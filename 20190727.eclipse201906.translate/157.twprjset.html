设置Web项目属性

您可以设置许多Web项目属性，这些属性会影响项目生命周期中的阶段及其对其他Web资源和Web开发过程的依赖性。在更新现有项目或将Web资源导入项目后，您可能需要验证或更新这些属性。

要查看或设置Web项目属性，请执行以下操作：

  1. 右键单击Web项目，然后从弹出菜单中选择“ 属性” 。
  2. 在“属性”对话框的左窗格中选择属性类型，以查看或更新Web项目中的属性。对于每个属性，您可以应用更改并修改默认设置。根据您是否具有静态或动态Web项目，可以使用以下属性类型：
    信息
    提供有关项目类型，位置和修改状态的一般信息。
    BeanInfo路径（仅限动态Web项目）
    如果在此项目上选择“启用Beaninfo内省”，则可以指定Java™Bean搜索路径的内容和顺序，以提供与项目相关的Bean信息。您可以添加，删除和重新排序（使用向上和向下按钮）用于包含Bean信息的包。
    建设者
    在构建顺序中添加，删除或重新排序外部工具。选择“ 新建”以在列表中创建新的外部工具时，将打开“新建外部工具”向导，以便您可以定义新的构建工具。
    Java EE（仅限动态Web项目）
    确定Web级别规范并修改上下文根和Web内容文件夹名称
    Java构建路径（仅限动态Web项目）
    使用“ 创建DynamicWeb项目”向导查看或更改有关使用“ 创建动态Web项目”向导创建项目时提供的Java构建路径的信息。每当重建Web项目时，创建或更新的任何类文件都将输出到指定的输出文件夹。
    注意：除非从WEB-INF / lib文件夹中删除实际的JAR文件，否则Java构建路径上的库条目将保留在那里。如果删除库路径条目但不删除JAR文件，则库条目将自动重新添加到路径中。
    Java编译器（仅限动态Web项目）
    定义是否使用工作区设置或项目设置来编译Java代码，包括“问题”，“样式”，“合规性”和“类文件”以及“构建路径”的设置。
    Javadoc位置（仅限动态Web项目）
    为项目可用的任何Javadoc定义默认位置（URL）。
    Java JAR依赖项（仅限动态Web项目）
    定义和重新排序项目的JAR依赖项。请注意，这些依赖项可能因Web项目与多个企业应用程序项目的关联而有所不同。
    JSP任务标签
    添加，编辑或删除JSP任务标记并分配标记优先级。这些“待办事项”项目显示在“任务”视图中。
    JSP编译
    指定在构建过程中何时应将JSP设置为compile
    JSP Fragment（仅限动态Web项目）
    为Web项目中的JSP片段文档定义默认编码，页面指令和内容类型值。
    链接验证/重构
    定义项目级链接管理和验证设置。这些设置会覆盖当前项目的Workbench Preference设置中的相应设置。
    项目参考
    引用工作台工作区中存在的其他Web项目。
    服务器
    选择用于为项目执行必要的开发，测试和发布功能的服务器实例。
    验证
    指定应为项目运行哪些验证程序，以及在将资源更新写入项目时是否应自动进行验证。可用的验证程序基于项目中的资源类型。默认情况下，在Web项目中保存资源时会自动运行验证程序。如果您不希望在保存这些资源时自动运行验证程序 ，请启用“ 覆盖验证首选项”选项并在保存对资源的更改时自动禁用“ 运行验证”选项。
    注意：您还可以通过禁用Workbench属性上的资源修改时自动执行构建来禁用所有项目的自动验证（和构建）。如果禁用自动构建，则可以通过选择“ 项目” >“ 重建项目”来手动运行构建（和验证）。
    网页内容设置
    为Web项目提供任何适用的Web内容设置。您可以指定文档类型，CSS配置文件和目标设备。
    Web项目功能
    选择要为Web项目启用的功能，例如页面模板支持。
  3. 完成Web项目设置的更新后，单击“ 确定”以保存并关闭“属性”对话框。
