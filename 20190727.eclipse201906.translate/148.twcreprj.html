创建动态Web项目

您可以在Web项目中为Web应用程序创建和维护资源。

与静态 Web项目不同，动态Web项目使您能够创建JavaServer Pages和servlet等资源。

要创建新的动态Web项目，请完成以下步骤：

  1. 打开Java EE透视图。
  2. 在Project Explorer中，右键单击Dynamic Web Projects，然后从上下文菜单中选择New > Dynamic Web Project 。“新建动态Web项目”向导启动。
  3. 按照项目向导提示进行操作。

一般信息

项目方面
facet表示Web项目中的一个功能单元。例如，动态Web模块方面可以将项目部署为动态Web模块。选择项目构面时，会在向导中显示项目构面的简要说明。请注意，在许多情况下，您可以通过右键单击构面并从弹出菜单中选择项目约束来查看项目构面的约束。
目标运行时
使用此字段定义新安装的运行时环境。运行时在构建时用于编译项目。
企业应用项目（EAR项目）
必须将新的或现有的企业应用程序项目（EAR项目）与新的Web项目相关联，以便于部署。如果要覆盖企业应用程序项目的默认设置，可以使用向导执行此操作。在向导结束时创建Web项目时，还会使用EAR项目字段中指定的名称创建新的Enterprise Application项目。请注意，默认值是附加了EAR的Web项目的名称（除非在打开向导时选择了ear项目。）
上下文根
上下文根是Web应用程序根目录，它是应用程序部署到Web服务器时的顶级目录。使用项目“属性”对话框创建项目后，可以更改上下文根，可以从项目的弹出菜单中访问该对话框。链接构建器还可以使用上下文根，以确保在移动和重命名项目内的文件时，链接仍可以发布。
