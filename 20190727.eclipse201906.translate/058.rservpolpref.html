WS-I合规性偏好

“服务策略首选项”页面允许您选择符合WS-I简单SOAP绑定配置文件（SSBP）和WS-I附加配置文件（AP）的级别。可以在“服务策略”首选项页面上找到它们。

设置WS-I首选项

WS-I Basic Profile概述了WSDL和Web服务协议（SOAP / HTTP）流量必须遵守的要求才能声明WS-I一致性。Web服务WS-I验证工具目前支持WS-I Simple SOAP Binding Profile 1.0（WS-I SSBP），它构建在WS-I Basic Profile和WS-I Attachments Profile 1.0（WS-I AP）之上。要查看规范，请参阅WS-I Web站点，在Deliverables下选择Basic Profile 。

有关WS-I的更多信息，请访问： http ： //www.ws-i.org/

可以通过打开“首选项”笔记本在工作区级别设置WS-I首选项，或者在项目级别通过右键单击项目并选择“首选项”来设置WS-I首选项。

您可以选择WS-I AP 1.0或WS-I BP 1.1 + SSBP 1.0配置文件，并选择要忽略，建议或要求合规性的Web服务工具。如果您尝试生成不符合规范的代码，Suggest将生成警告，而require将阻止您生成不符合规范的代码。

注意：如果在“任务”视图中列出WS-I不合规性时更改了WS-I合规性级别，则“任务”视图将不会自动更新。在从“任务”视图中删除警告之前，必须重新验证工作空间。
