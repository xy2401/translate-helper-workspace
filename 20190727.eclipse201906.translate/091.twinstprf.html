定义服务器运行时环境首选项

使用“首选项”页面，您可以指示工作台使用应用程序服务器的特定运行时环境来编译，测试或运行应用程序。

服务器运行时环境由以下定义组成：
  • 用于标识服务器运行时环境的名称
  • 指定服务器的供应商和版本级别的服务器类型
  • 服务器本地安装的目录路径
  • 在某些情况下，还包括对Java运行时环境的引用
如果服务器运行时环境引用服务器的存根目录，则只能针对存根服务器编译应用程序。但是，如果引用服务器的完整安装，则可以针对服务器的安装来编译，测试和运行应用程序。
要定义服务器运行时环境，请完成以下步骤：
  1. 从“窗口”菜单中选择“首选项” 。
  2. 在“首选项”窗口中，展开“ 服务器”文件夹，然后选择“ 运行时环境” 。 将显示“首选项”对话框的“服务器运行时环境”部分。
  3. 可选：单击“ 搜索”按钮。这会在本地目录中搜索服务器运行时环境。不支持在远程主机上搜索。
    1. 将打开“搜索运行时环境”对话框。在“ 文件夹”字段中，浏览或键入您希望工作台开始搜索服务器运行时环境的目录。单击确定 。
    2. 如果显示安装列表，请选择所需的目标服务器，然后在显示的对话框中单击“ 确定 ”;然后在Preferences对话框中单击OK 。如果未找到任何应用程序服务器的安装，请继续完成此任务的步骤。
  4. 单击添加 。 将打开“新建服务器运行时环境”向导。
    1. 在“ 选择运行时环境类型”列表下，选择服务器和版本级别作为目标服务器运行时环境。
      提示：在“ 选择运行时环境类型”标签下的文本字段中，可以使用关键字替换文本type filter text以过滤可用应用程序服务器列表。以下是过滤器关键字的示例：
      Apache
      过滤供应商为Apache的可用服务器列表
      WebSphere
      过滤名称为WebSphere的可用服务器列表
      ejb
      过滤支持ejb模块的可用服务器列表。
      web
      筛选支持Web模块的可用服务器列表
      5.0
      过滤版本为5.0或支持JEE 5规范级别的可用服务器列表。
      1.2
      筛选支持J2EE 1.2规范级别的可用服务器列表或可能具有1.2版本级别的任何服务器。
    2. 添加服务器运行时环境时，默认情况下会创建一个服务器并将其添加为Servers视图中的条目。如果只想添加服务器运行时环境而不在“服务器”视图中创建服务器，请清除“ 创建新的本地服务器”复选框。
    3. 单击下一步 。显示服务器面板。
    4. 对于“ 安装目录”或“ 位置”字段，键入或浏览到目标服务器的位置。
      提示：如果为目标服务器指定了错误位置，请使用工作台来组装模块，然后使用应用程序服务器来部署模块。
      如果选择目标服务器作为J2EE运行时库 （通用J2EE容器），请指定包含.jar文件的目录，例如用于Java™开发人员工具包（JDK）安装的\ bin \ lib目录。
    5. 单击完成 。 目标服务器名称和类型将添加到“服务器运行时环境”部分的表中。
    6. 在Preferences对话框中，单击OK 。

要更改可用目标服务器列表，请单击窗口>首选项>服务器>运行时环境以转到“首选项”对话框的“服务器运行时环境”部分。使用“ 添加” ，“ 编辑” ，“ 删除”或“ 搜索”选项可根据需要更改目标服务器列表。

要更改模块的目标服务器：
  1. 在Project Explorer视图中右键单击该模块，然后单击Properties> Project Facets 。Project Facets页面打开。
  2. 选择“ 运行时”选项卡。
  3. 在“ 运行时”表下，选择模块的目标服务器。
