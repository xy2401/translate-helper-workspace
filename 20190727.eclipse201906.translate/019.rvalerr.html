常见的验证错误和解决方案

此表列出了验证项目时可能遇到的常见错误消息。

消息前缀 信息 说明
应用程序客户端验证器
CHKJ1000 验证失败，因为应用程序客户端文件无效。确保部署描述符有效。 无法加载application-client.xml文件。无法从application-client.xml文件初始化项目元数据。
  1. 确保以下内容：
    • META-INF文件夹存在于应用程序客户端项目中
    • META-INF包含application-client.xml文件
    • META-INF在项目的类路径中。
  2. 验证application-client.xml文件的语法：在Navigator视图中，突出显示application-client.xml文件，右键单击，然后选择Validate XML file 。
  3. 如果1）和2）都可以，请关闭项目，重新打开项目，然后重建项目。项目元数据将刷新。
EAR验证器
CHKJ1001 EAR项目{0}无效。 无法加载application.xml文件。无法从application.xml文件初始化项目元数据。
  1. 确保以下内容：
    • META-INF文件夹存在于EAR项目中
    • META-INF包含application.xml
    • META-INF在项目的类路径中。
  2. 验证application.xml文件的语法：在Navigator视图中，突出显示application.xml文件，右键单击，然后选择Validate XML file 。
  3. 如果1）和2）都可以，请关闭项目，重新打开项目，然后重建项目。项目元数据将刷新。
EJB验证器
CHKJ2019 {0}密钥类必须在运行时可序列化。 EJB符合EJB规范。此消息表示可能会出现问题。当类型需要在运行时可序列化并且在编译时无法验证可序列化时，会出现警告。如果类型在运行时是基本类型，基本数组，远程对象，或者它实现了java.io，则类型是可序列化的。序列化。此消息标记java.lang。对象，它不能被禁用。您可以在编译时使对象可序列化，也可以忽略警告。
CHKJ2412 返回类型必须在运行时可序列化。
CHKJ2413 {0}的参数{1}必须在运行时可序列化。
CHKJ2102 必须定义查找器描述符或{0}类上的匹配自定义查找器方法。 每个finder方法必须存在finder描述符。
CHKJ2873 将此bean的数据源绑定迁移到CMP连接工厂绑定。  
CHKJ2874 将此EJB模块的默认数据源绑定迁移到默认的CMP连接工厂绑定。  
CHKJ2875E 每个包含此EJB模块的EAR文件中必须存在<ejb-client-jar> {0}。 如果在ejb-jar.xml中指定了<ejb-client-jar>则相应的EJB客户端项目必须包含主接口和远程接口以及客户端需要的任何其他类型。如果这些类型都包含在单个EJB项目中，请删除部署描述符中的<ejb-client-jar>行。否则，请确保EJB客户端项目存在，处于打开状态，并且是每个使用此EJB项目作为模块的EAR中的项目实用程序JAR。
CHKJ2905 EJB验证程序未运行，因为无法加载ejb-jar.xml。运行XML验证程序以获取更多信息。 CHKJ2905表示无法从ejb-jar.xml初始化项目的元数据。
  1. 确保以下内容：
    • EJB项目中存在META-INF文件夹
    • META-INF包含ejb-jar.xml
    • META-INF在项目的类路径中。
  2. 验证ejb-jar.xml文件的语法：在Navigator视图中，突出显示ejb-jar.xml文件，右键单击，然后选择Validate XML file 。
  3. 如果1）和2）都可以，请关闭项目，重新打开项目，然后重建项目。项目元数据将刷新。
JSP验证器
IWAW0482 没有有效的JspTranslator 项目存在路径问题; JSP Validator需要访问WAS运行时代码。如果IWAW0482E出现在所有Web项目中，请检查Variable或JRE路径：
  1. 检查全局首选项（“ 窗口”>“首选项”>“Java”>“已安装的JRE” ），并确保JRE的位置指向有效的JRE目录。
  2. 确保正确设置了类路径变量（ Window> Preferences> Java> Classpath Variables ）。
WAR验证器
CHKJ3008 WAR文件丢失或无效。 无法加载web.xml文件。无法从web.xml文件初始化项目元数据。
  1. 确保以下内容：
    • web-INF文件夹存在于Web项目中
    • WEB-INF包含web.xml文件
    • WEB-INF位于项目的类路径中。
  2. 验证web.xml文件的语法：在Navigator视图中，突出显示web.xml文件，右键单击，然后选择Validate XML file 。
  3. 如果1）和2）都可以，请关闭项目，重新打开项目，然后重建项目。项目元数据将刷新。
XML验证器
元素类型“ejb-jar”的内容不完整，必须匹配“（描述？，显示名称？，小图标？，大图标？，企业bean，汇编描述符？，ejb-client-jar） ？）”。 EJB 1.1和2.0规范要求EJB .jar文件中必须至少存在一个企业bean。在开发EJB .jar文件期间，此错误消息是正常的，在执行生产操作（例如导出或部署代码）之前，可以忽略此错误消息。在项目中定义至少一个企业bean。
