动态Web项目和应用程序

Web项目有两种类型：动态和静态 。除了静态资源（如图像和HTML文件）之外，动态Web项目还可以包含动态Java EE资源，如servlet，JSP文件，过滤器和关联的元数据。静态Web项目仅包含静态资源。创建Web项目时，可以包括级联样式表和JSP标记库（用于动态Web项目），以便您可以使用更丰富的项目资源开始开发。

动态Web项目始终嵌入在企业应用程序项目中。用于创建动态Web项目的向导还将创建企业应用程序（EAR）项目（如果该项目尚不存在）。该向导还将更新指定的Enterprise Application项目的application.xml部署描述符，以将Web项目定义为模块元素。如果要导入WAR文件而不是创建新的动态Web项目，则WAR导入向导要求您指定一个已经需要EAR项目的Web项目。

如果您只想创建一个静态的，基于内容的Web应用程序，它不包含动态文件（例如JSP文件或servlet），那么Java EE约定可能会带来额外的开销。在这种情况下，当您只需要最基本的Web项目时，您可能希望使用静态 Web项目类型（请参阅静态Web项目 ）。请注意，通过从“项目”菜单中选择“ 转换为动态Web项目” ，可以将静态Web项目转换为动态Web项目。

Java EE模型，更具体地说是Sun Microsystems Java™Servlet 2.3规范 ，定义了Web应用程序目录结构，该结构指定Web内容文件，类文件，类路径，部署描述符和支持元数据的位置。Web项目层次结构反映了从项目创建的Web应用程序的层次结构。在工作台中，您可以使用“ 新建Web项目”向导来创建新的Web项目。

注意：在Project Explorer视图中，Web项目将过滤到文件夹节点中，以自定义Web资源的显示，以便在开发期间轻松管理。有关筛选结构的信息，请参阅Project Explorer视图 。
Web部署描述符
标准Web应用程序部署描述符（ web.xml文件）。
的JavaSource
包含项目的类，bean和servlet的Java源代码。将这些资源添加到Web项目后，它们会自动编译，生成的文件将添加到WEB-INF / classes目录中。除非在创建WAR文件时指定了选项，否则源目录的内容不会打包在WAR文件中。
注意：虽然为该文件夹指定的默认名称是JavaSources，但您可以通过右键单击Project Explorer中的名称并单击Refactor > Rename来更改名称。
imported_classes文件夹
此文件夹可能在WAR导入期间创建，并包含没有附带源的类文件。imported_classes文件夹是Java类文件夹;也可以使用Web项目Java Build Path属性页面创建Java类文件夹。
WebContent文件夹
所有Web资源的强制位置，包括HTML，JSP，图形文件等。如果文件未放在此目录中（或此目录下的子目录结构中），则在服务器上执行应用程序时，这些文件将不可用。Web内容文件夹表示将部署到服务器的WAR文件的内容。不在Web内容文件夹下的任何文件都被视为开发时资源（例如.java文件，.sql文件和.mif文件），并且在项目进行单元测试或发布时不会部署。
注意：虽然为文件夹指定的默认名称是WebContent ，但您可以通过右键单击文件夹并选择Refactor Rename或从项目的Properties对话框的Web页面更改Project Explorer中的名称。在动态Web项目中，更改文件夹名称将更新Java构建输出目录。
META-INF
此目录包含MANIFEST.MF文件，该文件用于映射同一企业应用程序项目中其他项目中存在的从属JAR文件的类路径。此文件中的条目将更新运行时项目类路径和Java构建设置以包括引用的JAR文件。
主题
用于级联样式表和其他与样式相关的对象的建议目录。
WEB-INF
该目录基于Sun Microsystems Java Servlet 2.3规范 ，包含Web应用程序的支持Web资源，包括web.xml文件以及classes和lib目录。
/班
此目录用于servlet，实用程序类和Java编译器输出目录。应用程序类加载器使用此目录中的类来加载类。此目录中的文件夹将映射包名和类名，如：/ /WEB-INF/classes/com/mycorp/servlets/MyServlet.class 。

不要将任何.class文件直接放在此目录中。当Java编译器编译Java Resources目录中的Java源文件时，.class文件将自动放在此目录中。直接放在此目录中的任何文件在运行时都将被Java编译器删除。

/ lib目录
Web应用程序引用的支持JAR文件。放置在此目录中的.jar文件中的任何类都可用于您的Web应用程序
图书馆
Web应用程序引用的支持JAR文件。此文件夹镜像lib文件夹的内容。此外，Web库项目是“虚拟”JAR文件，它们实际上不在Web项目中，但与工作区中其他位置的Java项目相关联，包含在此文件夹中。导出应用程序的WAR文件时，它们与您的项目一起打包。
注意：除非从WEB-INF / lib文件夹中删除实际的JAR文件，否则Java构建路径上的库条目将保留在那里。如果删除库路径条目但不删除JAR文件，则库条目将自动重新添加到路径中。
