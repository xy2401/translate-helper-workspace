创建听众

侦听器向导可帮助您创建应用程序生命周期侦听器，方法是引导您完成创建过程，并为您提供可以使用的输出文件，或者可以修改以用于Web应用程序的输出文件。侦听器可以在符合Java EE的Web服务器上运行。

要创建侦听器，请完成以下步骤：

  1. 从Java EE角度，在Project Explorer视图中展开动态项目 。
  2. 右键单击Listener图标，然后从弹出菜单中选择New > Listener 。 将出现“ 创建侦听器”向导。
  3. 按照项目向导提示进行操作。
应用程序生命周期监听器
Sun Microsystems Java™Servlet 2.3规范定义了以下表示应用程序生命周期监听器的接口。您应该至少选择一个接口。
  • 的javax.servlet。ServletContextListener：此接口由javax.servlet包提供，可用于侦听ServletContext生命周期事件。此接口有两个方法：contextInitialized和contextDestroyed。当Web应用程序准备好为请求提供服务时调用contextInitialized方法，并在将要关闭servlet上下文时调用contextDestroyed方法。
  • 的javax.servlet。ServletContextAttributeListener：此接口由javax.servlet包提供。当任何属性添加到ServletContext或者更改或删除任何ServletContext的属性时，可以使用它来通知。这个inetface有三个方法： attributeAdded，attributeRemoved和attributeReplaced 。
  • 的javax.servlet。ServletRequestListener：此接口由javax.servlet包提供。当请求进入或超出Web组件的范围时，可以使用它来通知它。此接口有两个方法：requestInitialized和requestDestroyed。requestInitialized方法在即将进入每个Web应用程序中的第一个servlet或过滤器时被调用。requestDestroyed方法在退出最后一个servlet或链中的第一个过滤器时被调用。
  • 的javax.servlet。ServletRequestAttributeListener：此接口由javax.servlet包提供，可用于在更改任何请求属性时收到通知。该接口有三个方法：attributeAdded，attributeRemoved和attributeReplaced。将新属性添加到servlet请求时，将调用attributteAdded方法。从servlet请求中删除现有属性时，将调用attributeRemoved方法。在servlet请求上替换属性时，将调用attributeReplaced方法。
  • javax.servlet.http包。HttpSessionListener：此接口由javax.servlet.http包提供，可用于侦听会话生命周期事件。此接口有两个方法：sessionCreated和sessionDestroyed。在创建HttpSession对象时调用sessionCreated方法。当HttpSession对象失效时，将调用sessionDestroyed方法。
  • javax.servlet.http包。HttpSessionActivationListener：此接口由javax.servlet.http包提供。它可用于监听容器事件，通知会话将被钝化或激活。此接口有两个方法：sessionDidActivate和sessionWillPasivate。刚刚激活会话时调用sessionDidActivate，并在会话即将被钝化时调用sessionWillPasivate 。
  • javax.servlet.http包。HttpSessionAttributeListener：此接口由javax.servlet.http包提供，可用于侦听与会话属性相关的事件。该接口有三个方法：attributeAdded，attributeRemoved和attributeReplaced。将属性添加到HttpSession对象时，将调用attributeAdded方法。分别删除或替换HttpSession属性时，将调用attributeRemoved和attributeReplaced方法。
  • javax.servlet.http包。HttpSessionBindingListener：此接口由javax.servlet.http包提供，可用于在对象绑定到会话或从会话解除绑定时收到通知。此接口有两个方法：valueBound和valueUnbound。当对象绑定到会话并标识会话时，将调用valueBound方法。当对象从会话解除绑定并标识会话时，将调用valueUnbound方法。
修饰符
Sun Microsystems Java Servlet 2.3规范声明Listener类必须是公共的而不是抽象的 。因此，您无法更改这些修饰符。唯一可用于更改的是最终修饰符。
接口
您在向导上一页中选择的接口在Interfaces字段中作为默认值提供。要添加其他接口，请单击此字段旁边的“ 添加”按钮。
界面选择对话框
如果选择向servlet添加接口，则会出现此对话框。在键入要添加的接口的名称时，“ 匹配项目”列表框中列出的可用接口列表会动态更新，以仅显示与该模式匹配的接口。选择一个界面以查看限定符，然后在完成后单击“ 确定” 。
