听众

过滤器是Sun Microsystems Java Servlet 2.3规范中的新增功能。它们使您能够拦截请求在到达资源之前拦截请求。换句话说，过滤器允许您在将HttpServletRequest和HttpServletResponse对象传递给servlet之前访问它们。

过滤器非常有用。例如，您可以编写一个记录所有传入请求的过滤器，并记录发出请求的计算机的IP地址。您还可以使用过滤器作为加密和解密设备。其他用途包括用户身份验证，数据压缩，用户输入验证等。

您还可以在链中放置一组过滤器。将首先调用链中的第一个过滤器，然后将控制权传递给第二个过滤器，依此类推。过滤器链确保您可以编写执行特定任务的过滤器，但在另一个过滤器中添加一些功能。

在编写过滤器时，您基本上处理javax.servlet包中的以下三个接口：
  • 的javax.servlet。过滤器：表示过滤器的生命周期。
  • 的javax.servlet。FilterConfig：表示过滤器的配置。
  • 的javax.servlet。FilterChain： servlet容器向开发人员提供的对象，用于查看已过滤的资源请求的调用链。

有关过滤器的更多信息，请参阅java.sun.com/products/servlet/download.html上的Sun Microsystems Java™Servlet 2.3规范 。
