创建应用程序客户端项目

您可以使用向导创建新的应用程序客户端项目，并将其添加到新的或现有的企业应用程序项目中。

应用程序客户端项目包含在联网客户端系统上运行的应用程序客户端模块和程序所需的资源。应用程序客户端项目部署为JAR文件。

与其他类型的项目一样，应用程序客户端项目可以包含一个或多个项目构面，它们代表项目中的功能单元。新的应用程序客户端项目应具有Application Client模块构面。根据您要使用项目的内容，您可能希望为项目启用其他方面。

要创建J2EE应用程序客户机项目，请完成以下步骤：

  1. 在Java™EE透视图中，单击“ 文件” >“ 新建” >“ 项目” 。
  2. 在New Project Wizard中，选择Java EE > Application Client Project ，然后单击Next 。
  3. 在“ 项目名称”字段中，键入应用程序客户端项目的名称。
  4. 可选： 要更改默认项目位置，请清除“ 项目内容 ”下的“ 使用默认值”复选框，然后使用 “ 浏览”按钮选择新位置。如果指定另一个项目已在使用的非默认项目位置，则项目创建将失败。
    注意：如果键入新的EAR项目名称，则将根据正在创建的项目的版本，在默认位置创建具有最低兼容Java EE版本的EAR项目。如果要为企业应用程序指定其他版本或不同位置，则必须使用“新建企业应用程序项目”向导。
  5. 可选： 要将新项目添加到企业应用程序项目，请选中“ 将项目添加到EAR”复选框，然后在“ EAR项目名称”列表中选择一个项目。如果选择将项目添加到现有EAR项目，则目标运行时字段将被禁用，因为新项目的目标运行时将与EAR项目的目标运行时相同。
  6. 在“ 目标运行时”字段中，选择项目的目标运行时。
  7. 可选： 要为项目使用预定义配置，请在“ 配置”列表中选择一个配置。您可以单击“ 修改”按钮执行以下操作：
    • 自定义项目方面。选中希望项目具有的构面旁边的复选框，然后为每个构面选择版本号。选择构面名称可在“ 详细信息”选项卡中查看有关该构面的更多信息。您还可以从“ 配置”列表中选择预设的构面组合。
    • 限制您的项目与一个或多个运行时兼容。单击“ 显示运行时”选项卡，然后选择希望项目与之兼容的运行时。
    单击下一步 。
  8. 在“ 源文件夹”字段中，输入要用于源代码的文件夹的名称。
  9. 可选： 要为模块创建默认类，请选中“ 创建默认主类”复选框。
  10. 单击完成 。
