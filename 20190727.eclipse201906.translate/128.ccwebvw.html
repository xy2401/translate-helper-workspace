Project Explorer视图和Web开发

Project Explorer视图提供以下显着特征：
  • 你可以拖放从Windows®资源管理器或桌面文件到导航视图。
  • 从“导航器”视图“ 菜单”按钮中选择“ 过滤器” ，即可支持视图过可以按名称，项目类型或内容类型过滤资源。默认情况下会过滤掉以句点开头的文件。
  • 状态行显示所选资源的完整路径。
  • 将.java文件从Navigator视图拖动到JSP文件中将插入usebean标记，这与将.class文件拖入JSP文件时显示的行为相同。
  • 资源上的错误和警告（包括Java™，HTML / JSP和Links Builder错误和警告）以红色错误表示或黄色警告带有错误的资源旁边，以及直到项目的父容器。这适用于所有项目类型，而不仅适用于Web项目。
  • 项目弹出菜单中“ 新建”级联菜单中提供的项目是上下文相关的。所有菜单都有Project和Other选项。

组织Project Explorer视图

Project Explorer视图显示所有Web项目的自定义视图。以下是项目节点下方显示的一些值得注意的顶级对象（基于默认文件夹名称）。

Web内容文件夹 - 此文件夹包含要发布到服务器的项目。默认情况下，此文件夹将命名为WebContent，用于新创建的静态和动态Web项目。
注意：您可以在创建向导Web构面页面中更改名称。
  • META-INF - 此目录包含MANIFEST.MF文件，该文件用于映射同一企业应用程序项目中其他项目中存在的从属JAR文件的类路径。此文件中的条目将更新运行时项目类路径和Java构建设置以包括引用的JAR文件。
  • WEB-INF - 保留Web应用程序Web资源的目录（例如：.xmi文件，.xml文件和web.xml）。
