定义和使用注释

您可以使用@Interface注释来定义自己的注释定义。

定义自己的注释

使用@Interface批注定义自己的批注定义：
  • 注释定义类似于接口定义
  • 注释方法声明既没有参数也没有throws子句，并返回以下元素之一：
    • 原语
    • 串
    • 类
    • 枚举
    • 以上类型的数组
  • 方法可能具有默认值
public @interface CreatedBy{
     String name();
     String date();
     boolean contractor() default false;
}          
@CreatedBy(name = "Mary Smith",date="02/02/2008");
public class MyClass{....}
元注释 ：元注释（注释注释）提供有关如何使用注释的附加信息：
  • @目标
    • 限制注释的使用
    • 单个参数必须来自Enum ElementType
      • {TYPE，FIELD，METHOD，PARAMETER，CONSTRUCTOR，LOCAL_VARIABLE，ANNOTATION_TYPE}
  • @保留
    • 指示将保留注释信息的位置
    • 单个参数必须来自Enum RetentionPolicy
      • {SOURCE，CLASS，RUNTIME}
  • @Documented
    • 标记应该包含在Javadoc中的注释
  • @遗传
    • 类型注释的标记，由子类型继承
其他内置注释 ：
  • @Overrides
    • 应用于一种方法
    • 指示如果方法实际上不覆盖超类方法，编译器应生成错误。
  • @Deprecated
    • 应用于一种方法
    • 指示在外部使用方法时编译器应生成警告
  • @SuppressWarnings
    • 适用于类型或方法
    • 表示编译器应该抑制该元素和所有子元素的警告
      @Deprecated
public void oldMethod() {...}

@ SupressWarnings
public void yesIknowIuseDeprecatedMethods() {...}

  • 注释的类型
    Java EE 6和Java EE 5定义了许多类型或注释组，在许多Java规范请求（JSR）中定义。
