创建服务器

您可以创建服务器以标识要用于测试项目资源的运行时环境。创建服务器一词定义了创建从工作台到应用服务器的现有安装的指针 。

要创建服务器：
  1. 选择以下任一选项：
    • 在Servers视图（ Window > Show View > Servers ）中，右键单击并选择New > Server 。
    • 在菜单栏中，单击“ 文件”>“新建”>“其他” 。展开Server文件夹，选择Server ，然后单击Next 。
  2. 将打开“新建服务器”向导。此向导定义了一个新服务器，其中包含指向特定运行时环境以进行本地或远程测试或发布到应用程序服务器所需的信息。
  3. 在“ 选择服务器类型”列表中，选择要发布或测试资源的服务器或测试环境的类型。
    1. （可选）在“ 选择服务器类型”标签下的文本字段中，可以使用关键字替换文本type filter text以过滤可用应用程序服务器列表。 以下是过滤器关键字的示例：
      Apache
      过滤供应商为Apache的可用服务器列表
      WebSphere
      过滤名称为WebSphere的可用服务器列表
      ejb
      过滤支持ejb模块的可用服务器列表。
      web
      筛选支持Web模块的可用服务器列表
      5.0
      过滤版本为5.0或支持JEE 5规范级别的可用服务器列表。
      1.2
      筛选支持J2EE 1.2规范级别的可用服务器列表或可能具有1.2版本级别的任何服务器。
    2. 有关从服务器类型列表中选择的服务器的简要说明。
  4. 在服务器的主机名字段中，您可以提供运行服务器的主机的完全限定DNS名称或IP地址。默认情况下，此字段预先填充了默认地址： localhost 。
  5. （可选）在“ 服务器名称”字段中，键入标签以在“服务器”视图中标识此服务器条目。默认情况下，此字段使用以下命名约定完成： 主机名的 服务器类型 ，例如Tomcat v6.0 Server at localhost 。
  6. 当您从运行时环境首选项页面（ 窗口 > 首选项 > 服务器 > 运行时环境 ）中的服务器类型列表中选择包含运行时环境条目的服务器时，可以使用服务器运行时环境列表。 在“ 服务器运行时环境”列表中，选择应用程序服务器的运行时环境，以便编译，测试或运行应用程序。要修改服务器运行时环境，请选择“ 配置运行时环境”链接。要添加新的服务器运行时环境，请选择“ 添加”链接。有关运行时环境的更多详细信息，请参阅本主题底部的“ 定义服务器运行时环境首选项”链接。

    如果服务器运行时环境列表不可用，请继续执行“新建服务器”向导的下一页，它将提示您输入服务器的安装目录。完成此“新建服务器”向导后，工作台会在“运行时环境”首选项页面中自动添加运行时环境条目。

  7. 单击“ 下一步”，然后按照向导中的说明指定要创建的服务器的详细信息。 请参阅本主题末尾的链接，因为在继续完成此向导以创建您在服务器类型列表中选择的服务器时，可能存在可用的文档。
  8. 完成“新建服务器”向导后，单击“ 完成” ，新服务器将显示在“服务器”视图中。
