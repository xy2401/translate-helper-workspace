开发Java EE应用程序

Java™EE 5编程模型简化了创建Java应用程序的过程。

在Java EE规范中，编程要求已经简化，XML部署描述符是可选的。相反，您可以使用Java注释指定组装和部署的许多详细信息。Java EE将在许多情况下提供默认值，因此不需要明确指定这些值。

代码验证，内容辅助，快速修复和重构简化了代码的使用。代码验证器检查您的项目是否有错误。找到错误后，您可以在产品工作台的“问题”视图中双击它以转到错误位置。对于某些错误类型，您可以使用“快速修复”自动更正错误。对于Java源代码和Java注释，您可以依靠内容辅助来简化编程任务。重构源代码时，工具会自动更新关联的元数据。

有关Java EE的其他信息，请参阅官方规范：
  • Java EE 5 ： JSR 244：Java平台企业版5（Java EE 5）规范
  • Java EE 6 ： JSR 316：JavaTM平台，企业版6（Java EE 6）规范

  • Java EE 5：概述
    使用Java平台企业版（Java EE）体系结构，您可以构建分布式Web和企业应用程序。此体系结构可帮助您专注于演示和应用程序问题，而不是系统问题。
  • Java EE 5和Java EE 6支持注释
    Java EE 5和Java EE 6平台开发的目标是最小化您必须创建和维护的工件数量，从而简化开发过程。Java EE 5和Java EE 6支持在源代码中注入注释，以便您可以在源代码中嵌入资源，依赖项，服务和生命周期通知，而无需在其他地方维护这些工件。
