Java EE 5：概述

使用Java™Platform，Enterprise Edition（Java EE）体系结构，您可以构建分布式Web和企业应用程序。此体系结构可帮助您专注于演示和应用程序问题，而不是系统问题。

您可以使用Java EE 5工具和功能来创建围绕具有不同目的的模块构建的应用程序，例如Web站点和Enterprise JavaBeans™（EJB）应用程序。使用EJB 3.0组件时，可以创建具有事务支持的分布式安全应用程序。在开发访问持久数据的应用程序时，可以使用新的Java Persistence API（JPA）。该标准简化了持久性实体的创建和使用，以及添加新功能。对于开发表示逻辑，您可以使用JavaServer Pages（JSP）或JavaServer Faces（JSF）等技术。

使用Java EE 5平台企业版（Java EE），您可以比以前的版本更快速，更方便地开发应用程序。Java EE 5平台取代了Java 2 Enterprise Edition（J2EE）1.4版。产品工具支持这两个版本。Java EE 5显着增强了易用性
  • 缩短开发时间
  • 降低了应用复杂性
  • 提高应用性能
Java EE 5提供了简化的编程模型，包括以下工具：
  • 带注释的内联配置，使部署描述符现在可选
  • 依赖注入，隐藏资源创建和从应用程序代码查找
  • Java持久性API（JPA）允许在没有显式SQL或JDBC的情况下进行数据管理
  • 将普通旧Java对象（POJO）用于Enterprise JavaBeans和Web服务
Java EE 5为企业应用程序提供了简化的打包规则：
  • Web应用程序我们。WAR文件
  • 资源适配器使用。RAR文件
  • 企业应用使用。EAR文件
  • lib目录包含共享。JAR文件
  • 一个 。带有Main-Class JAR文件意味着应用程序客户端
  • 一个 。带有@Stateless注释的JAR文件意味着EJB应用程序
  • 许多简单的应用程序不再需要部署描述符，包括
    • EJB应用程序（。JAR文件）
    • 仅使用JSP技术的Web应用程序
    • 应用客户
    • 企业应用程序（。EAR文件）
Java EE 5使用依赖注入提供简化的资源访问：
  • 在依赖注入模式中，外部实体自动提供对象的依赖关系。
    • 该对象不需要显式请求这些资源
  • 在Java EE 5中，依赖项注入可以应用于组件所需的所有资源
    • 应用程序代码隐藏了资源的创建和查找
  • 可以在整个Java EE 5技术中应用依赖注入：
    • EJB容器
    • Web容器
    • 客户端
    • 网页服务
