创建企业应用程序项目

使用向导创建企业应用程序项目允许您在项目中打包许多Web应用程序和模块，并将这些模块部署为J2EE企业应用程序。

企业应用程序项目包含对企业应用程序所需资源的引用，并且可以包含Web模块，JAR文件，连接器模块，EJB模块和应用程序客户端模块的组合。企业应用程序项目以EAR文件的形式部署，因此有时称为EAR项目。企业应用程序项目中的模块映射到其他J2EE项目。映射信息存储在企业应用程序项目中的元数据文件中。元数据文件用于将项目导出到EAR文件以及在服务器上运行项目。

与其他类型的项目一样，企业应用程序项目可以包含一个或多个项目构面，它们代表项目中的功能单元。要部署为EAR文件，新项目必须具有EAR方面。根据您要使用项目的内容，您可能希望为项目启用其他方面。

创建企业应用程序项目时，建议您不要为其指定包含双字节字符集（DBCS）字符的名称。

要创建J2EE企业应用程序项目，请完成以下步骤：

  1. 在Java™EE透视图中，单击“ 文件” >“ 新建” >“ 项目” 。
  2. 在New Project Wizard中，选择J2EE > Enterprise Application Project ，然后单击Next 。
  3. 在“ 项目名称”字段中，键入新项目的名称。
  4. 可选： 要更改默认项目位置，请清除“ 项目内容 ”下的“ 使用默认值”复选框，然后使用 “ 浏览”按钮选择新位置。
  5. 在“ 目标运行时”字段中，选择项目的目标运行时。 您可以单击“ 新建”按钮为项目创建要使用的新运行时。
  6. 可选： 要为项目使用预定义配置，请在“ 配置”列表中选择一个配置。您可以单击“ 修改”按钮执行以下操作：
    • 自定义项目方面。选中希望项目具有的构面旁边的复选框，然后为每个构面选择版本号。选择构面名称可在“ 详细信息”选项卡中查看有关该构面的更多信息。您还可以从“ 配置”列表中选择预设的构面组合。
    • 限制您的项目与一个或多个运行时兼容。单击“ 显示运行时”选项卡，然后选择希望项目与之兼容的运行时。
    单击下一步 。
  7. 在要添加到向导的EAR页面的J2EE模块上，选择要添加到新企业应用程序项目的现有模块。
  8. 可选： 要创建要添加到项目的新模块 ，请单击“ 新建模块”按钮以执行以下操作之一：
    选项 描述
    创建一个模块 清除“ 创建默认模块”复选框，选择要创建的模块类型，单击“ 下一步” ，然后按照该项目类型的“新建项目”向导进行操作。
    创建两个或更多模块 选中“ 创建默认模块”复选框，选中要创建的每种项目类型的复选框，然后单击“ 完成” 。
    您可以输入每个模块的名称。这些模块中的每一个都将具有该类型项目的默认设置，并且它们将具有与新企业应用程序相同的服务器目标。
  9. 单击完成 。
