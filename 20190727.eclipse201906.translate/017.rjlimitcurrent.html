J2EE开发工具的局限性

本主题概述了当前已知的J2EE工具限制和限制。

企业应用程序中的JAR URI不支持的空间

企业应用程序中的模块或实用程序JAR文件的URI不支持空格。JAR文件或模块中MANIFEST.MF文件的“Class-Path：”属性是企业应用程序中以空格分隔的相对路径列表。如果引用的JAR文件的URI包含空格，则JAR文件将无法引用EAR中的另一个JAR文件。

企业应用程序项目名称不应包含DBCS字符

创建企业应用程序项目时，建议您不要为其指定包含双字节字符集（DBCS）字符的名称。

删除对Utility JAR文件的依赖关系时，Java™构建路径更新

删除对Utility JAR的依赖关系时，仅当依赖JAR仍由EAR项目引用时，才会从Java构建路径中删除相应的Java项目。例如，假设您创建了一个J2EE 1.3 Web项目和EAR以及JUnit Java Example项目。接下来，将JUnit项目添加为EAR中的Utility JAR，然后将JUnit添加为Web项目的Java JAR依赖项。如果您希望删除JUnit和Web项目之间的依赖关系，请首先从Web项目中删除Java JAR依赖项，然后从EAR中删除Utility JAR。按照此顺序确保此操作正常。

Java JAR Dependencies页面无法更新Java构建路径

Java JAR依赖项页面未与项目属性对话框中的Java构建路径页面同步。因此，在一个中应用的更改可能不会在同一对话会话中反映在另一个中。还有一些情况，当单击“ 确定”按钮或在“ 确定”按钮之前单击“ 应用”按钮时，在页面之间来回切换将导致从一个更新以从另一个更新取消更新。通常，这看起来好像添加了JAR依赖项，但项目未添加到Java构建路径中。解决方法是重新打开属性对话框，切换到JAR依赖关系页面，清除并重新选择依赖的JAR文件，然后单击“ 确定” 。

为新J2EE项目使用非默认项目位置时出现“项目描述无效”错误

创建新的J2EE项目（包括Java，企业应用程序，动态Web，EJB，应用程序客户端和连接器项目）时，不能使用工作台中另一个项目已使用的项目位置。如果选择另一个项目使用的项目位置，向导将显示“无效的项目描述”错误对话框或消息。如果在收到此消息后，通过单击“浏览”按钮选择有效的项目位置，项目创建仍将无法完成。解决方法是单击“取消”并重新打开项目创建向导。
