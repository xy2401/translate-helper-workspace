Java EE 5和Java EE 6支持注释

Java™EE 5和Java EE 6平台开发的目标是最大限度地减少必须创建和维护的工件数量，从而简化开发过程。Java EE 5和Java EE 6支持在源代码中注入注释，以便您可以在源代码中嵌入资源，依赖项，服务和生命周期通知，而无需在其他地方维护这些工件。

注释是一个修饰符或元数据标记，它为Java类，接口，构造函数，方法，字段，参数和局部变量提供附加数据。注释替换了样板代码，某些应用程序所需的通用代码。例如，注释可以替换Web服务所需的配对接口和实现。注释还可以替换程序所需的其他文件，这些文件是单独维护的。通过使用注释，不再需要此单独的文件。例如，注释可以取代对JavaBeans™单独维护的部署描述符的需求。

注释
  • 为大多数目的替换描述符
  • 不需要标记接口（如java.rmi。远程）
  • 允许应用程序设置在它们影响的组件中可见
Java EE 5为以下任务提供注释，其中包括：
  • 开发Enterprise JavaBean应用程序
  • 定义和使用Web服务
  • 将Java技术类映射到XML
  • 将Java技术类映射到数据库
  • 将方法映射到操作
  • 指定外部依赖项
  • 指定部署信息，包括安全属性
Java EE 5定义了许多可以注入源代码的注释。要声明注释，只需在关键字前面加上“at”符号（@）即可。
package com.ibm.counter;

import javax.ejb.Stateless;

@Stateless

public class CounterBean {

}

有关Java EE 5和Java EE 6支持的注释类别的更多信息，请参阅注释类型 。
