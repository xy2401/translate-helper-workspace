J2EE验证器

此表列出了可用于不同项目类型的验证器，并提供了每个验证器的简要说明。

验证器名称 描述
应用程序客户验证器 Application Client Validator验证以下Application Client项目资源：
  • 部署描述符（application-client.xml）
  • EJB引用
  • 资源参考
连接器验证器 Connector验证器检查连接器项目中的无效J2EE规范级别。
DTD验证器 DTD验证器确定DTD的当前状态是否在语义上有效。根据W3C Web站点的XML规范可扩展标记语言（XML）1.0验证XML文件。同样，DTD验证器检查错误，例如对实体和不存在的元素的引用。
EAR验证器 EAR Validator验证以下内容：
  • EAR部署描述符（application.xml）
  • 企业应用程序项目中所有模块项目的EJB引用
  • 安全角色
  • 资源参考
  • 清除所有包含或引用的模块和实用程序JAR文件的文件
  • 企业应用程序项目与任何实用程序和模块项目之间的目标服
  • 企业应用程序中定义的每个模块的项目的存在

请注意，EAR Validator仅确保模块项目相对于企业应用程序项目的有效性和依赖性。
EJB验证器 EJB Validator验证EJB项目中包含的企业bean是否符合Sun Enterprise JavaBeans™规范（1.1,2.0和2.1），具体取决于bean的级别。不支持EJB 1.0规范的代码验证。具体来说，EJB Validator验证以下资源：
  • 作为企业bean成员的Java™.class文件（home接口，远程接口，企业bean类，如果bean是实体bean，则是密钥类）
  • ejb-jar.xml中
EL语法验证器  
HTML语法验证器 HTML语法验证器在以下Web项目资源中验证HTML基本语法和HTML DTD合规性：
  • HTML文件
  • JSP文件
JSP语法验证器 JSP语法验证器通过将项目中的JSP文件转换为相应的Java代码，然后检查Java代码是否存在编译错误来验证项目中的JSP文件。
战争验证员 War Validator验证以下Web项目资源：
  • 部署描述符（web.xml）
  • Servlet的
  • 安全角色
  • Servlet和servlet映射
  • EJB引用
WSDL验证器 WSDL验证器检查WSDL文件中的以下内容：
  • XML语法
  • <types>部分中的XML Schema类型
  • WSDL中各种构造的引用完整性
验证器还包括一个扩展点，允许其他验证器插入WSDL验证，以提供对WSDL文件的额外验证。通过这种机制，通过针对WS-I配置文件验证WSDL文件来检查互操作性。
WS-I消息验证器 WS-I消息验证器根据WS-I配置文件检查SOAP消息。用户可以使用TCP / IP Monitor捕获和验证SOAP消息。验证程序检查保存为项目资源的消息日志（.wsimsg）。该日志符合WS-I指定的格式。
XML Schema Validator XML模式验证器确定XML模式文件的当前状态是否在语义上有效。XML模式根据W3C Web站点的XML Schema规范XML Schema Part 1：Structures进行验证。
XML验证器 XML验证器可确保XML文件格式正确。它还验证XML文件是否有效 - 也就是说，它遵循与XML文件关联的DTD或XML模式中建立的约束。
