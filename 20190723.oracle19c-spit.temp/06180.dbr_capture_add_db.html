Description of the illustration dbr_capture_add_db.gif

Three input fields appear below the title of the Add page. The first is Capture Name, which has a value of "NewEmpCapture." The second field is Target Database, which is read-only, and has a value of "database." The third field is Database Version, which has a value of "12.1.0.1.0." To the right of the Capture Name, Database Target, and Database Version fields is the Capture Description text field, which is blank.

Below these fields are two sections: Database Credential on the left, and Database Host Credential on the right. The Database Credential section has the text "Please provide credential to access database." For the Credential field, the Preferred option is selected. The Preferred Credential Name list is blank. The Credential Details field includes the text "Default preferred credentials are not set."

The Database Host Credential section has the text "Please provide credential to access database-name." For the Credential field, the Preferred option is selected. The Preferred Credential Name list is blank. The Credential Details field includes the text "Default preferred credentials are not set.".

Below the Database Host Credential section is a Database Capture Intermediate Storage Root Location input field, which has a value of "/scratch/destination_dir." To the right of this field is the Browse button. Below the field is the Information text "This root directory is used to temporarily store the captured workload while the capture is in progress." Under the Information text is the Use intermediate storage location as final storage location option, which is enabled.

OK and Cancel buttons appear in the lower right corner of the pop-up.

End of description.
