5.13 RC_BACKUP_DATAFILE_SUMMARY

RC_BACKUP_DATAFILE_SUMMARY provides summary information about available backups of data files.

This view is primarily intended to be used internally by Enterprise Manager.

Column Data Type Description

DB_KEY

NUMBER

The primary key for this database in the recovery catalog. Use this column to join with almost any other catalog view.

DB_NAME

VARCHAR2(8)

The DB_NAME of the database incarnation to which this record belongs.

NUM_FILES_BACKED

NUMBER

Number of data files backed up for this value of DB_KEY and DB_NAME.

NUM_DISTINCT_FILES_BACKED

NUMBER

Number of distinct files backed up for this value of DB_KEY and DB_NAME.

NUM_DISTINCT_TS_BACKED

NUMBER

Number of distinct tablespaces backed up for this value of DB_KEY and DB_NAME.

MIN_CHECKPOINT_CHANGE#

NUMBER

Minimum checkpoint of any data file backed up for this value of DB_KEY and DB_NAME.

MAX_CHECKPOINT_CHANGE#

NUMBER

Maximum checkpoint change number of any data file backed up for this value of DB_KEY and DB_NAME.

MIN_CHECKPOINT_TIME

DATE

Minimum checkpoint time for any data file backed up for this value of DB_KEY and DB_NAME.

MAX_CHECKPOINT_TIME

DATE

Maximum checkpoint time for any data file backed up for this value of DB_KEY and DB_NAME.

INPUT_BYTES

NUMBER

Total input bytes read for all files backed up for this value of DB_KEY and DB_NAME.

OUTPUT_BYTES

NUMBER

Sum of sizes of all output pieces generated by backups for this value of DB_KEY and DB_NAME.

COMPRESSION_RATIO

NUMBER

Compression ratio across all backups.

INPUT_BYTES_DISPLAY

VARCHAR2(32K)

Same value as INPUT_BYTES, but converted to a user-displayable format, for example, 798.01M or 5.25G.

OUTPUT_BYTES_DISPLAY

VARCHAR2(32K)

Same value as OUTPUT_BYTES, but converted to a user-displayable format, for example, 798.01M or 5.25G.
