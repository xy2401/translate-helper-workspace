版本8.7.0的发行说明

新特点和发展

使用单元格规范检查文本的更改

  • 带有单元格规格的表格操作检查文本现在正在进一步搜索。这意味着如果找到的第一个值不匹配，但是还有另一行，列组合具有指定值，则还将检查此值。这仅适用于行。我们仍然不支持重复的表头名称。

嵌入式AUTAgent for API

  • 现在可以在API中启动AUTAgent。因此，您不需要任何Jubula安装来运行测试。有关更多信息，请参阅开发人员手册

从AUTAgent获取ComponentIdentifier

  • 组件名称和编码的ComponentIdentifier可以轻松地从外部或嵌入式API AUTAgent获取。可以通过AUTAgent上下文菜单启动简单的组件收集模式。只有在没有ITE / testexec / API连接到代理时才可以执行此操作。有关更多信息，请参阅开发人员手册

可以禁用鼠标移动策略

  • 现在可以使用环境变量TEST_DISABLE_MOUSE_MOVEMENT=true禁用鼠标移动策略。但请注意，它可能会破坏测试，因为某些事件不会因鼠标移动而触发。

已知问题和其他信息

HTML支持将在下一版本中删除

  • 由于浏览器领域的快速发展，我们决定在下一版本中放弃对HTML Toolkit的支持。我们建议您直接使用Selenium。

ALM支持将在下一版本中删除

  • 测试执行后向ALM系统（如Bugzilla）的报告将在下一个版本中删除。

Java 9及更高版本

  • 必须使用jvm参数'--add-opens = javafx.controls / javafx.scene.control.skin = ALL-UNNAMED-illegal-access = permit'启动Java 9及更高版本的应用程序。这是必要的，因为我们使用内部信息来获取组件的某些位置信息。

通过AUTAgent进行对象映射（OM）

  • 如果通过AUTAgent上下文菜单使用OM，则存在无法加载扩展的问题。如果您使用嵌入式代理API，则可以将新的Testerclasses注册到ToolkitInformation，然后启动Embedded代理。

在Windows中执行外部命令

  • 有一个问题，有时我们无法杀死通过执行外部命令启动的已启动进程。

新特点和发展

  • 我们已将JavaFX和SWT树表操作移至具体工具箱。因此，不推荐使用unbound_modules_javafx和unbound_modules_swt项目中的相关模块。请使用项目unbound_modules_concrete中的对应模块替换这些模块。另请注意，某些树表操作仍仅针对JavaFX工具包实现（例如，检查列中值的存在性）。如果在SWT树表上执行，这些操作现在导致所选工具包错误不支持Action 。

改进的替换测试用例引用操作

  • 现在可以在“替换测试用例参考”操作（非编辑器版本）中为不匹配的参数选择默认值。这些默认值不能包含引用，但所有其他类型都可用（例如变量和函数）。

双击Test Case Editor根节点

  • 为了帮助在编辑器之间导航，我们添加了一个新功能：当双击测试用例编辑器的根节点时，如果打开一个引用给定测试用例的单个编辑器，则激活该编辑器。如果有多个此类编辑器打开，您可以选择一个在对话框中激活。如果没有打开这样的编辑器，您可以选择引用测试用例的测试用例或测试套件，并打开其编辑器。

  • 现在可以尝试跳转到通过getCentralTestDataSetValue函数由参数引用的数据集。如果行和单元格可以从函数的参数中识别出来，那么它们也会在数据集视图中被选中。

版本8.4.0的发行说明

新特点和发展

条件语句和循环

  • 我们引入了改变测试执行流程的新节点：条件语句和循环。

  • If-Then-Else节点允许您指定一个或多个测试用例或测试步骤作为条件。根据条件节点的结果，将输入“then”或“else”分支，并执行包含节点。这种情况也可以否定。

  • While-Do-node和Do-While-Node将重复“do”-branch中的测试节点，直到不再满足条件为止。与While-Do-node相反，Do-While-node在每次运行“do”-branch后检查条件。因此，Do-While-node将始终至少运行一次。

  • Repeat-node重复“do”-branch中的节点指定的次数。

调用外部方法

  • 现在可以通过Reflection组件调用外部（AUT独立）方法。外部JAR文件必须放在<installation-directory> / ite / externaljars目录中 。

组件名称

  • 组件名称类型的计算和处理方式与此版本不同。该类型仅变为装饰性，因此从现在开始允许导致无效组件名称类型的操作，但是，在导致无效类型的任何操作之前警告用户。具有无效类型的组件名称由错误标记修饰，就像组件名称一样，映射到组件中，该组件未实现可能在组件名称上执行的操作之一。

  • 组件名称浏览器中添加了一项新功能：现在可以显示组件名称的使用计数器。这显示了当前项目中使用组件名称的位置数。用户可以在“ 测试 - 编辑器”首选项窗口中打开和关闭此功能。

使用timeout参数检查操作

  • 大多数检查操作现在都有一个超时参数。如果超时值大于0，则检查操作将多次检查是否已发生所需状态。如果在达到超时之前没有发生这种情况，则检查被标记为检查失败。

跳过测试步骤

  • 现在可以通过将测试步骤的至少一个参数的值更改为“## jbskip ##”来跳过测试步骤。

  • 希望在多个平台上测试Eclipse应用程序的现有客户应激活AUT设置中的新生成名称选项。

  • 此选项可改进RCP向导和对话框中按钮的生成名称和对象识别。它在现有项目中停用，并在新项目中激活。激活此选项意味着从AUT重新映射受影响的组件。

  • 通过使用以下文本过滤对象映射编辑器，可以轻松找到受影响的组件：
    * swt.widgets.button 。
    删除受影响的组件，然后刷新对象映射编辑器。然后，您将看到需要重新映射的组件名称。

Mac OSX BETA支持

  • 希望试用BETA for Mac支持的测试人员可以这样做。

  • 许可证管理器无法在Mac系统上运行;它必须安装在Windows或Linux系统上。

新动作

  • 复制到剪贴板的操作已添加到应用程序组件中。这可以在测试期间用于本机对话框，输入文件路径等。

  • 我们更新了对表的操作，以允许按标题值和列索引选择和检查单元格。输入一个整数将导致根据其位置找到该列。输入字符串意味着根据其标题文本找到该列。标题文本目前仅支持equals运算符。旧表操作现已弃用，应交换新操作。

  • 现在可以选择并检查SWT树上复选框的选择。

RCP组件名称生成

  • 在AUT设置中，RCP AUT有一个新选项，可以为对话框和向导中的按钮生成唯一的名称。

  • 此选项在新项目中自动启用，在现有项目中停用。

  • 如果您的测试必须与平台无关，我们建议您为现有项目激活此选项。有关如何轻松重新映射受影响组件的信息，请参阅上面的Eclipse 3.5部分。

在对象映射编辑器中过滤

  • 对象映射编辑器现在有一个像浏览器一样的文本过滤器。

  • 与浏览器不同，对象映射编辑器过滤器根据编辑器中的文本和技术组件类型进行搜索。

控制装饰

  • 我们在用户界面中为各种组件添加了控件装饰，以帮助新用户了解一些更高级的选项。

新的备忘单

  • 帮助菜单中提供了使用事件处理程序的新备忘单。

显示未使用的测试用例

  • 测试用例浏览器中有一个新的过滤器 - 显示未使用的测试用例。

  • 使用此过滤器通过识别不再引用哪些测试用例来重构和清理测试。

显示测试用例的用途

  • 此操作现在也适用于重用项目的测试用例。

  • 这意味着您可以轻松地从项目中的Test Case库中搜索对已弃用的未绑定模块的引用以进行切换。

新的偏好

  • 在首选项中，您可以选择隐藏测试步骤名称背后的信息。

  • 您还可以取消激活该选项，以便在重命名重用的测试用例时在新名称后面显示原始测试用例名称。

其他信息

Eclipse中的Coolbars和工具栏

  • 为了解决Eclipse中对coolbars的识别问题，我们为coolbars和工具栏引入了更好的组件识别。

  • 测试RCP应用程序的客户必须重新映射他们的工具栏和酷吧按钮，除非这些按钮已由开发团队给出。

命令中的空格

  • 操作执行外部命令已更改为允许命令和参数中的空格。

  • 可以使用包含的空格简单地写入相对路径。如果您使用路径片段（例如./或../ ）作为相对路径，则需要在路径或命令周围使用引号（“）。

  • 使用绝对路径时，请在包含空格的命令或参数周围使用引号（“）。

例如，而不是：
C：\ Program Files \ guidancer \ guidancer.exe
-data C：\ Program Files \ guidancer \ ws
输入： "C:\Program Files\guidancer\guidancer.exe"
-data“C：\ Program Files \ guidancer \ ws”

  • 在Linux中，可以在命令和参数周围放置引号。Windows cmd.exe只能接受参数或命令中的引号。

  • 请注意，不会检查引号中的字符串是否有效。

表格中的绝对和相对搜索

  • 表格中的相对搜索现在从所选项目之后的下一个项目开始搜索。

  • 在表搜索中相对于鼠标位置移动动作。

键盘映射

  • 我们添加了如何将键盘映射文件添加到RCP访问器的文档。

键盘修饰符的新参数

  • 参数»Cmd«和»Mod«已被添加到组合键的操作中，以允许在MAC系统上进行测试，并确保跨Windows，Unix和MAC系统的独立于平台的测试。

已知的问题

连接到AUT代理

  • 当没有项目打开时，连接到AUT代理并再次断开连接导致无法再连接到AUT代理，直到ITE重新启动。

退出代码11

  • 重新启动RCP AUT可能会导致Eclipse中出现错误，该错误会显示错误代码11。

  • 将启动器更新到Eclipse版本3.3并使用参数--launcher.supressErrors (Executable)可能会妨碍这一点。

单击组件

  • 当滚动条围绕组件时，我们不建议使用百分比0和100来单击组件。

GIJ和OpenJDK

  • 这些不受支持。
