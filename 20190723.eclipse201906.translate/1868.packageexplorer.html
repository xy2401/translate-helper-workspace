对象团队包资源管理器

Object Teams包资源管理器提供以下功能（除了标准JDT 包资源管理器视图的功能外 ）：

  • 显示工作台中Object Teams项目的Object Teams元素层次结构 ，包括
    • 具体课程：
      - 团队课（ ）。
      - 角色类（ ）。
      - 嵌套团队 （ ）既是一个角色，也是一个团队。
    • 角色类装饰：
      - 绑定角色类（ ） - 有一个playedBy声明。
      - 重写角色类（ ） - 覆盖超级团队的角色
    • 方法：
      - callin方法标有装饰 。
    • 方法绑定：
      - callin ：之前（ ），之后（ ），替换（ ）
      - 标注 :( ）
    • 守卫谓词：
      - 常规后卫 （ ）
      - 基地守卫 （ ）
  • 物理/逻辑显示角色文件 （ ）：
    • 物理显示使用文件和目录结构，即角色文件将出现在代表封闭团队的特殊文件夹/包中。
    • 逻辑显示将角色文件和内联角色统一显示为其封闭团队的直接子项。
  • 可以通过选择Preferences-> General-> Appearance-> Label Decorations-> Java Type Indicator来启用编译单元 （团队或角色文件）和包 （团队包）的叠加图标 。
    • 编译单位：
      - 举办团队课程（ ）
      - 持有角色类（ 要么 ）
    • 包：
      - 团队套餐（ ）

对象团队插件项目中的Aspect绑定

如果Object Teams Plug-in项目具有方面绑定，则这些项目将直接显示在项目根目录下的包浏览器中：

在名为OT / Equinox Aspect Bindings的节点下方，您可以找到所有的列表由给定的OT 插件改编的基本插件 。每个基本插件依次列出所有实施此改编的团队 。
应用强制导出的基本插件用一个小小的左上箭头装饰：

双击基础插件或团队在编辑器中显示相应的元素：

  • 一个基本插件，作为PDE扩展编辑器中aspectBinding扩展的子元素。
  • 源代码编辑器中的团队类
