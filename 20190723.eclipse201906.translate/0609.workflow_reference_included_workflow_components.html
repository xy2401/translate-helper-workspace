包含的工作流程组件

建模工作流引擎附带了几个有用的工作流组件。需要详细描述这些组件中的一些。

DirectoryCleaner（org.eclipse.emf.mwe.utils。DirectoryCleaner）

此组件允许您在（重新）生成这些工件之前清理包含生成的工件的目录。

DirectoryCleaner组件提供以下配置属性：

表格1。DirectoryCleaner属性

财产名称 描述
目录 设置应清理的目录的路径。所有子目录也将被清理。
useDefaultExcludes 此布尔值确定是否应从清理中排除版本控制系统内部使用的目录。如果启用此属性，则不会清除Subversion内部使用的目录。
排除 此属性允许添加应从清理过程中排除的其他目录。

IfComponent（org.eclipse.emf.mwe.core.container。IfComponent）

IfComponent是一个非常简单的组件，允许例如暂时禁用一个或多个子组件。

例如：

<workflow> <component class =“my.first。WorkflowComponent“> <aProp value =”test“/> </ component> <if cond =”false“> <component class =”my.second。WorkflowComponent“> <anotherProp value =”test2“/> </ component> </ if> <component class =”my.third。WorkflowComponent“> <prop value =”test“/> </ component> </ workflow>

此组件仅提供单个属性：

表2。IfComponent属性

财产名称 描述
条件 确定是否执行子组件。仅允许值true或false 。

StandaloneSetup（org.eclipse.emf.mwe.utils。StandaloneSetup）

StandaloneSetup类不是狭义上的工作流组件。但是，需要在工作流内引用此类，以便在独立模式下设置EMF元模型。

StandaloneSetup提供以下属性：

表3。StandaloneSetup属性

财产名称 描述
platformUri 设置平台的路径。通常，应将其设置为当前工作空间的路径。
URIMAP 注册URI映射，允许将一个URI映射到另一个URI。
延期 允许为特定文件扩展名注册工厂类。
registerGeneratedEPackage 允许注册外部生成的EPackage。
registerEcoreFile 允许注册Ecore文件。

作家（org.eclipse.emf.mwe.utils。作家）

Writer组件允许您将存储在模型槽中的模型写入文件。

Writer组件提供以下配置属性：

表4。Writer属性

财产名称 描述
modelSlot 设置要写出的模型槽的名称。
cloneSlotContents 此布尔值确定在将其写入资源之前是否克隆指定插槽中包含的模型。
multipleResourcesInCaseOfLists 此布尔属性指定在指定的模型槽包含模型列表的情况下的写入行为。如果此属性设置为true ，则每个模型将写入单独的资源。否则，所有模型都将写入单个资源。
