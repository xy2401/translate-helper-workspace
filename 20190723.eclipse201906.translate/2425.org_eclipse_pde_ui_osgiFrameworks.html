OSGi框架

标识符：
org.eclipse.pde.ui.osgiFrameworks

以来：
3.3

此扩展点已从org.eclipse.pde.ui移至3.6中的org.eclipse.pde.launching 。但是，其合格的名称和可见性保持不变。

描述：

此扩展点用于注册新的OSGi框架。每个框架都与在OSGi Framework启动配置中选择OSGi框架时调用的启动器委托相关联。

已注册的OSGi框架出现在Plug-in Development> OSGi框架首选项页面中，其中可以设置默认框架。

配置标记：

<！ELEMENT 扩展 （ 框架 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 框架 EMPTY>

<！ATTLIST框架

id CDATA #REQUIRED

名称CDATA #REQUIRED

launcherDelegate CDATA #REQUIRED

初始化程序CDATA #IMPLIED>

  • id - 框架的唯一标识符
  • name - OSGi框架的可读名称
  • launcherDelegate - 启动OSGi框架的启动配置委托。此属性的值是扩展org.eclipse.pde.launcher.的Java类的完全限定名称org.eclipse.pde.launcher.AbstractPDELaunchConfiguration 。
  • 初始化程序 - 使用适当的默认值初始化新的OSGi Framework启动配置。此属性的值必须是扩展默认实现org.eclipse.pde.launcher.的Java类的完全限定名称org.eclipse.pde.launcher.OSGiLaunchConfigurationInitializer 。如果未指定，则使用默认初始值设定项org.eclipse.pde.launcher.OSGiLaunchConfigurationInitializer被实例化。

例子：

以下是扩展点的示例：

<extension point = “org.eclipse.pde.launching.osgiFrameworks” > <framework launcherDelegate = “org.eclipse.pde.launcher。EquinoxLaunchConfiguration“ id = ”org.eclipse.pde.ui。EquinoxFramework“ name = ”％Equinox.shortcut.label“ initializer = ”org.eclipse.pde.internal.launcher。EquinoxInitializer“ /> </ extension>

API信息：

属性launcherDelegate的值必须是扩展org.eclipse.pde.launcher.的Java类的完全限定名org.eclipse.pde.launcher.AbstractPDELaunchConfiguration 。如果启动程序要提供自己的源查找顺序，则应覆盖方法setDefaultSourceLocator 。

属性initializer值设定项的值必须是扩展org.eclipse.launcher.的Java类的完全限定名称org.eclipse.launcher.OSGiLaunchConfigurationInitializer 。

提供的实施：

PDE提供Equinox OSGi Framework扩展以启动Equinox框架。
