VM安装类型UI页面

标识符：
org.eclipse.jdt.debug.ui.vmInstallTypePage

描述：
此扩展点提供了一种用于提供UI的机制，该UI将显示在启动配置对话框的JRE选项卡中。仅当在JRE选项卡中选择了指定安装类型的VM时，才会显示UI。

配置标记：

<！ELEMENT 扩展 （ vmInstallTypePage *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT vmInstallTypePage EMPTY>

<！ATTLIST vmInstallTypePage

id CDATA #REQUIRED

vmInstallTypeID IDREF #REQUIRED

class CDATA #REQUIRED>

  • id - 指定此vm安装类型UI页面的唯一标识符。
  • vmInstallTypeID - 指定此UI页面适用的VM安装类型（对应于VM安装类型的ID）。
  • class - 指定实现ILaunchConfigurationTab的Java类的完全限定名称。

例子：
以下是VM安装类型页面扩展点的示例：

<extension point = “org.eclipse.jdt.debug.ui.vmInstallTypePage” > <vmInstallTypePage id = “com.example。ExampleVMInstallTypePage“ vmInstallTypeID = ”com.example。ExampleVMInstallTypeIdentifier“ class = ”com.example。ExampleVMInstallTypePage“ > </ vmInstallTypePage> </ extension>
在上面的示例中，只要当前选定的JRE具有com.example的VM安装类型标识符，就会在启动配置对话框的JRE选项卡中显示提供的页面。ExampleVMInstallTypeIdentifier 。

API信息：
属性类的值必须是实现接口org.eclipse.debug.ui的Java类的完全限定名。ILaunchConfigurationTab 。
