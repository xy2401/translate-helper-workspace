目标位置提供者

标识符：
org.eclipse.pde.ui.targetLocationProvisioners

以来：
3.8

描述：

此扩展点用于注册将创建新目标位置的目标位置配置向导。当用户尝试从目标平台首选项页面或目标编辑器向目标定义添加插件时，将列出每个插件供应商。所选供应商负责提供要添加到目标定义的目标位置。

客户端可以通过org.eclipse.pde.core.targetLocations扩展点提供自己的目标位置实现。此扩展点允许用户添加自定义位置实现。使用适配器框架处理其他UI交互（请参阅org.eclipse.pde.core.target.ITargetDefinition ）。

配置标记：

<！ELEMENT 扩展 （ locationProvider +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT locationProvider （ 描述 ？）>

<！ATTLIST locationProvider

id CDATA #REQUIRED

名称CDATA #REQUIRED

icon CDATA #IMPLIED

class CDATA #REQUIRED>

  • id - 供应者的唯一标识符
  • name - 供应商的人类可读名称
  • icon - 将用于直观表示供应商的图标的相对路径。
  • class - 为用户提供向目标添加内容的向导的类。此属性的值是实现org.eclipse.pde.ui.target.ITargetLocationWizard 。

<！元素描述 （#PCDATA）>

供应商的人类可读描述



例子：

以下是扩展点的示例：


    <extension
         point="org.eclipse.pde.ui.targetLocationProvisioners">
      <locationProvider
            class="org.eclipse.example.exampleTargetWizard"
            icon="icons/obj16/exampleTargetLocation.png"
            id="org.eclipse.example.exampleTargetProvisioner"
            name="%targetLocationProvider.example.name"
         <description>
            %targetLocationProvider.example.description
         </description>
      </locationProvider>
   </extension>

API信息：
此扩展点提供的向导必须实现org.eclipse.pde.ui.target.ITargetLocationWizard 。

提供的实施：

PDE提供内部位置配置程序实现，以添加来自更新站点和p2存储库的插件。请参阅org.eclipse.pde.ui.shared.target.InstallableUnitWizard 。
