新的LookupCall向导

New Scout Lookup Call向导可用于创建查找调用和相应的查找服务。要启动向导使用文件 ▸ 新建 ▸ 其他...或按Ctrl + N为在节中描述侦察员奇才 。

图1。新的LookupCall向导

在图1的情况下，在包资源管理器中选择了包org.eclipse.scout.apps.helloworld.shared.helloworld 。然后，需要手动填写的唯一向导字段是Name和Key Class字段。

通过单击“ 完成”按钮，将启动向导并创建指定的组件。

向导字段和默认值

根据IDE的当前上下文， 图1中的大多数字段将填充默认值。上下文可以从Package Explorer中选择的包或活动Java编辑器中的类派生。

源文件夹

用于创建查找调用类的Maven共享模块的源文件夹。默认值是Maven共享模块中的src/main/java文件夹。

包

包含查找调用类的Java包。Scout SDK将尝试从当前上下文中猜测包名称，并为Maven服务器模块派生匹配的包名称。

名称

查找调用类的名称。根据Scout约定，类名以LookupCall后缀LookupCall 。

超级

表格的超类。LookupCall是默认值。

关键班

要用于查找调用的键值的类。

服务器源文件夹

用于创建服务类的Maven服务器模块的源文件夹。默认值是Maven服务器模块中的src/main/java文件夹。

服务超级

与此查找调用关联的查找服务的超类。默认值为AbstractLookupService 。

创建组件

在上面显示的图1示例中，Scout SDK将创建以下组件。

  • 在Maven模块helloworld.shared中

    • 在文件夹src/main/java和包org.eclipse.scout.apps.helloworld.shared.helloworld

      • MyLookupCall类

      • IMyLookupService接口

  • 在Maven模块helloworld.server中

    • 文件夹src/generated/java的MyLookupService服务类和包org.eclipse.scout.apps.helloworld.server.helloworld
