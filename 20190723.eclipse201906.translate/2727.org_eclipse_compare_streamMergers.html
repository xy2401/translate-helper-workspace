流合并

标识符：
org.eclipse.compare.streamMergers

以来：
3.0

描述：
此扩展点允许插件为特定内容类型注册流合并。流合并期望在三个输入流上执行三向合并，并将结果写入输出流。扩展点必须实现接口org.eclipse.compare.IStreamMerger 。

配置标记：

<！ELEMENT 扩展 （ streamMerger *， contentTypeBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

已过时。应该使用org.eclipse.team.core.storageMergers扩展点。


  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT streamMerger EMPTY>

<！ATTLIST streamMerger

id CDATA #REQUIRED

class CDATA #REQUIRED

扩展CDATA #IMPLIED>

  • id - 可用于引用流合并的唯一标识符
  • class - 实现org.eclipse.compare.IStreamMerger
  • extensions - 以逗号分隔的文件扩展名列表，例如“java，properties”

<！ELEMENT contentTypeBinding EMPTY>

<！ATTLIST contentTypeBinding

contentTypeId IDREF #REQUIRED

streamMergerId IDREF #REQUIRED>

contentTypeBinding将流合并绑定到内容类型。


  • contentTypeId - 使用org.eclipse.core.contenttype.contentTypes扩展点定义的内容类型的id。
  • streamMergerId - 使用此扩展点的streamMerger元素定义的流合并的id（即org.eclipse.compare.streamMergers ）

例子：
以下是属性文件的流合并示例（扩展名“properties”）：


<extension point = "org.eclipse.compare.streamMergers"> 
     <streamMerger 
         id="org.eclipse.compare.internal.merge.TextStreamMerger" 
         class="org.eclipse.compare.internal.merge.TextStreamMerger" 
         extensions="properties" 
    /> 
</extension>

API信息：
贡献的类必须实现org.eclipse.compare.IStreamMerger

提供的实施：
Compare UI插件为面向行的文本文件定义流合并。
