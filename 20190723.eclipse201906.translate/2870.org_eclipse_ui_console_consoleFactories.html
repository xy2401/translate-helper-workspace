控制台工厂

标识符：
org.eclipse.ui.console.consoleFactories

以来：
3.1

描述：
控制台工厂创建或激活控制台，并在控制台视图中显示为操作。

配置标记：

<！ELEMENT 扩展 （ consoleFactory ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT consoleFactory EMPTY>

<！ATTLIST consoleFactory

标签CDATA #REQUIRED

icon CDATA #IMPLIED

class CDATA #REQUIRED>

  • label - 要在菜单中显示的操作标签
  • icon - 与动作一起显示的图标的可选插件相对路径
  • class - 实现org.eclipse.ui.console.类org.eclipse.ui.console.IConsoleFactory调用相关的操作时将被调用

例子：
以下是控制台工厂扩展点的示例：
<extension point = “org.eclipse.ui.console.consoleFactories” > <consoleFactory label = “Command Console” class = “com.example。CommandConsoleFactory“ icon = ”icons / cmd_console.png“ > </ consoleFactory> </ extension>
此扩展将导致条目显示在控制台视图的Open Console下拉菜单中，该下拉菜单标有带有指定图标的Command Console 。当用户调用该操作时， com.example类上的方法openConsole（） 。将调用CommandConsoleFactory 。工厂可以决定创建新控制台或激活现有控制台。

API信息：
属性类的值必须是实现接口org.eclipse.ui.console的Java类的完全限定名称。IConsoleFactory 。

提供的实施：
控制台插件提供控制台工厂以打开新的控制台视图。
