模板关联

标识符：
org.eclipse.cdt.core.templateAssociations

以来：
CDT 4.0

描述：
此扩展点允许开发人员指定应启用的工具链，以便与预先存在的模板一起使用。这解决了ISV如何定义模板并随CDT一起提供支持自己的工具链的问题，而无需修改CDT本身。

配置标记：

<！ELEMENT 扩展 （ 模板 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

每个扩展由一系列模板元素组成，每个模板元素定义新工具链的子元素，这些元素应在新项目创建期间启用以供选择。


  • id - 此扩展的唯一标识符

<！ELEMENT toolChain EMPTY>

<！ATTLIST工具链

id IDREF #REQUIRED>

此元素用于通过其唯一标识符引用现有工具链。


  • id - 工具链的唯一标识符，该标识符对org.eclipse.cdt.managedbuilder.core.buildDefinitions扩展点有贡献。

<！ELEMENT 模板 （ toolChain *）>

<！ATTLIST模板

id IDREF #REQUIRED>

此元素引用现有模板贡献的唯一标识符，以便可以在项目创建时选择单独为模板提供的工具链。


  • id - 模板贡献的唯一标识符（plugin.xml中的id而不是template.xml），它应该在新项目创建期间启用选择的指定附加工具链。

例子：
如果您使用ids“com.foobar.toolchain1.base”和“com.foobar.toolchain2.base”开发了工具链，您希望为cdt核心中内置的空项目类型启用它，您可以使用以下方法来实现此目的：
<extension point = “org.eclipse.cdt.core.templateAssociations” > <template id = “org.eclipse.cdt.build.core.templates。EmptyProject“ > <toolChain id = ”com.foobar.toolchain1.base“ /> <toolChain id = ”com.foobar.toolchain2.base“ /> </ template> </ extension>
