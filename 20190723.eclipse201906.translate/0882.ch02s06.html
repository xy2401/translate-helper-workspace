RCP工具包的完全可执行的示例设置，显示基本的SimpleAdder AUT启动 - 参见图�2.6，“通过API启动RCP-SimpleAdder AUT” - 和测试运行 - 参见图�2.7，“执行第一个测试” - 可以是在每个ITE安装中包含的git-Repository中找到。它位于<ITEInstallationFolder> /development/git/org.eclipse.jubula.core.git中 。只需创建该存储库的克隆并检查Eclipse项目org.eclipse.jubula.examples.api.adder.rcp。* 。对于API可以使用的所有三种形状，都有一个单独的预配置项目：

  • plain JAR：此示例设置的项目是org.eclipse.jubula.examples.api.adder.rcp.jar 。您仍然需要调整路径（默认情况下链接到位于git-clone本身内的相对/本地工件）library-directory（lib）。只需调整此路径即可指向例如<ITEInstallationFolder> / development / api / JARs / 。

  • m2 / maven：此示例设置的项目是org.eclipse.jubula.examples.api.adder.rcp.m2 。您仍然需要在该项目的pom.xml中调整本地maven存储库的路径，并指向安装中包含的路径： <ITEInstallationFolder> / development / api / m2-repo 。

  • OSGi：此示例设置的项目是org.eclipse.jubula.examples.api.adder.rcp.osgi 。您仍然需要将（例如本地的）Jubula p2-repository及其工件添加到目标平台： <ITEInstallationFolder> /development/org.eclipse.jubula.repo.zip
