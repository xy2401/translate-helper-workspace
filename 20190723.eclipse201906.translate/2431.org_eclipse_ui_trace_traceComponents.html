跟踪组件

标识符：
org.eclipse.ui.trace.traceComponents

以来：
1.0

描述：
此扩展点允许插件将“调试跟踪”选项提供给“跟踪”首选项页面（“首选项”>“常规”>“跟踪”）。来自一组插件.options文件的所有选项路径都可以从首选项页面进行编辑。由于选项将通过OSGi DebugOptions服务动态启用或禁用，因此指定的插件的跟踪必须侦听更改（请参阅org.eclipse.osgi.service.debug。DebugOptions）。

配置标记：

<！ELEMENT 扩展 （ 组件 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 组件 （ 包 *）>

<！ATTLIST组件

id CDATA #REQUIRED

标签CDATA #IMPLIED>

跟踪组件


  • id - 此组件的unqiue标识符
  • label - 将在跟踪对话框中显示的组件的可读标签。如果未提供标签，则会显示标签“缺少标签”。

<！ELEMENT 捆绑 EMPTY>

<！ATTLIST捆绑

名称CDATA #IMPLIED

消耗（true | false）>

Eclipse包。


  • name - 特定包的符号名称或定义一组包的名称的正则表达式。
  • consume - 描述此组件是否使用了捆绑包（或捆绑包）。使用的bundle是除了此组件之外不会显示在任何其他跟踪组件中的bundle。

例子：
以下是扩展点的示例，其中名为“Equinox Framework”的组件包含org.eclipse.osgi包中.options文件中的所有选项路径。同样，名为“JDT Core”的第二个组件包括匹配org.eclipse.jdt。* core *的任何捆绑包中的.options文件中的所有选项路径。跟踪首选项页面将在树中显示这两个类别，其中所有已知选项路径均为子项。 


   <extension
         point="org.eclipse.ui.trace.traceComponent">
      <component
            id="org.eclipse.ui.trace.osgitrace"
            label="Equinox Framework">
         <bundle
               name="org.eclipse.osgi">
         </bundle>
      </component>
   </extension>
   <extension
         point="org.eclipse.ui.trace.traceComponent">
      <component
            id="org.eclipse.ui.trace.jdtcoretrace"
            label="JDT Core">
         <bundle
               name="org.eclipse.jdt.*core*">
         </bundle>
      </component>
   </extension>
