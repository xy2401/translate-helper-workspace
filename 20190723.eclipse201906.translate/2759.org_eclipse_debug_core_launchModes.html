启动模式

标识符：
org.eclipse.debug.core.launchModes

以来：
3.0

描述：
此扩展点提供了一种为调试平台提供启动模式的机制。调试平台定义“运行”，“调试”和“配置文件”的模式。

配置标记：

<！ELEMENT 扩展 （ launchMode +）>

<！ATTLIST扩展

点CDATA #REQUIRED>

  • point - 目标扩展点的完全限定标识符

<！ELEMENT launchMode EMPTY>

<！ATTLIST launchMode

模式CDATA #REQUIRED

标签CDATA #REQUIRED

launchAsLabel CDATA #IMPLIED>

  • mode - 指定此启动模式的唯一标识符。调试平台提供的启动模式是“运行”，“调试”和“配置文件”。
  • label - 描述启动模式的人类可读标签
  • launchAsLabel - 用于启动级联菜单的人类可读标签。例如“Run As”。在3.2中添加了此属性以允许标签正确外部化。未指定时，将通过串联创建默认标签。

例子：
以下是启动委托扩展点的示例：

<extension point = “org.eclipse.debug.core.launchModes” > <launchMode mode = “profile” label = “Profile” > </ launchMode> </ extension>
在上面的示例中，提供了配置文件启动模式。
