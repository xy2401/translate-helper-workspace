调试模型演示

标识符：
org.eclipse.debug.ui.debugModelPresentations

描述：
此扩展点允许工具处理调试模型的表示方面。调试模型演示文稿负责为特定调试模型中的元素提供标签，图像和编辑器。

配置标记：

<！ELEMENT 扩展 （ debugModelPresentation *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT debugModelPresentation EMPTY>

<！ATTLIST debugModelPresentation

class CDATA #REQUIRED

id CDATA #REQUIRED

detailsViewerConfiguration CDATA #IMPLIED>

  • class - 实现org.eclipse.debug.ui.的Java类的完全限定名称org.eclipse.debug.ui.IDebugModelPresentation接口。从3.1开始，调试模型演示文稿可以选择实现IColorProvider和IFontProvider来覆盖调试元素的默认字体和颜色。
  • id - 此演示文稿负责的调试模型的标识符
  • detailsViewerConfiguration - 作为org.eclipse.jface.text.source.实例的Java类的完全限定名称org.eclipse.jface.text.source.SourceViewerConfiguration 。指定后，在与此调试模型演示文稿关联的调试模型中显示元素的详细信息时，将在变量和表达式视图的“详细信息”区域中使用源查看器配置。未指定时，使用默认配置。

例子：
以下是调试模型演示扩展点的示例：

<extension point = “org.eclipse.debug.ui.debugModelPresentations” > <debugModelPresentation class = “com.example。JavaModelPresentation“ id = ”com.example。JavaDebugModel“ > </ debugModelPresentation> </ extension>
在上面的示例中，类com.example。JavaModelPresentation将用于呈现和显示源自com.example标识的调试模型的调试元素。JavaDebugModel 。

API信息：
action属性类的值必须是实现org.eclipse.debug.ui的Java类的完全限定类名。IDebugModelPresentation 。从3.1开始，调试模型演示文稿可以选择实现IColorProvider和IFontProvider来覆盖调试元素的默认字体和颜色。
