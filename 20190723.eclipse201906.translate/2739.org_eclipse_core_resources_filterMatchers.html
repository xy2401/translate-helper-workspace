过滤匹配器

标识符：
org.eclipse.core.resources.filterMatchers

以来：
3.6

描述：
过滤器匹配扩展点允许插件提供匹配器。应用于容器（文件夹或项目）的过滤器使用匹配器在填充资源树时包含或排除某些文件系统对象。

配置标记：

<！ELEMENT 扩展 （ filterMatcher +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT filterMatcher EMPTY>

<！ATTLIST filterMatcher

class CDATA #REQUIRED

argumentType（none | string | filterMatcher | filterMatchers）

名称CDATA #REQUIRED

描述CDATA #IMPLIED

订购（第一|最后）

id CDATA #REQUIRED>

  • 课程 -
  • argumentType -
  • 名字 -
  • 说明 -
  • ordering - 一个提示，指定应在给定容器的活动过滤器列表中首先或最后调用此过滤器。通常出于性能原因而指定。
  • id -

例子：
以下是使用modelProvider扩展点的示例。（在文件plugin.xml中）
<extension point = “org.eclipse.core.resources.filterMatchers” > <filterMatcher argumentType = “string” class = “org.eclipse.core.internal.resources。RegexFileInfoMatcher“ > </ filterMatcher> </ extension>

API信息：
class属性的值必须表示org.eclipse.core.resources.filtermatchers的子类。AbstractFileInfoMatcher 。

提供的实施：
核心资源插件提供正则表达式匹配器，允许用户指定与java.util.regex支持的规范匹配的字符串参数。图案。
