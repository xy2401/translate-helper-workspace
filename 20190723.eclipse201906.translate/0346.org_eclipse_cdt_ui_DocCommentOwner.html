DocCommentOwner

标识符：
org.eclipse.cdt.ui。DocCommentOwner

5

描述：
此扩展点允许将文档注释检测，演示和编辑行为贡献给CDT编辑器。

在C和C ++定义单行和多行注释的情况下，文档工具无法识别差异注释的相应语言级别定义。这里有一个列表： Wikipedia文档生成器的比较

单行和多行注释的自定义由同一接口的两个贡献实现处理

  • org.eclipse.cdt.ui.text.doctools。IDocCommentViewerConfiguration
此接口的javadoc描述了可以提供的各个编辑器功能。

需要对eclipse和CDT编辑器基础结构有一定程度的了解才能为此扩展点做出贡献。一个很好的起点是查看包中的GenericDocTag类 - 请参阅下面的API Information部分。

配置标记：

<！ELEMENT 扩展 （ 所有者 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 所有者 EMPTY>

<！ATTLIST所有者

id IDREF #REQUIRED

名称CDATA #REQUIRED

多行CDATA #REQUIRED

单行CDATA #REQUIRED>

  • id - 表示此所有者的全局唯一ID。
  • name - CDT UI的可读名称
  • multiline - 应该用于检测，显示和编辑C / C ++多行注释的注释查看器配置。
  • singleline - 应该用于检测，显示和编辑C / C ++单行注释的注释查看器配置。

例子：
<extension point = “org.eclipse.cdt.ui。DocCommentOwner“ > <owner id = ”org.eclipse.cdt.ui.doxygen“ name = ”Doxygen“ multiline = ”org.eclipse.cdt.ui.text.doctools.doxygen。DoxygenMultilineConfiguration“singleline = ”org.eclipse.cdt.ui.text.doctools.doxygen。DoxygenSingleConfiguration“ > </ owner> </ extension>

javadoc被认为是插件实现者的主要信息来源。这里只给出了一个简洁的轮廓

  • org.eclipse.cdt.ui.text.doctools。IDocCommentViewerConfiguration
  • org.eclipse.cdt.ui.text.doctools。IDocCommentOwnershipListener
关键实现是：
  • 包中的实现org.eclipse.cdt.ui.text.doctools.generic

提供的实施：
org.eclipse.cdt.ui.text.doctools.generic包中给出了一些关键接口的抽象实现。
