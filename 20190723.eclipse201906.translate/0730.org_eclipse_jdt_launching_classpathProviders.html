Java运行时类路径提供程序

标识符：
org.eclipse.jdt.launching.classpathProviders

以来：
2.1

描述：
此扩展点允许客户端动态计算和解析Java启动配置的类路径和源查找路径。Java启动配置可以通过启动配置attriubte ATTR_CLASSPATH_PROVIDER与自定义类路径提供程序相关联，并通过属性ATTR_SOURCE_PATH_PROVIDER与自定义源路径提供程序相关联。指定后，启动配置属性对应于类路径提供程序扩展的ID。

配置标记：

<！ELEMENT 扩展 （ classpathProvider *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT classpathProvider EMPTY>

<！ATTLIST classpathProvider

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 可用于引用此类路径提供程序的唯一标识符
  • class - 实现此类路径提供程序的类。该类必须实现IRuntimeClasspathProvider

例子：
以下是类路径提供程序的示例：


 <extension point="org.eclipse.jdt.launching.classpathProviders">
  <classpathProvider
   class="com.example.ProviderImplementation"
   id="com.example.ProviderId">
  </classpathProvider>
 </extension>

提供的实施：
为未指定自定义类路径提供程序的所有启动配置提供了默认实现。
