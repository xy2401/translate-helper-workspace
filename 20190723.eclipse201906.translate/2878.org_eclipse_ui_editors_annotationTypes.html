注释类型

标识符：
org.eclipse.ui.editors.annotationTypes

以来：
3.0

描述：
注释是附加到文本文档的某个区域的一条信息。可以使用此扩展点定义新类型的注释。注释通过其注释模型附加到文档，并可以在文本编辑器和视图中显示。注释类型形成层次结构：注释类型可以通过在其super属性中指定它来优化另一种类型。一些注释用作标记的UI对应物（请参阅org.eclipse.core.resources.IMarker ），而其他人则独立存在而没有可持续的形式。标记和注释类型之间的映射由可选的markerType和markerSeverity属性定义。

配置标记：

<！ELEMENT 扩展 （ 类型 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 类型 EMPTY>

<！ATTLIST类型

名称CDATA #REQUIRED

markerType IDREF #IMPLIED

超级IDREF #IMPLIED

markerSeverity（0 | 1 | 2）>

标记类型定义。


  • name - 此批注类型的唯一名称。每当引用注释类型时都会使用它，例如，在指定org.eclipse.ui.editors.markerAnnotationSpecification 。
  • markerType - 此注释类型对应的标记类型（如果有）
  • super - 父类型的名称，如果此类型是另一个注释类型的后代
  • markerSeverity - 此注释类型的可选严重性，用于将注释类型映射到标记。任何超出IMarker。SEVERITY_INFO，IMarker。SEVERITY_WARNING，SEVERITY_ERROR。

例子：
这是JDT UI的plugin.xml的摘录，它添加了java编译器错误和警告注释：
<extension point = “org.eclipse.ui.editors.annotationTypes” > <type name = “org.eclipse.jdt.ui.error” super = “org.eclipse.ui.workbench.texteditor.error” markerType = “org .eclipse.jdt.core.problem“ markerSeverity = ”2“ > </ type> <type name = ”org.eclipse.jdt.ui.warning“ super = ”org.eclipse.ui.workbench.texteditor.warning“ markerType = “org.eclipse.jdt.core.problem” markerSeverity = “1” > </ type> <type name = “org.eclipse.jdt.ui.info” super = “org.eclipse.ui.workbench.texteditor。 info“ markerType = ”org.eclipse.jdt.core.problem“ markerSeverity = ”0“ > </ type> </ extension>

API信息：
请参阅org.eclipse.jface.text.source。注释类和org.eclipse.ui.editors.markerAnnotationSpecification扩展点。
