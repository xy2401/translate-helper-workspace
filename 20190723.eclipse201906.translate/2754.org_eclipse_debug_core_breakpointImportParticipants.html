断点导入参与者

标识符：
org.eclipse.debug.core.breakpointImportParticipants

以来：
3.5

描述：
此扩展点允许贡献者明确定义在导入工作空间时应如何比较断点，以及如果工作空间中已存在类似断点，应如何更新断点。

配置标记：

<！ELEMENT 扩展 （ importParticipant ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT importParticipant EMPTY>

<！ATTLIST importParticipant

参与者CDATA #REQUIRED

输入IDREF #REQUIRED>

描述一个导入参与者


  • 参与者 - 参与断点导入的类
  • type - 支持此参与者关心的断点类型的标记的id。例如'org.eclipse.jdt.debug.javaLineBreakpointMarker'，Java行断点的支持标记的id

例子：
以下是断点参与者扩展的示例：


<extension point="org.eclipse.debug.core.breakpointImportParticipants">
 <importParticipant
     participant="x.y.z.BreakpointImportParticipant"
     type="org.eclipse.jdt.debug.javaLineBreakpointMarker">
 </importParticipant>
</extension>
在上面的例子参与者关心的Java行断点：其中org.eclipse.jdt.debug.javaLineBreakpointMarker是使用所定义的一个断点型org.eclipse.debug.core.breakpoints在扩展点org.eclipse.jdt.debug

API信息：
type属性是支持此参与者关心的断点类型的标记的id。类型必须是此参与者关心的断点的确切标记ID，不考虑标记层次结构。例如'org.eclipse.jdt.debug.javaLineBreakpointMarker'，Java行断点的支持标记的id participant属性必须指定实现org.eclipse.debug.core.model.IBreakpointImportParticipant接口。

提供的实施：
Platform Debug提供了一般断点的默认实现，它将断点与参与者支持之前的比较方式进行比较：基于断点类型和行号信息。
