IM机器人

标识符：
org.eclipse.ecf.presence.bot.imRobot

以来：
1.0.0里程碑6

描述：
此扩展点允许实现者定义im bot。

配置标记：

<！ELEMENT 扩展 （ imRobot ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT imRobot EMPTY>

<！ATTLIST imRobot

id CDATA #REQUIRED

containerFactoryName CDATA #REQUIRED

connectId CDATA #REQUIRED

名称CDATA #IMPLIED

connectPassword CDATA #IMPLIED

>

imrobot扩展点。


  • id - im机器人所需的id。这必须是相对于所有其他im机器人的唯一值。对于要使用此机器人的消息处理程序，它们必须提供与此值匹配的imrobotid。
  • containerFactoryName - 所需的容器工厂名称。这应该是给定协议的容器工厂名称。例如，对于内置的ECF IRC提供程序，值为ecf.xmpp.smack 。
  • connectId - 用于连接到给定容器的连接ID。例如，对于XMPP，连接ID的预期ID语法是
    username @ host [：port]例如：slewis@ecf.eclipse.org
  • name - 机器人的可选名称。
  • connectPassword - 可选的连接密码。如果需要密码才能连接到给定容器，则必须提供适当的值。

例子：
<extension point = “org.eclipse.ecf.presence.bot.imRobot” > <imRobot connectID = “slewis@ecf.eclipse.org” containerFactoryName = “ecf.xmpp.smack” id = “org.eclipse.ecf.presence .imbot.defaultbot“ /> </ imRobot> </ extension>
注意，为了让imrobot接收im消息并处理它们，还应该使用上面给出的imrobot的id来定义immessagehandler。例如：
<extension point = “org.eclipse.ecf.presence.bot.immessagehandler” > <handler imrobotid = “org.eclipse.ecf.presence.imbot.defaultbot” class = “org.eclipse.ecf.presence.imbot。DefaultIMMessageHandler“ > </ handler> </ extension>

API信息：
请参阅immessagehandler的API信息。
