ExportProjectProvider

CIndex的这个子元素允许替代IExportProjectProvider实现的贡献。然后可以在命令行工具中使用完全限定的类名称来引用它们（请参阅选项-pprovider）。

将应用程序作为无头应用程序调用此示例ant文件显示了如何无头地调用该工具，相同的方法可以在shell或批处理文件中使用。

ReadOnlyPDOMProvider

CIndex的这个子元素允许ISV将只读的预建PDOM文件提供给CDT索引。所需的唯一信息是org.eclipse.cdt.core.index的实现的完全限定类名。IOfflinePDOMProvider。在eclipse会话期间将查询此实现，以获取适当的只读内容以添加到逻辑索引。可以通过org.eclipse.core.index访问逻辑索引。IIndex API。贡献预建只读pdom的一个例子：

<CIndex> <ReadOnlyPDOMProvider class = “com.acme.ide.index。AcmeSDKProvider“ /> </ CIndex>

ReadOnlyIndexFragmentProvider

CIndex的这个子元素允许ISV向CDT索引提供只读索引片段。该扩展点对如何收集和存储索引信息没有限制。例如，索引片段可以基于从远程服务器动态提取的数据。

此扩展点所需的唯一信息是实现org.eclipse.cdt.core.index的类的完全限定名称。IIndexFragmentProvider接口。在eclipse会话期间将查询此类，以便将适当的只读内容添加到逻辑索引中。可以通过org.eclipse.core.index访问逻辑索引。IIndex API。


  • class - 实现IIndexFragmentProvider的类的完全限定名称。

<！ELEMENT FragmentProviderUsage EMPTY>

<！ATTLIST FragmentProviderUsage

导航（true | false）

content_assist（true | false）

add_import（true | false）

call_hierarchy（true | false）

type_hierarchy（true | false）

include_browser（true | false）

搜索（true | false）

编辑器（true | false）>

定义应使用片段提供程序的工具。


  • 导航 - 片段提供者是否应用于执行“开放声明”和“开放元素”。
  • content_assist - 片段提供者是否应用于执行内容辅助。
  • add_import - 片段提供程序是否应用于执行“添加导入”。
  • call_hierarchy - 片段提供程序是否应用于填充调用层次结构。
  • type_hierarchy - 片段提供程序是否应用于填充类型层次结构。
  • include_browser - 片段提供程序是否应用于填充include浏览器。
  • search - 片段提供程序是否应用于C / C ++搜索。
  • editor - 片段提供程序是否应用于C / C ++编辑器。
