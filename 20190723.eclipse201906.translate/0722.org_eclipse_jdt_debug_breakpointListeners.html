Java断点监听器

标识符：
org.eclipse.jdt.debug.breakpointListeners

以来：
3.5

描述：
允许客户端为Java断点通知提供侦听器。例如，在命中断点并即将暂停执行时调用侦听器。监听器可以投票以恢复或暂停调试会话。可以以编程方式将侦听器添加到特定Java断点（由断点侦听器标识符指定）或从中删除侦听器，或者注册侦听器以侦听所有Java断点的通知。

配置标记：

<！ELEMENT 扩展 （ breakpointListener ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT breakpointListener EMPTY>

<！ATTLIST breakpointListener

id IDREF #REQUIRED

class CDATA #REQUIRED

过滤器（*）>

  • id - 断点侦听器的唯一标识符
  • class - 断点通知的委托 - 必须是IJavaBreakpointListener的实例
  • filter - 控制是否为所有断点通知断点侦听器。目前，唯一允许的选项是“*”表示所有断点。如果未指定，则必须通过其id以编程方式将侦听器添加到断点。

例子：
以下是定义断点侦听器的示例。在此示例中，仅向侦听器通知其以编程方式注册的特定断点的断点事件。 


<extension
       point="org.eclipse.jdt.debug.breakpointListeners">
   <breakpointActionDelegate
          class="com.example.BreakpointActionDelegate"
          id="com.example.breakpoint.action">
    </breakpointActionDelegate>
</extension>

API信息：

  • breakpointListener元素中的属性类的值必须是实现org.eclipse.jdt.debug.core的Java类的完全限定名称。IJavaBreakpointListener 。
  • 监听器以编程方式添加到断点并从断点中删除。请参阅IJavaBreakpoint.addBreakpointListener(String id)和IJavaBreakpoint.removeBreakpointListener(String id)

    • 提供的实施：
    没有
