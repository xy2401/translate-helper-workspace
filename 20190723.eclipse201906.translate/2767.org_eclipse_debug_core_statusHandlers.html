状态处理程序

标识符：
org.eclipse.debug.core.statusHandlers

描述：
此扩展点提供了一种分离错误的生成和解决方案的机制。错误源和解决方案之间的交互是客户端定义的。在发生错误情况时，查找并委派给状态处理程序是客户的责任。

配置标记：

<！ELEMENT 扩展 （ statusHandler *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT statusHandler EMPTY>

<！ATTLIST statusHandler

id CDATA #REQUIRED

class CDATA #REQUIRED

插件CDATA #REQUIRED

代码CDATA #REQUIRED>

  • id - 指定此状态处理程序的唯一标识符。
  • class - 指定实现IStatusHandler的Java类的完全限定名称。
  • plugin - 插件标识符，对应于此处理程序注册状态的插件。 （即IStatus.getPlugin() ）。
  • code - 指定此处理程序注册的状态代码。

例子：
以下是状态处理程序扩展点的示例：

<extension point = “org.eclipse.debug.core.statusHandlers” > <statusHandler id = “com.example。ExampleIdentifier“ class = ”com.example。ExampleStatusHandler“ plugin = ”com.example。ExamplePluginId“ code = ”123“ > </ statusHandler> </ extension>
在上面的示例中，将注册指定的状态处理程序以处理具有com.example的插件标识符的状态对象。ExamplePluginId和状态代码123 。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core的Java类的完全限定名称。IStatusHandler 。
