断点组织者

标识符：
org.eclipse.debug.ui.breakpointOrganizers

以来：
3.1

描述：
断点组织者根据某些特定标准对断点进行分类。例如，提供断点组织器以按项目对断点进行分类。具有指定名称的组织者将由调试平台自动创建，并作为分组断点的选项呈现给用户。提供的类，必须实现org.eclipse.debug.ui。IBreakpointOrganizerDelegate将仅在必要时加载，以避免早期插件激活。

配置标记：

<！ELEMENT 扩展 （ breakpointOrganizer +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT breakpointOrganizer EMPTY>

<！ATTLIST breakpointOrganizer

id CDATA #REQUIRED

class CDATA #REQUIRED

标签CDATA #REQUIRED

icon CDATA #IMPLIED

othersLabel CDATA #IMPLIED>

  • id - 此断点组织者的唯一标识符。
  • class - org.eclipse.debug.ui.实现org.eclipse.debug.ui.IBreakpointOrganizerDelegate执行分类。从3.3开始，该类可以选择实现org.eclipse.debug.ui.IBreakpointOrganizerDelegateExtension 。
  • label - 此组织者的标签，适合呈现给用户。
  • icon - 可以为此管理器显示的图标的可选路径。
  • othersLabel - 此组织者的可选标签，适用于向用户显示描述不属于此组织者提供的类别的断点。例如，如果组织者按工作集对断点进行分类，但断点不属于工作集，则将使用此标签。未指定时，使用“其他”。

例子：
以下是断点组织者扩展的示例。 
<extension point = “org.eclipse.debug.ui.breakpointOrganizers” > <breakpointOrganizer class = “com.example。BreakpointOrganizer“ id = ”com.example。BreakpointOrganizer“ label = ”示例管理器“ icon = ”icons / full / obj16 / example_org.png“ /> </ extension>
在上面的示例中，提供的工厂将包含在用于分组断点的选项列表中（“分组依据>示例管理器”）。选中后，关联的管理器将用于对断点进行分类。

API信息：
属性类的值必须是实现接口org.eclipse.debug.ui的Java类的完全限定名。IBreakpointOrganizerDelegate 。从3.3开始，类可以选择实现org.eclipse.debug.ui。IBreakpointOrganizerDelegateExtension 。

提供的实施：
Debug Platform为项目，文件，断点类型和工作集提供断点组织器。
