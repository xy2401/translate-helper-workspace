UI插件与帮助上下文相关联

DITA映射上下文敏感的帮助插件必须包含一个<topicmeta>元素作为<map>元素的第一个孩子，并为每个UI插件他们的帮助上下文ID在地图上被识别，即<topicmeta>元素必须包含一个标识UI插件的<othermeta>元素。

注意：标识UI插件的<othermeta>元素应该是<topicmeta>元素中的最后一个子元素。<topicmeta>元素可以包含任何其他有效的子元素（在标识UI插件的<othermeta>元素之前）。
<othermeta>元素的名称和内容属性值将用于标识org.eclipse.help.contexts扩展中的UI插件，该插件在上下文相关的插件清单（plugin.xml文件）中声明帮助插件。例如：
    <map id="org.eclipse.datatools.ui.doc">
        <topicmeta>
            ...
            <othermeta name="ui-plugin"
                content="org.eclipse.datatools.connectivity.ui"/>
            <othermeta name="ui-plugin"
                content="org.eclipse.datatools.connectivity.ui.dse"/>
        </topicmeta>
        ...
    </map>

name属性值“ui-plugin”是一个固定的文字字符串。content属性值是UI插件的Eclipse插件ID。

提供与上下文相关的帮助主题的嵌套映射必须包含相同的标记，以标识在该映射中标识其上下文ID的每个UI插件。
