标记帮助

标识符：
org.eclipse.ui.ide.markerHelp

以来：
3.0（最初在2.0版中添加为org.eclipse.ui.markerHelp）

描述：
该扩展点用于将帮助上下文id与标记的特定“种类”（特定类型的标记或具有某些属性值）相关联。

配置标记：

<！ELEMENT 扩展 （ markerHelp *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT markerHelp （ attribute *）>

<！ATTLIST markerHelp

markerType IDREF #IMPLIED

helpContextId CDATA #IMPLIED

helpContextProvider CDATA #IMPLIED

matchChildren（true | false）>

  • markerType - 帮助上下文适用的标记的唯一类型。
  • helpContextId - 帮助上下文的唯一ID。仅在未设置helpContextProvider时才使用此选项。
  • helpContextProvider - 实现IMarkerHelpContextProvider的类的名称。如果指定了，则忽略helpContextId。
  • matchChildren - 帮助上下文是否也适用于给定markerType的子项。

    如果省略，则假定为false。

    自：3.14


<！ELEMENT 属性 EMPTY>

<！ATTLIST属性

名称CDATA #REQUIRED

值CDATA #REQUIRED>

  • name - 要指定其值的属性的名称。
  • value - 属性的指定值。

例子：
以下是标记帮助扩展的示例（请注意子元素和使用的方式属性）：

<extension point = “org.eclipse.ui.ide.markerHelp” > <markerHelp markerType = “org.eclipse.ui.examples.readmetool.readmemarker” helpContextId = “org.eclipse.ui.examples.readmetool.marker_example1_context” > < attribute name = “org.eclipse.ui.examples.readmetool.id” value = “1234” /> </ markerHelp> </ extension>
在上面的示例中，帮助上下文id与org.eclipse.ui.examples.readmetool.readmemarker类型的标记相关联，其org.eclipse.ui.examples.readmetool.id属性的值为1234。

API信息：
开发人员需要确保为给定标记仅提供单个帮助上下文ID。如果为给定类型的标记提供了两个或更多帮助上下文ID，则工作台不会定义将返回的标记。但是，工作台确实定义了对于给定标记始终返回“最具体”的上下文ID。也就是说，在仅与两个相关联的上下文id之前将返回与三个匹配属性值相关联的上下文id。
