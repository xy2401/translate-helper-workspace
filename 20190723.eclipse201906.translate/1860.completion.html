3。创建callout / callin方法绑定

对代码完成的特定支持是为了便于创建标注 （ 3. ）和callin （ 4. ）方法绑定。
完成可用于
  • 创建完整的方法绑定（callout / callin）
  • 完成部分标注/ callin绑定
  • callin创建后的更正。

3.1创建完整的方法绑定（callout / callin）

只需在角色类的主体中的任何位置激活完成，您就可以创建完整的标注或callin绑定到现有的基本方法（或字段 - “callout to field”，请参阅 3.5 ）。
按Ctrl-space ，选择所需的绑定分三步进行：
a）选择基本方法
b）如果需要，更改角色方法名称。
c）选择绑定类型。

3.1.1创建字段标注

通过在调用完成之前键入“ get ”或“ set ”，可以生成另外两种标注：

打字后get 为字段getter创建一个标注。
键入set 为字段设置器创建一个标注。

3.1.2使用提升/降低来映射签名

绑定到基本方法/字段时，其签名包含当前团队中存在绑定角色的其他基本类型，完成允许您选择绑定的角色端是否应使用基本类型或角色类型。如果选择了角色类型，它们将通过隐式使用提升来符合相应的基本类型（ 2.3 ）或降低（ 2.2 ）。

3.2完成部分callin / callout绑定

如果已经键入了callin或callout绑定并包含“ <- ”或“ -> ”标记，则可以使用completion来扩展指定基本方法或字段的右侧。对于callin绑定，您可以选择首先键入callin修饰符（ before, replace或after ）。否则，系统将提示您输入所需的callin修饰符。在这种情况下，推断出建议的修饰符与绑定的基本方法匹配：

常规方法默认值是before （也after是合法的）。
callin方法默认是replace

通常，在“ -> ”（以及“ <- ”）之后，完成将尝试扩展基本方法 ，如果对于标注已经键入了一个修饰符get或set ，则搜索字段 。

3.3 callin创建后的更正

使用完成生成callin绑定后，以下两个更正可能特别有用：
  • 调整角色方法的callin修饰符，以匹配绑定的callin修饰符。
  • 创建缺少的角色方法。
两种更正都可以通过错误的角色方法规范提供的quickfix轻松执行。
