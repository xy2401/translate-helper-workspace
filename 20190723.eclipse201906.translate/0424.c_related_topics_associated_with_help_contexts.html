相关主题与帮助上下文关联

上下文相关帮助插件的DITA映射包含<resourceid>元素，每个元素都标识与DITA主题关联的具体帮助上下文ID。

<resourceid>元素是<topicmeta>元素的子元素。例如：
<topicmeta> <resourceid id =“help_context_ID_string”/> </ topicmeta>
对于具有帮助上下文ID相关联的每个DITA主题，指向该主题的<topicref>元素包含一个<topicmeta>元件（具有<RESOURCEID>子元素）。这定义了帮助上下文ID与该主题的关联。例如：
<topicref navtitle =“contexts.xml主题元素中的标签属性”href =“path / to / topic.xml”> <topicmeta> <resourceid id =“help_context_ID_string”/> </ topicmeta> </ topicref>
通过在<topicmeta>元素中插入必要的<resourceid>子元素，可以将任何DITA主题映射到多个帮助上下文ID。例如：
<topicref navtitle =“contexts.xml主题元素中的标签属性”href =“path / to / topic.xml”> <topicmeta> <resourceid id =“help_context_ID_1”/> <resourceid id =“help_context_ID_2”/> <resourceid id =“help_context_ID_3”/> </ topicmeta> </ topicref>

最好在helpKey属性文件中组合帮助上下文，而不是将多个帮助上下文的映射定义到DITA映射中的单个主题。这是负责维护DITA映射的文档团队的判断请求。有关更多信息，请参阅定义帮助上下文ID。

定义上下文相关帮助插件的DITA映射的其他注意事项：
  • 仅对将成为上下文相关帮助目标的DITA主题（即“帮助”视图中显示的“相关主题”）需要上下文到主题映射的标记。
  • 当单个DITA主题在地图中出现多次时，只有指向该主题的<topicref>的第一个实例需要上下文相关帮助标记。
  • 提供与帮助上下文相关的DITA主题的嵌套映射必须包含相同的上下文相关帮助标记。
