编制参与者

标识符：
org.eclipse.jdt.core.validationParticipant

以来：
3.2

描述：
此扩展点允许客户端通过org.eclipse.jdt.core.compiler.validationParticipant在构建和协调的各个阶段接收通知来参与验证过程。

配置标记：

<！ELEMENT 扩展 （ validationParticipant *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT validationParticipant （ 需要 *， managedMarker *）>

<！ATTLIST validationParticipant

class CDATA #REQUIRED

id CDATA #REQUIRED

modifiesEnvironment（true | false）

createsProblems（true | false）

requiredSourceLevel CDATA #IMPLIED>

编译参与者的定义。该定义允许订购参与者。参与者由小组管理。首先运行修改环境的参与者组，然后运行创建问题的参与者组，最后运行其他参与者组。在每个组中，参与者使用他们的“需求”属性进行排序。如果'requires'属性指向不属于该组的参与者，则会忽略该属性。


  • class - 实现此编译参与者的类。此类必须使用公共0参数构造函数实现org.eclipse.jdt.core.compiler.validationParticipant的公共子类。
  • id - 此参与者的唯一标识符
  • modifiesEnvironment - 可选择指定此编译参与者是否修改环境，从而影响绑定解析。如果未指定，则假定为false。
  • createsProblems - 可选择指定此编译参与者是否产生问题。如果未指定，则假定为false。
  • requiredSourceLevel - 此参与者所需的源级别（JavaCore之一）。VERSION *常数）。如果项目的源级别低于此必需的源级别，则不会调用该参与者。如果未指定此属性，则将为任何源级别调用参与者。

<！ELEMENT 需要 EMPTY>

<！ATTLIST要求

id CDATA #REQUIRED>

需要运行此编译参与者的参与者


  • id - 所需参与者的唯一标识符

<！ELEMENT managedMarker EMPTY>

<！ATTLIST managedMarker

markerType CDATA #REQUIRED>

根据记录的问题创建托管标记（请参阅BuildContext＃recordNewProblems（CategorizedProblem []）），并在重新构建相应的源文件或清除项目时删除


  • markerType - 托管标记的类型

例子：
validationParticipant声明的示例：
<extension id = “apt” name = “％annotationProcessingName” point = “org.eclipse.jdt.core.validationParticipant” > <validationParticipant class = “org.eclipse.jdt.apt.core.internal。AptvalidationParticipant“ id = ”APT“ requiredSourceLevel = ”1.5“ > <managedMarker markerType = ”org.eclipse.jdt.apt.core.compile.problem“ /> </ validationParticipant> </ extension>
