设置商店

标识符：
org.eclipse.rap.ui.settingstores

以来：
RAP 1.1

描述：
此扩展点允许开发人员提供其他SettingStore实例。

SettingStore为设置（字符串的键值对）提供持久存储，然后可以在单个会话的范围之外使用。SettingStoreFactory负责创建特定类型的SettingStore 。

想要实现自己的设置存储的开发人员必须提供设置存储工厂，并使用此扩展点将其注册到框架。

请注意，在运行时，将只使用一个 （在所有可用的）工厂中创建新的SettingStore实例。使用哪个工厂可以通过两种方式控制：

  • 在正常的RAP部署中，您可以通过将以下属性放在OSGi config.ini文件中来指定要使用的工厂： org.eclipse.rap.rwt.settingStoreFactory= id其中id标识特定的工厂元素。例：
    org.eclipse.rap.rwt.settingStoreFactory =文件
  • 在仅RWT部署中，您可以指定要在应用程序配置中使用的工厂：
    public class SimpleConfiguration实现ApplicationConfiguration {public void configure（Application application）{...application.setSettingStoreFactory（new FileSettingStoreFactory（））; ...}}
  • 如果未指定任何内容，则将在常规RAP部署中使用WorkbenchFileSettingStoreFactory类。在仅RWT部署中，将使用类FileSettingStoreFactory 。

配置标记：

<！ELEMENT 扩展 （ 工厂 ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT factory EMPTY>

<！ATTLIST工厂

id CDATA #REQUIRED

class CDATA #REQUIRED>

此元素用于定义设置存储工厂。如果这些元素中的多个元素具有相同的id ，则只有最后声明的元素（按读取注册表的顺序）才被认为是有效的。


  • id - 此工厂的唯一标识符
  • class - 实现org.eclipse.rap.rwt.service.的类的完全限定名org.eclipse.rap.rwt.service.SettingStoreFactory接口。

例子：
<extension point = “org.eclipse.rap.ui.settingstores” > <factory class = “org.eclipse.rap.ui.internal.preferences。WorkbenchFileSettingStoreFactory“ id = ”file“ > </ factory> <factory class = ”org.eclipse.rap.rwt.service。FileSettingStoreFactory“ id = ”rwtfile“ > </ factory> </ extension>

API信息：
要获取设置存储，请使用RWT.getSettingStore() 。这将使用配置的org.eclipse.rap.rwt.service.SettingStoreFactory生成并返回一个合适的org.eclipse.rap.rwt.service.此会话的SettingStore实例。

希望提供自己的设置商店的开发人员必须实现SettingStore和SettingStoreFactory并使用此扩展点贡献他们的工厂。

提供的实施：
RAP提供以下默认实现：
  • WorkbenchFileSettingStoreFactory ， FileSettingStoreFactory和FileSettingStore
