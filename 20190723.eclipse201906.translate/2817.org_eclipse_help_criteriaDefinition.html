标准定义

标识符：
org.eclipse.help.criteriaDefinition

以来：
3.5

描述：
用于注册提供的帮助内容的标准定义。

配置标记：

<！ELEMENT 扩展 （ criteriaDefinition | criteriaDefinitionProvider ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT criteriaDefinition EMPTY>

<！ATTLIST标准定义

文件CDATA #REQUIRED>

通过提供XML文件做出的标准贡献


  • file - 条件定义文件的名称，其中包含条件ID列表和帮助内容的名称。

    标准定义文件的配置标记：

    <！ELEMENT标准定义（标准）*>

    <！ELEMENT标准（标准值）*>
    <！ATTLIST标准ID CDATA #REQUIRED>
    <！ATTLIST标准名称CDATA #REQUIRED>

    <！ELEMENT标准值EMPTY>
    <！ATTLIST标准值id CDATA #REQUIRED>
    <！ATTLIST标准值名称CDATA #REQUIRED>

    criteriaDefinition元素

    criteriaDefinition元素表示条件定义的实例。CriteriaDefinition由条件组成，每个条件对应一个特定的id及其名称。

    标准要素

    标准元素表示标准。每个标准可以包含与标准相关联的若干标准值。

    标准值元素

    标准值元素为标准类别提供值。它还具有与之关联的id和名称。


<！ELEMENT criteriaDefinitionProvider EMPTY>

<！ATTLIST criteriaDefinitionProvider

class CDATA #REQUIRED>

（ 自3.5起 ）通过插入代码做出的标准定义贡献


  • class - 条件定义提供程序的实现类。该类必须实现org.eclipse.help。AbstractCriteriaDefinitionProvider接口。

例子：
以下是使用条件定义扩展点的示例。

（在文件plugin.xml中 ）

<extension point = “org.eclipse.help.criteriaDefinition” > <criteriaDefinition file = “criteria.xml” /> </ extension>

（在文件criteria.xml中 ）

<criteriaDefinition>
<criterion id =“version”name =“Version”>
<criterion-value id =“1.0”name =“1.0”/>
<criterion-value id =“2.0”name =“2.0”/>
</标准>
<criterion id =“product”name =“Product”>
<criterion-value id =“DB2”name =“DB2”/>
<criterion-value id =“WAS”name =“WAS”/>
</标准>
</ criteriaDefinition>

国际化可以翻译标准定义XML文件，并将生成的副本放在nl / <language> / <country>或nl / <language>目录中。<language>和<country>代表语言环境代码中使用的双字母语言和国家/地区代码。例如，繁体中文翻译应放在nl / zh / TW目录中。nl / <language> / <country>目录的优先级高于nl / <language>。只有在nl / <language> / <country>中找不到文件时，才会使用驻留在nl / <language>中的文件。最后将搜索插件的根目录。

API信息：
org.eclipse.help的一个实现。如果使用criteriaDefinitionProvider，则必须提供AbstractCriteriaDefinitionProvider 。

提供的实施：
Eclipse平台提供的帮助系统UI的默认实现完全支持criteriaDefinition扩展点。
