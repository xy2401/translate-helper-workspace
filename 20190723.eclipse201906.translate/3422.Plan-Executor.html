计划执行人

PlanExecutor是一个简单的工作流引擎，可以执行计划。该计划由运营部门分开。PlanExecutor的另一个输入是Initializer接口实现之一。Initializer应初始化Fragment对象，并能够存储阶段和操作的额外信息。在处理共享对象的执行操作期间，称为fragment。进程的输出是片段。

建筑模块

分段

该片段代表执行者正在操作的人工制品。在CPS模型生成器中 ，片段指的是构建的CPS实例模型。该片段作为类型参数引入，以确保操作可以在片段对象上以类型安全的方式调用方法。

初始化

初始化程序负责在执行开始时创建初始片段对象。此方法允许创建初始化对象，而无需在需要之前执行可能的昂贵处理来创建片段。

interface Initializer <FragmentType> {def FragmentType getInitialFragment（）; }

手术

为给定的片段类型定义操作，并作为阶段的一部分执行。可以在多个阶段中使用相同的操作实现。该操作在执行时接收片段对象作为参数。

interface IOperation <FragmentType> {def boolean execute（FragmentType fragment）; }

相

阶段表示独立执行步骤，用于定义相同片段类型的操作列表。在组装操作时，阶段接收片段对象作为参数，这意味着操作列表可以取决于片段的当前状态。

interface IPhase <FragmentType> {def Iterable <IOperation <FragmentType >> getOperations（FragmentType fragment）; }

计划

通过添加已由相同片段准备的阶段来创建计划。在执行期间，执行者请求阶段列表。

interface IPlan <FragmentType> {def void addPhase（IPhase <FragmentType> phase）; def Iterable <IPhase <FragmentType >> getPhases（）; }

MyPhase ：

class MyPhase实现IPhase <MyFragment> {override getOperations（MyFragment fragment）{Lists.newArrayList（new AddSecretOperation（），new EncryptStoreOperation（），）; }}

AddSecretOperation ：

class AddSecretOperation实现IOperation <MyFragment> {覆盖执行（MyFragment fragment）{fragment.addSecretInformation（“我的秘密信息”）;返回true; }}

class EncryptStoreOperation实现IOperation <MyFragment> {覆盖执行（MyFragment fragment）{fragment.encryptStore;返回true; }}
