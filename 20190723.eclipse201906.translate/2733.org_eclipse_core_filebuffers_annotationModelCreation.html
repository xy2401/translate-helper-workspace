注释模型创建

标识符：
org.eclipse.core.filebuffers.annotationModelCreation

以来：
3.0

描述：
此扩展点用于自定义此插件的默认文本文件缓冲区管理器的注释模型创建行为。它允许指定应使用哪个注释模型工厂，以便创建为特定文件内容类型，文件扩展名或文件名创建的文本文件缓冲区的注释模型实例。

配置标记：

<！ELEMENT 扩展 （ 工厂 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT factory EMPTY>

<！ATTLIST工厂

class CDATA #REQUIRED

扩展CDATA #IMPLIED

fileNames CDATA #IMPLIED

contentTypeId CDATA #IMPLIED>

注释模型工厂的规范。为了找到给定文件的工厂，按以下顺序查询每个工厂规范的属性：contentTypeId，fileNames，extensions。如果为给定文件找到多个同样特定的工厂规格，则不指定使用哪个工厂。


  • class - 工厂实现类的完全限定名称。该类必须实现org.eclipse.core.filebuffers.IAnnotationModelFactory接口。
  • extensions - 以逗号分隔的文件扩展名列表，应使用此工厂
  • fileNames - 以逗号分隔的文件名列表，应使用此工厂
  • contentTypeId - 应该使用此工厂的org.eclipse.core.runtime.contentTypes扩展点定义的内容类型的id

例子：

   <extension
      point="org.eclipse.core.filebuffers.annotationModelCreation">
      <factory
            extensions="xzy"
            class="org.eclipse.ui.texteditor.ResourceMarkerAnnotationModelFactory">
      </factory>
   </extension>

API信息：
注释模型工厂必须实现org.eclipse.core.filebuffers。IAnnotationModelFactory。
