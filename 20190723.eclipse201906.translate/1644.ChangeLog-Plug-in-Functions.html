ChangeLog插件功能

ChangeLog插件有两个功能： 插入ChangeLog和准备ChangeLog 。

第一次调用特定项目的任一功能时，将出现“ 文件夹选择”菜单。此菜单允许您选择保存该项目的 ChangeLog文件的位置。

选择保存 ChangeLog文件的位置后，Eclipse将在工作区编辑器中打开它。然后，ChangeLog插件将生成适当的预编译条目，供您进一步编辑。保存后可以关闭 ChangeLog文件;当您再次调用Insert ChangeLog或Prepare ChangeLog函数时，Eclipse将使用相同的 ChangeLog文件。

以下部分更详细地描述了这两个功能。

插入ChangeLog

此函数为编辑器中当前选定的文件生成单个ChangeLog条目。要使用此功能，请导航到编辑 > 更改 日志条目 。或者，您也可以使用组合键 Ctrl + Alt + C

为同一项目中的另一个文件调用Insert ChangeLog函数将为该文件创建一个新的ChangeLog条目。如果文件的条目已存在，则调用该文件的Insert ChangeLog函数会将插入点放在相应的条目中。

准备ChangeLog

此函数根据您对项目及其源代码控制存储库所做的本地更改之间的差异生成完整的ChangeLog条目。因此，使用此功能需要访问项目的源代码控制存储库。截至今天，支持Subversion和CVS项目。

Prepare ChangeLog函数生成的ChangeLog条目将包括新文件，删除的文件和修改。要调用此函数，首先选择要将更改写入的ChangeLog文件，然后导航到Project > Prepare ChangeLog 。或者，您也可以使用组合键 Ctrl + Alt + p 。

格式化ChangeLog

此函数根据所选的格式策略格式化ChangeLog编辑器中的条目。格式化策略在Window > Preferences > ChangeLog > Formatters列表中指定。目前，只有GNU格式随ChangeLog功能一起提供，但可以提供其他策略。

要调用格式，请选择一段文本，然后右键单击并选择Format ChangeLog，或者使用组合键Esc Ctrl +' f '。请注意Esc之后的空格，这意味着您不会同时按下所有三个键。只需要将最后两个键组合在一起。按下Esc后，您会看到一个对话框弹出窗口，其中包含以相同方式启动的各种组合键。您可以使用鼠标选择合适的一个，选择Format ChangeLog时按Enter键 ，或继续并键入Ctrl +' f '。
