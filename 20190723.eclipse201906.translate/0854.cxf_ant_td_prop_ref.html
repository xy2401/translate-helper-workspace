自上而下的CXF Web服务Ant任务属性文件参考

cxf_tdjava.properties文件用于在创建CXF Web服务时将数据传递到Web服务Ant任务。此文件显示可用选项。

参数

ScenarioType
选项： 服务
这是一个必需元素，它告诉Ant任务是创建Web服务还是Web服务客户端。
InitialSelection
输入URI到WSDL文件。此URI可以是工作空间相对URI或指向远程URI。例如/dynamicWebProjectName/WebContent/myService.wsdl或http：// localhost：8080 / dynamicWebProjectName / services / MyServicePort？wsdl
ListRuntimes
选项： true false
如果选择true ，则在运行Ant任务时将提供有效运行时选项列表。
清单服务器
选项： true false
如果选择true ，则在运行Ant任务时将提供有效服务器选项列表。
服务。RuntimeId
这是CXF Web服务运行时的ID。不应修改此参数。
服务。SERVERID
选项：
  • org.eclipse.jst.server.tomcat.x （其中x是60或70 ）
  • org.eclipse.jst.server.generic.jbossx （其中x是42或5 ）
这是目标服务器的ID。删除或注释掉此参数，以使Ant任务根据Dynamic Web项目的服务器设置选择正确的值。
ServiceProjectName
默认情况下，服务项目被赋予从中创建它的WSDL文件的名称。使用此参数可以使用其他名称创建服务项目。
ServiceEarProjectName
默认情况下，服务EAR项目被赋予从中创建它的WSDL文件的名称。使用此参数可以使用其他名称创建服务EAR项目。
映射
输入包含映射的属性文件的工作空间相对URI。
映射用于覆盖WSDL名称空间以生成生成的Java类的包名称映射。属性文件的内容必须采用wsdl-namespace = package格式。您需要转义属性文件中的一些特殊字符。例如， http：// someNamespace = somePackage应为http \：// someNamespace = somePackage 。否则，冒号（:)将被视为分隔符，导致尝试将http映射到// someNamespace = somePackage 。
JavaOutput
为生成的Java代码输入现有Java源文件夹的工作空间相对URI。
服务名称
指定要为其生成代码的WSDL服务的名称。缺省情况是为WSDL文档中的每个服务生成代码
GenServer
选项： true false
如果为true ，则将生成使用JAX-WS API的服务器主线。
GenImpl
选项： true false
如果为true ，则将生成实现对象的起始点代码。
默认值
选项： true false
如果为true ，则将在实现和客户端中生成默认值。如果未生成实现或客户端，则忽略
ExtSOAPHeaders
选项： true false
如果将启用扩展soap头消息绑定的真正处理。
DefaultNamespace的
选项： true false
如果将启用默认命名空间包名称映射的真实加载。
DefaultExcludesNamespace
选项： true false
如果为true ，则将启用默认排除名称空间映射的加载。
AutoNameResolution
选项： true false
如果启用了真正的自动名称解析。
XJC Args：指定要直接传递给XJC的参数
xjc_dv
选项： true false
如果将使用其默认值初始化true字段。
xjc_ts
选项： true false
如果为true，则将toString（）方法添加到生成的类中。
xjc_ts_multi
选项： true false
如果为true，则toString（）方法将生成多行输出。
xjc_ts_simple
选项： true false
如果为true，则toString（）方法将生成单行简洁输出。
xjc_locator
选项： true false
如果为true ，则将启用生成代码的源位置支持。
xjc_sync_methods
选项： true false
如果使用'synchronized'关键字生成真正的访问器方法。
xjc_mark_generated
选项： true false
如果为true，则会将@Generated注释添加到生成的代码中。
绑定
绑定文件。逗号分隔JAXWS或JAXB绑定文件的本地文件系统中的绝对路径字符串。
