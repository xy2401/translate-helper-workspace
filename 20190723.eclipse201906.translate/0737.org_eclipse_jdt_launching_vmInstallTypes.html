Java VM安装类型

标识符：
org.eclipse.jdt.launching.vmInstallTypes

描述：
此扩展点表示不同类型的Java运行时环境和开发工具包。每个扩展必须实现org.eclipse.jdt.launching.IVMInstallType 。IVMInstallType负责创建和管理其相应IVMInstall类的一组实例。通过创建不同的IVMInstall对象， IVMInstallType允许各种Java VM的特定行为。用于管理IVMInstall的UI由Java Debug UI插件提供。

配置标记：

<！ELEMENT 扩展 （ vmInstallType *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT vmInstallType EMPTY>

<！ATTLIST vmInstallType

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 可用于引用此IVMInstallType的唯一标识符。
  • class - 实现此VM安装类型的类。该类必须实现IVMInstallType。

例子：
以下是J9 VM的IVMInstallType示例：


 <extension point="org.eclipse.jdt.launching.vmInstallTypes">
  <vmInstallType
   class="org.eclipse.jdt.internal.launching.j9.J9VMInstallType"
   id="org.eclipse.jdt.internal.launching.j9.J9Type">
  </vmInstallType>
 </extension>

提供的实施：
提供了IVMInstall和IVMInstallType的抽象实现。Java开发工具为标准1.1。*级JRE定义VM安装类型，并为符合标准命令行选项（1.2,1.3,1.4,5.0,6.0和7.0级JRE）的JRE安装类型。此外，还为执行环境描述文件定义的JRE提供了安装类型。
