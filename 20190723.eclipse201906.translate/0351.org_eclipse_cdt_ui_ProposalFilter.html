完成提案过滤器

标识符：
org.eclipse.cdt.ui。ProposalFilter

以来：
CDT 3.1

描述：
当用户请求标识符前缀的代码完成时，则从各种来源收集提议。通常会收集几个相同完成的提案。因此，在收集之后，有必要过滤提议并以最佳顺序向用户呈现它们。什么是“最佳”的概念可能因应用程序而异。该扩展点允许在任何给定情况下最适合用户需求的完成过滤器的贡献。关于哪个过滤器实际将被使用的决定仍然是用户可以从所有提供的过滤器列表中选择他的偏好（菜单窗口 - 首选项 - C / C ++ - 编辑器 - 内容辅助）。

配置标记：

<！ELEMENT 扩展 （ ProposalFilter ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT ProposalFilter EMPTY>

<！ATTLIST ProposalFilter

id IDREF #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - ProposalFilter所需的唯一ID
  • name - ProposalFilter的必需名称。此名称将显示在首选项组合中，因此应为用户提供信息。
  • class - 必须实现org.eclipse.cdt.ui.text.contentassist的实现类所需的完全限定名称。IProposalFilter

例子：
请参阅org.eclipse.cdt.internal.ui.text.contentassist中的默认实现。以DefaultProposalFilter为例。

API信息：
ProposalFilters必须实现接口org.eclipse.cdt.ui.text.contentassist.IProposalFilter .

提供的实施：
org.eclipse.cdt.internal.ui.text.contentassist中的默认实现。DefaultProposalFilter是默认的过滤方法，只要不更改用户首选项，就会使用该方法。
