BreakpointExtension

标识符：
org.eclipse.cdt.debug.core。BreakpointExtension

描述：
此扩展点定义了一种机制，用于定义C断点的调试模型特定扩展。

配置标记：

<！ELEMENT 扩展 （ breakpointExtension *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT breakpointExtension EMPTY>

<！ATTLIST breakpointExtension

id CDATA #REQUIRED

markerType CDATA #REQUIRED

class CDATA #REQUIRED

debugModelId CDATA #REQUIRED>

  • id - 指定此断点扩展类型的唯一标识符。
  • markerType - 指定此扩展适用的断点的相应标记定义的完全限定标识符（id）
  • class - 指定实现ICBreakpointExtension的Java类的完全限定名称。
  • debugModelId - 指定此扩展适用的调试模型的完全限定标识符（id）

例子：
以下是断点扩展扩展点的示例。 


 <extension point="org.eclipse.cdt.debug.BreakpointExtension">
  <breakpointExtension
   id="com.example.ExampleBreakpointExtension"
   markerType="com.example.ExampleBreakpointMarker"
   debugModeId="com.example.debug"
   class="com.example.BreakpointExtensionImpl">
  </breakpointExtension>
 </extension>
在上面的示例中，指定类型的断点扩展由类“com.example”实现。BreakpointExtensionImpl”。此扩展将适用于标记扩展为“com.example”的断点。ExampleBreakpointMarker“，并调试ID为”com.example.debug“的模型。

API信息：
属性类的值必须是实现接口org.eclipse.cdt.debug.core.model的Java类的完全限定名称。ICBreakpointExtension ，它为构造函数提供类型为org.eclipse.cdt.debug.core.model的单个参数。ICBreakpoint 。
