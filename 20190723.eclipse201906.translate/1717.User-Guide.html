概观

欢迎使用SystemTap插件帮助页面。除了简要介绍SystemTap插件的新用户结构和SystemTap的简要说明之外，本节旨在为用户提供有关SystemTap脚本语言本身的信息和链接。

SystemTap插件

SystemTap插件旨在帮助您创建，编辑和运行SystemTap脚本。它还提供了绘制脚本输出图并远程运行SystemTap脚本的功能。

它提供SystemTap IDE透视图。在此透视图中，您应该能够看到Probe视图，Function视图和Project Explorer。您还应该能够创建SystemTap脚本。

SystemTap的

SystemTap提供免费软件（GPL）基础架构，以简化有关正在运行的Linux内核的信息收集。这有助于诊断性能或功能问题。SystemTap使开发人员无需经历繁琐且具有破坏性的工具，重新编译，安装和重新启动可能需要收集数据的序列。

SystemTap提供了一个简单的命令行界面和脚本语言，用于为实时运行的内核编写工具。内部tapset库以及已发布的样本可用于帮助重用和抽象。

目前的项目成员包括Red Hat，IBM，Intel和Hitachi。

安装

要求

硬件要求：

  • x86，x86_64，ppc64，s390

软件要求：

  • Linux内核v2.6.14或更新版本
    • Linux内核开发和debuginfo包
  • Eclipse v3.8 / v4.2或更高版本
  • SystemTap v0.7或更高版本

安装所有依赖项后，安装Eclipse的SystemTap插件的最简单方法是通过“ 软件更新”和“附加组件”菜单。有关如何使用此菜单的信息，请参阅此链接 。

内核源位置

为了使用内核源代码浏览器，您需要先将其指向内核源代码。默认情况下，SystemTap IDE将使用默认的内核源路径进行设置，但可能需要手动设置它。通过在浏览器窗格中选择内核源浏览器或转到W indow-> P references ，然后转到SystemTap-> IDE - >'Kernel Source Path'选项卡来执行此操作。可以在本地计算机或用于运行SystemTap脚本的同一台计算机中浏览内核源代码。输入您的选项并通知内核源位置。如果选择了远程选项，则需要在SystemTap - >“远程服务器”选项卡上设置远程计算机信息。
