编译对象团队程序

Object Teams程序使用扩展Java构建器 自动和增量编译。

编译器接受的语言在OTJLD （OT / J语言定义）中定义，OTJLD与OTDT捆绑在一起。

编译器检测到的问题被分类为警告或错误 。
警告的存在并不禁止执行该程序;某些警告仅表示不推荐的编程样式，其他警告可能会报告可能不安全的情况，以致在运行时可能会发生未声明的异常。
编译时错误（由OT / J语言规范指定）意味着字节代码生成不完整（甚至可能运行具有编译错误的程序，但程序一旦尝试执行错误的方法就会终止）。

配置诊断

对于许多问题，您可以指定是否希望OT / J编译器将它们报告为警告，错误或忽略它们。要更改默认设置，请使用窗口>首选项> Java>编译器（OT / J）首选项页面（或更改项目属性以指定项目特定设置）。
为便于参考，在首选项页面上，每组诊断程序都提到语言定义中定义手头问题的段落。

另请注意，可以使用快速修复程序修复许多诊断。
