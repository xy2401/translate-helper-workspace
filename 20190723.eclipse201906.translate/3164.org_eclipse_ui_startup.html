启动

标识符：
org.eclipse.ui.startup

以来：
2.0

描述：
此扩展点用于注册要在启动时激活的插件。作为startup元素的属性给出的类必须实现接口org.eclipse.ui.IStartup 。启动工作台后，将从单独的线程调用earlyStartup（）方法。如果startup元素具有class属性，则将实例化该类，并在结果上调用earlyStartup（）。否则，将在插件类上调用此方法。不要将插件类指定为class属性的值，否则它将被实例化两次（一次通过常规插件激活，一次通过此机制）。如果扩展没有在启动元素上提供类作为属性，则插件的激活器（插件类）必须实现org.eclipse.ui.IStartup 。请注意，此表单已弃用，不应再使用。它的功能依赖于org.eclipse.core.runtime.compatibility插件和org.eclipse.core.runtime.compatibility.registry片段的可用性。在工作台首选项中列出了为此扩展点提供扩展的插件，用户可以在早期启动时禁用任何插件。

配置标记：

<！ELEMENT 扩展 （ 启动 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 启动 EMPTY>

<！ATTLIST启动

class CDATA #IMPLIED>

  • class - 实现org.eclipse.ui.IStartup 。如果未指定，则使用插件类。不要将插件类指定为显式值，否则它将被实例化两次（一次通过常规插件激活，一次通过此机制）。自3.0发布以来。

例子：
以下是启动扩展的示例：

   <extension point="org.eclipse.ui.startup">
     <startup class="org.eclipse.example.StartupClass"/>
   </extension>

API信息：
请参阅接口org.eclipse.ui.IStartup 。
