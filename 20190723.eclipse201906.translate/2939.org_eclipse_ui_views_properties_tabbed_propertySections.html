标识符：
org.eclipse.ui.views.properties.tabbed.propertySections

描述：
描述要在选项卡属性表页面中提供的选项卡中显示的部分列表。

配置标记：

<！ELEMENT 扩展 （ propertySections ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT propertySections （ propertySection +）>

<！ATTLIST属性部分

contributorId CDATA #REQUIRED>

  • contributorId - 属性表页面参与者的唯一标识符。

<！ELEMENT propertySection （ input *）>

<！ATTLIST属性部分

选项卡CDATA #REQUIRED

id CDATA #REQUIRED

class CDATA #REQUIRED

afterSection CDATA #IMPLIED

过滤CDATA #IMPLIED

enableFor CDATA #IMPLIED>

  • 选项卡 - 显示此部分的选项卡。
  • id - 该部分的唯一ID。
  • class - 实现该部分的类，即扩展org.eclipse.ui.views.properties.tabbed。AbstractPropertySection
  • afterSection - 当选项卡中有多个部分时，部分按afterSection属性排序。
  • filter - 实现节过滤器的类，即实现org.eclipse.jface.viewers。IFilter 。过滤器用于覆盖类型映射器和输入以进行选择。指定过滤器时，将忽略类型映射器和输入。
  • enableFor - 一个值，指示启用该部分必须满足的选择计数。如果指定且不满足条件，则不显示该部分。如果省略，则部分启用不受影响。目前支持以下属性格式：
    ñ - 选择的精确数量的项目。例如：enableFor =“1”仅在选择1项时启用该部分

<！ELEMENT 输入 EMPTY>

<！ATTLIST输入

键入CDATA #REQUIRED>

  • type - 将在选项卡中的部分上启用显示的所选类或接口。

例子：

   <extension
         point="org.eclipse.ui.tests.views.properties.tabbed.propertySections">
      <propertySections contributorId="org.eclipse.ui.views.properties.tabbed.views.TestsView">
         <propertySection
               class="org.eclipse.ui.tests.views.properties.tabbed.sections.NameSection"
               enablesFor="1"
               id="propertySection1"
               tab="propertyTab1">
            <input type="org.eclipse.ui.tests.views.properties.tabbed.model.Element">
            </input>
         </propertySection>
      </propertySections>
   </extension>
