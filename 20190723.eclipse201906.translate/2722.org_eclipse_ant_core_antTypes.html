蚂蚁类型

标识符：
org.eclipse.ant.core.antTypes

描述：
允许插件定义Ant基础结构使用的任意Ant数据类型。标准的Ant基础结构允许添加任意数据类型。不幸的是，Ant Core插件不太可能在其类路径（或其任何先决条件）上具有这些数据类型所需的类。为解决此问题，客户端应定义一个插入此扩展点的扩展，并将数据类型名称映射到类。然后，Ant插件可以请求声明插件加载指定的类。

配置标记：

<！ELEMENT 扩展 （ antType *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT antType EMPTY>

<！ATTLIST antType

名称CDATA #REQUIRED

uri CDATA #IMPLIED

class CDATA #REQUIRED

库CDATA #REQUIRED

无头（真|假）

eclipseRuntime（true | false）>

  • name - 要定义的类型的名称
  • uri - 这个定义应该存在的uri。仅在使用比1.6更新的Ant时使用。
  • class - 实现数据类型的Java类的完全限定名称。通常，此类必须是org.apache.tools.ant.types.的子类org.apache.tools.ant.types.DataType 。
  • library - 相对于包含该类型的库的插件安装位置的路径。
  • headless - 表示此类型是否适合在“无头”Ant环境中使用。如果无头运行且属性为false ，则不会加载或定义类型。同样，插件类加载器不会作为父类加载器添加到Ant类加载器中。未指定时，隐含值为true 。
  • eclipseRuntime - 指示此类型是否需要Eclipse运行时（即必须在与Eclipse相同的VM中运行）。未指定时，隐含值为true 。

例子：
以下是Ant类型扩展点的示例：

<extension point = “org.eclipse.ant.core.antTypes” > <antType name = “coolType” class = “com.example。CoolType“ library = ”lib / antSupport.jar“ /> </ extension>
