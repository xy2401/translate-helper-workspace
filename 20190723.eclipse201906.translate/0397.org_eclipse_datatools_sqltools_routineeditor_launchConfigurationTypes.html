例行启动配置类型

标识符：
org.eclipse.datatools.sqltools.routineeditor.launchConfigurationTypes

描述：
此扩展点提供了一种可配置机制，用于在“运行”以外的模式下启动SQL过程对象。每个启动配置类型都有一个名称，一个或多个模式（已经处理了运行模式），并指定了一个负责启动应用程序实现的委托。

配置标记：

<！ELEMENT 扩展 （ launchConfigurationType *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT launchConfigurationType EMPTY>

<！ATTLIST launchConfigurationType

id CDATA #REQUIRED

委托CDATA #REQUIRED

模式CDATA #REQUIRED

名称CDATA #REQUIRED>

  • id - 指定此启动配置类型的唯一标识符。
  • delegate - 指定实现ILaunchConfigurationDelegate的Java类的完全限定名称。启动此类型的配置实例将委派给此类的实例以执行启动。
  • modes - 指定这种类型的lauch配置支持的模式的逗号分隔列表 - “debug”。
  • name - 为此类型的启动配置指定一个人类可读的名称。

例子：
以下是启动配置类型扩展点的示例：

 
<extension point=
"org.eclipse.datatools.sqltools.routineeditor.launchConfigurationTypes"
>

  
<launchConfigurationType
   id=
"com.example.ExampleIdentifier"

   delegate=
"com.example.ExampleLaunchConfigurationDelegate"

   modes=
"debug"

   name=
"Example Application"
>

  
</launchConfigurationType>

 
</extension>
在上面的示例中，指定类型的启动配置支持调试模式。

API信息：
属性delegate的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。ILaunchConfigurationDelegate 。
