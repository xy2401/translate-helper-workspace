3.12.5. Editing Test Cases

From the Test Case Editor, you can edit a Test Case. You can:

  • add other Test Cases (Section 3.10.4, “Adding items to editors”).

  • add and edit Test Steps (Section 3.18.2, “Specifying Test Steps”).

  • add and edit conditions and loops (Section 3.13.1, “Overview of Conditional Statements and Loops”).

  • rename the Test Case (Section 3.9.2, “Renaming items in browsers”) or the Test Cases it contains (Section 3.10.6, “Renaming items in editors”).

  • add a comment to the Test Case (Section 3.10.7, “Adding comments to items in editors”).

  • add or edit data for the Test Case (Section 3.14.11, “Using the Data Sets View to enter data loops”).

  • extract Test Cases from within this Test Case to make reusable modules (Section 3.10.12, “Extracting Test Cases from editors: Refactoring”).

  • deactivate specific Test Cases in this Test Case so they are ignored during execution (Section 3.10.10, “Commenting out items in editors”).

  • replace one or more Test Cases in it with another Test Case (Section 3.10.13, “Replacing Test Cases in editors: Refactoring”).
