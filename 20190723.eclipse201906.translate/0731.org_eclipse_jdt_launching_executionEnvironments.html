执行环境

标识符：
org.eclipse.jdt.launching.executionEnvironments

以来：
3.2

描述：
允许执行环境和执行环境分析器。执行环境代表一种JRE - 例如J2SE5。分析人员根据环境对JRE进行分类。

配置标记：

<！ELEMENT 扩展 （ 环境 *， 分析器 *， ruleParticipant *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 环境 EMPTY>

<！ATTLIST环境

id CDATA #REQUIRED

描述CDATA #IMPLIED

ruleParticipant CDATA #IMPLIED

profileProperties CDATA #IMPLIED

合规CDATA #IMPLIED>

  • id - 呈现给用户的此执行环境的唯一标识符 - 例如，“J2SE-1.4”。
  • description - 此执行环境的简短的人类可读描述
  • ruleParticipant - 实现org.eclipse.jdt.launching.environments.IAccessRuleParticipant 。指定时，此类用于在类路径解析期间为与此执行环境关联的VM库创建访问规则。此参与者最后创建规则，在任何ruleParticipants作为元素提供之前。此属性已在3.3版本中添加。
  • profileProperties - 包含标准OSGi配置文件属性的属性文件的插件相对路径。属性为工具提供有关环境的额外信息 - 例如，系统包以及在使用此环境进行编译时要使用的默认源和目标级别设置。此属性已在3.5版本中添加。最低限度，应指定以下属性。请注意，提供的任何“org.eclipse.jdt.core.compiler。*”属性都将用作针对此执行环境构建的项目的默认编译器设置。
    • org.osgi.framework.system.packages - 可用（API）系统包的逗号分隔列表
    • org.osgi.framework.bootdelegation - 一个逗号的Spearated包列表，Framework必须将类加载委托给引导类路径
    • org.osgi.framework.executionenvironment - 此环境提供的以逗号分隔的执行环境列表（此环境的子集环境列表，包括此环境）
    • osgi.java.profile.name - 环境标识符/名称
    • org.eclipse.jdt.core.compiler.compliance - 编译器合规性级别 - 1.4,1.5等
    • org.eclipse.jdt.core.compiler.source - 源级 - 1.3,1.4等
    • org.eclipse.jdt.core.compiler.codegen.targetPlatform - 目标等级 - 1.2,1.3等
    • org.eclipse.jdt.core.compiler.problem.assertIdentifier - 在源代码中使用'assert'作为标识符的严重性：错误，警告或忽略
    • org.eclipse.jdt.core.compiler.problem.enumIdentifier - 在源代码中使用“enum”作为标识符的严重性：错误，警告或忽略
  • 合规性 - 如果没有profileProperties，这将为org.eclipse.jdt.core.compiler.compliance提供值。

<！ELEMENT 分析仪 EMPTY>

<！ATTLIST分析仪

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 此执行环境分析器的唯一标识符
  • class - 实现org.eclipse.jdt.launching.environments.IExecutionEnvironmentAnalyzer

<！ELEMENT ruleParticipant EMPTY>

<！ATTLIST ruleParticipant

id CDATA #REQUIRED

class CDATA #REQUIRED>

这个元素在3.3版本中添加了。


  • id - 此规则参与者的唯一标识符
  • class - 实现org.eclipse.jdt.launching.environments.IAccessRuleParticipant 。指定时，此类用于在类路径解析期间为与此执行环境关联的VM库创建访问规则。此参与者在ruleParticipant直接由执行环境贡献之前创建规则，允许参与者覆盖环境的默认规则。

例子：
以下是执行环境和分析器的示例定义。 


<extension point="org.eclipse.jdt.launching.executionEnvironments">
      <environment
            id="J2SE-1.4"
            description="Java 2 Platform, Standard Edition 1.4"/>
      <analyzer
            class="com.example.ExecutionEnvironmentAnalyzer"
            id="com.example.eeAnalyzer"/>
</extension>
