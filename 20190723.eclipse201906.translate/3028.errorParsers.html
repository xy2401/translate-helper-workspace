识别编译器错误：Cray，PGI和Open64

  • 配置错误分析程序
  • 识别Cray编译器优化/ Loopmark信息

构建C / C ++或Fortran应用程序时，编译器的输出（包括任何错误消息）将显示在“控制台”视图中。但是，CDT / Photran可以“识别”来自许多流行编译器的错误和警告消息，将问题描述放在Problems视图中，并用源图标标记源文件中的相应行。

默认情况下，CDT不会识别来自Cray，PGI或Open64 C / C ++编译器的错误或警告消息。但是，安装PTP时可以这样做。

配置错误分析程序

要识别来自Cray，PGI或Open64 C / C ++编译器的错误和警告：

  1. 在Project Explorer视图中，右键单击C / C ++或Fortran项目。
  2. 在上下文菜单中，选择“属性”。这将打开“ 项目属性”对话框。
  3. 在左侧的树中，导航到C / C ++ Build> Settings 。
  4. 在对话框的右侧站点中，选择“错误分析程序”选项卡。请注意，该列表包括以下条目：
    • CDT Cray C / C ++错误分析器
    • CDT PGI C / C ++错误分析器
    • CDT Open64 C / C ++错误分析器
  5. 选择与您使用的或将来可能用于编译项目的所有C / C ++和Fortran编译器相对应的错误解析器。
  6. 单击“ 确定”按钮以关闭“ 项目属性”对话框。


识别Cray编译器优化/ Loopmark信息

Cray C / C ++和Fortran编译器可以生成有关执行或未执行的编译器优化的信息性消息。当启用Cray错误解析器时（请参阅上面的配置错误解析器 ），Eclipse还将识别这些信息性消息，标记相应的源代码行。

要指示Cray C / C ++或Fortran编译器生成信息性消息，请按如下所示更改Makefile：

  • 使用-h msgs或-h negmsgs选项进行编译。例如：
        cc -O3 -h nsgmsgs my_program.c
  • OR，编译使用-h表= m和猫产生的.lst文件。例如：
        cc -O3 -h list = m my_program.c && cat my_program.lst
