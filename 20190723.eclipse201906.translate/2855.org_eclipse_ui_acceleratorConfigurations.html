标识符：
org.eclipse.ui.acceleratorConfigurations

以来：
2.0

描述：

警告：此扩展点已弃用。
请勿使用此扩展点，将在本产品的未来版本中将其删除。相反，请使用扩展点org.eclipse.ui.commands

此扩展点用于注册加速器配置扩展。加速器配置是可以注册加速器集的配置。工作台允许用户从Workbench首选项页面中选择加速器配置。一次只能激活一个加速器配置。

加速器配置表示Workbench操作的加速器的一般样式或主题。例如，Workbench提供“Emacs”加速器配置。当“Emacs”加速器配置处于活动状态时，属于注册到“Emacs”配置的加速器组的加速器处于活动状态。这些加速器被定义为模仿Emacs中的加速器（开发人员中的流行文本编辑器）。

加速器集通过将配置的id列为其“configurationId”属性的值来注册加速器配置（请参阅加速器集扩展点）。许多加速器组可以注册到相同的加速器配置。

请注意，呈现给用户的加速器配置名称与org.eclipse.ui.acceleratorConfigurations扩展点的扩展元素的属性“name”的值相同。

配置标记：

<！ELEMENT 扩展 （ acceleratorConfiguration *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

不推荐使用acceleratorConfiguration元素

<！ELEMENT acceleratorConfiguration EMPTY>

<！ATTLIST加速器配置

id CDATA #REQUIRED

名称CDATA #REQUIRED

说明CDATA #REQUIRED>

  • id - 可用于标识此加速器配置的唯一名称。
  • name - 要呈现给用户的加速器配置的可翻译名称。
  • description - 加速器配置的简短描述。

例子：
以下是加速器配置扩展的示例：


   <extension point="org.eclipse.ui.acceleratorConfigurations"> 
      <acceleratorConfiguration 
        id="org.eclipse.ui.viAcceleratorConfiguration" 
        name="VI" 
        description="VI style accelerator configuration"> 
      </acceleratorConfiguration> 
      <acceleratorConfiguration 
        id="org.eclipse.ui.jonDoeAcceleratorConfiguration" 
        name="Jon Doe" 
        description="Personal accelerator configuration for Jon Doe"> 
      </acceleratorConfiguration> 
   </extension>

提供的实施：
工作台提供Default和Emacs加速器配置。
