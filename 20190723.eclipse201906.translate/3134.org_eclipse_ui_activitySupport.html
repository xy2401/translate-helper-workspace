活动支持

标识符：
org.eclipse.ui.activitySupport

以来：
3.1

描述：

此扩展点用于注册与活动基础架构相关的各种支持扩展。

配置标记：

<！ELEMENT 扩展 （ triggerPoint | triggerPointAdvisor | triggerPointAdvisorProductBinding | categoryImageBinding | activityImageBinding ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT triggerPoint （ 提示 *）>

<！ATTLIST triggerPoint

id CDATA #REQUIRED>

指定触发点。触发点是一个标识符，活动引擎使用该标识符来确定操作是否应该导致活动的启用。


  • id - 此触发点的唯一标识符

<！ELEMENT 提示 EMPTY>

<！ATTLIST提示

id CDATA #REQUIRED

值CDATA #REQUIRED>

提示表示可以由eclipse基础结构用于确定与主机触发点相关的活动支持的行为的一些数据。

以下提示是“众所周知的”，并且预计触发点顾问会确认它们：

互动
此触发点是否以“交互”方式提示。即：它是用户执行的操作的明确结果，例如在向导对话框中激活向导。接受的值是true和false

  • id - 此提示的唯一标识符
  • value - 此提示的值

<！ELEMENT triggerPointAdvisor EMPTY>

<！ATTLIST triggerPointAdvisor

id CDATA #REQUIRED

class CDATA #REQUIRED>

触发点顾问程序是一种策略机制，只要系统执行已禁用与其关联的活动的操作，就会咨询该策略机制。顾问有责任回答是否应该采取行动，以及是否可以开展哪些活动。


  • id - 此触发点顾问的唯一标识符
  • class - 实现org.eclipse.ui.activities.ITriggerPointAdvisor接口。

<！ELEMENT triggerPointAdvisorProductBinding EMPTY>

<！ATTLIST triggerPointAdvisorProductBinding

productId CDATA #REQUIRED

triggerPointAdvisorId IDREF #REQUIRED>

指定RAP商标和顾问程序之间的绑定。这些绑定确定哪个顾问适合当前的RAP标记（由品牌扩展的id属性定义）。


  • productId - 品牌扩展的唯一ID。要将顾问程序绑定到内置品牌，请使用值“ org.eclipse.rap.rwt.branding.default ”。
  • triggerPointAdvisorId - 触发点顾问的唯一ID

<！ELEMENT categoryImageBinding EMPTY>

<！ATTLIST categoryImageBinding

id IDREF #REQUIRED

icon CDATA #REQUIRED>

该元素允许将图标绑定到类别。这些图标可以由希望以某种方式可视化类别的用户界面组件使用。


  • id - 要将图标绑定到的类别的ID
  • icon - 将用于此类别的图标的名称

<！ELEMENT activityImageBinding EMPTY>

<！ATTLIST activityImageBinding

id IDREF #REQUIRED

icon CDATA #REQUIRED>

此元素允许将图标绑定到活动。这些图标可以由希望以某种方式可视化活动的用户界面组件使用。


  • id - 要将图标绑定到的活动的ID
  • icon - 将用于此活动的图标的名称

例子：
以下是非交互式触发点的示例：


  <extension point="org.eclipse.ui.activitySupport">
     <triggerPoint
        id="com.example.xyz.myTriggerPoint">
        <hint id="interactive" 
              value="false" />
     </triggerPoint>
  </extension>
以下是绑定到默认RAP标记的触发点顾问程序的示例，从而替换默认的WorkbenchTriggerPointAdvisor ：


  <extension point="org.eclipse.ui.activitySupport">
     <triggerPointAdvisor
        id="com.example.xyz.myTriggerPointAdvisor" 
        class="com.example.xyz.AdvisorImpl"/>
     <triggerPointAdvisorProductBinding 
      productId="org.eclipse.rap.rwt.branding.default"
      triggerPointAdvisorId="com.example.xyz.myTriggerPointAdvisor" />
  </extension>
以下是绑定到特定RAP品牌的触发点顾问程序的示例：


  <extension
         point="org.eclipse.rap.ui.branding">
      <branding
            favicon="icons/eclipse.gif"
            id="example.activity.branding1"
            servletName="rap2"
            title="Activity Example"/>
  </extension>
  <extension point="org.eclipse.ui.activitySupport">
     <triggerPointAdvisor
        id="com.example.xyz.myTriggerPointAdvisor" 
        class="com.example.xyz.AdvisorImpl"/>
     <triggerPointAdvisorProductBinding 
      productId="example.activity.branding1"
      triggerPointAdvisorId="com.example.xyz.myTriggerPointAdvisor" />
  </extension>
以下是将图像绑定到活动和类别的示例：


  <extension point="org.eclipse.ui.activitySupport">
    <activityImageBinding
       id="some.activity.id"
       icon="icons/someIcon.gif"/>    
    <categoryImageBinding
       id="some.category.id"
       icon="icons/someIcon.gif"/>
  </extension>

API信息：
triggerPointAdvisor标记的class属性的值必须是实现org.eclipse.ui.activities.ITriggerPointAdvisor 。

提供的实施：
工作台实现（ org.eclipse.ui.activities.WorkbenchTriggerPointAdvisor ）可供客户端子类化和重用。
