Http服务资源

标识符：
org.eclipse.equinox.http.registry.resources

以来：
1.0

描述：
此扩展模仿并借用了OSGi HttpService registerResource方法的语义。

配置标记：

<！ELEMENT 扩展 （ serviceSelector ？， resource +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 资源 EMPTY>

<！ATTLIST资源

别名CDATA #REQUIRED

基名CDATA #IMPLIED

httpcontextId CDATA #IMPLIED>

  • 别名 - 相当于registerResource中的OSGi Http Services“alias”。
  • base-name - 与registerResource中的OSGi Http Services“name”等效。
  • httpcontextId - 与registerResource中的OSGi Http Services httpcontext等效。httpcontextId与匹配id的“httpcontexts”扩展名匹配。如果未提供此属性，则使用OSGi Http Service的DefaultHttpContext语义。

<！ELEMENT serviceSelector EMPTY>

<！ATTLIST serviceSelector

过滤CDATA #IMPLIED

class CDATA #IMPLIED>

serviceSelector允许过滤用于注册资源的HttpService实例


  • filter - serviceSelector允许过滤用于注册资源的HttpService实例。
  • class - 如果使用，则该类必须是org.osgi.framework的实例。过滤

例子：
以下是resources扩展的示例：

     <extension
           id="testResource"
           point="org.eclipse.equinox.http.registry.resources">
        <resource
              alias="/testresource"
              base-name="/test"
              httpcontextId="testintest"/>
     </extension>
这相当于OSGi Http Service的registerResource方法。
