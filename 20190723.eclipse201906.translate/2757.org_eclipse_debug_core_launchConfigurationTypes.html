启动配置类型

标识符：
org.eclipse.debug.core.launchConfigurationTypes

描述：
此扩展点为启动应用程序提供了可配置的机制。每个启动配置类型都有一个名称，支持一种或多种模式（运行和/或调试），并指定负责启动应用程序实现的委托。从3.2开始，类型还可以定义迁移委托，该委托可用于设置项目与启动它们的启动配置之间的资源映射。

配置标记：

<！ELEMENT 扩展 （ launchConfigurationType +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

  • id - 指定此启动配置类型的唯一标识符。
  • delegate - 指定实现ILaunchConfigurationDelegate的Java类的完全限定名称。启动此类型的配置实例将委派给此类的实例以执行启动。也可以通过launchDelegates扩展为启动launchDelegates提供启动配置类型。由launchConfigurationType扩展指定的启动委托负责在同一扩展指定的所有模式下启动。要扩展现有配置类型的启动功能，或者当不同的启动模式组合需要不同的委托时，应使用launchDelegates扩展来提供启动委托。
  • modes - 指定这种类型的lauch配置支持的模式的逗号分隔列表 - “run”和/或“debug”。也可以通过launchDelegates扩展为启动配置类型提供启动模式。
  • name - 为此类型的启动配置指定一个人类可读的名称。
  • public - 指定用户是否可以访问此启动配置类型。如果未指定，则默认为true 。
  • category - 一个可选属性，指定此启动配置类型的类别。默认值为unspecified（ null ）。类别是客户定义的。此属性已在2.1版本中添加。
  • sourcePathComputerId - sourcePathComputer扩展的唯一标识符，用于计算此类型的启动配置的默认源查找路径。这个引用在3.0版本中添加。从3.3开始， sourcePathComputerId也可以通过launchDelegates扩展点指定。
  • sourceLocatorId - sourceLocator扩展的唯一标识符，用于为使用此类型的启动配置启动的会话创建源定位器。这个引用在3.0版本中添加。从3.3开始， sourceLocatorId也可以通过launchDelegates扩展点指定。
  • migrationDelegate - 指定实现org.eclipse.debug.core.的Java类的完全限定名称org.eclipse.debug.core.ILaunchConfigurationMigrationDelegate 。从3.2开始，用于迁移此类型的启动配置以与当前工具兼容的可选委托
  • delegateName - 关联的启动委托的可读名称 - 例如，“Eclipse JDT Launcher”。当多个启动器可用于启动配置且用户必须选择首选启动器时，将向用户显示此名称。此属性已在3.3版本中添加。
  • delegateDescription - 此类型的启动委托的描述 - 例如“Eclipse JDT Java Application Launcher支持运行和调试本地Java应用程序”。当多个启动器可用于启动配置并且用户必须选择首选启动器时，将向用户呈现此描述。此属性已在3.3版本中添加。
  • allowPrototypes - 指定此启动配置类型是否允许原型。如果未指定，则默认为false 。此属性已在4.7版本中添加。
  • allowCommandLine - 指定此启动配置类型是否允许命令行。如果未指定，则默认为false 。此属性已在4.7版本中添加。

不推荐使用fileExtension元素

<！ELEMENT fileExtension EMPTY>

<！ATTLIST fileExtension

扩展CDATA #REQUIRED

default（true | false）>

不再使用此元素。而是使用org.eclipse.debug.ui.launchShortcuts扩展点。


  • extension - 指定此启动配置类型可用于的文件扩展名。
  • default - 指定此启动配置类型是否应为指定文件扩展名的默认启动配置类型。如果未指定，则默认为false 。

例子：

以下是启动配置类型扩展点的示例：

<extension point = “org.eclipse.debug.core.launchConfigurationTypes” > <launchConfigurationType id = “com.example。ExampleIdentifier“ delegate = ”com.example。ExampleLaunchConfigurationDelegate“ modes = ”run，debug“ name = ”示例应用程序“ migrationDelegate = ”com.example.migrationDelegate“ sourceLocatorId = ”com.example.sourceLookupDirector“ sourcePathComputerId = ”com.example.sourcePathComputer“ delegateName = ”示例应用程序启动工具“ delegateDescription = “此示例工具将运行或调试示例代码。” > </ launchConfigurationType> </ extension>

在上面的示例中，指定类型的启动配置支持运行和调试模式。指定的类型还具有关联的迁移委托，源定位器标识，源路径计算机以及启动委托名称和描述。

API信息：

属性delegate的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。ILaunchConfigurationDelegate 。migrationDelegate属性的值必须是实现org.eclipse.debug.core的Java类的完全限定名称。ILaunchConfigurationMigrationDelegate 。
