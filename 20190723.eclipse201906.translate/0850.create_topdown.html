使用Apache CXF从WSDL文档生成SEI和框架实现bean

先决条件：

  1. 安装和配置Apache CXF运行时环境。有关详细信息，请参阅“ 要求和安装”页面。
  2. 创建动态Web项目并为Apache CXF Web服务开发启用它，如下所述： 为CXF Web服务创建项目 。
  3. 创建WSDL文档或将其导入Dynamic Web Project的Web Content文件夹。
使用Apache CXF运行时环境从WSDL文档创建SEI和框架实现bean：
  1. 选择从顶级菜单中选择Window> Open Perspective> Java EE切换到Java EE透视图。
  2. 在Project Explorer视图中，选择您创建或导入到Dynamic Web Projects Web Content文件夹中的WSDL文档。
  3. 选择打开Web服务向导 文件>新建>其他...> Web服务>顶级菜单中的Web服务 。
  4. Web Services页面：选择自上而下的Java bean Web Service作为Web服务类型，并浏览以选择从中生成SEI和实现bean的WSDL文档：

    1. 将服务滑块移动到您希望完成的Web服务开发阶段。
      • 开发：使用首选项默认值和向导中收集的信息调用Apache CXF wsdl2java工具，以生成带注释的服务端点接口和支持类。该向导还会生成JAX-WS端点配置文件。
      • 汇编：如果目标服务器需要，此阶段将承载Web服务的项目组装并配置到EAR中。
      • 部署：生成Web服务的部署工件。
      • 安装：在目标服务器上安装和配置Web模块和EAR。
      • 开始：一旦安装了Web服务，就启动目标服务器。
      • 测试：使用Web服务资源管理器测试生成的Web服务。
    2. 服务器：选择要将Web服务部署到的服务器。
    3. Web服务运行时：选择Apache CXF 2.x.
    4. 服务项目：在工作区中选择Apache CXF将生成代码的项目。
    5. 如果要为Web服务生成客户端，请选择Java Proxy作为客户端类型：
      • 将客户端滑块移动到Develop客户端阶段。
      • 客户端项目：选择要生成客户端代码的其他项目。如果项目不存在，向导将为您创建项目，并使用CXF Facet进行配置。
    6. 发布Web服务：启动Web Services Explorer以将Web服务发布到UDDI注册中心。
    7. 监视Web服务：使用TCP / IP Monitor监视Web服务的SOAP流量。
  5. 单击Next>转到第一个Apache CXF wsdl2java配置页面：
    1. 输出目录：指定生成代码的服务项目中的源文件夹。
    2. 包名称：键入要用于生成的代码的Java包名称。
    3. 将WSDL命名空间指定为包名称映射：可选择指定Java包名称的WSDL命名空间映射。
    4. 服务名称：指定从中生成代码的WSDL服务的名称。缺省情况是为WSDL文档中的每个服务生成代码。
    5. 绑定文件：指定一个或多个JAX-WS或JAXB绑定文件。
  6. 单击Next>转到第二个Apache CXF wsdl2java配置页面，该页面允许您覆盖wsdl2java默认首选项：

    确保选中Generate Implementation复选框，然后单击Next>

  7. 测试Web服务页面。如果第一页上的服务滑块设置为测试服务，将显示此页面。

    选择启动按钮以使用Web Services Explorer测试Web Service。

  8. Apache CXF JAX-WS客户端配置页面：如果Web服务页面上的客户端滑块设置为“ 开发客户端”，则将显示此页面。

    有关此页面上显示的选项的信息，请参阅“ 创建JAX-WS Web服务客户端”页面。

  9. Web服务发布页面：选择是否要将此Web服务发布到UDDI注册中心。单击完成
