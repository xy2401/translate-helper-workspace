RWT可主题小部件

标识符：
org.eclipse.rap.ui.themeableWidgets

以来：
RAP 1.0

描述：
此扩展点用于注册也支持主题的自定义RWT小部件。为了支持主题，窗口小部件提供了几个与主题相关的文件。这些文件必须符合命名约定。在使用此扩展点注册自定义窗口小部件后，可以按名称找到与主题相关的文件。因此，只能注册自定义窗口小部件类。

自定义窗口小部件必须在包中提供与主题相关的文件，并以模式<package>.internal.<widget>kit命名，其中<package>是包名称， <widget>是自定义的小写类名称控制。这是包含生命周期适配器（LCA）的相同包。

主题相关文件包括：

  • 主题定义文件，名称为<Widget>.theme.xml 。此文件定义新的主题键。
  • 一个实现ControlThemeAdapter主题适配器类，名为<Widget>ThemeAdapter 。主题适配器为控件提供依赖于主题的属性的当前值。
  • 名为<Widget>.appearances.js的外观片段文件。此文件定义了新的qooxdoo外观。
这些文件都不是必需的，找到的文件都是受到尊重的，但丢失的文件不会受到损害。

配置标记：

<！ELEMENT 扩展 （ 小部件 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 小部件 EMPTY>

<！ATTLIST小部件

id CDATA #REQUIRED

名称CDATA #IMPLIED

class CDATA #REQUIRED>

  • id - 标识此可主题窗口小部件的唯一名称
  • name - 可配置小部件的可选名称
  • class - 实现自定义窗口小部件的类的名称

例子：
以下是可主题窗口小部件扩展定义的示例：
<extension point = “org.eclipse.rap.ui.themeableWidgets” > <widget id = “my.custom.datepicker” class = “my.custom.controls。DatePicker“ > </ widget> </ extension>
