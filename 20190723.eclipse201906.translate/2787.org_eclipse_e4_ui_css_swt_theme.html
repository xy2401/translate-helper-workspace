主题

标识符：
org.eclipse.e4.ui.css.swt.theme

以来：
0.9

描述：
此扩展点提供了为已定义主题定义主题和/或提供样式表和资源的可能性

配置标记：

<！ELEMENT 扩展 （ stylesheet *， theme *， themeAssociation *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 样式表 （ themeid *， osgiresourcelocator *）>

<！ATTLIST样式表

uri CDATA #REQUIRED

os CDATA #IMPLIED

是CDATA #IMPLIED>

为主题提供样式表


  • uri - 将相对uri绑定到样式表
  • os - 样式适用的操作系统或空的适用于所有的操作系统
  • ws - 风格适用的窗口系统或适用于所有人的空白

<！ELEMENT 主题 EMPTY>

<！ATTLIST主题

id CDATA #REQUIRED

标签CDATA #REQUIRED

basestylesheeturi CDATA #REQUIRED

os CDATA #IMPLIED

是CDATA #IMPLIED

os_version CDATA #IMPLIED>

  • id - 主题的ID。可能多次使用相同的id，但它必须具有不同的OS和WS过滤器值
  • label - 显示给用户时使用的标签
  • basestylesheeturi - 相对于包的基本样式表uri
  • os - 主题适用的操作系统或空的适用于所有操作系统
  • ws - 主题适用的窗口系统或适用于所有人的空白
  • os_version - 主题适用的操作系统版本，或者适用于所有操作系统的版本。该值可以是逗号分隔的字符串，以指定多个版本。

<！ELEMENT 题材 EMPTY>

<！ATTLIST主题

refid IDREF #IMPLIED>

  • refid - 引用的样式表

<！ELEMENT osgiresourcelocator EMPTY>

<！ATTLIST osgiresourcelocator

uri CDATA #REQUIRED>

  • uri - 以platform：/ plugin开头的url

<！ELEMENT themeAssociation EMPTY>

<！ATTLIST themeAssociation

themeId IDREF #REQUIRED

colorAndFontId IDREF #REQUIRED

os_version IDREF #IMPLIED>

  • themeId - Eclipse CSS主题ID
  • colorAndFontId - Eclipse遗留的颜色和字体主题ID
  • os_version - 用于更好地匹配主题的可选属性

例子：

一个仅适用于Win32 / WinXP的主题：


<theme
  basestylesheeturi="css/e4_default_winxp_blu.css"
  id="org.eclipse.e4.ui.css.theme.e4_default"
  label="e4DefaultWinXPBlue"
  os="win32"
  ws="winxp">
</theme>

样式表为所有系统做出了贡献：


<stylesheet
  uri="css/default.css">
</stylesheet>

CSS主题和传统的颜色和字体关联：


<themeAssociation
 themeId="org.eclipse.e4.ui.css.theme.e4_classic"
 colorAndFontId="org.eclipse.ui.defaultTheme"> 
</themeassociation>
