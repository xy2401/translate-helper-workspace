命令图像

标识符：
org.eclipse.ui.commandImages

以来：
3.2

描述：

图像扩展点提供了将不同类型的图像和图标链接到特定命令的方式。无论命令出现在用户界面中的哪个位置，都可以使用这些图像。为了适应一些边界情况（例如，工具栏下拉项），可以将特定命令的图像分组为“样式”。然后，当要显示命令时，可以特别请求这些样式。

使用org.eclipse.ui.menus扩展点放置在菜单中的命令将使用默认图像（如果可用），放置在工具栏中的命令将使用toolbar样式图像（如果可用），后跟默认图像。

配置标记：

<！ELEMENT 扩展 （ 图像 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 图像 EMPTY>

<！ATTLIST图片

commandId IDREF #REQUIRED

icon CDATA #REQUIRED

disabledIcon CDATA #IMPLIED

hoverIcon CDATA #IMPLIED

风格CDATA #IMPLIED>

要与特定命令关联的图像。


  • commandId - 与图像关联的命令的标识符。
  • icon - 应在默认情况下使用的图标的路径，或ISharedImages常量。当图标启用且鼠标没有悬停在图标上时，将显示此信息。它也将用于没有这种更具体的图标的任何情况。
  • disabledIcon - 禁用命令时显示的图标。
  • hoverIcon - 用户将鼠标悬停在命令上时显示的图标。
  • style - 此图标块适用的特定方案。例如，当命令显示为工具栏下拉项时，可能需要具有不同的图标。目前，org.eclipse.ui.menus扩展将使用菜单图标的默认样式（无样式）和toolbar栏贡献的toolbar样式。如果该命令没有可用的toolbar条目，则它使用默认样式。

例子：

<extension
 point="org.eclipse.ui.commandImages">
 <image
  commandId="org.eclipse.example.ProfileLast"
  hoverIcon="icons/full/etool16/profile.png"
  disabledIcon="icons/full/dtool16/profile.png"
  icon="icons/full/etool16/profile.png" />
 <image
  commandId="org.eclipse.example.ProfileLast"
  hoverIcon="icons/full/etool16/history.png"
  disabledIcon="icons/full/dtool16/history.png"
  icon="icons/full/etool16/history.png"
  style="toolbar" />
</extension>

API信息：

在工作台中，可以使用org.eclipse.ui.commands.获取命令的图像org.eclipse.ui.commands.ICommandImageService接口。可以从支持的工作台对象（例如IWorkbench本身， IWorkbenchWindow或IWorkbenchPartSite检索此接口。要检索服务，您可以调用workbench.getService(ICommandImageService.class) 。
