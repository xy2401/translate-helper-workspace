启动快捷方式

标识符：
org.eclipse.pde.ui.launchShortcuts

以来：
3.3

描述：

此扩展点用于在PDE编辑器中注册启动快捷方式。启动快捷方式显示在插件清单编辑器的“概述”页面的“测试”部分中。这允许客户端在编辑器中显示其自定义启动配置的快捷方式，或者为现有Eclipse Application和OSGi Framework启动配置提供配置文件模式的缩略图。

配置标记：

<！ELEMENT 扩展 （ 快捷键 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 快捷键 EMPTY>

<！ATTLIST快捷方式

id IDREF #REQUIRED

标签CDATA #REQUIRED

模式（运行|调试|配置文件）

osgi（true | false）

projectSpecific（true | false）>

  • id - 相应的org.eclipse.debug.ui.launchShortcuts扩展名的ID，该扩展名应在单击链接时启动。
  • label - 要在编辑器中显示的人类可读标签。
  • mode - 启动快捷方式的模式。允许的值是run ， debug和profile 。
  • osgi - 如果设置为true ，则仅当插件清单编辑器的表示设置为仅显示与OSGi相关的部分时才显示该链接，即。当扩展名和扩展点页面不存在时。如果未指定，则假定该值为false 。
  • projectSpecific - 如果为true ，则此启动快捷方式仅显示在通过IBundleProjectDescription关联的项目中。如果未指定，则假定此值为false 。此属性已添加到3.6版本中，以支持Manifest编辑器中的自定义启动快捷方式。

例子：

以下是扩展点的示例：


   <extension
         point="org.eclipse.pde.ui.launchShortcuts">
      <shortcut
            id="org.eclipse.pde.ui.runtimeWorkbenchShortcut"
            label="Launch an Eclipse Runtime" 
            mode="run"
            osgi="false"/>
   </extension>

API信息：

属性id的值必须是已定义的org.eclipse.debug.ui.launchShortcuts扩展的id。匹配的org.eclipse.debug.ui.launchShortcuts扩展中指定的类将被实例化，并将调用其launch(IEditorPart, String)以启动。

属性mode的值必须run ， debug或profile 。

提供的实施：

PDE提供快捷方式扩展以启动Eclipse应用程序和OSGi框架。
