导入跟踪点数据

“Trace Compass用户指南”中的本节中的某些信息是多余的。有关更多详细信息，请参阅“ 参考”部分中的“跟踪罗盘用户指南”条目。

创建跟踪项目

在Project Explorer视图中，右键单击并从上下文菜单中选择New > Project ....在New Project对话框中，选择Tracing > Tracing Project ，单击Next ，为项目命名，然后单击Finish 。

导入GDB跟踪

在跟踪项目中，右键单击Traces文件夹并选择Import ...。浏览或输入源目录。在树中选择跟踪文件。（可选）将跟踪类型设置为GDB：GDB Trace 。单击完成 。

或者，可以从任何外部文件管理器将跟踪拖放到Traces文件夹中。

选择GDB跟踪类型

右键单击Traces文件夹中导入的跟踪，然后从上下文菜单中选择Select Trace Type ... > GDB > GDB Trace 。如果在导入时选择或检测到跟踪类型，则可以省略此步骤。

跟踪将使用GDB图标进行更新 。

选择跟踪可执行文件

必须标识创建跟踪点数据的可执行文件，以便可以正确启动C / C ++ Postmortem Debugger。

右键单击Traces文件夹中的GDB跟踪，然后从上下文菜单中选择Select Trace Executable ....浏览或输入可执行文件的路径，然后按确定 。

GDB必须将所选文件识别为可执行文件。
