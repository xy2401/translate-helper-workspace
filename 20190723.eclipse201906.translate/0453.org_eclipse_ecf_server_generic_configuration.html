ECF通用服务器配置

标识符：
org.eclipse.ecf.server.generic.configuration

以来：
1.0.0

描述：
此扩展点允许通过扩展点定义配置ECF通用服务器实例。

配置标记：

<！ELEMENT 扩展 （ 连接器 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT 连接器 （ 组 +）>

<！ATTLIST连接器

主机名CDATA #IMPLIED

端口CDATA #IMPLIED

keepAlive CDATA #IMPLIED

发现（真|假）

>

连接器定义ECF通用服务器的端口和其他信息，以接收来自客户端的连接请求。


  • hostname - 定义服务器ID中预期的主机名...例如“localhost”。如果未指定，则默认为“localhost”。
  • port - 要侦听的连接器的端口（例如3282）。如果未指定，则设置为3282。
  • keepAlive - 与ECF通用服务器故障检测器关联的超时（以毫秒为单位）。如果未指定，则默认为30000（30秒）。
  • discovery - 发现标志确定是否通过IDiscoveryService发布给定服务以进行发现

<！ELEMENT group EMPTY>

<！ATTLIST集团

名称CDATA #REQUIRED

>

父连接器的特定命名组。例如ecftcp：// localhost：3282 / server中的'server'


  • name - ID中组的名称：例如ecftcp：// localhost：3282 / server中的“server”

例子：
以下是org.eclipse.ecf.server.generic.configration扩展点的示例用法：

   <extension
         point="org.eclipse.ecf.server.generic.configuration">
      <connector
            hostname="localhost"
            keepAlive="30000"
            port="3282">
         <group
               name="server">
         </group>
      </connector>
   </extension>
使用groupID生成服务器：ecftcp：// localhost：3333 / groupName

API信息：
没有公共程序化API
