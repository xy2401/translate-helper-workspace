本地分析PHP脚本

此过程描述如何使用内部PHP可执行文件从工作空间分析PHP脚本。

  1. 单击“配置文件”按钮旁边的箭头在工具栏上，选择配置文件配置... - 或 - 从主菜单转到运行|配置文件配置... - 或者在Project Explorer视图中右键单击并选择Profile As |配置文件配置...
  2. 将出现配置文件启动配置对话框。
  3. 双击“ PHP CLI应用程序”选项以创建新的配置文件配置。

  4. 输入新配置的名称。
  5. 从以下选项中选择：
    • 工作区默认PHP - 选择使用工作区/项目默认PHP
    • 执行环境 - 从下拉列表中选择要用于调试配置的执行环境。
    • 备用PHP - 选择使用另一个PHP，可以从已安装的PHP列表中选择。
  6. 在PHP文件文本字段中输入您的PHP文件，或单击浏览并选择您的文件。
  7. 如有必要，可以在“ PHP脚本参数”选项卡中添加参数以模拟命令行输入。
  8. 如果不需要有关代码覆盖的信息，请选择“ Profiler”选项卡并取消选中“ 显示代码覆盖率”选项。

  9. 单击Apply ，然后单击Profile 。
  10. 将显示一个确认对话框，询问您是否要打开性能分析透视图。单击是。（如果您希望将来默认打开性能分析透视图，请选中“记住我的决定”复选框。）

PHP Profile Perspective将打开，显示Profiling Monitor窗口，其中包含各种Profiling视图。

有关配置文件会话运行后显示的信息的更多信息，请参阅PHP配置文件透视图 。
