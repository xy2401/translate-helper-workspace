结构创作者

标识符：
org.eclipse.compare.structureCreators

描述：
此扩展点允许插件为特定内容类型注册结构创建者。期望结构创建者为给定内容创建IStructureComparator树。该树用作结构比较的输入。扩展点必须实现org.eclipse.compare.structuremergeviewer接口。IStructureCreator 。

配置标记：

<！ELEMENT 扩展 （ structureCreator *， contentTypeBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT structureCreator EMPTY>

<！ATTLIST structureCreator

id CDATA #REQUIRED

class CDATA #REQUIRED

扩展CDATA #IMPLIED>

  • id - 可用于引用结构创建者的唯一标识符
  • class - 实现org.eclipse.compare.structuremergeviewer的类的完全限定名称。IStructureCreator
  • extensions - 以逗号分隔的文件扩展名列表，例如“java，properties”

<！ELEMENT contentTypeBinding EMPTY>

<！ATTLIST contentTypeBinding

contentTypeId IDREF #REQUIRED

structureCreatorId IDREF #REQUIRED>

contentTypeBinding将结构创建者绑定到内容类型。


  • contentTypeId - 使用org.eclipse.core.contenttype.contentTypes扩展点定义的内容类型的id。
  • structureCreatorId - 使用此扩展点的structureCreator元素定义的结构创建者的id（即org.eclipse.compare.structureCreators ）

例子：
以下是java文件的结构创建者示例（扩展名“java”）：

<extension point = “org.eclipse.compare.structureCreators” > <structureCreator id = “org.eclipse.compare。JavaStructureCreator“ class = ”org.eclipse.compare。JavaStructureCreator“ extensions = ”java“ /> </ extension>

API信息：
贡献的类必须实现org.eclipse.compare.structuremergeviewer.IStructureCreator

提供的实施：
Compare UI插件定义了zip存档的结构创建者。
