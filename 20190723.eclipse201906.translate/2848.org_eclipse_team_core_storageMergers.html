存储合并

标识符：
org.eclipse.team.core.storageMergers

以来：
3.2

描述：
此扩展点允许插件为特定内容类型注册存储合并。期望存储合并在三个输入存储上执行三向合并，并将结果写入输出流。扩展点必须实现接口org.eclipse.team.core.mapping。IStorageMerger 。

配置标记：

<！ELEMENT 扩展 （ storageMerger *， contentTypeBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT storageMerger EMPTY>

<！ATTLIST storageMerger

id CDATA #REQUIRED

class CDATA #REQUIRED

扩展CDATA #IMPLIED>

  • id - 可用于引用存储合并的唯一标识符
  • class - 实现org.eclipse.team.core.mapping的类的完全限定名称。IStorageMerger
  • extensions - 以逗号分隔的文件扩展名列表，例如“java，properties”

<！ELEMENT contentTypeBinding EMPTY>

<！ATTLIST contentTypeBinding

contentTypeId IDREF #REQUIRED

storageMergerId IDREF #REQUIRED>

  • contentTypeId - 使用org.eclipse.core.contenttype.contentTypes扩展点定义的内容类型的id。
  • storageMergerId -

例子：
以下是属性文件的存储合并示例（扩展名“properties”）：

<extension point = “org.eclipse.team.core.storageMergers” > <storageMerger id = “org.eclipse.team.internal.core.merge。PropertiesStorageMerger“ class = ”org.eclipse.team.internal.core.merge。PropertiesStorageMerger“ extensions = ”properties“ /> </ extension>

API信息：
贡献的类必须实现org.eclipse.team.core.mapping.IStorageMerger

提供的实施：
Team UI插件为面向行的文本文件定义存储合并。
