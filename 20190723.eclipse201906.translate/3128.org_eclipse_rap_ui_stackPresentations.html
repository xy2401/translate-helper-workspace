StackPresentations

标识符：
org.eclipse.rap.ui.stackPresentations

以来：
RAP 1.2

描述：
此扩展点用于注册ConfigurableStack 。此对象扩展了普通的StackPresentation因此可用于设置LayoutPart样式。要注册这样的ConfigurableStack只有在PresentationFactory是org.eclipse.rap.presentation.才会发生org.eclipse.rap.presentation.PresentationFactory 。因为只有这个PresentationFactory才能处理ConfigurableStack 。
此扩展点还用于将ConfigurationAction耦合到此类ConfigurableStack 。有了这个，您可以为不同的ConfigurableStack实现实现不同的配置行为。
您可以使用org.eclipse.rap.ui.branding扩展点为零件定义默认的StackPresentations。

配置标记：

<！ELEMENT 扩展 （ stackPresentation +）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT stackPresentation EMPTY>

<！ATTLIST stackPresentation

id CDATA #REQUIRED

class CDATA #REQUIRED

名称CDATA #REQUIRED

type（编辑|查看| standaloneview）

actionClass CDATA #IMPLIED

actionIcon CDATA #IMPLIED

menuIcon CDATA #IMPLIED>

  • id - ConfigurableStack的标识符。这是为了加载并保存这样的零件演示文稿。
  • class - ConfigurableStack的实现
  • name - ConfigurableStack的名称，这只是为了给它一个可读的名称。
  • type - 定义可以使用ConfigurableStack的类型。有三种可能的类型：Editor，View和StandaloneView。
  • actionClass - ConfigurationAction的实现。通过此操作，您可以实现配置bahaviour，例如，此操作可以创建一个显示配置元素的弹出对话框。
  • actionIcon - ConfigurationAction的图标。
  • menuIcon - 视图菜单的图标（如果有）。

例子：
<extension point = “org.eclipse.ui.presentations。StackPresentations“ > <stackPresentation actionClass = ”org.eclipse.rap.ui.interactiondesign.example.configaction。ExampleConfigAction“ actionIcon = ”icons / configure.png“ class = ”org.eclipse.rap.ui.interactiondesign.example.stacks。NavigationPaneStackPresentation“ id = ”org.eclipse.rap.ui.interactiondesign.navigationPaneStackPresentation“ menuIcon = ”icons / menuIcon.gif“ name = ”Navigation Pane“ type = ”view“ > </ stackPresentation> </ extension>

API信息：
每个stackPresentation都必须扩展org.eclipse.rap.ui.interactiondesign.ConfigurableStack用于处理不同的机制，例如stackPresentation即时更改等等。

提供的实施：
可以在org.eclipse.rap.presentation.example项目中找到详细的实现示例。该项目定义了两个不同的ConfigurableStack实现。
