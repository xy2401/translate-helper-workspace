Context Provider委托

ContextProviderDelegate允许UI控件提供实际的帮助上下文ID并帮助搜索表达式到IContextProvider的方法，同时将这些字符串的引用作为抽象帮助键（helpKey常量）处理。

为了支持从帮助上下文ID到helpKey常量的抽象，UI控件必须：
  • 作为键适应IContextProvider.class对象
  • 创建ContextProviderDelegate的实例，传入与帮助上下文关联的插件的符号名称
  • 实现IContextProvider的静态方法（getContext，getContextChangeMask和getSearchExpression），每个方法都返回ContextProviderDelegate实例的相应方法
  • 使用IWorkbench.setHelp定义帮助上下文（例如，在createPartControl方法中），通过传递抽象的helpKey常量字符串

ContextProviderDelegate方法将实际的帮助上下文ID，更改掩码或帮助搜索表达式返回给IContextProvider方法。

注意： DTP help-helper在通过ContextProviderDelegate方法返回它们之前，处理属性文件中定义的非限定帮助上下文ID以对它们进行完全限定。
