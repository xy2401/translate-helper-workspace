结构合并查看器

标识符：
org.eclipse.compare.structureMergeViewers

描述：
此扩展点允许插件为结构内容类型注册比较/合并查看器。期望观众扩展org.eclipse.jface.viewers。观众 。但是，由于查看器没有默认构造函数，因此扩展点必须为查看器org.eclipse.compare实现工厂接口。IViewerCreator 。

配置标记：

<！ELEMENT 扩展 （ viewer *， contentTypeBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT viewer EMPTY>

<！ATTLIST观众

id CDATA #REQUIRED

class CDATA #REQUIRED

扩展CDATA #REQUIRED

标签CDATA #IMPLIED>

  • id - 可用于引用查看器的唯一标识符
  • class - 为结构合并查看器实现工厂并实现org.eclipse.compare的 类的完全限定名称。IViewerCreator
  • extensions - 以逗号分隔的文件扩展名列表，例如“zip，jar”
  • label - 自3.5起。可翻译标签，将在此查看器的UI中使用。

<！ELEMENT contentTypeBinding EMPTY>

<！ATTLIST contentTypeBinding

contentTypeId IDREF #REQUIRED

structureMergeViewerId IDREF #REQUIRED>

contentTypeBinding将结构合并查看器绑定到内容类型。


  • contentTypeId - 使用org.eclipse.core.contenttype.contentTypes扩展点定义的内容类型的id。
  • structureMergeViewerId - 使用此扩展点的viewer元素定义的结构合并查看器的id（即org.eclipse.compare.structureMergeViewers ）

例子：
以下是zip文件的比较/合并查看器示例（扩展名“zip”）：

<extension point = “org.eclipse.compare.structureMergeViewers” > <viewer id = “org.eclipse.compare。ZipCompareViewer“ class = ”org.eclipse.compare。ZipCompareViewerCreator“ extensions = ”zip“ label = ”Binary Compare“ /> </ extension>

API信息：
贡献的类必须实现org.eclipse.compare.IViewerCreator

提供的实施：
Compare UI插件定义了zip存档的结构比较查看器。
