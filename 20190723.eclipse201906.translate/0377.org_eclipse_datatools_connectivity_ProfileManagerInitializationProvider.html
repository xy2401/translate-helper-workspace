ProfileManagerInitializationProvider

标识符：
org.eclipse.datatools.connectivity。ProfileManagerInitializationProvider

以来：
DTP 1.6

描述：
此扩展点将为在启动时填充样本配置文件提供一种方法。

配置标记：

<！ELEMENT 扩展 （ 连接 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT 连接 （ 属性 *）>

<！ATTLIST连接

jarList CDATA #IMPLIED

connectionProfileName CDATA #REQUIRED

connectionProfileID CDATA #REQUIRED

driverDefinitionName CDATA #REQUIRED

driverTemplateID CDATA #REQUIRED

connectionInitializer CDATA #REQUIRED

>

  • jarList - 驱动程序定义的jar列表。
  • connectionProfileName - 生成的连接配置文件的名称。
  • connectionProfileID - 要生成的连接配置文件的ID。
  • driverDefinitionName - 生成的驱动程序定义的名称。
  • driverTemplateID - 要从中生成驱动程序定义的驱动程序模板的ID。
  • connectionInitializer - 初始化连接的类。

<！ELEMENT 属性 EMPTY>

<！ATTLIST财产

id CDATA #REQUIRED

值CDATA #REQUIRED

>

用于指定驱动程序模板所需属性的属性集合。如果属性值需要插件的状态位置的位置，请输入以下值：value =“Plugin_State_Location / sample_value”> Plugin_State_Location将替换为声明扩展的状态位置。


  • id - 驱动程序模板属性的ID。
  • value - 驱动程序模板属性的值。
