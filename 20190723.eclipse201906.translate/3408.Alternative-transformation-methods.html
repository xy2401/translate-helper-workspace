替代转型方法

我们为CPS到部署M2M转换提供了多种实现。变体在至少一个特征方面彼此不同以支持比较和评估。特点包括：

  • 执行模式（批处理或增量）

  • 二手技术（Xtend，VIATRA，其他转换工具）

  • 模型遍历与事件驱动的转换

  • 部分转换模型的执行

每个变体的详细描述可以在相应的维基页面上找到。

变种

这些实现将在存储库中的transformations文件夹下以及org.eclipse.viatra.query.examples.cps.xfrom.m2m.<id>命名空间下创建。

批量

  1. Xtend（ id = batch.simple ， details ）

  2. Xtend + VIATRA查询（ id = batch.eiq ， 详细信息 ）

  3. VIATRA Batch API（ id = batch.viatra ， 详细信息 ）

增加的

对于增量备选方案，区分因素是识别源模型中必须与目标同步的变化的方法（“脏标志”）

  1. 直接通知（ id = incr.direct ）

    • 脏：EMF适配器/ IQBase

  2. 显式可追溯性（ id = incr.expl ）

    • 脏：可追溯性模型状态

    • 细节在这里

  3. 查询结果可追溯性（ id = incr.qrt ）

    • dirty：查询结果书签，可追溯性保存在deltamonitor中

    • 细节在这里

  4. 部分批量转换（ id = incr.aggr ）

    • 脏：将查询结果视为通知/事件，低级别更改聚合（转换更大的部分）

    • 高级：独立于源模型的通知，转换可以在后台运行

    • 详细信息<< Partial-batch-M2M-transformation＃，here>

  5. 变更驱动的转换（ id = incr.cdt ）

    • 脏：EVM激活生命周期

  6. 基于VIATRA EMF的转换API （ id = incr.viatra ）

摘要

我们目前实现了上面提到的不同变体（如果您对VIATRA查询很好，您可能会认为基于LS的应该被视为一个单独的:-)）用于此转换，并且将来可能会添加一些基于我们的技术堆栈的其他变体。但是，我们希望使用其他转换工具邀请变体。由于演示者是开源的，这可能涉及其他工具的用户，甚至作为案例提交给转型工具竞赛 。
