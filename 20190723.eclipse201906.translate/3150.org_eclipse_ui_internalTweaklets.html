％ExtPoint.tweaklets

标识符：
org.eclipse.ui.internalTweaklets

以来：
3.3

描述：
此内部扩展点不被视为API。它用于定义tweaklet。tweaklet是可用于调整Workbench外观的任何东西。

配置标记：

<！ELEMENT 扩展 （ tweaklet ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT tweaklet EMPTY>

<！ATTLIST tweaklet

id CDATA #REQUIRED

名称CDATA #REQUIRED

描述CDATA #IMPLIED

定义CDATA #IMPLIED

实现CDATA #IMPLIED

icon CDATA #IMPLIED>

  • id - 将用于标识此tweaklet的唯一名称
  • name - 将在此tweaklet的UI中使用的可翻译名称
  • description - 这个tweaklet的可翻译简短描述，将在UI中使用
  • definition - 工作台中tweaklet定义的标识符，通常是完全限定的类型名称
  • 实现 - 扩展器提供的tweaklet实现的标识符，通常是完全限定的类名
  • icon - 将与tweaklet关联的图标的相对名称。

例子：
以下是一个示例tweaklet：

<extension point = “org.eclipse.ui.internalTweaklets” > <tweaklet id = “org.eclipse.ui.internal.tweaklets。ConcreteXYZTweaklet“ name = ”XYZ Tweaklet“ description = ”调整的简短描述“ definition = ”org.eclipse.ui.internal.tweaklets。AbstractXYZTweaklet“ implementation = ”org.eclipse.ui.internal.tweaklets。ConcreteXYZTweaklet“ icon = ”icons / XYZ.gif“ /> </ extension>

API信息：
通常，尽管不是必需的，但definition属性的值是抽象类的完全限定名称或工作台定义的接口，而实现属性的值是由非抽象类提供的完全限定名称。扩展插件。

提供的实施：
可以在Eclipse Platform孵化器中找到Tweaklet实现。
