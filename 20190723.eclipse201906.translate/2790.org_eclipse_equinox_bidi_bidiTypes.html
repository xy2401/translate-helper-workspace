需要特殊双向处理的结构化文本类型

标识符：
org.eclipse.equinox.bidi.bidiTypes

以来：
0.10

描述：
使用此扩展点来描述需要特殊双向处理的字符串的处理，例如文件路径，SQL语句或Java代码。

配置标记：

<！ELEMENT 扩展 （ typeDescription +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #REQUIRED

名称CDATA #IMPLIED>

  • id - 此扩展的唯一标识符

<！ELEMENT typeDescription EMPTY>

<！ATTLIST类型描述

键入CDATA #REQUIRED

class CDATA #REQUIRED

说明CDATA #IMPLIED>

  • type - 此处理器将处理的结构化文本的类型。
  • class - 为此类字符串提供专门双向处理的处理器。
  • description - 字符串类型的描述。

例子：
支持“减”分隔符的结构化文本字符串的处理器可以描述为：
public class StructuredTextSepMinus扩展StructuredTextTypeHandler {public StructuredTextSepMinus（）{super（“ - ”）; }}
