物业测试员

标识符：
org.eclipse.core.expressions.propertyTesters

以来：
3.0

描述：
此扩展点允许向现有类型添加属性。然后可以在表达式语言的测试表达式元素中使用这些属性。

配置标记：

<！ELEMENT 扩展 （ propertyTester *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT propertyTester EMPTY>

<！ATTLIST propertyTester

id CDATA #REQUIRED

键入CDATA #REQUIRED

名称空间CDATA #REQUIRED

属性CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 属性测试器的唯一标识符
  • type - 此属性测试程序要扩展的类型
  • namespace - 确定添加属性的名称空间的唯一ID
  • properties - 此属性测试程序提供的以逗号分隔的属性列表
  • class - 实现测试方法的类的名称。该类必须是公共的并扩展org.eclipse.core.expressions。PropertyTester具有公共0参数构造函数。

例子：
以下是属性测试人员贡献的示例：

<extension point = “org.eclipse.core.expressions.propertyTesters” > <propertyTester id = “org.eclipse.jdt.ui。IResourceTester“ type = ”org.eclipse.core.resources。IResource“ namespace = ”org.eclipse.jdt.ui“ properties = ”canDelete“ class = ”org.eclipse.jdt.ui.internal。ResourceTester“ > </ propertyTester> </ extension>

API信息：
贡献类必须扩展org.eclipse.core.expressions.PropertyTester
