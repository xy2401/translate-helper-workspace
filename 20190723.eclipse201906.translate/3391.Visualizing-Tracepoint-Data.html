可视化跟踪点数据

打开GDB跟踪

在Traces文件夹中，双击GDB跟踪或右键单击它，然后从上下文菜单中选择“ 打开 ”。

跟踪点数据将在事件编辑器中打开，并将启动C / C ++ Postmortem Debugger实例。

如果在工作空间中可用，则还将在C / C ++编辑器中打开与第一个跟踪记录对应的源代码。

此时，建议将事件编辑器重新定位到默认编辑器区域之外，以便C / C ++编辑器不会隐藏它。

查看跟踪数据

在“事件”编辑器中，将显示一个表，其中每个跟踪记录都有一行。“ 跟踪帧”列显示顺序跟踪记录号。Tracepoint列显示GDB在收集时为此跟踪点分配的编号。“ 文件”列显示设置跟踪点的文件名，行号和方法。“ 内容”列显示跟踪点在运行时收集的数据。

通过在列标题中输入正则表达式，可以在任何列上完成搜索和过滤。

浏览GDB跟踪

可以使用键盘或鼠标在“事件”编辑器中选择跟踪记录。将更新Debug视图中的C / C ++ Postmortem Debugger以显示当前跟踪记录的堆栈跟踪。

也可以通过单击“ 下一个跟踪记录”或“ 上一个跟踪记录”按钮从“ 跟踪控制”视图导航跟踪 。事件编辑器和调试视图将更新。
