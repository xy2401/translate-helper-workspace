创建Autotools项目

有许多方法可以创建Autotools项目。

从头开始创建新项目

第一种方法是通过CDT C和C ++项目向导，它们可以分别从文件 - >新建 - > C项目和文件 - >新建 - > C ++项目菜单项激活，这些项目在C / C ++透视图中可用。通过单击File - > New - > Project ...菜单项，然后打开那里找到的C / C ++文件夹，也可以找到这两个向导。

看看C向导，我们看到有一个GNU Autotools类别。

打开类别，有三个选项：空项目，Autotools共享库项目或Hello World Ansi C Autotools项目。Empty Project模板意味着没有文件将提供给新项目，而Hello World项目将提供使用Autotools配置构建的示例hello world程序。Autotools共享库项目是如何使用Autotools创建共享库的示例。C ++项目向导还具有相同的GNU Autotools类别和三个项目模板：空项目，Autotools共享库和Hello World C ++ Autotools示例程序。如果从头开始创建项目，建议您使用Hello World示例项目或Autotools共享库示例项目并进行修改。否则，您需要为Autotools插件创建一个configure脚本，以便在构建时调用或提供Makefile，以便配置步骤不会失败。

导入时创建新项目

导入现有Autotools项目时，也可以使用C和C ++向导。例如，使用文件 - >导入... - > CVS - >从CVS导入项目时，您可以选择“签出为新项目向导配置的项目”。

从New Project菜单中，打开C / C ++类别并选择C Project或C ++ Project，具体取决于要导入的项目类型。如果代码是C和C ++，请选择C ++ Project。

假设你有一个使用Autotools的C应用程序并选择C Project。这将打开前面显示的C Project Wizard。在这种情况下，从GNU Autotools类别中选择Empty Project，因为您已经准备好了所有配置文件。

导入后转换存在项目

创建Autotools项目的另一种方法是转换现有项目。例如，如果我们从SVN或CVS导入项目而不使用新项目向导进行配置，则只需将项目下载到工作区中。此时，无法构建或访问CDT中的各种工具。

要转换它，请在Project Explorer中选择项目，然后打开C / C ++ Perspective中的File - > New - > Convert to C / C ++ Project菜单项。如果在另一个角度来看，可以通过选择File - > New - > Other ... - > C / C ++ - > Convert to a C / C ++ Project找到同一个向导。这将打开C / C ++转换向导。

请注意，有一组项目可供转换。此列表包含任何不是C或C ++项目的项目。AC或C ++项目包括常规CDT Managed Make和Makefile项目。另请注意，向导打开时会自动选择您选择的项目。

可以选择C或C ++项目以及一组可供选择的项目类型。对于Autotools，选择GNU Autotools类别，这将转换为Autotools项目。转换后，此项目将不会显示为转换向导中的候选项目。

转换存在项目（任何）

创建Autotools项目的最后一种方法是使用Autotools Conversion向导。该向导允许用户从任何形式的项目转换为Autotools项目。显然，转换实际上不是C或C ++并使用Autotools的项目将导致未定义的行为。您需要先选择要转换的项目。选择后，转到文件 - >新建 - >其他... - > C / C ++，打开类别，选择“转换为C / C ++ Autotools项目”，然后单击下一步按钮。

这会打开以下对话框：

同样，有转换的候选人。此列表包含当前不是C或C ++ Autotool项目的所有项目。与标准C / C ++转换向导一样，您可以选择C或C ++。只需按“完成”按钮即可，但如果按下“下一步”按钮并继续，则可以根据需要选择引用的项目。

转换为Autotools项目后，没有UI方法可以转换回常规的C / C ++项目。如果需要，您需要从项目目录中手动删除.project，.cproject和.autotools文件。这将有效地删除项目的性质和CDT设置，允许您根据需要再次转换它。
