目标位置

标识符：
org.eclipse.pde.core.targetLocations

以来：
3.8

描述：
此扩展点允许客户端贡献自己的目标位置实现，这些实现为目标平台提供插件和其他内容。

配置标记：

<！ELEMENT 扩展 （ targetLocation ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT targetLocation EMPTY>

<！ATTLIST targetLocation

键入CDATA #REQUIRED

locationFactory CDATA #IMPLIED

canUpdate（true | false）>

  • type - 此目标位置实例的字符串标识符。必须与ITargetLocation实现中getType（）方法返回的类型匹配。
  • locationFactory - 实现ITargetLocationFactory工厂。用于创建此目标位置的新实例。
  • canUpdate - 指定此目标位置是否可以自行更新。如果要求更新，则提供true将通知该位置。如果为false，则不会发送更新通知。

例子：
以下是targetLocations扩展的示例：

 <extension
  id="org.eclipse.pde.core.customlocation"
  name="Custom Location"
  point="org.eclipse.pde.core.targetLocations">
  <targetLocation
   locationFactory="org.eclipse.pde.core.CustomLocationFactory"
   type="CustomLocation">
  </targetLocation>
 </extension>
在上面的示例中，目标的名称和id在plugin.xml中声明，但工厂是org.eclipse.pde.core.CustomLocationFactory来自贡献插件。

API信息：
扩展必须提供实现ITargetLocationFactory的工厂。该工厂将用于实例化ITargetLocation对象。要在PDE UI中显示实现，客户端必须提供适配器工厂，使其ITargetLocation实现适应以下内容： IWizardNode ：提供IWizardNode对象以提供用于创建此类型的新位置的向导ILabelProvider ：提供标签文本和图像UI中的位置实现有关更多信息，请参阅ITargetLocation
