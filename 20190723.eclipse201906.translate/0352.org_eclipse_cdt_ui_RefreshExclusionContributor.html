刷新排除贡献者

标识符：
org.eclipse.cdt.ui。RefreshExclusionContributor

以来：
5.3

描述：
此扩展点允许向CDT的刷新排除系统贡献新类型的刷新排除。刷新排除本质上是一些任意规则，可以针对给定资源进行测试并评估为true或false。如果应该应用排除，则排除返回true（即应排除测试的资源）。规则可以嵌套，因此可以排除排除。

配置标记：

<！ELEMENT 扩展 （ exclusionContributor ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT exclusionContributor EMPTY>

<！ATTLIST排除贡献者

class CDATA #REQUIRED

id IDREF #REQUIRED

名称CDATA #REQUIRED

isTest（true | false）>

新型刷新排除的贡献。


  • class - 与贡献相对应的Java类，负责充当创建排除的具体实例的工厂，以及提供用于修改特定于排除的属性的UI。
  • id - 标识此贡献的唯一字符串。
  • name - 用于在UI中引用此排除类型的可读名称。
  • isTest - 表示此贡献的属性用于测试目的，因此不应考虑给定的贡献者对UI的贡献。

提供的实施：
资源排除提供了一个实现的实现。
