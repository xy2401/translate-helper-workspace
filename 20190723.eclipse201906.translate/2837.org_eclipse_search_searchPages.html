搜索页面

标识符：
org.eclipse.search.searchPages

描述：
此扩展点允许插件注册搜索页面以进行专门搜索。对资源执行搜索操作时，搜索基础结构会为其找到最具体的注册搜索页面。

配置标记：

<！ELEMENT 扩展 （ 第 * 页 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 页面 EMPTY>

<！ATTLIST页面

id CDATA #REQUIRED

标签CDATA #REQUIRED

class CDATA #REQUIRED

icon CDATA #IMPLIED

sizeHint CDATA #IMPLIED

tabPosition CDATA #IMPLIED

扩展CDATA #IMPLIED

searchViewHelpContextId CDATA #IMPLIED

showScopeSection（true | false）

启用（true | false）

canSearchEnclosingProjects（true | false）>

  • id - 将用于标识此搜索页面的唯一名称
  • label - 将在搜索页面选项卡中使用的可翻译标签
  • class - 实现org.eclipse.search.ui的类的名称。ISearchPage 。我们建议继承org.eclipse.jface.dialogs。DialogPage 。
  • icon - 将用于与指定扩展名匹配的所有资源的图像的相对名称。如果省略，搜索页面的选项卡将仅包含标签。
  • sizeHint - 页面初始大小的提示。这是一个包含以逗号分隔的宽度和高度的字符串（例如“50,60”）。在省略该提示的情况下，页面将不大于其他页面。
  • tabPosition - 一个整数，用于定义页面在页面簿中的放置位置。该值不定义绝对位置，而是定义相对于其他页面的位置。页面的排序如下：
    1. 如果两个页面都没有定义标签位置，那么它们将根据标签按字母顺序排序
    2. 如果两个页面都指定了标签位置，那么具有较低值的页面将首先出现。如果值相等，则将页面视为值不存在（参见1.）
    3. 如果只有一个页面具有指定的值，则此页面首先出现
  • extensions - 带有文件扩展名的逗号分隔列表，搜索页面可以在其上运行。每个扩展名还必须包含一个权重（0表示最低权重），使搜索基础结构能够找到最合适的页面。通过结肠将重物与延伸部分隔开。如果搜索页面可以搜索所有可能的资源，则可以使用“*”。
    注意：如果有org.eclipse.search.ui.ISearchPageScoreComputer适配器用于返回更高权重的所选元素，然后将使用更高的权重。
  • searchViewHelpContextId - 搜索视图的可选帮助上下文ID，显示此页面的结果。如果缺少此属性，则使用默认搜索帮助上下文ID（ org.eclipse.search.search_view_context ）。
  • showScopeSection - 如果缺少此属性或将其设置为“false”，则“搜索”对话框中不会显示范围部分。要查看范围部分，必须将此属性设置为“true”。添加自己的搜索页面并希望查看范围部分的插件必须将其添加到其plugin.xml中。
  • enabled - 如果此属性缺失或设置为“false”，则该页面最初不会显示在“搜索”对话框中。用户可以通过“搜索”对话框中的“自定义...”按钮激活该页面。
  • canSearchEnclosingProjects - 如果此属性缺失或设置为“false”，则“Enclosing Projects”搜索范围不会显示在搜索对话框的范围部分中。如果缺少属性“showScopeSection”或将其设置为“false”，则将忽略此属性。

例子：
以下是搜索页面扩展定义的示例：

<extension point = “org.eclipse.search.searchPages” > <page id = “org.eclipse.search.ui.text。TextSearchPage“ label = ”文本搜索“ icon = ”icons / full / obj16 / tsearch_pref.gif“ sizeHint = ”250,160“ tabPosition = ”1“ extensions = ”*：1“ showScopeSection = ”true“ class = ”org.eclipse。 search.ui.text。TextSearchPage“ > </ page> </ extension>

API信息：
贡献的类必须实现org.eclipse.search.ui.ISearchPage 。

提供的实施：
搜索基础结构提供全文搜索的搜索页面。
