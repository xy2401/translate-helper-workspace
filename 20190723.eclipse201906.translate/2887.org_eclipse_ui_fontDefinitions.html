字体定义

标识符：
org.eclipse.ui.fontDefinitions

以来：
2.1

描述：
此扩展点用于向JFace FontRegistry注册字体，并使用工作台首选项存储以供字体首选项页面使用。此扩展点已在3.0中弃用。您现在应该将fontDefinition元素添加到org.eclipse.ui.themes。

配置标记：

<！ELEMENT 扩展 （ fontDefinition *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT fontDefinition （ 描述 ？）>

<！ATTLIST fontDefinition

id CDATA #REQUIRED

标签CDATA #REQUIRED

值CDATA #IMPLIED

categoryId CDATA #IMPLIED

defaultsTo CDATA #IMPLIED>

  • id - 可用于标识此字体定义的唯一名称。
  • label - 要呈现给用户的字体的可翻译名称。
  • 价值 -

    字体值。它的格式为： fontname-style-height其中fontname是字体的名称， style是字体样式（ "regular" ， "bold" ， "italic"或"bold italic" ）， height是表示字体高度的整数。

    示例： Times New Roman-bold-36 。

    只能使用一个（或两个）值或默认值。

  • categoryId - 此字体所属的演示文稿类别的可选ID。
  • defaultsTo - 另一个字体定义的id，它是接收器的默认设置。如果没有对此字体的首选项，则字体注册表将在注册表中为其设置defaultsTo值。

    只能使用一个或两个值或默认值。


<！元素描述 （#PCDATA）>

字体用法的简短描述



例子：
以下是字体定义扩展的示例：


   <extension point="org.eclipse.ui.fontDefinition"> 
      <fontDefinition 
         id="org.eclipse.examples.textFont" 
         label="Text"> 
         <description>
            The text font
         </description>
      </fontDefinition> 

      <fontDefinition 
        id="org.eclipse.examples.userFont" 
        label="User"
        defaultsTo="org.eclipse.jface.textFont"> 
        <description>
           The user font
        </description>
      </fontDefinition> 
   </extension>

API信息：
defaultsTo标记用作Workbench的指令，每当更新defaultsTo fontDefinition时，将字体定义的值设置为defaultsTo的值。仅当fontDefinition处于其默认值时才会出现这种情况 - 一旦用户设置了此更新，则不会发生此更新。工作台提供4种字体：

org.eclipse.jface.bannerfont 。横幅字体用于向导横幅。
org.eclipse.jface.dialogfont 。对话框字体是对话框中窗口小部件的字体。
org.eclipse.jface.headerfont 。标题字体用于复合文本页面中的节标题。
org.eclipse.jface.textfont 。文本字体由文本编辑器使用。

提供的实施：
工作台提供文本，对话框，横幅和标题字体的字体定义。
