复合作弊表

标识符：
org.eclipse.ui.cheatsheets.composite_schema

以来：
3.2

描述：
复合备忘单内容文件的架构定义。复合备忘单由组成任务组的一组任务组成。每个任务可以是简单的备忘单或用户贡献的任务类型。（ 机器可读的XML模式。

配置标记：

<！ELEMENT compositeCheatsheet （ taskGroup | task ）>

<！ATTLIST compositeCheatsheet

名称CDATA #REQUIRED>

复合cheatsheet的根元素


  • name - 打开复合备忘单时将以大字体显示的复合备忘单的名称。

<！ELEMENT taskGroup （（ task | taskGroup ）+， intro ？， onCompletion ？， dependsOn *）>

<！ATTLIST任务组

kind（set | sequence | choice）“set”

名称CDATA #REQUIRED

id CDATA #IMPLIED

skip（true | false）“false”>

任务组表示相关任务的集合。如果类型是“选择”，则只需要完成其中一个子任务。如果种类是“设定”或“顺序”，则必须完成所有儿童。请注意，子元素<task>，<taskGroup>，<intro>，<onCompletion>和<dependsOn>可以按任何顺序出现。<task>和<taskGroup>元素的顺序决定了它们的显示顺序，在序列的情况下，它们的顺序决定了它们的执行顺序。


  • kind - 这个任务组的类型可以是“set”，“sequence”或“choice”。如果已完成此任务组的所有子任务/任务组，则此任务组已设置或序列已完成。此外，必须按顺序完成序列的子任务。任何子任务完成后，选择即告完成。
  • name - 将在任务资源管理器中显示的此任务组的名称。
  • id - 此任务组的id，如果dependsOn元素引用此任务组，则该ID是必需的。
  • skip - 如果为true，则可以跳过这组任务。

<！ELEMENT 任务 （ 简介 ？， onCompletion ？， param *， dependsOn *）>

<！ATTLIST任务

亲切的CDATA“集”

名称CDATA #REQUIRED

id CDATA #IMPLIED

skip（true | false）“false”>

复合备忘单中的叶任务。任务没有子项，但它有一个任务编辑器，显示在备忘单视图的下/右窗格中。任务类型确定将打开哪个任务编辑器，任务类型的“cheatsheet”表示简单的备忘单。可以贡献其他任务类型。请注意，子元素<intro>，<onCompletion>，<param>和<dependsOn>可以按任何顺序出现。


  • 善良 - 任务类型。任务类型的“cheatsheet”代表一个简单的cheatsheet，其他任务类型可以使用扩展点org.eclipse.ui.cheatsheets.cheatSheetContent提供。
  • name - 将在任务资源管理器中显示的此任务的名称。
  • id - 此任务组的id，如果dependsOn元素引用此任务组，则该ID是必需的。
  • skip - 如果为true，则可以跳过此任务。

<！ELEMENT param EMPTY>

<！ATTLIST param

名称CDATA #REQUIRED

值CDATA #REQUIRED>

复合备忘单中任务的参数。每个参数都有一个名称和值，两者都是字符串。任务可以具有任意数量的参数，单个任务的两个参数可以不共享相同的名称。


  • name - 此参数的名称。kind =“cheatsheet”的任务可以包含以下参数：

    id：使用扩展点org.eclipse.ui.cheatsheets.cheatSheetContent注册的备忘单的id。这标识了与此任务相关联的备忘单。应指定id或path参数（但不是两者）。

    path：备忘单内容文件的URL。这可以是绝对URL，或相对于复合备忘单的内容文件。如果同时指定了id和path，则将使用该路径来定位内容文件，并忽略id参数。

    showIntro：布尔参数，默认值为true。如果“假”，则启动时的备忘单将首先显示第一步而不是介绍。

  • value - 此参数的值。

<！ELEMENT 简介 （#PCDATA）>

包含在此任务启动之前显示的文本。可能包含表单文本标记 。



<！ELEMENT onCompletion （#PCDATA）>

包含将在此任务的完成面板中显示的文本。可能包含表单文本标记 。



<！ELEMENT 依赖于 EMPTY>

<！ATTLIST依赖于

任务CDATA #REQUIRED>

在两个任务或子任务之间创建依赖关系。


  • task - 在启动此任务之前必须完成的任务或任务组的ID。
