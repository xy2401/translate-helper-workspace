运行和分析调试器结果

启动其中一个调试会话（PHP脚本，PHP Web页面或URL）后，您可以使用调试过程中显示的视图来控制和监视调试过程。

控制调试过程：

可以使用“调试”视图中的各种按钮来控制调试过程。

1。Debug过程将在每个断点处自动停止。

2。各种视图仅显示有关调试过程的信息。

3。您可以使用调试视图中的各个按钮来决定如何继续调试过程：

  • 单击“继续”按钮继续调试过程直到下一个断点，或者直到调试过程结束。
  • 单击“终止”按钮停止调试过程。
  • 单击“跳过” 按钮在当前正在执行的代码行中跳过下一个方法调用（不输入它）。该方法仍将执行。
  • 单击“步骤返回”按钮从已经进入的方法返回。返回时跳过的其余代码仍然执行。
  • 单击Step Into按钮在当前执行的代码行中进入下一个方法调用。
  • 单击“使用步骤过滤器”按钮更改是否应在当前的“调试视图”中使用步骤过滤器。
  • 调试过程终止后，您可以单击“删除终止启动”按钮从列表中删除任何已终止的调试会话。

在调试过程中（对于PHP脚本，PHP Web页面或URL），各种视图将提供以下信息：

  • 调试视图 - 您可以在此控制（停止，暂停和恢复）调试过程。您还可以决定是否进入，步骤或步骤返回（退出）某些功能。
  • 变量 - 将在脚本中显示各种变量。
  • 断点 - 将显示您输入的断点
  • 编辑器窗口 - 将根据在“调试视图”窗口中选择的行显示相关部分的代码。
  • 调试输出 - 将显示脚本的文本输出。这将在调试过程继续时更新。
  • 浏览器输出 - 将脚本的输出显示给浏览器。这将在调试过程继续时更新。
  • 控制台视图 - 显示任何错误和警告消息。
  • 任务视图 - 显示添加到脚本中的任务（如果适用）。

在调试过程中（对于JavaScript调试），各种视图将提供以下信息：

  • Debug View [Debug Perspective] - 您可以在此控制（停止，暂停和恢复）调试过程。您还可以决定是否进入，步骤或步骤返回（退出）某些功能。
  • 变量视图[Debug Perspective] - 将在脚本中显示各种变量。
  • 断点视图[Debug Perspective] - 将显示您输入的断点。
  • 脚本视图 - 显示可用脚本的列表。双击脚本以在编辑器中查看其代码。
  • 编辑器窗口 - 将根据在“调试视图”窗口中选择的行显示相关部分的代码。
  • 内部Web浏览器 - 在浏览器中显示您正在调试的代码。
  • 调试输出视图[Debug Perspective] - 将显示脚本的文本输出。这将在调试过程继续时更新。
  • 浏览器输出视图[Debug Perspective] - 将脚本的输出显示给浏览器。这将在调试过程继续时更新。
  • 控制台视图 - 显示任何错误和警告消息。
  • 任务视图 - 显示添加到脚本中的任务（如果适用）。

注意：

有关将在调试期间显示的视图的更多信息，请参阅“ PHP Debug Perspective ”。
