Http服务过滤器

标识符：
org.eclipse.equinox.http.registry.servlets

以来：
1.1

描述：
此扩展模仿并借用了Equinox Extended HttpService registerFilter方法的语义。

配置标记：

<！ELEMENT 扩展 （ serviceSelector ？， filter +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 过滤器 （ init-param *）>

<！ATTLIST过滤器

class CDATA #REQUIRED

别名CDATA #REQUIRED

httpcontextId CDATA #IMPLIED

load-on-startup（true | false）>

  • class - 相当于registerFilter中Equinox Extended Http Service的“filter”。Filter由Extension Point提供程序实例化，并且必须是javax.servlet的实例。过滤
  • 别名 - 相当于registerFilter中的Equinox Extended Http Services“alias”。
  • httpcontextId - 相当于registerFilter中的Equinox Extended Http Services httpcontext。httpcontextId与匹配id的“httpcontexts”扩展名匹配。如果未提供此属性，则使用OSGi Http Service的DefaultHttpContext语义。
  • load-on-startup - 确定是否应立即初始化Filter。

<！ELEMENT init-param EMPTY>

<！ATTLIST init-param

名称CDATA #REQUIRED

值CDATA #REQUIRED>

类似于Servlet规范及其部署描述符的init-param元素。


  • name - 初始参数的名称
  • value - 初始参数的值

<！ELEMENT serviceSelector EMPTY>

<！ATTLIST serviceSelector

过滤CDATA #IMPLIED

class CDATA #IMPLIED>

  • filter - serviceSelector允许过滤用于注册过滤器的HttpService实例。
  • class - 如果使用，则该类必须是org.osgi.framework的实例。过滤

例子：
以下是filters扩展的示例：

     <extension
           id="testFilter"
           point="org.eclipse.equinox.http.registry.filters">
        <filter
              alias="/sp_test"
              class="org.eclipse.equinox.http.registry.internal.TestFilter">
           <init-param
                 name="testParam"
                 value="test param value">
           </init-param>
        </filter>
     </extension>
这相当于Equinox Extended Http Service的registerFilter方法。

API信息：
过滤器必须是javax.servlet的一个实例。过滤
