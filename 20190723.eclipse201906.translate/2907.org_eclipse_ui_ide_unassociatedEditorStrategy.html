无关联的编辑策略

标识符：
org.eclipse.ui.ide.unassociatedEditorStrategy

以来：
3.12

描述：

此扩展点允许插入不同的策略以支持IDE内的无关联文件。策略可以是强制特定编辑器，要求用户选择编辑器，轮询远程注册表以查找编辑器......

用户和扩展程序可以通过将相关的org.eclipse.ui.ide.unassociatedEditorStrategy首选项设置为其最喜欢的策略的ID来选择要使用的策略。
默认策略是检查系统编辑器，如果找不到相关编辑器，则打开Eclipse文本编辑器。

配置标记：

<！ELEMENT 扩展 （ 策略 ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 策略 EMPTY>

<！ATTLIST战略

id CDATA #REQUIRED

class CDATA #REQUIRED

标签CDATA #REQUIRED

interactive（true | false）>

  • id - 将用于标识此无关联编辑器策略的唯一ID
  • class - 实现接口org.eclipse.ui.ide的完全限定类名。IUnassociatedEditorStrategy
  • label - 描述策略的简短标签。将用于在Preference页面和其他相关UI条目中向用户呈现各种策略。
  • 交互式 - 声明策略是否是交互式的，需要用户操作或控制。在某些情况下，交互式策略将被忽略，因为操作必须在没有交互的情况下发生。

例子：
请参阅org.eclipse.ui.internal.ide。SystemEditorOrTextEditorStrategy。

API信息：
这会将新的接口IUnassociatedEditorStrategy添加到org.eclipse.ui.ide的API中。
