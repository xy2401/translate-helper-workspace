标尺栏

标识符：
org.eclipse.ui.workbench.texteditor.rulerColumns

以来：
3.3

描述：
标尺列贡献是基于行的信息区域，通常显示在文本编辑器的垂直标尺区域中。

配置标记：

<！ELEMENT 扩展 （ 列 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 列 （（ targetEditor | targetContentType | targetClass ）+， placement ）>

<！ATTLIST专栏

id CDATA #REQUIRED

名称CDATA #IMPLIED

icon CDATA #IMPLIED

class CDATA #REQUIRED

enabled（true | false）“true”

global（true | false）“true”

includeInMenu（true | false）“true”>

标尺列贡献的规范。


  • id - 此列的唯一ID
  • name - 列的可选名称，可以在用户界面中使用（例如，在首选项页面中）
  • icon - 列的可选图标。可以在用户界面中使用，例如在首选项页面上。
  • class - 实现列的类;必须实现org.eclipse.ui.texteditor.rulers.IContributedRulerColumn
  • enabled - 此列的默认启用状态
  • 全局 - 控制是否切换此列的可见性对所有目标编辑器或仅活动编辑器起作用。如果为true，则在用户启用列时将标尺添加到所有适用的编辑器中;如果为false，则标尺仅添加到活动编辑器中。
  • includeInMenu - 是否在标尺上下文菜单中包含此列的“显示/隐藏”菜单项

<！ELEMENT targetEditor EMPTY>

<！ATTLIST targetEditor

id IDREF #REQUIRED>

描述该列所贡献的编辑器。请参阅org.eclipse.ui.editors扩展点。请注意，rulerColumns扩展点通常仅受基于行的文本编辑器支持。


  • id - 目标编辑器的编辑器ID

<！ELEMENT targetContentType EMPTY>

<！ATTLIST targetContentType

id CDATA #REQUIRED>

描述列所贡献的内容类型。请参阅org.eclipse.core.runtime.contentTypes扩展点。请注意，rulerColumns扩展点通常仅受基于行的文本编辑器支持。


  • id - 目标内容类型的ID

<！ELEMENT targetClass EMPTY>

<！ATTLIST targetClass

class CDATA #REQUIRED>

描述该列所贡献的编辑器类。此colum贡献也会指定指定类的子类。


  • class - 实现org.eclipse.ui.texteditor.ITextEditor 。该类不需要是可实例化的。请注意，将标尺列提供给贡献者命名空间之外的内部编辑器类被认为是不好的做法，因为类名可能会更改，恕不另行通知。

<！ELEMENT 放置 （（ 之前 | 之后 ））*>

<！ATTLIST安置

重力CDATA“1.0”>

描述此标尺列相对于其他贡献列的放置。列可以请求放在其他已知列之前或之后 （由其ID标识 ）。最终位于相同位置的列按重力排序（见下文）。如果订单是矛盾的，或者找不到满足所有放置请求的订单，则会丢弃一些放置要求，直到可以实现放置。如果列未完全指定其位置，则将其放置在任意但一致的位置。


  • gravity - 一个介于0.0和1.0之间的浮点数，表示列想要与编辑器的距离，其中0.0表示尽可能远，1.0表示尽可能接近。在评估约束之前和之后 ，最终位于相同位置的列按其重力值排序。

<！EMPTY 之前的元素>

<！ATTLIST之前

id IDREF #REQUIRED>

请求在另一列之前放置。之前的含义类似于SWT.LEAD标志：在左到右的环境， 手段的 ， 左 前右到左的环境中，它意味着的权利 。


  • id - 另一个rulerColumns贡献的id

<！EMPTY 之后的元素>

<！ATTLIST之后

id IDREF #REQUIRED>

请求在另一列之后放置。after的含义类似于SWT.TRAIL标志：在从左到右的环境中， 在右到右的环境中 ，在右到左的环境中，它意味着在左边 。


  • id - 另一个rulerColumns贡献的id

例子：
以下是标尺列定义的示例：

<extension
  point="org.eclipse.ui.workbench.texteditor.rulerColumns">
  <column
    id="org.eclipse.ui.editors.columns.linenumbers"
    name="Line Numbers"
    class="org.eclipse.ui.internal.texteditor.LineNumberColumn"
    enabled="false"
    global="true"
    includeInMenu="false">
    <placement
       gravity="0.9">
       <after id="org.eclipse.ui.editors.columns.annotations"/>
    </placement>
    <targetClass
       class="org.eclipse.ui.texteditor.AbstractDecoratedTextEditor">
    </targetClass>
  </column>
</extension>

API信息：
org.eclipse.ui.texteditor.rulers包支持扩展点。RulerColumnRegistry提供对所有贡献的访问权限，而这些贡献又由RulerColumnDescriptor实例描述。

想要支持扩展点的编辑器需要为IColumnSupport提供适配器。AbstractTextEditor已经适应IColumnSupport ，因此它的子类应该使用建议的实现覆盖createColumnSupport() ，而不是再次提供适配器。AbstractDecoratedTextEditor子类自动安装此支持。

提供的实施：
所有标尺列贡献必须实现org.eclipse.ui.texteditor.rulers.IContributedRulerColumn 。许多贡献者会找到提供的org.eclipse.ui.texteditor.rulers.AbstractContributedRulerColumn很有用，它作为贡献标尺列的超类。其他人可能会从org.eclipse.jface.text.source.AbstractRulerColumn接收基于行的列的大多数绘制和滚动方面，并且可以在各种粒度级别进行扩展。那些继承后一类的人必须实现org.eclipse.ui.texteditor.rulers.IContributedRulerColumn org.eclipse.ui.texteditor.rulers.AbstractContributedRulerColumn作为模板。
