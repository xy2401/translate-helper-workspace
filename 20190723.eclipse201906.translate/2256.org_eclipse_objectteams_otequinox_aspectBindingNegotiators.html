用于方面绑定请求的OT / Equinox协商器

标识符：
org.eclipse.objectteams.otequinox.aspectBindingNegotiators

以来：
OTDT 1.2.6

描述：
允许客户端插件参与协商是否应拒绝或授予方面绑定请求（包括强制导出）。

只要一个方面插件请求方面绑定或强制导出，在eclipse安装和工作空间的持久存储中都找不到权限，就会询问所有扩展。

  • 只要任何扩展名DENYs给定请求，此方面插件就会被阻止。
  • 如果没有扩展程序GRANTs请求，方面插件也被阻止。
  • 只有当至少一个扩展名对请求进行GRANT并且没有扩展DENY时，该方面才可以继续并且将被编织到其基本实体中。

配置标记：

<！ELEMENT 扩展 （ 谈判者 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 谈判代表 EMPTY>

<！ATTLIST谈判代表

class CDATA #REQUIRED>

方面绑定协商协议的参与者，它能够授予或拒绝方面捆绑包的请求。


  • class - 应参与协商协议的类的完全限定名称。

例子：
negotiator声明的例子：
<？xml version = “1.0” encoding = “UTF-8” ？> <？eclipse版本= “3.2” ？> <plugin> <extension point = “org.eclipse.objectteams.otequinox.aspectBindingNegotiators” > <negotiator class = “yesser。AspectRequestGranter“ > </ negotiator> </ extension> </ plugin>

API信息：
class属性中指定的class必须实现org.eclipse.objectteams.otequinox.IAspectRequestNegotiator接口。
