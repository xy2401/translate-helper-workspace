3.4。�通过客户端API地址自定义控件

要通过客户端API在自定义控件/自定义CAP上执行CAP，您必须执行两个额外步骤。

首先，您必须使用以下命令为完全限定的新组件类名注册负责的测试器类：

public interface org.eclipse.jubula.toolkit。ToolkitInfo {/ ** *允许将组件类的测试器类添加到工具箱中* @param componentClassName组件类的完全限定名称* @param testerClassName测试器类的完全限定名称* @return以前注册的测试器类用于组件类*或<code> null </ code>如果没有* / @Nullable public String registerTesterClass（@NonNull String componentClassName，@ NonNull String testerClassName）; ...

然后，此扩展的ToolkitInformation用于从AUT-Agent检索AUT：

public interface org.eclipse.jubula.client。AUTAgent扩展了远程{.../ ** * @param autID * autID获取AUT @ * param信息*有关工具包的信息* @return a AUT  - 注意：目前仅底层实现*支持<b> ONE </ b>连接远程运行AUT的时间; *多个连接只能按顺序建立！* @throws CommunicationException *如果远程端出现通信问题* / AUT getAUT（AUTIdentifier autID，ToolkitInfo信息）抛出CommunicationException; ...

然后您可以使用CapBuilder创建自定义CAP实例：

公共类org.eclipse.jubula.toolkit。CapBuilder {...public CapBuilder addParameter（@NonNull String value）{...}public CapBuilder addParameter（@NonNull Integer value）{...}public CapBuilder addParameter（@NonNull Boolean value）{...}public CapBuilder setComponentIdentifier（@Nullable ComponentIdentifier ci）{...}public CAP build（）{} ...}

然后，这些自定义CAP实例可以像往常一样在AUT上下文中执行。

构建的方法签名（通过随后调用addParameter（...））必须匹配在给定测试器类上使用给定的rcMethodName远程调用的方法。
