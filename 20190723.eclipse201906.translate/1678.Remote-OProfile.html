远程OProfile

OProfile插件允许分析存储在远程机器中的项目。在为不同的体系结构开发应用程序时，这可能特别有用。

建立

远程OProfile插件没有安装脚本，但需要在远程计算机中进行一些设置。首先，远程计算机的用户必须能够以root身份运行OProfile。为此，以root身份登录远程计算机并编辑/ etc / sudoers文件以添加以下行：

<user> ALL=(ALL) NOPASSWD: <linux-tools-path>/opcontrol

哪里：

  • <user> - 远程机器的用户
  • <linux-tools-path> - 为项目设置的linux工具的路径。默认值为/ usr / bin。通过右键单击远程项目，单击“属性”并选择“Linux工具路径”选项卡，可以修改此路径。

还要确保以下行不存在或注释掉：

Defaults       requiretty

这对于使用sudo运行远程命令是必需的。

启动远程配置文件

要远程运行OProfile，请右键单击项目并导航到“Profiling Tools”，然后导航到“Profile Configurations”。（注意：远程OProfile没有单击启动）。在新窗口中，双击Profile with OProfile（远程）配置文件配置。

在主选项卡中，单击“C / C ++可执行文件”下的“浏览”按钮，选择要分析的二进制文件。

常规OProfile的相同事件配置事件配置选项也适用于远程OProfile。

配置配置文件后，单击配置文件按钮以启动配置文件。与本地OProfile视图相同的OProfile视图将显示结果。
