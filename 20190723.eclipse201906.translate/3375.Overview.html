概观

Trace Compass是一个Java工具，用于查看和分析任何类型的日志或跟踪。它的目标是提供视图，图形，指标等，以帮助从跟踪中提取有用的信息，其方式比大文本转储更加用户友好和信息丰富。

关于追踪

跟踪是一种故障排除技术，用于通过收集有关其执行路径的信息来了解已检测应用程序的行为。跟踪器是用于跟踪的软件。跟踪可用于解决各种非常具有挑战性的错误。这些包括例如复杂并行系统或实时系统中的性能问题。

跟踪类似于日志记录：它包括记录在选定执行位置的系统中发生的事件。但是，与日志记录相比，它通常针对开发人员，并且通常以高速率记录低级别事件。跟踪器通常每秒可生成数千个事件。生成的跟踪可以轻松包含数百万个事件，并且大小从几兆字节到几十千兆字节。因此，必须优化跟踪器以处理大量数据，同时对系统产生很小的影响。

跟踪可以包括来自操作系统内核的事件（IRQ处理程序进入/退出，系统调用进入/退出，调度活动，网络活动等）。它们还可以包括应用程序事件（也称为UST - 用户空间跟踪）或两者的混合。

对于最高级别的详细信息，可以将跟踪事件视为日志文件。但是，跟踪分析器和查看器可用于从原始数据中获取有用信息以及跟踪程序的知识。这些程序必须专门设计用于快速处理跟踪可能包含的大量数据。

特征

Trace Compass具有许多功能，可以有效处理非常大的跟踪（和大型跟踪集）：

  • 支持任意大的跟踪（大于可用内存）
  • 支持关联多个按时间顺序的跟踪
  • 支持在迹线或迹线集的任何部分上缩小到纳秒
  • 查看当前所选时间或时间范围和窗口时间范围的同步
  • 高效搜索和过滤事件
  • 支持跟踪书签
  • 支持导入和导出跟踪包

还支持非LTTng跟踪类型的集成：

  • 内置CTF解析器
  • 动态创建自定义解析器（用于XML和文本跟踪）
  • 动态创建自定义状态系统（来自XML文件）
  • 动态创建自定义视图（来自XML文件）

Trace Compass提供以下主要视图：

  • Project Explorer - 为跟踪项目量身定制的标准Eclipse Project视图的扩展
  • 事件 - 一种多功能视图，以表格格式显示原始事件，支持搜索，过滤和书签
  • 统计信息 - 一种视图，它按类型提供有关事件发生的简单统计信息
  • 直方图 - 显示跟踪中时间相对于事件密度的视图

可以针对特定跟踪类型（例如内核，硬件，用户应用程序）扩展或定制这些视图。

LTTng集成

Trace Compass的主要功能之一是LTTng集成。LTTng（Linux Trace Toolkit，下一代）是一种高效的Linux跟踪工具，可用于跟踪内核和应用程序性能问题，以及解决涉及多个并发进程和线程的问题。它由一组内核模块，守护进程（用于收集原始跟踪数据）和一组工具来控制，可视化和分析生成的数据组成。它还为用户空间应用程序检测提供支持。有关LTTng的更多信息，请参阅项目站点

注意 ：本用户指南涵盖了Eclipse中最新LTTng（最高v2.4）的集成。

LTTng插件提供了对LTTng跟踪器控制的集成，以及对所产生的跟踪的获取和可视化。它还为用户定义的分析工具提供了基础。

目前，LTTng插件支持以下面向内核的视图：

  • 控制流 - 可视化流程状态转换
  • 资源 - 可视化系统资源状态转换
  • CPU使用率 - 根据跟踪中的时间可视化处理器的使用情况
  • 内核内存使用 - 可视化系统内存的相对使用情况
  • IO用法 - 可视化输入/输出设备的使用
  • 系统调用 - 在表视图中显示所有系统调用
  • 系统调用统计信息 - 显示所有系统调用统计信息
  • 系统调用密度 - 显示按持续时间显示的系统调用
  • 系统调用与时间 - 可视化系统调用何时发生

此外，LTTng插件支持以下用户空间跟踪视图：

  • 内存使用 - 可视化跟踪中每个线程的内存使用情况
  • 调用堆栈 - 可视化调用堆栈随时间的演变
  • 功能持续时间密度 - 可视化按持续时间显示的功能调用
  • Flame Graph - 可视化CPU忙的原因

最后，LTTng插件支持以下控件视图：

  • 控制 - 控制跟踪器并配置跟踪点

虽然控制和提取部分以LTTng跟踪器为目标，但底层框架也可用于处理符合通用跟踪格式 （ CTF ）的任何跟踪。CTF指定了一种非常高效和紧凑的二进制跟踪格式，旨在与应用程序，体系结构和语言无关。
