采用4.9机制和API

本节介绍了在尝试更改4.8插件以采用4.9机制和API时所需的更改。

  1. 应该升级Apache Batik CSS的直接用法
  2. 应升级IBM ICU4j的直接用法
  3. 应该升级Apache Felix Gogo的直接用法

1。应该升级Apache Batik CSS的直接用法

受影响的是：客户端直接使用Apache Batik CSS捆绑版本1.9.1 。

描述：由于安全漏洞，我们已将Eclipse Platform使用的Apache Batik软件包更新为1.10.0版。以前它使用的是1.9.1版本。直接使用Apache Batik CSS API版本1.9.1客户端应该升级其代码以使用版本1.10.0 。其他客户不应受到影响。有关详细信息，请参阅Apache Batik网站 。

需要采取的行动：客户应：

  • 升级他们的代码以使用Apache Batik CSS版本1.10.0

2。应升级IBM ICU4j的直接用法

受影响的是：直接使用早于62.1.0 IBM ICU4j软件包版本的客户端。

描述： Eclipse Platform使用的IBM ICU4j软件包已更新至62.1.0版，以利用对Unicode版本10和11的支持。以前它使用的是版本58.2.0 。直接使用IBM ICU4j API的客户端应升级并确保其代码继续与版本62.1.0 。其他客户不应受到影响。有关详细信息，请参阅IBM ICU4j网站 。

需要采取的行动：客户应：

  • 升级其代码以使用IBM ICU4j版本62.1.0

3。应该升级Apache Felix Gogo的直接用法

受影响的是：客户端直接使用早于1.1.0 Apache Felix Gogo Runtime或Apache Felix Gogo Shell软件包版本。

描述： Eclipse Platform用于提供OSGi控制台的两个Apache Felix Gogo软件包已更新至1.1.0版。以前它使用的是版本的Apache Felix Gogo Runtime 1.0.6和Apache Felix Gogo Shell 1.0.0 。直接使用Apache Felix Gogo API的客户端应升级并确保其代码继续与两个捆绑包的1.1.0版一起使用。其他客户不应受到影响。

需要采取的行动：客户应：

  • 升级他们的代码以使用Apache Felix Gogo Runtime版本1.1.0和Apache Felix Gogo Shell版本1.1.0
