启动配置选项卡

标识符：
org.eclipse.debug.ui.launchConfigurationTabs

以来：
3.3

描述：
此扩展点是将启动配置选项卡（一个或多个）提供给现有启动配置选项卡组。

配置标记：

<！ELEMENT 扩展 （ tab ）+>

<！ATTLIST扩展

点CDATA #REQUIRED>


<！ELEMENT 选项卡 （ placement ？， associatedDelegate *）>

<！ATTLIST标签

id CDATA #REQUIRED

组IDREF #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED>

描述了单个启动配置选项卡贡献


  • id - 此贡献选项卡的唯一ID
  • group - 应该为此选项卡提供的existsnig ILaunchConfigurationTabGroup的唯一ID
  • name - 此选项卡的可读名称。提供的名称不必是选项卡本身上显示的名称，但应充分描述选项卡。
  • class - 实现ILaunchConfigurationTab的java类的完全限定名。

<！ELEMENT 放置 EMPTY>

<！ATTLIST安置

在CDATA #REQUIRED>之后

此元素允许贡献者为其提供的选项卡指定相对区域设置


  • after - 指定此选项卡在选项卡从左到右排列后应立即显示的选项卡的ID。此处指定的ID应与org.eclipse.debug.ui返回的值匹配。AbstractLaunchConfigurationTab.getID（），它可能与此扩展点的选项卡声明中指定的ID不同。

<！ELEMENT associatedDelegate EMPTY>

<！ATTLIST associatedDelegate

委托IDREF #REQUIRED>

此属性用于将受控制的选项卡映射到一个或多个启动委托。这允许根据选择的启动委托作为当前默认值从启动对话框中过滤选项卡。例如，如果两个插件为同一启动模式提供启动委托和选项卡，则将根据用户选择的启用委托向用户显示一组选项卡。


  • delegate - 与此选项卡关联的launchDelegate的id。该关联允许根据代理支持的模式集过滤选项卡。

例子：
以下示例将选项卡“FooTab”添加到本地Java应用程序启动选项卡组。 

<extension point = “org.eclipse.debug.ui.launchConfigurationTabs” > <tab class = “org.eclipse.jdt.debug.ui.launchConfigurations。FooTab“ group = ”org.eclipse.jdt.debug.ui.launchConfigurationTabGroup.localJavaApplication“ id = ”org.eclipse.jdt.debug.ui.fooTab“ name = ”Foo Tab“ > </ tab> </ extension>

提供的实施：
调试平台提供了两个抽象类，客户端在实现启动选项卡时应该将其子类化。请参阅org.eclipse.debug.ui包中的AbstractLaunchConfigurationTab和AbstractOptionLaunchConfigurationTab 。
