配置项目的PHP构建路径

通过配置项目的PHP构建路径，您可以选择将在构建过程中包含或排除的资源。

注意：

如果在项目创建期间选择了“项目布局”类别下的“使用项目作为源文件夹”，则默认情况下，该文件夹中的所有资源都将添加到“构建路径”中。如果您在项目创建期间选择“为源文件和公共资源创建单独的文件夹”，则默认情况下，“公共”文件夹中的资源将从“构建路径”中排除。

可以通过项目的Build Path属性页面配置项目的PHP Build Path，右键单击所需项目并选择Build Path |配置构建路径。

此页面允许您向构建路径添加文件夹和资源，并配置包含/排除项目中某些资源的规则。

您可以根据下面列出的选项配置构建路径。

完成更改后，单击“确定”进行保存。

注意：

默认情况下，构建路径中的所有文件夹都将包含在“包含路径”中。

配置项目的包含/排除模式

通过为构建路径配置包含/排除模式，您可以选择包含或排除项目中与定义模式匹配的所有资源。

要包含/排除构建路径中的资源：

  1. 在“构建路径属性”页面中，展开要配置其包含/排除模式的文件夹旁边的节点，然后单击“编辑”。 - 或者 - 在Project Explorer视图中，右键单击要配置其包含/排除模式的文件夹，然后选择“构建路径”|“构建路径”。配置包含/排除过滤器。将显示“包含和排除模式”对话框。
  2. 单击“包含或排除”模式窗格旁边的“添加”。将显示“添加包含/排除模式”对话框。
  3. 输入或选择要包含/排除的所需资源或模式，然后单击“确定”。
  4. 单击完成。

项目中与包含模式匹配但与排除模式不匹配的所有资源都将添加到构建路径中。

为您的项目配置不同的包含/排除模式

您可以为所选目录中的子文件夹配置不同的包含/排除规则。但是，您需要从父目录的构建路径中排除选定的子文件夹，并将其添加为单独的构建路径源文件夹。这样做是为了解决为父文件夹配置一个模式而为子文件配置另一个模式可能引起的任何冲突。

可以从PHP Build Path属性页面或Project Explorer视图中创建和/或添加源文件夹。

要将目录添加为单独的源文件夹：

从PHP Build Path Properties页面：

  1. 单击添加文件夹...将显示“源文件夹选择”对话框。
  2. 如有必要，请单击“创建新文件夹”以创建新文件夹。
  3. 选择所需文件夹并单击“确定”。
  4. 该文件夹将作为单独的源文件夹添加到“构建路径”列表中。

注意：

为了克服嵌套冲突，您应该从父文件夹的构建路径中排除刚刚添加的文件夹。有关如何排除文件夹及其中包含的所有资源的信息，请参阅上面的“ 配置包含/排除模式 ”。

从Project Explorer视图：

  1. 右键单击该项目，然后选择“构建路径”新的源文件夹。将显示“新建源文件夹”对话框。
    1。在“文件夹名称”字段中输入新文件夹的名称。
  2. 将文件夹添加到项目的构建路径可能会导致为项目根配置的包含/排除模式与为所选文件夹配置的包含/排除模式之间发生冲突。为了避免这些嵌套冲突，请选择以下选项之一：
    • 替换现有项目源文件夹条目以解决嵌套 - 这将从构建路径配置中删除您的父目录作为源文件夹，并将其替换为当前选定的文件夹。
    • 更新其他源文件夹中的排除过滤器以解决嵌套问题 - 这将从父目录的构建路径中排除当前选定的文件夹，但会将其添加为单独的实体，以便可以为其配置不同的包含/排除模式。
  3. 单击完成。

您现在可以按照上面“ 配置包含/排除模式 ”下的说明配置包含/排除源文件夹资源的模式 。

将外部源文件夹添加到构建路径

您可以添加外部源文件夹作为项目的链接，在构建过程中将扫描该项目。

要将外部源文件夹添加到项目的构建路径：

  1. 在“构建路径属性”页面中，单击“链接源...”按钮。 - 或者 - 在Project Explorer视图中，右键单击项目并选择Build Path |链接来源。将显示“链接源”对话框。

    1。单击“浏览...”按钮并浏览到要添加到构建路径的文件夹的位置。或者，单击“变量”并选择指向所需资源的变量。
  2. 文件夹名称字段将自动填充原始文件夹的名称。如果需要，请编辑此条目。
  3. 将文件夹添加到项目的构建路径可能会导致为项目根配置的包含/排除模式与为所选文件夹配置的包含/排除模式之间发生冲突。为了避免这些嵌套冲突，请选择以下选项之一：
    • 替换现有项目源文件夹条目以解决嵌套 - 这将从构建路径配置中删除您的父目录作为源文件夹，并将其替换为当前选定的文件夹。
    • 更新其他源文件夹中的排除过滤器以解决嵌套问题 - 这将从父目录的构建路径中排除当前选定的文件夹，但会将其添加为单独的实体，以便可以为其配置不同的包含/排除模式。
    • 忽略嵌套冲突 - 所选文件夹将作为单独的实体添加到“构建路径”列表中，但需要从项目根目录的“构建路径”中手动排除，以避免嵌套冲突。
  4. 单击完成。

该文件夹将添加到您的构建路径并显示在构建路径列表中。

您现在可以按照上面“ 配置包含/排除模式 ”下的说明配置包含/排除链接文件夹资源的模式 。
