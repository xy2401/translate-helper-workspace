如何清除任务的外发更改？

如果任务有您想要丢弃的外发更改，请右键单击该任务，然后选择标记为→清除外发。如果发生空中碰撞，请按任务编辑器工具栏中的同步按钮以引入传入的更改。您的传出更改将保留。

为什么我的任务没有出现在任务列表中？

任务列表具有过滤机制，可以限制任务列表中可见的项目。如果任务列表中缺少任务，请使用以下步骤删除所有过滤器：

  1. 检查视图菜单中的过滤器设置。确保未启用过滤器已完成任务，并且在“过滤器优先级低于”菜单中选中了所有项目。
  2. 如果工具栏中显示“Go Up To Root”图标，请选择该图标以使任务列表中的所有项目都可见。
  3. 如果按下Work on Workweek按钮，选择它以取消过滤任务列表。
  4. 如果在查找中键入文本，请使用查找框旁边的清除按钮。
  5. 切换到全部以使所有工作集可见。

为什么任务出现在Unmatched容器中？

如果需要显示任务但未与任何查询匹配，则将自动显示未匹配的容器。这是确保相应任务不会消失所必需的，例如，如果查询被设置为仅匹配已解决/已完成的任务（使用反模式，因为Mylyn具有其他机制来阻止已完成的任务在任务中显示列表）。

为了确保您不会错过提醒或通知，以下任务将始终显示在任务列表中，即使它们已从类别或查询中删除：

  • 计划在本周或逾期的任务。使用“ 计划”弹出菜单选项将其推迟到将来的一周或清除计划，从而删除它们。
  • 具有传入更改的存储库任务，例如注释。通过读取它们或通过标记→读取弹出菜单将它们标记为“已读”来删除它们。

要删除Unmatched容器中的任务，您可以：

  • 创建与任务匹配的新查询。
  • 右键单击一个或多个任务以删除它们。

如何更改任务列表颜色？

使用窗口→首选项→常规→外观→颜色和字体 。

“不匹配”类别包含许多不相关的任务，如何清理它？

如果您创建了非常广泛的查询，则最终可能会在未匹配的容器中执行数千个任务。除了清除工作区/ .metadata / .mylyn / tasklist.xml.zip和workspace/.metadata/.mylyn/offline文件夹（请注意，这将完全重置您的任务列表），最简单的选择是：

  • 清理查询以仅包含感兴趣的任务（请注意，建议包括已完成/已解决的任务）。
  • 在任务列表中关闭Focus on Workweek 。
  • 从“未匹配的容器”中删除所有不需要的任务。请注意，如果任务与查询不匹配，则会永久删除这些任务，并且您添加到其中的任何本地注释或计划信息都将丢失。

Mylyn如何计算任务的活动时间？

每当您处理任务时，Mylyn都会积累您积极处理任务所花费的时间。可以在任务编辑器的“ 个人计划”部分查看此时间。当您不与Eclipse交互时，默认情况下，时间会在3分钟后自动超时。这意味着在处理任务时不会捕获Eclipse外部的活动，并且Mylyn中的计时是在任务上花费的总时间的下限（捕获在Eclipse之外完成的工作的计时时间涉及操作系统特定的扩展）。

如何防止长时间运行的任务添加到进度条？

如果本周安排了许多长时间运行或重复执行的任务，它们可能会影响任务列表每周进度条。这可能会产生误导，因为它们可能永远无法完成。当前解决方案是防止长时间运行的任务被包含在每周进度条中，是将其估计时间安排为0（零）。或者，您可以仅计划本周计划在该任务上花费的时间。

任务列表是否替换了Eclipse Tasks视图？

SDK的“ 任务”视图用于显示标记，例如“todo”标记，用于指示资源的本地问题，类似于编译器警告。因此，这些“任务”的粒度比Mylyn的任务更精细，一项任务可能涉及清理多个待办事项。为了仅使用特定任务上下文中的标记，例如在提交之前进行清理，可以在“ 问题”和“ 任务”视图中使用“ 应用Mylyn”过滤器。

如果我使用多个工作区怎么办？

Mylyn对多个工作空间的支持目前有限（请参阅错误130658进行讨论），因为任务列表被认为是特定于个人的，而不是工作区。此外，Eclipse对多个工作区的支持仅限于基于导入/导出的使用，Mylyn继承了此限制。我们强烈建议您将Eclipse工作区配置为使用工作集，而不是依赖于多个工作区。

如果无法使用单个工作区，则可以执行以下操作，Mylyn确实提供了使用工作区外部相同数据目录的高级工具。但是，通常不建议这样做，因为如果同时启动两个工作空间，它可能会导致您覆盖现有的任务列表。

  • 在首选项→任务→任务列表中，将数据目录设置为共享位置，并对两个工作区执行此操作。
  • 请务必避免同时启动两个Eclipse工作区，因为一个工作区中的更改可能会覆盖其他工作区中的任务列表。
  • 如果升级Eclipse或移动工作区但看不到任务，请检查“ 数据目录”设置。

请注意，使用文件→导入→任务数据是将现有任务列表放入工作区的另一种方法。如果使用此机制在工作空间之间共享任务列表 ，则应在切换时导出任务列表 ，因为在此模式下，您正在使用两个单独的任务列表 。虽然大部分任务表明您的工作存储在共享任务存储库中，但所有读取状态，活动历史记录，计划信息和本地任务都保留在任务列表中 ，因此最终可能会因为两个不同的列表而变得很麻烦。管理。

为什么Mylyn使用术语“任务”？

有许多工作项构成了开发人员的工作日。许多问题跟踪器和项目管理工具将这些引用为：错误，缺陷，操作，故障单，故事，增强功能，并且列表继续存在。我们将所有此类工作项称为“任务”，因为单词任务很短并且通常用于时间管理工具中。任务存储库连接器可以自定义任务的显示，例如，指示哪个是缺陷，哪个是操作项。

为什么Linux上的封闭任务不会变得灰暗？

如果您从KDE运行Eclipse，请转到KControl→GTK样式和字体，然后在GTK样式部分选择“使用其他样式”（ 错误206399 ）。

如果样式选择不起作用，请参阅KDE的推荐GTK设置 。

为什么不支持主演任务？

主演任务的当前机制是将它们安排在今天 ，这与其他UI中的主演效果非常相似。如果您为今天安排任务，该任务将突出显示为蓝色并始终显示（即，在任务列表聚焦时保证可见性）。如果这太明显了，您可以安排本周 ，在这种情况下，任务将始终显示但不会变为蓝色。

将共享任务标记为传出的替代方法是将文本添加到其中（例如“[review]”）。ougtoing更改还将使任务保证可见性。这是目前“主演”已完成任务的唯一解决方法。

以任务为中心的界面考虑与资源（例如，文件和网页）正交的任务。资源对星/书签有意义，因为它们的主要驻留在某种结构层次结构中（例如文件夹或类型层次结构）。Starring提供了一种机制，用于定位大型层次结构中最相关的部分（代价是我们的已加星标/书签列表膨胀并成为管理的另一件事）。任务本质上与资源不同，因为它们的主要住所是及时的（例如到期日，里程碑，过去完成的事情）。因此，对任务主演的等价物是安排在“当前”时间窗口（例如今天或本周）中查看它们。有关此问题的讨论，请参阅错误168363 。
