模板扩展点

标识符：
org.eclipse.cdt.core.templates

以来：
CDT 4.0

描述：
此模板扩展点有助于用户将其模板XML提供给模板引擎插件。

配置标记：

<！ELEMENT 扩展 （ 模板 +， 类别 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

扩展点添加到Template Engine插件。任何打算向模板引擎提供XML模板的插件都必须扩展此扩展点，并添加模板元素。


  • id - 扩展程序插件中扩展名的ID。
  • name - 可选的name属性。

<！ELEMENT 模板 （ toolChain *， parentCategory *）>

<！ATTLIST模板

id CDATA #REQUIRED

位置CDATA #REQUIRED

projectType CDATA #IMPLIED

isCategory（true | false）

filterPattern CDATA #IMPLIED

pagesAfterTemplateSelectionProvider CDATA #IMPLIED>

通过添加模板扩展点，用户可以将其模板XML提供给Template Engine插件。


  • id - 此模板贡献的唯一标识符。这不需要与它所贡献的模板的id（在template.xml中定义）相同。这允许多次提供相同的template.xml（例如，对于多个项目类型，或使用备用页面序列）。
  • location - template.xml的路径（相对于包含插件的基础）。例如“templates / TemplateOne / template.xml”。此属性是必需的。
  • projectType - 此属性是一个项目类型ID，引用模板将与之关联的cdt项目类型。项目类型提供给org.eclipse.cdt.managedbuilder.core.buildDefinitions扩展点。此属性是必需的。CDT预定义了构建工件类型属性的以下值：
    • “org.eclipse.cdt.build.core.buildArtefactType.exe” - 表示可执行文件
    • “org.eclipse.cdt.build.core.buildArtefactType.staticLib” - 表示静态库
    • “org.eclipse.cdt.build.core.buildArtefactType.sharedLib” - 表示共享库
    • ISV可以通过贡献org.eclipse.cdt.managedbuilder.core.buildProperties扩展点来定义自己的自定义构建工件值。
    它不再是强制性的，因为模板不再按项目类型组织。模板catogories取代了该功能。
  • isCategory - 如果为true，则此模板贡献应显示为文件夹。默认为false。
  • filterPattern - filterPattern是一个java.util.regex。模式格式字符串，用于匹配构建配置ID。这是一个可选属性，如果不存在，则所有配置都匹配。
  • pagesAfterTemplateSelectionProvider - 实现org.eclipse.cdt.templateengine的类。IPagesAfterTemplateSelectionProvider接口，用于创建将附加到TemplatesChoiceWizard.getPagesAfterTemplateSelection（）返回的页面的页面。

<！ELEMENT toolChain EMPTY>

<！ATTLIST工具链

id CDATA #REQUIRED>

此元素用于通过其唯一标识符引用现有工具链。


  • id - 工具链的唯一标识符，该标识符对org.eclipse.cdt.managedbuilder.core.buildDefinitions扩展点有贡献。

<！ELEMENT 类别 （ parentCategory *）>

<！ATTLIST类别

id CDATA #REQUIRED

标签CDATA #REQUIRED>

用于在UI中的层次结构中排列模板。


  • id -
  • 标签 -

<！ELEMENT parentCategory EMPTY>

<！ATTLIST parentCategory

id CDATA #REQUIRED>

父类别的ID。


  • id -

例子：
以下是扩展点用法的示例：


   <extension point="org.eclipse.cdt.core.templates">
      <template
         id="org.foobar.templates.MyExampleTemplate"
         location="templates/MyExampleTemplate/template.xml"
         projectType="org.eclipse.cdt.build.core.buildArtefactType.exe"
         filterPattern=".*">
      </template>
   </extension>
有关如何定义自己的模板的更多详细信息，请查看org.eclipse.cdt.ui.templateengine下提供的示例

提供的实施：
org.eclipse.cdt.templateengine提供了此扩展点的实现
