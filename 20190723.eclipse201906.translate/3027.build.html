构建同步项目

本节讨论

  • 构建配置

构建配置

构建配置提供了构建项目所需的信息。请注意，只有C，C ++或Fortran同步项目才具有构建配置。

项目的构建配置最初基于所选项目类型和选定的远程和本地工具链，但实际上对可以使用构建配置的主机没有限制。通常对远程和本地主机使用相同的构建配置。“远程”与“本地”的区别仅适用于同步配置。

构建配置信息在项目属性中指定。

右键单击Project Explorer视图中的项目名称，然后从上下文菜单列表的底部选择Properties ，打开Project Properties 。

将显示项目的项目属性对话框。

选择C / C ++ Build以查看项目的Build Properties 。

可以在此页面顶部选择构建配置 。可以在对话框的左侧选择以下页面：

C / C ++ Build - 构建的主要属性页面。允许您配置构建命令。默认的构建命令是“make”，但可以将其更改为任何内容。

C / C ++ Build下列出的其他页面包括：

  • 构建变量 - 创建和管理可在其他构建配置页面中使用的变量
  • 发现选项 - 用于发现编译器设置的过时选项页面（使用C / C ++常规>预处理器包含路径 。）
  • 环境 - 修改或添加传递给构建的环境变量
  • 日志记录 - 启用或禁用构建日志记录
  • 设置
    • 二进制解析器选择（用于在Project Explorer中显示二进制文件）
    • 错误解析器选择（用于解析编译器命令的输出，并填充Problems视图）
    • 工具链设置（仅限托管构建项目 - 不用于Makefile项目）
  • 工具链编辑器 - 允许修改特定工具链中的工具
  • XL C / C ++编译器 - XL C / C ++编译器的编译器设置（如果已安装）

选择构建配置

多个构建配置可用于项目。可以为不同目标平台上的不同体系结构指定构建配置。

构建按钮将使用活动的构建配置被选中。

您可以使用构建图标旁边的箭头同时构建和切换。

或者，您只能使用构建图标旁边的“日”图标来更改活动配置。

您还可以使用“ 构建配置”项目上下文菜单更改活动配置。为此，右键单击项目，选择Build Configurations> Set Active>并选择所需的构建配置。

构建同步项目

构建配置和同步配置是独立的。

当构建按钮如果选中，则活动构建配置将在活动同步配置指定的远程系统上构建 。

可以在“ 控制台”视图中查看结果。

可以更改哪个构建配置处于活动状态，但要确保在同步配置中指定的远程系统上有意义。

构建配置可以与同步配置相关联，以便在更改同步配置时自动选择它。

同步配置

通过同步配置（同步配置或简称同步配置），可以为可以构建项目的每个位置单独设置构建配置。

可以从两者定位同步配置

  • Project Explorer中项目的Synchronize上下文菜单
  • “ 项目属性”对话框。

要从项目上下文菜单访问同步配置，请在Project Explorer中的项目上使用鼠标右键，然后选择Synchronize> Manage ...

这将打开“ 管理同步配置”对话框。

可以在“ 项目属性”的“ 同步”页面上访问相同的信息。

顶部区域列出了同步配置，指示可以构建项目的不同位置。选择同步配置，并在CDT Build Configuration下列出默认构建配置

另请参见与同步工作项目上的同步项目页面获取更多的讨论。

环境管理

如果远程系统安装了环境系统（例如模块），则可以配置一组自定义模块来构建C / C ++项目。

在“ 管理同步配置”对话框中，选择要在顶部更改的配置，然后选中“ 使用环境管理系统”以自定义远程构建环境 。

查询远程系统以获取模块的可用性，并使用可用模块和当前选定的模块填充对话框。从Available Modules列表中选择一个模块，然后单击Add - >按钮将其添加到Selected Modules列表中。

使用< - Remove按钮从Selected Modules列表中删除模块。

使用“ 筛选器列表”字段可以快速查找具有给定名称的模块。

使用向上和向下按钮更改所选模块的顺序。

单击“ 设置默认值”以仅加载新登录shell中存在的那些模块。

使用模块构建

为了构建项目，Eclipse将会

  • 打开一个新的Bash登录shell
  • 执行模块清除
  • 为每个选定的模块执行模块加载
  • 运行make

模块命令在构建期间显示在“ 控制台”视图中。

注意必须按特定顺序加载的模块，或包含/bin或/usr/bin等公共路径的模块。

有关详细信息，请参阅配置环境模块。

有关同步项目的更多详细信息，请参阅同步项目 。
