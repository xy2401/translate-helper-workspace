helpKey常量接口类

要引用在ResourceBundle属性文件中定义的外化帮助上下文ID，UI组件必须实现一个接口类来声明抽象的helpKey常量。

以下示例来自org.eclipse.datatools.connectivity.ui源代码。 

包org.eclipse.datatools.connectivity.internal.ui; / ** * helpKey_constants_for_plug-in：org.eclipse.datatools.connectivity.ui * / public interface IHelpConstants {/ * * CONTEXT_ID_CP_PROPERTY_PAGE = *基本概要文件名称/描述/自动连接属性页* / public static final String CONTEXT_ID_CP_PROPERTY_PAGE =“CONTEXT_ID_CP_PROPERTY_PAGE “; // $ NON-NLS-1 $ / * * CONTEXT_ID_CP_WIZARD_PAGE = *新建连接配置文件向导中的向导选择页面* / public static final String CONTEXT_ID_CP_WIZARD_PAGE =“CONTEXT_ID_CP_WIZARD_PAGE”; // $ NON-NLS-1 $ / * * CONTEXT_ID_INTRO_WIZARD_PAGE = *新连接配置文件向导的基本介绍页面* / public static final String CONTEXT_ID_INTRO_WIZARD_PAGE =“CONTEXT_ID_INTRO_WIZARD_PAGE”; // $ NON-NLS-1 $ ...}
请注意helpKey常量接口类的以下功能：
  • 包和类声明之间出现Javadoc注释块。注释标识将使用此类将抽象的helpKey常量传递给ContextProviderDelegate的每个UI插件。
  • 该类仅声明helpKey常量;此类中没有声明其他常量。
  • 该类声明至少一个UI插件的所有helpKey常量。
    注意：该类可以为多个UI插件声明helpKey常量，但任何一个UI插件的所有helpKey常量必须仅在一个helpKey常量接口类中声明。
  • 每个helpKey常量声明都应以注释开头，注释必须为文档团队提供足够的信息，以确定关联UI控件的位置和用途。
  • 每个helpKey常量都声明为public static final String，格式如下：
    public static final String MY_HELP_KEY =“MY_HELP_KEY”;
    其中MY_HELP_KEY是用作UI控件中的helpKey常量的文字字符串。
    注意： helpKey常量应仅声明为文字字符串值。应避免间接声明（例如HELP_KEY = PREFIX +“HELP_KEY_STRING” ）。
  • helpKey常量字符串中允许的唯一字符是：大写和小写字母（az，AZ），数字（0-9）和下划线（_）。
    注意： helpKey常量字符串不得以完全限定的帮助上下文ID的格式显示。如果helpKey常量在helpKey属性文件中映射到自身（作为非限定帮助上下文ID），则任何句点（。）或空格字符都将导致问题。
