使用JavaScript内容辅助

这些过程描述了如何启用和配置JavaScript
内容辅助选项。

注意：

为了使项目外部的库和文件中的JavaScript元素可用于内容辅助列表，您必须为项目启用JavaScript支持并设置JavaScript构建路径 。要访问Dojo工具包内容辅助选项，必须在配置构建路径时在项目中设置Dojo集成。

要访问JavaScript内容辅助选项：

  1. 输入相关的HTML和JavaScript代码：
  2. JavaScript内容辅助选项现在可用。
  3. 键入所需元素的前几个字母。将显示内容辅助列表（如果未自动显示内容辅助列表，请按Ctrl + Space或转至JavaScript内容辅助首选项页面以配置自动激活首选项）。
  4. 通过双击或按Enter键从“内容辅助”窗口中选择相关选项。
  5. 如果您选择了JavaScript类，请键入句点“。”在类的名称之后，显示具有类的相关功能和方法的内容辅助窗口。
  6. 选择完成JavaScript代码所需的选项。

JavaScript内容辅助配置

可以从JavaScript内容辅助首选项页面配置JavaScript内容辅助选项。

配置JavaScript内容辅助选项：

  1. 转到从JavaScript |访问的JavaScript内容辅助首选项页面偏好|网络| JavaScript |编辑|内容辅助 。
  2. 根据您的偏好配置以下选项：

插入

  • 完成插入/完成覆盖 - 选择是否从内容辅助列表中选择项目将导致输入新代码或覆盖现有代码。
  • 自动插入单个提案 - 如果启用，只有一个内容辅助选项存在时，将自动插入内容辅助建议
  • 自动插入公共前缀 - 如果启用，Content Assist将自动插入所有可能的完成的公共前缀，类似于Unix shell扩展。即使在显示“内容辅助”窗口时，也可以重复使用此功能。
  • 在完成时填充参数名称 - 如果启用，Content Assist将在完成方法时添加参数。
  • 猜测填充函数参数 - 如果启用，Content Assist将根据上下文填充具有最佳匹配函数的参数。

排序和过滤

  • 对提案进行排序 - 选择应在内容辅助列表中对提案进行排序的方式。
  • 隐藏在调用上下文中不可见的提议 - 如果启用，则Java元素提议受可见性规则的限制。例如，不会显示其他类的私有字段提案。
  • 显示驼峰大小写匹配 - 如果启用，将显示驼峰大小写匹配（例如，NPE扩展为NullPointerException）。
  • 隐藏禁用引用 - 如果启用，则不会显示对访问规则禁止的JavaScript元素的引用。
  • 隐藏不鼓励的引用 - 如果启用，则不会显示对访问规则阻止的JavaScript元素的引用。
  • 隐藏已弃用的引用 - 如果启用，则不会显示对已弃用的JavaScript元素的引用。

自动激活

  • 启用自动激活 - 如果启用，则在键入元素的第一个字母时将自动显示“内容辅助”列表。

注意：

如果未标记，则可以按Ctrl + Space显示“内容辅助”列表。

  • 自动激活延迟 - 确定自动显示“内容辅助”框之前的延迟。
  • JavaScript的自动激活触发器 - 设置确定JavaScript上下文的字符。
  • JSDoc的自动激活触发器：设置确定JSDoc上下文的字符。

单击“应用”以应用设置。
