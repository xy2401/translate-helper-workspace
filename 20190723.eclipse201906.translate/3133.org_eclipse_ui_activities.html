活动

标识符：
org.eclipse.ui.activities

以来：

3.0

描述：

org.eclipse.ui.activities扩展点用于声明活动和关联元素。平台使用活动来过滤来自用户视图的某些插件贡献，直到他们表达对它们的兴趣为止。这允许Eclipse根据用户的使用模式动态增长。

配置标记：

<！ELEMENT 扩展 （ activity ， activityRequirementBinding ， activityPatternBinding ， category ， categoryActivityBinding ， defaultEnablement ）*>

<！ATTLIST扩展

id CDATA #IMPLIED

名称CDATA #IMPLIED

点CDATA #REQUIRED>

  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称
  • point - 目标扩展点的完全限定标识符

<！ELEMENT 活动 （ 启用时 ？）>

<！ATTLIST活动

描述CDATA #IMPLIED

id CDATA #REQUIRED

名称CDATA #REQUIRED>

此元素用于定义活动。如果这些元素中不止一个具有相同的id属性，则只有最后声明的元素（按读取注册表的顺序）才被视为有效。


  • description - 用于在UI中显示的此活动的可翻译简短描述
  • id - 此活动的唯一标识符
  • name - 要在UI中显示的此活动的可翻译名称

<！ELEMENT activityRequirementBinding EMPTY>

<！ATTLIST activityRequirementBinding

requiredActivityId IDREF #REQUIRED

activityId IDREF #REQUIRED>

此元素允许将活动绑定到活动。这种关系是这样的，即如果启用了activityId，那么也启用了requiredActivityId。


  • requiredActivityId - 要绑定的活动的唯一标识符
  • activityId - 要绑定的活动的唯一标识符

<！ELEMENT activityPatternBinding EMPTY>

<！ATTLIST activityPatternBinding

activityId IDREF #REQUIRED

模式CDATA #REQUIRED

isEqualityPattern（true | false）>

此元素允许将活动绑定到模式。


  • activityId - 要绑定的活动的唯一标识符
  • pattern - 要绑定的模式。模式是与唯一标识符匹配的正则表达式。请参阅Java文档以获取java.util.regex.进一步细节的Pattern 。
  • isEqualityPattern - 如果模式应该“按原样”考虑。这是一个可选属性，如果省略则值为false如果为false，则在确定给定标识符是否与模式匹配时，模式字符串将被编译为正则表达式。如果是这样，则执行直接相等比较。自3.4

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

描述CDATA #IMPLIED

id CDATA #REQUIRED

名称CDATA #REQUIRED>

此元素用于定义类别。如果这些元素中不止一个具有相同的id属性，则只有最后声明的元素（按读取注册表的顺序）才被视为有效。


  • description - 此类别的可翻译简短描述，用于在UI中显示
  • id - 此类别的唯一标识符
  • name - 要在UI中显示的此类别的可翻译名称

<！ELEMENT categoryActivityBinding EMPTY>

<！ATTLIST categoryActivityBinding

activityId IDREF #REQUIRED

categoryId IDREF #REQUIRED>

此元素允许将类别绑定到活动。


  • activityId - 要绑定的活动的唯一标识符
  • categoryId - 要绑定的类别的唯一标识符

<！ELEMENT defaultEnablement EMPTY>

<！ATTLIST defaultEnablement

id IDREF #REQUIRED>

此元素允许指定默认情况下应启用给定活动。


  • id - 活动的唯一标识符

<！ELEMENT enabledWhen （ not | and | 或 | instanceof | test | systemTest | equals | count | with | resolve | adapt | iterate | reference ）>

包含工作台处理程序代理使用的核心表达式，以确定何时启用此处理程序而不加载它。



例子：
以下是几个活动和类别定义以及相关绑定的示例。 

<extension point = “org.eclipse.ui.activities” > <activity id = “com.xyz。活动“ description = ”过滤来自com.xyz的贡献“ name = ”My Activity“ /> <activity id = ”com.xyz。OtherActivity“ description = ”过滤com.xyz的其他贡献“ name = ”我的其他活动“ /> <！ - 其他活动需要活动 - > <activityRequirementBinding activityId = ”com.xyz。OtherActivity“ requiredActivityId = ”com.xyz。活动“ /> <category id = ”com.xyz。类别“ description = ”com.xyz活动“ name = ”我的类别“ /> <！ - 将活动放在类别中 - > <categoryActivityBinding activityId = ”com.xyz。活动“ categoryId = ”com.xyz。类别“ /> <！ - 绑定来自插件com.xyz的所有贡献 - > <activityPatternBinding id = ”com.xyz。活动“ pattern = ”com \ .xyz /.*“ /> <！ - 从插件com.xyz.other绑定my.contribution  - > <activityPatternBinding id = ”com.xyz。OtherActivity“ pattern = ”com \ .xyz \ .other / my.contribution“ /> <！ - 相同，但没有正则表达式，因为模式应该完全匹配id  - > <activityPatternBinding id = ” com.xyz。OtherActivity“ isEqualityPattern = ”true“ pattern = ”com.xyz.other / my.contribution“ /> <！ - 我们的活动应默认启用 - > <defaultEnablement id = ”com.xyz。活动“ /> </ extension>

API信息：

目前没有用于声明此扩展点以外的活动或关联元素的公共API。工作台中的活动状态可通过org.eclipse.ui.访问org.eclipse.ui.IWorkbench.getActivitySupport() 。从这里，您可以查询和更新当前启用的活动集。

提供的实施：
工作台没有提供“默认活动”。活动旨在在产品级别定义，例如Eclipse SDK，以便紧密集成产品包含的所有（已知）组件。
