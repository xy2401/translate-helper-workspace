介绍部分

标识符：
org.eclipse.ui.intro

以来：
3.0

描述：

此扩展点用于注册特殊工作台部件的实现，称为引入部件，负责将产品引入新用户。通常在产品第一次启动时显示介绍部分。通过此扩展点也可以提供将介绍部件实施与特定产品相关联的规则。

生命周期如下：
  • 介绍区域是在工作台启动时创建的。与编辑和查看区域一样，此区域由介绍站点管理（实现org.eclipse.ui.intro.IIntroSite ）。
  • 当前产品的ID（Platform.getProduct（））用于选择相关的介绍部件实现。
  • 介绍部分类（实现org.eclipse.ui.intro.IIntroPart ）使用介绍站点创建并初始化。
  • 当介绍部分向用户显示时，它可以在完全模式和待机模式之间来回切换（由用户以编程方式或显式方式）。
  • 最终，引入部分被关闭（由用户以编程方式或显式方式）。当前透视图占据整个工作台窗口区域。

配置标记：

<！ELEMENT 扩展 （ intro *， introProductBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 简介 EMPTY>

<！ATTLIST介绍

id CDATA #REQUIRED

icon CDATA #IMPLIED

class CDATA #REQUIRED

contentDetector CDATA #IMPLIED

标签CDATA #IMPLIED>

指定简介。介绍是针对产品启动时向首次使用的用户显示的特定于产品的演示文稿。


  • id - 此简介的唯一标识符
  • icon - 与此简介相关联的图标的插件相对文件名
  • class - 实现org.eclipse.ui.intro.的类的完全限定名称org.eclipse.ui.intro.IIntroPart接口。通常的做法是子类org.eclipse.ui.part.intro。IntroPart为了继承默认功能。这个类实现了介绍。
  • contentDetector - 扩展org.eclipse.ui.intro.的类的完全限定名称org.eclipse.ui.intro.IntroContentDetector 。此可选类用于检测新添加的介绍内容。如果有新内容可用，则会再次打开显示介绍的视图。
  • label - 此介绍的字符串标签，用于构建open动作以及部件标签3.2以后

<！ELEMENT introProductBinding EMPTY>

<！ATTLIST introProductBinding

productId IDREF #REQUIRED

introId IDREF #REQUIRED>

指定产品和简介之间的绑定。这些绑定确定哪种引入适合当前产品（由org.eclipse.core.runtime.定义） org.eclipse.core.runtime.Platform.getProduct() ）。


  • productId - 产品的唯一ID
  • introId - 介绍的唯一ID

例子：
以下是介绍部分扩展的示例，该扩展有助于特定介绍并将其与特定产品相关联：

<extension point = “org.eclipse.ui.intro” > <intro id = “com.example.xyz.intro.custom” class = “com.example.xyz.intro。IntroPart“ /> <introProductBinding productId = ”com.example.xyz。产品“ introId = ”com.example.xyz.intro.custom“ /> </ extension>

API信息：
class属性的值必须是实现org.eclipse.ui.intro.的类的完全限定名org.eclipse.ui.intro.IIntroPart接口通过IIntroPart org.eclipse.ui.part.intro.IntroPart 。

提供的实施：
初始用户体验没有默认实现。每个基于Eclipse的产品都负责提供与其品牌和功能紧密匹配的产品。
