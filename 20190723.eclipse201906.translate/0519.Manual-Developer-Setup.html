手动开发者设置

获取来源

EGit和JGit是在Git中自托管的。您可以在Web上浏览存储库： EGit ， JGit

下面的第一部分描述了如何克隆存储库，如果您之前已经这样做，可以跳过它。下一节将列出存储库及其URL。

克隆

在命令行上

git clone <enter URL>

之后，使用Import> Existing Projects into Workspace将项目导入Eclipse。

使用EGit（参见下载页面 ）

首先，验证主Git首选项页面上设置的默认存储库文件夹是否符合您的喜好。

然后，克隆存储库并导入项目：

  • 打开文件 > 导入...，然后从Git中选择Git > Projects
  • Selet URI
  • 输入网址（参见下一节）
  • 从新创建的工作目录将现有项目导入工作空间

库

要开发EGit，需要EGit和JGit存储库，其他存储库是可选的。要开发JGit，只需要JGit。

例如：It

网址： https ： //git.eclipse.org/r/egit/egit.git

这是主存储库，其中开发了标准EGit功能。它包含UI和Eclipse集成的代码。

JGit

网址： https ： //git.eclipse.org/r/jgit/jgit.git

这是EGit使用的Git Java实现，用于处理Git存储库。

EGit GitHub集成

网址： https ： //git.eclipse.org/r/egit/egit-github.git

EGit还提供了与GitHub和Mylyn任务集成的工具。

要获取依赖项，请打开文件org.eclipse.mylyn.github-feature/github.target （ 在Web上查看 ）并选择Set as Target Platfrom 。

EGit PDE工具

网址： https ： //git.eclipse.org/r/egit/egit-pde.git

EGit还提供了与PDE Build和Eclipse RelEng Tools集成的工具。如果您是使用PDE Build和/或Eclipse RelEng工具的Eclipse开发人员，您可能会对以下内容感兴趣。否则你可能会跳过本节。

除了JGit和EGit所需的依赖项之外，您还需要在目标平台中使用Eclipse PDE（> = 3.6.1）以及org.eclipse.releng.tools ，或者在工作区中从Git中检出。

开发IDE配置

如果您还没有，请从此处下载并安装Eclipse软件包“Eclipse IDE for Eclipse Committers”或“Eclipse for RCP and RAP Developers”：

https://www.eclipse.org/downloads/packages/

工具

要安装所有必要的工具来处理EGit / JGit，有一个egit-developer-tools.p2f文件，你可以使用如下：

  • 文件>导入>安装>从文件安装软件项
  • 浏览...
    • 转到egit存储库的位置，打开tools目录并选择egit-developer-tools.p2f
    • 或者，如果您只想为JGit做贡献，请从上面的链接下载文件并选择它
  • 应自动选择您尚未拥有的所有项目
  • 完成向导
  • 重新开始

Java要求

EGit和JGit将Java 8.0和Eclipse Platform 4.4（Luna）作为最低要求，因此必须避免对较新的Java和平台版本的依赖。

我们正在使用API工具环境描述 （请参阅JGit和EGit的更改）以便于检测不适用于Java 8的代码。如果您按照上面“ 工具”部分中的说明操作，则应已安装必要的说明。否则，请从发行版存储库中安装API工具环境描述 ，请参阅安装执行环境描述 。

依赖

在Eclipse中导入EGit和JGit项目后，由于缺少依赖项，它们将无法编译。设置目标平台以解决此问题

  • 打开org.eclipse.egit.target项目
  • 选择与Eclipse平台版本匹配的egit- <version> .target文件（例如4.5 for Mars）并打开它（这可能需要一段时间，因为它下载了目标平台所指的p2存储库的索引）
  • 在结果编辑器中，单击右上角的“ 设置为目标平台”链接（这可能还需要一段时间，因为它下载了依赖项）

在那之后，工作区应该干净利落地构建。如果没有，请尝试Project> Clean ...> All。如果这也无法帮助打开首选项>插件开发>目标平台，请选择已选中的目标平台并单击“重新加载...”，这将刷新PDE的捆绑缓存并重新下载目标平台中列出的工件。

有不同的目标定义，一个用于EGit支持的每个Eclipse版本。如果要尝试功能或错误修复，您选择的那个将是启动的那个。

您始终可以在它们之间切换以在不同的Eclipse版本上进行测试。例如，当您开发一些主要的UI功能时，您应该使用最早支持的Eclipse版本来尝试它，以确保它不依赖于仅在更高版本中可用的API。
