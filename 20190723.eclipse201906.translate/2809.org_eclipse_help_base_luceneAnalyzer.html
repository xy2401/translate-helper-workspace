Lucene分析仪

标识符：
org.eclipse.help.base.luceneAnalyzer

以来：
3.0（最初在2.0版中添加为org.eclipse.help.luceneAnalyzer）

描述：
此扩展点用于在索引和搜索文档时注册文本分析器以供帮助使用。

帮助利用Lucene搜索引擎的功能，允许对令牌流（单词流）进行索引。分析器从字符流中创建标记。他们检查文本内容并提供与索引一起使用的令牌。可以通过许多独特方式对文本流进行标记。一个简单的分析器可以在白色空间标记化流，不同的分析器可以根据应用程序的需要执行标记过滤。由于文档主要是人类可读的文本，因此希望帮助系统使用的分析器执行语言和语法识别标记化以及索引文本的标准化。对于某些语言，如果对索引文本执行停用词删除和词干，则搜索质量会显着提高。

对此扩展点有贡献的分析器将覆盖Eclipse帮助系统为给定语言环境提供的分析器。

配置标记：

<！ELEMENT 扩展 （ 分析器 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 分析仪 EMPTY>

<！ATTLIST分析仪

locale CDATA #REQUIRED

class CDATA #REQUIRED>

  • locale - 标识语言环境的字符串，提供的分析器将使用该语言环境。如果提供了两个字母，语言，则分析器将可用于该语言的所有语言环境。
  • class - 扩展org.apache.lucene.analysis的Java类的完全限定名称。分析仪 。

例子：
以下是Lucene Analyzer配置的示例：

<extension id = “com.xyx。XYZ“ point = ”org.eclipse.help.base.luceneAnalyzer“ > <analyzer locale = ”ll_CC“ class = ”com.xyz.ll_CCAnalyzer“ /> </ extension>

API信息：
locale属性的值必须表示五个或两个字符的区域设置字符串。如果通过指定双字母语言指定为某种语言配置分析器，则分析器将用于该语言的所有语言环境。如果分析器配置为匹配五个字符的区域设置，则将使用它。

class属性的值必须表示扩展org.apache.lucene.analysis的类。分析仪 。建议此分析器对可能通过使搜索区分大小写来增加搜索命中次数的语言执行小写过滤。

提供的实施：
Eclipse帮助系统为所有语言提供分析器。对于英语和德语，分析仪执行停用词过滤，小写过滤和词干分析。对于所有其他语言，提供的分析器仅执行小写过滤。
