分析配置

如上所述，每个快捷方式在Profile as ...上下文菜单下创建一个菜单项（例如，右键单击C / C ++可执行文件）。除了每个类别的快捷方式外，最终用户还可以选择配置文件配置...

这会打开一个对话框，允许用户修改当前的启动配置或创建新配置：

所有Linux Tools分析配置中都提供了以下选项卡：

  • 主要
    • 此选项卡允许用户设置C / C ++可执行文件和Eclipse项目。
  • 参数
    • 此选项卡设置在分析时传递给C / C ++程序的参数。
  • 环境
    • 此选项卡设置在运行程序和分析时要设置的任何环境变量。您可以选择覆盖当前环境或完全替换它。
  • 资源
    • 此选项卡设置源查找路径，Linux Tools Profiling插件可以使用该路径通过UI打开源代码。
  • 共同
    • 此选项卡允许指定配置的存储方式，将其标记为最喜欢的配置文件配置，以及在使用C / C ++标准输入/输出的情况下设置控制台。

通过所有快捷方式，可以为您设置一组默认参数，例如：可执行文件将是您右键单击的项目，项目将自动设置为其项目。如果您希望从头开始创建新配置，只需通过单击左侧树视图选择要创建的性能分析启动配置类型，然后按左上角找到的图标。这将创建具有一些默认值的新配置，但某些选项可能需要手动设置（例如可执行文件和项目）。

除了常用选项卡之外，每个类别都为自己的设置提供了一个选项卡。例如，对于Profile Memory，添加了一个Memory选项卡：

类别选项卡将具有类别的名称（即在本例中为内存），并在顶部有一个下拉列表，允许您指定要使用的工具。默认情况下，它将在“性能分析类别”首选项页面中为类别或初始默认值选择当前工具。您可以通过打开下拉框并选择其他工具（如果为该类别安装了一个工具）来覆盖此配置。

在此之下，请注意还有另一组选项卡。这些选项卡特定于为顶部选择的性能分析工具设置选项。有关选项含义的详细信息，请参阅相应的插件用户指南，提供用于分析的工具（例如，在上面的“内存”选项卡中，这将是Valgrind工具的用户指南）。

当您从Profile as ...上下文菜单（例如Profile Memory）中选择一个快捷方式时，如果当前对于所选的C / C ++可执行文件不存在，将为您创建启动配置。如果有多个配置指向同一个可执行文件（显然具有不同的配置名称），则会提供一个对话框，供您选择要使用的配置文件。

如果C / C ++可执行文件的启动配置已存在，但它指定了与“概要分析类别”首选项页面中设置的当前设置不同的默认工具，则还将创建新配置。因此，如果您已经有一个启动配置，那么现在将有两个配置，这将触发多个配置选择对话框。新配置的唯一名称是从旧的配置创建的，添加数值（例如（2））。您可以自由重命名任何配置，以便通过“性能分析配置”对话框进行选择。
