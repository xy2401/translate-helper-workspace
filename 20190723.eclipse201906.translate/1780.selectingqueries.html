选择查询

Memory Analyzer提供了一组查询来分析堆转储。这些查询分为几个类别。

您可以从下拉工具栏菜单中选择一个查询。



在菜单中，您还可以通过历史记录条目访问已执行查询的列表。查询选择器可以使用Ctrl + Q组合键快速访问。



查询选择器包含一个输入字段，您可以在其中搜索查询，在输入字段下方可以找到已执行查询的历史记录，在历史记录下方可以查看按类别分组的所有可用查询的列表。标记查询时，底部会显示查询的详细说明。此描述还包含有关此查询的参数的信息。

查询浏览器（“搜索查询...”）也可在上下文菜单中使用。它仅提供对当前选择有效的查询，并且更易于使用，因为它可以显示部分或全部查询，并且还显示每个查询的帮助。

如果查询具有参数，则在选择查询时会打开向导。所有参数都显示在表中，强制参数为粗体。在消息区域中，显示有关必需参数的顶部信息。在底部，详细说明显示在查询选择器中。



提供查询参数

要将参数添加到查询，可以使用查询浏览器或参数对话框。



强制性参数以粗体显示。对话框下方的工具提示和帮助infopop提供参数说明。对话框本身的消息区域显示有关所选参数的信息，如果输入的值无效，则显示错误消息。有些查询需要一组对象作为参数。您可以使用对话框的简单或高级模式提供这些对象。在下图中，您可以看到简单模式。一组对象由模式定义。



模式语法

模式支持的元字符是： （ ， [ ， { ， \ ， ^ ， - ， $ ， | ， ] ， } ， ） ,? ， *和+ 。

模式语法 细节 
[abc]
 a，b或c（简单类） 
[^abc]
 除a，b或c之外的任何字符（否定） 
outbounds( object )
 出境推荐人 
[a-zA-Z]
 a到z，或A到Z，包括（范围） 
[a-d[m-p]]
 a到d，或m到p：[a-dm-p]（联合） 
[a-z&&[def]]
 d，e或f（十字路口） 
[a-z&&[^bc]]
 a到z，b和c除外：[ad-z]（减法） 
[a-z&&[^m-p]]
 a到z，而不是m到p：[a-lq-z]（减法） 
\d
 数字：[0-9] 
\D
 非数字：[^ 0-9] 
\s
 空格字符：[\ t \ n \ x0B \ f \ r \ n] 
\S
 非空白字符：[^ \ s] 
\w
 单词字符：[a-zA-Z_0-9] 
\W
 非单词字符：[^ \ w]

在高级模式下，您可以使用模式，对象地址或对象查询语言（OQL）查询定义一组对象。图标和帮助消息为您提供有关参数类型的信息。



您可以为每行输入一个参数，例如模式，对象地址或OQL查询。此类参数的附加行会自动添加，因此可以输入多个参数。

对象地址语法

对象地址具有十六进制形式，即以十六进制前缀0x开头，由一个或多个字母AF和数字0-9组成。

示例： 0x36d99c88

有关OQL的更多信息，请参阅OQL语法详细信息
