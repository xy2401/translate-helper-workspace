逻辑结构类型

标识符：
org.eclipse.debug.core.logicalStructureTypes

以来：
3.0

描述：
此扩展点允许调试器提供值的替代逻辑结构。通常，复杂的数据结构在其逻辑结构方面更方便导航，而不是在其实现方面。例如，无论如何实现列表（链接，数组等），用户可能只想根据有序集合来查看列表中的元素。此扩展点允许逻辑结构类型的贡献，以提供从原始实现值到逻辑值的转换。

配置标记：

<！ELEMENT 扩展 （ logicalStructureType *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT logicalStructureType EMPTY>

<！ATTLIST logicalStructureType

id CDATA #REQUIRED

描述CDATA #IMPLIED

class CDATA #REQUIRED

modelIdentifier CDATA #REQUIRED>

  • id - 此逻辑结构类型的唯一标识符
  • description - 对此逻辑结构的描述。从3.1开始，此属性是可选的。未指定时，逻辑结构类型委托还必须实现ILogicalStructureTypeDelegate2以提供为其提供逻辑结构的值的描述。
  • class - 实现ILogicalStructureTypeDelegate的Java类的完全限定名称。该类可以选择实现ILogicalStructureTypeDelegate2 。
  • modelIdentifier - 此逻辑结构类型与之关联的调试模型的标识符

例子：
以下是逻辑结构类型扩展点的示例：

<extension point = “org.eclipse.debug.core.logicalStructureTypes” > <logicalStructureType id = “com.example。ExampleLogicalStructure“ class = ”com.example。ExampleLogicalStructureDelegate“ modelIdentifier = ”com.example.debug.model“ description = ”Ordered Collection“ > </ logicalStructureType> </ extension>
在上面的示例中，将为com.example.debug.model调试模型中的值的备用逻辑值查询指定的逻辑结构类型，因为它们显示在变量视图中。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。ILogicalStructureTypeDelegate 。
