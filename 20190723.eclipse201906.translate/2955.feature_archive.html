特色档案

功能包装信息放在单独的Java .jar中。标准Java jar工具用于构建功能存档。功能存档引用单独打包的插件存档（请参阅下一节）和非插件文件。

使用基于提供商互联网域名的结构化标识符来标识特征。例如，组织eclipse.org生成功能org.eclipse.jdt。用于功能标识符的字符集与插件标识符一样（请参阅插件清单 ）。

命名功能存档的建议约定是
<ID> _ <版本>的.jar

其中<id>是要素标识符， <version>是相应feature.xml中包含的完整版本标识符。请注意，这是建议的约定，可以最大限度地减少冲突的可能性，但Eclipse体系结构不需要这样做。例如，以下是有效的功能存档名称

org.eclipse.jdt_2.0.0.jar
org.eclipse.pde_2.0.jar
my_feature.jar

在内部，每个功能存档都相对于其功能目录打包（但不包括目录路径元素）。存档具有以下结构

feature.xml的
feature <_locale> .properties（参见“翻译的特征信息”）
其他功能文件和子目录（TBD）
META-INF /
Java jar清单和安全文件

请注意，功能存档不包含其组成插件和片段。
