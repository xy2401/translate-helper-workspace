插件内容向导

标识符：
org.eclipse.pde.ui.pluginContent

描述：
此扩展点提供了有助于向导创建PDE插件项目的其他内容的向导。在创建插件清单和密钥文件之后，可以使用这些向导向初始结构添加更多文件和扩展。此向导的典型实现将基于根据向导中的用户选项自定义的参数化模板添加内容。目标是在创建后立即创建一个有用的插件（例如，提供视图，编辑器等）。

配置标记：

<！ELEMENT 扩展 （ 向导 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 向导 （ 描述 ？）>

<！ATTLIST向导

id IDREF #REQUIRED

名称CDATA #REQUIRED

icon CDATA #IMPLIED

class CDATA #REQUIRED

类别CDATA #IMPLIED

ui-content（true | false）“true”

java（true | false）“true”

rcp（true | false）“false”

pureOSGi（true | false）“false”

requiresActivator（true | false）“false”>

  • id - 将用于标识此向导的唯一名称。
  • name - 将在此向导的UI表示中使用的可翻译名称。
  • icon - 将用于直观表示向导的图标的相对路径。
  • class - 实现org.eclipse.pde.ui的类的完全限定名称。IPluginContentWizard并从org.eclipse.jface扩展。向导 。
  • category - 可选标记，可用于将内容向导与不同的目标项目相关联。
  • ui-content - 一个标志，指示向导是否将使用用户界面内容提供代码。此标志将影响将选择哪个插件类（UI插件扩展AbstractUIPlugin类，而非UI插件扩展Plugin基类）。由于对Eclipse的许多贡献都有UI内容，因此默认情况下此属性为true 。
  • java - 一个标志，指示向导将提供Java内容。由于大多数Eclipse插件都具有Java代码，因此默认情况下该属性为true 。如果插件没有Java代码（例如，仅限文档文件），请将其设置为false 。
  • rcp - 从3.1开始。一个布尔标志，指示向导是否提供独立的全功能富客户端应用程序。如果设置为true ，则仅当用户选择“富客户端应用程序”选项时，向导才会显示在“新建插件项目”向导中。
  • pureOSGi - 自3.2。一个布尔标志，指示向导是否提供纯OSGi包。如果设置为true ，则仅当用户选择创建纯OSGi包时，向导才会出现在New Plug-in Project向导中。
  • requiresActivator - 自3.2。一个布尔标志，指示向导是否提供激活器类。如果设置为true ，则仅当用户选择生成激活器以控制插件生命周期时，向导才会出现在New Plug-in Project向导中。

<！元素描述 （#PCDATA）>

这个向导的简短描述。



例子：
以下是此扩展点的示例：
<extension point = “org.eclipse.pde.ui.pluginContent” > <wizard name = “Example Plug-in Content Generator” icon = “icons / content_wizard.png” class = “com.example.xyz。ContentGeneratorWizard“ id = ”com.example.xyz。ExampleContentGenerator“ > <description>添加视图和首选项页面。</ description> </ wizard> </ extension>

API信息：
插入此扩展点的向导必须实现org.eclipse.pde.ui。IPluginContentWizard接口并期望扩展org.eclipse.jface.wizard.Wizard 。

提供的实施：
PDE提供基于可自定义模板的贡献内容向导的API。PDE UI本身提供了许多基于这些模板的具体向导。
