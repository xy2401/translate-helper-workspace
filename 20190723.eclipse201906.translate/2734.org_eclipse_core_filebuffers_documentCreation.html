文件创作

不推荐使用此扩展点

标识符：
org.eclipse.core.filebuffers.documentCreation

以来：
3.0

描述：
此扩展点用于自定义此插件的默认文本文件缓冲区管理器的文档创建行为。它允许指定应使用哪个文档工厂，以便创建为特定文件内容类型，文件扩展名或文件名创建的文本文件缓冲区的文档实例。

自3.2以来已弃用：
必须不再使用此扩展名。我们敦促现有客户取消他们的贡献：
  • 文件缓冲区管理器现在提供ISynchronizable文档。需要同步文档的客户端可以设置锁定对象。覆盖现有的锁定对象被认为是不好的做法，可能会破坏其他客户端：如果文档已经设置了锁定对象，那么客户端应该使用此对象 。
  • 需要不同的org.eclipse.jface.text.IDocument实现在其文档提供程序中创建相应的文档。自定义文档将包装文件缓冲区提供的文档。

配置标记：

<！ELEMENT 扩展 （ 工厂 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT factory EMPTY>

<！ATTLIST工厂

class CDATA #REQUIRED

扩展CDATA #IMPLIED

fileNames CDATA #IMPLIED

contentTypeId CDATA #IMPLIED>

文件工厂的规范。为了找到给定文件的工厂，按以下顺序查询每个工厂规范的属性：contentTypeId，fileNames，extensions。如果为给定文件找到多个同样特定的工厂规格，则不指定使用哪个工厂。


  • class - 工厂实现类的完全限定名称。该类必须实现org.eclipse.core.filebuffers.IDocumentFactory接口。
  • extensions - 以逗号分隔的文件扩展名列表，应使用此工厂
  • fileNames - 以逗号分隔的文件名列表，应使用此工厂
  • contentTypeId - 应该使用此工厂的org.eclipse.core.runtime.contentTypes扩展点定义的内容类型的id

例子：

<extension
     id="org.eclipse.jdt.debug.ui.SnippetDocumentFactory"
     name="%snippetDocumentFactory.name"
     point="org.eclipse.core.filebuffers.documentCreation">
  <factory
        extensions="jpage"
        class="org.eclipse.jdt.internal.debug.ui.snippeteditor.SnippetDocumentFactory">
  </factory>
</extension>

API信息：
文档工厂必须实现org.eclipse.core.filebuffers。IDocumentFactory。
