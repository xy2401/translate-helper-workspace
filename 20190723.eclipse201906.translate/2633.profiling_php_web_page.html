分析PHP Web页面

此过程描述如何配置服务器上已有的整个应用程序，项目，文件或文件集。

  1. 单击“配置文件”按钮旁边的箭头在工具栏上，选择配置文件配置... - 或 - 从主菜单转到运行|配置文件配置... - 或者在Project Explorer视图中右键单击并选择Profile As |配置文件配置...
  2. 将出现配置文件启动配置对话框。
  3. 双击“ PHP Web应用程序”选项以创建新的配置文件配置。

  4. 输入新配置的名称。
  5. 从PHP Server：列表中选择您的服务器。如果尚未配置服务器，请单击“ 新建” ，将打开“PHP服务器创建向导”。
  6. 在“ PHP文件”下 ，单击“ 浏览”并选择“配置文件目标”文件（从中开始配置文件处理的文件）。
  7. 有关进一步的性能分析选项，请选择“ Profiler”选项卡，其中包含以下选项：
    • SSH隧道 - 使用此组选项可以使用SSH隧道来执行配置文件会话
    • 在浏览器中打开 - 标记您是否希望在浏览器中显示该应用程序

  8. 单击Apply ，然后单击Profile 。
  9. 将显示一个确认对话框，询问您是否要打开性能分析透视图。单击是。（如果您希望将来默认打开性能分析透视图，请选中“记住我的决定”复选框。）

PHP Profile Perspective将打开，显示Profiling Monitor窗口，其中包含各种Profiling视图。

有关配置文件会话运行后显示的信息的更多信息，请参阅PHP配置文件透视图 。

注意：

对于Xdebug，请确保已启用xdebug.profiler_enable_trigger 。

注意：

使用Xdebug探查器，您必须手动导入配置文件会话 。
