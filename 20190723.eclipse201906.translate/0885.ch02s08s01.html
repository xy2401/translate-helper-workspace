2.8.1。�使用API进行测试的嵌入式代理的一般信息和设置

可以在测试中启动AUT代理。因此，无需其他Jubula应用程序即可完全编写和执行测试。除HTML外，所有工具包均受支持。

常规设置信息

以下部分描述了在何处查找使用EmbeddedAgent for API的相关JAR /工件。提供的EmbeddedAgent具有Java 8所需的最低运行时环境。

除了客户端API jar之外，还必须使用JAR / maven工件/ OSGi包（请参阅第2 章， 通过客户端API编写UI测试 ）。这些Jars必须位于类路径中：

  1. 每个项目都需要依赖于API的所有 工具包中性部分 ：

    • org.eclipse.jubula.autagent.api

    • org.eclipse.jubula.autagent.common

    • org.eclipse.jubula.client.api

    • org.eclipse.jubula.toolkit.api

    • org.eclipse.jubula.rc.common.agent

    • org.eclipse.jubula.tools

    • org.eclipse.jubula.tools.exec

    • org.eclipse.jubula.communication

  2. 以下是类路径上需要的第三方库。

    • org.apache.commons.lang

    • org.apache.commons.io

    • org.apache.commons.exec

    • org.apache.commons.codec

    • org.apache.commons.beanutils

    • org.apache.oro

    • org.eclipse.osgi

    • javax.xml

    • javax.xml.stream

    • org.slf4j.api

    • ch.logback.classic

    • ch.logback.core

    • org.xmlpull

    • com.thoughtworks.xstream

  3. 以下jar可以在类路径上，但不是必需的。但重要的是它们与org.eclipse.jubula.autagent.api位于同一目录中。以下是独立于工具包的需要：

    • org.eclipse.jubula.rc.common

    每个项目都需要依赖于工具包特定元素的 各个部分，这些元素依赖于API的使用工具包：

    • org.eclipse.jubula.rc.javafx

    • org.eclipse.jubula.rc.javafx.j8u40

    • org.eclipse.jubula.rc.swing

    • org.eclipse.jubula.rc.swt

    未列出RCP，因为rc.rcp插件已部署到AUT中

您可以在<ITEInstallationFolder> / development / api文件夹中找到不同形状的工件（纯JAR和maven工件）。要定义与API工件的OSGi依赖关系，您只需在目标平台（定义）中包含<ITEInstallationFolder> /development/org.eclipse.jubula.repo.zip ，并定义对“Jubula Functional Testing - API Embedded”的依赖关系 -特征。
