关键字索引

标识符：
org.eclipse.help.index

以来：
3.2

描述：
用于为贡献的帮助内容注册关键字索引。

配置标记：

<！ELEMENT 扩展 （ index | indexProvider ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 索引 EMPTY>

<！ATTLIST指数

文件CDATA #REQUIRED>

通过提供XML文件做出的索引贡献


  • file - 索引文件的名称，其中包含帮助内容的关键字列表和相关主题。

    索引文件的配置标记：

    <！ELEMENT索引（条目）*>

    <！ELEMENT条目（条目|主题|见）*>
    <！ATTLIST条目关键字CDATA #REQUIRED>

    <！ELEMENT主题EMPTY>
    <！ATTLIST主题href CDATA #REQUIRED>
    <！ATTLIST主题标题CDATA #IMPLIED>

    <！ELEMENT见（子路径）*>
    <！ATTLIST请参阅关键字CDATA #REQUIRED>

    <！ELEMENT子路径EMPTY>
    <！ATTLIST子路径关键字CDATA #IMPLIED>

    索引元素

    index元素表示索引的实例。索引由条目组成，每个条目对应一个特定的关键字。所有提供的索引都合并到帮助系统提供给用户的一个主索引。构建索引时，条目按字母顺序附加。

    入口元素

    entry元素表示索引的关键字。每个条目可能包含多个链接，以帮助与关键字相关联的主题。条目也可以是其他条目（子条目）的容器，以形成关键字的层次结构。允许它进入子条目的容器并同时拥有主题。

    主题元素

    topic元素提供对与关键字相关的帮助内容的引用。href属性与清单文件所属的插件相关。如果需要访问另一个插件中的文件，可以使用该语法

    <topic label =“另一个插件中的主题”href =“../ other.plugin.id/concepts/some_other_file.html”/>

    title属性用于命名链接以在多个主题与单个关键字关联时帮助内容。如果未指定该属性，则标题来自label属性，否则来自拥有该链接的TOC。如果链接不属于任何TOC，则必须指定title或label属性，否则标题将变为未定义。

    see元素 see元素表示索引中条目的同义词。单击查看链接将导致索引导航到这是同义词的索引条目。keyword属性表示同义词的顶级元素。仅当同义词不在索引的顶级并且表示同义词路径的其他组件时，才使用子路径元素。


<！ELEMENT indexProvider EMPTY>

<！ATTLIST indexProvider

class CDATA #REQUIRED>

（ 自3.3起 ）通过插入代码做出的索引贡献


  • class - 索引提供程序的实现类。该类必须实现org.eclipse.help。AbstractIndexProvider接口。

例子：
以下是使用索引扩展点的示例。

（在文件plugin.xml中 ）

<extension point = “org.eclipse.help.index” > <indexProvider class = “com.myplugin。MyIndexProvider“ /> </ extension> <extension point = ”org.eclipse.help.index“ > <index file = ”index.xml“ /> </ extension>

（在文件index.xml中 ）

国际化可以翻译索引XML文件，并将生成的副本（带有翻译的关键字）放在nl / <language> / <country>或nl / <language>目录中。<language>和<country>代表语言环境代码中使用的双字母语言和国家/地区代码。例如，繁体中文翻译应放在nl / zh / TW目录中。nl / <language> / <country>目录的优先级高于nl / <language>。只有在nl / <language> / <country>中找不到文件时，才会使用驻留在nl / <language>中的文件。最后将搜索插件的根目录。

API信息：
org.eclipse.help的一个实现。如果使用indexProvider，则必须提供AbstractIndexProvider 。

提供的实施：
Eclipse平台提供的帮助系统UI的默认实现完全支持索引扩展点。
