出口奇才

标识符：
org.eclipse.ui.exportWizards

描述：
此扩展点用于注册导出向导扩展。导出向导在“导出对话框”中显示为选项，用于从工作台导出资源。

向导可以选择指定描述子元素，其主体应包含有关向导的简短文本。

配置标记：

<！ELEMENT 扩展 （ 类别 | 向导 ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT keywordReference EMPTY>

<！ATTLIST keywordReference

id IDREF #REQUIRED>

导出向导对关键字的引用。请参阅关键字扩展点。


  • id - 要引用的关键字的ID。

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

id CDATA #REQUIRED

名称CDATA #REQUIRED

parentCategory IDREF #IMPLIED>

  • id - 可用于标识此类别的唯一名称
  • name - 将在对话框中使用的类别的可翻译名称
  • parentCategory - 如果应将此类别添加为子类，则为另一个类别的路径

<！ELEMENT 向导 （ 描述 ？， 选择 *， 关键字参考 *）>

<！ATTLIST向导

id CDATA #REQUIRED

名称CDATA #REQUIRED

类别CDATA #IMPLIED

class CDATA #REQUIRED

icon CDATA #IMPLIED>

将用于创建导出向导的元素


  • id - 将用于标识此向导的唯一名称
  • name - 将在对话框中用于表示此向导的可翻译名称
  • category - 类别ID的斜杠分隔路径（'/'）。路径中的每个标记必须表示此前或其他插件先前定义的有效类别ID。如果省略，则向导将添加到“其他”类别。
  • class - 实现org.eclipse.ui的类的完全限定名称。IExportWizard界面
  • icon - 将与导出引擎列表中的向导名称一起使用的图标的相对名称。

<！元素描述 （#PCDATA）>

一个可选的子元素，其主体应代表导出引擎功能的简短描述。



<！ELEMENT 选择 EMPTY>

<！ATTLIST选择

名称CDATA #IMPLIED

class CDATA #REQUIRED>

一个可选元素，用于限制调用向导时可以选择的对象的类型和名称。


  • name - 可选的名称过滤器。工作台选择中的每个对象都必须与要传递给向导的名称过滤器匹配。
  • class - 完全限定的类名。如果工作台选择中的每个对象都实现此接口，则选择将传递给向导。否则，传递空选择。

例子：
以下是导出扩展定义的示例：

<extension point = “org.eclipse.ui.exportWizards” > <wizard id = “com.xyz。ExportWizard1“ name = ”XYZ Web Exporter“ class = ”com.xyz.exports。ExportWizard1“ icon = ”./ icons / import1.gif“ > <description>导出Web项目的简单引擎</ description> <selection class = ”org.eclipse.core.resources。IProject“ /> </ wizard> </ extension>

API信息：
class属性的值必须是实现org.eclipse.ui的类的名称。IExportWizard 。

提供的实施：
工作台预装了文件和目录的基本导出引擎。
