过滤器和过滤资源

Remote System Explorer是一组本地客户端工具，用于组织您希望如何使用远程服务器上的文件夹和文件等资源。下图用作Remote System Explorer的一些主要组件的介绍：

在“远程系统资源管理器”透视图中，您将查询服务器以列出远程资源（如文件夹和文件）并执行操作。您可以使用在工作站上显示这些资源的过滤器在这些远程资源上执行这些任务。

Remote System Explorer透视图旨在允许您直接在远程系统上操作资源。可用的操作取决于您要连接的系统类型以及资源的识别方式。例如，您的选择可以定义过滤器字符串，以查找与特定目录中的* .c匹配的所有文件。

然后，过滤器显示远程系统上的资源。它们显示的资源类型取决于您正在查看的子系统。例如，文件子系统显示远程系统上的文件和目录。流程子系统将显示正在运行的流程。

文件子系统的功能非常丰富。例如，您可以在远程系统之间复制文件和文件夹，而不管您与这些远程系统的连接类型如何。只需快速说明：如果过滤器显示目录，则将该目录复制到远程系统将导致该文件夹中的所有文件被复制，而不仅仅是过滤器显示的文件。这与Eclipse中其他地方管理资源的方式一致。

过滤池

如果您已经使用远程系统资源管理器一段时间，您的工作区可能包含太多过滤器，无法轻松导航。或者，您可能只想将过滤器组分开，例如，如果您需要在远程系统资源管理器中表示两个不同的环境，而不管您拥有多少个过滤器。无论哪种情况，您都可以将过滤器分组到过滤器池中 。

例如，一个筛选器池可以包含应收帐款程序的筛选器，而另一个筛选器池包含工资核算程序的筛选器。或者，如果服务器上有两个不同的环境：生产环境和测试环境，则可以为每个环境定义不同的环境和一组过滤器。

如果没有过滤器池，则所有过滤器都会在所有连接中一起显示。但是，在创建过滤器池时，您在该过滤器池中创建的任何过滤器都与该连接不同，并且不会出现在任何其他连接中。（有关过滤器池的更多信息，请参阅相关链接。）

注意：此时此筛选器池名称不能包含三个连续的下划线字符。

共享过滤器

您可以通过使用过滤器池引用在多个连接之间共享过滤器池 。筛选器池引用是一种在任何其他连接中显示来自一个连接的筛选器池的机制，因此当您对原始筛选器池进行更改时，您的更改将反映在筛选器池引用中。

所有连接，过滤器，过滤器池等都属于您在首次连接到服务器时定义的远程系统资源管理器配置文件 。如果删除连接，您可能想知道过滤器或过滤器池会发生什么情况，特别是如果您在两个或多个连接之间共享过滤器。请放心，过滤器不会被删除，因为过滤器实际上属于配置文件，而不是连接。

请记住，如果您需要在远程系统资源管理器中拥有全新的过滤器池，过滤器和连接配置，则始终可以创建新的配置文件。有关更多信息，请参阅相关链接。
