调试PHP网页

此过程描述如何调试服务器上已有的整个应用程序，项目，文件或文件集合。

注意：

您的服务器必须运行Xdebug或Zend Debugger才能使远程调试功能正常运行。

要调试PHP Web页面：

1。单击调试按钮旁边的箭头在工具栏上，选择打开调试对话框 - 或 - 选择运行|打开调试对话框。将打开“调试”对话框。

2。双击PHP Web Application选项以创建新的调试配置。

3。输入新配置的名称。

4。从PHP Server列表中选择您的服务器。如果尚未配置服务器，请单击“新建”。PHP服务器创建向导将打开。按照PHP Servers Preferences页面下的“ 添加新服务器 ”的说明配置服务器。

5。在“文件”下，单击“浏览”并选择“调试目标”文件（调试进程将从该文件开始的文件）。

6。将根据文件名和服务器地址自动创建要调试的URL。如果URL未指向调试目标的位置，请取消选中“自动生成”复选框并修改URL。

7。要设置附加到所选PHP服务器的调试器的选项，请单击“ 调试器”选项卡。

8。要检查服务器连接是否已正确配置，以及PDT是否可以与服务器调试器通信，请单击“测试”按钮。

9。通过标记/取消标记“在第一行中断”复选框，选择调试器是否应停在第一行代码处。

10。您可以设置一些与调试器类型相关的其他选项，例如Zend Debugger可以使用以下选项：

  • 通过SSH隧道调试 - 标记是否要使用SSH隧道进行调试。
  • 在浏览器中打开 - 标记您是否希望在调试时将应用程序显示在PDT的内部浏览器中。
  • 选择是否：
  • 调试所有页面 - 调试指定页面和链接到它的所有页面。浏览器在显示之前等待每个页面的调试。
  • 仅调试第一页 - 仅调试第一页。
  • 从中启动调试 - 选择要从中启动调试过程的URL。
  • 从此页面继续调试 - 选择此选项将导致链接到URL的所有页面都被调试。

11。单击Apply，然后单击Debug。

12。如果询问是否打开PHP Debug Perspective，请单击Yes。

有关调试过程结果的更多信息，请参阅“ 运行和分析调试器结果 ”主题。

注意：

如果文件包含对项目中未包含的文件的“包含”或“要求”调用，则必须将它们添加到项目的“包含路径”中，以便模拟生产环境。此外，如果调用使用服务器位置的绝对路径定义的文件（有关绝对文件位置的更多信息，请参阅“ 包含路径 ”），将显示“路径映射”对话框。有关更多信息，请参阅路径映射
