结果分拣机

不推荐使用此扩展点

标识符：
org.eclipse.search.searchResultSorters

描述：
此扩展点允许插件将搜索结果排序器提供给（旧）搜索结果视图的排序上下文菜单。此扩展点自3.0以来已弃用。搜索结果视图现在由客户端提供，分类器由这些页面直接管理。

配置标记：

<！ELEMENT 扩展 （ 分拣机 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 分拣机 EMPTY>

<！ATTLIST分拣机

id CDATA #REQUIRED

pageId IDREF #REQUIRED

标签CDATA #REQUIRED

工具提示CDATA #IMPLIED

icon CDATA #IMPLIED

class CDATA #REQUIRED>

  • id - 将用于标识此搜索结果排序器的唯一名称。
  • pageId - 将为其激活分拣机的已注册搜索页面的ID。 “*”激活所有页面的分拣机。
  • label - 可翻译标签，将用作菜单项的标签。
  • tooltip - 可翻译的文本，将用作菜单项的工具提示。如果省略，菜单项将没有工具提示。
  • icon - 将在上下文菜单中显示的图像的相对名称以及标签。如果省略，菜单条目将仅包含标签。
  • class - 扩展org.eclipse.jface.viewers.ViewerSorter

例子：
以下是搜索页面扩展定义的示例：


   <extension point="org.eclipse.search.searchResultSorters"> 
    <sorter 
        id="org.eclipse.search.internal.ui.FileNameSorter" 
        pageId="*" 
        label="%FileNameSorter.label" 
        tooltip="%FilenNameSorter.tooltip" 
        icon="icons/full/ecl16/search_sort.gif" 
        class="org.eclipse.search.internal.ui.FileNameSorter"> 
    </sorter> 
   </extension>

API信息：
贡献的类必须实现org.eclipse.jface.viewers.ViewerSorter

提供的实施：
搜索基础结构提供了一个排序器，可以按资源名称对匹配进行排序。
