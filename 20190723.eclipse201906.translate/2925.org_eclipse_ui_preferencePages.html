偏好页面

标识符：
org.eclipse.ui.preferencePages

描述：
工作台为首选项提供了一个通用对话框。此扩展点的目的是允许插件将页面添加到首选项对话框中。打开首选项对话框（从菜单栏启动）时，以这种方式提供的页面将添加到对话框中。

首选项对话框提供页面的分层分组。因此，页面可以选择指定类别属性。此属性表示由以“/”分隔的父页面ID组成的路径。如果省略此属性或者无法找到路径中的任何父节点，则将在根级别添加该页面。

配置标记：

<！ELEMENT 扩展 （ 第 * 页 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 页面 （ keywordReference *）>

<！ATTLIST页面

id CDATA #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED

类别IDREF #IMPLIED>

  • id - 将用于标识此页面的唯一名称。
  • name - 将在此页面的UI中使用的可翻译名称。
  • class - 实现org.eclipse.ui的完全限定类的名称。IWorkbenchPreferencePage 。
  • category - 指示首选项树中页面位置的路径。路径可以是父节点ID或由'/'分隔的ID序列，表示来自根节点的完整路径。

<！ELEMENT keywordReference EMPTY>

<！ATTLIST keywordReference

id IDREF #REQUIRED>

首选项页面对关键字的引用。请参阅关键字扩展点。


  • id - 要引用的关键字的ID。

例子：
以下是首选项扩展点的示例：

<extension point = “org.eclipse.ui.preferencePages” > <page id = “com.xyz.prefpage1” name = “XYZ” class = “com.xyz.prefpages。PrefPage1“ > <keywordReference id = ”xyz。关键字“ /> </ page> <page id = ”com.xyz.prefpage2“ name = ”Keyboard Settings“ class = ”com.xyz.prefpages。PrefPage2“ category = ”com.xyz.prefpage1“ > </ page> </ extension>

API信息：
属性类的值必须表示实现org.eclipse.ui的类的完全限定名称。IWorkbenchPreferencePage 。

提供的实施：
工作台添加了几个页面来设置平台的首选项。通过此扩展程序注册的页面将根据其类别信息添加到其后。
