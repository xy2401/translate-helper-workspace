配置复制映射

标识符：
org.eclipse.ui.externaltools.configurationDuplicationMaps

以来：
3.0

描述：
这是一个内部扩展点，用于声明在将现有配置复制为项目构建器时应创建的启动配置类型。客户端无意使用此扩展点。

配置标记：

<！ELEMENT 扩展 （ configurationMap *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT configurationMap EMPTY>

<！ATTLIST配置映射

sourceType IDREF #REQUIRED

builderType IDREF #REQUIRED>

  • sourceType - 指定为其提供此映射的启动配置类型的标识符
  • builderType - 指定将“sourceType”类型的配置导入为项目构建器时应创建的启动配置类型的标识符

例子：
以下示例指定当用户选择导入类型为“org.eclipse.ui.externaltools”的启动配置时。ProgramLaunchConfigurationType“，类型的新启动配置”org.eclipse.ui.externaltools。应该创建ProgramBuilderLaunchConfigurationType“。 
<extension point = “org.eclipse.ui.externaltools.configurationDuplicationMaps” > <configurationMap sourceType = “org.eclipse.ui.externaltools。ProgramLaunchConfigurationType“ builderType = ”org.eclipse.ui.externaltools。ProgramBuilderLaunchConfigurationType“ > </ configurationMap> </ extension>
