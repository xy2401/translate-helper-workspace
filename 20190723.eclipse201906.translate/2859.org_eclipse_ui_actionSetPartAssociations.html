操作集部件关联

不推荐使用此扩展点

标识符：
org.eclipse.ui.actionSetPartAssociations

描述：

警告：此扩展点已弃用。
请勿使用此扩展点，将在本产品的未来版本中将其删除。相反，请使用扩展点org.eclipse.ui.commands

此扩展点用于定义在透视图中打开零件（视图或编辑器）时应添加到透视图的操作集。对于编辑器，当编辑器是当前编辑器时，操作集将保持可见。对于视图，当视图是活动部件时，操作集将可见。

配置标记：

<！ELEMENT 扩展 （ actionSetPartAssociation *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

不推荐使用actionSetPartAssociation元素

<！ELEMENT actionSetPartAssociation （ part *）>

<！ATTLIST actionSetPartAssociation

targetID IDREF #REQUIRED>

  • targetID - 与特定工作台视图和/或编辑器关联的操作集的唯一标识符（在注册表中指定）。

不推荐使用part元素

<！ELEMENT part EMPTY>

<！ATTLIST部分

id CDATA #REQUIRED>

  • id - 要与操作集关联的部件（视图或编辑器）的唯一标识符。

例子：
以下是操作集部件关联的示例（请注意子元素和使用的方式属性）：


 <extension point="org.eclipse.ui.actionSetPartAssociations">
  <actionSetPartAssociation targetID="org.eclipse.jdt.ui.refactoring.actionSet">
   <part id="org.eclipse.jdt.ui.PackageExplorer"/>
   <part id="org.eclipse.jdt.ui.CompilationUnitError"/>
  </actionSetPartAssociation>
 </extension>
在上面的示例中，视图或编辑器与重构操作集相关联。

API信息：
用户可以使用自定义透视对话框覆盖这些关联。无论这些关联如何，用户关闭的动作集将永远不会出现，并且用户打开的动作集将始终可见。
