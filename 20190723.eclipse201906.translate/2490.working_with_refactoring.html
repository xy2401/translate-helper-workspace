使用重构

本教程的目的是教您如何重构代码以轻松地重命名和移动文件和元素，而不会损坏您的项目或切断引用项之间的链接，以及在单独的文件中创建PHP元素之间的必要链接。

内容：

  • 目的和用法
  • 重命名文件
  • 移动文件
  • 重命名文件中的元素

目的和用法

重构功能允许您：

  • 重命名并移动这些文件中的文件和元素，同时保持项目之间的链接。重命名或移动元素或文件后，项目中该项目的所有实例都将自动更新以反映其新名称/位置。
  • 组织包含，以便可以在另一个文件中引用一个文件中的元素。

重组项目并更改文件和元素的名称和位置时，应使用重构。

注意：

重构只能在Project Explorer视图中使用。

重命名文件

重命名文件将导致自动重命名在项目中引用该文件的所有实例。

此过程演示如何重命名文件：

  1. 使用以下代码创建名为“RenFile1”的PHP文件：
<？php $ a = 5; ？>
  1. 在同一个项目中，使用以下代码创建第二个名为RenFile2的PHP文件：
<？php require（RenFile1.php）; $ a = 8; ？>
  1. 保存这两个文件。
  2. 在Project Explorer视图中，右键单击RenFile1并选择Refactor |重命名 - 或 - 选择它并转到Refactor |从主菜单重命名 。
    将出现“重命名文件”对话框。
  3. 在“重命名文件”对话框中，将RenFile1重命名为RenFile3。
  4. 选中“更新参考”框，然后单击“ 预览” 。
  5. 在“预览”窗口中，滚动查看更改并注意，以及更改文件本身的名称，RenFile2中对文件的引用也将更改。
  6. 按确定以应用更改。

RenFile2中对RenFile1的引用将更新为：

要求（RenFile3.php）;

为了反映文件名的变化。

移动文件

移动文件将导致自动更新项目中引用该文件的所有实例，以反映其位置的更改。

此过程演示如何移动文件：

  1. 按照上面“重命名文件”下的步骤1到3中的说明创建文件'RenFile1'和'RenFile2'。
  2. 在同一项目中，创建一个名为“RenFolder”的附加文件夹。
  3. 在Project Explorer视图中，右键单击RenFile1并选择Refactor |移动 - 或 - 选择它并转到Refactor |从主菜单移动 。
  4. 在“移动”对话框中，选择“RenFolder”。
  5. 单击预览 。预览窗口将显示移动将应用于脚本的更改。请注意，RenFile1的新位置将在RenFile2的引用中自动更新。
  6. 单击“ 确定”以执行“移动”。

重命名文件中的元素

所有PHP元素（例如类，接口，函数，方法）也可以在编辑器窗口中重命名和重构。

此过程演示如何使用重命名元素功能：

  1. 按照上面“重命名文件”下的步骤1到3中的说明创建文件'RenFile1'和'RenFile2'。
  2. 在RenFile2的编辑器窗口中，突出显示第3行的变量“a”。
  3. 右键单击并选择Refactor |重命名 - 或 - 单击Alt + Shift + R.
  4. 在“重命名全局变量”对话框中，将变量重命名为b。
  5. 选中“更新文本事件”框，然后单击“ 预览” 。
  6. 在“预览”窗口中，滚动查看更改并注意变量“$ a”的出现将在RenFile1和RenFile2中更改。
  7. 按确定以应用更改。
