测试种类

此扩展点是内部的

标识符：
org.eclipse.jdt.junit.internal_testKinds

以来：
3.2

描述：
内部扩展点注册JUnit测试种类。此扩展点不适合客户端使用。它可以随时更改或删除。

配置标记：

<！ELEMENT 扩展 （ 种类 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 类 （ runtimeClasspathEntry +）>

<！ATTLIST那种

id CDATA #REQUIRED

displayName CDATA #REQUIRED

finderClass CDATA #REQUIRED

loaderPluginId CDATA #REQUIRED

loaderClass CDATA #REQUIRED

precedesTestKind CDATA #IMPLIED>

  • id - 唯一的测试种类ID
  • displayName - 此测试类型的名称
  • finderClass - 测试finder类实现org.eclipse.jdt.internal.junit.launcher。ITestFinder
  • loaderPluginId - 测试加载程序插件ID
  • loaderClass - 实现org.eclipse.jdt.internal.junit.runner的测试加载器类。ITestLoader（在插件“loaderPluginId”中声明）
  • precedesTestKind - 以逗号分隔的种类ID列表。此测试类型应在其所有其他类型之后显示。

<！ELEMENT runtimeClasspathEntry EMPTY>

<！ATTLIST runtimeClasspathEntry

pluginId CDATA #REQUIRED

pathToJar CDATA #IMPLIED>

  • pathToJar -
