重构持久类型和属性类型

警告：

  • 如果使用不同的方法重命名或移动持久性类型（例如，通过Project Explorer ），则持久性类型将从JPA图中消失。

  • 重构属性类型时，将删除属性访问方法中的任何现有业务逻辑。

概观

使用JPA图编辑器，您可以执行以下重构操作：

  • 重命名持久类型类：从持久类型名称的上下文菜单中，选择Refactor Persistent Type Class → Rename ，然后完成向导。

  • 将持久性类型移动到另一个源包：从持久性类型名称的上下文菜单中，选择“ 重构持久性类型类” →“ 移动” ，然后完成向导。

  • 重构属性类型：从属性的上下文菜单中，选择“ 重构属性类型” ，更改类型，然后选择“ 确定” 。

查看和编辑持久类型的源代码

要查看和编辑特定持久类型的源代码，请双击图上相应持久性类型块的标题。

Eclipse IDE在标准源代码编辑器中打开持久类型类。

将更改应用于源代码时，它们会自动与图表同步。

从图中删除所有持久类型

注意：

此操作仅从JPA图中删除持久性类型，也就是说，它们不会从底层应用程序数据模型中删除。

  1. 使用辅助鼠标按钮单击JPA图。

  2. 要将任何未保存的更改保存到应用程序数据模型，并从图中删除所有持久性类型 ，请选择从图表中删除所有持久性类型 → ...并保存更改

  3. 要从图中删除所有持久性类型而不保存对数据模型的任何未保存更改，请选择从图中删除所有持久性类型 → ...并放弃更改 。
