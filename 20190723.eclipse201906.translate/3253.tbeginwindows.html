连接到远程Windows服务器

以下文档说明如何安装Windows服务器代码，启动服务器守护程序以及建立与远程Windows服务器的连接。在此处查看在Linux，General UNIX或Mac上设置服务器以及其他配置选项。

安装服务器代码

  1. 创建要安装服务器代码的目录。这些说明的其余部分将假定目录名称和位置为C：\ rseserver，但您可以自由使用您选择的任何目录。
  2. 找到包含服务器的包。服务器代码通常与包含产品一起打包，您应该参考该产品的文档来查找和安装服务器包。但是，该服务器也可以在Eclipse Target Management下载站点上以包rseserver- <version> - <os> .zip的形式提供 。例如，rseserver-2.0-windows.zip包含适用于Windows的2.0版服务器。
  3. 将rseserver.zip复制到C：\ rseserver目录（这可能位于不同的计算机上）。
  4. 使用解压缩实用程序将服务器代码解压缩到C：\ rseserver目录。

启动服务器

您可以手动启动RSE通信服务器，也可以使用守护程序启动RSE通信服务器。

要使用服务器守护程序启动服务器：
  1. 只需双击daemon.bat程序即可启动服务器守护程序。
  2. 您可以编辑daemon.bat文件以更改守护程序的属性，例如要使用的特定守护程序端口或强制服务器的端口范围（以符合防火墙）。

请注意，服务器守护程序不会强制执行任何用户身份验证。如果运行服务器守护程序，则任何用户都可以连接到计算机，使用文件系统并运行命令。 建议不要在Windows系统上使用服务器守护程序 。

默认情况下，服务器守护程序在端口4075上运行。如果需要，可以传递可选的daemonPort参数以强制使用其他端口。
如果您的守护程序在防火墙后运行，您可能需要指定可选的serverPortRange参数，以将选定的服务器端口限制为给定的范围：
daemon.bat 4075 10000-10010
要手动启动服务器：
  1. 只需双击server.bat程序即可启动dstore服务器。服务器将选择第一个可用端口并打印端口号。默认情况下，通常为4033。然后，您必须在远程系统资源管理器中为连接的“文件”子系统输入端口属性中的此端口号。
  2. 出于安全原因，服务器只会等待一段有限的时间，直到客户端连接（默认为12000秒）。
  3. 要使用精确指定的端口或超时启动服务器，请打开Windows命令提示符并输入：
    C：
    cd \ rseserver
    server.bat [port] [timeout]
  4. 将RSE连接到服务器时，只要断开客户端，服务器就会终止。但是，守护程序不会终止。
要连接到远程Windows服务器：
  1. 切换到Remote System Explorer透视图。
  2. 在“远程系统”视图中， 将自动展开“ 新建连接”以显示可通过“远程系统资源管理器”连接的各种远程系统。展开Windows以调用对话框并配置连接。
  3. 输入第一个配置文件的名称，然后单击“ 下一步”（如果您之前从未定义过连接，则只会执行此步骤。）
  4. 输入连接名称。此名称显示在树视图中，并且必须对配置文件是唯一的。
  5. 在“ 主机名”字段中输入Windows服务器的名称或TCP / IP地址，例如jsandler。
  6. 输入描述 （可选）;创建连接后，描述将显示在“属性”视图中。
  7. 单击“ 完成”以定义系统。
注意：要检查端口号，请在“远程系统”视图中右键单击您的连接或子系统，然后选择“ 属性” 。单击子系统以查看相关信息。如果您的端口为“0”，则您的Remote System Explorer通信服务器将选择Windows服务器上的任何空闲端口。如果在启动服务器时指定了端口号，则需要在此处输入端口号，例如，使用防火墙。
