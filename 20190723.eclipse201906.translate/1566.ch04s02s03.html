4.2.3。�设计Swing中的可测试性

4.2.3.1。�命名组件

虽然AUT中的组件即使没有被开发人员命名也可以被识别，但是使用当前Swing组件类的setName方法肯定会使测试AUT变得更容易。即使AUT的整个区域已更改，仍将根据此唯一名称找到该组件。

4.2.3.2。�添加对文本检索的支持

您可以在不使用getText（）方法的情况下为Swing组件添加对呈现器的支持，以便访问在测试执行期间不可读的文本。

  • 可以在此处找到适配器机制的示例：

    http://git.eclipse.org/c/jubula/org.eclipse.jubula.core.git/
    树/ org.eclipse.jubula.examples.extension.swing.rc.renderer

  • 这并不能取代对开发人员可以直接更改的自定义Swing渲染器的支持。

  • 如果您能够自己更改渲染器，仍可以在渲染器中实现以下某项：

    public String getTestableText（）; public String getText（）;
