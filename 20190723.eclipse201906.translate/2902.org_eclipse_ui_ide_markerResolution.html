标记决议

标识符：
org.eclipse.ui.ide.markerResolution

以来：
3.0（最初在2.0版中添加为org.eclipse.ui.markerResolution）

描述：
此扩展点用于将标记分辨率生成器与标记的特定“种类”相关联。（某种类型的标记或具有某些属性值）。

配置标记：

<！ELEMENT 扩展 （ markerResolutionGenerator *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT markerResolutionGenerator （ attribute *）>

<！ATTLIST markerResolutionGenerator

class CDATA #REQUIRED

markerType IDREF #IMPLIED>

  • class - 实现IMarkerResolutionGenerator的类的名称。为了获得更好的性能，它还应该实现IMarkerResolutionGenerator2 。
  • markerType - 此标记分辨率生成器适用的标记类型。它不适用于标记子类型。

<！ELEMENT 属性 EMPTY>

<！ATTLIST属性

名称CDATA #REQUIRED

值CDATA #REQUIRED>

  • name - 要指定其值的属性的名称。
  • value - 属性的指定值。

例子：
以下是标记分辨率生成器扩展的示例（请注意子元素和使用的方式属性）：

<extension point = “org.eclipse.ui.ide.markerResolution” > <markerResolutionGenerator class = “org.eclipse.ui.examples.readmetool。ReadmeMarkerResolutionGenerator“ markerType = ”org.eclipse.ui.examples.readmetool.readmemarker“ > <attribute name = ”org.eclipse.ui.examples.readmetool.id“ value = ”1234“ /> </ markerResolutionGenerator> </ extension>
在上面的示例中，标记分辨率生成器与org.eclipse.ui.examples.readmetool.readmemarker类型的标记关联，其org.eclipse.ui.examples.redmetool.id属性的值为1234。

API信息：
只要目标属性按名称或值不同，就可以为给定标记提供多个标记分辨率生成器。
