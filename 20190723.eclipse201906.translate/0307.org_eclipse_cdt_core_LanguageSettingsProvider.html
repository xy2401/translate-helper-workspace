LanguageSettingsProvider

标识符：
org.eclipse.cdt.core。LanguageSettingsProvider

以来：
CDT 8.1

描述：
此扩展点用于提供新的语言设置提供程序。语言设置提供程序用于将编译器选项（例如包含路径（-I）或预处理器定义（-D））添加到项目模型中。

配置标记：

<！ELEMENT 扩展 （ 提供者 ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

此扩展点用于提供新的语言设置提供程序。语言设置提供程序用于将编译器选项（例如包含路径（-I）或预处理器定义（-D））添加到项目模型中。


  • id - 未使用的扩展点的ID
  • name - 未使用的扩展点的名称

<！ELEMENT 提供者 （ 语言范围 *， 条目 *）>

<！ATTLIST提供商

class CDATA #IMPLIED

id IDREF #REQUIRED

名称CDATA #REQUIRED

参数CDATA #IMPLIED

prefer-non-shared（true | false）“false”>

语言设置提供程序的定义。


  • class - 实现org.eclipse.cdt.core.language.settings.providers接口的Java类的完全限定名称。如果为空， 则为org.eclipse.cdt.core.language.settings.providers。默认情况下使用LanguageSettingsBaseProvider ，它提供此扩展点定义的基本功能。
    如果提供者需要非共享，按项目/配置设置或在UI中可编辑 - 提供者必须实现org.eclipse.cdt.core.language.settings.providers。ILanguageSettingsEditableProvider 。有一个通用的实现org.eclipse.cdt.core.language.settings.providers。LanguageSettingsGenericProvider可以在大多数情况下使用。
    如果需要以更深思熟虑的方式配置提供程序，则可以在扩展LanguageSettingsBaseProvider的类中使用attribute 参数 。
    此类的默认构造函数（不带参数的构造函数）必须是公共的，并且要导出包以便能够通过扩展点进行实例化。
  • id - 提供者的唯一ID。
  • name - 提供者的名称。该名称将在UI中呈现给用户。
  • parameter - 初始化提供程序的自定义参数。例如，用于为GCCBuiltinSpecsDetector发送命令。
  • prefer-non-shared - 一个标志，指示提供者应该由配置拥有，而不是在工作空间中全局并在项目之间共享（当尚未通过其他方式指示选择时）。例如，当用户从扩展提供者列表中添加提供者时，参考该偏好以便初始设置“共享”属性。此属性的值“true”仅对能够非共享的提供者有意义，即提供者扩展ILanguageSettingsEditableProvider。

<！ELEMENT 语言范围 EMPTY>

<！ATTLIST语言范围

id CDATA #REQUIRED>

语言范围的定义。包括此提供程序适用的语言ID列表。如果语言范围不存在，提供者将提供任何语言。


  • id - 此提供程序为其提供条目的语言的ID。例如，这些是由CDT提供的语言（参见扩展org.eclipse.cdt.core.language）：

    - C的“ org.eclipse.cdt.core.gcc ”，

    - 用于C ++的“ org.eclipse.cdt.core.g ++ ”。


<！ELEMENT 条目 （ 标志 *）>

<！ATTLIST入门

kind（includePath | macro | includeFile | libraryPath | libraryFile | macroFile）

名称CDATA #REQUIRED

值CDATA #IMPLIED>

语言设置条目用于向编译器选项（例如包含路径（-I）或预处理器定义（-D）等）添加到项目模型中。


  • kind - 一种映射到编译器选项的语言设置条目。例如，以下映射用于gcc选项：
    “ -I ”：includePath
    “ -D ”：宏
    “ -include ”：includeFile
    “ -L ”：libraryPath
    “ -l ”：libraryFile
    “ -imacros ”：macroFile
  • name - “name”属性映射到表示文件夹或文件路径的条目的路径，以及表示名称 - 值对的宏类型的名称。例如：
    “ / usr / include / ”
    “ MACRO ”（对于#define MACRO值 ）
    请注意，相对路径在构建工作目录中被视为root（如果适用）。
  • value - “value”属性仅用于表示名称 - 值对的宏类型。它不用于表示路径的条目。例如：
    “ value ”（对于#define MACRO值 ）

<！ELEMENT 标志 EMPTY>

<！ATTLIST标志

值（BUILTIN | LOCAL | RESOLVED | VALUE_WORKSPACE_PATH | UNDEFINED | EXPORTED）>

条目的组合标志。


  • value - 标志的值。对应于ICSettingEntry标志，有关详细信息，请参阅JavaDoc。以下是Javadoc的摘录，用于与此扩展点一起使用的标志（其他可能不受支持）：
    - BUILTIN ：表示在工具（编译器）本身内置的设置。这种设置不作为选项传递给编译器，但索引器或其他客户端可能需要它们。
    - LOCAL ：仅适用于includePath ，可以是本地（#include“...”）或系统（#include <...>）。如果includePath未标记为LOCAL，则将其视为系统。
    - 已解决 ：表示该项目并不需要为环境变量的扩张来解决这样的，工作目录正常化的路径对构建等。
    - VALUE_WORKSPACE_PATH ：用于指示该条目是由工作空间中的eclipse管理的资源。该路径以工作空间根目录为根。
    - UNDEFINED ：表示不应定义该条目，对应于gcc编译器的-U选项。如果定义了此标志，它将为列表中的所有提供者否定具有相同名称（和种类）的条目。
    - EXPORTED ：表示该条目已导出到引用项目。它将传递给引用条目所属配置的项目配置。

API信息：
想要扩展此扩展点的插件必须实现org.eclipse.cdt.core.language.settings.providers。ILanguageSettingsProvider接口。
对于未动态更改提供的设置条目（表示编译器选项）的情况，配置默认提供的现有类LanguageSettingsBaseProvider就足够了。
