2.5。�一般设置信息

以下部分描述了在何处查找使用API的相关JAR /工件。提供的API具有Java 6所需的miminum运行时环境。

通常，客户端API的使用分为三类JAR / maven工件/ OSGi包：

  1. 每个项目都需要依赖于API的所有 工具包中性部分 ：

    • org.eclipse.jubula.client.api

    • org.eclipse.jubula.client.api.commands

    • org.eclipse.jubula.toolkit.api

    • org.eclipse.jubula.tools

    • org.eclipse.jubula.communication

  2. 每个项目都需要依赖于依赖从API的使用工具箱工具包的特定元素的indiviual部分组成：

    • org.eclipse.jubula.toolkit.base.api

    • org.eclipse.jubula.toolkit.concrete.api

    • org.eclipse.jubula.toolkit.javafx.api

    • org.eclipse.jubula.toolkit.swt.api

    • org.eclipse.jubula.toolkit ...TOOLKIT_ID ... API

    请记住， 工具包是相互派生的 。因此，如果您正在设置JavaFX项目，还必须将直接依赖项包含在特定工具包工件“org.eclipse.jubula.toolkit.javafx.api”本身以及基于它的工具包中：“org .eclipse.jubula.toolkit.concrete.api“和它的基础”org.eclipse.jubula.toolkit.base.api“。

  3. 每个项目还需要依赖于API的所有 第三方依赖项：

    • Apache Commons Lang 2.6

    • Apache Commons Codec 1.6

    • SLF4j API

    • XStream 1.3.1

    • xmlpull 1.1.3.4a

您可以在<ITEInstallationFolder> / development / api文件夹中找到不同形状的工件（纯JAR和maven工件）。要定义API工件的OSGi依赖关系，您只需在目标平台（定义）中包含<ITEInstallationFolder> /development/org.eclipse.jubula.repo.zip ，并定义对“Jubula Functional Testing - API Core”的依赖关系 -特征。

API中随时间发生的更改列表位于此处 。
