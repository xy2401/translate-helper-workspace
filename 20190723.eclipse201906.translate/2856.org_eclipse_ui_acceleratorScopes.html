以来：

描述：

警告：此扩展点已弃用。
请勿使用此扩展点，将在本产品的未来版本中将其删除。相反，请使用扩展点org.eclipse.ui.commands

此扩展点用于注册加速器范围扩展。加速器范围是加速器组可能适用的范围。例如，如果加速器组适用于名为“文本编辑器范围”的范围，则该加速器组的加速器仅在“文本编辑器范围”或其子项之一处于活动状态时运行（换句话说，如果处于活动状态） part是参与文本编辑器）。

加速器集通过将作用域的id列为其“scopeId”属性的值来声明其适用的范围（请参阅加速器集扩展点）。许多加速器组可适用于相同的加速器范围。

配置标记：

<！ELEMENT 扩展 （ acceleratorScope *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

不推荐使用acceleratorScope元素

<！ELEMENT acceleratorScope EMPTY>

<！ATTLIST acceleratorScope

id CDATA #REQUIRED

名称CDATA #REQUIRED

说明CDATA #REQUIRED

parentScope CDATA #IMPLIED>

  • id - 可用于标识此加速器范围的唯一名称。
  • name - 加速器范围的可翻译名称。
  • description - 加速器范围的简短描述。
  • parentScope - 一个可选属性，表示此范围处于活动状态时处于活动状态的作用域。对于大多数范围，org.eclipse.ui.globalScope将是父范围

例子：
以下是加速器范围扩展的示例：

<extension point = “org.eclipse.ui.acceleratorScopes” > <acceleratorScope id = “org.eclipse.ui.globalScope” name = “Global” description = “除非明确覆盖，否则适用于所有视图和编辑器的动作加速键。” > </ acceleratorScope> <acceleratorScope id = “org.eclipse.ui.javaEditorScope” name = “Java Editor” description = “动作加速键仅在java编辑器处于活动状态时适用。” parentScope = “org.eclipse.ui.globalScope” > </ acceleratorScope> </ extension>

API信息：
方法public IKeyBindingService getKeyBindingService（）已添加到IEditorSite。

提供的实施：
工作台提供全局加速器范围和文本编辑器加速器范围。
