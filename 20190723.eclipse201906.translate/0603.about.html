2010年4月14日

内容的使用

ECLIPSE基金会为开放源项目提供可用的软件，文档，信息和/或其他材料（统称为“内容”）。内容的使用受本协议的条款和条件和/或许可协议的条款和条件或下面显示或引用的通知的约束。使用本协议，即表示您同意您对本协议的使用受本协议和/或任何适用的许可协议的条款和条件或下面显示或引用的通知的管辖。如果您不同意本协议的条款和条件以及任何适用的许可协议的条款和条件或下面显示或引用的通知，那么您可能不会使用该内容。

适用的许可证

除非另有说明，否则Eclipse Foundation提供的所有内容均根据Eclipse Public License Version 1.0（“EPL”）的条款和条件提供给您。本内容提供了EPL的副本，也可从http://www.eclipse.org/legal/epl-v10.html获取 。就EPL而言，“程序”将指内容。

内容包括但不限于源代码，目标代码，文档和在软件模块（“模块”）中的Eclipse Foundation源代码存储库（“存储库”）中维护的其他文件，并且可用作可下载的存档（“下载”） ）。

  • 可以将内容结构化并打包成模块，以便于传递，扩展和升级内容。典型的模块可以包括插件（“插件”），插件片段（“片段”）和特征（“特征”）。
  • 每个插件或片段可以打包为名为“plugins”的目录中的子目录或JAR（Java™ARchive）。
  • 功能是一个或多个插件和/或片段以及相关材料的捆绑。每个功能都可以打包为名为“功能”的目录中的子目录。在功能中，名为“feature.xml”的文件可能包含与该功能关联的插件和/或片段的名称和版本号列表。
  • 功能还可能包括其他功能（“包含的功能”）。在功能中，名为“feature.xml”的文件可能包含“包含的功能”的名称和版本号列表。

管理插件和片段的条款和条件应包含在名为“about.html”（“Abouts”）的文件中。管理功能和包含功能的条款和条件应包含在名为“license.html”（“功能许可证”）的文件中。关于和功能许可可能位于下载或模块的任何目录中，包括但不限于以下位置：

  • 顶级（根）目录
  • 插件和片段目录
  • 内部插件和碎片打包为JAR
  • 某些插件名为“src”的目录的子目录
  • 功能目录

注意：如果使用Provisioning Technology（如下定义）安装Eclipse Foundation提供的功能，则必须在安装过程中同意许可证（“功能更新许可证”）。如果功能包含包含的功能，功能更新许可证应向您提供管理包含功能的条款和条件，或通知您可以在何处找到它们。功能更新许可证可以在功能中找到的名为“feature.properties”的文件的“许可证”属性中找到。此类“关于”，“功能许可”和“功能更新许可”包含管理您对该目录中关联内容的使用的条款和条件（或对此类条款和条件的引用）。

关于EPL或其他许可协议，通知或条款和条件的相关信息，功能许可和功能更新许可可能会引用。其中一些许可协议可能包括（但不限于此）：

  • 通用公共许可证版本1.0（可从http://www.eclipse.org/legal/cpl-v10.html获得 ）
  • Apache软件许可证1.1（可从http://www.apache.org/licenses/LICENSE获取 ）
  • Apache Software License 2.0（可从http://www.apache.org/licenses/LICENSE-2.0获取 ）
  • Metro Link公共许可证1.00（可从http://www.opengroup.org/openmotif/supporters/metrolink/license.html获取 ）
  • Mozilla公共许可证版本1.1（可从http://www.mozilla.org/MPL/MPL-1.1.html获取 ）

在使用内容之前，您有义务阅读并接受所有这些条款和条件。如果未提供“关于”，“功能许可”或“功能更新许可”，请与Eclipse Foundation联系以确定哪些条款和条件适用于该特定内容。

使用供应技术

Eclipse Foundation提供可用的配置软件，其示例包括但不限于p2和Eclipse Update Manager（“供应技术”），目的是允许用户安装软件，文档，信息和/或其他材料（统称“可安装软件”）。提供此功能的目的是允许此类用户安装，扩展和更新基于Eclipse的产品。有关打包可安装软件的信息，请访问http://eclipse.org/equinox/p2/repository_packaging.html （“规范”）。

您可以使用Provisioning Technology允许其他方安装Installable Software。您应负责根据规范向供应技术的用户提供和接受与可安装软件相关的适用许可协议。通过以这种方式使用供应技术并按照规范提供，您进一步确认您同意并获得允许以下内容的所有必要权利：

  1. 可以发生一系列动作（“供应过程”），其中用户可以在机器（“目标机器”）上执行供应技术，目的是安装，扩展或更新基于Eclipse的产品的功能。
  2. 在供应过程中，供应技术可能会导致第三方可安装软件或其中一部分被访问并复制到目标机器。
  3. 根据规范，您将向用户提供管理可安装软件使用的条款和条件（“可安装软件协议”），并且应根据规范从目标机器访问此类可安装软件协议。此类可安装软件协议必须告知用户有关可安装软件的条款和条件，并且必须以此类可安装软件协议中规定的方式征求最终用户的同意。在用户表示同意后，供应技术将完成可安装软件的安装。
