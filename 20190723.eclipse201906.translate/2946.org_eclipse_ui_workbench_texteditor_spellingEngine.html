拼写引擎

标识符：
org.eclipse.ui.workbench.texteditor.spellingEngine

以来：
3.1

描述：
允许贡献者添加拼写引擎。

配置标记：

<！ELEMENT 扩展 （ 引擎 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 引擎 EMPTY>

<！ATTLIST引擎

class CDATA #REQUIRED

标签CDATA #IMPLIED

id CDATA #REQUIRED

默认（true | false）

preferencesClass CDATA #IMPLIED>

拼写引擎的定义。


  • class - 引擎的类，必须实现org.eclipse.ui.texteditor.spelling.ISpellingEngine
  • label - 引擎的显示标签，将显示在允许用户选择拼写引擎的菜单中
  • id - 唯一标识此引擎的字符串
  • default - 如果此标志设置为true ，则默认情况下将安装此拼写引擎。如果安装了多个引擎并且遇到了标志集，则会采用第一个引擎。
  • preferencesClass - org.eclipse.ui.texteditor.spelling.ISpellingPreferenceBlock

例子：
以下是拼写引擎定义的示例。 


  <extension point="org.eclipse.ui.workbench.texteditor.spellingEngine">
    <engine
        default="true"
        label="%defaultSpellingEngine.label"
        class="org.eclipse.jdt.internal.ui.text.spelling.DefaultSpellingEngine"
        id="org.eclipse.jdt.internal.ui.text.spelling.DefaultSpellingEngine">
    </engine>
  </extension>

提供的实施：
org.eclipse.jdt.ui插件提供DefaultSpellingEngine 。以其实现为例。
