特定于上下文的帮助内容

ditamap需要特定的标记来实现上下文相关的帮助。

包装<resourceid>元素的每个<topicmeta>元素可以选择包含一个<searchtitle>元素或一个<shortdesc>元素，或两者都包含（每个元素之一）：
  • <searchtitle>元素用于提供Eclipse上下文XML文件中<context>元素的title属性值。
  • <shortdesc>元素用于提供Eclipse上下文XML文件中<description>元素的内容。例如：
    <topicref navtitle =“contexts.xml主题元素中的标签属性”href =“path / to / topic.xml”> <topicmeta> <searchtitle>可选文本以覆盖帮助关于标题。</ searchtitle> <shortdesc>上下文描述。</ shortdesc> <resourceid id =“help_context_ID_string”/> </ topicmeta> </ topicref>
注意： <searchtitle> ， <shortdesc>和<resourceid>元素必须按照上面显示的顺序出现在<topicmeta>元素中。
包含上面示例的DITA映射在Eclipse上下文XML文件中生成以下内容：
<上下文> ......<context id =“help_context_ID_string”title =“覆盖帮助关于标题的可选文本。”> <description>上下文描述的文本。</ description> <topic label =“contexts.xml主题元素中的标签属性”href =“ PLUGINS_ROOT / doc_plugin_ID /path/to/topic.xml“/> </ context> ...</上下文>
其中doc_plugin_ID是<map>元素上id属性的值。
