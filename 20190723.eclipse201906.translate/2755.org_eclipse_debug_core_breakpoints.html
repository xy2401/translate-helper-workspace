断点

标识符：
org.eclipse.debug.core.breakpoints

描述：
此扩展点定义了一种用于定义新类型断点的机制。

配置标记：

<！ELEMENT 扩展 （ 断点 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 断点 EMPTY>

<！ATTLIST断点

id CDATA #REQUIRED

markerType IDREF #REQUIRED

class CDATA #REQUIRED

名称CDATA #IMPLIED>

  • id - 指定此断点类型的唯一标识符。
  • markerType - 指定此类型断点的相应标记定义的完全限定标识符（id）。
  • class - 指定实现IBreakpoint的Java类的完全限定名称。
  • name - 为此断点类型指定用户可呈现的名称。例如，“Java Line Breakpoint”。此属性已在3.1中添加，以支持断点类型自动分组断点。如果未指定此属性，则无法按类型自动对此类型的断点进行分组。

例子：
以下是启动配置类型扩展点的示例：


 <extension point="org.eclipse.debug.core.breakpoints">
  <breakpoint
   id="com.example.ExampleBreakpoint"
   markerType="com.example.ExampleBreakpointMarker"
   class="com.example.ExampleBreakpointImpl">
  </breakpoint>
 </extension>
在上面的示例中，指定类型的断点由类“com.example”实现。BreakpointImpl”。“com.example”有关联的标记定义。ExampleBreakpointMarker“，定义此断点的属性。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。IBreakpoint 。
