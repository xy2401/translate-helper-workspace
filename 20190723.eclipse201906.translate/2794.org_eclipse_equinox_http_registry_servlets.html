Http Service Servlets

标识符：
org.eclipse.equinox.http.registry.servlets

以来：
1.0

描述：
此扩展模仿并借用了OSGi HttpService registerServlet方法的语义。

配置标记：

<！ELEMENT 扩展 （ serviceSelector ？， servlet +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT servlet （ init-param *）>

<！ATTLIST servlet

class CDATA #REQUIRED

别名CDATA #REQUIRED

httpcontextId CDATA #IMPLIED

load-on-startup（true | false）>

  • class - 与registerServlet中的OSGi Http Services“servlet”等效。Servlet由Extension Point提供程序实例化，并且必须是javax.servlet的实例。Servlet的
  • 别名 - 与registerServlet中的OSGi Http Services“别名”等效。
  • httpcontextId - 与registerResource中的OSGi Http Services httpcontext等效。httpcontextId与匹配id的“httpcontexts”扩展名匹配。如果未提供此属性，则使用OSGi Http Service的DefaultHttpContext语义。
  • load-on-startup - 确定是否应立即初始化Servlet。

<！ELEMENT init-param EMPTY>

<！ATTLIST init-param

名称CDATA #REQUIRED

值CDATA #REQUIRED>

类似于Servlet规范及其部署描述符的init-param元素。


  • name - 初始参数的名称
  • value - 初始参数的值

<！ELEMENT serviceSelector EMPTY>

<！ATTLIST serviceSelector

过滤CDATA #IMPLIED

class CDATA #IMPLIED>

  • filter - serviceSelector允许过滤用于注册servlet的HttpService实例。
  • class - 如果使用，则该类必须是org.osgi.framework的实例。过滤

例子：
以下是servlets扩展的示例：
<extension id = “testServlet” point = “org.eclipse.equinox.http.registry.servlets” > <servlet alias = “/ sp_test” class = “org.eclipse.equinox.http.registry.internal。TestServlet“ > <init-param name = ”testParam“ value = ”test param value“ > </ init-param> </ servlet> </ extension>
这相当于OSGi Http Service的registerServlet方法。
