流程工厂

标识符：
org.eclipse.debug.core.processFactories

以来：
3.0

描述：
此扩展点提供了一种机制，用于指定要与启动配置一起使用的流程工厂，以创建适当的IProcess实例。启动配置将需要DebugPlugin。ATTR_PROCESS_FACTORY_ID属性设置为将用于创建IProcess的相应进程工厂ID

配置标记：

<！ELEMENT 扩展 （ processFactory *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT processFactory EMPTY>

<！ATTLIST processFactory

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 指定此流程工厂的唯一标识符。
  • class - 指定实现IProcessFactory的Java类的完全限定名称。

例子：
以下是流程工厂扩展点的示例：


 <extension point="org.eclipse.debug.core.processFactories">
  <processFactory
       id="com.example.ExampleIdentifier"
       class="com.example.ExampleProcessFactory">
  </processFactory>
 </extension>

API信息：
属性类的值必须是实现接口org.eclipse.debug.core的Java类的完全限定名称。IProcessFactory 。
