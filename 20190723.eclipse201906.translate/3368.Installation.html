安装

本节介绍LTTng跟踪器和Trace Compass插件的安装及其依赖性。

LTTng Tracer

虽然Eclipse插件可以在标准的Eclipse平台（Linux，Mac，Windows）上运行，但LTTng跟踪器及其附带的工具可以在Linux上运行。

自12.04以来，可以在Ubuntu中下载跟踪器和工具。可以使用以下命令轻松安装它们：

> sudo apt-get install lttng-tools

对于其他发行版，较旧的Ubuntu发行版或最新的LTTng示踪器，请参阅LTTng网站以获取安装信息。

注意 ：只有在您想要创建自己的跟踪（通常情况）时才需要LTTng跟踪器（和随附的工具）。如果您打算简单地分析现有跟踪，则无需安装跟踪器。

跟踪指南针插件

安装Eclipse的Trace Compass插件的最简单方法是通过Install New Software菜单。有关如何使用此菜单的信息，请参阅此链接 。

Trace Compass主插件的结构为一堆功能/插件，如下所示：

  • CTF - CTF解析器，也可以用作独立组件
    • 功能 ：org.eclipse.tracecompass.ctf
    • 插件 ：org.eclipse.tracecompass.ctf.core，org.eclipse.tracecompass.ctf.parser
  • 国家体系核心 - TMF的国家体系
    • 插件 ：org.eclipse.tracecompass.statesystem.core
  • TMF - 跟踪和监视框架用于通用跟踪处理的框架
    • 功能 ：org.eclipse.tracecompass.tmf
    • 插件 ：org.eclipse.tracecompass.tmf.core，org.eclipse.tracecompass.tmf.ui。 org.eclipse.tracecompass.tmf.analysis.xml.core，org.eclipse.tracecompass.tmf.analysis.xml.ui
  • CTF支持TMF - CTF支持TMF功能
    • 功能 ：org.eclipse.tracecompass.tmf.ctf
    • 插件 ：org.eclipse.tracecompass.tmf.ctf.core
  • LTTng控件 - LTTng跟踪器控件的包装器。可用于内核或应用程序跟踪。
    • 功能 ：org.eclipse.tracecompass.lttng2.control
    • 插件 ：org.eclipse.tracecompass.lttng2.control.core，org.eclipse.tracecompass.lttng2.control.ui
  • LTTng内核 - 特定于Linux内核跟踪的分析组件
    • 功能 ：org.eclipse.tracecompass.lttng2.kernel
    • 插件 ：org.eclipse.tracecompass.analysis.os.linux.core，org.eclipse.tracecompass.analysis.os.linux.ui，org.eclipse.tracecompass.lttng2.kernel.core，org.eclipse.tracecompass。 lttng2.kernel.ui
  • LTTng UST - 特定于Linux用户空间跟踪的分析组件
    • 功能 ：org.eclipse.tracecompass.lttng2.ust
    • 插件 ：org.eclipse.tracecompass.lttng2.ust.core，org.eclipse.tracecompass.lttng2.ust.ui

LTTng控制依赖关系

Eclipse LTTng Control功能通过ssh连接控制LTTng跟踪器，如果跟踪器在本地运行，它可以使用或绕过ssh连接。

使用ssh时 ，目标系统（运行跟踪器的位置）需要运行ssh服务器以及您有权连接的sftp服务器（用于文件传输）。

在主机端（运行Eclipse的位置），您还需要安装Eclipse Remote Services来处理SSH连接和传输。远程服务是作为LTTng控件功能的依赖项为您安装的。如有必要，可以使用标准方式手动安装（ 帮助 > 安装新软件... > 通用工具 > 远程服务 ）。

安装验证

如果您没有任何痕迹，可以在http://lttng.org/files/samples找到样本LTTng跟踪。此页面包含一些LTTng 2.0内核跟踪示例的链接。跟踪需要解压缩才能打开。跟踪也可以作为存档直接导入，有关详细信息，请参阅导入部分。

以下是使用LTTng跟踪验证安装是否正常运行的快速步骤：

  • 启动Eclipse
  • 打开LTTng Kernel透视图
  • 创建跟踪项目
    • 在Project Explorer视图中单击鼠标右键，然后选择New，Tracing Project
    • 输入项目名称（例如“MyLTTngProject”）
    • 该项目将被创建。它将包含2个空文件夹：“Traces”和“Experiments”
  • 打开并可视化样本跟踪
    • 右键单击新创建的项目“Traces”文件夹，然后选择“Open Trace ...”
    • 导航到要显示的示例LTTng跟踪，并选择跟踪文件夹中的任何文件
    • 新导入的跟踪应显示在Traces文件夹下
    • 应加载跟踪并填充视图

如果显示错误消息，您可能需要仔细检查是否正确设置了跟踪类型（右键单击跟踪和“选择跟踪类型...”）。

有关视图及其用法的详细说明，请参阅跟踪透视图。
