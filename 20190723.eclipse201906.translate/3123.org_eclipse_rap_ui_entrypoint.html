入口点

标识符：
org.eclipse.rap.ui.entrypoint

以来：
RAP 1.0

描述：
对于每个RAP应用程序，必须在框架中注册入口点。该框架将为每个用户会话创建此入口点的单独实例。入口点必须实现EntryPoint接口。

应用程序可以注册多个入口点，每个入口点位于不同的URL路径。要访问入口点，必须将其注册的路径附加到应用程序的上下文路径。

作为注册入口点的替代方法，Eclipse应用程序也可以使用接口IApplication 。

配置标记：

<！ELEMENT 扩展 （ 入口点 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 入口点 EMPTY>

<！ATTLIST入口点

id CDATA #REQUIRED

路径CDATA #REQUIRED

class CDATA #IMPLIED

applicationId IDREF #IMPLIED

brandingId IDREF #IMPLIED>

  • id - 此入口点的标识符。
  • path - 注册入口点的URL路径。必须以斜线开头。
  • class - 启动应用程序的EntryPoint的实现。必须指定此属性或applicationId。
  • applicationId - 要启动的IApplication扩展的id而不是EntryPoint类。
  • brandingId - 指定在启动此入口点时应使用的（可选）品牌。

例子：
<extension id = “org.eclipse.rap.demo.demoentrypoint” point = “org.eclipse.rap.ui.entrypoint” > <entrypoint id = “example.entrypoint” class = “example。MyEntrypoint“ path = ”/ myapp“ /> </ extension>

API信息：
每个入口点必须实现EntryPoint接口。在createUI（）方法中，创建应用程序的用户界面。简单RAP应用程序的入口点可能如下所示：
public class MyEntryPoint实现EntryPoint {public int createUI（）{Display display = new Display（）; Shell shell = new Shell（显示）; shell.setLayout（new GridLayout（））; Label label = new Label（shell，SWT.NONE）; label.setText（“Hello World！“）; shell.layout（）; shell.open（）;返回0;}}
