标识符：
org.eclipse.team.core.bundleImporters

以来：
3.6

描述：
实验 。此扩展点已添加为正在进行的工作的一部分。无法保证此API可以正常工作，或者在将来的版本中保持不变。未经团队团队咨询，请不要使用此API。

捆绑导入器能够基于捆绑包清单在工作空间中创建项目。例如，CVS包导入器利用“Eclipse-SourceReferences”标头从CVS导入项目。

配置标记：

<！ELEMENT 扩展 （ 进口商 ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 导入器 （ 支持 ）*>

<！ATTLIST进口商

id CDATA #REQUIRED

描述CDATA #IMPLIED

名称CDATA #REQUIRED

存储库IDREF #REQUIRED>

  • id - 此项目工厂扩展的唯一标识符。
  • description - 适用于向最终用户显示的此导入程序的简短描述。
  • name - 此捆绑导入程序的名称，适合显示给最终用户。
  • repository - 与此导入程序关联的org.eclipse.team.core.repository扩展的标识。

<！ELEMENT 支持 EMPTY>

<！ATTLIST支持

前缀CDATA #REQUIRED>

  • prefix - 此导入器支持的SCM URL前缀。

例子：
以下是捆绑导入程序扩展的示例。 

<extension point = “org.eclipse.team.core.bundleImporters” > <importer id = “com.example。ExampleIdentifier“ name = ”示例导入程序“ repository = ”com.example。ExampleNature“ > <supports prefix = ”scm：example：“ > </ supports> </ importer> </ extension>

提供的实施：
CVS提供了一个bundle importer扩展，它能够从'Eclipse-SourceReferences'标题引用的CVS中导入项目。
