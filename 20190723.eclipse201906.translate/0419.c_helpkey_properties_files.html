helpKey属性文件

helpKey属性文件定义了helpKey常量（属性键）的映射，可以是实际的帮助上下文ID，也可以是上下文特定的帮助搜索表达式。

helpKey属性文件通常成对提供，一个用于上下文ID，另一个用于搜索表达式。

helpKey属性文件中定义的帮助上下文ID字符串不能是完全限定的上下文ID。它们不得包含任何空格或句点（。）字符。在通过ContextProviderDelegate将它们返回到UI控件之前，DTP help-helper处理属性文件中定义的非限定帮助上下文ID以完全限定它们。

以下摘录来自helpKey属性文件，该文件由org.eclipse.datatools.doc.user.contexts插件提供，用于定义具体的帮助上下文ID字符串。 

    ...
    #	CONTEXT_ID_CP_PROPERTY_PAGE =
    #	Basic profile name/description/auto-connect property page
    CONTEXT_ID_CP_PROPERTY_PAGE = CONNECTION_PROFILE_COMMON_PROPERTY_PAGE
    
    #	CONTEXT_ID_CP_WIZARD_PAGE =
    #	wizard selection page in New Connection Profile wizard
    CONTEXT_ID_CP_WIZARD_PAGE = NEW_CONNECTION_PROFILE_WIZARD
    
    #	CONTEXT_ID_INTRO_WIZARD_PAGE =
    #	Basic intro page for new connection profile wizard
    CONTEXT_ID_INTRO_WIZARD_PAGE = NEW_CONNECTION_PROFILE_WIZARD
    ...

请注意，两个抽象帮助上下文（由helpKey常量CONTEXT_ID_CP_WIZARD_PAGE和CONTEXT_ID_INTRO_WIZARD_PAGE ）映射到一个具体的帮助上下文ID（ NEW_CONNECTION_PROFILE_WIZARD ）。这是抽象帮助上下文提供的灵活性的一个示例，允许UA组件在单个具体的帮助上下文ID中组合多个抽象帮助上下文，这独立于UI组件中的帮助上下文实现。

以下示例显示了helpKey常量到helpKey属性文件中特定于上下文的帮助搜索表达式的映射。 

    ...
    #	 * CONTEXT_ID_CP_PROPERTY_PAGE =
    #	 * Basic profile name/description/auto-connect property page
    CONTEXT_ID_CP_PROPERTY_PAGE = "connection profile" AND "properties"
    ...

搜索表达式字符串必须符合Apache Lucene查询解析器语法的规则： http ： //lucene.apache.org/java/docs/queryparsersyntax.html

注意：搜索表达式中的术语（关键字）不区分大小写，但布尔运算符（例如，AND，NOT，OR）必须全部为大写。
  • helpKey属性文件名
    helpKey属性文件的名称是任意的（只要它们以.properties结尾）。但是，上下文相关的帮助插件可以为多个UI插件提供成对的helpKey属性文件，如果这些文件位于同一目录中，则需要具有唯一的名称。
