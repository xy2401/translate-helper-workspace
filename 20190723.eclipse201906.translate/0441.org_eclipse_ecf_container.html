容器

标识符：
org.eclipse.ecf.container

以来：
ECF核心捆绑v 3.1.0。

描述：
此扩展点允许扩展指定创建给定类型的容器实例。

配置标记：

<！ELEMENT 扩展 （ 容器 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT 容器 （ 参数 +）？>

<！ATTLIST容器

factoryName CDATA #REQUIRED

containerId CDATA #IMPLIED

>

容器扩展点。扩展可以定义在ECF核心初始化时创建的新容器。


  • factoryName - 定义要创建的容器类型所需的containerFactoryName（例如ecf.generic.client或ecf.r_osgi.peer）。
  • containerId - 可选的containerId，用于定义新容器实例的id。

<！ELEMENT 参数 EMPTY>

<！ATTLIST参数

名称CDATA #REQUIRED

值CDATA #REQUIRED

>

容器实例创建的参数。由参数元素名称和值属性定义的名称值对将添加到Map，然后传递给容器工厂实例创建。


  • name - 容器参数的名称。
  • value - 容器参数的值。

例子：
<extension point = “org.eclipse.ecf.container” > <container containerId = “foo1” factoryName = “ecf.generic.client” > <parameter name = “name1” value = “value1” > </ parameter> <parameter name = “name2” value = “value2” > </ parameter> </ container> <container containerId = “adfasdf” factoryName = “bogus.factory.name” > </ container> <container containerId = “ecftcp：// localhost ：32111 / server“ factoryName = ”ecf.generic.server“ > </ container> <container factoryName = ”ecf.generic.client“ > </ container> </ extension>

API信息：
没有可用的API。

提供的实施：
ECF核心v3.1.0软件包提供此扩展点的实现。
