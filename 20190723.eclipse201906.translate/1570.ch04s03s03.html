4.3.3。�设置RCP AUT进行测试

如果要测试Rich Client Platform应用程序，则必须确保将“ RCP远程控制 ”插件（org.eclipse.jubula.rc.rcp）添加到AUT中，并在AUT启动时启动它。如果您正在进行持续构建和测试过程，那么我们强烈建议您将此作为产品构建的一部分，或者之后（第4.3.3.1节，“设置RCP AUT以作为构建的一部分进行测试）过程“ ）。出于测试目的，并且为了更快地开始，这些步骤也可以手动完成，如下所示：

请确保您遵循所有这些步骤！

  1. 找到安装目录，然后打开开发文件夹。

  2. 将rcp-support.zip文件夹的内容解压缩到RCP AUT的plugins目录中。

  3. RCP应用程序通常有一个配置/ config.ini文件，其中包含参数osgi.bundles 。需要修改此参数以允许RCP远程控制插件在AUT启动时加载。您必须添加org.eclipse.jubula.rc.rcp @ start （逗号对于将此参数与其他参数分隔开来很重要）到osgi.bundles参数的末尾。如果您的AUT使用简单的配置程序，那么您将在config.ini中看到osgi.bundles参数。在这种情况下，osgi.bundles参数将引用simple configurator的bundles.info所在的位置。找到bundles.info文件并确保将org.eclipse.jubula.rc.rcp插件设置为start（即设置为true）。如果要从一个版本的RCP远程控制迁移到新版本，则还可以检查RCP远程控制插件的版本号。

  4. 正常启动AUT（即不是AUT）。关闭它，然后再次启动它。

  5. 在AUT中，打开“ 帮助” - >“ 安装详细信息” 。

  6. 在“ 配置”选项卡中，您将在“ 插件注册表 ”部分中找到已启动插件的列表。您应该能够看到以下插件已启动（活动），并且它们的正确版本（根据您的Jubula安装）正在运行：

    • org.eclipse.jubula.rc.rcp

    • org.eclipse.jubula.rc.rcp.common

  7. 如果您可以看到这些插件，那么您可以通过例如AUT配置（ 章节�3.8.4.2，“配置从 AUTE 启动Java AUT” ）或通过autrun继续并启动AUT（ Section�3.8.4.6， “使用autrun命令启动Java AUT” 。

  8. 如果您看不到这些插件，那么您应该与开发团队的成员交流，以实现以下部分中描述的建议（ 章节�3.3.3.1，“设置RCP AUT以作为构建过程的一部分进行测试”） ）。

安装新版本的ITE时，必须使用新的RCP远程控制插件重复这些步骤。您必须删除以前的RCP远程控制并将其替换为新ITE安装中的新远程控制，并更新config.ini或bundles.info文件。我们建议您使用-clean启动AUT一次，以确保使用新的远程控制插件。

如果您不遵循这些步骤，AUT代理将无法与您的AUT通信！

4.3.3.1。�设置RCP AUT以作为构建过程的一部分进行测试

我们建议您通过以下方式之一自动将RCP访问器添加到AUT。您选择哪种方式取决于您的AUT，构建过程和Eclipse版本，您应该与开发团队核实哪种方法最好：

在产品构建期间：

在构建AUT时将RCP Accessor作为插件添加到AUT是确保它在AUT启动时存在和启动的一种方法。

通过OSGI控制台：

如果在构建期间添加Accessor不是一个选项，那么您可以在构建后通过OSGI控制台添加它。此选项的可用性取决于您使用的Eclipse版本以及您的AUT：它必须允许此类型的事后包含插件才能使用此选项。

通过P2导演：

或者，您可以使用P2控制器在产品构建后添加它。此选项的可用性取决于您使用的Eclipse版本以及您的AUT：它必须允许此类型的事后包含插件才能使用此选项。

通过帮助 - > 安装新软件...菜单（如果有）：

文件<installation-directory> /development/rcp-accessor-site.zip是一个P2更新站点，您可以通过帮助 - > 安装新软件轻松安装Accessor ...菜单。请注意，此菜单的可用性取决于RCP应用程序的设置。
