价值变量

标识符：
org.eclipse.core.variables.valueVariables

以来：
3.0

描述：
此扩展点提供了一种用于定义用于字符串替换的变量的机制。值变量具有静态值。

配置标记：

<！ELEMENT 扩展 （ 变量 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 变量 EMPTY>

<！ATTLIST变量

名称CDATA #REQUIRED

readOnly（true | false）

initialValue CDATA #IMPLIED

initializerClass CDATA #IMPLIED

说明CDATA #IMPLIED>

  • name - 指定此变量的唯一名称。
  • readOnly - 指定此变量是否为只读。如果为true，则此变量的值将始终从扩展派生，并且不能由用户编辑。如果为false，则此变量的值将从任何initialValue属性或初始化程序类初始化，并可由用户修改。修改读写变量后，其值将保持不变并覆盖扩展的initialValue或initializer类指定的任何值。未指定时，此属性的值为false 。此属性已在3.3版本中添加。
  • initialValue - 指定此变量的初始值。指定时，不能指定initializerClass属性。
  • initializerClass - 指定实现org.eclipse.core.variables.的Java类的完全限定名称org.eclipse.core.variables.IValueVariableInitializer 。指定时，不能指定initialValue属性。
  • description - 指定此变量的可读描述。

例子：
以下是具有初始值的值变量贡献的示例：

<extension point = “org.eclipse.core.variables.valueVariables” > <variable name = “FOO_HOME” initialValue = “/ usr / local / foo” > </ variable> </ extension>
在上面的示例中，使用初始值“/ usr / local / foo”创建指定的变量。以下是具有初始化程序类的值变量贡献的示例：

<extension point = “org.eclipse.core.variables.valueVariables” > <variable name = “FOO_HOME” initializerClass = “com.example。FooLocator“ > </ variable> </ extension>
在上面的示例中，创建了变量FOO_HOME，并且创建了类“com.example”。FooLocator“将用于在第一次请求时初始化值。

API信息：
initializerClass属性的值必须是实现接口org.eclipse.core.variables的Java类的完全限定名称。IValueVariableInitializer 。
