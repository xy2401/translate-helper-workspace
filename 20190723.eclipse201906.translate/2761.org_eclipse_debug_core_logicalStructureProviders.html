逻辑结构提供者

标识符：
org.eclipse.debug.core.logicalStructureProviders

以来：
3.1

描述：
此扩展点允许贡献者为值提供多种逻辑结构类型。这是扩展点org.eclipse.debug.core.logicalStructureTypes提供的逻辑结构支持的扩展，它允许每个值使用一个逻辑结构类型。

配置标记：

<！ELEMENT 扩展 （ logicalStructureProvider *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT logicalStructureProvider EMPTY>

<！ATTLIST logicalStructureProvider

class CDATA #REQUIRED

modelIdentifier CDATA #REQUIRED>

  • class - 实现ILogicalStructureProvider的Java类的完全限定名称。
  • modelIdentifier - 此逻辑结构提供程序与之关联的调试模型的标识符

例子：
以下是逻辑结构类型扩展点的示例：

<extension point = “org.eclipse.debug.core.logicalStructureProviders” > <logicalStructureProvider class = “com.example。ExampleLogicalStructureProvider“ modelIdentifier = ”com.example.debug.model“ > </ logicalStructureProvider> </ extension>
在上面的示例中，将为com.example.debug.model调试模型中的值的替代逻辑结构查询指定的逻辑结构提供程序，因为它们显示在变量视图中。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core的Java类的完全限定名称。ILogicalStructureProvider 。
