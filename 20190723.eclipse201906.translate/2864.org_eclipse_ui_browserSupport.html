浏览器支持

标识符：
org.eclipse.ui.browserSupport

以来：
3.1

描述：
此扩展点用于提供工作台浏览器支持。该支持负责打开所有Eclipse插件的URL。Workbench提供了一个非常基本的实现，可以作为可选的RCP插件提供更完整的实现。

作为标准支持随产品一起提供的贡献应标记为default 。这样，可以使用未标记为default其他贡献覆盖支持。但请注意，在任何时间点只能激活一个支持。在多个默认和/或非默认贡献的情况下，结果是非确定性的。

配置标记：

<！ELEMENT 扩展 （ 支持 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的标准标识符。
  • id - 扩展实例的可选标识符。
  • name - 扩展实例的可选名称。

<！ELEMENT 支持 EMPTY>

<！ATTLIST支持

class CDATA #REQUIRED

default（true | false）“false”>

  • class - 为工作台提供浏览器支持的类。这个类应该扩展org.eclipse.ui.browser.AbstractWorkbenchBrowserSupport 。
  • default - 指示此支持是否为默认值。如果浏览器支持通常随产品一起提供，则应将其标记为默认浏览器支持。浏览器支持需要覆盖默认支持，此标志应设置为false 。当工作台遇到两个扩展时，它将选择非默认值而不是默认值。

例子：
以下是浏览器支持贡献的示例：
<extension point = “org.eclipse.ui.browserSupport” > <support default = “true” class = “com.example.xyz。MyBrowserSupport“ > </ support> </ extension>

API信息：
期望贡献者提供一个扩展org.eclipse.ui.browser.的类org.eclipse.ui.browser.AbstractWorkbenchBrowserSupport 。

提供的实施：
工作台提供了浏览器支持的简单实现，在注册表中找不到任何贡献时使用该支持。
