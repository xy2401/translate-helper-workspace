CDT语言

标识符：
org.eclipse.cdt.core.language

以来：
CDT 3.1

描述：
此扩展点用于声明语言或语言变体。语言定义如何创建文件的C模型（例如，填充大纲视图）。它还提供了一些钩子，可以将翻译单元的低级解析覆盖到抽象语法树中。实际上，这仅限于C / C ++语言变体。使用此扩展点无法合理地建模完全不同的语言。

配置标记：

<！ELEMENT 扩展 （（ 语言 | pdomLinkageFactory ））+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 语言 （ contentType ）*>

<！ATTLIST语言

id CDATA #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 语言的唯一标识符。请注意，标识符的前缀是贡献插件的ID。
  • name - 语言的可读和可翻译名称。
  • class - 扩展org.eclipse.core.model.的类org.eclipse.core.model.AbstractLanguage 。

<！ELEMENT contentType EMPTY>

<！ATTLIST contentType

id CDATA #IMPLIED>

  • id -

<！ELEMENT pdomLinkageFactory EMPTY>

<！ATTLIST pdomLinkageFactory

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id -
  • class - 创建链接对象的工厂。

例子：
<language class = “org.eclipse.cdt.core.dom.ast.gnu.c。GCCLanguage“ id = ”gcc“ name = ”GNU C“ > <contentType id = ”org.eclipse.cdt.core.cSource“ /> <contentType id = ”org.eclipse.cdt.core.cHeader“ /> </语言>

API信息：
客户端需要扩展抽象类org.eclipse.core.model.AbstractLanguage 。

提供的实施：
CDT附带以下内置语言： org.eclipse.cdt.core.language.gcc （GNU C）和org.eclipse.cdt.core.language.g++ （GNU C ++）。

相应的语言实现是org.eclipse.cdt.core.dom.ast.gnu.c.GCCLanguage和org.eclipse.cdt.core.dom.ast.gnu.cpp.GPPLanguage
