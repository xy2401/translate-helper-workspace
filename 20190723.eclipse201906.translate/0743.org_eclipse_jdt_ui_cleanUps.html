干净的Ups

标识符：
org.eclipse.jdt.ui.cleanUps

以来：
3.5

描述：
此扩展点允许添加可以解决编译单元中的问题的清理。

配置标记：

<！ELEMENT 扩展 （ cleanUp *， cleanUpConfigurationUI *， cleanUpOptionsInitializer *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT cleanUp EMPTY>

<！ATTLIST cleanUp

class CDATA #REQUIRED

id CDATA #REQUIRED

在IDREF #IMPLIED>之后运行

  • class - 实现接口org.eclipse.jdt.ui.cleanup.的完全限定类名org.eclipse.jdt.ui.cleanup.ICleanUp
  • id - 将用于标识此清理的唯一ID
  • runAfter - 可选的清理id，指定此清理应在其后运行

<！ELEMENT cleanUpConfigurationUI EMPTY>

<！ATTLIST cleanUpConfigurationUI

名称CDATA #REQUIRED

class CDATA #REQUIRED

cleanUpKind（cleanUp | saveAction）>

给定清理类型的清理配置UI。每个贡献清理必须至少提供允许禁用它的UI 。


  • name - 允许在UI中标识此配置的名称
  • class - 实现接口org.eclipse.jdt.ui.cleanup.的完全限定类名org.eclipse.jdt.ui.cleanup.ICleanUpConfigurationUI
  • cleanUpKind - 此配置UI配置的清理方式。目前仅支持以下内容：
    • cleanUp：正常清理
    • saveAction：在Java编辑器中保存时执行的清理

<！ELEMENT cleanUpOptionsInitializer EMPTY>

<！ATTLIST cleanUpOptionsInitializer

cleanUpKind（cleanUp | saveAction）

class CDATA #REQUIRED>

一个选项初始值设定项，它返回清理类型的默认选项。每次清理都会查询所有提供的初始化程序，以检查它是否已启用。


  • cleanUpKind - 初始化此初始化程序的清理方式。目前仅支持以下内容：
    • cleanUp：正常清理
    • saveAction：在Java编辑器中保存时执行的清理
  • class - 实现接口org.eclipse.jdt.ui.cleanup.的完全限定类名org.eclipse.jdt.ui.cleanup.ICleanUpOptionsInitializer

例子：
以下是更新保存版权的清理示例：

<extension point = “org.eclipse.jdt.ui.cleanUps” > <cleanUp id = “org.eclipse.jdt.copyrightsaveaction.copyright_clean_up” class = “org.eclipse.jdt.ui.internal.copyrightupdater。CopyrightUpdaterCleanUp“ > </ cleanUp> <cleanUpOptionsInitializer class = ”org.eclipse.jdt.ui.internal.copyrightupdater。CopyrightOnSaveOptionsInitializer“ cleanUpKind = ”saveAction“ > </ cleanUpOptionsInitializer> <cleanUpConfigurationUI class = ”org.eclipse.jdt.ui.internal.copyrightupdater。CopyrightTabPage“ name = ”％cleanUpConfigurationUI.name“ cleanUpKind = ”saveAction“ > </ cleanUpConfigurationUI> </ extension>
