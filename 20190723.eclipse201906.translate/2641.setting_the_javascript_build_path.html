设置JavaScript构建路径

JavaScript构建过程扫描项目的JavaScript构建路径上的所有文件，文件夹，项目和库，以便项目中引用的资源可用于内容辅助选项。因此，必须将所需资源包含在JavaScript构建路径中。

JavaScript的Build Path允许您选择要包含/排除此进程的JavaScript资源。而不是自动扫描所有引用的项目和库，配置JavaScript构建路径允许您选择将扫描哪些资源，因此可以显着加快构建过程。

注意：

必须先为项目启用 JavaScript支持，然后才能设置JavaScript构建路径。

JavaScript构建路径由项目的JavaScript库属性页面中包含的资源决定。

配置项目的JavaScript构建路径：

  1. 在Project Explorer视图中，右键单击所需项目，然后选择Properties | JavaScript | JavaScript库 。在此属性页面中，您可以将以下元素添加到项目的JavaScript构建路径中：
  2. 库 - 内置和用户定义的库包含预先编写的JavaScript源文件集合，这些文件具有原型对象/类定义和JsDoc。
  3. 源 - 文件系统中的其他文件夹/文件/变量或从外部源链接。
  4. 项目 - 工作区或文件系统中的其他项目
  5. 全局超级类型 - 添加的库中的每个对象/字段/功能都将添加到项目的全局范围中。然后，项目中的每个JavaScript文件都可以访问这些对象/字段/函数。

库选项卡

如果您的项目引用了JavaScript库，则应将其添加到此列表中，以使其元素可用于内容辅助选项。

通过Libraries选项卡，您可以添加PDT的内置库，或创建和添加您自己的用户库。

注意：

默认的运行时库（脚本语言库，对JSDT和ECMA 3浏览器支持库的Web项目支持）会自动添加到JavaScript构建路径中。

要将内置库添加到JavaScript构建路径：

  1. 单击“ 添加运行时库”....将出现“添加库”对话框。
  2. 选择所需的库，然后单击下一步并完成。

要将用户库添加到JavaScript构建路径：

  1. 单击“ 添加运行时库”....将出现“添加库”对话框。
  2. 选择User Library ，然后单击Next。将显示先前配置的用户库列表。
  3. 如果您尚未配置用户库：
  4. 单击“用户库”按钮以进入“JavaScript用户库”首选项页面（也可从“ 窗口”|“首选项”|“JavaScript”|“包括路径”|“用户库”访问 ）。
  5. 单击新建。将打开“新建用户库”对话框。
  6. 输入要添加的库的名称，然后单击“确定”。库名称将添加到“用户库”列表中。这只是一个占位符名称，您将添加所需的文件以创建自己的库。
  7. 从列表中选择名称，然后单击“添加.js”。“
  8. 在.js文件选择对话框中，浏览到文件系统上所需的JavaScript文件，然后单击“打开”。
  9. JavaScript文件将添加到您的用户库中。
  10. 单击“确定”返回“用户库”选择列表。
  11. 选中要添加到项目的构建路径的用户库的复选框，然后单击完成。用户库将添加到项目的JavaScript Libraries Build Path列表中。

要将工作区中的库文件夹添加到JavaScript构建路径：

  1. 单击“添加库文件夹”按钮。库文件夹选择对话框将显示可用文件夹列表。
  2. 选择所需的文件夹，然后单击“确定”。

来源标签

Source选项卡允许您从JavaScript Build Path中包含/排除特定文件，文件夹和变量。

要从Workspace添加文件夹：

  1. 单击添加文件夹。将显示“源文件夹选择”对话框。
  2. 选择所需文件夹并单击“确定”。

要添加外部文件夹：

  1. 单击添加外部文件夹。将显示“链接源”对话框。
  2. 单击“浏览”从文件系统中选择文件夹，然后单击“确定”。

要从文件系统添加JavaScript文件：

  1. 单击“从文件系统添加文件”。
  2. 选择JavaScript文件，然后单击“打开”。

要从Workspace添加JavaScript文件：

  1. 单击“从项目添加文件”。
  2. 选择所需的文件，然后单击“确定”。

使用Classpath Variable可以指向JavaScript库，而无需指定它的本地文件系统位置。在团队中共享资源时，这很重要。

要添加变量类路径条目：

  1. 单击添加变量。
  2. 将显示已配置的类路径变量列表。如果尚未配置任何类路径变量：
  3. 单击Configure Variables按钮以打开JavaScript Include Path Variables首选项页面（也可从Window | Preferences | JavaScript | Include Path | Include Path Variables访问）。
  4. 单击新建。将出现“新建变量输入”对话框。
  5. 输入变量的名称和路径（单击“文件”或“文件夹”按钮以浏览到路径）。
  6. 单击确定。变量将添加到“包含路径变量”列表中。
  7. 单击“确定”返回“变量类路径条目”选择列表。
  8. 选择要添加到项目的JavaScript构建路径的变量，然后单击“确定”。

全局超类型选项卡

项目的全局范围包含已添加到JavaScript构建路径的库中包含的所有对象，字段和函数。全局SuperType中的元素将可用于项目中的所有PHP和JavaScript文件。

此外，Global SuperType选项卡允许您选择“SuperType”类。所有未在其自己的类中定义的JavaScript元素将自动添加到此类中。这将允许您根据所选的SuperType类名从内容辅助列表中选择这些元素。

要配置SuperType类：

  1. 单击“选择并重新订购库”按钮。将打开JavaScript Sourcefile SuperType Selection对话框。
  2. 选择SuperType类的类名，然后单击“确定”。未在类中定义的所有元素都将添加到选定的类中。输入类名将在“内容辅助”列表中显示这些元素。

注意：

变量指向文件夹。要选择文件夹中的存档，请单击“扩展”。

单击确定。

您可以选择在已添加的源文件夹中包含或排除特定资源或文件名模式。

要包含/排除JavaScript构建路径中的特定资源：

  1. 展开要包含/排除资源的文件夹旁边的节点。将显示包含和排除的资源列表（默认情况下，包括文件夹中的所有资源）。
  2. 选择“包含”或“排除”列表，然后单击“编辑”。将显示“包含和排除模式”对话框。
  3. 将显示“添加包含/排除模式”对话框。
  4. 输入或选择包含/排除所需的资源/模式，然后单击“确定”。
  5. 单击完成。文件夹中与包含模式匹配但与排除模式不匹配的所有资源都将添加到JavaScript构建路径中。

项目标签

“项目”选项卡允许您从Workspace中选择要添加到JavaScript构建路径的项目。

要将项目添加到JavaScript构建路径：

  1. 单击“添加..”按钮。将显示“必需的项目选择”对话框。
  2. 选择所需项目，然后单击“确定”。所选项目将添加到JavaScript构建路径中。

注意：

在构建过程中，引用的项目将在项目本身之前构建。

保存更改

对JavaScript构建路径进行所有必要的更改后，单击“确定”。

单击“包含或排除”模式窗格旁边的“添加”。根据您配置的设置，将重建您的项目以反映更改。
