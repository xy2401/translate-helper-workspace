入门

警告：Eclipse QVTd 0.14.0执行不够成熟，不仅仅是实验/研究用途。

要快速演示QVTc或QVTr，您可以安装相应的示例项目。

QVTr示例项目

调用File-> New-> Example ...然后选择Examples然后选择QVT（查询/查看/转换语言）项目然后选择QVTr HSTM2FSTM 项目然后选择Finish以创建一个名为org.eclipse.qvtd.examples.qvtrelation的小示例项目。 hstm2fstm 。

将打开QVTr或QVTc编辑器以显示转换。

该图显示了一个带有hovertext的映射，详细说明了name引用的声明。

QVT编辑器扩展了OCL编辑器，因此OCL上下文菜单提供了有用的功能，例如以Abstyract语法形式保存转换。

QVTr项目目前只是一个普通的Java项目。（未来可能会添加QVTd性质以支持自动构建。）

QVTr Hierarchical to Flat State Machine示例包含

  • HierarchicalStateMachine2FlatStateMachine.qvtr - 转换

  • HierarchicalStateMachine.ecore - 源元模型

  • FlatStateMachine.ecore - 目标元模型

  • hier.xmi - 执行执行的示例模型

  • expected.xmi - 预期的执行结果

  • HierarchicalStateMachine2FlatStateMachine.qvtr.launch - 用于执行的启动配置

您可以通过调用Run-> Run Configurations ...然后QVTr（Relations）转换然后HierarchicalStateMachine2FlatStateMachine.qvtr来对hier.xmi输入执行转换。

目前，该项目的发布信息过多，而且还没有实施设施。重要的领域是：

项目/转型

项目名称提供了一个默认名称，解析其他文件名以减少屏幕混乱。

转换名称是要执行的转换。

该模式目前只能强制创建/覆盖输出模型。

方向在执行多方向转换时选择输出方向。

输入

必须指定绑定到每个输入域的文件。

新产出

必须指定绑定到每个输出域的文件。

中间体

识别转换链中每个中间文件的位置。它们的默认位置是转换父文件夹的临时子文件夹。你可以改变它们。

如果单击“ 编译” ，则可以看到编辑进度，因为灰色过时文本更改为绿色就绪文本。

建立

解释复选框在解释执行（快速启动，慢速运行）或代码生成执行（慢启动，快速运行）之间进行选择。

解释的编译合成QVTc中间元模型和genmodel。

  • HierarchicalStateMachine2FlatStateMachine.ecore

  • HierarchicalStateMachine2FlatStateMachine.genmodel

生成的代码编译还合成了一个Java类来实现转换和QVTc中间模型的Java类。

  • HierarchicalStateMachine2FlatStateMachine.java

生成的文件生成到不同的src-gen树，您可能需要刷新该文件以查看所有文件。Java问题标记在合成和构建期间来来往往。在执行过程之前，错误应该全部消失。消除警告是一项正在进行的工作。

跑

如果您尚未单击“ 运行”，则会自动编译 。

结果应该是flat.xmi文件，类似于expected.xmi文件。

QVTc示例项目

可以通过以下方式安装传统UML到RDBMS转换的OMG QVTc变体：

调用File-> New-> Example ...然后选择Examples然后选择QVT（查询/查看/转换语言）项目然后选择QVTc UML2RDBMS Project然后选择Finish以创建一个名为org.eclipse.qvtd.examples.qvtcore的小示例项目。 uml2rdbms 。

QVTc编辑器打开以显示转换。

QVTc工具与QVTr工具非常相似。主要区别在于，对于QVTc，中间模型及其genmodel由用户设计，而QVTr中间模型和genmodel是自动合成的。如果需要代码生成的Java执行，则使用genmodel为中间模型生成Java代码也是手动用户的责任。
