调试器

Debuggers首选项页面允许您配置调试器设置。

可以从Window |访问“调试器首选项”页面偏好| PHP |调试|调试器 。

例如，配置Zend Debugger设置：

  1. 选择Zend Debugger。
  2. 单击配置。Zend Debugger Settings对话框将打开。

您可以配置以下内容：

  • 客户端主机/ IP - 输入将返回调试结果的客户端主机/ IP。PDT将自动搜索并识别客户端主机/ IP，但是输入特定的主机/ IP将加快调试过程并降低会话超时的可能性。
  • 调试端口 - Zend Debugger将使用的端口。默认端口为10137。
  • 调试响应超时（毫秒） - 调试器响应超时（以毫秒为单位）。
  • 广播端口 - 广播端口允许您的Zend浏览器工具栏检测您的调试首选项。此处输入的广播端口号必须与Zend Browser Toolbar中输入的广播端口匹配。默认端口为20080。
  • 虚拟文件名 - 这是PHP脚本调试器用于测试指定服务器上的PHP调试会话的文件。该名称应保留为默认的dummy.php。但是，如果更改了此项，请确保已在服务器上进行更改。
  • 运行时显示调试信息 - 标记执行运行配置时要显示的调试信息的复选框，例如控制台视图和浏览器输出。
  • 使用新协议 - 默认选中此复选框，并应用最新Zend Debugger引擎的使用。如果您遇到Zend Debugger的任何问题，请取消选中此复选框以恢复到之前的Zend Debugger协议。

注意：

某些调试器全局设置（负责与调试器引擎建立连接）可以在相应的调试器所有者级别（PHP Server或PHP Executable）上重写。如果为调试器所有者设置了单独的连接设置，那么在从IDE触发调试会话时将考虑这些设置。如果无法在IDE中找到调试器所有者设置（例如，在使用浏览器工具栏触发IDE外部的调试会话时），将使用全局设置来建立连接。
