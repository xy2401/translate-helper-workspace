的ServiceHandler

标识符：
org.eclipse.rap.ui.serviceHandler

以来：
RAP 1.3

描述：
服务处理程序可用于处理自定义请求，例如将文件，图像或其他内容传送到客户端。在用户的会话范围中调用服务处理程序，即它们可以访问会话信息。它们可以像servlet一样实现，即它们可以从请求对象访问头和参数，并将结果写入响应编写器或输出流。具有特定参数的URL可以访问服务处理程序。此URL可以从RWT.getServiceManager().getServiceHandlerUrl( "download" )获取RWT.getServiceManager().getServiceHandlerUrl( "download" )

配置标记：

<！ELEMENT 扩展 （ serviceHandler +）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT serviceHandler EMPTY>

<！ATTLIST serviceHandler

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 此服务处理程序的唯一ID。此ID用于获取访问服务处理程序的URL。
  • class - 服务处理程序实现。

例子：
这是一个注册两个服务处理程序的扩展示例：
<extension point = “org.eclipse.rap.ui.serviceHandler” > <serviceHandler class = “org.eclipse.rap.ui.impl。ServiceHandler1“ id = ”myHandler1“ > </ serviceHandler > < serviceHandler class = ”org.eclipse.rap.ui。ServiceHandler2“ id = ”myHandler2“ > </ serviceHandler > </ extension>
