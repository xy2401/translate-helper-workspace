Java逻辑结构

标识符：
org.eclipse.jdt.debug.javaLogicalStructures

以来：
3.1

描述：
此扩展点允许开发人员为指定类型的Java对象定义逻辑结构。通过评估提供的代码片段来创建逻辑值。

配置标记：

<！ELEMENT 扩展 （ javaLogicalStructure +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT javaLogicalStructure （ 变量 +）？>

<！ATTLIST javaLogicalStructure

键入CDATA #REQUIRED

subtypes（true | false）“true”

值CDATA #IMPLIED

说明CDATA #REQUIRED>

  • type - 类型的完全限定名称。
  • subtypes - 指定是否应将此Java逻辑结构也用于指定类型的子类型的对象，或仅用于指定类型的对象。此属性是可选的，默认值为true 。
  • value - 要评估以创建逻辑值的代码段。此属性是可选的，如果未指定，则扩展必须声明一个或多个变量。
  • description - 对此逻辑结构的描述。

<！ELEMENT 变量 EMPTY>

<！ATTLIST变量

名称CDATA #REQUIRED

值CDATA #REQUIRED>

此类型对象的逻辑值的一个变量。


  • name - 要创建的变量的名称
  • value - 将作为变量值计算的代码段

例子：
以下是具有两种结构的Java逻辑结构扩展点的示例：

<extension point = “org.eclipse.jdt.debug.javaLogicalStructures” > <javaLogitalStructure subtypes = “true” value = “return entrySet（）。toArray（）;” type = “java.util。Map“ /> <javaLogitalStructure subtypes = ”true“ type = ”java.util。Map $ Entry“ > <variable value = ”return getKey（）;“ name = ”key“ /> <variable value = ”return getValue（）;“ name = ”value“ /> </ javaLogitalStructure> </ extension>
在上面的示例中，Map被转换为其条目，Map $ Entry被转换为其键和值。
