Ant任务

标识符：
org.eclipse.ant.core.antTasks

描述：
允许插件定义Ant基础结构使用的任意Ant任务。标准的Ant基础结构允许添加任意任务。不幸的是，Ant Core插件不太可能在其类路径（或其任何先决条件）上具有这些任务所需的类。为了解决这个问题，客户端应该定义一个插入此扩展点的扩展，并将任务名称映射到一个类。然后，Ant插件可以请求声明插件加载指定的类。

配置标记：

<！ELEMENT 扩展 （ antTask *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT antTask EMPTY>

<！ATTLIST antTask

名称CDATA #REQUIRED

uri CDATA #IMPLIED

class CDATA #REQUIRED

库CDATA #REQUIRED

无头（真|假）

eclipseRuntime（true | false）>

  • name - 要定义的任务的名称
  • uri - 这个定义应该存在的uri。仅在使用比1.6更新的Ant时使用。
  • class - 实现任务的Java类的完全限定名称。通常，此类必须是org.apache.tools.ant.的子类org.apache.tools.ant.Task 。
  • library - 相对于包含任务的库的插件安装位置的路径。
  • headless - 表示此任务是否适合在“无头”Ant环境中使用。如果无头运行且属性为“false”，则不会加载或定义任务。同样，插件类加载器不会作为父类加载器添加到Ant类加载器中。未指定时，隐含值为true。
  • eclipseRuntime - 指示此任务是否需要Eclipse运行时（即必须在与Eclipse相同的VM中运行）。未指定时，隐含值为true。

例子：
以下是Ant任务扩展点的示例：

<extension point = “org.eclipse.ant.core.antTasks” > <antTask name = “coolTask” class = “com.example。CoolTask“ library = ”lib / antSupport.jar“ /> </ extension>
在上面的示例中，必须使用Ant项目构建器创建antSupport.jar 。有关创建项目构建器及其工作方式的更多信息，请参阅以下帮助主题： 贡献任务和类型 ，将Ant构建文件作为项目构建器和创建项目构建器Ant构建文件

提供的实施：
平台本身提供了许多任务，包括eclipse.incrementalBuild和eclipse.refreshLocal。
