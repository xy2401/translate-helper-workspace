标识符：
org.eclipse.core.resources.teamHook

以来：
2.1

描述：
用于提供TeamHook的实现，该实现仅用于团队提供者可用的机制。此扩展点最多可容忍一个扩展。

配置标记：

<！ELEMENT 扩展 （ teamHook ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT teamHook EMPTY>

<！ATTLIST团队挂钩

class CDATA #REQUIRED>

  • class - org.eclipse.core.resources.team.TeamHook

例子：
以下是使用teamHook扩展点的示例。（在文件plugin.xml中）

   <extension point="org.eclipse.core.resources.teamHook"> 
      <teamHook class="org.eclipse.team.internal.TeamHook"/> 
   </extension>

API信息：
class属性的值必须表示org.eclipse.core.resources.team的子类。TeamHook 。

提供的实施：
Team组件通常会提供团队钩子的实现。任何其他客户端都不应使用扩展点。
