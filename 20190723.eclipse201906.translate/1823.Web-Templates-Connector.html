Web模板连接器

我在哪里可以找到Web模板连接器？

它可以从孵化器更新站点获得。有关更新站点的列表，请访问http://www.eclipse.org/mylyn/downloads/ 。（ http://www.eclipse.org/mylyn/downloads本身不是更新站点。）

为什么我无法使用现有模板之一进行连接？

Web连接器在“查询”首选项对话框的“高级配置”部分中有一个“预览”按钮。如果出现错误或预览表未显示预期列表，则可以使用“打开”按钮验证Web连接器下载的网页是否正确。

或者，您可以使用查询模式（{Description}。+？）（{Id} \ n）在预览表中查看检索到的页面的内容，并将其与加载到Web浏览器中的相同URL的网页源进行比较。

如果检索到的网页正确，但“预览”未显示预期结果，则需要检查“查询模式”。一个流行的错误是一个没有掩盖的'？'用于分隔url中的请求参数。例如：

<a href =“show_bug.cgi \？ID \ =（。+？）“>。+？<span class =“summary”>（。+？）</跨度>

如果您没有看到正确的网页，请检查以下内容：

  • 查询URL或替换参数是正确的
    • 注意：只有参数是URL编码的，例如查询URL x：$ {y} ，替换参数$ {y}设置为a：b得到“ x：a％3Ab ”（只有替换参数是URL编码的）
  • Web应用程序要求用户在Web连接器存储库配置中登录和登录配置是否正确（请参阅Web连接器存储库设置）

要进一步调试Web应用程序，您可以使用wget工具（在Windows上它可用作独立的命令行工具，也包含在Cygwin中）并运行以下命令：

wget -v -d -o query.log -O query.html“http：//your.query.url”

如果存储库需要基本的http auth，则需要使用--http-user = USER和--http-password = PASS选项。

然后在Web浏览器中打开文件query.html ，并检查它是否显示您需要从Web服务器获取的问题列表。如果它确实具有正确的列表，则可能存在用于解析的查询模式的问题。

如果query.html没有显示预期的问题列表，您可以检查query.log以获取服务器响应，并尝试确定服务器是否需要身份验证，在某处重定向您的请求或发出任何其他错误。

已知限制

  • 在Mylyn 1.0中：如果Web服务器通过在获取资源时请求重定向到绝对URL来响应，则连接器会错误地将“/”添加到URL，并且请求将失败。解决方法是使用重定向所指向的最终URL（您可以如上所述找到它）。有关详细信息，请参阅错误167282 。
