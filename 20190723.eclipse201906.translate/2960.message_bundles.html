消息包

描述

标准Java ResourceBundle具有非常低效的空间特性。由于正在运行的Eclipse往往有许多外化消息，因此我们实现了一个新的消息包系统，以便在Eclipse中使用。该机制非常简单，完全通用 - 可以在任何地方使用。

新方法摘要：

  • messages.properties - 此文件与之前的文件相同，但所有密钥都必须是有效的Java标识符。
  • 每个消息文件都有一个相应的Java类。
  • 文件中的每个键/值对都有一个public static String字段，其名称与消息键相同。
  • 加载消息包时，字段的值将设置为messages.properties文件中的值。
  • 消息属性文件将从内存中清除。

创建新消息时：

  • 在Messages.java文件中创建一个字段。
  • 在messages.properties文件中创建一个键/值对，其中键名与字段名称匹配，
  • 要引用该消息，只需引用该字段（例如Messages.my_key ）而不是标准查找。

示例文件：

客户代码

旧代码：

public class MyClass {public void myMethod（）{String message; ...// no args message = Messages.getString（“key.one”）; // $ NON-NLS-1 $ ...//绑定一个arg message = MessageFormat.format（Messages.getString（“key.two”），new Object [] {“example usage”}）; // $ NON-NLS-1 $ // $ NON-NLS-2 $ ...}}

新守则：

public class MyClass {public void myMethod（）{String message; ...//没有args message = Messages.key_one; ...//绑定一个arg消息= NLS.bind（Messages.key_two，“example usage”）; // $ NON-NLS-1 $ ...}}

Messages.java

旧代码：

public class Messages {private static final String BUNDLE_NAME =“org.eclipse.core.utils.messages”; // $ NON-NLS-1 $ private static final ResourceBundle bundle = ResourceBundle.getBundle（BUNDLE_NAME）; public static String getString（String key）{try {return bundle.getString（key）; } catch（MissingResourceException e）{return key; }}}

新守则：

import org.eclipse.osgi.util。NLS;
公共类消息扩展NLS {private static final String BUNDLE_NAME =“org.eclipse.core.utils.messages”; // $ NON-NLS-1 $ public static String key_one; public static String key_two; ...static {NLS.initializeMessages（BUNDLE_NAME，Messages.class）; }}

messages.properties

旧代码：

key.one = Hello world。key.two =这是一个带有一个参数的绑定{0}。

新守则：

key_one = Hello world。key_two =这是一个带有一个参数的绑定{0}。
