扩展模板

标识符：
org.eclipse.pde.ui.templates

以来：
2.0

描述：
此扩展点注册用于为新扩展生成代码的插件项目内容模板。模板在两种情况下使用：

  • 一个或多个模板组合在一个向导中，该向导使用org.eclipse.pde.ui.pluginContent扩展点作为插件内容向导提供。这些模板为新创建的插件项目创建了有趣的内容。此外，使用此扩展点贡献的所有模板都可以在插件内容向导的特殊版本中看到，该向导列出了模板，并允许用户通过在列表中检查模板来自由组合模板。
  • 可以使用模板将新扩展添加到现有插件。

配置标记：

<！ELEMENT 扩展 （ 模板 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 模板 EMPTY>

<！ATTLIST模板

id IDREF #REQUIRED

名称CDATA #REQUIRED

icon CDATA #IMPLIED

class CDATA #REQUIRED

contributionsId IDREF #REQUIRED>

  • id - 将用于标识此模板的唯一名称。
  • name - 将在此模板的UI表示中使用的可翻译名称。
  • icon - 将用于直观表示模板的图标的相对路径。
  • class - 实现org.eclipse.pde.ui.templates.的类的完全限定名称org.eclipse.pde.ui.templates.ITemplateSection界面。
  • contributionsId - 此模板将贡献的扩展点的标识符。

例子：
以下是模板注册的示例：

<extension point = “org.eclipse.pde.ui.templates” > <template contributionsId = “org.eclipse.ui.actionSets” name = “XYZ Action Set Generator” class = “com.example.xyz。XYZActionSetTemplate“ id = ”com.example.xyz。ActionSetTemplate“ > </ template> </ extension>

API信息：
每个模板都必须提供一个实现org.eclipse.pde.ui.templates.的类org.eclipse.pde.ui.templates.ITemplateSection界面。但是，可以使用实现接口并可以扩展的抽象类。

提供的实施：
PDE UI提供了许多模板，可以为最流行的扩展点（如编辑器，视图，首选项等）创建扩展。
