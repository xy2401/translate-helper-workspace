Java编辑器的新增功能

Scout使用Eclipse IDE的标准Java编辑器。为了简化开发，Scout SDK为Java编辑器提供了其他功能。

为了创建Scout框架中由内部类表示的新Scout组件，Scout工具扩展了Eclipse Java编辑器的Java内容辅助。提供的提案是特定于上下文的。根据Java编辑器中当前的光标位置，可能的Scout组件将添加到提议列表中。

在表示表单中的组框的类中，Scout内容辅助添加了针对各种表单字段的提议，并且在表类中，内容辅助添加了添加表列或上下文菜单的提议。这些提议会触发为表单字段，表格列或代码创建内部类。可以通过键入Ctrl + Space来启动Eclipse内容辅助。

创建新的表单字段

要向表单添加其他表单域，当前编辑位置需要位于Scout组框内。键入Ctrl + Space然后可以访问最常用的Scout小部件， 如图1所示。

图1。建议在GroupBox中创建新的表单字段

选择模板后，可以通过使用Tab键在不同的编辑组之间导航来自定义模板（这与Eclipse编辑器中的其他模板完全相同）。使用此机制，您可以快速定义类名，父类和其他属性。要退出编辑模式，只需按Enter键 。

创建新的表列

要在表中添加新列，请在Scout表中设置当前编辑位置。Scout表本身可以位于TableField的内部， 如图2所示，也可以位于Scout TablePage的内部。

图2。建议在表中创建新列

在添加列之后， 图2中所示的内容辅助也可用于向表添加键击操作和菜单。

创建新代码

内容辅助支持向现有CodeType添加新代码， 如图3所示。

图3。建议在CodeType中创建新代码

NLS-Entry支持

当前编辑位置在TEXTS.get()代码的String参数内时，内容辅助（使用Ctrl + Space打开）提供对NLS条目的支持， 如图4所示。

图4。与NLS支持相对应的提案。

选择一个提案条目（例如示例中的“DateOfBirth”）会在右侧显示可用的翻译。要选择特定的提案条目，您可以双击该条目或按Enter键。要创建一个新的文本输入选择新建文本......在提案列表的末尾。

选择“新文本...”然后打开NLS条目向导 。
