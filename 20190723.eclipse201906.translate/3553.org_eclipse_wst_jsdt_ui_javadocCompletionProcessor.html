Jsdoc完井处理器

不推荐使用此扩展点，使用org.eclipse.wst.jsdt.ui.javaCompletionProposalComputer作为替代。

标识符：
org.eclipse.wst.jsdt.ui.javadocCompletionProcessor

描述：
此扩展点允许添加JSdoc完成处理器以提供新的JSdoc标记。

自2001年起不推荐使用。扩展器应该为org.eclipse.wst.jsdt.ui.javaCompletionProposalComputer扩展点做出贡献，它不仅可以为jsdoc提供其他提议，还可以为JavaScript编辑器的任何部分提供其他提议。

配置标记：

<！ELEMENT 扩展 （ javadocCompletionProcessor +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT javadocCompletionProcessor EMPTY>

<！ATTLIST javadocCompletionProcessor

id CDATA #REQUIRED

名称CDATA #IMPLIED

class CDATA #IMPLIED>

  • id - Jsdoc完成处理器的唯一标识符
  • name - Jsdoc完成处理器的本地化名称
  • class - 实现此Jsdoc完成处理器的类的名称。该类必须是公共的并实现org.eclipse.wst.jsdt.ui.text.java.IJavadocCompletionProcessor与一个公共0参数构造函数。

例子：
以下是Jsdoc完成处理器贡献的示例：

<extension point = “org.eclipse.wst.jsdt.ui.javadocCompletionProcessor” > <javadocCompletionProcessor id = “XDocletJavadocProcessor” name = “XDoclet Javadoc Processor” class = “com.example。XDocletJavadocProcessor“ > </ javadocCompletionProcessor> </ extension>
