编码

标识符：
org.eclipse.ui.encodings

以来：
3.1

描述：
用户可用于选择文件编码的编码列表。此列表对应于一系列Charsets的Charset＃getName（）的值。指定但不可用的那些将在工作台中如此指示，并且将不能被用户选择。

配置标记：

<！ELEMENT 扩展 （ 编码 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 编码 EMPTY>

<！ATTLIST编码

名称CDATA #REQUIRED>

  • name - charset的名称。如果将此值发送到Charset＃getName（String），则将返回相应的Charset。

例子：
以下是编码扩展的示例：


   <extension 
       point="org.eclipse.ui.encodings">
      <encoding name="ISO-8859-1"/>
      <encoding name="UTF-8"/>
      <encoding name="US-ASCII"/>
   </extension>

API信息：
使用方法org.eclipse.ui.WorkbenchEncoding.getDefinedEncodings用于检索此扩展点定义的编码。使用方法org.eclipse.ui.WorkbenchEncoding.getWorkbenchDefaultEncoding从虚拟机中检索默认编码。

提供的实施：
工作台提供了一些常用的编码。
