CPS到部署单元测试

xform.m2m.tests项目指定了大量独立于转换实现的测试用例，并且是为了允许基于规范的转换的测试驱动开发而创建的。

根据CPS-to-Deployment转换规范中的规则，将测试组合在一起成为类：

  • HostMappingTest

  • ApplicationMappingTest

  • StateMachineMappingTest

  • StateMappingTest

  • TransitionMappingTest

  • ActionMappingTest

此外， TransformationApiTest类确保所有变体对于不正确的输入参数的行为方式相同，而WrapperTest类可以包含针对特定变换变量包装器的其他测试。

技术细节

运行测试

只需右键单击xform.m2m.tests项目，然后选择Run as…​ → JUnit Plugin Test 。

测试用例结构

所有与规范相关的测试都扩展了CPS2DepTest类，它定义了一些扩展字段：

class CPS2DepTest {protected extension Logger logger = Logger.getLogger（“cps.xform。CPS2DepTest“）受保护的扩展CPSTransformationWrapper xform受保护的扩展CPSModelBuilderUtil模型构建器

一个简单的测试用例的结构如下：

@Test def hostIncremental（）{val testId =“hostIncremental”info（“START TEST：”+ testId）//使用模型构建器创建初始模型val cps2dep = prepareEmptyModel（testId）//使用转换包装器进行初始化（例如创建规则） ）cps2dep.initializeTransformation //使用转换包装器执行executeTransformation //修改模型val instance = cps2dep.prepareHostInstance //重新执行转换（增量通常忽略此调用）executeTransformation //检查结果cps2dep.assertHostMapping（instance）info（“END测试：“+ testId）}

使用现有输入模型进行测试

如果您想在自己的输入CPS模型上测试转换，请查看<cps>.xform.m2m.tests.integration包中InstanceModelTest.xtend中的specificInputModel测试用例。

@Ignore @Test def specificInputModel（）{val testId =“specificInputModel”info（“START TEST：”+ testId）val cpsUri =“file：//my-cps-git-location/models/org.eclipse.viatra.query .examples.cps.instances / example.cyberphysicalsystem“val cps2dep = prepareCPSModel（cpsUri）cps2dep.initializeTransformation executeTransformation info（”END TEST：“+ testId）}

只需将资源URI更改为完整文件路径，删除或注释掉@Ignore并运行测试。
