使用团队支持

您可以使用任何供应商提供的任何支持Eclipse的存储库。以下信息描述了使用Eclipse附带的Concurrent Versions System（CVS）支持的项目共享。步骤和用户界面元素将根据您已安装的存储库支持而有所不同。
注意：第一次将RemoteSystemsConnection项目与存储库同步时，您将看到许多文件存在冲突。这是因为它需要一次同步才能将服务器的编码版本信息与本地文件正确对齐。这只发生一次，您只需接收所有传入的更改，然后您的本地工作区将与服务器同步。在与中央服务器同步后，您可能还会在Team视图中找到属于其他队友的非活动配置文件。如果要在本地工作站上的远程系统资源管理器中查看配置文件的内容，可以选择是否在“组”视图中激活这些配置文件。

在团队编程环境中，团队成员可以在自己的工作台中工作，与其他人隔离。最终，他们希望与队友分享他们的工作。Remote System Explorer允许它们共享其连接，过滤池和过滤器。分享这些资源：

  1. 在“远程系统资源管理器”透视图中，选择“团队”选项卡，该选项卡默认位于与“远程系统”视图相同的窗格中。这将使Team视图位于Remote System Explorer的前端。你也可以使用 “远程系统”视图上的按钮，然后选择“ 使用配置文件”操作。
  2. 展开RemoteSystemsConnections 。将显示您已定义的配置文件以及团队配置文件。如果您不想在创建和管理的配置文件中共享它们，则远程系统资源管理器将创建团队配置文件，以用于共享连接，过滤器池和过滤器。
  3. 右键单击RemoteSystemsConnections，然后选择Team > Share Project 。
  4. 选择CVS并单击Next 。
  5. 如果已定义CVS存储库， 请选择“ 使用现有存储库位置” 。如果需要定义CVS存储库，请选择“ 创建新的存储库位置”（此信息应该来自在服务器上配置CVS存储库的人员）。要创建CVS存储库定义：
    1. 在“ 主机”字段中，键入服务器的地址，例如mymachine.com。
    2. 在Repository path字段中，键入服务器上存储库的路径，例如/ home / test，d：/ test。
    3. 在“ 用户”字段中，键入用于连接到存储库的用户名。
    4. 在“ 密码”字段中，键入用户名的密码。
    5. 从“ 连接类型”列表中，选择CVS服务器的身份验证协议。Eclipse CVS客户端有三种连接方法：
      • pserver ：CVS特定的连接方法。
      • extssh ：工作台附带的SSH 1.0客户端。
      • ext ：使用SSH等外部工具连接存储库的CVS ext连接方法。ext使用的工具在Team > CVS > EXT Connection Method首选项页面中配置。
    6. 如果服务器使用自定义端口，请启用“ 使用端口”并输入端口号。
  6. 单击完成 。存储库位置已创建。将远程创建RemoteSystemsConnections项目，并且将打开“同步”视图，并允许您将其资源提交到服务器。如果项目已远程存在，则“同步”视图将在本地和远程存在的任何文件上显示冲突。如果 RemoteSystemsConnections 项目在您第一次共享时已存在于存储库中，则可能存在许多冲突。通常，这些不是真正的冲突，而只是在第一次同步时考虑冲突，因为工作区中的资源没有本地CVS信息。如果您尚未在团队配置文件中创建任何资源，则可以安全地发送或接收所有冲突文件。如果您已在团队配置文件中创建了资源，则应检查每个冲突的文件以确保确实没有差异，如果有，请通过合并解决它们。
  7. 要将资源与关联存储库中的资源同步，请右键单击RemoteSystemsConnection项目，然后选择Team > Synchronize with Repository 。
  8. 共享或同步，并从存储库接收传入资源后，右键单击RemoteSystemsConnection项目，然后选择“ 重新加载远程系统资源管理器” 。这将刷新Remote System Explorer和Team视图以反映新的或更改的资源。

在“导航器”视图中查看项目资源

“导航器”视图提供工作空间中所有资源的分层视图。从这里，您可以查看共享项目的内容，包括RemoteSystemsConnection项目，并选择特定资源上与团队相关的操作。默认情况下，Navigator视图包含在Resources透视图中。要将其添加到当前视角，
  1. 选择窗口 > 显示视图 > 其他 。
  2. 展开基本。
  3. 选择导航器。
