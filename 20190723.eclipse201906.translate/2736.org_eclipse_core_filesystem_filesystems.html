文件系统提供商

标识符：
org.eclipse.core.filesystem.filesystems

以来：
org.eclipse.core.filesystem 1.0

描述：
此扩展点用于提供抽象文件系统API org.eclipse.core.filesystem的实现。IFileSystem。文件系统实现控制在某些后备存储中查询和修改文件和目录树的所有方面。每个文件系统扩展都负责由特定URI方案标识的文件系统命名空间。

配置标记：

<！ELEMENT 扩展 （ 文件系统 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 文件系统 （ 运行 ）>

<！ATTLIST文件系统

方案CDATA #REQUIRED>

  • scheme - 这种文件系统的URI方案。

<！ELEMENT 运行 EMPTY>

<！ATTLIST跑

class CDATA #REQUIRED>

  • class - 继承org.eclipse.core.filesystem.provider的类的完全限定名。文件系统

例子：
以下是文件系统扩展的示例，该扩展使用“gopher”URI方案提供对文件系统的访问。 

<extension id = “gopherFS” name = “Gopher File System” point = “org.eclipse.core.filesystem.filesystems” > <filesystem scheme = “gopher” > <run class = “com.xyz.fs.GopherFileSystem“ /> </ filesystem> </ extension>
如果此扩展名是在ID为“com.xyz.coolplugin”的插件中定义的，则此文件系统扩展名的完全限定名称为“com.xyz.coolplugin.gopherFS”。

API信息：
class属性的值必须表示org.eclipse.core.filesystem.provider的子类。文件系统 。

可以通过org.eclipse.core.filesystem方法获取文件系统实例。FileSystemCore.getFileSystem（String） 。

提供的实施：
该平台为“文件”方案提供文件系统实现。
