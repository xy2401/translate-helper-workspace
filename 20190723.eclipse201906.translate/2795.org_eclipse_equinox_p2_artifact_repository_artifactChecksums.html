Artifact Checksum支持

标识符：
org.eclipse.equinox.p2.artifact.repository.artifactChecksums

以来：
2.4.0

描述：
提供校验和算法id和java.security.的映射java.security.MessageDigest实现。

p2通过使用MD5散列函数计算伪像校验和来检测伪像传输/存储期间可能引入的错误。由于碰撞攻击和针对MD5的选择前缀冲突攻击已经在公开场合得到证明，它可以用于检测无意识的腐败。当前实现不可扩展，并且不允许使用其他更具抗冲突性的哈希函数。

有了这个扩展点，任何java.security.来自Java Security API的MessageDigest可用于检查工件的完整性。有关这些API的更多信息，请参阅Java Cryptography Architecture（JCA）参考指南 。

配置标记：

<！ELEMENT 扩展 （ artifactChecksum +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT artifactChecksum EMPTY>

<！ATTLIST artifactChecksum

id CDATA #REQUIRED

算法CDATA #REQUIRED>

  • id - 校验和算法的id，即sha-512 。由于其他工具将依赖于此ID，请考虑使用一些明确定义的值（即散列函数文本名称 ）。
  • algorithm - java.security.所期望的消息摘要算法的名称java.security.MessageDigest的方法getInstance(String) 。另请查看Java Cryptography体系结构标准算法名称文档中的MessageDigest部分，以获取有关标准算法名称的信息。

例子：
要启用由静态注册的提供程序提供的MessageDigest实现的支持，贡献包应该只注册扩展：
<extension point = “org.eclipse.equinox.p2.artifact.repository.artifactChecksums” > <artifactChecksum algorithm = “SHA3-512” id = “sha3-512” > </ artifactChecksum> </ extension>
如果MessageDigest实现由自定义提供程序（来自贡献包本身或其他一些包）提供，则应首先动态注册：
import java.security。安全; import org.bouncycastle.jce.provider。BouncyCastleProvider; public class Activator实现BundleActivator {public void start（BundleContext context）throws Exception {Security.addProvider（new BouncyCastleProvider（））; }}

提供的实施：
使用id sha-256提供校验和算法，并将其映射到SHA-256 MessageDigest实现。
