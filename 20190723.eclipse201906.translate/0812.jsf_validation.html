验证JSF应用程序


有效的JSF应用程序具有有效的Faces配置和JSP页面，这些页面在语法和语义上都是正确的。JSF工具在WTP工具的支持下，将尽可能在设计时验证应用程序是否可以在运行时执行。

验证应用程序

从资源管理器中的应用程序节点中，选择“验证”。此操作将使用为“手动”启用的所有验证程序验证项目中的所有资源。从JSF应用程序的角度来看，这将验证以下内容：

  • 面对配置文件
  • JSP文件

要访问工作区的验证器： 窗口 - >首选项...->验证

标记为黄色的验证器是那些适用于JSF Web应用程序的验证器。


验证面部配置


此时，当验证了faces配置时，它仅验证XML在语法上是否正确以及数据是否符合faces-configuration模式。未来版本将在单个面配置文件中执行语义检查，并使用应用程序在所有面配置文件中执行语义检查。


验证JSF JSP页面 - JSF视图验证器


JSF View Validator将验证JSP页面中为某些标记提供的属性值是否有效。此功能依赖于插件提供程序为标记库提供的其他元数据和类。JSF Tooling团队为以下方面提供了验证支持：

  • JSF核心标记库
  • JSF HTML标记库

出现问题时，标尺中会出现注释标记，问题视图中会出现问题标记，而源代码中的问题会出现“波浪形”。错误是红色的，必须处理，而黄色警告可能不会导致运行时问题，有时可能会被忽略。

验证器将使用和不使用表达式语言（EL）验证属性值。这些验证的处理方式不同。JSF EL由属性值中的“＃{some expression}”标识。

如果启用了JSF View Validator，则验证不仅会在Build和/或Manual验证时进行，还会按照您的类型进行。由于EL的验证可能“昂贵”，您可以选择单独关闭此选项。

要访问EL Validator首选项： Window - > Preferences ... - > Web and XML - > JavaServer Faces Tools - > Validation
