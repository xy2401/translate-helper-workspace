团队装饰者

标识符：
org.eclipse.team.ui.teamDecorators

以来：
3.2

描述：
此扩展点用于将存储库提供程序ID与提供团队状态修饰的装饰器ID相关联。模型提供程序使用它来确定何时需要对其模型元素进行标签更新。

配置标记：

<！ELEMENT 扩展 （ teamDecorator ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT teamDecorator EMPTY>

<！ATTLIST团队指导员

repositoryId IDREF #REQUIRED

decoratorId IDREF #REQUIRED

decoratedDirection CDATA #IMPLIED>

将存储库提供程序与装饰器关联


  • repositoryId - org.eclipse.team.core.repository扩展的标识，用于标识与装饰器关联的存储库提供程序。
  • decoratorId - org.eclipse.ui.decorators扩展的id，用于标识与存储库提供程序关联的装饰器。
  • decoratedDirection - 装饰方向状态的逗号分隔字符串。有效的状态标志是INCOMING和OUTGOING。如果省略decorateDirection字段，则假定bith方向。

例子：
以下是teamDecorators扩展示例

   <extension
         point="org.eclipse.team.ui.teamDecorators">
      <teamDecorator
            decoratorId="org.eclipse.team.cvs.ui.decorator"
            repositoryId="org.eclipse.team.cvs.core.cvsnature"/>
   </extension>

API信息：
repositoryId必须对应于使用org.eclipse.team.core.repository扩展点注册的提供程序的id。类似地，decoratorId必须对应于使用org.eclipse.ui.decorators扩展点注册的装饰器。

提供的实施：
没有与此扩展点直接关联的实现。
