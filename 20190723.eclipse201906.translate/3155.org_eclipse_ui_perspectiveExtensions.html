透视扩展

标识符：
org.eclipse.ui.perspectiveExtensions

描述：
此扩展点用于扩展其他插件注册的透视图。透视图定义窗口操作栏（菜单和工具栏）的初始内容以及工作台页面中的初始视图集及其布局。其他插件可以为透视选择时出现的透视提供动作或视图。其他插件的可选添加内容将附加到初始定义中。

配置标记：

<！ELEMENT 扩展 （ perspectiveExtension *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT perspectiveExtension （ actionSet | viewShortcut | perspectiveShortcut | newWizardShortcut | view | showInPart | hiddenMenuItem | hiddenToolBarItem ）*>

<！ATTLIST perspectiveExtension

targetID IDREF #REQUIRED>

  • targetID - 作出贡献的透视图的唯一标识符（在注册表中指定）。如果该值设置为“*”，则扩展名将应用于所有透视图。

<！ELEMENT actionSet EMPTY>

<！ATTLIST actionSet

id IDREF #REQUIRED>

  • id - 将添加到透视图的操作集的唯一标识符。

<！ELEMENT viewShortcut EMPTY>

<！ATTLIST viewShortcut

id IDREF #REQUIRED>

  • id - 视图的唯一标识符，该标识符将添加到“Window”菜单的透视图“Show View”子菜单中。

<！ELEMENT perspectiveShortcut EMPTY>

<！ATTLIST perspectiveShortcut

id IDREF #REQUIRED>

  • id - 将添加到“Window”菜单的透视图“Open Perspective”子菜单中的透视的唯一标识符。

<！ELEMENT newWizardShortcut EMPTY>

<！ATTLIST newWizardShortcut

id IDREF #REQUIRED>

  • id - 新向导的唯一标识符，它将添加到“文件”菜单的透视图“新建”子菜单中。

<！ELEMENT showInPart EMPTY>

<！ATTLIST showInPart

id IDREF #IMPLIED>

  • id - 视图的唯一标识符，该标识符将添加到“导航”菜单中透视图的“显示在...”提示符中。

<！ELEMENT 视图 EMPTY>

<！ATTLIST观点

id IDREF #REQUIRED

相对IDREF #IMPLIED

关系（堆栈|左|右|上|下|快|）

比率CDATA #IMPLIED

可见（true | false）

closeable（true | false）

可移动的（真|假）

独立的（真|假）

showTitle（true | false）

最小化（true | false）“false”>

  • id - 将添加到透视布局的视图的唯一标识符。
  • relative - 透视图中已存在的视图的唯一标识符。这将用作放置视图的参考点。这两个视图之间的关系由relationship定义。如果关系“快”，则忽略。
  • relationship - 指定id和relative之间的关系。支持以下值：
      fast - 视图扩展名将创建为快速视图。
      stack - 视图扩展将与文件夹中的相对视图堆叠在一起。
      left，right，top，bottom - 视图扩展名将放在相对视图旁边。在这种情况下，还必须定义ratio 。
  • ratio - 相对视图中将捐赠给视图扩展的区域百分比。如果视图扩展名是快速视图，则比率是快速视图在活动时将覆盖的工作台的百分比。必须将其定义为浮点值，介于0.05和0.95之间。
  • visible - 打开透视图时视图是否最初可见。如果使用，此属性应具有值“true”或“false”。如果未使用此属性，则默认情况下该视图最初将可见。
  • closeable - 视图是否在目标透视图中可关闭。如果使用，此属性应具有值“true”或“false”。如果未使用此属性，则视图将是可关闭的，除非透视本身标记为已修复。
  • 可移动 - 视图是否可移动。不可移动的视图既不能在同一文件夹中移动，也不能在透视图中的文件夹之间移动。如果使用，此属性应具有值“true”或“false”。如果未使用此属性，则视图将是可移动的，除非透视本身标记为已修复。
  • standalone - 视图是否为独立视图。独立视图无法与同一文件夹中的其他视图停靠在一起。如果使用，此属性应具有值“true”或“false”。如果关系属性为“fast”或“stacked”，则忽略此属性。如果未使用此属性，则视图将是常规视图，而不是独立视图（默认为“false”）。
  • showTitle - 是否显示视图的标题。如果使用，此属性应具有值“true”或“false”。此属性仅适用于独立视图。如果未使用此属性，将显示视图的标题（默认为“true”）。
  • 最小化 - 如果透视扩展将导致创建新的视图堆栈（即'relationship'属性是left，right，top或bottom之一），则此字段确定新堆栈的初始显示状态。

<！ELEMENT hiddenMenuItem EMPTY>

<！ATTLIST hiddenMenuItem

id CDATA #REQUIRED>

  • id - 要从菜单中删除的Command的唯一标识符。 警告：这被认为是“产品级别”扩展，不应在未经过精心处理的情况下用于耗材插件。

<！ELEMENT hiddenToolBarItem EMPTY>

<！ATTLIST hiddenToolBarItem

id CDATA #REQUIRED>

  • id - 要从工具栏中删除的Command的唯一标识符。 警告：这被认为是“产品级别”扩展，不应在未经过精心处理的情况下用于耗材插件。

例子：
以下是透视扩展的示例（请注意子元素和使用的方式属性）：


    <extension point="org.eclipse.ui.perspectiveExtensions"> 
        <perspectiveExtension 
            targetID="org.eclipse.ui.resourcePerspective"> 
            <actionSet id="org.eclipse.jdt.ui.JavaActionSet"/> 
            <viewShortcut id="org.eclipse.jdt.ui.PackageExplorer"/> 
            <newWizardShortcut id="org.eclipse.jdt.ui.wizards.NewProjectCreationWizard"/> 
            <perspectiveShortcut id="org.eclipse.jdt.ui.JavaPerspective"/> 
            <view id="org.eclipse.jdt.ui.PackageExplorer" 
                relative="org.eclipse.ui.views.ResourceNavigator" 
                relationship="stack"/> 
            <view id="org.eclipse.jdt.ui.TypeHierarchy" 
                relative="org.eclipse.ui.views.ResourceNavigator" 
                relationship="left" 
                ratio="0.50"/> 
        </perspectiveExtension> 
    </extension>

在上面的示例中，动作集，视图快捷方式，新向导快捷方式和透视快捷方式将提供给Resource Perspective的初始内容。此外，Package Explorer视图堆叠在Resource Navigator上，Type Hierarchy View添加在Resource Navigator旁边。

API信息：
透视扩展中定义的项目将提供给目标透视图的初始内容。在此之后，用户可以从工作台用户界面中删除任何贡献或将其他贡献添加到透视图中。
