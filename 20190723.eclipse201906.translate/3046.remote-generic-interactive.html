远程通用交互式

本节介绍如何使用“远程通用交互”作为并行工具平台（PTP）的目标系统配置来设置运行配置。

此目标系统配置（TSC）是在远程系统上启动作业的简单方法，尤其是并行作业。

如运行并行程序中所述 ，通过运行>运行配置创建运行配置...并创建一个新的并行应用程序 。

使用PTP运行远程程序的细节在运行配置的“ 资源”选项卡上指定。对于Target System Configuration ，选择Remote-Generic-Interactive

然后指定一个连接。如果已经指定了连接（例如，对于同步项目或远程项目），则可以选择它，或者可以使用“ 新建...”按钮在此时创建新连接。

有关如何创建连接，请参阅创建远程连接和本地连接

注意：要在创建连接后编辑连接信息，例如更改用户名或密码，请参阅编辑远程连接 。

选择Connection后，它将要求在系统上运行命令。选择是继续。检查此配置的“ 请勿再次运行”命令，以避免再次询问此启动配置。命令结果中的信息通常用于填充对话框中的信息。

在“ 应用程序”选项卡上（如运行并行应用程序的运行配置设置中所述），对于Remote-Generic-Interactive目标系统配置，应用程序名称不应是可执行文件名，而是用户提供的运行脚本。

  • 在本例中，我们创建了一个包含“hello world”脚本的简单同步项目
    #!/bin/bash
# hello.sh
echo "Hello PTP Remote Generic Interactive World"
使用“ 浏览...”按钮从远程目标系统上的某个位置选择脚本。

选择“ 运行”按钮以执行脚本。这个简单的“hello world”脚本的结果显示在工作台窗口底部的Console视图中。
