Trac连接器

服务器要求是什么？

Trac连接器支持两种不同的访问方法：推荐的XML-RPC模式提供与Mylyn的完全集成，但需要所有用户可能无法使用的其他设置和权限。Web模式提供的功能较少，但可以与任何公共Trac存储库一起使用。

XML-RPC（推荐）

要求：

  • Trac 0.10或更高版本
  • XmlRpcPlugin rev。 1950年或以后

此访问方法提供丰富编辑器中的任务编辑，附件支持和离线编辑。它需要为访问的存储库启用Trac的XmlRpcPlugin 。XmlRpcPlugin提供了Trac存储库的远程接口，并与Trac分开分发（参见＃217 ）。有关要求和文档，请参阅这些安装说明 。

为了通过XML_RPC访问存储库，每个用户都需要具有相应的权限，可以通过Trac的管理工具进行配置：

trac-admin </path/to/projenv> permission add <user> XML_RPC

卷筒纸

要求：

  • Trac 0.9或更高版本

在此模式下，标准Trac Web界面用于存储库访问。可以通过Web浏览器创建和编辑故障单。不支持脱机编辑和附件。

推荐的Trac版本

Mylyn可以使用版本0.9或更高版本的任何稳定的Trac版本（请参阅下面的已知限制）。当与Trac的XML-RPC插件（见上文）一起使用时，Mylyn效果最佳，但我们不建议使用特定的Trac版本。

Trac连接器测试针对这些Trac版本运行：

  • Trac 0.11.6，XML-RPC插件1.0.6（r7194）
  • Trac 0.10.5，XML-RPC插件0.0.2（r2125）
  • Trac 0.9.6（支持以Mylyn 3.3结束）

Mylyn是否支持Trac 0.11？

当Mylyn 3.2.1或更高版本与SVN主干的最新版Trac XmlRpcPlugin一起使用时 ，Mylyn Trac连接器功能齐全（ bug 175211 。

创建包含非ASCII字符的故障单时，为什么会出现HTTP Error 500内部服务器错误？

当使用Python 2.3运行Trac时，已经报告了与字符编码相关的问题。升级到Python 2.4可能有助于解决此问题。如果在创建包含非ASCII字符的故障单时遇到内部服务器错误，请评论错误＃188363 。

已知限制

  • Trac 0.10.1
    • 已知与Trac XML-RPC插件不兼容。有关详细信息，请参阅错误164272 。
  • Trac 0.10.3
    • 已知与旧版Trac HttpAuthPlugin不兼容（在修订版1890或更高版本中已修复）。当使用Trac AccountManagerPlugin进行基于表单的身份验证时，该插件为XML-RPC启用基本身份验证。有关详细信息，请参阅错误168413 。
  • Trac 0.11
    • 已知与XML-RPC插件版本1.0.2 - 1.0.5不兼容。建议使用1.0.6或更高版本。

为什么在Web浏览器中而不是在富编辑器中打开任务？

请确保将存储库属性中的访问类型设置为XML-RPC。这需要Trac 0.10和XML-RPC（见上文）。

Mylyn在Trac存储库中访问哪些URL？

XML-RPC：

  • 预期的URL是/ xmlrpc或/ login / xmlrpc（如果提供了登录凭据）

网址：

  • 身份验证：/ login
  • 查询：/查询？格式=标签...
  • 同步票证详细信息：/ ticket / ...
  • 获取存储库配置以填充查询对话框：/ query或/ newticket

Web模式依赖于屏幕抓取，如果Trac存储库的设计（即HTML输出）被大量定制，则可能会失败。

在Linux上打开Web编辑器时出现问题

如果您在Linux上打开Web任务编辑器时出现问题，并且错误日志中出现消息“ 无法创建浏览器”页面：XPCOM错误-2147221164 ，请尝试在Linux发行版上安装软件包xulrunner和xulrunner-gnome-support 。

Mylyn支持哪些Trac插件？

  • XML-RPC插件，用于丰富编辑和附件支持
  • 用于基于表单的身份验证的帐户管理器插件 （Mylyn 2.0或更高版本）
  • 用于子任务支持的Master Tickets插件 （Mylyn 2.3或更高版本）
