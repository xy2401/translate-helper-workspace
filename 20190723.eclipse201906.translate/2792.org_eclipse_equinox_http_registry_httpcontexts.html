Http Service Httpcontexts

标识符：
org.eclipse.equinox.http.registry.httpcontexts

以来：
1.0

描述：
提供由id标识的OSGi Http Service HttpContext。

配置标记：

<！ELEMENT 扩展 （ httpcontext +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT httpcontext （ resource-mapping *， mime-mapping *）>

<！ATTLIST httpcontext

id CDATA #REQUIRED

class CDATA #IMPLIED>

  • id - 与此httpcontext关联的标识符。在此捆绑包之外，必须通过其全名来引用它，包括bundle命名空间。
  • class - 与registerServlet和registerResource中的OSGi Http Services“httpcontext”等效。HttpContext由Extension Point提供程序实例化，并且必须是org.osgi.service.http的实例。HttpContext的。

<！ELEMENT mime-mapping EMPTY>

<！ATTLIST mime-mapping

扩展CDATA #REQUIRED

mime-type CDATA #REQUIRED>

类似于Servlet规范及其部署描述符的mime-mapping元素。


  • extension - 要关联的文件扩展名。例如：ps
  • mime-type - 相关的哑剧类型。例如：application / postscript

<！ELEMENT 资源映射 EMPTY>

<！ATTLIST资源映射

路径CDATA #IMPLIED

捆绑CDATA #IMPLIED>

  • path - 可以从中加载资源的包路径。
  • bundle - 从中加载资源的包。如果“bundle”属性不存在，则使用贡献包。

例子：
以下是httpcontexts扩展的示例：
<extension id = “testHttpcontext” point = “org.eclipse.equinox.http.registry.httpcontexts” > <httpcontext id = “testintest” > <resource-mapping path = “/ test” /> </ httpcontext> </ extension >
这个httpcontexts扩展将通过OSGi Http Service的HttpContext getResource方法从贡献包中的“/ test”创建资源。

API信息：
httpcontext类必须是org.osgi.service.http的实例。HttpContext的。
