资源标记

标识符：
org.eclipse.core.resources.markers

描述：
工作空间支持任意资源上的标记概念。标记是一种元数据（类似于属性），可用于使用用户信息标记资源。每当工作区保存或快照完成时，标记可选地由工作空间保留。

用户可以定义和查询给定类型的标记。标记类型在支持多继承的层次结构中定义。标记类型定义还指定了必须或可能存在于该类型的标记上的数字属性以及是否应该保留该类型的标记。

标记扩展点允许标记编写者在符号名称下注册其标记类型，然后在工作区内使用它来创建和查询标记。符号名称是标记扩展名的ID。在定义标记扩展时，鼓励用户为“name”属性包含人类可读的值，该属性标识其标记并可能呈现给用户。

配置标记：

<！ELEMENT 扩展 （ 超级 *， 持久性 ？， 属性 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #REQUIRED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的必需标识符
  • name - 扩展实例的可选名称

<！ELEMENT super EMPTY>

<！ATTLIST超级

输入IDREF #REQUIRED>

  • type - 标记超类型的完全限定id（即，由另一个标记扩展定义的标记类型）

<！ELEMENT persistent EMPTY>

<！ATTLIST持续存在

value（true | false）>

  • value - “ true ”或“ false ”，指示工作区是否应保留此类型的标记。如果未指定持久性特性，标记类型不持续。

<！ELEMENT 属性 EMPTY>

<！ATTLIST属性

名称CDATA #REQUIRED>

  • name - 可能出现在此类标记上的属性的名称

例子：
以下是标记配置的示例：

<extension id = “com.xyz.coolMarker” point = “org.eclipse.core.resources.markers” name = “Cool Marker” > <persistent value = “true” /> <super type = “org.eclipse.core .resources.problemmarker“ /> <super type = ”org.eclipse.core.resources.textmarker“ /> <attribute name = ”owner“ /> </ extension>

API信息：
所有标记，无论其类型如何，都是org.eclipse.core.resources的实例。IMarker 。

提供的实施：
平台本身具有许多预定义的标记类型。特定产品安装可根据需要包括额外的标记。
