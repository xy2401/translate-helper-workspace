什么是QVT ？

查询/查看/转换语言是OMG （对象管理组）指定的模型转换语言。或者更确切地说，它是Eclipse QVTd添加五种中间语言的三种指定语言。

  • QVTo - Operation Mappings是Eclipse QVTo项目支持的命令式转换语言。

  • QVTr - 关系是一种丰富的声明性转换语言。

  • QVTc - Core是一种简单的核心声明转换语言。

QVTc和QVTr，通常称为QVTd，支持Eclipse QVTd项目 （QVT声明）并在此处记录。

Eclipse QVTd支持涉及从QVTr到QVTc到可执行形式的转换链。这个链中的各个阶段是：

  • QVTu - 单向声明性转换语言 - QVTc的简化。

  • QVTm - 一种最小的声明性转换语言 - QVTc的进一步简化。

  • QVTp - 分区声明性转换语言 - QVTc的进一步简化。

  • QVT - 适用于计划确定的图形声明性转换语言。

  • QVTi - 一种可执行的命令式转换语言 - QVTc的一种变体。

  • Java - 最终的可执行形式

注意，QVTu，QVTm，QVTp，QVT和QVTi不是由当前的OMG规范定义的。它们可能有助于未来的规范。

建模图层

  • EMF是建模结构

Eclipse Modeling Framework（ EMF ）支持结构元模型的定义以及随后使用符合这些元模型的模型。EMF还支持生成Java代码以表示元模型。可以提供附加的Java代码以向结构元模型添加行为。

  • OCL是EMF和建模行为

OCL提供了一种建模语言，允许将行为嵌入到结构元模型中，或者作为这些元模型的补充提供。作为一种建模语言，OCL理解模型，因此OCL代码比同等Java更紧凑。可以静态检查OCL代码，而相应的Java代码通常使用反射，因此无法检查。

  • QVT是建模模型执行

Eclipse QVTd是OMG QVT 1.3规范的一个实现，用于Ecore和UML元模型。

Eclipse QVTd利用了Eclipse OCL Pivot模型的可扩展性。
