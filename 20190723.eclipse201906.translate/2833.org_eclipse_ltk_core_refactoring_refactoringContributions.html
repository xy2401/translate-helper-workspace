重构贡献

标识符：
org.eclipse.ltk.core.refactoring.refactoringContributions

以来：
3.2

描述：
扩展点用于定义重构贡献。重构贡献提供了一种基于RefactoringDescriptor动态构建可立即执行的重构实例的方法。

配置标记：

<！ELEMENT 扩展 （ 贡献 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 贡献 EMPTY>

<！ATTLIST的贡献

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 提供的重构类型的全局唯一ID。id通常是一个完全限定的标识符，由贡献插件的名称和重构的唯一标识符组成，例如org.eclipse.ltk.core.renameResource。
  • class - 扩展org.eclipse.ltk.core.refactoring.的重构贡献实现的完全限定名称org.eclipse.ltk.core.refactoring.RefactoringContribution 。

例子：
下面的示例定义了RenameResourceRefactoringContribution类为重命名资源重构实现的重构贡献，该重构具有唯一ID org.eclipse.ltk.core.rename.resource ：
<extension point = “org.eclipse.ltk.core.refactoring.refactoringContributions” > <contribution class = “org.eclipse.ltk.core.refactoring。RenameResourceRefactoringContribution“ id = ”org.eclipse.ltk.core.rename.resource“ /> </ extension>

API信息：
贡献的实现类必须扩展org.eclipse.ltk.core.refactoring.RefactoringContribution
