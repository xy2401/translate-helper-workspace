发射台

不推荐使用此扩展点

标识符：
org.eclipse.debug.core.launchers

描述：
此扩展点已由launchConfigurationTypes扩展点替换。此版本的扩展在2.0版本中已过时，因此被忽略。此扩展点用于贡献发射器。启动器负责启动调试会话或运行程序并向启动管理器注册结果。

配置标记：

<！ELEMENT 扩展 （ 启动器 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 启动器 EMPTY>

<！ATTLIST发射器

id CDATA #REQUIRED

class CDATA #REQUIRED

模式CDATA #REQUIRED

标签CDATA #REQUIRED

向导CDATA #IMPLIED

public（true | false）

描述CDATA #IMPLIED

透视CDATA #IMPLIED

icon CDATA #IMPLIED>

  • id - 可用于引用此启动程序的唯一标识符。
  • class - 实现org.eclipse.debug.core.model.的Java类的完全限定名称org.eclipse.debug.core.model.ILauncherDelegate 。
  • modes - 此启动器支持的逗号分隔模式列表。两种支持的模式是“运行”和“调试” - 如org.eclipse.debug.core.所定义org.eclipse.debug.core.ILaunchManager 。发射器可能能够以一种或两种模式发射。
  • label - 用于启动器的标签。调试UI使用此属性。
  • wizard - 实现org.eclipse.debug.ui.的类的完全限定名称org.eclipse.debug.ui.ILaunchWizard 。调试UI使用此属性。启动器可以提供允许用户配置和启动特定属性的向导。
  • public - 是否在调试UI中公开显示启动器。如果为true ，则启动器将在调试UI中可用 - 启动器将显示为默认启动器的选项，此启动器创建的启动将显示在启动历史记录中，启动器将从下拉运行中获得/调试工具栏动作。
  • description - 启动器的描述。目前仅在指定了向导属性时使用。
  • perspective - 成功启动时将切换到的透视图的标识符。默认值是调试透视图的标识符。调试UI使用此属性。
  • icon - 图标的相对路径，如果指定，将表示UI中的启动器。

例子：
以下是启动器扩展点的示例：

<extension point = “org.eclipse.debug.core.launchers” > <launcher id = “com.example。ExampleLauncher“ class = ”com.example.launchers。ExampleLauncher“ modes = ”run，debug“ label = ”示例Launcher“ wizard = ”com.example.launchers.ui。ExampleLaunchWizard“ public = ”true“ description = ”启动示例程序“ perspective = ”com.example。JavaPerspective“ > </ launcher> </ extension>
在上面的示例中，指定的启动程序支持运行和调试模式。成功启动后，调试UI将更改为Java透视图。当调试UI向用户显示可供选择的启动器列表时，“示例启动器”将作为选项之一出现，其中“启动示例程序”作为描述，并且com.example.launchers.ui指定了向导。 。ExampleLaunchWizard将用于配置任何启动特定的详细信息。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core的Java类的完全限定类名。ILauncherDelegate 。属性向导的值必须是实现org.eclipse.debug.ui的Java类的完全限定类名。ILaunchWizard 。
