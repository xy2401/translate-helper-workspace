测试

JGit单元测试

JGit单元测试在maven构建期间执行。要从Eclipse工作台运行它们，请使用作为测试包源代码的一部分的启动配置。

JGit HTTP测试

org.eclipse.jgit.http.test中的JGit HTTP测试依赖于Jetty Web容器。

要从Eclipse运行这些测试，需要Jetty功能。使用依赖项中描述的目标平台之一。

或者，从http://download.eclipse.org/jetty/updates/jetty-bundles-9.x/9.2.10.v20150310/安装“Jetty 9.2.10.v20150310”。

EGit核心测试

EGit Core测试在包org.eclipse.egit.core.test的maven构建期间执行。

要从Eclipse工作台运行它们，请使用启动配置，该配置是测试包org.eclipse.egit.core.test的源代码的一部分。

EGit UI测试

EGit UI测试使用SWTBot，使用'SWTBot for Eclipse Testing'功能。

您需要至少安装“SWTBot for Eclipse Testing”和“SWTBot IDE Feature”：

  • http://download.eclipse.org/technology/swtbot/snapshots/

从Eclipse启动UI测试：

  • 选择测试类或测试方法
  • 单击Run As> SWTBot Test

在UI测试运行时不要触摸鼠标或键盘，因为这可能会通过例如将当前焦点移动到另一个窗口来干扰UI测试。

在Maven Build期间

测试在默认Maven生命周期的集成测试阶段执行。

如果要跳过UI测试的执行（仅执行核心测试）：

mvn -P skip-ui-tests clean install

如果您想跳过所有测试：

mvn clean install -Dmaven.test.skip=true

辅助测试工具

任何代码（包括测试代码）并不总能达到您的预期。最常见的故障可能是未能真正执行您要测试的部分代码。像EclEmma这样的代码覆盖工具可以轻松地查看正在执行的代码的哪个部分。
