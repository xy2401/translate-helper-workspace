组

描述

该组是可以在多个页面之间共享的用户界面的可重用部分。它可以包含各种小部件及其验证规则和特定布局。

属性

  • identifier：组的标识符用于标识组。
  • labelExpression：标签表达式用于计算组的标签。该表达式必须返回一个字符串。
  • domainClass：域类用于确保只有在根据包含该组的页面的语义候选表达式计算的页面的当前上下文与域类匹配时才会显示该组。
  • semanticCandidateExpression：语义候选表达式用于以与页面的语义候选表达式类似的方式计算组的当前上下文。此表达式必须返回一个对象或一组对象，以用作该组的新上下文。下面提供了更多信息，以便从一个定义创建多个组。
  • preconditionExpression：前置条件表达式用于确定是否应创建组。该表达式必须返回一个布尔值。
  • controls：此组中使用的所有窗口小部件和容器的列表。
  • semanticValidationRules：组的验证规则。有关EEF中验证的更多详细信息，请查看验证文档。
  • propertyValidationRules：组的小部件的验证规则。有关EEF中验证的更多详细信息，请查看验证文档。
  • style：组的样式用于修改用户界面中组的外观。
  • conditionalStyles：条件样式是满足特定条件时可以使用的所有备用外观的列表。
  • actions：用于在页面工具栏中创建按钮的工具栏操作列表。这些动作可以包含一个tooltipExpression，它应该返回一个用作工具提示的字符串，一个用户点击按钮时执行的actionExpression和一个用于配置按钮外观的imageExpression。imageExpression必须返回具有以下结构PROJECT_NAME / IMAGE_PATH的字符串，例如：org.eclipse.sample / icons / Icon.gif

变量

语义候选表达式可以访问两个不同的变量，从页面的语义候选表达式self计算并input视图的输入。使用这些变量，组的语义候选表达式将计算一个新值，该值将以名称self的形式提供给组中的所有其他表达式。如果您使用了组的语义候选表达式的其他值而不是包含页面中可用的值，则无法直接访问页面的self值。您可以从组中的self值计算出来。如果您没有指定任何语义候选表达式，它将重用包含页面的self值，就像页面使用视图中的self值一样。

标签表达式，前提条件表达式，控件，样式和验证规则都可以访问从组的语义候选表达式和self （视图的输入）计算的input值。

从一个定义创建多个组

以与页面中可以完成的方式类似的方式，组的单个定义可以在用户界面中创建多个组。为此，您只需使用将返回集合的语义候选表达式。有关此机制的其他详细信息，请查看该页面的文档。

样式和条件样式

Eclipse Sirius对EEF影响的另一个方面是引入样式。样式允许您轻松自定义用户界面元素（此处为组）的外观。EEF组样式包含以下属性：

  • backgroundColorExpression：背景颜色表达式用于计算组的背景颜色。此表达式必须返回基于十六进制的颜色的字符串，例如#FFFFFF ，或基于rgb的颜色，例如rgb(30,30,30) 。
  • foregroundColorExpression：前景色表达式用于计算组标签的颜色。此表达式必须返回基于十六进制的颜色的字符串，例如#FFFFFF ，或基于rgb的颜色，例如rgb(30,30,30) 。
  • fontNameExpression：字体名称表达式用于计算组标签使用的字体名称。此表达式必须返回带有字体名称的字符串。
  • fontSizeExpression：字体大小表达式用于计算组标签字体的大小。该表达式必须返回一个整数。
  • barStyle：组的栏的样式。此样式可以是常规标题栏，简短标题栏或根本没有标题栏。
  • toggleStyle：组的切换样式。这可以是twistie，tree node或者根本没有。
  • expandedByDefault：指示默认情况下是应该展开还是折叠组。

除常规组样式外，您还可以访问条件样式。条件样式是需要前提条件才能应用的样式。例如，只有在数据中存在问题时才应用的红色样式。所有这些表达式都可以访问变量self ，由组的语义候选表达式计算， input ，视图的输入。
