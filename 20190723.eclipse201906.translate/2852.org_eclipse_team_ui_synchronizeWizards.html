同步参与者创建向导

标识符：
org.eclipse.team.ui.synchronizeWizards

以来：
3.0

描述：
此扩展点用于注册同步参与者创建向导。这些向导用于创建将出现在“同步视图”中的同步参与者。提供者通常将创建创建向导以允许用户执行和管理特定类型的同步参与者。提供者可以为此扩展点提供扩展，以及org.eclipse.jface.wizard的实现。IWizard 。

配置标记：

<！ELEMENT 扩展 （ 向导 ？）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 向导 EMPTY>

<！ATTLIST向导

名称CDATA #REQUIRED

说明CDATA #REQUIRED

icon CDATA #REQUIRED

class CDATA #REQUIRED

id CDATA #REQUIRED>

  • name - 同步参与者创建类型的名称。例如“CVS”，“CVS Merge”，“WebDAV”。
  • description - 创建向导的描述。
  • icon - 向用户显示此向导类型时显示的图标。
  • class - 实现org.eclipse.jface.wizard的Java类的完全限定名称。IWizard 。
  • id - 此扩展的唯一标识符。

例子：
以下是同步参与者创建向导扩展的示例：

<extension point = “org.eclipse.team.ui.synchronizeWizards” > <wizard name = “WebDAV” description = “创建WebDAV参与者以查看工作区资源与其远程WebDAV位置之间的更改” icon = “webdav.gif” 类= “com.xyz。DAVWizard“ id = ”com.xyz.dav.synchronizeWizard“ > </ wizard> </ extension>

API信息：
class属性的值必须表示实现org.eclipse.jface.wizard的类。IWizard 。

提供的实施：
插件org.eclipse.team.cvs.ui包含synchronizeWizards扩展点的示例定义。
