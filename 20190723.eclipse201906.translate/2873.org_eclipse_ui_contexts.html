上下文

标识符：
org.eclipse.ui.contexts

以来：
3.0

描述：

org.eclipse.ui.contexts扩展点用于声明上下文和关联元素。

配置标记：

<！ELEMENT 扩展 （ 上下文 *）>

<！ATTLIST扩展

id CDATA #IMPLIED

名称CDATA #IMPLIED

点CDATA #REQUIRED>

  • id - 扩展实例的可选标识符。
  • name - 扩展实例的可选名称。
  • point - 目标扩展点的标准标识符。

<！ELEMENT context EMPTY>

<！ATTLIST背景

描述CDATA #IMPLIED

id CDATA #REQUIRED

名称CDATA #REQUIRED

parentId IDREF #IMPLIED>

此元素用于定义上下文。如果这些元素中不止一个具有相同的id属性，则只有最后声明的元素（按读取注册表的顺序）才被视为有效。


  • description - 用于在UI中显示的此上下文的可翻译简短描述。
  • id - 此上下文的唯一标识符。
  • name - 要在UI中显示的此上下文的可翻译名称。
  • parentId - 此上下文的父级的唯一标识符。

例子：

org.eclipse.ui插件中的plugin.xml文件使用org.eclipse.ui.contexts扩展点。

API信息：

通常，最好静态地声明所有上下文（在plugin.xml ）。这样用户就可以在密钥绑定中使用它们。但是，可以在运行时声明上下文。为此，请检索org.eclipse.ui.commands.来自工作台组件的IContextService ，调用getContext(contextId)然后调用Context#define(...) 。如果已卸载，则必须通过插件清除以编程方式定义的上下文。
