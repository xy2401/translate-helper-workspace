具有对象团队功能的Java编辑器

扩展的Java编辑器提供了用于编辑OT / J代码的专用功能。原始Java编辑器已扩展为以下功能：
  • 对象团队元素的导航支持（F3 = 开放声明 ，见下文）
  • OT / J关键字和修饰符的语法高亮显示 。
    根据正在编辑的文件的内容启用OT / J关键字。这样，相同的编辑器仍然可以处理普通Java源代码而无需解释OT / J关键字。
    但是，在OT / J文件中，语法突出显示是以无上下文的方式确定的，这意味着作用域关键字（ A.0.1 ）像after, base, before, get, result, set被突出显示，即使在它们没有特殊含义的情况下也是如此。
  • 对象团队程序元素的内容/代码辅助 （代码完成） 。
  • 对象的模板（源模式）使用内容辅助（ Ctrl + Space ）构建特定语言结构，例如within （ Expression ） 语句中
  • OT / J语法的代码格式 。
  • 垂直标尺中的新绑定标记注释（对于使用playingBy / callin / callout绑定的基类/方法），允许导航到影响角色。

导航

普通F3导航的明显增强功能如下：
  • 使用playedBy子句转到角色的基类。
  • 使用callout或callin绑定转到绑定的基本方法或绑定的字段
    （ 结合标记支持相反的方向）。
浏览角色文件 （ 1.2.5 ）：
  • 在team package声明中击中F3会跳到封闭团队。
  • 要从团队导航到其角色文件，请Ctrl-O两次Ctrl-O使用快速大纲：
    将提供角色文件以供内联成员选择。
  • 如果建议使用javadoc @role标记列出当前团队的所有角色文件（ 1.2.5（d） ），这些标签中的角色名称也支持F3导航到相应的角色。
导航隐式角色层次结构 （ 1.3.1 ）：
  • 从超级团队覆盖相应角色的角色类使用覆盖标记进行修饰（ ）在垂直标尺中，它支持Open Super Implementation动作。
  • 从任何显式或隐式超级角色重写相应方法的角色方法也使用覆盖标记进行修饰（ ）支持导航到重写方法。
