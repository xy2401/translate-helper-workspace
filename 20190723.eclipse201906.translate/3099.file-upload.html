上传文件

本文介绍如何允许用户将文件从其本地文件系统上载到RAP服务器或其他HTTP服务器。这可以使用RAP FileUpload小部件， 拖放 API或FileDialog来完成 。

FileUpload小部件

FileUpload小部件看起来像一个按钮，但它会在单击时打开用户浏览器的文件选择器对话框（它包装了一个HTML <input type="file"/>元素）。当选择了一个或多个文件（由Selection事件指示）时，可以使用submit方法将这些文件发送到任何URL。

要在服务器上接收文件，RAP在bundle org.eclipse.rap.fileupload中提供服务。要使用此服务，请创建FileUploadHandler的实例，并从其方法getUploadUrl获取上载URL。这是一个上传文件并将其存储在临时目录中的最小示例：

final DiskFileUploadReceiver receiver = new DiskFileUploadReceiver（）; final FileUploadHandler uploadHandler = new FileUploadHandler（receiver）; uploadHandler.addUploadListener（new FileUploadListener（）{public void uploadProgress（FileUploadEvent event）{} public void uploadFailed（FileUploadEvent event）{} public void uploadFinished（FileUploadEvent event）{System.out.println（“Stored file：”+ receiver.getTargetFiles （）[0] .getAbsolutePath（））;}}）; final FileUpload fileUpload = new FileUpload（parent，SWT.NONE）; fileUpload.setText（“选择文件”）; fileUpload.addSelectionListener（new SelectionAdapter（）{@ Override public void widgetSelected（SelectionEvent e）{fileUpload.submit（uploadHandler.getUploadUrl（））;}}）;

要一次上传多个文件，请使用SWT.MULTI标志创建FileUpload 。这些文件无限期地存储在临时目录中，因此一旦不再需要它们，您应该小心删除它们。

文件拖放

现代浏览器允许使用拖放从文件系统中选择文件。在RAP中，此功能已集成到SWT拖放API中。要使控件成为文件放置和删除的目标，请使用ClientFileTransfer创建DropTarget的实例。drop事件 数据将包含一个ClientFile对象数组：

DropTarget dropTarget = new DropTarget（control，DND.DROP_MOVE）; dropTarget.setTransfer（new Transfer [] {ClientFileTransfer.getInstance（）}）; dropTarget.addDropListener（new DropTargetAdapter（）{@Override public void drop（DropTargetEvent event）{handleFileDrop（（ClientFile []）event.data）;}}）;

请注意，与普通拖动手势不同，拖动反馈（附加到鼠标光标的图标）由操作系统呈现，因此在RAP中可能看起来与行为不同。此外，即使用户持有修改键，丢弃操作也将始终移动。

ClientFile对象表示在客户端上选择的文件，但尚未上载。为此，您必须使用ClientFileUploader服务，并且与上面的FileUpload小部件一样 ，使用FileUploadHandler ：

private void handleFileDrop（ClientFile [] files）{final DiskFileUploadReceiver receiver = new DiskFileUploadReceiver（）; final FileUploadHandler uploadHandler = new FileUploadHandler（receiver）; uploadHandler.addUploadListener（new FileUploadListener（）{public void uploadProgress（FileUploadEvent event）{} public void uploadFailed（FileUploadEvent event）{} public void uploadFinished（FileUploadEvent event）{System.out.println（“Stored file：”+ receiver.getTargetFiles （）[0] .getAbsolutePath（））;}}）; RWT.getClient（）。getService（ClientFileUploader.class）.submit（uploadHandler.getUploadUrl（），files）; }

FileDialog的

这基本上是一体化解决方案。FileDialog提供与SWT兼容的API，允许用户选择一个或多个文件供应用程序打开。使用FileUpload （文件选取器）或文件拖放选择文件后，立即上载它们。该对话框显示上传进度（使用ServerPush ）和 - 除非取消 - 只能在所有上传完成后关闭。

FileDialog几乎可以像在SWT中一样在RAP中使用，但是存在一些差异：

  • FileDialog 不是 RWT包的一部分。由于它的附加依赖性，它包含在它自己的一个包中，名为org.eclipse.rap.filedialog 。
  • 与SWT FileDialog不同，RAP版本只能选择要打开的文件（即上传）。使用SWT.SAVE标志创建FileDialog无效。但是，支持SWT.MULTI 。
  • 如果未选择任何文件，则对话框将显示占位符消息。此消息可以本地化（请参阅本地化RAP消息 ）。相应的属性包含在文件RWTMessages.properties中 ，它们以RWT_FileDialog为前缀。

  • 与所有对话框一样 ，必须在JEE兼容模式下以不同方式打开FileDialog ：

    final FileDialog fileDialog = new FileDialog（parent，SWT.APPLICATION_MODAL）; // SWT兼容模式：System.out.println（“Stored file：”+ fileDialog.open（））; // JEE兼容模式：fileDialog.open（new DialogCallback（）{public void dialogClosed（int returnCode）{System.out.println（“Stored file：”+ fileDialog.getFileName（））;}}）;
