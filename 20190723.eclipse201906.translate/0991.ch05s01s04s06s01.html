5.1.4.6.1. Check Property (General Components)

Check Property

  • Use this action to check a specific property of a component.

  • You can check whether a value you enter for a given property matches the actual value for the property.

  • Every component has properties (attributes). These can include the text size, the size (width/height) of the component, the color etc.

  • You must first find out the name of the property you want to check, and then what value it should have. You can find out what properties and values a component has in the Object Mapping Mode (for Swing and SWT/RCP AUT's ). When you collect a component, its property information is displayed in the Properties View .

  • For HTML AUTs using a Webdriver you can use this action to test CSS attributes or any other property of an element. This is how the action works on such AUTs:

    • First the attributes of the element are searched, and if the property is found there, its value is returned.

    • If the property is not yet found, the element's CSS properties are searched.

    • If the property is still not found, the HTML element's properties are searched, using / as a separator.

    As an example, there are two ways to check the color of an HTML element: either by checking the property color, or the property style/color. Please note that the actual value returned is strongly browser-dependent, e.g. for color, Firefox would return something like rgb(255, 0, 0) and Chrome rgba(255, 0, 0, 0).

  • Parameters:

    • Property Name

    • Property Value

    • Operator

    • Timeout in Millisecs
