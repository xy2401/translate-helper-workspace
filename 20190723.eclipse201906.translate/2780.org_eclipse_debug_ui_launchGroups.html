启动组

标识符：
org.eclipse.debug.ui.launchGroups

以来：
2.1

描述：
此扩展点支持定义要在启动配置对话框中一起查看的一组启动配置，并支持启动历史记录（最近和最喜欢的启动）。

配置标记：

<！ELEMENT 扩展 （ launchGroup *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT launchGroup EMPTY>

<！ATTLIST launchGroup

id CDATA #REQUIRED

模式IDREF #REQUIRED

类别IDREF #IMPLIED

标签CDATA #REQUIRED

图片CDATA #REQUIRED

bannerImage CDATA #REQUIRED

公共CDATA #IMPLIED

标题CDATA #IMPLIED>

  • id - 指定此启动组的唯一标识符。
  • mode - 指定与该组关联的启动模式 - 即运行或调试。
  • category - 指定此组中的启动配置类别。未指定时，类别为null 。
  • label - 指定用于呈现此组的可翻译标签。
  • image - 指定用于在树，列表，制表符等中呈现此组的图像的插件相对路径。
  • bannerImage - 指定用于在向导或对话框横幅区域中呈现此组的图像的插件相对路径。
  • public - 指定此启动组是否为公共组，并且应在调试首选项中具有可见的启动历史记录选项卡。未指定时，隐含值为true 。
  • title - 打开此启动组时在启动向导中显示的标题 - 例如“选择或配置要调试的应用程序”（自3.1起）

例子：
以下是启动组扩展点的示例：


 <extension point="org.eclipse.debug.ui.launchGroups">
  <launchGroup
     id="com.example.ExampleLaunchGroupId"
     mode="run"
     label="Run"
     image="icons\run.png"
     bannerImage="icons\runBanner.png">
  </launchGroup>
 </extension>
在上面的示例中，启动组将包含所有启动配置，没有支持运行模式的类别。
