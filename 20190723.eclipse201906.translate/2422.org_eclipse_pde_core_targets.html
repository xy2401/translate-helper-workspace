目标

标识符：
org.eclipse.pde.core.targets

以来：
3.2

描述：
用于定义和注册目标定义的扩展点。PDE使用户能够选择和切换目标。PDE还提供编辑和扩展目标的功能。目标的主要内容位于.target文件中。除其他外，目标定义文件包含插件/功能内容，目标环境，执行环境和启动参数的描述。

应使用PDE提供的目标定义编辑器编辑目标定义文件。文件格式不是API，可能会在发行版之间进行更改。

配置标记：

<！ELEMENT 扩展 （ 目标 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 目标 （ 描述 ？）>

<！ATTLIST目标

id IDREF #REQUIRED

名称CDATA #REQUIRED

定义CDATA #REQUIRED>

  • id - 目标的唯一标识符
  • name - 目标的可读名称
  • definition - 贡献插件中目标定义文件的相对路径。

<！元素描述 （#PCDATA）>

人类可读的目标描述



例子：
以下是targets扩展的示例：
<extension point = “org.eclipse.pde.core.targets” > <target id = “org.eclipse.pde.core.rcpTarget” name = “Eclipse RCP” definition = “targets / rcp.target” /> </扩展>
在上面的示例中，目标的名称和标识在plugin.xml中声明，但其内容存储在位于贡献插件中的targets/rcp.target中。
.target文件必须通过File> New> Other ...> Plug-in Development> Target Definition创建，并且应使用PDE目标定义编辑器进行编辑。文件格式不是API，可能会在发行版之间进行更改。

API信息：
此扩展点不需要Java代码。

提供的实施：
org.eclipse.pde.core插件提供了两个扩展。
