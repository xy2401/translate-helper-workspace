自下而上的CXF Web服务Ant任务属性文件参考

cxf_bujava.properties文件用于在创建CXF Web服务时将数据传递到Web服务Ant任务。此文件显示可用选项。

参数

ScenarioType
选项： service
这是一个必需元素，它告诉Ant任务是创建Web服务还是Web服务客户端。
InitialSelection
输入工作空间中Java bean的工作空间相对URI。例如/dynamicWebProjectName/src/myBean.java
ListRuntimes
选项： true false
如果选择true ，则在运行Ant任务时将提供有效运行时选项列表。
清单服务器
选项： true false
如果选择true ，则在运行Ant任务时将提供有效服务器选项列表。
服务。RuntimeId
这是CXF Web服务运行时的ID。不应修改此参数。
服务。SERVERID
选项：
  • org.eclipse.jst.server.tomcat.x （其中x是60或70 ）
  • org.eclipse.jst.server.generic.jbossx （其中x是42或5 ）
这是目标服务器的ID。删除或注释掉此参数，以使Ant任务根据Dynamic Web项目的服务器设置选择正确的值。
ServiceProjectName
默认情况下，服务项目被赋予从中创建它的Java bean的名称。使用此参数可以使用其他名称创建服务项目。
ServiceEarProjectName
默认情况下，服务EAR项目被赋予从中创建它的Java bean的名称。使用此参数可以使用其他名称创建服务EAR项目。
GenClient
选项： true false
如果为true ，则将生成使用JAX-WS API的客户端主线。
GenServer
选项： true false
如果为true ，则将生成使用JAX-WS API的服务器主线。
GenWrapperFaultBeans
选项： true false
如果为true ，则将生成包装器和故障bean。
GenWSDL
选项： true false
如果为true则将生成WSDL文件。
GenXSDImports
选项： true false
如果将true模式输出到单独的文件，则将使用导入将它们加载到wsdl而不是内联它们。
UseSOAP12Binding
选项： true false
如果为true则生成的WSDL文件将包含SOAP 1.2绑定。
WsdlFileName
设置生成的WSDL文件的名称。
