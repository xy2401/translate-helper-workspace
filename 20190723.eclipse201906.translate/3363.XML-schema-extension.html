XML架构扩展

数据驱动的XML分析通过开发自己的分析和视图来增加Trace Compass，而无需编写任何代码。现在，外部插件可以扩展XSD架构以添加其分析扩展和解析器，同时利用Trace Compass XML分析框架。

扩展架构

想要将自己的元素添加到XSD架构的插件可以通过扩展额外元素并定义扩展基类型extraType的复杂类型来实现 。这些附加元素位于XSD的根级别，位于tmfxml元素下。以下示例显示了另一个callstack元素的XSD文件：

<？xml version =“1.0”encoding =“UTF-8”standalone =“no”？> <xs：schema xmlns：xs =“ http://www.w3.org/2001/XMLSchema ”attributeFormDefault =“unqualified”elementFormDefault =“qualified”>
<xs：element name =“callstack”substitutionGroup =“extra”type =“callstackType”/>
<xs：complexType name =“callstackType”> <xs：complexContent> <xs：extension base =“extraType”> [...类型定义...]</ xs：extension> </ xs：complexContent> </ xs：complexType>
</ XS：模式>

解析模式

添加扩展点

要宣传此架构扩展和解析器，必须为插件指定org.eclipse.tracecompass.tmf.analysis.xml.core.xsd扩展名。 

<extension point =“org.eclipse.tracecompass.tmf.analysis.xml.core.xsd”> <xsdfile file =“xsd_files / xmlCallstack.xsd”> </ xsdfile> <schemaParser class =“my.package。CallstackXmlSchemaParser“> </ schemaParser> </ extension>
