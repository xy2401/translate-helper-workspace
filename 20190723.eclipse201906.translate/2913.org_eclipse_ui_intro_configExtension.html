简介部件配置扩展

标识符：
org.eclipse.ui.intro.configExtension

以来：
3.0

描述：

此扩展点用于通过提供更多内容，其他StandbyContentParts或其他IntroUrl操作来扩展现有的介绍配置。

从3.2开始，此扩展点还用于提供演示主题，欢迎实现可以使用它来分隔演示文稿和内容。

配置标记：

<！ELEMENT 扩展 （ configExtension | standbyContentPart | action | theme ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT configExtension EMPTY>

<！ATTLIST configExtension

configId IDREF #REQUIRED

内容CDATA #REQUIRED>

定义介绍配置的扩展。如果通过定义锚点声明了可扩展性，则可以扩展介绍部件配置中的任何页面或组。


  • configId - 将扩展的介绍贡献的ID
  • content - 介绍内容文件。内容文件是一个XML文件，其中包含介绍（ 介绍内容文件格式规范）的细节 。内容文件在运行时由介绍框架解析。根据此文件中的设置，打开简介时会向用户显示一定数量的页面，组和链接。

<！ELEMENT standbyContentPart EMPTY>

<！ATTLIST standbyContentPart

id CDATA #REQUIRED

pluginId CDATA #REQUIRED

class CDATA #REQUIRED>

standbyContentPart注册。注册后，可以通过以下格式的introURL操作启动备用部件：

http：//org.eclipse.ui.intro/showStandby？partId =＆lt; standbyContentPart＆gt;

  • id - 标识此standbyContentPart的唯一ID。
  • pluginId - 包含“class”属性中定义的类的插件的名称。
  • class - 实现org.eclipse.ui.intro.config.的类的完全限定类名org.eclipse.ui.intro.config.IStandbyContentPart用于处理显示备用备用内容，例如备忘单。

<！ELEMENT 动作 EMPTY>

<！ATTLIST行动

名称CDATA #REQUIRED

替换CDATA #REQUIRED>

自定义简介URL操作注册。这可用于创建新的简介URL操作或对预定义操作的快捷方式。


  • name - 标识此操作的唯一名称。
  • 替换 - 替换简介URL中的操作名称的宏。

<！ELEMENT 主题 （ 属性 *）>

<！ATTLIST主题

id CDATA #REQUIRED

名称CDATA #REQUIRED

路径CDATA #REQUIRED

default（true | false）“false”

previewImage CDATA #IMPLIED

可扩展的（true | false）“false”>

  • id - 介绍主题的唯一标识符
  • name - 将在UI中呈现的简介主题的可翻译名称
  • path - 样式表和关联图形所在文件夹的插件相对路径。
  • default - 一个可选标志，指示此主题是否为默认主题。如果未明确选择主题，则将使用默认主题。
  • previewImage - 4x3比率的可选图像，用于在UI中预览此主题。建议尺寸为160x120。
  • 可伸缩 - 可伸缩主题仅使用相对字体。使用可缩放主题时，将创建工具栏项以通过设置包含元素的绝对大小来放大或缩小字体大小。

<！ELEMENT 属性 EMPTY>

<！ATTLIST财产

名称CDATA #REQUIRED

值CDATA #REQUIRED>

一个可选的主题属性，介绍配置器可以使用它来进一步自定义介绍演示。


  • name - 属性的唯一名称
  • 价值 - 财产的价值

例子：
以下是此扩展点的示例实现：

<extension point = “org.eclipse.ui.intro.configExtension” > <configExtension configId = “com.org.xyz.introConfig” content = “extensionContent.xml” /> <standbyContentPart id = “com.org.xyz.myStandbyPart “ class = ”com.org.xyz.internal。MyStandbyContent“ pluginId = ”com.org.xyz“ /> <action name = ”shortcutAction“ replaceces = ”http：//org.eclipse.ui.intro/showStandby？partId = com.org.xyz.myStandbyPart“ /> < action name = “customAction”replaceces = “runAction？pluginId = com.org.xyz＆安培;类= com.org.xyz。CustomAction＆amp; param1 = value1“ /> <theme id = ”com.example.intro.flowers“ name = ”A Nice Flowers Welcome“ path = ”$ nl $ / themes / flowers“ previewImage = ”$ nl $ / themes / flowers /preview.png“ > </ theme> </ extension>

API信息：
有关更多详细信息，请参阅org.eclipse.ui.intro.config API包的规范。

提供的实施：
有三个提供的实现：
  • org.eclipse.jdt，使用configExtension
  • org.eclipse.pde，使用configExtension
  • org.eclipse.platform，使用standbyContentPart
