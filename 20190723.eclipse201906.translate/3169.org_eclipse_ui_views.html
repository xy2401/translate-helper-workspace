的org.eclipse.ui.views

描述：
此扩展点用于定义工作台的其他视图。视图是工作台页面中的可视组件。它通常用于导航信息层次结构（如工作区），打开编辑器或显示活动编辑器的属性。用户可以从“窗口”>“显示视图”菜单中查看视图，也可以从视图本地标题栏中关闭视图。

为了减少“显示视图”对话框中的视觉混乱，应使用类别对视图进行分组。

配置标记：

<！ELEMENT 扩展 （ 类别 | 视图 | stickyView ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

id CDATA #REQUIRED

名称CDATA #REQUIRED

parentCategory IDREF #IMPLIED>

  • id - 将用于标识此类别的唯一名称
  • name - 将在此类别的UI中使用的可翻译名称
  • parentCategory - 由以“/”分隔的类别ID组成的可选路径。这允许创建类别层次结构。

<！ELEMENT 视图 （ 描述 ？）>

<！ATTLIST观点

id CDATA #REQUIRED

名称CDATA #REQUIRED

类别IDREF #IMPLIED

class CDATA #REQUIRED

icon CDATA #IMPLIED

fastViewWidthRatio CDATA #IMPLIED

allowMultiple（true | false）

restorable（true | false）“true”>

  • id - 将用于标识此视图的唯一名称
  • name - 将在此视图的UI中使用的可翻译名称
  • category - 一个可选属性，由以“/”分隔的类别ID组成。必须在相应的类别元素中声明每个引用的类别。
  • class - 实现org.eclipse.ui的类的完全限定名称。IViewPart 。通常的做法是子类org.eclipse.ui.part。ViewPart为了继承默认功能。
  • icon - 将与视图关联的图标的相对名称。
  • fastViewWidthRatio - 视图将作为活动快速视图占用的工作台宽度的百分比。必须将其定义为浮点值，介于0.05和0.95之间。如果未提供任何值，则将使用默认比率。
  • allowMultiple - 指示此视图是否允许使用IWorkbenchPage.showView（String id，String secondaryId）创建多个实例的标志。默认值为false。
  • restorable - 指示此视图是否允许在工作台重新启动时恢复的标志。如果设置为false，则在工作台重新启动后视图将不会打开。默认值为true。

    自：3.4


<！元素描述 （#PCDATA）>

一个可选的子元素，其主体应包含提供视图简短描述的文本。



<！ELEMENT stickyView EMPTY>

<！ATTLIST stickyView

id IDREF #REQUIRED

位置（右|左|上|下）

closeable（true | false）

moveable（true | false）>

粘性视图是一个窗口，默认情况下会在窗口中的所有透视图中显示。它的初始位置由位置属性控制，但没有任何东西阻止用户移动或关闭它。使用此元素只会导致创建视图的占位符，它不会显示视图。请注意，此元素的使用应非常谨慎，并且只应用于真正需要跨越视角的视图。从3.0开始


  • id - 要粘贴的视图的ID。
  • location - 可选属性，指定粘性视图相对于编辑器区域的位置。如果不存在，视图将停靠在编辑器区域的右侧。
  • closeable - 可选属性，指定视图应该是否可关闭。如果没有，它将是可以关闭的。
  • moveable - 可选属性，指定视图应该是可移动的。如果没有，它将是可移动的。

例子：
以下是扩展点的示例：

<extension point = “org.eclipse.ui.views” > <category id = “com.xyz.views。XYZviews“ name = ”XYZ“ /> <view id = ”com.xyz.views。XYZView“ name = ”XYZ查看“ category = ”com.xyz.views。XYZviews“ class = ”com.xyz.views。XYZView“ icon = ”icons / XYZ.gif“ /> </ extension>
以下是粘性视图声明的示例：

<extension point = “org.eclipse.ui.views” > <stickyView id = “com.xyz.views。XYZView“ /> </ extension>

API信息：
class属性的值必须是实现org.eclipse.ui的类的完全限定名。IViewPart 。通常的做法是继承org.eclipse.ui.part。开发新视图时的ViewPart 。

提供的实施：
Eclipse Platform提供了许多标准视图，包括Navigator，Properties，Outline和Tasks。从用户的角度来看，这些视图与插件提供的任何其他视图没有区别。所有视图都可以从“窗口”菜单的“显示视图”子菜单中显示。视图的位置是持久的：它在视图关闭时保存，并在单个会话中重新打开视图时恢复。工作台会话之间也保持该职位。
