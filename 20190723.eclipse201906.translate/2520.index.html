PHP调试透视图

运行Debug会话时，可以自动启动PHP Debug Perspective。它包含允许您控制和监视调试过程的视图。

PHP Debug Perspective包含以下视图：

  • 调试视图 - 您可以在此控制（停止，暂停和恢复）调试过程。您还可以决定是进入，跨步还是退回（退出）某些功能。
  • 变量视图 - 显示脚本中的各种变量。
  • 断点视图 - 显示您输入的断点。
  • 调试输出视图 - 显示脚本的文本输出。这将在调试过程继续时更新。
  • 浏览器输出视图 - 将脚本的输出显示给浏览器。这将在调试过程继续时更新。
  • 表达式视图 - 显示所选变量的进度。仅当您选择观看变量时才会显示视图。
  • 脚本视图 - 显示可用脚本的列表。
  • 编辑器 - 根据在“调试视图”窗口中选择的行，显示相关部分的代码。
  • 控制台视图 - 显示任何错误和警告消息。
  • 任务视图 - 显示添加到脚本中的任务（如果适用）。

注意：

默认情况下，将出现一个对话框，询问您是否要在运行调试会话时打开Debug Perspective。要更改此行为，请转到Window |打开Perspectives Preferences对话框偏好|运行/调试|透视并在“启动时打开关联透视”类别中选择“始终”，“从不”或“提示”。
