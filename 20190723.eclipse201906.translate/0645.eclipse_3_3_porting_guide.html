Eclipse JDT 3.3插件迁移指南

本指南介绍了如何将Eclipse JDT 3.2（或更早版本）插件迁移到Eclipse JDT 3.3。

Eclipse 3.3的目标之一是尽可能地向前移动Eclipse，同时保持与先前版本的兼容性。也就是说，尽管API发生了变化，但针对Eclipse 3.2 API编写的插件仍应在3.3中继续工作。

兼容性的关键类型是API合同兼容性和二进制兼容性。API合同兼容性意味着3.2 API的有效使用对3.3仍然有效，因此无需重新访问工作代码。二进制兼容性意味着API方法签名等不会以导致现有编译（“二进制”）代码不再链接和运行新3.3库的方式发生变化。

虽然我们尽一切努力避免破损，但仍存在一些不相容的方面。本文档描述了Eclipse在3.2和3.3之间以不兼容的方式更改的区域，并提供了将3.2插件迁移到3.3的说明。
