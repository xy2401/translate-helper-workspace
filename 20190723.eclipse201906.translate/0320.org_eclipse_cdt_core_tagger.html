节点标签

标识符：
org.eclipse.cdt.core.tagger

以来：
8.2

描述：
此扩展点允许扩展为已解析的节点做出贡献。标记PDOM节点时，标记的内容将存储到数据库中。其他节点的信息，例如AST，存储在存储器中。

配置标记：

<！ELEMENT 扩展 （ bindingTagger +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT bindingTagger EMPTY>

<！ATTLIST bindingTagger

class CDATA #REQUIRED

local-id CDATA #REQUIRED>

  • 课程 -
  • local-id - 此标记器贡献的唯一标识符。本地ID将附加到贡献插件的id（用点'。'分隔），以形成此标记器的全局唯一标识符。此id用于将标记与此插件唯一关联。

例子：
以下是Tagger贡献的示例：


<extension
    point="org.eclipse.cdt.core.tagger"
    id="example"
    name="Example Tagger Extension">
    <bindingTagger
        local-id="my-tagger"
        class="com.example.internal.ExampleTagger">
        <enablement>
            <with variable="projectNatures">
                <iterate operator="or">
                    <equals value="com.example.my-nature"/>
                </iterate>
            </with>
        </enablement>
    </bindingTagger>
</extension>

API信息：
贡献类必须实现org.eclipse.cdt.core.dom.ast.tag.IBindingTagger
