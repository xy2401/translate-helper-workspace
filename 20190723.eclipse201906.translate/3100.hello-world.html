使用RAP创建Hello World

本文档介绍如何使用RAP快速创建和启动第一个用户界面。完整的应用程序包括一些组件，具体取决于您的应用程序设置 。

前提条件

  • 已安装RAP工具
  • 安装并激活RAP目标平台

创建一个新项目

首先，我们需要为您的应用程序创建一个项目。除非您想在没有OSGi的情况下编写RAP应用程序 ，否则建议使用插件项目 。在后一种情况下，您还可以在此处创建纯Java项目 。（如果这样做，项目设置将与本教程略有不同。最值得注意的是，您必须在下一步中将org.eclipse.rap.rwt添加到java构建路径/ Required Projects 。）

从菜单中，选择“ 文件”>“新建”>“插件项目” 。我们来调用插件org.example.rap.helloworld 。在Target Platform框中，选择Eclipse version和3.5或更高 版本以创建与Eclipse兼容的bundle。

在向导的第二页上，禁用“ 生成激活器 ”选项。（此选项将创建一个包激活器，但我们现在不需要它。）确保选中此插件选项将对UI做出贡献，并且不会创建任何富客户端应用程序 。

在下一页上，选择RAP Hello World模板。

设置“ 应用程序”窗口标题和“ 包名” ，然后单击“完成”。

该向导将为您的RAP应用程序创建所有需要的类。

创建应用程序UI

每个RAP应用程序都需要一个入口点类，用于创建应用程序的初始UI。

该向导会自动创建一个名为BasicEntryPoint的类，该类在指定的包中扩展AbstractEntryPoint 。在createContents方法中，我们已经有了一些带有两个按钮的SWT代码。 

public class BasicEntryPoint extends AbstractEntryPoint {

  @Override
  protected void createContents(Composite parent) {
    parent.setLayout(new GridLayout(2, false));
    Button checkbox = new Button(parent, SWT.CHECK);
    checkbox.setText("Hello");
    Button button = new Button(parent, SWT.PUSH);
    button.setText("World");
  }

}

该向导还会创建一个名为BasicApplication的类，该类实现ApplicationConfiguration接口。请注意，如果重命名此类，或将其移动到另一个包，则还必须在文件OSGI-INF/contribution.xml手动重命名该类。

启动应用程序

现在我们可以使用项目根目录中生成的启动配置org.examples.rap.helloworld.launch启动此应用程序。从启动配置文件的上下文菜单中选择Run As> org.examples.rap.helloworld 。浏览器视图应该很快打开并显示正在运行的应用程序。关于开展RAP应用程序的更多信息可在章节中找到启动RAP应用程序 ，其中包括对部分故障排除 。如果浏览器最初显示404错误，通常可以通过刷新固定（按F5）。
