问题标记过滤器

标识符：
org.eclipse.cdt.core。ProblemMarkerFilter

以来：
8.3

描述：
此扩展点允许过滤掉不需要的问题标记。例如，在使用QML文件工具构建Qt基础项目期间，Qt Linguist可以在某个qml文件中报告语法错误。这些错误在qml文件中显示为“C / C ++ Problems”，因为它们与CDT期望的错误格式相匹配。如果已经安装了处理QML文件的插件，则忽略此类错误是明智的，因为它们已被报告为具有更有意义描述的“QML问题”。

配置标记：

<！ELEMENT 扩展 （ problemMarkerFilter +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT problemMarkerFilter （ 启用 ？）>

<！ATTLIST problemMarkerFilter

class CDATA #REQUIRED>

  • class - 问题标记过滤器的实现，允许过滤掉不需要的或双重的问题标记。

例子：
以下是qmakeEnvProvider贡献的示例：

<extension point = “org.eclipse.cdt.core。ProblemMarkerFilter“ id = ”example“ name = ”示例QMake Env Provider Extension“ > <problemMarkerFilter class = ”com.example.internal。ProblemMarkerFilter“ > <enablement> <with variable = ”projectNatures“ > <iterate operator = ”或“ > <equals value = ”com.example.my-nature“ /> </ iterate> </ with> </ enablement> < / problemMarkerFilter> </ extension>

API信息：
贡献类必须实现org.eclipse.cdt.core.IProblemMarkerFilter 。
