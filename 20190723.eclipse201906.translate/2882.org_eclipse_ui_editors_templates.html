编辑模板

标识符：
org.eclipse.ui.editors.templates

以来：
3.0

描述：
模板是文本或代码的片段，可帮助用户将重复出现的模式输入到文本编辑器中。模板可能包含在插入模板的上下文中解析的变量。

配置标记：

<！ELEMENT 扩展 （ 模板 *， 解析器 *， contextType *， 包含 *， contextTypeRegistry *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT contextType EMPTY>

<！ATTLIST contextType

id CDATA #REQUIRED

class CDATA #REQUIRED

名称CDATA #IMPLIED

registryId IDREF #IMPLIED>

上下文类型定义评估模板的上下文。上下文类型使用其解析器来解析模板。


  • id - 明确标识此上下文类型。建议使用限定名称。
  • class - org.eclipse.jface.text.templates.的子类org.eclipse.jface.text.templates.TemplateContextType
  • name - 此上下文的显示名称
  • registryId - 自动注册此上下文类型的上下文类型注册表的id。从3.5开始。

<！ELEMENT resolver EMPTY>

<！ATTLIST解析器

contextTypeId IDREF #REQUIRED

键入CDATA #REQUIRED

class CDATA #REQUIRED

描述CDATA #IMPLIED

名称CDATA #IMPLIED

icon CDATA #IMPLIED>

模板变量解析器可以在特定上下文中解析模板变量。


  • contextTypeId - 引用此解析程序所贡献的上下文类型
  • type - 此变量解析程序的类型。创建后，此属性将在解析程序上设置。
  • class - org.eclipse.jface.text.templates.的子类org.eclipse.jface.text.templates.TemplateVariableResolver
  • description - 此变量解析程序的描述。创建后，此属性将在解析程序上设置。
  • name - 此解析程序的显示名称
  • icon - 可以在用户界面中显示的图标

<！ELEMENT 模板 （ 模式 ）>

<！ATTLIST模板

id CDATA #REQUIRED

contextTypeId IDREF #REQUIRED

名称CDATA #REQUIRED

描述CDATA #IMPLIED

icon CDATA #IMPLIED

autoinsert（true | false）“true”>

模板是将在给定上下文中评估的代码或文本片段。可以使用$ {variable_type}表示法指定将在该上下文中解析的变量 。


  • id - 明确标识此模板。建议使用限定名称。
  • contextTypeId - 引用此模板所贡献的上下文类型
  • name - 将在UI中显示的模板的可国际化名称，例如模板提议中
  • description - 此模板的描述
  • icon - 可以在此模板的UI中显示的图标，例如在内容辅助提议中
  • autoinsert - true （默认）使模板自动插入， false为不允许自动插入。从3.1开始。

<！元素模式 （#PCDATA）>

模板模式。



<！ELEMENT 包括 EMPTY>

<！ATTLIST包括

文件CDATA #REQUIRED

翻译CDATA #IMPLIED>

编码为XML的模板集合可以通过此元素作为整体包含在内。


  • file - 要从中导入模板的XML文件。它在<templates> ... </ templates>中定义了至少一个<template>元素，具有以下属性：
    • id - uniqe id
    • 名称 - 名称
    • context - 上下文id
    • description - 描述（可选，默认：“”）
    • autoinsert（可选，默认值：true）
    • 启用（可选，默认值：true）
    有关这些属性的详细信息，请参见上面的模板元素
  • 转换 - 一个可选的属性文件，包含文件中指定的模板的资源。
    注意：转换文件不被视为资源包，因此文件名必须保留原样，即不要将语言环境添加到文件名中。翻译的文件必须位于'nl'目录中，即如果[plugin]在其plugin.xml中定义模板翻译文件，例如：
    ...translations = $ nl $ / [插件相对文件路径] ...
    然后翻译必须进入：
    [plugins nl fragment] / nl / [locale] / templates / [plugin-relative path to file]

<！ELEMENT contextTypeRegistry EMPTY>

<！ATTLIST contextTypeRegistry

id CDATA #IMPLIED>

为上下文类型定义注册表。定义“registryId”的上下文类型会自动添加到由new org.eclipse.ui.editors.text.templates.创建的注册表中new org.eclipse.ui.editors.text.templates.ContributionContextTypeRegistry(registryId) 。从3.5开始。


  • id - 此注册表的ID

例子：
<extension point = “org.eclipse.ui.editors.templates” > <template name = “％ant.tasks.javac.name” contextTypeId = “org.eclipse.ui.examples.templateeditor.antcontext” id = “org。 eclipse.ui.examples.templateeditor.templates.javac“ description = ”％ant.tasks.javac.description“ > <pattern> <javac srcdir = ”$ {src}“ destdir = ”$ {dst}“ classpath = ”$ {classpath}“ debug = ”$ {debug}“ /> </ pattern> </ template> <resolver contextTypeId = ”org.eclipse.ui.examples.templateeditor.antcontext“ type = ”src“ class = ”org.eclipse .ui.examples.templateeditor.editors。AntVariableResolver“ > </ resolver> <resolver contextTypeId = ”org.eclipse.ui.examples.templateeditor.antcontext“ type = ”dst“ class = ”org.eclipse.ui.examples.templateeditor.editors。AntVariableResolver“ > </ resolver> </ extension>
