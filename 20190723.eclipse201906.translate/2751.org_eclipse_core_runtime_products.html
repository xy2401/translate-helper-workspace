制品

标识符：
org.eclipse.core.runtime.products

以来：
3.0

描述：
产品是Eclipse的品牌单位。产品扩展由希望定义一个或多个产品的插件提供。每个扩展名必须有一个产品，因为扩展ID用于处理和识别产品。

产品扩展有两种可能的形式，静态和动态。静态产品扩展直接包含有关产品的所有相关信息。动态产品扩展标识了一个类（ IProductProvider ），它能够在查询时定义一个或多个产品。

配置标记：

<！ELEMENT 扩展 （（ 产品 | 提供商 ））>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 产品 （ 财产 *）>

<！ATTLIST产品

申请CDATA #REQUIRED

名称CDATA #REQUIRED

说明CDATA #IMPLIED>

  • application - 运行此产品时要运行的默认应用程序
  • name - 此产品的可读名称
  • description - 该产品的人类可读描述

<！ELEMENT 属性 EMPTY>

<！ATTLIST财产

名称CDATA #REQUIRED

值CDATA #REQUIRED>

  • name - 存储此属性的键
  • value - 此属性的值

<！ELEMENT 提供者 （ 运行 ）>

产品提供商的详细信息



<！ELEMENT 运行 EMPTY>

<！ATTLIST跑

class CDATA #REQUIRED>

  • class - 实现org.eclipse.core.runtime.IProductProvider 。

例子：
以下是静态产品声明的示例：


   <extension id="coolProduct" point="org.eclipse.core.runtime.products"> 
      <product name="%coolName" application="coolApplication" description="%coolDescription"> 
          <property name="windowImages" value="window.gif"/> 
          <property name="aboutImage" value="image.gif"/> 
          <property name="aboutText" value="%aboutText"/> 
          <property name="appName" value="CoolApp"/> 
          <property name="welcomePage" value="$nl$/welcome.xml"/> 
          <property name="preferenceCustomization" value="plugin_customization.ini"/> 
      </product> 
   </extension>
以下是动态产品（产品提供者）声明的示例：以下是应用程序声明的示例：


   <extension id="coolProvider" point="org.eclipse.core.runtime.products"> 
      <provider> 
         <run class="com.example.productProvider"/> 
      </provider> 
   </extension>

API信息：
此处提供的静态产品扩展在运行时由IProduct实例IProduct 。动态产品扩展必须标识IProductProvider的实现者。请参阅org.eclipse.ui.branding.IProductConstants ，了解Eclipse UI定义的与品牌相关的产品属性的详细信息。

提供的实施：
没有提供IProductProvider实现。
