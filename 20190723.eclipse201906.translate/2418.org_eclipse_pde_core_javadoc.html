Javadoc附件

标识符：
org.eclipse.pde.core.javadoc

以来：
3.2

描述：
此扩展点将插件与其包含的代码的javadoc位置相关联。PDE在计算插件的类路径时使用这些映射自动将javadoc附加到插件库。

配置标记：

<！ELEMENT 扩展 （ javadoc +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT javadoc （ 插件 +）>

<！ATTLIST javadoc

路径CDATA #REQUIRED

archive（true | false）“false”>

  • path - path是URL或相对于插件声明扩展名的路径。
  • archive - 一个可选属性，指示javadoc是否位于存档内。如果未指定，则默认值为false 。仅当path属性指定相对路径时，才应使用此属性。如果path属性指定URL，则此属性无效。

<！ELEMENT 插件 EMPTY>

<！ATTLIST插件

id CDATA #REQUIRED>

  • id - 由path属性指定其javadoc位置的插件的id

例子：
以下是javadoc扩展的示例：

 <extension point="org.eclipse.pde.core.javadoc">
     <javadoc path="doc.zip!/references/api/" archive="true">
        <plugin id="com.example.abc"/>
        <plugin id="com.example.def"/>
     </javadoc>
     
     <javadoc path="http://www.sample-url.org/doc/">
        <plugin id="com.example.ghi"/>
     </javadoc>
     
     <javadoc path="doc/">
        <plugin id="com.example.jkl"/>
     </javadoc>     
  </extension>
在此示例中，两个插件的javadoc： com.example.abc和com.example.def将其javadoc放在插件中references/api/ ZIP文件内的doc.zip目录中，声明此扩展名。插件com.example.ghi的javadoc位于http://www.sample-url.org/doc/ 。插件com.example.jkl的javadoc位于目录doc相对于插件的位置声明此扩展名。

API信息：
此扩展点不需要Java代码。

提供的实施：
Eclipse SDK附带了使用此扩展点的ISV文档插件。
