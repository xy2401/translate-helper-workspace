远程系统资源管理器的第一步

即使没有与远程系统的实际连接，您也可以开始尝试使用本地主机上的Remote System Explorer UI，默认情况下在Local节点下显示：
  • 浏览文件系统，选择contextmenu> show in Table ，然后观察Properties视图。
  • 创建一个新的过滤器 ，仅显示文件系统中的特定资源。
  • 启动RSE Shell （ Shell节点>启动）
    • 在Shell中，执行命令，如ls，dir，ps，gcc，make，并查看输出的解释
    • 在shell命令输入字段上使用Ctrl + Space Content Assist
对于实际远程系统上的操作，您也可以
  • 使用“ 仅限SSH ”系统类型（“ 新建”>“其他”>“远程系统资源管理器”>“连接” ）或
  • 在远程系统上启动dstore服务器守护程序并使用任何其他连接类型。

RSE是一个支持插入许多不同通信协议的框架。默认情况下，提供dstore，FTP和ssh协议插件，dstore是功能最丰富的插件。
DStore需要服务器在远程系统上运行。有几种方法可以为特定用户启动服务器，最容易设置的是守护进程方法。
