启动代表

标识符：
org.eclipse.debug.core.launchDelegates

以来：
3.0

描述：
此扩展点提供了一种机制，用于将启动委托提供给一个或多个启动模式的现有启动配置类型。由于启动模式是可扩展的，因此可能需要将其他启动委托贡献给现有的启动配置类型。每个启动委托都是针对特定的启动配置类型提供的。启动委托支持一种或多种启动模式，并指定负责启动实现的委托。

配置标记：

<！ELEMENT 扩展 （ launchDelegate +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT launchDelegate （ modeCombination +）？>

<！ATTLIST launchDelegate

id CDATA #REQUIRED

名称CDATA #IMPLIED

委托CDATA #REQUIRED

模式CDATA #IMPLIED

输入IDREF #IMPLIED

sourcePathComputerId IDREF #IMPLIED

sourceLocatorId IDREF #IMPLIED

delegateDescription CDATA #IMPLIED>

  • id - 指定此启动委托的唯一标识符。
  • name - 此委托的可读名称 - 例如，“Eclipse JDT Launcher”。当多个启动器可用于启动配置且用户必须选择首选启动器时，将向用户显示此名称。此属性已添加到3.3中。发布。
  • delegate - 指定实现ILaunchConfigurationDelegate的Java类的完全限定名称。启动此委托类型的配置实例将委托给此类的实例，以在此启动委托支持的模式下执行启动 - 请参阅modes属性和modeCombination元素。
  • modes - 此委托支持的各个启动模式的逗号分隔列表。

    当启动委托支持单一启动模式时，应使用此属性 - 例如，在运行模式或调试模式下启动。当启动委托支持以混合模式启动时（例如，调试和配置文件）， modeCombination元素应该用于指定支持的启动模式组合。

    此属性可与modeCombination元素一起使用。建议练习提供一个或另一个（ modes属性中的逗号分隔列表，或modeCombinations ）。在modes属性中指定'run，debug'类似于指定两个modeCombination贡献，一个用于'run'而另一个用于'debug'。

  • type - 此启动委托能够启动的现有启动配置类型的标识符。
  • sourcePathComputerId - sourcePathComputer扩展的唯一标识符，用于计算此类型的启动配置的默认源查找路径。从3.1开始，当在关联的launchConfigurationType扩展中未指定时，此属性cab在launchDelegate扩展中指定。每种启动配置类型只能指定一个源路径计算机。
  • sourceLocatorId - sourceLocator扩展的唯一标识符，用于为使用此类型的启动配置启动的会话创建源定位器。从3.1开始，当在关联的launchConfigurationType扩展中未指定时，可以在launchDelegate扩展中指定此属性。每种启动配置类型只能指定一个源定位器。
  • delegateDescription - 此启动委托的描述 - 例如“Eclipse JDT Java Application Launcher支持运行和调试本地Java应用程序”。当多个启动器可用于启动配置并且用户必须选择首选启动器时，将向用户呈现此描述。此属性已在3.3版本中添加。

<！ELEMENT modeCombination EMPTY>

<！ATTLIST模式组合

模式CDATA #REQUIRED

透视CDATA #IMPLIED>

此元素指定此委托支持的启动模式组合。每个modeCombination元素指定一个逗号分隔的模式列表，指定支持的混合启动模式。例如，同时支持调试，分析和代码覆盖的委托将指定由'debug，profile，coverage'组成的modes元素。此元素还可用于指定支持的单个启动模式 - 例如“debug”的modes属性。这个元素在3.3版本中添加了。


  • modes - 逗号分隔列表，指定关联委托支持的模式组合。

    与launchDeleaget元素定义的modes属性不同，此逗号分隔列表表示混合启动模式，而不是单个支持的启动模式。例如：指定“profile，debug” modeCombination的modes元素表示委托可以同时调试和分析。相反，如果'profile，debug'由modes元素的launchDelegate属性指定，它将指示委托可以调试或分析，但不能同时。

  • perspective - 一个字符串，表示启动时要切换到的persepctive的id。

例子：
以下是启动委托扩展点的示例：


 <extension point="org.eclipse.debug.core.launchDelegates">
  <launchDelegate
    id="com.example.ExampleProfileDelegate"
    delegate="com.example.ExampleProfileDelegate"
    type="org.eclipse.jdt.launching.localJavaApplication"
    name="Eclipse JDT Java Profile Tooling">
   <modeCombination 
    modes="run, profile"
    perspective="com.example.Perspective">
   </modeCombination>
   <modeCombination modes="debug, profile, coverage"></modeCombination>
  </launchDelegate>
 </extension>
在上面的示例中，指定的启动委托用于在配置文件模式下启动Java应用程序。该委托支持混合模式启动。委托在运行或调试模式下启动时支持分析，代码覆盖率仅在调试模式下。使用模式组合或运行和配置文件时，在此示例中，透视com.example。透视也可以打开。

API信息：
属性delegate的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。ILaunchConfigurationDelegate 。
