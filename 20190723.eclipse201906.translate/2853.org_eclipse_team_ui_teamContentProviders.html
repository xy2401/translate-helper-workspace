团队内容提供商

标识符：
org.eclipse.team.ui.teamContentProviders

以来：
3.2

描述：
此扩展点将模型提供者与内容提供者相关联。存储库提供程序将在执行团队操作时使用此信息来启用适当的内容提供程序。

配置标记：

<！ELEMENT 扩展 （ teamContentProvider ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT teamContentProvider EMPTY>

<！ATTLIST teamContentProvider

modelProviderId IDREF #REQUIRED

contentExtensionId IDREF #REQUIRED

icon CDATA #IMPLIED

preferencePage CDATA #IMPLIED

supportsFlatLayout（true | false）>

将模型提供者与内容扩展相关联。


  • modelProviderId - 与内容扩展相关联的org.eclipse.core.resoures.modelProvider扩展的标识。
  • contentExtensionId - 与此模型提供程序关联的org.eclipse.ui.navigator.navigatorContent扩展的标识。
  • icon - 在列表和菜单中显示此模型时将使用的图标。
  • preferencePage - 可以由承载内容扩展的视图显示的首选项页面
  • supportsFlatLayout - 指示此内容提供程序是否支持平面布局。默认情况下，内容提供商不支持平面布局。请参阅org.eclipse.team.ui.mapping类中的PROP_PAGE_LAYOUT常量。有关更多信息，请访问ITeamContentProviderManager。

例子：
这是一个示例teamContentProvider扩展

   <extension
         id="jdtContentProvider"
         point="org.eclipse.team.ui.teamContentProviders">
      <teamContentProvider
            contentExtensionId="org.eclipse.jdt.ui.resourceContent"
            modelProviderId="org.eclipse.jdt.ui.modelProvider"/>
   </extension>

API信息：
modelProviderId必须对应于使用org.eclipse.core.resources.modelProviders扩展点注册的模型提供程序。同样，contentExtensionId必须对应于使用org.eclipse.ui.navigator.navigatorContent扩展点注册的导航器内容扩展。

提供的实施：
没有与此扩展点直接关联的实现。
