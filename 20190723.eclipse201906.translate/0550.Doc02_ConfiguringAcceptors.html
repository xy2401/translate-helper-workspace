配置接受者

  

作者：Eike Stepper

CDO服务器的接受者在cdo-server.xml文件中配置。这是一个例子：

 CDO-的server.xml      
<？xml version = “1.0” encoding = “UTF-8” ？>
 <cdoServer>

   <acceptor type = “tcp” listenAddr = “0.0.0.0” port = “2036” />

   <！ - 其他接受者和存储库 - >

 </ cdoServer>

以下部分描述了各种元素和属性。

目录

1元素接受者
2元素谈判者

1元素接受器

定义IAcceptor实例。有关接受器和连接器的详细信息，请参阅Net4j信令平台文档。

type属性对应于通过org.eclipse.net4j.util.factories扩展点以及org.eclipse.net4j.acceptors的产品组提供的acceptor工厂的类型。

其余属性取决于指定的type属性值。随附的分发可能具有以下值（取决于用户提供的扩展名）：

  • tcp ： ITCPAcceptor用于快速，基于I / O的快速套接字连接。识别以下附加属性：
    • listenAddr ：服务器套接字应绑定的网络地址。值"0.0.0.0"是默认值（可以省略整个属性）并告诉套接字监听所有可用地址。
    • port ：服务器套接字应绑定的网络端口。值2036是默认值（可以省略整个属性）。
  • ssl ：与tcp类似，但具有传输级安全性（TLS）。
  • jvm ：用于JVM内部（基于非套接字）连接的IJVMAcceptor 。
  • http ：实验IHTTPAcceptor 。

2元谈判代表

定义由封闭的acceptor元素使用的INegotiator实例。有关协商器的详细信息以及可用于传输级别的身份验证和授权的可插入安全性概念，请参阅Net4j信令平台文档。

type属性对应于通过org.eclipse.net4j.util.factories扩展点以及org.eclipse.net4j.negotiators的产品组提供的协商工厂的类型。

其余属性取决于指定的类型属性值。随附的分发可能具有以下值（取决于用户提供的扩展名）：

  • 挑战 ：谈判者简单而有效且加密安全的挑战/响应式谈判。识别以下附加属性：
    • description ：本地文件系统中文件的绝对路径，其中包含user-id: password形式的user-id: password凭据。
