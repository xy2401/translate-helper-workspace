ECF开始

标识符：
org.eclipse.ecf.start

以来：
2.0.0M4

描述：
ECF核心启动扩展点。此扩展点（ org.eclipse.ecf.start ）允许扩展在ECF核心包启动时运行。请注意，当运行时启动ECF核心包（ID： org.eclipse.ecf ）时，将运行扩展，而不一定在平台启动时启动。

配置标记：

<！ELEMENT 扩展 （ 运行 ）+>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT 运行 EMPTY>

<！ATTLIST跑

class CDATA #REQUIRED

异步（true | false）

>

  • class - 必须实现IECFStart的类。
  • asynchronous - 用于确定给定运行类是同步（默认）还是异步运行的属性

例子：
以下是此扩展点的示例用法：
<extension point = “org.eclipse.ecf.start” > <run class = “org.eclipse.ecf.example.collab.start。CollabStart“ /> </ extension> <extension point = ”org.eclipse.ecf.start“ > <run class = ”org.eclipse.ecf.example.collab.start。CollabStart“ asynchronous = ”true“ /> </ extension>
请注意，CollabStart类必须实现org.eclipse.ecf.start。IECFStart接口。这是一个示例实现类：
公共类CollabStart实现IECFStart {public IStatus run（IProgressMonitor monitor）{...}}

API信息：
org.eclipse.ecf.startup。IECFStartup接口：
/ ** *必须通过org.eclipse.ecf.start *扩展点的扩展来实现的接口。这样的扩展将在ECF启动时通过新的*作业调用它们的start方法。* / public interface IECFStart {/ ** *运行一些启动任务。* * @return IStatus启动状态* / public IStatus run（IProgressMonitor monitor）; }

提供的实施：
没有为此扩展点提供实现。
