喜好

标识符：
org.eclipse.equinox.preferences.preferences

以来：
3.0

描述：
首选项扩展点允许bundle将新的首选项作用域添加到Eclipse首选项机制，以及指定要在运行时初始化缺省首选项值的类。

配置标记：

<！ELEMENT 扩展 （ 范围 *， 初始化 *， 修饰符 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 范围 EMPTY>

<！ATTLIST范围

名称CDATA #REQUIRED

class CDATA #IMPLIED

存储CDATA #IMPLIED>

描述客户端对新偏好范围的定义的元素。


  • name - 范围的名称。
  • class - 类的名称。
  • storage - 实现首选项存储的类。

<！ELEMENT 初始化程序 EMPTY>

<！ATTLIST初始化程序

class CDATA #REQUIRED>

定义用于运行时首选项初始化的类的元素。


  • class - 类的名称。

<！ELEMENT 修饰符 EMPTY>

<！ATTLIST修饰符

class CDATA #REQUIRED>

定义用于首选项修改侦听的类的元素。


  • class - 类的名称。

例子：
以下是首选范围声明的示例。此示例声明此插件将为范围“foo”提供首选项实现。它还声明当为此插件加载默认值时，类“MyPreferenceInitializer”包含要在运行时初始化首选项默认值的代码。 

<extension point = “org.eclipse.equinox.preferences.preferences” > <scope name = “foo” class = “com.example。FooPrefs“ /> <initializer class = ”com.example。MyPreferenceInitializer“ /> <modifier class = ”com.example。MyModifyListener“ /> </ extension>
如果客户端选择让Eclipse平台处理首选项节点但希望定义用于持久化数据的自定义位置/机制，则他们可以在“scope”元素上设置“storage”属性。请注意，只能同时定义“class”和“storage”属性中的一个。“storage”属性中引用的类必须扩展AbstractPreferenceStorage类。在以下示例中，“FooStorage”类处理“foo”首选项的加载/保存。 

<extension point = “org.eclipse.equinox.preferences.preferences” > <scope name = “foo” storage = “com.example。FooStorage“ /> <initializer class = ”com.example。MyPreferenceInitializer“ /> <modifier class = ”com.example。MyModifyListener“ /> </ extension>

API信息：
首选项服务（OSGi服务org.eclipse.core.runtime.preferences.IPreferencesService ）是Eclipse偏好机制的钩子。

提供的实施：
org.eclipse.equinox.preferences包为“配置”，“实例”和“默认”范围提供首选项实现。org.eclipse.core.resources包提供了“项目”首选项的实现。
