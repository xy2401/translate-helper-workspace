介绍

Eclipse的Autotools插件是CDT（C / C ++开发工具）的可选功能，它增加了对使用GNU Autotools维护和构建C / C ++项目的支持。作为Eclipse Juno版本的一部分，Autotools插件从Linux Tools项目迁移到CDT。

GNU Autotools是一组用于使项目可移植到多个系统或构建环境的工具。这些工具可帮助开发人员创建配置脚本，该脚本应在执行构建之前调用。配置脚本可以执行测试，例如测试当前平台，操作系统，本地安装的内容或任何数量的内容。也可以传递参数以提供无法计算的信息（例如，将运行项目的目标平台是什么）。测试和参数的结果用于为构建创建Makefile，或者在某些情况下，使用其他文件（如头文件或代码序列）。

开发人员可以通过输入到Autotools的特殊输入文件完全控制测试内容。通常，最常用的Autotools是'autoconf'，'automake'和'aclocal'。'autoconf'工具采用'configure.in'或'configure.ac'输入文件，并创建前面讨论过的'configure'脚本。在各个子目录中可以有多个配置脚本，但是应该设计一个项目，以便有一个顶级配置脚本自动调用任何低级别的脚本。

'automake'工具接受'Makefile.am'输入文件，并创建一个'Makefile.in'文件，配置脚本在配置时使用该文件作为创建Makefile的模板。每个Makefile都有自己对应的Makefile.in文件。可能有if / else逻辑用于确定Makefile中的内容，或者可能有请求直接替换在configure步骤中计算的变量。

'aclocal'工具创建了一个宏库，这些宏直接或间接地在'autoconf'输入文件中指定。提供这样的宏以执行常用的测试或动作（例如，测试某个头文件存在或找到C编译器）。这些宏用一种名为m4的语言编写，开发人员可以自由创建自己的宏来添加到存储库中。

Autotools插件增加了在构建之前运行configure脚本的支持。此外，还添加了对调用Autotools本身以及维护其使用的文件的支持。Autotools项目扩展了CDT Makefile项目。因此，CDT中用于支持此类项目的功能也适用于Autotools项目（例如，C / C ++属性页面）。

有关GNU Autotools的更多详细信息，请访问http://www.gnu.org/software/autoconf/和http://www.gnu.org/software/automake/
