步骤过滤器

标识符：
org.eclipse.debug.core.stepFilters

以来：
3.9

描述：
此扩展点允许贡献者为模型标识符提供多个步骤过滤器。

配置标记：

<！ELEMENT 扩展 （ stepFilter *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT stepFilter EMPTY>

<！ATTLIST stepFilter

class CDATA #REQUIRED

modelIdentifier CDATA #REQUIRED>

  • class - 实现IStepFilter的Java类的完全限定名称。
  • modelIdentifier - 此步骤过滤器与之关联的调试模型的标识符

例子：
以下是步骤过滤器扩展点的示例：

<extension point = “org.eclipse.debug.core.stepFilters” > <stepFilter class = “com.example。ExampleStepFilter“ modelIdentifier = ”com.example.debug.model“ > </ stepFilter> </ extension>
在上面的示例中，指定的步骤过滤器将用于com.example.debug.model调试模型。

API信息：
属性类的值必须是实现接口org.eclipse.debug.core.model的Java类的完全限定名称。IStepFilter 。
