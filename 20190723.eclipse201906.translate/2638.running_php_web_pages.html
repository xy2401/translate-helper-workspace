运行PHP网页

此过程描述如何运行服务器上的整个应用程序，项目，文件或文件集合。使用此过程，您可以运行位于服务器上的文件副本或位于工作区本地的文件（如果可用）。

注意：

您的服务器必须运行Xdebug或Zend Debugger才能使远程调试功能正常运行。

要运行PHP Web页面：

  1. 单击“运行”按钮旁边的箭头在工具栏上，选择“运行配置” - 或 - 转到“运行”|运行配置。“运行”对话框将打开。
  2. 双击PHP Web Page选项以创建新的运行配置。
  3. 输入新配置的名称。
  4. 选择要使用的服务器调试程序。PDT支持Zend Debugger和Xdebug。
  5. 从PHP Server列表中选择您的服务器。如果尚未配置服务器，请单击“新建”。PHP服务器创建向导将打开。按照PHP Servers Preferences页面下的“ 添加新服务器 ”的说明配置服务器。
  6. 要检查服务器连接是否正确配置以及PDT是否可以与服务器调试器通信，请单击“测试调试器”按钮。
  7. 在PHP文件下，单击“浏览”并选择要运行的文件。
  8. 将根据文件名和服务器地址自动创建要运行的URL。如果URL未指向文件的位置，请取消选中“自动生成”复选框并修改URL。

注意：

即使您要选择运行文件的本地副本，也要在服务器上存在要运行的文件。

单击“应用并运行”。

您的应用程序将在浏览器中运行并显示。

注意：

如果文件包含对项目中未包含的文件的“包含”或“要求”调用，则必须将它们添加到项目的“包含路径”中，以便模拟生产环境。此外，如果调用使用服务器位置的绝对路径定义的文件（有关绝对文件位置的更多信息，请参阅“ 包含路径 ”），将显示“路径映射”对话框。有关更多信息，请参阅路径映射
