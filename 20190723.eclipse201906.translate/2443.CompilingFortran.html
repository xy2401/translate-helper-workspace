要编译项目，Photran通常会执行系统路径上的make命令。（在大多数系统上，这将调用GNU Make。）如上所述，在创建项目时，您可以决定Photran是否会自动生成使用GNU Fortran，Intel Fortran或IBM XL Fortran的Makefile，或者您可以选择手写自己的Makefile。

  • 要使用自动生成的Makefile，请从项目类型菜单中选择“可执行文件（XYZ上的Gnu Fortran）”，其中XYZ是您的平台（Windows，Linux或Mac OS X）。英特尔Fortran和IBM XL Fortran也有类似的选项。
  • 要使用自定义Makefile创建项目，请在项目类型选择框中展开Makefile Project文件夹，然后选择Empty Project。

建设项目

编译Fortran项目与编译C / C ++项目完全相同。 有关构建项目的更多信息，请单击Eclipse中的“帮助”>“帮助内容”，然后导航到“ C / C ++开发用户指南”>“概念”>“构建”>“构建C / C ++项目”。

设置制作目标

为Fortran程序设置make目标也与为C / C ++程序设置make目标相同。 有关make目标的更多信息，请单击Eclipse中的Help> Help Contents，然后导航到C / C ++ Development用户指南>任务>构建项目>创建make目标。

标记来自Fortran编译器的错误消息（错误解析器）

构建（编译）Fortran程序时，Fortran编译器的输出（包括任何错误消息）将显示在Console视图中。但是，对于许多编译器，Photran可以“识别”错误消息，将问题描述放在Problems视图中，并用红色X图标标记源文件中的相应行。为此，您必须在Fortran项目的构建设置中选择正确的错误解析器 。要修改这些设置，请右键单击Fortran项目，转到“属性”>“Fortran构建”>“设置”，然后选择“错误分析器”选项卡。显示错误解析器列表;选择与您正在使用的Fortran编译器对应的那个，然后单击“确定”。

识别可执行文件（二进制解析器）

为了运行（执行）您的Fortran应用程序，您需要告诉Eclipse哪个Binary Parser用于您的项目。这可以在项目属性中完成（右键单击项目>属性> Fortran构建>设置>二进制分析器选项卡）。您要使用哪个二进制解析器取决于您正在使用的操作系统环境。例如，默认选择是Elf Parser，适用于Linux环境。Windows用户需要取消选择Elf Parser并选择PE Windows Parser和/或Cygwin PE Parser。Mac用户应该选择Mach-O解析器。
