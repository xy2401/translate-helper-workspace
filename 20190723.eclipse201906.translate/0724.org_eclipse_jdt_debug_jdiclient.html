JDI客户端

此扩展点是内部的

标识符：
org.eclipse.jdt.debug.jdiclient

以来：
2.0

描述：
这是一个内部扩展点，指定与Java调试器一起使用的JDI实现。客户端无意使用此扩展点。

配置标记：

<！ELEMENT 扩展名 EMPTY>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • name - 扩展点的名称是实现com.sun.jdi.的Java类的完全限定名称com.sun.jdi.VirtualMachineManager用于引导JDI实现。

例子：
Java调试器使用的JDI客户端默认是Eclipse SDK提供的JDI实现：
<extension-point id = “jdiclient” name = “org.eclipse.jdi.internal。VirtualMachineManagerImpl“ />

API信息：
扩展点的名称是实现com.sun.jdi.的Java类的完全限定名称com.sun.jdi.VirtualMachineManager 。

提供的实施：
Java调试器提供com.sun.jdi.的实现com.sun.jdi.VirtualMachineManager
