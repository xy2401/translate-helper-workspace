ODA连接配置文件存储

标识符：
org.eclipse.datatools.connectivity.oda.profile.profileStore

以来：
3.2.7（DTP 1.9.2）

描述：
profileStore扩展点允许客户端为没有文件扩展名的任何连接配置文件存储定义默认文件扩展名。如果未注册此扩展点的扩展名，则用户可以指定不带扩展名的概要文件存储文件。如果注册了多个默认的fileExtension元素，则只应用一个元素。采用者应确保其客户端应用程序仅安装此扩展点的一个扩展。

配置标记：

<！ELEMENT 扩展 （ fileExtension ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>


<！ELEMENT fileExtension EMPTY>

<！ATTLIST fileExtension

默认CDATA #REQUIRED

>

定义与连接配置文件存储关联的文件扩展名。


  • default - 定义默认文件扩展名，以自动与没有用户定义文件扩展名的连接配置文件存储关联。该属性值可以包括文件扩展名之前的可选点（例如“profiles”或“.profiles”）。

例子：

     <extension
           point="org.eclipse.datatools.connectivity.oda.profile.profileStore">
        <fileExtension
              default="profiles">
        </fileExtension>
     </extension>
此扩展示例指定默认文件扩展名“profiles”以自动与没有用户定义文件扩展名的连接配置文件存储关联。
