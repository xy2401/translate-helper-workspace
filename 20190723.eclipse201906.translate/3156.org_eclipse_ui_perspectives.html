展望

标识符：
org.eclipse.ui.perspective

描述：
此扩展点用于将透视工厂添加到工作台。透视工厂用于定义透视图的初始布局和可见操作集。用户可以通过调用“窗口”菜单的“打开透视图”子菜单来选择透视图。

配置标记：

<！ELEMENT 扩展 （ 透视 *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 透视 （ 描述 ？）>

<！ATTLIST的观点

id CDATA #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED

icon CDATA #IMPLIED

fixed（true | false）>

  • id - 将用于标识此透视图的唯一名称。
  • name - 将在工作台窗口菜单栏中用于表示此透视图的可翻译名称。
  • class - 实现org.eclipse.ui的类的完全限定名称。IPerspectiveFactory接口。
  • icon - 将与此透视关联的图标的相对名称。
  • fixed - 指示透视的布局是否固定。如果为true，则透视工厂创建的视图不可关闭，并且无法移动。默认值为false。

<！元素描述 （#PCDATA）>

一个可选的子元素，其主体应包含提供透视的简短描述的文本。



例子：
以下是透视扩展的示例：

<extension point = “org.eclipse.ui.perspectives” > <perspective id = “org.eclipse.ui.resourcePerspective” name = “Resource” class = “org.eclipse.ui.internal。ResourcePerspective“ icon = ”icons / MyIcon.gif“ > </ perspective> </ extension>

API信息：
class属性的值必须是实现org.eclipse.ui的类的完全限定名。IPerspectiveFactory 。当工作台询问时，类必须为透视提供初始布局。

plugin_customization.ini文件用于定义默认透视图。默认透视图是在安装后启动产品时显示的第一个透视图。当用户打开没有指定透视图的页面或窗口时，也会使用它。默认透视图定义为plugin_customization.ini中的属性，如下所示。用户还可以从工作台透视图首选项页面覆盖此透视图。 

defaultPerspectiveId = org.eclipse.ui.resourcePerspective
“Open Perspective”菜单中显示的透视图是透视选择的快捷方式。此集由活动透视本身定义，并通过perspectiveExtensions扩展点进行扩展。

提供的实施：
工作台提供“资源透视图”。插件可以添加其他透视图。使用“窗口”菜单的“打开透视图”子菜单选择它们。
