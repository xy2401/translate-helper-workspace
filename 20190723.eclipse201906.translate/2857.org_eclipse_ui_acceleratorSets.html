org.eclipse.ui.acceleratorSets

以来：

描述：

警告：此扩展点已弃用。
请勿使用此扩展点，将在本产品的未来版本中将其删除。相反，请使用扩展点org.eclipse.ui.commands

此扩展点用于注册加速器集扩展。加速器集正如其名称所暗示的那样，加速器组。加速器是一个或多个加速键序列与工作台动作之间的关联。加速键序列的长度可以是1或更大。

加速器集注册了加速器配置（请参阅加速器配置扩展点），适用于加速器范围（请参阅加速器范围扩展点）。

配置标记：

<！ELEMENT 扩展 （ acceleratorSet *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

不推荐使用acceleratorSet元素

<！ELEMENT acceleratorSet （ 加速器 +）>

<！ATTLIST加速器集

configurationId CDATA #REQUIRED

scopeId CDATA #REQUIRED>

@deprecated请改用“org.eclipse.ui.bindings”扩展点。使用key元素。


  • configurationId - 标识此加速器集注册到的加速器配置的唯一名称。
  • scopeId - 标识此加速器集适用的加速器范围的唯一名称。@deprecated迁移到新绑定扩展点时，建议您将“org.eclipse.ui.globalScope”的任何出现更改为“org.eclipse.ui.contexts.window”。

不推荐使用加速器元素

<！ELEMENT 加速器 EMPTY>

<！ATTLIST加速器

id CDATA #IMPLIED

关键CDATA #REQUIRED

locale CDATA #IMPLIED

平台CDATA #IMPLIED>

@deprecated请改用“org.eclipse.ui.bindings”扩展点。使用key元素。


  • id - 与此加速器关联的操作的操作定义的唯一标识符。如果未指定id，则此加速器将删除具有相同密钥的任何映射。这用于删除特定Locale的键绑定。
  • key - 表示用于执行与此加速器关联的操作的加速键的序列的属性。序列用“||”分隔，序列中的各个键用空格分隔。可以通过CTRL，ALT或SHIFT键修改密钥。根据键盘布局，一些键（'？'例如）可能需要访问SHIFT但是应该在没有SHIFT的情况下指定加速器，因此它将独立于键盘布局。例如，如果CTRL +？被指定为加速器，用户可能必须按CTRL + SHIFT +？取决于键盘布局。
  • locale - 一个可选属性，指定加速器适用的语言环境。如果未指定此属性，则加速器适用于所有语言环境。
  • platform - 一个可选属性，指定加速器适用的平台。如果未指定此属性，则加速器适用于所有平台。

例子：
以下是加速器集扩展的示例：

<extension point = “org.eclipse.ui.acceleratorSets” > <acceleratorSet configurationId = “org.eclipse.ui.exampleAcceleratorConfiguration” scopeId = “org.eclipse.ui.globalScope” > <accelerator id = “org.eclipse.ui。ExampleActionA“ key = ”CTRL + R CTRL + A“ > </ accelerator> <accelerator id = ”org.eclipse.ui。ExampleActionB“ key = ”CTRL + R CTRL + B“ > </ accelerator> <accelerator id = ”org.eclipse.ui。ExampleActionC“ key = ”CTRL + R CTRL + C || CTRL + SHIFT + DELETE“ > </ accelerator> </ acceleratorSet> </ extension>

API信息：
可以为加速器组中的相同动作指定多个加速器，但仅使用一个加速器。

如果指定了语言环境和/或平台，则将使用与当前语言环境和平台更匹配的加速器。当前语言环境由API Locale.getDefault（）确定，平台由API SWT.getPlatform（）确定。如果指定了平台和/或区域设置并且它与当前区域设置和/或平台不匹配，则会丢弃加速器。如果加速器A仅定义语言环境而B仅定义平台，则使用B.如果加速器A将“ja”定义为其语言环境而B定义“ja_JP”，则在当前语言环境为“ja_JP”的情况下使用B.

如果在不同插件中的加速器集中定义了两个加速器，则所选择的加速器将取决于插件。如果插件A依赖于B，则使用B中定义的加速器。如果A和B不相互依赖，则它们将按插件ID按字母顺序排序。

如果在不同范围中定义了两个加速器，则将使用当前范围中定义的加速器。如果未在当前范围或其父级之一中定义加速器，则将其丢弃。如果在父级和子级作用域中定义了加速器，则使用子级中的加速器。

提供的实施：
工作台为Default和Emacs加速器配置提供加速器集。
