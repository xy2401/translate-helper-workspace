介绍

Eclipse的Meson功能是CDT（C / C ++开发工具）的可选功能，它增加了对使用Meson构建系统维护和构建C / C ++项目的支持。Meson构建系统类似于Autotools，因为项目在构建之前已配置。与Autotools一样，配置涉及测试支持的构建系统和工具集。配置完成后，项目将由第二个工具构建，类似于Autotools中使用的Make工具。虽然Meson支持多个Make-like工具来构建项目，但CDT Meson插件目前只支持“ninja”。

项目的配置数据存储在名为“meson.build”的文件中。该文件包含由meson命令解释的指令。指令包括使用哪些语言，创建可执行文件/目标，源，特殊配置选项以及任何特殊构建时测试等。许多测试都是自动的，因此配置文件不需要太多。

以下是meson.build文件示例：

项目（'你好'，'c'）
可执行文件（'hello'，'hello.c'）

该文件告诉meson项目名称hello是一个C项目，它创建了一个名为hello的可执行文件，它是由hello.c构成的 。Meson将找到在这个构建机器上构建这个项目所需的C编译器等。

在包含meson.build的目录中运行介子将在构建目录（或指定的构建目录）中创建一个ninja.build文件。
