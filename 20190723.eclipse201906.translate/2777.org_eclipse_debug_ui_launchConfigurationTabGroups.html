启动配置选项卡组

标识符：
org.eclipse.debug.ui.launchConfigurationTabGroups

描述：
此扩展点提供了一种机制，用于为启动配置类型的启动配置对话框提供一组选项卡。

配置标记：

<！ELEMENT 扩展 （ launchConfigurationTabGroup *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT launchConfigurationTabGroup （ launchMode *）>

<！ATTLIST launchConfigurationTabGroup

id CDATA #REQUIRED

输入IDREF #REQUIRED

class CDATA #REQUIRED

helpContextId CDATA #IMPLIED

说明CDATA #IMPLIED>

  • id - 指定此启动配置选项卡组的唯一标识符。
  • type - 指定此选项卡组适用的启动配置类型（对应于启动配置类型扩展的ID）。
  • class - 指定实现ILaunchConfigurationTabGroup的Java类的完全限定名称。
  • helpContextId - 一个可选标识符，指定与此选项卡组的启动配置类型关联的帮助上下文
  • description - 启动配置类型的描述

<！ELEMENT launchMode EMPTY>

<！ATTLIST launchMode

模式IDREF #REQUIRED

透视IDREF #IMPLIED

说明CDATA #IMPLIED>

启动模式元素指定此选项卡组特定的启动模式。选项卡组可以与一个或多个启动模式相关联。为了向后兼容（3.0之前），不需要指定启动模式。未指定时，选项卡组将注册为关联的启动配置类型的默认选项卡组（即适用于尚未提供特定选项卡组的所有支持的启动模式）。


  • mode - 此选项卡组特定的启动模式的标识符。
  • perspective - 与此启动配置类型和启动模式关联的默认透视标识符。这允许扩展在向调试插件注册相应的启动时引起透视切换（或打开）。未指定时，表示默认情况下不应出现透视切换。
  • description - 特定于此launchMode的启动配置类型的描述。

例子：
以下是启动配置选项卡组扩展点的示例：

<extension point = “org.eclipse.debug.ui.launchConfigurationTabGroups” > <launchConfigurationTabGroup id = “com.example。ExampleTabGroup“ type = ”com.example。ExampleLaunchConfigurationTypeIdentifier“ class = ”com.example。ExampleLaunchConfigurationTabGroupClass“ > </ launchConfigurationTabGroup > </ extension>
在上面的示例中，将为具有标识符com.example的启动配置类型显示提供的选项卡组。ExampleLaunchConfigurationTypeIdentifier 。

API信息：
属性class值必须是实现接口org.eclipse.debug.ui.的Java类的完全限定名org.eclipse.debug.ui.ILaunchConfigurationTabGroup 。
