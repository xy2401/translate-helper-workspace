创作巫师

标识符：
org.eclipse.ui.newWizards

描述：
此扩展点用于注册资源创建向导扩展。创建向导在“新建对话框”中显示为选项，通常用于创建文件夹和文件。

在“新建对话框”中，向导被组织成通常反映特定问题域的类别。例如，面向Java的插件可以定义称为“Java”的类别，该类别适用于“类”或“包”创建向导。一个插件定义的类别可以使用category属性由其他插件引用。未分类的向导以及具有无效类别路径的向导将最终出现在“其他”类别中。

向导可以选择指定描述子元素，其主体应包含有关向导的简短文本。

配置标记：

<！ELEMENT 扩展 （ 类别 | 向导 | primaryWizard ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

id CDATA #REQUIRED

名称CDATA #REQUIRED

parentCategory IDREF #IMPLIED>

  • id - 可用于标识此类别的唯一名称
  • name - 将在对话框中使用的类别的可翻译名称
  • parentCategory - 如果应将此类别添加为子类，则为另一个类别的路径

<！ELEMENT 向导 （ 描述 ？， 选择 *， 关键字参考 *）>

<！ATTLIST向导

id CDATA #REQUIRED

名称CDATA #REQUIRED

icon CDATA #IMPLIED

类别CDATA #IMPLIED

class CDATA #REQUIRED

project（true | false）

finalPerspective IDREF #IMPLIED

preferredPerspectives CDATA #IMPLIED

helpHref CDATA #IMPLIED

descriptionImage CDATA #IMPLIED

canFinishEarly（true | false）

hasPages（true | false）>

  • id - 可用于标识此向导的唯一名称
  • name - 将在对话框中使用的向导的可翻译名称
  • icon - 将与名称一起使用的图标的相对路径，以将向导表示为创建对话框中的选项之一。
  • category - 类别ID的斜杠分隔路径（'/'）。路径中的每个标记必须表示此前或其他插件先前定义的有效类别ID。如果省略，则向导将添加到“其他”类别。
  • class - 实现org.eclipse.ui的Java类的完全限定名称。INewWizard 。
  • project - 一个可选属性，指示向导将创建IProject资源。还会导致向导在“新建项目对话框”中显示为选项。
  • finalPerspective - 一个可选属性，用于标识在完成IProject资源创建时要激活的透视图。
  • preferredPerspectives - 一个可选属性，指定以逗号分隔的透视ID列表。如果当前透视图位于此列表中，则在完成IProject资源创建时不会发生透视激活。
  • helpHref - 一个可以详细描述该向导的帮助URL。

    从3.0开始

  • descriptionImage - 一个更大的图像，可以帮助描述此向导。

    从3.0开始

  • canFinishEarly - 向导是否能够在不向用户显示页面的情况下完成。
  • hasPages - 向导是否提供任何页面。

<！元素描述 （#PCDATA）>

一个可选的子元素，其主体包含一个简短的文本，描述了启动时向导将执行的操作



<！ELEMENT 选择 EMPTY>

<！ATTLIST选择

class CDATA #REQUIRED

名称CDATA #IMPLIED>

  • class - 完全限定的类名。如果工作台选择中的每个对象都实现此接口，则选择将传递给向导。否则，传递空选择
  • name - 可选的名称过滤器。工作台选择中的每个对象都必须与要传递给向导的名称过滤器匹配

<！ELEMENT primaryWizard EMPTY>

<！ATTLIST primaryWizard

id IDREF #REQUIRED>

一种声明向导在UI中是“主要”的方法。新向导对话框中强调了主向导。请注意，此元素不适合插件开发人员使用！存在此元素，以便产品经理可以强调其产品的一组向导。


  • id - 应该成为主要向导的id。

<！ELEMENT keywordReference EMPTY>

<！ATTLIST keywordReference

id IDREF #REQUIRED>

“新建向导”对关键字的引用。请参阅关键字扩展点。


  • id - 要引用的关键字的ID。

例子：
以下是创建向导配置的示例：

<extension point = “org.eclipse.ui.newWizards” > <category id = “com.xyz。XYZ“ name = ”XYZ Wizards“ > </ category> <category id = ”com.xyz。XYZ.Web“ name = ”Web向导“ parentCategory = ”com.xyz。XYZ“ > </ category> <wizard id = ”com.xyz.wizard1“ name = ”XYZ artifact“ category = ”com.xyz。XYZ / com.xyz。XYZ.Web“ icon = ”./icons/XYZwizard1.png“ class = ”com.xyz。XYZWizard1“ > <description>创建一个简单的XYZ工件并设置初始内容</ description> <selection class = ”org.eclipse.core.resources。IResource“ /> </ wizard> </ extension>

API信息：
class属性的值必须表示实现org.eclipse.ui的类。INewWizard 。如果从“新建向导”中创建向导，则会将其插入现有向导中。如果向导作为快捷方式启动（从“文件新建”菜单或工具栏按钮），它将独立显示为单独的对话框。

提供的实施：
工作台附带了向导，用于创建以下类型的空资源：项目，文件夹和文件。使用与上述相同的机制注册这些向导。根据特定的平台安装，还可能出现其他向导。
