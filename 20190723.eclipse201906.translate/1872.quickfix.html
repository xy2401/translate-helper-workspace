OT / J快速修复

特定的快速修复程序可用于修复与OT / J相关的问题，包括：
类修饰符
- 缺少team修改器，
- 角色的非法修饰符，
- 降低（显式或隐式）继承角色的可见性，
- 抽象相关角色（ 2.5（b） ）。
方法和方法绑定的修饰符
- 方法的callin修饰符和callin绑定的修饰符（ before/after/replace ）之间不匹配，
- callin替换绑定中的静态/非静态不兼容性，
- callin与常规角色方法的不匹配覆盖（ 4.2（d） ），
- 缺少callin绑定修饰符（ before/after/replace ）。
- 错误的标注令牌（ -> vs. => ， 3.1（e） ）。
- 角色方法/构造函数/字段的可见性不足。
- 可以修复速记标注绑定的可见性不足，可以预先设置可见性修饰符（受保护或公开，请参阅 3.1（i） ）。
callout / callin方法绑定
- 实现推断的标注绑定（ 3.1（j） ， 3.5（h） ）
- 缺少callin优先声明（ 4.8 ）。
- 在callout / callin方法绑定中添加/删除签名 （ 3.1（c） ， 4.1（c） ）。
  如果无法解析其中一个方法规范，则此方法可用作quickfix，否则此更改将归类为快速辅助 。
打字
- 为外化角色的类型添加缺少的锚点（ 1.2.2 ， 9.2 ）（根据上下文推断提案），
- 将缺少的final修饰符添加到用作类型锚点的变量中，
- 将所需的类型参数添加到捕获协变基本方法的callin绑定中（ 4.9.3 ）。
角色文件
- 添加引用团队角色文件的@role javadoc标签（ 1.2.5.d ）。
编码风格
- 将导入更改为“ import base ...”，（ 2.1.2（d） ）
- 将对角色基类的限定引用更改为使用简单名称和“ import base ...”，
- 从“ anchor.R “到” R<@anchor> “（ 1.2.2（b） ）。
- 向角色类添加缺少的@Override注释，该角色类覆盖超级团队中的相应角色（ 1.3.1（c） ）。
- 向角色方法添加缺少的@Override注释，该方法覆盖隐式超级角色中的相应方法。
此外，一些标准的Java快速修正和内容辅助也已扩展为适用于OT / J元素。
抑制警告
- @SuppressWarnings注释也可以添加到方法绑定中。
创建方法
- 也适用于callout / callin绑定中未解决的方法规范。
- 生成继承的抽象方法也尊重隐式继承。
组织进口
- 如果合适，创建“ import base ...”
