Java元素过滤器扩展

标识符：
org.eclipse.jdt.ui.javaElementFilters

描述：
此扩展点用于使用筛选器扩展Java UI视图。

配置标记：

<！ELEMENT 扩展 （ 过滤器 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 过滤器 EMPTY>

<！ATTLIST过滤器

id CDATA #IMPLIED

名称CDATA #IMPLIED

描述CDATA #IMPLIED

targetId CDATA #IMPLIED

启用（true | false）

模式CDATA #IMPLIED

class CDATA #IMPLIED>

  • id - 将用于标识此过滤器的唯一ID
  • name - 允许在UI中标识此过滤器的唯一名称。此属性应为已翻译的字符串。虽然模式过滤器（即使用pattern属性的那些）不需要此属性，但我们建议提供名称，否则模式字符串本身将用于表示UI中的过滤器。
  • description - 此过滤器的简短描述
  • targetId - 提供此过滤器的目标的ID。如果缺少此属性，则过滤器将提供给使用org.eclipse.jdt.ui.actions.customFiltersActionGroup所有视图。这将替换已弃用的属性“viewId”。
  • enabled - 如果此属性为“true”或此属性不存在，则将启用过滤器。用户很可能在UI中覆盖此设置。
  • pattern - 将隐藏名称与此模式匹配的元素。此属性在此处是为了向后兼容，不应再使用。允许插入过滤器的所有视图也允许直接通过UI添加模式过滤器。
  • class - 用于过滤视图的类的名称。该类必须扩展org.eclipse.jface.viewers.ViewerFilter 。如果此属性在此处，则不得提供pattern属性。

例子：
以下是Java元素过滤器定义的示例。它过滤掉内部类并且最初被选中。 


 <extension point="org.eclipse.jdt.ui.javaElementFilters">
  <filter
   id="org.eclipse.jdt.ui.PackageExplorer.LibraryFilter"
   name="%HideReferencedLibraries.label"
   description="%HideReferencedLibraries.description"
   targetId="org.eclipse.jdt.ui.PackageExplorer"
   class="org.eclipse.jdt.internal.ui.filters.LibraryFilter"
   enabled="false">
  </filter>
 </extension>
