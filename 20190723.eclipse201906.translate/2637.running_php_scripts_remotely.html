远程运行PHP脚本

此过程描述如何使用服务器的调试器在工作区中运行文件。如果要在生产服务器上“实时”测试文件的执行，请使用此功能。如果您的服务器已加载扩展，这尤其重要。

注意：

您的服务器必须运行Xdebug或Zend Debugger才能使远程调试功能正常运行。

要远程运行PHP脚本：

  1. 单击“运行”按钮旁边的箭头在工具栏上，选择“运行配置” - 或 - 转到“运行”|运行配置 。“运行”对话框将打开。
  2. 双击PHP Script选项以创建新的运行配置。
    1。输入新配置的名称。
  3. 选择Debugger Location类别下的PHP Web Server选项。
  4. 从列表中选择您的服务器。如果尚未配置服务器，请单击带下划线的“PHP Servers”快捷方式。“服务器”首选项页面将打开。按照PHP Servers Preferences页面下的“ 添加新服务器 ”的说明配置服务器。有关配置PDT与远程服务器之间的通信的详细信息，请参阅设置远程调试 。
  5. 在PHP文件类别下，单击“浏览”并选择您的文件。
  6. 如有必要，可以在“PHP脚本参数”选项卡中添加参数以模拟命令行输入。
  7. 单击Apply，然后单击Run。

您的脚本将在浏览器中运行并显示。

注意：

如果文件包含对项目中未包含的文件的“包含”或“要求”调用，则必须将它们添加到项目的“包含路径”中，以便模拟生产环境。此外，如果调用使用服务器位置的绝对路径定义的文件（有关绝对文件位置的更多信息，请参阅“ 包含路径 ”），将显示“路径映射”对话框。有关更多信息，请参阅路径映射
