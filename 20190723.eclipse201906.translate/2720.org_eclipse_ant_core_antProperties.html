Ant属性

标识符：
org.eclipse.ant.core.antProperties

以来：
3.0

描述：
允许插件定义Ant属性以在Ant构建文件中使用。

配置标记：

<！ELEMENT 扩展 （ antProperty *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT antProperty EMPTY>

<！ATTLIST antProperty

名称CDATA #REQUIRED

值CDATA #IMPLIED

class CDATA #IMPLIED

无头（真|假）

eclipseRuntime（true | false）>

  • name - 属性的名称。
  • value - 属性的值。如果此处提供了值，则忽略“class”属性。
  • class - 如果未提供“value”属性，则调用此类以返回Ant属性的动态值。如果返回null ，则不设置该值。
  • headless - 指示此属性是否适合在“无头”Ant环境中使用。如果运行headless且属性为“false”，则不会设置该属性和任何指定的org.eclipse.ant.core.IAntPropertyValueProvider将不会被实例化。未指定时，隐含值为true 。
  • eclipseRuntime - 指示是否仅在与Eclipse在同一VM中运行时才考虑此属性。未指定时，隐含值为true 。

例子：
以下是Ant属性扩展点的示例：


   <extension point="org.eclipse.ant.core.antProperties">
      <antProperty name="eclipse.home"
           class="org.eclipse.ant.internal.core.AntPropertyValueProvider"/>
      <antProperty name="eclipse.running"
           value="true"/>
   </extension>

API信息：
class属性中指定的类必须实现org.eclipse.ant.core。IAntPropertyValueProvider接口。

提供的实施：
该平台使用此机制将Ant属性eclipse.home设置为Eclipse安装目录并设置eclipse.running属性。
