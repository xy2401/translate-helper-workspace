浏览器

标识符：
org.eclipse.ui.browser.browsers

以来：
3.0（最初在2.0版中添加为org.eclipse.help.ui.browser）

描述：
用于提供能够在给定URL处显示html文档的Web浏览器。

配置标记：

<！ELEMENT 扩展 （ 浏览器 ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 浏览器 （ 位置 ）*>

<！ATTLIST浏览器

id CDATA #REQUIRED

factoryclass CDATA #IMPLIED

名称CDATA #REQUIRED

os CDATA #IMPLIED

可执行的CDATA #IMPLIED>

将显示在首选项页面上的外部Web浏览器提供os属性匹配的General / Web浏览器和在至少一个指定位置中找到文件。


  • id - 浏览器的唯一ID。
  • factoryclass - 浏览器工厂的实现类。该类必须实现org.eclipse.ui.browser.BrowserFactory接口。
  • name - 浏览器的名称（可翻译）。
  • os - 逗号分隔的操作系统列表，用于在指定位置搜索此浏览器。仅使用具有匹配OS的浏览器。
  • 可执行文件 - 可执行文件名

<！ELEMENT 位置 （#PCDATA）>

默认安装位置。这些位置不应包含初始路径，因为它将替换为所有已知驱动器。例如，在Windows上，“test.exe”的位置将为所有已知驱动器查找“c：\ test.exe”，“d：\ test.exe”等。

在Windows上，路径也可以以环境变量开头，例如“%% ProgramFiles％\ folder \ test.exe”。请注意开头的双倍％以防止本地化。



例子：
以下是浏览器扩展点的示例用法：


 <extension point="org.eclipse.ui.browser.browsers">
  <browser
     id="org.eclipse.ui.browser.firefox"
     name="Firefox"
     os="Win32"
     executable="firefox.exe"
     factoryclass="org.eclipse.ui.internal.browser.browsers.MozillaFactory">
       <location>%%ProgramFiles%\Mozilla Firefox\firefox.exe</location>
       <location>%%ProgramFiles(x86)%\Mozilla Firefox\firefox.exe</location>
  </browser>
 </extension>

API信息：
提供的工厂类必须实现org.eclipse.help.browser.IBrowserFactory接口。该接口中的方法确定工厂是否在给定系统上可用，即能够提供浏览器实例，以及创建实现IBrowser接口的浏览器实例。

提供的实施：
org.eclipse.help.base和org.eclipse.help.ui插件包含常见平台上的浏览器实现。其他插件可以提供不同的实现。在首选项中，用户可以从可用的浏览器中选择默认浏览器。
