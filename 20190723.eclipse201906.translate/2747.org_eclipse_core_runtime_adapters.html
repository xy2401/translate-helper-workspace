适配器

标识符：
org.eclipse.core.runtime.adapters

以来：
3.0

描述：
适配器扩展点允许插件以声明方式注册适配器工厂。运行时XML表达式语言使用此信息来确定适配器的存在，而不会导致加载插件。通过扩展点注册适配器工厂，无需在插件启动时手动注册适配器工厂。

配置标记：

<！ELEMENT 扩展 （ 工厂 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT 工厂 （ 适配器 +）>

<！ATTLIST工厂

adaptableType CDATA #REQUIRED

class CDATA #REQUIRED>

  • adaptableType - 此工厂为其提供适配器的类的完全限定名称（通常实现IAdaptable）。
  • class - 适配器工厂类的完全限定名称。必须实现org.eclipse.core.runtime。IAdapterFactory 。

<！ELEMENT 适配器 EMPTY>

<！ATTLIST适配器

键入CDATA #REQUIRED>

  • type - 此工厂可以适应的Java类或接口的完全限定名称。

例子：
以下是适配器声明的示例。此示例声明此插件将提供适配器工厂，该工厂将IFile类型的对象调整为MyFile类型的对象。 


      <extension point="org.eclipse.core.runtime.adapters">
         <factory 
            class="com.xyz.MyFileAdapterFactory" 
            adaptableType="org.eclipse.core.resources.IFile">
            <adapter type="com.xyz.MyFile"/>
         </factory>
      </extension>

API信息：
使用此扩展点注册的适配器工厂可以使用IAdapterManager.hasAdapter方法查询，也可以使用IAdapterFactory上的某个 getAdapter方法进行检索 。使用此扩展点注册的适配器工厂不需要在运行时使用IAdapterFactory.registerAdapters进行注册。

提供的实施：
平台中的几个插件为许多不同的IAdaptable对象提供适配器。
