SCM URL导入页面

标识符：
org.eclipse.team.ui.scmUrlImportPages

以来：
3.6

描述：
实验 。此扩展点已添加为正在进行的工作的一部分。无法保证此API可以正常工作，或者在将来的版本中保持不变。未经团队团队咨询，请不要使用此API。

此扩展点用于注册页面以根据其SCM URL导入项目。

提供者可以为此扩展点提供扩展，并提供org.eclipse.team.ui.IScmUrlImportWizardPage ，允许设置和检索项目导入描述。

配置标记：

<！ELEMENT 扩展 （ scmUrlImportPage ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT scmUrlImportPage EMPTY>

<！ATTLIST scmUrlImportPage

页CDATA #REQUIRED

进口商IDREF #REQUIRED>

  • page - 向导页面，可以由向导根据SCM URL中的信息导入项目来显示。
  • importer - 与页面扩展名关联的org.eclipse.team.core.bundleImporters扩展名的id。

例子：
以下是导入页面CVS的示例：


<extension point="org.eclipse.team.ui.scmUrlImportPages">
 <scmUrlImportPage
  importer="org.eclipse.team.core.cvs.importer"
  page="org.eclipse.team.internal.ccvs.ui.wizards.CVSScmUrlImportWizardPage">
 </scmUrlImportPage>
</extension>

API信息：
页面属性的值必须是实现接口org.eclipse.team.ui的Java类的完全限定名称。IScmUrlImportWizardPage 。

提供的实施：
CVS UI提供了一个页面扩展，能够配置从CVS导入。
