运行对象团队程序

技术上运行对象团队程序涉及一个特殊的运行时环境（ OTRE - 对象团队运行时环境 ），它提供基本类并执行加载时编织，以便将方面与其基础连接起来。所有这些都是隐藏的，因为运行对象团队程序就像运行Java程序一样简单。仅在定位Java 8时，才需要选择适当的目标编织方案 。

以前版本的OTDT引入了特定的启动类型（ 对象团队应用， 对象团队Eclipse应用程序 OT / Equinox Framework）不再受支持。

要启动OT / J程序，请使用Eclipse提供的任何启动技术：例如，您可以从上下文菜单中调用Run As / Debug As选项，或使用其中一个运行/调试工具栏按钮。从这里只需选择一种标准的启动配置类型：

  • Java应用程序
  • 远程Java应用程序
  • JUnit的
  • JUnit插件测试
  • Eclipse应用程序
  • OSGi框架

所有这些启动现在始终提供一个附加复选框，显示在运行时JRE配置的旁边。

对于Java应用程序 （ ， ）此复选框位于 JRE标签：

  • 如果选中新选项，将启动应用程序，启用对象团队运行时环境（OTRE），这是将callin绑定编织到基类中所需的。相反，禁用OTRE可以运行OT / J应用程序，禁用所有callin-bindings。但请注意，更多的OT / J机制也不会起作用（如解封装）。
  • 对于OT / J项目，默认情况下启用此选项。
  • 如果另外选中了启用OTRE 将显示“ Team Activation选项卡 。
    （如果在禁用OTRE时打开了启动配置，则需要重新打开此配置（例如，取消选择并选择），以便显示新选项卡。）
  • 在幕后， 当选中“ 启用OTRE”时，将根据启动中涉及的项目配置选择正确的编织方案 。

团队激活：

我希望从基本应用程序和许多方面编写程序，您可以使用启动配置来选择应包含哪些方面。使用选项卡团队激活以选择现有团队并将其置于所需的顺序中。

结果是，在程序的主要方法执行之前，将实例化并激活列表中的每个团队。如果检查调试中的Stop是否正在进行调试，实际上可以观察到这一点：程序将在进入main 之前实际停止，下一步将执行所有待处理的团队实例化/激活，然后才启动程序（您可能希望使用团队监视器来观察团队实例化/激活）。

由于团队可以在基础应用程序中拦截方法调用，因此只需将团队添加到现有应用程序中就可以产生不同的行为。
