源路径计算机

标识符：
org.eclipse.debug.core.sourcePathComputers

以来：
3.0

描述：
定义一个扩展点以注册可以描述启动配置的默认源查找路径的计算机。源路径计算机通过launchConfigurationTypes扩展点与启动配置类型相关联。同样，源路径计算机可以通过启动配置属性ATTR_SOURCE_PATH_COMPUTER_ID与特定的启动配置相关联。

配置标记：

<！ELEMENT 扩展 （ sourcePathComputer *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT sourcePathComputer EMPTY>

<！ATTLIST sourcePathComputer

id CDATA #REQUIRED

class CDATA #REQUIRED>

定义一个扩展点以注册可以描述启动配置的默认源查找路径的计算机。


  • id - 用于引用此计算机的唯一ID。
  • class - 实现ISourcePathComputerDelegate的类。

例子：
以下是源路径计算机定义的示例：
<extension point = “org.eclipse.debug.core.sourcePathComputers” > <sourcePathComputer id = “org.eclipse.example.exampleSourcePathComputer” class = “org.eclipse.example。SourcePathComputer“ > </ sourcePathComputer> </ extension>

API信息：
属性类的值必须是实现接口ISourcePathComputer的Java类的完全限定名称。
