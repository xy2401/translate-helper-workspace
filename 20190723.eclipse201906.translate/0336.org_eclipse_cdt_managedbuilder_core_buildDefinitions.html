表示一个项目类，它充当用户将创建的项目的模板，例如，Linux静态库。项目类型包含一系列配置。项目类型以继承层次结构排列，其中项目类型从其父级继承配置列表，并可添加或覆盖此列表中的配置。


  • id - 由构建模型用于唯一标识项目类型。
  • name - 人类可读的项目类型名称，例如“可执行文件”。这将是用户在UI中显示的名称。
  • superClass - 此projectType派生自的projectType的id。
  • isAbstract - 将项目类型标记为抽象的可选字段。如果为true，则项目类型不会显示在UI中。项目类型被其他项目类型用作其“超类”。默认值为false。
  • unusedChildren - 超分类子项的子ID的分号分隔列表，不应由此元素自动继承。注意：此属性尚未实现。
  • isTest - 一个可选字段，用于将项目类型标记为仅测试。如果为true，则项目类型不会显示在UI中。例如，可以在JUnit测试中以编程方式操作项目类型。默认值为false。
  • projectEnvironmentSupplier - 指定实现IProjectEnvironmentVariableSupplier的类的名称，以便提供项目级环境变量。
  • projectMacroSupplier - 指定实现IProjectBuildMacroSupplier的类的名称，以便提供项目级构建宏。
  • configurationNameProvider - 包含实现接口的类的名称，该接口具有返回配置的默认名称的方法。用户项目中的配置名称必须是唯一的。projectType可以包含具有相同名称的配置子项。在这种情况下，必须指定configurationNameProvider，以便在“新建项目”和“新建配置”对话框中向用户显示名称之前使其成为唯一名称。
  • convertToId - projectType的标识符，使用此定义加载的项目类型应转换为。MBS将调用适当的转换器。
  • buildProperties - 指定以逗号分隔的构建属性类型列表 - 要应用于此项目类型的所有配置的值对。属性类型 - 值对以type_id = value_id格式指定。
  • buildArtefactType - 指定Build Artefact Type的属性。可以包含“org.eclipse.cdt.build.core.buildArtefactType”构建属性的值id。构建系统提供的默认值为“org.eclipse.cdt.build.core.buildArtefactType.exe” - 表示可执行文件，“org.eclipse.cdt.build.core.buildArtefactType.staticLib” - 表示静态库，“org。 eclipse.cdt.build.core.buildArtefactType.sharedLib“ - 表示动态库自定义值可以通过”org.eclipse.cdt.managedbuilder.core.buildProperties“扩展点提供。有关更多详细信息，请参阅该扩展点的说明。指定此属性完全等同于通过buildProperties属性指定“org.eclipse.cdt.build.core.buildArtefactType”。buildArtefactType属性，“buildArtefactType”属性值优先于通过“buildProperties”属性指定的artefact类型

配置用于将某些默认工具和选项聚集在一起，以某种方式构建项目。例如，“调试”配置可能为工具提供设置为使用调试符号构建的选项，而“发布”配置将为工具提供选项集以创建最佳性能。


  • id - 配置的唯一标识符。
  • name - 要在UI中使用的配置的描述性名称。
  • parent - 克隆此配置的配置。
  • artifactName - 这是与配置关联的最终构建工件的名称。用户将指定这是UI，因此无需提供默认值。
  • artifactExtension - 这是将对配置创建的任何构建工件应用（如果需要）的扩展。
  • cleanCommand - 此属性维护用于删除特定配置的中间文件和输出文件的命令。例如，在Linuc，Solaris或Cygwin等POSIX平台上，命令为rm -rf而在Win32平台上则为del /F /S /Q
  • errorParsers - 要与此配置一起使用的默认错误解析器的以分号分隔的列表。按顺序排列列表，首先调用列表中的第一个错误解析器，然后调用第二个错误解析器，依此类推。该列表可能包含CDT定义的错误解析器和/或其他已安装的错误解析器扩展。用户可以在每个配置的基础上更改要使用的错误解析器列表。指定后，将覆盖工具链errorParsers属性。
  • languageSettingsProviders - 以分号分隔的提供者ID列表，用于实现ILanguageSettingProvider接口。可以使用$ {Toolchain}关键字使用工具链级提供程序列表修改此字段。提供者ID可以带有“ - ”前缀，这将导致从前面的列表中删除id，包括使用$ {Toolchain}关键字定义的提供者。如果未指定此字段，则为“org.eclipse.cdt.managedbuilder.core”。默认情况下使用MBSLanguageSettingsProvider“（MBS语言设置提供程序）。
  • prebuildStep - 指定预构建命令，该命令在标准MBS构建之前运行。
  • postbuildStep - 指定构建后命令，该命令在标准MBS构建之后运行。
  • preannouncebuildStep - 指定运行预构建命令步骤时要显示的字符串。
  • postannouncebuildStep - 指定运行构建后命令步骤时要显示的字符串。
  • description - 指定在创建项目和管理配置时将向用户显示的配置的描述。描述仅显示在UI中 - 它不被视为配置名称的一部分。
  • buildProperties - 指定以逗号分隔的构建属性类型列表 - 为此配置定义的值对。属性类型 - 值对以type_id = value_id格式指定。
  • buildArtefactType - 指定Build Artefact Type的属性。可以包含“org.eclipse.cdt.build.core.buildArtefactType”构建属性的值id。构建系统提供的默认值为“org.eclipse.cdt.build.core.buildArtefactType.exe” - 表示可执行文件，“org.eclipse.cdt.build.core.buildArtefactType.staticLib” - 表示静态库，“org。 eclipse.cdt.build.core.buildArtefactType.sharedLib“ - 表示动态库自定义值可以通过”org.eclipse.cdt.managedbuilder.core.buildProperties“扩展点提供。有关更多详细信息，请参阅该扩展点的说明。指定此属性完全等同于通过buildProperties属性指定“org.eclipse.cdt.build.core.buildArtefactType”。buildArtefactType属性，“buildArtefactType”属性值优先于通过“buildProperties”属性指定的artefact类型

工具集成商定义的有序工具集，用于将项目资源转换为项目的输出。工具链可以定义为配置的一部分，也可以定义为通过工具链superClass属性从单独配置引用的独立规范。


  • id - 工具链的唯一标识符。
  • name - 要在UI中使用的工具链的描述性名称。
  • superClass - 此toolChain派生自的toolChain的id。
  • isAbstract - 一个可选字段，用于将工具链标记为抽象。抽象工具链必须在模型定义中定义为顶级对象，并且用户不能在UI中选择，但是从此工具链派生的工具链将继承其属性和子项。默认值为false。
  • unusedChildren - 超分类子项的子ID的分号分隔列表，不应由此元素自动继承。注意：此属性尚未实现。
  • osList - 此字段列出运行工具链的主机操作系统，托管构建系统使用该主机操作系统来确定工具链何时适用。该值应为逗号分隔列表。有效值是Eclipse API Platform.getOS（）返回的字符串，包括“win32”，“linux”，“solaris”，“hpux”，“aix”等字符串。请勿指定此属性或指定“all”以允许所有主机操作系统。
  • archList - 此字段列出工具链运行的主机体系结构，托管构建系统使用该体系结构来确定工具链何时适用。该值应为逗号分隔列表。有效值是Eclipse API Platform.getOSArch（）返回的字符串，包括“ia32”之类的字符串。请勿指定此属性或指定“all”以允许所有主机体系结构。
  • errorParsers - 要与此工具链一起使用的默认错误解析器的分号分隔列表。按顺序排列列表，首先调用列表中的第一个错误解析器，然后调用第二个错误解析器，依此类推。该列表可能包含CDT定义的错误解析器和/或其他已安装的错误解析器扩展。指定时，将覆盖工具链的工具子项和工具链的构建器子项的工具errorParsers属性。
  • languageSettingsProviders - 以分号分隔的提供者ID列表，用于实现ILanguageSettingProvider接口。可以在相应属性中的配置级别上调整此列表。
  • scannerConfigDiscoveryProfileId - 指定扫描仪配置发现配置文件的ID，用于收集工具链的内置编译器设置。
  • targetTool - 指定可以创建最终构建工件（构建的最终目标）的工具的分号分隔列表。使用配置中找到的第一个工具。例如，当工具链具有针对不同项目性质的不同工具时，需要列表。
  • secondaryOutputs - 除了targetTool的主outputType之外，其他outputType的ID的分号分隔列表，也被视为构建工件。构建文件生成器将确保构建输出。
  • isToolChainSupported - 指定实现IManagedIsToolChainSupported的类的名称。这提供了一种方法，用于确定系统上当前是否安装了对工具链的支持。MBS使用此信息来过滤提交给CDT用户的选择，并在未安装项目所需的支持时通知用户。如果工具链定义未提供isToolChainSupported回调，则工具链将被视为受支持。如果不支持为给定项目类型定义的所有配置，则将项目类型视为不受支持。
  • configurationEnvironmentSupplier - 指定实现IConfigurationEnvironmentVariableSupplier的类的名称，以便提供配置级环境变量。
  • configurationMacroSupplier - 指定实现IConfigurationBuildMacroSupplier的类的名称，以便提供配置级别的构建宏。
  • versionsSupported - 指定此工具链的版本分隔的逗号分隔列表，可以在不调用转换器的情况下加载。
  • convertToId - 工具链的标识符，使用此定义加载的工具链应转换为。MBS将调用适当的转换器。
  • optionPathConverter - 基于Cygwin的Gnu工具之类的工具链可以接受对OS平台无效的路径。例如，“/ usr / include”对于Cygwin gcc编译器来说是很好的，而它对于java.io来说不是一个有意义的路径。文件。因此，工具链可以指定pathConverter，该路径转换器将应用于托管构建系统中配置的包含和库路径。工具链的pathConverter适用于工具链的所有工具，除非工具定义了它自己的pathConverter。在这种情况下，工具链提供的pathConverter将被忽略。
  • supportsManagedBuild - 指定工具链是否支持托管构建。默认值为true。
  • isSystem - 系统使用系统元素满足特定需求，并且不在UI中显示

存储与整个配置不同的各个资源的构建属性的位置。

  • toolsToInvoke - 标识由分号分隔的子工具ID列表调用的工具。适用范围如下：1。默认为找到的订单中的所有工具2。使用指定的有序子列表来调用3。如果为空字符串，则将其视为没有资源配置，即使用项目级别工具。

  • icon - 要显示的16x16像素大图标的路径，而不是默认图标。该路径相对于定义.buildDefinitions的插件目录。
  • versionsSupported - 指定此工具的逗号分隔列表，可以在不调用转换器的情况下加载。
