文档设置

标识符：
org.eclipse.core.filebuffers.documentSetup

以来：
3.0

描述：
此扩展点用于通过此插件的默认文本文件缓冲区管理器为文本文件缓冲区管理器自定义文档的初始化过程。它允许指定在为特定文件内容类型，文件扩展名或文件名创建的文本文件缓冲区的初始化过程中应该参与哪个文档设置参与者。

注意：参与者必须意识到其他参与者的存在，即他们应该始终以不干扰他人的方式设置文档。例如，当参与者想要在文档上安装分区时，它必须使用org.eclipse.jface.text.IDocumentExtension3 API并选择一个唯一的分区ID。从3.2开始，如果不遵循此警告，则会向日志文件写入警告。

配置标记：

<！ELEMENT 扩展 （ 参与者 ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 参与者 EMPTY>

<！ATTLIST参与者

class CDATA #REQUIRED

扩展CDATA #IMPLIED

fileNames CDATA #IMPLIED

contentTypeId CDATA #IMPLIED>

文档设置参与者的规范。为了找到给定文件的所有参与者，按以下顺序查询每个参与者规范的属性：contentTypeId，fileNames，extensions。如果找到多个参与者，则不指定调用它们的顺序。


  • class - 参与者实现类的完全限定名称。该类必须实现org.eclipse.core.filebuffers.IDocumentSetupParticipant接口。
  • extensions - 以逗号分隔的文件扩展名列表，应使用此参与者
  • fileNames - 应使用此参与者的以逗号分隔的文件名列表
  • contentTypeId - 应使用此参与者的org.eclipse.core.runtime.contentTypes扩展点定义的内容类型的ID。

例子：

   <extension
         id="JavaDocumentSetupParticipant"
         name="%javaDocumentSetupParticipant"
         point="org.eclipse.core.filebuffers.documentSetup">
      <participant
            extensions="java"
            class="org.eclipse.jdt.internal.ui.javaeditor.JavaDocumentSetupParticipant">
      </participant>
   </extension>

API信息：
文档设置参与者必须实现org.eclipse.core.filebuffers。IDocumentSetupParticipant。
