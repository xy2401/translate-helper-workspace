使用户界面可访问的提示

以下是一系列使应用程序与辅助功能程序兼容的技巧，例如Windows提供的辅助功能界面。

使用组而不是标签。
如果使用Label标记一组相关小部件，请删除标签并将其父组合替换为其文本与标题Label相同的组 。
 
避免中间复合材料。
辅助功能工具将尽可能读取具有焦点的窗口小部件的父层次结构。确保树中的任何位置都没有没有文本的小部件。
 
使用只读文本而不是标签。
可以使用键盘访问文本，如果您希望键盘导航可以访问标签中的信息，则应使用该文本 。请注意，文本旁边的标签将被视为标题，因此如果您有标题/值对，则只需要将值小部件设为文本 。
 
阅读并理解IBM核对表。
IBM提供了一个有用的清单，以便在http://www.ibm.com/able/guidelines/software/accesssoftware.html上获得良好的可访问性
 
将助记符分配给所有菜单和菜单项。
确保它们在给定菜单中是唯一的。如果菜单是由多个插件动态组合的，那么最好不要分配助记符，因为通常无法避免冲突（例如“ 文件”>“新建”列表或“ 窗口”>“显示视图”列表）
 
将助记符分配给对话框/首选项页面/属性页面中的所有控件标签（例如按钮，复选框，单选按钮等）
确保它们在对话框中是唯一的。小心避免与默认按钮发生冲突（例如，恢复和默认值，并在首选项页面中应用;＆向下，向后和向导完成向导）。不要将助记符分配给“确定”和“取消”按钮。如果你确定shell的默认按钮，并且取消等同于关闭shell，那么默认情况下Enter和Esc映射到这些。一般用Esc或Enter做一些事情是个坏主意。
 
确保没有标签的控件前面有标签。
如果控件没有自己的标签（例如文本字段），请使用以“：”结尾的在前标签并为其指定助记符。当控件具有焦点时，像JAWS这样的屏幕阅读器将读取此标签（请参见窗口>首选项>常规 ）
 
避免使用额外的独立标签。
您无法使用键盘和JAWS等屏幕阅读器导航到独立标签，因为它们不会引起注意
 
不要将助记符分配给主窗口中的控件。
不要将助记符放在主窗口中的控件上（主菜单和主菜单项除外），即使它看起来像一个对话框（例如org.eclipse.ui.forms中的表单编辑器），因为这些通常会与菜单助记符冲突
 
为常用功能（以及仅常用功能）分配快捷键。
目前只有两种方法可以在SWT中挂钩快捷键：
  • 通过在主菜单栏中的菜单项上设置加速器（在上下文菜单中忽略它们） - JFace操作支持此功能
  • 通过在特定控件上挂钩关键监听器（例如，在视图或编辑器的实现中）
从General> Keys首选项页面查阅Eclipse SDK中提供的Eclipse SDK快捷键表，以避免冲突。
 
避免使用Alt + {key}，Ctrl + Alt + {key}和Ctrl + Space + {key}组合。
  • Alt + {key}组合可能与菜单助记符冲突
  • Ctrl + Alt + {key}组合可能与在国际键盘上输入特殊字符冲突（alt Gr = Ctrl + Alt）
  • Ctrl + Space + {key}组合可能与Ctrl-Space冲突，用于亚洲语言的模式切换。
尝试保存导航上下文。
例如，在Window> Preferences中 ，我们现在记住您最后选择的页面。这避免了每次都必须在列表中导航
 
指派团队中的特定人员负责项目的可访问性。
一切重要的东西都需要倡导者。确保团队中的每个人都知道良好的可访问性至关重要，并愿意为此人提供全面的合作。
 
测试可访问性。
让您的团队偶尔“拔掉您的鼠标日”，他们尝试仅使用键盘来使用产品。如果您正在开发Window，请获取JAWS TM的副本并确保您的UI可用于它
