在写作时，Mylyn捆绑了主要的EPP发行版（ jee，java，cpp ）。如果您希望手动安装Mylyn，则有两种方法，具体取决于Eclipse的版本。方法1概述了如何使用Eclipse 3.4更新管理器进行安装。下面的方法2描述了如何使用Update Manager将Mylyn安装到Eclipse 3.3及更低版本中。

安装 - Eclipse 3.4及更高版本

  1. 选择“帮助”>“软件更新...”
  2. 选择可用软件选项卡
  3. 按添加站点...按钮
  4. 输入Mylyn更新站点网址：
    1. http://download.eclipse.org/mylyn/releases/latest
    2. 其他扩展更新站点来自下载页面
  5. 按OK后，站点列表中将显示更新站点
  6. 展开Mylyn更新站点节点，然后选择所需的所有组件
  7. 按安装...按钮安装Mylyn

  • 发布版本：查看项目计划

需要什么版本的Java？

每个Mylyn版本都有自己的指定Java要求，可以在版本的评论页面上找到。例如， Mylyn 3.19需要Java 7或更高版本。

要检查启动了Eclipse的Java虚拟机的版本，请转到“ 帮助”→“关于Eclipse SDK”→“配置详细信息”，并验证它是否满足这些要求。

Mac用户应参阅有关错误1163477的最新评论，以获取有关如何更改1.4默认值的说明。

  • 如果您没有所需的Java版本，可以从Oracle的网站下载。
  • 如果您有多个VM，则需要指定Eclipse应使用正确的JDK VM。

在Unix中，将环境变量JAVA_HOME为JDK安装的根目录和/或设置PATH变量以将JDK可执行目录放在任何其他VM可执行目录之前。例如，在Unix下的bash下：

export JAVA_HOME="(location of JDK root)"
export PATH=$JAVA_HOME/bin:$PATH

我们不建议在Eclipse 3.1上使用JDK 1.6。 （它适用于Eclipse 3.2或3.3。）要在Eclipse 3.1上使用JDK 1.6，必须在config.ini文件中添加以下行：

org.osgi.framework.executionenvironment=OSGi/Minimum-1.0,OSGi/Minimum-1.1,JRE-1.1,J2SE-1.2,
J2SE-1.3,J2SE-1.4,J2SE-1.5,JavaSE-1.6

什么版本的Mylyn随Eclipse下载一起发布？

默认的Eclipse下载包含以下Mylyn重新分发。由于重新分发的版本可能缺少重要的错误修复或功能添加，我们建议使用最新版本的Mylyn 。

  • 适用于Java开发人员的Eclipse IDE：除了Team集成（例如自动更改集）集成之外的所有Mylyn。安装Eclipse CVS集成，然后安装最新的Mylyn构建以获取此组件。
  • 用于Java EE开发人员的Eclipse IDE：所有Mylyn重新分发，手动安装
  • 用于C / C ++开发人员的Eclipse IDE：没有Mylyn重新分发，请手动安装
  • Eclipse for RCP / Plug-in Developers：所有Mylyn都重新分发
  • Eclipse Classic：没有Mylyn重新分发，请手动安装

两种最有可能的可能性是：

  1. 您没有打开任何Mylyn视图。选择Window→Show View→Other ，然后选择Mylyn ，您应该看到可用的Mylyn Views。
  2. 如果您仍然没有看到任何内容，那么您可能没有使用所需的JDK VM。请参阅配置Java 。

这可能意味着您正在运行Linux，并且没有将Eclipse和Web浏览器配置为协同工作。请参阅在Linux上安装 。

  1. 使用“帮助”>“软件更新”>“更新”以确保安装了所有功能的最新版本
  2. 再次安装

Eclipse 3.3及更早版本的安装故障排除

更新失败

首先，尝试通过帮助→软件更新→搜索新功能再次运行更新...并确保所有Mylyn功能都已更新。

在早于3.3的Eclipse版本（最终版）上，在更新Mylyn时仅使用“搜索新功能...”选项。如果使用“搜索更新...”，Update Manager将允许部分安装，这可能导致Mylyn无法启动，并且您将需要再次运行更新。有关详细信息，请参阅下面的功能配置问如果您遇到此问题，请考虑投票使用Platform bug 132450 。

如果您已将Eclipse 3.3更新为Eclipse 3.4里程碑 ，则无法更新3.3版本，因为Mylyn有两个单独的Eclipse 3.3和3.4下载。此外，并非所有3.3版本的Mylyn都可以在Eclipse 3.4中使用。安装最新的3.4版本： http ： //www.eclipse.org/mylyn/downloads/

更新JEE Eclipse包时出现Java Persistence API Tools错误

如果您尝试安装其他功能并获得此错误，则会遇到错误194959 ，应尽快解决。解决方法是检查Europa Discovery Site并安装Data Tools Platform的前两个组件。

Subclipse相关问题

如果您看到以下消息：

  Subclipse Mylyar Integration (1.0.1) requires plug-in "org.eclipse.mylar.tasks.core (0.9.2)",
  or later version

您需要卸载Subclipse和Mylar集成的旧版（2.0之前版本）。大多数用户不需要这样做，因为旧的Mylar 1.x版本在更新到2.0后自行禁用。但如果你看到这个错误卸载通过：

  • 帮助→软件更新→管理配置
  • 卸载Subclipse Mylar Integration（1.0.1）
  • 卸载旧版本的Mylar

不兼容的VM（例如JDK 1.4）

如果您使用的是错误的VM，则会在日志文件中看到如下错误。

Root exception: java.lang.UnsupportedClassVersionError:
org/eclipse/mylar/tasklist/MylarTasklistPlugin (Unsupported major.minor version 49.0)

请参阅配置Java以解决此问题。

不兼容的Eclipse版本

Eclipse 3.1和3.2 ）存在单独的版本和更新站点，在这种情况下，您可能会在<workspace>/.metadata/.log文件或Mylyn视图中看到如下错误：

Mylyn功能配置问题

如果上面没有解决问题，最简单的方法是卸载任何旧版本并更新到最新的Mylyn。您的任务不会丢失，因为默认情况下它们存储在<workspace>/.metadata/.mylyn文件夹中，该文件夹将在下次Mylyn正确启动时读取。

  • 首先，使用帮助→软件更新→管理配置卸载旧版本的Mylyn。
  • 您需要通过右键单击禁用所有Mylyn功能。
  • 禁用Eclipse后允许Eclipse重新启动。
  • 重新启动后，请确保按下第3个工具栏按钮（如下图所示），以便您可以看到要卸载的已禁用功能。
  • 使用弹出菜单卸载所有已禁用的功能。

如果您不卸载，Update Manager会认为您拥有最新版本并告诉您没有更新。

注意：手动删除插件和功能可能会导致配置错误。

卸载后，通过添加下载页面上指定的正确更新站点来更新Eclipse，然后自动或手动更新将安装正确的版本。

在安装中我需要做什么才能将Mylyn任务管理功能与bug /任务/问题跟踪器一起使用？

安装时，请确保为错误/任务/问题跟踪软件选择连接器。例如，要使用Bugzilla，您必须安装Bugzilla连接器组件。

错误“搜索过程中遇到的网络连接问题”是什么意思？

如果您在尝试下载Mylyn时收到该消息，则表示Eclipse无法找到您输入的位置。这可能是因为您错误地复制了某些内容（注意额外的字符 - 甚至额外的空格可能会导致错误），或者因为网站出现故障。您可以通过将URL复制到Web浏览器来查看站点是启动还是关闭。

“更新管理器失败”消息的含义是什么？

这意味着Eclipse无法访问更新站点，或者它对Eclipse的配置状态感到困惑。首先尝试再次更新以查看更新站点是否可访问。

如果您尝试更新JIRA连接器，您也可以尝试取消选择该功能，以防Tigris.org更新站点无法访问。使用搜索新功能进行安装...安装时可以帮助避免此问题。如果不起作用，请参阅下面的功能配置疑难解答。

    • 名称： Mylyn for Eclipse 3.3

在Linux上安装

如何让SWT内部浏览器在Linux下运行？

Mylyn使用标准窗口小部件工具包浏览器，用户在Linux上遇到SWT浏览器问题。这不是Mylyn特定的问题，如果您尝试使用Eclipse的浏览器视图也会出现这种情况。要测试您的浏览器是否配置正确，请选择窗口→显示视图→其他→常规→内部Web浏览器 ，然后尝试加载网页。如果内部浏览器存在问题，请考虑通过Window→Preferences→General→Web Browser启用外部默认浏览器（即Firefox），然后选择Use external Web browser选项。

我收到了“无法创建浏览器页面：没有更多句柄”错误

当浏览器没有正确配置例外时，例如“无法创建浏览器页面：没有更多句柄（ java.lang.UnsatisfiedLinkError: … ）“将在尝试打开任务时出现。请参阅SWT浏览器指南，了解哪些浏览器可以使用。

对于使用Sun JVM的Linux性能不稳定的用户，请尝试使用IBM JVM ，这需要您在下载之前向IBM注册。我们也有使用JRockit JVM的好报告。

  • $ HOME / .kde /股/配置/的gtkrc-2.0

解决KDE环境变量设置问题

以下大多数已知的UI问题都是由于环境变量设置中断。环境变量GTK2_RC_FILES包含一个搜索路径，用于查找GTK应用程序要使用的GTK配置文件，并且可以使用

 env | grep GTK2_RC_FILES

通过获得正确的设置

 export GTK2_RC_FILES=$HOME/.gtkrc-2.0-kde:$HOME/.kde/share/config/gtkrc-2.0 # Bourne shell
 setenv GTK2_RC_FILES $HOME/.gtkrc-2.0-kde:$HOME/.kde/share/config/gtkrc-2.0 # C shell

重要说明：使用的环境设置似乎根据KDE启动应用程序的方式而有所不同：从Konsole，使用“运行命令...”，使用桌面图标等。请使用此简单脚本检查不同的方式：

 #!/bin/bash
 env | grep GTK2_RC_FILES >/tmp/GTK2_RC_FILES.env

并查看/ tmp中的结果输出。

考虑针对显示此不一致行为的分发提交错误。

为什么要看java.lang.OutOfMemoryError: PermGen space错误？

如果您的Eclipse崩溃，或者在安装Mylyn或其他插件后发现上述错误，您很可能会遇到臭名昭着的MaxPermSize错误。这不是Mylyn特定的问题，而是Sun Java VM的一般问题，如果安装了许多插件，则通常会在Eclipse 3.2及更高版本上触发。

要修复它，只需将以下内容添加到启动参数即可。这通常是您的快捷方式：

   -vmargs -XX:MaxPermSize=128m

或者到eclipse / configuration / config.ini文件：

   -XX:MaxPermSize=128m

注意：对于具有Equinox P2配置文件配置支持的Eclipse 3.4，也可以在当前P2配置文件中修改此设置。使用SDK的默认安装，请参阅：

@ config.dir /../ P2 / org.eclipse.equinox.p2.engine / profileRegistry / <名称>的.profile / <时间戳>的.profile

有关更多信息，请参阅： http ： //wiki.eclipse.org/Equinox_p2_Admin_UI_Users_Guide

如果您使用大量插件（例如WTP）并仍然出现此错误，则可能需要将数量增加到256M。请注意，在某些VM上，大小可能需要为2的幂，如果不被接受，可能会下降到默认值（例如32M）。

Eclipse 3.3.1用户：请注意，由于平台错误203325，您需要使用上面的说明，并且无法使用-launcher.设置大小-launcher.XXMaxPermSizeL ，将被忽略。

有关更多信息，请参阅Eclispe FAQ条目 。有关问题的详细信息，请参见平台错误92250 。

Mylyn的表现简介是什么？

无论您的工作空间有多大，Mylyn对Eclipse的速度或内存使用率都没有明显的影响 。您不需要增加Eclipse运行的内存量来使用Mylyn。任何性能问题都应报告为错误 。

目前的表现形象是：

  1. Mylyn仅在任务处于活动状态时运行，如果没有任务处于活动状态，则对Eclipse没有影响。
  2. 任务上下文模型具有可忽略的内存开销。
  3. 当任务处于活动状态时，需要额外的视图刷新才能根据兴趣模型更改来更新视图。在Windows上刷新非常快，但在其他平台上可能更明显，这一点不应该引人注意。
  4. 激活任务上下文的时间主要取决于Eclipse在上下文中打开编辑器所花费的时间。您可以在Mylyn首选项页面中设置要打开的编辑器的首选项（例如，设置为1将显着缩短激活时间，但也消除了使开放编辑器对应于任务上下文的好处）。您还可以在Mylyn Tasks视图下拉菜单中完全关闭编辑器管理。
  5. 如果Eclipse关闭时任务处于活动状态，则Eclipse启动速度会降低（4）。
  6. 在较慢的计算机上，Active Search视图运行的低优先级后台搜索可能会很明显。

如果您看到性能问题，这可能是一个错误，也可能是由其他Eclipse插件中的其他性能问题引起的。如果您遇到性能问题，我们建议增加Eclipse可用的内存量。这对于非常大的Java项目工作空间特别有用，JDT的元素缓存大小将与可用内存量成比例增长。我们建议使用几百个大型项目启动工作区的设置是：

 -Xmx768M -XX:MaxPermSize=128M

如果您看到内容辅助超时表明Mylyn提议计算机没有足够快地完成，请注意问题不在于Mylyn，而是标准内容辅助机制因计算密集而超时（例如，大量匹配）。在这种情况下，切换或禁用Mylyn提议计算机无助于提高性能，尽管它会消除这些消息。
