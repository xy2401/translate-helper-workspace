自动刷新提供程序

标识符：
org.eclipse.core.resources.refreshProviders

以来：
3.0

描述：
工作空间支持一种模式，在该模式下，文件系统中发生的更改将自动检测并与内存中的工作空间进行协调。默认情况下，这是通过创建轮询文件系统并定期搜索更改的监视器来完成的。监视工厂扩展点允许客户端创建更高效的监视器，通常是通过挂钩到某些本机文件系统工具来进行更改回调。

配置标记：

<！ELEMENT 扩展 （ refreshProvider ）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT refreshProvider EMPTY>

<！ATTLIST refreshProvider

名称CDATA #REQUIRED

class CDATA #REQUIRED>

  • name - 监视工厂的可读名称
  • class - 实现org.eclipse.core.resources.refresh.的类的完全限定名称org.eclipse.core.resources.refresh.RefreshProvider 。

例子：
以下是适配器声明的示例。此示例声明此插件将提供适配器工厂，该工厂将IFile类型的对象调整为MyFile类型的对象。 

<extension id = “coolProvider” point = “org.eclipse.core.resources.refreshProviders” > <refreshProvider name = “Cool Refresh Provider” class = “com.xyz。CoolRefreshProvider“ > </ refreshProvider> </ extension>

API信息：
刷新提供程序实现必须在org.eclipse.core.resources.refresh包中继承抽象类型RefreshProvider 。刷新请求和失败应该提供给IRefreshResult 。客户端还必须提供IRefreshMonitor的实现，工作区可以通过该实现请求卸载刷新监视器。

提供的实施：
org.eclipse.core.resources.win32片段提供了一个使用win32文件系统通知回调的本机刷新监视器。工作区还提供默认的基于天线轮询的监视器，可用于没有可用本机刷新回调的文件系统。
