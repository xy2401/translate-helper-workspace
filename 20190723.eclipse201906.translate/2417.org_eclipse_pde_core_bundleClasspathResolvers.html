Bundle Classpath Resolver

Identifier:
org.eclipse.pde.core.bundleClasspathResolvers

Since:
3.8

Description:
This extension point allows clients to add additional bundle classpath entries and source lookup path entries to a plug-in project classpath

Configuration Markup:

<!ELEMENT extension (resolver*)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT resolver EMPTY>

<!ATTLIST resolver

nature CDATA #REQUIRED

class  CDATA #REQUIRED>

  • nature - Nature of the projects this resolver is enabled for.
  • class - The class that implements this dynamic bundle classpath entry resolver. The class must implement IBundleClasspathResolver

Examples:
The following is an example of a classpath provider: 


  <extension
        point="org.eclipse.pde.core.bundleClasspathResolvers">
     <resolver
           class="org.eclipse.example.pde.SomeResolver"
           nature="org.eclipse.example.projectNature">
     </resolver>
  </extension>

API Information:
Each classpath resolver must provide a class that implements org.eclipse.pde.core.IBundleClasspathResolver interface.
