源容器类型

标识符：
org.eclipse.debug.core.sourceContainerTypes

以来：
3.0

描述：
此扩展点允许调试平台源查找工具提供可扩展的源容器类型集。

配置标记：

<！ELEMENT 扩展 （ sourceContainerType *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT sourceContainerType EMPTY>

<！ATTLIST sourceContainerType

id CDATA #REQUIRED

名称CDATA #REQUIRED

class CDATA #REQUIRED

说明CDATA #IMPLIED>

  • id - 用于引用此类型的唯一ID
  • name - 用于演示目的的此源容器类型的名称。
  • class - 实现ISourceContainerTypeDelegate的类
  • description - 此源容器的简短描述，用于演示目的。

例子：
以下是源容器类型定义的示例：
<extension point = “org.eclipse.debug.core.sourceContainerTypes” > <sourceContainerType name = “Project” class = “org.eclipse.debug.internal.core.sourcelookup.containers。ProjectSourceContainerType“ id = ”org.eclipse.debug.core.containerType.project“ description = ”工作空间中的项目“ > </ sourceContainerType> </ extension>

API信息：
属性类的值必须是实现接口ISourceContainerType的Java类的完全限定名称。
