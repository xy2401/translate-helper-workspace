Java运行时类路径条目解析器

标识符：
org.eclipse.jdt.launching.runtimeClasspathEntryResolvers

描述：
此扩展点允许客户端动态解析在运行时类路径和源查找路径上使用的条目，以用于相应的类路径变量和类路径容器。

配置标记：

<！ELEMENT 扩展 （ runtimeClasspathEntryResolver *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT runtimeClasspathEntryResolver EMPTY>

<！ATTLIST runtimeClasspathEntryResolver

id CDATA #REQUIRED

class CDATA #REQUIRED

变量CDATA #IMPLIED

容器CDATA #IMPLIED

runtimeClasspathEntryId CDATA #IMPLIED>

  • id - 可用于引用此解析程序的唯一标识符。
  • class - 实现此解析程序的类。该类必须实现IRuntimeClasspathEntryResolver 。
  • variable - 此解析程序注册的类路径变量的名称。必须至少指定一个变量或容器，并且最多可以为变量或容器注册一个解析器。
  • container - 此解析程序注册的类路径容器的标识符。必须至少指定一个变量或容器，并且最多可以为变量或容器注册一个解析器。
  • runtimeClasspathEntryId - 此解析程序与之关联的运行时类路径条目的标识符

例子：
以下是解析器的示例：

<extension point = “org.eclipse.jdt.launching.runtimeClasspathEntryResolvers” > <runtimeClasspathEntryResolver class = “com.example。ResolverImplementation“ id = ”com.example。ResolverId“ variable = ”CLASSPATH_VARIABLE“ > </ runtimeClasspathEntryResolver> </ extension>

提供的实施：
为标准JRE_LIB类路径变量和JRE_CONTAINER类路径容器提供了实现。
