超链接检测器

标识符：
org.eclipse.ui.workbench.texteditor.hyperlinkDetectors

以来：
3.3

描述：
此扩展点用于插入超链接检测器。

配置标记：

<！ELEMENT 扩展 （ hyperlinkDetector +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT hyperlinkDetector EMPTY>

<！ATTLIST的超链接检测器

id CDATA #REQUIRED

名称CDATA #REQUIRED

描述CDATA #IMPLIED

targetId IDREF #REQUIRED

class CDATA #REQUIRED

activate（true | false）“true”

modifierKeys CDATA #IMPLIED>

  • id - 将用于标识此超链接检测器的唯一ID
  • name - 允许在UI中标识此超链接检测器的名称
  • description - 此超链接检测器的可翻译描述。目前，Eclipse SDK中的任何位置都不会显示此描述。
  • targetId - 已通过org.eclipse.ui.workbench.texteditor.hyperlinkTargets扩展点注册的超链接目标的id
  • class - 实现接口org.eclipse.jface.text.hyperlink.IHyperlinkDetector 。子类org.eclipse.jface.text.hyperlink.AbstractHyperlinkDetector如果要访问目标提供的上下文。
  • activate - 一个属性，用于指示在给定目标中进行超链接检测时是否激活贡献插件
  • modifierKeys - 需要按下此超链接检测器才能激活的修饰键。有效值为：“Alt”，“Ctrl”，“Shift”，“Command”，“M1”，“M2”，“M3”，“M4”以及它们的组合由“+”分隔。请注意，不允许“Shift”本身，因为“Shift”+鼠标左键单击设置选择。 如果未指定此属性，则使用默认修饰键。

例子：
以下是超链接检测器定义的示例：

<extension
  point="org.eclipse.ui.workbench.texteditor.hyperlinkDetectors">
  <hyperlinkDetector
    id="org.eclipse.example.jdt.internal.debug.ui.JavaHyperlinkDetector"
    targetId="org.eclipse.jdt.ui.javaCode"
    class="org.eclipse.example.jdt.internal.debug.ui.JavaHyperlinkDetector"
    name="%javaHyperlinkDetector">
  </hyperlinkDetector>
</extension>
