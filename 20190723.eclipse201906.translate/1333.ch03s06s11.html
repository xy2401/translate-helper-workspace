3.6.11. Versioning Projects

  1. To create a different version of a Project, select:

    Test-->Create new version.

  2. An automatic suggestion for the next version number is provided.

  3. You can accept this version number or enter a different one.

  4. Project versions can have major, minor and micro versions as well as qualifiers. You can enter just a version number, e.g. "1" or "1.0" or "1.0.0", just a qualifier, or a version number and a qualifier, e.g. "1_release", "1.0_release" or "1.0.0_release". Project versions must be unique.

  5. Click ”OK” to create the new version.

  6. The new version of the Project becomes active in the client.

    Any test result summaries for the Project are not duplicated in the new version. Tests that ran for previous versions of the Project are, however, still in the database to be used for long term analysis.

You can also create a new version using the dbtool (Section 3.26.3.4, “Creating new versions of Projects”)
