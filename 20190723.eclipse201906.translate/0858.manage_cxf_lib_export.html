在部署时管理CXF库的导出

默认情况下，使用CXF方面配置的项目将在部署时将CXF库解析为WEB-INF / lib文件夹。

然而，这极大地增加了部署的Web模块的大小。如果要部署的Web或应用程序服务器已配置了Apache CXF库，则可以在Apache CXF首选项中禁用此选项。

要在已使用CXF Facet配置的现有Dynamic Web Project中启用或禁用此功能，请右键单击该项目，然后选择“ 属性” 。

选择Deployment Assembly属性页面并删除Apache CXF Library 。

使用CXF 2.6.0

弹簧网罐不包含在CXF 2.6.0中。需要此jar才能将使用CXF 2.6.0配置的项目部署到Web和应用程序服务器。

解决方法是使用Deployment Assembly属性页面添加依赖项。

jar文件可以在Maven 2存储库中的/ org / springframework / spring-web /目录下找到，也可以在早期或以后的CXF安装的lib目录下找到。

spring-web jar已经添加回CXF 2.6.1中的lib目录。
