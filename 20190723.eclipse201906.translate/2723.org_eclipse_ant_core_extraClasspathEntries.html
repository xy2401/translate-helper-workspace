额外的Ant类路径条目

标识符：
org.eclipse.ant.core.extraClasspathEntries

描述：
允许插件定义任意JAR以供Ant基础结构使用。这些JAR在运行时被放入Ant类路径中。除了JAR之外，提供JAR的插件的插件类加载器也被添加到类路径中。

配置标记：

<！ELEMENT 扩展 （ extraClasspathEntry *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT extraClasspathEntry EMPTY>

<！ATTLIST extraClasspathEntry

库CDATA #REQUIRED

无头（真|假）

eclipseRuntime（true | false）>

  • library - 相对于库的插件安装位置的路径。
  • headless - 指示此额外的类路径条目是否适合在“无头”Ant环境中使用。如果运行headless且属性为false ，则此条目不会添加到Ant类路径中。同样，插件类加载器不会作为父类加载器添加到Ant类加载器中。未指定时，隐含值为true 。
  • eclipseRuntime - 指示是否应该仅考虑在与Eclipse相同的VM中运行的构建的额外类路径条目。未指定时，隐含值为true。

例子：
以下是额外的类路径条目扩展点的示例：


 <extension point="org.eclipse.ant.core.extraClasspathEntries">
  <extraClasspathEntry library="myExtraLibrary.jar"/>
 </extension>

提供的实施：
平台本身提供了一个Ant支持jar（antsupportlib.jar）。
