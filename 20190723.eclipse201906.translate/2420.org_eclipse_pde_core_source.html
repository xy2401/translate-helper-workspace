来源地点

标识符：
org.eclipse.pde.core.source

以来：
2.0

描述：
此扩展点允许PDE在基于Eclipse的产品中找到的Eclipse插件中查找库的源存档。它用于提供包含源存档的位置。这些位置应包含与“plugins”目录相同的布局。

对于每个插件或片段，应存在{id} _ {version}形式的目录。目录的内容对应于插件/片段位置。它应该包含{library name} src.zip形式的源代码zip文件，其中库名是与源代码匹配的Java库的名称。对于名为“。”的JAR'd插件和库，源zip文件必须简单地命名为src.zip 。

此外，它应该包含使用src.includes变量在build.properties中指定的任何文件或目录。

配置标记：

<！ELEMENT 扩展 （ 位置 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT location EMPTY>

<！ATTLIST位置

路径CDATA #REQUIRED>

  • path - 存储源内容的贡献插件中目录的相对路径。该文件夹必须包含{id} _ {version}形式的一个或多个目录，其中id是匹配的插件或片段标识符， version是匹配的插件/片段版本。这些目录又应包含源存档以及使用相应插件/片段的build.properties文件中的src.includes变量指定的任何其他文件或文件夹。

例子：
以下是source扩展的示例：
<extension point = “org.eclipse.pde.core.source” > <location path = “src” /> </ extension>
在上面的示例中，已注册了贡献插件中的源位置src 。

API信息：
此扩展点不需要Java代码。

提供的实施：
Eclipse SDK附带了源插件，其中包含Eclipse SDK中所有插件和片段的源信息。
