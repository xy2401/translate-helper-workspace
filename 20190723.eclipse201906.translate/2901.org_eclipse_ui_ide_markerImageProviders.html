标记图像提供者

标识符：
org.eclipse.ui.ide.markerImageProviders

以来：
3.0（最初在2.1版中添加为org.eclipse.ui.markerImageProviders）

描述：
markerImageProvider扩展点是指定定义插件中标记类型的图像的点。

配置标记：

<！ELEMENT 扩展 （ imageprovider *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT imageprovider EMPTY>

<！ATTLIST imageprovider

id CDATA #REQUIRED

markertype IDREF #REQUIRED

icon CDATA #IMPLIED

class CDATA #IMPLIED>

  • id - 可用于标识此markerImageProvider的唯一名称。
  • markertype - markertype是org.eclipse.core.resources.定义的类型的id org.eclipse.core.resources.IMarker这个定义适用于。
  • icon - 如果没有定义类，则使用icon属性定义将应用于此类标记的图标。
  • class - 该类是将用于查找图像的类的完全限定名称。该类必须实现IMarkerImageProvider 。

例子：
以下是两种形式的标记图像提供者的示例。第一个是图像不会改变并直接声明的图像。对于第二个，图像必须由IMarkerImageProvider的实例确定。 

<extension point = “org.eclipse.ui.ide.markerImageProviders” > <imageprovider markertype = “org.eclipse.core.resources.taskmarker” icon = “taskicon.png” id = “myPlugin.declarativeMarkerProvider” > </ imageprovider> <imageprovider markertype = “org.eclipse.core.resources.problemmarker” class = “myPlugin。MyIMarkerImageProvider“ id = ”myPlugin.implementedMarkerProvider“ > </ imageprovider> </ extension>

API信息：
markerImageProvider目前是内部的，不应被视为API。

提供的实施：
图像提供者目前已注册书签，任务和问题标记。
