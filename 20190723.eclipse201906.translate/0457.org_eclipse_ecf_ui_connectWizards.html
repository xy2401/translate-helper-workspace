ECF连接向导

标识符：
org.eclipse.ecf.ui.connectWizards

以来：
ECF 0.9.4

描述：
此扩展点允许提供程序注册用于连接IContainer实例的向导。

配置标记：

<！ELEMENT 扩展 （ 类别 | 向导 ）*>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED

>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

id CDATA #REQUIRED

名称CDATA #REQUIRED

parentCategory CDATA #IMPLIED

>

  • id - 可用于标识此类别的唯一名称
  • name - 将在对话框中使用的类别的可翻译名称
  • parentCategory - 如果应将此类别添加为子类，则为另一个类别的路径

<！ELEMENT 向导 （ 描述 ？）>

<！ATTLIST向导

id CDATA #REQUIRED

名称CDATA #REQUIRED

类别CDATA #IMPLIED

class CDATA #REQUIRED

icon CDATA #IMPLIED

containerFactoryName CDATA #REQUIRED

>

将用于创建导出向导的元素


  • id - 将用于标识此向导的唯一名称
  • name - 将在对话框中用于表示此向导的可翻译名称
  • category - 类别ID的斜杠分隔路径（'/'）。路径中的每个标记必须表示此前或其他插件先前定义的有效类别ID。如果省略，则向导将添加到“其他”类别。
  • class - 实现org.eclipse.ecf.ui的类的完全限定名。IConnectWizard界面
  • icon - 将与导出引擎列表中的向导名称一起使用的图标的相对名称。
  • containerFactoryName - 要创建的容器的ContainerTypeName。例如，“ecf.generic.client”。

<！元素描述 （#PCDATA）>

一个可选的子元素，其主体应代表导出引擎功能的简短描述。



例子：
此扩展点的实现者必须指定org.eclipse.ecf.ui.connectWizards扩展点的扩展名。 
<extension point = “org.eclipse.ecf.ui.connectWizards” > <wizard class = “org.my.client。MyClientConnectWizard“ containerFactoryName = ”org.my.client“ id = ”my.client.connect.wizard“ name = ”My Client Connect Wizard“ icon = ”icons / connect.png“ > </ wizard> </ extension>

API信息：
“class”属性（必需）必须具有实现org.eclipse.ecf.ui的类。IConfigurationWizard界面：
