IBM并行环境

本节介绍如何使用IBM Parallel Environment将运行配置设置为PTP目标系统配置

在使用IBM PE运行并行应用程序时，首先创建一个C / C ++ / Fortran项目，然后构建它。有关更多信息，请参阅

  • 创建MPI项目简介 - 介绍创建简单项目，包括同步项目
  • 同步项目 - 有关在远程计算机上创建和构建项目的更多详细信息

如运行并行程序中所述 ，接下来通过运行>运行配置创建运行配置...并创建一个新的并行应用程序 。

使用PTP运行IBM PE并行应用程序的细节在运行配置的“ 资源”选项卡上指定。对于Target System Configuration ，选择IBM Parallel Environment

然后指定一个连接。如果您已经指定了连接，例如，对于同步项目或远程项目，它将被选中。如果要启动本地应用程序（在运行Eclipse的同一台计算机上），请为“ 连接类型”选择“ 本地 ”。

请参阅创建远程连接和本地连接

注意：要在创建连接后编辑连接信息，例如更改用户名或密码，请参阅编辑远程连接 。

选择IBM PE后，“资源”选项卡将填充特定于此目标配置的信息。此区域通常（取决于目标系统配置）包括更多选项卡，以分割您可以提供的其他信息。填写您需要的IBM PE选项。在这里，我们在第一个子选项卡Tasks上指定使用LoadLeveler调度，每个节点的任务，节点和任务的数量。

文本字段是指定在提交/启动作业时将传递给IBM PE的信息的简便方法。将鼠标悬停在文本字段上可查看有关数据值的其他信息，包括将使用您输入的值填充的环境变量。

要查看将通过环境变量等传递给目标系统的信息，请单击“资源”选项卡区域底部的“ 查看配置”按钮。

根据需要在其他选项卡上填写其他信息。例如，要填写LoadLeveler节点分配池，池ID，请使用“ 节点”选项卡。

对于没有相应文本字段或其他方式在UI中输入信息的环境变量，可以使用“ 环境”选项卡。例如，要填写LoadLeveler预留ID，请单击“ 新建”按钮创建新的环境变量。输入变量的名称和值，然后单击“ 确定”按钮。环境变量将随作业一起提交。

现在继续运行配置设置以运行并行应用程序 ，方法是在“ 应用程序”选项卡上指定应用程序，以及所需的任何其他信息。
