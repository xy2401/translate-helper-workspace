同步参与者

标识符：
org.eclipse.team.ui.synchronizeParticipants

以来：
3.0

描述：
此扩展点用于注册同步参与者。同步参与者是显示资源之间的更改的组件，通常允许用户操作更改。例如，CVS定义了工作空间同步参与者，该参与者允许显示工作空间资源与用于共享这些资源的远程位置的资源之间的更改。同步参与者通常通过同步参与者向导创建，或者可以通过插件操作创建，然后向ISynchronizeManager注册。“同步视图”显示同步参与者。

参与者是一个通用组件，提供对创建页面的访问权限，并向用户显示，以及为页面定义公共配置参数的配置。“同步视图”不会对如何向用户显示更改强制实施任何限制，而是仅管理参与者。

配置标记：

<！ELEMENT 扩展 （ 参与者 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的标准标识符
  • id - 扩展实例的可选标识符。
  • name - 此扩展实例的可选名称。

<！ELEMENT 参与者 EMPTY>

<！ATTLIST参与者

icon CDATA #IMPLIED

id CDATA #REQUIRED

class CDATA #REQUIRED

名称CDATA #REQUIRED

persistent（true | false）“false”

helpContextId CDATA #IMPLIED>

  • icon - 在列表和菜单中显示此参与者时将使用的图标。
  • id - 将用于标识此类型参与者的唯一名称。
  • class - 扩展org.eclipse.team.ui.synchronize的类的完全限定名。AbstractSynchronizeParticipant 。
  • name - 参与者的姓名。这将显示在UI中。
  • persistent - 默认情况下，参与者将在会话之间保持持久性。如果此参与者不应在会话之间保留，请将此属性设置为false。
  • helpContextId - 从3.5开始。唯一标识符，指示此参与者的帮助上下文。

例子：

<extension point = “org.eclipse.team.ui.synchronizeParticipants” > <participant name = “CVS Workspace” icon = “icons / full / cview16 / server.gif” class = “org.eclipse.team.internal.ccvs。 ui.subscriber。WorkspaceSynchronizeParticipant“ id = ”org.eclipse.team.cvs.ui.cvsworkspace-participant“ helpContextId = ”org.eclipse.team.cvs.ui.cvs_workspace_sync_view_context“ > </ participant> <participant name = ”CVS Merge“ icon = ” icons / full / obj16 / prjversions_rep.gif“ class = ”org.eclipse.team.internal.ccvs.ui.subscriber。MergeSynchronizeParticipant“ type = ”dynamic“ id = ”org.eclipse.team.cvs.ui.cvsmerge-participant“ > </ participant> </ extension>

API信息：
class属性的值必须表示实现org.eclipse.team.ui.synchronize的类。AbstractSynchronizeParticipant 。
