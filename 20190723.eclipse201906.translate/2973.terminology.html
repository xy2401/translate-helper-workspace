专业术语

当一个人用不同的术语引用相同的东西时，这可能会令人困惑，如果他们使用相同的术语引用不同的东西，则会产生彻头彻尾的误导。这就是为什么在引用注释，代码和文档中的内容时选择正确的单词很重要。

Eclipse

Eclipse是支持构建用于开发应用程序的集成工具的整个项目的名称。它代表了一系列相关项目 ，包括Eclipse平台， Java开发工具（JDT）和插件开发环境（PDE） 。
Eclipse平台
Eclipse Platform是创建插件扩展的核心框架和服务的名称。它提供了加载和运行插件的运行时。为了使用具有正确度（im）精度的术语，了解平台的一些内容是有用的。首先，Eclipse Platform本身并不是真正的产品。平台的直接消费者是工具构建者或ISV，因为他们将价值添加到Eclipse平台，使其对人们有用。
Eclipse平台分为Core和UI 。任何被归类为“UI”的东西都需要一个窗口系统，而被归类为“核心”的东西则可以“无头”。Eclipse Platform的UI部分称为Workbench 。Eclipse Platform的核心部分简称为Platform Core或Core。
所以Eclipse平台只是工具构建者构建工具插件的核心。
Eclipse SDK
Eclipse SDK是Eclipse Platform ， JDT和PDE 。除了Platform之外，SDK还提供了所需的开发工具，其中包括使Eclipse成为自己的开发环境。

平台 - “ Eclipse平台 ”的缩写 。

富客户端平台（RCP）
Eclipse Platform的一个子集，它构成了一个通用的应用程序框架，而不是构建开发工具的框架。此子集不包含Resources插件或与资源相关的任何UI。

Workbench - “Eclipse Platform UI”的缩写。

Workbench是一个高级UI框架，用于构建具有由可插入组件构建的复杂UI的产品。Workbench构建于JFace ， SWT和Platform Core之上。
核心 - “Eclipse平台核心”的缩写。
Eclipse平台的所有无UI基础架构。主要部门是：平台运行时和插件管理，工作空间和资源管理以及版本和配置管理。
运行时 - “Eclipse Platform Core Runtime”的缩写。
Platform Core的最低级别部分，负责插件注册表和插件。请注意，Platform Core Runtime不包含工作空间和资源（它们位于Resources插件中）。
工作区
工作空间是用于管理Eclipse平台中的资源的通用工具。请注意，工作空间和资源是平台的可选部分;平台的某些配置将没有工作空间。
UI - “Eclipse Platform UI”的缩写。
Eclipse平台的UI部分的全包术语。
JFace的
JFace是一个中级UI框架，可用于构建复杂的UI部分，例如属性查看器。JFace与SWT合作 。
SWT
SWT（标准窗口小部件工具包）是一个小型，快速的窗口小部件工具包，具有可移植的API和本机实现。目前，SWT可用于Windows，Linux（GTK），AIX（GTK），Solaris（GTK），HP-UX（GTK）和Mac OS X（Cocoa）。
JDT
Java开发工具（出于法律原因，小写的“开发工具”）将Java程序开发功能添加到Eclipse平台。
PDE
插件开发环境添加了用于开发Eclipse插件的专用工具。
