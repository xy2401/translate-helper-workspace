启动个人资料

OProfile插件的目的是以用户友好的方式提供有用的配置文件信息。为此，它首先收集所需的信息。插件挂钩到Eclipse和CDT的启动设施;在这样做时，分析变得像程序的正常运行一样简单。

在配置文件运行期间，OProfile插件将根据需要在后台启动，停止和执行其他OProfile任务，而正在配置的二进制文件正常运行。目前，可以在运行用户二进制文件时开始分析，并在二进制文件返回后停止，以及在任意时间启动/停止分析的能力。

一键启动

大多数用户都希望分析程序在执行期间花费最多CPU时间的位置。Eclipse的OProfile插件包含一键式启动功能，可对此进行分析。单击启动使用执行时间事件为配置文件配置设置适当的默认值，并启动配置文件而无需进一步的用户干预。

要使用一键启动，请右键单击项目 ， 二进制文件或在打开的编辑器中单击属于该项目的源文件。这里有两种方式来启动发布：

导航到性能分析工具并单击“ 使用OProfile配置文件”

或导航到分析工具 ，单击Profile Timing并将计时工具设置为Oprofile。

要正确设置首选项，您可以转到C / C ++ - > Profiling-> Categories-> Timing ，并将默认计时工具设置为Oprofile。

或者，您可以使用项目Properties-> C / C ++ General-> Profiling Categories-> Timing来设置项目特定首选项，以覆盖工作区首选项。

您还可以使用Profile as ...-> Local C / C ++ Application对您的应用程序进行概要分析，从而将Profiler选项卡中的概要分析工具设置为Oprofile。

启动自定义配置文件

OProfile插件允许您使用多个可用的OProfile选项配置配置文件运行。这些选项通过启动配置以用户友好的方式公开，允许更复杂的配置文件。

要自定义配置文件，请右键单击项目 ， 二进制文件或在打开的编辑器中单击属于该项目的源文件。然后，导航到“性能分析工具” >“ 配置文件配置...”以打开“ 配置文件配置”菜单。有关配置配置文件的详细信息，请参阅“ 配置文件配置”部分。

配置配置文件后，单击配置文件按钮以启动配置文件。

手动配置文件

从0.3.0版本开始，用户可以通过两种方式分析其应用程序：自动或手动配置文件。Automatic是先前描述的方法，其中OProfile守护程序在二进制启动之前启动并在二进制返回之后停止。手动配置文件使用适当的设置配置守护程序，但是可以使用控制对话框，用户可以使用该对话框随意启动或停止OProfile守护程序。每种模式都有自己的配置类型和图标，以消除配置文件历史记录中条目的配置文件类型。对话框中的反馈列表向用户确保已经接收到操作，因为有时操作可能需要几秒钟才能完成。此功能对于暂停输入的应用程序或者不希望多次启动和返回的大型应用程序最有用。当您认为您的应用程序花费时间时，建议不要仅使用此功能进行概要分析; OProfile的统计性质将解释这一点。

运行手动OProfile的基本工作流程是

  1. 简介 - > OProfile简介（手册）
  2. 在控制对话框中，点击“启动守护程序”
  3. 运行应用程序二进制
  4. 在控制对话框中，点击“刷新视图”（Oprofile视图应该用剖析数据刷新）
  5. 在控制对话框中，点击“停止守护进程”
