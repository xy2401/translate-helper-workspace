启动

标识符：
org.eclipse.ui.startup

以来：
2.0

描述：
此扩展点用于注册要在启动时激活的插件。作为startup元素的属性给出的类必须实现接口org.eclipse.ui。IStartup 。启动工作台后，将从单独的线程调用earlyStartup（）方法。将实例化由startup元素的class属性指定的类，并在结果上调用earlyStartup（）。不要将插件类指定为class属性的值，否则它将被实例化两次（一次通过常规插件激活，一次通过此机制）。在工作台首选项中列出了为此扩展点提供扩展的插件，用户可以在早期启动时禁用任何插件。

配置标记：

<！ELEMENT 扩展 （ 启动 +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 启动 EMPTY>

<！ATTLIST启动

class CDATA #REQUIRED>

  • class - 实现org.eclipse.ui的类的完全限定名称。IStartup 。自3.0以来必须提供。不要指定插件类，因为它将被实例化两次（一次通过常规插件激活，一次通过此机制）。

例子：
以下是启动扩展的示例：
<extension point = “org.eclipse.ui.startup” > <startup class = “org.eclipse.example。StartupClass“ /> </ extension>

API信息：
请参阅接口org.eclipse.ui。IStartup 。
