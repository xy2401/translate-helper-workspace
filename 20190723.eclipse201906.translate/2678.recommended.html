采用3.6机制和API

本节介绍在尝试更改3.5插件以采用3.6机制和API时所需的更改。

  1. 使用IResource #setDerived（boolean，IProgressMonitor）
  2. IFile＃create和IFolder＃的新原因造成了失败
  3. Workbench向导中的关键字支持
  4. 使用IWorkbench＃getModalDialogShellProvider（）

1。使用IResource #setDerived（boolean，IProgressMonitor）

从3.6开始，不推荐使用IResource#setDerived(boolean) 。应该使用IResource#setDerived(boolean, IProgressMonitor)并在随后的资源更改事件中报告此更改。

2。IFile＃create和IFolder＃的新原因造成了失败

由于3.6，还有一个新的原因时IFile#create和IFolder#create时，在一个虚拟文件夹中创建一个非虚拟资源失效，即。在#create之前进行初步验证的此API的调用者应该适应此更改并支持新案例。特别是，当#create可能失败时，创建新资源并在标题区域显示错误的向导应该通过显示新错误而不是失败并写入日志来处理该情况。

3。Workbench向导中的关键字支持

从3.6开始，New / Import / Export向导支持org.eclipse.ui.keywords扩展。建议添加关键字并与这些向导关联，以便用户可以轻松过滤

4。使用IWorkbench＃getModalDialogShellProvider（）

要获得模态对话框的最佳父shell，请使用IWorkbench#getModalDialogShellProvider() 。
