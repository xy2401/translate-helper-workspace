5.1.6.1。�商店物业/价值（一般组件）

商店物业

  • 使用此操作可以存储组件的特定属性的值。

  • 然后，您可以在测试中使用此值，或使用“检查应用程序”组件上的“数字值”和“检查字符串值”操作来检查存储的两个值是相同还是不同。

  • 每个组件都有属性（属性）。这些可以包括文本大小，组件的大小（宽度/高度），颜色等。

  • 您必须首先找到要检查的属性的名称，然后找出它应具有的值。您可以在对象映射模式中找到组件具有的属性和值（对于Swing和SWT / RCP AUT）。收集组件时，其属性信息将显示在“属性”视图中。

  • 对于使用Webdriver的HTML AUT，您可以使用此操作来测试CSS属性或元素的任何其他属性。这就是动作如何在这样的AUT上运行：

    • 首先搜索元素的属性，如果在那里找到属性，则返回其值。

    • 如果尚未找到该属性，则搜索该元素的CSS属性。

    • 如果仍未找到该属性，则使用/作为分隔符搜索HTML元素的属性。

    例如，有两种方法可以检查HTML元素的颜色：通过检查属性color或属性style/color 。请注意，返回的实际值与浏览器有很大关系，例如对于颜色，Firefox会返回类似rgb(255, 0, 0)和Chrome rgba(255, 0, 0, 0) 。

  • 参数：

    • 变量名

    • 物业名称

如果属性是私有的 ，则无法存储，远程控制组件无法访问它。

存储值（在显示文本的组件上）

  • 使用此操作从字段中读取值，以便您可以将其用作其他测试步骤的数据。

  • 参数：

    • 变量名
