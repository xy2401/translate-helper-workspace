编辑协会覆盖

标识符：
org.eclipse.ui.ide.editorAssociationOverride

以来：
3.8

描述：
允许覆盖IDE的编辑器关联以解决冲突的贡献。

有很多方法可以找到给定输入的编辑器并打开编辑器。此扩展点仅适用于通过org.eclipse.ui.ide.IDE或客户端代码通过调用org.eclipse.ui.ide.IDE 。主要目的是允许拦截如何通过IDE UI 打开编辑器，如Open ， Open With和Open Resource 。此扩展点不会影响比较编辑器。

注意：此扩展点仅应用于产品汇编时出现冲突的编辑器贡献并且没有其他方法可以解决这些冲突的情况。如果您只想将现有编辑器绑定到内容类型，请考虑使用org.eclipse.ui.editors / editorContentTypeBinding 。

配置标记：

<！ELEMENT 扩展 （ editorAssociationOverride +）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT editorAssociationOverride EMPTY>

<！ATTLIST editorAssociationOverride

id CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - 将用于标识此编辑器关联覆盖的唯一ID
  • class - 实现接口org.eclipse.ui.ide.IEditorAssociationOverride

例子：

   <extension
      point="org.eclipse.ui.ide.editorAssociationOverride">
         <editorAssociationOverride
            id="exampleOverride"
            class="example.ExampleOverride">
         </editorAssociationOverride>
   </extension>

API信息：
请参阅org.eclipse.ui.ide.IEditorAssociationOverride接口。
