偏好转移

标识符：
org.eclipse.ui.preferenceTransfer

以来：
3.1

描述：
工作台为维护首选项提供支持。此扩展点的目的是允许插件添加特定支持以保存和加载特定的首选项组。通常，这用于导入和导出等操作。

配置标记：

<！ELEMENT 扩展 （ transfer *，（ settingsTransfer *））>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>

  • point - 目标扩展点的完全限定标识符
  • id - 扩展实例的可选标识符
  • name - 扩展实例的可选名称

<！ELEMENT 转移 （ 映射 +， 描述 ？）>

<！ATTLIST转移

id CDATA #REQUIRED

名称CDATA #REQUIRED

icon CDATA #IMPLIED>

传输是用于指定可以一起导出的一组首选项的元素。


  • id - 将用于标识此传输的唯一名称
  • name - 在列出此项目时将在UI中使用的可翻译名称
  • icon - 显示传输时将使用的图标的相对名称

<！元素描述 （#PCDATA）>

一个可选的子元素，其主体应代表传输引擎功能的简短描述。



<！ELEMENT 映射 （ 条目 *）>

<！ATTLIST映射

范围CDATA #REQUIRED>

表示org.eclipse.core.runtime.preferences的子元素。IPreferenceFilter 。它指定范围org.eclipse.core.runtime.preferences中的一个或多个映射。IScope到0或更多节点。每个节点指定0个或更多个键。过滤器的导出和导入行为规则可以在org.eclipse.core.runtime.preferences中的规范中找到。IPreferenceFilter 。


  • scope - 一个属性，指定要将提供的节点和键映射到的作用域的名称。通常使用org.eclipse.core.runtime.preferences和org.eclipse.core.resources中定义的范围。它们是配置（org.eclipse.core.runtime.preferences。ConfigurationScope。SCOPE）实例（org.eclipse.core.runtime.preferences。InstanceScope。SCOPE）默认（org.eclipse.core.runtime.preferences。DefaultScope。SCOPE）默认（org.eclipse.core.resources。ProjectScope。范围）

<！ELEMENT 条目 （ 键 *）>

<！ATTLIST入门

节点CDATA #IMPLIED>

子元素，指定要为给定范围传输的节点和键


  • node - 指定要传输的给定范围内的首选项节点的属性。此首选项节点将对应于定义首选项的插件的包ID。

<！ELEMENT 键 EMPTY>

<！ATTLIST键

名称CDATA #REQUIRED

匹配（前缀）>

  • name - 指定要传输的首选项键。在未指定匹配节点的情况下指定键属性将导致运行时错误。
  • match - 如果match设置为“prefix”，则将传输首选节点上以指定名称开头的所有键。如果未指定匹配类型，则仅包括与名称相等的键。在3.6中添加。

<！ELEMENT 设置转移（ 描述 ？）>

<！ATTLIST设置转移

名称CDATA #REQUIRED

id CDATA #REQUIRED

class CDATA #REQUIRED

helpContext CDATA #IMPLIED>

设置转移是内部状态从一个工作台转移到另一个工作台的定义。在3.3中添加。


  • name - 传输的可读名称。
  • id - 传输的ID。
  • class - org.eclipse.ui.preferences.的子类org.eclipse.ui.preferences.SettingsTransfer指定此传输。
  • helpContext - 传输的帮助上下文。请参阅IWorkbench＃getHelpSystem（）。

例子：

导出所有传输的示例，导出指定范围的所有节点。 

<extension point = “org.eclipse.ui.preferenceTransfer” > <transfer icon = “XYZ.png” name = “Export All Transfer Test” id = “org.eclipse.ui.tests.all” > <mapping scope = “ instance“ /> <mapping scope = ”configuration“ /> <mapping scope = ”project“ /> <description>导出所有转移，导出指定范围的所有节点</ description> </ transfer> </ extension>

Bare Bones Transfer仅提供所需信息，不再提供。 

<extension point = “org.eclipse.ui.preferenceTransfer” > <transfer name = “Bare Bones Transfer Test” id = “org.eclipse.ui.tests.all” > <mapping scope = “instance” /> </ transfer > </ extension>

导出许多键和节点组合的示例

<extension point = “org.eclipse.ui.preferenceTransfer” > <transfer icon = “XYZ.png” name = “export many preferences” id = “org.eclipse.ui.tests.all” > <mapping scope = “instance “ > <entry node = ”org.eclipse.ui“ > <key name = ”DOCK_PERSPECTIVE_BAR“ /> </ entry> <entry node = ”org.eclipse.ui.workbench“ > <key name = ”RUN_IN_BACKGROUND“ /> <key name = “OPEN_” match = “prefix” /> </ entry> <entry node = “org.eclipse.ui.ide” /> <entry node = “org.eclipse.core.resources” /> </ mapping> <mapping scope = “configuration” /> <description>导出密钥和节点的多种组合</ description> </ transfer> </ extension>
