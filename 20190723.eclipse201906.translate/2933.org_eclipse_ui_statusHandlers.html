状态处理程序

标识符：
org.eclipse.ui.statusHandlers

以来：
3.3

描述：

此扩展点用于在工作台中注册状态处理程序的实现。处理程序可以与产品相关联，并且此关联也通过此扩展点提供。

状态处理程序是状态处理工具的一部分。该工具负责处理基于Eclipse的应用程序中的错误和其他重要问题。处理程序负责通过记录或显示错误对话框来显示此错误。

如果没有与产品关联的状态处理程序，则状态处理工具使用工作台顾问程序中定义的应用程序处理程序。

配置标记：

<！ELEMENT 扩展 （ statusHandler *， statusHandlerProductBinding *）>

<！ATTLIST扩展

点CDATA #REQUIRED

id CDATA #IMPLIED

名称CDATA #IMPLIED>


<！ELEMENT statusHandler （ 参数 *）>

<！ATTLIST statusHandler

id CDATA #REQUIRED

class CDATA #REQUIRED>

此元素用于定义状态处理程序。


  • id -

    此错误处理程序的唯一标识符。id用于设置产品中的默认错误处理程序。如果错误处理程序是默认处理程序，则必须在org.eclipse.ui / ERROR_HANDLER_ID属性中的产品首选项文件中设置其id。首选项文件的名称在产品扩展“preferenceCustomization”属性中设置。

  • 课程 -

    扩展org.eclipse.ui.statushandlers.的类的完全限定名称org.eclipse.ui.statushandlers.AbstractStatusHandler 。


<！ELEMENT 参数 EMPTY>

<！ATTLIST参数

名称CDATA #REQUIRED

值CDATA #REQUIRED>

此元素定义指定状态处理程序类的实例的参数。指定状态处理程序类的实例应该理解此参数。

处理策略可以使用处理程序参数例如，默认策略使用处理程序“prefix”参数。默认策略针对这些前缀查找给定状态检查状态pluginId的最特定处理程序。


  • name - 参数的名称。
  • value - 参数的值。

<！ELEMENT statusHandlerProductBinding EMPTY>

<！ATTLIST statusHandlerProductBinding

productId CDATA #REQUIRED

handlerId CDATA #REQUIRED>

指定产品和状态处理程序之间的绑定。这些绑定确定哪个处理程序适合当前产品（由org.eclipse.core.runtime.定义） org.eclipse.core.runtime.Platform.getProduct() ）。


  • productId - 产品的唯一ID。
  • handlerId - 状态处理程序的唯一ID。

例子：

以下是带前缀参数的处理程序定义的示例。 

<extension point = “org.eclipse.ui.statusHandlers” > <statusHandler class = “org.eclipse.ui.statushanders。SampleStatusHandler“ id = ”sampleStatusHandler“ > <parameter name = ”prefix“ value = ”org.eclipse.ui.ide“ > </ parameter> </ statusHandler> </ extension>

以下是default（product）处理程序定义的示例。示例中定义的处理程序对于id为“productId”的产品是默认的。 

<extension point = “org.eclipse.ui.statusHandlers” > <statusHandler class = “org.eclipse.ui.statushandlers。SampleStatusHandler“ id = ”sampleStatusHandler“ > </ statusHandler> <statusHandlerProductBinding handlerId = ”sampleStatusHandler“ productId = ”productId“ > </ statusHandlerProductBinding> </ extension>

当然，我们只能定义绑定（例如在产品插件中）。处理程序可以在其他插件中定义。 

<extension point = “org.eclipse.ui.statusHandlers” > <statusHandlerProductBinding handlerId = “sampleStatusHandler” productId = “productId” > </ statusHandlerProductBinding> </ extension>

API信息：
statusHandler元素中class属性的值必须是扩展org.eclipse.ui.statushandlers.的类的完全限定名称org.eclipse.ui.statushandlers.AbstractStatusHandler 。

的parameter中的元素statusHandler元件增加了可以在所定义的处理程序状态在处理过程中使用的参数class属性。

该值productId在属性statusHandlerProductBinding是与定义ID的处理器产品的全名handlerId在属性statusHandlerProductBinding将与之相关联。

提供的实施：

该平台提供org.eclipse.ui.statushandlers.基本实现org.eclipse.ui.statushandlers.AbstractStatusHandler 。它是org.eclipse.ui.statushandlers.WorkbenchErrorHandler ，它是默认的工作台状态处理程序。
