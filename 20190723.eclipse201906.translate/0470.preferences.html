代码覆盖率首选项

可以在Eclipse Preferences对话框中调整EclEmma插件的行为。从窗口 → 首选项菜单中打开对话框。EclEmma的设置位于Java → Code Coverage页面上。或者只需在过滤器字段中键入coverage 。“ 代码覆盖率”首选项页面包含以下选项。

  • 自动打开覆盖视图：每当新的覆盖会话变为活动状态时， Coverage视图可以自动显示在当前工作台窗口中。（默认值：开）
  • 激活新的覆盖会话：在覆盖启动终止或导入会话时创建新会话。此选项确定新会话是否应自动激活，即其结果数据显示在coverage视图和Java编辑器中。（默认值：开）
  • 删除已删除启动的覆盖会话：覆盖会话将一直可用，直到在Coverage视图中手动删除它们。对于频繁的测试运行，使用相关的启动自动删除它们可能更方便。（默认：关闭）
  • 在转储时重置执行数据：对于正在运行的应用程序，可以随时从Coverage视图转储数据。此选项确定在执行此类中间转储后是否重置执行数据。（默认：关闭）

当在代码覆盖模式下执行启动配置时，EclEmma首次选择默认情况下将在代码覆盖率分析范围内的类路径条目。有几个选项可以修改默认选择行为：

  • 仅限源文件夹：仅考虑基于源的类路径条目。（默认值：开）
  • 仅限相同项目：仅从同一项目中选择类路径条目。此选项仅适用于具有项目关联的启动配置，即Java应用程序。（默认：关闭）
  • 仅匹配的路径条目：逗号分隔的必须与类路径条目匹配的字符串列表。如果类路径条目包含给定字符串之一，则它将匹配过滤器。 （例如“src / main / java”，默认值：无过滤器）

警告：如果您的设置与项目中的任何类路径条目都不匹配，则覆盖模式下的每个新启动都将具有空分析范围。

由于技术原因，可能有必要从代码覆盖率分析中排除某些类。以下选项配置coverage代理以从分析中排除某些类。除性能优化或技术角落情况外，通常不需要这些选项。

  • 包含：应包含在执行分析中的类名列表。列表条目用冒号（:)分隔，可以使用通配符（*和？）。（默认值：*）
  • 排除：应从执行分析中排除的类名列表。列表条目用冒号（:)分隔，可以使用通配符（*和？）。（默认值： 空 ）
  • 排除类加载器：应从执行分析中排除的类加载器名称列表。列表条目用冒号（:)分隔，可以使用通配符（*和？）。如果特殊框架与JaCoCo代码检测冲突，特别是无法访问Java运行时类的类加载器，则可能需要此选项。（默认值：sun.reflect。DelegatingClassLoader）

警告：请谨慎使用这些选项！无效的条目可能会破坏代码覆盖率启动器。也不要使用这些选项来定义分析范围。被排除的课程仍将显示为未涵盖。

其他相关的偏好页面

各种Eclipse视图中的Java元素（如Package Explorer）可能会直接使用当前会话中的coverage数据进行修饰 。这可以在常规 → 外观 → 标签装饰器页面上启用。相应的条目是

  • Java代码覆盖率 （默认值：关闭）

可以在常规 → 编辑器 → 文本编辑器 → 注释页面上调整编辑器中代码覆盖的注释样式。相应的条目是：

  • 全覆盖 （默认：绿色突出显示）
  • 部分覆盖范围 （默认值：黄色突出显示）
  • 无覆盖 （默认：红色突出显示）
