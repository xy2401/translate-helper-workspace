新的和值得注意的

以下是对1.8版本的Memory Analyzer进行的一些更有趣或重要的更改的描述。

增强功能和修复

Eclipse Memory Analyzer 1.8支持从Java 9，Java 10以及Java 1.8和以前的版本生成和分析转储。已为此增强了获取转储对话框和集合查询。

  • Memory Analyzer RCP现在使用Photon构建。可以使用以下独立版本：
    • Windows（x86）
    • Windows（x86_64）
    • Mac OSX（Mac / Cocoa / x86_64）
    • Linux（x86 / GTK +）
    • Linux（x86_64 / GTK +）
    • Linux（PPC64 / GTK +）
    • Linux（PPC64le / GTK +）
    注意：由于删除了针对Oxygen的某些操作系统的Eclipse平台支持 ，以后以下内存分析器1.8无法使用以下独立版本：
    • Linux（PPC / GTK +）
    • Linux（s390 / GTK +）
    • Linux（s390x / GTK +）
    • Solaris 8（x86 / GTK +）
    • Solaris 8（SPARC / GTK +）
    • HP-UX（IA64 / GTK +）
    • AIX（PPC / GTK +）
    • AIX（PPC64 / GTK +）
    对于那些平台或Eclipse Memory Analyzer的早期版本，可能会将Memory Analyzer 1.8安装到Eclipse Neon中，但尚未经过测试。
    注意：由于Eclipse Neon及更高版本的要求，运行RCP需要Java 1.8或更高版本。 以前版本的Eclipse Memory Analyzer可以与Java 1.6,1.7和1.8一起运行。
  • Memory Analyzer现在可以使用Java虚拟机，版本9和版本10，以及1.8。仍在处理JVM 1.5,1.6和1.7版本的转储。Memory Analyzer已使用以下类型的JVM进行了测试：
    • Oracle JVM
    • OpenJDK JVM
    • IBM JVM
    • OpenJ9 JVM
  • 修复了新的Java 9字符串格式
  • 获取堆转储 - 各种改进：
    • Attach API提供程序现在可以使用com.sun.tools.attach工作，允许它与Oracle JVM和IBM JVM，版本1.8 SR5及更高版本一起使用。早期的IBM JVM使用com.ibm.tools.attach继续工作
    • 对PID和提供者的列进行排序
    • 使用窗口调整列的大小
    • 更多帮助，包括提供者帮助的链接
    • 对斜体标记的转储不可用的进程
    • Attach API提供可以使用HotSpot MX bean为Oracle和OpenJDK JVMS生成HPROF转储。
    • 从进程中选择时，配置堆转储提供程序页面跳转到适当的提供程序。
  • Equinox Bundle Explorer现在可以处理更新的Eclipse版本。
  • 由集合查询处理的更多集合，包括来自Java 9和Java 10的集合。
  • 修复520908 ： ThreadOverviewQuery中的无效线程对象导致整个查询失败
  • 修复了521999 ：未显示分析某些IBM核心转储的pString值
