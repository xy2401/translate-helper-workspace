Chapter 9. 使用图形用户界面安装RHEL

本节包含有关使用图形用户界面（GUI）安装Red Hat Enterprise Linux的信息。当您从CD，DVD或USB闪存驱动器或使用PXE的网络引导系统时，GUI是安装Red Hat Enterprise Linux的首选方法。

注意

联机帮助与客户门户上发布的内容之间可能存在一些差异。有关最新更新，请参阅客户门户上的安装内容。

图形安装工作流程

完成以下步骤以使用图形用户界面安装Red Hat Enterprise Linux：

脚步

  1. 配置语言和位置设置。有关详细信息，请参阅“配置语言和位置设置”一节 。
  2. 配置本地化设置。有关详细信息，请参阅“配置本地化选项”一节 。
  3. 选择所需的安装源和软件包。有关详细信息，请参阅“配置软件选项”一节 。
  4. 配置安装目标，KDUMP，网络，安全策略和系统用途。有关详细信息，请参阅“配置系统选项”一节 。
  5. 配置存储。有关详细信息，请参阅“配置存储设备”一节 。
  6. 开始安装并创建用户帐户和密码。有关详细信息，请参阅“启动安装程序”一节 。
  7. 完成图形安装。有关详细信息，请参阅“图形安装完成”一节 。

注意

从网络位置安装时，必须先配置网络，然后才能选择要安装的软件包。

配置语言和位置设置

安装程序使用您在安装期间以及已安装系统上选择的语言。

先决条件

  1. 您创建了安装媒体。有关详细信息，请参阅“创建安装介质”一节 。
  2. 如果使用Boot ISO映像文件，则指定了安装源。有关详细信息，请参阅“准备安装源”一节 。
  3. 你启动了安装。有关更多信息，请参见第5章， 引导安装 。

程序

  1. 从Welcome to Red Hat Enterprise Linux窗口的左侧窗格中，选择一种语言。或者，在“ 搜索”字段中键入首选语言。

    注意

    默认情况下会预先选择一种语言。如果配置了网络访问，即，如果从网络服务器而不是本地介质引导，则预选语言由GeoIP模块的自动位置检测功能确定。如果在引导命令行或PXE服务器配置中使用了inst.lang=选项，则会选择使用引导选项定义的语言。

  2. 在Welcome to Red Hat Enterprise Linux窗口的右侧窗格中，选择特定于您所在地区的位置。

  3. 单击“ 继续”以继续执行“安装摘要窗口”窗口部分。

    重要

    如果要安装Red Hat Enterprise Linux的预发行版本，则会显示有关安装介质的预发布状态的警告消息。单击我要继续安装，或者我想退出以退出安装并重新启动系统。

配置本地化选项

本节包含有关配置键盘，语言支持以及时间和日期设置的信息。

重要

如果使用不能接受拉丁字符的布局（如俄语） ，请添加英语（美国）布局并配置键盘组合以在两种布局之间切换。如果仅选择不具有拉丁字符的布局，则可能无法在安装过程中稍后输入有效的root密码和用户凭据。这可能会阻止您完成安装。

配置键盘，语言以及时间和日期设置

注意

键盘，语言和时间和日期设置默认配置为“配置语言和位置设置”一节的一部分。要更改任何设置，请完成以下步骤，否则请转至“配置软件选项”一节 。

过程：配置键盘设置

  1. 在“ 安装摘要”窗口中，单击“ 键盘” 。默认布局取决于“配置语言和位置设置”一节中选择的选项。

    1. 单击+打开“ 添加键盘布局”窗口并更改为其他布局。
    2. 通过浏览列表或使用“ 搜索”字段选择布局。
    3. 选择所需的布局并单击“ 添加” 。新布局显示在默认布局下。
    4. 单击“ 选项”以选择性地配置可用于在可用布局之间循环的键盘开关。将打开“ 布局切换选项”窗口。

    5. 要配置切换的组合键，请选择一个或多个组合键，然后单击“ 确定”以确认您的选择。

      注意

      选择布局时，单击“ 键盘”按钮以打开一个新对话框，该对话框显示所选布局的直观表示。

    6. 单击“完成”以应用设置并返回“安装摘要”窗口部分 。

过程：配置语言设置

  1. 在“ 安装摘要”窗口中，单击“ 语言支持” 。将打开“ 语言支持”窗口。左侧窗格列出了可用的语言组。如果配置了组中的至少一种语言，则会显示复选标记，并突出显示支持的语言。

    1. 在左侧窗格中，单击一个组以选择其他语言，然后从右侧窗格中选择区域选项。对您需要的语言重复此过程。
    2. 单击“完成”以应用更改并返回“安装摘要”窗口部分 。

过程：配置时间和日期设置

  1. 在“ 安装摘要”窗口中，单击“ 时间和日期” 。时间和日期窗口打开。

    注意

    默认情况下， 时间和日期设置基于您在“配置语言和位置设置”一节中选择的设置进行配置 。

    城市和地区列表来自由互联网号码分配机构（IANA）维护的时区数据库（ tzdata ）公共域。Red Hat无法将城市或区域添加到此数据库。您可以在IANA官方网站上找到更多信息。

    1. 从“ 区域”下拉菜单中，选择一个区域。

      注意

      选择Etc作为您的区域，以配置相对于格林威治标准时间（GMT）的时区，而无需将您的位置设置为特定区域。

    2. 从“ 城市”下拉菜单中，选择城市或在同一时区中距离您最近的城市。

    3. 切换网络时间开关以使用网络时间协议（NTP）启用或禁用网络时间同步。

      注意

      只要系统可以访问互联网，启用网络时间开关可以使您的系统时间保持正确。缺省情况下，配置一个NTP池;您可以添加新选项，或通过单击网络时间开关旁边的齿轮按钮禁用或删除默认选项。

    4. 单击“完成”以应用更改并返回“安装摘要”窗口部分 。

    注意

    如果禁用网络时间同步，则窗口底部的控件将变为活动状态，允许您手动设置时间和日期。

配置安装源

按照此过程中的步骤将Binary DVD ISO映像配置为安装源，这是安装Red Hat Enterprise Linux的推荐方法。

先决条件

  • 您已下载二进制DVD ISO映像，详细信息请参阅“下载安装ISO映像”一节 。
  • 您已创建可启动安装介质，详见“创建安装介质”一节 。
  • “ 安装摘要”窗口已打开。

注意

首次打开“ 安装摘要”窗口时，安装程序会尝试根据用于引导系统的介质类型配置安装源。完整的Red Hat Enterprise Linux Server DVD将源配置为本地媒体。

程序

  1. 在“ 安装摘要”窗口中，单击“ 安装源” 。将打开“ 安装源”窗口。

    1. 查看“ 自动检测到的安装”部分以验证详细信息。如果从包含安装源（例如DVD）的介质启动安装程序，则默认选择此选项。
    2. 单击“ 验证”以检查介质完整性。

    3. 查看“ 其他存储库”部分，并注意默认情况下已选中“ Appstream”复选框。

      重要

      • 无需其他配置，因为BaseOS和Appstream存储库是作为完整安装映像的一部分安装的。
      • 如果要安装完整的Red Hat Enterprise Linux 8，请不要禁用Appstream存储库复选框。

  2. 选择“ 在网络上”选项以从网络位置而不是本地介质下载和安装软件包。

    注意

    • 如果您不想从网络位置下载和安装其他存储库，请转至“配置软件选择”一节 。
    • 仅当网络连接处于活动状态时，此选项才可用。有关如何在GUI中配置网络连接的信息，请参阅“配置网络和主机名选项”一节 。

    1. 选择“ 在网络上”下拉菜单以指定下载程序包的协议。此设置取决于您要使用的服务器。

      警告

      如果选择“ 在网络上” ，然后决定还原为“ 自动检测到的安装 ”，则会禁用“Appstream存储库”复选框。您必须选中Appstream复选框以启用Appstream存储库。

    2. 在地址字段中键入服务器地址（不含协议）。如果选择NFS，则会打开第二个输入字段，您可以在其中指定自定义NFS装入选项 。该字段接受nfs(5)手册页中列出的选项。

      重要

      当选择NFS安装源，则必须用冒号地址（ : ）字符从路径分离的主机名。例如：

      server.example.com:/path/to/directory

      注意

      以下步骤是可选的，仅在您使用代理进行网络访问时才需要。

    3. 单击代理设置...可配置的代理HTTP或HTTPS源。
    4. 选中“ 启用HTTP代理”复选框，然后在“ 代理主机”字段中键入URL。
    5. 如果代理服务器需要身份验证，请选中“ 使用身份验证
    6. 输入您的用户名和密码。

    7. 单击确定完成配置并退出代理设置...对话框。

      注意

      如果HTTP或HTTPS URL引用存储库镜像菜单，请从URL类型下拉列表中选择所需选项。完成源配置后，可以选择所有环境和加载项。

  3. 单击+以添加存储库。
  4. 单击-删除存储库。
  5. 打开“ 安装源”窗口时，单击箭头图标将当前条目还原为该设置。

  6. 要激活或取消激活存储库，请单击列表中每个条目的“已启用”列中的复选框。

    注意

    您可以使用与网络上主存储库相同的方式命名和配置其他存储库。

  7. 单击“完成”以应用设置并返回“ 安装摘要”窗口。

配置软件选择

使用“ 软件选择”窗口选择所需的软件包。这些包由Base Environments和Add-Ons组织。

  • 基本环境是预定义的包。您只能选择一个基本环境，可用性取决于用作安装源的安装ISO映像。
  • 附加组件是基础环境的附加软件包。您可以选择多个加载项。

使用预定义的环境和加载项来自定义系统，但在标准安装中，您无法选择要安装的单个程序包。要查看特定环境或附加组件中包含的软件包，请参阅repository /repodata/*-comps- repository . architecture .xml安装源介质（DVD，CD，USB）上的repository /repodata/*-comps- repository . architecture .xml文件。XML文件包含作为基本环境或附加组件的一部分安装的软件包的详细信息。可用环境由<environment>标记标记，加载项由<group>标记标记。

如果您不确定要安装哪些软件包，Red Hat建议您选择Minimal Install基础环境。最小安装只需少量额外软件即可安装基本版本的Red Hat Enterprise Linux。系统完成安装并首次登录后，您可以使用Yum软件包管理器安装其他软件。有关Yum包管理器的详细信息，请参阅配置基本系统设置文档。

注意

  • yum group list命令列出yum存储库中的所有包组。有关更多信息，请参阅配置基本系统设置文档。
  • 如果需要控制安装的软件包，可以使用Kickstart文件并在%packages部分中定义%packages 。有关使用Kickstart安装Red Hat Enterprise Linux的信息，请参阅执行高级RHEL安装文档。

先决条件

  • 您已配置安装源。
  • 安装程序下载了包元数据。
  • “ 安装摘要”窗口已打开。

程序

  1. 在“ 安装摘要”窗口中，单击“ 软件选择” 。“ 软件选择”窗口打开。

  2. 从“ 基础环境”窗格中，选择基本环境。您只能选择一个基本环境。

    注意

    具有GUI基本环境的服务器是默认的基本环境，它会在安装完成后启动初始设置应用程序并重新启动系统。

  3. 从“ 所选环境的附加组件”窗格中，选择一个或多个加载项。
  4. 单击“完成”以应用设置并返回“安装摘要”窗口部分 。

配置系统选项

本节包含有关配置安装目标，KDUMP，网络和主机名，安全策略和系统用途的信息。

配置安装目标

使用“ 安装目标”窗口配置存储选项，例如，要用作Red Hat Enterprise Linux安装的安装目标的磁盘。您必须至少选择一个磁盘。

警告

如果您计划使用已包含数据的磁盘，请备份数据。例如，如果要缩小现有Microsoft Windows分区并将Red Hat Enterprise Linux安装为第二个系统，或者要升级以前版本的Red Hat Enterprise Linux。操纵分区总是存在风险。例如，如果进程因任何原因中断或失败，则磁盘上的数据可能会丢失。

重要

  • 某些BIOS类型不支持从RAID卡启动。在这些情况下，必须在RAID阵列外部的分区上创建/boot分区，例如在单独的硬盘驱动器上。有问题的RAID卡需要使用内部硬盘驱动器来创建分区。软件RAID设置也需要/boot分区。如果选择自动分区系统，则应手动编辑/boot分区。
  • 要将Red Hat Enterprise Linux引导加载程序配置为从其他引导加载程序进行链式加载，必须通过单击“ 安装目标”窗口中的“ 完整磁盘摘要和引导加载程序”链接手动指定引导驱动器。
  • 在具有多路径和非多路径存储设备的系统上安装Red Hat Enterprise Linux时，安装程序中的自动分区布局会创建包含多路径和非多路径设备混合的卷组。这违背了多路径存储的目的。建议您在“ 安装目标”窗口中选择多路径或非多路径设备。或者，继续进行手动分区。
条件

“ 安装摘要”窗口已打开。

程序

  1. 在“ 安装摘要”窗口中，单击“ 安装目标” 。“ 安装目标”窗口将打开。

    1. 从“ 本地标准磁盘”部分，选择所需的存储设备;白色复选标记表示您的选择。安装过程中不使用没有白色复选标记的磁盘;如果选择自动分区，则会忽略它们，并且它们在手动分区中不可用。

      注意

      所有本地可用的存储设备（SATA，IDE和SCSI硬盘驱动器，USB闪存和外部磁盘）都显示在“ 本地标准磁盘”下 。未检测到安装程序启动后连接的任何存储设备。如果使用可移动驱动器安装Red Hat Enterprise Linux，则在删除设备时系统将无法使用。

    2. 可选：如果要配置其他本地存储设备以连接新硬盘，请单击窗口右下角的“ 刷新”链接。将打开“ 重新扫描磁盘”对话框。

      注意

      单击“ 重新扫描磁盘”时，在安装过程中所做的所有存储更改都将丢失。

      1. 单击“ 重新扫描磁盘”并等待扫描过程完成。
      2. 单击“ 确定”返回“ 安装目标”窗口。所有检测到的磁盘（包括任何新磁盘）都显示在“ 本地标准磁盘”部分下。

  2. 可选：要添加一个专门的存储设备，单击添加磁盘...。

    将打开“ 存储设备选择”窗口，其中列出了安装程序可以访问的所有存储设备。有关如何添加专用磁盘的信息，请参阅“使用高级存储选项”一节 。

  3. 可选：在“ 存储配置”下 ，选择“ 自动”单选按钮。

    重要

    自动分区是分区存储的推荐方法。您还可以配置自定义分区，有关详细信息，请参阅“配置手动分区”一节。

  4. 可选：要从现有分区布局中回收空间，请选中“ 我想要增加可用空间”复选框。例如，如果您要使用的磁盘已包含不同的操作系统，并且您希望使该系统的分区更小，以便为Red Hat Enterprise Linux留出更多空间。

  5. 可选：选择加密我的数据以使用Linux Unified Key Setup （LUKS）加密除引导系统所需的所有分区（例如/boot ）。建议加密硬盘。

    1. 如果选择“加密我的数据” ，则会打开“ 磁盘加密密码”对话框。

      1. 在密码和确认字段中键入您的密码。

      2. 单击“ 保存密码”以完成磁盘加密。

        警告

        如果丢失了LUKS密码，则任何加密分区及其数据都将完全无法访问。没有办法恢复丢失的密码短语。但是，如果执行Kickstart安装，则可以在安装期间保存加密密码并创建备份加密密码。有关信息，请参阅执行高级RHEL安装文档。

  6. 可选：单击窗口左下角的“ 完整磁盘摘要和引导加载程序”链接，以选择包含引导加载程序的存储设备。有关更多信息，请参阅“配置引导加载程序”一节 。

    注意

配置启动加载程序

红帽企业Linux使用GRand Unified Bootloader版本2（ GRUB2 ）作为AMD64和Intel 64，IBM Power Systems和ARM的引导加载程序。对于IBM Z，使用zipl引导加载程序。

引导加载程序是系统启动时运行的第一个程序，负责加载控制并将控制转移到操作系统。GRUB2可以启动任何兼容的操作系统（包括Microsoft Windows），也可以使用链式加载将控制权转移到其他引导加载程序，以用于不受支持的操作系统。

警告

安装GRUB2可能会覆盖现有的引导加载程序。

如果已安装操作系统，则Red Hat Enterprise Linux安装程序会尝试自动检测并配置引导加载程序以启动其他操作系统。如果未检测到引导加载程序，则可以在完成安装后手动配置任何其他操作系统。

如果要安装具有多个磁盘的Red Hat Enterprise Linux系统，则可能需要手动指定要安装引导加载程序的磁盘。

程序

  1. 在“ 安装目标”窗口中，单击“ 完整磁盘摘要和引导加载程序”链接。将打开“ 选定的磁盘”对话框。

    引导加载程序安装在您选择的设备上，或安装在UEFI系统上;在引导分区期间，在目标设备上创建EFI系统分区。

  2. 要更改引导设备，请从列表中选择一个设备，然后单击“ 设置为引导设备” 。您只能将一个设备设置为引导设备。
  3. 要禁用新的引导加载程序安装，请选择当前标记为引导的设备，然后单击“不安装引导加载程序” 。这可确保GRUB2未安装在任何设备上。

警告

如果选择不安装引导加载程序，则无法直接引导系统，必须使用其他引导方法，例如独立的商业引导加载程序应用程序。仅当您有其他方法来引导系统时，才使用此选项。

引导加载程序还可能需要创建特殊分区，具体取决于系统是使用BIOS还是UEFI固件，或者引导驱动器是否具有GUID分区表 （GPT）或主引导记录 （MBR，也称为msdos ）标签。如果使用自动分区，则安装程序将创建分区。

配置Kdump

Kdump是一种内核崩溃转储机制。在系统崩溃的情况下， Kdump会在发生故障时捕获系统内存的内容。可以分析捕获的内存以查找崩溃的原因。如果启用了Kdump ，则必须将系统内存（RAM）的一小部分保留给自身。主内核无法访问此保留内存。

程序

  1. 在“ 安装摘要”窗口中，单击“ Kdump” 。Kdump窗口打开。
  2. 选中“ 启用kdump”复选框。

  3. 选择自动或手动内存预留设置。

    1. 如果选择“ 手动” ，请使用+和-按钮在要保留的存储器字段中输入要保留的内存量（以兆字节为单位）。预留输入字段下方的可用系统内存读数显示在保留您选择的RAM量后，主系统可以访问的内存量。
  4. 单击“完成”以应用设置并返回“安装摘要”窗口部分 。

注意

您保留的内存量取决于您的系统架构（AMD64和Intel 64具有与IBM Power不同的要求）以及系统内存总量。在大多数情况下，自动预订令人满意。

重要

其他设置（例如将保存内核崩溃转储的位置）只能在使用system-config-kdump图形界面进行安装后配置 ，或者在/etc/kdump.conf配置文件中手动配置。

关于安全政策

红帽企业Linux安全策略遵守安全内容自动化协议（SCAP）标准定义的限制和建议（合规性策略）。包是自动安装的。但是，默认情况下，不会强制实施任何策略，因此除非特别配置，否则不会在安装期间或之后执行任何检查。

应用安全策略不是安装程序的强制功能。如果将安全策略应用于系统，则使用您选择的配置文件中定义的限制和建议进行安装。openscap-scanner软件包已添加到您的软件包选择中，提供了一个用于合规性和漏洞扫描的预安装工具。安装完成后，系统会自动扫描以验证是否符合要求。此扫描的结果将保存到已安装系统上的/root/openscap_data目录中。您还可以从HTTP，HTTPS或FTP服务器加载其他配置文件。

使用syspurpose命令行工具配置系统目的

按照此过程中的步骤使用syspurpose命令行工具在安装后配置系统用途。

先决条件

  • 您已经安装并注册了Red Hat Enterprise Linux 8系统。
  • 您以root用户身份登录。
程序

按照此过程中的步骤使用syspurpose命令行工具设置和取消设置系统用途选项。

  1. 在终端窗口中，运行以下命令以设置系统的预期角色：

    # syspurpose set-role "VALUE"

    将VALUE替换为您要分配的角色：

    • 红帽企业Linux服务器
    • 红帽企业Linux工作站
    • 红帽企业Linux计算节点

    例如：

    # syspurpose set-role "Red Hat Enterprise Linux Server"

    运行以下命令以取消设置角色：

    # syspurpose unset-role

  2. 运行以下命令以设置系统的预期sla：

    # syspurpose set-sla "VALUE"

    将VALUE替换为要分配的sla：

    • 额外费用
    • 标准
    • 自助支持

    例如：

    # syspurpose set-sla "Standard"

    运行以下命令以取消设置sla：

    # syspurpose unset-sla

  3. 运行以下命令以设置系统的预期用途：

    # syspurpose set-usage "VALUE"

    将VALUE替换为您要分配的用法：

    • 生产
    • 灾难恢复

    • 开发/测试

      例如：

      # syspurpose set-usage "Production"

安装到NVDIMM设备

非易失性双列直插式内存模块（NVDIMM）设备在没有供电时将RAM的性能与类似磁盘的数据持久性相结合。在特定情况下，Red Hat Enterprise Linux 8可以从NVDIMM设备启动和运行。

使用NVDIMM设备作为安装目标的标准

您可以在nd_pmem驱动程序支持的Intel 64和AMD64架构上以扇区模式将Red Hat Enterprise Linux 8安装到非易失性双列直插式内存模块（NVDIMM）设备。

使用NVDIMM设备作为存储的条件

要将NVDIMM设备用作存储，必须满足以下条件：

  • 该系统的架构是Intel 64或AMD64。
  • NVDIMM设备配置为扇区模式。安装程序可以将NVDIMM设备重新配置为此模式。
  • Nd_pmem驱动程序必须支持NVDIMM设备。
从NVDIMM设备启动的条件

在以下条件下可以从NVDIMM设备引导：

  • 满足使用NVDIMM设备作为存储的所有条件。
  • 系统使用UEFI。
  • NVDIMM设备必须由系统上可用的固件或UEFI驱动程序支持。可以从设备自身的选项ROM加载UEFI驱动程序。
  • 必须在命名空间下提供NVDIMM设备。

在引导期间利用NVDIMM设备的高性能，将/boot和/boot/efi目录放在设备上。引导期间不支持NVDIMM设备的“就地执行”（XIP）功能，并且内核已加载到常规内存中。

创建软件RAID

按照此过程中的步骤创建独立磁盘冗余阵列（RAID）设备。RAID设备由多个存储设备构成，这些存储设备被布置为提供增强的性能，并且在一些配置中，具有更高的容错性。

只需一步即可创建RAID设备，并根据需要添加或删除磁盘。您可以为系统中的每个物理磁盘配置一个RAID分区，因此安装程序可用的磁盘数决定了可用的RAID设备的级别。例如，如果您的系统有两个硬盘驱动器，则无法创建RAID10设备，因为它需要4个单独的分区。

注意

在IBM Z上，存储子系统透明地使用RAID。无需手动配置软件RAID。

先决条件

  • 在RAID配置选项可见之前，您已选择两个或更多磁盘进行安装。创建RAID设备至少需要两个磁盘。
  • 您已经创建了一个挂载点。通过配置安装点，可以配置RAID设备。
  • 您已在“ Installation Destination窗口中选择“ Custom单选按钮。

程序

  1. 从“ 手动分区”窗口的左窗格中，选择所需的分区。
  2. 在“ 设备”部分下，单击“ 修改” 。将打开“ 配置挂载点”对话框。
  3. 选择要包含在RAID设备中的磁盘，然后单击“ 选择” 。
  4. 单击“ 设备类型”下拉菜单，然后选择“ RAID” 。
  5. 单击“ 文件系统”下拉菜单，然后选择首选文件系统类型。
  6. 单击RAID Level下拉菜单，然后选择您喜欢的RAID级别。
  7. 单击“ 更新设置”以保存更改。
  8. 单击“完成”以应用设置并返回“ 安装摘要”窗口。

如果指定的RAID级别需要更多磁盘，则会在窗口底部显示一条消息。

创建LVM逻辑卷

逻辑卷管理（LVM）提供了底层物理存储空间的简单逻辑视图，例如硬盘驱动器或LUN。物理存储上的分区表示为可以组合到卷组中的物理卷。您可以将每个卷组划分为多个逻辑卷，每个逻辑卷类似于标准磁盘分区。因此，LVM逻辑卷充当可跨越多个物理磁盘的分区。

注意

LVM配置仅在图形安装程序中可用。

重要

在文本模式安装期间，LVM配置不可用。要创建LVM配置，请按Ctrl + Alt + F2以使用其他虚拟控制台，然后运行lvm命令。要返回文本模式安装，请按Ctrl + Alt + F1 。

程序

  1. 从“ Manual Partitioning窗口的左侧窗格中，选择安装点。

  2. 单击“ Device Type下拉菜单，然后选择“ LVM 。将显示“ Volume Group下拉菜单，其中包含新创建的卷组名称。

    注意

    您无法在配置对话框中指定卷组的物理范围的大小。大小始终设置为默认值4 MiB。如果要创建具有不同物理范围的卷组，则必须通过切换到交互式shell并使用vgcreate命令手动创建它，或者使用带有volgroup --pesize= size命令的Kickstart文件。有关Kickstart的更多信息，请参阅执行高级RHEL安装文档。

其他资源

  • 有关LVM的详细信息，请参阅配置和管理逻辑卷文档。

  2. 单击“ 设备类型”下拉菜单，然后选择“ LVM 。将显示“ Volume Group下拉菜单，其中包含新创建的卷组名称。

  6. 从“ 大小”策略下拉菜单中，选择卷组的大小策略。

    可用的政策选项包括：

    • 自动 ：自动设置卷组的大小，使其足够大以包含已配置的逻辑卷。如果您不需要卷组中的可用空间，这是最佳选择。

配置root密码

您必须配置root密码才能完成安装过程并登录用于系统管理任务的管理员（也称为超级用户或root用户）帐户。这些任务包括安装和更新软件包以及更改系统范围的配置，例如网络和防火墙设置，存储选项以及添加或修改用户，组和文件权限。

警告

root帐户可以完全控制系统。如果未经授权的人员可以访问该帐户，他们可以访问或删除用户的个人文件。

程序

    • 必须至少八个字符
    • 可能包含数字，字母（大写和小写）和符号
    • 区分大小写
