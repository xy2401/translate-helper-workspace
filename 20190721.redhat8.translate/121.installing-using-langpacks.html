Chapter 9. 安装和使用langpacks

langpacks介绍

Langpacks是元数据包，它为系统上安装的每个软件包安装额外的附加软件包，其中包含翻译，字典和语言环境。

在Red Hat Enterprise Linux 8系统上， langpacks安装基于langpacks-<langcode>语言元数据包和RPM弱依赖项（Supplements标记）。

能够为所选语言使用langpacks有两个先决条件：

  1. 已在系统上安装所选语言的langpacks-<langcode>语言元包。

    在Red Hat Enterprise Linux 8上，使用Anaconda安装程序初始安装操作系统时会自动安装langpacks元软件包，因为这些软件包可在Application Stream存储库中找到。

    要查找提供langpacks的语言，请执行以下命令：

    ~]# yum list langpacks-*
  2. 您要为其搜索本地程序包的基本程序包已安装在系统上。

如果满足这些先决条件，语言元包将在事务集中自动为所选语言提取其langpack。

使用RPM弱依赖性langpacks

查询RPM弱依赖性langpacks

要列出已安装的语言支持，请运行以下命令：

~]# yum list installed langpacks*

要检查是否有其他语言的语言支持，请运行以下命令：

~]# yum list available langpacks*

要列出为任何语言安装的软件包，请运行以下命令：

~]# yum repoquery --whatsupplements langpacks-<locale_code>

安装语言支持

要添加新的语言支持，请以root用户身份运行以下命令：

~]# yum install langpacks-<locale_code>

删除语言支持

要删除任何已安装的语言支持，请以root用户身份运行以下命令：

~]# yum remove langpacks-<locale_code>

使用glibc-langpack- <locale_code>节省磁盘空间

目前，所有语言环境都存储在/usr/lib/locale/locale-archive文件中，这需要大量磁盘空间。

在磁盘空间是关键问题的系统上，例如容器和云映像，或者只需要几个语言环境，您可以使用glibc locale langpack软件包（ glibc-langpack-<locale_code> ）。通过使用以下命令而不是“安装语言支持”一节中描述的命令，您可以单独安装区域设置，从而获得更小的软件包安装占用空间：

~]# yum install glibc-langpack-<locale_code>

使用Anaconda安装操作系统时，将安装glibc-langpack-<locale_code> ，用于安装期间使用的语言以及您选择作为其他语言的语言。请注意，默认情况下会安装包含所有语言环境的glibc-all-langpacks ，因此会复制某些语言环境。如果为一个或多个所选语言安装了glibc-langpack-<locale_code> ，则可以在安装后删除glibc-all-langpacks以节省磁盘空间。

请注意，仅安装选定的glibc-langpack-<locale_code>包而不是glibc-all-langpacks会影响运行时性能。

注意

如果磁盘空间不是问题，请使用glibc-all-langpacks包保持安装所有语言环境。
