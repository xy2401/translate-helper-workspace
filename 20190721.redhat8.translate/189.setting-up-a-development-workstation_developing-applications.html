Chapter 9. 设置开发工作站

Red Hat Enterprise Linux 8支持自定义应用程序的开发。要允许开发人员这样做，必须使用所需的工具和实用程序设置系统。本章列出了最常见的开发用例和要安装的项目。

先决条件

  • 必须安装系统，包括图形环境，并订阅。

启用调试和源存储库

标准安装的Red Hat Enterprise Linux不支持调试和源存储库。这些存储库包含调试系统组件和测量其性能所需的信息。

程序

  • 启用源和调试信息包通道：

    # subscription-manager repos --enable rhel-8-for-$(uname -i)-baseos-debug-rpms
# subscription-manager repos --enable rhel-8-for-$(uname -i)-baseos-source-rpms
# subscription-manager repos --enable rhel-8-for-$(uname -i)-appstream-debug-rpms
# subscription-manager repos --enable rhel-8-for-$(uname -i)-appstream-source-rpms

    $(uname -i)部分会自动替换为系统架构的匹配值：

    建筑名称值

    64位Intel和AMD

    x86_64的

    64位ARM

    aarch64

    IBM POWER

    ppc64le

    IBM Z.

    s390x

设置以管理应用程序版本

有效的版本控制对于所有多开发人员项目至关重要。红帽企业Linux与分布式版本控制系统Git一起发布。

程序

  1. 安装git包：

    # yum install git

  2. 可选：设置与您的Git提交相关联的全名和电子邮件地址：

    $ git config --global user.name "Full Name"
$ git config --global user.email "email@example.com"

    将全名和email@example.com替换为您的实际姓名和电子邮件地址。

  3. 可选：要更改由Git启动的默认文本编辑器，请设置core.editor配置选项的值：

    $ git config --global core.editor command

    将命令替换为用于启动所选文本编辑器的命令。

其他资源

  • Git和教程的Linux手册页：

    $ man git
$ man gittutorial
$ man gittutorial-2

    请注意，许多Git命令都有自己的手册页。作为示例，请参阅git-commit（1） 。

  • Git用户手册 - Git的 HTML文档位于/usr/share/doc/git/user-manual.html 。
  • Pro Git - Pro Git书籍的在线版本提供了Git，其概念及其用法的详细说明。
  • 参考 - Git的Linux手册页的在线版本

设置使用C和C ++开发应用程序

Red Hat Enterprise Linux包含用于创建C和C ++应用程序的工具。

Preregquisites

  • 必须启用调试和源存储库。

程序

  1. 安装开发工具包组，包括GNU编译器集合（GCC），GNU调试器（GDB）和其他开发工具：

    # yum group install "Development Tools"

  2. 安装基于LLVM的工具链，包括clang编译器和lldb调试器：

    # yum install llvm-toolset

  3. 可选：对于Fortran依赖项，请安装GNU Fortran编译器：

    # yum install gcc-gfortran

设置为调试应用程序

红帽企业Linux提供了多种调试和工具工具来分析和排除内部应用程序行为。

Preregquisites

  • 必须启用调试和源存储库。

程序

  1. 安装用于调试的工具：

    # yum install gdb valgrind systemtap ltrace strace

  2. 安装yum-utils包以使用debuginfo-install工具：

    # yum install yum-utils

  3. 运行SystemTap帮助程序脚本以设置环境。 

    # stap-prep

    运行此脚本将安装内核debuginfo包。这些包的大小超过2 GiB。

  4. 确保SELinux策略不仅允许相关应用程序正常运行，而且还允许在调试情况下运行。有关更多信息，请参阅配置和管理安全性 。

设置以衡量应用程序的性能

Red Hat Enterprise Linux包含多个应用程序，可以帮助开发人员确定应用程序性能下降的原因。

Preregquisites

  • 必须启用调试和源存储库。

程序

  1. 安装性能测量工具：

    # yum install perf papi pcp-zeroconf valgrind strace sysstat systemtap

  2. 运行SystemTap帮助程序脚本以设置环境。 

    # stap-prep

    运行此脚本将安装内核debuginfo包。这些包的大小超过2 GiB。

  3. 启用并启动Performance Co-Pilot（PCP）收集器服务：

    # systemctl enable pmcd && systemctl start pmcd
