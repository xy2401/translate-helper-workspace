在RHEL 8中分发内容

RHEL 8内容通过两个主存储库分发： BaseOS和AppStream 。

BaseOS
BaseOS存储库中的内容旨在提供底层操作系统功能的核心集，为所有安装提供基础。此内容以RPM格式提供，并受类似于以前版本的Red Hat Enterprise Linux中的支持条款的约束。
AppStream
AppStream存储库中的内容包括其他用户空间应用程序，运行时语言和数据库，以支持各种工作负载和用例。AppStream中的内容有两种格式可供选择 - 熟悉的RPM格式和称为模块的RPM格式的扩展。

重要

BaseOS和AppStream内容集都是基本RHEL安装所必需的，并且可用于所有RHEL订阅。有关安装说明，请参阅“ 安装和部署RHEL”文档。

应用程序流

红帽企业Linux 8引入了应用程序流的概念 - 用户空间组件的版本。现在，这些组件的多个版本比核心操作系统软件包更频繁地提供和更新。这为定制Red Hat Enterprise Linux提供了更大的灵活性，而不会影响平台或特定部署的底层稳定性。

作为Application Streams提供的组件可以打包为模块或RPM包，并通过Red Hat Enterprise Linux 8中的AppStream存储库提供。每个ApplStream组件都有一个给定的生命周期。

注意

并非所有模块都是Application Streams。其他模块的依赖关系不被视为AppStream组件。

每个模块都定义了自己的生命周期，这个生命周期更接近应用程序的自然生命，而不是RHEL生命周期。要了解可用的应用程序流以及它们的支持长度，请参阅Red Hat Enterprise Linux 8应用程序流生命周期页面。

RHEL 8中的包装方法

AppStream存储库包含以两种方式打包的内容：

各个RPM包
传统的RPM软件包可立即安装。
模块
模块是表示逻辑单元的包的集合：应用程序，语言堆栈，数据库或一组工具。这些包是一起构建，测试和发布的。

在RHEL 8中使用YUM进行包管理

YUM软件包管理工具现在基于DNF技术，它增加了对新模块化功能的支持。

处理单个RPM软件包时， YUM的使用尚未更改。为了处理模块化内容，添加了yum module命令。有关其他详细信息，请参阅第4章， 安装RHEL 8内容 。

如果需要，模块化功能会自动选择适当的模块和流组合，以便安装逻辑包包以方便使用。
