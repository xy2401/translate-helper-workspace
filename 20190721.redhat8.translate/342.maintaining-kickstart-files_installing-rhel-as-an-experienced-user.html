Chapter 9. 维护Kickstart文件

您可以对Kickstart文件运行自动检查。通常，您需要验证新的或有问题的Kickstart文件是否有效。

安装Kickstart维护工具

要使用Kickstart维护工具，必须安装包含它们的软件包。

程序

  • 安装pykickstart包：

    # yum install pykickstart

验证Kickstart文件

使用ksvalidator命令行实用程序验证您的Kickstart文件是否有效。当您对Kickstart文件进行大量更改时，这非常有用。

程序

  • 在Kickstart文件上运行ksvalidator ：

    $ ksvalidator /path/to/kickstart.ks

    将/path/to/kickstart.ks替换为要验证的Kickstart文件的路径。

重要

验证工具无法保证安装成功。它仅确保语法正确并且文件不包含已弃用的选项。它不会尝试验证Kickstart文件的%pre ， %post和%packages部分。
