Chapter 9. 配置系统目的

系统管理员使用System Purpose来记录组织对Red Hat Enterprise Linux 8系统的预期用途。设置系统目的时，授权服务器会接收有助于自动附加满足系统预期用途的订阅的信息。本节介绍如何使用Kickstart配置系统目的。

概观

您可以通过以下方式输入系统用途数据：

  • 在图像创建期间。
  • 在使用图形用户界面进行安装期间。
  • 使用Kickstart自动化脚本。
  • 使用syspurpose命令行工具。

您可以配置以下组件：

  • 作用 ：

    • 红帽企业Linux服务器
    • 红帽企业Linux工作站
    • 红帽企业Linux计算节点

  • 服务水平协议 ：

    • 额外费用
    • 标准
    • 自助支持

  • 用法 ：

    • 生产
    • 灾难恢复
    • 开发/测试

好处包括：

  • 系统管理员和业务运营的深入系统级信息。
  • 在确定采购系统的原因及其预期目的时减少开销。
  • 改进了Subscription Manager自动附加的客户体验以及系统使用的自动发现和协调。

在Kickstart文件中配置系统目的

按照此过程中的步骤使用syspurpose命令在Kickstart配置文件中配置系统用途。

注意

虽然强烈建议您配置System Purpose，但它是Red Hat Enterprise Linux安装程序的可选功能。如果要在安装完成后启用“系统用途”，则可以使用syspurpose命令行工具执行此操作。

可以使用以下操作：

角色

设置系统的预期角色。此操作使用以下格式：

syspurpose --role=

分配的角色可以是：

  • Red Hat Enterprise Linux Server
  • Red Hat Enterprise Linux Workstation
  • Red Hat Enterprise Linux Compute Node
SLA

设置系统的预期sla。此操作使用以下格式：

syspurpose --sla=

分配的sla可以是：

  • Premium
  • Standard
  • Self-Support
用法

设置系统的预期用途。此操作使用以下格式：

syspurpose --usage=

分配的使用量可以是：

  • Production
  • Disaster Recovery
  • Development/Test
