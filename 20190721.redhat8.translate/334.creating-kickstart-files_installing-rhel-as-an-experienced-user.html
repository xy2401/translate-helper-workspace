Chapter 9. 创建Kickstart文件

您可以使用以下方法创建Kickstart文件：

  • 使用在线Kickstart配置工具。
  • 复制由于手动安装而创建的Kickstart文件。
  • 手动编写整个Kickstart文件。请注意，从其他方法编辑现有文件的速度更快，因此不建议使用此方法。

请注意，只能通过手动编辑Kickstart文件来配置某些高度特定的安装选项。

通过执行手动安装来创建Kickstart文件

创建Kickstart文件的推荐方法是使用手动安装Red Hat Enterprise Linux创建的文件。安装完成后，安装期间所做的所有选择都将保存到名为anaconda-ks.cfg的Kickstart文件中，该文件位于已安装系统的/root/目录中。您可以使用此文件以与以前相同的方式重现安装。或者，复制此文件，进行所需的任何更改，并使用生成的配置文件进行进一步安装。

程序

  1. 安装RHEL。有关更多详细信息，请参阅执行标准RHEL安装 。

    在安装期间，创建具有管理员权限的用户。

  2. 完成安装并重新启动到已安装的系统。
  3. 使用管理员帐户登录系统。

  4. 将文件/root/anaconda-ks.cfg复制到您选择的位置。

    • 要在终端中显示文件内容：

      # cat /root/anaconda-ks.cfg

      您可以复制输出并保存到您选择的另一个文件中。

    • 要将文件复制到其他位置，请使用文件管理器。请记住更改副本的权限，以便非root用户可以读取该文件。

    警告

    该文件包含有关用户和密码的信息。

其他资源

使用Kickstart配置工具创建Kickstart文件

拥有Red Hat Customer Portal帐户的用户可以使用客户门户实验室中的Kickstart Generator工具在线生成Kickstart文件。此工具将引导您完成基本配置，并使您可以下载生成的Kickstart文件。

注意

该工具目前不支持任何高级分区。

先决条件

  • 您必须拥有Red Hat Customer Portal帐户。

程序

  1. 打开Kickstart生成器实验室信息页面， 网址为https://access.redhat.com/labsinfo/kickstartconfig
  2. 单击标题左侧的“ 转到应用程序”按钮，然后等待下一页加载。
  3. 在下拉菜单中选择Red Hat Enterprise Linux 8 ，然后等待页面更新。

  4. 使用表单中的字段描述要安装的系统。

    您可以使用表单左侧的链接在表单的各个部分之间快速导航。

  5. 要下载生成的Kickstart文件，请单击页面顶部的红色“ 下载”按钮。

    您的Web浏览器将保存该文件。
