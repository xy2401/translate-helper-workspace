Chapter 9. 删除RHEL 8内容

以下部分描述了如何删除Red Hat Enterprise Linux 8中的内容：

  • “删除已安装的软件包”一节介绍了如何删除软件包。
  • “删除已安装的模块”一节介绍了如何删除从模块流中安装的内容。
  • 称为“重置模块流”的部分描述了将模块流重置为初始状态。
  • 名为“删除内容的命令”一节总结了删除内容的命令。

删除已安装的包

本节介绍如何删除包。

程序

  • 删除包：

    # yum remove package

    该包与任何其他依赖包一起被删除。

删除已安装的模块

删除模块会删除当前启用的模块流的配置文件所安装的所有软件包，以及依赖于这些软件包的任何其他软件包和模块。

从此模块流安装的软件包未在其任何配置文件中列出，仍保留在系统上，可以手动删除。

先决条件

  • 您要删除的模块必须已安装某些配置文件。
  • 您必须了解模块化依赖项解析 。

程序

  1. 列出从模块安装的包：

    $ yum module info module-name | grep module+el8 | sed 's/.*: //g;s/\n/ /g' | xargs yum list installed

    将module-name替换为模块的名称 。这将列出从此模块安装的所有软件包。

  2. 删除上一步中列出的包：

    # yum remove package

    使用上一步中列出的包替换包 。您可以提供由空格分隔的多个包名称。yum工具将显示更改摘要并要求确认。

  3. 将模块配置文件标记为已卸载：

    # yum module remove module-name

    当前启用的模块流保持启用状态。

  4. （可选）重置或禁用流。

重置模块流

重置模块是一种将其所有流恢复到其初始状态的操作 - 既不启用也不禁用。如果模块具有默认流，则该流由于重置模块而变为活动状态。

程序

  • 重置模块状态：

    # yum module reset module-name

    模块的所有流都返回到初始状态。没有删除已安装的内容。

删除内容的命令

本节列出了用于删除内容的常用命令。

命令列表

删除包
# yum remove package
删除已安装的模块流配置
# yum module remove module-name
将模块的所有流重置为初始状态
# yum module reset module-name
禁用模块及其所有流
# yum module disable module-name
