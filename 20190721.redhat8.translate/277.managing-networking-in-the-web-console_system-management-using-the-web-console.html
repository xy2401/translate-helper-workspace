Chapter 9. 在Web控制台中管理网络

RHEL 8 Web控制台支持基本网络配置。您可以：

  • 配置IPv4 / IPv6网络设置
  • 管理网桥
  • 管理VLAN
  • 管理团队
  • 管理债券
  • 检查网络日志

注意

RHEL 8 Web控制台构建于NetworkManager服务之上。

有关详细信息，请参阅使用NetworkManager管理网络入门 。

先决条件

  • 已安装并启用RHEL 8 Web控制台。

    有关NetworkManager的详细信息，请参阅安装Web控制台 。

在Web控制台中配置网桥

网桥用于将多个接口连接到具有相同IP地址范围的一个子网。

在Web控制台中添加网桥

本节介绍如何使用Web控制台在多个网络接口上创建软件桥接。

程序

  1. 登录RHEL Web控制台。

    有关详细信息，请参阅登录Web控制台 。

  2. 开放网络 。

  3. 单击“ 添加桥”按钮。

  4. 在“ 桥设置”对话框中，输入新桥的名称。
  5. 在“ 端口”字段中，选择要放入一个子网的接口。

  6. （可选）您可以选择生成树协议（STP）以避免桥接环路和广播辐射。

    如果您没有强烈的偏好，请保留预定义的值。

  7. 单击“ 创建”

如果成功创建了网桥，则Web控制台将在“ 网络”部分中显示新网桥。检查新桥接行中“ 发送”和“ 接收”列中的值。

如果您可以看到通过网桥发送和接收零字节，则连接无法正常工作，您需要调整网络设置。

在Web控制台中配置静态IP地址

系统的IP地址可以由DHCP服务器自动从池中分配，也可以手动配置IP地址。IP地址不受DHCP服务器设置的影响。

本节介绍如何使用RHEL Web控制台配置网桥的静态IPv4地址。

程序

  1. 登录RHEL Web控制台。

    有关详细信息，请参阅登录Web控制台 。

  2. 打开“ 网络”部分。

  3. 单击要设置静态IP地址的接口。

  4. 在界面详细信息屏幕中，单击IPv4配置。

  5. 在“ IPv4设置”对话框中，在“ 地址”下拉列表中选择“ 手动 ”。

  6. 单击“ 应用”

  7. 在“ 地址”字段中，输入所需的IP地址，网络掩码和网关。

  8. 单击“ 应用”

此时，已配置IP地址，接口使用新的静态IP地址。

使用Web控制台从网桥中删除接口

网桥可以包括多个接口。您可以从桥中删除它们。每个已删除的界面都将自动更改为独立界面。

本节介绍从RHEL 8系统中创建的软件桥中删除网络接口。

先决条件

  • 在系统中拥有一个具有多个接口的桥接器。

程序

  1. 登录RHEL Web控制台。

    有关详细信息，请参阅登录Web控制台 。

  2. 开放网络 。

  3. 单击要配置的网桥。

  4. 在网桥设置屏幕中，向下滚动到端口表（接口）。

  5. 选择界面，然后单击-图标。

RHEL 8 Web控制台从桥接器中删除接口，您可以在“ 网络”部分中将其作为独立接口查看。

在Web控制台中删除网桥

您可以在RHEL Web控制台中删除软件网桥。桥中包含的所有网络接口都将自动更改为独立接口。

先决条件

  • 在你的系统中有一座桥梁。

程序

  1. 登录RHEL Web控制台。

    有关详细信息，请参阅登录Web控制台 。

  2. 打开“ 网络”部分。

  3. 单击要配置的网桥。

  4. 在网桥设置屏幕中，向下滚动到端口表。

  5. 单击删除 。

在此阶段，返回Networking并验证Interfaces选项卡上是否显示所有网络接口。作为桥接器一部分的接口现在可以处于非活动状态。因此，您可能需要激活它们并手动设置网络参数。

在Web控制台中配置VLAN

VLAN（虚拟LAN）是在单个物理以太网接口上创建的虚拟网络。

每个VLAN由一个ID定义，该ID表示唯一的正整数，并作为独立接口使用。

以下过程介绍如何在RHEL Web控制台中创建VLAN。

先决条件

  • 在您的系统中具有网络接口。

程序

  1. 登录RHEL Web控制台。

    有关详细信息，请参阅登录Web控制台 。

  2. 开放网络 。

  3. 单击“ 添加VLAN”按钮。

  4. 在“ VLAN设置”对话框中，选择要为其创建VLAN的物理接口。
  5. 输入VLAN Id或只使用预定义的数字。

  6. 在“ 名称”字段中，您可以看到由父接口和VLAN ID组成的预定义名称。如果没有必要，请保留名称。

  7. 单击“ 应用”

已创建新VLAN，您需要单击VLAN并配置网络设置。
