Chapter 9. 软件管理

YUM堆栈的显着变化

在Red Hat Enterprise Linux（RHEL）8上，新版YUM工具确保安装软件，该工具基于DNF技术（ YUM v4 ）。

YUM v4优于YUM v3的优势

与以前在RHEL 7上使用的YUM v3相比， YUM v4具有以下优势：

  • 提高性能
  • 支持模块化内容
  • 精心设计的稳定API，可与工具集成

有关新YUM v4工具与RHEL 7以前版本YUM v3之间差异的详细信息，请参阅DNF CLI与YUM相比的更改 。

如何使用YUM v4

安装软件

从命令行使用，编辑或创建配置文件时， YUM v4与YUM v3兼容。

对于安装软件，您可以使用与RHEL 7相同的方式使用yum命令及其特定选项。

查看有关使用yum安装软件的更多详细信息。

插件的可用性

旧版YUM v3插件与新版YUM v4不兼容。选定的yum插件和实用程序已移植到新的DNF后端，并且可以使用与RHEL 7中相同的名称进行安装。它们还提供兼容性符号链接，因此可以在通常的位置找到二进制文件，配置文件和目录。

如果不再包含插件，或者替换不符合可用性需求，请联系红帽支持以请求功能增强，如我如何打开和管理客户支持案例中所述门户网站？

有关更多信息，请参阅插件接口 。

API的可用性

请注意， YUM v3提供的旧版Python API不再可用。建议用户将其插件和脚本迁移到YUM v4 （DNF Python API）提供的新API，该API是稳定且完全支持的。上游项目记录了新的DNF Python API - 请参阅DNF API参考 。

Libdnf和Hawkey API（包括C和Python）被认为是不稳定的，并且可能在RHEL 8生命周期中发生变化。

YUM v4的功能表现不同

一些YUM V3功能可以表现不同的YUM V4。如果任何此类更改对您的工作流程产生负面影响，请打开Red Hat支持案例，如如何在客户门户上打开和管理支持案例中所述？

yum list显示重复的条目

使用yum list命令列出软件包时，可能会显示重复的条目，每个存储库对应一个名称和版本相同的软件包。

这是有意的，它允许用户在必要时区分这些包。

例如，如果package-1.2在repo1和repo2中都可用，则YUM v4将打印两个实例：

[…​]
package-1.2    repo1
package-1.2    repo2
[…​]

相比之下，传统的YUM v3命令过滤掉了这样的重复项，因此只显示了一个实例：

[…​]
package-1.2    repo1
[…​]

值得注意的RPM功能和更改

红帽企业Linux（RHEL）8随RPM 4.14一起发布。此版本引入了许多针对RPM 4.11的增强功能，可在RHEL 7中使用。

值得注意的功能包括：

  • debuginfo包可以并行安装
  • 支持弱依赖
  • 支持rich或boolean依赖项
  • 支持打包大小超过4 GB的文件
  • 支持文件触发器

请参阅RHEL 8中有关新RPM功能的更多信息。

值得注意的变化包括：

  • 更严格的spec-parser
  • 简化签名检查非详细模式下的输出
  • 宏中的添加和弃用
