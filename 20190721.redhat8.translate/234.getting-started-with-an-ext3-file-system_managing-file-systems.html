Chapter 9. 开始使用ext3文件系统

作为系统管理员，您可以创建，装载，调整大小，备份和还原ext3文件系统。ext3文件系统本质上是ext2文件系统的增强版本。

ext3文件系统的功能

以下是ext3文件系统的功能：

  • 可用性：意外断电或系统崩溃后，由于提供了日记功能，因此不需要进行文件系统检查。默认日志大小大约需要一秒钟才能恢复，具体取决于硬件的速度

    注意

    ext3中唯一支持的日记记录模式是data=ordered （默认值）。有关更多信息，请参阅RHEL是否支持EXT日记选项“data = writeback”？ 知识库文章。

  • 数据完整性：ext3文件系统可防止在意外电源故障或系统崩溃期间丢失数据完整性。
  • 速度：尽管不止一次写入一些数据，但ext3在大多数情况下的吞吐量高于ext2，因为ext3的日志功能可优化硬盘驱动器的运动。
  • 轻松过渡：很容易从ext2迁移到ext3，并且无需重新格式化即可获得强大的日记文件系统的优势。

创建ext3文件系统

作为系统管理员，您可以使用mkfs.ext3命令在块设备上创建ext3文件系统。

先决条件

  • 磁盘上的分区。有关创建MBR或GPT分区的信息，请参阅“在磁盘上创建分区表”一节 。

    或者，使用LVM或MD卷。

程序

  1. 要创建ext3文件系统：

    • 对于常规分区设备，LVM卷，MD卷或类似设备，请使用以下命令：

      # mkfs.ext3 /dev/block_device

      将/ dev / block_device替换为块设备的路径。

      例如， /dev/sdb1 ， /dev/disk/by-uuid/05e99ec8-def1-4a5e-8a9d-5945339ceb2a ，或/dev/my-volgroup/my-lv 。通常，默认选项对于大多数使用方案是最佳的。

    注意

    • 在创建文件系统时指定UUID：

      # mkfs.ext3 -U UUID /dev/block_device

      将UUID替换为您要设置的UUID：例如， 7cd65de3-e0be-41d9-b66d-96d749c02da7 。

      将/ dev / block_device替换为ext3文件系统的路径以添加UUID：例如， /dev/sda8 。

    • 在创建文件系统时指定标签：

      # mkfs.ext3 -L label-name /dev/block_device

  2. 要查看创建的ext3文件系统：

    # blkid

挂载ext3文件系统

作为系统管理员，您可以使用mount实用程序安装ext3文件系统。

先决条件

  • 一个ext3文件系统。有关创建ext3文件系统的信息，请参阅“创建ext3文件系统”一节 。

程序

  1. 要创建装入点以装入文件系统：

    # mkdir /mount/point

    用必须创建分区的挂载点的目录名替换/ mount / point 。

  2. 要挂载ext3文件系统：

    • 要安装没有额外选项的ext3文件系统：

      # mount /dev/block_device /mount/point
    • 要永久挂载文件系统，请参阅“永久挂载文件系统”一节 。

  3. 要查看已安装的文件系统：

    # df -h

调整ext3文件系统的大小

作为系统管理员，您可以使用resize2fs实用程序调整ext3文件系统的大小。resize2fs实用程序以文件系统块大小为单位读取大小，除非使用指示特定单元的后缀。以下后缀表示具体单位：

  • s（扇区） - 512字节扇区
  • K（千字节） - 1,024字节
  • M（兆字节） - 1,048,576字节
  • G（千兆字节） - 1,073,741,824字节
  • T（兆兆字节） - 1,099,511,627,776字节

先决条件

  • 一个ext3文件系统。有关创建ext3文件系统的信息，请参阅“创建ext3文件系统”一节 。
  • 适当大小的基础块设备，用于在调整大小后保存文件系统。

程序

  1. 要调整ext3文件系统的大小，请执行以下步骤：

    • 要缩小和增大未安装的ext3文件系统的大小：

      # umount /dev/block_device
# e2fsck -f /dev/block_device
# resize2fs /dev/block_device size

      将/ dev / block_device替换为块设备的路径，例如/dev/sdb1 。

      使用s ， K ， M ， G和T后缀将大小替换为所需的调整大小值。

    • 使用resize2fs命令挂载时可以生成ext3文件系统：

      # resize2fs /mount/device size

      注意

      扩展时，size参数是可选的（通常是冗余的）。resize2fs自动扩展以填充容器的可用空间，通常是逻辑卷或分区。

  2. 要查看已调整大小的文件系统：

    # df -h
