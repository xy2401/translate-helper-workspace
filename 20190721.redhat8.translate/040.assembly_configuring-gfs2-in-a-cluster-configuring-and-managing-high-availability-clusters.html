Chapter 9. 集群中的GFS2文件系统

本节提供：

  • 设置包含GFS2文件系统的Pacemaker集群的过程

在群集中配置GFS2文件系统

此过程概述了设置包含GFS2文件系统的Pacemaker群集所需的步骤。此示例在三个逻辑卷上创建三个GFS2文件系统。

作为此过程的先决条件，您必须在所有节点上安装和启动集群软件，并创建基本的双节点集群。您还必须为群集配置防护。有关创建Pacemaker集群和为集群配置防护的信息，请参阅使用Pacemaker创建Red Hat高可用性集群 。

  1. 在群集的两个节点上，安装lvm2-lockd ， gfs2-utils和dlm软件包。lvm2-lockd软件包是AppStream通道的一部分， gfs2-utils和dlm软件包是Resilient Storage通道的一部分。 

    yum install lvm2-lockd gfs2-utils dlm

  2. 设置dlm资源。这是在群集中配置GFS2文件系统所必需的依赖项。此示例将dlm资源创建为名为locking的资源组的一部分。 

    [root @ z1~] pcs resource create dlm --group locking ocf:pacemaker:controld op monitor interval=30s on-fail=fence

  3. 克隆locking资源组，以便资源组可以在群集的两个节点上处于活动状态。 

    [root @ z1~] pcs resource clone locking interleave=true

  4. 设置lvmlockd资源作为组locking一部分。 

    [root @ z1~] pcs resource create lvmlockd --group locking ocf:heartbeat:lvmlockd op monitor interval=30s on-fail=fence

  5. 检查群集的状态以确保已在群集的两个节点上启动locking资源组。 

    [root @ z1~] pcs status --full群集名称：my_cluster [...]在线：[z1.example.com（1）z2.example.com（2）]完整资源列表：smoke-apc（stonith：fence_apc）：已启动z1.example.com克隆集：locked-clone [locking]资源组：锁定：0 dlm（ocf :: pacemaker：controld）：已启动z1.example.com lvmlockd（ocf :: heartbeat：lvmlockd）：已启动z1.example.com资源组：锁定：1 dlm（ocf :: pacemaker： controld）：已启动z2.example.com lvmlockd（ocf :: heartbeat：lvmlockd）：已启动z2.example.com已开始：[z1.example.com z2.example.com]

  6. 验证lvmlockd守护程序是否在群集的两个节点上运行。 

    [root @ z1~] ps -ef | grep lvmlockd root 12257 1 0 17:45？00:00:00 lvmlockd -p /run/lvmlockd.pid -A 1 -g dlm [root @ z2~]＃ps ps -ef | grep lvmlockd root 12270 1 0 17:45？00:00:00 lvmlockd -p /run/lvmlockd.pid -A 1 -g dlm

  7. 在群集的一个节点上，创建两个共享卷组。一个卷组将包含两个GFS2文件系统，另一个卷组将包含一个GFS2文件系统。

    以下命令在/dev/vdb上创建共享卷组shared_vg1 。 

    [root @ z1~] vgcreate --shared shared_vg1 /dev/vdb成功创建物理卷“/ dev / vdb”。卷组“shared_vg1”成功创建了VG shared_vg1，启动dlm lockspace开始锁定。等到锁准备好了......

    以下命令在/dev/vdc上创建共享卷组shared_vg2 。 

    [root @ z1~] vgcreate --shared shared_vg2 /dev/vdc成功创建物理卷“/ dev / vdc”。卷组“shared_vg2”成功创建了VG shared_vg2，启动dlm lockspace开始锁定。等到锁准备好了......

  8. 在群集中的第二个节点上，为每个共享卷组启动锁定管理器。 

    [root @ z2~] vgchange --lock-start shared_vg1 VG shared_vg1启动dlm lockspace开始锁定。等到锁准备好了......[root @ z2~] vgchange --lock-start shared_vg2 VG shared_vg2启动dlm lockspace开始锁定。等到锁准备好了......

  9. 在群集中的一个节点上，创建共享逻辑卷并使用GFS2文件系统格式化卷。确保为群集中的每个节点创建足够的日志。 

    [root @ z1~] lvcreate --activate sy -L5G -n shared_lv1 shared_vg1创建逻辑卷“shared_lv1”。[root @ z1~] lvcreate --activate sy -L5G -n shared_lv2 shared_vg1创建逻辑卷“shared_lv2”。[root @ z1~] lvcreate --activate sy -L5G -n shared_lv1 shared_vg2创建逻辑卷“shared_lv1”。[root @ z1~] # mkfs.gfs2 -j2 -p lock_dlm -t my_cluster:gfs2-demo1 /dev/shared_vg1/shared_lv1 [root @ z1~] # mkfs.gfs2 -j2 -p lock_dlm -t my_cluster:gfs2-demo2 /dev/shared_vg1/shared_lv2 [root @ z1~] # mkfs.gfs2 -j2 -p lock_dlm -t my_cluster:gfs2-demo3 /dev/shared_vg2/shared_lv1

  10. 为每个逻辑卷创建LVM-activate资源，以自动激活所有节点上的逻辑卷。

    1. 为卷组shared_vg1的逻辑卷shared_lv1创建名为sharedlv1的LVM-activate资源。此命令还会创建包含资源的资源组shared_vg1 。在此示例中，资源组与包含逻辑卷的共享卷组具有相同的名称。 

      [root @ z1~] pcs resource create sharedlv1 --group shared_vg1 ocf:heartbeat:LVM-activate lvname=shared_lv1 vgname=shared_vg1 activation_mode=shared vg_access_mode=lvmlockd

    2. 为卷组shared_vg1的逻辑卷shared_lv2创建名为sharedlv2的LVM-activate资源。此资源也将是资源组shared_vg1 。 

      [root @ z1~] pcs resource create sharedlv2 --group shared_vg1 ocf:heartbeat:LVM-activate lvname=shared_lv2 vgname=shared_vg1 activation_mode=shared vg_access_mode=lvmlockd

    3. 为卷组shared_vg2的逻辑卷shared_lv1创建名为sharedlv3的LVM-activate资源。此命令还会创建包含资源的资源组shared_vg2 。 

      [root @ z1~] pcs resource create sharedlv3 --group shared_vg2 ocf:heartbeat:LVM-activate lvname=shared_lv1 vgname=shared_vg2 activation_mode=shared vg_access_mode=lvmlockd

  11. 克隆两个新资源组。 

    [root @ z1~] pcs resource clone shared_vg1 interleave=true [root @ z1~] pcs resource clone shared_vg2 interleave=true

  12. 配置排序约束以确保首先启动包含dlm和lvmlockd资源的locking资源组。 

    [root @ z1~] pcs constraint order start locking-clone then shared_vg1-clone添加锁定克隆shared_vg1-clone（种类：必填）（选项：first-action = start then-action = start）[root @ z1~] pcs constraint order start locking-clone then shared_vg2-clone添加锁定克隆shared_vg2-clone（种类：必填）（选项：first-action = start then-action = start）

  13. 在群集中的两个节点上，验证逻辑卷是否处于活动状态。可能会延迟几秒钟。 

    [根@ Z1〜]＃ lvs LV VG的Attr LSIZE shared_lv1 shared_vg1 -Wi-A -----5.00克shared_lv2 shared_vg1 -Wi-A -----5.00克shared_lv1 shared_vg2 -Wi-A ----- 5.00克[根@ Z2〜]＃ lvs LV VG的Attr LSIZE shared_lv1 shared_vg1 -Wi-A -----5.00克shared_lv2 shared_vg1 -Wi-A -----5.00克shared_lv1 shared_vg2 -Wi-A ----- 5.00克

  14. 创建文件系统资源以在所有节点上自动挂载每个GFS2文件系统。

    您不应将文件系统添加到/etc/fstab文件，因为它将作为Pacemaker群集资源进行管理。可以使用options= options将挂载选项指定为资源配置的一部分。运行pcs resource describe Filesystem命令以获取完整配置选项。

    以下命令可创建文件系统资源。这些命令将每个资源添加到包含该文件系统的逻辑卷资源的资源组。 

    [root @ z1~] pcs resource create sharedfs1 --group shared_vg1 ocf:heartbeat:Filesystem device="/dev/shared_vg1/shared_lv1" directory="/mnt/gfs1" fstype="gfs2" options=noatime op monitor interval=10s on-fail=fence [root @ z1~] pcs resource create sharedfs2 --group shared_vg1 ocf:heartbeat:Filesystem device="/dev/shared_vg1/shared_lv2" directory="/mnt/gfs2" fstype="gfs2" options=noatime op monitor interval=10s on-fail=fence [root @ z1~] pcs resource create sharedfs3 --group shared_vg2 ocf:heartbeat:Filesystem device="/dev/shared_vg2/shared_lv1" directory="/mnt/gfs3" fstype="gfs2" options=noatime op monitor interval=10s on-fail=fence

  15. 验证GFS2文件系统是否已安装在群集的两个节点上。 

    [root @ z1~] mount | grep gfs2 / mnt / gfs1上的mount | grep gfs2 / dev / mapper / shared_vg1-shared_lv1类型gfs2（rw，noatime，seclabel）/ dev / mapper / shared_vg1-shared_lv2 / mnt / gfs2类型gfs2（rw，noatime，seclabel）/ dev / mapper / mnt / gfs3上的/ shared_vg2-shared_lv1类型gfs2（rw，noatime，seclabel）[root @ z2~] mount | grep gfs2 / mnt / gfs1上的mount | grep gfs2 / dev / mapper / shared_vg1-shared_lv1类型gfs2（rw，noatime，seclabel）/ dev / mapper / shared_vg1-shared_lv2 / mnt / gfs2类型gfs2（rw，noatime，seclabel）/ dev / mapper / mnt / gfs3上的/ shared_vg2-shared_lv1类型gfs2（rw，noatime，seclabel）

  16. 检查群集的状态。 

    [root @ z1~] pcs status --full群集名称：my_cluster [... 1完整资源列表：smoke-apc（stonith：fence_apc）：已启动z1.example.com克隆集：locked-clone [locking]资源组：锁定：0 dlm（ocf :: pacemaker：controld）：已启动z2.example.com lvmlockd（ocf :: heartbeat：lvmlockd）：已启动z2.example.com资源组：锁定：1 dlm（ocf :: pacemaker） ：controld）：已启动z1.example.com lvmlockd（ocf :: heartbeat：lvmlockd）：已启动z1.example.com已启动：[z1.example.com z2.example.com]克隆集：shared_vg1-clone [shared_vg1]资源组：shared_vg1：0 sharedlv1（ocf :: heartbeat：LVM-activate）：已启动z2.example.com sharedlv2（ocf :: heartbeat：LVM-activate）：已启动z2.example.com sharedfs1（ocf :: heartbeat：Filesystem） ：已启动z2.example.com sharedfs2（ocf :: heartbeat：Filesystem）：已启动z2.example.com资源组：shared_vg1：1 sharedlv1（ocf :: heartbeat：LVM-activate）：已启动z1.example.com sharedlv2（ocf） :: heartbeat：LVM-activate）：已启动z1.example.com sharedfs1（ocf :: heartbeat ：Filesystem）：已启动z1.example.com sharedfs2（ocf :: heartbeat：Filesystem）：已启动example.co已启动：[z1.example.com z2.example.com]克隆集：shared_vg2-clone [shared_vg2]资源组： shared_vg2：0 sharedlv3（ocf :: heartbeat：LVM-activate）：已启动z2.example.com sharedfs3（ocf :: heartbeat：Filesystem）：已启动z2.example.com资源组：shared_vg2：1 sharedlv3（ocf :: heartbeat： LVM-activate）：已启动z1.example.com sharedfs3（ocf :: heartbeat：Filesystem）：已启动z1.example.com已启动：[z1.example.com z2.example.com] ...

将GFS2文件系统从RHEL7迁移到RHEL8

在Red Hat Enterprise Linux 8中，LVM使用LVM锁守护程序lvmlockd而不是clvmd来管理主动/主动群集中的共享存储设备。这要求您将主动/主动群集所需的逻辑卷配置为共享逻辑卷。此外，这要求您使用LVM-activate资源来管理LVM卷，并使用lvmlockd资源代理来管理lvmlockd守护程序。有关配置包含使用共享逻辑卷的GFS2文件系统的Pacemaker群集的完整过程，请参阅在群集中配置GFS2文件系统。

要在配置包含GFS2文件系统的RHEL8群集时使用现有的Red Hat Enterprise Linux 7逻辑卷，请从RHEL8群集执行以下过程。在此示例中，群集RHEL 7逻辑卷是卷组upgrade_gfs_vg 。

注意

RHEL8群集必须与包含GFS2文件系统的RHEL7群集具有相同的名称，以使现有文件系统有效。

  1. 确保包含GFS2文件系统的逻辑卷当前处于非活动状态。仅当所有节点都已停止使用卷组时，此过程才是安全的。

  2. 从群集中的一个节点强制将卷组更改为本地。 

    [root @ rhel8-01~] #vgchange vgchange --lock-type none --lock-opt force upgrade_gfs_vg强制将VG锁定类型更改为none？[y / n]：y卷组“upgrade_gfs_vg”已成功更改

  3. 从群集中的一个节点，将本地卷组更改为共享卷组

    [root @ rhel8-01~] #vgchange vgchange --lock-type dlm upgrade_gfs_vg卷组“upgrade_gfs_vg”已成功更改

  4. 在群集中的每个节点上，开始锁定卷组。 

    [root @ rhel8-01~] #vgchange vgchange --lock-start upgrade_gfs_vg VG upgrade_gfs_vg启动dlm lockspace开始锁定。等到锁准备好了......[root @ rhel8-02~] #vgchange vgchange --lock-start upgrade_gfs_vg VG upgrade_gfs_vg启动dlm lockspace开始锁定。等到锁准备好了......

执行此过程后，您可以为每个逻辑卷创建LVM-activate资源。
