Chapter 9. 概观

Red Hat Enterprise Linux 8.0基于Fedora 28和上游内核4.18，为用户提供了跨混合云部署的稳定，安全，一致的基础，以及支持传统和新兴工作负载所需的工具。该版本的亮点包括：

分配

  • 内容可通过BaseOS和Application Stream（ AppStream ）存储库获得。
  • AppStream存储库支持传统RPM格式的新扩展 - 模块 。这允许组件的多个主要版本可用于安装。

有关详细信息，请参阅第3章， RHEL 8中的内容分发 。

软件管理

  • YUM包管理器现在基于DNF技术，它提供对模块化内容的支持，提高的性能以及与工具集成的精心设计的稳定API。

有关详细信息，请参阅“软件管理”一节 。

外壳和命令行工具

  • RHEL 8提供以下版本控制系统 ： Git 2.18 ， Mercurial 4.8和Subversion 1.10 。

有关详细信息，请参阅“Shell和命令行工具”一节 。

动态编程语言，Web和数据库服务器

  • Python 3.6是RHEL 8中的默认Python实现;提供了对Python 2.7有限支持。默认情况下不安装任何版本的Python。
  • Node.js是RHEL中的新功能。自RHEL 7以来，其他动态编程语言已经更新： PHP 7.2 ，Ruby 2.5 ， Perl 5.26 ， SWIG 3.0现已推出。
  • 以下数据库服务器随RHEL 8一起分发： MariaDB 10.3 ， MySQL 8.0 ， PostgreSQL 10 ， PostgreSQL 9.6和Redis 5 。
  • RHEL 8提供Apache HTTP Server 2.4并引入了一个新的Web服务器 nginx 1.14 。
  • Squid已更新至4.4版，现在包含一个新的代理缓存服务器 ： Varnish Cache 6.0 。

有关详细信息，请参阅“动态编程语言，Web和数据库服务器”一节 。

桌面

  • GNOME Shell已重新定位到版本3.28。
  • GNOME会话和GNOME显示管理器使用Wayland作为其默认显示服务器。X.Org服务器也是RHEL 7中的默认显示服务器。

有关详细信息，请参阅“桌面”一节 。

安装程序和映像创建

  • Anaconda安装程序可以使用LUKS2磁盘加密，并在NVDIMM设备上安装系统。
  • Image Builder工具使用户能够以各种格式创建自定义系统映像，包括为在各种提供商的云上部署而准备的映像。
  • RHEL 8中提供了使用IBM Z上的硬件管理控制台（ HMC ）和支持元素（ SE ）从DVD进行安装。

有关更多详细信息，请参阅“安装程序和映像创建”一节。

核心

  • 扩展的Berkeley数据包过滤（ eBPF）功能使用户空间可以将自定义程序附加到各种点（套接字，跟踪点，数据包接收）以接收和处理数据。此功能作为技术预览版提供 。
  • BPF编译器集合（ BCC ）是一种用于创建高效内核跟踪和操作程序的工具，可作为技术预览版提供 。

有关更多信息，请参阅“内核”一节 。

文件系统和存储

  • LUKS版本2（ LUKS2 ）格式取代了传统的LUKS（LUKS1）格式。dm-crypt子系统和cryptsetup工具现在使用LUKS2作为加密卷的默认格式。

有关详细信息，请参阅“文件系统和存储”一节 。

安全

  • 默认情况下，将应用系统范围的加密策略 ，该策略配置核心加密子系统，涵盖TLS，IPsec，SSH，DNSSEC和Kerberos协议。使用新的update-crypto-policies命令，管理员可以轻松地在模式之间切换：default，legacy，future和fips。
  • 使用PKCS＃11支持智能卡和硬件安全模块（ HSM ）现在在整个系统中保持一致。

有关详细信息，请参阅“安全性”一节 。

联网

  • nftables框架替换默认网络数据包过滤工具角色中的iptables 。
  • firewalld守护程序现在使用nftables作为其默认后端。
  • 已经引入了对支持多个容器的网络连接的IPVLAN虚拟网络驱动程序的支持。
  • 作为扩展伯克利数据包过滤（ eBPF）功能的一部分，eXpress数据路径（ XDP ），用于流量控制的XDP（ tc ）和地址族eXpress数据路径（ AF_XDP ）可用作技术预览 。有关更多详细信息，请参阅“网络”一节 。

有关其他功能，请参阅“网络”一节 。

虚拟化

  • 现在支持并在RHEL 8中创建的虚拟机中自动配置更现代的基于PCI Express的机器类型（ Q35 ）。这为虚拟设备的功能和兼容性提供了各种改进。
  • 现在可以使用RHEL 8 Web控制台（也称为Cockpit）创建和管理虚拟机。
  • QEMU仿真器引入了沙盒功能，该功能为QEMU可以执行的系统调用提供了可配置的限制，从而使虚拟机更加安全。

有关详细信息，请参阅“虚拟化”一节 。

编译器和开发工具

  • 基于版本8.2的GCC编译器支持更新的C ++语言标准版本，更好的优化，新的代码强化技术，改进的警告和新的硬件功能。
  • 用于代码生成，操作和调试的各种工具现在可以通过实验方式处理DWARF5调试信息格式。
  • 对于某些工具，例如BCC ， PCP和SystemTap ，可以使用对eBPF跟踪的内核支持。
  • 基于版本2.28的glibc库增加了对Unicode 11的支持，更新的Linux系统调用，DNS存根解析器的关键改进，额外的安全加固以及改进的性能。
  • RHEL 8提供OpenJDK 11，OpenJDK 8，IcedTea-Web和各种Java工具，如Ant ， Maven或Scala 。

有关其他详细信息，请参阅“编译器和开发工具”一节 。

高可用性和集群

  • Pacemaker群集资源管理器已升级到上游版本2.0.0，它提供了许多错误修复和增强功能。
  • 在RHEL 8中， pcs配置系统完全支持Corosync 3， knet和节点名称。

有关详细信息，请参阅“高可用性和群集”一节 。
