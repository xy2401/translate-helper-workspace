Chapter 9. 不推荐使用的功能

本部分概述了Red Hat Enterprise Linux 8.0中已弃用的功能。

在Red Hat Enterprise Linux 8生命周期结束之前，将继续支持不推荐使用的功能。在此产品的未来主要版本中可能不支持不推荐使用的功能，不建议用于新部署。有关特定主要版本中最新的已弃用功能列表，请参阅最新版本的发行文档。

建议不要将弃用的硬件组件用于当前或未来主要版本的新部署。硬件驱动程序更新仅限于安全性和关键修复程序。Red Hat建议在合理可行的情况下尽快更换此硬件。

可以弃用软件包，不建议将其用于进一步使用。在某些情况下，可以从产品中删除包装。然后，产品文档会识别更新的软件包，这些软件包提供与已弃用的软件类似，相同或更高级的功能，并提供进一步的建议。

有关RHEL 7中存在但已在RHEL 8中删除的功能的信息，请参阅采用RHEL 8时的注意事项 。

安装程序和映像创建

不推荐使用--interactive Kickstart命令的ignoredisk选项

在未来版本的Red Hat Enterprise Linux中使用--interactive option将导致致命的安装错误。建议您修改Kickstart文件以删除该选项。

（BZ＃1637872）

几个Kickstart命令和选项已被弃用

使用RHEL 8中的以下命令和选项Kickstart文件将在日志中打印警告。

  • auth或authconfig
  • device
  • deviceprobe
  • dmraid
  • install
  • lilo
  • lilocheck
  • mouse
  • multipath
  • bootloader --upgrade
  • ignoredisk --interactive
  • partition --active
  • reboot --kexec

如果仅列出特定选项，则base命令及其他选项仍可用，不会弃用。

有关Kickstart的更多详细信息和相关更改，请参阅采用RHEL 8文档时的注意事项的Kickstart更改部分。

（BZ＃1642765）

文件系统和存储

已禁用UDP上的NFSv3

默认情况下，NFS服务器不再打开或侦听用户数据报协议（UDP）套接字。此更改仅影响NFS版本3，因为版本4需要传输控制协议（TCP）。

RHEL 8不再支持NFS over UDP。

（BZ＃1592011）

不推荐使用elevator内核命令行参数

在早期的RHEL版本中使用了elevator内核命令行参数来为所有设备设置磁盘调度程序。在RHEL 8中，不推荐使用该参数。

上游Linux内核已取消对elevator参数的支持，但出于兼容性原因，它仍可在RHEL 8中使用。

请注意，内核根据设备类型选择默认磁盘调度程序。这通常是最佳设置。如果您需要不同的调度程序，Red Hat建议您使用udev规则或Tuned服务进行配置。匹配所选设备并仅为这些设备切换调度程序。

有关详细信息，请参阅以下文章： 为什么'elevator ='参数在RHEL8中不再起作用 。

（BZ＃1665295）

VDO包中的VDO Ansible模块

对VDO Ansible模块当前被提供vdo RPM包。在将来的版本中，VDO Ansible模块将被移动到Ansible RPM软件包。

（ BZ＃1669537 ）

联网

RHEL 8中不推荐使用网络脚本

Red Hat Enterprise Linux 8中不推荐使用网络脚本，默认情况下不再提供它们。基本安装提供了ifup和ifdown脚本的新版本，它通过nmcli工具调用NetworkManager服务。在Red Hat Enterprise Linux 8中，要运行ifup和ifdown脚本，必须运行NetworkManager。

请注意， /sbin/ifup-local ， ifdown-pre-local和ifdown-local脚本中的自定义命令不会执行。

如果需要这些脚本中的任何一个，则仍可以使用以下命令在系统中安装已弃用的网络脚本：

~]# yum install network-scripts

ifup和ifdown脚本链接到已安装的旧网络脚本。

调用旧网络脚本会显示有关其弃用的警告。

（BZ＃1647725）

安全

DSA在RHEL 8中已弃用

数字签名算法（DSA）在Red Hat Enterprise Linux 8中被认为已弃用。依赖于DSA密钥的身份验证机制在默认配置中不起作用。请注意，即使在OpenSSH系统范围的加密策略级别， LEGACY客户端也不接受DSA主机密钥。

（BZ＃1646541）

SSL2 Client Hello已在NSS弃用

传输层安全性（ TLS ）协议版本1.2及更早版本允许以与向后兼容安全套接字层（ Client Hello ）协议版本2的方式格式化的SSL消息开始协商。不推荐使用网络安全服务（ NSS ）库中对此功能的支持，默认情况下禁用此功能。

需要支持此功能的应用程序需要使用新的SSL_ENABLE_V2_COMPATIBLE_HELLO API来启用它。在未来的Red Hat Enterprise Linux 8版本中，可能会完全删除对此功能的支持。

（BZ＃1645153）

不推荐使用TLS 1.0和TLS 1.1

在DEFAULT系统范围的加密策略级别禁用TLS 1.0和TLS 1.1协议。如果您的方案（例如，Firefox Web浏览器中的视频会议应用程序）需要使用已弃用的协议，请将系统范围的加密策略切换到LEGACY级别：

# update-crypto-policies --set LEGACY

有关更多信息，请参阅RHEL 8中的强加密默认值以及 Red Hat Customer Portal和update-crypto-policies(8)手册页上的弱加密算法知识库文章的弃用 。

（ BZ＃1660839 ）

虚拟化

RHEL 8中未正确支持虚拟机快照

目前创建虚拟机（VM）快照的机制已被弃用，因为它无法可靠地运行。因此，建议不要在RHEL 8中使用VM快照。

请注意，新的VM快照机制正在开发中，将在RHEL 8的未来次要版本中完全实现。

（ BZ＃1686057 ）

Cirrus VGA虚拟GPU类型已被弃用

随着红帽企业Linux的未来重大更新，KVM虚拟机将不再支持Cirrus VGA GPU设备。因此，Red Hat建议使用stdvga ， virtio-vga或qxl设备代替Cirrus VGA。

（BZ＃1651994）

virt-manager已被弃用

Virtual Machine Manager应用程序（也称为virt-manager ）已被弃用。RHEL 8 Web控制台（也称为Cockpit ）旨在成为后续版本的替代品。因此，建议您使用Web控制台在GUI中管理虚拟化。但是，在Red Hat Enterprise Linux 8.0中，某些功能只能从virt-manager或命令行访问。

（JIRA：RHELPLAN-10304）
