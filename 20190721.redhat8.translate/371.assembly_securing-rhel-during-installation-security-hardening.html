Chapter 9. 在安装过程中保护RHEL

甚至在开始安装Red Hat Enterprise Linux之前，安全性就开始了。从一开始就安全地配置系统，以便以后更容易实现其他安全设置。

BIOS和UEFI安全性

BIOS（或等效的BIOS）和引导加载程序的密码保护可以防止对系统进行物理访问的未授权用户使用可移动介质进行引导或通过单用户模式获取root权限。您应采取的防范此类攻击的安全措施取决于工作站上信息的敏感性和机器的位置。

例如，如果在贸易展览中使用机器并且不包含敏感信息，则防止此类攻击可能并不重要。但是，如果员工的笔记本电脑带有用于公司网络的私有，未加密的SSH密钥在同一个贸易展览会上无人看管，则可能导致严重的安全漏洞，并对整个公司产生影响。

但是，如果工作站位于只有经过授权或信任的人才能访问的位置，则可能不需要保护BIOS或引导加载程序。

BIOS密码

密码保护计算机BIOS的两个主要原因是[1] ：

  1. 防止更改BIOS设置 - 如果入侵者可以访问BIOS，则可以将其设置为从CD-ROM或闪存驱动器启动。这使他们可以进入救援模式或单用户模式，这反过来又允许他们在系统上启动任意进程或复制敏感数据。
  2. 防止系统启动 - 某些BIOS允许启动过程的密码保护。激活后，攻击者必须在BIOS启动引导加载程序之前输入密码。

由于设置BIOS密码的方法因计算机制造商而异，因此请参阅计算机手册以获取具体说明。

如果您忘记了BIOS密码，可以使用主板上的跳线或断开CMOS电池来重置密码。因此，最好在可能的情况下锁定计算机机箱。但是，在尝试断开CMOS电池之前，请参阅计算机或主板的手册。

磁盘分区

Red Hat建议为/boot ， / ， /home /tmp和/var/tmp/目录创建单独的分区。每个的原因都不同，我们将解决每个分区。

/boot
此分区是系统在引导期间读取的第一个分区。用于将系统引导至Red Hat Enterprise Linux 8的引导加载程序和内核映像存储在此分区中。不应加密此分区。如果此分区包含在/中且该分区已加密或以其他方式变为不可用，则您的系统将无法启动。
/home
当用户数据（ /home ）存储在/而不是单独的分区中时，分区可能会填满，导致操作系统变得不稳定。此外，当您将系统升级到下一版本的Red Hat Enterprise Linux 8时，将数据保存在/home分区中会更容易，因为在安装过程中不会覆盖它。如果根分区（ / ）损坏，您的数据可能会永远丢失。通过使用单独的分区，可以防止数据丢失。您还可以将此分区作为常用备份的目标。
/tmp和/var/tmp/
/tmp和/var/tmp/目录都用于存储不需要长时间存储的数据。但是，如果大量数据充斥其中一个目录，则会占用您的所有存储空间。如果发生这种情况并且这些目录存储在/那么您的系统可能会变得不稳定并崩溃。因此，将这些目录移动到自己的分区中是个好主意。

注意

在安装过程中，会向您显示加密分区的选项。用户必须提供密码。此密码短语将用作解锁批量加密密钥的密钥，该密钥用于保护分区的数据。

在安装过程中限制网络连接

安装Red Hat Enterprise Linux 8时，安装介质代表特定时间系统的快照。因此，它可能无法与最新的安全修复程序保持同步，并且可能容易受到仅在安装介质提供的系统发布后修复的某些问题的影响。

安装可能存在漏洞的操作系统时，请始终将暴露限制在最近的必要网络区域。最安全的选择是“无网络”区域，这意味着在安装过程中将机器断开连接。在某些情况下，LAN或Intranet连接就足够了，而Internet连接是最危险的。要遵循最佳安全实践，请在从网络安装Red Hat Enterprise Linux 8时选择与存储库最近的区域。 ：实验：

安装所需的最少数量的包

最佳做法是仅安装您将使用的软件包，因为您计算机上的每个软件都可能包含漏洞。如果要从DVD介质进行安装，请抓住机会在安装过程中准确选择要安装的软件包。如果您发现需要其他软件包，可以随时将其添加到系统中。

安装后程序

以下步骤是安装Red Hat Enterprise Linux后应立即执行的安全相关过程。

  1. 更新您的系统。以root身份输入以下命令：

    # dnf update

  2. 即使防火墙服务firewalld通过安装Red Hat Enterprise Linux自动启用，也有可能明确禁用它的情况，例如在kickstart配置中。在这种情况下，建议考虑重新启用防火墙。

    要启动firewalld ，请以root身份输入以下命令：

    # systemctl start firewalld
# systemctl enable firewalld

  3. 要增强安全性，请禁用不需要的服务。例如，如果计算机上未安装打印机，请使用以下命令禁用cups服务：

    # systemctl disable cups

    要查看活动服务，请输入以下命令：

    $ systemctl list-units | grep service

[1]由于制造商之间的系统BIOS不同，有些可能不支持任何一种类型的密码保护，而其他类型可能支持一种类型但不支持另一种类型。
