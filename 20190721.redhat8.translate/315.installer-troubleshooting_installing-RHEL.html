附录C.故障排除

以下各节介绍了在诊断安装问题时可能有用的各种故障排除信息。

安装过程中的控制台和日志记录

Red Hat Enterprise Linux安装程序使用tmux终端多路复用器来显示和控制除主界面之外可以使用的多个窗口。这些窗口中的每一个都有不同的用途 - 它们显示多个不同的日志，可用于解决安装过程中的任何问题，其中一个窗口提供具有root权限的交互式shell提示符，除非使用引导专门禁用此提示选项或Kickstart命令。

注意

通常，除非您需要诊断安装问题，否则没有理由离开默认的图形安装环境。

终端多路复用器在虚拟控制台1中运行。要从实际安装环境切换到tmux ，请按Ctrl + Alt + F1 。要返回在虚拟控制台6中运行的主安装界面，请按Ctrl + Alt + F6 。

注意

如果选择文本模式安装，则将在虚拟控制台1（ tmux ）中启动，切换到控制台6将打开shell提示符而不是图形界面。

运行tmux的控制台有5个可用窗口;它们的内容在下表中描述，以及用于访问它们的键盘快捷键。请注意，键盘快捷键分为两部分：首先按Ctrl + b ，然后释放两个键，并按下要使用的窗口的数字键。

您也可以使用Ctrl + b n和Ctrl + b p分别切换到下一个或上一个tmux窗口。

表C.1。可用的tmux窗口

捷径内容

Ctrl + b 1

主安装程序窗口。包含基于文本的提示（在文本模式安装期间或使用VNC直接模式时），以及一些调试信息。

Ctrl + b 2

具有root权限的交互式shell提示符。

Ctrl + b 3

安装日志;显示存储在/tmp/anaconda.log消息。

Ctrl + b 4

存储日志;显示存储在/tmp/storage.log内核和系统服务与存储设备相关的消息。

Ctrl + b 5

程序日志;显示来自其他系统实用程序的消息，存储在/tmp/program.log 。

保存截图

您可以在图形安装过程中随时按Shift + Print Screen来捕获当前屏幕。这些屏幕截图保存到/tmp/anaconda-screenshots 。

恢复中断的下载尝试

您可以使用curl命令恢复中断的下载。

条件

您已导航到红帽客户门户的产品下载部分， 网址为https://access.redhat.com/downloads ，并选择了所需的变体，版本和体系结构。您已右键单击所需的ISO文件，并选择了“ 复制链接位置”以将ISO映像文件的URL复制到剪贴板。

程序

  1. 从新链接下载ISO映像。添加--continue-at -选项以自动恢复下载：

    $ curl --output directory-path/filename.iso 'new_copied_link_location' --continue-at -

  2. 使用校验和实用程序（如sha256sum）在下载完成后验证映像文件的完整性：

    $ sha256sum rhel-8.0-x86_64-dvd.iso
			`85a...46c rhel-8.0-x86_64-dvd.iso`

    将输出与Red Hat Enterprise Linux 产品下载网页上提供的参考校验和进行比较。

例C.1。恢复中断的下载尝试

以下是部分下载的ISO映像的curl命令示例：

$ curl --output _rhel-8.0-x86_64-dvd.iso 'https://access.cdn.redhat.com//content/origin/files/sha256/85/85a...46c/rhel-8.0-x86_64-dvd.iso?_auth=141...963' --continue-at -
