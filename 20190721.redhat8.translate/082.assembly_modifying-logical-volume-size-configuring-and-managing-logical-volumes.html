Chapter 9. 修改逻辑卷的大小

创建逻辑卷后，可以修改卷的大小。

不断增长的逻辑卷

要增加逻辑卷的大小，请使用lvextend命令。

扩展逻辑卷时，可以指示扩展卷的大小，或扩展卷后想要的大小。

以下命令将逻辑卷/dev/myvg/homevol为12千兆字节。 

lvextend -L12G /dev/myvg/homevol lvextend  - 将逻辑卷“/ dev / myvg / homevol”扩展到12 GB lvextend  - 自动备份卷组“myvg”lvextend  - 逻辑卷“/ dev / myvg / homevol“成功扩展

以下命令将另一个千兆字节添加到逻辑卷/dev/myvg/homevol 。 

lvextend -L+1G /dev/myvg/homevol lvextend  - 将逻辑卷“/ dev / myvg / homevol”扩展到13 GB lvextend  - 自动备份卷组“myvg”lvextend  - 逻辑卷“/ dev / myvg / homevol“成功扩展

与lvcreate命令一样，您可以使用lvextend命令的-l参数指定增加逻辑卷大小的扩展区数。您还可以使用此参数指定卷组的百分比，或卷组中剩余可用空间的百分比。以下命令扩展名为testlv的逻辑卷，以填充卷组myvg中的所有未分配空间。 

lvextend -l +100%FREE /dev/myvg/testlv将逻辑卷lvextend -l +100%FREE /dev/myvg/testlv扩展到68.59 GB逻辑卷testlv成功调整大小

扩展逻辑卷后，必须增加文件系统大小以匹配。

默认情况下，大多数文件系统大小调整工具会将文件系统的大小增加到基础逻辑卷的大小，因此您无需担心为这两个命令中的每个命令指定相同的大小。

在逻辑卷上增加文件系统

要在逻辑卷上扩展文件系统，请执行以下步骤：

  1. 确定现有卷组中是否有足够的未分配空间来扩展逻辑卷。如果没有，请执行以下过程：

    1. 使用pvcreate命令创建新的物理卷。
    2. 使用vgextend命令将包含逻辑卷的卷组扩展到您正在增长的文件系统，以包括新的物理卷。
  2. 一旦卷组足够大以包含较大的文件系统，请使用lvresize命令扩展逻辑卷。
  3. 调整逻辑卷上的文件系统的大小。

请注意，您可以使用lvresize命令的-r选项来扩展逻辑卷并使用单个命令调整基础文件系统的大小

缩小逻辑卷

您可以使用lvreduce命令减小逻辑卷的大小。

注意

GFS2或XFS文件系统不支持缩小，因此您无法减小包含GFS2或XFS文件系统的逻辑卷的大小。

如果要还原的逻辑卷包含文件系统，为防止数据丢失，必须确保文件系统未使用正在减少的逻辑卷中的空间。因此，建议在逻辑卷包含文件系统时使用lvreduce命令的--resizefs选项。使用此选项时， lvreduce命令会在缩小逻辑卷之前尝试减少文件系统。如果文件系统收缩失败，如果文件系统已满或文件系统不支持收缩，则lvreduce命令将失败并且不会尝试收缩逻辑卷。

警告

在大多数情况下， lvreduce命令会警告可能的数据丢失并要求确认。但是，您不应该依赖这些确认提示来防止数据丢失，因为在某些情况下您将看不到这些提示，例如逻辑卷处于非活动状态或未使用--resizefs选项时。

请注意，使用lvreduce命令的--test选项并不指示操作的安全位置，因为此选项不检查文件系统或测试文件系统调整大小。

以下命令将卷组vg00的逻辑卷lvol1缩小为64 MB。在此示例中， lvol1包含一个文件系统，该命令与逻辑卷一起调整大小。此示例显示命令的输出。 

lvreduce --resizefs -L 64M vg00/lvol1来自util-linux的fsck 2.23.2 / dev / mapper / vg00-lvol1：clean，11/25688文件，8896/102400 block resize2fs 1。42。9（2014年12月28日）将/ dev / mapper / vg00-lvol1上的文件系统大小调整为65536（1k）块。/ dev / mapper / vg00-lvol1上的文件系统现在为65536个块。逻辑卷vg00 / lvol1的大小从100.00 MiB（25个范围）更改为64.00 MiB（16个范围）。逻辑卷vg00 / lvol1已成功调整大小。

在调整大小值之前指定 - 符号表示将从逻辑卷的实际大小中减去该值。以下示例显示了您将使用的命令，而不是将逻辑卷缩小到64 MB的绝对大小，而是希望将卷缩小64 MB。 

lvreduce --resizefs -L -64M vg00/lvol1

扩展条带化逻辑卷

为了增加条带化逻辑卷的大小，组成卷组的基础物理卷上必须有足够的可用空间来支持条带。例如，如果您的双向条带占用整个卷组，则向卷组添加单个物理卷将无法扩展条带。相反，您必须向卷组添加至少两个物理卷。

例如，考虑一个由两个底层物理卷组成的卷组vg ，如以下vgs命令所示。 

vgs #PV #LV #SN Attr VSize VFree vg 2 0 0 wz  -  n- 271.31G 271.31G

您可以使用卷组中的整个空间量创建条带。 

lvcreate -n stripe1 -L 271.31G -i 2 vg使用默认条带化64.00 KB向上舍入大小到完整物理范围271.31 GB逻辑卷“stripe1”创建#lvs lvs -a -o +devices LV VG Attr LSize Origin Snap％移动日志复制％Devices stripe1 vg -wi-a- 271.31G / dev / sda1（0），/ dev / sdb1（0）

请注意，卷组现在没有更多可用空间。 

vgs VG #PV #LV #SN Attr VSize VFree vg 2 1 0 wz  -  n- 271.31G 0

以下命令将另一个物理卷添加到卷组，然后该卷具有135千兆字节的额外空间。 

vgextend vg /dev/sdc1卷组“vg”成功扩展vgs VG #PV #LV #SN Attr VSize VFree vg 3 1 0 wz  -  n- 406.97G 135.66G

此时，您无法将条带化逻辑卷扩展到卷组的完整大小，因为需要两个底层设备才能对数据进行条带化。

vgextend vg /dev/sdd1卷组“vg”成功扩展vgs VG #PV #LV #SN Attr VSize VFree vg 4 1 0 wz  -  n- 542.62G 271.31G＃ lvextend vg/stripe1 -L 542G使用stripeize最后一个段64.00 KB将逻辑卷stripe1扩展到542.00 GB逻辑卷stripe1已成功调整大小
