从文本编辑器迁移

本指南适用于使用文本编辑器（如Vim，Emacs或Sublime Text）的Python开发人员，需要有关切换到PyCharm的信息。它包括对IDE概念的简要讨论，然后探讨具体的差异。

什么是IDE？

Python开发人员长期以来使用各种工具来编写代码。尽管线条模糊，但这些工具分为两大类：文本编辑器和集成开发环境（IDE）。文本编辑器过于简单化，专注于编辑单个文件，提供非常精简和平均的体验。

顾名思义，IDE具有更广泛的视野。他们希望查看整个项目以及所有与编码相关的活动，并将这些活动统一到一致，强大的UI中。虽然这两个部分重叠，但整体范围是主要差异。

因为像PyCharm这样的IDE会查看所有代码 - 在项目中，在依赖项中以及在平台本身中 - 它可以在编码活动中提供很多帮助。这种分析和帮助显示在许多强大的设施中：

  • 通过完成输入， 自动完成可加快准确编码速度

  • 快速修复发现常见错误并提供特定于语言的更正

  • 代码意图建议对常见的Python模式进行优化和改进

  • 代码重构将经常使用的Python重构中的苦差事带走了

  • 模板自动执行重复任务

  • 代码导航分析所有代码的结构和语义，以及它使用的代码，以提供在软件中移动的丰富方法

虽然文本编辑器提供了一些这样的功能，但PyCharm不仅仅是面向字符串的操作，还能解决代码和语言的语义问题，提供智能帮助。PyCharm理解Python并理解您的代码，因此它可以跳转到定义符号的位置，或者使用类的位置，或者在代码库中重构方法及其用法。实际上，这些功能超越了Python：来自IntelliJ系列中的兄弟，PyCharm继承了Web开发和许多其他领域的一流IDE支持。

除了这些编码辅助工具之外，PyCharm还将相关操作与相同的环境联系起来：版本控制，数据库管理，性能分析，测试运行，远程环境等。这些设施中的每一个都以集成方式提供：应用更改，同时查看差异可以自动触发重新运行测试。

最后，这种力量伴随着以产品为中心的思维方式：一切正常。虽然其他工具有很多可能的功能，但您可以自行组装和维护聚合。这对那些喜欢修补的人来说很好。使用PyCharm作为您的IDE，这些功能可以开箱即用，并且年复一年地得到专业支持。

PyCharm接口

由于诸如PyCharm之类的IDE具有如此广泛的范围，因此用户界面（UI）需要比简单文本编辑器更多的方向。例如，这是PyCharm中的典型项目：

  1. 左侧的项目视图显示项目文件。

  2. 右侧的编辑器 ，您实际编写代码的地方。它有标签，可以在打开的文件之间轻松导航。

  3. 编辑器上方的导航栏还允许您快速运行和调试应用程序以及执行基本的VCS操作 。

  4. 左侧排水沟 ，编辑器旁边的垂直条纹，显示您拥有的断点，并提供了一种方便的方式来浏览代码层次结构，如转到定义/声明。它还显示行号和每行VCS历史记录。

  5. 正确的排水沟 ，位于编辑器的右侧。PyCharm不断监控代码的质量，并始终在正确的装订线中显示代码检查的结果：错误，警告等。右上角的指示器显示整个文件的代码检查的总体状态。

  6. 工具窗口是连接到工作区底部和侧面的专用窗口，可以访问典型任务，例如项目管理，源代码搜索和导航，与版本控制系统的集成等。

  7. 状态栏指示项目和整个IDE的状态，并显示各种警告和信息消息，如文件编码，行分隔符，检查配置文件等。

还是太多了？如果您正在从文本编辑器转换并想要简洁的外观，PyCharm有几个解决方案：

  • 在需要之前，可以隐藏Windows和工具栏

  • 无干扰模式和其他模式消除了大部分视觉效果，专注于通过键盘快捷键和动作搜索完成交互的代码

实际上，PyCharm通过以键盘为中心，以搜索为导向的交互来展现其大部分业务，从而接受了现代，整洁的趋势。不想单击菜单重新格式化代码？不仅有按键序列，而且您可以按Ctrl + Shift + A搜索“重新格式化代码”的操作：

项目

如上所述，PyCharm超越了文件，而是将项目视为完整的软件解决方案。不仅是项目附带的软件，还有相关的任务。

该项目不仅包含这些工件。在项目中，您可能按特定顺序在选项卡中打开3个文件。该信息保存在项目的元数据中，位于项目根文件夹的.idea子目录中。因此，下次打开项目时，您将恢复到该配置。这适用于配置：PyCharm 保存和还原在编辑时更改 ，由本地历史记录支持，可以将代码返回到先前的状态。PyCharm在IDE中有很多，它是项目配置的一部分：Python设置，运行代码的场景，数据库的连接信息等等。

你的第一个项目

如果您当前使用文本编辑器，那么您进入PyCharm的第一步将询问您有关创建项目的信息。您有很多选择和机会。我们来讨论一下。

首先，您可能从空白平板开始，或从磁盘上的现有目录开始，或者从尚未检出的远程存储库开始。PyCharm 有特定的设施来帮助解决所有这些问题。例如，在选择项目的起点时，能够浏览您的GitHub帐户的存储库真的很不错。

接下来，您可能有一个或多个要与项目一起使用的Python解释器。PyCharm支持本地安装的Python 2和3，以及制作或重用虚拟环境。它还支持基于Anaconda的口译员，远程口译员，Vagrant和Docker。您可以在启动PyCharm的“新建项目”向导之前设置它们，或者在项目创建期间指定（甚至创建）这些。同样，您可以在创建项目后改变主意。

使用PyCharm专业版，我们可以轻松开始使用流行的项目类型。例如，创建Django项目将生成遵循Django最佳实践的目录结构，并添加一个“运行配置”，在特殊工具窗口中启动Django服务器。

IdeaVim

长期的vim用户在如何使用他们的编辑器方面投入了大量资金。幸运的是JetBrains有一个非常强大且支持良好的IntelliJ插件，名为IdeaVim ，它是一个“基于IntelliJ平台的IDE的Vim仿真插件”。

使用此插件，您可以在PyCharm中输入Vim仿真器模式。更多信息可在上面链接的GitHub页面，以及Twitter帐户和PyCharm IdeaVim 截屏视频中找到 。虽然有点过时，但这是一个史诗评论 ，由前vim用户深入研究。

研究在PyCharm（IdeaVim）中使用Vim编辑器仿真来学习如何在PyCharm中使用IdeaVim插件。

也可以将vim配置为外部工具 ，如PyCharm中所述：在Vim，Emacs或Sublime Text中打开当前文件 。

Emacs和emacsIDEAs

Emacs和vim是两个流行的文本编辑器，适用于核心开发人员。PyCharm提供了一个匹配许多Emacs键序列的键映射 。

虽然PyCharm没有Emacs的插件与IdeaVim的全部广度相匹配，但社区有一个名为emacsIDEAs的活动插件，它提供了一些来自Emacs和Emacs扩展（如AceJump）的有用功能。

您可以将Emacs配置为PyCharm的外部工具 ，并将其用于编辑文件。本教程中描述了此过程。

定制和扩展

PyCharm的IntelliJ基础非常广泛（许多IDE在其上构建）并且非常成熟，这也带来了另一个好处：PyCharm是非常可定制的。例如，虽然默认情况下它附带一个浅色主题，但它还提供了一个可选的，更暗的主题，叫做“Darcula”，它与编辑器中的现代趋势相匹配。除了调整`heme之外，还可以自定义多种外观选项：字体大小，颜色方案，工具栏设置，气球行为等。

“它看起来如何”只是定制选项的开始。我们讨论了切换到不同的捆绑键映射 。但是，您可以通过多种方式扩展和自定义键映射，将不同的键击映射到不同的操作。除此之外，PyCharm的行为可以扩展：安装插件，操作的自定义“模板”，更改代码意图等等。

多个游标

这是一个功能特性，其他工具的开发人员发誓：多个游标。Sublime Text率先推出了这一概念，IntelliJ为其平台上的所有IDE添加了这一概念。使用PyCharm的多重插槽，可以使用多个插入符号，IDE操作适用于每个插入符号。这在PyCharm Multiple Selection截屏视频中有详细显示。

标签和拆分窗口

成熟的文本编辑器（如Emacs和vim）以其丰富的方式而闻名，他们可以将屏幕分成多个区域，允许一次显示多个文件。这些工具不仅仅是基础知识：您可以以丰富的方式在这些区域之间移动，垂直或水平分割，重新分割，并通过键盘完成所有这些工作。

PyCharm还具有一组成熟的功能来显示多个文件。最基本的是，您可以将多个文件打开到选项卡中，可以手动重新组织或按字母顺序排序。标签可以固定或分离到单独的窗口中。

除了选项卡之外，您还可以将编辑器窗口拆分为独立窗格，以便一次显示多个文件/选项卡。分割可以垂直或水平完成。您可以通过将选项卡移动到相反的组来重新排列当前打开的文件。

最后，所有这些都可以以键盘为中心的方式完成，这样就可以轻松使用多个文件和窗口，而无需使用鼠标。

运行代码

虽然文本编辑器提供了处理Python代码运行的工具，但PyCharm更进一步使用了一种称为运行配置的工具 。这些允许您定义与运行Python（或其他语言，如JavaScript）代码相关联的参数。然后，当您运行该运行配置时，将应用所有关联的设置。

此外，PyCharm在专用工具窗口中运行您的代码，该窗口提供了许多用于处理运行代码的功能：停止，重新运行等。输出显示在迷你控制台中，错误显示带有可以单击以跳转到该文件的行号的行的回溯。此工具窗口以及运行和重新运行代码都可以通过键盘驱动。

这些运行配置也适用于特殊类型的运行：测试运行器配置，带有用于显示测试输出的专用窗口，使用可视化调试器的调试配置，测试覆盖率运行配置，分析器运行配置，甚至是并发图配置。所有这些都为专业Python开发人员的工作流程提供了真正的价值。

最后，PyCharm Professional为它支持的框架添加了新类型的运行配置。例如，Pyramid运行配置了解Pyramid配置文件和Pyramid独有的功能。

生产力提示

从命令行运行

虽然PyCharm传统上像其他应用程序一样启动（从桌面），但您也可以从命令行使用PyCharm这些可以在初始安装期间进行设置。例如，在Linux和macOS上，/ usr / local / bin / charm可用于在PyCharm中打开文件。您还可以运行检查并使用PyCharm的diff工具。

打开一个文件

您不必创建项目来打开某个特定文件。PyCharm的Open菜单（或命令行“charm”程序）可以在现有项目的窗口或新窗口中打开单个文件。

划痕文件

如果你需要的代码或内容片段临时编辑，并且不希望将其保存在一个项目甚至一个文件，PyCharm提供了一个名为设施划痕 。您可以创建临时文件（与语言关联，因此突出显示语法，代码完成，可运行等）或临时缓冲区（小块文本）。

这些划痕具有许多功能：您可以列出它们（最多五个），关闭并删除它们，将它们与语言关联，然后重新组织它们。划痕保存在项目的特殊区域中。

内置提示

当您第一次安装PyCharm时，每次启动时它都会为您提供友好的提示。它还可以让您关闭这些提示。如果您以后想要查看更多提示 ，请选择帮助|主菜单上的每日提示 。

安装插件

如前所述，PyCharm和IntelliJ拥有庞大的IDE插件生态系统。PyCharm本身有近500个插件，涵盖了许多不同的类别。

实际上，PyCharm IDE本身是由插件组成的。这就是为什么，例如，PyCharm从WebStorm继承了许多Web功能：它们通过插件共享相同的核心功能。

通过PyCharm的设置/首选项对话框，可以浏览，安装，更新和删除插件 。此外，还会与PyCharm本身一起检查插件的更新。如果更新了插件，系统会要求您下载并重新启动PyCharm。

多语言发展

虽然文本编辑器支持许多语言和Python，但PyCharm在Python之外的功能是独一无二的。例如，PyCharm是一流的Web开发IDE。它不仅仅是语法高亮和自动完成HTML，CSS和JavaScript。它提供了全方位的功能：语义感知代码检查，重构，调试，启动实时服务器，将SASS文件转换为CSS，浏览npm运行脚本等。

这也适用于Web开发：IPython Notebook仿真，reStucturedText等。PyCharm如何做到这一点？如上所述，它与其他基于IntelliJ的IDE兄弟共享JetBrains插件。

小地图

其他工具，如Sublime Text和Visual Studio（通过附加组件），提供了滚动大文档的可视模式。这样，普通滚动条将替换为文档的图形缩略图和当前滚动位置。

PyCharm用户也可以使用CodeGlance JetBrains插件获取此插件，该插件是为了符合所选主题而编写的。
