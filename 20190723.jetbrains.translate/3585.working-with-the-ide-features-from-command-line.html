WebStorm 2019.1帮助

使用命令行中的WebStorm功能

使用WebStorm，您可以从命令行打开，格式化，检查，比较和合并文件。

在Linux上，将WebStorm安装为快照包会自动创建名为webstorm的命令行启动webstorm 。工具|因此，无法创建命令行启动器命令。

启用命令行启动器

对于Windows

  • 将WebStorm bin文件夹的Path添加到Path环境变量（例如， C:\Program Files\JetBrains\WebStorm\bin ）。

对于macOS和Linux

在macOS和Linux上，创建命令行启动程序以将WebStorm与shell集成。确保创建的启动程序脚本位于shell的搜索路径中，以便可以在系统范围内调用它。

  1. 从主菜单中，选择工具|创建命令行启动器 。

  2. 在打开的“ 创建启动器脚本”对话框中，键入启动器的名称及其路径，或接受建议的默认值webstorm 。

  3. 在WebStorm外部，将路径和启动器脚本的名称添加到path 。

对于macOS和Linux，可以在创建期间自定义默认名称webstorm ，请参阅为macOS和Linux启用命令行启动程序 。

启动WebStorm

  • 在命令提示符处，键入<WebStorm>命令，然后按Enter键 。 <WebStorm>是特定于平台的启动器。

webstorm.exe
webstorm

打开文件，文件夹和项目

  • 在命令提示符下，键入：

    <WebStorm>路径

  • 对于文件，可以选择使用以下格式之一指定光标所在的行：

    <WebStorm> [ - line <number> ]路径<WebStorm>路径：<number>

  • 如果文件或文件夹属于现有项目，则将打开此项目。

  • 如果该文件不属于任何项目，则将自动创建该文件的临时项目。

  • 如果该文件夹不属于任何项目，则将在此文件夹中创建一个新项目。

例子

句法
webstorm.exe --line <number>路径
替代语法
webstorm.exe路径：<number>
例子
webstorm.exe --line 3 C： \ M yProject \ s cripts \ n umbers.js webstorm.exe C： \ M yProject \ s cripts \ n umbers.js：3
句法
webstorm --line <number>路径
替代语法
webstorm路径：<number>
例
webstorm --line 3 ~ / MyProject / scripts / numbers.js webstorm~ / MyProject / scripts / numbers.js：3

运行代码检查

有关详细信息，请参阅代码检查

除了从主菜单或项目工具窗口的上下文菜单中运行代码检查之外，您还可以从命令行启动检查器，而无需实际运行WebStorm。

这样，您可以作为开发过程的一部分执行常规代码检查，这对于大型项目尤为重要。检查结果以XML格式存储，您可以随时下载和查看这些结果。

从命令行启动代码检查

  1. 退出WebStorm否则inspect失败。

  2. 在命令提示符下，键入：

    <WebStorm>检查项目路径检查路径输出路径[ -vX ] [ -d子目录路径]
    其中<WebStorm>是特定于平台的启动器。

查看脱机检查的结果

  1. 打开运行检查的项目。

  2. 在主菜单上，选择Code |查看离线检查结果 。

  3. 在打开的“ 选择路径”对话框中，单击存储带有检查结果的XML文件的目录。

  4. 单击确定 。检验结果显示在检验结果工具窗口的“ 脱机视图”选项卡中。

命令行选项

选项

描述

项目路径

包含要检查的项目的目录的完整路径。

检查路径

检查项目的检查配置文件的完整路径。默认情况下，检查配置文件存储在以下目录中：

  • Windows： ％HOMEPATH％\。WebStorm2019.1 \ CONFIG \检查

  • macOS： 〜/ Library / Preferences / WebStorm2019.1 /检查

  • Linux： 〜/。WebStorm2019.1 /配置/检查

输出路径

将存储包含检查报告的XML文件的现有目录的完整路径。

-vx

（可选）输出的详细级别，其中X为0表示安静， 1表示噪声， 2表示额外噪声。

-d子目录路径

（可选）要检查的子目录的完整路径。

比较文件

另请参阅比较文件，文件夹和文本源 。

  • 在命令提示符下，键入：

    <WebStorm> diff path1 path2
    其中<WebStorm>是特定于平台的启动程序， path1和path2是要比较的文件的完整路径。WebStorm在差异查看器中打开文件 。

例子

句法
webstorm.exe diff path1 path2
例
webstorm.exe diff C： \ M yProject \ R eadme.md C： \ M yProject \ R eadme.md.bak
句法
webstorm diff path1 path2
例
webstorm diff~ / MyProject / Readme.md~ / MyProject / Readme.md.bak

合并文件

如果跳过path3 ，WebStorm会将output视为基本修订。

  1. 在命令提示符下，键入：

    <WebStorm>合并path1 path2 path3输出
    其中<WebStorm>是特定于平台的启动程序， path1和path2是要合并的文件的完整路径， path3是可选的，指向文件发散的基本修订版本， output是文件的结果路径合并将被存储。

  2. 在打开的“ 合并”对话框中，浏览更改并按照解决冲突中的描述逐个接受或丢弃每个更改。

例子

句法
webstorm.exe合并path1 path2 path3输出
例
webstorm.exe合并C： \ M yProject \ R eadme.md.bak C： \ M yProject \ R eadme.md C： \ A rchive \ R eadme.md C： \ N ewProject \ R eadme.md
句法
webstorm merge path1 path2 path3输出
例
webstorm merge~ / MyProject / Readme.md.bak~ / MyProject / Readme.md~ / Archive / Readme.md~ / NewProject / Readme.md
