配置Python解释器

在PyCharm中，您不仅可以使用任何单个Python解释器。您可以实现多个，并且这样做可以选择您希望用于任何特定项目的解释器。

PyCharm支持：

  • 标准Python解释器 （请参阅支持的版本 ）

  • 其他Python实现（ IronPython ， PyPy ， Jython ， CPython ）

  • Virtualenv环境： Virtualenv ， Pipenv和Conda 。

  • 远程Python解释器（ SSH ， Vagrant ， WSL（仅适用于Windows） ）。

    仅在PyCharm Professional中受支持。

  • 基于Docker的解释器（ Docker ， Docker Compose ）。

    仅在PyCharm Professional中受支持。

可以在以下级别配置Python解释器：

  • 当前项目 ：选定的Python解释器将用于当前项目。

    在此级别上配置Python解释器如下所述。您可以创建新的解释器或使用现有的解释器之一 。

  • 新项目 ：选定的Python解释器将用于新项目而不是默认项目。

    请参阅创建纯Python项目一节。

创建一个新的项目解释器

要向当前项目添加新解释器：

  1. 通过以下任一方式打开“ 添加Python解释器”对话框：

    • 当您在编辑器中时 ，最方便的方法是使用状态栏中的Python解释器小部件。单击小部件并选择Add Interpreter ...

    • 如果您位于“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，请选择“ 项目<项目名称>”项目口译员 。点击图标，然后选择添加 。

  2. 选择要添加的解释器类型并执行特定设置：

    在“ 添加Python解释器”对话框的左侧窗格中，选择“ Virtualenv环境” 。以下操作取决于之前是否存在虚拟环境。

    如果选择了新环境 ：

    1. 在文本字段中指定新虚拟环境的位置，或单击并在文件系统中查找位置。请注意，新虚拟环境所在的目录必须为空！

    2. 从列表中选择基本解释器，或单击并在您的文件系统中找到基本解释器。

    3. 如果要继承全局站点包目录，请选中“ 继承全局站点包”复选框。此复选框对应于virtualenv工具的--system-site-packages选项。

    4. 如果需要， 请选中“ 使所有项目可用”复选框。

    如果选择了现有环境 ：

    1. 指定所需的解释器：使用列表，或单击并在您的文件系统中找到一个。

    2. 如果需要，选中“ 使所有项目可用 ”复选框。

    单击“ 确定”完成任务。

    有关详细信息，请参阅配置虚拟环境 。

    如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径：

    单击“ 确定”完成任务。

    如果您尚未修改PATH变量，PyCharm会显示错误消息： 找不到Pipenv可执行文件 。发现pipenv安装过程中描述的正确可执行路径，并在Pipenv可执行文件字段中输入目标字符串，例如： C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts \ pipenv.exe （Windows）或/Users/jetbrains/.local/bin/pipenv（macOS ）。

    单击“ 确定”保存更改并完成任务。

    有关更多详细信息，请参阅配置Pipenv环境 。

    在“ 添加Python解释器”对话框的左侧窗格中，选择“ Conda环境” 。以下操作取决于之前是否存在Conda环境。

    如果选择了新环境 ：

    1. 在文本字段中指定新Conda环境的位置，或单击并在文件系统中查找位置。请注意，新Conda环境所在的目录必须为空！

    2. 从列表中选择Python版本。

    3. 在文本字段中指定Conda可执行文件的位置，或单击并在Conda安装目录中找到位置。

    4. 如果需要， 请选中“ 使所有项目可用”复选框。

    如果选择了现有环境 ：

    1. 指定所需的解释器：使用列表，或单击并在您的文件系统中找到一个（例如， C：\ Users \ jetbrains \ Anaconda3 \ python.exe ）。

    2. 如果需要，选中“ 使所有项目可用 ”复选框。

    单击“ 确定”完成任务。

    有关更多详细信息，请参阅配置Conda虚拟环境 。

    在“ 添加Python解释器”对话框的左侧窗格中，选择“ 系统解释器” 。在“ 解释器”字段中，键入所需解释器可执行文件的完全限定路径，或单击在打开的Select Python Interpreter对话框中，选择所需的Python可执行文件，然后单击OK 。请注意，您需要管理员权限才能安装，删除和升级系统解释程序包。尝试通过意图操作安装解释程序包时，您可能会收到以下错误消息：根据提示，请考虑为项目使用虚拟环境。单击“ 确定”完成任务。

    有关更多详细信息，请参阅配置系统解释程序

    在“ 添加Python解释器”对话框的左侧窗格中，单击“ SSH解释器” 。在右侧窗格中，选择“ 新建服务器配置” ，然后指定服务器信息（主机，端口和用户名）。或者，您可以选择“ 现有服务器配置”，然后从列表中选择任何可用的部署配置。如果需要，请单击查看所选部署配置的“ 连接”设置 ，“ 映射 ”和“ 排除”路径 。单击“ 下一步”继续配置解释程序。

    在下一个对话框窗口中，提供身份验证详细信息以连接到目标服务器。

    选择密码或密钥对（OpenSSL或PuTTY）并输入您的密码或密码。

    单击“ 下一步”继续执行最终配置步骤。

    在下一个对话框窗口中，验证所需Python的路径是否已解释。您可以接受默认值，也可以指定其他默认值。您必须配置本地项目和服务器之间的路径映射。为此，请单击在“ 同步文件夹”字段旁边，输入本地项目文件夹的路径以及远程服务器上文件夹的路径。

    您还可以选择最低复选框以启用对远程服务器的本地更改的自动上载。

    有关更多详细信息，请参阅使用SSH配置远程解释器 。

    在“ 添加Python解释器”对话框的左侧窗格中，单击“ Vagrant” ：

    单击“浏览”按钮在Vagrant实例文件夹字段旁边，并指定所需的Vagrant实例文件夹。

    这导致显示到Vagrant主机URL的链接。

    Python解释器路径字段显示所需Python可执行文件的路径。您可以接受默认值，也可以指定其他默认值。单击确定 。配置的远程解释器将添加到列表中。

    有关更多详细信息，请参阅使用Vagrant配置远程解释器 。

    在对话框的左侧窗格中，单击WSL 。

    选择Linux发行版并在所选Linux发行版中指定python可执行文件的路径。

    有关更多详细信息，请参阅使用WSL配置远程解释器 。

    在打开的对话框中，选择Docker选项，从下拉列表中选择Docker服务器（如果缺少服务器，单击New ... ），并指定映像名称。

    Python解释器路径应该具有默认值：

    单击“ 确定”完成任务。

    有关更多详细信息，请参阅使用Docker配置远程解释器 。

    在打开的对话框中，选择Docker Compose选项，从下拉列表中选择Docker服务器，Docker Compose服务（此处为web ），配置文件（此处docker-compose.yml ）和图像名称（此处为python ）。

    为什么我们选择web ？这个选择的原因在于，在配置基于Docker-Compose的解释器之后，我们将能够创建定期运行配置，这将改变我们选择的容器的行为。因此，如果我们想调试容器中的代码，那就是我们应该在这里选择的代码。compose文件中的所有其他容器将始终与此一起启动，但是您无法从PyCharm影响它们的行为 - 它们总是表现得就像使用命令docker-compose up命令启动它们一样线。

    接下来，等待PyCharm启动Docker-Compose配置进行扫描和索引：

    单击“ 确定”完成任务。

    有关更多详细信息，请参阅使用Docker Compose配置远程解释器 。

添加远程Python解释器时，首先将PyCharm助手复制到远程主机。PyCharm助手需要远程运行打包任务，调试器，测试和其他PyCharm功能。接下来，生成二进制库的骨架并在本地复制。此外，所有Python库源都是从远程主机上的Python路径收集的，并与生成的骨架一起在本地复制。需要在本地存储骨架和所有Python库源才能使解析和完成正常工作。PyCharm在每次远程运行时检查远程助手版本，因此如果您更新PyCharm版本，新助手将自动上传，您无需重新创建远程解释器。将帮助程序复制到服务器需要SFTP支持。

设置现有项目解释器

您可以随时使用Python Interpreter小部件或项目Settings / Preferences中的项目解释器进行切换。

使用Python Interpreter小部件更改项目解释器

Python解释器小部件位于状态栏上 。这是切换项目解释器最方便，最快捷的方法。只需单击它并选择目标解释器：

仅当在编辑器中打开至少一个项目文件时，才会显示“ 项目解释器”窗口小部件。

在项目设置中更改项目解释程序

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 项目<项目名称> |”项目口译员 。

  2. 展开可用解释器列表，然后单击“ 全部显示”链接。或者，单击图标，然后选择全部显示 。

  3. 选择目标解释器。

    当PyCharm停止支持任何过时的Python版本时，相应的项目解释器被标记为不受支持。

  4. （可选）您可以单击并键入替代解释器名称。

    名称字段中指定的Python解释器名称在可用解释器列表中可见。单击“ 确定”以应用更改。
