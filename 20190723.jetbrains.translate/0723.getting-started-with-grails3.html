Grails入门3

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA支持Grails 3.0及更高版本。

在你开始之前

在使用Grails框架开始创建项目之前，请确保已在计算机上下载最新的Grails SDK版本。您可以从Grails页面下载最新的SDK版本。此外，如果您想使用Grails 3.0版本，请确保使用IntelliJ IDEA终极版，版本15或更高版本。查看最新版本 。

创建Grails 3项目

  1. 打开项目向导 ，在左侧窗格中选择Grails 。

  2. 在页面的右侧，指定以下信息：

    • 项目JDK，您将用于您的项目。

    • Grails SDK Home - 您的本地Grails 3安装，由库代表。

    • 创建create-app或create-plugin - 根据您要创建的内容选择其中一个选项。

    • 选项 - 将此字段用于其他选项。例如，您可以为Grails 3指定--profile=web等配置文件。

    单击下一步 。

  3. 在向导的下一页上，指定项目的信息，然后单击“ 完成” 。

IntelliJ IDEA使用适当的目录和工件创建Grails应用程序。

导入Grails 3项目

由于Grails 3与Gradle构建系统集成，您可以从Gradle模型导入项目。

  1. 从主菜单中，选择File |打开 build.gradle文件或包含build.gradle文件的目录。即使之前未打开或导入项目，IntelliJ IDEA也将导入Gradle项目。

    或者，您可以选择文件|新的|来自现有源的项目，并按照向导步骤操作。

  2. 在打开的页面上，指定必要的选项，然后单击“ 确定” 。

    IntelliJ IDEA导入Grails项目，创建特定的文件夹结构和所有必要的工件。

如果您需要有关Gradle项目导入的更多信息，请参阅Gradle导入部分。

探索Grails应用程序

Grails 3项目结构与Grails 2相比有一些显着的差异：

  • build.gradle - Grails版本3使用Gradle进行构建。导入Grails 3项目时，应通过Gradle模型导入它 。

  • config目录 - config目录的内容使您可以使用YAML文件或Groovy文件进行配置。日志配置也可用。

  • init目录 - 此目录包含主应用程序文件，允许您使用默认设置运行应用程序。

  • Grails 3支持Grails 视图 。

    所有项目视图设置也可用于Grails视图。

  • “ 项目”视图中提供了Grails操作。所有工件图标都更改为Grails图标（ '项目名称'|新建 ）。

有关Grails版本3更改的更多信息，请参阅Grails页面。

运行Grails 3应用程序

创建Grails应用程序后，它就可以运行了。

点击工作区右上角的图标。

如果要从编辑器运行应用程序，请执行以下步骤：

  1. 在“项目”工具窗口中，单击init文件夹。

  2. 从列表中选择Application.groovy以在编辑器中打开该文件。

  3. 在编辑器中，在左侧排水沟中单击图标，然后在打开的窗口中单击运行'Application main（）' 。

您的应用程序在默认浏览器中启动，地址栏中包含以下URL：

HTTP：//本地主机：8080 /

调试Grails 3应用程序

IntelliJ IDEA允许您使用Application.groovy调试Grails 3应用程序 。

  1. 在“项目”工具窗口中，打开init目录，然后右键单击Application.groovy

  2. 从列表中选择Debug Grails：'name'

您还可以使用编辑器启动调试过程。

使用Gradle任务调试Grails 3应用程序

IntelliJ IDEA还允许您使用Gradle任务调试Grails 3应用程序。

  1. 打开Gradle工具窗口 。

  2. 从任务列表中，单击“ 应用程序” ，然后在打开的列表中右键单击“ 运行” 。

  3. 从打开的列表中，选择Debug Grails：'name' 。

Grails 3编码协助

  • IntelliJ IDEA提供所有相关文件（如域类，视图，服务和控制器）之间的导航。在编辑器中按Ctrl + Alt + Home并选择目标。

  • IntelliJ IDEA提供注入的依赖项之间的导航。

  • IntelliJ IDEA还提供控制器方法和.gsp文件之间的导航。

  • IntelliJ IDEA允许您打开并检查域类的层次结构。在编辑器中按Ctrl + H打开“ 层次结构”工具窗口。

  • IntelliJ IDEA还允许您检查域类依赖性。
