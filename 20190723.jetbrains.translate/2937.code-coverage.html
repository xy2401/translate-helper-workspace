代码覆盖范围

使用WebStorm，您还可以监控测试覆盖的代码量。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记已覆盖和未覆盖的行。

  • “ 覆盖率”工具窗口工具窗口显示每个文件夹的覆盖类和行的百分比以及每个文件中覆盖行的百分比。

  • 在编辑器中打开文件时，模拟期间执行的行标记为绿色，未涉及的行标记为红色。

覆盖测量的结果保存在覆盖套件中 。您还可以将它们与任何现有套件合并。在这种情况下，如果一条线被至少一次测试运行覆盖，则认为该线被覆盖。

测量代码覆盖率可用于Mocha测试和Karma测试 。

运行覆盖测试

  • 创建Mocha或Karma运行配置，从工具栏上的列表中选择它，然后单击在列表旁边。

  • 或者，使用编辑器中的测试图标快速运行特定套件或覆盖测试：

配置代码覆盖行为

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 构建”，“执行”，“部署”下的“ 覆盖范围 ”。Coverage页面打开。

  2. 定义如何处理收集的覆盖率数据：

    • 要在每次启动具有代码覆盖率的新运行配置时显示“ 代码覆盖率”对话框，请在将coverage应用于编辑器之前选择“ 显示选项” 。

    • 要放弃新的代码覆盖率结果，请选择“不应用已收集的覆盖范围” 。

    • 要在每次启动具有代码覆盖率的新运行配置时丢弃活动套件并使用新套件 ，请选择“ 使用新套件替换活动套件” 。

    • 要在每次启动具有代码覆盖率的新运行配置时将新代码覆盖套件附加到活动套件，请选择“ 添加到活动套件” 。

  3. 使用coverage运行应用程序或测试时，定义Coverage工具窗口的行为：

    • 要自动打开Coverage工具窗口，请选中Activate Coverage View复选框。

    • 要手动打开Coverage工具窗口，请清除Activate Coverage View复选框。

或者，只需点击在统计弹出窗口中。

配置颜色以突出显示代码覆盖率结果

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 颜色方案 ” ，然后单击“ 常规” 。“ 颜色方案”页面打开。
