进行测试

通常，PyCharm通过运行您创建的运行/调试配置，以与其他应用程序相同的方式运行和调试测试。这样做时，它会将指定的测试类或方法传递给测试运行器。

在许多情况下，您可以从上下文菜单启动测试会话。为此，在某些上下文菜单中提供了“运行”和“ 调试”命令。例如，这些命令可用于项目工具窗口中的测试类，目录或包。它们也可用于您当前在编辑器中处理的测试类或方法。

如果运行没有永久运行/调试配置的测试，则会创建临时配置。然后，如果要在以后重复使用，可以使用“ 运行/调试”配置对话框保存此类配置。

测试在后台运行，因此您可以同时执行多个测试。

每个运行配置在“ 运行”工具窗口 （“ 测试结果”选项卡 ）中都有自己的选项卡 。一个选项卡可以聚合多个测试。

另请注意，上下文菜单中显示的命令是上下文相关的，即显示的测试命令取决于测试运行程序以及调用此命令的位置。

运行或调试测试

要开始运行或调试测试，可以使用主工具栏或项目工具窗口或编辑器中的上下文菜单：

  • 使用主工具栏：

    1. 从主工具栏上的列表中选择所需的运行/调试配置。

    2. 单击运行 或调试 在列表的右侧。（或者，转到主菜单上的Run | Run （ Shift + F10 ）或Run | Debug （ Shift + F9 ）。）

    要查看可用的运行或调试配置列表并快速选择所需的配置，可以使用以下键盘快捷键： Shift + Alt + F10用于运行配置， Shift + Alt + F9用于调试配置。

  • 使用上下文菜单：

    在“ 项目”工具窗口中右键单击测试文件或测试类，或在编辑器中将其打开，然后右键单击背景。从上下文菜单中，选择Run <class name> / Run <filename>或Debug ....

    对于测试方法，在编辑器中打开该类，然后右键单击方法中的任意位置。上下文菜单建议命令Run / Debug <method name> 。

在目录中运行所有测试

  1. 在“项目”工具窗口中，选择包含要执行的测试的目录。

  2. 从上下文菜单中，选择相应的运行命令。

    如果目录包含属于不同测试框架的测试 ，请选择要使用的配置。

    例如，选择<目录名>中的运行pytest' 。

  3. 探索测试跑步者的结果 。

对于Django 1.1及更高版本，PyCharm支持自定义测试运行器，如果此测试运行器是一个类。

默认情况下，所有测试都是逐个执行的。您可以更改pytest测试框架的此行为并并行执行测试。

并行运行测试

启用测试多处理以优化pytest测试的执行。

要显式指定测试执行的CPU数量：

  1. 按照安装，卸载和升级包中的说明安装 pytest-xdist 软件包 。

  2. 指定pytest作为项目测试框架。有关更多详细信息，请参阅选择测试框架 。

  3. 从主工具栏上的“运行/调试”配置列表中选择“ 编辑配置... ”。在“ 运行/调试配置”对话框中，展开Pytest测试组，然后在<目录名>中选择pytest 。

    当您第一次在目录中运行所有测试时，PyCharm会创建此配置。如果您尚未执行测试，请单击图标并指定运行/调试配置参数 。

  4. 在“ 运行/调试配置”对话框的“ 附加参数”字段中，指定运行测试-n <number of CPUs> ： -n <number of CPUs>并保存更改。

  5. 现在再次运行目录中的所有测试，并在“ 运行”工具窗口中检查输出。在所示示例中，总执行时间为12秒79毫秒，而相应的运行相同测试时为30秒。测试报告提供有关用于运行测试和执行时间的CPU的信息。

或者，您可以在pytest.ini文件中指定运行测试的CPU数。例如，

[ pytest ] addopts = - n3

如果在pytest.ini文件和运行/调试配置中指定了不同的CPU编号值，则后者优先于pytest.ini文件中的设置。
