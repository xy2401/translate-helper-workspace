IntelliJ IDEA 2019.1帮助

镖

使用IntelliJ IDEA，您可以开发，运行和调试Dart Web和命令行应用程序。IntelliJ IDEA可帮助您提供代码完成，错误和语法突出显示，代码检查和快速修复，搜索和导航，重构等等。IntelliJ IDEA还与pub工具和Dart Analysis Server集成。

在你开始之前

  • 如管理插件中所述，在“ 插件”页面上安装并启用Dart存储库插件。

下载Dart工具

从Dart官方网站了解有关Dart开发工具的更多信息。

  • 下载并安装Dart SDK 。

    Dart SDK包含用于开发命令行和Web Dart应用程序的所有工具。它结合了Dart虚拟机 ， Dart库以及所有命令行工具，包括编译器（ dart2js和dartdevc ）和pub工具。

创建一个新的Dart应用程序

如果您还没有应用程序，则可以从Stagehand模板生成具有Dart特定结构的IntelliJ IDEA项目。或者，创建一个空的IntelliJ IDEA项目并在其中配置Dart支持，如下面的“ 使用现有Dart应用程序启动”中所述。

Dart SDK是为每个项目单独配置的。这意味着您可以在IntelliJ IDEA中使用多个Dart SDK，并在它们之间从一个项目切换到另一个项目。

从Stagehand模板创建Dart项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ Dart ”。

  3. 在右侧窗格中，指定Dart SDK的路径。IntelliJ IDEA检测并显示Dart版本 。

    要在项目中创建示例应用程序，请选中Generate sample content复选框，然后从下面的列表中选择相关的Stagehand模板。如果清除该复选框，IntelliJ IDEA将创建一个空项目。

    单击下一步 。

  4. 在向导的第二页上，指定项目名称以及将存储项目相关文件的文件夹的路径。单击“ 完成”后 ，IntelliJ IDEA将设置项目结构并根据所选的Stagehand模板生成一些源。

创建一个空的IntelliJ IDEA项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，再次选择Static Web ，然后单击Next 。

  4. 在向导的第二页上，指定项目文件夹和名称，然后单击“ 完成” 。

从现有的Dart应用程序开始

如果要继续开发现有的Dart应用程序，请在IntelliJ IDEA中打开它，在其中配置Dart，并按照下面的管理Dart依赖关系中所述下载所需的依赖项。

打开计算机上已有的应用程序源

  • 单击欢迎屏幕上的打开或选择文件|从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中检出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”或选择“ VCS”从主菜单中查看版本控制 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

在现有项目中配置Dart支持

  1. 在“设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 语言和框架”下的“ Dart ”。Dart页面打开。

  2. 选中项目<项目名称>复选框的启用Dart支持 。

  3. 在Dart SDK Path字段中，指定下载的Dart SDK的位置。手动键入路径或单击并在打开的对话框中选择路径。如果IntelliJ IDEA正确识别Dart SDK，则其版本号将显示在“ 版本只读”字段中。

  4. 在“ 为以下模块启用Dart支持”区域中，选中需要Dart支持的模块名称旁边的复选框。

在一个IntelliJ IDEA项目中使用多个Dart项目（包）

要将Dart项目（包）附加到现有的IntelliJ IDEA项目，您需要将其根文件夹添加为内容根或新模块。

将Dart项目（包）添加到现有的IntelliJ IDEA项目

  • 如“添加内容根”中所述，在“ 模块”页面 （“ 文件”|“项目结构”|“模块” ）上添加内容根 。

  • 将模块添加到项目中：选择File |新的|来自现有来源的模块，然后在打开的对话框中选择相关模块。

管理Dart依赖项

或者在编辑器中打开pubspec.yaml ，或在“ 项目”工具窗口中选择它，然后从选择的上下文菜单中调用pub操作。

您也可以从嵌入式本地终端以命令行模式使用该工具。

IntelliJ IDEA与pub工具集成，允许您直接从编辑器运行其操作。只要打开pubspec.yaml文件，IntelliJ IDEA就会在其编辑器选项卡的顶部显示一个窗格。使用此窗格上的链接可以调用pub get ， pub upgrade和pub cache repair操作。

pub工具将下载的包保存在缓存中，并在pubspec.yaml文件旁边创建一个.packages文件和一个pubspec.lock文件。

IntelliJ IDEA在消息工具窗口中记录pub命令的执行，您也可以通过单击重新运行上次执行的命令在工具栏上。

构建Dart应用程序

或者，从pubspec.yaml的上下文菜单中选择webdev build 。

对于早于Dart SDK 2的版本，将调用操作pub构建 。

  1. 在编辑器中打开pubspec.yaml文件或切换到打开它的选项卡。

  2. 在选项卡顶部的窗格上，单击“ 生成” 。这将调用webdev构建操作。

  3. 在打开的对话框中，指定要从中获取源的输入文件夹以及要存储已编译应用程序的输出文件夹。

运行和调试Dart命令行应用程序

使用IntelliJ IDEA，您可以运行和调试Dart命令行应用程序。IntelliJ IDEA支持两种调试模式：

  • 本地调试 ：在此模式下，您的应用程序从IntelliJ IDEA启动，并在您的计算机上本地运行。要运行或调试它，请使用Dart命令行应用程序配置。

  • 调试远程应用程序 ：在此模式下，应用程序在远程环境中运行，例如，在Docker容器中运行。要调试它，请使用Dart Remote Debug配置。

运行Dart命令行应用程序

  1. 打开Dart文件以启动应用程序或在项目工具窗口中选择它。该文件必须包含main()方法。

  2. 从选择的上下文菜单中，选择Run'<dart_file_name>' 。IntelliJ IDEA生成Dart Command Line App类型的运行/调试配置，并使用它启动您的应用程序。

在本地调试Dart命令行应用程序

  1. 在Dart代码中配置和设置断点 。

  2. 打开Dart文件以启动应用程序或在项目工具窗口中选择它。该文件必须包含main()方法。

  3. 从选择的上下文菜单中，选择Debug'<dart_file_name>' 。IntelliJ IDEA生成Dart命令行应用程序的运行/调试配置，并启动它的调试会话。

  4. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

创建Dart远程调试运行/调试配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择Dart Remote Debug 。运行/调试配置：Dart远程调试打开。

  2. 在“ 主机”字段中，指定运行Dart虚拟机的计算机的地址，默认值为localhost 。

  3. 指定调试器将通过其连接到远程应用程序的端口，默认值为5858 。启动远程VM只读字段时 ， 使用命令行参数中显示指定的端口。请注意，必须使用这些参数准确启动远程应用程序。

  4. 如果IntelliJ IDEA项目包含多个配置为内容根的Dart项目，请从“ 搜索源”列表中选择要调试的Dart项目 。

启动远程调试会话

  1. 使用Dart Remote Debug运行配置中远程Dart VM字段的命令行参数中的VM选项启动远程Dart应用程序，例如， --enable-vm-service:5858 --pause_isolates_on_start 。应用程序启动，由于--pause_isolates_on_start参数立即暂停，并等待调试器连接。

  2. 在Select run / debug configuration列表中选择新创建的Dart Remote Debug配置，然后单击 。

  3. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

运行和调试Dart Web应用程序

或者，在编辑器中打开HTML文件，按Alt + F2 ，然后从列表中选择一个浏览器。

默认Webdev服务器端口为53322 。如果此端口已经忙，IntelliJ IDEA将使用最接近的端口。

确保此URL地址中的端口与“ 调试器”页面上的内置服务器端口和Chrome扩展设置中的端口相同 。

您可以在任何浏览器中运行Dart Web应用程序，而仅在Dartium和Chrome中支持调试。要运行Dart Web应用程序，请在浏览器中打开应用程序的主HTML文件。调试Dart Web应用程序是通过JavaScript Debug类型的运行配置启动的。

IntelliJ IDEA与webdev服务工具集成，可在必要时将Dart代码编译为JavaScript。对于早于Dart SDK 2的版本，将使用pub serve工具。

当您在浏览器中打开Dart Web应用程序时，它通常以内置服务器URL开头，例如http://localhost:63342/project-name/web/index.html 。但是，内置服务器不用于为应用程序提供服务。相反，IntelliJ IDEA会自动在Dart对话框中指定的端口上启动webdev服务器，并且浏览器页面将重定向到webdev服务器URL（如http://localhost:53322/index.html ）。

webdev服务器的工作记录在专用的Dart Webdev工具窗口中。在当前IntelliJ IDEA会话期间首次开始运行或调试Dart Web应用程序时，将打开工具窗口。您可以通过单击停止该工具在工具栏上。当您再次开始运行或调试时，webdev服务器会自动重新启动。

运行Dart Web应用程序

  1. 使用Dart引用打开HTML文件，或在“ 项目工具”窗口中选择它。

  2. 从编辑器或选择的上下文菜单中，选择“ 在浏览器中打开”，然后在列表中选择所需的浏览器。

创建JavaScript调试运行/调试配置

  1. 打开引用Dart的HTML文件或在“ 项目工具”窗口中选择文件。

  2. 从上下文菜单中，选择Create'<HTML_file_name>' 。将打开“ 运行/调试配置：JavaScript调试”对话框 。

  3. 指定用于调试应用程序的浏览器。

    • 使用Dart SDK 2，您只能使用Chrome，并且Dart代码可以通过dart2js或dartdevc工具编译为JavaScript。

    • 使用Dart SDK 1.x，您可以使用Dartium ，它具有内置的Dart虚拟机，因此Dart代码是本机执行的。

      要使用Dartium，请按照配置浏览器中的说明下载并配置它。确保浏览器名称为Dartium或至少包含Dartium 。

  4. URL字段已经以http://localhost:<built-in server port>/<project-name>/<relative path to the HTML file>格式显示应用程序的URL地址。在调试会话期间，浏览器将从此URL重定向到webdev服务器URL。

开始调试

  1. 在Dart代码中配置和设置断点 。

  2. 启动调试会话：从工具栏上的“ 编辑配置”列表中选择创建的运行配置，然后单击 。IntelliJ IDEA在所选浏览器中打开指定的URL。

  3. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

测试Dart应用程序

如果选择了某个文件夹，则测试运行器仅在名称格式为*_test.dart的文件中查找测试。

IntelliJ IDEA支持运行和调试使用dart测试包编写的Dart测试。您可以在任何目标平台上运行测试，仅支持VM测试进行调试。

您可以运行和调试单个测试，测试组以及整个文件和文件夹中的测试。IntelliJ IDEA使用默认设置创建运行/调试配置，并启动测试。您可以稍后保存此配置以供进一步重复使用。

运行或调试单个测试

  • 在编辑器中打开测试文件，右键单击test（）方法的调用，然后从上下文菜单中选择Run'<test_name>'或Debug'<test_name>' 。

运行或调试组

  • 在编辑器中打开测试文件，右键单击group（）方法的调用，然后从上下文菜单中选择Run'<group_name>'或Debug'<group_name>' 。

从文件运行或调试Dart测试

  • 在“ 项目工具”窗口中 ，选择要运行测试的文件，然后从上下文菜单中选择“运行'<file_name>”或“ 调试”<file_name> 。

从文件夹运行或调试Dart测试

  • 在“ 项目工具”窗口中 ，选择要运行测试的文件夹，然后从上下文菜单中选择“运行'<folder_name>”或“ 调试”<folder_name> 。

保存自动生成的默认配置

  • 测试会话结束后，从测试，测试组，测试文件或文件夹的上下文菜单中选择Save <default_test_configuration_name> 。

使用先前保存的运行/调试配置运行或调试测试

  • 从工具栏上的列表中选择所需的Dart测试配置，然后单击要么 。

配置语法突出显示

您可以根据自己的喜好和习惯配置支持Dart的语法突出显示。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案|飞镖 。

  2. 选择颜色方案，接受从默认值继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
