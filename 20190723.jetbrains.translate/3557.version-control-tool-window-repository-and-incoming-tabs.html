存储库和传入选项卡

VCS |显示更改视图 - 存储库/传入
查看|工具窗口|版本控制 - 存储库/传入
ALT + 9

存储库和传入选项卡仅适用于非分布式版本控制系统（即WebStorm支持的除Git和Mercurial之外的所有VCS）。

“ 存储库”选项卡显示在当前项目中VCS根目录下提交到存储库的更改。“ 传入”选项卡显示其他团队成员提交到存储库的更改，但尚未在本地签出。两个选项卡都显示存储在历史记录缓存中的信息。显示的更改列表的数量取决于缓存范围 。

每个选项卡包含以下窗格：

  • “ 更改列表”窗格显示更改列表。

  • “ 更改的文件”窗格显示在所选更改列表中修改和提交的文件列表。

请注意，如果在服务器和本地工作副本中都使用SVN 1.5或更高版本，则“ 存储库”选项卡还具有“ 合并信息”窗格，该窗格用于配置其他两个窗格中的视图，并提供对分支之间集成的控制。

变更列表窗格

该窗格显示已提交并存储在历史记录缓存中的更改列表。单击更改列表时，受选定提交影响的文件将显示在“ 更改的文件”窗格中。

承诺的变更列表通常对应于跟踪系统中的问题。您可以直接在“ 更改列表”窗格中在浏览器中打开此类问题。此功能具有以下先决条件：

  • 错误跟踪系统的模式在“ 问题导航 设置”对话框中指定。

  • 提交消息中提到了相应的问题编号。

配置问题导航后，提交消息中的问题编号将呈现为链接。单击此链接可转到问题跟踪器的相应页面。

存储库选项卡

过滤

使用此下拉列表隐藏您不感兴趣的更改列表，并仅查看满足特定条件的更改列表。

可以使用以下选项：

  • 用户 ：选择此选项以过滤用户的提交。

  • 结构 ：选择此选项可过滤目标模块或文件夹的提交。

  • 客户端 ：选择此选项可以过滤提交它们的计算机的提交。

  • 无 ：选择此选项可关闭过滤并返回默认视图。

两个标签

通过...分组

使用此下拉列表按照特定条件对更改列表进行分组。

可以使用以下选项：

  • 日期 ：选择此选项可按日期对提交进行分组。

  • 用户 ：选择此选项可按用户对提交进行分组。

  • 客户端 ：选择此选项可以通过计算机对提交进行分组。

两个标签

搜索

使用此文本框输入搜索模式，并找到其提交消息与指定字符串匹配的提交。在您键入时，列表会动态缩减以显示包含与指定模式匹配的提交消息的更改列表。

要保存搜索模式，请按Enter键 。

要查看最近搜索模式列表，请单击按钮。

要清除搜索模式列表，请单击按钮。

合并信息窗格

只有在服务器和本地工作副本中都使用SVN 1.5或更高版本时，该窗格才可用。

在此窗格中，指定要与其他要集成的分支的一对分支。该变更列表窗格将显示与指定的分支机构变更列表并提供有关的信息整合状态的每个变更表。

如果涉及多个项目或根，则可以指定多对分支。
