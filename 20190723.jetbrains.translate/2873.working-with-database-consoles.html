使用数据库控制台

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

以下内容仅在安装并启用数据库工具和SQL插件时有效！

基本

数据库控制台允许您为PyCharm中定义的数据库编写和执行SQL语句作为数据源。它们还允许您分析和修改检索到的数据。

支持以下标准化和DBMS供应商特定的SQL方言：DB2，Derby，H2，HSQLDB，MySQL，Oracle，Oracle SQL * Plus，PostgreSQL，SQL Server，SQL92，SQLite和Sybase。

创建数据源时，会自动为数据源创建一个数据库控制台。如有必要，您可以创建其他控制台。

数据库控制台是持久的：它们存储为SQL文件。

然后，可以从任何其他项目访问在其中一个项目中创建的数据库控制台。

创建数据库控制台

要创建控制台，请在“ 数据库”工具窗口中使用以下操作之一（“ 查看”|“工具Windows”|“数据库” ）。

  • 单击数据源，然后选择“ 文件”|新的|控制台 。

  • 右键单击数据源，然后选择“ 新建”|控制台 。

  • 单击数据源，按Alt + Insert ，然后选择控制台 。

  • 单击数据源，按Ctrl + Shift + F10 ，然后选择新建控制台 。

新控制台表示与数据源的新连接。您只能使用一个数据源连接，并且所有控制台都使用相同的一个连接。在启用单连接模式中，阅读有关单连接模式的更多信息。

打开一个控制台

  • 在“ 文件”工具窗口（“ 查看”|“工具Windows”|“文件” ）中，导航到“ 划痕和控制台”数据库控制台| <data_source_name> ，双击控制台文件。

  • 在“ 数据库”工具窗口中，单击“ 跳转到控制台”图标（ ）并选择一个控制台。

  • 单击数据源，按F4 ，然后选择所需的控制台。

修改控制台设置

  • 打开控制台，然后单击“ 设置”图标（ ）。

    在控制台中更改SQL方言

    右键单击输入窗格的编辑区域，选择“ 更改方言”（<current_dialect>） ，然后选择所需的方言。

    默认情况下，SQL控制台方言由附加到控制台的数据源定义。

    关闭控制台

    1. 点击 （ Ctrl + Shift + F4 ）关闭数据库控制台工具窗口。

    2. 点击在编辑器选项卡（ Ctrl + F4 ）上关闭输入窗格。

    管理数据库控制台

    要管理数据库控制台，请使用“ 项目”工具窗口的“ 划痕和控制台”目录。

    要打开控制台列表，请从标题栏左侧的列表中选择“ 划痕” 。

    该目录包含按数据源分组的现有数据库控制台（由SQL文件表示）（显示为文件夹）。默认控制台（由PyCharm自动创建的控制台）与相应的数据源具有相同的名称。

    您可以：

    • 创建新的控制台。选择目标数据源或其中的节点，然后执行以下操作之一：

      • 选择文件|新的|主菜单中的控制台文件 。

      • 选择新建|上下文菜单中的控制台文件

      • 按Alt + Insert并选择控制台文件 。

    • 重命名您的游戏机。创建新控制台时，它具有数据源的名称，最后带有数字，例如MySQL_1 。如果要为控制台提供更具描述性的名称，请选择控制台并执行以下操作之一：

      • 选择Refactor |从主菜单或上下文菜单重命名 。

      • 按Shift + F6 。

      然后，在打开的对话框中指定新名称。

    • 将控制台文件保存在任意目录中。选择控制台，然后选择Refactor |复制 （ F5 ）。在打开的对话框中指定文件名和位置。

    • 分组你的游戏机。这可以通过创建文件夹然后将控制台拖到这些文件夹中来完成。

    • 打开你的游戏机。选择感兴趣的控制台并执行以下操作之一：

      • 选择查看|从主菜单跳转到Source 。

      • 从上下文菜单中选择Jump to Source 。

      • 按F4 。

    • 查看控制台的更改历史记录。选择文件|当地历史|从主菜单或本地历史|中 显示历史 记录从上下文菜单中显示历史记录 。

    • 删除单个控制台和控制台组。使用编辑|从上下文菜单中删除 ， 删除或在键盘上删除 。
