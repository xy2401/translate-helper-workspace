IntelliJ IDEA 2019.1帮助

管理插件

插件扩展了IntelliJ IDEA的核心功能。例如，他们可以：

  • 提供与版本控制系统，应用程序服务器和其他工具的集成

  • 添加对各种语言和框架的支持

  • 通过快捷提示，实时预览，文件监视器等提高您的工作效率

  • 通过编码练习和验证帮助您学习新的编程语言

打开插件设置

  • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

在“ 插件”对话框中，您可以：

  • 使用Marketplace选项卡可以从JetBrains插件存储库或自定义插件存储库中浏览和安装插件。

  • 使用“ 已安装”选项卡浏览已安装的插件，启用，禁用或删除它们。禁用不必要的插件可以提高性能。

  • 使用“ 更新”选项卡浏览需要更新的插件。

大多数插件可以与任何JetBrains产品一起使用。有些仅限于商业产品。还有一些插件需要单独的许可证。

如果插件依赖于其他插件，IntelliJ IDEA将通知您有关依赖项的信息。如果您的项目依赖于某些插件，请将它们添加到所需插件列表中。

如果现有插件未涵盖某些功能，您可以为IntelliJ IDEA创建自己的插件。有关开发插件的信息，请参阅开发自己的插件 。

默认情况下，IntelliJ IDEA包含许多捆绑插件。您可以禁用捆绑的插件，但无法删除它们。您可以从插件存储库或本地存档文件（ZIP或JAR）安装其他插件。

从存储库安装插件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

  2. 在Marketplace中找到该插件，然后单击“ 安装” 。

从磁盘安装插件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

  2. 在“ 插件”对话框中，单击然后单击“ 从磁盘安装插件” 。

  3. 选择插件存档文件，然后单击“ 确定” 。

  4. 单击“ 确定”以应用更改，并在出现提示时重新启动IDE。

您还可以拖动下载的插件存档文件并将其放入IntelliJ IDEA欢迎屏幕。

自定义插件存储库

默认情况下，IntelliJ IDEA配置为使用JetBrains插件存储库中的插件 。这是托管所有社区插件的地方，您可以在那里托管您的插件。但是，如果您开发仅供内部使用的插件，则可以为它们设置自定义插件存储库。

有关设置自定义插件存储库的信息，请参阅IntelliJ Platform SDK文档 。

设置插件存储库后，将其添加到IntelliJ IDEA：

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

  2. 在“ 插件”对话框中，单击然后单击“ 管理插件存储库” 。

  3. 在“ 自定义插件存储库”对话框中，单击“确定” 并指定您的存储库URL。它必须指向updatePlugins.xml文件的位置。该文件可以与自定义插件位于同一服务器上，也可以位于专用插件上。

  4. 在“ 自定义插件存储库”对话框中单击“ 确定 ”以保存插件存储库列表。

  5. 单击“ 插件”对话框中的“ 确定 ”以应用更改。

要浏览自定义插件存储库，请在“ 插件”对话框的“ 市场”选项卡中键入repository:然后键入repository:的URL。例如：

存储库：“http://plugins.example.com:8080/updatePlugins.xml”myPlugin

或者，您可以使用自定义存储库URL替换默认的JetBrains插件存储库。如果您只想从IntelliJ IDEA获得自定义存储库插件，这将非常有用。为此，请编辑平台属性或VM选项文件，如下所述。有关更多信息，请参阅调整IntelliJ IDEA 。

用自定义插件替换默认插件存储库

  1. 在主菜单上，选择“ 帮助”|编辑自定义属性或帮助|编辑自定义VM选项 。

  2. 将idea.plugins.host属性添加到VM选项文件的平台属性文件或-Didea.plugins.host选项。例如：

    idea.plugins.host = “http://plugins.example.com:8080/”
    -Didea.plugins.host = “http://plugins.example.com:8080/”

    要添加多个URL，请使用分号（ ; ）分隔它们。

    确保没有plugins.jetbrains.com网址。

  3. 重启IntelliJ IDEA。

如果使用自定义插件库替换默认插件存储库，则“ 插件”对话框的“ 市场”选项卡上的搜索字段将仅浏览自定义存储库中的插件。

必需的插件

项目可能需要添加对某些技术或框架的支持的插件。您可以将此类插件添加到当前项目所需的插件列表中，以便IntelliJ IDEA验证插件是否已安装并启用。如果您忘记了某个插件，或者您的团队中的某个人在项目上工作时不了解依赖关系，它会通知您。

为您当前的项目添加所需的插件

  1. 确保安装了所需的插件。

  2. 在Settings / Preferences对话框（ Ctrl + Alt + S ）中，选择Build，Execution，Deployment |必需的插件 。

  3. 在Required Plugins对话框中，单击并选择插件。（可选）指定插件的最小和最大版本。

要指定所需的IntelliJ IDEA版本，请将IDE Core添加到所需插件列表中。

添加所需插件后，当您在IntelliJ IDEA中打开项目时，如果插件已禁用，未安装或需要更新，它将通知您。

单击通知消息中的链接以快速启用，安装或更新所需的插件。

开发自己的插件

您可以使用任何版本的IntelliJ IDEA来开发插件。它提供了一个开放的API，一个专用的SDK，模块和运行/调试配置来帮助您。

建议的工作流程是使用Gradle。还支持使用内部IntelliJ IDEA构建系统的旧工作流程。有关更多信息，请参阅IntelliJ Platform SDK开发人员指南 。
