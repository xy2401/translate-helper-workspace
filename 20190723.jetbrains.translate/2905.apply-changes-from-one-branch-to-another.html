将更改从一个分支应用到另一个分支

在Git中，有几种方法可以将更改从一个分支集成到另一个分支： 合并分支 ， Rebase分支 ，或者从一个分支到另一个分支应用单独的提交（樱桃选择） 。

合并分支

假设您已经创建了一个功能分支来处理特定任务，并希望在完成并测试您的功能后将工作结果集成到主代码库中：

将您的分支合并为master是最常用的方法。

当你在你的功能分支中工作时，你的队友继续承诺他们的工作是很常见的：

运行merge ，功能分支中的更改将集成到目标分支的HEAD中：

Git创建一个新的提交（M），称为合并提交，它是通过组合来自功能分支和主服务器的更改从两个分支分叉的点开始的。

合并分支

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 检出 ”以切换到该分支。

  2. 单击WebStorm窗口底部的“ 分支”弹出窗口，选择要合并到目标分支的分支，然后从子菜单中选择“ 合并到当前 ”。

如果您的工作树是干净的（这意味着您没有未提交的更改），并且您的功能分支和目标分支之间不会发生冲突，Git将合并这两个分支，并且合并提交将显示在日志中 ：

如果分支与目标分支之间发生冲突，系统将提示您解决冲突 （请参阅解决冲突 ）。如果合并后仍有未解决的冲突，则“ 合并冲突”节点将显示在“ 本地更改”视图中的相应更改列表中，并带有解决它们的链接。

如果您有本地更改将被合并覆盖，WebStorm将建议执行智能合并 。如果选择此选项，WebStorm将藏匿未提交的修改，进行合并，然后unstash的变化。

Rebase分支机构

将分支重新rebase到另一个分支时，可以在第二个分支中的HEAD提交之上应用第一个分支的提交，而不是将它们合并到目标分支中。

当您执行rebase操作你整合你已经在你的特性分支做的修改master通过对当前头顶将你的提交分公司提交master ：

将当前分支重新基于另一个分支

  1. 在“ 分支”弹出窗口中，选择要将当前分支重新分支到的分支。

  2. 从弹出菜单中选择Rebase Current to Selected 。

在当前分支的顶部重新建立分支

  1. 在“ 分支”弹出窗口中，选择要在当前分支上重新分组的分支。

  2. 从弹出菜单中选择Checkout和Rebase to Current 。

您可以通过从Git Branches弹出窗口顶部分别选择Abort或Continue操作来取消未完成的rebase操作或恢复中断的rebase。

有关如何在rebase期间跳过或压缩提交的详细信息，请参阅通过执行交互式rebase来编辑项目历史记录 。

樱桃挑选单独的提交

有时您只需要将单个提交（甚至是对提交中的单个文件的更改）应用于其他分支，而不是重新绑定或合并整个分支。这可能很有用，例如，如果您正在使用功能分支并希望集成在两个分支发生分支后提交的master中的修补程序。或者您可能希望将修补程序向后移植到先前版本分支。你可以使用Cherry-pick动作来完成。

将整个提交应用于另一个分支

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 检出 ”以切换到该分支。

  2. 打开“ 版本控制”工具窗口（ Alt + 9 ）并切换到“ 日志”选项卡。

  3. 找到包含您要挑选的更改的提交。

    您可以按分支，用户或日期过滤提交。你也可以点击在工具栏上，选择突出显示|非挑选提交按钮使已经应用于当前分支的提交变灰。如果您知道提交哈希，或者正在寻找标记提交，您还可以使用Go to Hash / Branch / Tag操作（在日志视图中按Ctrl + F ，或单击在工具栏上）。

  4. 选择所需的提交。使用“ 提交详细信息”区域中的信息确保这些是您要转移到另一个分支的更改。

  5. 单击Cherry-pick按钮在工具栏上。WebStorm将显示“ 提交更改”对话框，其中包含自动生成的提交消息。如果要在将代码提交到目标分支之前查看更改甚至修改代码，可以在此对话框中提供的差异查看器中执行此操作。

  6. 完成后，单击“ 提交”以选择所选更改。

    请注意，如果单击“ 取消” ，将创建一个单独的更改列表，其中包含您可以在“ 本地更改”选项卡中看到的所选更改。您可以查看这些更改，并在必要时稍后提交。

  7. 将更改推送到目标分支。

应用单独的更改

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 检出 ”以切换到该分支。

  2. 打开“ 版本控制”工具窗口（ Alt + 9 ）并切换到“ 日志”选项卡。

  3. 找到包含要应用的更改的提交。

    您可以按分支，用户或日期过滤提交。你也可以点击在工具栏上，选择突出显示|非挑选提交按钮使已经应用于当前分支的提交变灰。如果您知道提交哈希，或者正在寻找标记提交，您还可以使用Go to Hash / Branch / Tag操作（在日志视图中按Ctrl + F ，或单击在工具栏上）。

  4. 在右侧的“ 提交详细信息”窗格中，选择包含要应用于目标分支的更改的文件，然后从上下文菜单中选择“ 应用所选更改 ”。

  5. 在打开的对话框中，选择现有的更改列表或输入新更改列表的名称，然后单击“ 确定” 。

  6. 提交更改，然后将它们推送到目标分支。
