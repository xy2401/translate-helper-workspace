TSLint

此功能仅在Ultimate版本中受支持。

您还可以使用ESLint检查TypeScript代码，请参阅使用ESLint的Linting TypeScript代码 。

默认情况下，IntelliJ IDEA根据TSLint配置文件中的严重性级别标记检测到的问题。请参阅配置TSLint突出显示以了解如何覆盖这些设置。

您可以使用IntelliJ IDEA内部的TSLint代码验证工具，并检查TypeScript代码以查找最常见的错误，而无需运行应用程序。激活该工具后，它会删除所有打开的TypeScript文件并标记检测到的问题。

要查看问题的描述，请将鼠标悬停在突出显示的代码上。

安装TSLint

在开始之前，请确保您的计算机上有Node.js.

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install tslint typescript --save-dev

    从TSLint官方网站了解更多信息。

在IntelliJ IDEA中激活和配置TSLint

从TSLint官方网站了解更多信息。

默认情况下，IntelliJ IDEA使用项目node_modules文件夹中的TSLint包和存储当前文件的文件夹中的tslint.json配置文件。如果在当前文件文件夹中找不到tslint.json ，IntelliJ IDEA将在其父文件夹中查找一个直到项目根目录的文件夹。

如果您有几个将TSLint列为依赖项的package.json文件，IntelliJ IDEA会为每个package.json启动一个单独的进程并处理它下面的所有内容。这使您可以将特定TSLint版本或特定插件集应用于monorepo中的每个路径或具有多个TSLint配置的项目。

在所有新的IntelliJ IDEA项目中，此行为是默认的。要在先前创建的项目中启用它，请转到语言和框架| TypeScript | “ 设置/首选项”对话框中的TSLint （ Ctrl + Alt + S ），然后选择“ 自动TSLint配置”选项。

您也可以手动配置TSLint以使用自定义TSLint包和tslint.json，并指定一些其他规则。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| TypeScript | TSLint 。

  2. 选择“ 手动配置”选项。

  3. 指定要使用的Node.js解释器和TSLint包的路径。

  4. 在“ 配置文件”区域中，指定要使用的配置。

    默认情况下，IntelliJ IDEA首先查找tslint.json ， tslint.yaml或tslint.yml配置文件。IntelliJ IDEA从存储要检查的文件的文件夹开始搜索，然后在父文件夹中搜索，依此类推，直到到达项目根目录。如果未找到tslint.json ， tslint.yaml或tslint.yml配置文件，则TSLint将使用其默认的嵌入式配置文件。因此，您必须定义要在tslint.json ， tslint.yaml或tslint.yml配置文件中应用的配置，或者在自定义配置文件中应用，或者依赖于默认的嵌入式配置。

    • 要让IntelliJ IDEA查找tslint.json ， tslint.yaml或tslint.yml文件，请选择“ 自动搜索”选项。

    • 要使用自定义文件，请选择“ 配置文件”选项，并在“ 路径”字段中指定文件的位置。从列表中选择路径，或手动键入，或单击并从打开的对话框中选择相关文件。

  5. 如有必要，请在“ 附加规则目录”字段中，使用其他代码验证规则指定文件的位置。这些规则将在tslint.json ， tslint.yaml ， tslint.yml或来自上面指定的自定义配置文件的规则之后应用，因此将覆盖它们。

配置TSLint的突出显示

有关详细信息，请参阅代码检查和更改检查严重性

默认情况下，IntelliJ IDEA根据TSLint配置文件中的严重性级别标记检测到的错误和警告。例如，错误用红色波浪线突出显示，而警告用黄色背景标记。

更改TSLint配置中规则的严重性级别

  • 在TSLint配置文件中，找到要编辑的规则，并将其ID设置为warning或error 。从TSLint官方网站了解更多信息。

您可以从TSLint配置文件中覆盖严重性，以便IntelliJ IDEA忽略它们，并将linter报告的所有内容显示为错误，警告或自定义颜色。

忽略配置中的严重性级别

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|检查 。将打开“ 检查”页面 。

  2. 在中央窗格中，转到TypeScript | TSLint 。

  3. 在右侧窗格中，清除“ 从配置文件中使用规则严重性”复选框，然后选择要使用的严重性级别而不是默认级别。

从TSLint配置文件导入代码样式

IntelliJ IDEA了解tslint.json ， tslint.yaml或tslint.yml配置文件中描述的一些TSLint规则，并允许您将它们应用于项目中的TypeScript代码样式配置 。

  • 当您第一次打开项目时，IntelliJ IDEA 会自动从项目tslint.json ， tslint.yaml或tslint.yml导入代码样式。

  • 如果更新了tslint.json ， tslint.yaml或tslint.yml （手动或从您的版本控制），请在编辑器中将其打开，然后从上下文菜单中选择“ 应用TSLint代码样式规则” 。

    或者，只需对“从TSLint应用代码样式”回答“ 是 ”？“文件顶部的问题。

TSLint快速修复

IntelliJ IDEA允许您自动修复TSLint报告的一些问题。

  • 要修复特定错误，请将光标放在突出显示的代码上，按Alt + Enter ，然后选择TSLint：从列表中修复当前错误 。

  • 要修复文件中检测到的所有问题，请选择TSLint：修复当前文件 。

您可以抑制当前文件的TSLint规则，甚至是当前行。IntelliJ IDEA以格式/* tslint:disable:<rule name>自动生成禁用注释/* tslint:disable:<rule name>或// tslint:disable-next-line:<rule name>并将它们分别放在文件顶部或当前行上方。

要动态抑制TSLint规则

  1. 将光标置于TSLint报告的错误或警告处，然后按Alt + Enter 。

  2. 选择要禁用的规则的快速修复，然后按“ 向右” 。

  3. 从列表中， 为当前文件选择“ 抑制<规则名称>”或为当前行选择“ 抑制<规则名称>” 。
