IntelliJ IDEA 2019.1帮助

Heroku入门

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA提供了在不使用命令行界面的情况下将应用程序部署到Heroku的功能。您可以在IDE内部部署，重新部署和删除云应用程序。

先决条件

在IntelliJ IDEA中使用Heroku之前，应满足以下先决条件：

  • Heroku用户帐户已创建 。

  • 已安装Heroku集成插件。请参阅管理插件以了解如何执行此操作。

将Heroku用户帐户添加到IntelliJ IDEA

首先，您需要将现有的Heroku帐户添加到IntelliJ IDEA：

  1. 打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），然后选择“ 构建，执行，部署”云 。

  2. 点击按钮并选择Heroku 。

  3. 指定Heroku帐户的凭据：

    • 名称 - 指定服务器名称。稍后将使用此名称来创建Heroku 部署配置 。

    • 用户名（电子邮件） - 指定电子邮件地址。

    • 密码 - 指定帐户密码。

    • API密钥 - 指定为您的帐户生成的API密钥。

  4. 确保显示“ 连接成功”消息。然后，单击“ 确定” 。

创建Heroku部署配置

在将应用程序部署到Heroku之前，您需要创建Heroku部署配置：

  1. 选择运行|从主菜单中编辑配置 。

  2. 点击按钮（ Alt + Insert ）并从弹出窗口中选择Heroku Deployment 。

  3. 指定部署设置：

    • 名称 - 指定部署配置名称。

    • 服务器 - 确保选中了上一段中创建的服务器。

    • 部署 - 选择要部署的项目。

    • 使用自定义应用程序名称 - 指定用于访问应用程序的名称。

  4. 指定所有必需设置后，单击“ 确定” 。

部署应用程序

创建部署配置后，我们准备部署应用程序：

  1. 按Ctrl键两次，然后开始键入部署配置名称。

  2. 从列表中选择配置，然后按Enter键 。

  3. 在“ 提交和推送”对话框中，编写提交消息，然后单击“ 不提交推送” 。

  4. 在Git Login对话框中，指定任何用户名 。将Heroku API密钥复制并粘贴到密码字段。单击“ 登录” 。

    启用“ 记住”选项以保存凭据。在这种情况下，您可以在不指定API密钥的情况下重新部署应用程序。

  5. 等到部署应用程序。您可以在“ 应用程序服务器”窗口中查看该过程。

    部署应用程序后，单击链接以在浏览器中打开它。

重新部署应用程序

要发布已更改的应用程序版本：

  1. 在“ 应用程序服务器”窗口中，右键单击您的应用程序，然后选择“ 重新部署” 。

  2. 在“ 提交和推送”对话框中，编写提交消息，然后单击“ 提交并推送” 。

  3. （可选）在Git Login对话框中，指定任何用户名 。在“ 密码”字段中，输入Heroku API密钥。单击“ 登录” 。

  4. 部署应用程序后，切换到Web浏览器并重新加载页面以查看更改。

删除该应用程序

要从Heroku中删除您的应用程序及其源代码，请执行以下操作：

  1. 在“ 应用程序服务器”窗口中，右键单击您的应用程序，然后选择“ 取消部署” 。

  2. 在打开的对话框中，输入您的Heroku密码，然后单击“ 确定” 。

  3. 打开Web浏览器，刷新应用程序页面并确保您的应用程序不可用。
