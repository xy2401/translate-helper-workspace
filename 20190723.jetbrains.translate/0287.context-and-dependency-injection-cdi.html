上下文和依赖注入（CDI）

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA中的上下文和依赖关系注入 （CDI）支持基于Java EE：Context和Dependency Injection插件。此插件与IDE捆绑在一起并默认启用。

您可以在创建项目或模块时启用CDI支持。您还可以为现有模块添加CDI支持。

模块中的CDI由模块依赖项中的相应CDI库表示。您可以直接从IDE中下载所有必需的库文件。您还可以在需要时更改正在使用的CDI版本。

确保已启用CDI插件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件”已安装 。

  2. 在列表中找到Java EE：Contexts和Dependency Injection插件，并确保选中它旁边的复选框。

在创建项目或模块时启用CDI支持

  1. 要创建新项目，请在“欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件”|新的|项目 。

    要创建新模块，请打开所需项目并选择File |新的|模块 。

  2. 在打开的向导中，选择左侧的Java Enterprise 。

  3. 在向导的右侧，指定要使用的JDK，选择要支持的Java EE版本，并指定要使用的应用程序服务器。

  4. 从Additional Libraries and Frameworks列表中，选择CDI：Context and Dependency Injection复选框。

  5. 配置库。您可以使用现有库，创建和使用新库，下载库文件，或推迟设置库，直到以后。

    单击下一步 。

  6. 在向导的下一页上，指定新项目或模块的名称和位置。单击完成 。

如果您的计算机上没有JDK，请参阅创建项目时配置JDK部分，以获取有关如何下载和安装它的更多信息。

为现有模块添加CDI支持

  1. 在“项目”工具窗口中，右键单击所需的模块，然后选择“ 添加框架支持” 。

    要打开工具窗口，请从主菜单中选择View |工具窗口|项目 。

  2. 从左侧列表中，选择CDI：Context和Dependency Injection复选框。

  3. 配置库。您可以使用现有库，创建和使用新库，下载库文件，或推迟设置库，直到以后。

  4. 应用更改并关闭对话框。

因此，您指定的CDI库将添加到模块依赖项列表中。

更改CDI版本

  1. 点击工具栏上的图标（或按Ctrl + Shift + Alt + S ）打开“ 项目结构”对话框。

  2. 选择左侧的Modules并打开Dependencies选项卡。

  3. 选择必要的CDI库并单击 。

  4. 在“ 配置项目库”对话框中，单击“ 更改版本” 。

  5. 打开“ 版本”列表，然后选择新的库版本。应用更改并关闭对话框。
