在开始之前，请确保您的计算机上有Node.js.

IntelliJ IDEA与Gulp.js任务运行器集成。IntelliJ IDEA解析Gulpfile.js文件，识别任务定义，在树视图中显示任务，允许您在树中的任务及其在Gulpfile.js文件中的定义之间导航，并支持运行和调试任务。

Gulp.js任务可以从专用Gulp工具窗口中的任务树运行，也可以从Gulpfile.js文件运行 ，方法是启动Gulp.js运行配置 ，或者从另一个运行配置启动之前的任务。IntelliJ IDEA显示在“ 运行”工具窗口中执行任务的结果。工具窗口显示Grunt输出，报告发生的错误，列出尚未找到的包或插件等。最后执行的任务的名称显示在工具窗口的标题栏上。

安装Gulp.js

您也可以按照npm和Yarn中的说明在Node.js和NPM页面上安装软件包。

要在IntelliJ IDEA项目中使用Gulp，您需要两个包：

  • 全局安装的gulp-cli软件包（Gulp命令行界面），用于执行Gulp命令。

  • 一gulp包安装为发展依赖建项目任务树，并提供在编辑编码辅助Gulpfile.js文件。从Gulp.js官方网站了解有关Gupfile.js更多信息。

在全球范围内安装gulp-cli

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install -global gulp-cli

在项目中安装Gulp.js.

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install gulp --save-dev

从任务树运行Gulp.js任务

在当前IntelliJ IDEA会话期间第一次构建任务树时， Gulp工具窗口尚未打开。

IntelliJ IDEA允许您从Gulp工具窗口的任务树轻松快速地运行Gulp.js任务。IntelliJ IDEA会自动创建临时运行配置，如有必要，您可以保存并稍后使用。

通过在Project工具窗口中的Gulpfile.js的上下文菜单中选择Show Gulp Tasks或在编辑器中打开的Gulpfile.js ， Gulpfile.js在您调用Gulp.js时开始构建任务树 。树是根据调用Gulp.js的Gulpfile.js文件构建的。如果项目中有多个Gulpfile.js文件，则可以为每个文件构建一个单独的任务树，并在不放弃以前构建的任务树的情况下运行任务。每棵树都显示在一个单独的节点下。

从技术上讲，IntelliJ IDEA根据默认的Gulp.js运行配置调用Gulp.js并处理Gulpfile.js 。这是默默完成的，不需要你身边的任何步骤。

打开Gulp工具窗口

  • 在Project工具窗口中选择所需的Gulpfile.js文件，或在编辑器中打开它，然后从上下文菜单中选择Show Gulp Tasks 。

默认情况下，IntelliJ IDEA无法识别Gulpfile.js ES6，也无法构建任务树。要解决此问题，请更新默认的Gulp.js运行配置 。

从ES6 Gulpfile.js构建任务树

  1. 在主菜单上，选择“运行”|编辑配置 。

  2. 在“ 模板”节点下，单击“ Gulp.js” 。

  3. 在运行/调试配置：Gulp.js对话框打开，键入--harmony在节点选项字段，然后单击确定 。

从Gulp工具窗口构建任务树

  • 在Gulp工具窗口中，单击在工具栏上，从列表中选择所需的Gulpfile.js文件。默认情况下，IntelliJ IDEA会在项目的根目录中显示Gulpfile.js文件。

  • 如果您有另一个Gulpfile.js文件，请单击Choose Gulpfile.js并在打开的对话框中选择所需的Gulpfile.js文件。IntelliJ IDEA在其标题上添加一个新节点，其中包含所选Gulpfile.js文件的路径，并在新节点下构建任务树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的任务进行排序

  • 点击在工具栏上，从菜单中选择排序依据 ，然后选择名称 。
    默认情况下，树以Gulpfile.js （选项定义顺序 ）中的顺序显示任务。

运行任务

  • 双击任务。

  • 在树中选择任务，然后按Enter键或在上下文菜单中选择“运行<任务名称> ”。

运行默认任务

  • 在树中选择根节点，然后从上下文菜单中选择“运行默认值 ”。

运行几个任务

  • 使用多选模式：按住Shift （对于相邻项）或Ctrl （对于非相邻项）键并选择所需任务，然后在所选内容的上下文菜单中选择“运行”或“ 调试 ”。

导航到任务的定义

  • 在树中选择所需任务，然后在所选内容的上下文菜单中选择“ 跳转到源 ”。

从Gulpfile.js运行任务

  1. 将光标定位到要运行的任务的定义，然后从上下文菜单中选择“运行<任务名称> ”。IntelliJ IDEA使用所选任务的名称创建并启动临时运行配置。

  2. 要保存自动创建的临时运行配置，请将光标定位到创建它的任务的定义，然后从选择的上下文菜单中选择保存<任务名称> 。

根据运行配置运行和调试任务

除了使用IntelliJ IDEA自动创建的临时运行配置外，您还可以创建和启动自己的Gulp.js运行配置。

创建Gulp.js运行配置

  1. 在主菜单上，选择“运行”|编辑配置 。

  2. 点击在工具栏上，从列表中选择Gulp.js。运行/调试配置：Gulp.js对话框打开。

  3. 指定运行配置的名称，要运行的任务（使用空格作为分隔符），定义这些任务的Gulpfile.js文件的位置，以及当前项目根目录下安装的Gulp软件包的路径。

  4. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    （可选）指定Node.js的环境变量和执行任务的参数。使用格式--<parameter_name> <parameter_value> ，例如：-- --env development 。从Gulp官方网站了解更多信息。

运行任务

  • 从主工具栏上的列表中选择新创建的运行配置，然后单击在列表旁边。输出显示在“ 运行”工具窗口中 。

调试任务

  1. 创建Gulp.js运行/调试配置如上所述 。

  2. 在编辑器中打开Gulpfile.js文件，并在必要时在其中设置断点。

  3. 要启动调试会话，请从主工具栏上的列表中选择所需的调试配置，然后单击在列表旁边或选择Run |从主菜单中调试<配置名称> 。

  4. 在打开的“ 调试”工具窗口中，分析暂停的任务执行，逐步执行任务，等等，如检查暂停程序和单步执行程序中所述 。

将Gulp任务作为启动前任务运行

  1. 选择Run |打开Run / Debug Configurations对话框在主菜单上编辑配置 ，从列表中选择所需的配置，或通过单击重新创建并选择相关的运行配置类型。

  2. 在打开的对话框中，单击在Before launch区域中，从列表中选择Run Gulp任务 。

  3. 在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

自动运行Gulp.js任务

如果您有一些定期运行的任务，则可以将相应的运行配置添加到启动任务列表中。任务将在项目启动时自动执行。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 启动任务 ”。

  2. 在打开的“ 启动任务”页面上，单击“确定” 在工具栏上。

  3. 从列表中，选择所需的Gulp.js运行配置。配置将添加到列表中。

    如果项目中没有适用的配置，请单击然后选择“ 编辑配置” 。然后在打开的运行/调试配置：Gulp.js页面上定义具有所需设置的配置 。保存新配置时，它会自动添加到启动任务列表中。
