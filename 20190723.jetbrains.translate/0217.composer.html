IntelliJ IDEA |偏好|工具|命令行工具支持 macOS
按Ctrl + Alt + S

单击“ 添加”按钮并在“ 选择要添加的工具”对话框中选择“ Composer”时 ，将打开该对话框。
在此对话框中，通过指定启动Composer和指定文件以查找Composer命令的方式，在命令行模式下启用Composer Dependency Manager 。

要使此功能可用，您需要从插件存储库安装命令行工具支持插件。

IntelliJ IDEA为Composer命令解析指定的.phar存档或可执行文件的内容。完成文件分析后，IntelliJ IDEA将返回到“ 命令行工具支持”页面，其中指定的文件将添加到IntelliJ IDEA中可用的命令行工具列表中。

IntelliJ IDEA级别提供与Composer的集成，因此一旦配置，该工具可用于所有IntelliJ IDEA项目。只需根据您的需要激活或取消激活它。

描述

Composer.phar或PHP脚本

选择此选项可通过PHP脚本启动Composer ，或者让IntelliJ IDEA检测并启动composer.phar存档中的启动器。在此模式下，IntelliJ IDEA提供编码帮助并允许您执行脚本。

  • PHP可执行文件的路径：在此字段中，指定PHP引擎安装文件夹的位置。

  • composer.phar或composer的路径：在此文本框中，指定composer.phar存档的位置。

Composer可执行文件

选择此选项可通过Composer可执行文件启动Composer。在此模式下，您无法获得编码帮助，也无法执行脚本，因为没有为其指定PHP引擎。
在“可执行文件的路径”字段中，指定composer可执行文件的位置。

该工具仅适用于本地PHP解释器 。
