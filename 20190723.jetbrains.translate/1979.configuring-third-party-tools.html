配置第三方工具

在PyCharm中，您可以指定第三方独立应用程序并将其作为外部工具运行。您可以将上下文信息传递给这些工具（如当前选定的文件或项目源路径），查看IDE中的输出，将工具的启动配置为运行/调试配置的一个步骤等。

“ 外部工具”对话框中定义的工具在“ 工具”|中显示为命令外部工具菜单和各种上下文菜单。与大多数其他操作一样，可以为外部工具分配键盘快捷键以便快速访问。

调用外部工具的方法：

  • 从工具|中的可用工具列表中主菜单上的外部工具

  • 通过设置/首选项|中分配给该工具的键盘快捷键按键图

  • 作为运行/调试配置的Before启动步骤

有关如何配置和使用外部工具的示例，请参阅下一章。

示例：添加pylint作为外部工具

pylint是一种代码分析工具，可以检查您的代码并检测任何样式，逻辑和使用问题。它可能是PyCharm提供的代码验证功能的一个很好的补充。

安装pylint

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 项目”|项目口译员 。

  2. 点击按钮安装新包 。

  3. 在“ 可用包”窗口中，搜索“pylint”，然后在包列表中选择它，并单击“ 安装包” 。

    等到包安装完毕并关闭窗口。

现在您已在系统上安装了pylint，您可以配置它与Pycharm的集成。

配置pylint

  1. 在“ 设置/首选项”对话框中（ Ctrl + Alt + S ），导航到“ 工具”|外部工具 。

  2. 点击按钮添加新的外部工具。

  3. 添加配置选项，如下所示：

    • Group - 要在Tools |中显示的外部工具组的名称外部工具菜单

    • 名称 - 外部工具名称

    • 描述 - 可选描述

    • 程序 - pylint可执行文件的路径（ $PyInterpreterDirectory$是放置当前项目的Python解释器的目录）。
      例：

      $ PyInterpreterDirectory $ \ pylint的
      $PyInterpreterDirectory$/pylint （Linux，macOS）

    • 参数 - 指定应检查哪些文件和文件夹（本例中为car.py和test ），并设置pylint错误的输出格式。
      例：

      --msg-template =“$ FileDir $ \ {path}：{line}：{column}：{C} :( {symbol}）{msg}”car.py test
      --msg-template =“$ FileDir $ / {path}：{line}：{column}：{C} :( {symbol}）{msg}”car.py test

    • 工作目录 - 项目根目录

    • 建议从“ 高级选项”部分启用所有选项，并设置“ 输出”过滤器以将包含错误的文件的链接插入到pylint输出中，这样您就可以快速跳转到代码中的错误或警告。

  4. 单击“ 确定”以保存更改。单击“ 外部工具”窗口中的“ 应用”和“ 确定” ，完成添加外部工具 。

现在，您可以从Tools |访问此外部工具外部工具 。

您可以使用此菜单运行pylint。但是，您可能还希望将其添加到应用程序的运行/调试配置中。

在应用程序运行之前执行pylint

  1. 从主菜单中，选择“运行”|编辑配置 ，然后在“ 编辑配置”对话框中单击在发布之前部分。

  2. 从“ 添加新配置”列表中选择“运行外部”工具并指定pylint。它将添加到应用程序启动之前要执行的脚本列表中。

  3. 单击“ 确定”以保存更改

  4. 运行运行/调试配置（ Shift + F10 ）。

    您应该期望在“ 工具”窗口中看到带有以下示例输出的pylint选项卡：

您可以检查报告的错误，单击相应的链接以导航到有问题的代码，并进行所需的更改。
