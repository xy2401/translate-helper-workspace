文件编码

档案|设置|编辑|适用于Windows和Linux的文件编码
IntelliJ IDEA |偏好|编辑| macOS的文件编码
按Ctrl + Alt + S

使用此设置页面可以全局配置编码选项，为当前项目或项目中的任何文件或目录配置编码选项。

文件或目录编码优先于项目编码 ，而项目编码优先于全局编码 。

如果未定义文件或目录编码，则使用项目编码。如果无法进行项目编码（例如，如果尚未创建项目），则使用全局编码。

项目

描述

全局编码

如果未指定其他编码选项，请从此下拉列表中选择默认情况下应使用的编码。例如，当您指定默认项目的设置或从版本控制存储中签出源时，将应用编码。选择“ 系统默认值”以使用操作系统的默认编码或选择特定编码。

项目编码

从此下拉框中，选择要在下面的默认编码字段中未指定编码的文件夹中使用的默认编码 。选择“ 系统默认值”以使用操作系统的默认编码或选择特定编码。

路径

此列显示文件或目录的路径。

默认编码

此列显示文件或目录的编码（如果适用）。如果在文件中定义了编码，则无法对其进行配置，并以灰色字体显示。如果编码是可配置的，请单击所选文件或目录的“ 默认编码”列，然后从下拉列表中选择“编码”。

嵌入在文件中的编码信息会覆盖所选择的信息;嵌套文件或目录的编码信息会覆盖外部目录或整个项目的编码信息。

属性文件的默认编码

使用此下拉列表定义项目中属性文件的编码。根据Java API， java.util.的load(InputStream) / store(OutputStream, String)方法java.util.Properties类，对输入/输出流使用ISO 8859-1编码。除非您有特殊原因需要更改，否则建议使用此编码。在这种情况下，您可以从下拉列表中选择所需的编码;特别是，您可以使用为整个项目定义的编码。

透明的native-to-ascii转换

选择此选项可在属性文件中显示国家字符（非ISO 8859-1），存储为转义序列。

有关更多详细信息，请参阅配置属性文件的编码 。

如果未选中此复选框，则不会显示国家字符。

创建UTF-8文件

使用此选择器选择IntelliJ IDEA应如何创建UTF-8文件：具有字节顺序标记 （BOM），无BOM或仅适用于Windows的BOM。如果您使用Windows以外的操作系统并希望更灵活地使用UTF-8文件编码，这可能会有所帮助。请记住， 带有BOM选项的UTF-8可能会导致与BOM不兼容的软件或需要由解释程序运行的脚本出现问题。
