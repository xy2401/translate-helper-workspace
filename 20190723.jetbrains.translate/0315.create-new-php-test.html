描述

测试文件模板

在此列表中，选择将根据其生成测试类的测试文件模板 。IntelliJ IDEA为以下测试框架提供了内置模板： PHPUnit ， Codeception和PHPSpec 。

要测试的类（可选）

在此区域中，指定生产类以生成测试类。在“ 完全限定名称”文本框中键入类的完全限定名称 。指定的名称将用于提出测试类名称 。要使用完成，请按Ctrl + Space并从弹出列表中选择相关的生产类。

测试类

  • 名称：在此文本框中，指定要生成的测试类的名称。IntelliJ IDEA根据所选测试框架的命名约定自动组成生产类名称中的名称，例如， PHPUnit测试类的<production class> Test.php 。

  • 命名空间：在此文本框中，指定测试类将属于的命名空间。IntelliJ IDEA根据“ 目录”文本框的值自动完成命名空间。更改“ 目录”文本框时，“ 名称空间”文本框的值会相应更改。要使用完成，请按Ctrl + Space并从弹出列表中选择相关的命名空间。

  • 文件名：在此文本框中，指定将为测试类定义创建的文件的名称。默认情况下，名称与测试类名称相同。但是，如果在一个文件中定义了多个生产类，则对于每个生成的测试类，IntelliJ IDEA将创建一个单独的文件。

  • 目录：在此文本框中，指定将存储包含测试类的文件的目录。默认情况下，它是存储生产类的目录。要指定其他目录，请单击“ 浏览” ，然后在打开的对话框中选择相关目录。要使用完成，请按Ctrl + Space并从弹出列表中选择相关目录。

生成测试方法

在表中，选择要为其生成测试方法存根的生产类方法旁边的复选框。要包含父类中的继承方法，请选中“ 显示继承的方法”复选框。IntelliJ IDEA会自动将测试方法的名称组成test<production method> 。

您可以在“ 文件和代码模板”设置页面的“代码”选项卡上自定义用于生成测试方法存根的代码模板 。
