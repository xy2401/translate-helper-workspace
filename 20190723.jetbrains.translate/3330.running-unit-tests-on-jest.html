笑话

Jest是客户端JavaScript应用程序和React应用程序的测试平台。从Jest官方网站了解有关该平台的更多信息。

您可以在WebStorm中使用Jest运行和调试测试。您可以在树视图中查看测试结果，并从那里轻松导航到测试源。测试状态显示在编辑器的测试旁边，可以选择快速运行或调试它。

在开始之前，请确保您的计算机上有Node.js.

安装Jest

您还可以在Node.js和NPM页面上安装jest包，如npm和Yarn中所述 。

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install --save-dev jest

    另请参阅Jest官方网站上的入门 。

运行测试

使用WebStorm，您可以直接从编辑器快速运行单个Jest测试，或创建运行/调试配置以执行部分或全部测试。

从编辑器运行单个测试

  • 点击要么在左侧排水沟中，从列表中选择“运行<test_name> ”。

    由于测试状态图标，您还可以在编辑器中查看测试是否已通过或失败在左边的排水沟里。

创建Jest运行配置

  1. 在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。将打开“ 运行/调试配置：Jest”对话框。

  2. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

  3. 指定jest ， react-scripts ， react-script-ts ， react-super-scripts或react-awesome-scripts包的位置。

  4. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  5. （可选）指定要使用的jest.config文件：从列表中选择相关文件，或单击并在打开的对话框中选择它，或者只在字段中键入路径。如果该字段为空，WebStorm将使用jest键查找package.json文件。搜索在文件系统中从工作目录向上执行。如果找不到合适的package.json文件，则会动态生成Jest默认配置 。

  6. （可选）配置在相关源文件中的更改时自动重新运行测试。为此，请在Jest选项字段中添加--watch标志。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右侧。

  2. 测试服务器自动启动，无需您执行任何步骤。在“ 运行”工具窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在Test Runner选项卡中 ，单击在工具栏上。WebStorm将执行上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请在其上下文菜单中选择“运行<测试名称> ”。

有关详细信息，请参阅重新运行测试 。

导航

使用WebStorm，您可以在文件和相关测试文件之间跳转，或者从Test Runner选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择Go To |测试或转到|从上下文菜单中测试主题 ，或者只需按Ctrl + Shift + T.

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源 ”。测试文件在编辑器中打开，光标位于测试定义中。

  • 对于失败的测试，WebStorm会从堆栈跟踪中引导您进入测试中的故障线。如果确切的行不在堆栈跟踪中，您将进入测试定义。

测试文件应遵循流行的命名约定，例如，具有.test. ， .spec.或_spec.后缀，应位于源文件旁边或test文件夹中。

快照测试

WebStorm与Jest的集成支持快照测试等强大功能。如果快照与呈现的应用程序不匹配，则测试失败。这表示代码中的某些更改导致此不匹配或快照已过时且需要更新。您可以在WebStorm内置差异查看器中查看导致这种不匹配的原因。

更新失败测试的快照

  • 使用“ 运行”工具窗口的“ 测试运行器”选项卡中的“ 单击以更新快照”链接：

从测试文件导航到相关的快照文件

  • 点击在测试旁边的左边排水沟里。

  • 将光标toMatchSnapshot() ，按Ctrl + Alt + Home并从列表中选择所需的快照。

将实际输出与预期快照进行比较

  • 使用“ 测试运行器”选项卡右侧窗格中的“ 单击以查看差异”链接。

调试测试

使用WebStorm，您可以直接从编辑器快速开始调试单个Jest测试，或创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 点击要么在左侧排水沟中，从列表中选择Debug <test_name> 。

通过运行/调试配置启动测试调试

  1. 如上所述创建Jest运行/调试配置。

  2. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右侧。

  3. 在打开的调试工具窗口中，像往常一样继续： 逐步执行测试 ， 停止并恢复测试执行， 暂停时检查测试 ，等等。

监控代码覆盖率

使用WebStorm，您还可以监控Jest测试涵盖的代码量。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记已覆盖和未覆盖的行。

使用coverage运行测试

  1. 如上所述创建Jest运行/调试配置。

  2. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右侧。
    或者，从编辑器快速运行特定套件或测试覆盖：单击要么在左侧排水沟中，从列表中选择带有Coverage的Run <test_name> 。

  3. 在Coverage工具窗口中监视代码覆盖率。该报告显示测试覆盖了多少文件以及其中包含的行的百分比。从报告中，您可以跳转到该文件并查看覆盖的线条 - 标记为绿色 - 以及未覆盖的线条 - 标记为红色：
