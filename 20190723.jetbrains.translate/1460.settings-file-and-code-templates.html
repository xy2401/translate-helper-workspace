文件和代码模板

档案|设置|编辑|适用于Windows和Linux的文件和代码模板
IntelliJ IDEA |偏好|编辑| macOS的文件和代码模板
按Ctrl + Alt + S

IntelliJ IDEA使用文件模板创建文件。使用此页面可以查看，编辑和创建此类模板。

不同的模板组位于不同的选项卡上 。

选择模板时，其内容和说明将显示在页面的右侧。

  • 每个项目与默认方案
  • 标签
  • 工具栏
  • 模板设置和内容

每个项目与默认方案

项目

描述

方案

选择文件和代码模板设置是否属于整个工作区或当前项目：

  • 当文件和代码模板是全局模板时，选择默认方案。

  • 如果要使用可共享项目特定的文件和代码模板，则选择项目方案。

标签

标签

描述

档

此选项卡显示可用的文件模板。

您可以编辑现有模板，也可以创建新模板。

以粗体显示的模板无法删除;他们的名字和扩展名无法编辑。

包括

此选项卡显示可以包含在文件模板中的可重用片段的模板。

您可以编辑现有模板，也可以创建新模板。

码

此选项卡显示内置片段 - IntelliJ IDEA可在各种典型情况下生成的代码片段模板，例如，用于生成已实现或已重写的方法体。

您可以编辑现有代码段，但无法创建新代码段。

其他

此选项卡显示与各种应用程序服务器和框架相关的模板组。

您可以编辑现有模板，但不能创建新模板。

提示

描述

创建模板

单击此按钮可在当前打开的选项卡中创建新模板。此选项仅在“ 文件”和“ 包含”选项卡中可用。

新模板的位置由Schema列表定义。

删除模板

单击此按钮可删除所选模板。此选项仅适用于“ 文件”和“ 包含”选项卡中的某些模板。

复制模板

单击此按钮可创建所选模板的副本。此选项仅在“ 文件”和“ 包含”选项卡中可用。

重置为默认

单击此按钮可将选定的模板还原为其原始状态。此选项仅适用于已修改的模板（此类模板以蓝色突出显示）。

N / A

重启

当您开始编辑模板时，此链接将显示在页面的右上角。单击此链接会将所有未保存的更改重置为任何选项卡中的任何模板。

模板设置和内容

项目

描述

名称

创建新模板时会显示此字段。指定新模板的名称。

延期

在此字段中，指定扩展名。IntelliJ IDEA将在创建此类型的新文件时应用此模板。

模板文本

编辑模板内容。您可以使用：

  • 纯文本。

  • 使用模板包含的 #parse指令。

  • 自定义变量。变量的名称可以通过#set指令直接在模板中定义，也可以在文件创建期间定义。

    请注意，IntelliJ IDEA不会提示使用#set定义的Velocity变量的值。

  • 要变换为${<variable_name>}格式的相应值的${<variable_name>} 。

    可以在文件模板中使用以下预定义变量：

    变量

    描述

    $ {DATE}

    当前系统日期

    $ {DAY}

    当月的当天

    $ {} DS

    美元符号（ $ ）。此变量用于转义美元字符，因此不会将其视为模板变量的前缀。

    $ {} END

    完成编辑变量后的插入位置

    $ {FILE_NAME}

    新PHP文件的名称（如果启用了PHP插件 ）

    $ {HOUR}

    当前时间

    $ {} MINUTE

    当前分钟

    $ {MONTH}

    这个月

    $ {} MONTH_NAME_FULL

    当月的全名（1月，2月等）

    $ {} MONTH_NAME_SHORT

    当前月份名称的前三个字母（Jan，Feb等）

    $ {NAME}

    新实体的名称（文件，类，接口等）

    $ {} PACKAGE_NAME

    创建新类或接口文件的目标包的名称

    $ {PRODUCT_NAME}

    IDE的名称（例如，IntelliJ IDEA）

    $ {} PROJECT_NAME

    当前项目的名称

    $ {TIME}

    当前系统时间

    $ {USER}

    当前用户的登录名

    $ {YEAR}

    今年

    IntelliJ IDEA为PHP包含模板提供了一组附加变量，用于定义可重用的代码片段 ，例如文件头和PHPDoc注释 。

    PHP包含模板中可以使用以下附加预定义变量：

    变量

    描述

    $ {} CARET

    完成编辑变量后的插入位置。仅在文件创建期间生成并插入PHPDoc注释时应用。当通过Code |创建PHPDoc注释时生成| PHPDoc块 ，可以选择多种功能或方法。可以为多个类，函数，方法或字段创建此案例中的文档注释，因此忽略${CARET}变量。

    $ {} CLASS_NAME

    定义了生成PHPDoc注释的字段或方法的类的名称

    $ {NAME}

    将为其生成PHPDoc注释的类，字段或函数（方法）的名称

    $ {命名空间}

    类或字段命名空间的完全限定名称（不带前导斜杠）

    $ {} PARAM_DOC

    参数的文档注释。计算一组@param type name行。如果生成注释的函数不包含任何参数，则求值为空字符串。

    $ {} STATIC

    如果生成PHPDoc注释的函数（方法）或字段是静态的，则由static关键字替换。否则计算为空字符串。

    $ {} THROWS_DOC

    文档注释异常。计算一组@throws type行。如果生成注释的函数不抛出任何异常，则求值为空字符串。

    $ {} TYPE_HINT

    返回函数（方法）的值以生成PHPDoc注释。如果通过函数（方法）的静态分析无法检测返回类型，则求值为void 。

    对待美元符号

    • 您可以防止将模板变量中的美元字符（ $ ）视为前缀。如果您需要一个美元字符（按原样插入$ ，请使用${DS}文件模板变量。应用模板时，此变量的计算结果为普通美元字符（ $ ）。

      例子：

      • 要在默认类模板中使用某些版本控制关键字（例如$Revision$ ， $Date$等），请写入${DS}而不是美元前缀（ $ ）。

      • 模板代码${DS}this将呈现为$this 。

根据风格重新格式化

选中此复选框，以根据“ 代码样式”页面上定义的样式重新生成IntelliJ IDEA生成的存根文件。

此选项仅在“ 文件”选项卡中可用。

启用实时模板

选中此复选框以在文件模板中使用实时模板。在这样做时，必须将实时模板片段放入Velocity转义语法中。

例如：

＃[[$ MY_VARIABLE $ $ END $]]＃

因此，可以指定光标位置。请注意，这里需要使用实时模板变量 ！

描述

此只读字段提供有关模板，其预定义变量及其工作方式的信息。

此字段在自定义模板中不可用。
