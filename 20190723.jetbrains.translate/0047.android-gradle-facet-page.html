Android-Gradle Facet

档案|项目结构|模块| <module> | Android的摇篮
档案|项目结构|方面| Android-Gradle（<module>）

使用此页面配置附加到特定模块的Android-Gradle构面的设置。

在这个部分：

  • 属性选项卡
  • 签名标签
  • Flavors选项卡
  • 构建类型选项卡
  • 依赖选项卡

属性选项卡

此选项卡允许您指定build.gradle文件的属性。

项目

描述

编译SDK版本

使用此下拉列表选择编译目标。

构建工具版本

使用此下拉列表选择构建工具的版本。

图书馆资料库

指定库存储库。默认情况下， jcenter存储库用于Android-Gradle应用程序。有关更多信息，请参阅jcenter存储库 。

忽略资产模式

使用此字段指定要忽略的资产模式。

递增的Dex

使用此下拉列表设置增量dexing。您可以选择true或false选项，或将其留空。

源兼容性

使用此下拉列表选择Java源编译的Java版本兼容性。

目标兼容性

使用此下拉列表选择Java版本以生成Java类。

签名标签

此选项卡允许您指定签名配置的设置。在选项卡的左侧区域中添加config类时，这些字段将变为可编辑。您还可以删除不需要的config类。

项目

描述

名称

此字段显示您选择的config类的名称。您可以修改名称，但它应该是可识别的Java标识符。

关键别名

使用此字段输入私钥存储文件的名称。

密码密码

使用此字段输入私钥的密码。

存储文件

使用此字段输入私钥存储文件的位置。或者，单击并在打开的对话框中选择目标文件夹。

存储密码

使用此字段输入商店文件的密码。

Flavors选项卡

此选项卡允许您指定flavor的配置设置。在选项卡的左侧区域添加flavor时，这些字段将变为可编辑。您还可以删除不需要的flavor条目。

项目

描述

名称

此字段显示您选择的构建flavor的名称。您可以修改名称。

Min Sdk版本

使用此字段为flavor添加最低SDK版本。

申请ID

使用此字段添加应用程序ID。

Proguard文件

使用此字段输入Proguard文件的位置。或者，单击并在打开的对话框中选择目标文件夹。

签名配置

使用此下拉列表选择在“ 签名”选项卡中选择的config类的名称。

目标Sdk版本

使用此下拉列表为您的构建风格选择目标SDK版本。

测试仪器跑步者

使用此字段输入用于运行测试用例的类的名称。默认情况下，测试类名称为android.test.InstrumentationTestRunner 。

测试应用程序ID

使用此字段输入测试的应用程序ID。

版本代码

使用此字段输入一个整数值，该值表示相对于其他版本的应用程序代码的版本。

版本名称

使用此字段输入表示应用程序代码的发行版本的字符串值，因为它应显示给用户。

构建类型选项卡

此选项卡允许您指定Android构建类型的设置。默认情况下，会创建调试版和发布版，并显示在选项卡的左侧区域。您可以添加新条目或删除现有条目。

项目

描述

名称

此字段显示已添加的构建类型的名称。您可以修改名称。

可调试

使用此属性将其设置为true或false具体取决于构建类型。

Jni Debug Build

使用此属性将其设置为true或false具体取决于构建类型。

签名配置

使用此下拉列表选择在“ 签名”选项卡中选择的config类的名称。

Renderscript Debug Build

使用此属性可将Renderscript调试版本设置为true或false具体取决于构建类型。

Renderscript Optim Level

使用此属性设置renderscript编译器的优化级别。默认值为3。

Minify Enabled

使用此属性可将代码的缩小设置为true或false具体取决于构建类型。

伪区域设置已启用

使用此属性可将伪语言环境设置为true或false具体取决于构建类型。

Proguard文件

使用此字段输入Proguard文件的位置。或者，单击并在打开的对话框中选择目标文件夹。

应用程序ID后缀

使用此字段输入应用程序ID的后缀。

版本名称后缀

使用此字段输入版本名称后缀。

拉链对齐

使用此下拉列表为文件设置zipalign 。您可以使用true或false具体取决于构建文件。

依赖选项卡

使用此选项卡指定应用程序的依赖项 。

项目

描述

范围

使用此设置可分别为源和测试源以及构建和运行阶段控制类路径。（当1）编译源代码时，类路径可能不同2）编译测试源3）运行已编译的源代码4）运行测试。Scope选项定义包含依赖项的类路径。）

您可以选择以下选项：

  • 编译 - 使用此选项在编译和运行阶段将依赖项包含在源的类路径中。

  • 提供 - 使用此选项包括运行时环境提供的依赖项。

  • APK - 使用此选项将依赖项包含在将由Android模块编译生成的.apk文件中。

  • 测试编译 - 使用此选项在测试编译阶段将依赖项包含在测试源的类路径中。

  • 调试编译 - 使用此选项在编译阶段将相关性包含在调试构建类型的类路径中。

  • 发布编译 - 使用此选项在编译阶段将依赖项包含在发布构建类型的类路径中。

或Alt + Insert

使用此图标或快捷方式将库或模块添加到模块依赖项列表中。选择以下选项之一：>

  • 罐子或目录。 选择此选项可创建新模块库并将其添加到依赖项列表中。在打开的对话框中 ，选择要包含在库中的文件和文件夹。这些文件可能是包含此类文件的单个.class和.java文件，目录和存档（ .jar和.zip ）以及包含Java本机库（ .dll ， .so或.jnilib ）的目录。

  • 图书馆。 选择此选项可将一个或多个现有项目，全局或应用程序服务器库添加到依赖项列表中。在打开的对话框中，选择库或库，然后单击“ 添加所选项” 。

  • 模块依赖。 选择此选项可指定当前模块应依赖的模块。

或Alt +删除

使用此图标或快捷方式从依赖项列表中删除所选项。

或Alt + Up和或Alt + Down

使用这些图标和快捷方式在列表中向上或向下移动所选项目。请参阅模块依赖性 。
