常规选项卡

使用此选项卡可以编辑此类构建配置设置，如名称， 类型 ，输出文件名和文件夹等。

项目

描述

名称

使用此字段可以编辑构建配置的名称。

类型

构建配置类型的简要说明，例如， 用于Web的ActionScript应用程序（Flash Player） ，表示：

  • 目标平台是Web。

  • 输出类型是Application（SWF）。

  • 该应用程序是纯ActionScript（即它不使用Flex框架）。

要更改构建配置类型，请单击“ 更改” ，然后在打开的对话框中指定构建配置属性。

主要课程

对于Application和Runtime-loaded模块输出类型： SWF文件的主类 。

在字段中编辑类的名称或单击 （ Shift + Enter ）并在“ 选择主类”对话框中选择所需的类。

输出文件名

使用此字段可以编辑输出文件的名称和扩展名。

导出目录

生成输出文件的文件夹。

在字段中编辑此文件夹的路径，或单击 （ Shift + Enter ）并在打开的对话框中选择所需的文件夹。

使用HTML包装器

对于Web应用程序：如果要将HTML包装器包含在输出中，请选择此选项。
  • 带模板的文件夹。 指定构成HTML包装器模板的文件所在的文件夹的路径。
    • 要使用包装器模板文件选择现有文件夹，请单击 （ Shift + Enter ）并在打开的对话框中选择文件夹。

      请注意，相应的文件夹必须包含文件index.template.html并且此文件必须包含令牌${swf} 。

    • 要使用包装器模板文件创建新文件夹，请单击“ 创建” ，然后在“ 创建HTML包装器模板”对话框中指定文件夹位置和包装器选项。

在编译期间，模板中包含的标记（ index.template.html ）将替换为适当的值。例如， ${swf}将替换为.swf文件名。生成的.html包装器文件将与.swf文件具有相同的名称。

另请参阅使用SWF元数据标记来控制HTML包装器属性 。

运行时加载的模块

对于Web和桌面应用程序：如果您的应用程序具有模块化结构（请参阅Flex文档中的模块化应用程序概述 ），则可以使用此字段指定对相应的运行时加载模块（RLM）的依赖关系。

点击 （ Shift + Enter ）。在打开的Runtime-Loaded Modules对话框中：

  • 要添加依赖项，请单击 （ Alt + Insert ）并在打开的“选择运行时加载模块的主类”对话框中选择相应RLM的主类 。

  • 要优化模块SWF文件大小，请选中“ 优化”复选框。

  • 要用不同的类替换类，请单击相应的Main Class表单元格，单击 （ Shift + Enter ），然后在打开的对话框中选择所需的类。

  • 要从列表中删除依赖项，请选择要删除的依赖项（RLM）并单击 （ Alt + Delete ）。

请注意，还有一种指定RLM依赖关系的替代方法：您可以在“ 依赖关系”选项卡上添加RLM构建配置的依赖关系 。

另请参阅配置模块化应用程序的依赖关系 。

运行时样式表

对于Web和桌面应用程序：如有必要，请指定应编译为运行时样式表的.css文件。

点击 （ Shift + Enter ）。在打开的CSS文件编译对话框中：

  • 要将.css文件添加到列表，请单击 （ Alt + Insert ）并在打开的对话框中选择所需的文件。

  • 要使用其他文件替换列表中已有的文件，请单击相应的条目，单击 （ Shift + Enter ），然后在打开的对话框中选择文件。

  • 要从列表中删除项目，请选择要删除的项目，然后单击 （ Alt + Delete ）。

跳过编译

如果您不打算使用此构建配置编译模块，请选择此选项。如果这样做，构建配置将仅影响代码验证和（错误）突出显示。
