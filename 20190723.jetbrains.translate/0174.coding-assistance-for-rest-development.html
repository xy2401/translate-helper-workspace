代码检查和快速修复

IntelliJ IDEA支持代码检查，并建议在以下情况下快速修复 ：

问题

默认快速修复

方法注释与方法返回类型之间的不一致： @GET注释方法返回void值。

更改为字符串。

资源方法错误。

删除@Path注释。

参数类型不正确。

验证使用@QueryParam或@PathParam注释注释的参数。

@DefaultValue问题。

@GET 公共 字符串 得到 （@DefaultValue（ “33.5”）@QueryParam（ “STR”）INT STR）{ 返回 “你好”; }
DefaultValue标记为红色，描述为“无法转换为int”。

休息@Path和@PathParam注释检查。

Rest References解决问题。

@PathParam注释中的引用将解析为@Path注释中的模板

此方法应该只有一个HTTP方法指示符。

例：

@GET @POST 公共 字符串 得到 （）{ 回报 “你好”; }
@POST标记为红色。

代码编辑器中的HTTP客户端

IntelliJ IDEA代码编辑器中的专用HTTP客户端提供了以下用于处理HTTP请求的工具：

  • 编写和执行请求

  • 显示服务器响应和执行的请求历史记录

  • 完成 Java代码中指定的@Path和@Produces注释的HTTP请求部分

更改Java代码

在编辑器中更改HTTP请求

@Path注释已更新。

调用代码完成时（ Ctrl + Space ）， Request-URI建议列表的内容会发生变化。

@Produces注释已更新。

调用代码完成时（ Ctrl + Space ）， Accept头字段建议列表的内容会发生变化。
