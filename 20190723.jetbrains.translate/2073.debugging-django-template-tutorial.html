在你开始之前

确保：

  • 您正在使用PyCharm专业版。本教程是使用PyCharm版本2017.2编写的。

  • Django被指定为项目模板语言。有关详细信息，请参阅配置模板语言部分

准备一个例子

首先，让我们使用应用程序poll 创建一个Django项目 MyDjangoProject1 。

接下来，打开文件views.py进行编辑（ F4 ）并查看import语句已经存在。在第二行中，您会看到手动执行某些操作的邀请：

来自 django.shortcuts import render ＃在这里创建你的看法。

输入以下代码：

来自 django.shortcuts import render def index （ request ）： return render （ request ， 'index.html' ， context = { 'hello' ： 'world' }）

您看到index.html被标记为未解析的引用：

按Alt + Enter或单击并选择创建缺少的模板：

在index.html文件中键入以下代码：

{ ％ for char in hello ％ } {{ char }} { ％ endfor ％ }

这意味着变量hello的值world将打印一个接一个的字符。

接下来，打开以编辑urls.py文件（ F4 ）。

在此文件中找到（ Ctrl + F ）字符串path ，在admin站点的url之后，键入以下代码：

path （ 'index /' ， index ），

不要忘记import语句！

编辑和启动运行/调试配置

请注意， Django服务器运行/调试配置是自动创建的，唯一需要的是编辑它。要更改此运行/调试配置，请在下拉列表中选择“ 编辑配置...”命令：

不要为“ 模板”节点下列出的默认运行/调试配置设置工作目录。这可能会导致新创建的运行/调试配置中的未解析目标。

例如，更改端口号。这里将是8123 。

接下来，让我们在调试模式下启动此运行/调试配置：从下拉列表中选择此配置并单击 ：

您可以在下拉列表中看到运行/调试配置的名称按钮和调试工具窗口用点标记，这意味着运行/调试配置变为活动状态。

运行/调试配置显示错误消息（404），因为管理站点和索引页都不存在。但是，我们需要在断点处暂停我们的应用程序。为此，我们需要将名称index添加到地址栏的内容中。页面仍然找不到，但在PyCharm中，您会看到带有命中断点的模板：

和调试工具窗口。

使用“调试”工具窗口

在Django应用程序中， 调试工具窗口的所有功能都可用，与纯Python脚本相同。例如，您可以单步执行应用程序，计算表达式，查看变量等。

单步执行模板

步进工具栏变为活动状态，并且步进按钮可用。例如，单击并看到char变量的值更改为单词world的下一个字母。

有关详细信息，请参阅“单步执行程序”部分。

评估表达式

按Alt + F8 ，或在步进工具栏上单击 。在打开的对话框中，键入要评估的表达式，然后单击“ 评估” ：

有关详细信息，请参阅“ 评估表达式”部分。

看变量

假设，您希望始终关注模板的某个变量，比如char 。怎么做？

在“变量”选项卡中，确保按下按钮（如果按下此按钮，则在“变量”选项卡中可以看到手表），然后单击按钮。输入感兴趣的变量的名称，现在您的手表始终位于“变量”选项卡的顶部：

有关详细信息，请参阅添加，编辑和删除监视 。
