IntelliJ IDEA 2019.1帮助

运行/调试配置：Spring Boot

此功能仅在Ultimate版本中受支持。

配置选项卡

主要课程

指定将要执行的类的完全限定名称（传递给JRE）。

环境

VM选项

如有必要，请指定要在服务器启动时传递到服务器JVM的命令行选项。

  • 使用空格分隔选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或参数本身用双引号括起来，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号作为参数的一部分，请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性：

    - Dfoo = $ { MY_ENV_VAR }

程序参数

以您在命令行中使用的格式指定要传递给程序的参数。使用与指定VM选项相同的规则。

工作目录

如有必要，请指定正在运行的应用程序将使用的目录。此目录是所有相对输入和输出路径的起点。默认情况下指定项目目录。

环境变量

单击“ 浏览”以创建变量并指定其值。

使用模块的classpath

选择将使用其类路径运行应用程序的模块。

包含具有“提供”范围的依赖项

启用此选项可将具有提供范围的依赖项添加到运行时类路径。

JRE

默认情况下，模块依赖项中的最新JDK用于运行应用程序。您可以指定备用JDK或JRE。

缩短命令行

如果类路径太长或者您有许多超出OS命令行长度限制的VM参数，请选择将用于缩短命令行的方法：

  • none ：IntelliJ IDEA不会缩短长类路径。如果命令行超出操作系统限制，IntelliJ IDEA将无法运行您的应用程序，并将显示一条消息，建议您指定缩短方法。

  • JAR清单 ：IntelliJ IDEA将通过临时classpath.jar传递一个长类classpath.jar 。原类路径中所定义的manifest文件作为class-path在属性classpath.jar 。请注意，如果使用此方法缩短了完整命令行，则可以预览完整命令行，而不仅仅是临时classpath.jar 。

  • classpath.file ：IntelliJ IDEA会将长类路径写入文本文件。

  • 用户本地默认值 ：为IntelliJ IDEA版本2017.3之前创建的项目自动设置此旧版选项。IntelliJ IDEA将根据ide/workspace.xml和idea.config.path/options/options.xml文件中设置的属性配置此设置。

如果您选择共享运行/调试配置，则会共享此设置，因此无论操作系统如何，都将对您的团队成员应用相同的方法。

春季启动

启用调试输出

通过添加-Ddebug VM选项启用日志记录调试输出。

隐藏横幅

通过添加-Dspring.main.banner-mode=OFF VM选项隐藏Spring Boot启动-Dspring.main.banner-mode=OFF （对于Spring Boot 1.4.x及更早版本，选项为-Dspring.main.banner=false ）。

启用启动优化

选择此选项可加快应用程序启动速度。将添加-XX:TieredStopAtLevel=1和-noverify VM选项。

启用JXM代理

通过添加以下VM选项（需要Spring Boot 1.3.0或更高版本），启用显示应用程序端点数据的JMX代理：

  • -Dcom.sun.management.jmxremote
  • -DSpring.liveBeansView.mbeanDomain
  • -Dspring.application.admin.enabled =真

对于具有FAT文件系统的Windows用户，默认情况下不提供Spring Boot端点功能。要启用它，请将-XX:+PerfBypassFileSystemCheck添加到VM选项 ，并确保%TMP%\hsperfdata_username具有完整文件夹访问权限。

IntelliJ IDEA使用本地JMX连接器检索Spring Boot执行器端点的数据。但是，如果Spring Boot应用程序和IntelliJ IDEA JVM具有不同的位数且您使用的是OpenJ9 JDK，则无法获取本地JMX连接器地址。在这种情况下，请将以下行添加到VM选项 ：

  • -Dcom.sun.management.jmxremote.port = {some_port}
  • -Dcom.sun.management.jmxremote.authenticate = FALSE
  • -Dcom.sun.management.jmxremote.ssl = FALSE

正如Oracle官方文档中所述，此配置不安全。任何知道（或猜测）您的端口号和主机名的远程用户都将能够监视和控制您的Java应用程序和平台。

在'更新'动作

指定在单击“ 更新<应用程序名称>应用程序”后IDE将执行的操作：

  • 更新资源 - 编译所有已修改和相关的资源文件。

  • 更新类和资源 - 编译所有已修改和相关的文件。

  • 更新触发器文件 - 更新触发器文件。通过添加-Dspring VM选项设置触发器文件名。

  • 如果失败，热交换类和更新触发器文件 - （仅在调试模式下可用）编译所有已修改和相关的文件，热交换类，如果交换失败，则更新触发器文件。通过添加指定触发器文件名
    -Dspring.devtools.restart.trigger-file=.restartTriggerFile VM选项。

在帧停用时

指定切换到另一个应用程序（IntelliJ IDEA除外）后IDE将执行的操作：

  • 更新资源 - 编译所有已修改和相关的资源文件。

  • 更新类和资源 - 编译所有已修改和相关的文件。

活动档案

指定要在此运行/调试配置中使用的活动Spring配置文件 。

覆盖参数

使用此表指定要覆盖的Spring Boot配置参数。这些参数通常在配置文件中定义。在运行/调试配置中列出它们允许您通过修改运行/调试配置而不是配置文件来轻松切换参数，或者使用具有不同参数值的多个运行/调试配置。

代码覆盖率选项卡

使用此选项卡配置代码覆盖率监视选项。

选择覆盖跑步者

选择所需的代码覆盖率运行器。

默认情况下，IntelliJ IDEA使用自己的覆盖引擎和采样模式。您也可以选择JaCoCo或Emma来计算覆盖率。

采样

选择此选项可以最小化速度来测量代码覆盖率。

追踪

选择此选项可收集准确的分支覆盖范围。此模式仅适用于IntelliJ IDEA代码覆盖运行器。

跟踪每个测试覆盖率

选中此复选框可检测一个测试所涵盖的行以及所有涉及行的测试。

用于记录代码覆盖率数据的包和类

点击和用于指定要测量的类和包的按钮。您还可以通过在列表中选择它们并单击来从列表中删除类和包 。

在测试文件夹中启用覆盖

如果选中此复选框，则标记为test的文件夹包含在代码覆盖率分析中。

日志选项卡

使用此选项卡指定运行或调试时生成的日志文件必须显示在“运行/调试”工具窗口的控制台中。

要在控制台中显示的日志文件

在此表中，指定将在“运行/调试”工具窗口中显示的文件或一组文件。

将控制台输出保存到文件

将控制台输出保存到指定位置。

将消息打印到标准输出流时显示控制台

如果关联进程正在写入Standard.out，则启用输出控制台并将其转发。

将消息打印到标准错误流时显示控制台

如果关联进程正在写入Standard.err，请启用输出控制台并将其转发。

在发布选项之前

使用此部分指定在启动运行/调试会话之前必须执行的任务。您指定的任务将按它们在列表中显示的顺序执行。

显示此页面

此选项在启动运行/调试会话之前显示运行/调试设置。

激活工具窗口

运行会话后，选中该复选框以打开“运行/调试”工具窗口。
