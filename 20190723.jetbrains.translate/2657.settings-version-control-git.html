使用此页面指定将应用于Git控件下项目目录的版本控制设置。

描述

Git可执行文件的路径

在此字段中，指定Git可执行文件的路径。键入路径或单击“ 浏览” 并在打开的对话框中指定路径。

测试

单击此按钮以验证Git可执行文件的路径。

仅为当前项目设置此路径

如果要为不同的项目使用不同的路径，并且不希望全局应用此设置，请选择此选项。

在所有根上执行分支操作

只有在具有多根项目的情况下，此选项才可用，即在单个项目中有多个Git存储库。

如果要将分支操作（如checkout ， merge等）同步应用于项目中的所有存储库，请选择此选项。

樱桃挑选自动提交

当您选择特定提交时，将显示“ 提交更改”对话框。如果选择了自动执行提取选项选项，则单击樱桃选择按钮将以静默方式提交所选提交 ，不显示“ 提交更改”对话框。

在选择推送到受保护分支的提交时，添加“从<hash>挑选的樱桃”后缀

如果要在从受保护分支中挑选提交时保留对原始提交的引用，请选择此选项。默认情况下，选择更改时不会添加后缀。

如果要提交CRLF行分隔符，请发出警告

选择此选项可启用LF和CRLF行分隔符的智能处理。PyCharm将分析您的配置，警告您是否要将CRLF提交到存储库，并建议将core.autocrlf设置更改为true或input具体取决于您的操作系统。

请注意，此设置不适用于已设置任何相关Git属性的文件 。在这种情况下，PyCharm假设您清楚地了解自己在做什么，并从分析中排除这些文件。

如果取消选择此选项，则必须使用“ 差异查看器”对话框手动修复行结尾问题。

在分离的HEAD中或在rebase期间提交时发出警告

如果希望PyCharm在从分离的头或rebase执行提交时显示警告，请选择此选项，因为这可能会导致问题和代码丢失。

在分支弹出窗口中标记具有传入/传出提交的分支

如果希望传入/传出标记出现在“ 分支”弹出窗口中的分支名称旁边，以指示存在尚未提取到本地存储库的未决传入提交，或者尚未推送的传出本地提交，请选择此选项。

更新方法

使用此列表选择将本地存储库与远程存储同步的策略。当push操作被拒绝时（如果启用了当前分支的推送时自动更新选项被启用），或者当您调用更新项目操作时，将使用所选方法。可以使用以下选项：

  • 合并：选择此选项以应用合并策略。结果与运行git fetch ; git merge的结果相同git fetch ; git merge或git pull --no-rebase 。

  • Rebase：选择此选项以应用rebase策略。结果与运行git fetch ; git rebase的结果相同git fetch ; git rebase或git pull --rebase 。

  • 分支默认值：选择此选项可以应用分支的默认命令。默认命令在.git/config配置文件的branch.<name>部分中指定。

如果拒绝当前分支的推送，则自动更新

如果希望在拒绝当前分支到其跟踪分支的push操作时自动更新当前分支，请选中此复选框。

如果取消选择此选项，则在推送 分支时，PyCharm将显示“ 推送已拒绝”对话框，因为您的本地存储库和远程存储未同步。

请注意以下事项：

  • 如果您之前从未见过“ 推送已拒绝”对话框，并且您最初启用了复选框，则PyCharm将通过merge操作以静默方式更新冲突的本地分支。

  • 如果您已经遇到Push Rejected对话框并选择了Remember the update method choice ...选项，PyCharm将保存您的最后选择（ rebase或merge ）并将其应用于静默更新冲突的本地分支。

    因此，要更改“记住”设置，请清除该复选框，访问“ 推送已拒绝”对话框，选择“ 自动更新，如果推送...已拒绝”选项，并调用另一个更新策略。

显示提交和推送的推送对话框

如果要在“ 提交更改”对话框中单击“ 提交并推送”后显示“ 推送”对话框 ，请选择此选项。否则，您的更改将自动推送到受影响的存储库。

仅在提交受保护分支时显示“推送”对话框

如果您只想在“ 提交更改”对话框中单击“ 提交并推送”时推送到受保护的分支，则只需要显示“ 推送更改”对话框，请选择此选项。否则，您的更改将自动推送到受影响的存储库。

受保护的分支

如果要禁用对某些分支强制推送更改的功能，请在此处列出（这是存储在.idea/vcs.xml的团队共享参数）。

您可以列出由分号分隔的多个分支，或者提供分支模式，因为输入被视为正则表达式列表。
