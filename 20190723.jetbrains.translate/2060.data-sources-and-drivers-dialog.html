    数据源设置

    数据源的“设置”窗格包括以下选项卡：“ 常规” ，“ 选项” ，“ SSH / SSL” ，“ 架构” ，“ 高级” 。

    一般

    “ 常规”选项卡包含您需要为数据库连接指定的设置。

    选项卡上的字段和控件集取决于在“ 连接类型”列表中选择的选项。

    项目

    描述

    连接类型

    要用于数据库连接的连接类型。

    • 仅限URL此选项通常用于直接编辑数据库连接URL。

      选择此选项时，只有以下字段可用： 用户 ， 密码和URL 。

      您应该在字段中右侧编辑URL。必要时，您的用户名和密码在相应的字段中指定，或者在URL中以适合您正在使用的JDBC驱动程序的格式指定。

    • 当使用任何其他的选项（选项是DBMS特异性），PyCharm形成数据库连接URL自动使用上述的URL字段的字段中的信息。在所有这些情况下，通常，您不需要编辑URL（尽管您可以，如果您愿意）。

    司机

    驱动程序列表中要用于连接的驱动程序 。

    文件

    数据库文件的路径。

    使用“ 打开”图标（ ）选择现有的数据库文件。

    保存图标（ ）创建数据库文件的副本并更改复制位置的路径。

    路径

    数据库文件或文件夹的路径。

    使用“ 打开”图标（ ）选择现有数据库文件或文件夹。

    主办

    主机名（域名）或数据库所在计算机的IP地址。如果数据库位于本地计算机上，请指定localhost或127.0.0.1 。

    如果使用SSH，则必须可以通过运行SSH代理的计算机中指定的域名或IP地址访问数据库主机。请参阅SSH / SSL选项卡 。

    港口

    数据库端口号。

    数据库

    目标数据库或模式的名称。

    用户

    数据库用户帐户的名称。

    密码

    数据库用户的密码。

    保存

    保存密码字段的设置。您可以选择以下选项来存储密码：

    • 从不 ：每次与数据库建立连接时都会提示输入密码。

    • 在重新启动之前 ：仅为当前的PyCharm运行保存密码。如果退出PyCharm并再次打开它，则必须再次提供密码。

    • 对于会话 ：仅为与数据库的当前连接会话保存密码（直到与其断开连接）。您可以通过按Disconnect图标终止连接（ ）或Ctrl + F2 。

    • 永远 ：密码保存在PyCharm存储中。下次打开PyCharm时无需提供密码。

    创建数据库

    向URL添加参数以创建数据库。此选项仅在某些数据库管理系统中可用（例如，在Apache Derby（嵌入式）中 ）。

    网址

    PyCharm用于连接数据库的URL。用于指定URL的用户界面因连接类型列表中选择的选项而异。

    测试连接

    验证数据库连接设置是否正确，PyCharm是否可以与数据库通信。要复制通信信息，请单击“ 复制到剪贴板”图标（ ）。

    选项

    “ 选项 ” 选项卡包含数据库连接的设置。

    选项卡上的字段和控件集取决于在“ 连接类型”列表中选择的选项。

    项目

    描述

    只读

    设置只读状态。选中该复选框以保护数据源免遭意外数据修改。

    如果选中该复选框，则无法在数据编辑器中修改数据。

    如果驱动程序不支持只读状态，则可以在控制台中进行数据修改。

    交易控制

    设置数据库事务的隔离级别以及提交事务的方式。

    • 当你提交你的本地修改数据库服务器自动将当前事务被自动提交。

    • 手动提交到数据库服务器的更改将累积在可以提交或回滚的事务中。

    切换架构

    定义PyCharm用于切换模式的模式。

    • 自动 ：自动或手动切换模式。

    • 手动 ：仅手动切换模式。

    • 禁用 ：不切换模式。

    单连接模式

    启用单连接模式。单连接模式意味着数据源和所有控制台使用同一个连接。此模式允许您查看数据库树中的临时对象，或在不同的控制台中使用相同的事务。有关控制台的更多信息，请阅读数据库控制台 。

    每N秒运行keep-alive查询

    运行保持活动查询以保持连接活动。您可以在驱动程序设置中为不支持的数据库定义自定义查询。

    N秒后自动断开连接

    在指定的秒数后断开与数据库的连接。

    单数据库模式

    在数据库树视图中，仅显示并启用您在连接设置中指定的数据库。

    当您连接到数据源时，PyCharm可以检索并显示数据源所具有的所有数据库。但在某些情况下（例如，使用PgBouncer的某些设置），您可以或仅被允许使用某个数据库。启用单数据库模式后，您只能在数据库树视图中看到您在连接设置中指定的数据库。

    考虑使用此设置，适用于PostgreSQL，Azure SQL数据库，Greenplum，Amazon Redshift。

    启动脚本

    每次建立连接时运行SQL查询。

    注意 ：如果清除了单一连接模式复选框，则每个新控制台都会创建一个新连接。

    自动同步

    自动同步数据库的实际状态。

    如果清除了自动同步复选框，则只有在单击“ 同步”图标时，“ 数据库”工具窗口中的数据源视图才会与数据库的实际状态同步 （ ）或按Ctrl + Alt + Y ）。

    如果启用此选项，则会自动更新数据源的视图：

    • 在“ 数据源和驱动程序”对话框中单击“ 确定”时。

    • 在与数据源关联的数据库控制台中运行DDL SQL语句时。

    注意 ：对整个数据库执行自动同步。此操作可能非常耗时。考虑为小型数据库启用此选项。对于大型数据库，请考虑手动同步状态，并且仅针对相应的数据库部分（例如单独的表）。

    加载源

    加载所选类型的模式的数据库对象的源代码。

    要同时更改多个数据源的此设置，请选择相应的数据源。右键单击选择并导航到“ 加载源”并选择一个选项。

    编辑过时的DDL时发出警告

    如果缓存的DDL与数据库中的实际DDL不同，则显示通知。

    每次打开源代码时，PyCharm都会显示在索引过程中加载的版本。但是，如果有人从另一个工作站更改源代码，则索引版本将过时。

    如果您的版本早于数据库中的版本，则会看到以下通知： Cached object differs from the object in the database 。

    使用JDBC元数据进行内省

    切换到基于JDBC的introspector。

    要检索有关数据库对象（DB元数据）的信息，PyCharm使用以下内省检查器：

    • 本机introspector（某些DBMS可能不可用）。本机introspector使用特定于DBMS的表和视图作为元数据源。它可以检索特定于DBMS的详细信息，并生成更精确的数据库对象图片。

    • 基于JDBC的introspector（适用于所有DBMS）。基于JDBC的introspector使用JDBC驱动程序提供的元数据。它只能检索有关数据库对象及其属性的标准信息。

    当本机introspector失败或不可用时，请考虑使用基于JDBC的intorspector。

    当您的数据库服务器版本早于PyCharm支持的最低版本时，本机introspector可能会失败。

    您可以尝试切换到基于JDBC的introspector，以修复从数据库中检索数据库结构信息的问题。例如，当数据库工具窗口中未显示数据库中存在的模式或模式级别下的数据库对象时。

    代码风格

    选择要用于模式的代码样式。有关代码样式自定义的更多信息，请阅读SQL的代码样式 。

    图案

    定义模式以选择必要的数据库和模式。要获取有关要使用的语法的信息，请将光标放在该字段中，然后按Ctrl + Q.

    • @ ：当前数据库或架构。

    • * ：每个数据库或架构。您可以在*:之后列出模式。

    请考虑以下示例：

    • *:* ：所有数据库中的所有模式。

    • @:* ：来自当前数据库的所有模式

    • @:@ ：只有当前架构

    • *:dbo|@:@|db1:s1,s2,s3 ：来自所有数据库的dbo模式，当前模式，来自db1数据库的模式s1,s2,s3 。

    对象过滤器

    限制数据库工具窗口中显示的每个数据源的表集和其他数据库对象

    <type>:[-]<pattern> ，其中：

    • <type>可能是聚合，整理，事件，fdw，ftable，mview，运算符，包，角色，例程，序列，同义词，表，用户，视图，vtable。

    • <pattern>是一个正则表达式 。要排除项目，请在- - （减号）前加上。

    示例： table:CUR_.*或table:-OLD_.*||routine:DO.*||view:V_.*

    VM选项

    数据库驱动程序的JVM选项。

    对于某些Oracle数据库版本（例如，Oracle版本9），当您和您的数据库服务器位于不同的时区时，可能会出现连接问题。您可以在VM Option字段中指定时区的时间偏移量（例如， -Duser.timezone=UTC+03:00 ）。或者，在“ 高级”选项卡上的表中将变量oracle.jdbc.timezoneAsRegion设置为false 。
