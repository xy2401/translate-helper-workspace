在Firefox中调试JavaScript

使用WebStorm，您可以使用Firefox远程调试功能在Firefox，版本36及更高版本中调试客户端JavaScript。但是，强烈建议您使用Chrome或Chrome系列的任何其他浏览器，原因如下：

  • WebStorm在FireFox中的调试会话期间不支持源映射 ，这会阻止压缩或机器生成的JavaScript的调试。

  • 在Firefox中进行调试时，您无法动态预览对HTML，CSS或JavaScript代码的更改 - 此功能（称为“ 实时编辑 ”）仅在Google Chrome中受支持。

您可以调试在WebStorm内置Web服务器或外部服务器上运行的应用程序。

在Firefox中启用远程调试

  1. 打开Firefox浏览器，单击在工具栏上，转到Web Developer |切换工具 。“ 开发工具”窗格将打开。

  2. 在窗格的工具栏上，单击 ，从列表中选择“设置” ，然后在“ 高级设置”下选择“ 启用浏览器镶边和附加调试”和“ 启用远程调试”复选框。

使用调试器服务器重新启动Firefox

  • 打开终端工具窗口（ Alt + F12 ）并在命令提示符下键入start firefox --start-debugger-server <port number> 。稍后您将在运行配置中指定此端口号。您可以设置任何端口号，但建议您使用6000或更高版本。

  • 对于61及更早版本，您无需重新启动Firefox 。

    只需转到工具|开发者|开发人员工具栏并在浏览器底部打开的控制台中键入listen <port number> 。

调试应用程序

  1. 根据需要在JavaScript代码中设置断点 。

  2. 创建Firefox Remote类型的调试配置：从主菜单中选择Run |编辑配置 ，单击在工具栏上，从列表中选择Firefox Remote 。

  3. 在“ 运行/调试配置：打开的Firefox远程”对话框中，在“ 主机”字段中键入localhost 。在“ 端口”字段中，键入在Firefox中启用远程调试时指定的端口。默认值为6000 。

  4. 在Firefox中打开您的应用程序。浏览器在代码执行后显示应用程序，也就是说，您设置的断点尚未生效。

  5. 在工具栏上的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击“确定” 。

  6. 在打开的对话框中，单击“ 确定”以允许来自浏览器的传入连接。

  7. 在浏览器中刷新应用程序页面：该页面显示直到第一个断点的代码执行结果。

  8. 在调试工具窗口中，像往常一样继续： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ， 查看实际的HTML DOM ，等等
