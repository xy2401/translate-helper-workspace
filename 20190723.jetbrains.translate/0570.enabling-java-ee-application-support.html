IntelliJ IDEA 2019.1帮助

启用Java EE应用程序支持

此功能仅在Ultimate版本中受支持。

本主题讨论打开JavaEE应用程序选项时可用的功能。

  • 先决条件
  • 功能概述
  • 打开JavaEE Application选项
  • 管理部署描述符
  • 管理应用程序工件

先决条件

对于JavaEE Application选项和相关功能，可以使用：

  • 您应该使用IntelliJ IDEA的ULTIMATE版本。（社区版中没有相应的功能。）

  • 必须启用Java EE：EJB，JPA，Servlets 插件 。（此插件与IDE捆绑在一起并默认启用。）

功能概述

当您打开JavaEE应用程序选项时，IntelliJ IDEA：

  • 创建META-INF/application.xml ，一个企业应用程序归档部署描述符 。

  • 创建一个Java EE Application构面，允许您指定application.xml和特定于application.xml程序服务器的部署描述符的位置（例如glassfish-application.xml ， jboss-app.xml ）。

  • 创建爆炸EAR 工件配置。

  • 在“ 项目结构”对话框中提供各种快速修复，例如，将application.xml与EAR工件的结构同步。

  • 使JavaEE：App工具窗口可用。

如果在创建项目或模块时打开JavaEE Application选项并指定应用程序服务器，IntelliJ IDEA还会为该服务器创建运行/调试配置 。

打开JavaEE Application选项

您可以打开JavaEE Application选项：

  • 创建项目或模块时（ 文件|新建|项目或文件|新建|模块 ）。在New Project或New Module向导的第一页上，选择Java Enterprise ，然后选择Additional Libraries and Frameworks下的JavaEE Application复选框。

  • 对于现有模块。在“ 项目”工具窗口（“ 查看”|“工具Windows”|“项目” ）中，右键单击模块文件夹，然后选择“ 添加框架支持” 。然后在打开的对话框中选择JavaEE Application复选框。

管理部署描述符

您可以在“ 项目结构”对话框中管理application.xml和特定于服务器的部署描述符文件：

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 在最左侧的窗格中，选择“ 模块”或“ 构面” 。

  3. 在右侧窗格中，选择javaEEApplication 。

  4. 在对话框右侧打开的页面上 ，形成应用程序的部署描述符列表。

管理应用程序工件

要将应用程序部署到服务器，您需要一个应用程序工件 。对于Java EE应用程序，IntelliJ IDEA提供以下工件格式：

  • JavaEE应用程序：爆炸。 这是一个解压缩的企业应用程序归档（ EAR ），一种可以部署到应用程序服务器上的目录结构。

  • JavaEE应用程序：存档。 这显然是一个EAR文件。

要管理工件配置，请使用“ 项目结构”对话框（“ 文件”|“项目结构”|“工件” ）。

另请参阅使用工件 。
