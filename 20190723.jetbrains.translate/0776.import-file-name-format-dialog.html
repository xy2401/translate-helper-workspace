导入文件对话框

将分隔符分隔值（例如CSV，TSV）导入数据库时， 将打开此对话框。

在左侧部分中，指定如何将分隔符分隔值转换为表格格式。在右侧部分中，指定数据库中目标表的设置。

  • 转换设置
  • 表名，结构和数据映射
  • 数据和DDL预览
  • 编码，将错误写入文件并将不可转换的值插入为null

转换设置

处理转换设置时，请使用表格设置下方对话框右侧的表格预览。

项目

描述

格式

选择文件格式并检查表格预览。如果尚未达到预期效果，请调整设置。

如果您已更改设置并想要保存更改，请单击此图标并选择以下选项之一：

  • 保存更改。 设置以“相同名称”保存，而不创建新格式。（事实上，格式是一组命名的设置。）

  • 另存为。 设置将“以不同的名称”保存：创建新格式，您可以指定该新格式的名称。

值分隔符

选择或键入用于分隔单个值的字符。

行分隔符

选择或键入应被视为行分隔符的字符。

空值文本

如果单元格包含null （未知值），则用作值的文本。

添加行前缀/后缀

行前缀和后缀是字符序列，除了行分隔符之外还指示行的开头和结尾。

如有必要，请单击该链接，并在显示的字段中指定行前缀和后缀。

行情

报价范围内的每一行都是报价模式（参见报价值 ）。报价模式包括：

  • 左引号字符，在值之前插入的字符。

  • 右引号字符，插入一个值后的字符;通常，与左引号字符相同。

  • 引号字符是值的一部分时的转义方法或字符。例如转义：重复意味着如果引号字符出现在某个值中，则会加倍。（您可以改为指定自己的转义字符。）

如果存在多个模式，则使用第一个模式。

使用 ， ， 和创建，删除和重新排序模式。

要开始编辑现有模式，只需单击感兴趣的模式即可。

报价值

指定应在哪些情况下引用值（即括在引号字符中）。

  • 需要的时候。 仅当值包含值和/或行分隔符时才引用该值。

  • 总是。 任何值都以其文本表示形式引用。

修剪空格

如果未选中此复选框，则值分隔符之前和之后的Unicode空白字符将被视为相应值的一部分。如果选中此复选框，则会忽略或删除相应的空白字符。

第一行是标题

如果选中此复选框，则第一行将被视为包含列名称。在标题格式下显示的设置与上面的设置具有相同的含义，但应用于第一行。

第一列是标题

如果选中此复选框，则会将第一列视为包含行名称。

列/键等

  • 要删除列，请选择相应的行并单击 。

描述

编码

将错误写入文件

如果选中该复选框，则导入期间发生的错误将写入指定的文本文件。对于每个错误，指示它出现的位置。

将不可转换的值插入为null

如果选中该复选框，则会在表中插入NULL以获取无法转换的数据。
