木偶

IntelliJ IDEA支持Puppet语言语法，允许您使用Puppet模块和环境。这些功能是通过使用Puppet Support插件实现的。

请参阅管理插件以了解如何启用或禁用插件。

主要支持的功能是：

  • 支持Puppet语言语法，包括Puppet 3，Puppet 4+和EPP模板语言 。

  • 根据Puppet样式指南突出显示和格式化代码的语法。

  • 代码洞察功能，例如完成 ， 重命名重构 ， 导航支持 ， 查找用法等。

先决条件

使用Puppet模块和环境需要以下gem：

  • 木偶宝石 。

  • （可选）需要librarian-puppet gem来安装 Puppetfile或metadata.json指定的依赖项 。

创建一个Puppet模块

创建一个新的Puppet模块

  1. 选择文件|主菜单上的新项目...

  2. 在New Project对话框中，选择左侧的Puppet Module 。

  3. 然后，选择所需的Ruby SDK。如果未在所选SDK中安装puppet gem ，IntelliJ IDEA将建议您执行此操作。

  4. 最后，指定作者用户名 ， 模块名称 ，然后单击“ 下一步” 。在下一页上，指定项目名称，位置，然后单击“ 完成” 。创建的项目包含Puppet模块结构。

    您可以从相应的文档中学习如何创建Puppet模块。

打开Puppet模块和环境

IntelliJ IDEA自动识别打开的 Puppet 模块或环境 （例如，通过metadata.json或environment.conf文件的可用性），并使用项目树中的相应图标标记它们。

打开模块或环境后，您可以安装其依赖项 。

安装依赖项

IntelliJ IDEA允许您安装以下文件中指定的模块/环境依赖项：

  • 如果模块包含.fixtures.yml文件，IntelliJ IDEA将使用rake任务将依赖项安装到spec / fixtures / modules目录中。在这种情况下，IDE不会检查下一个源。

  • 如果有Puppetfile ，IntelliJ IDEA会将使用librarian-puppet的依赖项安装到.dependencies目录（或环境模块 ）中。

  • 如果项目包含metadata.json ，IntelliJ IDEA使用librarian-puppet以与Puppetfile相同的方式安装依赖Puppetfile 。

要安装模块/环境依赖项，请在“项目”工具窗口中右键单击该模块，然后选择“ 为模块<名称> 安装依赖项”或“ 为环境<名称>安装依赖项” 。

如果项目的SDK没有安装librarian-puppet gem，IntelliJ IDEA会建议你这样做。

配置与Puppet相关的设置

设置/首选项对话框（ Ctrl + Alt + S ）允许您配置与Puppet相关的以下设置：

  • 语言和框架| Puppet页面允许您配置Puppet语言版本和librarian-puppet的路径。

  • 编辑|代码风格| Puppet包含Puppet语言的代码样式设置。

  • 编辑|检查允许您配置检查设置。

  • 在编辑|在实时模板页面中，您可以使用与其他语言相同的方式为Puppet语言创建自定义实时模板 。
