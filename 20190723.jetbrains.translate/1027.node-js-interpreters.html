Node.js解释器对话框

此功能仅在Ultimate版本中受支持。

档案|设置|语言和框架|适用于Windows和Linux的Node.js和NPM
IntelliJ IDEA |偏好|语言和框架| Node.js和NPM用于macOS

只有在安装并启用了Node.js Remote Interpreter插件时，才能配置远程Node.js解释器 。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

单击时将打开该对话框在“ 运行/调试配置：Node.js”对话框中的Node Interpreter字段旁边或Node.js和NPM页面上。

使用此对话框在Windows子系统Linux上配置本地或远程 Node.js解释器或Node.js.

术语local Node.js interpreter表示计算机上的Node.js安装。术语remote Node.js interpreter表示远程主机或虚拟环境中的Node.js安装。

提示

描述

Node.js口译员

该列表显示了所有已配置的Node.js解释器，包括本地和远程解释器。对于本地解释器，IntelliJ IDEA还显示Node.js可执行文件的路径以及相关的npm可执行文件，请参阅npm和Yarn 。

加

单击此按钮可将新的Node.js解释器添加到列表中。从下拉菜单中，选择Add Local ， Add Remote或Add WSL 。根据您的选择，选择相关的本地Node.js安装，或在Configure Node.js Remote Interpreter对话框中配置远程解释器 ，或者在Add WSL Node中指定您正在使用的Linux发行版和Node.js的路径解释器对话框。

请注意，只有从“运行/调试配置：Node.js”打开对话框时，“ 添加远程”才可用。

删除

单击此按钮可从列表中删除选定的解释器。

编辑

单击此按钮可创建一个新的解释器，其中包含从所选设置复制的设置。

节点解释器

此只读字段显示所选本地解释器的路径。

Npm包

在此字段中，选择要在当前项目中使用的包管理器（npm或Yarn），有关详细信息，请参阅为项目配置包管理器 。
