运行配置

运行XSLT脚本就像打开编辑器上下文菜单并创建永久运行配置或只是选择运行以立即运行选定的XSLT脚本一样简单。

创建运行配置

可以通过Run |创建永久运行配置在主菜单上编辑配置 。您将找到另一个名为XSLT的选项卡，以便为XSLT脚本添加运行配置。

也可以通过上下文菜单中的Create“<name>”操作创建运行配置。

在这种情况下，配置的名称将从样式表的文件名中获取，其路径将已填充到XSLT脚本文件文本字段中。如果样式表定义了参数，那么这些参数将填充到带有空值的Parameters表中。以蓝色突出显示的参数尚未分配值，并且在执行期间不会传递给样式表，因此如果不应将任何内容传递给样式表，则不需要删除这些自动创建的参数值。

运行配置设置

输入

XSLT运行配置具有可以调整的各种设置。最重要的是XSLT脚本文件的位置和应该转换的XML输入文件。这些是必需的，指定的文件必须存在，否则无法执行配置。

XML输入文件组合框通过文件关联功能列出与所选样式表关联的所有XML文件。

产量

关于如何处理脚本输出有三种不同的选择。第一个是在默认控制台中显示 。选择此选项后，输出将显示在正常运行控制台中，以及来自XSLT转换器的任何警告和错误消息，以及脚本生成的消息，例如xsl:message 。

默认情况下，选择Show in extra console选项卡 ，它将在名为XSLT Output的额外选项卡中显示输出 。此选项可以根据IntelliJ IDEA中提供的不同文件类型突出显示生成的输出。但这是一种实验性功能，因此可以通过选择“ 禁用”选项完全关闭它。然后输出将写入临时文件，该文件由普通日志查看器显示。

最后一个选项Save to file可用于直接将输出写入文件。该字段不能为空，可以引用任何现有或不存在的文件。执行后选中“ 在编辑器中打开文件 ”选项，在脚本正常结束后在IntelliJ IDEA中打开文件。可以检查Web浏览器中的打开文件选项，以在执行完成后在配置的Web浏览器中打开生成的文件。

指定的输出文件将被覆盖而无需确认。

参数

Parameters表用于指定应传递给脚本的参数名称及其值。按“ 添加”和“ 删除”按钮修改列表。新添加的参数默认情况下不会分配任何值，因此如果未编辑该值，则不会将其传递给脚本。

高级选项

此选项卡允许您控制常规运行配置不需要的一些选项。

智能错误处理

清除此复选框可在执行期间发生错误时查看完整的错误消息，包括其完整的堆栈跟踪。选中该复选框后，将禁止这些堆栈跟踪，并且仅在控制台中显示有关错误的相关信息。

VM参数

允许您将任意VM参数传递给用于运行XSLT脚本的VM。

工作目录

要使用的工作目录。保留为空时，工作目录将是XSLT脚本文件所在的目录。

Classpath和JDK

允许您选择运行脚本的环境。默认设置是XSLT脚本文件所属的模块。选项From模块还将包括所选模块的完整类路径。如果脚本使用自定义XSLT扩展函数，则可能需要这样做 。

使用JDK选项允许您选择JDK，而不将任何与模块或项目相关的内容包含在类路径中。显式选择特定的JDK以测试脚本可能很有用。
