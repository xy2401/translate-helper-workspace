文件和代码模板

文件模板是创建新文件时要生成的默认内容的规范。根据您创建的文件类型，模板提供了预期在该类型的所有文件中的初始代码和格式（根据行业标准，您的公司政策或其他原因）。

WebStorm为所有支持的文件类型提供预定义模板，这些模板是在创建新文件时建议的。

文件模板在编辑器上管理WebStorm设置的文件和代码模板页面（ Ctrl + Alt + S ）此设置页面可以在两个范围之一中配置：

  • 默认范围控制与整个工作区相关的模板。它们存储在fileTemplates下的IDE配置目录中 。

  • 项目范围基于每个项目控制模板。它们存储在.idea / fileTemplates下的项目文件夹中。这些模板可以在团队成员之间共享。

默认情况下，模板列表仅包含WebStorm提供的预定义模板。其中一些是内部的 ，这意味着它们无法删除或重命名。内部模板的名称以粗体显示。您修改的模板名称以及手动创建的自定义模板以蓝色显示。

WebStorm使用几种类型的模板，分为以下选项卡：

  • “ 文件”选项卡包含可用于创建新文件的文件模板。

  • “ 包含”选项卡包含可以插入到文件模板中的可重用内容的模板（例如，作为标题）。

  • “ 代码”选项卡包含WebStorm用于生成各种构造的代码片段（片段）的内部模板。您可以在此选项卡上编辑可用的代码段，但无法创建新的代码段。

以下过程描述了如何创建文件模板。类似的过程可用于创建包含模板 。

创建新文件模板

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 编辑器”|文件和代码模板 。

  2. 在“ 文件”选项卡上，单击并指定模板的名称，文件扩展名和正文。

  3. 应用更改并关闭对话框。

复制现有文件模板

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 编辑器”|文件和代码模板 。

  2. 在“ 文件”选项卡上，单击并根据需要修改模板的名称，文件扩展名和正文。

  3. 应用更改并关闭对话框。

将文件另存为模板

  1. 在编辑器中打开一个文件。

  2. 选择工具|从菜单中将文件另存为模板 。

  3. 在打开的“ 将文件另存为模板”对话框中，指定新模板名称并根据需要编辑正文。

  4. 应用更改并关闭对话框。

从VTL参考指南中了解更多信息。

文件和代码模板使用Velocity Template Language （VTL）编写，它允许您使用以下结构：

  • 固定文本（标记，代码，注释等），按原样呈现。

  • 变量 ，由其值替换。

  • 各种指令，包括＃ #set ， #if #set ， #if等。

以下示例显示了在WebStorm中创建JavaScript类的默认模板：

/ ** * $ { USER } on $ { DATE }创建 * /

在此模板中， ${USER}和${DATE}是模板变量。

创建新的JavaScript文件时，此模板将生成一个文件，其内容类似于以下内容：

/ ** *由John创建。史密斯于6/7/11 * /
最后修改日期：2019年3月25日
