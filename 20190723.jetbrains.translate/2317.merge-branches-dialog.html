合并分支对话框

VCS | Git |合并更改

使用此对话框指定在本地Git存储库中合并分支的参数。

项目

描述

Git Root

从此下拉列表中，选择要在其中合并分支的本地存储库的路径。

现任分公司

此只读字段显示当前在所选本地存储库中检出的分支的名称。这是目标分支，来自所选源分支的更改将应用于它。

该字段的内容取决于Git Root下拉列表中的选择。

分支合并

使用此列表框指定将从中应用更改的源分支到目标分支。

该列表仅显示包含适用提交的分支。适用的提交是在与目标分支分离的分支之后进行的提交。

战略

从此下拉列表中，选择合并策略 。可用选项包括：

  • 默认

  • 解析 - 如果需要解析两个HEAD，请选择此选项，其中一个是当前分支，另一个HEAD是您在“ 分支到合并”列表中选择的分支 。选择此选项后，将应用3向合并算法。

  • 递归 - 用于将当前分支与一个分支合并的默认合并策略。如果需要通过应用3向合并算法解析两个HEAD并且有多个可用于3向合并的共同祖先，请选择此选项。

  • 八达通 - 用于将当前分支与多个分支合并的默认合并策略。

    不执行需要手动解决冲突的合并。

  • 我们 - 如果您需要解决多个HEAD，请选择此选项。合并的结果始终是当前分支的HEAD。

  • 子树 - 修改后的递归策略。

在“ 分支到合并”列表框中选择两个或多个源分支时，只有Octopus和Ours选项可用。

没有提交

如果需要检查并在必要时在提交结果之前调整合并结果，请选中此复选框。合并已执行但未自动提交，就好像失败一样。

没有快进

选中此复选框以生成合并提交，即使合并已解析为快进 。

壁球提交

选中此复选框可在当前分支之上创建单个提交，而不是合并一个或多个其他分支。生成工作树和索引状态就好像发生了真正的合并，但是没有执行提交并且不移动HEAD。

添加日志信息

选中此复选框可使PyCharm除分支名称外还填充一条日志消息，其中包含正在合并的实际提交中的单行描述。

提交消息

在此文本框中，提供提交的说明。

仅当未选中“ 无提交”复选框时，该文本框才可用。

合并

单击此按钮可根据定义的设置启动合并本地存储库中的指定分支。
