模块依赖

模块可以依赖于SDK，JAR文件（库）或项目中的其他模块。编译或运行代码时，模块依赖项列表用于构成编译器或JVM的类路径。

添加新的依赖项

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）并单击模块|依赖性 。

  2. 点击 （ Alt + Insert ）并选择依赖类型：

    • JAR或目录 ：从计算机上的文件中选择Java归档或目录。

    • 库 ：选择现有库或创建新库，然后将其添加到依赖项列表中。

    • 模块依赖关系 ：选择项目中的另一个模块。

删除依赖项

  • 要删除依赖项，请选择它，然后单击或按Alt + Delete 。

在删除依赖项之前，请确保它未在项目的其他模块中使用。为此，请选择必要的依赖关系，然后按Alt + F7 。您还可以使用上下文菜单中的“ 查找用法”选项。

指定依赖范围

通过指定依赖关系范围，您可以控制应在何种构建阶段使用依赖关系。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）并单击模块|依赖性 。

  2. 从Scope列的列表中选择所需的范围 ：

    • 编译 ：构建，测试和运行项目（默认范围）所必需的。

    • 测试 ：编译和运行单元测试所需。

    • 运行时 ：用于测试和运行项目的类路径的艺术。

    • 提供 ：用于构建和测试项目。

IntelliJ IDEA以不同于其他构建工具（例如，Gradle和Maven）的方式处理测试源的依赖项。

如果您的模块（比如模块A）依赖于另一个模块（模块B），则IntelliJ IDEA假定A中的测试源不仅依赖于B中的源，还依赖于其自己的测试源。因此，B的测试源也包含在相应的类路径中。

分析依赖关系

如果要检查项目中是否仍存在依赖项，并找到其确切用法，可以运行依赖项分析：

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）并单击模块|依赖性 。

  2. 右键单击必要的依赖关系，然后选择“ 分析此依赖关系” 。

您可以逐个分析多个依赖项，而无需关闭对话框。每个分析的结果将在Dependency Viewer工具窗口的单独选项卡中打开。分析所有必需的依赖项后，可以关闭“ 项目结构”对话框并查看结果。

如果IntelliJ IDEA在项目中找不到依赖项用法，系统将提示您删除此依赖项。

排序依赖项

依赖关系的顺序很重要，因为IntelliJ IDEA将按照列表中指定的顺序处理它们。

在编译期间，依赖项的顺序定义了编译器（javac）查找类以解析相应引用的顺序。在运行时，此列表定义JVM搜索类的顺序。

您可以按名称和范围对依赖项进行排序。你也可以使用和在列表中上下移动项目。
