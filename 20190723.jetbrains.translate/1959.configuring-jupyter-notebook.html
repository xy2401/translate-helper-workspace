配置Jupyter Notebook

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

在PyCharm中，您可以使用以下命令执行代码单元：

  • 托管服务器 - 由PyCharm为当前项目自动启动的Jupyter服务器。当你关闭PyCharm时它将被终止。

  • 已配置的服务器 - 您通过指定其URL和令牌连接到的任何本地或远程Jupyter服务器。

启动Jupyter服务器

要运行Jupyter服务器，只需执行任何代码单元 。当您启动单元格执行时，PyCharm使用任何可用端口在本地主机上启动Jupyter服务器（默认情况下，它是8888端口）。您可以切换到Jupyter Server工具窗口以预览服务器的配置详细信息。

启动服务器后， 它将在Jupyter工具栏的Jupyter Server小部件中显示为受管服务器。您还可以在内核列表中看到自动创建的服务器内核。该内核基于PyCharm项目解释器 。

停止Jupyter服务器

要停止任何正在运行的服务器，请切换到Jupyter Server工具窗口并单击图标或在Jupyter快速列表中选择相应的命令。在工具窗口中预览状态。

关闭服务器后，当前会话终止。下次使用时启动服务器图标或Jupyter快速列表 ，所有以前的会话和所有笔记本的执行结果将丢失。

您可以在特定的Jupyter服务器上执行笔记本代码单元。

连接到已配置的Jupyter服务器

  1. 单击Jupyter Server小部件，然后选择Configure Jupyter Server 。

  2. 在Jupyter Server对话框中，选择Configured Server并指定服务器的路径，包括URL和令牌。

    单击“ 应用”和“ 确定

  3. 由于仍保留受管服务器，因此您需要显式切换到已配置的服务器。单击Jupyter Server小部件，然后选择切换到当前的Jupyter Server 。

  4. 到目前为止，服务器已配置但尚未运行。

    执行任何代码单元以运行服务器。

  5. 如果看到以下警告，则新配置的服务器的内核与当前项目解释程序不匹配。例如，它可能具有不同的已安装软件包集。

    您可以通过单击相应的链接在配置的服务器上将项目解释程序注册为内核。之后，您可以在配置的服务器的内核列表中看到它。

您可以随时开始再次使用受管服务器。单击Jupyter Server小部件并选择Configure Jupyter Server ，然后在Jupyter Server对话框中选择Managed Server 。
