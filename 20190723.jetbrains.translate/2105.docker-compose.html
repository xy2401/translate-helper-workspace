以下内容仅在安装并启用Docker Integration Plugin时有效！

先决条件

确保满足以下先决条件：

  • 已安装Docker Compose，如Docker Compose页面所述。

  • 在开始使用Docker Compose之前，请确保已启用Docker Integration插件。该插件与PyCharm捆绑在一起，默认情况下处于激活状态。如果未激活插件 ，请在“ 管理插件 ”中所述的“设置/首选项”对话框的“ 插件”页面上启用它。

请注意，PyCharm支持高达3.x的docker-compose配置版本。

重要的提示

在Docker Compose配置文件中声明的所选服务作为Python解释器受支持。基于此解释器的Python运行配置的操作类似于docker-compose up命令，另外还将项目源映射到所选的服务容器中。

该功能适用于Linux和本地安装的Docker和macOS，它不会在运行在Windows平台上的PyCharm中显示。在macOS上，Docker Compose配置文件必须位于默认的共享文件夹/Users/ 。

使用Docker Compose

在Settings / Preferences对话框（ Ctrl + Alt + S ）框中，展开节点Build，Execution，Deployment ，然后在Docker页面中，添加所需的Docker，然后指定Docker Compose可执行文件 。

因此， Docker Compose选项出现在Configure Remote Python Interpreter对话框中。

PyCharm允许您指定多个docker-compose配置文件，并以与docker-compose相同的方式处理它们。
