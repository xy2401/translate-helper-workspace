按对话框（Mercurial，Git）

VCS | Git |推
VCS | Mercurial |推

此对话框可用于以下版本控制系统：

  • 混帐

  • 水银

该对话框由两个窗格（“ 存储库”窗格和“ 提交详细信息”窗格）和“ 推送控件”区域组成。

  • 存储库窗格
  • 提交详细信息窗格
  • 推控制

存储库窗格

左侧窗格显示Git和/或Mercurial存储库的列表（以及将哪个本地分支/活动书签推送到哪个远程分支），以及每个存储库中执行的提交列表。

  • 将鼠标悬停在提交上：显示工具提示，显示提交编号，日期和时间，作者和提交消息。如果提交的作者与当前用户不同，则此提交将标记为星号。

  • 选中要推送的每个存储库旁边的复选框。

    如果您有一个多根项目，其中不同步控制存储库，则默认情况下仅选择当前存储库（或在项目视图中选择多个存储库）。有关如何启用/禁用同步存储库控制的详细信息，请参阅以下资源：

    • 对于Git ： 版本控制设置：Git

    • for Mercurial ： 版本控制设置：Mercurial

  • 要修改要推送的目标分支（以蓝色突出显示），请单击它。标签变为文本字段，您可以在其中指定目标分支。您也可以通过选择要修改的分支并按Enter键切换到编辑模式。

  • 您也可以使用与远程分支相同的方式编辑远程存储库（如果有多个存储库）。请注意，如果未指定遥控器，则将显示“ 定义远程”链接而不是远程名称。单击它以添加遥控器。

  • 如果存储库中没有遥控器，则会显示“ 定义远程”链接。单击此链接并在打开的对话框中指定远程名称和URL。

提交详细信息窗格

右侧窗格显示所选提交中包含的文件。如果在左窗格中选择多个分支，则将显示所有相应的提交。

此区域中的工具栏提供以下选项：

项目

工具提示和快捷方式

描述

显示差异

按Ctrl + d

单击此按钮可打开“ 文件差异查看器”对话框，其中显示所选文件的已提交版本与先前版本之间的差异。

编辑来源

F4

单击此按钮可在编辑器中打开所选文件。

按目录分组

按Ctrl + P

单击此按钮可在平面视图和目录视图之间切换。

全部折叠/全部展开

Ctrl + NumPad - / Ctrl + NumPad Plus

单击这些按钮可折叠/展开目录树中的所有节点。如果选择了平面视图，则这些按钮不可用。

推控制

此区域中的控件允许您选择以下推送选项：

项目

描述

推送标签

此选项仅在您使用Git时可用。

默认情况下，执行push操作时，不会将标记发送到远程存储库。如果要使用提交推送标记，请选择此选项。

  • 如果要推送所有标记，请选择“ 全部” ，包括不属于您要推送的所选分支的标记（相当于push --tags ）。

  • 如果您只想推送属于您要推送的所选分支的标记，请选择当前分支 （相当于自Git 1.8.3以来可用的push --follow-tags ）。

导出活动书签

此选项仅在您使用Mercurial时可用。

默认情况下，执行push操作时，书签不会发送到远程存储库。如果要使用提交推送活动书签，请选择此选项。

推

单击此按钮并从下拉菜单中选择要执行的操作： push或push --force 。

有关如何使用push --force命令及其可能有用的说明，请参阅：

  • 对于Git：将更改推送到远程存储库

  • 对于Mercurial： 使用强制推动
