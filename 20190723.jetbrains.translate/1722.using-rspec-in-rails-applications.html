在Rails应用程序中使用RSpec

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

在Rails应用程序中使用RSpec需要将rspec-rails gem添加到您的项目中。因此，依赖的rspec gem会自动添加到项目中。添加rspec-rails gem后，相应的生成器rspec:install显示在可用生成器列表中。

IntelliJ IDEA支持rspec-rails gem版本2.12.0及更高版本。

在Rails应用程序中启用RSpec支持时，IntelliJ IDEA提供：

  • 生成RSpec测试基础设施。

  • RSpec特定的代码完成：

  • 使用Zeus服务器 。

  • RSpec运行/调试配置

根据您的特定Rails版本，可以使用不同的工作流程。如果您在使用Rails 3.0和更高版本，建议使用捆绑 ，并指定rspec-rails中的Gemfile宝石。因此，IntelliJ IDEA建议以下工作流程：

要使用rspec，请遵循以下常规步骤

  1. 确保安装了所需的gem。

  2. 在Gemfile中，添加以下行：

    宝石'rspec-rails'
  3. 生成RSpec测试基础架构。为此，请按Alt + Insert ，在弹出菜单中选择Run Rails Generator ，开始键入rspec生成器名称，然后从列表中选择rspec:install ：

    在“ Setup RSpec Support”对话框中，指定生成器选项。执行相应的脚本，并在“运行”工具窗口的控制台选项卡中显示其输出。

  4. 以与其他Rails应用程序元素相同的方式生成测试模板。

  5. 运行或调试您的规范。

    如果您正在使用捆绑器，请确保在RSpec运行/调试配置中选中“在捆绑包的上下文中运行脚本 ”复选框。
    请注意，您可以在脚本中的文件夹，特定测试脚本或单个示例中执行所有测试。

    • 要在文件夹中执行所有测试，请在“项目”工具窗口中右键单击包含规范的文件夹，然后在上下文菜单中选择“运行：<文件夹名称>中的所有规范” 。

    • 要执行测试脚本，请右键单击“项目”工具窗口中的规范，或在编辑器中打开此规范，然后右键单击各个示例之外的某个位置。然后选择Run <spec name> 。在上下文菜单上。

    • 要执行单个示例，请在编辑器中打开所需的规范，右键单击要执行的示例，然后选择“运行<规范名称> <示例名称>” 。
