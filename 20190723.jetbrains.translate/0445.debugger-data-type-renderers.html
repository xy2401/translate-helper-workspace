Java类型渲染器

IntelliJ IDEA允许您在逐个类的基础上指定调试器中不同对象的显示方式。您可以指定要显示的表达式，而不是依赖于对象的String表示。

例如，如果一个对象代表一个用户，您可能希望它由登录名表示;或者，对于缓存条目对象，其内容可能是适当的。IntelliJ IDEA将这些称为类型渲染器 。

支持所有对象类型（包括基元类型和数组）。

如果未定义渲染方案，则此对话框不显示任何控件。要开始使用渲染器，请单击 。

项目

描述

单击此图标可将新的渲染方案添加到列表中。

单击此图标可从列表中删除所选方案。

单击此图标可创建所选方案的副本。

单击这些图标可在列表中向上或向下移动选定项目一行。

请注意，顺序确定在类继承产生歧义的情况下使用哪个渲染器。

渲染器名称

指定新渲染器的名称，或编辑现有渲染器名称。

将渲染器应用于类型的对象（完全限定名称）

指定将由此渲染器表示的对象类型。输入完全限定的对象名称，或单击“ 浏览” 并从“ 渲染器参考类型”对话框的列表中选择所需的类型。

渲染节点时

此选项确定在折叠节点时对象在调试器中的显示方式：

  • 显示类型和对象ID ：如果清除，则显示的类型没有类信息或id。

  • 使用默认渲染器 ：选择此选项以默认方式显示节点。

  • 使用以下表达式 ：输入要用于标识对象的Java表达式。您可以使用对象属性，常量甚至字符串数学作为渲染器的一部分。

    请注意，您可以在定义表达式时使用代码完成（ Ctrl + Space ）。

    所有方法调用和成员变量访问都与您正在呈现的对象相关。使用this可引用渲染器应用的实例。

    • 在渲染器中使用沉重的表达式可能会减慢视图中的数据渲染速度。

    • 由于可能存在副作用，应谨慎使用方法调用。

扩展节点时

此选项确定在展开节点时对象在调试器中的显示方式。

通常，在调试器中展开节点会列出对象的成员变量（使用适合相应对象类型的渲染器）。此选项允许您覆盖此行为并选择要显示的单个表达式或一系列表达式。您可以使用它来限制显示的信息量，或者更准确地显示信息的显示方式。

  • 使用默认渲染器 ：选择此选项可以默认方式显示节点子节点。

  • 使用以下表达式 ：输入要用于标识对象的Java表达式。

    测试节点是否可以展开（可选） ：输入布尔表达式。如果为true ，则渲染器将显示已定义对象的可扩展节点。否则，不显示任何节点。

  • 使用表达式列表 ：创建要计算的单独表达式列表，并将其显示为节点子项。使用：

    （ Alt + Insert ）创建一个新表达式。

    （ Alt + Delete ）从列表中删除选定的表达式。

    （ Alt + Up ）将所选表达式在列表中向上移动一行。

    （ Alt + Down ）将所选表达式在列表中向下移动一行。

如果选中渲染器旁边的“按需”列中的复选框，则将根据需要对此表达式进行评估。当您需要在变量 ， 手表或其他视图中对其进行评估时，只需单击此表达式，而不是自动评估它。

您可以在定义表达式时使用代码完成（ Ctrl + Space ）。

附加默认子项

选中此复选框可将默认子项添加到表达式列表中。仅当选中“ 使用表达式列表 ”复选框时，此复选框才可用。
