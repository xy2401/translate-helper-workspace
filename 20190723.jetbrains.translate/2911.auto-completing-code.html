自动完成代码和路径

代码完成是WebStorm编辑器的基本功能之一。为了向您显示完成建议，WebStorm会使用源代码浏览项目文件，并将其中定义的类，方法，函数和变量添加到特殊的内部索引中。来自JSDoc注释， TypeScript类型定义等的附加信息也可以显着提高完成率。

本节介绍了各种上下文感知代码完成技术，可以加快编码过程。

完成也适用于来自第三方代码的符号。在大多数情况下，您只需要将所需的文件添加到项目中，有关详细信息，请参阅配置JavaScript库 。

基本完成

基本代码完成可帮助您完成可见范围内的类，方法，字段和关键字的名称。当您调用代码完成时，WebStorm会分析上下文并建议可从当前插入符号位置访问的选项（建议还包括实时模板 ）。

调用基本完成

  1. 开始输入名称。默认情况下，WebStorm会在您键入时自动显示代码完成弹出窗口。

    或者，按Ctrl + Space或选择Code |完成|主菜单中的基本菜单。

  2. 要获得更多建议，请再次按Ctrl + Space （或按Ctrl + Alt + Space ）。

  3. 要缩小建议列表，请键入预期名称或关键字的任何部分（即使接受中间某处的字符），也可以在点分隔符后调用代码完成。WebStorm显示包含任何位置的键入字符的建议。

通过以下方式之一接受列表中的建议

  • 按Enter键或双击相关列表项将其插入插入符号的左侧。

  • 按Tab键从插入符号右侧替换字符。

  • 使用Ctrl + Shift + Enter使当前代码构造在语法上正确（平衡括号，添加缺少的大括号和分号，等等）。

  • 使用特定语言和上下文相关键来插入所选的完成建议，请参阅配置完成设置以获取详细信息。

查看所选建议的参考

  • 要快速查找文档，请在建议列表中选择一个项目，然后按Ctrl + Q.WebStorm在“ 文档”弹出窗口中向您显示快速文档。

    要在具有更多控件的工具窗口中查看文档，请固定文档弹出窗口。要在“ 文档”弹出窗口和“ 文档”工具窗口之间切换，请按顺序按Ctrl + Q.

  • 要查看所选建议的定义 ，请按Ctrl + Shift + I.

查看代码层次结构

  • 在建议列表中选择一个条目，然后按以下快捷方式之一：

    • 按Ctrl + H查看类型层次结构

    • Ctrl + Alt + H可查看视图调用层次结构。

    • 按Ctrl + Shift + H查看方法层次结构。

基本完成也适用于非英语键盘布局。

对于CamelCase或snake_case名称，WebStorm仅自动识别并匹配首字母。

或者，按照配置完成设置中的说明启用自动文档查找。

有关详细信息，请参阅构建层次结构

对新变量名称的建议

除了完成现有符号的调用之外，WebStorm还可以在声明过程中为新变量建议名称。这些建议基于项目中定义的类，类型和接口的名称，您正在使用的库以及标准API中的名称。

默认情况下，此功能已关闭。要激活它，请打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），转到“ 语言和框架”| JavaScript ，然后选择Suggest变量名称复选框。

声明完成

或者，在左括号后面按Ctrl + Shift + Enter 。

您可以使用语句完成Ctrl + Shift + Enter创建语法正确的代码构造。它插入必要的语法元素（括号，大括号和分号），并使您处于可以开始键入下一个语句的位置。

  • 要完成方法声明，请开始键入function并按Ctrl + Shift + Enter 。WebStorm创建方法的整个构造，并将光标放在方法体内。

  • 要完成代码构造，请开始键入它，例如，键入if ，然后按Ctrl + Shift + Enter 。WebStorm自动添加所需的标点符号并将光标放在下一个编辑点。

嬉皮完成

嬉皮完成是一个完成引擎，可以在可见范围内分析您的文本，并从当前上下文中生成建议。它可以帮助您完成任何当前打开的文件中的任何单词。

将插入符号处的字符串展开为现有单词

  1. 键入初始字符串并执行以下操作之一：

    • 按Alt + /或选择代码|完成|循环展开Word以在插入符号之前搜索匹配的单词。

    • 按Shift + Alt + /或选择代码|完成|循环展开Word（向后）以在插入符号和其他打开文件之后搜索匹配的单词。

    出现第一个建议值，原型在源代码中突出显示。

  2. 接受建议，或按住Alt键并按住/直到找到所需的单词。

Postfix代码完成

JavaScript模板也适用于TypeScript文件。

从JavaScript Postfix模板了解更多信息。

Postfix代码完成可帮助您在编写代码时减少向后插入符号跳转。您可以根据在点，表达式类型及其上下文之后键入的后缀将已键入的表达式转换为其他表达式。

WebStorm附带了一组预定义的后缀模板，可让您为JavaScript和TypeScript定义自己的自定义模板。可以复制，更新和删除自定义模板。对于预定义模板，您只能更改其后缀，例如将短键替换为较短的键。

  • 要查看可用的后缀模板的完整列表，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|一般|后缀完成 。

  • 选择Tab ， Space或Enter以展开后缀模板。

完成覆盖

用于覆盖的方法体的生成可在JavaScript和TypeScript中使用。

如果要从父类或接口覆盖方法并从完成建议列表中选择此方法，WebStorm会自动添加参数，生成super()调用，并在可能的情况下添加类型信息。

关闭替代方法体的生成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript 。

  2. 在打开的JavaScript页面上，清除“ 覆盖完成时展开方法体”复选框。

完成标签，属性和属性

如果启用了Emmet支持 ，则Emmet模板也会在HTML和XML上下文中自动完成。

WebStorm自动完成许多文件类型中标记和属性的名称和值：

  • HTML / XHTML，包括完成CSS类和JSX中的HTML标记 。

  • XML / XSL ，包括命名空间的完成。

  • JSON，请参阅编辑package.json以获取详细信息。

标签和属性名称的完成基于与文件关联的DTD或模式 。如果没有架构关联，WebStorm将使用文件内容（标记和属性名称及其值）来完成输入。

  1. 按<并开始键入标签名称。WebStorm显示当前上下文中适当的标记名称列表。使用ArrowUp和ArrowDown按钮滚动列表。

  2. 按Enter键接受列表中的选择。如果您的文件与架构或DTD相关联，则WebStorm会根据它自动插入必需属性。

配置代码完成设置

默认情况下，选中“ 键入时显示建议”复选框。

您可以通过在“ 代码完成”页面上更新其设置来自定义默认完成行为。要打开页面，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 常规 ” ，然后单击“代码完成”。

在键入时自动显示建议列表

  • 选中“ 键入时显示建议”复选框。如果清除该复选框，则必须通过按Ctrl + Space进行基本完成或按Ctrl + Shift + Space进行智能完成来显式调用代码完成。

    您还可以选择在只有一个选项时自动插入建议：在自动插入单个建议下选择完成类型。

以字母顺序显示建议

  • 选中按字母顺序排序建议复选框。默认情况下，该复选框已清除，建议按其相关性排序。

  • 或者使用要么在建议列表的右下角切换这些模式。请注意，对于仅包含少量条目的列表，不会显示这些图标。

在案例列表中包含建议

  • 选择匹配大小写复选框，然后选择是仅匹配首字母大小写还是匹配所有字母大小写。

打开自动文档查找

  • 选中“ 显示文档”弹出复选框。从基本完成了解更多信息：文档查找 。

使用特定键插入建议

  • 按空格，点或其他上下文相关键复选框，选择“ 插入所选建议” 。这些键取决于语言和调用完成的上下文。

虽然此设置可帮助您节省时间，但打开它可能会导致插入项目。

完成一条路

这种类型的完成加速了文件和文件夹的选择。用于此目的的对话框经常被调用，例如，当您指定外部工具或版本控制引擎的安装文件夹或配置Node.js解释器时。

  1. 通过选择“ 文件”菜单上的“ 打开文件 ”或单击“ 浏览”打开“ 选择路径”对话框 在需要路径的字段旁边。

  2. 确保Path字段可见。如果该字段已隐藏，请单击“ 显示路径”链接。

  3. 开始输入路径。WebStorm建议一个列表，您可以在其中快速找到所需的目录。在您键入时，建议列表会缩小以仅显示匹配的路径。您也可以按Ctrl + Space显示建议列表。

  4. 从列表中选择相关建议。

macOS上不提供文件和文件夹路径的完成。

故障排除

如果代码完成不起作用，可能是由于以下原因之一：

  • 省电模式开启（ 文件|省电模式 ）。通过消除后台操作（包括错误突出显示，动态检查和代码完成），打开它可以最大限度地降低笔记本电脑的功耗。

  • 您的文件不驻留在内容根目录中，因此它不会获得代码完成所需的必需类定义和资源。

  • 具有要在代码完成中使用的符号的文件被标记为纯文本文件 。

  • 包含要在代码完成中使用的符号的第三方文件未配置为外部JavaScript库 。
