创建表单初始化代码

完成构建GUI Designer表单后 ，至少应该有一个绑定到表单的类，以及绑定到各种表单组件的类中的组件类型字段。

如果将二进制类文件指定为输出选项，则会在类文件中生成所有运行时初始化代码。如果表单有绑定类，则不会在那里看到自动生成的初始化代码。如果您选择了Java源代码作为输出选项，则表单的绑定类将包含自动生成的$$$setupUI$$$()方法。

不要手动编辑GUI Initializer源代码！在任何编译中，IntelliJ IDEA都会在此部分自动生成新代码，您的所有更改都将丢失。

有时您可能需要提供自己的初始化代码。例如，您希望GUI组件由具有特定参数的非默认构造函数实例化。在这种情况下，IntelliJ IDEA不会生成组件的实例化，您有责任在createUIComponents()方法中提供对构造函数的调用。否则，将报告空指针异常。遵循下面描述的一般技术。

要为特定组件创建自定义GUI初始化程序源代码，请遵循此常规过程

  1. 选择所需的组件。

  2. 在“检查器”中，选中“ 自定义创建 ”选项。

  3. 选择组件后，按F4 ，或在上下文菜单中选择“ 跳转到源 ”。

  4. 在文本编辑器中，找到方法createUIComponents() ，然后键入所需的源代码。编译时不会删除此方法中的代码。

例

例如，您希望为单选按钮1和2提供非默认构造函数，并让GUI Designer为单选按钮3创建默认构造函数：
