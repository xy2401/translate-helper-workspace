Node.js的

此功能仅在Ultimate版本中受支持。

Node.js是一个轻量级的运行时环境，用于在服务器端执行JavaScript。IntelliJ IDEA与Node.js集成，可在配置，编辑，运行，调试，测试，分析和维护应用程序方面提供帮助。

如果您只需要Node.js来管理npm包或运行JavaScript链接，构建工具，测试框架等，只需安装Node.js。如果您遵循标准安装过程，则在大多数情况下，IntelliJ IDEA会检测Node.js本身。

要将Node.js用作应用程序的运行时，需要将其配置为本地或远程解释器。要做到这一点，在上安装插件的NodeJS 插件的描述页面管理插件 。

配置本地Node.js解释器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。

  2. 在打开的Node.js和NPM页面上，单击 节点解释器列表旁边。

  3. 在Node.js Interpreters对话框中打开，其中包含所有当前配置的解释器的列表，单击在工具栏上。在打开的对话框中，在上下文菜单中选择“ 添加本地 ”，然后选择Node.js的本地安装，然后单击“ 确定” 。您将返回Node.js Interpreters对话框，其中Node解释器只读字段显示所选解释器的路径。

  4. 在“ 包管理器”字段中，选择要在当前项目中使用的包管理器（npm或Yarn），有关详细信息，请参阅为项目配置包管理器 。

单击“ 确定”后 ，将返回Node.js和NPM页面，其中节点解释器字段显示新的解释器。

使用系统Node.js版本

使用IntelliJ IDEA，您可以将默认系统节点别名设置为项目的Node.js版本。这意味着如果您安装新节点版本并使其成为系统中的默认版本，则在Node.js解释器字段中指定此系统别名的IntelliJ IDEA中的所有工具和运行配置将使用此较新版本。

使用nvm时，此功能特别有用。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。

  2. 从Node interpreter下拉列表中，选择node 。

  3. 在适用的地方指定这个新的Node.js解释器，例如在运行/调试配置或特定工具的设置中。

在Windows子系统Linux上使用Node.js.

IntelliJ IDEA允许您使用Windows子系统Linux上的 Node.js运行和调试Node.js应用程序。您可以在WSL上选择Node.js作为当前项目的默认解释器，或者您可以在Node.js运行/调试配置中配置和使用此节点版本。

在WSL上配置Node.js作为默认项目节点解释器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。

  2. 点击在“ 节点解释器”字段旁边的“打开的Node.js解释器”对话框中，单击“确定” ，然后从列表中选择添加WSL 。

  3. 在打开的“ 添加WSL节点解释器”对话框中，选择您正在使用的Linux发行版，并指定Node.js的路径。

使用Node.js应用程序快速入门

如果您的计算机上只有一个Node.js并且您遵循标准安装过程，则IntelliJ IDEA会自动检测您的Node.js.否则，从下拉列表中选择相关的解释器，请参阅上面的配置本地Node.js解释器 。

如果您还没有应用程序，可以从模板生成具有Node.js特定结构的IntelliJ IDEA项目，或者创建一个空的IntelliJ IDEA项目并在其中配置Node.js，如下面的开始使用现有Node.js应用程序中所述。 。

创建新的Node.js应用程序

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择Node.js和NPM 。

  3. 在右侧窗格中，选择Node.js Express App ，然后单击Next 。

  4. 在向导的第二页上，指定项目文件夹，Node.js解释器，包管理器（ npm或Yarn ，有关详细信息，请参阅npm和Yarn ）。

    对于Node.js Express应用程序，请指定express-generator的版本并选择要使用的模板语言和样式表语言。

  5. 单击完成。

    根据所选模板，IntelliJ IDEA会生成一个Node.js Express特定项目，其中包含所有必需的配置文件，或者只运行npm init命令，该命令会添加package.json文件并启用Node.js核心API的代码完成。

创建一个空的IntelliJ IDEA项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，再次选择Static Web并单击Next 。

  4. 在向导的第二页上，指定项目文件夹和名称，然后单击“ 完成” 。

从现有的Node.js应用程序开始

如果您要继续开发现有的Node.js应用程序，请在IntelliJ IDEA中打开它，在其中配置Node.js，然后下载所需的依赖项。

打开计算机上已有的应用程序源

  • 单击欢迎屏幕上的打开或选择文件|从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中检出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”或选择“ VCS”从主菜单中查看版本控制 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

要下载项目依赖项，请执行以下操作之一：

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm安装

  • 从项目根目录中的package.json文件的上下文菜单中选择Run'npm install' 。

在项目中配置Node.js.

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。

  2. 在Node Interpreter字段中，为当前项目指定默认的Node.js解释器。每次在创建运行/调试配置或配置Node.js相关工具（例如，Prettier或Yeoman）时从节点解释器列表中选择Project别名时，IntelliJ IDEA将自动使用它。

    从下拉列表或单击时打开的对话框中选择解释器 。如果选择node ，则使用系统Node.js版本 。

  3. 选中Node.js的编码辅助复选框，将Node.js核心模块源配置为JavaScript库 ，并将其与项目关联。因此，IntelliJ IDEA为fs ， path ， http和编译到Node.js二进制文件中的其他核心模块提供代码完成，参考解析，验证和调试功能。

    配置完成后，IntelliJ IDEA将显示有关当前配置版本的信息。

  4. （可选）配置Node.js Core源作为库处理的范围：

    1. 单击“ 管理范围” 。“ 使用范围”对话框打开

    2. 单击相关目录，对于每个目录，从列表中选择新配置的Node.js核心库。
