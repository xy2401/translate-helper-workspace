Grails入门1/2

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA与Grails紧密集成，可以在IDE中使用Grails应用程序，使您无需使用命令行。IntelliJ IDEA中的Grails支持允许您执行以下操作：

  • 在你开始之前
  • 创建Grails项目
  • 探索Grails应用程序
  • 在Grails项目中创建元素
  • 运行应用程序

在你开始之前

在开始创建Grails项目之前，请确保已在计算机上下载Grails SDK。您可以从Grails页面下载最新的SDK版本。此外，请确保您使用的是IntelliJ IDEA终极版，版本9或更高版本。查看最新版本 。

如果您打算使用Grails 3.0版本，请参阅Grails 3入门 。

创建Grails项目

执行以下操作之一：

  • 如果要创建新项目：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件”|新的|项目 。

    结果，“ 新建项目”向导将打开。

  • 如果要将模块添加到现有项目中：打开要添加模块的项目，然后选择“ 文件”|新的|模块 。

    结果，“ 新建模块”向导将打开。

在向导的第一页上，在左侧窗格中，选择Grails 。在页面的右侧，指定以下设置：

  • 您将要使用的Project SDK 。

  • Grails SDK Home - 选择由库表示的本地Grails安装。

  • 创建create-app或create-plugin - 根据您要创建的内容选择其中一个选项。

  • 选项 - 指定其他选项。

单击下一步 。

指定名称和位置设置。

单击完成 。

由于我们选择创建应用程序，IntelliJ IDEA执行create-app目标，该目标生成Grails应用程序的目录结构。所有输出信息都显示在控制台中 ：

探索Grails应用程序

IntelliJ IDEA使您可以从两个不同的角度探索Grails应用程序：

  • Project工具窗口显示典型的Grails目录结构。

  • Grails工具窗口显示Grails应用程序元素的逻辑集（域类，控制器，视图等）

运行应用程序

为了使您的应用程序有用，您可能需要做更多的事情，但让我们尝试使用默认设置立即运行它。为此，请按Shift + F10 ，在控制台中出现 http://localhost:8080/GrailsDemo消息后，您的应用程序将在默认浏览器中启动，地址栏中包含以下URL： http://localhost:8080/GrailsDemo 。
在该页面上，您将看到如下内容：

单击控制器链接以打开书籍列表，默认情况下为空。现在，您可以尝试填写图书馆管理系统的条目，例如，单击“新书”按钮添加图书：如您所见，我们的基本图书馆管理系统已准备就绪。如果您想扩展其功能，或者您对Grails生成的代码不满意，您可以修改IntelliJ IDEA编辑器中的文件以满足您的特定需求，并重新运行该应用程序。
