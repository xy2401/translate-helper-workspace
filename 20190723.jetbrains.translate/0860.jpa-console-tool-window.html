JPA控制台工具窗口

从“ 持久性”工具窗口 （对于持久性单元或其中的任何节点）：

  • 在标题栏上

  • 上下文菜单中的控制台

  • 按Ctrl + Shift + F10

当您打开JPA控制台工具窗口时，首先打开输入窗格。这是您编写JPQL查询的地方。

当您运行第一个查询时（ ），输出窗格在输入窗格上方打开。基本上，这是在控制台中执行的操作的日志。

如果查询检索数据（例如select ），则“结果”窗格也会打开，以表格格式显示检索到的数据。

此外，您可以打开“ 参数”窗格（ ）管理查询中的参数。

在本页面：

  • 工具栏图标和快捷方式
  • 输出窗格
  • 结果窗格

另请参阅使用JPA控制台 。

输出窗格

此窗格显示您运行的查询以及有关其执行的信息。发生错误时，相应的信息也会显示在此窗格中。

对于大多数事件，提供以下信息：

  • 时间戳，即事件发生的时间。

  • 对于数据操作操作 - 受影响的行数（例如，更改或删除）。对于数据检索操作 - 检索了多少行。

  • 持续时间，以毫秒为单位。

使用以下上下文菜单命令：

  • 复制 （ Ctrl + C ）将输出窗格中选定的文本复制到剪贴板。

  • 与剪贴板比较，将输出窗格中选择的文本与剪贴板的内容进行比较。

  • 全部清除以清除输出窗格的所有内容。

结果窗格

此窗格以表格格式显示从数据库检索的数据。您可以对数据进行排序以及执行其他相关任务。

  • 主要功能

数据提取器：<current_format>

使用此按钮或命令打开一个菜单，您可以在其中选择数据的输出格式。

除输出格式外，还有以下选项和命令：

  • 允许换位。 对于分隔符分隔值格式（TSV，CSV）：如果表格已转置，并且您正在将所选单元格或行复制到剪贴板（例如Ctrl + C ），则选项将被转置（如图所示）否则，不转置（如原始表中）。

  • 跳过生成的列（SQL）。 对于SQL INSERT和UPDATE：复制或保存数据时，不要包含自动增量字段。

  • 添加表定义（SQL）。 对于SQL INSERT和UPDATE：复制或保存数据时，添加表定义（CREATE TABLE）。

  • 配置CSV格式。 打开CSV格式对话框 ，该对话框允许您管理分隔符分隔值格式（例如CSV，TSV）。

  • 转到脚本目录。 切换到存储将表数据转换为各种输出格式的脚本的目录。

全部复制到剪贴板/全部保存到文件

  • 全部复制到剪贴板。 使用此命令将表数据复制到剪贴板上。

  • 全部保存到文件。 使用此命令将表数据保存在文件中。在打开的对话框中，指定文件的位置和名称。

注意：

  • 应用当前活动的数据提取器。

  • 如果当前仅显示相应行的子集，则无论如何都将所有相应的行复制到剪贴板或保存在文件中。（当前显示的行数可能受结果集页面大小参数的限制。）

参数窗格

“ 参数”窗格显示在输入窗格中检测到的参数，并允许您编辑它们的值。要打开或关闭此窗格，请使用在工具栏上。

参数值可以指定为文本或数字，也可以指定为包含对象引用和方法调用的Groovy表达式。例如，查询中date参数的值

SELECT o FROM Order o WHERE o 。 日期 > ： 日期

可以指定为

新的 java 。 sql 。日期 （ 系统 的currentTimeMillis（） - 24 * 3600 * 1000）

要开始编辑值，请切换到相应的表格单元格并开始键入。要指示您已完成编辑值，请按Enter或切换到其他单元格。要退出编辑模式并恢复初始值，请按Escape键。
