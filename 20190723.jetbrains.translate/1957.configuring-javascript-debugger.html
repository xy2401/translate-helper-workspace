配置JavaScript调试器

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

通过JavaScript Debugger插件支持在PyCharm中调试JavaScript。该插件默认激活。如果插件已禁用，请在“ 插件”页面上启用它，如管理插件中所述 。

Google Chrome和Chrome系列的其他浏览器支持调试JavaScript代码。

为确保成功调试，只需指定内置Web服务器端口并接受PyCharm为其他调试器选项建议的默认设置即可。

如果您使用JetBrains Chrome扩展程序进行调试，它还会使用此端口连接到正在运行的页面，请参阅使用JetBrains Chrome扩展程序 进行调试以及使用默认Chrome用户数据启动调试会话 。

设置内置Web服务器端口

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 构建，执行，部署”下的“ 调试器 ”。将打开“ 调试器”页面。

  2. 在“ 内置服务器”区域中，指定内置Web服务器运行的端口。默认情况下，此端口设置为默认的PyCharm端口63342 ，PyCharm通过该端口接受来自服务的连接。您可以将端口号设置为以1024及更高版本开头的任何其他值。

可选

  • 通过分别清除“ 可以接受外部连接”或“ 允许未签名的请求”复选框，禁止从其他计算机或PyCharm 外部调用内置服务器上的文件。

  • 选择删除断点的方法，默认设置为“ 单击” 。

  • 在调试器上。“ 调试器”节点下的“ 数据视图”页面，配置高级调试器选项：启用或禁用内联调试 ，指定何时查看具有对象值和表达式评估结果的工具提示等。

使用默认Chrome用户数据启动调试会话

或者，始终从“ 浏览器”列表中选择此Chrome浏览器配置。

您可能会注意到，您的调试会话是在新窗口中启动的，其中包含自定义Chrome用户数据而非默认用户数据 。因此，窗口看起来很不寻常，例如，您的书签，浏览器历史记录和扩展程序都缺失，这些都会破坏您的开发体验。这是因为PyCharm使用Chrome调试协议并使用--remote-debugging-port选项运行Chrome。但是，如果Chrome已启动，则无法为具有相同用户数据的任何新的或现有的Chrome实例打开调试端口。因此，当使用Chrome调试协议时，PyCharm始终在带有自定义用户数据的新窗口中启动调试会话。

摘要

  • 要在同一个Chrome实例中启动调试会话，请像以前一样使用JetBrains Chrome扩展程序 。

  • 要使用您熟悉的外观打开新的Chrome实例，请在PyCharm中配置Chrome以开始使用您的用户数据。在这种情况下，在开始调试会话之前，请务必确保Chrome尚未运行您的用户数据。否则，PyCharm仍会使用您的用户数据启动另一个Chrome实例，但无法为其打开调试端口。因此，PyCharm调试器无法连接到新Chrome实例中的应用程序，并且调试会话无法启动。

在PyCharm中配置Chrome以开始使用Chrome用户数据

  1. 将Chrome用户数据保存在计算机的任何位置。

  2. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ Web浏览器 ”。将打开“ Web浏览器”页面。

  3. 要创建新的Chrome配置，请点击 。列表中将显示一个新项目。在“ 路径”字段中，指定Chrome安装文件夹的路径。

  4. 选择新配置并单击 。Chrome设置对话框随即打开。

  5. 选择使用自定义用户数据目录复选框并指定路径到您的用户数据目录中PyCharm设置。

  6. 按照选择默认PyCharm浏览器中的说明标记您的Chrome浏览器配置默认 设置 ，并且在创建运行/调试配置时不要忘记从浏览器列表中选择默认值 。
