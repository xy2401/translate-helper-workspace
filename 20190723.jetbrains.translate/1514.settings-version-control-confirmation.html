创建文件时

在本节中，指定是否以及如何将在IntelliJ IDEA中创建的文件置于版本控制之下：

  • 在添加到版本控制之前显示选项 ：在打开的对话框中指定选项后，新创建的文件将置于版本控制之下。

  • 静默添加 ：新创建的文件会自动添加到版本控制中，而不会发出任何通知。

  • 不要添加 ：新创建的文件保持未版本化，您可以稍后将其置于版本控制之下。

删除文件时

在本节中，指定删除文件时是否以及如何从版本控制中删除文件：

  • 从版本控制中删除之前显示选项 ：在打开的对话框中选择文件后，也会从VCS中删除本地删除的文件。

  • 静默删除 ：本地删除的所有文件都将从VCS中删除而不要求确认。

  • 不删除 ：本地删除的文件仍在版本控制之下。

空更改列表变为非活动状态时

在本节中，指定删除空更改列表时的IntelliJ IDEA行为。
  • 在删除之前显示选项 ：IntelliJ IDEA在删除已失去其活动状态的空更改列表之前要求确认。

    如果您选择删除此类更改列表，IntelliJ IDEA建议您选择另一个更改列表以标记为活动。

  • 静默删除 ：IntelliJ IDEA会自动删除变为非活动状态的空更改列表，但默认更改列表除外。

  • 不要删除 ：在删除活动状态时不会删除空的更改列表。

调用这些命令时显示选项对话框

在此区域中，指定在调用下面选择的命令时是否希望IntelliJ IDEA要求您进行确认。

  • 命令的可用性取决于特定的版本控制系统。

  • 如果通过选择“以后不显示此对话框”复选框来抑制显示特定于操作的选项对话框，请使用此区域中的控件恢复原始设置。

显示清除只读状态对话框

选中此复选框，以便在编辑器中打开文件并尝试修改文件时，IntelliJ IDEA明确要求取消只读状态。

只读状态可以通过两种方式清除：

  • 使用当前VCS：该文件将添加到更改列表中

  • 使用文件系统：该文件未添加到更改列表中。

此选项与将签出与开始编辑分开的版本控制系统相关。

恢复分支切换的工作原理

如果希望在切换到该分支时恢复工作空间（即一组打开的文件，当前运行配置和与某个分支关联的断点），请选择此选项。

创建后在资源管理器中显示补丁

使用此下拉列表可在创建修补程序时定义IntelliJ IDEA行为。可用选项包括：

  • 是 ：如果选择此选项，则始终打开本机文件管理器以显示修补程序文件。

  • 否 ：如果选择此选项，则不会打开本机文件管理器。

  • 问 ：如果选择此选项，IntelliJ IDEA将显示一个对话框，通知成功创建补丁，以及在本机文件管理器中找到补丁文件的建议。
