在ActionScript中更改方法签名

此功能仅在Ultimate版本中受支持。

在ActionScript中，您可以使用Change Method Signature重构：

  • 更改方法名称和返回类型。

  • 添加新参数并删除现有参数。请注意，您还可以使用专用的Extract Parameter重构添加参数。

  • 重新排序参数。

  • 更改参数名称和类型。

  • 通过方法调用层次结构传播新参数。

初始化程序，默认值和新参数的传播

对于添加到函数的每个新参数，您可以指定：

  • 用于初始化参数的值（或表达式）（IntelliJ IDEA中的初始化字段）。

  • 默认值（或表达式）（ 默认值字段）。

您还可以将引入的参数传播到调用要更改其签名的函数的函数。

重构结果取决于您是否指定这些值并使用传播。

传播。 新参数可以传播到调用要更改其签名的函数的任何函数。在这种情况下，通常，呼叫功能的签名相应地改变。但是，这些更改还取决于初始化程序和为新参数设置的默认值的组合。

初始化程序。 Initializer字段中指定的值将作为默认参数值添加到函数定义中。这使相应的参数成为可选参数。（请参阅Flex / ActionScript文档中对必需参数和可选参数的讨论。）

如果未指定新参数的默认值 （在默认值字段中），则无论是否使用传播，函数调用和调用函数的签名都不会更改。

如果同时指定了初始值设定项和默认值，则重构结果取决于是否使用了传播：

  • 如果未使用传播，则初始化值不会影响函数调用和调用函数的签名。

  • 如果使用传播，则将初始化值添加到调用函数的定义中，作为相应参数的默认值（与您正在更改其签名的函数中的方式相同）。

默认值。 通常，这是要添加到函数调用的值。

如果新参数未传播到调用函数，则此函数内的函数调用也将使用此值。

如果使用传播，则该值对于调用函数内的函数调用无关紧要。

更改方法签名

  1. 在编辑器中，将光标放在要更改其签名的方法的名称中。

  2. 执行以下操作之一：

    • 按Ctrl + F6 。

    • 选择Refactor |在主菜单中更改签名 。

    • 选择Refactor |从上下文菜单中更改签名 。

  3. 在“ 更改签名”对话框中 ，对方法签名进行必要的更改，并指定需要哪些其他相关更改。

    您可以：

    • 通过编辑“ 返回类型”字段的内容来更改方法返回类型。

      代码完成在此字段中以及包含函数参数的表的某些字段中可用。

    • 更改方法名称。为此，请在“ 名称”字段中编辑文本。

    • 使用参数表和右侧的按钮管理方法参数：
      • 要添加新参数，请单击 （ Alt + Insert ）并在相应字段中指定新参数的属性。

        添加参数时，您可能希望将这些参数传播到调用当前方法的方法。

      • 要删除参数，请单击相应行中的任何单元格，然后单击 （ Alt + Delete ）。

      • 要重新排序参数，请使用 （ Alt + Up ）和 （ Alt + Down ）。例如，如果要在列表中的第一个参数中创建某个参数，则可以单击与该参数对应的行中的任何单元格，然后单击“确定”。 所需的次数。

      • 要更改参数的名称或类型，请在相应的表格单元格中进行必要的编辑。

    • 沿着调用当前方法的方法的层次结构传播新方法参数（如果有）。

      （可能有方法调用您正在更改其签名的方法。反过来，这些方法可以通过其他方法调用，依此类推。您可以通过调用方法的层次结构将您所做的更改传播到当前方法的参数，还可以指定哪些调用方法应该受影响，哪些不应该受影响。）

      要传播新参数：

      1. 点击 （ Alt + G ）。

      2. 在“ 选择要传播新参数的方法”对话框的左侧窗格中，展开必要的节点，然后选择要将新参数传播到的方法旁边的复选框。

        为了帮助您选择必要的方法，调用方法和被调用方法的代码显示在对话框的右侧（分别在Caller方法和Callee方法窗格中）。

        当您在左侧窗格中的方法之间切换时，右侧部分中的代码会相应更改。

      3. 单击确定 。

  4. 要立即执行重构，请单击“ 重构” 。

    要在实际执行重构之前查看预期的更改并进行必要的调整，请单击“ 预览” 。
