使用WordPress命令行工具WP-CLI

此功能仅在Ultimate版本中受支持。

要在命令行模式下运行WordPress，您需要一组命令行工具，您可以使用Composer依赖关系管理器或下载wp-cli.phar存档来安装wp-cli/wp-cli软件包。下载的命令行工具必须在IntelliJ IDEA中注册，如PHP命令行工具中所述 。

有关在命令行模式下运行该工具的信息，请参阅PHP命令行工具 。

在你开始之前

  • 确保PHP解释器在PHP页面上的IntelliJ IDEA中配置 ，如配置本地PHP解释器和配置远程PHP解释器中所述 。

  • 确保已安装并启用了PHP和命令行工具支持插件。

    插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装它们，如管理插件中所述 。启用后，插件可在IDE级别使用，也就是说，您可以在所有IntelliJ IDEA项目中使用它们。

  • 配置PHP解释器以使用WordPress ，如配置远程PHP解释器所述 。请注意， WordPress需要PHP 5.3或更高版本。

使用Composer依赖关系管理器安装wp-cli包

在开始之前，请确保您的计算机上已安装Composer，并在Composer Dependency Manager中所述的当前项目中进行初始化。

单击完成 。将使用所选包调用create-project Composer命令。因此，将创建Composer项目，其配置和结构取决于所选的包。之后，创建的IntelliJ IDEA项目将打开。

  1. 在composer.json的上下文菜单中，选择Composer |管理依赖关系 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择wp-cli/wp-cli包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。

    在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

或者选择工具|作曲家|在主菜单上管理依赖项 。

将wp-cli配置为IntelliJ IDEA命令行工具

  1. 在WordPress CLI下载wp-cli.phar 。

  2. 按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的|首适用于MacOS，然后单击工具下的命令行工具支持 。

  3. 在“ 命令行工具支持”页面上，单击“ 添加”按钮。在打开的“选择要添加的工具”对话框中，选择WP-CLI 。

  4. 在打开的WP-CLI对话框中，选择运行WordPress的方法 ：

    • 通过PHAR安装：选择此选项以通过PHP脚本启动WordPress或让IntelliJ IDEA检测并启动wp-cli.phar存档中的启动器。

      从PHP解释器列表中选择一个配置的PHP解释器 。有关详细信息，请参阅配置远程PHP解释器

      在Path to phar文本框中，指定wp-cli.phar存档的位置。手动键入路径或单击“ 浏览” 并在打开的对话框中选择所需的位置。

    • 可执行文件（通过Composer等安装）：选择此选项可通过可执行文件启动WordPress ，该文件在使用包管理工具（例如Composer）安装WordPress时可用。

      在Path to wp.bat字段中，指定wp.bat或wp可执行文件的位置。如果您使用的是Composer，则默认位置为\ vendor \ wp \ cli \ bin \ wp或\ vendor \ wp \ cli \ bin \ wp.bat 。手动键入路径或单击“ 浏览” 并在打开的对话框中选择所需的位置。

  5. 单击“ 确定”后 ，将关闭“ WP-CLI”对话框，IntelliJ IDEA将返回“ 命令行工具支持”页面，其中将wp添加到可用工具列表中。

  6. 从“ 控制台编码”下拉列表中，选择字符集以在“ 命令行工具控制台工具”窗口中显示工具的输出。

  7. 在“ 显示控制台”区域中，选择“ 弹出 窗口”以在弹出窗口或“ 工具”窗口中打开“输入”窗格，以将其显示为“ 命令行工具控制台”工具窗口底部的字段。

运行wp-cli工具

您可以使用专用的命令行工具控制台工具窗口在IntelliJ IDEA中运行WP-CLI命令并分析其输出，就像任何其他命令行工具的命令一样，请参阅PHP命令行工具 。
