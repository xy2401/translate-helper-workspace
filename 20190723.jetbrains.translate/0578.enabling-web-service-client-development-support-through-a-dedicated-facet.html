IntelliJ IDEA 2019.1帮助

通过专用方面启用Web服务客户端开发支持

此功能仅在Ultimate版本中受支持。

如果要为以下类型之一的Web服务开发客户端，则此方法适用：

  • GlassFish中/ JAXWS2。X RI / Netro 1。X / JWSDP2.0 。

  • Apache Axis 。

  • RESTful Web服务 。

IntelliJ IDEA创建相关的Web服务客户端模块结构，并通过专用的Web服务客户端方面自动识别所有必需的库。此外，IntelliJ IDEA可以找到以前下载的必需库，也可以建议将它们下载到您选择的位置。

可以直接将Web服务客户端构面添加到模块，而无需父Web构面。

要通过专用方面启用Web Service Client开发支持，请执行以下操作之一

在任何一种情况下，IntelliJ IDEA都将下载预定义的WS引擎实现版本。

  • 使用专用Web服务客户端构面创建Java模块 。除了设置相关的模块结构并自动提供所有必需的库之外，IntelliJ IDEA还将创建一个带有HelloWorldClient类的示例包，您可以将其用作示例，或者根据需要填充它以开发您的应用程序。

  • 将专用构面添加到现有模块。

为Web服务客户端应用程序创建模块

  1. 执行以下操作之一：

    • 如果要创建新项目：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件”|新的|项目 。

      结果，“ 新建项目”向导将打开。

    • 如果要将模块添加到现有项目中：打开要添加模块的项目，然后选择“ 文件”|新的|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页上，在左侧窗格中，选择Java 。在页面的右侧，指定要使用的JDK 。

  3. 在Additional Libraries and Frameworks下 ，选择WebServices Client复选框。

  4. 从“ 版本”列表中选择所需的WS引擎实现。

  5. 您将需要一个实现所选WS引擎的库。您可以选择使用现有库，创建和使用新库，下载库文件（如果它们尚未在您的计算机上可用），或推迟设置库直到以后。

    • 使用库 - 从列表中选择要使用的库（如果已在IntelliJ IDEA中定义了相应的库）。

      创建 - 如果计算机上已有相应的库文件（ .jar ），则可以在库中排列这些文件并使用该新库。为此，请单击“ 创建” ，然后在打开的对话框中选择所需的文件。

      （可选）单击“ 配置”以编辑选定的库。

    • 下载 - 选择此选项可下载WS引擎实施。（下载的文件将安排在库中。）

      （可选）单击“ 配置”以编辑库设置和内容。

    • 稍后设置库 - 选择此选项以推迟设置库直到稍后的时间。

    配置 - 单击此按钮可编辑“ 使用库”或即将下载的库旁边所选库的设置。

    单击下一步 。

  6. 指定名称和位置设置。

    单击完成 。

将专用构面添加到现有模块

  1. 打开项目工具窗口（例如View | Tool Windows | Project ）。

  2. 右键单击感兴趣的模块，然后选择“ 添加框架支持” 。

  3. 在打开的“ 添加框架支持”对话框的左侧窗格中，选中“ Web服务客户端”复选框。

  4. 如果要生成示例客户端代码，请选中相应的复选框。

  5. 从“ 版本”列表中选择所需的WS引擎实现。

  6. 您将需要一个实现所选WS引擎的库。您可以选择使用现有库，创建和使用新库，下载库文件（如果它们尚未在您的计算机上可用），或推迟设置库直到以后。

    • 使用库 - 从列表中选择要使用的库（如果已在IntelliJ IDEA中定义了相应的库）。

      创建 - 如果计算机上已有相应的库文件（ .jar ），则可以在库中排列这些文件并使用该新库。为此，请单击“ 创建” ，然后在打开的对话框中选择所需的文件。

      （可选）单击“ 配置”以编辑选定的库。

    • 下载 - 选择此选项可下载WS引擎实施。（下载的文件将安排在库中。）

      （可选）单击“ 配置”以编辑库设置和内容。

    • 稍后设置库 - 选择此选项以推迟设置库直到稍后的时间。

    配置 - 单击此按钮可编辑“ 使用库”或即将下载的库旁边所选库的设置。

  7. 在Add Frameworks Support对话框中单击OK 。
