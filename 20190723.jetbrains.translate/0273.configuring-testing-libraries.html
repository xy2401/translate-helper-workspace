配置测试库

JUnit和TestNG的库随IntelliJ IDEA一起提供，但默认情况下不包含在项目或模块的类路径中。因此，在创建测试类时，不会解析对TestCase类或测试注释的引用。

要将必要的库添加到类路径，可以使用向模块添加依赖项的一般过程。相应的库位于以下目录中：

  • JUnit库（ junit.jar和junit-4.12.jar ）： <IntelliJ IDEA directory>\lib 。

  • TestNG库（ testng-jdk15.jar ）： <IntelliJ IDEA directory>\plugins\testng\lib 。

IntelliJ IDEA可以自动将必要的库添加到类路径中。在为类创建测试或编写测试代码时，可以使用相应的功能。

在为类创建测试时，将测试库添加到类路径中

  1. 在编辑器中，将光标放在包含类声明的行中。

  2. 按Alt + Enter可查看可用的意图操作 。

  3. 选择Create Test 。

  4. 在“ 创建测试”对话框的文本右侧，通知您找不到相应的库，单击“ 修复” 。

在编写测试代码时，将测试库添加到类路径中

  1. 在测试类的源代码中，将光标放在未解析的TestCase或注释的引用中。

  2. 按Alt + Enter可查看可用的意图操作 。

  3. 选择Add <library>到classpath 。
