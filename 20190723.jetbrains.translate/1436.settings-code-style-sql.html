一般

使用此选项卡可在代码中指定某些格式化行为。结果显示在“ 预览”窗格中。

项目

描述

单词案例

在此节点中，指定关键字，标识符或带引号的标识符大小写是否应自动更改。单击右侧列，然后在菜单上检查所需的行为以将其打开。

可能的选择是：

  • 要更高 ：元素会自动转换为大写。

  • 降低 ：元素自动转换为小写。

  • 不要改变 ：元素的情况保持原样。

标识符引用

单击右侧列，然后在菜单上检查所需的行为以将其打开。

可能的选择是：

  • Quote ：标识符会自动引用。

  • 取消引用 ：标识符自动不加引号。

  • 请勿更改 ：标识符保持不变 。

新线前/后

使用这些节点可以定义在重新格式化后希望IntelliJ IDEA保留并插入代码的空白行数。对于每种类型的SQL元素，选择清除右侧的复选框。结果在“ 预览”窗格中突出显示。

以分号分隔的新行

使用此复选框可以管理分号周围的行分隔符。

对准

选中所需源代码元素旁边的复选框，使其自动对齐。

标签和缩进

项目

描述

使用制表符

  • 如果选中此复选框，则使用制表符：

    • 按Tab键

    • 缩进

    • 用于代码重新格式化

  • 清除该复选框后，IntelliJ IDEA使用空格而不是制表符。

智能标签

  • 如果选中此复选框，则由代码块嵌套定义的缩进部分由制表符和（如果需要）空格组成，而由对齐定义的缩进部分仅由空格构成。

  • 如果清除此复选框，则仅使用选项卡。这意味着符合指定选项卡大小的一组空格会自动替换为选项卡，这可能会导致细微对齐。

如果选中“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

标签大小

在此字段中，指定选项卡中包含的空格数。

缩进

在此字段中，指定要为每个缩进级别插入的空格数（如果选中“ 使用选项卡字符”复选框，则为选项卡 ）。

继续缩进

在此字段中，指定要在数组元素之间，表达式，方法声明和方法调用之间插入的空格数（或选中“ 使用选项卡字符”复选框）。

在空行上保持缩进

如果选中此复选框，则IntelliJ IDEA将在空行上保留缩进，就好像它们包含一些代码一样。
如果清除此复选框，IntelliJ IDEA将删除制表符和空格。

空间

使用此选项卡可以指定代码中的空格位置。要让IntelliJ IDEA自动在某个位置插入空格，请选中列表中此位置旁边的复选框。结果显示在“ 预览”窗格中。

包裹和支撑

在此选项卡中，自定义代码样式选项，IntelliJ IDEA将在重新格式化源代码时应用这些选项。左侧窗格包含异常列表（ 重新格式化时保留 ），以及各种代码构造（列表，语句，操作，注释等）的放置和对齐选项。右侧窗格显示预览。

对齐优先于缩进选项 。

重新格式化时保留

使用此节点中的复选框配置IntelliJ IDEA在重新格式化源代码时将执行的异常。例如，默认情况下，选中“ 线制动”复选框。如果您的代码包含的行短于标准约定，则可以在重新格式化源代码之前通过禁用“ 行制动”复选框来转换它们。

包裹在里面

使用此节点为冗长元素定义包装样式。
您可以使用左侧窗格的下拉列表中提供的不同包装样式选项 。

值表达式

使用此选项可为值表达式定义包装样式。您可以使用左侧窗格的下拉列表中提供的不同包装样式选项 。

包装样式选项

包装样式适用于左侧窗格中指定的各种代码构造（例如，表达式或赋值语句）。
从下拉列表中，选择所需的包装样式：
  • 不包装 - 选择此选项时，不应用特殊包装样式。

    如果选择此选项，则会忽略嵌套的对齐和大括号设置。

  • 如果长时间换行 - 选择此选项可使线条超出右边距并用适当的缩进包裹。

  • 如果长时间缩小 - 选择此选项可使列表中的元素超出右边距，这样每行有一个元素具有适当的缩进。

  • 总是换行 - 选择此选项可以将列表中的所有元素都包装起来，这样每行有一个元素并且有适当的缩进。

代码生成

该选项卡包含主键和外键约束名称以及索引的模板。在“ 创建表”或“ 修改表”对话框中创建约束和索引时，这些模板用于生成约束和索引的默认名称。

模板可以包含变量（例如， {table} ）和文本。生成名称时，将按字面再现指定的文本。

要获取有关变量的信息以及如何使用它们，请将光标放在感兴趣的字段中，然后按Ctrl + Q.

{columns}和{ref_columns} （取决于具体情况）是列的名称，或列名称以下划线（_）分隔的列表。

{独特？u:}检查索引是否唯一（ unique? ），如果是，则插入指定的字符序列? 和:在这个例子中，它是u ）。如果索引不是唯一的，则插入:和}之间的序列（在此示例中，它没有任何内容）。

例。 使用模板{table}_{columns}_{unique?u:}index ，您要在表persons中创建FirstName和LastName列的索引。如果索引是唯一的，则默认情况下，其名称为persons_FirstName_LastName_uindex 。如果索引不是唯一的，则其名称将为persons_FirstName_LastName_index 。
