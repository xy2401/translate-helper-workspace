配置SDK Gemsets

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

IntelliJ IDEA提供了使用RVM和rbenv 版本管理器的 gemsets的功能。

要查看已安装解释器的 gemsets，请打开“ 产品结构”对话框（ Ctrl + Shift + Alt + S ）并转到“ 模块”页面。然后，转到Ruby SDK和Gems页面。在此页面上，gemsets显示为解释器子项：

对于RVM，您可以选择某个gemset并将项目gem安装到此gemset。另一方面，rbenv版本管理器提供了为项目选择多个gemset的功能：

在这种情况下，宝石将安装到第一个启用的gemset。有关如何安装gem的说明，请参阅将Gems安装到Local Interpreter 。

为现有项目创建RVM gemsets

要在“ 产品结构”对话框中创建RVM gemset（ Ctrl + Shift + Alt + S ），请转到“ 模块”| Ruby SDK和Gems页面并执行以下步骤：

  1. 点击按钮：

  2. 在弹出窗口中，选择New RVM gemset ... ：

  3. 在打开的对话框中，选择将为其创建gemset的Ruby SDK 。然后，指定gemset名称并单击“ 确定” ：

  4. Ruby SDK和Gems页面将gemset显示为所选SDK的子项：

    请注意，星号将附加到gemset名称。这意味着尚未创建gemset。要创建它，请在“ 产品结构”对话框中单击“ 应用”或“ 确定 ”

使用.rbenv-gemset创建Rbenv gemsets

IntelliJ IDEA使您可以使用rbenv-gemsets插件使用rbenv gemsets 。在这种情况下， .rbenv-gemsets文件管理项目gemsets。IntelliJ IDEA将Ruby SDK和Gems页面中.rbenv-gemsets与.rbenv-gemsets指定的.rbenv-gemsets 。例如，如果取消选中Ruby SDK和Gems中的gemset，则会从.rbenv-gemsets文件中删除相应的记录。

此外，您可以在编辑器中创建和禁用gemsets：

请注意，使用远程解释器时不支持rbenv gemsets。

  1. 打开项目的.rbenv-gemsets文件。

  2. 在单独的行上键入所需的gemset名称。

  3. 使用Create gemset'name' 快速修复来创建gemset：

    此gemset将显示为当前SDK的子级：

  4. 如果从.rbenv-gemsets删除此gemset，请单击“ 更新库” ：

    将为当前SDK禁用gemset。

确保启用了针对gemsets更改 检查的Scan .rbenv-gemset（s）文件，以便能够使用此快速修复。
