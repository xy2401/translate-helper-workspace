创建一个新的Maven项目

IntelliJ IDEA允许您创建Maven项目或向任何现有项目添加Maven支持。

Maven 2版本未与IntelliJ IDEA捆绑在一起。要继续使用Maven 2版本，您需要在本地安装它。

  1. 启动“ 新建项目”向导 。如果当前未在IntelliJ IDEA中打开任何项目，请单击“欢迎”屏幕上的“ 创建新项目 ”：否则，请选择“ 文件”|新的|从主菜单中选择项目 。

  2. 从左侧的选项中选择Maven 。

  3. 如果要使用预定义的项目模板（通过单击“ 添加原型”配置自己的原型），请指定项目的SDK（JDK）或使用默认的SDK和原型 。
    单击下一步 。

  4. 在向导的下一页上，指定添加到pom.xml文件的以下Maven基本元素 ：

    • GroupId - 一个新项目的包。

    • ArtifactId - 项目的名称。

    • 版本 - 新项目的一个版本。默认情况下，会自动指定此字段。

    单击下一步 。

  5. 如果使用Maven原型创建项目，IntelliJ IDEA将显示可用于设置Maven主目录和Maven存储库的Maven设置。此外，您可以检查原型属性。单击下一步 。

  6. 指定名称和位置设置。单击完成 。

默认情况下，新创建或导入的项目存储在外部。这意味着生成的.iml和库文件存储在idea.system.path而不是.idea目录中。通过版本控制共享项目可能会有所帮助。

如果需要更改此行为，请打开Maven设置（ “设置”|“Maven”|“导入” ）并清除“ 外部存储生成的项目文件”复选框。

您还可以查看有关如何开始使用Maven项目的快速视频教程。

导入Maven项目

您可以选择文件|新的|从主菜单上的现有源项目或单击欢迎屏幕上的导入项目 。按照导入项目向导的说明，您可以快速导入Maven项目。

导入Maven工件时，IntelliJ IDEA会尝试尽可能使用Maven配置并保留工件的扩展。

  1. 在主菜单上，选择“ 文件”|打开 。

  2. 在打开的对话框中，选择要导入的项目的pom.xml 。单击确定 。

  3. 在“ 导入项目”向导的第一页上，在“ 从外部模型导入项目”中选择“ Maven” ，然后单击“ 下一步” 。

    （如果选择了pom.xml则不会显示此页面。）

  4. 指定Maven设置或使用默认选择。
    默认设置通常足以用于项目。但是，您可以选择以下（常用）选项：

    • 以递归方式搜索项目 - 如果选择此选项，则会找到并正确设置子项目（如果有）。

    • 自动导入Maven项目 - 如果选择此选项，则每次更改POM文件时都会自动导入项目，并且您无需手动控制何时导入更改。但请注意，重新导入大型项目可能需要一些时间。另请注意，IntelliJ IDEA项目中所做的更改（例如，通过“ 项目结构”对话框向项目添加依赖项）将在POM重新导入时被覆盖，因为IntelliJ IDEA将POM文件视为单一事实来源。

    单击下一步 。

    如果IntelliJ IDEA检测到项目中的配置文件 ，则会显示它们。

  5. IntelliJ IDEA显示找到的项目，您可以选择需要导入的项目。
    单击下一步 。

  6. 指定项目的SDK，然后单击“ 下一步” 。

  7. 指定项目的名称和位置。
    单击完成 。

将新的Maven模块添加到现有项目中

您可以将Maven模块添加到您已经在其中工作的项目中。

  1. 在“ 项目”工具窗口中，右键单击项目文件夹，然后选择“ 新建”|模块 。或者，在主菜单上，选择“ 文件”|新的|模块打开新的模块向导。

  2. 如果使用主菜单添加模块，则添加模块的过程与创建新的Maven项目相同 。
    如果要通过右键单击根文件夹来添加子模块，则添加新模块的过程会更短。您需要在ArtifactId字段中指定模块的名称。其余信息会自动添加，您可以使用默认设置或根据您的偏好更改它们。
    另请注意，默认情况下， Add as module to和Parent字段显示您尝试添加模块的项目的基本Maven属性（groupId，artifactId和version）。你可以点击更改显示的信息。

配置多模块Maven项目

IntelliJ IDEA允许您创建多模块Maven项目。多模块项目由具有多个子模块的父POM文件定义。

IntelliJ IDEA支持自定义pom文件名。您可以成功导入和执行具有不同名称的POM。

  1. 创建一个Maven父项目 。IntelliJ IDEA创建一个标准的Maven布局，包括一个src文件夹。

  2. 在Project工具窗口中，去掉src文件夹（你会需要它非常罕见的情况下，你的一般项目，你不需要src父POM文件夹）。

  3. 在“ 项目”工具窗口中，右键单击项目（或在主菜单中单击“ 文件” ），然后选择“ 新建”|用于添加子项目的模块 。

  4. 在“ 新建模块”向导中，按照有关如何添加模块的说明进行操作，指定必要的信息，然后单击“ 完成” 。
    src文件夹是自动创建的，您可以打开POM并添加所需的包装。IntelliJ IDEA将模块添加到父项目。IntelliJ IDEA还将子项目的名称和描述添加到父POM。

    请注意，父POM中的packaging被定义为pom因为它是父项目的适当包装，引用其他子项目。
    最后，但并非最不重要的是，IntelliJ IDEA将父POM的描述添加到子项目的POM中。你可以点击在左侧排水沟中快速打开子项目中的父POM。
  5. 您还可以将依赖项添加到子项目将继承的父POM。

  6. 打开Maven项目工具窗口，查看父POM中所做的所有更改都会反映在子项目中。

将常规项目转换为Maven项目

您可以打开现有的非Maven项目，并通过IntelliJ IDEA UI添加Maven支持。

  1. 打开现有项目，例如Java项目。

  2. 在“ 项目”工具窗口中，右键单击项目，然后选择“ 添加框架支持” 。

  3. 在打开的对话框中，从左侧的选项中选择Maven ，然后单击“ 确定” 。
    IntelliJ IDEA将默认POM添加到项目中，并在“ 项目”工具窗口中生成标准Maven布局。

    IntelliJ IDEA还在Maven项目工具窗口中创建了具有生命周期和插件的相应结构。

  4. 打开生成的POM并指定groupId 。artifactId和version是自动指定的。

    每次更改POM时，IntelliJ IDEA都会显示一个弹出窗口，建议您导入更改。

此时，您可以使用Maven进一步开发项目。我们建议在POM中进行所有项目更改，因为IntelliJ IDEA将pom.xml视为单一事实来源。

您可以结束以下可选步骤来创建可执行JAR。

  1. 点击建立项目。IntelliJ IDEA生成target文件夹。请注意，IntelliJ IDEA仅编译源，不会创建JAR文件或Manifest文件。

  2. 在resources目录中创建清单文件。

  3. 在POM中指定Manifest文件信息，这样您就可以使用Maven生成可执行的.jar文件。

  4. 在Maven Projects工具窗口的Lifecycle列表中，双击install命令以生成.jar文件。
    IntelliJ IDEA在目标文件夹中生成适当的信息，在项目工具窗口中生成可执行的JAR。

    您可以右键单击生成的JAR，然后选择“运行”以执行该文件。

如果您现有的项目较大且包含多个模块，则将此项目转换为Maven项目变得非常具有挑战性。由于IntelliJ IDEA在您转换项目时仅从POM识别项目设置，因此您需要检查并调整以下设置：

  • 注释设置 - 为模块更改它们。

  • 编译器输出 - 为模块更改。

  • 资源设置 - 它们被POM忽略和覆盖。

  • 模块依赖 - 需要检查它们。

  • 语言和编码设置 - 它们针对模块进行了更改。

此外，没有生成POM模板。所有依赖项（包括模块依赖项）都需要手动包含在POM中。

在这种情况下，我们建议您创建一个外部POM来描述项目并导入 POM，就像使用File |导入常规Maven项目一样。新的|来自Existing Sources命令的项目 。

IntelliJ IDEA将POM添加到项目中，并为现有元素添加Maven布局。

IntelliJ IDEA还在Maven Projects工具窗口中生成相应的结构。

将构建和运行操作委派给Maven

IntelliJ IDEA允许您将所有构建和运行操作委派给Maven。构建项目（ Build | Build Project ）时，IntelliJ IDEA会调用相应的Maven目标。此外， 运行菜单中的运行和调试操作是使用Maven执行的。

HotSwap也会被触发，并在调试过程中重新加载类。

  1. 点击在Maven Projects工具窗口中。或者，在主菜单上选择File |设置/首选项|构建，执行，部署|构建工具| Maven 。

  2. 单击Maven，然后从列表中选择Runner 。

  3. 在Runner页面上，选择Delegate IDE build / run actions to maven 。

  4. 单击确定 。

  5. 使用pom.xml文件打开任何Java项目。

  6. 调用Build Project操作（ Ctrl + F9 ）。Maven编译代码并在Build工具窗口中显示它。
    如果运行main方法（ Ctrl + Shift + F10 ），IntelliJ IDEA也将使用Maven来运行该类。

使用Maven依赖项

IntelliJ IDEA允许您管理项目中的Maven依赖项。您可以添加，导入Maven依赖项，并以图表形式查看它们。

添加Maven依赖项

IntelliJ IDEA允许您向项目添加Maven依赖项。我们建议您在POM中指定依赖项。您在IntelliJ IDEA模块设置中手动设置的依赖关系将在下一个Maven项目导入时被丢弃。

您可以在Maven项目中启用注释处理器。在POM中指定annotationProcessors和annotationProcessorPaths ，IntelliJ IDEA将添加适当的路径到位于Settings / Preferences |中的Annotation处理器设置。构建，执行，部署|重新导入项目后编译器 。

  1. 在编辑器中打开您的POM。

  2. 按Alt + Insert打开Generate上下文菜单。

  3. 从上下文菜单中，选择“ 依赖关系”或“ 依赖关系模板”以进行快速搜索。

  4. 在打开的对话框中，如果切换到“ 搜索类”选项卡，则会搜索工件或类 。
    搜索maven工件的语法如下：
    group - id ： artifact - id ： version
    您还可以使用:*:通配符来指定搜索位置:充当group-id，artifact-id和version之间的分隔符。

    例如，键入*:fest:*将返回其artifact-id中包含“fest”部分的artifact-id 。

    键入easy:*:4将返回其版本中具有“4”且在其group-id “简单”的工件。单击添加 。IntelliJ IDEA将依赖项添加到您的pom.xml 。IntelliJ IDEA还将依赖项添加到Maven项目工具窗口中的Dependencies节点以及Project工具窗口中的External Libraries 。
    如果添加的依赖项具有自己的传递依赖项，则IntelliJ IDEA会在两个工具窗口中显示它们。

在类中搜索时，IntelliJ IDEA会搜索所有可用的工件，并添加所有库，其中检测到具有指定名称的类。

集中依赖性信息

在多模块Maven项目中，父POM中的依赖项将由所有子项目继承。您可以使用dependencyManagement来合并和集中依赖项版本的管理。

  1. 在编辑器中打开您的POM。

  2. 按Alt + Insert打开Generate上下文菜单。

  3. 从上下文菜单中，选择Managed Dependency选项，该选项将显示在多模块项目中父POM的dependencyManagement部分中定义的dependencyManagement列表。

  4. 选择所需的依赖项并单击“ 确定” 。

    依赖关系被添加到POM中。您不需要指定从DependencyManagement获取的DependencyManagement 。但是，如果要覆盖定义的版本，则需要在将托管依赖项添加到POM时包含version 。

添加Maven依赖项的范围

您可以使用POM为依赖项添加范围。在这种情况下，IntelliJ IDEA将在指定阶段执行依赖项。

  1. 在POM中，在依赖关系描述中添加scope并使用代码完成添加范围的名称。

  2. 导入您的更改。范围的名称显示在Maven Projects工具窗口中。在“ 项目结构”对话框的“ 模块”页面上，您可以看到还显示了依赖关系的范围。

    请注意，在“ 项目结构”对话框中更改依赖项的范围不会影响pom.xml文件。

您还可以在定义依赖项时使用Maven作用域系统将自定义.jar文件添加为依赖项。但请注意，此依赖关系仅在您的计算机上可用，您只能将其用于本地部署。

使用Maven传递依赖项

IntelliJ IDEA允许您查看使用添加或导入的Maven依赖项引入的传递依赖项。您可以检查他们的版本，提出它们，或完全排除这些依赖项。

Maven项目工具窗口显示直接依赖项及其所引入的所有传递依赖项。

  1. 在项目的POM中，按Ctrl键并将鼠标悬停在依赖项上。

  2. 单击依赖项以打开依赖项POM。

  3. 在依赖项POM中，查看活动依赖项，其传递依赖项及其版本。

    您可以检查从中获取依赖关系的来源。

如果需要，可以排除传递依赖关系。

您可以使用Maven依赖关系图中的上下文菜单中的Exclude命令快速从POM和相应的工具窗口中排除指定的依赖关系。

  1. 打开依赖项POM并找到要排除的传递依赖项。复制groupId和artifactId 。

  2. 在项目POM中，在活动依赖项下，输入exclusions并使用代码完成粘贴要排除的依赖项的复制信息。

    依赖项也从Project和Maven Projects工具窗口中排除。

导入Maven依赖项

IntelliJ IDEA允许您将依赖项导入Maven项目。当IntelliJ IDEA导入添加的依赖项时，它应该解析依赖项并自动设置项目。

  • 在Maven Projects工具窗口中，单击图标打开Maven |导入设置并选中“ 自动导入Maven项目”复选框。此外，请确保导入程序的JDK与您尝试使用的JDK版本匹配。
    在这种情况下，每次更改POM时都会自动更新依赖项。

  • 按在Maven Projects工具窗口中。在这种情况下，您手动触发重新导入过程和依赖项的更新。

如果由于某种原因未正确导入依赖项，请尝试执行以下操作：

  • 您可以尝试在Maven更新远程存储库更新的Maven指数|存储库设置。

  • 您可以在Maven中检查本地maven存储库存储库设置并尝试更新它。

  • 您可以检查本地.m2存储库的.jar文件，以查看它是否已正确下载。

  • 您可以检查有效的POM以确定将哪个Maven存储库用作依赖项的原点。

  • 您可以在Maven设置中选择“ 始终更新快照”选项。在这种情况下，IntelliJ IDEA会检查下载的依赖项的最新版本并相应地进行更新。

使用Maven目标

IntelliJ IDEA允许您在项目中创建，调试和管理Maven目标。

运行Maven目标

IntelliJ IDEA允许您使用多种方式运行Maven目标。您可以从命令行运行Maven目标，使用Maven Projects工具窗口中的上下文菜单，或创建运行/调试配置以运行一个或多个Maven目标。

此外，当您在无干扰模式下工作（没有工具栏和工具窗口）时，您可以从Run Anything窗口运行Maven目标或命令。按两次Ctrl键将其打开，键入命令并按Enter键 。

您可以查看Mastering快捷方式部分以获取更多信息。

从命令行运行Maven目标

  1. 在Maven Projects工具窗口的工具栏上，单击图标。

  2. 在“ 执行Maven目标”对话框的“ 命令行”字段中，开始输入目标名称。您可以看到IntelliJ IDEA显示Maven目标列表，您可以从中选择合适的目标。单击“ 执行” 。

  3. IntelliJ IDEA运行所选目标并在“ 运行”工具窗口中显示结果。

从上下文菜单中运行Maven目标

  1. 在Maven Projects工具窗口中，单击Lifecycle以打开Maven目标列表。

  2. 右键单击所需目标，然后从上下文菜单中选择“运行目标名称” 。IntelliJ IDEA运行指定的目标并将其添加到“ 运行配置”节点。

通过运行配置运行Maven目标或一组目标

IntelliJ IDEA允许您为一个特定目标或一组目标创建运行配置。

  1. 在Maven Projects工具窗口中，单击Lifecycle以打开Maven目标列表。

  2. 右键单击要为其创建“运行”配置的目标。（要选择多个Maven目标，请按Ctrl键并突出显示所需目标。）

  3. 从列表中选择创建'目标名称' 。

  4. 在Create Run / Debug Configuration：'goal name'对话框中，指定目标设置（您可以指定任何Maven命令和参数），然后单击OK 。

    IntelliJ IDEA显示“ 运行配置”节点下的目标。

  5. 双击目标以运行它或右键单击目标，然后从上下文菜单中选择“运行” 。

配置Maven目标的触发器

IntelliJ IDEA允许您在项目执行之前运行Maven目标，或使用目标激活配置设置其他条件。

  1. 在Maven Projects工具窗口中，单击Lifecycle以打开目标列表。

  2. 在打开的列表中，右键单击要为其设置触发器的目标。

  3. 从上下文菜单中，选择激活阶段。例如， 在构建之前执行 。

    激活阶段的名称将添加到Maven项目工具窗口中的选定目标。

您还可以创建依赖于Maven目标的运行/调试配置。

  1. 在主菜单上，选择“运行”|编辑配置以打开项目的运行/调试配置。

  2. 在“ 运行/调试配置”对话框的“ 启动前”部分中，单击图标。

  3. 在打开的列表中，选择“运行Maven目标” 。

  4. 在“ 选择Maven目标”对话框中，指定要在启动项目之前执行的项目和目标。

  5. 单击确定 。

将Maven目标与键盘快捷键相关联

您可以将Maven目标与键盘快捷键相关联，并使用单个键击执行目标。

  1. 在Maven项目工具窗口中 ，右键单击所需目标。

  2. 从上下文菜单中，选择“ 指定快捷方式” 。“ 键盘映射”对话框打开。

  3. 在Keymap对话框中，在Maven节点下导航到您的目标。

  4. 右键单击目标，然后从打开的列表中选择要分配的快捷方式类型。

  5. 在打开的对话框中，根据快捷方式的类型，配置快捷方式，然后单击“ 确定” 。

    在Maven项目工具窗口中，您的目标将显示该快捷方式。

在“ 键盘映射”对话框中，您可以添加要为其指定快捷方式的新目标。

  1. 在“ 键盘映射”对话框的“ Maven”节点下，单击“选择阶段/目标”以指定快捷方式 。

  2. 在打开的对话框中，选择所需的目标，然后单击“ 确定” 。

    目标将添加到Maven节点下的列表中。现在您可以配置快捷方式 。

调试Maven目标

IntelliJ IDEA允许您为一个或多个Maven目标创建调试配置，或者您可以选择目标并启动调试会话。

  1. 在主菜单上，选择“运行”|编辑配置以创建Maven运行/调试配置。

  2. 在打开的对话框中，单击左侧打开“ 添加新配置”列表。

  3. 从打开的列表中，选择Maven以创建新的Maven配置。

  4. 在对话框的右侧，指定信息，例如配置名称，项目目录，命令行参数和配置文件。您可以为其余部分保留默认设置。单击确定 。

  5. 在主菜单上，单击开始你的调试会话。

您还可以在Maven项目工具窗口中为单个Maven目标或可能包含多个Maven目标的Maven运行配置启动调试会话。

  1. 打开Maven Projects工具窗口。

  2. 在“ 生命周期”节点下，选择要为其启动调试会话的目标。（在“ 运行配置”节点下查找现有的Maven运行配置，以启动运行配置的调试会话。）

  3. 右键单击目标，然后从上下文菜单中选择Debug [目标名称] 。IntelliJ IDEA启动调试会话。

在Maven项目中运行测试

IntelliJ IDEA允许您使用默认的IntelliJ IDEA测试运行器运行测试。当您运行JUnit或TestNg测试和Maven Failsafe插件参数以运行集成测试时，您还可以传递Maven Surefire插件参数。Maven surefire插件默认在超级POM中声明，但您可以在项目的POM中调整其设置。

您可以像平常一样为Java项目创建和运行测试。请参阅执行测试部分。

您可以使用test Maven目标运行所有测试，或使用Maven命令运行单个测试。

  1. 打开Maven Projects工具窗口。

  2. 在Lifecycle节点下选择测试 。请注意，在此阶段将激活Maven surefire插件中指定的目标，并且将运行项目或模块中的所有测试。

如果只想运行单个测试而不是项目中声明的所有测试，请使用Maven -Dtest='TestName' test命令。

  1. 打开Maven Projects工具窗口。

  2. 在工具栏上，单击图标。

  3. 在“ 选择Maven目标”对话框中，指定包含测试的项目或模块，然后在“ 命令行”字段中输入-Dtest='TestName' test命令。单击“ 执行” 。

    Maven运行指定的测试并在“ 运行”工具窗口中显示结果。

或者，您可以使用相同的Maven命令创建Maven运行配置以运行单个测试。运行配置将保存在“ 运行配置”节点下。

  1. 在Maven Projects工具窗口的Lifecycle节点下，右键单击test目标。

  2. 从上下文菜单中，选择“ 创建模块/项目的名称和目标名称” 。

  3. 在打开的对话框中，指定包含要运行的测试的工作目录，并在“ 命令行”字段中指定阶段（自动指定）和-Dtest='TestName' test命令。单击确定 。

  4. 打开“ 运行配置”节点，然后双击要运行的配置。

    Maven运行测试并在“ 运行”工具窗口中显示结果。

例如，当您只想编译项目并且不想等待Maven完成测试执行时，您可以跳过运行测试。

  1. 点击 Maven Projects工具窗口中的图标用于打开Maven设置，并从左侧的选项中选择Runner 。

  2. 在Runner页面上，选择Skip tests并单击OK 。

    IntelliJ IDEA取消激活Lifecycle节点下的test目标。当您执行其他目标时， Maven运行工具窗口中会显示通知已跳过测试的相应消息。

使用Maven配置文件

IntelliJ IDEA允许您使用Maven构建配置文件 ，它可以帮助您为特定环境（例如，生产或开发）自定义构建。

声明Maven配置文件

IntelliJ IDEA允许您在项目的POM中明确声明配置文件。使用代码完成，您可以在profiles标记内放置许多不同的配置，并覆盖POM中为Maven插件，依赖项，存储库等指定的默认配置。

  1. 在编辑器中打开您的POM。

  2. 指定<profiles>部分并声明配置文件。

    IntelliJ IDEA将它们显示在Maven项目工具窗口的“ 配置文件”列表中。

或者，您可以使用以下方法之一声明配置文件：

  • 您可以在Maven设置目录%USER_HOME%/.m2/settings.xml本地定义它们。

  • 您可以在全局Maven设置${maven.home}/conf/settings.xml全局定义它们。

  • 您可以在位于项目基本目录（profiles.xml）中的概要描述符中定义它们。请注意，Maven 3.0不支持此选项。请参阅Maven 3兼容性说明 。

激活Maven个人资料

当IntelliJ IDEA导入Maven项目时，它会检测配置文件并允许您在导入期间激活它们。

  1. 开始导入 Maven项目。

  2. 在“ 从Maven导入”页面中，IntelliJ IDEA显示配置文件，激活您需要的配置文件。

  3. 单击“ 下一步”完成导入。

您可以使用-P命令或使用“ 配置文件”节点和相应的配置文件复选框在Maven项目工具窗口中手动激活配置文件。

  1. 打开Maven Projects工具窗口。

  2. 在工具栏上，单击图标。

  3. 在打开的对话框中，在“ 命令行”字段中输入-P和配置文件的名称。如果您需要排除某些配置文件，请指定! 在个人资料名称前面。即使默认激活该配置文件，也会将其排除在外。单击确定 。

或者，您可以使用Maven项目工具窗口中的“ 配置文件”节点来激活配置文件。

  1. 打开Maven Projects工具窗口。

  2. 单击“ 配置文件”节点以打开已声明的配置文件列表。

  3. 选择相应的复选框以激活所需的配置文件。您可以拥有多个活动配置文件。激活它们时，将根据POM配置文件声明合并其配置。

您还可以根据一系列上下文条件自动激活配置文件，例如，JDK版本，操作系统名称和版本，是否存在特定文件或属性，但您仍需要指定POM中的所有参数。

如果在POM中使用activeByDefault标记声明此类配置文件，则可以默认自动激活Maven配置文件。

IntelliJ IDEA在Maven项目工具窗口中显示activeByDefault配置文件，其中所选复选框显示为灰色。您可以通过单击复选框手动取消激活此配置文件。另请注意，如果手动激活任何其他配置文件，则将取消激活activeByDefault配置文件。使用多模块项目时，请记住，如果在POM中指定了activeByDefault配置文件，则在手动激活任何其他配置文件时，即使它在不同模块的POM中声明，也将取消激活。

每次更改pom.xml文件时都不要忘记同步，以便在Maven Projects工具窗口中查看更改。更改pom.xml后，将显示一个弹出窗口，建议导入更改。您可以启用自动导入，也可以单击导入更改 。你也可以点击 Maven Projects工具窗口工具栏上的按钮。
