Java FX选项卡

档案|项目结构|文物| <artifact> | Java FX

使用此选项卡指定打包JavaFX应用程序或应用程序预加载器的设置 。

设置取决于工件类型（JavaFx Application或JavaFx Preloader）。

  • JavaFx应用程序设置
  • JavaFx Preloader设置

JavaFx应用程序设置

项目

描述

应用类

合格的应用程序主类名称。通常，这是扩展javafx.application.的类javafx.application.Application类并包含main()方法。

标题

申请标题。在技术术语中，这是相应部署描述符JNLP文件中 <title>元素的信息。

如果未指定，则使用标题Sample JavaFX Application 。

供应商

应用程序供应商的名称（即JNLP文件中<vendor>元素的信息）。

如果未指定，则使用文本Unknown vendor 。

描述

应用程序的简要描述（即JNLP文件中<description>元素的信息）。

如果未指定，则使用Sample JavaFX 2.0 application 。

宽度

应用程序窗口的宽度（以像素为单位）。此参数主要由嵌入在网页中的应用程序使用。

高度

应用程序窗口的高度（以像素为单位）。此参数主要由嵌入在网页中的应用程序使用。

HTML参数

您可能希望将动态参数传递到在Web Start中运行的应用程序或从相应的HTML页面嵌入浏览器模式。在这种情况下，您应该创建一个.properties文件，并在该文件中指定必要的命名参数集。然后，您应该在此字段中指定.properties文件的路径。

应用参数

您可能希望将命名和未命名参数传递给应用程序。在这种情况下，您应该创建.properties文件并在该文件中指定必要的参数。然后，您应该在此字段中指定.properties文件的路径。（指定的参数将包含在生成的部署描述符JNLP文件中 。）

例

如果.properties文件包含

ARG1 =值1
ARG2

将在<jfx:javafx-desc>生成以下元素：

<fx：param name =“arg1”value =“value1”/>
的<fx：引> ARG2 </ FX：自变量>

在后台更新

使用此复选框可以在JNLP文件中设置<update>元素的check属性。（此元素用于指定Java Web Start应如何处理应用程序更新。）

如果选中，则<update check="background"/> 。在启动应用程序时，JNLP客户端将在后台检查更新。

如果未选中，则<update check="always"/> 。JNLP客户端将始终在启动应用程序之前检查更新。

本机包

要生成的本机包。（本机捆绑包是特定于操作系统的独立应用程序包。这样的包包含一个应用程序本身以及一个JRE，JavaFX运行时和一个特定于平台的应用程序启动器。）

选择：

  • 没有。 不会创建特定于操作系统的捆绑包。该应用程序打包到以下文件中：

    • <ARTIFACT_NAME>的.jar。 此文件包含已编译的类文件和图像。

    • <ARTIFACT_NAME>的.jnlp。 这是用于Web部署模式的部署描述符JNLP文件（Web Start和嵌入在浏览器中）。

    • <ARTIFACT_NAME> html的。 此文件包含在Web Start中运行应用程序并嵌入浏览器模式的基本代码。

  • 所有。 将生成下面列出的所有捆绑包。

  • DEB。 生成Debian GNU / Linux的Debian软件包。

  • DMG。 生成用于macOS的Apple磁盘映像文件。

  • 可执行程序。 生成Windows的可执行文件。

  • 图片。 将生成您正在使用的OS的捆绑映像。

  • 微星。 生成Windows Installer（Microsoft Installer）文件。

  • 转。 生成用于Linux的Red Hat Package Manager文件。

将css转换为bin

如果希望将应用程序CSS文件转换为二进制格式，请选中此复选框。（这可能会提高应用程序性能，尤其是“大型”CSS文件。）

启用签名

如果要对应用程序包进行数字签名，请选中此复选框。

IntelliJ IDEA可以生成密钥和相应的自签名证书，或者可以使用现有密钥对包进行签名。

要选择应使用哪种签名方式，请单击“ 编辑证书” 。然后，在打开的“选择证书”对话框中，选择：

  • 自签。 该包将使用生成的自签名证书进行签名。

  • 按键签名。 该软件包将使用您的私钥进行签名。指定关联设置：

    • 别名。 指定您的私钥别名。

    • 密钥库。 指定密钥库文件的路径。（这是您的私钥和相应证书的存储位置。）

    • Storepass用于。 指定访问密钥库的密码。

    • 关键通过。 指定访问密钥的密码。
