测试Node.js.

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

以下内容仅在安装并启用Node.js插件时有效！

使用PyCharm，您可以使用大量框架测试Node.js应用程序。自2018年5月以来，不推荐使用Nodeunit。有关详细信息，请参见Nodeunit官方网站。

在你开始之前

  1. 安装Node.js并将其配置为本地Node.js解释器 。

  2. 下载并安装测试框架。

  3. 在插件页面上安装并启用NodeJS存储库插件，如管理插件中所述 。

为Node.js应用程序创建和运行单元测试

  1. 启用nodeunit支持 。

  2. 写单元测试 。

  3. 要让PyCharm识别单元测试并检测相应的生产源代码，请将单元测试存储的文件夹标记为测试文件夹 。

  4. 创建 Nodeunit类型的运行配置 。

  5. 启动单元测试并在“ 运行”工具窗口中监控测试结果 。

创建Nodeunit测试

  1. 在与src文件夹相同的级别创建文件夹测试

  2. 填充测试文件夹。对于每个生产文件，创建一个单独的测试文件。

  3. 将存储测试的文件夹标记为测试文件夹 。

创建Nodeunit运行配置

  1. 选择Run |打开Run / Debug Configuration对话框在主菜单上编辑配置 。

  2. 单击“ 添加”按钮在工具栏上，选择Nodeunit配置类型。

  3. 在打开的对话框中，指定以下内容：

    1. 用于标识配置的名称。

    2. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    3. 工作目录。这可以是项目根文件夹或测试文件夹的父目录。

    4. 要运行的测试范围。

      • 要让PyCharm运行文件夹中的所有测试文件，请从“ 运行”下拉列表中选择目录中的所有JavaScript测试文件 。在“ 目录”字段中，提供相对于工作目录的测试文件夹的路径。

      • 要执行特定测试，请从“ 运行”下拉列表中选择JavaScript测试文件 。在JavaScript测试文件字段中，提供相对于工作目录的文件路径。

  4. 应用更改并关闭对话框。
