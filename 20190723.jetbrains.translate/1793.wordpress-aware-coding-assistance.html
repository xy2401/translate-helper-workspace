IntelliJ IDEA 2019.1帮助

WordPress特定编码协助

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA为开发WordPress提供完整的编码帮助，包括WordPress -aware代码完成，搜索钩子注册函数和指定为钩子注册参数的函数，钩子注册和钩子调用之间的导航，回调，根据的配置编码风格的可能性WordPress代码样式，从IntelliJ IDEA编辑器查看官方WordPress文档，等等。

将WordPress添加到项目中

要利用IntelliJ IDEA提供的编码帮助，您正在使用的WordPress安装应在项目中配置为外部库 。因此， WordPress核心文件将参与索引，这是解析引用和提供代码完成，导航，搜索等的基础。

如果wp-content文件夹在WordPress安装之外，则需要将其添加为包含路径或内容根 。

将WordPress安装配置为项目包括路径

要将WordPress作为外部库添加到项目中，需要将其添加到包含路径列表中。在配置包含路径中了解更多信息。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP 。

  2. 在打开的PHP页面上，将WordPress安装文件夹的路径添加到“ 包含路径”列表中：单击按钮并在打开的对话框中指定安装文件夹的路径。

将wp-content文件夹添加到项目中

如果wp-content文件夹位于WordPress安装之外，则除了WordPress核心文件之外，还需要将其单独添加到项目中。这可以通过两种方式完成：

  • 要使wp-content文件夹参与索引而不将其置于项目版本控制之下，请将其添加到包含路径列表中：

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP 。

    2. 在打开的PHP页面上，将wp-content文件夹的路径添加到Include Paths列表：单击按钮并在打开的对话框中指定文件夹的路径。

  • 要使wp-content文件夹参与索引并将其置于项目版本控制之下，请将其添加为内容根目录 ：

    1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

    2. 在左侧窗格中，选择“ 模块” 。

    3. 在右侧窗格中，选择所需的模块。

    4. 在对话框的右侧，选择“ 源”选项卡。

    5. 单击“ 添加内容根” 。

    6. 在打开的对话框中 ，找到wp-content目录，然后单击“ 确定” 。

    详细了解如何在内容根中配置内容根 。有关将文件和文件夹添加到版本控制的信息，请参阅启用版本控制 。

配置WordPress代码样式

在IntelliJ IDEA中，您可以根据WordPress编码标准使用WordPress代码样式 。详细了解配置代码样式 。

配置本机WordPress代码样式

  • 请执行下列操作：

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 编辑器”|代码风格| PHP 。

    2. 关于代码风格。打开的PHP页面 ，单击Set from ...链接。

    3. 从上下文菜单中，选择“ 预定义样式” ，然后选择“ WordPress” 。

  • 激活WordPress集成后，IntelliJ IDEA会显示一个弹出窗口，提供您设置WordPress代码样式。单击弹出窗口中的“ 设置”链接，然后在“代码样式”上设置代码样式。打开的PHP页面。

根据Wordpress编码标准检查代码

使用IntelliJ IDEA，您可以结合WordPress编码标准使用PHP Code Sniffer工具检测编码标准问题， WordPress编码标准为PHP Code Sniffer提供了一组Wordpress特定标准。这将确保您的代码干净，一致并且没有一些常见错误。

首先，使用安装和配置PHP Code Sniffer中描述的任何技术安装PHP Code Sniffer 。可能最简单的方法是使用Composer安装它。

安装PHP Code Sniffer

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择squizlabs/php_codesniffer包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

接下来，安装WordPress编码标准 ，它将为PHP Code Sniffer提供Wordpress特定的标准。

安装WordPress编码标准

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择wp-coding-standards/wpcs包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

将安装WordPress编码标准包，并将自动选择相应的Wordpress标准进行PHP Code Sniffer验证检查。如有必要，您可以在编辑器|上进一步自定义检查“ 设置/首选项”对话框的“ 检查”页面（ Ctrl + Alt + S ）。有关详细信息，请参阅将PHP代码嗅探器配置为IntelliJ IDEA检查 。

要一次安装这两个软件包，请将它们添加到composer.json的require-dev部分，然后按Composer中的说明运行Install 。

钩子支持

IntelliJ IDEA索引在WordPress核心和包含的插件中声明的所有挂钩。这使得在使用钩子时编码辅助的基础。

完成Action和Filter函数的参数

在动作和过滤器函数的标准参数的代码完成中可以使用钩子名称（ add_action()和add_filter() ）。

调用参数的完成

  1. 开始键入add_action()或add_filter()函数的声明。

  2. 按Ctrl + Space并从列表中选择相关参数。

从挂钩注册导航到挂钩调用

您可以从钩子注册（ add_action()或add_filter()函数）导航到钩子调用。

  • 点击钩子注册旁边的装订区域中的图标可以从中导航。

    IntelliJ IDEA打开调用挂钩的文件，并将光标定位在调用方法中，在当前示例中为do_action 。

来自挂钩注册的回调

您可以导航到指定为钩子注册的第二个参数的函数或方法的声明（ add_action()或add_filter()函数）。

  • 按住Ctrl / ⌘键盘键，将光标悬停在感兴趣的参数上。IntelliJ IDEA显示一条弹出信息消息，其中包含指定为此参数的函数或方法的定义：

    该参数变为链接。单击此链接时，IntelliJ IDEA将打开声明函数或方法的文件，并定位光标和声明。

导航到挂钩调用

在“ 搜索文件中的目标”和“ 源代码”导航中了解有关搜索和导航的详细信息。

搜索一个钩子调用并导航到它

  • 执行以下操作之一：

    • 使用“ 导航到符号”功能：

      1. 选择导航|主菜单上的符号 。

      2. 在出现的输入符号名称弹出窗口中，开始在搜索字段中键入钩子名称。如有必要，请选中“ 包括非项目项”复选框。搜索字段下方列表的内容会在您键入时更改。

      3. 单击列表中的相关挂钩。IntelliJ IDEA打开调用它的文件，并将光标定位在调用位置。

    • 使用Search Everywhere功能：

      1. 按两次Shift键盘键。

      2. 开始在搜索字段中键入钩子名称。如有必要，请选中“ 包括非项目项”复选框。搜索字段下方列表的内容会在您键入时更改。

      3. 单击列表中的相关挂钩。IntelliJ IDEA打开调用它的文件，并将光标定位在调用位置。

搜索挂钩注册功能

您可以搜索挂钩注册函数的出现以及指定为挂钩注册的第二个参数的函数或方法的出现。

了解有关在项目中搜索用法的更多信息。

列出挂钩注册功能的用法

  1. 将光标定位到所需挂钩注册功能的名称，然后按Alt + F7或选择编辑|找到|在主菜单上找到用法 。

  2. 从弹出列表中，根据挂钩/功能名称显示可用选项，选择相关选项。将打开“ 查找工具”窗口，其中包含一个新选项卡，显示所选检测到的所选功能的所有发生

  3. 浏览列出的事件。要导航到相关的，请单击它。IntelliJ IDEA使用所选的功能或方法打开文件，并将光标定位在该文件上。

列出指定为挂钩注册的第二个参数的函数或方法的出现次数

  1. 将光标定位在感兴趣的参数上，然后按Alt + F7或选择编辑|找到|在主菜单上找到用法 。将打开“ 查找工具”窗口，其中包含一个新选项卡，显示指定为调用搜索参数的函数或方法的所有检测到的事件。

  2. 浏览列出的事件。要导航到相关的，请单击它。IntelliJ IDEA使用所选的功能或方法打开文件，并将光标定位在该文件上。

查看IntelliJ IDEA的官方WordPress文档

您可以直接从IntelliJ IDEA编辑器查看http://wordpress.org/上的官方WordPress文档 。

  1. 选择您感兴趣的文本。

  2. 从选择的上下文菜单中，选择在WordPress.org上搜索 。

    IntelliJ IDEA在默认浏览器中打开包含相关文档的页面。
