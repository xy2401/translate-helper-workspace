IntelliJ IDEA 2019.1帮助

在Zeus服务器上执行测试

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

基本

运行测试套装有时会涉及开销，因为每次执行测试套件时，都应重新加载整个环境（例如，应用程序的整个环境）。您可以使用只加载环境一次的Zeus服务器来避免它。

Zeus服务器在后台运行，您可以选择使用Zeus服务器或本地执行任何测试脚本。

使用Zeus服务器运行测试脚本

  1. 确保使用IntelliJ IDEA启动Zeus服务器并在后台运行。

  2. 运行测试脚本或其中一个示例。请注意， Zeus选项会在相应的运行配置中自动选择。如果要在本地运行此测试，则必须选择“ 无 ”选项。

如果您要在Zeus服务器下运行RSpec测试，请确保使用Zeus版本0.13.4.pre2或更高版本。

要调试测试，使用Zeus时，请遵循以下常规步骤

  1. 确保在要调试的测试脚本中设置断点。

  2. 在主工具栏上，单击运行/调试配置选择器，然后选择“ 编辑配置” 。

  3. 在“ 运行/调试配置对话框”对话框中，为Zeus服务器创建运行/调试配置。有关该过程的详细说明，请参阅创建和编辑运行/调试配置一节。

  4. 在调试模式下启动Zeus。为此，选择Zeus run / debug配置，单击 ，或按Shift + F9 。

  5. 在Zeus服务器以调试模式运行时，运行所需的测试脚本。

请注意，如果您调试测试脚本，它将在没有Zeus服务器的情况下执行。

技巧和窍门

  • IntelliJ IDEA为Zeus服务器创建临时运行/调试配置。稍后您可以根据需要更改此运行/调试配置，将其保存为永久性，并使用它来启动服务器。

  • 如果Zeus和Spork DRb服务器同时运行，那么Zeus就会获得优先权。
