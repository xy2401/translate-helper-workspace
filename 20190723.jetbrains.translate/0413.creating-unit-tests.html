创建单元测试

在Android中，单元测试基于JUnit ， JUnit的平面使用足以测试专门基于Java代码的功能。

但是，要测试特定于Android的功能，您需要在JUnit之上构建一堆包装类。IntelliJ IDEA简化了围绕Android测试项目构建的大部分任务。

1。确保您的代码是可测试的

单元测试要求源代码的组合方式使得模块之间的依赖关系可以通过模拟轻松抵消。此外，单元测试要求功能彼此很好地隔离。

因此， HelloDroid类的代码不容易测试。在我们进行单元测试之前，让我们首先应用快速重构。

  1. 打开HelloDroid类并选择TapDroid方法中引用显示消息生成的代码部分：

  2. 以这种方式重写TapDroid方法，以便它调用新创建的公共帮助器方法（ GetStringFor Display ），如下所示：

    private void TapDroid （） { counter ++; String temp = getStringForDisplay （ counter ）; 消息 。 的setText（ 字符串 格式 （ “你碰了机器人％S”， 温度 ）。）; } 公共 字符串 getStringForDisplay（ 诠释 计数 ）{ 字符串 温度 ; switch （ count ） { case 1 ： temp = “once” ; 打破 ; 案例 2 ： temp = “两次” ; 打破 ; default ： temp = String 。 格式 （ “％d次” ， 计数 ）; } return temp ; }

getStringForDisplay方法现在更容易测试，并且TapDroid方法的主体已经大大简化了。

2。创建一个测试模块

现在让我们创建一个新的测试模块并将HelloDroid设置为测试模块。这可确保测试模块保存对包含您要测试的源代码的模块的引用。

  1. 从主菜单中，选择File |新的|模块推出新模块向导 。

  2. 在向导的第一页上，在左侧窗格中选择Android ，在右侧选择Test Module ：

  3. 在第二页上，指定新模块名称，例如“ Tests 。保持其他字段不变。HelloDroid模块自动指定为测试模块，此时，这是项目中唯一的模块。

新节点将附加到名为Tests的项目中。该模块有自己的manifest文件和src目录。manifest文件链接到android.test库以构建测试类。

新创建的模块在src文件夹中有一个名为HelloDroidTest的测试文件。您可以通过添加更多Java类来添加更多测试文件，如下所示：

公共 类 HelloDroidTest 扩展 ActivityInstrumentationTestCase2 < HelloDroid > { public HelloDroidTest （） { super （ “com.example。HelloDroid“ ， HelloDroid 。 类 ）; } }

测试类继承自ActivityInstrumentationTestCase2<T> ，其中T是您要测试的活动的名称。

请注意，添加构造函数是必需的，因为没有为父类定义的默认构造函数。

3。添加测试方法

在编辑器中，右键单击HelloDroid测试类，然后单击Generate （或者，单击Alt + Insert ）。从打开的弹出菜单中，选择“ 测试方法” ：

IntelliJ IDEA创建一个名为testName的新方法存根，您可以在其中轻松地将Name后缀更改为在上下文中更有意义的内容：

如果您使用的是Android中的默认测试框架JUnit 3 ，则需要方法名称中的test前缀。使用JUnit 4 ，您必须使用方法名称注释来指示必须将给定方法作为测试方法处理。

4。编写测试方法的逻辑

在内部，测试方法首先获得对其尝试测试的活动的引用，然后调用目标方法，最后将有效结果与预期结果进行比较。

将以下代码添加到测试方法：

public void testStringForDisplay （） throws Exception { int count = 1 ; HelloDroid activity = getActivity （）; 字符串 结果 = 活动 。 getStringForDisplay （ count ）; 断言 。 assertEquals （ 结果 ， “一次” ）; }

断言是通过JUnit框架的服务实现的，需要在源文件中正确引用。弹出意图操作以引用它时按Alt + Enter ：

5。为测试创建运行/调试配置

要运行测试，您需要创建专用的运行/调试配置。设置测试模块时会自动为您创建默认配置。

要编辑其设置，请在主菜单中选择“运行”|在左窗格中编辑配置并选择Android 测试下的测试 ：

您可以选择在模块中运行所有测试，或将测试限制为给定类中的方法。

6。运行测试

要运行测试，请确保在编辑器右上角的下拉列表中选择了相应的运行/调试配置，然后单击“ 运行”按钮在它的旁边：

测试结果显示在自动激活的“运行”工具窗口的“ 测试运行器”选项卡中。如果测试成功完成，编辑器右上角会出现绿色方块图标。如果有警告，则图标为黄色，如果测试失败，则为红色。您可以单击该图标以获取更多详细信息。

您可以通过单击“ 导出测试结果”图标将测试报告导出为各种格式在“ 测试”选项卡工具栏中 。
