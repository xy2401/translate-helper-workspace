监控和管理测试

测试进度和结果显示在“ 运行”工具窗口的专用测试 运行器 选项卡中。

哪里：

  1. 运行工具栏与“ 运行”工具窗口几乎相同，但具有特定于测试的按钮。

  2. 左侧窗格显示当前运行/调试配置中所有测试的树视图。

    • 根节点表示选择运行的测试。

    • 嵌套节点表示测试套件和测试用例的层次结构。

    • 叶节点代表各个测试。

    每个测试的状态由图标指示。双击节点以在编辑器中打开相应的测试类或测试方法。

  3. 测试工具栏提供的控件使您可以监视测试并分析结果。一些命令在测试树节点的上下文菜单上重复。

  4. 输出窗格显示当前测试套件的输出。

  5. 内联统计信息显示执行测试的列表以及每个测试的执行时间。

您可以像运行应用程序一样重新运行 ， 终止和暂停执行测试。除了常见的运行操作，您还可以在测试运行器中：

  • 使用箭头键浏览测试用例列表。

  • 使用。在导航失败的测试之间导航和按钮或Ctrl + Alt +向上或Ctrl + Alt +向下键盘快捷键。

  • 查看当前会话中正在运行的测试总数。

  • 按字母顺序排序测试并按持续时间 。

  • 单击显示或隐藏有关传递的测试的信息 。

  • 通过单击，在当前运行/调试配置或测试类中的所有测试的树视图中显示被忽略的测试 。

  • 通过单击“ 输出”窗格中的超链接，从堆栈跟踪导航到源代码中的问题位置。

  • 该图标打开一个附加菜单，允许您：

    • 监视当前测试的执行情况。

    • 在完成诉讼后自动选择第一个失败的测试。

    • 导航到堆栈跟踪。

    • 启用自动滚动到源代码。

    • 在例外情况下打开相应的源代码。

    • 显示内联统计信息。
