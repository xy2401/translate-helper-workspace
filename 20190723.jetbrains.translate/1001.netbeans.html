使用项目

如何在IntelliJ IDEA中打开NetBeans项目？

使用文件|新的|从现有源项目中选择并选择NetBeans项目目录。

“ 导入项目”向导打开后，选择“ 从现有源创建项目”选项，然后按照向导的说明进行操作。

IntelliJ IDEA会将必要的定义文件（ .idea目录）添加到项目目录中。NetBeans .nbproject目录和build.xml将保持不变，您将能够将IntelliJ IDEA与NetBeans一起使用。

在导入期间，IntelliJ IDEA将修复缺少的库，为不同的Web框架添加构面并创建运行配置。

如果您将Maven与NetBeans一起使用，并且想要将Maven项目导入IntelliJ IDEA，请选择File |打开然后选择项目的pom.xml 。您仍然需要配置运行配置，但是，所有项目依赖项都将得到解决。

项目和模块之间有什么区别？

IntelliJ IDEA为整个代码库创建一个项目，并为每个组件创建一个模块。因此，IntelliJ IDEA模块更像是一个NetBeans项目。

IntelliJ IDEA中是否有基于目录的项目格式？

是的，有一个.idea目录，其中存储了项目定义XML文件。有关更多信息，请参阅项目 。

如何更改项目的JDK？

  1. 打开“ 项目结构”对话框（“ 文件”|“项目结构”或“ Ctrl + Shift + Alt + S” ）。

  2. 在“ 平台设置”下 ，选择“ SDK” 。

  3. 点击 ，选择JDK并指定JDK安装目录。

  4. 单击“ 应用”

  5. 在项目设置下 ，选择项目 。

  6. 在Project SDK下 ，从列表中选择JDK。

  7. 单击确定 。

有关更多信息，请参阅SDK 。

如何将库添加到项目中？

  1. 打开“ 项目结构”对话框（“ 文件”|“项目结构”或“ Ctrl + Shift + Alt + S” ）。

  2. 在“ 项目设置”下 ，选择“ 库” 。

  3. 点击 ，选择Java并指定库位置。

  4. 选择将使用此库的模块。

  5. 单击确定 。

有关更多信息，请参阅使用库 。

如何为我的项目配置Web框架？

在IntelliJ IDEA Ultimate中（Community Edition中没有相应的功能）：

  1. 打开项目工具窗口（例如View | Tool Windows | Project ）。

  2. 右键单击必要的模块，然后选择“ 添加框架支持” 。（在模块级别启用了框架支持。）

  3. 在打开的“ 添加框架支持”对话框中，选择要支持的框架，指定关联的设置，然后单击“ 确定” 。

有关更多信息，请参阅添加框架和技术 ，例如，为现有模块启用JSF支持 。

“运行”按钮已禁用。如何运行我的应用程序？

“运行”按钮已禁用，因为项目中没有运行配置 。

如果您有一个带有main()方法的Java类，请在编辑器中打开相应的文件，右键单击编辑区域并选择Run'<FileName> .main（）' 。因此，将自动创建必要的运行配置，然后执行。

您也可以自己创建运行配置：在主菜单中，选择Run |编辑配置| 。点击添加新配置并选择运行应用程序的方式。

如何为项目生成Ant构建脚本？

选择Build |生成Ant构建 。有关更多信息，请参阅生成Ant构建文件 。

如何同时在IntelliJ IDEA中打开多个项目？

使用IntelliJ IDEA可以同时处理多个项目。要实现此目的，您只需打开一个项目，而另一个项目已经打开，并选择添加到当前打开的项目 。

如何关闭项目？

选择文件|关闭项目 。您也可以使用File |退出以关闭所有打开的项目并退出IntelliJ IDEA。

选项对话框在哪里？

在IntelliJ IDEA中，“ 设置”对话框用于类似目的。要打开此对话框，请按Ctrl + Alt + S.

还有Project Structure对话框（ Ctrl + Shift + Alt + S ），它允许您管理JDK，库，模块依赖项等。

有关更多信息，请参阅设置/首选项对话框和项目结构对话框 。

如何从VCS集成开始？

IntelliJ IDEA支持最流行的版本控制系统，包括Git，Subversion，Mercurial，Perforce，CVS，TFS等。可以在“ 设置/首选项”对话框的“ 版本控制”页面上配置项目的VCS集成。请参阅IntelliJ IDEA详细信息的版本控制 。

使用代码编辑器

我可以在IntelliJ IDEA中使用NetBeans密钥绑定吗？

是的你可以。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为”下的“ 键盘映射 ”。

  2. 在对话框的右侧，在Keymaps旁边，从列表中选择NetBeans 6.5 。

IntelliJ IDEA中的代码完成如何工作？

键入一个或两个字母后，代码完成建议列表会自动出现。要缩小此列表，请使用：

  • Ctrl +空格 。该列表简化为关键字以及当前上下文中可用的类，方法和字段的名称。请注意，当您按Ctrl + Space第二次或第三次时，列表会更改。

  • Ctrl + Shift + Space 。仅显示适合当前上下文的类型。

有关更多信息，请参阅代码完成 。

IntelliJ IDEA中的本地历史记录与NetBeans中的本地历史记录有何不同？

IntelliJ IDEA中的本地历史通常更详细。无论您对目录，文件，类，方法或字段或代码块执行什么操作都会反映在您的本地历史记录中。本地历史记录还包括VCS操作。

有关更多信息，请参阅本地历史记录

IntelliJ IDEA中是否有任何特殊的代码分析功能？

IntelliJ IDEA可以分析依赖关系，数据流和堆栈跟踪，查找重复项并评估代码质量。只需查看Analyze菜单中的选项即可。

我可以在IntelliJ IDEA中启用“标记出现”吗？

您可以。IntelliJ IDEA中的相应选项称为高亮显示插入符号元素的用法 。默认情况下启用此选项。

以防万一：

  1. 打开“ 设置”对话框（例如Ctrl + Alt + S ）。

  2. 在编辑器类别中，选择常规 。

  3. 在对话框的右侧，在“插入运动的突出显示”下，选中“ 在插入符号处突出显示元素的使用”复选框。

  4. 单击确定 。

我可以在IntelliJ IDEA中启用“保存时编译”吗？

您可以。

要在每次保存（或自动保存）时启用自动编译， 请在“ 设置”对话框中启用“自动生成项目”选项：

  1. 打开“ 设置”对话框（例如Ctrl + Alt + S ）。

  2. 在“ 构建，执行，部署”类别中，选择“ 编译器” 。

  3. 在对话框的右侧，选择“ 自动生成项目”复选框。

  4. 单击确定 。

另请注意，默认情况下，IntelliJ IDEA会自动保存更改的文件，因此您不需要像在其他IDE中那样频繁地使用Ctrl + S.

我可以在IntelliJ IDEA中启用“在保存时部署”吗？

IntelliJ IDEA设置中没有此选项，但是，通过在相应的运行配置中选择适当的应用程序更新选项，可以获得类似的结果。

有关更多信息，请参阅更新Application Server上的应用程序 。

（相应的功能仅在IntelliJ IDEA Ultimate中可用。Community Edition不提供与应用程序服务器的集成。）

使用插件

我可以在IntelliJ IDEA中使用NetBeans插件吗？

不幸的是。但是，IntelliJ IDEA“开箱即用”中提供了许多作为NetBeans插件实现的功能。此外，还有很多IntelliJ IDEA插件，因此您总能找到一个IntelliJ IDEA插件，其功能类似于您喜欢的NetBeans插件。

我如何找到我需要的插件？

与使用插件相关的所有功能都在“ 设置”对话框的“ 插件”页面上（ Ctrl + Alt + S |插件 ）。您可以查找，下载，安装和更新插件，以及启用和禁用它们。

有关更多信息，请参阅插件和管理插件 。

如何安装我的计算机上可用的插件？

  1. 打开“ 设置”对话框（例如Ctrl + Alt + S ）。

  2. 在左侧窗格中，选择“ 插件” 。

  3. 在“ 插件”页面的下半部分，单击“ 从磁盘安装插件” 。

  4. 在打开的对话框中，选择插件文件（通常是JAR或ZIP）。

  5. 单击确定 。

  6. 如果询问，请重新启动IntelliJ IDEA。

我想为IntelliJ IDEA编写一个插件。有没有说明？

是的，看看：

  • IntelliJ Platform SDK开发人员指南 。

  • 有关插件开发人员的信息，请参见IntelliJ IDEA插件页面

  • 开放API和插件开发

是否可以使用IntelliJ IDEA构建NetBeans RCP应用程序？

但是，您可能无法获得与NetBeans（向导，菜单操作等）相同的支持。查看使用IntelliJ IDEA进行NetBeans平台开发 。
