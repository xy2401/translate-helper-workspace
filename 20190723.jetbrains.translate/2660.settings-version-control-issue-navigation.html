PyCharm |偏好|版本控制|发布 macOS 导航
按Ctrl + Alt + S

使用此对话框可以创建所谓的问题导航模式列表 。问题导航模式将提交消息中的问题ID模式与引用问题的URL地址进行映射。这使您可以从已提交的更改导航到与这些更改相关的问题。一旦PyCharm在提交消息中遇到与问题ID模式的匹配，匹配就会在版本控制工具窗口中显示为链接。如果您提到几个问题，所有问题都将显示为链接。单击此链接可在默认浏览器中打开匹配问题。

描述

问题

此只读字段显示问题模式。

链接

此只读字段显示了从当前行中的问题模式导航到错误跟踪系统中的问题的链接。

单击此按钮可创建新问题导航模式和链接。将打开“ 添加问题导航链接”对话框，您可以在其中指定：

  • 用于定义问题ID的正则表达式。

  • 用于定义问题的导航链接的正则表达式。

有关在正则表达式中使用特殊字符的详细信息，请参阅正则表达式语法参考 。

单击此按钮可创建新的JIRA模式。将打开“ 创建JIRA问题导航模式”对话框，您可以在其中指定JIRA安装的URL。定义模式的正则表达式会自动添加。

单击此按钮可为YouTrack创建新模式。在打开的对话框中，指定YouTrack安装的URL。定义模式的正则表达式会自动添加。

单击此按钮可更新选定的问题导航链接。

单击此按钮可从列表中删除选定的问题导航链接。

例

下面的示例显示了PyCharm如何应用上述规则来检测提交消息中对问题的引用，并在问题跟踪系统中编写指向它的链接。

问题ID模式

正则表达式，用于定义在提交消息中引用问题的格式。

[AZ] + \ - \ d +
此正则表达式匹配由两个以n-dash字符分隔的子字符串组成的所有字符串：

  1. 子串1：无限数量的大写字母字符。

  2. 子串2：无限数量的数字字符。

问题链接模式

问题跟踪系统的URL地址和标识其中问题的正则表达式的组合。

HTTP：// mytracker /问题/ $ 0
这里$0表示对整个比赛的反向引用。这意味着只要PyCharm在提交消息中检测到匹配，就会将其添加到跟踪器的URL地址中。

匹配问题ID

PyCharm在感兴趣的提交消息中检测到以下对问题的引用：

MYPROJECT-110

组成问题链接

根据上述问题导航模式，检测到的匹配参考按原样添加到跟踪器的URL，因此引用问题的链接组成如下：

HTTP：// mytracker /问题/ MYPROJECT-110
