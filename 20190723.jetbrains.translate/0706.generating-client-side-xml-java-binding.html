IntelliJ IDEA 2019.1帮助

生成客户端XML-Java绑定

此功能仅在Ultimate版本中受支持。

要从客户端到目标Web服务开发格式良好且有效的请求，您需要知道Web服务的可用方法，它使用的数据类型，服务的接口，可接受的请求格式，格式生成的回复等所有这些数据都显示在目标Web服务的WSDL描述符中。

IntelliJ IDEA可以基于所需的WSDL描述符生成必要的客户端XML-Java绑定，从而在开发客户端请求时为您提供有效的编码帮助。您只需要指定WSDL描述符的URL地址，IntelliJ IDEA将检索必要的数据并生成Java类。

Java代码生成在“从Wsdl或Wadl生成Java代码”对话框中配置，主要在启用Web服务客户端开发支持时打开。

配置客户端XML-Java绑定的生成

  1. 通过执行以下操作之一， 从Wsdl或Wadl对话框中打开“ 生成Java代码” ：

    • 创建一个模块并启用其中的Web服务客户端支持 。

    • 在开发过程中的任何时候，在Project视图中选择所需的客户端模块，然后选择WebServices |在上下文菜单中从Wsdl或Wadl生成Java代码 。

  2. 在Web Service WSDL URL字段中，指定所需Web服务WSDL描述符的URL地址。

    如果Status read only字段通知您WSDL URL连接异常，请确保目标Web服务正在运行且其WSDL描述符的URL地址是正确的。

  3. 从“ 输出路径”下拉列表中，选择用于放置生成的文件的目录。

  4. 在“ 包前缀”下拉列表中，指定要将已编译的Java类放入的包。

  5. 对于Apache Axis Web服务的客户端，请使用以下字段为代码生成过程指定其他配置选项：

    • 类型映射版本

    • 生成TestCase

    • 为模式数组生成类

    • 生成未引用的元素

    • 支持包装文档/文字样式
