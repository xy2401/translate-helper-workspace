I18nize硬编码字符串

Alt + Enter - i18nize硬编码字符串文字

使用此对话框将硬编码的字符串文字提取到指定的.properties文件。启用硬编码字符串检查时，此意图操作可用。

项目

描述

属性文件

在此文本框中，指定.properties文件以存储提取的字符串文字。手动键入文件的路径，或单击“ 浏览” 打开“ 选择属性文件”对话框，您可以在其中使用项目树视图或按名称搜索来选择所需位置。在您键入搜索字符串时，建议列表会缩小以仅显示匹配的属性文件。

更新资源包中的所有属性文件

选中此复选框可更新目标包中的所有属性文件。

属性键

默认情况下，此文本框根据要提取的字符串的值显示建议的键名称。接受默认名称或键入所需的名称。

适当的价值

默认情况下，此字段显示要提取的字符串的值。接受默认值或键入所需的值。

资源包表达式

默认情况下，此字段显示源代码中资源包声明的资源包表达式。如果未在源代码中声明资源包，则该字段将显示红色的无效值。要改善这种情况，请定义所需的表达式。执行以下操作之一：

  • 输入ResourceBundle类型的表达式，如使用java.util的 硬编码字符串文字部分所述。ResourceBundle实用程序类。

  • 使用自己的自定义实用程序类。

此字段中提供了基本代码完成（ Ctrl + Space ）。

编辑i18n模板

单击此链接以打开“ 文件和代码模板”对话框，您可以在其中更改I18nized Expression模板以指向将用于访问资源包的自定义实用程序类的方法。

更改的文件模板是影响所有项目的全局设置。如果要恢复默认值，请打开“ 文件和代码模板”对话框，在“ 代码”选项卡中找到I18nized Expression模板，然后单击“ 重置”按钮 。
