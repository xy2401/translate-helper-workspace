调试工具窗口

查看|工具窗口|调试
Alt + 5

开始调试时，此工具窗口可用。

它显示应用程序的调试会话生成的输出。如果要调试多个应用程序，则每个应用程序的输出将显示在以相应的运行/调试配置命名的单独选项卡中。

对于每个应用程序，都有以下嵌套选项卡：

  • 控制台 ：显示系统信息和错误消息，以及应用程序的控制台输入和输出。

  • 调试器 ：此选项卡分为以下几个区域：

    • 框架 / 线程

    • 变量

    • 手表

  • 转储 ：在获取线程转储时打开在调试器工具栏上启用了选项。

  • 元素 ：如果您使用Chrome浏览器进行调试，则会显示。

每个区域都有一个上下文菜单 ，允许您配置其行为并在选项卡之间导航。

每个选项卡和区域都可以隐藏/恢复 ，或移动到您选择的位置。

  • 显示方法返回值 ：选择此选项可显示上次执行的方法的返回值。

  • 自动变量模式 ：如果希望IntelliJ IDEA调试器自动评估某些变量（断点处的变量加上断点之前和之后的几行），请选择此选项。

Pin标签

步进工具栏

项目

工具提示和快捷方式

描述

显示执行点
Alt + F10

单击此按钮可突出显示编辑器中的当前执行点，并在“ 帧”窗格中显示相应的堆栈帧。

跳过
F8

单击此按钮执行程序，直到当前方法或文件中的下一行，跳过当前执行点（如果有）引用的方法。如果当前行是方法中的最后一行，则执行步骤到此方法之后执行的行。

踏入
F7

单击此按钮可使调试器步入当前执行点调用的方法。

如果通过“ 设置”对话框的“ 步进”页面（例如，如果它是标准Java SDK类或简单的getter）来抑制单步调用被调用的方法，则将跳过该方法。更改设置或使用Force Step Into命令。

强行进入
SHIFT + ALT + F7

单击此按钮可使调试器步入当前执行点中调用的方法，即使要跳过此方法也是如此。

走出去
按住Shift + F8

单击此按钮可使调试器从当前方法中退出，直到执行后执行的行。

丢帧

中断执行并返回到方法执行的初始点。在此过程中，它会从堆栈中删除当前方法帧。

运行到光标
Alt + F9

单击此按钮可恢复程序执行并暂停，直到执行点到达编辑器中当前光标位置的行。不需要断点。实际上，在插入符号处为当前行设置了临时断点，一旦程序执行暂停，该断点就会被删除。因此，如果插入符号位于已经执行的行，则程序将刚刚恢复以进一步执行，因为无法回滚到先前的断点。当您深入了解方法序列并需要一次性退出多个方法时，此操作尤其有用。

如果在将您带到指定行之前为应执行的行设置了断点，则调试器将在第一个遇到的断点处暂停。

当在特定行需要一种临时断点时，请使用此操作，不应中断程序执行。

评估表达
Alt + F8

单击此按钮可打开“ 评估表达式”对话框。

跟踪当前流链

当调试器在Stream API调用链之前或之内停止时，单击此图标以获取每个转换中每个元素发生情况的可视化（有关详细信息，请参阅分析Java Stream操作 ）。

恢复“框架”视图

恢复“手表”视图

单击此按钮可使“ 手表”区域可见。单击后，此按钮变为可用 。

恢复“线程”视图

恢复'变量'视图

移动标签和区域

如果您对“调试”工具窗口的默认布局不满意，则可以始终移动选项卡和区域。为此，只需将标签或区域拖动到所需位置即可。可能的目标会突出显示。

将选项卡或区域拖放到突出显示的位置。

要恢复选项卡和区域的默认布局，请单击在调试工具栏中。

选项卡的上下文菜单

使用“ 框架 / 线程” ，“ 变量”或“ 手表”区域的上下文菜单配置这些区域的行为或在选项卡之间导航。
