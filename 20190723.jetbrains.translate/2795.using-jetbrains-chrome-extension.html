PyCharm 2019.1帮助

使用JetBrains Chrome扩展程序进行调试

您不再需要安装和使用适用于Chrome的JetBrains IDE支持扩展来调试PyCharm中的应用程序。但是，如果要在同一个Chrome实例中启动调试会话，仍可以使用它。默认情况下，PyCharm会打开一个包含自定义Chrome用户数据的新窗口，请参阅使用默认Chrome用户数据启动调试会话 。

在使用JetBrains Chrome扩展程序进行调试会话期间，可以使用HTML，CSS和JavaScript进行实时编辑 。您还可以查看DOM树和实际页面的源代码。

安装JetBrains Chrome扩展程序

  1. 在Chrome Web Store中打开JetBrains IDE支持页面。

  2. 点击添加到Chrome按钮 ，然后在打开的“ 确认新扩展”对话框中单击“ 添加” 。“ 添加到Chrome”按钮更改为“ 添加到Chrome” 。

使用JetBrains Chrome扩展程序启用实时编辑和调试

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 构建，执行，部署”下的“ 调试器 ”，然后单击“ 实时编辑” 。“ 实时编辑”页面将打开。

  2. 选择“ 使用JetBrains IDE支持”扩展以进行调试和“实时编辑”复选框。

激活，取消激活和卸载JetBrains Chrome扩展程序

  • 通过chrome：//扩展页面提供对JetBrains Chrome扩展程序的控制。要打开该页面，只需在Chrome地址栏中输入chrome://extensions即可。或者，点击自定义并控制Google Chrome （ ），从上下文菜单中选择“设置” ，然后在打开的chrome：//设置页面上单击“ 扩展 ”。

  • 要取消激活扩展，请清除已启用复选框。复选框名称将更改为“ 启用” 。

  • 要激活扩展，请选中“ 启用”复选框。

  • 要卸载扩展程序，请点击从Chrome移除按钮 。

在Chrome扩展选项中指定实际的PyCharm端口

在调试会话期间， Chrome扩展程序会侦听从中调用扩展程序的JetBrains IDE的端口。包括PyCharm在内的每个IDE都有自己的默认端口。

如果由于某种原因，默认的PyCharm端口已经忙，PyCharm会找到最近的可用端口并从中启动。这会导致冲突：PyCharm正在“新”端口上运行，而Chrome扩展程序仍然会侦听先前启动的产品的端口。

当您使用Live Edit启动调试会话时，冲突显示：扩展无法通过默认端口连接，PyCharm等待来自扩展的连接，并显示以下消息及其实际运行的端口号（例如， current port 63342 ）：

  1. 右键点击并在上下文菜单中选择“ 选项 ”。带有Chrome扩展程序选项的网页会打开，显示连接到PyCharm的参数。

  2. 在“ IDE连接”区域中，在“ 端口”旋转框中指定实际的PyCharm端口。

覆盖默认的CORS设置

假设您使用Live Edit调试的页面请求一个资源，出于安全原因，该资源可以防止通过CORS设置进行访问。您可以通过更改Chrome扩展程序选项来启用对受保护资源的访问权限。

  1. 右键点击并在上下文菜单中选择“ 选项 ”。带有Chrome扩展程序选项的网页会打开，显示连接到PyCharm的参数。

  2. 在“ 强制CORS”文本框中，键入定义要使其可访问的URL地址的模式，例如：http： http://youtrack.jetbrains.com/rest/* 。
