创建一个空的Python项目

以下内容仅在安装并启用Python插件时有效！

要创建一个空项目：

  1. 在主菜单上，选择“ 文件”|新的|单击项目或在欢迎屏幕上 ，单击新建项目 。

    “新建项目”对话框打开。

  2. 在“ 新建项目”对话框的左侧窗格中，选择Python作为项目类型。

  3. 在右侧窗格中，指定项目SDK。如果列表中缺少所需的SDK，请单击“ 新建”并创建以下Python SDK之一：

    在“ 添加Python解释器”对话框的左侧窗格中，选择“ Virtualenv环境” 。以下操作取决于之前是否存在虚拟环境。

    如果选择了新环境 ：

    1. 在文本字段中指定新虚拟环境的位置，或单击并在文件系统中查找位置。请注意，新虚拟环境所在的目录必须为空！

    2. 从列表中选择基本解释器，或单击并在您的文件系统中找到基本解释器。

    3. 如果要继承全局站点包目录，请选中“ 继承全局站点包”复选框。此复选框对应于virtualenv工具的--system-site-packages选项。

    4. 如果需要， 请选中“ 使所有项目可用”复选框。

    如果选择了现有环境 ：

    1. 指定所需的解释器：使用列表，或单击并在您的文件系统中找到一个。

    2. 如果需要，选中“ 使所有项目可用 ”复选框。

    单击“ 确定”完成任务。
    在“ 添加Python解释器”对话框的左侧窗格中，选择“ Pipenv” 。

    如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径：

    单击“ 确定”完成任务。

    如果您尚未修改PATH变量，IntelliJ IDEA将显示错误消息： 找不到Pipenv可执行文件 。发现pipenv安装过程中描述的正确可执行路径，并在Pipenv可执行文件字段中输入目标字符串，例如： C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts \ pipenv.exe （Windows）或/Users/jetbrains/.local/bin/pipenv（macOS ）。

    单击“ 确定”保存更改并完成任务。

    在“ 添加Python解释器”对话框的左侧窗格中，选择“ Conda环境” 。以下操作取决于之前是否存在Conda环境。

    如果选择了新环境 ：

    1. 在文本字段中指定新Conda环境的位置，或单击并在文件系统中查找位置。请注意，新Conda环境所在的目录必须为空！

    2. 从列表中选择Python版本。

    3. 在文本字段中指定Conda可执行文件的位置，或单击并在Conda安装目录中找到位置。

    4. 如果需要， 请选中“ 使所有项目可用”复选框。

    如果选择了现有环境 ：

    1. 指定所需的解释器：使用列表，或单击并在您的文件系统中找到一个（例如， C：\ Users \ jetbrains \ Anaconda3 \ python.exe ）。

    2. 如果需要，选中“ 使所有项目可用 ”复选框。

    单击“ 确定”完成任务。
    在“ 添加Python解释器”对话框的左侧窗格中，选择“ 系统解释器” 。在“ 解释器”字段中，键入所需解释器可执行文件的完全限定路径，或单击在打开的Select Java Interpreter对话框中，选择所需的Java可执行文件，然后单击OK 。请注意，您需要管理员权限才能安装，删除和升级系统解释程序包。尝试通过意图操作安装解释程序包时，您可能会收到以下错误消息：根据提示，请考虑为项目使用虚拟环境。单击“ 确定”完成任务。
    在“ 添加Python解释器”对话框的左侧窗格中，单击“ SSH解释器” 。在右侧窗格中，选择“ 新建服务器配置” ，然后指定服务器信息（主机，端口和用户名）。或者，您可以选择“ 现有服务器配置”，然后从列表中选择任何可用的部署配置。如果需要，请单击查看所选部署配置的“ 连接”设置 ，“ 映射 ”和“ 排除”路径 。单击“ 下一步”继续配置解释程序。

    在下一个对话框窗口中，提供身份验证详细信息以连接到目标服务器。

    选择密码或密钥对（OpenSSL或PuTTY）并输入您的密码或密码。

    单击“ 下一步”继续执行最终配置步骤。

    在下一个对话框窗口中，验证所需Java解释的路径。您可以接受默认值，也可以指定其他默认值。您必须配置本地项目和服务器之间的路径映射。为此，请单击在“ 同步文件夹”字段旁边，输入本地项目文件夹的路径以及远程服务器上文件夹的路径。

    您还可以选择最低复选框以启用对远程服务器的本地更改的自动上载。

    在“ 添加Python解释器”对话框的左侧窗格中，单击“ Vagrant” ：

    单击“浏览”按钮在Vagrant实例文件夹字段旁边，并指定所需的Vagrant实例文件夹。

    这导致显示到Vagrant主机URL的链接。

    Python解释器路径字段显示所需Python可执行文件的路径。您可以接受默认值，也可以指定其他默认值。单击确定 。配置的远程解释器将添加到列表中。

    在打开的对话框中，选择Docker选项，从下拉列表中选择Docker服务器（如果缺少服务器，单击New ... ），并指定映像名称。

    Python解释器路径应该具有默认值，例如python ：

    单击“ 确定”完成任务。

    在打开的对话框中，选择Docker Compose选项，从下拉列表中选择Docker服务器，Docker Compose服务（此处为web ），配置文件（此处docker-compose.yml ）和图像名称（此处为python ）。

    接下来，等待IntelliJ IDEA启动Docker-Compose配置进行扫描和索引：

    单击“ 确定”完成任务。

    要添加任何远程解释器，请确保已安装并启用了SSH Remote Run ， Vagrant和Docker Integration插件。如果未激活插件 ，请按照管理插件中的说明在“设置/首选项”对话框的“ 插件”页面上启用它们。

  4. 如果需要，请选择其他库和框架以启用项目。选择特定框架时，请确保指定的项目SDK支持该框架。另外，提供所需的参数。

  5. 单击下一步 。

  6. 如果要将项目或框架模板用于新的Python项目，请在下一页上选择“ 从模板创建项目”复选框。

  7. 指定项目名称和位置。

  8. 单击“ 完成”以完成任务。
