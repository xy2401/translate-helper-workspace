部署：连接选项卡

使用此选项卡可选择访问Web服务器的方式并指定连接设置。控件集取决于所选的服务器访问类型。

所有服务器类型的通用设置

项目

描述

仅对此项目可见

使用此复选框可配置服务器访问配置或服务器组的可见性。

  • 选中该复选框以限制将配置或服务器组用于当前项目。此类配置或服务器组不能在当前项目之外重用。它不会出现在其他项目的可用配置列表中。例如，如果在SFTP配置中选中此复选框，则无法使用其SSH凭据来配置远程解释程序。

    服务器记录与项目一起存储在.idea目录中，允许通过VCS在团队成员之间共享它们。

    在左侧窗格的服务器访问配置列表中，仅在当前项目中可见的配置标有图标。

  • 清除该复选框后，配置或服务器组在所有WebStorm项目中都可见。其设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

有关设置解释程序的详细信息，请参阅配置Remote Node.js解释程序以获取详细信息。

类型

在此列表中，选择访问服务器的方式。可用选项包括：

  • FTP：选择此选项可让WebStorm通过FTP 文件传输协议访问服务器。

  • SFTP：选择此选项可让WebStorm通过SFTP文件传输协议访问服务器。

  • FTPS：选择此选项可让WebStorm通过SSL（ FTPS扩展）上的FTP文件传输协议访问服务器。

  • 本地或装入文件夹：如果Web服务器在本地或装入文件夹中运行且其文档根目录 不是项目根目录的父目录 ，请选择此选项。

  • 就地：如果Web服务器在您的计算机上运行，您的项目位于其文档根目录下，并且您直接在服务器上进行开发，请选择此选项。

Web服务器URL

在此字段中，指定Web服务器根文件夹 URL地址。点击在该字段中，以确保可访问指定的服务器根URL地址并指向正确的网页。

支持HTTP和HTTPS协议。

FTP和FTPS设置

项目

描述

主办

在此字段中，指定要将文件上载到的服务器的主机名。

港口

在此字段中，指定要使用的端口。默认值为21。

根路径

在此字段中，指定在注册帐户时定义的相对于用户主 目录的服务器配置根目录 。此文件夹将是通过当前服务器配置可访问的文件夹结构中的最高文件夹。

执行以下操作之一：

  • 接受默认/路径，该路径指向服务器上的用户主文件夹。

  • 手动键入路径。

  • 点击并在打开的“选择根路径”对话框中选择所需的文件夹。

  • 单击“ 自动检测”按钮，让WebStorm检测FTP / SFTP服务器上的用户主文件夹设置，并根据它们设置根路径 。只有在指定了用户名和密码后才能启用该按钮。

自动侦测

单击此按钮可让WebStorm检测服务器上的用户主文件夹设置，并相应地设置根路径 。

用户名

在此字段中，键入用于对服务器进行身份验证的用户名。

密码

在此字段中，键入用于对服务器进行身份验证的密码。

以匿名身份登录

选中此复选框以使用您的电子邮件地址作为密码启用对服务器的匿名访问 。

保存密码

选中此复选框可让WebStorm记住指定的密码。

测试连接

单击此按钮可检查指定的设置是否确保连接成功。只有在指定了用户凭据后才能启用该按钮。

FTP和FTPS高级设置

在“ 高级”区域中，通过为FTP / FTPS服务器配置指定其他特定于协议的选项来自定义上载/下载。

项目

描述

可以用来

TLS

选择调用客户端安全性的方法：

  • 显式 ：应用显式（活动）安全性 。建立连接后，计算机上的FTP客户端立即向服务器发送命令，通过默认FTP端口建立安全控制连接。

  • 隐式 ：应用隐式（被动）安全性 。在这种情况下，在建立与服务器的连接时自动提供安全性，该服务器指定用于安全连接的单独端口。

    请注意， Implicit方法被认为已弃用，因此不鼓励使用它。

FTPS

数据通道保护级别

从此列表中，为PROT命令设置保护级别：

  • <default> ：应用默认服务器策略。

  • 清除 ：原始数据在不应用安全性的情况下传输

  • 专用 ：应用传输层安全性（TLS）或安全套接字层（SSL）。

有关更多信息，请参阅有关TLS ， SSL 2.0和SSL 3.0的文章 。

FTPS

重用SSL会话

选中此复选框以连接到需要在控制和数据连接之间重用SSL会话的服务器。

FTPS

被动模式

选中此复选框可将计算机上的客户端设置为被动模式 ，连接到服务器以通知处于被动模式，接收要侦听的端口号，以及通过端口建立的数据连接和接收到的号码。当您的计算机位于防火墙后面时，此模式很有用。

FTP，FTPS

显示和处理隐藏文件

选中此复选框时：

  1. 隐藏文件和目录显示在远程主机工具窗口中 。

  2. 差异文件和目录涉及差异和同步操作。

隐藏文件或目录的名称以点（ . ）开头。

FTP，FTPS

使用LIST命令

使用此区域中的选项配置LIST命令用法。

  • 与旧版本的列表子项 兼容 ：选中此复选框以确保与FTP服务器的子文件命名兼容 。

    如果远程FTP服务器报告以下错误，此选项很有用：

    无效的 后代 文件 名 < 文件 名 >
    选择此选项可能会降低与服务器的同步速度。

  • 而不是MLSD ：选中此复选框以使用标准LIST命令列出而不是MLSD命令。这样可以避免出现问题，例如，如果FTP服务器支持MLSD并返回cdir ，则在上载过程中出现无效后代文件名异常的cdir 。

FTP，FTPS

并发连接限制

选中此复选框可让WebStorm限制同时支持的连接数，并指定字段中允许的最大连接数。

FTP，FTPS

检索文件时间戳

使用此列表指定MDTM FTP命令调用策略，以检索远程主机上给定文件的上次修改时间。

可用选项包括：

  • 始终 - 选择此选项可为远程主机工具窗口中显示的每个文件调用MDTM。

  • 在复制 - 选择此选项以在以下情况下调用MDTM：

    • 在“ 选项”对话框中的“ 覆盖最新文件”复选框被清除时，检查文件是否是最新的。

    • 在下载期间保留文件的实际时间戳。

  • 从不 - 选择此选项以禁止调用MDTM。

FTP，FTPS

每次发送保持活动消息

在此字段中，指定WebStorm向服务器发送命令以重置超时的频率，从而保留连接。

FTP，FTPS

保持活力命令

从此列表中，选择要发送到服务器的命令以重置超时，从而保留连接。

FTP，FTPS

用于客户端 - 服务器通信的编码

在此字段中，指定与服务器使用的编码匹配的编码。如果您不确定它是否支持UTF-8编码，请接受默认值。

FTP，FTPS

认证

从此列表中，选择客户端身份验证方法。可用选项包括：

  • 密码 - 选择此选项可通过密码使用标准身份验证。

  • 密钥对（OpenSSH或PuTTY） - 选择此选项以通过密钥对使用SSH身份验证 。

    要应用此身份验证方法，您必须在客户端计算机上拥有私钥，并在远程服务器上拥有公钥。WebStorm支持使用OpenSSH实用程序生成的私钥。

有关使用SSH密钥的详细信息，请参阅生成新的SSH密钥并将其添加到ssh-agent教程。

SFTP高级设置

在“ 高级”区域中，通过为SFTP服务器配置指定其他特定于协议的选项来自定义上载/下载。

项目

描述

并发连接限制

选中此复选框可让WebStorm限制同时支持的连接数，并指定字段中允许的最大连接数。

每次发送保持活动消息

在此字段中，指定WebStorm向服务器发送命令以重置超时的频率，从而保留连接。

用于客户端 - 服务器通信的编码

在此字段中，指定与服务器使用的编码匹配的编码。如果您不确定它是否支持UTF-8编码，请接受默认值。

忽略信息消息

在某些SFTP服务器上，可能会启用SSH横幅 。每次建立连接时，都会显示带有信息消息的弹出窗口，如果要继续，则需要单击“ 确定” 。

要禁止显示信息弹出窗口，请选中忽略信息消息复选框。

项目

描述

夹

在此字段中，指定服务器配置根 。

服务器配置根是服务器上文件树中可以通过服务器配置访问的最高文件夹。最简单的方法是使用Web服务器配置文件中定义的Web服务器的文档根目录。但是，您可以在文档根目录下指定任何其他现有文件夹。
