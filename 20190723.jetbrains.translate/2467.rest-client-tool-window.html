REST客户端工具窗口

工具| HTTP客户端|测试RESTful Web服务
查看|工具窗口| REST客户端

使用REST Client工具窗口测试RESTful Web服务 。该工具窗口用于基于服务API编写和向Web服务方法提交测试请求，以及查看和分析服务器响应。

请注意以下事项：

  • 查看|工具窗口| REST客户端 - 使用Tools |打开工具窗口后，可以通过这种方式访问工具窗口HTTP客户端|测试RESTful Web Service命令。

  • 仅当启用REST Client捆绑插件时，工具窗口才可用。该插件默认处于活动状态。如果没有，请在“ 设置”对话框的“ 插件”页面中将其激活。

在本页面：

  • 常见请求设置
    • 工具栏
  • 请求标签
  • 饼干标签
  • 响应选项卡
  • 响应标题选项卡

常见请求设置

在此区域中，选择请求方法并指定要从中组成请求URI的数据。

服务器响应代码和内容长度显示在REST客户端工具窗口的右上角。

描述

HTTP方法

在此下拉列表中，指定请求方法 。可用选项包括：

  • 得到

  • POST

  • 放

  • 补丁

  • 删除

  • 头

  • OPTIONS

主机/端口

在此文本框中，键入部署目标Web服务的主机的URL地址以及它侦听的端口。默认情况下，端口号为80 。如果使用其他端口，请以<host URL>:<port number>格式明确指定。

路径

在此下拉列表中，指定目标方法的相对路径。

您可以在主机/端口字段中输入要测试的方法的完整URL地址。无论选择哪种HTTP方法 ，按下时都是如此 PyCharm会将URL地址拆分为主机/端口和方法的路径。提取的相对路径将显示在“ 路径”字段中，提取的参数将添加到“ 请求”选项卡的“ 请求参数”窗格中的列表中。

项目

工具提示和快捷方式

描述

提交请求

单击此按钮可将生成的测试请求提交给服务器。

如果服务器不受信任，PyCharm会显示一个建议接受服务器的对话框，或拒绝它。如果您将服务器接受为受信任，则PyCharm会将其证书写入信任存储区。在下一次连接到服务器时，将不会显示此对话框。

重播最近的请求

单击此按钮可显示“ 最近请求”列表，并选择相关请求。字段将填入所选请求的设置。单击“ 提交请求”按钮 。

出口申请

单击此按钮可将当前请求设置保存在XML文件中，以便在另一个PyCharm会话中可用。在打开的对话框中，指定要保存设置的文件的名称及其父文件夹。必要时，您可以检索已保存的设置并再次运行请求。

进口申请

单击此按钮可以从XML文件中检索先前保存的请求的设置。在打开的对话框中，选择相关的XML文件。

生成授权标头

单击此按钮以打开“ 生成授权标头”对话框，并指定用于访问目标RESTful Web服务的用户名和密码。基于这些凭证，PyCharm将生成一个用于基本身份验证的身份验证标头 。欲了解更多信息， 请访问http://www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm 。

配置HTTP代理

单击此按钮可在打开的对话框中指定代理服务器设置。

关

关闭REST客户端工具窗口。

救命

显示此页面。

请求标签

使用此选项卡可以通过GET请求的查询字符串或其他请求类型的请求正文来指定要在生成的测试请求中传递给服务的参数。还可以通过指定服务和客户端接受的数据格式来配置客户端和Web服务之间的交互。

项目

描述

头

在此窗格中，指定请求标头中包含的技术数据。这些数据通过头字段传递，并定义输入参数（ 接受字段）的格式，响应格式（ 内容类型字段），缓存机制（ 缓存控制字段）等。

要向列表中添加字段，请单击“ 添加” ，然后在“ 名称”文本框中指定字段名称，并在“ 值”下拉列表中指定字段值。

字段集及其值应符合Web服务API。换句话说，指定的输入格式应该是Web服务所期望的格式，并且预期的响应格式应该是服务返回的格式。

对于accept ， content-type和其他一些字段，PyCharm提供了一个建议值列表。从“ 值”下拉列表中选择相关的格式类型。

请求参数

在此窗格中，通过URL中的查询字符串指定要传递给目标方法的参数。此方法用于GET类型的请求。默认情况下，窗格显示带有一行的空列表。

  • 要添加参数，请单击“ 添加” ，然后在“ 名称”文本框中指定参数的名称，并在“ 值”下拉列表中指定参数的值 。

  • 要从列表中删除参数，请选择该参数，然后单击“ 删除” 。

  • 要禁止发送指定的查询字符串参数并禁用“ 请求参数”窗格中的控件， 请按“不发送任何内容”切换按钮 。

参数集及其类型应符合Web服务API，特别是它们应与目标方法的输入参数完全相同。

  • 选中此复选框这可能会有所帮助，例如，如果要通过请求正文中的其他请求方法测试传递参数，但仍保留在“ 查询字符串参数”窗格中键入的数据。

请求机构

选择GET ， DELETE ， HEAD或OPTIONS请求方法时，将禁用该窗格。在此窗格中，指定要传递给请求消息正文中的目标方法的输入参数。

  • 清空：选择此选项以发送空体的请求。

  • 文本：选择此选项以使用带有值的参数字符串发送请求。在选项旁边的文本框中指定参数。

  • 文件内容：选择此选项可以从本地文本文件中将参数插入请求正文中。在“ 要发送的文件”字段中指定源文本文件。

  • 文件上载（multipart / form-data）：如果要将二进制文件作为参数传递，请选择此选项，这需要转换文件。在“ 要发送的文件”字段中指定源二进制文件。

饼干标签

使用此选项卡可以创建，保存，编辑和删除Cookie，这些Cookie都是通过响应接收并手动创建的。如果尚未达到到期日期 ，则cookie的名称和值将自动包含在与为cookie指定的域和路径匹配的URL地址的每个请求中。

该选项卡显示您通过响应收到或手动创建的所有当前可用cookie的列表。Cookie按照添加到列表中的顺序显示。单击cookie时，其详细信息将变为可编辑状态并显示在文本框中。

项目

描述

名称

在此字段中，指定要包含在请求中的cookie的名称。

值

在此字段中，指定要包含在请求中的cookie的值。

域

在此字段中，指定必须提供的请求的主机和端口以及cookie的名称和值 。

路径

在此字段中，指定必须提供的请求的URL路径以及cookie的名称和值 。

到期日

在此字段中，指定cookie的到期日期。

单击此按钮可向列表中添加新行并在其中定义新的苦力。

单击此按钮可从列表中删除选定的cookie。

响应选项卡

使用此选项卡可以查看Web服务的响应。默认情况下，响应以纯文本格式显示。使用选项卡的图标使它们以HTML，XML和JSON格式显示在编辑器中。

项目

提示

描述

以HTML格式查看

单击此按钮可在主编辑器窗口中打开新选项卡，并将响应显示为HTML。

以XML格式查看

单击此按钮可在主编辑器窗口中打开新选项卡，并将服务器响应显示为XML。

以JSON格式查看

单击此按钮可在主编辑器窗口中打开新选项卡，并在此处显示服务器响应为JSON。

在浏览器中打开

单击此按钮可在默认Web浏览器中查看响应。

响应标题选项卡

该选项卡显示Web服务响应标头中提供的技术数据。
