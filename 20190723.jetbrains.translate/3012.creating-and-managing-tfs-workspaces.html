创建和管理TFS工作区

TFS服务器和本地项目之间的交互是通过工作空间配置的。工作区主要将存储库中的文件夹及其副本映射到计算机上。

在WebStorm中，您可以配置对多个TFS服务器的访问权限，并根据需要在其下具有任意数量的工作空间。可用的TFS服务器访问配置和工作空间列表通过“ 管理TFS服务器和工作空间”对话框进行处理。

在本主题中：

  • 打开“管理TFS服务器和工作空间”对话框
  • 配置对TFS服务器的访问
  • 创建工作区

打开“管理TFS服务器和工作区”对话框

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。“ 管理TFS服务器和工作空间”对话框显示其中所有可用服务器和工作空间的列表。

配置对TFS服务器的访问

要放弃服务器访问配置，请在列表中选择服务器，然后单击“ Team Servers”区域中的“ Remove”按钮。

  1. 打开“管理TFS服务器和工作区”对话框，其中包含所有可用服务器和工作区的列表。

  2. 单击“ Team Servers”区域中的“ Add”按钮。

  3. 在“打开的添加Team Foundation Server”对话框中，在“ 地址”文本框中指定目标服务器的URL地址。

    在“身份验证”字段中，指定用于访问服务器的身份验证协议。TFS使用NTLM身份验证 ，因此本机Windows应用程序（即Microsoft Team Explorer ）使用系统凭据进行静默身份验证。由于Java Runtime构成的限制，WebStorm用户必须始终指定其用户名和密码。

    要通过OAuth进行身份验证（Windows Live ID ），请从“ 身份 验证”下拉列表中选择“ 备用 ”。

  4. 对于NTLM和备用身份验证类型，请指定您的凭据：

    • 对于NTLM ：TFS服务器所在的网络域，TFS用户名和TFS密码。

    • 对于备用 ：您的TFS用户名和密码。

  5. 单击确定 。 WebStorm返回“ 管理TFS服务器和工作空间”对话框，其中新服务器已添加到列表中。

创建服务器工作区

工作空间由其名称及其所有者的名称标识，包含计算机的名称，工作空间所属的服务器的URL地址，以及可通过工作空间访问的远程和本地工作文件夹之间的一组映射。

  1. 打开“管理TFS服务器和工作区”对话框，然后选择有问题的服务器。要刷新可用服务器工作空间的列表，请单击“ 重新加载”工作空间按钮。

  2. 单击“ 工作空间”区域中的“ 创建”按钮。

  3. 在打开的“ 创建工作空间”对话框中，指定工作空间名称。（可选）在“ 注释”文本框中提供工作区的简要说明。

  4. 在“ 工作文件夹”区域中，定义映射。

    1. 单击“ 添加”按钮 。新的行添加到映射列表中。

    2. 在“ 服务器路径”文本框中，指定需要使用的服务器上的文件夹。

    3. 在“ 本地路径”文本框中，指定用于存储下载数据的本地文件夹。

    4. 在“ 状态”下拉列表中指定映射的状态 。

      • 要根据映射启用从服务器检索数据，请选择“ 活动” 。

      • 要防止根据映射从服务器下载数据，请选择Cloaked 。

    5. 要放弃映射，请在列表中选择它，然后单击“ 删除”按钮 。
