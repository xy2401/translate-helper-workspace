远程调试

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Python插件时有效！

IntelliJ IDEA提供了两种远程调试方法：

  • 使用远程解释器
  • 使用Python调试服务器

使用远程解释器进行远程调试

如果使用远程Python解释器执行远程调试，则所有操作都在单个SSH连接中完成，并且不需要端口号。

要使用远程解释器远程调试，请遵循以下常规步骤

  1. 确保配置了远程解释器。

  2. 使用常规运行/调试配置启动调试过程 。

Python调试器在PyPi中可用，因此可以安装它以便使用pip进行远程调试。

在调试在另一台机器上运行的进程时，可以在IntelliJ IDEA的安装下debug-eggs pip install pydevd而不是从debug-eggs目录复制存档。

使用Python调试服务器进行远程调试

对于远程调试，IntelliJ IDEA提供带有pydev目录的存档。根据Python版本，这些存档是：

  • pycharm-debug.egg for Python版本2

  • Python 3的pycharm-debug-py3k.egg

此存档位于以下位置，具体取决于平台：

  • Windows：下.用户主目录的IntelliJIDEAXXXX.X\config\plugins\python目录。

  • * NIX：在.用户主页的IntelliJIDEAXXXX.X/config/plugins/python目录。

  • macOS：在/Users/jetbrains/Library/Application Support/IntelliJIDEAXXXX.X/python/目录下。

远程调试过程涉及以下一般步骤：

  • 配置远程调试服务器 。

  • 准备调试 。

  • 启动调试服务器 。

  • 从外部启动脚本 。

配置远程调试服务器

  1. 打开“ 运行/调试配置”对话框，如“ 创建运行/调试配置”一节中所述，然后选择“ Python远程调试”配置类型。

  2. 指定本地主机名以及将运行调试服务器的端口号。如果您未指定任何值，IntelliJ IDEA将随机提供端口号。

  3. 如果要调试驻留在远程计算机上的脚本，请指定远程脚本的源根目录以及要保留映射的本地源的根目录。

准备远程调试

  1. 在IntelliJ IDEA中，打开本地脚本进行编辑，并添加以下命令（您可以从远程调试配置对话框中复制它）：

    pydevd.settrace（<主机名>，端口= <端口号>）

    另外，将相应的import语句添加到脚本中：

    导入pydevd

  2. 将本地脚本复制到要调试它的远程位置。

  3. 包括pycharm-debug.egg存档。您可以通过多种方式执行此操作，例如：

    • 将存档添加到PYTHONPATH 。

    • 将存档附加到sys.path 。

    • 只需将pydev从存档复制到远程脚本所在的目录即可。

启动调试服务器

如果进程没有停止，但您仍想停止跟踪并断开与Python远程调试服务器的连接，请将以下函数添加到正在调试的远程脚本的末尾：

pydevd 。 stoptrace （）

此函数停止使用pydevd.settrace()启动的远程调试过程。因此，Python远程调试服务器将传递到等待新连接的状态。

  1. 选择所需的远程配置：

  2. 点击 ，或按Shift + F9 。IntelliJ IDEA在指定的主机和端口启动调试服务器，该调试服务器显示在“ 调试”工具窗口的“ 控制台”窗格中。

调试远程脚本

  1. 找到远程脚本，然后根据特定操作系统的要求启动它。例如，在Windows上，使用命令：

    python <script name>

    如果存在相应的本地脚本，IntelliJ IDEA将在挂起模式下在编辑器中打开它;第一个可执行语句标有蓝色条带。此外，IntelliJ IDEA在“ 调试”工具窗口中显示其第一个可执行语句的框架和变量。如果找不到相应的本地脚本，IntelliJ IDEA将打开一个专用的编辑器选项卡，并建议执行以下操作之一：

    • 在远程调试配置对话框中编辑本地和远程根。

    • 自动检测映射文件，然后从具有匹配名称的遇到文件列表中选择要用作本地版本的映射文件。

    • 将远程文件下载到指定位置。

  2. 通过脚本 ， 设置监视和评估表达式 。
