管理任务和上下文

在开始处理任务之前，请确保在“ 设置/首选项”中启用了“ 任务管理”插件插件窗口。

在处理项目时，您可以在需要完成的较小任务中组织工作。

这些可以是您自己设定的任务。在PyCharm中，您可以将大型任务划分为与专用更改列表关联的较小任务。

或者这些可以是来自您的问题跟踪器的任务。例如，您可以直接从PyCharm处理分配给您的任务和错误。为此，请首先在IDE和跟踪器帐户之间配置连接。

将IDE连接到跟踪器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|任务|服务器 。

  2. 点击按钮并从列表中选择所需的问题跟踪器。

  3. 输入连接细节。请注意，设置因问题跟踪器而异。

    通常，您必须指定服务器URL和连接凭据。

    选择“ 共享URL”选项以允许您的团队中的其他成员访问服务器。

    如果要通过代理服务器访问服务器，请单击“ 代理设置” 。您可以在HTTP代理部分中找到有关代理设置的更多信息。

    （ 可选 ）启用为更改列表添加提交消息并配置消息模板。

    对于某些跟踪器（例如，对于不支持开箱即用的跟踪器），您还必须配置服务器参数。指定用于访问任务的URL和请求类型，选择跟踪服务器响应的格式 - 用于XPath的XML，用于JSONPath的 JSON或用于正则表达式的文本。您还可以使用选择器表来指定要从服务器获取的任务的详细信息。例如，这可以是创建任务的日期或其URL。请注意，您可以在此窗口中使用代码完成。

如果您使用“慢”问题跟踪系统，则特别推荐同步。

指定其他集成选项

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|任务 。

  2. 配置必要的选项：

    • 更改列表名称格式 - 当您打开或创建新任务时，PyCharm将提示您创建与此任务关联的新更改列表 。在此字段中，您可以指定将用于为新更改列表生成名称的模板。使用{id} ， {summary} ， {number} ， {project}占位符。

    • 功能分支名称格式 - 当您创建或打开新任务时，PyCharm将提示您创建新功能分支。在此字段中，您可以配置模板以生成新功能分支的名称。使用{id} ， {summary} ， {number} ， {project}占位符。

    • 任务历史记录长度 - PyCharm将存储的任务数。

    • 在提交时保存上下文 - 每次提交更改时，PyCharm都会创建一个新的已关闭的本地任务，该任务将保留您使用过的文件。这样，您将能够在将来的任何时间快速恢复与该任务关联的所有选项卡。

    • 启用问题缓存 - 优化PyCharm与问题跟踪器之间的同步。

      PyCharm缓存从跟踪器加载的问题列表并重复更新它们。您可以指定应缓存的问题数量以及PyCharm应更新有关它们的信息的频率。

处理任务

在PyCharm中，有两种类型的任务。

从问题跟踪器加载到PyCharm的任务。这些是跟踪器任务跟踪器任务与问题跟踪器中的相应问题相关联。这允许您直接从PyCharm监视和更新它们。

最初在PyCharm中创建的任务。这些是本地任务 。本地任务与问题跟踪器无关。

如果您创建了任一类型的至少一个任务，则工具栏上将显示一个名为task combo的下拉列表。

打开跟踪器任务

将跟踪器任务连接到问题跟踪器后，跟踪器任务将加载到您的PyCharm。

  1. 转到工具|任务和背景|打开任务 （ Shift + Alt + N ），或单击工具栏上的任务组合。

  2. 从列表中选择所需的任务。

  3. 在“ 打开任务”对话框中，您可以更新问题状态。

    如果要关闭当前在编辑器中打开的所有选项卡，请选中“ 清除当前上下文”复选框。

  4. 在VCS操作部分中，您可以创建新的更改列表，选择要为其贡献的现有分支，或创建新的分支。

    您也可以搁置当前更改以便稍后返回。

创建本地任务

在PyCharm中，您可以创建不是源自问题跟踪器的本地任务。

  1. 转到工具|任务和背景|打开任务 。您也可以使用任务组合或只需按Shift + Alt + N.

  2. 在“ 输入任务名称”对话框中，选择“ 创建新任务” 。

  3. 输入新任务的名称。

    如果要关闭当前在编辑器中打开的所有选项卡，请选中“ 清除当前上下文”复选框。

  4. 在VCS操作部分中，您可以创建新的更改列表 ，选择要为其贡献的现有分支，或创建新分支。

    您也可以搁置当前更改以便稍后返回。

查看任务说明

选择要切换到的任务时，任务列表仅显示任务ID。这些信息并不总是足够的，因为它既不反映导致问题的步骤，也反映相关的讨论。

  1. 转到工具|任务和背景|打开任务 。

  2. 打开必要的任务，然后按Ctrl + Q在PyCharm中打开任务描述，或按Shift + Alt + B在浏览器中查看描述。

或者，转到工具|任务和上下文并单击显示“ 任务ID ”描述或在浏览器中打开“ 任务ID ” 。

查看已关闭的任务

如果整个项目或受影响的目录位于版本控制下，则关闭的本地任务是与更改列表无关的任务。

关闭的跟踪器任务是在问题跟踪器中具有关闭状态的任务。

  1. 单击任务组合，然后单击“ 打开任务” 。

  2. 选中包含已关闭的任务复选框，或按Shift + Alt + N.

关闭任务

  • 转到工具|任务和上下文 ，然后单击关闭活动任务 。

这将关闭PyCharm中的当前上下文。选择必要的复选框以提交更改，并可选择合并已创建的分支。对于跟踪器任务，您还可以更改其状态。新状态将传播到您的问题跟踪器。

删除任务

如果您不需要在PyCharm中显示任务，则可以将其从任务列表中删除。

  1. 单击主工具栏上的任务组合。

  2. 选择一个或多个要删除的任务。
    使用Shift （对于相邻项目）或Ctrl （对于非相邻项目）键进行多项选择。

  3. 单击右箭头按钮，然后选择“ 删除” 。

删除跟踪器任务时，将其从IDE中删除。它们将保留在您的问题跟踪器中。在这种情况下，本地任务将被完全删除，因为它们未连接到您的问题跟踪器。

时间跟踪

使用PyCharm，您可以跟踪在编辑器中工作的任务所花费的时间。对于本地任务，如果您想知道在处理项目时完成任务所需的时间，这些信息可能会有所帮助。

对于跟踪器任务，如果您的问题跟踪器配置要求您记录在任务上花费的时间，则此选项很有用。在这种情况下，您可以将时间日志从PyCharm发送到跟踪器。

请注意，时间跟踪功能仅在专业版中提供。

确保在“ 设置/首选项”中启用了“ 时间跟踪”插件插件窗口。

启用时间跟踪

  1. 在“ 设置/首选项”对话框中，转到“ 工具”|任务|时间跟踪 ，然后选中启用时间跟踪复选框。

  2. （可选）更改暂停延迟值。在这里，您可以指定在任务被视为暂停之前您必须保持非活动状态的时间。

在“时间跟踪”工具窗口中，使用“ 自动”模式 用于自动时间记录，或用于活动任务的启动计时器 和活动任务的停止计时器 手动时间记录的选项。

将时间日志发送到跟踪器

要将录制的时间日志发送到问题跟踪器：

  1. 单击“将工作项目发布到错误跟踪器” 在时间跟踪工具窗口中。

  2. 指定要记录的时间间隔。（可选）添加评论。

  3. 单击“ 确定”将日志推送到跟踪器。

使用上下文

上下文是与任务连接的一组文件。使用PyCharm，您可以保存和清除上下文，而无需将它们与特定任务相关联。

保存上下文

  1. 从主菜单中，选择工具|任务和背景|保存上下文 ，或按Shift + Alt + S.

  2. （可选）在“ 保存上下文”对话框中，根据需要指定注释。

要将新文件添加到现有上下文，请加载此上下文，打开要添加的文件，然后再次保存上下文。

在上下文之间切换

使用PyCharm，您可以在与特定任务无关的上下文之间切换。这将允许您处理任务并在它们之间切换，而不会混合两个任务之间所做的更改。

  1. 从主菜单中，选择工具|任务和背景|加载上下文 ，或按Shift + Alt + L.

  2. 在“ 加载上下文”弹出窗口中，从列表中选择必要的上下文。
    或者，单击向右箭头并选择“ 加载” 。

清除背景

  • 要清除当前上下文而不加载另一个上下文，请选择工具|任务和上下文|从主菜单中清除上下文 ，或按Shift + Alt + X.

删除上下文

任务完成后，或者您不再需要上下文时，可以将其删除。要删除上下文：

  1. 从主菜单中，选择工具|任务和背景|加载上下文 ，或按Shift + Alt + L.

  2. 在“ 加载上下文”弹出窗口中，单击向右箭头并选择“ 删除” 。
