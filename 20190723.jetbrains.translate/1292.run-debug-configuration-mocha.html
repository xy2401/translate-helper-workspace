运行/调试配置：Mocha

此功能仅在Ultimate版本中受支持。

运行|编辑配置| |摩卡

在此对话框中，使用Mocha测试框架创建用于运行和调试JavaScript单元测试的配置。

在你开始之前

  1. 下载并安装Node.js.

  2. 安装摩卡测试框架中描述摩卡 。

特定于Mocha的配置设置

项目

描述

Node.js解释器

在此字段中，指定要使用的Node.js解释器。选择一个配置的解释器或单击并配置一个新的，如配置本地Node.js解释器中所述 。如果选择Project别名，IntelliJ IDEA将自动使用Node.js和NPM页面上的Node解释器字段中的项目默认解释器 。

节点选项

在此字段中，键入要传递给Node.js可执行文件的Node.js特定命令行选项。有关详细信息，请参阅节点

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

环境变量

在此字段中，指定Node.js可执行文件的环境变量 （如果适用）。单击“ 浏览” 在该字段的右侧并在“ 环境变量”对话框中配置变量列表，该对话框将打开：

  • 要定义新变量，请单击“ 添加” 并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击“ 删除” 。

  • 准备好后，单击“ 确定”

变量的定义显示在“ 环境变量只读”字段中，并以分号作为分隔符。可接受的变量是：

  • NODE_PATH ：A :以模块搜索路径为前缀的NODE_PATH目录列表。

  • NODE_MODULE_CONTEXTS ：设置为1以在其自己的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

摩卡包

在此字段中，指定Mocha安装home /npm/node_modules/mocha 。如果您通过节点包管理器定期安装Mocha ，IntelliJ IDEA会自动检测Mocha安装。或者，手动键入可执行文件的路径，或单击“ 浏览” 并在对话框中选择打开的位置。

用户界面

从此下拉列表中，选择用于编写测试文件夹中的测试的接口 。IntelliJ IDEA将仅识别符合所选接口的测试。如果在测试执行期间IntelliJ IDEA针对另一个接口的测试运行，则测试会话将被取消并显示错误。这意味着必须根据相同的接口写入指定测试文件夹中的所有测试，并且必须从下拉列表中选择此接口 。

额外的摩卡选项

在此字段中，指定其他Mocha命令行选项 。例如，添加一个--watch标志来打开类似自动测试的跑步者 。因此，当前运行配置中的任何测试都会在更改相关源代码时自动重新启动，而无需单击“ 重新运行”按钮 。

测试

在此区域中，指定要执行的测试。可用选项包括：

  • 全部在目录中：选择此选项可以从存储在文件夹中的文件运行所有测试，并在“ 测试目录”字段中指定此文件夹的路径。要在指定目录下的子文件夹中查找测试，请选中“ 包含子目录”复选框。

  • 文件模式：选择此选项可在名称与特定掩码匹配的所有文件中查找测试，并在“ 测试文件模式”字段中指定掩码。

  • 测试文件：选择此选项仅从一个文件运行测试，并在“ 测试文件”字段中指定此文件的路径。

  • 套件：选择此选项可从测试文件运行单个套件，并在“ 测试文件”字段中指定此文件的路径。单击“ 套件名称”字段并配置要运行的套件列表。要将套件添加到列表，请单击并键入所需套件的名称。要删除套件，请在列表中选择它，然后单击

  • 测试：选择此选项可从测试文件运行单个测试，并在“ 测试文件”字段中指定此文件的路径。单击“ 测试名称”字段并配置要运行的测试列表。要将测试添加到列表，请单击并键入所需测试的名称。要删除测试，请在列表中选择它并单击

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

工具栏

运行/调试配置的树视图有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

进入新文件夹/创建新文件夹

您可以通过将运行/调试配置放入文件夹来对其进行分组。

要创建文件夹，请选择类别中的配置，单击 ，并指定文件夹名称。如果仅关注某个类别，则会创建一个空文件夹。

然后，要将配置移动到文件夹中，文件夹之间或文件夹之外，请使用拖放或和纽扣。

要删除分组，请选择一个文件夹并单击 。

排序配置

单击此按钮可按字母顺序对配置进行排序。

在启动选项之前

在此区域中，您可以指定在启动所选运行/调试配置之前必须执行的任务。任务按照它们在列表中出现的顺序执行。

项目

捷径

描述

Alt +插入

单击此图标可添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择要运行的一个或多个应用程序。如果尚未在IntelliJ IDEA中定义，请添加其定义。有关更多信息，请参阅配置第三方工具和外部工具 。

  • 运行另一个配置 ：选择执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • Build ：选择编译指定的模块。将执行Build Module命令 。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建项目 ：选择编译整个项目。Build Project命令将被执行。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建，无错误检查 ：与Build选项相同，但IntelliJ IDEA将尝试启动运行/调试配置，而不管编译结果如何。

  • 构建工件 ：选择此选项可构建工件或工件。在打开的对话框中，选择应构建的工件或工件。

  • 运行Ant目标 ：选择此选项以运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项以运行Grunt任务。在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项以运行Gulp任务。在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项以运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项可自动运行bundler，作为运行或调试会话的一部分。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过start npm任务运行开发服务器。为此，请单击 ，然后在Configure React Native对话框中，选择npm脚本并从列表中选择start 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上载到远程主机 ：选择此选项可根据默认服务器访问配置将应用程序文件自动上载到 服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：在运行或调试之前添加要执行的Rake任务 。要选择Rake任务，请单击“浏览”按钮 ，然后从可用任务列表中选择所需任务。

    请注意，此处可以使用代码完成 。

  • 运行JRuby编译器 ：选择此选项以使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。

Alt + Delete键

单击此图标可从列表中删除所选任务。

输入

单击此图标可编辑选定的任务。在打开的对话框中进行必要的更改。

/

Alt + Up / Alt + Down

单击这些图标可将所选任务在列表中向上或向下移动一行。（任务按照它们在列表中出现的顺序执行。）

显示此页面

选中此复选框可在实际启动运行/调试配置之前显示运行/调试配置设置。

激活工具窗口

默认情况下，选中此复选框，并在启动运行/调试配置时打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，当配置运行时，您可以通过按Alt + 4或Alt + 5 自行打开相应的工具窗口。
