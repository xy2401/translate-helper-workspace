安慰

本节中的设置与显示数据库控制台中的各种信息有关。

项目

描述

在新标签中显示查询结果

您可以选择在单个选项卡上查看查询结果，也可以在同一个选项卡上查看查询结果。

如果选中该复选框，则每次运行查询（ SELECT ）时都会打开带有查询结果的新选项卡。通过这种方式，您可以保留已运行的所有查询的结果。

如果未选中此复选框，则会使用相同的选项卡显示查询结果。运行查询时，选项卡上的信息将更新以显示结果，并且不会打开新选项卡。

在这种情况下，当您获得要保留的结果时，可以将选项卡固定到工具窗口（ 在工具栏上或选项卡的上下文菜单中的“ 引脚”选项卡上）。

在编辑器中显示错误通知

如果未选中此复选框，则仅在输出窗格中显示有关错误的信息。此外，如果您希望错误通知栏显示在输入窗格中，请选中该复选框。

运行SQL语句序列时，错误通知栏可能特别有用。如果在这种情况下发生错误，则错误通知栏允许您选择如何做出反应。

有关更多信息，请参阅使用错误通知栏 。

始终在执行前查看参数

当您使用参数运行语句时，IntelliJ IDEA会记忆参数值。每次下次执行语句时：

  • 如果启用此选项，IntelliJ IDEA将显示上次使用的参数值，以便您可以在实际运行语句之前更改它们。

  • 如果此选项关闭，IntelliJ IDEA会立即执行该语句，而不显示参数值。

跟踪数据库/模式的创建和删除

创建新架构或数据库，或删除架构或数据库时（请参阅例如创建数据库或模式 ）：

  • 如果启用此选项，则新模式或数据库将立即显示在“ 数据库”工具窗口中。删除的模式和数据库将立即从“ 数据库”工具窗口中的“ 模式”弹出窗口以及“ 数据源和驱动程序”对话框中“ 模式”选项卡上的列表中删除。

  • 如果禁用此选项，则不会显示新架构或数据库，除非您手动将其可视化，请参阅显示和隐藏架构 。除非取消选择，否则选定要查看的已删除模式和数据库将保留在相应的列表中。

在控制台中执行

本节包含Execute命令的选项。您最多可以分配三种不同的执行配置，每种配置都有自己的行为和快捷方式。

项目

描述

在语句内部执行时

如果光标位于语句内，则可以使用以下选项：

  • 询问执行什么。 显示可以运行的语句列表，您可以选择一个或多个语句。

  • 最小的陈述。 执行最小的可能语句。例如，当光标位于子查询中时，将执行子查询。

  • 最大的声明。 执行最大可能的语句。例如，当光标位于子查询中时，将执行外部语句。

  • 最大的声明或批次。 对于Transact-SQL（SQL Server和Sybase），将执行当前批处理语句。对于所有其他方言 - 与上一个选项相同。

  • 整个脚本。 所有的陈述都被执行了。

否则执行

如果光标位于语句之外，例如在空行或注释中，则可以使用以下选项：

  • 没有。 没有任何陈述被执行。

  • 整个脚本。 所有的陈述都被执行了。

  • 插入物以下的一切。 执行光标位置后的所有语句。

选择执行

如果当前选择了某些内容，则可以使用以下选项：

  • 正如一个声明。 确切地说，所选内容将作为单个语句执行。

  • 正如声明一样。 确切地说，执行了什么。如果选择包含多个语句，则语句将作为单独的语句执行。

  • 智能扩展到脚本。 如果选择中至少有一个语句边框，则扩展选择以形成一系列有效语句。然后执行该序列。

    否则，执行精确选择的内容。
