IntelliJ IDEA 2019.1帮助

内容根源

IntelliJ IDEA中的内容是一组包含源代码，构建脚本，单元测试和文档的文件。这些文件通常按层次结构组织。顶级文件夹称为内容根 。

模块通常有一个内容根。您可以添加更多内容根。例如，如果您的代码片段存储在计算机上的不同位置，这可能很有用。

同时，模块可以在没有内容根源的情况下存在。在这种情况下，您可以将它们用作其他模块的依赖项集合。

IntelliJ IDEA中的内容根目录标有图标。

要快速打开“项目结构”对话框，请单击在工具栏上。

添加新内容根

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ），然后单击项目设置|模块 。

  2. 选择所需的模块，然后打开对话框右侧的“ 源”选项卡。

  3. 单击“ 添加内容根”，然后指定要添加为新内容根的文件夹。单击确定

要删除内容根，请单击 。IntelliJ IDEA将选定的根标记为常规文件夹;文件夹本身及其内容不会被删除。

文件夹类别

内容根目录中的文件夹可以分配到多个类别。

来源根源

包含应编译的生产代码

生成的源根

IDE认为Generated Sources根文件夹中的文件是自动生成的，而不是手动编写的，可以重新生成。

测试源根

这些文件夹将与测试相关的代码与生产代码分开。源和测试源的编译结果通常放在不同的文件夹中。

生成的测试源根

IDE认为此文件夹中的文件是自动生成的，而不是手动编写的，可以重新生成。

资源根

与源类似，您可以指定生成资源。您还可以指定应将资源复制到的输出文件夹中的哪个文件夹。

（仅限Java）应用程序中使用的资源文件（图像，配置XML和属性文件等）。在构建过程中，资源文件将按原样复制到输出文件夹。

测试资源根

这些文件夹用于与测试源关联的资源文件。

排除

代码完成，导航和检查将忽略排除文件夹中的文件。这就是为什么当您排除目前不需要的文件夹时，可以提高IDE性能。

通常，编译输出文件夹被标记为已排除。

除了排除整个文件夹外，您还可以排除特定文件 。

加载路径根

（仅限Ruby）加载路径是require和load语句查找文件的路径。

配置文件夹类别

  1. 在“项目”工具窗口中右键单击文件夹。

  2. 从上下文菜单中选择标记目录 。

  3. 选择必要的类别。

这样，您也可以为子文件夹分配类别。

要还原文件夹的上一类别，请再次右键单击此文件夹，选择“将目录 标记为” ，然后选择“ 取消标记为<文件夹类别>” 。对于排除的文件夹，请选择取消排除 。

您还可以在“ 项目结构”中配置文件夹类别模块|来源 。

排除文件

如果您不需要特定文件，但又不想完全删除它们，则可以暂时从项目中排除这些文件。代码完成，导航和检查会忽略排除的文件。

不能排除Java文件和二进制文件。

要排除文件，您需要将其标记为纯文本文件。您始终可以将排除的文件恢复为原始状态。

  1. 右键单击“项目”工具窗口的目录树中的必要文件。

  2. 从菜单中选择标记为纯文本 。

纯文本文件标有目录树中的图标。

要还原更改，请右键单击该文件，然后从菜单中选择标记为<文件类型> 。

按名称模式排除文件和文件夹

在某些情况下，逐个排除文件或文件夹是不方便的。例如，如果源代码文件和自动生成的文件（例如编译器）放在同一目录中，并且您只想排除生成的文件，则这可能不方便。在这种情况下，您可以为特定内容根配置一个或多个名称模式。

如果位于所选内容根目录内的文件夹或文件名与其中一个图案匹配，则会将其标记为已排除。所选内容根目录之外的对象不会受到影响。

排除文件夹中的所有文件也将被排除。

  1. 从主菜单中，选择File |项目结构 ，或按Ctrl + Shift + Alt + S.

  2. 单击“ 项目设置”部分下的“ 模块 ” ，然后选择一个模块。如果此模块中有多个内容根，请选择要从中排除文件或文件夹的内容。

  3. 在位于对话框底部的“ 排除文件”字段中，输入模式。例如，输入*.aj以排除AspectJ文件。

    您可以配置多个模式并将它们与; （分号）符号。

为Java源分配包前缀

在Java中，您可以将包前缀分配给文件夹，而不是手动配置文件夹结构。可以将包前缀分配给源文件夹，生成的源文件夹，测试源文件夹和生成的测试源文件夹。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ），然后单击模块 。

  2. 选择所需的模块，然后打开“ 源”选项卡。

  3. 在右侧窗格中，单击 源文件夹或测试源文件夹旁边。

  4. 指定包前缀，然后单击“ 确定” 。

更改资源的输出路径

在构建项目时，默认情况下会将资源复制到编译输出文件夹中。您可以在输出文件夹中指定其他目录以放置资源。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ），然后单击模块 。

  2. 选择所需的模块，然后打开“ 源”选项卡。

  3. 在右侧窗格的“ 资源文件夹”或“ 测试资源文件夹”下 ，单击“确定” 在必要文件夹（文件夹路径）的右侧。

  4. 指定相对于输出文件夹根目录的路径，然后单击“ 确定” 。
