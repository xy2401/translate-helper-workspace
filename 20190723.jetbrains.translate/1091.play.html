如果您只想使用IntelliJ IDEA来运行play命令行实用程序（也称为Play控制台），则可以安装和使用Play框架的2.xx版本。

播放框架支持概述

请注意，IntelliJ IDEA版本11支持Play框架版本1.x.更高版本的IntelliJ IDEA支持Play 2.x版本。此外，对于IntelliJ IDEA版本14.0，Play 2.x框架与Scala插件合并。

IntelliJ IDEA中的Play框架支持包括：

  • 能够在IDE中使用play命令行实用程序（Play控制台）。

  • 编码协助：

    • 在Application.java的方法和相应的模板之间导航。

    • 通过相应Application.java方法的变量更新模板。

    • 关键字，标签，变量，参数和函数的代码完成 。

    • 错误和语法突出显示。

    • 代码格式和折叠 。

  • 高级源代码搜索和替换功能 。

  • 结构视图 。

准备Play应用程序开发

准备Play应用程序开发：

  1. 下载并安装IntelliJ IDEA支持的所需Play框架版本。请参阅安装说明 。

  2. 创建一个新的Play应用程序 （ play new 应用程序 ）或获取现有应用程序 （您将在IntelliJ IDEA中使用的应用程序）。

  3. 准备您的应用程序，以便在IntelliJ IDEA中打开它（ play idea或play idealize ）。

  4. 确保在IntelliJ IDEA中正确指定了Play框架设置 。

  5. 在IntelliJ IDEA中打开应用程序 。

  6. （可选）将项目转换为基于目录的格式 。

创建一个Play应用程序

要创建新的Play应用程序，请在IntelliJ IDEA外部的命令行shell中运行play new命令：

  1. 打开命令行shell。

  2. 切换到要在其中创建Play应用程序的目录。

  3. 运行以下命令：
    玩新的<dir>

    其中<dir>是将在当前目录中创建的新Play应用程序目录（例如helloworld ）的名称。

  4. 当询问应用程序名称时，键入名称并按Enter键 。

因此，将在指定的目录中创建具有指定名称的Play应用程序。

准备Play应用程序以在IntelliJ IDEA中打开它

要准备Play应用程序以在IntelliJ IDEA中打开它，请从Play应用程序的根目录运行play idea或play idealize命令：

  1. 打开命令行shell。

  2. 切换到Play应用程序的根目录。

  3. 运行以下命令：
    发挥想法

    （或者，您可以使用play idealize命令。）

因此，所有必需的IntelliJ IDEA配置文件都在当前目录中创建。这些包括<app_name>.ipr项目文件，您现在可以在IntelliJ IDEA中打开它 。

指定Play框架设置

在IntelliJ IDEA中，Play框架设置在“ 播放配置”页面的“设置”对话框中指定。

  1. 打开“ 设置”对话框（例如Ctrl + Alt + S ）。

  2. 在对话框的左侧窗格中，选择“ 播放配置” 。

  3. 在对话框右侧打开的“ 播放配置”页面上：

    • 在Home字段中，指定Play框架安装目录。

    • 在“ 工作目录”字段中，指定Play框架工作目录。这是运行play命令行实用程序命令的目录。

    • 根据您的偏好，打开或关闭“在控制台上运行”选项。

  4. 在Settings对话框中单击OK 。

在IntelliJ IDEA中打开Play应用程序

要在IntelliJ IDEA中打开Play应用程序，您应该打开先前生成的<app_name>.ipr项目文件：

  1. 选择文件|打开 。

  2. 在“ 打开文件或项目”对话框中，转到“播放应用程序”根文件夹，选择<app_name>.ipr文件，然后单击“ 确定” 。

在打开Play应用程序的项目中，由IntelliJ IDEA模块表示。

如有必要，将项目转换为基于目录的格式 。

另一种为Play应用程序创建IntelliJ IDEA项目的方法

在IntelliJ IDEA中开始使用Play应用程序的最快捷，最方便的方法是运行play idea命令 ，然后在IntelliJ IDEA中打开生成的.ipr文件 。

或者，您可以使用File |为Play应用程序源创建一个新项目新的|来自Existing Sources的项目 ，然后将必要的Play框架资产（ <play_dir>\framework\lib和<play_dir>\framework\play-<version>.jar ）添加到生成的IntelliJ IDEA模块的依赖项中：

  1. 选择文件|新的|来自现有来源的项目 。

  2. 在打开的对话框中 ，选择Play应用程序根目录。

  3. 在“ 导入项目”向导的第一页上，选择“ 从现有源创建项目” ，然后单击“ 下一步” 。

  4. 在向导的下一页的“ 项目位置”字段中，指定Play应用程序根目录的路径。其余设置并不重要。单击下一步 。

  5. 按照向导的说明进行操作。（通常，您只需在接受默认设置的每个页面上单击“ 下一步 ”。）

    创建项目后，添加必要的模块依赖项：

  6. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  7. 在对话框的左侧窗格中，选择“ 模块” 。

  8. 在对话框右侧显示的Module页面上 ，选择Dependencies选项卡 ，单击并选择Jars或目录 。

  9. 在Attach Files或Directories对话框中，转到Play框架安装目录，选择目录framework\lib和文件framework\play-<version>.jar ，然后单击OK 。

  10. 在Project Structure对话框中单击OK 。

访问IntelliJ IDEA中的播放命令行实用程序

如果当前在IntelliJ IDEA中打开了Play应用程序，则可以访问play命令行实用程序（Play控制台）并从IDE直接运行它：

  1. 选择工具|玩Playframework 。

  2. 在Play Configuration对话框中，指定Play框架设置，然后单击OK 。 （如果在Play框架设置中禁用“在控制台上显示运行”选项，则不会显示此对话框。）

因此， play命令行实用程序在“ 运行”工具窗口中启动。

将Play模块添加到IntelliJ IDEA模块

Play应用程序使用的Play模块应作为模块内容的根目录添加到相应的IntelliJ IDEA模块中。例如，如果您的应用程序使用（或即将使用）Play模块secure ：

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 在对话框的左侧窗格中，选择“ 模块” 。

  3. 在对话框右侧显示的“ 模块”页面上 ，选择“ 源”选项卡 ，然后单击“ 添加内容根” 。

  4. 在“ 选择内容根目录”对话框中，转到“播放框架”安装文件夹，选择目录modules\secure ，然后单击“ 确定” 。（如果您想要的Play模块未包含在Play框架发行版中，则应首先下载该模块以使其在本地可用。）

  5. 在Project Structure对话框中单击OK 。

运行Play应用程序

要运行播放应用程序，可以使用play run命令：

  1. 启动播放命令行实用程序 （ 工具|使用Playframework播放 ）。

  2. 在“ 运行”工具窗口中， play后，键入run并按Enter键 。

  3. 打开Web浏览器并转到http://localhost:9000以查看应用程序主页。

作为替代方案，您可以创建IntelliJ IDEA 运行/调试配置 ，并使用该配置来运行您的应用程序。

要创建运行Play应用程序的运行配置：

  1. 打开“ 运行/调试配置”对话框（例如“运行”|“编辑配置” ）。

  2. 点击 （ Alt + Insert ）并选择Application 。

  3. 在以下字段中，指定：

    • 主要课程。 play.server.服务器

    • VM选项。 键入-Dapplication.path="."

    • 工作目录。 指定Play应用程序根目录。

  4. 如有必要，请更改运行配置名称，然后单击“ 确定” 。

现在，要执行此运行配置，您可以使用Run |跑 。（有关更多信息，请参阅运行应用程序 。）

运行Play应用程序测试

要为Play应用程序运行测试，可以使用play test命令：

  1. 启动播放命令行实用程序 （ 工具|使用Playframework播放 ）。

  2. 在“ 运行”工具窗口中，在play后，键入test并按Enter键 。

  3. 打开Web浏览器并转到http://localhost:9000/@tests以运行测试。

为了能够使用IntelliJ IDEA运行/调试配置来运行测试：

  • 将<play_dir>\modules\testrunner\lib\play-testrunner.jar到模块的依赖项中。

  • 修改运行应用程序的运行配置 ：在VM选项字段中，在-Dapplication.path="."之后-Dapplication.path="." ，键入space然后键入-Dplay.id=test

    因此，最后， VM选项字段将包含：

    -Dapplication.path = “”。 -Dplay.id =测试

    （如果要为测试单独运行配置，可以创建现有运行配置的副本（ ）然后相应地修改该副本。）

调试Play应用程序：流程概述

  1. 创建运行/调试配置以进行调试 。这应该是Remote类型的配置;此配置中指定的端口应对应于Java调试器端口（默认情况下为8000 ）。

  2. 在代码中设置一个或多个断点 。见断点 。

  3. 如果要使用IntelliJ IDEA运行配置来启动应用程序，请修改相应的运行配置 ，或创建新配置 。

  4. 使用play run命令（不带任何选项和参数）或执行相应的运行配置来运行应用程序。

  5. 启动用于调试的运行/调试配置。

  6. 在Web浏览器中，转到http://localhost:9000然后使用该应用程序尝试访问断点。

  7. 到达断点时，切换到IntelliJ IDEA并仔细检查可疑代码片段。

  8. 除非您在代码中本地化问题，否则继续调试。

另请参阅“ 不要暂停代码执行” 。

创建运行/调试配置以进行调试

  1. 打开“ 运行/调试配置”对话框（例如“运行”|“编辑配置” ）。

  2. 点击 （ Alt + Insert ）并选择Remote 。

  3. 在“ 端口”字段中，指定8000 。（这是Java调试器将连接的端口。默认情况下，Play框架使用端口8000 ）

  4. 如有必要，请更改运行/调试配置名称，然后单击“ 确定” 。

修改运行应用程序的运行配置

如果您使用IntelliJ IDEA 运行配置来运行Play应用程序 ，则必须修改此配置，以便Java调试器可以连接到正在运行的应用程序。这是通过添加相应的JVM选项来完成的。可以从用于调试的运行/调试配置中复制必要的选项。

  1. 打开“ 运行/调试配置”对话框（例如“运行”|“编辑配置” ）。

  2. 在对话框的左侧窗格中，选择用于调试的运行/调试配置。

  3. 点击在包含JVM命令行参数的上部字段的右侧。结果，字段内容被复制到剪贴板。

  4. 选择运行应用程序的运行配置，然后将选项粘贴到VM选项字段中。该字段最终应包含-Dapplication.path="." ，然后是一个空格，然后是你刚刚粘贴的选项。这是一个例子：
    -Dapplication.path = “”。 -agentlib：JDWP =运输= dt_socket，服务器= Y，暂停= n时，地址= 8000

    （如果使用相同的运行配置来运行应用程序和测试 ，则可能需要删除-Dplay.id=test 。）

  5. 单击确定 。
