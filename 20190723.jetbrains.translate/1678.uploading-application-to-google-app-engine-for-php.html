IntelliJ IDEA 2019.1帮助

运行，调试和上传应用程序到Google App Engine for PHP

所描述的功能由Google App Engine支持PHP插件处理。该插件不再维护，与IntelliJ IDEA 2018.1及更高版本不兼容。

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

IntelliJ IDEA提供了在PHP Development Server上本地运行和调试应用程序的可能性，然后再将它们上传到Google PHP Runtime Environment 。应用程序根据专用的App Engine for PHP运行/调试配置在本地启动。

在这个部分：

  • 使用app.yaml文件
  • 创建App Engine for PHP类型的运行配置
  • 在PHP开发服务器上运行应用程序
  • 在PHP开发服务器上调试应用程序
  • 将应用程序上载到Google PHP运行时环境

使用app.yaml文件

在App.yaml文件中配置了Google App Engine PHP应用程序。如果生成项目存根（请参阅准备为PHP应用程序开发Google App ），则会自动创建app.yaml文件。如果您在现有项目中激活Google App Engine支持，则需要手动创建app.yaml文件。在任何一种情况下，请确保runtime属性设置为php55 ，因为Google App Engine for PHP SDK包含内置的PHP 5.5解释器。

创建App Engine for PHP类型的运行配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择App Engine for PHP 。运行/调试配置：PHP的App Engine打开。

  2. 指定运行开发服务器和应用程序的主机（默认为localhost ）。

  3. 指定IntelliJ IDEA与开发服务器通信的端口（默认端口为8080 ）。

  4. （可选）在“ 命令行”区域中，在命令行模式下指定在PHP开发服务器上运行和调试应用程序的设置：

    • 在Interpreter选项字段中，指定要传递给内置PHP解释器的PHP可执行文件的选项，有关详细信息，请参阅命令行参数 。

    • 在“ 自定义工作目录”字段中，指定包含源的文件夹外部的文件的位置，并通过相对路径引用。手动键入路径或单击“ 浏览” 并在打开的对话框中选择所需的文件夹。

    • 在Yaml files字段中，指定要使用的.yaml配置文件。此字段是可选的，当您的应用程序包含多个模块时使用它，因此使用了几个.yaml配置文件。

    • 在“ 环境变量”字段中，在此字段中，指定要传递给内置服务器的环境变量 。有关详细信息，请参阅Apache中的环境变量 。

在PHP开发服务器上运行应用程序

  1. 如上所述， 为PHP运行配置创建App Engine 。

  2. 从列表中选择配置，然后单击在工具栏上。

  3. 在“ 运行”工具窗口的控制台中查看和分析应用程序的输出。

  4. 要查看应用程序执行结果，请在http://localhost:8080打开浏览器。

在PHP开发服务器上调试应用程序

  1. 如上所述， 为PHP运行配置创建App Engine 。

  2. 在代码中设置断点，请参阅断点 。

  3. 从列表中选择配置，然后单击

  4. 在打开的调试工具窗口中， 逐步执行程序 ，停止并恢复程序，暂停时检查程序等。有关详细信息，请参阅单步执行程序 ， 暂停和恢复调试器会话以及检查暂停程序 。

将应用程序上载到Google PHP运行时环境

通过在开发服务器上本地运行和调试应用程序来测试应用程序后，可以将其部署到Google PHP运行时环境 。

  1. 在主菜单上，选择“ 工具”|适用于PHP的Google App Engine |上传App Engine PHP应用 。

  2. 访问http://<your-application-id>.appspot.com/ 。
