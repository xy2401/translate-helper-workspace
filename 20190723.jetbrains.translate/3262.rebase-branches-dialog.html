Rebase分支对话框

VCS | Git |变基

使用此对话框指定分支到rebase，新基础，变基模式以及配置变基程序。

项目

描述

Git Root

从此下拉列表中，选择要在其中重新分支分支的本地存储库的路径。

科

从此下拉列表中，选择要转换的分支。

默认情况下，选择当前工作分支。如果指定另一个分支，它将首先自动检出。

互动

选中此复选框可查看并可能编辑要重新定位的提交列表。

保留合并

选中此复选框可以重新创建合并而不是忽略它们。仅当选中“ 交互”复选框时，该复选框才可用。

选中此复选框后，Git不支持压缩提交。

在

使用此下拉列表指定所选分支的新基础。要指定所需的提交，请键入其提交哈希或使用表达式，例如，以下结构：
<branch>~<number of commits backwards between the latest commit (HEAD) and the required commit> 。
有关详细信息，请参阅Git 提交命名约定。

如果未指定提交，则将所选分支的HEAD用作新基础。

验证

单击此按钮以检查Onto字段中指定的提交是否存在，并查看其中受影响的文件。

从

使用此下拉列表指定要将分支应用于新基础的提交。键入所需的提交哈希或使用表达式，例如，以下结构：
<branch>~<number of commits backwards between the latest commit (HEAD) and the required commit> 。
有关详细信息，请参阅Git 提交命名约定。

要应用整个分支，请将该字段留空。

验证

单击此按钮以检查“ 发件人”字段中指定的提交是否存在，并查看其中受影响的文件。

显示标签

选中此复选框可以在Onto和From下拉列表中包含标记的提交 。

显示远程分支

选中此复选框可在远程存储库中包含Onto下拉列表中的分支。

合并策略

从此下拉列表中，选择合并策略 。可用选项包括：

  • 默认

  • 解析 - 如果需要解析两个HEAD，请选择此选项，其中一个是当前分支，另一个HEAD是从中提取更改的分支。选择此选项后，将应用3向合并算法。

  • 递归 - 拉动一个分支的默认合并策略。如果需要通过应用3向合并算法解析两个HEAD并且有多个可用于3向合并的共同祖先，请选择此选项。

  • 八达通 - 用于提取多个分支的默认合并策略。

    不执行需要手动解决冲突的合并。

  • 我们的 - 如果您需要取代侧支的旧开发历史，请选择此选项。通过应用此策略，可以解析任意数量的HEAD，但合并的结果始终是当前分支的HEAD。

  • 子树 - 修改后的递归策略。

不要使用合并策略

选中此复选框后，在rebase期间不会应用合并策略。

变基

单击此按钮可根据定义的设置启动变基。
