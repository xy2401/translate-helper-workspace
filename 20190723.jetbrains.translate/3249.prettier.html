WebStorm 2019.1帮助

漂亮

在开始之前，请确保您的计算机上有Node.js.

漂亮的格式化工具.js ， .ts ， .css ， .less ， .scss ， .vue ，并.json代码。使用WebStorm，您可以使用Reformat with Prettier操作格式化选定的代码片段以及整个文件或目录。只要将Prettier安装为项目中的依赖项或计算机上的全局依赖项，WebStorm就会添加此操作。

安装Prettier

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install --save-dev --save-exact更漂亮

    • npm install --global prettier

    从Prettier官方网站了解有关安装模式的更多信息。

在WebStorm中配置Prettier

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |更漂亮 。

  2. 在Prettier页面上打开，指定prettier包的路径，并选择要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

用Prettier重新格式化代码

  • 选择要在编辑器中重新格式化的代码片段，或在“ 项目”工具窗口中选择文件或文件夹，然后按Ctrl + Shift + Alt + P或从上下文菜单中选择“ 使用Prettier重新格式化” 。

  • 或者，按Ctrl + Shift + A并从“ 查找操作”列表中选择“ 使用Prettier重新格式化 ” 。

WebStorm可以将Prettier配置中的密钥代码样式规则应用于WebStorm代码样式设置，以便生成的代码（例如，在重构或快速修复之后）和已经使用Prettier处理的代码的格式一致。

要应用更漂亮的代码样式规则

  • 在启用Prettier的项目中，打开package.json ，然后在选项卡顶部的窗格中单击“ 是 ”。

  • 要重新应用Prettier代码样式（在窗格中单击No或修改代码样式后），请按Ctrl + Shift + A并从Find Action列表中选择Apply Prettier Code Style Rules 。
