IntelliJ IDEA 2019.1帮助

的WebPack

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA与webpack模块捆绑器集成。此支持通过考虑webpack模块解析和解决别名来改进JavaScript文件中的编码帮助。对于webpack版本2及更高版本，IntelliJ IDEA为webpack配置文件中的选项提供代码完成和快速文档查找。

在开始之前，请确保您的计算机上有Node.js.

在IntelliJ IDEA中配置webpack

请参阅webpack官方网站上的入门 。

  1. 确保webpack列在package.json的dependencies或devDependencies对象中。

    如果缺少webpack，请打开嵌入式终端 （ Alt + F12 ）并键入：

    npm install --save-dev webpack

  2. 在项目根目录或其他位置创建配置文件（ 新建| JavaScript文件 ）。从webpack官方网站了解更多信息。

  3. 指定要使用的webpack配置文件。默认情况下，IntelliJ IDEA分析项目根目录中的webpack配置文件。

    要使用其他webpack配置文件，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中转到“ 语言和框架”| JavaScript | Webpack并指定要使用的配置文件的位置。

基于对webpack配置文件的分析，IntelliJ IDEA了解webpack配置并在JavaScript文件中提供编码帮助，请参阅下面的解析模块 。

编辑webpack配置文件

对于webpack版本2及更高版本，IntelliJ IDEA在webpack.config.js的配置对象中提供代码完成和文档查找。代码完成即时提供。要查看符号的文档，请按Ctrl + Q.

IntelliJ IDEA仅在webpack配置文件中包含webpack字符串且webpack列在package.json时才提供编码帮助。

解决模块

当您打开项目或编辑在Settings / Preferences |上指定的webpack.config.js 语言和框架| JavaScript | Webpack ，IntelliJ IDEA在后台分析配置，并根据收到的信息，正确理解项目解析根和解决别名 。由于对项目配置的这种理解，IntelliJ IDEA为JavaScript文件中的导入和导出符号提供了更精确的代码完成。因此，一切正常，没有任何步骤。
