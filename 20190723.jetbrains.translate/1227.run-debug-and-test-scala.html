IntelliJ IDEA 2019.1帮助

运行，调试和测试Scala

IntelliJ IDEA允许您像通常使用IntelliJ IDEA中的任何其他应用程序一样运行，调试和测试Scala应用程序。IntelliJ IDEA还允许您使用coverage运行Scala代码并配置代码覆盖率设置。

要在执行运行或调试过程时调整配置设置，请使用“ 运行/调试配置”对话框。要访问该对话框，请选择“运行”|在主菜单上编辑配置 。

运行Scala应用程序

您可以通过IntelliJ IDEA运行Scala代码，使用sbt shell，或使用Scala工作表进行快速代码评估。

通过Intellij IDEA运行Scala应用程序

  1. 创建或导入Scala项目，就像在IntelliJ IDEA中创建或导入任何其他项目一样。

  2. 在编辑器中打开您的应用程序。

  3. 按Shift + F10执行应用程序。或者，在编辑器的左侧排水沟中，单击图标，然后选择运行'名称' 。

你也可以点击主工具栏上的图标，用于运行您的应用程序。

使用sbt shell运行Scala应用程序

您可以使用sbt shell运行您的应用程序，该shell是任何sbt项目的一部分。

  1. 打开你的sbt项目。

  2. 如果要将构建和导入委托给sbt，请在sbt projects工具窗口中单击图标打开sbt设置。
    在sbt设置页面上，选择“ 使用sbt shell进行构建和导入（必需sbt 0.13.5+）”选项，然后单击“ 确定” 。

    （当您创建或导入sbt项目时，此选项也可用。）
    如果您不想将构建和导入委托给sbt，您仍然可以在sbt shell中工作（ 并直接从它运行sbt命令 ，包括运行您的应用程序。

  3. 在sbt projects工具窗口中，单击sbt Tasks节点。

  4. 在打开的列表中，选择将运行main方法的运行任务。

    执行结果显示在sbt shell工具窗口中。

使用Scala工作表运行Scala代码

IntelliJ IDEA允许您创建Scala工作表以动态评估Scala代码结果。

  1. 右键单击您的项目，然后选择New | Scala工作表 。

    我们建议您在src目录中创建工作表，以简化项目类在类路径中的包含。这可能有助于测试目的。

  2. 在“ 新建Scala工作表”窗口中，键入Scala工作表的名称，然后单击“ 确定” 。

    因此，将打开一个扩展名为.sc的文件。

  3. 输入您的代码，然后按看到结果。

  4. 使用.sc文件时，您可以执行以下操作：

    • 您可以评估Scala对象。

    • 您可以折叠输出而不影响左侧的代码，只展开与特定语句匹配的输出块。

    • 您可以单击设置工作表编辑器工具栏上的图标，并配置将影响其他工作表的整个项目的当前工作表设置或默认工作表设置。

      • 如果要自动查看结果，请选中“ 交互模式”复选框。

      • 清除“生成项目”复选框以提高评估性能。在这种情况下，禁用项目更改的自动检查。

      • 您可以选择“ 运行类型”选项以查看不同的评估模式。例如，选择REPL类型以快速评估Scala表达式。添加新表达式时，它们会以递增方式执行。

      • 请注意， 模块的“ 使用类路径”选项已禁用，因为工作表是一个可以在其他项目之间共享的文件，应该保留在创建它的位置。您可以在默认设置中更改其位置，但它将应用于整个项目，并将影响其他工作表。如果您需要更改类路径，我们建议您创建并使用临时文件 。

      • 您还可以为工作表选择编译器配置文件。点击访问Scala编译器设置以获取更多选项。请记住，如果您通过编译器设置页面添加/更改配置文件并且您有一个sbt项目，那么您的更改将在下一个项目导入时被丢弃。

      • 有关工作表设置的更多选项，请选择“ 文件” | 设置 | 语言和框架 |单击Scala并单击“ 工作表”选项卡。例如，在“ 工作表”选项卡上，您可以设置输出截止限制，指定要在哪种模式下运行工作表，或者配置IntelliJ IDEA如何处理.sc和临时文件。

        如果选中“ 使用eclipse兼容性”模式复选框，则对象内的所有语句都将包含在工作表的输出中。

要删除工作表的结果，请按垃圾箱图标。

要将代码和评估结果复制到一个文件中，请单击图标。

调试Scala代码

IntelliJ IDEA允许您使用IntelliJ IDEA调试器或sbt shell调试代码。

使用Intellij IDEA调试Scala代码

  1. 在编辑器中打开Scala应用程序。

  2. 在左侧装订线中，为要调试的代码行设置断点。有关断点的更多信息，请参阅断点主题。

  3. 如果需要，可以访问运行/调试配置 （运行|编辑配置）并调整设置，但通常默认设置足以成功启动和完成调试会话。

  4. 按Shift + F9 。或者，在主工具栏上单击图标以启动调试过程。

  5. 在“ 调试”工具窗口中评估结果。

    有关如何在“ 调试”工具窗口中使用选项的信息，请参阅调试工具窗口参考 。

如果为run或test任务启用了分叉 ，则通过sbt shell调试任务将不起作用。

使用sbt shell调试Scala代码

  1. 打开你的sbt项目。

  2. 在编辑器中打开您的应用程序。

  3. 在编辑器中，在左侧装订线中，为要调试的代码行设置断点。

    您无法调试实际.sbt文件中定义的代码，但您可以调试可以从build.sbt调用的Scala文件中的代码。

  4. 在sbt projects工具窗口中，单击图标，启动sbt shell。

  5. 在sbt shell工具窗口中，单击用于连接调试器服务器并启动调试会话的图标。
    IntelliJ IDEA还为调试会话创建运行/调试配置 。

    如果需要，可以编辑运行/调试配置设置，但默认设置应足以成功调试代码。

  6. 运行你的程序。

    在“ 调试”工具窗口中评估结果。

测试Scala应用程序

IntelliJ IDEA允许您使用ScalaTest和Specs2测试Scala应用程序。您还可以设置和运行测试范围。

使用Scala Test测试Scala应用程序

  1. 打开你的项目。
    • 如果您有一个sbt项目，请打开build.sbt文件并为ScalaTest指定以下依赖项：
      libraryDependencies + = “org.scalactic” %% “scalactic” ％ “3.0.1”

      libraryDependencies + = “org.scalatest” %% “scalatest” ％ “3.0.1” ％ “test”

      点击 sbt projects工具窗口中的图标用于刷新项目或使用sbt设置中指定的自动导入选项，以便在每次更改build.sbt时自动刷新项目。

    • 如果您有常规Scala项目，请使用“ 项目结构”对话框配置测试库。

  2. 在编辑器中打开一个类，要为其创建测试并将光标放在包含类声明的行中。

  3. 按Ctrl + Shift + T并选择“ 创建新测试” 。

  4. 在打开的对话框中，指定测试设置，然后单击“ 确定” 。

  5. 在编辑器中打开测试，按Ctrl + Shift + F10或右键单击测试类，然后从上下文菜单中选择Run'test name' 。

  6. IntelliJ IDEA会自动为测试创建运行/调试配置，但如果要编辑配置中的设置，请单击“运行”|在主菜单上编辑配置 。

  7. 在“ 运行/调试配置”对话框的右侧，指定测试套件的设置，然后单击“ 确定” 。
    配置具有标准选项，您可以在“ 执行测试”部分中找到更多详细信息。但是，您还可以指定以下与Scala相关的选项：

    • 您可以选择使用sbt shell复选框通过sbt shell运行测试，使用sbt 使用UI以与平台测试运行器相同的格式显示测试结果。

    • 您可以选择“ 测试种类”选项以指定要运行的测试类型 。例如，您可以选择正则表达式并设置类和测试模式。

  8. 在主工具栏上，单击用于运行测试的图标。

  9. 在“ 运行”工具窗口中评估结果。

您可以使用test命令打开sbt shell，运行测试并评估sbt shell的结果。

或者，要创建测试套件，请右键单击该类，然后从上下文菜单中选择Go to |测试

要在打开“ 运行/调试配置”对话框后为测试创建新的运行/调试配置，请单击在左上角的conner中，从打开的列表中选择ScalaTest或Specs2 ，分别创建ScalaTest运行配置或Specs2运行配置。

要编辑Scala测试文件的默认模板，请按Ctrl + Alt + S，然后从左侧的选项中选择编辑器|文件和代码模板 。从右侧的选项中，选择“ 代码”选项卡并调整模板。

使用Specs2测试Scala应用程序

使用Specs2测试Scala应用程序的过程与使用ScalaTest测试Scala应用程序部分中描述的过程相同，但以下选项除外：

  • 您需要为sbt项目指定以下依赖项：
    libraryDependencies ++ = Seq （ “org.specs2” %% “specs2-core” ％ “3.9.5” ％ “test” ）

    对于常规Scala项目，请使用“ 项目结构”对话框来配置测试库。

  • 使用sbt选项的Use UI以与平台测试运行器相同的格式显示测试结果不可用。

您可以在作用域内运行测试，也可以在Scala项目中测试整个作用域。

  2. 在编辑器中，根据您的测试范围，您可以执行以下操作：

使用coverage运行Scala测试

IntelliJ IDEA允许您运行具有代码覆盖率的测试套件。

  1. 打开你的项目。

  2. 在编辑器中打开相关测试。

  3. 在左侧排水沟中，单击图标并选择带有Coverage选项的Run'name' 。
    IntelliJ IDEA运行测试并显示Coverage工具窗口，其中包含代码覆盖率信息。

    如果需要，您还可以调整默认代码覆盖率设置或代码覆盖率行为。

    如果您已经运行了测试套件，IntelliJ IDEA会自动创建运行/调试配置。您可以打开创建的运行配置并调整设置。
    如果要为测试套件创建新的运行配置，请执行以下操作：

    1. 从主菜单中，选择“运行”|编辑配置 。

    2. 在“ 运行/调试配置”对话框中，单击“ 左侧选项中的图标。

    3. 从打开的列表中，选择所需的配置。

    4. 从右侧的选项中，单击“ 代码覆盖率”选项卡。

    5. 调整默认设置，然后单击“ 确定” 。
      有关更多信息，请参阅配置代码覆盖率测量

    您还可以调整代码覆盖率行为。

    1. 按Ctrl + Alt + S打开“ 设置/首选项”页面。

    2. 从左侧的选项中，选择Build，Execution，Deployment |覆盖范围 。

    3. 从右侧的选项中，调整设置，然后单击“ 确定” 。

或者，您可以选择“运行”|从主菜单运行Coverage ，以覆盖范围运行代码。
