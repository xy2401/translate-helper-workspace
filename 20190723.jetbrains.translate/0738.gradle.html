创建一个新的Gradle项目

  1. 启动“ 新建项目”向导 。如果IntelliJ IDEA中当前未打开任何项目，请单击欢迎屏幕上的“ 创建新项目 ”：否则，请选择“ 文件”|新的|从主菜单中选择项目 。

  2. 从左侧的选项中选择Gradle 。

  3. 指定项目SDK和其他框架或库（IntelliJ IDEA将相应的插件添加到build.gradle文件中）。单击下一步 。

  4. 在向导的下一页上，指定类似于Maven命名约定的字段。如果您决定将项目部署到Maven存储库，这些设置可能会有所帮助。您指定的字段将添加到build.gradle文件中。

    • GroupId - 新项目的groupId 。如果您计划在本地部署项目，则可以省略此字段。

    • ArtifactId - artifactId ，作为新项目的名称添加。

    • 版本 - 新项目的version 。默认情况下，会自动指定此字段。

    如果指定了父Gradle项目，则可以从父项继承指定的字段（单击继承 ）。
    单击下一步 。

  5. 在向导的下一页上，为项目配置自动导入 ， 源集和Gradle版本 。

    您还可以指定如何在项目中对模块进行分组。例如，如果要对模块进行排序并按其限定名称对其进行分组，请使用using qualified names选项。在这种情况下，使用Gradle group生成模块组时，也会考虑模块名称中的前缀。
    有关更多信息，请参阅“ 分组模块”部分。

    单击下一步 。

  6. 指定名称和位置设置。单击完成 。

默认情况下，新创建或导入的项目存储在外部。这意味着生成的.iml和库文件存储在idea.system.path而不是.idea目录中。通过版本控制共享项目可能会有所帮助。

如果需要更改此行为，请打开Gradle设置（ 设置| Gradle ）并清除外部存储生成的项目文件复选框。

为项目配置Gradle版本

IntelliJ IDEA允许您在创建或导入Gradle项目时使用不同的选项来配置Gradle版本。您可以使用默认的Gradle包装器，使用Gradle包装器作为任务，或配置本地Gradle分发。

选择在Gradle项目工具窗口中快速访问Gradle设置页面。

  1. 在创建或导入Gradle项目时，在Gradle设置页面上，选择以下选项之一：

    • 使用默认gradle包装器（推荐） - 选择此选项以使用Gradle包装器 。在这种情况下，您将Gradle版本的更新委派给Gradle并获取构建的自动Gradle下载。它还允许您使用精确的Gradle版本进行构建。Gradle版本保存在项目的Gradle目录中的gradle-wrapper.properties文件中。我们建议您使用此选项来消除项目中的任何Gradle版本问题。

    • 使用gradle任务配置 - 选择此选项可根据wrapper任务配置配置Gradle包装wrapper 。如果由于某种原因不想使用gradle-wrapper.properties文件，或者想要控制项目要使用的Gradle版本，可能会很方便。请注意，如果您最初使用默认的Gradle包装器选项，然后在更改任务中的任何内容（例如更改Gradle版本）时决定使用Gradle包装器任务配置，则无需手动运行包装器任务，因为gradle-wrapper.properties文件更新由IntelliJ IDEA在导入期间隐式完成。

      另请注意，例如，如果您在项目中使用VCS并且每个团队成员通过gradle同步项目，则gradle/wrapper/gradle-wrapper.properties文件将更新，如果您使用此类型的包装器可能会出现不一致组态。

    • 使用Gradle本地分发 - 如果要手动下载并使用特定的Gradle版本，请选择此选项。指定Gradle安装和JVM的位置，在导入指定的Gradle项目时以及执行其任务时，IntelliJ IDEA将在该位置运行Gradle。

  2. 按“ 确定” （在向导中，按“ 下一步” ）。

将新的Gradle模块添加到现有项目中

您可以将Gradle模块添加到您已在其中工作的项目中。

  1. 在项目中，在主菜单上，选择“ 文件”|新的|模块打开新的模块向导。

  2. 如果现有项目不是Gradle项目，则添加模块的过程与创建新Gradle项目相同 。
    如果现有项目是Gradle项目，则添加新模块的过程会更短。您需要在ArtifactId字段中指定模块的名称。其余信息会自动添加，您可以使用默认设置或根据您的偏好更改它们。
    另请注意，默认情况下，将模板添加到字段会显示您尝试添加模块的项目的名称。你可以点击如果您有其他链接的Gradle项目，请选择其他名称。

从Gradle模型导入项目

  1. 启动“ 新建项目”向导 。如果IntelliJ IDEA中当前未打开任何项目，请单击欢迎屏幕上的“ 导入项目 ”。否则，请选择文件|新的|来自主菜单中的现有来源的项目 。

    请注意，您也可以选择文件|从主菜单中打开 ，然后选择build.gradle文件或包含build.gradle文件的目录。即使之前未打开或导入项目，IntelliJ IDEA也将导入Gradle项目。

  2. 在打开的对话框中，选择包含要导入的项目的目录或包含Gradle项目描述的文件（ build.gradle ）。单击确定 。

  3. 在Import Project向导的第一页上，在Import Model from External model中 ，选择Gradle并单击Next 。

    （如果IntelliJ IDEA已猜到您要导入的内容，则不会显示此页面。）

  4. 在“ 导入项目”向导的下一页上，指定Gradle项目设置 ，该设置与创建Gradle项目时相同。
    另外，指定以下全局Gradle设置：

    • 脱机工作 - 使用此复选框可在脱机模式下使用Gradle。在这种情况下，Gradle将使用缓存中的依赖项。Gradle不会尝试访问网络以执行依赖项解析。如果缓存中不存在所需的依赖项，则构建执行将失败。

    • 服务目录路径 - 使用此字段覆盖默认的Gradle主目录位置目录。

    • Gradle VM选项 - 使用此字段为Gradle项目指定VM选项。

    单击完成 。

如果使用gradle idea命令生成Gradle项目，则可能会出错。我们建议您改为导入项目。

    • 在“ 项目结构”对话框的“ 模块”页面上，单击“确定” 图标，然后选择导入模块 。

    • 在Gradle项目工具窗口的工具栏上，单击图标。

  2. 在打开的对话框中，选择要导入的模块，然后单击“ 确定” 。

  3. 如有必要，请执行“ 导入Gradle项目”部分中描述的步骤，然后单击“ 完成” 。

  4. 如果您有多模块项目，则会打开“ 选择要导入的项目数据”对话框。
    在“ 选择要导入的项目数据”对话框中，选择要包含在项目中的模块或数据，然后单击“ 确定” 。

使用Gradle项目

IntelliJ IDEA可让您管理Gradle项目。您可以链接，忽略项目并同步Gradle和IntelliJ IDEA项目中的更改。此外，您还可以配置Gradle组合构建。

将Gradle项目链接到IntelliJ IDEA项目

通过从Gradle模型导入创建项目时，将自动建立项目链接，并启用Gradle Projects工具窗口。

如果IntelliJ IDEA项目未链接到Gradle项目，则禁用Gradle工具窗口。
在这种情况下，IntelliJ IDEA会显示一条消息，其中包含一个链接，可以快速导入Gradle项目并启用Gradle Projects工具窗口。
如果Gradle Projects工具窗口处于活动状态，则表示您至少链接了一个Gradle项目。

  1. 打开Gradle Projects工具窗口。

  2. 在Gradle Projects工具窗口中，单击附加gradle项目。

  3. 在打开的对话框中，选择所需的build.gradle文件，然后单击“ 确定” 。

  4. 在“ 从Gradle导入模块”窗口中，指定要尝试链接的Gradle项目的选项，然后单击“ 确定” 。

    该项目是相互关联的。Gradle Projects工具窗口显示工具栏和Gradle实体的树视图。

或者，如果将项目作为Gradle模块从现有源导入 ，则可以链接Gradle项目。在这种情况下，即使Gradle Projects工具窗口不可用，也可以链接Gradle项目。

导航到build.gradle文件

  1. 在Gradle项目工具窗口中，右键单击链接的项目。

  2. 从上下文菜单中选择 。

    IntelliJ IDEA导航到相应的Gradle配置文件，并在编辑器中打开相关的build.gradle文件。

分离或忽略链接的Gradle项目

  1. 在Gradle项目工具窗口中，右键单击链接的项目。

  2. 从上下文菜单中选择 （ 删除 ）。或者，您可以选择链接的项目并单击在Gradle工具栏上。

  3. 在“ 导入Gradle项目”弹出窗口中，如果您不想从IntelliJ IDEA 项目工具窗口中删除项目，请清除模块的复选框。

  4. 单击确定 。

    Gradle项目与IntelliJ IDEA项目分离，不再同步。

您还可以使用“ 忽略Gradle项目”选项取消激活Gradle项目。

  1. 在Gradle Projects工具窗口中，右键单击要忽略的项目。

  2. 从上下文菜单中，选择Ignore Gradle Project 。

  3. 在打开的窗口中，选择要取消激活的项目和模块，然后单击“ 确定” 。

    在这种情况下，IntelliJ IDEA将从IntelliJ IDEA 项目工具窗口中删除所选项目和模块，并且不再导入它们。但是，您仍然可以在Gradle Projects工具窗口中看到忽略的Gradle模块和项目列表。

    如果要激活Gradle项目或模块，请从上下文菜单中选择Unignore Gradle Projects 。

如果从build.gradle文件中手动删除模块，则build.gradle部分导入时忽略模块，或者通过选择每个源集创建单独的模块或使用限定名称选项来更改导入设置中的模块 名称 - 在导入项目时所有这些情况，IntelliJ IDEA会提示您恢复已删除的模块。您可以在“ 孤立模块”对话框中选择要还原的那些。

刷新链接的Gradle项目

  1. 在Gradle项目工具窗口中，右键单击链接的项目。

  2. 从上下文菜单中选择 。

配置Gradle组合构建

在开始配置复合构建之前，请确保为项目配置了Gradle 3.1或更高版本。

另请注意， 使用Gradle组合构建存在一些限制 。

您可以使用settings.gradle文件为Gradle组合构建包含Gradle构建。

  1. 在编辑器中打开settings.gradle文件。

  2. 使用includeBuild命令，指定要作为项目依赖项添加的构建的位置。

您还可以使用Gradle项目工具窗口配置复合构建。

  1. 打开Gradle项目。

  2. 链接要用于复合构建的其他Gradle项目。

  3. 在Gradle项目工具窗口中，右键单击主项目，然后从上下文菜单中选择Composite Build Configuration 。

  4. 在Gradle Project Build Composite对话框中，选择要包含在Gradle组合构建中的项目。

  5. 刷新您的主要Gradle项目。

    IntelliJ IDEA查找包含的Gradle项目并将其视为IntelliJ IDEA模块。

在Gradle项目中添加包前缀

如果在Gradle项目中使用包前缀，请在build.gradle文件中指定它们。这样，当您重新导入项目时，一切都会被保存。

  1. 打开build.gradle文件。

  2. 添加以下插件以支持包前缀：

    plugins { id “org.jetbrains.gradle.plugin.idea-ext” version “0.5” }

  3. 添加包前缀。例如，您有以下一组源集：

    sourceSets { main 。 java 。 srcDirs = [] 主要 。 java 。 srcDirs + = “src” main 。 java 。 srcDirs + = “src / main / java” main 。 java 。 srcDirs + = “../ other-root/src/main/java” }
    使用以下代码将包前缀（在我们的示例中为“org.example”）添加到它们：
    idea { module { settings { packagePrefix [ “src” ] = “org.example” packagePrefix [ “src / main / java” ] = “org.example” packagePrefix [ “../other-root/src/main/java” ] = “org.example” } } }

  4. 重新导入您的更改或使用自动导入 。

有关更多信息，请参阅https://github.com/JetBrains/gradle-idea-ext-plugin 。

在build.gradle文件中指定项目设置

使用https://github.com/JetBrains/gradle-idea-ext-plugin插件，您可以描述项目设置，例如项目编码，以及build.gradle文件中属性文件的编码。

  1. 打开build.gradle文件。

  2. 添加以下插件以支持编码配置：

    plugins { id “org.jetbrains.gradle.plugin.idea-ext” version “0.5” }
  3. 使用以下代码描述项目编码：
    import org.jetbrains.gradle.ext。EncodingConfiguration。BomPolicy 想法 {{ 项目 设置 {{ 编码 编码 = ' 窗口 - 1251' bomPolicy = BomPolicy。WITH_NO_BOM 属性 { 编码 = '< 系统 默认 >' transparentNativeToAsciiConversion = FALSE} 映射 [ '../ 样品 - 的gradle - 游离 / 模块 '] = ' 窗 - 1251' 映射 [ ' 模块 '] = ' 窗 - 1251' 映射 [ ' module2 / src / main / java ' ] = ' windows - 1251 ' } } } }

    <System Default>是一个字符串常量，表示将使用系统默认编码设置。

  4. 重新导入您的更改或使用自动导入 。

使用buildSrc

如果您有一个包含多个Java，Groovy或Kotlin类的大型Gradle脚本，则可以将这些类移动到buildSrc目录并从主Gradle脚本中引用它们。在这种情况下，您可以确保build.gradle文件的可读性。

  1. 如果您没有和现有的buildSrc ，请将其作为Gradle 模块添加到主项目中。

  2. 在编辑器中打开main build.gradle文件，并将所需的类移动到buildSrc目录的main子目录中。

  3. 从项目的build.gradle文件中运行任务。

如果需要更改Gradle脚本中引用的类的名称，可以使用重命名 （ Shift + F6 ）重构。IntelliJ IDEA将更改应用于所有引用。

查找用法 （ Alt + F7 ）操作将包括搜索中的gradle文件以及常规文件。

委派构建和运行操作

默认情况下，IntelliJ IDEA会将所有构建和运行操作委派给Gradle。

构建项目（ Build | Build Project ）时，IntelliJ IDEA使用Gradle调用正确的任务。此外，“ 运行 ”菜单中的“运行”和“ 调试”操作也是使用Gradle执行的。

HotSwap也会被触发，并在调试过程中重新加载类。

Gradle编译器不支持链式表达式的代码完成，而是使用IntelliJ IDEA编译器。

如果需要将构建委派给IntelliJ IDEA，请执行以下操作：

  1. 点击在Gradle项目工具窗口中。或者，在主菜单上选择File |设置|构建，执行，部署|构建工具| Gradle 。

  2. 单击Gradle，然后从列表中选择Runner 。

  3. 在Runner页面上，清除Delegate IDE构建/运行操作到Gradle选项。

    请注意， 运行测试使用选项保持活动状态，即使您将所有构建和运行操作委派给IntelliJ IDEA，也可以选择运行测试的方式。

  4. 单击确定 。

现在，如果您构建Gradle项目，它将使用IntelliJ IDEA构建。

如果已链接项目，则可以配置构建每个链接项目的方式。

  1. 在Settings / Preferences对话框（ Ctrl + Alt + S ）中，转到Build，Execution，Deployment | Gradle 。

  2. 在“ Gradle设置”页面的“ 链接的Gradle项目”中 ，选择要为其配置委派设置的项目。

  3. 在“ 委派设置”部分中， 使用选项指定“ 生成并运行”和“运行测试” 。单击确定 。

您还可以使用Gradle构建WAR或EAR工件。

  1. 创建Gradle Web项目。（有关更多说明，请参阅创建Gradle项目 。）

  2. 创建Tomcat服务器运行/调试配置。

  3. 单击“ 部署”选项卡，然后为部署添加工件。

  4. 点击启动Tomcat服务器配置。
    在“ 运行”工具窗口中，您可以看到工件由Gradle构建，并由IntelliJ IDEA Tomcat集成部署。

配置和使用Gradle测试运行器

IntelliJ IDEA允许您使用不同的测试运行程序在Gradle项目中执行测试。您可以使用JUnit来运行测试。在这种情况下，由于增量编译，测试运行得更快。您可以将测试过程委派给Gradle。在这种情况下，当您运行测试时，您将在持续集成（CI）服务器上获得相同的结果。此外，无论您的Gradle项目有多难，在命令行中运行的测试将始终在IDE中运行。您还可以决定每个测试专门使用哪个测试运行器。

  1. 在Gradle项目工具窗口中，单击 。

  2. 在“ 设置”对话框中，右键单击“ Gradle”，然后从列表中选择“ Runner” 。

  3. 在打开的页面上，在“ 运行测试使用”列表中，选择要用于测试运行的选项。

    （默认值为Platform Test Runner 。）

  4. 单击确定 。

  5. 在Gradle项目中，在编辑器中，创建或选择要运行的测试。

  6. 从上下文菜单中，选择“运行<测试名称>” 。
    或者，单击左侧排水沟中的图标。

  7. 根据您选择的测试运行程序，IntelliJ IDEA可通过以下方式之一运行测试：

    • 如果选择了Platform Test Runner ，IntelliJ IDEA将使用JUnit测试运行器运行测试，并在Run JUnit工具窗口中显示输出。

    • 如果选择了Gradle Test Runner ，IntelliJ IDEA将使用Gradle测试运行器运行测试，并在Run Gradle工具窗口中显示输出。

      您还可以选择带覆盖的运行'name（）'以查看测试的代码覆盖率。它适用于平台测试运行器以及Gradle测试运行器。

    • 如果您选择让我选择每个测试 ，IntelliJ IDEA允许您在JUnit或Gradle之间进行选择以运行测试。

      一旦您选择了测试运行器，IntelliJ IDEA将记住您的选择并自动使用您选择的选项运行测试。
    您可以在“ 赛跑者设置”页面中更改您的选择。

如果选择了让我选择每个测试选项，并且您决定调试测试（从装订线的上下文菜单中选择Debug'测试名称' ），IntelliJ IDEA将清除Run / debug配置中的Enable Gradle脚本调试选项。禁用Gradle脚本中的断点。

这样做是为了加快调试过程。您可以通过选择或清除“ 启用Gradle脚本调试”选项，在Gradle脚本中手动启用或禁用断点。

在Run Gradle工具窗口中，您可以单击图标以获取Gradle测试报告。

你也可以点击用于检查以前保存的测试结果的图标。

运行内部Gradle测试服

有时，在多模块项目中，您可能希望查看有关Gradle内部测试套件的信息。

例如，当并行运行测试时，可以使IntelliJ IDEA显示有关测试过程的信息。IntelliJ IDEA还显示有多少进程同时工作。

  1. 在“ 运行”工具窗口中，单击图标并选择显示内部Gradle测试套装 。

  2. 重新运行测试以查看包含Gradle内部测试结果的测试结果。

使用Gradle任务

IntelliJ IDEA允许您在项目中创建，调试和管理Gradle任务。

运行Gradle任务

IntelliJ IDEA允许您使用几种不同的方式运行Gradle任务，例如使用Gradle工具窗口中的工具栏，使用运行配置，使用上下文菜单，甚至使用一个运行配置运行多个任务。

此外，当您在无干扰模式下工作（没有工具栏和工具窗口）时，您可以从Run Anything窗口运行Gradle任务或命令。按两次Ctrl键将其打开，键入命令并按Enter键 。

您可以查看Mastering快捷方式部分以获取更多信息。

从Gradle工具栏运行Gradle任务

  1. 在Gradle项目工具窗口的工具栏上，单击 。

  2. 在“ 运行Gradle任务”对话框的“ 命令行”字段中，开始输入任务的名称。您可以看到IntelliJ IDEA显示Gradle任务列表，您可以从中选择适当的任务。单击确定 。

  3. IntelliJ IDEA运行所选任务并在“ 运行”工具窗口中显示结果。

    IntelliJ IDEA还将任务保存在“ 运行配置”节点下的Gradle项目中 。

通过运行配置运行Gradle任务

  1. 打开Gradle项目工具窗口。

  2. 右键单击要为其创建“运行”配置的任务。

  3. 从上下文菜单中选择“ 创建'任务名称' 。

  4. 在“ 创建运行/调试配置：'任务名称'中 ，指定任务设置，然后单击” 确定“ 。

    IntelliJ IDEA显示“ 运行配置”节点下的任务。

  5. 双击任务以运行它，或右键单击任务，然后从上下文菜单中选择“运行” 。

从上下文菜单中运行Gradle任务

  1. 打开Gradle项目工具窗口。

  2. 右键单击要运行的任务。

  3. 从上下文菜单中选择“运行'任务名称' 。

使用一个运行/调试配置运行多个Gradle任务

  1. 选择运行|编辑配置 （ Shift + Alt + F10 ）。
    “ 运行/调试配置”对话框将打开。

  2. 在“ 运行/调试配置”对话框中，单击“确定” 并选择Gradle以添加新配置。

  3. 在“ 运行/调试配置”对话框的右侧，在“ 名称”字段中，输入配置的名称。
    例如，指定以下设置：

    • Gradle项目 - 单击并选择已注册的Gradle项目。

    • 任务 - 指定要使用此配置执行的任务。您可以运行多个任务。
      例如，指定clean和build 。

    • VM选项 - 您可以自定义VM选项。例如，指定-Xmx3g 。

    • 参数 - 您可以指定Gradle命令行参数 。例如，指定--debug 。

  4. 单击确定 。
    创建的配置将添加到Gradle Projects工具窗口中的Run Configurations节点，并被视为Gradle任务。

  5. 双击配置以运行任务，或右键单击配置并选择“运行” 。

调试Gradle任务和脚本

您可以像调试常规应用程序一样调试Gradle任务和脚本。

您可以从编辑器或Gradle项目工具窗口启动调试会话。

调试Gradle脚本任务

  1. 在编辑器中打开build.gradle 。

  2. 设置断点并在左侧排水沟中右键单击“运行”（ ）图标。从上下文菜单中，选择Debug <name> 。

  3. IntelliJ IDEA启动调试会话，您可以在调试工具窗口中检查结果。

使用Gradle项目工具窗口调试实现org.gradle.process. Gradle任务org.gradle.process.JavaForkOptions接口，例如， 测试或运行 。

调试Gradle任务

  1. 在Gradle项目工具窗口的“ 任务”区域中，双击Gradle项目。

    任务列表打开。

  2. 在任务列表中，选择要调试的任务，右键单击它，然后从上下文菜单中选择Debug 。

    将启动调试过程，并将任务添加到“ 运行配置”节点下的最近任务列表中。

为Gradle任务分配快捷方式

IntelliJ IDEA允许您为可以以多种方式运行的Gradle任务分配快捷方式，并使用单个按键来执行这些任务。您还可以为可包含多个任务的Gradle运行/调试配置分配快捷方式。

  1. 在Gradle项目工具窗口中，右键单击所需任务。

  2. 从上下文菜单中，选择“ 指定快捷方式” 。

    “ 键盘映射”对话框打开。

  3. 在“ 键盘映射”对话框中的“ Gradle”节点下，导航到您的任务。

  4. 右键单击该任务，然后从打开的列表中选择要分配的快捷方式类型。

  5. 在打开的对话框中，根据快捷方式的类型，配置快捷方式，然后单击“ 确定” 。
    在我们的例子中，让我们添加一个键盘快捷键。

    您可以在Gradle项目工具窗口中看到快捷方式是针对您的任务显示的。

在“ 键盘映射”对话框中，您可以添加要为其分配快捷方式的新任务。

  1. 在“ 键盘映射”对话框的“ Gradle”节点下，单击“选择任务以指定快捷方式” 。

  2. 在打开的对话框中，选择所需的任务，然后单击“ 确定” 。

    该任务将添加到Gradle节点下的列表中。现在您可以配置快捷方式 。

为Gradle任务配置运行触发器

IntelliJ IDEA允许您在项目执行之前运行Gradle任务，或使用任务激活配置设置其他条件。

  1. 在Gradle项目工具窗口中，右键单击Gradle项目。

  2. 从上下文菜单中，选择“ 任务激活” 。

  3. 在“ 任务激活”对话框中，单击 。

  4. 在“ 选择激活阶段”菜单上，选择运行任务的时间，例如“ 生成前” ，“ 同步后”等。

  5. 在“ 选择任务”菜单上，选择实际任务。

    任务和激活阶段将添加到“ 任务激活”对话框的列表中。您还可以在Gradle项目工具窗口中看到针对所选任务的激活阶段名称。

或者，当您在Gradle项目工具窗口中右键单击要执行的任务时，可以从上下文菜单中选择激活阶段名称。

您还可以创建依赖于Gradle任务的运行/调试配置。

  1. 在主菜单上，选择“运行”|编辑配置以打开项目的运行/调试配置。

  2. 在“ 运行/调试配置”对话框的“ 启动前”部分中，单击图标。

  3. 在打开的列表中，选择“运行Gradle”任务 。

  4. 在“ 选择Gradle任务”对话框中，指定在启动项目之前要执行的项目和任务。您可以指定Gradle链接项目或任何其他Gradle项目。请注意，如果您的Gradle项目未链接，则IntelliJ IDEA将使用默认配置（例如，捆绑的Gradle版本）来运行任务。

  5. 单击确定 。
