使用SSH创建基于WSL的远程解释器

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

您可以使用Windows子系统Linux（WSL）来使用Linux发行版中提供的Python解释器。

完成初步步骤

  1. 单击屏幕左下角的Windows按钮，然后开始键入System Information 。要确保您的系统能够很好地使用WSL，请将Windows升级到最新版本。

  2. 按照“ WSL安装指南”中的说明安装适用于Linux的Windows子系统并初始化Linux发行版。

通过WSL配置远程解释器

  1. 通过以下任一方式打开“ 添加Python解释器”对话框：

    • 当您在编辑器中时 ，最方便的方法是使用状态栏中的Python解释器小部件。单击小部件并选择Add Interpreter ...

    • 如果您位于“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，请选择“ 项目<项目名称>”项目口译员 。点击图标，然后选择添加 。

  2. 在对话框的左侧窗格中，单击WSL 。

  3. 选择Linux发行版并在所选Linux发行版中指定python可执行文件的路径。

完成后，新的解释器将添加到您的项目中，并将设置默认的mnt映射 。

请注意，使用WSL无法创建虚拟环境 ：您安装的所有软件包都将添加到相应的系统解释程序中。系统会要求您输入sudo密码。

PyCharm允许您使用在WSL上运行的自定义Linux发行版 。这可以通过在检测到WSL后自动编辑PyCharm创建的wsl.distributions.xml配置文件来完成。要添加自定义分发，请执行以下操作：

  1. 打开％\ HOMEPATH％\。PyCharm2019.1 \ config \ options \ wsl.distributions.xml文件。

  2. 添加descriptor元素并提供访问自定义分发的设置。请注意， id值应该是唯一的。例如：

    <！ - ... - > <descriptor> <id> Arch </ id> <microsoft-id> Arch </ microsoft-id> <executable-path> c：/linux/arch.exe </ executable- path> <presentable-name> Arch </ presentable-name> </ descriptor> <！ - ... - >

    请参阅Windows注册表以检查id和executable-path值。

  3. 重新启动PyCharm并将您的自定义分发添加为远程解释器， 如上所述 。
