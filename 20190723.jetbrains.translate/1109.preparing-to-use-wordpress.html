准备使用WordPress

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA为开发和运行WordPress应用程序提供专用接口，并提供WordPress -aware编码帮助，请参阅WordPress特定编码辅助 。要在项目中访问此功能，您需要下载WordPress ，在IntelliJ IDEA中注册它，并在当前项目中激活WordPress集成。

或者，您可以下载WordPress并创建一个WordPress目标项目的存根。在这种情况下，项目中的WordPress集成将自动激活。

无论是在现有项目中启用WordPress集成还是创建WordPress应用程序的存根，IntelliJ IDEA都会检查是否为WordPress开发正确配置了开发环境。如果配置不符合要求，IntelliJ IDEA将显示带有Fix链接的弹出窗口。

在你开始之前

  1. 确保PHP解释器在PHP页面上的IntelliJ IDEA中配置 ，如配置本地PHP解释器和配置远程PHP解释器中所述 。

  2. 确保已安装并启用了PHP和WordPress支持插件。

    插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装它们，如管理插件中所述 。启用后，插件可在IDE级别使用，也就是说，您可以在所有IntelliJ IDEA项目中使用它们。

  3. 配置PHP解释器以使用WordPress ，如配置远程PHP解释器所述 。请注意， WordPress需要PHP 5.3或更高版本。

下载并安装WordPress

  1. 通过https://wordpress.org/download/下载WordPress.zip存档。

  2. 提取存档：

    • 为了能够在本地Web服务器上运行WordPress应用程序，请将解压缩的文件存储到配置PHP解释器的服务器的文档根目录中。对于使用Apache HTTPD Web服务器的本地开发环境，将WordPress.zip解压缩到htdocs文件夹。

    • 要在就地服务器上运行WordPress应用程序，请将提取的文件存储在项目根目录下。

    • 要在远程服务器上运行WordPress应用程序，请将提取的文件存储在项目中，然后将它们自动上载到远程服务器的文档根目录。

      了解有关服务器访问配置的详细信息，请参阅配置与Web服务器的同步 。有关配置上载到服务器的信息，请参阅上载和下载文件 。

    • 如果您不打算运行WordPress应用程序，只需要从IntelliJ IDEA获得编码帮助，请将提取的文件存储在计算机的任何位置。在这种情况下，您必须将安装配置为包含路径 ，请参阅配置包含路径 。

要在命令行模式下运行WordPress，您需要一组命令行工具，您可以使用Composer依赖关系管理器或下载wp-cli.phar存档来安装wp-cli/wp-cli软件包。有关详细信息，请参阅使用WordPress命令行工具WP-CLI 。

在项目中激活WordPress安装

  1. 按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，然后单击“ 语言和框架”下的“ 框架”页面 PHP 。

  2. 在打开的“ 框架”页面上，选中“ 启用WordPress集成”复选框。

  3. 在“ WordPress安装路径”文本框中，指定安装WordPress的文件夹。该文件夹应包含wp-admin和wp-includes子目录。

  4. 单击“ 应用”以保存WordPress注册，然后单击“ PHP” 。

  5. 在打开的PHP页面上，将WordPress安装文件夹的路径添加到“ 包含路径”列表中：单击按钮并在打开的对话框中指定安装文件夹的路径。在配置包含路径中了解更多信息。

生成WordPress应用程序存根

在项目创建期间，IntelliJ IDEA可以生成用于开发WordPress应用程序的项目存根。项目结构是根据WordPress要求设置的。

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的创建新项目 。“ 项目类别和选项”对话框将打开。

  2. 在左侧窗格中，单击列表中的PHP ，然后在右侧窗格中选择WordPress插件 ，然后单击“ 下一步” 。

  3. 在向导的第二页上，指定项目名称以及将在其中创建项目的文件夹。

    在“ WordPress安装路径”文本框中，指定安装WordPress的文件夹。该文件夹应包含wp-admin和wp-includes子目录。单击“ 完成”以开始生成项目存根。

  4. 如果新创建的项目位于WordPress安装文件夹之外，请将其配置为外部库，方法是将其添加到包含的路径列表中。

    1. 按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，然后单击Languages＆Frameworks下的PHP 。

    2. 在打开的PHP页面上，将WordPress安装文件夹的路径添加到“ 包含路径”列表中：单击按钮并在打开的对话框中指定安装文件夹的路径。在配置包含路径中了解更多信息。
