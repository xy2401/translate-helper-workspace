在调试模式下生成APK

除了生成要在物理设备上部署的已签名包，或要在模拟器上运行的未签名包之外，您还可以在调试模式下提取和签署应用程序包。此签名足以用于测试和调试应用程序，但不允许发布它们。只有通过配置工件才能在调试模式下对应用程序包进行签名。

在调试模式下 ，您可以使用默认证书或自定义证书对APK进行签名。

如果您决定使用默认证书 ，IntelliJ IDEA将使用调试密钥库或Android SDK工具生成的密钥在调试模式下对提取的包进行签名，并具有预定义的名称和密码：

  • 密钥库名称： debug.keystore

  • 密钥库密码： android

  • 关键别名： androiddebugkey

  • 密码： android

  • CN （ 通用名 ）： CN=Android Debug,O=Android,C=US

这些是IntelliJ IDEA中的默认设置。这意味着如果您未手动配置工件并在“ 运行/调试配置：Android应用程序”对话框中选择“ 部署默认APK”选项，则IntelliJ IDEA将使用证书中的预定义值来生成.apk 。

如果使用自定义证书 ，IntelliJ IDEA将使用调试密钥库或您自己指定的密钥在调试模式下对提取的包进行签名。您可以生成新证书或重用现有证书。后一种方法很有用，例如，如果您有多个应用程序，并且您希望它们都使用相同的证书进行签名，那么它们可以存储在设备上的同一文件夹中。

在调试模式下签名包

  1. 选择文件|从主菜单中选择项目结构 ，然后单击左窗格中的Artifacts 。

  2. 单击“ 新建”按钮并从上下文菜单中选择Android Application 。

  3. 在弹出菜单中，通过选择以下选项之一来定义工件内容：

    • 要创建空布局定义，请选择“ 空” 。

    • 要在工件中包含模块数据，请选择From module <module name>

    新工件的常规设置将显示在右侧的“ 工件布局”窗格中。

  4. 指定工件的常规设置 。在“ 输出目录”文本框中，指定目标包.apk文件的位置。

  5. 使用以下步骤完成工件定义：

    • 配置工件结构 。

    • 向工件添加资源 。

    • 排列工件中包含的元素 。

    • 如有必要，请在“ 预处理”和“ 后处理”选项卡中指定在构建工件之前和之后要执行的其他活动。

  6. 切换到Android选项卡，并从“ 类型”下拉列表中指定要使用的证书：

    • 选择使用默认证书签名的调试，以使IntelliJ IDEA使用调试密钥库或由Android SDK工具生成的密钥，并具有预定义的名称和密码。

    • 选择使用自定义证书签名的调试，以使用您选择的证书对包进行签名。通过执行以下操作之一指定要使用的密钥和包含它的密钥库文件：
      • 使用现有密钥库文件中的密钥对包进行签名：

        1. 在密钥库路径文本框中指定文件位置。手动键入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择该文件。在密钥库密码文本框中，键入所选密钥库的密码。

        2. 指定密钥别名并输入密码以访问密钥。

      • 要在现有密钥库中生成新密钥：

        1. 在密钥库路径文本框中指定文件位置。手动键入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择该文件。在密钥库密码文本框中，键入所选密钥库的密码。

        2. 单击“ 创建新”按钮，然后通过填写打开的“ 新建密钥库”对话框中的数据来配置要生成的版本密钥 。

      • 要使用新密钥生成新的密钥库文件：

        1. 单击“ 创建新”按钮。在打开的“ 新建密钥库”对话框中，在“ 密钥库路径”文本框中指定要生成的文件的位置。手动键入路径或单击“ 浏览” ，然后选择父文件夹并指定文件的名称。

        2. 指定并确认密码以访问密钥库。

        3. 通过填写Key区域中的数据来配置新的释放密钥 。

    单击下一步 。

  7. 要让IntelliJ IDEA在打包过程中对应用程序进行模糊处理 ，请选中Run ProGuard复选框并指定proguard.txt配置文件的位置。该文件在项目创建时生成，并存储在项目根目录中。IntelliJ IDEA在“ 配置文件路径”文本框中建议此默认位置。单击“ 浏览”接受建议或指定自定义配置文件并在打开的对话框中选择所需的文件。
