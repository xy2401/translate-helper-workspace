在开始之前，请确保您的计算机上有Node.js.

PyCharm与Grunt JavaScript Task Runner集成。PyCharm解析Gruntfile.js文件，识别任务和目标的定义，在树视图中显示任务和目标，让您在树中的任务或目标及其在Gruntfile.js文件中的定义之间导航，并支持运行和调试任务和目标。

使用PyCharm，您可以在专用Grunt工具窗口中的任务树中运行Grunt任务，或使用Grunt运行配置 ，或作为启动前任务。PyCharm显示在“ 运行”工具窗口中执行任务的结果。工具窗口显示Grunt输出，报告发生的错误，列出尚未找到的包或插件等。最后执行的任务的名称显示在工具窗口的标题栏上。

安装Grunt

您也可以按照npm和Yarn中的说明在Node.js和NPM页面上安装软件包。

要在PyCharm项目中使用Grunt，您需要两个包：

  • 全局安装的grunt-cli包（Grunt命令行界面），用于执行Grunt命令。

  • 作为开发依赖项安装的grunt包，用于构建项目任务树并在编辑Gruntfile.js或Gruntfile.coffee文件时提供编码帮助。从Grunt官方网站了解有关Gruntfile.js更多信息。

在全球范围内安装grunt-cli

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install -g grunt-cli

在项目中安装Grunt

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install grunt --save-dev

从任务树运行Grunt任务

在PyCharm会话期间第一次构建任务树时， Grunt工具窗口尚未打开。

请注意，从任务树运行任务时， 强制执行和详细模式选项不可用。因此，您无法配置PyCharm，例如，忽略警告或提供详细日志。要使用这些选项，请按照下面的运行配置中的运行和调试任务中的说明从运行配置运行任务或目标。

PyCharm允许您从Grunt工具窗口中的任务树轻松快速地运行Grunt任务。PyCharm会自动创建一个临时运行配置，如有必要，您可以保存并稍后使用。

打开Grunt工具窗口

  • 在“ 项目”工具窗口中选择所需的Gruntfile.js文件，或在编辑器中将其打开，然后从上下文菜单中选择“ 显示Grunt任务 ”。

从Grunt工具窗口构建任务树

  • 在Grunt工具窗口中，单击在工具栏上，从列表中选择所需的Gruntfile.js文件。默认情况下，PyCharm在项目的根目录中显示Gruntfile.js文件。

  • 如果您有另一个Gruntfile.js文件，请单击Choose Gruntfile.js并在打开的对话框中选择所需的Gruntfile.js文件。PyCharm在其标题上添加一个新节点，其中包含所选Gruntfile.js文件的路径，并在新节点下构建任务树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的任务进行排序

  • 点击在工具栏上，从菜单中选择排序依据 ，然后选择名称 。
    默认情况下，树以Gruntfile.js （选项定义顺序 ）中的顺序显示任务。

运行任务或目标

  • 双击任务或目标。

  • 在树中选择任务或目标，然后按Enter或在上下文菜单中选择“运行<任务名称> ”。

运行默认任务

  • 在树中选择根节点，然后从选择的上下文菜单中选择“运行默认值 ”。

运行多个任务或目标

  • 使用多选模式：按住Shift （对于相邻项）或Ctrl （对于非相邻项）键并选择所需的任务或目标，然后在选择的上下文菜单中选择“运行”或“ 调试 ”。

跳转到任务或目标的定义

  • 在树中选择所需的任务或目标，然后在选择的上下文菜单中选择“ 跳转到源 ”。

根据运行配置运行和调试任务

除了使用PyCharm从任务tress运行任务或目标时自动创建的临时运行配置，您还可以创建并启动自己的Grunt.js运行配置。

创建Grunt.js运行/调试配置

  1. 在主菜单上，选择“运行”|编辑配置 。

  2. 点击在工具栏上，从列表中选择Grunt.js 。运行/调试配置：Grunt.js对话框打开。

  3. 指定运行配置的名称，要运行的任务（使用空格作为分隔符），定义这些任务的Gruntfile.js文件的位置，以及全局安装的grunt-cli包的路径。

  4. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    （可选）指定Node.js特定的选项参数以及要传递给Node.js的环境变量 。

运行任务

  • 从主工具栏上的列表中选择新创建的运行配置，然后单击在列表旁边。PyCharm在“ 运行”工具窗口中显示任务输出。

调试任务

  1. 创建Grunt.js运行/调试配置如上所述 。

  2. 在编辑器中打开Gruntfile.js文件，并在必要时在其中设置断点。

  3. 要启动调试会话，请从主工具栏上的列表中选择所需的调试配置，然后单击在列表旁边或选择Run |从主菜单中调试<配置名称> 。

  4. 在打开的“ 调试”工具窗口中，分析暂停的任务执行，逐步执行任务，等等，如检查暂停程序和单步执行程序中所述 。

将Grunt任务作为启动前任务运行

  1. 选择Run |打开Run / Debug Configurations对话框在主菜单上编辑配置 ，从列表中选择所需的配置，或通过单击重新创建并选择相关的运行配置类型。

  2. 在打开的对话框中，单击在Before launch区域中，从列表中选择Run Grunt任务 。

  3. 在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

自动运行Grunt任务

如果您有定期运行的某些任务或目标，则可以将相应的运行配置添加到启动任务列表中。任务将在项目启动时自动执行。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|启动任务 。

  2. 在打开的“ 启动任务”页面上，单击“确定” 在工具栏上。

  3. 从列表中，选择所需的Grunt运行配置。配置将添加到列表中。

    如果项目中没有适用的配置，请单击然后选择“ 编辑配置” 。然后在运行/调试配置：打开的Grunt.js页面上定义具有所需设置的配置 。保存新配置时，它会自动添加到启动任务列表中。
