IntelliJ IDEA |偏好|工具| macOS的终端
按Ctrl + Alt + S

虽然IntelliJ IDEA终端 （ Alt + F12 ）中的默认shell工作正常，但许多开发人员更喜欢使用他们喜欢的shell。例如，Windows用户可能希望使用PowerShell或Cmder ，Linux和macOS用户可能希望使用zsh而不是默认的终端shell。在此设置页面上，您可以自定义将在终端中使用的shell。

IntelliJ IDEA通过捆绑插件实现终端功能，可以通过清除IntelliJ IDEA设置的插件页面上的终端复选框（ Ctrl + Alt + S ）完全禁用该插件 。

项目

描述

启动目录

指定终端将启动的工作目录。

壳路径

指定默认运行的shell。以下是不同shell的一些值：

  • PowerShell ： powershell

  • Cmder ： "cmd" /k ""\vendor\init.bat"" （注意必须设置CMDER_ROOT环境变量）

  • Cygwin ： "C:\cygwin\bin\bash.exe" --login -i

  • Zsh ： /bin/zsh

  • Bash ： /bin/bash （或Windows的bash： bash.exe ）

标签名称

指定新会话选项卡的默认名称。请注意，可以重命名会话选项卡。

听见钟声

如果选择此选项，控制台将在传入的转义序列上播放铃声。

结束时关闭会话

如果选择此选项，则当相应进程结束时（例如，kill），当前会话将自动结束。

鼠标报告

如果选择此选项，则嵌入式本地终端支持鼠标指针。

选择时复制到剪贴板

如果选择此选项，则在终端中选择的文本将自动复制到剪贴板。

粘贴鼠标中键单击

如果选择此选项，则可以通过单击鼠标中键来粘贴剪贴板内容。

覆盖IDE快捷方式

如果选择此选项，“终端”工具窗口将处理与IntelliJ IDEA不同的键盘快捷方式。
如果清除此复选框，则使用IntelliJ IDEA键绑定。

壳牌整合

如果选择此选项，终端首先加载自定义rc配置文件（位于IntelliJ IDEA发行版plugins下的terminal文件夹中），该文件提供额外的设置，然后是用户的rc文件。

请注意，目前shell集成适用于Bash / sh （ bashrc ）， zsh （ zshrc ）和fish shell （ config.fish ）。
