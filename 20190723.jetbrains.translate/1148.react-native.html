在开始之前，请确保您的计算机上有Node.js.

使用React Native，您可以使用JavaScript和React为iOS和Android开发本机移动应用程序。它由Facebook创建，用于Instagram，Airbnb和现在JetBrains自己的YouTrack移动应用程序等知名应用程序。从React Native官方网站了解更多信息。

IntelliJ IDEA可帮助您创建，编辑，修改，运行，调试和维护React Native应用程序。IntelliJ IDEA还为React和Flow符号提供代码完成。

全局安装react-native-cli

  • 打开内置终端 （ View | Tool Windows | Terminal ）并在命令提示符下键入npm install -g react-native-cli 。

创建一个新的React Native应用程序

在IntelliJ IDEA中创建React Native应用程序的推荐方法是使用专用项目生成器。

创建应用程序

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，选择“ React Native” ，然后单击“ 下一步” 。

  4. 在向导的第二页上，指定项目名称和要在其中创建的文件夹。

    在Node Interpreter字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择“ 添加”以配置新的解释器

    在React Native字段中，指定react-native-cli包的路径。

  5. 单击“完成”后，IntelliJ IDEA将生成一个包含所有必需配置文件的React Native特定项目。

  6. 安装其他工具以开始使用React Native，例如iOS模拟器 。这些工具的列表取决于您的操作系统以及您要定位应用程序的移动平台。有关详细的安装说明，请参阅“ React Native入门指南” 。

要下载项目依赖项，请执行以下操作之一：

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm安装

  • 从项目根目录中的package.json文件的上下文菜单中选择Run'npm install' 。

要继续开发现有的React Native应用程序，请在IntelliJ IDEA中打开它并下载所需的依赖项。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

运行和调试React Native应用程序

您需要只安装一次iOS模拟器，之后IntelliJ IDEA会自动使用react-native run-ios启动它。

每次开始使用应用程序时，都需要准备Android设备。

您可以在物理设备或模拟器上运行和调试应用程序。在启动应用程序之前，请确保模拟器已在运行，或者如果您使用的是物理设备，则它已连接到您的计算机。

IntelliJ IDEA使运行和调试React Native应用程序非常灵活。例如，如果您是第一次启动应用程序，则可以选择运行React Native捆绑器，构建应用程序，并在模拟器上打开它 - 所有这些都是运行或调试会话的一部分。如果捆绑程序已在运行，您也可以跳过启动它，如果自上次运行后未对其本机代码进行任何更改，则可以拒绝构建应用程序。

如果您使用的是Expo，则可能需要更改默认的bundler主机以确保与调试器成功连接。此连接可能会失败，因为默认情况下IntelliJ IDEA使用localhost开始调试，而Expo需要127.0.0.1或外部IP地址，具体取决于Metro捆绑器配置的“连接”字段中选择的内容。有关详细信息，请参阅调试使用Expo的React Native应用程序 。

创建React Native运行/调试配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择React Native 。运行/调试配置：React Native打开。

  2. 选择是否要IntelliJ IDEA为您构建和启动应用程序：

    • 如果您是第一次启动应用程序，或者自上次运行以来您已更新其本机代码，请选中“ 生成并启动”复选框。

    • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，IntelliJ IDEA会等待您在模拟器中打开您的应用程序，并启用远程调试，如React Native官方网站上所述。

    • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序会自行处理该过程。

  3. 如果您选择了Build and launch复选框，请选择目标平台Android或iOS。根据您的选择，IntelliJ IDEA将使用react-native run-ios或react-native run-android运行bundler。

    （可选）键入要传递给React Native的参数，例如，通过‑‑simulator标志指定模拟器类型： - ‑‑simulator="iPhone 4s" 。

  4. 在Bundler主机字段中，指定运行React Native捆绑程序的主机，默认值为localhost 。

  5. 在Bundler端口字段中，指定运行React Native捆绑器的端口，默认选择8081 ，从React Native官方网站了解更多信息。

  6. 选择要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    指定react-native-cli的路径和应用程序的工作目录 。（可选）为react-native run-android或react-native run-ios键入环境变量。

  7. 默认情况下，当您调用运行/调试配置时，IntelliJ IDEA会自动启动React Native捆绑器。如果您已从IntelliJ IDEA外部启动了捆绑程序，例如，从命令行启动捆绑程序，则可以在不停止和重新启动的情况下重新使用它。在“ 启动前”区域中选择您的捆绑器，然后单击 。

准备Android设备

  • 启动Android虚拟设备或启用USB调试并通过USB 连接到物理Android设备 。从React Native官方网站了解更多信息。

准备iOS模拟器

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install --global ios-sim

运行应用程序

  • 在工具栏上的“ 选择运行/调试配置”列表中选择新创建的React Native配置，然后单击“确定” 在它的旁边。IntelliJ IDEA打开“ 运行”工具窗口，首先在新的“ React Native”选项卡中启动React Native捆绑器。之后，执行react-native run-ios或react-native run-android命令，具体取决于所选的目标平台。如果构建成功，模拟器将显示您的应用程序：

调试应用程序

适用于MacO和Linux版本的Chrome 59及更高版本以及Windows版本60及更高版本的Headless模式。

使用IntelliJ IDEA，您可以调试使用本机React Native捆绑程序和使用Expo的应用程序。

在调试React Native应用程序时，IntelliJ IDEA依赖于Chrome运行时，它由React Native本身使用。您还可以将DevTools与IntelliJ IDEA一起使用。当您启动调试会话时，IntelliJ IDEA会启动一个新的Chrome实例并附加到它。如果您不想看到任何新的Chrome窗口，请使用Chrome无头模式 。

调试使用本机捆绑程序的React Native应用程序

  1. 根据需要在代码中设置断点。

  2. 如上所述创建新的React Native运行/调试配置。如果捆绑器已在IntelliJ IDEA外部运行，请在“启动前”区域中选择它，然后单击 。

  3. 要启动应用程序，请从工具栏上的“ 选择运行/调试配置”列表中选择新创建的React Native配置，然后单击在它的旁边。IntelliJ IDEA打开运行工具窗口并在新的React Native选项卡中运行bundler。

  4. 完成构建并且应用程序出现在模拟器中时， 打开In-App Developer菜单并选择Debug JS Remotely 。从React官方网站了解更多信息。

    内置的IntelliJ IDEA调试器连接到模拟器。

  5. 当第一个断点被点击时，继续调试会话 - 逐步执行断点 ，在帧之间切换，即时更改值， 检查暂停的程序 ， 计算表达式和设置监视 。

调试使用Expo的React Native应用程序

  1. 根据需要在代码中设置断点。

  2. 如上所述创建新的React Native运行/调试配置并更改默认设置，如下所示：

    1. 清除构建和启动复选框，因为此捆绑程序负责处理过程本身。

    2. 在Bundler主机字段中，将默认localhost设置更改为127.0.0.1或外部IP地址，具体取决于在Metro捆绑程序配置的“连接”字段中选择的内容。

    3. 在Bundler端口字段中，将默认8081设置更改为19001 。

    4. 在Before launch区域中，选择默认选择的Start React Native Bundler任务，然后单击 。然后，在Configure React Native对话框中，选择npm script ，并选择start或启动应用程序的其他任务。

  3. 选择新创建的配置，然后单击 。

  4. 在手机或模拟器上打开您的Expo客户端应用程序，选择当前应用程序，并启用远程调试，从Expo官方网站了解更多信息。

  5. 当第一个断点被命中时，继续调试使用本机捆绑程序的应用程序 。

在IntelliJ IDEA中配置Chrome Headless

  1. 点击在Browser for debugging字段旁边。“ Web浏览器”对话框将打开。

  2. 选择Chrome ，然后点击 ，并将复制的配置重命名为例如Chrome Headless 。

  3. 选择新的浏览器配置，单击 ，并在Chrome设置对话框中键入--headless用于macOS和Linux的无头 - 或用于Windows的--headless --disable-gpu 。

  4. 在运行/调试配置中，从浏览器中选择新的Chrome Headless配置以进行调试 。
