按Ctrl + Alt + S

单击时将打开该对话框在PHP页面的Development environment部分的CLI Interpreter下拉列表旁边。

使用此对话框将PHP引擎配置为解释器 ，请参阅配置本地PHP解释器和配置远程PHP解释器 。在此对话框中，您可以添加新的解释器并编辑或删除现有的解释器。

该对话框由两个窗格组成。左侧窗格列出了所有已配置的PHP解释器，包括本地和远程解释器，并包含用于添加，删除和复制PHP解释器配置的工具栏。右侧窗格的内容取决于左侧窗格中当前选择的解释器的类型。使用此窗格中的控件编辑所选解释器的设置，如有必要，甚至可以选择其类型。

左侧窗格

左侧窗格列出了所有已配置的PHP解释器，包括本地和远程解释器，并包含用于添加，删除和复制PHP解释器配置的工具栏。

项目

提示

描述

加

单击此按钮可将新的PHP解释器添加到列表中。从下拉列表中，选择解释器的类型（ 本地或远程 ），并在右侧窗格中配置本地解释器，或在打开的“ 配置PHP远程解释器对话框”对话框中配置远程解释器 。

删除

单击此按钮可从列表中删除选定的解释器。

复制

单击此按钮可创建一个新的解释器，其中包含从所选设置复制的设置。

右侧窗格

右侧窗格的内容取决于左侧窗格中当前选择的解释器的类型。使用此窗格中的控件编辑所选解释器的设置，如有必要，甚至可以选择其类型。

口译员类型

描述

本地

  • 名称 ：在此字段中，键入标识符以将解释器与其他解释器区分开来，例如， php_installation_<version> 。

  • PHP可执行文件 ：在此字段中，指定PHP引擎的路径。手动键入路径或单击并在打开的对话框中选择可执行文件。IntelliJ IDEA检测PHP的版本并将其显示在PHP版本的只读字段中。

    IntelliJ IDEA还检测调试引擎和php.ini配置文件。与PHP解释器及其版本关联的调试引擎的类型显示在Debugger只读字段中。如果未检测到调试器或您已在php.ini文件中禁用它（请参阅在按需模式下配置Xdebug以使用 ），则该字段显示Debugger：Not installed 。

    php.ini的位置显示在配置文件只读字段中。要编辑IntelliJ IDEA中的php.ini ，请单击“ 在编辑器中打开” 。

  • （ 重新加载 ）：单击此按钮以检查指定的PHP主目录是否实际包含PHP可执行文件。如果在指定位置未检测到PHP可执行文件，IntelliJ IDEA将显示相应的错误消息。

  • （ 显示phpinfo ）：单击此按钮可使IntelliJ IDEA显示单独的信息窗口，其中包含安装详细信息以及已加载扩展和已配置选项的列表。请注意，未列出CLI解释器对话框的“ 配置选项”字段中指定的其他选项 。

  • 调试器扩展：在此字段中，指定Xdebug扩展的位置，以使IntelliJ IDEA在必要时激活它。

    IntelliJ IDEA 2016.2及更高版本支持按需模式，您可以在其中禁用全局PHP安装的调试器，并且只有在调试命令行脚本或需要代码覆盖率报告时才能按需自动启用它。这使您的命令行脚本（包括Composer和单元测试）运行得更快。

  • 配置选项 ：使用此字段通过编写要通过-d命令行选项传递的配置指令字符串来自定义安装的配置设置，从而将新条目添加到php.ini文件中。点击打开“ 配置选项”对话框并在其中创建新的php.ini条目列表。

  • 用户名：在该字段中，键入在SSH服务器上注册的用户名。

  • 验证类型：从此列表中选择验证方法。

    • 密码 ：使用密码访问主机。指定密码，然后选择保存密码复选框以将密码保存在IntelliJ IDEA中。

    • 密钥对（OpenSSH或PuTTY） ：使用密钥对的SSH身份验证 。要应用此身份验证方法，您必须在客户端计算机上拥有私钥，并在远程服务器上拥有公钥。IntelliJ IDEA支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，并在相应字段中键入密码（如果有）。要让IntelliJ IDEA记住密码，请选中Save passphrase复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭据帮助应用程序管理的SSH密钥（例如，Windows上的Pageant或Mac和Linux上的ssh-agent ）。
