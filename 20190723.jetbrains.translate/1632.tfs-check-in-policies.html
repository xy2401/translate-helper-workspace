TFS登机政策

签入策略是在每次签入之前执行的规则，以确保所选的变更集可以提交。标准策略存储在服务器上，并在客户端计算机上执行。

自定义策略实现为IntelliJ IDEA的自定义插件。这些插件的ID存储在服务器上，而策略本身则在本地应用。因此，要在团队中使用策略，所有团队成员都应安装相应的插件。

在本页面：

  • 定义默认策略设置
  • 禁止在当前项目中使用默认的IntelliJ IDEA范围的签入策略设置
  • 管理可用策略列表
  • 引入自定义签到策略

定义要在IntelliJ IDEA级别应用的默认策略设置

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，选中Checkin策略兼容性区域中的适用复选框。

    • 评估团队资源管理器策略：选中此复选框以在客户端计算机上安装并执行Microsoft团队资源管理器策略定义。

    • 评估Teamprise策略：选中此复选框以在客户端计算机上安装并执行Teamprise策略定义。

    • 警告未安装的策略：选中此复选框可在未安装指定的策略定义的情况下显示警告。

禁止将默认签入策略设置应用于项目

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。

  3. 在“ 管理TFS服务器和工作空间”对话框中，打开该对话框，从“ 团队项目”下拉列表中选择有问题的项目 。

  4. 在“ 兼容性”区域中，选中“ 替代团队项目<项目名称>的默认设置”复选框。

  5. 通过选择或清除下面相应的复选框重新定义默认设置。

    • 评估团队资源管理器策略：选中此复选框以在客户端计算机上安装并执行Microsoft团队资源管理器策略定义。

    • 评估Teamprise策略：选中此复选框以在客户端计算机上安装并执行Teamprise策略定义。

    • 警告未安装的策略：选中此复选框可在未安装指定的策略定义的情况下显示警告。

管理可用策略列表

可用策略列表包括标准第三方策略和自定义用户定义策略。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。

  3. 在打开的“ 管理TFS服务器和工作空间”对话框中，选择所需的工作空间，然后单击“ 签入策略”按钮。

  4. 在打开的“ 编辑签入策略”对话框中，配置策略列表：

    • 要激活策略，请选中它旁边的已启用复选框。

    • 要取消策略，请清除其旁边的“已启用”复选框。

    • 要永久丢弃策略，请在列表中选择该策略，然后单击“ 删除”按钮。

引入自定义登记政策

  1. 将所需策略实现为自定义插件。

  2. 按照管理插件中的说明下载，安装和启用插件 。
