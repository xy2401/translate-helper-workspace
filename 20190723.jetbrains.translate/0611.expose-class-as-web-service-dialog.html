将类公开为Web服务对话框

Alt + Enter - 将类公开为Web服务

该对话框仅通过专门的意图操作可用。要调用它，请将光标放在类名称上，然后按Alt + Enter或单击黄色灯泡图标 。

使用该对话框为整个类配置Web服务WSDL生成，并将其所有方法公开为Web服务操作并进行部署。对话框的内容取决于Web服务类型。

项目

描述

Web服务类型

服务名称

使用此下拉列表来指定服务的名称出版（如为空文本ROOT上下文或mycontext用于/mycontext ）。

Apache Axis

服务类名称

在此文本框中，指定要作为Web服务公开的类。

所有

服务命名空间

在此文本框中，键入Web服务名称空间前缀。

Apache Axis

服务方式

使用此下拉列表指定要生成的WSDL文档的样式。可用选项包括：
  • RPC - 选择此选项以生成rpc WSDL。

    默认情况下选择此选项。

  • 文档 - 选择此选项以生成文档 WSDL。

  • Wrapped - 选择此选项以使用包装方法生成WSDL。

    如果选择此选项，则会在“ 使用项目绑定”下拉列表中自动预选Literal 。

Apache Axis

项目的使用

从此下拉列表中，选择应如何使用生成的WSDL文档。可用选项包括：

  • Literal - 选择此选项时，XML Schema定义请求的XML表示形式。

  • 编码 - 选择此选项可在生成的WSDL中指定SOAP编码。

Apache Axis

所有
