File Watcher是一个WebStorm系统，用于跟踪文件的更改并运行第三方独立应用程序。WebStorm为许多此类标准流行的第三方工具（ 编译器 ，压缩器 ，美化器和其他工具）提供了预定义的文件监视器模板。您还可以配置自定义文件监视器以运行任何其他第三方工具。

文件观察者有两个专门的代码检查 ：

  • 文件观察程序可用检查在适用预定义文件监视器的每个文件中运行。如果项目没有配置相关的文件监视器，WebStorm建议添加一个。

  • 文件观察器问题检查由正在运行的文件监视器调用，并突出显示特定于它的错误。

该页面由两部分组成：

  • 当前项目中可用的文件监视器列表。要激活文件观察程序 ，请选中它旁边的复选框。如果在文件观察程序运行时发生错误，则会自动禁用文件监视器 。

  • 用于管理此列表的工具栏。

工具提示/
和捷径

描述

加
Alt +插入

单击此按钮可打开“ 选择模板”弹出窗口，然后选择“ 文件观察器”的相关类型。之后，WebStorm打开New Watcher对话框，根据当前项目的设置自定义预定义的File Watcher 。

编辑
输入

单击此按钮可在“ 编辑观察器”对话框中更新所选文件观察器的设置。此更新仅应用于当前项目File Watcher ，它不会影响预定义的WebStorm级别模板。

去掉
Alt + Delete键

单击此按钮可删除选定的文件观察程序 。文件观察程序不再应用于当前项目中的文件。请注意，此操作不会影响WebStorm级别仍可用的相应预定义模板。

 

向上（ Ctrl + Alt +向上 ）
向下（ Ctrl + Alt +向下 ）

使用这些按钮更改列表中文件监视器的顺序。如果启用了多个文件监视器 ，则这将确定启动文件监视器的顺序。

复制

使用此按钮可以创建所选文件观察程序的副本。

进口

单击此按钮可导入现有文件观察程序并将其添加到可用文件查看器列表中。

出口

单击此按钮可将选定的观察者导出到位于用户家中的watchers.xml文件。
