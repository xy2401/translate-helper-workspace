项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 要在相应字符串值之前添加的字符序列。

  • 后缀。 要在相应字符串值之后添加的字符序列。

前缀和后缀是可选的。有关详细信息，请参阅 。

XML标记

在此区域中，指定属性受定义的配置影响的XML标记。
  • 本地名称 - 在此文本框中，指定不带名称空间前缀的标记名称。使用正则表达式指定多个标记名称（ name1|name2 ），不区分大小写的名称（ (?i)tagname匹配标记名以及TagName ）等。

    不允许使用空格字符，因为它们会影响匹配结果。

  • 命名空间 - 在此文本框中，指定XML标记的命名空间URI。

两个字段都是可选的。但是，如果“ 本地名称”文本框为空，则配置将应用于与配置的名称匹配的任何属性，而不管其是否包含XML标记。

XML属性

在此区域中，定义XML标记属性，该属性指示包含在具有此类属性的标记中的文本应被视为所选语言。
  • 本地名称 - 在此文本框中，指定不带名称空间前缀的属性名称。使用正则表达式：例如，要匹配HTML事件处理程序属性，请在文本框中键入on.* 。

    除非XML标记区域中的本地名称文本框为空，否则该字段是可选的。如果未指定属性本地名称，则配置将应用于封闭标记的所有属性。

  • 命名空间 - 在此文本框中，指定属性的命名空间URI。

高级

在此区域中，指定其他设置以对注射过程进行更细粒度的控制。
  • 值模式 - 在此文本框中，键入一个正则表达式，该表达式确定要将语言注入的属性值的一部分。通过使用模式的第一个捕获组作为注入目标，可以将过程配置为仅将语言注入与特定模式匹配的值或注入与模式匹配的多个部分。
    • 单个文件 - 如果选项关闭，则与值模式匹配的片段将作为不同的“文件”单独处理 - 例如从片段编辑器的角度。

      如果选项打开，则相应的片段将合并在一起以形成单个单元或“文件”。

      鉴于价值模式

      xxx（。+）yyy（。+）zzz

      和片段

      xxx select * yyy from family zzz ，

      如果选项关闭， select *和from family将被视为两个独立的片段（或“文件”）。如果选项处于启用状态，则select * from family将被视为单个单元或“文件”。

  • XPath条件 - 在此文本框中，指定XPath表达式以更精确地处理注入目标。评估表达式的上下文是属性本身。

    要使该字段处于活动状态，必须启用XPathView + XSLT支持插件 。
