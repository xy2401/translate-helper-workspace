File Cache Conflict

An external process has changed a file, opened and unsaved in WebStorm, which results in two conflicting versions of a file. Resolve this conflict using the following options.

Item

Description

Load FS Changes

Click this button to load the file version produced outside of WebStorm, and overwrite your local changes.

Keep Memory Changes

Click this button to preserve the version produced in WebStorm and stored in cache.

Show Difference

Click this button to invoke the differences viewer that shows the version in the file system to the left, and WebStorm version to the right. You can merge differences as required and load the desired version to WebStorm. By default, the cache version is saved.
