初步步骤

安装Erlang OTP

设置Erlang环境的第一件事是安装Erlang OTP，这是一组对开发至关重要的Erlang库。

视窗

如果您是Windows用户，请下载Erlang OTP包并运行安装向导。安装完成后，将安装路径+ \bin到PATH环境变量中。

苹果系统

如果您是macOS用户，要安装Erlang OTP，请在终端提示符下键入以下内容（确保在您的计算机上安装了Homebrew ）：

brew安装erlang

如果您更喜欢MacPorts到Homebrew，您的命令行应该是不同的：

port install erlang + ssl

Linux的

Linux的安装过程与macOS类似，不同之处在于您必须使用apt-get （Linux软件包管理实用程序）而不是brew或port ：

apt-get install erlang

您可以随时为任何操作系统下载最新版本的Erlang OTP软件包。

验证Erlang OTP安装

要验证是否正确安装了Erlang OTP，请在终端提示符下键入erl来运行Erlang shell：

要了解有关Erlang shell的更多信息，请阅读其用户指南 。

安装钢筋

除了Erlang OTP之外，你还需要Rebar ，一个帮助编译和测试Erlang应用程序的构建工具。在您的计算机上安装它的最简单方法是下载其源并在本地构建它：

git clone git：//github.com/rebar/rebar.git $ cd rebar $ ./bootstrap重新编译：src / getopt ...重新编译：src / rebar_utils == >钢筋（编译）

恭喜！您现在在当前工作目录中有一个名为“rebar”的自包含脚本。将此脚本放在路径中的任何位置，您可以使用rebar构建符合OTP的应用程序。

设置IntelliJ IDEA

现在，当设置Erlang OTP和Rebar时，是时候下载并安装IntelliJ IDEA了。请记住，对于Erlang开发，您可以使用IntelliJ IDEA Community Edition（免费和开源）。

IDE启动后，您会看到其欢迎屏幕 ，请转到配置|插件 ，然后单击浏览存储库 ，找到Erlang插件并安装它：

安装插件后，重新启动IntelliJ IDEA。

配置Erlang SDK

配置IntelliJ IDEA还需要做的一件事就是添加一个Erlang SDK。

为此，请更改默认项目的结构。以下列两种方式之一打开默认项目结构：

  • 在欢迎屏幕上 ，转到配置|项目默认值|项目结构

  • 在主菜单上，选择“ 文件”|其他设置|新项目的结构

然后，通过指定Erlang OTP安装目录的路径来添加Erlang SDK。

如果您不知道安装了Erlang OTP的位置，请检查以下目录：

  • Windows ： C:\Program Files\erl<version>

  • Linux ： /usr/lib/erlang/<version>

  • MacPorts，macOS ： /opt/local/lib/erlang/<version>

  • 自制软件，macOS ： /usr/local/Cellar/erlang/<version>

创建一个钢筋项目

您可能希望创建一个Rebar项目，而不是纯Erlang项目。为此，请在终端提示符下键入以下代码：

rebar create-app appid = <项目名称>

创建项目后， 将其导入IntelliJ IDEA，以便在IDE中打开此项目。

将项目导入IntelliJ IDEA

您可以通过多种方式将项目导入 IntelliJ IDEA的。让我们从欢迎屏幕探索导入。

要将现有项目导入IntelliJ IDEA，请在“欢迎”屏幕上单击“ 导入 ”，然后选择项目目录。IntelliJ IDEA允许您从现有源导入项目，或从外部模型（构建文件）导入项目。

如果您的项目使用Rebar，请在询问时选择相应的选项。
