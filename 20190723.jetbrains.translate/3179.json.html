WebStorm 2019.1帮助

JSON

JSON格式通常用于存储数据和配置文件。WebStorm可以帮助您处理JSON文件 - 它会检查它们的语法和格式。在流行的配置文件类型中，WebStorm提供了代码完成功能，这要归功于JSON Schema ，它是一种用于描述此类文件的结构和内容的特殊格式。您还可以使用自定义JSON模式在JSON文件中启用代码完成并验证它们。

启用JSON5

WebStorm认可了许多最受欢迎的JSON标准，包括JSON5 。WebStorm默认将json5扩展名的文件视为JSON5文件，并在其中支持这种新语法。

将JSON5语法扩展为所有JSON文件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|文件类型 。将打开“ 文件类型”页面。

  2. 在“已识别的文件类型”列表中，选择“ JSON5” 。

  3. 在“已注册的模式”区域中，单击并在打开的“ 添加通配符”对话框中键入*.json 。

使用JSON Schema Store中的模式

JSON Schema Store中的模式也可以应用于YAML文件。

WebStorm可以自动从JSON架构存储中下载和使用架构，该架构托管许多常用配置文件的架构文件。只要打开名称与其中一个可用模式关联的文件（例如tslint.json ），WebStorm就会下载并使用此模式。应用架构的名称显示在状态栏上。

如果配置文件具有自定义名称，请单击状态栏上的“ 无JSON模式” ，然后从列表中选择所需的模式，或单击“ 新建模式映射”以打开“ JSON模式映射”页面并配置新的自定义模式 。

默认情况下，启用从JSON架构存储自动下载架构。如果由于某种原因它被关闭，您可以随时再次启用它。

从JSON架构存储启用自动下载架构

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”|架构和DTD |远程JSON模式 。

  2. 选中“ 允许从远程源下载JSON模式”和“ 使用schemastore.org JSON模式目录”复选框。

WebStorm捆绑了许多流行的模式。虽然这些模式会定期自动更新，但它们仍可能过时。

使用捆绑模式的最新版本

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”|架构和DTD |远程JSON模式 。

  2. 选中始终下载最新版本的模式复选框。

使用自定义JSON模式

自定义架构必须符合JSON架构标准 。

如果指定URL，请确保选中“ 允许从远程源下载JSON模式”复选框“ 远程JSON模式”页面。

WebStorm仅在当前项目中搜索具有指定名称的文件和文件夹，因此您无需指定文件和文件夹的完整路径。

除了来自JSON Schema Store的模式 ，WebStorm还允许您配置和使用来自其他存储的自定义模式。您可以下载所需的模式并将其存储在项目根目录下，或指定资源的URL，以便WebStorm可以自动下载模式。

配置自定义JSON架构

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”|架构和DTD | JSON模式映射 。将打开JSON Schema Mappings页面。

  2. 在中央窗格中，显示所有先前配置的自定义架构，单击在工具栏上。

  3. 指定架构的名称以及架构符合的架构规范版本 。在“ 架构文件或URL”字段中，指定先前下载的架构文件的位置，或键入所需架构可用的URL。

  4. 创建要针对此架构验证的文件或文件夹列表。该列表可能包含特定文件的名称，整个目录的名称和文件名模式。根据列表，WebStorm在内部检测要验证的文件。

    要将项目添加到列表，请单击并指定文件或文件夹的路径或键入文件模式。

从远程源启用JSON模式的自动下载

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”|架构和DTD |远程JSON模式 。

  2. 选择允许从远程源下载JSON模式 。

    清除该复选框后，将禁用JSON架构周围的任何网络活动，包括来自JSON架构存储的架构。

处理模式范围之间的冲突

当文件，文件夹或模式属于两个或多个模式的范围时，会发生冲突。WebStorm以两种模式分析范围：

  • 静态分析检测自定义架构范围内的冲突。如果检测到冲突，WebStorm将在“ 架构详细信息”窗格中显示警告。要查看重叠范围，请单击“ 显示详细信息”链接。WebStorm显示一个弹出窗口，其中列出了冲突的范围和模式：

  • 动态分析检测系统和自定义架构范围内的冲突。当您打开属于特定范围的文件时，将启动此类分析。如果检测到冲突，WebStorm会在编辑器选项卡的顶部显示警告：

    单击该链接以打开JSON Schema Mappings页面并编辑冲突的自定义架构的范围。请注意，您无法编辑系统架构的范围。

在JSON模式中使用HTML描述

配置语法突出显示

您可以根据自己的喜好和习惯配置支持JSON的语法突出显示。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案| JSON 。

  2. 选择颜色方案，接受从默认值继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
