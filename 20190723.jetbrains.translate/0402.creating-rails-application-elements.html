创建Rails应用程序元素

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

IntelliJ IDEA可帮助您使用所有必需元素的存根来填充Rails应用程序：控制器，视图，模型等。您可以通过以下几种方式启动Rails生成器：

  • 按Ctrl键两次，然后开始键入所需的命令或其部分（例如， rails g controller ）。然后，从建议列表中选择此命令，然后按Enter键 。

  • 使用工具|运行Rails Generator菜单命令。

  • 转到文件|新建 （ Alt + Insert ）并选择Run Rails Generator 。

在本主题中，我们将向您展示如何创建Rails应用程序元素的几个示例。

创建控制器和操作

要生成控制器及其操作（可选），请执行以下操作：

  1. 按Ctrl键两次，然后开始键入rails g controller 。选择rails g controller并按Enter键 。

  2. 在调用的“ 添加新控制器”对话框中，指定控制器名称。（可选）添加由空格分隔的操作名称（例如， new create ）。单击确定 。

    其他选项允许您提供其他参数。例如，您可以添加--no-test-framework参数。

  3. IntelliJ IDEA生成所需文件（控制器，视图和其他）并在“运行”工具窗口中显示输出。

    您可以单击任何文件以在编辑器中快速打开它。

从操作创建视图

您可以在编辑器中为指定的控制器操作创建视图。去做这个：

  1. 在编辑器中打开控制器文件。

  2. 找到所需的操作，然后单击编辑器左侧排水沟中的图标。

  3. 如果存在与该方法关联的视图，则在编辑器中打开该视图。如果视图不存在，IntelliJ IDEA建议创建一个新视图。

    指定视图文件的名称，然后单击“ 确定” 。请注意，您可以创建ERB或Haml视图。

创建模型

要创建模型，请执行以下步骤：

  1. 按两次Ctrl键开始键入rails g model 。选择rails g model并按Enter键 。

  2. 在调用的“ 添加新模型”对话框中，指定模型名称，属性及其类型（例如， 文章标题：字符串文本：文本 ）。单击确定 。

    按Ctrl + Space可完成属性类型。

  3. IntelliJ IDEA生成所需文件（模型和迁移）并在“运行”工具窗口中显示输出。

    您可以单击任何文件以在编辑器中快速打开它。

    要查看已创建模型的依赖关系图，请按Ctrl + Shift + Alt + U ，在调用的弹出窗口中选择Rails Model Dependency Diagram ，然后按Enter键 。

运行迁移

要运行迁移，您需要执行相应的Rake命令：

  1. 按Ctrl键两次，然后输入rake db：migrate 。选择rake db:migrate并按Enter键 。

  2. 在调用的对话框中，您可以选择最新的或选择特定的迁移。单击确定 。

  3. IntelliJ IDEA执行此迁移并在“运行”工具窗口中显示创建的表。
