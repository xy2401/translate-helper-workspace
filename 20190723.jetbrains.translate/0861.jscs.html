JSCS

此功能仅在Ultimate版本中受支持。

在开始之前，请确保您的计算机上有Node.js.

编辑JSCS配置文件时，IntelliJ IDEA会检测错误并提供代码完成。

IntelliJ IDEA允许您使用JSCS linter验证JavaScript代码。

要全局安装JSCS

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install jscs -g

在IntelliJ IDEA中启用JSCS并配置其行为

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |代码质量工具| JSCS 。在打开的JSCS页面上，选中“ 启用”复选框。之后，页面上的其他控件变为可用。

  2. 在Node Interpreter字段中，指定Node.js的路径。如果您遵循标准安装过程，IntelliJ IDEA会检测路径并填写字段本身。

  3. 在JSCS Package字段中，指定当前项目中安装的jscs包的位置，请参阅安装JSCS 。

  4. 指定要使用的配置。

    默认情况下，IntelliJ IDEA首先在当前项目的package.json文件中查找jscsConfig属性。如果未找到此类属性，IntelliJ IDEA将查找.jscsrc或.jscs.json配置文件。IntelliJ IDEA从存储要检查的文件的文件夹开始搜索，然后在父文件夹中搜索，依此类推，直到到达项目根目录。因此，您必须将配置定义为在package.json文件或.jscsrc或.jscs.json配置文件中或在自定义JSON配置文件中应用为jscsConfig属性。

    您还可以单独应用预定义的规则集，也可以与配置文件结合使用。在后一种情况下，配置文件中的规则会覆盖预定义的规则。

    • 要让IntelliJ IDEA在package.json文件或.jscsrc或.jscs.json文件中查找jscsConfig属性，请选择Search for config（s）选项。

    • 要使用自定义文件，请选择“ 配置文件”选项，并在“ 路径”字段中指定文件的位置。从列表中选择路径，或手动键入，或单击并从打开的对话框中选择相关文件。

    • 要应用预定义的一组或规则，请从“ 代码样式预设”列表中选择所需的一组。

  5. 如有必要，从“ 代码样式预设”列表中，选择与您使用的代码样式关联的预定义规则集。
