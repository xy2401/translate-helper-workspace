语言注入设置对话框：XML标记注入

项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 要在相应字符串值之前添加的字符序列。

  • 后缀。 要在相应字符串值之后添加的字符序列。

前缀和后缀是可选的。有关详细信息，请参阅 。

XML标记

在此区域中，定义XML标记，该标记指示此标记中包含的文本应被视为所选语言。
  • 本地名称 - 在此文本框中，指定不带名称空间前缀的标记名称。使用正则表达式指定多个标记名称（ name1|name2 ），不区分大小写的名称（ (?i)tagname匹配标记名以及TagName ）等。

    请务必不要输入任何空格字符，因为它们对于匹配很重要。

  • 命名空间 - 在此文本框中，指定XML标记的命名空间URI。

    该字段是可选的。

子标记

选中此复选框以递归方式包含所有子标签。

高级

在此区域中，指定其他设置以对注射过程进行更细粒度的控制。
  • 值模式 - 在此文本框中，键入一个正则表达式，该表达式确定要将语言注入的XML文本值的一部分。通过使用模式的第一个捕获组作为注入目标，可以将过程配置为仅将语言注入与特定模式匹配的值或注入与模式匹配的多个部分。
    例子：
    [$＃] \ {（。*？）\}匹配JSP / JSF表达式语言使用的模式。
    ^javascript:(.*)匹配可以在hyperlink-hrefs中用于执行JavaScript代码的javascript协议。
    • 单个文件 - 如果选项关闭，则与值模式匹配的片段将作为不同的“文件”单独处理 - 例如从片段编辑器的角度。

      如果选项打开，则相应的片段将合并在一起以形成单个单元或“文件”。

      鉴于价值模式

      xxx（。+）yyy（。+）zzz

      和片段

      xxx select * yyy from family zzz ，

      如果选项关闭， select *和from family将被视为两个独立的片段（或“文件”）。如果选项处于启用状态，则select * from family将被视为单个单元或“文件”。

  • XPath条件 - 在此文本框中，指定XPath表达式以更精确地处理注入目标。评估表达式的上下文是周围的XML标记。

    可以使用Jaxen XPath引擎提供的XPath 扩展函数 ，例如lower-case() 。此外，还有三个附加功能可用于确定当前文件的名称，扩展名和文件类型： file-name() ， file-ext()和file-type() 。或者，可以通过标准代码完成来检索可用功能的列表。

    要使该字段处于活动状态，必须启用XPathView + XSLT支持插件 。

    出于性能原因，建议您尽可能简化这些表达式。特别是导致整个文档被扫描的表达式，例如//foo/bar可能会导致大文件出现性能问题。
