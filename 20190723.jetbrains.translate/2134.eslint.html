ESLint

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

默认情况下，PyCharm根据ESLint配置中的严重性级别标记检测到的问题。请参阅配置ESLint突出显示以了解如何覆盖这些设置。

在开始之前，请确保您的计算机上有Node.js.

PyCharm与ESLint集成，它带来了各种各样的linting规则，也可以通过插件进行扩展。当您键入时，PyCharm会在编辑器中显示ESLint报告的警告和错误。使用ESLint，您还可以使用JavaScript标准样式 。

要查看问题的描述，请将鼠标悬停在突出显示的代码上。

安装ESLint

  1. 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install eslint -g用于全局安装。

    • npm install eslint --save-dev将ESLint安装为开发依赖项。

  2. （可选）安装其他插件，例如eslint-plugin-react to lint React应用程序。

在PyCharm中激活和配置ESLint

了解有关从ESLint官方网站配置ESLint的更多信息。

从ESLint官方网站了解有关ESLint CLI选项的更多信息。

默认情况下，PyCharm使用项目node_modules文件夹中的ESLint包和存储当前文件的文件夹中的.eslint。*配置文件。如果在当前文件夹中找不到.eslint。* ，PyCharm将在其父文件夹中查找一个直到项目根目录。

如果你有几个将ESLint列为依赖项的package.json文件，PyCharm会为每个package.json启动一个单独的进程并处理它下面的所有内容。这使您可以将特定的ESLint版本或特定的插件集应用于monorepo中的每个路径或具有多个ESLint配置的项目。

在所有新的PyCharm项目中，此行为是默认的。要在先前创建的项目中启用它，请转到语言和框架| JavaScript | 设置/首选项对话框中的ESLint （ Ctrl + Alt + S ），然后选择自动ESLint配置选项。

您也可以手动配置ESLint以使用自定义ESLint包和配置。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript | ESLint 。

  2. 选择“ 手动配置”选项以使用自定义ESLint包和配置。

  3. 在Node Interpreter字段中，指定Node.js的路径。如果您遵循标准安装过程，PyCharm会检测路径并填充字段本身。

  4. 在ESLint Package字段中，指定eslint或standard包的位置。

  5. 选择要使用的配置。

    • 使用自动搜索 ，PyCharm会查找.eslintrc文件或尝试检测package.json中 eslintConfig下定义的配置。PyCharm首先在文件夹中查找.eslintrc或package.json ，然后在其父文件夹中，依此类推到项目根目录。

    • 选择“ 配置文件”以使用自定义文件，并在“ 路径”字段中指定文件位置。

  6. 可选：

    • 在Extra ESLint Options字段中，指定运行ESLint的其他命令行选项，使用空格作为分隔符。

    • 在“ 附加规则目录”字段中，使用其他代码验证规则指定文件的位置。这些规则将在.eslintrc或上面指定的自定义配置文件的规则之后应用，因此将覆盖它们。

配置ESLint的突出显示

有关详细信息，请参阅代码检查和配置检查严重性 。

默认情况下，PyCharm根据ESLint配置中的严重性级别标记检测到的错误和警告。例如，错误用红色波浪线突出显示，而警告用黄色背景标记。

更改ESLint配置中规则的严重性级别

  • 在.eslintrc或下eslintConfig中的package.json，找到要修改其ID设置为规则1 （ warn ）或2 （ error ）。从ESLint官方网站了解更多信息。

您可以从ESLint配置中覆盖严重性，以便PyCharm忽略它们并将linter报告的所有内容显示为错误，警告或自定义颜色。

忽略配置中的严重性级别

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 编辑器”|检查 。将打开“ 检查”页面 。

  2. 在中央窗格中，转到JavaScript |代码质量工具| ESLint 。

  3. 在右侧窗格中，清除“ 从配置文件中使用规则严重性”复选框，然后选择要使用的严重性级别而不是默认级别。

从ESLint导入代码样式

您可以将一些ESLint代码样式规则导入PyCharm JavaScript代码样式设置 。这使PyCharm能够在自动完成，生成或重构代码或添加import语句时为您的项目使用更准确的代码样式选项。当您使用重新格式化操作时，PyCharm将不再从ESLint透视图中破坏格式正确的代码。

PyCharm了解所有官方格式的ESLint配置： .eslintrc JSON文件，带有eslintConfig字段的package.json文件，以及JavaScript和YAML配置文件。

  • 当您第一次打开项目时，PyCharm会自动从项目ESLint配置中导入代码样式。

  • 如果更新了ESLint配置（手动或从版本控制），请在编辑器中将其打开，然后从上下文菜单中选择“ 应用ESLint代码样式规则” 。

    或者，只需对“从ESLint应用代码样式”回答“ 是 ”？“文件顶部的问题。

    应用规则列表显示在“ 事件日志”工具窗口中：

使用JavaScript标准样式

您可以将JavaScript标准样式设置为应用程序的默认JavaScript代码样式，以便在键入代码或重新格式化代码时应用其主要规则。由于Standard基于ESLint，您还可以通过PyCharm ESLint集成使用Standard。

安装JavaScript标准版

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install standard --save-dev

    从JavaScript Standard Style官方网站了解更多信息。

通过ESLint启用标准linting

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |代码质量工具| ESLint 。

  2. 在打开的ESLint页面上，选中“ 启用”复选框，并在“ ESLint包”字段中指定standard包的位置。

将JavaScript标准样式设置为默认值

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|代码风格| JavaScript ）。

  2. 关于代码风格。打开的JavaScript页面，单击“ 设置自” ，然后选择“ 预定义样式” JavaScript标准风格 。该样式将取代您当前的方案。

如果打开项目的package.json文件中列出standard的项目，PyCharm会自动使用标准进行linting。

使用ESLint的Linting TypeScript代码

另见TSLint 。

从@ typescript-eslint / parser官方文档中了解更多信息。

如果您全局安装了ESLint，请同时全局安装@typescript-eslint/eslint-plugin 。从@ typescript-eslint / eslint-plugin官方文档了解有关安装和版本兼容性的更多信息。

如果您更喜欢使用ESLint检查TypeScript代码，则可以在编辑器中安装其他解析器和插件，并在*.ts和*.tsx文件中查看ESLint错误和警告。

如果您已经在使用这些解析器或插件，但不想使用ESLint检查TypeScript代码，请将*.ts或*.tsx添加到.eslintignore文件中。

使用@ typescript-eslint / parser

  1. 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install @ typescript-eslint / parser --save-dev

  2. 在根据.eslintrc配置文件或eslintConfig中的package.json，添加：

    { “parser” ： “@ typescript-eslint / parser” }

使用@ typescript-eslint / eslint-plugin

  1. 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install @ typescript-eslint / parser @ typescript-eslint / eslint-plugin --save-dev

  2. 在根据.eslintrc配置文件或eslintConfig中的package.json，添加：

    { “parser” ： “@ typescript-eslint / parser” ， “plugins” ： [ “@ typescript-eslint” ] }

ESLint 4.0

从babel-eslint官方文档了解有关安装和版本兼容性的更多信息。

从typescript-eslint-parser官方文档中了解更多信息。

如果您全局安装了ESLint，请同时全局安装eslint-plugin-typescript。从eslint-plugin-typescript官方文档中了解更多信息。

如果您更喜欢以前版本的ESLint，则必须安装babel-eslint ， babel-eslint typescript-eslint-parser或eslint-plugin-typescript typescript，因为ESLint 4.0及更早版本不支持作用域包。

使用babel-eslint

  1. 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install eslint babel-eslint --save-dev

  2. 在根据.eslintrc配置文件或eslintConfig中的package.json，添加：

    { “parser” ： “babel-eslint” }

使用typescript-eslint-parser

  1. 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install typescript-eslint-parser --save-dev

  2. 在根据.eslintrc配置文件或eslintConfig中的package.json，添加：

    { “parser” ： “typescript-eslint-parser” }

使用eslint-plugin-typescript

  1. 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install typescript-eslint-parser eslint-plugin-typescript --save-dev

  2. 在根据.eslintrc配置文件或eslintConfig中的package.json，添加：

    { “parser” ： “typescript-eslint-parser” ， “plugins” ： [ “eslint-plugin-typescript” ] }
