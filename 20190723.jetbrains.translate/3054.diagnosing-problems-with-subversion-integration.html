用Subversion集成诊断问题

如果Subversion集成出现任何问题，请随时联系我们的支持 。为便于检测，查找和解决您的问题，请提供有关Subversion与WebStorm集成的详细信息。本主题列出了必要的信息，并说明了如何检索它。

诊断Subversion问题通常需要以下数据：

  • WebStorm版本和内部版本号，

  • 使用的操作系统。

在本页面：

  • 常规VCS设置
  • Subversion设置
  • 本地工作副本格式
  • 使用的分支的父文件夹
  • 启用svnkit日志记录

常规VCS设置

应用于项目的常规版本控制设置在“设置/首选项”对话框的“ 版本控制”页面上指定。按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或WebStorm | macOS的首选项 ，然后单击版本控制 。

Subversion设置

Subversion特定设置在Subversion页面上的Settings / Preferences对话框的Version Control节点下配置。按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或WebStorm | macOS的首选项 ，然后单击版本控制下的Subversion 。

本地工作副本格式

工作副本格式是Subversion格式，符合创建工作副本的格式。要查看工作副本格式，请选择“ 查看”|工具窗口|主菜单上的版本控制 。在打开的“ 版本控制”工具窗口中，切换到“ Subversion工作副本信息”选项卡。

使用的分支的父文件夹

要查看分支的配置，请在“ 版本控制”工具窗口的“ Subversion工作副本信息”选项卡中，单击“ 配置分支”链接。

启用svnkit日志记录

如果保存在WebStorm日志中的数据不足以解决问题，并且问题与通信协议或身份验证有关，请启用svnkit日志记录 ，重现问题并附加idea.log 。

要启用-Djavasvn.log=true日志记录，请根据所使用的操作系统将-Djavasvn.log=true添加到以下文件之一：

  • Windows ： WebStorm.exe.vmoptions

  • Linux ： WebStorm.vmoptions

  • macOS ： http ： //stackoverflow.com/a/13581526/72788

请参阅调整WebStorm以了解WebStorm idea.properties和vmoptions文件位置。
