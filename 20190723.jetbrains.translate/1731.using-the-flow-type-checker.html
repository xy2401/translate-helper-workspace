IntelliJ IDEA 2019.1帮助

流

此功能仅在Ultimate版本中受支持。

在开始之前，请确保您的计算机上有Node.js.

Flow是一个静态类型检查器，它为JavaScript提供了类型注释。IntelliJ IDEA可识别Flow结构，并在所有操作系统上为它们提供语法高亮显示。

安装Flow

从Flow官方网站了解更多信息。

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install --global flow-bin全局安装Flow。

    • npm install --save-dev flow-bin将Flow作为开发依赖项安装。

在IntelliJ IDEA中配置流程

要让IntelliJ IDEA识别Flow结构，提供正确的语法突出显示，正确报告错误并避免误报错误突出显示，将JavaScript语言级别更改为Flow ，将.flowconfig配置文件添加到项目中，并提供要检查的每个文件在// @flow注释顶部。

将语言级别更改为Flow

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript 。JavaScript页面打开。

  2. 从JavaScript语言版本列表中，选择Flow 。

  3. 在Flow包或可执行文件字段中，指定node_modules\flow-bin包或Flow二进制可执行文件的路径。要使用node_modules\.bin\flow确保将Node.js的路径添加到PATH环境变量中。

  4. 在Use Flow server for：区域中，通过选中或清除以下复选框指定编码帮助的基础：

    • 类型检查：选中此复选框后，将根据从Flow服务器接收的数据提供语法和错误突出显示。清除该复选框后，只有基本的内部IntelliJ IDEA突出显示可用。

    • 导航，代码完成和类型提示：选中此复选框时，参考分辨率和代码完成的建议列表包含从与Flow集成检索的建议和IntelliJ IDEA计算的建议。清除该复选框后，仅通过IntelliJ IDEA计算解析引用。

    仅当指定了Flow可执行文件的路径时，复选框才可用。

  5. 保持选中“ 自动保存所有已修改文件”复选框以确保连续应用Flow，因为Flow仅在保存所有其他修改文件后才检查当前文件。

在项目中生成.flowconfig配置文件

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    flow init

在文件中启用Flow

  • 在其顶部添加// @flow注释：只需键入flow ，按Tab键 ，IntelliJ IDEA将其扩展为// @flow 。
