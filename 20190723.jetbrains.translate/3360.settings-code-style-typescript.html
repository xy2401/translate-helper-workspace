标签和缩进

使用制表符

  • 如果选中此复选框，则选项卡字符用于缩进和代码重新格式化。

  • 清除该复选框后，WebStorm使用空格而不是制表符。

智能标签

如果选中“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

缩进由两部分组成。一部分来自嵌套代码块，另一部分是通过对齐来确定的。

  • 如果选中此复选框，则嵌套产生的部分包含制表符和空格（如果需要），而由对齐定义的部分仅包含空格。

  • 如果清除此复选框，则仅使用选项卡。这意味着在重新格式化一组符合指定选项卡大小的空格后，将自动替换为选项卡，这可能会导致细微对齐。

标签大小

在此文本框中，指定适合选项卡的空格数。

缩进

在此文本框中，指定要为每个缩进级别插入的空格数（如果选中“ 使用选项卡字符”复选框，则为选项卡 ）。

继续缩进

在此文本框中，指定要在数组元素之间，表达式，方法声明和方法调用之间插入的空格数（或选中“ 使用选项卡字符”复选框）。

在空行上保持缩进

如果选中此复选框，则WebStorm会在空行上保留缩进，就好像它们包含一些代码一样。如果清除该复选框，WebStorm将删除空行上的制表符和空格。

缩进链式方法

在函数声明中，链中的第二个和更多方法显示在单独的行上。

  • 选中该复选框后，链中的第二个和其他方法将与第一个调用对齐。

  • 清除该复选框后，链中的第二个和更多方法将与调用它们的对象对齐。

缩进组中的所有链接调用

仅当选中“ 缩进链接方法”复选框时，该复选框才可用。

空间

使用此选项卡指定WebStorm自动插入空格的位置。选中相关位置描述旁边的复选框，然后在“ 预览”窗格中检查结果。

包裹和括号

在此选项卡中，自定义WebStorm在重新格式化源代码时将应用于各种代码构造的异常，大括号放置和对齐选项。在“ 预览”窗格中检查结果。

对齐优先于缩进选项。

硬包装
在此字段中，指定元素右侧所需的空格数。如果接受“ 默认”选项，则使用全局设置中的值。

打字输入
在此字段中，指定编辑的文本在指定的Hard wrap at字段中的拟合方式。

  • 默认 - 选择此选项以使用全局设置中的Wrap on键值。

  • 是 - 选择此选项以使用右边距字段中的值。

  • 否 - 如果选择此选项，则行可以超过右边距中指定的值。

可视指南在此字段中，指定多个右边距。您可以保留默认值或输入保证金的空格数。如果要指定多个边距，请输入以逗号分隔的数字。

重新格式化时保留
使用复选框配置WebStorm在重新格式化源代码时将进行的异常。例如，默认情况下，选中“ 换行符”复选框。如果您的代码包含比标准约定更短的行，则可以在重新格式化之前通过禁用换行符复选框来转换它们。

包装选项
包装样式适用于左侧窗格中指定的各种代码构造（例如，方法调用参数或赋值语句）。

不要换行

选择此选项后，不会应用任何特殊的包装样式，将忽略嵌套的对齐和大括号设置。

包裹多久

选择此选项可以使用适当的缩进来包围超出右边距的线条。

永远包裹

选择此选项可将所有元素包装在列表中，以便每行有一个元素并具有适当的缩进。

如果长时间砍下来

选择此选项可以在超出右边距的列表中包装元素，以便每行有一个元素并且具有适当的缩进。

对齐选项

多线时对齐

如果选中此复选框，则代码构造将在每个下一行的同一列开始。否则，代码构造的位置由当前缩进级别确定。

下一行<character(s)>

选中此复选框可在包装线条时将指定的一个或多个字符移动到下一行。

新线上的“其他”

使用此复选框将相应的语句或字符移动到下一行。

<character>之后的新行

选中此复选框可将指定字符后面的代码移动到新行。

特别是如果治疗

如果选中此复选框， else if语句位于同一行。

否则， else if语句被移动到相应缩进级别的下一行。

缩进案例分支

如果选中此复选框，则case语句位于相应的缩进级别。否则， case语句与switch一起放在相同的缩进级别。

对象

从下拉列表中，选择如何对齐对象：

  • 不对齐 - 连续行中的属性将不对齐。

  • 在冒号上 - 连续行中的属性将与冒号对齐。

  • 在值上 - 连续行中的属性将与值对齐。

变量声明

选择以下选项之一以配置相等符号的对齐：

  • 不要对齐 - 等号不对齐。

  • 多行时对齐 - 通过插入额外的空格来对齐多行var语句中的等号。

  • 分组时对齐 - 通过插入其他空格来对齐多个var语句中的相等符号。

ES6导入/导出

 对齐'from'子句：选中此复选框后，WebStorm会自动对齐ECMAScript 6代码中的import和export语句，使您的代码更易于阅读和维护。在“ 预览”窗格中比较代码片段的外观与对齐以及没有它的情况。

启用此选项后，WebStorm将动态对齐新代码。在按Ctrl + Alt + L重新格式化代码后，将对齐现有的import和export语句，有关详细信息，请参阅“ 重新格式化源代码”部分。

大括号放置选项

大括号放置方式

使用此下拉列表在类声明 ， 方法声明 ， 函数声明和其他类型的声明中指定左大括号的位置。可用选项包括：

  • 行尾 - 选择此选项可将开括号放在声明行末尾。

  • 如果换行，则选择下一行 - 选择此选项可将左大括号放在多行声明行之后的行的开头。

  • 下一行 - 选择此选项可将开始括号放在声明行之后的行的开头。

  • 下一行移动 - 选择此选项可在声明行移动到相应的缩进级别后将左大括号放在该行上。

  • 每个移动的下一行 - 选择此选项以在声明行移动到相应的缩进级别后将开始括号放置在行上，并将下一行移动到下一个缩进级别。

强制支撑

从此下拉列表中，为if ， for ， while和do () while语句选择大括号引入方法。可用选项包括：

  • 不要强制 - 选择此选项可禁止自动引入括号。

  • 多行时 - 如果语句占用多行 ，请选择此选项以自动插入大括号。请注意，WebStorm分析整个语句中的行数，但不仅仅是它的条件。

  • 始终 - 选中此复选框后，WebStorm将始终自动插入大括号。

空白行

使用此选项卡可以定义WebStorm在重新格式化后保留并插入代码中的空白行的位置和数量。结果显示在“ 预览”窗格中。

保持最大空行

在此区域中，指定重新格式化后要保留的额外空行数。

最小空行

在此区域中，配置是否在import语句块之后以及类，字段，方法或函数周围有额外的空行。在每个选项旁边的文本框中，指定要保留的最小额外空行数。

这些设置不会影响第一个项目之前和之后的空白行数。

标点

使用此选项卡中的下拉列表可以自动插入终止分号，单引号和双引号以及尾随逗号的指令。

分号终止语句

  • 使用分号终止新代码中的语句

  • 使用分号始终终止语句

  • 不要使用分号来终止新代码中的语句

  • 不要用分号来终止始终声明

行情

  • 在新代码中使用双引号

  • 始终使用双引号

  • 在新代码中使用单引号

  • 始终使用单引号

尾随逗号

使用此下拉列表可以配置是否要在对象，数组中使用尾随逗号 ，以及在方法定义和调用中使用参数。可用选项包括：
  • 保持
  • 去掉
  • 多线时添加

代码生成

在此选项卡上，配置生成代码的代码样式。

使用'public'修饰符

使用此复选框可在生成的代码中插入或省略公共访问修饰符。
例如，在从以下方面生成公共方法期间：
class Test { public test （） ： void { var x = 1 ; } }

  • 如果选中该复选框，则会在生成的代码中自动插入公共访问修饰符：

    class Test { public test （） ： void { this 。 extract （）; 萃取 } 公共 （）{ 风险 X = 1; } }

  • 如果清除该复选框，则在代码生成期间将省略公共访问修饰符：

    class Test { public test （） ： void { this 。 extract （）; } 萃取（）{ 风险 X = 1; } }
请参阅TypeScript语言手册 ，私有/公共修饰符一章。

命名约定

在这方面：

  • 配置或接受将自动添加到生成的字段和属性名称的默认前缀。

  • 选择生成文件名称的样式。例如，当WebStorm建议将类移动到新的自动生成的文件时，将根据所选样式从类名生成该文件的建议名称。了解更多信息保持类的名称和包含文件的符合性

评论代码

在此区域中，为生成的注释配置代码样式。

  • 在JSDoc中包含类型 - 选中此复选框可自动为@return @param ， @return和其他标记添加类型。请注意，类型信息与TypeScript无关，因为从代码本身检索类型。因此，默认情况下清除该复选框。

  • 第一列的行注释 - 选中此复选框以在第一列开始行注释。清除该复选框后，行注释将在代码中对齐。

  • 在注释开始时添加空格 - 选中此复选框后，将在行注释字符和注释行的第一个字符之间插入一个空格。

选中第一列的行注释时， 在注释开始时添加空格复选框不可用。

进口

合并来自同一模块的成员的导入

  • 选中此复选框后，同一模块中的导入符号将在一个导入语句中列出，并使用逗号作为分隔符。成员按其导入顺序列出。要按字母顺序排列它们，请选中Sort imported members复选框并运行Code |优化进口 。

  • 清除此复选框后，对于每个导入的符号，将生成单独的import语句。

使用相对于tsconfig.json路径

选中此复选框后，WebStorm将使用tsconfig.json文件作为根计算导入路径。清除此复选框后，WebStorm将计算相对于项目根目录的导入路径。

例如，如果您的项目结构如下：

选中该复选框后，WebStorm将生成以下import语句：

从 'directory_2 / file_2' 导入 { ClassName }

如果清除该复选框，则会生成以下import语句：

从 '../directory_2/file_2' 导入 { ClassName }

使用目录导入（节点式模块解析）

  • 选中此复选框后， 将生成符合Node.js模块解析策略的 import语句。

  • 清除此复选框后， 将生成符合TypeScript经典模块解析策略的 导入语句。

在模块名称中使用.js文件扩展名

选中此复选框后，WebStorm会自动将js扩展名添加到import语句中的ES6模块名称中。当您在需要完整文件名的浏览器中使用ES6模块时（例如，在Chrome中），这一点非常重要。WebStorm在自动生成导入语句，代码完成或使用Alt + Enter调用导入生成时都应用此设置。

不要完全从中导入

在此字段中，指定WebStorm在自动导入符号期间应跳过的确切路径。相反，WebStorm将寻找导入符号的替代路径。

这对于允许导入子模块而不是整个模块的模块特别有用。例如，要将import {Observable} from 'rxjs/Observable'中的import {Observable} from 'rxjs/Observable'等import {Observable} from 'rxjs/Observable'更喜欢import {Observable} from 'rxjs/Observable'中更import {Observable} from 'rxjs' ，请将rxjs添加到列表中。

要管理要跳过的模块列表：

  1. 点击在田野的右边。

  2. 在打开的“ 更改模块”对话框中，单击“确定” 并在“ 添加模块”对话框中指定模块名称。要从列表中删除模块，请选择该模块并单击 。

排序导入的成员

  • 选中此复选框后，WebStorm将按字母顺序列出合并导入语句中的导入成员。请注意，成员按照导入顺序以逗号分隔列出，并仅在运行Code |时重新排序优化进口 。

  • 清除此复选框后，合并导入语句中的成员始终按照导入顺序以逗号分隔列出。

按模块排序导入

  • 选中此复选框后，运行Code |时，模块名称将按字母顺序重新排序import语句优化进口 。

  • 清除此复选框后， 导入语句将始终按生成顺序显示，并且在运行Code |后不会更改此顺序优化进口 。

安排

在此选项卡中，定义一组规则以根据您的首选项重新排列TypeScript代码。

分组规则

使用此区域设置分组规则。

  • 具有相应getter / setter的组属性字段

匹配规则

使用此区域将元素顺序定义为规则列表，其中每个规则都有一组匹配项，例如修饰符或类型。

  • - 使用此按钮添加规则。空规则区域打开。

  • - 使用此按钮从列表中删除规则。

  • - 使用此按钮编辑现有规则。要查看此按钮，请导航到要编辑的规则，然后单击按钮。在打开的弹出窗口中，修改规则字段。

  • - 使用这些按钮向上或向下移动所选规则。

空规则

使用此区域可以创建新的匹配规则或编辑现有规则。您可以从以下过滤器中进行选择：

  • 类型 - 使用此过滤器为规则选择构造函数，方法，字段或属性。
    请注意，单击类型关键字两次会使条件无效。

  • 修改器 - 使用此过滤器选择规则的修改器类型。
    请注意，单击修饰符关键字两次会使条件无效。

  • 名称 - 使用此字段指定规则中的条目名称。此筛选器仅匹配条目名称，例如字段名称，方法名称，类名称等。筛选器支持正则表达式并使用标准语法 。匹配是针对整个名称执行的。

  • 订单 - 使用此下拉列表选择规则的排序顺序。当多个元素使用相同的匹配规则时，此选项很有用。在这种情况下，选择Keep order将保持与重新排列之前设置的顺序相同，并选择Order by Name将按名称对具有相同匹配规则的元素进行排序。

从“ 订单”列表中选择“按名称排序”时，将显示此图标。该图标表示此规则中的项目按字母顺序排序。
