运行和调试TypeScript

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

您可以使用TypeScript编写应用程序，然后以与使用JavaScript编写的客户端应用程序相同的方式运行和调试它。

在运行或调试之前，您需要将TypeScript代码编译为JavaScript。PyCharm允许您通过与TypeScript编译器集成来动态执行此操作，如验证TypeScript并将其编译为JavaScript中所述 。您还可以使用其他工具，例如webpack 。从TypeScript官方网站了解更多信息。

运行客户端TypeScript应用程序

您不需要任何配置来从PyCharm运行TypeScript。

  1. 将TypeScript代码编译为JavaScript 。

  2. 在编辑器中，打开HTML文件，并引用生成的JavaScript文件。此HTML文件不一定必须是实现应用程序起始页面的HTML文件。

  3. 执行以下操作之一：

    • 选择查看|在主菜单上的浏览器中打开或按Alt + F2 。然后从列表中选择所需的浏览器。

    • 将鼠标指针悬停在代码上以显示浏览器图标栏： 单击指示所需浏览器的图标。

调试TypeScript

在开始之前，按照配置JavaScript调试器中的描述配置内置调试器 。

通常，您可能希望调试在外部开发Web服务器上运行的客户端应用程序，例如由Node.js提供支持。如果您的应用程序在内置PyCharm服务器上运行 ，您也可以像在内置服务器上运行的JavaScript一样调试它。

使用PyCharm，您可以在原始TypeScript代码中设置断点，而不是实际执行的已编译JavaScript。PyCharm使用源映射识别和处理TypeScript中的断点。

在编译期间生成源映射

  • 打开tsconfig.json文件并确保sourceMap属性设置为true 。

调试在外部Web服务器上运行的TypeScript应用程序

  1. 将TypeScript代码编译为JavaScript，请参阅编译TypeScript 。确保生成源映射。

  2. 在TypeScript代码中配置和设置断点 。

  3. 在开发模式下运行该应用程序。通常你需要为此运行npm start 。开发服务器准备就绪后，在浏览器中复制运行应用程序的URL地址 - 您需要在运行/调试配置中指定此URL地址。

  4. 选择运行|从主菜单中编辑配置 ，单击在工具栏上，从列表中选择JavaScript Debug 。在“ 运行/调试配置：打开的JavaScript调试”对话框中，指定运行应用程序的URL地址。可以从浏览器的地址栏复制此URL，如上面的步骤3中所述。

  5. 在工具栏上的“ 选择运行/调试配置”下拉列表中选择新创建的配置，然后单击“确定” 。运行配置中指定的URL地址将在所选浏览器中打开，并显示“ 调试工具窗口 ”。

  6. 在调试工具窗口中，像往常一样继续： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ， 查看实际的HTML DOM ，等等
