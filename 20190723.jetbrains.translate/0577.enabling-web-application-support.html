IntelliJ IDEA 2019.1帮助

启用Web应用程序支持

此功能仅在Ultimate版本中受支持。

本主题讨论打开“ Web应用程序”选项时可用的功能。

  • 先决条件
  • 功能概述
  • 打开Web应用程序选项
  • 管理部署描述符，Web资源目录和Java Web源根
  • 管理应用程序工件

先决条件

对于Web应用程序选项和相关功能：

  • 您应该使用IntelliJ IDEA的ULTIMATE版本。（社区版中没有相应的功能。）

  • 必须启用Java EE：EJB，JPA，Servlets 插件 。（此插件与IDE捆绑在一起并默认启用。）

功能概述

当您打开Web应用程序选项时，IntelliJ IDEA：

  • 创建一个Web资源目录web其中index.jsp旨在作为应用程序的起始页面，还可以选择Web应用程序部署描述符 WEB-INF/web.xml 。

  • 创建Web构面，使您可以管理部署描述符，Web资源目录和Java Web 源根 。

  • 创建爆炸的WAR 工件配置。

  • 使Web工具窗口可用。

如果在创建项目或模块时打开Web应用程序选项并指定应用程序服务器，IntelliJ IDEA还会为该服务器创建运行/调试配置 。

打开Web应用程序选项

您可以打开Web应用程序选项：

  • 创建项目或模块时（ 文件|新建|项目或文件|新建|模块 ）。在New Project或New Module向导的第一页上，选择Java Enterprise ，然后选择Additional Libraries and Frameworks下的Web Application复选框。

  • 对于现有模块。在“ 项目”工具窗口（“ 查看”|“工具Windows”|“项目” ）中，右键单击模块文件夹，然后选择“ 添加框架支持” 。然后在打开的对话框中选择“ Web应用程序”复选框。

管理部署描述符，Web资源目录和Java Web源根

您可以在“ 项目结构”对话框中管理Web应用程序部署描述符 ，Web资源目录和Java Web 源根目录：

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 在最左侧的窗格中，选择“ 模块”或“ 构面” 。

  3. 在右侧窗格中，选择Web或Web（<ModuleName>） 。

  4. 在对话框右侧打开的页面上：

    部署描述符。 形成Web应用程序的部署描述符列表。

    Web资源目录。 指定包含Web应用程序资源的目录，例如网页，图像等。

    来源根源。 选择包含Web应用程序Java类（servlet，过滤器，托管bean等）的源根。

    有关详细信息，请参阅Web构面页面 。

管理应用程序工件

要将应用程序部署到服务器，您需要一个应用程序工件 。对于Java Web应用程序，IntelliJ IDEA提供以下工件格式：

  • Web应用程序：爆炸。 这是一个解压缩的Web应用程序归档（ WAR ），一种可以部署到Web服务器上的目录结构。

  • Web应用程序：存档。 这显然是一个WAR文件。

要管理工件配置，请使用“ 项目结构”对话框（“ 文件”|“项目结构”|“工件” ）。

另请参阅使用工件 。
