配置Pipenv环境

Pipenv是一个工具，它为Python项目创建虚拟环境提供了所有必要的手段。在安装或卸载软件包时，它会通过Pipfile文件自动管理项目包。

Pipenv还生成Pipfile.lock文件，该文件用于生成确定性构建并创建工作环境的快照。当项目要求和软件包版本至关重要时，这可能对安全敏感部署特别有用。有关pipenv更多详细信息，请参阅项目文档docs.pipenv.org 。

要将Pipenv与PyCharm一起使用，您需要实现几个准备步骤。

安装Pipenv

  1. 运行以下命令以确保在系统中安装了pip：

    $ pip --version

    您应该期望收到指示pip版本的系统响应。如果未发现pip，请按照安装说明中的说明进行安装 。或者，您可以从http://python.org下载并安装Python。

  2. 通过运行以下命令安装pipenv ：

    $ pip install --user pipenv

    安装完成后，您将看到以下消息：

  3. 为方便起见，您可以将用户库的二进制目录添加到PATH环境变量中。如果跳过此过程，PyCharm将在添加pipenv环境时提示您指定pipenv可执行文件的路径。

    1. 运行以下命令：

      $ py -m site --user-site

      示例输出可以是：

      C：\用户\ JetBrains公司\应用程序数据\漫游\ Python的\ Python37 \站点包

    2. 将site-packages替换为此路径中的Scripts以接收用于添加到PATH变量的字符串，例如：

      $ setx PATH“％PATH％; C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts”

    1. 运行以下命令以查找用户库的二进制目录：

      $ python -m site --user-base

      样本输出可以是

      /Users/jetbrains/.local

    2. 将bin添加到此路径以接收用于添加到~/.bashrc文件的字符串，例如：

      $ export PATH =“$ PATH：Users / jetbrains / .local / bin”

    3. 运行以下命令以使更改生效：

      $ source~ / .bashrc

    4. 确保您已在bash_profile启用bashrc 。

您可以随时在项目设置中更改pipenv可执行文件的指定路径。在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 工具”| Python集成工具 ，并在Path ot Pipenv可执行文件字段中键入目标路径。

完成准备步骤后，您可以使用pipenv为新项目或现有项目创建虚拟环境。

为新的Python项目设置pipenv

  1. 如创建纯Python项目中所述，启动创建新的Python项目 。

  2. 在“ 新建项目”对话框中，单击要展开Project Interpreter节点，请选择New environment using ，然后从可用虚拟环境列表中选择Pipenv 。

  3. 如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径：

  4. 单击“ 确定”完成任务。

  5. 如果您尚未修改PATH变量，PyCharm会显示错误消息： 找不到Pipenv可执行文件 。发现pipenv安装过程中描述的正确可执行路径，并在Pipenv可执行文件字段中输入目标字符串，例如： C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts \ pipenv.exe （Windows）或/Users/jetbrains/.local/bin/pipenv（macOS ）。

    单击“ 确定”保存更改并完成任务。

您还可以为任何早期创建的项目设置pipenv。

为现有的Python项目配置pipenv

  1. 通过以下任一方式打开“ 添加Python解释器”对话框：

    • 当您在编辑器中时 ，最方便的方法是使用状态栏中的Python解释器小部件。单击小部件并选择Add Interpreter ...

    • 如果您位于“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，请选择“ 项目<项目名称>”项目口译员 。点击图标，然后选择添加 。

  2. 在此对话框的左侧窗格中，单击“ Pipenv环境” 。

  3. 如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径。

    如果您看到找不到Pipenv可执行文件错误消息，请按照pipenv安装过程发现可执行文件路径并将其添加到Pipenv可执行文件字段。

  4. 单击“ 确定”完成任务。

完成所有步骤后，将为您的项目设置新的pipenv环境，并安装Pipfile中列出的软件包。

如果您打开一个添加了Pipfile文件但没有配置任何解释器的项目，PyCharm会让您使用Pipenv环境。

如果选择此选项，PyCharm会自动为您设置pipenv。或者，您可以单击“ 配置Python解释器”以遵循标准工作流程。

同样，当您第一次在PyCharm中打开带有Pipfile文件的项目时，例如，通过从版本控制中检出它，将自动配置Pipenv虚拟环境。

将Pipenv虚拟环境设置为项目解释程序后，将从Pipfile Pipfile中定义的源添加所有可用程序包。通过pipenv而不是通过pip在软件包列表中安装，删除和更新软件包。

对于任何配置的Python解释器，您可以：

  • 安装和卸载解释器路径
  • 安装和卸载解释器包
