在开始之前，请确保您的计算机上有Node.js.

React是一个JavaScript库，用于从封装组件构建复杂的交互式用户界面。从React官方网站了解有关该库的更多信息。

WebStorm与React集成，在配置，编辑，linting，运行，调试和维护应用程序方面提供帮助。

创建一个新的React应用程序

或者，打开内置终端并键入：

  1. create-react-app <application-name>来创建应用程序。

  2. cd <application-name>切换到应用程序文件夹。

  3. npm start启动Node.js服务器。

您可以使用create-react-app包或创建一个空的WebStorm项目并在其中安装React。

使用create-react-app生成React应用程序

WebStorm仅保证使用react-scripts包运行和调试Jest测试。

创建React App是开始构建新的React单页面应用程序的推荐方法。因此，您的开发环境已预先配置为使用webpack，Babel，ESLint和其他工具。了解有关从React官方网站安装React和创建React应用程序的更多信息。

全局安装create-react-app

  • 打开内置终端 （ View | Tool Windows | Terminal ）并在命令提示符下键入npm install -g create-react-app 。

创建应用程序

  1. 选择文件|新的|项目主菜单上，或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择React App 。

  3. 在右侧窗格中，指定项目文件夹，要使用的Node.js解释器以及create-react-app包的路径。

  4. 可选：
    在项目生成期间指定要使用的自定义程序包而不是react-scripts 。这可以是从react-scripts派生的一个包，例如， react-awesome-scripts ， custom-react-scripts ， react-scripts-ts等等。

  5. 单击“ 创建”时 ，WebStorm将生成一个包含所有必需配置文件的React特定项目。

要下载项目依赖项，请执行以下操作之一：

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm安装

  • 从项目根目录中的package.json文件的上下文菜单中选择Run'npm install' 。

使用ESLint

除了提供内置代码检查之外，WebStorm还集成了用于JSX代码的连接器，例如ESLint 。ESLint带来了各种各样的linting规则，这些规则也可以通过插件进行扩展。当您键入时，WebStorm会在编辑器中显示ESLint报告的警告和错误。使用ESLint，您还可以使用JavaScript标准样式 。

要让ESLint正确理解React JSX语法，您需要eslint-plugin-react 。使用此插件时，您会收到警告，例如，当没有为React组件设置显示名称时，或者使用某些危险的JSX属性时：

在WebStorm中开始使用ESLint

  1. 在内置终端 （ View | Tool Windows | Terminal ）中，键入npm install --save-dev eslint和npm install --save-dev eslint-plugin-react 。

  2. 将ESLint配置文件.eslintrc添加到项目中。

  3. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |代码质量工具| ESLint 。在打开的ESLint页面上，选中“ 启用”复选框。WebStorm将自动在项目的node_modules文件夹中找到ESLint，然后默认使用.eslintrc配置。

.eslintrc结构示例（带反应插件的ESLint 1.x）

  • 在ecmaFeatures对象中，添加"jsx" = true 。您还可以在此处指定要使用的其他语言功能，例如ES6类，模块等。

  • 在plugins对象中，添加react 。

  • 在rules对象中，您可以列出要启用的ESLint内置规则 ，以及通过react插件提供的规则 。

{ “parser” ： “babel-eslint” ， “env” ： { “browser” ： true }， “ecmaFeatures” ： { “jsx” ： true }， “plugins” ： [ “react” ]， “rules” ： { “半” ： 2 } }
从ESLint官方网站了解有关ESLint的更多信息并对插件配置react 。

运行和调试React应用程序

仅适用于使用create-react-app 。

仅适用于使用create-react-app 。

建议开始构建新的React单页面应用程序的方法是Create React App 。只有在这种情况下，您的开发环境才会预先配置为使用webpack和Babel。否则，您需要首先配置构建管道 。

要运行React应用程序，请执行以下操作之一：

  • 在npm工具窗口 （ View | Tool Windows | npm ）中，双击start任务。

    感谢Webpack Hot Module Replacement ，当开发服务器运行时，只要更改任何源文件并保存更新，就会自动重新加载应用程序。

调试React应用程序

  1. 通过双击npm工具窗口中的start任务，在开发模式下启动应用程序。

  2. 等到编译应用程序并且Webpack开发服务器准备就绪。在http://localhost:3000/打开浏览器以查看应用程序。

  3. 复制运行应用程序的URL地址（默认情况下为http://localhost:3000/ ），稍后在创建调试配置时需要此URL。

  4. 创建一个新的JavaScript调试配置：选择Run |编辑配置 ，单击 ，并从列表中选择JavaScript Debug 。在“ 运行/调试配置：JavaScript调试”对话框中，将保存的URL（ http://localhost:3000/ ）粘贴到URL字段中。保存配置。

  5. 在代码中设置断点并单击启动调试会话在配置列表旁边。

  6. 当第一个断点被点击时，切换到调试工具窗口并照常进行： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ，浏览调用堆栈和变量，设置监视，评估变量， 查看实际HTML DOM ，等等。

如果在现有WebStorm项目中安装了React，则需要设置构建过程。了解从React官方网站为React应用程序配置构建管道的各种方法。

测试React应用程序

您可以在使用create-react-app创建的 React应用程序中运行和调试Jest测试 。在开始之前，请确保将react-scripts包添加到package.json的dependencies对象中。

您可以通过运行/调试配置运行和调试Jest测试，也可以直接从编辑器或项目工具窗口运行和调试Jest测试，有关详细信息，请参阅Jest 。

创建Jest运行/调试配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行”|“编辑配置 ”）。

  2. 点击在工具栏上，从列表中选择Jest 。将打开“ 运行/调试配置：Jest”对话框。

  3. 指定要使用的节点解释器和应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  4. 在Jest包字段中，指定react-scripts包的路径。

  5. 在Jest选项字段中，键入--env=jsdom 。

运行测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右侧。

  2. 测试服务器自动启动，无需您执行任何步骤。在“ 运行”工具窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

调试测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右侧。

  2. 在打开的调试工具窗口中，像往常一样继续： 逐步执行测试 ， 停止并恢复测试执行， 暂停时检查测试 ，等等。

或者，在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。

一些已知的限制

  • 当您第一次在调试会话期间打开应用程序时，可能会发生在页面加载时执行的代码中的某些断点未被命中。原因是要停止原始源代码中的断点，WebStorm需要从浏览器获取源映射。但是，只有在页面完全加载至少一次后，浏览器才能传递这些源映射。作为解决方法，请自行在浏览器中重新加载页面。

  • 如果您使用的是早于2的Webpack版本的webpack-dev-server，建议您在WebStorm中禁用安全写入功能。否则，应用程序将不会在更改时按时更新。此问题已在Webpack 2中修复。
