配置范围和文件颜色

范围是项目中文件和目录的过滤子集。启用“ 范围”视图时， 会将 搜索和代码检查等操作限制为所选范围。本节介绍如何配置属于这些范围的文件的范围和颜色。

创建新范围

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 外观和行为”|范围 。

  2. 点击 ，从下拉列表中选择“ 本地”或“ 共享 ”，然后指定范围的名称。

    • 本地范围仅供个人使用，并存储在您的工作区中。默认情况下，本地作用域存储在<project_folder> \。idea \ workspace.xml中 。

    • 共享范围是为当前项目定义的，可通过VCS访问团队成员。默认情况下，共享范围存储在<project_folder> \。idea \ scopes文件夹中。

您可以稍后使用页面底部的“ 共享范围”复选框更改共享状态。

配置范围中的项目列表

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 外观和行为”|范围并选择要配置的范围。

  2. 要定义范围的模式，请执行以下操作之一：

    • 要在范围中包含文件和目录，请选择文件和目录，然后按Include或Include Recursively （with subdirectories）按钮。要排除文件或目录，请使用“ 排除”或“ 递归排除” 。WebStorm创建一个表达式并将其显示在Pattern字段中。

    • 使用范围语言语法手动在“ 模式”字段中指定模式。

  3. 单击“ 应用”

将文件颜色与范围相关联

属于不同范围的文件可以在WebStorm的用户界面中以不同颜色突出显示：在导航列表中 ，在编辑器选项卡中，在项目工具窗口中 。这样可以在大型项目中更快，更轻松地导航。

将文件颜色与范围相关联

  1. 导航到文件|设置|外观与行为|文件颜色 。

  2. 单击添加 （ ）。

  3. 在“ 添加颜色标签”对话框中，选择范围并为其选择颜色。

  4. 单击确定 。

  5. 定义要使用范围颜色的区域。要定义区域，请选中“ 在编辑器选项卡中 使用”或“ 在项目视图中使用”复选框。

  6. 选择范围颜色关联的辅助功能级别。要共享颜色关联，请单击“ 共享”图标（ ）。默认情况下，文件颜色关联存储在<project_folder> \。idea \ fileColors.xml中 。

  7. 应用更改。

安排范围的顺序

如果某个文件包含在多个范围内，则范围的顺序变得很重要：WebStorm使用最上面范围的颜色（显示在“ 范围设置”页面中）来突出显示此类文件。当然，您可以更改范围的顺序，从而更改结果突出显示。

安排范围的顺序

  1. 导航到文件|设置|外观与行为|范围 。

  2. 选择要移动的范围。

  3. 要更改顺序，请单击“上移” （ ）或下移图标（ ）。

  4. 单击“ 应用”
