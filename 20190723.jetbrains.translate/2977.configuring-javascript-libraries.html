配置JavaScript库

在WebStorm中， 库是一个文件或一组文件，除了WebStorm从您编辑的项目代码中检索的函数和方法之外，其功能和方法也被添加到WebStorm的内部知识中。在项目范围内，默认情况下其库是写保护的。

WebStorm仅使用库来增强编码帮助（即代码完成，语法突出显示，导航和文档查找）。请注意，库不是管理项目依赖项的方法。

TypeScript社区存根也称为TypeScript定义文件，TypeScript声明文件或DefinitelyTyped存根，或仅d.ts文件。

同样，您可以配置自动创建的node_modules库的范围，请参阅配置node_modules库 。

使用TypeScript社区存根（TypeScript定义文件）

在WebStorm中，可以配置DefinitelyTyped存根并将其用作库，这在以下情况下特别有用：

  • 要改进代码完成，请解析对于WebStorm静态分析而言过于复杂的库或框架的符号，并为这些符号添加类型信息。

  • 从测试框架中解析全局定义的符号。

  2. 单击“ 下载” ，在打开的“ 下载库”对话框中，选择所需的库，然后单击“ 下载并安装” 。

配置Node.js核心库

要获得编译为Node.js二进制文件的fs ， path ， http和其他核心模块的代码完成和参考解析，您需要将Node.js核心模块源配置为JavaScript库。

配置Node.js核心

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ Node.js”和“NPM ”。Node.js和NPM页面打开。

  2. 选中Node.js的编码辅助复选框。

    请注意，Node.js核心库是特定于版本的。因此，如果您在Node.js和NPM页面上更改Node.js的版本，则需要再次选中该复选框。之后，WebStorm为这个新版本创建了一个新库。

配置node_modules库

为了提供项目依赖项的代码完成，WebStorm自动创建一个node_modules库，以便Node.js模块保存在项目中，但不会对它们运行检查，从而提高性能。

在“ 项目”工具窗口中， node_modules也标记为库：但是， node_modules库仅包含项目package.json文件的dependencies对象中列出的模块。WebStorm不将依赖项的依赖项包含到node_modules库中，但实际上将它们从项目中排除。

配置库的范围

您可能会不时注意到WebStorm建议不相关的完成，例如，客户端代码中的Node.js API。发生这种情况是因为默认情况下，WebStorm在整个项目文件夹中使用库来完成。WebStorm允许您通过配置库的范围来调整代码完成。

配置库的范围

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。JavaScript。将打开“ 库”页面，其中显示了所有已有库的列表。

  2. 选择所需的库，清除其旁边的“已启用”复选框，然后单击“ 管理范围” 。JavaScript库。使用范围对话框打开。

  3. 点击并选择要包含在库范围中的文件或文件夹。WebStorm将您带回JavaScript Libraries Usage Scopes对话框，其中Path字段显示所选文件或文件夹。

  4. 对于每个添加的文件或文件夹，从“ 库”列表中选择要配置的库。

示例：配置HTML和Node.js核心库的范围

在WebStorm中处理全栈JavaScript应用程序时，您经常会注意到代码完成在客户端代码中建议了一些Node.js API，在Node.js代码中建议了DOM API。这是因为默认情况下在整个项目中启用带有DOM API的HTML库和带有Node.js API的Node.js核心库。要删除不相关的完成建议，您需要配置这些库的范围。

配置HTML和Node.js核心库的范围

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。JavaScript。将打开“ 库”页面，其中显示为当前项目配置的所有库。

  2. 清除HTML和Node.js核心项旁边的已启用复选框。

  3. 单击管理范围 。将打开“ JavaScript库使用范围”对话框。

  4. 要配置HTML库的范围，请单击 ，使用客户端代码选择文件夹，然后从“ 库”下拉列表中选择“ HTML ”。

  5. 要配置Node.js核心库的范围，请单击再次，选择具有服务器端代码的文件夹，然后从Libraries下拉列表中选择Node.js Core 。

    现在，WebStorm解析了HTML和Node.js核心库中的项目，并仅在这些所选项目文件夹的文件中建议它们完成。

配置通过CDN链接添加的库

当通过CDN链接引用库.js文件时，它可用于运行时，但对于WebStorm是不可见的。要将此类库.js文件中的对象添加到完成列表，请下载该文件并将其配置为外部库。

下载库

  • 将光标定位到库的CDN链接，按Alt + Enter ，然后从列表中选择下载库 ：

    该库将下载到WebStorm缓存（但不会下载到您的项目中），并显示带有信息消息的弹出窗口：在JavaScript Libraries页面上，下载的库将添加到列表中，并在当前项目的范围内启用它：在“ 项目”工具窗口中，库显示在“ 外部库”节点下：

更改库的可见性

  • 默认情况下，WebStorm将下载的库标记为全局 ，这意味着您可以在任何其他WebStorm项目中启用和重用它。要更改此默认设置，请在列表中选择下载的库，单击“ 编辑” ，然后在打开的“ 编辑库”对话框中选择“ 项目 ”。

配置自定义第三方JavaScript库

假设您的项目或计算机上的其他地方有一个JavaScript框架文件，并且您希望WebStorm将其视为库，而不仅仅是您编辑的项目代码。

配置自定义库

  1. 下载所需的框架文件。

  2. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。关于JavaScript。打开的库页面，单击“ 添加” 。“ 新建库”对话框将打开。

  3. 指定外部JavaScript库的名称，单击 ，然后从列表中选择附加文件或附加目录 。在打开的对话框中，选择包含下载框架的文件或文件夹。

可选

  • 默认情况下，库是Global ，这意味着您可以将其附加到任何其他项目。要禁止重新使用库，请在“ 新建库”对话框中选择“ 项目 ”。

  • 默认情况下，库在整个当前项目的范围内启用。您可以按配置库的范围中所述更改此默认设置。

  • 在“ 文档URL”区域中，指定库或框架的官方文档的路径。当您在此库中的符号上按Shift + F1时，WebStorm将打开此URL。

查看与文件关联的库

  • 在编辑器中打开文件或在项目视图中选择它，然后单击在状态栏上。WebStorm打开一个弹出窗口，列出与当前文件关联的库。要更改列表，请单击“ 范围内的库”链接，然后在打开的“ 管理范围”对话框中编辑范围设置。

  • 或者，在编辑器中打开文件，然后在上下文菜单中选择“ 使用JavaScript库 ”。WebStorm打开一个包含可用配置库的列表。与当前文件关联的库标有刻度。

    • 要从库的范围中删除当前文件，请清除此库旁边的复选框。

    • 要将库与当前文件相关联，请选中此库旁边的复选框。

删除库

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。JavaScript。将打开“ 库”页面，其中显示了所有已有库的列表。

  2. 选择所需的库，然后单击“ 删除” 。
