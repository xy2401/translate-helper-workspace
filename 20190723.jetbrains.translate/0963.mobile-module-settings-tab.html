Java Wireless Toolkit（WTK）的设置

项目

描述

MIDlet的名称

在此文本框中，指定MIDlet套件名称（对应于JAD MIDlet-Name属性）。此选项对于识别设备上的MIDlet套件是必需的。

的MIDlet-JAR-URL

在此文本框中，指定MIDlet JAR位置。此选项对于识别设备上的MIDlet套件也是必需的。之后将从MIDlet-JAR-URL位置安装JAR文件。您可以手动指定位置，或单击省略号按钮并在打开的对话框中选择所需位置。

MIDlet的供应商

在此文本框中，指定MIDlet供应商名称，即MIDlet套件提供程序。

的MIDlet-版本

在此文本框中，指定您的MIDlet版本号。

可选设置

单击此按钮可打开“ 可选MIDP设置”对话框，您可以在其中设置通常不需要的其他选项。

定义的MIDlet

在此区域中，定义MIDlet以形成MIDlet套件。已定义的MIDlet包含以下信息：

  • 将MIDlet呈现给用户的MIDlet名称。

  • 用于指示MIDlet的图标（可选）。

  • 实现MIDlet的类的名称，应用程序管理器将调用它来加载MIDlet。

使用“ 添加” ，“ 编辑”和“ 删除 ”按钮创建，更新和删除MIDlet定义。

使用“上移”和“ 下移”按钮更改MIDlet的顺序。此顺序确定调用MIDlet的顺序。

保留用户定义的JAD文件

选中此复选框以使用上面指定的设置创建JAD文件。

DoJa的设置

项目

描述

AppName的

在此文本框中，键入应用程序名称（最多50个字节）。

PACKAGEURL

在此文本框中，指定访问应用程序的URL地址。确保URL是ASCII格式的字符串。此外，无法直接指定IP地址。

AppClass

在此文本框中，指定应用程序主类名称（最多255个字节）。这应该是com.nttdocomo.ui.的子类名com.nttdocomo.ui.IApplication 。

可选设置

单击此按钮可打开“ 可选MIDP设置”对话框，您可以在其中设置通常不需要的其他选项。

保留用户定义的JAM文件

选中此复选框以使用上面指定的设置创建JAM文件。
