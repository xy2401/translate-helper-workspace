从XML模式生成Java代码

介绍

本主题描述如何获取XML模式的Java表示，其中涉及将XML模式的元素映射到Java类的成员。使用IntelliJ IDEA，可以使用以下数据绑定器之一完成此转换：

  • JAXB生成类并在Java包中对它们进行分组。包由Java类名和ObjectFactory类组成。后者是一个用于返回绑定Java类实例的工厂。

  • XmlBeans将XML Schema转换为Java类，对其进行编译，并放置在指定的输出.jar文件中。

使用JAXB

使用JAXB从XML Schema生成Java类

  1. 在活动编辑器选项卡中，打开所需的架构（ .xsd ）文件或包含所需架构的XML文档。然后选择工具| JAXB |从XML模式生成Java代码使用主菜单上的JAXB 。

  2. 在使用JAXB从Xml架构生成Java对话框中打开配置生成过程：

    • 在Schema / DTD / WSDL Path下拉列表中，指定要用作代码生成基础的文件。默认情况下，该字段显示当前文件的完整路径。接受此建议或单击“ 浏览” 并在打开的JAXB生成的Select XML Schema File中选择所需的文件。

    • 从“ 输出路径”下拉列表中，选择要将生成的Java类放入的模块源目录。

    • 在“ 程序包前缀”下拉列表中，指定要包括生成的存根的程序包。

    • 使用复选框，配置其他选项，例如生成注释，设置只读状态，下载和安装其他库。

使用XmlBeans

使用XmlBeans从XML Schema生成和编译Java类

  1. 在活动编辑器选项卡中，打开所需的架构（ .xsd ）文件或包含所需架构的XML文档。然后选择工具| XmlBeans |从XML模式生成Java代码使用主菜单上的XmlBeans 。

  2. 在“ 使用XmlBeans生成Java模式的XML模式”对话框中，打开配置生成过程：

    • 在“ 架构路径”下拉列表中，指定要用作代码生成基础的文件。默认情况下，该字段显示当前文件的完整路径。接受此建议或单击“ 浏览” 并在打开的“ 选择用于生成的XML架构/ WSDL文件”对话框中选择所需的文件。

    • 在“ 输出路径”下拉列表中，指定.jar文件的名称以放置生成和编译的Java代码。默认情况下，IntelliJ IDEA建议创建一个新的文件types.jar 。要覆盖现有文件，请单击“ 浏览” 并在打开的“ 选择XML架构/ Wsdl文件以生成”对话框中选择所需的文件。

    • 要自动下载和安装缺少的库，请选中“ 添加必要的库以便生成代码编译和工作”复选框。
