使用Docker配置远程解释器

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

介绍

PyCharm与Docker的集成允许您在Docker容器中部署的各种配置的开发环境中运行应用程序。

先决条件

确保满足以下先决条件：

  • 已安装Docker，如Docker Docs页面所述。您可以在各种平台上安装Docker，但在这里我们将使用Windows安装。

    请注意，您可能希望在不同平台上重复本教程;然后使用Docker安装macOS和Linux （Ubuntu，其他与发行版相关的指令也可用）。

  • 在开始使用Docker之前，请确保已启用Docker Integration和Python Docker插件。这些插件与PyCharm捆绑在一起并默认激活。如果未激活插件 ，请按照管理插件中的说明在“设置/首选项”对话框的“ 插件”页面上启用它们。

  • 此外，对于Windows，右键单击Docker whale图标，从上下文菜单中选择Settings，然后在General页面中选择Expose daemon ...复选框：

将Docker配置为远程解释器

现在我们已经准备好了我们的示例，让我们定义一个基于Docker的远程解释器。

通过以下任一方式打开“ 添加Python解释器”对话框：

  • 当您在编辑器中时 ，最方便的方法是使用状态栏中的Python解释器小部件。单击小部件并选择Add Interpreter ...

  • 如果您位于“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，请选择“ 项目<项目名称>”项目口译员 。点击图标，然后选择添加 。

在打开的对话框中，选择Docker选项，从下拉列表中选择Docker服务器（如果缺少服务器，单击New ... ），并指定映像名称。

当应用程序以任何方式执行时，PyCharm创建一个新容器非常重要。无论是运行，调试，运行覆盖，测试 - 每次执行都需要一个新的容器！

Docker工具窗口

但是没有终端可以看到所有容器吗？PyCharm说 - 是的。您可以使用Docker Tool Window作为Docker命令行客户端的UI。

如果您已将Docker配置为远程解释器，您将在主PyCharm窗口的底部看到Docker工具窗口按钮：

单击此按钮，看到Docker工具窗口打开：

摘要

让我们总结一下在PyCharm的帮助下所做的工作：

  • 我们创建了一个项目并添加了一个Python脚本。

  • 我们配置了远程解释器。

  • 我们在Docker容器中运行并调试了我们的脚本。

  • 最后，我们启动了Docker工具窗口，并在终端中看到了所有细节。
