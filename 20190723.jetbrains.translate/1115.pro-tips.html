IntelliJ IDEA专业技巧

本指南面向已熟悉其基本功能并希望了解更多信息的IntelliJ IDEA用户。如果您对IntelliJ IDEA比较陌生，我们建议您在深入研究之前阅读Discover IntelliJ IDEA指南 。

编码协助

输入信息

如果您想了解有关插入符号的更多信息，例如它来自哪里或类型是什么， 快速文档就是您的朋友。按Ctrl + Q调用它，您将看到包含这些详细信息的弹出窗口。如果您不需要完整信息，请改为使用“类型信息”操作：它仅显示所选表达式的类型，但不会占用大量屏幕空间。

代码完成区分大小写

默认情况下，IntelliJ IDEA代码完成区分大小写仅影响您键入的第一个字母。可以更改此策略在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）， 编辑器|一般|代码完成 ，因此您可以根据更适合您的偏好，使IDE对所有字母敏感或使其对案例不敏感。

热门提示 ：在这里您还可以关闭Autopopup代码完成选项。如果您希望代码完成弹出窗口仅在您明确调用它时才显示，这是有意义的。

禁用突出显示插入符号元素的用法

在更好地学习IntelliJ IDEA之后，谈论您可能想要更改的默认值，我们不能错过编辑器 中插入符号选项中元素的突出用法。 “设置/首选项”对话框的常规页面。如果您知道Ctrl + Shift + F7快捷键并且不希望编辑器中的突出显示在每次移动插入符时出现和消失，则不需要此选项。

CamelHumps

默认情况下，当您在编辑器中选择任何内容时，IntelliJ IDEA对单词的大小写不敏感。如果您更喜欢根据CamelCase选择单词，例如，不是选择整个单词，而是选择它的一部分，您可以在编辑器中启用它。 一般| “设置/首选项”对话框的智能键 。

嬉皮完成

IntelliJ IDEA通过Ctrl + Space提供基本完成 ，通过Ctrl + Shift + Space提供智能完成 ，通过Ctrl + Shift + Enter提供 语句完成 。所有这些功能都基于对代码结构的实际理解。但是，有时您可能需要一个更简单但更灵活的逻辑，它可以建议当前文件中使用的单词，甚至是项目的上下文。此功能称为嬉皮完成 ，可通过Alt + /获得 。

重构

撤消重构

使用IntelliJ IDEA，您无需担心重构代码时的后果，因为您可以通过方便的Ctrl + Z快捷方式调用撤消来撤消任何操作。

提取字符串片段

IntelliJ IDEA不仅可以重构可执行代码，还可以重构字符串文字。选择字符串的任何片段，调用Extract variable / constant / field / parameter将其作为常量提取并在整个代码中替换它的用法。

输入迁移

重构时，通常会重命名符号，或者在代码中提取和移动语句。然而，重构不仅仅是那个。例如， 类型迁移 （通过Ctrl + Shift + F6可用）允许您更改变量，字段，参数或方法的返回值（ int → String ， int → Long等）的类型，更新相关代码并解决可能的冲突。

反转布尔值

如果IntelliJ IDEA可以自动化类型迁移，为什么不对语义做同样的事情呢？要反转布尔符号的所有用法，只需使用反转布尔重构 。

代码分析

依赖结构矩阵

IntelliJ IDEA允许您分析代码中的组件彼此之间的紧密程度，并且您需要密切关注它，因为当依赖关系太多时，它可能会导致各种问题 。依赖结构矩阵操作 （可通过Analyze菜单获得）将帮助您可视化和探索模块，包和类之间的依赖关系。

尽管外观复杂，但它是一种非常易于使用的工具。只需选择一个类或包，看看它的使用位置和使用方法。

结构搜索和替换

结构搜索和替换（SSR ）非常强大（在您学会正确使用它之后），并且可用于静态代码分析和重构自动化。简而言之，它允许您在代码中搜索特定模式，并使用参数化模板替换它们。为此，它配备了自己的语言来定义代码模式， 本文将对此进行更详细的描述。

编辑

与剪贴板比较

IntelliJ IDEA具有内置的Diff查看器，用于代码，jar文件，修订甚至图像。要调用它，请选择任意一对文件，然后按Ctrl + D.

如果您选择了单个文件，IDE将提示您选择要比较的文件。要快速比较活动编辑器和剪贴板，请选择“ 查看”|与剪贴板比较 。

粘贴历史记录

说到剪贴板，IntelliJ IDEA会记录您放置的所有内容。只要您想粘贴其中一个先前复制的项目，请按Ctrl + Shift + V.

多项选择

多选是一种相对较新的，功能非常强大的编辑器功能，可让您一次快速选择和编辑多个（相邻或不相邻）代码。

简而言之，这就是发生的事情。您可以按Alt + J开始 （然后IntelliJ IDEA在插入符号处选择一个符号），或者您可以像往常一样选择一些符号。

然后，按Alt + J ，IntelliJ IDEA将向前搜索当前文件，直到找到匹配的文本，然后将其添加到选择中。您可以再次按Alt + J继续前进，或按Shift + Alt + J返回，但请注意，当搜索到达文件末尾时，它将从文件的开头重新开始。

选择完成后，您可以开始编辑所有片段，就像它们是一个片段一样。

热门提示 ：克隆插入符号的另一种方法是按两次Ctrl （ Alt for macOS），然后用箭头或简单地用鼠标向上或向下移动插入符号。

版本控制

修改变更

在Commit Changes对话框中， IntelliJ IDEA提供执行各种操作的功能。其中之一是修改提交 ，当您想要更改上次提交并将当前更改加入其中时，这非常有用。

货架和补丁

Shelves是一个类似于Git Stash的IDE功能，但它适用于所有VCS：当您需要暂停当前工作并从存储库中取出某些内容以便尽快修复它时，它会有所帮助，然后继续处理您正在处理的任何内容。此功能处理本地更改的文件而不提交它们，因此不会丢失更改或仓促进行合并提交。

有关更多详细信息，请参阅Git-Stash页面和Stashing and Unstashing部分。

修补程序允许您将一组更改保存到可以通过电子邮件（或任何其他古老介质）传输的文本文件，然后应用于代码。当您的飞机失事降落在荒岛上之后真的需要投入某些事情时，这是非常有用的，或者您在某种情况下让自己处于没有可靠宽带连接的情况下。

有关更多详细信息，请参阅使用修补程序一节。

调试

动作或方法断点

有时您可能想要在特定代码行评估某些内容而不实际停止。您可以使用方法断点来完成此操作。要创建一个，只需单击持有Shift的装订线。

这样，您可以在不更改代码的情况下将任何表达式打印到输出。在调试库或远程应用程序时，这尤其有用。

现场断点或现场观察点

除了上面提到的操作断点之外，您还可以使用Field观察点 。当访问与其关联的字段时，此断点将停止执行。要创建现场观察点，只需单击持有Alt的装订线（对于macOS，为Ctrl + Cmd ）。

自定义数据渲染器

评估表达式 ， 变量 ， 监视器和内联调试器都使用标准方法来呈现变量值，主要基于类的toString实现。并非所有人都知道您可以为任何类定义自己的自定义渲染器。为此，请从“ 调试”工具窗口的上下文菜单中选择“ 自定义数据视图” 。

当您使用的库中的某些类没有提供有意义的toString实现时，这尤其有用 - 因此您可以在库外部自己定义它。

丢帧

如果您想在调试时“回到过去”，可以通过Drop Frame操作完成。如果你错误地走得太远，这将是一个很大的帮助。这不会恢复应用程序的全局状态，但至少会通过堆栈框架返回。

强制回归

如果您想要跳转到未来，并强制从当前方法返回而不执行任何更多指令，请使用强制返回操作（要调用它，按Ctrl + Shift + A并键入操作名称） ）。如果方法返回一个值，则必须指定它。

DCEVM

有时，当您快速更改代码时，您希望立即看到它们在工作应用程序中的行为方式。不幸的是，Java HotSwap VM有很多限制：比如说，你不能在类中添加新方法或字段并执行热交换;在热交换期间，您唯一可以实际更改的是方法体。

有关详细信息，请参阅重新加载已修改的类和HotSwap部分。

幸运的是，有一种方法可以使用新的开源项目Dynamic Code Evolution VM来修改这种情况，这是对Java HotSwap VM的修改，可以无限制地支持在运行时重新加载类。

使用专用插件可以轻松地在IntelliJ IDEA中使用它。启用插件后，IDE将为您提供下载适用于您的环境的DCEVM JRE。然后，您必须在备用JRE列表中选择它。
