单击“ 添加”按钮并在“ 选择要添加的工具”对话框中选择“ Composer”时 ，将打开该对话框。
在此对话框中，通过指定启动WordPress的方式，在命令行模式下启用WordPress内容管理系统 。

描述

通过PHAR安装

选择此选项可通过PHP脚本启动WordPress ，或让IntelliJ IDEA检测并启动wp-cli.phar存档中的启动程序 。

  • PHP解释器：从执行区域的列表中选择一个配置的PHP解释器。

  • phar的路径：在此文本框中，指定wp-cli.phar存档的位置。

可执行文件（通过Composer等安装）

选择此选项可通过可执行文件启动WordPress ，该文件在使用包管理工具（例如Composer）安装WordPress时可用。

wp.bat的路径

在此字段中，指定wp.bat或wp可执行文件的位置。如果您使用的是Composer，则默认位置为\ vendor \ wp \ cli \ bin \ wp或\ vendor \ wp \ cli \ bin \ wp.bat 。手动键入路径或单击“ 浏览” 并在打开的对话框中选择所需的位置。

该工具仅适用于本地PHP解释器 。
