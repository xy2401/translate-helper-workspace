编辑与网络层对话框相关的Subversion选项

档案|设置|版本控制| Widnwos和Unix的Subversion
WebStorm |偏好|版本控制| Subversion for macOS

单击“ 设置/首选项”对话框的“ Subversion”页面上的“ 编辑网络选项”按钮时，将打开该对话框。在此对话框中，指定存储在服务器Subversion运行时配置文件中的Subversion网络设置。

该对话框包含两个选项卡：

  • 系统文件 - 此选项卡显示系统管理员指定的默认网络配置设置。

  • 用户文件 - 在此选项卡中，自定义默认网络配置设置。

该对话框由两个窗格组成：

  • 在左侧窗格中，添加，编辑和删除配置配置文件。网络配置设置被安排为两种类型的配置文件：

    • 组 - 来自此类配置文件的设置适用于由glob模式定义的特定组。

    • 全局 - 此配置文件中的设置将应用于与任何glob模式不匹配的所有服务器。

  • 在右侧窗格中，指定所选配置概要文件的设置。

HTTP代理设置

项目

描述

网址格式

在此文本框中，键入用于定义要通过代理访问的存储库的URL地址的模式。使用逗号分隔模式。

例外

在此文本框中，键入定义要直接访问的存储库的URL地址的模式，而不使用代理。使用逗号分隔模式。

服务器

在此文本框中，指定要使用的代理服务器的名称或IP地址。

港口

在此文本框中，指定代理服务器侦听的端口号。

连接超时

在此文本框中，指定Subversion客户端取消尝试与服务器建立连接后的时间段（以秒为单位）。

用户

在此文本框中，键入要在指定代理服务器上进行身份验证的用户名或登录名。

密码

在此文本框中，键入与指定的登录名或用户名匹配的密码。

在此文本框中，指定包含Subversion客户端在访问存储库时接受的证书颁发机构（CA）文件的证书的文件的路径。

SSL客户端证书文件

在此文本框中，指定SSL客户端证书文件的位置。手动键入文件的路径，或单击“ 浏览” 并在打开的对话框中选择位置。

SSL客户端证书密码

在此文本框中，键入要使用的SSL客户端证书密码。

信任默认CA.

选中此复选框可使Subversion集成信任OpenSSL附带的默认证书颁发机构文件集。
