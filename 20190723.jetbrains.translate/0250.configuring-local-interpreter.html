配置本地PHP解释器

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

本地PHP解释器是安装在计算机上的PHP引擎，与远程PHP解释器相对，可以安装在远程主机上或在Vagrant实例中设置的虚拟环境中。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| PHP 。

  2. 在打开的PHP页面上，单击 CLI解释器列表旁边。

  3. 在打开的CLI解释器对话框中，单击在左侧窗格中，然后从弹出菜单中选择“ 本地 ”。

    如果您已在IntelliJ IDEA中配置了本地解释器，则它也会从菜单中显示，并且菜单项将更改为“ 其他本地” 。

  4. 在对话框的右侧窗格中，指定PHP解释器的设置。

    1. 在“ 名称”字段中，键入标识符以将解释器与其他解释器区分开来，例如， php_installation_<version> 。

    2. 在PHP可执行文件字段中指定PHP引擎安装目录。手动键入路径或单击并在打开的对话框中选择相关文件夹。

      IntelliJ IDEA显示在指定文件夹中检测到的PHP引擎版本以及php.ini文件中与此PHP引擎关联的调试器。

  5. （可选）您可以在CLI解释器对话框的“ 附加”区域中自定义Java安装的配置设置。

    • 在Debugger扩展字段中，指定Xdebug的路径。这使得IntelliJ IDEA可以在必要时激活Xdebug，如果您已在php.ini文件中禁用它，请参阅在按需模式下配置Xdebug以进行使用 。

    • 在“ 配置选项”字段中，编写要通过-d命令行选项传递的配置指令字符串，从而将新条目添加到php.ini文件中。此字段中指定的指令覆盖IntelliJ IDEA生成的默认指令，例如-dxdebug.remote_enable=1 ， -dxdebug.remote_host=127.0.0.1 ， -dxdebug.remote_port=9001 ， -dxdebug.remote_mode=req 。

      例如，如果指定-dxdebug.remote_mode=jit指令，它将覆盖默认的-dxdebug.remote_mode=req指令，从而将Xdebug切换到实时（JIT）模式，请参阅Just-In-中的调试详细时间模式 。

      为此，请单击在“ 配置选项”字段旁边，然后在“打开的配置选项”对话框中创建条目列表。

      • 要添加新条目，请单击 。在添加到列表中的新行中，分别在“ 名称”和“ 值”字段中指定新条目的名称及其值 。

        您可以根据需要添加任意数量的条目，请记住它们将转换为命令行，其长度限制为256个字符。

      • 要删除条目，请在列表中选择该条目，然后单击 。

      • 要更改条目的顺序，请单击要么 。

      单击“ 确定”后 ，将返回“ CLI解释器”对话框，其中条目将转换为命令行。

  6. 点击让IntelliJ IDEA显示一个单独的信息窗口，其中包含安装详细信息以及已加载扩展和已配置选项的列表。请注意，未列出CLI解释器对话框的“ 配置选项”字段中指定的其他选项 。
