该页面仅在启用PHP插件时可用。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

Code Sniffer

在本节中，选择要使用的Code Sniffer脚本。

项目

描述

组态

选择Code Sniffer脚本：

  • 要使用与特定远程PHP解释器关联的脚本，请选择此解释器的名称。

  • 要使用本地脚本，请选择“ 本地” 。在这种情况下，无论在项目中使用哪个PHP解释器（本地或远程），都将执行本地Code Sniffer。请注意，Code Sniffer只能有一个本地配置，因为IntelliJ IDEA运行一个脚本（ 适用于Windows的phpcs.bat或适用于Linux和macOS的phpcs ），其中包含PHP引擎的路径。

  • 要使用与默认项目解释程序关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释程序” 。

忽略的文件

单击以打开“ 代码嗅探器忽略的文件”对话框，其中列出了代码嗅探器跳过的文件。IntelliJ IDEA建议在等待来自Code Sniffer的响应超过Tool process timeout字段中指定的限制时，在检查期间向列表添加新文件。这样做是为了防止减慢处理速度。对于每个文件，IntelliJ IDEA都会显示其名称和位置。

  • 要添加文件，请单击并在打开的对话框中找到所需的文件。

  • 要从列表中删除文件并让Code Sniffer再次处理它，请选择该文件并单击“ 删除文件”按钮 。

  • 要从列表中删除所有文件，请单击 。

Code Sniffer对话框

单击时将打开该对话框在“ 代码嗅探器”页面上的“ 配置”列表旁边。

使用此对话框配置与远程PHP解释器关联的本地Code Sniffer脚本或脚本，以及配置Code Sniffer的行为。有关详细信息，请参阅PHP Code Sniffer 。

项目

描述

PHP Code Sniffer（phpcs）路径

在此字段中，指定Code Sniffer实用程序phpcs或phpcs.bat 。

如果脚本与PHP解释器关联，IntelliJ IDEA会检测到它的路径并自动填充该字段，但您可以根据需要进行编辑。手动键入路径或单击并在打开的对话框中选择路径。

要检查phpcs.bat或phpcs的指定路径是否确保IntelliJ IDEA和Code Sniffer之间的交互，也就是说，该工具可以从IntelliJ IDEA启动，IntelliJ IDEA将从中接收问题报告，单击Validate按钮。此验证等同于运行phpcs --version命令。如果验证成功通过，IntelliJ IDEA将显示检测到的Code Sniffer版本的信息。

代码质量工具脚本（phpcs.bat用于Windows或Linux和MacOS phpcs）包含对全系统的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统path 。

翻译员

该字段显示所选的PHP解释器以使用Code Sniffer。

每个文件的最大消息数

在此字段中，设置要为文件报告的消息总数的上限。超出此限制的所有邮件都将被拒绝。IntelliJ IDEA将在代码中显示以下警告： Too many PHP Code Sniffer messages并建议将文件添加到Ignored files列表中。

工具进程超时

在此字段中，指定IntelliJ IDEA等待PHP Code Sniffer结果的时间。如果超时超时，则会终止进程以防止CPU和内存使用过多。这使您可以根据计算机的配置和使用的规则集来微调PHP Code Sniffer流程行为。

PHP代码美化和修复程序设置

在此区域中，在phpcbf路径字段中提供PHP Code Beautifier和Fixer工具的路径 。手动键入路径或单击并在打开的对话框中选择路径。
