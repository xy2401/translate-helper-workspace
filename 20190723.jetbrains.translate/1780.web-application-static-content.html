配置静态内容资源

此功能仅在Ultimate版本中受支持。

配置静态Web资源包括：

  1. 指定服务器上静态Web内容的位置 。

    部署之后，实现用户界面的所有静态内容资源都应位于Web应用程序根目录下 。默认情况下，IntelliJ IDEA将目标Web应用程序根目录映射到启用Web开发支持后创建的web文件夹。默认应用程序主页映射到index.jsp存根文件，该文件也是自动创建的。

    您可以通过两种方式在模块中排列静态内容资源：

    • 将所需的资源和目录存储在<project root>\web文件夹下。部署之后，这些资源的整个文件夹层次结构将被复制到应用程序根目录下的服务器。

    • 将所需资源存储在本地适合的任何位置，并将它们映射到服务器上的文件夹 。

  2. 在要部署的工件中包含静态Web内容 。

配置静态Web内容资源

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 单击左侧窗格中的“ 模块 ”，然后展开所需的模块。

  3. 单击Web构面。对话框的右侧窗格显示所有可用设置。

  4. 在“ Web资源目录”区域中，管理本地文件夹与所需资源之间的映射列表以及服务器上用于部署资源的目标目录。<project root>\web文件夹和应用程序rood目录之间的默认映射已在列表中：

    • 要配置映射，请单击“ 新建”按钮。在打开的“ Web资源目录路径”对话框中，指定所需的本地文件夹以及相对于应用程序根文件夹的目标位置。默认情况下，IntelliJ IDEA在部署后将<project root>\web文件夹映射到应用程序的根文件夹。例如，如果键入正斜杠/ ，则Web资源目录中的文件将复制到部署根目录中。

    • 要编辑映射，请在列表中选择它，单击“ 编辑”按钮，然后在打开的“ Web资源目录路径”对话框中更新映射。

    • 要放弃映射，请在列表中选择，然后单击“ 删除”按钮。
