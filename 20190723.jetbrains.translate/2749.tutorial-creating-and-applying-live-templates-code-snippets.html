创建和应用实时模板（代码片段）

本教程的内容是什么？

PyCharm附带了各种各样的实时模板......Python有几个预定义的实时模板。本教程旨在指导您为Python类声明创建实时模板，并使用此实时模板。

Python编程超出了本教程的范围。有关详细信息，请参阅Python文档。

此处不讨论实时模板的基础知识和用法。您可以在实时模板部分中找到有关实时模板的类型，缩写，变量和存储的所有必要信息。

先决条件

确保您使用的是PyCharm 2.7或更高版本。如果您还没有PyCharm，请从此页面下载。要安装PyCharm，请按照说明操作，具体取决于您的平台。

本教程是使用PyCharm Professional 2016.1版创建的。

创建自己的实时模板

1。创建存根实时模板

  • 打开“设置/首选项”对话框（ 在主工具栏上，或按Ctrl + Alt + S ），展开“ 编辑器”节点，然后单击“ 实时模板” ：

  • 点击 。首先，选择Template Group ...并指定其名称（在我们的例子中，它是user ）。创建的组获得焦点。

  • 点击再次。这一次，选择实时模板 。新模板显示在包含焦点的组中，即user 。

接下来发生什么？

  • 首先，在组user ，有一个存根实时模板，现在称为<abbreviation> 。

  • 其次，输入模板缩写，描述，正文和上下文的字段显示：

2。定义模板缩写和上下文

那么，让我们输入模板缩写 。在我们的示例中，我们将在字段缩写中键入单词class ，然后输入description（这是可选的，但仍然......）。

接下来，让我们选择应用新模板的上下文。到目前为止，您看到上下文未定义 - 因此单击链接Define ，然后选择上下文（在我们的示例中，这是Python）。将来，当已定义上下文时，链接将更改为“ 更改” 。

扩展键将是Tab ，默认情况下指定。

3。定义模板文本

在“ 模板”文本字段中键入以下代码：

class $ class $ （ $ object $ ）： “”“$ cls_doc $”“” def __init __ （ self ， $ args $ ）： “”“$ class $的构造函数”“” $ END $

$ sign中包含的模板文本部分是模板变量 。您可以从整个模板文本中轻松告诉他们，因为它们突出了颜色：

这些模板变量尚无效，所以让我们先定义它们......所有，除了一个（稍等一下，了解原因）。

4。编辑模板变量

单击按钮编辑变量 ：

在对话框编辑模板变量中 ，您会看到所有列表（但只有一个！）变量：

对于变量$object$ ，让我们定义它的默认值（ object ），然后单击OK ：

关于$ END $的附注

您已经注意到，建议不要编辑模板变量$END$ 。这是因为模板变量$END$是预定义的 ，因此不可编辑。它始终代表模板扩展后填充所有必填字段时光标的位置。所以在我们的例子中，游标将停留在新类声明的末尾。

5。记住新的实时模板

您只需单击“ 确定”在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）。扩展为存根Python类的新实时模板（或代码片段）已准备就绪。我们来试试吧......

使用您自己的实时模板

首先，创建一个Python文件（ Alt + Insert - Python文件），并将其命名为sample.py 。

新的Python文件打开以进行编辑。接下来，让我们在其中创建一个类声明。为此，请键入模板缩写class 。

哇...我们的新实时模板现在位于建议列表中 - 您可以通过描述告诉我们我们输入的内容以防万一：

按Tab键选择此选项。

正如所料，缩写扩展为存根Python类。红线标记下一个入口点：当您键入类名（变量class ）时，它将在此特定位置输入。

顺便提一下，请注意，变量class已被多次使用 - 在类声明中，在类构造函数的注释中。在这种情况下， class的字段已自动填写。

键入类名后，按Enter键 ，看到红线（框架）移动到下一个字段。因此，您必须填写所有必填字段，并在结尾处按Enter键 。

在不同IDE之间共享模板的示例

在PyCharm中创建user模板后，您可以重复使用它来在另一个IDE中使用Python脚本，例如IntelliJ IDEA 。您只需导出 PyCharm Live Template配置并将其导入 IntelliJ IDEA即可。
