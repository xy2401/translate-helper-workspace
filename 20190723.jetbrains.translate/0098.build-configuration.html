介绍

IntelliJ IDEA中的每个Flash模块都包含一个或多个构建配置。构建配置定义如何将模块源文件转换为目标输出类型（SWF或SWC），然后打包。

创建模块时会创建一个构建配置。您可以根据需要添加更多构建配置。

构建配置类型

构建配置类型由以下定义：

  • 目标平台（即将使用开发内容的环境）：Web（用于Flash播放器/ Web浏览器目标内容），桌面（用于Adobe AIR目标内容）或Mobile（AIR Mobile，用于内容）适用于Android和iOS移动设备）。

  • 构建配置是使用Flex框架还是纯ActionScript。

  • 输出类型：应用程序（可运行的应用程序，SWF文件），库（SWC文件）或运行时加载的模块（可动态加载的模块 ，SWF文件）。

构建配置的主要选项

定义构建配置类型后，可以指定以下主要选项：

  • 对于Applications和运行时加载的模块： 主类 。

  • 输出文件名。

  • 输出文件夹。

  • 对于Web应用程序：包含HTML包装器模板的文件夹。

  • 对于Web和桌面应用程序： 模块和运行时样式表 。

构建配置依赖项（构建路径）

通常，每个构建配置的构建路径可以通过以下方式定义：

  • Flex SDK。根据构建配置类型自动选择必要的SDK SWC。此外，您可以通过选择以下内容来管理SWC集：

    • 对于Web目标平台：Flash播放器版本。如果SDK包含多个播放器版本，您可以选择应使用哪个相应的SWC。

    • 对于基于Flex框架的构建配置：Flex 4组件集。您可以指定只应使用Spark或MX或两个组件集。

      对于Flex组件，您可以指定其框架链接类型 （Merged Into Code，RLS或External）。

  • 生成库和运行时加载模块的其他构建配置。

  • 第三方库，包括SWC和原始ActionScript。

编译器选项

每个构建配置都与一组编译器选项相关联。

IntelliJ IDEA在IDE，项目和模块级别提供默认的编译器选项集。

较低级别的默认值可以从较高级别继承或重新定义。在单个构建配置级别，类似地，编译器选项可以从模块级别默认值继承。或者，可以指定构建配置特定值。

IntelliJ IDEA提供了一个方便的界面，用于编辑各种级别的默认值。还可以恢复已更改的值的默认值。

主动构建配置

其中一个构建配置对相应的模块设置为活动状态。

活动构建配置为编辑器中的源代码突出显示提供了基础。因此，当您更改活动配置时，模块中突出显示的代码也会更改。

管理构建配置及其设置

要管理构建配置及其设置，请按照下列步骤操作

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 在项目设置下 ，选择模块 。

  3. 在该地区下 ，展开必要的Flash模块节点，然后选择任何构建配置。现在你可以：

    • 在对话框右侧显示的Build Configuration页面上查看和编辑所选构建配置的设置。

    • 创建所选构建配置的副本。为此，请单击或在上下文菜单中选择复制 。在打开的对话框中指定复制构建配置的设置。请注意，根据设置，构建配置的副本可能具有与原始类型不同的类型 。请参阅更改构建配置类型时的可能更改 。

    • 在项目中查找所选构建配置的用法。为此，请单击 ，按Alt + F7或在上下文菜单中选择查找用法 。

    • 删除所选的构建配置。为此，请单击 ，按Delete或在上下文菜单中选择“ 删除 ”。

    • 创建新的构建配置。要做到这一点：

      1. 点击 ，按Alt + Insert或在上下文菜单中选择“ 新建 ”。

      2. 选择Flash构建配置 。

      3. 在打开的“ 添加构建配置”对话框中，指定名称并选择新构建配置的主要选项，然后单击“ 确定” 。

      4. 如有必要，请在“ 构建配置”页面上编辑构建配置设置。

    • 更改构建配置类型 。为此，请单击“ 常规”选项卡上的“ 更改 ”（显示构建配置类型的区域右侧），然后在打开的对话框中指定构建配置属性。请参阅更改构建配置类型时的可能更改 。

  4. 在Project Structure对话框中单击OK 。

另请参阅使用快捷方式打开构建配置设置，请按照下列步骤操作 。

选择活动的构建配置

要选择活动的构建配置，请按照下列步骤操作

当前处于活动状态的构建配置的名称显示在状态栏的右侧。

要激活其他构建配置，请单击此名称。

因此，将显示Active Flash构建配置菜单，您可以在其中选择将变为活动状态的其他构建配置。

使用.as或.mxml文件时，也可以从编辑器访问Active Flash构建配置菜单。Active Flash Build Configuration命令可用于此目的。

使用快捷方式打开构建配置设置

要使用快捷方式打开构建配置设置，请按照下列步骤操作

如前所述， Active Flash构建配置菜单用于选择活动构建配置（请参阅选择活动构建配置，请按照以下步骤操作 ）。

此外，此菜单提供了访问构建配置设置的快捷方式（“ 项目结构”选项）。

在Active Flash构建配置菜单中选择“ 项目结构”时，将打开“ 项目结构”对话框，显示活动构建配置的设置。

另请参阅， 要管理构建配置及其设置，请按照下列步骤操作 。

更改构建配置类型时可能的更改

更改构建配置类型时 （通过使用“更改类型”命令或创建构建配置的副本），如果构建配置输出类型已更改（例如，从“应用程序”到“库”），则可能会发生以下更改：

  • 输出文件扩展名可能会更改（从.swf到.swc ，反之亦然）。

  • 如果输出类型已更改为库，则会删除对运行时加载的模块（如果存在）的依赖性。

  • 如果对具有已更改类型的构建配置的依赖性变得不合适，则会删除此类依赖项。

创建构建配置的副本时，还会发生以下更改：

  • 输出文件名更改。

  • 如果适用，包文件名或名称会更改。
