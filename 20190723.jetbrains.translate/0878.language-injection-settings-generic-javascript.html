单击时将打开该对话框并在上下文菜单中选择Generic JS ，或选择一个条目并单击 。

项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 要在相应字符串值之前添加的字符序列。

  • 后缀。 要在相应字符串值之后添加的字符序列。

前缀和后缀是可选的。有关详细信息，请参阅 。

地方模式

在此字段中，键入定义您希望IntelliJ IDEA将文字识别为注入的上下文的规则。

规则是从程序结构接口模式构建的，实际上是内部IntelliJ IDEA语言方法的链接调用。程序结构界面显示IntelliJ IDEA处理文件的结构。

这些规则是IntelliJ IDEA内部，强烈建议您使用预定义的注入配置并避免创建新的注入配置。

高级

在此区域中，指定其他设置以缩小注射适用的上下文，从而实现对注射过程的更细粒度控制。

  • 值模式 - 在此字段中，键入一个正则表达式，用于确定将语言注入的上下文。通过使用模式的第一个捕获组作为注入目标，可以将过程配置为仅将语言注入与特定模式匹配的值或注入与模式匹配的多个部分。例如， ^javascript:(.*)匹配可以在超链接hrefs中用于执行JavaScript代码的javascript协议。

  • 单个文件 - 如果选项关闭，则与值模式匹配的片段将作为不同的“文件”单独处理 - 例如从片段编辑器的角度。

    如果选项打开，则相应的片段将合并在一起以形成单个单元或“文件”。

    鉴于价值模式

    xxx（。+）yyy（。+）zzz

    和片段

    xxx select * yyy from family zzz ，

    如果选项关闭， select *和from family将被视为两个独立的片段（或“文件”）。如果选项处于启用状态，则select * from family将被视为单个单元或“文件”。
