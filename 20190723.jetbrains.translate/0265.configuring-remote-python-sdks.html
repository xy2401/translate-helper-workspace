配置远程Python解释器

以下内容仅在安装并启用Python插件时有效！

IntelliJ IDEA提供与远程主机上运行的Python解释器的完全集成。

在开始使用远程解释器之前，请确保已安装并启用了SSH Remote Run ， Vagrant和Docker Integration插件。如果未激活插件 ，请按照管理插件中的说明在“设置/首选项”对话框的“ 插件”页面上启用它们。

根据远程解释器的类型，请遵循以下过程之一：

配置SSH解释器：

  1. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目结构”对话框中，选择“ 平台设置”部分下的SDK ，然后单击 ，从弹出菜单中选择Python SDK 。

  3. 在“ 添加Python解释器”对话框的左侧窗格中，单击“ SSH解释器” 。

  4. 在右侧窗格中，选择“ 新建服务器配置” ，然后指定服务器信息（主机，端口和用户名）。

    或者，您可以选择“ 现有服务器配置”，然后从列表中选择任何可用的部署配置。如果需要，请单击查看所选部署配置的“ 连接”设置 ，“ 映射 ”和“ 排除”路径 。单击“ 下一步”继续配置解释程序。

  5. 在下一个对话框窗口中，提供身份验证详细信息以连接到目标服务器。

    选择密码或密钥对（OpenSSL或PuTTY）并输入您的密码或密码。

    单击“ 下一步”继续执行最终配置步骤。

  6. 在下一个对话框窗口中，验证所需Python的路径是否已解释。您可以接受默认值，也可以指定其他默认值。您必须配置本地项目和服务器之间的路径映射。为此，请单击在“ 同步文件夹”字段旁边，输入本地项目文件夹的路径以及远程服务器上文件夹的路径。

    您还可以选择最低复选框以启用对远程服务器的本地更改的自动上载。

要配置Vagrant：

  1. 确保在您的计算机上创建了Vagrant实例并正确初始化。

  2. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ）。

  3. 在“ 项目结构”对话框中，选择“ 平台设置”部分下的SDK ，然后单击 ，从弹出菜单中选择Python SDK 。

  4. 在“ 添加Python解释器”对话框的左侧窗格中，单击“ Vagrant” ：

  5. 单击“浏览”按钮在Vagrant实例文件夹字段旁边，并指定所需的Vagrant实例文件夹。

    这导致显示到Vagrant主机URL的链接。

  6. Python解释器路径字段显示所需Python可执行文件的路径。您可以接受默认值，也可以指定其他默认值。

  7. 单击确定 。配置的远程解释器将添加到列表中。

要将Docker配置为远程解释器：

  1. 确保已在计算机上设置Docker 。

  2. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ）。

  3. 在“ 项目结构”对话框中，选择“ 平台设置”部分下的SDK ，然后单击 ，从弹出菜单中选择Python SDK 。

  4. 在打开的对话框中，选择Docker选项，从下拉列表中选择Docker服务器（如果缺少服务器，单击New ... ），并指定映像名称。

    Python解释器路径应该具有默认值，例如python ：

  5. 单击“ 确定”完成任务。

要将Docker Compose配置为远程解释器：

  1. 确保已在计算机上设置并运行Docker Compose 。

  2. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ）。

  3. 在“ 项目结构”对话框中，选择“ 平台设置”部分下的SDK ，然后单击 ，从弹出菜单中选择Python SDK 。

  4. 在打开的对话框中，选择Docker Compose选项，从下拉列表中选择Docker服务器，Docker Compose服务（此处为web ），配置文件（此处docker-compose.yml ）和图像名称（此处为python ）。

    接下来，等待IntelliJ IDEA启动Docker-Compose配置进行扫描和索引：

  5. 单击“ 确定”完成任务。

对于任何配置的Python解释器，您可以：

  • 安装和卸载解释器路径
  • 安装和卸载解释器包
