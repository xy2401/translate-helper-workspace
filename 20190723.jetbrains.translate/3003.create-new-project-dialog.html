创建新项目对话框

档案|新项目

欢迎屏幕|创建新项目

使用此对话框可以创建空项目或生成用于开发应用程序的特定于框架的项目存根。

对话框的左侧窗格显示了可以为其生成存根的项目类型列表。可用项目类型集取决于已安装和已激活的插件。右侧窗格的内容取决于所选的项目类型。“ 位置”字段对于所有项目类型都是通用的。

项目类型

描述

空项目

选择此选项只能获取没有任何内容的项目文件夹。在“ 位置”文本框中指定项目文件夹的路径，然后单击“ 创建” 。

HTML5 Boilerplate

选择此选项可以设置项目结构，并根据HTML5 Boilerplate模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从“ 版本”列表中，选择要使用的模板版本，然后单击“ 创建” 。

Web入门套件

选择此选项可创建项目，设置其结构，并根据Web Starter Kit要求生成某些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从“ 版本”列表中，选择要使用的模板版本，然后单击“ 创建” 。

反应App

选择此选项以使用全局安装的create-react-app包生成项目，有关详细信息，请参阅使用create-react-app生成React应用程序 。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在Node Interpreter字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择“ 添加”以配置新的解释器

  3. 在create-react-app字段中，指定create-react-app包的路径。

  4. （可选）在“ 脚本版本”字段中，指定要在项目生成期间使用的自定义程序包而不是react-scripts 。这可以是从react-scripts派生的一个包，例如， react-awesome-scripts ， custom-react-scripts ， react-scripts-ts等等。

引导

选择此选项可以创建项目，设置其结构，并基于Bootstrap模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从“ 版本”列表中，选择要使用的模板版本，然后单击“ 创建” 。

基础

选择此选项可以设置项目结构，并根据Foundation框架模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从“ 版本”列表中，选择要使用的模板版本，然后单击“ 创建” 。

AngularJS

选择此选项可以设置项目结构，并根据AngularJS框架模板生成一些源。

在右侧窗格中，指定将存储与项目相关的文件的文件夹的路径。

角度CLI

选择此选项以获取存根，以后您可以使用Angular CLI命令行界面自动生成特定结构，例如类 ， 组件 ， 路由 ， 管道 ， 服务等。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在Node Interpreter字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择“ 添加”以配置新的解释器

  3. 在Angular CLI字段中，指定@angular/cli包的路径。

反应原生

选择此选项可获取用于开发React Native应用程序的存根。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在Node Interpreter字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择“ 添加”以配置新的解释器

  3. 在React Native字段中，指定react-native-cli包的路径。

Node.js Express App

选择此选项可以设置项目结构，并根据Express框架生成一些项目源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. Node解释器，请参阅配置本地Node.js解释器 。

  3. 包管理器使用- NPM或纱线 ，看到NPM和纱线 。

  4. 要使用的Express模板引擎 。从“ 模板引擎”下拉列表中，选择以下选项之一：

    • 灰尘

    • EJS

    • 把手

    • Hogan.js

    • 帕格 （ 帕格（玉） ）

    • 枝条

    • VASH

    • 静态HTML

  5. 要使用的CSS引擎。从CSS引擎下拉列表中，选择以下选项之一：

    • 简单的CSS

    • 唱针

    • 减

    • 指南针 。

    • 萨斯 。

Cordova App

选择此选项可以设置项目结构，并根据PhoneGap ， Apache Cordova和Ionic框架生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 指定可执行文件phonegap.cmd或cordova.cmd或ionic.cmd （请参阅安装PhoneGap / Cordova / Ionic ）。

约曼

选择此选项以使用Yeoman工具获取用于生成特定于框架的项目存根的界面。

右侧窗格显示所有以前安装的Yeoman生成器 。从列表中选择所需的生成器，单击“ 下一步” ，然后按照Yeoman中的说明继续操作。

流星应用程序

选择此选项可以设置项目结构，并根据Meteor框架生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

  3. 从“ 模板”列表中，选择要生成的样本。要生成基本项目结构，请选择“ 默认”选项。

  4. 在“ 文件名”文本框中，键入将生成的相互关联的.js ， .html和.css文件的名称。仅当从“ 模板”下拉列表中选择“ 默认”样本类型时，该字段才可用。

Vue.js

选择此选项以使用Vue CLI创建Vue.js应用程序。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在Node Interpreter字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择“ 添加”以配置新的解释器

  3. 在vue-cli或@ vue / cli字段中，指定存储vue-cli或@vue/cli包的文件夹。

  4. 从“ 项目模板”列表中，选择要使用的Vue.js模板 。

镖

选择此选项可以设置项目结构，并为Dart应用程序生成一些源。
