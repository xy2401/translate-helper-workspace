版本控制

档案|设置|适用于Windows和Linux的版本控制
IntelliJ IDEA |偏好| macOS的版本控制
按Ctrl + Alt + S

此节点下的设置允许配置与不同版本控制系统的集成。

无论使用哪种版本控制系统，都应用于项目文件的常用设置：

  • 确认

  • 背景

  • 忽略的文件

  • 问题导航

  • 变更列表

配置与特定版本控制系统集成的设置位于以下节点下：

  • GitHub上

  • 混帐

  • 水银

  • Perforce公司

  • 颠覆

  • TFS

指定将用于特定目录或整个项目的版本控制系统 。

项目

描述

目录

此字段显示项目目录或项目根目录的路径。

对于启用了Git或Mercurial集成的项目，IntelliJ IDEA会扫描项目目录以检查是否存在不受IDE控制的Git / Mercurial存储库。如果找到这样的存储库，它们将在未注册的根目录下列出 ，并标记为灰色。要添加未注册的根，请在列表中选择它，然后单击“ 添加”按钮 。

IntelliJ IDEA还检查注册的根是否有效，换句话说，是否存在指定路径上的Git / Mercurial存储库。如果检测到无效的存储库，则会将其标记为红色。

VCS

选择指定目录的版本控制系统。

该列表仅显示为其启用相应插件的版本控制系统。

单击此按钮可打开“ 版本控制配置”对话框，并更新所选VCS的配置设置。

单击此按钮可将目录映射添加到列表中。将打开“ 添加VCS目录映射”对话框，您可以在其中指定所需目录，为其选择VCS，并打开“ 版本控制配置”对话框以根据需要配置指定的VCS。

单击此按钮可编辑选定的目录映射。将打开“ 编辑VCS目录映射”对话框，您可以在其中更新所选映射并根据需要配置指定的VCS。

单击此按钮可从列表中删除选定的目录映射。

限制历史记录

选中此复选框以指定文件历史记录的显示行数。如果选中此复选框，则会启用历史记录深度的文本框和旋转框。

显示具有已更改后代的目录

如果选中此复选框，则包含更改的目录将进行颜色标记。

颜色可在“编辑器”设置的“ 颜色方案”页面中进行配置（文件状态 - 立即更改子项，已更改后代）。

在分布式版本控制系统下存放文件的基本修订版。

此选项仅适用于Git和Mercurial 。选中此复选框可自动搁置Git或Mercurial版本控制下的文件的基本修订。

默认情况下，IntelliJ IDEA始终“记住”最后一次提交哈希。但是，如果由于运行rebase操作而自上次提交以来历史记录已更改，则此信息是不够的。在这种情况下，拥有基本修订版的副本可能会有所帮助。

显示在最后<number>天内更改

选中此复选框可在堆栈跟踪分析和调试期间应用文件状态的颜色指示 。将在相应的时间段内突出显示在特定时间段内更改的文件的名称。

指定天数。

按范围筛选更新项目信息

如果启用此选项并选择了作用域，则属于此作用域的文件将在版本控制工具窗口的“ 更新项目信息”选项卡中以粗体标记。如果单击“ 过滤器”按钮在“ 更新项目信息”选项卡的工具栏中，将按范围过滤文件，即仅显示属于所选范围的文件。

单击“ 管理范围”链接以打开“ 范围设置”对话框并配置范围。

提交消息右边距（列）

在此字段中，指定可以适应“ 提交更改”对话框右边距的符号数。

如果希望在达到最大字符数时将文本传输到下一行，请选择“ 键入到右边距”选项时的“ 换行”。

在“提交”对话框中显示无版本文件

选择此选项可在“ 提交更改”对话框的“未版本控制的文件”节点下查看尚未添加到版本控制的新添加文件。

检查提交邮件拼写

如果要自动检查提交消息的拼写，请选中此复选框。
