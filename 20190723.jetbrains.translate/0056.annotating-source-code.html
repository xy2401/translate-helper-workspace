注释

Java注释是一些元数据，提供有关它们使用的代码的信息，并可以指示IDE如何处理此代码。

在Java中，有一组内置注释。最重要的是，IntelliJ IDEA提供了一组注释，可作为单独的项目使用 。您可以将它们添加到类路径中并在代码中使用。

将org.jetbrains.annotations添加到Gradle或Maven项目

  1. 要将带注释的库添加到Gradle项目，请将compile 'org.jetbrains:annotations:16.0.2'依赖项添加到build.gradle文件。

  2. 对于Maven项目，将org.jetbrains:annotations:16.0.2依赖项添加到pom.xml 。

将org.jetbrains.annotations添加到其他项目

  1. 打开“ 项目结构”对话框（ Ctrl + Shift + Alt + S ），然后选择“ 库” 。

  2. 点击 ，并选择From Maven 。

  3. 在搜索字段中，如果使用JDK 1.8或更高版本，请键入org.jetbrains:annotations:16.0.2 。

    对于JDK org.jetbrains:annotations-java5:16.0.2或1.7，请键入org.jetbrains:annotations-java5:16.0.2 。

  4. 单击确定 。

在编辑器中添加org.jetbrains.annotations

您还可以使用意图操作启用注释。

  1. 在编辑器中，键入注释，例如@NotNull ，然后按Alt + Enter ：

  2. 从列表中，选择“ 添加'注释”到类路径 。

    IDE将提示您从Maven下载带有注释的库。

annotations工件需要JDK 1.8或更高版本。如果您的项目是使用JDK annotations-java5或1.7编译的，请使用annotations-java5工件。

JetBrains注释

有关JetBrains注释的更多信息，请参阅GitHub上的JetBrains注释页面

  • @Nls注释表明带注释的代码元素是需要本地化的字符串。

  • @NonNls注释表明带注释的代码元素是用户不可见的字符串，它不需要本地化，也不包含需要本地化的字符串。当您使用@NonNls注释元素时，本地化工具将跳过此元素和其中的字符串。

  • @PropertyKey批注指示方法参数接受必须是特定资源束中的有效属性键的参数。当字符串文字不是包中的属性键作为参数传递时，IntelliJ IDEA会将其突出显示为错误。注释还用于在作为参数传递的字符串文字中提供完成。

  • @TestOnly注释指示必须仅从测试代码调用方法或构造函数。

  • @Contract注释允许您指定方法必须遵循的一组规则（合同）。如果违反合同，IntelliJ IDEA会报告问题。

  • @Nullable注释指示可以为null的变量，参数或返回值。

  • @NotNull注释指示不能为null的变量，参数或返回值。
