创建和编辑ActionScript和Flex应用程序元素

此功能仅在Ultimate版本中受支持。

在处理ActionScript和Flex应用程序时，您可以创建和编辑应用程序元素，例如包 ，ActionScript 类和接口以及MXML组件 。

对于ActionScript类，接口和MXML组件，IntelliJ IDEA提供了许多预定义的文件模板 。

创建包

  1. 在“ 项目”工具窗口中，选择源根文件夹（例如src ）或要在其中创建新包的包。选择文件|新的|打包 ，或按Alt + Insert并选择包 。

    或者，右键单击相应的文件夹或包，然后选择“ 新建”|从上下文菜单中打包 。

  2. 在打开的“ 新建包”对话框中，指定包名称，然后单击“ 确定” 。

    请注意，如果使用点（ . ）分隔包名称，则可以一次创建多个包。例如，如果您键入myPackage.mySubpackage并且当前不存在这些包， mySubpackage创建这些包（ myPackage和mySubpackage ）。

    另请注意，您可以在创建新的ActionScript类或接口或MXML组件时创建新包。请参阅创建ActionScript类或接口以及创建MXML组件 。

创建ActionScript类或接口

  1. 在“ 项目”工具窗口中，选择源根文件夹（例如src ）或要在其中创建新ActionScript类或接口的包。选择文件|新的| ActionScript Class ，或按Alt + Insert并选择ActionScript Class 。

    或者，右键单击相应的文件夹或包，然后选择“ 新建”|上下文菜单中的ActionScript类 。

  2. 在打开的“ 新建ActionScript类”对话框中，指定类的名称，包，要使用的文件模板 ，然后单击“ 创建” 。如果使用Class with Supers模板 ，您还可以指定类应该实现的超类和/或一个或多个接口。

    请注意，如果指定尚不存在的包，则将创建相应的包。

另请参阅在编辑器中启动ActionScript类创建 。

创建MXML组件

  1. 在“ 项目”工具窗口中，选择源根文件夹（例如src ）或要在其中创建新MXML组件的包。选择文件|新的| MXML组件 ，或按Alt + Insert并选择MXML组件 。

    或者，右键单击相应的文件夹或包，然后选择“ 新建”|上下文菜单中的ActionScript类 。

  2. 在打开的“ 新建MXML组件”对话框中，指定组件的名称，包，要使用的文件模板 ，父组件，然后单击“ 创建” 。

    请注意，如果指定尚不存在的包，则将创建相应的包。

在编辑器中启动ActionScript类创建

使用类的源代码时，可以在编辑器中开始创建其子类。同样，您可以启动创建实现接口的类。出于这些目的，IntelliJ IDEA分别提供了名为Create Subclass和Implement Interface的意图操作 。

以下是使用Create Subclass意图操作的示例。（以类似的方式访问实现接口操作。）

  1. 在编辑器中，将光标放在包含类声明的行中。

  2. 按Alt + Enter并选择Create Subclass 。

  3. 在打开的“ 新建ActionScript类”对话框中，指定新类的设置，然后单击“ 创建” 。

基于模板的ActionScript类，接口和MXML组件

根据文件模板创建ActionScript类，接口和MXML组件。可以使用以下预定义模板：

  • ActionScript类。 此模板生成如下所示的文件内容（在创建类时指定了目标包和类的名称）：

    package myPackage { public class MyClass { public function MyClass （） { } } }

  • 带有超级的ActionScript类。 此模板生成下面显示的文件内容（创建类时，目标包，类名，超类和/或类实现的接口）：

    包 mypackage的 { 公共 类 MyClass1的 延伸 MyClass的 实现 IMyInterface1，IMyInterface2 { 公共 函数 MyClass1的 （）{ 超级 （）; } } }

  • ActionScript接口。 此模板生成如下所示的文件内容（创建接口时指定目标包和接口名称）：

    包 mypackage的 { 公共 接口 IMyInterface的 {}}

  • Flex 3组件。 此模板生成下面显示的文件内容（根标记由创建组件时指定的父组件定义;生成以下示例时， mx.core.Application被指定为父组件）：

    <？xml 版本 = “1.0” ？> < mx ： 应用程序 xmlns ： mx = “http://www.adobe.com/2006/mxml” > </ mx ： 应用程序 >

  • Flex 4组件。 此模板生成如下所示的文件内容（根标签由创建组件时指定的父组件定义;生成以下示例时， spark.components.Application被指定为父组件）：

    <？xml 版本 = “1.0” ？> < s ： 应用程序 xmlns ： fx = “http://ns.adobe.com/mxml/2009” xmlns ： s = “library：//ns.adobe.com/flex/spark” > </ s ： 应用程序 >

如有必要，您可以修改预定义模板或创建自己的自定义文件模板。请参阅文件和代码模板以及为MXML组件创建自定义文件模板的示例 。

另请参阅ActionScript和Flex的预定义文件模板变量 。

ActionScript和Flex的预定义文件模板变量

对于ActionScript和Flex文件模板， 预定义模板变量列表更广泛。此外，还提供以下预定义变量：

  • ${Superclass} - ActionScript类的超类或MXML组件的父组件。

  • ${SuperInterfaces} - ActionScript类实现的接口列表。

为MXML组件创建自定义文件模板的示例

如前所述，预定义的MXML 4 Component文件模板只生成MXML组件的根标记。假设您希望另外生成<fx:Declarations>和<fx:Script>标记。

如果由于某种原因您希望保持预定义模板不变，则可以创建相应的自定义文件模板。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中， 编辑器下的clicm 文件和代码模板 。

  2. 在对话框右侧打开的“ 文件模板”页面上，选择“ 模板”选项卡。

  3. 选择Flex 4 Component。

  4. 现在，要创建此模板的副本，请单击 。

  5. 例如，将模板的名称更改为带声明和脚本的Flex 4组件 。

  6. 在模板体中（ 根据样式复选框下面的区域显示复选框），后面的行

    <$ {Superclass} xmlns：fx =“http://ns.adobe.com/mxml/2009”>

    加

    <fx：声明> </ fx：声明> <fx：Script> <！[CDATA [ ]]> </ fx：Script>

  7. 在Settings对话框中单击OK 。

现在，检查结果：

  1. 选择源根文件夹（例如src ）并按Alt + Insert 。请注意，现在可以在“ 新建”菜单中使用带有声明和脚本选项的Flex 4组件 （对应于新模板的名称）。

  2. 选择MXML组件 。在打开的“ 新建MXML组件”对话框中，检查“ 模板”列表的内容。请注意， 带有声明和脚本的Flex 4组件也已添加到此列表中。

使用SWF元数据标记来控制HTML包装器属性

您可以在主应用程序类中使用SWF元数据标记来设置HTML包装器中的标题，背景颜色以及width和height属性。

例如，如果ActionScript类包含

打包 myPackage { 端口 flash 。 显示 。雪碧 ; WF （ pageTitle = “hello” ， backgroundColor = “＃ccddee” ， width = “400” ， height = “200” ）] public class MyClass extends Sprite { ...

相应的属性将在HTML包装器中设置。

以类似的方式，如果相应的文件包含以下内容，SWF元数据标记将适用于MXML组件：

<？xml版本= “1.0”？> <s：应用程序 xmlns：fx = “http://ns.adobe.com/mxml/2009” xmlns：s = “library：//ns.adobe.com/flex/spark” > <fx：元数据> [ SWF（pageTitle =“hello”，backgroundColor =“＃ccddee”，width =“400”，height =“200”）] </ fx：元数据> ... </秒：应用>

默认情况下，标题与嵌入的.swf文件的名称相同，背景颜色为白色（ #ffffff ）， width和height均为100% 。

编辑ActionScript和Flex源

编辑ActionScript和Flex源时，可以使用以下IntelliJ IDEA功能：

  • 语法和错误突出显示。请注意，突出显示代码的方式由活动的构建配置定义。

  • 代码完成 。

  • 周围有标签或代码构造（ Ctrl + Alt + T和Ctrl + Alt + J ），并删除封闭的标签（ Ctrl + Shift + Delete ）。

  • 意图操作和快速修复 （ Alt + Enter ）。

     

  • 代码检查。

  • MXML组件和ActionScript类的结构视图 。

  • 跳转到声明 （ Ctrl + B ）。

  • 代码重构。

  • 自动代码生成 （ Alt + Insert ）：生成getter和setter，可绑定的getter和setter，事件处理程序等。
