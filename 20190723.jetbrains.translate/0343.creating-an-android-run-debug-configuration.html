创建Android运行/调试配置

  1. 要开始创建Android运行配置，请选择“运行”|从主菜单编辑配置 。或者，单击Shift + Alt + F10并从列表中选择“ 编辑配置 ”。单击“ 添加新配置”按钮在工具栏上，从列表中选择Android应用程序 。在“ 运行/调试配置：打开的Android应用程序”页面上，指定配置名称并选择将应用此配置的模块。

  2. 指定将在目标设备上部署的.apk文件，并指定将在应用程序启动时启动的活动。

    • 要运行或调试整个应用程序，请从“ 安装选项”区域的“ 部署”下拉列表中选择“ 默认APK” ，然后从“ 启动 选项”区域的“ 启动”下拉列表中选择“ 默认活动” 。IntelliJ IDEA将上传从上面的Module下拉列表中指定的模块构建的.apk 。.apk是自动构建的，不需要从您身边进行初步的工件配置。

    • 要运行或调试稍后将嵌入到应用程序中的自定义.apk ，请从“ 安装选项”区域的“ 部署”下拉列表中选择“ 自定义工件”选项，然后选择要从中构建.apk的工件。在这种情况下，您必须在创建运行/调试配置之前手动定义工件（有关详细信息，请参阅通过工件生成签名版本APK和使用工件 ）。然后从Launch Options区域的Launch下拉列表中选择Specified Activity选项，并从所选工件（ .apk ）中指定启动活动。手动键入活动名称或单击“ 浏览” 并在打开的“ 选择活动类”对话框中选择它。

      可用活动列表由模块的选择决定。

    • 如果要为已在运行的应用程序启动调试会话，请选择“不在 包中 部署任何内容 ”以禁止将数据上载到设备，然后在“ 活动”区域中选择“不启动活动 ”。使用这些设置执行运行配置与单击“ 将调试器附加到Android进程”按钮相同在工具栏上。

  3. 在“ 部署目标选项”区域中，指定将启动应用程序的设备。
    • 要指定虚拟设备，请选择“ 模拟器”选项，然后从“ 首选Android虚拟设备”下拉列表中选择一个虚拟设备。

      如果没有可用的虚拟设备，请单击“ 浏览” 启动Android虚拟设备（AVD）管理器并配置新的模拟器（有关详细信息，请参阅管理虚拟设备 ）。

    • 如果要手动选择目标设备，请选择“ 显示选择器”对话框选项。每次启动运行/调试会话并应用此配置时，IntelliJ IDEA都将显示“ 选择设备”对话框。

    • 要在应用程序启动时让IntelliJ IDEA检测到插入的USB设备，请选择USB设备选项。

    如果要在稍后插入的物理设备上运行应用程序，则选择“ 显示选择器”对话框或“ USB设备”选项可能会有所帮助，因此无法预见可用设备集。
