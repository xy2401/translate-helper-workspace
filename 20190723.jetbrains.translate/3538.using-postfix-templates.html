JavaScript Postfix模板

所有后缀模板的默认扩展键是Tab 。请参阅更改默认扩展密钥以了解如何选择另一个。

Postfix代码完成允许您在刚刚键入的表达式周围添加模板代码。当您在点后面键入缩写（后缀）并按下扩展键或在代码完成弹出窗口中选择缩写时，模板会展开。

WebStorm附带了一组预定义的后缀模板，可让您为JavaScript和TypeScript定义自己的自定义模板。请注意，JavaScript模板也适用于TypeScript文件。

可以复制，更新和删除自定义模板。对于预定义模板，您只能更改其后缀，例如将短键替换为较短的键。

激活后缀完成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 常规 ” ，然后单击“ 后缀完成” 。

  2. Postfix Completion页面打开，显示可用的后缀列表，旁边有相应的模板。选择后缀时，“ 描述”窗格会显示相应的转换，显示扩展模板之前和之后的代码段。

  3. 选中“ 启用后缀模板”复选框以及需要激活的后缀旁边的复选框。

要应用后缀模板

  1. 键入表达式后跟一个点。

  2. 键入后缀并按扩展键（默认值为Tab ）或从建议列表中选择后缀。如有必要，请选择要包围或替换的表达式。

创建自定义模板

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ Editor下的Editor 常规 Editor ，然后单击“ 后缀完成” 。

  2. 在打开的Postfix完成页面上，单击 ，并选择模板的工作语言。您可以选择JavaScript或TypeScript，JavaScript模板也适用于TypeScript文件。

  3. 在打开的“ 创建新模板”对话框中，键入模板后缀，选择语言级别，键入模板代码，并在需要插入初始表达式的位置添加$EXPR$ 。在您希望光标位于末尾的位置添加$END$ 。

您还可以创建一个与现有模板略有不同的新模板，例如，在其语言上下文或光标的最终位置。WebStorm允许您复制原始模板并在该副本中进行必要的更改。

从现有模板创建新的自定义模板

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ Editor下的Editor 常规 Editor ，然后单击“ 后缀完成” 。

  2. 在打开的Postfix Completion页面上，选择要从中创建新模板的自定义模板，然后单击在工具栏上。

  3. 在打开的“ 编辑模板”对话框中，根据需要编辑模板并键入新的后缀。

要更改默认扩展密钥

  • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ Editor下的Editor 常规 Editor ，然后单击“ 后缀完成” 。在打开的Postfix Completion页面上，从Expand templates with list中选择一个新密钥。

禁用后缀完成

  • 要禁止展开所有已配置的后缀模板，请清除“ 启用后缀模板”复选框。

  • 要禁止展开特定模板，请清除其后缀旁边的复选框。

创建React组件模板

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ Editor下的Editor 常规 Editor ，然后单击“ 后缀完成” 。

  2. 在打开的Postfix完成页面上，单击 ，并选择JavaScript 。将打开“ 创建新模板”对话框。

  3. 在“ 密钥”字段中，键入rcomp缩写。

  4. 从Minimum language level列表中，选择React JSX 。这意味着该模板仅用于语言级别为React JSX或Flow的 JavaScript文件中。

  5. 使用$EXPR$键入模板文本，其中将显示初始表达式（以.rcomp ）。不要忘记将$END$放在扩展模板后光标停止的位置：

    class $ EXPR $ extends Component { render（）{ return （ < div > $ END $ < / div> ）; }} 出口 默认 $ $ EXPR;

要应用新的React组件模板

  1. 在编辑器中，键入新组件Button的名称，后跟一个点。
