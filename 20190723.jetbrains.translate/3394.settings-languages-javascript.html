JavaScript语言版本

从此下拉列表中，选择代表要在项目中使用的语言功能集的JavaScript语言版本。可用选项包括：

  • ECMAScript 3

  • ECMAScript 5.1

  • JavaScript 1.8.5

  • ECMAScript 6 ：此版本增加了对ECMAScript 2015-2017中引入的功能的支持以及对该标准的一些当前提议。

  • React JSX ：此版本在ECMAScript 6之上添加了对JSX语法的支持

  • Flow ：此版本增加了对Flow语法的支持。

只有基于类型的完成

  • 清除此复选框后，完成列表包含复杂情况下的多个变体。

  • 选中该复选框后，完成列表很大程度上取决于WebStorm类型推断。这使得完成更加精确，但是如果推断不良，则列表可能是空的。

默认情况下，清除该复选框。

建议变量名称

选中此复选框后，WebStorm会在声明过程中为新变量建议名称。这些建议基于项目中定义的类，类型和接口的名称，您正在使用的库以及标准API中的名称。

默认情况下，清除该复选框。

在完成中展开方法体以进行覆盖

如果要从父类或接口覆盖方法并从完成建议列表中选择此方法，WebStorm会自动添加参数，生成super()调用，并在可能的情况下添加类型信息。

默认情况下，选中该复选框。清除它以禁止在完成期间为覆盖自动生成方法体。

流程包或可执行文件

在此字段中，指定node_modules\flow-bin包或Flow二进制可执行文件的路径。要使用node_modules\.bin\flow确保将Node.js的路径添加到PATH环境变量中。

仅当从JavaScript语言版本列表中选择Flow时，该字段才可用。

使用Flow服务器：

在此区域中，通过选中或清除以下复选框指定编码帮助的基础：

  • 类型检查：选中此复选框后，将根据从Flow服务器接收的数据提供语法和错误突出显示。清除该复选框后，只有基本的内部WebStorm突出显示可用。

  • 导航，代码完成和类型提示：选中此复选框后，参考分辨率和代码完成的建议列表包含从与Flow集成检索的建议和WebStorm计算的建议。清除该复选框后，仅通过WebStorm计算解析引用。

仅当指定了Flow可执行文件的路径时，复选框才可用。

自动保存所有修改的文件

选中此复选框以确保连续应用Flow，因为Flow仅在保存所有其他已修改文件后才检查当前文件。
