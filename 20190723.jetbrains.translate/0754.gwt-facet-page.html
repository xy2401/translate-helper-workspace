GWT安装的路径
目录

在此下拉列表中，选择GWT SDK的位置。如有必要，请单击并在打开的对话框中选择所需的位置。

目标Web方面

在此下拉列表中，选择GWT组件应归属的位置。

可用选项包括：

  • 无：生成的文件不会归因于任何Web方面。

  • Web：生成的文件将归于war / explosion目录。

    仅当GWT应用程序放置在具有Web构面的Web模块中时，此选项才可用。

JavaScript输出样式

在此下拉列表中，选择编译器将应用于生成的JavaScript代码的优化样式。可以使用以下选项：
  • 混淆。选择此选项可使输出更小，从而加快加载速度。

    输出JavaScript代码中的函数名称是不可预测的。

  • 漂亮。选择此选项可使输出对人类可读。

  • 详细。选择此选项可获得更详细的输出，例如，详细的变量名称。

编译器最大堆大小

在此文本框中，指定将由GWT编译器使用的Mb中的最大堆大小。

为了启动GWT Compiler，您需要将GWT Compiler Output元素添加到工件中。（项目结构|工件|输出布局）

因此，GWT Compiler在工件构建时启动，生成的文件存储在工件的输出目录中。

其他编译器VM选项

在此文本框中，指定其他VM选项，例如，用于编译和运行大型GWT项目的VM选项。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

    - Dfoo = $ { MY_ENV_VAR }

编译器参数

在此文本框中，以命令行的格式指定要传递给GWT编译器进程的选项 。

使用与指定VM选项相同的规则。

GWT模块

此只读列表显示项目中的所有GWT模块。要使make过程中涉及GWT模块，请选中列表中模块名称旁边的复选框。

make过程自动重新编译自上次编译以来已更改的所有模块或项目源。

输出相对路径

在此文本框中，指定相对于GWT模块编译输出根的输出路径。

下载GWT

单击此按钮可下载GWT SDK。

创建神器

要启动GWT编译器，您需要在“ 输出布局”选项卡中添加一个配置了GWT Compiler输出元素的工件。单击此按钮可自动创建具有已配置输出布局的工件。
