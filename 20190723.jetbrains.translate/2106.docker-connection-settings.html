使用连接到Docker守护程序

适用于Mac的Docker。 仅适用于macOS：如果您使用Docker for macOS，这是推荐的连接选项。

Unix套接字。 仅适用于Linux：这是Linux的推荐连接选项。

如果您获得权限被拒绝错误，请确保将当前用户添加到计算机上的docker组。

Docker Machine。 如果您使用的是Docker Toolbox for Windows或macOS，这是连接Docker API的推荐选项。

应立即显示“ 连接成功”消息。如果没有，请检查Docker上的Docker Machine可执行文件设置工具页面 。

如果您安装并运行了多台Docker Machine，请使用该列表选择应使用哪台计算机。

TCP套接字。 如果您使用的是Docker for Windows，这是常用的连接选项。此选项也适用于Linux，Docker for macOS和Docker Toolbox。

  • 引擎API URL。 取决于Docker版本和操作系统：

    • Docker for Windows： tcp://localhost:2375

      在Docker设置的“ 常规”部分中，打开tcp：// localhost：2375上的Expose守护程序，不带TLS选项。

    • 适用于macOS或Linux的Docker： unix:///var/run/docker.sock

    • 适用于Windows或macOS的Docker Toolbox： https://192.168.99.100:2376 ：//192.168.99.100： https://192.168.99.100:2376

  • 证书文件夹证书文件夹的路径。根据您的Docker版本和操作系统：

    • 适用于Windows，macOS或Linux的Docker：此字段必须为空。

    • 适用于Windows的Docker Toolbox： <your_home_directory>\.docker\machine\machines\default

    • 适用于macOS的Docker Toolbox：通常是<your_home_directory>/.docker/或其子目录。

    请注意以下事项：由于Certificates文件夹字段指定具有证书的任何文件夹，因此该字段对应于环境变量DOCKER_CERT_PATH 。请参阅Docker文档的客户端模式页面上的详细信息。

路径映射

对于Windows和macOS：指定要映射到容器卷的文件夹的主机 - 虚拟机路径映射。

使用编辑现有映射，或创建一个新的。在打开的对话框中：

  • 本地路径。 要为卷绑定提供的本地文件夹的路径。

  • 虚拟机路径。 Docker虚拟机文件系统中的相应目录路径。
