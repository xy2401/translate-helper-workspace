准备为PHP应用程序开发Google App

所描述的功能由Google App Engine支持PHP插件处理。该插件不再维护，与IntelliJ IDEA 2018.1及更高版本不兼容。

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA负责创建特定的目录结构和设置。IntelliJ IDEA可以为特定于PHP的项目存根生成Google App Engine，也可以在现有项目中启用对Google App Engine for PHP的支持。

  • 在你开始之前
  • 为PHP应用程序创建Google App Engine的项目存根
  • 在现有项目中激活Google App Engine支持

在你开始之前

  1. 下载并安装Python 2.7版。

  2. 下载并安装适用于PHP的Google App Engine 。

  3. 如管理插件中所述，在插件页面上安装适用于PHP插件的Google App Engine支持。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

  4. 在http://www.google.com上创建一个Google帐户。

  5. 在Google App Engine的“ 创建应用程序”页面上创建应用程序并记住其应用程序ID，您必须在创建项目期间指定它。

为PHP应用程序创建Google App Engine的项目存根

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ PHP ”。

  3. 在右侧窗格中，选择App Engine Project ，然后单击“ 下一步” 。

  4. 在向导的第二页上，指定以下内容：

    1. 项目的名称以及要在其中创建的文件夹。

    2. 要使用的模板版本。

    3. 在“ 应用程序ID”字段中，键入在“ 创建应用程序”页面上指定的应用程序的标识符。

    4. 在SDK目录字段中，指定安装Google App Engine SDK for PHP的文件夹的路径。

    5. Python可执行文件的位置。在Python可执行文件字段中手动键入路径，或单击“ 浏览” 并在打开的对话框中选择Python可执行文件。

    准备好后单击完成 。

IntelliJ IDEA使用main.php文件和app.yaml配置文件创建项目。app.yaml文件包含生成的运行时设置（应该是php55 ）， api_version ， applicationID ， 应用程序版本 （默认为1）， threadsafe元素具有true值以启用发送多个并行请求，以及脚本处理程序的 url和script （值为main.php ）。

在现有项目中激活Google App Engine支持

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ PHP ”，然后单击“ Google App Engine for PHP” 。

  2. 在打开的Google App Engine for PHP页面上，选中“ 启用Google App Engine for PHP支持”复选框并指定以下内容：

    1. 存储Google App Engine for PHP SDK的文件夹。

    2. Python 2.7可执行文件的路径。

    3. 在App Engine帐户设置区域中，选择向开发服务器进行身份验证的方式，可用选项包括：

      • 通过OAuth2使用无密码登录：选择此选项以使用OAuth 2.0协议 。要保存通过Google Developers Console实现的令牌 ，请清除“不保存令牌”复选框。

      • 使用电子邮件和密码登录：选择此选项以使用您的Gmail地址和密码。
