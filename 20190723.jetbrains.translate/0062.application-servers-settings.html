IntelliJ IDEA |偏好|构建，执行，部署| macOS的应用程序服务器
从运行/调试配置对话框：编辑运行/调试配置为应用程序服务器配置的时候。

使用此页面或对话框管理支持的应用程序服务器及其设置的配置。

已在IntelliJ IDEA中定义的服务器显示在窗格下方和 。在此窗格中选择服务器时，相应的配置设置和关联的控件将显示在右侧区域中。（如果要编辑特定服务器（例如Tomcat）的运行配置，则仅显示此特定服务器的现有配置。）

使用创建服务器配置和删除它们。

当您开始创建服务器配置时，必须已在您的计算机上安装相应的服务器。（对于CloudBees，这不是必需的。）

请注意，您可以使用的服务器列表取决于当前启用的服务器集成插件 。请参阅启用应用程序服务器集成插件 。

  • 工具栏
  • 主要设置和控制
  • 图书馆
  • 框架的附加库
  • 配置文件列表（适用于Jetty 7或更高版本）

工具栏

项目

描述

或Alt + Insert

使用此图标或快捷方式创建服务器配置（即在IntelliJ IDEA中定义服务器）：

  1. 如果显示服务器列表，请选择感兴趣的服务器。

  2. 在打开的对话框中，指定服务器设置，然后单击“ 确定” 。

    对于大多数服务器，您只需指定服务器安装目录（称为服务器主目录）。有关更多信息，请参阅主要设置和控件 。

或Alt +删除

使用此图标或快捷方式删除所选的应用程序服务器配置。

主要设置和控制

项目

描述

名称

服务器配置名称。

<服务器>主页

应用程序服务器安装文件夹的路径。

点击并在打开的对话框中选择文件夹。

<服务器>版本

检测到的应用程序服务器版本（只读）。

<Server>基目录（用于Jetty，Tomcat和TomEE）

服务器基目录的路径。

点击并在打开的对话框中选择目录。

注册模式（适用于JBoss）

为了能够在IntelliJ IDEA中编辑JBoss XML配置文件（例如standalone.xml ， domain.xml等），可以在JBoss安装中注册XML模式（XSD）：

  1. 单击注册模式 。

  2. 要注册所有模式，只需在打开的对话框中单击“ 确定 ”。

    如果存在您不想注册的模式，请在“ 外部模式和DTD”下选择这些模式，单击“确定” 然后单击“ 确定” 。

要在以后查看或编辑模式列表，请使用“ 设置”对话框中的“ 模式和DTD”页面 。

下载客户端库（适用于CloudBees）

嵌入在CloudBees中的Tomcat实例用作服务器。此Tomcat实例包含在您可以下载的CloudBees客户端库中。

如果该按钮处于非活动状态，则IntelliJ IDEA中已存在客户端库（以及相应的Tomcat实例）。

另请参见“ 库” ， “框架的附加库”和“ 配置文件”列表（适用于Jetty 7或更高版本） 。

图书馆

创建服务器配置时，通常会创建关联的应用程序服务器库 。通常，此库包括服务器分发中可用的Servlet，JSP和EJB实现。

使用“ 库”下的控件来管理库的内容。

项目

描述

（ Alt + Insert ）

将项目（类，源，文档等）添加到库中。

在打开的对话框中，选择所需的文件和文件夹。这些文件可能是单独的.class和.java文件，也可能是包含此类文件的目录和存档（ .jar和.zip ）。

IntelliJ IDEA将分析选定的文件和文件夹，并自动将其内容分配给相应的库类别（类，源，文档等）。

当IntelliJ IDEA无法猜出类别时（例如，当您选择空文件夹时），将显示一个对话框，您可以在其中自行指定类别。

为了能够使用在线提供的外部文档，请单击此图标并指定外部文档的URL。

排除项目（请参阅排除库项目 ）。在打开的对话框中，选择您希望IntelliJ IDEA忽略的项目（存档中的文件夹，存档和文件夹），然后单击“ 确定” 。

（ 删除 ）

此选项允许您：

  • 从库中删除选定的普通库。

  • 恢复所选的已排除项目。这些物品本身将留在图书馆。

框架的附加库

对于某些应用程序服务器配置（例如GlassFish），有一个带有复选框的附加库框架列表。

此列表中的每个项目都可能是实现相应框架的库 。（相应的库文件包含在服务器分发中。）

在列表中选中一个复选框后，将创建相应的应用程序服务器库。打开的对话框允许您选择应使用此库的模块。（因此，库将添加到所选模块的依赖项中。）

配置文件列表（适用于Jetty 7或更高版本）

对于Jetty 7或更高版本，有一个部分列出了服务器配置（.xml）或模块（.mod）文件。

本节提供了一个用于编辑文件<jetty_home>\start.ini 。 （ start.ini包含<jetty_home>\start.jar start.ini的选项，用于启动Jetty。）

使用复选框使文件处于活动或非活动状态。使用 ， ， ， 和添加，删除，替换和重新排序文件。

项目

描述

活性

使用复选框使文件处于活动或非活动状态。

取消选择文件后，会立即对start.ini的相应行进行注释。使文件处于活动状态时，将取消注释start.ini的相应条目。

路径

显示相应文件的路径（只读）。

对于Jetty安装目录中的文件，路径是相对于安装目录的。对于所有其余文件，将显示绝对路径。

或Alt + Insert

使用此图标或快捷方式将文件添加到列表中。在打开的对话框中 ，选择所需文件，然后单击“ 确定” 。

将文件添加到列表会导致向start.ini添加新的（注释）条目。

或Alt +删除

使用此图标或快捷方式从列表中删除所选文件。请注意，此操作不会物理删除文件。但是，相应的条目将从start.ini删除。

或输入

使用此图标或快捷方式替换所选文件。在打开的对话框中 ，选择替换文件，然后单击“ 确定” 。

或者Alt + Up

使用此图标或快捷方式将所选文件在列表中向上移动一行。

或Alt + Down

使用此图标或快捷方式将所选文件在列表中向下移动一行。
