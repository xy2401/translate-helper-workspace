分析Xdebug分析数据

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

当启用与Xdebug探查器的集成时，IntelliJ IDEA提供了探查器快照的可视化表示。IntelliJ IDEA打开一个单独的编辑器选项卡，其中包含四个视图，其中数据基于不同的标准显示。

要收集分析数据并对其进行分析，请执行以下常规步骤：

  • 启动Xdebug调试会话
  • 检索分析器累积的数据
  • 检查分析数据

启动Xdebug调试会话

要启动Xdebug会话，请执行以下操作之一：

  • 要开始调试整个应用程序，请创建PHP Web页面类型的调试配置，然后单击Debug启动调试 。

    有关详细信息，请参阅使用PHP网页调试配置进行调试 。

  • 要调试特定的PHP HTTP请求，请定义PHP HTTP Request类型的调试配置，然后单击Debug启动调试 。

    有关详细信息，请参阅调试PHP HTTP请求 。

  • 要启动零配置调试会话：

    1. 切换“ 开始侦听PHP调试连接”按钮在IntelliJ IDEA工具栏上，它更改为 。之后，IntelliJ IDEA开始侦听当前项目中使用的调试引擎的端口。调试端口在语言和框架|的IntelliJ IDEA级别设置PHP | “ 设置/首选项”对话框的“ 调试”页面（ Ctrl + Alt + S ）。

    2. 在浏览器中打开应用程序的起始页面，选择Start debugger bookmark从浏览器激活调试引擎，重新加载当前页面（应用程序的起始页面），然后返回到IntelliJ IDEA。

检索分析器累积的数据

  1. 在主菜单上，选择“ 工具”|分析Xdebug Profiler快照 。

  2. 在打开的“ 选择Xdebug探查器快照”对话框中，选择存储分析数据的文件夹和文件 。

    IntelliJ IDEA在单独的编辑器选项卡中显示收集的分析数据，其中包含所选探查器输出文件的名称。

检查分析数据

当您请求累积的分析数据时，IntelliJ IDEA会在单独的编辑器选项卡中打开其可视化的演示文稿。该选项卡以选定的探查器输出文件命名，并包含多个视图。在视图之间切换，以根据各种分析标准分析分析数据。

  • 在“ 执行统计信息”视图中，检查有关每个被调用函数的执行度量的摘要信息。

  • 在“ 调用树”视图中，浏览所有调用函数的执行路径。

  • 要探索特定函数的执行路径，请在“ 调用树”视图中选择该函数，并在“ 被调用者”视图中查看其被调用者 。

  • 要探索可能导致调用特定函数的所有路径，请在“ 调用树”视图中选择该函数，并在“ 调用者”视图中检查其可能的调用者 。
