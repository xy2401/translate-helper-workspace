Linked Gradle项目

此区域包含已链接到IntelliJ IDEA项目的已注册Gradle项目列表。

项目级设置

此区域包含Gradle项目的设置。您可以选择以下选项：

  • 使用自动导入 - 选中此复选框可在每次刷新项目时自动解决对Gradle项目所做的所有更改。

  • 自动为空内容根创建目录 - 选择此选项可在从Gradle模型导入项目时自动将src目录添加到项目中。

  • 组模块-您可以组件组 使用显式模块组或使用合格的名称 。

  • 为每个源集创建单独的模块 - 选中此复选框以在解析Gradle项目时使用源集功能。

  • 外部存储生成的项目文件。

    默认情况下，选择此选项，并在外部存储新创建或导入的项目。这意味着生成的.iml和库文件存储在idea.system.path而不是.idea目录中。通过版本控制共享项目可能会有所帮助。

  • 使用默认gradle包装器（推荐） - 选中此复选框以使用Gradle Wrapper 。在默认位置生成或签出包装文件时，可以使用此选项。

  • 使用gradle wrapper任务配置 - 选中此复选框以自定义Gradle Wrapper 。如果您的磁盘上还没有包装文件，或者您使用其他位置，则可以使用此选项。在这种情况下，IntelliJ IDEA引用Gradle包装器任务定义，并根据任务配置生成或更新文件。当您不想检入二进制包装器文件或为多个项目重用相同的包装器文件时，此选项非常有用。

    Gradle 1.7或更高版本支持此选项。

  • 使用本地gradle分发 - 选择此选项以运行本地构建脚本。

  • Gradle home - 在此文本字段中，指定Gradle安装的完全限定路径。

    如果Gradle位置已由环境变量GRADLE_HOME或PATH定义，则IntelliJ IDEA会推导出此位置，并将此路径建议为默认值。

    如果尚未从环境变量推断出Gradle位置，请手动指定它，或单击“ 浏览”按钮，然后在打开的对话框中选择所需的目录。请注意，在此字段中输入的值优先于环境变量。

  • Gradle JVM - 使用此列表选择用于运行Gradle项目的JVM。默认设置为项目JDK。

  • 委派设置 - 使用此部分指定在指定的链接项目中运行测试，构建或运行操作时IntelliJ IDEA应使用的内容。您可以委派所选链接项目的执行（在Gradle设置的Linked Projects部分中进行选择）到Gradle或使用IntelliJ IDEA。

    请注意，Gradle编译器不支持链式表达式的代码完成 ，您需要使用IntelliJ IDEA编译器。

全局Gradle设置

此区域包含Gradle全局设置选项。您可以选择以下选项：

  • 脱机工作 - 使用此复选框可在脱机模式下使用Gradle。在这种情况下，Gradle将使用缓存中的依赖项。Gradle不会尝试访问网络以执行依赖项解析。如果依赖项的缓存中不存在所需的依赖项，则构建执行将失败。

  • 服务目录路径 - 使用此字段覆盖默认的Gradle主目录位置目录。

  • Gradle VM选项 - 使用此字段为Gradle项目指定VM选项。

    指定JVM选项时，请遵循以下规则：

    • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

    • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

    • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

    • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

      - Dfoo = $ { MY_ENV_VAR }
