配置代码覆盖率测量

IntelliJ IDEA可以配置代码覆盖率测量的各个方面。在这个部分：

  • 配置处理coverage套件的方式
  • 配置代码覆盖率测量选项
  • 更改覆盖突出显示的颜色

配置代码覆盖行为

  1. 打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），然后单击“ 构建”，“执行”，“部署”下的“ 覆盖范围 ”。Coverage页面打开。

  2. 定义如何处理收集的覆盖率数据：

    • 要在每次启动具有代码覆盖率的新运行配置时显示“ 代码覆盖率”对话框，请在将coverage应用于编辑器之前选择“ 显示选项” 。

    • 要放弃新的代码覆盖率结果，请选择“不应用已收集的覆盖范围” 。

    • 要在每次启动具有代码覆盖率的新运行配置时丢弃活动套件并使用新套件 ，请选择“ 使用新套件替换活动套件” 。

    • 要在每次启动具有代码覆盖率的新运行配置时将新代码覆盖套件附加到活动套件，请选择“ 添加到活动套件” 。

  3. 使用coverage运行应用程序或测试时，定义Coverage工具窗口的行为：

    • 要自动打开Coverage工具窗口，请选中Activate Coverage View复选框。

    • 要手动打开Coverage工具窗口，请清除Activate Coverage View复选框。

配置代码覆盖率选项

  1. 打开“编辑运行/调试配置”对话框，添加所需的运行/调试配置，然后单击“ 代码覆盖率”选项卡。

  2. 在“ 代码覆盖率”选项卡中，定义以下选项：
    • 从“ 选择覆盖运行器”下拉列表中，选择所需的代码覆盖运行器。可用选项包括：

      • EMMA

      • IntelliJ IDEA

    • 选择所选跑步者的选项：

      • 对于EMMA跑步者，只有采样模式可用。

      • 对于IntelliJ IDEA运行器，您可以在采样或跟踪模式之间进行选择。

    • 指定范围以测量代码覆盖率。执行以下操作之一：

      • 要指定类，请单击“ 添加类”按钮。

      • 要指定包，请单击“ 添加包”按钮。

    • 要为包含测试的文件夹收集代码覆盖率统计信息，请选中“ 在测试文件夹中启用覆盖率”复选框。

或者，只需点击在统计弹出窗口中。

配置代码覆盖颜色

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 颜色方案 ” ，然后单击“ 常规” 。“ 颜色方案”页面打开。
