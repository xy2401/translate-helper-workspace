使用SSH配置远程解释器

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

先决条件

  • ssh服务器应该在远程主机上运行，因为PyCharm通过ssh-session运行远程解释器。

  • 如果要将源复制到远程计算机，请创建部署配置，如创建远程服务器配置一节中所述。

通过SSH凭据配置远程Python解释器

通过SSH凭据配置远程Python解释器

  1. 通过以下任一方式打开“ 添加Python解释器”对话框：

    • 当您在编辑器中时 ，最方便的方法是使用状态栏中的Python解释器小部件。单击小部件并选择Add Interpreter ...

    • 如果您位于“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，请选择“ 项目<项目名称>”项目口译员 。点击图标，然后选择添加 。

  2. 在“ 添加Python解释器”对话框的左侧窗格中，单击“ SSH解释器” 。

  3. 在右侧窗格中，选择“ 新建服务器配置” ，然后指定服务器信息（主机，端口和用户名）。

    或者，您可以选择“ 现有服务器配置”，然后从列表中选择任何可用的部署配置。如果需要，请单击查看所选部署配置的“ 连接”设置 ，“ 映射 ”和“ 排除”路径 。单击“ 下一步”继续配置解释程序。

  4. 在下一个对话框窗口中，提供身份验证详细信息以连接到目标服务器。

    选择密码或密钥对（OpenSSL或PuTTY）并输入您的密码或密码。

    单击“ 下一步”继续执行最终配置步骤。

  5. 在下一个对话框窗口中，验证所需Python的路径是否已解释。您可以接受默认值，也可以指定其他默认值。您必须配置本地项目和服务器之间的路径映射。为此，请单击在“ 同步文件夹”字段旁边，输入本地项目文件夹的路径以及远程服务器上文件夹的路径。

    您还可以选择最低复选框以启用对远程服务器的本地更改的自动上载。

  6. 使用带有root权限的解释器通过sudo复选框选择执行代码，以使用sudo运行SSH解释器。只有启动远程解释器才会启用root权限。文件将以您当前的用户权限复制到远程服务器。

  7. 单击“ 完成”以完成添加解释器。如果以root用户身份添加解释器，系统会要求您提供密码。

    选择记住即使重新启动PyCharm也要继续使用此解释器的密码。添加了root权限的解释器在可用解释器列表中标有sudo 。
