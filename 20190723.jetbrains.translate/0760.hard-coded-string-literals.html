硬编码的字符串文字

如果源代码包含硬编码字符串文字，则可以启用国际化代码检查以突出显示它们。然后，您可以将这些字符串提取到属性文件中进行本地化，或者如果它们不是要本地化则忽略它们。

启用突出显示硬编码字符串文字

  1. 打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），展开“ 编辑器” ，然后单击“ 检查” 。

  2. 选择所需的配置文件，然后在Java下找到节点国际化问题 。

  3. 启用硬编码字符串检查以在编辑器中突出显示硬编码字符串文字。

  4. 应用更改并关闭对话框。

现在编辑器将突出显示硬编码的字符串文字，如下面的屏幕截图所示：

您可以配置此检查的严重性，并准确指定应识别硬编码字符串的情况。例如，您可以选择跳过toString()方法的内容，或者不包含字母字符的文字。

提取硬编码的字符串文字

IntelliJ IDEA提供特殊意图操作i18nize硬编码字符串文字，以将字符串文字提取到属性文件中。您可以使用java.util访问资源包。ResourceBundle类或自定义资源包类 。

使用java.util提取字符串文字。资源包

  1. 在源代码中，使用以下格式指定将用于存储提取的文字的资源包：

    private static ResourceBundle < field name > = ResourceBundle 。 getBundle （ “<bundle name>” ）;

    例如：

    private static ResourceBundle myBundle = ResourceBundle 。 getBundle （ “awesomeBundle” ）;

  2. 单击突出显示的字符串，按Alt + Enter ，然后选择i18nize硬编码字符串文字 。

  3. 在I18nize Hard Coded String Literal对话框中，指定目标属性文件，属性的键和值以及资源包表达式。

    如果已在源代码中声明ResourceBundle字段，则默认情况下IntelliJ IDEA会建议其名称。如果尚未在源代码中声明此字段，则仍可以在对话框中立即定义所需的表达式。为此，请在“ 资源包”表达式字段中输入ResourceBundle类型的有效表达式。

  4. 单击确定 。具有硬编码字符串文字的行被替换。

    • 如果资源包已在源代码中声明：

      系统 。 出 。 的println（myBundle 的getString（ “hello.world”））;

    • 如果已在对话框中定义资源包：

      系统 。 出 。 的println（ 的ResourceBundle 的getBundle（ “awesomeBundle”） 的getString（ “hello.world”））;

您可以跳过此步骤，并在I18nize Hard Coded String Literal对话框中指定所需的资源包表达式。

指定资源包表达式时，可以使用基本代码完成 。键入类名称，然后在句点后按Ctrl + Space以选择方法。

使用自定义资源包类提取字符串文字

  1. 确保IntelliJ IDEA安装下的redist/annotations.jar存档已添加到模块依赖项中。

  2. 在项目中创建一个新的Java类，并粘贴以下代码：

    import org.jetbrains.annotations。PropertyKey ; import org.jetbrains.annotations。NonNls ; import java.util。ResourceBundle ; import java.text。MessageFormat ; public class I18nSupport { @NonNls private static final ResourceBundle bundle = ResourceBundle 。 getBundle （ “com.intellij。FontChooser“ ）; public static String i18n_str （ @PropertyKey （ resourceBundle = ”com.intellij。FontChooser”）String 键 ， 对象 ... PARAMS）{ 字符串 值 = 束 的getString（ 键 ）; 如果 （PARAMS 长度 > 0） 返回 的MessageFormat 格式 （ 值 ，则 params）; 返回 值 ;}}

  3. 在包含硬编码文字的类中，单击突出显示的字符串，按Alt + Enter ，然后在意图操作列表中选择i18nize硬编码字符串文字 。“ I18nize Hardcoded String”对话框打开时没有有效的资源包表达式。

  4. 单击编辑i18n模板 。在“ 文件模板”对话框中，将I18nized Expression更改为指向自定义资源包类的方法。

    单击“ 确定”以保存更新的模板并关闭对话框。

  5. 在“ I18nize Hardcoded String”对话框的“ 预览”部分中，验证建议的替换，然后单击“ 确定” 。源代码相应地改变：

    系统 。 出 。 的println（I18nSupport i18n_str（ “hello.world”）。）;

这种变化是全球性的，并影响到所有项目！

忽略硬编码的字符串文字

如果要忽略硬编码的字符串文字，请使用“ 不需要国际化注释”。

  1. 按Alt + Enter以显示字符串文字的意图操作。

  2. 从操作列表中选择Annotate as @NonNls 。

  3. 指定要存储annotations.xml文件的位置。
