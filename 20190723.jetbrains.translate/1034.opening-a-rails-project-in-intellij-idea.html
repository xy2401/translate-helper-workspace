IntelliJ IDEA 2019.1帮助

打开并运行Rails项目

此功能仅在Ultimate版本中受支持。

在本教程中，我们将向您展示如何克隆现有的Rails应用程序，配置Ruby解释器，安装依赖项以及运行此应用程序。以下应用程序将用于此目的： 示例Rails应用程序 。

先决条件

在使用IntelliJ IDEA中的Rails项目之前，请确保满足以下先决条件：

  • Ruby插件已安装并启用。请参阅管理插件以了解更多信息。

  • Ruby SDK安装在您的计算机上。

检查一个项目

首先，我们需要克隆包含示例应用程序的存储库：

  1. 在欢迎屏幕上 ，单击从版本控制中 签 出并选择Git 。

  2. 在调用的Clone Repository对话框中，将以下存储库地址插入URL字段： https ： //bitbucket.org/rubyminedoc/sample_rails_app.git 。

    然后，单击“ 克隆”按钮。

  3. 克隆存储库后，系统将提示您从这些源创建IntelliJ IDEA项目。

    单击是 。

  4. 在调用的“ 导入项目”向导中，在向导的每个页面上单击“ 下一步” ，保留默认设置。

  5. 在最后一页上，单击“ 完成” 。

配置Ruby SDK

现在我们需要为项目配置Ruby解释器：

  1. 转到文件|项目结构 ，或按Ctrl + Shift + Alt + S.

  2. 在调用的Project Structure对话框中，切换到Modules页面并选择Ruby解释器。

    单击确定 。

安装宝石

要安装项目的Gemfile中指定的gem，请执行以下操作：

  1. 按两次Ctrl键并输入以下命令： gem install bundler 。

    按Enter键并等待，直到安装了Bundler gem。

  2. 再次按Ctrl键两次，然后开始键入bundle install 。然后，从列表中选择bundle install命令，然后按Enter键 。

  3. 在调用的“ Bundle Install”对话框中，单击“ Install” 。

  4. 等到IntelliJ IDEA安装所有宝石。

运行迁移

要运行我们的Rails应用程序，首先我们需要迁移数据库：

  1. 按两次Ctrl键并键入db：migrate 。在下拉列表中选择rake db:migrate ，然后按Enter键 。

  2. 保留调用的Execute'db：migrate'对话框中的默认设置，然后单击OK 。

  3. 等到迁移完成。

运行该应用程序

要运行我们的Rails应用程序，我们需要创建相应的Rails配置：

  1. 按Ctrl + Shift + A并开始键入编辑配置 。选择Edit Configurations ，然后按Enter键 。

  2. 在调用的“运行/调试配置”对话框中，单击“确定” 并选择Rails 。

  3. 将创建的配置名称更改为开发并启用“ 运行”浏览器选项。

  4. 按两次Ctrl键 ，然后开始键入已创建配置的名称（ 开发 ）。从列表中选择它，然后按Enter键 。

  5. 在“运行”工具窗口中，IntelliJ IDEA将显示准备要运行的应用程序的过程...

    ...然后在浏览器中打开应用程序。
