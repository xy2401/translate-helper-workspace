工具Windows查看模式

您可以通过各种方式查看和调整工具窗口，以便在编辑器或其他工具窗口中工作时快速访问它们并节省空间。

  • 从主菜单中，选择Window |活动工具窗口|查看模式和实际模式选项。

  • 在工具窗口的标题栏上，单击 ，从选项列表中，选择查看模式和实际模式选项。

  • 右键单击工具窗口或标题栏，然后从选项列表中选择“ 查看模式”和“实际模式”选项。

您可以在“ 外观和行为”|“ 窗口选项 ”下为工具窗口配置许多常规查看选项IntelliJ IDEA设置的外观页面（ Ctrl + Alt + S ）。

停靠固定/停靠未固定/未固定模式

即使您的焦点位于其他位置，也可以使工具窗口保持打开状态，例如，在编辑器或其他工具窗口中。要实现此目的，请在选择查看模式时使用Dock固定 模式 。

要在切换焦点时关闭工具窗口，请使用Dock unpinned或Undock模式。

在停靠模式下，工具窗口的所有边都附加到周围元素（编辑器，其他工具窗口等）。在此模式下，工具窗口和相邻元素共享主窗口中的可用空间。

取消停靠时，工具窗口的所有侧面（工具窗口栏上的除外）都与周围元素分离。窗口移动到“上层”，覆盖用于与之共享空间的元素。在其中一个方向（沿工具窗口栏），窗口伸展并占用所有可用空间。在另一个方向上，其中一个窗口边框变得松散，可以移动而不会影响其他底层元素的大小。

浮动/窗口模式

您可以使用Float或Window模式分离工具窗口。

最初，所有工具窗口都处于“固定”模式，这意味着它保持在主窗口内。

  • 在浮动模式下，工具窗口可以在屏幕上移动，但它仍然位于主IntelliJ IDEA窗口的顶部。

  • 在窗口模式下，工具窗口表现为单独的应用程序窗口：它可以在屏幕上移动并与主IntelliJ IDEA窗口重叠。

分割模式

此模式与多少个窗口有关可以一次显示固定到同一工具窗口栏的一个窗口（一个或两个）。

通常，沿工具窗口栏的空间在两组对接的工具窗口之间共享。

在每个时刻，只有来自每个组的一个窗口可见。

因此，如果停靠在工具窗口栏上的所有工具窗口都关闭了拆分模式，则一次只能显示一个工具窗口。在这种情况下，可见的工具窗口占用工具窗口栏上的所有可用空间。因此，当您显示某个窗口时，会自动隐藏前一个可见窗口。

如果对接到同一工具窗口栏的所有窗口都启用了拆分模式，则会出现相同的行为。

为了能够同时看到两个窗口，相应的窗口应该选择Docked Pinned 。

组选项卡选项

如果工具窗口中有多个视图可用，则如果在Window |中选择了Group选项卡选项，则相应的视图可能会显示在单独的选项卡上 。主菜单的活动窗口 。

宽屏支持

IntelliJ IDEA使工具窗口可以使用屏幕的整个宽度和高度。在“ 设置”对话框（ Ctrl + Alt + S ）中，展开“ 外观和行为 ”节点，然后在“ 外观”页面中，使用左侧/右侧的“ 宽屏工具窗口布局”和“并排布局 ”复选框来优化工具窗口的放置。

您可以通过工具窗口之间的拆分器上的Ctrl + MouseClick打开或关闭并排布局。

有关详细信息，请参阅宽屏工具窗口布局 。
