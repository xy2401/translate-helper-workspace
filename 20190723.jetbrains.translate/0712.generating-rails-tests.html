生成Rails应用程序的测试

介绍

如果由于某种原因你有一个没有相关测试的Rails实体，请注意IntelliJ IDEA建议在RSpec和Test :: Unit测试框架中为控制器，邮件程序，模型和帮助程序生成测试的方法。

生成测试

要为Rails实体生成测试，请遵循以下常规步骤

  1. 在编辑器中，将插入符号放在所需的Rails实体上。

  2. 执行以下操作之一：

    • 在主菜单上，选择“ 导航”|测试

    • 在上下文菜单中，选择“ 转到”测试

    • 按Ctrl + Shift + T。

  3. 如果尚不存在所需的测试，则会出现弹出的“ 生成测试 ”。在此窗口中，选择测试框架：

    如果缺少相关的测试框架，IntelliJ IDEA将建议安装缺少的gem（例如， rspec-rails ）：

  4. 单击Bundle it将gem添加到Gemfile，然后在Bundle Install对话框中运行bundler：

IDE将安装缺少的gem，并在spec或test目录下的Rails实体名称（例如， controllers或models ）的目录中以指定格式生成test ，具体取决于所选的测试框架。

如果出现问题，您可以回滚此操作。

  • 生成缺失测试功能不会创建帮助程序。因此，应手动创建生成的测试的测试助手。

  • 其他新框架将在稍后添加。请参阅https://youtrack.jetbrains.com/issue/RUBY-19144 。
