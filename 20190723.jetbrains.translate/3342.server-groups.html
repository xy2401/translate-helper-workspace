WebStorm 2019.1帮助

将服务器组织到组中

通过服务器组，您可以将服务器配置组合在一起，并与单个实体一起使用它们。如果需要将代码部署到多个服务器，则可以使用服务器组并避免单独部署到每个服务器。

创建服务器组

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在设置/首选项（Ctrl + Alt + S），选择下生成，执行，部署 部署 。

    • 选择工具|部署|在主菜单上配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加” 并在弹出菜单中选择“ 服务器组 ”。

  3. 在打开的“ 创建新组”对话框中，键入要创建的组的名称，然后单击“ 确定” 。将关闭“ 创建新组”对话框，并返回“ 部署”节点的“ 连接”选项卡。

  4. 要在组内创建新的服务器配置，请在左侧窗格中选择该组，然后单击右侧窗格中的“ 添加新服务器”链接或工具栏按钮。然后根据其类型配置新创建的服务器，如配置与Web服务器同步中所述 。

    要将现有服务器配置添加到组，请将其拖到组中。要删除服务器配置，请将其拖出组。

  5. 单击“ 用作默认值”工具栏按钮在以下情况下让WebStorm以静默方式应用当前服务器组：

    • 自动上传已更改的文件 。

    • 无需选择目标主机即可手动上载和下载文件。

    • 编辑远程主机上的单个文件 。

  6. 使用“ 当前项目”复选框配置服务器组可见性，该可见性也适用于组中包含的所有服务器。

    • 选中该复选框以限制将组用于当前项目。这样的组不能在当前项目之外重用。它不会出现在其他项目的可用配置列表中。

    • 清除该复选框后，该组在所有WebStorm项目中都可见。其设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

创建的服务器组现在可用于上载和下载文件到包含的服务器。
