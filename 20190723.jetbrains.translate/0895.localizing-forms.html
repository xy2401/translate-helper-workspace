本地化表格

GUI Designer将I18N支持扩展到GUI表单，使您能够查找包含硬编码字符串的组件并将这些字符串传输到属性文件。生成各种语言环境后，您可以在设计时更改它们并预览本地化表单的外观。

在本节中，您可以找到将i18n支持应用于GUI表单的一般建议：

  • 一般本地化程序

  • 抑制I18nize硬编码字符串快速修复

  • 更改设计时区域设置

一般本地化程序

要本地化GUI表单，请执行以下主要步骤

  1. 确保激活Hard-Coded Strings检查。有关详细信息，请参阅识别硬编码字符串文字部分。如果激活检查，则在UI Designer工具窗口中显示组件的相应工具提示，并在“表单工作区和UI设计器”工具窗口中显示快速修复i18nize属性<name> 。

  2. 为要为GUI提供的每个区域设置创建属性文件 ，以及不带后缀的默认属性文件。

    例如，对于使用英语开发并本地化为德语和法语的对话框，您可以创建以下属性文件：

    1. Subscription.properties

    2. Subscription_de.properties

    3. Subscription_fr.properties

  3. 浏览表单，然后选择包含硬编码字符串的每个组件。单击快速修复图标，或按Alt + Enter显示可用意图操作列表，然后选择i18nize属性<name> ，如下图所示：

  4. 在I18nize Hard Coded String Literal对话框中，将硬编码的字符串文字提取到指定的属性文件。有关详细信息，请参阅硬编码字符串文字部分。

  5. 编辑资源包中提取的字符串。有关详细信息，请参阅资源包部分。

抑制I18nize硬编码字符串快速修复

抑制i18nize快速修复

  1. 选择包含硬编码文本属性的组件。

  2. 单击快速修复图标，或按Alt + Enter以显示可用意图操作的列表。

  3. 将鼠标光标悬停在i18nize属性<name>快速修复上，然后单击向右箭头图标 （或按右箭头键）显示子菜单，如下图所示：

  4. 单击选项Suppress for component或Suppress for all components 。列表抑制检查显示在GUI Designer工具窗口中：

要取消抑制，请在Suppressed Inspections节点下选择所需的检查，然后按Delete键。

更改设计时区域设置

在设计时更改区域设置

  1. 在GUI Designer选项卡具有焦点的情况下，单击“ 更改本地”组合框在主工具栏上。

  2. 从下拉列表中选择所需的区域设置。
