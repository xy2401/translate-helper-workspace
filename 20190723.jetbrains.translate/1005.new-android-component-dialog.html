新的Android组件对话框

项目工具窗口|包含应用程序类的包的上下文菜单新的 - Android组件

在此对话框中，指定实现新Android组件的类，组件类型以及要向用户显示的标题。

项目

描述

名称

在此文本框中，键入实现该组件的类的名称。

类

在此下拉列表中，指定组件类型。可用选项包括：

  • AIDL ：用于进程间通信的Android接口定义语言（AIDL）接口。

  • 活动 ：实现一个窗口，您可以在其中放置UI以与用户交互。

  • Android Auto ：允许您扩展应用程序以用于车辆。您可以添加媒体服务或邮件服务活动。

  • 文件夹：根据您为此组件选择的活动创建源根。

  • 片段 ：表示活动中的行为或用户界面的一部分。

  • Google：允许您为Google地图和AdMob广告活动创建活动。

  • 应用程序 ：Android软件包，即包含Android应用程序和安装程序内容的.apk存档。

  • 服务 ：表示应用程序要求在不与用户交互的情况下执行操作，或为其他应用程序提供功能。

  • 其他：允许您将以下组件添加到您的应用程序：
    • Android清单文件
    • 广播接收器
    • 内容提供商
    • 梦话

  • UI组件 ：允许您向应用程序添加自定义视图。

  • 磨损 ：允许您扩展您的应用程序以用于Android磨损。

  • 小部件 ：允许您为应用程序添加不同类型的小部件。

  • XML ：允许您为Android布局和值添加不同类型的.xml文件。

标签

在此文本框中，键入将向用户显示的标题。

  • 如果未指定标题，则将显示分配给整个应用程序的标签。

  • 没有为远程接口组件指定标签。

标记为启动活动

选中此复选框可在应用程序启动时默认显示活动类型的新组件。

只有在Kind下拉列表中指定了Activity组件类型时，该复选框才可用。

创建布局文件

  • 选中此复选框可让IntelliJ IDEA生成相关布局定义文件的存根，即新活动或片段的内容视图。

  • 如果清除该复选框，则必须手动创建新活动或片段的内容视图。

该复选框仅适用于活动或片段组件类型。
