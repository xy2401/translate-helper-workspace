XPath表达式评估

此功能仅在Ultimate版本中受支持。

在使用程序代码或XSLT脚本之前或之前，XPath表达式需要进行评估以对其进行测试，从而对XML文档进行结构化查询。

IntelliJ IDEA允许您以两种模式评估XPath表达式：

  • 在简单模式下，您可以输入不需要任何自定义名称空间前缀的简单单行表达式。此模式不允许您配置上下文设置或使用预定义变量。

  • 在高级模式下，您可以方便地在多行模式下编辑长表达式并编辑XPath上下文。

一些错误检查和XPath检查还为检测到的问题提供快速修复，例如，可以按意图将未解析的名称空间前缀映射到URI。

评估XPath表达式

  1. 从活动编辑器选项卡的上下文菜单中选择“ 评估XPath” ，或转到“ 编辑”|找到|评估XPath 。将打开“ 评估XPath表达式”对话框。

  2. 要切换评估模式，请单击“ 高级 / 简单”按钮。在任一模式下，对话框都包含最近评估的表达式，完成，语法检查和突出显示的历史记录，以及对输入表达式的一些语义错误检查。语义检查包括使用的命名空间前缀的验证，无用的XPath表达式（例如@comment() ）和元素/属性名称的节点测试，这些测试不会出现在上下文文档中，也不会成功匹配。

  3. 浏览表达式的历史记录：

    • 在简单模式下，可以从下拉列表中选择最近使用的最近表达式。

    • 在高级模式下，使用 / 或按Alt +向上 / Alt +向下 。

  4. 要重新配置XPath上下文，请单击“ 编辑上下文” 。在打开的对话框中，将自定义前缀分配给上下文文档中使用的名称空间URI，并定义要在重复表达式的查询中使用的变量。

    分配较短的前缀，解决前缀冲突或实际定义默认命名空间的前缀可能很有用。这可能是必要的，因为XPath不会自动匹配默认命名空间中的元素，而不指定要匹配的元素的前缀。编辑名称空间及其前缀和

    可以为表中的每个变量分配一个表达式，该表达式将在执行查询时进行一次计算。然后，所得到的值可用于多次使用，无需额外的计算成本。

  5. 可选：

    • 选择突出显示结果复选框，突出显示当前编辑器中的匹配节点。不突出显示不属于当前编辑器的匹配节点（可能通过使用document()函数发生）。建议在Find Usages工具窗口中显示此类跨文档结果。

    • 选中“ 在使用情况视图中显示结果”复选框以在“ 查找用户”工具窗口中显示所有匹配的节点。选中“ 在新选项卡中打开”复选框以在新选项卡中打开结果，而不是重复使用最后一个选项卡。
