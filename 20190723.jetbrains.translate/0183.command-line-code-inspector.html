从命令行启动代码检查

除了从主菜单或项目工具窗口的上下文菜单中运行代码检查之外，您还可以从命令行启动检查器，而无需实际运行IntelliJ IDEA。

这样，您可以作为开发过程的一部分执行常规代码检查，这对于大型项目尤为重要。检查结果以XML格式存储，您可以随时下载和查看这些结果。

请注意，如果未定义SDK，则检查将失败。SDK描述应存储在config \ options \ jdk.table.xml中 。了解如何在此处配置SDK。

从命令行启动代码检查

  • 在命令行中，使用以下语法运行inspect命令：

    检查项目路径检查路径输出路径[ -vX ] [ -d子目录路径]

如果IntelliJ IDEA正在运行，则inspect实用程序将失败，因此您必须在执行命令之前退出IDE。

命令选项如下：

选项

描述

检查

的完整路径inspect.bat（inspect.sh在MacOS和Linux）脚本位于<IDE_HOME> / bin目录。

项目路径

包含要检查的项目的目录的完整路径。

请注意，您需要指定完整路径。不接受相对路径。

检查路径

检查项目的检查配置文件的完整路径。默认情况下，检查配置文件存储在以下目录中：

  • Windows： ％HOMEPATH％\。IntelliJIdea2019.1 \ CONFIG \检查

  • macOS： 〜/ Library / Preferences / IntelliJIdea2019.1 /检查

  • Linux： 〜/。IntelliJIdea2019.1 /配置/检查

输出路径

将存储包含检查报告的XML文件的现有目录的完整路径。

-vx

（可选）输出的详细级别，其中X为0表示安静， 1表示噪声， 2表示额外噪声。

-d子目录路径

（可选）要检查的子目录的完整路径。

查看脱机检查的结果

执行离线检查并将检查结果导出到XML格式的目录后，您始终可以下载并查看这些结果。

  1. 打开运行检查的项目。

  2. 在主菜单上，选择Analyze |查看离线检查结果 。

  3. 在打开的“ 选择路径”对话框中，单击存储带有检查结果的XML文件的目录。

  4. 单击确定 。检验结果显示在检验结果工具窗口的“ 脱机视图”选项卡中。

或者，您可以在IntelliJ IDEA或任何其他文本处理器中打开相关的XML文件，而无需打开已检查的项目。
