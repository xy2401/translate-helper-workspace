运行/调试配置：Node.js

运行|编辑配置| | Node.js的

在此对话框中，创建用于在本地运行和调试Node.js应用程序的配置。当前上下文中的“本地”意味着WebStorm本身启动计算机上安装的Node.js，从而启动运行或调试会话。

在你开始之前

下载并安装Node.js.

配置选项卡

项目

描述

节点解释器

在此字段中，指定要使用的Node.js解释器。这可以是本地或远程 Node.js解释器或Windows子系统Linux上的Node.js.从列表中选择一个解释器或单击并在打开的对话框中配置一个新的。

要在此项目的所有运行/调试配置中默认使用WSL上的Node.js，请选择它作为默认项目节点解释器 。

节点参数

在此文本框中，键入要传递给NodeJS可执行文件的Node.js特定命令行选项。最常见的选项是：

  • 使用--require coffee-script/register可以在运行期间将CoffeeScript文件动态编译为JavaScript。此模式要求register.js文件（它是coffee-script包的一部分）应位于项目内。因此，您需要在本地安装Node.js页面上的coffee-script包，如npm和Yarn中所述 。

  • 使用Node.js v7进行Chrome调试协议支持时，请使用--inspect参数。否则，默认情况下，调试进程将使用V8调试协议 。

有关完整列表，请参阅Node.js命令行选项 。

工作目录

在此字段中，指定应用程序的工作目录 。默认情况下，该字段显示项目根文件夹 。

JavaScript文件

在此字段中，指定启动它的应用程序的主文件的路径（例如， Node.js Express应用程序的 bin / www ）。

如果要调试CoffeeScript，请使用源映射指定生成的JavaScript文件的路径。该文件可以在外部生成，也可以使用文件监视器进行编译。有关更多详细信息，请参阅调试CoffeeScript 。

应用参数

在此文本框中，键入要通过process.argv数组启动时传递给应用程序的Node.js特定参数。

环境变量

在此字段中，指定Node.js可执行文件的环境变量 （如果适用）。单击“ 浏览” 在该字段的右侧并在“ 环境变量”对话框中配置变量列表，该对话框将打开：

  • 要定义新变量，请单击“ 添加” 并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击“ 删除” 。

  • 准备好后，单击“ 确定”

变量的定义显示在“ 环境变量只读”字段中，并以分号作为分隔符。可接受的变量是：

  • NODE_PATH ：A :以模块搜索路径为前缀的NODE_PATH目录列表。

  • NODE_MODULE_CONTEXTS ：设置为1以在其自己的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

Docker容器设置

仅当选择基于Docker的远程解释器时，才会显示此字段。

手动键入设置或单击并在打开的“ 编辑Docker容器设置”对话框中指定设置，或选择“ 自动配置”复选框以让WebStorm自动执行此操作。

  • 禁用网络 ：选中此复选框可禁用网络连接。这对应于--net="none" ，这意味着在容器内部外部网络资源不可用。

  • 网络模式 ：对应于选项--net的其他值。

    • bridge是默认值。将为网桥上的容器分配IP地址，并通过此网桥将流量路由到容器。
      容器默认情况下可以通过其IP地址进行通信。要按名称进行通信，必须将它们链接起来。

    • host ：使用容器内的主机网络堆栈。

    • container:<name|id> ：使用通过其名称或id指定的另一个容器的网络堆栈。

    有关详细信息，请参阅网络设置文档。

  • 链接 ：使用此部分将要创建的容器与其他容器链接。这适用于Network mode = bridge ，对应于--link选项。

  • 发布所有端口 ：将所有容器端口公开给主机。这对应于选项--publish-all 。

  • 端口绑定 ：指定端口绑定列表。与使用docker run的-p选项类似。

  • 额外主机 ：这对应于--add-host选项。有关详细信息，请参阅管理/ etc / hosts页面。

  • 卷绑定 ：使用此字段指定特殊文件夹卷与运行Docker守护程序的计算机文件夹之间的绑定。这对应于-v选项。
    有关详细信息，请参阅管理数据容器

  • 环境变量 ：使用此字段指定环境变量及其值的列表。这对应于-e选项。有关详细信息，请参阅ENV（环境变量）页面。

点击扩展表格。点击 ， ， 要么列出清单。

自动配置

选中此复选框可让WebStorm配置容器设置。在自动配置模式下：

  • WebStorm创建一个新映像并在其中安装npm模块。

  • WebStorm使用新映像运行容器，将项目文件夹绑定到容器中的/opt/project文件夹以确保更新时的同步，并将/opt/project/node_modules到OS临时目录。

即使使用自动配置，您仍然需要将运行应用程序的端口与容器的端口绑定。这些公开的端口可在Docker主机的IP地址上使用（默认为192.168.99.100）。调试Node.js Express应用程序的客户端时需要这样的绑定。在这种情况下，您需要从计算机打开浏览器并通过应用程序中指定的端口访问容器主机上的应用程序。

配置端口绑定

  1. 点击在Docker Container Settings字段中，展开打开的Edit Docker Container Settings对话框中的Port bindings区域。

  2. 点击在打开的“ 端口绑定”对话框中，按如下方式映射端口：

    • 在“ 容器端口”文本框中，键入应用程序中指定的端口。

    • 在“ 主机端口”文本框中，键入要从计算机在浏览器中打开应用程序的端口。

    • 在“ 主机IP”文本框中，键入Docker主机的IP地址，默认IP为192.168.99.100。主机在“设置/首选项”对话框的“ Docker”页面上的API URL字段中指定。

    • 单击“ 确定”返回“ 编辑Docker容器设置”对话框，其中将新端口映射添加到列表中。

  3. 单击“ 确定”返回“ 运行/调试配置：Node.js”对话框。

WebStorm将package.json复制到映像中的/tmp/project_modules文件夹，运行npm install ，然后将模块复制到容器中的项目文件夹中。因此，更改项目中的package.json导致重新构建映像。

浏览器/实时编辑选项卡

在此选项卡中，配置浏览器的行为并启用调试应用程序的客户端代码。此功能是通过JavaScript Debug运行配置提供的，因此从技术上讲，WebStorm为服务器端和客户端代码创建单独的运行配置，但您在一个专用的Node.js运行配置中指定了所有设置。

描述

打开浏览器

在此区域的字段中，根据要在内置Web服务器上启动的项目根目录，指定项目HTML文件以创建此文件的正确URL。例如，如果选择project_root / inner_folder / index.html，则生成的URL将为http：// localhost：63342 / project_root / inner_folder / index.html 。如果选择After Launch复选框，浏览器将在应用程序启动后自动打开此页面。或者，您可以通过在您选择的浏览器中手动打开包含此URL地址的页面来查看相同的结果。

发布后

选中此复选框可自动打开浏览器。从列表中选择要使用的浏览器：

  • 要使用系统默认浏览器，请选择“ 默认” 。

  • 要使用自定义浏览器，请从列表中选择它。请注意，仅在Chrome中完全支持实时修改 。

  • 要配置浏览器，请单击并调整打开的Web浏览器对话框中的设置。有关更多信息，请参阅配置浏览器 。

V8性能分析选项卡

在此选项卡中，启用V8 CPU和内存分析 。

项目

描述

记录CPU分析信息

选中此复选框可在启动应用程序时开始记录CPU性能分析数据。以下区域中的控件已启用。指定以下内容：

  • 日志文件夹：在此字段中，指定用于存储记录日志的文件夹。分析数据存储在日志文件isolate-<session number> 。

  • 所有隔离区的一个日志文件（V8实例）：选中此复选框以仅为所有V8实例创建一个日志文件（以及相应的一个性能分析结果视图）。清除复选框以为每个实例创建单独的文件。Node.js可以创建一个额外的V8实例，例如用于调试过程。

允许获取堆快照

如果要运行内存分析，请选中此复选框。

日志选项卡

在此选项卡中，启用在“运行”工具窗口的“ 控制台”选项卡中显示由morgan等日志记录工具生成的信息。有关详细信息，请参阅运行Node.js应用程序时管理日志 。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

共享运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

禁用时，每次启动新的运行/调试配置时，WebStorm都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

在启动选项之前

在此区域中，您可以指定在启动所选运行/调试配置之前必须执行的任务。任务按照它们在列表中出现的顺序执行。

项目

捷径

描述

Alt +插入

单击此图标可添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择要运行的一个或多个应用程序。如果尚未在WebStorm中定义，请添加其定义。有关更多信息，请参阅配置第三方工具和外部工具 。

  • 运行另一个配置 ：选择执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行文件监视器 ：选择此选项可让WebStorm应用所有当前活动的文件监视器 。

  • 运行Grunt任务 ：选择此选项以运行Grunt任务。在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项以运行Gulp任务。在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项可自动运行bundler，作为运行或调试会话的一部分。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过start npm任务运行开发服务器。为此，请单击 ，然后在Configure React Native对话框中，选择npm脚本并从列表中选择start 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上载到远程主机 ：选择此选项可根据默认服务器访问配置将应用程序文件自动上载到 服务器 。

否则，如果清除该复选框，则不会显示工具窗口。但是，当配置运行时，您可以通过按Alt + 4或Alt + 5 自行打开相应的工具窗口。
