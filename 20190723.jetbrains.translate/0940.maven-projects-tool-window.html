Maven项目工具窗口

查看|工具窗口| Maven项目

此工具窗口标有图标 。

使用Maven Projects工具窗口查看可用的Maven项目，下载源和Javadoc，以及执行构建生命周期的各个阶段。工具窗口显示每个Maven项目的节点，以及Lifecycle和Plugins子节点。如果至少有一个pom.xml文件包含配置文件定义，则“ 配置文件”节点将添加到工具窗口中。此节点包含Maven项目中定义的所有配置文件。如果将依赖项添加到项目中， Maven项目工具窗口还会显示“ 依赖项”节点。

工具栏按钮

项目

描述

单击此按钮可将所有Maven项目与IntelliJ IDEA项目同步。请参阅Maven集成对话框的导入选项卡 。

单击此按钮可启动Maven目标，以便为源和测试目录生成源和资源，并读取生成的目录结构。根据这种生成的结果，IntelliJ IDEA文件夹被正确标记为源或测试根。

请参阅导入设置 。

单击此按钮可下载缺少的源和文档。从子菜单中选择所需的下载选项。
您可以在Maven Integration对话框的Importing页面设置源和文档的自动下载。

单击此按钮可添加Maven项目。在打开的对话框中选择所需的pom.xml文件。

单击此按钮可执行构建生命周期的选定阶段或插件目标。如果选择了多个目标，它们将以与树中相同的顺序执行。请注意，默认情况下，此按钮处于禁用状态，要激活它，您需要选择要运行的构建阶段或插件目标。

单击此按钮可使用命令行执行maven目标。

单击此按钮可切换离线模式。

单击此按钮可打开Maven选项跳过测试模式 ，并省略运行单元测试。

单击此按钮可在UML弹出框架中显示当前Maven项目的依赖项。

单击此按钮可折叠选定Maven项目下的所有节点。

单击此按钮可在Maven Integration对话框中配置当前Maven项目的设置。

单击此按钮可显示节目选项的菜单：

  • 组模块 ：选择此选项可按目录对节点进行分组。

  • 显示忽略的项目 ：选择此选项后，忽略的节点将显示在工具窗口中，并带有删除线。否则，隐藏被忽略的节点。

  • 仅显示基本阶段 ：选择此选项时，IntelliJ IDEA仅显示基本构建阶段;否则，显示完整的阶段列表。

  • 始终显示ArtifactId ：选择此选项可显示Maven项目的pom.xml中指定的artifactId。

  • 显示版本 ：选择此选项后，IntelliJ IDEA将显示pom.xml中指定的Maven项目的版本。

  • 显示工具栏 - 选择此选项可显示Maven项目的工具栏。

  • 固定，停靠，浮动，窗口，分割模式

  • 组选项卡 - 如果工具窗口中有多个视图，则取消选择此选项可在单独的选项卡上查看视图。

  • 移至 - 选择此选项可将Maven项目工具窗口移至顶部，左侧或右侧。

  • 调整大小 - 选择此选项可调整Maven项目工具窗口的大小。

生命周期阶段的上下文菜单命令

命令

描述

创建<project> [阶段]

选择此命令可为生命周期的选定阶段创建运行/调试配置 。

运行<project> [phase]

选择此命令以使用特定于阶段的运行/调试配置运行生命周期的选定阶段。

调试<project> [阶段]

选择此命令可使用特定于阶段的运行/调试配置调试生命周期的选定阶段。

Make之前执行，Make之后执行

选择这些命令可为生命周期的选定阶段设置相应的标志。这样做， 在Make之前和之后Make评论出现在节点的名称旁边。

在重建之前执行，在重建之后执行

选择这些命令可为生命周期的选定阶段设置相应的标志。在这种情况下， 重建前和重建后的注释会出现在节点名称旁边。

在Run / Debug之前执行

选择此命令以指定运行/调试配置，在此之前应执行生命周期的选定阶段。这样做， 之前运行注释出现在节点名称旁边。

分配快捷方式

选择此命令可将选定的相位与键盘快捷键相关联。这样，带有快捷方式的注释将出现在节点名称旁边。

Maven项目的上下文菜单命令

本节仅介绍工具栏中不可用的上下文菜单命令。

命令

描述

忽略项目/ Unignore项目

选择忽略项目命令以忽略构建中的项目，或者相反，包括构建先前忽略的项目。
忽略的项目不会导入IntelliJ IDEA。

删除项目

选择此命令可从Maven结构中删除选定的Maven模块。因此，您可以选择从IntelliJ IDEA项目中删除相应的Maven模块。

创建/打开'settings.xml'
创建/打开'profiles.xml'

选择此命令可以创建“settings.xml”或“profile.xml”，或者打开此类文件（如果已创建）。

显示有效的POM

选择此命令可将有效POM生成为此构建的XML，并将活动配置文件和超级POM考虑在内。有效的POM显示以下信息：

  • 默认项目源文件夹结构

  • 输出目录

  • 需要插件

  • 资料库

  • Maven将在执行所需目标时使用的报告目录

跳转到源头

在编辑器中打开所选Maven项目的pom.xml文件。
