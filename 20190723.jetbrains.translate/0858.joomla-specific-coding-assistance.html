配置Joomla！支持

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

IntelliJ IDEA检查是否为Joomla正确配置了开发环境！发展。

在事件日志工具窗口中报告任何检测到的不一致，并作为弹出窗口针对每个差异，IntelliJ IDEA建议修复。要应用建议，请单击报告的事件旁边的链接。

改变Joomla！设置

启用或禁用Joomla！积分

  • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |框架 。在打开的Frameworks页面上，切换Enable Joomla！集成复选框以激活或停用Joomla！在当前的IntelliJ IDEA项目中。
    要使用另一个Joomla！安装，键入相关安装文件夹的路径。

设置代码样式

  • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 编辑器”|代码风格| PHP页面并根据需要更新设置。

根据Joomla编码标准检查代码

使用IntelliJ IDEA，您可以使用PHP Code Sniffer工具，它可以检测编码标准问题，并结合Joomla Coding Standards ，它为PHP Code Sniffer提供了一组Joomla特定的标准。这将确保您的代码干净，一致并且没有一些常见错误。

首先，使用安装和配置PHP Code Sniffer中描述的任何技术安装PHP Code Sniffer 。可能最简单的方法是使用Composer安装它。

安装PHP Code Sniffer

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择squizlabs/php_codesniffer包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

接下来，安装Joomla Coding Standards ，它将为PHP Code Sniffer提供Joomla特定的标准。

安装Joomla编码标准

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择joomla/coding-standards软件包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

将安装Joomla Coding Standards软件包，并将自动选择相应的Joomla标准进行PHP Code Sniffer验证检查。如有必要，您可以在编辑器|上进一步自定义检查“ 设置/首选项”对话框的“ 检查”页面（ Ctrl + Alt + S ）。有关详细信息，请参阅将PHP代码嗅探器配置为IntelliJ IDEA检查 。

要一次安装这两个软件包，请将它们添加到composer.json的require-dev部分，然后按Composer中的说明运行Install 。
