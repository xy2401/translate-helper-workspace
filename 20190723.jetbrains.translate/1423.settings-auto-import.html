  • 代码完成时：如果选中此复选框，则当您完成在另一个项目文件中导出的符号时，IntelliJ IDEA会自动在TypeScript代码中插入import语句。

  • 使用导入弹出窗口：如果选中此复选框，则当您将光标放在可导入的未解析符号时，IntelliJ IDEA会显示带有导入建议的工具提示。清除该复选框后，只能按Alt + Enter调用导入生成。

  • 动态明确的导入：选中此复选框后，IntelliJ IDEA会在您键入代码时添加import语句，或者如果只有一个源从中导入符号，则粘贴带有尚未导入的符号的片段。

PHP

此表仅在下载并安装PHP插件时可用。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

在粘贴上插入导入

选择IntelliJ IDEA如何处理粘贴代码块中缺少的类和方法的引用：

  • All ：将为粘贴的代码块中找到的所有缺失类和方法自动添加import语句。

  • 问 ：IntelliJ IDEA将提示您选择要导入的类和方法。

    如果已通过别名在目标代码中引用了粘贴的类，则IntelliJ IDEA将提示您重用此别名，而不是创建新的import语句。

  • 无 ：不会添加任何导入语句，也不会询问您是否有未解析的引用。

请注意，只有在代码中正确解析复制的元素（即未通过未定义的类或未定义的方法检查突出显示）并且项目索引已完成时，才能在粘贴时添加导入。

在文件范围中启用自动导入

选中此复选框可让IntelliJ IDEA在您输入不属于任何特定命名空间的类或文件时自动导入PHP命名空间，添加use语句以及动态完成短类名 。默认情况下，清除该复选框。

在命名空间范围内启用自动导入

选中此复选框可让IntelliJ IDEA在您键入属于某个命名空间的类或文件时自动导入PHP命名空间，添加use语句并动态完成短类名 。默认情况下选中该复选框。

处理来自全球空间的符号

在此区域中，分别为类，函数和常量从全局命名空间配置自动导入。

  • 首选FQN ：如果选中，IntelliJ IDEA会自动从全局命名空间插入符号的完全限定名称，前缀为反斜杠，例如：

    命名空间 A ; $ myException = new \ Exception （）; $ a = \ strlen （ “测试” ）; echo \ PHP_EOL ;

  • 首选导入 ：如果选择此选项，IntelliJ IDEA会自动为全局命名空间中的符号添加use语句（如果这不会导致冲突），例如：

    命名空间 A ; 使用 例外 ; 使用 const PHP_EOL ; 使用 函数 strlen ; $ myException = new Exception （）; $ a = strlen （ “测试” ）; echo PHP_EOL ;

  • 首选后备 ：如果选中，IntelliJ IDEA既不会插入函数的完全限定名称，也不会插入常量，也不会通过use语句导入它们。该回退全局函数或常量是优选在这种情况下，例如：

    命名空间 A ; 使用 例外 ; $ myException = new Exception （）; $ a = strlen （ “测试” ）; echo PHP_EOL ;
