IntelliJ IDEA与Vagrant集成，可帮助您创建由VagrantFile配置文件定义的可重现的开发环境。您可以创建和删除虚拟框 ， 初始化Vagrant框 ，通过从Vagrant配置文件中读取设置来配置远程解释器，并在不离开IDE的情况下执行其他Vagrant命令。执行vagrant up命令（Multiple Vagrant配置）时，您还可以在多个虚拟框中进行选择。

在IntelliJ IDEA的上下文中，以下术语用于表示Vagrant特定的概念：

  • 在当前文档中的IntelliJ IDEA的用户界面或流浪碱箱 A 流浪盒是一个盒子中的本机流浪术语。它表示纯图像，骨架，在其基础上，在您的计算机上自定义，配置和部署特定环境。

  • 实例是虚拟机 。换句话说，它是个性化，配置和颠沛流离的底盒的基础上，并按照部署你的机器上一个特定的环境Vagrantfile 。在其他类似的产品和文档中， 实例可以称为虚拟机 。

  • 实例文件夹是初始化后存储相关Vagrantfile的文件夹，以及IntelliJ IDEA将查找的文件夹。默认情况下，它是项目根文件夹。

准备与Vagrant合作

在开始之前，请确保完成以下步骤和检查：

  1. 如管理插件中所述，在插件页面上安装并启用Vagrant存储库插件。

  2. 确保在计算机上安装了Vagrant和Oracle的VirtualBox ，并将其可执行文件的父文件夹添加到系统PATH变量中。如果您遵循标准安装过程，则会自动添加这些文件夹。

  3. 确保将以下可执行文件的父文件夹添加到系统PATH变量中：

    • 您的Vagrant安装中的vagrant.bat或vagrant 。这应该由Vagrant安装程序自动完成。

    • Oracle的VirtualBox安装中的VBoxManage.exe或VBoxManage 。

在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，验证或修改Vagrant设置：

  1. 单击工具下的Vagrant 。将打开Vagrant页面页面。

  2. 指定Vagrant可执行文件。由于可执行文件的父文件夹已添加到系统PATH变量中，因此只需键入可执行文件的名称。

  3. 在“ 实例文件夹”字段中，指定初始化和存储Vagrantfile的目录的完全限定路径。Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定共享哪些文件夹以及应安装哪些第三方软件。根据Vagrantfile您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行配置 ，并部署在您的计算机上。通过vagrant init命令创建Vagrantfile 。

    通过vagrant up命令或通过Tools |调用实例 （ 虚拟机 ）的创建时流浪汉|向上菜单选项，IntelliJ IDEA在Instance文件夹字段中指定的目录中查找Vagrantfile 文件 。有关更多信息，请参阅http://docs.vagrantup.com/v2/vagrantfile/ 。

    您可以在任何目录中创建Vagrantfile并将其指定为实例文件夹 。如果该字段为空，IntelliJ IDEA会将项目根视为实例文件夹，并在其中查找Vagrantfile 。

  4. 在Vagrant Boxes区域中，配置IntelliJ IDEA 中可用的预定义Vagrant基本框的列表。每个项目都呈现一个Vagrant基础框 ，Vagrant在其上配置并启动其实例 （ 虚拟机 ）。此列表的条目对应于命令vagrant box list的输出。

    • 要下载新的基本框 ，请单击“ 添加”按钮 。在打开的对话框中，指定访问基本框的URL地址以及在IntelliJ IDEA中引用它的名称。默认情况下，IntelliJ IDEA会建议lucid32框的URL。

      此命令对应于vagrant box add <name> <URL> 。因此，指定的基本框将下载到您的计算机。

    • 要删除基本框 ，请在列表中选择它，然后单击“ 删除”按钮 。基本框和嵌套文件将从磁盘中物理删除。此命令对应于vagrant box remove <name> 。

初始化Vagrant文件

Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定共享哪些文件夹以及应安装哪些第三方软件。根据Vagrantfile您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行配置 ，并部署在您的计算机上。通过vagrant init命令创建Vagrantfile 。

您可以在任何文件夹中初始化Vagrantfile ，请记住，应该在“设置”对话框的“ Vagrant”页面上将此文件夹指定为实例文件夹 。否则，IntelliJ IDEA将无法在创建实例 （ 虚拟机 ）期间找到相关的Vagrantfile 。

要初始化Vagrantfile ，请执行以下操作之一：

初始化Vagrantfile

  1. 要在项目根目录中创建Vagrantfile ，请选择“ 工具”|流浪汉|在主菜单上的Project Root上初始化，然后从打开的列表中选择目标项目根目录。init命令的输出显示在“ 运行”工具窗口中。

  2. 要在特定文件夹中创建Vagrantfile ，请打开嵌入式终端 （ Alt + F12 ），然后键入以下命令：

    cd <目录初始化> vagrant init <base box name> <base box url>中的Vagrantfile

创建Vagrantfile后，您可以在编辑器中预览它：它包含可用配置选项的详细列表。如果需要，请更改或添加配置设置。请注意，Vagrantfile启用了代码完成功能。

成功完成Vagrantfile初始化后，您就可以导入基本框 ，根据Vagrantfile配置和部署，从而创建自己的实例 （ 虚拟机 ）。

创建和启动实例（虚拟机）

创建实例 （ 虚拟机 ）意味着根据实例文件夹中的Vagrantfile导入和配置基本框 。

创建实例，请按照下列步骤操作：

  1. 执行以下操作之一：

    • 要在项目根目录中创建实例，请转到“ 工具”|流浪汉|起来 。

    • 要在特定文件夹中创建实例，请打开嵌入式终端 （ Alt + F12 ），然后键入以下命令：

      cd <instance folder> vagrant up

  2. 从建议列表中选择所需的虚拟机配置：

停止，挂起，恢复，重新加载和销毁实例（虚拟机）

重新加载实例

  • 如果您对Vagrantfile进行了一些更改并希望根据它们更新正在运行的虚拟机，请选择Tools |流浪汉|重新加载主菜单或在嵌入式终端中运行以下命令：

    流浪汉重装
    有关详细信息，请参阅http://docs.vagrantup.com/v2/cli/reload.html 。

暂停实例

  • 要在正在运行的虚拟机（ 来宾计算机 ）上临时停止操作系统并保存当前环境的确切状态，以便可以从此时开始恢复，请选择“ 工具”|“工具”。流浪汉|挂起主菜单或在嵌入式终端中运行以下命令：

    流浪者暂停
    有关详细信息，请参阅http://docs.vagrantup.com/v2/cli/suspend.html 。

恢复实例

  • 要在虚拟机（ 来宾计算机 ）上恢复以前挂起的操作系统并使其从暂停时保存的状态运行，请选择“ 工具”|“工具”。流浪汉|在主菜单上恢复或在嵌入式终端中运行以下命令：

    流浪的简历
    有关详细信息，请参阅http://docs.vagrantup.com/v2/cli/resume.html 。

关闭一个实例

  • 要关闭虚拟机（ 来宾计算机 ）上的操作系统，请选择“ 工具”|流浪汉|在主菜单上暂停或在嵌入式终端中运行以下命令：

    流浪汉停止
    有关详细信息，请参阅http://docs.vagrantup.com/v2/cli/halt.html 。

  • 要关闭虚拟机（ 来宾计算机 ）上的操作系统，请停止虚拟机本身，并在创建期间删除其上配置的资源（ vagrant up ），选择工具|流浪汉|在主菜单上销毁或在嵌入式终端中运行以下命令：

    流浪汉毁灭
    有关详细信息，请参阅http://docs.vagrantup.com/v2/cli/destroy.html 。
