IntelliJ IDEA 2019.1帮助

Java SE

以下基本知识旨在帮助您开始在IntelliJ IDEA中使用Java。

将源文件移动到子文件夹中

您不应将源（.java）文件保留在项目根目录中。如果是这种情况，请在项目根目录中创建一个子目录，并将所有源文件移动到该子目录中。

为.java文件启用编码帮助

只要您将.java文件标记为包含源代码的文件夹， Java的编码帮助就会自动打开。在“ 项目”工具窗口（“ 视图”|“工具窗口”|“项目” ）中，右键单击该文件夹，指向“将目录标记为”，然后选择“ 源根” 。

使Java API可供您的代码访问

在Java中编写时，通常会重用（即引用） Java API类。这些类在JDK中可用。因此，要使代码可以访问Java API，您应该下载然后指定您的JDK：

  1. 在编辑器中打开.java文件：在“ 项目”工具窗口中选择文件，然后按F4 。

    如果尚未指定JDK，则编辑区域上方将未定义Project SDK 。

  2. 单击Setup SDK 。

  3. 在打开的对话框中，选择JDK并单击“ 确定” ，或单击“ 配置” ，单击“确定” ，选择JDK ，然后选择安装了所需JDK的目录。

为包创建文件夹结构或指定包前缀

如果您的软件包还没有适当的文件夹结构（让我们假设您的软件包的名称是com.example.mypackage ），您有两个选项导致大致相同的结果：

  • 在源目录中创建相应的文件夹结构 （例如<your-source-dir>/com/example/mypackage ）：

    在编辑器中，在package语句中（例如package com.example.mypackage; ），将光标放在包名称中，按Alt + Enter并选择Move to package'com.example.mypackage' 。

  • 为源文件夹分配包前缀，而不实际创建文件夹结构：

    1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ），选择“ 模块”并选择模块。

    2. 在“ 源”选项卡的右侧部分中，单击源文件夹旁边的 。

    3. 在打开的对话框中，指定前缀（例如com.example.mypackage ）。

使JAR中的课程可供您的应用访问

比如，您有一个JAR ，其中包含您要在应用中重复使用的类。在这种情况下，您应该将JAR添加到模块的依赖项中。因此，JAR类可用于在代码中引用，并且在构建时它们包含在您的应用程序中。

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ），选择“ 模块”并选择模块。

  2. 选择Dependencies选项卡，单击并选择JAR或目录 。

  3. 在打开的对话框中，选择您的JAR文件。

编译.java文件

如果您按照本页前面的讨论完成了所有操作，那么编译.java文件或文件不会有问题。只需选择以下选项之一： Build |构建项目或构建|构建模块'module_name' 。

如果编译有问题，IntelliJ IDEA将通知您原因并提供解决问题的提示。

运行你的应用程序

如果编译成功 ，您可以运行您的应用程序：

  1. 在编辑器中使用main（）方法打开该类。

  2. 在左边距中，在main（）方法旁边，单击运行标记（绿色箭头）并选择Run'<ClassName> .main（）' 。

将您的应用程序打包在JAR中

要将应用程序打包到JAR中，您应该配置JAR工件然后构建它：

  1. 打开“ 项目结构”对话框（ Ctrl + Shift + Alt + S ）并选择“ 工件” 。

  2. 点击 ，选择JAR并从具有依赖项的模块中选择。

  3. 在打开的对话框中，选择具有main（）方法的类 （ Main Class字段）。

  4. 单击“ 项目结构”对话框中的“ 确定”保存工件设置。

  5. 选择Build |构建工件 ，选择工件并选择构建 。（如果只有一个工件，则可以在弹出Build Artifact菜单时按Enter键 。）
