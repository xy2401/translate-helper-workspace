Composer依赖管理器

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

IntelliJ IDEA与Composer依赖关系管理器集成，允许您将Packagist中的包声明为项目的依赖关系，并从IDE管理它们。

IntelliJ IDEA为常用的Composer命令提供专用的用户界面：

  • 使用Composer 创建新项目 ，

  • 在现有的IntelliJ IDEA项目中初始化Composer ，

  • 管理项目依赖项 ，

  • 执行其他任务 。

这些命令只能在当前项目中使用。

运行上述命令时，IntelliJ IDEA将通过Composer Log控制台显示其输出。

如果要在IntelliJ IDEA项目中使用所有Composer命令 ，可以将Composer配置和运行为外部命令行工具 。

下载并安装Composer

您可以手动下载和安装Composer作为系统范围的独立工具。为此，请参阅Composer官方文档 。

或者，当您创建新的Composer项目 ， 初始化Composer或调用现有项目中的其他Composer命令时，IntelliJ IDEA将提示您自动下载Composer。在这种情况下，Composer通过配置的本地PHP解释器运行。

创建一个新的Composer项目

创建新的Composer项目时，IntelliJ IDEA会根据您选择的Composer包创建项目，解析依赖项并将其安装在项目的供应商目录下。

创建新的Composer项目

  1. 选择文件|新的|单击项目或单击欢迎屏幕上的创建新项目 。

    “ 新建项目”对话框打开：

  2. 在对话框中，指定项目的参数：

    • 选择项目类型和位置：

      • 在左侧窗格中，单击列表中的PHP ，然后在右侧窗格中选择Composer Project ，然后单击“ 下一步” 。

      • 在向导的第二页上，指定项目名称以及将在其中创建项目的文件夹。

    • 选择Composer命令的执行方式：

      • 要通过全局安装的composer可执行文件运行命令，请选择“composer”可执行文件并在字段中指定其位置。

      • 要通过本地安装的composer.phar存档运行命令，请选择composer.phar 。然后，在字段中指定现有composer.phar文件的位置，或从getcomposer.org复选框中选择Download composer.phar以下载该文件的新实例。composer.phar文件将保存在“ 位置”文本框中指定的项目根文件夹下。

        然后，从PHP解释器列表中选择一个配置的本地PHP解释器 。有关详细信息，请参阅配置本地PHP解释程序。

      （可选）在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，一个包被添加到composer.json文件的require-dev部分，而不是默认的require部分。

    • 在“ 包”区域中，从列表中选择包，然后从“ 要安装的版本”列表中选择相关版本。要搜索包，请使用“ 筛选包”搜索字段。

  3. 单击完成 。将使用所选包调用create-project Composer命令。因此，将创建Composer项目，其配置和结构取决于所选的包。之后，创建的IntelliJ IDEA项目将打开。

在现有项目中初始化Composer

在项目中初始化Composer时，会创建composer.json文件。此文件列出项目依赖项和其他元数据。

在现有项目中初始化Composer

  1. 从主菜单中，选择“ 工具”|作曲家| Init Composer 。

    “ Composer设置”对话框将打开：

  2. 选择Composer命令的执行方式：

    • 要通过全局安装的composer可执行文件运行命令，请选择“composer”可执行文件并在字段中指定其位置。

    • 要通过本地安装的composer.phar存档运行命令，请选择composer.phar 。然后，在字段中指定现有composer.phar文件的位置，或从getcomposer.org复选框中选择Download composer.phar以下载该文件的新实例。composer.phar文件将保存在“ 位置”文本框中指定的项目根文件夹下。

      然后，从PHP解释器列表中选择一个配置的本地PHP解释器 。有关详细信息，请参阅配置本地PHP解释程序。

  3. 单击“ 确定”关闭“ Composer设置”对话框。IntelliJ IDEA创建composer.json文件的存根，并在编辑器中打开它：

    完成代码或接受生成的值。有关使用composer.json文件的详细信息，请参阅Composer.json：Project Setup 。

您可以在一个IntelliJ IDEA项目中包含多个composer.json文件。对于每个composer.json ，可以从编辑器或项目视图中的上下文菜单调用操作，也可以使用编辑器面板顶部的快捷方式链接调用操作。

使用composer.json编辑器面板顶部的快捷方式链接快速安装或更新项目依赖项，打开Composer Log并访问其他命令 。

在IntelliJ IDEA中打开现有的Composer项目

当您打开现有的Composer项目时，IntelliJ IDEA使用composer.json文件中包含的信息自动应用某些配置选项，例如项目的内容根和PHP语言级别 。

  1. 单击欢迎屏幕上的打开或选择文件|在主菜单上打开 ，然后选择存储Composer项目的文件夹。

    IntelliJ IDEA将显示通知：

  2. 单击“ 初始化Composer设置” 。在打开的Composer对话框中，指定composer.json文件的位置。如果IntelliJ IDEA检测到现有文件，则会自动填充composer.json路径字段：

  3. 配置项目设置：

    • 默认情况下， vendor / * / *目录下的所有软件包都从项目中排除，并添加为写保护库。要保护它们不被编辑，请选中“ 将包添加为库”复选框。否则，如果要编辑vendor / * / *下的Composer包，请清除“ 将包添加为库”复选框。

    • 确保选中“ 使用composer.json同步IDE设置”复选框以自动检测PHP语言级别，并根据composer.json中的配置配置项目源和测试根。

      IntelliJ IDEA知道在composer.json的autoload和autoload-dev部分中声明的PSR-0 / PSR-4源根及其名称空间前缀。IntelliJ IDEA还根据require部分中的php设置检测PHP语言级别。

      因为composer.json包含有关项目配置的最新信息，所以此自动同步可确保Source和Test文件夹与项目结构完全匹配，并自动设置正确的PHP语言级别。

      要了解有关PSR和自动加载的更多信息，请参阅Composer官方网站 。有关同步设置的示例和详细信息，请参阅PhpStorm博客文章 。

    单击“ 确定”以应用指定的项目设置并关闭对话框。

  4. 设置Composer命令执行：

    • 执行以下操作之一：

      • 打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），然后导航到“ 语言和框架”| PHP |作曲家 。

      • 如果项目中有一个composer.json文件，请选择“ 工具”|作曲家|在主菜单上安装以打开“ Composer设置”对话框。

    • 选择Composer命令的执行方式：

      • 要通过全局安装的composer可执行文件运行命令，请选择“composer”可执行文件并在字段中指定其位置。

      • 要通过本地安装的composer.phar存档运行命令，请选择composer.phar 。然后，在字段中指定现有composer.phar文件的位置，或从getcomposer.org复选框中选择Download composer.phar以下载该文件的新实例。composer.phar文件将保存在“ 位置”文本框中指定的项目根文件夹下。

        然后，从PHP解释器列表中选择一个配置的本地PHP解释器 。有关详细信息，请参阅配置本地PHP解释程序。

  5. 安装项目依赖项：

    • 如果项目中有一个composer.json文件，请选择“ 工具”|作曲家|在主菜单上安装 。

    • 否则，如果您有几个composer.json文件，请选择Composer |从相关的上下文菜单安装 。另请参见在IntelliJ IDEA项目中指定默认的composer.json 。

    您还可以在代码编辑器中打开composer.json文件，然后单击编辑器面板顶部的“ 安装”快捷方式链接。

在IntelliJ IDEA项目中指定默认的composer.json

  4. 确保选中“ 使用composer.json同步IDE设置”复选框以自动检测PHP语言级别，并根据composer.json中的配置配置项目源和测试根。

    要了解有关PSR和自动加载的更多信息，请参阅Composer官方网站 。有关同步设置的示例和详细信息，请参阅PhpStorm博客文章 。

安装依赖项

您可以单独安装特定软件包，IntelliJ IDEA将自动更新您的composer.json 。或者，您可以在composer.json的require和require-dev部分列出所需的包，然后一次性安装它们。

如果composer.lock文件存在于当前目录中，则将安装其中列出的确切包版本。否则，如果缺少composer.lock文件，Composer将在安装软件包后创建它，从而将项目锁定到其特定版本。

了解有关在Composer官方网站上添加依赖项的更多信息。

安装特定包

  1. 执行以下操作之一：

    • 从composer.json上下文菜单中，选择Composer |管理依赖关系 。

    • 要从默认的composer.json安装包，请选择Tools |作曲家|在主菜单上管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择所需的包。要查找包，请使用搜索字段。该列表显示了所有可用的包;已安装的软件包标有复选标记。

    从要安装的版本列表中选择相关版本。

  3. （可选）自定义程序包安装。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，一个依赖项被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击“ 安装”开始安装。要退出对话框，请单击“ 关闭” 。

安装软件包后，IntelliJ IDEA会在供应商下创建一个新的子文件夹，并将新软件包存储在此子文件夹中。相应的包记录将添加到composer.json文件的require或require-dev部分以及composer.lock文件中。

一次安装所有包

  • 请执行以下任何操作：

    • 从composer.json上下文菜单中，选择Composer |安装

    • 在编辑器中打开composer.json ，然后单击编辑器面板顶部的“ 安装”快捷方式链接。

    • 要安装默认composer.json中列出的所有软件包，请选择Tools |作曲家|在主菜单上安装 。

IntelliJ IDEA安装composer.json文件的require和require-dev部分中列出的所有软件包。

更新依赖项

您可以单独将特定软件包更新到其最新版本，IntelliJ IDEA将自动更新您的composer.json 。或者，您可以一次更新composer.json文件的require和require-dev部分中列出的所有包。

更新特定包

  1. 执行以下操作之一：

    • 从composer.json上下文菜单中，选择Composer |管理依赖关系 。

    • 要从默认的composer.json更新包，请选择“ 工具”|作曲家|在主菜单上管理依赖项 。

  2. 从标有tick的软件包中的Available Packages列表中选择所需的软件包，然后单击Update 。

要一次更新所有包

    • 从上下文菜单composer.json上下文菜单中，选择Composer |更新 。

    • 在编辑器中打开composer.json ，然后单击编辑器面板顶部的“ 更新”快捷方式链接。

    • 要更新默认composer.json中列出的所有包，请选择“ 工具”|作曲家|在主菜单上安装 。

IntelliJ IDEA更新composer.json文件的require和require-dev部分中列出的所有包。

卸载依赖项

卸载包

  1. 请执行以下任何操作：

验证

运行此命令以对composer.json文件执行多个有效性检查，在将composer.json提交到VCS之前，建议使用该文件。验证结果显示在Composer Log控制台中。

等效的Composer CLI命令是validate 。

在执行命令之前，在“ 验证设置”对话框中选择所需的选项：

  • 允许未绑定的版本约束 ：如果选中，则允许不带上限的版本约束（例如* ， >3.1 ）。请注意，这将允许更新任何未来版本的依赖项，这可能会导致依赖项意外更改时出现向后兼容性问题。

    Composer CLI命令的等效选项是--no-check-all 。

  • 检查composer.lock达最新 ：如果选择，在composer.lock列出的版本约束将针对composer.json列出的检查。如果版本约束不匹配，建议通过更新包来解决此问题。将软件包更新为所需版本后，将同步composer.lock和composer.json文件。

    Composer CLI命令的等效选项是--no-check-lock 。

  • 检查包是否为Packagist-ready ：如果选中，将根据Packagist发布要求检查包的composer.json 。通常，对于要打包Packagist的包，必须指定其composer.json文件中的name ， description和require部分。有关发布包的详细信息，请参阅Packagist文档 。

    Composer CLI命令的等效选项是--no-check-publish 。

  • 验证包依赖关系 ：如果选中，则还将验证当前包的所有已安装依赖关系的composer.json文件。

    Composer CLI命令的等效选项是--with-dependencies 。

转储自动加载器

运行此命令以生成自动加载的类列表。如果列表已存在，则将重新生成。这在您在项目中引入新类并需要将其包含在自动加载类列表中的情况下非常有用。有关使用Composer自动加载类的更多详细信息，请参阅Composer文档 。

等效的Composer CLI命令是dump-autoload 。

在执行命令之前，请在“ Autoloader Generation Settings”对话框中选择所需的选项：

  • 跳过脚本执行 ：如果选中，则不执行composer.json文件的scripts部分中定义的scripts 。

    Composer CLI命令的等效选项是--no-scripts 。

  • 优化自动加载器（将PSR-0/4自动加载器转换为类图） ：如果选中，Composer将扫描给定的PSR-0/4自动加载目录/文件以获取php和inc文件，并生成单个autoload_classmap键值数组以加快速度自动加载。请注意，如果类的数量很大，则运行此命令可能非常耗时。

    Composer CLI命令的等效选项是--optimize 。

  • 仅来自类映射的自动加载类 ：如果选中，则只会自动加载类映射中列出的类。Composer不会为类映射中未列出的类搜索回退PSR-4目录。由于这自然需要现有的类映射，因此Composer还将执行Optimize自动加载器（将PSR-0/4自动加载器转换为类映射）命令以生成一个，这可能是一个耗时的过程。

    Composer CLI命令的等效选项是--classmap-authoritative 。

  • 使用APCu缓存已发现/未找到的类 ：如果选中，将启用通过APCu进行缓存。当在类映射中找到（或未找到）类时，相应的记录将缓存在APCu中，以便可以在下一个请求中快速检索它。

    Composer CLI命令的等效选项是--apcu 。

  • 禁用autoload-dev依赖项 ：如果选中， autoload-dev部分中列出的自动加载的命名空间和文件不包含在自动加载的类列表中。

    Composer CLI命令的等效选项是--no-dev 。

从命令行运行Composer

命令行模式为您提供全系列的Composer命令。要在此模式下使用Composer，您需要将其配置为外部命令行工具。

在开始之前，请确保按照管理插件中的说明安装命令行工具支持存储库插件。

将Composer与IntelliJ IDEA集成为外部命令行工具

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|命令行工具支持 。

    在打开的命令行工具支持页面上，单击 。

  2. 在打开的“ 命令行工具”对话框中，从列表中选择“ Composer ”。使用“ 可见性”选项按钮，指定该工具是全局可用的，即在任何IntelliJ IDEA项目中，还是仅在当前项目中可用：

    单击“ 确定”以应用更改。

  3. 在打开的“ 命令行工具支持：编写器”对话框中，通过选择以下选项之一指定启动Composer的方式： composer.phar或php脚本或composer可执行文件 。根据您的选择，指定PHP安装文件夹和composer.phar或composer可执行文件的路径。

    IntelliJ IDEA为Composer命令解析指定的.phar存档或可执行文件的内容。完成文件分析后，IntelliJ IDEA将返回到“ 命令行工具支持”页面，其中指定的文件将添加到IntelliJ IDEA中可用的命令行工具列表中。

    单击“ 确定”以应用更改。Composer工具将添加到“ 命令行工具支持”页面上的工具列表中。

  4. 选中添加工具旁边的“ 启用”复选框，指定要在调用中使用的别名，根据需要自定义命令集，并选择显示“运行”命令的“ 输入”窗格的位置。有关详细信息，请参阅自定义工具和输入窗格位置 。

您可以配置多个Composer实例，并通过在Composer初始化期间指定相关项来在它们之间切换。

如果选择composer可执行文件 ，则IntelliJ IDEA不提供编码帮助，因为没有指定PHP解释器，所以无法执行脚本。

现在添加了Composer命令行工具，您可以使用它运行命令。打开“输入”窗格（“ 工具”|“运行命令” ）并键入<alias> （默认为c ），然后按Ctrl + Space以调用完成。命令执行结果显示在“ 输出”选项卡中，并带有命令的名称。了解更多信息如何运行命令？ 。

如果为Composer指定了自定义别名而不是默认c ，请键入<tool alias> init 。
