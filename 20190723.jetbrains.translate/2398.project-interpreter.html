项目口译员

对于在“ 项目”窗格中选择的项目 ，从列表中选择Python解释器。

列表中的解释器使用按钮，如查看可用Python解释器列表一节中所述。

当PyCharm停止支持任何过时的Python版本时，相应的项目解释器被标记为不受支持。

路径映射

此字段仅适用于远程口译员 ！

此字段显示所需本地路径与相应远程路径之间的只读映射。PyCharm允许通过单击配置路径在田野旁边。在打开的对话框中 ，单击要么用于根据需要更改路径映射的按钮。

单击此按钮配置项目解释器：

从列表中选择要添加的一个：

  • 添加 ：选择此选项可添加本地驻留的Python解释器 。这还包括虚拟环境和Conda环境 。此外，选择此选项可添加远程驻留的Python解释器。

  • 全部显示 ：选择此选项可打开当前项目可用的所有解释器列表。

包

此列显示驻留在可访问远程存储库中的软件包列表。对于每个包，您可以查看其名称和存储库。其他信息显示在工具提示中。

最新

此列显示软件包的最新可用版本。如果版本已过时且包可以升级， 标记出现在此列中。

工具提示和快捷方式

描述

安装
Alt +插入

单击此按钮可从可访问的远程存储库安装新软件包。

卸载
Alt + Delete键

单击此按钮可卸载选定的本地程序包。

升级

单击此按钮可安装最新的软件包版本。此按钮仅适用于过时的软件包版本。

显示早期版本

如果要将最新可用版本的范围扩展到任何预发行版本（例如测试版或候选发布版 ），请单击此按钮。

使用Conda Package Manager

此按钮仅适用于Conda口译员！
如果按下此按钮，则使用Conda包管理器。如果释放此按钮，则使用pip 。

对于基于Docker和基于 Docker-Compose的解释器，此工具栏上的按钮被禁用。

所有软件包都应该已经安装在Docker镜像中。如果缺少某些软件包，则必须创建一个新的Docker镜像，如页面快速入门指南：Compose和Django所述 。
