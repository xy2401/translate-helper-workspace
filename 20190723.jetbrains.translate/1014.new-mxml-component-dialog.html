新的MXML组件对话框

档案|新的| MXML组件

使用此对话框指定要创建的MXML组件的设置。如果指定尚不存在的包，也可以创建新包。

项目

描述

名称

指定组件的名称。这是将创建的.mxml文件的名称。

当光标位于此字段中时，您可以使用向上和向下箭头键浏览模板列表的内容。

包

指定应在其中创建组件的包的完全限定名称。为此，您可以：

  • 从列表中选择包。

  • 点击 （ Shift + Enter ）并在打开的“选择目标包”对话框中选择包 。

  • 输入字段。要启用包名称，请按Ctrl + Space 。

该字段可以留空。在这种情况下，组件将在默认根包中创建（通常对应于src文件夹）。

如果键入尚不存在的包的名称（此例中的名称显示为红色），则将创建相应的包。

您可以一次创建多个包。例如，如果您键入myPackage.mySubpackage并且当前不存在这些包， mySubpackage创建这些包（ myPackage和mySubpackage ）。

模板

选择要使用的文件模板 。

最初，根据与活动构建配置关联的Flex SDK版本（Flex SDK 4的MXML 4组件或Flex SDK 3的MXML 3组件） ，只有一个选项不同。

  • 类。 选择此模板以创建不扩展另一个类或实现接口的类。

  • 有超级的班级。 选择此模板以创建扩展另一个类和/或实现一个或多个接口的类。

  • 接口。 选择此模板以创建界面。

要编辑现有文件模板或创建新模板，请单击 （ Shift + Enter ）。“ 文件模板”对话框将打开。

如果所选模板包含未定义值的变量，则对话框中将显示“ 下一步”按钮，而不是“ 创建”按钮。在这种情况下，如果单击“ 下一步” ，将打开另一个对话框，您可以在其中设置变量值。（如果使用包含自定义变量的自定义模板 ，则可能会发生这种情况。）

另请参阅ActionScript和Flex的预定义文件模板变量以及 为MXML组件创建自定义文件模板的示例 。

父组件

为要创建的组件指定父组件。父组件定义新MXML组件的根标签，例如<s:Application> ， <s:Module> ， <s:ComboBox> 。

要指定父组件，您可以：

  • 点击 （ Shift + Enter ）并在打开的“选择超类”对话框中选择组件。

  • 输入字段。要启用组件名称，请按Ctrl + Space 。
