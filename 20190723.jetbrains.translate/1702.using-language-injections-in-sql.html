在SQL中使用语言注入

以下内容仅在安装并启用IntelliLang，Database Tools和SQL插件时有效！

此功能仅在Ultimate版本中受支持。

对于SQL中的语言注入，IntelliJ IDEA提供以下附加功能（有关常规信息，请参阅语言注入 ）：

使用自动注入XML和JSON

对于定义为XML和JSON类型的值，将自动注入相应的语言。

例

  1. 创建一个SQL文件并在编辑器中打开它。

  2. 将PostgreSQL指定为该文件的SQL方言。

  3. 将以下内容复制到SQL文件中：

    CREATE TABLE test （ my_xml XML DEFAULT '' ）;

  4. 将光标放在引号之间。

  5. 检查灯泡菜单（ Alt + Enter ）：那里有Edit XML Fragment命令，这意味着已经自动注入了XML。

对用户定义的数据类型使用基于模式的注入

您可以创建模式 - 例如，用户定义的数据类型 - 并将这些模式与语言相关联。因此，当IntelliJ IDEA遇到与模式匹配的数据类型时，将注入为该模式指定的语言。

在下面的示例中，我们将为以DATA结尾的数据类型创建模式，并将该模式与XML相关联。

例

  1. 在您的SQL文件中，将XML替换为MYDATA 。

  2. 将光标放在引号之间。

  3. 按Alt + Enter 。请注意，菜单中不再有Edit XML Fragment命令。

    选择“ 按类型注入” ，然后选择“ XML（XML文件）” 。

  4. 在打开的对话框中，在“ 类型模式”字段中，指定(?一世）。*DATA 。（使用正则表达式指定类型模式。在这个例子中， (?i)打开不区分大小写的模式; .*代表任意数量的任何字符。）

  5. 再次检查灯泡菜单（ Alt + Enter ）。编辑XML片段命令已变为可用，这意味着已为MYDATA类型的值注入了XML。

  6. 要删除刚刚创建的模式（如果不需要），请打开“ 设置/首选项”对话框（例如Ctrl + Alt + S ），转到编辑器|语言注入页面，查找和删除模式。
