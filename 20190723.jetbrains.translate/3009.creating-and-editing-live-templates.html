创建实时模板

以下示例过程说明了如何使用当前日期，用户名和扩展到输入字段的两个用户定义变量为TODO注释创建模板：

创建一个实时模板

  1. 导航到文件|设置/首选项|实时模板 。

  2. 选择要在其中创建新实时模板的模板组（例如， other ）。如果未选择模板组，则会将实时模板添加到用户组。

  3. 单击添加 在工具栏上。添加并选择新模板项。

  4. 指定以下值：

    领域

    描述

    示例值

    缩写

    指定要扩展模板的字符

    		去做

    描述

    指定可选描述以标识模板的用途

    插入TODO注释，其中包含TODO建议的当前日期，用户和两个输入字段以及与其相关的项目名称。

    模板文本

    使用变量指定模板的主体

    // TODO在$ DATE $ $ $ $上添加：考虑项目$ project $的$ todo $

  5. 单击底部的“ 定义 ”以选择适用的上下文（例如， JavaScript，如果它应该应用于JavaScript源文件）。

  6. 单击Edit Variables并为$USER$选择$DATE$和user()的date()函数。如有必要，使用更改列表中变量的顺序和 。此顺序确定光标在扩展模板中的变量之间跳转的顺序。例如，如果向下移动$todo$变量，则在$project$ input字段完成后，光标将跳转到其输入字段。

    单击“ 确定”关闭对话框。

  7. 单击“ 确定”以应用更改。

如果要在多个组中重复使用相同的模板，或者要基于另一个模板创建新模板，则可以复制现有模板。

复制现有模板

  1. 在“ 设置/首选项”对话框的“ 实时模板”页面上，选择要复制的模板。

  2. 单击复制 （ ）在工具栏上。新模板项目将添加到与原始模板相同的组中，并被选中。

  3. 指定模板的新缩写，可选描述（用于标识模板的用途），并在必要时修改模板主体。如果模板已定义变量 ，请单击“ 编辑变量”以配置变量。

  4. 单击“ 确定”以应用更改。

从代码片段创建新模板

  1. 在编辑器中，选择文本片段以从中创建实时模板。

  2. 在“ 工具”菜单上，单击“ 另存为实时模板” 。将打开“ 实时模板”页面，并创建并选择新模板。

  3. 指定模板的缩写，可选描述（用于标识模板的用途），以及修改模板主体。如果模板已定义变量 ，请单击“ 编辑变量”以配置变量。

  4. 单击“ 确定”以应用更改。
