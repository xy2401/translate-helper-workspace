在此页面上，在PyCharm中启用Vagrant支持，指定VagrantFile的位置，并处理用于创建虚拟框（实例）的Vagrant基本框列表。

在开始使用Vagrant之前，请确保：

  1. Vagrant已下载并安装。

  2. 在开始使用Vagrant之前，请确保已启用Vagrant插件。该插件与PyCharm捆绑在一起，默认情况下处于激活状态。如果未激活插件 ，请在“ 管理插件 ”中所述的“设置/首选项”对话框的“ 插件”页面上启用它。

项目

描述

流浪的可执行文件

指定可执行文件的完全限定地址：用于Windows的vagrant.bat ，用于Unix和macOS的vagrant 。手动键入路径，或单击“浏览”按钮并在“ 选择流浪者可执行文件”对话框中找到所需的文件。

实例文件夹

在此处指定目录的完全限定路径，其中执行任务vagrant init ，并初始化和存储VagrantFile 。

请注意，您可以在任何位置创建实例文件夹，例如，在项目根目录中。创建新的远程解释器时，默认情况下会将此项目根位置建议为实例文件夹。

Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定共享哪些文件夹以及应安装哪些第三方软件。根据Vagrantfile您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行配置 ，并部署在您的计算机上。通过vagrant init命令创建Vagrantfile 。

通过vagrant up命令或通过Tools |调用实例 （ 虚拟机 ）的创建时流浪汉|向上菜单选项，PyCharm在Instance文件夹字段中指定的目录中查找Vagrantfile 。有关更多信息，请参阅http://docs.vagrantup.com/v2/vagrantfile/ 。

您可以在任何目录中创建Vagrantfile并将其指定为实例文件夹 。如果该字段为空，PyCharm会将项目根视为实例文件夹，并在其中查找Vagrantfile 。

提供商

使用此字段指定vagrant up命令要使用的提供程序 。如果此字段留空，则使用默认提供程序。

环境变量

单击省略号按钮或按Shift + Enter以指定用于配置提供程序行为的shell变量。

框和插件选项卡

盒

此列表显示PyCharm 中可用的预定义Vagrant基本框 。

每个项目都呈现一个Vagrant基础框 ，Vagrant在其上配置并启动其实例 （ 虚拟机 ）。此列表的条目对应于命令vagrant box list的输出。
Alt +插入

单击此按钮可下载新的基本框。此命令对应于vagrant box add <name> <URL> 。默认情况下，PyCharm建议使用lucid32框的URL。有关详细信息，请参阅 。

Alt + Delete键

单击此按钮可删除选定的Vagrant基本框 。框和嵌套文件将从磁盘中物理删除。此命令对应于vagrant box remove <name>有关详细信息，请参阅 。

插件

使用此表可以查看和更改可用插件列表。

Alt +插入

单击此按钮可安装新的Vagrant插件。

Alt + Delete键

单击此按钮可删除所选插件。

单击此按钮可更新所选插件。

使用此按钮将许可证附加到所选插件。
