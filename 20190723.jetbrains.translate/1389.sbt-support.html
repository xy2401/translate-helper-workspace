IntelliJ IDEA 2019.1帮助

SBT

要创建新项目，请启动“ 新建项目”向导，然后按照向导中建议的步骤操作，例如：

  1. 选择Scala和sbt 。

  2. 指定项目的名称，位置，JDK以及sbt和Scala版本。（sbt和Scala版本会自动获取。）

导入sbt项目

  1. 单击“ 导入项目”或在欢迎屏幕上打开 。

  2. 在打开的对话框中，选择包含sbt项目的目录或只是build.sbt 。单击确定 。

  3. 按照“ 导入项目”向导中建议的步骤操作。
    您可以使用建议的默认设置，因为它们足以成功导入项目。
    我们建议您在使用代码生成或在sbt中修改构建过程的其他功能时启用使用sbt shell进行构建和导入（需要sbt 0.13.5+）选项。如果您的sbt项目不在IntelliJ IDEA项目根目录中，我们建议您跳过此选项。
    您还可以选择适当的选项，以便在项目中对模块进行分组。

确保sbt和Scala版本的兼容性

通常，您在团队中共享项目，需要使用特定版本的sbt。

您可以在项目的build.properties文件中覆盖sbt版本。

  1. 创建或导入您的sbt项目。

  2. 在“ 项目”工具窗口的源根目录中，找到build.properties文件并在编辑器中将其打开。

  3. 在编辑器中显式指定要在项目中使用的sbt的版本。
    sbt 。 version = xxx

    请注意，较新的sbt版本（1.0 +）将自动创建build.properties文件（如果它不存在）。

  4. 刷新您的项目。（点击在sbt projects工具窗口中。）

当您尝试导入包含旧版本sbt的sbt项目时，您可能会收到错误。我们建议您升级到与Scala版本2.12兼容的sbt 1.0及更高版本（需要Java 8）。

管理sbt项目

sbt项目结构

当您创建或导入sbt项目时，IntelliJ IDEA会生成以下sbt结构：

  • sbt项目（正确构建），它定义一个项目并包含build.sbt文件， src和目标目录，模块;任何与常规项目相关的事情。

  • sbt 构建项目，它在项目子目录中定义。它包含作为构建定义一部分的其他代码。

  • sbt projects工具窗口，其中包含可以执行的sbt任务，命令和设置。

有关sbt项目结构的更多信息，请参阅相关的sbt文档 。

当您使用sbt项目时，您使用build.sbt文件对项目进行主要更改，因为IntelliJ IDEA将sbt配置视为单一事实来源。

将库添加到sbt项目

您可以通过build.sbt文件添加sbt依赖项 ，也可以在.scala文件中使用import语句。

  1. 在编辑器中打开.scala文件。

  2. 指定要导入的库。

  3. 将光标放在未解析的包上，然后按Alt + Enter 。

  4. 从可用意图操作列表中，选择添加sbt依赖关系 。

  5. 按照打开的向导中建议的步骤操作，然后单击“ 完成” 。

  6. IntelliJ IDEA下载工件，将依赖项添加到build.sbt文件和sbt项目工具窗口。

  7. 一旦IntelliJ IDEA检测到build.sbt更改，就会出现建议刷新项目的通知。刷新您的项目。（点击在sbt projects工具窗口中。）
    或者，使用位于sbt设置中的自动导入选项。

使用sbt shell

sbt shell嵌入在sbt项目中，可在项目启动时使用。您可以使用sbt shell执行sbt命令和任务，以运行和调试项目。

  • 要启动sbt shell，请按Ctrl + Shift + S （对于Windows）或Cmd + Shift + S （对于Mac OS X）。或者，单击在位于屏幕底部的工具栏上。

  • 要使用sbt shell进行构建和导入过程，请选择sbt设置中的使用sbt shell进行构建和导入（需要sbt 0.13.5+）选项，并使用sbt shell部分执行Run a Scala应用程序中描述的步骤。请注意，sbt版本为0.13.16。建议使用+ / 1.0.3。+。

  • 要使用sbt shell进行调试，请参阅使用sbt shell部分进行调试 。

  • 要从sbt shell运行测试：

    1. 打开运行/调试配置（ 运行|编辑配置 ）。

    2. 创建测试配置并从可用设置中选择使用sbt选项。

您还可以通过在sbt shell中输入test/Only来运行test或test/Only命令。

运行sbt任务

  • 您可以通过在sbt项目工具窗口的sbt Tasks目录中选择所需的任务来运行sbt任务。

  • 您可以在sbt shell中手动输入任务（支持代码完成）并直接从那里运行它。

  • 您可以为任务创建运行配置。例如，您可以创建一个自定义任务，该任务不属于sbt projects工具窗口中的任务列表。

    1. 打开（ Shift + Alt + F10 ）运行配置。

    2. 指定运行配置设置，然后单击“ 确定” 。如果需要，可以在运行配置之前添加其他配置或要执行的任务。点击 “ 启动前”部分中的图标以及打开的列表中的图标选择您需要执行的操作。

  • 当您在无干扰模式下工作时（没有工具栏和工具窗口），您可以从Run Anything窗口运行SBT任务或命令。按两次Ctrl键将其打开，键入命令并按Enter键 。

    您可以查看Mastering快捷方式部分以获取更多信息。

IntelliJ IDEA在sbt shell窗口中显示结果。

您可以像执行sbt任务一样执行sbt 命令和设置 。

使用sbt设置

要访问sbt设置，请单击在sbt项目工具窗口中。
您可以将sbt设置用于以下显着操作：

  • 如果您希望每次更改build.sbt时sbt自动刷新项目，请选择“ 使用自动导入” 。

  • 要将正在运行的构建委派给sbt，请选择“ 使用sbt shell进行构建和导入” 。

  • 要通过sbt shell调试代码，请选择启用调试sbt shell选项以启用调试按钮（ ）在sbt shell工具窗口中。要启动调试会话，只需单击此按钮即可。有关调试的更多信息，请参阅使用sbt进行调试 。

  • 要更改项目中的.ivy缓存位置或设置其他sbt属性 ，请使用VM参数字段。

要查看最常见的sbt问题和解决方法，请参阅sbt疑难解答部分。
