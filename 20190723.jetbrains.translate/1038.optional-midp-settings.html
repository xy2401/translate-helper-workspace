可选的MIDP设置对话框

单击“ 从头开始新建项目向导”的 “ 移动SDK特定选项”页面上的“ 可选设置”按钮时，将打开该对话框。

使用此对话框指定通常不需要的其他选项。对话框的内容取决于使用的移动开发工具：

  • Java Wireless Toolkit（WTK） 。

  • DoJa 。

WTK

项目

描述

的MIDlet-说明

在此文本框中，提供MIDlet套件的说明。

MIDlet的图标

在此文本框中，指定.jar MIDlet套件演示文稿.png文件的位置。

的MIDlet-DATA-大小

在此文本框中，键入MIDlet所需的持久数据的最小字节数。

MIDlet的信息，网址

在此文本框中，指定URL以访问有关MIDlet套件和/或供应商的更多信息。

的MIDlet-Delete（删除），确认

在此文本框中，键入在提示用户确认删除MIDlet套件时要显示的消息。

MIDlet的安装，通知

在此文本框中，指定发送POST请求的URL以确认此MIDlet套件的成功安装。

用户定义的设置

在此区域中，编写与特定MIDlet相关的用户定义属性列表。对于每个属性，请指定键和值 。使用“ 添加”和“ 删除”按钮管理列表的内容。使用“上移”和“上移”按钮定义设置顺序。此顺序确定应用设置的优先级。

DoJa标准

项目

描述

AppVersion

在此文本框中，指定应用程序版本（最多10个字节）。

ConfigurationVer

在此文本框中，指定J2ME配置版本，例如CLDC-1.0 。

ProfileVer

在此文本框中，指定i-mode Java Application运行时环境概要文件的版本，例如DoJa-1.5oe 。

SPsize

在此文本框中，以字节为单位指定ScratchPad的大小。

AppParam

在此文本框中，指定主类的参数（最多255个字节）。

UseNetwork的

在此文本框中，为使用网络功能的应用程序指定http 。

TargetDevice

如果应用程序针对特定模型（最多128个字节），请在此处指定模型名称。不要为针对所有模型的应用程序设置此选项。

LaunchAt

要在特定时间自动启动应用程序，请在此文本框中指定所需的时间。

apptrace会

在i-Appli终止后，选择On值以使用System.out.println()或System.err.println()输出一些信息。

DrawArea

在此文本框中，指定应用程序绘图区域的大小，例如， 120x130 。

GetUtn

选择此选项后，应用程序会参考其SIM或UIM卡上的手机识别码和IC芯片信息。
