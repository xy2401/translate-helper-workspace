节点解释器

在此字段中，为当前项目指定默认的Node.js解释器。每次在创建运行/调试配置或配置Node.js相关工具（例如，Prettier或Yeoman）时从节点解释器列表中选择Project别名时，WebStorm将自动使用它。

从下拉列表或单击时打开的对话框中选择解释器 。如果选择node ，则使用系统Node.js版本 。

在这里，您可以选择或配置仅本地Node.js解释器，即计算机上安装的Node.js，或Windows Subsystem for Linux上的Node.js ，请参阅配置本地Node.js解释器和使用Node.js有关详细信息，请参阅Windows Subsystem for Linux 。

远程解释器在“ 配置Node.js远程解释器对话框”对话框中配置，该对话框可从“ 运行/调试配置：Node.js”访问 。有关详细信息，请参阅配置Remote Node.js解释程序。

版

此只读字段显示Node.js的当前版本。

Node.js的编码辅助

选中此复选框可将Node.js核心模块源配置为JavaScript库并将其与项目关联。因此，WebStorm为fs ， path ， http和编译到Node.js二进制文件中的其他核心模块提供代码完成，参考解析，验证和调试功能。

配置完成后，WebStorm将显示有关当前配置版本的信息。

可选
配置Node.js Core源被视为库的范围：

  1. 单击“ 管理范围” 。“ 使用范围”对话框打开

  2. 单击相关目录，对于每个目录，从列表中选择新配置的Node.js核心库。

包

Packages区域显示了计算机上当前安装的所有NPM相关软件包的列表。

  • 包：此只读字段显示包的名称，与在命令行模式下安装包时的名称完全相同。

  • 版本：此只读字段显示计算机上安装的软件包的版本。

  • 最新：此只读字段显示该软件包的最新发布版本。如果包不是最新的，则标记为 。

  • 点击安装新包。在打开的“ 可用包”对话框中，选择相关包。要全局安装软件包，请选中“ 选项”复选框，然后在“ 选项”字段中键入-g 。全局安装使程序包在WebStorm级别可用，因此可以在任何WebStorm项目中使用。准备好后单击Install Package 。

  • 点击删除选定的包。

  • 点击使用最新发布的版本替换所选软件包的当前版本。仅当所选项目不是最新时，才会启用该按钮。
