编译

选择应使用哪些可用的Flex编译器：
  • 内置编译器shell。 IntelliJ IDEA编译器shell，它使用Flex SDK编译器API。

    此编译器shell可以执行增量编译。作为多线程shell，它能够同时并行运行多个编译。

  • mxmlc的/ compc命令。 Flex SDK中提供的mxmlc / compc编译器。

    此编译器无法逐步编译。但是，它可以同时运行多个独立的编译过程，这显着提高了编译性能。

无论您使用哪种编译器选项，IntelliJ IDEA都会跟踪自上次编译以来没有任何变化的模块。因此，不会编译最新的SWF和SWC文件。

首选ActionScript Compiler 2.0以实现纯ActionScript构建配置

如果选中此复选框，则ActionScript Compiler 2.0（ASC 2.0）将用于纯ActionScript构建配置 。

使用最多<this_many>线程或进程进行并行编译

指定同时运行的最大编译线程数（对于内置编译器shell）或进程（对于mxml / compc编译器）。

编译器堆大小

指定启动编译器的进程可用的最大堆大小。

默认的512 Mb适用于大多数用途。但是，作为单进程编译器的内置编译器shell可能需要更多内存用于大型项目。

VM选项

如有必要，在IntelliJ IDEA启动编译器时键入要传递给VM的字符串。

如果您需要更多空间来键入，请单击在该字段旁边，可以访问文本输入区域较大的Flex Compiler VM选项对话框。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

    - Dfoo = $ { MY_ENV_VAR }
