调试

启动调试会话

  1. 要调试CoffeeScript ， TypeScript和Dart代码，您需要为其生成源映射 。这将设置原始代码和生成的JavaScript代码中的行之间的对应关系。如果未生成源映射，则无法正确识别和处理断点。

  2. 为要调试的应用程序定义运行/调试配置 。

  3. 创建一个断点 。

  4. 点击图标。阅读有关在启动调试器会话中调试会话的更多信息。

根据需要暂停或恢复调试会话。

在调试器会话期间， 逐步执行断点 ， 计算表达式 ，即时更改值以检查挂起的程序 ，以及设置监视 。

有关详细信息，请参阅调试CoffeeScript ， TypeScript以及运行和调试Dart Web应用程序 。

在您启动调试会话后， 标记“ 调试”工具窗口切换到的图标表示调试过程处于活动状态。

调试

为项目配置运行配置后 ，可以按Shift + F9以调试模式启动它。

在“ 调试”工具窗口中，您可以看到包含状态，变量和监视的框架和线程列表。选择框架时，您会看到与所选框架对应的变量。

断点

自定义断点设置

要自定义断点设置，请按Ctrl + Shift + F8 。要查看项目中的所有断点（使用其他设置），请单击“ 更多”或再次按相同的快捷键Ctrl + Shift + F8 。

不要暂停代码执行

使用操作断点来评估特定代码行的变量，而不会暂停代码执行。要创建操作断点，请在按住Shift的同时单击装订线。

创建一个临时断点

要创建仅停止一次的断点，请在按住Shift + Alt的同时单击左侧装订线。有关更多信息，请参阅断点部分。

禁用断点

要禁用断点，请在按Alt键的同时单击断点。有关详细信息，请参阅断点部分。

调试器会话

聪明的步入

有时会发生这样的情况，即你停留在一条线上并想要进入一个特定的方法而不是第一个被调用的方法。在这种情况下，通过按Shift + F7选择特定方法，使用Smart步进 。

有关详细信息，请参阅选择要进入的方法一节。

运行到光标

  1. 创建一个断点。

  2. 运行调试会话。要运行调试会话，请单击“ 运行应用程序”图标（ ）在装订区域中选择Debug <configuration_name> 。

要在光标位置停止代码执行而不添加其他断点，请单击“ 运行到光标”图标（ ）或按Alt + F9 。或者，您可以单击装订线区域中的行号。

图标在调试工具窗口的工具栏参考中描述。

评估表达

在调试模式下，您可以通过按Alt + F8来评估任何表达式 。

此工具提供代码完成，就像在编辑器中一样，因此输入任何表达式都非常容易：

有关详细信息，请参阅评估表达式一节。

设置

配置调试器设置

要更改调试器设置，请单击“ 文件”|设置 。在“ 设置”菜单中，单击“ 构建，执行，部署”调试器 。

有用的调试器快捷方式

行动

热键

简历程序F9
过来F8
踏入F7

停止

按Ctrl + F2
查看断点详细信息/所有断点按Ctrl + Shift + F8

在插入符调试代码

Shift + F9 （在主方法中），或Shift + Alt + F9
