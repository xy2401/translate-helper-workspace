为测试创建运行/调试配置

您可以使用运行/调试配置运行测试（测试用例，测试套件等），其方式与运行普通应用程序类似。

要创建必要的运行/调试配置，可以使用一般过程（请参阅创建和编辑运行/调试配置 ）或IntelliJ IDEA中提供的各种快捷方式。使用这些快捷方式（实现为上下文菜单命令），您可以为以下内容创建运行/调试配置：

  • 个别测试用例。

  • 目录或包中的所有测试。

  • 选定的测试类，可以位于相同或不同的目录，包或模块中。

    此选项仅适用于JUnit。

  • 个人测试方法。

即使不为它们创建运行/调试配置，也可以运行测试。为此，请在相应的上下文菜单中使用“ 运行” （ Ctrl + Shift + F10 ）或“ 调试”命令。

因此，将生成临时运行/调试配置，然后您可以保存。

要为测试创建运行/调试配置，请遵循以下常规步骤：

  1. 要开始创建运行/调试配置，请打开所关注项目的上下文菜单，然后选择相应的“ 创建”命令。因此，根据您要为其创建运行/调试配置的内容，请执行以下操作之一：

    • 如果要为单个测试或目录或包中的所有测试创建运行/调试配置：

      1. 在“ 项目工具”窗口中右键单击相应的测试类，目录或包。

      2. 从上下文菜单中选择Create ...命令。

        命令名称取决于您在“ 项目”工具窗口中的当前选择，可能类似于：

        • 为源或测试目录创建“所有测试” 。

        • 为包创建“测试'<package_name>'” 。

        • 为测试类创建“<test_name>” 。

    • 如果要为选定的测试创建运行/调试配置：

      1. 在项目工具窗口中选择感兴趣的测试。

      2. 右键单击其中一个选定的测试以打开上下文菜单。

      3. 在上下文菜单中，选择Create“<first_test_name> and <n> more” 。

    • 如果要为测试方法创建运行/调试配置：

      1. 在编辑器中打开包含感兴趣的方法的测试类。

      2. 要打开上下文菜单，请右键单击方法代码中的某个位置。

      3. 在上下文菜单中，选择Create“<method_name>（）” 。

      类似的上下文菜单命令可用于编辑器中的整个测试类。要打开类的上下文菜单，请右键单击方法代码所占区域之外的某个位置（例如，在包含类声明的行中）。

  2. 在打开的对话框中，指定运行/调试配置参数，然后单击“ 确定” 。有关更多信息，请参阅JUnit或TestNG的相应对话框描述。
