基本

为了支持ActionScript和Flex ，IntelliJ IDEA提供：

  • Flash / Flex支持插件 。此插件与IDE捆绑在一起，必须启用。

  • 专用模块类型（Flash）。

  • 构建各种目标平台（Web，桌面和移动）和输出类型（SWF和SWC）的配置。

  • 专用的运行/调试配置类型（Flash App，FlexUnit和Flash Remote Debug）。

  • ActionScript Profiler和Flash UI Designer插件。这些插件可从JetBrains存储库下载。有关更多信息，请参阅在Flash和Flex应用程序中分析CPU 。另请参阅管理插件 。

ActionScript和Flex支持

ActionScript和Flex支持包括：

  • 代码完成 ，包括完成语句 （ Ctrl + Shift + Enter ）和智能类型完成 （ Ctrl + Shift + Space ）。

  • 错误和语法突出显示。

  • ActionScript和Flex代码重构 ：

    • 更改方法签名 。

    • 代表方法。

    • 提取界面 。

    • 提取方法 。

    • 提取超类 。

    • 内联 。

    • 介绍常数 。

    • 介绍Field 。

    • 提取参数 。

    • 介绍变量 。

    • 将类或接口移动到包 。

    • 将内部移动到上层以将在包外声明的类，函数，变量，常量和命名空间移动到包中。

    • 移动/复制文件。

    • 移动静态成员 。

    • 向上拉成员，向下推送成员 ， 重构允许您通过将类成员移动到子类或子接口来清理类层次结构。然后，成员仅重定位到直接子类/接口。这提高了内部类的一致性。 。

    • 重命名文件，函数，变量，参数，属性或标签（直接和通过引用）。

    • 安全删除文件。

  • ActionScript和Flex代码检查和快速修复 。

  • 用于创建各种应用程序元素的意图操作 。

  • 代码格式和折叠 。

  • 高级搜索和导航 ，以及结构视图 。

  • 使用天沟图标增强导航 。

  • 从CSS属性和选择器导航到ActionScript中的声明（ Ctrl + B ）。

  • 使用各种编译器shell和编译选项构建ActionScript和Flex应用程序的可能性。

  • 支持断点和特定的运行/调试配置 ，可直接从IntelliJ IDEA调试ActionScript和Flex应用程序。

  • ActionScript和Flex-aware 调试器 ，允许您逐步执行应用程序，计算表达式，检查相关信息并查找运行时错误。

  • AsDoc的快速Javadoc （ Ctrl + Q ）。

  • BlazeDS支持。

  • AIR应用程序开发支持应用程序开发周期的各个阶段。还支持开发用于移动设备的AIR应用程序。有关基本的操作方法信息，请参阅IntelliJ IDEA 10.5中的新增功能：为Android开发移动AIR应用程序 。

  • 可以创建纯ActionScript应用程序。

  • ActionScript 实时模板 ： 文件|设置|实时模板| ActionScript和JavaScript组。

  • 能够在ActionScript和MXML中包装和解包代码结构（ Ctrl + Alt + T和Ctrl + Shift + Delete ）。

  • 为Flex源（ *.mxml和*.as文件）键入层次结构（ Ctrl + H ），方法层次结构（ Ctrl + Shift + H ）和调用层次结构（ Ctrl + Alt + H ）。

  • 轻松导入在Adobe Flash Builder中创建的Flex项目 。

  • 导入Flexmojos项目。请参阅相关导入设置的说明。

  • ActionScript和Flex UML类图 。这些功能包括以结构化可视化形式查看 ActionScript源文件中的更改 。

FlexUnit支持

IntelliJ IDEA支持FlexUnit的 0.9和4版本， FlexUnit是Flex和ActionScript应用程序和库的单元测试框架。

FlexUnit支持包括：

  • 专用的FlexUnit运行/调试配置，用于运行单个测试方法，测试套件，特定测试类中的所有方法或给定包中的所有测试类。

  • 能够在运行和调试模式下执行测试。

  • 通过FlexUnit 4测试运行器支持Flunit测试。

  • 支持FlexUnit的代码检查（默认情况下已关闭）。
