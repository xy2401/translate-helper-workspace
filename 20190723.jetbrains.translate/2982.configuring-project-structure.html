WebStorm 2019.1帮助

配置项目结构

在WebStorm中， 内容是您当前使用的文件集合，可能以子文件夹的层次结构进行组织。项目的顶级文件夹是其内容根 。

您可以添加任何文件夹作为项目的一部分。为此，只需在“ 目录”页面上将此文件夹添加为新的内容根目录 ，如使用多个内容根 目录中所述。

配置内容根目录中的文件夹

您也可以排除特定文件 。

或者，单击“ 设置/首选项”对话框中的“ 目录 ” （ Ctrl + Alt + S ），然后在打开的“ 目录”页面上配置文件夹类别。

在内容根目录中，WebStorm可以区分包含您编辑的源代码的文件夹和包含其他类型内容的文件夹。默认情况下，任何文件夹都被视为源，并用除非您将其标记为其他类型：

  • 测试 （ ）文件夹允许您将与测试相关的代码与生产代码分开。

  • 资源根 （ ）文件夹：可以相对于这些文件夹引用资源根文件夹中的文件，而不是指定完整路径。

  • 排除 （ ）代码完成，导航和检查会忽略文件夹。排除目前不需要的文件夹以提高WebStorm性能。

配置文件夹的类别

  • 在“ 项目”工具窗口中右键单击该文件夹，从上下文菜单中选择“ 标记目录” ，然后选择所需的类别。

要还原文件夹的上一个类别

  • 在“ 项目”工具窗口中右键单击该文件夹，从上下文菜单中选择“ 标记目录” ，然后选择“ 取消标记为<当前类别>” 。对于Excluded folders，选择Cancel Exclusion 。

排除文件

如果您现在不需要文件但又不想完全删除它，则可以暂时从项目中排除该文件。代码完成，导航和检查将忽略排除的文件。

  • 要排除文件，请在“ 项目”工具窗口中右键单击该文件，然后从菜单中选择“ 标记为纯文本 ”。该文件标有目录树中的图标。

  • 要还原更改，请右键单击该文件，然后从菜单中选择标记为<文件类型> 。

按名称模式排除文件和文件夹

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 目录”。 将打开“ 目录”页面。

  2. 在“ 排除文件”字段中，键入定义要排除的文件和文件夹名称的掩码，例如*.min.js 。使用分号作为分隔符。
    如果文件名与任何这些模式匹配，则将该文件视为已排除。如果文件夹的名称与模式匹配，则此文件夹及其子文件夹中的所有文件都标记为已排除。

使用多个内容根

您可以在项目中拥有多个内容根。这使您可以处理来自几个没有公共直接父级的目录中的文件。当您使用静态内容（例如图标）时，这很有用。您可以将它们全部保存在一个文件夹中，然后将此文件夹作为内容根添加到多个项目中。

在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 目录” 。将打开“ 目录”页面。

  • 要添加内容根，请单击右侧窗格中的“ 添加内容根 ”，然后在打开的对话框中选择所需的文件夹。

  • 要删除内容根，请在右侧窗格中选择它，然后单击在它的旁边。确认删除。请注意，不会物理删除相应的文件夹。
