解决冲突

当您在团队中工作时，当有人将更改推送到您当前正在处理的文件时，您可能会遇到这种情况。如果这些更改不重叠（即对不同的代码行进行了更改），则会自动合并冲突的文件。但是，如果相同的行受到影响，Git不能随意选择一侧而是另一侧，并要求您解决冲突。

在Git中，当您尝试执行以下操作之一时可能会出现冲突： pull ， merge ， rebase ， cherry-pick ， unstash changes或apply patch 。如果存在冲突，则这些操作将失败，系统将提示您接受上游版本，更喜欢您的版本或合并更改：

IntelliJ IDEA提供了一种在本地解决冲突的工具。该工具由三个窗格组成。左窗格显示只读本地副本;右侧窗格显示签入存储库的只读版本。中央窗格显示了一个功能齐全的编辑器，其中显示了合并和解决冲突的结果。最初，此窗格的内容与文件的基本修订版本相同，即从中派生两个冲突版本的修订版本。

解决冲突

  1. 单击“ 冲突”对话框中的“ 合并 ”，或在编辑器中选择冲突文件，然后选择“ VCS” <your_VCS> |从主菜单中解决冲突 。

  2. 要自动合并所有非冲突的更改，请单击 （ 应用所有非冲突的更改 ）在工具栏上。你也可以使用 （ 从左侧应用非冲突的更改 ）和 （ 从右侧应用非冲突更改 ）分别合并对话框左/右部分的非冲突更改。

  3. 要解决冲突，您需要选择要应用的操作（接受或忽略 ）到左侧（本地）和右侧（存储库）版本，并在中央窗格中检查生成的代码：

    您还可以右键单击中央窗格中的冲突，然后使用上下文中的命令。Resolve使用Left和Resolve使用Right命令提供了从一侧接受更改并从另一侧忽略它们的快捷方式：

    对于简单冲突（例如，如果在不同的文件修订版中修改了同一行的开头和结尾），则可以使用Resolve选项 ，允许在一次单击中合并更改：

    “ 应用所有非冲突更改”操作无法解决此类冲突，因为您必须确保它们已正确解析。

  4. 比较不同版本以解决冲突也可能很有用。使用工具栏按钮，用于调用选项列表。请注意， Base指的是本地和存储库版本源自的文件版本（最初显示在中间窗格中），而Middle指的是生成的版本。

  5. 在中央窗格中查看合并结果，然后单击“ 应用” 。

生产力提示

自动应用非冲突的更改

您可以将IntelliJ IDEA配置为始终自动应用非冲突更改，而不是通过“ 合并”对话框告知它执行此操作。为此，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|在左侧进行差异合并 ，然后选择自动应用非冲突更改选项。

处理与LF和CRLF行结尾相关的冲突

通常，在团队中工作并为同一存储库做贡献的人使用不同的操作系统。这可能会导致行结束的问题，因为Unix，Linux和MacOS使用LF ，而Windows使用CRLF来标记行的结尾。

IntelliJ IDEA在差异查看器中显示行结尾的差异 ，因此您可以手动修复它们。如果您希望Git自动解决此类冲突，您需要在Windows core.autocrlf属性设置为true并在Linux和MacOS上input （有关更多详细信息，请参阅处理行结尾 ）。您可以通过在Windows上运行git config --global core.autocrlf true或在Linux和macOS上运行git config --global core.autocrlf input来手动更改配置。

但是，IntelliJ IDEA可以自动分析您的配置，如果您要将CRLF提交到远程存储库，则会发出警告，并建议将core.autocrlf设置为true或input具体取决于您的操作系统。

要启用LF和CRLF行分隔符的智能处理，请打开“ 设置”对话框（ Ctrl + Alt + S ），然后选择“ 版本控制”| “ 版本控制”。左边的Git节点。 如果要提交CRLF行分隔符选项，请启用警告 。

启用此选项后，每当您要使用CRLF分隔符提交文件时，IntelliJ IDEA将显示行分隔符警告对话框 ，除非您在受影响的文件中设置了任何相关的Git属性 （在这种情况下，IntelliJ IDEA假设你清楚地了解你在做什么，并从分析中排除这个文件。

在“ 行分隔符警告”对话框中 ，单击以下选项之一：

  • Commit As是忽略警告并使用CRFL分隔符提交文件。

  • 修复并提交将core.autocrlf属性设置为true或input具体取决于您的操作系统。因此，在提交之前， CRLF行分隔符将替换为LF 。

如果稍后需要查看在合并期间如何解决冲突，可以在“ 日志”视图中找到所需的合并提交，在右侧的“ 提交详细信息”窗格中选择包含冲突的文件，然后单击或按Ctrl + D （有关详细信息，请参阅查看更改的合并方式）。
