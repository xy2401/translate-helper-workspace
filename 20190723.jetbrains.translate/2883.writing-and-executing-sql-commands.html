编写和执行SQL语句

选择默认架构或数据库

您可以使用工具栏右侧的列表选择默认架构或数据库。如果这样做，您将能够在语句中省略该架构或数据库的名称。

控制PostgreSQL和Redshift的模式搜索路径

使用PostgreSQL或Redshift数据源时，除非您指定其他搜索路径，否则将使用默认搜索路径（数据库中的一个集合）。

要控制搜索路径，请使用工具栏右侧的弹出窗口。弹出窗口还允许您切换数据库。

要选择数据库或为其设置默认搜索路径，请单击数据库或按Enter键 。

要更改当前数据库的搜索路径，请打开架构列表。

如果搜索路径应仅包含一个架构，请单击必要的架构。以同样的方式，您可以在单模式搜索路径中将模式替换为另一模式。

要形成包含两个或更多模式的搜索路径，请使用：

  • 空间高亮显示的架构添加到搜索路径，并删除从搜索路径的模式。

  • Alt + Up和Alt + Down可重新排序搜索路径中的模式。

  • 单击确定以应用更改。

弹出窗口中提供了更多说明和使用提示。

编写SQL语句

在编写SQL语句时，请使用：

  • 预定义模式（ Ctrl + J或代码|插入实时模板 ）。

  • 自动完成和突出显示SQL关键字以及表和列名称。

  • 列的数据类型提示（ Ctrl + P或View | Parameter Info ）。

  • 高级查找和替换功能（ Ctrl + F或编辑|查找|查找 ， Ctrl + R或编辑|查找|替换 ）。

  • 快速评估（ Ctrl + Alt + F8 ）。它们可用于表和列名称以及SQL关键字，并在撰写语句时为您提供有关数据和潜在结果的提示。

  • 控制台历史记录（ 或Ctrl + Alt + E ）。请参阅执行自动记忆的语句 。

另请参见在“数据库”工具窗口中导航到表或列视图 。

以表格格式编辑INSERT语句的数据

  1. 选择感兴趣的INSERT语句。

  2. 从上下文菜单中选择“ 编辑为表格 ”。

  3. 使用上下文菜单命令和相关的快捷方式来处理数据。

导航到“数据库”工具窗口中的表或列视图

在编写语句时，有时可以查看表的结构，或者查看有关其所属表的上下文中的列（字段）的信息。出于此目的，PyCharm提供了从输入窗格中的表或列的名称切换到数据库工具窗口中的视图的功能。

以下方法可用于使用此功能：

  • 将光标放在感兴趣的表或列的名称中。然后使用Ctrl + B.（或者，您可以从主菜单中使用“ 导航”|“声明”或从上下文菜单中使用“ 转到”|“声明” 。）

  • 按住Ctrl键，然后指向感兴趣的名称。当文本变为超链接时，单击超链接。

配置执行命令

执行命令（ 在工具栏上， Ctrl + Enter或从上下文菜单执行 ）用于运行您的语句。

PyCharm为Execute命令提供了许多选项，具体取决于光标位置和是否有选择。

选项在工具|中指定“ 设置/首选项”对话框中的“ 数据库”页面（“ 文件”|“设置”|“工具”| “Windows和Linux上的数据库 ; PyCharm |首选项|工具| macOS上的数据库 ”）。有关更多信息，请参阅在控制台中执行 。

执行SQL语句

  1. 将光标放在语句中。

  2. 按Ctrl + Enter 。或者，单击“ 执行”图标在工具栏上。

    如果您有多个语句，请选择是要执行所有语句还是单个语句。建议列表始终包含用于运行所有语句的项目。

执行参数化语句

您的语句可以包含参数，但是，在执行此类语句时，必须指定参数的值。有以下指定参数值的方法：

  • 点击在工具栏上或按Ctrl + Enter执行该语句。在打开的对话框中，指定参数值，然后单击“ 确定” 。

    （要开始编辑值，请切换到相应的表格单元格并开始键入。要指示您已完成编辑值，请按Enter或切换到其他单元格。要退出编辑模式并恢复初始值，请按Escape键。）

  • 或者，您可以在“ 数据库控制台”工具窗口中打开“ 参数”窗格（ 在工具栏上）并在那里指定相应的值。（这些值的编辑方式与对应的对话框相同。）然后执行声明（ 在工具栏上或按Ctrl + Enter ）。

    有关更多信息，请参阅参数窗格 。

另请参阅“ 用户参数”和“ 始终在执行前查看参数” 。

执行一组语句

要在控制台中执行一组彼此跟随的语句，请选择（突出显示）语句（选择所有语句，使用Ctrl + A ）并执行以下操作之一：

按Ctrl + Enter 。或者，单击“ 执行”图标在工具栏上。

另请参阅使用错误通知栏和控制台中的执行 。

执行所有陈述

要执行控制台中包含的所有语句，可以使用Run console.sql命令作为Execute命令的替代方法。

此命令在上下文菜单中可用，其等效键盘为Ctrl + Shift + F10 。

通常， 运行console.sql命令运行得更快但是：

  • 带参数的语句不会运行。

  • 未显示SELECT语句的检索数据。

执行语句的一部分（例如子查询）

要执行语句的一部分（例如子查询），请选择（突出显示）要执行的片段，然后执行以下操作之一：

按Ctrl + Enter 。或者，单击“ 执行”图标在工具栏上。

另请参阅“ 在控制台中执行” 。

执行自动记忆的语句

当您在控制台中运行SQL语句时，PyCharm会记住它们。因此，稍后您可以查看已运行的语句，并在必要时再次运行它们。

要打开显示自动记忆语句的对话框（“ 历史记录”对话框），请执行以下操作之一：

  • 点击在工具栏上。

  • 按Ctrl + Alt + E.

“ 历史记录”对话框中有两个窗格。左侧窗格显示您已运行的语句列表。对于“长”语句，只显示它们的开头。在此窗格中选择语句时，整个语句将显示在右侧窗格中。

您可以过滤信息：只需开始输入即可。因此，只会显示包含键入文本的语句。

您可以将“ 历史记录”对话框中的语句复制到控制台的输入窗格中。要复制语句，请执行以下操作之一：

  • 双击要复制的语句。

  • 选择感兴趣的语句，然后按Enter键 。

  • 选择语句，然后单击“ 确定” 。

（一旦语句在输入窗格中，您就可以立即运行它。）

您可以删除不必要的记忆语句。要删除语句，请在“ 历史记录”对话框中选择该语句，然后按Delete键。

将SELECT语句的结果输出到文件中

您可以将SELECT语句的结果输出到文件中，而不是“ 服务”工具窗口的结果窗格。

  1. 右键单击感兴趣的SELECT语句。

  2. 指向执行到文件并选择输出格式。

  3. 指定输出文件位置和名称。

使用错误通知栏

如果在运行语句时发生错误，则会在输入窗格的下半部分显示错误通知栏。

执行一系列语句时，此栏可能特别有用（请参阅执行一组语句 ），因为在这种情况下，它允许您选择如何做出反应。

选项是：

  • 重试。 从导致错误的语句开始执行语句序列。

  • 忽视。 跳过错误的语句并从下一个语句开始执行序列。如果发生其他错误，将再次出现错误通知栏。

  • 忽略所有。 跳过错误的语句并从下一个语句开始执行序列。如果发生其他错误，将跳过所有错误的语句，并且不会为这些语句显示错误通知栏。

  • 停止。 停止执行序列。

在“ 设置”对话框中启用或禁用显示输入窗格中的错误通知栏（“ 数据库”页面上的“ 在编辑器中显示错误通知”复选框）。

取消正在运行的语句

要终止当前语句的执行，请执行以下操作之一：

  • 点击在输入窗格的工具栏上，或在Dababase控制台工具窗口的工具栏上。

  • 按Ctrl + F2 。

管理数据库事务

您可以选择自动或手动提交事务。要更改提交模式，请使用Tx开关在工具栏上。

如果提交模式设置为Auto ，则每个SQL语句都在其自己隐式提交的事务中执行。因此，无法回滚在此模式下执行的SQL语句。

如果提交模式设置为“ 手动” ，则会通过以下方式显式提交或回滚事务要么在工具栏上。

Tx开关还可用于选择事务的隔离级别 。

运行迁移

DEFINER子句指定例程执行的安全上下文（访问权限）。生成迁移脚本时，请选择“ 忽略定义器”以跳过此子句。

要还原更改，请单击“ 还原”图标（ ）。

要查看原始代码与更改之间的并排差异，请单击“ 显示差异”图标（ ）。

运行存储过程

  1. 右键单击要执行的存储函数。

  2. 在“ 执行例程”窗口中，键入所有必需的参数值，然后单击“ 确定” 。

PyCharm生成一个SQL代码，用于从JDBC驱动程序中检索结果集。

显示执行计划

执行计划是一组用于访问数据库中数据的步骤。PyCharm支持两种类型的执行计划：

  • 说明计划 ：结果以专用计划选项卡上的混合树和表格格式显示。您可以单击“ 显示可视化”图标（ ）创建一个可视化查询执行的图表。

  • 解释计划（原始） ：结果以表格格式显示。

要创建执行计划，请在数据库控制台中右键单击查询，然后选择“ 解释计划” 。如果要为查询创建图表，请单击“ 显示可视化”图标（ ）。

显示Oracle的DBMS_OUTPUT

对于Oracle，您可以在输出窗格中启用或禁用显示DBMS_OUTPUT缓冲区的内容。为此，请使用在“ 服务”工具窗口的工具栏上（ Ctrl + F8 ）。
