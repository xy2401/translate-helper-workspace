使用向导生成签名版本APK

要在物理设备上部署和运行Android应用程序，您需要以数字方式对应用程序进行签名 。使用IntelliJ IDEA，您可以在包提取时使用现有发行密钥签署Android应用程序包（ .apk文件）。IntelliJ IDEA还包含一个可在打包过程中调用的释放密钥生成工具。生成的密钥保存在密钥库二进制文件中。

您可以根据需要拥有任意数量的密钥库文件和密钥，并使用现有密钥，或在现有密钥库中创建新密钥，甚至创建新的密钥库。

如果您使用生成签名APK向导 ，IntelliJ IDEA将在提取时签署包。

在本页面：

  • 使用向导提取和签署Android应用程序包
  • 生成新的释放密钥

使用向导提取和签署Android应用程序包

  1. 从主菜单中，选择Build |生成签名APK 。生成签名APK向导启动。

  2. 在第一页上 ，指定要使用的发行密钥以及包含它的密钥库文件。执行以下操作之一：
    • 使用现有密钥库文件中的密钥对包进行签名：

      1. 在密钥库路径文本框中指定文件位置。手动键入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择该文件。在密钥库密码文本框中，键入所选密钥库的密码。

      2. 指定密钥别名并输入密码以访问密钥。

    • 要在现有密钥库中生成新密钥：

      1. 在密钥库路径文本框中指定文件位置。手动键入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择该文件。在密钥库密码文本框中，键入所选密钥库的密码。

      2. 单击“ 创建新”按钮，然后通过填写打开的“ 新建密钥库”对话框中的数据来配置要生成的版本密钥 。

    • 要使用新密钥生成新的密钥库文件：

      1. 单击“ 创建新”按钮。在打开的“ 新建密钥库”对话框中，在“ 密钥库路径”文本框中指定要生成的文件的位置。手动键入路径或单击“ 浏览” ，然后选择父文件夹并指定文件的名称。

      2. 指定并确认密码以访问密钥库。

      3. 通过填写Key区域中的数据来配置新的释放密钥 。

    单击下一步 。

  3. 在下一步中 ，在“ 目标APK路径”文本框中，指定将保存.apk的文件夹。手动键入路径，或单击“ 浏览” 并在打开的对话框中选择目标文件夹。

  4. 要让IntelliJ IDEA在打包过程中对应用程序进行模糊处理，请选中Run ProGuard复选框并指定proguard.txt配置文件的位置。该文件在项目创建时生成，并存储在项目根目录中。IntelliJ IDEA在“ 配置文件路径”文本框中建议此默认位置。单击“ 浏览”接受建议或指定自定义配置文件并在打开的对话框中选择所需的文件。

  5. 单击“ 完成”以生成并签名包。IntelliJ IDEA会通知您操作成功完成。

    • 要打开生成的.apk文件所在的文件夹，请单击“ 打开文件位置”按钮。

    • 要完成向导，请单击“ 关闭” 。

生成新的释放密钥

  1. 通过执行以下操作之一打开“ 新建密钥库”对话框：

    • 在Generate Signed APK向导的第一页上，单击Create new按钮。

    • 创建Android应用程序工件定义时 ，单击Android选项卡中的创建新按钮。

  2. 指定密钥库位置并输入密码以访问它。

    如果要将新密钥添加到现有密钥库并已在向导或“ 工件”选项卡中选择它，则这些字段已填写。

  3. 在别名文本框中为新键指定别名。将使用此别名引用密钥。

  4. 在密码文本框中输入密钥密码并确认。

  5. 根据应用程序的预期使用寿命指定密钥的有效期。

  6. 提供以下个人信息，然后单击下一步 ：

    • 名字和姓氏

    • 组织单元

    • 组织

    • 城市或地区

    • 州或省

    • 国家代码
