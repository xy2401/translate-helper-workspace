ALT + 0

只有安装并启用了Dart插件时，工具窗口才可用。该插件默认激活。如果插件已禁用，请在“ 插件”页面上启用它，如管理插件中所述 。

WebStorm与Dart Analyzer工具集成，可对Dart源代码执行静态分析。在Dart分析工具窗口中报告所有不一致和潜在问题，可以导航到检测到问题的源代码片段。

在本页面：

  • 消息列表
  • 分组和排序
  • 工具栏按钮
  • 上下文菜单

消息列表

所有问题报告都显示在一个表中，该表由两列组成， 描述和位置 ，以及下面的摘要。

  • “ 描述”列显示消息本身，可以是错误消息 ， 警告或提示 。

  • “ 位置”列显示以下数据：

    • 出现问题的Dart包的名称。Dart包名称对应于pubspec.yaml中的project name 。

    • 从Dart包根到创建问题的文件的相对位置。

    • 出现问题的行号。

  • 表格下方的“ 摘要”区域显示检测到的错误，警告或提示的数量以及过滤器状态。

分组和排序

可以对Dart分析工具窗口中的消息进行分组和排序。默认情况下，消息主要按其严重性排序，即错误消息显示在列表顶部，然后出现警告，最后显示提示。在每个严重性组中，消息按Dart包名称分组，这对于具有多个pubspec.yaml文件的项目非常方便。在每个包组中，邮件按其文件路径分组。最后，在每个文件组中，问题按发生的行号排序。

您可以重新配置二级排序：

  • 单击“ 位置”列标题可在Dart包名称+文件路径排序的升序和降序之间切换。请注意，这不会影响严重性组的顺序，即始终首先显示错误。也不考虑行号，对于每个特定文件，问题总是以行号的升序显示。

  • 单击“ 描述”列标题，按问题描述的字母顺序执行二级排序。请注意，这不会影响严重性的主要分组。

要按严重性禁用主分组，请按严重性切换按钮释放组 在工具栏上。之后，根本不考虑问题的严重性，问题报告仅按其描述或位置 （ Dart包名称+文件路径 ）进行排序。此排序方法可确保将每个特定文件的所有问题组合在一起，而不管其严重性如何。

过滤

单击此按钮可打开“ Dart Problems Filter”弹出窗口，您可以在其中配置根据工具窗口中是否显示问题报告的条件。

  • 在Severity区域中，指定要显示的消息类型，可用类型为Errors ， Warnings和Hints 。要显示严重性级别的问题，请选中此严重性级别旁边的复选框。

  • 在“ 文件”区域中，选择要查看问题报告的范围。可用选项包括：
    • 整个项目
    • 当前内容根

    • 当前包：当选择此选项但从当前文件开始在文件夹层次结构中未检测到pubspec.yaml文件时，将根据当前文件所在的内容根执行过滤。

    • 当前文件

  • 单击“ 重置所有过滤器”链接以恢复默认过滤器设置。

更改当前设置后立即应用过滤。
