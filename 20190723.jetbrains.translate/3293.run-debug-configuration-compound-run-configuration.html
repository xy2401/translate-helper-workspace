运行/调试配置：复合

运行|编辑配置| |复合

要按顺序启动配置，请将第一个添加为Before启动任务。例如，您可以将Node.js运行/调试配置添加到JavaScript调试配置。

使用此对话框可以创建包含可以一次启动的多个运行/调试配置的配置。例如，如果要启动各种自动化任务或测试套件，这很有用。每个配置都将在“ 运行”或“ 调试”工具窗口的单独选项卡中提供。

点击选择要包含在Compound配置中的现有配置，并填写以下字段：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

共享运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。
