IntelliJ IDEA代码编辑器中的HTTP客户端

此功能仅在Ultimate版本中受支持。

测试Web服务时，您可以直接在IntelliJ IDEA 代码编辑器中创建，编辑和执行HTTP请求。

HTTP请求存储在HTTP和REST文件中，并标有图标。

对HTTP文件的支持包括以下功能：

  • 代码突出显示

  • 方法类型和标题字段的代码完成

  • 请求标题字段和文档标记的内联文档

  • 查看 HTTP请求的结构

  • 请求消息正文中的语言注入

  • 移动重构

  • 实时模板

在开始之前，如有必要，请在“设置/首选项” （ Ctrl + Alt + S ）对话框的“ HTTP代理”页面上配置代理设置 。

通过捆绑的HTTP客户端插件支持测试Web服务 ，该插件在“设置/首选项”中默认启用插件 。

创建HTTP请求文件

您可以使用暂存文件或HTTP请求类型的物理文件处理HTTP请求 。

Scratch文件可用于在开发期间测试HTTP请求。临时文件不存储在项目中，因此IntelliJ IDEA可以修改它并添加有关请求的其他信息。从临时文件执行HTTP请求时，响应输出文件的链接将添加到请求下方和请求历史记录文件的顶部。

创建HTTP请求临时文件

  • 按Ctrl + Shift + Alt + Insert并选择HTTP请求 。

物理文件可用于记录，测试和验证HTTP请求。物理文件存储在项目中，IntelliJ IDEA不会修改它。从物理文件执行HTTP请求时，不会修改此文件。有关已执行请求以及响应输出文件链接的信息将添加到请求历史记录文件的顶部。

创建物理HTTP请求文件

  • 在“ 文件”菜单上，指向“ 新建” ，然后单击“ HTTP请求” 。

您可以使用移动重构（ F6 ）将HTTP请求从划痕移动到物理文件，以及物理文件之间。

移动HTTP请求

  1. 在编辑器中，将插入符号放在要移动的请求上，并执行以下操作之一：

    • 从主菜单或上下文菜单中，选择Refactor |移动

    • 按F6 。

  2. 在打开的对话框中，单击选择文件或键入要将请求移动到的文件的完整路径。请注意，您可以指定不存在的文件的名称，在这种情况下，将自动创建具有提供的名称的新文件。

撰写HTTP请求

您可以使用以下常规语法直接在创建的HTTP请求文件中键入HTTP请求：

方法Request-URI HTTP-Version标头字段：标头值Request-Body

IntelliJ IDEA使用编辑器格式的HTTP请求 ，它提供了一种创建，执行和存储有关HTTP请求的信息的简单方法。要了解其可能性的概述，您可以浏览HTTP请求集合 ，这是一系列组合请求。

从HTTP请求集合中打开请求

  1. 点击在编辑器的右上角或选择“ 工具”| HTTP客户端|在主菜单中打开HTTP请求集合 。

  2. 从列表中，选择您要打开的HTTP Requests集合：

有关完整格式说明，请参阅编辑器语法中的探索HTTP请求以获取语法和功能概述，以及编辑器规范中的HTTP请求 。

要加快撰写HTTP请求的速度，请使用实时模板 。例如， gtr扩展为简单的GET请求; mptr扩展为multipart/form-data POST请求。

使用环境变量

编写HTTP请求时，可以使用环境变量对其元素进行参数化。例如，您可以使用{{host}}占位符代替在请求中明确提供主机名。然后在项目中定义一组包含所需主机定义的环境变量。执行请求时，可以在已定义的环境中进行选择，例如，将请求发送到的主机 ：

在查看请求的结构并在浏览器中打开请求或为其创建运行/调试配置时，所选环境将用作默认环境。

变量的名称可以是任意的，但可以只包含字母，数字，下划线符号（ _ ）或连字符号（ - ）。变量可以保存以下请求组件的值：

  • 请求主机，端口和路径

  • 查询参数或值

  • 标头值

  • 直接在请求正文中提供或在外部文件中提供的任意值。

环境变量在环境文件中定义，必须存储在模块中：

  • rest-client.env.json （或http-client.env.json ）是一个常规文件，旨在与您的项目一起分发。

  • rest-client.private.env.json （或http-client.private.env.json ）是一个私有文件，可能包含密码，令牌，证书和其他敏感信息。默认情况下，此文件将添加到VCS忽略的文件列表中。私有文件中指定的变量值将覆盖环境文件中的值。

您可以通过client.global.get和client.global.set方法在响应处理程序脚本中以编程方式管理变量。

示例：使用环境文件

以下示例http-client.env.json环境文件定义了两个环境： 开发和生产 。附加的http-client.private.env.json文件包含授权敏感数据。

{ “development” ： { “host” ： “localhost” ， “id-value” ： 12345 ， “username” ： “” ， “password” ： “” ， “my-var” ： “my-dev-value” } ， “production” ： { “host” ： “example.com” ， “id-value” ： 6789 ， “username” ： “” ， “password” ： “” ， “my-var” ： “my-prod-value “ } }
{ “development” ： { “username” ： “dev-user” ， “password” ： “dev-password” }， “production” ： { “username” ： “user” ， “password” ： “password” } }

示例HTTP请求如下：

获取http：// {{ host }} / api / json / get？id = {{ id-value }} ＆ key = {{ unresolved_var }}授权：基本{{ username }} {{密码}}内容类型：application / json { “key” ： {{ my-var }} }

执行上述请求时，IntelliJ IDEA可在已定义的执行环境之间进行选择：

根据您的选择，生成的请求将是以下之一：

GET http：// localhost / api / json / get？id = 12345 ＆ key = {{ unresolved_var }}授权：基本开发用户dev-password内容类型：application / json { “key” ： “my-dev-价值“ }
获取http://example.com/api/json/get?id = 6789 ＆ key = {{ unresolved_var }}授权：基本用户密码内容类型：application / json { “key” ： “my-prod-value” }

由于{{unresolved-var}}变量未在环境文件中定义，因此IntelliJ IDEA将在两种情况下发送{{unresolved-var}}文本作为请求的一部分。

使用响应处理程序脚本

使用响应处理程序脚本，您可以以编程方式对收到的HTTP响应做出反应。通过使用这些脚本，您可以自动处理接收的数据，并根据您指定的条件对其进行验证。响应处理程序脚本作为HTTP请求文件中请求的一部分提供，并在收到响应后立即执行。要查看响应处理示例，请打开auth-requests或test-responses 请求集合 。

您可以在适当的位置或通过引用外部文件将响应处理程序脚本插入到请求中。

将脚本插入请求中

  • 要在脚本中插入脚本，请在其前面加上>并将其括在{% %} ：

    GET host / api / test> { ％//响应处理程序脚本...％ }

  • 要从外部文件插入脚本，请在>前面添加：

    GET host / api / test> scripts / my-script.js

响应处理程序脚本使用JavaScript ECMAScript 5.1编写，编码帮助和文档由捆绑的HTTP Response Handler库HTTP Response Handler 。对于就地脚本，此功能会自动启用。对于外部脚本，您需要手动启用它。

在JavaScript文件中启用HTTP响应处理程序编码帮助

  1. 在编辑器中打开脚本文件。

  2. 在上下文菜单中，选择“ 使用JavaScript库” HTTP响应处理程序 。

    有关详细信息，请参阅配置JavaScript库 。

HTTP Response Handler库公开了两个用于组合响应处理程序脚本的对象：

  • client对象存储会话元数据，可以在脚本内部进行修改。在关闭IntelliJ IDEA之前，将保留client状态。 client.global保存为variable_name每个变量都可以作为{{variable_name}}对后续HTTP请求进行访问。

  • response包含有关收到的响应的信息：其内容类型，状态，响应正文等。

响应处理程序脚本可以包含测试，允许您将HTTP客户端用作测试框架。要创建测试，请调用client.test(testName, function)方法。在测试中，您可以通过调用client.assert(condition, message)方法来断言client.assert(condition, message)例如：

GET https://httpbin.org/status/200> { ％client.test （ “请求执行成功” ， function （） { client.assert （ response.status === 200， “响应状态不是200” ） ; }） ; ％ }

将插入符号放在库对象上，然后按Ctrl + B在编辑器中打开HTTP Response Handler库。

执行HTTP请求

  1. 如果要测试自己的Web服务，请确保已部署并运行该服务。

  2. 请执行以下任何操作：

    • 在编辑器中，单击“运行” 在您要执行的请求旁边的装订线中。从列表中，选择“运行<请求名称>” 。

    • 在编辑器中，从请求的上下文菜单中选择“运行<请求名称> ”。

    • 将插入符号放在要执行的请求上，按Alt + Enter并选择“ 运行<请求名称>意图操作”。

    执行请求时，IntelliJ IDEA会自动为其创建专用的临时HTTP请求运行/调试配置 。如有必要，可以将其保存为永久运行/调试配置 。

在浏览器中打开请求

您可以在“设置/首选项”对话框的“ Web浏览器”页面上指定的浏览器中打开HTTP请求。选择系统默认浏览器或您选择的浏览器。

  • 请执行以下任何操作：

    • 将插入符号放在请求的第一行，然后选择View |在主菜单上跳转到Source ，或按Ctrl + B或F4 。

    • Ctrl +单击 （对于Windows和Linux）或⌘+单击 （对于macOS）请求行：

使用HTTP请求运行/调试配置

您可以使用HTTP请求类型的运行配置来执行HTTP请求。使用运行配置，您可以随时执行请求，将其添加到“运行仪表板 ”或复合配置 。

有关详细信息，请参阅使用运行/调试配置 。

创建HTTP请求运行/调试配置

  1. 请执行以下任何操作：

    • 在编辑器中，从要为其创建运行/调试配置的请求的上下文菜单中选择“ 创建<配置名称> ”。

    • 或者，选择Run |从主菜单中编辑配置 ，然后单击并从列表中选择HTTP请求 。

  2. 提供运行/调试配置参数：

    • 在环境列表中，选择一个环境 ，将定义一组请求中使用环境变量。

    • 在“ 文件”字段中，提供HTTP请求文件的路径。您可以手动键入路径并在键入时使用路径完成（ Ctrl + Space ），或单击并在打开的对话框中选择所需的文件夹。

    • 如果您的请求文件包含多个请求，请在“ 请求”列表中选择要执行的请求的索引。

当您从编辑器执行HTTP请求时 ，IntelliJ IDEA会自动使用请求参数创建临时运行/调试配置。临时运行/调试配置的工作方式与永久运行/调试配置相同。您可以使用“ 运行/调试配置”对话框更改其设置，并可选择将其另存为永久设置。

保存临时HTTP请求运行/调试配置

  • 请执行以下任何操作：

    • 在“ 运行/调试配置”选择器中，选择“ 保存<配置名称>” 。

    • 在“ 运行/调试配置”对话框中，选择配置，然后单击“ 保存”图标（ ）。

    • 在编辑器中，从执行的请求的上下文菜单中选择Save <configuration name> 。

使用运行/调试配置执行请求

  • 请执行以下任何操作：

    • 在“ 运行/调试配置”选择器中，选择所需的运行配置。然后单击“ 运行”图标（ ）在主工具栏上或按Shift + F10 。

    • 按Shift + Alt + F10 ，从列表中选择所需的运行配置，然后按Enter键 。

查看收到的回复

  1. 切换到“ 运行”工具窗口 ，该窗口会在收到响应后立即自动打开。

  3. 作为响应处理程序脚本的一部分执行的测试结果（如果有）将显示在“ 运行”工具窗口的“ 测试”选项卡上。

查看请求历史记录

IntelliJ IDEA自动将最近执行的50个请求保存到http-requests-log.http文件中，该文件存储在.idea / httpRequests /目录下的项目级别。使用请求历史记录，您可以快速导航到特定响应并再次发出任何请求 。如果从请求历史记录再次发出请求，则其执行信息和响应输出的链接将添加到请求历史记录文件的顶部。

打开请求历史记录

  • 点击在编辑器的右上角或选择“ 工具”| HTTP客户端|在主菜单上显示HTTP请求历史记录 。

如有必要，可以通过在请求之前添加带有@no-log标记的注释行来阻止将请求保存到请求历史记录 。如果请求包含一些敏感数据，并且您不想记录它，这可能会有所帮助。

配置代理设置

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为 ”下的“ 系统设置” ，然后选择“ HTTP代理” 。

  2. 在打开的“ 代理”对话框中，选择“ 手动代理配置”并指定以下内容：

    • 在主机名和端口号字段中输入代理主机名和端口号。

    • 要启用授权，请选中“ 代理身份验证”复选框，然后在相应字段中键入用户名和密码。
