在你开始之前

确保启用Web和JavaEE：WebServices（JAX-WS）捆绑插件。默认情况下会激活插件。如果禁用了插件，请在“ 插件”页面上启用它们，如管理插件中所述 。

基本

要在IntelliJ IDEA中开发Web服务，必须相应地配置相应的模块，并提供所有必需的库和servlet引用。本节介绍满足这些要求的最常用方法。

通过专用方面启用Web服务开发支持

如果要开发GlassFish / JAXWS2类型的Web服务，则此方法适用。X RI / Netro 1。X / JWSDP2.0或Apache Axis 。

通过Web和Web服务方面启用与Struts的集成。这些专用方面包含设置，配置文件路径和验证规则。此信息确定模块的结构，以便IntelliJ IDEA检测如何处理模块内容。

Web服务构面只能作为Web构面的子项添加。请注意，模块中只允许一个Web服务构面。

您可以使用专用构面创建新模块，也可以将构面添加到现有模块。在任何一种情况下，IntelliJ IDEA都会下载预定义的WS引擎实现版本。

要使用Web Services构面创建模块：

  1. 执行以下操作之一：

    • 如果要创建新项目：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件”|新的|项目 。

      结果，“ 新建项目”向导将打开。

    • 如果要将模块添加到现有项目：打开要添加模块的项目，然后选择“ 文件”|新的|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页上，在左侧窗格中，选择Java 。在页面的右侧，指定要使用的JDK 。

  3. 在“ 其他库和框架”下 ，选中“ Web应用程序”复选框。

    从“ 版本”列表中选择要支持的Servlet规范的版本 。

    如果要创建部署描述符web.xml文件，请选中Create web.xml复选框。

  4. 选择WebServices复选框。

  5. 从“ 版本”列表中选择所需的WS引擎实现。

  6. 您将需要一个实现所选WS引擎的库。您可以选择使用现有库，创建和使用新库，下载库文件（如果它们尚未在您的计算机上可用），或推迟设置库直到以后。

    • 使用库 - 从列表中选择要使用的库（如果已在IntelliJ IDEA中定义了相应的库）。

      创建 - 如果计算机上已有相应的库文件（ .jar ），则可以在库中排列这些文件并使用该新库。为此，请单击“ 创建” ，然后在打开的对话框中选择所需的文件。

      （可选）单击“ 配置”以编辑选定的库。

    • 下载 - 选择此选项可下载WS引擎实施。（下载的文件将安排在库中。）

      （可选）单击“ 配置”以编辑库设置和内容。

    • 稍后设置库 - 选择此选项以推迟设置库直到稍后的时间。

    配置 - 单击此按钮可编辑“ 使用库”或即将下载的库旁边所选库的设置。

    单击下一步 。

  7. 指定名称和位置设置。

    单击完成 。

要将Web服务构面添加到现有模块：

    将必要的库添加到现有模块

    如果先前已下载必要的库，则此方法适用。

    1. 打开模块设置。

    2. 选择所需模块后，单击“ 依赖关系”选项卡。

    3. 要启用Web开发，请单击“ 添加”按钮，然后从上下文菜单中选择“ 单条目模块库 ”。在打开的对话框中 ，选择javaee.jar库，然后单击“ 确定” 。

    4. 再次单击“ 添加”按钮，然后从上下文菜单中选择“ 单条目模块库 ”。在打开的对话框中 ，选择javaee.jar库和所需的特定于Web服务的库。

      • javaee.jar库的位置是在安装IntelliJ IDEA期间定义的。

      • Web服务特定库的位置在下载期间定义。

    5. 准备好后单击确定 。

    启用额外的WS引擎

    除了大多数常见的WS引擎，通过专用方面支持，使用IntelliJ IDEA，您还可以使用以下方法：

    • Apache Axis2

    • XFire 1。X / CXF

    • 将JBossWS

    • WebSphere 6。X

    此外，您可以使用任何所需的WS引擎实现版本，而不是将自己限制为预定义的版本。

    要使用自定义WS引擎：

    1. 下载所需的WS引擎实现。

    2. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”|左窗格中的Web服务 ，并指定外部Web服务引擎，服务器名称和端口等的路径（有关详细信息，请参阅Web服务 ）。
