配置Spring facet

此功能仅在Ultimate版本中受支持。

要在项目中使用Spring，您需要包含库和UI元素的Spring构面，以配置特定于Spring的设置。IntelliJ IDEA可以自动检测代码中的Spring配置。它将通知您缺少配置并将建议必要的操作。

如果要创建全新的项目或模块，可以使用向导选择必要的框架和库。

如果IntelliJ IDEA无法自动检测配置文件，您可以手动添加构面：

  1. 按Ctrl + Shift + Alt + S打开“项目结构”对话框。

  2. 从左侧列表中，选择模块 。

  3. 选择所需的模块，然后单击中间部分的按钮。

  4. 从列表中选择Spring 。

  5. 添加构面时可能需要设置库。在这种情况下，单击警告消息旁边窗口底部的“ 修复” 。

    如果您已有Spring库，则可以按原样使用它，也可以使用计算机上的JAR文件创建新库。在这种情况下，请选择“ 使用库”选项。

    如果您没有库，请选择下载 。

如果Spring facet配置正确，您将看到标记每个组件，bean等的Spring gutter图标。要配置装订线图标，请按Ctrl + Alt + S并转到编辑器|一般|天沟图标 。

创建应用程序上下文

Spring应用程序上下文是一种在IntelliJ IDEA中对配置文件进行分组的方法。创建上下文时，您可以让IntelliJ IDEA了解此上下文中包含的配置文件之间的关系。

您可以根据需要创建任意数量的应用程序上下文;任何配置文件都可以包含在多个上下文中。

  1. 按Ctrl + Shift + Alt + S并选择Facets 。

  2. 从列表中选择Spring facet，然后单击右侧部分的按钮。

  3. 在“ 新建应用程序上下文”对话框中，输入名称，然后选择要包含在上下文中的文件。

在某些情况下，您不需要配置上下文。例如，Spring MVC Web应用程序对其配置有严格的规定。IntelliJ IDEA中的Spring支持可以推导出它们并为您创建自动检测的应用程序上下文，以及设置Web构面。

配置父上下文

IntelliJ IDEA允许您配置上下文之间的父子关系。来自父上下文的Bean对于子上下文中的bean是可见的，但反之亦然。因此，来自子上下文的bean可以使用父上下文中的配置。

例如，Spring MVC应用程序通常有两个上下文。一个上下文属于Web层bean，另一个上下文用于服务和存储库。在这种情况下，Web层上下文将是子上下文，因为您需要将服务注入控制器，而不是其他方式。

要配置父上下文，请使用“ 新建应用程序上下文”对话框。

“ 多个上下文”面板显示在编辑器顶部，用于包含在两个或多个应用程序上下文中的文件。您可以使用此面板选择另一个活动上下文，例如，如果要使用不同配置运行应用程序，并更改突出显示。

要禁用面板，请单击按钮，并清除“ 显示多个上下文”面板复选框。

IntelliJ IDEA可以自动配置父上下文。例如，如果IDE检测到Spring Cloud上下文，它将使其成为Spring Boot的父应用程序上下文。

导航Spring依赖项

Spring图可帮助您分析应用程序中的依赖项。您可以双击图表上的bean或文件，以在编辑器的专用选项卡中打开它。

要使用图表，必须启用捆绑的UML插件。

对于Spring Boot，您还可以构建Runtime Beans图 。

Spring Beans依赖

Spring Beans依赖关系图使您可以查看和分析项目中bean之间的依赖关系。

要打开图表：

  1. 右键单击要为其构建图的bean。您还可以为类构建一个图，以查看此类中的bean及其依赖项。

  2. 从菜单中选择Diagrams 。

  3. 选择Show Diagram Popup （ Ctrl + Alt + U ）以在本地弹出窗口中打开图表，或选择Show Diagram （ Ctrl + Shift + Alt + U ）以在编辑器中打开图表。

  4. 点击 Spring Beans依赖 。

    如果要仅在图上查看bean之间的直接依赖关系，请选择“ 邻居”模式 。

Spring应用程序上下文依赖项

Spring Application Context Dependencies图允许您查看多个配置文件之间的依赖关系，并分析它们如何包含和引用彼此。

图上的红色箭头标记错误（例如循环依赖关系），并允许您轻松地从Spring配置中发现和删除它们。

要打开图表：

  1. 右键单击要为其构建图的配置文件。

  2. 从菜单中选择Diagrams 。

  3. 选择Show Diagram Popup （ Ctrl + Alt + U ）以在本地弹出窗口中打开图表，或选择Show Diagram （ Ctrl + Shift + Alt + U ）以在编辑器中打开图表。

  4. 点击 Spring应用程序上下文依赖项 。

蓝色实线代表@ComponentScan注释依赖项。

黄色实线代表@Enable注释依赖项。

绿色虚线代表@Import注释依赖项。

红线代表错误。

虚线灰色线代表其他依赖关系。

在Spring工具窗口中浏览依赖项

Spring工具窗口可帮助您在Spring组件和依赖项之间进行导航。您可以查看项目中使用的Spring bean的定义，并查看它们与其他bean的关系。

要访问Spring工具窗口，请转到View |工具窗口|春天

Beans选项卡显示项目中使用的Spring bean的定义，文档以及它们与其他bean的关系。每个豆都标有图标 - （在XML文件中定义的bean）; （通过@Component注释声明的可自动发现的bean）; （由Spring添加的服务bean没有明确定义）。

使用左框架上的图标来更改bean的显示方式，查看文档或构建图形。

MVC选项卡允许您查看Spring MVC框架的控制器映射。你也可以使用 “请求方法”选项，用于通过HTTP方法筛选映射。

如果您的配置包含Spring Data，则“ 数据”选项卡将显示存储库列表，查询及其预测。

更改活动配置文件

Spring允许您将特定的上下文或bean映射到不同的配置文件 - 例如， 测试或生产 。这样，您可以在不同的环境中激活不同的配置文件。
如果您在项目中定义了至少一个配置文件，IntelliJ IDEA将在编辑器顶部显示一个特殊面板。您可以使用此面板查看当前配置文件名称并更改活动配置文件：

您还可以在Spring工具窗口中更改活动的配置文件。右键单击组件，然后选择“ 更改活动弹簧配置文件” 。

  1. 单击面板上的“ 更改配置文件 ”。

  2. 选择要将配置文件映射到的组件。这可以是整个项目，当前模块或上下文。

  3. 选择要将此组件映射到的配置文件。

如果要隐藏面板，请单击在上面。在下一个对话框中，清除“ 显示配置文件面板”复选框。
