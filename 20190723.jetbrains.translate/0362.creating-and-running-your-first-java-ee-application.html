开发Java EE应用程序

此功能仅在Ultimate版本中受支持。

本教程说明了Java EE应用程序开发工作流程。

我们要开发的应用程序将是最小的。它将是一个JSP页面Java Web应用程序。但是，此处显示的IntelliJ IDEA功能适用于任何复杂的Java EE应用程序。

在你开始之前

确保您的计算机上安装了以下软件：

  • IntelliJ IDEA ULTIMATE版。

  • GlassFish Server，版本3.0.1或更高版本。下载GlassFish 。（您可以使用任何其他支持Java EE的应用程序服务器。这里使用GlassFish作为示例。）

  • 一个Web浏览器。

创建一个项目

  1. 单击欢迎屏幕上的创建新项目 ，或选择文件|新的|项目 。

    “ 新建项目”向导将打开。

  2. 在左侧窗格中，选择Java Enterprise 。

  3. 指定要使用的JDK （ Project SDK字段）：从列表中选择一个，单击New并选择JDK安装文件夹，或单击Download JDK 。

  4. 指定应用程序服务器。（我们将使用GlassFish Server。）

    如果尚未在IntelliJ IDEA中定义GlassFish，请单击Application Server字段右侧的New ，然后选择Glassfish Server 。

    在Glassfish服务器对话框中，指定GlassFish Server安装目录。

  5. 在“ 其他库和框架”下 ，选中“ Web应用程序”复选框。

    单击下一步 。

  6. 指定新项目的名称（例如JavaEEHelloWorld ）。

    单击完成并在IntelliJ IDEA创建项目时等待。

探索项目结构

创建项目后，您将在“ 项目”工具窗口中看到与此类似的内容。

  • JavaEEHelloWorld是一个模块文件夹（在本例中与项目文件夹重合）。.idea文件夹和文件JavaEEHelloWorld.iml包含项目和模块的配置数据。文件夹src用于Java源代码。文件夹web用于应用程序的Web部分。此文件夹包含部署描述符WEB-INF/web.xml和文件index.jsp用作应用程序的起始页面。

  • 外部库包括用于处理GlassFish的JDK和JAR文件。

开发源代码

我们的应用程序将是一个JSP页面应用程序。它唯一的功能就是输出文本Hello，World！

  1. 打开index.jsp进行编辑：在Project工具窗口中选择文件，然后按F4 。

  2. 在<body>和</body>输入Hello, World!

    此步骤的代码已准备就绪。

运行应用程序

在工作区的右上角，单击 。

IntelliJ IDEA编译您的源代码并构建应用程序工件 。

之后，将打开“ 运行”工具窗口。IntelliJ IDEA启动服务器并将工件部署到其上。

最后，您的默认Web浏览器启动，您会看到应用程序输出Hello, World! 那里。

修改代码并观察更改

  1. 在index.jsp ，更改Hello, World! Hello! 。

  2. 在“ 运行”工具窗口中，单击“更新” 。

  3. 在Update对话框中，选择Update resources并单击OK 。（有关更多信息，请参阅应用程序更新选项 。）

  4. 切换到Web浏览器并重新加载页面以查看更改。

另请参阅更新应用程序服务器上的应用程

探索运行配置

在创建项目时，我们将GlassFish指定为应用程序服务器。因此，IntelliJ IDEA为GlassFish创建了一个运行配置。

当我们执行Run命令时（ ），我们开始运行配置。现在让我们看看运行配置，看看它的设置如何映射到我们刚刚观察到的事件。

  1. 单击运行配置选择器，然后选择“ 编辑配置” 。

    将打开“ 运行/调试配置”对话框，并显示GlassFish运行配置的设置。

    Before启动任务列表（在对话框的下半部分）指定应编译应用程序代码，并在执行运行配置之前构建相应的工件。

  2. 选择“ 启动/连接”选项卡以查看服务器在运行，调试和代码覆盖模式下的启动方式。

  3. 选择“ 部署”选项卡以查看服务器启动后部署的工件。

  4. 返回“ 服务器”选项卡。

    “ 打开”浏览器下的设置指定在启动后（即在服务器启动并将工件部署到其上之后），默认Web浏览器应该启动并转到指定的URL（ http://localhost:8080/JavaEEHelloWorld_war_exploded ）。

    “更新”操作右侧的设置指定单击时的设置在“ 运行”工具窗口中，应显示“ 更新”对话框，默认情况下应使用“ 更新资源”选项。（ 上次使用的更新选项成为默认选项 ）。

  5. 单击确定 。

探索工件配置

在创建项目时，我们表示我们将开发一个Web应用程序。因此，IntelliJ IDEA创建了用于构建Web应用程序工件的配置。我们来看看这个配置。

  1. 打开“ 项目结构”对话框： 文件|项目结构或Ctrl + Shift + Alt + S.

  2. 在“ 项目设置”下 ，选择“ 工件” 。

    可用的工件配置显示在右下方的窗格中和 。（目前只有一种配置。）

    工件设置显示在对话框的右侧部分。

    类型。 工件类型是Web Application：Exploded。这是一个解压缩的Web应用程序归档（WAR），一种可以部署到Web服务器上的目录结构。

    输出目录。 构建时，工件将放入<project_folder>/out/artifacts/JavaEEHelloWorld_war_exploded 。

    输出布局。 工件结构显示在“ 输出布局”选项卡的左侧窗格中。

    <output root>对应于输出目录。其他元素具有以下含义：

    • 'JavaEEHelloWorld'编译输出表示编译的Java类，其源位于src目录中。它们放在输出目录中的WEB-INF/classes中。

    • “Web”构面资源代表web目录的内容。

将应用程序打包到WAR文件中

当您对应用程序感到满意时进入舞台，您可能希望将其置于WAR （Web应用程序存档）中。为此，您应该创建适当的工件配置，然后构建工件：

  1. 点击 ，指向Web Application：Archive并选择For'JavaEEHelloWorld：war explosion' 。

    将创建一个新的工件配置，其设置将显示在对话框的右侧。

  2. 为存档创建清单文件：单击Create Manifest并同意IntelliJ IDEA建议的位置（ web/META-INF/MANIFEST.MF ）。

  3. 在Project Structure对话框中单击OK 。

  4. 选择Build |构建工件 。

  5. 在Build Artifact弹出窗口中，指向JavaEEHelloWorld：war并选择Build 。

    现在，如果您查看out/artifacts/JavaEEHelloWorld_war文件夹，您将在那里看到存档。

将应用程序打包到EAR中：使用Java EE Application支持

要将Java EE应用程序打包到EAR中 ，您应该：

  1. 创建Java EE 部署描述符 application.xml 。

  2. 配置EAR 工件 。

  3. 构建该工件。

正如我们将要看到的，IntelliJ IDEA为您执行大部分这些任务，作为其Java EE应用程序支持的一部分：

  1. 在“ 项目”工具窗口中，右键单击模块文件夹，然后选择“ 添加框架支持” 。

  2. 在打开的对话框中，选中JavaEE Application复选框，然后单击OK 。

    请注意在模块文件夹中创建的描述符文件META-INF/application.xml 。

  3. 在编辑器中打开文件（ F4 ）。

    目前，该文件几乎是空的。

  4. 现在让我们看一下工件配置。

    请注意，出现了一个新配置，一个用于爆炸EAR工件的配置。

    目前，工件中仅包含JavaEE应用程序构面资源（ META-INF/application.xml ）。

  5. 让我们将爆炸的WAR工件的副本添加到EAR工件结构中。为此，在“ 可用元素”下 ，展开“ 工件”节点，然后双击爆炸的WAR工件。这是结果。

    （获得相同结果的另一种方法是 |神器| JavaEEHelloWorld：战争爆发了 。）

  6. 请注意消息Web facet未在application.xml中注册 。单击修复 。（稍后，我们将通过此快速修复查看对application.xml所做的更改。）

  7. 为EAR工件创建配置： | JavaEE应用程序：存档|对于'JavaEEHelloWorld：耳朵爆炸' 。

  8. 要创建清单文件，请单击“ 创建清单”并同意默认文件位置（ <project_folder>/META-INF/MANIFEST.MF ）。

  9. 在Project Structure对话框中单击OK 。

    看到您的application.xml已更改。这是应用快速修复的结果。

    现在让我们创建一个用于构建和部署EAR工件的运行配置。

  10. 单击运行配置选择器，然后选择“ 编辑配置” 。然后，在“ 运行/调试配置”对话框中，选择 | GlassFish Server |远程 。

  11. 为运行配置指定描述性名称，例如GlassFishRemoteEAR 。

  12. 在部署列表中包含EAR工件：切换到“ 部署”选项卡并选择 |神器| JavaEEHelloWorld：耳朵 。

    请注意， Build'JavaEEHelloWorld：ear'工件任务自动包含在Before启动任务列表中。

  13. 切换到“ 服务器”选项卡，然后在“ 打开浏览器”部分中检查URL。http://localhost:8080/的部分应该与application.xml的<context-root>元素相对应。

  14. 在“ 运行/调试配置”对话框中单击“ 确定 ” 。

  15. 执行运行配置（ ）。

    和以前一样，在“ 运行”工具窗口中打开另一个选项卡，显示运行配置输出。

    然后，应用程序输出显示在浏览器中。
