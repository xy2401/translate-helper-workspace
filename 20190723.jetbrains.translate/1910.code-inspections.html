代码检查

此功能在PyCharm的教育版中不可用。

IntelliLang包含一组检查，用于验证所提供注释（或自定义配置注释）的正确使用。可以通过常规设置|配置这些检查错误配置对话框。

语言注入

此类别中的检查适用于与@Language注释相关的语言注入功能。

  • 未知语言ID
  • 语言不匹配
  • 模式验证
  • 验证带注释的模式
  • 模式注释不适用
  • 非注释方法会覆盖@Pattern方法

未知语言ID

此检查为使用不存在的语言ID提供了验证。它标记@Language值属性的错误值的用法，例如@Language("NonExistingID") 。

语言不匹配

验证是否使用对注释为包含不同语言或根本没有注释的元素的引用。检查提供了一个Quick-Fix，用于为预期语言添加正确的注释来注释这些元素。

注射不适用

此检查检查@Language或任何派生的注释是否用于除String或String[]类型的元素之外的任何内容。

模式验证

此类别包含有关验证@Pattern或其派生注释的使用的检查。

  • 验证带注释的模式
  • 模式注释不适用
  • 非注释方法会覆盖@Pattern方法

验证带注释的模式

此检查验证表达式（字符串文字，以及其他编译时常量或替换表达式）是否与@Pattern注释所需的模式匹配。检查可以选择忽略包含非替换引用的非常量表达式，并提供Quick-Fix以在适用的情况下添加替换。

模式注释不适用

检查模式验证注释（ @Pattern或派生的注释）是否有效应用于带注释的元素。只能对String类型的元素进行注释。

非注释方法会覆盖@Pattern方法

此检查检查没有任何@Pattern或派生注释的方法是否从其基类覆盖带注释的方法。这对于编辑器中的错误突出显示不是必需的，但是运行时检查检测不会从基类方法中获取注释。

提供了一个Quick-Fix来添加一个与基类方法相匹配的注释。这可确保运行时检查工具正常工作。
