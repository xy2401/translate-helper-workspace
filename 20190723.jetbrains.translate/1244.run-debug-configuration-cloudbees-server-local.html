发布后

选中此复选框可在启动服务器并部署工件后启动Web浏览器。

从列表中选择浏览器。点击 （ Shift + Enter ）配置您的Web浏览器。

使用JavaScript调试器

如果选中此复选框，则启动Web浏览器并启用JavaScript调试器。

请注意，JavaScript调试仅适用于Firefox和Google Chrome。当您第一次在Firefox中调试JavaScript时，会安装JetBrains Firefox扩展。

下面的字段发布后

指定浏览器启动时应该访问的URL。最有可能的是，默认的http://localhost:8080会这样做。

VM选项

如有必要，请指定要在服务器启动时传递到服务器JVM的命令行选项。

如果您需要更多空间来键入，请单击旁边的字段打开VM选项对话框，在文本输入面积较大。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

    - Dfoo = $ { MY_ENV_VAR }

在'更新'动作

为更新应用程序功能选择必要的选项（ 或运行或调试工具窗口中的N / A ）。

  • 热插拔类。 更改的类将在运行时重新编译并重新加载。此选项仅适用于调试模式。

  • 重新部署。 重建并重新部署应用程序工件。

  • 重启服务器。 服务器重新启动。重建并重新部署应用程序工件。

显示对话框

如果要在每次使用“ 更新”应用程序功能时都看到“ 更新”对话框，请选中此复选框。

“ 更新”对话框用于在实际更新应用程序之前选择“ 更新”选项。

在帧停用时

指定从IDE切换到其他应用程序（例如，Web浏览器）时IntelliJ IDEA应执行的操作。（帧停用意味着切换到不同的应用程序。）

“不执行任何操作”以外的选项与“ 更新”操作的含义相同。

JRE

默认情况下，项目JDK用于运行应用程序。如果要在此处指定备用JDK或JRE，请从下拉列表中选择它。

HTTP端口

服务器HTTP端口。

JNDI端口

服务器JNDI端口。

部署选项卡

指定要在服务器上部署的工件或外部资源。（外部资源是指可部署的Web组件，例如WAR文件，它不是由项目工件表示的。）

部署列表中应该只有一个项目。不能使用“爆炸”工件。

要将项目添加到列表，请单击并执行以下操作之一：

  • 要添加工件，请选择工件，然后在打开的对话框中选择所需的工件。

  • 要添加外部资源，请选择“ 外部源”，然后在打开的对话框中选择所需资源的位置。

（ Alt + Delete ）从列表中删除所选项目。

（ F4 ）打开所选工件的“ 项目结构”对话框的“ 工件”页面 。

日志选项卡

使用此选项卡指定在运行或调试时生成的日志文件应显示在控制台中，即“ 运行”或“ 调试”工具窗口的专用选项卡上。

日志文件输入

此列中的只读字段列出要显示的日志文件。该列表可以包含：

  • 特定文件的完整路径。

  • Ant模式 ，用于定义要显示的文件范围。

    如果日志条目模式定义了多个文件，则选项卡标题显示文件的名称而不是日志条目别名。

将控制台输出保存到文件

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

在启动选项之前

在此区域中，您可以指定在启动所选运行/调试配置之前必须执行的任务。任务按照它们在列表中出现的顺序执行。

项目

捷径

描述

Alt +插入

单击此图标可添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择要运行的一个或多个应用程序。如果尚未在IntelliJ IDEA中定义，请添加其定义。有关更多信息，请参阅配置第三方工具和外部工具 。

  • 运行另一个配置 ：选择执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • Build ：选择编译指定的模块。将执行Build Module命令 。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建项目 ：选择编译整个项目。Build Project命令将被执行。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建，无错误检查 ：与Build选项相同，但IntelliJ IDEA将尝试启动运行/调试配置，而不管编译结果如何。

  • 构建工件 ：选择此选项可构建工件或工件。在打开的对话框中，选择应构建的工件或工件。

  • 运行Ant目标 ：选择此选项以运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项以运行Grunt任务。在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项以运行Gulp任务。在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项以运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：在运行或调试之前添加要执行的Rake任务 。要选择Rake任务，请单击“浏览”按钮 ，然后从可用任务列表中选择所需任务。

    请注意，此处可以使用代码完成 。

  • 运行JRuby编译器 ：选择此选项以使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。
