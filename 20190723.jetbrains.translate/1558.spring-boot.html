开始使用Spring Boot

Spring Boot仅在Ultimate版本中提供。

Spring Initializr是一个向导，允许您在创建项目或模块时选择必要的配置。例如，您可以选择构建工具，并添加Spring Boot启动器和依赖项。

创建一个Spring Boot项目

  1. 从主菜单中，选择File |新的|项目 ，然后选择Spring Initializr 。

  2. 从Project SDK列表中，选择要在此Spring Boot项目中使用的JDK版本。如果未安装必要的版本，请参阅创建项目时配置JDK一节以了解如何下载和安装项目 。

  3. 输入要使用的Initializr服务的URL，或保留默认值。单击下一步 。

  4. 配置项目元数据：选择语言（Java，Kotlin或Groovy），构建工具（Maven或Gradle），并指定工件ID和版本。

    如果要在不同于项目JDK版本的Java版本上构建项目，可以在此处选择它。

    单击下一步 。

  5. 选择启动器和依赖项。如果选择需要其他插件的技术，IDE将在创建项目后通知您，并建议安装或启用它们。

因此， Spring Initializr将使用已定义的构建文件（ pom.xml或build.gradle ），源文件，测试文件和空的application.properties文件创建项目。

如果先前已配置高级代码格式设置，则它们将应用于预定义文件。如果您希望Spring Initializr使用默认代码格式设置创建项目，请打开“ 设置/首选项”对话框，转到“ 语言和框架”| “设置” 。春天| Spring Boot并禁用“ 重新格式化代码”选项。

配置自定义配置文件

Spring Initializr创建一个可能并不总是足以进行开发的默认配置文件。如果您不想使用默认配置文件，或者想要在不同环境中运行代码，则可以使用自定义配置文件。

为此，您必须让IntelliJ IDEA知道您项目中的哪些文件是配置文件。这将启用相关的突出显示和编码帮助。

  1. 转到文件|项目结构|方面 。

  2. 点击 （ 自定义Spring Boot ）在工具栏中。

  3. 如果要使用自定义配置文件而不是默认配置文件，请在搜索框中键入新自定义配置文件的名称。

    如果要使用多个配置文件，请单击按钮并从项目树中选择文件。

  4. 单击“ 确定”并应用更改。

    如果自定义配置文件设置正确，则会标记为图标。

某些自定义配置文件是自动配置的。例如，名称与当前命名模式匹配的特定于配置文件的配置文件将添加到上下文中。

监视Spring Boot端点

Spring Boot具有内置功能，允许您通过调用不同的端点（例如运行状况或Bean详细信息）来获取关键指标并监视生产环境中应用程序的状态。

在IntelliJ IDEA中，您可以在“ 端点”选项卡上查看端点。启动应用程序后，此选项卡将显示在“运行”仪表板上或“运行”工具窗口中。

“端点”选项卡使您可以导航到方法和bean。双击Mappings或Beans选项卡上的必要项（或按Enter键 ）以在编辑器中打开所选方法或bean。

在“运行”工具窗口中访问端点

  • 启动应用程序后，“ 运行”工具窗口将自动打开。

    如果您已关闭工具窗口并想要重新打开，请按Alt + 4或转到查看|工具窗口|跑 。

在“调试”工具窗口中访问端点

  • 一旦开始调试应用程序，就会打开“ 调试”工具窗口。

    如果您已关闭工具窗口并想要重新打开，请按Alt + 5 ，或转到查看|工具窗口|调试 。

在“运行”仪表板上访问端点

  1. 从主菜单中，选择“运行”|编辑配置 。

  2. 选择模板 。

  3. 在“ 运行仪表板”部分的“ 可用配置”下 ，单击“ 配置 ” 按钮，然后选择Spring Boot 。

  4. 在“ 运行/调试配置”对话框中，打开必要的Spring Boot配置，然后选中“ 启用JMX代理”复选框。

此外，在运行应用程序之前，请确保将org.springframework.boot.spring-boot-starter-actuator依赖项添加到模块中。

对于映射端点，还要添加org.springframework.boot.spring-boot-starter-web starter。

要显示或隐藏仪表板，请转到“ 查看”|工具Windows ，然后单击“运行仪表板” 。

Run仪表板允许您管理多个应用程序并监控其状态。如果在运行时检测到任何错误，仪表板将显示有助于识别和解决错误的信息。有关更多信息，请参阅管理多个运行配置 。

当您运行应用程序时， 图标显示在bean声明的装订线上，表明在上下文中找到了bean。您可以使用此图标查看依赖和注入的bean。

如果未显示图标，请确保在“运行/调试配置”对话框中选中“ 启用JMX代理”复选框。

在Beans选项卡上，您可以使用装订线中的图标来更改显示Bean信息的方式。例如，您只能查看您的bean并禁用显示库bean（ ），您可以启用和禁用显示文档（ ）， 等等。每个图标都有一个标识它的工具提示。

查看运行时bean之间的依赖关系

要使用图表，必须启用捆绑的UML插件。

Spring Runtime Beans图可帮助您可视化Spring Boot应用程序的运行时bean之间的依赖关系。

您可以在访问图端点 | “ 运行”仪表板 ，“ 运行”工具窗口或“ 调试 ”工具窗口中的“ Bean”选项卡。启动应用程序，然后单击Diagram Mode图标。

访问HTTP请求映射

在开始之前，请确保在“运行/调试配置”对话框中选中“ 启用JMX代理”复选框，并将org.springframework.boot.spring-boot-starter-actuator依赖项添加到构建文件中。

运行Spring Boot Web应用程序后， 图标将显示在带有@RequestMapping注释的方法的装订线中，以指示它们已被Spring注册为处理程序。

单击此图标以选择下一个操作。您可以立即运行请求，也可以在编辑器中使用HTTP扩展在临时文件中打开它，如果需要，可以在其中编辑它，然后通过REST客户端运行。对于具有@GetMapping注释的方法，您还可以在浏览器中打开映射的URL。

您也可以访问请求映射端点 | “ 运行”仪表板 ，“ 运行”工具窗口或“ 调试 ”工具窗口中的“ 映射”选项卡。单击必要的请求，然后选择下一个操作：

使用devtools配置应用程序更新策略

使用spring-boot-devtools模块，每次类路径上的文件发生更改时，应用程序都将重新启动。但是，如果您希望应用程序仅在特定时刻重新启动，则可以设置触发器文件。在这种情况下，只有在修改触发器文件后，应用程序才会重新启动。

IntelliJ IDEA提供了更多正在运行的应用程序的更多选项。启用devtools后， 更新<应用程序名称>应用程序 选项（ Ctrl + F10 ）在运行仪表板的左侧框架，运行/调试工具窗口以及主菜单（ 运行|更新<应用程序名称>应用程序 ）中可用。根据您的需要，您可以配置单击此选项后IDE将执行的操作。

您可以使用不同的更新策略创建多个运行/调试配置，并在它们之间切换。

  1. 从主菜单中，选择“运行”|编辑配置 。

  2. 选择必要的Spring Boot配置，或创建一个新配置（单击并选择 春季启动 ）。

  3. 指定必须应用此配置的主类。

  4. 从“更新”操作列表中，选择一个策略。您可以仅更新资源，更新类和资源（构建应用程序），更新触发器文件（重新启动）或执行类热交换，如果失败，IntelliJ IDEA将更新触发器文件（重新启动）。

从On frame deactivation列表中，您还可以选择切换到另一个应用程序后IDE将执行的操作。IntelliJ IDEA可以更新资源或构建应用程序。

对于更新触发器文件和热交换和更新触发器文件（如果策略失败） ，IDE会在您启动应用程序时设置触发器文件。
