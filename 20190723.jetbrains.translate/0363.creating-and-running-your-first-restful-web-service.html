在GlassFish Application Server上创建和运行您的第一个RESTful Web服务

此功能仅在Ultimate版本中受支持。

本教程演示了如何开发一个简单的RESTful Web服务并将其部署到GlassFish应用程序服务器。

  • 在你开始之前
  • 在IntelliJ IDEA中配置GlassFish服务器
  • 配置JDK
  • 创建一个项目
  • 探索项目结构
  • 开发源代码
  • 检查生成的工件配置
  • 探索并完成运行配置
  • 运行应用程序

在你开始之前

  1. 确保您使用的是IntelliJ IDEA ULTIMATE版。

  2. 安装Java SE Development Kit（JDK），版本1.8或更高版本，请参阅下载Oracle JDK 。

  3. 下载GlassFish应用程序服务器，版本3.0.1或更高版本，请参阅下载GlassFish 。

  4. 确保您的计算机上有可用的Web浏览器。

在IntelliJ IDEA中配置GlassFish服务器

  1. 按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，然后单击构建，执行，部署下的应用程序服务器 。

  2. 在打开的Application Servers页面上，单击在中央窗格上方，从列表中选择GlassFish Server 。

  3. 在右侧窗格中，在GlassFish Home字段中指定GlassFish Server安装文件夹。手动键入路径或单击并在打开的对话框中选择安装文件夹。IntelliJ IDEA检测应用程序服务器的版本并自动填写“ 名称”字段，如下所示： GlassFish <version> 。在我们的示例中，它是GlassFish 4.1.1 。

    其他字段自动填充或是可选的，因此只需单击“ 确定” 。

配置JDK

  1. 按Ctrl + Shift + Alt + S或选择文件|主菜单上的项目结构 。

  2. 在打开的“ 项目结构”对话框中，选择“ 平台设置”下的SDK 。

  3. 在打开的SDK页面上，单击在中央窗格上方，选择JDK 。

  4. 在右侧窗格中，指定要使用的Java SE Development Kit（JDK）的安装文件夹。手动键入路径或单击并在打开的对话框中选择安装文件夹。IntelliJ IDEA检测JDK的版本并自动在Name字段中输入它。在我们的例子中它是1.8 。

    其他选项卡中的所有必填字段都会自动填写，因此只需单击“ 确定”即可 。

创建一个项目

  1. 单击欢迎屏幕上的创建新项目 ，或选择文件|新的|项目在主菜单上。

    “ 新建项目”向导将打开。

  2. 在向导的第一个项目类别和选项页面：

    1. 在左侧窗格中，选择Java Enterprise 。

    2. 从Project SDK列表中，选择要使用的JDK。在我们的例子中，它是1.8 。

    3. 从Application Server列表中，选择GlassFish 4.1.1 。

    4. 从JavaEE Version下拉列表中，选择JavaEE 7 。

    5. 在Additional Libraries and Frameworks区域中，选中Web Application和RESTful Web Service复选框。

    6. 在“ 附加库和框架”列表下方的区域中选择“ 下载”选项。只有在选择了“ Web应用程序”和“ RESTful Web服务”复选框后，才会显示该区域。

    7. 单击下一步 。

  3. 在向导的第二个“ 项目名称和位置”页面上，指定新项目的名称，在我们的示例中为Rest_glassfish_hello_world 。

    单击完成并在IntelliJ IDEA创建项目时等待。

探索项目结构

创建项目后，您将在“ 项目”工具窗口中看到与此类似的内容：

  • rest_glassfish_hello_world是一个模块文件夹（在本例中与项目文件夹重合）。

  • .idea文件夹和文件Rest_glassfish_hello_world.iml包含项目和模块的配置数据。

  • 文件夹src用于Java源代码。

  • 文件夹web用于应用程序的Web部分。此文件夹包含部署描述符WEB-INF\web.xml 。

  • 外部库包括用于处理GlassFish Server的 JDK和JAR文件。

检查生成的工件配置

除了构建RESTful特定的项目结构外，IntelliJ IDEA还为我们配置了一个工件 。

IntelliJ IDEA中的单词工件可能意味着以下之一：

  • 工件配置，即为项目生成的输出规范;

  • 根据这样的规范（配置）生成的实际输出。

我们来看看这个配置。

  1. 按Ctrl + Shift + Alt + S或选择File |打开Project Structure对话框主菜单上的项目结构 。

  2. 在“ 项目设置”下 ，选择“ 工件” 。

    可用的工件配置显示在中央窗格下和 。目前只有一个配置rest_glassfish_hello_world:war exploded ，它是一个解压缩的Web应用程序归档（WAR），一个可以部署到Web服务器上的目录结构。

  3. 工件设置显示在对话框的右侧窗格中：

    了解有关使用工件和工件的工件的更多信息。

    IntelliJ IDEA已经填写了所有必填字段，我们不需要进行任何更改，因此只需单击取消即可退出对话框。

探索并完成运行配置

在IntelliJ IDEA中，任何应用程序都是根据专用的运行配置启动的 。在项目创建期间，我们已将GlassFish Server指定为运行应用程序的应用程序服务器。基于此选择和代码中的注释，IntelliJ IDEA创建了一个运行配置并填充了几乎所有必填字段。

  1. 在主菜单上，选择“运行”|编辑配置 。

  2. 在打开的“ 编辑配置”对话框中，展开“ GlassFish Server”节点，然后单击“ GlassFish 4.1.1” 。右侧窗格显示自动生成的运行配置的设置。

    • Application Server字段显示GlassFish 4.1.1 ，它是在项目创建期间选择的GlassFish Server的安装。Name字段还显示GlassFish 4.1.1 ，IntelliJ IDEA在指定的应用程序服务器之后自动命名生成的配置。

    • 在“ 打开”浏览器区域中，选中“ 启动后”复选框，因此将自动打开包含应用程序输出的页面。

      在下面的字段中，我们需要指定要打开的页面的URL地址。在我们的示例中，它是http://localhost:8080/rest_glassfish_hello_world_war_exploded/helloworld 。

    • 要在启动运行配置时自动部署rest_glassfish_hello_world：war爆炸，必须将工件标记为部署。如果已成功完成项目创建步骤，则会自动将工件标记为部署。如果不是，IntelliJ IDEA将显示警告没有标记为部署的工件和修复按钮。

      单击“ 修复”时 ，IntelliJ IDEA将打开“ 部署”选项卡，其中rest_glassfish_hello_world：war爆炸将添加到服务器启动列表上的 “ 部署”中。
    所有其他字段都自动填写或是可选的，因此只需单击“ 确定”即可保存运行配置。

运行应用程序

点击在工具栏上。之后：

  1. IntelliJ IDEA编译您的源代码并构建应用程序工件。

  2. “ 运行工具”窗口打开。IntelliJ IDEA启动服务器并在其上部署工件。

  3. 最后，您的默认Web浏览器启动，您在那里看到应用程序输出Hello World 。
