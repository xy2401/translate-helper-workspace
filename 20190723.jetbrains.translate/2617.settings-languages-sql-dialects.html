此页面允许您指定在各种范围中使用的SQL方言（特定于DBMS的SQL版本）。

要使此页面可用，必须启用数据库工具和SQL插件。请参阅管理插件 。

方言设置

项目

描述

全局SQL方言

计算机上所有.sql和.ddl文件的SQL方言;可以在较窄的范围内重新定义 - 在项目级别，和/或单个文件和目录。

项目SQL方言

当前项目中所有.sql和.ddl文件的SQL方言。如果指定<None> ，则继承全局SQL方言。

路径/ SQL方言

单个文件和目录的SQL方言 - 如果与全局或项目方言不同。

如果本节中未指定任何内容，则项目中的所有.sql和.ddl都将继承项目方言，以及项目外的所有文件 - 全局方言。

要指定文件或目录的方言，请单击并在打开的对话框中选择文件或目录。然后点击或者SQL Dialect单元格，然后选择方言。

明确指定的方言以黑色显示。继承的方言（除非您关闭对话框）以灰色斜体显示。

方言选项

指定方言时，除了特定的方言外，您还可以选择：

  • <无>或<清除> 。结果，继承了更高层次的方言。

  • <通用SQL> 。 <Generic SQL>支持SQL关键字，表和列名称的完成和突出显示。禁用此选项的语法错误突出显示（文件内容显示为语法正确）。

例

比如，您计算机上的大多数SQL脚本文件都是针对PostgreSQL的。在当前项目中，您正在为Oracle开发脚本，但在项目的一个目录中有MySQL脚本。在这种情况下，您需要指定：

  • 全局SQL方言： PostgreSQL

  • Project SQL方言： Oracle

  • <PathToMySQLScriptsFolder> ：MySQL
