使用服务器运行/调试配置

此功能仅在Ultimate版本中受支持。

要在应用程序服务器上运行或调试Java EE或Web应用程序，需要应用程序服务器运行/调试配置 。

IntelliJ IDEA可以自动创建一个这样的配置。如果您在创建项目或模块时指定要使用的应用程序服务器，则会发生这种情况（请参阅创建项目或模块时定义服务器 ）。

您可以根据需要创建更多服务器运行/调试配置。

本地和远程运行配置

服务器运行/调试配置可以是本地的或远程的。

本地配置适用于计算机上安装的服务器。此类配置包括定义如何启动相应服务器的设置。因此，当您执行本地配置时，IntelliJ IDEA会启动服务器。

远程配置无法启动服务器。通常，它们用于在不同（远程）计算机上运行的服务器。如果您不希望运行配置启动（或停止）服务器，它们也可以用于本地安装的服务器（在您的计算机上）。

执行远程配置时，IntelliJ IDEA连接到服务器以便能够部署应用程序工件或执行其调试。那时，相应的服务器必须已经运行。

总而言之，主要区分本地和远程配置的不是服务器的物理安装位置（尽管这也很重要），而是服务器是否通过相应的运行配置启动（或停止）。

启动服务器运行配置时会发生什么

当您启动服务器运行/调试配置时，IntelliJ IDEA通常会执行以下操作：

  1. 执行Before启动任务。默认情况下，这些是Make和Build工件 ：IntelliJ IDEA编译项目并构建应用程序工件。Build Artifacts任务最初不存在于运行配置中，而是在您指定要部署到服务器上的工件时自动添加。

  2. 启动服务器（用于本地配置）或连接到服务器（用于远程配置）。（此时，将打开“ 运行”或“ 调试”工具窗口，以便您可以监视和控制该过程。）

  3. 将指定的工件（和/或项目外部的可部署组件）部署到服务器。

  4. 如果已指定，则启动Web浏览器并打开指定的URL（通常对应于应用程序的起始页）。

创建服务器运行/调试配置

  1. 打开“ 运行/调试配置”对话框（例如“运行”|“编辑配置” ）。

  2. 点击 （ Alt + Insert ），选择感兴趣的服务器（例如Tomcat Server ），如果可用，选择Local或Remote 。（请参阅本地和远程运行配置 。）

  3. 在对话框的右侧，指定运行/调试配置设置，然后单击“ 确定” 。（有关可用设置的信息，请参阅“ 运行/调试配置”对话框中相应的特定于服务器的主题）。

另请参阅创建和编辑运行/调试配置 。

启动服务器运行/调试配置

与任何其他运行配置一样，应用程序服务器运行/调试配置可以通常的方式启动。请参阅运行应用程序和启动调试器会话 。

或者，您可以使用“ 应用程序服务器”工具窗口 ：

  1. 打开Application Servers工具窗口（例如View | Tool Windows | Application Servers ）。

  2. 选择要使用的服务器运行/调试配置。

  3. 在工具窗口的工具栏上，单击：

    • 在运行模式下启动所选配置。

    • 在调试模式下启动所选配置。

    结果，将打开“ 运行”或“ 调试”工具窗口 。
