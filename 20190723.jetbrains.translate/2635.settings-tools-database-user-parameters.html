一般来说，只有问号（ ? ）被视为SQL语句中的参数。在此页面上，您可以指定应将哪些其他字符及其序列视为参数以及在哪些位置。

SQL参数的模式通过正则表达式指定。

描述

在控制台和SQL文件中启用

如果选中该复选框，则参数模式将应用于SQL（在SQL文件和数据库控制台中）。如有必要，使用范围可以限制在各个模式的级别。

如果未选中此复选框，则不会在SOL文件和控制台中使用这些模式，而不管为各个模式指定了哪个使用范围。

使用SQL注入在字符串文字中启用

如果选中该复选框，则参数模式将应用于使用SQL 注入的字符串文字。如有必要，使用范围可以限制在各个模式的级别。

如果未选中此复选框，则无论为各个模式指定了哪个使用范围，都不会在字符串文字中使用这些模式。

参数模式

该表显示了参数模式及其使用范围。

使用正则表达式指定模式。括号中的值被视为参数名称。最初可用的模式具有以下含义：

  • \？（\d+) - 一个问号后跟一个或多个数字，例如?69在这种情况下， 69将参数名称。

  • :(\w+) - 冒号后跟一个或多个单词字符，例如:x ， :value ， :parameter_1 。

  • %w+ - %后跟一个或多个单词字符，例如%xyz 。

  • \$\{([^$\{\}]*)\} - $ ，然后{ ，然后是除$ ， {或}之外的任何字符零次或多次，然后} ，例如${} ， ${value} 。

  • \$\(([^\)]+)\) - $ ，然后( ，然后任何字符除外)一次或多次，然后) ，例如$（x）。

  • \$(\w+)\$ - $ ，然后是一个或多个单词字符，然后是$ ，例如$x1$ 。

  • \#(\w+)\# - # ，然后是一个或多个单词字符，然后再# ，例如#field_3# 。

使用 （ Alt + Insert ）， （ Alt + Delete ）， （ Alt + Up ）和 （ Alt + Down ）添加，删除和重新排序模式。

要编辑模式或其使用范围，请单击模式并使用以下控件：

  • 在脚本中。 如果不应在SQL文件和数据库控制台中使用该模式，请清除此复选框。

  • 在文字中。 如果不应在SQL注入的字符串文字中使用该模式，请清除此复选框。

  • 全部 （链接文本可能不同）。单击该链接并取消选择不应使用该模式的语言。
