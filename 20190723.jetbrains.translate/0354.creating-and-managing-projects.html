IntelliJ IDEA 2019.1帮助

项目

创建新项目时，必须选择项目类型，配置SDK并指定项目名称和位置。

您还可以在“ 新建项目”向导中指定必要的设置，以便在第一次打开项目时，它将进行预配置。例如，IDE将加载库和创建文件。对于Maven或Gradle ，IntelliJ IDEA将定义任务并声明依赖关系。

在创建项目时配置JDK

要在IntelliJ IDEA中开发应用程序，您需要Java SDK（JDK）。JDK是一个软件包，包含库，用于开发和测试Java应用程序的工具（开发工具），以及用于在Java平台上运行应用程序的工具（Java Runtime Environment - JRE）。

JRE可以与JDK分开获得，但它不适合应用程序开发，因为它没有必要的组件，如编译器和调试器。

请注意，捆绑的JRE用于运行IDE本身，并不足以开发Java应用程序。在开始使用Java开发之前，必须获取并安装独立的JDK 。

除了JDK之外，一些框架还需要自己的SDK。例如，Android或Grails。

IntelliJ IDEA没有随JDK一起提供，因此如果您没有必要的JDK版本，则必须下载并安装它。

  1. 在New Project向导中，单击Project SDK列表下的Download JDK 。您将被重定向到Oracle官方网站 。

  2. 选择并下载必要的JDK版本。

  3. 运行Java安装向导并按照其步骤操作，直到安装Java。

  4. 单击Project SDK列表旁边的New ，然后选择JDK安装文件夹（主目录）。

    在Windows上，转到C：\ Program Files \ Java ，在macOS上转至/ System / Library / Java 。然后选择Java目录，例如， jdk1.8.0_144 （数字代表您已安装的JDK版本）。

    要确保您指定JDK目录的路径，请检查是否存在bin/javac可执行文件，因为JRE目录没有。

    您可以在处理项目时随时指定其他JDK版本，配置其他SDK以及更改SDK级别。

项目格式

在IntelliJ IDEA中，有两种类型的格式可以存储项目的配置 - 基于文件的格式格式（旧版）和基于目录的格式（默认和推荐）。

对于基于文件的项目，IDE将创建.ipr ， .iws和.iml文件。对于以基于目录的格式存储的项目，IDE将创建.iml文件和保留项目设置的.idea目录。

为项目中的每个模块创建.iml文件。有关更多信息，请参阅使用模块 。

如果要将项目的格式更改为基于目录，请转到“ 文件”|保存为基于目录的格式 。

如果要通过VCS与其他开发人员共享项目，请参阅IntelliJ IDEA支持文章如何在版本控制系统下管理项目 。

导入项目

要在IntelliJ IDEA中导入项目，请转到文件|新的|来自现有来源的项目 。

在IntelliJ IDEA中，您可以导入来自Eclipse和Flash Builder的项目 。您还可以导入源文件集合以从中创建新项目。

如果您要导入使用Maven或Gradle等构建工具的项目，则建议您导入关联的构建文件（ pom.xml或build.gradle ）。在这种情况下，IntelliJ IDEA将识别项目中的依赖项和工件，并将自动恢复其配置。

对于Maven和Gradle项目，如果通过VCS共享项目，请考虑在项目导入对话框中启用“ 外部存储项目文件”选项。

通常，每次在描述符文件中进行更改时，IntelliJ IDEA都会重新创建.iml文件，并导入这些更改。您可以避免将.iml文件置于版本控制之下，但在这种情况下，模块配置中的某些更改将不会被共享（例如，添加的facet）。使用Store生成的项目文件外部选项，生成的文件将单独存储，您不必从版本控制中排除文件。

项目设置

对于IntelliJ IDEA中的项目，您可以配置名称，指定SDK，设置语言级别和编译器输出。要更改项目设置，请转到文件|项目结构或按Ctrl + Shift + Alt + S ，然后选择项目 。

项目设置与每个特定项目一起存储为.idea文件夹下的一组XML文件。

并不是说更改只会应用于当前项目。如有必要，您可以配置将应用于所有新创建的项目的全局SDK，语言级别和编译器输出。要更改全局项目设置，请转到文件|其他设置|新项目的结构|项目 。

有关如何配置IDE的说明（包括代码样式，检查等），请参阅配置IDE 。

在IntelliJ IDEA中打开文件夹后，会将.idea子文件夹添加到IntelliJ IDEA存储其内部配置设置的位置，例如，项目代码样式或版本控制系统。
