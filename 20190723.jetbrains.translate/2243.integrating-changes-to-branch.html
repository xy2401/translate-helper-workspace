将更改集成到分支

PyCharm允许您将更改集成到选定的分支中，并将此类集成的结果提交到存储库。请执行下列操作：

  1. 在“ 版本控制”工具窗口中，切换到“ 存储库”选项卡 。

  2. 右键单击要集成的更改列表，然后选择Subversion |从上下文菜单集成到Branch ：

    如果要集成单独的文件而不是整个更改列表，请在“ 更改的文件”窗格中选择它们，然后选择“ Subversion”从上下文菜单中集成到分支 。

    如果您正在使用SVN 1.5或在服务器上，并在本地工作副本更高，选择变更列表相关的变更列表/文件（S）或更改的文件窗格，然后单击在工具栏上。

    将打开“ 集成到分支”对话框，其中显示源和目标分支的URL地址以及可用的本地工作副本列表。

  3. 从“ 集成到工作副本”列表中，选择要将所选更改列表集成到的本地工作副本的路径。

    要添加列表的路径，请单击按钮。

    要从列表中删除路径，请单击按钮。

    确保指定的工作副本目录在Subversion版本控制下！

  4. 要通过启用svn命令的--dry-run开关来预览合并结果，请选择Try merge，但不进行任何更改复选框。

    如果未选中此复选框，则会以静默方式合并源。

  5. 单击确定 。 “ 提交更改”对话框将打开

  6. 查看摘要，指定必要的选项，然后运行commit。
