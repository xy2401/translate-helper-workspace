IntelliJ IDEA 2019.1帮助

ColdFusion的

IntelliJ IDEA为ColdFusion文件提供了广泛的编辑支持，并为将应用程序部署到ColdFusion服务器提供了便利。

IntelliJ IDEA使用捆绑插件实现ColdFusion功能，可以通过清除IntelliJ IDEA设置的插件页面上的CFML支持复选框（ Ctrl + Alt + S ）来完全禁用。

应在您的计算机上下载并安装ColdFusion服务器 。

ColdFusion支持

ColdFusion文件标有图标。

ColdFusion支持包括：

  1. 编码协助：

    • 代码完成标签，属性，属性值，函数和当前作用域的变量，函数参数，Java类的功能和通过'createObject'函数创建的组件，用于'createObject'中的组件和Java类的名称，用于继承的方法一个组件。

    • 错误和语法突出显示。

    • 代码格式和折叠 。

  2. 浏览源代码的众多方法，其中包括：

    • 能够打开具有指定名称的文件或组件 。

    • 使用结构视图导航 。

    • 导航|声明 （ Ctrl + B ）。

  3. 搜索源代码的高级工具。

  4. 代码生成

    • 在文件创建期间基于文件模板生成代码存根。

    • 使用实时模板插入，扩展和生成代码块。

    • 通过意图动作创建各种应用程序元素

    • 能够创建行和块注释 （ Ctrl + / / Ctrl + Shift + / ）。

  5. 查看方法参数信息。

  6. 运行/调试 ColdFusion的配置 。

配置部署到ColdFusion服务器

  1. 按Ctrl + Alt + S打开“ 设置/首选项”对话框，然后转到“ 语言和框架”| ColdFusion 。

  2. 在打开的ColdFusion页面上，指定本地文件夹与应用程序源和服务器上的路径之间的映射。
    • 要添加新映射，请单击 （ Alt + Insert ）并在“ 目录路径”字段中指定具有应用程序源的本地文件夹。手动键入路径或单击并在打开的对话框中选择文件夹。

      在“ 逻辑路径”字段中，键入要将指定本地文件夹的内容部署到的服务器的URL地址。

    • 要从列表中删除映射，请选择映射并单击 （ Alt + Delete ）。
