移动命名空间对话框

只有安装并启用了PHP插件，该对话框才可用。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

当您选择要移动的PHP命名空间并选择Refactor |时，将打开该对话框在主菜单上或从选择的上下文菜单中移动 。

IntelliJ IDEA假设项目中的命名空间按照PSR0标准排列，并强制您在移动命名空间时按照此标准保留文件夹结构和命名空间层次结构。

当您指定名称空间以将名称空间移动到时，IntelliJ IDEA会自动更新“ 目标目标目录”字段，该字段显示与相关名称空间对应的文件夹的路径。

项目

描述

新命名空间名称

对话框打开后，该字段将显示所选命名空间的完全限定名称。指定新的命名空间名称。仅使用反斜杠 （ \ ）作为命名空间分隔符。

目标目标目录

对话框打开时，该字段显示与当前名称空间对应的文件夹的路径。

路径以以下格式显示：

... \ < 项目 根 > < 目前的 命名空间 的文件夹 相 对于 所述 项目 根 > \
指定新的命名空间名称时，路径会自动更新。但是，如果要将命名空间移动到另一个父命名空间，IntelliJ IDEA将不会建议正确的文件夹，除非您通过在“设置/首选项”对话框的“ 目录”页面上将相关文件夹标记为“ 源”来为命名空间结构指定根文件夹（ Ctrl + Alt + S ）。

执行以下操作之一：

  • 接受字段中显示的预选路径。

  • 从列表中选择另一个路径。所有这些都是从命名空间根目录或当前目录进行评估的，因此可以安全地选择其中任何一个。

  • 点击并在打开的对话框中选择一个文件夹。

  • 按F2并编辑预选路径。请记住，这可能会导致将来自动加载出现问题。

重构

单击此按钮以打开“使用相关命名空间移动文件”对话框，并指定要移动到新命名空间和新文件夹的类和文件。

对话框的上部窗格列出了与命名空间相关的类和文件的目标命名空间和文件夹。列表中的每个项目对应一个类/文件。将光标移动到项目时，底部窗格显示与其相关的文件的内容。

  • 要将类和相应的文件移动到目标命名空间和目标文件夹，请选中命名空间/文件夹旁边的复选框。

  • 要将所有项目添加到列表中或从列表中删除所有项目，请分别单击全 选或取消全选 。
