TypeScript工具窗口

查看|工具窗口|打字稿

只有在TypeScript页面上激活TypeScript语言服务时，PyCharm才会显示此工具窗口，如TypeScript中所述 。使用此工具窗口可以查看TypeScript语言服务检测到的错误。

错误

该选项卡列出了TypeScript语言服务检测到的代码中的差异。更改代码时，列表会动态更新。

默认情况下，列表仅包含活动编辑器选项卡中文件的错误，并且顶部显示该文件的完整路径。要显示整个项目中的错误，请按Show project errors toggle按钮在工具栏上。错误消息按检测到它们的文件分组。

要导航到相关代码，请双击相应的错误消息或选择它，然后在上下文菜单中选择“ 跳转到源 ”。

编译错误

默认情况下， Compile All代表整个项目中的编译所有TypeScript文件 。要更改此行为，请在TypeScript页面的“ 编译范围”字段中定义另一个默认范围。

单击时将打开选项卡并从列表中选择编译范围：

该选项卡列出了在所选范围中检测到的所有编译错误。此列表不受您对代码所做的更改的影响，仅在再次手动调用编译时才会更新。

错误消息按检测到它们的文件分组。要导航到相关代码，请双击相应的错误消息或选择它，然后在上下文菜单中选择“ 跳转到源 ”。

工具提示和快捷方式

描述

编

单击此按钮并选择要编译的文件范围。

  • 要在活动编辑器选项卡中编译文件，请选择文件的路径。

  • 要使用tsconfig.json文件中定义的tsconfig.json ，请选择所需作用域的路径。

  • 如果选择“ 全部编译” ，则默认情况下，编译器将在整个项目中运行。要更改此行为，请在TypeScript页面的“ 编译范围”字段中定义另一个默认范围。

配置

单击此按钮可打开Typescript页面并编辑TypeScript设置。

重新启动TypeScript服务

单击此按钮可清除选项卡并重新运行TypeScript语言服务。

显示项目错误

默认情况下，此切换按钮已释放，选项卡显示当前TypeScript文件中的错误。按此切换按钮可显示整个项目中的错误。

关闭Ctrl + Shift + F4

单击此按钮以终止编译器和TypeScript语言服务并关闭工具窗口。


展开全部
Ctrl + NumPad Plus
全部收缩
Ctrl + NumPad -

使用这些按钮可以展开或折叠所有节点。

全部清除

单击此按钮可从当前活动窗格中删除所有错误消息。

上下文菜单

上下文菜单对于“ 错误”和“ 编译错误”选项卡很常见。

跳转到源头

复制

选择此选项可将选定的错误消息与文件，行和检测到错误的列的信息一起复制。

安慰

该选项卡显示自打开工具窗口以来TypeScript语言服务的日志。
