IntelliJ IDEA 2019.1帮助

PHP CS Fixer

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

除了内置的编码帮助外，IntelliJ IDEA还通过与PHP CS Fixer工具集成来检查源代码，该工具可检测代码中的编码标准问题。

要使用IntelliJ IDEA而不是命令行的PHP CS Fixer，您需要在IntelliJ IDEA中注册它并将其配置为IntelliJ IDEA 代码检查 。在IntelliJ IDEA中安装并启用后，该工具可在任何已打开的PHP文件中使用，无需其他步骤即可启动它。每次更新文件时都会激活动态代码检查，从而轻松排除发现的问题。

PHP CS Fixer即时报告的错误和警告显示为弹出消息。在批处理模式下运行工具时，错误和警告将显示在“ 检查结果”工具窗口中。每条消息都有PHP CS Fixer前缀，以区别于IntelliJ IDEA内部检查。

您可以应用预定义规则或定义自己的自定义规则集 。

在你开始之前

确保启用了PHP插件。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

安装和配置PHP CS Fixer

PHP CS Fixer脚本可以用作本地脚本，与PHP解释器关联的脚本，或者声明为项目依赖项的脚本，并通过Composer安装，这是首选和推荐的方式。

代码质量工具脚本（PHP-CS-fixer.bat用于Windows或PHP-CS-定影液用于Linux和MacOS）包含对全系统的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统path 。

使用Composer安装PHP CS Fixer

在开始之前，请确保您的计算机上已安装Composer，并在Composer Dependency Manager中所述的当前项目中进行初始化。

当您使用Composer安装PHP CS Fixer时，IntelliJ IDEA会自动下载必要的脚本，在IDE中注册它们，并可选择启用和配置相应的代码检查 。

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择friendsofphp/php-cs-fixer 包 ，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

手动配置PHP CS Fixer

您可以使用手动下载的本地PHP代码质量工具脚本或与PHP解释器关联的脚本。可以有许多本地和远程PHP解释器，在“ 设置/首选项”对话框的PHP页面上指定的解释器被视为“ 项目默认” 。在配置远程PHP解释器或配置本地PHP解释器中了解有关配置PHP解释器的更多信息。

选择要使用的PHP CS Fixer脚本

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS修复程序”区域。从配置列表中，

    选择PHP CS Fixer脚本：

    • 要使用与特定远程PHP解释器关联的脚本，请选择此解释器的名称。

    • 要使用本地脚本，请选择“ 本地” 。在这种情况下，无论在项目中使用哪个PHP解释器（本地或远程），都将执行本地PHP CS Fixer。请注意，只能有一个PHP CS定影液本地配置，因为IntelliJ IDEA的运行脚本（PHP-CS-fixer.bat用于Windows或PHP-CS-定影液用于Linux和MacOS），其中包含的PHP引擎的路径。

    • 要使用与默认项目解释程序关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释程序” 。

配置本地PHP CS修复程序脚本

  1. 下载并安装PHP CS Fixer脚本。

    要检查PHP CS Fixer安装，请切换到安装目录并运行以下命令：

    php-cs-fixer --version

    如果该工具可用，您将收到以下格式的消息：

    PHP CS定影液2.13.0瑜珈的烧烤被法比安斯基Potencier和达留Ruminski

    要根据您自己的自定义编码标准检查代码，请创建它 。存储规则以及指向规则集根目录中的.php_cs或php_cs.dist文件。

  2. 在IntelliJ IDEA中注册本地PHP CS Fixer脚本：

    • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |质量工具 。

    • 在打开的“ 质量工具”页面上，展开“ PHP CS修复程序”区域，然后单击“确定” 在配置列表旁边。

    • 在打开的PHP CS Fixer对话框中，在PHP CS Fixer路径字段中指定php-cs-fixer.bat或php-cs- fixer PHP CS Fixer可执行文件的位置。手动键入路径或单击并在打开的对话框中选择相关文件夹。

      要检查到PHP-CS-fixer.bat或php-CS-定影液指定的路径确保的IntelliJ IDEA和PHP CS修复器之间的互动，也就是，该工具可以从IntelliJ IDEA的和IntelliJ IDEA的推出将收到来自它的问题报告，单击“ 验证”按钮。此验证等同于运行php-cs-fixer --version命令。如果验证成功通过，IntelliJ IDEA将显示有关检测到的PHP CS Fixer版本的信息。

配置与PHP解释器关联的PHP CS Fixer脚本

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS修复程序”区域，然后单击“确定” 在配置列表旁边。将打开“ PHP CS Fixer”对话框，其中显示左侧窗格中所有已配置的PHP CS Fixer脚本的列表，其中一个是Local类型，另一个是以与脚本关联的PHP解释器命名的。点击在工具栏上。

    请注意，目前IntelliJ IDEA仅支持PHP CS Fixer 2.8.0及更高版本。

  3. 在打开的远程解释器的PHP CS Fixer对话框中，选择远程PHP解释器以使用相关的脚本。如果列表中不包含相关解释器，请单击并在CLI解释器对话框中配置远程解释器 ，如配置远程PHP解释器中所述 。

    单击“ 确定”后 ，IntelliJ IDEA将返回到“ PHP CS修复程序”对话框，其中新的PHP CS修复程序配置已添加到列表中，右侧窗格显示所选的远程PHP解释程序，即与之关联的PHP CS修复程序的路径它，以及高级PHP CS Fixer选项。

配置高级PHP CS Fixer选项

IntelliJ IDEA允许您指定高级PHP CS Fixer选项，从而根据计算机的配置和使用的规则集微调PHP CS Fixer进程行为。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS修复程序”区域。在“ 每个文件的最大邮件数”字段中，设置要为文件报告的邮件总数的上限。超出此限制的所有消息都将被拒绝，并且IntelliJ IDEA将在执行检查后在编辑器选项卡或检查结果工具窗口中显示警告消息。

  3. 在“ 工具进程超时”字段中，指定IntelliJ IDEA等待PHP CS Fixer结果的时间长度，然后执行该进程以防止过多的CPU和内存使用。

将PHP CS Fixer配置为IntelliJ IDEA检查

手动配置PHP CS Fixer检查

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 检查 ” 。

  2. 在打开的Inspections页面上，展开PHP |质量工具节点，然后选中PHP CS Fixer验证旁边的复选框。

  3. 在页面的右侧窗格中，使用“ 选项”区域中的控件配置PHP CS Fixer工具：

    1. 从Severity列表中，选择PHP CS Fixer检查的严重性级别 。所选值确定IntelliJ IDEA处理检测到的差异的严重程度，并在检查结果中显示 。

    2. 从“ 范围”列表中，选择要将检查应用程序限制为的范围 。

    3. 要允许有风险的规则 ，即可以更改代码行为的规则，请选中“ 允许有风险的规则”复选框。默认情况下，不允许使用有风险的规则。

    4. 指定编码标准申请。

      • 要使用其中一个预定义的编码标准 ，请选择编码标准列表，指定编码样式以检查您的代码。该列表包含在主php-cs-fixer目录结构中声明的所有编码标准。

      • 要根据您之前定义的编码标准检查代码，请选择“ 自定义” 。单击“ 浏览” 在打开的“ 自定义编码标准”对话框中，在“ 路径规则”字段中为您自己的编码标准指定.php_cs或.php_cs.dist文件的路径 。手动键入路径或单击“ 浏览” 并在打开的对话框中选择相关文件夹。

使用Composer配置PHP CS Fixer检查

您可以在composer.json的scripts部分中包含有关默认和自定义PHP CS Fixer规则集的信息。安装或更新项目依赖项时，将检测指定的规则集，并自动启用PHP CS Fixer验证检查。

如果在composer.json的scripts部分中未指定规则集，则IntelliJ IDEA将另外检查项目根目录以找到具有.php_cs或.php_cs.dist默认名称的规则集。如果文件存在，它将自动选为检查的自定义规则集。

使用Composer配置PHP CS Fixer检查

  • 在composer.json的scripts部分中，将php-cs-fixer PHP CS Fixer启动命令添加到其中一个叶元素中。

    提供--rules参数以表示使用的编码标准。例如，添加以下记录将编码标准设置为PSR1 ：

    “scripts” ： { “php-cs- fixer ” ： “php-cs-fixer --rules = @ PSR1” }

    或者，提供--config参数和规则集文件的路径，该文件将编码标准设置为Custom ：

    “scripts” ： { “php-cs-fixer” ： “php-cs-fixer --config =。/.php_cs“ }

与团队共享自定义编码风格

  1. 将编码标准的根目录放在项目根目录下。

  2. 将PHP CS Fixer配置为IntelliJ IDEA检查 。

  3. 指定您的编码标准 。

  4. 确保在“ 检查”页面顶部选择了项目配置文件 。此类配置文件保存在特定项目的.idea目录中（例如， $ PROJECT_DIR / .idea / inspectionProfiles ）。有关详情，请参阅配置配置

  5. 在“ 设置/首选项”对话框的“ 版本控制”页面上，确保将.idea目录置于版本控制之下 。

在批处理模式下运行PHP CS Fixer

  1. 通过选择Code |来 运行检查 检查主菜单上的代码 。指定检查范围和配置文件。

  2. 在检验结果工具窗口中查看检验结果。通过PHP CS定影液报告的错误和警告的前缀PHP CS Fixer从IntelliJ IDEA的内部检查区分它们。

从PHP CS Fixer检查中排除文件

等待PHP CS Fixer响应超过PHP CS Fixer对话框中Tool tool timeout字段中指定的限制时，IntelliJ IDEA建议将该文件添加到忽略列表中 。

管理被忽略的文件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 语言和框架”| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS修复程序”区域，然后单击“ 显示忽略的文件”链接。

    • 要添加文件，请单击并在打开的对话框中找到所需的文件。

    • 要从列表中删除文件并让PHP CS Fixer再次处理它，请选择该文件并单击 。

    • 要从列表中删除所有文件，请单击“ 清除列表”按钮 。
