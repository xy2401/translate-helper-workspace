npm和Yarn

WebStorm与npm和Yarn包管理器集成，因此您可以从IDE内部安装，定位，升级和删除可重用代码包。Node.js和NPM页面提供了用于管理包的专用UI。当然，您也可以从内置终端的命令行执行此操作。

WebStorm还允许您运行和调试npm脚本 。WebStorm解析package.json文件，识别脚本的定义，在树视图中显示脚本，并允许您在树中的脚本及其在package.json文件中的定义之间进行导航。

WebStorm使用Yarn工作区检测项目，并索引工作区的package.json文件中列出的所有依赖项，但这些依赖项位于根node_modules文件夹中。

在你开始之前

  1. 下载并安装Node.js.请注意，npm也已安装，因此如果您要使用它，则需要执行初步步骤。

  2. 要使用Yarn，请按照纱线官方网站上的说明进行安装。

选择项目包管理器

WebStorm使用分配给这些管理器的当前系统路径的npm和yarn别名。如果选择另一个Node.js版本，WebStorm还会将npm别名路径更改为指向与所选Node.js版本捆绑在一起的npm版本。

使用WebStorm，您可以选择是否在项目中使用npm或Yarn。默认情况下，WebStorm建议使用npm。但是，如果使用yarn.lock文件打开项目并在yarn.lock上安装了Yarn，则WebStorm会自动将此项目的包管理器更改为Yarn。您还可以将Yarn设置为默认的WebStorm包管理器 。

为当前项目选择包管理器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。Node.js和NPM页面打开。

  2. 从包管理器列表中，选择npm或yarn以使用系统默认安装。要使用包管理器的自定义安装，请单击“ 选择” ，然后在打开的对话框中选择包管理器的相关路径。

    当您从“ 运行/调试配置：NPM”对话框的“ 包管理器”列表中选择“ Project别名时，WebStorm会自动使用所选的包管理器。每次调用Run'npm install' / Run'yarn install'命令或运行npm / Yarn脚本时，WebStorm也会使用所选包管理器的路径，有关详细信息，请参阅运行和调试脚本 。

将Yarn设置为默认的WebStorm包管理器

  1. 在“ 设置新项目”对话框（“ 文件”|“新项目的设置...” ）中，单击“ 语言和框架”下的“ Node.js”和“NPM ”。

  2. 在打开的Node.js和NPM页面上，从包管理器列表中选择yarn 。之后，每次创建新项目时，WebStorm都会将Yarn建议为默认值。

使用内置终端安装软件包

您可能知道，npm可以在全局或作为项目依赖项或开发依赖项安装软件包，请从npm官方网站了解更多信息。使用Yarn，您还可以在全局或作为项目依赖项安装软件包，有关详细信息，请参阅Yarn官方网站。

安装包

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install --global <package_name>或yarn global add <package_name>进行全局安装。

    • npm install --save <package_name> ， npm install --save-dev <package_name>或yarn add <package_name> --dev以将程序包安装为项目依赖项或开发依赖项。

从package.json文件安装所有依赖项

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm安装

    • 纱线安装

    因此，您可以从当前文件夹中获取package.json列出的所有依赖项。从编辑package.json了解更多信息。

在Node.js和NPM页面上安装和升级软件包

WebStorm显示Node.js和NPM页面上所有当前安装的软件包。要打开页面，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ Node.js”和“NPM ”。全局安装的软件包列在顶部。对于每个包，WebStorm显示其当前安装的版本和最新的可用版本。

安装包

  1. 点击在打开的“ 可用包”对话框中，选择所需的包。

  2. 默认情况下，安装最新的软件包版本。要安装另一个，请选中“ 指定版本”复选框，然后从列表中选择所需的版本。

  3. 默认安装是本地安装。要使用其他安装类型，请选中“ 选项”复选框并键入-g以进行全局安装，或键入--save / --save-dev以将程序包安装为依赖项/开发依赖项。从npm和Yarn官方网站了解更多信息。

查看包的位置

  • 将鼠标指针悬停在包名称上。WebStorm在工具提示中显示包的路径。

将软件包升级到最新版本

  • 在列表中选择包，然后单击 。

删除包

  • 在列表中选择包，然后单击 。

编辑package.json

当您从require或import语句调用文档查找时，WebStorm还会显示npm包的readme文件。

WebStorm可帮助您处理package.json文件中的项目依赖项，从而提供广泛的编码帮助：

  • 包名称的代码完成。

  • 有关最新可用软件包版本的信息。

  • 以前的软件包版本的代码完成。按Ctrl + Space或开始键入与最新版本不同的版本时，WebStorm会显示包含所有先前版本软件包的建议列表。

  • 快速查找包的文档。

要从package.json文件安装依赖项，请执行以下操作之一：

  • 在编辑器中打开相关的package.json文件，或在Project工具窗口中选择它，然后从上下文菜单中选择Run'npm install' 。

  • 单击弹出窗口中的“运行'npm install' ：

    当尚未安装依赖项或更改它们时，WebStorm会显示此弹出窗口。如果关闭弹出窗口或通过选择“不再显示”将其关闭，您仍然可以使用“ 运行'npm install”操作或内置终端来安装依赖项。

更新项目依赖项

  • 单击弹出窗口中的“运行'npm install' 。

    每次打开项目，从版本控制更新它或编辑package.json ，WebStorm都会显示此弹出窗口。

    WebStorm还运行检查，检查是否安装了dependencies或devDependencies的包，并且它们的版本是否与指定范围匹配。

    如果检查发现任何不匹配，则建议快速修复。要应用它，请按Alt + Enter并从列表中选择运行'npm install' ：

运行和调试脚本

您可以从编辑器中的package.json文件，专用npm工具窗口中的脚本树，根据专用运行配置启动npm或Yarn脚本，或者自动启动npm或Yarn脚本作为启动或启动 前任务。

脚本执行的结果显示在“ 运行工具”窗口中 。工具窗口显示npm或Yarn脚本输出，报告发生的错误，列出尚未找到的包或插件等。最后执行的脚本的名称显示在工具窗口的标题栏上。

在脚本调试会话期间，WebStorm打开调试工具窗口 ，您可以在其中单步执行程序 ， 停止和恢复程序执行， 在挂起时检查它 ， 查看实际的HTML DOM ，等等

启用调试脚本

  • 将$NODE_DEBUG_OPTION添加到package.json文件中的定义，例如：

    { “name” ： “application-name” ， “version” ： “0.0.1” ， “scripts” ： { “main” ： “node $ NODE_DEBUG_OPTION ./app-compiled.js” } }

从package.json运行和调试脚本

WebStorm允许您从package.json文件快速启动单个脚本。要运行或调试多个脚本，请使用运行配置或npm工具窗口。

您也可以按Ctrl + Shift + F10运行脚本。

  • 在编辑器中打开package.json文件，单击在脚本旁边的装订线中，从上下文菜单中选择“运行<script_name>”或“ Debug <script_name> ”。

从npm工具窗口运行和调试脚本

在Project工具窗口中选择package.json文件或在编辑器中打开它并在上下文菜单中选择Show npm Scripts时 ，将打开工具窗口。一旦调用npm或Yarn，它就会开始构建一个在调用它的package.json文件的scripts属性中定义的scripts树。如果项目中有多个package.json文件，则可以为每个文件构建一个单独的脚本树，并运行脚本而不删除以前构建的树。每棵树都显示在一个单独的节点下。

如果尚未打开npm工具窗口，则打开它

  • 在Project工具窗口中选择所需的package.json文件，或在编辑器中打开它，然后从上下文菜单中选择Show npm Scripts 。将打开npm工具窗口，其中显示根据所选或已打开的package.json文件构建的脚本树。

在已打开的npm工具窗口中构建脚本树

  • 在npm工具窗口中，单击在工具栏上，从列表中选择所需的package.json文件。默认情况下，WebStorm会在项目的根目录中显示package.json文件。

  • 如果您有另一个package.json文件，请单击Choose package.json并在打开的对话框中选择所需的package.json文件。WebStorm在其标题上添加一个新节点，其中包含所选package.json文件的路径，并在新节点下构建一个脚本树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的脚本进行排序

  • 点击在工具栏上，从菜单中选择排序依据 ，然后选择名称 。
    默认情况下，树按照在package.json中定义的顺序显示脚本（选项定义顺序 ）。

运行一个脚本

  • 双击脚本。

  • 在树中选择脚本，然后按Enter或在上下文菜单中选择“运行<脚本名称> ”。

调试一个脚本

  • 在树中选择脚本，然后从上下文菜单中选择Debug <script_name> 。

运行或调试多个脚本

  • 使用多选模式：按住Shift （对于相邻项）或Ctrl （对于非相邻项）键并选择所需的脚本，然后在选择的上下文菜单中选择“运行”或“ 调试 ”。

通过运行配置运行脚本

从npm官方网站和Yarn官方网站了解有关包管理器的命令行界面的更多信息。

从编辑器或npm工具窗口运行或调试脚本时，WebStorm会自动创建临时运行配置。除了使用它们，您还可以创建和启动自己的npm运行配置。

  1. 在主菜单上，选择“运行”|编辑配置 。

  2. 点击在工具栏上，从列表中选择npm 。运行/调试配置：npm对话框打开。

  3. 指定要执行的CLI命令，要运行的脚本（使用空格作为分隔符），以及定义这些脚本的package.json文件的位置。（可选）键入用于执行脚本的命令行参数 。

  4. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    （可选）指定Node.js特定的选项参数以及要传递给Node.js的环境变量 。

  5. 指定要使用的包管理器。如果选择Project别名，WebStorm将使用Node .js页面中的默认项目包管理器。您还可以选择相关的包别名（ npm或yarn ）或指定包管理器的自定义安装的显式路径。

  6. 从mtoolbar栏上的列表中选择新创建的运行配置，然后单击 。输出显示在“ 运行”工具窗口中 。

在启动时自动运行脚本

如果您有一些定期运行的脚本，则可以将相应的运行配置添加到启动任务列表中。任务将在项目启动时自动执行。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 工具”|启动任务 。

  2. 在打开的“ 启动任务”页面上，单击“确定” 在工具栏上。

  3. 从列表中，选择所需的npm运行配置。配置将添加到列表中。

    如果项目中没有适用的配置，请单击然后选择“ 编辑配置” 。然后在运行/调试配置：打开的NPM页面上定义具有所需设置的配置 。保存新配置时，它会自动添加到启动任务列表中。

将脚本作为启动前任务运行

  1. 选择Run |打开Run / Debug Configurations对话框在主菜单上编辑配置 ，从列表中选择所需的配置，或通过单击重新创建并选择相关的运行配置类型。

  2. 在打开的ddialog中，单击在Before启动区域中，从下拉列表中选择Run npm script

  3. 在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。
