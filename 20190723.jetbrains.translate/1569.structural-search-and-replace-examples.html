结构搜索和替换示例

一个声明

$ Statement $ ;

将出现次数增加到一定数量，您可以找到包含最多指定数量元素的语句序列。

方法调用

$ Instance $ 。$ MethodCall $ （ $ Parameter $ ）

此模板与方法调用表达式匹配。如果出现次数为零，则意味着可以省略方法调用。

@Deprecated $ Instance $ 。$ MethodCall $ （ $ Parameter $ ）

您可以使用此模板查找已弃用的方法，并将其用作创建其他带注释的方法模板的原型。特别是对于搜索方法调用已弃用的方法，您可以选择已经存在的模板方法调用已弃用的方法 。

有关如何创建搜索模板，请参阅结构搜索和替换 。

如果声明

if （ $ Expr $ ） { $ ThenStatements $ ; } else { $ ElseStatements $ ; }

在注释和/或字符串文字中搜索

考虑一个人想找到包含'foo'的评论或文字。搜索模板就像$SomethingWeWantToFind$或"$SomethingWeWantToFind$" 。如果想要找到包含某些特定单词的注释/字符串（例如，foo作为单词），则应将其指定为文本约束。

添加try / catch / finally代码

如果想要用try/catch/finally结构替换语句，可以建议使用以下一对搜索和替换模板。搜索模板是：

$ Statements $ ;

具有指定为约束的特定最大出现次数。

替换模板是：

试试 { $ Statements $ ; } catch （ Exception ex ） { }

查找实现某个接口的类或所有类的所有后代

请考虑以下搜索模板：

class $ Clazz $ 扩展 $ AnotherClass $ {}

要么

class $ Clazz $ 实现 $ SomeInterface $ {}

作为变量$ AnotherClass $或$ SomeInterface $的文本约束，指定基类或已实现接口的名称。

找到所有这些方法

要查找相同接口方法的不同实现，请使用以下搜索模板：

class $ a $ { public void $ show $ （）; }

为$show$变量指定文本约束，并启用选项此变量是搜索的目标 。

使用@Modifier查找包本地和实例方法

IntelliJ IDEA为类的包本地和实例字段建议预定义的模板。这些模板使用@Modifier注释，当无法使用自然语言表达方式表达搜索目标时，它有助于描述搜索目标。

但是，如果需要搜索包本地方法或实例方法，则必须自己创建相应的搜索模板，并应用@Modifier注释。

要指定使用visibility modifiers package local和instance查找所有方法的条件，请使用以下搜索模板：

class $ Class $ { @Modifier （ “packageLocal” ） @Modifier （ “Instance” ） $ ReturnType $ $ MethodName $ （ $ ParameterType $ $ Parameter $ ）; } }

在搜索中使用“包含在约束中”字段

现有示例使用以下模板：

记录 。 debug （ $ params $ ）;

放置if('_a) { '_st*; }其中_a和_st是变量， *表示零只或多个出现在包含的领域约束并选择完全吻合变量的反转条件复选框会导致搜索未包含在日志语句if语句。

搜索XML和HTML标记，属性及其值

搜索标记的最简单模板是<$a$/>

通过在变量$a$上设置约束，您可以指定要查找的变量。例如，如果指定text / regexp约束app.+ ，则会找到名称以app开头的标记。

用于搜索XML和HTML的更通用的模板是<$tag$ $attribute$="$value$"/>通过使用具有正确指定的搜索设置和约束的模板，您几乎可以找到XML或HTML中可能出现的任何内容。例如，如果为变量$attribute$指定text / regexp约束width ，则会找到具有width属性的所有标记。
