使用pipfile管理依赖项

Pipfile是Pipenv虚拟环境用于管理项目依赖关系的专用文件。此文件对于使用Pipenv至关重要。为新项目或现有项目创建Pipenv环境时，将自动生成Pip文件 。同样，当您第一次在PyCharm中使用Pipfile文件打开项目时，会自动配置Pipenv虚拟环境。

考虑从头开始创建依赖项列表的任务。

记录Pipfile中的依赖项以管理项目包

  1. 当PyCharm为新的pipenv虚拟环境创建Pip文件时，该文件如下所示：

    python_version参数是您在创建新的pipenv环境时指定的基本解释器的版本。packages部分是您可以列出项目所需packages的位置。

  2. 请注意，PyCharm建议您安装Pipfile特定插件。单击相应的链接以安装Tom's Obvious，Minimal Language（ TOML ）的插件。您将不得不重新启动PyCharm以启用该插件。重新启动后，您将看到确认已识别Pipfile格式的消息：

  3. 通过修改packages部分添加新的包依赖项。

    [ 包 ] django = “*”

  4. 每次修改Pipfile文件时，PyCharm都会建议以下操作之一：

    • pipenv lock - 将新要求记录到Pipfile.lock文件中。

    • pipenv update - 将新需求记录到Pipfile.lock文件，并在项目解释器上安装缺少的依赖项。

    这一步非常重要，因为PyCharm根据Pipfile.lock文件中记录的信息管理项目依赖性。因此，任何添加到Pipfile但未锁定的要求都将被忽略。

    让我们选择pipenv update命令来安装Django包。

  5. 确保Django位于已安装软件包的列表中。在Settings / Preferences对话框（ Ctrl + Alt + S ）中，展开Project <项目名称>节点，单击Project Interpreter ，然后检查包列表。

  6. 现在探索相反的工作流程。在“ 项目解释器”对话框中，单击添加Flask包。请注意，pipenv使用标准过程限制配置索引。所有可用的包都是从Pipfile中指定的源添加的。通过pipenv而不是通过pip安装，删除和更新软件包。

  7. 一旦添加了Flask包，关闭Available packages对话框并查看Pipfile 。[packages]部分现在看起来如下：

    [ 包 ] django = “*” flask = “*”

[packages]和[dev-packages]部分之间的区别在于[packages]定义了生产和开发环境的要求，而[dev-packages]仅列出了用于开发目的的要求。

如果不满足Pipfile中列出的任何要求，PyCharm会跟踪并建议您应用受影响的依赖项。
