无法连接到数据库

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

步骤1。检查您的网络设置

数据库可以在本地，服务器或云中运行。对于服务器和云数据库，您需要网络连接。要验证连接是否可用，请使用ping和telnet命令。

使用ping命令，可以确保可以从源计算机访问目标计算机。打开命令行并键入以下命令： ping -a <host_IP> ，其中-a是将地址解析为主机名的命令选项（如果可能）。如果将主机名与ping命令一起使用，则会将主机名解析为IP地址。例如， ping -a example.com解析为PING example.com (93.184.216.34) 。

ping - 一个 < host_IP >

使用telnet命令，您可以测试与远程计算机的连接并发出命令。如果将端口指定为telnet命令的参数，则可以测试与给定端口上的远程主机的连接。如果连接成功，您将看到以下消息： Connected to <host_IP> 。

telnet < host_IP > < port_number >

出于安全原因，DBMS通常会丢弃所有telnet连接。telnet命令允许您检查端口是否已打开以进行通信。

第2步。检查您的连接属性

每个数据库（MySQL，PostgreSQL，Oracle或任何其他供应商）都有自己的连接设置。大多数数据库包括连接设置：

  • 主机 ：存储数据库的计算机或其他设备的主机名。它可以是IP地址（ 127.0.0.1 ）或域名（ localhost ）。

  • 数据库 ：要连接的数据库的名称。您可以在数据库服务器的设置中找到数据库名称，也可以询问数据库管理员。在某些情况下，可以在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW DATABASES; 。

  • 用户 ：具有足够权限以对数据库执行操作的用户的名称。在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW GRANTS; 。

  • 密码 ：用户的密码。

  • 端口 ：标识主机之间连接点的数字。主机使用端口号来确定必须建立连接的应用程序，服务或进程。不同的数据库供应商为其数据库使用不同的端以下列表是默认端口号列表。

第3步。检查驱动程序版本

使用JDBC驱动程序，您可以与PyCharm中的数据库管理系统（DBMS）进行交互。每个DBMS都需要自己的JDBC驱动程序。确保驱动程序版本和DBMS版本彼此兼容。

从PyCharm，您可以下载所有支持的供应商的驱动程序。您可以在“ 驱动程序”列表中查看支持的供应商的完整列表（ Ctrl + Alt + S ）。或者，您可以将自己的驱动程序添加到现有供应商，或者为不在“ 驱动程序”列表中的供应商创建新的驱动程序条目。

下载驱动程序并选择驱动程序版本

要从JetBrains FTP服务器下载驱动程序 ，请从“ 驱动程序”列表中选择供应商，然后单击“ 下载”版本。 驱动程序文件窗格中的<version_number>链接。

要更改驱动程序版本，请单击该版本。 驱动程序文件窗格中的<version_number>链接，然后选择所需的驱动程序版本。

使用JDBC驱动程序创建与数据库的连接

如果在数据源列表中找不到数据库供应商的名称，请下载数据库管理系统（DBMS）的JDBC驱动程序，并在PyCharm中创建连接。使用JDBC驱动程序，您可以连接到DBMS并开始工作。

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ），然后单击数据源属性图标（ ）。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ）并选择驱动程序和数据源 。

  3. 单击驱动程序链接。

  4. 在“ 驱动程序文件”窗格中，单击“ 添加”图标，然后选择“ 自定义JAR” 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击“ 确定” 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 单击“ 应用”

  8. 返回创建的数据源连接。

  9. 指定数据库连接细节或者，将JDBC URL粘贴到URL字段中。

    要设置空密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  10. 要确保与数据源的连接成功，请单击“ 测试连接” 。

将JDBC驱动程序添加到现有连接

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ），然后单击数据源属性图标（ ）。

  2. 在“ 数据源和驱动程序”对话框中，选择要更改驱动程序的数据源，然后单击数据源设置中的“ 驱动程序”链接。

  3. 在“ 驱动程序文件”窗格中，单击“ 添加”图标（ ）并选择Custom JARs 。

  4. 导航到JDBC驱动程序的JAR文件，选择它，然后单击“ 确定” 。

  5. 在“ 类”字段中，指定要用于驱动程序的值。

  6. 单击“ 应用”

第4步。检查是否需要使用SSH或SSL连接

要使数据库连接更安全，某些服务需要使用SSH或SSL。对于某些服务，您需要应用某些JDBC选项以使用SSL或SSH隧道（例如，当您尝试连接到Heroku Postgres时 ）。

使用SSL连接到数据库

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ），然后单击数据源属性图标（ ）。

  2. 选择要更改连接设置的数据源配置文件。

  3. 单击SSH / SSL选项卡，然后选择使用SSL复选框。

  4. 在CA文件字段中，导航到CA证书文件（例如， ca.pem ）。

  5. 在“ 客户端证书文件”字段中，导航到客户端证书文件（例如， client-cert.pem ）。

  6. 在客户端密钥文件字段中，导航到客户端密钥文件（例如， client-key.pem ）。

  7. 要确保与数据源的连接成功，请单击“ 测试连接” 。

建议使用PEM证书。

对于自签名证书，在某些情况下使用受信任的根实体颁发的证书，在使用最新的JDBC驱动程序版本时可能会遇到错误。如果Java密钥库不接受证书链，则SSL连接可能会失败。作为临时解决方案，尝试降级JDBC驱动程序（例如，对于MySQL连接器，您需要切换到5.1.40版本。）

使用SSH连接到数据库

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ），然后单击数据源属性图标（ ）。

  2. 选择要更改连接设置的数据源配置文件。

  3. 单击“ SSH / SSL”选项卡，然后选中“ 使用SSH隧道”复选框。

  4. 在代理主机 ， 代理用户 ， 代理密码和端口字段中，指定连接详细信息。

  5. 从Auth类型列表中，选择一种身份验证方法：

    • 密码 ：使用密码访问主机。指定密码，然后选择保存密码复选框以将密码保存在PyCharm中。

    • 密钥对（OpenSSH或PuTTY） ：使用密钥对的SSH身份验证 。要应用此身份验证方法，您必须在客户端计算机上拥有私钥，并在远程服务器上拥有公钥。PyCharm支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，并在相应字段中键入密码（如果有）。要让PyCharm记住密码，请选中Save passphrase复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭据帮助应用程序管理的SSH密钥（例如，Windows上的Pageant或Mac和Linux上的ssh-agent ）。

  6. 要确保与数据源的连接成功，请单击“ 测试连接” 。

有关使用SSH密钥的详细信息，请参阅生成新的SSH密钥并将其添加到ssh-agent教程。

使用PuTTY创建SSH隧道（Windows）

  1. 下载并运行最新版本的PuTTY SSH和Telnet客户端（从https://www.putty.org/下载客户端）。

  2. 在PuTTY Configuration对话框中，导航到Connection | SSH |认证

  3. 在“ 用于身份验证的私钥文件”字段中，指定私钥文件的路径，然后单击“ 打开” 。

  4. 在命令行窗口中，指定用于SSH隧道的用户名，然后按Enter键 。不要关闭命令行窗口。

  5. 在“数据库”窗口（“ 查看”|“工具Windows”|“数据库” ）中，单击“ 数据源属性”图标（ ）在工具栏上。

  6. 选择要更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选中“ 使用SSH隧道”复选框。

  8. 从Auth类型列表中，选择OpenSSH config和authentication agent 。

  9. 在代理主机 ， 代理用户和端口字段中，指定连接详细信息。

  10. 要确保与数据源的连接成功，请单击“ 测试连接” 。

使用Pageant创建SSH隧道（Windows）

Pageant是PuTTY，PSCP，PSFTP和Plink的SSH身份验证代理。Pageant存储您的私钥，只要它正在运行，它就会向PuTTY或其他工具（如PyCharm）提供解锁的私钥。您可以在Windows任务栏中找到Pageant图标。

  1. 下载最新版本的Pageant（从https://www.putty.org/下载客户端）。

  2. 在Windows任务栏中，右键单击“Pageant”图标，然后选择“ 添加密钥” 。

  3. 在“ 选择私钥文件”对话框中，导航到私钥文件（PPK文件），然后单击“ 打开” 。

  4. （可选）输入私钥密码，然后按Enter键 。

  5. 在“数据库”窗口（“ 查看”|“工具Windows”|“数据库” ）中，单击“ 数据源属性”图标（ ）在工具栏上。

  6. 选择要更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选中“ 使用SSH隧道”复选框。

  8. 从Auth类型列表中，选择OpenSSH config和authentication agent 。

  9. 在代理主机 ， 代理用户和端口字段中，指定连接详细信息。

  10. 要确保与数据源的连接成功，请单击“ 测试连接” 。

使用ssh-agent创建SSH隧道（macOS和Linux）

在命令行中运行ssh-agent的所有命令。

  1. 确保ssh-agent正在运行。

    ssh - 代理商

  2. 将密钥添加到代理程序（在以下示例中，密钥路径为~/.ssh/id_rsa ）。

    ssh - 添加 〜/。 ssh / id_rsa

  3. （可选）在macOS上，可以在ssh-add命令中添加-K选项，以在密钥链中存储密码。在macOS Sierra及更高版本中，您需要使用以下文本在~/.ssh/创建config文件：

    主机 * UseKeychain 是 AddKeysToAgent 是 IdentityFile ~ /。 ssh / id_rsa

    如果.ssh目录中有其他.ssh ，请为每个密钥添加一个IdentityFile行。例如，如果第二个密钥具有id_ed25519名称，则添加IdentityFile ~/.ssh/id_ed25519作为第二个私钥的附加行。

  4. 列出所有添加的密钥。

    ssh - add - L.

  5. 在“数据库”窗口（“ 查看”|“工具Windows”|“数据库” ）中，单击“ 数据源属性”图标（ ）在工具栏上。

  6. 选择要更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选中“ 使用SSH隧道”复选框。

  8. 从Auth类型列表中，选择OpenSSH config和authentication agent 。

  9. 在代理主机 ， 代理用户和端口字段中，指定连接详细信息。

  10. 要确保与数据源的连接成功，请单击“ 测试连接” 。
