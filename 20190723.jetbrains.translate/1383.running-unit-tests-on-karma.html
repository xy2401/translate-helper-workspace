此功能仅在Ultimate版本中受支持。

Karma是一种用于测试客户端JavaScript的工具。Karma针对在真实浏览器中运行的应用程序执行测试，以确保测试结果的正确性和可信度。IntelliJ IDEA与Karma集成，因此您可以从IDE内部运行，调试和监控测试的覆盖范围。您可以在树视图中查看测试结果，并从那里轻松导航到测试源。测试状态显示在编辑器的测试旁边，可以选择快速运行或调试它。

在你开始之前

  1. 下载并安装Node.js.

  2. 如管理插件中所述，在插件页面上安装并启用Karma存储库插件。

安装Karma和插件

您还可以在Node.js和NPM页面上安装karma包，如npm和Yarn中所述 。

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install如果已经在package.json定义了Karma和所有必需的插件。

    • 要将Karma和所需的插件（例如karma-jasmine或jasmine-core ）安装为开发依赖项 ：

      npm install - save - dev karma npm install - save - dev < required_karma_plugin > < another_required_karma_plugin >

    在Karma官方网站上了解更多信息 。

生成Karma配置文件

Karma测试根据在交互模式下生成的karma.conf.js配置文件运行。如果您的项目中已经有karma.conf.js ，请跳过此步骤。有关Karma配置的更多详细信息，请参阅Karma官方网站 。

创建Karma配置文件

  1. 打开终端并根据您的操作系统键入以下内容之一启动karma.conf.js生成向导：

    • 对于macOS和Linux：
      ./node_modules/karma/bin/karma init

    • 对于Windows：
      npm install -g karma-cli
      业力初始化

  2. 回答向导的问题，指定要使用的测试框架以及自动捕获的浏览器。

    另请参见Karma文件：模式匹配 。

运行测试

另外，添加'ChromeHeadless'到browsers中karma.conf.js选项。

使用IntelliJ IDEA，您可以直接从编辑器快速运行单个Karma测试，或创建运行/调试配置以执行部分或全部测试。

从编辑器运行单个测试

  • 点击要么在左侧排水沟中，从列表中选择“运行<test_name> ”。

    由于测试状态图标，您还可以在编辑器中查看测试是否已通过或失败在左边的排水沟里。

创建Karma运行配置

  1. 在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。运行/调试配置：Karma对话框打开。

  2. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    （可选）指定Node.js特定的选项参数以及要传递给Node.js的环境变量 。

  3. 指定karma包的位置和karma.conf.js的路径。

  4. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  5. （可选）指定要传递给Karma的命令行选项，以覆盖karma.conf.js配置文件中的默认设置。

    例如，要在Headless Chrome中运行或调试测试， --browsers ChromeHeadless在Karma选项字段中键入--browsers ChromeHeadless 。有关详细信息，请参阅使用Headless Chrome进行自动测试 。

    要查看所有可用的CLI选项，请在终端 （ Alt + F12 ）中键入karma start --help 。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右侧。

  2. Karma测试服务器自动启动，无需您执行任何步骤。在“运行”工具窗口的“ 因果服务器”选项卡中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在Test Runner选项卡中 ，单击在工具栏上。IntelliJ IDEA将执行上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请在其上下文菜单中选择“运行<测试名称> ”。

有关详细信息，请参阅重新运行测试 。

导航

使用IntelliJ IDEA，您可以在文件和相关测试文件之间跳转，或者从Test Runner选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择Go To |测试或转到|从上下文菜单中测试主题 ，或者只需按Ctrl + Shift + T.

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源 ”。测试文件在编辑器中打开，光标位于测试定义中。

  • 对于失败的测试，IntelliJ IDEA会从堆栈跟踪中将您带到测试中的故障线。如果确切的行不在堆栈跟踪中，您将进入测试定义。

测试文件应遵循流行的命名约定，例如，具有.test. ， .spec.或_spec.后缀，应位于源文件旁边或test文件夹中。

调试测试

使用IntelliJ IDEA，您可以直接从编辑器快速开始调试单个Karma测试，或创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 点击要么在左侧排水沟中，从列表中选择Debug <test_name> 。

通过运行/调试配置启动测试调试

  1. 如上所述创建Karma运行/调试配置。

  2. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右侧。

  3. 在打开的调试工具窗口中，像往常一样继续： 逐步执行测试 ， 停止并恢复测试执行， 暂停时检查测试 ，等等。

监控代码覆盖率

使用IntelliJ IDEA，您还可以监控Karma测试涵盖的代码量。IntelliJ IDEA在专用工具窗口中显示此统计信息，并在编辑器中直观地标记已覆盖和未覆盖的行。要监控覆盖范围，您需要安装karma-coverage软件包并更新karma.conf.js 。

安装业力报道

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install --save-dev karma-coverage

将karma-coverage定义添加到配置文件中

  1. 在编辑器中打开karma.conf.js 。

  2. 找到reporters定义并将coverage添加到以下格式的值列表中：

    记者 ： [ '进展' ， '覆盖' ]

  3. 添加preprocessors定义并以以下格式指定coverage范围：

    预处理器 ： { '** / * .js' ： [ 'coverage' ]}

启动覆盖范围的测试

  1. 如上所述创建Karma运行/调试配置。

  2. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右侧。

    或者，使用编辑器中的测试图标快速运行特定套件或覆盖测试。

  3. 在Coverage工具窗口中监视代码覆盖率。

    每次运行Karma测试时，都会在磁盘上生成覆盖率报告。可以在配置文件中配置覆盖率报告的格式，例如：

    // karma.conf.js 模块 。 exports = function （ config ） { config 。 集 （{ ... //可选，配置报告者 coverageReporter： { type ： ' html ' ， dir ： ' coverage / ' } ... }）;};

    可接受以下type值：

    • html生成一堆带有注释源代码的HTML文件。

    • lcovonly生成一个lcov.info文件。

    • lcov生成HTML + .lcov文件。默认情况下应用此格式。

    • cobertura生成一个cobertura-coverage.xml文件，以便于Hudson集成。

    • text-summary生成一个紧凑的文本摘要，通常是控制台。

    • text生成详细的文本表，其中包含所有文件的覆盖范围。
