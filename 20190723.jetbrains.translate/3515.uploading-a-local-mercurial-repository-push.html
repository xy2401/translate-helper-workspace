WebStorm 2019.1帮助

将变化推向上游（推送）

请执行下列操作：

  1. 从主菜单中，选择VCS | Mercurial |推 。Push Commits对话框打开，显示所有Mercurial存储库（用于多存储库项目）， 并列出自上次推送以来在每个存储库中的当前分支中进行的所有提交。

    如果您的项目使用多个不同步控制的存储库，则默认情况下仅选择当前存储库。有关如何启用同步存储库控制的详细信息，请参阅版本控制设置：Mercurial 。

  2. 如有必要，可以通过单击修改远程存储库的路径。标签变为文本字段，您可以在其中键入新路径或按Ctrl + Space调用完成。

    如果存储库中没有遥控器，则会显示“ 定义远程”链接。单击此链接并在打开的对话框中指定远程名称和URL。

  3. 如果要在推送更改之前预览更改，请选择所需的提交。右侧窗格显示所选提交中包含的更改。您可以使用工具栏按钮来检查提交详细信息 。

    如果提交的作者与当前用户不同，则此提交将标记为星号。

  4. 如果要使用提交推送活动书签（默认情况下它们不会发送到远程存储库），请选择“ 导出活动书签”选项。

  5. 准备好后单击“ 推送”按钮，然后从下拉菜单中选择要执行的操作： push或push --force 。

您也可以按Enter或Alt + Enter切换到所选元素的编辑模式。

您可以按Ctrl + Q进行所选提交以显示额外信息，例如提交作者，时间，哈希和提交消息。

如果选择整个存储库，则所有提交的所有文件都将在右窗格中列出。

如果在多次提交中修改了相同的文件，则只有在选择这些提交或整个存储库时才会列出一次，如果为此文件调用差异查看器 ，则所有更改将一起压缩。

使用强制推动

当您运行push ，如果远程存储库包含您缺少的更改并且您要使用存储库的本地副本覆盖，Mercurial将拒绝完成操作。通常，在使用更改进行更新之前，需要执行pull以与远程同步。

--force push命令禁用此检查，并允许您覆盖远程存储库，从而删除其历史记录并导致数据丢失。

当您可能仍需要执行--force push时，可能会出现这样的情况：当您重新--force push推送的分支然后想要将其推送到远程服务器时。在这种情况下，当您尝试推送时，Mercurial将拒绝您的更改，因为远程引用不是本地引用的祖先。如果你在这种情况下执行pull ，你将得到两个你需要合并的分支副本。

除非绝对必要（例如，如果您不小心推送了一些敏感数据），应该避免重新推出推送分支并修改其历史记录。

使用--force会导致所有新分支被推到所有分支上，这可能会给您的团队造成混乱。

如果您决定强制推送重新分支的分支并且您正在团队中工作，请确保：

  • 没有人拉你的分支并对它进行了一些局部更改

  • 所有挂起的更改都已提交并推送

  • 您拥有该分支的最新更改
