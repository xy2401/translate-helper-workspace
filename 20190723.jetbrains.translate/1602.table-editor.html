数据编辑器

从“ 数据库”工具窗口 （对于DB数据源中的任何表）：

  • 在工具栏上（如果工具栏当前未隐藏）

  • 从上下文菜单中打开编辑器

  • F4

概观

数据编辑器提供用于处理表数据的GUI。它允许您对数据进行排序，过滤，添加，编辑和删除，以及执行其他相关任务。

工具栏控件，数据单元格和键盘快捷键的上下文菜单命令

通过工具栏上的控件，数据单元的上下文菜单命令以及相关的键盘快捷键访问大多数可用功能。

数据提取器：<current_format>

使用此按钮或命令打开一个菜单，您可以在其中选择数据的输出格式。

除输出格式外，还有以下选项和命令：

  • 允许换位。 对于分隔符分隔值格式（TSV，CSV）：如果表格已转置，并且您正在将所选单元格或行复制到剪贴板（例如Ctrl + C ），则选项将被转置（如图所示）否则，不转置（如原始表中）。

  • 跳过生成的列（SQL）。 对于SQL INSERT和UPDATE：复制或保存数据时，不要包含自动增量字段。

  • 添加表定义（SQL）。 对于SQL INSERT和UPDATE：复制或保存数据时，添加表定义（CREATE TABLE）。

  • 配置CSV格式。 打开CSV格式对话框 ，该对话框允许您管理分隔符分隔值格式（例如CSV，TSV）。

  • 转到脚本目录。 切换到存储将表数据转换为各种输出格式的脚本的目录。

查看查询

使用此按钮可以查看用于生成当前表视图的查询。要关闭显示查询的窗格，请按Escape 。

此图标提供对以下命令的访问：

  • 置。 打开或关闭转置的表格视图。（在转置视图中，行和列互换。因此，行显示为列，反之亦然。）

  • 重置视图。 重新排序或隐藏列或排序数据后，还原初始表视图。

  • 通过ORDER BY排序。 打开或关闭相应的选项。

    如果启用了Sort via ORDER BY选项，则您执行的所有排序操作都会反映在相应的SELECT语句中（添加或修改ORDER BY子句），该语句将立即执行。结果，整个表的数据由相应的数据库系统排序。

    如果要将与数据库的交互保持在最低限度（例如，当表非常大或数据库连接“慢”时），请不要打开此选项。

    如果此选项关闭，则数据将由IntelliJ IDEA“本地”排序，并且仅针对当前显示的行。

  • 行过滤器。 显示或隐藏过滤器框。

  • 设置。 打开“ 设置”对话框的“ 数据库”页面 ，以查看或编辑数据库，Hibernate和JPA控制台，数据编辑器和数据库工具窗口的设置。
