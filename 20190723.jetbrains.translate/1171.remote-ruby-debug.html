远程Ruby调试

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

在这个部分：

  • 介绍

介绍

IntelliJ IDEA使用专用的Ruby远程调试配置为Ruby和Rails应用程序提供远程调试的可能性。这样做，IntelliJ IDEA保持本地源和服务器端源之间的映射。

可以同时启动多个调试过程。这样，每个远程调试过程都在Debug工具窗口的单独选项卡中启动，其名称为<script name>的Remote debug 。您可以像往常一样处理每个调试会话。

远程调试脚本或应用程序

要远程调试脚本或应用程序，请执行以下常规步骤

  1. 启动远程调试会话。

    要在远程主机上运行的命令的通用格式如下：

    rdebug-ide --host 0.0.0.0 --port <端口号> --dispatcher-port <端口号> - $ COMMAND $

    您可以在Ruby远程运行/调试配置对话框中看到此命令作为提示。

    请注意，Ruby脚本和Rails应用程序之间存在差异。例如，Ruby脚本的命令将是：

    rdebug - ide - host 0 。 0 。 0 。 0 - 端口 1234 - 调度程序 - 端口 26162 - / home / user / RubymineProjects / remote - debug - example / math_wiz 。 RB

    Rails应用程序的命令将是：

    rdebug - ide - 端口 1236 - 调度程序 - 端口 26166 - 主机 0 。 0 。 0 。 0 - bin / rails s - b 0 。 0 。 0 。 0

    在这两种情况下，port都是远程主机上的端口号。 dispatcher-port是本地主机上的端口号。

  2. 创建要调试的脚本或应用程序的本地副本，然后设置所需的断点。

  4. 在IntelliJ IDEA中，选择所需的Ruby远程调试配置，在编辑器中打开Ruby脚本或Rails应用程序，然后单击在主工具栏上。

请注意，“ 连接到调试器”进度条中添加了一个取消按钮，为您提供了手动取消远程调试会话的方法，而不是等待超时。
