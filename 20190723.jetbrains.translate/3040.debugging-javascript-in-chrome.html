在Chrome中调试JavaScript

默认情况下，调试会话在具有自定义Chrome用户数据的新窗口中启动。要使用您熟悉的外观打开新的Chrome实例，请在WebStorm中配置Chrome以开始使用您的用户数据，有关详细信息，请参阅使用默认Chrome用户数据启动调试会话 。

WebStorm为与Chrome配合使用的客户端 JavaScript代码提供内置调试器。视频和下面的说明将指导您完成此调试器的基本步骤。

在开始之前，按照配置JavaScript调试器中的描述配置内置调试器 。要在浏览器中立即显示对HTML，CSS或JavaScript代码所做的更改而不重新加载页面，请激活实时编辑功能。有关详细信息，请参阅HTML，CSS和JavaScript中的实时编辑 。

调试在内置服务器上运行的应用程序

WebStorm有一个内置的Web服务器，可用于预览和调试您的应用程序。此服务器始终在运行，不需要任何手动配置。根据项目结构，所有项目文件都在内置服务器上提供，其根URL为http://localhost:<built-in server port>/<project root> 。

开始调试

  1. 根据需要在JavaScript代码中设置断点 。

  2. 打开引用要调试的JavaScript的HTML文件，或在“ 项目工具”窗口中选择HTML文件。

  3. 在编辑器或选择的上下文菜单上，选择Debug <HTML_file_name> 。WebStorm生成调试配置并通过它启动调试会话。该文件将在浏览器中打开，并显示“ 调试”工具窗口 。

  4. 在调试工具窗口中，像往常一样继续： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ， 查看实际的HTML DOM ，等等

要保存自动生成的配置以供进一步重用，请在调试会话结束后在上下文菜单中选择“ 保存<HTML_file_name> ”。

例

假设您有一个由index.html文件和index.js文件组成的简单应用程序，其中index.html引用index.js 。要使用内置服务器开始调试此应用程序，请在编辑器中打开index.html ，然后在上下文菜单中选择Debug'index.html' ：

WebStorm自动创建运行/调试配置 ，并启动调试会话：要重新启动新的运行/调试配置，请单击在WebStorm窗口的右上角或选择Run |在主菜单上调试 ：

调试在外部Web服务器上运行的应用程序

通常，您可能希望调试在外部开发Web服务器上运行的客户端JavaScript，例如由Node.js提供支持。

开始调试

  1. 根据需要在JavaScript代码中设置断点 。

  2. 在开发模式下运行该应用程序。通常你需要为此运行npm start 。开发服务器准备就绪后，在浏览器中复制运行应用程序的URL地址 - 您需要在运行/调试配置中指定此URL地址。

  3. 创建JavaScript Debug类型的调试配置：
    选择运行|在主菜单上编辑配置 ，单击在工具栏上，从列表中选择JavaScript Debug 。在“ 运行/调试配置：打开的JavaScript调试”对话框中，指定运行应用程序的URL地址。可以从浏览器的地址栏复制此URL，如上面的步骤2中所述。单击“ 确定”以保存配置设置。

  4. 在工具栏上的“ 选择运行/调试配置”下拉列表中选择新创建的配置，然后单击“确定” 在列表旁边。运行配置中指定的URL地址将在浏览器中打开，并显示“ 调试”工具窗口 。

  5. 在调试工具窗口中，像往常一样继续： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ， 查看实际的HTML DOM ，等等

有关示例，请参阅调试反应应用程序和调试角度应用程序 。

调试异步代码

WebStorm支持调试异步客户端JavaScript代码。WebStorm识别异步代码中的断点，停止它们，并让您进入这样的代码。只要异步函数内部出现断点或您进入异步代码，就会在“ 调试器”选项卡的“ 帧”窗格中添加一个Async call from <caller>的新元素Async call from <caller> 。WebStorm显示一个完整的调用堆栈，包括调用者和异步操作开始的整个方式。

下图显示了JavaScript调试会话的示例。

调试器在第3行（断点）停止，然后在第5行（断点）停止。单击Step into ，调试器将在第5行停止（在function ），然后将移至第6行。

默认情况下，异步调试模式处于打开状态。要禁用异步堆栈跟踪，请将Registry中的 js.debugger.async.call.stack.depth设置为0 。

调试工人

WebStorm支持调试服务工作者和Web工作者 。WebStorm识别每个工作程序中的断点，并在调试工具窗口的“ 调试 器”选项卡的“ 框架”窗格中将其调试数据显示为单独的线程。

请注意，WebStorm只能调试专用工作程序 ，目前不支持对共享工作程序进行调试。

调试工人

  1. 在Workers中设置断点以进行调试。

  2. 如果您使用的是Service Workers ，请确保选中“ 调试器”页面上的“ 允许未签名的请求”复选框。否则，您的服务工作者在调试会话期间可能不可用：

  3. 如上所述调试在外部Web服务器上运行的客户端JavaScript，创建JavaScript Debug类型的调试配置。

  4. 在工具栏的“ 选择运行/调试配置”下拉列表中选择新创建的配置，然后单击“ 调试” 。

    运行配置中指定的HTML文件将在所选浏览器中打开，并打开“ 调试工具”窗口，其中“ 框架”下拉列表将显示所有工作人员 ：

    要检查Worker的数据（变量，监视等），请在列表中选择其线程，并在Variables和Watches窗格中查看其数据。当您选择另一个Worker时 ，窗格的内容会相应更新。
