从TFS存储库签出

Checkout可帮助您获取存储库文件夹的本地工作副本，您可以根据需要进行编辑。进行必要的更改后，您可以通过提交或检入存储库的更改来发布结果。

TFS服务器与本地项目之间的交互是通过工作空间配置的，这些工作空间将存储库中的文件夹与计算机上的副本进行映射。您可以根据现有工作空间的映射下载远程文件夹，也可以在下载期间定义所需的映射，并自动生成相应的工作空间。在任何一种情况下，都使用TFS向导Checkout。结账时，将围绕下载的源创建WebStorm项目。

在本页面：

  • 将数据下载到自动生成的新工作区
  • 将数据下载到现有工作区

生成新工作区

  1. 在主菜单上，选择VCS |从版本控制检出| TFS 。从TFS向导启动Checkout。

  2. 在“结帐模式”页面上，选择“ 自动创建工作空间”选项，指定用于标识工作空间的名称，然后单击“ 下一步” 。

  3. 在“源服务器”页面上，指定要从中下载数据的服务器。执行以下操作之一：

    • 选择一个现有的服务器访问配置。

    • 单击“ 添加”，然后在“打开的添加Team Foundation Server”对话框中定义新的服务器访问配置 。

    准备好后单击“ 下一步” 。

  4. 在“ 选择源和目标路径”页面上，定义工作文件夹映射以生成工作区：

    1. 在“ 源路径”区域中，选择要下载的远程文件夹。

    2. 在“ 目标路径”文本框中，指定存储下载数据的位置。手动键入路径或单击“ 浏览” 并在对话框中选择该文件夹，该文件夹将打开。

    单击下一步 。

  5. 在“ 摘要”页面上，检查要生成的工作空间的详细信息，然后单击“ 完成”以启动结帐过程。结帐完成后，WebStorm会围绕下载的源创建项目并建议打开它。

将数据下载到现有工作区

  1. 在主菜单上，选择VCS |从版本控制检出| TFS 。从TFS向导启动Checkout。

  2. 在“结帐模式”页面上，选择“手动选择工作区”选项，然后单击“ 下一步”

  3. 在“ 源工作区”页面上，选择要从中下载数据的服务器以及此服务器的工作区以添加工作文件夹。如有必要，请创建新的服务器访问配置和工作区 。

    准备好后单击“ 下一步” 。

  4. 在“ 选择源路径”页面上，指定要下载的远程文件夹。仅当所选工作空间包含文件夹本身或其父项的映射时，此类文件夹才可用。如果缺少此类映射，请返回“ 源工作区”页面并添加所需的映射 。

    准备好后单击“ 下一步” 。

  5. 在“ 摘要”页面上，检查要生成的工作空间的详细信息，然后单击“ 完成”以启动结帐过程。结帐完成后，WebStorm会围绕下载的源创建项目并建议打开它。
