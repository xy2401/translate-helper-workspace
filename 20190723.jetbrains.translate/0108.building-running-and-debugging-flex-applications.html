运行和调试ActionScript和Flex应用程序

此功能仅在Ultimate版本中受支持。

在这个部分：

  • 运行/调试配置类型
  • 从编辑器中运行或调试应用程序

  • 隐藏或显示[SWF]和[UnloadSWF]调试器消息

有关一般准则，请参阅运行应用程序和不暂停代码执行 。

运行/调试配置类型

以下运行/调试配置类型可用于Flash模块：

  • Flash App配置允许您编译然后运行或调试Flash（ActionScript和Flex）应用程序。您可以在运行或调试之前创建必要的配置。在使用源代码时，您还可以直接在编辑器中开始运行或调试应用程序（请参阅在编辑器中运行或调试应用程序 ）。

  • FlexUnit配置允许您编译然后运行或调试FlexUnit测试。请参阅测试ActionScript和Flex应用程序 。

  • Flash Remote Debug配置允许您调试已编译的应用程序，并在必要时打包，并准备在本地或远程计算机或移动设备上运行。请参阅使用Flash远程调试配置 。

为了能够调试您的应用程序：

  • 应用程序必须是调试就绪的，即包含必要的调试信息。

  • 对于以Web为目标的应用程序，您必须为Web浏览器安装Flash播放器的调试版或Flash Player插件的调试版。通常，此软件包含在Flex SDK中。

    您也可以从Adobe Flash Player下载页面单独下载相应的软件。

从编辑器中运行或调试应用程序

在编辑器中使用类源代码时，如果适用，您可以使用当前类作为主应用程序类来运行或开始调试应用程序 。为此，右键单击编辑器区域中的某个位置以打开上下文菜单，然后选择：

  • 运行“<class_name>” （ Ctrl + Shift + F10 ）运行应用程序。

  • 调试“<class_name>”以开始调试应用程序。

如果适用于该任务的Flash App运行/调试配置已存在，则选择并使用此配置。否则，将创建新的运行/调试配置并将其另存为临时运行/调试配置 。有关详细信息，请参阅IntelliJ IDEA如何选择或创建特定于类的运行/调试配置 。

除了Run和Debug之外 ，根据具体情况，可能还有以下相关命令：

  • 创建“<class_name>”。 如果未找到适当的运行/调试配置，则可以使用此命令创建新的运行/调试配置并使其成为当前配置。（“ 创建运行/调试配置”对话框将打开。）

  • 保存“<class_name>”。 如果相应的运行/调试配置可用作临时配置，则可以使用此命令保存配置，从而使其永久化 。

  • 选择“<class_name>”。 如果要使相应的临时或永久运行/调试配置保持最新，请使用此命令。

上述所有功能也可以作为项目或收藏夹工具窗口中的上下文菜单命令进行访问。

IntelliJ IDEA如何选择或创建特定于类的运行/调试配置

如前所述，当您从编辑器中运行或调试应用程序时 ，IntelliJ IDEA首先尝试使用您当前使用的类作为主类来查找现有的运行/调试配置。

如果未找到此类配置，则会创建新的运行/调试配置。

如果找到具有合适主类的多个配置，IntelliJ IDEA将根据以下条件对配置进行优先级排序，并选择具有最高优先级的配置：

  1. 运行/调试配置基于活动构建配置，构建配置输出类型为Application，主类在运行/调试配置中不被覆盖。

  2. 运行/调试配置基于具有输出类型Application的构建配置，主类在运行/调试配置中不被覆盖。

  3. 运行/调试配置基于活动的构建配置，主类在运行/调试配置中被覆盖。

  4. 具有适当（重写）主类的任何运行/调试配置。

创建新的运行/调试配置时，IntelliJ IDEA会尝试查找具有合适主类的构建配置。如果找到，新的运行/调试配置将基于这样的构建配置。否则，将使用活动的构建配置，并且将在新的运行/调试配置中覆盖主类。

使用Flash远程调试配置

  1. 选择并启动必要的Flash Remote Debug配置（ 或Shift + F9 ）。

    结果，“ 调试”工具窗口打开;调试器正在等待应用程序连接。

  2. 现在，要连接到调试器，请执行以下操作之一：

    • 在本地计算机上启动Flash或AIR应用程序;应用程序将自动连接到调试器。

    • 如果应用程序已在本地或远程计算机上的Web浏览器或Flash播放器中运行，请右键单击浏览器中的相应页面或播放器中的应用程序，然后从上下文菜单中选择“ 调试器 ”。

    • 在移动设备上启动应用程序。如果设备能够与您的计算机通信，则应用程序将自动连接到调试器。

隐藏或显示[SWF]和[UnloadSWF]调试器消息

您的Flash Player调试器输出可能包含许多[SWF]和[UnloadSWF]消息（“ 调试”工具窗口的“ 控制台”选项卡）。有些情况下你不想看到它们。

您可以使用隐藏或显示[SWF]和[UnloadSWF]消息工具栏上的按钮。

请注意，更改此切换的状态不会更改当前控制台内容。也就是说，如果打开过滤器，则不会隐藏输出中已存在的消息。仅显示新消息。
