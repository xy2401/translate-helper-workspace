Cucumber.js

此功能仅在Ultimate版本中受支持。

Cucumber.js是行为驱动的JavaScript开发的测试框架。Cucumber.js测试使用人类可读的Gherkin语言编写，并存储在具有feature扩展的功能文件中。IntelliJ IDEA与Cucumber.js集成并识别用Gherkin编写的功能，因此您可以直接从IDE运行Cucumber，js test。

在你开始之前

  1. 下载并安装Node.js.

  2. 如管理插件中所述，在插件页面上安装Cucumber.js和Gherkin存储库插件 。

安装Cucumber.js

您也可以在npm和Yarn中描述的Node.js和NPM页面上安装cucumber包。

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install cucumber在项目中npm install cucumber以进行本地安装。

    • npm install -g cucumber用于全局安装。

    • npm install --save-dev cucumber安装Cucumber.js作为开发依赖项 。

    另请参阅Cucumber.js官方网站上的Cucumber.js演示 。

创建测试定义

使用IntelliJ IDEA，您可以在JavaScript和TypeScript中编写步骤定义。IntelliJ IDEA检测并突出显示缺少定义的步骤，并建议快速修复以生成定义。

  1. 将光标放在没有定义的步骤上，按Alt + Enter ，然后选择创建步骤定义或创建所有步骤定义 。

    或者，将鼠标悬停在步骤上并使用弹出窗口中的链接。

  2. 在打开的对话框中，指定新文件的名称和要在其中写入定义的语言版本。

    如果此类文件已存在，IntelliJ IDEA将显示一个弹出窗口，您必须选择要添加定义的文件或选择创建新文件。

运行测试

要为一个方案创建运行/调试配置，请从编辑器的上下文菜单中选择Create Scenario：<Scenario_name> 。

如果为一个方案创建运行/调试配置（ 创建方案：<Scenario_name> ），IntelliJ IDEA会自动在“ 名称过滤器”字段中填写方案的名称 。

Cucumber.js测试仅通过运行/调试配置启动。

创建Cucumber.js运行配置

  1. 在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。运行/调试配置：Cucumber.js对话框打开。

  2. 在Feature文件或目录字段中，指定要运行的测试。如果要运行一组功能，请键入特定.feature文件或文件夹的路径。

  3. 指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

  4. 在Cucumber包字段中，根据您的操作系统，指定存储cucumber包的文件夹的路径或cucumber-js.cmd ， cucumber-js.bat或其他可执行文件的位置。

  5. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  6. 如果您使用TypeScript进行步骤定义，请单击在Before启动区域中，从列表中选择Compile TypeScript ，然后指定要使用的tsconfig.json文件。在运行Cucumber.js之前，IntelliJ IDEA将使用此tsconfig.json运行TypeScript编译器。

    要调试用TypeScript编写的测试，请打开tsconfig.json并设置"sourceMap": true以启用源映射。

  7. 可选：

    • 在“ 名称过滤器”字段中，键入要运行的特定方案的名称，而不是功能文件或目录中的所有方案。

    • 指定要传递给可执行文件的命令行参数，例如-r （-- --require LIBRARY|DIR ）， - -t （ - --tags TAG_EXPRESSION ）或--coffee 。有关详细信息，请参阅cucumber-js --help命令提供的本机内置帮助。

或者，将TypeScript编译设置为另一个构建步骤。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Cucumber.js运行/调试配置，然后单击在列表的右侧。

  2. 测试服务器自动启动，无需您执行任何步骤。在“ 运行”工具窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

导航

使用IntelliJ IDEA，您可以在文件和相关测试文件之间跳转，或者从Test Runner选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择Go To |测试或转到|从上下文菜单中测试主题 ，或者只需按Ctrl + Shift + T.

  • 要从.feature文件中的步骤跳到步骤定义，请将光标放在步骤上，然后按Ctrl + Shift + T.

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源 ”。测试文件在编辑器中打开，光标位于测试定义中。

仅识别具有正则表达式的TypeScript定义。

测试文件应遵循流行的命名约定，例如，具有.test. ， .spec.或_spec.后缀，应位于源文件旁边或test文件夹中。

配置语法突出显示

您可以根据自己的喜好和习惯配置Cucumber.js感知语法突出显示。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案|黄瓜 。

  2. 选择颜色方案，接受从默认值继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
