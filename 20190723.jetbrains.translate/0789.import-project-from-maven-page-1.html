IntelliJ IDEA 2019.1帮助

从Maven导入。第1页

使用此页面指定导入首选项。默认导入设置在Maven Integration对话框中定义。

项目

描述

根目录

指定要导入的Maven项目所在的目录。

递归搜索项目

选中此复选框以使导入查找嵌套的pom.xml文件。

项目格式

选择要存储项目的格式。有关两种可用格式的详细信息，请参阅“ 项目”部分。

保留项目文件

选中此复选框以指定存储项目和模块文件的所需位置。

自动导入Maven项目

如果希望IntelliJ IDEA在每次更改pom.xml时自动执行重新导入，请选中此复选框。

为聚合器项目创建IntelliJ IDEA模块（使用'pom'包装）

清除此复选框以跳过创建IntelliJ IDEA模块。如果您导入的Maven项目是聚合器，并且其packaging元素的值为pom ，则可能需要这样做。

为多模块Maven项目创建模块组

如果选中此复选框，IntelliJ IDEA将从聚合Maven项目创建模块组，其中嵌套模块包含在该组中。

保持源和测试文件夹重新导入

如果选中此复选框，则在重新导入项目时将保留源文件和测试文件夹。

排除构建目录
（PROJECT_ROOT /目标）

清除此复选框可禁用自动排除目标文件夹。

请注意，如果/target目录包含/target/generated-sources ，则无法排除/target目录，因为/target/generated-sources目录被标记为sources 。排除/target目录中的其他子目录。

使用Maven输出目录

如果未选中此复选框，则会在常规IntelliJ IDEA的输出目录USER_HOME\IdeaProjects\<project>\classes\Production\创建构建。
如果选中此复选框，则会在Maven的输出目录中生成构建，并重新使用IntelliJ IDEA的编译结果。但是，IntelliJ IDEA本身不会重用Maven构建结果并从头开始执行编译。

生成的源文件夹

重新导入项目时指定源根目录。
您可以选择以下选项之一：
  • 自动检测这是默认选项。选择此选项时，IntelliJ IDEA会自动检测生成的源的位置。IntelliJ IDEA还会检测要标记为源根目录的目录。

    请注意，IntelliJ IDEA仅在target/generated-sources和target/generated-sources/*目录中搜索target/generated-sources/* 。

  • target / generated-sources此选项使您可以手动将目录标记为源根。

  • “target / generated-sources”的子目录此选项使您可以手动将子目录标记为源根目录。

用于文件夹更新的阶段

选择要用于文件夹更新的Maven阶段。如果您调整插件以便在某个阶段加载其他源，这可能很有用。

自动下载

在打开Maven项目时，指定将源和文档注释自动下载到本地存储库中。
您可以选择以下一个或两个复选框：

  • 来源

  • 文档

依赖类型

使用此字段可指定要包含在导入中的依赖关系类型。

环境设置

单击此按钮以指定Maven环境设置。
