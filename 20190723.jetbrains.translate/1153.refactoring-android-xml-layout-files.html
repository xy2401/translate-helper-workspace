重构Android XML布局文件

除了常见的重构之外 ，IntelliJ IDEA还为Android布局定义XML文件提供了许多特定于Android的重构。这些重构中的大多数都可用于两种设计模式 ：

  • 手动模式，位于打开目标布局定义文件的编辑器选项卡中，以及“ 结构”工具窗口中 。

  • 在Android UI Designer中 ，从“ 设计”窗格或“ 组件树”中 。

您可以通过在打开布局定义文件的编辑器中的“ 设计”和“ 文本”选项卡之间切换来在这些模式之间切换。

在本主题中：

  • 提取样式
  • 内联风格
  • 提取布局
  • 内联布局重构

提取样式

Android样式有助于将应用程序设计与应用程序功能分开，就像层叠样式表（CSS）一样 。

通过提取样式重构，您可以从布局XML标记创建样式。IntelliJ IDEA在/res/values/styles.xml文件中创建样式声明，并将标记属性转换为在item元素中声明的样式属性。好处是现在您不再需要解析整个资源定义文件来更新标记属性以编辑布局。您需要做的就是更新样式定义，然后在引用样式的任何位置自动应用更改。

要应用提取样式重构

  1. 执行以下操作之一：

    • 在可视模式下 ，在“ 设计”窗格或“ 组件树”中的画布上的定义中选择使用源XML标记的组件 。

    • 在手动模式下 ，将光标定位在源XML标记内的任何位置。

  2. 从主菜单中，选择Refactor |重构此按钮或按Ctrl + Shift + Alt + T.在出现的弹出窗口中，从“ 提取”组中选择“ 样式 ”。或者，选择Refactor |提取|在选择的上下文菜单中提取样式 。

  3. 在打开的“ 提取Android样式”对话框中，指定要创建的样式的名称，并选择要包括为样式属性的标记属性。

  4. 要让IntelliJ IDEA通过使用对新创建的样式的引用替换标记属性来自动更新布局定义，请选中“在提取样式后启动可能的使用样式”复选框。

内联风格

此重构与Extract Style重构相反，并导致将样式中定义的所有属性添加到应用此样式的一个或所有组件。当您需要将父样式与其继承者合并时，也会使用此重构。

您可以从样式定义或应用样式的组件调用内联样式重构。在第一种情况下，IntelliJ IDEA删除样式定义并将相应的属性添加到使用它的所有组件。从组件调用重构时，可以将其应用于具有此样式的所有组件，或仅应用于当前组件。

要应用内联样式重构

执行以下操作之一：

  • 要从样式定义调用重构：

    1. 打开样式定义文件，将光标定位到要内联的样式的名称，然后选择Refactor |从上下文菜单中内联或按Ctrl + Alt + N.

    2. 打开的“ 内联Android样式”对话框仅提供默认选中的一个选项（ 内联所有引用并删除样式 ）。因此，IntelliJ IDEA将样式的属性转换为XML标记，将它们添加到使用样式的所有组件的定义中，并删除样式定义。

      • 要立即应用更改，请单击“ 重构” 。

      • 要在应用之前打开“ 查找”工具窗口并预览更改 ，请单击“ 预览” 。

  • 要从组件定义调用重构：
    1. 选择具有相关样式的组件：

      • 在可视模式下 ，在“ 设计”窗格或“ 组件树”中选择具有此样式的组件 。然后选择Refactor |主菜单或选择的上下文菜单中的内联样式 。

      • 在手动模式下 ，在Structure视图中选择具有此样式的组件，然后选择Refactor |主菜单或选择的上下文菜单中的内联样式 。

    2. 在打开的内联Android样式对话框中，配置重构：

      • 要让IntelliJ IDEA将样式的属性转换为XML标记，请将它们添加到使用样式的所有组件的定义中，并删除样式定义，选择“ 内联所有引用”并删除样式 。

      • 要仅将重构应用于当前组件，请选择“ 内联此用法”并保留样式 。

    3. 指定要应用更改的方式。

      • 要立即应用更改，请单击“ 重构” 。

      • 要在应用之前打开“ 查找”工具窗口并预览更改 ，请单击“ 预览” 。

提取布局

IntelliJ IDEA支持重用Android布局 ：IntelliJ IDEA将现有布局定义的一部分移动到单独的布局定义文件，并通过自动插入的<include/>标记在原始布局定义中引用它。在IntelliJ IDEA中，此操作称为提取布局重构 。

提取布局

  1. 选择要提取的片段作为单独的布局。执行以下操作之一：

    • 在手动模式下 ，选择代码片段以将光标提取或定位在片段的根元素中。

    • 在可视模式下 ，选择要在“ 设计”中提取的组件或一组组件 。窗格或组件树中 。

  2. 从主菜单中，选择Refactor |重构此按钮或按Ctrl + Shift + Alt + T.在显示的列表中，从“ 提取”组中选择“ 布局 ”。或者，选择Refactor |提取|从选择的上下文菜单中提取布局 。

  3. 在打开的“ 提取Android布局”对话框中，指定将存储提取的布局定义的文件的名称以及其父文件夹的名称。您无需事先创建文件和文件夹，IntelliJ IDEA将自动创建它们。

  4. 为布局创建配置限定符列表。此列表确定必须提供的备用资源集，以便在特定目标设备配置中运行应用程序。

    • 要向集合添加限定符 ，请在“ 可用限定符”列表中选择它，然后单击按钮，并指定限定符的值。

    • 要从集合中删除限定符，请在“ 选择的限定符”列表中选择该限定符 ，然后单击按钮。

    有关限定符，其含义，可接受的值和格式的详细信息，请参阅替代资源的配置限定符的规范 。

内联布局重构

此重构与Extract Layout重构相反，并导致嵌入通过<include/>标记引用并删除标记的布局。

从布局定义文件中的<include/>标记调用内联布局重构。您可以让IntelliJ IDEA将其应用于所有引用并删除布局定义文件，或仅应用于当前引用。

内联布局

  1. 打开引用相关布局的XML定义文件。

  2. 导航到引用相关布局的<include/>标记，并将光标放在布局名称上。

  3. 执行以下操作之一：

    • 选择Refactor |从主菜单内联 。

    • 选择Refactor |从主菜单中重构这个 ，然后在打开的列表中选择Inline ，

    • 按Ctrl + Alt + N.

    • 选择Refactor |从上下文菜单中内联 。

  4. 在打开的Inline Android Layout对话框中，配置重构：

    • 要让IntelliJ IDEA将布局嵌入到通过<include/>标记引用它的所有文件中并删除引用的布局定义文件，请选择内联所有引用并删除该文件 。

    • 要将重构仅应用于当前引用，请选择“ 内联此用法”并保留该文件 。

  5. 指定要应用更改的方式：

    • 要立即应用更改，请单击“ 重构” 。

    • 要在应用之前打开“ 查找”工具窗口并预览更改 ，请单击“ 预览” 。
