在开始之前，请确保您的计算机上有Node.js.

使用React Native，您可以使用JavaScript和React为iOS和Android开发本机移动应用程序。它由Facebook创建，用于Instagram，Airbnb和现在JetBrains自己的YouTrack移动应用程序等知名应用程序。从React Native官方网站了解更多信息。

PyCharm可帮助您创建，编辑，lint，运行，调试和维护React Native应用程序。PyCharm还为React和Flow符号提供代码完成。

全局安装react-native-cli

  • 打开内置终端 （ View | Tool Windows | Terminal ）并在命令提示符下键入<nmp_path>\npm install -g react-native-cli 。

创建一个新的React Native应用程序

创建应用程序

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的创建新项目 。“ 新建项目”对话框打开。

  2. 在左侧窗格中，选择React Native 。

  3. 在右侧窗格中，指定项目文件夹，要使用的Node.js解释器以及react-native-cli包的路径。

  4. 单击“ 创建”时 ，PyCharm将生成一个包含所有必需配置文件的React Native特定项目。

要下载项目依赖项，请执行以下操作之一：

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    <nmp_path> \ npm install

  • 从项目根目录中的package.json文件的上下文菜单中选择Run'npm install' 。

从现有的React Native应用程序开始

要继续开发现有的React Native应用程序，请在PyCharm中打开它并下载所需的依赖项。

打开计算机上已有的应用程序源

  • 单击欢迎屏幕上的打开或选择文件|从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中检出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”或选择“ VCS”从主菜单中查看版本控制 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

下载依赖项

  • 单击弹出窗口中的“运行'npm install' ：

安装其他工具以开始使用React Native，例如iOS模拟器 。这些工具的列表取决于您的操作系统以及您要定位应用程序的移动平台。有关详细的安装说明，请参阅“ React Native入门指南” 。

编码协助

PyCharm在JavaScript代码中为React API和JSX提供代码完成。代码完成适用于React方法，特定于React的属性，HTML标记和组件名称， React事件 ，组件属性等。有关详细信息，请参阅React：完成代码 。

运行和调试React Native应用程序

你需要只安装一次iOS模拟器，之后PyCharm会自动使用react-native run-ios启动它。

每次开始使用应用程序时，都需要准备Android设备。

您可以在物理设备或模拟器上运行和调试应用程序。在启动应用程序之前，请确保模拟器已在运行，或者如果您使用的是物理设备，则它已连接到您的计算机。

PyCharm使运行和调试React Native应用程序非常灵活。例如，如果您是第一次启动应用程序，则可以选择运行React Native捆绑器，构建应用程序，并在模拟器上打开它 - 所有这些都是运行或调试会话的一部分。如果捆绑程序已在运行，您也可以跳过启动它，如果自上次运行后未对其本机代码进行任何更改，则可以拒绝构建应用程序。

如果您使用的是Expo，则可能需要更改默认的bundler主机以确保与调试器成功连接。此连接可能会失败，因为默认情况下，PyCharm使用localhost开始调试，而Expo需要127.0.0.1或外部IP地址，具体取决于Metro捆绑器配置的“连接”字段中选择的内容。

创建React Native运行/调试配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择React Native 。

  2. 选择是否要PyCharm为您构建和启动应用程序：

    • 如果您是第一次启动应用程序，或者自上次运行以来您已更新其本机代码，请选中“ 生成并启动”复选框。

    • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，PyCharm会等待您在模拟器中打开您的应用程序，并启用远程调试，如React Native官方网站上所述。

    • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序会自行处理该过程。

  3. 如果您选择了Build and launch复选框，请选择目标平台Android或iOS。根据您的选择，PyCharm将使用react-native run-ios或react-native run-android运行bundler。

    （可选）键入要传递给React Native的参数，例如，通过‑‑simulator标志指定模拟器类型： - ‑‑simulator="iPhone 4s" 。

  4. 在Bundler主机字段中，指定运行React Native捆绑程序的主机，默认值为localhost 。

  5. 在Bundler端口字段中，指定运行React Native捆绑器的端口，默认选择8081 ，从React Native官方网站了解更多信息。

  6. 选择要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

    指定react-native-cli的路径和应用程序的工作目录 。（可选）为react-native run-android或react-native run-ios键入环境变量。

  7. 默认情况下，当您调用运行/调试配置时，PyCharm会自动启动React Native捆绑器。如果您已经从PyCharm外部启动了捆绑程序，例如，从命令行启动捆绑程序，则可以在不停止和重新启动的情况下重新使用它。在“ 启动前”区域中选择您的捆绑器，然后单击 。

准备Android设备

  • 启动Android虚拟设备或启用USB调试并通过USB 连接到物理Android设备 。从React Native官方网站了解更多信息。

准备iOS模拟器

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    <nmp_path> \ npm install --global ios-sim

运行应用程序

  • 在工具栏上的“ 选择运行/调试配置”列表中选择新创建的React Native配置，然后单击“确定” 在它的旁边。PyCharm打开Run工具窗口，首先在新的React Native选项卡中启动React Native bundler。之后，执行react-native run-ios或react-native run-android命令，具体取决于所选的目标平台。如果构建成功，模拟器将显示您的应用程序：
