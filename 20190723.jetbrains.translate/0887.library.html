库是模块可依赖的已编译代码的集合。在IntelliJ IDEA中，可以在三个级别定义库： 全局 （可用于许多项目）， 项目 （可用于项目中的所有模块）和模块 （可用于一个模块）。

Java库可以包括具有类文件的类文件，存档和目录，以及具有Java本机库（ .dll ， .so或.jnilib ）的目录。

定义一个库

定义库并将其添加到模块依赖项后，IDE将在您编写代码时向您提供其内容。IntelliJ IDEA还将使用库中的代码来构建和部署您的应用程序。

您还可以从位于项目内容根目录中的JAR文件创建新库。在“ 项目”工具窗口中选择这些文件，然后从上下文菜单中选择“ 添加为库 ”。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 平台设置”下 ，选择“ 全局库” 。

  3. 点击按钮并选择如何添加新库：您可以从计算机上的文件添加Java和Kotlin库，或从Maven下载库。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目设置”下 ，选择“ 库” 。

  3. 点击按钮并选择如何添加新库：您可以从计算机上的文件添加Java和Kotlin库，或从Maven下载库。

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目设置”下 ，选择“ 模块”部分。

  3. 打开“ 依赖关系”选项卡。

  4. 点击按钮并选择库 。

  5. 单击对话框底部的New Library ，然后选择您要如何添加新库：您可以从计算机上的文件添加Java和Kotlin库，或从Maven下载库。

从Maven下载库

  1. 按Ctrl + Shift + Alt + S打开“ 项目结构”对话框，然后单击“ 库” 。

  2. 点击 ，并选择From Maven 。

  3. 在下一个对话框中，指定库工件（例如， org.jetbrains:annotations:16.0.2 ）。如果您不知道其确切名称，请输入关键词并单击 。

    您还可以指定其他库位置，并选择是否要下载传递依赖项，源文件，JavaDoc文件或注释。

IntelliJ IDEA将从Maven或Nexus公共存储库下载该库。您还可以配置自定义远程存储库。

配置自定义远程存储库

您可以查看远程存储库的完整列表，并在设置中添加自定义存储库。请注意，即使您不使用Maven作为项目的构建工具，IntelliJ IDEA也可以从Maven加载库。

  1. 在Settings / Preferences对话框（ Ctrl + Alt + S ）中，转到Build，Execution，Deployment |远程Jar存储库 。

  2. 单击相应对话框部分中的“ 添加 ”，然后指定存储库URL。

将库添加到模块依赖项

在将它们添加到模块依赖项之前，全局和项目库不可用：

  1. 打开“ 项目结构”对话框（ Ctrl + Shift + Alt + S ），然后选择“ 模块” 。

  2. 打开Dependencies选项卡，然后单击 。

  3. 选择要添加的库，然后单击对话框底部的“ 添加所选项”。

包括特定的传递依赖项

如果只想包含特定的传递依赖项，可以使用库属性编辑器。

  1. 按Ctrl + Shift + Alt + S打开“ 项目结构”对话框，然后转到“ 模块”|依赖性 。

  2. 选择必要的Maven库并单击 。

  3. 在下一个对话框中，单击“ 编辑” ，然后单击“ 包含传递依赖关系”选项旁边的“ 配置” 。

  4. 选择要包含在库中的依赖项，然后单击“ 确定” 。

排除库项目

IntelliJ IDEA允许您临时排除库项目以提高IDE性能。您可以排除存档中的文件夹，存档（例如JAR）和文件夹。

来自排除包的类不会显示在代码完成建议列表中，对这些类的引用将在编辑器中显示为未解析等。但是，在编译或运行代码时，无论是否存在该库中的排除项，库仍将作为整体使用。

从项目或全局库中排除项目

  1. 按Ctrl + Shift + Alt + S打开“ 项目结构”对话框，然后单击“ 库” 。

  2. 点击 ，然后选择要排除的库项目。

从模块库中排除项目

  1. 按Ctrl + Shift + Alt + S打开“ 项目结构”对话框，然后转到“ 模块”|依赖性 。

  2. 点击 ，然后选择要排除的库项目。

被排除的项目将标有图标。要将库项目恢复为其原始状态，请删除已排除的项目。

更改库级别

在IntelliJ IDEA中，您可以将项目或模块库移动到更高级别。如果您想扩展其使用范围，这将非常有用。例如，如果要在整个项目或整个IDE中使用模块库。要将库移动到更高级别，请在“ 项目结构”对话框中右键单击此库，然后选择“ 移动到项目库”或“ 移动到全局库” 。

您还可以在较低级别创建库的副本。例如，如果要向项目库添加更多类，但只想在一个模块中使用它们。

右键单击必要的全局库，然后选择“ 复制到项目库” 。然后指定是否还需要库文件的副本以及应在何处创建该副本。对于模块依赖项中包含的全局或项目库，要在模块级别开始创建副本，请右键单击所需的库，然后选择“ 复制到模块库” 。
