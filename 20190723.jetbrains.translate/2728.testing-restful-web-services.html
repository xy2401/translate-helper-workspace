测试RESTful Web服务

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

介绍

使用PyCharm，您可以测试RESTful Web服务。PyCharm提供了通过编写和运行请求来模拟与WebService交互的工具，就好像您是实际调用该服务的应用程序一样。除此之外，您还可以创建，保存，编辑和删除Cookie，这些Cookie都是通过响应接收并手动创建的。如果尚未达到到期日期 ，则cookie的名称和值将自动包含在与为cookie指定的域和路径匹配的URL地址的每个请求中。

通过HTTP Client捆绑插件支持测试RESTful Web服务，默认情况下启用该插件。如果没有，请在“ 设置”对话框的“ 插件”页面中将其激活。

当您需要撰写并运行对RESTful Web服务的请求时，有两个主要用例：

  • 当您开发和部署RESTful Web服务并希望确保它按预期工作时：它可以按照规范访问，并且可以正确响应。

  • 在开发解决RESTful Web服务的应用程序时。在这种情况下，在开始开发之前调查对服务的访问和所需的输入数据是有帮助的。在开发过程中，您还可以从应用程序外部调用Web服务。这可能有助于在应用程序导致意外输出时找到错误，同时在代码中未检测到逻辑错误，并且您怀疑瓶颈是与Web服务的交互。

测试RESTful Web服务包括以下检查：

  • 根据服务部署端点和方法注释正确构成URL地址。

  • 生成的服务器请求调用相应的方法。

  • 该方法返回可接受的数据。

PyCharm使您能够通过组合并向服务器提交请求 ，查看和分析服务器响应 ，从REST客户端工具窗口运行这些检查。

如有必要，请在“设置”对话框的“ HTTP代理”页面上配置“代理”设置。

编写测试请求并将其提交给Web服务方法

  1. 如果要测试自己的Web服务，请确保已部署并运行。

  2. 选择工具| HTTP客户端|测试RESTful Web服务 。将打开REST客户端专用工具窗口 。

  3. 要让PyCharm生成将在基本身份验证中使用的身份验证标头 ，请单击Generate Authorization Header按钮，然后在打开的对话框中，指定用于访问目标RESTful Web服务的用户名和密码。基于这些凭证，PyCharm将生成一个用于基本身份验证的身份验证标头 。欲了解更多信息， 请访问http://www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm 。

  4. 从HTTP方法下拉列表中选择测试请求方法 。可用选项包括：

    • 得到

    • POST

    • 放

    • 补丁

    • 删除

    • 头

    • OPTIONS

  5. 提供数据以计算目标方法的URL地址：

    1. 在“ 主机/端口”文本框中，键入部署Web服务的主机的URL地址。

    2. 在“ 路径”字段中，指定要调用的方法的相对路径。

    您可以在主机/端口字段中输入要测试的方法的完整URL地址。无论选择哪种HTTP方法 ，按下时都是如此 PyCharm会将URL地址拆分为主机/端口和方法的路径。提取的相对路径将显示在“ 路径”字段中，提取的参数将添加到“ 请求”选项卡的“ 请求参数”窗格中的列表中。

  6. 在“ 标头数据”窗格中，指定请求标头中包含的技术数据。这些数据通过头字段传递，并定义输入参数（ 接受字段）的格式，响应格式（ 内容类型字段），缓存机制（ 缓存控制字段）等。

    要向列表中添加字段，请单击“ 添加” ，然后在“ 名称”文本框中指定字段名称，并在“ 值”下拉列表中指定字段值。

    字段集及其值应符合Web服务API。换句话说，指定的输入格式应该是Web服务所期望的格式，并且预期的响应格式应该是服务返回的格式。

    对于accept ， content-type和其他一些字段，PyCharm提供了一个建议值列表。从“ 值”下拉列表中选择相关的格式类型。

  7. 创建一组要传递给目标方法的参数并指定它们的值。根据所选的请求方法，您可以通过两种方式创建参数列表：
    • 对于GET请求，请指定要作为URL内的查询字符串传递的参数。使用“ 请求参数”窗格。默认情况下，窗格显示带有一行的空列表。

      • 要添加参数，请单击“ 添加” ，然后在“ 名称”文本框中指定参数的名称，并在“ 值”下拉列表中指定参数的值 。

      • 要从列表中删除参数，请选择该参数，然后单击“ 删除” 。

      • 要禁止发送指定的查询字符串参数并禁用“ 请求参数”窗格中的控件， 请按“不发送任何内容”切换按钮 。

    • 要将参数传递给请求消息正文中的目标方法，请使用“ 请求 正文”窗格或将它们插入到本地文件的请求中。选择GET ， DELETE ， HEAD或OPTIONS请求方法时，将禁用“ 请求正文”窗格。

      • 要显式指定参数，请选择“ 文本”选项，然后在文本框中键入参数和值。

      • 要从文本文件中插入参数，请选择“ 文件内容”选项，然后在“ 要发送的文件”字段中指定文件位置。

      • 要在请求中转换和发送二进制文件，请选择“ 文件上载（multipart / form-data）”选项，并在“ 要发送的文件”字段中指定文件位置。

    在任何一种情况下，参数集及其类型都应符合Web服务API，特别是它们应与目标方法的输入参数完全相同。
  8. 要向服务器提交请求 ，请单击“ 提交请求”按钮 。

    请注意，服务器可能缺少证书，或者不受信任。

    如果服务器不受信任，PyCharm会显示一个建议接受服务器的对话框，或拒绝它。如果您将服务器接受为受信任，则PyCharm会将其证书写入信任存储区。在下一次连接到服务器时，将不会显示此对话框。

查看和分析Web服务的响应

  1. 要查看对服务器请求的响应 ，请切换到“ 响应”选项卡。收到回复后，该选项卡会自动打开。默认情况下，服务器响应以格式显示，在请求标头中通过content-type字段指定。

  2. 要将响应转换为另一种格式并在编辑器的单独选项卡中打开，请使用View as HTML ， 以XML格式查看 ， 以JSON格式查看 纽扣。

  3. 要查看Web服务响应标头中提供的技术数据，请切换到“ 响应标头”选项卡。

使用cookie

使用专用的Cookie选项卡，您可以创建，保存，编辑和删除Cookie，这些Cookie都是通过响应接收并手动创建的。如果尚未达到到期日期 ，则cookie的名称和值将自动包含在与为cookie指定的域和路径匹配的URL地址的每个请求中。

该选项卡显示您通过响应收到或手动创建的所有当前可用cookie的列表。Cookie按照添加到列表中的顺序显示。单击cookie时，其详细信息将变为可编辑状态并显示在文本框中。有关详细信息，请参阅REST Client工具窗口

  • 保存通过响应收到的cookie不需要特定步骤，从服务器收到的所有cookie都会自动保存。要编辑收到的cookie，请单击列表中包含cookie的行，并更新现在显示在可编辑文本框中的详细信息。

  • 要手动创建cookie，请单击并指定以下内容：

    • 要包含在请求中的cookie的名称和值 。

    • 必须提供的请求的域和路径 ，其中包含cookie的名称 ， 值和到期日期 。

  • 要从列表中删除cookie，请选择包含cookie的行并单击 。

配置代理设置

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为 ”下的“ 系统设置” ，然后选择“ HTTP代理” 。

  2. 在打开的“ 代理”对话框中，选择“ 手动代理配置”并指定以下内容：

    • 在主机名和端口号字段中输入代理主机名和端口号。

    • 要启用授权，请选中“ 代理身份验证”复选框，然后在相应字段中键入用户名和密码。

重用请求

在PyCharm会话期间，PyCharm会保持跟踪或请求，您可以运行任何以前执行的请求。您还可以将请求的设置保存在XML文件中，以便它们可以在另一个PyCharm会话中使用。必要时，您可以检索已保存的设置并再次运行请求。

在PyCharm会话中重新运行请求

  1. 单击“ 重播最近请求”按钮 。

  2. 从“ 最近请求”列表中，选择相关请求。字段将填入所选请求的设置。

  3. 单击“ 提交请求”按钮 。

保存请求的设置，以便可以在另一个PyCharm会话中检索它们

  • 单击“ 导出请求”按钮 。在打开的对话框中，指定要保存设置的文件的名称及其父文件夹。

运行在上一次PyCharm会话期间保存的请求

  1. 单击“ 导入请求”按钮 。

  2. 在打开的对话框中，选择相关的XML文件。字段将填入所选请求的设置。

  3. 单击“ 提交请求”按钮 。
