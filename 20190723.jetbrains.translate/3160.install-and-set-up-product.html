安装并设置WebStorm

安装要求

硬件要求

  • 最低2 GB RAM，建议4 GB RAM

  • 1.5 GB硬盘空间+至少1 GB用于缓存

  • 1024x768最低屏幕分辨率

软件要求

JRE 1.8与WebStorm发行版捆绑在一起。您无需在计算机上安装Java即可运行WebStorm。

视窗

苹果系统

Linux的

32位或64位版本的Microsoft Windows 10,8,7（SP1）或Vista（SP2）

macOS 10.8.3或更高版本
（仅支持64位系统）

  • OS Linux（请注意，32位JDK未捆绑，因此建议使用64位系统）

  • 建议使用KDE，Gnome或Unity桌面环境

  • GCC / C ++或Clang

下载并安装WebStorm

  1. 下载适用于您的操作系统的WebStorm 。

  2. 根据您的操作系统执行以下操作：
    • Windows 安装 ：

      1. 运行您下载的WebStorm-*.exe文件。

      2. 按照安装向导中的说明进行操作。

    • macOS 安装 ：

      1. 打开已下载的.dmg软件包，然后将WebStorm拖到“ 应用程序”文件夹中。

      2. 打开已下载的WebStorm-*.dmg软件包，然后将WebStorm拖到Applications文件夹中。

    • Linux 安装 ：
        1. 如果当前的Downloads文件夹不支持文件执行，请将已下载的WebStorm-*.tar.gz文件解压缩到其他文件夹：
          tar xfz WebStorm - * .tar.gz <new_archive_folder>

          根据文件系统层次结构标准（FHS）推荐的安装位置是/opt 。要将WebStorm安装到此目录中，请键入以下命令：

          sudo tar xfz WebStorm - * .tar.gz -C / opt /

        2. 切换到bin子目录：

          cd / opt / WebStorm - * / bin

        3. 从bin子目录运行webstorm.sh 。

    32位系统的JRE未与WebStorm捆绑在一起。如果您使用的是32位版本的Windows， 请在安装向导中选中“ 下载并通过Jetbrains安装JRE x86”复选框，以自动下载和安装JRE。

    不得在现有实例上提取新实例。目标文件夹必须为空。

    早期的WebStorm版本可以在Previous WebStorm Releases页面上下载。

    第一次运行WebStorm

    导入WebStorm设置

    首次启动WebStorm时，或者从先前版本升级后，将打开“ 完整安装”对话框，您可以在其中选择是否要导入IDE设置。

    如果这是您的第一个WebStorm实例，请选择“不导入设置”选项。

    您可以使用File |在以后手动导入和导出设置 导入设置和文件|主菜单上的“ 导出设置”命令。

    选择用户界面主题

    接下来，系统将提示您选择UI主题。您可以选择Default和Darcula主题。

    下载并安装其他插件

    在下一步中，WebStorm会提示您从WebStorm插件存储库下载并安装其他插件。

    在WebStorm中启动项目

    完成初始WebStorm配置后，将显示“ 欢迎”屏幕。它允许您：

    • 创建一个新项目

    • 打开现有项目

    • 或从版本控制系统检出现有项目（从远程存储库克隆）

    也可以将现有项目目录或单独的文件拖到“欢迎”屏幕，这样就可以在WebStorm中打开它。

    注册WebStorm

    要尝试评估WebStorm，您可以免费下载并安装其试用版。试用版可以使用30天，因此您需要获取并注册许可证。

    WebStorm版本可作为Early Access Program的一部分下载，不需要任何注册，并附带30天的许可证。

    1. 执行以下操作之一：

      • 在“ 欢迎”屏幕上，单击“ 配置”|管理许可

      • 在主菜单上，选择“ 帮助”|寄存器

    2. 选择注册WebStorm的方式：

      选项

      描述

      JetBrains账户

      使用JetBrains帐户注册。

      有关更多信息，请参阅什么是JetBrains帐户？

      激活码

      使用激活码注册。

      许可服务器

      使用许可证服务器注册。

      要在初始WebStorm启动期间覆盖系统代理的URL，请使用-Djba.http.proxy属性，该属性可以作为JVM选项添加。

      在多台计算机上执行静默安装或管理WebStorm安装时，可以设置JETBRAINS_LICENSE_SERVER环境变量以将安装指向许可证服务器URL。

    如果您的插件需要许可证，则可以通过单击“ 帮助”|打开“ 许可证”对话框，而不是“ 许可证激活”对话框主菜单中的许可证 。

    “ 许可证”对话框包含许可证列表并显示其状态：

    • 已过期 ：续订许可证以继续。

    • 宽限期 ：许可证到期前有几天时间。

    • 评估期 ：无需有效许可即可使用。

    • 有效许可证 ：使用到有效期。

    要配置许可证，请单击“ 编辑”按钮，这将打开“ 许可证激活”对话框。

    更新WebStorm

    1. 下载最新版本的WebStorm。

    2. 按照下载和安装WebStorm中的说明启动安装程序 。

    3. 选择是否要卸载现有WebStorm版本及其设置，缓存和本地历史记录，并按照安装向导的说明进行操作。

    通过Toolbox App管理WebStorm

    Toolbox App是一个控制面板，允许您从单一访问点管理所有JetBrains开发人员工具，包括WebStorm以及您的项目。它允许您启动IDE，维护同一工具的不同版本，安装更新并在需要时回滚。它还会记住您的JetBrains帐户，并在安装和注册新工具时使用它自动登录。

    1. 下载工具箱应用程序 。

    2. 启动安装文件。

    3. 安装完成后，接受JetBrains隐私政策并登录JetBrains帐户。

    现在，您可以管理现有工具，安装新工具和下载更新：

    在Windows上执行静默安装

    无需任何用户界面即可执行静默安装。网络管理员可以使用它在多台计算机上安装WebStorm，避免中断其他用户。

    要执行静默安装，请使用以下开关运行安装程序：

    • /S ：启用静默安装

    • /D ：指定安装目录的路径

    • /CONFIG ：指定静默配置文件的路径

    例如：

    WebStorm.exe / S /CONFIG=d:\temp\silent.config / D = d：\ IDE \ WebStorm

    要在安装过程中检查问题，请添加带有日志文件路径和名称的/LOG开关。安装程序将生成指定的日志文件。例如：/ /LOG=d:\JetBrains\IDEA\install.log : /LOG=d:\JetBrains\IDEA\install.log 。

    无提示配置文件

    您可以在https://download.jetbrains.com/webstorm/silent.config下载WebStorm的静默配置文件。

    静默配置文件定义了安装WebStorm的选项。使用默认选项，仅对当前用户（ mode=user ）执行静默安装。如果要为所有用户安装WebStorm，请使用文本编辑器打开静默配置文件，更改安装模式选项的值（ mode=admin ）并以管理员身份运行安装程序。

    默认的静默配置文件对于每个JetBrains产品都是唯一的。您可以根据需要对其进行修改以启用或禁用各种安装选项。

    静默卸载WebStorm

    要以静默方式卸载WebStorm，请以管理员身份使用/S开关运行卸载程序。卸载程序位于bin下的安装目录中。

    以管理员身份运行cmd （Windows命令提示符），切换到WebStorm安装目录，然后运行以下命令：

    bin \ uninstall 。 exe / S.

    在Linux上将WebStorm安装为快照包

    您可以在Linux上将WebStorm安装为独立的快照包。由于快照会自动更新，因此您的WebStorm安装将始终是最新的。

    要使用快照，请按照安装指南中的说明在计算机上安装并运行snapd服务。

    在Ubuntu 16.04 LTS及更高版本上，此服务已预先安装。

    WebStorm通过两个渠道分发：

    • 稳定通道仅包含稳定版本。要安装最新的稳定版WebStorm，请运行以下命令：

      $ sudo snap install webstorm --classic

    • 边缘通道包括EAP构建。要安装最新的WebStorm EAP版本，请运行以下命令：

      $ sudo snap install webstorm --classic --edge

    安装快照后，您可以通过运行webstorm命令启动它。

    要列出所有已安装的快照，请运行sudo snap list 。有关更多信息，请参阅Snapcraft文档 。

    --classic选项是必需的，因为WebStorm快照需要对系统的完全访问权限，如传统打包的应用程序。
