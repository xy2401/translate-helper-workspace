项目

描述

Gradle项目

指定要导入的项目的build.gradle文件的地址。手动键入路径，或单击“浏览”按钮并在文件选择器对话框中找到所需的文件。

自动为空内容根创建目录

选择此选项可在从Gradle模型导入项目时自动将src目录添加到项目中。

为每个源集创建单独的模块

选中此复选框以在解决Gradle项目时使用源集功能。

使用默认gradle包装器（未为当前项目配置）

选择此选项以使用Gradle包装器。使用Gradle包装器可以获得构建的自动Gradle下载。它还允许您使用精确的Gradle版本进行构建。

使用可自定义的gradle包装器

选择此选项可在build.gradle文件中配置Gradle包装器。在这种情况下，IntelliJ IDEA可确保配置的可自定义包装设置用于处理Gradle。

使用本地gradle分发

选择此选项可运行本地构建脚本。

Gradle回家

默认情况下，IntelliJ IDEA显示Gradle安装的路径，如GRADLE_HOME环境变量中所定义，或指定为Gradle home项目设置 。

如果路径未知，或者您要使用其他Gradle安装，请单击“浏览”按钮并在路径选择器对话框中找到所需的目录。

项目格式

选择要存储项目的格式。有关两种可用格式的详细信息，请参阅“ 项目”部分。

全局Gradle设置

使用此区域设置全局Gradle设置。您可以选择以下选项：

  • 脱机工作 - 使用此复选框可在脱机模式下使用Gradle。在这种情况下，Gradle将使用缓存中的依赖项。Gradle不会尝试访问网络以执行依赖项解析。如果依赖项的缓存中不存在所需的依赖项，则构建执行将失败。

  • 服务目录路径 - 使用此字段覆盖默认的Gradle主目录位置目录。

  • Gradle VM选项 - 使用此字段为Gradle项目指定VM选项。

    指定JVM选项时，请遵循以下规则：

    • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

    • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

    • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

    • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

      - Dfoo = $ { MY_ENV_VAR }
