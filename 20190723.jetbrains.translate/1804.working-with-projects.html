IntelliJ IDEA 2019.1帮助

配置项目

IntelliJ IDEA中的项目允许您方便地管理模块并在它们之间共享设置。这些设置包括SDK，语言级别和编译器输出。

模块是独立的实体。每个模块都可以有自己的设置，可以配置为开发特定于框架的应用程序。模块可以引用其他模块，即相互依赖。

在IntelliJ IDEA中，您可以将项目与构建工具集成，例如Maven或Gradle 。在这种情况下，IDE将为您提供编码帮助，以便编辑构建文件，代码完成，检查和快速修复，重构等。

使缓存无效

IntelliJ IDEA缓存大量文件，因此系统缓存可能会过载。有时，如果您经常使用频繁的短期项目，则永远不会再需要缓存。

当您使缓存无效时，IntelliJ IDEA将重建在当前版本的IDE中运行的所有项目。

  • 在重新启动IntelliJ IDEA之前，不会删除文件。

  • 打开和关闭项目不会导致删除任何文件。

清除系统缓存

  1. 从主菜单中，选择File |使高速缓存/重新启动无效 。

  2. 在“ 无效缓存”对话框中，选择一个操作。您可以使高速缓存无效并重新启动IDE，在不重新启动IDE的情况下使高速缓存无效，或者只是重新启动IDE。

路径变量

路径变量是占位符，代表链接到项目的资源的路径。它们提供共享灵活性，因为您不必引用计算机上的固定位置。

例如，如果您在项目或模块级别定义了第三方库，并且它存储在项目目录或模块内容根目录之外（对于通过VCS共享的项目），则路径变量很有用。这样的库以其绝对路径引用，并且无法保证此路径在您的队友计算机上是相同的。如果使用路径变量，则会在每台计算机上单独配置每个变量时使路径相对。

在IntelliJ IDEA中，有一些预定义的变量：

  • $USER_HOME$ ：代表您的主目录。

  • $PROJECT_DIR$ ：代表存储项目的目录。

  • $MODULE_DIR$ ：代表保存模块配置文件（ IML ）的目录。

要配置路径变量，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中选择“ 外观和行为”路径变量 。

示例：创建新的路径变量

例如，您有一个未存储在项目目录中的第三方库。如果要在从VCS更新项目后确保队友计算机上的路径正确，则可以创建新变量。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为”|路径变量 ，然后单击按钮。

  2. 输入新变量的名称（例如， PATH_TO_LIB ）及其指向磁盘上库位置的值。

  3. 通过版本控制系统共享IML文件。

  4. 在队友从VCS更新项目后，他们将更改PATH_TO_LIB变量值，使其指向计算机上的库位置。

忽略路径变量

当您打开项目时，IntelliJ IDEA会检查是否存在任何未解析的路径变量。如果IDE检测到任何，它将要求您为它们定义值。如果由于某种原因您不想这样做（例如，如果您不打算使用具有未解析的路径变量的文件或目录），则可以将它们添加到忽略的变量列表中。

例如，如果在运行/调试配置中传递给JVM的程序参数与内部($SOME_STRING$)路径变量具有相同的格式，则还可以使用忽略变量列表。在这种情况下，您可以将此参数添加到忽略变量列表中，以避免混淆。在“ 路径变量”对话框的“ 忽略的变量”字段中输入SOME_STRING 。

资源文件

资源包括属性文件 ，图像，DTD和XML文件。这些文件位于应用程序的类路径中，通常使用以下方法从类路径加载：

  • ResourceBundle.getBundle()用于属性文件和资源包

  • loadResourceAsStream()用于图标和其他文件

构建应用程序时，IntelliJ IDEA会将所有资源复制到输出目录中，从而保留相对于源路径的资源的目录结构。默认情况下，以下文件类型被识别为资源：

可以在“ 编译器”对话框（ 设置/首选项 （ Ctrl + Alt + S ） |构建，执行，部署|编译器 ）中将已识别资源文件的模式配置为正则表达式。使用“ 资源模式”字段，您可以添加自己的文件扩展名并创建自定义资源列表。
