构建ActionScript和Flex应用程序

此功能仅在Ultimate版本中受支持。

您的ActionScript和Flex源根据相应的构建配置进行编译，特别是：

  • 编译Flash模块或其部件（ Build | Make Module'<name>'或Build | Compile'<target_name>' （ Ctrl + Shift + F9 ））时，通常会为所有构建配置生成编译输出与模块相关联。

    要禁用某些构建配置的编译 ，请在相应的构建配置中启用“ 跳过编译”选项 。

  • 编译整个项目（ Build | Make Project （ Ctrl + F9 ）， Build | Rebuild Project ）时，类似地，为未明确禁用编译的所有构建配置生成编译输出。

  • 使用运行/调试配置编译源时，仅为关联的构建配置生成编译输出。

编译过程

在编译期间，对于每个单独的构建配置 ，将处理以下文件，并将处理结果放入输出文件夹：

  • 资源文件（例如，图像文件）。

    对于应用程序，您可以显式指定是否要将模块源根目录中的资源文件复制到输出文件夹（“ 构建配置”页面上的“ 将资源文件复制到输出文件夹”选项 ）。您还可以选择不应复制的单个文件和文件夹（“ 设置”对话框的“ 编译器”|“排除”页面 ）。

    对于库和RLM，资源文件永远不会复制到输出文件夹。

  • 源文件（ .as和.mxml ）。

    编译源文件，并根据构建配置输出类型生成相应的.swf或.swc文件。

  • 要编译成运行时样式表的.css文件（适用于Web和桌面应用程序）。

    显然，编译了指定的.css文件（ Build Configuration页面上的Runtime Style Sheets字段 ）。他们的文件名不会改变;扩展名更改为.swf 。

  • HTML包装器模板文件（用于Web应用程序）。

    如果这样指定（“ 构建配置”页面上的“ 使用HTML包装器”选项 ），则会处理构成HTML包装器模板的文件。（这些文件存储在一个名为html-template或类似文件的单独文件夹中。）

    index.template.html文件被重命名：生成的.html包装文件将与应用程序.swf文件具有相同的名称。此文件中包含的标记（例如${title} ， ${swf}将替换为适当的值。例如， ${swf}将替换为.swf文件名。另请参阅使用SWF元数据标记来控制HTML包装器属性 。

    构成HTML包装器的其他文件将复制到输出文件夹而不进行任何更改。

  • 应用程序描述符模板（用于桌面和移动应用程序）。

    根据构建配置设置，在输出文件夹中创建自动生成的描述符，或使用指定的模板文件。在后一种情况下，模板的<content>元素中的文本将替换为应用程序文件的名称和扩展名（ .swf ）。
