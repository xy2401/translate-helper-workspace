配置Web应用程序部署

此功能仅在Ultimate版本中受支持。

Web应用程序可以作为展开的目录部署到服务器，其中文件和文件夹在文件系统中作为单独的项目呈现，或者作为包含所有必需文件的Web归档 （ WAR文件）。因此，您需要配置项目输出的布局，以便可以使用其中一种形式将其部署到服务器。在IntelliJ IDEA中，项目输出的布局是通过工件定义的。

在模块中启用Web开发时，IntelliJ IDEA会使用以下基本结构配置爆炸类型的工件 ：

您可以使用此预定义工件 ，可能需要进行必要的自定义，或配置新工件 。

配置要部署的工件涉及：

  1. 指定工件类型，名称和输出目录 。

  2. 添加静态Web内容资源 。

建议的部署配置过程反映了可以根据您的首选项和特定Web应用程序的要求灵活定制的基本工作流程。

配置基本工件设置

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 单击“ 工件”以打开“ 工件”页面 。

  3. 执行以下操作之一：

    • 要使用预定义的展开目录工件，请从左侧窗格的列表中选择<module name>war:exploded artifact。如有必要，请在右侧窗格的相应字段中更改工件的名称和输出目录。

    • 要创建新工件，请单击“ 新建” 在左侧窗格中，从“ 新建”下拉列表中选择工件类型。

      • 要将应用程序部署为目录，请选择“ Web应用程序：已爆炸” 。

      • 要以打包的形式部署应用程序，请选择“ Web应用程序：存档” 。

    在右侧窗格中，在相应字段中指定工件的常规设置 ，例如名称和输出目录。

添加静态Web内容资源

  1. 打开所需的工件并切换到右侧窗格的“ 输出布局”选项卡。

  2. 选择选择的输出根节点后 ，在上下文菜单中选择“ 创建目录”项，或单击“ 创建目录”工具栏按钮 。在打开的对话框中指定新文件夹的名称，例如Resources ：

  3. 选择新文件夹后，在上下文菜单中选择“ 添加副本”项，或单击“ 添加副本”工具栏按钮 。

  4. 在上下文菜单上，选择上下文菜单中的“ 目录内容”项。在打开的对话框中 ，选择存储所需Web内容资源的目录。
