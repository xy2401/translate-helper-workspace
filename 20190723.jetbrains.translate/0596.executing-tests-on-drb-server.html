IntelliJ IDEA 2019.1帮助

在DRb服务器上执行测试

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用Ruby插件时有效！

基本

运行测试套装有时会涉及开销，因为每次执行测试套件时，都应重新加载整个环境（例如，应用程序的整个环境）。您可以使用Spork DRb服务器来避免它， 它只加载环境一次。

Spork DRb服务器在后台运行，您可以选择使用Spork DRb服务器或本地执行任何测试脚本。

先决条件

  • 在启动Spork DRb服务器之前，请确保spork gem以及相应的测试宝石（ rspec-rails ， cucumber ， cucumber-rails等）在您的应用程序中使用。

  • spork gem被添加到Gemfile中。

在开始使用之前，请确保这样做插件已安装并启用 。

启动Spork DRb服务器

  1. 在主菜单上，选择“ 工具”|运行Spork DRb服务器....

  2. 在Spork DRb启动选项对话框中，选择所需的测试框架，然后单击运行 。

    如果您要在项目中第一次使用spork ，请启动选中Perform bootstrap复选框的操作。因此，Spork将修补测试脚本。

Spork DRb服务器在“运行”工具窗口的单独选项卡中启动。

使用Spork DRb服务器运行测试脚本

  1. 确保使用IntelliJ IDEA启动Spork DRb服务器并在后台运行。

  2. 运行测试脚本或其中一个示例。请注意， Spork DRb选项会在相应的运行配置中自动选择。如果要在本地运行此测试，则必须选择“ 无 ”选项。

要调试测试，请在使用Spork DRb时遵循以下常规步骤

  1. 确保在要调试的测试脚本中设置断点。

  2. 在主工具栏上，单击运行/调试配置选择器，然后选择“ 编辑配置” 。

  3. 在“ 运行/调试配置对话框”对话框中，为Spork DRb服务器创建运行/调试配置。有关该过程的详细说明，请参阅创建和编辑运行/调试配置一节。

  4. 在调试模式下启动Spork DRb。为此，选择Spork DRb运行/调试配置，单击 ，或按Shift + F9 。

  5. 在Spork DRb服务器以调试模式运行时，运行所需的测试脚本。

请注意，如果您调试测试脚本，它将在没有Spork DRb服务器的情况下执行。

管理Spork DRb服务器

请参阅“ 运行”工具窗口的说明。特别是，使用以下按钮：

  • - 停止Spork DRb服务器而不关闭“运行”工具窗口中的选项卡。

  • - 关闭Spork DRb服务器选项卡。

  • - 在同一选项卡中重新运行Spork DRb服务器。

技巧和窍门

  • IntelliJ IDEA为Spork DRb服务器创建临时运行/调试配置。稍后您可以根据需要更改此运行/调试配置，将其保存为永久性，并使用它来启动服务器。

  • 如果您正在使用TeamCity ，请通过在Spork.preload添加以下语句来避免开销：

    要求 ' teamcity / spec / runner / formatter / teamcity / formatter '

  • Spork DRb服务器不适用于Ruby 2.0。

  • 如果Zeus和Spork DRb服务器同时运行，那么Zeus就会获得优先权。
