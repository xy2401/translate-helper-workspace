流星

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

Meteor是一个全栈框架，允许您在客户端和服务器端使用JavaScript。PyCharm与Meteor集成，因此您可以在IDE内部使用它。 PyCharm中的流星支持包括：

  • 通过检测.meteor文件夹并从项目中排除.meteor/local文件夹来自动识别Meteor项目。有关详细信息，请参阅隐藏排除文件

  • 将预定义的Meteor库自动附加到项目。这样可以启用语法突出显示，解析引用和代码完成。

  • 通过把手支持Spacebars，完成if和each指令。PyCharm识别Spacebars模板，但作为副作用标记Meteor项目中的HTML文件 。PyCharm提供JavaScript源代码和模板之间的导航， 转到声明 （ Ctrl + B ）。

  • 用于在一个调试会话中调试客户端和服务器端代码的专用复杂Meteor运行/调试配置，请参阅调试Meteor应用程序 。

在你开始之前

按照管理插件中的说明在插件页面上安装Meteor和Handlebars / Moustache存储库插件 。

安装流星

需要macOS 10.6或更高版本。

安装过程取决于您使用的操作系统。从Meteor官方网站了解更多信息。

在Windows上安装Meteor

  • 在Meteor官方网站下载LaunchMeteor.exe安装程序。

在macOS或Linux上安装Meteor

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    $ curl https://install.meteor.com | / bin / sh的

创建一个新的Meteor应用程序

如果您还没有应用程序，可以从Meteor样板模板生成具有Meteor特定结构的PyCharm项目。或者，创建一个空的PyCharm项目并在其中配置Meteor支持，如下面的开始使用现有的Meteor应用程序中所述。

从样板模板创建Meteor项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的创建新项目 。“ 新建项目”对话框打开。

  2. 在左侧窗格中，选择Meteor App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

    3. 从“ 模板”列表中，选择要生成的样本。要生成基本项目结构，请选择“ 默认”选项。

    4. 在“ 文件名”字段中，键入将生成的相互关联的.js ， .html和.css文件的名称。仅当从“ 模板”下拉列表中选择“ 默认”样本类型时，该字段才可用。

创建一个空的PyCharm项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的创建新项目 。“ 新建项目”对话框打开。

  2. 在左侧窗格中，选择Empty Project 。在右侧窗格中，指定应用程序文件夹，然后单击“ 创建” 。

从现有的Meteor应用程序开始

如果您要继续开发现有的Meteor应用程序，请在PyCharm中打开它，在其中配置Meteor，并按照下面的下载Meteor依赖项中所述下载所需的依赖项。

打开计算机上已有的应用程序源

  • 单击欢迎屏幕上的打开或选择文件|从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中检出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”或选择“ VCS”从主菜单中查看版本控制 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

在现有项目中配置Meteor支持

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |流星 。Meteor页面打开。

  2. 指定Meteor可执行文件的路径。如果您遵循标准安装过程，PyCharm会自动检测该文件。

  3. 要将.meteor/local文件夹及其内容包含在索引中，请清除“打开项目时自动排除”.meteor / local“目录复选框。有关详细信息，请参阅下面隐藏排除的文件 。

  4. 确保选中“ 自动导入Meteor软件包作为外部库”复选框。

    • 选中该复选框后，PyCharm会自动从meteor/packages文件中导入外部包。因此，PyCharm提供全范围编码帮助：解析对Meteor内置函数的引用，例如check(true) ，以及来自第三方软件包的函数，提供正确的语法和错误突出显示，支持使用源映射进行调试，等等。

    • 清除此复选框后，PyCharm不会自动从meteor/packages文件导入外部包。结果，没有提供编码辅助。要改善这种情况，请在编辑器中打开meteor/packages文件，然后单击Import packages as library链接或运行meteor --update命令。

  5. 确保PyCharm已将Meteor库附加到项目中。

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |图书馆 。

    2. 关于JavaScript。打开的库页面，确保选中“ 库”列表中Meteor项目库旁边的复选框。

导入Meteor包

标记为已排除的文件夹和文件不会在代码完成中使用。

除了确保基本特定流星编码的援助预定义的流星库，你可以下载额外的软件包中定义的.meteor/local/packages文件。

下载其他Meteor软件包

  1. 在编辑器中打开.meteor/local/packages文件。

  2. 单击屏幕右上角的Import Meteor Packages链接。

  3. 在打开的对话框中，根据要在项目中开发的应用程序的类型指定要下载的包。

    • 客户
    • 服务器

    • Cordova ：选择此选项可导入支持iOS和Android Meteor应用程序开发的软件包，有关详细信息，请参阅Meteor Cordova Phonegap Integration 。

PyCharm自动标记.meteor / local文件夹，用于存储构建的应用程序， 排除但仍显示在项目树中。

隐藏.meteor / local文件夹

  • 点击 “ 项目”工具窗口工具栏上的按钮，然后删除“ 显示排除的文件”选项旁边的勾选。

运行Meteor应用程序

从技术上讲，PyCharm为服务器端和客户端代码创建单独的运行配置，但是您可以在一个专用的Meteor运行配置中指定所有设置。

PyCharm根据Meteor类型的运行配置运行Meteor应用程序。如果您从样板模板创建了应用程序，PyCharm会为您生成Meteor运行配置。

创建Meteor运行配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择Meteor 。

  2. 在“ 配置”选项卡中，根据安装指定Meteor可执行文件的路径（请参阅安装Meteor ）。

  3. 指定存储要运行的应用程序文件的文件夹。此文件夹必须在根目录中有一个.meteor子文件夹，以便PyCharm将您的应用程序识别为Meteor项目 。

    默认情况下，工作目录是项目根文件夹。

可选

  1. 在“ 程序参数”字段中，指定命令行在启动时传递给可执行文件的其他参数（如果适用）。这些可以是，例如， --dev ， --test ，或--prod以指示应用程序正在运行（ 开发 ， 测试或生产环境中），以便不同的资源被加载在启动时的环境。

  2. 默认情况下，PyCharm在“ 运行”工具窗口中显示应用程序输出。要查看客户端代码执行的结果，请在“ 浏览器/实时编辑”选项卡中选择“ 启动后”复选框，然后从列表中选择要打开的浏览器。在下面的字段中，指定用于打开应用程序的URL地址。默认值为http://localhost:3000 。

运行Meteor应用程序

  1. 从主工具栏上的列表中选择所需的运行配置，然后转到“运行”|在主菜单上运行<configuration name>或单击 。

  2. 如果您将浏览器配置为在应用程序启动时打开，请在“ 运行”工具窗口或浏览器中查看应用程序输出， 如上所述 。

调试Meteor应用程序

调试器还会在/packages文件夹中存储的源中设置的断点处暂停。客户端和服务器端代码都支持此功能。

使用PyCharm，您可以在一个调试会话中调试Meteor JavaScript代码的客户端和服务器端 。仅通过专用的Meteor运行配置启动调试会话。

从技术上讲，实现不同应用程序的几个Meteor项目可以在一个PyCharm项目中组合使用。要独立运行和调试这些应用程序，请使用相关工作目录为每个应用程序创建单独的运行配置。为避免端口冲突，这些运行配置应使用不同的端口。在“ 程序参数”字段中，为每个运行配置指定一个单独的端口，格式为--port=<port_number> 。

调试Meteor应用程序

  1. 必要时在代码中设置断点 。

  2. 如上所述创建Meteor运行/调试配置。在“ 浏览/实时编辑”选项卡中，选择“ 启动后”复选框，从列表中选择“ Chrome ”，然后选中“ 使用JavaScript调试器”复选框。

  3. 要启动调试会话，请从主工具栏上的列表中选择所需的调试配置，然后单击在列表旁边或选择Run |从主菜单中调试<配置名称> 。

    调试工具窗口打开，显示两个选项卡：一个用于调试标记为的服务器端代码另一个用于调试标有的客户端代码 。

  4. 浏览暂停的程序并逐步完成该程序 。

（可选） 如下所述，即时预览应用程序的更改。

预览浏览器中的更改

如果自动上载仍然失败，请单击以重新启动应用程序在工具栏上。

在调试会话期间，您可以动态预览对HTML，CSS或JavaScript代码的更改。您编辑的页面的实时内容显示在“ 调试”工具窗口的“ Elements选项卡中。更新策略取决于您正在编辑的应用程序的哪个部分。

要预览客户端代码的更改，请执行以下操作之一：

  • 切换到<配置名称> JavaScript 选项卡并单击在工具栏上。

  • 通过选择Meteor页面上的Enable Meteor Hot code push复选框配置自动上载更新。从Meteor官方网站了解更多信息。

要预览服务器端代码的更改，请执行以下操作之一：

  • 切换到<配置名称> 选项卡并单击在工具栏上。

  • 使用实时编辑功能配置自动上传，如HTML，CSS和JavaScript中的实时编辑中所述。建议您在“ 实时编辑”页面上选中“ 如果热插拔失败时重新启动”复选框，那么如果没有更改，则PyCharm将尝试重新加载页面。
