结束（在空白行）

插入对括号

插入对报价

选中此复选框可使IntelliJ IDEA分别为每个打开的单引号或双引号自动添加结束单引号或双引号。

键入“}”时重新格式化块

如果选中此复选框，则在键入结束花括号时，如果此代码块的格式与所选代码样式不匹配，则会自动重新格式化随附的代码块。

使用'CamelHumps'字样

选中此复选框可让IntelliJ IDEA在CamelHump名称中识别单独的单词。名称中的单词应以大写字母或下划线开头。此选项会影响某些编辑器操作，例如：

  • 插入符号移动 （ Ctrl +向右 / Ctrl +向左 ）

  • 使用选择移动插入符号 （ Ctrl + Shift +向右 / Ctrl + Shift +向左 ）

  • 在Caret中选择Word （ Ctrl + W ）

  • 删除到Word开始/结束 （分别为Ctrl + Backspace和Ctrl + Delete ）

  • 双击 （如果在编辑|常规设置页面上启用了使用双击选择时的荣誉“CamelHumps”字设置 ）。

IntelliJ IDEA还提供类似的操作，其工作方式与Use'CamelHumps'字样设置中选择的模式相反：

  • 将Caret移动到不同“CamelHumps”模式中的上一个单词
  • 将Caret移动到上一个单词，选择不同的“CamelHumps”模式
  • 将Caret移动到不同“CamelHumps”模式下的下一个单词
  • 将Caret移动到下一个单词，选择不同的“CamelHumps”模式
  • 在不同的“CamelHumps”模式下删除到Word End
  • 删除到Word以不同的“CamelHumps”模式启动

例如，如果启用了使用'CamelHumps'字样 ，则将Caret移动到不同“CamelHumps”模式下的下一个字的操作会将插入符号移动到单词的结尾，而不管该单词中的大写字符;如果禁用 “CamelHumps”字样 ，则插入符号移动到该单词中的下一个CamelHump。

这些操作没有默认的键盘快捷键，并且不包含在菜单中，但您可以从“ 转到操作” （ Ctrl + Shift + A ）调用它们：

您可以使用您选择的快捷方式将它们绑定，如配置键盘快捷方式部分所述。

键入引号或括号的环绕选择

如果选中此复选框，则键入引号，双引号或括号的所选文本将被这些字符包围。如果未选中此复选框，则键入的引号，双引号或大括号将替换选择。

使用箭头键在双Ctrl / Add上添加多个插入符号

如果选中此复选框，则：

  • 按Ctrl键 （对于Windows或* NIX）或⌥ （对于macOS）两次加上/向下箭头键可以创建多个插入符号。

  • 按Ctrl （对于Windows或* NIX）或⌥ （对于macOS）两次加上左/右箭头键或Home / End导致创建选择。

有关更多信息，请参阅“ Multicursor”部分。

跳转到闭合括号/引用Tab

如果选中此复选框，则在键入括号/引号内时按Tab键会将插入符号移到结束括号/引号之外。如果未选中此复选框，则按Tab键将插入Tab字符。

请注意，在初始打字这仅适用于：后续的编辑过程中，按Tab键括号内/引号将插入Tab符。

输入

使用此区域可以按Enter键来定义要调用的操作。
  • 智能缩进 - 选中此复选框以使IntelliJ IDEA添加新行并将插入符号放入其中，并使用IntelliJ IDEA假定在当前代码点中合理的缩进（缩进基于当前的代码样式设置）。

    如果清除该复选框，则在空白行中按Enter键时，IntelliJ IDEA会添加一个新行并将插入符号放在当前的非空格字符列中。

  • 插入对'}' - 选中此复选框可让IntelliJ IDEA在空行中按Enter键时自动将右括号}定位在正确的列上。在这种情况下，IntelliJ IDEA向后寻找最近的未闭合的开口支撑{并将关闭的一个放在相应的压痕水平。

  • 插入文档注释存根 - 此复选框定义在打开文档标记后按Enter键的行为。
    • 如果选中此复选框，IntelliJ IDEA将生成文档注释存根。

      对于方法注释，此存根包含必需的标记（每个方法参数的@param标记， @return @throws或@throws ）。有关详细信息，请参阅文档代码和创建JSDoc注释 。

    • 如果未选中此复选框，则仅生成结束标记。

    请注意，此复选框指的是JavaScript，Java以及具有特殊文档注释开头的其他语言。

退格

使用此下拉列表可以通过按Backspace键来定义要调用的操作。可用选项包括：

  • 禁用 - 按退格键一次将插入符号返回一个位置。

  • 到最近的缩进位置
  • 适当的缩进

重新格式化粘贴

使用此下拉列表指定粘贴代码块的放置方式。可用选项包括：

  • 无 - 粘贴的代码作为纯文本插入到插入符号位置，无需重新格式化或缩进。

  • 缩进块 - 根据当前的代码样式设置 ，粘贴的代码块位于正确的缩进级别，但其内部结构不会更改。

  • 缩进每一行 - 根据当前的代码样式设置 ，粘贴的代码块的每一行都位于适当的缩进级别。

  • 重新格式化块 - 根据当前的代码样式设置重新格式化粘贴的代码块。

此功能适用于包含尾随换行符的行。

XML / HTML

在此区域中，定义编辑XML或HTML代码时要自动调用的操作。

  • 在标记完成时插入结束标记 ：选中此复选框以使IntelliJ IDEA在输入相应的开始标记时自动插入结束XML或HTML标记。

  • 在标记完成时插入必需的属性 ：选中此复选框可使IntelliJ IDEA显示包含类型标记的所有必需属性的模板。

  • 在标记完成时插入必需的子标记 ：选中此复选框可使IntelliJ IDEA显示包含所有必需子标记的模板。

  • 标记完成时启动属性 ：选中此复选框可使IntelliJ IDEA显示带有类型标记的第一个必需属性的模板。

  • 在键入'='时为属性值添加引号 ：选中此复选框可让IntelliJ IDEA自动为当前键入的属性值添加引号。

  • 键入“</”时自动关闭标记 ：选中此复选框可在输入后自动添加结束标记</。清除此复选框以关闭此类自动完成功能。

  • 同时<tag> </ tag>编辑 ：

    • 选中此复选框并编辑开始标记后，相应的结束标记会自动更改。

    • 如果清除此复选框，则编辑开始标记不会影响保持不变的结束标记。结果，打开和关闭标签不匹配，并且整个构造被加下划线作为错误。

    此复选框控制IntelliJ IDEA在以下上下文中的行为：

    • HTML文件

    • JavaScript代码中的HTML注入

    • 带有模板Handlebars / Moustache模板的HTML

    • Handlebars / Moustache模板文件，扩展名为.hbs

    • XML，XHTML文件

    • DTD文件

    • JSX文件

    • JSP文件

    • PHP文件中的HTML注入

 在Enter上插入字符串连接。 如果您使用的DBMS支持多行字符串文字，则可能需要关闭此选项：

比如，PostgreSQL text值notes有以下片段：

PHP

使用此区域在PHP上下文中定义编辑器的行为：
  • 启用智能功能参数完成 ：选中此复选框后，您可以使用“自动”实时模板，该模板为传递到函数，方法或类构造函数的参数提供完成列表。

    要调用magic live模板，请在函数，方法或类的调用中键入params关键字作为第一个参数：

    IntelliJ IDEA显示一个实时模板，其中参数自动使用函数声明中定义的变量名完成。要移至下一个参数，请按Enter或Tab键 。要移至上一个参数，请按Shift + Tab 。

    完成列表包含下一个顺序中的本地范围的变量：具有相同类型，具有相似名称的变量，在附近定义。您可以通过按Ctrl + Space或只键入列表中没有的任何内容来切换到通常的完成模式。将自动插入具有相似名称的变量。

  • 双击时不选择$符号选择变量名称 ：选中此复选框时，双击或按Ctrl + W仅选择$符号后面的变量名称。如果您经常需要在没有$情况下复制变量名称，这将非常有用：只需双击并复制选择即可。
    如果仍然需要带有$ selected的变量名，请将光标放在$符号前面并双击它或按Ctrl + W.
