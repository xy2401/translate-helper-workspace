模块SDK

选择模块SDK。（要将项目SDK与模块关联，请选择Project SDK 。请注意，如果稍后更改项目SDK，模块SDK将相应更改。）

如果列表中不存在所需的SDK，请单击“ 新建”并选择所需的SDK类型。然后，在打开的对话框中 ，选择SDK主目录，然后单击“ 确定” 。

要查看或编辑所选SDK的名称和内容，请单击“ 编辑” 。（ SDK页面将打开。）

依赖项列表

模块SDK下面显示的是模块依赖项列表。使用 ， ， 和添加，删除和重新排序项目（ 库和模块）。使用标题行中的单元格对列表进行排序，请参阅排序依赖项列表 。

编译或运行代码时，依赖项列表用于构成编译器或JVM的类路径 。（本机库位置（如果有）将添加到java.library.path而不是类路径。）

范围。 此设置允许您分别为源和测试源以及构建和运行阶段控制类路径。（当1）编译源代码时，类路径可能不同2）编译测试源3）运行已编译的源代码4）运行测试。Scope选项定义要包含依赖项的类路径。）

从列表中选择必要的选项：

  • 编译。 在编译和运行阶段，依赖项包含在源和测试源的类路径中。

  • 测试。 在编译和运行阶段，依赖项仅包含在测试源的类路径中。

  • 运行。 依赖项包含在源和测试源的类路径中，但仅限于运行阶段。

  • 提供。 对于您的源，依赖项仅在编译阶段包含在类路径中。当存在在运行时提供相应依赖性的容器（例如，应用程序服务器的Web容器 ）时，这很有用。

    对于测试源，依赖项在构建和运行阶段都包含在类路径中。

    通常，应用程序服务器库包含在提供范围的依赖关系列表中。

请注意，IntelliJ IDEA与处理测试源的依赖关系的方式不同于其他一些构建工具（例如Gradle和Maven）。如果您的模块（模块A）依赖于另一个模块（模块B），则IntelliJ IDEA假定模块A中的测试源不仅依赖于模块B中的源，还依赖于其测试源。因此，B的测试源也包含在相应的类路径中。

出口。 此选项允许您控制依赖于此模块的模块的编译类路径。

如果希望将项目作为依赖项与模块一起导出，请选中该复选框。也就是说，如果存在依赖于此模块的模块，则具有“导出”选项的项目将包含在依赖模块的编译类路径中。

请注意，此设置不会影响运行时类路径。在运行时，当前模块的所有依赖项都包含在依赖于此模块的模块的类路径中。

另请注意，导出时依赖项范围可能会更改。例如：

  • 模块A依赖于模块B，并且此依赖关系的范围是Compile。

  • B依赖于库L，并且此依赖关系的范围是Test。

如果L启用了“导出”选项，则A对L的依赖关系范围将实际为“测试”。

用于处理依赖项的控件

（ Alt + Insert ）。使用此图标或快捷方式将库或模块添加到模块依赖项列表中。选择以下选项之一：

  • 罐子或目录。 选择此选项可创建新模块库并将其添加到依赖项列表中。在打开的对话框中 ，选择要包含在库中的文件和文件夹。这些文件可能是包含此类文件的单个.class和.java文件，目录和存档（ .jar和.zip ）以及包含Java本机库（ .dll ， .so或.jnilib ）的目录。

  • 图书馆。 选择此选项可将一个或多个现有项目，全局或应用程序服务器库添加到依赖项列表中。在打开的对话框中，选择库或库，然后单击“ 添加所选项” 。

  • 模块依赖。 选择此选项可指定当前模块应依赖的模块。

（ Alt + Delete ）。使用此图标或快捷方式从依赖项列表中删除所选项。

（ Alt + Up ）和 （ Alt + Down ）。使用这些图标和快捷方式在列表中向上或向下移动所选项目。请参阅模块依赖性 。

（ F4 ）。使用此图标或快捷方式打开“ 配置库”对话框以编辑库。

依赖存储格式

选择存储依赖项的格式（作为IntelliJ IDEA模块或Eclipse项目）。此选项对使用不同开发工具的团队很有帮助。

对于库：使用此命令编辑选定的库。（“ 配置库”对话框将打开。）

使用此命令可以查看使用所选依赖项的模块列表。单击该列表中的模块 ，打开相应模块的模块页面 。

使用此命令对所选项执行依赖关系分析，并在依赖关系查看器中显示结果。

对于模块库：使用此命令将选定的库移动到更高级别 （项目或全局）。

复制到模块库

对于全局或项目库：使用此命令在模块级别创建所选库的副本。

对依赖项列表进行排序

您可以通过单击标题行中的单元格按名称和范围对依赖项进行排序。

如果单击一次单元格，则列表将按相应的列按升序排序。排序标记出现在单元格中： 。单击第二次单元格时，信息按降序排序。为了显示这一点，排序标记会改变其外观： 。最后，当您第三次单击该单元格时，将使用初始未排序状态。

请注意，排序操作不会更改依赖项的实际顺序。

对列表进行排序后，用于更改依赖关系顺序的图标将处于非活动状态。
