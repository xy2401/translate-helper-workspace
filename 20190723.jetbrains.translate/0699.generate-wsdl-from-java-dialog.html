从Java Dialog生成WSDL

工具|网络服务|从Java代码生成WSDL

使用该对话框配置Web服务WSDL生成，并选择要作为Web服务操作公开并部署的方法。对话框的内容取决于Web服务类型。

项目

描述

Web服务类型

用于生成WSDL的类

此只读字段显示类的名称

所有

Web服务URL

在此文本框中，指定Web服务可用的URL地址。

所有

操作方法

在此区域中，指定要将所选类的哪些方法部署为Web服务操作。

  • 公开方法 - 此列显示所选类中所有方法的列表。

  • 添加到部署 - 选择要作为操作部署的方法旁边的复选框。

如果无法选择方法，则工具提示会说明原因。

Apache Axis

Web服务命名空间

在此文本框中，键入Web服务名称空间前缀。

Apache Axis

SOAP动作

从此列表中，选择<wsdlsoap:operation />字段的soapAction属性的值。可用选项包括：

  • 默认 - 选择此选项时， soapAction根据操作的元数据设置（通常为“”）。

  • 操作 - 选择此选项后， soapAction将设置为操作名称。

  • 无 - 选择此选项时， soapAction设置为“”。

Apache Axis

装订风格

使用此下拉列表指定要生成的WSDL文档的样式。可用选项包括：
  • RPC - 选择此选项以生成rpc WSDL。

    默认情况下选择此选项。

  • 文档 - 选择此选项以生成文档 WSDL。

  • Wrapped - 选择此选项以使用包装方法生成WSDL。

    如果选择此选项，则会在“ 使用项目绑定”下拉列表中自动预选Literal 。

Apache Axis

在绑定中使用项目

从此下拉列表中，选择应如何使用生成的WSDL文档。可用选项包括：

  • Literal - 选择此选项时，XML Schema定义请求的XML表示形式。

  • 编码 - 选择此选项可在生成的WSDL中指定SOAP编码。

Apache Axis

类型映射版本

使用此下拉列表指定默认类型映射注册表，以使用指定的Serializer将Java类映射到XML限定名称。可用选项包括：

  • 1.1表示符合SOAP 1.1 JAX-RPC。

  • 1.2表示SOAP 1.1编码。

Apache Axis
