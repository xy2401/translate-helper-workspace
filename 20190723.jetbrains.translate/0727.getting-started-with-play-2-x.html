IntelliJ IDEA 2019.1帮助

Play 2.x入门

此功能仅在Ultimate版本中受支持。

在开始生成Play项目之前，请确保在IntelliJ IDEA中下载并启用了Scala插件。

IntelliJ IDEA支持Play版本2.4及更高版本。

让我们看一下在IntelliJ IDEA中生成Play 2项目的以下常见场景：

  • 如果您已有一个Play 2项目，则可以将其导入或从版本控制中签出。在这种情况下，您可以按照常规过程导入或克隆项目。

  • 如果您不熟悉Play 2框架并想学习如何使用它，请通过IntelliJ IDEA创建一个Play项目，该项目将生成可执行应用程序。

  • 如果要查看可以生成Play 2项目的类型，请使用Lightbend Project Starter并使用其中一个提供的模板创建项目。

  • 如果要手动添加对Play 2框架的支持，请创建一个sbt项目。

创建一个新的Play 2项目

  1. 打开新建项目向导。从右侧的选项中，选择Play 2.x.

  2. 按照向导中建议的步骤操作，然后单击“ 完成” 。

IntelliJ IDEA使用适当的结构，依赖项和可执行应用程序生成Play 2项目。

使用Lightbend项目启动器创建Play 2模板

  1. 打开一个新项目向导，然后从右侧的选项中选择Lightbend Project Starter 。

  2. 在向导的下一页上，选择要用于项目的模板。例如，选择Play Scala Seed模板，该模板将创建用于开发Play应用程序的可执行Play模板项目。

    单击完成 。

使用Play 2框架创建一个sbt项目

  1. 创建一个sbt项目 。

    按照向导中建议的步骤操作，然后单击“ 完成” 。
    IntelliJ IDEA创建一个sbt项目。

  2. 在编辑器中打开build.sbt文件，并将Play添加为项目依赖项。

  3. 在“ 项目”工具窗口的项目源根目录中，选择“ 新建”|文件创建plugins.sbt文件。

  4. 在编辑器中打开plugins.sbt文件并添加Play 2插件。

IntelliJ IDEA下载相应的依赖项，并将它们添加到sbt projects工具窗口中的Dependencies节点。

请注意，未生成Play 2框架项目结构，您需要手动添加所有内容。检查Play 2项目结构。

此时，您可以开始创建Play 2模板，因为一旦添加了对Play 2框架的支持，IntelliJ IDEA就会支持它们，并进一步开发您的项目。您可以在Play 2文件中使用代码完成，导航和动态代码分析功能。IntelliJ IDEA还支持路由文件和代码检查的代码帮助。

运行和调试Play 2应用程序

  • 如果要运行该应用程序：

    1. 在项目工具窗口中，右键单击该应用程序。

    2. 在上下文菜单中，选择Run Play 2 App 。

  • 如果需要访问“ 运行/调试配置”对话框：

    1. 在主菜单上，选择“运行”|编辑配置 。

    2. 点击并选择Play 2 App以创建新配置。

    3. 指定运行/调试配置设置。默认设置足以进行运行和调试。

    4. 在主工具栏上，单击要运行的图标或用于调试应用程序的图标。

您可以在默认浏览器http://localhost:9000检查应用程序的输出。如果要停止服务器会话，请按Alt + D （对于macOS，为Cmd + P ）。

使用Play2设置

您可以使用Play 2设置来配置编译器选项并配置路由文件的格式。您还可以查看和编辑默认模板导入列表。

  1. 按Ctrl + Alt + S，然后从左侧的选项中选择Languages Frameworks | Play2 。

  2. 根据您要配置的设置，在Play2设置页面上，选择相应的选项卡并调整设置。使用“ 其他”选项卡可以检查和编辑刷新项目时从描述符中检索的模板导入列表。

    要手动添加模板，删除或编辑现有模板 ，请选择“ 手动选择模板导入”选项，以使“ 导入”部分变为活动状态并相应地调整导入列表。
