设置本地Mercurial存储库

尽管Mercurial在安排数据和使用存储库工作方面提供了高度灵活性，但以下方案最常用于设置本地Mercurial存储库：

  • 克隆现有远程存储库并使用下载的数据创建新项目。

  • 创建一个本地存储库 ，如有必要，可以稍后将其推送到远程位置。

克隆远程Mercurial存储库

  1. 在主菜单上，选择VCS |从版本控制检出| Mercurial 。将打开“ 克隆Mercurial存储库”对话框。

  2. 在Mercurial Repository URL文本框中，键入要克隆的远程存储库的URL。

  3. 单击Mercurial Repository URL文本框旁边的Test Repository按钮，检查是否可以成功建立与远程存储库的连接。

  4. 在“ 父目录”文本框中，指定希望IntelliJ IDEA为本地Mercurial存储库创建文件夹的目录。使用浏览 按钮，如有必要。

  5. 在“ 目录名称”文本框中，指定要将克隆存储库的新文件夹的名称。单击克隆 。

  6. 通过接受IntelliJ IDEA显示的相应建议，基于克隆数据创建新项目。

创建本地Mercurial存储库

  1. 打开要存储在存储库中的项目。

  2. 在主菜单上，选择VCS |导入版本控制|创建Mercurial存储库 。将打开“ 创建Mercurial存储库”对话框。

  3. 指定新存储库的位置。
    • 要在项目根目录中创建存储库 ，请选择“ 为整个项目创建存储库”选项。IntelliJ IDEA将在项目根文件夹中创建.hg目录。

      默认情况下选择此选项。

    • 要在另一个位置创建新存储库，请选择“ 选择创建存储库的位置”选项，并在下面的文本框中指定存储库位置的路径。手动键入路径或单击“ 浏览” 并在打开的“ 选择hg init目录”对话框中选择相关文件夹。

      Mercurial不支持外部路径。因此，如果您选择另一个目录，请注意它必须包含项目根所在的树。

      如果选择已在Mercurial控件下的目录，IntelliJ IDEA将打开Directory Is Under hg对话框，您可以在其中选择在指定位置创建存储库或保留在父存储库中。

  4. 将所需文件放在Mercurial版本控制下。这些文件显示在“ 默认”节点下的“ 版本控制”工具窗口中。

    请注意，如果在“ 版本控制”对话框中将Mercurial指定为目录的版本控制系统，IntelliJ IDEA将建议将每个新文件放在Mercurial控件下的此目录中。
