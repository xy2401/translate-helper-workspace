可能的原因

解

项目设置

项目解释程序在“首选项/设置”中的可用解释程序列表中标记为不受支持项目|项目口译员 。

标记解释器的Python版本已过时，PyCharm不支持它

您可以从可用解释器列表中删除不受支持的解释器。有关受支持的Python版本的更多信息，请参阅Python支持 。

代码编辑

PyCharm Editor限制代码编辑。

您已安装IdeaVim插件，因此启用了vim编辑模式。

在“ 工具”菜单上取消选择“ Vim仿真器 ”。有关更多详细信息，请参阅在PyCharm中使用Vim编辑器仿真 。

编辑器中未突出显示Python代码。

PY文件与文本文件格式相关联。

在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 编辑器”|单击文件类型 ，从识别文件类型列表中选择文本 ，从已注册模式列表中选择* .py ，然后单击 。

代码完成

代码完成操作不可用。

省电模式已启用。

从主菜单中，选择“ 文件”并清除“ 省电模式”复选框。

您的文件不驻留在内容根目录中，因此它不会获得代码完成所需的必需类定义和资源。

重构源文件。

包含要在完成建议列表中显示的类和函数的文件是纯文本文件。

重新考虑并修改（如果需要）源文件的格式。

调试

调试Gevent代码时，Python Debugger会挂起。

未启用Gevent兼容模式。

在Settings / Preferences对话框（ Ctrl + Alt + S ）中，导航到Build，Execution，Deployment | Python Debugger并选择Gevent compatible复选框。

PyQt代码中报告了一些导入错误。

PyQt安装在解释器上，但未在应用程序代码中导入。

在Settings / Preferences对话框（ Ctrl + Alt + S ）中，导航到Build，Execution，Deployment | Python Debugger ，并清除中的PyQt compatible复选框。默认情况下启用此模式。

调试过程很慢。

调试器不仅在进程以异常终止时停止，而且在每次抛出异常时停止（即使它被捕获并且没有导致进程终止）。

清除“ 断点”对话框中的“ 提升”复选框（ “运行”|“查看断点” ）。

第一次尝试附加到本地进程时出现以下错误消息： ptrace: Operation not permitted 。

这是Ubuntu的一个已知问题 。

确保kernel.yama.ptrace_scope内核参数设置为0。您可以使用命令sudo sysctl kernel.yama.ptrace_scope=0临时更改其值，或者通过将kernel.yama.ptrace_scope = 0行添加到/etc/sysctl.d/10-ptrace.conf文件中来永久设置它。

Cython加速

显示以下错误消息：

Python.h: no such file or directory 。

您缺少Python dev的头文件和静态库。

使用包管理器为系统范围内的Python dev安装头文件和静态库。

例如，您可以使用以下命令：

  • 对于Python 2.x：

    sudo apt-get install python-dev

  • 对于Python 3.5：

    sudo apt-get install python3-dev

  • 对于Python 3.6：

    sudo apt-get install python3.6-dev

显示以下错误消息：

Command 'gcc' failed with exit status 1 。

你没有C编译器。

安装C编译器以便为调试器构建Cython扩展。

您已经单击了“ 安装”链接
在Cython加速扩展通知中
但PyCharm反复提示安装它。

您没有权限写入PyCharm使用的目录 。

检查并修改您的权限。

打包

包安装失败。

pip不适用于特定的Python解释器，或者不满足任何安装要求。

  • 如果pip不可用，请为所需的Python解释器安装它（请参阅相应的故障排除提示 ）

  • 请参阅特定软件包的产品文档，以确保它可以安装在您的操作系统和目标Python解释器中。另外，检查是否应满足任何其他系统要求。

无法连接到Docker

Docker未运行，或者您的Docker连接设置不正确。

如果您使用的是Docker for Windows，请在Docker设置的“ 常规”部分中启用tcp：// localhost：2375上的Expose守护程序，而不使用TLS选项。

如果您使用的是Docker Toolbox，请确保Docker Machine正在运行，并且在Build，Execution，Deployment |下的Settings / Preferences对话框（ Ctrl + Alt + S ）中正确指定了它的可执行文件。 Docker |工具 。

当您尝试拉取图像时，将显示以下消息：

Failed to parse dockerCfgFile: <your_home_dir>/.docker/config.json, caused by: ... {"credsStore":"wincred"}

使用credsStore时无效的Auth配置文件： http ： credsStore

转到<your_home_dir> /.泊坞目录并删除config.json文件。

Docker Compose可执行文件指定不正确。

在Build，Execution，Deployment |下的Settings / Preferences对话框（ Ctrl + Alt + S ）中指定Docker Compose可执行文件Docker |工具

诊断行动

在PyCharm和终端窗口中运行应用程序时，您遇到的应用程序行为不同。

使用PyCharm项目设置中指定的python版本运行脚本：

  1. 在PyCharm中，打开“ 设置”对话框（ Ctrl + Alt + S ），导航到“ 项目<项目名称>”项目口译员 。

  2. 点击在Project Interpreter字段附近，选择Show All ....您在项目中使用的解释器将在可用的解释器列表中选择。

  3. 点击并从Interpreter路径字段中复制路径 。

  4. 现在，使用复制的路径在终端窗口中运行脚本

    <copied python interpreter path> <path to the python script>\my_script.py 。
    例如， C:\Python27\python.exe C:\Samples\my_script.py

  1. 在PyCharm中，打开“ 首选项”对话框（ Ctrl + Alt + S ），导航到“ 项目<项目名称>”项目口译员 。

  2. 点击在Project Interpreter字段旁边，选择Show All ....您在项目中使用的解释器将在可用的解释器列表中选择。

  3. 点击并从Interpreter路径字段中复制路径 。

  4. 现在，使用复制的路径在终端窗口中运行脚本

    <copied python interpreter path> <path to the python script>/my_script.py 。
    例如， /Users/jetbrains/bin/python3 /Users/jetbrains/Samples/my_script.py

如果应用程序的行为仍然不同，请联系支持服务pycharm-support@jetbrains.com

未为特定Python解释器安装所需的包。
为特定Python解释器安装的软件包版本已过时。

运行以下命令以安装所需的包：

<python可执行文件的路径> \ python.exe -mpip install'<package>'

例如，
C：\ Python27 \ python.exe -mpip install'Flask'

执行后，这些命令将安装指定软件包的最新版本。

要安装特定版本的软件包，请使用以下表达式：

  • '<package>==<package_version>' - 安装特定版本的软件包

  • '<package>>=<package_version>' - 安装大于指定或更高版本的软件包

  • '<package>~=<package_version>' - 安装与指定版本的软件包兼容的版本

例如，以下命令安装与1.0.2版兼容的Flask：
C：\ Python27 \ python.exe -mpip install'Flask~ = 1.0.2'

<python可执行文件的路径/ python -mpip install'<package>'

例如，
/ Users / jetbrains / anaconda3 / bin / python -mpip install'Flask'

执行后，这些命令将安装指定软件包的最新版本。

要安装特定版本的软件包，请使用以下表达式：

  • '<package>==<package_version>' - 安装特定版本的软件包

  • '<package>>=<package_version>' - 安装大于指定或更高版本的软件包

  • '<package>~=<package_version>' - 安装与指定版本的软件包兼容的版本

例如，以下命令安装与1.0.2版兼容的Flask：
/ Users / jetbrains / anaconda3 / bin / python -mpip install'Flask~ = 1.0.2'

在尝试安装软件包时，您会发现pip不适用于特定的Python解释器。

尝试从标准库引导pip：

<python可执行文件的路径> \ python.exe -m ensurepip --default-pip

例如，
C：\ Python27 \ python.exe -m ensurepip --default-pip

<python可执行文件的路径/ python -m ensurepip --default-pip

例如，
/ Users / jetbrains / anaconda3 / bin / python -m ensurepip --default-pip

有关pip安装的更多信息，请参阅packaging.python.org
