在本教程中，您将以科学模式运行并使用Matplotlib和NumPy包来运行和调试具有数据可视化的Python代码。

在开始之前，请确保安装以下内容：

  • 康达翻译

  • Matplotlib包

  • NumPy包

创建科学项目

要创建预定义科学模式设置的PyCharm项目：

  1. 打开New Project向导（ File | New | Project ）。

  2. 选择科学项目类型。

  3. 在项目设置对话框窗口中，指定项目名称，确保选择Conda作为新环境，并更改默认数据文件夹（如果需要），然后单击“ 创建” 。

  4. 打开main.py文件并添加以下代码示例：

运行

运行您的科学项目（ Shift + F10 ）。代码被执行并在SciView中显示两个图形。单击预览缩略图将显示相应的图表：

您可以修改项目代码，一次只绘制一个图形。在科学模式中，您可以通过创建代码单元来执行代码片段。

要执行代码单元格：

  1. 通过添加“＃%%”行修改main.py文件。

  2. 在左侧排水沟中，单击图标 （ Ctrl + Enter ）与散点图单元格标记一致。只会构建散点图。

  3. 现在单击图标或者在带有y对x的单元格标记的行上按Ctrl + Enter 。应该出现相应的图表。

调试

我们在这一行放一个断点：

plt 。 show （）

此行在我们的示例代码中出现两次，因此将有两个断点。

右键单击编辑器背景，然后在上下文菜单中选择Debug'ample' 。

您可以在编辑器中看到“ 调试”工具窗口和灰色字符。这是内联调试的结果，该调试已启用。

第一个断点的行以蓝色突出显示。这意味着调试器已在断点处停止，但尚未执行。如果我们执行此行（例如，通过单击 “调试工具”窗口的步进工具栏上的按钮），我们将看到图形：

接下来，查看Debug工具窗口的Variables选项卡。如果单击area数组附近的View as Array链接，将打开SciView窗口中的“数据”选项卡：

处理过量数据时，如果调试器同步或异步加载变量值，则可能会降低调试性能。建议通过选择相应的加载策略切换到按需模式。有关更多信息，请参阅管理变量加载策略 。

请注意SciView中“数据”选项卡中唯一的一行数字 - 它是由area数组是一维的事实来解释的。
