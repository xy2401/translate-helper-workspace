WebStorm 2019.1帮助

生成代码

WebStorm提供了多种方法来生成公共代码构造和重复元素，从而帮助您提高生产力。这些可以是预定义的模板，也可以是自定义模板，它们根据上下文，各种包装器和字符的自动配对而不同地应用。此外，WebStorm还提供代码完成和Emmet支持。

在主菜单上，选择Code |生成 （ Alt + Insert ）以打开具有可生成的可用构造的弹出菜单。

使用实时模板创建代码构造

有关详细信息，请参阅实时模板：参数化模板 。

WebStorm为许多常见的代码构造提供了许多预定义的实时模板 。您还可以定义自定义模板以涵盖特定于工作流的用例。

您可以通过插入和扩展实时模板以及使用环绕模板包装代码片段来创建代码构造。

插入实时模板

  1. 将插入符号放在您希望模板展开的位置。

  2. 键入模板缩写并按下调用键（默认情况下为Tab ）。

    或者，要使用代码完成，请按Ctrl + J或从上下文菜单中选择“ 插入实时模板” ，然后从建议列表中选择所需的模板。要查看所选建议的快速文档 ，请按Ctrl + Q.

  3. 如果所选模板需要用户输入，则第一个输入字段标有红框。在此框架中键入您的值，然后按Enter或Tab键完成输入并传递到下一个输入字段。完成最后一个输入字段后，插入符移动到构造的末尾，编辑器返回常规操作模式。

使用环绕模板包装代码片段

  1. 在编辑器中，选择要包装的代码段并按Ctrl + Alt + J或从上下文菜单或选择中选择Surround With Live Template ....

  2. 在建议列表中，选择所需的模板。

查看可用的实时模板列表

  • 在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）

    ，单击编辑器下的实时模板 。打开的“ 实时模板”页面显示按语言分组的所有已配置的实时模板。有关更多信息，请参阅创建实时模板 。

展开和删除语句

WebStorm允许您快速从封闭语句中解包或提取表达式。如果...... else ， for ， while和do ...同时控制结构以及XML和HTML标记， 则此操作可用于JavaScript。

  1. 将插入符号放在要提取或解包的表达式上。

  2. 选择代码|在主菜单上展开/删除或按Ctrl + Shift + Delete 。WebStorm显示一个弹出窗口，其中包含当前上下文中可用的所有操作。要提取的语句显示在蓝色背景上，要删除的语句显示在灰色背景上。

  3. 单击所需操作或使用向上和向下箭头键选择它，然后按Enter键 。

完成配对字符

WebStorm可以自动添加结束括号和其他配对元素。

  1. 在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）

    ，单击编辑器下的常规 ，然后单击智能键 。将打开“ 智能钥匙”页面 。

  2. 要在按Enter键时自动添加结束括号，请在Enter区域中选中Insert pair}复选框。

  3. 通过选择XML / HTML区域中的相关复选框，配置结束标记的自动插入。

缩进级别在“ 代码样式”设置中定义。

生成构造函数

WebStorm可以生成一个构造函数，该构造函数使用相应参数的值初始化特定字段。

class Person { private readonly name ： string private readonly lastName ： string }
class Person { private readonly name ： string private readonly lastName ： string constructor （ name ： string ， lastName ： string ） { this 。 name = name ; 这个 。 lastName = lastName ; } }

  1. 从类的上下文菜单中选择“ 生成 ”或按Alt + Insert 。

  2. 在“ 生成”弹出窗口中，单击“ 构造函数” 。

  3. 如果类包含字段，请选择要由构造函数初始化的字段，然后单击“ 确定” 。

生成getter和setter

WebStorm可以为类中的字段生成访问器和增变器方法（ getter和setter ）。WebStorm根据您的代码生成命名首选项生成getter和setter的名称，有关详细信息，请参阅JavaScript代码样式：代码生成和TypeScript代码样式：代码生成 。

class Person { private name ： string private lastName ： string }
class Person { get name （） ： string { return this 。 _name ; } 得到 的lastName（）： 字符串 { 退货 。 _lastName ; } private _name ： string private _lastName ： string }
class Person { set name （ value ： string ） { this 。 _name = value ; } 设定 lastName的 （ 值 ： 字符串 ）{ 此 。 _lastName = value ; } private _name ： string private _lastName ： string }

  1. 从类的上下文菜单中选择“ 生成 ”或按Alt + Insert 。

  2. 在“ 生成”弹出窗口中，单击以下选项之一：

    • Getter生成访问器方法以获取类字段的当前值。

    • Setter用于生成用于设置类字段值的mutator方法。

    • Getter和Setter生成访问器和mutator方法。

  3. 在打开的对话框中，选择要为其生成getter或setter的字段，然后单击“ 确定” 。

实现接口或抽象类的方法

如果一个类被声明为实现接口或抽象类，则它必须实现父类或基接口的方法。WebStorm为已实现的方法创建存根。

  1. 从类的上下文菜单中选择Generate （ Alt + Insert ），然后从弹出窗口中选择Implement members或者按Ctrl + I.

  2. 在打开的对话框中，选择要实现的方法，然后单击“ 确定” 。

覆盖超类的方法

另请参阅自动完成代码和路径 。

您可以通过从预定义模板生成必要的代码来覆盖父类的任何方法。WebStorm创建一个包含对超类方法的调用的存根，您只需在方法体中提供一些有意义的源代码。

  1. 从类的上下文菜单中选择Generate （ Alt + Insert ），然后从弹出窗口中选择Override method或者按Ctrl + O.

  2. 在打开的对话框中，选择要覆盖的方法。方法列表不包括已被覆盖的方法，或者无法从当前子类访问的方法。

  3. 单击“ 确定”并提供方法主体的源代码。使用标记左侧装订线中的重写方法以查看基类名称的图标，以及打开重写方法声明的图标。

您还可以使用代码完成并从完成建议列表中选择父方法。WebStorm会自动添加参数，生成super()调用，并在可能的情况下显示类型信息。

关闭替代方法体的生成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript 。

  2. 在打开的JavaScript页面上，清除“ 覆盖完成时展开方法体”复选框。
