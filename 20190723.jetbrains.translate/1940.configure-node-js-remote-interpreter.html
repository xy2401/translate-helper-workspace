配置Node.js远程解释器对话框

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

只有在安装并启用了Node.js Remote Interpreter插件时，该对话框才可用。该插件不与PyCharm捆绑在一起，但它可以从JetBrains插件存储库安装，如管理插件中所述 。

单击“ 添加”后，将打开该对话框在Node.js Interpreters对话框中，从下拉菜单中选择Remote ....仅当您从运行/调试配置：Node.js打开Node.js解释器对话框时，此菜单项才可用。

使用此对话框可以配置对远程主机或Vagrant实例中设置的开发环境中的Node.js安装的访问。

项目

描述

SSH凭据

选择此选项可通过SSH凭据配置对远程主机上的Node.js解释器的访问。在对话框的字段中，指定以下内容：

  • 主机：在此字段中，键入安装解释器的主机的名称。

  • 端口：在此字段中，键入远程主机上的SSH服务器侦听的端口。默认端口号为22。

  • 用户名：在该字段中，键入在SSH服务器上注册的用户名。

  • 验证类型：从此列表中选择验证方法。

    • 密码 ：使用密码访问主机。指定密码，然后选择保存密码复选框以将密码保存在PyCharm中。

    • 密钥对（OpenSSH或PuTTY） ：使用密钥对的SSH身份验证 。要应用此身份验证方法，您必须在客户端计算机上拥有私钥，并在远程服务器上拥有公钥。PyCharm支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，并在相应字段中键入密码（如果有）。要让PyCharm记住密码，请选中Save passphrase复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭据帮助应用程序管理的SSH密钥（例如，Windows上的Pageant或Mac和Linux上的ssh-agent ）。

要使用解释器配置，您需要路径映射来设置项目文件夹之间的对应关系，服务器上用于复制项目文件的文件夹，以及用于访问服务器上复制数据的URL地址。PyCharm首先尝试通过处理所有可用的应用程序级配置来检索路径映射。如果PyCharm发现具有与上面指定的主机相同的主机的配置，则在“ 主机”字段中，将自动合并这些配置的映射。如果未找到与此主机的配置，PyCharm将显示一条错误消息，通知您未配置路径映射。

要解决此问题，请打开“ 构建，执行，部署”节点下的“ 部署”页面，选择服务器访问配置，切换到“ 映射”选项卡，然后将本地文件夹映射到服务器上的文件夹，如创建远程服务器配置 ，“ 映射”一节中所述。 本地文件夹到服务器上的文件夹和URL地址访问它们 。

流浪汉

仅当安装并启用Vagrant存储库插件时，此选项才可用。该插件不与PyCharm捆绑在一起，但它可以从JetBrains插件存储库安装，如管理插件中所述 。

选择此选项可使用您的Vagrant凭据配置对Vagrant实例中安装的Node.js解释器的访问。从技术上讲，它是所需环境的VagrantFile配置文件所在的文件夹。根据此设置，PyCharm会检测Vagrant主机并将其显示为Vagrant Host URL只读字段中的链接。

要使用解释器配置，您需要路径映射来设置项目文件夹之间的对应关系，服务器上用于复制项目文件的文件夹，以及用于访问服务器上复制数据的URL地址。PyCharm评估来自VagrantFile配置文件的路径映射。

部署配置

仅当启用了“ 远程主机访问”插件时，此选项才可用。该插件默认激活。如果插件已禁用，请在“ 插件”页面上启用它，如管理插件中所述 。

选择此选项可使用服务器访问配置配置对远程主机上的Node.js解释器的访问 。仅当您具有至少一个SFTP类型的服务器访问配置时 ，此选项才可用，请参阅创建远程服务器配置 。

从“ 部署配置”下拉列表中，选择您希望PyCharm连接到目标主机的SFTP类型的服务器访问配置 。如果所选配置中指定的设置确保连接成功，则PyCharm会在“ 部署主机URL”字段中将目标主机的URL地址显示为链接。

要使用解释器配置，您需要路径映射来设置项目文件夹之间的对应关系，服务器上用于复制项目文件的文件夹，以及用于访问服务器上复制数据的URL地址。默认情况下，PyCharm从所选的服务器访问（部署）配置中检索路径映射。如果配置不包含路径映射，PyCharm将显示相应的错误消息。

要解决此问题，请打开“ 构建，执行，部署”节点下的“ 部署”页面，选择相关的服务器访问配置，切换到“ 映射”选项卡，然后将本地文件夹映射到服务器上的文件夹，如“ 映射本地文件夹”中所述服务器上的文件夹和访问它们的URL地址部分。

搬运工人

仅当启用了Node.js ， Node.js远程解释器和Docker集成插件时，此选项才可用。默认情况下会激活插件。如果禁用了插件，请在“ 插件”页面上启用它们，如管理插件中所述 。

选择此选项可配置对在Docker容器中运行的Node.js解释器的访问。

  1. 在“ 服务器”字段中，指定要使用的Docker配置 ，有关详细信息，请参阅使用Docker：流程概述 。从列表中选择配置或单击并在打开的Docker对话框中创建一个新配置。

  2. 在“ 图像名称”字段中，指定要使用的基本Docker镜像。从下拉列表中选择以前下载的或自定义图像之一，或手动键入图像名称，例如node:argon或mhart/alpine-node 。稍后启动运行配置时，Docker将在您的计算机上搜索指定的映像。如果搜索失败，将从注册表页面上指定的映像存储库下载映像。

  3. Node.js解释器路径字段显示指定映像中默认Node.js解释器的位置。

  4. 单击“ 确定”后 ，PyCharm将关闭“ 配置Node.js远程解释器”对话框 ，并将您带到Node.js解释器对话框，其中新的解释器配置将添加到列表中。单击“ 确定”返回运行配置。

Node.js解释器路径

在此字段中，根据所选远程开发环境的配置指定Node.js可执行文件的位置。

  • 对于远程主机，PyCharm默认建议/ usr / bin / node文件夹。

  • 对于Vagrant实例，PyCharm默认建议/ usr / bin / node文件夹。

  • 对于Docker容器，默认情况下PyCharm会建议节点文件夹。

单击“ 确定”后 ，PyCharm将检查Node.js可执行文件是否实际存储在指定的文件夹中。

  • 如果找不到Node.js可执行文件，PyCharm会显示一条错误消息，询问您是继续搜索还是保存解释器配置。

  • 如果找到Node.js可执行文件，则返回Node.js Interpreters对话框，其中显示安装文件夹和检测到的Node.js解释器版本。
