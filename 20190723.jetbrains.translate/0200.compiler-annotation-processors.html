注释处理器设置

项目

描述

启用注释处理

如果要在编译期间运行注释处理器，请选中此复选框并指定关联选项：

  • 从项目类路径获取处理器 ：如果希望IntelliJ IDEA从项目类路径获取注释处理器，请选择此选项。例如，如果您将自定义注释处理器用作项目的一部分，或者处理器存储在作为库附加到所有相应模块的.jar文件中，则这很有用。

  • 处理器路径 ：如果不希望将处理器包含在项目库或项目库中，请选择此选项并在右侧的字段中指定注释处理器的路径。

  • 存储生成的源相对于 ：使用下面的字段来定义由注释处理器生成的源的存储位置，并覆盖配置文件的默认行为。

    • 模块输出目录 ：默认情况下，注释处理器生成的源相对于模块输出目录存储。

    • 模块内容根 ：选择此选项可覆盖配置文件的默认行为。

    稍后，如果要将生成的类用作自己的源，可以将相应的目录标记为源根。

    在重建时，存储生成的源的目录将作为普通输出目录进行清理。因此，建议不要在此类目录中存储非生成的源。否则，重建时将丢失相应的源。

处理器FQ名称

指定处理器完全限定名称。

使用 （ Alt + Insert ）或 （ Alt + Delete ）组成要运行的注释处理器列表。

注释处理器选项

如有必要，请将处理器运行选项指定为-key=value或key=value 。

使用空格分隔各个选项。

使用 （ Alt + Insert ）或 （ Alt + Delete ）组成要传递给注释处理器的选项列表。
