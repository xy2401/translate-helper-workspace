云

档案|设置|构建，执行，部署|适用于Windows和Linux的云
IntelliJ IDEA |偏好|构建，执行，部署| macOS的云

使用此页面可管理您的云访问配置。每种配置都包括您的云用户帐户信息和相关设置。

要创建新配置，请单击并选择感兴趣的云平台。（当您开始创建云访问配置时，您必须已拥有相应的云用户帐户。请参阅使用云平台 。）

  • CloudBees的
  • Cloud Foundry
  • Google App Engine
  • Heroku的
  • OpenShift
  • 工具栏

CloudBees的

指定您的CloudBees用户帐户详细信息和相关设置。有关其他信息，请参阅CloudBees文档 。

项目

描述

名称

CloudBees配置的名称。

域

您的CloudBees域。

用户名（电子邮件）

您的CloudBees帐户的用户名（例如，您的电子邮件地址）。

密码

您的CloudBees帐户的密码。

Cloud Foundry

指定Cloud Foundry用户帐户详细信息和相关设置。有关其他信息，请参阅Cloud Foundry文档 。

项目

描述

名称

Cloud Foundry配置的名称。

版

您正在使用的Cloud Foundry版本（也称为Service Broker API版本）：

  • 版本1的V1 。

  • 版本2的V2 。

用户名（电子邮件）

您的Cloud Foundry实例的用户名（例如，您的电子邮件地址）。

密码

您的Cloud Foundry实例的密码。

提供者（API URL）

Cloud Foundry实例中的Cloud Controller的URL（也称为API端点或目标URL）。

信任自签名证书

对于版本2（ V2 ）：选中此复选框以查看要信任的自签名云身份验证证书。否则，将拒绝自签名证书。

组织

对于版本2（ V2 ）：目标组织（用于部署应用程序）。

空间

对于版本2（ V2 ）：目标空间（在组织内）。

云实例

对于版本1（ V1 ）：以下之一：

  • 全球托管的Cloud Foundry。

  • 用于Cloud Foundry VCAP安装的本地（vcap.me） 。

  • Micro-offline for Micro Cloud Foundry在离线模式下使用。

港口

对于Cloud Foundry VCAP实例（ Local（vcap.me） ）：http端口（默认为80 ）。

域

对于Micro Cloud Foundry实例（ Micro - offline ）：与您的Micro Cloud Foundry实例关联的域名。

Google App Engine

指定您的Google App Engine云用户帐户详细信息和相关设置。有关其他信息，请参阅Google App Engine Cloud文档 。

项目

描述

名称

Google App Engine配置的名称。

通过OAuth2使用无密码登录

选择此选项以使用OAuth 2.0身份验证。

用电子邮件和密码登录

选择此选项可使用您的电子邮件和密码登录云。

电子邮件

您的Google电子邮件地址。

密码

你的密码。

记住密码

如果您希望IntelliJ IDEA记住您的密码（例如将其保存在密码数据库中），请选中此复选框。请参阅密码 。

Heroku的

指定您的Heroku用户帐户详细信息和相关设置。有关其他信息，请参阅Heroku文档 。

项目

描述

名称

Heroku配置的名称。

用户名（电子邮件）

您的Heroku帐户的用户名（例如，您的电子邮件地址）。

密码

您的Heroku帐户的密码。

API密钥

API令牌用于验证Heroku API请求。

上传公共SSH密钥

为了能够使用云，您应该将公共SSH密钥上传到它。（如果您之前这样做，则没有必要这样做。）

单击链接，单击在“上载公共SSH密钥”对话框中，在打开的对话框中选择公共SSH密钥文件（ .pub ）。

指定您的OpenShift用户帐户详细信息和相关设置。有关其他信息，请参阅OpenShift文档 。

项目

描述

'oc'的路径
