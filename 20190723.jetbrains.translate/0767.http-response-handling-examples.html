检查响应标头，正文和内容类型

在这个例子中，我们将创建几个测试来验证以下内容：

  • 请求成功执行，即响应状态为200 。

  • 在响应正文中收到标题。

  • 响应的内容类型是application/json 。

要创建测试，我们调用客户端对象的test方法。在测试中，我们可以通过调用client对象的assert方法来声明特定条件，并引用响应对象的各种属性来验证它们。

使用环境变量

在这个例子中，我们将从接收到的响应中捕获一个值到一个环境变量中，然后可以在后续请求中使用它。

所述第一请求包括一个响应处理程序脚本，从而节省了从所接收的响应身体的认证令牌到auth_token下可变client.global 。为此，我们使用client.global.set(VariableName, VariableValue)构造：

POST https://httpbin.org/post Content-Type：application / json { “token” ： “my-secret-token” } //保存变量> { ％client.global.set （ “auth_token” ，response。 body.json.token ） ; ％ }

执行请求后，可以通过client.global.get("auth_token")构造从后续请求{{auth_token}}和响应处理程序脚本访问auth_token变量。

//访问一个变量 GET HTTPS：//httpbin.org/headers 授权 ： 承载 {{ 的auth_token}}

要从响应头获取值，请使用headers对象的valueOf方法。如果收到多个具有相同名称的标头（这是Set-Cookie标头的常见情况），请改用valuesOf方法。这将返回所有响应头值的数组。

POST https://httpbin.org/cookies //从第一个Set-Cookie标题中保存一个cookie> { ％client.global.set （ “my_cookie” ，response.headers.valuesOf （ “Set-Cookie” ）[ 0 ] ） ; ％ }
