使用PyCharm进行远程调试

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

使用PyCharm，您可以使用位于另一台计算机上的解释器调试应用程序，例如，在Web服务器或专用测试机器上。

PyCharm提供了两种远程调试方法：

  • 通过远程口译员 。

    案例：使用此方法可利用远程计算机上可用的扩展调试功能。

    要求：从本地计算机到远程服务器的SSH访问。

  • 使用Python 远程调试服务器配置。

    案例：使用此方法将调试过程集成到远程服务器上的一系列正在运行的进程中。当您无法显式运行应用程序进行调试或需要某些准备任务时，这可能会有所帮助。

    要求：从本地计算机到远程服务器的SSH访问，使用任何预定义端口从远程服务器访问本地计算机。

为远程解释器创建部署配置

在此示例中，运行应用程序的计算机被引用为本地计算机，具有远程解释程序的计算机被引用为远程计算机 。

配置远程解释器

  1. 确保您具有对远程计算机的SSH访问权限。

  2. 按照使用SSH配置远程解释器指定连接到远程计算机的凭据中所述，向项目添加新的远程解释器。

  3. 为项目创建远程解释器后，将创建相应的部署配置。要预览它，请单击Ctrl + Alt + S以打开本地计算机上的“设置”对话框窗口，然后单击“ 构建”，“执行”，“部署”节点和“ 部署”节点。

  4. 如果需要，您可以接受所有默认设置或更改它们。对于此示例，让我们为部署配置使用有意义的名称，例如“MySFTPConnection”。

  5. 确保根路径值反映在创建的SSH解释器的相应设置中指定的路径（检查项目|项目解释器设置/首选项中的路径解释 器 ）。

现在您的部署配置已准备就绪。

将应用程序部署到远程主机

接下来，必须将应用程序部署到远程主机。

  1. 在“ 工具”菜单上，选择“ 部署”|上传到MySFTPConnection 。

  2. 出现文件传输工具窗口。验证传输的文件数。

调试您的应用程序

  1. 右键单击编辑器背景，然后选择Debug <name> （此处为Debug'quadratic_equation' ）。

  2. 查看调试输出。请注意，调试实际上发生在指定的远程服务器上。

创建SFTP连接

  1. 在远程计算机上，创建一个应该上载文件quadratic_equation.py的目录。您可以在终端窗口中执行此操作：

    $ cd / tmp $ mkdir pycharm_project_986

  2. 在本地计算机上，创建连接配置文件。在主菜单上，选择“ 工具”|部署 - 配置......在打开的对话框中，单击 ，在“ 添加服务器”对话框中选择连接类型（此处为SFTP）并输入其名称（此处为MySFTPConnection 。）

  3. 在“ 连接”选项卡中，指定SFTP主机（远程计算机的地址），该计算机的用户名和密码。

    请注意，指定的用户应具有对远程主机的SSH访问权限。

  4. 单击“ 映射”选项卡，然后在服务器中输入部署路径。服务器是MySFTPConnection ，因此单击浏览按钮并选择所需的文件夹（ / tmp / pycharm_project_986 ）。请注意，浏览按钮显示远程主机的内容。应用更改并关闭对话框。

将文件部署到远程计算机

将以下文件部署到远程计算机： pydevd-pycharm.egg和quadratic_equation.py 。

  1. 在本地计算机的“ 项目工具”窗口中 ，选择文件，右键单击所选内容，然后选择“ 部署”|上传到MySFTPConnection 。

  2. 检查“ 文件传输”对话框窗口，确保将本地计算机上的文件上载到远程服务器。

启动调试服务器

  1. 选择创建的运行/调试配置，然后单击 ：

  2. 确保“ 调试工具”窗口显示“ 等待进程连接...”消息。在远程计算机上启动脚本之前，将显示此消息，此脚本将连接到调试服务器。

在远程计算机上执行Python文件

  1. 在远程计算机上，导航到tmp/pycharm_project_986目录。

  2. 在远程主机上启动quadratic_equation.py文件。为此，请在“终端”窗口中输入以下命令：

对于Python 3

$ python3 quadratic_equation.py

对于Python 2

$ python quadratic_equation.py

此调试方法最有用的方面是，当远程调试是计划任务的一部分或需要在运行Python脚本之前执行一些准备步骤时，可以使用任何bash脚本运行Python文件。如果是这种情况，请将以下行添加到bash脚本的适当位置：

对于Python 3

cd / tmp / pycharm_project_986 python3 quadratic_equation.py

对于Python 2

cd / tmp / pycharm_project_986 python quadratic_equation.py

调试应用程序

  • 在本地计算机上，切换到“ 调试工具”窗口 。它应该显示与pydev调试器的连接。

    您的代码实际上是在远程主机上执行的，但在本地计算机上进行了调试。

摘要

为了使用远程解释器进行调试，您必须通过PyCharm启动程序，这并不总是可行的。另一方面，使用调试服务器时，您可以连接到正在运行的进程。

比较两种方法。

在第一种情况下，我们

  • 配置了远程解释器 。

  • 将脚本部署到远程主机 。

  • 调试了脚本 。

在第二种情况下，我们

  • 创建了一个调试配置 （Debug Server）。

  • 启动了调试服务器 。

  • 在远程计算机上执行Python脚本 。该脚本连接到调试服务器。

  • 调试本地计算机上的脚本 。
