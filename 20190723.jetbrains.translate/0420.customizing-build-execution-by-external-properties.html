通过外部配置属性来自定义构建执行

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

Phing构建文件通常包含属性元素 ，可以在构建文件本身中手动配置，也可以在Phing Build工具窗口中外部配置。外部配置的属性在启动构建执行时通过命令行传递给Phing，因此每次动态重新计算它们。外部配置的属性始终覆盖在构建文件中手动设置的属性。

在这个部分：

  • 访问Phing Buide工具窗口
  • 在外部配置属性

访问Phing Buide工具窗口

  1. 在编辑器中打开Phing构建文件或在“ 项目”工具窗口中选择它，然后从选择的上下文菜单中选择“ 添加为Phing构建文件 ”。

  2. 选择查看|工具窗口| Phing Build在主菜单上。在编辑器或“项目”工具窗口中通过Phing构建文件的上下文菜单打开工具窗口后，可以访问该工具窗口。

在外部配置属性

  1. 打开Phing Build工具窗口。

  2. 在列表中选择所需的构建文件，然后单击“ 设置”按钮在工具栏上。

  3. 在打开的Phing Settings对话框中，切换到Properties选项卡并配置要通过命令行传递给Phing的列表。

    1. 要向列表中添加新属性，请单击“ 添加”按钮。

    2. 在“ 属性”文本框中，键入属性的名称。

    3. 在“ 值”文本框中，指定要传递的属性值。执行以下操作之一：

      • 手动键入值。

      • 要使用IntelliJ IDEA宏 ，请单击“ 插入宏”按钮并在打开的“ 宏”对话框中配置相关宏定义的列表。要添加宏，请在列表中选择它，然后单击“ 插入” 。

  4. 准备好属性列表后，在“ Phing设置”对话框中单击“ 确定 ”。
