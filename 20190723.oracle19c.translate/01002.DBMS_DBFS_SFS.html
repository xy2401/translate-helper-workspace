52 DBMS_DBFS_SFS

DBMS_DBFS_SFS包提供了一个接口，用于为DBMS_DBFS_CONTENT包中描述的内容接口操作基于SecureFile的存储（SFS）。

52.1 DBMS_DBFS_SFS概述

DBMS_DBFS_SFS包是实现和扩展DBMS_DBFS_CONTENT_SPI接口的包的示例实现。它提供了存储在RDBMS中的符合POSIX的文件系统。

52.2 DBMS_DBFS_SFS安全模型

DBMS_DBFS_SFS包与AUTHID CURRENT_USER DBMS_DBFS_SFS运行。

52.4.1 CREATEFILESYSTEM过程

此过程创建文件系统存储。

句法

DBMS_DBFS_SFS.CREATEFILESYSTEM（store_name IN VARCHAR2，schema_name IN VARCHAR2 DEFAULT NULL，tbl_name IN VARCHAR2 DEFAULT NULL，tbl_tbs IN VARCHAR2 DEFAULT NULL，lob_tbs IN VARCHAR2 DEFAULT NULL，use_bf IN BOOLEAN DEFAULT FALSE，DBMS_DBFS_CONTENT_PROPERTIES_T DEFAULT NULL，create_only IN BOOLEAN FALSE，use_objects在BOOLEAN DEFAULT FALSE中，with_grants在BOOLEAN DEFAULT FALSE中，do_dedup在BOOLEAN DEFAULT FALSE中，do_compress IN BOOLEAN DEFAULT FALSE压缩在VARCHAR2 DEFAULT COMPRESSION_DEFAULT，do_encrypt IN BOOLEAN DEFAULT FALSE，加密IN VARCHAR2 DEFAULT ENCRYPTION_DEFAULT，do_partition IN BOOLEAN DEFAULT FALSE，npartition in NUMBER DEFAULTDEFAULT_PARTITIONS ，partition_key IN NUMBER DEFAULT PARTITION_BY_ITEM，partition_guidi IN BOOLEAN DEFAULT FALSE，partition_pathi IN BOOLEAN DEFAULT FALSE，partition_prop IN BOOLEAN DEFAULT TRUE）;

参数

表52-6 CREATEFILESYSTEM过程参数

参数 描述

STORE_NAME

商店名称

SCHEMA_NAME

存储的架构，默认为当前架构

tbl_name

商店条目表。如果未指定，则使用内部生成的名称。

tbl_tb

存储的表空间，默认为模式的默认表空间

lob_tbs

用于创建LOB段的表空间。它默认为用户的默认表空间。

use_bf

如果为TRUE ，则使用BasicFile LOB ;否则使用SecureFile LOB 。

性能

用于配置商店属性的（名称，值，类型代码）元组的表。目前，未定义或使用此类属性。

create_only

如果为TRUE ，则创建文件系统，但不向当前用户注册

use_objects

如果为TRUE ，则创建具有对象类型列（使用嵌套表）的单个基表以备份新文件系统。否则，使用一对（父，子）表来备份文件系统。在这两种情况下，对象类型嵌套表或子表仅用于用户定义的属性。

with_grants

如果为TRUE ，则在创建文件系统时将DBFS_ROLE授予DML和查询访问权限。否则，访问文件系统需要显式授权（或现有权限）。

do_dedup

如果为TRUE ，请对基础SecureFile列执行重复数据删除

do_compress

如果为TRUE ，则对基础SecureFile列进行压缩

压缩

使用压缩算法（ 见表52-1 ）

do_encrypt

如果为TRUE ，则加密基础SecureFile列

加密

加密算法使用（ 见表52-2 ）

do_partition

如果为TRUE ，则对用于存储的表进行分区

nPartition的

为表创建的分区数（ 参见表52-3 ）。

partition_key

如何对表进行分区：按项目名称，按路径名称或按GUID进行分区（请参阅表52-4 ）。

partition_guidi

如果为TRUE ，则在GUID上构建索引

partition_pathi

如果为TRUE ，则在路径名上构建索引

partition_prop

如果为TRUE ，则对属性表进行分区

使用说明

该过程像DDL一样执行，因为它在执行之前和之后自动提交。

52.4.3 DROPFILESYSTEM程序

此过程将删除DBFS SFS存储，清除与存储关联的所有字典信息，并删除基础文件系统表。

句法

DBMS_DBFS_SFS.DROPFILESYSTEM（schema_name IN VARCHAR2 DEFAULT NULL，tbl_name IN INTEGER）; DBMS_DBFS_SFS.DROPFILESYSTEM（store_name IN VARCHAR2）;

参数

表52-8 DROPFILESYSTEM过程参数

参数 描述

SCHEMA_NAME

架构名称

tbl_name

表空间的名称

STORE_NAME

商店路径的名称

使用说明

  • 如果指定的存储表由当前用户注册，则将从DBMS_DBFS_CONTENT包和POSIX元数据表中描述的内容接口取消注册。

  • 在取消注册之后，将尝试存储表格。如果其他用户当前正在使用此商店表，则此操作可能会失败。

  • 尝试删除商店下面的表的用户实际上必须具有完成删除操作的权限（作为表的所有者，或作为跨模式操作的足够特权的用户）。

  • 该过程像DDL一样执行，因为它在执行之前和之后自动提交。
