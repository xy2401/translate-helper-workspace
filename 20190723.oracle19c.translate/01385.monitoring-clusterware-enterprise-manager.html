5.1使用Oracle Enterprise Manager监控Oracle Clusterware

使用Oracle Enterprise Manager监视Oracle Clusterware环境。

使用客户端浏览器登录Oracle Enterprise Manager时，将显示Cluster Database Home页面，您可以在其中监视Oracle Database和Oracle Clusterware环境的状态。Oracle Clusterware监控包括以下详细信息：

  • 群集目标上Oracle Enterprise Manager中的当前和历史群集运行状况监视器数据

  • 通知是否有任何VIP重新安置

  • 使用通过群集验证实用程序（CVU）获取的信息在群集的每个节点上使用Oracle Clusterware的状态

  • 节点应用程序（ nodeapps ）启动或停止时的通知

  • Oracle Clusterware警报日志中的问题通知，Oracle Cluster Registry，投票文件问题（如果有）和节点驱逐

Cluster Database Home页面类似于单实例Database Home页面。但是，在Cluster Database主页上，Oracle Enterprise Manager显示系统状态和可用性。系统状态和可用性包括有关警报消息和作业活动的摘要，以及指向所有数据库和Oracle自动存储管理（Oracle ASM）实例的链接。例如，跟踪群集上的服务问题，包括服务未在所有首选实例上运行或何时未满足服务响应时间阈值。

使用Oracle Enterprise Manager Interconnects页面监视Oracle Clusterware环境。“互连”页面显示以下详细信息：

  • 集群上的公共和专用接口

  • 专用互连的总吞吐量

  • 每个网络接口上的单独吞吐量

  • 错误率（如果有的话）

  • 由互连上的数据库实例提供的负载

  • 如果数据库实例由于配置错误而使用公共接口，则发出通知

  • 互连上各个实例提供的吞吐量

前面列出的所有信息也可用作具有历史视图的集合。历史视图对于群集缓存一致性非常有用，例如在诊断与群集等待事件相关的问题时。单击Cluster Database主页上的Interconnect选项卡，访问Interconnects页面。

此外，Oracle Enterprise Manager 群集数据库性能页面还提供了数据库性能统计信息的快速概览。统计信息将在图表中的群集数据库中的所有实例中汇总。使用图表旁边的链接，您可以获得更具体的信息并执行以下任何任务：

  • 确定性能问题的原因

  • 决定是否必须添加或重新分配资源

  • 调整SQL计划和架构以获得更好的优化

  • 解决性能问题

“群集数据库性能”页面上的图表包括以下内容：

  • 图群集主机负载平均 ：群集数据库性能页面群集主机负载平均线图显示，在数据库以外的潜在问题。该图表显示前一小时群集中可用节点的最大，平均和最小负载值。

  • 全局缓存块访问延迟的图表 ：每个群集数据库实例在其系统全局区域（SGA）中都有自己的缓冲区缓存。使用缓存融合，Oracle RAC环境在逻辑上组合每个实例的缓冲区缓存，以使数据库实例能够处理数据，就像数据驻留在逻辑上组合的单个缓存一样。

  • 图表平均活动会话 ：群集数据库性能页面的平均活动会话图表显示了数据库内部潜在的问题。类别（称为等待类）显示数据库使用资源的程度，例如CPU或磁盘I / O.将CPU时间与等待时间进行比较有助于确定有效工作消耗了多少响应时间，而不是等待其他进程可能拥有的资源。

  • 图表数据库吞吐量 ： 数据库的吞吐量图表总结出现的平均活动会话图表中的任何资源争用，并且还显示数据库多少工作执行代表用户或应用程序。“ 每秒”视图显示与登录次数相比的事务数，以及与每秒的重做大小相比的物理读取量。“ 每个事务”视图显示与每个事务的重做大小相比的物理读取量。登录是登录到数据库的用户数。

此外，“ 群集数据库性能”页面上的“ 顶级活动”下拉菜单使您可以通过等待事件，服务和实例查看活动。此外，您可以通过移动图表上的滑块来查看有关SQL /会话的详细信息。
