3.53 ALL_SCHEDULER_JOBS

ALL_SCHEDULER_JOBS显示有关当前用户可访问的Scheduler作业的信息。

相关观点

  • DBA_SCHEDULER_JOBS显示有关数据库中所有Scheduler作业的信息。

  • USER_SCHEDULER_JOBS显示有关当前用户拥有的Scheduler作业的信息。此视图不显示OWNER列。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

调度程序作业的所有者

JOB_NAME

VARCHAR2（128）

调度程序作业的名称

JOB_SUBNAME

VARCHAR2（128）

调度程序作业的子名称（用于运行链式步骤的作业）

JOB_STYLE

VARCHAR2（17）

工作作风：

  • 定期

  • 轻量级

  • IN_MEMORY_RUNTIME

  • IN_MEMORY_FULL

JOB_CREATOR

VARCHAR2（128）

这份工作的原始创造者

CLIENT_ID

VARCHAR2（65）

创建作业的用户的客户端标识符

GLOBAL_UID

VARCHAR2（33）

创建作业的用户的全局用户标识符

PROGRAM_OWNER

VARCHAR2（4000）

与作业关联的程序的所有者

PROGRAM_NAME

VARCHAR2（4000）

与作业关联的程序的名称

工作类型

VARCHAR2（16）

内联作业操作类型：

  • PLSQL_BLOCK

  • STORED_PROCEDURE

  • EXECUTABLE

  • 链

  • SQL_SCRIPT

  • BACKUP_SCRIPT

  • EXTERNAL_SCRIPT

JOB_ACTION

VARCHAR2（4000）

内联工作行动

NUMBER_OF_ARGUMENTS

数

内联的作业参数数量

SCHEDULE_OWNER

VARCHAR2（4000）

作业使用的计划的所有者（可以是窗口或窗口组）

SCHEDULE_NAME

VARCHAR2（4000）

作业使用的计划的名称（可以是窗口或窗口组）

时间安排：SCHEDULE_TYPE

VARCHAR2（12）

作业使用的计划类型：

  • IMMEDIATE - 开始日期和重复间隔为NULL

  • ONCE - 重复间隔为NULL

  • PLSQL - 用作计划的PL / SQL表达式

  • CALENDAR - 用作计划的Oracle日历表达式

  • EVENT -事件时间表

  • NAMED - 命名时间表

  • WINDOW - 用作计划的窗口

  • WINDOW_GROUP - 用作计划的窗口组

开始日期

TIMESTAMP（6）带时区

作业的原始预定开始日期（对于内联计划）

REPEAT_INTERVAL

VARCHAR2（4000）

内联计划PL / SQL表达式或日历字符串

EVENT_QUEUE_OWNER

VARCHAR2（128）

将引发事件的源队列的所有者

EVENT_QUEUE_NAME

VARCHAR2（128）

将引发事件的源队列的名称

EVENT_QUEUE_AGENT

VARCHAR2（256）

用户在事件源队列上使用的AQ代理的名称（如果它是安全队列）

EVENT_CONDITION

VARCHAR2（4000）

布尔表达式，用作源队列上事件的预订规则

EVENT_RULE

VARCHAR2（261）

协调器用于触发基于事件的作业的规则的名称

FILE_WATCHER_OWNER

VARCHAR2（261）

此作业所基于的文件观察程序的所有者

FILE_WATCHER_NAME

VARCHAR2（261）

此作业所基于的文件观察程序的名称

结束日期

TIMESTAMP（6）带时区

作业将不再运行的日期（对于内联计划）

JOB_CLASS

VARCHAR2（128）

与作业关联的作业类的名称

启用

VARCHAR2（5）

指示作业是启用（ TRUE ）还是禁用（ FALSE ）

AUTO_DROP

VARCHAR2（5）

指示作业在完成（ TRUE ）之后是否将被删除（ FALSE ）

RESTART_ON_RECOVERY

VARCHAR2（5）

指示是否应在数据库恢复（ TRUE ）上重新启动该步骤（ FALSE ）

RESTART_ON_FAILURE

VARCHAR2（5）

指示是否应在应用程序失败（ TRUE ）时重新启动该步骤（ FALSE ）

州

VARCHAR2（15）

目前的工作状态：

  • 禁用

  • 重新计划

  • 排定

  • BLOCKED

  • RUNNING

  • 已完成

  • 破碎

  • 失败

  • 远程

  • RESOURCE_UNAVAILABLE

  • SUCCEEDED

  • CHAIN_STALLED

JOB_PRIORITY

数

相对于同一班级中的其他工作，工作的优先级

RUN_COUNT

数

作业运行的次数

UPTIME_RUN_COUNT

数

自上次重新启动数据库以来的运行次数。对于内存中作业，将填充此列，但不会填充RUN_COUNT列。

对于所有其他作业，此列为NULL 。

MAX_RUNS

数

作业计划运行的最大次数

FAILURE_COUNT

数

作业无法运行的次数

UPTIME_FAILURE_COUNT

数

自上次重新启动数据库以来的失败次数。对于内存中作业，将填充此列，但不会填充FAILURE_COUNT列。

对于所有其他作业，此列为NULL 。

MAX_FAILURES

数

在标记为已损坏之前允许作业失败的次数

RETRY_COUNT

数

作业重试的次数，如果正在重试

LAST_START_DATE

TIMESTAMP（6）带时区

作业开始运行的最后日期

LAST_RUN_DURATION

间隔日（9）至第二（6）

在上次运行期间完成作业所需的时间

NEXT_RUN_DATE

TIMESTAMP（6）带时区

计划运行作业的下一个日期

SCHEDULE_LIMIT

间隔日（3）至第二（0）

一段时间之后，尚未运行的工作将被重新安排

MAX_RUN_DURATION

间隔日（3）至第二（0）

允许作业运行的最长时间

LOGGING_LEVEL

VARCHAR2（11）

将完成与作业有关的日志记录量：

  • 关闭

  • 游程

  • 失败的运行

  • 充分

STORE_OUTPUT

VARCHAR2（5）

指示作业的所有作业输出消息是否都存储在*_JOB_RUN_DETAILS视图的OUTPUT列中，以用于记录的作业运行。可能的值：

  • TRUE ：作业的所有作业输出消息都存储在*_JOB_RUN_DETAILS视图的OUTPUT列中，用于记录的作业运行。这是新作业的默认值。新工作是使用Oracle Database 12 c软件创建的工作。

  • FALSE ：作业的作业输出消息未存储在*_JOB_RUN_DETAILS视图的OUTPUT列中。这是现有作业的默认值。现有作业是使用Oracle Database 12 c之前的软件创建的作业。

STOP_ON_WINDOW_CLOSE

VARCHAR2（5）

指示如果与作业关联的窗口关闭（ TRUE ）（ FALSE ），作业是否将停止（ FALSE ）

INSTANCE_STICKINESS

VARCHAR2（5）

指示作业是粘性（ TRUE ）还是不粘性（ FALSE ）

RAISE_EVENTS

VARCHAR2（4000）

为工作筹集的工作活动清单：

  • JOB_SUCCEEDED

  • JOB_FAILED

  • JOB_BROKEN

  • JOB_COMPLETED

  • JOB_CHAIN_STALLED
