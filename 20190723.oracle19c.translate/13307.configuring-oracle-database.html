3配置Oracle数据库

本章介绍如何配置Oracle Database for Oracle产品。它包含以下部分：

  • 使用配置助手

  • 重新链接可执行文件

3.1使用配置助手

Oracle数据库软件随附配置助手，可用于执行各种数据库管理操作。

本节包含以下主题：

  • 使用Oracle Net Configuration Assistant

  • 使用Oracle数据库升级助手

  • 使用Oracle数据库配置助手

3.1.1使用Oracle Net Configuration Assistant

Oracle Net Configuration Assistant使您能够在安装期间配置基本网络组件，包括侦听器名称和协议地址，命名方法， tnsnames.ora文件中的网络服务名称以及目录服务器使用情况。

安装完成后，您可以通过输入以下命令使用Oracle Net Configuration Assistant创建更详细的配置：

$ $ ORACLE_HOME / bin / netca

注意：

使用DBCA创建数据库时，它会自动更新网络配置文件以包含新数据库的信息。

3.1.2使用Oracle数据库升级助手

Oracle数据库升级助手（DBUA）以交互方式指导您完成数据库升级并为新版本配置数据库。DBUA通过执行通常手动执行的所有任务来自动执行升级。DBUA为表空间和联机重做日志等配置选项提供建议。

要启动DBUA，请运行以下命令：

$ $ ORACLE_HOME / bin / dbua

有关DBUA可用的命令行选项的信息，请使用-help或-h命令行参数。例如：

$ $ ORACLE_HOME / bin / dbua -help

3.1.3使用Oracle数据库配置助手

Oracle数据库配置助手（DBCA）是一个图形用户界面，可帮助您创建和配置默认数据库或自定义数据库。它还使您能够配置现有数据库以添加Oracle数据库功能，并创建Oracle自动存储管理磁盘组。DBCA还会生成一组shell和SQL脚本，您可以在以后检查，修改和运行这些脚本以创建数据库。

要启动DBCA，请运行以下命令：

$ $ ORACLE_HOME / bin / dbca

有关DBCA可用的命令行选项的信息，请使用-help或-h命令行参数。例如：

$ $ ORACLE_HOME / bin / dbca -help

3.2重新链接可执行文件

您可以使用位于$ORACLE_HOME/bin目录中的relink shell脚本手动重新链接产品可执行文件。每次应用操作系统修补程序或操作系统升级后，都必须重新链接产品可执行文件。

注意：

在重新链接可执行文件之前，必须关闭在Oracle主目录中运行的所有重新链接可执行文件。此外，关闭与Oracle共享库链接的应用程序。relink脚本将all和as_installed作为参数。如果未指定任何参数，则考虑all参数。

根据Oracle主目录中已安装的产品， relink脚本会relink所有Oracle产品可执行文件。
