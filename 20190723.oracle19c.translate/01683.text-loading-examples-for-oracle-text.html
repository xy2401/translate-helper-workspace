C.1 SQL INSERT示例

一个简单的方法来填充文本表创建一个表有两列， id和text ，使用CREATE TABLE ，然后使用INSERT语句加载数据。此示例使id列成为主键，这是可选的。text列是VARCHAR2 ：

create table docs（id number primary key，text varchar2（80））;

要填充text列，请使用INSERT语句，如下所示：

插入docs值（1，'这是第一个文档的文本'）;插入文档值（12，'这是第二个文档的文本'）;

C.2.1创建表

此示例加载一个表articles_formatted ，如下所示：

CREATE TABLE articles_formatted（ARTICLE_ID NUMBER PRIMARY KEY，AUTHOR VARCHAR2（30），FORMAT VARCHAR2（30），PUB_DATE DATE，TITLE VARCHAR2（256），TEXT BLOB）;

article_id列是主键。文档将加载到text列中，该列的类型为BLOB 。

C.2.2发出SQL * Loader命令

以下命令启动加载程序，该加载程序读取控制文件LOADER1.DAT ：

sqlldr userid = demo / password control = loader1.dat log = loader.log

  • 示例控制文件：loader1.dat

  • 示例数据文件：loader2.dat

C.2.2.1控制文件示例： loader1.dat

此SQL * Loader控制文件定义要加载的列，并指示加载程序从loader2.dat加载数据到articles_formatted表中。loader2.dat每一loader2.dat包含一个逗号分隔的要加载的字段列表。 

- 加载文件示例加载数据INFILE'loading2.dat'INTO TABLE articles_formatted APPEND FIELDS TERMINATED BY'，'（article_id SEQUENCE（MAX，1），作者CHAR（30），格式，pub_date SYSDATE，title，ext_fname FILLER CHAR（80 ），文本LOBFILE（ext_fname）由EOF终止）

此控制文件指示加载程序以下列方式将数据从loader2.dat加载到articles_formatted表：

  1. 描述loader2.dat文档字段的行的序号位置被写入article_id列。

  2. 该行的第一个字段写入author列。

  3. 该行的第二个字段将写入format列。

  4. SYSDATE给出的当前日期将写入pub_date列。

  5. 文档的标题是该行的第三个字段，写入title列。

  6. 要加载的每个文档的名称都读入ext_fname 临时变量，并在text BLOB列中加载实际文档：

C.3.1导入文件格式

导入文件必须对同义词库中的条目使用以下格式：

phrase   BT broader_term   NT narrower_term1   NT narrower_term2 。 。 。
NT narrower_termN BTG broader_term NTG narrower_term1 NTG narrower_term2 。 。 。NTG narrower_termN BTP broader_term NTP narrower_term1 NTP narrower_term2 。 。 。NTP narrower_termN BTI broader_term NTI narrower_term1 NTI narrower_term2 。 。 。NTI narrower_termN SYN synonym1 SYN synonym2 。 。 。SYN synonymN使用synonym1或查阅synonym1 or PT synonym1 RT related_term1 RT related_term2 。 。 。RT related_termN SN text language_key：term
phrase

是一个词或短语，被定义为具有同义词，更宽泛的术语，更窄的术语或相关术语。

根据ISO-2788标准，可以在短语之前放置TT标记，以指示该短语是层次结构中的顶级术语;但是，不需要TT标记。实际上，ctxload在导入期间忽略TT标记。

最高术语被标识为没有更广泛术语（BT，BTG，BTP或BTI）的任何短语。

注意：

同义词查询运算符（ SYN ， PT ， BT ， BTG ， BTP ， BTI ， NT ， NTG ， NTP ， NTI和RT ）是保留字，因此不能用作同义词条目中的短语。

BT，BTG，BTP，BTI wider_termN

是指示wider_termN是短语的更广泛（通用|实例）术语的标记。

broader_termN 是一个词或短语，在概念上提供词组更一般的描述或类别。例如， 大象这个词可能有更广泛的陆地哺乳动物 。

NT，NTG，NTP，NTI narrower_termN

是指示narrower_termN是短语的较窄（通用|实例）术语的标记。

如果短语没有更广泛的（通用|实例）术语，但是具有一个或多个更窄（通用|实例）的术语，则在相应的层次结构中创建短语作为顶级术语（在Oracle Text同义词库中， BT / NT，BTG / NTG，BTP / NTP和BTI / NTI层次结构是分开的结构）。

narrower_termN是概念上为短语提供更具体描述的单词或短语 。例如， 大象这个词可能有一个较窄的印度语 大象和非洲大象 。

SYN同义词N.

是一个标记，表示短语和同义词N是同义词环内的同义词。

同义词N是对短语具有相同含义的单词或短语 。例如，单词dog可以具有犬的同义词。

注意：

同义词环未在Oracle Text叙词中明确定义。它们是由同义词的传递性质创造的。

使用SEE PT同义词1

是指示短语和synonym1是同义词环（类似于SYN）内的同义词标记。

标记USE，SEE或PT还表示同义词1是同义词环的优选术语。这些标记中的任何一个都可用于定义同义词环的首选术语。

注意：

如果要使用用户定义的同义词库来编译知识库，则必须在声明同义词环时指定首选术语。使用关键字USE，SEE或PT之一指定报告查询匹配时要使用的同义词。只有一个术语可以是首选术语。

不使用这些关键字之一可能导致无法返回由单词的同义词定义的结果。当编译声明相同同义词环的元素的两个或更多个叙词表时，两个文件中的首选术语必须相同，这确保只有一个单词被定义为同义词环中的首选单词。

RT related_termN

是标记，表示related_termN是短语的相关术语。

related_termN是具有与短语相关但不一定与短语同义的含义的单词或短语 。例如，单词dog可能有一个相关的狼术语。

注意：

相关术语不具有传递性。如果短语具有两个或更多相关术语，则这些术语仅与父短语相关，而不是彼此相关。

SN文字

是标记，指示以下文本是前一个条目的范围注释（例如，注释）。

language_key术语

term是将短语翻译成language_key指定的语言 。

C.3.2备用层次结构

符合thesauri标准，加载文件支持格式化层次结构（BT / NT，BTG / NTG，BTP，NTP，BTI / NTI），方法是缩进最高术语下的术语并使用NT（或NTG，NTP，NTI）标记包括该术语的级别：

phr ase NT1 narrower_term1 NT2 narrower_term1.1 NT2 narrower_term1.2 NT3 narrower_term1.2.1 NT3 narrower_term1.2.2 NT1 narrower_term2。 。 。
   NT1 narrower_termN

使用此方法，可以在加载文件中分层表示顶级项的整个分支。

C.3.3导入文件中术语的使用说明

以下条件适用于导入文件中条目的结构：

  • 每个条目（ 短语 ， BT，NT或SYN ）必须位于一行，后跟换行符。

  • 条目可以包含单个单词或短语。

  • 条目（ 短语，BT，NT或SYN ）的最大长度为255个字节，不包括BT，NT和SYN标记或换行符。

  • 条目不能包含括号或加号。

  • 以关系（ BT，NT等）开头的文件的每一行必须以至少一个空格开头。

  • 短语可以在文件中出现多次。

  • 每个短语可以具有一个或多个较窄的术语条目（ NT，NTG，NTP ），更宽泛的术语条目（ BT，BTG，BTP ），同义词条目和相关术语条目。

  • 每个更宽泛的术语，更窄的术语，同义词和首选术语输入必须以适当的标记开始，并且标记必须是大写字母。

  • 词组的更宽泛的术语，更窄的术语和同义词可以按任何顺序排列。

  • 在每个使用它们的地方，必须使用括号内的括号。

    例如： cranes (birds), cranes (lifting equipment)

  • 复合项用每个因子之间的加号表示（例如，建筑物+建筑）。

  • 复合词仅允许作为其他术语的同义词或首选术语，从不作为术语本身或在层次关系中。

  • 术语后面的行可以跟随范围注释（ SN ），总最大长度为2000字节。

  • 允许使用多行范围注释，但在注释的每一行上都需要SN标记。

SN使用不正确的示例：

VIEW CAMERAS SN相机通过镜头聚焦和镜头平面相对于胶片平面的一系列移动

正确SN使用示例：

VIEW CAMERAS SN相机通过镜头聚焦和镜头平面相对于镜头平面的SN运动的SN范围

  • 多字词不能以保留字开头（例如， 使用是保留字，因此使用其他门不是允许的术语;但是， 使用是允许的术语）。

C.3.4导入文件中关系的使用说明

以下条件适用于为导入文件中的条目定义的关系：

  • 相关术语条目必须遵循短语或其他相关术语条目

  • 相关术语条目以一个或多个空格开始， RT标记，后跟空格，然后是同一行上的相关术语

  • 多个相关术语需要多个RT标记

    不正确的RT使用示例：

    移动图像摄像机RT电视摄像机电视摄像机

    正确使用RT的示例：

    移动图像摄像机RT电视摄像机RT电视摄像机

  • 条款允许包含多个更广泛的术语，更窄的术语和相关术语
