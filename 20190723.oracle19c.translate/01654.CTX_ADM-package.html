5.1关于CTX_ADM包程序

CTX_ADM PL / SQL包提供管理索引首选项的管理过程。

CTX_ADM包中包含以下存储过程。

名称 描述

MARK_FAILED

将索引的状态从LOADING更改为FAILED 。

恢复

清除已删除文本表的数据库对象。

RESET_AUTO_OPTIMIZE_STATUS

重置CTX_AUTO_OPTIMIZE_STATUS视图。

SET_PARAMETER

设置索引创建的系统级默认值。

注意：

只有CTXSYS用户可以使用CTX_ADM包中的过程。

CTX_ADM程序包中的API不支持以架构或所有者名称为前缀的标识符。

5.2 MARK_FAILED

使用MARK_FAILED过程将索引的状态从LOADING更改为FAILED 。

在极少数情况下，如果CREATE INDEX或ALTER INDEX失败，索引可能会保留状态LOADING 。当索引处于LOADING状态时，将阻止使用RESUME INDEX进行恢复的任何尝试。对于这种情况，使用CTX_ADM.MARK_FAILED强制将状态从LOADING更改为FAILED以便您可以使用RESUME INDEX 。

您必须以CTXSYS身份登录才能运行CTX_ADM.MARK_FAILED 。

警告：

请谨慎使用CTX_ADM.MARK_FAILED 。它只应作为最后的手段使用，并且只有在没有其他会话触及索引时才会使用。通常，如果另一个会话正在使用CREATE或ALTER INDEX主动构建索引，则CTX_ADM.MARK_FAILED不会成功。但是，索引创建或更改可能包括CTX_ADM.MARK_FAILED可以成功的时间窗口，即使正在由另一个会话构建索引，也会将索引标记为失败。

CTX_ADM.MARK_FAILED适用于本地分区索引。但是，它会将所有分区的状态更改为FAILED 。因此，在使用CTX_ADM.MARK_FAILED后，应使用ALTER INDEX REBUILD PARTITION PARAMETERS ('RESUME') CTX_ADM.MARK_FAILED ALTER INDEX REBUILD PARTITION PARAMETERS ('RESUME')重建所有索引分区。如果在此操作之后运行ALTER INDEX PARAMETER ('RESUME') ，则Oracle会将索引分区状态重置为有效。Oracle不会重建在MARK_FAILED操作之前成功构建的索引分区。

句法

CTX_ADM.MARK_FAILED（VARCHAR2中的owner_name，VARCHAR2中的index_name）;
owner_name

要更改其状态的索引所有者的名称。

index_name

要更改其状态的索引的名称。

注意：

index_name不得以架构或所有者名称为前缀。

例

开始CTX_ADM.MARK_FAILED（'owner_1'，'index_1'）;结束;

5.3恢复

RECOVER过程清除文本数据字典，删除诸如剩余首选项之类的对象。

句法

CTX_ADM.RECOVER;

例

开始ctx_adm.recover;结束;

5.4 RESET_AUTO_OPTIMIZE_STATUS

使用RESET_AUTO_OPTIMIZE_STATUS过程重置（或删除） CTX_AUTO_OPTIMIZE_STATUS视图的内容。

您必须以CTXSYS身份登录才能运行CTX_ADM.RESET_AUTO_OPTIMIZE_STATUS 。

句法

CTX_ADM.RESET_AUTO_OPTIMIZE_STATUS;

例

开始ctx_adm.reset_auto_optimize_status;结束;

5.5 SET_PARAMETER

SET_PARAMETER过程为索引创建和近实时索引设置系统级参数。

句法

CTX_ADM.SET_PARAMETER（param_name IN VARCHAR2，param_value IN VARCHAR2）;
param_name

指定要设置的参数的名称，该参数可以是以下参数之一：

  • max_index_memory （允许索引的最大内存）

  • default_index_memory （为索引分配的默认内存）

  • ctx_doc_key_type （ CTX_DOC过程的默认输入密钥类型）

  • auto_optimize （自动优化的ENABLE或DISABLE ）

  • default_datastore （默认数据存储区首选项）

  • default_filter_file （存储在文件中的数据的默认过滤器首选项）

  • default_filter_text （默认文本过滤器首选项）

  • default_filter_binary （默认二进制过滤器首选项）

  • default_section_html （默认的html部分组首选项）

  • default_section_xml （默认的xml节组首选项）

  • default_section_text （默认文本部分组首选项）

  • default_lexer （默认词法分析器首选项）

  • default_wordlist （默认wordlist首选项）

  • default_stoplist （默认停止列表首选项）

  • default_storage （默认存储首选项）

  • default_ctxcat_lexer （CTXCAT索引的默认词法分析器首选项）

  • default_ctxcat_stoplist （CTXCAT索引的默认停止列表首选项）

  • default_ctxcat_storage （默认CTXCAT索引存储

  • default_ctxcat_wordlist （CTXCAT索引的默认wordlist首选项）

  • default_ctxrule_lexer （CTXRULE索引的默认词法分析器）

  • default_ctxrule_stoplist （CTXRULE索引的默认停止列表）

  • default_ctxrule_storage （CTXRULE索引的默认存储）

  • default_ctxrule_wordlist （CTXRULE索引的默认词表）

    也可以看看：

    要了解有关这些参数的默认值的更多信息，请参阅Oracle Text Indexing Elements中的 “ 系统参数 ”

注意：

无法再修改log_directory （ CTX_OUTPUT文件的目录）和auto_optimize_logfile （自动优化日志文件的基本文件名）。对这些参数忽略对API的任何调用。
param_value

指定要分配给参数的值。对于max_index_memory和default_index_memory ，您指定的值必须具有以下语法：

数[K | M | G]

其中K代表千字节，M代表兆字节，G代表千兆字节。

对于每个其他参数，请指定要用作索引的默认值的首选项的名称 。

对于auto_optimize ，您指定的值必须是ENABLE或DISABLE 。将此参数设置为ENABLE ，可以启动自动优化作业。将此参数设置为DISABLE ，不能启动自动优化作业，并且终止所有当前运行的优化作业。

例

要修改MAX_INDEX_MEMORY值：

exec ctx_adm.set_parameter（'MAX_INDEX_MEMORY'，100G）;

索引语句中的内存参数可以高达256 GB（如果MAX_INDEX_MEMORY参数未明确指定为较低的值）。 

在mytab（textcol）上创建索引myindex1 indextype是ctxsys.context参数（'memory 256G'）; exec ctx_ddl.sync_index（'myindex2'，memory =>'256G'）;

例

开始ctx_adm.set_parameter（'default_lexer'，'my_lexer'）;结束;
