80 ORA-12500至ORA-12699

ORA-12500：TNS：侦听器无法启动专用服务器进程

原因：启动专用服务器进程的过程失败。找不到可执行文件，或者可能未正确设置环境。

操作：在ADMIN级别启用跟踪并重新执行该操作。验证是否存在ORACLE Server可执行文件并启用了执行权限。确保在LISTENER.ORA中正确指定了ORACLE环境。正在调用的Oracle协议适配器可能未安装在本地硬盘驱动器上。请检查是否已成功链接正确的协议适配器。如果错误仍然存在，请联系Oracle客户支持。

ORA-12502：TNS：侦听器未收到客户端的CONNECT_DATA

原因：没有将CONNECT_DATA传递给侦听器。

操作：检查从TNSNAMES.ORA解析的服务名称是否具有连接描述符的CONNECT_DATA组件。

ORA-12503：TNS：侦听器从客户端收到无效的REGION

原因：无效的REGION传递给侦听器。

操作：检查连接描述符的CONNECT_DATA组件中指定的REGION名称。

ORA-12504：TNS：侦听器未在CONNECT_DATA中获得SERVICE_NAME

原因：侦听器未配置默认服务，并且侦听器收到的CONNECT_DATA中缺少SERVICE_NAME。

操作：可能的解决方案是： - 使用有效的服务名称在LISTENER.ORA中配置DEFAULT_SERVICE参数。使用reload [<listener_name>]重新加载侦听器参数文件。这应该由LISTENER管理员完成。- 如果使用服务名称，请检查与TNSNAMES.ORA中的服务名称对应的连接描述符是否在CONNECT_DATA中具有SERVICE_NAME或SID组件。

ORA-12505：TNS：侦听器当前不知道连接描述符中给出的SID

原因：侦听器收到建立与数据库或其他服务的连接的请求。侦听器接收的连接描述符为实例（通常是数据库实例）指定了一个SID，该实例尚未向侦听器动态注册或尚未为侦听器静态配置。这可能是一个临时条件，例如在侦听器启动之后，但在数据库实例向侦听器注册之前。

行动： - 稍等片刻，尝试连接第二次。- 通过执行以下命令检查侦听器当前已知的实例：lsnrctl services listener name - 检查连接描述符中的SID参数是否指定侦听器已知的实例。- 检查listener.log文件中的事件。

ORA-12506：TNS：侦听器基于服务ACL过滤拒绝连接

原因：客户端不在服务访问控制列表中。

操作：在服务ACL中添加客户端的地址。

ORA-12508：TNS：侦听器无法解析给定的COMMAND

原因：不支持给予侦听器的COMMAND。

行动：这在Oracle Net的正常使用中是看不到的。

ORA-12509：TNS：侦听器无法将客户端重定向到服务处理程序

原因：调度员意外终止

操作：尝试再次连接，如果发生相同的错误，请联系DBA以检查此SID的调度程序的状态。如果问题仍然存在，请在侦听器中启用跟踪以确定重定向导致的TNS错误。

ORA-12510：TNS：数据库暂时缺少处理请求的资源

原因：调度程序似乎忙于处理其他请求。

操作：再次尝试连接。如果错误仍然存在，请要求DBA增加调度员数量和/或调度员对他们可以接受的连接数/会话数的限制。

ORA-12511：TNS：已找到服务处理程序，但它不接受连接

原因：调度员通知听众他们暂时不接受新连接。

操作：再次尝试连接。如果错误仍然存在，请联系DBA以检查调度员的状态和/或要求他增加调度员的数量。

ORA-12513：TNS：已找到服务处理程序但已注册其他协议

原因：为此服务注册的调度程序通过与客户端不同的网络协议连接到侦听器。

操作：联系DBA以在协议上注册调度程序。

ORA-12514：TNS：侦听器当前不知道连接描述符中请求的服务

原因：侦听器收到建立与数据库或其他服务的连接的请求。侦听器接收的连接描述符指定了服务（通常是数据库服务）的服务名称，该服务尚未向侦听器动态注册或尚未为侦听器静态配置。这可能是一个临时条件，例如在侦听器启动之后，但在数据库实例向侦听器注册之前。

行动： - 稍等片刻，尝试连接第二次。- 通过执行以下命令检查侦听器当前已知的服务：lsnrctl services listener name - 检查所使用的网络服务名称的连接描述符中的SERVICE_NAME参数是否指定侦听器已知的服务。- 如果使用了简单连接命名连接标识符，请检查指定的服务名称是否为侦听器已知的服务。- 检查listener.log文件中的事件。

ORA-12515：TNS：侦听器找不到此演示文稿的处理程序

原因：没有任何侦听器的已知服务处理程序注册为支持连接客户端所需的表示协议。

操作：检查目标服务是否配置为接受表示协议。

ORA-12516：TNS：侦听器找不到具有匹配协议栈的可用处理程序

原因：给定SERVICE_NAME的已知和可用服务处理程序都不支持客户端的协议堆栈：传输，会话和表示协议。

操作：检查以确保给定SERVICE_NAME的服务处理程序（例如调度程序）已向侦听程序注册，正在接受连接，并且已正确配置它们以支持所需的协议。

ORA-12518：TNS：侦听器无法切换客户端连接

原因：将客户端连接切换到另一个进程的过程失败。

操作：打开侦听器跟踪并重新执行操作。验证是否已正确配置侦听器和数据库实例以进行直接切换。如果问题仍然存在，请致电Oracle支持。

ORA-12519：TNS：找不到合适的服务处理程序

原因：侦听器找不到适合客户端连接的任何可用服务处理程序。

操作：运行“lsnrctl services”以确保实例已向侦听器注册，并且正在接受连接。

ORA-12520：TNS：侦听器无法找到所请求的服务器类型的可用处理程序

原因：请求的服务器类型（专用或共享）的已知和可用服务处理程序均不适用于客户端连接。

操作：运行“lsnrctl services”以确保实例已向侦听器注册，并且相应的处理程序正在接受连接。

ORA-12521：TNS：侦听器当前不知道连接描述符中请求的实例

原因：侦听器收到建立与数据库或其他服务的连接的请求。除了服务名称之外，由侦听器接收的连接描述符是实例（通常是数据库实例）的实例名称，该实例尚未动态地向侦听器注册或者尚未为侦听器静态配置。这可能是一个临时条件，例如在侦听器启动之后，但在数据库实例向侦听器注册之前。

行动： - 稍等片刻，尝试连接第二次。- 通过执行以下命令检查侦听器当前已知的实例：lsnrctl services listener name - 检查连接描述符中的INSTANCE_NAME参数是否指定侦听器已知的实例名称。- 检查listener.log文件中的事件。

ORA-12522：TNS：侦听器找不到给定INSTANCE_ROLE的可用实例

原因：没有向侦听器注册任何可用且适当的数据库实例，这些实例是由连接描述符中给出的SERVICE_NAME标识的服务的一部分，并且具有指定的INSTANCE_ROLE（如果指定，则为INSTANCE_NAME）。

操作：检查以确保指定的INSTANCE_ROLE正确。运行“lsnrctl services”以确保实例已向侦听器注册并且它们已准备好接受连接。

ORA-12523：TNS：侦听器找不到适合客户端连接的实例

原因：侦听器找不到适合客户端连接的任何可用（数据库）实例。

操作：运行“lsnrctl services”以确保实例已向侦听器注册，并且状态为READY。

ORA-12524：TNS：侦听器无法解析连接描述符中给出的HANDLER_NAME

原因：在指定的SERVICE_NAME和INSTANCE_NAME的侦听器表中找不到CONNECT_DATA中的HANDLER_NAME。

操作：检查以确保指定的HANDLER_NAME是正确的。

ORA-12525：TNS：听众未及时收到客户的请求

原因：侦听器断开了客户端，因为客户端未能在允许的时间间隔内提供必要的连接信息。这可能是网络或系统延迟的结果;或者这可能表示恶意客户端试图对侦听器造成拒绝服务攻击。

操作：如果由于网络或系统速度较慢而发生错误，请将INBOUND_CONNECT_TIMEOUT重新配置为更大的值。如果怀疑是恶意客户端，请使用listener.log中的地址来标识源并限制访问。打开跟踪以获取更多信息。

ORA-12526：TNS：listener：所有适当的实例都处于受限模式

原因：支持客户端请求的服务的数据库实例处于受限模式。监听器不允许以受限模式连接到实例。这种情况可能是暂时的，例如在执行数据库管理期间。

操作：再次尝试连接。如果错误仍然存在，请联系数据库管理员以更改实例的模式（如果适用）。

ORA-12527：TNS：侦听器：所有实例都处于受限模式或阻止新连接

原因：支持客户端请求的服务的所有相应数据库实例都报告它们处于受限模式或阻止了新连接。监听器不允许连接到此类实例。这种情况可能是暂时的，例如在启动时。

操作：再次尝试连接。如果错误仍然存在，请与数据库管理员联系以检查实例的状态。

ORA-12528：TNS：listener：所有适当的实例阻止新连接

原因：支持客户端请求的服务的所有实例都报告他们阻止了新连接。这种情况可能是暂时的，例如在启动时。

操作：再次尝试连接。如果错误仍然存在，请联系管理员以检查实例的状态。

ORA-12529：TNS：根据当前过滤规则拒绝连接请求

原因： Connection Manager及其侦听器配置了过滤规则，指定拒绝连接请求。

操作：如果应允许此连接请求，请与管理员联系以修改过滤规则。

ORA-12530：TNS：听众：达到了费率限制

原因：监听器已达到连接或服务速率限制，因此拒绝连接。

操作：再次尝试连接。如果错误频繁发生，请与数据库管理员联系以提高连接或服务速率限制。

ORA-12531：TNS：无法分配内存

原因：无法分配足够的内存来执行所需的活动。

操作：为TNS释放一些资源，或者为机器添加更多内存。有关更多详细信息，请启用跟踪并重新执行该操作。

ORA-12533：TNS：非法ADDRESS参数

原因：指定了一组非法的协议适配器参数。在某些情况下，如果无法与协议传输建立连接，则会返回此错误。

操作：验证是否可以使用指定的协议访问目标。检查TNSNAMES.ORA的ADDRESS部分中的参数。可以在适用于您的平台的Oracle操作系统特定文档中找到Legal ADDRESS参数格式。解析传输层名称的协议（如DECnet对象名称）如果配置不正确或名称拼写错误，则容易受到此错误的影响。

ORA-12538：TNS：没有这样的协议适配器

操作：使用服务器所需的服务配置客户端（最佳解决方案）或从服务器的配置文件中删除要求（最不安全）。

ORA-12665：NLS字符串打开失败

原因：本机服务无法使字符串可供国家语言支持组件使用。

行动：确保国家语言支持组件已正确使用。如果有，请启用跟踪并将问题报告给客户支持。

ORA-12666：专用服务器：与入站不同的出站传输协议

原因：从专用服务器（数据库链接）为外部标识的出站连接指定的协议与用于入站连接的协议不同。Oracle Net无法对使用与用于连接到专用服务器的协议不同的协议的代理连接进行身份验证。

操作：在出站连接的Oracle Net连接描述符中指定与用于入站连接的协议相同的协议。

ORA-12667：共享服务器：与入站不同的出站传输协议

原因：为来自共享服务器（数据库链接）的外部标识的出站连接指定的协议与用于入站连接的协议不同。Oracle Net无法对使用与用于连接到共享服务器的协议不同的协议的代理连接进行身份验证。

操作：在出站连接的Oracle Net连接描述符中指定与用于入站连接的协议相同的协议

ORA-12668：专用服务器：出站协议不支持代理

原因：指定用于从专用服务器执行外部标识的代理连接（数据库链接）的协议不支持代理连接。

操作：在用于支持外部认证代理连接的连接的Oracle Net连接描述符中指定协议。注意：由于Oracle Net中的限制，用于代理连接的协议必须与用于从客户端到服务器的连接的协议相同。

ORA-12669：共享服务器：出站协议不支持代理

原因：指定用于从共享服务器执行外部标识的代理连接（数据库链接）的协议不支持代理连接。

操作：在用于支持外部认证代理连接的连接的Oracle Net连接描述符中指定协议。注意：由于Oracle Net中的限制，用于代理连接的协议必须与用于从客户端到服务器的连接的协议相同。

ORA-12670：角色密码不正确

原因：身份验证服务无法验证为角色提供的密码。

操作：提供正确的密码。

ORA-12671：共享服务器：适配器无法保存上下文

原因：当尝试通过共享服务器保存代理连接（数据库链接）所需的数据时，身份验证服务的适配器失败。

操作：启用跟踪以确定确切的错误。如果原因不明显，请联系Oracle客户支持。

ORA-12672：数据库登录失败

原因：正在使用的身份验证服务适配器遇到错误，它尝试验证用户的登录尝试。

操作：启用跟踪以确定适配器遇到的确切错误。

ORA-12673：专用服务器：未保存上下文

原因：连接被标记为来自专用服务器的代理连接（数据库链接），但不存在入站上下文。

操作：用户通常不会看到此错误。联系Oracle客户支持。

ORA-12674：共享服务器：未保存代理上下文

原因：连接被标记为来自共享服务器的代理连接（数据库链接），但不存在入站上下文。

操作：用户通常不会看到此错误。联系Oracle客户支持。

ORA-12675：外部用户名尚不可用

原因：正在使用的身份验证服务无法返回ORACLE服务器用户的外部名称，因为该服务尚不可用。

操作：这只是一条信息性消息，通常不应对用户可见。如果出现错误，请联系Oracle客户支持。

ORA-12676：服务器收到客户端的内部错误

原因：服务器进程收到来自客户端的错误，表明发生了内部Oracle Net本机服务错误。

操作：在两个进程上启用跟踪并尝试重新创建问题。如果问题再次出现，请联系Oracle客户支持。

ORA-12677：数据库链接不支持身份验证服务

原因：代理进程（数据库链接）使用的身份验证服务无法在其身份验证机制列表中找到客户端使用的适配器。

操作：指定客户端共享的身份验证适配器以及用于数据库链接的服务器。

ORA-12678：身份验证已禁用但必需

原因：控制是禁用身份验证还是需要身份验证的配置参数都设置为TRUE。

操作：将一个或两个参数设置为FALSE。

ORA-12679：本机服务被其他进程禁用但需要

原因：远程进程已禁用本机服务，但本地进程需要它们。

操作：在远程进程上启用本机服务或在本地禁用它们。

ORA-12680：已禁用本机服务但需要

原因：该进程已禁用本机服务，但至少需要一项服务。

操作：启用本机服务或更改配置文件，以便不需要任何可用服务。

ORA-12681：登录失败：SecurID卡还没有密码

原因：用于登录Oracle的SecurID卡没有分配密码。

操作：使用Security Dynamics提供的程序之一为芯片分配密码。

ORA-12682：登录失败：SecurID卡处于下一个PRN模式

原因： SecurID卡和SecurID服务器不同步，服务器需要下一个卡代码才能重新同步卡。

操作：使用Security Dynamics提供的程序之一重新同步SecurID卡。

ORA-12683：加密/加密校验和：没有Diffie-Hellman种子

原因： SQLNET.ORA参数文件中缺少“sqlnet.crypto_seed”参数。

操作：将此行添加到SQLNET.ORA：sqlnet.crypto_seed =“随机选择的文本”

ORA-12684：加密/加密校验和：Diffie-Hellman种子太小

原因： SQLNET.ORA参数文件中的“sqlnet.crypto_seed”参数太小。

操作：可能使用网络管理器向其中添加更多随机选择的文本。

ORA-12685：本地服务需要远程，但在本地禁用

原因：远程进程需要本机服务，但本地服务已在本地禁用。

操作：在本地启用本机服务或更改远程主机上的配置参数，以便不需要本机服务。

ORA-12686：为服务指定的命令无效

原因：为本机服务指定了不存在的操作。

操作：这是编程错误，通常不应对用户可见。如果出现错误，请联系Oracle客户支持。

ORA-12687：凭证已过期。

原因：用于为请求的连接验证用户的凭据已过期。

操作：续订凭据。有关如何执行此操作，请参阅特定于网络身份验证适配器的文档。

ORA-12688：登录失败：SecurID服务器拒绝了新的密码

原因： SecurID服务器拒绝密码的原因有很多： - 用户可能无权编写自己的密码。- 密码太短或太长。有效的密码由最少的四个字符组成，但不超过八个字符。- 密码包含任何非字母数字字符。

操作：重新执行操作并确保使用满足上述要求的密码。如果问题仍然存在，请在连接的Oracle Server端打开跟踪，并检查跟踪文件以查找确切的错误。

ORA-12689：需要服务器身份验证，但不受支持

原因：此连接需要服务器身份验证，但连接的两端都不支持。

操作：确保连接的两端都具有正确版本的高级网络选项，并且身份验证适配器支持服务器身份验证。

ORA-12690：服务器身份验证失败，登录已取消

原因：需要服务器身份验证，但客户端发现服务器的凭据无效。

操作：确保服务器具有一组有效的凭据。有关如何执行此操作，请参阅身份验证适配器特定文档。

ORA-12691：流量控制器模式下Oracle Connection Manager不支持TTC RPC（ string ）

原因： Traffic Director模式下的Oracle Connection Manager不支持指定的双任务公共（TTC）远程过程调用（RPC）。

操作：不要使用发布指定TTC RPC的功能，这是在流量控制器模式下Oracle Connection Manager不允许的

ORA-12692：流量控制器模式下Oracle Connection Manager不支持的功能（ string ）

原因： Traffic Director模式下的Oracle Connection Manager不支持指定的功能。

操作：请勿在Traffic Director模式下使用Oracle Connection // Manager不支持的功能。

ORA-12693：在流量控制器模式下，Oracle Connection Manager中未配置PRCP

原因：客户端尝试使用代理驻留连接池（PRCP）进行连接，而未在流量控制器模式下的Oracle连接管理器中配置PRCP。

操作：仅当在流量控制器模式下在Oracle Connection Manager中配置PRCP时，客户端才应使用PRCP连接。

ORA-12694：在流量控制器模式下在Oracle Connection Manager中配置PRCP时请求的非PRCP连接

原因：客户端正在尝试非代理驻留连接池（PRCP）连接，而在Exchange连接管理器中以流量控制器模式配置PRCP。

操作：在流量控制器模式下在Oracle Connection Manager中配置PRCP时，客户端应仅使用PRCP连接。

ORA-12695：在启用PRCP的流量控制器模式下，无法使用Oracle Connection Manager执行此语句

原因：在启用了代理驻留连接池（PRCP）的流量控制器模式下，客户端未发送足够的信息来使用Oracle Connection Manager执行该语句。

操作：此客户端需要一个可以解决问题的补丁。或者，它需要升级到Oracle Database Release 18.1或更高版本。请联系Oracle客户支持以获取该补丁。

ORA-12696：双重加密已开启，不允许登录

原因：用户正在使用加密打开的安全协议适配器以及ANO加密。

操作：请执行以下操作之一： - 关闭ANO加密。- 将SQLNET.IGNORE_ANO_ENCRYPTION_FOR_TCPS参数设置为TRUE。 - 在客户端的TNS连接字符串的SECURITY子句中将IGNORE_ANO_ENCRYPTION_FOR_TCPS设置为TRUE。有关更多详细信息，请参阅数据库网络服务参考。

ORA-12697：PRCP：内部错误\ n

原因：收集统计信息时检测到无效的PRCP状态

操作：在Traffice Director模式管理中检查CMAN以从trace和sqlnet.log获取更多信息。

ORA-12699：本机服务内部错误

原因：本机服务组件中发生内部错误。

操作：启用跟踪以确定确切的错误。联系Oracle客户支持。
