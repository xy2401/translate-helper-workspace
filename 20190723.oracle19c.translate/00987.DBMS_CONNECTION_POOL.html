36.1.2 CONFIGURE_POOL过程

此过程使用高级选项配置池。

句法

DBMS_CONNECTION_POOL.CONFIGURE_POOL（pool_name IN VARCHAR2 DEFAULT'SYS_DEFAULT_CONNECTION_POOL'，msize IN NUMBER DEFAULT 4，maxsize IN NUMBER DEFAULT 40，incsize IN NUMBER DEFAULT 2，session_cached_cursors in NUMBER DEFAULT 20，inactivity_timeout IN NUMBER DEFAULT 300，max_think_time IN NUMBER DEFAULT 120，max_use_session IN NUMBER DEFAULT 500000，max_lifetime_session IN NUMBER DEFAULT 86400，max_txn_think_time IN NUMBER）;

参数

表36-4 CONFIGURE_POOL过程参数

参数 描述

POOL_NAME

要配置的池。目前仅支持默认池名称。

MINSIZE

池中池化服务器的最小数量

MAXSIZE

池中允许的最大池化服务器

incrsize

当应用程序请求时池服务器不可用时，池将按此数量的池服务器递增

SESSION_CACHED_CURSORS

为池中的所有连接启用SESSION_CACHED_CURSORS 。这是一个现有的init 。 ora参数

inactivity_timeout

池中空闲会话的TTL （生存时间）。此参数有助于在未将池用于其最大容量时缩小池。如果此时池中的连接保持空闲状态，则会将其终止。

max_think_time

客户端从池中获取池化服务器且其中没有打开事务的最长不活动时间（以秒为单位）。从池中获取池化服务器后，如果客户端应用程序未在MAX_THINK_TIME指定的时间内发出数据库调用，则释放池化服务器并终止客户端连接。

max_use_session

可以将连接释放到池中的最大次数

max_lifetime_session

汇集会话的TTL （生存时间）

max_txn_think_time

客户端从打开事务的池中获取池化服务器后，不活动的最长时间（以秒为单位）。从池中获取池化服务器后，如果客户端应用程序未在MAX_TXN_THINK_TIME指定的时间内发出数据库调用，则释放池化服务器，并终止客户端连接。此参数的默认值是MAX_THINK_TIME参数的值。应用程序可以将MAX_TXN_THINK_TIME参数的值设置为高于MAX_THINK_TIME值的值，以便为打开事务的连接留出更多时间。

例外

表36-5 CONFIGURE_POOL过程异常

例外 描述

ORA-56500

找不到连接池

ORA-56507

连接池更改配置失败

使用说明

  • 所有时间表达都以秒为单位

  • 应根据统计请求模式设置所有参数。

  • 应该设置minsize ，记住它会降低服务器资源消耗的下限。这是为了防止超时拖动池太低，因为短暂的不活动时间。

  • 应该设置maxsize ，记住它在并发和响应时间以及服务器资源消耗上设置了上限。

  • session_cached_cursors通常设置为最常用语句的数量。它占用服务器上的游标资源

  • 有疑问，不要设置increment和inactivity_timeout 。该池将具有合理的默认值。

  • max_use_session和max_lifetime_session max_use_session获得全新的会话，允许软件更新或防御潜在的错误，泄漏，累积等问题的方法。

  • 连接池保留5％的池服务器进行身份验证，并且始终保留至少一个池服务器进行身份验证。设置maxsize参数时，请确保有足够的池服务器用于身份验证和连接。

36.1.3 START_POOL过程

此过程启动池以进行操作。只有在此调用之后，连接类才能使用池来创建会话。

句法

DBMS_CONNECTION_POOL.START_POOL（pool_name IN VARCHAR2 DEFAULT'SYS_DEFAULT_CONNECTION_POOL'）;

参数

表36-6 START_POOL过程参数

参数 描述

POOL_NAME

游泳池即将开始。目前仅支持默认池名称。

例外

表36-7 START_POOL过程异常

例外 描述

ORA-56500

找不到连接池

ORA-56501

连接池启动失败

使用说明

如果重新启动实例（关闭然后启动），则会自动启动池。

36.1.4 STOP_POOL过程

此过程将停止池并使其不可用于已注册的连接类。

句法

DBMS_CONNECTION_POOL.STOP_POOL（pool_name IN VARCHAR2 DEFAULT'SYS_DEFAULT_CONNECTION_POOL'）;

参数

表36-8 STOP_POOL过程参数

参数 描述

POOL_NAME

游泳池被停止。目前仅支持默认池名称。

例外

表36-9 STOP_POOL过程异常

例外 描述

ORA-56500

找不到连接池

ORA-56506

连接池关闭失败

使用说明

这会停止池并使其脱机。这不会破坏与池关联的持久数据（例如，池名称和配置参数）。

句法

表36-10 RESTORE_DEFAULTS过程参数

POOL_NAME

游泳池将被恢复。目前仅支持默认池名称。
