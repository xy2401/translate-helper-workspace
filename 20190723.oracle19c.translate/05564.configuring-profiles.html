12.2在安装过程中配置配置文件

在客户端和服务器上安装软件后，Oracle Universal Installer会启动Oracle Net Configuration Assistant 。Oracle Net Configuration Assistant配置计算机用于将连接标识符解析为连接描述符的命名方法的顺序

在安装期间使用Oracle Net Configuration Assistant进行配置会导致sqlnet.ora文件中的条目类似于以下内容：

NAMES.DIRECTORY_PATH =（EZCONNECT，TNSNAMES）

NAMES.DIRECTORY_PATH参数指定用于解析连接标识符的命名方法的优先级顺序。如果安装的配置不足，则使用Oracle Net Manager更改sqlnet.ora配置。

12.3.1关于客户端的默认域

在客户端经常从特定域请求名称的环境中，使用NAMES.DEFAULT_DOMAIN参数在客户端sqlnet.ora文件中设置默认域是合适的。此参数可用于本地和外部命名方法。

设置默认域后，它会自动附加到连接字符串中给出的任何非限定网络服务名称，然后与存储在tnsnames.ora文件中的网络服务名称进行比较。

例如，如果客户端tnsnames.ora文件包含sales.us.example.com的网络服务名称，并且默认域为us.example.com ，则用户可以输入以下连接字符串：

CONNECT scott @ sales输入密码： password

在前面的示例中， sales被搜索为sales.us.example.com 。

如果连接字符串包含域扩展名，例如在CONNECT scott@sales.us.example.com ，则不会追加域。

如果tnsnames.ora文件中的网络服务名称不是域限定的并且设置了NAMES.DEFAULT_DOMAIN参数，则必须输入网络服务名称，并在名称末尾添加句点（ . ）。例如，如果域设置为us.example.com并且客户端tnsnames.ora文件包含sales2的网络服务名称，则用户将输入以下连接字符串：

CONNECT scott @ sales2。输入密码： password

在前面的示例中，客户端将连接到sales2 ，而不是sales2.us.example.com 。

12.5关于高级配置文件信息

表12-3说明了可以设置的高级sqlnet.ora文件设置。

表12-3 sqlnet.ora中的高级设置

Oracle Net Manager字段/选项 sqlnet.ora文件参数 描述

发送操作超时

SQLNET.SEND_TIMEOUT

指定数据库客户端或服务器在建立连接后等待来自对等方的数据的时间（毫秒，秒或分钟）。

为客户端设置此参数可确保接收操作不会无限期地长时间处于等待状态，或者由于服务器进程或服务器繁忙状态的异常终止而长时间处于等待状态。如果客户端未在指定的时间内收到响应数据，则会记录 ORA-12535: TNS:operation timed out和 ORA-12609: TNS: Receive timeout occurred sqlnet.log文件中ORA-12609: TNS: Receive timeout occurred错误。

您还可以在服务器端设置此参数，以指定服务器在建立连接后等待客户端数据的时间（毫秒，秒或分钟）。如果客户端未在指定时间内发送任何数据，则数据库服务器会记录ORA-12535: TNS:operation timed out和ORA-12609: TNS: Receive timeout occurred消息到sqlnet.log文件。如果没有此参数，数据库服务器可能会继续等待来自可能已关闭或遇到困难的客户端的数据。服务器通常会阻止来自客户端的输入，并且如果设置为较低值，则会频繁获取这些超时。

接收操作超时

SQLNET.RECV_TIMEOUT

指定数据库服务器在建立连接后等待客户端数据的时间（以秒为单位）。客户端必须在指定的时间间隔内发送一些数据。

建议在客户端偶尔或异常关闭的环境中设置此参数。如果客户端未在指定的时间内发送任何数据，则数据库服务器会记录ORA-12535: TNS:operation timed out并且 ORA-12609: TNS: Receive timeout occurred sqlnet.log文件ORA-12609: TNS: Receive timeout occurred错误。

如果没有此参数，数据库服务器将继续等待来自可能已关闭或遇到困难的客户端的数据。

您还可以在客户端使用此设置来指定客户端在建立连接后等待来自数据库服务器的响应数据的时间（以秒为单位）。如果没有此参数，客户端可能会等待很长一段时间，以便来自数据库服务器的请求不堪重负。

连接超时

SQLNET.INBOUND_CONNECT_TIMEOUT

指定客户端与数据库服务器连接的时间（以秒为单位），并提供必要的身份验证信息。

总发送缓冲区大小

SEND_BUF_SIZE

为会话的发送操作指定缓冲区空间限制。

总接收缓冲区大小

RECV_BUF_SIZE

为会话的接收操作指定缓冲区空间限制。

TNS超时值

SQLNET.EXPIRE_TIME

指定发送探针以验证客户端/服务器连接是否处于活动状态的时间间隔（以分钟为单位）。设置大于0的值可确保由于客户端终止异常，连接无法无限期保持打开状态。如果探测发现终止连接或不再使用的连接，则它会返回错误，导致服务器进程退出。此设置适用于数据库服务器，该服务器通常一次处理多个连接。

使用此终止连接检测功能的限制是：

  • 遗留连接不允许这样做。

  • 即使它非常小，探测数据包也会产生额外的流量，可能会降低网络性能。

  • 根据正在使用的操作系统，服务器可能需要执行额外的处理以将连接探测事件与发生的其他事件区分开。这也可能导致网络性能下降。

客户注册ID

SQLNET.CLIENT_REGISTRATION

为客户端指定唯一标识符。使用任何连接请求将此标识符传递给侦听器。标识符可以是长度不超过128个字符的任何字符串。

登录验证协议版本

SQLNET.ALLOWED_LOGON_VERSION_CLIENT

SQLNET.ALLOWED_LOGON_VERSION_SERVER

定义连接到Oracle数据库实例时允许的最低身份验证协议。参数名称中的术语VERSION指的是身份验证协议的版本，而不是Oracle数据库版本。如果客户端或服务器未满足其伙伴指定的最低版本，则身份验证将失败，并显示ORA-28040错误。

支持的值包括：

  • 12a for Oracle Database 12 c认证协议（最强保护）

  • 12关键补丁更新CPUOct2012及更高版本Oracle Database 11g身份验证协议（推荐）

  • 11适用于Oracle Database 11g身份验证协议

    10用于Oracle数据库10g身份验证协议

  • 9用于Oracle9 i数据库身份验证协议

  • 8用于Oracle8 i数据库身份验证协议

默认值为12或12a 。请注意将值设置为11或12的以下含义：

  • 不得使用SEC_CASE_SENSITIVE_LOGON=FALSE设置。如果将其设置为FALSE ，则用户帐户和安全角色将变为不可用。SEC_CASE_SENSITIVE_LOGON初始化参数启用或禁用密码的区分大小写。

  • 要充分利用Oracle数据库11克推出的密码保护，用户必须更改其密码。

  • 在Oracle Database 10g和所有版本的JDBC瘦客户机无法使用基于密码的身份验证对Oracle数据库进行身份验证之前，OCI客户端的发布。

当Oracle数据库12c的服务器有一个数据库链接到运行早期Oracle数据库版本的服务器，将Oracle数据库12c的服务器上SQLNET.ALLOWED_LOGON_VERSION_CLIENT参数的值以匹配服务器的运行越早SQLNET.ALLOWED_LOGON_VERSION设置发布。

关闭UNIX信号处理

BEQUEATH_DETACH

打开或关闭UNIX信号处理。

因为客户端应用程序通过遗留协议在内部生成服务器进程作为子进程，所以客户端应用程序负责在子进程完成时清理它。当服务器进程完成其连接职责时，它将成为已终止的进程。信号处理程序负责清理这些终止的进程。设置此参数将配置客户端配置文件，以通过禁用信号处理程序将此过程传递到UNIX初始化过程。

禁用带外中断

DISABLE_OOB

打开或关闭带外休息。

如果取消选择或设置为off ，则Oracle Net可以使用底层协议提供的紧急数据请求发送和接收中断消息。启用后，此功能将应用于此客户端使用的所有协议。

如果选择或设置为on ，则它将禁用使用基础协议的紧急数据请求发送和接收中断消息的功能。

禁用带外自动中断选项

DISABLE_OOB_AUTO

在连接时检查客户端到服务器路径的TCP OOB（带外）支持。仅当服务器路径支持OOB时，客户端才启用OOB。
