Oracle Database Release 19c，版本19.1中的更改

以下是Oracle数据库性能调优指南（适用于Oracle数据库版本19c，版本19.1）中的更改。

新功能

以下功能是此版本中的新增功能：

  • Memoptimized Rowstore - 快速摄取

    Memoptimized Rowstore的快速摄取功能优化了来自应用程序（如物联网（IoT）应用程序）的高频单行数据插入的处理。

    请参阅“ 使用快速摄取 ” 。

  • 可插拔数据库（PDB）的自动数据库诊断监视器（ADDM）支持

    您现在可以使用ADDM分析PDB中的AWR数据，以识别和解决与性能相关的问题。

    请参阅“ 在多租户环境中使用ADDM ” 。

不受支持的功能

此版本不支持以下功能。

  • Oracle Streams

    从Oracle Database 19c开始，不支持Oracle Streams功能。使用Oracle GoldenGate替换Oracle Streams的所有复制功能。

Oracle Database Release 18c，版本18.1中的更改

以下是Oracle数据库性能调优指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

新功能

以下功能是此版本中的新增功能：

  • Memoptimized Rowstore

    Memoptimized Rowstore为使用MEMOPTIMIZE FOR READ子句指定的表启用高性能读取。此功能对于主要基于非常高频率的主键值（例如物联网（IoT）应用程序）查询表的应用程序特别有用。

    请参阅“ 使用Memoptimized Rowstore启用高性能数据流 ” 。

Oracle Database 12c第2版（12.2）中的更改

以下是Oracle Database 12 c第2版（12.2）的Oracle数据库性能调整指南中的更改。

新功能

以下功能是此版本中的新增功能：

  • 每进程PGA限制

    消耗过多PGA内存的失控查询可能会在Oracle数据库中造成严重的性能问题。在多租户容器数据库（CDB）中，此类查询可能会影响所有可插拔数据库（PDB）的性能。要防止出现此问题，您现在可以指定特定使用者组中每个会话可以使用的PGA内存量的绝对限制。

    请参阅“ 使用资源管理器调整程序全局区域大小 ” 。

  • Prespawned服务器进程

    当启用专用代理连接模式或启用线程执行模式时，Oracle数据库现在会预先确定服务器进程池。此功能改进了这些数据库操作模式下的客户端连接性能。

    请参阅“ 由于预先确定的进程而改善的客户端连接性能 ” 。

  • 自动工作负载存储库（AWR）支持多租户环境

    您现在可以在多租户环境中捕获PDB的自动工作负载存储库（AWR）数据。此功能可在多租户环境中对PDB进行性能调整。

    请参阅“ 在多租户环境中管理自动工作负载存储库 ” 。

  • 自动工作负载存储库（AWR）支持Oracle Active Data Guard备用数据库

    您现在可以捕获Oracle Active Data Guard备用数据库的自动工作负载存储库（AWR）数据。此功能可以对Oracle Active Data Guard备用数据库进行性能调整。

    请参阅“ 在Active Data Guard备用数据库中管理自动工作负载存储库 ” 。

  • 直接NFS客户端性能改进功能 - 并行NFS和直接NFS调度程序

    并行NFS是NFS版本4.1中引入的Oracle Direct NFS Client的可选功能。此功能支持高度可扩展的分布式NAS存储，以实现更好的I / O性能。

    Direct NFS调度程序功能将从数据库实例创建的TCP连接数合并到NFS服务器。在大型数据库部署中，使用Direct NFS调度程序可提高可伸缩性和网络性能。

    请参阅“ 使用Direct NFS Client提高I / O性能 ” 。

  • 面向服务的缓冲区缓存访问优化

    集群管理的服务在各种Oracle Real Application Clusters（Oracle RAC）数据库实例之间分配工作负载。这些服务还从Oracle RAC数据库实例访问数据，并将其提供给请求数据的应用程序。面向服务的缓冲区高速缓存访问优化功能允许Oracle RAC缓存每个服务经常访问的特定于服务的数据，从而改善服务的数据访问时间。在跨Oracle RAC数据库实例访问数据时，这种依赖于数据的缓存还可以使响应时间更加一致。此功能在Oracle RAC中永久启用。

Oracle Database 12c第1版（12.1.0.2）中的更改

以下是Oracle Database 12 c第1版（12.1.0.2）的Oracle数据库性能调整指南中的更改。

新功能

以下功能是此版本中的新增功能：

  • 内存列存储

    内存中列存储（IM列存储）是SGA的可选区域，它以列式格式存储表，分区和其他数据库对象的副本，这些格式针对快速扫描进行了优化。IM列存储可加快分析，数据仓库和联机事务处理（OLTP）应用程序的数据库性能。

    请参阅“ 使用内存中的列存储来提高查询性能 ” 。

  • 内存列存储的可管理性支持

    SQL Monitor报告，ASH报告和AWR报告现在显示各种内存中操作的统计信息。

  • 强制完全数据库缓存模式

    强制完全数据库缓存模式使您可以将整个数据库缓存在内存中，这可以在执行全表扫描或访问LOB时提供显着的性能改进。

    请参阅“ 配置数据库缓存模式 ” 。

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle Database 12 c第1版（12.1.0.1）的Oracle数据库性能调整指南中的更改。

新功能

以下功能是此版本中的新增功能：

  • 实时ADDM

    实时ADDM可帮助您分析和解决无响应或挂起数据库中的问题，而无需重新启动数据库。

    请参阅“ 实时ADDM分析 ” 。

  • 限制计划全球区域（PGA）的规模

    PGA_AGGREGATE_LIMIT初始化参数使您可以指定PGA内存使用的硬限制。Oracle数据库通过终止消耗最多PGA内存的会话或进程来确保PGA大小不超过此限制。

    请参阅“ 限制程序全局区域的大小 ” 。

其他变化

以下是此版本中的其他更改：

  • 新书

    “Oracle数据库性能调优指南”正在对此版本进行重大改写。结果，书籍结构发生了重大变化。本书现在仅包含与数据库本身相关的调优主题。所有与SQL相关的调优主题都将移至Oracle Database SQL Tuning Guide 。
