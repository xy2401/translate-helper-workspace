修改Oracle所有者用户组

如果您已创建Oracle软件安装所有者帐户，但它不是要指定为OSDBA，OSOPER，OSDBA for ASM，ASMADMIN或其他系统特权组的组的成员，则修改该用户的组设置在安装之前。

警告：

每个Oracle软件所有者必须是同一中央库存组的成员。请勿修改现有Oracle软件所有者帐户的主要组，或将不同的组指定为OINSTALL组。如果Oracle软件所有者帐户具有不同的组作为其主要组，则可以破坏中央库存。

在安装过程中，安装软件的用户应将OINSTALL组作为其主要组，并且它必须是适合您的安装的操作系统组的成员。例如：
＃/ usr / sbin / usermod -g oinstall -G dba，asmdba，backupdba，dgdba，kmdba，racdba [，oper] oracle
