5.1 OPG_APIS.ANALYZE_PG

格式

OPG_APIS.ANALYZE_PG（graph_name IN VARCHAR2，estimate_percent IN NUMBER，method_opt IN VARCHAR2，degree IN NUMBER，cascade IN BOOLEAN，no_invalidate IN BOOLEAN，force IN BOOLEAN DEFAULT FALSE，options IN VARCHAR2 DEFAULT NULL）;

描述

Hathers，对于给定的属性图，VT $，GE $，IT $和GT $表的统计信息。

参数

graph_name

属性图的名称。

estimate_percent

在架构表中估计的行的百分比（NULL表示计算）。有效范围是[0.000001,100]。使用常量DBMS_STATS.AUTO_SAMPLE_SIZE让Oracle数据库确定适当的样本大小以获得良好的统计信息。这是通常的默认值。

mrthod_opt

对内部属性图模式表接受以下任一选项或两者的组合：

  • FOR ALL [INDEXED | HIDDEN] COLUMNS [size_clause]

  • FOR COLUMNS [size clause] column | attribute [size_clause] [，column | attribute [size_clause] ...]

size_clause定义为size_clause := SIZE {integer | REPEAT | AUTO | SKEWONLY}

  • integer ：直方图桶的数量。必须在[1,254]范围内。

  • REPEAT ：仅在已有直方图的列上收集直方图。

  • AUTO ：Oracle数据库根据数据分布和列的工作负载确定要收集直方图的列。

  • SKEWONLY ：Oracle数据库根据列的数据分布确定要收集直方图的列

column定义为column := column_name | (extension)

  • column_name ：列的名称

  • extension ：可以是(column_name, column_name [, ...])格式的列组或表达式。

通常的默认值是： FOR ALL COLUMNS SIZE AUTO

学位

属性图模式表的并行度。degree的通常默认值为NULL，这意味着使用CREATE TABLE或ALTER TABLE语句中DEGREE子句指定的表默认值。使用常量DBMS_STATS.DEFAULT_DEGREE根据初始化参数指定默认值。AUTO_DEGREE值自动确定并行度。根据对象的大小，这是1（串行执行）或DEFAULT_DEGREE （基于CPU数和初始化参数的系统默认值）。

级联

收集有关属性图模式表的索引的统计信息。使用常量DBMS_STATS.AUTO_CASCADE让Oracle数据库确定是否要收集索引统计信息。这是通常的默认值。

no_invalidate

如果为TRUE ，则不会使从属游标无效。如果为FALSE ，则立即使相关游标无效。如果DBMS_STATS.AUTO_INVALIDATE （通常的默认值）生效，Oracle数据库将决定何时使相关游标无效。

强制

如果为TRUE ，即使锁定了一个或多个基础表，也会执行操作。

选项

（保留供将来使用。）

使用说明

只有属性图的所有者才能调用此过程。

例子

以下示例收集属性图mypg统计信息。 

EXECUTE OPG_APIS.ANALYZE_PG（'mypg'，estimate_percent => 0.001，method_opt =>'FOR ALL COLUMNS SIZE AUTO'，degree => 4，cascade => true，no_invalidate => false，force => true，options => NULL） ;

5.2 OPG_APIS.CF

格式

OPG_APIS.CF（edge_tab_name在VARCHAR2中，edge_label在VARCHAR2中，rating_property在VARCHAR2中，迭代在NUMBER DEFAULT 10中，min_error在NUMBER DEFAULT 0.001中，k在NUMBER DEFAULT 5中，learning_rate在NUMERT DEFAULT 0.0002中，decrease_rate在NUMBER DEFAULT 0.95中，正则化在NUMBER DEFAULT中0.02，dop IN NUMBER DEFAULT 8，wt_l IN / OUT VARCHAR2，wt_r IN / OUT VARCHAR2，wt_l1 IN / OUT VARCHAR2，wt_r1 IN / OUT VARCHAR2，wt_i IN / OUT VARCHAR2，wt_ld IN / OUT VARCHAR2，wt_rd IN / OUT VARCHAR2， tablespace IN VARCHAR2 DEFAULT NULL，options IN VARCHAR2 DEFAULT NULL）;

描述

在给定图表上使用矩阵分解运行协同过滤。矩阵乘积的结果因子将存储在左表和右表中。

参数

edge_tab_name

属性图表边缘表（GE $）的名称。

edge_label

包含rating属性的边的标签。

rating_property

（保留供将来使用：评级属性的名称。）

次迭代

应执行的最大迭代次数。默认= 10。

min_error

达到的最小错误。如果在某个迭代中错误值低于此值，则过程结束。默认值= 0.001。

k

左侧和右侧产品的功能数量。默认= 5。

learning_rate

梯度下降的学习率。默认值= 0.0002。

decrease_rate

（保留供将来使用：如果学习率对于有效梯度下降而言太大，则降低率。默认= 0.95。）

正规化

一个额外的参数，以避免过度拟合。默认= 0.02

dop

并行度。默认= 8。

wt_l

保存矩阵分解左侧的工作表的名称。

wt_r

保存矩阵分解右侧的工作表的名称。

wt_l1

在梯度下降中保持左侧中间步骤的工作台的名称。

wt_r1

在梯度下降中保持右侧中间步骤的工作台的名称。

wt_I

保存中间矩阵产品的工作表的名称。

wt_ld

在梯度下降中保持中间左侧三角形的工作台的名称。

wt_rd

在梯度下降中保持中间右侧三角形的工作台的名称。

表空间

用于存储中间数据的表空间的名称。

选项

其他操作设置。带有一个或多个（逗号分隔）以下值的可选字符串：

  • 'INMEMORY = T'是使用'inmemory'子句创建模式表的选项。

  • 'IMC_MC_B = T'使用INMEMORY MEMCOMPRESS BASIC子句创建模式表。

使用说明

有关使用RDF数据进行协作过滤的信息，请参阅基于SQL的属性图分析 ，尤其是协作过滤概述和示例 。

如果工作表已存在，则可以为工作表相关参数指定其名称。在这种情况下，算法可以继续前一次迭代的进度而无需重新创建表。

如果工作表不存在，或者您不想使用现有工作表，则必须首先调用OPG_APIS.CF_PREP过程，该过程将创建必要的工作表。

协同过滤算法的最终结果是工作表wt_l和wt_r ，它们是矩阵乘积的两个因子。在对协同过滤进行预测时，应使用这些矩阵因子。

如果（且仅当）您没有兴趣保留输出矩阵因子和算法的当前进度以供将来使用，您可以调用OPG_APIS.CF_CLEANUP过程来删除包含中间表的所有工作表和输出矩阵因子。

例子

以下示例调用OPG_APIS.CF_PREP过程以创建工作表，然后调用OPG_APIS.CF过程以使用带有rating标签的边缘在phones图上运行协同过滤。 

DECLARE wt_l varchar2（32）; wt_r varchar2（32）; wt_l1 varchar2（32）; wt_r1 varchar2（32）; wt_i varchar2（32）; wt_ld varchar2（32）; wt_rd varchar2（32）; edge_tab_name varchar2（32）：='phonesge $'; edge_label varchar2（32）：='rating'; rating_property varchar2（32）：='';迭代整数：= 100; min_error数字：= 0.001; k整数：= 5; learning_rate数：= 0.001; decrease_rate数：= 0.95;正则数：= 0.02; dop数：= 2; tablespace varchar2（32）：= null; options varchar2（32）：= null; BEGIN opg_apis.cf_prep（edge_tab_name，wt_l，wt_r，wt_l1，wt_r1，wt_i，wt_ld，wt_rd）; opg_apis.cf（edge_tab_name，edge_label，rating_property，iterations，min_error，k，learning_rate，decrease_rate，regularization，dop，wt_l，wt_r，wt_l1，wt_r1，wt_i，wt_ld，wt_rd，tablespace，options）;结束; /

以下示例假定先前已运行OPG_APIS.CF_PREP，并且它指定在该运行期间创建的各种工作表。在这种情况下，前面的示例会自动为工作表的名称分配“$$ CFL57”等后缀。（输出名称可以在生成时打印，也可以在调用OPG_APIS.CF_PREP时由用户定义。）因此，以下示例可以使用OPG_APIS.CF运行算法的更多迭代，而无需首先调用OPG_APIS.CF_PREP，从而继续前一次运行的进度。

DECLARE wt_l varchar2（32）='phonesge $$ CFL57'; wt_r varchar2（32）='phonesge $$ CFR57'; wt_l1 varchar2（32）='phonesge $$ CFL157'; wt_r1 varchar2（32）='phonesge $$ CFR157'; wt_i varchar2（32）='phonesge $$ CFI57'; wt_ld varchar2（32）='phonesge $$ CFLD57'; wt_rd varchar2（32）='phonesge $$ CFRD57'; edge_tab_name varchar2（32）：='phonesge $'; edge_label varchar2（32）：='rating'; rating_property varchar2（32）：='';迭代整数：= 100; min_error数字：= 0.001; k整数：= 5; learning_rate数：= 0.001; decrease_rate数：= 0.95;正则数：= 0.02; dop数：= 2; tablespace varchar2（32）：= null; options varchar2（32）：= null; BEGIN opg_apis.cf（edge_tab_name，edge_label，rating_property，iterations，min_error，k，learning_rate，decrease_rate，regularization，dop，wt_l，wt_r，wt_l1，wt_r1，wt_i，wt_ld，wt_rd，tablespace，options）;结束; /

OPG_APIS.SPARSIFY_GRAPH（edge_tab_name IN VARCHAR2，阈值IN NUMBER DEFAULT 0.5，min_keep IN INTEGER DEFAULT 1，dop IN INTEGER DEFAULT 4，wt_out_tab IN OUT VARCHAR2，wt_und_tab IN OUT VARCHAR2，wt_hsh_tab IN OUT VARCHAR2，wt_mch_tab IN OUT VARCHAR2，tbs IN VARCHAR2 DEFAULT NULL，options IN VARCHAR2 DEFAULT NULL）;

描述

对属性图边缘表执行稀疏化（边缘修剪）。

参数

edge_tab_name

属性图表边缘表（GE $）的名称。

门槛

控制需要执行多少稀疏化的数值。值越低，将删除的边越多。一些典型值是：0.1,0.2，...，0.5

min_keep

一个正整数，表示每个顶点至少应保留多少个相邻边。建议值为1。

dop

操作的并行度。

wt_out_tab

用于保存输出的工作表，一个稀疏图。

wt_und_tab

用于保存原始图形的无向版本的工作表。

wt_hsh_tab

用于保存图的最小哈希值的工作表。

wt_mch_tab

用于保存最小哈希值的匹配计数的工作表。

tbs

用于保存工作表数据的工作表。

选项

其他操作设置。带有一个或多个（逗号分隔）以下值的可选字符串：

  • 'INMEMORY = T'是使用'inmemory'子句创建模式表的选项。

  • 'IMC_MC_B = T'使用INMEMORY MEMCOMPRESS BASIC子句创建模式表。

使用说明

调用此过程需要CREATE TABLE权限。

使用的稀疏算法是基于min hash的局部稀疏化。请参阅“可扩展聚类的本地图解析”，2011 ACM SIGMOD国际数据管理会议论文集： https ：//cs.uwaterloo.ca/~tozsu/courses/CS848/W15/presentations/ElbagouryPresentation-2.pdf

稀疏化仅涉及图的拓扑。所有属性（K / V）都不相关。

例子

以下示例执行mypg的edge表的准备工作，打印出工作表名称，并运行sparsification。输出是一个稀疏图，存储在名为LEAN_PG的表中，该表有两列SVID和DVID。

SQL>在DECLARE上设置serveroutput my_lean_pg varchar2（100）：='lean_pg'; - 输出表wt2 varchar2（100）; wt3 varchar2（100）; wt4 varchar2（100）; BEGIN opg_apis.sparsify_graph_prep（'mypgGE $'，my_lean_pg，wt2，wt3，wt4，null）; dbms_output.put_line（'wt2'|| wt2 ||'，wt3'|| wt3 ||'，wt4'|| wt4）; opg_apis.sparsify_graph（'mypgGE $'，0.5,1,4，my_lean_pg，wt2，wt3，wt4，'SEMTS'，null）;结束; / wt2“MYPGGE $$ TWSPAU275”，wt3“MYPGGE $$ TWSPAH275”，wt4“MYPGGE $$ TWSPAM275”SQL>描述lean_pg;名字空？输入----------------------------------------- -------- ---------------------------- SVID NUMBER DVID NUMBER
