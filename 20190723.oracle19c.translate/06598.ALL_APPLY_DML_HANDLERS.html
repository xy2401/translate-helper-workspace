2.32 ALL_APPLY_DML_HANDLERS

ALL_APPLY_DML_HANDLERS在当前用户可访问的表上显示有关DML处理程序的信息。

相关视图

DBA_APPLY_DML_HANDLERS显示有关数据库中所有表的DML处理程序的信息。

柱 数据类型 空值 描述

OBJECT_OWNER

VARCHAR2（128）

不是NULL

指定DML处理程序的对象的所有者

OBJECT_NAME

VARCHAR2（128）

不是NULL

指定DML处理程序的对象的名称

OPERATION_NAME

VARCHAR2（13）

使用DML处理程序的DML操作的名称：

  • 默认

  • 插入

  • UPDATE

  • 删除

  • LOB_UPDATE

  • ASSEMBLE_LOBS

USER_PROCEDURE

VARCHAR2（98）

用户指定的DML处理程序的名称，该处理程序处理包含对象上OPERATION_NAME列中的DML操作的行逻辑更改记录

ERROR_HANDLER

VARCHAR2（1）

指示DML处理程序是否仅处理导致应用错误（ Y ）或所有相关行逻辑更改记录（ N ）的相关行逻辑更改记录

APPLY_DATABASE_LINK

VARCHAR2（128）

应用更改的数据库链接。如果为null，则更改将应用于本地数据库。

APPLY_NAME

VARCHAR2（128）

给定对象的应用过程的名称

ASSEMBLE_LOBS

VARCHAR2（1）

指示LOB程序集是否用于处理程序（ Y ）处理的逻辑更改记录（LCR）中的LOB列（ N ）

在将LCR传递给处理程序之前，LOB程序集将LOB列的多个LCR组合在一起，从而将单行更改为一行LCR。

通过设置

VARCHAR2（10）

设置处理程序的实体。可能的值包括：

  • 加洲的金门大桥

  • 用户
