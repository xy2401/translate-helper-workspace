8.1寻找最佳属性

有时过多的信息会降低数据挖掘的效率。为构建和测试模型而组装的一些数据属性列不会为模型提供有意义的信息。有些确实会降低模型的质量和准确性。

例如，您希望收集有关特定人群的大量数据，因为您希望预测该组中某种疾病的可能性。其中一些信息（可能是其中的大部分）对疾病的易感性几乎没有影响。诸如每个家庭的汽车数量之类的属性可能无论如何都没有效果。

不相关的属性会给数据增加噪音并影响模型的准确性。噪声会增加模型的大小以及模型构建和评分所需的时间和系统资源。

具有许多属性的数据集可以包含相关的属性组。这些属性实际上测量相同的底层特征。它们在构建数据中的存在可能会扭曲算法的逻辑并影响模型的准确性。

宽数据（许多属性）通常呈现数据挖掘算法的处理挑战。模型属性是算法使用的处理空间的维度。处理空间的维数越高，算法处理所涉及的计算成本就越高。

为了最小化噪声，相关性和高维度的影响，某种形式的降维有时是数据挖掘的理想预处理步骤。特征选择和提取是降维的两种方法。

  • 特征选择 ：选择最相关的属性

  • 特征提取 ：将属性组合成一组新的特征

8.2关于特征选择和属性重要性

找出最重要的预测因素是一些数据挖掘项目的目标。例如，模型可能试图找出构成高信用风险的客户的主要特征。

Oracle Data Mining支持属性重要性挖掘功能，该功能根据属性在预测目标时的重要性对属性进行排名。属性重要性实际上不执行特征选择，因为所有预测变量都保留在模型中。在真实特征选择中，从模型中移除排名低于给定重要性阈值的属性。

特征选择可用作预处理步骤，以提高预测建模的计算效率。当启用自动数据准备（ADP）时，Oracle Data Mining在决策树算法和朴素贝叶斯内实现功能选择以进行优化。可以将广义线性模型（GLM）配置为执行特征选择作为预处理步骤。

8.2.1属性重要性和评分

Oracle Data Mining不支持属性重要性的评分操作。属性重要性的结果是根据其预测影响排序的构建数据的属性。排名和重要性度量可用于选择分类模型的训练数据。

8.3关于特征提取

特征提取是属性约简过程。不像功能选择，选择并保留最重要的属性，功能提取实际上转换属性。变换后的属性或特征是原始属性的线性组合。

特征提取过程会产生更小，更丰富的属性集。功能的最大数量可由用户指定或由算法确定。默认情况下，算法会确定它。

基于提取的特征构建的模型可以具有更高的质量，因为更少和更有意义的属性描述数据。

特征提取将具有较高维度的数据集投影到较小数量的维度上。因此，它对于数据可视化是有用的，因为当复杂数据集减少到二维或三维时，可以有效地可视化。

特征提取的一些应用是潜在语义分析，数据压缩，数据分解和投影以及模式识别。特征提取还可用于提高监督学习的速度和有效性。

特征提取可用于提取文档集合的主题，其中文档由一组关键字及其频率表示。每个主题（特征）由关键字组合表示。然后可以根据发现的主题来表示集合中的文档。

8.3.1特征提取和评分

Oracle Data Mining支持特征提取的评分操作。作为无监督挖掘函数，特征提取不涉及目标。应用时，特征提取模型将输入转换为一组要素。

8.4属性重要性和特征提取算法

了解用于属性重要性和特征提取的算法。

Oracle Data Mining支持以下属性重要性算法：

  • 最小描述长度

  • CUR矩阵分解

Oracle Data Mining支持以下特征提取算法：

  • 显式语义分析 （ESA） 。

  • 非负矩阵分解 （NMF） 。

  • 奇异值分解 （SVD）和预测成分分析 （PCA） 。

注意：

Oracle Data Mining使用NMF作为默认值特征提取算法。
