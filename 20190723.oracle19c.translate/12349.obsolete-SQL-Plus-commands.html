C.2 BTI [TLE] 文本 （过时的旧表格）

在每个报告页面的底部显示标题。

旧形式的BTITLE提供的格式化功能比新表单的格式更有限，但提供了与UFI（SQL * Plus的前身）的兼容性。旧表单将底部标题定义为空行，后跟带有居中文本的行。有关详细信息，请参阅TTI [TLE]文本（过时的旧表单） 。

C.3 COL [UMN] { column | expr } DEF [AULT]（已废弃）

将给定列的显示属性重置为默认值。

与COLUMN CLEAR具有相同的效果。

C.4 DOC [UMENT]（已废弃）

在脚本中开始一个文档块。

有关在脚本中插入注释的当前方法的信息，请参阅关于在脚本中放置注释部分和 REMARK命令 。

键入DOCUMENT并输入[Return]后，SQL * Plus将显示提示DOC>代替SQL>，直到您结束文档。一行上的“磅”字符（＃）本身就会结束文档。

如果已将DOCUMENT设置为OFF，则SQL * Plus会禁止显示DOCUMENT命令创建的文档块。有关更多信息，请参阅SET DOC [UMENT] {ON | OFF}（已废弃） 。

C.5 NEWPAGE [1 | n]（已废弃）

进展假脱机输出n行超出下一页的开头。

见SET NEWP [年龄] {1 | n | NONE}获取有关推进假脱机输出的当前方法的信息。

C.6 SET BUF [FER] { 缓冲区 | SQL}（已废弃）

使指定的缓冲区成为当前缓冲区。

最初，SQL缓冲区是当前缓冲区。SQL * Plus不需要使用多个缓冲区;单独的SQL缓冲区应该满足您的需求。

如果输入的缓冲区名称不存在，则SET BUFFER定义（创建并命名）缓冲区。退出SQL * Plus时，SQL * Plus会删除缓冲区及其内容。

运行查询会自动使SQL缓冲区成为当前缓冲区。要将文本从一个缓冲区复制到另一个缓冲区，请使用GET和SAVE命令。要清除当前缓冲区中的文本，请使用CLEAR BUFFER。要在使用其他缓冲区时从SQL缓冲区中清除文本，请使用CLEAR SQL。

C.7设置COM [PATIBILITY] {V7 | V8 | NATIVE }（已过时）

指定要使用的SQL语言分析语法的版本。

将兼容性设置为适用于Oracle7的V7，或适用于适用于Oracle8或更高版本的V8。兼容性始终默认为NATIVE。为要在连接的数据库上使用的Oracle数据库SQL语法版本设置COMPATIBILITY，否则。

默认兼容性设置NATIVE是现代Oracle数据库最相关的设置。

有关SQL * Plus版本兼容性设置的信息，请参阅SET SQLPLUSCOMPAT [IBILITY] {xy [.z]} 。

C.8 SET CLOSECUR [SOR] {ON | OFF}（已废弃）

设置游标使用行为。

On或OFF设置光标是否在每个SQL语句后关闭并重新打开。在某些情况下，此功能可能有助于释放数据库服务器中的资源。

C.9 SET DOC [UMENT] {ON | OFF}（已废弃）

显示或禁止DOCUMENT命令创建的文档块。

SET DOCUMENT ON会使文档块回显到屏幕上。设置DOCUMENT OFF会禁止显示文档块。

有关DOCUMENT命令的信息，请参阅DOC [UMENT]（已废弃） 。

C.10 SET MAXD [ATA] n （已废弃）

设置SQL * Plus可以处理的最大总行宽。

在SQL * Plus中，最大行宽现在是无限的。SQL * Plus会忽略使用SET MAXDATA设置的任何值。

C.11 SET SCAN {ON | OFF}（已废弃）

控制扫描是否存在替换变量和参数。OFF禁止替换变量和参数的处理; ON启用正常处理。

ON功能与SET DEFINE ON相同。

C.12 SET SPACE {1 | n }（已过时）

设置输出中列之间的空格数。n的最大值为10。

SET SPACE 0和SET COLSEP“命令具有相同的效果。SET COLSEP已废弃此命令，但您仍可以将其用于向后兼容。您可能更喜欢使用COLSEP，因为SHOW命令识别COLSEP并且无法识别SPACE。

C.13 SET TRU [NCATE] {ON | OFF}（已废弃）

控制SQL * Plus是否截断或包装对于当前行宽而言太长的数据项。

ON的功能与SET WRAP OFF相同，反之亦然。您可能更喜欢使用WRAP，因为SHOW命令识别WRAP并且无法识别TRUNCATE。

C.14 TTI [TLE] 文本 （过时的旧表格）

在每个报告页面的顶部显示标题。

旧形式的TTITLE提供的格式化功能比新表单的格式更有限，但提供了与UFI（SQL * Plus的前身）的兼容性。旧表单将顶部标题定义为日期左对齐且页码右对齐的行，后跟带有居中文本的行，然后是空行。

您输入的文本定义了TTITLE显示的标题。

SQL * Plus基于SET LINESIZE确定的行大小来居中文本。分隔符（|）开始一个新行;一行中的两个行分隔符（||）插入一个空行。您可以使用SET HEADSEP更改行分隔符。

您可以通过定义名为“_page”的变量来控制旧形式的TTITLE和BTITLE中页码的格式。_page的默认值是格式化字符串“page＆P4”。要更改格式，可以使用新的格式字符串DEFINE _page，如下所示：

SET ESCAPE / SQL> DEFINE _page ='Page /＆P2'

此格式化字符串将使用首字母大写字母打印单词“page”，并将页码格式设置为宽度为2。您可以将任何文本替换为“页面”，将任何数字替换为宽度。必须设置escape，以便SQL * Plus不会将＆符号（＆）解释为替换变量。请参见SET ERRORL [OGGING] {ON | OFF} [TABLE [schema。] tablename] [TRUNCATE] [IDENTIFIER identifier] ，了解有关设置转义字符的更多信息。

如果有效的new-form子句不紧跟命令名，SQL * Plus会以旧格式解释TTITLE。

如果你想使用带有TTITLE的CENTER并在一行上放置多个单词，你应该使用新形式的TTITLE。有关更多信息，请参阅TTITLE命令。

例

要使用旧形式的TTITLE在一行上设置左对齐日期和右对齐页码，然后在下一行设置SALES DEPARTMENT，在第三行设置PERSONNEL REPORT，请输入

TTITLE'销售部门|人事报告'
