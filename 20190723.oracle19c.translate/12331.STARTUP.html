12.46启动

句法

STARTUP db_options | cdb_options | upgrade_options

其中db选项具有以下语法：

[FORCE] [RESTRICT] [PFILE = filename ] [QUIET] [MOUNT [ dbname ] | [ OPEN [ open_db_options ] [ dbname ]] | NOMOUNT]

其中open_db_options具有以下语法：

阅读{仅| | WRITE [RECOVER]} |恢复

其中cdb_options具有以下语法：

root_connection_options | pdb_connection_options

其中root_connection_options具有以下语法：

PLUGGABLE DATABASE pdbname [FORCE] | [升级] | [限制] [ OPEN { open_pdb_options }]

其中pdb_connection_options具有以下语法：

[FORCE] | [升级] | [限制] [ OPEN { open_pdb_options }]

其中open_pdb_options具有以下语法：

READ WRITE |只读

以及upgrade_options具有以下语法的位置：

[PFILE = filename ] {UPGRADE | DOWNGRADE} [QUIET]

使用多个选项启动Oracle数据库实例，包括装入和打开数据库。

PDB STARTUP的先决条件

当前容器是可插拔数据库（PDB）时，只有在以下情况下才能使用STARTUP命令：

  • 当前用户具有SYSDBA，SYSOPER，SYSBACKUP或SYSDG系统特权。

  • 该权限通常在PDB中被授予或在本地授予。

  • 当前用户在连接时使用AS SYSDBA，AS SYSOPER，SYSBACKUP或AS SYSDG来执行权限。

  • PDB处于MOUNTED模式，不包括使用FORCE选项。

  • PDB必须处于READ ONLY或READ WRITE模式才能处于挂载模式。

有关更多信息，请参见“ Oracle数据库管理员指南”

注意：

仅使用db_options启动数据库（合并或其他）

仅在连接到Root时使用root_connection_options启动可插拔数据库。

仅使用pdb_options启动您连接的可插入数据库。

仅使用upgrade_options启动数据库（Consolidated或其他）进行升级或降级。

如果既没有READ WRITE也不READ ONLY指定，一个PDB将在被打开READ ONLY如果CDB其所属用作物理备用数据库，否则PDB将被打开READ WRITE 。

有关使用Consolidated和Pluggable Databases的更多信息，请参阅创建和配置Oracle数据库 。

条款

力

在重新启动之前，使用SHUTDOWN模式ABORT关闭当前的Oracle数据库实例（如果它正在运行）。如果当前实例正在运行且未指定FORCE，则会导致错误。在调试和异常情况下，FORCE很有用。通常不应该使用它。 

限制

只要使具有RESTRICTED SESSION系统特权的Oracle数据库用户能够连接到数据库 。稍后，您可以使用ALTER SYSTEM命令禁用受限会话功能。 

PFILE = 文件名

指定启动实例时要使用的客户机参数文件。如果未指定PFILE，则服务器尝试访问缺省服务器参数文件（spfile）。如果未找到默认spfile，则服务器将尝试访问默认pfile。默认文件是特定于平台的。例如，UNIX上的默认文件是$ ORACLE_HOME / dbs / init $ ORACLE_SID.ora，Windows上的默认文件是ORACLE_HOME \ database \ initORCL.ora。 

安静

禁止显示起始实例的系统全局区域信息。

安装 DBNAME

安装一个数据库但不打开它。

dbname是要装入或打开的数据库的名称。如果未指定数据库名称，则从初始化参数DB_NAME中获取数据库名称。

打开

坐骑和打开指定的数据库。 

NOMOUNT

导致在实例启动时不挂载数据库。

不能与MOUNT或OPEN一起使用。

恢复

指定必要时，应在启动实例之前执行介质恢复。STARTUP RECOVER与发出RECOVER DATABASE命令和启动实例具有相同的效果。只有RECOVER选项才能完全恢复。

如有必要，恢复将继续进行，就像AUTORECOVERY设置为ON一样，无论是否启用AUTORECOVERY。如果在预期位置找不到重做日志文件，则通过提示您使用建议的位置和需要应用的后续日志文件的名称，恢复将继续，就像禁用AUTORECOVERY一样。

升级

以OPEN UPGRADE模式启动数据库，并将系统初始化参数设置为启用数据库升级脚本所需的特定值。只有在首次使用新版本的Oracle数据库服务器启动数据库时才应使用升级。

运行时，升级脚本会将已安装的Oracle数据库版本或版本转换为更高版本，例如，将Oracle9 i数据库升级到Oracle Database 10 g 。升级完成后，应关闭数据库并正常重新启动。 

DOWNGRADE

以OPEN DOWNGRADE模式启动数据库，并将系统初始化参数设置为启用数据库降级脚本以运行所需的特定值。

运行时，降级脚本会将已安装的Oracle数据库版本或版本转换为先前版本，例如，将Oracle10 g数据库降级为Oracle9 i数据库。降级完成后，应关闭数据库并正常重新启动。 

可插入的数据库

使用pluggable database pdbname选项指定要在其上执行STARTUP命令的可插入数据库。

用法

您必须以SYSDBA，SYSOPER，SYSBACKUP或SYSDG身份连接到数据库。您无法通过调度程序连接到共享服务器。

没有参数的STARTUP等同于STARTUP OPEN。

即使恢复失败，STARTUP OPEN RECOVER也会安装并打开数据库。

例子

要使用标准参数文件启动CDB或非CDB实例，请装入默认数据库，然后打开数据库，输入

启动

或输入

STARTUP OPEN数据库

要使用标准参数文件启动实例，请装入默认数据库，然后打开数据库，输入

STARTUP FORCE RESTRICT MOUNT

要在不安装数据库的情况下使用参数文件TESTPARM启动实例，请输入

STARTUP PFILE = testparm NOMOUNT

要关闭特定数据库，请立即重新启动并打开它，仅允许具有RESTRICTED SESSION权限的用户访问，并使用参数文件MYINIT.ORA。输入

STARTUP FORCE RESTRICT PFILE = myinit.ora OPEN数据库

要启动实例并挂载但不打开数据库，请输入

CONNECT / as SYSDBA
