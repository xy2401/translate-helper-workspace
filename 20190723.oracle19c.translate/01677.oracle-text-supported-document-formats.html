B Oracle Text支持的文档格式

Oracle Text使用Oracle Outside In的HTML导出技术进行自动过滤。本附录提供了此版本的自动AUTO_FILTER过滤技术支持的文档和图形文件格式的表格。

B.1关于文档过滤技术

Oracle Text中的自动过滤技术使您可以使用CTX_DOC包将文档转换为HTML以进行文档呈现。

要使用自动过滤进行索引和DML处理，必须在过滤器首选项中指定AUTO_FILTER对象。

要使用自动过滤技术将文档转换为使用CTX_DOC包的HTML，您无需使用AUTO_FILTER索引首选项。

B.1.2格式支持的限制

本附录中列出的格式是AUTO_FILTER识别的格式。识别格式并不一定意味着可以从中提取文本。例如，扫描的文档通常是图像， AUTO_FILTER不执行光学字符识别。同样，无法从多媒体文件类型中提取文本以进行索引。

AUTO_FILTER过滤器不支持受密码保护的文档和受密码保护的内容的AUTO_FILTER 。

有关其他限制，请参阅有关特定文档类型的“ 支持的文档格式 ” 。

B.1.5 PDF过滤限制

过滤PDF文件时，以下限制适用：

  • 如果使用字符ID键控（CID）字体，预定义的CJK CMap文件或ToUnicode字体编码创建PDF文档，并且文档不包含嵌入字体，则支持多字节PDF。

  • PDF文档中的嵌入字体未正确过滤。它们通常使用问号（？）显示替换角色。

  • 在浏览器或查看窗口中显示时，PDF中的超链接无效。

  • 不支持注释，如音符，声音或电影。

B.1.6环境变量

用户不需要设置环境变量。

B.1.7一般限制

AUTO_FILTER过滤技术具有以下限制：

  • 任何少于0x20（十进制32）的ASCII字符都将转换为十六进制数字。

  • 不处理大于2GB的文件。

MIME支持说明

支持以下格式：

  • MIME格式

    • EML

    • MHT（网络档案）

    • NWS（新闻组单部分和多部分）

    • 简单文本邮件（在RFC 2822中定义）

  • TNEF格式

  • MIME编码，包括

    • base64（在RFC 1521中定义）

    • 二进制（在RFC 1521中定义）

    • binhex（在RFC 1741中定义）

    • BTOA

    • quoted-printable（在RFC 1521中定义）

    • utf-7（在RFC 2152中定义）

    • UUE

    • XXE

    • yenc

另外，消息的主体可以以多种方式编码。支持以下编码：

  • HTML

  • RTF

  • TNEF

  • 文本/丰富（在RFC 1523中定义）

  • Text / richtext（在RFC1341中定义）

  • 嵌入式邮件消息（在RFC 822中定义） - 这是作为新消息的链接处理的

MIME消息的附件可以以多种格式存储。Oracle Corporation处理其技术支持的所有附件类型。
