1.324 SQL92_SECURITY

SQL92_SECURITY指定是否必须为表授予用户SELECT权限才能执行引用WHERE或SET子句中的表列值的UPDATE或DELETE语句。

属性 描述

参数类型

布尔

默认值

真正

可修改

没有

可在PDB中修改

是

价值范围

是的|假

基本

没有

SQL标准指定安全管理员应该能够在执行引用WHERE或SET子句中的表列值的UPDATE或DELETE语句时要求用户对表具有SELECT权限。

值

  • 真正

    用户必须SELECT一个列特权引用它在WHERE一个子句DELETE或UPDATE语句，对在赋值的右边SET的子句UPDATE语句。

  • 假

    对DELETE语句的目标表具有DELETE特权的用户可以引用WHERE子句中该目标表的任何列。对UPDATE语句的目标表具有UPDATE特权的用户可以在WHERE子句中或在UPDATE语句的SET子句中的任何赋值的右侧引用该目标的任何列。
