ASMCMD文件访问控制命令

本主题提供ASMCMD Oracle ASM文件访问控制命令的摘要。

有关Oracle ASM文件访问控制的信息，请参阅管理磁盘组的Oracle ASM文件访问控制 。

管理具有ASMCMD的用户时，用户必须是现有操作系统用户，并且其用户名必须具有相应的操作系统用户ID。但是，只能验证与Oracle ASM实例位于同一群集中的用户。

注意：

修改文件的所有权时，即使文件的所有权发生更改，也无法在文件仍处于打开状态时删除文件的先前所有者。

chgrp命令

目的

更改文件或文件列表的用户组。

语法和说明

chgrp 用户组 文件 [ file ...]

表10-95列出了chgrp命令的语法选项。

表10-95 chgrp命令的选项

选项 描述

用户组

用户组的名称。

文件

文件名。

只有文件所有者或Oracle ASM管理员才能使用此命令。如果用户是文件所有者，则该用户还必须是该命令的所有者或组成员才能成功。

CHMOD

目的

更改文件或文件列表的权限。

语法和说明

chmod 模式 文件 [ file ...]

mode可以是以下形式之一：

  • { ugo | ug | uo | go | u g | o | a } { + | - } { r | w | rw }

    a指定所有用户的权限， u指定文件所有者的权限， g指定组权限， o指定其他用户的权限。

  • { 0 | 4 | 6 } { 0 | 4 | 6 } { 0 | 4 | 6 }

    第一个数字指定owner权限，第二个数字指定group权限，第三个数字指定other权限。

表10-96列出了chmod命令的语法选项。

表10-96 chmod命令的选项

选项 描述

6

读写权限

4

只读权限

0

没有权限

ü

所有者权限，与r或w

G

组权限，与r或w

Ø

其他用户权限，与r或w

一个

所有用户权限，与r或w

+

添加与r或w使用的权限

-

删除与r或w使用的权限

[R

阅读权限

w ^

写权限

文件

文件名

此命令接受由空格分隔的文件名或多个文件名。

您只能将文件权限设置为读写，只读和无权限。您无法将文件权限设置为只写。

如果更改打开文件的权限设置，则使用旧权限设置完成当前在文件上运行的操作。新设置在需要重新验证时生效。

要查看文件的权限，请使用带有--permission选项的ASMCMD ls命令。见“ ls ” 。

例子

以下是更改指定文件权限的chmod命令的示例。

例10-98使用ASMCMD chmod命令

ASMCMD [+ fra / orcl / archivelog / flashback]> chmod ug + rw log_7.264.684968167 log_8.265.684972027 ASMCMD [+ fra / orcl / archivelog / flashback]> chmod 640 log_7.264.684968167 log_8.265.684972027 ASMCMD [+]> ls  - permission + fra / orcl / archivelog / flashback用户组权限名称grid asm_fra rw -r ----- log_7.264.684968167 grid asm_fra rw -r ----- log_8.265.684972027

CHOWN

目的

更改文件或文件列表的所有者。

语法和说明

chown user [： usergroup ] 文件 [ file ...]

表10-97列出了chown命令的语法选项。

表10-97 chown命令的选项

选项 描述

用户

成为新所有者的用户的名称。

用户组

用户所属的用户组的名称。

文件

文件名。

user通常是指拥有数据库实例主目录的用户。Oracle ASM文件访问控制使用操作系统（OS）名称来标识数据库。

此命令接受由空格分隔的文件名或多个文件名。

如果更改打开文件的所有权设置，则以下情况适用：

  • 当前在该文件上运行的操作使用旧的所有权和用户组设置完成。新设置在需要重新验证时生效。

  • 在以群组方式重新启动群集中的所有实例之前，无法从磁盘组中删除该文件的新所有者。

只有Oracle ASM管理员才能使用此命令。

例子

以下是chown命令的示例，该命令将指定文件的所有者更改为oracle1操作系统用户。

例10-99使用ASMCMD chown命令

ASMCMD [+ fra / orcl / archivelog / flashback]> chown oracle1 log_7.264.684968167 log_8.265.684972027 ASMCMD [+ fra / orcl / archivelog / flashback]> chown oracle1：asm_fra log_9.264.687650269

grpmod

目的

向现有Oracle ASM用户组添加或删除操作系统（OS）用户。

语法和说明

grpmod {--add | --delete} diskgroup usergroup user [ user ...]

表10-99列出了grpmod命令的语法选项。

表10-99 grpmod命令的选项

选项 描述

- 加

指定将用户添加到用户组。

- 删除

指定从用户组中删除用户。

磁盘组

用户组所属的磁盘组的名称。

用户组

用户组的名称。

用户

要在用户组中添加或删除的用户的名称。

只有用户组的所有者才能使用此命令。该命令需要SYSASM权限才能运行。

此命令接受操作系统用户名或由空格分隔的多个用户名。操作系统用户通常是数据库实例主目录的所有者。

rpusr

目的

将一个操作系统（OS）用户替换为磁盘组中的另一个用户。

语法和说明

rpusr diskgroup user1 user2

表10-107列出了rpusr命令的语法选项。

表10-107 rpusr命令的选项

选项 描述

磁盘组

指定磁盘组的名称。

USER1

要替换的用户的名称。

用户2

替换磁盘组中现有用户的用户名。

rpusr命令将一个操作系统用户替换为Oracle ASM磁盘组中的另一个操作系统用户。替换用户不能是当前位于磁盘组用户列表中的用户。

如果命令成功，则当前用户以前拥有的所有文件现在都由替换用户拥有。将自动从磁盘组的用户列表中删除当前用户。可以将当前用户添加回磁盘组，但此用户不会拥有任何文件。

更改打开文件的所有权时，在重新启动集群中的所有实例之前，不能使用所有者的文件删除新所有者。

该命令需要SYSASM权限才能运行。

例子

以下是rpusr命令的示例，该命令用data磁盘组中的oracle2用户替换oracle1用户。

示例10-109使用ASMCMD rpusr命令

ASMCMD [+]> rpusr数据oracle1 oracle2
