关于Oracle Grid Infrastructure名称解析选项

在开始安装之前，每个节点上必须至少配置两个接口：一个用于专用IP地址，另一个用于公用IP地址。

在安装过程中，系统会要求您确定Oracle Universal Installer（OUI）在群集节点上检测到的每个网络接口的计划用途。将每个接口标识为公共或私有接口，或者作为您不希望Oracle Grid Infrastructure或Oracle ASM使用的接口。公共和虚拟互联网协议（VIP）地址在公共接口上配置。私有地址在私有接口上配置。

使用以下选项之一配置IP地址：

使用多集群或标准Oracle网格命名服务（GNS）分配动态IP地址

如果选择此选项，则网络管理员将委派GNS解析的子域（标准或多集群）。GNS要求根据您配置区域委派的方式而有所不同。如果您使用区域委派配置GNS，则GNS将被委派为其解析服务请求的域。如果您在没有区域委派的情况下配置GNS，则GNS具有由DNS解析的虚拟IP地址。

对于带区域委派的GNS：

  • 对于IPv4，群集使用的公共网络上运行的DHCP服务

  • 对于IPv6，群集使用的公共网络上运行的自动配置服务

  • DHCP服务器上有足够的地址为每个节点提供一个IP地址，并为群集的单客户端访问名称（SCAN）使用的群集提供三个IP地址

使用现有的GNS配置

从Oracle Grid Infrastructure 12c第1版（12.1）开始，多个集群可以使用单个GNS实例。要将GNS用于多个群集，DNS管理员必须委派一个区域供GNS使用。此外，必须在网络上的某处启动GNS实例，并且GNS实例必须是可访问的（不会被防火墙阻止）。向GNS实例注册的所有节点名称必须是唯一的。

使用DNS或主机文件解析分配静态IP地址

如果选择此选项，则网络管理员会为群集中的每个物理主机名分配固定IP地址，并为Oracle Clusterware管理的VIP分配IP。此外，每个节点都使用基于域名服务器（DNS）的静态名称解析，或者必须更新群集和客户端的主机文件，从而导致SCAN功能受限。选择此选项要求您在修改群集时请求网络管理更新。

对于没有区域委派的GNS，请为群集配置GNS虚拟IP地址（VIP）。要启用Oracle Flex Cluster，您必须至少配置GNS虚拟IP地址。
