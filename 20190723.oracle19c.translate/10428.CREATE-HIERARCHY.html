创造层次

目的

使用CREATE HIERARCHY语句创建层次结构。层次结构指定属性维度的级别之间的层次关系。

小费：

您可以在Oracle Live SQL网站上查看和运行创建层次结构的SQL脚本，网址为https://livesql.oracle.com/apex/livesql/file/index.html 。该网站提供脚本和教程，演示分析视图的创建和使用。

先决条件

要在自己的架构中创建层次结构，您必须具有CREATE HIERARCHY系统特权。要在另一个用户的模式中创建层次结构，您必须具有CREATE ANY HIERARCHY系统特权。

语义

或替换

指定OR REPLACE以使用不同的定义替换层次结构的现有定义。

力量和NOFORCE

指定FORCE以强制创建层次结构，即使它未成功编译。如果指定NOFORCE ，则层次结构必须成功编译，否则会发生错误。默认为NOFORCE 。

模式

指定用于创建层次结构的架构。如果未指定架构，则Oracle Database会在您自己的架构中创建层次结构。

等级制度

指定层次结构的名称。

sharing_clause

指定是否将层次结构创建为应用程序公共对象。指定METADATA共享层次结构的元数据，但其数据对每个容器都是唯一的。指定DATA共享层次结构对象;它的数据对于应用程序容器中的所有容器都是相同的，并且数据仅存储在应用程序根目录中。指定NONE排除层次结构的共享。

classification_clause

使用classification子句指定CAPTION或DESCRIPTION分类的值，并指定用户定义的分类。分类提供描述性元数据，应用程序可用于提供有关分析视图及其组件的信息。

您可以为同一对象指定任意数量的分类。分类的最大长度为4000字节。

对于CAPTION和DESCRIPTION分类，你可以使用快捷键DDL CAPTION ' caption '和DESCRIPTION ' description '或完全分类的语法。

您可以按语言更改分类值。要为CAPTION或DESCRIPTION分类指定语言，必须使用完整语法。如果未指定语言，则分类的语言值为NULL 。语言值必须为NULL或有效的NLS_LANGUAGE值。

hier_using_clause

指定具有层次结构成员的属性维。

level_hier_clause

指定层次结构级别的组织。

hier_attrs_clause

指定包含层次结构属性的描述性元数据的分类。一个hier_attr_clause对于给定hier_attr_name可能只出现在列表中一次。

所有层次结构始终包含所有层次结构属性，但层次结构属性没有与之关联的描述性元数据，除非您使用此子句指定它。

hier_attr_clause

指定层次结构属性并为其提供一个或多个分类。

hier_attr_name

指定分层属性。
