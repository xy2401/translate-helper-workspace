Oracle Grid Infrastructure IP名称和地址要求

查看有关Oracle Grid Infrastructure IP名称和地址要求的此信息。

对于小型群集，您可以使用IP地址的静态配置。对于大型群集，手动维护大量所需的IP地址变得过于繁琐。将Oracle Grid Naming Service与大型群集配合使用可以降低网络管理成本。

  • 关于Oracle Grid Infrastructure名称解析选项
    在开始安装之前，每个节点上必须至少配置两个接口：一个用于专用IP地址，另一个用于公用IP地址。
  • 群集名称和SCAN要求
    在选择群集名称和SCAN之前，请查看此信息。
  • 网格命名服务（GNS）的IP名称和地址要求
    查看此信息，了解网格命名服务（GNS）的IP名称和地址要求。
  • 多群集GNS的IP名称和地址要求
    多集群GNS与标准GNS的不同之处在于，多集群GNS在一组集群中提供单一网络服务，而不是单个集群的网络服务。
  • 手动配置群集的IP名称和地址要求
    对于Oracle Flex Clusters和Oracle Member Cluster for Applications，如果未启用GNS，请配置静态群集节点名称和地址。
  • 确认SCAN的DNS配置
    使用nslookup命令确认DNS正确地将SCAN与地址相关联。
