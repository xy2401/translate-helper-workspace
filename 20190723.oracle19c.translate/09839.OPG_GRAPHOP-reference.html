6 OPG_GRAPHOP包子程序

OPG_GRAPHOP包中包含Oracle数据库中属性图上各种操作的子程序。

要使用本章中的子程序，您必须了解本书前几章中的概念和用法信息。

本章按字母顺序提供有关子程序的参考信息。

  • OPG_GRAPHOP.POPULATE_SKELETON_TAB

6.1 OPG_GRAPHOP.POPULATE_SKELETON_TAB

格式

OPG_GRAPHOP.POPULATE_SKELETON_TAB（图形IN VARCHAR2，dop IN INTEGER DEFAULT 4，tbs IN VARCHAR2 DEFAULT NULL，选项IN VARCHAR2 DEFAULT NULL）;

描述

填充骨架表（<graph-name> GT $）。默认情况下，在填充表之前，会截断（删除）骨架表中的所有现有内容。

参数

图表

属性图的名称。

dop

操作的并行度。

tbs

用于保存框架表的索引数据的表空间的名称。

选项

可用于自定义骨架表填充的选项。（一个或多个，逗号分隔。）

  • 'KEEP_DATA = T'导致在填充表之前不删除任何现有表。在现有行之后添加新行。

  • 'PDML = T'跳过默认索引创建。

使用说明

您必须具有CREATE TABLE和CREATE INDEX权限才能调用此过程。

骨架表（GE $）的EID列上有唯一的索引约束。因此，如果指定KEEP_DATA=T选项，并且新数据与现有数据重叠，则将违反唯一键约束，从而导致错误。

例子

以下示例填充名为mypg的属性图的骨架表。 

EXECUTE OPG_GRAPHOP.POPULATE_SKELETON_TAB（'mypg'，4，'pgts'，'PDML = T'）;
