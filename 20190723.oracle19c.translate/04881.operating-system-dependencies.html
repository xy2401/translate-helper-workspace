一个操作系统依赖性

COBOL编程的一些细节因系统而异。本附录是有关Pro * COBOL的所有系统特定问题的集合。在适用的情况下，会向文档集中的其他来源提供参考。

A.1本手册中的系统特定参考

系统特定的参考在以下部分中描述，按主题区域分组。

A.1.1 COBOL版本

Pro * COBOL预编译器支持为您的操作系统（通常为COBOL-85或COBOL-74）标准实施COBOL。某些平台可能同时支持COBOL实现。检查Oracle系统特定的文档。

A.1.2主机变量

声明和命名主机变量的方式取决于您使用的COBOL编译器。有关声明和命名主机变量的详细信息，请查看COBOL用户指南。

A.1.2.1声明

根据COBOL规则声明主机变量，指定Oracle支持的COBOL数据类型。表4-6显示了您可以指定的COBOL数据类型和假型。但是，您的COBOL实现可能不包括所有这些实现。

A.1.2.2命名

主机变量名称必须仅包含字母，数字和连字符。他们必须以一封信开头。它们可以是任意长度，但只有前30个字符才有意义。您的编译器可能允许不同的最大长度。

由于Pro * COBOL限制，在与SQLLIB（C例程）交互时，除非正确对齐主机变量的边界，否则可能会出现一些不可预测的结果。有关定义主变量边界对齐的特定信息，请参阅COBOL文档。解决方法可能包括：

  • 使用FILLER手动对齐

  • 使用01级条目强制边界

  • 如果数据源是第三方代码，则使用77级条目或01级条目的临时变量，并将其用作主机变量。

A.1.3包含声明

您可以包含任何文件。预编译Pro * COBOL程序时，每个EXEC SQL INCLUDE语句都将替换为语句中指定的文件的副本。

如果您的系统使用文件扩展名但未指定文件扩展名，则Pro * COBOL预编译器将采用源文件（通常为COB）的默认扩展名。默认扩展名取决于系统。检查Oracle系统特定的文档。

如果系统使用目录，则可以通过指定预编译器选项INCLUDE = path来为包含的文件设置目录路径 。必须使用INCLUDE为非标准文件指定目录路径，除非它们存储在当前目录中。指定目录路径的语法是特定于系统的。检查Oracle系统特定的文档。

A.1.4 MAXLITERAL默认值

使用MAXLITERAL预编译器选项，您可以指定预编译器生成的字符串文字的最大长度，以便不超过编译器限制。MAXLITERAL默认值为256，但您可能必须指定较低的值。

例如，如果您的COBOL编译器无法处理长度超过132个字符的字符串文字，请指定“MAXLITERAL = 132”。检查COBOL编译器用户指南。有关MAXLITERAL选项的更多信息，请参阅预编译器选项

A.1.5用于多字节全球化支持字符的PIC N或Pic G子句

某些COBOL编译器可能不支持使用PIC N或PIC G子句声明多字节全球化支持字符变量。在编写使用这些子句声明多字节字符变量的源代码之前，请检查COBOL用户指南。

A.1.6返回代码特殊注册可能不可预测。

在任何SQL语句或SQLLIB函数之后，RETURN-CODE特殊寄存器（对于那些支持它的系统）的内容是不可预测的。

A.1.7二进制数据的字节顺序

在某些平台上，COBOL编译器会反转二进制数据的字节顺序。有关COMP5预编译器选项，请参阅特定于平台的文档。
