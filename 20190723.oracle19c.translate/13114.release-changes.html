新功能

以下功能是此版本中的新增功能：

  • 自动解决SQL计划回归

    SQL计划管理在自动工作负载存储库（AWR）中搜索SQL语句。通过最高负载确定优先级，SPM Evolve Advisor在所有可用来源中查找备用计划，自动为SQL计划基准添加性能更佳的计划。

    请参阅“i 关于SPM Evolve Advisor任务 ” 。

  • 报告提示使用情况

    Oracle数据库包括一个提示使用报告机制，用于报告在计划生成期间是否使用了提示。在某些情况下，该报告解释了为什么没有使用提示，例如，语法错误或提示之间的冲突。您可以使用标准DBMS_XPLAN显示功能生成提示使用情况报告。

    请参阅“ 提示报告 ” 。

  • 计划比较功能

    DBMS_XPLAN.COMPARE_PLANS函数采用参考计划和测试计划列表，并突出显示它们之间的差异。用户可以通过识别差异来源来分类计划重现性问题。

    请参阅“ 比较执行计划 ” 。

  • 实时统计

    Oracle数据库在传统DML操作期间自动收集实时统计信息 。这些统计信息可以增强DBMS_STATS作业收集的标准统计信息。

    请参阅“ 实时统计 ” 。

  • 高频自动优化器统计收集

    此轻量级任务定期收集陈旧对象的统计信息。默认时间间隔为15分钟。与自动统计信息收集作业相反，高频任务不执行诸如清除不存在的对象的统计信息或调用Optimizer Statistics Advisor等操作。

    请参阅“ 配置高频自动优化器统计信息收集 ” 。

  • 统计维护增强功能

    分区移动，合并和合并操作维护全局和分区级统计信息。

    请参阅“ 在线统计信息收集 ” 。

  • 为开发人员提供实时SQL监控

    没有SELECT_CATALOG_ROLE数据库用户可以为自己的SQL语句生成和查看SQL Monitor报告，包括执行计划和性能指标。

    请参阅“ 生成和访问SQL监视器报告 ” 。

  • 基于统计的查询转换

    在某些情况下，数据库可以通过访问表统计信息而不是表本身来满足简单的聚合查询，从而显着减少查询响应时间。

    请参阅“ 基于统计的查询转换 ” 。

  • 隔离失控的SQL语句

    Oracle数据库自动隔离由Oracle数据库资源管理器（资源管理器）终止的SQL语句，以破坏资源限制。通过将计划放在“黑名单”上，数据库可以防止资源占用语句再次执行。

    请参阅“ 关于隔离的SQL计划 ” 。

  • 自动索引

    自动后台任务监视系统工作负载，然后创建和维护适合工作负载中语句的索引。数据库验证索引的性能影响，并不断保持或改变其决策，以最大限度地提高性能。例如，当索引显着提高查询性能时，数据库可能会在语句谓词中使用的列上创建索引。

    请参阅“ 自动索引 ” 。

新功能

以下功能是此版本中的新增功能：

  • 私人临时表

    专用临时表是临时数据库对象，在事务或会话结束时自动删除。私有临时表存储在内存中，仅对创建它的会话可见。私有临时表将临时表的范围限制为会话或事务，从而为应用程序编码提供更大的灵活性，从而简化代码维护并提供更好的即用功能。

    请参阅“ 全局临时表的统计信息 ” 。

  • 近似前N个查询处理

    要比传统查询更快地获得“前n个 ”查询结果，请将APPROX_SUM和APPROX_COUNT SQL函数与APPROX_RANK一起APPROX_RANK 。

    请参阅“ 关于近似查询处理 ” 。

  • Oracle数据库云服务器的SQL Tuning Advisor增强功能

    SQL Tuning Advisor可以推荐支持Exadata的SQL配置文件。在Oracle数据库云服务器上，智能扫描的成本取决于系统统计信息I / O查找时间（ ioseektim ），多块读取计数（ mbrc ）和I / O传输速度（ iotfrspeed ）。这些统计信息的值通常在Exadata上有所不同，因此可能影响计划的选择。如果系统统计信息过时，并且如果收集它们可以提高性能，则SQL Tuning Advisor建议接受可识别Exadata的SQL配置文件。

    请参阅“ 统计分析 ”和“ SQL配置文件中的统计信息 ” 。

  • 用于管理SQL调优集的新包

    您可以使用DBMS_SQLSET而不是DBMS_SQLTUNE来创建，修改，删除和执行所有其他SQL调优集操作。

    请参阅“ SQL调整集的命令行界面 ” 。

  • 可扩展序列

    当将数据加载到使用序列值作为键的表中时，可伸缩序列减轻了索引叶块争用。

  • 将OPTIMIZER_ADAPTIVE_STATISTICS与性能反馈分离

    与以前的版本不同，将OPTIMIZER_ADAPTIVE_STATISTICS初始化参数设置为TRUE或FALSE现在对性能反馈没有影响。
