2.8.4自动上传集合

配置Oracle ORAchk和Oracle EXAchk以自动将检查结果上载到Oracle Health Check Collections Manager数据库。

指定连接字符串和密码以连接到数据库。Oracle运行状况检查集合管理器将连接详细信息存储在加密钱包中。

要配置Oracle ORAchk和Oracle EXAchk以自动上载检查结果：

  1. 使用–setdbupload选项指定连接详细信息。对于默认选项，请使用–setdbupload all 。
    orachk -setdbupload all
    exachk -setdbupload all

    Oracle Health Check Collections Manager会提示您输入连接字符串和密码的值。Oracle Health Check Collections Manager将这些值存储在加密的钱包文件中。

  2. 使用–getdbupload选项验证钱包中设置的值。
    orachk -getdbupload
    exachk -getdbupload

    Oracle ORAchk和Oracle EXAchk自动使用RAT_UPLOAD_USER和RAT_ZIP_UPLOAD_TABLE环境变量中设置的默认值。

  3. 如果Oracle ORAchk和Oracle EXAchk成功连接到数据库，请使用–checkdbupload选项进行验证。
    orachk -checkdbupload
    exachk -checkdbupload
  4. 为Oracle ORAchk和Oracle EXAchk检查结果设置数据库上载。
    orachk -setdbupload all
    exachk -setdbupload all

    注意：

    如上例所示，使用连接字符串的完全限定地址。不要使用tnsnames.ora文件中的别名。

    使用完全限定的地址消除了在运行该工具的所有服务器上依赖tnsnames.ora文件名解析的需要。

  5. 查看Oracle ORAchk和Oracle EXAchk数据库检查结果上载。
    orachk -getdbupload
    exachk -getdbupload

示例2-1检查Oracle ORAchk和Oracle EXAchk检查结果上载

$ ./orachk -checkdbupload配置很适合将结果上传到数据库。

在运行状况检查收集结束时，Oracle ORAchk和Oracle EXAchk会检查是否已设置所需的连接详细信息（在钱包或环境变量中）。如果正确设置了连接详细信息，则Oracle ORAchk和Oracle EXAchk会上载收集结果。

要配置许多Oracle ORAchk和Oracle EXAchk实例：

  1. 使用-setdbupload all选项创建一次钱包，然后在出现提示时输入值。

  2. 将生成的wallet目录复制到每个Oracle ORAchk和Oracle EXAchk实例目录。

您还可以将环境变量RAT_WALLET_LOC设置为指向wallet目录的位置。

其他可配置的上传值包括：

  • RAT_UPLOAD_USER ：控制要连接的用户（默认为ORACHKCM ）。

  • RAT_UPLOAD_TABLE ：控制表名以存储非压缩的集合结果（默认情况下不使用）。

  • RAT_PATCH_UPLOAD_TABLE ：控制表名以存储非压缩补丁结果（默认情况下不使用）。

  • RAT_UPLOAD_ORACLE_HOME ：控制在建立连接和上载时使用的ORACLE_HOME 。

    默认情况下， ORACLE_HOME环境变量设置为Oracle ORAchk和Oracle EXAchk发现的Oracle Grid Infrastructure Grid主目录。

RCA13_DOCS ：不可配置为使用Oracle Health Check Collections Manager，因为RCA13_DOCS是Oracle Health Check Collections Manager查找的表。

RAT_UPLOAD_TABLE和RAT_PATCH_UPLOAD_TABLE ：默认情况下不使用，因为压缩的集合详细信息存储在RCA13_DOCS 。

如果您使用自己的自定义应用程序来查看集合结果，请配置RAT_UPLOAD_TABLE和RAT_PATCH_UPLOAD_TABLE环境变量。

您还可以在钱包中设置这些值。

例如：
$ ./orachk -setdbupload all
$ ./exachk -setdbupload all

这会提示您输入并设置RAT_UPLOAD_CONNECT_STRING和RAT_UPLOAD_PASSWORD ，然后使用

$ ./orachk -setdbupload RAT_PATCH_UPLOAD_TABLE，RAT_PATCH_UPLOAD_TABLE
$ ./exachk -setdbupload RAT_PATCH_UPLOAD_TABLE，RAT_PATCH_UPLOAD_TABLE

注意：

或者，使用环境变量设置钱包中设置的所有值。如果使用环境变量RAT_UPLOAD_CONNECT_STRING设置值，则将值括在双引号中。

例如：
export RAT_UPLOAD_CONNECT_STRING =“（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（HOST = myserver44.example.com）（PORT = 1521））（CONNECT_DATA =（SERVER = DEDICATED）（SERVICE_NAME = orachkcm.example.com）））”

2.8.5查看和重新尝试失败的上载

配置Oracle ORAchk和Oracle EXAchk以显示并重新尝试上载失败的上载。

这些工具将值存储在collection_dir /outfiles/check_env.out文件中，以记录先前的数据库上载是否成功。

以下示例显示已设置数据库上载，但上次上载失败：
DATABASE_UPLOAD_SETUP = 1 DATABASE_UPLOAD_STATUS = 0

要查看和重新尝试失败的上传：

  1. 要查看失败的集合，请使用-checkfaileduploads选项。
    ./orachk-checkfaileduploads
    ./exachk -checkfaileduploads
    例如：
    $ ./orachk -checkfaileduploads上传集合失败的列表/home/oracle/orachk_myserver_042016_232011.zip /home/oracle/orachk_myserver_042016_231732.zip /home/oracle/orachk_myserver_042016_230811.zip /home/oracle/orachk_myserver_042016_222227.zip / home / oracle / orachk_myserver_042016_222043。压缩
  2. 要重新尝试收集上载，请使用-uploadfailed选项
    指定all以上载所有集合或指定以逗号分隔的集合列表：
    ./orachk -uploadfailed all | list of failed collections
    ./exachk -uploadfailed all | list of failed collections
    例如：
    ./orachk -uploadfailed “ /home/oracle/orachk_myserver_042016_232011.zip ， /home/oracle/orachk_myserver_042016_231732.zip ”

    注意：

    由于SQL唯一约束，您无法上载先前上载的集合。

2.8.6编写用户定义的检查

定义，测试和维护您自己的特定于您的环境的检查。

Oracle支持用于创建和运行用户定义检查的框架，但不支持检查的逻辑。您有责任测试，验证，创作，维护和支持用户定义的检查。在运行时，Oracle ORAchk和Oracle EXAchk脚本运行用户定义的检查，并在HTML报告的“ 用户定义的检查”部分中显示结果。

用户定义的检查存储在Oracle Health Check Collections Manager模式中，并输出到XML文件，该文件与ORAchk脚本位于同一位置。在系统上运行时，ORAchk 12.1.0.2.5及更高版本会尝试查找XML文件。如果找到，则Oracle ORAchk运行其中包含的检查，并将结果包含在标准HTML报告中。

编写用户定义的检查：

  1. 单击“ 用户定义的检查”选项卡，然后选择“ 添加新检查” 。

    图2-16用户定义的检查选项卡


    “图2-16用户自定义检查选项卡”的说明
  2. 选择OS Check或SQL Check作为审核检查类型 。

    操作系统检查使用系统命令来确定检查状态。SQL检查运行SQL语句以确定检查状态。

    图2-17用户定义的检查选项卡 - 审核检查类型


    “图2-17用户自定义检查选项卡 - 审核检查类型”的说明

    选择审核检查类型后 ，Oracle Health Check Collections Manager将更新适用的字段。

    在创作过程中的任何时候，单击字段标题以查看特定于该字段的帮助文档。

    支持操作系统和SQL命令。不支持以root身份运行用户定义的检查。

    图2-18用户定义的检查选项卡 - 审核检查类型 - 操作系统检查


    “图2-18用户自定义检查选项卡 - 审核检查类型 - 操作系统检查”的说明

    创建检查后，检查将在“ 可用审计检查”部分中列出。

    使用此页面上的过滤器过滤检查。

    图2-19用户定义的检查选项卡 - 可用的审核检查


    “图2-19用户自定义检查选项卡 - 可用审计检查”的说明
  3. 单击“ 生成XML” 。

    在右侧，找到下载生成的user_defined_checks.xml文件的链接。

    生成的XML文件包括已创作但尚未保留的所有检查。保留支票等同于逻辑删除。如果检查有问题或逻辑不完美，则将检查置于保持状态。保留的检查不包含在XML文件中。如果检查是生产就绪，则删除保留以在下次生成XML文件时包括检查。

  4. 下载user_defined_checks.xml文件并将其保存到与Oracle ORAchk和Oracle EXAchk工具相同的目录中。

    Oracle ORAchk和Oracle EXAchk在下次运行时运行用户定义的检查。

    图2-20用户定义的检查选项卡 - 下载用户定义的检查


    “图2-20用户自定义检查选项卡 - 下载用户自定义检查”的说明
  5. 或者，要仅运行用户定义的检查，请使用配置文件user_defined_checks 。
    使用此选项时，用户定义的检查是唯一运行的检查，“ 用户定义的检查”部分是唯一一个在报告中显示结果的部分。
    ./orachk -profile user_defined_checks
    ./exachk -profile user_defined_checks
  6. 要在运行时省略用户定义的检查，请使用–excludeprofile选项。
    ./orachk -excludeprofile user_defined_checks
    ./exachk -excludeprofile user_defined_checks
