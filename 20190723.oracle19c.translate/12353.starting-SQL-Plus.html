3.1登录用户名和密码

当您启动SQL * Plus时，您需要一个用户名和密码以登录Oracle数据库架构。您的用户名和密码将您标识为Oracle数据库架构的授权用户。

数据库管理员（DBA）负责使用必要的权限创建数据库帐户，并为您提供用于访问帐户的用户名和密码。

将创建默认登录，并在Oracle数据库安装期间提示您输入关联的密码。创建的一些默认登录用户名是：

  • SYS

  • 系统

  • HR

在Oracle数据库安装期间，将在消息中创建并显示登录。

有关默认登录的详细信息，请参阅Oracle数据库用户类型 。

登录后，您可以使用其他用户名进行连接使用CONNECT 命令。用户名和密码必须对数据库有效。例如，要使用密码FOX将用户名TODD连接到默认数据库，您可以输入

连接TODD

系统会提示您输入密码FOX。

在命令行界面中，如果省略用户名和密码，SQL * Plus会提示您输入用户名和密码。由于CONNECT首先将您与当前数据库断开连接，因此如果在CONNECT命令中使用无效的用户名和密码，则将保持未连接到任何数据库。

如果以帐户已过期的用户身份登录或连接，系统将提示您更改密码，然后才能进行连接。

如果帐户被锁定，则会显示一条消息，并且在您的DBA解锁帐户之前，不允许以该用户身份进行连接。

您可以使用DISCONNECT 命令断开与数据库的连接没有离开在SQL * Plus。

3.1.1安全外部密码存储

作为应用程序使用密码凭据连接到数据库的大规模部署的命令行替代方法，可以将此类凭据存储在客户端Oracle钱包中。Oracle钱包是一个安全的软件容器，用于存储身份验证和签名凭据。

在客户端Oracle钱包中存储数据库密码凭证，无需在应用程序代码中嵌入用户名和密码， 批量作业，或脚本。这样可以降低在脚本和应用程序代码中以明文形式泄露密码的风险，并简化维护，因为每次用户名和密码更改时都不需要更改代码。此外，无需更改应用程序代码也可以更轻松地为这些用户帐户实施密码管理策略。

将客户端配置为使用外部密码存储时，应用程序可以使用以下语法连接到使用密码身份验证的数据库：

CONNECT / @ database_alias

请注意，您无需在此CONNECT语句中指定数据库登录凭据。相反，您的系统会在客户端钱包中查找数据库登录凭据。

也可以看看：

“Oracle数据库管理员指南” ，了解有关配置客户端以使用安全外部密码存储的信息以及有关管理其中的凭据的信息。

3.1.2过期密码

在命令行界面中，如果密码已过期，则SQL * Plus会在您尝试登录时提示您更改密码。成功更改密码后即可登录。

3.1.3关于更改密码

在命令行界面中，您可以使用PASSWORD命令更改密码。见密码 。

3.2关于连接数据库

您必须先连接到Oracle数据库（实例），然后才能查询或修改该数据库中的数据。您可以连接到默认数据库以及可通过网络访问的其他数据库。要通过网络连接到另一个数据库，两个数据库都必须配置Oracle Net，并具有兼容的网络驱动程序。您必须输入连接标识符或网络服务名称才能连接到默认数据库以外的数据库。

输入连接标识符或网络服务名称：

  • 作为启动命令行会话时SQLPLUS程序语法的参数。

  • 作为当前会话的CONNECT命令的参数。有关详细用法，请参阅“ Oracle数据库管理员指南”中的访问CDB中的容器 。

3.2.1网络服务名称

您的DBA负责创建您使用和定义的数据库 tnsnames.ora文件中的网络服务名称。

tnsnames.ora文件中的网络服务名称定义具有以下语法：

net_service_name =（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = host ）（PORT = port ））（CONNECT_DATA =（SERVICE_NAME = service_name ）））

要使用网络服务名称（别名），它必须在运行SQL * Plus的计算机上的tnsnames.ora文件中有一个条目。如果使用连接标识符，则不需要tnsnames.ora中的条目。

示例3-1 sales数据库的tnsnames.ora条目

SALES1 =（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = sales.us.acme.com）））

示例3-2使用网络服务名称启动到sales数据库的命令行会话

SQLPLUS hr @ SALES1

有关数据库连接和网络服务名称定义的详细信息，请参阅Oracle Net Services的 配置参数和配置以及管理 。

3.2.2完整连接标识符

根据您的配置，使用完整连接标识符语法如：

（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = host ）（PORT = port ））（CONNECT_DATA =（SERVICE_NAME = service_name ）））

SERVICE_NAME是在数据库创建期间输入的全局数据库名称。它将数据库名称与域名组合在一起。例如， SERVICE_NAME sales.us.acme.com的数据库名称为sales ，域名为us.acme.com 。

INSTANCE_NAME是您在创建期间为数据库实例提供的名称。它默认为您在数据库创建期间输入的SID。

Oracle系统标识符（ SID ）标识特定的Oracle 8.0版数据库实例。

您可以选择使用INSTANCE_NAME代替SERVICE_NAME短语。

连接到Oracle 8.0或更早版本的数据库时，请使用SID代替SERVICE_NAME 。

示例3-3 SALES1的完整连接标识符

SQLPLUS hr @ \“（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = sales.us.acme.com）））\”

3.2.3简易连接标识符

容易或缩写连接标识符具有以下语法：

[//] host [： port ] [/ service_name ]

示例3-4使用简易连接标识符启动到sales数据库的命令行会话

sqlplus hr @ \“sales-server：1521 / sales.us.acme.com \”

示例3-5使用简易连接标识符连接到销售数据库

省略密码时，需要引用连接字符串。

connect hr @“sales-server：1521 / sales.us.acme.com”

只要您可以使用完整连接标识符或网络服务名称，就可以使用简单连接标识符。简单的语法不太复杂，不需要tnsnames.ora条目。

注意：

从Oracle Database 19c版开始，easy connect语法具有改进的功能，称为Easy Connect Plus。有关更多信息，请参阅关于Easy Connect Plus 。

3.2.4与/ NOLOG的无连接会话

在命令行界面中，可以不使用SQL * Plus 连接到数据库。这对于执行某些数据库管理任务，编写可传输脚本或使用SQL * Plus编辑命令来编写或编辑脚本非常有用。

您使用SQLPLUS命令的/ NOLOG参数来启动无连接命令行会话。SQL * Plus启动后，您可以使用CONNECT命令连接到数据库。

示例3-6使用/ NOLOG启动无连接SQL * Plus会话

3.4关于退出SQL * Plus命令行

如果你不能登录到SQL * Plus因为您的用户名或密码无效或由于某些其他原因，SQL * Plus返回的错误状态等同于 EXIT FAILURE命令。有关详细信息，请参阅EXIT命令。

当你完成工作使用SQL * Plus 并希望返回操作系统，输入EXIT 在SQL * Plus提示符下输入或退出，或输入文件结束字符，在UNIX上输入Ctrl + D或在Windows上输入Ctrl + Z.

在返回操作系统提示符之前，SQL * Plus会显示断开连接的Oracle数据库的版本以及通过SQL * Plus提供的工具版本。

3.5 SQL * Plus程序语法

您在操作系统上使用SQLPLUS命令提示启动命令行SQL * Plus ：

SQLPLUS [[ Options ] [ Logon | / NOLOG] [ Start ]]

其中： 选项具有以下语法： 

-H [ELP] | -V [ERSION] | [[ -  C [OMPATIBILITY] { x 。 y [。 z ]] [-F [ast]] [-M [ARKUP] markup_option ] [-L [OGON]] [-NOLOGINTIME] [-R [ESTRICT] {1 | 2 | 3}] [-S [ILENT]] ]

和markup_option包括：

  • csv_option
  • html_option

csv_option具有以下语法：

CSV {ON | OFF} [DELIMI [TER] 字符 ] [QUOTE {ON | OFF}]

html_option具有以下语法：

HTML [ON | OFF] [HEAD 文本 ] [BODY 文本 ] [TABLE 文本 ] [ENTMAP {ON | OFF}] [SPOOL {ON | OFF}] [PRE [FORMAT] {ON | OFF}]

Logon具有以下语法：

{ username [/ password ] [@ connect_identifier ] | /} [AS {SYSASM | SYSBACKUP | SYSDBA | SYSDG | SYSOPER | SYSRAC | SYSKM}] [edition = value ]

其中Start具有以下语法：

@ { url | file_name [。 ext ]} [ arg ...]

警告：

以明文形式包含密码存在安全风险。您可以通过省略密码来避免此风险，并仅在系统提示时输入密码。

您可以选择输入登录。如果未指定登录但指定start，则SQL * Plus假定脚本的第一行包含有效登录。如果既未指定启动也未指定登录，则SQL * Plus会提示输入登录信息。

3.5.1.6 MARKUP选项
-M [ARKUP]

您可以使用MARKUP选项通过查询或脚本生成HTML或CSV（字符分隔值）格式的输出。

MARKUP目前支持HTML 4.0过渡和CSV格式。

使用SQLPLUS -MARKUP以HTML或CSV格式生成输出。

注意：

根据您的操作系统，SQL * PLUS命令的完整markup_option子句可能需要包含在引号中。

对于HTML输出，请使用SQLPLUS -MARKUP HTML ON或SQLPLUS -MARKUP HTML ON SPOOL ON来生成独立的网页。SQL * Plus将生成使用<HTML>和<BODY>标记自动封装的完整HTML页面。执行SPOOL OFF或SQL * Plus退出时，将关闭假脱机文件中的HTML标记。

与-MARKUP结合使用时，-SILENT和-RESTRICT命令行选项可能很有用。

您可以使用MARKUP HTML ON在<PRE>标记或HTML表格中生成HTML输出。输出到表使用标准HTML <TABLE>，<TR>和<TD>标记来自动编码查询产生的行和列。当HTML选项设置为ON时，输出到HTML表是默认行为。您可以通过设置PREFORMAT ON来使用HTML <PRE>标记生成输出。

对于CSV输出，请使用SQLPLUS -MARKUP CSV ON以CSV格式生成输出。您可以使用DELIMITER选项指定分隔符。您还可以使用QUOTE OFF输出不带引号的文本。

使用SHOW MARKUP命令查看MARKUP选项的状态。

SQLPLUS -MARKUP命令具有与SET MARKUP命令相同的功能。本节介绍了这些选项。有关SET MARKUP命令的其他信息，请参阅SET命令。 

CSV {ON | OFF}

CSV是必需的MARKUP参数指定要生成的输出类型为CSV。可选的CSV参数ON和OFF指定是否生成CSV输出。默认为OFF。您可以在会话期间根据需要打开和关闭CSV输出。 

HTML {ON | OFF}

HTML是强制性的MARKUP参数指定要生成的输出类型是HTML。可选的HTML参数ON和OFF指定是否生成HTML输出。默认为OFF。

MARKUP HTML ON使用指定的MARKUP选项生成HTML输出。

您可以在会话期间根据需要打开和关闭HTML输出。

头文字

头部 文本选项使您可以指定<HEAD>标记的内容。默认情况下， 文本包含默认的内嵌级联样式表和标题。

如果文本包含空格，则必须用引号括起来。SQL * Plus不会测试此自由文本条目的HTML有效性。您必须确保输入的文本对HTML <HEAD>标记有效。这使您可以灵活地为浏览器或特殊需求定制输出。 

身体文字

身体 文本选项使您可以指定<BODY>标记的属性。默认情况下，没有属性。如果文本包含空格，则必须用引号括起来。SQL * Plus不会测试此自由文本条目的HTML有效性。您必须确保输入的文本对HTML <BODY>标记有效。这使您可以灵活地为浏览器或特殊需求定制输出。 

表格文字

桌子 text选项使您可以输入<TABLE>标记的属性。您可以使用TABLE 文本来设置HTML <TABLE>标记属性，例如BORDER，CELLPADDING，CELLSPACING和WIDTH。默认情况下，<TABLE> WIDTH属性设置为90％，BORDER属性设置为1。

如果文本包含空格，则必须用引号括起来。SQL * Plus不会测试此自由文本条目的HTML有效性。您必须确保输入的文本对HTML <TABLE>标记有效。这使您可以灵活地为浏览器或特殊需求定制输出。 

ENTMAP {ON | OFF}

ENTMAP ON或OFF指定SQL * Plus是否分别用HTML实体＆lt;，＆gt;，＆amp;和＆替换特殊字符<，>，“和＆。ENTMAP默认设置为ON。

您可以在会话期间根据需要打开和关闭ENTMAP。例如，使用ENTMAP OFF，SQL * Plus屏幕输出为：

SQL> PROMPT A> B A> B.

使用ENTMAP ON，SQL * Plus屏幕输出为：

SQL＆GT;提示A> B A＆gt;乙

由于未映射<HEAD>和<BODY>标记中的实体，因此必须确保在MARKUP HEAD和BODY选项中使用有效实体。

如果未映射实体，则Web浏览器可能将数据视为无效HTML，并且所有后续输出可能显示不正确。ENTMAP OFF使用户能够编写自己的HTML标签来自定义输出。

注意：

ENTMAP仅在HTML选项设置为ON时生效。有关在输出中使用实体的更多信息，请参阅COLUMN命令。 

SPOOL {ON | OFF}

SPOOL ON或OFF指定SQL * Plus是否将HTML开始标记<HTML>和<BODY>以及结束标记</ BODY>和</ HTML>写入由每个创建的文件的开头和结尾。 SQL * Plus SPOOL文件名命令。默认为OFF。

您可以在会话期间根据需要打开和关闭SPOOL。

注意：

区分SET MARKUP HTML SPOOL选项和SQLPLUS SPOOL filename命令非常重要。

SET MARKUP HTML SPOOL ON选项可以将<HTML>标记写入假脱机文件。未创建假脱机文件，并且在发出SQLPLUS SPOOL 文件名命令之前，SET MARKUP HTML SPOOL ON选项启用的页眉和页脚标记不会写入假脱机文件 。有关更多信息，请参见SPOOL命令。

发出SPOOL filename命令时，SQL * Plus会将多个HTML标记写入假脱机文件。

当您发出任何SQL * Plus命令：EXIT，SPOOL OFF或SPOOL文件名时，SQL * Plus会附加以下结束标记并关闭文件：

</ BODY> </ HTML>

您可以使用HEAD和BODY选项指定<HEAD>标记内容和<BODY>属性

PRE [FORMAT] {ON | OFF}

PREFORMAT ON或OFF指定SQL * Plus是否将输出写入<PRE>标记或HTML表。默认为OFF，因此默认情况下输出将写入HTML表。您可以在会话期间根据需要打开和关闭PREFORMAT。

注意：

要使用HTML <PRE>标记生成报告输出，必须将PREFORMAT设置为ON。例如：

SQLPLUS -M“HTML ON PREFORMAT ON”

3.5.1.7 MARKUP使用说明

MARKUP HTML ON

当使用MARKUP HTML ON PREFORMAT OFF时，最初用于格式化纸质报告的命令对于用于Web表的报告具有不同的含义：

  • PAGESIZE是HTML表中的行数，而不是行数。每行可以包含多行。每个PAGESIZE行重复TTITLE，BTITLE和列标题。

  • 如果打开包装，LINESIZE可能会对数据产生影响，或者对于非常长的数据可能会产生影响。根据数据大小，可以在单独的行上生成输出，浏览器可以将其解释为空格字符。

  • TTITLE和BTITLE内容输出到三个线位置：左，中，右，最大线宽预设为浏览器窗口的90％。由于Web输出的处理方式，这些元素可能无法按预期与主输出对齐。TTITLE和BTITLE中的实体映射与MARKUP命令中指定的常规ENTMAP设置相同。

  • 如果在输出中使用标题，则SQL * Plus会在标题后面显示输出行的新HTML表。您的浏览器可能会以不同方式格式化每个表的列宽，具体取决于每列中的数据宽度。

  • 当PREFORMAT为ON时，SET COLSEP，RECSEP和UNDERLINE仅在HTML报告中生成输出。

MARKUP CSV ON

使用MARKUP CSV ON时，查询输出将以CSV格式显示。

通过在SQL * Plus命令行中使用-M [ARKUP] CSV ON选项，可以在登录用户会话时启用CSV标记。有关更多信息，请参阅SQL * Plus程序语法 。登录到用户会话后，可以使用SET MARKUP CSV ON命令启用CSV标记。

您可以使用DELIMITER选项指定分隔符。您还可以使用QUOTE OFF输出不带引号的文本。

用户名 [/ 密码 ]

警告：

以明文形式包含密码存在安全风险。您可以通过省略密码来避免此风险，并仅在系统提示时输入密码。
