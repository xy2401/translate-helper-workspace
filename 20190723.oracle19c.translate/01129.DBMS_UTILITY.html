179.1 DBMS_UTILITY已弃用的子程序

Oracle Database 12 c发行版12.2中不推荐使用这些DBMS_UTILITY子程序。

注意：

Oracle建议您不要在新应用程序中使用已弃用的过程。对不推荐使用的功能的支持仅用于向后兼容。

  • GET_PARAMETER_VALUE函数

    直接查询v$_parameter以查找init.ora参数的值。

  • ANALYZE_PART_OBJECT过程

    使用DBMS_STATS收集统计信息。

  • GET_DEPENDENCY程序

    这个子程序没有替代品。但是，您可以直接查询字典视图。

这些子程序没有替代品。

179.2 DBMS_UTILITY安全模型

DBMS_UTILITY以NAME_RESOLVE过程和COMPILE_SCHEMA过程的调用用户的权限运行。这是必要的，以便SQL正常工作。

该软件包不以SYS身份运行。使用DBMS_DDL检查权限。

179.6.5 CANONICALIZE程序

此过程规范化给定的字符串。该过程处理单个保留或关键字（例如“表”），并剥离单个标识符的空格，以便“表”成为TABLE。

句法

DBMS_UTILITY.CANONICALIZE（名称IN VARCHAR2，canon_name OUT VARCHAR2，canon_len IN BINARY_INTEGER）;

参数

表179-9 CANONICALIZE过程参数

参数 描述

名称

要规范化的字符串

canon_name

规范化的字符串

canon_len

规范化的字符串长度（以字节为单位）

返回值

返回第一个canon_len以字节为单位canon_name 。

使用说明

  • 如果name为NULL，则canon_name变为NULL。

  • 如果name不是带点名称，并且name以双引号开头和结尾，请删除两个引号。或者，使用NLS_UPPER转换为大写。请注意，此案例不包含带有特殊字符的名称，例如空格，但不会加倍引用。

  • 如果name是带点名称（例如。“b”.c），则对于组件以双引号开头和结尾的情况下的虚线名称中的每个组件，将不对此组件执行转换。或者，使用NLS_UPPER转换为大写，并将开始和结束双引号应用于此组件的大写形式。在这种情况下，每个规范化组件将在输入位置连接在一起，用“。”分隔。

  • [.b] *之后的任何其他字符都将被忽略。

  • 该程序不处理像'A B'这样的案件。

例子

  • a成为A

  • "a"变成a

  • "a".b变为"a"."B"

  • "a".b,cf变为"a"."B"与",cf"被忽略。

表179-10 COMMA_TO_TABLE过程参数

参数 描述

逗号分隔的'名称'列表列表，其中名称应具有以下格式用于第一次重载： a [. b [. c ]][ @ d ]

参数

表179-18 EXPAND_SQL_TEXT过程参数

参数 描述

例外

表179-19 EXPAND_SQL_TEXT过程异常

例外 描述

ORA-00942

当前用户对input_sql_text递归引用的所有视图和表没有select权限

ORA-24251

input_sql_text不是SELECT语句

ORA-00900

输入无效

ORA-29477

输入LOB大小超过最大大小4GB -1

使用说明

成功完成后，展开和合并的SQL语句文本将复制到output_sql_text。生成的查询文本仅包含对基础表的引用，并且在语义上等同于一些警告：

  • 如果存在从任何视图调用的调用者权限函数，则如果视图所有者与最终编译/运行扩展SQL文本的用户不同，则可以在生成的查询文本中将它们作为不同的用户调用。

  • 如果提供了用于生成动态WHERE子句的函数，则VPD策略会以不同方式扩展。此函数将以不同的方式返回，例如，如果用户标识导致扩展不同。

  • 如果存在对远程对象的引用，则结果不确定。

179.6.15 FORMAT_CALL_STACK函数

此函数格式化当前调用堆栈。这可以在任何存储过程或触发器上使用以访问调用堆栈。这对调试很有用。

句法

DBMS_UTILITY.FORMAT_CALL_STACK RETURN VARCHAR2;

编译指示

pragma restrict_references（format_call_stack，WNDS）;

返回值

这将返回调用堆栈，最多2000个字节。

179.6.16 FORMAT_ERROR_BACKTRACE函数

即使从外部作用域中的异常处理程序调用子程序，此函数也会在引发异常的位置显示调用堆栈。

输出类似于SQLERRM函数的输出，但不受相同的大小限制。

句法

DBMS_UTILITY.FORMAT_ERROR_BACKTRACE RETURN VARCHAR2;

返回值

使用说明

  • 如果对象不存在或已经有效或者是无法验证的对象，则不会引发错误。

  • 如果要验证的对象在指定版本中不是实际的，则子程序会在验证之前自动切换到对象实际的版本。也就是说，对VALIDATE的调用将不会实现指定版本中的对象。

  • INVALIDATE过程使数据库对象无效，并可选择更改其PL / SQL编译器参数设置。要使其无效的对象由其object_id指定。子程序在失效之前自动切换到对象实际的版本。也就是说，对INVALIDATE的调用将不会实现当前版本中的对象。

179.6.38 WAIT_ON_PENDING_DML函数

此函数一直等到所有事务（除了调用者自己的事务）锁定列出的表并在指定的scn之前开始已经提交或回滚。

句法

DBMS_UTILITY.WAIT_ON_PENDING_DML（表IN VARCHAR2，超时IN BINARY_INTEGER，scn IN OUT NUMBER）返回BOOLEAN;

参数

表179-35 WAIT_ON_PENDING_DML函数参数

参数 描述

表

以逗号分隔的一个或多个表名列表。该列表必须对COMMA_TO_TABLE过程有效，并且每个项对NAME_RESOLVE过程有效。名称中不允许使用列说明符或DBLINK（数据库链接）说明符，并且每个名称必须解析为本地数据库中的现有表。

超时

在所有表/事务中总计等待的最大秒数。NULL或负值将导致等待很长时间。

SCN

SCN之前必须开始认为交易与此请求相关。如果值为NULL或未在输入中识别为有意义的scn，则将使用跨所有实例的最新SCN，并将其作为输出设置为传递的参数。如果传入有意义的值，则其值将保留在输出中。

返回值

TRUE ，如果所有相关的交易已提交或回滚， FALSE如果之前提交或正在回滚所有相关交易发生超时
