关于Oracle Database Vault报告

Oracle Database Vault提供了显示数据库中与安全相关的信息的报告。

这些报告还显示自定义Oracle Database Vault审核事件信息。如果启用了统一审核，则报告将捕获统一审核策略的结果。

报告分为两类：

  • Database Vault Reports。 这些报告允许您检查领域，命令规则，因素，因子身份，规则集和安全应用程序角色的配置问题。这些报告还揭示了领域违规，审计结果等。

  • 一般安全报告。 通过这些报告，您可以检查对象权限，数据库帐户系统权限，敏感对象，权限管理，功能强大的数据库帐户和角色，初始化参数，配置文件，帐户密码，安全审核和其他安全漏洞报告的状态。

运行Oracle Database Vault Reports

已被授予相应角色的用户可以从Database Vault Administrator运行Oracle Database Vault报告。

  1. 以已授予DV_OWNER ， DV_ADMIN或DV_SECANALYST角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 在“主页”页面的“报告”下，选择“ 数据库Vault报告” 。
  3. 在左侧，选择所需的报告类别。

    • Database Vault配置问题

    • 数据库保险库执行审计报告

    • Database Vault配置更改

  4. 在“报告”页面中，展开包含该报告的类别。

    例如，要查找“规则集配置问题”报告，必须展开“ 数据库Vault配置问题” 。

  5. 选择报告（例如， 规则集配置问题 ）。

    该报告显示在右侧窗格中。

  6. （可选）使用“ 搜索”字段筛选报告。

    例如，您可以搜索涉及特定规则集的报告事件。“搜索”字段内容因报告而异。

  7. 查看完报告后，单击“ 确定”按钮。

领域审计报告

领域审核报告显示领域保护和领域授权操作生成的审核记录。

您可以使用规则集管理域授权，然后审核规则集处理结果。当对受领域保护的对象执行操作的数据库帐户未被授权执行该操作时，会发生域违规。即使您未指定附加到域的任何规则集，Oracle Database Vault也会审核违规。配置域时，可以将其设置为审计域违规的实例。您可以使用此信息来调查破坏安全性的尝试。

Oracle Database Vault常规安全报告

常规安全报告跟踪信息，例如与PUBLIC相关的对象权限或授予数据库帐户或角色的权限。

  • 对象权限报告
    对象权限报告跟踪受PUBLIC ，直接对象权限和对象依赖性影响的权限。
  • 数据库帐户系统权限报告
    数据库帐户系统特权报告跟踪直接，间接，分层和ANY系统特权等活动。
  • 敏感对象报告
    敏感对象报告跟踪活动，例如对SYS模式对象的EXECUTE特权的授予和对敏感对象的访问。
  • 特权管理 - 摘要报告
    权限管理摘要报告跟踪受助者，所有者和权限的权限分配。
  • 强大的数据库帐户和角色报告
    功能强大的数据库帐户和角色报告跟踪有关已被授予电源权限的用户的信息，例如WITH ADMIN权限。
  • 初始化参数和配置文件报告
    初始化参数和配置文件报告跟踪数据库参数，资源配置文件和系统限制。
  • 数据库帐户密码报告
    数据库帐户密码报告跟踪数据库帐户的默认密码和帐户状态。
  • 安全审计报告：核心数据库审计报告
    核心数据库审计报告列出了数据库审计跟踪记录。
  • 其他安全漏洞报告
    其他安全漏洞报告可跟踪Java策略授予操作系统目录对象等活动时出现的漏洞。

对象依赖关系报告

对象依赖关系报告描述了过程，包，函数，包体和触发器之间数据库中的依赖关系。

该报告包含对没有任何数据库链接创建的视图的依赖性。

此报告可以帮助您使用现有应用程序的最小权限原则开发安全策略。如果数据库对象（如UTL_FILE包）具有授予PUBLIC或某个其他全局角色的权限，则可以使用“对象依赖关系报告”来确定可能依赖于该对象的帐户，并确定该帐户如何使用该对象。要运行报告，请在“报告参数”页面中输入要检查依赖关系的数据库帐户及其可能依赖的对象。

“报告结果”页面显示依赖对象和对象类型以及源对象名称和类型。此报告显示可能敏感对象的使用位置。通过查看多个帐户，您可能会看到可以帮助您开发受限角色的模式。这些受限制的角色可以取代广泛使用的敏感对象上的PUBLIC授权。

如果触发器不是可信数据库应用程序的一部分，则它可以窃取敏感数据（可能来自通过Oracle Label Security或虚拟专用数据库（VPD）保护的表），并将其放入未受保护的表中以供后续查看或导出。
