261 UTL_MATCH

UTL_MATCH包有助于匹配两个记录。这通常用于匹配名称，例如两个名字或两个姓氏。

本章包含以下主题：

  • 概观

  • 安全模型

261.1 UTL_MATCH概述

在确定匹配时， UTL_MATCH可以使用编辑距离算法或Jaro-Winkler算法。

编辑距离，也称为Levenshtein距离（以俄罗斯科学家Vladimir Levenshtein命名，他于1965年设计算法），是两个字符串s1和s2之间相似性的度量。距离是将s1变换为s2所需的插入，删除或替换的数量。

字符串shack le ford和shack el ford之间的编辑距离= 2。

“Jaro-Winkler算法”是计算两个字符串之间的编辑距离的另一种方法。在美国人口普查中开发的这种方法是一种字符串比较器度量，它给出了两个字符串之间部分一致的值。字符串比较器考虑字符串的长度，并部分地解释了在字母数字字符串中产生的典型人为错误。

261.2 UTL_MATCH安全模型

UTL_MATCH包使用UTL_MATCH的权限运行。必须在SYS下创建UTL_MATCH 。此程序包提供的操作是使用SYS权限执行的。

261.3.1 EDIT_DISTANCE函数

此函数计算将string-1转换为string-2所需的插入，删除或替换次数。

句法

UTL_MATCH.EDIT_DISTANCE（在VARCHAR2中为s1，在VARCHAR2中为s2）RETURN PLS_INTEGER;

参数

表261-3 EDIT_DISTANCE函数参数

参数 描述

S1

要转换的字符串

S2

要转换s1的字符串

例子

SELECT UTL_MATCH.EDIT_DISTANCE（'shackleford'，'shackelford'）FROM DUAL; -------------返回2

261.3.2 EDIT_DISTANCE_SIMILARITY函数

此函数计算将string-1转换为string-2所需的插入，删除或子站的数量，并返回两个字符串之间的Edit Distance的Normalized值。

该值通常介于0（不匹配）和100（完美匹配）之间。

句法

UTL_MATCH.EDIT_DISTANCE_SIMILARITY（在VARCHAR2中为s1，在VARCHAR2中为s2）RETURN PLS_INTEGER;

参数

表261-4 EDIT_DISTANCE_SIMILARITY函数参数

参数 描述

S1

要转换的字符串

S2

要转换s1的字符串

例子

SELECT UTL_MATCH.EDIT_DISTANCE_SIMILARITY（'shackleford'，'shackelford'）FROM DUAL; --------------返回82
