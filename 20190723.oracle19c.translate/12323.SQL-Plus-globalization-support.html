11 SQL * Plus全球化支持

全球化支持支持以本机语言存储，处理和检索数据。可以存储在Oracle数据库中的语言由Oracle数据库支持的字符集进行编码。全球化支持可确保数据库实用程序，错误消息，排序顺序以及日期，时间，货币，数字和日历约定适应本机语言和区域设置。

11.1关于在命令行SQL * Plus中配置全球化支持

SQL * Plus通过NLS_LANG环境变量支持多种语言。显示另一个 SQL * Plus中的语言，在启动SQL * Plus之前必须这样做配置：

  • SQL * Plus客户端环境中的NLS_LANG。

  • 安装期间的Oracle数据库。

11.1.1 SQL * Plus客户端

通过设置NLS_LANG环境变量来配置SQL * Plus客户端环境，该变量在启动时由SQL * Plus读取。

11.1.2 Oracle数据库

通过使用所需的字符集创建数据库来配置Oracle数据库环境。

11.2 NLS_LANG环境变量

NLS_LANG环境变量有三个组件，每个组件控制全球化功能的子集。

您的操作系统和键盘必须能够支持您选择的字符集。您可能需要安装其他支持软件。有关NLS_LANG和软件支持的更多信息，请参阅设置全球化支持环境 。

使用NLS参数可以在SQL * Plus客户端上设置特定于语言环境的行为。可以以多种方式指定这些参数，包括作为服务器上的初始化参数。有关控制服务器行为的设置，请参阅NLS数据库参数 。

NLS_LANG具有以下语法：

NLS_LANG = language_territory 。 字符集

其中language指定用于Oracle数据库消息，排序顺序，日期和月份名称的约定。例如，要接收日语消息，请将语言设置为JAPANESE。如果未设置语言 ，则默认为AMERICAN。

区域指定默认日期的约定，以及货币和数字格式。例如，要使用日语区域格式，请将区域设置为JAPAN。如果没有设置领土 ，默认值是从语言值获得，所以设置了美国。

其中，在SQL * Plus命令行中， charset指定SQL * Plus用于数据处理的字符集编码，通常适用于用户终端的编码。可以设置不合逻辑的组合，但不起作用。例如，使用西欧字符集不支持日语，例如：

NLS_LANG = JAPANESE_JAPAN.WE8DEC

但是，Unicode字符集可以支持日语。例如：

NLS_LANG = JAPANESE_JAPAN.UTF8

11.2.1查看NLS_LANG设置

您可以通过输入SELECT命令来查看NLS_LANG设置：

SELECT * FROM NLS_SESSION_PARAMETERS;

NLS_TERRITORY和NLS_LANGUAGE值对应于NLS_LANG变量的语言和区域组件。

您还可以通过查询NLS动态性能视图表V $ NLS_VALID_VALUES来获取NLS_SORT，NLS_LANGUAGE，NLS_TERRITORY和NLS_CHARACTERSET的有效值列表。

11.3设置NLS_LANG

您可以设置NLS_LANG环境变量来控制全球化功能。

示例11-1在Windows上配置SQL * Plus中的日语支持

  1. 确保您已退出当前的SQL * Plus会话。

  2. 从开始>设置>控制面板打开系统。

  3. 单击“高级”选项卡，然后选择“环境变量”。

  4. 创建一个新的环境变量NLS_LANG，其值为Japanese_Japan。JA16SJIS。

  5. 您可能需要重新启动Windows才能使此设置生效。

示例11-2在UNIX上的SQL * Plus中配置日语支持

  1. 确保您已退出当前的SQL * Plus会话。

  2. 根据您使用的UNIX shell，使用set或setenv设置NLS_LANG变量。例如，在csh中，您将输入：

    setenv NLS_LANG Japanese_Japan。UTF8

    要么

    setenv NLS_LANG Japanese_Japan。JA16SJIS

    要么

    setenv NLS_LANG Japanese_Japan。JA16EUC

    UNIX终端的语言环境设置确定NLS_LANG参数的确切值。有关NLS_LANG设置的详细信息，请参阅指定NLS_LANG的值 。

示例11-3在Oracle数据库中配置日语支持

要使用UTF-8字符编码将数据存储在日语字符集中，请确保已使用AL32UTF8字符集创建Oracle数据库。有关使用US7ASCII以外的字符集创建数据库的信息，请参见“Oracle数据库安装指南”。
