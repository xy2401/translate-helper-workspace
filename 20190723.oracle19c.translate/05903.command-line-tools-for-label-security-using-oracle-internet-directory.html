这些命令执行目录中条目的更新，插入和删除，并通过名为olsadmintool的脚本olsadmintool ，您可以从$ORACLE_HOME/bin/olsadmintool调用该脚本。除了olsadmintool ，您还可以使用olsoidsync命令执行引导操作。

注意：

您还可以使用Oracle Enterprise Manager提供的图形用户界面来管理Oracle Label Security。可以在Oracle Enterprise Manager帮助中找到详细的文档。

关于olsadmintool命令

从命令提示符运行olsadmintool命令，可以使用特殊字符执行特定操作。

在olsadmintool命令中，一些参数是可选的，通过在括号内包含这样的参数来指示。最常见的两个示例是[ -b admin context ]和[-p port ] ，表示可以选择指定命令的管理上下文或连接到Oracle Internet Directory的端口。（默认端口是389.）

除了b ， h ， p ， D和w之外的所有参数都需要使用两个破折号（ -- ，没有空格），这些参数前面有一个破折号。双短划线表示需要指定正在使用的名称或参数的完整版本或长版本。如果任何此类名称或参数包含空格，则必须用双引号括起来，例如“这是一个非常长的名称或参数”。

olsadmintool addadmin

olsadmintool addadmin命令将企业用户添加到策略的管理组。

这使用户能够创建，修改或删除指定策略的元数据。您必须提供策略名称和新管理员的DN。该组应仅包含企业用户。

句法

olsadmintool addadmin --polname policy_name --admindn admin_DN [-b admin_context ] -h OID_host [-p port ] -D bind_DN 输入绑定密码： bind_password

例

olsadmintool addadmin --polname defense --admindn“cn = scott，c = us”-h sales_west -D cn = lbacsys 输入绑定密码： bind_password

olsadmintool dropadmin

olsadmintool dropadmin命令从策略的管理组中删除企业用户。

所有olsadmintool命令可以根据需要使用-b标志指定订户/管理上下文。

通过将参数括在括号内来显示指定参数是可选的事实，例如端口或管理上下文。最常见的两个示例是[ -b admin context ]和[-p port ] 。

因为每个命令都必须指定主机，绑定DN和密码，并且如果需要，还可以指定管理上下文， 表C-2使用缩写CON将所有这些连接参数表示为一个组：

olsadmintool参数的摘要

olsadmintool具有适应不同需求类别的参数，例如策略，管理和审计。

表C-2总结了几个类别中的命令。

  • 策略：创建，更改或删除策略或其组件，即级别，组和隔离专区

  • 数据标签：创建，更改或删除它们

  • 管理员和策略创建者：添加或删除它们

  • 用户：从配置文件添加或删除用户

  • 审计选项：设置审计策略的选项

  • 配置文件：创建，列出，描述或删除它们

  • 默认读取或行标签：设置它们

在表C-2和表C-3中 ，列标题仅显示参数，而不是必须在它们之前的关键字。例如， 表C-2显示了policyname和column-name作为createpolicy命令的参数，但没有显示必须在它们之前的关键字（ --name和--colname ）。

    • LABEL_UPDATE

    • CHECK_CONTROL

    • READ_CONTROL

    • WRITE_CONTROL

    • INSERT_CONTROL

    • DELETE_CONTROL

    • UPDATE_CONTROL

    • ALL_CONTROL

    • 无控制

  • OptionsA表示审计选项，即以下条目的逗号分隔组合： SET ， APPLY ， REMOVE或PRIVILEGE 。

olsoidsync命令参考

olsoidsync命令从Oracle Internet Directory中提取策略信息，并填充数据库中的信息（引导）。

您必须提供数据库TNS名称，数据库用户名，数据库用户密码，管理上下文（如果有），Oracle Internet Directory主机名，绑定DN和绑定密码以及可选的Oracle Internet Directory端口号。

句法

olsoidsync --dbconnectstring“ database_connect_string_in_host：port：sid_format ” --dbuser database_user [-c] [-r] [-b admin_context ] -h OID_host [-p port ] -D bind_DN输入数据库密码： database_user_password 输入绑定密码： bind_password

在本规范中：

  • -c删除数据库中的所有现有策略，并使用Oracle Internet Directory中的策略信息对其进行刷新。可选的。
  • -r删除所有策略元数据（不自行删除策略），并使用Oracle Internet Directory中的新元数据刷新策略。可选的。

如果没有这两个开关，该命令将仅从Oracle Internet Directory创建新策略，并将暂停刷新期间遇到的任何错误。

例

olsoidsync --dbconnectstring sales_srvr：1521：ora101 --dbuser lbacsys -c -b“ou = Americas，o = ExampleCorp，c = US”-h sales_srvr -D cn = policycreator输入数据库密码： database_user_password 输入绑定密码： bind_password
