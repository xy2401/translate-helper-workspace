1.194 MAX_STRING_SIZE

MAX_STRING_SIZE控制SQL中VARCHAR2 ， NVARCHAR2和RAW数据类型的最大大小。

属性 描述

参数类型

串

句法

MAX_STRING_SIZE = {标准|扩展}

基本

Oracle RAC

多个实例必须使用相同的值。

脚注1

仅在数据库处于UPGRADE模式时使用ALTER SYSTEM，然后运行utl32k.sql脚本，如本节所述。

STANDARD意味着Oracle数据库12之前的长度的限制的Oracle数据库版本Ç申请（例如，4000个字节为VARCHAR 2和NVARCHAR2 ，和2000个字节为RAW ）。

EXTENDED表示Oracle Database 12 c中引入的32767字节限制适用。

必须将COMPATIBLE初始化参数设置为12.0.0.0或更高才能设置MAX_STRING_SIZE = EXTENDED 。

您可以将MAX_STRING_SIZE的值从STANDARD更改为EXTENDED 。但是，您无法将MAX_STRING_SIZE的值从EXTENDED更改为STANDARD 。

通过设置MAX_STRING_SIZE = EXTENDED ，用户正在采取可能在其数据库中引入应用程序不兼容的显式操作。可以重写不想使用扩展数据类型的应用程序，以便与任一设置兼容;例如，这些应用程序可以使用显式CAST在CREATE TABLE AS SELECT期间修复VARCHAR2表达式的长度。

更改MAX_STRING_SIZE将更新数据库对象并可MAX_STRING_SIZE它们无效，如下所示：

  • 具有虚拟列的表将使用VARCHAR2(4000) ，4000字节NVARCHAR2或RAW(2000)类型的虚拟列的新数据类型元数据进行更新。

    • 如果对其关联虚拟列的更改导致索引键超出索引键长度限制，则功能索引将变得不可用。使用ORA-01450: maximum key length exceeded尝试重建此类索引将失败ORA-01450: maximum key length exceeded 。

  • 如果视图包含VARCHAR2(4000) ，4000字节NVARCHAR2或RAW(2000)类型表达式列，则视图将无效。

  • 物化视图将使用VARCHAR2(4000)数据VARCHAR2(4000) ，4000字节NVARCHAR2和RAW(2000)类型表达式列进行更新

增加非CDB中VARCHAR2，NVARCHAR2和RAW列的最大大小

要增加非CDB中VARCHAR2 ， NVARCHAR2和RAW列的最大大小：

  1. 关闭数据库。

  2. 以UPGRADE模式重新启动数据库。

  3. 将MAX_STRING_SIZE的设置更改为EXTENDED 。

  4. 运行rdbms/admin/utl32k.sql脚本。您必须连接AS SYSDBA才能运行该脚本。

  5. 以NORMAL模式重新启动数据库。

    注意：

    utl32k.sql脚本为需要此视图的视图增加了VARCHAR2 ， NVARCHAR2和RAW列的最大大小。由于写入这些视图的SQL的方式，该脚本不会增加某些视图中VARCHAR2 ， NVARCHAR2和RAW列的最大大小。

  6. 运行rdbms/admin/utlrp.sql脚本以重新编译无效对象。您必须连接AS SYSDBA才能运行该脚本。

增加CDB中VARCHAR2，NVARCHAR2和RAW列的最大大小

要增加CDB中以及CDB中所有PDB中VARCHAR2 ， NVARCHAR2和RAW列的最大大小：

  1. 连接到CDB AS SYSDBA。

  2. 在根目录中，将MAX_STRING_SIZE的设置MAX_STRING_SIZE为EXTENDED ：

    ALTER SESSION SET CONTAINER = CDB $ ROOT; ALTER SYSTEM SET max_string_size = extended SCOPE = SPFILE;

    注意：

    即使在MAX_STRING_SIZE设置为EXTENDED之后，root仍继续使用STANDARD语义。在根中设置MAX_STRING_SIZE为EXTENDED的原因是CDB中的所有PDB都可以从根继承EXTENDED设置。

  3. 关闭CDB。

  4. 在UPGRADE模式下重新启动CDB。

增加PDB中VARCHAR2，NVARCHAR2和RAW列的最大大小

要增加PDB中VARCHAR2 ， NVARCHAR2和RAW列的最大大小：

  1. 关闭PDB。

  2. 在迁移模式下重新打开PDB。

    注意：

    当前容器是PDB时，以下SQL语句可用于在迁移模式下重新打开PDB：

    ALTER PLUGGABLE DATABASE pdb-name OPEN UPGRADE;

  3. 将PDB中MAX_STRING_SIZE的设置更改为EXTENDED 。

  4. 在PDB中运行rdbms/admin/utl32k.sql脚本。必须连接AS SYSDBA才能运行utl32k.sql脚本。

  5. 在NORMAL模式下重新打开PDB。

    注意：

    utl32k.sql脚本为需要此视图的视图增加了VARCHAR2 ， NVARCHAR2和RAW列的最大大小。由于写入这些视图的SQL的方式，该脚本不会增加某些视图中VARCHAR2 ， NVARCHAR2和RAW列的最大大小。

  6. 在PDB中运行rdbms/admin/utlrp.sql脚本以重新编译无效对象。您必须连接AS SYSDBA才能运行该脚本。

增加Oracle RAC数据库中VARCHAR2，NVARCHAR2和RAW列的最大大小

要增加Oracle RAC数据库中VARCHAR2 ， NVARCHAR2和RAW列的最大大小，请执行以下操作：

  1. 关闭所有Oracle RAC数据库实例，除了一个。

  2. 以UPGRADE模式重新启动Oracle RAC数据库实例。

  3. 将MAX_STRING_SIZE的设置更改为EXTENDED 。

  4. 在Oracle RAC数据库实例中运行rdbms/admin/utl32k.sql脚本。您必须连接AS SYSDBA才能运行该脚本。

  5. 以NORMAL模式重新启动所有Oracle RAC数据库实例。

    注意：

    utl32k.sql脚本为需要此视图的视图增加了VARCHAR2 ， NVARCHAR2和RAW列的最大大小。由于写入这些视图的SQL的方式，该脚本不会增加某些视图中VARCHAR2 ， NVARCHAR2和RAW列的最大大小。

  6. 运行rdbms/admin/utlrp.sql脚本以重新编译无效对象。您必须连接AS SYSDBA才能运行该脚本。

增加Oracle Data Guard逻辑备用数据库中VARCHAR2，NVARCHAR2和RAW列的最大大小

要增加Oracle Data Guard逻辑备用数据库中VARCHAR2 ， NVARCHAR2和RAW列的最大大小，请执行以下操作：

  1. 关闭Oracle Data Guard主数据库和逻辑备用数据库。

  2. 以UPGRADE模式重新启动主数据库和逻辑备用数据库。

  3. 在主数据库和逻辑备用数据库MAX_STRING_SIZE的设置更改为EXTENDED 。

  4. 在主数据库和逻辑备用数据库上运行rdbms/admin/utl32k.sql脚本。您必须连接AS SYSDBA才能运行该脚本。

  5. 以NORMAL模式重新启动主数据库和逻辑备用数据库。

    注意：

    utl32k.sql脚本为需要此视图的视图增加了VARCHAR2 ， NVARCHAR2和RAW列的最大大小。由于写入这些视图的SQL的方式，该脚本不会增加某些视图中VARCHAR2 ， NVARCHAR2和RAW列的最大大小。

  6. 在主数据库和逻辑备用数据库上运行rdbms/admin/utlrp.sql脚本以重新编译无效对象。您必须连接AS SYSDBA才能运行该脚本。

  7. 重新启动SQL Apply。
