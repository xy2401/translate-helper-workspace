网关密码加密工具

网关密码加密工具（ tg4pwd ）已被一般功能所取代，该功能现在是异构服务的一部分。请参阅“电源系统（64位）上的IBM AIX的Oracle数据库网关安装和配置指南”中的第15章“安全注意事项”，Linux x86-64，SPARC上的Oracle Solaris（64位）和x86上的Oracle Solaris-有关详细信息，请参见64位（64位）或适用于Microsoft Windows的Oracle Database Gateway安装和配置指南。

部分IPv6支持

Oracle数据库与网关之间存在完整的IPv6支持，但网关与IMS Connect之间尚不支持IPv6。

已知的限制

  • PGAU的已知限制

网关的已知限制

Oracle Database Gateway for APPC具有以下限制。

数字数据和子句不支持多字节字符集

APPC的Oracle数据库网关支持从版本3.4开始的COBOL PIC G数据类型的多字节字符集数据。但是，允许的非数字字符数据（如$，（，），+， - ，。） 仍必须指定DISPLAY数据类型和PIC 9编辑掩码 EBCDIC。非数字字符数据不受MBCS转换的约束。

CICS事务不允许PF密钥仿真

使用Oracle Database Gateway for APPC执行CICS事务时，无法模拟CICS PF密钥。

不支持APPC PIP数据

您无法在此版本的Oracle Database Gateway for APPC中定义和传输APPC PIP数据。

不支持浮点数据类型转换

Oracle Database Gateway for APPC不支持floating数据类型转换。

交易程序负责所有数据压缩和解压缩

适用于APPC的Oracle数据库网关不提供压缩和解压缩工具的出口。网关和事务之间交换的所有数据必须采用未压缩格式。

不支持IBM VS COBOL II编译器

IBM VS COBOL II编译器已被不支持。但是，字符串“ IBMVSCOBOLII ”仍用作编译器名称参数的值，以表示您选择使用的任何COBOL编译器。IBMVSCOBOLII使用值IBMVSCOBOLII ，并且不会对任何特定版本的编译器创建依赖关系。

PGAU的已知限制

当COBOL输入到 PGAU DEFINE DATA语句包含COPY REPLACE子句，只进行第一次替换。
