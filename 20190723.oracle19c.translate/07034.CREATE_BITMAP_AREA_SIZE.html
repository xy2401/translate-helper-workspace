1.56 CREATE_BITMAP_AREA_SIZE

CREATE_BITMAP_AREA_SIZE指定为位图创建分配的内存量（以字节为单位）。

属性 描述

参数类型

整数

默认值

0

可修改

没有

可在PDB中修改

没有

价值范围

依赖于操作系统

基本

没有

此参数仅与包含位图索引的系统相关。

注意：

除非使用共享服务器选项配置实例，否则Oracle不建议使用CREATE_BITMAP_AREA_SIZE参数。Oracle建议您通过设置PGA_AGGREGATE_TARGET来启用SQL工作区的自动调整大小。保留CREATE_BITMAP_AREA_SIZE以实现向后兼容性。

较大的值可以加速索引创建。

基数是列中与表中行数相关的唯一值的数量。如果基数非常小，则可以为此参数设置较小的值。例如，如果基数只有2，则该值可以是千字节而不是兆字节。作为一般规则，基数越高，获得最佳性能所需的内存就越多。
