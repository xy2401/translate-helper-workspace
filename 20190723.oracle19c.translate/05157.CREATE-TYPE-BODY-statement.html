14.14 CREATE TYPE BODY声明

CREATE TYPE BODY定义或实现使用CREATE TYPE语句创建的类型规范中定义的成员方法。

对于未指定call_spec的类型规范中指定的每个方法，必须在类型主体中指定相应的方法主体。

先决条件

ADT的CREATE TYPE规范中的每个成员声明必须在CREATE TYPE或CREATE TYPE BODY语句中具有相应的构造。

要在模式中创建或替换类型主体，您必须具有CREATE TYPE或CREATE ANY TYPE系统特权。要在另一个用户的模式中创建类型，您必须具有CREATE ANY TYPE系统特权。要替换另一个用户的架构中的类型，您必须具有DROP ANY TYPE系统特权。

语义

create_type_body

或替换

如果存在，则重新创建类型主体，然后重新编译它。

在重新定义类型主体之前被授予权限的用户仍然可以访问类型主体而无需重新授予权限。

您可以使用此子句将成员子程序定义添加到使用ALTER TYPE添加的规范中...REPLACE声明。

[版权所有|不可挽回的]

如果未指定此属性，则类型主体将继承类型规范中的EDITIONABLE或NONEDITIONABLE 。如果确实指定了此属性，则它必须与类型规范的属性匹配。

plsql_type_body_source

模式

包含类型主体的模式的名称。默认值：您的架构。

TYPE_NAME

ADT的名称。

subprog_decl_in_type

与类型规范关联的函数或过程子程序的类型。

您必须在每个过程或函数声明的类型规范中定义相应的方法名称和可选参数列表。对于函数，还必须指定返回类型。

map_order_func_declaration

无论声明了多少MEMBER或STATIC方法，都可以声明一个MAP方法或一个ORDER方法。如果声明MAP或ORDER方法，则可以比较SQL中的对象实例。

如果未声明任何一种方法，则只能比较对象实例的相等性或不等式。只有当每对对应的属性相等时，相同类型定义的实例才相等。

MAP会员

声明或实现MAP成员函数，该函数返回给定实例在对象的所有实例的排序中的相对位置。隐式调用MAP方法，并通过将对象实例映射到预定义标量类型的值来指定对象实例的排序。PL / SQL使用排序来评估布尔表达式并执行比较。

如果MAP方法的参数为null，则MAP方法返回null，并且不调用该方法。

类型主体只能包含一个MAP方法，该方法必须是一个函数。MAP函数除了隐式SELF参数之外不能有任何参数。

订单会员

指定ORDER成员函数，该函数将对象的实例作为显式参数和隐式SELF参数并返回负整数，零或正整数，指示隐式SELF参数小于，等于或大于分别比显式参数。

如果ORDER方法的任一参数为null，则ORDER方法返回null并且不调用该方法。

在ORDER BY子句中比较相同ADT定义的实例时，数据库将调用ORDER MEMBER func_decl_in_type 。

对象规范只能包含一个ORDER方法，该方法必须是返回类型为NUMBER的函数。

proc_decl_in_type

程序子程序声明。

constructor_declaration

用户定义的构造函数子程序声明。构造函数的RETURN子句必须是RETURN SELF AS RESULT 。此设置指示构造函数返回的值的最具体类型是传递给构造函数的SELF参数的最具体类型。
