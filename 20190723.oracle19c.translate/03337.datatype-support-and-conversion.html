7针对DRDA的Oracle数据库提供程序中的数据类型支持和转换

考虑Oracle中的数据类型支持以及Oracle和DRDA数据类型之间的转换。

数据类型转换概述

DRDA利用格式化数据对象内容体系结构（FD：OCA）进行数据类型编码。有几种类型与Oracle本机类型没有直接的类比，需要转换。此外，某些Oracle数据类型在FD中没有直接编码支持：OCA。例如，考虑Oracle NUMBER ，它可能包含各种值，包括整数和浮点值 。这种二元性使其无法映射到特定的DRDA类型，从而减少数字值的损失。任何类型的选择都会在极端值范围内损失精度或比例。

Oracle数据库提供程序为DRDA使用了两种数据类型转换：将DRDA元数据描述符转换为Oracle OCI接口类型，以及将Oracle列类型转换为DRDA元数据描述符。对于应用程序编程人员，这些是通过绑定变量的SQL类型或描述的列类型来描述的。请参阅“DRDA数据类型与Oracle数据类型之间的转换”和“将Oracle数据类型转换为DRDA”部分。

映射Oracle NUMBER一般机制在“数据类型等价和重映射”一节中介绍。

数值范围考虑;一般

在Oracle NUMBER ，IEEE浮点，IBM十六进制浮点（HEX浮点，S390或System390浮点）和十进制浮点（ DECFLOAT ）数据类型之间进行转换时，请注意它们具有不同的范围和功能。例如，客户端程序中的IBM HEX FLOAT绑定变量的所有值都适合Oracle NUMBER ，但并非所有Oracle NUMBER值都可以在IBM HEX FLOAT正确返回; DECFLOAT34是更好的选择。

其他一些考虑因素包括：

  • 无穷大。 一些浮点类型支持正负无穷大。

    当无穷大用于不支持它们的数据类型时，使用正无穷大的最高可能数和负无穷大的负数。

  • 浮点。 可以在Oracle中使用BINARY_FLOAT和BINARY_DOUBLE类型定义IEEE FLOAT列。在DB2 z / OS中，浮点类型（ REAL ， FLOAT ， DOUBLE和DOUBLE PRECISION ）是IBM HEX浮点。在DB2 / 400和DB2 LUW中，浮点类型（ REAL ， FLOAT ， DOUBLE和DOUBLE PRECISION ）是IEEE浮点数。

  • 不是数字。 某些数据类型支持非数字（NAN），这是一个特殊值，用于指示未分配任何值，或者计算结果无效或未定义。

COBOL用户的数值范围考虑因素

DRDA数据库为整数类型提供三个选项： SMALLINT （2个二进制字节）， INTEGER （4个二进制字节）和BIGINT （8个二进制字节）。在转换期间，必须根据使用情况而不是DB2 CREATE TABLE定义中使用的类型来定义包含等效值的Oracle列。

DRDA SMALLINT ， INTEGER和BIGINT的实际范围如下：

  • SMALLINT的下限为-32,768 ，上限为32,767

  • INTEGER的下限为-2,147,483,648 ，上限为2,147,483,647

  • BIGINT的下限为-9,223,372,036,854,775,808 ，上限为9,223,372,036,854,775,807

但是，在应用程序级别，保存这些DRDA列值的COBOL变量可以使用固定的十进制数字声明，也可以使用相应DRDA整数数据类型的完整二进制精度声明。

在COBOL中，等效的二进制整数数据类型定义如下：

  • USAGE的BINARY ， COMPUTATIONAL ， COMP ， COMPUTATIONAL-4和COMP-4 ;这些是等价的

  • 对于2字节整数， S9(1-4) PICTURE ，对于4字节整数， S9(5-9)对于8字节整数， S9(10-18) 。

该值通常限于图片中的位数。

例如， PICTURE S9(4) COMP是一个2字节的整数，通常范围为-32,768到+32,767 。但是，生成的COBOL代码仅允许该值的范围为-9,999到+9,999 。当仅使用这些类型的绑定变量访问和更新DRDA SMALLINT ， INTEGER和BIGINT列时，将Oracle中的列定义为NUMBER( n ) ，其中n与上述PICTURE S9( n )定义匹配。

当使用BINARY ， COMPUTATIONAL ， COMP ， COMPUTATIONAL-4和COMP-4 COBOL变量与TRUNC(BIN) COBOL编译器选项时，二进制整数可以达到数据类型的完整范围。无论TRUNC编译器选项是否有效，使用COMPUTATIONAL-5或COMP-5产生相同的效果。在COBOL，C，PL / I或汇编程序中使用全范围的二进制整数进行编程时，将Oracle列定义为NUMBER( n +1) ，其中n与上述PICTURE S9( n )定义匹配。

根据DRDA中的数据类型和用法，以下是推荐的替代Oracle数据类型：

与COBOL COMP一起使用：

  • SMALLINT应转换为Oracle NUMBER(4)

  • INTEGER应转换为Oracle NUMBER(9)

  • BIGINT应转换为Oracle NUMBER(18)

与COBOL COMP，TRUNC（BIN），COMP-5，C，PL / I或汇编程序二进制整数变量一起使用：

  • SMALLINT应转换为Oracle NUMBER(5)

  • INTEGER应转换为Oracle NUMBER(10)

  • BIGINT应转换为Oracle NUMBER(19)

限制Oracle NUMBER

使用全范围的二进制整数值时，建议实现Oracle约束并将值限制为相应数据类型的范围。

例如，DRDA SMALLINT获得一个支持全范围SMALLINT值的等效Oracle NUMBER列，如例7-1所示 。

数据类型等价和重映射

Oracle不提供离散数据库数据类型，例如SMALLINT ， INTEGER或BIGINT DRDA数据类型。在某些情况下，通常要限制列的值范围，可能需要定义具有特定精度或比例的数字列。因此，Oracle提供了一种更灵活的数字数据库数据类型Oracle NUMBER ，它可以通过指定的精度和比例来定义。如果没有为列定义特定的范围限制，Oracle NUMBER可能在同一列中包含整数和小数值。

因此，在从非Oracle数据库迁移数据时，必须考虑适当的数据库数据类型。在迁移期望处理有限范围或形式的数据的应用程序时，这一点尤其重要。

例如，如果应用程序接受特定于NUMBER(5)的数据范围，但该列由数据类型NUMBER定义，则可能会在列中插入不适当或无效的值，并在使用或检索时导致数据问题值。

如果表定义映射到原始非Oracle数据的近似值，则应该没有数据类型兼容性问题。但是，如果必须访问未准确建模的数据，或者查询使用的表达式产生非范围限制数据类型，则可能需要应用更兼容的备用数据类型。

请考虑COUNT(*)表达式导致非范围限制的Oracle NUMBER数据类型。如果应用程序期望使用COUNT的查询结果表示为DRDA INTEGER数据类型，则必须执行以下步骤之一以避免类型不匹配：

  • 更改应用程序以使用Oracle NUMBER

  • 将查询表达式更改为CAST结果为适当的表单

  • 重新映射生成的数据类型表单

通常，修改应用程序既不实际也不可行，重新映射数据类型是唯一可行的解决方案。

Application Server具有有限的功能，可以基于每个表或每个列将Oracle NUMBER数据类型结果转换为更离散的等效DRDA数据类型。当客户端AR无法正确转换Oracle NUMBER的默认映射到DRDA数据类型时，也可以使用此机制。有关所有支持的转换，请参阅“将Oracle数据类型转换为DRDA”。

应用数据类型映射

要应用数据类型映射，必须调用PL / SQL函数。请参阅“ SET_TYPEMAP ”部分。过程SET_TYPEMAP为指定的表和列表达式实现指定的类型转换映射。类型映射对象名称的语法是table_name : column_expression 。通配符*可用于代替表名，以包含具有指定列表达式的所有表。它还可用于指示评估为Oracle NUMBER的指定表的所有列表达式都是类型映射的。

从Oracle NUMBER转换为另一种数据类型的语法是NUMBER= datatype 。有关可用的数据类型名称，请参阅Oracle NUMBER TYPEMAP数据类型名称。

Oracle NUMBER的默认映射是DRDA DECFLOAT(34) 。“在直接使用列的查询中使用TYPEMAP ”表示直接使用列的查询可能会将检索到的列上的重新映射用作DRDA类型INTEGER 。当使用的功能的柱，可能有必要施加typemap为的表达，如“使用描述TYPEMAP在函数”。

在查询中使用TYPEMAP

假设应用程序期望EMPLOYEE_ID值采用DRDA类型INTEGER的格式。

示例7-2在直接使用列的查询中使用TYPEMAP

CREATE TABLE员工（employee_id NUMBER（6），first_name VARCHAR2（20），...）;

此映射强制执行范围限制。为了便于此映射，请为应用程序包ORACLE.MYPACKAGE应用以下typemap条目：

开始dbms_drdaas.set_typemap（'ORACLE'，'MYPACKAGE'，'EMPLOYEES：EMPLOYEE_ID'，'NUMBER = INTEGER'）;结束;

在函数中使用TYPEMAP

对列使用COUNT函数时，如

例7-3在函数中使用TYPEMAP

SELECT COUNT（employee_id）来自员工;

应用以下typemap表达式：

开始dbms_drdaas.set_typemap（'ORACLE'，'MYPACKAGE'，'EMPLOYEES：COUNT（EMPLOYEE_ID）'，'NUMBER = INTEGER'）;结束;
