Oracle Database Security 19c中的更改

Oracle数据库安全指南（适用于Oracle Database 19c）具有新的安全功能。

  • LOB定位器的基于签名的安全性
    从此版本开始，您可以为大对象（LOB）定位器配置基于签名的安全性。
  • 默认用户帐户现在只有架构
    使用Oracle数据库版本18c中的仅模式帐户功能，大多数Oracle数据库提供的模式（用户）现在都会删除其密码，以防止用户对这些帐户进行身份验证。
  • 特权分析现在可在Oracle数据库企业版中使用
    权限分析现在作为Oracle数据库企业版的一部分提供。
  • 能够从仅模式帐户授予或撤消管理权限
    现在可以将管理权限（如SYSOPER和SYSBACKUP授予仅架构（无密码）帐户。
  • 自动支持SASL和非SASL Active Directory连接
    从此版本开始，Microsoft Active Directory连接支持简单身份验证和安全层（SASL）以及传输层安全性（TLS）绑定。
  • 同时支持不同用户的Oracle Native Encryption和SSL身份验证
    在以前的版本中，Oracle数据库阻止同时使用Oracle本机加密（也称为高级网络选项（ANO）加密）和安全套接字层（SSL）身份验证。
  • 支持基于主机名的部分DN匹配以匹配服务器证书
    这种对部分DN匹配的新支持增加了客户端进一步验证服务器证书的能力。
  • 只能审核顶级SQL语句的能力
    统一审计顶级报表功能使您可以审计数据库中的顶级用户（或直接用户）活动，但无需收集间接用户活动审计数据。
  • 改进了统一审计试验的阅读性能
    存储统一审计跟踪记录的AUDSYS.AUD$UNIFIED系统表已经过重新设计，以使用分区修剪来提高读取性能。
  • PDB_GUID作为SYSLOG和Windows事件查看器的审计记录字段名称
    SYSLOG和Windows事件查看器的审计记录字段现在有一个新字段PDB_GUID ，用于标识与统一审计跟踪记录关联的可插入数据库。

LOB定位器的基于签名的安全性

从此版本开始，您可以为大对象（LOB）定位器配置基于签名的安全性。

此功能增强了Oracle数据库LOB的安全性，尤其是在分布式环境中使用LOB数据类型（CLOB和BLOB）的实例时。

LOB签名密钥可以位于多租户PDB中，也可以位于独立的非多租户数据库中。您可以通过执行ALTER DATABASE DICTIONARY ENCRYPT CREDENTIALS SQL语句来启用LOB签名密钥凭证的ALTER DATABASE DICTIONARY ENCRYPT CREDENTIALS ;否则，凭证以模糊格式存储。如果选择以加密格式存储LOB签名密钥，则数据库或PDB必须具有打开的TDE密钥库。

默认用户帐户现在只有架构

使用Oracle数据库版本18c中的仅模式帐户功能，大多数Oracle数据库提供的模式（用户）现在都会删除其密码，以防止用户对这些帐户进行身份验证。

此增强功能不会影响示例模式。示例模式仍然使用其默认密码进行安装。

对于仅作为模式的默认模式，管理员仍然可以使用密码更改这些帐户（如果需要对模式进行身份验证），但Oracle建议之后将模式更改回仅模式帐户。

此功能的好处是管理员不再需要定期轮换这些Oracle数据库提供的模式的密码。此功能还可以降低使用默认密码侵入这些帐户的攻击者的安全风险。

特权分析现在可在Oracle数据库企业版中使用

权限分析现在作为Oracle数据库企业版的一部分提供。

权限分析对用户和应用程序进行动态分析，以查找已使用和未使用的权限和角色。权限分析通过向您显示每个帐户使用和未使用的权限来减少实施最低权限最佳做法的工作。权限分析具有高度的性能，旨在用于测试，开发和生产数据库。

作为此更改的一部分，权限分析文档已从Oracle Database Vault管理员指南转移到Oracle数据库安全指南 。

能够从仅模式帐户授予或撤消管理权限

现在可以将管理权限（如SYSOPER和SYSBACKUP授予仅架构（无密码）帐户。

当前被授予管理权限的现有用户帐户（活动的，很少访问的和未使用的用户）可以更改为仅模式帐户。此增强功能可防止管理员管理这些帐户的密码。

自动支持SASL和非SASL Active Directory连接

从此版本开始，Microsoft Active Directory连接支持简单身份验证和安全层（SASL）以及传输层安全性（TLS）绑定。

对于集中管理的用户，Oracle数据库最初将尝试使用SASL bind连接到Active Directory。如果Active Directory服务器拒绝SASL绑定连接，则Oracle数据库将在没有SASL绑定的情况下再次自动尝试连接，但仍使用TLS进行保护。

Active Directory管理员负责配置Active Directory服务器的连接参数，但不需要配置数据库以匹配此新的Active Directory连接增强功能。数据库将自动从使用SASL调整为不使用SASL绑定。

同时支持不同用户的Oracle Native Encryption和SSL身份验证

在以前的版本中，Oracle数据库阻止同时使用Oracle本机加密（也称为高级网络选项（ANO）加密）和安全套接字层（SSL）身份验证。

例如，如果你设置了SQLNET.ENCRYPTION_CLIENT到客户端上的参数required和SQLNET.ENCRYPTION_SERVER服务器上required ，并且如果使用TCPS监听器，那么ORA-12696 Double Encryption Turned On, login disallowed错误出现。从此版本开始，您可以将新参数SQLNET.IGNORE_ANO_ENCRYPTION_FOR_TCPS设置为TRUE ，以便在使用TCPS客户端与将这两个参数中的任何一个设置为required时发生冲突时忽略SQLNET.ENCRYPTION_CLIENT或SQLNET.ENCRYPTION_SERVER 。

支持基于主机名的部分DN匹配以匹配服务器证书

这种对部分DN匹配的新支持增加了客户端进一步验证服务器证书的能力。

允许部分和完全DN匹配以进行证书验证，可以根据证书的创建方式提供更大的灵活性。

只能审核顶级SQL语句的能力

统一审计顶级报表功能使您可以审计数据库中的顶级用户（或直接用户）活动，但无需收集间接用户活动审计数据。

您可以使用此功能仅审核顶级用户直接发出的事件，而不会产生间接SQL语句的开销。顶级语句是用户直接发出的SQL语句。这些陈述对于安全性和合规性都很重要。从PL / SQL过程或函数中运行的SQL语句不被视为顶级，因此它们可能与审计目的不太相关。

改进了统一审计试验的阅读性能

存储统一审计跟踪记录的AUDSYS.AUD$UNIFIED系统表已经过重新设计，以使用分区修剪来提高读取性能。

这种重新设计需要在AUDSYS.AUD$UNIFIED表中添加一个新列。UNIFIED_AUDIT_TRAIL数据字典视图，使您可以查询AUDSYS.AUD$UNIFIED表审核记录，现在具有EVENT_TIMESTAMP_UTC列以与新的AUDSYS.AUD$UNIFIED表列对应。作为此增强功能的一部分， GV$UNIFIED_AUDIT_TRAIL视图中EVENT_TIMESTAMP列的数据类型已更改TIMESTAMP(6) 。

Oracle建议在查询UNIFIED_AUDIT_TRAIL视图时，在WHERE子句中包含EVENT_TIMESTAMP_UTC列以实现分区修剪。

PDB_GUID作为SYSLOG和Windows事件查看器的审计记录字段名称

SYSLOG和Windows事件查看器的审计记录字段现在有一个新字段PDB_GUID ，用于标识与统一审计跟踪记录关联的可插入数据库。

在多租户数据库部署中，必须在审计跟踪中标识生成统一审计跟踪记录的可插拔数据库。从此版本开始， SYSLOG和Windows事件查看器将有一个新字段PDB_GUID来捕获此信息。数据类型为VARCHAR2 。

Active Directory服务与Oracle数据库的集成

从此版本开始，您可以直接使用Microsoft Active Directory对用户进行身份验证和授权。

通过集中管理的用户（CMU），Oracle数据库用户和角色可以直接映射到Active Directory用户和组，而无需使用Oracle Enterprise User Security（EUS）或其他中间目录服务。EUS未被替换或弃用;如果您只想使用Active Directory对用户进行身份验证和授权，则此新功能是另一个更简单的选项。集中管理的用户旨在扩展为与其他符合LDAP版本3的目录服务一起使用，但Microsoft Active Directory是此版本中唯一支持的服务。

与目录服务的直接集成通过使用企业身份管理体系结构的更简单配置支持更好的安全性。过去，由于困难和复杂性，用户可能已经避免了将数据库与目录服务集成的安全实践。通过直接集成，您可以更轻松地将数据库集成到企业目录服务，从而改善您的安全状况。

能够加密数据字典中的敏感凭证数据

从此版本开始，您可以加密存储在数据字典SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL系统表中的敏感凭据数据。

在以前的版本中，默认情况下，在此版本中，这些表中的数据是模糊的。但是，由于互联网上可用的去混淆算法的兴起，使用更安全的解决方案来保护这类敏感数据非常重要。您可以使用ALTER DATABASE DICTIONARY SQL语句手动加密此数据。

PDB锁定配置文件增强功能

此版本引入了PDB锁定配置文件的几个增强功能。

这些增强功能如下：

  • 您现在可以在应用程序根目录以及CDB根目录中创建PDB锁定配置文件。在以前的版本中，您只能在CDB根目录中创建配置文件。在应用程序容器中创建PDB锁定配置文件的功能使您能够更精细地控制对与应用程序容器关联的应用程序的访问。

  • 您现在可以创建基于另一个PDB锁定配置文件的PDB锁定配置文件，可以是静态基本配置文件，也可以是动态基本配置文件。您可以控制对基本配置文件的后续更改是否反映在使用基本配置文件的新创建的配置文件中。

  • 此版本添加了三个默认的PDB锁定配置文件： PRIVATE_DBAAS ， SAAS和PUBLIC_DBAAS 。这些配置文件有益于云环境

  • 可以使用新的动态数据字典视图V$LOCKDOWN_RULES 。此视图使本地用户可以查看适用于PDB的锁定规则。

此功能有利于在PDB配置中需要强制安全性和隔离的环境。

新的身份验证和认证参数

此版本引入了四个可用于增强数据库安全性的新参数。

新参数如下：

  • ADD_SSLV3_TO_DEFAULT sqlnet.ora参数控制安全套接字层版本3的使用，该版本3易受Padding Oracle On Downgraded Legacy Encryption（POODLE）攻击的影响

  • ADG_ACCOUNT_INFO_TRACKING初始化参数通过使您能够跨所有Data Guard主数据库和备用数据库维护用户帐户信息的单个全局副本来控制Oracle Data Guard备用数据库的登录尝试。

    ACCEPT_MD5_CERTS sqlnet.ora参数启用或禁用MD5算法。

  • ACCEPT_SHA1_CERTS sqlnet.ora参数启用或禁用SHA-1算法。

能够将统一审计跟踪记录写入SYSLOG或Windows事件查看器

从此版本开始，您可以将统一的审计跟踪记录写入UNIX上的SYSLOG或Microsoft Windows上的Windows事件查看器。

在Microsoft Windows上，您可以启用或禁用此行为。在UNIX系统上，您可以指定要使用的SYSLOG工具和统一审计记录的类型日志记录类别，例如是警报还是紧急情况。要配置此行为，可以设置UNIFIED_AUDIT_SYSTEMLOG初始化参数。

从此版本开始，您可以使用Oracle Data Pump在完整或部分导出和导入操作中包含统一审计跟踪。

用户界面没有变化。执行数据库的导出或导入操作时，统一审计跟踪将自动包含在数据泵转储文件中。

与以前的版本一样，此功能使用户必须创建审计记录的转储文件。
