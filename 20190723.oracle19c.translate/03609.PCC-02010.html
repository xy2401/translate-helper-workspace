PCC-02323：找到表达式中使用的typedef名称（期望值）

PCC-02324：找到用作typedef名称的变量名

PCC-02325：非法间接操作

PCC-02326：非法结构参考操作

PCC-02331：未定义的SQL标识符

PCC-02334：SQL标识符未声明为游标

PCC-02336：主变量表达式具有无效类型

PCC-02337：无法为此类型声明多维数组

原因：无法声明具有多个维度的标量的主变量数组。允许多维度的唯一主变量是CHAR和VARCHAR。

PCC-02338：结构包含嵌套的结构或联合

PCC-02339：主变量不能是联合类型

PCC-02340：结构包含一个位字段

原因：主机变量中不允许使用位字段，因为它们对于数据库DML操作毫无意义。

PCC-02341：主机变量具有非法类型

原因：主变量具有未允许的类型（即;枚举，无效等）。

PCC-02344：主机变量数组大小不匹配。使用最小值： number

PCC-02345：使用嵌入式PL / SQL块时必须给出SQLCHECK = SEMANTICS

原因：嵌入式PL / SQL块要求使用命令行标志SQLCHECK = SEMANTICS。

PCC-02346：PL / SQL发现了语义错误

原因：引用了一个不存在的数据库实体，例如表或列名。这是编译时错误，而不是运行时错误。

PCC-02347：PL / SQL发现语法错误

操作：检查PL / SQL用户指南和参考以获取正确的语法。

PCC-02348：EXEC IAF声明中不允许使用指标

PCC-02356：行number ，列number ，文件string警告：\ n

PCC-02365：指标数组大小不能小于主机数组大小

PCC-02466：计划基线长度超过允许的限制

原因：计划基准名称<module_name> _ filename _ filetype _ seqno超出允许的限制。

操作：使用PLAN_PREFIX选项指定计划基准前缀名称。

PCC-02467：计划基准数已超过每个文件的限制

原因：可以创建的计划基准数超过了文件限制。每个文件的允许计划基准为10000。

操作：拆分文件，使每个文件的基线不超过10000个。

PCC-02468：无法创建计划基准

原因：无法创建计划基准，因为存在语法错误或该语句不支持基线。

操作：更正文件中使用的实际SQL中的语法错误，或参阅“Oracle数据库性能调整指南”以获取支持的语句列表。

PCC-02469：必须使用选项PLAN_BASELINE以及选项PLAN_ENABLED

原因：使用了计划基线创建所需的参数（PLAN_ENABLED）而未指定选项PLAN_BASELINE = [<module_name> / yes]。

操作：指定选项PLAN_BASELINE = [<module_name> / yes]。

PCC-02470：必须使用选项PLAN_BASELINE以及选项PLAN_FIXED

原因：使用了计划基线创建所需的参数（PLAN_FIXED）而未指定PLAN_BASELINE = [<module_name> / yes]。

操作：指定选项PLAN_BASELINE = [<module_name> / yes]。

PCC-02471：无法同时使用PLAN_BASELINE和OUTLINE

原因：指定了PLAN_BASELINE和OUTLINE选项。不同时支持这些选项。

操作：指定PLAN_BASELINE选项或OUTLINE选项，但不能同时指定两者。

PCC-02472：模块名称超过30个字节，将在数据库中截断

原因： SQL计划管理（SPM）不支持大于30个字节的模块名称，并且指定的模块名称超出此限制。

操作：缩短模块名称以保持在此限制内。

PCC-02473：程序有EXEC SQL DECLARE TABLE，因此忽略PLAN_RUN选项

原因：由于找到了DECLARE TABLE，因此忽略了PLAN_RUN。在预编译期间未创建基准计划。

操作：手动运行生成的SQL。

PCC-02474：无效的数值

原因：数值无效。

操作：指定正确的数值。
