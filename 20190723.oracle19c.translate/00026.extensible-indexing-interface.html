20.1.2 ODCIIndexAlter（）

使用以下方法之一更改域索引或域索引分区时调用：

  • 改变指数

  • ALTER INDEX PARTITION

  • TRUNCATE TABLE table_name

  • 重命名表

  • ALTER TABLE ... [ADD | TRUNCATE | SPLIT | MERGE]...划分

  • ALTER TABLE RENAME

  • ALTER TABLE RENAME COLUMN

要在创建本地域索引时填充索引分区，请为基表的每个分区调用此方法。

句法

静态函数ODCIIndexAlter（ia ODCIIndexInfo，parms IN OUT VARCHAR2，alter_option NUMBER，env ODCIEnv）返回数字
参数 描述 
IA

包含有关索引和索引列的信息

 
parms（IN）

参数字符串

  • 使用ALTER INDEX PARAMETERS或ALTER INDEX REBUILD ，包含用户指定的参数字符串

  • 使用ALTER INDEX RENAME ，包含域索引的新名称

  • 使用ALTER TABLE RENAME COLUMN ，包含新的域索引列名称

  • 使用ALTER TABLE RENAME or RENAME TABLE ，包含新表名

 
parms（OUT）

参数字符串

仅对ALTER INDEX PARAMETERS或ALTER INDEX REBUILD ;包含要存储在系统目录中的结果字符串

 
alter_option

指定以下选项之一：

  • 如果ALTER INDEX [PARTITION] PARAMETERS AlterIndexNone

  • AlterIndexRename如果ALTER INDEX RENAME [PARTITION]

  • AlterIndexRebuild如果ALTER INDEX REBUILD [PARTITION] [PARAMETERS]

  • 如果ALTER TABLE RENAME COLUMN AlterIndexRenameCol

  • 如果ALTER TABLE RENAME或RENAME TABLE AlterIndexRenameTab

  • 如果ALTER TABLE UPDATE BLOCK REFERENCES AlterIndexUpdBlockRefs

  • 如果域索引是用户管理的，则AlterIndexMigrate如果是ALTER INDEX COMPILE ，但其索引类型是系统管理的

 
ENV

传递给例程的环境句柄

返回

ODCIConst。Success的成功， ODCIConst.Error或ODCIConst.错误ODCIConst.否则Warning 。当调用以重建本地索引分区时，也可能返回ODCIConst.ErrContinue 。

使用说明

  • 此函数应实现为静态类型方法。

  • 可以通过多种方式为域索引调用ALTER INDEX语句。 

    ALTER INDEX index_name PARAMETERS（ parms ）;

    要么

    ALTER INDEX index_name REBUILD PARAMETERS（ parms ）;

    这两种情况的精确行为由实现定义。一种可能性是第一个语句只是根据参数重新组织索引，而第二个语句将从头开始重建。

  • 输入参数字符串的最大长度为1000个字符。parms参数的OUT值可以设置为结果参数字符串，以存储在系统目录中。

  • ALTER INDEX语句还可用于以下列方式重命名域索引：

    ALTER INDEX index_name RENAME TO new_index_name

  • 当创建域索引的表的名称更改时，使用alter_option=AlterIndexRenameTab调用ODCIIndexAlter() ，并将new_table_name传递给parms参数：

    ALTER TABLE table_name RENAME new_table_name

    要么

    RENAME table_name TO new_table_name

  • 当创建域索引的列的名称更改时，使用alter_option=AlterIndexRenameCol调用ODCIIndexAlter() ，并将new_column_name传递给parms参数：

    ALTER TABLE table_name RENAME COLUMN column_name TO new_column_name

  • 如果省略PARALLEL子句，则按顺序重建域索引或本地域索引分区。

  • 如果PARALLEL指定子句，平行度被传递到ODCIIndexAlter()在调用IndexParaDegree的属性ODCIIndexInfo ，以及Parallel的位IndexInfoFlags属性设置。平行度确定如下：

    • 如果指定了PARALLEL DEGREE 度 ，则通过deg 。

    • 如果仅指定了PARALLEL ，则传递一个常量以指示已指定默认的并行度。

  • 如果ODCIIndexAlter例程返回ODCIConst 。Success ，索引有效且可用。如果ODCIIndexAlter()例程返回ODCIConst 。Warning ，索引有效且可用，但会向用户返回警告消息。如果ODCIIndexAlter()返回错误（或异常），则域索引标记为FAILED 。

  • 当正在执行ODCIIndexAlter()例程时，域索引被标记为LOADING 。

  • ODCIIndexAlter()执行的每个SQL语句都被视为一个独立的操作。ODCIIndexCreate()所做的更改不保证是原子的。

  • AlterIndexUpdBlockRefs alter选项仅适用于索引组织表上的域索引。当最终用户执行ALTER INDEX domain_index UPDATE BLOCK REFERENCES ，调用ODCIIndexAlter()并设置AlterIndexUpdBlockRefs位，以使盒式开发人员有机会更新存储在逻辑rowid中的行的块位置的猜测。

  • AlterIndexMigrate alter选项仅适用于将用户管理的域索引迁移到系统管理的域索引。当用户管理的域索引标记为INVALID ，但其索引类型是系统管理的时，您必须进行ALTER INDEX domain_index COMPILE调用以重新验证域索引。这使用alter_option=AlterIndexMigrate调用ODCIIndexAlter()方法，以便有机会将域索引迁移到系统管理的方法。

20.1.4 ODCIIndexCreate（）

在CREATE INDEX...创建域索引时调用CREATE INDEX...INDEXTYPE是......PARAMETERS...用户发出的声明。域索引可以是非分区索引或本地分区域索引。可以在系统或用户管理的方案中创建本地分区域索引。

句法

功能ODCIIndexCreate（ia ODCIIndexInfo，parms VARCHAR2，env ODCIEnv）返回编号
参数 描述 
IA

包含有关索引和索引列的信息

 
PARMS

传入的PARAMETERS字符串不由Oracle解释。参数字符串的最大大小为1,000字符。 

ENV

传递给例程的环境句柄

返回

ODCIConst。Success ， ODCIConst.Error ， ODCIConst.Warning ， ODCIConst.ErrContinue如果在分区级别调用方法以创建本地分区索引，即使出现错误也继续到下一个分区，或ODCIConst.Fatal ，表示清理索引的所有字典条目，并回滚CREATE INDEX操作。返回此状态代码假定盒式磁带代码未创建任何对象（或清除创建的任何对象）。

使用说明

  • 此函数应实现为STATIC类型方法。

  • 创建对象（例如表）以存储索引数据，生成索引数据，并将数据存储在索引数据表中。

  • 此过程应该处理在空表和非空表上创建索引。如果基表不为空，则该过程可以扫描整个表并生成索引数据。

  • 当ODCIIndexCreate()例程正在运行时，域索引标记为LOADING 。

  • ODCIIndexCreate()执行的每个SQL语句都被视为独立操作。ODCIIndexCreate()所做的更改不保证是原子的。

  • 要创建ODCIConst.区域索引，调用ODCIIndexCreate()方法，唯一有效的返回码是ODCIConst.Success ， ODCIConst.Warning ， ODCIConst.Error或ODCIConst.Fatal 。如果操作返回ODCIConst.Fatal ， CREATE INDEX语句由服务器回滚。

  • 在IndexPartition区域索引中， IndexPartition ， TablePartition名称和callProperty应为NULL 。

  • 对于非分区域索引，平行度被传递到ODCIIndexCreate()在调用IndexParaDegree的属性ODCIIndexInfo ，以及Parallel的位IndexInfoFlags被设置。平行度确定如下：

    • 如果指定了PARALLEL DEGREE deg ，则通过deg 。

    • 如果仅指定了PARALLEL ，则传递指示默认并行度指定的常量。

    • 如果完全省略PARALLEL子句， PARALLEL顺序执行操作。

  • 如果ODCIIndexCreate()例程返回ODCIConst 。Success ，索引有效且可用。如果ODCIIndexCreate()例程返回ODCIConst 。Warning ，索引有效且可用，但会向用户返回警告消息。如果ODCIIndexCreate()例程返回ODCIConst 。Error （或异常），域索引标记为FAILED.

  • FAILED域索引允许的唯一操作是DROP INDEX ， TRUNCATE TABLE或ALTER INDEX REBUILD 。

  • 如果在包含REF属性的对象类型列上创建域索引，则在构建索引时不要取消引用REF 。取消引用REF从不同的表实例中获取数据。如果修改了另一个表中的数据，则域索引将变得不正确。请注意，不会通知用户。

  • ODCIIndexCreate()方法被调用两次以创建系统管理的本地域索引，唯一有效的返回码是ODCIConst.Success ， ODCIConst.Warning或ODCIConst.Error 。ODCIConst。第一次调用可以返回Fatal值，导致服务器回滚CREATE INDEX语句。分区数作为参数ODCIIndexInfo.IndexPartitionTotal传入。第一个调用应该创建所有索引存储表。最好对所有索引存储表进行系统分区，以获得本地域索引的好处。也：

    • 这些表必须与基表具有相同数量的分区。如果基表是复合分区的，则域索引存储表应具有与基表相同的总子分区数。

    • 用户应生成包含对象和分区级属性的create table语句。

  • 请注意，对象级别创建例程仅传递对象级别参数字符串。但是，要构造所有分区的存储属性，需要分区级别参数字符串。盒式磁带索引代码必须通过查询字典表上的*_ind_partitions视图来获取它们。不应在此阶段填充系统分区表。用户应等待后续调用ODCIIndexAlter()填充分区。此外，建议用户从索引名称和索引分区名称中派生存储表及其分区的名称。在这种情况下，用户应从*_ind_partitions视图中获取索引分区名称，并构造存储表的分区名称。

  • 在第二个ODCIIndexCreate()调用中，用户可以创建依赖于域索引存储表的对象，例如索引，约束和触发器。这些可以像以前一样通过直接使用SQL回调来创建。但是，对于系统分区存储表，不允许使用以下类型的索引：

    • 非分区索引

    • 全局分区索引

  • 可以使用回调创建序列号和同义词，并假设它们与分区无关。为非分区域索引创建的对象集与本地分区索引的对象集相同，并且在完成表或分区维护操作时不会影响这些对象。删除索引时，用户有责任删除这些对象。

  • 临时使用所需的其他（瞬态）对象可以像以前一样使用回调创建。用户提供的代码负责在创建调用结束时删除它们。

  • 可以创建临时表来保存中间数据。服务器不对这些表执行维护操作。

  • 可以创建外部对象（如文件）以供临时使用。

  • 在调用ODCIIndexCreate()或ODCIIndexAlter()之后剩下的所有表都应该是系统管理的，并且服务器在drop，truncate或分区维护操作期间对它们采取适当的操作。

  • 由于此例程处理多个事物，例如创建非分区索引或创建本地索引，因此必须特别注意对其进行适当的编码。

20.1.6 ODCIIndexDrop（）

使用DROP INDEX语句显式删除域索引时，或者通过DROP TABLE或DROP USER语句隐式删除ODCIIndexDrop()过程。

句法

功能ODCIIndexDrop（ia ODCIIndexInfo，env ODCIEnv）返回编号
参数 描述 
IA

ENV

传递给例程的环境句柄

返回

  • 对于ODCIIndexDrop()区域索引和系统管理本地域索引， ODCIIndexDrop()方法仅调用一次。使用系统管理的方法，不必删除索引存储表。这在调用完成后由内核自动完成。

句法

参数 描述 
IA

指定导出当前正在工作的索引

 
版

导出版本以11.2.0.1.00的形式进行11.2.0.1.00

0 （ FALSE ）：返回的字符串继续当前块。导出仅将返回的字符串写入转储文件，然后再次调用例程。

传递给例程的环境句柄

返回

  • 一个以null结尾的字符串，包含一段不透明的PL / SQL代码块

  • 零长度字符串表示没有更多数据;导出停止调用例程

使用说明

  • 此函数应实现为静态类型方法。

  • 重复调用该例程，直到返回字符串长度为0。如果索引没有使用PL / SQL导出的元数据，它应该在第一次调用时返回一个空字符串。

  • 此例程可用于构建一个或多个匿名PL / SQL代码块，以便通过导入执行。每个返回的块都通过导入独立调用。也就是说，如果一个块在导入时因任何原因失败，则仍然会调用后续块。因此，任何相关代码都应合并到一个块中。单个PL / SQL代码块的大小仅受其BUFFER参数控制的导入读缓冲区大小的限制。

  • 在导入时执行这些PL / SQL块被视为关联域索引创建的一部分。因此，它们的执行取决于索引的底层基表的成功导入以及用户对import的INDEXES=Y/N参数的设置，以及索引的创建。

  • 例程不应该传回打开和关闭PL / SQL代码的各个块的BEGIN / END字符串; export添加这些以标记各个执行单元。

  • 参数version是当前正在执行的导出客户端的版本号。由于导出和导入可用于将数据库降级到先前的功能点版本，因此它还表示您可以在导入时找到的最小服务器版本;它可能更高，但从不低。

  • 盒式磁带开发人员可以使用此信息来确定应将哪些版本的信息写入转储文件。例如，假设当前服务器版本是11.2.0.1.0 ，但输入的导出版本是11.1.0.1.0 。如果盒式磁带的元数据更改格式化这些版本，它将知道以11.1格式将数据写入转储文件，期望导入到11.2系统。

  • 交回出口的字符串中包含的数据必须完全独立于平台。也就是说，它们不应包含可能反映导出平台的字节序性质的二进制信息，这可能与导入平台不同。二进制信息可以作为十六进制字符串传递，并通过RAWTOHEX和HEXTORAW转换。

  • 字符串从导出服务器转换为导出客户端字符集，并将其写入转储文件。在导入时，它们从导出客户端字符集转换为导入客户端字符集，然后从导入客户端字符集转换为导入服务器字符集（在移交UPI接口时）。

  • 必须避免在执行任何PL / SQL块时指定目标模式，因为如果使用import的FROMUSER - > TOUSER模式复制功能，它经常会导致错误。例如，程序原型如：

    程序AQ_CREATE（模式IN VARCHAR2，que_name IN VARCHAR2）...

    应该避免，因为如果您在导入时将架构A重新映射到架构B，则会失败。您可以在导入时假设您已连接到目标架构。

  • 从特定版本导出转储文件必须可导入到所有将来的版本中。这意味着必须始终支持在写入转储文件的匿名PL / SQL块中调用的所有PL / SQL例程。您可能希望对某些版本信息进行编码，以帮助检测何时需要转换。

  • 如果参数CONSISTENT=Y则导出在只读事务中运行。在这种情况下，导出会话不允许写入。因此，此方法不得写入任何数据库状态。

  • 您可以尝试多次导入同一个转储文件，尤其是在使用import的IGNORE=Y参数时。因此，此方法必须生成幂等的PL / SQL代码，或者在执行多次时至少具有确定性。

  • 必须保留数据库对象名称的大小写;也就是说，名为' Foo '和' FOO '的对象是不同的对象。数据库对象名称应括在双引号（“”）中以保留大小写。

错误处理

任何不可恢复的错误都应该引发异常，允许它传播回get_domain_index_metadata ，然后返回导出。这会导致导出终止在转储文件中创建当前索引的DDL并继续下一个索引。

在导入时，执行任何元数据PL / SQL块的失败导致在假设元数据创建是索引创建的组成部分的情况下不创建关联索引。
