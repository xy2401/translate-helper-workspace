DROP DATABASE

目的

注意：

您无法回滚DROP DATABASE语句。

使用DROP DATABASE语句删除数据库。如果要在成功迁移到新主机后删除测试数据库或删除旧数据库，此语句很有用。

也可以看看：

有关删除数据库的详细信息，请参见“ Oracle数据库备份和恢复用户指南”

先决条件

您必须具有SYSDBA系统特权才能发出此语句。数据库必须以独占和受限模式安装，并且必须关闭。

句法

drop_database :: =


说明drop_database.eps的说明

语义

发出此语句时，Oracle数据库将删除数据库并删除控制文件中列出的所有控制文件和数据文件。如果数据库使用服务器参数文件（spfile），则还会删除spfile。

存档日志和备份不会被删除，但您可以使用Recovery Manager（RMAN）将其删除。如果数据库位于原始磁盘上，则此语句不会删除实际的原始磁盘特殊文件。
