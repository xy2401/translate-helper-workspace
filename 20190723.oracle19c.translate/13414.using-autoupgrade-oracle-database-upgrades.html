3使用AutoUpgrade进行Oracle数据库升级

了解如何使用AutoUpgrade简化升级任务。

  • 关于Oracle数据库AutoUpgrade
    AutoUpgrade实用程序可在升级之前识别问题，部署升级，执行升级后操作，并启动升级后的Oracle数据库。
  • 检查Java版本支持
    通常情况。您在要升级的Oracle主目录中启动AutoUpgrade JAR文件。但是，Oracle主目录必须包含Java 8。
  • AutoUpgrade工作流程的示例
    要指导升级，请查看与升级用例匹配的AutoUpgrade工作流示例。
  • 自动升级消息和过程描述术语
    要了解升级检查和操作的进行情况，请了解在实用程序运行时生成的AutoUpgrade实用程序消息。
  • 关于AutoUpgrade处理模式
    四种AutoUpgrade处理模式（分析，修复，部署和升级）表征AutoUpgrade在运行时执行的操作。
  • 了解自动升级工作流程和阶段
    AutoUpgrade工作流程可自动执行典型升级过程的每个步骤。运行的阶段取决于您选择的处理模式。
  • AutoUpgrade命令行参数
    查看AutoUpgrade参数，并选择Oracle数据库升级用例所需的参数和选项。
  • AutoUpgrade实用程序配置文件
    AutoUpgrade配置文件包含执行Oracle数据库升级所需的所有信息。
  • 自动升级配置文件示例
    使用这些示例可以了解如何修改自己的AutoUpgrade配置文件。
  • 自动升级内部设置配置文件
    内部配置设置控制AutoUpgrade的运行方式。
  • 自动升级日志文件结构
    AutoUpgrade实用程序生成包含作业状态和配置文件的日志文件结构
  • 为AutoUpgrade启用完全部署
    要启用保证还原点（GRP）以便可以闪回升级，必须设置存档，并且应完成其他任务以启用AutoUpgrade以完成升级。
  • 如何使用AutoUpgrade控制台
    AutoUpgrade控制台提供一组命令来监视AutoUpgrade作业的进度。运行AutoUpgrade实用程序时，控制台默认启动，并由参数console和noconsole启用或禁用。
  • 如何覆盖默认修正
    您可以使用RUNFIX列条目禁用自动修复，除非禁用修订违反安全性或Oracle策略。
  • 本地配置文件参数修正清单示例
    如果要在升级期间包含或排除单个数据库的特定修订，则可以使用本地配置文件核对表。
  • AutoUpgrade和Microsoft Windows ACL和CLI
    在Microsoft Windows系统上运行AutoUpgrade时，Oracle建议使用访问控制列表（ACL）和命令行界面（CLI）的其他最佳实践。
