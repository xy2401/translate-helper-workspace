你需要安装什么

为了能够开发示例应用程序，您需要安装以下产品和组件：

  • Oracle Database 12c第2版（12.2）

  • J2SE或JDK

  • Apache Maven

  • JDeveloper IDE

  • Web服务器（Tomcat）

  • Oracle Database 12 c第2版（12.2）

    要开发Java Web应用程序，您需要使用HR模式进行Oracle Database 12c第2版（12.2）服务器的工作安装，该模式通常随数据库一起提供。有两种方法可以安装Oracle Database 12c第2版（12.2）。

    Oracle数据库服务

    Oracle数据库云服务提供对Oracle的单个Oracle数据库的访问，为Oracle提供计算能力，物理存储和维护和管理操作工具。您可以在几分钟内访问工作数据库。在Oracle云上创建数据库实例后，您将可以完全访问Oracle数据库中可用的功能和操作，但不包括EECS选项。

    Oracle Cloud使您可以灵活地以适合您需求的成本和级别开始。之后，您可以适应不断变化的要求。

    Oracle数据库Exadata快速云服务（EECS） - 完全托管

    Oracle数据库Exadata云服务（ExaCS）

    Oracle数据库Exadata云计算机（ExaCM）

    在Oracle数据库云服务（DBCS）中创建一个帐户。这样您也可以访问HR模式。

解锁JDBC应用程序的HR模式

该 HR用户帐户最初被锁定，该帐户拥有本指南中用于Java应用程序的示例HR模式。您必须以具有管理权限（ SYS ）的用户身份登录并解锁帐户，然后才能以HR身份登录。

如果数据库是本地安装的，请使用“ 运行SQL命令行”解锁帐户，如下所示：

  1. 要访问“ 运行SQL命令行” ，请从“ 开始”菜单中选择“ 程序” （或“所有程序”），然后选择“ Oracle Database 12c第2版（12.2）” ，再单击“运行SQL命令行” 。以具有DBA权限的用户身份登录，例如：
    >以SYSDBA连接SYS;输入密码： 密码
  2. 运行以下命令：
    > ALTER USER HR帐户解锁;

    要么，

    >人力资源部门确定的用户人力资源;
  3. 测试连接如下：
    > CONNECT HR输入密码： 密码

您应该看到一条消息，指示您已连接到数据库。

注意：

有关使用Oracle Database 12 c第2版（12.2）创建和使用安全密码的信息，请参阅“ Oracle数据库安全指南” 。

此外， HR模式中存在的某些约束和触发器与本指南中创建的Java应用程序的范围不一致。您必须使用以下SQL语句删除这些约束和触发器，如下所示：

DROP TRIGGER HR.UPDATE_JOB_HISTORY; DROP TRIGGER HR.SECURE_EMPLOYEES;从JOB_HISTORY中删除;

本指南中开发的示例应用程序使用 JavaServer Pages（ JSP）技术显示信息并接受用户的输入。要部署这些页面，您需要一个带有Web服务器 servlet和JSP容器等 Apache Tomcat应用程序服务器。

本指南使用名为的嵌入式服务器 JDeveloper中的Oracle WebLogic Server，用于部署JSP页面。如果您选择不安装Oracle JDeveloper，那么可以使用任何Web服务器部署JSP页面应该足够了。

有关这些服务器的更多信息，请参阅特定于供应商的文档。

您可以通过JDeveloper使用maven命令编译或创建.war文件。

2。运行Maven

3。导航到compile / package / clean

使用Maven编译并在任何Java EE容器中运行应用程序

一个。使用Maven编译：

还可以使用Maven命令轻松编译HR Web应用程序。如果您使用的是Oracle maven存储库，请确保将settings.xml文件包含所有必需的详细信息以便于访问。下载HRWebApp_workspace并添加所需代码后，使用以下命令清理，编译和打包源代码。

执行以下命令：

mvn -s settings.xml干净

mvn -s settings.xml编译

mvn -s settings.xml包

湾在任何Java EE容器上部署.war文件

使用maven命令“mvn -s settings.xml package”打包源代码后，war文件将放在“target / JdbcWebSamples.war”文件夹下。将此war文件放在TOMCAT_HOME / webapps /下并启动tomcat服务器。启动tomcat后，使用URL http：// localhost：8080 / JdbcWebSamples /访问HR Web应用程序。

如果添加了登录/注销模块，则使用hradmin或hrstaff用户登录。
