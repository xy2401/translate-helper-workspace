14.15 DROP FUNCTION声明

DROP FUNCTION语句从数据库中删除独立函数。

注意：

不要使用此语句删除作为包的一部分的函数。相反，要么使用“ DROP PACKAGE语句 ”删除整个包，要么使用带有OR REPLACE子句的“ CREATE PACKAGE语句 ”重新定义包而不使用该函数。

先决条件

该函数必须位于您的架构中，或者您必须具有DROP ANY PROCEDURE系统特权。

句法

drop_function :: =


说明drop_function.eps的描述

语义

drop_function

模式

包含该函数的模式的名称。默认值：您的架构。

FUNCTION_NAME

要删除的函数的名称。

数据库使依赖或调用已删除函数的任何本地对象无效。如果随后引用其中一个对象，则数据库会尝试重新编译该对象，如果尚未重新创建已删除的函数，则会返回错误。

如果任何统计类型与该函数关联，则数据库将使用FORCE选项取消关联统计类型，并删除使用统计类型收集的任何用户定义的统计信息。

例

例14-32删除函数

此语句删除示例模式oe的函数SecondMax ，并使依赖于SecondMax所有对象无效：

DROP FUNCTION oe。SecondMax;
