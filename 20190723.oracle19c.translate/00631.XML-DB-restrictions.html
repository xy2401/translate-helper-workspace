B Oracle XML DB限制

此处列出了与Oracle XML DB相关的限制。

  • 某些 XMLType 函数 不支持精简JDBC驱动程序 - XMLType方法transform()仅适用于OCI驱动程序。不是所有的oracle.xdb.瘦JDBC驱动程序支持XMLType函数。如果你不使用oracle.xdb.XMLType类和OCI驱动程序，可以失去性能优势。

  • 不支持NCHAR，NVARCHAR2和NCLOB - Oracle XML DB不支持将SQL数据类型NCHAR ， NVARCHAR2和NCLOB用于以下任何方面：

    • 使用XML模式中的SQLType注释将XML元素或属性映射到这些数据类型

    • 使用SQL / XML函数XMLElement ， XMLAttributes和XMLForest从这些数据类型生成XML数据

    • 在SQL / XML函数XMLQuery和XMLTable ，使用XQuery函数ora:view （ desupported ）， fn:doc和fn:collection包含包含这些数据类型的列的表

    为了处理，存储或生成包含多字节字符的XML数据，Oracle强烈建议您使用AL32UTF8作为数据库字符集。

  • XML标识符长度限制 - Oracle XML DB仅支持最大32767字节或4000字节的XML标识符，具体取决于初始化参数MAX_STRING_SIZE的值。请参见Oracle数据库SQL语言参考 。

  • 存储库文件大小限制 - Oracle XML DB存储库中文件的最大大小为4千兆字节。这意味着对不同类型的文件数据有以下限制：

    • 任何LOB为4千兆字节，这意味着存储在数据库字符集AL32UTF8中的CLOB 2个千兆字节。

    • 用于二进制XML编码数据的4千兆字节，通常意味着在编码之前超过4千兆字节的外部XML数据。

    • 对于以对象方式存储的XML数据不确定。

  • 存储库范围的资源配置文件限制 - 您无法为存储库范围的配置创建超过125个资源配置文件。

  • 递归文件夹删除 - 使用递归删除选项无法删除超过50级嵌套文件夹。

  • 否列级加密 XMLType不支持列级加密- XMLType 。所有XMLType存储模型都支持表空间级加密。

  • 没有 XMLType复合分区 - XMLType表或列不支持复合分区（无论XMLType存储模型如何）。

  • 没有为分层启用的表分区 - 您无法对启用了层次结构的表进行分区。（有关启用层次结构的表的信息，请参阅存储库资源和数据库表安全性 。）

  • 没有针对 XMLType Oracle Real Application Testing（RAT） - XMLType不支持Oracle Real Application Testing（RAT）。

  • 无 数据库链接的 XMLType 访问 - 不支持访问远程XMLType表或列。

  • 某些功能需要Oracle JVM - 通常，Oracle XML DB的行为不依赖于您是否安装了Oracle JVM（Java虚拟机）。但是，如果使用Java servlet，则必须安装Oracle JVM。

  • 编辑视图与 XMLType 不兼容 - 编辑视图与以对象关系存储的XMLType数据不兼容。无法在包含持久对象类型的数据库模式中启用它们。

  • 可传输表空间和数据库整合 - 如果Oracle Database 11g第2版（11.2）中的Oracle XML DB存储库具有现有数据，则无法使用可传输表空间将该数据库直接插入容器数据库（CDB）。相反，将11.2数据库升级到12.1，拔掉它，然后将其插入。

  • 没有 XMLType 的对象关系存储虚拟列XMLType -你无法在虚拟添加XMLType列的对象关系XMLType表或到有一个对象-关系表中XMLType列。

  • XMLType或存储库没有分片 - XMLType数据或Oracle XML DB存储库不支持分片 。

  • 使用堆表而不是索引组织表（IOT） - 为了获得最佳结果，Oracle建议您使用堆表而不是IOT与Oracle XML DB。

  • XML架构URL唯一忽略协议部分 -属性schemaurl用于注册的XML模式必须是唯一的距协议的一部分。在测试唯一性时忽略协议部分（例如， http或https ）。
