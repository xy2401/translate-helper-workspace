7 SQL * Loader和PL / SQL演示

本章介绍如何构建和运行Oracle数据库提供的SQL * Loader和PL / SQL演示程序。它包含以下部分：

  • SQL * Loader演示

  • PL / SQL演示

  • 从64位Oracle数据库PL / SQL调用32位外部过程

注意：

要使用本章中描述的演示，必须安装Oracle Database 19c示例介质中包含的Oracle数据库示例。在创建演示之前，您必须解锁JONES帐户并设置密码。

7.1 SQL * Loader演示

运行ulcase.sh文件以运行SQL * Loader演示。要运行单个演示，请阅读文件中包含的信息以确定如何运行它。

7.2 PL / SQL演示

PL / SQL包含许多演示程序。在使用这些程序之前，必须构建数据库对象并加载示例数据。要构建对象并加载示例数据：

  1. 将目录更改为PL / SQL演示目录：

    $ cd $ ORACLE_HOME / plsql / demo

  2. 启动SQL * Plus，然后输入以下命令：

    $ sqlplus SQL> CONNECT JONES输入密码： 密码

  3. 运行以下命令以构建对象并加载示例数据：

    SQL> @ exampbld.sql SQL> @ examplod.sql

    注意：

    以具有足够权限的任何Oracle用户身份构建演示。以同一个Oracle用户身份运行演示。

PL / SQL内核演示

该软件提供以下PL / SQL内核演示：

  • examp1.sql到examp8.sql

  • examp11.sql到examp14.sql

  • sample1.sql到sample4.sql

  • extproc.sql

要编译和运行examp n .sql或sample n .sql PL / SQL内核演示：

  1. 启动SQL * Plus，然后输入以下命令：

    $ cd $ ORACLE_HOME / plsql / demo $ sqlplus SQL> CONNECT JONES输入密码： 密码

  2. 运行类似于以下命令运行演示，其中demo_name是演示的名称：

    SQL> @ demo_name

    注意：

    您在listener.ora文件中为SID_NAME指定的值必须与您在tnsnames.ora文件中为SID指定的值匹配。

    • 在Oracle Solaris，Linux和HP-UX上：

      SID_LIST_LISTENER =（SID_LIST =（SID_DESC =（SID_NAME = PLSExtProc）（ORACLE_HOME = oracle_home_path ）（ENVS = EXTPROC_DLLS = oracle_home_path /plsql/demo/extproc.so,LD_LIBRARY_PATH = oracle_home_path / plsql / demo）（PROGRAM = extproc）））

  3. 将目录更改为$ORACLE_HOME/plsql/demo 。
  4. 运行以下命令以创建extproc.so共享库，构建所需的数据库对象，并加载示例数据：
    $ make -f demo_plsql.mk extproc.so exampbld examplod

    或者，如果您已经构建了数据库对象并加载了示例数据，则运行以下命令：

    $ make -f demo_plsql.mk extproc.so

    SQL> CONNECT SYSTEM输入密码： system_password SQL> GRANT CREATE LIBRARY TO JONES; SQL> CONNECT JONES输入密码： 密码 SQL> CREATE OR REPLACE LIBRARY demolib IS 2 'oracle_home_path /plsql/demo/extproc.so'; 3 /

    注意：

    CREATE LIBRARY是一个非常高的特权。必须仅向受信任的用户授予此权限。

  6. 要开始演示，请运行以下命令：
    SQL> @extproc

7.3从64位Oracle数据库PL / SQL调用32位外部过程

注意：

本节适用于任何64位Oracle数据库。

从Oracle Database 11g第2版（11.2）开始，64位Oracle数据库安装不再提供extproc32 。因此，如果您需要从64位Oracle数据库运行32位外部过程，则必须通过为您的平台安装相应的32位客户端软件来获取32位extproc 。具体而言，您必须在32位客户端安装中选择自定义安装，然后选择Oracle数据库实用程序和Oracle侦听器。

换句话说，您需要一个单独的Oracle主目录（32位）来运行32位extproc 。可执行文件名称不再是extproc32 ，而只是extproc 。

要在64位Oracle数据库环境中启用32位外部过程，必须为extproc配置32位侦听器，并为extproc listener.ora条目指定Oracle主目录（从32位客户端安装）。
