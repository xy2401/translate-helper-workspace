创建尺寸

目的

使用CREATE DIMENSION语句创建维度 。维度定义列集对之间的父子关系，其中列集的所有列必须来自同一个表。但是，一个列集中的列（称为级别 ）可以来自与另一个列中的列不同的表。优化程序将这些与物化视图的关系用于执行查询重写 。SQL Access Advisor使用这些关系来建议创建特定的物化视图。

注意：

Oracle数据库不会自动验证您在创建维度时声明的关系。要验证hierarchy_clause和CREATE DIMENSION的dimension_join_clause中指定的关系，必须运行DBMS_OLAP 。 VALIDATE_DIMENSION过程。

先决条件

要在自己的架构中创建维度，您必须具有CREATE DIMENSION系统特权。要在另一个用户的架构中创建维度，您必须具有CREATE ANY DIMENSION系统特权。在任何一种情况下，您都必须对维度中引用的任何对象具有READ或SELECT对象特权。

语义

模式

指定将在其中创建维度的架构。如果未指定schema ，则Oracle数据库会在您自己的架构中创建维度。

尺寸

指定维度的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

level_clause

level_clause定义维度中的级别。级别定义维层次结构和属性。

水平

指定级别的名称。

level_table。 level_column

指定级别中的列。您最多可以指定32列。您在此子句中指定的表必须已存在。

空跳时跳过

指定此子句以指示如果指定的级别为NULL ，则跳过级别。此子句允许您通过跳过指定级别的备用路径来保留父子关系的层次结构链。请参阅hierarchy_clause 。

尺寸级别列的限制

维度级别列受以下限制：

  • 级别中的所有列必须来自同一个表。

  • 如果不同级别的列来自不同的表，则必须指定dimension_join_clause 。

  • 您指定的列集必须是此级别的唯一列。

  • 您指定的列不能在任何其他维度中指定。

  • 除非使用SKIP WHEN NULL指定级别，否则每个level_column必须为非null。非空列不需要具有NOT NULL约束。您为其指定SKIP WHEN NULL的列不能具有NOT NULL约束。

hierarchy_clause

hierarchy_clause定义维度中的级别的线性层次结构。每个层次结构在维度中的级别之间形成一系列父子关系。维度中的层次结构彼此独立。他们可能（但不一定）有共同的列。

维度中的每个级别最多应在此子句中指定一次，并且每个级别必须已在level_clause.命名level_clause.

等级制度

指定层次结构的名称。此名称在维度中必须是唯一的。

child_level

指定与父级别具有n ：1关系的级别的名称。level_columns的child_level不能为null，并且每个child_level值唯一地确定下一个命名的parent_level的值。

如果子level_table与父level_table level_table不同，则必须在dimension_join_clause指定它们之间的连接关系。

parent_level

指定级别的名称。

dimension_join_clause

dimension_join_clause允许您为其列包含在多个表中的维度指定内部等值连接关系。仅当层次结构中指定的列不在同一个表中时，才需要此子句。

child_key_column

指定一个或多个与父级别中的列连接兼容的列。

如果未指定每个child_column的模式和表，则会从hierarchy_clause的CHILD OF关系推断出模式和表。如果确实指定了child_key_column的模式和列，则模式和表必须与hierarchy_clause中parent_level子parent_level中的模式和列表相匹配。

parent_level

指定级别的名称。

加入维度的限制

加入尺寸受以下限制：

  • 您只能为同一层次结构中的给定一对级别指定一个dimension_join_clause 。

  • child_key_columns必须为非null，并且父键必须是唯一且非null。您无需定义约束来强制执行这些条件，但如果这些条件不正确，查询可能会返回不正确的结果。

  • 每个子键必须与parent_level表中的键连接。

  • 不允许自我加入。child_key_columns不能与parent_level在同一个表中。

  • 所有子键列必须来自同一个表。

  • 子键列的数量必须与parent_level的列数相匹配，并且列必须是可连接的。

  • 除非父级别包含多个列，否则无法指定多个子键列。

attribute_clause

attribute_clause允许您指定由层次结构级别唯一确定的列。level的列必须都与dependent_columns属于同一个表。该dependent_columns不必已在指定level_clause 。

例如，如果层次结构级别是city ， state和country ，那么city可能决定mayor ， state可能决定governor ， country可能决定president 。

extended_attribute_clause

此子句允许您为一个或多个级别到列关系指定属性名称。使用此子句创建的attribute_clause类型与使用attribute_clause创建的attribute_clause类型没有区别。唯一的区别是此子句允许您为与级别名称不同的属性指定名称。

例子

创建维度：示例

此语句用于在示例模式sh创建customers_dim维度：

CREATE DIMENSION customers_dim LEVEL客户IS（customers.cust_id）LEVEL city IS（customers.cust_city）LEVEL状态IS（customers.cust_state_province）LEVEL国家IS（countries.country_id）LEVEL子区域IS（countries.country_subregion）LEVEL区域IS（countries.country_region） ）HIERARCHY geog_rollup（客户CHILD OF城市儿童国家儿童国家儿童国家儿童地区儿童地区加入关键字（customers.country_id）参考国家）ATTRIBUTE客户决定（cust_first_name，cust_last_name，cust_gender，cust_marital_status，cust_year_of_birth，cust_income_level，cust_credit_limit）属性国家DETERMINES（countries.country_name）;

使用扩展属性创建维度：示例

或者，可以使用extended_attribute_clause而不是attribute_clause ，如以下示例所示：

CREATE DIMENSION customers_dim LEVEL客户IS（customers.cust_id）LEVEL city IS（customers.cust_city）LEVEL状态IS（customers.cust_state_province）LEVEL国家IS（countries.country_id）LEVEL子区域IS（countries.country_subregion）LEVEL区域IS（countries.country_region） ）HIERARCHY geog_rollup（客户儿童城市儿童国家儿童国家儿童国家儿童地区儿童地区加入关键字（customers.country_id）参考国家）ATTRIBUTE customer_info LEVEL客户决定（cust_first_name，cust_last_name，cust_gender，cust_marital_status，cust_year_of_birth，cust_income_level，cust_credit_limit） ATTRIBUTE国家/地区确定（countries.country_name）;

使用NULL列值创建维度：示例

以下示例显示如果其中一个级别列为空并且您希望保留分层链，则如何创建维。该示例使用cust_marital_status列以简化，因为它不是NOT NULL列。如果它有这样的约束，那么在使用SKIP WHEN NULL子句之前必须禁用约束。 

CREATE DIMENSION customers_dim LEVEL customer IS（customers.cust_id）LEVEL status IS（customers.cust_marital_status）SKIP WHEN NULL LEVEL city IS（customers.cust_city）LEVEL state IS（customers.cust_state_province）LEVEL country IS（countries.country_id）LEVEL subregion IS（ countries.country_subregion）跳过空级别区域IS（countries.country_region）HIERARCHY geog_rollup（客户城市儿童城市儿童国家儿童国家儿童区域儿童区域加入密钥（customers.country_id）参考国家）ATTRIBUTE客户决定（cust_first_name， cust_last_name，cust_gender，cust_marital_status，cust_year_of_birth，cust_income_level，cust_credit_limit）ATTRIBUTE country DETERMINES（countries.country_name）;
