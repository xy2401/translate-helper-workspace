7.32 V $ ARCHIVE_DEST_STATUS

V$ARCHIVE_DEST_STATUS显示归档重做日志目标的运行时和配置信息。

此视图中的信息不会在实例关闭时保留。

柱 数据类型 描述

DEST_ID

数

标识日志归档目标参数（1到31）

DEST_NAME

VARCHAR2（256）

记录归档目标参数名称

状态

VARCHAR2（9）

目的地的当前状态：

  • VALID - 已初始化且可用

  • INACTIVE - 没有目的地信息

  • DEFERRED - 用户手动禁用

  • ERROR - 打开或复制期间出错

  • DISABLED - 错误后DISABLED

  • BAD PARAM - 参数有错误

  • ALTERNATE - 目的地处于备用状态

  • FULL - 超出目的地的配额大小

类型

VARCHAR2（14）

档案目的地数据库的类型：

  • LOCAL - 本地到主数据库

  • PHYSICAL - 物理待机

  • CROSS-INSTANCE - 主要实例

  • LOGICAL - 逻辑备用

  • SNAPSHOT - 快照备用数据库

  • DOWNSTREAM - Streams下游捕获数据库

  • FAR SYNC -远东同步实例

DATABASE_MODE

VARCHAR2（15）

归档目标数据库的当前模式：

  • STARTED启动 - 实例已启动，未安装

  • MOUNTED - 已安装

  • MOUNTED-STANDBY - 已安装待机

  • OPEN - 打开读/写

  • OPEN_READ-ONLY - 打开只读

恢复模式

VARCHAR2（23）

档案目的地的当前申请模式：

  • IDLE - 托管恢复未激活

  • MANAGED - 托管恢复处于活动状态。挂载备用数据库时使用此值。在此恢复模式下，用户无法在备用数据库上查询数据。

  • MANAGED WITH QUERY管理 - 在此恢复模式下，托管恢复处于活动状态。在非CDB中，此值在数据库打开时使用，用户可以在备用数据库上查询数据。在CDB中，当CDB中的一个或多个PDB打开时使用此值，并且用户可以在备用数据库上查询数据。

  • MANAGED REAL TIME APPLY - 在此恢复模式下，日志应用服务在写入日志的同时从备用重做日志恢复重做数据，而不是在发生日志切换时从存档的重做日志中恢复重做。在此恢复模式下，用户无法在备用数据库上查询数据。

  • MANAGED REAL TIME APPLY WITH QUERY - 在此恢复模式下，日志应用服务在写入日志的同时从备用重做日志恢复重做数据，而不是在发生日志切换时从存档的重做日志中恢复重做。在非CDB中，此模式在数据库打开时使用，用户可以在备用数据库上查询数据。在CDB中，当CDB中的一个或多个PDB打开时使用此值，并且用户可以在备用数据库上查询数据。

  • LOGICAL REAL TIME APPLY - 实时SQL应用

  • LOGICAL APPLY - SQL Apply

PROTECTION_MODE

VARCHAR2（20）

指示数据库是否受保护：

  • 最大限度的保护

  • 最大可用性

  • 再同步

  • 最大的性能

  • UNPROTECTED

目的地

VARCHAR2（256）

指定要归档重做数据的位置

STANDBY_LOGFILE_COUNT

数

表示在备用数据库上创建的备用重做日志总数

STANDBY_LOGFILE_ACTIVE

数

表示备用数据库上处于活动状态的备用重做日志总数，包含主数据库联机重做日志信息

ARCHIVED_THREAD＃

数

标识在目标上收到的最新归档重做日志的线程号

ARCHIVED_SEQ＃

数

标识在目标上接收的最新归档重做日志的日志序列号

APPLIED_THREAD＃

数

标识在目标上接收的最近应用的重做日志的线程号

APPLIED_SEQ＃

数

标识在目标接收的最近应用的重做日志的日志序列号

错误

VARCHAR2（256）

显示错误文本

SRL

VARCHAR2（3）

指示是否在备用数据库上使用备用重做日志文件（ YES ）或否（ NO ）

DB_UNIQUE_NAME

VARCHAR2（30）

指定使用LOG_ARCHIVE_DEST_ n参数上的DB_UNIQUE_NAME属性定义的当前实例的唯一数据库名称

SYNCHRONIZATION_STATUS

VARCHAR2（22）

此列的可能值如下：

  • CHECK CONFIGURATION - 无法与此目标同步，因为此数据库未处于MAXIMUM PROTECTION或MAXIMUM AVAILABILITY数据保护模式，或者与此目标关联的LOG_ARCHIVE_DEST_ n参数尚未配置SYNC和AFFIRM属性。

  • CHECK STANDBY REDO LOG - 此目的地的备用重做日志配置不正确。

  • CHECK NETWORK - 此数据库的一个或多个实例无法将重做数据发送到此目标。

  • DESTINATION HAS A GAP - 此目标缺少与此数据库同步所需的重做数据。

  • OK - 此目标与此数据库同步。

  • NOT AVAILABLE -同步状态不可用。

另请参阅： Oracle Data Guard概念和管理 ，以获取有关重做传输配置的更多信息

SYNCHRONIZED

VARCHAR2（3）

可能的值是：

  • YES - 此目标与主数据库同步。

  • NO - 目标未与主数据库同步。

  • UNKNOWN - 无法确定此目标的同步状态。

GAP_STATUS

VARCHAR2（24）

重做差距状态：

  • NO GAP - 目的地没有重做差距。

  • LOG SWITCH GAP - 目标尚未收到上一个日志文件中的所有重做。

  • RESOLVABLE GAP - 目标有一个重做间隙，可以通过从此数据库中提取丢失的重做来自动解决。

  • UNRESOLVABLE GAP - 目标有一个重做间隙，无法通过从此数据库中提取丢失的重做来自动解决，并且没有其他目的地可以从中获取重做。

  • LOCALLY UNRESOLVABLE GAP - 目标有一个重做间隙，无法通过从此数据库中提取丢失的重做来自动解决。可以通过从另一个目的地获取丢失的重做来解决间隙。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
