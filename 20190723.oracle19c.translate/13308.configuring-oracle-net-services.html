5.1查找Oracle Net Services配置文件

Oracle Net Services配置文件通常（但不总是）位于$ORACLE_HOME/network/admin目录中。根据文件类型，Oracle Net使用不同的搜索顺序来查找文件。

sqlnet.ora和ldap.ora文件的搜索顺序如下：

  1. 如果设置了此环境变量，则由TNS_ADMIN环境变量指定的目录

  2. $ORACLE_HOME/network/admin目录

cman.ora ， listener.ora和tnsnames.ora文件的搜索顺序如下：

  1. 如果设置了此环境变量，则由TNS_ADMIN环境变量指定的目录
  2. 以下目录之一：

    • 在Oracle Solaris上：

      在/ var /选择/甲骨文

    • 在其他平台上：

      /等等
  3. $ORACLE_HOME/network/admin目录

对于某些系统级配置文件，用户可能在其主目录中存储相应的用户级配置文件。用户级文件中的设置会覆盖系统级文件中的设置。下表列出了系统级配置文件和相应的用户级配置文件：

5.3使用Oracle协议支持

Oracle协议支持是Oracle Net的一个组件。它包括以下内容：

  • IPC协议支持

  • TCP / IP协议支持

  • 支持安全套接字层协议的TCP / IP

具有安全套接字层协议支持的每个IPC，TCP / IP和TCP / IP都具有在Oracle Net Services配置文件和DISPATCHER初始化参数中使用的地址规范。以下部分描述了每种协议支持的地址规范。

5.3.1 IPC协议支持

仅当客户端程序和Oracle数据库安装在同一系统上时，才能使用IPC协议支持。此协议支持需要一个监听器。它被安装并链接到所有客户端工具和oracle可执行文件。

IPC协议支持需要以下格式的地址规范：

（ADDRESS =（PROTOCOL = IPC）（KEY = key ））

下表描述了此地址规范中使用的参数：

参数 描述

协议

要使用的协议。价值是IPC。它不区分大小写。

键

任何名称与同一系统上用于IPC KEY的任何其他名称不同。

以下是IPC协议地址示例：

（ADDRESS =（PROTOCOL = IPC）（KEY = EXTPROC））

5.3.2 TCP / IP协议支持

TCP / IP是用于通过网络进行客户端/服务器通信的标准通信协议。TCP / IP协议支持支持客户端程序和Oracle数据库之间的通信，无论它们是安装在相同还是不同的系统上。如果系统上安装了TCP / IP协议，则会安装TCP / IP协议支持并链接到所有客户端工具和oracle可执行文件。

TCP / IP协议支持需要以下格式的地址规范：

（ADDRESS =（PROTOCOL = TCP）（HOST = 主机名 ）（PORT = 端口 ））

下表描述了此地址规范中使用的参数：

参数 描述

协议

要使用的协议支持。值是TCP。它不区分大小写。

主办

主机名或主机IP地址。

港口

TCP / IP端口。将端口指定为映射到/etc/services文件中端口的数字或别名。Oracle建议值为1521。

以下是TCP / IP协议地址示例：

（ADDRESS =（PROTOCOL = TCP）（HOST = MADRID）（PORT = 1521））

5.3.3具有安全套接字层协议支持的TCP / IP

具有安全套接字层协议支持的TCP / IP使客户端上的Oracle应用程序能够通过TCP / IP和安全套接字层与远程Oracle数据库实例进行通信。

具有安全套接字层协议支持的TCP / IP需要采用以下格式的地址规范：

（ADDRESS =（PROTOCOL = TCPS）（HOST = 主机名 ）（PORT = 端口 ））

下表描述了此地址规范中使用的参数：

参数 描述

协议

要使用的协议。值为TCPS。它不区分大小写。

主办

主机名或主机IP地址。

港口

带有安全套接字层端口的TCP / IP。将端口指定为映射到/etc/services文件中端口的数字或别名。Oracle建议值为2484。

以下是带有安全套接字层协议地址的TCP / IP示例：

（ADDRESS =（PROTOCOL = TCPS）（HOST = MADRID）（PORT = 2484））

5.4使用安全套接字层设置TCP / IP或TCP / IP的侦听器

Oracle建议您在网络上每个Oracle Net Services节点的/etc/services文件中为侦听器保留一个端口。默认端口为1521。该条目列出了侦听器名称和端口号。例如：

oraclelistener 1521 / tcp

在此示例中， oraclelistener是listener.ora文件中定义的listener.ora器的名称。如果要启动多个侦听器，请保留多个端口。

如果您打算使用安全套接字层，那么您应该在/etc/services文件中使用安全套接字层定义TCP / IP端口。Oracle建议值为2484。例如：

oraclelistenerssl 2484 / tcps

在此示例中， oraclelistenerssl是listener.ora文件中定义的listener.ora器的名称。如果要启动多个侦听器，请保留多个端口。
