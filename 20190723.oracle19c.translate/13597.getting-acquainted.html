1.1什么是Oracle预编译器？

Oracle预编译器是一种编程工具，使您可以在高级主机程序中嵌入SQL语句。如图1-1所示，预编译器接受主机程序作为输入，将嵌入式SQL语句转换为标准的Oracle运行时库调用，并生成可以编译，链接和执行的源程序。

1.2为什么使用Oracle预编译器？

该 Oracle预编译器允许您将SQL的灵活性包含在应用程序中。您可以在流行的高级语言（如C和COBOL）中使用SQL。方便易用的界面使您的应用程序可以直接访问Oracle。

与许多应用程序开发工具不同，Oracle预编译器允许您创建高度自定义的应用程序。例如，您可以创建包含最新窗口和鼠标技术的用户界面。您还可以创建在后台运行的应用程序，而无需用户交互。

此外，使用Oracle预编译器，您可以对应用程序进行微调。它们允许密切监视资源使用情况，SQL语句执行和各种运行时指示器。有了这些信息，您可以调整程序参数以获得最佳性能。

1.3为什么要使用SQL？

如果要访问和操作Oracle数据，则需要SQL。交互式使用SQL还是嵌入应用程序中取决于手头的工作。如果作业需要C或COBOL的程序处理能力，或者必须定期完成，请使用嵌入式SQL。

SQL已成为首选的数据库语言，因为它灵活，功能强大且易于学习。作为非过程，它允许您指定您想要的内容，而无需指定如何执行。一些类似英语的语句可以轻松地一次操作一行或多行Oracle数据。

您可以从应用程序执行任何SQL（而不是SQL * Plus）语句。例如，你可以

  • 动态CREATE ， ALTER和DROP数据库表

  • SELECT ， INSERT ， UPDATE和DELETE数据行

  • COMMIT或ROLLBACK事务

在将SQL语句嵌入应用程序之前，您可以使用以交互方式测试它们在SQL * Plus。通常，从交互式SQL切换到嵌入式SQL只需要进行微小的更改。

1.4为什么要使用PL / SQL？

作为SQL的扩展，PL / SQL是一种事务处理语言，支持过程构造，变量声明和强大的错误处理。在同一个PL / SQL块中，您可以使用SQL和所有PL / SQL扩展。

嵌入式PL / SQL的主要优点是性能更好。与SQL不同，PL / SQL在逻辑上启用组SQL语句，并在块中而不是逐个发送到Oracle。这减少了网络流量和处理开销。

有关PL / SQL的更多信息（包括如何将其嵌入应用程序），请参阅运行Oracle预编译器 。

1.6 Oracle预编译器是否符合行业标准？

SQL已成为关系数据库管理系统的标准语言。本节介绍Oracle预编译器如何符合以下组织建立的最新SQL标准：

  • 美国国家标准协会（ANSI）

  • 国际标准组织（ISO）

  • 美国国家标准与技术研究院（ NIST）

这些组织采用了以下出版物中定义的SQL：

  • ANSI文档ANSI X3.135-1992， 数据库语言SQL

  • 国际标准ISO / IEC 9075：1992， 数据库语言SQL

  • ANSI文档ANSI X3.168-1992， 数据库语言嵌入式SQL

  • NIST联邦信息处理标准FIPS PUB 127-2， 数据库语言SQL

1.6.1要求

ANSI X3.135-1992（非正式地称为 SQL92）指定“符合要求的SQL语言”，并且为了允许分阶段实现，定义了三种语言级别：

  • 完整的SQL

  • 中级SQL（完整SQL的一个子集）

  • Entry SQL（中级SQL的一个子集）

符合标准的SQL实现必须至少支持Entry SQL。

ANSI X3.168-1992规定了在以标准编程语言（如COBOL，FORTRAN，Pascal或PL / I）编写的应用程序中嵌入SQL语句的语法和语义。

ISO / IEC 9075-1992完全采用ANSI标准。

FIPS PUB 127-2适用于为联邦使用而获得的RDBMS软件，也采用ANSI标准。此外，它还指定了数据库结构的最小大小参数，并且需要“FIPS Flagger”来标识ANSI扩展。

1.6.2合规性

Oracle预编译器100％符合ANSI，ISO和NIST标准。根据需要，它们支持Entry SQL并提供FIPS Flagger。

1.6.3 FIPS Flagger

根据FIPS PUB 127-1，“提供本标准未规定的附加设施的实现还应提供标记不合格SQL语言或符合可能以不合规方式处理的SQL语言的选项。”为了满足此要求，Oracle预编译器提供了标记ANSI扩展的FIPS Flagger 。扩展名是违反ANSI格式或语法规则的任何SQL元素，但权限实施规则除外。有关标准SQL的Oracle扩展的列表，请参阅Oracle数据库SQL语言参考 。

您可以使用FIPS Flagger识别

  • 如果将应用程序移动到符合要求的环境，则可能必须修改不符合的SQL元素

  • 符合在另一个处理环境中可能表现不同的SQL元素

因此，FIPS Flagger可帮助您开发便携式应用程序。

1.6.4 FIPS选项

名为FIPS的选项控制FIPS Flagger。要启用FIPS Flagger，请在内联或命令行中指定FIPS=YES 。有关命令行选项FIPS详细信息，请参阅“ FIPS ” 。

1.6.5认证

NIST使用SQL Test Suite测试了Oracle预编译器的ANSI Entry SQL合规性，该测试套件包含近300个测试程序。具体而言，程序测试是否符合COBOL和FORTRAN嵌入式SQL标准。因此，Oracle预编译器通过了100％ANSI认证。
