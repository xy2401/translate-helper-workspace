关于Oracle Grid Infrastructure升级和降级

您可以将Oracle Grid Infrastructure升级或降级到受支持的版本。

您可以通过以下任何方式升级Oracle Grid Infrastructure：

  • 滚动升级涉及升级单个节点而不停止群集中其他节点上的Oracle Grid Infrastructure。

  • 非滚动升级，包括关闭除一个节点之外的所有节点。当根脚本停止旧的Oracle Clusterware堆栈并在您启动升级的节点上启动新的Oracle Clusterware堆栈时，会发生完全的群集中断。升级完成后，将在所有节点上启动新的Oracle Clusterware。

请注意，当一个或多个节点正在升级时，某些服务将被禁用。所有升级都是异地升级，这意味着软件二进制文件放置在与先前版本使用的Grid主目录不同的Grid主目录中。

您可以从Oracle Grid Infrastructure 19c降级到Oracle Grid Infrastructure 18c，Oracle Grid Infrastructure 12c第2版（12.2），Oracle Grid Infrastructure 12c第1版（12.1）和Oracle Grid Infrastructure 11g第2版（11.2）。请注意，如果降级到先前版本，则群集必须符合该先前版本的配置要求，并且群集可用的功能仅包含Oracle Clusterware和Oracle ASM先前版本可用的功能。

您可以使用Oracle ASM Configuration Assistant（ASMCA）对Oracle ASM实例执行非现场升级。除了使用图形用户界面运行ASMCA之外，您还可以在非交互（静默）模式下运行ASMCA。

注意：

在尝试使用群集备份文件之前，必须完成升级。您无法将备份用于尚未完成升级的群集。
