Oracle Net Services和Oracle RAC的性能特征

Oracle RAC数据库提供了连接负载平衡和故障转移的重要优势。

  • 负载平衡连接到Oracle RAC数据库
    服务通过向本地侦听器和SCAN侦听器注册其工作负载或当前正在处理的工作量来协调其会话。
  • Oracle RAC数据库的连接故障转移
    Oracle RAC通过在多个节点上配置多个侦听器来管理同一数据库服务的客户端连接请求，从而为节点VIP地址提供故障转移。
  • Oracle RAC数据库的共享服务器配置
    独立Oracle数据库通过在共享服务器调度程序进程之间分配连接来执行负载平衡。默认情况下，DBCA使用专用服务器（而非共享服务器）配置Oracle RAC数据库。
