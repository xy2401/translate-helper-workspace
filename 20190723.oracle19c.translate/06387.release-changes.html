Oracle Database 12c第2版（12.2.0.1）中的更改

以下是Database Database 12c第2版（12.2.0.1）的Oracle数据库备份和恢复参考中的更改。

新功能

  • 在可插拔数据库（PDB）上执行闪回到指定的时间点

    您可以执行闪回数据库操作以将单个PDB回滚到先前的时间点。这使您可以反转对单个PDB所做的不需要的更改，而不会影响其余PDB的操作。

    请参见FLASHBACK DATABASE 。

  • 验证和恢复未记录的数据块

    在Data Guard环境中，您可以通过从主数据库或物理备用数据库中提取数据块来执行非记录数据块的恢复。您还可以执行验证以确定未记录的块范围中的数据块是否仍然无效。

    请参阅RECOVER和VALIDATE 。

  • 表恢复的增强功能

    您可以使用REMAP TABLE子句将表或表分区恢复为与这些对象最初存在的模式不同的模式。

    见RECOVER 。

  • DUPLICATE命令的增强功能

    使用DUPLICATE命令的FOR FARSYNC子句通过复制目标数据库来创建Oracle Data Guard远程同步实例。

    重复

    要复制使用透明加密和基于密码的密钥库的数据库，使用SET DECRYPTION WALLET OPEN命令将用于打开密钥库的密码提供给辅助实例。

    组

  • 跨平台传输的增强功能

    RMAN支持将PDB跨平台传输到目标CDB。您可以通过网络执行表空间的跨平台传输。加密的表空间也可以传输到不同的平台。

    请参阅BACKUP ， RECOVER ， RESTORE和foreignFileSpec 。

  • 稀疏数据库的备份和恢复

    RMAN使您能够在COMPATIBLE初始化参数设置为12.2或更高的情况下备份，还原，恢复和复制稀疏数据文件，表空间，CDB，PDB和整个数据库。

    请参阅BACKUP ， DUPLICATE ， RECOVER ， RESTORE

Oracle Database 12 c第1版（12.1.0.2）中的更改

以下是Oracle Database 12 c第1版（12.1.0.2）的备份和恢复参考中的更改。

新功能

  • 用于RMAN虚拟专用目录的Oracle虚拟专用数据库（VPD）

    使用VPD创建和管理RMAN目录。这为恢复目录提供了更好的性能和可伸缩性，尤其是对于大量虚拟专用目录。实施细节对用户是透明的。

    看到：

    • “ 创建目录 ”

    • “ DROP CATALOG ”

    • “ 升级目录 ”

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle Database 12 c第1版（12.1.0.1）的备份和恢复参考中的更改。

新功能

  • 支持多租户容器数据库和可插拔数据库

    RMAN支持多数据容器数据库（CDB）和可插拔数据库（PDB）的备份和恢复，这些数据库是在Oracle Database 12 c中引入的。该支持包括备份CDB和PDB，以及执行整个CDB或单个PDB的完整和时间点恢复。

    看到：

    • “ 备份 ”

    • “ 恢复 ”

    • “ 恢复 ”

  • 跨平台备份和还原增强功能

    RMAN使您可以使用完整和增量备份集跨平台传输数据。增量备份可用于减少跨平台传输表空间时所需的应用程序停机时间。

    看到：

    • “ 备份 ”

    • “ 恢复 ”

    • “ 恢复 ”

  • 恢复表和表分区

    RMAN可以将表和表分区从先前创建的RMAN备份恢复到指定的时间点。

    请参阅“ RECOVER ” 。

  • 通过网络恢复和恢复文件

    RMAN使您可以使用物理备用数据库中的备份集来恢复数据库，数据文件，表空间或控制文件。RMAN通过网络将备份集传输到目标主机。当您要同步备用数据库和主数据库时，这在Data Guard环境中很有用。

    看到：

    • “ 恢复 ”

    • “ 恢复 ”

  • 使用存储快照优化改进了对第三方快照技术的支持

    通过Storage Snapshot Optimization，您可以使用存储快照技术备份和恢复Oracle数据库，而无需将其置于BACKUP模式。快照技术必须符合某些条件。

    看到：

    • “ 恢复 ”

    • “ 使用存储快照进行恢复 ”

  • 增量和多节点备份改进

    RMAN提供的多节点技术允许通过多个通道并行备份和恢复非常大的文件，现在可以应用于增量备份和映像副本。

    看到：

    • “ 备份 ”

    • “ 恢复 ”

  • SYSBACKUP权限

    SYSBACKUP管理权限包含备份和恢复所需的权限，包括连接到已关闭数据库的权限。系统管理员可以向执行备份和恢复的用户授予SYSBACKUP而不是SYSDBA ，从而减少SYSDBA权限的扩散。与SYSDBA ， SYSBACKUP不包括数据访问权限，例如SELECT ANY TABLE 。

    看到：

    • “ 连接 ”

    • “ connectStringSpec ”

  • 活动数据库复制增强功能

    RMAN现在可以使用备份集来执行活动数据库复制。当分配了足够的辅助通道时，辅助实例连接到目标实例并通过网络检索备份集，从而减少目标实例上的处理负载。在复制过程中可以使用未使用的块压缩，从而减少通过网络传输的备份的大小。您还可以在执行活动数据库复制时加密备份并使用多节备份

    请参阅“ DUPLICATE ” 。

  • SQL接口增强功能

    您现在可以在RMAN中发出大多数SQL命令，而不必使用带有SQL关键字的命令，并将SQL命令括在引号中。当SQL命令本身需要引号时，这极大地简化了语法。对于RMAN和SQL中存在的一些命令，您可以指定SQL关键字以消除歧义。

    SQL ALTER命令替换RMAN命令。新的RMAN DESCRIBE命令提供SQL * Plus DESCRIBE命令的功能。

    看到：

    • “ DESCRIBE ”

    • “ SQL ”

  • DUPLICATE增强功能

    您可以使用NOOPEN子句指定在创建后不能使用RESETLOGS打开重复数据库。如果要更改重复数据库的初始化参数，或者打开重复数据库，可能会在重复数据库中启动与原始数据库冲突的服务，则可能不希望打开重复数据库。

    请参阅“ DUPLICATE ” 。
