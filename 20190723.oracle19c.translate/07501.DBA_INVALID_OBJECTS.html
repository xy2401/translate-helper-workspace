5.120 DBA_INVALID_OBJECTS

DBA_INVALID_OBJECTS描述数据库中的所有无效对象。您可以使用此视图在数据库升级之前和之后标识无效对象。

此视图消除了旧版本的对象类型。它只包含对象类型，如果是最新版本。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

对象的所有者

OBJECT_NAME

VARCHAR2（128）

对象的名称

SUBOBJECT_NAME

VARCHAR2（128）

子对象的名称（例如，分区）

OBJECT_ID

数

对象的字典对象编号。

DATA_OBJECT_ID

数

包含对象的段的字典对象编号。

注意： OBJECT_ID和DATA_OBJECT_ID显示数据字典元数据。不要将这些数字与Oracle数据库分配给系统中对象表中的行对象的唯一16字节对象标识符（ 对象ID ）混淆。

OBJECT_TYPE

VARCHAR2（23）

对象的类型（例如TABLE ， INDEX ）。

仅当该类型无效时，才会显示该类型的当前版本。换句话说，如果该类型的先前版本无效但该类型的最新版本有效，则它将不在此列表中。

CREATED

日期

用于创建对象的时间戳

LAST_DDL_TIME

日期

最后一次修改由DDL语句产生的对象和依赖对象的时间戳（包括授权和撤销）

TIMESTAMP

VARCHAR2（19）

对象规范的时间戳（字符数据）

状态

VARCHAR2（7）

对象的状态：

  • 有效

  • 无效

  • N / A

临时

VARCHAR2（1）

指示对象是否是临时的（当前会话只能看到它放在此对象中的数据）（ Y ）或不是（ N ）

GENERATED

VARCHAR2（1）

指示此对象的名称是否是系统生成的（ Y ）或不是（ N ）

二级

VARCHAR2（1）

指示这是否是Oracle数据盒（ Y ）的ODCIIndexCreate方法创建的辅助对象（ N ）

NAMESPACE

数

对象的命名空间

EDITION_NAME

VARCHAR2（128）

对象实际的版本名称

共享

VARCHAR2（13）

价值观：

  • METADATA LINK - 如果对象是元数据链接的，或者是根目录中对象的元数据链接

  • DATA LINK - 如果对象是数据链接的，或者是根目录中对象的数据链接

  • NONE - 如果以上都不适用

EDITIONABLE

VARCHAR2（1）

价值观：

  • Y -对于对象标记EDITIONABLE

  • N - 对于标记为NONEDITIONABLE对象

  • NULL - 对于类型不可在数据库中编辑的对象

ORACLE_MAINTAINED

VARCHAR2（1）

表示对象是由Oracle提供的脚本（例如catalog.sql还是catproc.sql）创建并维护的。除非运行Oracle提供的脚本，否则不得以任何方式更改此列具有值Y的对象。

应用

VARCHAR2（1）

指示对象是否是应用程序公共对象（ Y ）（ N ）

DEFAULT_COLLATION

VARCHAR2（100）

对象的默认排序规则

复制

VARCHAR2（1）

指示此对象是否在此分片（ Y ）上重复（ N ）

分片

VARCHAR2（1）

指示此对象是否为分片（ Y ）或不是（ N ）

创建对象的应用程序的ID

CREATED_VSNID

数

MODIFIED_APPID

数
