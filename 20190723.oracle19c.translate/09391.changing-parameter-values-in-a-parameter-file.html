1.4更改参数文件中的参数值

您可以通过多种方式更改参数文件中参数的值。

  • 通过编辑初始化参数文件

    在大多数情况下，新值将在下次启动数据库实例时生效。

  • 通过发布ALTER SYSTEM SET ...SCOPE=SPFILE语句用于更新服务器参数文件

  • 通过发出ALTER SYSTEM RESET语句来清除初始化参数值。

    也可以看看：

    有关使用ALTER SYSTEM RESET语句清除初始化参数值的详细信息，请ALTER SYSTEM RESET “Oracle数据库管理员指南”

1.4.2可修改的参数

在实例运行时，您可以使用ALTER SESSION或ALTER SYSTEM语句修改某些初始化参数。

使用以下语法修改初始化参数：

ALTER SESSION SET parameter_name = value ALTER SYSTEM SET parameter_name = value [DEFERRED]

每当使用ALTER SYSTEM语句修改参数时，Oracle数据库都会记录在警报日志中修改参数的语句。

ALTER SESSION语句在调用语句的会话期间更改指定参数的值。对于实例中的其他会话，参数的值不会更改。可以使用ALTER SESSION更改以下初始化参数的值：

1.4.3显示当前参数值

您可以使用SQL * Plus SHOW PARAMETERS语句查看初始化参数的当前设置。

例如：

 SQL> SHOW PARAMETERS

此语句按字母顺序显示所有参数及其当前值。

输入以下文本字符串以显示名称中包含BLOCK所有参数：

SQL> SHOW PARAMETERS BLOCK

您可以使用SPOOL命令将输出写入文件。

1.4.4参数文件中不应指定的参数

您不应在参数文件中指定这些类型的参数：

  • 除非Oracle指示解决问题，否则您永远不会更改的参数

  • 派生参数，通常不需要更改，因为它们的值由Oracle数据库服务器自动计算

1.4.5参数设置错误时

如果参数设置不正确，则不同参数的效果可能不同，或者基于参数设置的低或高的不同。

某些参数的最小设置低于该设置，Oracle实例将无法启动。对于其他参数，将值设置得太低或太高都可能导致Oracle执行不良，但仍会运行。此外，Oracle可能会将可接受范围之外的某些值转换为可用级别。

如果参数值太低或太高，或者您已达到某个资源的最大值，则Oracle会返回错误。通常，您可以等待一会儿，并在系统不忙时重试该操作。如果重复出现消息，则应关闭实例，调整相关参数，然后重新启动实例。
