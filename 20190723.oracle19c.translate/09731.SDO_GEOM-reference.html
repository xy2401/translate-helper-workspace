26 SDO_GEOM包（几何）

MDSIS.SDO_GEOM包中包含用于处理几何对象的子程序。

本章包含SDO_GEOM包中与几何相关的PL / SQL子程序的描述，可以将其分为以下类别：

  • 两个对象之间的关系（真/假）：RELATE，WITHIN_DISTANCE

  • 验证：VALIDATE_GEOMETRY_WITH_CONTEXT，VALIDATE_LAYER_WITH_CONTEXT，SDO_SELF_UNION

  • 单对象操作：SDO_ALPHA_SHAPE，SDO_ARC_DENSIFY，SDO_AREA，SDO_BUFFER，SDO_CENTROID，SDO_CONVEXHULL，SDO_CONCAVEHULL，SDO_CONCAVEHULL_BOUNDARY，SDO_DIAMETER，SDO_DIAMETER_LINE，SDO_LENGTH，SDO_MBC，SDO_MBC_CENTER，SDO_MBC_RADIUS，SDO_MAX_MBR_ORDINATE，SDO_MIN_MBR_ORDINATE，SDO_MBR，SDO_POINTONSURFACE，SDO_TRIANGULATE，SDO_VOLUME，SDO_WIDTH，SDO_WIDTH_LINE

  • 双对象操作：SDO_CLOSEST_POINTS，SDO_DISTANCE，SDO_DIFFERENCE，SDO_INTERSECTION，SDO_MAXDISTANCE，SDO_MAXDISTANCE_LINE，SDO_UNION，SDO_XOR

以下用法信息适用于几何子程序。（另请参阅每个子程序参考信息下的使用说明。）

  • 输入参数和操作的某些组合可以返回空值，即空几何。例如，请求两个不相交的几何对象的交集返回空值。

  • 空值（空几何）作为几何函数的输入参数（例如， SDO_GEOM.RELATE ）会产生错误。

  • 某些操作可以返回与一个或两个输入几何不同类型的几何。例如，线和重叠多边形的交点返回一条线;两条线的交点返回一个点;并且两个切线多边形的交点返回一条线。

  • SDO_GEOM子程序仅支持二维几何，除了以下几者，二维和三维几何都支持：

26.1 SDO_GEOM.RELATE

格式

SDO_GEOM.RELATE（geom1 IN SDO_GEOMETRY，dim1 IN SDO_DIM_ARRAY，掩码IN VARCHAR2，geom2 IN SDO_GEOMETRY，dim2 IN SDO_DIM_ARRAY）RETURN VARCHAR2;

要么

SDO_GEOM.RELATE（geom1 IN SDO_GEOMETRY，掩码IN VARCHAR2，geom2 IN SDO_GEOMETRY，tol IN NUMBER）RETURN VARCHAR2;

描述

检查两个几何对象以确定它们的空间关系。

参数

geom1

几何对象。

dim1

与geom1对应的维度信息数组，通常从xxx_SDO_GEOM_METADATA视图之一中选择（在几何元数据视图中描述）。

面具

指定要检查的关系列表。请参阅“使用说明”中的关键字列表。

geom2

几何对象。

dim2

与geom2对应的维信息数组，通常从xxx_SDO_GEOM_METADATA视图之一中选择（在几何元数据视图中描述）。

tol

公差值（见公差 ）。

使用说明

为了获得更好的性能，请使用SDO_RELATE运算符或其中一种便捷运算符格式（均在Spatial Operators中描述）而不是SDO_GEOM.RELATE函数，除非您需要使用该函数。例如，DETERMINE掩码关键字不适用于SDO_RELATE运算符。有关运算符和函数的性能注意事项的更多信息，请参阅Spatial Operators_ Procedures_和Functions 。

SDO_GEOM.RELATE函数可以返回以下类型的答案：

  • 如果传递一个列出一个或多个关系的mask ，则该函数将返回指定的mask值，如果该几何关系中的一个或多个关系为true。如果所有关系都为false，则过程返回FALSE。

  • 如果在mask传递DETERMINE关键字，则该函数返回与几何最匹配的一个关系关键字。

  • 如果在mask传递ANYINTERACT关键字，则如果两个几何不相交，则函数返回TRUE。

可以测试以下mask关系：

  • ANYINTERACT：如果对象不是不相交的，则返回TRUE。

  • CONTAINS：如果第二个对象完全在第一个对象内并且对象边界不接触，则返回CONTAINS;否则，返回FALSE。

  • COVEREDBY：如果第一个对象完全在第二个对象内并且对象边界在一个或多个点处触摸，则返回COVEREDBY;否则，返回FALSE。

  • COVERS：如果第二个对象完全在第一个对象内并且边界在一个或多个地方触摸，则返回COVERS;否则，返回FALSE。

  • DISJOINT：如果对象没有公共边界或内部点，则返回DISJOINT;否则，返回FALSE。

  • EQUAL：如果对象的边界和内部完全重叠，则返回EQUAL，包括任何孔（即，如果两个几何在拓扑上相等）;否则，返回FALSE。

  • INSIDE：如果第一个对象完全在第二个对象内并且对象边界不接触，则返回INSIDE;否则，返回FALSE。

  • ON：如果一条线（第一个对象）的边界和内部完全位于多边形（第二个对象）的边界上，则返回ON;否则，返回FALSE。

  • OVERLAPBDYDISJOINT：如果对象重叠，则返回OVERLAPBDYDISJOINT，但它们的边界不相互作用;否则，返回FALSE。

  • OVERLAPBDYINTERSECT：如果对象重叠，则返回OVERLAPBDYINTERSECT，并且它们的边界在一个或多个位置相交;否则，返回FALSE。

  • TOUCH：如果两个对象共享一个公共边界点但没有内部点，则返回TOUCH;否则，返回FALSE。

可以使用逻辑布尔运算符OR组合mask的值。例如，如果几何图形之间的关系是INSIDE或TOUCH或INSIDE和TOUCH两者之间的关系，则'INSIDE + TOUCH'返回INSIDE+TOUCH TOUCH;如果几何之间的关系既不是INSIDE也不是TOUCH，则返回FALSE 。

如果geom1和geom2基于不同的坐标系， geom2引发异常。

26.2 SDO_GEOM.SDO_ALPHA_SHAPE

格式

SDO_GEOM.SDO_ALPHA_SHAPE（geom IN SDO_GEOMETRY，tol IN NUMBER，radius IN NUMBER DEFALT NULL，flag IN BINARY_INTEGER DEFAULT 0）RETURN SDO_GEOMETRY;

描述

根据指定的半径值返回输入几何的alpha形状几何。

参数

geom

几何对象。

tol

公差值（见公差 ）。

半径

用于计算alpha形状的半径。如果此参数为null，则alpha形状是输入几何体的凸包。

标志

确定是否包含孤立的点和边： 0 （默认值）包括孤立的点和边，以便返回alpha形状; 1不包括孤立的点和边，因此仅返回alpha形状的多边形部分。

使用说明

α形状是凸包的概括。此函数从输入几何体获取所有坐标，使用它们计算Delaunay三角剖分和alpha形状。

如果为radius参数指定值，则可能首先要使用具有radius输出参数的格式调用SDO_GEOM.SDO_CONCAVEHULL函数。

如果geom是点类型，少于三个点或顶点，或由多个点全部在一条直线上组成，或者如果radius小于0，则会引发异常。

对于大地测量数据，此函数由近似值支持，如具有大地测量数据的近似支持的函数中所述 。

26.3 SDO_GEOM.SDO_ARC_DENSIFY

格式

SDO_GEOM.SDO_ARC_DENSIFY（geom IN SDO_GEOMETRY，dim in SDO_DIM_ARRAY params IN VARCHAR2）RETURN SDO_GEOMETRY;

要么

SDO_GEOM.SDO_ARC_DENSIFY（geom IN SDO_GEOMETRY，在VARCHAR2中输入数字参数）RETURN SDO_GEOMETRY;

描述

返回一个几何图形，其中输入几何中的每个圆弧变为由直线组成的圆弧的近似值，并且每个圆变为由近似圆形的一系列直线组成的多边形。

参数

geom

几何对象。

昏暗

与geom对应的维度信息数组，通常从xxx_SDO_GEOM_METADATA视图之一中选择（在几何元数据视图中描述）。

tol

公差值（见公差 ）。

参数

带引号的字符串，包含弧度容差值和可选的单位值。有关格式和含义的说明，请参阅“使用说明”。

使用说明

如果在投影坐标系中包含圆形或圆弧的几何，则可以使用此函数将它们加密为常规多边形。然后，您可以将生成的直线多边形几何用于任何空间操作，也可以将它们转换为任何投影或大地坐标系。

26.5 SDO_GEOM.SDO_BUFFER

格式

SDO_GEOM.SDO_BUFFER（geom IN SDO_GEOMETRY，dim in SDO_DIM_ARRAY，dist IN NUMBER [，params IN VARCHAR2]）RETURN SDO_GEOMETRY;

要么

SDO_GEOM.SDO_BUFFER（geom IN SDO_GEOMETRY，dist IN NUMBER，tol in NUMBER [，params IN VARCHAR2]）RETURN SDO_GEOMETRY;

描述

生成一个缓冲多边形在几何对象周围或内部。

参数

geom

几何对象。

昏暗

与geom对应的维度信息数组，通常从xxx_SDO_GEOM_METADATA视图之一中选择（在几何元数据视图中描述）。

dist

距离值。如果值为正，则在几何体周围生成缓冲区;如果值为负（仅对多边形有效），则在几何体内生成缓冲区。此参数的绝对值必须大于公差值，如维度数组（ dim参数）或tol参数中所指定。

tol

公差值（见公差 ）。

参数

带引号的字符串，可以包含以下一个或两个关键字：

  • unit和MDSYS.SDO_DIST_UNITS表中的SDO_UNIT值。它标识与dist参数值关联的度量单位，如果指定arc_tolerance关键字，则还标识弧度容差值。有关测量单位规范的更多信息，请参见测量支持单位。

  • arc_tolerance和弧度容差值。有关arc_tolerance关键字的更多信息，请参阅本章中SDO_GEOM.SDO_ARC_DENSIFY函数的使用说明。

例如： 'unit=km arc_tolerance=0.05'

如果输入几何是大地数据，并且未指定arc_tolerance ，则默认值是公差值乘以20。Spatial和Graph使用arc_tolerance值在计算结果时执行弧arc_tolerance化。如果输入几何是笛卡尔坐标或投影数据，则arc_tolerance无效，不应指定。

如果未为笛卡尔或投影几何指定此参数，或者如果为大地几何指定了arc_tolerance关键字但未指定unit关键字，则假定与数据关联的度量单位。

使用说明

此函数返回表示缓冲区多边形的几何对象。

此函数围绕点，线或多边形或多边形内部创建圆形缓冲区。void中的缓冲区也是圆形的，并且与外部边界的距离与外部边界的距离相同。（有关说明，请参阅空间关系和过滤中的距离缓冲图。）

如果缓冲区多边形几何体位于投影坐标系中，则它将包含弧形;如果要将该几何图形转换为大地坐标系，则必须首先使用SDO_GEOM.SDO_ARC_DENSIFY函数对其进行加密，然后转换加密的几何图形。

如果输入几何具有超过50个纵坐标，并且缓冲宽度小于连续坐标之间的均方根间距的0.1％（0.001），则原始几何将保持不变。

对于大地测量数据，此函数由近似值支持，如具有大地测量数据的近似支持的函数中所述 。

对于大地测量数据，此函数仅应用于相对较小的几何图形：用于内部计算的局部切平面投影不会引入明显的扭曲或误差的几何图形。这将源几何的适用范围（无论是线串还是多边形）限制在大约德克萨斯（美国），法国或满洲省（中国）的区域。

执行一个一致性检查有效几何类型，并在几何无效时返回上下文信息。该函数根据元素定义检查表中几何的表示。

参数

theGeometry

几何对象。

theDimInfo

使用说明

在对数据执行任何空间操作之前，应验证所有几何数据并修复任何验证错误，如“加载和验证空间数据的建议”中所述 。

如果几何有效，则此函数返回TRUE。 （对于用户定义的几何体，即SDO_GTYPE值为2000的几何体，此函数返回字符串NULL。）

如果几何无效，则此函数返回以下内容：

  • 基于几何无效的特定原因的Oracle错误消息编号，如果由于某些其他原因导致几何失败，则为FALSE

  • 错误的上下文（导致几何无效的坐标，边或环）。（参见本节“ 错误的上下文：详细信息 ” 。）

此功能检查类型一致性和几何一致性。

对于类型一致性，该函数检查以下内容：

  • SDO_GTYPE有效。

  • SDO_ETYPE值与SDO_GTYPE值一致。例如，如果SDO_GTYPE是2003，则几何体中应至少有一个POLYGON类型的元素。

  • SDO_ELEM_INFO_ARRAY具有有效的三元组值。

对于几何一致性，该函数根据特定几何类型检查以下内容：

  • 多边形至少有四个点，包括关闭多边形的点。（最后一点与第一点相同。）

  • 多边形不是自交叉的。

  • 线或多边形上没有两个顶点是相同的。

  • 多边形的方向正确。（外环边界必须逆时针方向，内环边界必须顺时针方向。）

  • 多边形的内部是连接的。

  • 线串至少有两个点。

  • SDO_ETYPE 1位和4位值在定义多边形环元素时不会混合（即两者都使用）。

  • 弧上的点不是共线的（即，不是直线）并且不是相同的点。

  • 几何图形位于适用的DIMINFO列值的指定范围内（来自USER_SDO_GEOM_METADATA视图）。

  • LRS几何（参见线性参考系统 ）具有三维或四维以及有效的测量尺寸位置（3或4，取决于尺寸的数量）。

对于COLLECTION类型几何，不执行上述某些检查。具体而言，不对COLLECTION类型几何的多边形元素执行内环检查和多边形 - 多边形重叠检查。

对于多点几何，此函数检查具有三维几何的重复顶点，但不检查二维几何。

对于三维几何，此函数还执行三维几何的验证检查中描述的检查 。

在检查几何一致性时，该函数会在确定线是否接触或点是否相同时考虑几何的公差值。

如果使用具有tolerance的函数格式，则不会进行检查以验证几何是否在USER_SDO_GEOM_METADATA视图的DIMINFO字段中存储的坐标系范围内。如果您的使用需要此检查，请使用带有theDimInfo的函数格式。

将flag10g参数值设置为TRUE会导致使用Oracle Spatial Release 10.2的验证逻辑，而不考虑几何的维度。这对于允许包含预发行版11.1格式的几何的三维几何在其他方式失败时通过验证检查非常有用。例如，如果三维线包含圆弧，则该三维线无效;并且将flag10g设置为TRUE将允许这样的几何形状避免仅因为圆弧而被认为是无效的。（稍后您应该根据当前版本的条件使这些几何图形有效，例如通过增加圆弧的密度。）

您可以在PL / SQL过程中使用此函数，以替代使用SDO_GEOM.VALIDATE_LAYER_WITH_CONTEXT过程。有关详细信息，请参阅SDO_GEOM.VALIDATE_LAYER_WITH_CONTEXT的使用说明。

错误的背景：细节

如果几何体无效，则结果可包括有关以下组合的信息：坐标，元素，环和边。

  • 坐标 ：坐标指的是几何中的顶点。在二维几何中，顶点是两个数字（X和Y，或经度和纬度）。在三维几何中，使用三个数字定义顶点;在四维几何中，使用四个数字定义顶点。（您可以使用SDO_UTIL.GETVERTICES函数返回几何中的坐标。）

    如果收到几何验证错误（例如13356）（几何中的相邻点是冗余的），则可以调用SDO_UTIL.GETVERTICES函数，指定rownum停止条件以包括大于错误指示的坐标的坐标。输出中显示的最后两个坐标是冗余坐标。这些坐标可以完全相同，或者它们可以在用户指定的公差范围内，因此被认为是相同的点。您可以使用SDO_UTIL.REMOVE_DUPLICATE_VERTICES函数删除冗余坐标。

  • 元素 ：元素是点，线串或具有零个或多个相应内部多边形的外部多边形。（也就是说，多边形元素包括外环和与外环相关的所有内环。）如果几何是多元素几何（例如，多个点，线或多边形），则第一个元素是元素1，第二个元素是元素2，依此类推。

  • 环 ：环仅用于多边形元素。多边形中的外环被认为是多边形元素，并且外环可以包括零个或多个内环（或孔）。每个内圈都有自己的环名称，但是环1与外部多边形本身相关联。例如，元素1，环1指的是几何中的第一个外部多边形;元素1，环2指的是第一外部多边形的第一内部多边形;和元素1，环3指的是第二内部多边形。如果几何体是多面体，则元素2，环1用于指代第二个外部多边形。如果存在与其关联的内部多边形，则元素2，环2指向第二外部多边形的第一内部多边形。

  • 边 ：边是指两个坐标之间的线段。边1指的是坐标1和坐标2之间的线段，边2指的是坐标2和3之间的线段，依此类推。在验证几何时，查看边缘错误的最常见位置是自相交多边形。（Open Geospatial Consortium简单要素规范不允许多边形自相交。）在这种情况下，Oracle报告错误13349（多边形边界自身交叉），包括发生自相交的元素，环和边数。

    如果针对经度超过119度的优化矩形返回错误13351（共享边缘），则此矩形上的某些查询将返回正确的结果，如大地测量MBR中所述 。
