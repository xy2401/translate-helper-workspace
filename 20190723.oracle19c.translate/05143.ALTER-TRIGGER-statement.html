14.5 ALTER TRIGGER声明

ALTER TRIGGER语句启用，禁用，编译或重命名数据库触发器。

注意：

此语句不会更改现有触发器的声明或定义。要重新声明或重新定义触发器，请使用带有OR REPLACE子句的“ CREATE TRIGGER语句 ” 。

先决条件

如果触发器位于SYS架构中，则必须以SYSDBA身份连接。否则，触发器必须位于您的架构中，或者您必须具有ALTER ANY TRIGGER系统特权。

此外，要更改DATABASE上的触发器，您必须具有ADMINISTER DATABASE TRIGGER系统特权。

语义

alter_trigger

模式

包含触发器的模式的名称。默认值：您的架构。

TRIGGER_NAME

要更改的触发器的名称。

[ENABLE |禁用]

启用或禁用触发器。

重命名为new_name

重命名触发器而不更改其状态。

重命名触发器时，数据库将在*_SOURCE静态数据字典视图中重建记住的触发器源。因此，即使触发源没有更改，注释和格式也可能会在这些视图的TEXT列中发生更改。

{版权|不可取的}

指定是否editioning为架构对象类型后启用触发器是否成为editioned或noneditioned对象TRIGGER的schema 。默认值： EDITIONABLE 。有关更改已编辑和未编辑的对象的信息，请参阅“ Oracle数据库开发指南” 。

对不可接受的限制

您不能为交叉触发器指定NONEDITIONABLE 。

例子

示例14-5禁用触发器

示例模式hr在employees表上创建了一个名为update_job_history的触发器。只要UPDATE语句更改了员工的job_id就会触发该触发器。触发器在job_history表中插入一行，其中包含员工的ID，上一个作业的开始和结束日期以及作业ID和部门。

创建此触发器时，数据库会自动启用它。您可以随后使用以下语句禁用触发器：

ALTER TRIGGER update_job_history DISABLE;

禁用触发器时，如果UPDATE语句更改了员工的作业，则数据库不会触发该触发器。

例14-6启用触发器

禁用触发器后，您可以随后使用以下语句启用它：

ALTER TRIGGER update_job_history ENABLE;

重新启用触发器后，只要UPDATE语句更改了员工的作业，数据库就会触发该触发器。如果在禁用触发器时更新了员工的工作，则数据库不会自动触发此员工的触发器，直到另一个事务再次更改job_id 。
