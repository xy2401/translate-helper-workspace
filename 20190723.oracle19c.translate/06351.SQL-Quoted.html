3.19 SQL（引用）

目的

SQL命令从RMAN中执行SQL语句或PL / SQL存储过程。此语法可与Oracle Database 11.2及更早版本兼容，但为了便于使用，请参阅SQL 。

也可以看看：

Oracle数据库SQL语言参考

先决条件

没有。

句法

quotedsqlcommand :: =


说明quotedsqlcommand.eps的说明

语义

语法元素 描述

CHANNEL channel_id

指定在RUN命令中执行RMAN命令时要使用的通道的区分大小写的名称。

必须首先在此RUN命令中由ALLOCATE CHANNEL分配ALLOCATE CHANNEL 。如果未设置此参数，则RMAN将使用默认通道。

' command '

指定要执行的SQL语句（请参阅示例3-70 ）。不允许使用SELECT语句。

当引用的字符串使用相同的引用样式时，必须使用重复的单引号将单引号插入带引号的字符串中。例如，如果RMAN传递给SQL的字符串包含文件名，则文件名必须用重复的单引号括起来，并且SQL关键字后面的整个字符串必须用双引号括起来（参见例3-71 ）。

注意：由于EXECUTE是SQL * Plus命令，因此无法通过在RMAN SQL命令中指定EXECUTE来EXECUTE PL / SQL程序单元。相反，您必须使用BEGIN和END关键字。例如，要使用SQL命令执行PL / SQL过程rman.rman_purge ，请发出以下命令：

SQL'BEGIN rman.rman_purge;结束;';

例子

示例3-70归档未归档的联机日志

此示例备份表空间，然后归档所有未归档的联机重做日志。

BACKUP TABLESPACE用户; sql'ALTER SYSTEM ARCHIVE LOG CURRENT';

示例3-71在带引号的字符串中指定文件名

此示例通过在双引号字符串中使用重复的单引号来指定文件名。

sql'ALTER TABLESPACE用户添加数据文件''/disk1/oradata/users02.dbf''SIZE 100K AUTOEXTEND ON NEXT 10K MAXSIZE 100K';
