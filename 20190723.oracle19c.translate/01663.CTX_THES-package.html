14 CTX_THES包

本章提供了使用CTX_THES包管理和浏览叙词表的参考信息。除非另有说明，否则这些同义词库函数基于ISO-2788和ANSI Z39.19标准。

了解信息如何存储在同义词库中有助于使用词库运算符编写查询。您还可以使用同义词库扩展知识库，该知识库用于ABOUT英语和法语查询以及生成文档主题。

14.1 ALTER_PHRASE

更改同义词库中的现有短语。只有CTXSYS或同义词库所有者才能更改短语。

句法

CTX_THES.ALTER_PHRASE（varchar2中的tname，varchar2中的短语，varchar2中的op，varchar2中的操作数，默认值为null）;
tname

指定同义词库名称。

短语

指定要更改的短语。

op（更改操作）

将alter操作指定为字符串或符号。您可以使用op和operand对指定以下操作之一：

操作（或改变操作） 含义 操作数

改名

要么

CTX_THES.OP_RENAME

重命名短语。如果同义词库中已存在新短语，则此过程会引发异常。

指定一个新短语。您可以包含限定符以更改，添加或删除短语中的限定符。

PT

要么

CTX_THES.OP_PT

使短语成为首选术语。同义词环中的现有优选术语变为非优选的同义词。

（没有）

SN

要么

CTX_THES.OP_SN

更改短语的范围说明。

指定新的范围注释。
操作数

指定alter操作的参数。请参阅表“ ”op（更改操作）“ ” 。

例子

在词库中纠正错误拼写的单词：

ctx_thes.alter_phrase（'thes1'，'tee'，'rename'，'tea'）;

从汞（金属）中删除限定符：

ctx_thes.alter_phrase（'thes1'，'mercury（metal）'，'rename'，'mercury'）;

为汞添加限定符：

ctx_thes.alter_phrase（'thes1'，'mercury'，'rename'，'mercury（planet）'）;

使Kowalski成为同义词环中的首选术语：

ctx_thes.alter_phrase（'thes1'，'Kowalski'，'pt'）;

更改视图相机的范围注释：

ctx_thes.alter_phrase（'thes1'，'view cameras'，'sn'，'带镜头聚焦的相机'）;

op

CTX_THES.OP_RENAME

14.3 BT

此函数返回指定同义词库中记录的短语的所有更广泛的术语。

语法1：表结果

CTX_THES.BT（restab IN OUT NOCOPY EXP_TAB，短语IN VARCHAR2，lvl IN NUMBER DEFAULT 1，tname IN VARCHAR2 DEFAULT'DEFAULT'）;

语法2：字符串结果

CTX_THES.BT（短语IN VARCHAR2，lvl IN NUMBER DEFAULT 1，tname IN VARCHAR2 DEFAULT'DEFAULT'）RETURN VARCHAR2;
restab

（可选）指定用于存储结果的扩展表的名称。此表必须是EXP_TAB类型，系统定义如下：

type exp_rec是record（xrel varchar2（12），xlevel number，xphrase varchar2（256））; type exp_tab是binary_integer的exp_rec索引表;

也可以看看：

有关EXP_TAB更多信息，请EXP_TAB Oracle文本结果表中的 “ CTX_THES结果表和数据类型 ”

短语

指定要在同义词库中查找的短语。

lvl

指定要返回的更广泛术语的级别。例如，2意味着获得更广泛的术语的更广泛的术语。

tname

指定同义词库名称。如果未指定，则使用系统默认同义词库。

返回

此函数以下列形式返回一系列更广泛的术语：

{bt1} | {bt2} | {bt3} ...

例

字符串结果

考虑一个名为MY_THES的词库，它具有cat的条目，如下所示：

猫BT1猫BT2哺乳动物BT3脊椎动物BT4动物

要查找最多两个级别的cat ，请输入以下语句：

在declare terms上设置serveroutput varchar2（2000）;开始术语：= ctx_thes.bt（'CAT'，2，'MY_THES'）; dbms_output.put_line（'CAT的更广泛扩展是：'||术语）;结束;

此代码生成以下输出：

CAT的更广泛扩展是：{cat} | {feline} | {mammal}

表结果

以下示例使用表结果为白狼执行更广泛的术语查找：

在declare xtab ctx_thes.exp_tab上设置serveroutput;开始ctx_thes.bt（xtab，'white wolf'，2，'my_thesaurus'）; for i in 1..xtab.count loop dbms_output.put_line（xtab（i）.rel ||''|| xtab（i）.phrase）;结束循环;结束;

此代码生成以下输出：

PHRASE WHITE WOLF BT WOLF BT CANINE BT ANIMAL

14.7 CREATE_PHRASE

CREATE_PHRASE过程将新短语添加到指定的同义词库。

注意：

即使您可以使用此过程创建同义词关系，Oracle建议您使用CTX_THES.CREATE_RELATION而不是CTX_THES.CREATE_PHRASE在同义词库中创建关系。

句法

CTX_THES.CREATE_PHRASE（tname IN VARCHAR2，短语IN VARCHAR2，rel IN VARCHAR2 DEFAULT NULL，relname IN VARCHAR2 DEFAULT NULL）;
tname

指定添加新短语或现有短语所在的同义词库的名称。

短语

指定要添加到同义词库的短语或为其创建新关系的短语。

rel

指定短语和relname之间的新关系。仅支持向后兼容性此参数。使用CTX_THES。 CREATE_RELATION在同义词库中创建新关系。

重新命名

指定与短语相关的现有短语 。仅支持向后兼容性此参数。使用CTX_THES。 CREATE_RELATION在同义词库中创建新关系。

返回

条目的ID。

例

在此示例中，在名为tech_thes的同义词库中创建了两个新短语（ 操作系统和操作系统 ）。 

开始ctx_thes.create_phrase（'tech_thes'，'os'）; ctx_thes.create_phrase（'tech_thes'，'操作系统'）;结束;

14.8 CREATE_RELATION

在同义词库中创建两个短语之间的关系。根据字长，同义词环的长度限制为约4000个同义词。

注意：

Oracle建议您使用CTX_THES.CREATE_RELATION而不是CTX_THES.CREATE_PHRASE在同义词库中创建关系。

只有同义词库所有者和CTXSYS可以在给定的同义词库上调用此过程。

句法

CTX_THES.CREATE_RELATION（varchar2中的tname，varchar2中的短语，varchar2中的rel，varchar2中的relphrase）;
tname

指定同义词库名称

短语

指定要更改或创建的短语。如果phrase是消除歧义的同形异义词，则必须指定限定符。如果同义词库中不存在phrase ，则会创建该phrase 。

rel

指定要创建的关系。关系是从phrase到relphrase 。您可以指定以下关系之一：

关系 含义 relphrase

BT * / NT *

添加层次关系。

指定相关短语。这种关系从短语解释为重新短语。

RT

添加关联关系。

指定要关联的短语。

SYN

将短语添加到同义词环。

在同义词环中指定现有短语。

指定语言

为短语添加翻译。

指定新的翻译短语。
重新定义

指定相关短语。如果relphrase不TNAME存在，则创建relphrase。请参阅表相对 。

笔记

您为rel指定的关系被解释为从短语到重新短语。例如，考虑更广泛的术语动物的狗：

狗BT动物

要添加此关系，请按如下所示指定参数：

开始CTX_THES.CREATE_RELATION（'thes'，'dog'，'BT'，'animal'）;结束;

注意：

为CTX_THES.CREATE_RELATION指定参数的顺序与使用CTX_THES.CREATE_PHRASE指定它们的顺序不同。

例子

创建关系VEHICLE NT CAR：

ctx_thes.create_relation（'thes1'，'vehicle'，'NT'，'car'）;

为您创建日语翻译：

ctx_thes.create_relation（'thes1'，'you'，'JAPANESE：'，'kimi'）;

14.9 CREATE_THESAURUS

CREATE_THESAURUS过程在同义词表中创建一个具有指定名称的空同义词库。

句法

CTX_THES.CREATE_THESAURUS（名称IN VARCHAR2，casesens IN BOOLEAN DEFAULT FALSE）;
名称

指定要创建的同义词库的名称。同义词库的名称必须是唯一的。如果已存在具有指定名称的同义词库，则CREATE_THESAURUS将返回错误并且不会创建同义词库。

案例

指定要创建的同义词库是否区分大小写。如果casesens为true ，则Oracle Text会保留在指定的同义词库中输入的所有术语的大小写 。因此，使用同义词库的查询区分大小写。

例

开始ctx_thes.create_thesaurus（'tech_thes'，FALSE）;结束;

14.10 CREATE_TRANSLATION

使用此过程为指定语言的短语创建新翻译。

句法

CTX_THES.CREATE_TRANSLATION（varchar2中的tname，varchar2中的短语，varchar2中的语言，varchar2中的转换）;
tname

使用不超过30个字符指定同义词库的名称。

短语

在同义词库中指定要添加翻译的短语。词组必须已存在于同义词库中，否则会引发错误。

语言

使用不超过10个字符指定翻译的语言。

翻译

指定翻译的术语，使用不超过256个字符。

如果此短语的翻译已存在，则只要原始翻译不相同，就会添加此新翻译而不删除原始翻译。两次添加相同的翻译会导致错误。

例

以下代码将dog的西班牙语翻译添加到my_thes ：

开始ctx_thes.create_translation（'my_thes'，'dog'，'SPANISH'，'PERRO'）;结束;

14.11 DROP_PHRASE

从同义词库中删除一个短语。只有同义词库所有者和CTXSYS可以在给定的同义词库上调用此过程。

句法

CTX_THES.DROP_PHRASE（varchar2中的tname，varchar2中的短语）;
tname

指定同义词库名称。

短语

指定要删除的短语。如果该短语是消除歧义的同形异义词，那么您必须包含限定符。如果tname中不存在该短语，则此过程会引发异常。

BT * / NT *关系在删除的短语周围打了补丁。例如，如果A具有BT B，并且B具有BT C，则在B被丢弃之后，A具有BT C.

当一个词有多个更宽泛的术语时，则为每个更宽泛的术语建立每个较窄术语的关系。

注意，BT，BTG，BTP和BTI是单独的层次结构，因此如果A具有BTG B，并且B具有BTI C，则当B被丢弃时，在A和C之间不存在隐式创建的关系。

RT关系没有修补。例如，如果A具有RT B，并且B具有RT C，则如果B被删除，则在A和C之间不存在关联关系。

例

假设您在mythes中定义了以下关系：

狼BT犬犬BT动物

你掉落短语犬 ：

开始ctx_thes.drop_phrase（'mythes'，'canine'）;结束;

生成的同义词库被修补，看起来像：

狼BT动物

14.12 DROP_RELATION

从同义词库中删除两个短语之间的关系。

注意：

CTX_THES.DROP_RELATION仅删除两个短语之间的关系。这个电话永远不会删除短语。

只有同义词库所有者和CTXSYS可以在给定的同义词库上调用此过程。

句法

（没有）
