22.1 DBMS_AQADM安全模型

最初，只有SYS和SYSTEM对DBMS_AQADM和DBMS_AQ的过程具有执行权限。已被授予DBMS_AQ和DBMS_AQADM EXECUTE权限的用户能够在自己的模式中创建，管理和使用队列。MANAGE_ANY AQ系统特权用于在其他模式中创建和管理队列，并且可以分别通过DBMS_AQADM.GRANT_SYSTEM_PRIVILEGE和DBMS_AQADM.REVOKE_SYSTEM_PRIVILEGE授予和撤销。从Oracle数据库12开始c释放2， MANAGE_ANY权限不允许访问SYS通过比其他用户拥有的队列SYS 。

用户角色

如果您希望用户具有此级别的控制权，则数据库管理员可以选择授予系统权限ENQUEUE_ANY和DEQUEUE_ANY ，直接向数据库用户执行DBMS_AQADM.GRANT_SYSTEM_PRIVILEGE和DBMS_AQADM.REVOKE_SYSTEM_PRIVILEGE 。

应用程序开发人员通过执行DBMS_AQADM.GRANT_QUEUE_PRIVILEGE和DBMS_AQADM.REVOKE_QUEUE_PRIVILEGE来授予和撤消对象级别的权限，从而为队列提供权限。从Oracle数据库12c的第2版开始， ENQUEUE_ANY和DEQUEUE_ANY权限不允许访问SYS通过比其他用户拥有的队列SYS 。

传播所需的安全性

传播作业由SYS拥有，但传播发生在队列表所有者的安全上下文中。先前的传播作业由用户调度传播所拥有，并且传播发生在设置传播调度的用户的安全上下文中。必须在DBMS_AQADM程序包上授予队列表所有者EXECUTE特权。否则，Oracle数据库快照进程不会传播并生成未定义错误标识符SYS.DBMS_AQADM跟踪文件。队列表所有者拥有的专用数据库链接可用于传播。连接字符串中指定的用户名必须对远程数据库上的DBMS_AQ和DBMS_AQADM程序包具有EXECUTE访问权限。

表22-12 ADD_SUBSCRIBER过程参数

参数 描述

规则

基于消息属性，消息数据属性和PL / SQL函数的条件表达式。使用类似于SQL查询的WHERE子句的语法将规则指定为布尔表达式。此布尔表达式可以包括有关消息属性，用户数据属性（仅限对象有效内容）以及PL / SQL或SQL函数（在SQL查询的where子句中指定）的条件。当前支持的消息属性是priority和corrid 。

22.4.4 ALTER_PROPAGATION_SCHEDULE过程

此过程会更改传播计划的参数。

句法

DBMS_AQADM.ALTER_PROPAGATION_SCHEDULE（queue_name IN VARCHAR2，destination IN VARCHAR2 DEFAULT NULL，duration IN NUMBER DEFAULT NULL，next_time IN VARCHAR2 DEFAULT NULL，latency IN NUMBER DEFAULT 60，destination_queue IN VARCHAR2 DEFAULT NULL）;

参数

表22-14 ALTER_PROPAGATION_SCHEDULE过程参数

参数 描述

queue_name

要传播其消息的源队列的名称，包括模式名称。如果未指定模式名称，则默认为用户的模式名称。

目的地

目标数据库链接。传播此目标中收件人的源队列中的邮件。如果为NULL ，则目标是本地数据库，并且消息将传播到本地数据库中的其他队列。此字段的长度当前限制为128个字节，如果名称不是完全限定的，则使用默认域名。

持续时间

传播窗口的持续时间，以秒为单位。NULL值表示传播窗口是永久的，或者直到传播未预定。

下次

日期函数，用于计算当前窗口末尾的下一个传播窗口的开始。如果此值为NULL ，则在当前窗口的末尾停止传播。例如，要在每天的同一时间启动窗口，则应将next_time指定为SYSDATE + 1 - duration/86400 。

潜伏

传播窗口中队列排队后传播的最长等待时间（以秒为单位）。默认值为60。注意：如果未为此调用指定延迟，则延迟将使用默认值覆盖任何现有值。

例如，如果延迟为60秒并且在传播窗口期间没有要传播的消息，则来自该队列的目标消息不会传播至少60秒。在再次检查队列以查找要为指定目标传播的消息之前，至少需要60秒。如果延迟为600，则队列将不会被检查10分钟，如果延迟为0，则作业队列进程将等待消息入队到目的地，并且一旦消息入队，它将是传播。

destination_queue

在再次检查队列以查找要为指定目标传播的消息之前至少60秒。如果延迟为600，则不检查队列10分钟，如果延迟为0，则作业队列进程将等待将消息排入目标。一旦消息入队，它就会被传播。
