Oracle RAC的目录结构

查看此主题以了解适用于Oracle RAC的示例最佳灵活体系结构（OFA）兼容数据库的分层目录树。

表B-1符合OFA标准的环境的目录结构

目录 描述

$ ORACLE_BASE

默认的ORACLE_BASE目录，其中软件所有者是oracle用户。例如：

/ U01 /应用/ ORACLE

installation_type

Oracle基目录下的安装类型。例如，安装Oracle Database 19c时，安装类型的值为product/19.0.0/dbhome_1 。例如：

/u01/app/oracle/product/19.0.0/dbhome_1

$ ORACLE_HOME

安装Oracle数据库软件的位置。如果您有多个软件安装，也可以选择添加计数器。例如，如果要为Oracle Database 19c软件创建第二个Oracle主目录，则路径如下：

/u01/app/oracle/product/19.0.0/dbhome_2

请注意，Oracle数据库主目录位于Oracle基本目录路径下。一个Oracle集群件目录一定不能在数据库可执行文件位于甲骨文基本目录下。

DBS

为数据库存储本地初始化参数文件的目录。

管理

管理目录。请注意，对于Oracle Database 11g， bdump ， cdump和udump文件将重定位到与ADR_BASE关联的目录。例如：

/ U01 /应用/ ORACLE /管理

DB_UNIQUE_NAME

数据库唯一名称;当数据库名称的长度为8个或更少时，这与dbname相同。例如，如果您的数据库名称是sales ，则目录路径为：

/ U01 /应用/ ORACLE /管理/销售

/ hdump

/ PFILE

数据库服务器的转储目标。

$ ADR_BASE / bdump

$ ADR_BASE / cdump

$ ADR_BASE / UDUMP

自动诊断存储库转储目标跟踪文件。请注意，此目录路径由初始化参数DIAGNOSTIC_DEST ，并且自动数据存储库的路径必须位于所有节点可用的共享存储位置。

默认情况下，此路径是Oracle基本目录的子集，位于以下路径中：

$ ORACLE_BASE /诊断/

适用于集群主目录的Oracle Grid Infrastructure（网格主页）

适用于Oracle Clusterware主目录的OFA兼容路径。默认值为：

/u01/app/19.0.0/grid

在用于群集安装的Oracle Grid Infrastructure期间，将安装Oracle Clusterware和Oracle自动存储管理（Oracle ASM）软件。root.sh脚本更改Oracle Clusterware主目录的所有父目录的权限，以仅向root用户授予写访问权限。因此，Oracle Clusterware主目录不能是Oracle基目录的子目录。

箱子

Oracle Clusterware和Oracle ASM可执行文件的子树。

网络

Oracle Net Services配置文件和实用程序的子树。
