16 Oracle Database Vault命令规则API

DBMS_MACADM PL / SQL包提供了配置命令规则的过程。 。

只有已被授予DV_OWNER或DV_ADMIN角色的用户才能使用这些过程。

  • CREATE_COMMAND_RULE过程
    CREATE_COMMAND_RULE过程创建命令规则并将其与规则集关联。
  • CREATE_CONNECT_COMMAND_RULE过程
    CREATE_CONNECT_COMMAND_RULE过程创建一个CONNECT命令规则，您可以将该规则与用户和规则集关联。
  • CREATE_SESSION_EVENT_CMD_RULE过程
    CREATE_SESSION_EVENT_CMD_RULE过程基于ALTER SESSION语句创建可与会话事件关联的命令规则。
  • CREATE_SYSTEM_EVENT_CMD_RULE过程
    CREATE_SYSTEM_EVENT_CMD_RULE过程基于ALTER SYSTEM语句创建可与系统事件关联的命令规则。
  • DELETE_COMMAND_RULE过程
    DELETE_COMMAND_RULE过程删除命令规则声明。
  • DELETE_CONNECT_COMMAND_RULE过程
    DELETE_CONNECT_COMMAND_RULE过程删除使用CREATE_CONNECT_COMMAND_RULE过程创建的CONNECT命令规则。
  • DELETE_SESSION_EVENT_CMD_RULE过程
    DELETE_SESSION_EVENT_CMD_RULE过程删除与事件关联的会话命令规则。
  • DELETE_SYSTEM_EVENT_CMD_RULE过程
    DELETE_SYSTEM_EVENT_CMD_RULE过程删除与事件关联的系统命令规则。
  • UPDATE_COMMAND_RULE过程
    UPDATE_COMMAND_RULE过程更新命令规则声明。
  • UPDATE_CONNECT_COMMAND_RULE过程
    UPDATE_CONNECT_COMMAND_RULE过程更新使用CREATE_CONNECT_COMMAND_RULE过程创建的CONNECT命令规则。
  • UPDATE_SESSION_EVENT_CMD_RULE过程
    UPDATE_SESSION_EVENT_CMD_RULE过程基于ALTER SESSION语句更新会话事件命令规则。
  • UPDATE_SYSTEM_EVENT_CMD_RULE过程
    UPDATE_SYSTEM_EVENT_CMD_RULE过程基于ALTER SYSTEM语句更新系统事件命令规则。

CREATE_COMMAND_RULE过程

CREATE_COMMAND_RULE过程创建命令规则并将其与规则集关联。

（可选）您可以使用它来启用规则集的规则检查的命令规则。在多租户环境中，您可以创建公共命令规则和本地命令规则。

句法

DBMS_MACADM.CREATE_COMMAND_RULE（命令IN VARCHAR2，rule_set_name IN VARCHAR2，object_owner IN VARCHAR2，object_name IN VARCHAR2，启用IN VARCHAR2，privilege_scope IN NUMBER，clause_name IN VARCHAR2，parameter_name IN VARCHAR2，event_name IN VARCHAR2，component_name IN VARCHAR2，action_name IN VARCHAR2，范围IN NUMBER DEFAULT）;

参数

表16-1 CREATE_COMMAND_RULE参数

参数 描述

命令

要保护的SQL语句。

另见以下内容：

  • 默认命令规则有关默认命令规则的信息

  • DBA_DV_COMMAND_RULE查看现有命令规则的列表

  • 可以通过命令规则保护的 SQL语句，以获取可以使用的可用SQL语句的列表

rule_set_name

与此命令规则关联的规则集的名称。

要在当前数据库实例中查找现有规则集，请查询DBA_DV_RULE_SET视图中描述的DBA_DV_RULE_SET视图 。

OBJECT_OWNER

将应用此命令规则的数据库模式。允许使用通配符% ，但SELECT ， INSERT ， UPDATE ， DELETE和EXECUTE语句除外。

要查找可用用户，请查询Oracle Database Reference中描述的DBA_USERS视图。

有关更多信息，另请参阅创建命令规则中的 “对象所有者”。

OBJECT_NAME

受命令规则保护的对象。（允许使用通配符％。有关受命令规则保护的对象的更多信息，请参阅创建命令规则中的 “对象名称”。）

要查找可用对象，请查询Oracle数据库参考中描述的ALL_OBJECTS视图。

启用

指定以下选项之一以设置命令规则的状态：

  • DBMS_MACUTL.G_YES或'y' （是）启用命令规则（默认）

  • DBMS_MACUTL.G_NO或'n'禁用命令规则，包括捕获模拟日志中的违规

  • DBMS_MACUTL.G_SIMULATION或's'使SQL语句能够执行，但在模拟日志中捕获违规

privilege_scope

过时的参数

clause_name

SQL语句中用于创建命令规则的子句。例如， ALTER SESSION SQL语句的命令规则可以将SET子句作为clause_name参数。

仅适用于ALTER SYSTEM和ALTER SESSION命令规则。

PARAMETER_NAME

clause_name参数中的参数。例如，对于ALTER SESSION命令规则，如果clause_name为SET ，则可以将parameter_name设置为EVENTS 。

仅适用于ALTER SYSTEM和ALTER SESSION命令规则。

EVENT_NAME

命令规则定义的事件。例如，假设ALTER SESSION命令规则使用SET的clause_name和EVENTS作为parameter_name 。如果要跟踪跟踪事件，则可以将event_name设置为TRACE 。

仅适用于将parameter参数设置为EVENTS ALTER SYSTEM和ALTER SESSION命令规则。

COMPONENT_NAME

event_name设置的一个组件。例如，对于TRACE事件， component_name可以是GCS 。

仅适用于将parameter参数设置为EVENTS ALTER SYSTEM和ALTER SESSION命令规则。

ACTION_NAME

component_name设置的操作。

仅适用于将parameter参数设置为EVENTS ALTER SYSTEM和ALTER SESSION命令规则。

范围

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）如果命令规则在当前PDB中是本地的

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果命令规则在应用程序根目录中

如果在应用程序根目录中创建公共命令规则并希望它对关联的PDB可见，则必须同步应用程序。例如：

ALTER PLUGGABLE DATABASE APPLICATION saas_sales_app SYNC;

例子

简单的命令规则

以下示例说明如何在OE.ORDERS表上为SELECT语句创建简单命令规则。此命令规则不使用命令规则。 

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'SELECT'，rule_set_name =>'Check User Role'，object_owner =>'OE'，object_name =>'ORDERS'，enabled => DBMS_MACUTL.G_YES）;结束; /

此示例显示如何创建命令规则，以检查用户是否可以启用或禁用hr_audit_pol统一审核策略。请注意，如果对象是统一的审核策略，那么您必须具有AUDIT POLICY ，而不仅仅是AUDIT ，用于command参数。 

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'AUDIT POLICY'，rule_set_name =>'检查审核能力'，object_owner =>'％'，object_name =>'hr_audit_pol'，enabled => DBMS_MACUTL.G_YES，scope => DBMS_MACUTL.G .SCOPE_LOCAL）;结束; /

ALTER SESSION命令规则使用SET子句

以下示例说明如何创建使用带有ERROR_ON_OVERLAP_TIME参数的SET子句的ALTER SESSION命令规则。 

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'ALTER SESSION'，rule_set_name =>'Test ERROR_ON_OVERLAP_TIME for FALSE'，object_owner =>'％'，object_name =>'％'，enabled => DBMS_MACUTL.G_YES，clause_name =>'SET' ，parameter_name =>'ERROR_ON_OVERLAP_TIME'，scope => DBMS_MACUTL.G_SCOPE_COMMON）;结束; /

在这个例子中：

  • rule_set_name ： ALTER SESSION SQL语句ERROR_ON_OVERLAP_TIME会话参数必须设置为TRUE或FALSE 。您可以创建一个规则集来检查此设置。例如，对于规则：

    EXEC DBMS_MACADM.CREATE_RULE（'RULE_TRUE'，'UPPER（PARAMETER_VALUE）=''TRUE'''）;

    与此规则一起使用的规则集可以类似于以下内容：

    BEGIN DBMS_MACADM.CREATE_RULE_SET（rule_set_name => '测试ERROR_ON_OVERLAP_TIME'，描述=> '检查该ERROR_ON_OVERLAP_TIME设置为TRUE或FALSE'，使能=> DBMS_MACUTL.G_YES，eval_options => DBMS_MACUTL.G_RULESET_EVAL_ALL，audit_options => DBMS_MACUTL.G_RULESET_AUDIT_FAIL + DBMS_MACUTL .G_RULESET_AUDIT_SUCCESS，fail_options => DBMS_MACUTL.G_RULESET_FAIL_SILENT，fail_message =>'重叠错误'，fail_code => 20461，handler_options => DBMS_MACUTL.G_RULESET_HANDLER_FAIL，handler =>''，is_static => false）;结束; / EXEC DBMS_MACADM.ADD_RULE_TO_RULE_SET（'Test ERROR_ON_OVERLAP_TIME'，'RULE_TRUE'）;

  • 对于ALTER SESSION和ALTER SYSTEM命令规则，必须将object_owner和object_name设置为% 。

  • enabled使用DBMS_MACUTL.G_YES常量在创建命令规则时启用它。

  • clause_name设置ALTER SESSION命令规则以使用ALTER SESSION PL / SQL语句的SET子句。

  • parameter_name设置为SET子句的ERROR_ON_OVERLAP_TIME参数。

  • scope使用DBMS_MACUTL.G_SCOPE_COMMON常量将命令规则设置为公共命令规则。此命令规则将位于多租户环境的应用程序根目录中，因此运行此过程的用户必须位于CDB根目录中。与此命令规则关联的任何规则或规则集必须是通用的。

    如果要在本地创建命令规则，则应将scope设置为DBMS_MACUTL.G_SCOPE_LOCAL 。在这种情况下，运行此过程的用户必须位于命令规则所在的PDB中。要查找现有PDB，可以查询DBA_PDBS数据字典视图。与此命令规则关联的任何规则或规则集都必须是本地的。

ALTER SYSTEM命令规则使用CHECKPOINT子句

此示例显示如何创建用户CHECKPOINT子句的ALTER SYSTEM命令规则。要对CHECKPOINT设置进行命令规则测试，必须创建规则集和规则，类似于上一示例中的ALTER SESSION命令规则。在此示例中，未指定parameter设置，因为CHECKPOINT设置没有参数。 

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'ALTER SYSTEM'，rule_set_name =>'Test CHECKPOINT Setting'，object_owner =>'％'，object_name =>'％'，enabled => DBMS_MACUTL.G_YES，clause_name =>'CHECKPOINT'， parameter_name =>''，scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

ALTER SESSION命令规则使用SET子句

以下ALTER SESSION命令规则使用SET子句指定event_name和component_name 。您只能使用event_name ， component_name和action_name如果参数clause_name参数指定SET 。

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'ALTER SESSION'，rule_set_name =>'Check Trace Events'，object_owner =>'％'，object_name =>'％'，enabled => DBMS_MACUTL.G_YES，clause_name =>'SET'， parameter_name =>'EVENTS'，event_name =>'TRACE'，component_name =>'GCS'，scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

有关此主题的概念性信息，另请参阅ALTER SESSION和ALTER SYSTEM命令规则 。

CREATE_CONNECT_COMMAND_RULE过程

CREATE_CONNECT_COMMAND_RULE过程创建一个CONNECT命令规则，您可以将该规则与用户和规则集关联。

在多租户环境中，您可以创建公共命令规则和本地命令规则。

句法

DBMS_MACADM.CREATE_CONNECT_COMMAND_RULE（user_name IN VARCHAR2，rule_set_name IN VARCHAR2，启用IN VARCHAR2，范围IN NUMBER DEFAULT）;

参数

表16-4 CREATE_CONNECT_COMMAND_RULE参数

参数 描述

用户名

CONNECT命令规则将应用于的用户。如果输入%通配符，则CONNECT命令规则将应用于每个数据库用户。

在多租户环境中，如果在root中执行此过程，则指定%适用于所有普通用户。如果在PDB中运行该过程，则它适用于有权访问此PDB的所有本地和普通用户。如果有两个命令规则，一个是公共命令，一个是本地，并且它们都适用于同一个对象，则两个命令规则必须成功评估才能使操作成功。

在多租户环境中，如果CONNECT命令规则是常见的，请确保此用户是公用的，如果CONNECT命令规则是本地的，则确保本地用户或公用用户。

rule_set_name

与此命令规则关联的规则集的名称。在多租户环境中，如果CONNECT命令规则是公共的，则确保此规则集是常见的，如果CONNECT命令规则是本地的，则确保本地规则集是本地规则集。

  • DBMS_MACUTL.G_NO或'n'禁用命令规则，包括捕获模拟日志中的违规

  • DBMS_MACUTL.G_SIMULATION或's'使SQL语句能够执行，但在模拟日志中捕获违规

范围

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果命令规则在应用程序根目录中

如果在应用程序根目录中创建公共CONNECT命令规则并希望它对关联的PDB可见，则必须同步应用程序。例如：

例子

以下示例说明如何在多租户环境中创建公共CONNECT命令规则。此命令规则将位于CDB根目录中，因此运行此过程的用户必须位于CDB根目录中。与此命令规则关联的任何用户名或规则集必须是通用的。

BEGIN DBMS_MACADM.CREATE_CONNECT_COMMAND_RULE（rule_set_name =>'允许会话'，user_name =>'C ## HR_ADMIN'，启用=> DBMS_MACUTL.G_SIMULATION，范围=> DBMS_MACUTL.G_SCOPE_COMMON）;结束; /

BEGIN DBMS_MACADM.CREATE_CONNECT_COMMAND_RULE（rule_set_name =>'允许会话'，user_name =>'PSMITH'，启用=> DBMS_MACUTL.G_SIMULATION，范围=> DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /
