3.1.3.1数据存储对象

流根据文档中的数据存储区读取开始，因为它们根据您的数据存储区首选项存储在系统中。例如，如果您将数据存储定义为FILE_DATASTORE,则通过从操作系统读取文件来启动流。您还可以将文档存储在Internet或Oracle数据库中。无论您的文件是物理存在的，Oracle数据库中的文本表都必须始终指向文件。

3.1.3.2过滤对象

然后流经过过滤器。您的FILTER首选项确定会发生什么。可以通过以下方式之一对流进行操作：

  • 指定NULL_FILTER首选项类型或format列的值为时，不会进行过滤 IGNORE.纯文本，HTML或XML文档无需过滤。

  • 当您指定AUTO_FILTER首选项类型或格式列的值为BINARY.时，格式化文档（二进制）将被过滤为标记文本BINARY.

3.1.3.3分区对象

经过筛选后，标记的文本将通过分区程序，分区程序将流分为文本和分区信息。部分信息包括部分在文本流中的开始和结束位置。提取的节的类型由节组类型决定。

文本传递给词法分析器。节信息直接传递给索引引擎，后者稍后使用它。

3.1.3.4 Lexer对象

您可以使用Oracle Text lexer类型之一创建词法分析器首选项，以指定要编制索引的文本的语言。词法分析器根据您的语言将文本分解为标记。这些代币通常是单词。要提取标记，词法分析器使用词法分析器首选项中定义的参数。这些参数包括分隔标记的字符的定义，例如空格。参数还包括是将文本转换为全部大写还是将其保留为大小写混合。

当您启用主题索引时，词法分析器会分析您的文本以创建用于索引的主题标记。

3.1.3.5索引引擎

索引引擎创建倒排索引，将标记映射到包含它们的文档。在此阶段，Oracle Text使用您指定的停止列表来排除停用词或从索引中停止主题。Oracle Text还使用您的定义中的参数 WORDLIST偏好。这些参数告诉系统如何创建前缀索引或子串索引（如果已启用）。

3.1.4关于索引列的更新

在Oracle Database 12c第2版（12.2）之前的版本中，当Oracle Text索引所基于的列发生更新时，在索引同步之前，该文档不可用于搜索操作。用户查询无法搜索此文档。从Oracle Database 12 c第2版（12.2）开始，您可以指定在更新后必须可以搜索文档，而无需立即执行索引同步。在索引同步之前，查询使用旧索引条目来获取旧文档的内容。在索引同步之后，用户查询将获取更新文档的内容。

索引的ASYNCHRONOUS_UPDATE选项使您可以在更新后保留文档的旧内容，然后使用此索引来回答用户查询。

3.1.5分区表和索引

在分区文本表上创建分区的CONTEXT索引时，必须按范围对表进行分区。不支持散列，复合和列表分区。

您可以创建分区文本表，以按日期对数据进行分区。例如，如果您的应用程序维护了一个大型的过时新闻文章库，您可以按月或按年划分您的信息。分区简化了大型数据库的可管理性，因为查询，插入，更新，删除操作以及备份和恢复可以作用于单个分区。

在具有多个表集的本地CONTEXT索引上，Oracle Text支持Oracle数据库支持的分区数。

注意：

的数量 Oracle Text支持的分区大约为1024K-1。此限制应该足够，不适用于分区表上的CONTEXT索引。

3.1.7并行索引

Oracle Text支持使用CREATE INDEX语句进行并行索引。

在非分区表上输入并行索引语句时，Oracle Text会将基表拆分为临时分区，生成子进程并将子进程分配给分区。然后每个子进程索引其分区中的行。将基表切分为分区的方法由Oracle确定，不受您的直接控制。对于实际生成的子进程数量也是如此，这取决于机器功能，系统负载，您的 init.ora设置和其他因素。由于这些变量，实际的并行度可能与请求的并行度不匹配。

由于索引是一种密集的I / O操作，因此并行索引最有效在分布式磁盘访问和多个CPU时减少索引时间。并行索引只能使用CREATE INDEX语句影响初始索引的性能。它不会影响ALTER INDEX,插入，更新和删除操作INDEX,并且对查询性能的影响很小。

由于并行索引会缩短初始索引时间，因此对以下方案很有用：

  • 数据暂存，当您的产品包含Oracle Text索引时

  • 基于大数据集合的应用程序的快速初始启动

  • 应用程序测试，当您需要在开发应用程序时测试不同的索引参数和模式

3.2.10.2混合格式列过滤

为一个混合格式列，例如包含Microsoft Word，纯文本和HTML文档的列，您可以通过在文本表中包含格式列来绕过纯文本或HTML的过滤。在format列中，标记TEXT或BINARY.每一行BINARY.标记为TEXT行不会被过滤。

例如，将HTML和纯文本行标记为TEXT ，将Microsoft Word行标记为BINARY.您可以在CREATE INDEX参数子句中指定format列。

如果不希望索引文档，可以使用第三种格式列类型IGNORE.此列类型很有用，例如，当混合格式表包含日语和英语的纯文本文档时，您只想处理英语文档。当混合格式表包含纯文本文档和图像时，此列类型也很有用。由于IGNORE是在数据存储级别实现的，因此您可以将其与所有过滤器一起使用。

3.2.10.3自定义过滤

您可以创建自定义筛选器以筛选文档以进行索引。您可以创建从文件系统执行的外部过滤器，也可以创建内部过滤器作为PL / SQL或Java存储过程。

对于外部自定义筛选，请使用USER_FILTER筛选器首选项类型。

对于内部过滤，请使用PROCEDURE_FILTER过滤器类型。

3.2.12文档字符集

索引引擎期望过滤的文本在数据库字符集中。使用AUTO_FILTER过滤器类型时，格式化的文档将转换为数据库字符集中的文本。

如果源是文本而文档字符集不是数据库字符集，则可以使用AUTO_FILTER过滤器类型转换文本以进行索引。

字符集检测

将CHARSET列设置为AUTO, AUTO_FILTER过滤器会检测文档的字符集，并将其从检测到的字符集转换为数据库字符集（如果存在差异）。

混合字符集列

如果文档集包含具有不同字符集的文档，例如 JA16EUC和 JA16SJIS，您可以索引文档，前提是您创建了一个CHARSET列，使用每行的文档字符集名称填充此列，并在CREATE INDEX语句的parameter子句中命名该列。

3.3文件语言

Oracle Text可以索引大多数语言。默认情况下，Oracle Text假定要编制索引的文本的语言是您在数据库设置中指定的语言。根据文档的语言，使用以下词法分类器类型之一：

  • BASIC_LEXER:索引以空格分隔的语言，如英语，法语，德语和西班牙语。对于其中一些语言，您可以启用备用拼写，复合词索引和基本字母转换。

  • USER_LEXER:创建自己的词法分析器以索引特定语言。

  • WORLD_LEXER:索引包含不同语言文档的表，并自动检测文档中的语言。

通过BASIC_LEXER首选项，Oracle Text为大多数语言提供了一个lexing解决方案。对于日语，中文和韩语，您可以在用户定义的词法分析器界面中创建自己的lexing解决方案。

  • BASIC_LEXER之外的语言功能： 用户定义的词法分析器界面使您能够创建PL / SQL或Java过程，以在索引和查询期间处理文档。使用用户定义的词法分析器，您还可以创建自己的主题lexing解决方案或语言处理引擎。

  • 多语言列： Oracle Text可以索引包含不同语言文档的文本列，例如包含用英语，德语和日语编写的文档的列。要索引多语言列，请在文本表中添加语言列，并使用MULTI_LEXER首选项类型。在索引多语言列时，还可以合并多语言停止列表。

3.4特殊字符

当你使用 BASIC_LEXER首选项类型，您可以指定非字母数字字符（如连字符和句点）如何与包含它们的标记相关联。例如，您可以指定Oracle Text在索引诸如副总裁之类的单词时包含或排除连字符（ - ） 。

根据索引期间所需的行为，这些字符属于BASIC_LEXER类别。将词法分析器设置为用于索引的行为的方式是它对查询解析的行为方式。

某些您可以设置的特殊字符如下：

  • Printjoin字符：当您希望在索引期间将此字符包含在令牌中时，将非字母数字字符定义为printjoin 。例如，如果希望索引包含连字符和下划线，请将它们定义为printjoins。这意味着像副总统这样的词被列为副总统。 在副总统查询不会副总统找到。

  • 其他字符：您可以指定其他字符来控制其他字符标记化行为，例如令牌分离（ startjoins， endjoins， 空格） 标点符号（标点符号）， 数字标记化（numjoins）和换行后的单词延续（ 延续）。这些类别的字符具有可修改的默认值。

3.6使用远期指数改进文档服务性能

当它在文档中搜索单词时，Oracle Text使用反向索引，然后通过计算该文档中的代码段来显示结果。为了计算片段，重新索引作为搜索结果的一部分返回的每个文档。当文档的大小非常大时，搜索操作会显着减慢。

前向索引克服了非常大的文档的性能问题。它使用$O映射表引用$I倒排索引表中的令牌偏移量。每个令牌偏移量被转换为原始文档中的字符偏移量，然后围绕字符偏移量的文本用于生成文本片段。

由于前向索引在计算代码段时不使用文档的内存索引，因此在搜索非常大的文档中的单词时，它提供了相对于反向索引的显着改进的性能。

正向索引提高了Oracle Text CTX_DOC包中以下过程的性能：

  • CTX_DOC.SNIPPET

  • CTX_DOC.HIGHLIGHT

  • CTX_DOC.MARKUP

3.6.1启用前向索引

3.6.3使用保存副本转发索引

使用前向索引和保存副本

要有效地使用转发索引，您应该以纯文本格式或过滤格式存储$D表中的文档副本，具体取决于您使用的CTX_DOC程序包过程。例如，使用SNIPPET过程时以纯文本格式存储文档，并在使用MARKUP或HIGHLIGHT过程时以过滤格式MARKUP文档。

您应该使用Oracle Text的“保存复制”功能将文档的副本存储在$D表中。使用save_copy属性或save_copy列参数实现该功能。

  • save_copy基本存储属性：

    以下示例将BASIC_STORAGE存储类型的save_copy属性值设置为save_copy PLAINTEXT.此示例使Oracle Text能够在$D表中保存文本文档的副本，同时在该文档中搜索单词。 

    exec ctx_ddl.create_preference（'mystore'，'BASIC_STORAGE'）; exec ctx_ddl.set_attribute（'mystore'，'save_copy'，'PLAINTEXT'）;

  • save_copy column索引参数：

    以下示例使用save_copy column索引参数将文本文档的副本保存到$D表中。create index语句创建$D表并将文档1（“hello world”）复制到$D表中。 

    create table docs（id number，txt varchar2（64），save varchar2（10））;插入docs值（1，'hello world'，'PLAINTEXT'）;在docs上创建索引idx（txt）indextype是ctxsys.context参数（'save_copy column save'）;

对于save_copy属性或列参数，您可以指定以下值之一：

  • PLAINTEXT以纯文本格式将文档副本保存在$D索引表中。纯文本格式定义为分段器的输出格式。使用SNIPPET过程时指定此值。

  • FILTERED在$D索引表中以过滤格式保存文档的副本。过滤后的格式定义为过滤器的输出格式。使用MARKUP或HIGHLIGHT过程时指定此值。

  • NONE不会将文档副本保存在$D索引表中。如果不使用SNIPPET, MARKUP,或HIGHLIGHT过程，并且索引VARCHAR2或CLOB. SNIPPET, MARKUP,请指定此值CLOB.

3.6.4没有保存副本的转发索引

在以下方案中，您可以利用前向索引的性能增强而不保存$D表中的所有文档的副本（即，不使用“保存复制”功能）：

  • 文档集包含HTML和纯文本：使用DIRECT_DATASTORE或MULTI_COLUMN_DATASTORE数据存储区类型将所有文档存储在基表中。

  • 文档集包含HTML，纯文本和二进制文件：使用DIRECT_DATASTORE数据存储区类型将所有文档存储在基表中。仅以过滤格式将二进制文档存储在$D表中。

3.8.2使用用户定义规则创建新实体类型的示例

本节中的示例显示如何使用用户定义的规则创建新的实体类型。您可以使用基于正则表达式的语法定义规则，并将规则添加到提取策略中。无论何时使用，都会应用该策略。

以下规则通过匹配以下任何表达式来识别股票指数的增加：

因此，您必须创建一种新类型的实体以及与任何表达式匹配的正则表达式。由于用户定义的实体必须以字母“x”开头，因此请按如下方式调用您的实体“xPositiveGain”：

ctx_entity.add_extract_rule（'mypolicy'，1，'<rule>'||'<expression>'||'（（攀爬|增加|增加|增加|反弹）'||'（（by | over | almost |超过））* \ d +（\。\ d +）？（百分比|％））'|| '</ expression>'|| '<type refid =“1”> xPositiveGain </ type>'|| '</规则>'）;

注意在示例中使用refid 。它通过引用其中的一对括号来告诉您要匹配正则表达式的哪一部分。例如，因为您需要整个表达式，即最外层（和第一个出现的）括号，即refid=1 。

ctx_entity.compile（ 'mypolicy'）;

ctx_entity.extract（'mypolicy'，mydoc，null，myresults）

3.9模糊匹配和词干

模糊匹配使您可以在查询中匹配类似拼写的单词。Oracle Text提供实体提取多种语言。

词干使您可以匹配具有相同语言根的单词。例如，对$ speak的查询，展开以搜索包含说话，说话，说话和口语的所有文档。

如果Oracle Text支持您的语言的此功能，则会在索引中自动启用模糊匹配和词干分析。

模糊匹配启用其模糊分数的默认参数及其最大扩展项数。模糊分数衡量扩展单词与查询单词的匹配程度。在索引时，您可以更改这些默认参数。

要自动检测文档的语言并执行必要的转换，请通过启用index_stems属性来创建词干索引。 AUTO_LEXER.使用与文档语言对应的词干分析器，并始终配置词干分析器以最大程度地调用文档。对于德语，芬兰语，瑞典语和荷兰语等复合词，如果将index_stems设置为YES,则会在文档中自动执行复合词词干。化合物总是分离成它们的组分茎。

要提高词干查询的性能，请通过启用index_stems属性来创建词干索引BASIC_LEXER.

3.10更好的通配符查询性能

通配符查询使您可以输入左侧截短， 右截断，和双截断查询，例如％ing， cos％或％benz％。 使用常规索引，这些查询有时会扩展为大型单词列表并降低查询性能。

当令牌前缀和子字符串记录在索引中时，通配符查询具有更好的响应时间。

默认情况下， 令牌前缀和子字符串不会记录在Oracle Text索引中。如果您的查询应用程序大量使用通配符查询，请考虑索引标记前缀和子字符串。为此，请使用 wordlist首选项类型。权衡是改进通配符搜索的更大指标。

3.12停用词和停止主题

停用词是您不希望编入索引的单词。停用词是通常是给定语言中的低信息词，例如 这和 那是英文的。

默认情况下，Oracle Text提供了一个用于索引给定语言的停止列表。修改此列表或使用。创建自己的列表 CTX_DDL包。在CREATE INDEX语句的参数字符串中指定停止列表。

stoptheme是一个被阻止被主题索引或被阻止对主题做出贡献的词。使用CTX_DDL包添加stopthemes。

  • 语言检测和停止列表：在查询时，查询的语言继承自查询模板或会话语言（如果未通过查询模板指定语言）。

  • 多语言停止列表：您可以创建多语言停止列表来保存特定于语言的停用词。当您使用MULTI_LEXER索引包含不同语言（如英语，德语和日语）的文档的表时，此停止列表非常有用。在索引创建时，将检查每个文档的语言列，并且仅消除该语言的停用词。在查询时，会话语言设置确定活动停用词，就像它确定多词法分析器的活动词法分析器一样。

3.15混合查询性能

如果您的CONTAINS()查询在非文本列上也有结构化谓词，那么请考虑索引这些列值。为此，请在CREATE INDEX语句的FILTER BY子句中指定这些列。然后，Oracle Text可以确定是否让Oracle Text索引处理结构化谓词以获得更好的性能。

此外，如果您的CONTAINS()查询在一个或多个结构化列上具有ORDER BY条件，则Oracle Text索引也可以索引这些列值。在CREATE INDEX语句的ORDER BY子句中指定这些列。然后，Oracle Text可以确定是否将排序推送到Oracle Text索引以获得更好的查询响应时间。
