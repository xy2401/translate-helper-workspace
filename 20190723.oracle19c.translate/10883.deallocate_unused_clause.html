deallocate_unused_clause

目的

使用deallocate_unused_clause显式释放数据库对象段末尾的未使用空间，并使该空间可用于表空间中的其他段。

您可以使用以下语句释放未使用的空间：

  • ALTER CLUSTER （参见ALTER CLUSTER )

  • ALTER INDEX ：从索引，索引分区或索引子分区中释放未使用的空间（请参阅ALTER INDEX )

  • ALTER MATERIALIZED VIEW ：从索引组织的物化视图的溢出段中释放未使用的空间（参见ALTER MATERIALIZED VIEW ）

  • ALTER TABLE ：从表中释放未使用的空间，表分区，表子分区，索引组织表的映射表，索引组织表的溢出段或LOB存储段（请参阅ALTER TABLE )

语义

本节描述deallocate_unused_clause的语义。有关其他信息，请参阅为特定数据库对象设置或重置此子句的SQL语句。

您不能在同一语句中同时指定deallocate_unused_clause和allocate_extent_clause 。

Oracle数据库仅释放高水位线以上的未使用空间（数据库块尚未格式化以接收数据的点）。Oracle从对象的末尾开始释放未使用的空间，并将对象的开头移动到高水位线。

如果扩展中完全包含扩展区，则可以释放整个扩展区以供重用。如果在重新分配中部分地包含范围，则使用到高水位线的部分成为范围，并且剩余的未使用空间被释放以供再次使用。

Oracle将释放空间量计入发生重新分配的表空间的用户配额。

释放的确切空间量取决于INITIAL ， MINEXTENTS和NEXT存储参数的值。有关这些参数的说明，请参阅storage_clause 。

保持 整数

指定在重新分配后数据库对象的段所具有的高水位线上方的字节数。

  • 如果省略KEEP并且高水位线高于INITIAL和MINEXTENTS的大小，则释放高水位MINEXTENTS所有未使用空间。当在高水位小于大小INITIAL或MINEXTENTS ，那么上述所有未使用的空间MINEXTENTS被释放。

  • 如果指定KEEP ，则保留指定的空间量并释放剩余空间。当剩余的区数小于MINEXTENTS ，Oracle MINEXTENTS调整为新的区数。如果初始范围变得小于INITIAL ，则Oracle会将INITIAL调整为新大小。

  • 在任何一种情况下，Oracle都会将NEXT存储参数的值设置为已解除分配的最后一个范围的大小。
