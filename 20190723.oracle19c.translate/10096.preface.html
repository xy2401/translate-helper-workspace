本文档的用例场景

使用此方案文档可帮助您将早期版本的非CDB转换为在不同的endian操作系统上运行的PDB。转换完成后，您可以将该PDB插入Oracle Database 19c。

此方案适用于以下非CDB：

  • Oracle Database 11g第2版（11.2.0.3）

  • Oracle Database 11g第2版（11.2.0.4）

  • Oracle Database 12c第1版（12.1.0.1）

  • Oracle Database 12c第1版（12.1.0.2）

  • Oracle Database 12c第2版（12.2.0.1）

此方案的先决条件

  • 源非CDB和目标CDB主机必须使用兼容的字符集和国家字符集。

  • 您已使用多租户架构作为配置选项在目标CDB主机上创建和配置了Oracle Database 19c，以便您拥有CDB和默认PDB。

  • 您准备将源非CDB上的用户定义表空间置于只读模式。

此场景的大纲

  1. 传输数据库 。将数据库传输到新平台

  2. 将数据转换为目标操作系统的Endian格式 。转换您的endian格式，然后将数据转换为新平台。

  3. 连接到Oracle数据库

  4. 迁移Oracle数据库 。将数据库迁移到目标多租户体系结构主机上的PDB。

这些步骤对应于本文档中的章节。
