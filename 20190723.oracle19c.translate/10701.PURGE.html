清除

目的

使用PURGE语句：

  • 从回收站中删除表或索引，并释放与该对象关联的所有空间

  • 从回收站中删除部分或全部已删除的表空间或表空间集

  • 删除整个回收站

注意：

您无法回滚PURGE语句，也无法在清除对象后恢复它。

要查看回收站的内容，请查询USER_RECYCLEBIN数据字典视图。您可以改用RECYCLEBIN同义词。以下两个语句返回相同的行：

SELECT * FROM RECYCLEBIN; SELECT * FROM USER_RECYCLEBIN;

也可以看看：

  • “Oracle数据库管理员指南” ，了解有关回收站中对象的回收站和命名约定的信息

  • FLASHBACK TABLE，用于从回收站检索已删除表的信息

  • 有关使用RECYCLEBIN初始化参数控制已删除表是否进入回收站的信息，请RECYCLEBIN Oracle数据库参考

先决条件

要清除表，该表必须驻留在您自己的模式中，或者您必须具有DROP ANY TABLE系统特权，或者您必须具有SYSDBA系统特权。

要清除索引，索引必须驻留在您自己的架构中，或者您必须具有DROP ANY INDEX系统特权，或者您必须具有SYSDBA系统特权。

要清除表空间或表空间集，您必须具有DROP TABLESPACE系统特权，或者您必须具有SYSDBA系统特权。

要清除表空间集，还必须以SDB用户身份连接到分片目录数据库。

要执行PURGE DBA_RECYCLEBIN操作，您必须具有SYSDBA或PURGE DBA_RECYCLEBIN系统特权。

语义

表或索引

在回收站中指定要清除的表或索引的名称。您可以指定原始用户指定的名称或在删除对象时分配给该对象的系统生成的名称Oracle Database。

  • 如果指定用户指定的名称，并且回收站包含多个具有该名称的对象，则数据库将清除回收站中最长的对象。

  • 系统生成的回收站对象名称是唯一的。因此，如果指定系统生成的名称，则数据库将清除该指定的对象。

当数据库清除表时，也会清除该表的所有表分区，LOB和LOB分区，索引以及其他依赖对象。

TABLESPACE或TABLESPACE SET

使用此子句清除驻留在指定表空间中的所有对象或从回收站设置的表空间。

USER 用户

使用此子句可回收为指定用户设置的表空间或表空间中的空间。当特定用户的指定表空间或表空间集的磁盘配额不足时，此操作很有用。

RECYCLEBIN

使用此子句清除当前用户的回收站。Oracle数据库将从用户的回收站中删除所有对象，并释放与回收站中的对象关联的所有空间。

DBA_RECYCLEBIN

仅当您具有SYSDBA或PURGE DBA_RECYCLEBIN系统特权时，此子句才有效。它允许您从系统范围的回收站中删除所有对象，相当于清除每个用户的回收站。例如，在向后迁移之前，此操作很有用。

例子

从回收站中删除文件：示例

以下语句从回收站中删除表test 。如果多个版本的测试驻留在回收站中，则Oracle数据库将删除最长的版本：

PURGE TABLE测试;

要确定要从回收站中删除的表的系统生成名称，请在回收站上发出SELECT语句。使用该对象名称，可以通过发出类似于以下语句的语句来删除该表。（系统生成的名称将与示例中显示的名称不同。） 

PURGE TABLE RB $$ 33750 $ TABLE $ 0;

删除回收站的内容：示例

要删除回收站的全部内容，请发出以下语句：

PURGE RECYCLEBIN;
