11 Spatial Web Services简介

本章介绍Oracle Spatial和Graph对空间Web服务的支持。

Web服务使Oracle Spatial和Graph应用程序的开发人员能够通过Web向其应用程序用户提供功能数据和元数据。

注意：

如果您使用空间和图形Web要素服务（WFS）或Web服务目录服务（CSW）支持，并且如果您拥有使用一个或多个SYS.XMLTABLEINDEX索引编制索引的先前版本的数据，则必须删除关联的在升级之前指数重新创建升级后的索引。

有关更多信息，请参阅升级前后的索引维护（WFS和CSW） 。

11.1空间Web服务的类型

Oracle Spatial and Graph提供以下类型的Web服务。

  • 地理编码，使用户能够将空间位置（经度和纬度坐标）与邮政地址相关联。地理编码地址数据中解释了地理编码支持。

  • 黄页，使用户能够根据他们与某个位置的关系，按名称或类别查找商家。商业目录（黄页）支持中解释了黄页支持 。

  • 路由，提供单个或多个路由的驾驶信息和说明。路由引擎中解释了路由支持。

  • OpenLS，提供基于开放式位置服务计划（OpenLS）规范的基于位置的服务，用于地理编码，映射，路由和黄页。OpenLS支持中解释了OpenLS支持 。

  • Web要素服务（WFS），使用户能够根据与某个位置或非空间属性的关系来查找要素（道路，河流等）。Web功能服务（WFS）支持中介绍了WFS支持 。

  • Web Coverage Services（WCS），提供对客户端呈现有用的表单中的coverage数据的访问，作为科学模型和其他客户端的输入。WCS支持在Web Coverage Service（WCS）支持中进行了解释 。

    有关WCS的概述，请参阅https://en.wikipedia.org/wiki/Web_Coverage_Service 。有关WCS与相关Web服务的介绍性比较，请参阅http://gis.stackexchange.com/questions/80948/what-are-the-differences-between-wms-wfs-wcs-wps 。

  • Web服务目录服务（CSW），描述了目录服务的Open GIS Consortium规范的Oracle Spatial and Graph实现。根据此规范：“目录服务支持发布和搜索数据，服务和相关信息对象的描述性信息（元数据）集合的能力。” 在Web服务目录服务（CSW）支持中解释了CSW支持 。

  • Web地图服务（WMS），支持空间数据的渲染。具体而言，WMS 1.1.1和1.3.0实现规范在Map Visualization Component中实现。在Oracle Spatial和Graph Map Visualization Developer's Guide 中的Map Visualization Component中的OGC WMS支持中描述了此实现。

11.2空间Web服务的用户类型

在“用户”一词的一般商业意义上，实现任何空间Web服务应用程序涉及以下类型的人。

  • 管理员设置Web服务基础结构。管理员可以为新的和现有的数据库用户创建数据库用户，授予权限和访问权限，以及执行影响多个数据库用户的其他操作。

    • 对于Web要素服务，管理员可以使用WFS管理控制台注册要素表并发布要素类型。

    • 对于Web服务的目录服务，管理员可以使用CSW Admin Console发布记录类型。

    • 对于Web覆盖服务，管理员可以使用WCS管理控制台发布coverage。

    例如，管理员可以设置基础结构以允许访问空间要素，例如道路和河流。

  • 应用程序开发人员创建和管理空间数据和元数据。它们创建空间数据表，创建空间索引，将行插入USER_SDO_GEOM_METADATA视图，并使用空间函数和过程来实现应用程序逻辑。

    例如，应用程序开发人员可以创建道路和河流表，并实现应用程序逻辑，使最终用户能够根据空间查询条件查找道路和河流。

  • 最终用户使用KVP，POST或SOAP协议通过HTTP请求访问服务。

    例如，最终用户可能会要求在特定河流的一英里范围内或与该河流相交（交叉）的所有道路。

从管理员的角度来看，应用程序开发人员和最终用户都是“用户”，因为必须创建数据库用户以满足他们的需求。应用程序开发人员将以具有足够权限的用户身份连接到数据库，以创建和管理空间表并使用Oracle Spatial and Graph功能和过程。最终用户将通过HTTP请求访问数据库。

有关Spatial和Graph Web服务的章节是为管理员和应用程序开发人员编写的，而不是为最终用户编写的。

11.3部署和配置Spatial Web服务

本主题描述适用于部署和配置空间Web服务的操作，尤其是WFS，WCS和CSW。

这些服务作为Java Web应用程序实现，可以部署为在WebLogic 12.1.3或更高版本上运行。所需的Java版本是JDK 1.8或更高版本。

  • WFS，CSW和CSW打包在sdows.ear文件中。

  • Geocoder服务打包在geocoder.ear.zip文件中。

  • 路由引擎打包在routeserver.ear.zip文件中。

除了本主题中的“常规”说明之外，请参阅有关计划用于任何其他部署和配置任务的每个特定空间Web服务的章节。

部署任何Oracle Spatial和Graph Web服务包括以下主要任务。

  1. 准备WebLogic Server（版本12.1.3或更高版本）

  2. 在WebLogic Server上创建域

  3. 在WebLogic Server上配置服务器启动参数

  4. 在WebLogic Server上部署Spatial Web Services

  5. 确保Web Service Web应用程序处于活动状态

  6. 配置每个空间Web服务

准备WebLogic Server（版本12.1.3或更高版本）

在部署Web服务引擎之前，建议您在WebLogic Server中创建受管服务器。

要在受管服务器上成功部署Web Service Engine，必须创建WebLogic域。

在WebLogic Server上创建域

您必须确保WebLogic Server上存在Web服务的域。若要创建空间域，请按照下列步骤操作。

  1. 登录WebLogic Server控制台。

  2. 选择“ 创建新的WebLogic域” ，然后单击“ 下一步” 。

  3. 选择“ 域来源：生成自动配置的域” ，然后单击“ 下一步” 。

  4. 配置管理员用户名和密码，然后单击“ 下一步” 。

  5. 对于服务器启动模式 ，选择开发或生产 。

  6. 对于JDK ，请选择一个可用的JDK。

  7. 对于“ 自定义环境和服务设置” ，请接受默认值或指定任何自定义项。

  8. 在Create WebLogic Domain页面上，单击Create ，然后单击Next 。

  9. 在“创建域”页面上，单击“完成” 。

有关创建和配置域的详细信息，请参阅WebLogic Server文档。

在WebLogic Server上配置服务器启动参数

Spatial Web Services Web应用程序需要一个文件夹来保存配置文件和日志文件。此文件夹作为名为sdows的Java参数传递给WebLogic Server。如果您正在运行节点管理器，请按照以下步骤使用WebLogic Server管理控制台配置此参数：

  1. 登录WLS控制台。

  2. 在环境 - 服务器中选择受管 服务器 。

  3. 在Configuration选项卡上，选择Server Start 。

  4. 将-Dsdows=<configuration folder path>添加到Arguments ，其中<configuration folder path>是可写文件夹的路径，其中将存储配置文件和日志文件。

如果未运行节点管理器，请编辑脚本编辑脚本<WLS domain> \ bin \ setDomainEnv。搜索set JAVA_PROPERTIES ，并添加到该行的末尾：

-Dsdows = <配置文件夹路径>

在WebLogic Server上部署Spatial Web Services

若要在WebLogic Server上部署空间Web服务，请按照下列步骤操作。

  1. 登录WLS控制台

  2. 单击“ 部署” ，然后单击“ 安装” 。

  3. 选择sdows.ear （文件），然后单击“ 下一步” 。

  4. 确保选中“ 将此部署安装为应用程序目标样式”，然后单击“ 下一步” 。

  5. 在要部署WFS引擎的潜在服务器列表中，选择您创建的受管服务器的名称，选择“ 我将从以下位置访问部署” ，然后单击“ 下一步” 。

  6. 确保部署名称为sdows ，然后单击“ 完成” 。

确保Web服务Web应用程序处于活动状态

完成空间Web服务的必要步骤后，请在“部署”页面上检查应用程序是否处于“ Active状态。

如果它处于“ Prepared状态，请单击“ 启动”以启动该应用程序。

配置每个空间Web服务

下一步是单独配置要使用的每个空间Web服务（WFS，WCS或CSW）。您必须执行特定任务，这些任务取决于您将支持在您的环境中使用哪些Web服务。您可能需要为数据库用户创建和授予权限。您可能需要下载和加载特殊数据（例如地理编码），修改配置文件或在WebLogic Server中创建数据源。

有关该服务的特定说明，请参阅每个相关空间Web服务的章节。

11.4空间Web服务管理控制台

Oracle Spatial and Graph提供了一个Spatial Web Services管理控制台，该控制台由几个组件组成，可帮助配置WFS，WCS和CSW Web服务引擎，测试每个服务以及显示错误日志以进行诊断。它在sdows.ear文件中提供。

要访问管理控制台，请按以下格式转到URL：

HTTP：// <系统名称>：<端口> / oraclespatial /

在使用任何“服务”页面之前，请从所有可用数据源名称列表中选择数据源。（当前选择的数据源显示在右上角，您可以随时在其中进行更改。）

  • 使用“ 配置文件”选项卡可以修改服务参数，例如GetCapabilities响应中显示的日志记录级别或服务提供者信息。

  • WFS，WCS和CSW的测试选项卡允许您为不同的操作创建简单的请求，您可以编辑这些请求以添加或修改参数，然后作为HTTP POST / XML请求发送。还会显示回复。

  • WFS，WCS和CSW的日志选项卡允许您显示每个服务的日志文件的内容。每个“日志”选项卡还允许您下载压缩为zip文件的日志，以便以后可以使用该信息来诊断问题。

控制台页面和WFS，WCS和CSW在有关这些服务的章节中有更详细的描述。
