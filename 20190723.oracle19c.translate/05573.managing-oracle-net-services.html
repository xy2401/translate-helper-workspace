7.1.1使用Oracle Enterprise Manager云控制配置Oracle Net Services

Oracle Enterprise Manager Cloud Control使您能够跨多个文件系统为任何Oracle主目录配置Oracle Net Services。它还为听众提供了通用的管理功能。Oracle Enterprise Manager Cloud Control提供了一个用于配置和管理Oracle Net Services的集成环境。

您可以使用Oracle Enterprise Manager Cloud Control从多个Oracle主目录配置和管理以下内容：

  • 监听器：配置监听器以接收客户端连接。

  • 命名：定义连接标识符并将它们映射到连接描述符以标识服务的网络位置。Oracle Net Manager支持在本地tnsnames.ora文件或集中式目录服务中配置连接描述符。

  • 文件位置：指定Oracle Net配置文件的文件位置。

  • 访问网络服务管理页面

7.1.1.1访问网络服务管理页面

以下过程介绍了如何使用Oracle Enterprise Manager云控制访问“网络服务管理”页面：

  1. 从“登录到数据库”页面，输入数据库凭据，然后单击“ 登录” 。

    将显示“选择企业管理器主页”。

  2. 从“目标”菜单中选择“ 所有目标 ”。

  3. 从“优化搜索”列表中选择“ 侦听器 ”。

  4. 双击侦听器名称以选择侦听器。

  5. 从Oracle Listener菜单中选择Net Services Administration 。

    将显示“网络服务管理”页面。

在“网络服务管理”页面中，您可以管理侦听器，命名方法，首选项等。管理程序在本书的其他章节中有所描述。

7.1.2使用Oracle Net Manager配置Oracle Net Services

Oracle Net Manager使您能够在本地客户端或服务器主机上为Oracle主目录配置Oracle Net Services。

您可以使用Oracle Net Manager配置以下网络组件：

  • 监听器：创建和配置监听器以接收客户端连接。

  • 命名：定义连接标识符并将它们映射到连接描述符，以标识网络位置和服务标识。Oracle Net Manager支持在本地tnsnames.ora文件或集中式目录服务中配置连接描述符。

  • 命名方法：配置解析连接标识符的方式以连接描述符。

  • 配置文件：配置在客户端或服务器上启用和配置Oracle Net功能的首选项。

7.1.3决定何时使用Oracle Enterprise Manager云控制和Oracle Net Manager

之前仅在Oracle Net Manager中可用的许多功能已与Oracle企业管理器云控制集成。Oracle Enterprise Manager Cloud Control提供跨多个文件系统管理多个Oracle主目录配置的功能。Oracle Net Manager仅允许您管理本地主机上的一个Oracle主目录的配置。以下是这些工具之间的主要区别。

Oracle企业管理器云控制

  • 配置以下功能：

    - 本地命名（ tnsnames.ora文件）

    - 目录命名

    - 听众

  • 跨多个文件系统提供Oracle主页支持

  • 提供搜索和排序本地和目录命名条目的功能

  • 将目录命名条目导出到tnsnames.ora文件

  • 为选定的侦听器执行以下管理任务：

    - 显示当前状态

    - 改变状态

    - 更改跟踪级别设置

    - 更改日志记录设置

    - 当有多个侦听器时，设置连接时故障转移和负载平衡方法

Oracle Net Manager

  • 配置以下功能：

    - 本地命名（ tnsnames.ora文件）

    - 目录命名

    - 听众

    - 个人资料

  • 为单个主机提供Oracle主页支持

  • 当有多个侦听器时，设置连接时故障转移和负载平衡方法

  • 为客户端和服务器设置以下选项：

    - 跟踪设置

    - 记录设置

    - 安全，身份验证和访问权限

    - 路由

注意：

启用自动诊断存储库（ADR）后，系统将忽略使用Oracle Enterprise Manager Cloud Control对跟踪和日志记录设置所做的任何更改。

父主题： 管理Oracle Net Services

7.2.1将用户添加到OracleNetAdmins组

以下过程介绍如何使用ldapmodify命令将用户添加到OracleNetAdmins组：

  1. 创建一个LDIF （轻量级目录交换格式）文件，指定要将用户添加到OracleNetAdmins组。

    您可以使用以下示例LDIF文件。对cn=OracleNetAdmins和要添加的用户使用适当的DN。 

    dn：cn = OracleNetAdmins，cn = OracleContext，...changetype：modify add：uniquemember uniquemember： DN of user being added to group

  2. 在命令行中输入以下命令以刷新文件：

    $ ldapmodify -h directory_host -p port -D binddn -q -f ldif_file

    在上面的命令中， directory_host是目录服务器主机， port是目录服务器的侦听TCP / IP端口， binddn是目录管理员或用户DN， ldif_file是输入文件名。如果未指定端口，则使用默认端口389。-q选项提示输入单个绑定密码。

父主题： 关于OracleNetAdmins组

7.2.2从OracleNetAdmins组中删除用户

以下过程介绍了如何使用ldapmodify命令从OracleNetAdmins组中删除用户：

  1. 创建一个LDIF文件，指定要将用户删除到OracleNetAdmins组。

    您可以使用以下示例LDIF文件。输入cn=OracleNetAdmins的适当DN以及要删除的用户。 

    dn：cn = OracleNetAdmins，cn = OracleContext，...changetype：modify delete：uniquemember uniquemember： DN of user being deleted from group

  2. 输入以下命令以删除用户：

    $ ldapmodify -h directory_host -p port -D binddn -q -f ldif_file

    在上面的命令中， directory_host是目录服务器主机， port是目录服务器的侦听TCP / IP端口， binddn是目录管理员或用户DN， ldif_file是输入文件名。如果未指定端口，则使用默认端口389。-q选项提示输入单个绑定密码。

父主题： 关于OracleNetAdmins组

7.2.3更改OracleNetAdmins组的所有权

以下过程介绍如何将组添加为OracleNetAdmins组的所有者：

  1. 创建LDIF文件，如下所示：

    1. 指定要添加为所有者的组。

      您可以使用以下示例LDIF文件。输入cn=OracleNetAdmins的相应DN以及要添加的组的DN。 

      dn：cn = OracleNetAdmins，cn = OracleContext，...changetype：modify add：owner owner： DN of group to add

      例如，以下LDIF语法将所有权从OracleNetAdmins组更改为另一个名为ExampleSecurityAdmins组。该组可以在Oracle Context内部或外部。 

      dn：cn = OracleNetAdmins，cn = OracleContext，...changetype：modify add：owner owner：cn = ExampleSecurityAdmins

    2. （可选）指定要作为所有者删除的组。

      dn：cn = OracleNetAdmins，cn = OracleContext，...changetype：modify delete：owner owner： DN of group to delete

  2. 在命令行中输入以下命令以刷新文件：

    $ ldapmodify -h directory_host -p port -D binddn -q -f ldif_file

    在上面的命令中， directory_host是目录服务器主机， port是目录服务器的侦听TCP / IP端口， binddn是目录管理员或用户DN， ldif_file是输入文件名。如果未指定端口，则使用默认端口389。-q选项提示输入单个绑定密码。

7.3使用监听器控制实用程序管理监听器

Oracle Net Services提供的工具可帮助您启动，停止，配置和控制每个网络组件。Listener Control实用程序使您可以管理侦听器。该实用程序由拥有Oracle安装的用户或指定组的成员在运行侦听器的同一台计算机上启动。该实用程序的基本语法如下：

lsnrctl command [ listener_name ]

例如，以下命令启动名为lsnr的侦听lsnr ：

lsnrctl START lsnr

您还可以在LSNRCTL>程序提示符下发出监听器控制实用程序命令。要获取提示，请在操作系统命令行中输入不带参数的lsnrctl 。运行lsnrctl ，将启动该实用程序，您可以从程序提示符输入必要的命令。

例如：

lsnrctl LSNRCTL> START lsnr

  • 有关监听器控制实用程序的其他信息，请参阅Oracle数据库网络服务参考
