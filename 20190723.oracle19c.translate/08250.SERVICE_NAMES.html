1.303 SERVICE_NAMES

SERVICE_NAMES指定客户端可以通过其连接到实例的一个或多个名称。

属性 描述

参数类型

串

句法

SERVICE_NAMES =

db_service_name [, db_service_name [ ...]]

默认值

如果已定义，则为DB_UNIQUE_NAME.DB_DOMAIN

可修改

改变系统

可在PDB中修改

没有

价值范围

任何ASCII字符串或以逗号分隔的字符串名称列表

基本

没有

Oracle RAC

不要为Oracle RAC环境设置SERVICE_NAMES参数。相反，使用Oracle Enterprise Manager定义服务并使用服务器控制（SRVCTL）实用程序管理这些服务。

该实例使用侦听器注册其服务名称。当客户端请求服务时，侦听器确定哪些实例提供所请求的服务并将客户端路由到适当的实例。

您可以指定多个服务名称以区分同一数据库的不同用途。例如：

SERVICE_NAMES = sales.example.com，widgetsales.example.com

您还可以使用服务名称来标识通过使用复制从两个不同数据库中获得的单个服务。

如果不使用域限定此参数中的名称，Oracle会使用DB_DOMAIN参数的值限定它们。如果未指定DB_DOMAIN ，则不会将域应用于非限定SERVICE_NAMES值。

使用此参数指定其他服务名称时，不会覆盖默认服务名称。默认服务名称加上使用此参数指定的其他服务名称是客户端可用于连接到数据库的服务名称。
