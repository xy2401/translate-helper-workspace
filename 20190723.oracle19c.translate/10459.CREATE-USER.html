创建用户

目的

使用CREATE USER语句创建和配置数据库用户 ，该用户可以通过该帐户登录数据库，并建立Oracle数据库允许用户访问的方法。

您可以在Oracle自动存储管理（Oracle ASM）群集中发出此语句，以将用户和密码组合添加到当前节点的Oracle ASM实例本地的密码文件中。每个节点的Oracle ASM实例都可以使用此语句更新自己的密码文件。密码文件本身必须由ORAPWD实用程序创建。

您可以使用户通过代理应用程序或应用程序服务器连接到数据库。有关语法和讨论，请参阅ALTER USER 。

先决条件

您必须具有CREATE USER系统特权。使用CREATE USER语句CREATE USER ，用户的权限域为空。要登录Oracle数据库，用户必须具有CREATE SESSION系统特权。因此，在创建用户之后，您应该至少为用户授予CREATE SESSION系统特权。有关更多信息，请参阅GRANT 。

只有经过身份验证的AS SYSASM用户才能发出此命令来修改Oracle ASM实例密码文件。

要指定CONTAINER子句，必须连接到多租户容器数据库（CDB）。要指定CONTAINER = ALL ，当前容器必须是根。要指定CONTAINER = CURRENT ，当前容器必须是可插拔数据库（PDB）。

语义

用户

指定要创建的用户的名称。此名称只能包含数据库字符集中的字符，并且必须遵循“ 数据库对象命名规则 ”一节中描述的规则 。Oracle建议用户名包含至少一个单字节字符，无论数据库字符集是否还包含多字节字符。

在非CDB中，用户名不能以C##或c##开头。

在CDB中，用户名的要求如下：

  • 公共用户的名称必须以与COMMON_USER_PREFIX初始化参数指定的前缀不区分大小写的字符开头。默认情况下，前缀为C## 。

  • 本地用户的名称不得以与COMMON_USER_PREFIX初始化参数指定的前缀不区分大小写的字符开头。无论COMMON_USER_PREFIX的值COMMON_USER_PREFIX ，本地用户的名称都不能以C##或c##开头。

注意：

如果COMMON_USER_PREFIX的值是空字符串，则对常见或本地用户名没有要求，但有一个例外：本地用户的名称永远不能以C##或c##开头。Oracle建议不要使用空字符串值，因为当PDB插入不同的CDB时，或者打开在创建普通用户时关闭的PDB时，它可能会导致本地用户名和普通用户名之间发生冲突。

注意：

Oracle建议用户名和密码仅以ASCII或EBCDIC字符编码，具体取决于您的平台。

身份证明条款

IDENTIFIED子句允许您指示Oracle数据库如何对用户进行身份验证。

通过密码

BY password子句允许您创建本地用户并指示用户必须指定password才能登录数据库。密码区分大小写。用于将此用户连接到数据库的任何后续CONNECT字符串必须使用此CREATE USER语句或后续ALTER USER语句中使用的相同大小写（上限，下限或混合）来指定密码。密码可以包含来自数据库字符集的任何单字节，多字节或特殊字符或这些字符的任意组合，但双引号（“）和返回字符除外。如果密码以非字母字符开头，或者包含字母数字字符以外的字符，下划线（_），美元符号（$）或井号（＃），则必须用双引号括起来。否则，将密码括在双引号中是可选的。

[HTTP] DIGEST条款

此子句允许您为用户ENABLE或DISABLE HTTP摘要访问身份验证。默认为DISABLE 。

HTTP关键字是可选的，并且是为了语义清晰度而提供的。

对[HTTP] DIGEST条款的限制

您不能为外部或全局用户指定此子句。

外部条款

指定EXTERNALLY以创建外部用户。 这样的用户必须由外部服务（例如操作系统或第三方服务）进行身份验证。在这种情况下，Oracle数据库依赖于操作系统或第三方服务的身份验证，以确保特定外部用户可以访问特定数据库用户。

AS 'certificate_DN'

此子句仅用于SSL身份验证的外部用户，并且仅用于此身份。certificate_DN是用户钱包中用户PKI证书中的可分辨名称。certificate_DN的最大长度为1024个字符。

AS 'kerberos_principal_name'

此子句仅用于Kerberos身份验证的外部用户并且仅用于Kerberos身份验证的外部用户。kerberos_principal_name的最大长度为1024个字符。

注意：

Oracle强烈建议您不要将IDENTIFIED EXTERNALLY用于具有固有弱登录安全性的操作系统。

全球条款

GLOBALLY子句允许您创建全局用户 。此类用户必须由企业目录服务（Oracle Internet Directory）授权。

directory_DN字符串可以采用以下两种形式之一：

  • 企业目录服务中标识此用户的X.509名称。它应该是CN= username,other_attributes形式，其中other_attributes是目录中用户的可分辨名称（DN）的其余部分。此表单使用LDAP数据交换格式（LDIF）并创建私有全局模式 。

  • 一个空字符串（''），表示企业目录服务将经过身份验证的全局用户映射到具有相应角色的此数据库模式。此表单与仅指定GLOBALLY关键字相同，并创建共享全局模式 。

directory_DN的最大长度为1024个字符。

您可以控制应用程序服务器作为指定用户进行连接的能力，并使用ALTER USER语句激活该用户的角色。

没有认证条款

使用NO AUTHENTICATION子句创建没有密码且无法登录的模式。这适用于仅模式帐户，并通过删除默认密码和旋转密码的任何要求来减少维护。

默认收集条款

此子句允许您为用户拥有的架构指定默认排序规则。默认排序规则分配给随后在架构中创建的表，视图和物化视图。

对于collation_name ，请指定有效的命名排序规则或伪排序规则。

如果省略此子句，则用户拥有的架构的默认排序规则将设置为USING_NLS_COMP伪排序规则。

您可以通过为表，实例化视图或视图指定CREATE或ALTER语句的DEFAULT COLLATION子句来覆盖此子句并为特定表，物化视图或视图指定不同的默认排序规则。您还可以通过设置会话的默认排序规则来覆盖数据库会话期间所有模式的默认排序规则。有关更多详细信息，请参阅ALTER SESSION的DEFAULT_COLLATION子句。

仅当COMPATIBLE初始化参数设置为12.2或更高，并且MAX_STRING_SIZE初始化参数设置为EXTENDED ，才能指定DEFAULT COLLATION子句。

DEFAULT TABLESPACE条款

为在用户架构中创建的对象指定默认表空间。如果省略此子句，则用户的对象将存储在数据库缺省表空间中。如果没有为数据库指定默认表空间，则用户的对象存储在SYSTEM表空间中。

对默认表空间的限制

您不能将本地管理的临时表空间（包括撤消表空间或字典管理的临时表空间）指定为用户的默认表空间。

[LOCAL] TEMPORARY TABLESPACE条款

为用户的临时段指定表空间或表空间组。如果省略此子句，则用户的临时段将存储在数据库缺省临时表空间中，如果未指定，则存储在SYSTEM表空间中。

  • 指定tablespace以指示用户的临时表空间。指定TEMPORARY TABLESPACE以指示共享临时表空间。指定LOCAL TEMPORARY TABLESPACE以指示本地临时表空间。如果您连接到CDB，则可以指定CDB$DEFAULT以使用CDB范围的默认临时表空间。

  • 指定tablespace_group_name以指示用户可以在tablespace_group_name指定的tablespace_group_name空间组中的任何表空间中保存临时段。本地临时表空间不能是表空间组的一部分。

临时表空间的限制

本条款受以下限制：

  • 表空间必须是临时表空间，并且必须具有标准块大小。

  • 表空间不能是撤消表空间或具有自动段空间管理的表空间。

QUOTA条款

使用QUOTA子句指定用户可在表空间中分配的最大空间量。

CREATE USER语句可以为多个表空间提供多个QUOTA子句。

UNLIMITED允许用户在没有绑定的情况下在表空间中分配空间。

您可以指定的最大空间量为2 TB（TB）。如果您需要更多空间，请指定UNLIMITED 。

对QUOTA条款的限制

您不能为临时表空间指定此子句。

简介条款

指定要分配给用户的配置文件。该配置文件限制了用户可以使用的数据库资源量。如果省略此子句，则Oracle Database会将DEFAULT配置文件分配给用户。

授予和创建资料

PASSWORD EXPIRE条款

如果希望用户密码过期，请指定PASSWORD EXPIRE 。此设置强制用户或DBA在用户登录数据库之前更改密码。

帐户条款

指定ACCOUNT LOCK以锁定用户的帐户并禁用访问权限。指定ACCOUNT UNLOCK以解锁用户的帐户并启用对帐户的访问权限。默认值为ACCOUNT UNLOCK 。

启用版本

该条款不可逆转。指定ENABLE EDITIONS以允许用户使用版本在此架构中创建多个版本的可编辑对象。未启用版本的模式中的可版本化对象无法编辑。

启用版本的限制

除了种子数据库中的示例模式之外，您无法为Oracle提供的任何模式启用版本。

集装箱条款

当您连接到CDB时， CONTAINER子句适用。但是，没有必要指定CONTAINER子句，因为它的默认值是唯一允许的值。

  • 要创建普通用户，您必须连接到根。您可以选择指定CONTAINER = ALL ，这是连接到根目录时的默认值。

  • 要创建本地用户，您必须连接到PDB。您可以选择指定CONTAINER = CURRENT ，这是连接到PDB时的默认值。

创建公共用户时，使用以下子句指定的任何缺省表空间，临时表空间或概要文件必须存在于属于CDB的所有容器中：

  • DEFAULT TABLESPACE

  • TEMPORARY TABLESPACE

  • 配额

  • 轮廓

如果所有容器中都不存在这些对象，则CREATE USER语句将失败。

例子

以下所有示例都使用example表空间，该表空间存在于种子数据库中，并且可供示例模式访问。

创建数据库用户：示例

如果使用PASSWORD EXPIRE创建新用户，则必须在用户尝试登录数据库之前更改用户密码。您可以通过发出以下语句来创建用户sidney ：

CREATE USER sidney通过out_standing1 DEFAULT TABLESPACE标识示例QUOTA 10M ON示例TEMPORARY TABLESPACE temp QUOTA 5M ON系统PROFILE app_user PASSWORD EXPIRE;

用户sidney具有以下特征：

  • 密码out_standing1

  • 默认表空间example ，配额为10兆字节

  • 临时表空间temp

  • 访问表空间SYSTEM ，配额为5兆字节

  • 对配置文件app_user定义的数据库资源的限制（在“ 创建配置文件：示例 ”中创建 ）

  • 过期的密码，必须在sidney登录数据库之前更改

创建外部数据库用户：示例

以下示例创建一个外部用户，在访问数据库之前必须由外部源标识：

CREATE USER app_user1 IDENTIFIED EXTERNALLY DEFAULT TABLESPACE示例QUOTA 5M ON示例PROFILE app_user;

用户app_user1具有以下附加特征：

  • 默认表空间example

  • 默认临时表空间example

  • 表空间example上的5M空间和数据库的临时表空间上的无限制配额

  • 对app_user配置文件定义的数据库资源的限制

要创建仅由操作系统帐户访问的另一个用户，请在用户名前添加初始化参数OS_AUTHENT_PREFIX的值。例如，如果此值为“ ops$ ”，则可以使用以下语句创建外部标识的用户external_user ：

CREATE USER ops $ external_user IDENTIFIED EXTERNALLY DEFAULT TABLESPACE示例QUOTA 5M ON示例PROFILE app_user;

创建全局数据库用户：示例

以下示例创建一个全局用户。创建全局用户时，可以指定在企业目录服务器上标识此用户的X.509名称：

CREATE USER global_user全局识别'CN =分析师，OU = division1，O = oracle，C = US'DEFAULT TABLESPACE示例QUOTA 5M ON示例;

在CDB中创建通用用户

以下示例在CDB中创建名为c ## comm_user的公共用户。在运行此CREATE USER语句之前，请确保表空间example和temp_tbs存在于CDB中的所有容器中。

CREATE USER c ## comm_user IDENTIFIED BY comm_pwd DEFAULT TABLESPACE示例QUOTA 20M ON示例TEMPORARY TABLESPACE temp_tbs;

用户comm_user具有以下附加特征：

  • 密码comm_pwd

  • 默认表空间example ，配额为20兆字节

  • 临时表空间temp_tbs
