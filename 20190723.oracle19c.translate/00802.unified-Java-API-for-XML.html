10用于XML的Unified Java API

提出了可扩展标记语言（XML）的统一Java应用程序接口（API）。描述了针对Oracle XML DB和Oracle XML Developer's Kit（XDK）统一的API。

适用于XML的Unified Java API概述

使用Unified Java API for XML，您可以使用Oracle XML DB和XDK所需的核心Java DOM API。您还可以使用新的Java类，它们提供基于Java DOM API构建的额外功能。

Unified Java API for XML结合了Oracle XML DB和XDK所需的功能。 Oracle XML DB使用Java包oracle.xdb.dom实现Java文档对象模型（DOM）API，XDK使用oracle.xml.parser.v2包实现它。

您可以使用Unified Java API，无论您的XML数据位于何处（在数据库内或其外部），因为Unified Java API使用会话池模型进行连接管理。如果未将连接类型指定为thick（使用OCI API且基于C）或thin（使用Java数据库连接（JDBC）API且纯粹基于Java），则使用Java DOM API进行连接到驻留在数据库外部的本地文档对象。

组件统一

在Unified Java API for XML中统一了仅由粗连接或仅由瘦连接支持的某些组件。

仅由瘦连接支持并已统一的那些包括以下内容：

  • DOM解析器

  • 用于XML处理的Java API（JAXP）Transformer

  • XML SQL实用程序（XSU）

  • 可扩展样式表语言转换（XSLT）

关于迁移到Unified Java API

Unified Java API提供了替换旧的oracle.xdb.dom Java类的新Java类。不推荐使用oracle.xdb.dom包中的所有类。如果您使用的是已弃用的类，则必须迁移到Unified Java API并使用oracle.xml.parser.v2类。

扩展API

除了W3C建议书之外，用于XML实现的Unified Java API还提供了扩展W3C DOM API的扩展API。您可以使用特定于Oracle的扩展API来执行基本功能（如连接到数据库）和性能增强。

XMLDocument是一个表示实例化XML文档的DOM的类。使用带有Document参数的XMLType构造函数从XML文档中检索XMLType值。例如：

XMLType createXML（Connection conn，Document domdoc）

取消引用节点手动的，也就是说，显式解引用从DOM树使用的文档片段freeNode()的扩展API的oracle.xml.parser.v2包（ XMLNode类）。

文档创建Java API

创建XMLDocument Java API必须创建精简文档或厚文档。由于厚文档需要Connection对象来与数据库建立通信，因此每个文档创建API都会扩展为接受Connection对象。

对于XMLType.createXML API，必须指定Connection类型，该类型确定对象的类型。旧文档创建API（仅为了向后兼容性而提供），除非另行指定，否则创建瘦（纯Java）对象。

这些对象，方法和类可用于统一Java API中的文档创建：

  • DOMParser对象和parse()方法

    使用DOMParser对象和parse()方法来解析XML文档。您必须指定对象类型 - 粗或细。对于厚对象，还必须使用DOMParser.setAttribute() API提供Connection类型。例如：

    DOMParser parser = new oracle.xml.parser.v2。的DOMParser（）; parser.setAttribute（为XMLDocument。KIND，XMLDocument。厚）; parser.setAttribute（为XMLDocument。连接，conn）;

  • DocumentBuilder对象和DocumentBuilderFactory类

    使用DocumentBuilder对象使用特定于Java的API（JAXP）解析XML文档。

    您必须使用DocumentBuilderFactory类创建DOM解析器工厂。DocumentBuilder从输入SAX事件构建DOM，从DocumentBuilderFactory上的属性集中获取Connection。例如：

    DocumentBuilderFactory.setAttribute（为XMLDocument。连接，conn）; DocumentBuilderFactory.setAttribute（为XMLDocument。善良，为XMLDocument。厚）;

    DocumentBuilderFactory将连接传递给DOMParser对象，该对象从这些API创建文档：

    DocumentBuilder.newDocument（）DocumentBuilder解析（InputStream）DocumentBuilder解析（InputStream，int）DocumentBuilder.parse（InputSource）

  • XSU方法

    每个XSU方法都将XMLDocument返回给用户。您可以指定用户是想要厚对象还是瘦对象。例如：

    OracleXMLUtil util = new OracleXMLUtil（...）; util.setAttribute（为XMLDocument。KIND，XMLDocument。厚）; util.setAttribute（为XMLDocument。连接，conn）; Document doc = util.getXMLDOMFromStruct（struct，enc）; OracleXMLQuery query = new OracleXMLQuery（...）; query.setAttribute（为XMLDocument。KIND，XMLDocument。厚）; query.setAttribute（为XMLDocument。连接，conn）; Document doc = query.getXMLDOM（root，meta）; OracleXMLDocGenDOM dgd = new OracleXMLDocGenDOM（...）; dgd.setAttribute（为XMLDocument。KIND，XMLDocument。厚）; dgd.setAttribute（为XMLDocument。连接，conn）; Document doc = dgd.getXMLDocumentDOM（struct，enc）;

  • XMLType方法

    您可以使用XMLType.createXML方法指定希望getDocument()方法获取的文档类型（thin或thick）。在此示例中，从OPAQUE推断出连接：

    XMLType xt = XMLType.createXML（orset.getOPAQUE（1），XMLDocument。厚）; Document doc = xt.getDocument（）;

注意：

您不能指定ResultSet.getObject()返回的XMLType对象的类型。对象类型由JDBC调用中使用的Connection确定，该调用从XMLType列获取ResultSet 。
