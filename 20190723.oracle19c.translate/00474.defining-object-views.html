6.2定义对象视图

对象视图允许您仅显示希望用户查看的数据。

定义对象视图的过程是：

  1. 定义对象类型，其中类型的每个属性对应于关系表中的现有列。
  2. 编写一个查询，指定如何从关系表中提取数据。以与对象类型中的属性相同的顺序指定列。
  3. 根据基础数据的属性指定唯一值作为对象标识符，使您能够为视图中的对象创建指针（ REF ）。您通常可以使用现有的主键。

    也可以看看：

    “ 用于标识行对象的对象标识符 ”

要更新对象视图，其中对象类型的属性与现有表中的列不完全对应，您可能需要执行以下操作：

编写一个INSTEAD OF触发器过程，以便Oracle在应用程序尝试更新对象视图中的数据时执行。请参见“ 对象视图操作 ” 。

完成这些步骤后，您可以像对象表一样使用对象视图。

示例6-1包含用于定义对象视图的SQL语句，其中视图中的每一行都是employee_t类型的对象：

示例6-1创建对象视图

CREATE TABLE emp_table（empnum NUMBER（5），ename VARCHAR2（20），salary NUMBER（9,2），job VARCHAR2（20））; CREATE TYPE employee_t AS OBJECT（empno NUMBER（5），ename VARCHAR2（20），salary NUMBER（9,2），job VARCHAR2（20））; / CREATE VIEW emp_view1 OF employee_t WITH OBJECT IDENTIFIER （empno）AS SELECT e.empnum，e.ename，e.salary，e.job FROM emp_table e WHERE job ='Developer';插入emp_table值（1，'John'，1000.00，'Architect'）;插入emp_table值（2，'Robert'，900.00，'开发者'）;插入emp_table值（3，'James'，2000.00，'Director'）; select * from emp_view1; EMPNO ENAME SALARY JOB ---------- -------------------- ---------- ------ -------------- 2 Robert 900开发者

要从关系表的empnum列访问数据，请访问对象类型的empno属性。
