1.289 RESOURCE_MANAGER_CPU_ALLOCATION

RESOURCE_MANAGER_CPU_ALLOCATION指定资源管理器应使用的CPU数。 资源管理器控制数据库会话如何使用系统的CPU。
属性 描述

参数类型

整数

默认值

操作系统报告的逻辑CPU数。

可修改

改变系统

可在PDB中修改

没有

价值范围

0到特定于操作系统

基本

没有

注意：

不推荐使用RESOURCE_MANAGER_CPU_ALLOCATION参数。它仅为了向后兼容而保留。请改用CPU_COUNT参数。

资源管理器根据DBA配置和启用的资源计划在CPU上调度数据库会话。通常，资源管理器会调度足够的数据库会话以保持所有CPU的使用。但是，在某些情况下，DBA可能只希望安排足够的数据库会话以保留所使用的CPU的子集。
