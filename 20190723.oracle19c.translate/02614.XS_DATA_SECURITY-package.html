XS_DATA_SECURITY包

XS_DATA_SECURITY包包括创建，管理和删除数据安全策略，关联数据领域约束，列约束和ACL参数的过程。

本节包括以下主题：

  • 安全模型

  • 对象类型，构造函数，同义词和授权

  • XS_DATA_SECURITY子程序摘要

XS_DATA_SECURITY包的安全模型

XS_DATA_SECURITY包是在SYS架构下创建的。DBA角色被授予ADMIN_ANY_SEC_POLICY ，允许它管理所有模式中的ACL，安全类和安全策略等模式对象。此外，授予ADMIN_ANY_SEC_POLICY用户可以调用以下过程： ENABLE_OBJECT_POLICY ， DISABLE_OBJECT_POLICY ， APPLY_OBJECT_POLICY和REMOVE_OBJECT_POLICY 。

如果已为模式授予RESOURCE角色，则用户可以在自己的模式中管理模式对象。RESOURCE角色和XS_RESOURCE应用程序角色包括管理模式中的模式对象所需的ADMIN_SEC_POLICY权限，以及在授予的模式中管理策略工件以在应用程序中实现策略管理。

如果已授予APPLY_SEC_POLICY权限，则用户可以管理架构上的策略实施。使用此权限，用户可以在授予的模式中管理策略实施，以在应用程序内实现策略管理。

CREATE_POLICY过程

CREATE_POLICY过程创建新的数据安全策略。

句法

XS_DATA_SECURITY.CREATE_POLICY（名称IN VARCHAR2，realm_constraint_list IN XS $ REALM_CONSTRAINT_LIST，column_constraint_list IN XS $ COLUMN_CONSTRAINT_LIST：= NULL，description IN VARCHAR2：= NULL）;

参数

参数 描述

名称

要创建的数据安全策略的名称。

该名称是模式限定的，例如， SCOTT.POLICY1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为POLICY1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.POLICY1 。

DELETE_ACL_PARAMETER过程

DELETE_ACL_PARAMETER过程删除数据安全策略的ACL参数。

句法

XS_DATA_SECURITY.DELETE_ACL_PARAMETER（策略IN VARCHAR2，参数IN VARCHAR2，delete_option IN PLS_INTEGER：= XS_ADMIN_UTIL.DEFAULT_OPTION）;

参数

参数 描述

政策

要删除ACL参数的数据安全策略的名称。

该名称是模式限定的，例如， SCOTT.POLICY1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为POLICY1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.POLICY1 。

参数

要删除的ACL参数的名称。

delete_option

要使用的删除选项。可以使用以下选项：

  • DEFAULT_OPTION （默认）：

    默认选项仅允许在其他地方未引用ACL参数时删除它。如果有其他实体引用ACL参数，则无法删除ACL参数。

  • CASCADE_OPTION ：

    级联选项删除ACL参数以及对它的任何引用。删除安全类的用户也必须具有删除这些引用的权限。

  • ALLOW_INCONSISTENCIES_OPTION：

    allow inconsistencies选项允许您删除实体，即使其他实体具有对它的后期绑定引用。如果实体是早期依赖项的一部分，则删除失败并引发错误。

例子

以下示例使用default选项从EMPLOYEES_DS数据安全策略中删除DEPT_POLICY ACL参数。 

BEGIN SYS.XS_DATA_SECURITY.DELETE_ACL_PARAMETER（'EMPLOYEES_DS'，'DEPT_POLICY'，XS_ADMIN_UTIL.DEFAULT_OPTION）;结束;

REMOVE_OBJECT_POLICY过程从指定的表或视图中删除数据安全策略而不删除它。REMOVE_OBJECT_POLICY删除由ENABLE_XDS在静态数据领域约束上构建的ACL物化视图。

要查找当前用户可用的表或视图的数据安全策略的状态，请查询DBA_XS_APPLIED_POLICIES数据字典视图。

在实施策略之前，将检查APPLY_SEC_POLICY权限。

句法

XS_DATA_SECURITY.REMOVE_OBJECT_POLICY（策略IN VARCHAR2，模式IN VARCHAR2，对象IN VARCHAR2）;

参数

参数 描述

政策

要删除的数据安全策略的名称。

该名称是模式限定的，例如， SCOTT.POLICY1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为POLICY1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.POLICY1 。

模式

包含要从中删除数据安全策略的表或视图的架构的名称。

宾语

要从中删除数据安全性的表或视图的名称。政策

例子

以下示例从sales架构中的products表中删除CUST_DS数据安全策略。 

BEGIN SYS.XS_DATA_SECURITY.REMOVE_OBJECT_POLICY（policy =>'CUST_DS'，schema =>'sales'，object =>'products'）;结束;

APPLY_OBJECT_POLICY过程

APPLY_OBJECT_POLICY过程为指定的数据库表或视图启用或重新启用数据安全策略。

要查找当前用户可用的表或视图的数据安全策略的状态，请查询DBA_XS_APPLIED_POLICIES数据字典视图。

在实施策略之前，将检查APPLY_SEC_POLICY权限。

句法

XS_DATA_SECURITY.APPLY_OBJECT_POLICY（策略IN VARCHAR2，模式IN VARCHAR2，对象IN VARCHAR2，row_acl IN BOOLEAN DEFAULT FALSE，owner_bypass IN BOOLEAN DEFAULT FALSE，statement_types IN VARCHAR2 DEFAULT NULL，aclmv IN VARCHAR2 DEFAULT NULL）;

参数

参数 描述

政策

要启用的数据安全策略的名称。

该名称是模式限定的，例如， SCOTT.POLICY1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为POLICY1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.POLICY1 。

模式

包含要启用或重新启用的关系表或视图的架构的名称。

宾语

用于启用或重新启用数据安全策略的关系表或视图的名称。

row_acl

默认值为FALSE 。设置为TRUE ，创建隐藏列SYS_ACLOD 。

owner_bypass

对象的所有者可以绕过数据安全策略。默认值为FALSE 。

statement_types

类型可以是： SELECT ， INSERT ， UPDATE ， DELETE和INDEX 。

请注意，如果您的应用程序安全性要求您必须更新表行并限制对同一表中某些列的读访问权限，则必须使用两个APPLY_OBJECT_POLICY过程来强制执行每个数据安全策略，以确保每个策略的精确实施。例如，一个APPLY_OBJECT_POLICY过程将强制更新表行所需的DML statement_types （例如， INSERT ， UPDATE ， DELETE ），而另一个APPLY_OBJECT_POLICY过程将仅对列约束强制执行SELECT的statement_types 。

aclmv

指定用户提供的MV名称，用于维护静态ACL信息。MV有两列： TABLEROWID和ACLIDLIST 。 aclmv的默认值为NULL 。

例子

以下示例为HR模式中的EMP表启用DEPT_POLICY数据安全策略。 

BEGIN sys.xs_data_security.apply_object_policy（policy =>'HR.EMPLOYEES_DS'，schema =>'HR'，object =>'EMPLOYEES'，statement_types =>'SELECT'，owner_bypass => true）;结束;
