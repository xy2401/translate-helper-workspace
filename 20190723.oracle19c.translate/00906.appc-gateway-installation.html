在你开始之前

配置在线事务处理器以允许网关访问需要对OLTP和主机操作系统的某些组件执行操作。虽然主机系统上未安装Oracle软件，但仍需要访问主机系统和OLTP以及一些知识。虽然以下主题包括有关主机系统和OLTP安装步骤的一些信息，但您必须确保具有适用的OLTP和主机系统文档。

OLTP上的某些配置操作可能需要您重新启动OLTP。为此做准备，请让主机系统程序员或DBA查看OLTP的说明，以便进行必要的准备工作。

使用单个安装和配置网关 Oracle数据库和单个OLTP，执行以下主题中描述的过程。

注意：

如果您的网关使用SNA通信协议 ，请按照本章， 配置Oracle网络和使用SNA通信协议的网关配置中的安装和配置说明进行操作。

如果您的网关使用TCP / IP通信协议 ，请按照本章， 配置Oracle网络和使用TCP / IP通信协议的网关配置中的安装和配置说明进行操作。

计划升级或迁移网关

如果您的系统上安装了早期版本的Oracle Database Gateway for APPC，那么您可能正在升级或迁移到当前版本。网关升级代表版本中的次要软件升级（例如，从版本9.0迁移到版本9.2.0）网关迁移表示从一个版本号到另一个版本号的重大更改（例如，从版本9.0迁移到10.2版） ）。

本节仅适用于具有以前版本的Oracle Database Gateway for APPC的客户。如果您之前安装了网关，则需要先执行特定任务，然后才能安装适用于APPC 12 c第2版（12.2）的Oracle数据库网关。

  • 阅读本节后，请阅读从现有网关迁移，以确定准备升级或迁移网关时必须采取的具体操作。如果要从版本4.01或更早版本迁移到Oracle Database Gateway for APPC 12 c Release 2（12.2），那么您将找到与从现有网关迁移中的网关迁移相关的特定材料。

如果您是第一次安装，请从“ 执行预安装过程 ”开始 。

执行升级前程序

执行以下步骤以准备将Oracle Database Gateway for APPC升级到当前版本：

  1. 备份已更改的PGA已发送文件。
  2. 删除或重命名任何旧网关目录。

升级和迁移注意事项

升级注意事项如下：

  • PGAU Gateway Release 8或9中的控制文件是向上兼容的，您无需更改它们。

  • 升级后，PG DD 包含所有早期条目而不进行修改。添加了新的PGAU控制信息以及一些列以支持新功能，但升级不会更改任何客户条目。

  • 所有由于以下内容的更改，必须重新编译来自Oracle Database Gateway for APPC版本4.0.1或更早版本的TIP：

    • PL / SQL兼容性

    • 网关服务器RPC接口

    • UTL_PG接口

  • 仅限TCP / IP： 如果您之前使用SNA通信协议在网关上生成了现有的TIP，并且您希望使用新的TCP / IP功能，则必须通过PGAU使用强制NLS_LANGUAGE和Side Profile Settings重新生成TIP。在DEFINE TRANSACTION命令中指定适当的ASCII字符集。

    这是因为网关假定正在使用IMS Connect中的适当用户出口，这将在适当的ASCII和EBCDIC字符集之间进行转换。

    警告：

    升级了早期版本的PGAU无法访问PG数据字典（PG DD）。

恢复

如果你想恢复以前版本的网关，然后您必须将以下组件恢复到以前的版本：

  • PGAU

  • PG DD

  • 网关服务器

执行预安装程序

在安装网关之前，请执行以下预安装程序：

  • 确保您的系统满足系统要求中指定的所有硬件和软件要求 。

  • 确保满足您的安全要求。

    有关OLTP上的连接和数据访问的安全要求的详细信息，请参阅系统要求 。

  • 填写工作表，确定配置系统和所选通信协议（SNA或TCP / IP）所需的唯一参数名称，该通信协议位于配置工作表中 。

  • 确定网关的SID（系统标识符）。 此SID用于配置网关 配置网关 。

    SID必须是唯一的，不得由任何其他网关使用系统上的Oracle数据库。

  • 仅限SNA ：必须先安装和配置SNA软件包，然后才能继续安装网关。确保您的系统可以使用平台所需的SNA Server与OLTP进行通信。有关设置和配置SNA for Windows的详细信息，请参阅在Windows上配置SNA通信包 。

  • 仅限TCP / IP ：您的必须先安装并配置TCP / IP包，然后才能继续安装网关。

    确保您的系统可以使用适用于您的平台的TCP / IP通信包与OLTP进行通信。

如果您需要有关安装Oracle产品和使用Oracle Universal Installer的一般信息，请参阅适用于Microsoft Windows的Oracle数据库安装指南

网关安装方法

您可以通过以下任一方式安装网关：

  1. 与现有系统相同 Oracle数据库但在不同的目录中。

    本节将讨论此类安装或升级的所有任务。

  2. 在与本地Oracle数据库不同的系统上。
  3. 与Oracle数据库位于同一系统上，并位于同一Oracle主目录中。请注意，在这种情况下，Oracle数据库和网关必须处于相同的发行版级别。

安装网关软件

有关安装Oracle产品以及如何使用Oracle Universal Installer的一般信息，请参阅适用于Microsoft Windows的Oracle数据库安装指南并首先执行所有必要的任务。

如果您的服务器版本是不是你的网关版本不同，则不要安装在同一个Oracle主目录作为Oracle数据库网关。如果网关可执行文件与更高版本的Oracle数据库库重新链接，则需要将网关与可能导致不兼容的Oracle数据库升级隔离开来。

将Windows用户帐户用作Oracle Home用户

在Windows中，您以具有管理员权限的用户身份登录以安装Oracle数据库软件。您还可以在安装期间指定Oracle Home User（基于低权限，非管理用户帐户）。

以下是Windows用户帐户：

  • Windows本地用户帐户

  • Windows域用户帐户

  • Windows托管服务帐户（MSA）

  • Windows内置帐户

也可以看看：

“ 适用于Microsoft Windows的Oracle数据库平台指南 ”中的“在Windows上使用Oracle Home用户”

安装步骤

如果您希望将网关安装在与Oracle数据库相同的Oracle主目录中，则两个产品的版本号必须相同。

第1步：登录到Windows系统

以Administrators组成员的身份登录Windows系统。如果您当前不是DBA用户，请与系统管理员联系以创建DBA登录用户ID。有关登录信息，请参阅“适用于Microsoft Windows的Oracle数据库安装指南” 。

第2步：确保最小磁盘空间量

如果您是第一次安装网关，请确保网络所在的磁盘上有足够的空间，如“ 磁盘空间要求 ”中所述 。

第3步：停止所有Oracle服务

在开始网关安装过程之前，必须停止当前正在运行的所有Oracle服务。跟着这些步骤：

  1. 单击开始 ，然后单击设置 ，然后单击控制面板
  2. 选择服务 。将显示所有Windows服务的列表。
  3. 选择Oracle服务（这些服务以Oracle开头）。
  4. 单击停止 。
  5. 继续选择并停止Oracle服务，直到停止所有活动的Oracle服务。

步骤4：插入Gateway产品安装介质

验证是否已将驱动器分配给您选择的逻辑驱动器，以及是否可以访问安装介质上的文件。

注意：

以下安装步骤假定安装介质位置已映射到D：驱动器。

安装介质包中包含适用于APPC的Oracle数据库网关和Oracle Universal Installer。

步骤5：启动Oracle Universal Installer

要启动Oracle Universal Installer，请运行setup.exe ：

  1. 从“开始”菜单中，选择“运行” 。
  2. 输入可执行文件名的路径。例如：
    d：\ DISK1 \ SETUP.EXE

第6步：逐步完成Oracle Universal Installer

注意：

Oracle Universal Installer自动安装Oracle提供的Java Runtime Environment（JRE）版本。运行Oracle Universal Installer和多个Oracle助手需要此版本。除非使用Oracle支持服务提供的修补程序，否则请勿修改JRE。Oracle Universal Installer还安装了JDK。

Oracle Universal Installer是一个菜单驱动的实用程序，可引导您通过提示安装网关行动项目。操作项及其显示顺序取决于您的平台。

默认文件名是InstallActions YYYY-MM-DD_HH-mm-SS-AM/PM .log ，其中：

  • YYYY是一年;
  • MM是月
  • DD是白天
  • HH是小时
  • mm是分钟
  • SS是秒
  • AM/PM是白天或晚上

日志文件名中的每个变量都代表产品的安装日期和时间。

删除适用于APPC的Oracle数据库网关

以下主题描述如何从Oracle主目录中删除Oracle Database Gateway。它包含有关以下主题的信息：

  • 删除Oracle软件

关于卸载工具

安装前，安装介质中提供了卸载工具（ deinstall ），安装后可在Oracle主目录中使用。它位于ORACLE_HOME\deinstall 。

deinstall命令停止Oracle软件，并删除操作系统上的Oracle软件和配置文件。

该脚本使用以下语法，其中变量内容以斜体表示：

deinstall -home home的完整路径 [-silent] [-checkonly] [-local] [-paramfile 输入参数属性文件的完整路径 ] [-params name1 = value name2 = value 。 。 。] [-o 保存文件的目录的完整路径 ] [-help | -H]

选项是：

  • -无声

    使用此标志以非交互模式运行命令。此选项需要一个属性文件，其中包含正在卸载或取消配置的Oracle主目录的配置值。

    要创建属性文件并提供所需参数，请参阅位于响应文件夹中的模板文件deinstall.rsp.tmpl, 。如果您愿意，可以使用-checkonly选项生成属性文件，以使deconfig从Oracle主目录中发现要卸载和取消配置的信息，而不是使用模板文件。该工具将生成属性文件，然后您可以使用-silent选项。

  • -checkonly

    使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行该命令不会删除Oracle配置。

  • -本地

    在多节点环境中使用此标志可在群集中取消配置Oracle软件。

    使用此标志运行deconfig时，它会在本地节点（运行deconfig的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

  • -paramfile complete path of input parameter property file

    使用此标志可以使用默认位置以外的参数文件运行deconfig。使用此标志时，请提供参数文件所在的完整路径。

    参数文件的默认位置取决于deconfig的位置：

    • 从安装介质或阶段位置： ORACLE_HOME\response

    • 来自Oracle Technology Network的解压缩归档文件： ziplocation\response

    • 从已安装的Oracle主目录安装后： ORACLE_HOME\deinstall\response

  • -params [name1= value name 2= value name3= value . . .]

    将此标志与参数文件一起使用，以覆盖要在已创建的参数文件中更改的一个或多个值。

  • -o complete path of directory for saving files

    使用此标志提供除保存属性文件的默认位置以外的路径。默认位置为\ response\deinstall.rsp.tmpl 。

    参数文件的默认位置取决于deconfig的位置：

    • 安装前从安装介质或阶段位置： ORACLE_HOME\

    • 来自Oracle Technology Network的解压缩归档文件： \ziplocation\response\

    • 从已安装的Oracle主目录安装后： ORACLE_HOME/deinstall/response

  • -help | -H

    使用help选项（ -help或-h ）获取有关可选标志的其他信息

删除Oracle软件

完成以下过程以删除Oracle软件：

  1. 以Administrators组成员的身份登录。
  2. 出现提示时，运行deinstall命令，提供有关Oracle系统标识符（SID）的信息。
