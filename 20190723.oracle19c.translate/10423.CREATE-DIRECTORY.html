创建目录

目的

使用CREATE DIRECTORY语句创建目录对象。目录对象指定服务器文件系统上的目录的别名，外部二进制文件LOB（ BFILE ）和外部表数据位于该目录中。在PL / SQL代码和OCI调用中引用BFILE时，可以使用目录名，而不是硬编码操作系统路径名，以提高管理灵活性。

所有目录都在单个命名空间中创建，并且不属于单个模式。通过将目录的对象权限授予特定用户，可以保护对存储在目录结构中的BFILE的访问。

先决条件

您必须具有CREATE ANY DIRECTORY系统特权才能创建目录。

创建目录时，会自动为该目录授予READ ， WRITE和EXECUTE对象权限，您可以将这些权限授予其他用户和角色。DBA还可以将这些权限授予其他用户和角色。

对目录的WRITE权限与外部表有关。它们让被授权者确定外部表代理是否可以将日志文件或错误文件写入目录。

对于文件存储，还必须创建相应的操作系统目录，Oracle自动存储管理（Oracle ASM）磁盘组或Oracle ASM磁盘组中的目录。您的系统或数据库管理员必须确保操作系统目录对Oracle数据库进程具有正确的读写权限。

为目录授予的权限是独立于为操作系统目录定义的权限而创建的，两者可能完全对应也可能不完全对应。例如，如果示例用户hr被授予目录对象的READ权限但相应的操作系统目录没有为Oracle数据库进程定义的READ权限，则会发生错误。

限制

打开BFILE对象时，目录对象路径或文件名中不允许使用符号链接。检查整个目录路径和文件名，如果找到任何符号链接，则返回以下错误：

ORA-22288：文件或LOB操作FILEOPEN路径中的软链接失败

解决方法

如果您尝试打开的数据库目录对象或文件名包含符号链接，请将其更改为提供实际路径和文件名。

语义

或替换

指定OR REPLACE以重新创建目录数据库对象（如果已存在）。您可以使用此子句更改现有目录的定义，而无需删除，重新创建和重新授予以前在目录上授予的数据库对象权限。

以前被授予重新定义的目录权限的用户仍然可以访问该目录，而无需重新授予权限。

共享

仅当在应用程序根目录中创建目录时，此子句才适用。这种类型的目录称为应用程序公共对象，它可以与属于应用程序根目录的应用程序PDB共享。要确定共享目录的方式，请指定以下共享属性之一：

  • METADATA - 元数据链接共享目录的元数据，但其数据对每个容器都是唯一的。这种类型的目录称为元数据链接的应用程序公共对象 。

  • NONE - 目录未共享。

如果省略此子句，则数据库使用DEFAULT_SHARING初始化参数的值来确定目录的共享属性。如果DEFAULT_SHARING初始化参数没有值，则默认值为METADATA 。

创建目录后，无法更改目录的共享属性。

目录

指定要创建的目录对象的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

Oracle数据库不会验证您指定的目录是否确实存在。因此，请注意在操作系统中指定有效目录。此外，如果操作系统使用区分大小写的路径名，请确保以正确的格式指定目录。您不需要在路径名的末尾包含尾部斜杠。

不要引用目录名中的父目录。例如，以下语法有效：

CREATE DIRECTORY mydir AS'/ scratch / data / file_data';

但是，以下语法无效：

CREATE DIRECTORY mydir AS'/ scratch /../ file_data';

路径名

指定文件所在服务器的操作系统目录的完整路径名。单引号是必需的，结果是路径名称区分大小写。

例子

创建目录：示例

以下语句创建一个指向服务器上的目录的目录数据库对象：

CREATE DIRECTORY admin AS'/ disk1 / oracle / admin';

以下语句重新定义目录数据库对象bfile_dir以启用对存储在操作系统目录/ usr/bin/bfile_dir中的BFILE的访问：

创建或替换目录bfile_dir AS'/ usr / bin / bfile_dir';
