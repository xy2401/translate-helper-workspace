DROP EDITION

目的

使用DROP EDITION语句删除版本及其包含的所有实际可编辑对象。实际可编辑对象是在版本中创建或修改的可编辑对象。

也可以看看：

CREATE EDITION以获取可编辑对象类型的列表

先决条件

您必须具有直接或通过角色授予的DROP ANY EDITION系统特权。

句法

drop_edition :: =


说明drop_edition.eps的描述

语义

成功后，此语句将删除指定的版本，包括与该版本关联的任何对象的版本。不删除其他版本中相同对象的版本。不会删除不可编辑或可编辑但尚未在当前版本中实现的对象。

如果指定的版本包含任何实际的可编辑对象，则必须指定CASCADE 。

本声明受以下条件和限制的约束：

  • 指定版本不能同时具有父版本和子版本。

  • 如果您尝试删除默认版本，则DROP EDITION将失败。

  • 如果您尝试删除根版本并且回收站包含至少一个曾在该版本中被删除的对象，则DROP EDITION将失败。在这种情况下，甚至DROP EDITION CASCADE也会失败。在这种情况下，您可以使用PURGE DBA_RECYCLEBIN语句从回收站中清除所有对象，然后删除该版本。有关更多信息，请参阅PURGE 。

    如果您尝试删除叶版本并且回收站包含至少一个在删除之前在该版本中的对象，则DROP EDITION也将失败。但是，在这些情况下， DROP EDITION CASCADE将成功。

    可能位于回收站中的唯一类型的已编辑对象是触发器。
