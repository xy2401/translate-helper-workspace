acfsutil标签信息

目的

显示Oracle ACFS文件系统中标记目录或文件名的标记名称。

语法和说明

acfsutil标签信息-h acfsutil标签信息[-r] [-c -t 标记名 ] 路径 [ 路径 ...]acfsutil标签信息[-c -t 标记名 ]

acfsutil tag info -h显示帮助文本和退出。

表16-27包含acfsutil tag info命令可用的选项。

表16-27 acfsutil tag info命令的选项

选项 描述

-t tagname

指定要显示的标记名称字符串。如果字符串包含空格，则将字符串括在引号中。

路径

指定一个或多个文件或目录的路径名。

-r

指定指定路径上的递归操作。

-C

指定标记名称的不区分大小写的部分匹配。

如果-r选项包含在指定的目录路径名中，则acfsutil tag info命令可以递归地将操作应用于所有子文件和子目录。

如果未指定路径名，则acfsutil tag info命令将在所有Oracle ACFS装入点中搜索已标记的文件。

仅显示Oracle ACFS文件系统中已标记文件的路径。如果包含-t tagname选项，则仅显示标记名称与指定tagname名完全匹配的文件路径。如果包含-c选项，则tagname可以是成功匹配的标记名称的不区分大小写的子字符串。例如， acfsutil tag info -c -t AG命令将显示具有标记名称（如tag1 ， ag或AG的文件的路径名。

任何用户都可以使用此命令在用户具有读访问权限的目录上显示标记信息。

在受安全领域保护的文件或目录上运行acfsutil tag info失败。如果用户在受安全领域保护的目录上使用-r选项运行acfsutil tag info ，并且用户对目录没有浏览权限，则不显示任何输出。要解决此问题，必须将用户添加到安全领域并授予READDIR权限。有关将对象添加到Oracle ACFS安全领域的信息，请参阅“ acfsutil sec realm add ” 。

acfsutil标签集

目的

将给定标记添加到Oracle ACFS文件系统中的指定文件或目录

语法和说明

acfsutil标记集-h acfsutil标记集[-v] [-r] 标记名 路径 [ path ...]

acfsutil tag set -h显示帮助文本和退出。

表16-28包含acfsutil tag set命令可用的选项。

表16-28 acfsutil tag set命令的选项

选项 描述

标签名称

指定标记名称字符串。如果字符串包含空格，则将字符串括在引号中。

标记字符串可以由ASCII字符组成，包括：ab，AZ，0-9，空格字符，连字符和下划线。标签名称的最大长度为32个字符。

all是保留的标记名称，不能在任何文件或目录上设置。

路径

指定一个或多个文件或目录的路径字符串。

-r

指定指定路径上的递归操作。

-v

显示操作的进度。

该命令可以递归地将操作应用于指定目录路径名的所有子文件和子目录。

将标记名称添加到文件或目录时，该文件或目录上的现有标记保持不变。在为目录分配标记后创建的新文件隐式继承其父目录中的标记。目录中的现有文件不继承新标记;必须为这些文件明确分配标记。重命名文件或将文件移动到同一文件系统中的子目录不会导致文件从新的父目录继承标记名称。在文件系统之间移动文件使用复制操作，新创建的文件继承父目录的标记名称。

Oracle ACFS文件系统或每个文件和目录没有固定的最大标记名称数。但是，标记名称的数量受标记名称的限制，每个文件和目录最多可容纳64 KB的特殊元数据存储。此元数据还包含用于管理扩展属性的信息。较长的标记名称会减少可在文件或目录上设置的标记名称的总数。例如，如果所有标记名称都是4个字节长，那么假设不存在其他扩展属性信息，则可以保存大约1730个标记名称。

任何有权修改目标文件或目录的用户都可以运行此命令。

如果用户运行acfsutil tag set与-r对安全领域保护的目录选项，并没有对目录浏览权限，则没有标签被应用。要解决此问题，必须将用户添加到安全领域并授予READDIR权限。有关将对象添加到Oracle ACFS安全领域的信息，请参阅“ acfsutil sec realm add ” 。

例子

以下示例显示了acfsutil tag set命令的acfsutil 。

示例16-25使用acfsutil标记set命令

$ / sbin / acfsutil标签设置repl_grp1 -r /acfsmounts/acfs1/myrepfiles/*.dat

acfsutil标签未设置

目的

从指定的文件或目录中删除给定的标记名称或所有标记名称。

语法和说明

acfsutil tag unset -h acfsutil tag unset [-v] [-r] {all | tagname } path [ path ...]

acfsutil tag unset -h显示帮助文本并退出。

表16-29包含acfsutil tag unset命令可用的选项。

表16-29 acfsutil tag unset命令的选项

选项 描述

所有

指定删除路径字符串中的所有标记。

标签名称

指定标记名称字符串。如果字符串包含空格，则将字符串括在引号中。

路径

指定一个或多个文件或目录的路径字符串。

-r

指定指定路径上的递归操作。

-v

显示操作的进度。

该命令可以递归地将操作应用于指定目录路径名的所有子文件和子目录。

从文件或目录中删除标记名称时，该文件或目录上的其他现有标记保持不变。

任何有权修改目标文件或目录的用户都可以运行此命令。

如果用户在受安全领域保护的目录上运行带-r选项的acfsutil tag unset且对该目录没有浏览权限，则不会删除任何标记。要解决此问题，必须将用户添加到安全领域并授予READDIR权限。有关将对象添加到Oracle ACFS安全领域的信息，请参阅“ acfsutil sec realm add ” 。
