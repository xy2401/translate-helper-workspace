使用响应文件配置软件二进制文件

在任何节点上安装或复制Oracle Grid Infrastructure软件时，可以将配置推迟到以后进行。使用配置向导（ gridSetup.sh ）在节点上安装或复制软件后，请查看此过程以完成配置。

要使用响应文件配置Oracle Grid Infrastructure软件二进制文件：

  1. 作为Oracle Grid Infrastructure安装所有者（ grid ），使用以下语法从Oracle Grid Infrastructure纯软件版本的Oracle Grid Infrastructure配置向导模式启动Oracle Universal Installer，其中filename是响应文件名：

    /u01/app/19.0.0/grid/gridSetup.sh [-debug] [-silent -responseFile filename ]

    例如：

    $ cd /u01/app/19.0.0/grid $ ./gridSetup.sh -responseFile /u01/app/grid/response/response_file.rsp

    配置脚本以配置向导模式启动Oracle Universal Installer。每个页面显示相同的用户界面，并执行OUI通常执行的相同验证检查。但是，配置向导模式不是运行安装，而是验证输入并在所有集群节点上配置安装。

  2. 完成配置值后，OUI会显示“摘要”页面，其中列出了您为群集提供的所有信息。验证摘要是否具有群集的正确信息，然后单击“ 安装”以开始配置本地节点。

    完成本地节点的配置后，OUI会将Oracle Grid Infrastructure配置文件复制到其他集群成员节点。

  3. 出现提示时，运行根脚本。

  4. 确认所有根脚本都已运行后，OUI将检查群集配置状态，并根据需要启动其他配置工具。
