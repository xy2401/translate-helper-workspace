在群集节点上启用SSH用户等效性

将包含所有密钥的authorized_keys文件复制到群集中的每个节点后，请完成以下过程。

在此示例中，Oracle Grid Infrastructure软件所有者名为grid 。
请执行下列操作：
  1. 在要运行OUI的系统上，以grid用户身份登录。
  2. 使用以下命令语法，其中hostname1 ， hostname2等是群集中节点的公共主机名（别名和完全限定域名），以便从本地节点到每个节点运行SSH，包括从本地节点到本身，从每个节点到每个节点：
    [grid @ nodename] $ ssh hostname1 date [grid @ nodename] $ ssh hostname2 date。 。 。
    在此过程结束时，应在known_hosts文件中为所有其他群集节点注册每个成员节点的公共主机名。如果您使用远程客户端连接到本地节点，并且您看到类似于“ Warning: No xauth data; using fake authentication data for X11 forwarding ”的消息，那么这意味着您的授权密钥文件配置正确，但您的SSH配置已启用X11转发。要解决此问题，请参阅设置远程显示和X11转发配置 。
  3. 在每个群集节点成员上重复步骤2。
如果您已正确配置SSH，则现在可以使用ssh或scp命令而不会提示输入密码。例如：
[grid @ node1~] $ ssh node2 date Mon Feb 26 23:34:42 UTC 2009 [grid @ node1~] $ ssh node1 date Mon Feb 26 23:34:48 UTC 2009

如果任何节点提示输入密码，请验证该节点上的~/.ssh/authorized_keys文件是否包含正确的公钥，以及您是否已创建具有相同组成员身份和ID的Oracle软件所有者。
