32.2.1 OCIBinXmlCreateReposCtxFromConn（）

创建元数据连接上下文。

目的

从指定的OCIEnv / OCISvcCtx专用OCI连接创建元数据连接上下文（ OCIBinXmlReposCtx ）。请注意，此连接专用于元数据使用。

句法

sword OCIBinXmlCreateReposCtxFromConn（OCIEnv * env，OCISvcCtx * svcctx，OCIError * err，OCIBinXmlReposCtx ** ctx）;

参数

env（IN）

环境处理。

svcctx（IN）

用于访问元数据的连接句柄。

错误（IN）

错误句柄。

ctx（OUT）

创建并返回的元数据上下文。

返回

返回-1表示错误，0表示成功。err参数包含有关错误的更多信息。

32.2.2 OCIBinXmlCreateReposCtxFromCPool（）

从指定的连接池创建元数据连接上下文。

目的

从指定的连接池创建元数据连接上下文（ OCIBinXmlReposCtx ）。只要需要来自令牌存储库的任何信息，就会使用来自连接池的连接。

句法

剑OCIBinXmlCreateReposCtxFromCPool（OCIEnv * env，OCICPool * cpool，OCIError * err，OCIBinXmlReposCtx ** ctx）;

参数

env（IN）

环境处理。

cpool（IN）

用于访问元数据的连接句柄。

错误（IN）

错误句柄。

ctx（OUT）

创建并返回的元数据上下文。

返回

返回-1表示错误，0表示成功。err参数包含有关错误的更多信息。

32.2.3 OCIBinXmlSetFormatPref（）

指定在XML文档的客户端和服务器之间传输的图像是二进制XML格式。

目的

将来，所有通信都将采用二进制XML格式。二进制XML感知应用程序可以设置它。

句法

剑OCIBinXmlSetFormatPref（xmldomdoc * doc，ub4 formattype）;

参数

doc（IN）

指向要应用首选项的domdoc的指针。

formattype（IN）

用于酸洗的格式类型。目前唯一允许的值是OCIXML_FORMATTYPE_TEXT和OCIXML_FORMATTYPE_BINXML 。

返回

返回-1表示错误，0表示成功。

相关话题

  • OCIBinXmlSetReposCtxForConn（）

32.2.4 OCIBinXmlSetReposCtxForConn（）

将数据连接与元数据连接相关联。

目的

请注意，使用专用连接时，数据连接和元数据连接的环境句柄必须相同。

句法

剑OCIBinXmlSetReposCtxForConn（OCISvcCtx * dataconn，OCIBinXmlReposCtx * reposctx）;

参数

dataconn（IN）

数据连接句柄。

reposctx（IN）

指向元数据连接的指针。

返回

返回-1表示错误，0表示成功。err参数包含有关错误的更多信息。

32.2.6 OCIXmlDbInitXmlCtx（）

初始化XML上下文。

目的

从数据库初始化XML数据的XML上下文。

句法

xmlctx * OCIXmlDbInitXmlCtx（OCIEnv * envhp，OCISvcCtx * svchp，OCIError * errhp，ocixmldbparam * params，ub4 num_params）;

参数

envhp（IN）

OCI环境处理。

svchp（IN）

OCI服务处理。

errhp（IN）

OCI错误句柄。

参数（IN）

此数组中的可选可能值是指向OCI持续时间（默认值为OCI_DURATION_SESSION ，或指向用户注册的原型回调的错误处理程序：

void（* err_handler）（sword errcode，（const OraText *）errmsg）;

err_handler的两个参数是：

（OUT）

数字错误值。

errmsg（OUT）

错误消息文本。

num_params（IN）

从params.读取的参数数量params. 如果num_params的值超过数组params的大小，则会产生意外行为。

评论

也可以看看：

使用OCI XML DB函数作为用法示例

返回

返回：

  • 一个指向结构xmlctx的指针，其中错误处理程序和回调填充了适当的值。这稍后用于所有OCI调用。

  • 如果没有可用的数据库连接，则为NULL。
