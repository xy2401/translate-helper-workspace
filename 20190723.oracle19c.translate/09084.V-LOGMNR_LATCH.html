8.52 V $ LOGMNR_LATCH

V$LOGMNR_LATCH可以与V$LATCH和V$LATCH_CHILDREN视图连接，以获取有关活动LogMiner持久会话使用的不同锁存器的统计信息。

通过首次在逻辑备用数据库上启动Data Guard SQL Apply或创建复制捕获来创建持久LogMiner会话。

柱 数据类型 描述

SESSION_ID

数

LogMiner持久会话的唯一标识符

名称

VARCHAR2（32）

闩锁名称：

  • LogMiner internal state - 标识保护LogMiner READER ， PREPARER和BUILDER进程之间通信的锁存器

  • LogMiner memory allocation - 标识保护LogMiner内所有内存分配和释放的锁存器

  • LogMiner transaction list - 标识在逻辑备用数据库上的Data Guard SQL Apply期间保护LogMiner层与其客户端之间的交互的锁存器

CHILD_ADDR

RAW（4 | 8）

子锁存对象的地址。此列与V$LATCH_CHLDREN视图中的相应ADDR列匹配

州

VARCHAR2（6）

闩锁状态：

  • UNINIT - 闩锁结构未初始化;也就是说，它当前未分配给任何LogMiner持久会话

  • READY - 持久性LogMiner会话正在使用锁存器结构

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

也可以看看：

  • “ V $ LATCH ”

  • “ V $ LATCH_CHILDREN ”
