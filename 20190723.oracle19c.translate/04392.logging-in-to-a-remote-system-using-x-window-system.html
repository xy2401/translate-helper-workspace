使用X Window系统登录远程系统

使用此过程通过登录到运行时设置禁止直接登录到图形用户界面（GUI）的远程系统来运行Oracle Universal Installer（OUI）。

OUI是一个图形用户界面（GUI）应用程序。在运行时设置阻止GUI应用程序运行的服务器上，您可以将GUI显示重定向到连接到服务器的客户端系统。

注意：

如果您以其他用户身份登录（例如， oracle或grid ），则也要为该用户重复此过程。

  1. 启动X Window系统会话。如果从PC或类似系统使用X Window System终端仿真器，则可能需要配置安全设置以允许远程主机在本地系统上显示X应用程序。
  2. 使用以下语法输入命令以使远程主机能够在本地X服务器上显示X应用程序：
    #xhost + RemoteHost

    RemoteHost是完全限定的远程主机名。例如：

    ＃xhost + somehost.example.com somehost.example.com被添加到访问控制列表中
  3. 如果不在本地系统上安装软件，请使用ssh命令连接到要安装软件的系统：
    #ssh -Y RemoteHost

    RemoteHost是完全限定的远程主机名。-Y标志（“是”）使远程X11客户端可以完全访问原始X11显示器。例如：

    #ssh -Y somehost.example.com
  4. 如果您未以root用户身份登录，并且您正在执行需要root用户权限的配置步骤，请将用户切换到root 。

注意：

有关使用X Window System进行远程登录的详细信息，请参阅X服务器文档，或与X服务器供应商或系统管理员联系。根据您使用的X服务器软件，您可能必须以不同的顺序完成任务。
