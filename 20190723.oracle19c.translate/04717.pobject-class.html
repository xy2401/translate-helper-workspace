13.23 PObject类

OCCI提供对象导航调用，使应用程序能够对对象执行以下任何操作：

  • 创建，访问，锁定，删除，复制和刷新对象

  • 获取对象的引用

此类使类型定义器能够指定类何时可以具有持久或瞬态实例。从PObject派生的类的实例是持久的或瞬态的。例如，类A那是从持久能力的继承PObject类：

A类：PObject {...}

在NULL PObject上唯一有效的方法是setName（） ， isNull（）和operator =（） 。

某些方法（如lock（））仅适用于持久实例，不适用于瞬态实例。

表13-31 PObject类使用的枚举值

属性 选项 
LockOption

  • OCCI_LOCK_WAIT指示缓存仅在获取锁定后固定对象;如果对象被另一个用户锁定，则使用此选项的pin调用将等待，直到它可以在返回给调用者之前获取锁定;相当于SELECT FOR UPDATE

  • OCCI_LOCK_NOWAIT指示缓存仅在获取锁定后固定对象;如果对象当前被另一个用户锁定，则不等待;相当于SELECT FOR UPDATE WITH NOWAIT

 
UnpinOption

  • OCCI_PINCOUNT_RESET将对象的引脚数重置为0

  • OCCI_PINCOUNT_DECR将对象的引脚数递减1

13.23.17 unpin（）

此方法取消持久化对象。在默认模式下，对象的引脚数减1。使用OCCI_PINCOUNT_RESET调用此方法时，将重置对象的引脚数。如果引脚数重置，则此方法使指向此对象的所有引用（ Ref s）无效。缓存设置有资格释放的对象，如有必要，回收内存。

句法

void unpin（UnpinOption mode = OCCI_PINCOUNT_DECR）;
参数 描述 
模式

指定UnpinOption模式或引脚数应该递减还是重置为0 。见表13-31 。有效值为OCCI_PINCOUNT_RESET和OCCI_PINCOUNT_DECR 。
