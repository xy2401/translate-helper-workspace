65 O2I-00101至O2I-00133

O2I-00101：USERID参数的值无效

原因：为USERID参数提供的值不是合法的username / password [@ databasename ]字符串。

操作：将USERID选项设置为正确的username / password或username / password @ database name组合，以便在连接到数据库时使用。如果您有OPS $帐户，则可以省略USERID选项，在这种情况下，OTT将尝试使用userid OPS $ username连接到数据库。可以在命令行或CONFIG文件中指定USERID选项。

O2I-00102：无法连接到Oracle

原因： OTT无法使用用户名，密码以及提供的数据库链接（如果适用）连接到Oracle。USERID选项值不正确，或者，如果未提供USERID，则不接受用户标识OPS $ username 。

操作：检查用户名和密码是否是最新且正确的。运行另一个连接到Oracle的程序，以验证您是否可以使用该用户名和密码进行连接。

O2I-00103：两个文件名引用INTYPE文件中的相同HFILE文件

原因： INTYPE文件中使用了两个不同的文件名来引用相同的HFILE文件，或者在命令行和INTYPE文件中使用了不同的文件名来引用相同的HFILE文件。

操作：始终使用相同的文件名引用每个HFILE文件。

O2I-00110：OTT组件O2I中的内部错误

原因： OTT的O2I组件发生内部OTT错误。

行动：联系ORACLE客户支持。

O2I-00111：无法分配内存

原因： OTT无法分配内存。

行动：联系ORACLE客户支持。

O2I-00112：无法写入HFILE文件

原因：尝试写入HFILE文件时发生操作系统错误。

操作：确保您具有写入文件的操作系统权限。另外，请确保磁盘空间可用。

O2I-00113：未指定HFILE

原因：未指定OTT生成的C声明的HFILE文件。

操作：在命令行或CONFIG文件中指定全局HFILE选项。您可以通过在描述类型的INTYPE文件条目中指定不同的HFILE来为特定类型选择不同的HFILE。如果每个INTYPE文件条目指定HFILE，并且如果HFILE中的所有类型都不需要那里未提及的类型的声明，则可以省略全局HFILE选项。由先前的OTT调用生成的INTYPE文件满足这些条件。

O2I-00114：HFILE文件名无效

原因：指定的HFILE文件名在语法上不正确。

操作：确保HFILE文件名拼写正确。

O2I-00115：打开HFILE文件时出错

原因：尝试打开HFILE文件进行写入时发生操作系统错误。

操作：确保HFILE文件名拼写正确。尝试创建或修改HFILE以确保您具有该文件的写入权限。

O2I-00116：无法关闭HFILE文件

原因：尝试关闭HFILE文件时发生操作系统错误。

操作：尝试创建或修改HFILE文件以确保您能够访问该文件。

O2I-00117：内部错误：组件O2U没有消息文件

原因：找不到内部OTT组件O2U的消息文件。OTT可能未正确安装。

行动：联系ORACLE客户支持。

O2I-00118：在数据库中找不到此用户定义的类型

原因：在数据库中找不到INTYPE文件中指定的用户定义类型。

操作：确保类型的名称拼写正确。确保在数据库中实际声明了类型。

O2I-00119：子系统报告的警告：\ n

原因： OTT调用的子系统报告了警告。此警告紧跟在“子系统报告的警告”消息之后。

操作：检查子系统报告的警告以确定其原因。

O2I-00120：子系统报告的错误：\ n

原因： OTT调用的子系统报告错误。此错误紧跟在“子系统报告的错误”消息之后。

操作：检查sybsystem报告的错误消息以确定其原因。

O2I-00121：没有为CODE选项指定值

原因：未在命令行或配置文件中指定所需的CODE选项。

操作：在命令行或配置文件中指定CODE选项。目前，支持以下CODE选项值：CODE = ANSI_C，CODE = KR_C，CODE = C

O2I-00122：INITFILE文件的文件名无效

原因：为INITFILE文件指定的文件名在语法上不正确。

操作：确保INITFILE文件的文件名拼写正确。

O2I-00123：无法关闭INITFILE文件

原因：尝试关闭INITFILE文件时发生操作系统错误。

操作：尝试使用编辑器打开INITFILE文件以确保可以访问该文件。

O2I-00124：打开INITFILE文件进行写入时出错

原因：尝试打开INITFILE文件进行写入时发生操作系统错误。

操作：确保INITFILE文件名拼写正确。尝试创建或修改INITFILE文件以确保您具有该文件的写入权限。

O2I-00125：写入INITFILE文件时出错

原因：尝试写入INITFILE文件时发生操作系统错误。

操作：确保您具有写入文件的操作系统权限。另外，请确保磁盘空间可用。

O2I-00126：您必须使用SCHEMA_NAMES = FROM_INTYPE指定INTYPE文件

原因：选项SCHEMA_NAMES = FROM_INTYPE请求将模式名称写入INTYPE文件中给出的OUTTYPE文件。但是，未指定INTYPE文件。

操作：提供INTYPE文件的名称，或指定SCHEMA_NAMES选项ALWAYS或IF_NEEDED。

O2I-00127：非法的INITFUNC名称

原因： INITFUNC函数的名称不是合法的C或C ++标识符。

操作：将INITFUNC函数的名称指定为合法的C或C ++标识符。此名称可以在INTYPE文件中给出，在命令行中给出，或者从INITFILE名称派生。

O2I-00128：无法写入Java文件

原因：尝试写入Java源文件时发生操作系统错误。

操作：确保您具有写入文件的操作系统权限。另外，请确保磁盘空间可用。

O2I-00129：未指定Java文件

原因：未指定写入OTT生成的Java声明的文件。

操作：永远不应该看到此消息，因为Java文件是在其中声明的类之后命名的。联系ORACLE客户支持。

O2I-00130：无效的Java文件名

原因：指定的Java文件名在语法上不正确。

操作：确保Java文件名拼写正确。

O2I-00131：打开Java文件时出错

原因：尝试打开Java文件进行写入时发生操作系统错误。

操作：确保文件名拼写正确。尝试创建或修改文件以确保您具有该文件的写入权限。

O2I-00132：无法关闭Java文件

原因：尝试关闭Java文件时发生操作系统错误。

操作：尝试创建或修改Java文件以确保您能够访问该文件。

O2I-00133：发生错误，没有消息可用

原因：未报告此错误的原因。

行动：联系ORACLE客户支持。
