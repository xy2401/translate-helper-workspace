2.2 Oracle XML DB用例

Oracle XML DB适用于使用XML表示处理的部分或全部数据的任何应用程序。

Oracle XML DB提供高性能的数据库摄取，存储，处理和XML数据检索。它还允许您从现有关系数据快速轻松地生成XML。Oracle XML DB特别适合的应用程序包括：

  • 企业对企业（B2B）和应用程序到应用程序（A2A）集成

  • 互联网

  • 内容管理

  • 消息

  • 网页服务

典型的Oracle XML DB应用程序至少具有以下特征之一：

  • 必须提取或生成大量XML文档

  • 必须处理或生成大型XML文档

  • 无论是在文档中还是在大量文档集中，都需要进行高性能搜索

  • 需要高水平的安全性

  • 需要细粒度的安全性

  • 数据处理必须使用XML文档，并且数据必须存储在关系表中

  • 编程必须支持SQL，XML，XQuery，XPath和XSL等开放标准

  • 必须使用标准Internet协议访问信息，例如FTP，HTTP（S）/ WebDAV和Java数据库连接（JDBC）

  • 必须从SQL查询XML数据

  • 必须将分析功能应用于XML数据

  • 必须根据XML模式验证XML文档

2.3.2 XML数据的结构

XML数据的结构化程度以及它是否基于XML模式可以影响您存储它的方式。

如果您的XML数据不是基于XML模式的，那么无论结构如何，您都可以将其存储在XMLType表或视图中作为二进制XML，或者您可以将其存储为Oracle XML DB Repository文件夹中的文件。您不能以对象关系存储它。

如果您的XML数据是基于XML Schema的，则必须将其存储为二进制XML或对象关系。

2.3.4 XML处理选项

Oracle XML DB提供了全面的XML处理选项。

在设计Oracle XML DB应用程序时，可以使用以下内容：

  • XML生成和XMLType视图。是否需要生成（或重新生成）XML数据。请参阅从关系数据生成XML数据 。

  • 您的应用程序是以数据为中心还是以文档为中心，或两者兼而有之。请参阅如何使用Oracle XML DB概述 。

  • DOM保真度，文档保真度。XMLType存储（无论是对象关系还是二进制XML）都保留了DOM保真度 。也就是说，从存储为XMLType的XML文档创建的DOM与从原始文档创建的DOM相同。但是，微不足道的空白可能存在差异。请参阅DOM Fidelity ， SYS_XDBPD $和DOM Fidelity for Object-Relational Storage ，以及PL / SQL API for XMLType 。

    如果除了DOM保真度之外还需要保留文档保真度 （无效空白），则存储原始文档的两个副本：一个用作数据库使用和XML处理的XMLType实例，另一个用作提供文档保真度的CLOB实例。

  • XPath搜索。您可以使用嵌入在SQL语句中的XPath语法来查询数据库中的XML内容。请参阅使用RESOURCE_VIEW和PATH_VIEW 查询和更新XML数据 ， 访问Oracle XML DB存储库数据和存储库访问 。

  • XML文档访问，更新和操作的频率。请参阅XML数据的查询和更新 。

  • 是否需要更新片段或整个文档。您可以使用XPath表达式在更新期间指定文档的各个元素和属性，而无需重写整个文档。这样更有效，特别是对于大型XML文档。请参阅更新XML数据 。

  • 哪种索引最适合您的应用程序和数据。请参阅XMLType数据的索引 。

  • XSLT。是否需要将XML数据转换为HTML，WML或其他语言，如果需要，您的应用程序如何执行此操作。在Oracle XML DB中存储XML文档时，您可以选择确保其结构符合（验证）特定的XML模式。请参阅XMLType数据的转换和验证 。

2.3.5 Oracle XML DB存储库访问

使用Oracle XML DB Repository的应用程序的设计注意事项包括访问方法，安全性需求以及是否需要版本控制。

有两种主要的存储库访问方法：

  • 基于导航的访问或基于路径的访问。这适用于内容/文档和面向数据的应用程序。Oracle XML DB提供以下语言和访问API：

    • 通过资源和路径视图进行SQL访问。请参阅使用RESOURCE_VIEW和PATH_VIEW的存储库访问 。

    • 使用包DBMS_XDB或包DBMS_XDB_ADMIN ， DBMS_XDB_CONFIG和DBMS_XDB_REPOS PL / SQL访问。请参阅对Oracle XML DB存储库的PL / SQL访问 。

    • 使用HTTP（S）/ WebDAV或FTP的基于协议的访问，最适合面向内容的应用程序。请参阅使用协议的存储库访问 。

  • 基于查询的访问。这可能最适合面向数据的应用程序。Oracle XML DB通过以下API使用SQL查询提供访问：

    • Java访问（通过JDBC）。请参阅XML DOM的Java DOM API 。

    • PL / SQL访问。请参阅PLType / XML API以获取XMLType 。

访问Oracle XML DB存储库数据中还讨论了用于访问存储库数据的这些选项。

您还可以考虑以下访问条件：

  • 您需要什么级别的安全性。请参阅存储库访问控制 。

  • 是否需要对数据进行版本控制。请参阅资源版本 。

2.3.6 Oracle XML DB与其他数据库选项和功能配合使用

Oracle XML DB是Oracle数据库的集成部分，可与其他数据库选项和功能配合使用。

  • Oracle数据库高级队列（AQ） - 合并XML有效负载。请参阅使用Oracle数据库高级队列和Oracle数据库高级队列用户指南的 XML数据交换

  • Oracle GoldenGate和Oracle Active Data Guard - 复制和保护XML数据，或执行滚动升级。请参阅Oracle GoldenGate和Oracle Data Guard概念和管理

  • Oracle Exadata存储服务器软件 - XML数据的高性能，可扩展和高可用性。请参阅“ Oracle Exadata Storage Server软件用户指南” 。

  • Oracle Real Application Clusters（Oracle RAC） - 将XML数据与数据库实例集群一起使用。请参阅“ Oracle Real Application Clusters管理和部署指南”

  • Oracle Multitenant选项 - 将XML数据与多租户架构结合使用，其中每个可插拔数据库都有自己的Oracle XML DB存储库。请参阅Oracle Multitenant管理员指南

  • 压缩和加密 - 您可以压缩或加密使用SecureFiles LOB存储的二进制XML数据。对于以对象关系存储的XML数据，您可以单独压缩或加密XML元素和属性。

  • 并行执行 - 可以并行执行以下操作：

    • XMLType数据的查询

    • 使用SecureFiles LOB存储为二进制XML的XMLType数据的DML

    • 直接加载XMLType表，在该表上定义了Oracle Text CONTEXT索引
