35.1 SDO_UTIL.AFFINETRANSFORMS

格式

SDO_UTIL.AFFINETRANSFORMS（几何IN IN SDO_GEOMETRY，翻译IN VARCHAR2 DEFAULT'FALSE'，tx IN NUMBER DEFAULT 0.0，ty IN NUMBER DEFAULT 0.0，tz IN NUMBER DEFAULT 0.0，scale IN VARCHAR2 DEFAULT'FALSE'，psc1 IN SDO_GEOMETRY DEFAULT NULL，sx IN NUMBER DEFAULT 0.0，sy IN NUMBER DEFAULT 0.0，sZ IN NUMBER DEFAULT 0.0，旋转IN VARCHAR2 DEFAULT'FALSE'，p1 IN SDO_GEOMETRY DEFAULT NULL，line1 IN SDO_GEOMETRY DEFAULT NULL，angle IN NUMBER DEFAULT 0.0，dir IN NUMBER DEFAULT -1，剪切IN VARCHAR2 DEFAULT'FALSE'，shxy IN NUMBER DEFAULT 0.0，shxx IN NUMBER DEFAULT 0.0，shxx IN NUMBER DEFAULT 0.0，shzx IN NUMBER DEFAULT 0.0，shyz IN NUMBER DEFAULT 0.0，shzy IN NUMBER DEFAULT 0.0，ref IN IN VARCHAR2 DEFAULT'FALSE' ，pref IN SDO_GEOMETRY DEFAULT NULL，lineR IN SDO_GEOMETRY DEFAULT NULL，dirR IN NUMBER DEFAULT -1，planeR IN VARCHAR2 DEFAULT'FALSE'， n IN SDO_NUMBER_ARRAY DEFAULT NULL，bigD IN SDO_NUMBER_ARRAY DEFAULT NULL，）RETURN SDO_GEOMETRY;

描述

返回反映输入几何的仿射变换的几何。

参数

几何

输入几何体以执行仿射变换。

翻译

字符串值TRUE导致执行转换;字符串值FALSE （默认值）导致不执行转换。如果此参数为TRUE ，则执行关于（tx，ty）或（tx，ty，tz）处的点的转换。

tx

用于平移的X轴值。默认值为0.0。

ty

用于平移的Y轴值。默认值为0.0。

tz

用于平移的Z轴值。默认值为0.0。

缩放

字符串值TRUE导致执行缩放;字符串值FALSE （默认值）导致不执行缩放。

psc1

指向要执行缩放的输入几何体。如果scaling为TRUE ，则此几何应该是零点（具有0,0或0,0,0纵坐标的点几何，用于缩放原点）或非零点（具有纵坐标的点几何，用于缩放关于除了原点之外的点）起源）。如果scaling为FALSE ，则psc1可以为空值。

sx

用于缩放的X轴值（关于psc1参数或原点中指定的点）。默认值为0.0。

sy

用于缩放的Y轴值（关于psc1参数或原点中指定的点）。默认值为0.0。

sz

用于缩放的Z轴值（关于psc1参数或原点中指定的点）。默认值为0.0。

轮换

字符串值TRUE导致执行旋转;字符串值FALSE （默认值）导致不执行旋转。

对于二维几何，旋转使用p1和angle值。对于三维几何形状，旋转使用任一angle和dir值或line1和angle值。

p1

指向围绕指定点的二维几何旋转。

围绕指定轴旋转的线。

角度

用于围绕指定轴或围绕X，Y或Z轴旋转的角度旋转参数（以弧度表示）。默认值为0.0。

目录

x（0），y（1）或z（2）-axis roll的旋转参数。如果rotation参数值为TRUE但未使用dir参数，请使用值-1（默认值）

剪毛

字符串值TRUE会导致剪切;字符串值FALSE （默认值）导致剪切不执行。

对于二维几何，剪切使用shxy和shyx参数值。对于三维几何，剪切使用shxy ， shyx ， shxz ， shzx ， shyz和shzy参数值。

shxy

由于X沿Y方向的剪切值。默认值为0.0。

shyx

由于Y沿X方向的剪切值。默认值为0.0。

shxz

由于X沿Z方向的剪切值（仅限三维几何）。默认值为0.0。

shzx

Z方向沿X方向剪切的值（仅限三维几何）。

shyz

由于Y沿Z方向的剪切值（仅限三维几何）。

很蠢

由于Z沿Y方向的剪切值（仅限三维几何）。

反射

字符串值TRUE导致执行反射;字符串值FALSE （默认值）导致不执行反射。

对于二维几何，反射使用lineR值进行轴的反射，使用质心的pref值进行自反射。对于三维几何，反射使用lineR值进行关于轴的反射;关于yz，xz和xy平面的反射的dirR值;用于反射指定平面的planeR ， n和bigD值;和自我反射的质心的pref值。

pref

指向进行反射的点。

lineR

沿着哪条线进行反射。

dirR

表示执行反射的平面（通过）的数字：0表示yz平面，1表示xz平面，或2表示xy平面。如果reflection参数值为TRUE但未使用dirR参数，请使用值-1（默认值）。

planeR

字符串值TRUE导致关于要执行的任意平面的反射;字符串值FALSE （默认值）会导致无法执行任意平面的反射。

n

飞机的法线向量。

bigD

三维几何中平面方程的Delta值。

对于三维几何， bigD = delta和n =（A，B，C）其中n是三维空间中平面的法线。因此，平面方程是：

Ax + By + Cz + bigD = 3DDotProd（n， anypointonplane ）+ bigD = 0

使用说明

仿射变换的顺序很重要，因为它们是矩阵和向量乘法。

您应该使用SDO_GEOM.VALIDATE_GEOMETRY_WITH_CONTEXT函数验证生成的几何。

35.2 SDO_UTIL.APPEND

格式

SDO_UTIL.APPEND（geometry1 IN SDO_GEOMETRY，geometry2 IN SDO_GEOMETRY）RETURN SDO_GEOMETRY;

描述

将一个几何体附加到另一个几何体以创建新几何体。

参数

geometry1

要附加geometry2几何对象。

geometry2

要附加到geometry1几何对象。

使用说明

此函数仅应用于没有任何空间交互的几何（即，在不相交的对象上）。如果输入几何不是不相交的，则生成的几何可能无效。

此函数不执行并集操作或任何其他计算几何操作。要执行并集操作，请使用SDO_GEOM.SDO_UNION函数，该函数在SDO_GEOM包（几何）中描述。APPEND函数的执行速度比SDO_GEOM.SDO_UNION函数快。

生成的几何体的几何类型（SDO_GTYPE值）反映了输入几何的类型和追加操作。例如，如果输入几何是二维多边形（SDO_GTYPE = 2003），则生成的几何图形是二维多边形（SDO_GTYPE = 2007）。

如果geometry1和geometry2基于不同的坐标系，则会引发异常。

以下示例附加cola_a和cola_c几何。（该示例使用简单示例中的定义和数据：Inserting_ Indexing_和查询空间数据 。）

35.3 SDO_UTIL.BEARING_TILT_FOR_POINTS

格式

SDO_UTIL.BEARING_TILT_FOR_POINTS（START_point IN SDO_GEOMETRY，end_point IN SDO_GEOMETRY，tol in NUMBER，承载OUT NUMBER，倾斜OUT NUMBER）RETURN SDO_GEOMETRY;

描述

计算从起点到终点的方位和倾斜度。

参数

start_point

起点几何对象，用于计算轴承和倾斜度。点几何必须基于大地坐标系。

end_point

结束点几何对象用于计算轴承和倾斜。点几何必须基于与start_point相同的大地坐标系。

tol

公差值（见公差 ）。

轴承

从北方顺时针测量的弧度数。

倾斜

从法线测量的弧度数。

使用说明

输入点几何必须基于相同的大地坐标系。如果它们基于非大地坐标系，则输出方位为空值。

倾斜度被计算为高度值之间的差值的反正切除以点之间的距离（高度从距离计算中排除）。那就是： tilt = atan(height_difference/distance)

要将弧度转换为十进制度或将十进制度转换为弧度，可以使用SDO_UTIL.CONVERT_UNIT函数。要返回指定距离的点几何并从起点开始，可以使用SDO_UTIL.POINT_AT_BEARING函数。

例子

35.4 SDO_UTIL.CIRCLE_POLYGON

格式

SDO_UTIL.CIRCLE_POLYGON（指向SDO_GEOMETRY，半径IN NUMBER，arc_tolerance IN NUMBER，start_azimuth IN NUMBER DEFAULT NULL，end_azimuth IN NUMBER DEFAULT NULL，orientation IN NUMBER DEFAULT NULL，arc IN NUMBER DEFAULT NULL）RETURN SDO_GEOMETRY;

要么

SDO_UTIL.CIRCLE_POLYGON（center_longitude IN NUMBER，center_latitude IN NUMBER，radius IN NUMBER，arc_tolerance IN NUMBER）RETURN SDO_GEOMETRY;

要么

SDO_UTIL.CIRCLE_POLYGON（center_longitude IN NUMBER，center_latitude IN NUMBER，radius IN NUMBER，start_azimuth IN NUMBER，end_azimuth IN NUMBER，arc_tolerance IN NUMBER）RETURN SDO_GEOMETRY;

描述

在大地坐标系中创建圆和弧的多边形或折线近似值。

  • 第一种格式允许完全控制输出。

  • 第二种格式返回近似的多边形几何体，并由指定的圆覆盖。

  • 第三种格式返回一个线几何，它近似于从开始到结束方位角（圆弧）的圆的边界。

参数

点

圆的中心。必须是大地坐标系中的点几何。返回的几何体将位于相同的坐标系中。

center_longitude

用于创建返回几何的圆的中心经度（以度为单位）。

center_latitude

用于创建返回几何的圆的中心纬度（以度为单位）。

半径

用于创建返回几何的圆的半径的长度（以米为单位）。

arc_tolerance

用于构造多边形几何的数值。arc_tolerance参数值与SDO_GEOM.SDO_ARC_DENSIFY函数的params参数字符串中的arc_tolerance关键字值具有相同的含义和使用指南。与几何相关联的度量单位与arc_tolerance参数值相关联。（有关更多信息，请参阅SDO_GEOM包（几何）中 SDO_GEOM.SDO_ARC_DENSIFY函数的使用说明。）

start_azimuth

弧的起始角度（以度为单位），从正南方向逆时针测量。

如果start_azimuth和end_azimuth为null，则行为与第二种格式相当，后者返回近似并被指定圆覆盖的多边形几何。

end_azimuth

弧的终止角度（以度为单位），从正南方向逆时针测量。如果orientation没有明确指定，则弧会从圆的逆时针部分start_azimuth如果end_azimuth大于start_azimuth ，与圆弧将成为顺时针节中，如果end_azimuth小于start_azimuth 。

如果start_azimuth和end_azimuth为null，则行为与第二种格式相当，后者返回近似并被指定圆覆盖的多边形几何。

方向

控制使用从start_azimuth到end_azimuth的圆圈部分。它控制返回输出的形状，而不是输出的方向：返回的多边形总是逆时针定向，返回的弧总是从start_azimuth到end_azimuth 。值可以是以下之一：

  • 0或null（默认值）：Automatic（请参阅end_azimuth参数说明）。

  • 1或+1：从start_azimuth到end_azimuth逆时针绘制弧。

  • -1：从start_azimuth到end_azimuth顺时针绘制弧。

弧形

如果设置为1，结果将是一行;如果为0或null（默认值），则结果为多边形。如果start_azimuth和end_azimuth指定具有多边形结果的圆的子集，则返回的多边形将包括圆的中心（即，将是圆的扇区）。

使用说明

当无法使用真圆时，此函数的第一种格式对于在指定中心点周围创建圆形多边形非常有用（圆形对于使用Oracle Spatial和Graph的大地数据无效）。返回的几何体的SDO_SRID值为8307（ Longitude / Latitude (WGS 84) ）。

此函数的第二种和第三种格式对于创建圆弧的折线近似或表示圆的扇形的多边形非常有用。

如果指定了起始和结束方位角值，则它们必须彼此不相等，并且必须覆盖不超过360度的旋转。角度必须在-720到+720的范围内。

将始终使用至少四个不同的顶点（正方形）创建圆。

返回输入二维几何的所有元素和子元素，作为一个或多个几何的数组。返回SDO_GEOMETRY_ARRAY类型的对象，该对象定义为VARRAY OF SDO_GEOMETRY 。

参数

几何

从中提取所有元素和子元素的几何。必须是二维几何。

展平

一个标志，指示是否将环“展平”为包含外部环和一个或多个内环的几何形状的单个几何形状：

  • 0 （零）为每个元素返回一个几何，但不会将环展平为单个几何。（仍将为输入几何体的每个元素返回几何体。）

  • 1 （默认值）或任何其他非零值将环展平为单个几何。

例如，如果多边形包含外环和内环，则值0将返回包含两个环的单个几何，值1返回两个几何，每个几何包含一个环作为几何。

对于不包含外环和一个或多个内环的几何，此参数将被忽略。

使用说明

此功能仅适用于二维几何。对于三维几何，请使用SDO_UTIL.EXTRACT3D函数。

此功能使您可以从几何体中提取所有元素和子元素，无论几何体具有多少元素和子元素。具有SDO_GTYPE值（在SDO_GTYPE中解释）以1，2或3结尾的几何图形具有一个元素; SDO_GTYPE值以4,5,6或7结尾的几何可以包含多个元素。例如，SDO_GTYPE为2007的多面可能包含三个元素（多边形）。要提取单个元素，请改用SDO_UTIL.EXTRACT函数。

对于具有一个或多个孔的多边形，使用flatten参数的默认值，表示提取的内环的返回几何体将重新定向，以使其顶点以逆时针顺序呈现（与内环中的顺时针顺序相反）。但是，如果flatten参数值为0，则不执行重新定向。

如果geometry为null或者SDO_GTYPE值以0结尾，则此函数返回null几何。

geometry不能包含类型0（零）元素。类型0元素在类型0（零）元素中描述。

此函数不适用于具有任何null纵坐标值的几何。返回几何中的任何null纵坐标值都将替换为0（零）。

35.40 SDO_UTIL.SIMPLIFY

格式

SDO_UTIL.SIMPLIFY（几何IN IN SDO_GEOMETRY，阈值IN NUMBER容差IN NUMBER DEFAULT 0.0000005，remove_loops IN NUMBER DEFAULT 0）RETURN SDO_GEOMETRY;

描述

使用Douglas-Peucker算法，根据阈值简化输入几何。

参数

几何

几何要简化。

门槛

用于几何简化的阈值。应该是一个正数。（零会导致返回输入几何体。）如果输入几何是大地测量，则该值是米的数量;如果输入几何是非大地测量，则该值是与数据关联的单位数。

随着阈值的减小，返回的几何体可能更接近输入几何体;随着阈值的增加，返回的几何体中可能会有更少的点。有关更多信息，请参阅使用说明。

容差

公差值（见公差 ）。不得大于threshold ;为了获得更好的性能，不应该与threshold相同。如果未指定值，则默认值为0.0000005。

remove_loops

对于某些线几何，当线被简化时，它可能最终在中间具有自交环。虽然这是有效的几何（对于线），但在某些情况下，不希望在简化操作的结果中具有这些循环。值0 （默认值）不会删除此类循环;值为1 （或任何其他非零正数）将删除任何此类循环，并始终返回简单的线段。

使用说明

此函数还将弧转换为线条叮咬，消除重复的顶点，并纠正许多重叠的边多边形问题。此函数有时修复问题的原因是它在简化过程结束时在内部调用SDO_UTIL.RECTIFY_GEOMETRY函数，以确保返回有效的几何。但请注意，如果两个完美对齐的几何图形独立简化，则几何图形在简化后可能不会对齐。

如果希望几何体的分辨率低于原始几何体，则此功能非常有用。例如，如果显示分辨率无法在河流或政治边界中显示数百或数千个转弯，则如果将几何图形简化为仅显示主要转弯，则可能会产生更好的性能。

如果对具有两个以上维度的几何使用此函数，则在处理查询时仅使用前两个维度，并且只有返回几何中的前两个维度才被视为有效且有意义。

此功能使用Douglas-Peucker算法，该算法在多个制图教科书和参考文档中进行了解释。（在某些解释中，使用术语容差而不是阈值 ;但是，这与容差的Oracle Spatial和Graph含义不同。）

比较具有这种功能SDO_UTIL.SIMPLIFYVW ，它使用Visvalingham-Whyatt算法。

返回的几何体可以是多边形，线或点，具体取决于几何定义和阈值。以下注意事项适用：

  • 如果与几何相关联的阈值足够大，则多边形可以简化为线或点，并且线可以简化为点。例如，如果两个平行长边之间的距离小于阈值，则细矩形将简化为线，并且如果起点和终点之间的距离小于阈值，则线将简化为点。 。

  • 在具有孔的多边形中，如果外环或内环（孔）简化为直线或点，则内环从所得几何体中消失（不包括在其中）。

  • 简化后可能无法保持输入几何的拓扑特征。对于集合几何，元素的数量可能会增加，以防止各个元素重叠。在所有情况下，此函数都不会返回无效的几何体。

线性参考系统（LRS）几何（ 在线性参考系统中描述）不支持此功能。

格式

SDO_UTIL.TO_KMLGEOMETRY（几何IN IN SDO_GEOMETRY）RETURN CLOB;

描述

将Spatial和Graph几何对象转换为KML（Keyhole标记语言）文档。

参数

几何

要返回KML文档的几何图形。

使用说明

此函数不转换圆，包含任何圆弧的几何，LRS几何或SDO_ETYPE值为0的几何（0型元素）;在这些情况下它返回一个空的CLOB。

必须使用Oracle9 i及更高版本的Spatial和Graph的约定来定义多边形。也就是说，首先存储外边界（ETYPE = 1003），然后存储零个或多个内边界元素（ETYPE = 2003）。对于带孔的多边形，外边界必须首先存储在SDO_ORDINATES定义中，然后是内边界的坐标。

在传递给TO_KMLGEOMETRY函数之前，必须将LRS几何转换为标准几何（使用SDO_LRS.CONVERT_TO_STD_GEOM或SDO_LRS.CONVERT_TO_STD_LAYER函数）。

任何圆弧或圆必须是致密化（使用SDO_GEOM.SDO_ARC_DENSIFY功能）或表示为多边形（使用SDO_GEOM.SDO_BUFFER被传递给TO_KMLGEOMETRY功能之前功能）。

标签点被丢弃。也就是说，如果几何具有SDO_POINT字段的值以及SDO_ELEM_INFO和SDO_ORDINATES中的值，则不会在KML文档中输出SDO_POINT。

实体几何将转换为KML MultiGeometry对象，因为KML 2.1不支持实体。如果随后在MultiGeometry上使用SDO_UTIL.FROM_KMLGEOMETRY函数，则结果不是Oracle Spatial and Graph实体几何（即，其SDO_GTYPE值不反映几何类型的SOLID或MULTISOLID）。

KML输出未格式化;标签没有换行符或缩进。要在SQL * Plus中查看返回的CLOB的内容，请使用TO_CHAR（）函数或将SQL * Plus参数LONG设置为合适的值（例如， SET LONG 2000 ）。要获取格式化的GML输出或在SQLX或Oracle XML DB函数（如XMLELEMENT）中使用TO_KMLGEOMETRY的返回值，请使用XMLTYPE（clobval CLOB）构造函数。
