1.306会议

SESSIONS指定可以在系统中创建的最大会话数。

属性 描述

参数类型

整数

默认值

派生：（1.5 * PROCESSES ）+ 22

可修改

ALTER SYSTEM只能在PDB中用于更改该PDB的SESSIONS参数值。

ALTER SYSTEM 不能用于更改非CDB或CDB $ ROOT中SESSIONS参数的值。

可在PDB中修改

是

价值范围

1到2 16 （1到65536）

基本

是

由于每次登录都需要会话，因此该参数有效地确定了系统中的最大并发用户数。您应该始终将此参数显式设置为等于最大并发用户数的估计值，加上后台进程数，再加上递归会话的大约10％。

Oracle使用此参数的默认值作为其最小值。介于1和默认值之间的值不会触发错误，但Oracle会忽略它们并使用默认值。

ENQUEUE_RESOURCES和TRANSACTIONS参数的默认值来自SESSIONS 。因此，如果增加SESSIONS的值，则应考虑是否调整ENQUEUE_RESOURCES和TRANSACTIONS的值。（请注意， ENQUEUE_RESOURCES Oracle Database 10g第2版（10.2）起， ENQUEUE_RESOURCES已过时。）

在共享服务器环境中， PROCESSES的值可能非常小。因此，Oracle建议您将SESSIONS的值调整为大约1.1 *总连接数 。

对于CDB，根容器的SESSIONS参数指定数据库的会话总数。

PDB的SESSIONS参数指定该PDB可以使用的会话总数。其值默认为根容器的SESSIONS值。如果PDB尝试使用的会话数多于其SESSIONS参数配置的会话数，则会生成ORA-00018错误消息。对于PDB，SESSIONS参数不计算递归会话，因此不需要10％调整。

PDB的SESSIONS参数只能由PDB修改。它不能设置为高于CDB的SESSIONS值。
