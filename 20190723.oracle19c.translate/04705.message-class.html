13.19消息类

消息是排队的单元。Message对象包含其内容或有效内容及其属性。此类提供了获取和设置消息属性的方法。

表13-25消息类使用的枚举值

属性 选项 
MessageState

  • MSG_WAITING表示尚未达到消息延迟时间

  • MSG_READY表示该消息已准备好进行处理

  • MSG_PROCESSED表示邮件已被处理，并且正在保留

  • MSG_EXPIRED表示消息已移至异常队列。 

载荷类型

  • 生的

  • ANYDATA

  • 宾语

13.19.21 setDelay（）

指定延迟排队Message的时间（以秒为单位）。延迟结束后， Message可用于出队。

请注意， msgid出列会覆盖延迟规范。具有延迟的队列Message处于WAITING状态。延迟由Message的生产者设置。

句法

void setDelay（int delay）;
参数 描述 
延迟

延误。

13.19.22 setExceptionQueueName（）

如果无法成功处理，则指定Message对象移动到的队列的名称。队列名称必须有效。

请注意，如果移动时异常队列不存在，则将Message移动到与队列表关联的缺省异常队列;警报日志中会记录一条警告。

另请注意，如果使用默认异常队列，则参数在入队时返回NULL值;该属性必须引用有效的队列名称。

句法

void setExceptionQueueName（const string＆queue）;
参数 描述 
队列

异常队列的名称。
