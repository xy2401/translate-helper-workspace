ALTER LOCKDOWN PROFILE

目的

使用ALTER LOCKDOWN PROFILE语句更改PDB锁定配置文件。您可以在多租户环境中使用PDB锁定配置文件来限制可插拔数据库（PDB）中的用户操作。

使用CREATE LOCKDOWN PROFILE语句创建锁定配置文件后，立即为配置文件启用所有用户操作。然后，您可以使用ALTER LOCKDOWN PROFILE语句禁用配置文件的某些用户操作。当锁定配置文件应用于CDB，应用程序容器或PDB时，用户无法执行对配置文件禁用的操作。如果您以后想要重新启用某些禁用的用户操作，则可以使用ALTER LOCKDOWN PROFILE语句来执行此操作。

ALTER LOCKDOWN PROFILE语句允许您禁用或启用：

  • 与某些数据库功能关联的用户操作（使用lockdown_features子句）

  • 与某些数据库选项关联的用户操作（使用lockdown_options子句）

  • 发布某些SQL语句（使用lockdown_statements子句）

  • 您必须从CDB根或应用程序根发出ALTER LOCKDOWN PROFILE语句。

  • 您必须在发出语句的容器中具有ALTER LOCKDOWN PROFILE系统特权。

语义

PROFILE_NAME

指定要更改的PDB锁定配置文件的名称。

您可以通过查询DBA_LOCKDOWN_PROFILES数据字典视图找到现有PDB锁定配置文件的名称。

lockdown_features

此子句允许您禁用或启用与某些数据库功能关联的用户操作。

  • 指定DISABLE以添加指定功能的限制。将限制用户在配置文件适用的任何PDB中执行这些操作。

  • 指定ENABLE以删除指定功能的限制。将允许用户在配置文件适用的任何PDB中执行这些操作。

  • 使用feature指定要禁用或启用其操作的功能。表11-1列出了您可以指定的功能，并描述了与每个功能关联的操作。该表还指示每个功能的功能包。对于feature ，您可以指定功能区包名称以禁用或启用该包中所有功能的用户操作，也可以指定单个功能名称。您可以使用大写和小写字母的任意组合指定功能包名称和功能名称。

  • 使用ALL指定表中列出的所有功能。

  • EXCEPT指定的功能外，使用ALL EXCEPT指定表中列出的所有功能。

如果省略此子句，则默认为ENABLE ALL 。

lockdown_options

此子句允许您禁用或启用与某些数据库选项关联的用户操作。

  • 指定DISABLE以禁用指定选项的用户操作。将限制用户在配置文件适用的任何PDB中执行这些操作。

  • 指定ENABLE以启用指定选项的用户操作。将允许用户在配置文件适用的任何PDB中执行这些操作。

  • 对于option ，您可以使用大写和小写字母的任意组合指定以下数据库选项：

    • DATABASE QUEUING - 表示与Oracle Database Advanced Queuing选项关联的用户操作

    • PARTITIONING - 表示与Oracle Partitioning选项关联的用户操作

  • 使用ALL指定前面列表中的所有选项。

  • EXCEPT指定的选项外，使用ALL EXCEPT指定前面列表中的所有选项。

如果省略此子句，则默认为ENABLE OPTION ALL 。

lockdown_statements

此子句允许您禁用或启用某些SQL语句的颁发。

  • 指定DISABLE以禁用指定SQL语句的颁发。用户将被限制在该配置文件适用的任何PDB中发布这些语句。

  • 指定ENABLE以启用指定SQL语句的颁发。用户将被允许在配置文件适用的任何PDB中发布这些语句。

  • 对于SQL_statement ，您可以使用大写和小写字母的任意组合指定以下语句：

    • ALTER DATABASE

    • ALTER PLUGGABLE DATABASE

    • ALTER SESSION

    • ALTER SYSTEM

  • 使用ALL指定前面列表中的所有语句。

  • 使用ALL EXCEPT指定前面列表中除指定语句之外的所有语句。

如果省略此子句，则默认为ENABLE STATEMENT ALL 。

statement_clauses

此子句允许您禁用或启用指定SQL语句的特定子句。

  • 使用clause指定构成要禁用或启用的子句的SQL关键字。您可以使用大写和小写字母的任意组合指定子句。

  • 使用ALL指定SQL语句的所有子句。

  • 使用ALL EXCEPT指定除指定子句之外的SQL语句的所有子句。

对于clause ，必须至少指定足够的关键字以明确标识SQL语句的单个子句。以下是如何为ALTER SYSTEM语句指定clause一些示例：

  • 要指定archive_log_clause :: = ，请指定ARCHIVE 。这就足够了，因为没有其他ALTER SYSTEM子句以关键字ARCHIVE开头。或者，您可以指定ARCHIVE LOG以获得语义清晰度，但不需要LOG关键字。

  • 要指定rolling_migration_clauses :: =中的任何一个，必须指定START ROLLING MIGRATION或STOP ROLLING MIGRATION以区分这些子句与类似名称的rolling_patch_clauses :: = START ROLLING PATCH和STOP ROLLING PATCH 。

  • 您不能指定单个关键字FLUSH ，因为有几个ALTER SYSTEM子句以此关键字开头。您必须单独指定每个子句，例如FLUSH SHARED_POOL或FLUSH GLOBAL CONTEXT 。

没有必要在子句中指定可选关键字，因为它们没有效果。例如：

  • archive_log_clause :: =具有可选的INSTANCE关键字。但是，您不能仅启用或禁用包含INSTANCE关键字的ARCHIVE LOG子句。指定ARCHIVE LOG INSTANCE等同于指定ARCHIVE或ARCHIVE LOG 。

无需在子句中指定参数值，因为它们不起作用。例如：

  • shutdown_dispatcher_clause :: =要求您指定dispatcher_name 。但是，您无法启用或禁用包含特定调度程序名称的SHUTDOWN子句。指定SHUTDOWN dispatcher1等同于指定SHUTDOWN 。

也可以看看：

ALTER DATABASE ， ALTER PLUGGABLE DATABASE ， ALTER SESSION和ALTER SYSTEM ，提供有关这些声明条款的完整信息

clause_options

仅当为lockdown_statements和statement_clauses指定以下内容之一时，此子句才有效：

{DISABLE | ENABLE} STATEMENT =（'ALTER SESSION'）CLAUSE =（'SET'）{DISABLE | ENABLE} STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）

此子句允许您使用ALTER SESSION SET或ALTER SYSTEM SET语句禁用或启用特定选项的设置或修改。

  • 使用clause_option指定要禁用或启用的选项。

  • 使用clause_option_pattern指定匹配多个选项的模式。在模式中，指定百分号（％）以匹配选项名称中的零个或多个字符。例如，指定'QUERY_REWRITE_%'等同于指定QUERY_REWRITE_ENABLED和QUERY_REWRITE_INTEGRITY选项。

  • 您可以使用大写和小写字母的任意组合指定clause_option和clause_option_pattern 。

  • 使用ALL指定所有选项。

  • 使用ALL EXCEPT指定EXCEPT指定选项之外的所有选项。

也可以看看：

该alter_session_set_clause的条款ALTER SESSION和alter_system_set_clause的条款ALTER SYSTEM有关选项，您可以指定这些语句的完整信息

option_values

仅当为lockdown_statements ， statement_clauses和clause_options指定以下内容之一时，此子句才有效：

DISABLE STATEMENT =（'ALTER SESSION'）CLAUSE =（'SET'）OPTION = clause_option DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）OPTION = clause_option

此子句允许您在禁用该选项的设置时为选项指定默认值。对于采用数值的选项，此子句还允许您限制用户将选项设置为某些值。

  • VALUE子句允许您为clause_option指定默认的option_value ，该选项将在关闭并重新打开PDB后对配置文件应用的任何PDB生效。如果clause_option接受多个默认值，则可以在逗号分隔列表中指定多个option_value 。使用此子句的目的是同时为选项设置默认值，并限制用户设置或修改该值。

  • MINVALUE子句允许您限制用户将clause_option的值设置为小于option_value的值。您只能为采用数值的选项指定此子句。

  • MAXVALUE子句允许您限制用户将clause_option的值设置为大于option_value的值。您只能为采用数值的选项指定此子句。

  • 您可以同时指定MINVALUE和MAXVALUE子句，以限制用户将clause_options的值设置为小于MINVALUE或大于MAXVALUE任何值。

  • 当锁定配置文件分配给PDB时， MINVALUE和MAXVALUE设置立即生效;您无需关闭并重新打开PDB。

例子

以下语句创建PDB锁定配置文件hr_prof ：

CREATE LOCKDOWN PROFILE hr_prof;

本节中的其余示例更改了hr_prof 。

禁用PDB锁定配置文件的功能：示例

以下语句禁用功能包NETWORK_ACCESS中的所有功能：

ALTER LOCKDOWN PROFILE hr_prof DISABLE FEATURE =（'NETWORK_ACCESS'）;

以下语句禁用LOB_FILE_ACCESS和TRACE_VIEW ACCESS功能：

ALTER LOCKDOWN PROFILE hr_prof DISABLE FEATURE =（'LOB_FILE_ACCESS'，'TRACE_VIEW_ACCESS'）;

以下语句禁用除COMMON_USER_LOCAL_SCHEMA_ACCESS和LOCAL_USER_COMMON_SCHEMA_ACCESS功能之外的所有功能：

ALTER LOCKDOWN PROFILE hr_prof DISABLE FEATURE ALL EXCEPT =（'COMMON_USER_LOCAL_SCHEMA_ACCESS'，'LOCAL_USER_COMMON_SCHEMA_ACCESS'）;

以下语句禁用所有功能：

ALTER LOCKDOWN PROFILE hr_prof DISABLE FEATURE ALL;

为PDB锁定配置文件启用功能：示例

以下语句启用UTL_HTTP和UTL_SMTP功能，以及功能包OS_ACCESS中的所有功能：

ALTER LOCKDOWN PROFILE hr_prof ENABLE FEATURE =（'UTL_HTTP'，'UTL_SMTP'，'OS_ACCESS'）;

以下语句启用除AQ_PROTOCOLS和CTX_PROTOCOLS功能之外的所有功能：

ALTER LOCKDOWN PROFILE hr_prof ENABLE FEATURE ALL EXCEPT =（'AQ_PROTOCOLS'，'CTX_PROTOCOLS'）;

以下语句启用所有功能：

ALTER LOCKDOWN PROFILE hr_prof ENABLE FEATURE ALL;

禁用PDB锁定配置文件的选项：示例

以下语句禁用与Oracle Database Advanced Queuing选项关联的用户操作：

ALTER LOCKDOWN PROFILE hr_prof DISABLE OPTION =（'DATABASE QUEUING'）;

以下语句禁用与Oracle Partitioning选项关联的用户操作：

ALTER LOCKDOWN PROFILE hr_prof DISABLE OPTION =（'PARTITIONING'）;

启用PDB锁定配置文件的选项：示例

以下语句启用与Oracle Database Advanced Queuing选项关联的用户操作：

ALTER LOCKDOWN PROFILE hr_prof ENABLE OPTION =（'DATABASE QUEUING'）;

以下语句允许与Oracle Database Advanced Queuing选项和Oracle Partitioning选项关联的用户操作：

ALTER LOCKDOWN PROFILE hr_prof ENABLE OPTION ALL;

禁用PBB锁定配置文件的SQL语句：示例

以下语句禁用ALTER DATABASE语句：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER DATABASE'）;

以下语句禁用ALTER SYSTEM SUSPEND和ALTER SYSTEM RESUME语句：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SUSPEND'，'RESUME'）;

以下语句禁用ALTER PLUGGABLE DATABASE语句的所有子句，但DEFAULT TABLESPACE和DEFAULT TEMPORARY TABLESPACE除外：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER PLUGGABLE DATABASE'）条款ALL EXCEPT =（'DEFAULT TABLESPACE'，'DEFAULT TEMPORARY TABLESPACE'）;

下面的语句禁止使用ALTER SESSION语句设置或修改COMMIT_WAIT或CURSOR_SHARING ：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SESSION'）CLAUSE =（'SET'）OPTION =（'COMMIT_WAIT'，'CURSOR_SHARING'）;

以下语句禁用使用ALTER SYSTEM语句设置或修改PDB_FILE_NAME_CONVERT的值。它还将PDB_FILE_NAME_CONVERT的默认值PDB_FILE_NAME_CONVERT为'cdb1_pdb0', 'cdb1_pdb1' 。此默认值将在下次关闭并重新打开PDB时生效。 

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）OPTION =（'PDB_FILE_NAME_CONVERT'）VALUE =（'cdb1_pdb0'，'cdb1_pdb1'）;

以下语句禁用使用ALTER SYSTEM语句将CPU_COUNT的值设置或修改为小于8的值：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）OPTION =（'CPU_COUNT'）MINVALUE ='8';

以下语句禁用使用ALTER SYSTEM语句将CPU_COUNT的值设置或修改为大于2的值：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）OPTION =（'CPU_COUNT'）MAXVALUE ='2';

以下语句禁止使用ALTER SYSTEM语句将CPU_COUNT的值设置或修改为小于2或大于6 ：

ALTER LOCKDOWN PROFILE hr_prof DISABLE STATEMENT =（'ALTER SYSTEM'）CLAUSE =（'SET'）OPTION =（'CPU_COUNT'）MINVALUE ='2'MAXVALUE ='6';

为PBB锁定配置文件启用SQL语句：示例

以下语句启用除ALTER DATABASE之外的所有语句：

ALTER LOCKDOWN PROFILE hr_prof ENABLE STATEMENT ALL EXCEPT =（'ALTER DATABASE'）;

以下语句启用ALTER DATABASE MOUNT和ALTER DATABASE OPEN语句：

ALTER LOCKDOWN PROFILE hr_prof ENABLE STATEMENT =（'ALTER DATABASE'）CLAUSE =（'MOUNT'，'OPEN'）;

以下语句启用ALTER PLUGGABLE DATABASE语句的所有子句，但DEFAULT TABLESPACE和DEFAULT TEMPORARY TABLESPACE除外：

ALTER LOCKDOWN PROFILE hr_prof ENABLE STATEMENT =（'ALTER PLUGGABLE DATABASE'）条款ALL EXCEPT =（'DEFAULT TABLESPACE'，'DEFAULT TEMPORARY TABLESPACE'）;

下面的语句使使用ALTER SESSION语句设置或修改COMMIT_WAIT或CURSOR_SHARING ：

ALTER LOCKDOWN PROFILE hr_prof ENABLE STATEMENT =（'ALTER SESSION'）CLAUSE =（'SET'）OPTION =（'COMMIT_WAIT'，'CURSOR_SHARING'）;
