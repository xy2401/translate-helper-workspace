1.97 DBWR_IO_SLAVES

DBWR_IO_SLAVES指定DBW0进程使用的I / O服务器进程数。

属性 描述

参数类型

整数

默认值

0

可修改

没有

可在PDB中修改

没有

价值范围

0到操作系统相关

基本

没有

DBWR_IO_SLAVES仅与只有一个数据库写程序进程（ DBW0 ）的系统相关。DBW0进程及其服务器进程始终写入磁盘。默认情况下，该值为0并且不使用I / O服务器进程。

如果将DBWR_IO_SLAVES设置为非零值，则ARCH和LGWR进程使用的I / O服务器进程数将设置为4 。但是，仅当禁用异步I / O时，Recovery Manager使用的I / O服务器进程数才会设置为4 （您的平台不支持异步I / O或disk_asynch_io设置为false ）。

通常，I / O服务器进程用于模拟不支持异步I / O或低效实现异步I / O的平台上的异步I / O.但是，即使使用异步I / O，也可以使用I / O服务器进程。在这种情况下，I / O服务器进程将使用异步I / O.

即使启用了异步I / O，I / O服务器进程在具有非常大的I / O吞吐量的数据库环境中也很有用。
