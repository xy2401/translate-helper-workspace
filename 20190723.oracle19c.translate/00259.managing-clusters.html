22.1关于集群

集群提供了一种存储表数据的可选方法。集群由一组共享相同数据块的表组成。这些表组合在一起，因为它们共享公共列并且经常一起使用。

例如， emp和dept表共享deptno列。当您对emp和dept表进行集群时（参见图22-1 ），Oracle数据库实际存储了相同数据块中emp和dept表中每个部门的所有行。

由于集群将相关的不同表行存储在同一数据块中，因此正确使用的集群提供了两个主要好处：

  • 磁盘I / O减少，并且对于群集表的连接，访问时间得到改善。

  • 群集键是群集表共有的列或列组。您在创建群集时指定群集键的列。随后在创建添加到群集的每个表时指定相同的列。每个群集密钥值仅在群集和群集索引中存储一次，无论多少行的不同表包含该值。

    因此，在群集中存储相关表和索引数据可能需要的存储空间少于非群集表格式所需的存储空间。例如，在图22-1中 ，注意每个簇密钥（每个deptno ）如何只存储一次，对于emp和dept表中包含相同值的许多行。

创建集群后，您可以在集群中创建表。但是，在将任何行插入到集群表之前，必须创建集群索引。使用群集不会影响在群集表上创建其他索引;它们可以照常创建和删除。

不应将群集用于经常单独访问的表。

22.2.1为群集选择适当的表

将群集用于主要是查询并经常一起查询的表。

对于满足以下条件的表使用簇：

  • 主要查询表 - 即不主要插入或更新的表。

  • 表中的记录经常被一起查询或加入。

父主题： 管理群集的准则

22.2.2为群集密钥选择适当的列

仔细选择群集键列。如果在连接表的查询中使用多个列，请将群集键设置为组合键。通常，表示良好聚类索引的特征与任何索引的特征相同。

有关良好索引特征的信息，请参阅“ 索引管理指南 ” 。

良好的簇密钥具有足够的唯一值，使得与每个键值对应的行组填充大约一个数据块。每个群集密钥值的行数太少会浪费空间并导致可忽略的性能提升。除非在群集创建时指定了小SIZE ，否则只有少数几个行共享公共值的群集密钥才会导致块中的空间浪费（请参阅“ 指定平均群集密钥及其关联行所需的空间 ” ） 。

每个群集键值的行太多可能导致额外搜索以查找该键的行。过于笼统的值（例如， male和female ）上的群集键导致过度搜索，并且可能导致比没有群集时更差的性能。

群集索引不能是唯一的，也不能包含定义为long的列。

22.2.3指定平均群集密钥及其关联行所需的空间

CREATE CLUSTER语句有一个可选子句SIZE ，它是平均簇密钥及其关联行所需的估计字节数。

执行以下任务时，数据库使用SIZE参数：

  • 估计可以适合群集数据块的群集密钥（和关联的行）的数量

  • 限制放置在集群数据块中的集群键的数量。这最大化了群集中密钥的存储效率。

SIZE不限制给定群集密钥可以使用的空间。例如，如果设置SIZE使得两个群集密钥可以适合一个数据块，则任何一个群集密钥仍然可以使用任何数量的可用数据块空间。

默认情况下，数据库仅在群集数据段的每个数据块中存储一个群集密钥及其关联的行。虽然块大小可能因操作系统而异，但每个块的一个键的规则将保持不变，因为群集表将导入到其他系统上的其他数据库。

如果给定群集键值的所有行都不能适合一个块，则将这些块链接在一起以加速使用给定键访问所有值。集群索引指向块链的开头，每个块包含集群键值和关联的行。如果群集SIZE使得多个键适合块，则块可以属于多个链。

22.2.4指定每个群集和群集索引行的位置

始终在CREATE CLUSTER / INDEX语句中指定TABLESPACE子句以标识用于存储新集群或索引的表空间。

如果您具有适当的权限和表空间配额，则可以在当前联机的任何表空间中创建新的群集和关联的群集索引。

可以在不同的表空间中创建集群及其集群索引。实际上，在存储在不同存储设备上的不同表空间中创建集群及其索引允许以最小的磁盘争用同时检索表数据和索引数据。

22.2.5估计簇大小和设置存储参数

在创建集群之前，请估计集群大小并设置集群数据段的存储参数。

以下是在创建集群之前估计集群大小的好处：

  • 您可以使用群集的估计大小组合以及索引和重做日志文件的估计值来确定保存目标数据库所需的磁盘空间量。根据这些估算，您可以进行正确的硬件购买和其他决策。

  • 您可以使用单个群集的估计大小来更好地管理群集将使用的磁盘空间。创建群集后，您可以设置适当的存储参数并提高使用群集的应用程序的I / O性能。

使用CREATE CLUSTER或ALTER CLUSTER语句的STORAGE子句设置集群数据段的存储参数，而不是将表放入集群的单个CREATE或ALTER语句。创建或更改聚簇表时指定的存储参数将被忽略。为群集设置的存储参数会覆盖表存储参数。

  • 创建群集索引

父主题： 管理群集

22.3.1创建集群

您可以使用CREATE CLUSTER语句创建集群。

要在架构中创建集群，您必须具有CREATE CLUSTER系统特权以及用于包含集群的表空间的配额或UNLIMITED TABLESPACE系统特权。

要在另一个用户的模式中创建集群，您必须具有CREATE ANY CLUSTER系统特权，并且所有者必须具有用于包含集群的表空间的配额或UNLIMITED TABLESPACE系统特权。

以下语句创建一个名为emp_dept的集群，该集群存储由deptno列聚类的emp和dept表：

CREATE CLUSTER emp_dept（deptno NUMBER（3））SIZE 600 TABLESPACE用户存储（初始200K下一个300K MINEXTENTS 2 PCTINCREASE 33）;

如果未指定INDEX关键字，则在此示例中为true，默认情况下会创建索引集群。当指定了哈希参数（ HASHKEYS ， HASH IS或SINGLE TABLE HASHKEYS ）时，您还可以创建HASH集群。散列集群在管理散列集群中进行了描述。

父主题： 创建使用它们的群集和对象

22.3.2创建聚簇表

您可以使用带有CLUSTER子句的CREATE TABLE语句在集群中CREATE TABLE 。

要在集群中创建表，您必须具有CREATE TABLE或CREATE ANY TABLE系统特权。您不需要表空间配额或UNLIMITED TABLESPACE系统特权来在集群中创建表。

例如，可以使用以下语句在emp_dept集群中创建emp和dept表：

CREATE TABLE emp（empno NUMBER（5）PRIMARY KEY，ename VARCHAR2（15）NOT NULL，...deptno NUMBER（3）REFERENCES dept）CLUSTER emp_dept（deptno）; CREATE TABLE dept（deptno NUMBER（3）PRIMARY KEY，...）CLUSTER emp_dept（deptno）;

注意：

您可以在CREATE TABLE语句中指定聚簇表的架构。群集表可以与包含群集的架构位于不同的架构中。此外，列的名称不需要匹配，但它们的结构必须匹配。

22.3.3创建群集索引

必须先创建集群索引，然后才能将任何行插入任何集群表。

要创建集群索引，必须满足下列条件之一：

  • 您的架构包含群集。

  • 您具有CREATE ANY INDEX系统特权。

在任何一种情况下，您还必须具有用于包含集群索引的表空间的配额或UNLIMITED TABLESPACE系统特权。

以下语句为emp_dept集群创建集群索引：

CREATE INDEX emp_dept_index ON CLUSTER emp_dept TABLESPACE用户存储（初始50K下一个50K MINEXTENTS 2 MAXEXTENTS 10 PCTINCREASE 33）;

集群索引子句（ ON CLUSTER ）标识正在为其创建集群索引的集群emp_dept 。该语句还明确指定了群集和群集索引的多个存储设置。

22.4改变使用它们的集群和对象

您可以更改群集以更改其物理属性，大小和默认并行度。

  • 改变集群
    您可以使用ALTER CLUSTER语句更改集群。
  • 改变聚类表
    您可以使用ALTER TABLE语句更改聚簇表，但无法使用ALTER TABLE语句设置聚簇表的某些参数。
  • 改变群集索引
    您可以像处理其他索引一样更改集群索引。

父主题： 管理群集

22.4.1改变集群

您可以使用ALTER CLUSTER语句更改集群。

要更改群集，您的架构必须包含群集，或者您必须具有ALTER ANY CLUSTER系统特权。您可以更改现有群集以更改以下设置：

  • 物理属性（ INITRANS和存储特性）

  • 存储群集键值（ SIZE ）的所有行所需的平均空间量

  • 默认的并行度

此外，您可以为集群显式分配新扩展区，或者在集群末尾取消分配任何未使用的扩展区。数据库根据需要为群集的数据段动态分配其他扩展区。但是，在某些情况下，您可能希望为群集显式分配其他范围。例如，使用Real Application Clusters时，可以为特定实例显式分配集群范围。您可以使用带有ALLOCATE EXTENT子句的ALTER CLUSTER语句为集群分配新范围。

更改群集的群集大小参数（ SIZE ）时，新设置将应用于群集使用的所有数据块，包括已分配的块和随后为群集分配的块。已经为表分配的块在必要时（不是立即）重新组织。

当您更改集群的事务条目设置INITRANS ， INITRANS的新设置INITRANS适用于随后为集群分配的数据块。

存储参数INITIAL和MINEXTENTS不能更改。其他存储参数的所有新设置仅影响随后为群集分配的扩展区。

要更改群集，请使用ALTER CLUSTER语句。

22.4.2更改聚簇表

您可以使用ALTER TABLE语句更改聚簇表，但无法使用ALTER TABLE语句设置聚簇表的某些参数。

但是，您在群集表的ALTER TABLE语句中设置的任何数据块空间参数，事务条目参数或存储参数都会生成错误消息（ ORA-01771, illegal option for a clustered table ）。数据库使用群集的参数用于所有群集表。因此，您只能使用ALTER TABLE语句添加或修改列，删除非群集键列，或添加，删除，启用或禁用群集表的完整性约束或触发器。有关更改表的信息，请参阅“ 更改表 ” 。

22.5删除使用它们的集群和对象

使用DROP CLUSTER语句删除集群。使用DROP TABLE语句删除聚簇表。您使用DROP INDEX语句删除集群索引。

  • 掉落集群
    您可以使用DROP CLUSTER语句删除集群。
  • 删除集群表
    可以单独删除聚簇表，而不会影响群集，其他群集表或群集索引。使用DROP TABLE语句删除聚簇表，就像删除非聚簇表一样。
  • 删除群集索引
    可以删除集群索引，而不会影响集群或其集群表。但是，如果没有集群索引，则无法使用集群表;您必须重新创建群集索引以允许访问群集。

父主题： 管理群集

22.5.1丢弃集群

您可以使用DROP CLUSTER语句删除集群。

如果不再需要群集中的表，则可以删除群集。删除群集时，群集中的表以及相应的群集索引也将被删除。属于集群数据段和集群索引的索引段的所有扩展区都将返回到包含表空间，并可用于表空间中的其他段。

要删除不包含表的集群及其集群索引，请使用DROP CLUSTER语句。例如，以下语句删除名为emp_dept的空集群：

DROP CLUSTER emp_dept;

如果集群包含一个或多个集群表，并且您打算同时删除表，请添加DROP CLUSTER语句的INCLUDING TABLES子句，如下所示：

DROP CLUSTER emp_dept包括表格;

如果未包含INCLUDING TABLES子句且集群包含表，则会返回错误。

如果群集中的一个或多个表包含由群集外的表的FOREIGN KEY约束引用的主键或唯一键，则除非还删除了相关的FOREIGN KEY约束，否则无法删除该群集。这可以使用DROP CLUSTER语句的CASCADE CONSTRAINTS子句轻松完成，如以下示例所示：

DROP CLUSTER emp_dept包括表格级联约束;

如果不使用CASCADE CONSTRAINTS子句并且存在CASCADE CONSTRAINTS则数据库将返回错误。

22.5.2删除聚簇表

可以单独删除聚簇表，而不会影响群集，其他群集表或群集索引。使用DROP TABLE语句删除聚簇表，就像删除非聚簇表一样。

要删除群集，您的架构必须包含群集，或者您必须具有DROP ANY CLUSTER系统特权。即使群集表不归群集所有者所有，也不需要其他权限来删除包含表的群集。

请参阅“ 删除表列 ” 。

注意：

从群集中删除单个表时，数据库将单独删除表的每一行。要在删除整个集群时最大限度地提高效率，请使用带有INCLUDING TABLES子句的DROP CLUSTER语句删除包含所有表的集群。仅当您希望保留群集的其余部分时，才从群集中删除单个表（使用DROP TABLE语句）。

22.5.3删除群集索引

可以删除集群索引，而不会影响集群或其集群表。但是，如果没有集群索引，则无法使用集群表;您必须重新创建群集索引以允许访问群集。

有时会删除群集索引作为重建碎片群集索引的过程的一部分。

注意：

无法删除散列集群索引。
