14内存中初始化参数

几个初始化参数控制IM列存储的行为。

本章仅是摘要。Oracle数据库参考包含所有数据库视图的完整信息。

表14-1与IM列存储库相关的初始化参数

初始化参数 描述

INMEMORY_ADG_ENABLED

指示备用数据库上的In-Memory for Oracle Active Data Guard是启用（ TRUE ）还是禁用（ FALSE ）。默认值为TRUE 。

对于Active Data Guard，介质恢复必须在应用重做时检索In-Memory对象，并在查询超前后使相关对象无效。此参数控制介质恢复是否执行检索和无效。

您只能在未运行备用恢复时修改此系统级参数。如果备用数据库使用Oracle RAC，则必须在每个实例上将此参数设置为相同的值。

INMEMORY_AUTOMATIC_LEVEL

通过帮助确保工作数据集始终位于IM列存储中，自动管理IM列存储。

您可以设置以下值：

  • OFF ：这是默认值。设置此值后，将禁用“自动内存”。此值将IM列存储返回到Oracle Database 18c之前存在的行为。如果您不希望稳定的工作数据集，则将参数设置为DISABLE 。

  • LOW ：设置此值时，数据库会在内存压力下从IM列存储区中清除冷段。

  • MEDIUM ：此级别包括一个额外的优化，可确保首先填充因内存压力而未填充的任何热段。

INMEMORY_CLAUSE_DEFAULT

为新表和物化视图指定默认IM列存储子句。

此参数支持以下选项：

  • 要指定新表和物化视图没有默认IM列存储子句，请保留此参数未设置或将其设置为空字符串。将此参数设置为NO INMEMORY与将其设置为默认值（空字符串）具有相同的效果。

  • 要指定该子句是所有新表和INMEMORY化视图的缺省子句，请将此参数设置为有效的INMEMORY子句。该子句可以包括IM列存储压缩方法和数据填充选项的有效子句。选项是：

    • 如果子句以INMEMORY ，那么所有新表和物化视图（包括那些没有INMEMORY子句的INMEMORY ）都将填充在IM列存储中。

    • 如果该子句省略INMEMORY ，则它仅适用于在创建期间使用INMEMORY子句为IM列存储启用的新表和物化视图。

INMEMORY_EXPRESSIONS_USAGE

控制哪些IM表达式有资格在IM列存储中填充。

此参数支持以下选项：

  • ENABLE

    数据库将静态和动态IM表达式填充到IM列存储中。设置此值会增加某些表的内存占用量。这是默认值。

  • STATIC_ONLY

    静态配置使IM列存储能够缓存OSON（二进制JSON）列，这些列标记有IS_JSON检查约束。在内部，OSON列是名为SYS_IME_OSON的隐藏虚拟列。

  • DYNAMIC_ONLY

    数据库仅填充已作为SYS_IME隐藏虚拟列添加到表中的常用或“热”表达式。设置此值会增加某些表的内存占用量。

  • DISABLE

    数据库不会将任何IM表达式（无论是静态还是动态）填充到IM列存储中。

注意：

IM表达式不支持与NLS相关的数据类型。

INMEMORY_FORCE

启用或禁用IM列存储的表和实例化视图。

此参数支持以下选项：

  • 要允许INMEMORY或NO INMEMORY属性确定填充，请将此参数设置为DEFAULT （默认值）。

  • 要禁用IM列存储的所有表和实例化视图，请将此参数设置为OFF 。

INMEMORY_MAX_POPULATE_SERVERS

指定用于填充的最大空间管理工作进程数（W nnn ），以便进程不会使系统的其余部分超载。

根据系统中的核心数将此参数设置为适当的值。默认值是有效CPU线程数的一半或PGA_AGGREGATE_TARGET值除以512 MB（以较小者为准）。

注意：当INMEMORY_MAX_POPULATE_SERVERS设置为0 ，无法在IM列存储中填充对象

INMEMORY_OPTIMIZED_ARITHMETIC

控制NUMBER列是否以In-Memory优化格式存储。

此参数支持以下选项：

  • DISABLE （默认值）不使用优化编码。

  • ENABLE以优化的格式对数字进行编码，以便使用SIMD硬件进行本机计算。此优化适用于具有FOR QUERY LOW压缩的表。

INMEMORY_QUERY

指定是否允许In-Memory查询。

此参数支持以下选项：

  • ENABLE （默认值）允许查询访问填充的对象。

  • DISABLE阻止访问填充的对象。

INMEMORY_SIZE

设置数据库实例中IM列存储的大小。

默认值为0 ，用于禁用IM列存储。最小非零设置为100M 。

INMEMORY_TRICKLE_REPOPULATE_SERVERS_PERCENT

限制执行涓流重新填充的总人口和人口再增长过程的百分比。

此参数的值是INMEMORY_MAX_POPULATE_SERVERS初始化参数值的百分比。例如，如果INMEMORY_TRICKLE_REPOPULATE_SERVERS_PERCENT为5％，并且INMEMORY_MAX_POPULATE_SERVERS为20，则IM列存储使用平均1核（.05 * 20）进行涓流重新填充。

INMEMORY_VIRTUAL_COLUMNS

控制在IM列存储中填充的表达式。

此参数支持以下选项：

  • MANUAL （默认值）允许明确指定为INMEMORY的IM虚拟列的INMEMORY 。

  • ENABLE允许INMEMORY表中所有IM虚拟列的INMEMORY ，除非已从IM列存储中明确排除这些列。

  • DISABLE指定没有IM虚拟列符合填充条件。

OPTIMIZER_INMEMORY_AWARE

控制Database In-Memory中的优化程序成本模型增强功能。

此参数支持以下选项：

  • TRUE （默认值）优化引用INMEMORY对象的SQL语句。

  • FALSE在优化期间忽略表的INMEMORY属性。
