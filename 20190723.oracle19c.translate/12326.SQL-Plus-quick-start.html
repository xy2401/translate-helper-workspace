SQL * Plus概述

SQL * Plus是一个交互式批处理查询工具，随每个Oracle数据库安装一起安装。它有一个命令行用户界面。

还有SQL * Plus Instant Client，它是一个独立的命令行界面，可在支持OCI Instant Client的平台上使用。SQL * Plus Instant Client连接到任何可用的Oracle数据库，但不需要自己的Oracle数据库安装。有关OCI Instant Client的更多信息，请参阅Oracle Call Interface Programmer's Guide 。

SQL * Plus有自己的命令和环境，它提供对Oracle数据库的访问。它使您能够输入和执行SQL，PL / SQL，SQL * Plus和操作系统命令来执行以下操作：

  • 格式化，对查询结果执行计算，存储和打印

  • 检查表和对象定义

  • 开发和运行批处理脚本

  • 执行数据库管理

您可以使用SQL * Plus以交互方式生成报告，以批处理方式生成报告，并将结果输出到文本文件，屏幕或HTML文件以便在Internet上浏览。您可以使用SQL * Plus的HTML输出工具动态生成报告。

谁可以使用SQL * Plus

SQL * Plus，SQL和PL / SQL命令语言功能强大，足以满足具有一定数据库经验的用户的需求，而且对于刚刚学习使用Oracle数据库的新用户来说足够简单。

SQL * Plus 语言易于使用。例如，要使用标题“Family Name”重命名标记为LAST_NAME的列，请输入命令：

COLUMN LAST_NAME标题'姓氏'

同样，要列出EMPLOYEES表的列定义，请输入命令：

描述员工

SQL * Plus命令行架构

SQL * Plus命令行使用双层模型，包括：

  • 客户端（命令行用户界面）。

  • 数据库（Oracle数据库）。

这两层可能在同一台计算机上。

SQL * Plus客户端

命令行用户界面是基于字符的终端实现。

Oracle数据库

Oracle Database Net组件提供 SQL * Plus客户端和Oracle数据库之间的通信。

SQL * Plus先决条件

SQL * Plus是Oracle数据库的一个组件。安装Oracle数据库时，默认情况下会安装SQL * Plus。

Oracle数据库和SQL * Plus的某些方面因计算机和操作系统而异。“Oracle数据库安装指南”中针对SQL * Plus支持的每个操作系统讨论了这些主题。

在运行SQL * Plus之前有什么必要？

  • 安装Oracle数据库或Oracle客户端。有关您的操作系统，请参阅http://www.oracle.com/technetwork/的“Oracle数据库安装指南”。

  • 在安装期间或从数据库管理员处获取Oracle数据库登录用户名和密码。请参阅登录用户名和密码 。

  • 确保安装了示例数据库，并且在Oracle数据库安装期间您具有登录用户名和密码。请参阅关于示例模式和SQL * Plus 。

  • 在安装期间创建默认数据库，或从数据库管理员获取要连接到的Oracle数据库的连接标识符。请参见关于连接到数据库 。

  • 确保已启动要连接的数据库。请参阅STARTUP命令。

SQL * Plus日期格式

SQL * Plus中的默认日期格式由数据库NLS_DATE_FORMAT参数确定，并且可以使用显示两位数年份的日期格式。您可以在SELECT语句中使用SQL TO_CHAR函数或SQL * Plus COLUMN FORMAT命令来控制日期在报表中的显示方式。

启动SQL * Plus命令行

SQL * Plus可执行文件通常安装在$ ORACLE_HOME / bin中，它通常包含在操作系统PATH环境变量中。您可能需要将目录更改为$ ORACLE_HOME / bin目录以启动SQL * Plus。

在以下示例中，系统会提示您输入数据库帐户密码。

使用Easy Connection标识符连接到mymachine上运行的MYDB数据库中的HR模式的示例是：

sqlplus hr @ \“// mymachine.mydomain：port / MYDB \”

使用网络服务名称的示例是：

sqlplus hr @ MYDB

网络服务名称可以存储在多个位置，包括Oracle名称。有关详细信息，请参阅“网络服务参考指南”

如果要使用在本地Oracle Net tnsnames.ora文件中配置的网络服务名称，请将环境变量TNS_ADMIN设置为包含tnsnames.ora文件的目录。例如，在UNIX上，如果您的tnsnames.ora文件位于/ home / user1中，并且它定义了网络服务名称MYDB2：

TNS_ADMIN = / home / user1导出TNS_ADMIN sqlplus hr @ MYDB2

此示例假定已设置ORACLE_HOME环境变量，并且$ ORACLE_HOME / network / admin / tnsnames.ora或ORACLE_HOME \ network \ admin \ tnsnames.ora文件定义网络服务名称MYDB3：

sqlplus hr @ MYDB3

可以将TWO_TASK（在UNIX上）或LOCAL（在Windows上）环境变量设置为连接标识符。这样就无需在SQL * Plus或SQL * Plus Instant Client中建立连接时显式输入连接标识符。此UNIX示例连接到称为MYDB4的数据库：

TNS_ADMIN = / home / user1 export TNS_ADMIN TWO_TASK = MYDB4 export TWO_TASK sqlplus hr

启动SQL * Plus并连接到默认数据库

  1. 打开UNIX或Windows终端并输入SQL * Plus命令：
    SQLPLUS
  2. 出现提示时，输入您的Oracle数据库用户名和密码。如果您不知道Oracle数据库用户名和密码，请询问您的数据库管理员。
  3. 或者，以下列形式输入SQL * Plus命令：
    sqlplus 用户名

    系统会提示您输入密码。

  4. SQL * Plus启动并连接到默认数据库。

    现在，您可以在SQL>提示符下开始输入和执行SQL，PL / SQL和SQL * Plus语句和命令。

关于启动SQL * Plus Instant Client

SQL * Plus Instant Client是SQL * Plus命令行，无需安装Oracle数据库。有关使用它的信息，请参阅启动SQL * Plus命令行 。

由于SQL * Plus Instant Client不包含数据库，因此它始终是来自任何数据库服务器的“远程”。要连接到数据库，必须使用Oracle Net连接标识符指定数据库。

如果未设置TNS_ADMIN，则检查依赖于操作系统的目录集以查找tnsnames.ora。此搜索路径包括查找由network / admin / tnsnames.ora的ORACLE_HOME环境变量指定的目录。这是为SQL * Plus Instant Client设置ORACLE_HOME环境变量的唯一原因。如果在运行Instant Client应用程序时设置了ORACLE_HOME，则必须将其设置为存在的目录。

关于连接到不同的数据库

在现有的命令行会话中，输入以下格式的CONNECT命令：

SQL> connect username @ connect_identifier

系统会提示您输入密码。

关于示例模式和SQL * Plus

示例模式包含在Oracle数据库中。本指南中的示例使用人力资源（HR）示例模式的EMP_DETAILS_VIEW视图。此架构包含虚构公司的人事记录。要查看视图的列详细信息，请输入EMP_DETAILS_VIEW

描述EMP_DETAILS_VIEW

有关该示例的更多信息架构，请参阅Oracle数据库示例架构指南。

解锁样品表

人力资源（HR） 示例模式是作为默认Oracle数据库安装的一部分安装的。HR帐户默认为锁定状态。

您必须先解锁HR帐户，然后才能使用HR示例模式。要解锁HR帐户，请以SYSTEM用户身份登录并输入以下命令，其中your_password是您要为用户HR定义的密码：

由your_password ACCOUNT UNLOCK识别的ALTER USER HR;

有关解锁HR帐户的更多信息，请参阅Oracle数据库示例模式指南。HR用户主要是为了使您能够访问HR示例模式，并且必须使您能够运行本指南中的示例。

数据库中的每个表都由特定用户“拥有”。你可能想拥有自己的样本表的副本在尝试本指南中的示例时使用。获取自己的HR表副本 ，请参阅您的DBA或查看Oracle数据库示例模式指南，或者您可以使用脚本HR_MAIN.SQL创建HR表，该脚本位于UNIX上的以下目录中：

$ ORACLE_HOME /演示/模式/ human_resources / hr_main.sql

在Windows上的以下目录中：

ORACLE_HOME \ DEMO \ SCHEMA \ HUMAN_RESOURCES \ HR_MAIN.SQL

创造命令行SQL * Plus中的HR表，执行以下操作：

  1. 向您的DBA询问您的Oracle数据库帐户用户名和密码。
  2. 登录SQL * Plus。
  3. 在UNIX上，在SQL * Plus提示符处输入以下命令：
    SQL> @？/DEMO/SCHEMA/HUMAN_RESOURCES/HR_MAIN.SQL

    在Windows上，在SQL * Plus提示符处输入以下命令：

    SQL> @？\ DEMO \ SCHEMA \ HUMAN_RESOURCES \ HR_MAIN.SQL

去除示例表，执行相同的步骤，但将HR_DROP.SQL替换为HR_MAIN.SQL。
