4管理Oracle RAC One节点

Oracle Real Application Clusters一个节点（Oracle RAC One Node）是在集群中的一个节点上运行的Oracle Real Application Clusters（Oracle RAC）数据库的单个实例。此选项增加了Oracle为数据库整合提供的灵活性。您可以以最小的开销将许多数据库整合到一个集群中，同时还提供故障转移保护，在线滚动补丁应用程序以及操作系统和Oracle集群件的滚动升级的高可用性优势。

本章包括以下主题：

  • 创建Oracle RAC One节点数据库

  • 转换数据库

  • 在线数据库重定位

创建Oracle RAC One节点数据库

您可以使用Fleet Patching and Provisioning或Database Configuration Assistant（DBCA）创建Oracle RAC One Node数据库，与任何其他Oracle数据库一样（手动创建的脚本也是有效的替代方案）。

您可以使用Fleet Patching and Provisioning创建Oracle RAC One Node数据库，使用-dbtype RACONENODE参数rhpctl add database命令。您还可以使用rhpctl add workingcopy命令包含Oracle RAC One Node数据库。

Oracle RAC One Node数据库也可能是从单实例Oracle数据库（例如，使用rconfig ）或Oracle RAC数据库进行转换的结果。通常，Oracle提供的工具会将Oracle RAC One Node数据库注册到Oracle Clusterware。根据您的配置，可能不会自动注册Oracle RAC One Node数据库和Oracle Clusterware。如果是这种情况，请按照本节中的步骤向Oracle Clusterware注册Oracle RAC One Node数据库。

注意：

Oracle建议您使用服务器控制实用程序（SRVCTL）管理Oracle RAC One Node数据库。您只能使用SRVCTL执行某些操作（例如联机数据库重定位）。

如果Oracle RAC One Node数据库未自动向Oracle Clusterware注册，请使用srvctl add database命令将Oracle RAC One Node数据库添加到群集。例如：

$ srvctl add database -dbtype RACONENODE [-server server_list ] [-instance instance_name ] [-timeout timeout ]

添加管理员管理的Oracle RAC One Node数据库时，请使用-server选项和-instance选项。

对于Oracle RAC One Node数据库，您必须至少配置一个动态数据库服务（除了默认数据库服务之外，还与之相对）。使用管理员管理的Oracle RAC One Node数据库时，与任何其他Oracle RAC数据库一样执行服务注册。将服务添加到策略管理的Oracle RAC One Node数据库时，SRVCTL不接受任何放置信息，而是使用SERVER_POOLS属性的值配置这些服务。

注意：

添加管理员管理的Oracle RAC One Node数据库时，您可以选择使用srvctl add database命令的-instance instance_name选项提供实例前缀。然后，实例的名称将是prefix _1 。如果未指定实例前缀，则数据库唯一名称的前12个字符将成为前缀。实例名称在联机数据库重定位期间更改为prefix _2 ，并在后续联机数据库重定位期间恢复为prefix _1 。故障转移时使用相同的实例名称。

转换数据库

使用SRVCTL，你可以将具有一个实例的Oracle RAC数据库转换为Oracle RAC One Node数据库，反之亦然。

本节包括以下主题：

  • 将数据库从Oracle RAC转换为Oracle RAC One节点

  • 将数据库从Oracle RAC One节点转换为Oracle RAC

将数据库从Oracle RAC转换为Oracle RAC One节点

使用SRVCTL将Oracle RAC数据库转换为Oracle RAC One Node数据库。

在将Oracle RAC数据库转换为Oracle RAC One Node数据库之前，必须首先确保Oracle RAC数据库只有一个实例。如果您的Oracle RAC数据库是管理员管理的并且具有多个实例，则必须删除除使用srvctl remove instance命令之外的所有实例。如果您的Oracle RAC数据库是策略管理的并且具有多个实例，则必须使用srvctl stop instance命令srvctl stop instance除一个实例以外的所有实例。

如果Oracle RAC数据库是管理员管理的，则必须更改所有服务的配置，以将首选实例设置为要在转换后保留为Oracle RAC One Node数据库的实例。如果任何服务具有PRECONNECT TAF策略，则在开始转换过程之前，必须将其TAF策略更新为BASIC或NONE 。这些服务必须不再具有任何可用实例。

如果Oracle RAC数据库是策略管理的，那么在将Oracle RAC数据库转换为Oracle RAC One Node数据库之前，必须更改所有服务的配置，以便它们都使用相同的服务器池。

您可以使用srvctl convert database命令srvctl convert database具有一个实例的Oracle RAC数据库转换为Oracle RAC One Node数据库，如下所示：

$ srvctl convert database -db db_unique_name -dbtype RACONENODE [-instance instance_name -timeout timeout ] -w timeout ]

注意：

要转换为Oracle RAC One Node的Oracle RAC数据库必须使用Oracle Managed Files（以启用自动线程分配）或至少具有两个重做线程。

将数据库从Oracle RAC One节点转换为Oracle RAC

您可以通过以Oracle RAC One Node数据库所有者身份登录并输入以下SRVCTL命令，将Oracle RAC One Node数据库转换为Oracle RAC数据库：

srvctl转换数据库-db db_unique_name -dbtype RAC

如果要使用联机数据库重定位重定位要转换为Oracle RAC的数据库，或者联机数据库重定位失败，则必须在运行srvctl convert database命令之前退出或完成重定位。

运行此命令后，除数据库服务器池外，还必须为每个数据库服务创建服务器池。必须将数据库服务使用的服务器池的SERVER_NAMES值设置为从Oracle RAC One节点转换为Oracle RAC节点的节点。您可以使用CRSCTL实用程序或Oracle Enterprise Manager来创建和配置服务器池。

将管理员管理的Oracle RAC One Node数据库转换为Oracle RAC数据库可配置所有数据库服务，以便单实例数据库成为该服务的首选实例。转换数据库后，可以使用srvctl add instance命令srvctl add instance添加到数据库。

将策略管理的Oracle RAC One Node数据库转换为Oracle RAC数据库会将所有数据库服务设置为UNIFORM基数。它还会导致重用当前运行数据库的服务器池。转换重新配置数据库以在服务器池中的所有节点上运行。该命令不会启动任何其他实例，但运行srvctl start database命令会在服务器池中的所有节点上启动数据库。

在线数据库重定位

您可以使用联机数据库重定位功能将Oracle RAC One Node数据库重定位到另一个节点，同时仍保持服务可用性。

仅在计划的联机数据库重定位期间，才会创建Oracle RAC One Node数据库的第二个实例，以便在将数据库重定位到新节点时可以继续任何数据库会话。您只能使用Oracle RAC One Node数据库进行在线数据库重定位，但无论管理方式如何（管理员或策略管理），您都无法使用Oracle RAC数据库进行在线数据库重定位。

您可以使用srvctl relocate database命令配置在重新定位的数据库启动和迁移服务之后，在数据库的前一个实例停止之前的时间量。此配置的时间量不是整个操作所用时间的上限，而是仅控制重定位数据库在停止前一个实例之前等待连接从前一个实例迁移到新实例的时间。

发生在线数据库重定位，如下所示：

  1. 在其他位置启动新的数据库实例。

  2. 将所有服务移动到重定位的实例。

  3. 等待所有连接迁移到重定位的实例。

  4. 关闭以前的数据库实例，强制任何剩余的连接移动到重定位的实例。

联机重定位超时是您配置完成步骤3的时间。

如果您的Oracle RAC One Node数据库是管理员管理的，则在您启动重定位时，要将数据库实例重新定位到的目标节点必须是Free服务器池的一部分。如果Free服务器池中有备用目标节点，则此节点将添加到Oracle RAC One Node数据库的候选列表中。

注意：

  • 将数据库实例重定位到当前不在数据库的候选服务器列表中的目标节点时，必须将密码文件（如果已配置）复制到目标节点，除非您使用存储在Oracle ASM中的共享密码文件。

    当您使用基于密码文件的身份验证远程管理没有任何共享密码文件的Oracle RAC One Node数据库时，您必须在每个可以运行数据库的节点上有两个密码文件：一个名为SID_prefix _1 ，另一个名为SID_prefix _2 。每次更新密码文件时，都必须将这两个文件重新复制到所有候选节点。对于策略管理和管理员管理的数据库都是如此。

    Oracle建议使用Oracle Clusterware启动和停止数据库，并在数据字典中定义用户以进行其他管理。

  • 在重定位数据库实例之前，必须确保将数据库服务用户添加到钱包中。运行crsctl query wallet -type OSUSER -all以检查数据库服务用户是否在钱包中。如果没有，则运行crsctl add wallet -type OSUSER -user user_name -passwd将数据库服务用户添加到钱包。

使用srvctl relocate database命令启动Oracle RAC One Node数据库的重定位。例如：

$ srvctl重定位数据库-d rac1 -n node7
