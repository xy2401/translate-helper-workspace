如果Oracle库存不存在，则创建Oracle库存组

如果oraInst.loc文件不存在，则创建Oracle Inventory组。

OINSTALL组的成员被授予写入Oracle中央库存（ oraInventory ）的权限，以及Oracle安装所有者用户的其他系统权限。

Oracle安装所有者应始终将要指定为OINSTALL组（ oinstall ）的组作为其主要组。确保该组可用作所有计划的Oracle软件安装所有者的主要组。默认情况下，如果oraInst.loc文件不存在且未标识Oracle中央清单（ oraInventory ），则安装程序会将运行安装的安装所有者的主组指定为OINSTALL组。

以下示例使用组ID号54321创建oraInventory组oinstall 。 

＃/ usr / sbin / groupadd -g 54321 oinstall

注意：

对于Oracle Clusterware上的安装，群集中的所有节点上的组ID和用户ID必须相同。确保要在每个集群成员节点上使用要使用的组和用户标识，并确认集群安装所有者的每个Oracle Grid Infrastructure的主组具有相同的名称和组标识。
