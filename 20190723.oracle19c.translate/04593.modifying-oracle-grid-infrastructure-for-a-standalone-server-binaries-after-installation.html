安装后为独立服务器二进制文件修改Oracle Grid Infrastructure

安装后，如果不使用带有opatchauto标志的opatchauto修补二进制文件，则必须停止堆栈，修改软件，然后重新启动堆栈。

您必须首先停止Oracle Restart堆栈以修改Grid home中安装的软件。例如，要应用一次性修补程序或修改Oracle Restart或Oracle ASM使用的任何动态链接库（DLL），必须手动停止堆栈，修改软件，然后重新启动堆栈。

但是，如果使用auto选项opatchauto运行OPatch实用程序，则OPatch会为您停止并启动软件堆栈。如果使用apply选项运行OPatch， opatch apply ，则还必须手动停止并重新启动堆栈。

每次应用操作系统修补程序或操作系统升级后，都必须重新链接Oracle Restart和Oracle ASM二进制文件。

警告：

在重新链接可执行文件之前，必须关闭正在重新链接的Oracle主目录中运行的所有可执行文件。此外，关闭与Oracle共享库链接的应用程序。

修改独立服务器二进制文件的Oracle Grid Infrastructure

  1. 以独立服务器软件所有者用户身份登录Oracle Grid Infrastructure，并将目录更改为Grid_home /bin路径，其中Grid_home是独立服务器主目录的Oracle Grid Infrastructure路径：

    $ cd Grid_home / bin

  2. 使用以下命令关闭Oracle Restart堆栈：

    $ crsctl stop有-f

  3. 以root身份登录并解锁网格主页：

    #cd Grid_home / crs / install＃roothas.sh -unlock

  4. 使用opatch apply应用补丁。

  5. 重新链接二进制文件。

    $ export ORACLE_HOME = Grid_home $ Grid_home / bin / relink

  6. 将网格锁定在家：

    #cd Grid_home / crs / install＃roothas.sh -lock

  7. 输入以下命令以重新启动Oracle Restart堆栈：

    $ crsctl start有

重新链接Oracle Restart和Oracle ASM二进制文件

  1. 以root身份登录并解锁网格主页：
    #cd Grid_home / crs / install＃roothas.sh -unlock
  2. 以grid用户身份登录并重新链接二进制文件：
    $ export ORACLE_HOME = Grid_home $ Grid_home / bin / relink
  3. 再次以root身份登录并执行以下步骤：
    #cd Grid_home / rdbms / install /＃./rootadd_rdbms.sh #cd Grid_home / crs / install＃roothas.sh -lock

    注意：

    与Oracle数据库12起c解除1（12.1.0.2），该roothas.sh脚本替换roothas.pl在Oracle网格基础架构的家脚本。
