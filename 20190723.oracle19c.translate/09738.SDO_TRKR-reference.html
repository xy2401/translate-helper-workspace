33 SDO_TRKR包（位置跟踪）

MDSYS.SDO_TRKR包中包含使用位置跟踪服务器的子程序。

要使用这些子程序，您必须了解位置跟踪服务器中描述的概念和技术。

33.1 SDO_TRKR.CREATE_TRACKING_SET

格式

SDO_TRKR.CREATE_TRACKING_SET（tracking_set_name IN VARCHAR2，num_trkr_queues IN NUMBER（38）DEFAULT 4，num_loc_queues IN NUMBER（38）DEFAULT 1）;

描述

创建跟踪集，特别是位置跟踪服务器所需的表和队列。

参数

tracking_set_name

跟踪集的名称。

num_trkr_queues

要创建以管理跟踪消息的队列数。此参数的默认值为4。对于跟踪区域表中的大量区域，更多跟踪器队列可以提高性能。对于创建的每个跟踪器队列，还会创建一个调度程序作业，因此跟踪器队列的数量也应与系统资源平衡。

num_loc_queues

要创建以管理位置消息的队列数。此参数的默认值为1。如果正在跟踪大量对象，则更多位置队列可以提高性能。对于创建的每个位置队列，还会创建一个调度程序作业，因此位置队列的数量也应与系统资源平衡。

使用说明

此过程将创建位置跟踪服务器所需的必要表和队列。这是启用位置跟踪服务器所需的第一步

有关位置跟踪服务器的概念和使用信息，请参阅位置跟踪服务器 。

例子

以下示例创建一个名为TRACKING_EXAMPLE的跟踪网络，其中包含8个跟踪队列和8个位置队列。

EXECUTE SDO_TRKR.CREATE_TRACKING_SET（'TRACKING_EXAMPLE'，8,8）;

33.3 SDO_TRKR.GET_NOTIFICATION_MSG

格式

SDO_TRKR.GET_NOTIFICATION_MSG（tracking_set_name IN VARCHAR2，deq_wait IN NUMBER（38）DEFAULT DBMS_AQ.NO_WAIT，消息OUT NOTIFICATION_MSG）;

描述

从跟踪集通知队列中获取下一条通知消息。

参数

tracking_set_name

跟踪集的名称。此参数用于构建通知队列的名称。

deq_set_wait

如果队列中没有与搜索条件匹配的消息，则等待消息到达通知队列的秒数。此参数的DEFAULT是DBMS_AQ.NO_WAIT，这意味着操作不会等待。

消息

NOTIFICATION_MSG类型的输出参数。来自通知队列的下一条消息，如果等待时间已过期，则为null。

使用说明

有关位置跟踪服务器的概念和使用信息，请参阅位置跟踪服务器 。

例子

以下示例从跟踪集通知队列中获取通知消息，并将消息内容插入到跟踪集辅助通知表中。它将继续收到通知消息，直到它等待30秒而没有消息到达。

...LOOP SDO_TRKR.GET_NOTIFICATION_MSG（tracking_set_name =>'TRACKING_EXAMPLE'，message => message，deq_wait => 30）; IF（消息IS NULL）然后退出;万一; INSERT INTO tracking_example_notifications（object_id，region_id，time，x，y，state）VALUES（message.object_id，message.region_id，message.time，message.x，message.y，message.state）;结束循环; ...

33.4 SDO_TRKR.SEND_LOCATION_MSGS

格式

SDO_TRKR.SEND_LOCATION_MSGS（tracking_set_name IN VARCHAR2，location_msgs IN LOCATION_MSG_ARR）;

描述

为跟踪集中正在跟踪的对象发送一系列新位置信息。

参数

tracking_set_name

跟踪集的名称。

location_msgs

LOCATION_MSG_ARR类型的消息。

queue_no

应该用于此位置消息的特定位置队列。建议您不要指定此参数，在这种情况下，跟踪服务器会确定用于管理此位置消息的最佳队列。

使用说明

此过程添加新的位置消息以用于跟踪目的。每当为正在跟踪的对象获取新位置时，此过程可用于更新该对象的位置。

location_msgs参数的类型为LOCATION_MSG_ARR，它在位置跟踪服务器的数据类型中定义。

每条消息都包含一个时间戳值，该值是更新的时间，并与位置信息一起保存。

有关位置跟踪服务器的概念和使用信息，请参阅位置跟踪服务器 。

例子

以下示例添加新的位置消息。该消息包括对象id（1），当前时间以及对象新位置的X和Y坐标（2,2）

EXECUTE SDO_TRKR.SEND_LOCATION_MSGS（'TRACKING_EXAMPLE'，LOCATION_MSG_ARR（location_msg（1，CURRENT_TIMESTAMP（），2,2）））;

33.5 SDO_TRKR.SEND_TRACKING_MSG

格式

SDO_TRKR.SEND_TRACKING_MSG（tracking_set_name IN VARCHAR2，tracking_msg IN TRACKER_MSG）;

描述

将跟踪消息插入跟踪队列。每个跟踪的对象都需要一个TRACKER_MSG。如果在区域1,2和3中跟踪对象1，则需要三个TRACKER_MSG，每个对象ID /区域ID对一个。

参数

tracking_set_name

跟踪集的名称。

tracking_msg

TRACKER_MSG类型的消息。这用于在跟踪的对象与跟踪它们的区域之间创建关系。

使用说明

此过程在跟踪器表中创建新行，该行告知位置跟踪器服务器在哪些区域内跟踪哪些对象。

tracking_ms参数的类型为TRACKER_MSG，它在位置跟踪服务器的数据类型中定义。

有关位置跟踪服务器的概念和使用信息，请参阅位置跟踪服务器 。

例子

以下示例插入ID为4的新跟踪对象，以跟踪名为TRACKING_EXAMPLE的跟踪集中ID为8的区域。当对象在 （ I ）区域内时发送通知消息。如果O已经指定通知将一直与对象发送是区域外 ，而且T已经规定，通知只发送当对象转换进入或离开该区域。 

EXECUTE SDO_TRKR.SEND_TRACKING_MSG（'TRACKING_EXAMPLE'，TRACKER_MSG（4,8，'I'））;
