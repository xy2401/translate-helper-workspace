6在Linux上配置SNA通信包

APPC的Oracle数据库网关使用SNA高级程序编程通信（APPC / LU6.2）协议与OLTP进行通信。 Linux for Intel基于Pentium的32位系统支持APPC由IBM Communications Server for Linux提供。

以下主题介绍如何使用IBM Communications Server for Linux在Linux上为基于Intel Pentium的32位系统配置SNA Server以运行APPC的Oracle数据库网关。

  • 使用SNA安全验证

  • 处理入站连接

  • 独立对抗的LU

  • 为网关创建IBM Communications Server定义

  • 测试连接

注意：

本指南中使用的术语SNA Server通常是指IBM Communications Server。

使用SNA安全验证

当网关收到启动远程事务程序的RPC请求时，网关尝试使用联机事务处理（OLTP）启动APPC对话。在对话开始之前，会话必须在Linux逻辑单元（LU）和OLTP LU之间启动。

SNA及其各种访问方法实现（包括SNA Server和VTAM）在会话启动时提供安全验证，使每个LU能够对其伙伴进行身份验证。在网关和OLTP应用程序开始对话并处理对话级安全数据之前，此验证完全由网络软件执行。如果使用会话级安全性，则必须在网关SNA服务器定义中以及要访问的OLTP中的类似参数结构中建立正确的密码信息。有关此主题的详细信息，请参阅相应的通信软件产品文档。

处理入站连接

许多OLTP提供用于操纵入站（客户端）APPC会话请求的安全性行为的选项。有关此主题的详细信息，请参阅相应的OLTP文档。

注意：

对于CICS，网关不支持一个安全选项：在CICS CONNECTION定义中指定的ATTACHSEC=PERSISTENT需要网关中尚不可用的功能。

但是，网关完全支持ATTACHSEC=LOCAL ， ATTACHSEC=IDENTIFY ， ATTACHSEC=VERIFY和ATTACHSEC=MIXIDPE安全选项。

独立对抗的LU

Oracle建议您为APPC的Oracle数据库网关使用独立的LU，因为它们支持多个并行会话或对话。这意味着可以通过独立LU使用相同的OLTP同时激活多个Oracle客户端应用程序。

从属LU仅支持单个活动会话。CP（节点的控制点）在已经活动的对话后面从网关服务器排队其他对话请求。换句话说，对话是针对从属LU的单线程。如果正确定义了网关LU，则无需更改Oracle Database Gateway for APPC和网关服务器的配置。

依赖LU的操作影响是第一个客户端应用程序可以通过Oracle数据库网关与网关服务器发起对话。当该事务处于活动状态时（可能是几秒到几分钟到几小时，具体取决于客户端应用程序和事务的设计方式），启动与同一网关服务器的会话的任何其他客户端应用程序在等待上一个会话后都会挂起。

如果生产应用程序在任何时候只使用一个对话，那么应该没有任何影响。

但是，测试或其他应用程序开发可能需要额外的并发对话。每个对话都要求您在远程主机上定义其他从属LU，以及其他IBM Communication Server配置条目，这些条目定义网关系统上的其他从属LU。应定义附加的侧信息配置文件以使用新的从属LU。应创建新网关实例并将其配置为使用这些新的边信息配置文件。

定义类型

有几种类型的IBM Communications Server定义与网关APPC / LU6.2操作相关。可以使用相应的SNA节点配置菜单创建和编辑每个定义。与网关相关的定义在按层次结构顺序为网关创建IBM Communications Server定义中介绍。本节概述了与Oracle Database Gateway for APPC相关的IBM Communications Server定义。首先讨论层次结构中最低的那些定义类型。这与创建定义的逻辑顺序相匹配。有关IBM Communications Server定义的完整讨论，请参阅IBM Communications Server联机文档。

为网关创建IBM Communications Server定义

IBM Communications Server定义是使用SNA节点配置工具创建的，而服务器的操作是使用SNA节点操作工具完成的，这两个工具都随IBM Communications Server一起提供。SNA定义的维护通常由具有管理员权限的用户完成。

以下部分描述使用SNA节点配置工具为IBM Communications Server创建SNA定义的过程。本节中描述的所有任务都在SNA节点配置中执行。

创建配置

“SNA节点配置”屏幕会提示您指定是创建新配置还是加载现有配置。这些任务基于创建新配置的假设。

SNA节点配置接下来会提示您输入配置方案。

创建节点

每个SNA Server必须定义一个控制点。这通常称为节点定义。要定义节点：

  1. 单击节点 。

  2. 单击“ 创建”

    在“定义节点”对话框中：

    1. 选择基本选项卡。

    2. 在“控制点”，“本地节点ID”和“节点类型”框中输入信息。

    您可以根据SNA网络配置选择“ 高级”选项卡选项。

  3. 单击确定 。

创建设备

配置通信设备：

  1. 单击设备 。
  2. 单击“ 创建”

选择设备类型

选择要用于通信的设备类型。LAN类型是以太网或令牌环连接的网络设备的典型类型。

配置LAN设备

要配置LAN设备：

  1. 选择基本选项卡。
  2. 选择要使用的适配器和本地SAP。其他选项卡提供网络调整参数的选项。
  3. 单击确定 。

创建对等连接

要创建对等连接：

  1. 单击“ 对等连接” 。
  2. 单击“ 创建”

定义链接站

要定义链接站：

  1. 在“基本”选项卡中，输入此连接的“链接站”名称。
  2. 选择连接设备。
  3. 输入目标地址和远程SAP。

定义相邻节点

要定义相邻节点：

  1. 选择“ 相邻节点”选项卡。
  2. 输入远程系统的相邻CP名称，然后选择其CP类型。

    您可能必须选择与默认值不同的传输组（TG）。有关详细信息，请咨询SNA网络管理员。其他选项卡提供有关调整和重新激活的选项。

  3. 单击确定 。

创建本地LU

要为节点创建本地LU：

  1. 单击本地LU 6.2 LU 。
  2. 单击“ 创建”

定义本地LU

要定义本地LU：

  1. 在“基本”选项卡中，输入本地LU的名称，以及可选的别名。该名称必须与此节点的远程主机的本地LU定义匹配。您可以检查其他选项卡以获取同步支持和LU会话限制。
  2. 单击确定 。

创建合作伙伴LU

要创建合作伙伴LU：

  1. 单击Partner LU 6.2 LU 。
  2. 单击“ 创建”

定义合作伙伴LU

要定义合作伙伴LU：

  1. 在“基本”选项卡中，输入远程或伙伴LU的名称，以及可选的别名。
  2. 从现有列表中选择完全限定的CP。您可以检查逻辑记录限制和安全支持的其他选项卡。
  3. 单击确定 。

创建CPI-C边信息配置文件

要定义将用于创建网关的CPI-C配置文件：

  1. 单击CPI-C侧信息定义 。
  2. 单击“ 创建”
