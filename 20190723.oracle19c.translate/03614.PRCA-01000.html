PRCA-01000：ASM磁盘组{0}不存在

原因：尝试无法检索未配置的ASM磁盘组。

操作：如果指定的磁盘组不存在，则使用sqlplus创建磁盘组。在ASM中挂载磁盘组，使用CRS注册磁盘组。

PRCA-01001：ASM磁盘组资源不存在

原因：没有向CRS注册磁盘组资源。

操作：使用sqlplus创建磁盘组并在ASM中装入磁盘组，该磁盘组使用CRS注册磁盘组。

PRCA-01002：无法删除ASM磁盘组{1}的CRS资源{0}

原因：尝试删除指定ASM磁盘组的CRS资源失败。

操作：检查随附的错误消息以获取详细信息。

PRCA-01003：无法创建ASM {0}资源{1}

原因：尝试为ASM添加CRS资源失败。

操作：确保CRS堆栈正在运行，运行srvctl add asm的用户也是CRS主目录的所有者，并检查随附的错误消息以获取详细信息。

PRCA-01005：无法检索ASM {0}的服务器参数文件

原因：无法检索ASM服务器参数文件。

操作：确保CRS堆栈正在运行，已配置ASM资源，并且用户对ASM资源具有读取权限后，请重试该请求。

PRCA-01006：无法更新ASM {1}的服务器参数文件{0}

原因：无法更新ASM服务器参数文件。

操作：确保CRS堆栈正在运行，已配置ASM资源，并且用户对ASM资源具有更新权限后，请重试该请求。

PRCA-01007：ASM {0}未使用初始化参数：{1}

原因：指定的ASM实例不使用显示的初始化参数。

操作：检查随附的错误消息以获取详细信息。

操作：检查随附的错误消息以获取详细信息。

PRCA-01017：ASM {0}不存在

PRCA-01018：无法检索ASM {0}的ASM实例列表

操作：确保crs堆栈已启动且已配置ASM资源。

PRCA-01019：ASM {0}未运行

原因： ASM资源没有任何正在运行的实例。

操作：使用'srvctl start asm'命令启动asm资源。

PRCA-01020：无法为卷设备{1}创建ACFS文件系统资源{0}

原因：无法创建指定卷设备路径的ACFS文件系统资源。可能的原因：1）CRS堆栈不可用。2）ASM实例未运行。3）用户没有使用具有管理访问权限的帐户运行。 （root，Administrator等）4）无法识别用户请求资源使用的挂载点。

操作： 1）验证CRS堆栈是否可用。 （'crsctl check crs'）2）验证ASM是否正在运行。 （srvctl status asm）3）验证用户是否正在以管理访问权限运行。4）验证给予create resource命令的挂载点是否存在。

PRCA-01135：实例名称{0}不是IOServer的活动实例。

PRCA-01137：未为装入点路径{0}配置Oracle ACFS。

原因：尝试查询Oracle自动存储管理群集文件系统（Oracle ACFS）信息属性失败，因为没有为指定的安装点路径配置Oracle ACFS（显示在消息中）。

PRCA-01138：无法启动一个或多个文件系统资源：\ n {0}

原因：尝试启动一个或多个文件系统资源失败，因为一个或多个文件系统资源已在运行或已禁用。

操作：查看附带的消息，并确保在启动它们之前启用了文件系统资源并且未运行。

PRCA-01139：卷名称和磁盘组名称不相等

原因：尝试启动文件系统资源列表失败，因为卷名称和磁盘组名称的数量不同。

操作：重试'srvctl start filesystem'指定相同数量的卷和磁盘组名称。

PRCA-01140：装入点用户列表中包含装入点路径所有者{0}。

原因：尝试配置或修改安装点用户列表被拒绝，因为指示的安装点路径所有者包含在指定的安装点用户列表中。

操作：从装入点用户列表中删除装入点路径所有者。

PRCA-01141：用户{0}是装入点路径所有者。

原因：尝试从挂载点用户列表中添加或删除用户失败，因为指定的用户已配置为挂载点路径所有者。

操作：在命令'srvctl modify filesystem'的'-mountowner'选项中指定新的安装点路径所有者。

PRCA-01142：卷设备{0}已配置为主卷设备。

原因：尝试添加文件系统资源被拒绝，因为一个或多个指定的卷设备已配置为主卷设备。

操作：指定未配置为主卷设备的加速器卷设备。

PRCA-01143：卷设备{0}未格式化为主卷设备{1}的加速器卷设备。

原因：尝试添加文件系统资源被拒绝，因为一个或多个指定的卷设备未格式化为指定主卷设备的加速器卷设备。

操作：使用'mkfs'命令格式化指定的加速器卷设备，然后重试该操作。

PRCA-01144：未安装装入点路径{0}。

原因：尝试从指定的装入点路径查询文件系统信息被拒绝，因为提到的装入点路径未安装在群集中的任何节点中。

操作：使用'srvctl start filesystem'命令指定挂载的挂载点路径或启动文件系统资源。

PRCA-01158：指定的安装组“{0}”无效。

PRCA-01159：指定卷“{0}”的大小小于创建Oracle ACFS所需的最小大小{1} MB。

PRCA-01160：指定卷“{0}”的大小不支持两个以上的节点。

操作：重试该操作，指定最小大小为312 MB的卷乘以群集中的节点数。对于512 MB卷大小，请在双节点群集中重试该操作。
