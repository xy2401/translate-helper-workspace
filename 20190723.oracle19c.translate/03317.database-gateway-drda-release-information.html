远程插入Rowsource

远程插入行源功能允许远程插入，要求本地Oracle数据通过Oracle数据库和Oracle数据库网关工作。此功能要求Oracle数据库和Oracle Database Gateway为12.2或更高版本。

通过Oracle数据库设计，必须在数据库链接站点执行某些分布式语句。但在某些情况下，执行这些查询所需的数据必须从原始Oracle数据库中获取。在同构连接下，远程Oracle数据库将回调源Oracle数据库以获取此类数据。但是在异构连接中，这是不可行的，因为这意味着外部数据存储必须查询只能由发出查询的Oracle实例提供的回调函数或数据。通常，这些类型的语句不能通过Oracle数据库网关支持。

以下类别的SQL语句会导致回调：

  • 任何带有子选择的DML，它指的是Oracle数据库中的一个表。

  • 任何DELETE ， INSERT ， UPDATE或“ SELECT...FOR UPDATE..."包含需要在原始Oracle数据库中执行的SQL函数或语句的SQL语句。

    这些SQL函数包括USER ， USERENV和SYSDATE ;并涉及从原始Oracle数据库中选择数据。

  • 任何涉及Oracle数据库中的表以及远程表中的LONG或LOB列的SQL语句。

可以通过远程插入行源功能工作的远程INSERT语句的示例如下：

INSERT INTO gateway_table @gateway_link select * from local_table ;

结果集和存储过程

用于DRDA的Oracle数据库网关为返回结果集的存储过程提供支持。默认情况下，所有存储过程和函数都不会将结果集返回给用户。要启用结果集，请在初始化参数文件中设置HS_FDS_RESULTSET_SUPPORT参数。

也可以看看：

初始化有关编辑初始化参数文件和HS_FDS_RESULTSET_SUPPORT参数的HS_FDS_RESULTSET_SUPPORT参数。有关Oracle对非Oracle数据库中的结果集的支持的详细信息，请参阅“ Oracle数据库异构连接用户指南” 。

注意：

如果设置HS_FDS_RESULTSET_SUPPORT网关初始化参数，则必须更改所有现有存储过程的过程execute语句的语法，否则将发生错误。

通过Oracle Database Gateway for DRDA访问带有结果集的存储过程时，您将处于异构服务的顺序模式。在过程描述期间，网关将以下信息返回到异构服务：

  • 远程存储过程的所有输入参数

  • 没有输出参数

  • 类型引用游标的一个out参数（对应于存储过程返回的第一个结果集）

客户端程序必须使用虚拟包函数DBMS_HS_RESULT_SET.GET_NEXT_RESULT_SET来获取后续结果集的引用游标。返回的最后一个结果集是过程中的out参数。

访问结果集的限制如下：

  • 必须按照将它们放置在线路上的顺序检索远程存储过程返回的结果集。

  • 在执行存储过程时，将关闭先前执行的存储过程返回的所有结果集，而不管数据是否已被完全检索。

DB2注意事项

12 c版本2（12.2）中存在以下注意事项：

DD基本表和视图

的所有者 DD 基本表和视图是OTGDB2 。这不能改变。

SUBSTR功能后处理

该 SUBSTR函数可以与DRDA服务器不兼容的方式与Oracle数据库一起使用，例如DB2 UDB for z / OS。因此， SUBSTR功能经过后处理。但是，可以允许服务器处理它 原生使用“ Native Semantics ”功能。有关详细信息，请参阅开发应用程序

数据类型限制

有关数据类型的详细信息，请参阅“ DRDA数据类型到Oracle数据类型转换 ” 。

空值和存储过程

空值不会传入或返回， 打电话给存储过程通过网关。

字符串连接数字

DB2 Universal Database不支持字符串数字连接。例如，

SELECT 2 || 2 FROM table @ dblink

不被允许。

GLOBAL_NAMES初始化参数

如果 GLOBAL_NAMES 在Oracle数据库INIT.ORA文件中设置为TRUE ，然后为了能够连接到网关，您必须指定异构服务（HS）初始化参数， HS_DB_DOMAIN ，在网关初始化参数文件中匹配的值 Oracle数据库的DB_DOMAIN参数。请参阅POWER Systems（64位）上的IBM AIX上的Oracle Database Gateway安装和配置指南，Linux x86-64，SPARC上的Oracle Solaris（64位），x86-64（64位）上的Oracle Solaris和HP- 适用于Microsoft Windows的 UX Itanium或Oracle Database Gateway安装和配置指南 ，具体取决于您的平台，以获取更多信息。

DRDA包和DB2注意事项

网关利用包来执行语句。第一次网关连接到目标DB2系统时，将隐式绑定此程序包。确保连接到DB2系统的用户标识具有绑定程序包所需的权限。请参阅POWER Systems（64位）上的IBM AIX上的Oracle Database Gateway安装和配置指南，Linux x86-64，SPARC上的Oracle Solaris（64位），x86-64（64位）上的Oracle Solaris和HP- 适用于Microsoft Windows的 UX Itanium或Oracle Database Gateway安装和配置指南 ，具体取决于您的平台，以获取更多信息。

日期算术

通常，由于DB2，以下类型的SQL表达式表单无法与网关一起正常工作局限性：

日期+号码 +日期 日期 - 号码 date1  -  date2

DB2不允许使用日期数据类型添加或减少数字。日期和数字的加法和减法(date + number, number + date, date - number)表格将被发送到拒绝它们的DB2。

此外，DB2不会一致地执行日期减法。当您减去两个日期(date1 - date2) ，DB2中日期减法的不同解释会导致结果因服务器而异。

注意：

避免所有网关SQL表达式中的日期算术表达式，直到解决日期算术问题。

行长度限制

由于DRDA体系结构的限制，DRDA表示中聚合长度超过32 KB的行无法存储或检索。

LONG SQL * Plus中的数据类型

SQL * Plus无法从Oracle数据库网关获取DRDA的LONG列。

存储过程和事务完整性

IBM DB2为存储过程引入了一个名为Commit on Return的功能。此功能允许DB2在存储过程成功运行后执行自动提交。创建过程时启用此功能。为确保数据完整性，适用于DRDA的Oracle数据库网关在异构环境中不支持此功能。尝试调用启用了此功能的存储过程时，网关将通过网关返回错误ORA-28526或PLS-00201 （必须声明标识符）。

SQL限制

以下各节介绍了Oracle Database Gateway for DRDA的SQL限制：

Oracle ROWID专栏

DB2 ROWID列与Oracle ROWID列不兼容。由于不支持ROWID列，因此以下限制适用：

  • UPDATE和不支持DELETE WHERE CURRENT OF CURSOR条款。通过网关更新或删除特定行的条件样式必须使用WHERE子句。（ 错误号205538）

    在预编译器和PL / SQL程序中使用UPDATE和DELETE语句时，它们内部依赖于Oracle ROWID函数。

  • 不支持Oracle数据库和DB2之间的快照。

    快照内部依赖于Oracle ROWID列。

Oracle绑定变量

与网关一起使用时，Oracle绑定变量将成为SQL参数标记。因此，绑定变量受到与SQL参数标记相同的限制。

例如，不允许以下语句：

WHERE：x IS NULL WHERE：x =：y
不支持CONNECT BY

用于DRDA的Oracle数据库网关在SELECT语句中不支持CONNECT BY 。
