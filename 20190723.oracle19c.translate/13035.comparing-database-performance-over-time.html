8比较数据库性能随时间的变化

本章介绍如何使用自动工作负载存储库（AWR）比较期间报告比较数据库性能，并包含以下主题：

  • 关于自动工作负载存储库比较期间报告

  • 生成自动工作负载存储库比较期间报告

  • 解释自动工作负载存储库比较期间报告

8.1关于自动工作负载存储库比较期间报告

当您的数据库在过去以最佳状态执行时会发生数据库性能下降，但随着时间的推移逐渐降级到用户可能会注意到的程度。 AWR比较期间报告使您可以比较数据库性能。

AWR报告显示两个快照（或两个时间点）之间的一段时间内的AWR数据。另一方面，AWR比较期间报告显示两个时间段之间的差异（或两个AWR报告，相当于四个快照）。使用AWR比较期间报告可帮助您识别两个时间段之间不同的详细性能属性和配置设置。

例如，假设在维护窗口期间每天运行的批处理工作负载在晚上10点到午夜之间显示性能不佳，现在在凌晨2点完成。您可以生成AWR比较期间报告，该报告的时间段为性能良好的当天晚上10:00至午夜，以及性能差的当天上午10:00至凌晨2:00的另一个报告。然后，您可以比较这些报告，以确定这两个时间段之间不同的配置设置，工作负载配置文件和统计信息。基于这些差异，您可以更轻松地诊断性能下降的原因。

在AWR比较期间报告中选择的两个时间段可以具有不同的持续时间，因为报告按照每个时间段在数据库上花费的时间量来统计统计数据，并且呈现按时间段之间的最大差异排序的统计数据。

注意：

将AWR功能与可插拔数据库（PDB）一起使用时，数据可见性和权限要求可能会有所不同。有关可管理性功能（包括AWR功能）如何在多租户容器数据库（CDB）中工作的信息，请参阅Oracle Multitenant管理员指南 。

8.2生成自动工作负载存储库比较期间报告

如果数据库的性能随着时间的推移而降低，则AWR比较周期报告使您能够比较两个时间段，以确定可帮助您诊断性能下降原因的关键差异。

AWR比较期间报告分为多个部分。HTML报告包含可用于在各部分之间快速导航的链接。报告的内容包含所选快照范围的系统工作负载配置文件。

  • 用于生成AWR比较周期报告的用户界面

  • 使用命令行界面生成AWR比较周期报告

8.2.1用于生成AWR比较周期报告的用户界面

生成AWR比较周期报告的主要界面是Oracle企业管理器云控制（云控制）。尽可能使用Cloud Control生成AWR比较期间报告。

如果Cloud Control不可用，则通过运行SQL脚本生成AWR比较期间报告。运行这些脚本需要DBA角色。

也可以看看：

Oracle Database 2 Day +性能调整指南 ，了解有关使用Cloud Control生成AWR比较期间报告的信息

8.2.2使用命令行界面生成AWR比较周期报告

本主题介绍如何通过在命令行界面中运行SQL脚本来生成AWR比较期间报告。

  • 生成本地数据库的AWR比较周期报告

  • 生成特定数据库的AWR比较周期报告

  • 生成Oracle RAC AWR比较本地数据库的周期报告

  • 生成特定数据库的Oracle RAC AWR比较周期报告

8.2.2.1为本地数据库生成AWR比较周期报告

awrddrpt.sql SQL脚本生成HTML或文本报告，该报告比较本地数据库实例上两个选定时间段之间的详细性能属性和配置设置。

要使用命令行界面在本地数据库实例上生成AWR比较周期报告，请执行以下操作：

  1. 在SQL提示符下，输入：

    @ $ ORACLE_HOME / RDBMS /管理/ awrddrpt.sql

  2. 指定您是要HTML还是文本报告：

    输入report_type的值：html

    在此示例中，选择了HTML报告。

  3. 指定要在第一个时间段内列出快照ID的天数。

    输入num_days的值：2

    将显示指定时间范围的现有快照列表。在此示例中，将显示最近2天捕获的快照。

  4. 指定第一个时间段的开始和结束快照ID：

    输入begin_snap的值：102输入end_snap的值：103

    在此示例中，选择快照ID为102的快照作为开始快照，并选择快照ID为103的快照作为第一个时间段的结束快照。

  5. 指定要在第二个时间段内列出快照ID的天数。

    输入num_days2：1的值

    将显示指定时间范围的现有快照列表。在此示例中，将显示前一天捕获的快照。

  6. 指定第二个时间段的开始和结束快照ID：

    输入begin_snap2的值：126输入end_snap2的值：127

    在此示例中，选择快照ID为126的快照作为开始快照，选择快照ID为127的快照作为第二个时间段的结束快照。

  7. 输入报告名称，或接受默认报告名称：

    输入report_name的值：使用报告名称awrdiff_1_102_1_126.txt

    在此示例中，接受默认名称，并生成名为awrdiff_1_102_126的AWR报告。

8.3解释自动工作负载存储库比较期间报告

生成AWR比较期间报告后对于要比较的时间段，请检查其内容以确定性能随时间降低的可能原因。

AWR比较期间报告的内容分为以下几个部分：

  • AWR比较期间报告摘要

  • AWR比较期间报告的详细信息

  • AWR比较期间报告中的补充信息

8.3.1 AWR比较期报告摘要

报告摘要位于AWR比较期间报告的开头，并汇总了有关报告中使用的快照集和工作负荷的信息。

报告摘要包含以下部分：

  • 快照集

  • 主机配置比较

  • 系统配置比较

  • 加载配置文件

  • 前5个定时活动

8.3.1.1快照集

“快照集”部分显示有关用于此报告的快照集的信息，例如实例，主机和快照信息。

8.3.1.2主机配置比较

“主机配置比较”部分比较两个快照集中使用的主机配置。例如，该报告会比较物理内存和CPU数量。配置中的任何差异都被量化为％Diff列中的百分比不同。

8.3.1.3系统配置比较

“系统配置比较”部分将比较两个快照集中使用的数据库配置。例如，该报告比较系统全局区域（SGA）和日志缓冲区大小。配置中的任何差异都被量化为％Diff列中的百分比不同。

8.3.1.4负载配置文件

“加载配置文件”部分比较两个快照集中使用的工作负载。工作负载的任何差异都会量化为％Diff列中的百分比不同。

8.3.1.5前5个定时事件

“前5个定时事件”部分显示五个定时事件或操作，这些事件或操作占每个快照集中总数据库时间（数据库时间）的最高百分比。

8.3.2 AWR比较期报告的详细信息

细节部分遵循AWR比较期间报告的报告摘要，并提供有关报告中使用的快照集和工作负荷的详细信息。

报告详细信息包含以下部分：

  • 时间模型统计

  • 操作系统统计

  • 等待事件

  • 服务统计

  • SQL统计

  • 实例活动统计

  • I / O统计

  • 咨询统计

  • 等待统计

  • 撤消细分摘要

  • 锁存统计

  • 细分统计

  • 内存段细分

  • 字典缓存统计

  • 库缓存统计

  • 记忆统计

  • 流量统计

8.3.2.1时间模型统计

该 “时间模型统计”部分比较两个快照集中的时间模型统计信息。时间模型统计信息是根据两个快照集之间特定类型操作所花费的总DB时间的差异排序的，并按降序列出。本节顶部的时间模型统计信息具有两个快照集之间的最大差异，并且相关操作可能是导致性能随时间降低的原因。

也可以看看：

有关时间模型统计信息的“ 时间模型统计 ”

8.3.2.2操作系统统计

操作系统统计部分比较两个快照集中的操作系统统计信息。本节提供了比较两个时段中每个时段的操作系统的总体状态。

8.3.2.3等待事件

该 “等待事件”部分比较两个快照集中的等待事件。

第一部分列出了等待事件的类，包括用户I / O和系统I / O.这些类按DB时间列％的绝对值以降序列出。

第二部分列出了等待事件。等待事件根据两个快照集之间等待事件所花费的总DB时间的差异进行排序，并按降序列出。本节顶部的等待事件具有两个快照集之间的最大差异，并且可能导致性能随时间降低。

8.3.2.4服务统计

服务统计部分比较两个快照集中的服务。服务根据两个快照集之间在特定服务上花费的总DB时间的差异进行排序，并按降序列出。

8.3.2.5 SQL统计

SQL Statistics部分比较两个快照集中的顶级SQL语句。SQL语句是根据不同的比较方法排序的，但在所有情况下，都显示了两个快照集之间差异最大的前十个SQL语句。

本节中显示的SQL语句可能会导致性能随时间降低，并根据以下类别进行排序：

  • 按执行时间排名前十的SQL比较

  • CPU时间排名前十的SQL比较

  • Buffer获取的十大SQL比较

  • 物理读取的十大SQL比较

  • 执行的十大SQL比较

  • Parse Calls排名前十的SQL比较

  • SQL文本的完整列表

8.3.2.7 I / O统计

I / O统计部分比较在两个快照集之间对表空间和数据库文件执行的I / O操作。两个快照之间的I / O操作的急剧增加可能是性能随时间降低的原因。

对于每个表空间或数据库文件，读取，写入和缓冲区高速缓存等待（或缓冲区获取）的数量差异被量化为百分比。数据库文件是根据不同的比较方法排序的，但在所有情况下，都会显示两个快照集之间差异最大的前10个数据库文件。

I / O统计信息分为以下几类：

  • 表空间I / O统计信息

  • I / O的十大文件比较

  • 按读取时间排名前10位的文件

  • 缓冲等待的前10个文件比较

8.3.2.7.1表空间I / O统计

本小节中显示的表空间按两个快照集之间在表空间上执行的规范化I / O数量的差异排序，并按降序列出。标准化I / O是每秒平均读取和写入的总和。

8.3.2.7.2通过I / O进行的前10个文件比较

本小节中显示的数据库文件按两个快照集之间在数据库文件上执行的规范化I / O数量的差异排序，并按降序列出。标准化I / O是每秒平均读取和写入的总和。

8.3.2.7.3按读取时间排序的前10个文件

本小节中显示的数据库文件按两个快照集之间从数据库文件读取数据所花费的DB时间百分比的差异排序，并按降序列出。

8.3.2.7.4缓冲区等待的前10个文件比较

本小节中显示的数据库文件按两个快照集之间的数据库文件上执行的缓冲区等待数（缓冲区高速缓存中的空闲缓冲区查找期间导致的等待）的差异排序，并按降序列出。

8.3.2.11锁存统计

锁存统计部分比较两个快照集之间的锁存器的总睡眠数，按降序排列。

锁存器是简单的低级序列化机制，用于保护SGA中的共享数据结构。例如，锁存器保护当前访问数据库的用户列表以及描述缓冲区高速缓存中的块的数据结构。服务器或后台进程在操作或查找这些结构之一的同时获取锁存器非常短的时间。锁存器的实现取决于操作系统，特别是关于进程是否以及等待锁存器的时间长短。

  • ITL Waits排名前五的细分市场比较

  • 缓冲区忙等待的前5个段比较
