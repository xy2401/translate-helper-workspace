11使用JSON Merge修补程序更新JSON文档

您可以使用Oracle SQL函数json_mergepatch更新JSON文档的特定部分。您传递一个JSON Merge Patch文档，该文档指定对指定的JSON文档所做的更改。JSON Merge Patch是IETF标准。

您可以在UPDATE语句中使用json_mergepatch来更新JSON列中的文档。例11-3说明了这一点。

您可以在SELECT列表中使用json_mergepatch来修改所选文档。修改后的文档可以进一步返回或处理。例11-4说明了这一点。

JSON Merge Patch适用于更新主要使用对象作为结构的JSON文档，而不使用显式null值。您不能使用它来添加，删除或更改数组元素（除非显式替换整个数组）。并且您不能使用它将字段的值设置为null 。

JSON Merge Patch有点像UNIX patch实用程序：你给它（1）补丁的源文档和（2）指定要进行的更改的补丁文档，它返回更新的源文档的副本（修补） 。补丁文档指定源文档和结果文档之间的差异。对于UNIX patch ，差异是UNIX diff实用程序输出的形式。对于JSON Merge Patch，源和补丁都是JSON文档。

您可以将JSON Merge Patch视为合并源和补丁的内容。合并两个对象时，一个来自源，一个来自补丁，一个成员在一个对象中而不在另一个对象中的成员保留在结果中。一个例外是当源对象没有这样的字段时，将忽略字段值为null的补丁成员。

合并具有相同字段的对象成员时：

  • 如果补丁字段值为null ，则从源中删除该字段 - 它不包含在结果中。

  • 否则，该字段保留在结果中，但其值是将源字段值与补丁字段值合并的结果 。也就是说，在这种情况下，合并操作是递归的 - 它向下潜入其值本身就是对象的字段。

更确切地说，JSON Merge Patch的行为如下：

  • 如果补丁 不是 JSON对象，则通过补丁替换源。

  • 否则（补丁是对象），请执行以下操作：

    1. 如果源 不是对象，则表现为它是空对象（ {} ）。

    2. 迭代补丁对象的（ p-field : p–value ）成员。

      • 如果修补程序成员的p-value null则从源中删除相应的成员。

      • 否则， 递归 ： 将相应源字段的值替换为使用p-value （作为下一个补丁） 合并修补该值（作为下一个源）的结果。

如果补丁字段值null没有特殊含义（使用该字段删除相应的源成员），则可以将其用作字段值，以将相应的源字段值设置为null 。特殊删除行为意味着您无法将源字段值设置为null 。

例子：

  • 修补程序成员"PONumber":99999使用字段PONumber覆盖源成员， 将其值替换为修补程序指定的值99999 。

    json_mergepatch('{"User":"ABULL", "PONumber":1600}', '{"PONumber":99999}')导致{"User":"ABULL", "PONumber":99999} 。

  • 修补程序成员"tracking":123456使用字段tracking覆盖缺少的源成员，将该修补程序成员添加到结果中。并且源成员"PONumber":1600使用字段PONumber覆盖缺少的补丁成员 - 它保留在结果中。

    json_mergepatch('{"PONumber":1600}', '{"tracking":123456}')导致{"PONumber":1600, "tracking":123456}" 。

  • 补片部件"Reference":null覆盖与场源构件Reference ，从结果中移除 。

    json_mergepatch('{"PONumber":1600, "Reference":"ABULL-20140421"}', '{"Reference":null}')导致{"PONumber":1600} 。

  • 补丁值[1,2,3]会覆盖相应的源值[4,5,6] ， 替换它。

    json_mergepatch('{"PONumber":1600, "LineItems":[1, 2, 3]}', '{"LineItems":[4,5,6]}')导致{"PONumber":1600, "LineItems":[4, 5, 6]} 。

注意：

合并补丁程序 - 特别是对于非对象补丁没有递归行为这一事实 - 意味着您无法单独添加，删除或替换数组的值。要进行此类更改，您必须替换整个数组 。例如，如果源文档具有成员Phone:["999-555-1212", "415-555-1234"]然后要删除第二个电话号码，您可以使用其内容具有"Phone":["999-555-1212"]成员的补丁"Phone":["999-555-1212"] 。

例11-1 JSON合并补丁文档

如果应用于示例2-1中显示的文档，则此JSON Merge Patch文档将执行以下操作：

  • 添加成员"Category" : "Platinum" 。

  • 使用字段ShippingInstructions删除成员。

  • 用"Contact User SBELL"字符串替换字段Special Instructions的值。

  • 用空数组[]替换字段LineItems的值

  • 使用成员"Allow Partial Shipment" : false替换成员"AllowPartialShipment" : null "Allow Partial Shipment" : false （实际上重命名该字段，因为字段值已经为false ）。

示例11-3使用JSON Merge修补程序更新JSON列

此示例更新j_purchaseorder.po_document所有文档，删除字段"Special Instructions" 。 

更新 j_purchaseorder SET po_document = json_mergepatch（po_document，'{“Special Instructions”： null }'）;

示例11-4即时更新选定的JSON文档

此示例选择j_purchaseorder.po_document所有文档，返回漂亮打印的更新副本，其中"Special Instructions"字段已被删除。 

SELECT json_mergepatch（po_document，'{“Special Instructions”： null }'RETURNING CLOB PRETTY）from j_purchaseorder;
