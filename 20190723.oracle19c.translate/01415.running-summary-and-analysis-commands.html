E.3运行摘要和分析命令

使用这些命令可以查看Oracle Trace File Analyzer的部署和状态摘要，以及Oracle Trace File Analyzer检测到的更改和事件。

  • 进行分析
    使用tfactl analyze命令通过解析数据库，Oracle ASM和Oracle Grid Infrastructure警报日志，系统消息日志，OSWatcher Top和OSWatcher Slabinfo文件来获取系统分析。
  • tfactl改变
    使用tfactl changes命令查看Oracle Trace File Analyzer检测到的更改。
  • tfactl事件
    使用tfactl events命令查看Oracle Trace File Analyzer检测到的事件。
  • tfactl isa
    使用tfactl isa命令查看基础结构服务自动化（ISA）分数。
  • tfactl运行
    使用tfactl run命令运行请求的操作（可以是库存或扫描或任何支持工具）。
  • tfactl搜索
    使用tfactl search命令搜索存储在Oracle Trace File Analyzer索引中的所有元数据。
  • tfactl摘要
    使用tfactl summary命令查看Oracle Trace File Analyzer部署的摘要。
  • tfactl toolstatus
    使用tfactl toolstatus命令查看所有节点上的Oracle跟踪文件分析器支持工具的状态。

E.3.1 tfactl分析

使用tfactl analyze命令通过解析数据库，Oracle ASM和Oracle Grid Infrastructure警报日志，系统消息日志，OSWatcher Top和OSWatcher Slabinfo文件来获取系统分析。

按组件，错误类型和时间过滤命令的输出。

使用tfactl analyze命令，您可以从以下类型的日志文件分析中进行选择：

  • 显示日志中最常见的消息 ：此分析可快速指示发生较大问题的位置。Oracle跟踪文件分析器从警报日志中取出重要消息，从日志消息中删除无关信息，组织最常出现的消息，并按照从最常见到最不常见的顺序显示它们。默认情况下，Oracle跟踪文件分析器会分析错误消息，但您可以指定特定类型的消息进行分析。

  • 在日志消息中搜索文本 ：这类似于使用grep实用程序进行搜索，但速度更快，因为Oracle跟踪文件分析器会检查每条消息的时间，并仅显示与最后x分钟或任何时间间隔匹配的消息。

  • 分析Oracle OSWatcher日志统计信息 ：Oracle跟踪文件分析器读取OSWatcher日志文件中可用的各种统计信息，并提供详细分析，显示每个统计信息的第一，最高，最低，平均和最后三个读数。选择任何特定分钟或秒的间隔。Oracle跟踪文件分析器可选择为OSWatcher日志提供报告的每个值（数据点）的原始数据。

句法

tfactl analyze [-search“pattern”] [-comp db | asm | crs | acfs | os | osw | oswslabinfo | oratop | all] [-type error | warning | generic] [-last n [ h | d ]] [ - 来自“MMM / DD / YYYY HH24：MI：SS”] [ - “MMM / DD / YYYY HH24：MI：SS”] [ - “MMM / DD / YYYY HH24：MI：SS” ] [-node all | local | n1 ， n2 ，...][-verbose] [-o file ] [-examples]

例子

以下命令示例演示了如何使用Oracle Trace File Analyzer搜索收集的数据：

  • $ tfactl analyze -search“error”-last 2d

    Oracle跟踪文件分析器在过去两天内搜索包含不区分大小写的字符串“error”的消息的警报和系统日志文件。

  • $ tfactl analyze -comp os -for“Jul / 01/2016 11”-search“。”

    Oracle Trace File Analyzer显示2016年7月1日上午11点的所有系统日志消息。

  • $ tfactl analyze -search“/ ORA- / c”-comp db -last 2d

    Oracle跟踪文件分析器在数据库警报日志中搜索过去两天的区分大小写的字符串“ORA-”。

以下命令示例演示了如何使用Oracle Trace File Analyzer分析收集的数据：

  • $ tfactl analyze -last 5h

    Oracle跟踪文件分析器显示从过去五小时内的所有警报日志和系统消息中收集的事件的摘要。

  • $ tfactl analyze -comp os -last 1d

    Oracle跟踪文件分析器显示过去一天系统消息中的事件摘要。

  • $ tfactl analyze -last 1h -type generic

    Oracle Trace File Analyzer分析过去一小时内的所有通用消息。

以下命令示例演示了如何使用Oracle Trace File Analyzer分析OSWatcher Top和Slabinfo：

  • $ tfactl analyze -comp osw -last 6h

    Oracle跟踪文件分析器显示过去六小时的OSWatcher摘要。

  • $ tfactl analyze -comp oswslabinfo -from“2016-07-01”-to“2016-07-03”

    Oracle跟踪文件分析器显示指定时间段内的OSWatcher Slabinfo摘要。
