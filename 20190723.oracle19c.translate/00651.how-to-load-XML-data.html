35如何加载XML数据

将XML数据加载到Oracle XML DB的主要方法是使用SQL * Loader。

  • 将XMLType数据加载到Oracle数据库的概述
    您可以使用常规方法或直接路径方法使用SQL * Loader加载XMLType数据，而不管它是如何存储的（对象关系或二进制XML存储）。
  • 使用SQL * Loader加载XMLType数据
    SQL * Loader处理XMLType列和表，如对象关系列和表。当XMLType数据存储为LOB时，所有从主数据文件或LOBFILE值加载LOB数据的方法也适用于加载XMLType列和表。

35.1将XMLType数据加载到Oracle数据库的概述

您可以使用常规方法或直接路径方法使用SQL * Loader加载XMLType数据，而不管它是如何存储的（对象关系或二进制XML存储）。

从Oracle9i第1版（9.0.1）开始，Export-Import实用程序和SQL * Loader支持XMLType作为列类型。从Oracle Database 10g开始，SQL * Loader还支持加载XMLType表。

注意：

对于XML数据的对象关系存储 ，如果数据包括XML Schema类型的继承 （扩展或限制），然后使用SQL * Loader 不支持直接路径加载。

也就是说，如果一个XML模式包含complexType延伸或限制另一元件complexType元件（基型），那么这导致在其他SQL类型的术语被定义一些SQL类型。在这种情况下，对象关系存储不支持直接路径加载。

导出用户数据时， 不会导出Oracle XML DB存储库信息。资源和任何信息都不会导出。

35.2.1.1使用预定大小字段加载LOB数据

预定义的大小字段构成了一个非常快速且概念上简单的SQL * Loader格式，用于加载LOB。

注意：

由于您加载的LOB大小可能不同，因此可以使用空格填充LOB数据，以使LOB在特定数据字段中的长度相等。

父主题： 使用SQL * Loader加载XMLType LOB数据

35.2.1.2使用分隔字段加载LOB数据

分隔字段格式处理同一列（数据文件字段）中不同大小的LOB。但是，这种增加的灵活性会影响性能，因为SQL * Loader必须扫描数据，查找分隔符字符串。

与单字符分隔符一样，当您指定字符串分隔符时，应考虑数据文件的字符集。当数据文件的字符集与控制文件的字符集不同时，可以用十六进制（即十六进制字符串）指定分隔符。如果以十六进制表示法指定分隔符，则规范必须包含在输入数据文件的字符集中有效的字符。相反，如果未使用十六进制规范，则分隔符规范被视为在客户端（即控制文件）字符集中。在这种情况下，在SQL * Loader搜索数据文件中的分隔符之前，分隔符将转换为数据文件字符集。

35.2.1.3从LOBFILE加载包含LOB数据的XML列

LOB数据可以足够长，从LOBFILE而不是从主数据文件加载它是有意义的。

在LOBFILE中，LOB数据实例仍被视为字段（预定大小，分隔符，长度值），但这些字段未组织成记录（LOBFILE中不存在记录的概念）。因此，避免了处理记录的处理开销。这种类型的数据组织是LOB加载的理想选择。

不要求LOBFILE中的LOB适合内存。SQL * Loader以64 KB块的形式读取LOBFILE。

在LOBFILE中，数据可以是以下任何类型的字段，其中任何一个都可以用于加载XML列：

  • 单个LOB字段，可以读取文件的全部内容

  • 预定大小字段（固定长度字段）

  • 分隔字段（即TERMINATED BY或ENCLOSED BY ）

    PRESERVE BLANKS子句不适用于从LOBFILE读取的字段。

  • 长度 - 值对字段（可变长度字段）。

    要从此类型的字段加载数据，请使用VARRAY ， VARCHAR或VARCHAR2 SQL * Loader数据类型。

35.2.1.4指定LOBFILE

您可以静态指定LOBFILE（指定文件的名称）或动态指定LOBFILE（使用FILLER字段作为文件名的来源）。

在任何一种情况下，当达到LOBFILE的EOF时，文件被关闭，并且从该文件读取数据的其他尝试产生的结果相当于从空字段读取数据。

您不应该指定与两个不同字段的源相同的LOBFILE。如果这样做，那么通常，这两个字段独立地读取数据。

35.2.2使用SQL * Loader直接从控制文件加载XMLType数据

您可以直接从控制文件加载XMLType数据。SQL * Loader像处理任何标量类型一样处理XMLType数据。

例如，考虑一个包含NUMBER列的表，后跟一个以对象关系存储的XMLType列。用于此表的控制文件可以包含NUMBER列的值，后跟XMLType实例的值。

SQL * Loader适用于非常大的XMLType实例。您还可以选择从LOB文件加载此类数据。

35.2.3使用SQL * Loader加载大型XML文档

您可以使用SQL * Loader将大量XML数据加载到Oracle数据库中。

  1. 列在数据文件中，比如filelist.dat ，列出要加载的XML文档的位置。

  2. 使用处理数据文件中列出的文件的命令创建一个控制文件，例如load_data.ctl 。

  3. 调用SQL * Loader shell命令sqlldr ，向其传递控制文件的名称。

这在例35-1 ， 例35-2和例35-3中说明 。文件filelist.dat列出包含2002年采购订单的XML文件。

如果您的应用程序使用索引或约束，那么处理这些可能会影响加载性能。您可以使用程序包DBMS_XMLSTORAGE_MANAGE PL / SQL子程序disableIndexesAndConstraints和enableIndexesAndConstraints暂时禁用此处理。
