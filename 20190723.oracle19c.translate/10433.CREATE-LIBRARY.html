创建图书馆

目的

使用CREATE LIBRARY语句创建与操作系统共享库关联的模式对象。然后可以在CREATE FUNCTION或CREATE PROCEDURE语句的call_spec中使用此模式对象的名称，或者在声明包或类型中的函数或过程时使用此模式对象的名称，以便SQL和PL / SQL可以调用第三代语言（ 3GL）功能和程序。

先决条件

CREATE LIBRARY语句仅在支持共享库和动态链接的平台上有效。

要在自己的模式中创建库，您必须具有CREATE LIBRARY系统特权。要在另一个用户的模式中创建库，您必须具有CREATE ANY LIBRARY系统特权。

要在CREATE FUNCTION语句的call_spec中使用库，或者在声明包或类型中的函数时，必须具有库的EXECUTE对象特权和CREATE FUNCTION系统特权。有关CREATE FUNCTION语句的call_spec的信息，请参阅Oracle Database PL / SQL语言参考 。

要在CREATE PROCEDURE语句的call_spec中使用库，或者在声明包或类型中的过程时，必须具有库的EXECUTE对象特权和CREATE PROCEDURE系统特权。有关CREATE PROCEDURE语句的call_spec的信息，请参阅Oracle Database PL / SQL语言参考 。

要执行使用call_spec定义的过程或函数（包括在包或类型中定义的过程或函数），您必须对过程或函数具有EXECUTE对象特权（但您不需要对库具有EXECUTE对象特权） 。

语义

或替换

指定OR REPLACE以重新创建库（如果它已存在）。使用此子句可以更改现有库的定义，而无需删除，重新创建和重新授予对其授予的对象权限。

以前被授予重新定义的库权限的用户仍然可以访问该库而无需重新授予权限。

[版权所有|不可挽回的]

使用这些子句指定库是否为editioned或noneditioned对象，如果editioning为架构对象类型启用LIBRARY的schema 。默认为EDITIONABLE 。有关已编辑和未编辑的对象的信息，请参见“ Oracle数据库开发指南” 。
