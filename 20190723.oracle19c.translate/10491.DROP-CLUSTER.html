DROP CLUSTER

目的

使用DROP CLUSTER子句从数据库中删除集群。

注意：

删除群集时，回收站中曾经属于该群集的所有表都将从回收站中清除，并且无法再使用FLASHBACK TABLE操作进行恢复。

您无法取消群集单个表。相反，您必须执行以下步骤：

  1. 创建一个具有与旧结构和内容相同的结构和内容的新表，但没有CLUSTER子句。

  2. 放下旧桌子。

  3. 使用RENAME语句为新表提供旧表的名称。

  4. 授予新非聚集表的权限。旧聚簇表上的授权不适用。

    也可以看看：

    有关这些步骤的信息，请参见 CREATE TABLE ， DROP TABLE ， RENAME ， GRANT

先决条件

群集必须位于您自己的架构中，或者您必须具有DROP ANY CLUSTER系统特权。

句法

drop_cluster :: =


说明drop_cluster.eps的说明

语义

模式

指定包含群集的架构。如果省略schema ，则数据库假定集群在您自己的模式中。

簇

指定要删除的集群的名称。删除集群还会删除集群索引，并将所有集群空间（包括索引的数据块）返回到相应的表空间。

包括表格

指定INCLUDING TABLES以删除属于该群集的所有表。

级联约束

指定CASCADE CONSTRAINTS以从集群外部的表中删除所有引用完整性约束，这些表引用集群表中的主键和唯一键。如果省略此子句并且存在此类参照完整性约束，则数据库将返回错误并且不会丢弃该集群。
