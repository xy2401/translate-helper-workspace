122 DBMS_PLSQL_CODE_COVERAGE

DBMS_PLSQL_CODE_COVERAGE包提供了一个接口，用于在基本块级别上收集PL / SQL应用程序的代码覆盖率数据。

122.1 DBMS_PLSQL_CODE_COVERAGE概述

DBMS_PLSQL_CODE_COVERAGE包提供了一个接口，用于在PL / SQL应用程序的基本块级收集代码覆盖率信息。基本块是指PL / SQL代码的单个条目单个退出块。PL / SQL开发人员想知道他们的测试基础设施如何运用他们的代码。使用CREATE_COVERAGE_TABLES过程创建coverage表。

在会话中运行的典型代码覆盖涉及对以下内容的调用：

  • START_COVERAGE

  • 运行PL / SQL代码

  • STOP_COVERAGE

122.2 DBMS_PLSQL_CODE_COVERAGE安全模型

用户必须具有DBMS_PLSQL_CODE_COVERAGE包的EXECUTE特权。

用户必须具有该设备的CREATE权限才能收集有关此设备的覆盖信息。

不为包装单元收集覆盖范围信息。

122.3.1 CREATE_COVERAGE_TABLES过程

此过程创建用于coverage数据收集的表。

句法

DBMS_PLSQL_CODE_COVERAGE.CREATE_COVERAGE_TABLES（FORCE_IT in BOOLEAN DEFAULT FALSE）;

参数

参数 描述

FORCE_IT

如果coverage表已存在，则默认为引发错误。如果设置为TRUE，则表将以静默方式删除，如果表已存在，则会创建新表。

例外

表122-2 CREATE_COVERAGE_TABLES例外

例外 描述
COVERAGE_ERROR

FORCE_IT参数为FALSE且表已存在。

122.3.2 START_COVERAGE功能

此函数在用户会话中启动coverage数据集合，并返回运行的唯一标识符RUN_ID 。

句法

DBMS_PLSQL_CODE_COVERAGE.START_COVERAGE（run_comment IN VARCHAR2）返回号码;

参数

参数 描述

run_comment

允许用户命名运行并识别测试。

122.3.3 STOP_COVERAGE程序

此过程结束当前的覆盖范围运行。

句法

DBMS_PLSQL_CODE_COVERAGE.STOP_COVERAGE;

例外

表122-3 STOP_COVERAGE异常

例外 描述
COVERAGE_ERROR

如果coverage表不存在，则会引发错误。
