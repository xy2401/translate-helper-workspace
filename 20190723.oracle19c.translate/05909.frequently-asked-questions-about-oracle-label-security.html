Oracle Label Security如何解决我的安全需求？

Oracle Label Security可以标记数据并以高度粒度限制访问。

当多个组织或公司共享单个应用程序时，这尤其有用。敏感性标签可用于将应用程序用户限制为组织或组织内的数据子集。

数据隐私对消费者很重要，监管措施将继续公布。Oracle Label Security可用于实施数据隐私策略，限制只有那些需要知道的人才能访问。

我应该使用Oracle Label Security来保护我的所有表吗？

不，您不应该使用Oracle Label Security来保护所有表。

在大多数情况下，传统的Oracle自主访问控制（DAC）对象权限（如SELECT ， INSERT ， UPDATE和DELETE与数据库角色和存储过程相结合就足够了。您可以通过查询DBA_SYS_PRIVS数据字典视图来查找用户的权限。

此外，还有许多其他方法可以保护对数据库表的访问，例如使用Oracle虚拟专用数据库（VPD），Oracle Database Vault，Oracle数据编辑，透明数据加密（TDE）或透明敏感数据保护（TSDP） ）。

Oracle虚拟专用数据库和Oracle Label Security之间有什么区别？

Oracle数据库企业版免费提供Oracle虚拟专用数据库（VPD）。

Oracle Label Security是Oracle数据库企业版的附加安全选项。

Oracle VPD是一个术语，用于几个强大的安全功能，如细粒度访问控制（FGAC），应用程序上下文和全局应用程序上下文。VPD策略使用PL/SQL编写，可以分配给单个表或视图。访问受VPD保护的表或视图的信息请求将根据分配给表或视图的策略进行修改。

VPD策略可以像在工作时间强制访问一样简单。VPD策略可以通过将单个行中的属性值与应用程序上下文值进行比较来限制访问。全局应用程序上下文允许跨多个数据库会话访问应用程序上下文，从而减少或消除为每个用户会话创建单独的应用程序上下文的需要。

Oracle Label Security是行级安全性的开箱即用解决方案。无需编码或软件开发，允许管理员完全专注于策略。Oracle Label Security提供了一个界面，用于创建策略，指定实施选项，定义数据敏感性标签，建立用户标签授权以及保护单个表或方案。

数据敏感性标签提供了一种强大而灵活的限制数据访问的方法。例如，属于不同组织或公司的数据可以使用数据敏感性标签分开，并通过更改数据敏感性标签在公司之间选择性地共享。

根据安全策略的复杂性，Oracle虚拟专用数据库可能是实施安全策略的首选方法。Oracle Label Security最适合需要基于信息敏感性的访问控制决策的情况。

我可以将Oracle虚拟专用数据库和Oracle Label Security结合起来吗？

是。您可以使用WHERE子句或VPD策略。

  • WHERE子句可以附加到OLS策略，该策略提供更多级别的粒度。例如，用户（无论其标签授权如何）仅允许从特定IP地址或子网进行连接，并且仅在工作时间内进行连接。

  • VPD策略（无论是否为列敏感）可以评估用户标签并确定对列和行的访问，而无需应用数据标签。

Oracle Label Security是否提供列级访问控制？

不，Oracle Label Security不支持列。

虚拟专用数据库（VPD）可以使用此行为。可以编写VPD策略，使其仅在某个列是针对受保护表的SQL语句的一部分时才变为活动状态。如果列灵敏度开关打开，则VPD仅返回用户可访问敏感列值的行，或者返回敏感列中所有单元格为空的所有行，但允许用户使用的值除外查看。

什么是受信任的存储程序单元？

受信任的存储程序单元是使用定义器的系统和对象权限（DAC）执行的存储过程，函数和包。

如果调用者是具有Oracle Label Security用户许可（标签）的用户，则该过程将使用定义者的DAC权限和调用者的安全许可的组合来执行。

受信任的存储过程是被授予Oracle Label Security特权FULL或READ 。运行受信任的存储程序单元时，有效的策略权限是调用用户权限和程序单元权限的组合。

VPD或OLS是否向受保护的表添加其他列？

将Oracle Label Security（OLS）策略应用于表时，该策略会向表中添加一个附加列。

最初创建策略时，需要指定此列的名称。

现有列可用于存储OLS行标签。此列必须具有NUMBER(10)数据类型。

Oracle虚拟专用数据库（VPD）不会向受保护的表添加其他列。

为什么要隐藏其他OLS行标签列？

大多数应用程序都考虑了访问控制机制，因此Oracle Label Security必须透明地执行此操作。

当应用程序使用SELECT FROM tablename语句查询表时，它将返回所有列，包括未隐藏的标签列。现有应用程序可能无法显示附加列和故障。但是，如果标签列被隐藏，则仅当其名称包含在SQL语句中时才会显示。SELECT FROM tablename将返回应用程序所期望的所有列，不包括隐藏的OLS列。
