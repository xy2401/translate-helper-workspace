创建最小组，用户和路径的示例

您可以创建最小操作系统身份验证配置，如此示例中所述。

此配置示例显示以下内容：

  • 创建Oracle Inventory组（ oinstall ）

  • 创建单个组（ dba ）作为为所有Oracle Grid Infrastructure，Oracle ASM和Oracle数据库系统特权分配的唯一系统特权组

  • 创建Oracle Grid Infrastructure软件所有者（ grid ）和一个具有正确组成员身份的Oracle数据库所有者（ oracle ）

  • 创建和配置符合具有正确权限的OFA结构的Oracle基本路径

输入以下命令以创建最小操作系统身份验证配置：

#groupadd -g 54421 oinstall #groupadd -g 54422 dba #useradd -u 54321 -g oinstall -G dba oracle＃useradd -u 54322 -g oinstall -G dba grid＃mkdir -p /u01/app/19.0.0/grid #mkdir -p / u01 / app / grid #mkdir -p / u01 / app / oracle #chown -R grid：oinstall / u01＃chown oracle：oinstall / u01 / app / oracle＃chmod -R 775 / u01 /

运行这些命令后，您将拥有以下组和用户：

  • Oracle中央库存组或oraInventory组（ oinstall ）。将中央库存组作为其主要组的成员将被授予OINSTALL权限以写入oraInventory目录。

  • 一个系统特权组dba ，用于Oracle Grid Infrastructure，Oracle ASM和Oracle数据库系统特权。将dba组作为主要或辅助组的成员将被授予OSASM / SYSASM，OSDBA / SYSDBA，OSOPER / SYSOPER，OSBACKUPDBA / SYSBACKUP，OSDGDBA / SYSDG，OSKMDBA / SYSKM，OSM ASM / SYSDBA for ASM的操作系统认证，和OSOPER for ASM / SYSOPER for Oracle ASM管理Oracle Clusterware，Oracle ASM和Oracle数据库，并被授予SYSASM和OSOPER，以便Oracle ASM访问Oracle ASM存储。

  • 适用于集群所有者或网格用户（ grid ）的Oracle Grid Infrastructure，其中oraInventory组（ oinstall ）作为其主要组，OSASM组（ dba ）作为辅助组，具有其Oracle基本目录/u01/app/grid 。

  • Oracle数据库所有者（ oracle ），其oraInventory组（ oinstall ）作为其主要组，OSDBA组（ dba ）作为其辅助组，具有其Oracle基本目录/u01/app/oracle 。

  • /u01/app由grid:oinstall拥有grid:oinstall在安装之前具有775权限，在安装期间运行root.sh脚本之后由root用户执行。此所有权和权限使OUI能够在路径/u01/app/oraInventory创建Oracle Inventory目录。

  • /u01由grid:oinstall拥有grid:oinstall安装前的grid:oinstall ，以及在安装期间运行root.sh脚本之后的root。

  • grid:oinstall所拥有的/u01/app/19.0.0/grid grid:oinstall具有775权限的grid:oinstall 。这些权限是安装所必需的，并且在安装过程中会更改。

  • /u01/app/grid由grid:oinstall拥有775权限。这些权限是安装所必需的，并且在安装过程中会更改。

  • /u01/app/oracle由/u01/app/oracle拥有oracle:oinstall具有775权限。

注意：

您可以将一个安装所有者用于Oracle Grid Infrastructure和任何其他Oracle安装。但是，Oracle建议您为每个Oracle软件安装使用单独的安装所有者帐户。
