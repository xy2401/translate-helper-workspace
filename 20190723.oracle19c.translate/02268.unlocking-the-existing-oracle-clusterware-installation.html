解锁和卸载以前的版本网格主页

从先前版本升级后，如果要卸载以前版本的Oracle Grid Infrastructure主目录，则必须先更改先前版本Grid主目录的权限和所有权。

使用以下过程解锁Oracle Grid Infrastructure安装：

  1. 以root身份登录，并使用以下命令语法更改先前版本Grid home的权限和所有权，其中oldGH是以前版本的Grid home， swowner是Oracle Grid Infrastructure安装所有者， oldGHParent是以前的父目录发布网格主页：

    #chmod -R 755 oldGH #chown -R swowner oldGH #chown swowner oldGHParent

    例如：

    #chmod -R 755 /u01/app/18.0.0/grid -R grid /u01/app/18.0.0/grid grid /u01/app/18.0.0

  2. 更改先前版本Grid home的权限和所有权后，以Oracle Grid Infrastructure安装所有者（ grid ，在前面的示例中）登录，并使用以前版本中的deinstall命令Grid home（ oldGH ） $ORACLE_HOME/deinstall目录。

警告：

您必须使用同一版本中的deinstall命令删除Oracle软件。请勿从更高版本运行deinstall命令以从早期版本中删除Oracle软件。例如，不要从19.0.0.0.0 Oracle主目录运行deinstall命令，以从现有的18.0.0.0.0 Oracle主目录中删除Oracle软件。
