D.5 chactl校准

使用chactl calibrate命令创建具有更高灵敏度和准确度的新模型。

用户生成的模型对您的操作环境中的Oracle Real Application Clusters（Oracle RAC）监控系统有效，因为用户生成的模型使用来自目标的校准数据。Oracle Cluster Health Advisor将用户生成的模型添加到可用模型列表中，并将新模型存储在Oracle Cluster Health Advisor存储库中。

如果存在具有相同名称的模型，则使用-force选项使用新模型覆盖旧模型。

关键绩效和工作量指标

一组度量或关键性能指标描述了为校准选择的训练数据的高级约束。此集包含用于描述性能目标和资源利用带宽的相关度量标准，例如响应时间或CPU利用率。

关键性能指标也是操作系统和数据库信号，它们被监视，估计并与故障检测逻辑相关联。这些关键绩效指标中的大多数也是预测因子，即它们的状态与其他信号的状态相关，或者由其他信号预测。关键性能指标与其他信号相关的事实使得它们可用作训练或校准数据的滤波器。

关键性能指标范围用于query calibrate和calibrate命令以过滤掉数据点。

数据库支持以下关键性能指标：

  • CPUPERCENT - CPU利用率 - 百分比

  • IOREAD - 磁盘读取 - Mbyte / sec

  • DBTIMEPERCALL - 每个用户调用的数据库时间 - usec / call

  • IOWRITE - 磁盘写入 - Mbyte / sec

  • IOTHROUGHPUT - 磁盘吞吐量 - IO /秒

群集支持以下关键性能指标：

  • CPUPERCENT - CPU利用率 - 百分比

  • IOREAD - 磁盘读取 - Mbyte / sec

  • IOWRITE - 磁盘写入 - Mbyte / sec

  • IOTHROUGHPUT - 磁盘吞吐量 - IO /秒

句法

chactl校准{簇|数据库-db db_unique_name } -model model_name [-force] [-timeranges“开始= time_stamp ，端= time_stamp ，...'] [-kpiset'name = kpi_name min = val max = val ，...'] [ -  help]

在YYYY-MM-DD HH24:MI:SS格式中指定时间戳。

例子

chactl calibrate database -db oracle -model weekday -timeranges'start = start = 2016-09-09 16：00：00，end = 2016-09-09 23:00:00'
chactl calibrate database -db oracle -model weekday -timeranges'start = start = 2016-09-09 16：00：00，end = 2016-09-09 23:00:00'-kpiset'name = CPUPERCENT min = 10 max = 60'

错误消息

错误： 输入字符串“xc＃？％“被误解了

描述：确认给定的型号名称是否存在且警告： model_name已存在，请使用[-force]消息。

错误： start_time和/或end_time被误解

描述：输入时间说明符构造错误。

错误： 指定时间段内没有足够的校准数据，请重新选择另一个时间段

说明： Evaluator无法找到足够的校准数据。
