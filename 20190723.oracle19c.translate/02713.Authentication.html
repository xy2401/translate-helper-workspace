认证

数据库REST API使用Verisign颁发的证书颁发机构（CA）证书，以使客户端能够安全地连接到服务器。

注意：

确保您具有用于创建，管理和删除Database REST API实例的相应登录凭据。

您可以通过HTTPS访问Database REST API资源，并且必须提供以下信息以进行身份验证：

  • 用于对Verisign CA证书进行身份验证的SSL证书颁发机构（CA）证书文件或捆绑包。

  • 您的Database REST API帐户的用户名和密码。

  • 自定义标头X-ID-TENANT-NAME，用于标识标识域ID。

例如，要使用cURL进行身份验证：

  • 使用-u cURL选项传递用户名和密码（例如，username和userPassword1！）。

  • 使用-H cURL选项传递X-ID-TENANT-NAME自定义标头。

  • 仅从Windows命令shell运行cURL时，将cURL环境变量CURL_CA_BUNDLE设置为SSL证书颁发机构（CA）证书文件或包的位置，以对Verisign CA证书进行身份验证。或者，您可以使用--cacert命令行选项在命令行上指定本地CA证书包的位置。

您可以从http://curl.haxx.se/docs/caextract.html下载SSL CA证书包，也可以自行提供。有关使用cURL验证CA证书的信息，请参阅： http ： //curl.haxx.se/docs/sslcerts.html 。

以下是cURL命令的示例语法：

curl -i -X GET -u <用户名>：<密码> -H <请求标头>：<值> https：// <子域>。<域> .com：<端口> / <资源路径>
