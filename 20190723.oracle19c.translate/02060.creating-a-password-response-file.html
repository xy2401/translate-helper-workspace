创建密码响应文件

您可以创建密码响应文件，并将其与配置助手一起使用以执行静默安装。

执行以下步骤以创建密码响应文件：

  1. 创建一个名称格式为filename的响应文件。 properties ，例如：
    $ touch pwdrsp.properties
  2. 使用文本编辑器打开文件，并剪切并粘贴示例密码文件内容，如示例所示，根据需要进行修改。
  3. 更改权限以保护密码响应文件。例如：
    $ ls -al pwdrsp.properties -rw ------- 1 oracle oinstall 0 Apr 30 17:30 pwdrsp.properties

示例A-4 Oracle Grid Infrastructure的密码响应文件（ grid用户）

grid.crs | S_ASMPASSWORD = 密码 grid.crs | S_OMSPASSWORD = 密码 grid.crs | S_BMCPASSWORD = 密码 grid.crs | S_ASMMONITORPASSWORD = 密码

如果您没有BMC卡，或者您不想启用IPMI，请将S_BMCPASSWORD输入字段留空。

示例A-5用于独立服务器的Oracle Grid Infrastructure的密码响应文件（ oracle用户）

oracle.crs | S_ASMPASSWORD = 密码 oracle.crs | S_OMSPASSWORD = 密码 oracle.crs | S_ASMMONITORPASSWORD = 密码

示例A-6 Oracle数据库的密码响应文件（ oracle用户）

此示例提供了与数据库配置助手一起使用的密码响应文件的模板。

oracle.server | S_SYSPASSWORD = 密码 oracle.server | S_SYSTEMPASSWORD = 密码 oracle.server | S_EMADMINPASSWORD = 密码 oracle.server | S_DBSNMPPASSWORD = 密码 oracle.server | S_ASMSNMPPASSWORD = 密码 oracle.server | S_PDBADMINPASSWORD = 密码

如果您不想启用Oracle Enterprise Manager进行管理，请将这些密码字段留空。
