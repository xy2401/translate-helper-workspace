更改审计政策（统一审计）

本节介绍统一 审计的ALTER AUDIT POLICY语句。这种类型的审计是Oracle Database 12 c的新开始，它提供了一整套增强的审计功能。有关统一审计的更多信息，请参阅“ Oracle数据库安全指南” 。

目的

使用ALTER AUDIT POLICY语句修改统一的审核策略。

也可以看看：

  • 创建审计政策（统一审计）

  • DROP审计政策（统一审计）

  • 审计（统一审计）

  • NOAUDIT（统一审计）

先决条件

您必须具有AUDIT SYSTEM系统特权或AUDIT_ADMIN角色。

如果您连接到多租户容器数据库（CDB），然后要修改通用的统一审计策略，则当前容器必须是根，并且您必须具有通常授予的AUDIT SYSTEM权限或AUDIT_ADMIN公共角色。要修改本地统一审核策略，当前容器必须是创建审核策略的容器，并且您必须具有通常授予的AUDIT SYSTEM权限或AUDIT_ADMIN公共角色，或者您必须具有本地授予的AUDIT SYSTEM权限或容器中的AUDIT_ADMIN本地角色。

语义

政策

指定要修改的统一审核策略的名称。必须使用CREATE AUDIT POLICY语句创建策略。您可以通过查询AUDIT_UNIFIED_POLICIES视图找到所有统一审核策略的描述。

也可以看看：

  • 创建审计政策（统一审计）

  • 有关AUDIT_UNIFIED_POLICIES视图的更多信息，请AUDIT_UNIFIED_POLICIES Oracle数据库参考

ADD |下降

使用ADD子句添加要审核到policy权限。

使用DROP子句删除要从policy审核的权限。

请参阅privilege_audit_clause ， action_audit_clause和role_audit_clause的CREATE AUDIT POLICY这些条款的完整语义。

条件

使用此子句删除，添加或替换policy的审核条件。

指定DROP以从policy删除审核条件。

指定' audit_condition ' ...以添加或替换policy的审核条件。

有关这些条款的完整语义，请参阅audit_condition ， EVALUATE PER STATEMENT ， EVALUATE PER SESSION ，以及评估 CREATE AUDIT POLICY的每个实例 。

只有TOPLEVEL

指定此子句可将现有的统一审核策略更改为仅审核用户发出的顶级SQL语句。

示例：添加顶级审核

该示例将HR审核策略hr_audit_policy更改为仅捕获顶级语句。 

ALTER AUDIT POLICY hr_audit_policy仅添加TOPLEVEL

您可以从审核顶级SQL语句的现有审核策略中删除顶级审核。

示例：删除顶级审核

ALTER AUDIT POLICY hr_audit_policy只有TOPLEVEL

例子

以下示例修改了在CREATE AUDIT POLICY “ Examples ”中创建的统一审计策略。

将权限，操作和角色添加到统一审核策略：示例

以下语句将系统特权CREATE ANY TABLE和DROP ANY TABLE到统一审计策略dml_pol ：

ALTER AUDIT POLICY dml_pol ADD PRIVILEGES CREATE ANY TABLE，DROP ANY TABLE;

以下语句将系统操作CREATE JAVA ， ALTER JAVA和DROP JAVA到统一审计策略java_pol ：

ALTER AUDIT POLICY java_pol ADD ACTIONS CREATE JAVA，ALTER JAVA，DROP JAVA;

以下语句将角色dba添加到统一审计策略table_pol ：

ALTER AUDIT POLICY table_pol ADD ROLES dba;

以下语句将多个系统特权，操作和角色添加到统一审计策略security_pol ：

ALTER AUDIT POLICY security_pol ADD PRIVILEGES创建任何库，删除hr.employees上的任何库操作，hr.employees上的INSERT，hr.employees上的UPDATE，hr.departments上的ALL ROLES dba，connect;

从统一审核策略中删除权限，操作和角色：示例

以下语句从统一审计策略table_pol删除系统特权CREATE ANY TABLE ：

ALTER AUDIT POLICY table_pol DROP PRIVILEGES CREATE ANY TABLE;

以下语句删除hr上的INSERT和UPDATE操作。来自统一审计策略的employees dml_pol ：

ALTER AUDIT POLICY dml_pol对hr.employees进行DROP ACTIONS INSERT，对hr.employees进行UPDATE;

下面的语句删除角色java_deploy从统一的审计策略java_pol ：

ALTER AUDIT POLICY java_pol DROP ROLES java_deploy;

以下语句从统一审计策略hr_admin_pol删除系统特权，操作和角色：

ALTER AUDIT POLICY hr_admin_pol DROP PRIVILEGES CREATE ANY TABLE ACTIONS LOCK TABLE ROLES audit_viewer;

添加和删除统一审核策略的操作：示例

以下语句将Oracle Data Pump的EXPORT操作添加到统一审计策略dp_actions_pol并删除Oracle Data Pump的IMPORT操作：

ALTER AUDIT POLICY dp_actions_pol ADD ACTIONS COMPONENT = datapump EXPORT DROP ACTIONS COMPONENT = datapump IMPORT;

从统一审核策略中删除审核条件：示例

以下语句从统一审计策略order_updates_pol删除审计条件：

ALTER AUDIT POLICY order_updates_pol CONDITION DROP;
