在全局区域中部署Oracle RAC

此部署方案由多个服务器组成，您可以在全局区域中安装Oracle Real Application Clusters（Oracle RAC）。

  1. 创建rac_framework资源以支持Oracle Grid Infrastructure的安装。请参阅注册和配置RAC框架资源组 。
  2. 必要时创建存储框架资源以及任何所需的存储资源。
    请参阅创建存储管理资源 。
    您还可以通过从全局群集节点运行clsetup实用程序来配置存储资源，然后按照向导中的步骤操作。请参阅注册和配置Oracle Files的存储资源
  3. 准备环境，然后按照以下主题安装和配置Oracle Grid Infrastructure：
    • 有关在安装Oracle Grid Infrastructure之前配置用户，组和环境的信息，请参阅为Oracle Grid Infrastructure和Oracle Database配置用户，组和环境。
    • 有关为Oracle RAC数据库配置存储的信息，请参阅为Oracle Grid Infrastructure配置存储。
    • 有关安装Oracle Grid Infrastructure的信息，请参阅安装Oracle Grid Infrastructure。
  4. 按照“ Oracle Real Application Clusters安装指南”中的说明安装和配置Oracle RAC数据库。
  5. 为Oracle RAC数据库创建crs_framework和rac_server_proxy资源，如配置Oracle RAC数据库实例的资源中所述。
    crs_framework资源类型使Oracle Solaris Cluster和Oracle Clusterware能够通过启用Oracle Solaris Cluster来停止Oracle Clusterware来实现互操作。rac_server_proxy资源是Oracle RAC数据库服务器的代理资源。
  6. 从全局群集节点，验证Oracle RAC框架资源：
    #clresource status
