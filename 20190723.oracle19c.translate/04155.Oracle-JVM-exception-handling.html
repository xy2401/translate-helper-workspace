8.6 Oracle JVM如何处理异常

Java异常是对象，具有命名和继承层次结构。因此，您可以将子异常（即异常类的子类）替换为超级异常，即异常类的超类。

所有Java异常对象都支持toString()方法，该方法返回连接到可选字符串的异常类的完全限定名称。通常，字符串包含有关异常情况的数据相关信息。通常，构造异常的代码会将字符串与其关联。

当Java存储过程运行SQL语句时，抛出的任何异常都将作为java.sql.的子类实现给过程java.sql.SQLException 。该类具有getErrorCode()和getMessage()方法，分别返回Oracle错误代码和消息。

如果从SQL或PL / SQL调用的存储过程抛出异常并且未被Java捕获，则会出现以下错误消息：

ORA-29532 Java调用因未捕获的Java异常而终止

这是报告所有未捕获的异常（包括非SQL异常）的方式。
