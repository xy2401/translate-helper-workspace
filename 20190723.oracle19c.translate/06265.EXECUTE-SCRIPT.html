2.21执行脚本

目的

使用EXECUTE SCRIPT命令运行存储在恢复目录中的本地或全局RMAN脚本。

也可以看看：

  • Oracle数据库备份和恢复用户指南 ，了解如何使用存储的脚本

  • CREATE SCRIPT并REPLACE SCRIPT

先决条件

仅在RUN命令的大括号内使用EXECUTE SCRIPT 。必须使用CATALOG命令行选项或CONNECT CATALOG命令将RMAN连接到恢复目录。必须打开恢复目录数据库。

使用说明

在RUN块中运行EXECUTE SCRIPT命令时，RMAN将脚本的内容放在该RUN块中。因此，如果您也在脚本中分配了一个通道，请不要在RUN块中分配通道。

如果指定了GLOBAL ，则恢复目录中必须存在具有此名称的全局脚本;否则，RMAN返回错误 RMAN-06004 。如果未指定GLOBAL ，则RMAN将搜索为当前目标数据库定义的本地存储脚本。如果找不到具有此名称的本地脚本，则RMAN将使用相同的名称搜索全局脚本，并在找到该脚本时执行该脚本。

句法

executeScript :: =


插图的描述executioncript.eps

语义

语法元素 描述

全球

指定全局存储脚本而不是本地脚本的执行。

另请参阅： “ 使用说明 ” ，以解释全局脚本和本地脚本之间的区别

SCRIPT script_name

指定要执行的存储脚本的名称。当名称包含空格或保留字时，必须在脚本名称周围使用引号。
USING [ string_or_identifier | integer ]

指定一个或多个值，用于存储脚本中的替换变量（请参阅示例2-97 ）。

另请参阅： CREATE SCRIPT以了解如何使用替换变量创建存储脚本，以及RMAN和@ (at sign)以了解如何使用RMAN替换变量

示例2-96执行存储的脚本

示例2-97创建和执行使用替换变量的存储脚本

启动RMAN并连接到目标数据库和恢复目录后，使用REPLACE SCRIPT创建包含三个替换变量的备份脚本。RMAN会提示您输入变量的初始值（用户输入以粗体显示）。 

RMAN> REPLACE SCRIPT 2> backup_df {BACKUP DATAFILE＆1 TAG＆2.1 FORMAT'/ disk1 /＆3_％U';输入1： 1的值输入2的值： df1_backup输入3的值： df1启动恢复目录的完全重新同步完全重新同步完成创建的脚本backup_df

稍后，您可以使用不同的值执行backup_df脚本。以下示例将值3 ， test_backup和test传递给存储脚本中的替换变量：

RMAN> RUN {EXECUTE SCRIPT backup_df使用3 test_backup df3; }
