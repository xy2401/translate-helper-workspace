chdg

目的

根据XML配置文件更改磁盘组（添加磁盘，删除磁盘，调整磁盘大小或重新平衡磁盘组）。

语法和说明

chdg { config_file .xml | ' contents_of_xml_file '}

选项 描述

CONFIG_FILE

包含磁盘组更改的XML文件的名称。除非指定了路径，否则chdg将在启动ASMCMD的目录中搜索XML文件。

contents_of_xml_file

用单引号括起来的XML脚本。

故障组是可选参数。默认值导致每个磁盘属于其自己的故障组。

可以通过此操作执行从磁盘组中删除磁盘。单个磁盘可以通过其Oracle ASM磁盘名称引用。可以通过故障组名称指定属于故障组的一组磁盘。

您可以使用chdg调整磁盘组内的磁盘chdg 。如果在调整大小后没有足够的空间存储数据，则调整大小操作将失败。

iostat的

目的

显示已装入磁盘组中Oracle ASM磁盘的I / O统计信息。

语法和说明

iostat [--suppressheader] [-et] [--io] [ -  region] [-G diskgroup ] [ interval ]

iostat使用V$ASM_DISK_STAT视图列出磁盘组统计信息。

表10-58列出了iostat命令的语法选项。

表10-58 iostat命令的选项

选项 描述

-e

显示错误统计信息（Read_Err，Write_Err）。

-G diskgroup

显示磁盘组名称的统计信息。

--suppressheader

禁止列标题。

--io

以I / O数显示信息，而不是字节。

-t

显示时间统计信息（Read_Time，Write_Time）。

- 区域

显示冷磁盘区域和热磁盘区域的信息（Cold_Reads，Cold_Writes，Hot_Reads，Hot_Writes）。

间隔

根据间隔值（秒）刷新统计信息显示。使用CTRL + C停止间隔显示。

表10-59 iostat命令输出的属性说明

属性名称 描述

团队名字

磁盘组的名称。

Dsk_Name

磁盘名称。

从磁盘读取的字节数。如果输入--io选项，则该值显示为I / O数。

Hot_Reads

从热磁盘区域读取的字节数。如果输入--io选项，则该值显示为I / O数。

写入热磁盘区域的字节数。如果输入--io选项，则该值显示为I / O数。

如果未指定刷新间隔，则显示的数字表示总字节数或I / O.如果指定了刷新间隔，则显示的值（字节或I / O）是先前值和当前值之间的差值，而不是总值。

语法和说明

lsdg [--suppressheader] [-g] [ -  discover] [ pattern ]

禁止列标题。

将结果限制为仅具有成员身份等于CANDIDATE磁盘。

- 会员

将结果限制为仅具有成员身份等于MEMBER磁盘。

仅返回有关与提供的模式匹配的指定磁盘的信息。

  • 在连接模式下， lsdsk使用V$ASM_DISK_STAT和V$ASM_DISK动态视图来检索磁盘信息。默认情况下使用V$ASM_DISK_STAT视图。

  • 在非连接模式下， lsdsk扫描磁盘头以检索磁盘信息。某些信息在此模式下不可用，某些选项与此模式无效。

注意：

Windows不支持非连接模式。

pattern将输出限制为仅匹配指定模式的磁盘。通配符字符和斜杠（ /或\ ）可以是模式的一部分。应将pattern指定为该命令的最后一个选项。有关通配符的信息，请参阅“ 通配符 ” 。

-k ， -p ， -t和--statistics选项可修改每个磁盘显示的信息量。如果指定了任何选项组合，则输出显示与每个标志关联的属性的并集。

例

还原Oracle ACFS信息时请注意以下事项：

  • 还原文件系统的元数据时，用户标识将设置为创建元数据备份时拥有文件系统的用户标识。此用户标识应存在于还原群集上。

  • md_restore恢复Oracle ACFS元数据信息。要为Oracle ACFS文件系统运行元数据还原，请创建一个脚本。必须以root身份运行此脚本才能格式化文件系统。将恢复包含的Oracle ADVM卷的元数据，例如条带大小和冗余。还会还原Oracle ACFS文件系统配置元数据，例如压缩和调整大小属性，块大小和创建的快照。Oracle ACFS文件系统数据和ACFS CRS资源信息未恢复。

  • 如果在指定单个磁盘组时使用--silent选项，并且由于另一个磁盘组上的Oracle ADVM元数据或Oracle ACFS加速器而存在依赖关系，则会删除该依赖关系，并且您必须稍后手动重新创建该依赖关系。

  • 还原Oracle ACFS加密或安全性时，必须使用用户数据还原文件系统扩展属性（xattrs）。记录需要使用文件系统应用程序数据存储和恢复xattrs非常重要。没有xattrs，就没有安全性或加密。 xattrs描述文件所属的安全领域。没有xattr，文件最终没有领域。相反的是，一个带有xattrs但没有初始化安全性的文件最终没有任何领域。

  • 在元数据还原期间，将创建一个包含先前配置的快照列表的文件。由于无法重新创建时间点稀疏快照视图，因此不会还原快照数据。

  • 如果正在使用Oracle ACFS安全性，并且尚未备份文件系统的元数据安全信息，则安全管理员可以将此信息复制到元数据备份文件位置，然后使用其余的磁盘组信息进行恢复。

  • 如果正在还原Oracle ACFS安全性和加密，则可以预先初始化用户和密码。如果不是，则提供的用户名用于初始化Oracle ACFS安全性和加密，并显示默认密码。
