156 DBMS_SQL

DBMS_SQL包提供了一个接口，使用动态SQL来解析使用PL / SQL的任何数据操作语言（DML）或数据定义语言（DDL）语句。

156.1 DBMS_SQL概述

Oracle允许您编写使用动态SQL的存储过程和匿名PL / SQL块。动态SQL语句未嵌入源程序中;相反，它们存储在运行时输入到程序或由程序构建的字符串中。这使您可以创建更多通用过程。例如，动态SQL允许您创建一个过程，该过程对名称在运行时之前未知的表进行操作。

Native Dynamic SQL是DBMS_SQL的替代方法，它允许您将动态SQL语句直接放入PL / SQL块中。在大多数情况下，Native Dynamic SQL比DBMS_SQL更易于使用和执行。但是，Native Dynamic SQL本身有一些限制：

  • 不支持所谓的方法4（对于具有未知数量的输入或输出的动态SQL语句）

  • 有些任务只能使用DBMS_SQL.执行DBMS_SQL.有关需要DBMS_SQL任务，请参阅Oracle Database PL / SQL语言参考 。

在存储过程中使用动态SQL的能力通常遵循Oracle调用接口（OCI）的模型。

Oracle调用接口程序员指南

PL / SQL与其他常见的编程语言（例如C）略有不同。例如，地址（也称为指针）在PL / SQL中不是用户可见的。因此，Oracle Call Interface和DBMS_SQL包之间存在一些差异。这些差异包括以下内容：

  • OCI按地址绑定， DBMS_SQL包按值绑定。

  • 对于DBMS_SQL您必须调用VARIABLE_VALUE来检索匿名块的OUT参数的值，并且必须在获取行之后调用COLUMN_VALUE以将行中列的值检索到程序中。

  • DBMS_SQL包的当前版本不提供CANCEL游标过程。

  • 指示符变量不是必需的，因为完全支持NULLs作为PL / SQL变量的值。

156.2 DBMS_SQL安全模型

DBMS_SQL是使用AUTHID CURRENT_USER编译的SYS拥有的包。从匿名PL / SQL块调用的任何DBMS_SQL子程序都以当前用户的权限运行。

防止打开游标号码的恶意或意外访问

当使用不表示打开游标的游标号调用任何DBMS_SQL子程序时，将引发错误ORA-29471 。引发错误时，会向警报日志发出警报，并且DBMS_SQL在会话期间无法运行。

如果在调用IS_OPEN函数时光标号的实际值表示当前在会话中打开的游标，则返回值为TRUE 。如果实际值为NULL ，则返回值为FALSE 。否则，会引发ORA-29471错误。

防止不当使用光标

保护游标免受破坏已知现有游标的安全漏洞的影响。

绑定和执行时进行检查。可选地，可以对每个单独的DBMS_SQL子程序调用执行检查。检查是：

  • 调用子程序时current_user与调用最新解析时的相同。

  • 调用子程序时启用的角色必须与调用最新解析时启用的角色相同。

  • 调用子程序时的容器与调用最新解析时的容器相同。

与使用定义者权利子程序一致，角色不适用。

如果任一检查失败，则会引发ORA-29470错误。

定义何时执行检查的机制是OPEN_CURSOR子程序的新重载，该子程序采用形式参数security_level ，允许值为NULL ， 1和2 。

  • 当security_level = 1 （或为NULL ）时，仅在绑定和执行时进行检查。

  • 当security_level = 2 ，始终进行检查。

升级注意事项

这种安全制度比以前的版本更为严格。因此， DBMS_SQL用户在升级时可能会遇到运行时错误。

156.4 DBMS_SQL操作说明

这些操作说明描述了处理查询，处理更新，插入和删除以及定位错误。

处理查询

如果使用动态SQL处理查询，则必须执行以下步骤：

  1. 通过调用DEFINE_COLUMN过程 ， DEFINE_COLUMN_LONG过程或DEFINE_ARRAY过程 ，指定要接收SELECT语句返回的值的变量。

  2. 通过调用EXECUTE函数运行SELECT语句。

  3. 调用FETCH_ROWS函数 （或EXECUTE_AND_FETCH ）以检索满足查询的行。

  4. 调用COLUMN_VALUE过程或COLUMN_VALUE_LONG过程以确定FETCH_ROWS函数为查询检索的列的值。如果使用包含对PL / SQL过程的调用的匿名块，则必须调用VARIABLE_VALUE过程以检索分配给这些过程的输出变量的值。

处理更新，插入和删除

如果使用动态SQL处理INSERT ， UPDATE或DELETE ，则必须执行以下步骤：

  1. 通过调用EXECUTE函数运行INSERT ， UPDATE或DELETE语句。

  2. 如果语句具有returning子句，则必须调用VARIABLE_VALUE过程以检索分配给输出变量的值。

找到错误

DBMS_SQL包具有附加功能，用于获取有关会话中最后引用的游标的信息。这些函数返回的值仅在运行SQL语句后立即生效。此外，某些错误定位函数仅在某些DBMS_SQL调用之后才有意义。例如，在调用其中一个PARSE过程后立即调用LAST_ERROR_POSITION函数 。

156.5.3 BIND_VARIABLE，BIND_VARIABLE_PKG或BIND_ARRAY

许多DML语句要求将程序中的数据输入到Oracle。定义包含要在运行时提供的输入数据的SQL语句时，必须在SQL语句中使用占位符来标记必须提供数据的位置。

对于在SQL声明每一个占位符，必须调用的一个BIND_ARRAY程序 ，或BIND_VARIABLE程序 ，或BIND_VARIABLE_PKG程序给一个变量的程序中的值（或数组的值）提供的占位符。随后运行SQL语句时，Oracle使用程序在输出和输入中放置的数据，或绑定变量。

DBMS_SQL可以多次运行DML语句 - 每次都使用不同的绑定变量。BIND_ARRAY过程允许您绑定标量集合，每个标量的每个值都用作每个EXECUTE的输入变量一次。这类似于OCI支持的阵列接口。

请注意，绑定到占位符的值的数据类型不能是PL / SQL-only数据类型。

156.5.4 DEFINE_COLUMN，DEFINE_COLUMN_LONG或DEFINE_ARRAY

DEFINE_COLUMN ， DEFINE_COLUMN_LONG和DEFINE_ARRAY过程指定在查询上接收SELECT值的变量。

在SELECT语句中选择的行的列由它们在从左到右出现在选择列表中的相对位置来标识。对于查询，必须调用其中一个定义过程（ DEFINE_COLUMN过程 ， DEFINE_COLUMN_LONG过程或DEFINE_ARRAY过程 ）来指定要接收SELECT值的变量，就像INTO子句为静态查询所做的那样。

使用DEFINE_COLUMN_LONG过程定义LONG列，方法与使用DEFINE_COLUMN定义非LONG列的方式相同。在使用COLUMN_VALUE_LONG过程从LONG列获取之前，必须调用DEFINE_COLUMN_LONG 。

使用DEFINE_ARRAY过程定义要在单个SELECT语句中获取行的PL / SQL集合。DEFINE_ARRAY提供了一个获取多个行的接口。在使用COLUMN_VALUE过程获取行之前，必须调用DEFINE_ARRAY 。

156.5.7 VARIABLE_VALUE，VARIABLE_VALUE_PKG，COLUMN_VALUE或COLUMN_VALUE_LONG

呼叫类型决定使用哪个过程或功能。

对于查询，请调用COLUMN_VALUE过程以确定FETCH_ROWS函数检索的列的值。

对于包含对PL / SQL过程的调用的匿名块或带有returning子句的DML语句，请调用VARIABLE_VALUE过程或VARIABLE_VALUE_PKG过程以在运行语句时检索分配给输出变量的值。

要仅获取LONG数据库列的一部分（最大可达2 GB），请使用DEFINE_COLUMN_LONG过程 。您可以将偏移量（以字节为单位）指定为列值，以及要获取的字节数。

156.5.8 CLOSE_CURSOR

当您不再需要游标进行会话时，请通过调用CLOSE_CURSOR过程来关闭游标。如果您使用的是Oracle Open Gateway，那么您可能还需要在其他时间关闭游标。有关其他信息，请参阅Oracle Open Gateway文档。

散装阵列绑定

批量选择，插入，更新和删除可以通过将多个调用捆绑为一个来增强应用程序的性能。DBMS_SQL包允许您使用PL / SQL表类型处理数据集合。

表项是无界的同类集合。在持久存储中，它们就像其他关系表一样，没有内在的排序。但是当一个表项被带入工作空间时（通过查询或通过持久数据的导航访问），或者当它被创建为PL / SQL变量或参数的值时，其元素将被赋予可以与之一起使用的下标数组样式语法，用于获取和设置元素的值。

这些元素的下标不需要密集，可以是任何数字，包括负数。例如，表项只能包含位置-10,2和7的元素。

当表项从临时工作空间移动到持久存储时，不存储下标;表项在持久存储中是无序的。

在绑定时，表从PL / SQL缓冲区复制到本地DBMS_SQL缓冲区（与所有标量类型相同），然后从本地DBMS_SQL缓冲区操作表。因此，如果在绑定调用后更改表，则该更改不会影响执行的方式。

参数

表156-28 OPEN_CURSOR功能参数

参数 描述

安全级别

指定要对打开的游标强制执行的安全保护级别。有效的安全级别的值是0 ， 1 ，和2 。当为此重载提供NULL参数值时，以及使用不带security_level参数的open_cursor重载打开的游标时，将对打开的游标强制执行默认安全级别值1 。

156.9.26 PARSE程序

此过程解析给定游标中的给定语句。所有语句都会立即解析。此外，DDL语句在解析时立即运行。

PARSE程序有多个版本：

  • 将VARCHAR2语句作为参数

  • 采用分段字符串，一个采用VARCHAR2A ，一个TABLE OF VARCHAR2(32767) ，另一个采用VARCHAR2S ，一个TABLE OF VARCHAR2(256)作为参数。这些重载连接PL / SQL表语句的元素并解析生成的字符串。您可以使用这些过程通过拆分语句来解析长度超过单个VARCHAR2变量限制的语句。

  • 以CLOB语句为参数。您可以使用解析过程的CLOB重载版本来解析大于32K字节的SQL语句。

句法

每个版本都有多个重载。

DBMS_SQL.PARSE（c IN INTEGER，语句IN VARCHAR2，language_flag IN INTEGER [[，version IN VARCHAR2 DEFAULT NULL]，apply_crossedition_trigger IN VARCHAR2 DEFAULT NULL，fire_apply_trigger IN BOOLEAN DEFAULT TRUE] [，schema IN VARCHAR2 DEFAULT NULL] [，container IN VARCHAR2 ）];
DBMS_SQL.PARSE（c IN INTEGER，语句IN CLOB，language_flag IN INTEGER [[，版本IN VARCHAR2 DEFAULT NULL]，apply_crossedition_trigger IN VARCHAR2 DEFAULT NULL，fire_apply_trigger IN BOOLEAN DEFAULT TRUE] [，schema IN VARCHAR2 DEFAULT NULL] [，container IN VARCHAR2 ）];
DBMS_SQL.PARSE（c IN INTEGER，语句IN VARCHAR2A，lb IN INTEGER，ub IN INTEGER，lfflg in BOOLEAN，language_flag IN INTEGER [[，edition IN VARCHAR2 DEFAULT NULL]，apply_crossedition_trigger IN VARCHAR2 DEFAULT NULL，fire_apply_trigger IN BOOLEAN DEFAULT TRUE] [ ，schema IN VARCHAR2 DEFAULT NULL] [，container IN VARCHAR2]]; DBMS_SQL.PARSE（c IN INTEGER，语句IN VARCHAR2s，lb IN INTEGER，ub IN INTEGER，lfflg in BOOLEAN，language_flag IN INTEGER [[，edition IN VARCHAR2 DEFAULT NULL]，apply_crossedition_trigger IN VARCHAR2 DEFAULT NULL，fire_apply_trigger IN BOOLEAN DEFAULT TRUE] [ ，schema IN VARCHAR2 DEFAULT NULL] [，container IN VARCHAR2]];

参数

参数 描述

要在其中解析语句的游标的ID号。

声明

要解析的SQL语句。大于32K的SQL语句，可以存储在CLOB 。

指定在以下条件下运行语句的版本：

  • 如果NULL和container为NULL ，则该语句将在当前版本中运行。

  • 如果指定了有效容器，则传递NULL表示该语句将在目标容器的默认版本中运行。

  • 给定用户和用于执行语句的版本，用户必须具有该版本的USE特权。

以下一般条件适用。字符串的内容作为SQL标识符处理;如果版本的实际名称中包含特殊字符或小写字符，则双引号必须包围字符串的其余部分，如果未使用双引号，则内容将为大写。

apply_crossedition_trigger

指定要应用于指定SQL的前向交叉接触触发器的非限定名称。使用要在其中执行语句的edition和current_schema设置解析名称。触发器必须由将执行该语句的用户拥有。如果指定了非NULL值，则将执行指定的交叉触发器，假设fire_apply_trigger为TRUE ，触发器已启用，触发器在作为语句目标的表上定义，语句的类型与触发器的dml_event_clause匹配，满足任何有效的WHEN和UPDATE OF限制，等等。还可以执行其他前向交叉触发，使用应用的“交叉触发DML规则”来选择，就好像指定的触发器正在对作为语句目标的表进行进一步的DML。不会执行非交叉触发和反向交叉触发。字符串的内容作为SQL标识符处理;如果触发器的实际名称中存在特殊字符或小写字符，则双引号必须包围字符串的其余部分，如果未使用双引号，则内容将为大写。

fire_apply_trigger

指示指定的apply_crossedition_trigger本身是否要执行，或者只能是用于选择其他触发器的指南。当语句替换apply_crossedition_trigger本身将执行的操作时，通常将其设置为FALSE 。如果为FALSE ，则不执行指定的触发器，但仍会选择其他触发器进行触发，就好像指定的触发器正在对作为语句目标的表执行DML。如果语句不是DML，则忽略apply_crossedition_trigger和fire_apply_trigger参数。

模式

指定用于解析非限定对象名称的模式。如果为NULL ，则当前架构是有效用户的架构。

容器

要在其中运行游标的目标容器的名称。如果为NULL或未指定，则目标容器的名称是调用容器的名称，并且不执行容器切换。如果指定了有效的容器名称，则当前用户必须是具有SET CONTAINER权限的普通用户才能切换到目标容器。如果容器开关完成，则有效用户将具有其默认角色。

使用说明

  • 使用DBMS_SQL动态运行DDL语句可能导致程序停止响应。例如，对包中的过程的调用导致包被锁定，直到执行返回到用户端。任何导致冲突锁定的操作（例如在释放第一个锁之前动态尝试删除程序包）都会使程序停止运行。

  • 由于客户端代码无法引用远程包变量或常量，因此必须显式使用常量的值。

    例如，以下代码无法在客户端上编译：

    DBMS_SQL.PARSE （ cur_hdl ， stmt_str ， DBMS_SQL 。 NATIVE ）; - 使用常量DBMS_SQL.NATIVE

    以下代码适用于客户端，因为显式提供了参数：

    DBMS_SQL 。 PARSE （ cur_hdl ， stmt_str ， 1 ）; - 编译客户端

  • 目前支持VARCHAR2S类型以实现遗留代码的向后兼容性。但是，建议您使用VARCHAR2A以获得其卓越的功能，因为VARCHAR2S将在以后的版本中弃用。

  • 要解析大于32 KB的SQL语句，可以使用PARSE过程的新CLOB重载版本而不是VARCHAR2A重载。

  • 如果container参数值与调用容器相同，则不会发生容器切换。但是，当前用户的默认角色将生效。

例外

如果使用具有编译警告的DBMS_SQL创建类型，过程，函数或包，则会引发ORA-24344异常，并且仍会创建PL / SQL单元。
