29.1数据要求

了解数据如何存储和查看以进行数据挖掘。

数据挖掘活动需要在单个表或视图中定义的数据。每条记录的信息必须存储在单独的行中。数据记录通常称为案例 。每种情况都可以选择性地通过唯一标识 案例ID 。表或视图本身可以称为案例表 。

SH模式中的CUSTOMERS表是可用于挖掘的表的示例。每个客户的所有信息都包含在一行中。案例ID是CUST_ID列。以下示例中列出的行是从SH.CUSTOMERS中选择的。

注意：

Oracle Data Mining需要单记录除关联模型之外的所有类型模型的案例数据，可以在本机上构建交易数据。

29.1.1列数据类型

了解案例表中不同类型的列数据。

案例表的列包含描述每个案例的属性。在例29-1中 ，属性为： CUST_GENDER ， CUST_YEAR_OF_BIRTH和CUST_MAIN_PHONE_NUMBER 。属性是监督模型中的预测变量或无监督模型中的描述符。该案例ID， CUST_ID ，可以视为特殊属性;它不是预测器或描述符。

Oracle数据挖掘支持标准 Oracle数据类型以及以下集合类型：

  • DM_NESTED_CATEGORICALS
  • DM_NESTED_NUMERICALS
  • DM_NESTED_BINARY_DOUBLES
  • DM_NESTED_BINARY_FLOATS

29.1.2分类和回归的数据集

了解数据集如何用于培训和测试模型。

您需要两个案例表来构建和验证分类和回归模型。一组行用于训练 model，另一组行用于测试模型。派生出来通常很方便从同一数据集构建数据和测试数据。例如，您可以随机选择60％的行来训练模型;剩余的40％可用于测试模型。

实现其他挖掘功能的模型（如属性重要性，聚类，关联或特征提取）不使用单独的测试数据。

29.1.3评分要求

大多数数据挖掘模型可以应用于称为评分的过程中的单独数据。Oracle Data Mining支持评分操作分类， 回归， 异常检测， 聚类，和特征提取。

评分过程将评分数据中的列名称与用于构建模型的列的名称相匹配。评分过程不要求所有列都存在于评分数据中。如果数据类型不匹配，Oracle Data Mining会尝试执行类型强制。例如，如果名为PRODUCT_RATING的列在训练数据中为VARCHAR2但在评分数据中为NUMBER ，则Oracle数据挖掘会有效地应用TO_CHAR()函数进行转换。

测试或评分数据中的列必须经历相同的操作转换为构建数据中的相应列。例如，如果构建数据中的AGE列已从数字转换为值CHILD ， ADULT和SENIOR ，则评分数据中的AGE列必须经历相同的转换，以便模型可以正确评估它。

注意：

Oracle Data Mining可以嵌入用户指定的转换模型中的指令，并在应用模型时重新应用它们。当转换指令嵌入到模型中时，您无需为测试或评分数据集指定它们。

Oracle Data Mining也支持自动数据准备（ADP）。启用ADP后，算法所需的转换将自动执行，并与任何用户指定的转换一起嵌入到模型中。
