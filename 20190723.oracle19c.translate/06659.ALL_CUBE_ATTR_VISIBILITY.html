2.91 ALL_CUBE_ATTR_VISIBILITY

ALL_CUBE_ATTR_VISIBILITY描述了当前用户可访问的维，层次结构和级别可见的OLAP属性。

相关观点

  • DBA_CUBE_ATTR_VISIBILITY描述了数据库中维，层次结构和级别可见的所有OLAP属性。

  • USER_CUBE_ATTR_VISIBILITY描述了当前用户拥有的维，层次结构和级别可见的OLAP属性。此视图不显示OWNER列。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

多维数据集维度的所有者

DIMENSION_NAME

VARCHAR2（128）

多维数据集维度的名称（例如TIME ）

ATTRIBUTE_NAME

VARCHAR2（128）

维度属性的名称（例如LONG_DESCRIPTION或END_DATE ）

HIERARCHY_NAME

VARCHAR2（128）

维度层次结构的名称（例如CALENDAR ）

LEVEL_NAME

VARCHAR2（128）

维度级别的名称（例如MONTH ）

FROM_TYPE

VARCHAR2（10）

标识当前行从中派生属性可见性的维度类型。可能的值：

  • DIMENSION - 从自身派生属性可见性。

  • HIERARCHY - 从关联DIMENSION或其自身显式设置的VisibleAttributes属性可见性。

  • DIM_LEVEL - 从关联DIMENSION或其自身上显式设置的VisibleAttributes属性可见性。

  • HIER_LEVEL - 从关联的DIMENSION ， HIERARCHY ， DIM_LEVEL或其自身显式设置的VisibleAttributes属性可见性。

输入

VARCHAR2（10）

标识当前行的维度类型。可能的值：

  • DIMENSION - 当TO_TYPE为DIMENSION ，仅填充DIMENSION_NAME 。

  • HIERARCHY - 当TO_TYPE为HIERARCHY ，只会填充DIMENSION_NAME和HIERARCHY_NAME 。

  • DIM_LEVEL - 当TO_TYPE为DIM_LEVEL ，只会填充DIMENSION_NAME和LEVEL_NAME 。

  • HIER_LEVEL - 当TO_TYPE为HIER_LEVEL ，只会填充HIERARCHY_NAME和LEVEL_NAME 。
