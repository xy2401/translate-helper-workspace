64 DBMS_EPG

DBMS_EPG包实现了嵌入式PL / SQL网关，使Web浏览器能够通过HTTP侦听器调用PL / SQL存储过程。

64.1 DBMS_EPG概述

DBMS_EPG包是PL / SQL用户开发和部署PL / SQL Web应用程序的平台。嵌入式PL / SQL网关是在Oracle数据库中的XML数据库HTTP服务器中运行的网关的嵌入式版本。它在数据库中提供了mod_plsql的核心功能，但不需要Apache支持的Oracle HTTP服务器。

为了通过HTTP从浏览器访问PL / SQL应用程序，必须创建数据库访问描述符（DAD）并将其映射到虚拟路径。DAD是用于数据库访问的一组配置值，虚拟路径映射使应用程序可以在XML DB HTTP Server的虚拟路径下访问。DAD在XML DB HTTP Server中表示为servlet。

64.2 DBMS_EPG安全模型

需要XDBADMIN角色来调用配置界面。它可以由数据库用户“ XDB ”调用。

任何用户都可以调用授权接口。

64.6.1 AUTHORIZE_DAD过程

此过程授权DAD使用数据库用户的权限调用过程和访问文档表。调用者总是可以授权使用她/他自己的权限。

也可以看看：

授权该组中其他子程序的子程序

句法

DBMS_EPG.AUTHORIZE_DAD（dad_name IN VARCHAR2，路径IN VARCHAR2 DEFAULT NULL）;

参数

表64-5 AUTHORIZE_DAD过程参数

参数 描述

dad_name

要创建的DAD的名称

用户

具有取消授权权限的用户。如果使用，则假定调用者。

使用说明

  • 要授权使用其他用户的权限，调用者必须具有ALTER USER系统特权。

  • DAD必须存在，但其“database-username”DAD属性不必设置为用户进行授权。

  • 多个用户可以授权相同的DAD，并且由DAD的“数据库用户名”设置来决定使用哪个用户的权限。

例外

如果DAD或用户不存在，或者调用者没有所需的系统特权，则引发错误。

例子

DBMS_EPG.AUTHORIZE_DAD（ 'HR'）;
