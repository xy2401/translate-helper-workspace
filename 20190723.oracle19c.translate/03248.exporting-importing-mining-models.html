8.3导出和导入挖掘模型

您可以将数据挖掘模型导出到平面文件以备份正在进行的工作或将模型移动到Oracle Database Enterprise Edition的其他实例（例如从开发数据库到测试数据库）。

导出和导入模型的所有方法都基于 Oracle Data Pump技术。

DBMS_DATA_MINING包中包含用于导出和导入单个挖掘模型的EXPORT_MODEL和IMPORT_MODEL过程。EXPORT_MODEL和IMPORT_MODEL使用Oracle Data Pump的导出和导入功能。

8.3.1关于Oracle Data Pump

Oracle Data Pump由两个命令行客户端和两个PL / SQL包组成。命令行客户端expdp和impdp为Data Pump导出和导入实用程序提供了易于使用的界面。您可以使用expdp和impdp导出和导入整个模式或数据库。

Data Pump导出实用程序将架构对象（包括构成挖掘模型的表和元数据）写入转储文件集。Data Pump导入实用程序从转储文件集中检索架构对象（包括模型表和元数据），并将其还原到目标数据库中。

expdp和impdp不能用于导出/导入单个挖掘模型。

8.3.2出口和进口采矿模型的选项

列出导出和导入挖掘模型的选项。

导出模型的导出和导入选项如下表所述。

表8-1 Oracle Data Mining的导出和导入选项

任务 描述

导出或导入完整数据库

（仅限DBA）使用expdp导出完整数据库和impdp以导入完整数据库。包括数据库中的所有挖掘模型。

导出或导入架构

使用expdp导出模式和impdp以导入模式。包含模式中的所有挖掘模型。

导出或导入数据库中的单个模型

使用DBMS_DATA_MINING.EXPORT_MODEL导出单个模型，使用DBMS_DATA_MINING.IMPORT_MODEL导入单个模型。这些过程可以导出和导入与特定条件匹配的单个挖掘模型，所有挖掘模型或挖掘模型。

默认情况下， IMPORT_MODEL模型导回到导出它们的模式中。您可以指定schema_remap参数以将模型导入到其他模式中。您可以使用schema_remap指定tablespace_remap ，以将模型导入到使用不同表空间的模式中。

您可能需要特别的数据库中的特权，用于将模型导入不同的模式。这些权限由EXP_FULL_DATABASE和IMP_FULL_DATABASE角色授予，这些角色仅对特权用户（例如SYS或具有DBA角色的用户）可用。您不需要这些角色来导出或导入您自己的架构中的模型。

要导入模型，您必须具有与创建转储文件集的用户相同的数据库权限。否则，具有完全系统特权的DBA必须导入模型。

从远程数据库导出或导入单个模型

使用数据库链接将单个模型导出到远程数据库或从远程数据库导入单个模型。数据库链接是一个数据库中的模式对象，可以访问不同数据库中的对象。必须在执行EXPORT_MODEL或IMPORT_MODEL之前创建链接。

要创建专用数据库链接，您必须具有CREATE DATABASE LINK系统特权。要创建公共数据库链接，您必须具有CREATE PUBLIC DATABASE LINK系统特权。此外，您必须具有远程Oracle数据库的CREATE SESSION系统特权。必须在本地和远程Oracle数据库上安装Oracle Net。

8.3.3 EXPORT_MODEL和IMPORT_MODEL的目录对象

了解如何使用目录对象来标识转储文件集的位置。

EXPORT_MODEL和IMPORT_MODEL使用目录对象来标识转储文件集的位置。目录对象是主机上物理目录的数据库中的逻辑名称。

要导出数据挖掘模型，您必须具有对目录对象及其所代表的文件系统目录的写访问权。要导入数据挖掘模型，您必须具有对目录对象和文件系统目录的读访问权。此外，数据库本身必须能够访问文件系统目录。您必须具有CREATE ANY DIRECTORY权限才能创建目录对象。

以下SQL命令创建名为dmuser_dir的目录对象。它表示的文件系统目录必须已存在，并具有操作系统授予的共享读/写访问权限。 

创建或替换目录dmuser_dir AS'/ dm_path / dm_mining';

以下SQL命令为用户dmuser提供对dmuser_dir读写访问dmuser_dir 。 

GRANT READ，WRITE ON DIRECTORY dmuser_dir to dmuser;

8.3.4使用EXPORT_MODEL和IMPORT_MODEL

这些示例使用EXPORT_MODEL和IMPORT_MODEL说明了各种导出和导入方案。

这些示例使用示例8-1中显示的目录对象dmdir和两个模式dm1和dm2 。两种模式都具有数据挖掘权限。 dm1有两种型号。 dm2有一个型号。 

SELECT owner，model_name，mining_function，algorithm FROM all_mining_models;所有者MODEL_NAME MINING_FUNCTION ALGORITHM ---------- -------------------- ---------------- ---- -------------------------- DM1 EM_SH_CLUS_SAMPLE CLUSTERING EXPECTATION_MAXIMIZATION DM1 DT_SH_CLAS_SAMPLE分类DECISION_TREE DM2 SVD_SH_SAMPLE FEATURE_EXTRACTION SINGULAR_VALUE_DECOMP

示例8-1创建目录对象

- 以系统用户身份连接CREATE OR REPLACE DIRECTORY dmdir AS'/ scratch / dmuser / expimp';授权阅读，写入目录dmdir至dm1;授权阅读，写入目录dmdir to dm2; SELECT * FROM all_directories WHERE directory_name IN'DMDIR'; OWNER DIRECTORY_NAME DIRECTORY_PATH ---------- -------------------------- ----------- ----------------------------- SYS DMDIR / scratch / dmuser / expimp

示例8-2从DM1导出所有模型

- 连接为dm1 BEGIN dbms_data_mining.export_model（filename =>'all_dm1'，directory =>'dmdir'）;结束; /

在/scratch/dmuser/expimp中创建日志文件和转储文件，这是与dmdir关联的物理目录。日志文件的名称是dm1_exp_11.log 。转储文件的名称是all_dm101.dmp 。

示例8-3将模型导入DM1

在例8-2中导出的模型仍然存在于dm1 。由于导入不会覆盖具有相同名称的模型，因此必须先删除模型，然后再将模型导回到同一模式中。 

BEGIN dbms_data_mining.drop_model（'EM_SH_CLUS_SAMPLE'）; dbms_data_mining.drop_model（ 'DT_SH_CLAS_SAMPLE'）; dbms_data_mining.import_model（filename =>'all_dm101.dmp'，directory =>'DMDIR'）;结束; / SELECT model_name FROM user_mining_models; MODEL_NAME ------------------------------ DT_SH_CLAS_SAMPLE EM_SH_CLUS_SAMPLE

8.3.5导出和导入序列化模型

从Oracle Database Release 18c开始， EXPORT_SERMODEL和IMPORT_SERMODEL过程可用于导出和导入序列化模型。

序列化格式允许将模型移动到另一个平台（数据库外）进行评分。模型以BLOB导出，可以保存在BFILE 。导入例程获取BLOB的序列化内容以及要使用内容创建的模型的名称。

8.3.6从PMML导入

您可以导入预测模型标记语言（PMML）中表示的回归模型。

PMML是由Data Mining Group（ http://www.dmg.org ）指定的基于XML的标准。符合PMML的应用程序可以部署由任何供应商创建的符合PMML的模型。Oracle Data Mining支持PMML 3.1的核心功能，用于回归模型。

您可以导入PMML中表示的回归模型。模型必须是RegressionModel类型，可以是线性回归或二元逻辑回归。
