33.5创建文本策略

Oracle Text策略指定必须如何解释文本内容。您可以提供文本策略来管理模型，属性或模型和单个属性。

如果存在特定于模型的策略且一个或多个属性具有自己的策略，则Oracle Data Mining将使用指定属性的属性策略和其他属性的特定于模型的策略。

CTX_DDL.CREATE_POLICY过程创建文本策略。 

CTX_DDL.CREATE_POLICY（policy_name IN VARCHAR2，filter IN VARCHAR2 DEFAULT NULL，section_group IN VARCHAR2 DEFAULT NULL，lexer IN VARCHAR2 DEFAULT NULL，stoplist IN VARCHAR2 DEFAULT NULL，wordlist IN VARCHAR2 DEFAULT NULL）;

CTX_DDL.CREATE_POLICY的参数如下表所述。

表33-4 CTX_DDL.CREATE_POLICY过程参数

参数名称 描述

POLICY_NAME

新策略对象的名称。Oracle Text策略和文本索引共享相同的命名空间。

过滤

指定如何将文档转换为纯文本以进行索引。例子有： CHARSET_FILTER字符集和NULL_FILTER纯文本，HTML和XML。

有关filter值，请参阅Oracle Text Reference中的 “ 过滤器类型 ” 。

section_group

标识文档中的部分。例如， HTML_SECTION_GROUP定义HTML文档中的部分。

有关section_group值，请参阅Oracle Text Reference中的 “ Section Group Types ” 。

注意：您可以指定CONTEXT索引支持的任何节组。

词法分析器

标识要编制索引的语言。例如， BASIC_LEXER是用于使用以空格分隔的单词（例如英语和大多数西欧语言）的语言从文本中提取术语的词法分析器。

有关lexer值，请参阅Oracle Text Reference中的 “ 词汇表类型 ” 。

终止列表

指定要从术语提取中排除的单词和主题。例如，单词“the”通常位于英语语言文档的停止列表中。

默认情况下使用系统提供的停止列表。

请参阅Oracle Text Reference中的 “ Stoplists ” 。

词汇表

指定必须如何扩展词干和模糊查询。词干定义词的根形式，以便不同的语法形式具有单个表示。模糊查询包括单词表示中的常见拼写错误。

请参阅Oracle Text Reference中的 “ BASIC_WORDLIST ” 。
