验证要求

在执行本附录中的任何故障排除步骤之前，请确保系统满足要求并且您已完成相应主题中指定的所有预安装任务。

阅读发行说明

在安装之前，请阅读产品的发行说明。OTN网站上还提供了最新版本的发行说明：

http://docs.oracle.com

如果发生安装错误该怎么办

如果在安装过程中遇到错误：

  • 不要退出Oracle Universal Installer。

  • 如果在输入有关其中一个安装屏幕的错误信息后单击“ 下一步” ，则单击“上一步”返回到屏幕并更正信息。

  • 如果在Oracle Universal Installer复制或链接文件时遇到错误，请参阅查看安装会话日志 。

  • 如果在配置助手运行时遇到错误，请参阅配置助手故障排除 。

  • 如果无法解决问题，请按照“ 安装失败后清理”中列出的步骤删除失败的安装 。

查看安装会话的日志

在安装过程中，Oracle Universal Installer会在日志文件中记录它执行的所有操作。如果在安装过程中遇到问题，请查看日志文件以获取有关问题可能原因的信息。

要查看日志文件，请按照下列步骤操作：

  1. 如有必要，请输入以下命令以确定oraInventory目录的位置：

    对于AIX和Linux：

    $ cat /etc/oraInst.loc

    对于Solaris SPARC：

    #more /var/opt/oracle/oraInst.loc

    此文件中的inventory_loc参数指定oraInventory目录的位置。

  2. 输入以下命令将目录更改为Oracle Universal Installer日志文件目录，其中orainventory_location是oraInventory目录的位置：
    $ cd / orainventory_location / logs
  3. 输入以下命令以确定日志文件的名称：
    $ ls -ltr

    此命令按创建顺序列出文件，最后显示最新文件。安装程序日志文件的名称与以下内容类似，其中date_time表示安装开始的日期和时间：

    installActions date_time .log
  4. 要查看日志文件中最新的条目，其中最有可能出现有关问题的信息，请输入类似于以下内容的命令：
    $ tail -50 installActions date_time .log |更多

    此命令显示日志文件中的最后50行。

  5. 如果Oracle Universal Installer显示的错误或日志文件中列出的错误指示重新链接问题，请参阅以下文件以获取更多信息：
    $ ORACLE_HOME /安装/ make.log里

配置助手故障排除

要解决配置助手运行时发生的安装错误，请执行以下操作：

  • 查看“ 查看安装会话日志”中列出的安装日志文件。

  • 查看位于$ORACLE_HOME/cfgtoollogs目录中的特定配置助手日志文件。尝试解决导致错误的问题。

  • 如果你看到“致命错误。重新安装“消息，通过查看日志文件查找问题的原因。有关详细说明，请参阅致命错误 。

配置助手失败

安装屏幕底部会注明Oracle配置助手故障。配置助手界面显示其他信息（如果有）。配置助手执行状态存储在以下文件中：

oraInventory_location / logs / installActions date_time .log

执行状态代码列在下表中：

状态 结果代码

配置助手成功

0

配置助手失败

1

配置助手已取消

-1

致命错误

如果在配置助手运行时收到致命错误，则：

  1. 按照在安装失败后清理中所述删除失败的安装 。
  2. 纠正致命错误的原因。
  3. 重新安装Oracle软件。

静默模式响应文件错误处理

要确定静默模式安装是成功还是失败，请参阅以下日志文件：

/ oraInventory_location / logs / silentInstall date_time .log

如有必要，请参阅上一节以获取有关确定oraInventory目录位置的信息。

静默安装失败，如果：

  • 您没有指定响应文件

  • 您指定了不正确或不完整的响应文件

  • Oracle Universal Installer遇到错误，例如磁盘空间不足

Oracle Universal Installer或配置助手在运行时验证响应文件。如果验证失败，则静默模式安装或配置过程结束。 Oracle Universal Installer会处理具有错误上下文，格式或类型的参数的值，就好像文件中未指定任何值一样。

安装失败后清理

如果安装失败，则必须删除Oracle Universal Installer在尝试安装期间创建的文件并删除Oracle主目录。执行以下步骤以删除文件：

  1. 按运行Oracle Universal Installer中的说明启动Oracle Universal Installer 。
  2. 单击欢迎窗口上卸装产品 ，或点击任何安装程序窗口上安装的产品 。

    出现“清单”窗口，列出已安装的产品

  3. 选择包含要删除的产品的Oracle主目录，然后单击“ 删除” 。
  4. 手动删除在安装失败期间创建的Oracle主目录。
  5. 重新安装Oracle软件。
