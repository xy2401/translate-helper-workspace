Oracle安全备份（OSB）云模块使您能够利用Amazon Simple Storage Service（S3）提供的基于Internet的数据存储服务进行RMAN备份和恢复任务。

E.1关于使用Oracle Secure Backup云模块在云上进行备份

Oracle Secure Backup云模块是Oracle Secure Backup产品系列的一部分，可以灵活地将数据库备份到Amazon S3 Cloud和磁带。借助此云产品，本地磁盘备份可直接发送到Amazon S3以进行异地存储，并与Recovery Manager（RMAN）功能完全集成。

Oracle Secure Backup云模块有效地处理Oracle数据库备份到S3存储。您可以从Oracle Database 9 i Release 2或更高版本开始备份Oracle数据库。此外，Oracle Secure Backup云模块备份可与Oracle Enterprise Manager和您自定义的RMAN脚本等工具配合使用。Oracle Secure Backup云模块不备份操作系统文件。

Oracle Secure Backup云模块使用RMAN SBT（系统备份到磁带）界面扩展Amazon S3功能以进行Oracle备份操作。Oracle安全备份云模块提供易于管理，经济高效且可扩展的替代方案，可用于维护内部数据存储和管理本地完全配置的备份基础架构。

与传统的基于磁带的异地备份相比，Oracle Secure Backup云模块具有以下几个优势：

  • 连续可访问性

    始终可以访问存储在Amazon S3存储上的Oracle Secure Backup云模块备份。云存储服务可用性和访问模型可帮助组织简化恢复操作。例如，在执行还原之前无需发送或加载磁带。您仍然可以使用熟悉的标准工具（如Enterprise Manager），并且您组织的当前脚本将继续执行备份和还原任务。由于能够持续轻松地访问备份，因此可以大大减少恢复备份所花费的时间。

  • 提高可靠性

    由于S3存储是基于磁盘的，因此它本身比磁带介质更可靠。互联网存储服务提供商会保留数据的多个冗余副本，以实现可用性和可扩展性，并且这种做法可以为您的组织和数据带来更高的可靠性。

E.2.7在RMAN存储库中存储配置信息（O ptional ）

为避免每次调用备份时都必须提供配置信息，最好将Oracle Secure Backup云模块配置信息存储在RMAN存储库中。

此示例适用于11g之前的Release 2数据库：

RMAN>配置通道设备类型sbt parms“SBT_LIBRARY = / orclhome / lib / libosbwsll.so ENV =（OSB_WS_PFILE = / orclhome / dbs / osbwst1.ora）';
使用目标数据库控制文件而不是恢复目录新的RMAN配置参数：“SBT_LIBRARY = / orclhome / lib / libosbwsll.so ENV =（OSB_WS_PFILE = / orclhome / dbs / osbwst1.ora）';成功存储新的RMAN配置参数

此示例完成后，系统将配置为OSB云模块备份，您可以使用常用的RMAN备份和还原命令。

注意：对于11 g第2版数据库及更高版本，必须使用SBT_PARMS参数指定环境变量。

安装和配置Oracle Secure Backup云模块后，即可连接到目标数据库并配置RMAN通道。

您必须在命令中指定库和配置文件。

以下示例为Oracle 11g第2版数据库配置RMAN通道：

RMAN> run {allocate channel dev1 type sbt parms ='SBT_LIBRARY = / orclhome / lib / libosbws11.so,SBT_PARMS =（OSB_WS_PFILE = / orclhome / dbs / osbwst1.ora）'; }

此时，您可以发出通常的RMAN备份和还原命令。

注意：对于Oracle 11g第2版数据库及更高版本，您必须使用 SBT_PARMS参数，用于指定环境变量。对于Oracle 11g前版本的数据库，您仍然可以使用 PARMS选项的ENV参数用于指定环境变量。
