Oracle软件安装所有者的限制

查看为拥有Oracle软件而创建的用户的以下限制。

  • 如果您打算将多个Oracle软件所有者用于不同的Oracle数据库主目录，则Oracle建议您为Oracle Grid Infrastructure软件（Oracle Clusterware和Oracle ASM）创建单独的软件所有者，并使用该所有者运行Oracle Grid Infrastructure安装。

  • 在安装过程中，必须在集群成员节点之间设置SSH。SSH Universal Installer（安装程序）可以自动设置SSH。要使SSH能够自动设置，请在其配置文件中创建没有任何stty命令的Oracle安装所有者，并删除在向终端生成消息的登录期间触发的其他安全措施。这些消息，邮件检查和其他显示会阻止Oracle软件安装所有者帐户使用安装程序中内置的SSH配置脚本。如果未禁用它们，则必须手动配置SSH，然后才能运行安装。

  • 如果您计划安装Oracle数据库或Oracle RAC，那么Oracle建议您为Oracle Grid Infrastructure和Oracle数据库安装创建单独的用户。如果使用一个安装所有者，那么当您要执行管理任务时，必须将$ORACLE_HOME的值更改为要管理的实例（Oracle ASM，Oracle Grid Infrastructure主目录或Oracle主目录中的数据库） ，使用命令语法，例如以下示例，其中/u01/app/19.0.0/grid是Oracle Grid Infrastructure主目录：

    $ ORACLE_HOME = / u01 / app / 19.0.0 / grid;导出ORACLE_HOME

  • 如果在环境变量$ORACLE_HOME设置为其他Oracle主目录或网格主路径时尝试使用sqlplus ， lsnrctl或asmcmd命令管理Oracle主目录或Grid主目录实例，则会遇到错误。例如，从数据库主目录启动SRVCTL时，应将$ORACLE_HOME设置为该数据库主目录，否则SRVCTL将失败。例外情况是在Oracle Grid Infrastructure主页中使用SRVCTL时。在这种情况下，将忽略$ORACLE_HOME ，并且Oracle主环境变量不会影响SRVCTL命令。在所有其他情况下，您必须将$ORACLE_HOME更改$ORACLE_HOME要管理的实例。

  • 要为不同的Oracle软件安装创建单独的Oracle软件所有者和单独的操作系统特权组，请注意，每个用户都必须将Oracle中央库存组（ oraInventory组）作为其主要组。该组的成员被授予OINSTALL系统权限以写入Oracle中央库存（ oraInventory ）目录，并且还被授予对各种Oracle Clusterware资源，OCR密钥，Oracle Clusterware主目录中DBA需要写访问权限的目录的权限，以及其他必要的特权。此组的成员还被授予执行权限，以启动和停止Clusterware基础结构资源和数据库。在Oracle文档中，该组在代码示例中表示为oinstall 。

  • 每个Oracle软件所有者必须是同一中央库存oraInventory组的成员，并且他们必须将此组作为其主要组，以便所有Oracle软件安装所有者共享相同的OINSTALL系统特权。Oracle建议您没有多个Oracle安装的中央清单。如果Oracle软件所有者具有不同的中央库存组，则可能会损坏中央库存。
