创造包装身体

目的

包体使用PL / SQL定义。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用CREATE PACKAGE BODY语句创建存储包的主体，该包是相关过程，存储函数和存储在数据库中的其他程序对象的封装集合。包体定义了这些对象。在早期的CREATE PACKAGE语句中定义的包 规范声明了这些对象。

包是作为独立模式对象创建过程和函数的替代方法。

也可以看看：

  • 创建功能和创建过程以获取有关创建独立功能和过程的信息

  • CREATE PACKAGE用于讨论包，包括如何创建包

  • 有关修改包的信息，请参阅ALTER PACKAGE

  • 有关从数据库中删除包的信息，请参阅DROP PACKAGE

先决条件

要在您自己的架构中创建或替换包，您必须具有CREATE PROCEDURE系统特权。要在另一个用户的模式中创建或替换包，您必须具有CREATE ANY PROCEDURE系统特权。在这两种情况下，必须在与包相同的模式中创建包体。

要在Oracle数据库预编译器程序中嵌入CREATE PACKAGE BODY语句，必须使用关键字END-EXEC然后是特定语言的嵌入式SQL语句终止符来终止该语句。

语义

或替换

指定OR REPLACE以重新创建包体（如果已存在）。使用此子句可以更改现有包的主体，而无需删除，重新创建和重新授予先前在其上授予的对象权限。如果更改包体，则Oracle Database会重新编译它。

以前被授予重新定义的包特权的用户仍然可以访问该包而无需重新授予特权。
