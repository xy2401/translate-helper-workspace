7.99 V $ BACKUP_FILES

V$BACKUP_FILES显示有关所有RMAN备份（映像副本和备份集）和归档日志的信息。

此视图模拟LIST BACKUP和LIST COPY RMAN命令。此视图要求使用DBMS_RCVMAN.SETDATABASE过程设置数据库。

柱 数据类型 描述

PKEY

数

备份的主键

BACKUP_TYPE

VARCHAR2（32）

备份类型：

  • 备份集

  • 复制

  • 代理复制

文件类型

VARCHAR2（32）

文件类型：

  • DATAFILE

  • CONTROLFILE

  • SPFILE

  • REDO LOG

  • ARCHIVED LOG

  • COPY （用于图像复制备份）

  • PIECE （备用件）

保持

VARCHAR2（3）

指示备份是否具有与CONFIGURE RETENTION POLICY （ YES ）的值不同的保留策略（ NO ）

KEEP_UNTIL

日期

如果指定了BACKUP命令的KEEP UNTIL TIME子句，则此列显示备份过时的日期。如果KEEP_OPTIONS null且KEEP_OPTIONS不为null，则备份永远不会过时。

KEEP_OPTIONS

VARCHAR2（13）

KEEP备份选项：

  • LOGS - RMAN保留恢复备份所需的日志

  • NOLOGS - RMAN不保留恢复备份所需的日志

如果此列为null，则备份没有KEEP选项，并且将根据保留策略使其过时。

状态

VARCHAR2（16）

备份状态：

  • 可用

  • UNAVAILABLE

  • 过期

  • 其他

FNAME

VARCHAR2（1024）

文件名

标签

VARCHAR2（32）

作品，副本或代理副本的标记

媒体

VARCHAR2（80）

作品或代理副本的介质ID

RECID

数

重新记录控制文件中的记录

邮票

数

控制文件中记录的戳记

设备类型

VARCHAR2（255）

存储备份的媒体设备的类型

BLOCK_SIZE

数

备份的块大小（以字节为单位）

完成时间

日期

备份完成的时间

COMPRESSED

VARCHAR2（3）

指示备份件是否已压缩（ YES ）或否（ NO ）;仅在FILE_TYPE为PIECE有效。图像副本无法压缩。

已过时

VARCHAR2（3）

指示备份件或副本是否已过时（ YES ）或否（ NO ）;仅在FILE_TYPE为PIECE或COPY

BYTES

数

文件大小（以字节为单位）

BS_KEY

数

备份集的主键（仅当BACKUP_TYPE为BACKUP SET有效）

BS_COUNT

数

控制文件记录中的备份集计数（仅当BACKUP_TYPE为BACKUP SET有效）

BS_STAMP

数

从控制文件记录中设置备份集的戳记（仅当BACKUP_TYPE为BACKUP SET有效）

BS_TYPE

VARCHAR2（32）

备份集的类型（仅当BACKUP_TYPE为BACKUP SET有效）：

  • DATAFILE

  • 存档日志

BS_INCR_TYPE

VARCHAR2（32）

备份集的增量类型（仅当BACKUP_TYPE为BACKUP SET有效）

BS_PIECES

数

备份集中的备份件数（仅当BACKUP_TYPE为BACKUP SET有效）

BS_COPIES

数

备份集的副本数量（仅当FILE_TYPE为PIECE且BACKUP_TYPE为BACKUP SET ）

BS_COMPLETION_TIME

日期

备份集的完成时间（仅当BACKUP_TYPE为BACKUP SET有效）

BS_STATUS

VARCHAR2（16）

备份集的状态（仅当BACKUP_TYPE为BACKUP SET有效）：

  • 可用

  • UNAVAILABLE

  • 过期

  • OTHER - 作为备份集的一部分的片段没有统一的状态（也就是说，其中一些是可用的，有些则不可用）

BS_BYTES

数

备份集中所有备份片段大小的总和（仅当BACKUP_TYPE为BACKUP SET有效）

BS_COMPRESSED

VARCHAR2（3）

指示备份集的备份是否已压缩（ YES ）或否（ NO ）;仅在BACKUP_TYPE为BACKUP SET有效

BS_TAG

VARCHAR2（1024）

备份集的标签。如果片段具有不同的标签，则所有片段标签被连接并用逗号分隔。仅在BACKUP_TYPE为BACKUP SET ）

BS_DEVICE_TYPE

VARCHAR2（255）

备份集的设备类型。如果有多个设备类型，则用逗号分隔。仅在BACKUP_TYPE为BACKUP SET

BP_PIECE＃

数

备份集内的片段数（仅当FILE_TYPE为PIECE且BACKUP_TYPE为BACKUP SET ）

BP_COPY＃

数

备份集的副本数量（仅当FILE_TYPE为PIECE且BACKUP_TYPE为BACKUP SET ）

DF_FILE＃

数

数据文件的绝对文件号（仅当FILE_TYPE为DATAFILE有效）

DF_TABLESPACE

VARCHAR2（30）

数据文件的表空间名称（仅当FILE_TYPE为DATAFILE有效）

DF_RESETLOGS_CHANGE＃

数

创建控制文件或数据文件时最新RESETLOGS系统更改编号（SCN）（仅当FILE_TYPE为DATAFILE有效）

DF_CREATION_CHANGE＃

数

创建控制文件或数据文件的SCN（仅当FILE_TYPE为CONTROLFILE或DATAFILE ）

DF_CHECKPOINT_CHANGE＃

数

最新控制文件或数据文件检查点的系统更改编号（SCN）（仅当FILE_TYPE为CONTROLFILE或DATAFILE ）

DF_CKP_MOD_TIME

日期

SPFILE情况下的修改时间，否则控制文件或数据文件被检查点的时间（仅当FILE_TYPE为SPFILE ， CONTROLFILE或DATAFILE ）

RL_THREAD＃

数

归档日志的重做日志线程号（仅当FILE_TYPE为REDO LOG有效）

RL_SEQUENCE＃

数

归档日志的重做日志序列号（仅当FILE_TYPE为REDO LOG有效）

RL_RESETLOGS_CHANGE＃

数

创建记录时最新RESETLOGS系统更改编号（SCN）（仅当FILE_TYPE为REDO LOG有效）

RL_FIRST_CHANGE＃

数

重做日志的第一个SCN（仅当FILE_TYPE为REDO LOG有效）

RL_FIRST_TIME

日期

Oracle数据库切换到重做日志的时间（仅当FILE_TYPE为REDO LOG有效）

RL_NEXT_CHANGE＃

数

线程中下一个重做日志的第一个SCN（仅当FILE_TYPE为REDO LOG有效）

RL_NEXT_TIME

日期

线程中下一个重做日志的第一个时间戳（仅当FILE_TYPE为REDO LOG有效）

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
