4.6关于设置系统属性

在RDBMS会话中，只要在RDBMS会话中启动Java会话，就可以维护一组添加到系统属性的值。这组值在较长的Java会话状态持续时间内保持有效，这通常与RDBMS会话的持续时间相同。

DBMS_JAVA包中有一组PL / SQL函数，用于在内部RDBMS会话持续DBMS_JAVA中设置，检索，删除和显示键值对，其中一对的两个元素都是字符串（ VARCHAR2 ），并且最多只有一个对于给定的密钥配对。这些功能如下：

  • set_property

  • get_property

  • remove_property

  • show_property

set_property

此函数为系统属性建立一个值，然后在初始化Java会话时将其用于当前RDBMS会话的持续时间。第一个参数是属性的名称，第二个参数是要为其建立的值。除非出现一些错误，否则set_property的返回值为null。例如，如果尝试设置指定属性的值，则返回错误消息。 

FUNCTION set_property（名称VARCHAR2，值VARCHAR2）RETURN VARCHAR2;

get_property

此函数返回先前由set_property.建立的任何值set_property. 如果没有这样的值，它返回null。 

FUNCTION get_property（name VARCHAR2）RETURN VARCHAR2;

remove_property

此函数删除先前由set_property.建立的任何值set_property. 除非发生错误，否则返回值为null，在这种情况下会返回错误消息。 

功能remove_property（名称VARCHAR2）RETURN VARCHAR2;

show_property

如果name为null，则此函数显示输入名称的形式name = value的消息，或所有已建立的属性绑定的消息。成功完成后，此函数的返回值为null，否则为错误消息。输出显示在您当前指向Java输出的任何位置。 

功能show_property（名称VARCHAR2）RETURN VARCHAR2;

在初始化Java会话之前，此表中的值将添加到Oracle JVM已维护的一组默认系统属性值中。使用命令行界面运行Java方法时， -D选项确定的值（如果存在）将覆盖表中设置的值。一旦终止Java会话， -D选项建立的值就会过时，并且键将设置为表中存在的原始值。
