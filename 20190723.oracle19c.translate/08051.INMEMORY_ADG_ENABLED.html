1.137 INMEMORY_ADG_ENABLED

INMEMORY_ADG_ENABLED指示除了内存高速缓存大小之外是否启用Active Data Guard的内存。

Oracle RAC

应在所有实例上使用相同的值。如果在一个实例上更改了此参数的值，则应将参数的值更改为其他实例上的相同值。

对于Active Data Guard，介质恢复需要在应用重做时检索内存中的对象，并在查询前进后使相关对象无效。此参数控制介质恢复是否执行检索和无效。

应在备用数据库上设置此参数。此参数的值在主数据库上没有意义。

如果备用数据库是Oracle RAC实例，则所有不同的备用实例必须将此参数设置为相同的值。

此参数是一个动态系统可修改参数，仅当Active Data Guard恢复未运行时才能更改其值。

如果启动多实例重做应用（MIRA）Active Data Guard恢复，则MIRA中涉及的所有实例必须将参数设置为相同的值，否则MIRA将发出错误信号。

例

如果备用数据库是Oracle RAC实例，则应在所有Oracle RAC实例上将此参数设置为相同的值：

SQL> alter system set inmemory_adg_enabled = true sid ='*';
