5为WebSphere MQ安装Oracle Database Gateway

以下主题将指导您安装适用于WebSphere MQ的Oracle Database Gateway企业版。

安装

此表指导您完成安装过程。

表5-1指导您完成安装过程。对于每个屏幕，执行“响应”列中描述的操作。

表5-1为WebSphere MQ安装Oracle Database Gateway

Oracle Universal Installer屏幕 响应

欢迎

单击下一步 。

文件位置

屏幕的“源”部分用于指定Oracle Universal Installer用于为WebSphere MQ安装Oracle Database Gateway的源位置。您无需在“路径”字段中编辑文件规范 - 此字段的默认设置指向Oracle Database Gateway for WebSphere MQ CD-ROM上的Oracle Universal Installer文件。

“文件位置”屏幕的“目标”部分中的“路径”字段是您指定安装目标的位置。您无需在“路径”字段中编辑路径规范。此字段的默认设置指向ORACLE_HOME 。在“文件位置”屏幕中设置字段后，根据需要单击“ 下一步”继续。从CD-ROM加载必要的信息后，安装程序将显示“可用产品”屏幕。

可用产品

选择Oracle Database 12.2并单击Next继续。Oracle Universal Installer将显示“安装类型”屏幕。

安装类型

选择Custom ，然后单击Next继续。Oracle Universal Installer显示“可用产品组件”屏幕。

可用的产品组件

使用复选框指示要安装的产品组件。默认情况下，会为您选择所有可用组件。您需要通过单击复选框取消选择不需要的组件。单击“ 下一步”继续，Oracle Universal Installer将显示“安装WebSphere MQ队列管理器的位置”？屏幕。

WebSphere MQ队列管理器安装在哪里？

如果MQM在与网关相同的系统上运行，请选择“ 本地”;如果MQM在与网关不同的系统上运行，请选择“ 远程” 。单击“ 下一步”继续。

本地WebSphere MQ队列管理器名称

如果在安装WebSphere MQ队列管理器的位置为MQM选择Local？屏幕，然后显示本地WebSphere MQ队列管理器名称屏幕。在“队列管理器”字段中键入本地WebSphere MQ队列管理器名称。单击“下一步”继续，Oracle Universal Installer将显示“摘要”屏幕。

远程WebSphere MQ队列管理器名称

如果在“本地WebSphere MQ队列管理器名称”屏幕中为MQM选择“远程”，则会显示“远程WebSphere MQ队列管理器名称”屏幕。在“队列管理器”字段中输入远程WebSphere MQ队列管理器的名称，并在“通道”字段中输入WebSphere MQ通道名称。

有关服务器连接通道的信息，请参阅有关WebSphere MQ Clients的IBM出版物，或者向WebSphere MQ系统管理员询问您希望网关连接到的队列管理器的通道定义。

定义语法是：

CHANNEL_NAME / PROTOCOL / server_address [（port）]

其中CHANNEL_NAME和PROTOCOL必须为大写， server_address是服务器的TCP / IP主机名。端口值是可选的，是服务器正在侦听的TCP / IP端口号。

如果您未提供端口号，那么WebSphere MQ将使用QM.INI文件中指定的端口号。如果QM.INI文件中未指定任何值，那么WebSphere MQ将使用TCP / IP服务文件中标识的端口号来获取WebSphere MQ服务名称。如果服务文件中的此条目不存在，则使用默认值1414 。重要的是客户端使用的端口号和服务器侦听器程序使用的端口号是相同的。

例如： CHANNEL1/TCP/Sales

单击“ 下一步”继续。Oracle Universal Installer将显示“摘要”屏幕。

Oracle Universal Installer摘要

通过此屏幕，您可以查看此安装的选项和组件的树列表。单击“ 安装”以显示“安装状态”屏幕。

安装状态

“安装状态”屏幕显示安装过程中的状态，以及此安装会话的Oracle Universal Installer日志文件的位置。

请耐心等待Oracle Universal Installer处理软件安装。根据系统上的CPU，CD-ROM驱动器和硬盘驱动器，安装过程可能需要一段时间才能完成。

安装结束

这是安装过程中Oracle Universal Installer的最终屏幕。假设您的安装成功，请单击“ 退出”以退出安装程序。

在基于UNIX的系统上运行root.sh

本主题说明如何在基于UNIX的系统上运行root.sh。

完成安装后，执行以下步骤以运行root.sh脚本：

  1. 以root用户身份登录。
  2. 转到WebSphere MQ网关的$ORACLE_HOME/dg4mq/admin目录。
    提示> cd $ ORACLE_HOME / dg4mq / admin
  3. 运行root.sh脚本。
    提示> ./root.sh

    此脚本使WebSphere MQ网关能够为其运行严格的安全模型。

  4. 退出root帐户。
