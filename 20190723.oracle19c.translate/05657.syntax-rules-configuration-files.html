3.1配置文件语法概述

根据语法规则和标准约定构建Oracle Net Services配置文件。

Oracle Net Services配置文件由包含关键字 - 值对的参数组成。关键字 - 值对由括号括起：

parameter =（ keyword = value ）

有些关键字还有其他关键字 - 值对作为其值：

（ keyword =（ keyword 1 = value 1）（ keyword 2 = value 2））

例如，本地命名配置文件（ tnsnames.ora ）的地址部分可以包含以下行：

（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521））

设置配置文件，以便缩进反映哪个关键字是其他关键字 - 值对的父级或所有者。如果您不选择以这种方式缩进文件，那么您仍必须至少在一个空格中缩进包装行，否则它将被误读为新参数。可以接受以下语法：

（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521））

不接受以下语法：

（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521））

3.2配置文件的语法规则

遵循配置文件的结构，层次结构和字符要求。

以下规则适用于配置文件的语法：

  • 配置文件中任何以包含一个或多个关键字 - 值对的参数开头的关键字必须位于一行的最左侧列中。如果它由一个或多个空格缩进，则将其解释为前一行的延续。

  • 所有字符都必须属于网络字符集。

  • 关键字不区分大小写。但是，值可能区分大小写，具体取决于您的操作系统和协议。

  • 在关键字 - 值对中，等号（ = ）周围的空格是可选的。

  • 关键字的层次结构要求某些关键字始终跟随其他关键字。在层次结构的任何级别，关键字都可以按任何顺序列出。例如，以下条目同样有效：

    （ADDRESS =（PROTOCOL = TCP）（HOST =销售服务器）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp）（PORT = 1521）（HOST =销售服务器））

  • 关键字不能包含空格。

  • 除非带有空格的值包含在双引号（ " ）或单引号（ ' ）中，否则值不得包含空格。

  • 如果关键字 - 值对由单个字组成，或者在等号的两侧包含单词串联，则不需要括号。

  • 连接描述符的最大长度为4 KB。

  • 您可以在行的开头使用数字符号（ # ）来包含注释。数字符号后面的任何内容都被视为注释。

3.3关键字的网络字符集

对关键字值和网络字符集使用允许的字符集。

关键字值的网络字符集由以下字符组成。连接描述符必须由单字节字符组成。

AZ，az 0-9（）<> / \ ,. ：; '“=  -  _ $ + *＃＆！％？@

在此字符集中，保留以下符号：

（）= \“'＃

保留符号用作分隔符，而不是关键字或值的一部分，除非关键字或值具有引号。如果您有一个包含保留符号的值，则使用单引号或双引号括起该值。要在引号括起的值中包含引号，请使用不同的引号。反斜杠（ \ ）用作转义字符。

您可以在连接描述符中使用以下字符，但不能在关键字或值中使用：

  • 空间

  • 标签

  • 回程

  • 新队

3.4监听器和网络服务名称的字符集

了解如何为符合字符集要求的客户端创建侦听器名称和网络服务名称。

侦听器名称和网络服务名称仅限于以下字符集：

[a ... z] [A ...Z] [0 ... 9] _

侦听器名称或网络服务名称中的第一个字符必须是字母数字字符。通常，长度最多为64个字符的名称是可以接受的。此外，数据库服务名称必须与数据库管理员定义的全局数据库名称匹配，该名称由数据库名称和数据库域组成。网络服务名称和全局数据库名称都不区分大小写。
