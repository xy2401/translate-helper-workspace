PGA参数

该 PGA 参数控制网关的APPC接口部分。使用SET指定PGA参数网关初始化参数。例如：

SET pga_parm = value

哪里：

  • pga_parm是pga_parm列表中的PGA参数名称之一

  • value是一个字符串，其内容依赖于pga_parm

PGA_CAPABILITY

PGA交易能力。它控制是否允许通过网关进行更新。以下是有效值：

READ_ONLY或RO ：只读功能。

SINGLE_SITE或SS ：仅限单站点更新。这表明在分布式环境中，只有网关可以执行更新。Oracle事务中不会发生其他数据库更新。

COMMIT_CONFIRM或CC ：提交确认。这表明在分布式环境中，网关和Oracle事务中的其他参与者都可以执行更新。始终在此模式下提交网关，并且不允许其他提交确认站点参与Oracle事务。

默认值为SINGLE_SITE 。

PGA_CONFIRM

传入的APPC CONFIRM请求处理选项。这控制了当从远程事务程序接收到APPC CONFIRM请求时网关执行的操作。仅当对话运行且SYNCLEVEL设置为大于0的值时，此参数才有意义。以下是有效值：

ACCEPT - 使用APPC CONFIRMED响应响应传入的APPC CONFIRM请求。

REJECT - 将传入的APPC CONFIRM请求视为导致会话被解除分配的错误以及要发出的错误消息。

默认值为REJECT 。

PGA_LOG_DB

该网关维护其事务日志的Oracle数据库的Oracle Net服务名称。此参数的长度可以是1到255字符。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM时才需要此参数。

没有默认值。

PGA_LOG_PASS

甲骨文连接到PGA_LOG_DB参数指定的Oracle数据库时网关使用的密码。密码长度可以是1到30字符。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM时才需要此参数。密码可以加密。有关加密密码的详细信息，请参阅网关初始化文件中的密码 。

没有默认值。

PGA_LOG_USER

连接到PGA_LOG_DB参数指定的Oracle数据库时网关要使用的Oracle用户标识。用户ID的长度可以是1到30字符。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM时才需要此参数。

没有默认值。

PGA_RECOVERY_PASS

网关在使用PGA_RECOVERY_TPNAME参数指定的事务分配APPC对话时使用的密码。密码长度可以是1到8字符。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM且PGA_SECURITY_TYPE设置为PROGRAM时， PGA_CAPABILITY需要此参数。密码可以加密。有关加密密码的详细信息，请参阅网关初始化文件中的密码 。

没有默认值。

PGA_RECOVERY_TPNAME

该 OLTP中安装的事务的TP名称，用于commit-confirm FORGET和RECOVERY处理。TP名称长度可以是1到64字符。对于对于z / OS的CICS Transaction Server，TP名称限制为四个字符。对于 IMS / TM，TP名称限制为八个字符。其他OLTP可能对TP名称的长度有其他限制。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM时才需要此参数。

默认值为RECO 。

PGA_RECOVERY_USER

网关在使用PGA_RECOVERY_TPNAME参数指定的事务分配APPC对话时要使用的用户标识。用户ID的长度可以是1到8个字符。仅当PGA_CAPABILITY设置为COMMIT_CONFIRM且PGA_SECURITY_TYPE设置为PROGRAM或SAME时， PGA_CAPABILITY需要此参数。

没有默认值。

PGA_SECURITY_TYPE

APPC会话安全选项。这可以控制在对话分配时将哪些安全参数发送到FMH-5中的OLTP。以下是有效值：

NONE ：不发送安全参数

SAME ：仅发送用户ID

PROGRAM ：发送用户ID和密码

默认值为NONE 。

有关这些选项的更多信息，请参阅安全要求 。

PGA_SIGDANGER

（仅限AIX参数）

采取SIGDANGER从系统接收SIGDANGER信号，表明调页空间不足。以下是有效值：

IGNORE ：忽略信号

默认为IGNORE 。

TRACE_LEVEL

PGA跟踪级别。这将控制写入STDERR跟踪输出（ LOG_DESTINATION参数的目标）。该值必须是0到255之间的整数。

默认值为0 ，表示没有跟踪。

1到255之间的任何值都将打开跟踪。

PGA_CAPABILITY参数注意事项

选择设置时 PGA_CAPABILITY参数，请注意确保根据远程事务程序将要执行的操作使用正确的设置。

该应该始终使用READ_ONLY设置远程事务程序是只读的，即远程事务程序不执行数据库更新时。远程事务程序执行数据库更新时，永远不应使用READ_ONLY 。例如，如果选择了READ_ONLY设置，并且如果网关调用的远程事务程序对外部数据库执行更新，则 Oracle数据库不为这些更新提供任何完整性保护。此外， READ_ONLY模式使网关事务成为可能更新其他几个数据库的分布式事务的一部分。如果网关调用在这种情况下执行更新的远程事务程序，并且如果发生故障，则远程事务程序更新的数据库不会与其他数据库同步。

在远程事务程序对外部数据库执行更新的情况下，存在 PGA_CAPABILITY值的两个选项：

  • SINGLE_SITE

  • COMMIT_CONFIRM

通过允许，每个选项都可以防止数据完整性问题 COMMIT和 ROLLBACK请求转发到远程事务程序并通过通知 Oracle数据库，关于网关的分布式更新和恢复功能。特定选项取决于远程事务程序的设计和功能 OLTP（在线事务处理器）执行的位置。

如果OLTP具有LU6.2 SYNCLEVEL 1或2支持，那么 COMMIT_CONFIRM功能提供了有限的两阶段提交 Oracle数据库和OLTP，没有其他限制 commit-confirm站点（网关或Oracle）可以是分布式事务的一部分。如果无法使用COMMIT_CONFIRM ，那么SINGLE_SITE能力提供了Oracle数据库和更新之间的能力 OLTP，限制条件是只有OLTP可以执行更新，并且Oracle端不会发生更新。

用于更新控制的每个PGA_CAPABILITY选项都对远程事务程序和OLTP施加了特定要求。对于COMMIT_CONFIRM功能，这些要求将在Oracle Database Gateway for APPC用户指南的第5章“实现提交确认”中详细讨论。另请参阅本指南中的配置OLTP以进行提交确认 。对于SINGLE_SITE功能，远程事务程序负责执行所需的任务以响应代表网关从网关接收的COMMIT和ROLLBACK请求。 Oracle数据库。网关使用APPC CONFIRM和SEND_ERR请求分别实现COMMIT和ROLLBACK 。收到CONFIRM ，远程事务程序必须执行COMMIT处理，然后使用APPC CONFIRMED响应响应网关。收到SEND_ERR ，远程事务程序必须执行ROLLBACK处理。

因为分布式事务的能力 Oracle数据库受网关使用的PGA_CAPABILITY选项的影响，希望通过为每个应用程序使用不同的网关实例来分离查询和更新应用程序。一个网关可以被定义PGA_CAPABILITY设置为READ_ONLY和其他人PGA_CAPABILITY设置为SINGLE_SITE或COMMIT_CONFIRM 。

这允许只读事务程序在Oracle数据库的控制下参与分布式事务。例如，来自DB2的数据可以通过READ_ONLY网关由只查询的远程事务程序检索，然后可以在一个Oracle事务中用作Oracle数据库上数据库更新的输入。SINGLE_SITE网关只能用于访问远程事务程序，这些程序对Oracle数据库控件范围之外的外部数据库执行更新。可以从Oracle数据库可访问的任何数据库中读取数据，并且该数据可用于通过网关执行更新。

当需要更新Oracle端和OLTP端的资源时，可以使用COMMIT_CONFIRM网关，前提是OLTP和远程事务程序设置为实现commit-confirm。

设置多个所需的一切网关实例是为每个实例设置以下内容：

  • listener.ora的条目定义网关实例的sid的文件

  • tnsnames.ora的条目定义用于连接到listener.ora定义的网关实例的别名的文件

  • 一个 Oracle数据库中的数据库链接，用于指定在其USING参数中在tnsnames.ora文件中定义的别名

请注意，网关实例可以共享一个公共目录结构并使用相同的可执行文件。

例如，要设置两个网关， PGAI和 PGAU （分别用于查询和更新），需要执行以下步骤：

  1. 在listener.ora为两个sid ， PGAI和PGAU定义条目。

  2. 在tnsnames.ora中定义两个连接到两个新sid ， PGAI和PGAU 。

  3. 在中定义两个数据库链接 Oracle数据库，一个连接到PGAI ，另一个连接到PGAU 。

  4. 最后，创建初始化文件initPGAI.ora和initPGAU.ora 。

    在initPGAI.ora ，设置PGA_CAPABILITY到READ_ONLY ，并在initPGAU.ora ，设置PGA_CAPABILITY到SINGLE_SITE或COMMIT_CONFIRM 。然后，使用PGAI网关进行仅查询事务，并使用PGAU网关进行更新事务。

    可以使用相同的步骤来设置其他网关实例。

PGA_CONFIRM参数注意事项

在决定设置时 PGA_CONFIRM 参数，了解每个设置的效果很重要。首先，请记住，此参数仅影响在SYNCLEVEL 1运行的那些会话。默认设置PGA_CONFIRM = REJECT对大多数应用程序都是正确的。使用此设置，网关生成一个如果从远程事务程序收到CONFIRM请求，则会出错。如果你有一个使用远程交易CONFIRM以验证数据是由网关接收，则必须使用PGA_CONFIRM = ACCEPT ，以允许网关向这些传入响应CONFIRM与请求CONFIRMED响应。您必须知道，当Oracle应用程序发送了COMMIT请求时，网关会向远程事务发送CONFIRM请求。为使COMMIT处理正常工作，必须编写远程事务，以便在从网关收到CONFIRM请求时执行其本地提交处理，并在提交处理成功完成后使用CONFIRMED响应网关。如果在提交处理期间发生错误，然后远程事务必须使用SEND_ERR响应网关以指示提交失败。

使用PGA_CONFIRM = ACCEPT一个特例是IMS / TM版本7。使用IMS / TM版本7提供的“隐含APPC”支持时，在SYNCLEVEL 1上运行的会话的处理方式与在SYNCLEVEL 0上运行的会话的处理方式不同。当会话处于SYNCLEVEL 1时，IMS / TM会在每次APPC SEND后自动生成CONFIRM请求。 在网关侧，如果未指定PGA_CONFIRM = ACCEPT ，则IMS / TM发送的CONFIRM请求将导致网关生成错误。使用PGA_CONFIRM = ACCEPT缓解这一问题，使网关进入的回应CONFIRM与请求CONFIRMED响应。以这种方式运行的唯一限制是IMS提供的隐含APPC支持在从网关收到CONFIRM请求时不通知应用程序。这意味着网关不能使用CONFIRM来实现COMMIT ，从而禁止使用COMMIT和ROLLBACK来控制对话的IMS端的更新。
