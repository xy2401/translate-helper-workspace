安全套接字层如何在Oracle环境中工作：SSL握手

启动安全套接字层上的网络连接时，客户端和服务器在执行身份验证之前执行SSL握手。

握手过程如下：

  1. 客户端和服务器确定要使用的密码套件 。这包括哪些加密算法用于数据传输。

  2. 服务器将其证书发送到客户端，客户端验证服务器的证书是否由受信任的CA签名。此步骤验证服务器的标识。

  3. 同样，如果需要客户端身份验证，客户端会将自己的证书发送到服务器，服务器会验证客户端的证书是否由受信任的CA签名。

  4. 客户端和服务器使用公钥加密来交换密钥信息。基于该信息，每个都生成会话密钥 。客户端和服务器之间的所有后续通信都使用此会话密钥和协商的密码套件进行加密和解密。

验证过程如下：

  1. 在客户端上，用户使用SSL启动到服务器的Oracle Net连接。

  2. SSL执行客户端和服务器之间的握手。

  3. 如果握手成功，则服务器验证用户是否具有访问数据库的适当授权 。

关于公钥加密

传统的私钥或对称密钥加密需要由两方或多方共享的单个秘密密钥来建立安全通信。

该密钥用于加密和解密在各方之间发送的安全消息，要求事先将密钥安全地分发给各方。该方法的问题在于难以安全地传输和存储密钥。

公钥加密通过使用公钥和私钥对以及用于密钥分发的安全方法来提供该问题的解决方案。可免费获得的公钥用于加密只能由相关私钥的持有者解密的消息。私钥与其他安全凭证一起安全地存储在称为钱包的加密容器中。

公钥算法可以保证消息的保密性，但它们不一定保证安全通信，因为它们不验证通信方的身份。要建立安全通信，重要的是验证用于加密消息的公钥实际上是否属于目标接收者。否则，第三方可能会窃听通信并拦截公钥请求，用自己的公钥替换合法密钥（ 中间人攻击）。

为了避免这种攻击，有必要验证公钥的所有者，这是一个称为身份验证的过程。身份验证可以通过证书颁发机构 （CA）完成， 证书颁发机构 （CA）是两个通信方都信任的第三方。

CA颁发包含实体名称，公钥和某些其他安全凭证的公钥证书。此类凭证通常包括CA名称，CA签名和证书生效日期（从日期，到日期）。

CA使用其私钥来加密消息，而公钥用于解密，从而验证消息是否由CA加密。 CA公钥是众所周知的，并且不必在每次访问时都进行身份验证。这种CA公钥存储在钱包中。

证书颁发机构

证书颁发机构（CA）是受信任的第三方，用于验证实体的身份，例如用户，数据库，管理员，客户端和服务器。

当实体请求认证时，CA会验证其身份并授予证书，该证书使用CA的私钥签名。

颁发证书时，不同的CA可能具有不同的标识要求。一些CA可以使用驾驶执照来验证请求者的身份，一些CA可以验证与请求者的指纹的身份，而其他CA可能要求请求者将其证书请求表格公证。

CA发布自己的证书，其中包括其公钥。每个网络实体都有一个可信CA证书列表。在通信之前，网络实体交换证书并检查彼此的证书是否由其各自的可信CA证书列表上的其中一个CA签名。

网络实体可以从相同或不同的CA获取其证书。默认情况下，Oracle Database会在您创建新钱包时自动从VeriSign，RSA，Entrust和GTE CyberTrust安装可信证书。

证书

当实体的公钥由受信任的证书颁发机构（CA）签名时，将创建证书。

证书确保实体的标识信息是正确的，并且公钥实际上属于该实体。

证书包含实体的名称，公钥和到期日期，以及序列号和证书链信息。它还可以包含有关与证书关联的权限的信息。

当网络实体收到证书时，它会验证它是否是受信任的证书 ，即由受信任的证书颁发机构颁发和签名的证书 。证书在到期或撤销之前一直有效。

证书撤销列表

当CA签署将公钥对绑定到用户标识的证书时，证书在指定时间内有效。

但是，某些事件（例如用户名更改或受损私钥）可能会在有效期到期之前使证书无效。发生这种情况时，CA会撤消证书并将其序列号添加到证书吊销列表（CRL）中。当CA不再可接受使用特定公钥来验证其关联的用户身份时，CA会定期发布CRL以警告用户群。

当服务器或客户端在Oracle环境中接收用户证书时，他们可以通过检查证书的到期日期，签名和撤销状态来验证证书。通过根据已发布的CRL验证证书撤销状态来检查证书撤销状态。如果启用了证书吊销状态检查，则服务器将根据此功能的配置方式搜索相应的CRL。服务器按以下顺序在以下位置搜索CRL：

  1. 本地文件系统

  2. Oracle Internet Directory

  3. CRL分发点 ，CRL分发点（CRL DP）X.509版本3中指定的位置，证书颁发时的证书扩展。

注意：

要将CRL与其他Oracle产品一起使用，请参阅特定的产品文档。使用CRL进行证书验证的实现仅在Oracle Database 12c发行版1（12.1）和更高版本的SSL适配器中可用。

安全套接字层和防火墙

Oracle数据库支持两种基于应用程序代理和防火墙的状态包检测。

这些防火墙如下：

  • 基于应用程序代理的防火墙：例如Network Associates Gauntlet或Axent Raptor。

  • 状态包检测防火墙：例如Check Point Firewall-1或Cisco PIX Firewall。

启用SSL时，状态检查防火墙的行为类似于应用程序代理防火墙，因为它们不会解密加密的数据包。

防火墙不检查加密流量。当防火墙遇到发往Intranet服务器上SSL端口的数据时，它会根据其访问规则检查目标IP地址，并让SSL数据包通过允许的SSL端口，拒绝所有其他端口。

使用Oracle Net Firewall Proxy工具包（一些防火墙供应商提供的产品），防火墙应用程序可以为数据库网络流量提供特定支持。如果代理工具包在防火墙中实现，则会执行以下处理：

  • Net Proxy（Oracle Net Firewall Proxy工具包的一个组件）确定路由其流量的位置。

  • 数据库侦听器需要访问证书才能参与SSL握手。侦听器检查SSL数据包并标识目标数据库，返回目标数据库侦听客户端的端口。必须将此端口指定为SSL端口。

  • 客户端在所有后续连接中在此服务器指定的端口上进行通信。

关于安全套接字层密码套件

密码套件是一组用于在网络实体之间交换消息的认证，加密和数据完整性算法。

在安全套接字层握手期间，两个实体协商以查看他们在来回传输消息时将使用哪个密码套件。

安装Oracle数据库时，默认情况下会为您设置表21-1中列出的安全套接字层密码套件，并按列出的顺序进行协商。您可以通过设置SSL_CIPHER_SUITES参数来覆盖默认顺序。确保将括号中的SSL_CIPHER_SUITES参数设置括起来（例如， SSL_CIPHER_SUITES=(ssl_rsa_with_aes_128_cbc_sha256) ）。否则，密码套件设置将无法正确解析。

您可以优先考虑密码套件。当客户端与服务器协商有关使用哪个密码套件时，它遵循您设置的优先级。在确定密码套件的优先级时，请考虑以下事项：

  • 兼容性。 必须将服务器和客户端配置为使用兼容的密码套件才能成功连接。

  • 密码优先级和强度。 优先考虑从最强的密码套件开始到最弱的密码套件，以确保最高级别的安全性。

  • 您要使用的安全级别。

  • 对绩效的影响。

    注意：

    关于Diffie-Hellman匿名身份验证：

    • 如果将服务器设置为使用此密码套件，则还必须在客户端上设置相同的密码套件。否则，连接失败。

    • 如果您使用使用Diffie-Hellman匿名的密码套件，则必须将SSL_CLIENT_AUTHENTICATION参数设置为FALSE 。有关更多信息，请参阅步骤1E：在服务器上设置SSL客户端身份验证（可选） 。

    • 有一个已知的错误，即使使用带有DH_ANON的密码套件，OCI客户端也需要钱包，而DH_ANON不会对客户端进行身份验证。

步骤1G：在服务器和客户端上禁用SSLv3（可选）

SSLv3指的是安全套接字层版本3。

支持安全套接字层版本3（SSLv3）的应用程序很容易受到Padding Oracle On Downgraded Legacy Encryption（POODLE）攻击，即使它们使用最新版本的传输层安全性（TLS）。要防止POODLE攻击，应在服务器和客户端ADD_SSLV3_TO_DEFAULT sqlnet.ora参数设置为FALSE 。ADD_SSLV3_TO_DEFAULT仅在未设置SSL_VERSION参数时适用。 （这意味着您使用的是SSL版本的默认列表）。
  1. 登录数据库服务器或客户端服务器。
  2. 编辑sqlnet.ora参数文件，该文件默认位于$ORACLE_HOME/network/admin目录中，以包含ADD_SSLV3_TO_DEFAULT参数，如下所示：
    ADD_SSLV3_TO_DEFAULT = FALSE

    ADD_SSLV3_TO_DEFAULT默认为FALSE ，如果明确设置了SSL_VERSION ，则SSL_VERSION 。因此，默认情况下，如果SSL_VERSION明确设置SSL_VERSION ，则不允许SSLv3连接。将ADD_SSLV3_TO_DEFAULT设置为TRUE将允许SSLv3连接默认继续工作。

关于在客户端上配置服务器DN匹配和使用带有SSL的TCP / IP

除了验证服务器证书的证书链之外，您还可以通过服务器DN匹配执行额外检查。

您可以将Oracle Net服务名称配置为包括服务器DN匹配，并在客户端上使用TCP / IP和SSL。要完成此操作，必须将服务器的可分辨名称（DN）和TCPS为客户端网络配置文件中的协议，以启用服务器DN匹配和TCP / IP与SSL连接。服务器DN匹配是可选的，但Oracle推荐它，因为它为客户端添加了一层安全性：客户端可以对服务器执行此检查。

您可以配置部分DN匹配或完全DN匹配。将SSL_SERVER_DN_MATCH参数设置为TRUE ，将自动执行部分DN匹配。然后，客户端将检查服务器证书以获取DN信息。完全DN匹配使客户端能够与服务器的完整DN进行匹配。如果要执行完全DN匹配，则必须在SSL_SERVER_CERT_DN参数中指定服务器的DN。

根据您创建和管理主机证书的方式，使用部分或完全DN匹配的功能可以提供更大的灵活性。例如，假设客户端尝试使用hostname=finance.us.example.com连接到服务器。通过部分DN匹配，客户端检查服务器的证书以验证CN=finance是服务器的DN。此外，对于部分DN匹配，仅检查主机名（ finance ），而不是完全限定的域名（ finance.us.example.com ）。对于完全和部分DN匹配，必须将SSL_SERVER_DN_MATCH参数设置为TRUE 。

您必须手动编辑tnsnames.ora客户端网络配置文件以指定服务器的DN和使用SSL协议的TCP / IP。tnsnames.ora文件可以位于客户端或LDAP目录中。如果它位于服务器上，则它通常与listener.ora文件位于同一目录中。tnsnames.ora文件通常位于TNS_ADMIN环境变量指定的TNS_ADMIN中。如果未设置TNS_ADMIN ，则tnsnames.ora位于以下目录位置：

  • （UNIX） $ORACLE_HOME /network/admin/

  • （Windows） ORACLE_BASE \ ORACLE_HOME \network\admin\

关于设置客户端安全套接字层密码套件

密码套件是一组用于在网络实体之间交换消息的认证，加密和数据完整性算法。

在SSL握手期间，两个实体协商以查看他们在来回传输消息时将使用哪个密码套件。

安装Oracle数据库时，默认情况下会为您设置表21-1中列出的SSL密码套件。此表按照两个实体正在协商连接时尝试的顺序列出它们。您可以通过设置SSL_CIPHER_SUITES参数来覆盖默认值。例如，如果使用Oracle Net Manager添加密码套件SSL_RSA_WITH_RC4_128_SHA ，则会忽略默认设置中的所有其他密码套件。

您可以优先考虑密码套件。当客户端与服务器协商有关使用哪个密码套件时，它遵循您设置的优先级。在确定密码套件的优先级时，请考虑以下事项：

  • 您要使用的安全级别。例如，三重DES加密比DES强。

  • 对绩效的影响。例如，三重DES加密比DES慢。有关将SSL硬件加速器与Oracle数据库一起使用的信息，请参阅配置系统以使用硬件安全模块 。

  • 行政要求。为客户端选择的密码套件必须与服务器所需的密码套件兼容。例如，对于Oracle调用接口（OCI）用户，服务器要求客户端对自身进行身份验证。在这种情况下，您不能使用使用Diffie-Hellman匿名身份验证的密码套件，该密码套件不允许交换证书。

您通常优先考虑从最强的密码开始到最弱的密码套件。

表21-1列出了当前支持的安全套接字层密码套件。安装Oracle数据库时，默认情况下会设置这些密码套件。该表还列出了每个密码套件使用的身份验证，加密和数据完整性类型。

注意：

如果sqlnet.ora文件中的SSL_CLIENT_AUTHENTICATION参数设置为true ，则禁用所有使用Diffie-Hellman匿名身份验证的密码套件。否则，连接失败。

CRL检查的工作原理

Oracle数据库根据CRL检查证书吊销状态。

这些CRL位于文件系统目录，Oracle Internet Directory中，或从证书上的CRL分发点（CRL DP）扩展中指定的位置下载。

通常，CRL定义有效期为几天。如果将CRL存储在本地文件系统或目录中，则必须定期更新它们。如果使用CRL分发点 （CRL DP），则每次使用证书时都会下载CRL，因此无需定期刷新CRL。

服务器按列出的顺序在以下位置搜索CRL。当系统找到与证书CA的DN匹配的CRL时，它将停止搜索。

  1. 本地文件系统

    系统首先检查sqlnet.ora文件中的SSL_CRL_FILE参数，然后检查SSL_CRL_PATH参数。如果未指定这两个参数，则系统会检查钱包位置以查找任何CRL。

    注意：如果将CRL存储在本地文件系统上，则必须使用orapki实用程序定期更新它们。有关更多信息，请参阅使用哈希值重命名CRL以进行证书验证 。

  2. Oracle Internet Directory

    如果服务器无法在本地文件系统上找到CRL，并且已在ldap.ora文件中配置了目录连接信息，则服务器将在目录中进行搜索。它使用CA的可分辨名称（DN）和CRL子树的DN搜索CRL子树。

    服务器必须具有正确配置的ldap.ora文件才能在目录中搜索CRL。它无法使用Oracle Internet Directory的域名系统（DNS）发现功能。另请注意，如果将CRL存储在目录中，则必须使用orapki实用程序定期更新它们。有关详细信息，请参阅将CRL上载到Oracle Internet Directory

  3. CRL DP

    如果CA在颁发证书时指定CRL DP X.509版本3（证书扩展）中的位置，则会下载包含该证书的吊销信息的相应CRL。目前，Oracle数据库支持通过LDAP下载CRL。

    请注意以下事项：

    • 出于性能原因，仅检查用户证书。

    • Oracle建议您将CRL存储在目录而不是本地文件系统中。

orapki crl upload -crl /home/user1/wallet/crldir/crl.txt -ldap host1.example.com:3533 -user cn = orcladmin

与使用硬件安全模块关联的错误消息

                     

                        
可能会出现与使用PKCS＃11硬件安全模块相关的错误。

                        

                           

                              
 ORA-43000：PKCS11：找不到库

                              

                                 
原因：系统无法在创建钱包时指定的位置找到PKCS＃11库。只有在创建钱包后移动库时才会发生这种情况。
                                 

                              

                              

                                 
操作：将PKCS＃11库复制回创建钱包时的原始位置。
                                 

                              

                              
 ORA-43001：PKCS11：未找到令牌

                              

                                 
原因：硬件安全模块插槽中不存在用于创建钱包的智能卡。
                                 

                              

                              

                                 
操作：确保硬件安全模块插槽中存在创建钱包时使用的智能卡。
                                 

                              

                              
 ORA-43002：PKCS11：密码错误

                              

                                 
原因：如果在创建钱包时指定了错误的密码，或者在创建钱包后使用Oracle Wallet Manager在钱包中未更新PKCS＃11设备密码，则会发生这种情况。
                                 

                              

                              

                                 
操作：根据原因，执行以下操作之一：

                                 
如果您在创建钱包期间看到此错误，请检查以确保您拥有正确的密码并重新输入。

                                 
如果在创建钱包后密码已更改，则使用Oracle Wallet Manager打开钱包并输入新密码。
