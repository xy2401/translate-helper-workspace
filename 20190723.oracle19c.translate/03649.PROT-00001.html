141 PROT-00001至PROT-00607

PROT-00001：无法初始化ocrconfig

原因：无法初始化并启动ocrconfig。

操作：检查日志文件以获取有关失败的详细信息。

PROT-00002：导出文件' string '已存在

原因：请求的导出文件存在于给定的路径中。

操作：更改导出文件的位置或删除旧的导出文件。

PROT-00003：无法创建导出文件' string '

原因：无法创建或打开用于导出群集注册表数据的文件。

操作：检查所请求路径的访问权限或磁盘空间。

PROT-00004：无法从群集注册表中检索数据

原因：无法从群集注册表中检索必要的信息。

操作：从日志文件中检查群集注册表的可用性以及故障的详细信息。

PROT-00005：Clusterware运行不正常

原因：无法注册集群件以访问集群注册表。

操作：检查计算机上运行的集群件的状态。

PROT-00006：无法访问指定的Oracle Cluster Registry导入文件“ string ”。

原因：尝试从指示的导入文件导入Oracle Cluster Registry内容失败，因为导入文件不存在，未正确安装或用户无权打开导入文件。

操作：确保导入文件存在，安装正确，并且其权限正确并重试导入命令。如果文件位置是ASM磁盘组，请检查ASM警报日志文件中的相关条目以获取更多详细信息。

PROT-00007：无法打开指定的导入文件' string '

原因：无法访问指定的导入文件。

操作：检查导入文件的访问权限。

PROT-00008：无法将数据从指定文件导入群集注册表

原因：无法从导入文件中读取数据并导入到群集注册表。

操作：从日志文件中检查群集注册表的可用性以及故障的详细信息。

PROT-00010：无法打开指定的源群集注册表

原因：无法访问指定的源群集注册表。

操作：检查源群集注册表的访问权限和位置以及日志文件以获取更多详细信息。

PROT-00011：无法打开指定的目标群集注册表

原因：无法访问指定的目标群集注册表。

操作：检查目标群集注册表的访问权限和位置。还要检查日志文件以获取更多详细信息。

PROT-00012：要升级的Oracle Cluster Registry具有不受支持的版本

原因：要升级的Oracle Cluster Registry具有不受支持的版本。

操作：将 Oracle Cluster Registry升级到支持的版本。

PROT-00013：Oracle Cluster Registry是当前版本

原因： Oracle Cluster Registry处于当前版本。

行动：无需采取任何行动。

PROT-00014：要降级的Oracle Cluster Registry有早期版本

原因：要降级的Oracle Cluster Registry有早期版本。

行动：无需采取任何行动。

PROT-00015：未实现的功能

原因：当前版本中尚未实现请求的功能。

操作：在下一个版本中再次尝试该功能。

PROT-00016：内部错误

原因：这是ocrconfig的一般内部错误。

操作：联系Oracle支持服务。

PROT-00017：致命错误

原因：这是ocrconfig的一般致命错误。

操作：联系Oracle支持服务。

PROT-00018：无法打开指定的备份文件' string '

原因：文件不存在或用户没有打开文件的权限。

操作：确保文件存在且其权限正确。还要检查日志文件以获取更多详细信息。

PROT-00019：群集就绪服务正在运行时无法继续

原因：群集就绪服务正在运行。

操作：使用'crsctl stop crs'命令关闭本地节点上的Cluster Ready Service。

PROT-00020：没有足够的权限继续进行。需要特权用户

原因：用户不是特权用户。

操作：使用特权用户再次运行该命令。

PROT-00021：参数无效

原因：输入参数无效。

操作：检查日志文件以获取更多信息。

PROT-00022：存储空间太小

原因：指定的存储空间太小，无法镜像OCR内容。

操作：执行'ocrcheck'命令以检索总大小，并确保要添加的设备大小足以供群集注册表使用。

PROT-00023：无法备份Oracle Cluster Registry

原因：尝试制作Oracle Cluster Registry（OCR）的备份副本失败。此错误可能伴随着识别特定问题的其他消息。

操作：检查Oracle Cluster Registry的可用性以及将存储手动备份的磁盘组上的可用空间。如有必要，使用'ocrconfig -backuploc'指定新的备份位置。

PROT-00024：Oracle Cluster Registry的自动备份不可用

原因：尚未创建Oracle Cluster Registry的自动备份。它们每四个小时创建一次。

行动：无需采取任何行动。

PROT-00025：Oracle Cluster Registry的手动备份不可用

原因：尚未创建Oracle Cluster Registry的手动备份。

操作：可以使用'ocrconfig -manualbackup'命令创建手动备份。

PROT-00026：从本地副本检索Oracle Cluster Registry备份位置

原因：无法访问Oracle Cluster Registry文件。有关群集注册表备份的信息也在Oracle Local Registry中维护。显示的信息已从Oracle Local Registry中检索到。此信息可能已过时。

操作：从群集中的另一个节点运行“ocrconfig -showbackup”命令。将此节点上'ocrconfig -showbackup'的输出与来自群集中其他节点的相同命令的输出进行比较，以确定最新的备份副本。

PROT-00027：无法配置其他Oracle Cluster Registry位置，因为已配置最大Oracle Cluster Registry位置数（5）

原因：已配置最大Oracle Cluster Registry位置数（5）。

操作:(可选）使用'ocrcheck -config'命令验证配置。通过执行'ocrconfig -delete existing location '命令删除Oracle Cluster Registry设备，然后重试该操作。

PROT-00028：无法删除或替换唯一配置的Oracle Cluster Registry位置。

原因：当只有1个已配置的Oracle Cluster Registry位置时，执行了删除或替换操作。只有在至少有2个已配置的Oracle Cluster Registry位置时，才能执行上面列出的操作。

操作：执行'ocrconfig -add new location '以添加新的Oracle Cluster Registry位置，然后添加'ocrconfig -delete existing location '命令。

PROT-00029：已配置Oracle Cluster Registry位置

原因：要添加的位置与配置的Oracle Cluster Registry位置相同。

操作：使用'ocrcheck -config'命令验证配置。确保尚未配置要添加的位置。

PROT-00030：要添加的Oracle Cluster Registry位置不可用。

原因：要添加的Oracle Cluster Registry位置不存在，未正确安装，没有所需权限，或者没有足够的磁盘空间。此错误可能伴随着识别特定问题的其他消息。

操作：确保该位置存在，它已正确安装并且对群集中的所有节点可见，其权限正确，并且具有足够的磁盘空间。如果尝试添加ASM Oracle Cluster Registry位置，请检查ASM警报日志文件中的相关条目以获取更多详细信息。

PROT-00031：要添加的Oracle Cluster Registry位置是ASM磁盘

原因：要添加的Oracle Cluster Registry位置已由ASM使用。

操作：指定当前未使用的Oracle Cluster Registry位置。使用'crsctl query css votedisk'命令获取CSS使用的投票文件列表。使用'asmcmd lsdsk --member'命令获取ASM使用的磁盘列表。指定不在任一列表中的位置。

PROT-00032：要添加的Oracle Cluster Registry位置是CSS投票文件

原因：要添加的Oracle Cluster Registry位置已配置为CSS投票文件。

操作：指定当前未使用的Oracle Cluster Registry位置。使用'crsctl query css votedisk'命令获取CSS使用的投票文件列表。使用'asmcmd lsdsk --member'命令获取ASM使用的磁盘列表。指定不在任一列表中的位置。

PROT-00033：本地节点上的Oracle高可用性服务未运行

原因：本地节点上的Oracle高可用性服务未运行

操作：运行'crsctl start crs'命令以在本地节点上启动Oracle高可用性服务。

PROT-00034：未配置要删除的Oracle Cluster Registry位置。

原因：要删除的位置尚未配置为Oracle Cluster Registry位置。

操作：运行'ocrcheck -config'命令以获取已配置的Oracle Cluster Registry位置列表。确保已配置要删除的位置。

PROT-00035：无法访问已配置的OCR位置。

原因：配置的Oracle Cluster Registry（OCR）位置不存在，未正确装入，没有所需权限或磁盘空间不足。此错误可能伴随着识别特定问题的其他消息。

操作：确保位置存在，它们已正确安装并且对群集中的所有节点可见，其权限是正确的，并且它们具有足够的磁盘空间。如果使用Oracle自动存储管理（Oracle ASM）OCR位置，请检查Oracle ASM警报日志文件中的条目以获取更多详细信息。

PROT-00036：无法添加重复或以前使用的Oracle Cluster Registry位置string

原因：要添加的Oracle Cluster Registry位置具有与当前配置位置相同的内容。这可能是由于要添加的位置是具有不同路径或先前使用的当前配置位置之一的副本。

操作：确保当前未使用其他路径配置要添加的位置。如果该位置先前已配置为Oracle Cluster Registry位置，请确保在再次添加内容之前删除内容。

PROT-00037：无法删除Oracle Cluster Registry位置string因为OCR必须可用

原因：当可用的最低配置Oracle Cluster Registry位置时，请求执行删除操作。只有在大多数已配置的Oracle Cluster Registry位置可访问且联机时，才能执行上面列出的操作。

操作：运行'ocrcheck'命令以获取已配置的Oracle Cluster Registry位置的当前状态。执行'ocrconfig -add new location '以添加新的Oracle Cluster Registry位置，后跟'ocrconfig -delete existing location '命令。

PROT-00038：无法添加Oracle Cluster Registry位置，因为另一个已配置的位置处于脱机状态。

原因：当至少有一个配置的Oracle Cluster Registry位置无法访问或联机时，请求添加或替换操作。只有在删除或替换无法访问的Oracle Cluster Registry位置时，才能执行上面列出的操作。

操作：运行'ocrcheck'命令以获取已配置的Oracle Cluster Registry位置的当前状态。执行'ocrconfig -delete offline location '命令删除不可访问的Oracle Cluster Registry位置，后跟'ocrconfig -add new location '命令。

PROT-00039：指定的文件“ string ”格式无效

原因：指定的文件不是有效的Oracle Cluster Registry导出/导入文件。

操作：指定使用“ocrconfig -export”命令创建的有效Oracle Cluster Registry文件。

PROT-00040：指定的文件' string '格式无效

原因：指定的文件不是有效的Oracle Cluster Registry备份文件。

操作：使用'ocrconfig -showbackup'命令列出所有可用备份。指定上述命令列出的有效Oracle Cluster Registry备份文件。

PROT-00041：无法访问指定的备份位置“ string ”

原因：备份位置不存在或未授予所需权限。

操作：确保该位置存在并授予所需的文件创建权限。

PROT-00042：指定的位置“ string ”指定Oracle Cluster Registry备份文件的无效存储类型。

原因：命令'ocrconfig -backuploc'指定了一个备份位置，该位置为Oracle Cluster Registry备份文件指定了无效的存储类型。

操作：检查Oracle Cluster Registry备份位置的存储类型。

PROT-00043：不支持从Leaf节点管理Oracle Cluster Registry。

原因：尝试使用Leaf节点中的指定选项执行'ocrconfig'命令，该节点不受支持。

操作：从Hub节点执行'ocrconfig'命令。

PROT-00044：无法将Oracle Cluster Registry备份位置更改为ASM磁盘组。

原因：在ASM上未配置Oracle Cluster Registry时，尝试将Oracle Cluster Registry备份位置更改为ASM磁盘组。

操作：在将Oracle Cluster Registry备份位置更改为ASM磁盘组之前，将Oracle Cluster Registry迁移到ASM。

PROT-00045：指定的Oracle Cluster Registry备份位置string不可访问。

原因： Oracle Cluster Registry备份位置不存在，未正确安装或没有所需权限。

操作：确保备份位置存在，正确安装，以及其权限是否正确。检查ASM警报日志文件中的条目以获取更多详细信息。

PROT-00046：在本地节点上添加位置' string '时无法修复Oracle Cluster Registry配置

原因：要添加的Oracle Cluster Registry（OCR）位置不存在，未正确装入，没有所需权限，或磁盘组中的OCR文件不存在。此错误可能伴随着识别特定问题的其他消息。

操作：确保该位置存在且具有适当的权限。如果该位置是ASM磁盘组，则磁盘组中的OCR文件必须已存在，磁盘组必须正确安装;还要检查ASM警报日志文件中的条目以获取更多详细信息。

PROT-00048：无法删除Oracle Cluster Registry位置string因为OCR备份位置是ASM磁盘组。

原因：尝试删除唯一的ASM Oracle Cluster Registry（OCR）位置。

操作：确保至少有一个ASM Oracle Cluster Registry位置。

PROT-00049：Oracle Cluster Registry（OCR）完整性检查失败。

原因：一个或多个已配置的Oracle Cluster Registry位置的完整性检查失败。

操作：使用“ocrcheck”命令验证已配置的Oracle Cluster Registry位置的块完整性。使用命令'ocrconfig -delete corrupt location '删除corrupt location 。如果所有已配置的Oracle Cluster Registry位置都已损坏，请使用“ocrconfig -showbackup”命令检查备份的可用性，并还原兼容的备份。如果无法解决问题，请与Oracle支持服务联系。

PROT-00050：OCR仅限本地。仅限本地OCR不支持“-repair -add”和“-repair -delete”选项。

原因：尝试将“-repair -add”或“-repair -delete”选项与仅支持本地的Oracle Cluster Registry一起使用。

行动：没有

PROT-00051：为Oracle Cluster Registry位置指定的保留文件或设备名称

原因：发出了命令'ocrconfig -add'，'ocrconfig -repair'或'ocrconfig -replace'之一，指定了保留文件或设备名称。名称week.ocr，day.ocr，backup00.ocr，backup01.ocr和backup02.ocr是为Oracle保留的。

操作：选择不是保留名称之一的OCR位置。

PROT-00052：无法检索Oracle Cluster Registry的备份位置

原因：尝试显示Oracle Cluster Registry的备份位置失败，因为无法访问Oracle Cluster Registry。

操作：检查Oracle Cluster Registry的可访问性。

PROT-00053：为'ocrconfig -repair'，'ocrconfig -add'或'ocrconfig -replace'命令指定的文件名[ string ]指定Oracle Cluster Registry的无效存储类型。

原因：其中一个命令'ocrconfig -add'，'ocrconfig -repair'或'ocrconfig -replace'是使用文件名发出的，该文件名为Oracle Cluster Registry指定了无效的存储类型。

操作：检查要添加，修复或更换的设备的存储类型。

PROT-00054：无法将Oracle Cluster Registry备份文件复制到指定的目标“ string ”

原因：尝试将Oracle Cluster Registry备份文件复制到指定目标失败，因为它无法由用户写入，或者没有足够的空间，或者目标不是文件名。

操作：确保目标可由用户写入，并且文件有足够的可用空间，并且目标是文件名。

PROT-00055：为'ocrconfig -manualbackup -filename'命令指定的文件名[ string ]无效。

原因：尝试为Oracle Cluster Registry配置手动备份文件失败，因为指定的文件名参数未指定完整的路径名。

操作：重新发出命令，指定备份文件的完整路径名。

PROT-00056：为'ocrconfig -copy'命令指定的文件名[ string ]无效。

原因：尝试复制Oracle Cluster Registry文件失败，因为指定的目标文件名参数未指定完整的路径名。

操作：重新发出命令，指定目标文件的完整路径名。

PROT-00057：'ocrconfig -manualbackup -filename'命令中指定的备份文件已存在[ string ]。

原因：请求的备份文件已存在于给定路径中。

操作：更改“ocrconfig -manualbackup -filename”命令的备份文件位置，或删除旧的备份文件。

PROT-00058：成功将Oracle Cluster Registry内容导出到文件' string '

原因： Oracle Cluster Registry内容已成功导出到用户请求的指定文件。

行动：没有

PROT-00059：无法显示Oracle Cluster Registry的手动备份。

原因：尝试显示OCR手动备份位置失败，因为正在进行手动备份。

操作：在正在进行的备份完成后，使用“ocrconfig -showbackup manual”命令重试该操作以显示手动备份位置。

PROT-00060：Oracle Cluster Registry备份被复制到' string '。

原因： Oracle Cluster Registry备份内容已成功复制到指定的文件。

行动：没有

PROT-00061：创建的Oracle Cluster Registry备份文件是“ string ”。

原因：无，这是一条信息性消息。

行动：没有

PROT-00062：无法删除Oracle Cluster Registry备份文件“ string ”。

原因：可能无法从发出命令的节点访问Oracle Cluster Registry备份，或者用户可能没有相应的权限来删除备份文件。

操作：检查随附的错误并采取相应措施。

PROT-00607：检测到多个OCR存储类型。

原因：检测到多个Oracle Cluster Registry（OCR）存储类型。不支持同时在不同存储类型上存储OCR。

操作：使用相同的存储类型配置OCR。
