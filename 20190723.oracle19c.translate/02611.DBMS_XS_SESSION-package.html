DBMS_XS_SESSIONS包

DBMS_XS_SESSIONS包管理应用程序会话。

本节包括以下主题：

  • 安全模型

  • 常量

  • 对象类型，构造函数，同义词和授权

  • DBMS_XS_SESSIONS子程序摘要

安全模型

DBMS_XS_SESSIONS包在SYS架构中创建。执行包的权限授予PUBLIC 。执行用户必须具有特定操作的适当权限。

常量

以下常量定义传递给命名空间事件处理函数的操作码：

attribute_first_read_operation CONSTANT PLS_INTEGER：= 1; modify_attribute_operation CONSTANT PLS_INTEGER：= 2;

以下常量表示用于标识具有事件处理函数的命名空间中特定属性感兴趣的事件的位值：

attribute_first_read_event CONSTANT PLS_INTEGER：= 1; modify_attribute_event CONSTANT PLS_INTEGER：= 2;

以下常量定义可由命名空间事件处理函数返回的返回码：

event_handling_succeeded CONSTANT PLS_INTEGER：= 0; event_handling_failed CONSTANT PLS_INTEGER：= 1;

以下常量用作ADD_GLOBAL_CALLBACK ， DELETE_GLOBAL_CALLBACK和ENABLE_GLOBAL_CALLBACK过程的输入：

create_session_event CONSTANT PLS_INTEGER：= 1; attach_session_event CONSTANT PLS_INTEGER：= 2; guest_to_user_event CONSTANT PLS_INTEGER：= 3; proxy_to_user_event CONSTANT PLS_INTEGER：= 4; revert_to_user_event CONSTANT PLS_INTEGER：= 5; enable_role_event CONSTANT PLS_INTEGER：= 6; disable_role_event CONSTANT PLS_INTEGER：= 7; enable_dynamic_role_event CONSTANT PLS_INTEGER：= 8; disable_dynamic_role_event CONSTANT PLS_INTEGER：= 9; detach_session_event CONSTANT PLS_INTEGER：= 10; terminate_session_event CONSTANT PLS_INTEGER：= 11; direct_login_event CONSTANT PLS_INTEGER：= 12; direct_logoff_event CONSTANT PLS_INTEGER：= 13;

本节介绍以下DBMS_XS_SESSIONS子程序：

CREATE_SESSION过程

CREATE_SESSION过程为指定的用户名创建新的应用程序会话。它返回一个会话标识符，您可以使用该标识符在将来的调用中引用该会话。

可以使用常规应用程序用户或外部应用程序用户创建会话。可以在受信任模式或安全模式下创建会话。在受信任模式下，绕过数据安全检查;在安全模式下，它们是强制执行的。

不支持在受信任模式下常规会话的组合。支持其他组合，安全模式下的常规会话，受信任模式下的外部会话或安全模式下的外部会话。

namespaces参数是要创建的三元组命名空间，要创建的属性以及要设置的属性值的列表。这是一个可选参数。默认值为NULL 。 XS$GLOBAL_VAR和XS$SESSION命名空间及其属性始终可用于会话。

此功能不会将当前传统会话附加到新创建的应用程序会话。使用ATTACH_SESSION过程执行此任务。

执行该过程的用户必须具有username参数指定的应用程序用户的CREATE_SESSION应用程序特权。您还可以指定创建会话时要创建的命名空间列表。如果在创建会话期间指定名称空间，则需要在调用名称空间中为调用者授予应用程序权限MODIFY_NAMESPACE或MODIFY_ATTRIBUTE ，或者授予调用者ADMIN_NAMESPACE系统特权。

句法

CREATE_SESSION（用户名IN VARCHAR2，sessionid OUT NOCOPY RAW，is_external IN BOOLEAN DEFAULT FALSE，is_trusted IN BOOLEAN DEFAULT FALSE，名称空间IN DBMS_XS_NSATTRLIST DEFAULT NULL，cookie IN VARCHAR2 DEFAULT NULL）;

用户名

要为其创建应用程序会话的常规应用程序用户或外部应用程序用户的名称。

要查找当前会话的用户名和应用程序角色的列表，请查询DBA_XS_USERS数据字典视图。要查找所有应用程序用户和角色，请按如下方式查询DBA_XS_PRINCIPALS数据字典视图：

用户：

从DBA_XS_USERS中选择名称;

角色：

从DBA_XS_ROLES中选择名称;从DBA_XS_DYNAMIC_ROLES中选择名称;

会话ID

新创建的应用程序会话的会话ID。您可以使用以下方法之一获取会话ID：

  • SELECT XS_SYS_CONTEXT（'XS $ SESSION'，'SESSION_ID'）FROM DUAL;

  • 使用DBMS_XS_SESSIONS.GET_ATTRIBUTE过程。

is_external

指定是否将会话创建为外部主体会话。这是一个可选参数。默认值为FALSE ，表示要创建常规会话。NULL值表示FALSE 。

is_trusted

指定是以受信任模式还是以安全模式创建会话。在受信任模式下，绕过数据安全检查;在安全模式下，它们是强制执行的。这是一个可选参数。默认值为FALSE ，表示安全模式。NULL值表示FALSE 。

命名空间

名称，属性和属性值三元组的列表。如果会话无法访问命名空间，或者不存在此类命名空间模板，则会引发错误。

曲奇饼

指定要为会话设置的服务器cookie。这是一个可选参数。默认值为NULL 。 cookie的最大允许长度为1024字节。

例子

DECLARE nsList DBMS_XS_NSATTRLIST; sessionid RAW（16）; BEGIN nsList：= DBMS_XS_NSATTRLIST（DBMS_XS_NSATTR（'ns1'），DBMS_XS_NSATTR（'ns2'））; SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwuser1'，sessionid，FALSE，FALSE，nsList）;结束;

ATTACH_SESSION程序

ATTACH_SESSION过程将当前传统数据库会话附加到由会话ID（ session_id ）标识的应用程序会话。附加会话允许（直接或间接）为创建会话的应用程序用户授予的角色以及在此会话的最后一次分离之前启用的会话范围动态应用程序角色。如果执行ATTACH_SESSION使用可选的参数动态应用的角色列表enable_dynamic_roles ，所提供的动态应用角色为会话启用。要禁用动态角色列表，请使用可选参数disable_dynamic_roles指定列表。

您可以在附加操作期间指定三元组值列表（名称空间，属性，属性值）。然后创建名称空间和属性并设置属性值。这是会话中存在的任何名称空间和属性的补充。

要执行此过程，传统会话用户必须具有ATTACH_SESSION应用程序特权。如果指定名称空间，则需要在用户名或ADMIN_NAMESPACE系统权限上为用户授予应用程序权限MODIFY_NAMESPACE或MODIFY_ATTRIBUTE 。

如果从显式连接的会话（使用ATTACH_SESSION过程附加的会话或Java中的attachSession()方法）调用，则允许使用SQL * Plus PASSWORD命令更改自密码，前提是该会话具有ALTER_USER权限且用户名为提供了PASSWORD命令。

句法

ATTACH_SESSION（sessionid IN RAW，enable_dynamic_roles IN XS $ NAME_LIST DEFAULT NULL，disable_dynamic_roles IN XS $ NAME_LIST DEFAULT NULL，external_roles IN XS $ NAME_LIST DEFAULT NULL，authentication_time IN TIMESTAMP WITH TIME ZONE DEFAULT NULL，名称空间IN DBMS_XS_NSATTRLIST DEFAULT NULL）;

参数

参数 描述

会话ID

应用程序会话的会话ID。您可以使用以下方法之一获取会话ID：

  • SELECT XS_SYS_CONTEXT（'XS $ SESSION'，'SESSION_ID'）FROM DUAL;

  • 使用DBMS_XS_SESSIONS.GET_ATTRIBUTE过程。

enable_dynamic_roles

要授予在应用程序会话中启用的动态角色的列表。这是一个可选参数。如果指定的任何动态角色不存在，则附加会话将失败。如果会话是外部主体会话，则可以指定外部角色列表以启用。在分离之前，这些角色将保持启用状态，默认情况下不会在下次附加时启用。

要查找当前会话的应用程序角色列表，请查询DBA_XS_SESSION_ROLES数据字典视图。要查找所有动态应用程序角色的列表，请按如下方式查询DBA_XS_PRINCIPALS数据字典视图：

SELECT NAME，TYPE FROM DBA_XS_PRINCIPALS;

disable_dynamic_roles

要从会话中禁用的动态角色列表。这是一个可选参数。

external_roles

如果会话是外部主体会话，则为外部角色的列表。这是一个可选参数。这些外部角色在分离操作之前保持启用状态，默认情况下在下次附加时不会再次启用。

authentication_time

更新的会话身份验证时间。这是一个可选参数。必须按以下格式指定时间：

YYYY-MM-DD HH：MI：SS.FF TZR

命名空间

名称，属性和属性值三元组的列表。如果会话无法访问命名空间，或者不存在此类命名空间模板，则会引发错误。

例子

DECLARE nsList DBMS_XS_NSATTRLIST; sessionid RAW（16）; BEGIN nsList：= DBMS_XS_NSATTRLIST（DBMS_XS_NSATTR（'ns1'），DBMS_XS_NSATTR（'ns2'））; SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwuser1'，sessionid）; SYS.DBMS_XS_SESSIONS.ATTACH_SESSION（sessionid，NULL，NULL，NULL，NULL，nsList）;结束;

ASSIGN_USER程序

ASSIGN_USER过程将指定的应用程序用户分配给当前连接的匿名应用程序会话。

此操作后将保留当前会话中启用的角色。可选参数enable_dynamic_roles和disable_dynamic_roles指定要启用或禁用的其他动态角色列表。如果分配的用户是外部用户，则可以指定要启用的外部角色列表。

您可以在分配操作期间指定三元组值列表（命名空间，属性，属性值）。然后在会话和属性值集中创建名称空间和属性。这是对会话中已存在的任何名称空间和属性的补充。

要执行此过程，调度程序或连接用户必须具有ASSIGN_USER应用程序特权。如果指定名称空间，则需要在用户名或ADMIN_NAMESPACE系统权限上为用户授予应用程序权限MODIFY_NAMESPACE或MODIFY_ATTRIBUTE 。

句法

DBMS_XS_SESSIONS.ASSIGN_USER（用户名IN VARCHAR2，is_external IN BOOLEAN DEFAULT FALSE，enable_dynamic_roles IN XS $ NAME_LIST DEFAULT NULL，disable_dynamic_roles IN XS $ NAME_LIST DEFAULT NULL，external_roles IN XS $ NAME_LIST DEFAULT NULL，authentication_time IN TIMESTAMP WITH TIME ZONE DEFAULT NULL，名称空间IN DBMS_XS_NSATTRLIST DEFAULT NULL）;

SWITCH_USER程序

SWITCH_USER过程在当前连接的会话中切换应用程序用户。在执行切换操作之前，当前应用程序用户必须是目标应用程序用户的代理用户，方法是使用XS_PRINCIPAL.ADD_PROXY_USER PL / SQL API获取另一个应用程序用户权限的代理。在会话中启用目标用户的过滤应用程序角色列表。

您可以保留会话的当前应用程序名称空间或将其丢弃。您还可以指定要创建的命名空间列表以及要在切换后设置的属性值。如果指定名称空间，则需要在用户名或ADMIN_NAMESPACE系统权限上为用户授予应用程序权限MODIFY_NAMESPACE或MODIFY_ATTRIBUTE 。

句法

SWITCH_USER（用户名IN VARCHAR2，keep_state IN BOOLEAN DEFAULT FALSE，名称空间IN DBMS_XS_NSATTRLIST DEFAULT NULL）;

参数

参数 描述

用户名

要切换到其安全上下文的用户的用户名。

要查找现有应用程序用户的列表，请按如下方式查询DBA_XS_USERS数据字典视图：

从DBA_XS_USERS中选择名称;

KEEP_STATE

控制是否保留应用程序名称空间。

可能的值是：

  • TRUE ：将所有其他会话状态设置为保持不变。

  • FALSE ：清除会话中的先前状态。默认值。

命名空间

名称，属性和属性值三元组的列表。如果会话无法访问命名空间，或者不存在此类命名空间模板，则会引发错误。

例子

DECLARE nsList：= DBMS_XS_NSATTRLIST（DBMS_XS_NSATTR（'ns1'），DBMS_XS_NSATTR（'ns2'））; sessionid RAW（16）; BEGIN SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwuser1'，sessionid）; SYS.DBMS_XS_SESSIONS.ATTACH_SESSION（的sessionid）; SYS.DBMS_XS_SESSIONS.SWITCH_USER（username =>'lwuser2'， keep_state => TRUE， namespaces => nsList）;结束;

CREATE_NAMESPACE过程

CREATE_NAMESPACE过程在当前连接的应用程序会话中创建新的命名空间。与命名空间对应的命名空间模板必须存在于系统中，否则此操作会引发错误。执行此操作后，会话中可以使用命名空间及其在模板中创建的属性。

调用用户必须具有MODIFY_NAMESPACE应用程序权限。

句法

CREATE_NAMESPACE（名称空间IN VARCHAR2）;

参数

参数 描述

命名空间

要创建的命名空间的名称。必须存在具有此名称的现有命名空间模板文档。区分大小写的字符串的最大大小为128个字符。

要查找当前会话的现有命名空间列表，请在连接后查询V$XS_SESSION_NS_ATTRIBUTES数据字典视图。您可以查询DBA_XS_SESSION_NS_ATTRIBUTES数据字典视图，以查找所有应用程序会话中的所有命名空间。

您可以查询DBA_XS_NS_TEMPLATES和DBA_XS_NS_TEMPLATE_ATTRIBUTES数据字典视图，以获取命名空间模板和属性的列表。

例子

BEGIN SYS.DBMS_XS_SESSIONS.CREATE_NAMESPACE（'J_NS1'）;结束;

CREATE_ATTRIBUTE过程

CREATE_ATTRIBUTE过程在当前连接的应用程序会话中的指定命名空间中创建新的自定义属性。如果会话中尚未提供名称空间，或者不存在此类名称空间模板，则会引发错误。

调用用户需要被授予MODIFY_ATTRIBUTE应用程序权限。

句法

PROCEDURE create_attribute（名称空间IN VARCHAR2，属性IN VARCHAR2，值IN VARCHAR2 DEFAULT NULL，eventreg IN PLS_INTEGER DEFAULT NULL）;

参数

参数 描述

命名空间

创建属性的名称空间。如果会话中不存在命名空间，则会引发错误。区分大小写的字符串的最大大小为128个字符。

属性

要创建的属性的名称。区分大小写的字符串的最大大小为4000个字符。

值

属性的默认值。区分大小写的字符串的最大大小为4000个字符。

eventreg

为该属性执行处理程序的事件。这是一个可选参数。此参数可以采用以下值：

  • DBMS_XS_SESSIONS.attribute_first_read_event

    只要收到属性获取请求并且尚未设置属性的值，就会调用处理函数。仅当默认值设置为NULL才能注册此事件。此值对应于XS_NAMESPACE包或Admin API中的FIRSTREAD_EVENT常量。

  • DBMS_XS_SESSIONS .modify_attribute_event ：

    只要收到属性集请求，就会调用处理函数。此值对应于XS_NAMESPACE包或Admin API中的UPDATE_EVENT常量。

如果为第一个读取事件注册了该属性，则在返回该值之前，如果该属性未初始化，则执行该处理程序。如果注册了update事件，则只要修改了该属性，就会调用该处理程序。仅当命名空间具有事件处理程序时才能注册事件，否则将引发错误。

例子

BEGIN SYS.DBMS_XS_SESSIONS.CREATE_ATTRIBUTE（'NS1'，'NS1CUSTOM'，'NS1CUSTOMDEFAULT'）;结束; -  firstRead事件集的示例BEGIN SYS.DBMS_XS_SESSIONS.create_Attribute（'ns1'，'attr4'，NULL，DBMS_XS_SESSIONS.attribute_first_read_event）;结束;

SET_ATTRIBUTE过程

SET_ATTRIBUTE过程为与当前连接的会话关联的命名空间中的指定属性设置新值。如果为属性设置了update事件，则调用handler函数。如果命名空间不存在或被删除，则会引发错误。如果没有与存在的命名空间对应的模板，则会引发错误。

调用用户需要被授予MODIFY_ATTRIBUTE应用程序权限。

句法

SET_ATTRIBUTE（名称空间IN VARCHAR2，属性IN VARCHAR2，值IN VARCHAR2）;

参数

参数 描述

命名空间

与属性关联的命名空间的名称。区分大小写的字符串的最大大小为128个字符。

要查找当前会话的现有命名空间列表，请在连接后查询V$XS_SESSION_NS_ATTRIBUTES数据字典视图。您可以查询DBA_XS_SESSION_NS_ATTRIBUTES数据字典视图，以查找所有应用程序会话中的所有命名空间。

您可以查询DBA_XS_NS_TEMPLATES和DBA_XS_NS_TEMPLATE_ATTRIBUTES数据字典视图，以获取命名空间模板和属性的列表。

属性

现有命名空间中现有属性的名称。

要查找现有命名空间属性的列表，请查询V $ XS_SESSION_NS_ATTRIBUTES数据字典视图。

值

属性的新值。区分大小写的字符串的最大大小为4000个字符。

要查找与该属性关联的现有值的列表，请查询V $ XS_SESSION_NS_ATTRIBUTES数据字典视图。

例子

BEGIN SYS.DBMS_XS_SESSIONS.SET_ATTRIBUTE（'J_NS'，'JohnNSAttr1'，'John bio'）;结束;

GET_ATTRIBUTE过程

GET_ATTRIBUTE过程获取当前连接的会话中命名空间中指定属性的值。如果不存在与命名空间对应的模板，则会引发错误。如果指定的属性不存在，则返回空字符串。

如果属性值为NULL ，则设置firstRead事件，并且它是第一次获取属性值，然后调用属性的处理函数。

不要求主叫用户授予任何权限。

句法

GET_ATTRIBUTE（名称空间IN VARCHAR2，属性IN VARCHAR2，值OUT NOCOPY VARCHAR2）;

参数

参数 描述

命名空间

要检索的属性的命名空间。区分大小写的字符串的最大大小为128个字符。

要查找当前会话的现有命名空间列表，请在连接后查询V$XS_SESSION_NS_ATTRIBUTES数据字典视图。您可以查询DBA_XS_SESSION_NS_ATTRIBUTES数据字典视图，以查找所有应用程序会话中的所有命名空间。

属性

要检索的属性的名称。区分大小写的字符串的最大大小为4000个字符。要查找可用属性的列表，请查询V $ XS_SESSION_NS_ATTRIBUTES数据字典视图。

值

要检索的属性的值。

要查找可用属性值的列表，请查询V $ XS_SESSION_NS_ATTRIBUTES数据字典视图。

例子

DECLARE attrVal VARCHAR2（4000）;
BEGIN SYS.DBMS_XS_SESSIONS.GET_ATTRIBUTE（'J_NS1'，'JohnNS1Attr1'，attrVal）;结束;

RESET_ATTRIBUTE过程

RESET_ATTRIBUTE过程将属性的值重置为其默认值（如果存在）或当前附加会话中的命名空间中的NULL 。如果属性指定了默认值，则该值将重置为默认值。如果创建的属性没有默认值并标记为attribute_first_read_event ，则将该值设置为NULL并将该属性标记为未初始化。如果创建的属性没有默认值且未标记为attribute_first_read_event ，则该值设置为NULL 。

调用用户需要被授予MODIFY_ATTRIBUTE应用程序权限。

句法

属性

此操作不要求调用用户具有任何其他权限。

SET_SESSION_COOKIE程序

SET_SESSION_COOKIE过程使用指定的会话ID设置新的cookie值。如果指定的会话不存在或者cookie名称在所有用户应用程序会话中不唯一，则会引发错误。

要执行此过程，需要为用户授予MODIFY_SESSION应用程序特权。

句法

SET_SESSION_COOKIE（cookie IN VARCHAR2，sessionid IN RAW DEFAULT NULL）;

参数

参数 描述

曲奇饼

新cookie的名称。cookie的最大允许长度为1024个字符。Cookie名称必须是唯一的。

要查找当前会话的现有cookie列表，请查询XS_SYS_CONTEXT(XS$SESSION','COOKIE') 。

会话ID

应用程序会话的会话ID。默认值为NULL 。您可以使用以下方法之一获取会话ID：

  • SELECT XS_SYS_CONTEXT（'XS $ SESSION'，'SESSION_ID'）FROM DUAL;

  • 使用DBMS_XS_SESSIONS.GET_ATTRIBUTE过程。

如果未指定会话ID或输入NULL ，则SET_SESSION_COOKIE将使用当前应用程序会话的会话ID。

例子

DECLARE sessionid RAW（16）; BEGIN SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwuser1'，sessionid）; SYS.DBMS_XS_SESSIONS.SET_SESSION_COOKIE（'cookie1'，sessionid）;结束;

SET_INACTIVITY_TIMEOUT过程

SET_INACTIVITY_TIMEOUT过程为当前连接的会话设置不活动超时值（以分钟为单位）。不活动超时值表示Oracle数据库终止应用程序会话并回收资源之前允许的最长不活动时间。尝试为time参数设置负值会引发错误。如果指定了无效的会话ID或者会话不存在，则会引发错误。

设置超时值的另一种方法是使用setInactivityTimeout Java方法。您可以在xmlconfig.xml配置文件中设置默认的全局超时值。Oracle建议240（4小时）。

在附加传统会话时，应用程序会话不会因不活动而超时。每次传统会话附加到应用程序会话时，都会更新上次访问时间。

要执行此过程，需要为调用用户授予MODIFY_SESSION应用程序特权。

句法

SET_INACTIVITY_TIMEOUT（时间IN NUMBER，sessionid IN RAW DEFAULT NULL）;

参数

参数 描述

时间

不活动超时值，以分钟为单位。Oracle建议将time参数设置为240 （4小时）。零（0）值表示该值是无限的，并且会话永远不会因不活动而到期。

会话ID

应用程序会话的会话ID。默认值为NULL 。您可以使用以下方法之一获取会话ID：

  • SELECT XS_SYS_CONTEXT（'XS $ SESSION'，'SESSION_ID'）FROM DUAL;

  • 使用DBMS_XS_SESSIONS.GET_ATTRIBUTE过程。

如果未指定会话ID或输入NULL ，则SET_INACTIVITY_TIMEOUT将使用当前应用程序会话的会话ID。

例子

DECLARE sessionid RAW（16）; BEGIN SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwuser1'，sessionid）; SYS.DBMS_XS_SESSIONS.SET_INACTIVITY_TIMEOUT（300，sessionid）;结束; /

DETACH_SESSION程序

DETACH_SESSION过程将当前传统数据库会话与其所连接的应用程序会话分离。数据库会话将返回到附加到应用程序会话之前的上下文。任何用户都可以执行此过程，因为操作不需要任何特权来执行。

句法

DETACH_SESSION（在BOOLEAN DEFAULT FALSE中止）;

参数

参数 描述

退出

如果指定为TRUE ，则回滚在当前会话中执行的更改。如果指定为FALSE （默认值），则会话中执行的所有更改都将保留。如果为此参数指定了NULL值，则将其视为FALSE 。

例子

BEGIN SYS.DBMS_XS_SESSIONS.DETACH_SESSION;结束;

DESTROY_SESSION过程会破坏指定的会话。此过程还隐式地将所有传统会话与应用程序会话分离。在会话被销毁之后，不能再对会话进行任何附加。此操作无法销毁通过直接登录应用程序用户创建的会话。

要执行此过程，用户必须具有TERMINATE_SESSION应用程序特权。

句法

DESTROY_SESSION（在RAW中的sessionid，强制进入BOOLEAN DEFAULT FALSE）;

参数

参数 描述

会话ID

应用程序会话的会话ID。您可以使用以下方法之一获取会话ID：

  • SELECT XS_SYS_CONTEXT（'XS $ SESSION'，'SESSION_ID'）FROM DUAL;

  • 使用DBMS_XS_SESSIONS.GET_ATTRIBUTE过程。

如果未指定会话ID或输入NULL ，则DESTROY_SESSION将使用当前应用程序会话的会话ID。

力

如果设置为FALSE ，则此操作将引发错误，以防当前附加指定的会话。如果设置为TRUE ，则可以销毁当前连接的应用程序会话。这是一个可选参数。

例子

DECLARE sessionid RAW（16）; BEGIN SYS.DBMS_XS_SESSIONS.CREATE_SESSION（'lwtSession1'，sessionid）; SYS.DBMS_XS_SESSIONS.DESTROY_SESSION（sessionid）;结束;

ADD_GLOBAL_CALLBACK过程

ADD_GLOBAL_CALLBACK过程将现有的PL / SQL过程注册为事件处理程序，并使用event_type参数指定的会话操作。您可以为相同的会话操作添加多个事件处理程序，以便在发生关联事件时执行。添加全局回调过程会自动启用回调过程以执行。如果为同一会话事件添加了多个回调，则根据其注册顺序执行它们，即首先执行的首先注册的回调过程。如果指定了无效的事件类型或回调过程不存在，则此过程将引发错误。

成功执行此过程需要CALLBACK应用程序特权。这个角色可以通过PROVISIONER数据库角色获得。

句法

ADD_GLOBAL_CALLBACK（event_type IN PLS_INTEGER，callback_schema IN VARCHAR2，callback_package IN VARCHAR2，callback_procedure IN VARCHAR2）;

参数

参数 描述

事件类型

从以下事件类型中选择：

  • CREATE_SESSION_EVENT

  • ATTACH_SESSION_EVENT

  • CREATE_NAMESPACE_EVENT

  • GUEST_TO_USER_EVENT

  • PROXY_TO_USER_EVENT

  • REVERT_TO_USER_EVENT

  • ENABLE_ROLE_EVENT

  • DISABLE_ROLE_EVENT

  • ENABLE_DYNAMIC_ROLE_EVENT

  • DISABLE_DYNAMIC_ROLE_EVENT

  • DETACH_SESSION_EVENT

  • TERMINATE_SESSION_EVENT

  • DIRECT_LOGIN_EVENT

callback_schema

输入创建回调过程的模式的名称。

callback_package

输入创建回调过程的包的名称。如果回调过程是独立的，则应将NULL作为callback_package参数传递。仅当回调过程在包中时，此参数才是可选的。

callback_procedure

输入定义全局回调的过程的名称。

例子

BEGIN SYS.DBMS_XS_SESSIONS.ADD_GLOBAL_CALLBACK（DBMS_XS_SESSIONS.CREATE_SESSION_EVENT，'APPS1_SCHEMA'，'APPS2_PKG'，'CREATE_SESSION_CB'）;结束;
