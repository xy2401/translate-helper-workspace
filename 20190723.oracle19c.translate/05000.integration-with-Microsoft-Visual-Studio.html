G将Pro * C / C ++集成到Microsoft Visual Studio .NET中

本附录介绍如何将Pro * C / C ++集成到Microsoft Visual Studio .NET 2002/2003（或更高版本）集成开发环境中。

本附录包含以下主题：

  • 在Microsoft Visual Studio .NET项目中集成Pro * C / C ++

  • 将Pro * C / C ++添加到工具菜单

G.1在Microsoft Visual Studio .NET项目中集成Pro * C / C ++

本节介绍如何在Microsoft Visual Studio .NET项目中完全集成Pro * C / C ++。

所有预编译器错误和警告都显示在Microsoft Visual Studio .NET 2002/2003显示编译器和链接器消息的输出框中。您不必分别从Microsoft Visual Studio .NET 2002/2003构建环境预编译文件。更重要的是，Microsoft Visual Studio .NET 2002/2003维护.c和.pc文件之间的依赖关系。如果需要，Microsoft Visual Studio .NET 2002/2003维护依赖项和预编译文件。

本节中的所有过程都在Microsoft Visual Studio .NET中执行。

G.1.1指定Pro * C / C ++可执行文件的位置

要使Microsoft Visual Studio .NET运行Pro * C / C ++，它必须知道Pro * C / C ++可执行文件的位置。如果在安装任何Oracle发行版产品之前安装了Microsoft Visual Studio .NET，则必须添加目录路径。

要指定Pro * C / C ++可执行文件的位置：

  1. 从工具菜单中选择选项 。

    出现“选项”对话框。

  2. 单击“ 目录”选项卡。
  3. 从Show directories For中选择可执行文件 。
  4. 滚动到“目录”框的底部，然后单击虚线矩形。
  5. 输入ORACLE_BASE\ORACLE_HOME \bin目录。例如：

    C：\ ORACLE \ ora121 \ BIN

  6. 单击确定 。

G.1.2指定Pro * C / C ++头文件的位置

要指定Pro * C / C ++头文件的位置：

  1. 从工具菜单中选择选项 。出现“选项”对话框。
  2. 单击“ 目录”选项卡。
  3. 从显示目录中选择包含文件列表。
  4. 滚动到“目录”框的底部，然后单击虚线矩形 。
  5. 输入ORACLE_BASE\ORACLE_HOME \precomp\public目录。例如：

    C：\ ORACLE \ ora121 \预补偿\公

  6. 单击确定 。

G.2将.pc文件添加到项目中

创建项目后，需要添加.pc文件。

要将.pc文件添加到项目：

  1. 从Project菜单中选择Add To Project ，然后选择Files。将出现“将文件插入项目”对话框。

    插图描述ms2.gif

  2. 从文件列表中选择所有文件 。
  3. 选择.pc文件。
  4. 单击确定 。

G.2.1将.c文件的引用添加到项目中

对于每个.pc文件，您需要添加对预编译产生的.c文件的引用。

要将.c文件的引用添加到项目：

  1. 从“项目”菜单中选择“ 添加到项目 ”，然后选择“文件”。将出现“将文件插入项目”对话框。
  2. 在“文件名”框中键入.c文件的名称。
  3. 单击确定 。由于尚未创建.c文件，Microsoft Visual Studio .NET将显示以下消息：“指定的文件不存在。您是否想要添加对项目的引用？“
  4. 单击是 。

G.2.2将Pro * C / C ++库添加到项目中

Pro * C / C ++应用程序必须与库文件orasql12.lib链接。

要将Pro * C / C ++库添加到项目中：

  1. 从“项目”菜单中选择“ 添加到项目 ”，然后选择“ 文件” 。将出现“将文件插入项目”对话框。
  2. 从文件列表中选择所有文件 。
  3. 从ORACLE_BASE\ORACLE_HOME \precomp\lib\msvc目录中选择orasql12.lib 。
  4. 单击确定 。

G.2.3指定自定义构建选项


插图描述ms3.gif

要指定自定义构建选项：

  1. 在FileView中，右键单击.pc文件，然后选择“设置”。出现“项目设置”对话框，并显示“自定义构建”选项
  2. 在“构建命令”框中的一行中，将构建设置为使用与$ORACLE_HOME设置相同的硬编码路径。
  3. 在“输出文件”框中，输入以下内容之一：

    • 如果要生成.c文件，请输入$(ProjDir)\$(InputName).c 。

    • 如果要生成.cpp文件，请输入$(ProjDir)\$(InputName).cpp 。

    $(ProjDir)和$MSDEVDIR是Microsoft Visual Studio .NET中自定义生成命令的宏。构建项目时，Microsoft Visual Studio .NET会检查输出文件的日期，以确定是否需要为源代码所做的任何新修改重建它们。

  4. 单击确定 。

G.3将Pro * C / C ++添加到工具菜单


插图描述ms1.gif

您可以在Microsoft Visual Studio .NET的“工具”菜单中包含Pro * C / C ++作为选项。

要将Pro * C / C ++添加到“工具”菜单：

  1. 在Microsoft Visual Studio .NET中，从“工具”菜单中选择“ 自定义 ”。出现“自定义”对话框。
  2. 单击“ 工具”选项卡。
  3. 滚动到“菜单内容”框的底部，然后单击虚线矩形。
  4. 输入以下文本：

    PRO * C / C ++

  5. 在“命令”框中，键入图形Pro * C / C ++可执行文件的路径和文件名，或使用框右侧的“浏览”按钮选择文件名。例如：

    C：\ ORACLE \ ora121 \ BIN \ procui.exe

  6. 在“参数”框中，输入以下文本：

    $（的TargetName）

    从“工具”菜单中选择Pro * C / C ++时 ，Microsoft Visual Studio .NET使用$(TargetName)参数将当前开发项目的名称传递给Pro * C / C ++。Pro * C / C ++然后打开一个预编译项目，其名称与打开的项目相同，但在项目目录中具有.pre扩展名。

  7. 在“初始目录”框中，输入以下文本：

    $（WkspDir）

    现在，“自定义”对话框应如下图所示（尽管您的计算机上的Oracle主目录可能不同）。

  8. 单击关闭。 Microsoft Visual Studio .NET将Pro * C / C ++添加到“工具”菜单。
