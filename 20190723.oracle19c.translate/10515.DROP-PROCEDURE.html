DROP程序

目的

使用PL / SQL定义过程。有关创建，更改和删除过程的完整信息，请参阅Oracle数据库PL / SQL语言参考 。

使用DROP PROCEDURE语句从数据库中删除独立存储过程。请勿使用此语句删除属于包的过程。相反，要么使用DROP PACKAGE语句删除整个包，要么使用带有OR REPLACE子句的CREATE PACKAGE语句重新定义包而不使用该过程。

先决条件

该过程必须位于您自己的架构中，或者您必须具有DROP ANY PROCEDURE系统特权。

句法

drop_procedure :: =


插图说明drop_procedure.eps

语义

模式

指定包含该过程的模式。如果省略schema ，则Oracle数据库假定该过程位于您自己的模式中。

程序

指定要删除的过程的名称。

删除过程时，Oracle数据库会使依赖于已删除过程的任何本地对象失效。如果随后引用其中一个对象，则数据库会尝试重新编译该对象，如果尚未重新创建已删除的过程，则会返回错误消息。

例子

删除过程：示例

以下语句删除用户hr拥有的过程remove_emp ，并使依赖于remove_emp所有对象无效：

DROP PROCEDURE hr.remove_emp;
