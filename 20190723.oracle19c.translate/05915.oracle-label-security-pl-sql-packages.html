SA_AUDIT_ADMIN.AUDIT

SA_AUDIT_ADMIN.AUDIT过程启用特定于策略的审核。

每项政策的审计独立于其他政策。审计记录捕获Oracle Label Security管理操作以及在登录，DML执行和受信任的存储过程调用期间使用的Oracle Label Security特权。

句法

SA_AUDIT_ADMIN.AUDIT（policy_name IN VARCHAR2，用户IN VARCHAR2 DEFAULT NULL，audit_option IN VARCHAR2 DEFAULT NULL，audit_type IN VARCHAR2 DEFAULT NULL，成功IN VARCHAR2 DEFAULT NULL）;

参数

表E-1 SA_AUDIT_ADMIN.AUDIT参数

参数 描述

POLICY_NAME

指定策略。要查找现有策略，请查询ALL_SA_POLICIES数据字典视图的POLICY_NAME列。

用户

可选的。要审核的逗号分隔的用户名列表，如下所示：

  • 如果您正在审核OLS管理操作，请确保您输入的用户具有policy_ DBA角色和Oracle Label Security软件包的EXECUTE特权。

  • 如果您正在审核OLS权限的使用，则这些用户不需要是OLS管理员。

  • 如果未指定任何用户，则会审核所有用户。

要查找具有修改Oracle Label Security策略权限的用户，请查询DBA_SA_USERS视图的USER_NAME列。

audit_option

可选的。以逗号分隔的要审核的选项列表。选项如下：

  • APPLY ：将指定的Oracle Label Security策略的应用程序审核到表和模式

  • REMOVE ：审核从表和模式中删除指定的Oracle Label Security策略

  • SET ：审核用户授权的设置以及用户和程序权限

  • PRIVILEGES ：审计使用所有特定于策略的特权

如果未指定，则审核所有默认选项（即不包括权限的选项）。应通过指定PRIVILEGES选项显式设置特权操作的审计选项，该选项为所有特权设置审计选项。

audit_type

可选的。BY ACCESS或BY SESSION 。如果未指定，则审计记录由BY SESSION编写。

成功

可选的。SUCCESSFUL ，如果操作成功，或NOT SUCCESSFUL 。如果未指定，则为两者编写审计。

例子

以下示例审核用户psmith和rlayton任何失败的APPLY和REMOVE尝试。 

BEGIN SA_AUDIT_ADMIN.AUDIT（policy_name =>'hr_ols_pol'，users =>'jjones，rlayton'，audit_option =>'apply，remove'，audit_type =>'by access'，success =>'not successful'）;结束; /

如果您未指定任何审计选项，则审计除与权限相关的选项之外的所有选项。您必须明确指定权限审核。例如，如果输入以下语句，则会为hr_ols_pol策略设置默认选项：

EXEC SA_AUDIT_ADMIN.AUDIT（'hr_ols_pol'）;

启用审核时，将按会话对所有用户执行审核，无论其操作是否成功。

设置审核参数和选项时，新值仅适用于后续会话，而不适用于当前会话。

还要考虑一种情况，其中一个SA_AUDIT_ADMIN.AUDIT调用（未指定用户）启用对所有用户的APPLY操作的审计，然后第二个调用启用对特定用户的REMOVE操作的审计。例如：

EXEC SA_AUDIT_ADMIN.AUDIT（'hr_ols_pol'，null，'apply'）; EXEC SA_AUDIT_ADMIN.AUDIT（'hr_ols_pol'，'scott'，'remove'）;

在这种情况下， SCOTT将同时进行APPLY和REMOVE操作的审核。

SA_AUDIT_ADMIN.NOAUDIT

SA_AUDIT_ADMIN.NOAUDIT过程禁用Oracle Label Security特定于策略的审核。

句法

SA_AUDIT_ADMIN.NOAUDIT（policy_name IN VARCHAR2，用户IN VARCHAR2 DEFAULT NULL，audit_option IN VARCHAR2 DEFAULT NULL）;

参数

表E-6 SA_AUDIT_ADMIN.NO_AUDIT参数

参数 描述

POLICY_NAME

指定策略。要查找现有策略，请查询ALL_SA_POLICIES数据字典视图的POLICY_NAME列。

用户

可选的。以逗号分隔的已审核用户列表。如果未指定，则禁用所有用户的审核。

要查找具有修改Oracle Label Security策略权限的用户，请查询ALL_SA_AUDIT_OPTIONS视图的USER_NAME列。

audit_option

可选的。以逗号分隔的要禁用的选项列表。选项如下：

  • APPLY ：将指定的Oracle Label Security策略的应用程序审核到表和模式

  • REMOVE ：审核从表和模式中删除指定的Oracle Label Security策略

  • SET ：审核用户授权的设置以及用户和程序权限

  • PRIVILEGES ：审计使用所有特定于策略的特权

如果未指定，则禁用所有默认选项。必须明确禁用权限。

例子

以下示例禁用用户psmith和rlayton对APPLY和REMOVE尝试失败的审核。 

BEGIN SA_AUDIT_ADMIN.NOAUDIT（policy_name =>'hr_ols_pol'，users =>'jjones'，audit_option =>'apply，remove'）;结束; /

您可以禁用所有已启用选项的审核，也可以仅对已启用选项的子集执行审核。对于所有指定的用户（或所有用户，如果users参数为NULL ），将禁用对指定选项的所有审核。例如，以下语句禁用对用户John，Mary和Scott的APPLY和REMOVE操作的审计：

EXEC SA_AUDIT_ADMIN.NOAUDIT（'HR'，'JOHN，MARY，SCOTT'，'APPLY，REMOVE'）;

还要考虑一种情况，其中一个AUDIT调用启用特定用户的审核，第二个调用（未指定用户）启用所有用户的审核。例如：

EXEC SA_AUDIT_ADMIN.AUDIT（'HR'，'SCOTT'）; EXEC SA_AUDIT_ADMIN.AUDIT（'HR'）;

在这种情况下，在没有指定用户的情况下对NOAUDIT的后续调用（例如以下语句）不会反转在第一次调用中明确为SCOTT设置的审计。因此，继续在SCOTT上进行审计。

EXEC SA_AUDIT_ADMIN.NOAUDIT（'HR'）;

这样，即使为所有用户设置了SA_AUDIT_ADMIN.NOAUDIT ，Oracle Label Security仍会审核明确设置了审核的任何用户。

必须明确指定特权操作的审计。如果您运行没有选项的SA_AUDIT_ADMIN.NOAUDIT ，Oracle Label Security仍将继续审核特权操作。例如，如果启用了审核并且您输入了

EXEC SA_AUDIT_ADMIN.NOAUDIT（'HR'）;

然后将继续对特权操作（例如WRITEDOWN ）执行审计。

您设置的SA_AUDIT_ADMIN.NOAUDIT参数和选项仅适用于后续会话，而不适用于当前会话。

如果尝试启用已设置的审核选项，或者尝试禁用尚未设置的审核选项，则Oracle Label Security会处理该语句而不指示错误。 尝试指定无效选项会导致出现错误消息。您可以通过查询ALL_SA_AUDIT_OPTIONS数据字典视图来查找审计选项的状态。

SA_COMPONENTS.ALTER_COMPARTMENT过程更改与隔离专区关联的短名称和长名称。

设置后，无法更改comp_num参数。如果comp_num参数在任何现有的标签使用的，那么它的短名称不能改变，但其长期的名称可以更改。

SA_UTL.SET_LABEL

SA_UTL.SET_LABEL过程设置当前数据库会话的标签。

会话的写标签和行标签设置为标签的隔离专区和授权进行写访问的组的子集。

句法

SA_UTL.SET_LABEL（policy_name IN VARCHAR2，标签IN LBAC_LABEL）;

参数

表E-80 SA_UTL.SET_LABEL参数

参数 描述

POLICY_NAME

指定策略。要查找现有策略，请查询ALL_SA_POLICIES数据字典视图的POLICY_NAME列。

标签

要设置为会话标签的标签。要查找现有标签值，请查询ALL_SA_LABELS视图的LABEL列。

您必须将此参数作为TO_LBAC_DATA_LABEL函数的输出传递，该函数将字符形式的标签转换为LBAC_LABEL类型。（下一节中的示例显示了如何执行此操作。）

例

以下示例设置hr_ols_pol策略的标签。 

BEGIN SA_UTL.SET_LABEL（policy_name =>'hr_ols_pol'，label => to_lbac_data_label（'hr_ols_pol'，'hs：pii'））;结束; /

SA_UTL.SET_ROW_LABEL

SA_UTL.SET_ROW_LABEL过程设置当前数据库会话的行标签。

标签中的隔离专区和组必须是会话标签中有权进行写访问的隔离专区和组的子集。

句法

SA_UTL.SET_ROW_LABEL（policy_name IN VARCHAR2，标签IN BINARY_INTEGER）;

参数

表E-81 SA_UTL.SET_ROW_LABEL参数

参数 描述

POLICY_NAME

指定策略。要查找现有策略，请查询ALL_SA_POLICIES数据字典视图的POLICY_NAME列。

标签

要设置为会话默认行标签的标签。要查找现有标签值，请查询ALL_SA_LABELS视图的LABEL列。

例

以下示例将hr_ols_pol策略的行标签设置为3000 。 

BEGIN SA_UTL.SET_ROW_LABEL（policy_name =>'hr_ols_pol'，label => 1111）;结束; /
