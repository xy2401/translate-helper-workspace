13.49 PIPELINED条款

指示数据库迭代地返回表函数或多态表函数的结果 。

仅用于表函数，以指定它是流水线的。流水线表函数在处理该行后立即向其调用者返回一行并继续处理行。要向调用者返回一行（但不是控件），该函数使用“ PIPE ROW语句 ” 。

表函数返回集合类型。

多态表函数是一个表函数，其返回类型由参数确定。

您可以在查询的FROM子句中的函数名称之前使用TABLE关键字来查询这两种表函数。例如：

SELECT * FROM TABLE（ function_name （...））

当表函数参数列表或空列表（）出现时， TABLE运算符是可选的。例如：

SELECT * FROM function_name （）

然后，数据库返回由函数生成的行。

PIPELINED选项可以出现在以下SQL语句中：

  • 创建功能声明

  • CREATE PACKAGE声明

  • CREATE PACKAGE BODY声明

语义

pipelined_clause

pipelined_clause只能在函数中出现一次。

PIPELINED

要创建流水线函数，请在函数定义中包含pipelined_clause 。如果在定义流水线函数之前声明它，则必须在函数声明中指定PIPELINED选项。

{IS |使用}

  • 如果单独指定关键字PIPELINED （ PIPELINED IS ...），则PL / SQL函数体必须使用PIPE关键字。此关键字指示数据库从函数中返回集合的单个元素，而不是将整个集合作为单个值返回。

  • 您可以指定PIPELINED USING implementation_type子句来预定义包含start，fetch和close操作的接口。实现类型必须实现ODCITable接口，并且必须在创建表函数时存在。此子句对于在外部语言（如C ++和Java）中实现的表函数很有用。

    如果函数的返回类型是ANYDATASET ，那么还必须将describe方法（ ODCITableDescribe ）定义为函数实现类型的一部分。

[schema。] implementation_type

实现类型必须是包含ODCIAggregate子程序实现的ADT。如果未指定schema ，则数据库假定实现类型位于模式中。

对PIPELINED的限制

您不能为嵌套函数指定PIPELINED 。

注意：

您无法在数据库链接上运行管道化表函数。原因是流水线表函数的返回类型是SQL用户定义的类型，只能在单个数据库中使用（如Oracle数据库对象关系开发人员指南中所述 ）。虽然流水线表函数的返回类型可能看起来是PL / SQL类型，但数据库实际上将该PL / SQL类型转换为相应的SQL用户定义类型。

PIPELINED [行]表] POLYMORPHIC [使用[schema。] implementation_package]

多态表函数elaborator可以出现在独立函数声明或包函数声明中。

PIPELINED

定义多态表函数时需要。

行

当TABLE类型的单个输入参数使用任何单行确定新列时，指定ROW 。

表

指定TABLE当类型的单个输入参数TABLE使用当前行确定新的列和对整个表或表的一个逻辑分区操作。

POLYMORPHIC

对多POLYMORPHIC限制

POLYMORPHIC表函数不允许以下内容：

  • PARALLEL_ENABLE子句

  • RESULT_CACHE子句

  • DETERMINISTIC选项

  • AUTHID属性（Invoker的权利和定义者的权利条款）

[USING [schema。] implementation_package ]

引用多态表函数（PTF）实现包。规范必须包括DESCRIBE方法。OPEN ， FETCH_ROWS和CLOSE方法的规范是可选的。实现包的规范必须已存在（除非PTF及其实现位于同一个包中）。

如果多态表函数及其实现方法在同一个包中定义，则USING子句是可选的。
