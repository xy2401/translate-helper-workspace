269.1 UTL_URL概述

统一资源定位符（URL）是标识Web资源（例如页面或图片）的字符串。使用URL通过超文本传输协议（HTTP）访问此类资源。

例如，Oracle网站的URL是：

http://www.oracle.com

通常，URL包含英文字母字符，数字和标点符号。这些字符称为无保留字符 。URL中的任何其他字符（包括多字节字符或二进制八位字节代码）都必须进行转义，以便Web浏览器或Web服务器进行准确处理。一些标点符号（如美元符号（ $) ，问号（ ?) ，冒号（ :)和等号（ =)将保留为URL中的分隔符。它们被称为保留字符 。要从字面上处理这些字符，而不是将它们视为分隔符，它们必须被转义。

未保留的字符是：

  • A到Z, a到z,和0到9

  • 连字符（ - ），下划线（ _ ），句点（ . ），感叹号（ ! ），波浪号（ ~ ），星号（ * ），重音符号（ ' ），左括号（ ( ），右括号（ ) ）

保留的字符是：

  • 分号（ ; ）斜杠（ / ），问号（ ? ，结肠）（ ： :符号（ @ ），符号（ & ），等号（ = ），加号（ + ），美元符号（ $ ） ，百分号（ % ）和逗号（ , ）

UTL_URL包有两个函数，为URL字符提供转义和unescape机制。在使用URL之前使用escape函数转义URL以通过UTL_HTTP包获取Web页面。在从URL中提取信息之前，使用unescape函数来转义转义的URL。

有关更多信息，请参阅Request For Comments（RFC）文档RFC2396。请注意，此URL转义和unescape机制与HTML规范中描述的x-www-form-urlencoded编码机制不同：

http://www.w3.org/TR/html

269.3 UTL_URL示例

UTL_URL可用于编码和解码。

您可以使用UTL_URL.ESCAPE函数实现x-www-form-urlencoded编码，如下所示：

创建或替换函数form_url_encode（data IN VARCHAR2，charset IN VARCHAR2）RETURN VARCHAR2 AS BEGIN RETURN utl_url.escape（data，TRUE，charset）; - 注意使用TRUE END;

对于使用form-URL-encode scheme编码的数据进行解码，以下函数实现了解码方案：

创建或替换函数form_url_decode（data IN VARCHAR2，charset IN VARCHAR2）RETURN VARCHAR2 AS BEGIN RETURN utl_url.unescape（replace（data，'+'，''），charset）;结束;

269.4.1 ESCAPE功能

此函数返回一个URL，其中包含使用%2-digit-hex-code格式转义的非法字符（以及可选的保留字符）。

句法

UTL_URL.ESCAPE（url IN VARCHAR2 CHARACTER SET ANY_CS，escape_reserved_chars IN BOOLEAN DEFAULT FALSE，url_charset IN VARCHAR2 DEFAULT utl_http.body_charset）RETURN VARCHAR2;

参数

表269-3 ESCAPE功能参数

参数 描述

网址

原始网址

escape_reserved_chars

指示是否应转义URL保留字符。如果设置为TRUE ，则会保留保留和非法URL字符。否则，只会转义非法的URL字符。默认值为FALSE 。

url_charset

转义字符（单字节或多字节）时，确定在以十六进制代码格式转义字符之前应转换为字符的目标字符集。如果url_charset为NULL ，则假定数据库字符集，并且不会发生字符集转换。默认值是UTL_HTTP包的当前默认主体字符集，其默认值为ISO-8859-1 。字符集可以在Internet Assigned Numbers Authority（IANA）或Oracle命名约定中命名。

使用说明

使用此函数可以转义包含URL规范RFC 2396中定义的非法字符的URL。URL中的合法字符是：

  • A到Z, a到z,和0到9

  • 连字符（ - ），下划线（ _ ），句点（ . ），感叹号（ ! ），波浪号（ ~ ），星号（ * ），重音符号（ ' ），左括号（ ( ），右括号（ ) ）

保留字符包括：

  • 分号（ ; ）斜杠（ / ），问号（ ? ，结肠）（ ： :符号（ @ ），符号（ & ），等号（ = ），加号（ + ），美元符号（ $ ）和逗号（ , ）

许多保留字符用作URL中的分隔符。您应该使用escape_url转义超出此处列出的字符。此外，要在URL的查询字符串的名称 - 值对中使用保留字符，必须单独转义这些字符。escape_url无法识别是否需要转义这些字符，因为一旦在URL中，这些字符就会与实际的分隔符无法区分。例如，要将名称 - 值对$logon=scott/tiger传递到URL的查询字符串中，请将$和/分别作为%24logon=scott%2Ftiger并在URL中使用它。

通常，您将转义整个URL，其中包含不应转义的保留字符（分隔符）。例如：

utl_url.escape（'http://www.acme.com/a url with space.html'）

返回：

http://www.acme.com/a%20url%20with%20space.html

在其他情况下，您可能希望发送包含保留字符的值的查询字符串。在这种情况下，只完全转义值（将escape_reserved_chars设置为TRUE ），然后将其与URL的其余部分连接起来。例如：

url：='http：//www.acme.com/search？check ='|| utl_url.escape（'是否使用“$”符号好吗？'，TRUE）;

这个表达式转义了问号（ ? ），美元符号（ $ ）和空格字符' Is the use of the "$" sign okay?'但不是? 在表示使用查询字符串的URL中search之后。

您打算从中获取Web页面的Web服务器可能使用与数据库不同的字符集。在这种情况下，请将url_charset指定为Web服务器字符集，以便在目标字符集中转义需要转义的字符。例如，想要访问ASCII Web服务器的EBCDIC数据库用户应使用US7ASCII转义URL，以便将空格转义为%20 （ASCII中的空格的十六进制代码）而不是%40 （十六进制代码为EBCDIC中的空间）。

此功能不会验证正确URL格式的URL。

269.4.2 UNESCAPE功能

此函数将转义字符序列转换为URL中的原始形式，以将%XX转义字符序列转换为原始字符。

句法

UTL_URL.UNESCAPE（url IN VARCHAR2 CHARACTER SET ANY_CS，url_charset IN VARCHAR2 DEFAULT utl_http.body_charset）RETURN VARCHAR2;

参数

表269-4 UNESCAPE功能参数

参数 描述

网址

unescape的URL

url_charset

字符未转义后，假定该字符位于source_charset字符集中，并在返回URL之前将其从source_charset转换为数据库字符集。如果source_charset为NULL ，则假定数据库字符集，并且未进行字符集转换。默认值是UTL_HTTP包的当前默认正文字符集，其默认值为“ISO-8859-1”。字符集可以在Internet Assigned Numbers Authority（IANA）或Oracle命名约定中命名。

使用说明

从您收到URL的Web服务器可能使用与数据库不同的字符集。在这种情况下，请将url_charset指定为Web服务器字符集，以便在源字符集中未转义需要转义的字符。例如，从ASCII Web服务器接收URL的EBCDIC数据库的用户应该使用US7ASCII来US7ASCII URL，以便%20不作为空格转义（0x20是ASCII中空格的十六进制代码）而不是? （因为0x20不是EBCDIC中的有效字符）。

此功能不会验证正确URL格式的URL。
