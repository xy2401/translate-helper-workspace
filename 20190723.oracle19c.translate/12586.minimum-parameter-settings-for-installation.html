设置资源控制参数的准则

  • 除非另行指定，否则上表中的内核参数和shell限制值仅为最小值。验证上表中显示的内核参数是否设置为大于或等于显示的最小值的值。对于生产数据库系统，Oracle建议您调整这些值以优化系统性能。有关内核资源管理的更多信息，请参阅操作系统文档。

  • 如果任何参数的当前值大于上表中列出的值，则Fixup脚本不会更改该参数的值。

  • project.max-shm-memory资源控制值假定没有其他应用程序使用此项目中除Oracle实例之外的共享内存段。如果除Oracle实例之外的应用程序正在使用共享内存段，则必须将该共享内存使用量添加到project.max-shm-memory资源控制值。

  • project.max-shm-memory资源控制=在相应项目下启动的每个Oracle数据库实例上分配的所有共享内存的累积总和。

  • 确保memory_target或max_sga_size不超过process.max-address-space和project.max-shm-memory 。有关更多信息，请参阅My Oracle Support说明1370537.1 。
