10 SDO_GEOR_GDAL包参考

SDO_GEOR_GDAL包通过外部过程将开源软件GDAL与Oracle Database Server集成，并提供PL / SQL API来执行一组GDAL操作。本章为每个子程序提供一个或多个示例的参考信息。

SDO_GEOR_GDAL包中的函数和过程在Oracle数据库服务器计算机上执行，并且可以与任何其他GeoRaster PL / SQL API一起使用。

目前，SDO_GEOR_GDAL包仅适用于Windows和Linux操作系统。

有关更多信息（包括配置要求），请参阅使用SDO_GEOR_GDAL包 。

  • SDO_GEOR_GDAL.dem
  • SDO_GEOR_GDAL.translate

10.1 SDO_GEOR_GDAL.dem

格式

SDO_GEOR_GDAL.dem（inGeoRaster IN SDO_GEORASTER，outGeoRaster IN OUT SDO_GEORASTER，处理IN VARCHAR2 DEFAULT NULL，选项IN VARCHAR2 DEFAULT NULL，createOptions IN VARCHAR2 DEFAULT NULL，metadataOptions IN VARCHAR2 DEFAULT NULL，colorDirectory IN VARCHAR2 DEFAULT NULL，colorFileName IN VARCHAR2 DEFAULT NULL，openOptions IN VARCHAR2 DEFAULT NULL）;

描述

处理输入数字高程模型（DEM）以生成反映指定处理和转换选项的输出GeoRaster对象。

参数

GeoRaster中的

GeoRaster对象，通常是数字高程模型（DEM）。

outGeoRaster

GeoRaster对象保存操作的结果。必须是有效的初始化GeoRaster对象。不能与inGeoRaster GeoRaster对象相同

处理

指定时，标识要应用的DEM处理技术的名称：

  • aspect生成方面图。

  • color-relief生成颜色浮雕图。

  • hillshade生成阴影浮雕地图。

  • Roughness产生粗糙度图。

  • slope生成坡度图。

  • TPI生成地形位置索引图。

  • TRI生成Terrain Ruggedness Index的地图。

选项

指定时，标识GDAL转换操作的选项。有关可能的options参数值的名称和说明，请参阅使用说明。

示例： options => 'outputType=float32'

createOptions

指定时，标识特定于输出驱动程序的选项。格式'name = value'，选项用空格分隔。示例： "COMPRESS=JPEG-F GENPYRAMID=NN"

metadataOptions

指定后，指定特定于输出驱动程序的元数据值。格式'name = value'，选项用空格分隔。示例： "TIFFTAG_POINTAREA=AREA"

colorDirectory

指定时，标识与输入颜色表文件所在的文件系统目录相关的目录对象的名称。

colorFileName

指定后，标识GDAL兼容颜色表文件的基本文件名。

openOptions

指定时，标识特定于输入驱动程序格式的ptions。有关详细信息，请参阅GDAL支持的格式列表。

使用说明

openOptions参数可能的关键字列在下表中。

表10-1 demOptions参数dem操作的可能值

关键词 说明

ALG

指示是否使用ZevenbergenThorne算法而不是Horn的公式。布尔类型;默认为false 。

高度

仅适用于hillshade处理。指示灯的高度，以度为单位（如果光来自DEM，则为90，如果光是耙光，则为0）。

方位

仅适用于hillshade处理。默认值为315。它应该很少被更改，因为这是通常用于生成着色图的值。

带

标识DEM的带编号。值从1开始（默认值）。

综合

仅适用于hillshade处理。指示是否计算组合着色，斜率和倾斜着色的组合。

computeEdges

指示是否计算光栅边缘的值。布尔类型;默认为false 。

输出类型

输出像素数据类型。支持的值包括： Byte ， Int16 ， UInt16 ， UInt32 ， Int32 ， Float32 ， Float64 ， CInt16 ， CInt32 ， CFloat32和CFloat64 。示例： 'outputtype=float64'

如果未指定此选项，则将使用输入数据类型。

检查输出格式是否支持正在使用的数据类型。

规模

表示乘以垂直单位的比率。例如，使用scale=111120如果垂直单位是米（或scale=370400 ，如果他们在英尺），需要转换。

三角函数

仅用于aspect处理。指示是否返回三角形而不是方位角。 0deg是指东， 90deg是指北， 180deg指西，和270deg指南。

zeroForFlat

仅用于aspect处理。指示是否为slope=0平坦区域返回0，而不是-9999。

zFactor

仅对于hillshade处理，请指出垂直夸大。

为方便起见， options参数的参数也可以使用与GDAL gdal_dem命令行工具相同的格式输入。示例： ”-b 1 –scale 10”

例子

以下示例从输入DEM生成方位图。

DECLARE gr6 sdo_georaster; gr7 sdo_georaster; BEHIN从id = 7的图像中删除;插入图像值（7，sdo_geor.init（'dem_rdt'，7））将栅格返回到gr7;从图像中选择栅格到gr6，其中id = 6; sdo_geor_gdal.dem（inGeoRaster => gr6，outGeoRaster => gr7，processing =>'aspect'，options =>'outputType = float32'）;更新图像集raster = gr7其中id = 7;承诺;结束;

10.2 SDO_GEOR_GDAL.translate

格式

SDO_GEOR_GDAL.translate（inDirectory IN SDO_VARCHAR2，InFileName IN OUT SDO_VARCHAR2，outGeoRaster IN OUT SDO_GEORASTER，options IN VARCHAR2 DEFAULT NULL，createOptions IN VARCHAR2 DEFAULT NULL，metadataOptions IN VARCHAR2 DEFAULT NULL，openOptions IN VARCHAR2 DEFAULT NULL）;

要么

SDO_GEOR_GDAL.translate（inGeoRaster IN SDO_GEORASTER，outDirectory IN VARCHAR2，outFileName IN VARCHAR2，options IN VARCHAR2 DEFAULT NULL，createOptions IN VARCHAR2 DEFAULT NULL，metadataOptions IN VARCHAR2 DEFAULT NULL，openOptions IN VARCHAR2 DEFAULT NULL）;

要么

SDO_GEOR_GDAL.translate（inGeoRaster IN SDO_GEORASTER，outGeoRaster IN OUT SDO_GEORASTER，options IN VARCHAR2 DEFAULT NULL，createOptions IN VARCHAR2 DEFAULT NULL，metadataOptions IN VARCHAR2 DEFAULT NULL，openOptions IN VARCHAR2 DEFAULT NULL）;

描述

在不同格式之间转换栅格数据，可能会执行某些操作，如子集，重新取样和重新缩放过程中的像素

参数

inDirectory

指向包含输入文件的系统目录的目录对象的名称。

inFileName

任何GDAL兼容的栅格数据源规范的基本文件名（不带路径的文件名）。

GeoRaster中的

输入GeoRaster对象

outGeoRaster

GeoRaster对象保存操作的结果。必须是有效的初始化GeoRaster对象。不能与inGeoRaster GeoRaster对象相同

outDirectory

指向系统目录以包含输出文件的目录对象的名称。

outFileName

GeoRaster对象保存操作的结果。必须是有效的初始化GeoRaster对象。不能与inGeoRaster GeoRaster对象相同

选项

指定时，标识GDAL转换操作的选项。有关可能的options参数值的名称和说明，请参阅使用说明。

示例： options => 'srcwin=100,100,2500,2500'

createOptions

指定时，标识特定于输出驱动程序的选项。格式'name = value'，选项用空格分隔。示例： "COMPRESS=JPEG-F GENPYRAMID=NN"

metadataOptions

指定后，指定特定于输出驱动程序的元数据值。格式'name = value'，选项用空格分隔。示例： "TIFFTAG_POINTAREA=AREA"

openOptions

指定时，标识特定于输入驱动程序格式的ptions。有关详细信息，请参阅GDAL支持的格式列表。

使用说明

此函数执行与GDAL translate命令相同的操作。

options参数可能的关键字列在下表中。

表10-2选项翻译操作的参数可能值

关键词 说明

bandList

一组乐队号码。整数列表; 1是第一个乐队。示例： 'bandlist=4,3,2'

如果未指定，将使用输入文件中的波段列表。

生态

当projWin完全在外面时显示错误。布尔类型;默认为false 。示例： 'eco=true'

EPO

当projWin部分在外面时显示错误。布尔类型;默认为false 。示例： 'epo=true'

exponentList

使用幂函数应用非线性缩放。必须是积极的。与缩放选项一起使用。示例： 'exponent=10,100'

格式

支持数据集创建的GDAL驱动程序的名称。字符串类型。示例： 'format=gtiff' 。sdo_georaster输出不需要此选项。对于文件输出，如果未指定此选项，则假定为“GTIFF”。

GCP

地面控制点。可以多次提供以提供一组GCP。格式为“pixel,line,easting,northing,elevation” ，其中高程是可选的。示例： “gcp=10,40,234.2,734 gcp=13,54,28.4,837 gcp=20,90,285.2,934”

maskBandList

一组乐队号码。整数列表，其中1是第一个波段。示例： 'maskbandlist=1'

如果未指定，将使用输入文件中的波段列表。

没有数据

每个输出频段的Nodata值列表（或者none设置它）。示例： ”nodata=9990”

outputBounds

以“upper-left-x, upper-left-y, lower-right-x, lower-right-y”的格式指定输出图像的输出边界。示例： 'outputbounds=293.992,643.447,361.104, 118.648'

outputBoundSRS

outputBounds的空间参照系（SRS）。示例： outputBoundsSRS=EPSG:4326 。如果未指定，则假定输入图像SRS。

输出类型

输出像素数据类型。支持的值包括： Byte ， Int16 ， UInt16 ， UInt32 ， Int32 ， Float32 ， Float64 ， CInt16 ， CInt32 ， CFloat32和CFloat64 。示例： 'outputtype=float64'

如果未指定此选项，则将使用输入数据类型。

检查输出格式是否支持正在使用的数据类型。

如雷贯耳

设置由行和列定义的输出图像的大小或原始图像大小的相对百分比。例：

'超大= 50％，50％'，'超大= 1024,512'，'超大= 200％，200％'

如果未指定，将使用输入图像的原始大小。

projWin

来自在坐标中定义的输入图像的源窗口子区域，如“upper-left-x, upper-left-y, lower-right-x, lower-right-y” 。示例： 'srcwin=180,90,0,0' 。如果未指定，将使用整个输入图像。

projWinSRS

空间参照系（SRS），用于projWin 。示例： projWinSRS=EPSG:4326 。如果未指定，则假定输入图像SRS。

鼠

指示是否将源图像栅格属性表复制到输出图像。默认为true 。示例：“rat = false”

操作取决于输出格式的容量以支持栅格属性表。

resampleAlg

重采样模式{ nearest （默认）， bilinear ， cubic ， cubicspline ， lanczos ， average ， mode }。示例:: resample=cubic

rgbExpand

指示是否将具有1个波段的数据集与颜色表一起转换为具有3（RGB）或4（RGBA）波段的数据集。使用仅包含灰度级别的颜色表的数据集扩展到灰色索引数据集。 { gray|rgb|rgba }。示例： “rgbExpand=rgb”

规模

用于将像素值从输入图像重新缩放到输出图像的值列表。值cam可以定义为“src_min,src_max”或“src_min,src_max,dst_min,dst_max” 。示例： 'scale=-10,2400,0,255'

srcWin

来自输入图像的源窗口子区域以像素坐标定义，如“left_x, top_y, width, height” 。示例： 'srcwin=10,31,400,800'

如果未指定，将使用整个输入图像。

统计

计算并存储输出图像的统计数据。布尔类型;默认为false 。示例： 'stats=true'

严格

如果在转换为输出格式时数据类型不匹配和丢失数据，则引发错误。布尔类型;默认为false 。示例： 'strict=true'

xyRes

输出水平和垂直输出图像分辨率。示例:: 'xyres=30.0,30.0' 。如果未指定，将使用输入图像的分辨率。

为方便起见， options参数的参数也可以使用与GDAL gdal_translate命令行工具相同的格式输入。示例： “-srcwin 10 10 512 512 –outsize 1024 1024“

例子

以下示例将geotiff文件从文件系统加载到初始化的GeoRaster对象。

创建或替换目录mydata_dir AS'/ folder_name / data /';从id = 1的图像中删除BEGIN;插入图像值（1，sdo_geor.init（'imagery_rdt'，1））将栅格返回到gr; sdo_geor_gdal.translate（inDirectory =>'mydata_dir'，inFileName =>'sample.tif'，outGeoRaster => gr）;更新图像集raster = gr其中id = 1;承诺;结束;

以下示例将GeoRaster对象导出到geotiff文件。

创建或替换目录dump_dir AS'/ folder_name / dump /'; DECLARE gr sdo_georaster; BEGIN从图像中选择栅格到gr，其中id = 1; sdo_geor_gdal.translate（inGeoRaster => gr，outDirectory =>'dump_dir'，outFileName =>'copy_imagery_id_1.tif'）;结束;

以下示例将GeoRaster对象复制到图像的另一个裁剪部分，同时更改分辨率和值的比例。

DECLARE gr1 sdo_georaster; gr2 sdo_georaster; BEGIN从id = 2的图像中删除;选择栅格到gr1，其中id = 2;插入图像值（2，sdo_geor.init（'imagery'，2））将栅格返回到gr2; sdo_geor_gdal.translate（inGeoraster => gr1，options =>'srcwin = 100,100,2500,2500'||'scale = 0,255'||'resxy = 400,400'，outGeoRaster => gr）;更新图像集raster = gr2其中id = 2;承诺;结束;

以下示例将GeoRaster对象导出为Erdas .img格式的文件。 

创建或替换目录myoutput_dir AS'/ folder_name / data /'; DECLARE gr sdo_georaster; BEGIN选择栅格到gr，其中id = 3; sdo_geor_gdal.translate（inGeoraster => gr，outDirectory => myoutput_dir ,, outFilename =>'exported.img'，options =>'format = hfa'）;结束;
