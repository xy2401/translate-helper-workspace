13.11收集变量声明

集合变量是一个复合变量，其内部组件（称为元素）具有相同的数据类型。

集合变量的值及其元素的值可以更改。

您可以通过名称引用整个集合。您使用语法collection ( index )引用集合元素。

PL / SQL有三种集合类型：

  • 关联数组（以前称为PL / SQL表或索引表 ）

  • 可变大小的数组（varray）

  • 嵌套表

关联数组可以由字符串类型或PLS_INTEGER索引。 Varrays和嵌套表由整数索引。

您可以使用以下任一方式创建集合变量：

  • 定义集合类型，然后声明该类型的变量。

  • 使用%TYPE声明与先前声明的集合变量相同类型的集合变量。

注意：

本主题适用于您在PL / SQL块或包内定义的集合类型，这些集合类型与使用“ CREATE TYPE语句 ”创建的独立集合类型不同。

在PL / SQL块或包中，您可以定义所有三种集合类型。使用CREATE TYPE语句，您可以创建嵌套表类型和VARRAY类型，但不能创建关联数组类型。

语义

collection_type_definition

类型

您要定义的集合类型的名称。

assoc_array_type_def

关联数组的类型定义。

对assoc_array_type_def的限制

只能出现在块，子程序，包规范或包体的声明部分中。

数据类型

关联数组元素的数据类型。 datatype可以是除REF CURSOR之外的任何PL / SQL数据类型。

不是NULL

对关联数组的每个元素施加NOT NULL约束。有关此约束的信息，请参阅“ NOT NULL约束 ” 。

{PLS_INTEGER | BINARY_INTEGER}

指定关联数组的索引的数据类型是PLS_INTEGER 。

{VARCHAR2 | VARCHAR | STRING}（ v_size ）

指定关联数组的索引的数据类型是VARCHAR2 （或其子类型VARCHAR或STRING ），长度为v_size 。

您可以使用TO_CHAR函数（在Oracle数据库SQL语言参考中描述）使用任何类型的值填充关联数组的元素，该值可以转换为VARCHAR2 。

警告：

由字符串索引的关联数组可能受到国家语言支持（NLS）参数的影响。有关更多信息，请参阅“ NLS参数值影响由字符串索引的关联数组 ” 。

长

指定关联数组的索引的数据类型为LONG ，等同于VARCHAR2(32760) 。

注意：

Oracle仅支持LONG以便与现有应用程序向后兼容。对于新应用程序，请使用VARCHAR2(32760) 。

type_attribute，rowtype_attribute

指定关联数组的索引的数据类型是使用%ROWTYPE或%TYPE指定的数据类型。此数据类型必须表示PLS_INTEGER ， BINARY_INTEGER或VARCHAR2( v_size ) 。

varray_type_def

可变大小数组的类型定义。

SIZE_LIMIT

varray可以拥有的最大元素数。 size_limit必须是1到2147483647范围内的整数字面值。

数据类型

varray元素的数据类型。 datatype可以是除REF CURSOR之外的任何PL / SQL数据类型。

不是NULL

对varray的每个元素强加NOT NULL约束。有关此约束的信息，请参阅“ NOT NULL约束 ” 。

nested_table_type_def

嵌套表的类型定义。

数据类型

嵌套表的元素的数据类型。 datatype可以是除REF CURSOR或NCLOB之外的任何PL / SQL数据类型。

如果datatype是标量类型，则嵌套表具有该类型的单个列，称为COLUMN_VALUE 。

如果datatype是ADT，则嵌套表的列与ADT的名称和属性匹配。

不是NULL

对嵌套表的每个元素强加NOT NULL约束。有关此约束的信息，请参阅“ NOT NULL约束 ” 。

collection_variable_decl

new_collection_var

要声明的集合变量的名称。

assoc_array_type

先前定义的关联数组类型的名称; new_collection_var的数据类型。

varray_type

先前定义的VARRAY类型的名称; new_collection_var的数据类型。

nested_table_type

先前定义的嵌套表类型的名称; new_collection_var的数据类型。

collection_constructor

收集构造函数的数据类型new_collection_var ，其提供的初始值new_collection_var 。

collection_var_1

在相同的数据类型的先前声明集的变量的名称new_collection_var ，其提供的初始值new_collection_var 。

注意：

collection_var_1和new_collection_var必须具有相同的数据类型，而不仅仅是相同类型的元素。

collection_var_2

先前声明的集合变量的名称。

％类型

请参见“ ％TYPE属性 ” 。
