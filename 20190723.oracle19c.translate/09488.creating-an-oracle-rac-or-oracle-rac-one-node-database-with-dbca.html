选择DBCA选项以创建Oracle RAC或Oracle RAC One节点数据库

查看此信息以使用Database Configuration Assistant（DBCA）创建Oracle RAC或Oracle RAC One节点数据库。

注意：

您无法再从DBCA或Oracle Universal Installer（OUI）为Oracle RAC数据库设置电子邮件通知。

  • 启动DBCA
    要启动DBCA，请将安装所有者帐户（例如， oracle ）连接到安装了Oracle RAC的其中一个节点，将SSH密钥加载到内存中，然后从$ORACLE_HOME/bin目录输入命令dbca 。
  • 使用DBCA时的群集检测和节点选择
    当您启动DBCA时，如果它从中央Oracle Inventory检测到Oracle主机已启用Oracle RAC，它将自动显示Oracle RAC选项。
  • 使用DBCA选择要与任何Oracle RAC数据库一起使用的存储
    您可以选择使用Oracle ASM磁盘组或支持的群集文件系统作为存储。
  • 使用DBCA选择服务器池选项
    使用Database Configuration Assistant（DBCA）创建数据库时，可以选择服务器池来托管策略管理的数据库。
  • 使用DBCA为Oracle RAC指定数据库初始化参数
    如果要在群集中添加更多节点，请指定初始化参数。
  • DBCA针对Oracle RAC数据库执行的操作
    查看此信息以了解Oracle RAC数据库创建期间的DBCA操作。
