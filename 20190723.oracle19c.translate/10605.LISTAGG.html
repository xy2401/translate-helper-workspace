目的

对于指定的度量， LISTAGG对ORDER BY子句中指定的每个组内的数据进行LISTAGG ，然后连接度量列的值。

  • 作为单集聚合函数， LISTAGG对所有行进行操作并返回单个输出行。

  • 作为一个组集聚合，该函数对GROUP BY子句定义的每个组进行操作并返回一个输出行。

  • 作为分析函数， LISTAGG根据query_partition_clause中的一个或多个表达式将查询结果集划分为组。

函数的参数遵循以下规则：

  • ALL关键字是可选的，提供语义清晰度。

  • measure_expr是度量列，可以是任何表达式。度量列中的空值将被忽略。

  • delimiter指定用于分隔度量列值的字符串。此子句是可选的，默认为NULL 。

    如果measure_expr的类型为RAW ，则分隔符必须是RAW类型。可以通过指定的分隔符作为一个字符的字符串，可以隐式转换来实现这一RAW ，或由分隔符明确地转换为RAW ，例如，使用UTL_RAW.CAST_TO_RAW功能。

  • order_by_clause确定返回连接值的顺序。仅当ORDER BY列列表实现唯一排序时，该函数才是确定性的。

DISTINCT关键字从列表中删除重复值。

如果度量列的类型为RAW ，则返回数据类型为RAW 。否则，返回数据类型为VARCHAR2 。

返回数据类型的最大长度取决于MAX_STRING_SIZE初始化参数的值。如果MAX_STRING_SIZE = EXTENDED ，则VARCHAR2和RAW数据类型的最大长度为32767字节。如果MAX_STRING_SIZE = STANDARD ，则VARCHAR2数据类型的最大长度为4000字节， RAW数据类型的最大长度为2000字节。在确定返回值是否适合返回数据类型时，不包括最终分隔符。

listagg_overflow_clause

此子句控制当返回值超过返回数据类型的最大长度时函数的行为方式。

ON OVERFLOW ERROR如果指定此子句，则该函数返回ORA-01489错误。这是默认值。

ON OVERFLOW TRUNCATE如果指定此子句，则该函数返回一个截断的度量值列表。

  • truncation_indicator指定要附加到度量值的截断列表的字符串。如果省略此子句，则截断指示符是省略号（ ... ）。

    如果measure_expr的类型为RAW ，则截断指示符必须是RAW类型。可以通过指定截断指示器为一个字符的字符串，可以隐式转换来实现这一RAW ，或者通过截断指示器明确地转换为RAW ，例如，使用UTL_RAW.CAST_TO_RAW功能。

  • 如果指定WITH COUNT ，则在截断指示符之后，数据库会附加括在括号中的截断值的数量。在这种情况下，数据库会截断足够的度量值，以允许最终分隔符，截断指示符的返回值中的空格，以及括号中括起的数字值的24个字符。

  • 如果指定WITHOUT COUNT ，则数据库将忽略返回值中截断值的数量。在这种情况下，数据库会截断足够的度量值，以允许最终分隔符和截断指示符的返回值中的空格。

如果未指定WITH COUNT或WITHOUT COUNT ，则默认值为WITH COUNT 。
