2安装示例模式

从Oracle Database 12c第2版开始，最新版本的示例模式脚本可以在GitHub上获得， 网址为https://github.com/oracle/db-sample-schemas/releases/latest 。

在完整安装Oracle数据库期间，可以使用dbca选项在创建数据库时手动或自动安装HR模式。必须通过GitHub上提供的脚本手动安装所有其他示例模式。

本章包含以下主题：

  • 仅安装HR模式

  • 从GitHub安装示例模式

注意：

通过安装任何Oracle数据库示例模式，您将删除使用以下用户名的任何以前安装的模式： HR ， OE ， PM ， SH ， IX ， BI 。

如果您运行本节中描述的任何安装脚本，则任何这些模式中包含的数据都将丢失。您不应将示例模式用于个人或业务数据和应用程序。它们仅用于演示目的。

2.1.1使用Database Configuration Assistant安装HR模式

选择sample schemas选项以在数据库中安装HR模式。

在安装过程结束时，会出现一个对话框，显示已创建的帐户及其锁定状态。默认情况下，示例模式已锁定且密码已过期。在使用锁定帐户之前，您必须解锁并重置密码。您可以在安装过程中解锁帐户。或者，安装完成后，您可以使用ALTER USER ...解锁模式并重置其密码ALTER USER ...ACCOUNT UNLOCK声明。例如：

通过Password识别的ALTER USER hr帐户解锁;

如果需要删除HR模式，请在SQL * Plus命令行上运行以下脚本。

sqlplus system / systempw @connect_string @ drop_hr.sql

2.2从GitHub安装示例模式

从Oracle Database 12c第2版开始， $ORACLE_HOME/demo/schema/human_resources目录中只提供了HR示例模式SQL脚本。如果要使用HR以外的示例模式，例如OE, OC, PM和SH模式，则必须从GitHub存储库下载它们。

从GitHub安装示例模式的过程如下：

  1. 要查找最新版本的示例模式安装脚本，请访问以下GitHub网站： https ： //github.com/oracle/db-sample-schemas/releases/latest

    例如，如果您需要12.2.0.1版本的脚本，请转到https://github.com/oracle/db-sample-schemas/releases/tag/v12.2.0.1

  2. 克隆GitHub存储库，或从GitHub下载ZIP包并提取文件。

  3. 解压缩文件。

  4. 按照说明在zip文件中包含的README中创建模式。

本节包括以下主题：

2.2.1重置样本模式

要将示例模式重置为其初始状态，请使用SQL * Plus命令行界面中的以下语法：

sqlplus system / systempw @connect_string @mksample systempw syspw hrpw oepw pmpw ixpw shpw bipw users temp / your / path / to / log / connect_string

mksample脚本需要11个参数。提供SYSTEM和SYS的密码，以及模式HR ， OE ， PM ， IX和SH的密码。指定临时表和默认表空间，并确保使用尾部斜杠结束日志文件目录的名称。

mksample脚本生成多个日志文件：

  • mkverify.log是示例模式创建日志文件。

  • hr_main.log是HR模式创建日志文件。

  • oe_oc_main.log是OE模式创建日志文件。

  • pm_main.log是PM模式创建日志文件。

  • pm_p_lob.log是在SQL * Loader日志文件PM.PRINT_MEDIA 。

  • ix_main.log是IX模式创建日志文件。

  • sh_main.log是SH模式创建日志文件。

  • cust.log是在SQL * Loader日志文件SH.CUSTOMERS 。

  • prod.log是在SQL * Loader日志文件SH.PRODUCTS 。

  • promo.log是在SQL * Loader日志文件SH.PROMOTIONS 。

  • sales.log是SH.SALES的SQL * Loader日志文件。

  • sales_ext.log是外部表的日志文件SH.COSTS 。

在大多数情况下，首次安装示例模式或在先前安装的版本上重新安装示例模式之间没有区别。*_main.sql脚本删除模式用户及其所有对象。

2.2.2卸载示例模式

如果需要从安装中删除示例模式， drop_sch.sql在SQL * Plus命令行上运行脚本drop_sch.sql 。此脚本随Oracle数据库一起提供。

此脚本使用以下参数：

  • systempwd

  • 系统

  • CONNECT_STRING

systempwd是SYSTEM用户的密码， connect_string是数据库的连接字符串。

示例2-1如何卸载示例模式

sqlplus system / systempw @connect_string @drop_sch.sql
