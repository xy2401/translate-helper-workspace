13.1条款可以进入

ACCESSIBLE BY子句限制对单元和子程序的访问。

ACCESSIBLE BY子句限制其他单元对单元或子程序的访问。访问者列表 （也称为白名单）明确列出了可能具有访问权限的那些单元。可以在包中的各个子程序上定义访问者列表。除了在包本身上定义的访问者列表（如果有的话）之外，还检查该列表。此列表可能仅限制对子程序的访问 - 它无法扩展访问权限。此代码管理功能可用于防止无意中使用内部子程序。例如，将包重组为两个包可能不方便或不可行：一个用于需要限制访问的少量过程，另一个用于需要公共访问的其余单元。

ACCESSIBLE BY子句可能出现在对象类型，对象类型主体，包和子程序的声明中。

ACCESSIBLE BY子句可以出现在以下SQL语句中：

  • ALTER TYPE语句

  • 创建功能声明

  • CREATE PROCEDURE声明

  • CREATE PACKAGE声明

  • CREATE TYPE语句

  • CREATE TYPE BODY声明

语义

accessible_by_clause

访问

[ schema 。] unit_name

指定可以调用实体的已存储PL / SQL单元。

每个accessor指定可以访问包含ACCESSIBLE BY子句的实体的另一个PL / SQL实体。

当出现ACCESSIBLE BY子句时，只有该子句中指定的实体可以访问该子句出现的实体。

accessor可能在ACCESSIBLE BY子句中出现多次。

ACCESSIBLE BY子句只能在单元声明中出现一次。

不需要存在在accessor命名的实体。

当调用具有ACCESSIBLE BY子句的实体时，它会在执行所有其他检查后强制执行其他访问检查。这些检查是：

  • 被调用的单元必须包括一个accessor以相同的UNIT_NAME和unit_kind作为调用单元。

  • 如果accessor包含模式 ，则调用单元必须位于该模式中 。

  • 如果accessor不包含模式 ，则调用者必须来自与调用实体相同的模式。

unit_kind

指定单位是FUNCTION ， PACKAGE ， PROCEDURE ， TRIGGER还是TYPE 。

使用说明

unit_kind是可选的，但建议在单位具有相同名称时指定它以避免歧义。例如，可以定义与函数同名的触发器。

只有当呼叫是直接的时， ACCESSIBLE BY子句才允许访问。如果通过静态SQL， DBMS_SQL或动态SQL进行访问，则检查将失败。

将根据包规范的访问者列表检查对包规范或包体的初始化过程的任何调用。

一个单位总是可以访问自己。单元中的项目可以引用同一单元中的另一个项目。

对受保护子程序的RPC调用将始终失败，因为在编译时或运行时没有可用于检查调用有效性的上下文。

从条件编译指令调用受保护的子程序将失败。

示例13-1限制对同一模式中的顶级过程的访问

此示例显示顶级过程top_protected_proc只能由当前模式中的过程top_trusted_proc调用。用户无法直接调用top_proctected_proc。

实时SQL：

您可以在限制访问同一模式中的顶级过程的 Oracle Live SQL上查看和运行此示例

程序top_protected_proc ACCESSIBLE BY（PROCEDURE top_trusted_proc） AS BEGIN DBMS_OUTPUT.PUT_LINE（'执行top_protected_proc。“）;结束;程序top_trusted_proc AS BEGIN DBMS_OUTPUT.PUT_LINE（'top_trusted_proc调用top_protected_proc'）; top_protected_proc;结束; EXEC top_trusted_proc; top_trusted_proc调用top_protected_proc执行top_protected_proc。EXEC top_protected_proc; BEGIN top_protected_proc;结束; PLS-00904：访问对象TOP_PROTECTED_PROC的权限不足

示例13-2限制对任何类型的单元名称的访问

此示例显示如果未在ACCESSIBLE ACCESSIBLE BY子句中指定PL / SQL unit_kind ，则在单元名称匹配时允许来自任何单位类型的调用。如果ACCESSIBLE ACCESSIBLE BY子句中指定的unit_kind与任何现有对象都不匹配，则不会出现编译错误。可以定义与函数同名的触发器。建议指定unit_kind 。

示例13-3限制对存储过程的访问

此示例显示了只能由top_trusted_proc过程调用的程序包过程。子程序规范和正文的ACCESSIBLE BY子句必须匹配。如果使用在其访问者列表中不包含此过程的ACCESSIBLE BY子句调用现有过程，则会引发编译错误。
