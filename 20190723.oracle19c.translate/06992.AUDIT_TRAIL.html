1.25 AUDIT_TRAIL

AUDIT_TRAIL启用或禁用数据库审计。

属性 描述

参数类型

串

句法

AUDIT_TRAIL = {无| os | db [，扩展] | xml [，扩展]}

默认值

没有

可修改

没有

可在PDB中修改

没有

基本

没有

注意：

在已迁移到统一审计的Oracle数据库中，此参数的设置无效。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

值

  • 没有

    禁用标准审核。如果未在初始化参数文件中AUDIT_TRAIL参数，或者使用Database Configuration Assistant以外的方法创建数据库，则此值是缺省值。如果使用Database Configuration Assistant创建数据库，则默认值为db 。

  • 口

    将所有审核记录定向到操作系统文件。Oracle建议您使用os设置，尤其是在使用超安全数据库配置时。

  • D b

    将审计记录定向到数据库审计跟踪（ SYS.AUD$表），但始终写入操作系统审计跟踪的记录除外。将此设置用于通用数据库以实现可管理性。

    如果数据库以只读模式启动且AUDIT_TRAIL设置为db ，则Oracle数据库在内部将AUDIT_TRAIL为os 。查看警报日志以获取详细信息。

  • db，扩展

    执行AUDIT_TRAIL = db所有操作，并且还填充SYS.AUD$表的SQL绑定和SQL文本CLOB类型列（如果可用）。仅在指定此参数时才填充这两列。当标准审计与DB, EXTENDED ，虚拟专用数据库（VPD）谓词和策略名称也会填充在SYS.AUD$表中。

    如果数据库以只读模式启动，并且AUDIT_TRAIL设置为db, extended ，则Oracle Database会在内部将AUDIT_TRAIL为os 。查看警报日志以获取详细信息。

  • XML

    以XML格式写入操作系统审计记录文件。将除了Sql_Text和Sql_Bind之外的AuditRecord节点的所有元素记录到操作系统XML审计文件中。

  • xml，扩展

    执行AUDIT_TRAIL = xml所有操作，并在审计跟踪中包含SQL文本和SQL绑定信息。

无论此参数的设置如何，您都可以使用SQL AUDIT语句设置审核选项。

在CDB中，此初始化参数的设置范围是CDB。虽然在CDB中按PDB提供审计跟踪，但是不能为单个PDB配置此初始化参数。

例子

以下语句设置AUDIT_TRAIL参数的db, extended值。重新启动数据库后，新值将生效。 

SQL> alter system set AUDIT_TRAIL = db，extended scope = spfile;系统改变了。SQL>

以下语句设置AUDIT_TRAIL参数的xml, extended值。重新启动数据库后，新值将生效。 

SQL> alter system set AUDIT_TRAIL = xml，extended scope = spfile;系统改变了。SQL>

以下语句设置AUDIT_TRAIL参数的db值。重新启动数据库后，新值将生效。 

SQL> alter system set AUDIT_TRAIL = db scope = spfile;系统改变了。SQL>
