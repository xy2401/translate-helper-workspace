B用于SQL *加载程序，导出和导入的Instant Client

Oracle Instant Client允许您在不安装标准Oracle客户端或拥有Oracle主目录的情况下运行应用程序。

B.1什么是Tools Instant Client？

Tools Instant Client软件包可在支持OCI Instant Client的平台上使用。

Tools包包含多个命令行实用程序，包括SQL * Loader，Data Pump Export，Data Pump Import，Original（经典）Export和Original（经典）Import。Instant Client安装是独立的，具有产品命令行版本的所有功能。Instant Client连接到现有的远程Oracle数据库，但不包括自己的数据库。与使用命令行版本的产品所需的完整Oracle数据库客户端安装相比，它易于安装并且使用的磁盘空间显着减少。

有关OCI Instant Client的详细信息，请参阅Oracle Call Interface Programmer's Guide 。

要使用Tools Instant Client，您需要两个包：

  • 工具即时客户端包

  • Basic OCI Instant Client软件包或OCI Instant Client Light软件包。

使用Tools Instant Client所需的基本步骤如下。本附录中描述了每个步骤。

  1. 选择要使用的OCI包（基本或轻量级），并选择安装Instant Client文件的目录。

  2. 从已安装的Oracle实例复制Tools Instant Client Package和您选择的OCI Instant Client软件包，或从OTN下载它们。

  3. 安装（解压缩）Tools Instant Client程序包和OCI程序包。将创建一个新目录instantclient_12_2作为安装的一部分。

  4. 配置Instant Client。

  5. 使用要运行的实用程序连接到远程实例。

Tools包和OCI包都必须来自Oracle Database 12.2.0.0.0或更高版本，并且两者的版本必须相同。

B.2选择要安装的Instant Client

在安装Tools Instant Client Tools软件包之前，必须先选择Basic Instant Client或Instant Client Light。

基本即时客户端

与Instant Instant Client一起使用时，Tools Instant Client程序包可与Oracle数据库支持的任何NLS_LANG设置一起使用。它支持Oracle数据库中可用的所有字符集和语言设置。

即时客户端灯

Instant Client Light（英文）版本的Instant Client进一步减少了客户端安装的磁盘空间要求。通过删除除英语以外的语言的错误消息文件并且仅保留约250个支持的字符集定义，可以减小库的大小。

Instant Client Light适用于使用US7ASCII ， WE8DEC ， WE8ISO8859P1 ， WE8MSWIN1252或Unicode字符集的应用程序。NLS_LANG设置的LANGUAGE和TERRITORY字段没有限制，因此Instant Client Light可以使用任何语言和区域设置进行操作。由于Instant Client Light仅提供英语错误消息，因此即使NLS_LANG设置为AMERICAN以外的语言，客户端生成的错误消息（如网络连接错误）也始终以英语报告。如果在数据库实例的Oracle主目录中安装了相应的已翻译消息文件，则数据库端生成的错误消息（如SQL语句中的语法错误）采用所选语言。

B.3从OTN下载安装Tools Instant Client

Linux的OTN下载是RPM包。UNIX和Windows的OTN下载是zip文件。

永远不要在Oracle主目录上安装Instant Client软件包。

从Linux RPM软件包安装Instant Client

  1. 从OTora Instant Client页面下载包含Tools Instant Client软件包的RPM软件包和OCI软件包， 网址为http://www.oracle.com/technetwork/database/database-technologies/instant-client/overview/index.html 。两个软件包必须是12.2.0.0.0或更高版本，并且两者的版本必须相同。

  2. 使用rpm -i进行RPM软件包的初始安装，或使用rpm -u升级到更新版本的软件包。首先安装OCI包。

  3. 配置Instant Client。请参阅配置工具Instant Client Package 。

从UNIX或Windows Zip文件安装Instant Client

  1. 从OTora Instant Client页面下载包含Tools Instant Client软件包的RPM软件包和OCI软件包， 网址为http://www.oracle.com/technetwork/database/database-technologies/instant-client/overview/index.html 。两个软件包必须是12.2.0.0.0或更高版本，并且两者的版本必须相同。

  2. 创建一个新目录，例如UNIX上的/home/instantclient12_2或Windows上的c:\instantclient12_2 。

  3. 将两个包解压缩到新目录中。首先安装OCI包。

  4. 配置Instant Client。请参阅配置工具Instant Client Package

B.4从12 c客户端版本媒体安装Tools Instant Client

可以从客户端发布介质安装Tools Instant Client程序包。

  1. 在Oracle Database 12 c Client Release介质上运行安装程序，然后选择“管理员”选项。

  2. 在UNIX和Linux上创建新目录，例如/home/instantclient12_2 ，或在Windows上创建c:\instantclient12_2 。

  3. 将Tools Instant Client程序包和OCI Instant Client文件复制到新目录。必须从同一Oracle主目录复制所有文件。

  4. 配置Tools Instant Client程序包。请参阅配置工具Instant Client Package 。

B.5配置工具Instant Client Package

Tools Instant Client程序包可执行文件仅应与OCI Instant Client的匹配版本一起使用。不需要设置ORACLE_HOME或ORACLE_SID环境变量。

在Linux上配置Tools Instant Client Package（来自RPMS）

从OTN下载的RPM安装到/usr文件系统中的Oracle特定子目录中。子目录结构允许多个版本的Instant Client可用。

  1. 将包含Instant Client库的目录的名称添加到LD_LIBRARY_PATH 。删除所有其他Oracle目录。

    例如，要在Bourne或Korn shell中设置LD_LIBRARY_PATH ，请使用以下语法：

    LD_LIBRARY_PATH = / usr / lib / oracle / 18.1 / client / lib：$ {LD_LIBRARY_PATH} export LD_LIBRARY_PATH

    或者，要在C shell中设置LD_LIBRARY_PATH ，请使用以下语法：

    ％setenv LD_LIBRARY_PATH /usr/lib/oracle/18.1/client/lib:$LD_LIBRARY_PATH

  2. 确保从RPM安装的Tools可执行文件是PATH找到的第一个可执行文件。例如，为了测试这个，你可以进入which sqlldr应返回/usr/bin/sqlldr 。如果没有，则从PATH删除任何其他Oracle目录，或将/usr/bin放在PATH其他Tools可执行文件之前，或使用绝对或相对路径启动Tools Instant Client。

    例如，要在bash shell中设置PATH ：

    PATH = / usr / bin：$ {PATH}导出PATH

  3. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。

    NLS_LANG = AMERICAN_AMERICA.UTF8导出NLS_LANG

在Linux和UNIX上配置工具Instant Client Package（来自Client Media或Zip File）

  1. 将包含Instant Client文件的目录的名称添加到相应的共享库路径LD_LIBRARY_PATH ， LIBPATH或SHLIB_PATH 。删除所有其他Oracle目录。

    例如，在Bourne或Korn shell中的Solaris上：

    LD_LIBRARY_PATH = / home / instantclient12_2：$ {LD_LIBRARY_PATH} export LD_LIBRARY_PATH

  2. 将包含Instant Client文件的目录添加到PATH环境变量中。如果未设置，则必须使用绝对路径或相对路径来启动Tools包中提供的实用程序。从PATH删除任何其他Oracle目录。例如：

    PATH = / home / instantclient12_2：$ {PATH}导出PATH

  3. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。例如：

    NLS_LANG = AMERICAN_AMERICA.UTF8导出NLS_LANG

在Windows上配置Tools Instant Client Package

可以使用Windows命令提示符中的SET命令配置环境，也可以通过在“系统属性”中设置“环境变量”使其成为永久环境。

例如，要使用“系统属性”在Windows 2000中设置环境变量，请从“控制面板”中打开“ 系统 ”，单击“ 高级”选项卡，然后单击“ 环境变量” 。

  1. 将包含Instant Client文件的目录添加到PATH系统环境变量中。从PATH删除任何其他Oracle目录。

    例如，将c:\instantclient12_2添加到PATH的开头。

  2. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。

    例如，要为日语环境设置NLS_LANG ，请创建设置为JAPANESE_JAPAN.JA16EUC的用户环境变量NLS_LANG 。

B.6使用Tools Instant Client Package连接到数据库

安装并配置Tools Instant Client软件包后，即可连接到数据库。

Tools Instant Client中提供的实用程序始终远离任何数据库服务器。假设服务器已启动并运行Oracle实例并且正在运行TNS侦听器。对于Data Pump导出和导入客户端，转储文件驻留在远程服务器上;服务器上的Oracle目录对象必须存在，并且应具有相应的权限。

要连接到数据库，必须使用Oracle Net连接标识符指定数据库。以下信息使用SQL * Loader（sqlldr）实用程序，但该信息也适用于Tools Instant Client程序包中提供的其他实用程序。

使用Easy Connection标识符连接到mymachine上运行的MYDB数据库中的HR模式的示例是：

SQLLDR小时/ your_password @ \ “MYDOMAIN // mymachine上 ： 端口 / MYDB \”。

或者，您可以使用网络服务名称：

sqlldr hr / your_password @MYDB

网络服务名称可以存储在许多地方，包括LDAP。建议使用LDAP来利用Oracle Database 12 c的新功能

要使用在本地Oracle Net tnsnames.ora文件中配置的网络服务名称，请将环境变量TNS_ADMIN为包含tnsnames.ora文件的目录。例如，在UNIX上，如果您的tnsnames.ora文件位于/home/user1并且它定义了网络服务名称MYDB2 ：

TNS_ADMIN = / home / user1导出TNS_ADMIN sqlldr hr @ MYDB2

如果未设置TNS_ADMIN ，则检查依赖于操作系统的目录集以查找tnsnames.ora 。此搜索路径包括查找由network/admin/tnsnames.ora的ORACLE_HOME环境变量指定的目录。这是为SQL * Loader Instant Client设置ORACLE_HOME环境变量的唯一原因。如果在运行Instant Client应用程序时设置了ORACLE_HOME ，则必须将其设置为存在的目录。

此示例假定已设置ORACLE_HOME环境变量，并且$ORACLE_HOME/network/admin/tnsnames.ora或ORACLE_HOME\network\admin\tnsnames.ora文件定义网络服务名称MYDB3 ：

sqlldr hr @ MYDB3

可以将TWO_TASK （在UNIX上）或LOCAL （在Windows上）环境变量设置为连接标识符。这样就无需在SQL * Loader或SQL * Loader Instant Client中建立连接时显式输入连接标识符。此UNIX示例连接到称为MYDB4的数据库：

TNS_ADMIN = / home / user1 export TNS_ADMIN TWO_TASK = MYDB4 export TWO_TASK sqlldr hr

在Windows上，可以在“系统属性”中设置TNS_ADMIN和LOCAL 。

B.7卸载Instant Client

Tools Instant Client软件包可以与OCI Instant Client分开删除。

卸载Tools Instant Client软件包后，剩余的OCI Instant Client库会启用自定义编写的OCI程序或第三方数据库实用程序以连接到数据库。

卸载工具Instant Client

  1. 对于Linux上从RPM软件包安装，请仅在Tools Instant Client软件包上使用rpm -e

    要么

    对于UNIX和Windows上的安装以及从客户端发行介质安装Linux，请手动删除特定于Tools Instant Client的所有文件。要删除的文件应位于安装时指定的Instant Client目录中。确保不删除任何Oracle主文件。

    重置环境变量并在必要时删除tnsnames.ora 。

卸载Complete Instant Client

  1. 对于Linux上从RPM软件包安装，请使用rpm -qa查找Tools Instant Client和Basic OCI软件包名称，并运行rpm -e以删除它们

    要么

    对于UNIX和Windows上的安装以及来自客户端发行介质的Linux上的安装，请手动删除包含Tools可执行文件和Oracle库的目录。

  2. 重置环境变量，例如PATH ， LD_LIBRARY_PATH和TNS_ADMIN 。

  3. 如有必要，删除tnsnames.ora 。
