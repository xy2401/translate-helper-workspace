5创建Oracle Label Security策略

Oracle Label Security策略是一组命名的实现Oracle Label Security的命令。

  • 关于创建Oracle Label Security策略
    创建Oracle Label Security策略时，必须遵循一组常规步骤。
  • 第1步：创建标签安全策略容器
    Oracle Label Security策略容器是策略设置的存储位置。
  • 第2步：为标签安全策略创建数据标签
    创建策略容器后，您就可以为每个数据库表行创建数据标签。
  • 第3步：授权用户使用标签安全策略
    在用户可以访问受Oracle Label Security策略保护的数据之前，必须先授权他们。
  • 步骤4：向用户和受信任的存储程序单元授予权限
    您可以向用户授予权限，例如READ以便用户可以读取受Oracle Label Security策略保护的数据。
  • 步骤5：将策略应用于数据库表或架构
    在为Oracle Label Security策略创建授权授权和特权之后，可以将其应用于数据库表或模式。
  • 第6步：将策略标签添加到表行
    您必须向表行添加策略标签。
  • 步骤7 :(可选）配置审核
    您可以使用SA_USER_ADMIN P / L SQL包来审核Oracle Label Security策略。
  • 使用Enterprise Manager Cloud Control创建OLS策略
    您可以在Oracle Enterprise Manager云控制中创建Oracle Label Security策略。

关于创建Oracle Label Security策略

创建Oracle Label Security策略时，必须遵循一组常规步骤。

  1. 创建一个策略容器，用于定义策略名称，Oracle Label Security将添加到要保护的表的列的名称，是否隐藏此列，是否启用策略以及策略的默认强制选项。

    有关更多信息，请参阅步骤1：创建标签安全策略容器 。

  2. 为标签定义以下属性：灵敏度级别，以及可选的隔区和组，以进一步过滤标签敏感度。定义属性后，创建标签本身，然后将这些属性与标签相关联。

    请参阅步骤2：为标签安全策略创建数据标签 。

  3. 授权用户使用该策略。

    有关详细信息，请参阅步骤3：为用户授予标签安全策略 。

  4. 为这些用户或受信任的程序单元授予权限。

    有关详细信息，请参阅步骤4：为用户和可信存储的程序单元授予权限 。

  5. 将策略应用于数据库表。或者，您可以将策略应用于整个架构。

    有关更多信息，请参阅步骤5：将策略应用于数据库表或架构 。

  6. 将策略标签添加到表行。您必须更新用于策略的表。

    有关详细信息，请参阅步骤6：向表行添加策略标签 。

  7. （可选）配置用户的审核设置。

    有关更多信息，请参阅步骤7 :(可选）配置审核 。

父主题： 创建Oracle Label Security策略

关于标签安全策略容器

Oracle Label Security策略容器存储描述策略行为方式的元数据。

此容器定义策略名称，Oracle Label Security将添加到要保护的表的列的名称，是否隐藏此列以及策略的默认强制选项。

添加到要保护的表的列将包括基于特定列中的值分配给表中特定行的数据标签（稍后创建）。策略创建过程为策略创建特殊角色，并将此角色授予创建策略的用户。角色名称采用policy _DBA格式。例如，对于名为EMP_OLS_POL的策略，角色名称为EMP_OLS_POL_DBA 。只有在新用户会话开始后，此角色才会生效。

您可以在Oracle Enterprise Manager Cloud Control中创建策略容器，也可以使用SA_SYSDBA.CREATE_POLICY过程。

创建标签策略容器

您可以使用SA_SYSDBA.CREATE_POLICY过程来创建Oracle Label Security策略容器。

  • 要创建策略，请运行SA_SYSDBA.CREATE_POLICY ，指定策略名称，列名称和默认选项。

例如：

BEGIN SA_SYSDBA.CREATE_POLICY（policy_name =>'emp_ols_pol'，column_name =>'ols_col'，default_options =>'read_control，update_control'）;结束; /

关于策略级别敏感性组件

级别是表示其标记的信息的敏感度的排名。

信息越敏感，其级别越高。信息越不敏感，其级别越低。

数字形式

数字形式的范围为0到9999。它与用于级别的数字无关。隔间的数字形式并不表示灵敏度更高或更低。相反，它控制标签字符串中的短格式隔间名称的显示顺序。例如，假设创建了一个标签，其中包含表5-4中列出的所有三个隔离专区，以及一个SENSITIVE级别。当此标签以字符串格式显示时，它看起来像这样：

关于将策略组件与命名数据标签关联

定义数据标签组件后，您可以通过将其与现有级别相关联来创建数据标签。

（可选）您可以在此关联中包含隔离专区和组。

您可以使用Oracle Enterprise Manager云控制或SA_LABEL_ADMIN.CREATE_LABEL过程。标签的字符串表示使用以下语法：

level ： compartment1 ，...， compartmentn ： group1 ，...， groupn

指定标签的文本字符串最多可包含4,000个字符，包括字母数字字符，空格和下划线。标签名称不区分大小写。您可以以大写，小写或大小写形式输入它们，但字符串存储在数据字典中并以大写形式显示。用冒号分隔每组组件。您不需要在此语法中输入尾随分隔符。

例如，您可以创建有效的标签，例如：

敏感：金融，化学：EASTERN_REGION，WESTERN_REGION CONFIDENTIAL：财务：VP_GRP敏感度高度敏感：财务敏感:: WESTERN_REGION

父主题： 步骤2：为标签安全策略创建数据标签

将策略组件与命名数据标签相关联

SA_LABEL_ADMIN.CREATE_LABEL过程创建数据标签。

  • 运行SA_LABEL_ADMIN.CREATE_LABEL ，指定策略名称和有关策略组件的详细信息。

例如：

BEGIN SA_LABEL_ADMIN.CREATE_LABEL（policy_name =>'emp_ols_pol'，label_tag =>'1310'，label_value =>'S：FINCL，CHEM：ER，WR'，data_label => TRUE）;结束; /

在此示例中， label_value设置采用简短形式，转换为以下长形式：

敏感：金融，化工：EASTERN_REGION，WESTERN_REGION

创建数据标签时，会发生另外两个操作：

  • 标签自动指定为有效数据标签。此功能限制了可以分配给数据的标签。Oracle Label Security还可以在运行时动态创建有效的数据标签，这些标签来自Oracle Internet Directory中预定义的标签。但是，大多数用户更喜欢手动创建标签以限制数据标签的扩散。

  • 数字标签标签与表示标签的文本字符串相关联。这是标签标签，而不是文本字符串，它存储在受保护表的策略标签列中。

    注意：

    对于不使用Oracle Internet Directory的Oracle Label Security安装，动态创建有效数据标签使用TO_DATA_LABEL函数。它的使用应该严格控制。

父主题： 创建Oracle Label Security策略

关于将策略应用于数据库表或架构

将策略应用于表时，将自动启用该策略。

禁用策略是关闭其保护，尽管它仍然应用于表。启用策略是为特定表或模式打开并强制执行其保护。

删除策略是将其完全从表或模式中删除。但请注意，除非您明确删除它们，否则策略标签列和标签仍保留在表中。

您可以更改可能在架构中创建的将来表的默认策略强制选项。但是，这不会影响架构中现有表的策略实施选项。

要更改现有表的强制选项，必须先从表中删除策略，进行所需的更改，然后将策略重新应用于表。

请注意，您无法在外部表上强制执行Oracle Label Security策略。

为策略组件创建策略组件并配置用户授权，权限和审核后，可以将策略应用于数据库表或整个模式。

将策略应用于数据库表时，除了策略名称和目标模式表之外，还必须指定以下信息：

  • table_options ：用于表的以逗号分隔的策略实施选项列表。如果为NULL ，则使用策略的默认选项。

  • label_function ：调用函数的字符串，用于返回标签值以用作默认值。例如， my_label(:new.dept,:new.status)根据行中DEPT和STATUS列的新值计算标签。

  • predicate ：用于组合（使用AND或OR ）与READ_CONTROL的基于标签的predicate的附加谓词

请注意将Oracle Label Security策略与模式一起使用的以下方面：

  • 如果将策略应用于空模式，则每次在该模式中创建表时，都会应用该策略。将策略应用于架构后，您选择的默认选项将应用于添加的每个表。

  • 如果从表中删除策略以使其不受保护，然后运行SA_POLICY_ADMIN.ENABLE_SCHEMA_POLICY ，则表将保持不受保护。如果您希望再次保护表，则必须将策略应用于表，或者将策略重新应用于架构。

如果将策略应用于已包含受策略保护的表的模式，则所有将来的表都将具有应用策略时指定的新选项。现有表格将保留已有的选项。

将策略应用于架构

您必须更新要添加策略标签的表。

  1. 要向表中添加数据标签，请在SQL * Plus中使用以下语法输入UPDATE语句：
    UPDATE table_name SET ols_column = CHAR_TO_LABEL（' ols_policy '，' data_label '）WHERE UPPER（ table_column ）IN（ column_data ）;

    例如，假设LABCSYS已创建一个名为ACCESS_LOCATIONS的策略，并希望在HR.LOCATIONS表中将标签SENS添加到城市北京，东京和新加坡。策略标签列称为ROW_LABEL 。 UPDATE语句如下：

    更新地点设置ROW_LABEL = CHAR_TO_LABEL（'ACCESS_LOCATIONS'，'SENS'）上（城市）IN（'北京'，'东京'，'新加坡'）;
  2. 运行以下SELECT语句以确保将策略添加到表中：
    SELECT LABEL_TO_CHAR（ROW_LABEL）FROM LOCATIONS;
