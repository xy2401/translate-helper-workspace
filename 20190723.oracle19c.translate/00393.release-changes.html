Oracle Database 12 c第2版（12.2.0.1）中的更改

以下是Oracle Database 2 Day DBA for Oracle Database 12 c Release 2（12.2.0.1）中的更改。

新功能

Oracle Enterprise Manager Database Express（EM Express）中有几个新功能。

EM Express具有以下新功能：

  • 资源管理器支持多租户容器数据库（CDB），可插拔数据库（PDB）和非CDB。您可以创建，编辑，删除，激活和停用资源计划，以帮助您在这些数据库中有效地管理资源。

    有关EM Express中的资源管理器功能的信息，请参阅“ 管理数据库资源 ” 。

  • SQL性能分析器（SPA）和SQL性能分析器快速检查（SPA快速检查）支持，使您可以评估数据库更改对SQL性能的影响

    有关EM Express中SPA和SPA快速检查功能的信息，请参阅“ 分析数据库更改对SQL性能的影响 ” 。

  • 在CDB的根目录中，您可以配置单个端口（称为全局端口），这使您可以使用EM Express使用CDB的HTTPS端口连接到CDB中的所有PDB

    有关配置全局端口的信息，请参阅“ 为PDB启动EM Express ” 。

  • CDB和PDB的其他Performance Hub支持

    请参阅“ 使用Performance Hub监控性能 ”中的表，以获取可用于不同数据库（CDB，PDB和非CDB）的选项卡列表。

  • 支持管理新类型的PDB

    您可以使用EM Express管理可刷新的PDB，代理PDB，应用程序根和应用程序PDB。请参阅“ 关于可以在EM Express中管理的PDB ”和“ 访问应用程序根目录的容器页面 ” 。

    EM Express支持代理PDB的管理。代理PDB提供上下文以执行SQL语句并在代理PDB中执行远程PDB中的操作。

    在代理PDB中执行的所有语句（ ALTER DATABASE ， CREATE PLUGGABLE DATABASE ， ALTER PLUGGABLE DATABASE和DROP PLUGGABLE DATABASE语句除外）都将在远程PDB中执行。

  • 将远程CDB中的PDB热克隆到另一个CDB

    有关将PDB从远程CDB克隆到另一个CDB的信息，请参阅“ 通过使用EM Express从远程CDB克隆PDB来创建PDB ” 。

  • 将PDB从远程CDB重定位到另一个CDB，停机时间最短

    有关将PDB从远程CDB重定位到另一个CDB的信息，请参阅“重定位远程PDB ” 。

  • 创建可刷新的PDB

    有关在通过从远程CDB克隆现有PDB创建新PDB时使用“ 刷新选项”字段创建可刷新PDB的信息，请参阅“ 通过使用EM Express从远程CDB克隆PDB来创建PDB ” 。

  • 根据需要刷新可刷新的PDB

    有关刷新可刷新PDB的信息，请参阅“ 按需刷新PDB”。

  • 用于管理Oracle Data Guard备用数据库的新EM Express功能

    您可以使用EM Express：

    • 查看Performance Hub中备用数据库的实时性能统计信息

      有关查看备用数据库性能统计信息的信息，请参见“ 关于在备用数据库上查看性能统计信息” 。

    • 使用SQL Tuning Advisor调整备用数据库的SQL查询

      有关调整备用数据库的SQL语句的信息，请参见“ 关于调整备用数据库上的SQL语句 ”和“ 调整备用数据库上的 SQL语句”。

Oracle Database 12 c第1版（12.1.0.2）中的更改

以下是Oracle Database 2 Day DBA for Oracle Database 12 c Release 1（12.1.0.2）中的更改。

新功能

除管理非CDB之外，您还可以使用Oracle Enterprise Manager Database Express（EM Express）来管理多租户容器数据库（CDB）和可插拔数据库（PDB）。

  • 有关启动EM Express以管理CDB的信息，请参阅“ 为CDB启动EM Express ” 。

  • 有关启动EM Express以管理PDB的信息，请参阅“ 为PDB启动EM Express ” 。

  • 有关EM Express中用于管理PDB的功能的信息，请参阅“ 使用EM Express管理PDB”。许多这些功能都可以在CDB的Containers页面上找到。

EM Express为Oracle Database 12 c第1版（12.1.0.2）中引入的内存数据库功能提供支持：

  • “内存管理”页面上的“分配细分”和“分配历史记录”图表显示了“内存区域”信息。

  • CPU消耗量分为常规CPU使用率（CPU使用率与内存中操作无关）和内存中CPU使用率（内存中操作的CPU使用率）（如果适用）。这个CPU故障显示在几个地方：

    • 在“监视的SQL执行详细信息”页面上，显示CPU活动的任何位置。

    • 在“SQL详细信息”页面上的“活动％”列中。

    • 在Performance Hub的“活动”选项卡上，选择“按等级类消耗资源”时。

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle Database 2 Day DBA for Oracle Database 12 c Release 1（12.1.0.1）中的更改。

新功能

本节介绍本手册中针对Oracle Database 12 c第1版（12.1.0.1）所述的新功能。

  • Oracle Enterprise Manager Database Express（EM Express）是Oracle Database 12 c第1版（12.1）附带的用于管理和管理各个数据库的下一代管理工具。

    在以前的Oracle数据库版本中，Oracle Enterprise Manager数据库控制（DB Control）是主要的数据库管理工具。此版本中不包含DB Control。

    有关EM Express的更多信息，请参见“ Oracle Enterprise Manager Database Express简介 ” 。

  • SQL Developer用于管理模式对象。

    本书的早期版本描述了如何使用DB Control管理模式对象。此版本描述了如何使用SQL Developer管理架构对象。

    有关使用SQL Developer的详细信息，请参阅“ 管理架构对象 ” 。

  • Oracle Recovery Manager（RMAN）用于执行数据库备份和恢复操作。

    本书的早期版本描述了如何使用DB Control执行数据库备份和恢复操作。此版本描述了如何使用RMAN执行数据库备份和恢复操作。

    有关使用RMAN执行数据库备份和恢复的详细信息，请参阅“ 执行备份和恢复 ” 。
