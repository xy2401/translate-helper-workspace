Oracle Database 19c中VLDB和分区的更改

以下是Oracle数据库19c版19.1的超大型数据库和分区中的更改。

新功能

这些是Oracle数据库19c版19.1中的新功能，用于支持超大型数据库：

  • Oracle混合分区表

    Oracle混合分区表将经典内部分区表与外部分区表组合成一个更通用的分区，称为混合分区表。通过此功能，您可以轻松地将内部分区和外部分区（驻留在数据库外部的源上的分区）集成到单个分区表中。使用此功能还可以轻松地将非活动分区移动到外部文件，以获得更便宜的存储解决方案。

    混合分区表的分区可以驻留在Oracle表空间和外部源上，例如带有逗号分隔值（CSV）记录的Linux文件或带有Java服务器的Hadoop分布式文件系统（HDFS）上的文件。混合分区表支持外部分区的所有现有外部表类型： ORACLE_DATAPUMP ， ORACLE_LOADER ， ORACLE_HDFS ， ORACLE_HIVE 。

    并行分区SQL操作可以显着提高查询性能，从而提高响应时间。并行分区连接通常用于高效且快速地处理大型连接。

Oracle Database 12 c版本2（12.2.0.1）中VLDB和分区的更改

以下是Oracle数据库12 c版本2（12.2.0.1）中的超大型数据库和分区的更改。

  • 新功能

  • 不推荐的功能

新功能

这些是Oracle Database 12 c第2版（12.2.0.1）中支持超大型数据库的新功能：

  • 自动列表分区

    自动列表分区方法允许按需创建列表分区。自动列表分区表类似于常规列表分区表，但该分区表更易于管理。您可以仅使用已知的分区键值创建自动列表分区表。当数据加载到表中时，如果加载的分区键值与任何现有分区不对应，则数据库会自动创建新分区。由于分区是按需自动创建的，因此自动列表分区方法与现有的间隔分区方法类似。

    有关创建自动列表分区表的信息，请参阅创建自动列表分区表 。

  • 多列列表分区

    通过多列列表分区，您可以根据多列的列表值对表进行分区。与单列列表分区类似，各个分区可以包含包含值列表的集合。

    有关多列列表分区的信息，请参阅创建多列列表分区表 。

  • 自动列表分区和间隔子分区的延迟段创建

    自动列表复合分区表和间隔子分区仅在存在数据时创建子分区;这个动作节省了空间。在按需创建新分区时延迟创建子分区可确保仅在插入第一个匹配行时创建子分区段。

    有关创建自动列表分区表的信息，请参阅创建表和索引时指定分区 。

  • 只读分区

    您可以将表，分区和子分区设置为只读状态，以保护数据免受任何用户或触发器的无意DML操作。

    有关使用只读分区创建表的信息，请参阅使用只读分区或子分区创建表 。

  • 将非分区表转换为分区表

    可以将非分区表转换为分区表，并在ALTER TABLE SQL语句中添加MODIFY子句。此外，可以指定关键字ONLINE ，在转换过程中启用并发DML操作。

    有关在线转换为分区表的信息，请参阅将非分区表转换为分区表 。

  • 使用分区表创建Exchange表

    可以使用FOR EXCHANGE WITH子句创建表以与分区表的形状完全匹配，并且可以使用分区交换命令，除非索引不是作为此命令的操作创建的。由于此功能提供了非分区表和分区表之间的精确匹配，因此这是对CREATE TABLE AS SELECT语句的改进。

    有关创建用于与分区表交换的表的信息，请参阅使用分区表创建Exchange表 。

  • 过滤的分区维护操作

    分区维护操作支持添加数据过滤，实现分区和数据维护的组合。过滤的分区维护操作仅保留满足数据过滤的数据，作为分区维护的一部分。数据过滤的功能适用于MOVE PARTITION ， MERGE PARTITION和SPLIT PARTITION 。

    有关过滤维护操作的信息，请参阅过滤维护操作 。

  • 使用SPLIT操作进行在线分区维护

    支持SPLIT操作的分区维护作为联机操作，使用关键字ONLINE进行堆组织表，在分区维护操作正在进行时启用并发DML操作。

    有关拆分分区的信息，请参阅关于拆分分区和子分区 。

  • 创建分区外部表

    支持对外部表进行分区，以便对分区外部表上的查询进行更好的优化。

    有关创建分区外部表的信息，请参阅创建分区外部表 。

  • 区域地图

    区域映射是一种访问结构，它允许在表的扫描磁盘块期间根据其列上的谓词进行修剪。

    区域是一组连续的数据块，用于存储相关列的最小值和最大值。当SQL语句包含存储在区域中的列的谓词时，数据库会将谓词值与区域中存储的最小值和最大值进行比较，以确定在SQL执行期间要读取的区域。这显着提高了扫描的I / O和CPU成本。

    区域地图可以与属性聚类组合。属性集群表的主要优点是I / O修剪，可以显着降低表扫描的I / O成本和CPU成本。有关使用区域映射进行修剪的信息，请参阅使用区域映射进行分区修剪 。

    也可以看看：

    有关区域映射和属性群集的信息，请参见“Oracle数据库数据仓库指南”

    对于分区索引，您可以逐个分区地指定压缩类型。您还可以在索引分区上指定高级索引压缩，即使未压缩父索引也是如此。

    有关高级索引压缩和索引分区的信息，请参阅使用分区索引的高级索引压缩 。有关分区和表压缩的信息，请参阅分区和表压缩 。

    也可以看看：

    “Oracle数据库管理员指南” ，了解有关高级索引压缩的信息

    Oracle XML DB和使用域索引的其他应用程序可以使用散列分区方法。散列分区是在一系列分区上均匀地平衡I / O的有效方法。现在，XML DB的用户和使用域索引的其他应用程序可以使用此分区方法的优点。有关更多信息，请参阅散列分区 。

以下是Oracle数据库12 c第1版（12.1.0.1）的超大型数据库和分区中的更改。

新功能

这些是Oracle Database 12 c第1版（12.1.0.1）中的新功能，用于支持超大型数据库。

  • 多个分区的分区维护操作

    此功能可在多个分区上启用分区维护操作，从而简化应用程序开发并提高分区维护效率。

    多分区维护操作允许向表添加多个分区，删除多个分区，将多个分区合并为一个分区，将单个分区拆分为多个分区，以及使用单个SQL数据定义语言（DDL）语句截断多个分区。有关有效的维护操作的摘要，请参阅分区上支持的维护操作

    有关更多信息，请参阅添加多个分区 ， 删除多个分区 ， 合并多个分区 ， 拆分为多个分区以及截断多个分区 。

  • 热图

    此功能提供热数据和冷数据的热图，使您可以根据需要为数据库中的每一行指定存档状态，并帮助您实施信息生命周期管理（ILM）策略。可以根据其归档状态适当地管理数据库中的每一行，从而对数据库中所有数据的可见性，压缩和存储层进行细粒度控制。结合其他与ILM相关的功能，此功能使用户能够自动优化其数据库存储，从而最大限度地提高性能并最大限度地降低成本。

    您可以跟踪对行，段和表级别的数据的访问和修改操作。您还可以根据使用此功能跟踪的信息实施策略驱动的自动化。

    有关更多信息，请参阅使用热图和管理ILM热图和ADO与Oracle Enterprise Manager 。

    • 有关热图视图的信息，请参阅Oracle数据库参考

  • 自动数据优化

    此功能提供了用于在行，段和表空间级别指定策略的SQL语句选项，可帮助您实施信息生命周期管理（ILM）策略。

    您可以使用此功能自动在数据库中的不同存储层之间移动数据。这包括为每个层指定不同的压缩级别，以及控制数据移动发生的时间。您可以在数据库中的每个表中的行和段级别指定压缩。行级和段级压缩分层的组合提供了对数据库中数据存储和管理方式的细粒度控制。

    有关更多信息，请参阅使用自动数据优化和使用Oracle Enterprise Manager管理ILM热图和ADO 。

    也可以看看：

    • 有关ILM视图的信息，请参阅Oracle数据库参考

    通过数据库内存档和时间有效性功能，您可以管理实际情况下数据的有效性和可见性。

    有关更多信息，请参阅控制Oracle数据库中数据的有效性和可见性 。

  • 并行执行Union All

    此功能的主要好处是同时运行UNION或UNION ALL多个远程分支。

    有关信息，请参阅Union All的并发执行 。

  • 增量统计的增强功能

    增量统计已得到增强，以支持分区交换加载。加载到非分区表中的数据可以与表中的分区交换，Oracle可以自动准确地计算分区表的全局统计信息，使用非分区表和现有分区级统计信息中的统计信息。有关信息，请参阅关于交换分区和子分区 。

    此功能添加到现有并行语句排队功能。

    包含在此功能中：

    • 手动运行或取消排队的查询

      可以手动运行或取消排队查询。

    • 可插拔数据库（PDB）计划的并行服务器限制指令

      parallel_server_limit指令替换parallel_target_percentage指令。此伪指令指定在设置此伪指令时使用者组可以使用的可用并行服务器的百分比。

      对于多租户容器数据库（CDB）资源计划，此限制适用于可插拔数据库。对于PDB资源计划或非CDB资源计划，此限制适用于使用者组。

      有关更多信息，请参阅关于使用Oracle数据库资源管理器管理并行语句队列 。

        V$RSRC_SESSION_INFO.PQ_SERVERS指定此并行操作使用的并行服务器数。

        V$RSRC_SESSION_INFO.PQ_STATUS指定并行操作排队的原因。

  • 分区表的部分索引

    可以在表的分区的子集上创建本地和全局索引，从而在索引创建方面提供更大的灵活性。使用默认表索引属性支持此操作。创建或更改表时，可以为表或其分区指定默认索引属性。

    有关为分区表创建部分索引的信息，请参阅分区表的部分索引 。

  • DROP和TRUNCATE PARTITION的异步全局索引维护

    此功能使全局索引维护能够延迟并与DROP和TRUNCATE分区分离，而不会使全局索引不可用。增强功能包括更快的DROP和TRUNCATE分区操作以及将索引维护延迟到非高峰时间的能力。

    有关更多信息，请参阅“删除和截断分区的异步全局索引维护” 。

  • 区间参考分区

    此功能使引用分区表能够使用区间分区作为顶级分区策略，从而提供更好的分区建模。区间分区表可用作父表以进行引用分区。在插入到引用分区表中时，将创建与父表中的间隔分区对应的引用分区表中的分区。

    有关更多信息，请参阅创建间隔 - 参考分区表 。

  • TRUNCATE PARTITION和EXCHANGE PARTITION的级联功能

    此功能为参考和间隔引用分区表提供了TRUNCATE PARTITION和EXCHANGE PARTITION操作的CASCADE选项，这些操作将操作级联到引用分区子表。此功能通过启用从父表到子表的分区维护操作的继承，可以简化应用程序开发。默认情况下，级联选项处于关闭状态，因此它们不会影响兼容性。

    有关更多信息，请参阅关于使用级联选项交换分区和使用级联选项 截断分区 。

  • 在线移动分区

    此功能提供ALTER TABLE ..MOVE PARTITION选项，其中DML操作可以继续在正在移动的分区上不间断地运行。此外，在移动分区期间维护全局索引，因此不再需要手动索引重建。此功能消除了对ALTER TABLE MOVE PARTITION ONLINE命令的任何特殊停机时间的需要。

    有关更多信息，请参阅关于移动分区和子分区 。
