Fleet补丁和配置入门

了解如何开始使用并在标准部署中使用Fleet Patching和Provisioning。

如何开始进行Fleet补丁和配置？

您必须创建和配置Fleet Patching和Provisioning Server，您可以在其中创建和存储Oracle数据库，Oracle Grid Infrastructure和其他软件家庭的黄金映像。有关创建Fleet Patching和Provisioning Server的详细步骤，请参阅My Oracle Support说明2097026.1。

https://support.oracle.com/rs?type=doc&id=2097026.1

要设置Fleet Patching和Provisioning Server，请确保您的硬件满足My Oracle Support Note 2126710.1中所述的要求：

https://support.oracle.com/rs?type=doc&id=2126710.1

如何在标准操作环境中使用Fleet Patch和Provisioning？

查看这些主题，了解使用Fleet Patching and Provisioning配置，修补和升级软件的分步过程。

任务 描述
创建Oracle Grid Infrastructure 12 c第2版部署 在当前没有Grid主目录的两个节点上配置和配置Oracle Grid Infrastructure软件，然后配置Oracle Grid Infrastructure以形成多节点Oracle Grid Infrastructure安装。
配置Oracle数据库主页软件和创建数据库 配置Oracle Database 12 c第 2版（12.2）软件并创建Oracle数据库实例。
升级到Oracle Grid Infrastructure 12 c第2版 使用两个Fleet Patching和Provisioning命令将Oracle Grid Infrastructure集群从11 g版本2（11.2.0.4）升级到12 c版本2（12.2）。
在不更改Grid Home Path的情况下修补Oracle Grid Infrastructure 使用Fleet Patching和Provisioning为单个Oracle主目录启用Oracle分层文件系统（OLFS）以进行变更管理。
同时修补Oracle Grid Infrastructure和Oracle数据库 将群集上的Oracle Grid Infrastructure和Oracle数据库修补到最新的修补程序级别，而不会导致群集停机。
在没有停机的情况下修补Oracle Database 12 c第1版 使用最新的修补程序修补Oracle Database 12 c第 1版（12.1），而不会关闭数据库。
升级到Oracle Database 12 c第2版 使用Fleet Patching和Provisioning，使用单个命令将Oracle数据库从11.2升级到12 c版本2，这两个命令都适用于托管和非托管Oracle主目录。
将节点添加到群集并将Oracle RAC数据库扩展到节点 对于双节点群集，添加新节点，并使用Fleet Patching和Provisioning将Oracle RAC数据库扩展到新节点。
创建一个Fleet Patching和Provisioning Image 创建软件主页的黄金映像并将其存储在Fleet Patching and Provisioning Server上，以便稍后用于创建工作副本。
创建用于部署Apache Web服务器的用户操作 您可以使用Fleet Patching和Provisioning用户操作来安装和配置任何类型的软件。查看此过程以使用Fleet Patching和Provisioning自动部署Apache Web Server。
