配置Oracle软件所有者环境的过程

配置每个Oracle安装所有者用户帐户环境：

  1. 在运行安装的服务器上启动X终端会话（ xterm ）。
  2. 输入以下命令以确保X Window应用程序可以在此系统上显示，其中hostname是您从中访问服务器的本地主机的完全限定名称：
    $ xhost +主机名
  3. 如果您未以软件所有者用户身份登录，请切换到正在配置的软件所有者用户。例如，使用用户grid ：
    $ su  -  grid
    在无法运行su命令的系统上，请改用sudo ：
    $ sudo -u grid -s
  4. 要确定用户的默认shell，请输入以下命令：
    $ echo $ SHELL
  5. 在任何文本编辑器中打开用户的shell启动文件：
    • Bash shell（ bash ）：
      $ vi .bash_profile
    • Bourne shell（ sh ）或Korn shell（ ksh ）：
      $ vi .profile
    • C shell（ csh或tcsh ）：
      ％vi .login
  6. 输入或编辑以下行，为默认文件模式创建掩码指定值022 ：
    umask 022
  7. 如果在文件中设置了ORACLE_SID ， ORACLE_HOME或ORACLE_BASE环境变量，则从文件中删除这些行。
  8. 保存文件，然后退出文本编辑器。
  9. 要运行shell启动脚本，请输入以下命令之一：
    • Bash shell：
      $。 。/.bash_profile
    • Bourne，Bash或Korn shell：
      $。 。/。轮廓
    • C shell：
      ％ 资源 。/。登录
  10. 使用以下命令检查PATH环境变量：
    $ echo $ PATH
    删除所有Oracle环境变量。
  11. 如果不在本地系统上安装软件，请输入类似于以下内容的命令，以指示X应用程序在本地系统上显示：
    • Bourne，Bash或Korn shell：
      $ export DISPLAY = local_host：0.0
    • C shell：
      ％setenv DISPLAY local_host：0.0
    在此示例中， local_host是要在其上显示安装程序的系统（您的工作站或其他客户端）的主机名或IP地址。
  12. 如果/tmp目录的可用空间少于1 GB，则标识具有至少1 GB可用空间的文件系统，并设置TMP和TMPDIR环境变量以指定此文件系统上的临时目录：

    注意：

    您不能将共享文件系统用作Oracle RAC安装的临时文件目录（通常为/tmp ）的位置。如果将/tmp放在共享文件系统上，则安装将失败。

    1. 使用df -h命令标识具有足够可用空间的合适文件系统。
    2. 如有必要，请输入与以下内容类似的命令，以在您标识的文件系统上创建临时目录，并在目录上设置相应的权限：
      $ sudo  -  s #mkdir / mount_point / tmp #chmod 775 / mount_point / tmp #exit
    3. 输入与以下内容类似的命令以设置TMP和TMPDIR环境变量：

      Bourne，Bash或Korn shell：

      $ TMP = / mount_point / tmp $ TMPDIR = / mount_point / tmp $ export TMP TMPDIR

      C shell：

      ％setenv TMP / mount_point / tmp％setenv TMPDIR / mount_point / tmp
  13. 要验证是否已正确设置环境，请输入以下命令：
    $ umask $ env |更多
    验证umask命令显示的值22 ， 022 ，或0022和您在本节中设置环境变量的值正确。
