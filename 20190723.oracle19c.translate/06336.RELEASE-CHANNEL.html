3.3释放信道

目的

使用RELEASE CHANNEL命令释放正常或维护通道，同时保持与目标数据库实例的连接。使用ALLOCATE CHANNEL分配正常通道，而为维护通道分配ALLOCATE CHANNEL FOR MAINTENANCE 。

先决条件

要释放普通通道，请使用release :: = diagram中显示的语法。仅在RUN命令中执行此形式的RELEASE CHANNEL并使用ALLOCATE CHANNEL命令中使用的相同标识符指定通道名称。

要释放维护通道，请使用releaseForMaint :: =图中显示的语法。仅在RMAN提示符下执行此形式的RELEASE CHANNEL ，而不是在RUN命令中。

使用说明

维护通道不受RUN命令中发出的ALLOCATE CHANNEL和RELEASE CHANNEL命令的影响。

使用RELEASE CHANNEL在RUN释放通道是可选的，因为当RUN命令终止时，RMAN会自动释放所有正常通道。

语义

语法元素 描述

CHANNEL_ID

指定ALLOCATE CHANNEL使用的区分大小写的通道ID 命令（见例3-15 ）。

例子

示例3-15释放在RUN命令中分配的通道

此示例分配名为ch1的SBT通道，其中包含用于每日备份的一组磁带的参数，备份数据库，然后释放此通道。然后，该示例分配一个名为ch1的SBT通道，其中包含用于每周备份的一组磁带的参数，并进行另一个数据库备份：

RUN {ALLOCATE CHANNEL ch1 DEVICE TYPE sbt PARMS ='ENV =（OB_MEDIA_FAMILY = daily_bkp）';备份数据库; RELEASE CHANNEL ch1; ALLOCATE CHANNEL ch1 DEVICE TYPE sbt PARMS ='ENV =（OB_MEDIA_FAMILY = weekly_bkp）';备份数据库; }

RUN命令末尾的RELEASE CHANNEL命令是可选的，因为RMAN会自动释放通道ch1 。
