Oracle Net是Oracle Net Services的一个组件，支持从客户端应用程序到Oracle数据库服务器的网络会话。建立网络会话时，Oracle Net充当客户端应用程序和数据库的数据信使。它负责建立和维护客户端应用程序与数据库之间的连接，以及在它们之间交换消息。Oracle Net能够执行这些作业，因为它位于网络中的每台计算机上。

1.1.2.1关于位置透明度

每个数据库由一个或多个服务表示。服务由服务名称标识，例如sales.us.example.com 。客户端使用服务名称来标识它必须访问的数据库。有关数据库服务及其在网络中的位置的信息对客户端是透明的，因为连接所需的信息存储在存储库中。

存储库由一个或多个命名方法表示。命名方法是客户端应用程序在尝试连接到数据库服务时用于将连接描述符解析为连接描述符的解析方法。Oracle Net Services提供了几种支持每个客户端上的本地化配置的命名方法，或者可以由网络中的所有客户端访问的集中式配置。

例如，在图1-6中 ，公司有三个客户端可以访问的数据库。每个数据库都有一个不同的服务名称，例如sales.us.example.com ， hr.us.example.com和mktg.us.example.com 。

1.1.3了解共享服务器体系结构

Oracle数据库共享服务器体系结构提高了应用程序的可伸缩性以及可以同时连接到数据库的客户端数量。共享服务器体系结构还使现有应用程序可以扩展，而无需对应用程序本身进行任何更改。

使用共享服务器时，客户端不直接与数据库服务器进程通信，数据库服务器进程是代表数据库处理客户端请求的数据库进程。相反，客户端请求被路由到一个或多个调度程序 。调度程序将客户端请求放在公共队列中。来自共享服务器进程池的空闲共享服务器从队列中获取并处理请求。这意味着一小部分服务器进程可以为大量客户端提供服务。

图1-8和图1-9显示了共享服务器连接模型与传统专用服务器之间的基本区别连接模型。在共享服务器模型中，a 调度程序可以同时支持多个客户端连接。在专用服务器模型中，每个客户端都有一个服务器进程。每次收到连接请求时，服务器进程都会启动并专用于该连接，直到完成为止。这会导致处理延迟。

“图1-8共享服务器架构”的说明

“图1-9专用服务器架构”的说明

共享服务器是具有大量连接的配置的理想选择，因为它可以降低服务器内存要求。共享服务器非常适合Internet和Intranet环境。

使用Oracle Connection Manager可以进一步增强服务器资源的利用率。Oracle Connection Manager是一个Oracle Net Services组件，它允许多个客户端网络会话通过与数据库的单个网络连接进行多路复用或漏斗化。

会话多路复用功能通过使服务器能够为传入请求使用较少的网络连接端点，减少维护两个进程之间的多个网络会话所需的资源需求。通过这种方式，可以增加服务器可以处理的网络会话总数。一个具有多个网关的Oracle Connection Manager可以使数千个并发用户连接到服务器。

图1-10显示了如何在Web体系结构中使用会话多路复用。当Oracle Connection Manager与应用程序Web服务器在同一台计算机上运行时，应用程序Web服务器可以通过Oracle Connection Manager路由多个客户机会话，以确保这些会话可以连续访问Oracle数据库服务器。此功能对于会话可用性和响应时间是主要问题的Web应用程序尤其有用。

图1-10会话复用


“图1-10会话多路复用”的描述

以下是会话多路复用的优缺点。对于需要连续连接的网络，建议使用会话复用。

会话复用的优点

  • 限制用于每个进程的网络资源数量

  • 支持大客户群

  • 通过有限数量的进程连接最大化客户端/服务器会话的数量

  • 优化资源利用率

  • 实现对真实用户的识别和监控

  • 使中间层应用程序能够支持其他服务

  • 对于具有多个应用程序的客户端，只需要一个传输

  • 数据库链接只需要一个网络连接

会话多路复用的缺点

客户端必须连接到Oracle Connection Manager。

1.1.4.1监听器队列大小

如果您预计通过TCP / IP接收大量关于侦听进程的连接请求（例如侦听器或Oracle Connection Manager），则Oracle Net允许您将侦听队列配置为高于系统默认值。

父主题： 了解性能

1.1.4.2数据传输优化的会话数据单元大小

在通过网络发送数据之前，Oracle Net会将数据缓冲并封装到会话数据单元（SDU）中 。当缓冲区已满，刷新或数据库服务器尝试读取数据时，Oracle Net会发送存储在此缓冲区中的数据。当传输大量数据或消息大小一致时，调整SDU缓冲区的大小可以提高性能，网络利用率或内存消耗。您可以在客户端，应用程序Web服务器和数据库上部署SDU。

调整应用程序以减少通过网络的往返次数是提高网络性能的最佳方法。如果这样做，那么也可以进行优化通过调整SDU的大小来传输数据。

修改SDU大小的注意事项

在以下情况下修改SDU大小：

  • 从服务器返回的数据被分段为单独的数据包。

  • 您所在的广域网（WAN）有很长的延迟。

  • 数据包大小始终相同。

  • 返回大量数据。

在以下情况下，请勿修改SDU大小：

  • 可以调整应用程序以避免相邻列中列出的延迟。

  • 您有一个高速网络，数据传输的影响可以忽略不计。

  • 您的请求会从服务器返回少量数据。

注意：

从Oracle Database 11g开始 ，Oracle Net Services优化了某些组件的批量数据传输，例如Oracle SecureFiles LOB和Oracle Data Guard重做传输服务。网络参数文件中指定的SDU大小限制不适用于这些批量数据传输。启用ASO选项或使用TLS传输时，批量数据传输优化不适用。

1.1.4.3 TCP / IP的持久缓冲区刷新

在某些使用TCP / IP的应用程序的某些条件下，Oracle Net数据包可能无法立即刷新到网络。大多数情况下，当流式传输大量数据时会发生此行为。TCP / IP本身的实施是缺乏冲洗的原因，导致不可接受的延迟。要解决此问题，请在缓冲区刷新过程中指定无延迟。

1.1.4.4套接字直接协议

Oracle Net Services为InfiniBand高速网络提供支持。InfiniBand是一种高带宽I / O架构，旨在提高CPU，服务器端设备和网络子系统之间的通信速度。Oracle Net Services提供对套接字直接协议（ SDP ）的支持。SDP是一种行业标准的有线协议，旨在用于InfiniBand网络对等体之间。

SDP通过消除数据的中间复制并将大部分消息传递负担从CPU转移到网络硬件上来减少TCP / IP的开销。结果是低延迟 ，增加带宽，高吞吐量连接，减少了专用于网络处理的CPU周期。

客户端（包括Oracle WebLogic Server或任何其他第三方中间层客户端）与Oracle Database 12 c之间的通信可以利用高速互连优势。Oracle WebLogic Server在其安装过程中包含Oracle TCP / IP支持。

安装在Oracle WebLogic Server服务器上的驱动程序透明地将TCP / IP支持转换为SDP支持。然后，SDP请求将发送到InfiniBand交换机，该交换机处理从Oracle WebLogic Server服务器到数据库服务器的请求并将其转发。

1.1.4.5数据库可用性

数据库的可用性对任何网络都至关重要。您可以配置多个侦听器来处理同一数据库服务的客户端连接请求。这在Oracle Real Application Clusters配置中很有用，其中每个实例都有一个与之关联的侦听器。多个侦听器配置使您可以使用以下功能。

  • 如果第一个侦听器发生故障，则连接时故障转移使客户端可以请求不同的侦听器（通常在不同的节点上）。

  • 客户端负载平衡使客户端能够将请求随机化到多个侦听器，通常是在不同的节点上。这些功能可以一起使用或单独使用。它们共同确保对数据库的访问并分配负载，而不会使单个侦听器负担过重。

1.3.1关于Oracle Net

Oracle Net是一个驻留在客户端和Oracle数据库服务器上的软件层。它负责建立和维护客户端应用程序和服务器之间的连接，以及使用行业标准协议在它们之间交换消息。Oracle Net有两个软件组件：

  • Oracle Net Foundation Layer
  • Oracle协议支持

1.3.2关于Oracle Net Listener

Oracle数据库服务器通过接收初始连接 Oracle Net Listener 。Oracle Net Listener，在本文档中称为侦听器 ，代理客户端请求，将请求切换到服务器。侦听器配置有协议地址，配置了相同协议地址的客户端可以向侦听器发送连接请求。建立连接后，客户端和Oracle服务器将直接相互通信。

Oracle Net侦听器支持用于服务的ACL（访问控制列表），并且所有IP协议都支持此功能。

1.3.3关于Oracle Connection Manager

Oracle Connection Manager是驻留在其自己的计算机上的软件组件，与客户端或Oracle数据库服务器分开。它代理并屏蔽对数据库服务器的请求。此外，它还复用数据库会话。

在其会话多路复用角色中，Oracle Connection Manager通过与特定目标的单个传输协议连接来汇集多个会话。通过这种方式，Oracle Connection Manager可以通过使Oracle数据库服务器为传入请求使用较少的连接端点来减少维护两个进程之间的多个会话所需的资源需求。

作为访问控制过滤器，Oracle Connection Manager控制对Oracle数据库的访问。

注意：

通过在cman.ora设置tdm=yes ，Oracle Connection Manager可以充当流量控制器模式中的连接管理器。

流量控制器模式下的Oracle连接管理器提供改进的高可用性（HA）（计划内和计划外），连接多路复用支持和负载平衡。此功能还提供带内客户端通知机制，以将Oracle Connection Manager关闭和服务停止事件的计划关闭交付给OCI客户端。

1.3.4关于网络工具

Oracle Net Services提供用户界面工具和命令行实用程序来配置，管理和监控网络。

  • Oracle Net Configuration Assistant是一个独立的工具，可用于配置侦听器和命名方法。

  • Oracle Enterprise Manager Cloud Control将多个文件系统的配置功能与侦听器管理控制相结合，为配置和管理Oracle Net Services提供集成环境。

  • Oracle Net Manager为本地客户端或服务器主机上的Oracle主目录提供配置功能。

  • 用于配置，管理和监视网络组件的命令行控制实用程序，包括侦听器和Oracle Connection Manager。

使用Oracle Enterprise Manager Cloud Control或Oracle Net Manager，您可以微调使用Oracle Net Configuration Assistant创建的侦听器和命名方法配置。此外，Oracle企业管理器云控制和Oracle Net Manager提供内置向导和实用程序来测试连接，将数据从一种命名方法迁移到另一种命名方法，以及创建其他网络组件。

1.3.5关于Oracle Advanced Security

Oracle Advanced Security是一个可单独授权的产品，提供Oracle数据库透明数据加密（TDE）和Oracle数据编辑。TDE对数据进行加密，以便只有授权的收件人才能读取数据。Oracle Data Redaction使管理员可以使用以下类型的编辑来编辑（掩盖）列数据：

  • 完全编校会编辑列数据的所有内容。返回给查询用户的编辑值取决于列的数据类型。例如，NUMBER数据类型的列用零（0）编辑，字符数据类型用空格编辑。

  • 部分编辑会编辑列数据的一部分。例如，使用星号（*）屏蔽大部分信用卡号，但最后四位数除外。

  • 正则表达式允许使用数据模式进行编辑。例如，使用正则表达式来编辑可能具有不同字符长度的电子邮件地址。它仅适用于字符数据。

  • 随机编辑将编辑后的数据作为随机生成的值显示给查询用户。

  • 无编辑功能使管理员可以测试编校策略的内部操作，而不会影响对其上定义了策略的表的查询结果。
