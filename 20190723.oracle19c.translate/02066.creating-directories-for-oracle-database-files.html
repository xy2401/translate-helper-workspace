创建Oracle数据库文件的目录

您可以将Oracle数据库和恢复文件存储在与配置文件不同的文件系统中。

执行此过程以将Oracle数据库或恢复文件放在Oracle基本目录的单独文件系统上：

  1. 使用以下命令确定每个已装入文件系统上的可用磁盘空间：
    #df -h
  2. 从显示中标识要使用的文件系统：
    选项 描述

    数据库文件

    选择以下选项之一：

    • 单个文件系统，具有至少1.5 GB的可用磁盘空间

    • 两个或多个文件系统，总共至少有3.5 GB的可用磁盘空间

    恢复文件

    选择具有至少2 GB可用磁盘空间的文件系统

    如果对多种文件类型使用相同的文件系统，则为每种类型添加磁盘空间要求以确定总磁盘空间要求。

  3. 记下您标识的文件系统的安装点目录的名称。
  4. 如果执行安装的用户具有在计划安装Oracle数据库的磁盘上创建目录的权限，则DBCA将创建Oracle数据库文件目录和恢复文件目录。如果执行安装的用户没有写访问权限，则必须手动创建这些目录。

    例如，给定用户oracle和Oracle Inventory Group oinstall ，并使用路径/u03/oradata/wrk_area作为Oracle数据库文件，并使用/u01/oradata/rcv_area作为恢复区域，这些命令在每个子目录中创建推荐的子目录。挂载点目录并为它们设置适当的所有者，组和权限：

    • 数据库文件目录：

      #mkdir / u01 / oradata / #chown oracle：oinstall / u01 / oradata / #chmod 775 / mount_point / oradata

      数据库文件目录的缺省位置是$ORACLE_BASE/oradata 。

    • 恢复文件目录（快速恢复区）：

      #mkdir / u01 / oradata / rcv_area＃chown oracle：oinstall / u01 / app / oracle / fast_recovery_area＃chmod 775 / u01 / oradata / rcv_area

      默认的快速恢复区域是$ORACLE_BASE/fast_recovery_area 。

      Oracle建议您将快速恢复区域保留在与数据库文件目录不同的物理磁盘上。如果包含oradata的磁盘因任何原因无法使用，则此方法使您可以使用快速恢复区域来检索数据。
