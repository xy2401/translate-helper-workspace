1.183 LOG_FILE_NAME_CONVERT

LOG_FILE_NAME_CONVERT将主数据库上的新日志文件的文件名转换为备用数据库上的日志文件的文件名。

属性 描述

参数类型

串

句法

LOG_FILE_NAME_CONVERT = ' string1 ' , ' string2 ' , ' string4 ' , ... ' , ' string3 ' , ...

哪里：

  • string1是主数据库文件名的模式

  • string2是备用数据库文件名的模式

  • string3是主数据库文件名的模式

  • string4是备用数据库文件名的模式

您可以根据需要使用多对主要和备用替换字符串。您可以使用单引号或双引号。

以下是可接受的示例设置：

LOG_FILE_NAME_CONVERT ='/ dbs / t1 /'，'/ dbs / t1 / s'，'dbs / t2 /'，'dbs / t2 / s_'

默认值

没有默认值。

可修改

没有

可在PDB中修改

没有

基本

没有

如果将日志文件添加到主数据库，则必须将相应的文件添加到备用数据库。

如果指定奇数个字符串（最后一个字符串没有相应的替换字符串），则在启动期间会发出错误信号。如果要转换的文件名与模式/替换字符串列表中的多个模式匹配，则第一个匹配的模式生效。您可以在此参数中指定的对数量没有限制（除了多值参数的最大长度的硬限制之外）。

更新备用数据库时，此参数将主数据库上的日志文件名转换为备用数据库上的日志文件名。该文件必须存在于备用数据库中，并且必须是可写的，否则恢复过程将因错误而停止。

第一个字符串是在主数据库的日志文件名中找到的模式。第二个字符串是在备用数据库的日志文件名中找到的模式。

在表空间时间点恢复期间设置克隆数据库时，还应使用LOG_FILE_NAME_CONVERT重命名克隆控制文件中的日志文件。

注意：

LOG_FILE_NAME_CONVERT参数仅适用于联机日志（不适用于归档日志）。
