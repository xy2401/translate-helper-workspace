13从现有网关迁移

如果您遵循一些指导原则，从现有安装迁移到Oracle Database Gateway for APPC的新实例非常简单。以下主题提供了使这些新安装尽可能简单的信息。如果在网关上使用TCP / IP for IMS Connect通信协议，它们还提供所需的参数。

以下部分提供了特定于此版本的Oracle Database Gateway for APPC for IBM AIX for POWER Systems（64位），Linux x86-64，Oracle Solaris on SPARC（64位）和HP-UX Itanium的信息：

  • 使用SNA协议将现有网关实例迁移到新版本

  • 使用SNA从现有网关迁移到TCP / IP

使用SNA协议将现有网关实例迁移到新版本

按照以下步骤使用SNA通信协议将现有网关迁移到网关的12 c版本2（12.2）。

请注意，如果您使用的是网关对IMS Connect的TCP / IP支持，您不会将现有版本迁移到当前版本的网关。但是，您需要将有效的异构服务参数放入您的 init sid. ora文件。继续进行“ 参数更改：网关版本4到12c版本2（12.2） ” 。

第1步：安装新版本

如安装网关中所述，在单独的目录中安装新版本的网关 。

注意：

不要在先前存在的网关安装上安装网关。这样做会破坏现有安装。

步骤2：传输initsid.ora网关初始化文件参数

复制 init sid. ora文件从旧网关实例到新实例。

如果要从版本9.0.1或更早版本的网关迁移，则不支持PGA_TRACE 。您需要将参数修改为TRACE_LEVEL 。

注意：

如果您使用TRACE_LEVEL ，则必须设置路径 LOG_DESTINATION参数。

如果已使用dg4pwd在文件init sid .ora加密了某些参数，请将$ORACLE_HOMRE/dg4appc/admin/init sid .pwd文件从旧网关实例复制到新实例。

注意：

如果要从版本9.0.1或更低版本的网关迁移，则无需将此文件复制到新实例中。

迁移到新版本时的退出注意事项

Oracle建议您在安装新版本的网关并升级现有实例时保持旧网关Oracle主目录和实例配置的完整性和可操作性，以防升级出现问题。这有助于确保更改不同网关实例之间的停机时间最短。

Oracle Net注意事项

适用于APPC的Oracle数据库网关使用 Oracle和Oracle Net的异构服务（HS）工具。如果要从版本4网关升级，则需要稍微修改tnsnames.ora文件中的网关服务名称条目。添加一个(HS=)子句告诉Oracle Net网关使用HS设施。有关更多信息，请参阅配置Oracle网络 。

参数更改：版本4到12 c网关版本2（12.2）

如果要从版本4网关迁移到网关的12 c版本2（12.2），则此版本的Oracle Database Gateway for APPC会引入新的和更改的初始化参数。

注意：

如果要从Oracle数据库网关的APPC版本8迁移到版本10.2.0，则“ 参数更改：网关版本4到12c版本2（12.2） ”部分不适用于您。

如果您正在使用网关对IMS Connect的TCP / IP支持，您不会从版本4 迁移到当前版本的网关。但是，您需要将有效的异构服务参数放入您的 init sid. ora文件。

从现有网关迁移包含对运行网关所需的特定HS参数的引用。

注意：

有关Oracle产品中使用的所有HS参数的完整列表和说明，请参阅“ Oracle数据库异构连接用户指南” 。

新网关初始化参数

以下参数位于网关初始化文件中（ init sid. ora ）：

  • FDS_CLASS

  • FDS_INSTANCE

  • HS_FDS_FETCH_ROWS

  • LOG_DESTINATION

  • TRACE_LEVEL

  • PGA_TCP_DB （仅限TCP / IP）

  • PGA_TCP_USER （仅限TCP / IP）

  • PGA_TCP_PASS （仅限TCP / IP）

    注意：

    HS_参数特定于 O racle异类服务。有关详细信息 HS参数，请参阅Oracle数据库异构连接用户指南 。

重命名网关初始化文件参数

以下是一份清单网关初始化文件（ init sid. ora ）在此版本的网关中重命名的参数。参数的前名称显示在括号中。

  • HS_COMMIT_STRENGTH_POINT （以前称为COMMIT_STRENGTH_POINT ）

  • HS_DB_DOMAIN （以前称为DB_DOMAIN ）

  • HS_DB_INTERNAL_NAME （原DB_INTERNAL_NAME ）

  • HS_DB_NAME （以前称为DB_NAME ）

  • HS_DESCRIBE_CACHE_HWM （原DESCRIBE_CACHE_HWM ）

  • HS_LANGUAGE （原LANGUAGE ）

  • HS_NLS_DATE_FORMAT （原为NLS_DATE_FORMAT ）

  • HS_NLS_DATE_LANGUAGE （原为NLS_DATE_LANGUAGE ）

  • HS_OPEN_CURSORS （原OPEN_CURSORS ）

  • HS_ROWID_CACHE_SIZE （以前为ROWID_CACHE_SIZE ）

使用SNA从现有网关迁移到TCP / IP

以下部分适用于那些使用SNA协议具有现有网关版本但希望切换到使用TCP / IP支持IMS Connect的用户。

此版本的网关中的IMS Connect功能的TCP / IP支持使您可以继续使用现有的TIP。

使用现有的TIP与现有的侧面轮廓定义

请遵循以下说明：

  1. 确保你已经使用过 pg4tcpmap工具将有效参数值插入到 PGA_TCP_IMSC表。

    有关加载PGA_TCP_IMSC表的说明，请参阅使用TCP / IP通信协议的网关配置 。

  2. 确保TIP中的LANGUAGE参数设置为 american_america_us7ascii 。
  3. 使用PGAU重新生成IMS TIP。
  4. 将以下新TCP / IP参数添加到init sid. ora文件：

    • PGA_TCP_DB

    • PGA_TCP_USER

    • PGA_TCP_PASS

    您将在网关的参数更改：版本4到12c版本2（12.2）中找到有关添加这些参数的说明和信息。您还可以在TCP / IP通信协议的网关初始化参数中找到参数的说明。

    注意：

    如果已使用SideProfileName定义了先前版本的网关的TIP，并且NLS_ LANGUAGE参数设置为american_america_us7ascii的值，则无需重新编译这些TIP。您仍然需要使用pg4tcpmap工具映射参数值。

    注意：

    将通信协议从SNA更改为TCP / IP时，必须重新编译TIP 。

    如果您之前使用SNA通信包协议在网关上生成了现有TIP，并且您希望使用新的TCP / IP功能，则必须通过PGAU使用强制NLS_LANGUAGE和Side Profile Settings重新生成TIP。在DEFINE TRANSACTION命令中指定适当的ASCII字符集。

    这是因为网关假定正在使用IMS Connect中的适当用户出口，这将在适当的ASCII和EBCDIC字符集之间进行转换。
