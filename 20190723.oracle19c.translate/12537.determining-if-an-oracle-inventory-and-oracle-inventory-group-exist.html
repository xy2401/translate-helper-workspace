确定是否存在Oracle Inventory和Oracle Inventory Group

确定您是否具有现有Oracle中央库存，并确保为所有Oracle软件安装使用相同的Oracle Inventory。此外，请确保您打算用于安装的所有Oracle软件用户都有权写入此目录。

当您第一次在系统上安装Oracle软件时，OUI会创建oraInst.loc文件。此文件标识Oracle Inventory组的名称（默认情况下为oinstall ），以及Oracle中央清单目录的路径。如果您有现有的Oracle集中清单，请确保为所有Oracle软件安装使用相同的Oracle清单，并确保您打算用于安装的所有Oracle软件用户都有权写入此目录。
oraInst.loc文件包含以下格式的行，其中central_inventory_location是现有Oracle中央库存的路径， group是操作系统组的名称，其成员有权写入中央库存：
inventory_loc = central_inventory_location inst_group = group

使用more命令确定您的系统上是否有Oracle中央库存。例如：

#more /var/opt/oracle/oraInst.loc
inventory_loc = / u01 / app / oraInventory inst_group = oinstall

使用命令grep groupname /etc/group确认指定为Oracle Inventory组的组仍存在于系统上。例如：

$ grep oinstall / etc / group oinstall：x：54321：grid，oracle

注意：

不要将oraInventory目录放在Oracle基本目录下以进行新安装，因为这可能导致其他安装的用户权限错误。
