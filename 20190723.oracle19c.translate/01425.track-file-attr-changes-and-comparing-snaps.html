2.7跟踪文件属性更改和比较快照

使用Oracle ORAchk和Oracle EXAchk -fileattr选项和命令标志来记录和跟踪文件属性设置，并比较快照。

对所有者，组或权限等文件属性的更改可能会导致意外后果。在您的业务受到影响之前主动监控和缓解问题。

  • 使用文件属性检查守护程序
    在使用-fileattr之前，必须安装并运行Oracle Grid Infrastructure。
  • 获取文件属性快照
    默认情况下，Oracle Grid Infrastructure主目录和所有已安装的Oracle数据库主目录都包含在快照中。
  • 包括要检查的目录
    在文件属性更改检查中包含目录。
  • 不包括支票目录
    从文件属性更改检查中排除目录。
  • 重新检查更改
    将新快照与前一个快照进行比较以跟踪更改。
  • 将快照指定为基准
    将快照指定为基准，以与其他快照进行比较。
  • 限制系统检查
    限制Oracle ORAchk和Oracle EXAchk仅执行文件属性更改检查。
  • 删除快照
    努力删除快照。

2.7.1使用文件属性检查守护程序

在使用-fileattr之前，必须安装并运行Oracle Grid Infrastructure。

要对守护程序使用文件属性检查：

  1. 启动守护进程。
    ./orachk -d start
  2. 使用-fileattr选项启动客户端运行。
    ./orachk -fileattr start -includedir“/ root / myapp，/ etc / oratab”-excludediscovery ./orachk -fileattr check -includedir“/ root / myapp，/ etc / oratab”-excludediscovery
  3. 使用–output选项指定存储快照的输出目录。
    ./orachk -fileattr start -output“/ tmp / mysnapshots”
  4. 使用–tag选项为快照指定描述性名称，以标识快照。
    例如：
    ./orachk -fileattr start -tag“BeforeXYZChange”生成的快照目录 -  orachk_myserver65_20160329_052056_ BeforeXYZChange

2.7.5重新检查更改

将新快照与前一个快照进行比较以跟踪更改。

要重新检查更改：

  1. 使用check选项运行file attribute changes check命令以获取新快照，并运行正常的运行状况检查集合。

    –fileattr check命令将新快照与上一个快照进行比较。

    例如：
    ./orachk -fileattr检查
    ./exachk -fileattr检查

    注意：

    要在快照之间获得准确的比较，必须使用–fileattr check ，并使用与使用–fileattr start获得的先前快照集合相同的选项。

    例如，如果在运行–fileattr start时使用选项-includedir "/somedir" –excludediscovery获取了第一个快照，则必须在–fileattr check包含相同的选项以获得准确的比较。

2.7.6将快照指定为基准

将快照指定为基准，以与其他快照进行比较。

要将快照指定为基线：

  1. 使用–baseline path_to_snapshot选项运行file attribute changes check命令。

    如果要检查多个不同的基准， –baseline path_to_snapshot命令会将特定基准快照与其他快照进行比较。 

    ./orachk -fileattr check -baseline path_to_snapshot
    ./exachk -fileattr check -baseline path_to_snapshot
    例如：
    ./orachk -fileattr check -baseline“/ tmp / Snapshot”

2.7.7限制系统检查

限制Oracle ORAchk和Oracle EXAchk仅执行文件属性更改检查。

默认情况下， –fileattr check还执行完整运行状况检查。

限制系统检查：

  1. 使用–fileattronly选项运行file attribute changes check命令。
    ./orachk -fileattr check -fileattronly
    ./exachk -fileattr check -fileattronly
