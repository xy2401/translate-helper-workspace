147.1 DBMS_SERVICE概述

DBMS_SERVICE支持DBMS_SERVICE的高可用性，服务质量，作业调度和其他计划操作的工作负载管理，以用于工作负载测量，管理，优先级划分以及XA和分布式事务管理。

Oracle Real Application Clusters（RAC）可以通过SRVCTL管理跨实例的服务名称。 DBMS_SERVICE包允许在单个实例中创建，删除，启动和停止服务。此外，它还可以断开连接到具有服务名称的实例的所有会话。

147.2 DBMS_SERVICE安全模型

DBMS_SERVICE包具有某些安全要求。

特权

使用此程序包的客户端必须具有ALTER SYSTEM执行权限和V$SESSION表读取权限。

架构

必须在SYS架构下安装此程序包。

角色

程序包的EXECUTE特权仅授予DBA角色。

数

VARCHAR

STOP_OPTION_IMMEDIATE

VARCHAR

动态

数

使用说明

  • 如果已注册TAF回调，则忽略故障转移重试和故障转移延迟。如果发生错误，只要回调返回值OCI_FO_RETRY ，TAF就会继续重新尝试连接和身份验证。必须将任何延迟编码到回调逻辑中

  • 服务器端TAF设置会覆盖可能在TNS连接描述符中配置的客户端对应设备。如果未在客户端配置TAF，则必须至少将故障转移类型设置为启用TAF。如果在服务器端设置了故障转移类型，则故障转移方法默认为BASIC 。延迟和重试是可选的，可以单独指定。

147.6.1 CREATE_SERVICE过程

此过程在数据字典中创建服务名称。在service_name参数中设置服务或通过ALTER SYSTEM SET SERVICE_NAMES命令时，也会隐式地在数据字典中创建服务。

注意：

如果您的服务由Oracle Clusterware，Oracle Restart或Oracle Global Data Services管理，则不能使用第二版子程序。带参数数组接口的版本适用于不受Oracle Clusterware，Oracle Restart或Oracle Global Data Services管理的数据库。新属性仅可使用参数界面获得。

句法

DBMS_SERVICE.CREATE_SERVICE（service_name IN VARCHAR2，network_name IN VARCHAR2，parameter_array IN TABLE OF VARCHAR2（100））;

保持此重载以实现向后兼容：

DBMS_SERVICE.CREATE_SERVICE（service_name IN VARCHAR2，network_name IN VARCHAR2，目标IN NUMBER DEFAULT NULL，dtp IN BOOLEAN DEFAULT NULL，aq_ha_notifications IN BOOLEAN DEFAULT NULL，failover_method IN VARCHAR2 DEFAULT NULL，failover_type IN VARCHAR2 DEFAULT NULL，failover_retries IN NUMBER DEFAULT NULL，failover_delay IN NUMBER DEFAULT NULL，clb_goal IN NUMBER DEFAULT NULL，版本IN VARCHAR2 DEFAULT NULL）;

参数

参数 描述

服务名称

服务名称，数据字典中限制为64个字符

网络名字

用于客户端连接的SQLNet连接描述符中使用的服务的网络名称。这仅限于NET service_names字符集（请参阅Oracle数据库网络服务参考 ）。

parameter_array

具有服务属性的名称 - 值对的关联数组。支持的名称：

  • 目标

  • DTP

  • aq_ha_notifications

  • failover_method

  • failover_type

  • failover_retries

  • failover_restore

  • failover_delay

  • clb_goal

  • 版

  • commit_outcome

  • retention_timeout

  • replay_initiation_timeout

  • session_state_consistency

  • sql_translation_profile

  • drain_timeout

  • stop_option

目标

服务的工作负载管理目标指令。有效值：

  • DBMS_SERVICE 。 GOAL_SERVICE_TIME

  • DBMS_SERVICE 。 GOAL_THROUGHPUT

  • DBMS_SERVICE 。 GOAL_NONE

DTP

声明服务是用于X / Open分布式事务处理（DTP）或任何分布式事务（尤其是XA）

aq_ha_notifications

确定是否为OCI / OCCI / ODP启用快速应用程序通知（FAN）。在Oracle Database12 c中 ，FAN使用Oracle Notification Services（ONS）。此参数仍用于启用FAN。建议所有高可用性系统使用FAN，默认情况下，应用程序连续性处于启用状态

failover_method

应用程序连续性和TAF服务的故障转移TYPE 。如果在服务上将failover_type设置为TRANSACTION ，则会自动将COMMIT_OUTCOME设置为TRUE 。 JDBC Replay Driver使用TRANSACTION的FAILOVER_TYPE服务属性设置进行TRANSACTION故障转移。OCI使用SELECT和SESSION的旧设置。服务器仅接受带有TRANSACTION设置的FAILOVER_METHOD = BASIC 。

failover_type

应用程序连续性的TRANSACTION处理。TAF的SELECT或BASIC 。

failover_retries

应用程序连续性和TAF的连接重试次数。使用failover_retries和failover_delay参数，可以延迟failover_delay ，直到服务下次可用。此参数用于连接。它不控制故障转移的数量，对于应用程序连续性的每个事件，它都是3。

failover_delay

应用程序连续性和TAF的连接重试之间的延迟时间。应用程序连续性的默认值为10秒。如果服务需要时间进行故障转移和注册，请不要使用0秒延迟。长时间延迟有利于计划中断和故障转移到Data Guard。当服务已经可用时，短暂延迟与RAC配合良好。

版

如果此参数具有非NULL值，则为使用此服务但未指定版本的后续数据库连接提供初始会话版本。如果未指定任何值，则此参数无效。

在服务创建或修改期间，不对此参数执行验证。

在连接时，如果连接用户没有版本的USE权限，或者版本不存在，则会引发错误ORA-38802 （版本不存在）。

drain_timeout

如果定义了此参数，则使用快速连接故障转移（FCF）的客户端驱动程序和池将连接到该服务的所有会话。可以在服务上设置drain_timeout ，因此默认情况下停止和重新定位此时的排水。

stop_option

提供终止会话的选项。支持的值是：

  • TRANSACTIONAL

    drain_timeout适用于交易。在drain_timeout到期后，会话立即被终止。

  • 即时

    在drain_timeout到期后，会话将被终止。

  • 没有

    会话未终止。

但是，可以使用命令行覆盖这些参数值。

commit_outcome

确定COMMIT执行后是否可以访问事务COMMIT结果。虽然数据库保证COMMIT是持久的，但这确保了COMMIT的结果是持久的。应用程序使用该功能来探测中断后最后执行的提交的状态，并且应用程序可以使用该功能来确定结果。注意：

  • 调用DBMS_APP_CONT包的GET_LTXID_OUTCOME过程需要设置commit_outcome属性。

  • commit_outcome对活动Data Guard和只读数据库没有影响。

  • commit_outcome仅允许在数据库服务和用户定义的数据库服务上使用

retention_timeout

与commit_outcome结合使用时，它确定保留COMMIT_OUTCOME的时间（以秒为单位）。默认值为24小时（86400）。最大值为30天（2592000）。

replay_initiation_timeout

对于应用程序连续性， replay_initiation_timeout是请求的第一次操作的原始执行时间与成功重新连接后重放准备好开始的时间之间的差异。重播启动时间是从最初提交请求的时间到重放已连接并准备重播的时间来衡量的。在预期重播时，请保持此值高。默认值为900秒。

session_state_consistency

描述在请求期间如何更改非事务（值为DYNAMIC或STATIC ）。仅当failover_type设置为TRANSACTION for Application Continuity时才考虑此参数。会话状态的示例包括NLS设置，优化程序首选项，事件设置，PL / SQL全局变量，临时表，高级队列，LOB和结果缓存。如果在请求开始后这些值发生变化，请设置为DYNAMIC （默认值）。几乎所有应用程序都应使用DYNAMIC模式。如果您不确定，请使用DYNAMIC模式。

sql_translation_name

SQL转换单元的名称

clb_goal

用于连接负载平衡的方法（ 参见表147-2 ）

例子

DBMS_SERVICE.CREATE_SERVICE（ 'ernie.example.com'， 'ernie.example.com'）; DECLARE params dbms_service.svc_parameter_array; BEGIN params（'FAILOVER_TYPE'）：='TRANSACTION'; PARAMS（ 'REPLAY_INITIATION_TIMEOUT'）：= 1800; params（'RETENTION_TIMEOUT'）：= 86400; params（'FAILOVER_DELAY'）：= 10; params（'FAILOVER_RETRIES'）：= 30; params（'DRAIN_TIMEOUT'）：= 60; params（'STOP_OPTION'）：='DBMS_SERVICE.STOP_OPTION_IMMEDIATE'; params（'FAILOVER_RESTORE'）：='DBMS_SERVICE.FAILOVER_RESTORE_BASIC'; params（'commit_outcome'）：='true'; params（'aq_ha_notifications'）：='true'; DBMS_SERVICE.MODIFY_SERVICE（ '黄金'，则params）;结束;

147.6.3 DISCONNECT_SESSION过程

此过程断开与当前实例上的命名服务的会话。

句法

DBMS_SERVICE.DISCONNECT_SESSION（service_name IN VARCHAR2，disconnect_option IN NUMBER DEFAULT POST_TRANSACTION;

参数

表147-8 DISCONNECT_SESSION过程参数

参数 描述

服务名称

服务名称，数据字典中限制为64个字符

disconnect_option

选项包常量表示为NUMBER ：

  • POST_TRANSACTION = 0 ：当前事务提交或回滚后会话断开连接

  • IMMEDIATE = 1 ：会话立即断开连接

  • NOREPLAY = 2 ：会话立即断开连接并被标记为不被应用程序连续性重放，即IMMEDIATE和NOREPLAY在一起

注意 ： IMMEDIATE或POST_TRANSACTION和NOREPLAY自动翻译为1或0或2。但是，传递字符串文字（使用'或"字符引用，例如"IMMEDIATE"或'POST_TRANSACTION'或'NOREPLAY' ）会引发错误。

使用说明

  • 此过程可以在单个实例的上下文中使用，也可以在Oracle Real Application Clusters中使用。

  • 在断开所有相应的会话之前，此子程序不会返回。因此，如果调用者不想等待所有相应的会话断开连接，请使用DBMS_JOB包或将SQL会话置于后台。

例子

这会断开与service_name 'ernie.example.com' 。 

DBMS_SERVICE.DISCONNECT_SESSION（ 'ernie.example.com'）;

如果服务正在使用应用程序连续性，并且您不希望重播会话但只是终止，请使用以下命令：

EXECUTE DBMS_SERVICE.DISCONNECT_SESSION（' service name '，DBMS_SERVICE.NOREPLAY）;

对于Application Continuity，当设置failover_restore参数时，会在重播ODP.NET和Java之前恢复会话状态。使用LEVEL1作为ODP.NET，使用Java和Application Continuity来恢复初始状态。

对于AC OCI，对非STATIC应用程序使用NONE 。

failover_retries

应用程序连续性和TAF的连接重试次数。使用failover_retries和failover_delay参数，可以延迟failover_delay ，直到服务下次可用。此参数用于连接。它不控制故障转移的数量，对于应用程序连续性的每个事件，它都是3。

failover_delay

应用程序连续性和TAF的连接重试之间的延迟时间。应用程序连续性的默认值为10秒。如果服务需要时间进行故障转移和注册，请不要使用0秒延迟。长时间延迟有利于计划中断和故障转移到Data Guard。当服务已经可用时，短延迟与Oracle RAC配合良好。

drain_timeout

设置此参数后，使用快速连接故障转移（FCF）的客户端驱动程序和池将连接到该服务的所有会话。可以在服务上设置drain_timeout ，默认情况下停止和重新定位此时的排水。

版

如果此参数具有非NULL值，则为使用此服务但未指定版本的后续数据库连接提供初始会话版本。如果未指定任何值，则此参数无效。

在服务创建或修改期间，不对此参数执行验证。

在连接时，如果连接用户没有版本的USE权限，或者版本不存在，则会引发错误ORA-38802 （版本不存在）。

commit_outcome

确定COMMIT执行后是否可以访问事务COMMIT结果。虽然数据库保证COMMIT是持久的，但这确保了COMMIT的结果是持久的。应用程序使用该功能来探测中断后最后执行的提交的状态，并且应用程序可以使用该功能来确定结果。注意：

  • 调用DBMS_APP_CONT包的GET_LTXID_OUTCOME过程需要设置commit_outcome属性。

  • commit_outcome对活动Data Guard和只读数据库没有影响。

  • commit_outcome仅允许在用户定义的数据库服务上使用

retention_timeout

与commit_outcome结合使用时，它确定保留COMMIT_OUTCOME的时间（以秒为单位）。默认值为24小时（86400）。最大值为30天（2592000）。

replay_initiation_timeout

对于应用程序连续性， replay_initiation_timeout是请求的第一次操作的原始执行时间与成功重新连接后重放准备好开始的时间之间的差异。重播启动时间是从最初提交请求的时间到重放已连接并准备重播的时间来衡量的。在预期重播时，请保持此值高。默认值为900秒。

session_state_consistency

描述在请求期间如何更改非事务（值为DYNAMIC或STATIC ）。仅当failover_type设置为TRANSACTION for Application Continuity时才考虑此参数。会话状态的示例包括NLS设置，优化程序首选项，事件设置，PL / SQL全局变量，临时表，高级队列，LOB和结果缓存。如果在请求开始后这些值发生变化，请设置为DYNAMIC （默认值）。几乎所有应用程序都应使用DYNAMIC模式。如果您不确定，请使用DYNAMIC模式。

sql_translation_name

SQL转换单元的名称

modify_edition

如果为TRUE ，则更新版本服务属性以使用版本参数值。如果为FALSE或NULL ，则不更新版本属性。

clb_goal

用于连接负载平衡的方法（ 参见表147-2 ）

使用说明

  • 如果将集群管理服务与Oracle Clusterware一起使用，或者将Oracle Restart与单实例数据库一起使用，则必须使用srvctl命令而不是DBMS_SERVICE来修改服务。当Oracle Clusterware或Oracle Restart启动服务时，将在数据库中修改服务以匹配为Oracle Clusterware或Oracle Restart定义的资源。除非使用相应的srvctl命令进行更改，否则使用DBMS_SERVICE进行的任何更改都将丢失。从11.2.0.2开始，服务属性修改在srvctl启动或修改服务时立即生效。

  • 虽然用户可以在服务启动并运行时修改版本属性，但这样做可能不安全。用户必须谨慎行事，因为这会导致新版本连接新连接，而现有连接不受影响。这可能导致中间层操作连接到错误的版本。

147.6.5 START_SERVICE程序

此过程启动服务。在Oracle RAC中，实现此选项会对指定的实例起作用。

注意：

如果您的服务由Oracle Clusterware，Oracle Restart或Oracle Global Data Services管理，则无法使用此子程序。

句法

DBMS_SERVICE.START_SERVICE（service_name IN VARCHAR2，instance_name IN VARCHAR2）;

参数

表147-10 START_SERVICE过程参数

参数 描述

服务名称

数据字典中限制为64个字符的服务名称

INSTANCE_NAME

如果您的服务由Oracle Clusterware，Oracle Restart或Oracle Global Data Services管理，则无法使用此子程序。

句法

DBMS_SERVICE.STOP_SERVICE（service_name IN VARCHAR2，instance_name IN VARCHAR2;

参数

表147-11 STOP_SERVICE过程参数

参数 描述

服务名称

数据字典中限制为64个字符的服务名称

INSTANCE_NAME

必须停止服务的实例的名称（可选）。NULL导致本地停止服务。在单个实例中，这只能是当前实例或NULL 。 Oracle RAC和独占案例中的默认值为NULL 。指定DBMS_SERVICE 。 ALL_INSTANCES在所有已配置的实例上停止服务。
