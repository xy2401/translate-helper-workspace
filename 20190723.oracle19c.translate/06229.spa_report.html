摘要下方是“全球统计”部分，其中包括两个条形图。

左侧的图表标记为Projected Workload Elapsed Time。它具有标记为经过时间（秒）的y轴。y轴的值范围为0到1,500。x轴值是SQL试验1（蓝色）和SQL试验2（棕色）。SQL Trial 1（蓝色）的y轴值刚刚超过1,000。SQL Trial 2（棕色）的y轴值略高于SQL Trial 1。改善影响为0％。回归影响为-2％。整体影响为-2％。

右侧的图表标记为SQL Statement Count。y轴标记为SQL Count，范围为0到500。x轴标记为经过时间的变化。x轴具有以下值：Improved，Regressed和Unchanged。改进的y轴值为0。回归的y轴值为0。“未更改”栏为绿色，y轴值约为450，表示“相同计划”。没有橙色的条形图，代表新计划。

“SQL语句计数”图表右侧是“建议”部分。“建议”部分中的文本显示“Oracle提供了两个选项来修复因计划更改而导致的回归SQL：通过创建SQL计划基准，使用SQL试验1中更好的执行计划。”该文本下方是Create SQL Plan Baselines按钮。“创建SQL计划基准”按钮下方是此附加文本：“使用SQL Tuning Advisor探索备用执行计划”。该文本下方是Run SQL Tuning Advisor按钮。

“全局统计”部分下面是“基于对工作负载的影响的前10个SQL语句”部分。该表包含以下列：null，SQL ID，工作负载的净影响（％），已用时间（秒）SQL试验1，经过时间（秒）SQL试验2，对SQL的净影响（％），新计划。
