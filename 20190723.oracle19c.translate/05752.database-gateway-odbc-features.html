2 Oracle Database Gateway for ODBC功能和限制

安装和配置网关后，您可以使用网关访问非Oracle系统中的数据，将本机命令从应用程序传递到非Oracle系统，执行分布式查询以及复制数据。

使用传递功能

网关可以使用DBMS_HS_PASSTHROUGH包将本机命令或语句从应用程序传递到非Oracle系统。

使用PL / SQL块中的DBMS_HS_PASSTHROUGH包指定要传递给非Oracle系统的语句，如下所示：

DECLARE num_rows INTEGER; BEGIN num_rows：= DBMS_HS_PASSTHROUGH.EXECUTE_IMMEDIATE@SYBS（' command '）;结束; /

其中command不能是以下之一：

  • 开始交易

  • 承诺

  • ROLLBACK

  • 保存

  • 关掉

DBMS_HS_PASSTHROUGH包支持传递绑定值和执行SELECT语句。

注意：

TRUNCATE不能用于传递语句。

作为一般规则，建议您在传递中的每个DDL语句之后执行COMMIT ，尤其是在转到Sybase数据库时。

使用AUTO COMMIT

某些非Oracle数据库无需记录即可运行。如果在此类环境下需要只读功能，则需要设置HS_TRANSACTION_MODEL网关参数。

HS_TRANSACTION_MODEL参数可以设置如下：

HS_TRANSACTION_MODEL = READ_ONLY_AUTOCOMMIT

但是，如果仍需要更新功能，请在网关初始化参数文件中设置HS_TRANSACTION_MODEL=SINGLE_SITE_AUTOCOMMIT 。任何更新都会立即提交。没有记录的目标不允许提交确认。

已知的限制

如果遇到本节或“ 已知问题 ”中未列出的不兼容问题，请与Oracle支持服务联系。

以下是已知限制：

  • 传递查询无法读取BLOB和CLOB数据

  • 不允许在WHERE子句中包含不受支持的函数的更新或删除

  • Oracle Database Gateway for ODBC不支持存储过程

  • 不能参与分布式交易;仅支持单站点事务

  • 不支持多线程代理

  • 不支持使用绑定变量更新LONG列

  • 不支持rowid

PL / SQL游标循环中的COMMIT或ROLLBACK关闭打开游标

PL / SQL游标循环中发出的任何COMMIT或ROLLBACK关闭所有打开的游标，这可能会导致错误。

可能发生以下错误：

ORA-1002：取消顺序

要防止此错误，请在游标循环外移动COMMIT或ROLLBACK语句。

SQL语法限制

Oracle Database Gateway for ODBC对SQL语法有以下限制。

条款的当前条款

网关不支持带有WHERE CURRENT OF子句的UPDATE和DELETE语句，因为它们依赖于Oracle ROWID实现。要通过网关更新或删除特定行，必须使用条件样式WHERE子句。

通过条款连接

网关不支持SELECT语句中的CONNECT BY子句。

ROWID

不支持Oracle ROWID实现。

解释计划声明

不支持EXPLAIN PLAN语句。

  • 带有小写表名的SQL * Plus COPY命令

    用双引号包装小写表名。

    例如：

    复制自tkhouser / tkhouser @ inst1使用select * from“tkhodept”@ holink2插入loc_tkhodept;

  • 数据库链接

    网关不是多线程的，不支持共享数据库链接。每个网关会话都会生成一个单独的网关进程，并且无法共享连接。

CALLBACK链接支持

Oracle Database Gateway for ODBC不支持CALLBACK链接。

尝试与网关的CALLBACK链接将返回以下错误消息：

ORA-02025：SQL语句中的所有表必须位于远程数据库中

已知问题

本节介绍已知问题，并包括在可能的情况下进行纠正的建议。如果您对问题有任何疑问或疑虑，请联系Oracle支持服务。

加密格式登录

Oracle数据库自动加密密码。

Oracle数据库不再支持初始化参数DBLINK_ENCRYPT_LOGIN 。直到版本7.3，此参数的默认TRUE值阻止了通过网络发送登录用户ID的密码（以明文形式）。更高版本会自动加密密码。

日期算术

本主题描述了无法与网关一起正常运行的SQL表达式。

日期+号码+日期日期 - 号码date1  -  date2

具有前述表达式的语句将发送到非Oracle系统，而不进行任何转换。如果非Oracle系统不支持这些日期算术函数，则语句将返回错误。
