1.139 INMEMORY_CLAUSE_DEFAULT

INMEMORY_CLAUSE_DEFAULT使您可以为新表和物化视图指定默认的内存中列存储（IM列存储）子句。

属性 描述

参数类型

串

句法

INMEMORY_CLAUSE_DEFAULT = '[INMEMORY] [NO INMEMORY] [other-clauses] '

句法

其他子句:: =

[compression-clause] [priority-clause] [rac-clause]

句法

压缩子句:: =

NO MEMCOMPRESS | MEMCOMPRESS FOR { DML | QUERY [ LOW | HIGH ] | CAPACITY [LOW | HIGH] } Foot 1 Footref 1

句法

优先子句:: =

PRIORITY { LOW | MEDIUM | HIGH | CRITICAL | NONE } Footref 1

句法

外消旋子句:: =

[distribute-clause] [duplicate-clause] Footref 1

句法

分发子句:: =

DISTRIBUTE [ AUTO | BY ROWID RANGE ] Footref 1

句法

重复子句:: =

NO DUPLICATE | DUPLICATE [ ALL ] Footref 1

默认值

一个空字符串

可修改

ALTER SESSION，ALTER SYSTEM

可在PDB中修改

是

基本

没有

Oracle RAC

所有实例都应使用相同的值

脚注1

有关此子句的更多信息，请参见表1-2 。

如果取消设置INMEMORY_CLAUSE_DEFAULT参数或将其设置为空字符串（默认值），则只会将显式指定为INMEMORY表和物化视图填充到IM列存储中。将INMEMORY_CLAUSE_DEFAULT参数的值设置为NO INMEMORY与将其设置为默认值具有相同的效果。

如果设置了INMEMORY_CLAUSE_DEFAULT参数，则任何新创建的表或指定为INMEMORY物化视图INMEMORY将从此参数继承未指定的属性。这可以强制默认情况下某些内存中的选项未在语法中明确指定。例如，如果INMEMORY_CLAUSE_DEFAULT参数设置为MEMCOMPRESS FOR CAPACITY LOW并且表创建为INMEMORY PRIORITY HIGH ，则表被视为声明为INMEMORY MEMCOMPRESS FOR CAPACITY LOW PRIORITY HIGH 。

如果将INMEMORY指定为此参数的一部分，则所有新创建的表和物化视图将填充到IM列存储中，但显式指定为NO INMEMORY表和物化视图NO INMEMORY 。例如，如果此参数设置为INMEMORY MEMCOMPRESS FOR CAPACITY HIGH ，则将创建所有新表，就好像此子句存在于SQL CREATE TABLE语句中一样。如果给定段的表空间存在默认的INMEMORY值，则它将覆盖此参数的值。

表1-2 INMEMORY_CLAUSE_DEFAULT参数值的含义

句法 描述

在记忆中

指定所有新创建的表和物化视图填充IM列存储，除非它们在SQL CREATE TABLE或CREATE MATERIALIZED VIEW语句中指定为NO INMEMORY

没有内容

指定仅在SQL CREATE TABLE或CREATE MATERIALIZED VIEW语句中显式指定为INMEMORY表和INMEMORY CREATE MATERIALIZED VIEW填充IM列存储

压缩子句

指定应该为实例使用内存中压缩。使用MEMCOMPRESS FOR值指定内存中的压缩级别。

没有MEMCOMPRESS

如果指定了NO MEMCOMPRESS ，则IM列存储中不会进行内存中压缩。

MEMCOMPRESS FOR

MEMCOMPRESS FOR用于指示IM列存储的内存中压缩级别。

DML

当指定DML ，IM列存储针对DML操作进行了优化，并且可以进行一些轻量级内存中压缩。

QUERY

指定QUERY ，内存中压缩级别用于实现高性能。如果指定QUERY而没有LOW或HIGH ，则默认为QUERY LOW 。

QUERY LOW

指定QUERY LOW ，内存中压缩级别提供最高性能。

QUERY HIGH

指定QUERY HIGH ，内存中压缩级别在压缩和性能之间提供平衡，并对性能进行加权。

容量

如果指定CAPACITY没有LOW或HIGH ，则默认为CAPACITY LOW 。

容量低

当指定CAPACITY LOW ，内存中压缩级别是压缩和性能之间的平衡，加权容量。

容量高

指定CAPACITY HIGH ，内存中压缩级别用于最高容量。

优先条款

指定在IM列存储中填充表时要使用的优先级。使用PRIORITY值指定优先级。

默认情况下，IM列存储中的表的填充可以延迟，直到数据库确定它有用。

在数据库实例启动时，将按优先级顺序填充表。

优先权

指定PRIORITY NONE ，IM列存储中的表的填充可以延迟，直到数据库确定它有用。

这是未指定优先级时的默认值。

优先级低

为一个或多个表指定PRIORITY LOW时，IM列存储中这些表的填充是在未指定优先级的表之前完成的。

优先媒体

为一个或多个表指定PRIORITY MEDIUM时，IM列存储中这些表的填充是在没有指定优先级且指定了PRIORITY LOW表之前完成的。

优先级高

为一个或多个表指定PRIORITY HIGH时，IM列存储中这些表的填充是在没有指定优先级， PRIORITY LOW和PRIORITY MEDIUM表之前完成的。

优先关键

为一个或多个表指定PRIORITY CRITICAL时，IM列存储中这些表的填充是在没有指定优先级， PRIORITY LOW ， PRIORITY MEDIUM和PRIORITY HIGH表之前完成的。

RAC-条款

指定如何在Oracle Real Application Clusters（Oracle RAC）实例之间管理IM列存储中的表。使用distribute-clause和duplicate-clause指定如何在Oracle RAC实例中管理IM存储中的表。

对于非Oracle RAC数据库，这些设置无效，因为整个表或分区必须位于单个实例上。

分发子句

指定表在Oracle RAC实例之间的分布方式。

DISTRIBUTE AUTO

指定数据库将根据分区类型和duplicate-clause的值自动决定如何跨Oracle RAC实例分发IM列存储中的表。

DISTRIBUTE AUTO是默认值，当它自己指定DISTRIBUTE时也会使用它。

由ROWID RANGE分发

指定IM列存储中的表将按rowid范围分发到不同的Oracle RAC实例。

重复子句

指定IM列存储中表的每个内存中压缩单元（IMCU）的副本数将分布在所有Oracle RAC实例中。

注意：只有在工程系统上使用Oracle Real Application Clusters（Oracle RAC）时， duplicate-clause才适用。否则，将忽略duplicate-clause ，并且内存中每个IMCU只有一个副本。

没有重复

Oracle RAC实例中的数据不会重复。这是默认值。

重复

数据在另一个Oracle RAC实例上重复，导致数据总共存在于两个Oracle RAC实例中。

全部重复

所有Oracle RAC实例都会复制数据。如果指定DUPLICATE ALL ，则无论是否或如何指定distribute-clause ，数据库都将使用DISTRIBUTE AUTO设置。

例子

以下语句不会导致表填充IM列存储：

alter system set INMEMORY_CLAUSE_DEFAULT ='NO INMEMORY'cope = both;

以下语句导致新表和物化视图（指定为NO INMEMORY视图除外）以高容量压缩级别填充IM列存储：

alter system set INMEMORY_CLAUSE_DEFAULT ='INMEMORY MEMCOMPRESS FOR CAPACITY HIGH'scope = both;

以下语句会导致新表（即使指定为NO INMEMORY ）以LOW优先级填充最高性能压缩级别的IM列存储：

alter system set INMEMORY_CLAUSE_DEFAULT ='INMEMORY MEMCOMPRESS for QUERY LOW PRIORITY LOW'cope = both;

以下语句会导致新表（即使指定为NO INMEMORY ）在没有任何内存压缩的情况下填充IM列存储：

alter system set INMEMORY_CLAUSE_DEFAULT ='INMEMORY NO MEMCOMPRESS'cope = both;

除非在非工程系统上，否则以下语句会导致IM列存储中的表在每个Oracle RAC实例上重复。对于非工程系统，将忽略duplicate-clause （ DUPLICATE ALL ），并且列存储中的表将自动分布在Oracle RAC实例中，IM列存储中的表中每个IMCU只有一个副本：

alter system set INMEMORY_CLAUSE_DEFAULT ='INMEMORY MEMCOMPRESS for QUERY DISTRIBUTE AUTO DUPLICATE ALL'cope = both;

以下语句将INMEMORY_CLAUSE_DEFAULT参数的值设置回其默认值，即空字符串：

alter system set INMEMORY_CLAUSE_DEFAULT =''scope = both;
