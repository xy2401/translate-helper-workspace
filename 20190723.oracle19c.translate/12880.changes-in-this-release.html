Oracle Database Release 19c，版本19.1中的更改

以下是Oracle Database 2 Day +性能调优指南（适用于Oracle数据库19c版，19.1版）中的更改。

新功能

以下功能是此版本中的新增功能：

  • 可插拔数据库（PDB）的自动数据库诊断监视器（ADDM）支持

    您现在可以使用ADDM分析PDB中的AWR数据，以识别和解决与性能相关的问题。

    有关详细信息，请参阅“ 多租户环境的ADDM ” 。

  • 为非管理数据库用户启用实时SQL监视功能

    现在，没有管理权限的数据库用户也可以通过导航到Oracle Enterprise Manager云控制（云控制）的“受监控的SQL执行”页面来查看其SQL语句的执行计划和性能指标。

    有关详细信息，请参阅“ 在云控制中监控数据库操作 ”一节。

不受支持的功能

此版本不支持以下功能。

  • Oracle Streams

    从Oracle Database 19c开始，不支持Oracle Streams功能。使用Oracle GoldenGate替换Oracle Streams的所有复制功能。

Oracle Database Release 18c，版本18.1中的更改

Oracle Database 2 Day +性能调优指南 （Oracle数据库版本18c，版本18.1）中没有任何更改。有关此版本中新增的Oracle数据库性能相关功能，请参见“ Oracle数据库性能调整指南”和“ Oracle数据库SQL调优指南” 。

Oracle Database 12 c第2版（12.2）中的更改

Oracle Database 12 Day +性能调优指南（适用于Oracle Database 12 c第2版（12.2））没有任何更改。有关此版本中新增的Oracle数据库性能相关功能，请参阅“ Oracle数据库性能调整指南” 。

Oracle Database 12 c第1版（12.1.0.2）中的更改

以下是Oracle Database 12 Day +性能调优指南（适用于Oracle Database 12 c第1版（12.1.0.2））中的更改。

新功能

以下功能是此版本中的新增功能：

  • 内存列存储的可管理性支持

    新的Oracle数据库内存中列存储（IM列存储）功能可加快分析，数据仓库和联机事务处理（OLTP）应用程序的数据库性能。

    SQL Monitor报告，ASH报告和AWR报告现在显示各种内存中操作的统计信息。

    • SQL监视器报告中的内存中统计信息：“时间和等待统计信息”面板中的“活动％”，“计划统计信息”表中的“活动”列和“SQL监视器”报告中的“活动”选项卡显示执行内存中查询操作时SQL命令占用的CPU。SQL Monitor报告现在支持自适应计划。执行计划显示“解决”或“已解决”图标，具体取决于该计划的当前状态。“计划统计信息”选项卡包含一个下拉列表，用于显示当前计划，最终计划和完整计划。它还包含Plan Note按钮，单击该按钮时，将显示在SQL语句的说明计划中生成的注释。

    • ASH报告中的内存中统计信息：ASH报告标头表显示“内存区域”列下的内存池大小。Top Events，Top SQL和Activity After Time部分显示各种内存操作的CPU消耗。

    • AWR报告中的内存中统计信息：AWR报告包含一个新部分 - 内存中段信息统计 - 显示基于各种属性的内存中段消耗，例如扫描，数据块块更改，填充CU和重新填充CU。“时间模型统计”部分显示与内存中CPU使用率相关的统计信息，“实例活动统计信息”部分显示与内存中活动相关的统计信息。

其他变化

以下是此版本中的其他更改：

  • ASH Analytics页面中的更改

    在ASH分析页面上的“平均活动会话”图表中，您现在可以单击CPU等待类，以根据各种内存中操作以及用于内存操作之外的操作的总CPU来查看CPU使用情况。

    有关详细信息，请参阅“ 确定数据库活动中峰值的原因 ” 。

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle Database 12 Day +性能调优指南（适用于Oracle Database 12 c第1版（12.1.0.1））中的更改。

新功能

以下功能是此版本中的新增功能：

  • 实时数据库操作监控

    实时数据库操作监视有关活动和最近完成的数据库操作的跟踪和报告。您可以监视单个SQL或PL / SQL语句的执行详细信息。您还可以监视长时间运行的操作（如批处理作业或提取，转换和加载（ETL）处理）的进度。

    有关此功能的信息，请参阅“ 监视实时数据库操作 ” 。

不受支持的功能

Oracle不再支持Oracle Enterprise Manager Database Control。

其他变化

以下是此版本中的其他更改：

  • Oracle企业管理器云控制

    在以前版本的Oracle数据库中，您使用Oracle Enterprise Manager数据库控制（数据库控制）从图形用户界面管理数据库性能调整。在此版本中，您可以使用Oracle Enterprise Manager云控制（云控制）图形用户界面。云控制提供比数据库控制更多的功能。本指南中的过程使用Cloud Control。

    您必须与Oracle数据库分开安装Cloud Control。

    也可以看看：

    “Oracle企业管理器云控制基本安装指南”

  • ASH Analytics页面

    Cloud Control具有ASH Analytics页面，该页面以图形方式显示最近的活动会话历史记录信息。

    有关详细信息，请参阅“ 确定数据库活动中峰值的原因 ” 。

  • 实时ADDM

    Cloud Control具有实时ADDM页面，您可以从该页面实时运行自动数据库诊断监视，以诊断数据库缓慢或挂起的问题。

    有关详细信息，请参阅“ 实时诊断严重性能问题 ” 。
