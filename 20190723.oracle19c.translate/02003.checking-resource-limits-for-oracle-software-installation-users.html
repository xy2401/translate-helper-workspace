检查Oracle软件安装用户的资源限制

对于每个安装软件所有者用户帐户，请检查安装的资源限制。

在Oracle Linux系统上，Oracle建议您安装Oracle预安装RPM以满足预安装要求，例如配置操作系统以在limits.conf文件中设置资源限制。Oracle Preinstallation RPM仅为oracle用户配置limits.conf文件。如果要实现Oracle Grid Infrastructure作业角色分离，则将值从oracle用户复制到limits.conf文件中的grid用户。

要检查资源限制：

  1. 以安装所有者身份登录。
  2. 检查文件描述符设置的软限制和硬限制。确保结果在推荐范围内。例如：
    $ ulimit -Sn 1024 $ ulimit -Hn 65536
  3. 检查软限制和硬限制，以确定用户可用的进程数。确保结果在推荐范围内。例如：
    $ ulimit -Su 2047 $ ulimit -Hu 16384
  4. 检查堆栈设置的软限制。确保结果在推荐范围内。例如：
    $ ulimit -Ss 10240 $ ulimit -Hs 32768
  5. 对每个Oracle软件安装所有者重复此过程。

如有必要，请在/etc/security/limits.conf配置文件中更新安装所有者的资源限制。但是，配置文件可能是特定于分发的。请与系统管理员联系以获取特定于分发的配置文件

注意：

如果对Oracle安装用户帐户进行了更改并且该用户帐户已登录，则在将这些用户注销并重新登录之前，对limits.conf文件的更改不会生效。在使用这些帐户进行安装之前，必须执行此操作。
