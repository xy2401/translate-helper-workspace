7 Oracle Data Guard命令行界面参考

使用命令参考了解如何使用Data Guard代理命令行界面（DGMGRL）来管理代理配置。

DGMGRL使您可以直接从命令行或批处理程序或脚本管理Data Guard代理程序配置及其各种成员。您可以使用Data Guard代理命令行界面替代Oracle Enterprise Manager云控制（云控制）来管理Data Guard配置。

  • 启动Data Guard命令行界面

  • 退出Data Guard命令行界面

7.1启动Data Guard命令行界面

要启动Data Guard命令行界面（DGMGRL），请在安装了Oracle的系统上的命令行提示符处输入dgmgrl 。 

％dgmgrl

将显示DGMGRL命令提示符：

DGMGRL>

要运行DGMGRL，您必须具有SYSDG或SYSDBA管理权限。

7.1.1 DGMGRL可选参数

您可以在命令行上提供可选参数，以指示您希望Data Guard命令行界面如何显示输出。

输出包括命令提示，横幅和消息等项目。

另外，一个单命令模式可用。在此模式下，DGMGRL执行一个命令并在命令完成时退出。退出代码是命令的结果。如果退出代码为0，则命令成功完成。否则，出现错误。

DGMGRL的命令行如下：

％dgmgrl [<options>] [<logon> [<command>]]

在调用DGMGRL命令行界面时指定以下任何关键字：

  • <options>可以是以下选项之一：

    • -回声

      显示命令输入和输出到默认显示设备。如果不使用此参数，则仅显示命令的输出。

    • -logfile <file-spec> “< dgmgrl-command >”

      指定可以捕获DGMGRL命令行界面操作的文件。

      注意：

      自Oracle Database 12 c第2版（12.2.0.1）起，不推荐使用DGMGRL -logfile选项。它仅支持向后兼容性。而是应该使用START OBSERVER命令上的LOGFILE IS子句指定日志文件。

      也可以看看：

      • “ START OBSERVER ”命令

      • 捕获观察者日志文件中的观察者操作

    • -无声

      禁止在默认显示设备上显示DGMGRL（ DGMGRL> ）命令提示符。如果要将命令输出定向到文件或其他显示工具，则此选项很有用。

  • <logon>是：

    • 用户名[@connect-identifier]

      要连接到数据库，请输入username和（可选） connect-identifier 。然后会提示您输入密码。connect-identifier是完全指定的连接描述符或要由Oracle命名方法（例如，TNS）解析的名称。

      如果使用完全指定的连接描述符，则需要包含引号;否则连接将失败并出现invalid option错误。以下是使用引号连接的示例：
      dgmgrl sys @'（DESCRIPTION =（ADDRESS_LIST =（ADDRESS =（PROTOCOL = TCP）（HOST = sales-server）（PORT = 1521）））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））'
      输入密码： 密码

      无论是使用完全指定的连接描述符还是使用Easy Connect命名方法指定连接标识符，以下任何语法都是有效的（系统将提示您输入密码）：

      • dgmgrl username @' connect_identifier '

      • dgmgrl username @“ connect_identifier ”

      • dgmgrl username @“' connect_identifier '”

      警告：

      在调用DGMGRL时在命令行中包含密码是一种安全风险。通过在调用DGMGRL时省略密码并在提示时输入密码或使用外部身份验证方法，可以避免此风险。

    • 使用操作系统身份验证（远程数据库重新启动不起作用），安全套接字层（SSL）协议或存储在钱包中的数据库凭据时，可以以“/”连接。

  • <command>是一个命令。

    例如：

    dgmgrl sys“show database'North_Sales'”

    Password ： 密码

7.1.3 DGMGRL命令使用说明

此列表中的项目描述了DGMGRL特有的使用说明。

  • 该 DG_BROKER_START动态初始化参数设置为TRUE 。

  • 要启用需要重新启动实例而无需手动干预的代理操作，必须在包含主数据库实例和备用数据库实例的每台主机上配置Oracle Net Services。具体来说，listener.ora文件必须包含有关该实例的静态配置信息。必须将GLOBAL_DBNAME属性设置为db_unique_name _DGMGRL. db_domain 。有关其他信息，请参阅先决条件 。

  • 创建配置或添加数据库时使用的连接标识符必须可从配置中的任何主机解析。

  • 你必须有使用Oracle Data Guard命令行界面的SYSDG或SYSDBA权限。 不要在CONNECT命令中包含AS SYSDG或AS SYSDBA 。DGMGRL首先尝试AS SYSDG连接;如果失败，则尝试AS SYSDBA连接。

  • 如果在命令上指定了多个选项，则可以按任何顺序指定选项。

  • 每个DGMGRL命令结束时都需要分号。

  • a中指定的字符 DGMGRL命令字符串值被解释为小写字符，除非用双引号（“）或单引号（'）括起来。例如，数据库和DatAbaSe是等效的，但“database”和“DatAbaSe”不是。

  • 如果这些字符出现在字符串中，则可以使用反斜杠（\）来转义单引号（'），双引号（“）和反斜杠字符（\）本身。

  • 代理配置上的某些操作可能需要关闭并重新启动一个或多个数据库。在大多数情况下，如果满足以下条件，DGMGRL将自动关闭并重新启动给定数据库：

    • instance-name是SID（这适用于Cloud Control以及DGMGRL）。

    • 代理必须能够使用上一个CONNECT命令中给出的相同凭据连接到数据库，即使最后一个CONNECT命令用于连接到另一个数据库也是如此。

7.3 @（at sign）命令

@命令允许您执行存储在脚本文件中的DGMGRL命令。

您可以将一系列命令放入脚本文件中，然后使用@命令执行该文件。脚本中包含的命令按顺序执行。

格式

在DGMGRL中，语法如下：

DGMGRL> @ script_file_name

命令参数

旗 描述
-回声 打印脚本中的所有命令及其执行结果。

使用说明

使用此命令执行的脚本必须满足以下条件：

  • DGMGRL必须能够访问脚本;否则命令失败，因为DGMGRL无法打开文件。

  • 脚本中包含的每个DGMGRL命令都必须以分号结尾。

  • 允许递归@命令执行，但递归级别的限制为20。如果递归级别达到20，则执行中止并且不执行任何未执行的命令。因此，应谨慎使用@命令的自递归执行（例如，在abc.script本身中放置@ abc.script命令）。

  • 如果脚本中有START OBSERVER命令，则忽略之后的任何命令，因为START OBSERVER命令将DGMGRL会话转换为观察者。

    START OBSERVER IN BACKGROUND命令被视为正常命令;也就是说，执行后的任何命令。

  • 脚本中允许使用注释行，但必须使用分号终止它们。例如，脚本中允许以下注释：

    REM Hello World; -  再次问好！;

    双短划线必须后跟注释文本前的空格字符。

7.5添加数据库

DGMGRL ADD DATABASE命令将备用数据库添加到现有代理配置。

格式

ADD DATABASE 数据库名称 AS CONNECT IDENTIFIER是connect-identifier ;

命令参数

数据库名称

代理将用于引用此备用数据库的名称。它必须匹配（不区分大小写）相应数据库DB_UNIQUE_NAME初始化参数的值。

连接标识符

完全指定的连接描述符或要由Oracle Net Services命名方法（例如，TNS）解析的名称。您指定的值也用作DGConnectIdentifier数据库属性的初始值。

使用说明

  • 要发出此命令，必须连接到主数据库或已在配置中的已启用备用数据库。

  • 代理使用指定的connect-identifier与其他数据库中的指定数据库进行通信。因此，必须确保connect-identifier可用于从配置中的所有数据库寻址指定的数据库。例如，如果使用TNS作为命名方法，则必须确保作为配置一部分的每个数据库和实例上的tnsnames.ora文件包含connect-identifier的条目。连接标识符必须解析为相同的连接描述符。如果要添加的数据库是Oracle RAC数据库，则此处提供的connect-identifier必须到达Oracle RAC的所有实例，最好设置FAILOVER属性。

    也可以看看：

    Oracle数据库网络服务管理员指南

  • 如果无法建立连接，则代理不会将新数据库添加到配置中。

  • 必须先清除备用数据库上的所有远程重做传输目标，然后才能将其添加到配置中。

命令示例

以下示例显示如何添加名为South_Sales的数据库。 

DGMGRL> ADD DATABASE South_Sales AS CONNECT IDENTIFIER是South_Sales.example.com;添加了数据库“South_Sales”

7.9 CONVERT DATABASE

CONVERT DATABASE命令将物理备用数据库转换为快照备用数据库，或将快照备用数据库还原为物理备用数据库。

快照备用数据库是完全可更新的备用数据库。与物理或逻辑备用数据库一样，快照备用数据库从主数据库接收和归档重做数据。与物理或逻辑备用数据库不同，快照备用数据库不应用它接收的重做数据。在首次丢弃对快照备用数据库进行的任何本地更新之后，快照备用数据库转换回物理备用数据库之前，不应用快照备用数据库接收的重做数据。

快照备用数据库最适用于需要物理备用数据库的临时可更新快照的方案。请注意，由于快照备用数据库接收的重做数据在转换回物理备用数据库之前不会应用，因此执行角色转换所需的时间与需要应用的重做数据量成正比。

有关快照备用数据库的其他信息，请参阅Oracle Data Guard概念和管理 。

格式

CONVERT DATABASE db_unique_name TO {SNAPSHOT |物理}待命;

使用说明

  • 如果物理备用数据库是快速启动故障转移的目标，则无法将其转换为快照备用数据库。ORA-16668: operation cannot be performed on the fast-start failover target standby database错误将被返回。

  • 如果物理备用数据库的RedoRoutes配置属性设置为非NULL值，则无法将其转换为快照备用数据库。

  • 使用DGMGRL ADD DATABASE命令将现有快照备用数据库导入Oracle Data Guard代理配置。

  • 快照备用数据库不能成为切换或快速启动故障转移的目标。

  • 如果禁用快速启动故障转移，则快照备用数据库可以成为手动故障转移的目标。

  • 您可以使用SHOW CONFIGURATION或SHOW DATABASE命令验证转换结果。例如：

    DGMGRL> SHOW CONFIGURATION;配置 -  DRSolution保护模式：MaxPerformance成员：North_Sales  - 主数据库South_Sales  - 快照备用数据库快速启动故障转移：禁用配置状态：成功

  • 将快照备用数据库转换回物理备用数据库后，它将处于物理备用数据库APPLY-ON的缺省状态。

命令示例

示例1：将物理备用转换为快照备用

发出以下命令将物理备用数据库转换为快照备用数据库：

DGMGRL> CONVERT DATABASE'South_Sales'到SNAPSHOT STANDBY;将数据库“South_Sales”转换为快照备用数据库，请稍候...数据库“South_Sales”已成功转换

示例2：将快照备用转换回物理备用

发出以下命令将快照备用数据库转换回物理备用数据库：

DGMGRL> CONVERT DATABASE'South_Sales'到PHYSICAL STANDBY;将数据库“South_Sales”转换为Physical Standby数据库，请稍候...操作需要在数据库“South_Sales”上关闭实例“south_sales1”关闭实例“south_sales1”...数据库关闭。数据库已卸下。ORACLE实例关闭。操作需要在数据库“South_Sales”上启动实例“south_sales1”启动实例“south_sales1”...ORACLE实例启动了。数据库已安装继续转换数据库“South_Sales”...数据库“South_Sales”已成功转换

有关完整信息，请参阅启用快速启动故障转移 。

格式

启用FAST_START FAILOVER [仅限OBSERVE];

命令参数

OBSERVE ONLY:在发出此命令之前或之后启动的所有观察者都将以仅观察模式运行。

使用说明

  • 在发出此命令以启用快速启动故障转移之前，必须满足启用快速启动故障转移的先决条件中描述的先决条件 。

  • 发出ENABLE FAST_START FAILOVER命令不会触发故障转移，它只允许监视配置的观察者在条件允许故障转移时启动快速启动故障转移。

  • 您可以在连接到代理配置中的任何数据库时启用快速启动故障转移。

  • 如果在启用快速启动故障转移后未启动观察器，则会显示主备用数据库和目标备用数据库的ORA-16819警告。例如：

    DGMGRL> SHOW DATABASE'South_Sales';数据库 -  South_Sales角色：PRIMARY目标状态：TRANSPORT-ON实例：south_sales1数据库警告：ORA-16819：快速启动故障切换观察器未启动数据库状态：警告

  • 要为具有多个备用数据库的代理配置启用快速启动故障转移，主数据库上的FastStartFailoverTarget配置属性必须指定一个或多个可用的目标备用数据库。主数据库和目标备用数据库都必须具有：

    • 已配置备用重做日志

    • 必须在两个数据库中为已配置的保护模式正确配置重做传输

    Oracle还建议在主数据库和备用数据库上启用闪回数据库，以便在故障转移后恢复旧的主数据库。如果未启用，则在启用快速启动故障转移时将收到警告：

    DGMGRL> ENABLE FAST_START FAILOVER;警告：ORA-16827：已禁用闪回数据库

    启用快速启动故障转移的任务2和FastStartFailoverTarget提供了有关FastStartFailoverTarget配置属性的更多信息。

  • 启用快速启动故障转移后，必须遵守“启用快速启动故障转移时的限制”中所述的限制 。

7.43失败

FAILOVER命令调用故障转移，将命名（目标）备用数据库转换为主数据库的角色。

此类故障转移称为手动故障转移 。有关详细信息，请参阅手动故障转移

注意：

由于故障转移导致备用数据库转换为主要角色，因此应在主数据库发生故障或无法访问且无法及时恢复时执行故障转移。故障转移可能会也可能不会导致数据丢失，具体取决于故障转移时生效的保护模式以及目标备用数据库是否与主数据库同步。

如果主数据库未发生故障并且您希望当前主数据库和备用数据库切换角色而不丢失数据，请使用SWITCHOVER命令。

格式

FAILOVER TO 数据库名称 [IMMEDIATE];

命令参数

数据库名称

要故障转移到主数据库角色的物理，逻辑或快照备用数据库的名称。

使用说明

  • 始终尝试首先执行完整的故障转移，除非由于ORA-752或ORA-600 [3020]错误而导致Redo Apply停在故障转移目标上。如果发生了其中一个错误，则在继续之前，请按照http://support.oracle.com My Oracle Support说明1265884.1中的“解决待机恢复期间的ORA-752或ORA-600 [3020]”中的指导原则进行操作。只有在完全故障转移失败或刚刚记录的错误情况下才应执行立即故障转移。

  • 必须在主数据库失败之前启用指定的备用数据库。但是，已关闭的已启用备用数据库可能是故障转移操作的候选者。在这种情况下，使用DGMGRL STARTUP命令重新启动备用数据库，然后发出FAILOVER命令。

  • 故障转移在指定的备用数据库上运行，并将其角色更改为主数据库。旁观者备用数据库（未参与故障转移的数据库）仍处于备用角色。

  • 在发出FAILOVER命令之前，请验证您是否已连接到将成为新主数据库的备用数据库。如有必要，请发出CONNECT命令以连接到要进行故障转移的备用数据库。

  • 如果在没有任何选项的情况下发出FAILOVER命令，则选择作为故障转移目标的备用数据库会应用它在更改为主要角色之前收到的所有未应用的重做。这称为完全故障转移。

  • 如果代理配置在最大保护模式下运行，则手动故障转移操作将强制保护模式获得最大性能。重做传输服务设置不受影响。在故障转移操作之后，您需要为生成的配置恢复所需的保护模式。

    注意：

    通过快速启动故障转移，代理保留在故障转移之前生效的保护模式。

  • 如果使用IMMEDIATE选项发出FAILOVER命令，则不会尝试应用任何已接收的未应用重做。即使在备用数据库上配置了备用重做日志文件，此选项也可能导致应用程序数据丢失。此外，配置中的任何剩余备用数据库在恢复或重新创建之前无法正常运行。有关更多信息，请参阅角色更改后重新启用已禁用的数据库 。

  • 您可以执行手动故障转移或设置代理以执行快速启动故障转移。有关允许代理在条件允许故障转移时自动调用故障转移的信息，请参阅ENABLE FAST_START FAILOVER命令。

  • 如果启用了快速启动故障切换，则只能在快速启动故障切换目标备用数据库与主数据库同步或在主数据库的延迟限制内执行完全手动故障切换。 ，只有当观察者开始时。启用快速启动故障转移时，无法立即执行手动故障转移。

  • 如果在故障转移之前在前（失败）主数据库上启用了闪回数据库，则可以使用代理的REINSTATE命令恢复以前的主数据库（请参阅REINSTATE DATABASE命令）。

    如果对物理备用数据库执行了故障转移，则在备用数据库上启用了闪回数据库并且有足够的闪回日志可用时，可以恢复故障转移禁用的任何其他物理备用数据库。有关逐步说明，请参阅角色更改后重新启用已禁用的数据库 。

  • 原始主数据库只能在恢复或重新创建后作为备用数据库参与配置。

    警告：

    如果在故障转移之前仍有任何活动实例正在运行，则应关闭原始主数据库。

    也可以看看：

7.47 MIGRATE PLOGGABLE DATABASE

MIGRATE PLUGGABLE DATABASE命令允许您将可插拔数据库（PDB）从一个多租户容器数据库（CDB）迁移到同一主机上的另一个。

您可以将PDB从主CDB迁移到另一个主CDB，或将PDB从备用CDB故障转移到主CDB。 MIGRATE PLUGGABLE DATABASE命令具有以下先决条件：

  • 必须以这样的方式创建和启动目标CDB，使其能够在与源CDB相同的文件路径上访问PDB数据文件。

  • 源CDB和目标CDB必须各自处于不同的Data Guard代理配置中。

  • 源CDB可以是主数据库或物理备用数据库。

  • 如果源CDB是物理备用数据库，那么

    • 源CDB和目标CDB必须运行相同的Oracle版本和修补程序。

    • 源和目标CDB必须具有与COMPATIBLE初始化参数相同的设置。

    • 要迁移的PDB必须在其主CDB上关闭。

    • 如果PDB故障转移将导致数据丢失，则必须使用IMMEDIATE选项专门选择执行故障转移。

  • 如果源CDB是主数据库，那么

    • 目标CDB无法运行较低版本的Oracle。

    • COMPATIBLE初始化参数的设置不能设置为目标CDB上的值低于源CDB上的值。

  • 必须停止并从要迁移的源数据库的Oracle Clusterware存储库中删除PDB的服务。应对源数据库的Data Guard代理配置中的所有数据库执行此操作。

  • 目的地CDB必须是开放的。

  • 您必须具有源CDB和目标CDB的SYSDBA权限。

格式

MIGRATE PLUGGABLE DATABASE [IMMEDIATE] pdb-name
TO CONTAINER dest-cdb-name
使用XML-description-file
[CONNECT AS {/ @ dest-cdb-connect-identifer | dest-cdb-user / dest-cdb-pass @ dest-cdb-connect-identifer }];

命令参数

pdb-name
要迁移的PDB的名称。
dest-cdb-name
用于接收要迁移的PDB的CDB的数据库唯一名称。
XML-description-file
包含要迁移的PDB的描述的XML文件。该文件由MIGRATE PLUGGABLE DATABASE命令执行的SQL语句自动创建。
dest-cdb-user
具有SYSDBA访问目标CDB的用户的用户名。
dest-cdb-pass
与为dest-cdb-user指定的用户名关联的密码。
dest-cdb-connect-identifier
用于到达目标CDB的Oracle Net连接标识符。

使用说明

  • 默认情况下，当此命令用于PDB故障转移时，如果存在数据丢失的可能性，则会拒绝故障转移尝试。您可以使用IMMEDIATE选项覆盖此默认行为。

  • 如果源数据库是主数据库，则忽略IMMEDIATE选项。

  • 如果指定了连接标识符，则使用数据库凭据对目标CDB上的用户进行身份验证。

  • 操作系统凭据不能用于对目标CDB上的用户进行身份验证。必须指定连接标识符;不支持斜杠（/）。

  • 对于使用斜杠（/）指定连接标识符（例如， /@boston ）的情况，将从钱包中提取凭据。

  • 如果要指定连接字符串，可以使用以下选项：

    • /@ dest-cdb-connect-identifier （凭证从钱包中提取）

    • dest-cdb-user / dest-cdb-pass @ dest-cdb-connect-identifier （使用数据库凭据）

  • 要防止密码在命令行上可见，请仅指定具有连接标识符的用户名。然后会提示您输入密码。仅提供用户名和连接标识符时，支持以下选项：

    • dest-cdb-user @ dest-cdb-connect-identifier （使用数据库凭据）

    • dest-cdb-user /@ dest-cdb-connect-identifier （使用数据库凭据）

  • 如果完全从命令行中省略了连接字符串，则系统将提示您输入用户名和密码。支持以下选项：

    • /@ dest-cdb-connect-identifier （没有提示输入密码，从钱包中提取凭据）

    • dest-cdb-user @ dest-cdb-connect-identifier （使用数据库凭据）

    • dest-cdb-user /@ dest-cdb-connect-identifier （使用数据库凭据）

    • 中止

      以最快的速度关闭数据库，无需等待调用完成或用户断开与数据库的连接。未回滚未提交的事务。正在处理的客户端SQL语句将终止。连接到数据库的所有用户都是隐式断开连接，下一次数据库启动将需要实例恢复。如果后台进程异常终止，则必须使用此选项。

      不等待当前调用完成或用户断开与数据库的连接。禁止进一步连接。数据库已关闭并已卸除。实例已关闭，下次数据库启动时不需要实例恢复。

7.76 START OBSERVER

START OBSERVER命令在此主机（运行DGMGRL会话的位置）上启动快速启动故障转移观察器，如果此主机上没有为此配置运行的已注册观察器。

在使用此命令之前，必须先发出CONNECT命令以登录特定的代理配置。否则，将返回一条错误消息，指出您尚未登录。

格式

START OBSERVER [ observer-name ] [FILE is observer-file ] [LOGFILE IS logfile ] [TRACE LEVEL IS USER |支持];

命令参数

观察者名称
用于标识同一Data Guard代理配置中的观察者的名称。

  • 同一Data Guard代理配置中没有两个观察者可以具有相同的名称。

  • 如果没有为观察者指定名称，则使用默认观察者名称，即发出START OBSERVER命令的机器的主机名。

  • 观察者名称不区分大小写。

  • 字符串“NONAME”不能用作观察者名称。

观察者文件
指定运行时数据文件的路径和名称。如果未指定，则文件名默认为fsfo.dat ，路径为当前工作目录。
观察者日志文件的完整路径名。每个观察者都有自己的日志文件。

使用说明

  • 您最多可以注册三个观察者来监视单个Data Guard代理配置。每个观察者都由您在发出START OBSERVER命令时提供的名称标识。请参阅安装和启动Observer 。

  • 可选子句TRACE_LEVEL IS允许您控制完成并写入观察者日志文件的跟踪量。默认值为USER ，它将观察者日志内容限制为跟踪有关快速启动故障转移，主备用数据库和目标备用数据库的状态更改以及错误/警告消息的信息。将TRACE_LEVEL设置为SUPPORT增加跟踪信息量，以包括Oracle Support Services所需的较低级别信息。

  • 该必须在观察器计算机上安装Oracle客户端管理员工具包或完整的Oracle数据库企业版或Oracle个人版工具包，以监视要为其启用快速启动故障切换的代理配置。有关更多信息，请参阅启用快速启动故障转移的先决条件 。

  • 必须在观察器计算机上发出START OBSERVER命令。一旦观察者成功启动，在观察者停止之前（例如，通过从不同的客户端连接发出STOP OBSERVER命令），控件不会返回给用户。如果要与代理配置执行进一步交互，则必须通过另一个客户端进行连接。

    有关如何在后台启动观察器的信息，请参阅背景中的START OBSERVER 。

  • 如果使用LOGFILE IS子句，则所有观察者输出都记录在指定的文件中。如果无法访问指定的日志文件，或者未使用LOGFILE IS子句，则将观察者输出发送到标准输出。Observer输出对于解决观察者的问题和一般的快速启动故障转移非常有用。

  • 如果未使用FILE IS参数指定目录路径，则观察者将在当前工作目录中搜索fsfo.dat文件。如果fsfo.dat文件，这是第一次发出START OBSERVER命令，则观察者会创建一个fsfo.dat文件。

  • 主备用数据库DB_UNIQUE_NAME初始化参数和连接标识符存储在fsfo.dat配置文件中。Oracle建议您确保此文件免受未经授权的访问。

  • 在发出此命令之前，不需要启用快速启动故障转移。

    • 如果启用了快速启动故障转移，则观察器将从代理配置中检索主要和目标备用连接标识符，并开始监视配置。

    • 如果未启用快速启动故障切换，则观察器会持续监视何时启用快速启动故障切换。

  • 发出此命令时，只需要运行主数据库;将成为快速启动故障转移目标的备用数据库不需要运行，以便成功完成此命令。

  • 使用SHOW OBSERVER命令或SHOW CONFIGURATION VERBOSE命令，或查询主数据库上的V$FS_FAILOVER_OBSERVERS视图以查看观察者及其主计算机的状态。

  • 如果主数据库和目标备用数据库保持连接但它们失去与观察者的连接，则主数据库将进入未观察状态。该状态由经纪人的健康检查功能报告。

  • SHOW OBSERVER命令指示是否已启动一个或多个观察者。

    如果SHOW OBSERVER命令显示一个或多个已注册的观察者，但由于某些原因其中一些已不再运行，则可以执行以下任一操作：

    • 在最初启动它的同一观察器计算机上发出START OBSERVER命令，并在首次启动观察器时使用观察器配置文件。

    • 在任何计算机上发出STOP OBSERVER命令，然后发出START OBSERVER命令以启动观察者。

    如果SHOW OBSERVER命令显示一个或多个观察者并且一个观察者已经在一个位置运行，那么再次尝试在该位置启动观察者将失败，并显示以下错误：

    无法打开观察者文件

    如果SHOW OBSERVER命令显示三个已注册的观察者，并且您尝试在另一个位置启动观察者，则该命令将失败并显示以下错误：

    ORA-16647：无法启动超过三名观察员

7.77背景中的开始观察者

START OBSERVER IN BACKGROUND命令在此主机（运行此DGMGRL会话）上启动快速启动故障转移观察器作为后台进程。

发出此命令后，DGMGRL会报告START OBSERVER IN BACKGROUND命令是否成功提交。如果是，则控制返回给用户。（这与START OBSERVER命令的行为不同，在命令START OBSERVER控件不会返回给用户。）

此命令使用Oracle wallet获取凭据以登录数据库服务器并注册观察者。即使您已使用CONNECT命令成功连接到代理配置中的数据库服务器，此命令也会忽略现有连接并使用存储在Oracle钱包中的凭据。如果未配置钱包，则该命令将无法启动Observer。

格式

START OBSERVER [ observer-name ]在BACKGROUND [文件是观察者文件 ] [LOGFILE IS log-file ]
CONNECT IDENTIFIER是connect-identifier [TRACE_LEVEL IS USER |支持];

命令参数

观察者名称
用于标识同一数据保护代理配置中的观察者的名称。

  • 同一Data Guard Broker配置中没有两个观察者可以具有相同的名称。

  • 如果没有为观察者指定名称，则使用默认观察者名称，即发出START OBSERVER命令的机器的主机名。

  • 观察者名称不区分大小写。

  • 字符串“NONAME”不能用作观察者名称。

观察者文件
指定运行时数据文件的路径和名称。如果未指定，则文件名默认为fsfo.dat ，路径为当前工作目录。
观察者日志文件的完整路径。每个观察者都有自己的日志文件。
连接标识符
快速启动故障切换观察器将监视的代理配置中的数据库的连接标识符。Oracle wallet使用此标识符作为搜索oracle钱包的密钥。

使用说明

  • 即使成功提交START OBSERVER命令，观察者也可能由于凭据问题，间歇性网络连接或观察者注册失败而无法启动。要验证观察者是否已成功启动，请使用SHOW OBSERVERS命令或检查观察者日志文件。

  • 此命令忽略使用CONNECT命令对特定配置成员所做的任何连接。换句话说，即使您没有连接到代理配置中的特定成员，您仍然可以使用START OBSERVER IN BACKGROUND命令启动观察者。

7.79 STARTUP

STARTUP命令启动Oracle数据库实例，并允许您指定许多选项。

您可以指定的选项如下：

  • FORCE ：在重新启动之前，在SHUTDOWN ABORT模式下关闭当前的Oracle实例。

  • RESTRICT ：仅允许具有RESTRICTED SESSION系统特权的Oracle用户连接到该实例。

  • PFILE ：指定启动数据库实例时要使用的PFILE初始化参数文件。

  • MOUNT ：安装指定的数据库而不打开它。

  • OPEN ：安装并打开指定的数据库。

  • NOMOUNT ：在不挂载数据库的情况下启动指定的数据库实例。

格式

STARTUP [FORCE] [RESTRICT] [PFILE = filename ] [MOUNT |打开[ open_options ] | NOMOUNT];

命令参数

文件名

启动数据库实例时要使用的初始化参数文件的名称。如果未指定PFILE参数选项，则使用默认服务器参数文件（特定于您的操作系统）。

开放选项

您希望指定数据库启动的访问模式。可能的模式是：

  • 只读
  • 读写

使用说明

  • 使用不带参数的STARTUP命令等同于使用STARTUP OPEN命令。

  • 如果在使用STARTUP命令时未使用FORCE子句并且当前数据库实例正在运行，则会导致错误。在调试或发生错误情况时， FORCE子句很有用。否则，不应该使用它。

  • 使用RESTRICT子句仅允许具有RESTRICTED SESSION系统特权的Oracle用户连接到该实例。稍后，您可以通过SQL * Plus使用ALTER SYSTEM命令来禁用受限会话功能。

  • 如果不使用PFILE子句指定初始化参数文件，则STARTUP命令将使用默认服务器参数文件（如果存在）。否则， STARTUP命令使用默认初始化参数文件。默认文件是特定于平台的。

    有关默认参数文件的详细信息，请参阅特定于操作系统的文档。

  • 使用OPEN子句装入和打开指定的数据库。

  • NOMOUNT子句在不挂载数据库的情况下启动数据库实例。您不能将NOMOUNT子句与MOUNT或OPEN选项一起使用。

7.80停止观察员

STOP OBSERVER命令停止快速启动故障转移观察器。

格式

STOP OBSERVER [ 观察者名称 |所有];

命令参数

观察者名称
要停止的观察者的名称。如果未指定名称且配置只有一个已注册的观察者，则它将被停止;如果配置中有多个注册的观察者，则返回错误消息。

ALL关键字将停止在此代理配置中注册的所有观察者。

使用说明

  • 您可以在连接到代理配置中的任何数据库时发出此命令。

  • 此命令不会禁用快速启动故障转移，但在没有观察者的情况下无法启动快速启动故障转移。

  • 发出此命令时，无需启用快速启动故障转移。

  • 如果在发出STOP OBSERVER命令时启用了快速启动故障转移，则必须连接主数据库和备用数据库并相互通信。否则返回以下错误：

    ORA-16636快速启动故障转移目标备用数据库处于错误状态，无法停止观察者

    如果主数据库和备用数据库之间不存在连接，则可以在主数据库上发出DISABLE FAST_START FAILOVER FORCE命令，然后发出STOP OBSERVER命令。请注意，在与观察者和目标备用数据库断开连接的主数据库上，使用FORCE选项禁用快速启动故障转移不会阻止观察者启动到目标备用数据库的快速启动故障转移。

  • 如果在发出STOP OBSERVER命令时未启用快速启动故障转移，则在停止观察器时，只有主数据库必须正在运行。

  • 发出STOP OBSERVER命令时，观察者不会立即STOP OBSERVER 。观察者在下次观察者与经纪人联系时才发现它已被停止。

    一旦发出STOP OBSERVER命令，就可以在任何计算机上再次输入START OBSERVER命令。你可以立即开始一个新的观察者，即使旧的观察者还没有发现它被停止了。任何重启旧观察者的尝试都将失败，因为已经为代理配置启动了一个新的观察者。

  • 如果正在切换到新的快速启动故障转移目标或新的主STOP OBSERVER ，则STOP OBSERVER命令将失败。

  • 如果有两个或多个已注册的观察者并且您尝试仅停止主STOP OBSERVER ，则STOP OBSERVER命令将失败。

命令示例

以下示例将停止在代理配置中运行的所有观察者。

DGMGRL>停止观察所有;

7.82 SWITCHOVER

发出SWITCHOVER命令时，当前主数据库将成为备用数据库，指定的备用数据库将成为主数据库。

这称为切换操作。

格式

SWITCHOVER TO database_name [WAIT [ timeout_in_seconds ]];

WAIT选项指定您要在继续切换之前等待会话耗尽。代理确定所有当前活动服务的最大drain_timeout值，并在继续切换之前等待所有当前客户端请求处理的时间长度。drain-timeout值是SRVCTL实用程序的add service命令中指定的选项。WAIT选项对单个实例数据库没有影响;仅当服务配置了与Oracle Clusterware中的应用程序连续性相关的属性时，它才有效。

您还可以选择指定等待会话耗尽的秒数。您指定的值将覆盖drain_timeout值。

命令参数

数据库名称

要更改为主数据库角色的备用数据库的名称。

超时秒
在切换操作进行之前，允许资源耗尽的时间（以秒为单位）完成。

允许的值为“”（空字符串），0（零）或任何正整数。默认值为空字符串，表示未设置此选项。如果该值为零，则立即排空。在排水期间，处理所有当前客户端请求，但不接受新请求。

使用说明

  • 如果启用了快速启动故障切换，则可以仅切换到快速启动故障切换目标备用数据库。

  • 在开始切换之前，代理会验证主数据库和备用数据库是否处于以下状态：

    • 必须启用主数据库并处于TRANSPORT-ON状态，以便启动重做传输服务。

    • 必须启用备用数据库，并且处于APPLY-ON状态，并启动日志应用服务。

  • 只要您选择参与切换的备用数据库没有重做传输服务错误，代理就允许切换继续进行。但是，任何其他旁观者备用数据库发生的错误都不会阻止切换继续进行。

  • 当配置在最大保护模式下运行时，不允许切换到逻辑备用数据库。

  • 如果代理配置在最大保护模式或最大可用性模式下运行，则即使在操作之后，切换仍会保持保护模式（在执行切换操作之前进行了描述）。如果无法维护该模式，则不允许切换，因为切换的目标备用数据库是唯一满足保护模式要求的备用数据库。

  • 如果承担主要角色的备用数据库是物理备用数据库，则在切换完成后将重新启动旧的主数据库。如果备用数据库是逻辑备用数据库，则主数据库和逻辑备用数据库都不会重新启动。

  • 如果承担主要角色的备用数据库是物理备用数据库，则原始主数据库将成为物理备用数据库。

  • 如果承担主要角色的备用数据库是逻辑备用数据库，则原始主数据库将成为逻辑备用数据库。

  • 如果Oracle RAC主数据库正在成为物理备用数据库，则在执行切换之前，将关闭除主数据库的所有实例之外的所有实例。有关详细信息，请参阅切换

  • 您无法切换到快照备用数据库。

  • 如果承担主要角色的备用数据库是逻辑备用数据库，并且配置中有物理备用数据库，则在切换后，将禁用物理备用数据库。

    警告：

    因此，Oracle通常建议您指定物理备用数据库以进行切换而不是逻辑备用数据库。如果必须切换到逻辑备用数据库，请参阅在角色更改后重新启用已禁用的数据库以重新创建物理备用数据库。

    如果您打算相对较快地切换回原始主数据库，则可以允许物理和快照备用数据库保持禁用状态。完成切换回原始主数据库后，您可以重新启用物理和快照备用数据库，因为它们仍然是原始主数据库的可行备用数据库。

7.84验证数据库数据文件

VALIDATE DATABASE DATAFILE命令跨主数据库和备用数据库执行数据文件的验证。

数据文件的验证检测到任一数据库中丢失的写入。

格式

VALIDATE DATABASE [ database-name | ALL] DATAFILE [ datafile-name | datafile-number | ALL] OUTPUT =“ output-file-name ”;

命令参数

数据库名称

要为其显示信息的数据库的名称。VERBOSE关键字（如果使用）必须在database-name之前或返回错误。

如果要验证的数据库是primary或ALL ，则将所有备用数据库的数据文件与主数据库的数据文件进行比较。

如果要验证的数据库是备用数据库，则将其数据文件与主数据库的数据文件进行比较。

datafile-name |数据文件数
您可以通过名称（数据文件名称）或编号（数据文件编号）指定要比较的数据文件。

datafile-name是要验证的特定日期文件的名称。

datafile-number是数据文件的文件标识号（如V$DATAFILE视图的FILE#列所示）。

输出文件名
在服务器上生成的文件，您必须检查该文件以确定块比较是否已完成以及是否存在任何丢失的写入。输出文件在要比较的数据库的诊断trace目录中创建。

使用说明

  • VALIDATE DATABASE命令显示DATABASE的简短摘要，并报告检测到的任何错误或警告。

  • 发出VALIDATE DATABASE命令后，它会立即返回一条消息，表明数据文件已在数据库上启动，但这并不意味着数据文件比较已完成或数据文件之间没有丢失写入。您必须检查生成的输出文件，以确定数据文件比较是否已完成，或者是否有丢失的写入。

7.85 VALIDATE DATABASE SPFILE

VALIDATE DATABASE SPFILE命令执行主数据库与指定备用数据库之间的服务器参数文件（SPFILE）条目的比较。

格式

VALIDATE DATABASE { database-name } SPFILE;

命令参数

数据库名称

要为其显示信息的备用数据库的名称。VERBOSE关键字（如果使用）必须在数据库名称之前或返回错误。

如果要验证的数据库是备用数据库，则将主数据库上的服务器参数文件值与备用数据库上的服务器参数文件值进行比较。

使用说明

  • VALIDATE DATABASE SPFILE命令报告如果在主数据库和指定的备用数据库上没有差异以及参数列表中存在不同的值，则不会No parameter differences found 。

  • 发出VALIDATE DATABASE SPFILE命令后，它将根据DGConnectIdentifier属性的相应值建立与主数据库和指定备用数据库的连接。如果连接尝试无法成功完成，该命令将失败。
