1.171 LOG_ARCHIVE_DEST

使用LOG_ARCHIVE_DEST指定要将归档文件归档到的目标。

属性 描述

参数类型

串

句法

LOG_ARCHIVE_DEST = filespec

默认值

空值

可修改

改变系统

可在PDB中修改

没有

价值范围

除原始分区外的任何有效路径或设备名称

基本

没有

Oracle RAC

多个实例可以具有不同的值。

注意：

对于Enterprise Edition用户，不推荐使用此参数以支持LOG_ARCHIVE_DEST_ n参数。如果未安装Oracle Enterprise Edition或已安装Oracle Enterprise Edition，但尚未指定任何LOG_ARCHIVE_DEST_ n参数，则此参数有效。

只有在ARCHIVELOG模式下运行数据库或从归档重做日志恢复数据库时， LOG_ARCHIVE_DEST才适用。LOG_ARCHIVE_DEST与LOG_ARCHIVE_DEST_ n参数不兼容，并且当任何LOG_ARCHIVE_DEST_ n参数的值不是空字符串时，必须将其定义为空字符串（“”）或（''）。归档重做日志文件时，使用文本字符串指定磁盘文件或磁带设备的默认位置和根。（并非所有操作系统都支持存档到磁带。）该值不能是原始分区。

如果LOG_ARCHIVE_DEST没有明确定义，所有的LOG_ARCHIVE_DEST_ n参数为空字符串值， LOG_ARCHIVE_DEST设置为在实例启动的操作系统特定的缺省值。

要覆盖此参数指定的目标，请为手动归档指定其他目标，或使用SQL * Plus语句ARCHIVE LOG START filespec进行自动归档，其中filespec是新的归档目标。要永久更改目标，请使用语句ALTER SYSTEM SET LOG_ARCHIVE_DEST = filespec ，其中filespec是新的归档目标。

LOG_ARCHIVE_DEST和LOG_ARCHIVE_FORMAT都不必是完整的文件或目录说明符本身;在将变量替换为LOG_ARCHIVE_FORMAT并将两个参数连接在一起后，它们只需要形成一个有效的文件路径。
