acfsutil accel替换

目的

用新的加速器卷替换现有的加速器卷。

语法和说明

acfsutil accel replace -h acfsutil accel replace -a new_accel_volume primary_volume

acfsutil accel replace -h显示帮助文本和退出。

下表包含acfsutil accel replace命令可用的选项。

表16-122 acfsutil accel replace命令的选项

选项 描述

-a new_accel_volume

指定新加速器卷的名称。

primary_volume

指定装入Oracle ACFS文件系统的主卷的名称。

此命令允许管理员使用新的加速器卷替换现有的加速器卷。如果当前加速器卷已满并且无法增大，或者将加速器迁移到更快的卷，则该命令很有用。

必须在所有节点上卸载文件系统才能使用此命令。新加速器必须至少与现有加速器一样大。如果新加速器卷大于现有加速器卷，则管理员应在acfsutil accel replace操作之后重新装入文件系统后运行acfsutil resize -d new_accel_volume ，以使Oracle ACFS能够利用新加速器卷的整个空间。

您必须是root用户或Oracle ASM管理员用户才能运行此命令。

以下示例显示了acfsutil accel replace命令的acfsutil 。

例16-117使用acfsutil accel replace命令

$ / sbin / acfsutil accel replace -a new_accelerator_volume my_primary_volume

acfsutil compat set -h acfsutil compat set -c release_version mount_point

下表包含acfsutil compat set命令可用的选项。

表16-124 acfsutil compat set命令的选项

选项 描述

-c release_version

指定mount_point确定的Oracle ACFS文件系统的发行版本。

装入点

指定安装文件系统的目录。

使用-c release_version选项指定的值必须大于或等于磁盘组的COMPATIBLE.ADVM值，并且必须小于或等于正在运行的Oracle Grid Infrastructure版本。如果未指定—c release_version ，则使用COMPATIBLE.ADVM的值。设置兼容性后，无法降级。—c release_version选项可用于无法或不希望更新磁盘组的COMPATIBLE.ADVM和COMPATIBLE.ASM的情况，但您希望使用需要增加兼容性的Oracle ACFS功能。更新Oracle ACFS兼容性后，将无法再使用较早的Oracle Grid基础结构版本安装文件系统。有关Oracle ASM磁盘组兼容性属性的信息，请参阅“ 磁盘组兼容性 ” 。

acfsutil冻结

目的

暂时冻结文件系统中的修改活动。

语法和说明

acfsutil freeze -h acfsutil freeze [-f] mount_point

acfsutil freeze -h显示帮助文本和退出。

下表包含acfsutil freeze命令可用的选项。

表16-127 acfsutil freeze命令的选项

选项 描述

-F

指定在冻结文件系统之前将数据刷新到磁盘的命令。

装入点

指定安装文件系统的目录。

acfsutil freeze命令暂时停止对文件系统的修改活动。您可以使用acfsutil freeze在不停止应用程序的情况下跨不同快照创建时间点映像。到了后恢复活动acfsutil freeze命令已经在文件系统上发出，运行acfsutil thaw命令。有关acfsutil thaw信息，请参阅acfsutil解冻 。

请谨慎使用acfsutil freeze命令以防止应用程序超时和可能的实例驱逐。

如果操作修改磁盘上复制配置信息或操作是对冻结备用文件系统的更新，则在Oracle ACFS复制处于活动状态的文件系统上使用acfsutil freeze命令可能会阻止复制操作。可能受影响的复制操作包括：

  • acfsutil repl reverse standby

  • acfsutil repl sync

  • acfsutil repl terminate primary

  • acfsutil repl terminate standby

acfsutil信息文件

目的

显示Oracle ACFS文件系统中文件的信息。

语法和说明

acfsutil信息文件-h acfsutil信息文件[[-d] [-o acfs_extent_offset ] [-m] [-u] | [-c]] 路径 [ 路径 ...]

acfsutil info file -h显示帮助文本并退出。

表16-128包含acfsutil info file命令可用的选项。

表16-128 acfsutil info file命令的选项

选项 描述

-d

显示详细的范围信息，从Oracle ACFS到磁盘组中的Oracle ASM设备。

-o acfs_extent_offset

acfs_extent_offset - 将范围显示过滤到指定的Oracle ACFS文件偏移量。

-C

仅显示范围计数。

-m

显示有关文件元数据范围的详细信息-m选项与-d ， -o和-u兼容。-m选项与-c不兼容。

-u

显示范围使用信息。

路径

指定文件的完整路径名。

该acfsutil info file大约在Oracle ACFS文件系统中的文件命令显示的信息。该信息包括范围图，该范围图详细说明了包含文件的块的位置以及文件的使用与分配的存储。还显示文件的标记信息。

—u选项向标准acfsutil info file输出添加一个附加列，指示是否在快照父级和子级之间继承acfsutil info file 。-c选项显示总范围计数以及主要和加速器卷范围节点的数量，同时省略每个范围的详细信息。当您想要显示范围数量以及使用加速器时， —c选项的输出非常有用，以确保元数据位于加速器卷上。

您必须具有对指定文件的读访问权才能运行acfsutil info file 。

acfsutil info fs

目的

显示详细的Oracle ACFS文件系统信息。

语法和说明

acfsutil info fs -h acfsutil info fs [-l] [{-o option_item | -s [ interval [ count ] [ -  d]] | -f [-v]}] [ mount_point ]

acfsutil info fs -h显示帮助文本并退出。

表16-129包含acfsutil info fs命令可用的选项。

表16-129 acfsutil info fs命令的选项

选项 描述

-l

显示由装入点指定的Oracle ACFS文件系统的符号链接，或者如果未指定装入点，则显示所有已装载的Oracle ACFS文件系统的符号链接。

-o

显示以下列表中的特定文件系统选项：

  • allvolumes [ mount_point | primary_volume_device | auxiliary_volume_device ] - 列出与文件系统关联的所有卷。

  • autoresize - 如果启用自动调整大小，则为1，否则为0。

  • autoresizeincrement - 列出文件系统的自动调整大小增量。

  • autoresizemax - 列出自动调整大小的最大值。

  • auxvolume { mount_point | primary_volume_device } - 列出与文件系统关联的辅助卷。可以针对安装点或设备运行。

  • available - 文件系统可用的时间长度。

  • compatacfs - 列出ACFS兼容性设置

  • compatadvm - 列出ADVM兼容性设置。

  • diskgroup - 与path关联的磁盘组的名称。

  • fetasize - 文件条目表大小，它是inode的磁盘元数据占用的存储量。

  • freespace - 文件系统中可用空间量。

  • isadvm - 如果存在Oracle ADVM卷，则为1，否则为0。

  • isauxvolume { volume_device } - 如果卷是辅助卷则为1，否则为0。必须针对卷设备运行。

  • isavailable - 如果文件系统可用，则为1;如果文件系统不可用，则为0。

  • iscompression - 文件系统上的压缩状态。 1表示启用，0表示禁用。

  • iscorrupt - 如果文件系统损坏iscorrupt 1，如果文件系统未损坏则为0。

  • isencryption - 文件系统上的isencryption状态，0表示禁用，1表示已启用。

-o （续）

  • ismountpoint - 1，如果path是安装点，0，如果path是不是一个安装点。

  • isreplprimary - 如果是主文件系统isreplprimary 1，否则为0。

  • isreplstandby - 如果是备用文件系统isreplstandby 1，否则为0。

  • issecurity - 文件系统的安全状态，0表示禁用，1表示已启用。

  • metablocksize - 文件系统metablocksize数据块的大小。

  • mountpoints - 与文件系统关联的挂载点。

  • primaryvolume - 与文件系统关联的设备的路径名。

  • redundancy - Oracle ADVM卷使用的redundancy类型。

  • replication - 如果已将文件系统初始化为主要或备用文件系统进行复制，则为1;如果未初始化为复制，则为0。

  • resizeincrement - path使用的Oracle ADVM卷的分配单元大小（以字节为单位）。

  • stripewidth - path使用的Oracle ADVM卷的条带宽度。

  • stripecolumns - path使用的Oracle ADVM卷的条带集列。

  • totalspace - 文件系统的大小。

  • volumes - 与文件系统关联的volumes数。

-s [ interval [ count ] [ -d] ]

显示文件系统统计信息

可选的interval参数指定每个报告之间的时间量（以秒为单位）。第一个报告包含自文件系统装载以来的时间统计信息。每个后续报告都包含自上次报告以来间隔期间收集的统计信息。

可以使用interval参数指定可选的count参数。如果指定了count参数，则count的值将确定以间隔秒间隔生成的报告数。

如果指定了interval参数而没有count参数，则该命令会连续生成报告，直到您使用Ctrl+c中断命令。

可选的-d选项显示更详细的统计信息。

-f [ -v ]

显示文件系统碎片。-v选项显示其他碎片信息。

装入点

指定安装文件系统的目录。不是复制数据的有效选项。

acfsutil info fs显示有关Oracle ACFS文件系统的信息和统计信息。如果已在文件系统上启动压缩，加密，安全性或复制，则会显示其他标志和状态信息。

-o option_item选项显示指定的option_item文件系统信息。

-s选项显示当前在运行命令的节点的文件系统上的更改量和速率。

如果未指定任何选项，该命令将显示文件系统信息，其中包括卷设备名称，文件系统的大小，卷设备上可用的空间量，此节点上的文件系统装入时间，文件系统的状态，用户指定的块大小，Oracle ACFS快照的数量，文件系统中快照消耗的空间以及与文件系统关联的可选名称或卷标。可能的文件系统状态显示在flags行中。这些州包括：

  • Offline表示无法访问基础设备，可能是由于Oracle ASM实例故障，磁盘组强制卸载或无法恢复的I / O错误。此节点上的文件系统只能卸载。所有其他访问尝试都会导致错误。

  • Available表示文件系统已联机且可运行。

  • Corrupt表示应尽早检查和修复文件系统，以纠正检测到的不一致。例如，在Linux上运行fsck命令或在Windows上运行acfschkdsk命令以检查和修复文件系统。确保在修复模式下运行命令以更正问题。有关更多信息，请参阅“ fsck（离线模式） ”和“ acfschkdsk ” 。

在Windows系统上使用acfsutil info fs查看空间使用值时，这些值可能与Windows文件夹中的大小不同。Windows系统上的“ 文件夹 属性”使用的机制仅考虑文件，应视为近似值。

任何用户都可以运行acfsutil info fs 。

目的

表16-133包含acfsutil plugin enable命令可用的选项。

表16-133 acfsutil plugin enable命令的选项

选项 描述

-t tag ，...

选择要监视的Oracle ACFS文件的标记名称的可选列表。标签的最大数量为12。名称以逗号分隔。默认设置是监视Oracle ACFS文件系统中的所有文件。

标签名称由Oracle ACFS标记API管理。

-m metrictype

选择要使用每个调出消息从Oracle ACFS传输到插件模块的消息有效负载。Oracle ACFS提供acfsmetric1和acfsmetric2预定义的度量标准类型。

-i interval [ s | m ]

选择发布消息传递和发布间隔。Oracle ACFS插件机制在每个指定的消息间隔期间发布度量标准消息。

间隔是指标发布之前可以通过的最长时间。应用程序在API调用中阻止检索指标，直到发布为止。

间隔可以指定为s秒或m分钟的整数值。默认值为分钟。间隔允许的最大值为60分钟或3600秒。

如果指定了间隔，则请求Oracle ACFS以间隔为基础将指标消息发布到插件应用程序。

如果未指定间隔，则应用程序将轮询Oracle ACFS文件系统中的度量标准更新。这是默认操作。

装入点

指定安装文件系统的目录。

acfsutil plugin enable为应用程序插件服务enable Oracle ACFS文件系统。Oracle ACFS插件仅针对单个节点启用，可以是独立主机，也可以是网格群集的一个或多个单个节点。启用插件后，Oracle ACFS文件系统开始收集指定标记文件集或Oracle ACFS文件系统中所有文件的度量标准信息，并根据所选消息有效负载将标注消息传输到插件应用程序和交付参数。每次交付给应用程序后都会重置度量标准（无论是通过轮询还是发布）。

通常，从用户应用程序的角度收集度量。用于元数据或内部Oracle ACFS活动的I / O;例如快照写时复制，加密和复制文件;不包括在内。针对原始文件系统和快照文件中的两个文件的用户I / O收集度量标准。导致I / O操作的文件的内存映射记录在度量标准中。复制相关文件下.指标中不包含ACFS/repl目录。

Oracle ACFS提供acfsmetric1和acfsmetric2预定义的度量标准类型。

使用此命令需要管理员权限，或者您必须是Oracle ASM管理员组的成员。

仅调整指定设备的大小。

装入点

指定安装文件系统的目录

必须至少指定一个+或— ， —a或—x选项。

根据文件系统的块大小和Oracle ADVM卷设备文件的分配单元对大小值进行舍入。要确定Oracle ADVM卷设备调整大小增量，请检查V$ASM_VOLUME视图中的RESIZE_UNIT_MB字段， asmcmd volinfo输出中的Resize Unit ，或acfsutil info fs输出中的ADVM resize increment 。

在发出acfsutil size命令来增大或缩小文件系统之前，您可以发出acfsutil size -q来报告可能需要移动多少数据才能完成命令。此估计有助于预测命令可能需要多长时间才能完成。使用-q选项不会导致文件系统的任何大小调整。

对于ADVM兼容性属性设置为11.2.0.4或更高的磁盘组，可以扩展文件系统的次数没有限制。

对于ADVM兼容性属性设置为小于11.2.0.4的磁盘组，文件系统的内部存储位图限制为5个扩展区。这会导致任何尝试在文件系统增加四次或更多次后将文件系统增加失败。但是，如果在增加文件系统四次或更多次后文件系统大小减小，那么如果增加的大小小于减小的大小，则可以再次增加文件系统大小。当达到文件系统扩展的限制时，使用-a选项运行fsck或acfschkdsk可以合并内部存储位图，从而允许将来的文件系统扩展。

作为asmadmin组成员的root或用户可以运行此命令。有关操作系统组权限的信息，请参阅关于Oracle ASM的权限 。

手动调整大小选项

+或-选项增大或缩小已安装的Oracle ACFS及其底层Oracle ADVM存储，以匹配指定的新大小。此操作还会调整基础Oracle ADVM卷文件的大小，以匹配指定的新长度。但是，磁盘组必须具有足够的可用存储空间，以适应文件系统大小的任何请求增加。

您可以使用acfsutil size来扩展或收缩整个文件系统或仅扩展或收缩指定的设备。

减小文件系统大小会将位于文件系统末尾的未使用的存储空间返回到磁盘组。

在Oracle ASM版本18c之前，只有在操作不需要移动文件系统中的现有文件时，收缩操作才会成功。对于Oracle ASM版本18c，收缩过程在缩小文件系统时根据需要移动文件。这个移动文件的过程可能非常耗时。此外，增长文件系统的后续命令可能还需要移动数据，这可能是耗时的。

注意：

  • 尝试扩展或收缩文件系统可能部分成功，导致文件系统的大小比发出acfsutil size命令之前更大或更小。但是，由于错误或环境变化，新大小可能不是请求的大小。如果由于任何原因导致节点上的增长或收缩操作失败，则会向USM事件日志和OKS日志写入错误消息。

  • 加速器卷的缩小不会导致元数据从加速器卷迁移到主卷。如果加速器上有太多数据，则该过程失败。

  • 收缩命令将文件移出要移除的卷区域并进入卷的不同部分。这种文件移动可能会改变这些文件的碎片。这可能导致它们变得比文件系统缩小操作之前更加碎片化。它也可能导致它们变得不那么分散。

自动调整大小选项

自动调整大小选项（ -a选项）指定如果文件系统中的可用空间量低于增量指定的量，Oracle ACFS文件系统将自动增长的增量。您还可以指定使用自动调整大小选项时允许的最大大小（ -x选项）。acfsutil info fs命令的输出显示自动调整大小增量和最大量。如果在没有-a选项的情况下仅提供-x选项（并且之前未指定自动调整大小增量），则在增量仍为零时发出警告。如果先前通过先前的命令调用设置了增量，则命令成功。自动调整大小需要将ADVM兼容性属性设置为12.2或更高。

如果由于任何原因导致节点上的自动调整大小失败，则会暂时禁用节点上的自动调整大小。将错误消息写入USM事件日志和OKS日志。自动调整大小仍可能在另一个节点上进行;但是，如果失败是由于基础卷超出可用空间，则所有节点都会快速禁用自动调整大小。文件系统会继续定期尝试以越来越长的间隔自动调整大小。调整大小成功后，例如当更多空间添加到基础Oracle ASM磁盘组时，将在节点上重新启用自动调整大小。请注意，即使一个节点成功自动调整文件系统大小，其他节点也可能仍处于错误状态。可以通过重新安装文件系统或使用-a或-x选项再次运行acfsutil size命令来手动重新启用自动调整大小，以更新自动调整大小设置。当文件系统已增长到使用acfsutil size -x指定的最大大小时，自动调整大小仍保持启用状态，但在最大值增加之前不会进一步增大文件系统。

-d参数与-a和-x自动调整大小参数不兼容。除非使用-d指定卷设备，否则假定调用方正在指定应用于主卷的存储量，该值表示文件系统的大小。在使用加速器卷增长文件系统时， acfsutil size命令确定是否需要增加加速器大小。如果加速器必须更大且无法增加，则命令将失败并且主卷保持相同的大小。收缩文件系统时，只缩小主卷。如果使用-d选项指定卷设备，则指定的大小将应用于主卷或加速器卷（以指定的为准），并且仅更改该卷设备。如果生成的加速器卷大小不足以容纳主卷大小，则该命令会警告用户。

使用加速器卷并自动调整大小时，使用自动resize -a参数指定的增量将应用于主卷;但是，主要音量和加速器音量都受到监控。自动调整大小会使指定的增量增加主卷，或者根据需要增大。自动调整大小监视器并根据需要增加加速器音量以及更小的增量。当自动调整大小需要增加主卷时，如果加速器上没有足够的空间来容纳新的增长，它首先尝试增加加速器，如果失败则主卷保持相同的大小。

除了由于可用空间低于配置的自动调整大小增量而被触发之外，还会在其他条件下进行自动调整大小，例如：

  • 如果文件系统中的可用空间低于当前文件系统大小的10％或5 GB，则以较小者为准。

  • 如果写入时复制在文件系统中找不到足够的连续空间来为写入分配新存储空间。

  • 如果磁盘碎片整理程序找不到足够的连续空间来分配，以便对文件中8 MB的扩展区进行碎片整理。

这些条件可防止大型文件系统具有足够的总可用空间，因此无需自动调整大小，但空间非常分散，应用程序无法使用它。

调整非稀疏文件的大小以使其增长超过配置的自动调整大小间隔时，文件系统会自动调整自动调整大小增量的倍数，以便可以满足调整大小请求。

例子

以下示例显示了acfsutil size的使用。此示例将/acfsmounts/acfs1文件系统的主设备文件增加500 MB。

例16-136使用acfsutil size命令

$ / sbin / acfsutil size + 500M / acfsmounts / acfs1
