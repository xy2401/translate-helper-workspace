1.1.1 Oracle数据库示例

Oracle数据库示例包括各种示例和产品演示，您可以使用它们来了解Oracle数据库的产品，选项和功能。

其中许多例子都是为了与之合作而设计的示例模式，您可以在任何Oracle数据库中安装。Oracle文档库中的许多文档都使用Oracle数据库示例提供的示例程序和脚本。

1.1.1.1关于Oracle Data Mining演示

Oracle Data Mining演示包括可帮助您了解Oracle Data Mining API的数据挖掘示例程序。

数据挖掘示例程序说明了数据准备，算法选择，算法调整，测试和评分的典型方法。每个程序都在数据库中创建一个挖掘模型。所有程序都包含大量内联注释，以帮助您理解代码。

1.1.1.2关于Oracle全球化支持演示

Oracle全球化演示展示了Oracle全球化开发工具包的功能和API。

Oracle全球化开发工具包（GDK）提供了一个完整的框架，可以简化全球化互联网应用程序的开发。GDK示例演示演示了各个功能，例如区域设置敏感数据格式化，语言排序，区域设置映射，区域设置确定和语言检测。Oracle Store演示说明了如何集成GDK功能以构建完整的多语言J2EE Web应用程序，该应用程序为具有完全不同的区域设置首选项的用户提供服务。

1.1.1.3关于Oracle Message Gateway Common Companion Files

Oracle Messaging Gateway支持基于非Oracle消息传递系统和Oracle数据库高级队列的应用程序之间的通信。

Oracle Database Advanced Queuing提供两个Oracle Database Advanced Queuing队列之间的传播，以启用电子商务（HTTP到IDAP）。Messaging Gateway将此扩展到基于非Oracle消息传递系统的应用程序。为Messaging Gateway提供的示例说明了使用包含JMS，LCR和XML的消息格式与Oracle AQ，WebSphere MQ Series和Tibco Rendezvous一起传播的消息。

Messaging Gateway提供了七个不同的示例，每个示例都位于各自的子目录中。每个样本都包含一个README文件，其中包含有关设置和执行样本的详细信息。

1.1.1.4关于Oracle多媒体演示

Oracle Multimedia（以前称为Oracle inter Media）是一项功能，使Oracle数据库能够以与其他企业信息集成的方式存储，管理和检索图像，音频，视频或其他异构媒体数据。

Oracle Multimedia将Oracle数据库的可靠性，可用性和数据管理扩展到传统，互联网，电子商务和富媒体应用程序中的多媒体内容。它包括您可以使用的几个脚本和示例应用程序。它们包括OCI，SQL，Java，PL / SQL和ASP / VBScript示例应用程序（演示）。

1.1.1.5关于Oracle预编译器演示

Oracle预编译器演示向应用程序员展示了如何以与ANSI标准兼容的方式在C / C ++和COBOL代码中嵌入SQL操作。

Pro * C程序是包含嵌入式SQL语句的C程序。Oracle Pro * C / C ++和Pro * COBOL Demos提供了支持动态SQL操作的扩展，这些操作可以实时更改。也可以通过PL / SQL代码（称为匿名PL / SQL块）与Pro * C / C ++和Pro * COBOL应用程序一起使用动态SQL操作。典型应用程序包含比动态SQL更多的静态SQL。预编译器将每个嵌入式SQL语句转换为对预编译器运行时库（SQLLIB）的调用。

Oracle Spatial and Graph是一组集成的功能和过程，可以在Oracle数据库中快速有效地存储，访问和分析空间数据。空间数据表示真实或概念对象的基本位置特征，因为这些对象与它们存在的真实或概念空间相关。

Oracle Internet Directory Client演示程序向应用程序编程人员展示了如何使用LDAP C和PL / SQL API。

1.2.1可用磁盘空间

验证包含Oracle主目录的文件系统是否至少具有：

  • UNIX系统上1.1 GB的可用磁盘空间。

  • Windows系统上800 MB的可用磁盘空间。

以下主题包含有关在UNIX和Windows平台上确定可用磁盘空间的详细信息：

  • 确定UNIX平台上的可用磁盘空间

1.2.1.1确定UNIX平台上的可用磁盘空间

说明如何检查UNIX平台上的可用磁盘空间。

使用以下命令检查Oracle主目录是否有足够的空间：
  • df —k oracle_home_path
其中oracle_home_path是Oracle主目录的路径。
1.2.1.2确定Windows系统上的可用磁盘空间

说明如何确定Windows系统上的可用磁盘空间。

执行以下步骤以检查Oracle主目录是否有足够的空间：
  1. 双击“ 我的电脑”。
  2. 右键单击要安装Oracle软件的驱动器。
  3. 选择属性。

1.3访问安装软件

说明如何识别Oracle数据库安装使用的Oracle主目录。

在从Oracle数据库示例介质安装产品之前，必须确保您的系统可以访问Oracle Database 19c Oracle主目录。如果需要，您必须先安装或升级到此版本的Oracle数据库。

标识Oracle数据库安装使用的Oracle主目录。

在UNIX平台上， oratab文件位于/etc目录或/var/opt/oracle目录中。使用more命令检查oratab文件的内容。例如：

#more / etc / oratab

在Windows平台上：

  1. 从“开始”菜单中选择“ 程序” ，然后选择“ Oracle - HOME_NAME” ，“ Oracle安装产品” ，“ Universal Installer” 。
  2. 出现“欢迎”屏幕时，单击“ 已安装的产品” 。
    将显示“清单”屏幕，其中列出了系统上的所有Oracle主目录以及每个Oracle主目录中安装的产品。
  3. 在Inventory屏幕上，展开每个Oracle主目录并找到Oracle Database 19c 。
  4. 单击关闭 ，然后单击取消以退出Oracle Universal Installer。

1.4.1使用Oracle Universal Installer安装Oracle数据库示例

您必须在现有Oracle Database 19c Oracle主目录中安装Oracle数据库示例。

执行以下步骤以安装Oracle数据库示例：
  1. 登录安装Oracle数据库的计算机。

    在UNIX平台上，以Oracle软件所有者用户（ oracle ）身份登录并设置DISPLAY环境变量。

    在Windows平台上，以管理员或Administrators组成员的身份登录到要安装Oracle组件的计算机。如果要在主域控制器（PDC）或备份域控制器（BDC）上进行安装，请以Domain Administrators组成员的身份登录。

  2. 验证您计划使用的Oracle数据库是否可访问。
    有关详细信息，请参阅“访问安装软件” 。
  3. 导航到下载安装文件的目录。
  4. 解压缩zip文件的内容。
  5. 运行以下命令以启动Oracle Universal Installer：
    在UNIX平台上：
    $ ./runInstaller
    在Windows平台上：

    双击setup.exe 。

  6. 在“指定安装位置”屏幕上，输入以下内容：
    • Oracle Base：默认情况下显示Oracle基本路径。您可以根据需要更改路径。指定Oracle Base路径。

      Oracle基本目录是Oracle软件安装的顶级目录。

    • 软件位置：在“软件位置”字段中，接受默认值或输入要安装Oracle数据库示例的Oracle主目录路径。您必须在现有Oracle数据库主目录中安装Oracle数据库示例。

      注意：

      如果您尝试在与当前版本不同的Oracle数据库主目录中安装Oracle数据库示例，则安装将失败并显示错误。

    如果Oracle Real Application Clusters（Oracle RAC）安装在您选择的Oracle主目录中，则Oracle Universal Installer将显示“网格安装选项”屏幕。示例软件安装在所选Oracle RAC主目录中的所有节点上。表中列出的节点是Oracle Inventory已在其上注册已安装的Oracle RAC主目录的节点。
    单击“ 下一步”之前，请确保列出的每个节点都已启动并运行

    注意：

    在Oracle RAC安装上，该软件安装在所有群集节点上。
  7. 在“执行先决条件检查”屏幕上，检查并更正Oracle Universal Installer检查系统时可能发生的任何错误。如果单击“ 再次检查” ，则可以再次运行先决条件检查以查看是否满足最低要求以继续安装。如果希望安装程序修复问题并再次检查系统要求，请单击“ 修复并再次检查”。

    注意：

    Fix＆Check Again选项生成一个脚本（仅限UNIX），该脚本将某些系统参数设置为Oracle建议的值。您必须以root用户身份运行脚本。Oracle建议您不要修改此脚本的内容。
    从列表中选择一个选项：
    • 全部显示 ：获取运行的所有预先检查检查的列表。
    • 显示失败 ：获取失败的预审检查列表。
    • 显示成功 ：获取成功的预先检查检查列表。
  8. 查看“摘要”屏幕上显示的信息，然后单击“ 安装” 。

    注意：

    单击“ 保存响应文件”以将所有安装步骤保存到响应文件中。稍后，您可以使用此文件进行静默安装。
  9. 安装产品时会出现“安装产品”屏幕。安装操作包括执行诸如文件复制和在Windows系统上添加服务以及在UNIX系统上进行链接以及执行决策点和计算等操作。
    安装完成后，将显示“完成”屏幕。
  10. 单击“ 关闭”以退出Oracle Universal Installer。

1.4.2使用响应文件静默安装Oracle数据库示例

静默安装在后台运行，不需要用户输入。通常不会显示用户通常看到的交互式对话框。如果必须在多台计算机上多次安装Oracle产品，则无提示安装非常有用。

要执行Oracle数据库示例的静默安装，必须以静默或响应文件模式启动Oracle Universal Installer。输入类似于以下内容的命令：
  1. 在UNIX平台上：
    $ / directory_path / runInstaller [-silent] -responseFile responsefilename

    注意：

    不要指定响应文件的相对路径。如果指定相对路径，则Oracle Universal Installer会失败。

    哪里：

    • directory_path是DVD或硬盘驱动器上数据库目录的路径。
    • -silent以静默方式运行Oracle Universal Installer。
    • responsefilename是您配置的安装响应文件的完整路径和文件名。
  2. 在Windows平台上：
    DRIVE_LETTER：\ setup.exe_location setup [-silent]“variable = setting”[-nowelcome] [-noconfig] [-nowait] -responseFile

    哪里：

    • DRIVE_LETTER ：标识响应文件的完整路径。
    • setup.exe_location ：指示位置setup.exe 。
    • "variable=setting"是指响应文件中您可能更喜欢在命令行运行而不是在响应文件中设置的变量。将变量及其设置括在引号中。
    • -noconfig ：禁止在安装期间运行配置助手。
    • -nowait ：静默安装完成后关闭控制台窗口。
