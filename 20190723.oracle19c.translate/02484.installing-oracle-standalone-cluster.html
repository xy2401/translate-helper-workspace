安装Oracle Standalone Cluster

完成此过程以安装Oracle Standalone Cluster的Oracle Grid Infrastructure软件。

  1. 作为grid用户，下载Oracle Grid Infrastructure图像文件并将文件解压缩到Grid主目录。例如：
    $ mkdir -p /u01/app/19.0.0/grid $ chown grid：oinstall /u01/app/19.0.0/grid $ cd /u01/app/19.0.0/grid $ unzip -q download_location/grid.zip

    grid.zip是Oracle Grid Infrastructure映像zip文件的名称。

    注意：

    • 您必须将zip映像软件解压缩到您希望Grid主页所在的目录中。

    • 仅将Oracle Grid Infrastructure映像文件下载并复制到本地节点。在安装过程中，软件将复制并安装在群集中的所有其他节点上。

  2. 配置共享磁盘以与Oracle ASM筛选器驱动程序一起使用：
    1. 以root用户身份登录，并将环境变量ORACLE_HOME设置为Grid home的位置。

      对于C shell：

      $ su root #setenv ORACLE_HOME /u01/app/19.0.0/grid

      对于bash shell：

      $ su root #export ORACLE_HOME = / u01 / app / 19.0.0 / grid
    2. 使用Oracle ASM命令行工具（ASMCMD）配置磁盘设备以与Oracle ASM筛选器驱动程序一起使用。
      #cd /u01/app/19.0.0/grid/bin#./asmcmd afd_label DATA1 / dev / rdsk / cXtYdZsA --init＃。/ asmcmd afd_label DATA2 / dev / rdsk / cXtYdZsB --init＃。/ asmcmd afd_label DATA3 / dev / rdsk / cXtYdZsC --init
    3. 验证设备是否已标记为与Oracle ASMFD一起使用。
      ＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsA＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsB＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsC
  3. 以grid用户身份登录，并通过运行以下命令启动Oracle Grid Infrastructure安装程序：
    $ /u01/app/19.0.0/grid/gridSetup.sh
    安装程序启动，出现“选择配置选项”窗口。
  4. 选择“ 为新群集配置网格基础结构 ”选项，然后单击“ 下一步” 。
    将出现“选择群集配置”窗口。
  5. 选择“ 配置Oracle独立群集 ”选项，然后单击“ 下一步” 。
    选择“ 配置为扩展群集”选项可将Oracle RAC群集扩展到两个或多个单独的站点，每个站点都配有自己的存储。
    将出现“网格即插即用信息”窗口。
  6. 在“ 群集名称”和“ 扫描名称”字段中，输入在整个企业网络中唯一的群集和群集扫描的名称。

    如果已将域名服务器（DNS）配置为发送到子域GNS服务的GNS虚拟IP地址名称解析请求，则可以选择配置GNS ，如本指南中所述。

    对于集群成员节点公用和VIP网络地址，根据要配置的集群类型提供所需的信息：

    • 如果您计划使用通过GNS配置和解析的DHCP地址的自动群集配置，则您只需提供DNS上配置的GNS VIP名称。

    • 如果计划使用手动群集配置，并在DNS上配置并解析固定IP地址，则提供群集的SCAN名称，以及每个群集成员节点的公用名称和VIP名称。例如，您可以选择基于节点名称的公共前缀的名称。群集名称可以是mycluster ，群集SCAN名称可以是mycluster-scan 。

    单击下一步 。
    将出现“群集节点信息”窗口。
  7. 在群集节点表的“公共主机名”列中，您应该看到本地节点，例如node1.example.com 。

    以下是有关节点IP地址的其他信息的列表：

    • 仅对于本地节点，OUI会自动填写公共和VIP字段。如果您的系统使用供应商集群件，则OUI可能会填充其他字段。

    • 主机名和虚拟主机名不是域限定的。如果在安装期间在地址字段中提供域，则OUI将从该地址中删除该域。

    • 标识为专用于私有IP地址的接口不应作为公共接口访问。使用Cache Fusion的公共接口可能会导致性能问题。

    • 输入公共节点名称时，请使用每个节点的主要主机名。换句话说，使用/bin/hostname命令显示的名称。

    1. 单击“ 添加”以将另一个节点添加到群集。
    2. 输入第二个节点的公共名称（ node2 ）和虚拟IP名称（ node2-vip ），然后单击“ 确定” 。为所有群集节点提供虚拟IP（VIP）主机名，或者不提供。
      您将返回“群集节点信息”窗口。您现在应该看到群集节点表中列出的所有节点。
    3. 确保选中所有节点，然后单击窗口底部的SSH连接按钮。
      窗口的底部面板显示SSH连接信息。
    4. 输入Oracle软件所有者（ grid ）的操作系统用户名和密码。如果已在节点之间配置SSH连接，则选择“ 重用用户主目录中存在的私钥和公钥”选项。单击设置 。
      将出现一个消息窗口，指示可能需要几分钟才能在节点之间配置SSH连接。在短暂的一段时间后，将出现另一个消息窗口，指示已在群集节点之间建立无密码SSH连接。单击“ 确定”继续。
    5. 返回“群集节点信息”窗口时，单击“ 下一步”继续。
    将出现“指定网络接口使用”窗口。
  8. 选择显示的每个网络接口的使用类型。
    验证每个接口是否具有与之关联的正确接口类型。如果您具有Oracle Clusterware不应使用的网络接口，请将网络接口类型设置为“不使用” 。例如，如果您只有两个网络接口，则将公共接口设置为使用值为Public ，并将专用网络接口设置为使用ASM和私有值。
    单击下一步 。将出现“存储选项信息”窗口。
  9. 选择Oracle Cluster Registry（OCR）和投票文件的存储选项：
    1. 选择“ 使用Oracle Flex ASM存储”以在Oracle ASM磁盘组上存储OCR和表决文件。
    2. 选择“ 使用共享文件系统”将OCR和投票文件存储在共享文件系统上，然后单击“ 下一步” 。将出现“创建网格基础结构管理存储库选项”窗口
  10. 选择是否要为Oracle Standalone Cluster安装创建Grid Infrastructure Management存储库，然后单击“ 下一步” 。
    如果在此窗口中选择“ 是 ”，则会显示“网格基础结构管理存储库选项”窗口。否则，将显示“创建ASM磁盘组”窗口。
  11. 选择是否要将Grid Infrastructure Management存储库存储在单独的Oracle ASM磁盘组中，然后单击“ 下一步” 。
    将出现“创建ASM磁盘组”窗口。
  12. 提供Oracle ASM磁盘组的名称和规范。
    1. 在“ 磁盘组名称”字段中，输入磁盘组的名称 ，例如DATA。
    2. 选择此磁盘组的冗余级别。建议选择正常。
    3. 在“添加磁盘”部分中，选择要添加到此磁盘组的磁盘。

      在“添加磁盘”部分中，您应该看到在步骤2中标记的磁盘。如果没有看到磁盘，请单击“ 更改发现路径”按钮并提供磁盘的路径和模式匹配，例如/dev/sd*

      在安装期间，使用默认发现字符串时，标记为Oracle ASMFD磁盘或Oracle ASMLIB磁盘的磁盘将列为候选磁盘。但是，如果磁盘的标头状态为MEMBER，则它不是候选磁盘。

    4. 如果要使用Oracle ASM筛选器驱动程序（Oracle ASMFD）来管理Oracle ASM磁盘设备，请选择“ 配置Oracle ASM筛选器驱动程序 ”选项。
      如果要在Linux系统上进行安装，并且希望使用Oracle ASM筛选器驱动程序（Oracle ASMFD）来管理Oracle ASM磁盘设备，则必须在启动Oracle Grid Infrastructure安装之前卸载Oracle ASM库驱动程序（Oracle ASMLIB）。
    完成为磁盘组提供信息后，单击“ 下一步” 。
  13. 如果选择为GIMR使用其他磁盘组，则会显示“网格基础结构管理存储库选项”窗口。提供GIMR磁盘组的名称和规格。
    1. 在“ 磁盘组名称”字段中，输入磁盘组的名称 ，例如DATA。
    2. 选择此磁盘组的冗余级别。建议选择正常。
    3. 在“添加磁盘”部分中，选择要添加到此磁盘组的磁盘。
    完成为磁盘组提供信息后，单击“ 下一步” 。
    将出现“指定ASM密码”窗口。
  14. 为Oracle ASM SYS和ASMSNMP帐户选择相同的密码，或为每个帐户指定不同的密码，然后单击“ 下一步” 。
    出现“故障隔离支持”窗口。
  15. 选择“不使用智能平台管理界面（IPMI）”选项 ，然后单击“ 下一步” 。
    将出现“指定管理选项”窗口。
  16. 如果您的企业中安装了Enterprise Manager Cloud Control，请选择“ 注册企业管理器（EM）云控制 ”选项并提供EM配置信息。如果您的企业中未安装Enterprise Manager Cloud Control，请单击“ 下一步”继续。
    将显示“特权操作系统组”窗口。
  17. 接受Oracle ASM管理的默认操作系统组名称，然后单击“ 下一步” 。
    将出现“指定安装位置”窗口。
  18. 指定用于Oracle Grid Infrastructure安装的Oracle基础的目录，然后单击“ 下一步” 。Oracle基本目录必须与Oracle主目录不同。
    如果您按照步骤1中的说明将Oracle Grid Infrastructure安装文件复制到Oracle Grid主目录中，则Oracle基本目录的默认位置应显示为/u01/app/grid 。
    如果以前未在此计算机上安装Oracle软件，则会显示“创建库存”窗口。
  19. 如果需要，更改库存目录的路径。然后单击“ 下一步” 。
    如果您使用与本书中的示例相同的目录名称，则它应显示/u01/app/oraInventory 。oraInventory目录的组名应显示oinstall 。
    将出现“根脚本执行配置”窗口。
  20. 选择“ 自动运行配置脚本 ”选项。 输入root用户或sudo帐户的凭据，然后单击“ 下一步” 。
    或者，您可以在安装程序提示时在安装过程结束时以root用户身份手动运行脚本。
    将出现“执行先决条件检查”窗口。
  21. 如果任何检查的状态为“失败”且不可修复，则必须手动更正这些问题。解决问题后，可以单击“ 再次检查”按钮让安装程序重新检查要求并更新状态。根据需要重复，直到所有检查的状态为Succeeded。单击下一步 。

    将出现“摘要”窗口。

  22. 查看“摘要”窗口的内容，然后单击“ 安装” 。
    安装程序显示进度指示器，使您可以监视安装过程。
  23. 如果未配置根脚本的自动化，则需要以“执行配置脚本”窗口中指定的root用户身份运行某些脚本。直到运行的所有脚本不要单击确定 。按照指示在所有节点上按所示顺序运行脚本。

    例如，在Oracle Linux上执行以下步骤（请注意，为清楚起见，示例显示了提示中的当前用户，节点和目录）：

    1. 作为node1上的grid用户，打开终端窗口，然后输入以下命令：

      [grid @ node1 grid] $ cd / u01 / app / oraInventory [grid @ node1 oraInventory] $ su

    2. 输入root用户的密码，然后输入以下命令以在node1上运行第一个脚本：

      [root @ node1 oraInventory]＃。/ orainstRoot.sh

    3. orainstRoot.sh脚本在node1完成后，打开另一个终端窗口，并以grid用户身份输入以下命令：

      [grid @ node1 grid] $ ssh node2 [grid @ node2 grid] $ cd / u01 / app / oraInventory [grid @ node2 oraInventory] $ su

    4. 输入root用户的密码，然后输入以下命令以在node2上运行第一个脚本：

      [root @ node2 oraInventory]＃。/ orainstRoot.sh

    5. orainstRoot.sh脚本在node2完成后，转到您在此步骤的第a部分中打开的终端窗口。作为node1上的root用户，输入以下命令以运行第二个脚本root.sh ：

      [root @ node1 oraInventory] #cd /u01/app/19.0.0/grid [root @ node1 grid]＃。/ root.sh

      在提示符处按Enter键以接受默认值。

      注意：

      您必须在第一个节点上运行root.sh脚本并等待它完成。您可以在除运行脚本的最后一个节点之外的所有其他节点上同时运行root.sh脚本。与第一个节点一样，最后一个节点上的root.sh脚本必须单独运行。

    6. root.sh脚本在node1完成后，转到在此步骤的c部分中打开的终端窗口。作为node2上的root用户，输入以下命令：

      [root @ node2 oraInventory] #cd /u01/app/19.0.0/grid [root @ node2 grid]＃。/ root.sh

      root.sh脚本完成后，返回到Oracle Universal Installer窗口，安装程序在该窗口中提示您运行orainstRoot.sh和root.sh脚本。单击确定 。

      软件安装监视窗口重新出现。

  24. 继续监视安装，直到出现“完成”窗口。然后单击“ 关闭”以完成安装过程并退出安装程序。

警告：

安装完成后，请不要手动删除或运行在服务器上运行Oracle软件时删除/tmp/.oracle或/var/tmp/.oracle目录或其文件的cron作业。如果删除这些文件，则Oracle软件可能会遇到间歇性挂起。Oracle Clusterware安装可能会失败并显示错误：

CRS-0184: Cannot communicate with the CRS daemon 。

完成Oracle Grid Infrastructure安装后，可以在群集节点上安装Oracle数据库以获得高可用性，或者安装Oracle RAC。
