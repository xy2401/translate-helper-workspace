7.163 V $ DATAGUARD_CONFIG

V$DATAGUARD_CONFIG显示使用DB_UNIQUE_NAME和LOG_ARCHIVE_CONFIG初始化参数定义的唯一数据库名称，从配置中的任何数据库提供Oracle Data Guard环境的视图。

视图的第一行列出了使用DB_UNIQUE_NAME初始化参数指定的当前数据库的唯一数据库名称。其他行反映了配置中使用LOG_ARCHIVE_CONFIG初始化参数的DG_CONFIG关键字指定的其他数据库的唯一数据库名称。

柱 数据类型 描述

DB_UNIQUE_NAME

VARCHAR2（30）

唯一数据库名称

PARENT_DBUN

VARCHAR2（30）

父数据库的DB_UNIQUE_NAME ，也称为向目标提供实时重做的数据库。

例如，假设波士顿是DB_UNIQUE_NAME主数据库的，芝加哥是DB_UNIQUE_NAME远东同步实例的，和西雅图是DB_UNIQUE_NAME终端备用数据库。由于Boston是主数据库，因此它没有父数据库，因此Boston的PARENT_DBUN将为NULL 。自波士顿为芝加哥服务以来，芝加哥的PARENT_DBUN将成为波士顿。自芝加哥服务西雅图以来，芝加哥将成为西雅图的PARENT_DBUN 。

DEST_ROLE

VARCHAR2（17）

档案目的地数据库的类型：

  • LOCAL - 本地到主数据库

  • PHYSICAL - 物理待机

  • FAR SYNC INSTANCE -远东同步实例

  • CROSS-INSTANCE - 主要实例

  • LOGICAL - 逻辑备用

  • SNAPSHOT - 快照备用数据库

  • DOWNSTREAM - Streams下游捕获数据库

CURRENT_SCN

数

指定数据库已应用重做的SCN

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
