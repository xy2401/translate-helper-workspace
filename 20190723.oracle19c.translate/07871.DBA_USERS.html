PASSWORD_VERSIONS

VARCHAR2（12）

显示帐户现有的密码哈希值（也称为“验证程序”）的版本列表。

此列的值可包括：

  • 10G ：如果存在旧的不区分大小写的ORCL哈希

  • 11G ：如果存在SHA-1哈希

  • 12C ：如果存在基于去优化的基于PBKDF2的散列

  • HTTP ：如果存在MD5哈希（用于HTTP摘要式身份验证）

有关12C验证程序的更多信息，请参阅Oracle数据库概念 。

请注意，任何给定帐户都可以存在这些验证程序的任意组合。

EDITIONS_ENABLED

VARCHAR2（1）

指示是否已为相应用户（ Y ）启用了版本（ N ）

AUTHENTICATION_TYPE

VARCHAR2（8）

表示用户的认证机制：

  • NONE - 尚未为用户配置身份验证方法

  • EXTERNAL - CREATE USER user1 IDENTIFIED EXTERNALLY;

  • GLOBAL - CREATE USER user2 IDENTIFIED GLOBALLY;

  • PASSWORD - CREATE USER user3 IDENTIFIED BY user3 IDENTIFIED BY ;

PROXY_ONLY_CONNECT

VARCHAR2（1）

指示用户是否可以直接连接（ N ），或者该帐户是否只能由具有此帐户代理权限的用户（即，已被授予此帐户“connect through”权限的用户）代理（ Y ） 。

有关创建代理用户帐户和授权用户通过它们进行连接的详细信息，请参阅“ Oracle数据库安全指南” 。

共同

VARCHAR2（3）

指示给定用户是否通用。可能的值

  • 如果用户很常见，则为YES

  • NO如果用户是本地的（不常见）

上次登录

TIMESTAMP（9）带时区

ORACLE_MAINTAINED

VARCHAR2（1）

表示用户是由Oracle提供的脚本（例如catalog.sql还是catproc.sql）创建并维护的。除非运行Oracle提供的脚本，否则不得以任何方式更改此列具有值Y的用户。

遗传

VARCHAR2（3）

指示用户定义是否从其他容器继承（ YES ）（ NO ）

DEFAULT_COLLATION

VARCHAR2（100）

用户架构的默认排序规则

隐

VARCHAR2（3）

指示此用户是否是由隐式应用程序创建的公共用户（ YES ）或否（ NO ）

ALL_SHARD

VARCHAR2（3）

在分片数据库中，此列中的值指示是否已使用启用了分片DDL创建用户。可能的值是：

  • YES ：用户是在启用了分片DDL的情况下创建的。用户存在于所有分片和分片目录中。

  • NO ：创建的用户未启用分片DDL。用户仅存在于创建用户的数据库中。

在非分片数据库中，此列中的值始终为NO 。
