E.3.6 Oracle名称

Oracle数据库对象的名称（例如表和列）的长度不得超过30个字符。第一个字符必须是字母，但其余字母可以是字母，数字，美元符号（$），井号（＃）和下划线（_）的任意组合。

但是，如果名称用引号（“）括起来，则它可以包含任何合法字符组合，包括空格但不包括引号。

Oracle名称不区分大小写，除非用引号括起来。

关键词和参数 描述

使用说明

事务（或逻辑工作单元）是Oracle将其视为单个单元的一系列SQL语句。事务以COMMIT，ROLLBACK或与数据库的连接之后的第一个可执行SQL语句开始。事务以COMMIT结束声明，ROLLBACK 声明，或从数据库断开（有意或无意）。请注意，Oracle在处理任何数据定义语言语句之前和之后发出隐式COMMIT语句。

使用不带TO SAVEPOINT子句的ROLLBACK语句执行以下操作：

  • 结束交易

  • 撤消当前事务中的所有更改

  • 擦除事务中的所有保存点

  • 释放事务的锁

使用带有TO SAVEPOINT子句的ROLLBACK语句执行以下操作：

  • 在保存点之后回滚事务的一部分。

  • 丢失在该保存点之后创建的所有保存点。请注意，保留了命名的保存点，因此您可以多次回滚到同一个保存点 。之前的保存点也会保留。

  • 释放自保存点以来获取的所有表锁和行锁。请注意，已请求访问保存点之后锁定的行的其他事务必须继续等待，直到提交或回滚事务为止。尚未请求行的其他事务可以立即请求和访问行。

建议您使用COMMIT或ROLLBACK语句显式结束应用程序中的事务。如果您没有显式提交事务并且程序异常终止，则Oracle将回滚最后一个未提交的事务。

例I

以下语句回滚整个当前事务：

EXEC SQL ROLLBACK END-EXEC。

例II

以下语句将当前事务回滚到保存点SP5：

EXEC SQL ROLLBACK到SAVEPOINT SP5 END-EXEC。

分布式事务

具有分布式选项的Oracle使您可以执行分布式事务或修改多个数据库上的数据的事务。要提交或回滚分布式事务，您只需像发出任何其他事务一样发出COMMIT或ROLLBACK语句。

如果在分布式事务的提交过程中存在网络故障，则事务的状态可能是未知的或不确定的。在咨询事务中涉及的其他数据库的管理员之后，您可以决定在本地数据库上手动提交或回滚事务。您可以通过发出带有FORCE子句的ROLLBACK语句来手动回滚本地数据库上的事务。

您无法手动将不确定事务回滚到保存点。

带有FORCE子句的ROLLBACK语句仅回滚指定的事务。这样的陈述不会影响您当前的交易。

例III

以下语句手动回滚一个不确定的分布式事务：

EXEC SQL ROLLBACK WORK FORCE '25 .32.87'END-EXEC。
