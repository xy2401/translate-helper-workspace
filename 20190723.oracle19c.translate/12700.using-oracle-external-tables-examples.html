示例18-1使用ORACLE_LOADER创建分区外部表

此示例假定有四个数据文件包含以下内容：

p1a.dat：1，AAAAA Plumbing，01372,28，Sparkly Laundry，78907,13，Andi's Donuts，54570，p1b.dat：51，DIY Supplies，61614,87，Fast Frames，22201,89，Friendly Pharmacy，89901， p2.dat：121，Pleasant Pets，33893,130，Bailey the Bookmonger，99915,105，Le Bistrot du Chat Noir，94114，p3.dat：210，The Electric Eel Diner，07101,222，Everyt'ing General Store， 80118,231，Big Rocket Market，01754，

数据文件中有三个字段： CUSTOMER_NUMBER ， CUSTOMER_NAME和POSTAL_CODE 。外部表使用CUSTOMER_NUMBER上的范围分区来创建三个分区。

  • 分区1适用于小于100的customer_number

  • 分区2适用于小于200的customer_number

  • 分区3适用于小于300的customer_number

请注意，第一个分区有两个数据文件，而其他分区只有一个。以下是SQLPlus用于创建文件的输出。

18.2使用ORACLE_LOADER访问驱动程序创建分区混合表

本主题介绍如何使用ORACLE_LOADER访问驱动程序创建分区混合表。

混合分区表是一种通过允许某些分区驻留在数据库段中以及外部文件或源中的某些分区来扩展Oracle分区的功能。这显着增强了大数据SQL的分区功能，其中表的大部分可以驻留在外部分区中。

例18-2例子

以下是创建分区混合l表的语句示例：

CREATE TABLE hybrid_pt（time_id日期，客户编号）TABLESPACE TS1外部分区属性（类型ORACLE_LOADER默认目录data_dir0访问参数（字段终止于'，'）拒绝限制无限制分区范围（time_id）（PARTITION century_18 VALUES少于（'01） -01-1800'）EXTERNAL，< - 空外部分区PARTITION century_19值少于（'01 -01-1900'）外部默认目录data_dir1位置（'century19_data.txt'），PARTITION century_20 VALUES少于（'01  - 01-2000'）外部位置（'century20_data.txt'），PARTITION year_2000值小于（'01 -01-2001'）TABLESPACE TS2，PARTITION pmax VALUE小于（MAXVALUE））;

在此示例中，该表包含内部和外部分区。表中内部分区的默认表空间是TS1 。添加了一个EXTERNAL PARTITION ATTRIBUTES子句，用于指定在表级别应用于表中外部分区的参数。该子句对于混合分区表是必需的。在这种情况下，通过ORACLE_LOADER访问驱动程序访问外部分区，并在子句中指定访问驱动程序所需的参数。在分区级别，在每个外部分区中指定EXTERNAL子句，以及应用于该分区的任何外部参数。

在此示例中， century_18 ， century_19和century_20是外部分区。 century_18是一个空分区，因为它不包含位置。分区century_19的缺省目录是data_dir1 ，它覆盖了表级缺省目录。该分区的位置为data_dir1:century19_data.txt 。分区century_20具有位置data_dir0:century20_data.txt ，因为在分区中未指定缺省目录时，表级缺省目录将应用于某个位置。分区year_2000和pmax是内部分区。分区year_2000具有tablespace TS2 。当分区没有指定EXTERNAL子句或外部参数时，默认情况下它被假定为内部分区。

18.3使用ORACLE_DATAPUMP访问驱动程序创建分区外部表

本节中的示例显示了如何创建子分区外部表。

它还显示了如何使用虚拟列对表进行分区。

示例18-3使用ORACLE_DATAPUMP访问驱动程序创建分区外部表

在此示例中，使用的转储文件与使用ORACLE_LOADER访问驱动程序在上一个示例中创建的转储文件相同。但是，在此示例中，除了使用customer_number对数据进行分区之外，还使用postal_code对数据进行子分区。对于每个分区，都有一个子分区，其中postal_code小于50000，另一个子分区用于postal_code所有其他值。使用三个分区，每个分区包含两个子分区，总共需要六个文件。要创建文件，请使用SQL CREATE TABLE AS SELECT语句为分区选择正确的行，然后将这些行写入ORACLE_DATAPUMP驱动程序的文件中。

以下语句创建一个文件，其中包含分区p1 （ customer_number小于100）的第一个子分区（ postal_code小于50000）的数据。 

SQL> create table customer_list_dp_p1_sp1_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p1_sp1.dmp'））4作为5从customer_list_xt分区中选择customer_number，customer_name，postal_code 6（p1）7其中to_number（postal_code）<50000;表创建。SQL>

此语句创建一个文件，其中包含分区p1 （customer_number小于100）的第二个子分区（postal_code的所有其他值）的数据。 

SQL> create table customer_list_dp_p1_sp2_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p1_sp2.dmp'））4作为5从customer_list_xt分区中选择customer_number，customer_name，postal_code 6（p1）7其中to_number（postal_code）> = 50000;表创建。

其他分区的文件以类似的方式创建，如下所示：

SQL> create table customer_list_dp_p2_sp1_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p2_sp1.dmp'））4作为5从customer_list_xt分区中选择customer_number，customer_name，postal_code 6（p2）7其中to_number（postal_code）<50000;表创建。SQL> SQL> create table customer_list_dp_p2_sp2_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p2_sp2.dmp'））4 as 5 select customer_number，customer_name，postal_code 6 from customer_list_xt partition（p2）7 where to_number（postal_code）> = 50000;表创建。SQL> SQL> create table customer_list_dp_p3_sp1_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p3_sp1.dmp'））4作为5从customer_list_xt分区中选择customer_number，customer_name，postal_code 6（p3）7其中to_number（postal_code）<50000 ;表创建。SQL> SQL> create table customer_list_dp_p3_sp2_xt 2 organization external 3（类型oracle_datapump默认目录def_dir1 location（'p3_sp2.dmp'））4作为5从customer_list_xt分区中选择customer_number，customer_name，postal_code 6（p3）7其中to_number（postal_code）> = 50000;表创建。SQL>

您可以从每个外部表中进行选择，以验证它是否包含您要写出的数据。执行SQL CREATE TABLE AS SELECT语句后，可以删除这些外部表。

要使用虚拟列对表进行分区，请创建分区的ORACLE_DATAPUMP表。同样，该表在customer_number列上进行了分区，并在postal_code列上进行了分区。postal_code列是包含数字的字符字段，但此示例根据数值而不是字符串对其进行分区。为此，请创建一个虚拟列postal_code_num,其值为postal_code字段，转换为NUMBER数据类型。SUBPARTITION子句使用虚拟列来确定行的子分区。 

SQL> create table customer_list_dp_xt 2（customer_number number，3 CUSTOMER_NAME VARCHAR2（50），4 postal_code CHAR（5），5 postal_code_NUM as（to_number（postal_code）））6组织外部7（类型oracle_datapump默认目录def_dir1）8按范围划分（ customer_number）9按范围划分的子分区（postal_code_NUM）10（11个分区p1值小于（100）12（子分区p1_sp1值小于（50000）位置（'p1_sp1.dmp'），13个子分区p1_sp2值小于（MAXVALUE）位置（ 'p1_sp2.dmp'）），14分区p2值小于（200）15（子分区p2_sp1值小于（50000）位置（'p2_sp1.dmp'），16分区p2_sp2值小于（MAXVALUE）位置（'p2_sp2。 dmp'）），17分区p3值小于（300）18（子分区p3_sp1值小于（50000）位置（'p3_sp1.dmp'），19分区p3_sp2值小于（MAXVALUE）位置（'p3_sp2.dmp'） ）20）;表创建。SQL>

如果选择所有行，则返回的数据与使用ORACLE_LOADER访问驱动程序在上一个示例中返回的数据相同。 

SQL> select customer_number，customer_name，postal_code 2 from customer_list_dp_xt 3 order by customer_number; customer_number CUSTOMER_NAME POSTA --------------- -------------------------------- ------------------ ----- 1 AAAAA Plumbing 01372 13 Andi's Donuts 54570 28 Sparkly Laundry 78907 51 DIY Supplies 61614 87 Fast Frames 22201 89 Friendly Pharmacy 89901 105 Le Bistrot du Chat Noir 94114 121 Pleasant Pets 33893 130 Bailey the Bookmonger 99915 210 The Electric Eel Diner 07101 222 Everyt'ing General Store 80118 231 Big Rocket Market 01754 12行选择。SQL>

WHERE子句可以将读取的行限制为子分区。以下查询应该只读取第一个分区的第一个子分区。 

SQL> select customer_number，customer_name，postal_code 2 from customer_list_dp_xt 3其中customer_number <20，postal_code_NUM <39998 4 by customer_number; customer_number CUSTOMER_NAME POSTA --------------- -------------------------------- ------------------ ----- 1 AAAAA Plumbing 01372 1行选中。SQL>

SQL> select customer_number，customer_name，postal_code 2 from customer_list_dp_xt subpartition（p2_sp2）order by customer_number; customer_number CUSTOMER_NAME POSTA --------------- -------------------------------- ------------------ ----- 105 Le Bistrot du Chat Noir 94114 130 Bailey the Bookmonger 99915 2行选中。SQL>

18.5使用ORACLE_HIVE访问驱动程序创建分区外部表

要为ORACLE_HIVE表创建分区外部表，需要一个分区的Hive外部表。

然后，您需要使用PL / SQL过程DBMS_HADOOP.CREATE_EXTDDL_FOR_HIVE() 。此外，此示例从上一示例中使用的HDFS文件创建分区Hive表。在可以创建分区的Hive表之前，必须创建一个用于存储数据的HDFS目录。

示例18-5使用ORACLE_HIVE访问驱动程序创建分区外部表

这个示例使用前一个示例中的数据文件创建Hive表，显示如何使用ORACLE_HDFS创建分区外部表。以下命令均在Hive CLI内执行，因此它们使用Hive语法。

首先，使用Hive在HDFS数据文件之上创建Hive外部表，如下所示：

创建外部表customer_list_no_part（customer_number int，customer_name string，postal_code string）行格式分隔的字段以'，'结尾，存储为文本文件位置'/ user / doc / hdfs_pet'

然后执行以下命令告诉Hive动态创建分区：

set hive.exec.dynamic.partition = true set hive.exec.dynamic.partition.mode = nonstrict

创建分区的Hive表：

创建表customer_list（CUSTOMER_NUMBER int，CUSTOMER_NAME字符串）分区为（postal_code字符串）行格式分隔的字段以'\ t'位置'/ user / doc / doc_pet'终止

使用表customer_list_no_part数据填充配置单元表。这应该为每个具有正确行的分区创建文件。 

insert overwrite table customer_list partition（postal_code）select * from customer_list_no_part

Hive customer_list表填充了行。如下面的查询所示：

select * from customer_list order by customer_number 1 AAAAA Plumbing 01372 13 Andi's Donuts 01372 28 Sparkly Laundry 07101 51 DIY Supplies 07101 87 Fast Frames 01754 89 Friendly Pharmacy 01372 105 Le Bistrot du Chat Noir 01754 121 Pleasant Pets 01754 130 Bailey the Bookmonger 01754 210 The Electric Eel Diner 07101 222 Everyt'ing General Store 01372 231 Big Rocket Market 01754

现在，您可以返回SQL * Plus以在Oracle数据库中创建分区外部表。首先，使用PL / SQL函数DBMS_HADOOP.CREATE_EXTDDL_FOR_HIVE()来创建外部表。论点如下：

  1. Hadoop集群的名称

  2. 拥有该表的Hive用户的名称

  3. 分区Hive表的名称

  4. 一个布尔值，指示是否要将Hive表中的分区列作为列包含在外部表中

  5. 创建的分区ORACLE_HIVE表的名称

  6. 一个布尔值，指示是否执行CREATE DDL

  7. CLOB包含CREATE DDL字符串

此示例获取CREATE DDL字符串的CLOB ，并使用DBMS_OUTPUT.PUT_LINE()过程编写它。设置SERVEROUTPUT ON告诉SQL * Plus显示DBMS_OUTPUT.PUT_LINE()过程中的数据。将LINESIZE设置为132可确保在显示DBMS_OUTPUT.PUT_LINE()过程中的数据时，奇数位置没有换行符。 

SQL> SET LINESIZE 132 SQL> SET SERVEROUTPUT ON SQL> SQL> DECLARE 2 DDLtxt clob; 3 BEGIN 4 dbms_hadoop.create_extddl_for_hive 5（'hadoop_cl_1'，'default'，'customer_list'，6 TRUE，'CUSTOMER_LIST_HIVE'，TRUE，DDLtxt）; 7 dbms_output.put_line（'DDL Text is：'|| DDLtxt）; 8结束; 9 /外部表已成功创建。DDL文本是：CREATE TABLE“DOC”。“CUSTOMER_LIST_HIVE”（customer_number NUMBER，customer_name VARCHAR2（4000），postal_code VARCHAR2（4000））ORGANIZATION EXTERNAL（类型ORACLE_HIVE DEFAULT DIRECTORY DEFAULT_DIR访问参数（com.oracle.bigdata.cluster = hadoop_cl_1 com） .oracle.bigdata.tablename = default.customer_list））REJECT LIMIT UNLIMITED PARTITION BY LIST（postal_code）（PARTITION“P_293620257”VALUES（'01372'），PARTITION“P_292175793”VALUES（'01754'），PARTITION“P_717839126”VALUES（ '07101'））

由于Hive未指定STRING列的最大字符数，因此外部表的列定义为VARCHAR2(4000) 。如果您希望某些列的长度较小，则可以调用DBMS_HADOOP.CREATE_EXTDDL_FOR_HIVE()过程并指定您不想执行CREATE DDL。然后，您可以编辑CLOB返回的CREATE语句，以将VARCHAR2列的长度设置为更合适的值。

另请注意，分区名称中的数字可能会有所不同。

现在创建了表，执行SELECT *语句将返回所有行。请注意，上面执行的SET LINESIZE意味着SQL * Plus对customer_name和postal_code使用132个字符。 

SQL> select * from customer_list_hive order by customer_number; CUSTOMER_NUMBER --------------- CUSTOMER_NAME --------------------------------- -------------------------------------------------- -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  - -邮政编码  -  -  - -------------------------------------------------- -------------------------------------------------- -------------------- 1 AAAAA Plumbing 01372 13 Andi's Donuts 01372 28 Sparkly Laundry 07101 51 DIY Supplies 07101 87 Fast Frames 01754 89 Friendly Pharmacy 01372 105 Le Bistrot du Chat Noir 01754 121喜羊羊宠物01754 130 Bailey the Bookmonger 01754 210 The Electric Eel Diner 07101 222 Everyt'ing General Store 01372 231 Big Rocket Market 01754 12行选择。SQL>

18.6从外部表加载LOB

外部表对于从单个文件加载大量记录特别有用，因此每个记录都显示在表中自己的行中。

以下示例显示了如何执行此类加载。

示例18-6从外部表加载LOB

假设您定义了一个外部表my_ext_table ，如下所示：

CREATE TABLE my_ext_table（id NUMBER，作者VARCHAR2（30），创建日期，文本CLOB）ORGANIZATION EXTERNAL（类型ORACLE_LOADER默认目录MY_DIRECTORY访问参数（由0x'0A'字段删除的记录'，'选项包含''''）（ id CHAR（10），作者CHAR（30），创建日期“YYYY-MM-DD”，文本CHAR（131071）））LOCATION（MY_DIRECTORY：'external.dmp'））;

external.dmp文件的内容如下：

1，Roger，2015-08-08，快速的棕色狐狸跳过懒狗2，John，2012-01-01，“愤怒的aligator，独自行动，吃了羚羊”

转储文件中的第二行需要在整个文本字符串周围加引号;否则该字段将以逗号终止。

注意：

虽然在此示例中使用的转储文件中没有问题，但如果全文字符串中的某些内容包含引号，那么您可以将其括在另一组引号中，如下alone的单词：
2，John，2012-01-01，“愤怒的aligator，表演”“独自”，“吃羚羊”

如果全文可能包含记录分隔符（0x'0A'或换行符），则可以为每个文档指定单独的文件。外部表不支持填充字段，因此您必须使用COLUMN TRANSFORMS子句指定fname字段包含外部文件的名称：

DROP TABLE my_ext_table2; CREATE TABLE my_ext_table2（id NUMBER，作者VARCHAR2（30），创建DATE，文本CLOB）ORGANIZATION EXTERNAL（类型ORACLE_LOADER默认目录MY_DIRECTORY访问参数（由0x'0A'字段删除的记录'，'选项包含''''）（ id CHAR（10），作者CHAR（30），创建DATE“YYYY-MM-DD”，fname char（100））COLUMN TRANSFORMS（文本FROM LOBFILE（fname）FROM（MY_DIRECTORY）））LOCATION（'loader.txt' ））;

注意：

FROM (MY_DIRECTORY)子句实际上不是必需的，因为它已被指定为默认目录。但是，这里显示的是出于示例目的，因为如果loader.txt文件和各个CLOB文件位于不同的位置，则需要它。

一旦数据在外部表中，您可以将其保留在那里并在外部表上执行常规表操作（DML和大多数DDL），或者您可以使用外部表作为临时表将数据放入普通表中。要创建新的普通（非外部）表，可以使用以下SQL语句：

CREATE TABLE normaltable AS SELECT * FROM externaltable ;

您可以类似地使用以下SQL语句将数据插入新的普通表：

INSERT INTO normaltable AS SELECT * FROM externaltable ;
