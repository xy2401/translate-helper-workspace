如何显示Oracle数据字典的当前状态

使用以下三种方法之一检查Oracle数据字典的状态，以诊断升级和迁移。

运行dbupgdiag.sql脚本

dbupgdiag.sql脚本收集有关数据字典当前状态的升级和迁移诊断信息。

您可以在源数据库上升级之前以及在作为SYS用户升级的数据库上升级之后，在SQL * Plus中运行该脚本。有关使用dbupgdiag.sql脚本收集升级和迁移诊断信息的更多信息，请参阅My Oracle Support说明556610.1。

在DBA_REGISTERY上运行SQL查询

要显示字典的当前状态，请执行类似于以下示例的SQL查询：

SQL> spool /tmp/regInvalid.out SQL> set echo on  -  query registry SQL> set lines 80 pages 100 SQL> select substr（comp_id，1,15）comp_id，substr（comp_name，1,30）comp_name，substr（版本，1,10）版本，来自dba_registry的状态由修改后的顺序;

运行查询以检查无效对象

要查询无效对象，请执行类似于以下示例的SQL查询：

SQL> select owner，object_name，object_type from dba_invalid_objects order by owner，object_type;

升级数据库并运行utlrp.sql ，此视图查询应不返回任何行。
