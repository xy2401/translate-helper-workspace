1.315 SHARED_SERVERS

SHARED_SERVERS指定启动实例时要创建的服务器进程数。 如果系统负载减少，则维持此最小数量的服务器。因此，您应该注意不要在系统启动时将SHARED_SERVERS设置SHARED_SERVERS太高。
属性 描述

参数类型

整数

默认值

0 ，表示共享服务器未启用。

如果您使用的是共享服务器体系结构，或者如果设置DISPATCHERS参数使得调度程序的总数大于0，则默认值为1 。

可修改

改变系统

可在PDB中修改

是

价值范围

此参数的值应小于MAX_SHARED_SERVERS 。如果它大于或等于MAX_SHARED_SERVERS ，那么服务器的数量将不会自我调整，但将保持不变，如SHARED_SERVERS所指定。

基本

是

从Oracle Database 12 c第1版（12.1.0.2）开始，可以在PDB上设置SHARED_SERVERS参数。但是，与可以在PDB上设置的大多数其他参数不同，此参数只能在PDB中使用，以启用或禁用对该PDB的共享服务器的使用。因此，在PDB中，DBA可以将SHARED_SERVERS设置为0以禁用该PDB的共享服务器，或使用ALTER SYSTEM RESET SHARED_SERVERS为PDB重新启用共享服务器。 CDB的共享服务器配置只能在根目录中完成。
