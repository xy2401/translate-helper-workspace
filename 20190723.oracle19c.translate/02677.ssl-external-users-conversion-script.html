B SSL外部用户转换脚本

升级到Oracle Database 12 c第1版（12.1）及更高版本后，如果在Oracle Database 10g第2版（10.2）版之前使用经SSL验证的外部用户，则应运行SSL外部用户转换脚本。该脚本将Oracle数据库10g第2版（10.2）版本中经过SSL身份验证的外部用户转换为Oracle Database 12 c第1版（12.1）及更高版本中经过SSL身份验证的外部用户。

注意：

只有从Oracle Database 10g第2版（10.2）版本升级后，才需要运行SSL外部用户转换脚本。

关于使用安全的外部密码存储区

如果要使用安全的外部密码存储，请按照以下信息中的说明配置Oracle钱包;否则，可以交互式提供密码，您可以跳过此部分。

在运行extusrupgrade脚本之前， extusrupgrade客户端Oracle钱包配置为安全外部密码存储，以便应用程序可以使用存储在钱包中的密码凭据连接到数据库。在客户端Oracle钱包中存储数据库密码凭证，无需在应用程序代码，批处理作业或脚本中嵌入密码。这样可以降低在脚本和应用程序代码中以明文形式显示密码的风险，并允许您更轻松地管理用户帐户的密码策略，而无需在密码更改时更改应用程序代码或脚本。

有关使用mkstore命令行实用程序将客户端配置为使用外部密码存储库的步骤，请参阅配置客户端以使用外部密码存储库。

注意：

钱包的外部密码存储区与存储公钥基础结构（PKI）凭据的区域是分开的。因此，您无法使用Oracle Wallet Manager来管理钱包的外部密码存储中的凭据。而是使用命令行实用程序mkstore来管理这些凭据。

使用mkstore CreateCredential命令，通过提供<alias, username, password >信息来配置以下dbuser凭证，在该提示中将提示您输入用户的密码：

  • dbalias ， dbuser ， password

配置此用户凭据允许您在extusrupgrade脚本命令行上使用以下参数：

  • -dbalias = <DB-密码别名>

转换脚本示例使用为别名，用户名和密码提供的用户dbuser的以下钱包凭据信息。钱包位置如图所示指定。

  • dbmanager1 ， system ， password

  • WALLET_LOCATION = / ORACLE /产品/ 19.1.0 / DB_1 /钱包

转换脚本示例在命令行上使用以下条目：

  • -dbalias = dbmanager1

  • WALLET_LOCATION = / ORACLE /产品/ 19.1.0 / DB_1 /钱包

配置客户端钱包后，启用Oracle Wallets的自动登录，以允许运行extusrupgrade脚本的管理员访问并执行extusrupgrade服务，而无需提供必要的凭据。

也可以看看：

  • 管理密码凭据的安全外部密码存储，以获取有关创建客户端密码存储钱包以存储用户的别名，用户名和密码凭据的详细信息

  • 关于为Oracle Wallets使用自动登录的信息，以获取有关启用Oracle钱包自动登录的信息，这些信息支持基于PKI的服务访问，无需人工干预即可提供运行extusrupgrade脚本所需的必要用户名密码

使用SSL外部用户转换脚本

SSL外部用户转换脚本具有以下语法：

$ ORACLE_HOME / rdbms / bin / extusrupgrade --dbconnectstring 数据库连接字符串 --dbuser 数据库用户 [-dbalias 数据库用户密码别名 ] [-wallet_location 钱包位置 ] [-a] [-l username1，username2，... ][-f filename ] [-o] [-h]注意：-a升级所有合格用户-l由comma -f升级的用户升级列表由文件指定的用户升级列表。每行一个用户名-o将所有合格用户输出到标准输出。不与其他选项结合使用-h显示此帮助。

database connect string的格式应为hostname ： port_no ： sid ，其中hostname是运行数据库的主机的名称， port_no是侦听器端口号， sid是数据库实例的系统标识符。

如果使用mkstore命令行实用程序创建了安全外部密码存储库，则使用语法<alias, username, password>使用mkstore CreateCredential命令在钱包中创建dbuser凭据。例如， dbmanager1, system, password 。

接下来，启用Oracle钱包的自动登录。这允许运行extusrupgrade脚本的管理员用户访问extusrupgrade服务，而无需提供必要的凭据。

现在，您可以在命令行上使用数据库别名参数-dbalias <database user password alias>和钱包位置参数-wallet_location <wallet location>来运行extusrupgrade转换脚本。

以下示例假定钱包具有使用语法<alias, username, password>定义的dbuser凭证<alias, username, password>如dbmanager1, system, password 。例如，钱包位置显示为/oracle/product/19.1.0/db_1/wallets 。

使用-a选项转换所有经过SSL身份验证的外部用户。这是一个例子：

extusrupgrade --dbconnectstring mymachine：1521：ORA001 --dbuser system -dbialis dbmanager1 -wallet_location /oracle/product/19.1.0/db_1/wallets -a

使用-l选项指定要转换的逗号分隔的用户列表。例如：

extusrupgrade --dbconnectstring mymachine：1521：ORA001 --dbuser system -dbialis dbmanager1 -wallet_location /oracle/product/19.1.0/db_1/wallets -l user1，user2，user3

使用-f选项指定包含要转换的用户列表的文件。例如：

extusrupgrade --dbconnectstring mymachine：1521：ORA001 --dbuser system -dbialis dbmanager1 -wallet_location /oracle/product/19.1.0/db_1/wallets -f usernames.txt

指定文件中的每一行应该有一个用户名。这是一个示例usernames.txt文件：

用户＃1用户> 2个用户，3个用户4用户5

必须使用-f选项转换用户名中包含特殊字符（例如＃）的用户。

注意：

您可以在同一命令中组合-l和-f选项。该脚本结合了-l和-f选项的用户列表。如果使用-a选项以及-l option和-f选项，则忽略-a选项。

您可以使用-o选项将SSL身份验证的外部用户列表打印到标准输出设备。输出列出了可以使用extusrupgrade脚本转换的用户。-o选项不能与任何其他选项组合使用。 

extusrupgrade --dbconnectstring mymachine：1521：ORA001 --dbuser system -dbialis dbmanager1 -wallet_location /oracle/product/19.1.0/db_1/wallets -o

此示例输出可能是：

user1 user2 user3

小费：

您可以将命令输出重定向到文件以获取可以转换的用户列表。然后，您可以编辑该文件并将其与-f选项一起使用。

将全球用户转换为外部用户

Oracle数据库10g及更高版本允许经过SSL身份验证的外部用户和经过SSL身份验证的全局用户在数据库中共存。以前的版本限制所有SSL用户必须是全局用户或外部用户，具体取决于是否使用Oracle Internet Directory来验证用户。

如果您希望用户即使在Oracle Internet Directory不可用时也能够连接到数据库，则应将用户配置为外部用户。您可以使用SSL外部用户转换脚本将经过SSL身份验证的全局用户转换为经过SSL身份验证的外部用户。

如果使用mkstore命令行实用程序创建了安全外部密码存储区，并使用语法<alias, username, password>使用mkstore CreateCredential命令在钱包中创建了dbuser凭据。例如， dbmanager1, system, password 。例如，钱包位置显示为/oracle/product/19.1.0/db_1/wallets 。现在，您可以在运行extusrupgrade转换脚本时在命令行上使用数据库别名参数-dbalias <database user password alias>和钱包位置参数-wallet_location <wallet location> 。请注意，如果您已启用Oracle钱包的自动登录，则运行extusrupgrade脚本的管理员用户可以访问extusrupgrade服务，而无需提供必要的凭据。

例如：

extusrupgrade --dbconnectstring mymachine：1521：ORA001 --dbuser system -dbialis dbmanager1 -wallet_location /oracle/product/19.1.0/db_1/wallets -l user1，user2

上面的示例将两个全局用户转换为外部用户。
