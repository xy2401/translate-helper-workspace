6.1本地命名参数概述

tnsnames.ora文件是一个配置文件，其中包含映射到本地命名方法的连接描述符的网络服务名称 ，或映射到侦听器协议地址的网络服务名称。

网络服务名称是映射到a中包含的数据库网络地址的别名连接描述符。连接描述符包含通过协议地址的侦听器的位置以及要连接的数据库的服务名称。客户端和数据库服务器（即其他数据库服务器的客户端）在与应用程序建立连接时使用网络服务名称。

默认情况下， tnsnames.ora文件位于ORACLE_HOME/network/admin目录中。Oracle Net将检查配置文件的其他目录。例如，检查tnsnames.ora文件的顺序如下：

  1. 由TNS_ADMIN环境变量指定的目录。如果在指定的目录中找不到该文件，则假定该文件不存在。

  2. 如果未设置TNS_ADMIN环境变量，则Oracle Net将检查ORACLE_HOME/network/admin目录。

注意：

在Microsoft Windows上，如果在进程的环境中设置了TNS_ADMIN环境变量，则使用该变量。如果未在环境中定义TNS_ADMIN环境变量，或者该进程是没有环境的服务，则Microsoft Windows会在注册表中扫描TNS_ADMIN参数。

6.4 tnsnames.ora中的多个地址列表

tnsnames.ora文件还支持具有多个地址列表的连接描述符，每个地址列表都有自己的特征。在例6-3中 ，给出了两个地址列表。第一个地址列表具有客户端负载平衡和无连接时故障转移功能 ，仅影响其ADDRESS_LIST协议地址。第二个协议地址列表没有客户端负载加载平衡，但具有连接时故障转移，仅影响其ADDRESS_LIST那些协议地址。客户端首先随机尝试第一个或第二个协议地址，然后按顺序尝试协议地址3和4。

示例6-3 tnsnames.ora中的多个地址列表

注意：

  • Oracle Net Manager仅支持为连接描述符创建一个协议地址列表。

6.5使用Oracle连接管理器进行连接时故障转移和客户端负载平衡

当tnsnames.ora文件中的连接描述符包含至少两个Oracle Connection Manager的协议地址时，连接时故障转移和负载平衡的参数可以包含在文件中。

示例6-4 tnsnames.ora中的多个Oracle Connection Manager地址

此示例说明了多个Oracle Connection Manager协议地址的故障转移。

sample1 =（DESCRIPTION =（SOURCE_ROUTE = yes）（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST = host1）（PORT = 1630））＃1（ADDRESS_LIST =（FAILOVER = on）（LOAD_BALANCE = off）＃2（ ADDRESS =（PROTOCOL = tcp）（HOST = host2a）（PORT = 1630））（ADDRESS =（PROTOCOL = tcp）（HOST = host2b）（PORT = 1630）））（ADDRESS =（PROTOCOL = tcp）（HOST = host3） ）（PORT = 1521）））＃3（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））

这里，语法执行以下操作：

  1. 指示客户端连接到第一个Oracle Connection Manager的协议地址，如下所示：

    （ADDRESS =（PROTOCOL = TCP）（HOST =主机1）（PORT = 1630））

  2. 指示第一个Oracle Connection Manager连接到另一个Oracle Connection Manager的第一个协议地址。如果第一个协议地址失败，则它尝试第二个协议地址。使用以下配置指定此序列：

    （ADDRESS_LIST =（FAILOVER = on）（LOAD_BALANCE = off）（ADDRESS =（PROTOCOL = tcp）（HOST = host2a）（PORT = 1630））（ADDRESS =（PROTOCOL = tcp）（HOST = host2b）（PORT = 1630） ））

  3. Oracle Connection Manager使用以下协议地址连接到数据库服务：

    （ADDRESS =（PROTOCOL = TCP）（HOST = host3上）（PORT = 1521））

示例6-5 tnsnames.ora中的客户端负载平衡

此示例说明了两个Oracle Connection Manager和两个协议地址之间的客户端负载平衡：

sample2 =（DESCRIPTION =（LOAD_BALANCE = on）＃1（FAILOVER = on）（ADDRESS_LIST =（SOURCE_ROUTE = yes）（ADDRESS =（PROTOCOL = tcp）（HOST = host1）（PORT = 1630））＃2（ADDRESS =（ PROTOCOL = tcp）（HOST = host2）（PORT = 1521）））（ADDRESS_LIST =（SOURCE_ROUTE = yes）（ADDRESS =（PROTOCOL = tcp）（HOST = host3）（port = 1630））（ADDRESS =（PROTOCOL = tcp） ）（HOST = host4）（port = 1521）））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））＃3

这里，语法执行以下操作：

  1. 如果选择的ADDRESS_LIST失败，则指示客户端随机选择ADDRESS_LIST并故障转移到另一个。这由LOAD_BALANCE和FAILOVER参数设置为on 。

  2. 选择ADDRESS_LIST ，客户端首先使用Oracle Connection Manager协议地址连接到Oracle Connection Manager，该地址使用为ADDRESS_LIST指示的端口1630。

  3. 然后，Oracle Connection Manager使用为ADDRESS_LIST指示的协议地址连接到数据库服务。

6.7议定书地址科

tnsnames.ora文件的协议地址部分指定侦听器的协议地址。如果只有一个侦听器协议地址，则使用ADDRESS参数。如果有多个地址，则使用ADDRESS_LIST参数。

    ADDRESS网络参数位于tnsnames.ora文件中，它指定ADDRESS_LIST下的协议地址或一个侦听器的DESCRIPTION参数。

  • HTTPS_PROXY_PORT
  • ADDRESS_LIST

6.7.2 HTTPS_PROXY

目的

为隧道SSL客户端连接指定HTTP代理主机名。

使用说明

客户端可以使用HTTP CONNECT方法通过转发HTTP代理隧道安全连接。这有助于访问公共云数据库服务，因为它消除了在客户端防火墙上打开出站端口的要求。此参数仅适用于PROTOCOL=TCPS的连接描述符。这类似于要连接到Internet主机的Intranet用户的Web浏览器设置。仅当在sqlnet.ora设置SQLNET.USE_HTTPS_PROXY=1时，此值才会生效。根据客户端查询，将请求的转发Web代理读取超时增加到更高的值。否则，前向Web代理会在没有来自客户端的请求的情况下关闭连接。

此参数适用于服务器端的Oracle Connection Manager。

默认

没有

值

可以与Internet主机建立出站连接的HTTP代理主机名。

例

HTTPS_PROXY = www-proxy.mycompany.com

6.8.1启用

目的

要允许呼叫者检测已终止的远程服务器，通常需要2小时或更长时间才能注意到。

使用说明

通过在连接字符串中的DESCRIPTION参数下放置(ENABLE=broken) ，可以为网络服务客户端启用支持的TCP传输上的keepalive功能。在客户端， tcp_keepalive的默认值为off 。操作系统TCP配置（根据平台而异）定义了实际的保持活动时序细节。

值

破碎

例

net_service_name =（DESCRIPTION =（ENABLE = broken）（ADDRESS =（PROTOCOL = tcp）（HOST = sales1-svr）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp）（HOST = sales2-svr）（PORT = 1521） ）））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com））

虽然前面的示例有多个地址，但未使用ADDRESS_LIST参数。这是因为ADDRESS_LIST参数不是必需的。

6.8.2失败

目的

为多个协议地址启用或禁用连接时故障转移。

使用说明

将参数设置为on ， yes或true ，如果第一个协议地址失败，Oracle Net会在连接时将故障转移到另一个地址。将参数设置为off ， no或false ，Oracle Net会尝试一个协议地址。

将此参数放在DESCRIPTION_LIST参数， DESCRIPTION参数或ADDRESS_LIST参数下。

注意：

不要将GLOBAL_DBNAME在参数SID_LIST_ listener_name一节的listener.ora 。静态配置的全局数据库名称禁用连接时故障转移。

默认

on为DESCRIPTION_LIST ， DESCRIPTION ，和ADDRESS_LIST参数

值

  • yes | on | true

  • no | off | false

例

net_service_name =（DESCRIPTION =（FAILOVER = on）（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST = sales1-svr）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp）（HOST = sales2-svr）（ PORT = 1521）））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））

6.8.3 LOAD_BALANCE

目的

为多个协议地址启用或禁用客户端负载平衡。

使用说明

当您将参数设置为on ， yes或true ，Oracle Net会以随机顺序浏览地址列表，从而平衡各种侦听器或Oracle Connection Manager协议地址上的负载。将参数设置为off ， no或false ，Oracle Net会按顺序尝试协议地址，直到成功为止。

将此参数放在DESCRIPTION_LIST参数， DESCRIPTION参数或ADDRESS_LIST参数下。

值

  • no | off | false

6.9.2 FAILOVER_MODE

目的

如果第一个侦听器在运行时失败，则指示Oracle Net故障转移到其他侦听器。

使用说明

根据配置，会话或正在进行的任何SELECT语句将自动进行故障转移。

此类故障转移称为透明应用程序故障转移（TAF） ，不应与连接时故障转移FAILOVER参数混淆。

将此参数放在CONNECT_DATA参数下。

附加参数

FAILOVER_MODE支持以下参数：

  • BACKUP ：通过其网络服务名称指定故障转移节点。必须为故障转移节点创建单独的网络服务名称。

  • TYPE ：指定故障转移的类型。默认情况下， Oracle Call Interface（OCI）应用程序可以使用三种类型的Oracle Net故障转移功能：

    • SESSION ： SESSION失败。例如，如果用户的连接丢失，则会自动为备份上的用户创建新会话。此类故障转移不会尝试恢复选择。

    • SELECT ：允许具有打开游标的用户在失败后继续获取它们。但是，此模式涉及正常选择操作中客户端的开销。

    • NONE ：这是默认设置，不使用故障转移功能。这也可以明确指定，以防止发生故障转移。

  • METHOD ：指定从主节点到备份节点的故障转移速度：

    • BASIC ：在故障转移时建立连接。此选项在故障转移时间之前几乎不需要备份数据库服务器上的任何工作。

    • PRECONNECT ：预先建立连接。这提供了更快的故障转移，但要求备份实例能够支持来自每个受支持实例的所有连接。

  • TRANSACTION ：允许数据库在发生可恢复的错误后完成当前数据库事务。此参数与COMMIT_OUTCOME=TRUE参数一起使用。

  • RETRIES ：指定故障转移后尝试连接的次数。如果指定了DELAY ，则RETRIES默认为五次重试尝试。

  • DELAY ：指定连接尝试之间等待的时间（以秒为单位）。如果指定了RETRIES ，则DELAY默认为一秒。

注意：

如果注册了回调函数，则忽略RETRIES和DELAY参数。

6.11.1 CONNECT_TIMEOUT

目的

指定客户端建立到Oracle数据库的Oracle Net连接的超时持续时间（以ms ， sec或min为单位）。

使用说明

将此参数放在DESCRIPTION参数下。

CONNECT_TIMEOUT指定的超时间隔是TCP连接超时间隔的超集。它包括连接到提供所请求服务的数据库实例的时间，而不仅仅是TCP连接的持续时间。它接受不同的超时，在值和单位之间有或没有空格。如果没有提到单位，则默认单位为sec 。

超时间隔适用于ADDRESS_LIST每个ADDRESS以及主机名映射到的每个IP地址。

CONNECT_TIMEOUT参数等效于sqlnet.ora参数SQLNET.OUTBOUND_CONNECT_TIMEOUT ，并覆盖它。

例

net_service_name =（DESCRIPTION =（CONNECT_TIMEOUT = 10 ms）（RETRY_COUNT = 3）（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST = sales1-svr）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp）（HOST） = sales2-svr）（PORT = 1521）））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））
