13.23决定性条款

deterministic选项标记一个返回可预测结果且没有副作用的函数。

基于函数的索引，使用PL / SQL函数的虚拟列定义以及启用了查询重写的物化视图需要特殊的函数属性。DETERMINISTIC子句声明函数具有这些属性。

DETERMINISTIC选项可以出现在以下语句中：

  • 功能声明和定义

  • 创建功能声明

  • CREATE PACKAGE声明

  • CREATE TYPE BODY声明

语义

deterministic_clause

确定性

如果DETERMINISTIC子句出现在声明或函数定义中，则函数是确定性的。

DETERMINISTIC子句在函数声明中最多只出现一次，在函数定义中最多出现一次。

如果提供给两个调用的参数相同，则确定性函数必须在两个不同的调用上返回相同的值。

DETERMINISTIC功能可能没有副作用。

DETERMINISTIC函数可能不会引发未处理的异常。

如果具有DETERMINISTIC子句的函数违反任何这些语义规则，则其调用的结果，其值以及对其调用程序的影响都是未定义的。

使用说明

DETERMINISTIC子句是一个断言，该函数服从语义规则。如果函数没有，编译器，SQL执行或PL / SQL执行都不能诊断问题，并且可能会无声地生成错误的结果。

如果要在基于函数的索引的表达式中，在虚拟列定义中调用函数，或者从标记为REFRESH FAST或ENABLE QUERY REWRITE化视图的查询中调用此函数，则必须指定此关键字。当数据库遇到确定性函数时，它会尽可能使用先前计算的结果，而不是重新执行该函数。如果更改了该函数，则必须手动重建所有依赖于函数的索引和物化视图。

不要为结果取决于会话变量或模式对象状态的函数指定DETERMINISTIC ，因为结果可能因调用而异。

不要指定此子句来定义使用包变量的函数，或以任何可能影响函数返回结果的方式访问数据库的函数。

不允许为多态表函数指定此子句。

当出现DETERMINISTIC选项时，编译器可以使用该标记来提高函数执行的性能。

制作属于这些类别的函数是一种很好的编程习惯DETERMINISTIC ：

  • WHERE ， ORDER BY或GROUP BY子句中使用的函数

  • MAP类型或SQL类型的ORDER方法的函数

  • 有助于确定行是否出现在结果集中的函数
