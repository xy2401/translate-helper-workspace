7.101 V $ BACKUP_PIECE

V$BACKUP_PIECE显示有关控制文件中备份片段的信息。每个备份集由一个或多个备份片组成。

柱 数据类型 描述

RECID

数

备份件记录ID

邮票

数

备份件记录戳

SET_STAMP

数

备份集邮票

SET_COUNT

数

备份集计数

片＃

数

备件号（1-N）

复制＃

数

表示启用双工创建的备份片段的副本号。如果备份件未双面打印，则为1。

设备类型

VARCHAR2（17）

备份片所在的设备类型。对磁盘上的备份集设置为DISK。

另请参见： V$BACKUP_DEVICE

处理

VARCHAR2（513）

备份件句柄标识还原时的备份件

评论

VARCHAR2（64）

操作系统或存储子系统返回的注释。对磁盘上的备份片段设置为NULL。该值仅供参考;不需要恢复。

媒体

VARCHAR2（65）

备份片所在的媒体的名称。该值仅供参考;不需要恢复。

MEDIA_POOL

数

副本所在的媒体池。这与在Recovery Manager BACKUP命令的POOL操作数中输入的值相同。

同意

VARCHAR2（3）

（ YES | NO ）表示可以同时访问的媒体上的片段

标签

VARCHAR2（32）

备份件标签。标记在备份集级别指定，但存储在片级。

状态

VARCHAR2（1）

表示该片段的状态： A （可用）， D （已删除）或X （已过期）

开始时间

日期

起始时间

完成时间

日期

完成时间

ELAPSED_SECONDS

数

经过的秒数

DELETED

VARCHAR2（3）

（ YES / NO ） NO表示文件仍然存在。YES表示文件已不再存在，因为它已被删除。

BYTES

数

备份件的大小（以字节为单位）

IS_RECOVERY_DEST_FILE

VARCHAR2（3）

指示文件是否在快速恢复区域中创建（ YES ）（ NO ）

RMAN_STATUS_RECID

数

拥有V$RMAN_STATUS记录ID

RMAN_STATUS_STAMP

数

拥有V$RMAN_STATUS记录戳记

COMPRESSED

VARCHAR2（3）

指示备份件是否已压缩（ YES ）或否（ NO ）

BACKED_BY_VSS

VARCHAR2（3）

卷影复制服务（VSS）是否备份了该文件。此列保留供内部使用。

ENCRYPTED

VARCHAR2（3）

值YES表示加密备份，否则不表示加密备份。

BACKED_BY_OSB

VARCHAR2（3）

值为YES表示已对Oracle Secure Backup进行了备份。否则，由其他第三方磁带库备份。

FOR_XTTS

VARCHAR2（3）

指示这是否是跨平台备份：

  • YES ：这是一个跨平台备份。

  • NO ：这不是跨平台备份。

SAME_ENDIAN

VARCHAR2（3）

如果FOR_XTTS值为YES ，则该列指示备份片段是否具有与当前数据库相同的字节顺序（ YES ）或否（ NO ）;否则为NULL 。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

GUID

RAW（16）

备份所属的PDB的GUID。在删除PDB以识别备份所属的PDB之后，这很有用。
