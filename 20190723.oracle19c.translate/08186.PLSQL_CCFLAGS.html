1.263 PLSQL_CCFLAGS

PLSQL_CCFLAGS提供了一种机制，允许PL / SQL程序员独立控制每个PL / SQL库单元的条件编译。

属性 描述

参数类型

串

句法

PLSQL_CCFLAGS ='< v1 >：< c1 >，< v2 >：< c2 >，...，< vn >：< cn >'

默认值

空字符串

可修改

ALTER SESSION, ALTER SYSTEM

可在PDB中修改

是

价值范围

任何满足内部语法的字符串文字

基本

没有

例子

ALTER SESSION SET PLSQL_CCFLAGS ='DeBug：TruE';

ALTER SESSION SET PLSQL_CCFLAGS ='debug：TRUE';

值

  • < vi >具有不带引号的PL / SQL标识符的形式。它不受限制，可以是保留字或关键字。该案文对案件不敏感。每一个都被称为旗帜或旗帜名称。每个< vi >可以在字符串中出现多次，每次出现可以有不同的标志值，并且标志值可以是不同种类。

  • < ci >是以下之一：PL / SQL布尔文字， PLS_INTEGER文字或文字NULL。该案文对案件不敏感。每个都称为标志值，对应于标志名称。

您可以为PLSQL_CCFLAGS定义任何允许的值。但是，Oracle建议使用此参数来控制调试或跟踪代码的条件编译。建议不要将以下标识符用作标志名称值：

  • Oracle参数的名称（例如， NLS_LENGTH_SEMANTICS ）

  • ：与下列前缀的标识符PLS_ ， PLSQL_ ， PLSCC_ ， ORA_ ， ORACLE_ ， DBMS_ ， SYS_
