5配置Oracle网络

本节中的说明介绍了如果网关使用SNA或TCP / IP通信协议，如何配置网络。

网关必须定义为必须定义Oracle Net Listener和服务名称才能访问网关。为此，请执行以下步骤：

  1. 将网关条目添加到listener.ora文件：

    • 如果您使用SNA：

      （SID_DESC =（SID_NAME = PGA ）（ORACLE_HOME = / oracle / pga / 12.2 ）（PROGRAM = pg4asrv））

      其中： /oracle/pga/12.2是您的网关Oracle主页， PGA是网关SID名称。

    • 或者，如果您使用的是TCP / IP：

      （SID_DESC =（SID_NAME = PGA ）（ORACLE_HOME = / oracle / pga / 12.2 ）（PROGRAM = pg4t4ic））

      其中： /oracle/pga/12.2是网关Oracle主页， PGA是网关SID名称。

  2. 将网关的服务名称添加到Oracle数据库所在系统上的tnsnames.ora文件中。服务名称在。中指定 USING参数为从Oracle数据库访问网关而定义的数据库链接。例如，如果您使用IPC协议适配器并且网关sid是PGA，则将以下条目添加到tnsnames.ora ：
    pgaipc =（DESCRIPTION =（ADDRESS =（PROTOCOL = ipc）（KEY = key ））（CONNECT_DATA =（SID = PGA））（  HS =））

    在此示例中， key是IPC 密钥定义在 IPC的listener.ora文件协议。您只能使用IPC协议 Oracle数据库和网关位于同一系统上。

    如果你正在使用 TCP / IP协议适配器，如果您的网关sid是PGA，则将以下条目添加到tnsnames.ora ：

    pgatcp =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（Host = gateway ）（Port = port ））（CONNECT_DATA =（SID = PGA））（  HS =））

    在此示例中， port是TCP的listener.ora文件中定义的TCP端口 protocol和gateway是gateway所在系统的TCP / IP主机名。

    注意：

    对于以下情况：

    • 如果您的网关和Oracle数据库不在同一系统上，

    • 如果网关和Oracle数据库在同一系统上但是 Oracle数据库侦听器与网关侦听器不同

    您必须将Oracle数据库定义为 PGAU通过在网关所在的系统上为tnsnames.ora添加服务名称。例如：

    ora_server =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（PORT = port ）（HOST = ora_srv ））（CONNECT_DATA =（SID = ora_server ）））

    在这个例子中：

    • port是Oracle数据库中定义的TCP端口 TCP协议的listener.ora ;

    • ora_srv是Oracle数据库所在系统的TCP / IP主机名;和

    • ora_server是Oracle数据库的SID。

    确保启动已定义的侦听器。

示例5-1继续为网关配置通信包

如果您的通信协议是SNA，则必须现在为APPC连接配置SNA通信包配置文件。

配置配置文件以定义与OLTP的LU6.2对话。请参阅以下列表中的相应章节，以了解适用于您的平台的SNA通信包或TCP / IP包。

  • 对于基于Intel Pentium的32位系统的Linux ，请参阅在Linux上配置SNA通信包 。

  • 对于基于AIX的系统，请参阅在基于AIX的系统上配置SNA通信包 。

  • 对于Solaris操作系统（SPARC 64位），请参阅在Solaris上配置SNA通信包 。

此外，如果您的通信协议是SNA，请参阅SNA协议的网关初始化参数中的 “ 使用SNA的网关的示例listener.ora文件 ”和“ 使用SNA的网关的 示例tnsnames.ora文件 ” 。

如果您的通信协议是TCP / IP，请参阅TCP / IP通信协议的网关初始化参数中的 “ 使用TCP / IP的网关的 示例listener.ora文件 ”和“ 使用TCP / IP的网关的示例tnsnames.ora文件 ” 。
