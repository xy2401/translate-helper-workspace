说明dbseg_vm_001.png的说明

图dbseg_vm_001.png说明了连接的工作原理。从左到右，从上到下，是以下组件：

  • 未修补的Oracle 9i客户端（U），它尝试使用Oracle Net Services协议协商连接到固定主机Oracle数据库链接（H）。它失败。Oracle Database 12c的固定Oracle数据库链接连接到Oracle 9i目标（T）。

  • 10g客户端（E），可以连接，因为它已应用了关键补丁更新CPUOct2012。

  • 9i客户端之后是Oracle Database Release 11.2.0.3客户端（C），它成功使用Oracle Net Services协议首先连接到固定主机数据库链接（H），然后连接到目标Oracle 9i数据库（T） 。

  • 主机和目标数据库的SQLNET.ALLOWED_LOGON设置如下：

    • 固定主机12c（H）设置如下：

      H通过使用SQLNET.ALLOWED_LOGON_VERSION_CLIENT = 8充当T的客户端

      H还通过使用SQLNET充当（C）和（U）的服务器.ALLOWED_LOGON_VERSION_SERVER = 12

    • 目标9i数据库（T）设置为SQLNET.ALLOWED_LOGON_VERSION = 8
