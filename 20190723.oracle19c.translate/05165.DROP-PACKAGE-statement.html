14.17 DROP PACKAGE声明

DROP PACKAGE语句从数据库中删除存储的包。

该声明删除了包的正文和规范。

注意：

不要使用此语句从包中删除单个对象。相反，使用OR REPLACE子句使用“ CREATE PACKAGE语句 ”和“ CREATE PACKAGE BODY语句 ”重新创建没有对象的包。

先决条件

程序包必须位于您的架构中，或者您必须具有DROP ANY PROCEDURE系统特权。

句法

drop_package :: =


说明drop_package.eps的说明

语义

drop_package

身体

仅丢弃包装的主体。如果省略此子句，则数据库将同时删除包的主体和规范。

如果仅删除包的主体但不删除其规范，则数据库不会使依赖对象无效。但是，在重新创建包体之前，不能调用包规范中声明的过程或存储函数。

模式

包含该包的模式的名称。默认值：您的架构。

包

要删除的包的名称。

数据库使依赖于包规范的任何本地对象无效。如果随后引用其中一个对象，则数据库会尝试重新编译该对象，如果尚未重新创建已删除的包，则会返回错误。

如果任何统计类型与包关联，则数据库将使用FORCE子句取消关联统计类型，并删除使用统计类型收集的任何用户定义的统计信息。

  • 有关DISASSOCIATE STATISTICS语句的信息，请参见Oracle数据库SQL语言参考

例

例14-34删除包

此语句删除了在“ CREATE PACKAGE BODY语句 ”中创建的emp_mgmt软件包的规范和正文，使依赖于规范的所有对象无效：

DROP PACKAGE emp_mgmt;
