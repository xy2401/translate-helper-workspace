目的

LAST_VALUE是一种分析函数，可用于数据密集化。它返回有序值集中的最后一个值。

注意：

此语法的两种形式具有相同的行为。顶部分支是ANSI格式，Oracle建议ANSI兼容。

{ RESPECT | IGNORE } NULLS确定expr空值是否包含在计算中或从计算中消除。默认值为RESPECT NULLS 。如果设定的最后一个值是零，则函数返回NULL ，除非您指定IGNORE NULLS 。如果指定IGNORE NULLS LAST_VALUE ，则LAST_VALUE返回LAST_VALUE的最后一个非空值，如果所有值都为null，则返回NULL 。有关数据密集化的示例，请参阅“ 使用分区外连接：示例 ” 。

您不能通过使用LAST_VALUE或expr任何其他分析函数来嵌套分析函数。但是，您可以为expr使用其他内置函数表达式。有关有效形式的expr信息，请参阅“ 关于SQL表达式 ” 。

如果省略analytic_clause的windowing_clause ，则默认为RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW 。此默认值有时会返回意外值，因为窗口中的最后一个值位于窗口的底部，而不是固定的。它会随着当前行的变化而不断变化。对于预期的结果，指定windowing_clause为RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 。或者，您可以将windowing_clause指定为RANGE BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING 。

例子

以下示例为每行返回获得最低薪水的员工的雇用日期：

SELECT employee_id，last_name，salary，hire_date，LAST_VALUE（hire_date）OVER（按工资顺序排列在无限制下行和下达之后的行程）作为lv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 13-JAN-01 101 Kochhar 17000 21-SEP-05 13-JAN-01 102 De Haan 17000 13 -JAN-01 13-JAN-01

此示例说明了LAST_VALUE函数的非确定性。Kochhar和De Haan有相同的薪水，所以他们在相邻的行。首先出现Kochhar，因为子查询中的行按hire_date排序。但是，如果按照hire_ date按降序排序行，如下一个示例所示，则函数返回不同的值：

SELECT employee_id，last_name，salary，hire_date，LAST_VALUE（hire_date）OVER（按照UNBOUNDED PRECEDING和UNBOUNDED POLLOWING之间的工资DESC行顺序）as lv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 21-SEP-05 102 De Haan 17000 13-JAN-01 21-SEP-05 101 Kochhar 17000 21 -SEP-05 21-SEP-05

以下两个示例显示如何通过对唯一键进行排序来使LAST_VALUE函数具有确定性。通过使用salary和唯一键employee_id在函数内进行排序，无论子查询中的顺序如何，都可以确保相同的结果。 

SELECT雇员标识，姓氏，工资，聘用日期，LAST_VALUE（聘用日期）OVER（ORDER BY工资DESC，EMPLOYEE_ID之间UNBOUNDED PRECEDING，无节制的下列各行）像LV FROM（SELECT * FROM雇员WHERE部门= 90 ORDER BY聘用日期）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 13-JAN-01 101 Kochhar 17000 21-SEP-05 13-JAN-01 102 De Haan 17000 13 -JAN-01 13-JAN-01 SELECT雇员标识，姓氏，工资，HIRE_DATE，LAST_VALUE（HIRE_DATE）OVER（ORDER BY薪金DESC，EMPLOYEE_ID BETWEEN UNBOUNDED PRECEDING和无界下列各行）AS LV FROM（SELECT * FROM雇员WHERE部门= 90 ORDER BY hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 13-JAN-01 101 Kochhar 17000 21-SEP-05 13-JAN-01 102 De Haan 17000 13 -JAN-01 13-JAN-01

以下两个示例显示当您使用逻辑偏移（ RANGE而不是ROWS ）时， LAST_VALUE函数是确定性的。当找到ORDER BY表达式的重复项时， LAST_VALUE是expr的最高值：

SELECT employee_id，last_name，salary，hire_date，LAST_VALUE（hire_date）OVER（按照UNBOUNDED PRECEDING和UNBOUNDED FOLLOWING之间的工资DESC 范围排序）作为lv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 21-SEP-05 102 De Haan 17000 13-JAN-01 21-SEP-05 101 Kochhar 17000 21 -sep-05 21-SEP-05 SELECT雇员标识，姓氏，工资，HIRE_DATE，LAST_VALUE（HIRE_DATE）OVER（ORDER BY薪金DESC RANGE BETWEEN UNBOUNDED PRECEDING和无界的关注）AS LV FROM（SELECT * FROM雇员WHERE部门= 90 ORDER BY hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE LV ----------- ------------------------- --------- -  --------- --------- 100 King 24000 17-JUN-03 21-SEP-05 102 De Haan 17000 13-JAN-01 21-SEP-05 101 Kochhar 17000 21 -SEP-05 21-SEP-05
