FLASHBACK数据库

目的

使用FLASHBACK DATABASE语句将数据库返回到过去时间或系统更改号（SCN）。此语句提供了执行不完整数据库恢复的快速替代方法。

在FLASHBACK DATABASE操作之后，为了具有对闪回数据库的写访问权，必须使用ALTER DATABASE OPEN RESETLOGS语句重新打开它。

先决条件

您必须具有SYSDBA ， SYSBACKUP或SYSDG系统特权。

如果您连接到多租户容器数据库（CDB）：

  • 要闪回CDB，必须连接到root，并且必须具有通常授予的SYSDBA ， SYSBACKUP或SYSDG系统特权。

  • 要闪回PDB，您必须连接到根目录，并且必须具有通常授予的SYSDBA ， SYSBACKUP或SYSDG系统特权，或者您必须连接到要闪回的PDB，并且必须具有SYSDBA ， SYSBACKUP ，或SYSDG系统特权，通常授予或在该PDB中本地授予。

必须为数据库准备一个快速恢复区。除非您将数据库闪回到保证还原点，否则必须使用ALTER DATABASE FLASHBACK ON语句将数据库置于FLASHBACK模式。必须装入数据库但不能打开。

此外：

  • 数据库必须以ARCHIVELOG模式运行。

  • 必须使用当前控制文件装入数据库，但不能打开。控制文件不能是备份或重新创建。从备份还原或重新创建数据库控制文件时，将丢弃所有现有的闪回日志信息。

  • 数据库必须不包含使用SQL语句ALTER TABLESPACE禁用闪回功能的联机表空间...FLASHBACK OFF 。

语义

发出FLASHBACK DATABASE语句时，Oracle数据库首先验证是否所有必需的归档和联机重做日志都可用。如果它们可用，则它会将数据库中的所有当前在线数据文件还原到SCN或此语句中指定的时间。

  • 保留在数据库中的闪回数据量由DB_FLASHBACK_RETENTION_TARGET初始化参数和快速恢复区域的大小控制。您可以通过查询V$FLASHBACK_DATABASE_LOG视图来确定可以回退数据库的时间。

  • 如果数据库中仍有足够的数据来执行闪回，则可以使用标准恢复过程将数据库恢复到过去的某个时间点。

  • 如果一组数据文件的数据不足，则数据库将返回错误。在这种情况下，您可以使这些数据文件脱机并重新发出该语句以还原数据库的其余部分。然后，您可以尝试使用标准恢复过程恢复脱机数据文件。

支持

指定STANDBY以将备用数据库还原为较早的SCN或时间。如果数据库不是备用数据库，则数据库将返回错误。如果省略此子句，则database可以是主数据库或备用数据库。

PLUGGABLE

指定PLUGGABLE以闪回PDB。无论当前容器是要回闪的根还是PDB，都必须指定此子句。

闪回PDB的限制

  • 您无法闪回代理PDB。

  • 如果CDB处于共享撤消模式，则只能将PDB闪回到干净的PDB还原点。有关更多信息，请参阅CREATE RESTORE POINT的CLEAN子句。

数据库

如果要闪回非CDB或CDB，则可以选择指定要闪回的数据库的名称。如果省略database ，则Oracle数据库会闪回由初始化参数DB_NAME的值标识的数据库。

如果要闪回PDB并且当前容器是根，则使用database指定要闪回的PDB的名称。如果要闪回PDB并且当前容器是该PDB，则可以选择使用database来指定PDB名称。

到SCN条款

指定系统更改编号（SCN）：

  • TO SCN将数据库恢复到指定SCN的状态。

  • TO BEFORE SCN将数据库恢复到其指定SCN之前的系统更改号的状态。

您可以通过查询V$DATABASE视图的CURRENT_SCN列来确定当前SCN。这反过来允许您将SCN保存到假脱机文件，例如，在运行高风险批处理作业之前。

到TIMESTAMP条款

指定有效的日期时间表达式。

  • TO TIMESTAMP将数据库恢复到指定时间戳的状态。

  • TO BEFORE TIMESTAMP将数据库恢复到指定时间戳之前一秒的状态。

您可以将时间戳表示为与确定值（例如SYSDATE的偏移量，或者表示为绝对系统时间戳。

恢复点条款

指定此子句可将数据库闪回到指定的还原点。如果尚未启用闪回数据库，则这是您可以在此FLASHBACK DATABASE语句中指定的唯一子句。如果数据库不处于FLASHBACK模式，如上面“先决条件”部分所述，那么这是您可以为此语句指定的唯一子句。

RESETLOGS

指定TO BEFORE RESETLOGS闪数据库恢复到刚刚过去的RESETLOGS操作之前（ ALTER DATABASE OPEN RESETLOGS ）。

例子

假设您已为数据库准备了快速恢复区并启用了介质恢复，请启用数据库FLASHBACK模式并使用以下语句打开数据库：

STARTUP MOUNT ALTER DATABASE FLASHBACK ON; ALTER DATABASE OPEN;

在数据库打开至少一天后，您可以使用以下语句一天闪回数据库：

SHUTDOWN DATABASE STARTUP MOUNTBACK DATABASE TO TIMESTAMP SYSDATE-1;
