7在基于AIX的系统上配置SNA通信包

APPC的Oracle数据库网关使用SNA高级程序编程通信（APPC / LU6.2）协议与在线事务处理（OLTP）进行通信。所有基于AIX的APPC系统支持均由IBM提供 SNA服务器产品。

此产品需要一组存储的定义（称为配置文件），以支持使用OLTP在网关和应用程序之间建立连接。每配置文件由配置文件名称，配置文件类型和描述配置文件的一组字段组成。配置文件类型中的字段通常是操作参数值和与配置文件相关的其他SNA配置文件的名称的混合。

如果您的网关使用SNA通信协议，请参阅以下主题。主题描述如何创建和激活SNA服务器配置文件。

注意：

完成本章中的说明以配置通信协议后，请参阅配置Oracle网络以继续进行网络配置。

处理入站连接

许多OLTP提供用于操纵入站（客户端）APPC会话请求的安全性行为的选项。有关此主题的详细信息，请参阅相应的OLTP文档。

注意：

对于CICS，网关不支持一个安全选项。在CICS CONNECTION定义中指定的ATTACHSEC=PERSISTENT需要网关中尚不可用的功能。

但是，网关完全支持ATTACHSEC=LOCAL ， ATTACHSEC=IDENTIFY ， ATTACHSEC=VERIFY和ATTACHSEC=MIXIDPE 。

独立对抗的LU

Oracle建议使用适用于APPC的Oracle Database Gateway的独立LU，因为它们支持多个并行会话或会话。这意味着多个Oracle客户端应用程序可以通过独立LU与同一OLTP同时处于活动状态。

从属LU仅支持单个活动会话。CP（在这种情况下，用于AIX的SNA服务器）在已经活动的会话后面对来自网关服务器的其他会话请求进行排队。换句话说，对话是针对从属LU的单线程。

如果正确定义了从属LU，则无需更改Oracle数据库网关以进行APPC配置，主机事务或OLTP的启动方式。

依赖LU的操作影响是第一个客户端应用程序可以通过Oracle数据库网关与OLTP发起对话。当该事务处于活动状态时（可能是几秒到几分钟到几小时，具体取决于客户端应用程序和事务的设计方式），启动与同一OLTP实例的对话的任何其他客户端应用程序在等待前一个对话后都会挂起。

如果生产应用程序一次只使用一个会话或事务，那么应该没有任何影响。

但是，测试或其他应用程序开发可能需要额外的并发对话或事务。每个都要求在远程主机上定义其他从属LU，以及其他SNA服务器配置文件，这些配置文件定义IBM pSeries工作站上的其他从属LU。启动对话的TIP必须通过不同的辅助信息配置文件或通过覆盖LU名称来指定不同的Partner LU。

为网关创建SNA配置文件

您可以使用中的菜单创建和修改SNA服务器配置文件定义 AIX系统管理界面工具（ SMIT）。

SNA服务器配置文件的维护通常由具有root访问权限的用户完成。以下信息适用于为网关创建配置文件的用户。在阅读本节之前，您应该对SNA有一些了解。

通过使用 smit，你应该能够接受大多数默认值。分配给新的配置文件集中的许多字段的默认值对于网关是可接受的。

$ORACLE_HOME/dg4appc/sna子目录包含一个示例的网关配置文件集。 pgasna.export文件。

在构建SNA服务器配置文件之前，请检查相应的示例导出文件以确定所需的配置文件，其内容及其相互关系。导出文件格式是面向文本的，每个配置文件的每个字段都有明确标记。您可以在smit会话中使用配置文件时打印要使用的导出文件的副本。

档案类型

几种类型的SNA服务器配置文件与网关APPC / LU6.2操作相关。可以使用相应的smit菜单创建和编辑每个配置文件，该菜单可以从“通信应用程序和服务”主菜单选项中访问。

配置文件按层次顺序显示。首先讨论层次结构中最低的概要文件类型。这与创建配置文件的逻辑顺序相匹配。您可以使用smit list菜单指定配置文件名称。

SNA节点配置文件

SNA节点配置文件定义了各种SNA系统默认值。将“最大会话数”和“最大会话数”字段设置为足以处理预期的最大并发网关会话数的值，以及其他应用程序在系统上使用的任何非网关会话和会话。

设置“恢复资源管理器（RRM）”？“字段为no ，除非您已在AIX系统上运行其他应用程序，并且要求将此字段设置为yes 。例如，CICS pSeries和Encina需要将此字段设置为yes 。

链接站配置文件

Link Station Profile和相关的DLC配置文件描述和控制IBM pSeries与网络的连接。此处不讨论配置文件内容的详细信息，因为APPC的Oracle数据库网关不对这些配置文件强加特殊要求。样本资料分布在 pgasna.export包含为令牌环网络连接创建的配置文件。如果需要，将在合作伙伴LU位置配置文件中稍后指定链接站配置文件名称。

模式简介

模式配置文件指定用于确定的参数：

  • APPC / LU6.2并行会话限制

  • 发送和接收调步值

  • SNA RU大小

  • 在会话启动时发送到OLTP的模式名称

您指定的模式名称必须定义为 OLTP通讯软件。在咨询负责配置OLTP通信软件的用户之后，除了其他模式参数之外，还要选择模式名称。

与并行会话限制相关的参数确定网关实例与OLTP之间允许的最大并发对话数。这相当于通过网关实例的最大并发活动远程事务程序调用次数。

模式名称（例如ORAPLU62 ）稍后在边信息简介。

注意：

不要将模式配置文件与模式名称混淆。

本地LU配置文件

本地的 LU（逻辑单元）配置文件描述了网关通过的SNA LU。

一个必须将LU名称分配给网关。分配给网关的LU名称可能在SNA网络中的其他位置是必需的。请与SNA网络管理员联系，以确定要在配置文件中指定的正确LU名称。

要创建本地LU配置文件，请执行以下步骤：

  1. 将“本地LU名称”设置为分配给网关的LU名称。

  2. 应使用“Local LU别名”字段为LU分配别名。此别名稍后将在辅助信息配置文件中使用。

  3. 将“本地LU依赖”字段设置为no 。

本地LU配置文件名称稍后在辅助信息配置文件中指定。

合作伙伴LU简介

该 Partner LU Profile描述了网关与之通信的OLTP系统的SNA LU。必须在此配置文件中指定OLTP LU的名称和SNA网络的名称。请与您的SNA网络管理员联系，以确定正确的LU和网络名称。

要创建Partner LU配置文件，请执行以下步骤：

  1. 将“标准合作伙伴LU名称”字段设置为网络名称，后跟句点，后跟OLTP LU名称。例如， network.oltplu 。

  2. 您可以为合作伙伴分配别名 LU名称通过将“Partner LU别名”设置为您选择的值。这使您可以在不知道完全限定的合作伙伴LU名称的情况下引用合作伙伴LU，并在合作伙伴LU名称更改时最小化更改。

  3. 设置“支持并行会话？“字段为yes除非您的OLTP不支持并行会话。

  4. 如果您打算使用SNA会话或会话安全性，那么设置“支持会话安全性？“和”支持对话安全？“根据需要填写字段。这些设置需要会话安全和会话安全配置文件。

合作伙伴LU位置资料

该当合作伙伴LU所在的远程主机不是支持APPN的节点时，将使用合作伙伴LU位置配置文件。许多大型机系统没有APPN功能。例如，在版本4之前运行VTAM版本的z / OS系统不支持APPN。此外，如果您的硬件连接是通过在版本5之前运行NCP版本的前端处理器，则不支持APPN。在这些情况下，Partner LU Location Profile可用于指定系统服务控制点（SSCP）或控制点（CP）的名称，该控制点拥有与伙伴LU的网络连接。

将“完全限定的合作伙伴LU名称”字段设置为网络名称，后跟句点，然后是OLTP LU名称，例如network.oltplu 。

根据网络配置的要求设置“Partner LU location method”和相关字段。如果使用owning cp选项，则应将“完全限定拥有控制点（CP）名称”字段设置为SSCP或CP名称，该名称拥有与伙伴LU的网络连接。对于VTAM，SSCP名称是VTAM NETID启动参数的值，通常位于VTAMLST成员ATCSTR00 。

边信息简介

辅助信息配置文件是必需的配置文件，用于标识要通过Oracle Database Gateway for APPC访问的目标OLTP系统。

辅助信息配置文件标识以下内容：

  • 本地的 LU别名

  • 合作伙伴LU别名或完全限定名称

  • 远程事务程序名称（可选）

  • 模式名称

为每个辅助信息配置文件字段设置配置文件信息如下：

  • 将本地LU或控制点别名设置为分配给本地LU配置文件中的本地LU的别名。

  • 将模式名称设置为模式配置文件中指定的实际模式名称。

  • 使用实际远程TP名称或在执行时覆盖的虚拟名称设置远程事务程序名称（RTPN）。

  • 设置“十六进制的RTPN？“如果远程TP名称是十六进制，则字段为yes 。

  • 如果存在“Partner LU别名”字段，则在Partner LU Profile中添加分配给伙伴LU的别名。

  • 如果存在“完全限定合作伙伴LU名称”字段，则添加合作伙伴LU的完全限定合作伙伴LU名称。
