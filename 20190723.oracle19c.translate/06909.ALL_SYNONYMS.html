3.102 ALL_SYNONYMS

ALL_SYNONYMS描述了当前用户可访问的同义词。

以下条件确定ALL_SYNONYMS显示的同义词列表：

  • 登录用户拥有的所有私有同义词，即使指向的基础对象不可访问。

  • 所有公共同义词，即使指向的基础对象不可访问。

  • 由不同用户拥有的所有私有同义词，其中由该同义词或任何嵌套同义词链指向的最终基础对象，由于对登录用户的授予或对角色的授予而被知道是可访问的。本次会议的效果。

  • 如果当前会话具有以下任何特权，则会显示直接指向本地对象的所有同义词，因为假定会话可以访问这些对象：

    • 锁定任何表格

    • 读一个表

    • 选择任何表格

    • 插入任何表

    • 更新任何表格

    • 删除任何表格

    排除指向远程对象的同义词，因为刚刚列出的系统权限不会自动传达对这些远程对象的访问权限。此外，如果同义词指向除表和视图之外的对象（例如序列，PL / SQL过程等），则此规则可能会显示最终解析为此会话无法访问的对象的同义词。

  • 不同用户拥有的所有私有同义词（同义词是通过数据库链接）都被排除在外。

相关观点

  • DBA_SYNONYMS描述数据库中的所有同义词。

  • USER_SYNONYMS描述当前用户拥有的同义词。此视图不显示OWNER列。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

同义词的所有者

SYNONYM_NAME

VARCHAR2（128）

同义词的名称

TABLE_OWNER

VARCHAR2（128）

如果目标是公共同义词（即TABLE_NAME引用的对象），则由同义词引用的对象的所有者或引用同义词的创建者。

虽然该列称为TABLE_OWNER ，但所拥有的对象不一定是表。它可以是任何一般对象，例如视图，序列，存储过程，同义词等。

TABLE_NAME

VARCHAR2（128）

同义词引用的对象的名称。虽然该列名为TABLE_NAME ，但该对象不一定必须是表。它可以是任何一般对象，例如视图，序列，存储过程，同义词等。

DB_LINK

VARCHAR2（128）

引用的数据库链接的名称（如果有）

ORIGIN_CON_ID

VARCHAR2（256）

数据源自的容器的ID。可能的值包括：

  • 0 ：该值用于非CDB中的行。该值不用于CDB。

  • n ：此值用于包含源自容器ID为n的容器中的数据的行（如果行来自root，则n = 1）
