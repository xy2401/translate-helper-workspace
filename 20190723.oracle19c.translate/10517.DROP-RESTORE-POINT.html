DROP RESTORE POINT

目的

使用DROP RESTORE POINT语句从数据库中删除正常还原点或保证还原点。

  • 您无需删除正常的还原点。必要时，数据库会自动删除最旧的还原点，如restore_point的语义中所述。但是，如果要重用该名称，则可以删除正常的还原点。

  • 保证还原点不会自动删除。因此，如果要从数据库中删除保证还原点，则必须使用此语句显式执行此操作。

先决条件

要删除正常还原点，必须具有SELECT ANY DICTIONARY ， FLASHBACK ANY TABLE ， SYSBACKUP或SYSDG系统特权。

要删除保证还原点，你必须符合下列条件之一 ：

  • 您必须连接AS SYSDBA ，或AS SYSBACKUP或AS SYSDG 。

  • 您必须已被授予SYSDBA权限，并且正在使用多租户数据库。

  • 您必须以SYS用户身份运行，并且正在使用多租户数据库。

连接到多租户容器数据库（CDB）时，可以删除还原点，如下所示：

  • 要删除正常的CDB还原点，当前容器必须是根，并且您必须具有SELECT ANY DICTIONARY或FLASHBACK ANY TABLE系统特权，可以在根目录中SYSBACKUP或在本地授予，也可以具有SYSDBA ， SYSBACKUP或SYSDG系统特权一般授予。

  • 要删除保证的CDB还原点，当前容器必须是根，并且必须具有通常授予的SYSDBA ， SYSBACKUP或SYSDG系统特权。

  • 要删除正常的PDB还原点，当前容器必须是根，并且您必须具有通常授予的SELECT ANY DICTIONARY ， FLASHBACK ANY TABLE ， SYSDBA ， SYSBACKUP或SYSDG系统特权，或者当前容器必须是其中的PDB如果要创建还原点，则必须具有SELECT ANY DICTIONARY ， FLASHBACK ANY TABLE ， SYSDBA ， SYSBACKUP或SYSDG系统特权， SYSDG特权通常授予或在该PDB中本地授予。

  • 要删除保证的PDB还原点，当前容器必须是根，并且您必须具有通常授予的SYSDBA ， SYSBACKUP或SYSDG系统特权，或者当前容器必须是要在其中创建还原点的PDB和您必须具有SYSDBA ， SYSBACKUP或SYSDG系统特权， SYSDG特权通常授予或在该PDB中本地授予。

句法

drop_restore_point :: =

语义

restore_point

指定要删除的还原点的名称。

对于可插入的数据库

此子句使您可以在连接到根时删除PDB还原点。对于pdb_name ，请指定包含要删除的还原点的PDB的名称。

如果您已连接到要从中删除还原点的PDB，则无需指定此子句。但是，如果指定此子句，则必须指定要连接的PDB的名称。

例子

删除还原点：示例

以下示例删除了good_data还原点，该点在“ 创建和使用还原点：示例 ”中创建 ：

DROP RESTORE POINT good_data;
