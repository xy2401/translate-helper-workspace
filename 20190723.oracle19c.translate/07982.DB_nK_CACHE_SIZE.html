1.66 DB_ n K_CACHE_SIZE

DB_ n K_CACHE_SIZE （其中n = K_CACHE_SIZE ）指定n K缓冲区的高速缓存大小。

属性 描述

参数类型

大整数

句法

DB_[2 | 4 | 8 | 16 | 32]K_CACHE_SIZE = integer [K | M | G]

默认值

0 （默认情况下未配置其他块大小缓存）

可修改

改变系统

可在PDB中修改

没有

价值范围

最小值： 0 （大于零的值会自动修改为粒度大小*处理器组数，或4 MB * CPU数，以较大者为准）

最大值：取决于操作系统

基本

没有

仅当DB_BLOCK_SIZE的值不是n K时，才能设置此参数。例如，如果DB_BLOCK_SIZE = 4096 ，则指定参数DB_4K_CACHE_SIZE是非法的（因为4 KB块缓存的大小已由DB_CACHE_SIZE指定）。

如果有任何具有n K块大小的在线表空间，请不要将此参数设置为零。

特定于操作系统的块大小限制适用。例如，如果操作系统的最大块大小小于32 KB，则无法设置DB_32K_CACHE_SIZE 。此外，如果最小块大小大于2 KB，则无法设置DB_2K_CACHE_SIZE 。
