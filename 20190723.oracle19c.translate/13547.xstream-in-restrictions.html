C.1.1入站服务器的不支持的数据类型

入站服务器不支持某些数据类型。

入站服务器不会在以下数据类型的列中应用包含DML更改结果的行LCR：

  • ROWID

  • 嵌套表

  • 以下Oracle提供的类型： ANYTYPE ， ANYDATASET ，URI类型， SDO_TOPO_GEOMETRY ， SDO_GEORASTER和Expression

注意：

XStream不支持具有不同宽度多字节字符集的数据库中的LONG列。

如果入站服务器尝试应用包含有关不受支持的数据类型列的信息的行LCR，则会引发错误。此外，入站服务器无法将DML更改应用于以下类型的表：

  • 临时表

  • 包含不受支持的数据类型的对象表

如果入站服务器尝试应用此类更改，则会引发错误。当入站服务器引发LCR的错误时，它会将包含LCR的事务移动到错误队列中。

这些数据类型限制适用于普通（堆组织）表和索引组织表。

C.1.2应用处理程序的不支持的数据类型

应用处理程序不支持某些数据类型。

语句DML处理程序无法处理行LCR中的LONG ， LONG RAW或非组装LOB列数据。但是，语句DML处理程序可以处理由LOB程序集构造的行LCR中的LOB列数据。默认情况下，语句DML处理程序启用LOB程序集。

过程DML处理程序和错误处理程序无法处理行LCR中的LONG或LONG RAW列数据。但是，过程DML处理程序和错误处理程序可以处理行LCR中的非组装和已组装的LOB列数据，但这些处理程序无法修改非组装的LOB列数据。

C.1.3入站服务器忽略的DDL更改类型

入站服务器忽略某些类型的DDL更改。

入站服务器不支持以下类型的DDL更改。不应用这些类型的DDL更改：

  • ALTER MATERIALIZED VIEW

  • ALTER MATERIALIZED VIEW LOG

  • CREATE DATABASE LINK

  • CREATE SCHEMA AUTHORIZATION

  • CREATE MATERIALIZED VIEW

  • CREATE MATERIALIZED VIEW LOG

  • DROP DATABASE LINK

  • DROP MATERIALIZED VIEW

  • DROP MATERIALIZED VIEW LOG

  • FLASHBACK DATABASE

  • 改名

如果入站服务器收到指定无法应用的操作的DDL LCR，则入站服务器将忽略DDL LCR并在入站服务器跟踪文件中记录以下消息，后跟已忽略的DDL文本：

入站服务器忽略以下DDL：

如果应根据入站服务器规则集应用包含更改的DDL LCR，则入站服务器将应用所有其他类型的DDL更改。

注意：

  • 入站服务器应用ALTER object_type object_name RENAME更改，例如ALTER TABLE jobs RENAME 。因此，如果要进行重命名要应用的对象的DDL更改，请使用ALTER object_type object_name RENAME语句而不是RENAME语句。更改数据库对象的名称后，可能需要指定新数据库对象名称的新规则来复制对数据库对象的更改。

  • 名称“物化视图”与名称“快照”同义。入站服务器会忽略物化视图上语句的快照等效项。

C.2.1子集规则的限制

限制适用于子集规则。

以下限制适用于子集规则：

  • 具有子集规则中引用的表名的表必须与子集规则存在于同一数据库中，并且此表必须位于为子集规则中的表引用的相同模式中。

  • 如果子集规则位于入站服务器的正规则集中，则表必须包含子集条件中指定的列，并且每列的数据类型必须与计算到的行LCR中相应列的数据类型匹配对于子集规则为TRUE 。

  • 不支持为具有以下数据类型的一列或多列的表创建子集规则：LOB， LONG ， LONG RAW ，用户定义类型（包括对象类型， REF ，varrays，嵌套表）和Oracle提供的类型（包括Any类型，XML类型，空间类型和媒体类型）。

C.3.1声明性基于规则的转换的不支持的数据类型

除了添加列转换之外，对列进行操作的基于声明规则的转换支持入站服务器支持的相同数据类型。

添加列转换不能添加以下数据类型的列： BLOB ， CLOB ， NCLOB ， BFILE ， LONG ， LONG RAW ， ROWID ，用户定义类型（包括对象类型， REF ，varrays，嵌套表）和Oracle提供的类型（包括Any类型，XML类型，空间类型和媒体类型）。

C.4扩展数据类型的限制中的XStream

限制适用于XStream In中的扩展数据类型。

所述的最大大小VARCHAR2 ， NVARCHAR2 ，和RAW数据类型已在Oracle数据库12c中，当已增加COMPATIBLE初始化参数设置为12.0.0和MAX_STRING_SIZE初始化参数设置为EXTENDED 。 XStream In支持这些扩展数据类型。

但是，以下限制适用于扩展数据类型：

  • 有关扩展数据类型列的信息可能不包含在数据操作语言（DML）操作的原始LCR中。相反，XStream In可能会将扩展数据类型列视为与处理LOB列的方式类似。具体而言，其他LCR可能包含扩展数据类型列的信息。

  • 对于扩展数据类型列，XStream规则无法访问LCR中的数据。

  • 无法在子集规则子句中指定扩展数据类型列。

  • 扩展数据类型列不能用于冲突检测。

  • 扩展数据类型列不能用于替换主键，以便与DBMS_APPLY_ADM.SET_KEY_COLUMNS过程一起应用。

  • 扩展数据类型列不能用于以下类型的基于声明规则的转换：

    • 添加列

    • 保留专栏
