一个 Oracle数据库升级实用工具

Oracle升级实用程序脚本有助于执行Oracle数据库升级。

用于升级Oracle数据库的脚本

Oracle为升级之前，期间和之后运行的升级提供了一组工具和脚本。

注意：

Oracle提供的某些脚本无法在UPGRADE模式下运行。

下表列出了各种脚本和工具以及每种脚本和工具的说明。

表A-1升级，升级后和降级脚本

脚本 描述

catcon.pl

必须在UPGRADE模式下运行。升级CDB时使用此脚本。

catctl.pl

并行升级实用程序（ catctl.pl和shell脚本dbupgrade ）。您必须以升级模式运行这些脚本。在Oracle数据库12c与后来的版本中，并行升级程序代替了SQL升级程序catupgrd.sql 。使用Parallel Upgrade Utility，您可以并行运行升级脚本和进程。此功能可充分利用您的服务器CPU容量，并可缩短升级时间。DBUA使用此工具。

dbdowngrade

Shell脚本实用程序，它调用catdwgrd.sql脚本，并确保对catcon.pl脚本的调用使用推荐的降级值。此功能有助于减少由于升级过程中产生过多线程而导致的潜在错误。多租户架构（CDB）降级尤其有用。使用此脚本是Oracle建议的降级过程。

dbupgrade ， dbupgrade.cmd

 调用catctl.pl脚本的catctl.pl脚本。Oracle Database 12 c第 2版（12.2）及更高版本提供了这些shell脚本。这些脚本使您可以在shell命令提示符下输入dbupgrade命令。您可以使用默认值运行这些脚本，也可以使用与从Perl提示符运行catctl.pl相同的输入参数来运行catctl.pl 。使用dbupgrade Linux和UNIX系统，并dbupgrade.cmd的Windows系统。

catdwgrd.sql

这是降级脚本，在过程中用于降级到升级的早期版本。

catnoamd.sql

不要在UPGRADE模式下运行。升级后仅运行此脚本。catnoamd.sql脚本删除了OLAP目录组件，也称为AMD。

catnoexf.sql

升级过程为您运行此脚本;但是，您可以在升级之前运行catnoexf.sql脚本。不要在UPGRADE模式下运行此脚本。catnoexf.sql脚本删除了此版本中不支持的表达式筛选器（EXF）和数据库规则管理器（RUL）组件。

catuppst.sql

如果执行手动升级，则必须通过DBUA或手动运行此脚本。

DBUA自动运行catuppst.sql 。您只需单独运行此脚本以进行手动升级。

不要在升级模式下运行它。运行位于ORACLE_HOME /rdbms/admin目录中的catuppst.sql ，以执行不需要数据库处于UPGRADE模式的剩余升级操作。如果Oracle主目录中安装了Oracle捆绑包修补程序或修补程序集更新（PSU或BP），则此脚本会自动将该修补程序集更新应用于数据库。

警告：如果执行手动升级，并且未运行catuppst.sql ，则数据库会随着时间的推移而降低性能。

catuptabdata.sql

 catuptabdata.sql脚本由catuppst.sql自动运行，以便在升级期间受Oracle维护类型更改影响的任何Oracle维护表上运行ALTER TABLE UPGRADE。

您可以手动运行catuptabdata.sql脚本以升级因任何Oracle维护类型的更改而需要升级的任何Oracle维护表。必须使用已授予SYSDBA系统特权且已连接到AS SYSDBA的用户帐户运行该命令。

emdwgrd

不要在UPGRADE模式下运行此脚本。emdwgrd实用程序为您的11g版本数据库保存Oracle Enterprise Manager数据库控制（DB Control）配置文件和数据。

如果您可能希望将Oracle Database 18c降级到Oracle Database 11g （11.2），并且想要还原Oracle DB Control，则必须在升级之前运行emdwgrd 。

另请参阅： Oracle支持文档ID 1484775.1。

emremove.sql

emremove.sql脚本删除与Oracle Enterprise Manager相关的模式和对象。使用此脚本手动删除DB Control。在开始升级过程之前运行emremove.sql最大限度地减少停机时间。这是可选的升级前步骤，因为并行升级实用程序和DBUA会自动运行此脚本。

警告 ：如果要保留DB Control配置和数据以选择降级和还原DB Control，则必须首先执行使用emdwgrd的过程。

olspreupgrade.sql

不要在UPGRADE模式下运行此脚本。olspreupgrade.sql脚本是升级数据库所需的预处理脚本，该数据库早于使用Oracle Label Security（OLS）和Oracle Database Vault的Oracle Database 12.1版。

postupgrade_fixups.sql

postupgrade_fixups.sql脚本随Oracle数据库提供。升级后运行postupgrade_fixups.sql 。DBUA自动运行此脚本;但是，您可以在升级后随时运行它。

preupgrd.sql

（在12.2及更高版本中已过时） preupgrd.sql升级前信息工具随Oracle Database 12c第1版（12.1）提供。在升级之前随时运行preupgrd.sql以分析数据库。升级前信息工具提供DBUA检查和生成脚本（称为修复脚本）的项目预览，您可以运行这些脚本来解决源数据库中标记的问题。在版本12.1中， preupgrd.sql脚本替换了utlu121i.sql脚本。）在版本12.2中， preupgrade.jar替换了preupgrd.sql 。

preupgrade.jar

preupgrade.jar升级前信息工具随Oracle Database 12 c第 2版（12.2）提供。preupgrade.jar与preupgrd.sql具有相同的功能， preupgrade.jar替换。

utlrp.sql

使用utlrp.sql重新编译存储的PL / SQL和Java代码。DBUA自动运行此脚本。手动升级时，必须运行此脚本。

utlusts.sql

utlusts.sql升级后状态工具随Oracle数据库一起提供，并显示DBA_REGISTRY每个组件的版本和已用升级时间。升级数据库后，可以随时运行升级后状态工具。

utluptabdata.sql

升级后运行utluptabdata.sql脚本，以在依赖于升级期间更改的Oracle维护类型的任何用户表上执行ALTER TABLE UPGRADE命令。

必须使用为要升级的所有表分配ALTER TABLE系统特权的用户帐户运行utluptabdata.sql脚本，或者使用连接到AS SYSDBA的SYSDBA系统特权的用户帐户运行utluptabdata.sql脚本。

在数据库升级期间，用户表不会自动升级到新版本的类型，因此您可以在用户表空间设置为READ ONLY的情况下运行升级。
