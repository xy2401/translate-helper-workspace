270.1 WPG_DOCLOAD常量

WPG_DOCLOAD定义了在指定参数值时要使用的几个常量。

WPG_DOCLOAD常量如下所示：

  • NAME_COL_LEN

  • MIMET_COL_LEN

  • MAX_DOCTABLE_NAME_LEN

NAME_COL_LEN

文档表中的NAME列必须与name_col_len的值相同。 

name_col_len CONSTANT pls_integer：= 64;

MIMET_COL_LEN

文档表中的MIME_TYPE列必须与mimet_col_len的值相同。 

mimet_col_len CONSTANT pls_integer：= 48;

MAX_DOCTABLE_NAME_LEN

文档表的名称长度必须小于max_doctable_name_len 。 

max_doctable_name_len CONSTANT pls_integer：= 256;

子程序 描述

DOWNLOAD_FILE程序

下载文件， BLOBS和BFILES

270.2.1 DOWNLOAD_FILE程序

此下载文件过程有三个版本。

  • 第一个版本下载文件，并从文档下载过程中调用，以通知PL / SQL网关将p_filename从文档表下载到客户端的浏览器。

  • 可以从任何过程中调用第二个版本，以通知PL / SQL网关将p_blob下载到客户端的浏览器。

  • 可以在任何过程中调用第三个版本，以通知PL / SQL网关将p_bfile下载到客户端的浏览器。

句法

WPG_DOCLOAD.DOWNLOAD_FILE（p_filename IN VARCHAR2，p_bcaching IN BOOLEAN DEFAULT TRUE）;

WPG_DOCLOAD.DOWNLOAD_FILE（p_blob IN OUT NOCOPY BLOB）; WPG_DOCLOAD.DOWNLOAD_FILE（p_bfile IN OUT BFILE）;

参数

表270-2 DOWNLOAD_FILE过程参数

参数 描述

p_filename

要从文档表中下载的文件。

p_blob

BLOB下载。

p_bfile

要下载的BFILE （参见使用说明 ）。

p_bcaching

是否启用了浏览器缓存（请参阅使用说明 ）。

使用说明

  • 通常，文档将被下载到浏览器，除非浏览器向网关发送“If-Modified-Since”标头，指示它在其缓存中具有所请求的文档。在这种情况下，网关将确定浏览器的缓存副本是否是最新的，如果是，它将向浏览器发送HTTP 304状态消息，指示浏览器应显示缓存的副本。但是，由于文档URL和文档在PL / SQL Web Gateway中不一定具有一对一的关系，因此在某些情况下可能不希望显示文档的缓存副本。在这些情况下， p_bcaching参数设置为FALSE以指示网关忽略“If-Modified-Since”标头，并下载文档。

  • p_bfile和p_blob声明为IN OUT因为最初打开定位器以检查文件的可访问性和存在性。只有在定位器可写和可读时才能执行打开操作。
