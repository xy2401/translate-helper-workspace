2.12创建目录

目的

使用CREATE CATALOG命令创建基本恢复目录。将目录权限授予虚拟专用目录所有者时，将自动创建虚拟专用目录。

  • 基本恢复目录是包含一组目标数据库的RMAN元数据的数据库模式。

  • 虚拟专用目录是一组安全策略，用于限制用户对基本恢复目录子集的访问。

也可以看看：

  • Oracle数据库备份和恢复用户指南 ，了解如何创建恢复目录

  • “ RMAN兼容性 ” ，了解恢复目录与RMAN环境的其他组件的兼容性要求

先决条件

仅在RMAN提示符下执行此命令。必须通过CATALOG命令行选项或CONNECT CATALOG命令将RMAN连接到恢复目录数据库，并且必须打开目录数据库。不需要与目标数据库的连接。

必须为基本恢复目录的恢复目录所有者授予RECOVERY_CATALOG_OWNER角色。恢复目录在恢复目录所有者的默认表空间中创建。

注意：

从Oracle Database 12c第1版（12.1.0.2）开始，恢复目录数据库必须使用Enterprise Edition。

拥有虚拟专用目录的数据库用户必须被授予CREATE SESSION特权。从Oracle Database 12 c第1版（12.1.0.2）开始，只有在使用Oracle数据库企业版时才能创建虚拟专用目录。

如果要创建虚拟专用目录，则基本恢复目录所有者必须使用RMAN GRANT命令授予其中一个 CATALOG或REGISTER权限（参见例2-71 ）。

有关RMAN客户端来自发行版Oracle数据库10g或更早版本的RMAN客户端与虚拟目录的连接的限制，请参阅CONNECT CATALOG说明。

使用说明

通常，您在专门为此目的创建的数据库中创建恢复目录。不要在特权架构（如SYS或SYSBACKUP创建恢复目录。

最佳做法是创建一个恢复目录，作为许多数据库的中央RMAN存储库。因此，它被称为基本恢复目录 。

基本恢复目录的拥有者可以GRANT或REVOKE到类别到其他数据库用户的受限接入。每个受限用户都对其自己的元数据具有完全读/写访问权限，称为虚拟专用目录 。RMAN元数据存储在虚拟专用目录所有者的模式中。基本恢复目录的所有者控制每个虚拟目录用户可以访问的内容。

例子

示例2-70创建恢复目录并注册数据库

假设您启动SQL * Plus并使用管理员权限连接到恢复目录catdb 。您可以按如下方式执行CREATE USER语句，将密码替换为用户指定的密码（有关创建安全密码的信息，请参阅“ Oracle数据库安全指南” ）。SQL语句在数据库catdb创建用户rco ， catdb rco用户授予RECOVERY_CATALOG_OWNER角色。 

SQL> CREATE USER rco IDENTIFIED BY password 2 DEFAULT TABLESPACE cattbs 3 QUOTA UNLIMITED ON cattbs; SQL> GRANT recovery_catalog_owner TO rco; SQL> EXIT

然后启动RMAN并运行以下RMAN命令以rco身份连接到恢复目录数据库并创建恢复目录：

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> CREATE CATALOG;

在同一个RMAN会话中，使用操作系统身份验证连接到目标数据库，并使用REGISTER DATABASE命令在目录中注册此数据库：

RMAN> CONNECT TARGET / RMAN>注册数据库; RMAN>退出

示例2-71创建虚拟专用目录

假设您创建了恢复目录并注册了数据库，如例2-70所示。现在，您要为数据库用户vpc1创建虚拟专用目录。数据库版本是Oracle Database 12 c第1版（12.1.0.2）。启动SQL * Plus并使用管理员权限连接到恢复目录数据库catdb 。您创建vpc1用户并授予恢复目录的所有权如下，与用户指定的密码（请参阅更换密码 Oracle数据库安全指南的信息，创建安全密码）：

SQL> CREATE USER vpc1 IDENTIFIED BY password 2 DEFAULT TABLESPACE vpcusers; SQL> GRANT创建会话到vpc1; SQL> EXIT

然后，您启动RMAN并以目录所有者rco身份连接到恢复目录数据库。默认情况下，虚拟目录所有者无权访问基本恢复目录。您可以使用GRANT命令授予虚拟专用目录访问vpc1对数据库RMAN操作prod1 （但不是prod2 ）：

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> GRANT CATALOG FOR DATABASE prod1 TO vpc1; RMAN>退出;

现在，将使用虚拟专用目录vpc1的备份操作员已准备好创建虚拟目录。在以下示例中，备份操作员以vpc1连接到恢复目录数据库，并使用vpc1注册数据库vpc1 。

将目录权限授予虚拟专用目录所有者时，将自动创建虚拟专用目录。

RMAN> CONNECT CATALOG vpc1 @catdb恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> REGISTER DATABASE prod1; RMAN>退出;
