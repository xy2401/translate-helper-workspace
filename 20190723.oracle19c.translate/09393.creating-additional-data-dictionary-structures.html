B.2创建附加数据字典结构

Oracle数据库提供了其他脚本，可以创建可用于管理数据库和创建数据库应用程序的其他结构。 表B-2中列出了这些脚本。

utlexpt1.sql

约束

任何用户

用于Oracle数据库。创建用于存储启用约束的异常的默认表（ EXCEPTIONS ）。可以处理物理和逻辑rowid。

另请参阅： Oracle数据库管理员指南

utlfixdirs.sql 脚1

将数据库移动到新的Oracle主目录

SYS

将数据库移动到新的Oracle主目录后使用。更新目录对象以使用Oracle主目录和Oracle基目录的新路径名，由ORACLE_HOME和ORACLE_BASE环境变量的新值定义。

当您在CDB的根容器中运行此脚本时，它会更新根目录中的目录对象，以及PDB中任何Oracle维护的目录对象;您必须手动更新任何非Oracle维护的PDB目录对象。

utlip.sql

PL / SQL

SYS

主要用于升级和降级操作。它通过更改某些字典表使所有现有PL / SQL模块无效，以便随后的重新编译将以数据库所需的格式进行。它还重新加载STANDARD和DBMS_STANDARD软件包，它们是任何PL / SQL编译所必需的。

utlirp.sql

PL / SQL

SYS

用于从32位更改为64位字大小，反之亦然。此脚本以新数据库所需的格式重新编译现有PL / SQL模块。它首先改变一些数据字典表。然后重新加载使用PL / SQL所必需的软件包STANDARD和DBMS_STANDARD 。最后，它触发重新编译所有PL / SQL模块，例如包，过程和类型。

utllockt.sql

性能监控

SYS或SYSDBA

以树结构格式显示锁等待图

另请参阅： Oracle数据库管理员指南

utlpwdmg.sql

安全

SYS或SYSDBA

创建PL / SQL函数以进行默认密码复杂性验证。设置默认密码配置文件参数并启用密码管理功能。

另请参阅： Oracle数据库安全指南

utlrp.sql

PL / SQL

SYS

重新编译以前处于INVALID状态的所有现有PL / SQL模块，例如包，过程和类型。
