关于在Oracle RAC环境中使用克隆添加节点

克隆过程假定您已成功安装并配置了要添加节点和实例的Oracle RAC环境。要使用克隆将节点添加到Oracle RAC环境，首先扩展Oracle Clusterware配置，然后使用Oracle RAC扩展Oracle数据库软件，然后通过运行Oracle助手添加侦听器和实例

克隆脚本运行多个工具，每个工具都可以生成自己的日志文件。clone.pl脚本完成运行后，您可以查看日志文件以获取有关克隆过程的更多信息。有关详细信息，请参阅“ 查找和查看克隆过程中生成的日志文件 ” 。

克隆Linux和UNIX系统上的本地Oracle主目录

通过在Linux和UNIX系统环境中克隆本地（非共享）Oracle主目录，将节点添加到现有Oracle RAC环境。

完成以下步骤以使用Oracle RAC软件克隆Oracle数据库：

  1. 按照“准备克隆Oracle RAC”中的步骤创建Oracle主目录的副本，然后使用该副本在一个或多个节点上执行克隆过程。

  2. 使用tar实用程序在现有节点上创建Oracle主目录的存档，并将其复制到新节点。如果源节点上Oracle主目录的位置为$ORACLE_HOME ，则必须将此目录用作新节点上的目标位置。

  3. 在新节点上，配置环境变量ORACLE_HOME和ORACLE_BASE 。然后转到$ORACLE_HOME/clone/bin目录并运行以下命令，其中existing_node是要克隆的节点的名称， new_node2和new_node3是新节点的名称， Oracle_home_name是Oracle主Oracle_home_name的名称：

    perl clone.pl -O'CLUSTER_NODES = { existing_node,new_node2 ， new_node3 }'-O LOCAL_NODE = new_node2 ORACLE_BASE = $ ORACLE_BASE ORACLE_HOME = $ ORACLE_HOME ORACLE_HOME_NAME = Oracle_home_name -O -noConfig

  4. 运行以下命令以运行配置助手以在新节点上配置Oracle RAC：

    $ ORACLE_HOME / cfgtoollogs / configToolFailedCommands

    此脚本包含在安装期间失败，跳过或取消的所有命令。您可以使用此脚本在Oracle Universal Installer之外运行数据库配置助手。请注意，在运行脚本之前，您应该检查脚本以查看其中是否需要更新其中的任何密码。

  5. 在$ORACLE_HOME/oui/bin目录的现有节点上运行以下命令，以使用Oracle_home指定的Oracle RAC更新Oracle数据库主目录中的清单，其中existing_node是要克隆的原始节点的名称， new_node2和new_node3是新节点的名称：

    ./runInstaller -updateNodeList ORACLE_HOME = $ ORACLE_HOME -O“CLUSTER_NODES = { existing_node ， new_node2,new_node3 }”

  6. 在每个新节点上，转到$ORACLE_HOME目录并运行以下命令：

    ./root.sh

  7. 从您克隆的节点，运行Database Configuration Assistant（DBCA）以在新节点上添加Oracle RAC数据库实例。

克隆Linux和UNIX系统上的共享Oracle主目录

通过在Linux和UNIX系统环境中克隆共享Oracle主目录，将节点添加到现有Oracle RAC环境。

完成以下步骤以使用Oracle RAC软件克隆Oracle数据库：

  1. 按照“准备克隆Oracle RAC”中的步骤创建Oracle主目录的副本，然后使用该副本在一个或多个节点上执行克隆过程。

  2. 在新节点上，配置环境变量ORACLE_HOME和ORACLE_BASE 。然后转到$ORACLE_HOME/clone/bin目录并运行以下命令，其中existing_node是要克隆的节点的名称， new_node2和new_node3是新节点的名称， Oracle_home_name是Oracle主Oracle_home_name的名称， -cfs选项表示共享Oracle主目录：

    perl clone.pl -O'CLUSTER_NODES = { existing_node ， new_node2 ， new_node3 }'-O LOCAL_NODE = new_node2 ORACLE_BASE = $ ORACLE_BASE ORACLE_HOME = $ ORACLE_HOME ORACLE_HOME_NAME = Oracle_home_name [-cfs]

    注意：

    在上面的命令中：

    • 对Oracle RAC的共享Oracle数据库主目录使用-cfs选项。

    • ORACLE_HOME_NAME参数的值必须是要克隆的节点的值。

  3. 在$ORACLE_HOME/oui/bin目录的现有节点上运行以下命令，以使用Oracle_home指定的Oracle RAC更新Oracle数据库主目录中的清单，其中existing_node是要克隆的原始节点的名称， new_node2和new_node3是新节点的名称：

    ./runInstaller -updateNodeList ORACLE_HOME = $ ORACLE_HOME“CLUSTER_NODES = { existing_node ， new_node2,new_node3 }”

  4. 在每个新节点上，转到$ORACLE_HOME目录并运行以下命令：

    ./root.sh

  5. 从您克隆的节点，运行Database Configuration Assistant（DBCA）以将Oracle RAC数据库实例添加到新节点。

在Windows系统上克隆Oracle Homes

通过在Windows系统环境中克隆共享或本地Oracle主目录，将节点添加到现有Oracle RAC环境。

完成以下步骤以使用Oracle RAC软件克隆Oracle数据库：

  1. 如果您有本地Oracle主目录，则使用ZIP实用程序在现有节点上使用Oracle RAC创建Oracle数据库主目录的存档，并将其复制到新节点。否则，请继续执行下一步。

    使用Oracle RAC中的Oracle RAC主文件从Oracle RAC所在的Oracle数据库主目录所在的同一目录中的新节点上的ZIP文件中提取Oracle数据库。例如，假设新节点上目标Oracle RAC主目录的位置为%ORACLE_HOME% 。

  2. 在新节点上，转到%ORACLE_HOME%\clone\bin目录并运行以下命令，其中Oracle_Home是Oracle数据库主目录， Oracle_Home_Name是Oracle数据库主目录的名称， Oracle_Base是Oracle基本目录， user_name是正在克隆的Oracle主目录的Oracle主目录用户（非管理员用户）的名称， existing_node是现有节点的名称， new_node是新节点的名称：

    perl clone.pl ORACLE_HOME = Oracle_Home ORACLE_BASE = Oracle_Base ORACLE_HOME_NAME = Oracle_Home_Name ORACLE_HOME_USER = user_name new_node = { existing_node ， new_node }'-O LOCAL_NODE = new_node

    如果您有一个带有Oracle RAC的共享Oracle数据库主目录，则将-cfs选项附加到该命令以指示Oracle主目录是共享的，如以下示例所示：

    perl clone.pl ORACLE_HOME = Oracle_Home ORACLE_BASE = Oracle_Base ORACLE_HOME_NAME = Oracle_Home_Name ORACLE_HOME_USER = user_name new_node = { existing_node ， new_node }'-O LOCAL_NODE = new_node [-cfs -noConfig]

    注意：

    • 只有在克隆安全的Oracle主目录时才需要ORACLE_HOME_USER 。

    • 对Oracle RAC的共享Oracle数据库主-noConfig使用-cfs和-noConfig选项。

    • ORACLE_HOME_NAME参数的值必须是要克隆的节点的值。要获取ORACLE_HOME_NAME ，请在HKEY_LOCAL_MACHINE\SOFTWARE\oracle\KEY_OraCRs12c_home1下的ORACLE_HOME_NAME参数键的克隆节点上的注册表中HKEY_LOCAL_MACHINE\SOFTWARE\oracle\KEY_OraCRs12c_home1 。

  3. 在现有节点上，从%ORACLE_HOME%\oui\bin目录运行以下命令，以使用Oracle_home指定的Oracle RAC更新Oracle数据库主目录中的清单，其中existing_node是现有节点的名称， new_node是新节点的名称：

    setup.exe -updateNodeList ORACLE_HOME = Oracle_home “CLUSTER_NODES = { existing_node ， new_node }”LOCAL_NODE = existing_node

  4. 从您克隆的节点，运行DBCA以将Oracle RAC数据库实例添加到新节点。
