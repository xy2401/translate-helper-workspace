关于Oracle GoldenGate和在线数据库升级

使用Oracle GoldenGate复制可以使您在线执行升级，从而简化升级。

在Oracle GoldenGate复制环境中，您可以执行在线数据库升级到当前版本的Oracle数据库。使用Oracle GoldenGate复制环境可以最大限度地减少升级期间的数据库停机时间。除了平台迁移之外，Oracle GoldenGate是一种极好的方法，可以最大限度地减少计划内维护期间的停机时间，包括应用程序和数据库升级。Oracle GoldenGate是独立于Oracle Database for Oracle和第三方数据库管理系统销售的Oracle产品。

使用Oracle GoldenGate升级Oracle数据库的步骤概述

查看这些步骤以了解如何使用Oracle GoldenGate升级Oracle数据库。

使用Oracle GoldenGate升级到新的Oracle数据库版本包括以下高级步骤。

  1. 使用现有数据库备份设置运行早期数据库软件版本的备用数据库。

  2. 将备用数据库升级到新的Oracle数据库版本。

  3. 将备用数据库与生产数据库同步。

  4. 以主动/实时模式测试您的环境。

  5. 将应用程序切换到备用数据库。

  6. 在备用数据库上对新版本进行全面测试。

  7. 将主数据库升级到新的Oracle数据库版本
