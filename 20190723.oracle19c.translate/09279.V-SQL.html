9.45 V $ SQL

V$SQL列出了没有GROUP BY子句的共享SQL区域的统计信息，并且为输入的原始SQL文本的每个子项包含一行。 V$SQL中显示的统计信息通常在查询执行结束时更新。但是，对于长时间运行的查询，它们每5秒更新一次。这样可以很容易地看到长时间运行的SQL语句仍在进行中的影响。

数

  • 1 ：此值用于包含仅与根有关的数据的行

IS_REOPTIMIZABLE

VARCHAR2（1）

此列显示匹配此子游标的下一个执行是否将触发重新优化。价值观是：

  • Y ：如果下一次执行将触发重新优化

  • R ：如果子光标包含重新优化信息，但由于光标是在报告模式下编译的，因此不会触发重新优化

  • N ：如果子光标没有重新优化信息

IS_RESOLVED_ADAPTIVE_PLAN

VARCHAR2（1）

此列显示计划的所有自适应部分是否已解析为最终计划。解决方案后， DBMS_XPLAN显示的计划哈希值和计划在执行结束时不会更改。此列的值为：

  • NULL ：如果计划不适应

  • Y ：如果计划完全解决了

  • N ：如果计划尚未完全解决

另请参阅： Oracle数据库PL / SQL包和类型参考 ，以获取有关DBMS_XPLAN包的更多信息

DDL_NO_INVALIDATE

VARCHAR2（1）

指示DDL语句是否更新了依赖对象并且未使此游标无效。价值观是：

  • N ：没有一个DDL语句更新了一个依赖对象而没有使这个游标失效。

  • Y ：DDL语句更新了一个依赖对象并且没有使该游标无效，但是由于发生这种情况，游标没有执行。

  • X ：DDL语句更新了一个依赖对象，并没有使该游标失效，并且自从发生这种情况后游标已执行。

IS_ROLLING_INVALID

VARCHAR2（1）

指示此游标是否已滚动验证。价值观是：

  • N ：此游标不滚动无效。

  • Y ：此游标滚动无效，但游标未在此状态下执行。

  • X ：此游标滚动无效，并且游标已在此状态下执行。

IS_ROLLING_REFRESH_INVALID

VARCHAR2（1）

指示此游标是否已滚动验证并且需要执行时间刷新。价值观是：

  • N ：该游标不是滚动无效的游标，需要刷新执行时间。

  • Y ：此游标滚动无效并需要执行时刷新，但游标未在此状态下执行。

  • X ：此游标滚动无效并需要执行时刷新，并且游标已在此状态下执行。
