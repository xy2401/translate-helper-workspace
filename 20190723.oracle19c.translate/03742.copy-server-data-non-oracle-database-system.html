将数据从Oracle数据库服务器复制到非Oracle数据库系统

异构服务支持回调链接。

这样可以执行以下SQL语句：

INSERT INTO table_name @ dblink SELECT column_list FROM table_name ;

尽管异构服务支持回调功能，但并非所有网关都实现了它。如果您使用的网关尚未实现此功能，则前面的INSERT语句将返回以下错误消息：

ORA-02025：SQL语句中的所有表必须位于远程数据库中

有关支持回调链接的信息，请参阅网关文档

对于不支持回调链接的网关，可以使用SQL * Plus COPY命令。语法如下：

COPY FROM username @ db_name   -  INSERT destination_table -  USING 查询 ;

以下示例选择本地Oracle emp表中的所有行，将它们插入到非Oracle数据库的emp表中，并提交事务：

从SCOTT @ inst1复制 -  INSERT EMP @ remote_db  - 使用SELECT * FROM EMP;

COPY命令支持APPEND ， CREATE ， INSERT和REPLACE选项。但是， INSERT是复制到非Oracle数据库时唯一受支持的选项。SQL * Plus COPY命令不支持复制到具有小写表名的表。将以下PL / SQL语法与小写表名一起使用：

DECLARE v1 oracle_table.column1 ％TYPE; v2 oracle_table.column2 ％TYPE; v3 oracle_table.column3 ％TYPE; 。。。CURSOR cursor_name IS SELECT * FROM oracle_table ; BEGIN OPEN cursor_name ; LOOP FETCH cursor_name INTO v1，v2，v3， ...;退出时， cursor_name ％NOTFOUND; INSERT INTO destination_table VALUES（ v1，v2，v3，... ）;结束循环; CLOSE cursor_name ;结束;
