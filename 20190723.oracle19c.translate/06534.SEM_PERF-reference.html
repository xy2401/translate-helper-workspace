12 SEM_PERF封装子程序

SEM_PERF包中包含用于检查和增强Oracle数据库中资源描述框架（RDF）和Web本体语言（OWL）支持的性能的子程序。

要使用本章中的子程序，您必须了解RDF语义图概述和OWL概念中的概念和用法信息。

本章提供有关子程序的参考信息，按字母顺序列出。

  • SEM_PERF.DELETE_NETWORK_STATS
  • SEM_PERF.DROP_EXTENDED_STATS
  • SEM_PERF.EXPORT_NETWORK_STATS
  • SEM_PERF.GATHER_STATS
  • SEM_PERF.IMPORT_NETWORK_STATS

12.1 SEM_PERF.DELETE_NETWORK_STATS

格式

SEM_PERF.DELETE_NETWORK_STATS（cascade_parts IN BOOLEAN DEFAULT TRUE，cascade_columns IN BOOLEAN DEFAULT TRUE，cascade_indexes IN BOOLEAN DEFAULT TRUE，no_invalidate IN BOOLEAN DEFAULT DBMS_STATS.AUTO_INVALIDATE，force IN BOOLEAN DEFAULT FALSE，options IN VARCHAR2 DEFAULT NULL），network_owner IN VARCHAR2 DEFAULT'MDSYS' ，network_name IN VARCHAR2 DEFAULT NULL）;

描述

删除语义网络的统计信息。

参数

选项

控制操作范围：

  • 如果MDSYS.SDO_RDF.VALUE_TABLE_ONLY ，则该操作仅适用于MDSYS.RDF_VALUE $表。

  • 如果MDSYS.SDO_RDF.LINK_TABLE_ONLY ，则该操作仅适用于MDSYS.RDF_LINK $表。

  • 如果为null（缺省值），则该操作适用于MDSYS.RDF_VALUE $和MDSYS.RDF_LINK $表。

（其他参数）

请参阅Oracle数据库PL / SQL包和类型参考中的DBMS_STATS.DELETE_TABLE_STATS过程的参数说明，尽管此处force适用于网络统计信息。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

请参阅Oracle Database PL / SQL包和类型参考中有关DBMS_STATS包的信息。

另请参阅管理语义模型和语义网络的统计信息 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例删除语义网络的统计信息：

EXECUTE SEM_APIS.DELETE_NETWORK_STATS;

12.4 SEM_PERF.GATHER_STATS

格式

SEM_PERF.GATHER_STATS（just_on_values_table IN BOOLEAN DEFAULT FALSE，degree IN NUMBER（38）DEFAULT NULL，estimate_percent IN NUMBER DEFAULT DBMS_STATS.AUTO_SAMPLE_SIZE，value_method_opt IN VARCHAR2 DEFAULT NULL，link_method_opt IN VARCHAR2 DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

收集有关RDF和OWL表及其索引的统计信息。

参数

just_on_values_table

TRUE仅在包含三元组词汇值的表上收集统计信息; FALSE （默认值）收集与RDF和OWL数据存储相关的所有主要表的统计信息。

值TRUE减少了过程的执行时间;如果您只需要收集值表的统计信息（例如，如果您使用其他接口来收集您可能需要的任何其他统计信息），则可能就足够了。

度

并行度。有关并行执行的更多信息，请参见“ Oracle数据库VLDB和分区指南” 。

estimate_percent

确定MDSYS.RDF_LINK $和MDSYS.RDF_VALUE $中的行的百分比。

有效范围介于0.000001和100之间。您可以使用常量DBMS_STATS.AUTO_SAMPLE_SIZE （缺省值）使Oracle数据库能够确定适当的样本大小以获得最佳统计信息。

value_method_opt

对于MDSYS.RDF_VALUE $表，接受以下任一选项或两者的组合：

  • FOR ALL [INDEXED | HIDDEN] COLUMNS [size_clause]

  • FOR COLUMNS [size clause] column | attribute [size_clause] [，column | attribute [size_clause] ...]

size_clause定义为： size_clause := SIZE {integer | REPEAT | AUTO | SKEWONLY}

column定义为： column := column_name | (extension)

  • integer ：直方图桶的数量。必须在[1，2048]范围内。

  • REPEAT ：仅在已有直方图的列上收集直方图。

  • AUTO ：Oracle数据库根据数据分布和列的工作负载确定要收集直方图的列。

  • SKEWONLY ：Oracle数据库根据列的数据分布确定要收集直方图的列。

  • column_name ：列的名称

  • extension ：可以是（column_name，column_name [，...]）格式的列组或表达式。

通常的默认值是： FOR ALL COLUMNS SIZE 2048

link_method_opt

对于MDSYS.RDF_LINK $表，接受以下任一选项或两者的组合：

  • FOR ALL [INDEXED | HIDDEN] COLUMNS [size_clause]

  • FOR COLUMNS [size clause] column | attribute [size_clause] [，column | attribute [size_clause] ...]

size_clause定义为： size_clause := SIZE {integer | REPEAT | AUTO | SKEWONLY}

column定义为： column := column_name | (extension)

  • integer ：直方图桶的数量。必须在[1,254]范围内。

  • REPEAT ：仅在已有直方图的列上收集直方图。

  • AUTO ：Oracle数据库根据数据分布和列的工作负载确定要收集直方图的列。

  • SKEWONLY ：Oracle数据库根据列的数据分布确定要收集直方图的列。

  • column_name ：列的名称。

  • extension ：可以是(column_name, column_name [, ...])格式的列组或表达式。

通常的默认值是： FOR ALL COLUMNS SIZE AUTO FOR COLUMNS SIZE 2048 P_VALUE_ID CANON_END_NODE_ID START_NODE_ID G_ID (CANON_END_NODE_ID, START_NODE_ID) (P_VALUE_ID, CANON_END_NODE_ID) (P_VALUE_ID, START_NODE_ID)

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

要使用此过程，您必须以具有执行权限的用户身份进行连接。默认情况下，当Spatial和Graph作为Oracle数据库的一部分安装时，只有MDSYS用户可以执行此过程;但是，可以根据需要向用户授予对此过程的执行权限。

此过程收集可帮助您提高推理性能的统计信息，如增强推理性能中所述 。此过程在内部调用DBMS_STATS.GATHER_TABLE_STATS过程以收集有关RDF和OWL的表及其索引的统计信息，并将统计信息存储在Oracle数据库数据字典中。有关使用DBMS_STATS程序包的信息，请参阅Oracle Database PL / SQL程序包和类型参考 。

收集统计信息会占用大量系统资源，因此在不会对基本应用程序和操作产生负面影响时执行此过程。

另请参阅管理语义模型和语义网络的统计信息 。

例子

以下示例收集有关RDF和OWL相关表及其索引的统计信息。

EXECUTE SEM_PERF.GATHER_STATS;
