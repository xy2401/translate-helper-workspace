SRVCTL使用信息

默认情况下，SRVCTL安装在群集中的每个节点上。要使用SRVCTL，请登录节点的操作系统，并以区分大小写的语法输入SRVCTL命令及其参数。

  • 只能使用提供对正在创建或升级到Oracle数据库12c的 Oracle RAC数据库的Oracle数据库12c中的SRVCTL的版本。SRVCTL的版本必须与要管理的对象（侦听器，Oracle ASM实例，Oracle RAC数据库及其实例和服务）的版本相同。要确保版本是同一版本，请从您管理的数据库或对象的Oracle主目录运行SRVCTL命令。

  • SRVCTL不支持在同一对象上并发执行命令。因此，对于每个数据库，服务或其他对象，一次只运行一个SRVCTL命令。

  • 将逗号分隔的列表指定为SRVCTL命令的一部分时，列表中的项之间不应有任何空格。例如：

    srvctl add database -serverpool“serverpool1，serverpool3”

    在Windows环境中指定以逗号分隔的列表时，必须用双引号（ "" ）将列表括起来。您可以在Linux或UNIX环境中将逗号分隔的列表括在双引号中，但它们将被忽略。

  • 如果要输入SRVCTL命令，并且希望在新行上继续输入，则可以使用操作系统延续字符。在Linux中，这是反斜杠（\）符号。

  • 不产生输出的SRVCTL命令是成功的命令。并非所有SRVCTL命令在成功完成后都会返回消息。但是，如果SRVCTL命令失败，则它始终返回错误消息。

  • 您可以将-eval参数与多个SRVCTL命令一起使用。使用此参数时，可以模拟运行命令而无需对系统进行任何更改。SRVCTL返回输出，通知您在运行特定命令时会发生什么。例如，要了解重定位服务器时可能发生的情况：

将命令参数指定为关键字而不是单个字母

Oracle数据库12c中之前，SRVCTL命令行界面使用单字母参数。但是，这会限制可用于SRVCTL命令的唯一参数的数量。Oracle Database 12 c中引入的SRVCTL命令参数是全字而不是单个字母，例如-multicastport和-subdomain 。

要支持向后兼容性，可以使用单字母参数和新关键字参数的混合。使用关键字引入的新参数可以与单字母参数一起使用。

注意：

从Oracle Database 12 c开始 ，不推荐使用单字母参数来支持关键字参数，以避免使用相同的字母来实现不同的功能，具体取决于命令。

您可以通过在-help参数之后添加-compatible参数来获取单字母等效项（如果适用）。

使用SRVCTL帮助

本节包含有关对SRVCTL使用上下文相关帮助的信息。

要查看所有SRVCTL命令的帮助，请从命令行输入：

srvctl -help

要查看命令语法和每个SRVCTL命令的参数列表，请从命令行输入：

srvctl command （或verb ） object （或noun ）-help

当您使用-help请求命令的联机帮助时，SRVCTL会打印每个参数的完整单词。您可以通过在-help参数之后添加-compatible参数来获取单字母等效项（如果适用）。例如：

$ srvctl config database -help -compatible

上面的命令打印srvctl config database命令的用法信息，在适用的情况下，将所有参数列为全字，后跟括号中的单字母等效项。

要查看SRVCTL版本号，请输入：

$ srvctl -version

SRVCTL权限和安全性

要使用SRVCTL更改Oracle RAC数据库配置，请以您要管理的主目录的软件所有者身份登录操作系统。

例如，如果不同的用户安装了Oracle数据库和Oracle Grid Infrastructure， ora_db数据库软件所有者（例如， ora_db ）身份登录以管理数据库并以Oracle Grid Infrastructure软件所有者（例如， ora_asm ）身份ora_asm进行管理Oracle ASM实例。

作为OSDBA操作系统组成员的用户可以启动和停止数据库。要停止和启动Oracle ASM实例，您必须是OSASM操作系统组的成员。

要创建或注册侦听器，Oracle Notification Services和服务等对象，您必须以Oracle主目录的软件所有者身份登录操作系统。您为该Oracle主目录创建或注册的对象将在Oracle主目录的用户帐户下运行。数据库作为运行它们的主目录的数据库安装所有者运行。

要对任何对象执行srvctl add操作，您必须以运行该对象的主目录的Oracle帐户所有者身份登录。

对于某些SRVCTL命令，在Linux和UNIX系统上，您必须以root身份登录，并且在Windows系统上，您必须以具有管理员权限的用户身份登录才能运行它们。在本附录中，这些命令前面是命令示例中的根提示符（＃）。

其他SRVCTL主题

  • 使用 SRVCTL用于管理Oracle提供的资源（如侦听器，实例，磁盘组和网络），以及CRSCTL用于管理Oracle Clusterware及其资源。

    注意：

    Oracle强烈反对使用CRSCTL直接操作Oracle提供的资源（名称以ora开头的资源）。这可能会对群集配置产生负面影响。

  • 虽然您可以取消跑步通过按Control-C键进行SRVCTL命令，可能会通过执行此操作来破坏配置数据。

    强烈建议您不要以这种方式尝试终止SRVCTL。

srvctl command object [ parameters ]

在SRVCTL语法中：

- timeout

为Oracle RAC One Node数据库指定在线数据库重定位超时（以分钟为单位）。默认值为30 。

-role {PRIMARY | PHYSICAL_STANDBY | LOGICAL_STANDBY | SNAPSHOT_STANDBY}

在Oracle Data Guard配置中指定数据库的角色。默认值为PRIMARY 。
-startoption start_options

数据库的启动选项，例如OPEN ， MOUNT和NOMOUNT 。默认值为OPEN 。

笔记：

  • 对于多字启动选项，例如read only和read write ，使用空格分隔单词并用双引号（ "" ）括起来。例如， "read only" 。

  • 在Oracle Data Guard配置中执行切换时，切换后，成为主数据库的备用数据库的-startoption始终设置为OPEN 。

-stoption stop_options

指定数据库的停止选项，例如NORMAL ， TRANSACTIONAL ， IMMEDIATE和ABORT 。
-dbname db_name

如果数据库的名称与-db参数指定的唯一名称不同，请指定该数据库的名称。

-acfspath“ acfs_path_list ”

单个Oracle ACFS路径或以逗号分隔的Oracle ACFS路径列表，用双引号（ "" ）括起来，其中设置了数据库的依赖关系。

使用此参数可以在ORACLE_HOME以外的Oracle ACFS文件系统上创建依赖关系，例如，当数据库在与ORACLE_HOME文件系统不同的文件系统上使用ORACLE_BASE时。

-policy {AUTOMATIC |手动| NORESTART}
指定数据库的管理策略。

  • AUTOMATIC （默认值）：数据库在重新启动数据库主机时自动恢复到先前的运行状态（已启动或已停止）。

  • MANUAL ：重新启动数据库主机时，数据库永远不会自动重新启动。MANUAL设置不会阻止Oracle Clusterware在数据库运行时监视数据库，并在发生故障时重新启动数据库。

  • NORESTART ：与MANUAL设置类似，数据库永远不会在重新启动数据库主机时自动重新启动。但是，即使发生故障， NORESTART设置也不会重新启动数据库。

-serverpool“ server_pool_name ”[ - pqmppool“ pq_pool_name ”]

指定用于控制数据库放置的服务器池的名称。如果未指定此参数，则默认为通用服务器池。

您还可以选择指定数据库要使用的并行查询服务器池的名称。

笔记：

  • 此参数只能与Oracle Clusterware一起使用。您可以将此参数与-node参数一起使用，但服务器池必须具有MAX_SIZE=1且只有一个已配置的服务器（您在-node指定的服务器）。

  • 添加服务器池后，可以使用srvctl add service命令为其分配服务。

-diskgroup“ disk_group_list ”

如果数据库使用Oracle ASM存储，请指定以逗号分隔的Oracle ASM磁盘组列表。
-css_critical {YES |没有}

您可以通过将此参数设置为YES来为服务添加权重。如果群集中的节点发生故障，Oracle Clusterware将以最小的权重驱逐节点，确保关键服务保持可用。

注意：您只能在管理员管理的节点上使用此参数。如果节点成为策略管理，则在某些时候，此参数将不再适用。

-cpucount cpu_count

指定CPU数量。默认值为0。
-memorytarget memory_target

指定要为数据库分配的目标内存（MB）。默认值为0。
-maxmemory max_memory

指定要为资源分配的最大内存（MB）。如果指定-memorytarget但不-maxmemory ，然后-maxmemory将是默认值0。既-maxmemory和-memorytarget只要被验证-memorytarget小于或等于-maxmemory 。

-cpucap cpu_cap

指定1到100之间的百分比，即数据库所需的工作负载CPU的最大利用率。默认值为0。
-defaultnetnum network_number

指定在添加服务时未指定网络号的情况下服务将默认的网络号（整数）。该数字必须与添加网络时指定的-netnum参数的值匹配。

srvctl添加网络

添加静态或动态网络。

如果您的服务器连接到多个网络，则可以使用此命令为Oracle RAC配置其他网络接口，从而允许您在多个公共网络上创建VIP。

句法

srvctl add network [-netnum net_number ] -subnet subnet / netmask [/ if1 [| if2 | ...]][-nettype {static | dhcp | autoconfig | mixed}] [-leaf] [-pingtarget“ ping_target_list ”] [-verbose]

参数

表A-47 srvctl add network命令参数

参数 描述
-netnum net_number

指定网络号。默认值为1。
-subnet subnet / netmask [/ if1 [| if2 | ...]]

定义子网。如果未指定任何接口名称，则网络将使用给定子网上的任何接口。

如果为网络类型指定mixed ，则VIP资源同时使用静态IP地址和动态获取的IP地址，可以是IPv4的DHCP服务器，也可以是IPv6的无状态自动配置。

-叶

如果要创建仅在叶节点上运行且由应用程序VIP使用的应用程序VIP网络资源，请指定-leaf参数。

[-tartarget“ ping_target_list ”]

指定要以ping命名的逗号分隔的IP地址或主机名列表。
-verbose

详细输出。

  • 在Linux和UNIX系统上，您必须以root用户身份登录，并且必须以具有管理员权限的用户身份登录才能运行此命令。

  • Oracle仅支持默认网络的DHCP分配网络，而不支持后续网络。

  • 您还可以使用LISTENER_NETWORKS数据库初始化参数来控制客户端重定向到适当的网络。

使用说明

  • 在对服务属性执行联机更改（例如，故障转移延迟，运行时负载平衡目标等）时，更改仅在下次（重新）启动服务时生效。

  • 修改服务配置以便添加新的首选或可用实例时，不会影响现有服务的运行状态。但是，在发出srvctl start service命令之前，新添加的实例不会自动提供服务。

  • 如果有可用的服务实例，并且修改了服务配置以便删除首选或可用实例，则服务的运行状态可能会发生不可预测的更改：

    • 服务已停止，然后根据新服务配置在某些实例中删除。

    • 该服务可能正在从服务配置中删除的某些实例上运行。

    • 这些服务将重新定位到新服务配置中的下一个免费实例。

由于这些考虑因素，当修改在线服务时，即使没有删除实例，用户也可能在某些实例上遇到短暂的服务中断。或者，用户可能会在从服务中删除的实例上遇到短暂的服务中断。

将可用实例更改为首选实例的示例如下：

$ srvctl修改服务-db crm -service crm -available crm1 -toprefer

以下命令交换首选且可用的实例：

$ srvctl修改服务-db crm -service crm -modifyconfig -preferred“crm1”\ -available“crm2”
