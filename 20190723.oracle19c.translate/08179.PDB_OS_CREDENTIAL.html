1.258 PDB_OS_CREDENTIAL

PDB_OS_CREDENTIAL确定从PDB与操作系统交互时使用的操作系统用户（OS用户）的身份。

属性 描述

参数类型

串

句法

PDB_OS_CREDENTIAL = 凭证

默认值

没有

可修改

没有

可在PDB中修改

是

基本

没有

Oracle RAC

所有实例都应使用相同的值

注意：

从根目录与操作系统交互时，将继续使用Oracle OS用户。

Oracle OS用户通常是具有高权限的用户，不建议使用相同的用户进行每个PDB的操作系统交互。此外，使用相同的OS用户进行来自不同PDB的操作系统交互可能会损害属于给定PDB的数据。

相反，使用由名称指定为PDB_OS_CREDENTIAL参数值的凭证描述的OS用户有助于确保操作系统交互作为功能较弱的用户执行，并提供保护属于一个PDB的数据的能力。用户连接到另一个PDB。凭证是使用DBMS_CREDENTIAL包的CREATE_CREDENTIAL过程创建的对象。

作为凭证中指定的OS用户名完成的操作系统交互包括：

  • 尚未指定操作系统凭据的外部作业

  • 外部表预处理器

  • PL / SQL库执行

可以为CDB中的所有PDB指定此参数，但可以为特定PDB覆盖CDB范围的值，并且只能由具有DBMS_CREDENTIAL PL / SQL程序包和ALTER SYSTEM的EXECUTE特权的公共管理用户进行修改。系统特权。

如果没有为给定PDB的此参数设置值，则在从该PDB与操作系统交互时将继续使用Oracle OS用户。
