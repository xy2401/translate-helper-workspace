G服务器控制（SRVCTL）命令参考

使用服务器控件（SRVCTL）实用程序来管理群集中的各种组件和应用程序。

本附录包含服务器控制命令的参考信息，包括实用程序使用信息和SRVCTL命令的完整列表。

注意：

“ Oracle Real Application Clusters管理和部署指南”中介绍了特定于Oracle RAC管理操作的SRVCTL命令

SRVCTL使用信息

默认情况下，SRVCTL安装在群集中的每个节点上。要使用SRVCTL，请登录节点的操作系统，并以区分大小写的语法输入SRVCTL命令及其参数。

  • 只能使用提供对正在创建或升级到Oracle数据库12c的 Oracle RAC数据库的Oracle数据库12c中的SRVCTL的版本。SRVCTL的版本必须与要管理的对象（侦听器，Oracle ASM实例，Oracle RAC数据库及其实例和服务）的版本相同。要确保版本是同一版本，请从您管理的数据库或对象的Oracle主目录运行SRVCTL命令。

  • SRVCTL不支持在同一对象上并发执行命令。因此，对于每个数据库，服务或其他对象，一次只运行一个SRVCTL命令。

  • 将逗号分隔的列表指定为SRVCTL命令的一部分时，列表中的项之间不应有任何空格。例如：

    srvctl add database -serverpool“serverpool1，serverpool3”

    在Windows环境中指定以逗号分隔的列表时，必须用双引号（ "" ）将列表括起来。您可以在Linux或UNIX环境中将逗号分隔的列表括在双引号中，但它们将被忽略。

  • 如果要输入SRVCTL命令，并且希望在新行上继续输入，则可以使用操作系统延续字符。在Linux中，这是反斜杠（\）符号。

  • 不产生输出的SRVCTL命令是成功的命令。并非所有SRVCTL命令在成功完成后都会返回消息。但是，如果SRVCTL命令失败，则它始终返回错误消息。

  • 您可以将-eval参数与多个SRVCTL命令一起使用。使用此参数时，可以模拟运行命令而无需对系统进行任何更改。SRVCTL返回输出，通知您在运行特定命令时会发生什么。例如，要了解重定位服务器时可能发生的情况：

    $ srvctl重定位服务器-servers“rac1”-eval -serverpool pool2数据库db1将在节点上停止rac1将在节点rac7上启动服务mySrv1将在节点rac1上停止，它将不会在任何节点上运行服务myServ2将在节点上停止rac1将启动在节点rac6上，服务器rac1将从池myPoolX移动到池pool2

    -eval参数可用于以下命令：

    • srvctl添加数据库

    • srvctl添加服务

    • srvctl添加srvpool

    • srvctl修改数据库

    • srvctl修改服务

    • srvctl修改srvpool

    • srvctl重定位服务器

    • srvctl重新定位服务

    • srvctl删除srvpool

    • srvctl启动数据库

    • srvctl启动服务

    • srvctl停止数据库

    • srvctl停止服务

将命令参数指定为关键字而不是单个字母

Oracle数据库12c中之前，SRVCTL命令行界面使用单字母参数。但是，这会限制可用于SRVCTL命令的唯一参数的数量。Oracle Database 12 c中引入的SRVCTL命令参数是全字而不是单个字母，例如-multicastport和-subdomain 。

要支持向后兼容性，可以使用单字母参数和新关键字参数的混合。使用关键字引入的新参数可以与单字母参数一起使用。

注意：

从Oracle Database 12 c开始 ，不推荐使用单字母参数来支持关键字参数，以避免使用相同的字母来实现不同的功能，具体取决于命令。

您可以通过在-help参数之后添加-compatible参数来获取单字母等效项（如果适用）。

使用SRVCTL帮助

本节包含有关对SRVCTL使用上下文相关帮助的信息。

要查看所有SRVCTL命令的帮助，请从命令行输入：

srvctl -help

要查看命令语法和每个SRVCTL命令的参数列表，请从命令行输入：

srvctl command （或verb ） object （或noun ）-help

当您使用-help请求命令的联机帮助时，SRVCTL会打印每个参数的完整单词。您可以通过在-help参数之后添加-compatible参数来获取单字母等效项（如果适用）。例如：

$ srvctl config database -help -compatible

上面的命令打印srvctl config database命令的用法信息，在适用的情况下，将所有参数列为全字，后跟括号中的单字母等效项。

要查看SRVCTL版本号，请输入：

$ srvctl -version

SRVCTL权限和安全性

要使用SRVCTL更改Oracle RAC数据库配置，请以您要管理的主目录的软件所有者身份登录操作系统。

例如，如果不同的用户安装了Oracle数据库和Oracle Grid Infrastructure， ora_db数据库软件所有者（例如， ora_db ）身份登录以管理数据库并以Oracle Grid Infrastructure软件所有者（例如， ora_asm ）身份ora_asm进行管理Oracle ASM实例。

作为OSDBA操作系统组成员的用户可以启动和停止数据库。要停止和启动Oracle ASM实例，您必须是OSASM操作系统组的成员。

要创建或注册侦听器，Oracle Notification Services和服务等对象，您必须以Oracle主目录的软件所有者身份登录操作系统。您为该Oracle主目录创建或注册的对象将在Oracle主目录的用户帐户下运行。数据库作为运行它们的主目录的数据库安装所有者运行。

要对任何对象执行srvctl add操作，您必须以运行该对象的主目录的Oracle帐户所有者身份登录。

对于某些SRVCTL命令，在Linux和UNIX系统上，您必须以root身份登录，并且在Windows系统上，您必须以具有管理员权限的用户身份登录才能运行它们。在本附录中，这些命令前面是命令示例中的根提示符（＃）。

其他SRVCTL主题

  • 使用 SRVCTL用于管理Oracle提供的资源（如侦听器，实例，磁盘组和网络），以及CRSCTL用于管理Oracle Clusterware及其资源。

    注意：

    Oracle强烈反对使用CRSCTL直接操作Oracle提供的资源（名称以ora开头的资源）。这可能会对群集配置产生负面影响。

  • 虽然您可以取消跑步通过按Control-C键进行SRVCTL命令，可能会通过执行此操作来破坏配置数据。

    强烈建议您不要以这种方式尝试终止SRVCTL。

SRVCTL使用以下命令语法：

srvctl command object [ parameters ]

在SRVCTL语法中：

srvctl禁用asm

禁用Oracle ASM代理资源。

如果Oracle ASM包含Oracle Clusterware数据，或者节点重新启动并且在节点发生故障之前运行，则ASM将重新启动。srvctl disable asm命令还会阻止srvctl start asm命令启动Oracle ASM实例。

禁用Oracle ASM代理资源可防止数据库和磁盘组启动，因为它们依赖于Oracle ASM代理资源。

句法

srvctl disable asm [-proxy] [-node node_name ]

参数

表G-7 srvctl disable asm命令参数

参数 描述
-代理

（可选）您可以使用此参数指定是否禁用代理Oracle ASM配置。
-node node_name

（可选）您可以指定要禁用Oracle ASM的特定节点。

使用说明

  • 要在Oracle Database 12 c安装上管理Oracle ASM，请在Oracle Grid Infrastructure主目录中使用SRVCTL二进制文件作为集群（Grid home）。如果安装了Oracle RAC或Oracle数据库，则无法使用数据库主目录中的SRVCTL二进制文件来管理Oracle ASM。

  • Oracle ASM是Oracle Clusterware堆栈的一部分，当OCR和投票文件存储在Oracle ASM上时，Oracle ASM在OHASD启动Oracle Clusterware堆栈时启动。srvctl disable asm命令不会阻止OHASD管理的Oracle ASM实例启动。因此，Oracle ASM根据Oracle Clusterware堆栈的需要启动。

    srvctl disable asm命令可防止Oracle ASM Oracle Clusterware代理资源以及依赖它的任何资源启动。因此，该命令可防止Oracle Clusterware管理的对象（如数据库，磁盘组和依赖于Oracle ASM的文件系统）启动。例如， srvctl start database | diskgroup | filesystem命令无法在禁用Oracle ASM Oracle Clusterware代理资源的节点上启动任何这些对象。该命令还会阻止srvctl start asm命令在远程节点上启动Oracle ASM。

例

-path mountpoint_path
