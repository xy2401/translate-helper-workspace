关于使用Oracle Fleet补丁和配置部署Oracle Grid Infrastructure

Oracle Fleet Patching and Provisioning（Oracle FPP）是一种用于配置和维护Oracle主目录的软件生命周期管理方法。Oracle Fleet Patching and Provisioning支持大规模部署和维护数据库，集群和用户定义的软件类型的标准操作环境。

注意：

从Oracle Grid Infrastructure 19c开始，以前称为Rapid Home Provisioning（RHP）的功能现在是Oracle Fleet Patching and Provisioning（Oracle FPP）。

通过Oracle Fleet Patching和Provisioning，您可以安装集群，配置，修补，扩展和升级Oracle Grid Infrastructure，Oracle Restart和Oracle数据库主目录。支持的版本是11.2,12.1,12.2,18c和19c。您还可以使用Oracle Fleet Patching和Provisioning配置应用程序和中间件。

Oracle Fleet Patching and Provisioning是Oracle Grid Infrastructure中的一项服务，您可以使用以下任一模式：

  • 中央Oracle Fleet补丁和配置服务器

    Oracle Fleet Patching和Provisioning Server存储和管理标准化图像，称为黄金映像。黄金映像可以部署到数据中心的任意数量的节点。您可以在已部署的主目录上创建新的群集和数据库，并可以使用它们来修补，升级和扩展现有安装。

    Oracle Fleet Patching和Provisioning Server可以管理以下类型的安装：

    • 托管Oracle Fleet Patching和Provisioning Server本身的集群上的软件之家。

    • 运行Oracle Grid Infrastructure 12c第2版（12.2），18c和19c的Oracle Fleet补丁和配置客户端。

    • 运行Oracle Grid Infrastructure 11g第2版（11.2）和12c第1版（12.1）的安装。

    • 没有Oracle Grid Infrastructure的安装。

    Oracle Fleet Patching和Provisioning Server可以配置新安装，并且可以管理现有安装，而无需对现有安装进行任何更改。Oracle Fleet Patching和Provisioning Server可以在对等服务器之间自动共享黄金映像，以支持具有地理位置分散的数据中心的企业。

  • Oracle Fleet补丁和配置客户端

    可以从Oracle Fleet Patching和Provisioning Server管理Oracle Fleet Patching和Provisioning Client，也可以直接在客户端本身上执行命令。Oracle Fleet Patching and Provisioning Client是Oracle Grid Infrastructure中内置的一项服务，可在Oracle Grid Infrastructure 12c第2版（12.2）及更高版本中使用。Oracle Fleet Patching和Provisioning Client可以从Oracle Fleet Patching和Provisioning Server检索黄金映像，根据策略上载新映像，并将维护操作应用于自身。

Oracle Fleet补丁和配置

使用Oracle Fleet Patching and Provisioning部署Oracle软件具有以下优势：

  • 通过黄金映像和已部署软件的管理系列确保标准化并实现高度自动化。

  • 通过将新住宅作为图像（称为黄金映像）部署在一起，最大限度地减少停机时间，而不会中断活动数据库或集群。

  • 通过提供跨数据库版本和部署模型使用简单，一致的API调用的自动机来简化维护。

  • 通过内置验证和“干运行”模式降低维护风险，以测试操作。

  • 使您能够在出现无法预料的问题时恢复或重新启动命令，从而降低维护操作的风险。

  • 最大限度地减少并经常消除修补和升级的影响，其功能包括：

    • 零停机数据库升级，全自动升级，完全在部署中执行，无需任何额外节点或外部存储。

    • 滚动修补期间数据库会话和OJVM的自适应管理。

    • 用于管理整合部署的选项。

  • 部署和维护操作使自定义能够将特定于环境的操作包括在自动化工作流中。
