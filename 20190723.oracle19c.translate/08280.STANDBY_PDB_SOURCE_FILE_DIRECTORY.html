1.329 STANDBY_PDB_SOURCE_FILE_DIRECTORY

STANDBY_PDB_SOURCE_FILE_DIRECTORY指定备用数据库上的目录位置，其中可以找到用于实例化备用PDB的源数据文件。

属性 描述

参数类型

串

句法

STANDBY_PDB_SOURCE_FILE_DIRECTORY = datafiles-directory

默认值

空值

可修改

改变系统

可在PDB中修改

没有

基本

没有

Oracle RAC

可以在不同的实例上设置不同的值。

在Oracle Database 18c之前的Oracle数据库版本中，如果将PDB插入CDB，则在备用数据库上，数据文件应位于PDB的Oracle托管文件（OMF）目录位置。如果在那里找不到它们，则用户必须将数据文件复制到OMF位置，然后在备用数据库上重新启动重做应用。

为解决该缺陷， STANDBY_PDB_SOURCE_FILE_DIRECTORY指定备用数据库上的目录位置，其中可以找到用于实例化PDB的源数据文件。如果在那里找不到数据文件，将尝试将它们定位在备用数据库的OMF位置。

此参数还可用于在备用数据库中不存在源PDB文件的情况下尝试复制文件。例如，可以使用standbys = NONE创建源PDB。在这种情况下， STANDBY_PDB_SOURCE_FILE_DIRECTORY参数可以使备用数据库从某个位置复制文件（如果它们可用）。
