27.5.1 OCIObjectCopy（）

将源实例复制到目标。

目的

将源实例复制到目标。

句法

sword OCIObjectCopy（OCIEnv * env，OCIError * err，const OCISvcCtx * svc，void * source，void * null_source，void * target，void * null_target，OCIType * tdo，OCIDuration duration，ub1 option）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

svc（IN）

OCI服务上下文句柄，指定正在进行复制操作的服务上下文。

来源（IN）

指向源实例的指针;如果它是一个物体，它必须固定。

也可以看看：

OCIObjectPin（）

null_source（IN）

指向源对象的NULL结构的指针。

目标（IN）

指向目标实例的指针;如果它是一个物体，它必须固定。

null_target（IN）

指向目标对象的NULL结构的指针。

tdo（IN）

源和目标的TDO。可以使用OCIDescribeAny()检索。

持续时间（IN）

目标内存的分配持续时间。

选项（IN）

此参数当前未使用。传递为零或OCI_DEFAULT 。

评论

此函数将source实例的内容复制到target实例。此函数执行深层复制，以便复制以下所有信息：

  • 所有顶级属性（稍后参见例外）

  • 所有辅助内存（源代码）都可以从顶级属性访问

  • 实例的NULL结构

使用duration参数中指定的duration分配内存。

某些数据项不会被复制，例如，如果属性是内部LOB，则只复制源对象中的LOB定位符。在调用OCIObjectFlush()之前，不会复制LOB数据。在刷新目标对象之前，源和目标定位器都引用相同的LOB值。

必须已创建目标或目标的包含实例。这可以使用OCIObjectNew()或OCIObjectPin()来完成，具体取决于目标对象是否存在。

source实例和target实例必须属于同一类型。如果源和目标位于不同的数据库中，则两个数据库中必须存在相同的类型。

null_struct（IN）

对象或数组的NULL结构。

tdo（IN）

index_count（IN）[可选]

目前不支持。传递为(ub4)0 。

如果属性类型是原始的，则属性的NULL状态。

attr_null_struct（OUT）

此参数仅针对对象和不透明属性填充，而不针对集合填充。对于集合（传递OCICollGetElem ）， attr_null_struct为NULL 。对于集合，此参数指示整个集合是否为NULL 。

attr_value（OUT）

指向属性值的指针。看到

表24-1列出了此调用成功完成后，参数attr_value可以找到的数据类型和C数据类型。
attr_tdo（OUT）

指向属性的TDO的指针。

评论

此函数从对象获取值。返回的属性的位置被指定为路径表达式，它是名称数组和索引数组。如果参数instance指向对象，则路径表达式指定对象中属性的位置。假设对象已固定，并且返回的值有效，直到取消固定对象为止。返回的值被认为是只读的，不应以任何方式进行修改。

如果attr_null_status和attr_null_struct都为NULL ，则不返回NULL信息。

长度= {8,2,3,4}

indices = {5}

27.5.7 OCIObjectLockNoWait（）

锁定服务器上的持久对象但不等待锁定。

目的

锁定服务器上的持久对象但不等待锁定。如果锁不可用，则返回错误。

句法

sword OCIObjectLockNoWait（OCIEnv * env，OCIError * err，void * object）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

对象（IN）

指向被锁定的持久对象的指针。它必须已固定。

评论

此函数锁定服务器上的持久对象。但是，与OCIObjectLock()不同，如果另一个用户持有对象的锁定，则此函数不会等待，如果该对象当前被另一个用户锁定，则会返回错误。此函数还会为瞬态对象和值或不存在的对象返回错误。

在事务结束时释放对象的锁定。

OCIObjectLockNoWait()返回以下值：

  • OCI_INVALID_HANDLE ，如果环境句柄或错误句柄为NULL

  • OCI_SUCCESS ，如果操作成功

  • OCI_ERROR ，如果操作失败

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。句柄可以UTF-16（Unicode）模式初始化。请参阅OCIEnvNlsCreate()的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

svc（IN）

OCI服务处理。

typecode（IN）

实例类型的类型代码。

也可以看看：

的TypeCodes

tdo（IN）[可选]

指向类型描述符对象的指针。TDO描述了要创建的实例的类型。请参阅OCITypeByName()以获取TDO。创建命名类型（例如对象或集合）需要TDO。

表（IN）[可选]

指向表对象的指针，该表对象指定服务器中的表。如果没有给出表，则可以This参数设置为NULL 。请参阅以下描述以了解如何一起使用表对象和TDO来确定要创建的实例类型（持久性，瞬态，值）。另请参阅OCIObjectPinTable()以检索表对象。

这是一个重载参数。此参数的使用基于要创建的实例的类型。有关更多信息，请参阅表27-9 。

  • 对于实例的持久对象类型，此参数指定引脚持续时间。

  • 对于临时对象类型的实例，此参数指定分配持续时间和引脚持续时间。

  • 对于实例的值类型，此参数指定分配持续时间。

值（IN）

指定创建的对象是否为值。如果为TRUE ，则创建一个值。否则，创建可引用对象。如果实例不是对象，则忽略此参数。

实例（OUT）

新创建的实例的地址。如果环境句柄具有适当的设置且对象为OCIString ，则实例可以是UTF-16（Unicode）中的OCIString 。

评论

此函数创建由类型代码或TDO指定的类型的新实例。类型可以是复杂的或原始的。

用于记录

使用OCIObjectNew()创建包记录类型时，客户端在实例化记录类型时必须使用typecode OCI_TYPECODE_RECORD 。

记录在OCIObjectNew()指定的分配持续时间内分配。随后在该期限结束时释放它们。

在客户端上实例化的记录不支持记录字段初始值设定项。例如，给定以下mypack包定义，使用OCITypeByFullName()解析时会返回以下错误： OCI-22352: Type is unsupported or contains an unsupported attribute or element 。（因此，你永远不会调用OCIObjectNew() ）。 

创建或替换包mypack是类型r是记录（rec_field号：= 10）;结束;

所有记录均为空激活;也就是说，实例化记录的所有字段都设置为NULL 。为了与PL / SQL null语义保持一致，所有实例化的记录在原子上也不是NULL 。

收藏

创建包集合类型的新实例时， value参数必须为TRUE 。这是因为包集合类型不能是持久性的或可引用的，因此必须始终将它们实例化为值。使用FALSE值参数调用包集合类型的OCIObjectNew()导致错误。

包集合在OCIObjectNew()指定的分配持续时间内分配。随后在该期限结束时释放它们。

创建包时，客户端可以使用如下的类型代码：

  • 模式级集合和包集合类型的OCI_TYPECODE_NAMEDCOLLECTION

  • 索引表的OCI_TYPECODE_ITABLE

  • 嵌套表的OCI_TYPECODE_TABLE

  • OCI_TYPECODE_VARRAY的OCI_TYPECODE_VARRAY

对于布尔人

创建新的布尔类型时，客户端应使用OCI_TYPECODE_BOOLEAN 。

此函数使用给定值设置给定对象的属性。属性的位置被指定为路径表达式，它是名称数组和索引数组。

例

对于路径表达式stanford.cs.stu [5] .addr，数组显示为：

names = {“stanford”，“cs”，“stu”，“addr”}

长度= {8,2,3,4}

indices = {5}
