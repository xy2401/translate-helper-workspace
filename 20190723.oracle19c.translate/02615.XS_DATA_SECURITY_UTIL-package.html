XS_DATA_SECURITY_UTIL包

XS_DATA_SECURITY_UTIL包是一个实用程序包，它为用户表调度静态ACL的自动刷新，并将ACL刷新模式更改为on-commit或按需刷新。

本节包括以下主题：

  • 安全模型

  • 常量

  • XS_DATA_SECURITY_UTIL子程序摘要

安全模型

XS_DATA_SECURITY_UTIL包在SYS架构中创建。您需要在程序包上具有EXECUTE权限才能运行此程序包中包含的程序。

常量

以下是 ACLMV刷新模式的有效值：

ACLMV_ON_DEMAND CONSTANT VARCHAR2（9）：='ON DEMAND'; ACLMV_ON_COMMIT CONSTANT VARCHAR2（9）：='ON COMMIT';

以下是静态ACLMV上的刷新类型：

XS_ON_COMMIT_MV CONSTANT BINARY_INTEGER：= 0; XS_ON_DEMAND_MV CONSTANT BINARY_INTEGER：= 1; XS_SCHEDULED_MV常数BINARY_INTEGER：= 2;

以下是静态ACLMV的类型：

XS_SYSTEM_GENERATED_MV CONSTANT BINARY_INTEGER：= 0; XS_USER_SPECIFIED_MV CONSTANT BINARY_INTEGER：= 1;

SCHEDULE_STATIC_ACL_REFRESH程序

SCHEDULE_STATIC_ACL_REFRESH过程用于调用或调度静态ACL到用户表的自动刷新。如果将NULL值传递给start_date和repeat_interval参数，它可以立即开始刷新。

要查找为当前用户可用的表或视图完成的所有最新静态ACL刷新作业的状态，请查询ALL_XDS_LATEST_ACL_REFSTAT ， DBA_XDS_LATEST_ACL_REFSTAT和USER_XDS_LATEST_ACL_REFSTAT数据字典视图。可以在ALL_XDS_ACL_REFSTAT ， DBA_XDS_ACL_REFSTAT和USER_XDS_ACL_REFSTAT数据字典视图中找到所有静态ACL刷新作业状态历史记录。

句法

XS_DATA_SECURITY_UTIL.SCHEDULE_STATIC_ACL_REFRESH（schema_name IN VARCHAR2 DEFAULT NULL，table_name IN VARCHAR2，start_date IN TIMESTAMP WITH TIME ZONE DEFAULT NULL，repeat_interval IN VARCHAR2 DEFAULT NULL，comments IN VARCHAR2 DEFAULT NULL）;

参数

参数 描述

SCHEMA_NAME

指定表所属的架构的名称。

TABLE_NAME

与上述模式名称一起使用的表名，用于唯一标识静态ACL更新的表。

开始日期

此属性指定计划运行此刷新的第一个日期。如果重复调用该函数，则使用最新的给定start_date和repeat_interval来调度作业。由立即调用，提交时或刷新作业完成的ACL刷新的每个执行结果都会添加到XDS_ACL_REFSTAT 。

如果start_date和repeat_interval保留为NULL ，则立即启动刷新并删除任何现有刷新计划。要立即刷新， XDS_ACL_REFRESH中不会添加任何行，因为它不会更改刷新模式。

REPEAT_INTERVAL

此属性指定刷新应重复的频率。您可以使用DBMS_SCHEDULER程序包日历语法或使用PL / SQL表达式指定重复间隔。有关使用calendering语法的详细信息，请参阅Oracle Database PL / SQL包和类型参考 。

评估指定的表达式以确定下次应该运行刷新。如果未指定repeat_interval ，则作业仅在指定的开始日期运行一次。

start_date和repeat_interval用于通过使用DBMS_SCHEDULER包创建刷新作业，其中end_date默认为NULL 。

评论

此属性指定有关作业的注释。默认情况下，此属性为NULL

例子

SYS.XS_DATA_SECURITY_UTIL.SCHEDULE_STATIC_ACL_REFRESH（'aclmvuser'，'sales'，SYSTIMESTAMP，'freq = hourly; interval = 2'）;

ALTER_STATIC_ACL_REFRESH程序

ALTER_STATIC_ACL_REFRESH过程用于将ACL刷新模式更改为on-commit或按需刷新。

句法

XS_DATA_SECURITY_UTIL.ALTER_STATIC_ACL_REFRESH（schema_name IN VARCHAR2 DEFAULT NULL，table_name IN VARCHAR2，refresh_mode IN VARCHAR2）;

参数

参数 描述

SCHEMA_NAME

指定表所属的架构的名称。

TABLE_NAME

表名，与模式名一起使用，以唯一标识用于更改静态ACL刷新模式的表。

refresh_mode

ON COMMIT或ON DEMAND

例子

SYS.XS_DATA_SECURITY_UTIL.ALTER_STATIC_ACL_REFRESH（'aclmvuser'，'sales'，refresh_mode =>'ON COMMIT'）;
