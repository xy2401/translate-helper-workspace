274.2.1 AQ $ _AGENT类型

此类型标识消息的生产者或消费者。

句法

TYPE SYS.AQ $ _AGENT是OBJECT（名称VARCHAR2（512），地址VARCHAR2（1024），协议NUMBER）;

属性

表274-1 AQ $ _AGENT属性

属性 描述

名称

消息的生产者或消费者的名称。对于保留字符，该名称必须遵循Oracle数据库SQL语言参考中的对象名称准则。

地址

特定于协议的接收者地址。如果协议为0 ，则地址的格式为[ schema .] queue [@ dblink ] 。

例如，站点dbs1.net上HR队列中名为emp_messages的队列的地址为：hr.emp_messages@dbs1.net

协议

解释地址和传播消息的协议。协议1-127保留供内部使用。如果协议号在128 - 255范围内，则Oracle Database Advanced Queuing不会解释收件人的地址。

指定上下文类型。有效值为0 (RAW)或1 (ANYDATA)

payloadcbk

指定回调函数的有效负载

超时

指定以秒为单位的时间

ntfn_grouping_class

目前，只能设置以下标志来指定分组标准。默认值为0。如果ntfn_grouping_class为0，则所有其他通知分组属性必须为0。

  • NTFN_GROUPING_CLASS_TIME - 按时间分组的通知，即用户指定时间值，并在该时间结束时发布单个通知。

ntfn_grouping_value

以秒为单位指定的通知分组的时间段，表示在ntfn_grouping_repeat_count用尽之前定期发送分组通知的时间。

ntfn_grouping_type

  • NTFN_GROUPING_TYPE_SUMMARY - 时间间隔内发生的所有通知的摘要。（默认）

  • NTFN_GROUPING_TYPE_LAST - 间隔中发生的上次通知。

ntfn_grouping_start_time

通知分组开始时间。通知分组可以从用户指定的时间开始，该时间应该是带有时区的有效时间戳。如果在使用分组时未指定ntfn_grouping_start_time ，则默认为当前时区与时区

ntfn_grouping_repeat_count

分组通知将按通知分组重复计数的指定发送多次，然后再恢复为常规通知。ntfn_grouping_repeat_count，如果未指定，将默认为

  • NTFN_GROUPING_FOREVER - 继续发送分组通知。

274.2.13 DEQUEUE_OPTIONS_T类型

DEQUEUE_OPTIONS_T指定可用于出列操作的选项。

句法

TYPE DEQUEUE_OPTIONS_T是RECORD（consumer_name VARCHAR2（30）DEFAULT NULL，dequeue_mode BINARY_INTEGER DEFAULT REMOVE，navigation BINARY_INTEGER DEFAULT NEXT_MESSAGE，visibility BINARY_INTEGER DEFAULT ON_COMMIT，等待BINARY_INTEGER DEFAULT FOREVER，msgid RAW（16）DEFAULT NULL，correlation VARCHAR2（128）DEFAULT NULL，deq_condition VARCHAR2（4000）DEFAULT NULL，签名aq $ _sig_prop DEFAULT NULL，转换VARCHAR2（61）DEFAULT NULL，delivery_mode PLS_INTEGER DEFAULT PERSISTENT）;

属性

表274-9 DEQUEUE_OPTIONS_T属性

属性 描述

CONSUMER_NAME

消费者的名字。仅访问与消费者名称匹配的那些消息。如果没有为多个使用者设置队列，则应将此字段设置为NULL 。

对于安全队列，consumer_name必须是有效的AQ代理名称，通过dbms_aqadm.enable_db_access过程调用映射到执行出列操作的数据库用户。

dequeue_mode

指定与出列关联的锁定行为。可能的设置是：

BROWSE ：在不获取任何消息锁定的情况下阅读消息。该规范等同于select语句。

LOCKED ：读取并获取消息的写锁定。锁定持续到交易期间。此设置等同于select for update语句。

REMOVE ：阅读邮件并将其删除。此设置是默认设置。可以根据保留属性将消息保留在队列表中。

REMOVE_NODATA ：将邮件标记为已更新或已删除。可以根据保留属性将消息保留在队列表中。

导航

指定将检索的消息的位置。首先，确定位置。其次，应用搜索标准。最后，检索消息。可能的设置是：

NEXT_MESSAGE ：检索可用的下一条消息并匹配搜索条件。如果先前的消息属于消息组，则AQ将检索与搜索条件匹配且属于消息组的下一个可用消息。此设置是默认设置。

NEXT_TRANSACTION ：跳过当前事务组的剩余部分（如果有）并检索下一个事务组的第一条消息。仅当为当前队列启用了消息分组时，才能使用此设置。

FIRST_MESSAGE ：检索可用的第一条消息并匹配搜索条件。此设置将位置重置为队列的开头。

F IRST_MESSAGE_MULTI_GROUP ：表示对DBMS_AQ.DEQUEUE_ARRAY的调用会将位置重置为队列的开头，并使可用的消息（可能跨越不同的事务组）出列并匹配搜索条件，直到达到ARRAY_SIZE限制。请参阅TRANSACTION_GROUP属性以获取消息以区分事务组。

NEXT_MESSAGE_MULTI_GROUP ：表示对DBMS_AQ的调用。 DEQUEUE_ARRAY将使DEQUEUE_ARRAY的下一组消息（可能跨越不同的事务组）出列并与搜索条件匹配，直到达到ARRAY_SIZE限制。请参阅TRANSACTION_GROUP属性以获取消息以区分事务组。

能见度

指定新消息是否作为当前事务的一部分出列。使用BROWSE出队模式时，将忽略visibility参数。可能的设置是：

ON_COMMIT ：dequeue将是当前事务的一部分。此设置是默认设置。

我MMEDIATE: dequeue操作不是当前事务的一部分，而是一个在操作结束时提交的自治事务。

等待

如果当前没有可用的消息与搜索条件匹配，则指定等待时间。可能的设置是：

FOREVER ：永远等待。此设置是默认设置。

NO_WAIT ：不要等。

number：以秒为单位的等待时间。

MSGID

指定要出列的消息的消息标识符。

相关

指定要出列的消息的相关标识符。可以使用特殊模式匹配字符，例如百分号（ % ）和下划线（ _ ）。如果多个消息满足该模式，则出列的顺序是不确定的。

deq_condition

基于消息属性，消息数据属性和PL / SQL函数的条件表达式。

使用类似于SQL查询的WHERE子句的语法将deq_condition指定为布尔表达式。此布尔表达式可以包括有关消息属性，用户数据属性（仅限对象有效内容）以及PL / SQL或SQL函数（在SQL查询的WHERE子句中指定）的条件。消息属性包括队列表中的priority ， corrid和其他列

要在消息有效内容（对象有效内容）上指定出列条件，请在子句中使用对象类型的属性。您必须在每个属性前加上tab 。 user_data作为限定符，用于指示存储有效内容的队列表的特定列。deq_condition参数不能超过4000个字符。如果不止一条消息满足出队条件，则出列顺序未确定。

签名

目前尚未实施

转型

指定在将消息出列后将应用的转换。转换的源类型必须与队列的类型匹配。

delivery_mode

dequeuer指定它希望在出队选项中出列的消息的传递模式。它可以是BUFFERED或PERSISTENT或PERSISTENT_OR_BUFFERED 。出列消息的消息属性指示出列消息的传递模式。数组出列仅支持数组大小为“1”的缓冲消息。

274.2.14 ENQUEUE_OPTIONS_T类型

ENQUEUE_OPTIONS_T Type指定可用于入队操作的选项。

TYPE SYS.ENQUEUE_OPTIONS_T是RECORD（可见性BINARY_INTEGER DEFAULT ON_COMMIT，relative_msgid RAW（16）DEFAULT NULL，sequence_deviation BINARY_INTEGER DEFAULT NULL，transformation VARCHAR2（61）DEFAULT NULL，delivery_mode PLS_INTEGER NOT NULL DEFAULT PERSISTENT）;

属性

表274-10 ENQUEUE_OPTIONS_T属性

属性 描述

能见度

指定入队请求的事务行为。可能的设置是：

ON_COMMIT ：enqueue是当前事务的一部分。事务提交时操作完成。此设置是默认设置。

IMMEDIATE ：入队操作不是当前事务的一部分，而是在操作结束时提交的自治事务。这是排入非持久队列时唯一允许的值。

sequence_deviation

指定要排队的消息是否应该在队列中已有的其他消息之前出列。可能的设置是：

BEFORE ：消息在relative_msgid指定的消息BEFORE排队.

TOP ：消息在任何其他消息之前排队。

转型

指定在排队邮件之前应用的转换。转换函数的返回类型必须与队列的类型匹配。

delivery_mode

enqueuer指定它希望在enqueue选项中排队的消息的传递模式。它可以是BUFFERED或PERSISTENT 。排队消息的消息属性指示排队消息的传递模式。仅对阵列大小为“1”的缓冲消息支持数组入队。

274.2.16 SYS.MSG_PROP_T类型

此类型在PL / SQL通知中用作aq$_descriptor一个字段，用于将AQ消息的消息属性传递给PL / SQL通知客户端回调。

句法

创建或替换TYPE sys.msg_prop_t AS OBJECT（优先级NUMBER，延迟NUMBER，到期NUMBER，相关VARCHAR2（128），尝试NUMBER，exception_queue VARCHAR2（51），enqueue_time DATE，状态NUMBER，sender_id aq $ _agent，original_msgid RAW（16） ，delivery_mode NUMBER）;

参数

表274-12 SYS.MSG_PROP_T类型属性

参数 描述

优先

指定消息的优先级。数字越小表示优先级越高。优先级可以是任何数字，包括负数。

延迟

指定排队消息的延迟。延迟表示消息可用于出队的秒数。msgid会覆盖延迟规范。带有延迟集的队列消息处于WAITING状态，当延迟到期时，消息进入READY状态。DELAY处理需要启动队列监视器。但是，如果需要，系统会自动启动队列监视器。延迟由排队邮件的制作人设置。

可能的设置如下：

NO_DELAY ：该消息可立即出列

number：延迟消息的秒数

呼气

指定消息的到期时间。它以秒为单位确定消息可用于出列的持续时间。此参数是消息为出队准备的时间的偏移量。到期处理要求队列监视器正在运行。但是，如果需要，系统会自动启动队列监视器。

可能的设置如下：

NEVER ：消息不会过期

number：消息保持READY状态的秒数。如果消息在到期之前未出列，则将其移至EXPIRED状态的异常队列。

相关

在入队时返回消息生产者提供的标识符。

尝试

返回使消息出列的尝试次数。此参数不能在入队时设置。

exception_queue

如果无法成功处理消息，则指定消息移动到的队列的名称。

消息将自动移动到异常队列中。在以下情况下，消息将移动到异常队列中：

  • RETRY_COUNT ，出队不成功尝试的次数，已经超过了说明书MAX_RETRIES中参数DBMS_AQADM.CREATE_QUEUE队列创建过程中的过程。

    对于多消费者队列，即使失败的出队尝试超过仅有一个消费者的MAX_RETRIES参数，该消息也有资格被移动到异常队列。但是，在所有其他消费者成功使用消息或失败超过MAX_RETRIES之前，消息不会被移动。您可以查看MAX_RETRIES在队列ALL_QUEUES数据字典视图。

    如果由于服务器进程在实例上死亡（包括ALTER SYSTEM KILL SESSION ）或SHUTDOWN ABORT而导致dequeue事务失败，则RETRY_COUNT不会递增。

  • 在到期时间过去之前，消息未出列。

  • 传递到指定目标队列的消息失败，并出现以下错误之一：

    *多消费者目标队列没有收件人。

    *为单个使用者目标队列指定了收件人。

    *目标队列是一个异常队列

    *应用转换时出错。

缺省值是与队列表关联的异常队列。如果在移动时指定的异常队列不存在，则将消息移动到与队列表关联的缺省异常队列，并在警报日志中记录警告。如果指定了缺省异常队列，则该参数在出列时返回NULL值。

enqueue_time

指定邮件入队的时间。该值由系统确定，用户无法在入队时设置。

州

指定出列时的消息状态。此参数不能在入队时设置。可能的状态如下：

  • DBMS_AQ.READY：消息已准备好进行处理。

  • DBMS_AQ.WAITING：尚未达到消息延迟。

  • DBMS_AQ.PROCESSED：消息已处理并保留。

  • DBMSAQ.EXPIRED：消息已移至异常队列。

发件人ID

消息生产者在入队时指定的应用程序发送者标识。发件人ID的类型为aq$_agent 。

在入队时安全队列需要发件人姓名。这必须是有效的AQ代理名称，通过dbms_aqadm映射到执行入队操作的数据库用户。 enable_db_access过程调用。不应指定发件人地址和协议。

如果消息是从另一个队列传播的，则在出列时返回的消息属性中的Sender id可能具有发送方地址。如果是远程数据库，则地址的值为source_queue ，源数据库名称[ format source_queue@source_database_name ]
