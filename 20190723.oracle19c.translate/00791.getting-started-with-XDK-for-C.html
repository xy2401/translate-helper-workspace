3 Oracle XML Developer's Kit for C入门

解释了如何开始使用适用于C的Oracle XML Developer Kit（XDK）。

为C组件安装XDK

XDK for C组件是用于读取，操作，转换和验证可扩展标记语言（XML）的构建块。Oracle数据库包含XDK for C组件。

警告：

使用Oracle XML Developer Kit（XDK）的组件构建软件程序可以实现一些功能强大但具有潜在危险的功能，例如外部实体扩展和递归扩展。有关如何安全使用XDK的信息，请参阅Oracle XML Developer's Kit的安全注意事项 。

本章假定您已将XDK与Oracle数据库一起安装，并在Oracle数据库示例介质上安装了演示程序。有关安装说明和XDK目录结构的说明，请参见关于安装XDK 。

以下示例显示了XDK演示的UNIX目录结构以及XDK组件使用的库。子目录包含XDK for C组件的示例程序和数据文件。

UNIX上的C组件依赖关系的XDK

本节中描述的C库位于$ORACLE_HOME/lib 。

C和C ++组件的XDK包含在此库中：

libxml11.a

库中包含以下XKD组件：

  • XML解析器，用于检查XML文档的格式，可选择根据文档类型定义（DTD）或XML架构验证它，并支持文档对象模型（DOM）和Simple API for XML（SAX）接口以进行编程访问

  • 可扩展样式表语言转换（XSLT）处理器，它将XML文档转换为另一个XML文档

  • XSLT编译器，它将XSLT样式表编译为字节代码，供XSLT虚拟机（XSLT VM）使用

  • XSLTVM，是一个XSLT转换引擎

  • XML Schema处理器，它根据XML模式验证XML文件

Unified C API概述

统一的C API是一个编程接口，它统一了XDK for C和Oracle XML DB所需的功能。此API主要由XSLT和XML Schema使用。

如表3-4所示，统一的C API在xml.h头文件中声明。表3-8总结了C API的XDK。有关完整文档，请参阅Oracle数据库XML C API参考 。

表3-8适用于C API的Oracle XML Developer套件摘要

包 目的

回调API

定义为XML回调声明函数（或函数指针）的宏。

DOM API

使用DOM解析和操作XML文档。 API尽可能地遵循DOM 2.0标准，尽管它在从面向对象的DOM规范映射到平面C命名空间时更改了一些名称。例如，重载的getName()方法变为getAttrName() 。

范围API

创建和操作Range对象。

SAX API

使用SAX启用基于事件的XML解析。

架构API

将多个XML模式文档组装到单个模式中，该模式可用于验证特定的实例文档。

遍历API

启用DOM树的文档遍历和导航。

XML API

根据XML处理器必须如何读取XML数据及其必须提供给应用程序的信息来定义XML处理器。

XPath API

处理与XPath相关的类型和接口。

XPointer API

在XML文档中找到节点。

XSLT API

执行XSL处理。

XSLTVM API

实现可以运行已编译的XSLT代码的虚拟机。

API通过符合上下文来完成函数的统一。顶级XML上下文（ xmlctx ）共享协作XML组件之间的公共信息。该上下文定义了以下信息：

  • 数据编码

  • 错误消息语言

  • 低级分配回调

应用程序在解析文档并通过DOM或SAX接口提供编程访问之前需要此信息。

XDK for C和Oracle XML DB都需要针对顶级和服务上下文使用不同的启动和拆除功能。初始化函数采用特定于实现的参数并返回一致的上下文。

通过使用符合条件的上下文使得统一成为可能。符合上下文意味着返回的上下文必须以xmlctx ;它可能在标准头之后有任何其他特定于实现的部分。

应用程序获取xmlctx ，它使用统一的DOM调用，所有这些调用都将xmlctx作为第一个参数。

全球化支持XDK for C组件

XDK for C解析器支持超过300个IANA字符集。

这些字符集包括由XDK for C支持的字符集中列出的字符集 。

使用字符集时的注意事项：

  • Oracle建议您使用Internet Assigned Numbers Authority（IANA）字符集名称与其他XML解析器进行互操作。

  • XML解析器只需要支持UTF-8和UTF-16，因此这些字符集更可取。

  • 默认输入编码（“incoding”）是UTF-8。如果输入文档的编码不是不言自明的（通过HTTP字符集，字节顺序标记（BOM），XMLDecl等），则假定默认输入编码。如果仅使用单字节字符集（如US-ASCII或任何ISO-8859字符集），Oracle建议您显式设置默认编码，因为单字节性能最快。标志XML_FLAG_FORCE_INCODING指定默认输入编码始终应用于输入文档，忽略任何BOM或XMLDecl。然而，诸如HTTP字符集之类的协议声明总是受到尊重。

  • 请仔细选择DOM和SAX的数据编码（“outcoding”）。单字节编码是最快的，但只能代表一组非常有限的字符。下一个最快的是Unicode（UTF-16），最慢的是多字节编码，如UTF-8。如果输入数据无法在没有丢失的情况下转换为输出，则会发生错误。为了获得最大效用，请使用基于Unicode的输出，因为Unicode可以表示任何字符。如果未指定outcoding，则默认为解析的第一个文档的编码。
