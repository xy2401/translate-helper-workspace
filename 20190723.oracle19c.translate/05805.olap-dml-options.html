5.1关于选项

OLAP DML选项是一种特殊类型的分析工作空间对象，它指定Oracle OLAP如何计算或格式化数据或Oracle OLAP操作被激活的某些方面的特征。某些选项是只读的，而其他选项是可以指定值的读/写选项。读/写选项具有默认值。

您可以使用SET （=）命令将选项的值检索到预定义变量中，并为读/写选项指定新值。使用SHOW命令显示选项的值。

数据类型

整数

句法

AWWAITTIME = 秒

参数

秒

等待分析工作区可用的秒数。默认值为20秒。

使用说明

工作区共享

当您的用户标识具有适当的访问权限且没有用户具有对工作区的读/写独占访问权限时，无论有多少其他用户使用它，您都可以获得对分析工作区的只读访问权限。当另一个用户具有读/写访问权限并提交工作空间时，工作空间的视图不会改变;您必须分离并重新连接工作区才能查看更改。

5.8 BMARGIN

BMARGIN选项定义输出页面下边距的空白行数。仅当PAGING设置为YES且仅用于REPORT和DESCRIBE等语句的输出时，BMARGIN才有意义。 BMARGIN选项通常在报告程序的初始化部分中设置。

数据类型

整数

句法

BMARGIN = n

参数

n

一个INTEGER表达式，指定为报表中的下边距预留的行数。默认值为1 。

使用说明

为文件设置BMARGIN

要为文件设置BMARGIN，首先通过在OUTFILE语句中指定其名称，使文件成为当前的outfile，然后将BMARGIN设置为所需的值。在重置或直到使用OUTFILE语句将输出定向到不同的outfile之前，新值仍然有效。当您将输出定向到不同的outfile时，BMARGIN会返回该文件的默认值1 。

当您为默认输出文件设置BMARGIN时，新值将一直有效，直到您重置它为止，无论介入的输出发送到文件的OUTFILE语句如何。也就是说，BMARGIN的值会自动保存为默认的outfile

例子

例5-6设置报告页面的底边距

假设您希望能够在报告页面的底部做笔记。您可以设置5行的大底部边距。以下是您将在报告程序的初始化部分中包含的声明。

BMARGIN = 5

5.9 CALENDARWEEK

CALENDARWEEK选项确定周是否应与实际日历年保持一致。

注意：

您只能将此功能用于WEEK类型的尺寸。

数据类型

布尔

句法

CALENDARWEEK = { YES | NO}

参数

是的

（默认）指定周与日历年对齐。例如，如果已定义类型为WEEK的维度，则Oracle OLAP会对其值进行编号，以使日历年中的第一周为第1周，而日历年中的第二周为第2周，依此类推。无论WEEK维度定义中指定的任何开始或结束日期如何，周都与日历年对齐。

没有

指定周与日历年不对齐。而是对周数进行编号，以使它们与维度定义中指定的日期保持一致。例如，如果已定义具有开始日期或结束日期的WEEK类型的维度，则对其值进行编号，以使与维度定义中的日期对应的周为第1周，后一周为第2周，依此类推。

使用说明

财政年度

将CALENDARWEEK设置为NO会导致数周的编号，以便将数字1分配给在DEFINE DIMENSION语句中指定的日期开始或结束的周。然后将本周分配到会计年度，该会计年度是本周开始日期或之后的1月1日的日历年。例如，如果你定义类型WEEK的尺寸与起始日期02Jan1996 （或等价的，结束日期08Jan1996 ），开始的一周02Jan1996被认为是一周1财年的1997 。相反，如果你给维度的开始日期在02Jan1995和01Jan1996之间， 01Jan1996该日期开始的那一周是1996财政年度的第1周。

例子

5.10 COLWIDTH

COLWIDTH选项控制报表输出中数据列的默认宽度。对于来自ROW命令和HEADING命令的输出，COLWIDTH会影响除第一列之外的所有列。对于REPORT的输出，COLWIDTH会影响复合或联合维度的所有数据列和标签列。

注意：

对于单个列，COLWIDTH值始终由HEADING，REPORT或ROW命令中的WIDTH属性覆盖

数据类型

整数

句法

COLWIDTH = n

参数

n

一个INTEGER表达式，以字符数指定所需的列宽。您可以将COLWIDTH设置为1到4,000任何值。默认值为10 。

注意：

报告中行的最大宽度为4,000个字符。因此，报表的所有列的组合宽度不能超过4,000个字符。

5.11 COMMAS

COMMAS选项控制使用在数字输出中分隔数千和数百万的字符。这个字符通常是逗号;但是，根据您的NLS_TERRITORY设置，它可能会有所不同。THOUSANDSCHAR选项记录当前用于分隔数千个字符的字符。COMMAS选项控制字符是否出现在数字输出中。

COMMAS影响产生输出的所有命令，包括ROW命令，HEADING，REPORT和SHOW。

注意：

您可以使用HEADING，REPORT或ROW命令的COMMA和NOCOMMA属性来覆盖COMMAS设置。

数据类型

布尔

句法

COMMAS = {NO | 是的 }

参数

没有

数字输出不包含分隔数千，数百万等的字符。

是的

（默认）数字输出包含分隔数千，数百万等的字符。

例子

ID

以下文本表达式之一： 'MDY' ， 'DMY' ， 'YMD' ， 'YDM' ， 'MYD' ， 'DYM' 。每个字母代表日期的一个组成部分。M代表月份， D代表当天，Y代表年份。默认日期顺序为'MDY' 。

使用说明

暧昧的日期

当您输入明确的仅DATE值或将只有一种解释的文本值转换为日期时，无需咨询DATEORDER选项即可处理。例如，在03-24-97 ， 97只能指年份。考虑到剩下的， 24不能指月份，所以它必须是一天。只剩下03 ，所以必须是月份。但是，当解释不明确时，如值3-5-97 ， 3-5-97的当前值用于解释每个组件的含义。

要在将文本值转换为仅DATE值时覆盖当前的DATEORDER设置，请使用带有date-order参数的CONVERT函数。

基本日期组件

假设您要使用MAINTAIN语句将日期值分配给DAY，WEEK，MONTH，QUARTER或YEAR维度，或使用LIMIT命令将日期值分配给值集。当您以DATE-only表达式或文本文本的形式指定值时，Oracle OLAP使用DATEORDER选项来解释该值。提供文本文字时，可以使用任何有效的输入样式作为日期。但是，您必须仅提供标识您正在使用的特定类型的维或值集中的时间段所需的日期组件。例如，对于MONTH维度或其值集，您可以指定完整的日期，例如30jun97 ，或者您只能提供必要的组件，例如jun97或0697 。

5.16 DAYABBRLEN

DAYABBRLEN选项指定用于存储在DAYNAMES选项中的日期名称缩写的字符数。当您使用DATEFORMAT文本选项指定<WT> ， <WTXT>和<WTXTL>格式时，可以指定用于缩写特定日期名称的字符数。

句法

参数

表示缩写长度由长度定义的第一天和最后一天的数字。 这些数字位置适用于DAYNAMES选项中相应的文本行。您可以按相反顺序指定这些值范围， endpos [ - startpos ] ，如果您愿意。

长度

一个数字，指定缩写日期名称的字符长度（不是字节）。如果未在DAYNAMES选项中指定给定位置的缩写长度，或者明确将给定位置设置为零，则Oracle OLAP将使用<WT>的一个字符的默认缩写和<WTXT>和<WTXTL>三个字符<WTXTL> 。当您指定全名规范<WTEXT>和<WTEXTL>时，Oracle OLAP从不使用缩写。

使用说明

暧昧的日子名称

 LINENUM选项包含输出的当前行号。当输出线产生时，其值会自动递增。仅当PAGING设置为YES且仅用于REPORT和LISTNAMES等命令的输出时，LINENUM选项才有意义。

在输出中发送LINENUM

                     
当您生成包含LINENUM值的输出，并且此输出创建了一个新页面时，当您的输出由一行组成时，LINENUM的值将记录为1 。但是，当输出是多行值时，LINENUM的值可以记录为大于PAGESIZE的值。

5.49 MODDAMP

               

                  
 MODDAMP选项指定加权因子，当使用Gauss-Seidel方法求解模型中的联立方程时，该加权因子会抑制迭代之间的振荡。MODDAMP可以允许解决模型，否则这些模型将永远不会收敛，因为方程之间的振荡是稳定的。在这些情况下，振动不会在没有阻尼的情况下衰减。
                  

                  
使用Gauss-Seidel方法，Oracle OLAP在一组联立方程中测试每个迭代中的收敛或发散的每个模型方程。通过将当前迭代的结果与前一次迭代的结果进行比较来进行测试。当MODDAMP指定加权因子大于零时，Oracle OLAP在每次迭代后测试和存储的值是当前和之前结果的加权平均值。对于在迭代之间振荡的方程，因此可以使用MODDAMP来抑制振荡，并防止发散或加速方程的收敛。

                  

                     
数据类型

                     
DECIMAL

                  

                  
                  

                     
句法

                     
MODDAMP = { n | 0.00 }

                  

                  
                  

                     
参数

                     

                        
 n

                        

                           
小数值，大于或等于零且小于1，指定加权因子。MODDAMP越接近0.00，对当前迭代的值赋予的权重越大。默认值为0.00，它给出了当前迭代的全部权重。

                           
当MODDAMP大于零时，Oracle OLAP计算当前迭代的加权平均值，如下所示。

                           
calcvalue *（1  -  MODDAMP）+ weightavg

                           
哪里：

                           
    
                              
  • 
                                 
    calcvalue是从当前迭代中的模型方程计算的值。
                                 
    
                              
    • 
                              
  • 
                                 
    weightavg是在前一次迭代中计算的加权平均值。
                                 
    
                              
    • 
                           

                           
请参阅“ 存储的加权平均值 ” 。
                           

                        

                     

                  

                  
                  

                     
使用说明

                     
指定解决方法

                     
MODDAMP选项仅用于Gauss-Seidel方法以求解联立方程。MODSIMULTYPE选项确定正在使用的解决方法方法。MODSIMULTYPE的可能设置是GAUSS ，用于Gauss-Seidel方法， AITKENS用于Aitkens delta-squared方法。
                     

                     
MODDAMP对收敛速度的影响

                     
MODDAMP用于计算每个同时块中所有模型方程的结果，无论它们是否在迭代之间振荡。对于不振荡的方程，当MODDAMP的值大于零时，收敛速度会减慢。因此，当您的模型包含一些振荡的方程式而某些方程式不振荡时，您可以通过将MODDAMP设置为小的非零值（例如0.20来加速整体收敛。小的非零值仅略微减慢非振荡方程的收敛速度，同时加速振荡方程的收敛。
                     

                     
存储加权平均值

                     
当模型方程在当前迭代上不收敛或发散时，存储在当前迭代中计算的加权平均值。在下一次迭代中，Oracle OLAP将此存储的平均值用作加权平均值公式中的weightavg （即前一次迭代中计算的加权平均值）。
                     

                     
在块上的第一次迭代中，Oracle OLAP使用目标变量（或维值）的起始值作为weightavg （即，在上一次迭代中计算的加权平均值） 。

                     
迭代结果比较

                     
在每次迭代中收敛和发散的测试中，Oracle OLAP将当前迭代的结果与上一次迭代的结果进行比较。当MODDAMP大于零时，Oracle OLAP会测试按如下方式计算的比较值 。
                     

                     
（ weightavg - weightavg ）/（ weightavg PLUS MODGAMMA）

                     
其中weightavg是在前一次迭代中计算的加权平均值

                     
有关收敛测试的说明，请参阅MODTOLERANCE选项。有关分歧测试的说明，请参阅MODOVERFLOW选项。
                     

                     
控制同时块解决方案的选项

                     
改变MODDAMP的值只是尝试加速或实现同时块收敛的一步。MODEL列出了可用于控制同步块解决方案的其他选项，并提供有关运行和调试模型的信息。

5.51 MODGAMMA

               

                  
 MODGAMMA选项指定用于测试模型的同时块中的等式在迭代之间变化多少的值。MODGAMMA控制测试比较迭代之间的变化绝对量与比例变化的程度。MODGAMMA在测试导致非常小值的方程式时尤为重要。
                  

                  

                     
注意：

                     
“ 模型选项 ”用于描述控制同时块解决方案的所有选项。
                     

                  

                  

                     
数据类型

                     
整数

                  

                  
                  

                     
句法

                     
MODGAMMA = { n | 1 }

                  

                  
                  

                     
参数

                     

                        
 n

                        

                           
用于测试收敛和发散的INTEGER值。当Oracle OLAP计算同时块中的每个等式时，它构造一个比较值，该值基于当前迭代和前一次迭代的等式结果。当比较值通过公差测试时，该等式被认为已收敛。当比较值满足溢出测试时，该等式被认为是分歧的。
                           

                           
测试的比较值如下。
（ thisResult - prevResult ） DIVIDED BY（ prevResult PLUS MODGAMMA）
其中thisResult是此迭代的结果，而prevResult是前一次迭代的结果。
                           

                           
Oracle OLAP计算所包含表达式的绝对值。MODGAMMA的默认值为1 。
                           

                        

                     

                  

                  
                  

                     
使用说明

                     
测试融合

                     
在收敛测试中，MODTOLERANCE选项确定方程结果在连续迭代之间必须匹配的程度。对于MODTOLERANCE，默认值为3 ，当比较值小于0.001时，该等式被认为已收敛。
                     

                     
测试发散

                     
在发散测试中，MODOVERFLOW选项确定在连续迭代中方程的结果必须有多不相同。对于MODOVERFLOW，默认值为3时，当比较值大于1000时，该等式被认为已经发散。
                     

                     
比较价值

                     
Oracle OLAP在收敛和发散测试中评估的比较值基本上是比例值。它表示迭代之间的变化，作为先前结果的一部分。在收敛测试中，迭代之间的变化必须与先前的结果成比例地小。在分歧测试中，迭代之间的变化必须与之前的结果成比例。通过测试比例值，而不是测试绝对变化量，Oracle OLAP可以对所有方程应用相同的测试标准，无论方程结果的大小如何。

                     
但是，Oracle OLAP测试的比较值并不严格成比例。当方程的结果非常接近零时，严格比例的比较值的分母也将非常接近于零，因此比较值本身通常会很大。因此，难以满足收敛性测试，而易于满足发散测试。要解决此问题，Oracle OLAP会将MODGAMMA的值添加到比较值的分母中。当MODGAMMA使用默认值1时，MODGAMMA的效果如下：

                     
    
                        
  • 
                           
    当方程结果接近零时，分母接近于1，测试基本上是迭代之间绝对变化的测试。
    
                        
    • 
                        
  • 
                           
    当方程结果非常大时，将MODGAMMA添加到分母的效果可以忽略不计，并且测试接近于严格的比例测试。
    
                        
    • 
                     

                     
控制测试灵敏度

                     
对于接近于零的方程值，您可以通过更改MODGAMMA的值来控制对收敛和发散的测试的灵敏度。当方程值非常小时，您实际上在更改MODGAMMA的值时在迭代之间缩放模型值的更改。例如，当您将MODGAMMA从1更改为2时，比较值基本上减半。因此，您可以降低发散的可能性。

                     
提高模型方程收敛速度的方法

                     
在大多数情况下，MODGAMMA的默认值是合适的。当您增加MODGAMMA的值时，模型方程会更快地收敛，但结果不太精确。方程值越小，增加MODGAMMA的效果越明显;对于小模型值，相对更快地获得收敛，同时丢失更高的精度。

                     
您还可以通过降低MODTOLERANCE的值来强制模型的同步块更快地收敛，从而放宽测试收敛。但是，当您这样做时，您牺牲了所有结果的精度，而不仅仅是具有小值的方程的结果。

                     
因此，当模型包含一些具有较大值的方程和一些具有非常小值的方程时，可能最好增加MODGAMMA而不是降低MODTOLERANCE。通过增加MODGAMMA，您可以强制使用较小值的方程式更快地收敛，同时保留具有较大值的方程式的精度。

5.54 MODOVERFLOW

               

                  
 MODOVERFLOW选项用于测试模型的同时块中的任何方程是否已经发散。MODOVERFLOW确定方程的结果在连续迭代中必须是多么不相似，以使方程被认为是分歧的。
                  

                  

                     
注意：

                     
“ 模型选项 ”，用于控制同时块解决方案的所有选项的列表。
                     

                  

                  

                     
数据类型

                     
整数

                  

                  
                  

                     
句法

                     
MODOVERFLOW = { n | 3 }

                  

                  
                  

                     
参数

                     

                        
 n

                        

                           
用于测试分歧的INTEGER值。当Oracle OLAP计算同时块中的每个等式时，它构造一个比较值，该值基于当前迭代和前一次迭代的等式结果。当比较值满足发散测试时，该等式被认为是分歧的。
                           

                           
测试的比较值如下。

                           
（ thisResult - prevResult ）/（ prevResult + MODGAMMA）

                           
其中thisResult是此迭代的结果，而prevResult是前一次迭代的结果

                           
在前面的计算中，MODGAMMA是一个INTEGER选项，用于控制比较值表示迭代与比例变化之间的绝对变化量的程度。MODGAMMA的默认值为1 。
                           

                           
在差异测试中，Oracle OLAP测试比较值是否大于10的MODOVERFLOW功率。该测试的计算如下。
                           
比较值> 10 ** MODOVERFLOW
对于被认为具有分歧的等式，比较值必须满足前面描述的测试。MODOVERFLOW的默认值为3 。使用此默认值时，比较值在大于1000时符合测试。
                           

                        

                     

                  

                  
                  

                     
使用说明

                     
方程发散

                     
当方程出现偏差时，会发生错误。MODERROR选项控制Oracle OLAP在发生错误时采取的操作。
                     

                     
发展过程中分歧更快

                     
在开发模型时，有时可以通过使用较小的MODOVERFLOW值来节省时间。当Oracle OLAP在特定的同时块上执行多次迭代时，较小的MODOVERFLOW值会导致该块的快速分歧。当您将MODOVERFLOW选项设置为CONTINUE ，模型的执行会在发生分歧时继续执行，您可以专注于调试模型中的其他块。调试模型后，可以为MODOVERFLOW使用更大的值。

5.55 MODSIMULTYPE

               

                  
 MODSIMULTYPE选项指定用于解决模型中的同时块的方法。
                  

                  

                     
注意：

                     
“ 模型选项 ”，用于控制同时块解决方案的所有选项的列表。
                     

                  

                  

                     
数据类型

                     
ID

                  

                  
                  

                     
句法

                     
MODSIMULTYPE = { 'AITKENS' |'GAUSS'}

                  

                  
                  

                     
参数

                     

                        
 '

                        

                           
（默认）Oracle OLAP使用Aitkens delta-squared解决方案方法。在一组联立方程的每三次迭代的前两个中，使用来自先前迭代的值求解方程，并且测试结果的收敛和发散。在每第三次迭代中，结果不是通过求解方程而是通过进行下一次猜测计算而获得的。此计算使用前三次迭代的结果。猜测结果未针对收敛和发散进行测试，并且解决方案始终持续到下一次迭代。

                        

                        
 '

                        

                           
Oracle OLAP使用Gauss-Seidel解决方案方法。在块上的每次迭代中求解同时块中的等式。在每次迭代中测试结果的收敛和发散。

                        

                     

                  

                  
                  

                     
使用说明

                     
解决同时块

                     
Oracle OLAP使用迭代方法在同时块中求解方程。在每次迭代中，除了Aitkens解决方案中的下一次猜测迭代之外，根据当前迭代的结果和前一次迭代的结果计算比较值。当比较值落在指定的公差范围内时（参见MODTOLERANCE选项），该等式被认为已收敛。当比较值太大时（参见MODOVERFLOW选项），等式被认为已经发散并且块的解决方案结束。
                     

                     
当块中的所有方程收敛时，该块被认为是求解的。当任何方程出现偏差或任何方程在指定次数的迭代后无法收敛时（请参阅MODMAXITERS选项），块（和模型）的解决方案将失败，Oracle OLAP将生成错误。
                     

                     
下一个猜测计算

                     
Aitkens方法需要三个值来执行下一次猜测计算。因此，在同步块的前三次迭代中，Oracle OLAP解决方程式。第四次迭代是下一次猜测迭代，它使用计算中前三次迭代的结果。

                     
此后，每第三次迭代是下一次猜测迭代，其通过使用先前猜测来计算结果，并且来自中间两次迭代的等式结果。例如，第七次迭代进行下一次猜测计算，该计算基于来自第四次迭代的猜测，并且该方程式来自第五次和第六次迭代。

                     
需要内存

                     
Aitkens方法通常加速收敛，并且它通常产生比Gauss-Seidel方法更准确的结果。但是，Aitkens方法需要更多内存，因为存储了前三次迭代的结果。

                     
通常，使用Aitkens方法。仅当有限内存是系统问题时才使用Gauss-Seidel方法。

                     
在模型中求解同时块时处理NA值

                     
在计算方程式结果并进行下一次猜测计算时，Oracle OLAP会观察NASKIP2选项的设置。NASKIP2控制在执行+ （加号）和- （减号）操作时如何处理NA值。当您指定包含NA值的解决方案变量时，NASKIP2的设置很重要。由于求解变量中的值在同时块的第一次迭代中用作初始值，因此当解决方案变量中存在NA值时，等式的结果可能为NA 。一个NA结果在第一次迭代也可能会产生NA导致后面的迭代。因此，为避免获得结果的NA ，可以确保解决方案变量不包含NA值，或者可以在运行模型之前将NASKIP2设置为YES 。
                     

                     
数据类型问题

                     
当将数据分配给具有INTEGER数据类型的变量或者为基于维度的模型指定具有INTEGER数据类型的解决方案变量时，联立方程可能无法收敛。Oracle OLAP在计算每次迭代中的方程时将数据转换为十进制值，但结果存储在迭代之间的INTEGER变量中，这会使值四舍五入，从而干扰收敛的进展。

                     
功能问题

                     
当联立方程包含产生舍入值的函数（例如INSTRB或ROUND）或者包含在数据中引入不连续性的函数（例如MAX或MIN）时，联立方程可能无法收敛。

                     
起始价值问题

                     
同时块的解决方案对起始值敏感。例如，当模型在两个模型值之间具有比例关系时，则接近零的起始值会抑制收敛。因此，尝试使用对于要求解的方程合理的起始值。

                     
方程组

                     
同时块的解也对方程的阶数敏感。编译模型时，模型编译器会根据方程之间的依赖关系确定最佳方程顺序。

                     
要强制同时块中的方程按特定顺序求解，可以按所需顺序编写方程式，并在编译模型之前将MODINPUTORDER选项设置为YES 。当MODINPUTORDER为YES ，模型编译器按照它们在模型中出现的顺序将方程式保留在同步块中。
                     

                     
通过以特定顺序放置联立方程并在编译模型之前将MODINPUTORDER设置为YES ，您可能能够在某些模型中鼓励收敛。然而，通常，最好依靠模型编译器来对方程进行排序。
                     

                     
制作执行报告

                     
运行模型后，您可以使用MODEL.XEQRPT程序生成有关模型执行情况的报告。

                  

                  
                  

                     
例子

5.56 MODTOLERANCE

               

                  
 MODTOLERANCE选项用于测试模型的同时块中的每个方程是否已收敛。MODTOLERANCE确定方程的结果在连续迭代之间必须匹配的程度，以使方程被认为已收敛。
                  

                  

                     
数据类型

                     
整数

                  

                  
                  

                     
句法

                     
MODTOLERANCE = { n | 3 }

                  

                  
                  

                     
参数

                     

                        
 n

                        

                           
用于测试收敛的INTEGER值。当Oracle OLAP计算同时块中的每个等式时，它构造一个比较值，该值基于当前迭代和前一次迭代的等式结果。当比较值通过公差测试时，该等式被认为已收敛。
                           

                           
测试的比较值如下。
（ thisResult - prevResult ）/（ prevResult + MODGAMMA）
其中thisResult是此迭代的结果，而prevResult是前一次迭代的结果

                           
在前面的计算中，MODGAMMA是一个INTEGER选项，用于控制比较值表示迭代与比例变化之间的绝对变化量的程度。MODGAMMA的默认值为1。

                           
在容差测试中，Oracle OLAP测试比较值是否小于10到MODTOLERANCE的负功率。该测试的计算如下。
                           
比较值 <10 **  -  MODTOLERANCE
编写该计算的等效方法如下。
比较值 <（1 /（10 ** MODTOLERANCE））
对于被认为已收敛的等式，比较值必须满足前面描述的测试。MODTOLERANCE的默认值为3 。使用此默认值时，比较值在小于0.001时符合测试。
                           

                        

                     

                  

                  
                  

                     
使用说明

                     
未能融合

                     
当方程在指定次数的迭代之后未能收敛时，会发生错误。MODMAXITERS选项控制尝试的最大迭代次数。MODERROR选项控制Oracle OLAP在发生错误时采取的操作。
                     

                     
结果的精确度

                     
因为MODTOLERANCE控制方程的结果必须在迭代之间匹配的程度，因此它控制解决方案结果的精度。较小的MODTOLERANCE值导致精度较低，而较大的值则提供较高的精度。

                     
大小价值观

                     
当模型包含一些具有较大值的方程和一些具有非常小值的方程时，可能最好增加MODGAMMA选项的值而不是降低MODTOLERANCE。通过增加MODGAMMA，您可以强制使用较小值的方程式更快地收敛，同时保留具有较大值的方程式的精度。

                     
发展过程中更快的融合

                     
在开发模型时，您可能希望使用较小的MODTOLERANCE值。虽然这会得到不太精确的结果，但模型方程会更快地收敛。调试模型后，可以增加MODTOLERANCE的值，从而提高最终结果的精度。

                     
控制同时块解决方案的选项

                     
有关可用于控制同时块解决方案的所有选项的列表，请参阅“ 模型选项 ” 。
                     

                  

                  
                  

                     
例子

何时使用MULTIPATHHIER

 PAGENUM选项保存当前页面输出的数量。您可以将PAGENUM与PAGEPRG一起使用，以在报告的每个页面上生成页码。仅当PAGING设置为YES且仅用于REPORT和LISTNAMES等语句的输出时，PAGENUM选项才有意义。

                  

                     
数据类型

                     
整数

                  

                  
                  

                     
句法

                     
PAGENUM = n

                  

                  
                  

                     
参数

                     

                        
 n

                        

                           
一个INTEGER表达式，指定用于下一页输出的页码。默认值为1 。
                           

                        

                     

                  

                  
                  

                     
使用说明

                     
从第1页开始

                     
当您将输出发送到默认输出文件时，无论何时要从第1页开始生成报告，都要将PAGENUM和LINENUM都设置为1。您可以在报告程序的初始化部分设置这些选项。使用OUTFILE语句将输出发送到文件时，PAGENUM会自动设置为1 。
                     

                     
在中间页面设置PAGENUM

PAGING对PAGENUM的影响

                     
当您将PAGING设置为NO ，PAGENUM会停止计数并保持其最后一个值。当您将PAGING重置为YES ，PAGENUM将从其停止的页码开始计数。

使用OUTFILE语句将输出定向到文件时，文件的PAGENUM设置为1 。当您使用带有EOF关键字的OUTFILE语句将输出重定向到默认输出文件时，PAGENUM包含它最后为默认输出文件保留的数字。

（PRG = SALESREP）。 。 。

5.114 ZSPELL

               

                  
 ZSPELL选项保存默认文本，该文本用于表示HEADING，REPORT和ROW命令生成的输出中的数字零值。
                  

                  

                     
数据类型

                     
文本

                  

                  
                  

                     
句法

                     
ZSPELL = {' text '| 'OFF' }

                  

                  
                  

                     
参数

                     

                        
 文字

                        

                           
拼写用作数字零值的默认拼写。指定表达式而不是文本文字时，可以省略单引号。

                        

                        
关闭

                        

                           
（默认）为每个数字零值显示带有适当小数位数（由DECIMAL属性确定）的零（0）。

                        

                     

                  

                  
                  

                     
使用说明

                     
分配零值

                     
ZSPELL仅影响输出;它不会影响您分配零值的方式。例如，即使将ZSPELL设置为NONE ，仍然按如下方式指定零值。
                     
var1 = 0
显示小数位数

                     
默认值为OFF表示零值显示为0（零），小数位数由DECIMAL属性指示（例如， 0.00 ）。将ZSPELL设置为文本字符0 ，零值显示为0 ，没有小数位，无论DECIMAL规范如何。
                     

                     
ZSPELL对接近零值的影响

                     
当您的输出包含一个较小的数字（例如0.004 ，显示的小数位数会影响ZSPELL是否将该数字视为零。见例5-116 。
                     

                  

                  
                  

                     
例子

                     

                  

                  
                  

                     
示例5-115将零值显示为NONE

                     
此示例更改ZSPELL的值，以便DECIMAL变量testvar中的零值在报告输出中显示为NONE。当ZSPELL设置为其默认值OFF ，Oracle OLAP语句
testvar = 0.00 ROW testvar
产生以下输出。
0.00
相比之下，这些OLAP DML语句
ZSPELL ='无'ROW testvar
产生以下输出。
没有

                  
                  

                     
例5-116显示非常小的数字

                     
此示例说明了输出中显示的小数位数如何影响ZSPELL是否将非常小的数字视为零。当ZSPELL设置为其默认值OFF ，这些OLAP DML语句
ZSPELL ='OFF'testvar = 0.004 ROW DECIMAL 3 testvar
产生以下输出。
0.004
以下语句将ZSPELL设置为NONE并为输出指定两个小数位。
                     
ZSPELL ='NONE'行DECIMAL 2 testvar
这些语句产生以下输出。
没有
在ZSPELL仍设置为NONE ，以下语句指定输出的三个小数位。
                     
ROW DECIMAL 3 testvar
该语句产生以下输出。
0.004
