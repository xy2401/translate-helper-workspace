证明

SQL * Plus已针对特定于操作系统的Oracle数据库文档中列出的操作系统进行了认证。

SQL * Plus已通过Oracle Database 18c和所有受支持的Oracle Server版本的认证。

SQL * Plus版本19c中的新功能

本节介绍19c中SQL * Plus中引入的新功能。

新功能

  • 应用程序用于连接Oracle数据库的Easy Connect语法已得到增强，称为Easy Connect Plus。

不受支持的功能

  • 支持PRODUCT_USER_PROFILE表

    从Oracle Database 19c开始，不支持SQL * Plus表PRODUCT_USER_PROFILE（PUP表）。

    Oracle Database 19c中不提供SQL * Plus产品级安全功能。Oracle建议您使用Oracle数据库设置保护数据，以确保所有客户端应用程序的一致安全性。

SQL * Plus Release 18c，版本18.3中的新功能

此版本没有新功能。

不推荐的功能

  • PRODUCT_USER_PROFILE表的弃用

    从Oracle Database 18c开始，不推荐使用SQL * Plus表PRODUCT_USER_PROFILE （PUP）表。

    PRODUCT_USER_PROFILE （PUP）表的唯一用途是提供一种机制来控制SQL * Plus的产品级安全性。从Oracle Database 18c开始，此机制不再相关。Oracle Database 19c中将不提供此SQL * Plus产品级安全功能。Oracle建议您使用Oracle数据库设置保护数据，以确保所有客户端应用程序的一致安全性。

SQL * Plus Release 18c，版本18.1中的新功能

本节介绍18c中SQL * Plus中引入的新功能。

  • 显示查询的sql_id

    支持SET FEEDBACK命令以显示查询的sql_id。SET FEEDBACK命令的新SQL_ID选项显示当前执行的SQL或PL / SQL语句的sql_id。sql_id将分配给预定义变量_SQL_ID。

  • 设置查询显示的行数

    SET ROWLIMIT命令使用户可以为查询显示的行数限制。

  • 动态更改输出显示以适合屏幕大小

    SET LINESIZE命令有一个WINDOW选项，可根据屏幕的宽度和高度自动更改格式化输出的linesize和pagesize。

早期版本中的新功能

本节列出了以前版本中SQL * Plus中引入的新功能。

SQL * Plus 12.2.0.1.0生产中的新功能

SQL * Plus版本12.2.0.1是SQL * Plus版本12.1的超集。

本节介绍此版本的SQL * Plus中引入的新功能。某些功能可能受SQLPLUSCOMPATIBILITY设置的影响。请参阅SQL * Plus用户指南和参考中的第12章“SQL * Plus命令参考”中的SET SQLPLUSCOMPATIBILITY矩阵。

  • 支持命令历史记录

    命令历史记录功能使用户能够从当前会话的历史记录列表中运行，编辑或删除以前使用的SQL * Plus，SQL或PL / SQL命令。

  • 支持长标识符

    SQL * Plus支持128字节的对象长度。在以前的版本中，对象长度限制为30个字节。

  • 用于提高性能的命令

    SET LOBPREF [ETCH]，SET ROWPREF [ETCH]和SET STATEMENTC [ACHE]是新命令，使用户能够控制行和LOB预取以及语句缓存。

  • 新的行政特权

    SQLPLUS和CONNECT命令支持新用户权限SYSRAC以及现有的SYSBACKUP | SYSDG | SYSKM | SYSASM | SYSDBA | SYSOPER权限。

  • 支持输入绑定

    VARIABLE命令支持可在SQL和PL / SQL语句中使用的输入绑定。

  • 支持以CSV格式提取数据

    SET MARKUP命令支持以CSV格式显示数据。

  • 支持抑制查询返回的数据

    SET FEEDBACK命令的ONLY选项禁止显示查询返回的数据。显示查询选择和返回的行数。

  • 支持应用程序连续性

    SQL * Plus命令行选项-AC在会话中启用应用程序连续性。有关应用程序连续性的更多信息，请参阅Oracle Call Interface Programmer's Guide 。

  • 一般绩效改进

    SQL * Plus命令行选项-F [ast]可提高一般性能。

已知问题

错误24602051 - STARTUP OPEN RESTRICTED应在连接到应用程序容器根时返回语法错误。相反，它会导致PDB在正常模式下打开。

解决方法：使用STARTUP OPEN RESTRICT或ALTER PLUGGABLE DATABASE <pdb name> OPEN RESTRICTED。

错误修复

以下部分列出了SQL * Plus中修复的错误。问题描述后括号中的数字是指Oracle Bug数据库中的错误号。

SQL * Plus Release 18c，版本18.1中修复了错误

  • 使用STARTUP命令中的无效选项RESTRICTED将在连接到应用程序容器根目录时返回错误（24602051）。

  • 现在，SET FEEDBACK将具有与-F [ast]命令行选项（24642116）相同的设置。

  • 在描述DBMS包时，SQL * Plus不再崩溃（25097640）。

  • 现在，在返回不完整数据时，查询多字节字符集中的长数据列会显示正确的错误消息（25668186）。

  • SQL * Plus现在可以正确处理前导注释（17419163）。

  • SQL * Plus Instant Client不再泄漏内存（25127228）。

  • 将目录路径作为参数传递给文件现在可以正常工作（25699321）。

  • SQL * Plus现在在注册表中观察SQLPATH或在Windows上作为login.sql的环境变量（25804573）。

  • SQLPATH环境变量现在可以包含超过256个字符（25997107）。

  • DBMS_OUTPUT.PUT_LINE现在正确返回值（26163790）。

  • 内部修复（25361890,25390622,25475326,25475334）

在SQL * Plus 12.2.0.1.0生产中修复的错误

  • 使用Password命令时指定包含双引号（“）的用户名时，SQL * Plus不再抛出ORA-1005错误（17613757）

  • 如果在命令行中指定的“对象”的名称大于46个字符，则在编译包含错误的包体时SQL * Plus不再崩溃（18389912）

  • 长列名称不再显示为表格描述中的垃圾字符（20453052）

  • 在更改会话的NLS_SORT参数后使用“force”选项创建视图不再导致崩溃（20592522）

  • 内部修复（17830524,17847776,18925904,19655924,19674161,19953173,21609521）

SQL * Plus 12.1版中修复了错误

  • SQL * Plus现在正确执行包含WHENEVER SQLERROR EXIT子句的SQL脚本（10375866）

  • SPERRORLOG现在显示整个声明（9559937）

  • SQL * Plus命令现在可以在PRODUCT_USER_PROFILE（9298298）中正确显示

  • 在会话失效后，SHUTDOWN IMMEDIATE不再挂起（8299200）

  • INSERT现在可以正确报告插入的所有行（7047565）

  • 内部修复（9587813,9575131,8832471,8670263,8244438,7629280,6862613）
