4.109 DBA_AUDIT_MGMT_CLEAN_EVENTS

DBA_AUDIT_MGMT_CLEAN_EVENTS显示有关审计跟踪清除或清除事件历史记录的信息。

您应定期删除此视图的内容，以使其不会变得太大。

注意：

此视图仅用于传统审计（Oracle数据库12c之前的审计），不适用于统一审计。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

柱 数据类型 空值 描述

AUDIT_TRAIL

VARCHAR2（28）

在活动时清理的审计跟踪：

  • 标准审计跟踪

  • FGA审计跟踪

  • 标准和FGA审计跟踪

  • OS AUDIT TRAIL

  • XML审计跟踪

  • 操作系统和XML审计跟踪

  • 所有审计跟踪

RAC_INSTANCE

数

不是NULL

实例编号，指示在事件发生时已清理的Oracle RAC实例; 0意味着不适用

CLEANUP_TIME

TIMESTAMP（6）带时区

清理事件完成时的时间戳

DELETE_COUNT

数

在事件发生时删除的审计记录或审计文件的数量

被迫

VARCHAR2（3）

指示是否发生强制清理（ YES ）或否（ NO ）;强制清除会绕过上一个存档时间戳集

注意：

在只读数据库（包括Oracle Active Data Guard物理备用数据库）中，不会填充此视图。而是将清理事件的摘要写入相应的数据库实例的警报日志文件。
