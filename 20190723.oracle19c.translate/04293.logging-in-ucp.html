12.4 UCP日志记录概述

UCP利用JDK日志记录工具（ java.util.logging ）。默认情况下不启用日志记录，必须配置日志记录才能打印日志消息。可以使用日志配置文件以及API级配置来配置日志记录。

注意：

默认日志级别为null 。这可确保默认情况下使用父记录器的日志级别。

12.4.1使用日志属性文件

可以使用属性文件配置日志记录。必须将属性文件的位置设置为日志记录配置文件属性的Java属性。例如：

java -Djava.util.logging.config.file = log.properties

日志记录属性文件定义用于写入日志的处理程序，用于格式化日志的格式化程序，默认日志级别以及特定包或类的日志级别。例如：

handlers = java.util.logging。ConsoleHandler java.util.logging。ConsoleHandler.level =所有java.util.logging。ConsoleHandler.formatter = java.util.logging。SimpleFormatter oracle.ucp.level = FINEST oracle.ucp.jdbc。PoolDataSource =警告

UCP包含自定义格式化程序，可以输入格式化程序属性的值。例如：

java.util.logging的。ConsoleHandler.formatter = oracle.ucp.util.logging。UCPFormatter

您还可以从Oracle Technology Network（OTN）下载UCP随附的ucpdemos.jar文件。此文件包含示例日志记录属性文件的列表。例如，此文件包含可用于对快速连接故障转移（FCF）功能进行故障排除的日志记录属性文件。

12.4.2使用UCP和JDK API

可以通过UCP API或JDK API动态配置日志记录。使用UCP API时，使用连接池管理器配置日志记录。使用JDK时，使用java.util.logging实现配置日志记录。

以下示例演示如何使用UCP API配置日志记录：

UniversalConnectionPoolManager mgr = UniversalConnectionPoolManagerImpl。getUniversalConnectionPoolManager（）; mgr.setLogLevel（级别。精细）;

以下示例演示如何直接使用JDK日志记录实现：

Logger.getLogger（ “oracle.ucp”）。setLevel（级别。FINEST）; Logger.getLogger（“oracle.ucp.jdbc。PoolDataSource“）。setLevel（级别。FINEST）;

12.4.3在运行时启用或禁用特定于功能的日志记录

从Oracle Database 12 c第2版（12.2.0.1）开始，UCP支持在运行时启用和禁用所选功能的日志记录。例如，您可以仅为负载平衡功能启用日志记录，同时禁用UCP其他功能的日志记录。同样，在同一运行期间，您可以启用快速故障转移功能的日志记录并禁用负载平衡功能的日志记录。

默认情况下，启用所有功能的日志记录。

UCP的日志切换功能是OracleDiagnosabilityMBean的一部分。要使用此bean，请启动JConsole并连接到该应用程序。

显示支持的功能

从Oracle Database 12 c第2版（12.2.0.1）开始，您可以通过在日志记录属性文件中添加属性来启用或禁用特定功能的日志记录。默认情况下，为所有功能启用日志记录。否则，您可以使用以下语法为所有功能启用日志记录： clio.feature.all = on 。对于特定于功能的日志记录启用，您可以使用以下部分中提到的属性：

12.4.5支持的日志级别

以下列表描述了JDBC支持的每个日志级别。低于FINE级别产生的输出可能对用户没有意义。低于FINER水平将产生非常大量的输出。

  • INTERNAL_ERROR - 内部错误

  • SEVERE - SQL异常

  • WARNING - SQL警告和其他不可见的问题

  • INFO - 公共事件，例如连接尝试或Oracle RAC事件

  • CONFIG - SQL语句

  • FINE - 公共API

  • TRACE_10 - 内部事件

  • FINER - 内部API

  • TRACE_20 - 内部调试

  • TRACE_30 - 高容量内部API

  • FINEST - 高容量内部调试
