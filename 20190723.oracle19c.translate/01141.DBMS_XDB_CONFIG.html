188.1 DBMS_XDB_CONFIG概述

PL / SQL包DBMS_XDB_CONFIG是用于DBA的PL / SQL的Oracle XML DB资源应用程序接口（API），用于配置其系统。此API提供了使用PL / SQL访问和管理Oracle XML DB存储库资源的功能和过程。它包括管理资源安全性和Oracle XML DB配置的方法。

Oracle XML DB Repository以XML为模型，为任何数据提供数据库文件系统。存储库将路径名（或URL）映射到XMLType的数据库对象，并为这些对象提供管理功能。

PL / SQL包DBMS_XDB_CONFIG是一个可用于管理以下所有内容的API：

  • Oracle XML DB资源

  • 基于访问控制列表（ACL）的Oracle XML DB安全性。ACL是访问控制条目（ACE）的列表，用于确定哪些主体（用户和角色）可以访问哪些资源。

  • Oracle XML DB配置

188.2 DBMS_XDB_CONFIG安全模型

由XDB拥有， DBMS_XDB_CONFIG包必须由SYS或XDB创建。 EXECUTE特权被授予PUBLIC 。此包中的子程序使用当前用户的权限执行。

188.4.28 SETGLOBALPORTENABLED程序

此过程使Oracle Enterprise Manager Database Express（EM Express）客户端可以为会话使用单个端口（称为全局端口），而不是使用专用于PDB的端口。

句法

DBMS_XDB_CONFIG.SETGLOBALPORTENABLED（isenableable in BOOLEAN）;

参数

表188-20 SETGLOBALPORTENABLED过程参数

参数 描述

的IsEnabled

TRUE使EM Express客户端可以使用全局端口进行会话，而不是使用分配端口的PDB。

FALSE使端口ID能够确定要使用的PDB。

对于PDB，默认值为TRUE ，对于CDB根，默认值为FALSE 。

使用说明

  • 如果使用EM Express管理多租户环境中的数据库端口，请使用DBMS_XDB_CONFIG.SETGLOBALPORTENABLED过程。

  • 要使用全局端口，必须在CDB和PDB中启用它。 PDB默认启用它。对于CDB，您必须在CDB根目录中运行DBMS_XDB_CONFIG.SETGLOBALPORTENABLED过程。

  • 将DBMS_XDB_CONFIG.SETGLOBALPORTENABLED设置为TRUE ，必须指定用于全局端口的PDB。您可以通过在EM Express登录页面的“ 容器名称”字段中指定PDB名称来执行此操作。例如，如果输入hr_pdb ，则EM Express URL将变为http:// server_name :5500/hr_pdb/em 。此示例使用CDB根目录中的端口5500 ，后者又将消息路由到PDB hr_pdb的EM Express。端口可以是HTTP或HTTPS端口。

  • 您在EM Express登录页面中指定的PDB必须以读/写模式打开，然后才能运行DBMS_XDB_CONFIG.SETGLOBALPORTENABLED 。您可以通过连接到PDB然后查询V$PDBS动态视图的OPEN_MODE列来检查PDB的读/写模式状态。

  • DBMS_XDB_CONFIG.ISGLOBALPORTENABLED函数返回DBMS_XDB_CONFIG.SETGLOBALPORTENABLED设置的值。

  • 您可以使用XDB_PROTOCOLS锁定配置文件来控制全局端口的安全性。
