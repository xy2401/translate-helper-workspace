创建CONTROLFILE

注意：

Oracle建议您在使用此语句之前对数据库中的所有文件执行完整备份。有关更多信息，请参见“ Oracle数据库备份和恢复用户指南” 。

目的

只应在少数情况下使用CREATE CONTROLFILE语句。此语句如果正在使用的所有控制文件由数据库丢失，并且没有备份控制文件存在重新创建一个控制文件。您还可以使用此语句更改重做日志文件组，重做日志文件成员，归档重做日志文件，数据文件或可以同时装入和打开数据库的实例的最大数量。

要更改数据库的名称，Oracle建议您使用DBNEWID实用程序而不是CREATE CONTROLFILE语句。DBNEWID是首选，因为更改数据库名称后不需要OPEN RESETLOGS操作。

  • 有关DBNEWID实用程序的详细信息，请DBNEWID Oracle数据库实用程序

  • ALTER DATABASE “ BACKUP CONTROLFILE子句 ”，用于根据现有数据库控制文件创建脚本的信息

先决条件

要创建控制文件，您必须具有SYSDBA或SYSBACKUP系统特权。

任何实例都不能装入数据库。成功创建控制文件后，Oracle以CLUSTER_DATABASE参数指定的模式安装数据库。然后，DBA必须在打开数据库之前执行介质恢复。如果您将数据库与Oracle Real Application Clusters（Oracle RAC）一起使用，则必须先关闭并以SHARED模式重新装入数据库（通过将CLUSTER_DATABASE初始化参数的值设置为TRUE ），然后才能启动其他实例。

语义

发出CREATE CONTROLFILE语句时，Oracle数据库会根据您在语句中指定的信息创建新的控制文件。控制文件位于CONTROL_FILES初始化参数中指定的位置。如果该参数没有值，则数据库在默认控制文件目标中创建Oracle管理的控制文件，该文件是以下之一（按优先顺序排列）：

  1. DB_CREATE_ONLINE_LOG_DEST_ n初始化参数中指定的一个或多个控制文件。第一个目录中的文件是主控制文件。指定DB_CREATE_ONLINE_LOG_DEST_ n ，数据库不会在DB_CREATE_FILE_DEST或DB_RECOVERY_FILE_DEST （快速恢复区域）中创建控制文件。

  2. 如果没有为DB_CREATE_ONLINE_LOG_DEST_ n指定值，但为DB_CREATE_FILE_DEST和DB_RECOVERY_FILE_DEST设置了值，则数据库会在每个位置创建一个控制文件。DB_CREATE_FILE_DEST指定的位置是主控制文件。

  3. 如果仅为DB_CREATE_FILE_DEST指定了值，则数据库将在该位置创建一个控制文件。

  4. 如果仅为DB_RECOVERY_FILE_DEST指定了值，则数据库将在该位置创建一个控制文件。

如果没有为这些参数设置任何值，则数据库会在运行数据库的操作系统的默认位置创建控制文件。此控制文件不是Oracle托管文件。

如果省略任何子句，则Oracle数据库使用默认值而不是上一个控制文件的值。成功创建控制文件后，Oracle Database以初始化参数CLUSTER_DATABASE指定的模式安装数据库。如果未设置该参数，则默认值为FALSE ，并且数据库以EXCLUSIVE模式挂载。Oracle建议您关闭实例并对数据库中的所有文件进行完整备份。

REUSE

指定REUSE以指示可以重用由初始化参数CONTROL_FILES标识的现有控制文件，覆盖它们当前可能包含的任何信息。如果省略此子句并且已存在任何这些控制文件，则Oracle数据库将返回错误。

数据库条款

指定数据库的名称。此参数的值必须是先前CREATE DATABASE语句或CREATE CONTROLFILE语句建立的现有数据库名称。

SET DATABASE子句

使用SET DATABASE更改数据库的名称。数据库的名称可以长达8个字节。

指定此子句时，还必须指定RESETLOGS 。如果要重命名数据库并保留现有日志文件，则在发出此CREATE CONTROLFILE语句后，必须使用ALTER DATABASE RECOVER USING BACKUP CONTROLFILE语句完成完整数据库恢复。

logfile_clause

使用logfile_clause指定数据库的重做日志文件。您必须列出所有重做日志文件组的所有成员。

使用file_specification的redo_log_file_spec形式（请参阅file_specification ）列出操作系统文件系统中的常规重做日志文件或列出Oracle ASM磁盘组重做日志文件。使用ASM_filename形式时，不能指定redo_log_file_spec的autoextend_clause 。

如果在此子句中指定RESETLOGS ，则必须使用ASM_filename的文件创建形式之一。如果指定NORESETLOGS ，则必须指定ASM_filename的引用形式之一。

GROUP 整数

指定日志文件组编号。如果指定GROUP值，则Oracle Database会在数据库上次打开时使用GROUP值验证这些值。

如果省略此子句，则数据库使用系统默认值创建日志文件。此外，如果已设置DB_CREATE_ONLINE_LOG_DEST_ n或DB_CREATE_FILE_DEST初始化参数，并且已指定RESETLOGS ，则数据库将在DB_CREATE_ONLINE_LOG_DEST_ n参数中指定的缺省日志文件目标中创建两个日志，如果未设置，则在DB_CREATE_FILE_DEST参数。

RESETLOGS

如果希望Oracle数据库忽略LOGFILE子句中列出的文件的内容，请指定RESETLOGS 。这些文件不一定存在。如果已指定SET DATABASE子句，则必须指定此子句。

LOGFILE子句中的每个redo_log_file_spec都必须指定SIZE参数。数据库将所有联机重做日志文件组分配给线程1，并使该线程可供任何实例公开使用。使用此子句后，必须使用ALTER DATABASE语句的RESETLOGS子句打开数据库。

NORESETLOGS

如果希望Oracle数据库使用LOGFILE子句中的所有文件，就像上次打开数据库时一样，请指定NORESETLOGS 。这些文件必须存在，并且必须是当前的联机重做日志文件，而不是已还原的备份。数据库将重做日志文件组重新分配给先前分配给它们的线程，并重新启用先前启用的线程。

如果已指定SET DATABASE子句来更改数据库的名称，则无法指定NORESETLOGS 。有关更多信息，请参阅“ SET DATABASE子句 ” 。

DATAFILE子句

指定数据库的数据文件。您必须列出所有数据文件。这些文件必须全部存在，但它们可能是需要介质恢复的还原备份。

不要在DATAFILE子句中包含只读表空间中的任何数据文件。您可以稍后将这些类型的文件添加到数据库中。此外，请勿在此子句中包含任何临时数据文件（临时文件）。

使用file_specification的datafile_tempfile_spec形式（请参阅file_specification ）列出操作系统文件系统中的常规数据文件和临时文件，或列出Oracle ASM磁盘组文件。使用ASM_filename形式时，必须使用ASM_filename的引用形式之一。有关不同语法形式的信息，请参阅ASM_filename 。

对DATAFILE的限制

您无法在此DATAFILE子句中指定file_specification的autoextend_clause 。

MAXLOGFILES条款

指定可以为数据库创建的最大联机重做日志文件组数。Oracle数据库使用此值来确定在控制文件中为重做日志文件的名称分配多少空间。默认值和最大值取决于您的操作系统。您指定的值不应小于任何重做日志文件组的最大GROUP值。

MAXLOGMEMBERS条款

指定重做日志文件组的最大成员数或相同副本。Oracle数据库使用此值来确定在控制文件中为重做日志文件的名称分配多少空间。最小值为1。最大值和默认值取决于您的操作系统。

MAXLOGHISTORY条款

仅当您在ARCHIVELOG模式下使用Oracle数据库时，此参数才有用。指定数据库自动介质恢复所需的最大归档重做日志文件组的当前估计值。数据库使用此值来确定在控制文件中为归档重做日志文件的名称分配多少空间。

最小值为0。默认值是MAXINSTANCES值的倍数，取决于您的操作系统。最大值仅受控制文件的最大大小限制。数据库将根据需要继续向控制文件的相应部分添加额外空间，这样，如果原始配置不再适用，则无需重新创建控制文件。因此，此参数的实际值最终可能会超出您指定的值。

MAXDATAFILES条款

在CREATE DATABASE或CREATE CONTROLFILE时指定控制文件的数据文件部分的初始大小。尝试添加数量大于MAXDATAFILES但小于或等于DB_FILES的文件会导致控制文件自动扩展，以便数据文件部分可以容纳更多文件。

实例可访问的数据文件数也受初始化参数DB_FILES 。

最大条款

指定可以同时装入和打开数据库的最大实例数。该值优先于初始化参数INSTANCES的值。最小值为1。最大值和默认值取决于您的操作系统。

ARCHIVELOG | NOARCHIVELOG

指定ARCHIVELOG以在重用之前归档重做日志文件的内容。本节准备介质恢复以及实例或系统故障恢复的可能性。

如果省略ARCHIVELOG子句和NOARCHIVELOG子句，则Oracle数据库默认选择NOARCHIVELOG模式。创建控制文件后，可以使用ALTER DATABASE语句在ARCHIVELOG模式和NOARCHIVELOG模式之间进行ALTER 。

强制记录

在创建控制文件后，使用此子句将数据库置于FORCE LOGGING模式。当数据库处于此模式时，Oracle数据库会记录数据库中的所有更改，但临时表空间和临时段的更改除外。此设置优先于您为各个表空间指定的任何NOLOGGING或FORCE LOGGING设置，并且与您为各个数据库对象指定的任何NOLOGGING设置无关。如果省略此子句，则在创建控制文件后，数据库将不会处于FORCE LOGGING模式。

注意：

FORCE LOGGING模式可以产生性能影响。有关何时使用此设置的信息，请参阅Oracle数据库管理员指南 。

为数据可用性设置备用数据|负载性能

SET STANDBY NOLOGGING

SET STANDBY NOLOGGING禁用备用数据库的日志记录。您可以在两种模式中指定它：

  • SET STANDBY NOLOGGING FOR DATA AVAILABILITY保证完整数据复制到备用数据库。主数据库和备用数据库在加载期间同步。在网络拥塞的情况下，主数据库将限制其负载。

  • 为备份性能设置STANDBY NOLOGGING以保持主数据库负载的速度并在以后与备用数据库同步。

SET STANDBY NOLOGGING的限制

SET STANDBY NOLOGGING子句不能与FORCE LOGGING 。

character_set_clause

如果指定了字符集，则Oracle Database会重新构建控制文件中的字符集信息。如果随后需要对数据库进行介质恢复，则在打开数据库之前，此信息将可用，以便在恢复期间正确解释表空间名称。仅当您使用的默认字符集取决于您的操作系统时，才需要此子句。Oracle Database在启动期间将当前数据库字符集打印到$ORACLE_HOME/log中的警报日志。

如果要重新创建控制文件并使用Recovery Manager进行表空间恢复，并且如果从数据字典中存储的字符集指定其他字符集，则表空间恢复将不会成功。但是，在数据库打开时，控制文件字符集将使用数据字典中的正确字符集进行更新。

您无法使用此子句修改数据库的字符集。
