检查Linux的内核和程序包要求

验证内核和软件包，看它们是否满足最低安装要求。

  1. 要确定已安装的Linux的分发版和版本，请输入以下命令之一：
    #cat / etc / oracle-release #cat / etc / redhat-release #cat / etc / suSE-release＃lsb_release -id
  2. 要确定是否安装了所需的内核勘误表，请输入以下命令：
    #uname -r

    以下是此命令在Oracle Linux 7系统上显示的输出示例：

    4.1.12-112.16.7.el7uek.x86_64

    查看分发所需的勘误级别。如果勘误级别在所需的最小勘误更新之前，则从Linux经销商处获取并安装最新的内核更新。

  3. 要确定是否已安装所需的软件包，请输入与以下内容类似的命令：

    #rpm -q package_name

    或者，如果您需要特定的系统体系结构信息，请输入以下命令：

    #rpm -qa --queryformat“％{NAME} - ％{VERSION} - ％{RELEASE}（％{ARCH}）\ n”| grep package_name

    您还可以组合多个包的查询，并查看输出以获取正确的版本。例如：

    #rpm -q binutils compat-libstdc ++ gcc glibc libaio libgcc libstdc ++ \ make sysstat unixodbc
    如果未安装软件包，请从Linux分发介质安装软件包，或从Linux分发服务器的网站下载所需的软件包版本。
