1.2.5 Memory Guard简介

Memory Guard是一个Oracle Real Application Clusters（Oracle RAC）环境功能，用于监控集群节点，以防止由于内存不足而导致的节点压力。

默认启用，Memory Guard：

  • 每分钟一次分析过度提交的内存条件

  • 如果任何服务器存在风险，则会发出警报

  • 通过自动关闭服务器到新连接来保护应用程序

  • 在服务器上以事务方式停止所有CRS管理的服务

  • 一旦内存压力消退，重新打开服务器连接

由于过多的打开会话或失控的工作负载，企业数据库服务器可以使用所有可用内存。内存不足可能导致事务失败，或者在极端情况下，导致节点重新启动并导致应用程序资源可用性降低。

Memory Guard自动从Cluster Health Monitor收集每个节点内存的度量标准，以确定节点是否内存不足。如果内存不足，则Memory Guard会阻止创建新的数据库会话，从而允许现有工作负载完成并释放内存。当缓解内存压力时，会自动启动新会话。
