10为独立服务器安装和配置Oracle Grid Infrastructure

用于独立服务器的Oracle Grid Infrastructure包括Oracle Restart和Oracle自动存储管理。

如果为独立服务器安装Oracle Grid Infrastructure，然后创建数据库，则会自动将数据库添加到Oracle Restart配置中。Oracle Restart会在需要时自动重新启动数据库。

如果在已存在数据库的主机上为独立服务器安装Oracle Grid Infrastructure，则必须先手动将数据库，侦听器，Oracle ASM实例和其他组件添加到Oracle Restart配置中，然后才能够配置自动数据库重新启动。

注意：

独立服务器的Oracle Grid Infrastructure可以在单台主机上支持多个单实例数据库。

  • 关于基于映像的Oracle Grid Infrastructure安装
    从Oracle Grid Infrastructure 12c第2版（12.2）开始，通过基于映像的安装简化了Oracle Grid Infrastructure软件的安装和配置。
  • 安装向导用于创建映像的安装选项
    在启动Oracle数据库或Oracle Grid Infrastructure安装的安装向导之前，请确定是否要使用任何可用的映像创建选项。
  • 使用新数据库安装为独立服务器安装Oracle Grid Infrastructure
    完成以下步骤以为独立服务器安装Oracle Grid Infrastructure，然后创建由Oracle Restart管理的数据库。
  • 为现有数据库的独立服务器安装Oracle Grid Infrastructure
    按照本节中的高级说明为独立服务器安装Oracle Grid Infrastructure，并为现有Oracle数据库配置它。
  • 使用纯软件安装为独立服务器安装Oracle Grid Infrastructure
    纯软件安装仅为指定位置的独立服务器二进制文件安装Oracle Grid Infrastructure。安装软件后，必须完成一些手动配置步骤以启用Oracle Restart。
  • 测试Oracle自动存储管理安装
    在为单个实例安装Oracle Grid Infrastructure之后，使用ASMCMD命令行实用程序来测试Oracle ASM安装。
  • 安装后为独立服务器二进制文件修改Oracle Grid Infrastructure
    安装后，如果不使用带有opatchauto标志的opatchauto修补二进制文件，则必须停止堆栈，修改软件，然后重新启动堆栈。
  • 使用Oracle ASMCA手动配置Oracle ASM磁盘组
    在为独立服务器安装Oracle Grid Infrastructure之后，您还可以使用Oracle ASMCA创建和配置磁盘组，Oracle ADVM和Oracle ACFS。
  • 在Oracle重新启动配置上启用Oracle ACFS
    您必须启用root访问权才能使用Oracle ACFS。
  • 在Oracle Grid Infrastructure安装或升级期间应用修补程序
    从Oracle Grid Infrastructure 18c开始，您可以在Oracle Grid Infrastructure安装或升级期间下载并应用版本更新（RU）和一次性补丁。
