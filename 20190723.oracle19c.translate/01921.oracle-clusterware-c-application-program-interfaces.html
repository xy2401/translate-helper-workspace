概观

CLSCRS是一组用于Oracle Clusterware的基于C的API。CLSCRS API使您可以管理由Oracle Clusterware管理的实体的操作。这些实体包括资源，资源类型，服务器和服务器池。您可以使用API向Oracle Clusterware注册用户应用程序，以便集群件可以管理它们并保持高可用性。注册应用程序后，您可以对其进行管理并查询应用程序的状态。如果您不再需要该应用程序，则可以将其停止并从Oracle Clusterware取消注册。

Oracle Clusterware服务由作为Oracle Clusterware一部分运行的Cluster Ready Services提供。CLSCRS API使用在所有函数调用中显式命名的上下文。API不会在进程或线程级别存储任何内容。您可以使用回调进行诊断日志记录。

注意：

您可以从Oracle数据库客户端安装介质安装Oracle Clusterware高可用性API。

上下文初始化和持久化

要使用CLSCRS API，必须首先初始化clscrs上下文。创建和终止此上下文的调用是：

  • clscrs_init_crs ：初始化clscrs上下文

  • clscrs_term_crs ：终止clscrs上下文

调用者负责在不再需要时终止上下文。

线程支持

如果使用CLSCRS_FLAG_USETHREADS标志初始化，则CLSCRS API可能在内部生成线程。每个API函数都在调用线程的上下文中执行。多个线程可能不会同时使用API上下文对象。但是，客户端不需要线程关联。进程可以创建多个API上下文并在不同的线程上使用这些上下文，遵循一次一个线程的每个上下文一次的规则。

CLSCRS API数据结构

以下实体将传递给API调用并包含API调用的返回值：

  • clscrs_sp ： stringpair （ sp ）包含名称和值字符串。该值可以为NULL。它被创建和销毁，可以检查其内容并替换值。stringpair可以是一个stringpair列表（ splist ）的成员。

  • clscrs_splist ：字符串对列表（ splist ）是在各种上下文中使用的零个或多个字符串对的列表。API可以将字符串对添加到字符串对列表中或从字符串对列表中删除它们，或者API可以迭代字符串对。

  • clscrs_entity_type ：可用于标识Oracle Clusterware实体类型的枚举类型。

    枚举类型定义如下：

    / *枚举指定实体类型* / typedef enum {clscrs_entity_res = 1，/ * resource * / clscrs_entity_restype = 2，/ *资源类型* / clscrs_entity_serverpool = 3，/ *服务器池* / clscrs_entity_server = 4，/ *服务器* / clscrs_entity_resinst = 5，/ * resource instances * / clscrs_entity_config_policy = 6，/ * configuration policy * / clscrs_entity_config_policyset = 7，/ * configuration policy set * / clscrs_entity_server_category = 8 / * server category * /} clscrs_entity_type;

  • clscrs_crsentity ：此数据结构（ crsentity ）表示Oracle Clusterware实体，其中包含适用于使用Oracle Clusterware实体的上下文的名称和附加数据。有时，API包含Oracle Clusterware实体属性数据，有时它包含状态并返回有关操作的消息。单个实体可以是一个clscrs_crsentitylist的成员。

  • clscrs_crsentitylist ：实体列表（ crsentitylist ）是一个包含零个或多个clscrs_crsentity实例的数据结构。API可以向实体列表添加实体或从实体列表中删除实体，或者API可以迭代实体。

内存管理

CLSCRS API适用于元素和列表。元素将添加到列表中。元素和列表的内存通过显式API调用进行分配和释放。调用者有责任释放他们分配的内存。但是，当元素添加到列表时，只有列表必须被销毁：列表的销毁会隐式地销毁其元素。如果元素未添加到任何列表中，则必须销毁这些元素。对于递归列表，销毁父列表也会销毁其中包含的任何列表。clscrs_sp和clscrs_crsentity元素必须由调用者销毁。如果它们是clscrs_splist或clscrs_crsentitylist ，则销毁该列表会破坏相应的clscrs_sp和clscrs_crsentity实体。

例如，创建资源并将其添加到资源列表时，只能销毁资源列表，而不能销毁单个资源。销毁资源列表也会释放单个资源的内存。

错误处理和跟踪

交互式和非交互式CLSCRS API均使用不同的错误处理机制。

对于非交互式CLSCRS API，错误代码将作为函数调用的返回值返回。例如：

clscrsret clscrs_sp_get_value（clscrs_sp * sp，oratext ** value）;

错误代码作为clscrsret值返回。

对于交互式CLSCRS API，将显示输出结果，如下所示：

  1. 函数调用的返回值提供请求的高级输出。请求是否到达服务器？是完全成功，还是完全或部分不成功？成功的返回值意味着已收到，处理请求，并且所请求的所有实体的结果都是成功的。

  2. 对于请求运行的每个实体， op_status列表中存储有编程完成代码。如果值success ，则表示特定于处理特定对象请求的问题的高级类型。

  3. 可选地，API可以通过使用回调机制指示它想要接收本地化的，人类可读的错误，警告或状态消息。每次调用回调都会提供消息，消息类型（严重性）以及回调调用所属对象的ID。

例如：

CLSCRS_STAT clscrs_register_resource2（clscrs_crsentitylist * in_crsentitylist，uword flags，clscrs_msgf2 msgf，void * msgarg，clscrs_crsentitylist * op_status）;

  1. 该函数返回值为CLSCRS_STAT的错误代码。

  2. CRSD通过clscrs_msgf2回调将错误消息，警告消息和进度消息发送回客户端。客户端必须实现回调以处理CRSD返回的这些消息。

  3. 在以前的Oracle Clusterware版本中，API还包含Oracle Clusterware实体上每个操作的结果，作为op_status列表的一部分。您可以使用以下API访问该信息：

    clscrsret clscrs_entity_get_op_status（clscrs_entity * res，CLSCRS_STAT * status，oratext ** msg）;

    status参数包含有关Oracle Clusterware实体上的CRSD操作的状态代码。此外， msg参数包含来自CRSD的有关操作结果的消息。虽然op_status列表继续包含msg参数中每个Oracle Clusterware实体的CRSD操作的结果，但是现在已经不再使用msg参数来获取错误代码和消息，并且不支持任何使用API的API一个新的实体。仅支持预先存在的用例（特别是用于资源）。使用回调函数处理CRSD返回的任何消息。

回调机制

交互式CLSCRS API提供了一种回调机制，客户端可以使用该机制来处理CRSD发送的错误消息，警告消息和进度消息。

回调机制的签名是：

typedef void（* clscrs_msgf2）（void * usrp，const oratext * id，const oratext * msg，clscrs_msgtype msgtype）;

在上面的语法中：

  • usrp ：用户提供的指针，可能包含调用的上下文

  • id ：消息对应的实体的标识符

  • msg ：是输出文本

  • msgtype ：是消息的类型;错误，警告或进展

示例I-1回调机制

过滤器

您可以使用过滤器来缩小CLSCRS API运行的Oracle Clusterware实体的范围。简单过滤器是具有运算符的属性 - 值对。操作符必须用空格包围，如示例所示。您可以使用布尔运算符将简单过滤器组合到称为表达式过滤器

支持的过滤运算符是：

  • =
  • >
  • <
  • ！=
  • co ：包含
  • st ：从开始
  • en ：结束

支持的布尔运算符是AND和OR 。

过滤器的示例如下：

  • TYPE = type1

  • （（TYPE = type1）AND（CHECK_INTERVAL> 50））

  • （TYPE = type1）AND（（CHECK_INTERVAL> 30）OR（AUTO_START co never））

  • NAME en network.res

  • TYPE st ora.db

脚本代理使用

在脚本代理入口点中使用CLSCRS API时，请记住以下几点：

  1. 某些操作（如启动，停止和清除）是在资源实例上的锁定下执行的。因此，向服务器发出直接或通过扩展关系对资源起作用的请求导致死锁。

  2. 发出只读（ clscrs_stat2 ）通常是安全的，除非它是初始检查，其中脚本代理不能在服务器上回调，因为这也会导致死锁。使用clsagfw API查询检查入口点。

对现有CLSCRS API的更改

Oracle已将以下标志添加到 clscrs_stat2 API：

  • CLSCRS_STATFLAG_SERVERBYCATEGORY ：使用此标志查询与特定服务器类别匹配的服务器。

  • CLSCRS_STATFLAG_CATEGORYBYSERVER ：使用此标志查询与特定服务器匹配的服务器类别。

Oracle已将以下标志添加到clscrs_start_resource2和clscrs_stop_resource2 API：

  • CLSCRS_FLAG_T_HA_PREPARE ：将此标志与透明HA（ start2 / stop2 API）一起使用，以指示Oracle Clusterware后台驻留程序（CRSD）准备外部管理接口以对资源执行操作。当调用返回时，如果成功，则接口应该启动或停止资源并使用另一个标志（ CLSCRS_FLAG_T_HA_FINISH ） CLSCRS_FLAG_T_HA_FINISH 。

  • CLSCRS_FLAG_T_HA_FINISH ：将此标志与透明HA一起使用CLSCRS_FLAG_T_HA_PREPARE 。启动或停止操作完成后，必须在第二次调用CRSD（start2 / stop2 API）时使用此标志。请注意，客户端必须指明资源实例，而不是具有此标志的资源，因为CRSD必须知道API的调用适用于哪些实例。

  • CLSCRS_NO_STATE_RESTORATION ：此标志可用于启动和停止资源API，并指示CRSD跳过状态受影响的资源的资源状态恢复。除非您使用此标志，否则通常会尝试该过程。

非交互式CLSCRS API

您可以将非交互式CLSCRS API用于上下文初始化，为交互式API准备请求有效负载以及交互式API的后处理输出等功能。非交互式CLSCRS API不会调用CRSD。

回调错误报告机制不适用于非交互式CLSCRS API。除clscrs_stat2 ， clscrs_stat3和所有clscrs_whatif_* API之外的所有交互式CLSCRS API都使用此回调机制。这些API的客户端还使用回调机制从CRSD接收错误，警告和进度消息。

您还可以使用过滤器来减少Oracle Clusterware实体列表。您还可以在交互式API中使用过滤器来减少Oracle Clusterware实体的列表。

命令评估API

您可以使用命令评估API来预测Oracle Clusterware对假设的计划或计划外事件的响应。

Oracle Clusterware可以对系统中的事件做出反应并生成响应行动计划。此操作计划包含一系列资源状态转换或服务器池重新配置，或两者。命令评估API提供了一种机制，用于在外部公开此操作计划，并使客户端能够在事件发生之前预测事件的后果。

此外，通过为后续部分中列出的API指定标志，您可以获得合理的命令评估信息，该信息解释了Oracle Clusterware为此形成事件响应行动计划所做出的决策的原因。

命令评估响应计划适用于以下事件类别：

  • 资源 ：启动，停止，重定位，添加和修改

  • 服务器池 ：添加，删除和修改

  • 服务器 ：添加，删除和重定位

  • 政策 ：更改有效政策

  • 服务器类别 ：修改

合理的命令评估响应计划适用于以下事件类别：

  • 服务器池 ：添加，删除和修改

  • 服务器 ：添加和删除

  • 政策 ：更改有效政策

Oracle Clusterware提供命令评估输出作为操作列表，其中每个操作代表Oracle Clusterware执行的特定步骤。每个操作都由clscrs_action结构封装， clscrs_actionlist结构表示整个操作序列。Oracle集群件还提供了一组函数（ clscrs_action_*行动结构和clscrs_actionlist_*行动列表）来创建，查看，遍历，并破坏这些结构。它们的用法与相应的实体列表和实体函数的用法相同。

命令评估API还使客户端能够查询系统的预计状态。使用stat3格式的clscrs_querylist结构指定查询， clscrs_crsentitylist结构提供结果。有关其用法的详细信息，请参阅stat3部分。

Oracle Clusterware提供的每个命令评估响应都包含一个序列ID，它指示Oracle Clusterware的当前状态。对于Oracle Clusterware管理的每个新事件，序列ID都会递增。Oracle Clusterware保证，只要序列ID没有改变，就会对所涉及的事件执行所提供的行动计划。例如，Oracle Clusterware为whatif start resource FOO请求提供的操作计划将与Oracle Clusterware在提交实际start resource FOO请求时采取的操作相同，前提是序列ID未更改。

STAT3 API

Oracle Clusterware 11g第 2版（11.2）管理多个实体，例如资源，服务器池等。但是，该版本中的接口仅允许按类型检索（读取）实体，这意味着单个检索只能返回单个类型的实体。因此，客户需要获得不同类型的实体并且需要具有一致的数据结构视图以进行进一步的决策，这需要依赖于使用特殊事件序列ID的工作，并且如果需要，可以多次重新发出查询请求（理论上，在一个不断变化的状态/配置的系统中，这样的解决方案是无时间限制的。

Oracle Clusterware 12 c提供了一种机制，可以执行多种实体的一致读取。该机制透明地处理实体，因此添加新的管理实体不需要对机制进行任何更改。

这是通过clscrs_stat3 API实现的。

clscrs_stat3

获取有关qlist标识的Oracle Clusterware实体的信息。

句法

clscrs_stat3（clscrs_querylist * qlist，uword flags，clscrs_crsentitylist * out_entitylist）;

输入

qlist ：要查询的Oracle Clusterware实体列表。

产量

out_entitylist ：保存返回的实体信息的实体列表。

返回

  • CLSCRS_STAT_SUCCESS ：表示API已成功查询Oracle Clusterware。
  • CLSCRS_STAT_FAILURE ：显示查询Oracle Clusterware时是否有错误。
  • CLSCRS_STAT_CONNECTION ：显示是否存在通信错误。

使用说明

  • 信息在out_entitylist返回。执行查询以使后续输出集保持一致。

  • 在将对象传递给此函数之前，您可以创建clscrs_query对象并将它们附加到clscrs_querylist对象。clscrs_query对象中必须存在每个实体类型所需的任何属性。如果属性列表为空，则Oracle Clusterware将返回所有属性。

  • 必须创建out_entitylist并将其作为空列表传递。实体的任何错误都会在out_entitylist中返回。

  • 输出作为嵌套实体列表返回。单个查询的结果将作为out_entitylist相应实体返回。单个查询的输出将作为该特定实体的实体列表返回。该类型的实体结果可以通过调用验证clscrs_crsentity_get_type ，拿到那就是部分实体的Oracle集群实体类型out_entitylist 。

    例如，如果您有两个查询，一个用于资源，另一个用于资源类型，则out_entitylist将包含两个实体对象，每个对象用于上述每个Oracle Clusterware实体类型。每个实体对象的实体列表将包含查询的结果。要确定特定对象的实体对象类型，必须在该实体对象上调用clscrs_crsentity_get_type函数。如果查询返回零匹配，则该实体对象的实体列表的大小将为零。

注意：

Oracle每个实体类型仅支持一个clscrs_query对象。如果为同一实体类型创建了多个clscrs_query对象，则Oracle不保证stat3 API行为。
