目的

SQL / JSON函数JSON_ARRAYAGG是一个聚合函数。它将一列SQL表达式作为输入，将每个表达式转换为JSON值，并返回包含这些JSON值的单个JSON数组。

EXPR

对于expr ，您可以指定任何SQL表达式，其值为JSON对象，JSON数组，数字文字，文本文字或null。此函数将数字文字转换为JSON数字值，将文本文字转换为JSON字符串值。

格式JSON

使用此可选子句指示输入字符串是JSON，因此不会在输出中引用。

order_by_clause生成

此子句允许您在语句返回的JSON数组中对JSON值进行排序。有关此子句的完整语义，请参阅SELECT文档中的order_by_clause 。

JSON_on_null_clause

当expr求值为null时，使用此子句指定此函数的行为。

  • NULL ON NULL - 如果指定此子句，则该函数返回JSON null值。

  • ABSENT ON NULL - 如果指定此子句，则该函数将省略JSON数组中的值。这是默认值。

JSON_agg_returning_clause

使用此子句指定此函数返回的字符串的数据类型。您可以指定以下数据类型：

  • VARCHAR2[( size [BYTE,CHAR])]

    在SQL中的其他位置指定VARCHAR2数据类型时，您需要指定大小。但是，在本节中，您可以省略大小。

  • CLOB

如果省略此子句，或者如果指定VARCHAR2但省略size值，则JSON_ARRAYAGG将返回VARCHAR2(4000)类型的字符串VARCHAR2(4000) 。

有关上述数据类型的更多信息，请参阅“ 数据类型 ” 。

严格

指定STRICT子句以验证JSON生成函数的输出是否正确JSON。如果检查失败，则会引发语法错误。

有关示例，请参阅JSON_OBJECT 。

用独特的钥匙

指定WITH UNIQUE KEYS以保证生成的JSON对象具有唯一键。

例子

以下语句创建一个表id_table ，其中包含ID号：

CREATE TABLE id_table（id NUMBER）;插入id_table VALUES（624）; INSERT INTO id_table VALUES（null）;插入id_table VALUES（925）;插入id_table VALUES（585）;

以下示例根据表id_table的ID号构造JSON数组：

SELECT JSON_ARRAYAGG（id ORDER BY id RETURNING VARCHAR2（100））ID_NUMBERS FROM id_table; ID_NUMBERS ------------- [585,624,925]
