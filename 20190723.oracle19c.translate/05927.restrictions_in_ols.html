G Oracle标签安全限制

此Oracle Label Security版本中存在一些限制。

这些限制如下：

  • CREATE TABLE AS SELECT限制

    如果尝试在受Oracle Label Security策略保护的模式中执行CREATE TABLE AS SELECT ，则该语句将失败。

  • 标签标签限制

    标签标签在数据库中的策略中必须是唯一的。在数据库中使用多个策略时，不能在不同的策略中使用相同的数字标签标记。

  • 出口限制

    在Oracle Database 12c第1版（12.1）之前，由于在Oracle Label Security中使用了opaque类型，因此无法导出LBACSYS模式。除了不会导出LBACSYS模式之外，可以使用安装的Oracle Label Security导出整个数据库（参数FULL =Y ）。

    从Oracle Database 12c版开始，此限制已被删除。有关可以从中支持导出的数据库版本的其他详细信息，请参阅完整数据库导出 。

  • Oracle Label Security删除限制

    您无法删除Oracle Label Security，但可以禁用它。请参阅禁用Oracle Label Security 。

  • 共享架构支持限制

    Oracle Internet Directory中定义的用户帐户无法获得单独的Oracle Label Security授权。但是，可以将授权授予目录用户映射到的共享模式。

    可以通过编程方式使用Oracle Label Security函数SET_ACCESS_PROFILE来设置标签授权配置文件，以便在用户通过身份验证并映射到共享模式后使用。Oracle Label Security不会在Oracle Label Security中给予标签授权的用户与实际数据库用户之间强制执行映射。

  • 隐藏列限制

    PL / SQL无法识别对表中隐藏列的引用。将生成编译器错误。
