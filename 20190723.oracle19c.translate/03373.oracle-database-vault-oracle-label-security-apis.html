19 Oracle Database Vault Oracle Label Security API

您可以使用DBMS_MACADM PL / SQL包管理Oracle Database Vault中的Oracle Label Security标签和策略。

  • CREATE_MAC_POLICY过程
    CREATE_MAC_POLICY过程指定在计算因子标签或Oracle Label Security会话标签时合并标签的算法。
  • CREATE_POLICY_LABEL过程
    CREATE_POLICY_LABEL过程标记Oracle Label Security策略中的标识。
  • DELETE_MAC_POLICY_CASCADE过程
    DELETE_MAC_POLICY_CASCADE过程删除与Oracle Label Security策略相关的所有Oracle Database Vault对象。
  • DELETE_POLICY_FACTOR过程
    DELETE_POLICY_FACTOR过程删除了导致Oracle Label Security标签的因素。
  • DELETE_POLICY_LABEL过程
    DELETE_POLICY_LABEL过程从Oracle Label Security策略中的标识中删除标签。
  • UPDATE_MAC_POLICY过程
    UPDATE_MAC_POLICY过程指定在计算因子标签或Oracle Label Security会话标签时合并标签的算法。

CREATE_MAC_POLICY过程

CREATE_MAC_POLICY过程指定在计算因子标签或Oracle Label Security会话标签时合并标签的算法。

句法

DBMS_MACADM.CREATE_MAC_POLICY（policy_name IN VARCHAR2，算法IN VARCHAR2）;

参数

表19-1 CREATE_MAC_POLICY参数

参数 描述

POLICY_NAME

现有策略的名称。

要查找当前数据库实例中的现有策略，请查询DBA_DV_MAC_POLICY视图中描述的DBA_DV_MAC_POLICY视图 。

算法

Oracle Label Security合并两个标签时的合并算法。输入表19-2中列出的与所需合并算法相对应的代码。例如，如果要选择最大级别/联合/联合合并算法，请输入HUU 。

CREATE_POLICY_LABEL过程

CREATE_POLICY_LABEL过程标记Oracle Label Security策略中的标识。

句法

DBMS_MACADM.CREATE_POLICY_LABEL（identity_factor_name IN VARCHAR2，identity_factor_value IN VARCHAR2，policy_name IN VARCHAR2，label IN VARCHAR2）;

参数

表19-3 CREATE_POLICY_LABEL参数

参数 描述

identity_factor_name

被标记的因子的名称。

要查找当前数据库实例中的现有因子，请查询DBA_DV_FACTOR视图中描述的DBA_DV_FACTOR视图 。

要查找与Oracle Label Security策略关联的因子，请使用DBA_DV_MAC_POLICY_FACTOR ，如DBA_DV_MAC_POLICY_FACTOR View中所述 。

identity_factor_value

被标记因子的身份价值。

要查找当前数据库实例中现有因子的标识，请查询DBA_DV_IDENTITY视图中描述的DBA_DV_IDENTITY视图 。

POLICY_NAME

现有策略的名称。

要查找当前数据库实例中的现有策略，请查询DBA_DV_MAC_POLICY视图中描述的DBA_DV_MAC_POLICY视图 。

标签

Oracle Label Security标签名称。

要查找因子标识符的现有策略标签，请查询DBA_DV_POLICY_LABEL视图，如DBA_DV_POLICY_LABEL视图中所述 。

例

BEGIN DBMS_MACADM.CREATE_POLICY_LABEL（identity_factor_name =>'App_Host_Name'，identity_factor_value =>'Sect2_Fin_Apps'，policy_name =>'Access Locations'，label =>'Sensitive'）;结束; /
