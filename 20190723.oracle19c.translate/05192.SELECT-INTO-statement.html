13.59选择INTO声明

SELECT INTO语句从一个或多个数据库表中检索值（如SQL SELECT语句所做的那样）并将它们存储在变量中（SQL SELECT语句不这样做）。

警告：

带有BULK COLLECT子句的SELECT INTO语句容易受到别名的影响，这会导致意外的结果。有关详细信息，请参阅“ 选择BULK收集语句和别名 ” 。

语义

select_into_statement

DISTINCT |独特

如果希望数据库仅返回所选的每组重复行的一个副本，请指定DISTINCT或UNIQUE 。这两个关键字是同义词。重复行是选择列表中每个表达式具有匹配值的行。

DISTINCT和UNIQUE查询的限制

  • 所有选择列表表达式中的总字节数限制为数据块的大小减去一些开销。此大小由初始化参数DB_BLOCK_SIZE指定。

  • 如果select_list包含LOB列，则不能指定DISTINCT 。

所有

（默认值）使数据库返回所有选定的行，包括重复的所有副本。

select_list中

如果SELECT INTO语句不返回任何行，则PL / SQL会引发预定义的异常NO_DATA_FOUND 。要防止此异常，请选择聚合函数COUNT(*) ，即使没有符合条件的行，也会返回单个值。

into_clause

使用此子句， SELECT INTO语句从单个行检索一个或多个列，并将它们存储在一个或多个标量变量或一个记录变量中。有关更多信息，请参阅“ into_clause :: = ” 。

bulk_collect_into_clause

使用此子句， SELECT INTO语句将检索整个结果集并将其存储在一个或多个集合变量中。有关更多信息，请参阅“ bulk_collect_into_clause :: = ” 。

rest_of_statement

可以在SQL SELECT语句中跟随关键字FROM任何内容，如Oracle数据库SQL语言参考中所述 。
