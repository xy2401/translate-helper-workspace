1.232 OPTIMIZER_INDEX_CACHING

OPTIMIZER_INDEX_CACHING允许您调整基于成本的优化的行为，以支持嵌套循环连接和IN列表迭代器。

属性 描述

参数类型

整数

默认值

0

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

价值范围

0到100

基本

没有

当使用索引访问内部表时，使用IN列表迭代器执行索引或执行嵌套循环连接的成本取决于缓冲区高速缓存中该索引的缓存。缓存量取决于优化程序无法预测的因素，例如系统上的负载和不同用户的块访问模式。

您可以通过将此参数设置为0到100之间的值来修改优化程序关于嵌套循环连接和IN-list迭代器的索引缓存的假设，以指示优化程序应该假定在缓存中的索引块的百分比。将此参数设置为更高的值会使嵌套循环连接和IN列表迭代器看起来更便宜。因此，它更有可能通过哈希或排序合并连接选择嵌套循环连接，并使用IN列表迭代器选择索引而不是其他索引或全表扫描。此参数的默认值为0，这会导致默认的优化程序行为。
