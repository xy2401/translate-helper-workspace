网络接口配置选项

在安装过程中，系统会要求您确定Oracle Universal Installer（OUI）在群集节点上检测到的每个网络适配器（或网络接口）的计划使用情况。

可以将每个NIC配置为仅执行以下角色之一：

  • 上市

  • 私人的

  • ASM

  • ASM和私人

  • 不使用

网络接口配置选项

您必须对Oracle Clusterware和Oracle RAC使用相同的专用适配器。为网络选择的精确配置取决于要配置的群集的大小和使用以及所需的可用性级别。网络接口必须至少为1 GbE，建议使用10 GbE。或者，使用InfiniBand进行互连。

如果经过认证的网络附加存储（NAS）用于Oracle RAC，并且此存储通过基于以太网的网络连接，则您必须具有用于NAS I / O的第三个网络接口。在这种情况下，未能提供三个独立的接口可能会导致负载下的性能和稳定性问题。

冗余互连使用无法保护用于公共通信的网络适配器。如果您需要公共适配器的高可用性或负载平衡，请使用第三方解决方案。通常，粘合，中继或类似技术可用于此目的。

您可以通过选择多个网络适配器作为专用适配器来为专用网络启用冗余互连使用。当您将多个网络适配器标识为私有时，冗余互连使用会创建冗余互连。
