31.4 OCI字符串操作函数

列出并描述OCI字符串操作函数。

字符串操作支持两种类型的数据结构：

  • 多字节字符串

  • 宽字符串

多字节字符串以本机Oracle字符集编码。对多字节字符串进行操作的函数将字符串作为整个单元，字符串的长度以字节为单位计算。宽字符（ wchar ）字符串函数为字符串操作提供了更大的灵活性。它们支持基于字符和基于字符串的操作，字符串的长度以字符计算。

宽字符数据类型是特定于Oracle的，不应与ANSI / ISO C标准定义的wchar_t数据类型混淆。所有操作系统中的Oracle宽字符数据类型总是4个字节，而wchar_t的大小取决于实现和操作系统。Oracle宽字符数据类型规范化多字节字符，以便它们具有固定宽度以便于处理。这可确保Oracle宽字符集和本机字符集之间的往返转换不会丢失数据。

字符串操作可以分为以下几类：

  • 在多字节和宽字符之间转换字符串

  • 角色分类

  • 案例转换

  • 显示长度的计算

  • 常规字符串操作，例如比较，连接和搜索

31.4.1 OCIMultiByteInSizeToWideChar（）

将多字节字符串的一部分转换为宽字符串。

目的

将多字节字符串的一部分转换为宽字符串。

句法

剑OCIMultiByteInSizeToWideChar（void * hndl，OCIWchar * dst，size_t dstsz，const OraText * src，size_t srcsz，size_t * rsize）;

参数

hndl（IN / OUT）

OCI环境或用户会话句柄，用于确定字符串的字符集。

dst（OUT）

指向wchar的目标缓冲区的指针。当dstsz为零时，它可以是NULL指针。

dstsz（IN）

目标缓冲区大小，以字符数表示。如果为零，则此函数返回转换所需的字符数。

src（IN）

要转换的源字符串。

srcsz（IN）

源字符串的长度，以字节为单位

rsize（OUT）

写入目标缓冲区的字符数，如果dstsz为零， dstsz转换后的字符串的字符数。如果它是NULL指针，则不返回任何内容。

评论

此例程将多字节字符串的一部分转换为宽字符串。它会尽可能多地转换完整的字符，直到达到输出缓冲区大小限制或输入缓冲区大小限制，或者它到达源字符串中的NULL终止符。如果空间允许，则输出缓冲区为NULL 。如果dstsz为零，则此函数仅返回不包括转换后的字符串所需的结束NULL终止符的字符数。如果OCI_UTF16ID为SQL指定CHAR在数据OCIEnvNlsCreate()函数，则该函数产生一个错误。

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。
