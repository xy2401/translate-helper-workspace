Oracle数据库中的1个 JSON

Oracle数据库本身支持JavaScript对象表示法（JSON）数据，具有关系数据库功能，包括事务，索引，声明性查询和视图。

本文档介绍如何使用数据库语言和功能来处理存储在Oracle数据库中的JSON数据。特别是，它介绍了如何将SQL和PL / SQL与JSON数据一起使用。

注意：

Oracle还提供了一系列简单Oracle文档访问 （ SODA ）API，用于访问存储在数据库中的JSON数据。SODA专为无模式应用程序开发而设计，无需了解关系数据库功能或SQL和PL / SQL等语言。它允许您在Oracle数据库中创建和存储文档集合，检索它们并查询它们，而无需知道文档如何存储在数据库中。

SODA有几种实现方式：

  • SODA for REST - 使用能够进行HTTP调用的任何语言，表示性状态转移（REST）请求执行收集和文档操作。

  • SODA for Java - Java类和接口代表数据库，集合和文档。

  • 用于PL / SQL的SODA - PL / SQL对象类型表示集合和文档。

  • SODA for C - Oracle调用接口（OCI）句柄代表集合和文档。

有关SODA 的信息，请将Oracle视为文档存储 。

1.1 Oracle数据库中的JSON概述

JSON数据和XML数据可以类似的方式在Oracle数据库中使用。与关系数据不同，两者都可以存储，索引和查询， 而无需任何定义数据的模式 。Oracle数据库本身支持JSON和关系数据库功能，包括事务，索引，声明性查询和视图。

JSON数据通常存储在NoSQL数据库中，例如Oracle NoSQL Database和Oracle Berkeley DB。这些允许存储和检索不基于任何模式的数据，但它们不提供关系数据库的严格一致性模型。

为了弥补这个缺点，关系数据库有时与NoSQL数据库并行使用。使用存储在NoSQL数据库中的JSON数据的应用程序必须自己确保数据完整性。

Oracle数据库对JSON的本机支持避免了这种解决方法。它提供了与JSON一起使用的关系数据库功能的所有好处，包括事务，索引，声明性查询和视图。

结构化查询语言（SQL）查询是声明性的。使用Oracle数据库，您可以使用SQL将JSON数据与关系数据连接起来。您可以相关地投影JSON数据，使其可用于关系流程和工具。您还可以在数据库中查询存储在外部表中的Oracle数据库外部的JSON数据。

您可以像访问其他数据库数据一样访问存储在数据库中的JSON数据，包括使用Oracle调用接口（OCI），Microsoft .NET Framework和Java数据库连接（JDBC）。

在Oracle数据库中，使用常见的SQL数据类型VARCHAR2 ， CLOB和BLOB存储JSON数据（与XML数据不同，后者使用抽象SQL数据类型XMLType存储）。Oracle建议您始终使用is_json检查约束来确保列值是有效的JSON实例（请参阅示例4-1 ）。

根据定义，文本JSON数据使用Unicode编码（UTF-8或UTF-16）进行编码。您可以使用存储在非Unicode字符集中的文本数据，就像它是JSON数据一样，但在这种情况下，Oracle数据库会在处理数据时自动将字符集转换为UTF-8。

数据库表中的JSON列

Oracle数据库对可用于存储JSON文档的表没有任何限制。包含JSON文档的列可以与任何其他类型的数据库数据共存。表还可以包含多个包含JSON文档的列。

将Oracle数据库用作JSON文档存储时，包含JSON列的表通常也会包含一些非JSON管理列。这些通常跟踪有关JSON文档的元数据。

如果您使用JSON为主要关系应用程序添加灵活性，那么您的某些表可能还有一个JSON文档列，您可以使用该列来管理不直接映射到关系模型的应用程序数据。

将SQL与JSON数据一起使用

在SQL中，您可以使用专门的函数和条件或简单的点表示法访问存储在Oracle数据库中的JSON数据。大多数SQL函数和条件都属于SQL / JSON标准，但有一些是特定于Oracle的。

  • SQL / JSON 生成函数json_object ， json_array ， json_objectagg和json_arrayagg 。他们收集SQL数据以生成JSON数据（作为SQL值）。

  • Oracle SQL聚合函数json_dataguide也是如此，但它生成的JSON数据是一个数据指南 ，您可以使用它来发现有关数据库中其他JSON数据的结构和内容的信息。

  • SQL / JSON 查询函数json_value ， json_query和json_table ，SQL / JSON条件json_exists ， is json ， is not json ， json_textcontains ，而Oracle SQL条件是json_equal 。除了is json ， is not json和json_equal ，它们会针对JSON数据评估SQL / JSON路径表达式以生成SQL值。

  • 点表示法 ，其作用类似于查询函数json_value和json_query的组合，类似于SQL对象访问表达式，即抽象数据类型（ADT）的属性点表示法。这是查询数据库中JSON数据的最简单方法。

作为查询的简单说明，这里是存储在表j_purchaseorder列po_document中的文档的点符号查询（此处别名为po ）。它获取所有采购订单请求者（JSON字段Requestor ）。 

选择po.po_document。请求者来自j_purchaseorder po;

使用带有JSON数据的PL / SQL

您通常可以在PL / SQL代码中使用SQL代码，包括访问JSON数据的SQL代码。您不能在PL / SQL中使用的任何SQL代码中使用空的JSON字段名称。

以下SQL函数和条件也可用作内置PL / SQL函数： json_value ， json_query ， json_object ， json_array和json_exists 。（在PL / SQL中，SQL条件json_exists是一个布尔函数。）

与没有BOOLEAN数据类型的Oracle SQL的情况不同，PL / SQL BOOLEAN是SQL / JSON函数json_value的有效返回数据类型。

还有JSON的PL / SQL对象类型，可用于细粒度构造和内存中JSON数据的操作。您可以对其进行内省，修改并将其序列化为文本JSON数据。

1.2入门将JSON与Oracle数据库一起使用

通常，在Oracle数据库中使用JSON数据时，您将执行以下任务：（1）使用is json检查约束创建JSON列，（2）将JSON数据插入列，以及（3）查询JSON数据。

  1. 创建一个包含主键列和JSON列的表，并添加一个is json 检查约束，以确保JSON列仅包含格式良好的JSON数据。

    以下语句创建具有主键id和JSON列po_document表j_purchaseorder （另请参见示例4-1 ）。 

    CREATE TABLE j_purchaseorder（id VARCHAR2（32）NOT NULL PRIMARY KEY ，date_loaded TIMESTAMP（6）WITH TIME ZONE， po_document VARCHAR2（32767） CONSTRAINT ensure_json CHECK （po_document IS JSON ））;
  2. 使用Oracle Database可用的任何方法将JSON数据插入JSON列。

    以下语句使用SQL INSERT语句将一些简单的JSON数据插入到表j_purchaseorder的第三列中（这是列po_document - 请参阅上一页）。这里省略了一些JSON数据（ ... ）。有关这些详细信息，请参见例4-2 。 

    INSERT INTO j_purchaseorder VALUES（SYS_GUID（），to_date（'30 -DEC-2014'），'{“PONumber”：1600，“Reference”：“ABULL-20140421”，“Requestor”：“Alexis Bull”，“User” ：“ABULL”，“CostCenter”：“A50”，“ShippingInstructions”：{...}，“特殊说明”：null，“AllowPartialShipment”：true，“LineItems”：[...]}“）;
  3. 查询JSON数据。返回值始终是表示JSON值的VARCHAR2实例。这是一些简单的例子。

    以下查询从JSON列po_document每个文档中提取标量值，JSON编号是JSON列po_document对象的字段PONumber的值（另请参见示例12-1 ）：

    选择po.po_document。PONumber FROM j_purchaseorder po;

    以下查询从每个文档中提取JSON电话对象的数组 ，该对象的字段Phone的值是字段ShippingInstructions的值（另请参见示例12-2 ）：

    选择po.po_document。ShippingInstructions。电话来自j_purchaseorder po;

    以下查询从每个文档中提取多个值作为数组：数组Phone每个对象的字段type值。返回的数组不是存储数据的一部分，而是由查询自动构造。（未指定数组元素的顺序。） 

    选择po.po_document。ShippingInstructions。Phone.type FROM j_purchaseorder po;
