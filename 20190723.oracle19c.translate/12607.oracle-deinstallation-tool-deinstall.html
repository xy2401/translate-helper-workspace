Oracle卸载（卸载）

您可以在安装后从Oracle主目录运行deinstall命令。

目的

deinstall停止Oracle软件，并删除操作系统上针对特定Oracle主目录的Oracle软件和配置文件。

句法

deinstall命令使用以下语法：

（./deinstall [-silent] [-checkonly] [-paramfile complete path of input response file ] [-params name1 = value name2 = value . . . ][-o complete path of directory for saving files ] [-tmpdir complete path of temporary directory to use ] [-logdir complete path of temporary directory to use complete path of log directory to use ] [-local] [-skipLocalHomeDeletion] [-skipRemoteHomeDeletion] [-help]

参数

参数 描述
-无声

使用此标志以非交互模式运行deinstall 。此选项需要以下之一：

  • 可以访问的工作系统，用于确定安装和配置信息。-silent标志不适用于失败的安装。

  • 一个响应文件，其中包含正在卸载或取消配置的Oracle主目录的配置值。

您可以通过使用-checkonly标志运行deinstall来生成要使用或修改的响应文件。然后， deinstall会发现来自Oracle主目录的信息以进行卸载和解除配置。它会生成响应文件，然后您可以使用-silent选项。

您还可以修改位于$ORACLE_HOME/deinstall/response目录中的模板文件deinstall.rsp.tmpl 。

-checkonly 使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行deinstall不会删除Oracle配置。-checkonly标志生成一个响应文件，然后您可以使用deinstall命令和-silent选项。
-paramfile complete path of input response file

使用此标志可以使用非默认位置的响应文件运行deinstall 。使用此标志时，请提供响应文件所在的完整路径。

响应文件的默认位置是$ORACLE_HOME/deinstall/response 。

-params [ name1=value name2=value name3=value . . . ]

将此标志与响应文件一起使用，以覆盖要在已创建的响应文件中更改的一个或多个值。
-o complete path of directory for saving response files

使用此标志提供除保存响应文件（ deinstall.rsp.tmpl ）的默认位置以外的路径。

响应文件的默认位置是$ORACLE_HOME/deinstall/response 。

-tmpdir complete path of temporary directory to use

使用此标志指定非默认位置，其中deinstall写入deinstall的临时文件。

-logdir complete path of log directory to use

使用此标志指定非默认位置，其中deinstall写入deinstall的日志文件。

-本地

在多节点环境中使用此标志可以在群集中卸载Oracle软件。

使用此标志运行deinstall时，它会在本地节点（运行deinstall的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

-skipLocalHomeDeletion

在多节点环境中的Oracle Grid Infrastructure安装中使用此标志可以取消配置本地Grid主目录而不删除Grid主目录。
-skipRemoteHomeDeletion

在多节点环境中的Oracle Grid Infrastructure安装中使用此标志可以取消配置远程Grid主目录而不删除Grid主目录。
-救命

使用此选项可获取有关命令选项标志的其他信息。
