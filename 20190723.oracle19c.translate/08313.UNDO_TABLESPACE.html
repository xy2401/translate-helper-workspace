1.346 UNDO_TABLESPACE

UNDO_TABLESPACE指定实例启动时要使用的撤消表空间。 如果在实例处于手动撤消管理模式时指定了此参数，则会发生错误并且启动将失败。
属性 描述

参数类型

串

句法

UNDO_TABLESPACE = undoname

默认值

数据库中第一个可用的撤消表空间。

可修改

改变系统

可在PDB中修改

是

价值范围

现有撤消表空间的合法名称

基本

是

Oracle RAC

设置时，每个实例必须具有此参数的唯一值。

如果省略UNDO_TABLESPACE参数，则选择数据库中第一个可用的撤消表空间。如果没有可用的撤消表空间，则实例将在没有撤消表空间的情况下启动。在这种情况下，将使用SYSTEM回滚段执行用户事务。在正常情况下，您应该避免在此模式下运行。

在实例运行时，可以将undo表空间替换为另一个undo表空间。

注意：

在Oracle Data Guard配置中更新主数据库上的此参数时，还必须在配置中的所有物理备用数据库上更新它。这可确保备用数据库在成为主数据库时可以找到撤消表空间。
