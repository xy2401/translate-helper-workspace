99 DBMS_LOGSTDBY_CONTEXT

如Oracle数据库12 c释放1（12.1）的，SQL应用进程访问称为上下文命名空间LSBY_APPLY_CONTEXT 。您可以使用所提供的程序DBMS_LOGSTDBY_CONTEXT包来设置和检索相关的各种参数LSBY_APPLY_CONTEXT 。在使用DBMS_LOGSTBDY.SKIP和DBMS_LOGSTDBY.SKIP_ERROR过程编写使用SQL Apply注册的跳过过程时，这非常有用。

99.1 DBMS_LOGSTDBY_CONTEXT概述

SQL Apply进程可以访问名为LSBY_APPLY_CONTEXT的上下文命名空间。

下表列出了相关的预定义参数LSBY_APPLY_CONTEXT ，你可以设置和通过使用规定的程序检索DBMS_LOGSTDBY_CONTEXT包。在编写使用DBMS_LOGSTBDY.SKIP和DBMS_LOGSTDBY.SKIP_ERROR过程向SQL Apply注册的跳过过程时，以这种方式设置和检索参数的功能非常有用。

表99-1命名空间LSBY_APPLY_CONTEXT的预定义参数

参数 描述

声明

Apply进程正在处理的语句的前4000个字节。

STATEMENT_TYPE

有关各种语句类型的列表，请参见表97-15 。

PACKAGE_SCHEMA

拥有正在处理的PL / SQL包的模式（如果适用）。

包裹名字

正在处理的PL / SQL包的名称（如果适用）。

PROCEDURE_NAME

正在处理的PL / SQL过程的名称（如果适用）。

CURRENT_SCHEMA

在主服务器上执行DDL或PL / SQL过程时生效的当前模式。

XIDUSN

主数据库中事务的XIDUSN。

XIDSLT

主数据库中事务的XIDSLT。

XIDSQN

主数据库中事务的XIDSQN。

SCHEMA

拥有SQL Apply正在处理的对象的架构（在DML或DDL操作的情况下）。

名称

SQL Apply正在处理的对象名称（在DML或DDL操作的情况下）。

CONTAINER_NAME

目标对象或PL / SQL过程所在的容器（在多租户容器数据库（CDB）的情况下）。

错误

SQL Apply进程遇到的原始错误的文本（如果适用）。

NEW_ERROR

SQL Apply进程引发的新错误的文本（如果适用，请参阅DBMS_LOGSTDBY SKIP_ERROR过程 ）。

NEW_STMT

SQL Apply必须执行的替换语句的文本（如果适用，请参阅DBMS_LOGSTDBY SKIP过程 ）。

SKIP_ACTION

由SQL Apply执行的跳过操作（请参阅DBMS_LOGSTDBY SKIP过程 ）。
