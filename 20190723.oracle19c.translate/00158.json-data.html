2.1 JSON概述

JavaScript Object Notation （ JSON ）在标准ECMA-404（JSON数据交换格式）和ECMA-262（ECMAScript语言规范，第三版）中定义。ECMAScript的JavaScript方言是一种在Web浏览器和Web服务器中广泛使用的通用编程语言。

JSON几乎是JavaScript的对象文字符号的一个子集。Foot 1因为它可以用来表示JavaScript对象文字，所以JSON通常用作数据交换语言。在这方面，它与XML有很多共同之处。

因为它是（几乎是JavaScript表示法的一个子集），JSON通常可以在JavaScript程序中使用，而无需解析或序列化。它是一种表示JavaScript对象文字，数组和标量数据的基于文本的方式。

虽然它是在JavaScript的上下文中定义的，但JSON实际上是一种与语言无关的数据格式。各种编程语言可以解析和生成JSON数据。

JSON对于人类来说相对容易读写，并且易于软件解析和生成。它通常用于序列化结构化数据并通过网络进行交换，通常在服务器和Web应用程序之间进行。

2.2 JSON语法及其代表的数据

描述了JSON（和JavaScript）值，标量，对象和数组。

JSON 值是以下之一：object，array，number，string，Boolean（ true或false ）或null 。除对象和数组之外的所有值都是标量 。

注意：

就SQL而言，JSON值为null是一个值 。它不是NULL ，在SQL中表示缺少值（缺失，未知或不适用的数据）。特别是，SQL条件IS NULL对于JSON null值返回false，并且SQL条件IS NOT NULL返回true。

JavaScript对象是零个或多个属性名称对和关联的JSON值的关联数组或字典。Foot 2 JSON对象是JavaScript对象文字 。脚3它被写成括在大括号这样的属性列表（ { ， } ），用逗号（分隔的名称-值对, ），并用每一对由冒号分隔的名称和值（ : ）。（逗号或冒号之前或之后的空格是可选的，并且是无关紧要的。）

在JSON中，每个属性名称和每个字符串值必须用双引号（ " ）括起来。在JavaScript表示法中，对象文字中使用的属性名称可以但不必括在双引号中。它也可以用单引号（ ' ）括起来。

由于存在这种差异，实际上，使用不带引号或单引号属性名称表示的数据有时被简称为JSON表示，而JSON的某些实现（包括Oracle数据库实现）支持松散的语法，允许使用不带引号和单引号的属性名称 。

JSON中的字符串由Unicode字符组成，反斜杠（ \ ）转义。JSON编号（数字）以十进制表示法表示，可能是有符号的，可能包括十进制指数。

对象属性通常称为字段 。它有时被称为密钥 ，但是这个文档通常使用“字段”来避免与“密钥”这个词的其他用法混淆。对象属性名称 - 值对通常称为对象成员 （但有时成员只能表示属性）。对象成员之间的顺序并不重要。

注意：

  • JSON字段名称可以为空 （写为"" ）。脚4

  • 给定JSON对象中的每个字段名称不一定是唯一的;可以重复相同的字段名称。Oracle数据库采用的SQL / JSON 路径评估始终只使用具有给定字段名称的一个对象成员;任何其他具有相同名称的成员都将被忽略 。未指定使用多个这样的成员中的哪一个。

    另请参阅JSON对象中的唯一对应重复字段 。

JavaScript数组包含零个或多个元素。JSON数组由括号（ [ ， ] ）表示，它们围绕数组元素 （也称为项 ）的表示形式，用逗号（ , ）分隔，每个数组都是一个对象，一个数组或一个标量值。数组元素顺序很重要 。（括号或逗号之前或之后的空格是可选的，无关紧要。）

2.3 JSON与XML相比

JSON和XML（可扩展标记语言）通常用作数据交换语言。他们的主要区别列在这里。

JSON对于简单的结构化数据最有用。XML对结构化和半结构化数据都很有用。JSON通常以数据为中心，而不是以文档为中心; XML也可以。JSON不是标记语言;它仅用于数据表示。XML既是文档标记语言又是数据表示语言。

  • JSON数据类型很少并且是预定义的。XML数据可以是无类型的，也可以基于XML模式或文档类型定义（DTD）。

  • JSON具有简单的结构定义和文档组合结构：它缺少属性，命名空间，继承和替换。

  • JavaScript对象文字成员的顺序无关紧要。通常，订单在XML文档中很重要。

  • JSON缺少等效的XML文本节点（XPath节点测试text() ）。特别是，这意味着没有混合内容（这是说JSON不是标记语言的另一种方式）。

  • JSON没有日期数据类型（与XML和JavaScript不同）。使用可用的数据类型（如字符串）以JSON表示日期。在日期和JSON字符串之间进行转换有一些事实上的标准。但是，使用JSON的程序必须以某种方式处理日期表示转换。

由于其简单的定义和功能，JSON数据通常比XML数据更容易生成，解析和处理。涉及组合不同数据源的用例通常很适合使用XML，因为它提供了名称空间和其他构造，有助于模块化和继承。
