25.1关于奇异值分解

奇异值分解（SVD）与密切相关主成分分析（PCA）是完善的特征提取方法，具有广泛的应用。Oracle Data Mining将SVD实现为特征提取算法，将PCA实现为SVD模型的特殊评分方法。

SVD和PCA是正交线性变换，在捕获数据的基础方差时是最佳的。此属性对于减少高维数据的维度和支持有意义的数据可视化非常有用。

除降维外，SVD和PCA还有许多重要的应用。这些包括矩阵求逆，数据压缩和未知数据值的插补。

  • U矩阵由一组“左”正交基础组成
  • S矩阵是对角矩阵
  • V矩阵由一组“正确的”正交基础组成

S中的值称为奇异值。它们是非负的，它们的大小表明相应碱基（组分）的重要性。奇异值反映了基数捕获的数据方差的数量。第一个基础（具有最大奇异值的基础）位于最大数据方差的方向上。第二个基础捕获具有第二个最大方差的正交方向，依此类推。

SVD基本上执行坐标旋转，其将变换的轴与数据中的最大方差的方向对齐。在假设观测数据具有高信噪比并且大方差对应于感兴趣的数据内容而较低方差对应于噪声的情况下，这是有用的过程。

SVD假设基础数据是高斯分布的，并且可以用均值和协方差来很好地描述。

25.1.2低秩分解

为了降低维数，奇异值分解（SVD）保持低阶基（具有最大奇异值的基）并忽略高阶基（具有最小奇异值的基）。该策略背后的基本原理是，低阶基数保留了对其方差贡献最大的数据特征，并且可能捕获数据的最重要方面。

给定数据集X （ n × m ），其中n是行数， m是属性的数量，低秩SVD仅使用k个分量（ k <= min （ m ， n ））。在SVD的典型实现中， k的值需要目视检查与各个组件相关联的排序奇异值。在Oracle Data Mining中，SVD自动估算截止点，这对应于解释方差的显着下降。

SVD产生两组标准正交基（ U和V ）。这些碱基中的任何一个都可以用作新的坐标系。在Oracle Data Mining SVD中， V是新坐标系， U表示此坐标系中X的投影。该算法计算新数据的投影如下：

25.1.3可伸缩性

在Oracle数据挖掘中，奇异值分解（SVD）可以处理具有数百万行和数千个属性的数据集。Oracle Data Mining会根据数据自动推荐适当数量的功能，以减少维数。

当计算完全分解时，SVD具有行数和立方可伸缩性的线性可伸缩性以及属性数量。低秩分解通常与行数呈线性关系，与列数呈线性关系。降级的可伸缩性取决于排名与行数和列数的比较。当秩小得多时它可以是线性的，或者当它在相同的比例时它可以是立方的。

25.2.1模型大小

奇异值分解中的U矩阵具有与构建数据中的行数一样多的行。为避免创建大型模型，只有在启用特定于算法的设置时， U矩阵才会保留。默认情况下， U矩阵不会持久存在。

25.2.2表现

奇异值分解可以使用近似计算来提高性能。近似可能适用于具有许多列的数据集。近似的低秩分解以合理的计算成本提供了良好的解决方案。近似的质量取决于数据的特征。

25.3 SVD的数据准备

了解为奇异值分解（SVD）准备数据。

Oracle Data Mining为数值数据和分类数据实现了SVD。

使用SVD对构建数据进行评分时，自动数据准备不执行任何操作。使用主成分分析（PCA）对构建数据进行评分时，自动数据准备按平均值移动数值数据。

不需要缺少值处理，因为Oracle Data Mining算法会自动处理缺失值。SVD使用模式用平均值和分类缺失值替换数值缺失值。对于稀疏数据（嵌套列中缺少值），SVD用零替换缺失值。
