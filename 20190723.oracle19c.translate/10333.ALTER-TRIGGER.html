ALTER TRIGGER

目的

触发器使用PL / SQL定义。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用ALTER TRIGGER语句启用，禁用或编译数据库触发器。

注意：

此语句不会更改现有触发器的声明或定义。要重新声明或重新定义触发器，请使用带有OR REPLACE关键字的CREATE TRIGGER语句。

也可以看看：

  • 有关创建触发器的信息，请创建TRIGGER

  • DROP TRIGGER有关丢弃触发器的信息

  • Oracle Database Concepts获取有关触发器的一般信息

先决条件

触发器必须位于您自己的架构中，或者您必须具有ALTER ANY TRIGGER系统特权。

此外，要更改DATABASE上的触发器，您必须具有ADMINISTER DATABASE TRIGGER权限。

语义

模式

指定包含触发器的架构。如果省略schema ，则Oracle数据库假定触发器位于您自己的模式中。

TRIGGER_NAME

指定要更改的触发器的名称。

trigger_compile_clause

有关此子句的语法和语义以及有关创建和编译触发器的完整信息，请参阅Oracle数据库PL / SQL语言参考 。

ENABLE | DISABLE

指定ENABLE以启用触发器。您还可以使用ALTER TABLE的ENABLE ALL TRIGGERS子句来启用与表关联的所有触发器。请参见ALTER TABLE 。

指定DISABLE以禁用触发器。您还可以使用ALTER TABLE的DISABLE ALL TRIGGERS子句禁用与表关联的所有触发器。

重命名条款

指定RENAME TO new_name以重命名触发器。Oracle数据库重命名触发器并使其处于重命名之前的状态。

当您重命名一个触发器，数据库重建的触发器的记忆源USER_SOURCE ， ALL_SOURCE和DBA_SOURCE数据字典视图。因此，即使触发源没有更改，注释和格式也可能会在这些视图的TEXT列中发生更改。

版面| NONEDITIONABLE

使用这些子句指定是否editioning为架构对象类型后启用触发器是否成为editioned或noneditioned对象TRIGGER的schema 。默认为EDITIONABLE 。有关更改已编辑和未编辑的对象的信息，请参阅“ Oracle数据库开发指南” 。

对不可接受的限制

您不能为交叉触发器指定NONEDITIONABLE 。
