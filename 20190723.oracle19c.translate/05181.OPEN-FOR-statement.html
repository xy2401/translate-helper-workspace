13.46公开声明

OPEN FOR语句将游标变量与查询相关联，分配数据库资源以处理查询，标识结果集，并将光标定位在结果集的第一行之前。

如果查询具有FOR UPDATE子句，则OPEN FOR语句将锁定结果集的行。

open_for_statement

cursor_variable

游标变量的名称。如果cursor_variable是子程序的形式参数，则它必须没有返回类型。有关光标变量作为子程序参数的信息，请参见“ 光标变量作为子程序参数 ” 。

： host_cursor_variable

在PL / SQL主机环境中声明并作为绑定变量传递给PL / SQL的游标变量的名称。不要在冒号（:)和host_cursor_variable之间放置空格。

主机游标变量的数据类型与任何PL / SQL游标变量的返回类型兼容。

select_statement中

SQL SELECT语句（不是PL / SQL SELECT INTO语句）。通常， select_statement返回多行。

看到：

SELECT语句语法的Oracle数据库SQL语言参考

dynamic_string

字符串文字，字符串变量或数据类型为CHAR ， VARCHAR2或CLOB字符串表达式，表示SQL SELECT语句。通常， dynamic_statement表示返回多行的SQL SELECT语句。

using_clause

使用位置表示法指定绑定变量。

注意：

如果在dynamic_sql_statement重复占位符名称，请注意占位符与绑定变量关联的方式取决于动态SQL语句的类型。有关详细信息，请参阅“ 动态SQL语句中的重复占位符名称” 。

对using_clause的限制

当且仅当select_statement或dynamic_sql_stmt包含绑定变量的占位符时使用。

IN，OUT，IN OUT

绑定变量的参数模式。IN绑定变量将其值传递给select_statement或dynamic_string 。OUT绑定变量存储dynamic_string返回的值。IN OUT绑定变量将其初始值传递给dynamic_string并存储dynamic_string返回的值。默认值： IN 。

bind_argument

表达式，其值在运行时替换select_statement或dynamic_string中的相应占位符。您必须为每个占位符指定bind_argument 。

注意：

可以按任何顺序评估绑定变量。如果程序确定评估顺序，那么在程序执行此操作时，其行为是不确定的。

bind_argument的限制

  • bind_argument不能是由string索引的关联数组。

  • bind_argument不能是保留字NULL 。

    要将值NULL传递给动态SQL语句，请使用未初始化的变量，以便在其中使用NULL ，如例7-7所示 。
