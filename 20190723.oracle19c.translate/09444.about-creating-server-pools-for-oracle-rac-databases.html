关于为Oracle RAC数据库创建服务器池

您可以在创建Oracle RAC数据库时使用DBCA创建服务器池，但Oracle建议您在部署数据库软件和数据库之前创建服务器池。

Oracle还建议您：

  • 在群集中创建第一个服务器池之前启用角色分离。

  • 使用配置策略和相应的策略集创建和管理服务器池。

您可以通过以下两种方式之一实现角色分离管理：

  • 垂直实现 （层之间）描述了基于不同操作系统用户和用于技术堆栈中各层的组的角色分离方法。服务器池和资源的权限使用访问控制列表授予堆栈中每个层的不同用户（和组）。Oracle自动存储管理（ASM）提供角色分离功能，作为Oracle Grid Infrastructure安装的一部分，基于特定角色的操作系统组的粒度分配。

  • 水平实现 （在一个层内）描述了一种角色分离方法，该方法使用分配给服务器池和策略管理数据库或应用程序的访问控制列表授予的资源的访问权限来限制一个层内的资源访问。

    例如，考虑一个名为grid的操作系统用户，该用户具有主操作系统组oinstall ，用于安装Oracle Grid Infrastructure并创建两个数据库服务器池。操作系统用户ouser1和ouser2必须能够在服务器池中运行，但不应该能够修改这些服务器池并无意或有意地从其他服务器池中撤回硬件资源。
