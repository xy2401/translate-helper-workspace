5.1数据泵导出和导入的数据性能改进

本节介绍数据泵导出和导入的数据性能改进。

数据泵导出和导入实用程序的性能提高可归因于几个因素，包括：

  • 多个工作进程可以执行可交互和分区间并行，以在多个并行直接路径流中加载和卸载表。

  • 对于非常大的表和分区，当使用外部表方法访问数据时，单个工作进程可以通过多个并行查询和并行DML I / O服务器进程选择分区内并行性。

  • Data Pump使用并行性来构建索引和加载包体。

  • 转储文件由服务器直接读取和写入，因此不需要向客户端移动任何数据。

  • 转储文件存储格式是直接路径API的内部流格式。此格式与存储在表空间内的Oracle数据库数据文件中的格式非常相似。因此，不会执行客户端到INSERT语句绑定变量的转换。

  • 支持的数据访问方法，直接路径和外部表比传统SQL更快。直接路径API提供最快的单流性能。外部表功能可以有效地使用Oracle数据库的并行查询和并行DML功能。

  • 在导出期间，元数据和数据提取可以重叠。

5.2调整性能

Data Pump旨在充分利用所有可用资源，以最大限度地提高吞吐量并最大限度地缩短已用工作时间

为此，必须在CPU，内存和I / O之间平衡系统。此外，标准性能调整原则适用。例如，为了获得最佳性能，应确保作为转储文件集成员的文件驻留在不同的磁盘上，因为转储文件是并行写入和读取的。此外，磁盘不应与源或目标表空间所在的磁盘相同。

任何性能调优活动都涉及在性能和资源消耗之间进行权衡。

  • 控制资源消耗
    数据泵导出和导入实用程序允许您动态增加和减少每个作业的资源消耗。

    使用与压缩和加密相关的数据泵参数可以对性能产生积极影响，特别是在网络模式下执行作业的情况下。

5.2.1控制资源消耗

数据泵导出和导入实用程序允许您动态增加和减少每个作业的资源消耗。

这是使用Data Pump PARALLEL参数完成的，以指定作业的并行度。为了获得最大吞吐量，请不要将PARALLEL设置为CPU数量的两倍以上（每个CPU两个工作程序）。

随着您增加并行度，CPU使用率，内存消耗和I / O带宽使用率也会增加。您必须确保有足够数量的这些资源可用。如有必要，您可以跨不同的磁盘设备或通道分发文件以获得所需的I / O带宽。

要最大化并行性，必须为每个并行度提供至少一个文件。最简单的方法是在文件名中使用替换变量（例如， file%u.dmp ）。但是，根据您的磁盘设置（例如，简单的非条带磁盘），您可能不希望将所有转储文件放在一个设备上。在这种情况下，最好使用替换变量指定多个文件名，每个文件名在一个单独的目录中解析为单独的磁盘。即使CPU速度快，磁盘速度快，CPU和磁盘之间的路径也可能是可以维持的并行度的制约因素。

Data Pump PARALLEL参数仅在Oracle Database 11g或更高版本的企业版中有效。

5.2.2压缩和加密对性能的影响

使用与压缩和加密相关的数据泵参数可以对性能产生积极影响，特别是在网络模式下执行作业的情况下。

但是您应该意识到，由于对原始数据执行转换所需的额外CPU资源，还会对性能产生负面影响。双方都存在权衡取舍。

5.2.3导出和导入统计信息时的内存注意事项

使用12.1之前的版本创建并且包含大量统计数据的数据泵导出转储文件可能导致导入操作使用大量内存。

为避免导入操作期间内存不足，请确保在开始导入之前分配足够的内存。所需的确切内存量取决于您导入的数据量，您使用的平台以及配置特有的其他变量。

完全避免此问题的一种方法是在导出或导入操作上设置Data Pump EXCLUDE=STATISTICS参数。然后，您可以使用DBMS_STATS PL / SQL包在导入完成后重新生成目标数据库上的统计信息。

5.3影响数据泵性能的初始化参数

某些Oracle数据库初始化参数的设置可能会影响数据泵导出和导入的性能。

特别是，您可以尝试使用以下设置来提高性能，尽管在所有平台上效果可能不同。

  • DISK_ASYNCH_IO = TRUE

  • DB_BLOCK_CHECKING = FALSE

  • DB_BLOCK_CHECKSUM = FALSE

以下初始化参数必须将值设置得足够高以允许最大并行度：

  • PROCESSES

  • SESSIONS

  • PARALLEL_MAX_SERVERS

此外， SHARED_POOL_SIZE和UNDO_TABLESPACE初始化参数应该是宽大的。确切的值取决于数据库的大小。

5.3.1在GoldenGate复制环境中设置缓冲区高速缓存的大小

Oracle Data Pump使用GoldenGate Replication功能在进程之间进行通信。

如果设置了SGA_TARGET初始化参数，则STREAMS_POOL_SIZE初始化参数将自动设置为合理的值。

如果未设置SGA_TARGET初始化参数且未定义STREAMS_POOL_SIZE初始化参数，则流池的大小将自动默认为共享池大小的10％。

创建流池时，将从分配给缓冲区高速缓存的内存中获取所需的SGA内存，从而将高速缓存的大小减小到小于DB_CACHE_SIZE初始化参数指定的大小。这意味着如果缓冲区高速缓存仅配置了所需的最小SGA，则数据泵操作可能无法正常工作。建议STREAMS_POOL_SIZE最小大小为10 MB，以确保数据泵操作成功。

5.3.2管理多个用户数据泵作业的资源使用情况

如果有多个用户在同一数据库环境中执行数据泵作业，则可以使用MAX_DATAPUMP_JOBS_PER_PDB和MAX_DATAPUMP_PARALLEL_PER_JOB初始化参数来获得对资源利用率的更多控制。

初始化参数MAX_DATAPUMP_JOBS_PER_PDB确定每个可插拔数据库（PDB）的最大并发Oracle Data Pump作业数。使用Oracle Database 19c和更高版本，您可以将参数设置为AUTO 。此设置意味着Oracle Data Pump将MAX_DATAPUMP_JOBS_PER_PDB的实际值派生为SESSIONS初始化参数值的50％（ 50% ）。如果未将值设置为AUTO ，则默认值为100 。您可以将值设置为0到250 。

Oracle Database Release 19c及更高版本包含初始化参数MAX_DATAPUMP_PARALLEL_PER_JOB 。如果有多个用户在给定的数据库环境中同时执行数据泵作业，则可以使用此参数来获得对资源利用率的更多控制。参数MAX_DATAPUMP_PARALLEL_PER_JOB指定可用于每个Oracle Data Pump作业的最大并行进程数。您可以指定特定的最大进程数，也可以选择AUTO 。如果选择指定设定值，则此最大数量可以是1到1024 （默认值为1024 ）。如果选择指定AUTO ，则Oracle Data Pump MAX_DATAPUMP_PARALLEL_PER_JOB参数MAX_DATAPUMP_PARALLEL_PER_JOB的实际值派生为SESSIONS初始化参数值的25％（ 25% ）。
