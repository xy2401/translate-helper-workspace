确保要安装的Oracle RAC数据库与现有数据库兼容

如果安装了Oracle Clusterware，并且安装了其他Oracle软件的不同版本，则Oracle Clusterware版本必须晚于或等于Oracle Database软件版本。执行Oracle Grid Infrastructure 19c安装时，Oracle Clusterware和Oracle ASM都升级到19c。

如果您有现有的Oracle主目录，则可以创建新的Oracle主目录并将Oracle Database 19c安装到新的Oracle主目录中。确保Oracle Clusterware位于单独的Oracle Grid Infrastructure主目录中。用于集群安装的Oracle Grid Infrastructure无法安装在Oracle数据库的Oracle主目录中。
将文件从RAW设备迁移到支持的存储 如果您在RAW设备上存储了任何数据库数据，那么在启动Oracle Grid Infrastructure和Oracle RAC安装之前，必须使用RMAN将该数据复制到Oracle ASM或其他受支持的文件系统。
准备升级所有现有节点 Oracle RAC数据库实例正在您打算成为新集群安装成员的相同节点上运行。例如，如果在三节点群集上运行现有Oracle RAC数据库，则必须升级所有三个节点。您无法仅升级群集的两个节点，而是在升级中删除第三个实例。
确保Oracle RAC数据库版本等于或早于Oracle Clusterware的版本

您可以在群集上为Oracle数据库安装多个Oracle主目录。但是，这些家庭中的Oracle RAC数据库软件必须来自与安装的Oracle Clusterware版本相同或之前的版本。例如：

  • 如果您的服务器使用Oracle Grid Infrastructure 19c，那么您可以在一个节点上运行Oracle Database 19c单实例数据库，并分离Oracle Database 11g第2版（11.2），Oracle RAC 12c第1版（12.1），Oracle RAC 12c版本2（12.2），Oracle RAC 18c或Oracle RAC 19c数据库也在群集上运行。

  • 您不能在群集上安装Oracle Grid Infrastructure 11g第2版（11.2），并安装Oracle RAC 12c第1版（12.1）。
