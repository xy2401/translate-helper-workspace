关于RADIUS质询 - 响应用户界面

您可以使用第三方身份验证供应商自定义RADIUS质询 - 响应用户界面以适合特定设备。

您可以设置支持RADIUS标准的任何身份验证设备，以对Oracle用户进行身份验证。当您的身份验证设备使用质询 - 响应模式时，图形界面会首先提示最终用户输入密码，然后提示其他信息（例如，用户从令牌卡获取的动态密码）。此接口基于Java，以提供最佳的平台独立性。

身份验证设备的第三方供应商必须自定义此图形用户界面以适合其特定设备。例如，智能卡供应商会自定义Oracle客户端以向智能卡读卡器发出质询。然后，当智能卡接收到挑战时，它通过提示用户提供更多信息（例如PIN）来响应。

自定义RADIUS质询 - 响应用户界面

您可以通过创建自己的类来自定义OracleRadiusInterface接口。

  1. 打开sqlnet.ora文件。

    默认情况下， sqlnet.ora文件位于ORACLE_HOME /network/admin目录或TNS_ADMIN环境变量设置的位置。确保已正确设置TNS_ADMIN变量以指向正确的sqlnet.ora文件。

  2. 找到SQLNET.RADIUS_AUTHENTICATION_INTERFACE参数，并将其中列出的类的名称（ DefaultRadiusInterface ）替换为您已创建的新类的名称。

    在sqlnet.ora文件中进行此更改时，将在Oracle客户端上加载该类以处理身份验证过程。

  3. 保存并退出sqlnet.ora文件

第三方必须实现OracleRadiusInterface接口，该接口位于ORACLE.NET.RADIUS包中。

示例：使用OracleRadiusInterface接口

您可以使用OracleRadiusInterface接口来检索用户名和密码。

示例D-1显示了如何使用OracleRadiusInterface接口。

示例D-1使用OracleRadiusInterface接口

公共接口OracleRadiusInterface {public void radiusRequest（）; public void radiusChallenge（String challenge）; public String getUserName（）; public String getPassword（）; }

在本规范中：

  • radiusRequest提示最终用户输入用户名和密码，稍后将通过getUserName和getPassword检索。

  • getUserName提取用户输入的用户名。如果此方法返回空字符串，则假定用户想要取消操作。然后，用户接收指示认证尝试失败的消息。

  • getPassword提取用户输入的密码。如果getUserName返回有效字符串，但getPassword返回空字符串，则challenge关键字将被数据库替换为密码。如果用户输入有效密码，则RADIUS服务器可能会或可能不会返回质询。

  • radiusChallenge提供从RADIUS服务器发送的请求，以便用户响应服务器的质询。

  • getResponse提取用户输入的响应。如果此方法返回有效响应，则该信息将填充新的Access-Request数据包中的User-Password属性。如果返回空字符串，则通过返回相应的值从两侧中止操作。
