1.140 INMEMORY_EXPRESSIONS_USAGE

INMEMORY_EXPRESSIONS_USAGE控制将内存中表达式（IM表达式）填充到内存中列存储（IM列存储）中并可用于查询。

参数类型

串

句法

INMEMORY_EXPRESSIONS_USAGE = {STATIC_ONLY | DYNAMIC_ONLY | ENABLE |禁用}

默认值

ENABLE

可修改

改变系统

可在PDB中修改

是

基本

没有

Oracle RAC

必须在所有实例上使用相同的值。

此参数的四个值为：

  • STATIC_ONLY ：为内存启用并包含某些数据类型（如Oracle编号或JSON）的表将使用更高效的表示形式在IM列存储中填充这些列。请注意，此设置将增加某些表的内存占用空间。静态配置使IM列存储能够缓存OSON（二进制JSON）列，这些列标记有IS_JSON检查约束。在内部，OSON列是名为SYS_IME_OSON的隐藏虚拟列。相反，动态配置会自动创建和填充常用表达式。

  • DYNAMIC_ONLY ：如果与PL / SQL过程DBMS_INMEMORY.IME_CAPTURE_EXPRESSIONS一起使用，IM表达式将自动创建并填充到IM列存储中。请注意，设置此值将增加某些表的内存占用空间。不应在Oracle Cloud环境中使用此值。

  • ENABLE ：静态和动态IM表达式都将填充到IM列存储中，并可供查询使用。请注意，设置此值将增加某些表的内存占用空间。这是默认值。不应在Oracle Cloud环境中使用此值。

  • DISABLE ：不会将任何类型的IM表达式填充到IM列存储中。

更改参数的模式不会立即影响现有的内存数据。例如，如果inmemory_expressions_usage子句从DYNAMIC_ONLY更改为DISABLE ，则不会立即删除存储的IM表达式。下一次重新填充不会带回内存中的表达式，从而有效地删除表达式。
