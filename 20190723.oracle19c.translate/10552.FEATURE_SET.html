目的

FEATURE_SET为选择中的每一行返回一组要素ID和要素值对。返回值是具有字段名称FEATURE_ID和VALUE的对象的varray。两个字段的数据类型均为NUMBER 。

topN和cutoff

您可以指定topN和cutoff来限制函数返回的topN数。默认情况下， topN和cutoff都为null，并返回所有要素。

  • topN是N最高价值的特征。如果多个要素具有第N个值，则该函数选择其中一个。

  • cutoff是一个值阈值。仅返回大于或等于cutoff要素。要仅按cutoff过滤，请为topN指定NULL 。

要返回大于或等于cutoff N要素，请同时指定topN和cutoff 。

语法选择

FEATURE_SET可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，也可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态挖掘数据。选择语法或分析语法 ：

  • 语法 - 使用第一种语法使用预定义的模型对数据进行评分。提供特征提取模型的名称。

  • 分析语法 - 使用分析语法在没有预定义模型的情况下对数据进行评分。包括INTO n ，其中n是要提取的mining_analytic_clause数，以及mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见“ analytic_clause :: = ” 。）

在对分区模型进行评分时， FEATURE_SET函数的语法可以使用可选的GROUPING提示。请参阅GROUPING提示 。

mining_attribute_clause

mining_attribute_clause标识要用作评分预测变量的列属性。使用分析语法调用函数时，这些预测变量也用于构建瞬态模型。mining_attribute_clause行为与PREDICTION函数的描述相同。（参见“ mining_attribute_clause :: = ” 。）
