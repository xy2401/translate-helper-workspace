Oracle Text for Oracle数据库版本18c，版本18.1中的更改

本主题描述了Oracle Text for Oracle Database 18c版本18.1中的更改。

新功能

本节介绍Oracle Database 18c版本18.1中引入的Oracle Text的主要新功能。

分面导航支持

Oracle Text提供分面导航支持。您现在可以使用SDATA部分和结果集接口查询来构建具有分面导航支持的应用程序。

也可以看看：

使用分面导航

支持高效的通配符搜索

通配符索引支持对所有通配符表达式进行快速有效的通配符搜索。新的wordlist首选项WILDCARD_INDEX,替换当前选项（ SUBSTRING_INDEX, PREFIX_INDEX,和REVERSE_INDEX ）。启用WILDCARD_INDEX ，K-gram（固定长度子串粒子）索引会索引每个标记中的所有子字符串。新的词汇表首选项WILDCARD_INDEX_K,控制使用的克数（即每个索引的子字符串的长度）。

仅使用单字节字符的语言支持通配符索引。

也可以看看：

  • BASIC_WORDLIST示例：启用通配符索引

  • 有关BASIC_WORDLIST属性表以及WILDCARD_INDEX和WILDCARD_INDEX_K属性的更多信息，请BASIC_WORDLIST Oracle Text Reference

自动背景索引维护

当$G表格过于分散时，查询性能会下降。为避免这种情况，Oracle Text现在为每个索引或分区提供自动后台优化合并。

如果索引同步已运行一段时间，则索引表可能会碎片化。为避免此问题，Oracle Text现在运行自动后台作业以优化各种索引表。

也可以看看：

  • 有关ALTER INDEX语句的更多信息，请参阅Oracle Text Reference

  • 有关CREATE INDEX语句的更多信息，请参阅Oracle Text Reference

支持并发数据操作语言操作

对于使用SYNC (ON COMMIT)选项创建的索引，同步是作为同一事务的一部分执行的。如果存在致命索引同步错误，则回滚整个数据事务。CTX_USER_INDEX_ERRORS视图中记录了非致命（单个行）同步错误，但事务仍然完成。

  • Oracle Text Reference

CTX_DDL.OPTIMIZE_INDEX过程有两个新参数， maxtokens和section_type.

也可以看看：

Oracle Text Reference

支持超过64个字符的JSON密钥名称的索引

除CTXCAT和CTXRULE索引之外的所有Oracle Text索引类型现在都在VARCHAR2 (255 BYTE)类型的表列中存储标记。SDATA部分继续在VARCHAR2 (249 BYTE).类型的表列VARCHAR2 (249 BYTE).存储令牌VARCHAR2 (249 BYTE).

如果它们索引包含名称长度超过64字节的对象字段的JSON数据，则必须重建在Oracle Database 18c之前创建的任何JSON搜索索引和Oracle Text索引。有关更多信息，请参见Oracle数据库升级指南

Oracle Text 12 c第2版（12.2.0.1）中的更改

本主题描述了Oracle Text for Oracle Database 12 c第2版（12.2.0.1）中的更改。

新功能

本节介绍Oracle Database 12 c第2版（12.2.0.1）中引入的Oracle Text的主要新功能。

SDATA部分改进

Oracle Text为SDATA部分操作提供了增强功能。添加了一种新的SDATA部分。

也可以看看：

SDATA部分

在索引中保留更新的文档

Oracle Text可以保留更新的索引条目，以使用ASYNCHRONOUS_UPDATE选项搜索原始内容。

也可以看看：

关于索引列的更新

DML改进

Oracle Text支持新的存储首选项SMALL_R_ROW,用于索引查找。

Oracle Text停止了锁定的基表行。每个索引或分区的新$U表跟踪所有并发更新，并为此$U表引入新的$U_TABLE_CLAUSE存储子句。

也可以看看：

  • 有关BASIC_STORAGE属性表和SMALL_R_ROW属性的更多信息，请BASIC_STORAGE Oracle Text Reference

  • 查询涉及哪些表？

左截断查询的反向令牌索引

Oracle Text为左截断查询提供了新的REVERSE_INDEX属性。此属性是wordlist首选项的一部分，可以设置为TRUE或FALSE.默认设置为FALSE ，以禁用新功能。您可以使用CTX_DDL.SET_ATTRIBUTE过程设置此属性。您也可以使用ALTER INDEX REBUILD,添加它ALTER INDEX REBUILD,就像任何其他wordlist首选项一样。如果要为左截断查询提供更好的查询性能，请使用此属性。

也可以看看：

有关BASIC_WORDLIST属性表和REVERSE_INDEX属性的更多信息，请BASIC_WORDLIST Oracle Text Reference

特定于分区的近实时索引

Oracle Text支持特定于分区的STAGE_ITAB选项，该选项提供两级索引机制，以防止主索引因频繁插入，更新或删除而碎片化。例如，如果分区主要包含静态数据，则在分区级别设置此选项，而其他分区包含快速更改的数据。

STAGE_ITAB_PARALLEL存储选项控制用于将$G登台表中的数据合并回$I表的并行度。

要防止近实时$G索引表变得太大而无法放入内存，请指定表的最大大小。

也可以看看：

使用STEX_ITAB，STAGE_ITAB_MAX_ROWS和STAGE_ITAB_PARALLEL改进响应时间CONTEXT索引的选项

情感分析与配置

Oracle Text支持情绪分析和配置。通过情感分析，您可以识别与搜索字词相关的正面和负面趋势。通过Collocates，您可以识别与指定关键字相关或经常使用的其他关键字。

NEAR2运算符和NDATA运算符增强功能

Oracle Text提供了一个新的运算符NEAR2,它是现有NEAR运算符的增强版本。NEAR2运算符旨在结合PHRASE, NEAR,和AND运算符的语义。

NDATA操作员现在可以更好地控制角色和语音匹配的相似性得分。它还可以对操作返回的整体排名结果提供更多控制。

Oracle Text为日语VGRAM词法分析器和WORLD词法分析器提供连接字符支持。

新文档格式

Oracle Text提供了新的文本过滤器以支持新的文档格式。

提取文档中单词的同义词

Oracle Text在CTX_DOC包中提供了新选项以启用词库支持。您可以使用CTX_DOC.TOKENS和CTX_DOC.POLICY_TOKENS过程来提取索引标记的同义词。

只读MDATA部分

Oracle Text支持只读MDATA部分。查询节时，不会为每个MDATA运算符打开额外的游标。由于无法在不可更新的MDATA部分中添加或删除MDATA值，因此跟踪更新的MDATA值不会产生额外开销，并且查询运行得更快。

也可以看看：

MDATA部分

索引名称长度和长标识符支持Oracle Text Objects

Oracle Text索引名称可以与数据库对象名称一样长。Oracle Database 12.2兼容及后续版本的长度为128字节，早期版本的长度为30字节。Oracle Text支持Oracle Text对象的长标识符。最大大小增加到128个字节。

增加MAX_INDEX_MEMORY参数的默认值和上限

Oracle Text提供了MAX_INDEX_MEMORY参数的默认值和上限的MAX_INDEX_MEMORY ，可以为索引目的分配MAX_INDEX_MEMORY参数。大小增加到256 GB。

您可以使用更简单的替代语法在JSON上创建搜索索引。
