1.16 ARCHIVE_LAG_TARGET

ARCHIVE_LAG_TARGET在指定的时间量过后强制执行日志切换。

属性 描述

参数类型

整数

默认值

0 （禁用）

可修改

改变系统

可在PDB中修改

没有

价值范围

0或[60,7200]中的任何整数

基本

没有

Oracle RAC

多个实例应使用相同的值

0值禁用基于时间的线程前进功能;否则，该值表示秒数。大于7200秒的值对于在备用数据库中保持合理的延迟没有多大用处。典型或推荐值为1800（30分钟）。极低的值会导致频繁的日志切换，从而降低性能;此类值还可能使归档程序进程过于繁忙，无法归档连续生成的日志。
