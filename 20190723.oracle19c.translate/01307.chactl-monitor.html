D.1 chactl监视器

使用chactl monitor命令可以使用当前集模型开始监视特定Oracle Real Application Clusters（Oracle RAC）数据库的所有实例。

Oracle Cluster Health Advisor使用分配给数据库的相同模型监视此数据库的所有实例。

当您第一次开始监视目标时，Oracle Cluster Health Advisor使用Oracle提供的黄金模型。Oracle Cluster Health Advisor存储内部存储中目标的监视状态。当Oracle Cluster Health Advisor检测或重新检测新实例时，Oracle Cluster Health Advisor会开始监视任何新的数据库实例。

句法

chactl监控数据库-db db_unique_name [-model model_name [-force] [ -求助]
chactl监视器簇[-model model_name [-force]]

参数

表D-1 chactl monitor命令参数

参数 描述

DB_UNIQUE_NAME

指定数据库的名称。

型号名称

指定模型的名称。

力

使用-force选项可以使用指定的模型进行监视，而无需停止监视目标。

如果没有-force选项， chactl unmonitor首先运行chactl unmonitor ，然后使用模型名称运行chactl monitor 。

例子

  • 使用BlkFridayShopping默认模型监视SalesDB数据库：

    $ chactl monitor database -db SalesDB -model BlkFridayShopping

  • 使用Nov2014模型监控InventoryDB数据库：

    $ chactl monitor database -db InventoryDB -model Nov2014

    如果指定model_name ，则Oracle Cluster Health Advisor将使用指定的模型开始监视，并将模型存储在Oracle Cluster Health Advisor内部存储中。

    如果同时使用–model和–force选项，则Oracle Cluster Health Advisor将停止监视并使用指定的模型重新启动监视。

  • 要使用Dec2014模型监视SalesDB数据库：

    $ chactl monitor database -db SalesDB -model Dec2014

  • 使用Dec2014模型和-force选项监视InventoryDB数据库：

    $ chactl监控数据库-db InventoryDB -model Dec2014 -force

错误消息

错误： 群集中没有运行CHA资源。

描述：当没有运行Oracle Cluster Health Advisor服务的中心或叶节点时返回。

错误： 未配置数据库。

描述：在Oracle Cluster Health Advisor配置库或CRS资源中找不到数据库时返回。

错误： 输入字符串“xc＃？％“ 是无效的。

描述：无法解析命令行时返回。还显示顶级帮助文本。

错误： CHA已经在监视目标<dbname>。

描述：在已监视数据库时返回。
