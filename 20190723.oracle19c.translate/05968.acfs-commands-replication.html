acfsutil repl bg

目的

启动，停止或显示有关Oracle ACFS复制后台进程的信息。

语法和说明

acfsutil repl bg -h acfsutil repl bg {start |停止| info} [ snap_shot @ ] mount_point

acfsutil repl bg -h显示帮助文本并退出。

下表包含acfsutil repl bg命令可用的选项。

表16-31 acfsutil repl bg命令的选项

选项 描述

开始

启动复制后台处理。

停

停止复制后台处理

信息

通过关联的集群件资源显示有关复制后台进程的信息。如果资源在主群集中的节点上处于ONLINE状态，则该进程正在运行。否则，该进程未运行。

[ snap_shot@ ] mount_point

指定要复制的位置，装入点或已装入的主文件系统的快照。

Oracle ACFS复制使用主群集上的后台进程将文件系统更改传输到备用位置（文件系统或快照）。必须运行此过程才能使复制正常运行。使用acfsutil repl init命令启动复制时，将启动该进程并向Oracle Clusterware注册，从而在重新引导或系统崩溃后启用自动重新启动。这个过程是集群范围的。

要确定主站点上是否正在运行Oracle ACFS复制后台进程，可以在主群集上运行acfsutil repl bg info命令。有关使用该命令的示例 ，请参见例16-27 。

注意：

即使其复制后台进程正在运行，也始终可以卸载包含参与复制的位置的主文件系统。如果需要启用卸载，则umount命令（Windows上的acfsdismount命令）将停止后台进程。任何正在进行的复制操作都会中断。当重新装入包含参与复制的位置的主文件系统时，复制会自动重新启动并重新启动后台进程。

仅当复制操作当前未在其上进行时，才能卸载包含参与复制的位置的备用文件系统。如果正在进行复制操作，则尝试卸载文件系统将失败，并指示文件系统正忙。

acfsutil repl比较

目的

验证是否已在Oracle ACFS存储位置复制文件。

语法和说明

acfsutil repl compare -h acfsutil repl compare [-v] [-a [-s] | -t {all | tagname ，...} [-s]] [ snap_shot @ ] primary_mount_point [ snap_shot @ ] standby_mount_point

acfsutil repl compare -h显示帮助文本并退出。

下表包含acfsutil repl compare命令可用的选项。

表16-32 acfsutil repl compare命令的选项

选项 描述

-一个

比较所有文件及其扩展属性名称。

-t { all | tagname ， ... }

将文件与acfsutil repl init操作中指定的全部或部分标记名进行比较。all选项指定acfsutil repl init操作中提供的所有标记名称。以逗号分隔的标记名称列表必须仅使用在acfsutil repl init操作期间指定的标记名称。

-s

使用-a或-t选项时，跳过对符号链接的扩展属性和标记的比较。仍会检查符号链接的所有其他内容，例如权限和所有权。

[ snap_shot @ ] primary_mount_point

指定要复制的位置，装入点或已装入的主文件系统的快照。

[ snap_shot @ ] standby_mount_point

指定要复制的位置，装入点或装入的备用文件系统的快照。

acfsutil repl compare命令验证主要位置的全部或部分是否已复制到备用位置。必须在本地安装备用位置才能进行比较。当主要位置尽可能少地变化时，获得最佳结果。该命令始终检查主位置上的所有文件与备用位置上的文件。

在acfsutil repl init操作期间未指定标记名称时，通常使用-a选项。-a选项将主要位置上的所有文件与备用位置上的文件进行比较。-a选项还可以在备用位置上测试主数据库上不存在的额外文件。在测试额外文件时，该命令还会尝试检查扩展属性。但是，如果使用NFS在本地安装备用位置，则由于NFS协议的限制，无法检查备用数据库是否匹配扩展属性。

在acfsutil repl init操作期间指定标记时使用-t选项。-t操作使用指定的标记名称查找主位置上的所有文件名，并将它们与备用数据库上的相应文件进行比较。-t选项将all选项或逗号分隔的标记名称列表作为参数。如果-t选项与all参数一起使用，则选择在acfsutil repl init操作期间提供的所有标记名称。否则，任何指定的标记名称必须与acfsutil repl init操作期间提供的标记名称完全匹配。-t选项还测试备用位置上的额外文件，这些文件没有在acfsutil repl init操作期间指定的关联标记名称。如果使用NFS在本地挂载备用位置，则由于NFS协议的限制，无法检查备用数据库是否匹配标记名称和扩展属性。

您可以使用acfsutil repl info -c选项来确定在acfsutil repl init操作期间指定了哪些标记。

-a和-t选项不能同时指定。提供-a或-t ，还可以指定—s选项以跳过符号链接的扩展属性比较。如果既未提供-a也未提供-t选项，则在不测试标记名称和扩展属性的情况下完成主备文件比较。

可以指定-v选项以在比较文件之后输出每个文件的名称。

[ snap_shot@ ] mount_point

要显示有关复制统计信息的信息，请使用带-s选项的acfsutil repl info 。

要显示有关所有复制事件的信息，请使用不带选项的acfsutil repl info 。要显示有关特定类型事件的信息，请使用带-a ， -e或-t选项的acfsutil repl info 。

表16-34 acfsutil repl init命令的选项

选项 描述

主

指定主要位置的复制。

支持

指定备用位置的复制。

-s repluser @ 网络名

指定主站点与备用站点联系的标准ssh连接字符串。

连接字符串中的repluser是ssh在备用节点上登录的用户。选择最低权限的用户身份。用户应具有Oracle ASM管理员权限，并且应属于Oracle DBA组。

注意：

必须为主群集和备用群集上的复制用户指定相同的用户和组标识（包括所有uids和gids ）。

在连接字符串中的网络名指定网络端点，如主机名，VIP名称或IP地址。

-C

指定以常量模式复制。快照不断生成并复制到备用站点。每次快照的复制完成后，就会启动新快照的生成。

-M

指定手动模式下的复制。执行初始复制操作后，在通过运行acfsutil repl sync手动请求之前，不会再进行复制。

—i interval

指定间隔（调度）模式下的复制。如果可能，将以指定的频率拍摄并复制新快照。必须给出后缀以指定测量间隔的单位。后缀必须是s （秒）， m （分钟）， h （小时）， d （天）或w （星期）。例如， 30m是30分钟， 2h是2小时。

标签名称

指定要用于限制复制的一个或多个标记名称。指定标记名称时，仅复制标记有至少一个标记名称的文件。有关Oracle ACFS标记的信息，请参阅Oracle ACFS标记 。

-m [ snap_shot@ ] standby_mount_point

指定备用存储位置（如果它与主位置不同）。

-d trace_level

指定跟踪级别设置[0..6]。

-z on | off

打开或关闭从主数据库发送到备用数据库的复制数据流的压缩。

-o sshCmdPath= pathname

指定ssh命令的路径名。

-o sshStrictKey= ynvalue

指定ssh是否应使用严格的主机密钥检查。以y开头的值启用此检查，这是默认设置。以n开头的值将禁用检查。

-o sshCipher= ciphername

指定传递给ssh以加密其会话的密码。

-u

指定repluser 。

[ snap_shot @ ] standby_mount_point

指定要复制的位置，装入点或装入的备用文件系统的快照。

[ snap_shot @ ] primary_mount_point

指定要复制的位置，装入点或已装入的主文件系统的快照。

acfsutil repl init primary和standby命令在指定的主站点和指定的备用站点之间启动复制。必须首先运行acfsutil repl init standby命令。接下来，运行acfsutil repl init primary命令。

当acfsutil repl init primary命令成功完成时，处理开始将所有指定文件的初始副本复制到备用位置。此外，对这些文件的任何更改以及随后在主存储位置中创建的任何新文件都将被捕获并传输到备用位置。但是，成功完成该命令并不表示指定的文件已完全复制到备用位置。要监视所有指定文件初始复制到备用位置的acfsutil ，用户可以运行acfsutil repl info -c命令。

初始复制操作基于主要位置的快照，该快照完全传输到备用位置。每个后续复制操作都使用主要位置的更高版本快照，并且仅将该快照与上一个上一个快照之间的差异传输到备用位置。

即使指定了标记，主要位置中的所有目录也始终复制到备用数据库。

符号链接文件按原样复制。如果符号链接解析为绝对路径名，并且备用位置上不存在该路径名，则引用符号链接会导致错误。

在支持命名套接字，字符设备文件或块设备文件的平台上，不会复制这些文件类型。

包含主存储位置或备用存储位置的文件系统只能安装在其群集中的一个装入点上。文件系统不必安装在集群中的所有节点上，但如果已挂载，则必须将其挂载在使用acfsutil repl init命令指定的挂载点上。此外，其他节点上的挂载点上不能安装其他文件系统。

单个Oracle ACFS位置不能同时配置为主要和备用位置。如果在同一主机或同一群集中配置了复制，则主要和备用位置不能使用相同的命名安装点。

如果托管主要或备用位置的站点在Windows上运行，则两个站点必须在Windows上运行。在运行Linux，Solaris或AIX操作系统的任意组合的站点之间允许复制。

acfsutil repl反向

acfsutil repl upgrade命令支持升级活动的现有复制关系以使用基于快照的复制。升级过程的目的是在当前主文件系统和备用文件系统之间提供基于快照的活动复制关系，而无需终止或重新启动复制，也不会丢失复制过程中的任何数据。

在开始升级过程之前，必须确保主文件系统仅安装在其群集的一个节点上。您应该停止针对主文件系统或备用文件系统的任何应用程序活动，以确保最佳升级;但是，如果忽略此建议，则升级将继续。

注意：

升级完成后，必须完成该过程。可以使用-u选项撤消acfsutil repl upgrade prepare操作。但是，在启动acfsutil repl upgrade standby操作后，必须完成整个升级过程。从那时起，没有提供停止或撤消升级的规定。

要开始升级过程，请首先在主群集上运行acfsutil repl upgrade prepare命令。此命令指定用于基于快照的复制的用户和主机或接口名称以及主装入点。用户名和主机名使用—s选项给出，与基于快照的复制的acfsutil repl init primary命令选项相同。

接下来，升级备用站点。在备用数据库上，运行acfsutil repl upgrade standby命令。此命令指定用于基于快照的复制的备用安装点。

发出此命令后，必须通过发出acfsutil repl upgrade primary命令完成upgrade 。

在备用站点上运行acfsutil repl upgrade standby命令后，必须在主站点上运行acfsutil repl upgrade primary命令才能完成基于快照的复制的升级。此命令自动终止原始复制过程并启动基于快照的复制。 acfsutil repl upgrade primary接受acfsutil repl init primary命令为基于快照的复制接受的任何命令行选项，但-m选项， -s选项和标记名除外。-m和-s选项的必要信息已从其他来源获得。为原始复制关系指定的任何标记名称都将自动转移到新关系。有关Oracle ACFS标记的信息，请参阅Oracle ACFS标记 。

在支持命名套接字，字符设备文件或块设备文件的平台上，基于快照的复制不会复制这些文件类型。但是， acfsutil repl upgrade过程会保留备用文件系统上已存在的所有这些类型的文件。它们不会进一步更新，但不会作为升级的一部分删除。
