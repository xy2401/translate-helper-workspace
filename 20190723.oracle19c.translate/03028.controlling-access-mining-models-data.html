34.4.1创建数据挖掘用户

说明如何创建数据挖掘用户。

数据挖掘 user是一个数据库用户帐户，具有执行数据挖掘活动的权限。例34-6显示了如何创建数据库用户。例34-7显示了如何为用户分配数据挖掘权限。

注意：

要为Data Mining示例程序创建用户，必须运行两个 “数据挖掘示例程序”中描述的配置脚本。

示例34-6在SQL * Plus中创建数据库用户

  1. 使用登录到SQL * Plus 系统特权。 

    输入用户名：sys as sysdba输入密码： password

  2. 要创建名为dmuser的用户，请键入这些命令。指定您选择的密码。 

    创建用户dmuser标识密码默认表格用户临时表格TEMP QUOTA无限用户;承诺;

    USERS和TEMP表空间包含在Oracle随数据库介质提供的预配置数据库中。USERS主要由演示用户使用;它适用于运行“数据挖掘示例程序”中描述的示例程序 。TEMP是大多数数据库用户共享的临时表空间。

    注意：

    必须根据标准DBA实践分配数据挖掘用户的表空间，具体取决于系统负载和系统资源。

  3. 要以dmuser身份登录，请键入以下内容。 

    CONNECT dmuser输入密码： password

34.4.1.1授予数据挖掘权限

您一定有 CREATE MINING MODEL创建权限您自己架构中的模型。您可以对您拥有的模型执行任何操作。这包括应用模型，添加成本矩阵，重命名模型以及删除模型。

以下示例中的GRANT语句将一组基本数据挖掘权限分配给dmuser帐户。所有采矿活动都不需要其中一些特权，但谨慎的做法是将它们全部作为一个组进行授权。

启用或限制特定挖掘活动需要其他系统和对象特权。

示例34-7数据挖掘所需的权限

授予dmuser创建采矿模型;授予dmuser创建会话;授予dmuser创建表;授予dmuser创建视图;在CTXSYS.CTX_DDL上执行GRANT EXECUTE到dmuser;

READ或SELECT权限对于不在架构中的数据所必需的。例如，以下语句授予对sh.customers表的SELECT访问权限。 

GRUC选择on sh.customers TO dmuser;

34.4.2数据挖掘的系统权限

学习不同的权限来控制挖掘模型的操作。

一个系统特权赋予在数据库中执行特定操作或对一类模式对象执行操作的权限。例如，创建表空间和删除数据库中任何表的行的权限是系统特权。

如果您具有适当的系统特权，则可以在其他模式中对挖掘模型执行特定操作。例如， CREATE ANY MINING MODEL允许您在其他模式中创建模型。SELECT ANY MINING MODEL允许您应用驻留在其他模式中的模型。如果您具有COMMENT ANY MINING MODEL特权，则可以向模型添加注释。

要授予系统特权，您必须已使用ADMIN OPTION授予系统特权，或者已被授予GRANT ANY PRIVILEGE系统特权。
