12.21编辑

句法

ED [IT] [ file_name [。 分机 ]]

其中file_name [。 分机 ] 表示您要编辑的文件（通常是脚本 ）。

调用一个操作系统文本编辑器在指定文件的内容上或者关于缓冲区的内容 。

输入没有文件名的 EDIT来编辑SQL缓冲区的内容使用操作系统文本编辑器。

用法

如果省略文件扩展名，则SQL * Plus将采用默认的命令文件扩展名（通常为SQL）。有关更改默认扩展名的信息，请参阅 SET命令的SUFFIX变量。

如果指定文件名 ，SQL * Plus将在ORACLE_PATH设置的目录中搜索该文件。如果SQL * Plus无法在ORACLE_PATH中找到该文件，或者未设置ORACLE_PATH，则会在当前工作目录中搜索该文件。如果SQL * Plus无法在任一目录中找到该文件，则会创建具有指定名称的文件。

替换变量， _编辑 ，包含EDIT调用的文本编辑器的名称。您可以通过更改_EDITOR的值来更改文本编辑器。有关更改替换变量值的信息，请参阅DEFINE 。如果未定义_EDITOR，EDIT将尝试运行默认操作系统编辑器。

EDIT默认情况下将SQL缓冲区的内容放在名为AFIEDT.BUF的文件中（在当前工作目录中），并对文件内容运行文本编辑器。如果AFIEDT.BUF文件已存在，则会被缓冲区的内容覆盖。您可以使用SET EDITFILE命令更改默认文件名。有关为EDIT命令设置默认文件名的更多信息，请参阅SET命令的EDITFILE变量。

注意：

默认文件AFIEDT.BUF在某些操作系统上可能具有不同的名称。

如果未指定文件名且缓冲区为空，则EDIT将返回错误消息。

在Windows上的SQL * Plus 9.0及更早版本中，命令EDIT *在给出无效的文件名警告后打开了一个空白文件。在Windows上的SQL * Plus 10及更高版本中， EDIT *提供无效的文件名警告，并且不会打开空白文件。要保留SQL * Plus 9.0行为，请输入set命令，

SET SQLPLUSCOMPATIBILITY 9.0

要退出编辑会话并返回SQL * Plus，请按照文本编辑器的惯例终止编辑会话。当您离开编辑器时，SQL * Plus会将文件的内容加载到缓冲区中。

注意：

在Windows中，如果您使用写字板作为编辑器（ _EDITOR = write.exe），退出写字板时不重新加载缓冲区。在这种情况下，使用GET重新加载缓冲区。
