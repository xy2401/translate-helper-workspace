Oracle Database Release 19c，版本19.1中的更改

以下是Oracle数据库管理员指南（适用于Oracle数据库版本19c，版本19.1）中的更改。

  • 新功能
  • 不受支持的功能

上级主题： “Oracle数据库管理员指南”的本发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • Oracle数据库提供的用户帐户现在是仅模式帐户

    除了SYS和示例模式之外，大多数Oracle数据库提供的用户帐户现在都是仅模式帐户，也就是说，这些帐户是在没有密码的情况下创建的。这可以防止恶意用户登录这些帐户。您可以在需要对这些帐户进行身份验证时为其分配密码。要查找帐户的状态，请查询DBA_USERS数据字典视图的ACCOUNT_STATUS列。如果该帐户仅为架构，则状态为NONE 。

    请参阅“ 预定义的用户帐户 ” 。

  • 自动索引

    自动索引功能可自动执行索引管理任务，例如根据应用程序工作负载的变化在Oracle数据库中创建，重建和删除索引。此功能通过在Oracle数据库中自动管理索引来提高数据库性能。

    请参阅“ 管理自动索引 ” 。

  • 隔离用于消耗过多系统资源的SQL语句的执行计划

    您现在可以为资源管理器终止的SQL语句的执行计划配置隔离，因为它们过多地消耗了系统资源，例如CPU和I / O.不允许再次运行SQL语句的隔离执行计划，从而保护数据库免受性能下降的影响。

    请参阅“ 为消耗过多系统资源的SQL语句执行计划的隔离 ” 。

  • Memoptimized Rowstore - 快速摄取

    Memoptimized Rowstore的快速摄取功能优化了来自应用程序（如物联网（IoT）应用程序）的高频单行数据插入的处理。

    请参阅“ 使用Memoptimized Rowstore启用高性能数据流 ” 。

  • 混合分区表

    Oracle数据库现在支持混合分区表 。混合分区表是一个分区表，其中一些分区驻留在数据库中，一些分区驻留在数据库外部的外部文件中，例如操作系统文件或Hadoop分布式文件系统（HDFS）文件。

    请参见Oracle数据库VLDB和分区指南 。

    另请参阅“ 指定要创建的表的类型 ” 。

  • DBCA中的新功能处于静默模式

    以下是DBCA在静默模式下的新功能：

    • 能够通过克隆远程PDB来创建PDB

      您可以使用DBCA命令createPluggableDatabase的createFromRemotePDB参数通过克隆远程PDB来创建PDB。

      请参见“ createPluggableDatabase ” 。

    • 能够将PDB重新定位到另一个CDB

      您可以使用DBCA命令relocatePDB将PDB重定位到另一个CDB。

      请参阅“ relocatePDB ” 。

    • 能够创建Oracle数据库的副本

      您可以使用DBCA命令createDuplicateDB创建Oracle数据库的副本。

      请参见“ createDuplicateDB ” 。

  • 与SQL语句可诊断性相关的新功能

    SQL诊断和修复工具（如SQL Test Case Builder和SQL Repair Advisor）已得到增强，可为管理有问题的SQL语句提供更好的诊断和修复功能。

    • 新的SQL修复顾问功能DBMS_SQLDIAG.SQL_DIAGNOSE_AND_REPAIR

      新的SQL Repair Advisor函数DBMS_SQLDIAG.SQL_DIAGNOSE_AND_REPAIR创建诊断任务，执行它，并接受SQL补丁建议，该SQL补丁建议生成严重错误。

      请参见Oracle Database PL / SQL包和类型参考 。

      另请参阅“ 使用DBMS_SQLDIAG程序包子程序运行SQL Repair Advisor ” 。

    • 控制SQL Test Case Builder输出的新选项

      您现在可以在DBMS_SQLDIAG.EXPORT_SQL_TESTCASE过程中指定这些新选项来控制SQL测试用例生成器的输出： compress ， diag_event和problem_type 。

      请参阅“ SQL测试用例生成器的输出 ” 。

    • 新视图V$SQL_TESTCASES

      您可以通过查询V$SQL_TESTCASES视图来查看有关SQL Test Case Builder生成的所有测试用例的信息。

      请参见Oracle数据库参考 。

      另请参见“ SQL测试用例生成器的输出 ” 。

  • 用于构建数据字典的新SQL脚本catpcat.sql

    使用CREATE DATABASE语句CREATE DATABASE ，可以使用新脚本catpcat.sql运行catctl.pl ，而不是运行脚本catalog.sql和catproc.sql来构建数据字典。脚本catpcat.sql使用并行进程运行脚本catalog.sql和catproc.sql ，从而提高了手动构建数据字典的性能。

    请参阅“ 步骤11：运行脚本以构建数据字典视图 ” 。

  • 新的ALTER SYSTEM语句子句FLUSH PASSWORDFILE_METADATA_CACHE

    如果数据库密码文件名或位置已更改，则必须使用FLUSH PASSWORDFILE_METADATA_CACHE子句运行ALTER SYSTEM语句才能使更改生效。

    请参阅“ 使用ORAPWD创建数据库密码文件 ” 。

    另请参见Oracle数据库SQL语言参考 。

  • Oracle Database Resource Manager自动为Oracle Database In-Memory启用

    通过将INMEMORY_SIZE初始化参数设置为大于0的值，在启用Oracle Database In-Memory时自动启用资源管理器。

    请参阅“ 启用Oracle数据库资源管理器和切换计划 ” 。

  • 新视图DBA_REGISTRY_BACKPORTS

    新视图DBA_REGISTRY_BACKPORTS显示已应用于数据库的修补程序已修复的错误。此视图中仅显示已修改数据字典的错误修复。

    请参见Oracle数据库参考 。

不受支持的功能

此版本不支持以下功能。

  • Oracle多媒体

    从Oracle Database 19c开始，不支持Oracle多媒体。Oracle建议您将多媒体内容存储在SecureFiles LOB中，并使用第三方产品进行图像处理和转换。

  • Oracle Streams

    从Oracle Database 19c开始，不支持Oracle Streams功能。使用Oracle GoldenGate替换Oracle Streams的所有复制功能。

Oracle Database Release 18c，版本18.1中的更改

以下是Oracle数据库管理员指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

  • 新功能
  • 其他变化

上级主题： “Oracle数据库管理员指南”的本发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • 暗影失去写保护

    阴影丢失写保护提供快速检测和对丢失写入的即时响应。使用阴影丢失写保护可以最大限度地减少数据丢失和修复数据库所需的时间。您可以为数据库，表空间或数据文件启用阴影丢失写保护，而无需Oracle Data Guard备用数据库。

    请参阅“ 使用影子表空间管理丢失的写保护 ” 。

  • Oracle数据库密码文件的新默认位置

    Oracle数据库密码文件的缺省位置是ORACLE_BASE目录而不是ORACLE_HOME目录。此更改支持只读Oracle主目录。

    请参见“ ORAPWD语法和命令行参数说明 ” 。

    请参见“ 关于以只读模式配置Oracle主目录 ” 。

  • 私人临时表

    私有临时表存储在内存中，每个临时表仅对创建它的会话可见。

    请参阅“ 创建临时表 ” 。

  • 内联外部表

    内联外部表使外部表的运行时定义成为SQL语句的一部分，而无需将外部表创建为数据字典中的持久对象。

    请参阅“ 使用内联外部表 ” 。

  • 并行队列超时

    您可以通过设置PQ_TIMEOUT_ACTION资源管理器指令来指定对超时并行SQL语句执行的PQ_TIMEOUT_ACTION 。

    请参见“ 并行队列超时 ” 。

  • 取消会话中的SQL语句

    您可以使用ALTER SYSTEM CANCEL SQL语句在会话中ALTER SYSTEM CANCEL SQL语句。

    请参阅“ 取消会话中的SQL语句 ” 。

  • 可扩展序列

    可伸缩序列可用于提高Oracle RAC数据库中数据加载操作的性能。

    请参阅“ 使序列可扩展 ” 。

  • 重置序列

    可以在ALTER SEQUENCE语句中使用新的RESTART子句将序列号重置为特定值。

    请参阅“ 更改序列 ” 。

  • Memoptimized Rowstore

    Memoptimized Rowstore可以快速查找主要基于主键列查询的表的数据。此功能对物联网（IoT）等应用程序特别有用。

    请参阅“ 使用Memoptimized Rowstore启用高性能数据流 ” 。

  • 只读Oracle主页

    可以以只读模式配置Oracle主目录，从而防止在Oracle主目录ORACLE_HOME内创建或修改文件。只读Oracle主目录可用作可在多个数据库服务器之间共享的软件映像。这简化了修补和批量部署，因为只需更新一个Oracle主映像即可将补丁分发到多个数据库服务器。

    请参见“ 关于以只读模式配置Oracle主目录 ” 。

  • 使用Oracle自动存储管理（Oracle ASM）克隆多租户可插拔数据库（PDB）

    Oracle ASM可用于创建PDB的克隆。

    请参阅Oracle Multitenant管理员指南 。

  • 代理驻留连接池

    代理驻留连接池为数据库客户端提供了改进的高可用性，安全性和性能。代理驻留连接池使用可在流量控制器模式下使用Oracle Connection Manager配置的代理驻留连接池。

    请参阅“ 关于代理驻留连接池 ” 。

  • Microsoft Active Directory服务与Oracle数据库的集成

    Oracle数据库可以直接连接到Microsoft Active Directory并对用户进行身份验证和授权。与使用Oracle Enterprise User Security（EUS）和Oracle目录服务的全功能集成相比，这提供了更简单，更简单的选项。

    请参阅“ 关于集中用户和权限管理 ” 。

  • Active Data Guard的新nologging子句

    数据库nologging已得到扩展，可为Active Data Guard环境中的使用提供更好的支持，而不会显着增加重做日志生成的数量。引入了以下两种新的nologging模式作为现有nologging模式NOLOGGING替代方案：

    用于数据可用性的备用数据可以为负载性能提供支持

    请参阅“ FORCE LOGGING模式的性能注意事项 ” 。

  • DBCA静默模式的新功能

    • 创建多租户容器数据库（CDB）的副本

      可以使用-createDuplicateDB命令创建CDB的副本。

      请参见“ createDuplicateDB ” 。

    • 创建Oracle RAC数据库的副本

      可以使用-createDuplicateDB命令选项-databaseConfigType创建Oracle RAC数据库的副本，其值为RAC或RACONENODE 。

      请参见“ createDuplicateDB ” 。

    • 从现有数据库创建备用数据库

      可以使用-createDuplicateDB命令选项-createAsStandby从现有数据库创建备用数据库。

      请参见“ createDuplicateDB ” 。

    • 在创建数据库之前验证硬件和软件先决条件

      可以通过在需要创建数据库的系统上执行-executePrereqs命令来验证创建数据库的硬件和软件先决条件。

      请参阅“ executePrereqs ” 。

    • 创建数据库模板的副本

      可以使用-createTemplateFromTemplate命令创建数据库模板的副本。

      请参阅“ createTemplateFromTemplate ” 。

  • 创建可插拔数据库（PDB）的副本

    可以使用-createPluggableDatabase命令选项-createPDBFrom创建PDB的副本，其值为PDB 。

    请参见“ createPluggableDatabase ” 。

Oracle Database 12 c第2版（12.2）中的更改

以下是Oracle数据库管理员指南（适用于Oracle Database 12 c第2版）（12.2）中的更改。

  • 新功能

上级主题： “Oracle数据库管理员指南”的本发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • 对表的在线重新定义的改进：

    • 在以前的版本中，无法在线重新定义具有BFILE列的表。在Oracle Database 12 c第2版（12.2）中，可以在线重新定义具有BFILE列的表。

    • 对于未更改表的形状的联机表重新定义，您可以启用表的回滚以将表返回到其原始定义，并保留对表所做的DML更改。

      请参阅“ 回滚在线表重新定义 ” 。

    • 通过在REDEF_TABLE过程或START_REDEF_TABLE过程中将refresh_dep_mviews参数设置为Y ，可以在联机表重新定义期间刷新相关的快速可刷新物化视图。

      请参见“ 在线表重新定义期间刷新从属物化视图 ” 。

    • 优化的批量更新的性能与一个表EXECUTE_UPDATE在程序DBMS_REDEFINITION包。性能已优化，因为更新未记录在重做日志中。

      请参阅“ 优化批量更新的性能 ” 。

    • 您可以查询V$ONLINE_REDEF视图以监视联机表重定义操作的进度。

      请参阅“ 监控联机表重新定义进度 ” 。

    • 当联机表重新定义失败时，通常可以纠正导致失败的问题并重新启动上次停止的联机重新定义过程。

      请参阅“ 失败后重新启动联机表重新定义 ” 。

  • 使用资源管理器限制PGA使用

    资源管理器可以限制可以分配给特定使用者组中每个会话的PGA内存量。

    请参阅“ 程序全局区域（PGA） ” 。

  • 索引压缩的改进

    除了过去版本中可用的低级别之外，您还可以指定高级别的高级索引压缩。高级别的高级索引压缩比低级别节省更多空间。

    请参阅“ 使用高级索引压缩创建索引 ” 。

  • 可以为阵列插入启用混合列压缩

    使用混合列压缩可以压缩插入数组插入的行。在以前的版本中，只能使用Hybrid Columnar Compression压缩插入直接路径INSERT行。

    请参阅“ 关于表压缩 ” 。

  • 表移动操作的改进

    当ONLINE关键字包含在ALTER TABLE MOVE语句中时，在移动操作期间支持数据操作语言（DML）操作。此外，当包含ONLINE关键字并包含UPDATE INDEXES子句时，索引在移动操作期间可用。

    请参阅“ 关于将表移动到新段或表空间 ” 。

  • 新的SYSRAC职责分离管理权限

    Oracle数据库现在为与Oracle Real Application Clusters（Oracle RAC）操作相关的任务提供新的管理权限。

    请参阅“ 管理权限 ” 。

  • 用于长时间运行的事务的新数据库驻留连接池参数

    为防止由于MAX_THINK_TIME参数中指定的限制而回滚长时间运行的事务， DBMS_CONNECTION_POOL包中的子程序的新参数MAX_TXN_THINK_TIME指定正在进行事务的任何会话的最长时间（以秒为单位）。

    请参阅“ 数据库驻留连接池的配置参数 ” 。

  • 有关数据库驻留连接池中每个连接的状态的其他信息

    添加到V$CPOOL_CONN_INFO视图的新列提供了有关连接池中每个连接的当前状态的更多信息。

    请参阅“ 确定连接池中的连接状态 ” 。

  • 改进了数据库链接的监控

    新视图和提供的PL / SQL函数使您能够确定传出数据库链接的主机名，查看有关传入数据库链接的信息的详细信息，并确定传入数据库链接的高系统更改编号（SCN）活动的来源。

    请参阅“ 确定传出数据库链接的主机 ” ， “ 确定有关传入数据库链接的信息 ”和“ 确定传入数据库链接的高SCN活动源 ” 。

  • 对象隔离

    对象隔离隔离了引发错误的对象，并监视对象是否对系统产生影响。

    请参阅“ 监视隔离的对象 ” 。

  • 实例中止延迟

    INSTANCE_ABORT_DELAY_TIME初始化参数指定错误导致实例中止的延迟时间。

    请参阅“ 延迟实例中止 ” 。

  • Prespawned进程

    Oracle数据库可以预先确定进程以获得更好的客户端连接性能。

    请参阅“ 管理Prespawned进程 ” 。

  • 分区外部表

    对于大量数据，外部表的分区可提供快速查询性能和增强的数据维护。

    请参见“ 分区外部表 ” 。

Oracle Database 12 c第1版（12.1.0.2）中的更改

以下是Oracle Database Administrator for Oracle Database 12 c Release 1（12.1.0.2）中的更改。

  • 新功能

上级主题： “Oracle数据库管理员指南”的本发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • 内存列存储

    内存列存储（IM列存储）位于SGA的可选区域中，以压缩列格式存储整个表，表分区，单个列和物化视图。该数据库使用特殊技术非常快速地扫描柱状数据。IM列存储是对数据库缓冲区高速缓存的补充而不是替代。

    请参阅“ 使用Oracle Database In-Memory提高查询性能 ” 。

  • 内存中列存储的数据泵支持

    Data Pump可以保留，覆盖或删除正在导入的数据库对象的In-Memory子句。

    请参阅Oracle Database In-Memory指南 。

  • 强制完全数据库缓存模式

    要提高性能，可以强制实例将数据库存储在缓冲区缓存中。

    请参阅“ 使用强制完全数据库缓存模式 ” 。

  • 大表缓存

    自动大表缓存功能使并行查询能够使用缓冲区缓存。

    请参阅“ 内存架构概述 ” 。

  • 属性集群表

    属性集群为堆组织表指定了一个指令，用于将数据存储在磁盘附近，从而提供性能和数据存储优势。该指令仅适用于直接路径操作，例如批量插入或移动操作。

    请参阅“ 考虑使用属性聚簇表 ” 。

  • 区域地图

    区域是磁盘上的一组连续数据块。区域图跟踪所有单个区域的指定列的最小值和最大值。区域映射的主要优点是表扫描的I / O减少。

    请参阅“ 考虑使用区域地图 ” 。

  • 高级索引压缩

    高级索引压缩可实现出色的压缩率，同时仍可提供对索引的高效访问。高级索引压缩在块级别工作，为每个块提供最佳压缩，这意味着用户不需要了解数据特征。高级索引压缩会自动为每个块选择正确的压缩。

    请参阅“ 使用高级索引压缩创建索引 ” 。

  • Oracle Clusterware对Diagnosability Framework的支持

    Oracle Clusterware使用Diagnosability Framework和ADR记录诊断跟踪数据和Clusterware警报日志。

    请参阅“ Oracle Clusterware环境中的ADR ” 。

  • READ对象特权和READ ANY TABLE系统特权

    对象的READ特权使用户能够从对象中进行选择，而无需为用户提供任何其他特权。

    有关详细信息，请参见“ 外部表的系统和对象权限 ”和“ Oracle数据库安全性指南” 。

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle数据库管理员指南（适用于Oracle Database 12 c第1版）（12.1.0.1）中的更改。

  • 新功能
  • 不推荐的功能

上级主题： “Oracle数据库管理员指南”的本发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • 完全可运输的出口/进口

    完全可传输导出/导入使您可以将数据库从一个数据库实例移动到另一个数据库实例。传输数据库比移动数据库的其他方法（例如完整数据库导出/导入）快得多。此外，您可以使用完全可传输的导出/导入将非CDB（或Oracle Database 11g第2版（11.2.0.3）数据库）移动到作为CDB一部分的PDB中。

    请参阅传输数据 。

  • 分离职责的新行政特权

    Oracle数据库现在为与Oracle Recovery Manager（Oracle RMAN），Oracle Data Guard和透明数据加密相关的任务提供管理权限。每个新的管理权限都授予在每个管理区域中完成任务所需的最低权限。新的管理权限使您可以避免为许多常见任务授予SYSDBA管理权限。

    请参阅“ 管理权限 ”

  • 数据库智能闪存缓存支持多个闪存设备

    数据库实例可以访问和组合数据库智能闪存缓存的多个闪存设备，而无需卷管理器。

    请参阅“ 数据库智能闪存缓存初始化参数 ” 。

  • 临时撤消

    临时对象的撤消存储在临时表空间中，而不是存储在撤消表空间中。使用临时撤消可以减少撤消表空间中存储的撤消量以及重做日志的大小。它还使用Oracle Active Data Guard选项在物理备用数据库中的临时表上启用数据操作语言（DML）操作。

    请参阅“ 管理临时撤消 ” 。另请参阅Oracle Data Guard概念和管理，以获取有关Oracle Data Guard环境中临时撤消的好处的信息。

  • 在线移动数据文件

    您可以在数据文件联机和访问时移动数据文件。此功能简化了维护操作，例如将数据移动到其他存储设备。

    请参阅“ 重命名和重新定位在线数据文件 ” 。

  • 同一组列上的多个索引

    您可以在同一组列上创建多个索引，以执行应用程序迁移，而无需删除现有索引并使用不同的属性重新创建它。

    请参阅“ 了解何时在同一组列上创建多个索引 ” 。

  • 在线移动分区或子分区

    DML操作可以在不使用联机表重新定义的情况下继续在正在移动的分区或子分区上不间断地运行。

    请参阅“ 将表移动到新段或表空间 ” 。

  • 在一个步骤中在线重新定义表

    您可以使用REDEF_TABLE过程中的DBMS_REDEFINITION包到过程的单一调用执行表的存储属性的在线重新定义。

    请参阅“ 使用REDEF_TABLE过程执行在线重新定义 ” 。

  • 在线重新定义具有多个分区的表

    要在重新定义表中的多个分区时最大限度地减少停机时间，可以在单个会话中在线重新定义这些分区。

    请参阅“ 一个或多个分区的在线重新定义 ” 。

  • 使用虚拟专用数据库（VPD）策略在线重新定义表

    为了最大限度地减少停机时间，可以在线重新定义具有VPD策略的表。

    请参阅“ 在线重新定义期间处理虚拟专用数据库（VPD）策略 ” 。

  • FINISH_REDEF_TABLE过程中的新时间限制参数

    DBMS_REDEFINITION包中的FINISH_REDEF_TABLE过程中的dml_lock_timeout参数可以指定过程等待挂起的DML提交的时间。

    请参阅“ 在DBMS_REDEFINITION中使用多个过程执行在线重新定义 ”中的步骤8 。

  • 隐形列

    您可以使单个表列不可见。表的任何通用访问都不会显示表中的不可见列。

    请参阅“ 了解不可见的列 ” 。

  • 优化的ALTER TABLE ...ADD COLUMN与空列缺省值

    可空列是不使用NOT NULL约束创建的列。对于某些类型的表，在添加具有默认值的可空列时，数据库可以优化操作的资源使用和存储要求。它通过将新列的默认值存储为表元数据来实现，从而无需将值存储在所有现有记录中。

    请参阅“ 添加表列 ” 。

  • 使用CloneDB进行数据库的写时复制克隆

    使用CloneDB克隆数据库时，Oracle数据库可以基于写时复制技术在CloneDB数据库中创建文件，这样只有在CloneDB数据库中修改的块才需要在磁盘上进行额外的存储。

    请参见“ 使用CloneDB克隆数据库 ” 。

  • DDL日志

    启用DDL语句的日志记录时，DDL语句将记录在单独的DDL日志中而不是警报日志中。

    请参阅“ DDL日志 ” 。

  • 调试日志

    可用于调试问题的某些信息记录在单独的调试日志中，而不是警报日志中。

    请参见“ 调试日志 ” 。

  • 服务器控件（SRVCTL）实用程序的全字选项

    为了提高可用性，每个SRVCTL实用程序选项都是一个完整的单词而不是单个字母。

    请参见“ Oracle Restart的SRVCTL命令参考 ” 。

  • 交易保护和应用程序连续性

    Transaction Guard确保最多一次执行事务，以保护应用程序免受重复的事务提交和相关的逻辑错误。Transaction Guard启用应用程序连续性，即在可恢复的通信错误之后重放事务并继续处理的能力。

    请参阅“ 事务处理保护和应用程序连续性 ” 。

  • 增强的语句排队

    关键语句可以绕过并行语句队列。您可以为高优先级使用者组将资源计划指令PARALLEL_STMT_CRITICAL为BYPASS_QUEUE ，以便来自使用者组的并行语句绕过并行语句队列。

    请参阅“ 创建资源计划指令 ” 。

  • 新的工作类型

    添加了几个新脚本作业，允许使用SQL * Plus，RMAN解释器或计算机平台的命令shell运行自定义用户脚本。

    请参阅“ 脚本作业 ” 。

不推荐的功能

此版本中不推荐使用以下功能，并且可能在将来的版本中不支持这些功能：

  • ORAPWD的IGNORECASE参数

    为了支持强身份验证，Oracle建议您将IGNORECASE设置为n或完全省略IGNORECASE 。此可选ORAPWD参数的默认值为n 。

    有关详细信息，请参阅“ 使用ORAPWD创建数据库密码文件 ” 。

  • 带有服务器控制（SRVCTL）实用程序命令的单字符选项

    所有SRVCTL命令都已增强，可接受全字选项而不是单字母选项。此版本中添加的所有新SRVCTL命令选项仅支持全字选项，并且没有单字母等效项。在将来的版本中可能不支持将单字符选项与SRVCTL命令一起使用。

    有关详细信息，请参见“ Oracle Restart的SRVCTL命令参考 ” 。

  • FILE_MAPPING初始化参数

    不推荐使用FILE_MAPPING初始化参数。它仍然支持向后兼容性。

    有关FILE_MAPPING初始化参数的信息，请参见Oracle数据库参考 。

  • * _SCHEDULER_CREDENTIALS

    此视图继续可用于向后兼容。

    有关详细信息，请参阅“ 指定计划程序作业凭据 ” 。
