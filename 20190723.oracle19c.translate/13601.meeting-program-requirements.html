3.1声明部分

您必须在Declare Section中声明要在SQL语句中使用的所有程序变量（即所有主机变量）。如果你使用在SQL语句中未声明的主机变量，预编译器发出错误消息。有关错误消息的完整列表，请参阅Oracle数据库错误消息 。

声明部分以声明开头

EXEC SQL BEGIN DECLARE SECTION;

并以声明结束

EXEC SQL END DECLARE SECTION;

在COBOL中，语句终止符是END-EXEC 。在FORTRAN，这是一个回车。

在这两个语句之间，只允许以下项：

  • 主变量和指标变量声明

  • EXEC SQL DECLARE语句

  • EXEC SQL INCLUDE语句

  • EXEC SQL VAR语句

  • EXEC ORACLE声明

  • 主持语言评论

每个预编译单元中允许多个声明部分。此外，主程序可以包含几个独立预编译的单元。

3.1.1一个例子

在以下示例中，您将声明四个主机变量，以便稍后在程序中使用。 

EXEC SQL BEGIN DECLARE SECTION; emp_number INTEGER; emp_name CHARACTER（10）;工资真实;委托REAL; EXEC SQL END DECLARE SECTION;

有关声明主机变量的详细信息，请参阅“ 声明和引用主机变量 ” 。

3.2包括声明

INCLUDE语句允许您将文件复制到主机程序中。它类似于COBOL COPY命令。一个例子如下：

- 复制SQLCA文件EXEC SQL INCLUDE SQLCA;

预编译程序时，每个EXEC SQL INCLUDE语句都将替换为语句中指定的文件的副本。

您可以包含任何文件。如果文件包含嵌入式SQL，则必须包含它，因为只预编译了包含的文件。如果未指定文件扩展名，则预编译器将采用源文件的默认扩展名，这是与语言相关的（请参阅本指南的主机语言补充）。

您可以通过指定预编译器选项为包含的文件设置目录路径

INCLUDE = <路径>

其中path默认为当前目录。（在这种情况下， 目录是文件位置的索引。）

预编译器首先在当前目录中搜索，然后在INCLUDE指定的目录中搜索，最后在标准INCLUDE文件的目录中搜索。因此，您无需指定目录标准文件的路径，例如SQLCA和ORACA。您仍必须使用INCLUDE为非标准文件指定目录路径，除非它们存储在当前目录中。

如果您的操作系统区分大小写（例如UNIX），请确保指定存储文件的相同大小/小写文件名。指定目录路径的语法是特定于系统的。检查特定于系统的Oracle手册。

3.4 Oracle数据类型

Oracle识别两种数据类型： 内部和外部 。内部数据类型指定Oracle如何在数据库列中存储数据。Oracle还使用内部数据类型来表示数据库伪列。外部数据类型指定数据在主机变量中的存储方式。

在预编译时，Declare部分中的每个主机变量都与外部数据类型代码相关联。在运行时，SQL语句中使用的每个主机变量的数据类型代码都将传递给Oracle。Oracle使用代码在内部和外部数据类型之间进行转换。

注意：

您可以使用动态SQL方法4或数据类型等效来覆盖默认数据类型转换。有关动态SQL方法4的信息，请参阅“ 使用方法4 ” 。有关数据类型同等化的信息，请参阅“ 数据类型同等化 ”。

3.4.2 CHAR

您使用CHAR数据类型存储固定长度的字符数据。内部如何表示数据取决于数据库字符集。CHAR数据类型采用可选参数，允许您指定最大宽度为255个字节。语法如下：

 CHAR [（maximum_width）]

您不能使用常量或变量来指定最大宽度;你必须使用整数文字。如果未指定最大宽度，则默认为1。请记住，您以字节为单位指定CHAR （ n ）列的最大宽度，而不是字符。因此，如果CHAR （ n ）列存储多字节（2字节）字符，则其最大宽度小于n / 2个字符。

3。3。3日期

你用的是 DATE数据类型，用于存储7字节固定长度字段中的日期和时间。日期部分默认为当月的第一天;时间部分默认为午夜。

在内部， DATE以二进制格式存储。将DATE列值转换为程序中的字符串时，Oracle会使用会话的默认格式掩码。如果您需要其他日期/时间信息，例如日期 Julian days，使用带格式掩码的TO_CHAR函数。始终使用（外部）将DATE列值转换为字符串和从字符串转换 ）字符数据类型，如VARCHAR2或STRING 。

3.4.4长

您使用LONG数据类型来存储可变长度字符串。LONG列可以存储文本，字符数组甚至短文档。LONG数据类型与VARCHAR2数据类型类似，但LONG列的最大宽度为2147483647字节或2千兆字节。

您可以在UPDATE ， INSERT和（大多数） SELECT语句中使用LONG列，但是不在表达式，函数调用或SQL子句中，如WHERE ， GROUP BY和CONNECT BY 。每个数据库表中只允许一个LONG列，并且无法索引该列。

3.4.5 LONG RAW

您使用LONG RAW数据类型来存储可变长度的二进制数据或字节字符串。该 LONG RAW列的最大宽度为2147483647字节或2千兆字节。

LONG RAW数据类似于LONG数据，不同之处在于Oracle不会假设LONG RAW数据的含义，并且在将LONG RAW数据从一个系统传输到另一个系统时不进行字符集转换。适用于LONG数据的限制也适用于LONG RAW数据。

3.4.6 MLSLABEL

您使用MLSLABEL数据类型来存储可变长度的二进制操作系统标签。Oracle使用标签来控制对数据的访问。

您可以使用MLSLABEL数据类型来定义数据库列。您可以将任何有效的操作系统标签插入MLSLABEL类型的列中。如果标签是文本格式，Oracle会自动将其转换为二进制值。文本字符串最长可达255个字节。但是， MLSLABEL值的内部长度介于2到5个字节之间。

您还可以从MLSLABEL列中选择值到字符变量中。Oracle会自动将内部二进制值转换为VARCHAR2值。

3.4.7号码

你用的是 NUMBER数据类型，用于存储几乎任何大小的固定或浮点数。您可以指定精度 ，即总位数，和 scale ，用于确定舍入发生的位置。

NUMBER值的最大精度为38;幅度范围是1.0E-129到9.99E125。比例范围可以从-84到127。例如，比例为-3表示数字四舍五入到最接近的千位（3456变为3000）。等级2表示该值四舍五入到最接近的百分位数（3.456变为3.46）。

指定精度和比例时，Oracle会在存储数据之前执行额外的完整性检查。如果值超过精度，Oracle会发出错误消息;如果某个值超出了该比例，Oracle会对该值进行舍入。

3.4.8 RAW

您可以使用RAW数据类型来存储二进制数据或字节字符串（例如，一系列图形字符）。Oracle不会解释RAW数据。

RAW数据类型采用必需参数，允许您指定最大宽度为255个字节。语法如下：

RAW（maximum_width）

您不能使用常量或变量来指定最大宽度;你必须使用整数文字。

RAW数据与CHAR数据类似，不同之处在于，当您将RAW数据从一个系统传输到另一个系统时，Oracle不会假定RAW数据的含义，也不会进行字符集转换（例如，从7位ASCII到EBCDIC代码页500）。

3.4.9 ROWID

在内部，Oracle数据库中的每个表都有一个名为ROWID的伪列，它存储称为rowid的二进制值。ROWID唯一标识行并提供访问特定行的最快方法。

3.4.10 VARCHAR2

您使用VARCHAR2数据类型来存储可变长度字符串。字符串在内部如何表示取决于数据库字符集，例如，可能是7位ASCII或EBCDIC代码。

VARCHAR2数据库列的最大宽度为2000个字节。要定义VARCHAR2列，请使用语法

VARCHAR2（maximum_width）

其中maximum_width是1的范围内的整数字面值。2000。

您可以指定VARCHAR2 （ n ）列的最大宽度（以字节为单位），而不是字符。因此，如果VARCHAR2 （ n ）列存储多字节（2字节）字符，则其最大宽度小于n / 2个字符。

CURRVAL以指定的顺序返回当前数字。在引用CURRVAL之前，必须使用NEXTVAL生成序列号。

LEVEL返回树结构中节点的级别编号。根是1级，根的子级是2级，孙子级是3级，依此类推。

LEVEL用于SELECT CONNECT BY语句，以将表的一些或所有行合并到树结构中。在ORDER BY或GROUP BY子句中， LEVEL隔离树中每个级别的数据。

您可以使用PRIOR运算符指定查询沿树的方向（从根向下或从分支向上）。在START WITH子句中，指定标识树根的条件。

NEXTVAL以指定的顺序返回下一个数字。创建序列后，您可以使用它为事务处理生成唯一的序列号。在以下示例中，使用名为partno的序列来分配部件号：

EXEC SQL INSERT INTO PARTS VALUES（partno。NEXTVAL，：说明，：数量，价格）;

如果事务生成序列号，则在提交或回滚事务时序列会递增。对NEXTVAL的引用将当前序列号存储在CURRVAL 。

ROWID以十六进制返回行地址。

ROWNUM返回一个数字，表示从表中选择行的顺序。选择的第一行的ROWNUM为1，第二行的ROWNUM为2，依此类推。如果SELECT语句包含ORDER BY子句，则在排序完成之前将ROWNUM分配给选定的行。

您可以使用ROWNUM来限制SELECT语句返回的行数。此外，您可以在UPDATE语句中使用ROWNUM为表中的每一行分配唯一值。在WHERE子句中使用ROWNUM不会停止处理SELECT语句;它只是限制了检索的行数。在WHERE子句中唯一有意义的使用ROWNUM是

...WHERE ROWNUM <常数;

因为只有在检索到行时， ROWNUM的值才会增加。永远不会满足以下搜索条件，因为未检索到前四行：

...在哪里ROWNUM = 5;

SYSDATE返回当前日期和时间。

UID返回分配给Oracle用户的唯一ID号。

USER返回当前Oracle用户的用户名。

3.4.12 ROWLABEL列

SQL还识别Oracle为每个数据库表创建的特殊列ROWLABEL 。与其他列一样， ROWLABEL可以在SQL语句中引用。ROWLABEL返回行的操作系统标签。

ROWLABEL一个常见用途是过滤查询结果。例如，以下语句仅计算安全级别高于“未分类”的行：

EXEC SQL SELECT COUNT（*）INTO：head_count FROM EMP WHERE ROWLABEL>'UNCLASSIFIED';

3.4.15 CHARF

当MODE = ANSI时，Oracle会将CHAR数据类型分配给所有字符主机变量。您使用CHAR数据类型来存储固定长度的字符串。在大多数平台上，CHAR值的最大长度为65535（64K）字节。有关DBMS和MODE选项之间关系的更多信息，请参见表6-4 。

在输入上 。Oracle读取为输入主机变量指定的字节数， 不剥离尾随空白，然后存储在目标数据库列中的输入值。

如果输入值超过数据库列的已定义宽度，则Oracle会生成错误。如果输入值是全空的，Oracle会将其视为字符值。

关于输出 。Oracle返回为输出主机变量指定的字节数，如有必要，返回空白填充，然后将输出值分配给目标主机变量。如果返回null，则Oracle使用空白填充主机变量。

如果输出值长于主机变量的声明长度，Oracle会在将值分配给主变量之前截断该值。如果指示符变量可用，Oracle会将其设置为输出值的原始长度。

3.4.16 CHARZ

您使用CHARZ数据类型来存储固定长度，以null结尾的字符串。在大多数平台上， CHARZ值的最大长度为65,535字节。您不应该在Pro * COBOL或Pro * FORTRAN中使用此外部类型。

在输入时， CHARZ和STRING数据类型的工作方式相同。您必须空终止输入值。null终止符仅用于分隔字符串;它不是数据的一部分。

在输出时， CHARZ和CHAR数据类型的工作方式相同。Oracle在输出值后附加一个空终止符，如果需要，它也是空白填充的。

3.4.20 FLOAT

您可以使用FLOAT数据类型来存储具有小数部分或超过INTEGER数据类型容量的数字。该数字使用计算机的浮点格式表示，通常需要4或8个字节的存储空间。您必须为输入和输出主变量指定长度。

Oracle可以表示比浮点实现更精确的数字，因为Oracle数字的内部格式是十进制的。

注意：

在SQL语句中，比较FLOAT值时，请使用SQL函数ROUND因为FLOAT存储二进制（非十进制）数字;因此，分数不会完全转换。

3.4.21 INTEGER

您使用INTEGER数据类型来存储没有小数部分的数字。整数是带符号的2字节或4字节二进制数。单词中字节的顺序取决于系统。您必须为输入和输出主变量指定长度。在输出时，如果列值是浮点数，则Oracle会截断小数部分。

3.4.22 LONG

您使用LONG数据类型来存储固定长度的字符串。LONG数据类型与VARCHAR2数据类型类似，只是LONG值的最大长度为2147483647字节（2千兆字节）。

3.4.23 LONG RAW

您可以使用LONG RAW数据类型来存储固定长度的二进制数据或字节字符串。LONG RAW值的最大长度为2147483647字节（2千兆字节）。

LONG RAW数据类似于LONG数据，不同之处在于Oracle不会假设LONG RAW数据的含义，并且在将LONG RAW数据从一个系统传输到另一个系统时不进行字符集转换。

3.4.24 LONG VARCHAR

您使用LONG VARCHAR数据类型来存储可变长度字符串。LONG VARCHAR变量具有4字节长度字段，后跟字符串字段。字符串字段的最大长度为2147483643字节。在EXEC SQL VAR声明， 不包括4个字节的长度字段。

3.4.25 LONG VARRAW

您使用LONG VARRAW数据类型来存储二进制数据或字节字符串。LONG VARRAW变量有一个4字节长的字段，后跟一个数据字段。数据字段的最大长度为2147483643个字节。在EXEC SQL VAR声明， 不包括4个字节的长度字段。

3.4.26 MLSLABEL

您使用MLSLABEL数据类型来存储可变长度的二进制操作系统标签。Oracle使用标签来控制对数据的访问。您可以使用MLSLABEL数据类型来定义列。您可以将任何有效的操作系统标签插入MLSLABEL类型的列中。

在输入上。 Oracle将输入值转换为二进制标签，该标签必须是有效的操作系统标签。如果标签无效，Oracle会发出错误消息。如果标签有效，Oracle会将其存储在目标数据库列中。

关于输出。 Oracle将二进制标签转换为字符串，可以是CHAR ， CHARZ ， STRING ， VARCHAR或VARCHAR2 。

3.4.27号码

你用的是 NUMBER数据类型，用于存储固定或浮点Oracle数字。您可以指定精度和比例。NUMBER值的最大精度为38;幅度范围是1.0E-129到9.99E125。比例范围可以从-84到127。

NUMBER值以可变长度格式存储，从指数字节开始，后跟最多20个尾数字节。指数字节的高位是一个符号位，设置为正数。低位7位代表指数，它是一个基数为100的数字，偏移量为65。

每个尾数字节都是1的基数100位数。100。对于正数，将1加到数字上。对于负数，从101中减去该数字，除非有20个尾数字节，否则将包含102的字节附加到数据字节。每个尾数字节可以表示两个十进制数字。尾数被归一化并且不存储前导零。最多可以使用20个数据字节作为尾数，但只保证19个数据字节是准确的。19个字节，每个字节代表一个基数为100的数字，最大精度为38位。

在输出时，主机变量包含Oracle内部表示的数字。为了容纳尽可能大的数字，输出主机变量必须是21个字节长。仅返回用于表示该数字的字节。Oracle没有空白填充或空终止输出值。如果需要知道返回值的长度，请改用VARNUM数据类型。通常，没有理由使用此数据类型。

3.4.28 RAW

您使用RAW数据类型来存储固定长度的二进制数据或字节字符串。在大多数平台上， RAW值的最大长度为65535字节。RAW数据与CHAR数据类似，不同之处在于Oracle不会假定RAW数据的含义，并且在将RAW数据从一个系统传输到另一个系统时不进行字符集转换。

3.4.29 ROWID

您可以使用ROWID数据类型在（通常为13个字节）固定长度字段中存储二进制rowid。字段大小是特定于端口的。因此，请检查特定于系统的Oracle手册。您可以使用VARCHAR2主机变量以可读格式存储rowid。当您在VARCHAR2主机变量中选择或获取rowid时，Oracle会将二进制值转换为18字节的字符串，并以格式返回

BBBBBBBB.RRRR.FFFF

其中BBBBBBBB是数据库文件中的块，RRRR是块中的行（第一行是0），FFFF是数据库文件。这些数字是十六进制的。例如，rowid

0000000E.000A.0007

指向第7个数据库文件中第15个块的第11行。

通常，您将rowid提取到VARCHAR2主机变量中，然后将主机变量与UPDATE或DELETE语句的WHERE子句中的ROWID伪列进行比较。这样，您可以识别游标提取的最新行。

注意：

如果您需要完全可移植性或您的应用程序通过透明网关与非Oracle数据库通信，请在声明VARCHAR2主机变量时指定最大长度为256（而不是18）字节。如果您的应用程序通过Oracle Open Gateway与非Oracle数据源通信，请指定最大长度为256个字节。虽然您可以对其内容不做任何假设，但主机变量在SQL语句中将正常运行。

3.4.30 STRING

STRING数据类型与VARCHAR2数据类型类似，只是STRING值始终以null结尾。

在输入上。 Oracle使用指定的长度来限制对空终止符的扫描。如果未找到空终止符，Oracle将生成错误。如果未指定长度，Oracle将采用最大长度，在大多数平台上为65535。

STRING值的最小长度为2个字节。如果第一个字符是空终止符且指定的长度为2，则Oracle将插入null，除非该列定义为NOT NULL 。全空或 nullterminated值保存完整。

关于输出 。Oracle将空字节附加到返回的最后一个字符。如果字符串长度超过指定的长度，Oracle会截断输出值并附加一个空字节。

3.4.31未签名

您使用UNSIGNED数据类型来存储无符号整数。无符号整数是2或4字节的二进制数。单词中字节的顺序取决于系统。您必须为输入和输出主变量指定长度。在输出时，如果列值是浮点数，则Oracle会截断小数部分。您不应该在Pro * COBOL或Pro * FORTRAN中使用此外部类型。

3.4.32 VARCHAR

您使用VARCHAR数据类型来存储可变长度字符串。VARCHAR变量具有2字节长度字段，后跟<= 65533字节字符串字段。但是，对于VARCHAR数组元素，字符串字段的最大长度为65530字节。指定VARCHAR变量的长度时，请确保为length字段包含2个字节。对于较长的字符串，请使用LONG VARCHAR数据类型。在EXEC SQL VAR声明， 不包括2个字节的长度字段。

3.4.33 VARCHAR2

当MODE=ORACLE ，Oracle会将VARCHAR2数据类型分配给所有字符主机变量。您使用VARCHAR2数据类型来存储可变长度字符串。在大多数平台上， VARCHAR2值的最大长度为65535字节。

您可以指定VARCHAR2 （ n ）值的最大长度（以字节为单位），而不是字符。因此，如果VARCHAR2 （ n ）变量存储多字节字符，则其最大长度小于n个字符。

在输入上。 Oracle读取为输入主机变量指定的字节数，删除任何尾随空白，然后将输入值存储在目标数据库列中。小心。未初始化的主机变量可以包含空值。因此，始终将字符输入主变量空白填充到其声明的长度。（COBOL PIC X（ n ）和FORTRAN CHARACTER * n变量自动执行此操作。）

如果输入值超过数据库列的已定义宽度，则Oracle会生成错误。如果输入值是全空的，Oracle会将其视为null。

如果字符值表示有效数字，Oracle可以将字符值转换为NUMBER列值。否则，Oracle会生成错误。

关于输出。 Oracle返回为输出主机变量指定的字节数，如有必要，返回空白填充，然后将输出值分配给目标主机变量。如果返回null，则Oracle使用空白填充主机变量。

如果输出值长于主机变量的声明长度，Oracle会在将值分配给主变量之前截断该值。如果指示符变量可用，Oracle会将其设置为输出值的原始长度。

Oracle可以将NUMBER列值转换为字符值。字符主变量的长度决定了精度。如果主变量太短，则使用科学记数法。例如，如果将列值abcdefg89选择为长度为6的主机变量，则Oracle会将值“1.2E08”返回到主机变量。

3.4.34 VARNUM

VARNUM数据类型与NUMBER数据类型类似，只是VARNUM变量的第一个字节存储值的长度。在输入时，必须将主变量的第一个字节设置为值的长度。在输出时，主机变量包含长度，后跟Oracle内部表示的数字。为了容纳尽可能大的数字，主机变量必须是22个字节长。在VARNUM主变量中选择列值后，可以检查第一个字节以获取值的长度。

3.4.35 VARRAW

您使用VARRAW数据类型来存储可变长度的二进制数据或字节字符串。VARRAW数据类型与RAW数据类型类似，不同之处在于VARRAW变量具有2字节长度字段，后跟<= 65533字节数据字段。对于较长的字符串，请使用LONG VARRAW数据类型。在EXEC SQL VAR声明， 不包括2个字节的长度字段。要获取VARRAW变量的长度，只需引用其长度字段即可。

3.5数据类型转换

在预编译时，将在Declare Section中为每个主机变量分配外部数据类型。例如，预编译器将INTEGER外部数据类型分配给整数主机变量。在运行时，SQL语句中使用的每个主机变量的数据类型代码都将传递给Oracle。Oracle使用代码在内部和外部数据类型之间进行转换。

在将选定列（或伪列）值分配给输出主机变量之前，如有必要，Oracle会将列的内部数据类型转换为主机变量的数据类型。同样，在将输入主机变量的值分配或比较到数据库列之前，如有必要，Oracle会将主机变量的外部数据类型转换为列的内部数据类型。

但是，主机变量的数据类型必须与数据库列的数据类型兼容。您有责任确保价值可兑换。例如，如果您尝试将字符串值“YESTERDAY”转换为DATE列值，则会出现错误。

内部和外部数据类型之间的转换遵循通常的数据转换规则。例如，您可以将CHAR值“1234”转换为2字节整数。但是，您不能将CHAR值“65543”（数字太大）或“10F”（数字不是十进制）转换为2字节整数。同样，您不能将包含字母字符的字符串值转换为NUMBER值。

数字转换遵循Oracle初始化文件中全球化支持（全球化支持）参数指定的约定。例如，您的系统可能配置为识别逗号（，）而不是句点（。）作为小数字符。

3.5.1日期值

在字符主机变量中选择DATE列值时，Oracle必须将内部二进制值转换为外部字符值。因此，Oracle隐式调用SQL函数TO_CHAR ，它返回一个字符串默认日期格式。默认值由Oracle初始化参数Globalization Support_DATE_FORMAT 。要获取其他信息，如时间或Julian日期，则必须显式调用TO_CHAR与格式掩码。

将字符主机值插入DATE列时，还需要进行转换。Oracle隐式调用SQL函数TO_DATE ，它需要默认的日期格式。要以其他格式插入日期，必须使用格式掩码显式调用TO_DATE 。

3.5.2 RAW和LONG RAW值

在字符主机变量中选择RAW或LONG RAW列值时，Oracle必须将内部二进制值转换为外部字符值。在这种情况下，Oracle将RAW或LONG RAW数据的每个二进制字节作为一对字符返回。每个字符代表半字节的十六进制等效值（半个字节）。例如，Oracle将二进制字节11111111作为字符对“FF”返回。SQL函数RAWTOHEX执行相同的转换。

将字符主机值插入RAW或LONG RAW列时，还需要进行转换。host变量中的每对字符必须表示二进制字节的十六进制等效值。如果一个字符不代表a的十六进制值 nibble，Oracle发出以下错误消息：

ORA-01465：无效的十六进制数

3.8数据类型等价

数据类型等效允许您自定义Oracle解释输入数据的方式以及Oracle格式化输出数据的方式。在逐个变量的基础上，您可以将支持等效的主机语言数据类型添加到Oracle外部数据类型。

3.8.1为什么等价数据类型？

数据类型等效在几个方面很有用。例如，假设您要在COBOL程序中使用以null结尾的主机字符串。您可以声明PIC X主机变量，然后将其等效为外部数据类型STRING ，它始终以空值终止。

当您希望Oracle存储但不解释数据时，可以使用数据类型等效。例如，如果要在LONG RAW数据库列中存储整数主机数组，则可以将主机数组等效为外部数据类型LONG RAW 。

此外，您可以使用数据类型等效来覆盖默认数据类型转换。除非Oracle初始化文件中的全局化支持参数另行指定，否则如果在字符主机变量中选择DATE列值，Oracle将返回一个9字节的字符串，格式如下：

DD-MON-YY

但是，如果将字符主机变量等同于DATE外部数据类型，Oracle将以内部格式返回7字节值。

3.8.2主机变量等价

默认情况下，Oracle预编译器为每个主机变量分配特定的外部数据类型。（这些默认分配在本指南的补充中列出。）您可以通过在Declare部分中将主机变量等效为Oracle外部数据类型来覆盖默认分配。这称为主变量等效 。

您使用的语法是：

EXEC SQL VAR <host_variable> IS <ext_type_name> [（{<length> | <precision>，<scale>}）];

其中， host_variable是在Declare Section中先前声明的输入或输出主机变量（或主机数组）。VARCHAR和VARRAW外部数据类型具有2字节长度字段，后跟n字节数据字段，其中n位于范围1中。65533。因此，如果type_name是VARCHAR或VARRAW ，则host_variable的长度必须至少为3个字节。

LONG VARCHAR和LONG VARRAW外部数据类型具有4字节长度字段，后跟n字节数据字段，其中n位于范围1中。2147483643。因此，如果type_name为LONG VARCHAR或LONG VARRAW ，则host_variable的长度必须至少为5个字节。

ext_type_name是有效外部数据类型的名称，例如RAW或STRING 。

length是一个整数文字，指定有效的长度（以字节为单位）。length的值必须足够大以容纳外部数据类型。

当type_name为DECIMAL或DISPLAY ，您必须指定精度和比例而不是长度 。当type_name为VARNUM ， ROWID或DATE ，您无法指定长度，因为它是预定义的。对于其他外部数据类型， length是可选的。它默认为host_variable的长度。

指定长度时 ，如果type_name为VARCHAR ， VARRAW ， LONG VARCHAR或LONG VARRAW ，则使用数据字段的最大长度。预编译器考虑了长度字段。如果type_name为LONG VARCHAR或LONG VARRAW且数据字段超过65533字节，则在长度字段中输入“-1”。

precision和scale是整数文字，分别表示有效数字的数量和舍入将发生的点。例如， 比例为2 表示该值四舍五入到最接近的百分位数（3.456变为3.46）;比例为-3表示数字四舍五入到最接近的千位（3456变为3000）。

您可以指定1 .. 99的精度和-84的比例 。99。但是，数据库列的最大精度和比例分别为38和127。因此，如果精度超过38，则无法将host_variable的值插入数据库列。但是，如果列值的比例超过99，则无法选择或将值提取到host_variable中 。

仅当type_name为DECIMAL或DISPLAY时才指定精度和小数 位数 。

3.10多字节全球化支持字符集

该 Pro * COBOL预编译器扩展了对多字节全球化支持字符集的支持

  • 嵌入式SQL语句中预编译器识别多字节字符串。

  • ANSI标准COBOL PIC N数据类型声明子句，它指示预编译器将主机字符变量解释为双字节字符串。

Oracle通过预编译器运行时库SQLLIB支持多字节字符串。

3.10.1嵌入式SQL中的字符串

嵌入式SQL语句中的多字节全球化支持字符串由一个字符文字组成，该字符文字将字符串标识为多字节字符串，后跟用单引号括起来的字符串。

例如，一个嵌入式SQL语句就像

EXEC SQL SELECT empno INTO：emp_num FROM emp WHERE ename = N'Kuroda'END-EXEC。

包含多字节字符串，因为字符串“Kuroda”之前的N字符文字将其标识为多字节字符串。

3.10.2动态SQL

由于动态SQL语句未在预编译时处理，并且Oracle本身不处理多字节全球化支持字符串，因此无法在动态SQL语句中嵌入多字节全球化支持字符串。

3.10.3嵌入式DDL

存储多字节全球化支持数据的列不能用于嵌入式数据定义语言（DDL）语句。预编译时无法强制执行此限制，因此在嵌入式DDL语句中使用扩展列类型（如NCHAR）会导致执行错误，而不是预编译错误。

3.10.4多字节全球化支持主机变量

Pro * COBOL预编译器使用ANSI标准PIC N子句声明多字节字符数据的主机变量。使用PIC N子句声明的变量被识别为双字节字符的字符串变量。

3.10.6空白填充

将Pro * COBOL字符变量定义为多字节全局化支持变量时，将应用以下空白填充和空白清除规则，具体取决于变量的外部数据类型。请参见外部数据类型 。

CHARF。这是定义多字节字符串时的默认字符类型。输入数据被剥离任何尾随的双字节空格。但是，如果一个字符串只包含双字节空格，则缓冲区中会留下一个双字节空间作为标记。

VARCHAR。在输入时，主变量不会被删除尾随的双字节空格。假设length组件是以字符为单位的数据长度，而不是字节。

STRING / LONG VARCHAR。全球化支持数据不支持这些主变量，因为它们只能使用动态SQL或数据类型等效来指定，全局化支持数据都不支持这些变量。

光标控制。 诸如OPEN，FETCH和CLOSE之类的游标控制语句是例外 - 它们从不使用AT子句。如果要将游标与显式标识的数据库关联，请使用中的AT子句 DECLARE CURSOR语句，如下：

EXEC SQL AT：db_name DECLARE emp_cursor CURSOR FOR ...EXEC SQL OPEN emp_cursor ...EXEC SQL FETCH emp_cursor ...EXEC SQL CLOSE emp_cursor;

如果db_name是主机变量，则其声明必须在引用声明游标的所有SQL语句的范围内。例如，如果在一个子程序中打开游标，然后在另一个子程序中从中获取游标，则必须全局声明db_name或将其传递给每个子程序。

从光标打开，关闭或提取时，不使用AT子句。如果在游标声明中没有使用AT子句，则SQL语句在DECLARE CURSOR语句的AT子句中指定的数据库中执行，或者在缺省数据库中执行。

动态SQL 。动态SQL语句类似于游标控制语句，因为有些语句从不使用AT子句。对于动态SQL方法1，如果要在非默认连接上执行语句，则必须使用AT子句。一个例子如下：

EXEC SQL CONNECT：username IDENTIFIED BY：password USING：db-string;

3.11.6隐式登录

通过Oracle分布式数据库选项支持隐式登录，该选项不需要显式登录。例如，分布式查询允许单个SELECT语句访问一个或多个非默认数据库上的数据。

分布式查询工具依赖于数据库链接，该链接为CONNECT语句而不是连接本身分配名称。在运行时，嵌入式SELECT语句由指定的Oracle Server执行，该服务器隐式连接到非默认数据库以获取所需的数据。

3.11.7单一隐式登录

在下一个示例中，您将连接到单个非默认数据库。首先，您的程序执行以下语句来定义a 数据库链接（数据库链接通常由DBA或用户以交互方式建立）：

EXEC SQL CREATE DATABASE LINK db_link CONNECT TO username IDENTIFIED BY password USING'd：newyork-nondef';

然后，程序可以使用数据库链接查询非默认EMP表，如下所示：

EXEC SQL SELECT ENAME，JOB INTO：emp_name，：job_title FROM emp @ db_link WHERE DEPTNO =：dept_number;

数据库链接与嵌入式SQL语句的AT子句中使用的数据库名称无关。它只是告诉Oracle非缺省数据库的位置，路径，以及要使用的Oracle用户名和密码。数据库链接存储在数据字典中，直到显式删除。

在我们的示例中，默认Oracle Server使用数据库链接db_link通过SQL * Net登录到非默认数据库。查询将提交给默认服务器，但会“转发”到非默认数据库以供执行。

为了更容易地引用数据库链接，您可以按如下方式创建同义词（同样，这通常以交互方式完成）：

EXEC SQL CREATE SYNONYM emp FOR emp @ db_link;

然后，您的程序可以查询非默认EMP表，如下所示：

EXEC SQL SELECT ENAME，JOB INTO：emp_name，：job_title FROM emp WHERE DEPTNO =：dept_number;

这为emp提供了位置透明度。

3.11.8多个隐式登录

在以下示例中，您将同时连接到两个非默认数据库。首先，执行以下语句序列来定义两个数据库链接并创建两个同义词：

EXEC SQL CREATE DATABASE LINK db_link1通过password1使用'd：newyork-nondef'连接到username1; EXEC SQL CREATE DATABASE LINK db_link2通过password2使用'd：chicago-nondef'连接到username2; EXEC SQL CREATE SYNONYM emp FOR emp @ db_link1; EXEC SQL CREATE SYNONYM dept for dept @ db_link2;

然后，您的程序可以查询非默认的EMP和DEPT表，如下所示：

EXEC SQL SELECT ENAME，JOB，SAL，LOC FROM emp，dept WHERE emp。DEPTNO = dept。DEPTNO AND DEPTNO =：dept_number;

甲骨文执行由执行在db_link1非默认EMP表和db_link2非默认DEPT表之间的连接查询。

3.12嵌入OCI（Oracle呼叫接口）呼叫

该 Oracle预编译器允许您在主机程序中嵌入OCI调用。只需采取以下步骤：

  1. 声明OCI登录数据区（ LDA）在宣布部分之外。
  2. 使用嵌入式SQL语句CONNECT连接到Oracle，而不是OCI调用OLOG 。
  3. 调用Oracle运行时库例程SQLLDA将连接信息存储在LDA中。

这样，Oracle预编译器和OCI“知道”他们正在一起工作。但是，没有共享Oracle游标。

您无需担心声明OCI主机数据区（HDA），因为Oracle运行时库管理连接并为您维护HDA。

3.12.1关于设置LDA

您通过发出OCI呼叫来设置LDA

SQLLDA（LDA）;

其中lda标识LDA数据结构。此调用的格式取决于语言。如果CONNECT语句失败，则lda中的lda_rc字段设置为1012以指示错误。

3.12.2远程和多连接

对SQLLDA的调用为最近执行的SQL语句使用的连接设置LDA。要设置其他连接所需的不同LDA，只需在每个CONNECT之后使用不同的lda调用SQLLDA。在以下示例中，您同时连接到两个非默认数据库：

EXEC SQL BEGIN DECLARE SECTION;用户名CHARACTER（10）;密码CHARACTER（10）; db_string1 CHARACTER（20）; db_string2 CHARACTER（20）; EXEC SQL END DECLARE SECTION; lda1 INTEGER（32）; lda2 INTEGER（32）; set username ='SCOTT'; set password ='TIGER'; set db_string1 ='D：NEWYORK-NONDEF1'; set db_string2 ='D：CHICAGO-NONDEF2'; - 为每个数据库连接提供一个唯一的名称EXEC SQL DECLARE db_name1 DATABASE; EXEC SQL DECLARE db_name2 DATABASE; - 连接到第一个非默认数据库EXEC SQL CONNECT：username IDENTIFIED BY：password AT db_name1 USING：db_string1; - 为OCI设置第一个LDA使用SQLLDA（lda1）; - 连接到第二个非默认数据库EXEC SQL CONNECT：username IDENTIFIED BY：password AT db_name2 USING：db_string2; - 为OCI设置第二个LDA使用SQLLDA（lda2）;

请记住，不要在Declare部分声明db_name1和db_name2 ，因为它们不是主机变量。您仅使用它们来命名两个非默认节点上的默认数据库，以便以后的SQL语句可以按名称引用数据库。

3.13.1 Oracle特定问题

您可以使用Oracle预编译器开发符合X / Open标准的应用程序。但是，您必须满足以下要求。

3.13.2关于连接Oracle

X / Open应用程序不建立和维护与数据库的连接。相反，Oracle提供的事务管理器和XA接口透明地处理数据库连接和断开连接。因此，通常X / Open兼容的应用程序不执行CONNECT语句。

3.13.3交易控制

X / Open应用程序不得执行影响全局事务状态的语句，如COMMIT ， ROLLBACK ， SAVEPOINT和SET TRANSACTION 。例如，应用程序不能执行COMMIT语句，因为事务管理器处理提交。此外，应用程序不得执行SQL数据定义语句（如CREATE ， ALTER和RENAME因为它们会发出隐式提交。

如果应用程序检测到阻止进一步SQL操作的错误，则可以执行内部ROLLBACK语句。但是，这可能会在更高版本的XA界面中发生变化。

3.13.4 OCI呼叫

如果希望X / Open应用程序发出OCI调用，则必须使用运行时库例程SQLLD2，它为通过XA接口建立的指定连接设置LDA。请注意，X / Open应用程序无法发出OCOM，OCON，OCOF，ORLON，OLON，OLOG和OLOGOF。

3.13.5链接

要获得XA功能，必须将XA库链接到X / Open应用程序对象模块。有关说明，请参阅特定于系统的Oracle手册。
