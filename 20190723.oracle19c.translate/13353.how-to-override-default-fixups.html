如何覆盖默认修正

您可以使用RUNFIX列条目禁用自动修复，除非禁用修订违反安全性或Oracle策略。

在Analyze处理模式阶段生成属于AutoUpgrade过程的默认修正。您可以修改生成的修正列表以禁用自动修正，以便您可以运行自己的修正。

步骤顺序如下：

  1. 在分析模式下运行AutoUpgrade实用程序
  2. 打开并编辑在分析模式下生成的checklist.cfg文件，以便禁止手动执行的修补程序自动运行。
  3. 在AutoUpgrade配置文件中，在要禁止自动sid .checklist的作业的本地参数列表下，找到参数sid .checklist ，其中sid是要抑制其的数据库的系统标识符（SID）自动修复。编辑参数以提供已编辑的checklist.cfg文件的路径。

使用以下示例来帮助您完成此过程。

注意：

如果AutoUpgrade实用程序发现错误级别数据库条件，并且没有可用的修订，或者您已手动禁用可用修订，则包含具有错误条件的数据库的AutoUpgrade作业将停止。

示例3-6在分析模式下启动AutoUpgrade实用程序

java -jar autoupgrade.jar -config config.cfg -mode analyze

示例3-7查找和编辑checklist.cfg

Analyze模式生成一个文件名为checklist.cfg的fixup文件。导航到该文件，其中DATABASE_LOGS_DIR是为数据库的AutoUpgrade log_dir参数设置的值， job-id是指AutoUpgrade实用程序生成的作业标识符， sid是要自动抑制的数据库的系统标识符链接地址：

DATABASE_LOGS_DIR / job-id / prechecks / sid _checklist.cfg

使用文本编辑器打开checklist.cfg文件。以下是具有SID DB11204的数据库的checklist.cfg文件的DB11204 ：

[dbname] [DB11204] ========================================== [容器] [DB11204] ========================================== [checkname] AMD_EXISTS [stage] PRECHECKS [fixup_available] YES [runfix] YES [严重性]警告---------------------------------- ------------------ [checkname] DEPEND_USR_TABLES [stage] POSTCHECKS [fixup_available] YES [runfix] YES [severity]警告------------ ---------------------------------------- [checkname] DICTIONARY_STATS [stage] PRECHECKS [fixup_available ]是[runfix]是[严重性]推荐---------------------------------------- ------------ [checkname] EM_PRESENT [stage] PRECHECKS [fixup_available] YES [runfix] YES [severity]警告------------------ ----------------------------------。。。

该文件具有分层结构。它以数据库名称和checklist.cfg条目所适用的容器名称开头。该文件包含一系列自动应用的修正检查。对于每个条目，有5个相关值参数：

参数 描述

[检查名]

数据库检查的名称

[阶段]

执行检查的自动升级阶段。它可以是预先检查，也可以是检查后。

[可用修复]

自动修复的可用性。此参数值为YES （自动修正可用）或NO （自动修正不可用）。

runfix

运行修复程序的运行状态。此参数采用以下两个值之一：

YES ：运行修正。

NO ：不要运行修复程序。

严重

自动修复程序解决的问题的严重性类别。

对于要手动执行的每个修正，请将[runfix]参数值从YES更改为NO 。
