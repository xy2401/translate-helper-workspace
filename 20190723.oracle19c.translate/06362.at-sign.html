2.1 @（在标志处）

目的

使用@命令执行存储在具有指定路径名的操作系统文件中的一系列RMAN命令。

注意：

该文件必须包含完整的RMAN命令。部分命令会生成语法错误。

先决条件

命令文件必须包含完整的RMAN命令。

如果在RUN命令中使用@命令，那么@命令必须在它自己的行上（参见例2-2 ）。

使用说明

RMAN处理文件，就像输入内容而不是@命令一样。如例2-3所示，您可以在命令文件中指定替换变量，然后在执行期间将值传递给命令文件。

语义

语法元素 描述

文件名

指定命令文件的名称，例如@/oracle/dbs/cmd/cmd1.rman 。如果未指定绝对路径名，则假定当前工作目录，例如@cmd1.rman 。

任何文件扩展名（或没有文件扩展名）都有效。不要在字符串周围使用引号或在@关键字和文件名之间留下空格。

例子

示例2-1从操作系统命令行运行命令文件

此示例创建RMAN命令文件，然后从操作系统命令行执行它。

％echo“BACKUP DATABASE;” > backup_db.rman％rman TARGET / @ backup_db.rman

示例2-2在RMAN中运行命令文件

此示例显示如何从RMAN提示符和RUN命令中执行命令文件。用户输入的文本以粗体显示。 

RMAN> @ backup_db.rman RMAN> RUN { 2> @ backup_db.rman 3>备份数据库; 4> **文件结束** 5> }

例2-3指定替换变量

假设您使用文本编辑器创建命令文件whole_db.rman其中包含以下内容：

#name：whole_db.rman BACKUP TAG＆1 COPIES＆2 DATABASE;出口;

以下示例从操作系统提示符启动RMAN并连接到目标数据库。然后该示例运行@命令，将变量传递给命令文件以创建带有标记Q106两个数据库备份：

％rman TARGET / RMAN> @ / tmp / whole_db.rman Q106 2
