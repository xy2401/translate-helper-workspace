对DBMS_JOB 的支持

不推荐使用DBMS_JOB包。Oracle Scheduler替换了DBMS_JOB包。尽管仍然支持DBMS_JOB以实现向后兼容，但Oracle强烈建议您从DBMS_JOB切换到Oracle Scheduler。

  • Oracle Scheduler替换DBMS_JOB
    从Oracle Database 11g第2版（11.2）开始，Oracle Scheduler取代了DBMS_JOB 。 Oracle Scheduler比DBMS_JOB更强大，更灵活， DBMS_JOB是用于调度作业的包。尽管仍然支持DBMS_JOB以实现向后兼容，但Oracle强烈建议您从DBMS_JOB切换到Oracle Scheduler。

    本节说明了如何使用DBMS_JOB包创建作业并使用Oracle Scheduler重写它们的一些示例，Oracle Scheduler使用DBMS_SCHEDULER包进行配置和控制。

A.1 Oracle Scheduler替换DBMS_JOB

从Oracle Database 11g第2版（11.2）开始，Oracle Scheduler取代了DBMS_JOB 。 Oracle Scheduler比DBMS_JOB更强大，更灵活， DBMS_JOB是用于调度作业的包。尽管仍然支持DBMS_JOB以实现向后兼容，但Oracle强烈建议您从DBMS_JOB切换到Oracle Scheduler。

  • 配置DBMS_JOB
    JOB_QUEUE_PROCESSES初始化参数指定可以为执行作业创建的最大进程数。
  • 使用DBMS_JOB和Oracle Scheduler
    DBMS_JOB和Oracle Scheduler（调度程序）使用相同的作业协调程序来启动作业从属。

A.1.1配置DBMS_JOB

JOB_QUEUE_PROCESSES初始化参数指定可以为执行作业创建的最大进程数。

从Oracle Database 12 c第2版开始， JOB_QUEUE_PROCESSES默认为4000。作业协调程序进程仅根据要运行的作业数和可用资源启动所需的作业队列进程数。您可以将JOB_QUEUE_PROCESSES设置为较小的数字以限制作业队列进程的数量。

将JOB_QUEUE_PROCESSES设置为0将禁用DBMS_JOB作业和DBMS_SCHEDULER作业。

A.1.2同时使用DBMS_JOB和Oracle Scheduler

DBMS_JOB和Oracle Scheduler（调度程序）使用相同的作业协调程序来启动作业从属。

您可以使用JOB_QUEUE_PROCESSES初始化参数来限制DBMS_JOB和调度程序的作业从属数。

如果JOB_QUEUE_PROCESSES为0，则禁用DBMS_JOB和Oracle Scheduler作业。

父主题： 支持DBMS_JOB

A.2.1创建工作

示例说明了使用DBMS_JOB包和DBMS_SCHEDULER包创建作业。

以下示例使用DBMS_JOB创建作业：

VARIABLE jobno NUMBER; BEGIN DBMS_JOB.SUBMIT（：jobno，'INSERT INTO employees VALUES（7935，''SALLY''，''DOGAN''，'sally.dogan @ examplecorp.com''，NULL，SYSDATE，''AD_PRES''， NULL，NULL，NULL，NULL）;'，SYSDATE，'SYSDATE + 1'）;承诺;结束; /

以下是使用DBMS_SCHEDULER的等效语句：

BEGIN DBMS_SCHEDULER.CREATE_JOB（job_name =>'job1'，job_type =>'PLSQL_BLOCK'，job_action =>'INSERT INTO员工VALUES（7935，''SALLY''，''DOGAN''，'sally.dogan@examplecorp。 com''，NULL，SYSDATE，''AD_PRES''，NULL，NULL，NULL，NULL）;'，start_date => SYSDATE，repeat_interval =>'FREQ = DAILY; INTERVAL = 1'）;结束; /

A.2.2改变工作

示例说明使用DBMS_JOB包和DBMS_SCHEDULER包更改作业。

以下示例使用DBMS_JOB更改作业：

BEGIN DBMS_JOB.WHAT（31，'INSERT INTO employees VALUES（7935，''TOM''，''DOGAN''，''tom.dogan @ examplecorp.com''，NULL，SYSDATE，''AD_PRES''，NULL ，NULL，NULL，NULL）;'）;承诺;结束; /

这会更改JOB1的操作以插入不同的值。

以下是使用DBMS_SCHEDULER的等效语句：

BEGIN DBMS_SCHEDULER.SET_ATTRIBUTE（name =>'JOB1'，attribute =>'job_action'，value =>'INSERT INTO employees VALUES（7935，''TOM''，''DOGAN''，''tom.dogan@examplecorp。 com''，NULL，SYSDATE，''AD_PRES''，NULL，NULL，NULL，NULL）;'）;结束; /

A.2.3从作业队列中删除作业

示例说明了使用DBMS_JOB包和DBMS_SCHEDULER包删除作业。

以下示例使用DBMS_JOB删除作业，其中14144是正在运行的作业的编号：

BEGIN DBMS_JOB.REMOVE（14144）;承诺;结束; /

使用DBMS_SCHEDULER ，您将发出以下语句：

BEGIN DBMS_SCHEDULER.DROP_JOB（'myjob1'）;结束; /
