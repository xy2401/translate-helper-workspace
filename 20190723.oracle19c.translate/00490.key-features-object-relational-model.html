1.3对象关系模型的主要特征

Oracle数据库将对象类型模型实现为关系模型的扩展，同时继续支持标准关系数据库功能，例如查询，快速提交，备份和恢复，可伸缩连接，行级锁定，读取一致性等。

SQL和各种编程接口和语言（包括PL / SQL，Java，Oracle调用接口，Pro * C / C ++和C＃）已通过扩展来增强，以支持Oracle对象。结果是一个对象关系模型，它提供了对象接口的直观性和经济性，同时保留了关系数据库的高并发性和吞吐量。

1.3.1.3关于对象方法

对象方法是可以在对象类型定义中声明的函数或过程，以实现您希望该类型的对象执行的行为。

可以在类型定义中声明的一般方法类型是：

  • 成员方法

    使用成员方法，您可以提供对对象数据的访问，还可以定义应用程序对数据执行的操作。要执行操作，应用程序会在适当的对象上调用适当的方法。

  • 静态方法

    静态方法比较对象实例并执行不使用任何特定对象的数据的操作，而是对对象类型是全局的。

  • 构造方法

    默认构造函数方法是为每个对象类型隐式定义的，除非它被用户定义的构造函数覆盖。在类型上调用构造函数方法以构造或创建该类型的对象实例。

1.3.1.4如何在表中存储对象

对象可以存储在两种类型的表中：

  • 对象表：仅存储对象

    在对象表中，每行表示一个对象，称为行对象 。

  • 关系表：使用其他表数据存储对象

    存储为关系表的列或者是其他对象的属性的对象称为列对象 。示例1-2显示了存储person_typ对象实例的contacts表。

在包含它们的关系数据库之外具有含义的对象，或者在多个关系数据库对象之间共享的对象，应该可以作为行对象引用。也就是说，这些对象应该作为行对象存储在对象表中而不是存储在关系表的列中。

1.3.1.5用于标识行对象的对象标识符

您不能直接访问对象标识符，但可以对对象标识符进行引用（ REF ）并直接访问REF ，如“ 引用行对象 ”中所述 。

1.3.1.6对行对象的引用

REF是可以从对象标识符（OID）构造的行对象的逻辑指针或引用。

您可以使用REF获取，检查或更新对象。您可以更改REF ，使其指向同一对象类型层次结构的不同对象，或为其指定空值。

REF是Oracle数据库内置数据类型。REF和对象之间的REF模型关联集合，特别是多对一关系，从而减少了对外键的需求。REF提供了一种在对象之间导航的简便机制。

例1-6说明了REF的简单使用。

示例1-6对emp_person_typ对象使用REF

CREATE TYPE emp_person_typ AS OBJECT（名称VARCHAR2（30），经理REF emp_person_typ ）; / CREATE TABLE emp_person_obj_table OF emp_person_typ; INSERT INTO emp_person_obj_table VALUES（emp_person_typ（'John Smith'，NULL））; INSERT INTO emp_person_obj_table SELECT emp_person_typ（'Bob Jones'，REF（e））FROM emp_person_obj_table e WHERE e.name ='John Smith';

1.3.1.11用于访问关系数据的对象视图

对象视图是使用对象关系功能访问关系数据的一种方法。

通过对象视图，您可以开发面向对象的应用程序，而无需更改基础关系模式。

您可以像访问对象表中的行对象一样访问属于对象视图的对象。Oracle数据库还支持存储在关系模式和表中的数据的用户定义类型的物化视图对象。

使用对象视图可以利用类型层次结构可以实现的多态性 。多态表达式采用表达式声明类型的值或该类型的任何子类型。如果构建反映类型层次结构的部分或全部结构的对象视图层次结构，则可以查询层次结构中的任何视图，以便仅在您感兴趣的专业化级别访问数据。如果查询具有子视图的对象视图，则可以获取视图类型及其子类型的多态数据行。

1.3.1.12类型继承的使用

类型继承使您可以创建类型层次结构。

类型层次结构是一组日益专门化的子类型的连续级别，这些子类型派生自共同的祖先对象类型，称为超类型。派生的子类型继承父对象类型的功能，并可以扩展父类型定义。专用类型可以添加新属性或方法，或重新定义从父级继承的方法。生成的类型层次结构为管理应用程序模型的复杂性提供了更高级别的抽象。例如，可以从一般人物对象类型导出特定类型的人，例如学生类型或具有附加属性或方法的兼职学生类型。

1.3.1.13类型Evolution用于更改现有对象类型

类型evolution允许您修改或发展现有的对象类型，甚至是已经在表中使用的对象类型。

类型演化通过ALTER TYPE语句工作，使您能够通过对象类型的所有实例传播更改。

ALTER TYPE语句使用与CREATE TYPE语句基本相同的验证来检查要ALTER TYPE的依赖关系。如果类型或其任何依赖类型未通过类型验证，则ALTER TYPE语句将回滚。

使用更改类型的所有表和列的元数据将针对新类型定义进行更新，以便可以以新格式存储数据。现有数据可以一次性转换为新格式，也可以在更新时逐步转换为新格式。在任何一种情况下，即使数据仍以旧版本的格式存储，数据也始终以新类型定义的形式呈现。

外部程序

您可以在PL / SQL，Java，C或.NET中实现对象类型的数据库函数，过程或成员方法作为外部过程。外部过程最适合于使用C语言等低级语言更快速或更轻松地完成的任务。外部过程始终以数据库地址空间之外的安全模式运行。可以编写通用外部过程，将一个或多个参数声明为系统定义的泛型类型。因此，外部过程可以使用系统定义的泛型类型来处理任何内置或用户定义类型的数据。

对象类型转换器

对象类型转换器（OTT）通过使用来自Oracle数据字典的模式信息来生成包含Java类和C结构和指示符的头文件，从而为对象类型模式提供客户端映射。您可以在主机语言应用程序中使用这些生成的头文件来透明地访问数据库对象。

客户端缓存

Oracle数据库提供了一个对象缓存，用于有效访问存储在数据库中的持久对象。可以将对象的副本带入对象缓存中。一旦数据缓存在客户端中，应用程序就可以以内存速度遍历这些数据。通过使用Oracle Call Interface编程接口的对象扩展，可以将对缓存中的对象所做的任何更改提交到数据库。

Oracle调用接口和Oracle C ++调用接口

Oracle调用接口（OCI）和Oracle C ++调用接口为应用程序和工具开发人员提供了全面的应用程序编程接口。Oracle Call Interface提供了一个运行时环境，其中包含连接Oracle数据库的功能，以及控制访问数据库中对象的事务。它允许应用程序开发人员通过遍历连接对象的图形来导航地访问和操作客户端对象缓存中的对象及其属性，或者通过声明性SQL DML指定数据的性质来关联。 Oracle Call Interface提供了许多函数来访问运行时在数据库中定义的对象类型的元数据。

Pro * C / C ++对象扩展

Oracle Pro * C / C ++预编译器提供嵌入式SQL应用程序编程接口，并提供比Oracle调用接口更高级别的抽象。与Oracle Call Interface一样，Pro * C / C ++预编译器允许应用程序开发人员使用Oracle客户端对象缓存和对象类型转换器实用程序。Pro * C / C ++支持对Oracle对象类型使用C绑定变量。Pro * C / C ++还提供了简化的语法来分配和释放SQL类型的对象，并使用SQL DML或导航界面访问它们。

.NET Object Extensions

用于Visual Studio的Oracle开发人员工具（ODT）和用于.NET的Oracle数据提供程序（ODP.NET）支持映射到Oracle对象关系数据类型，集合和REF的.NET自定义对象。ODT是一组集成到Visual Studio集成开发环境中的工具，允许在Oracle数据库中管理这些数据类型。通过ODT自定义类向导，Oracle对象可以自动映射到.NET自定义类型，以简化Oracle数据库和.NET应用程序之间的数据共享。通过ODP.NET进行对这些.NET自定义类型的数据访问。
