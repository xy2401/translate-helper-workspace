关于CVU升级验证命令选项

查看有关运行升级验证的此信息。

  • 运行Oracle Universal Installer（OUI），并允许OUI中内置的Cluster Verification Utility（CVU）验证执行系统检查并生成修复脚本。

  • 运行CVU手动脚本cluvfy.sh以执行系统检查并生成cluvfy.sh脚本。

要使用OUI执行安装前检查并生成修复脚本，请像平常一样运行安装。OUI启动CVU，并在安装过程中执行系统检查。如果您认为已完成预安装检查，并且要确认系统配置满足最低安装要求，则选择OUI以执行这些检查尤为合适。

要为CVU使用cluvfy.sh命令行脚本，请导航到解压缩要升级的映像文件的新Grid主目录，其中包含runcluvfy.sh脚本，然后运行命令runcluvfy.sh stage -pre crsinst -upgrade to检查Oracle Clusterware安装是否准备好进行升级。使用-pre crsinst -upgrade选项运行runcluvfy.sh会执行系统检查，以确认群集是否处于从现有集群件安装升级的正确状态。

该命令使用以下语法，其中变量内容以斜体表示：

runcluvfy.sh stage -pre crsinst -upgrade [-rolling] -src_crshome src_Gridhome ] -dest_crshome dest_Gridhome -dest_version dest_release [-fixup] [ -  fixupnoexec] [ - 方法sudo -user user_name [-location dir_path ] [ -  method root] [ -详细]

选项是：

  • -rolling

    使用此选项可验证滚动升级的准备情况。

  • -src_crshome src_Gridhome

    使用此选项指示要升级的源Oracle Clusterware或Grid home的位置，其中src_Gridhome是要升级的主src_Gridhome的路径。

  • -dest_crshome dest_Gridhome

    使用此选项指示升级Grid home的位置，其中dest_ Gridhome是Grid home的路径。

  • -dest_version dest_release

    使用-dest_version选项指示升级的版本号，包括任何补丁集。版本号必须包含五个数字，指定发布到特定于平台的修补程序级别。例如： 19.0.0.0.0 。

  • -fixup [-method sudo -user user_name [-location dir_path ][-method root ]

    使用-fixup选项表示您要为完成所需的任何步骤生成说明，以确保您的群集已准备好进行升级。默认位置是CVU工作目录。

    -fixup -method选项定义运行root脚本的方法。-method标志需要以下选项之一：

    • sudo ：在sudoers列表中以用户身份运行。

    • root ：以root用户身份运行。

    如果选择sudo ，则输入-location选项以在服务器上提供Sudo的路径，并输入-user选项以向用户帐户提供Sudo权限。

  • -fixupnoexec

    如果指定了该选项，则在验证失败时，将生成修正数据，并显示手动执行生成的修正的指令。

  • -verbose

    使用-verbose标志生成单个检查的详细输出。
