C.1添加brokerconfig

将Oracle Data Guard代理配置添加到全局数据服务池。

句法

add brokerconfig -connect connect_identifier [-pwd password ] [-gdspool gdspool_name ] [-region region_name ] [-savename] [-force]

选项

表C-1 GDSCTL添加brokerconfig选项

选项 描述

-connect connect_identifier

指定Oracle Net连接描述符或网络服务名称，该名称解析为代理配置中数据库的连接描述符。

-力

如果指定，则删除现有GDS配置。删除现有的正在运行的SDB，只有在想要删除整个SDB时才能使用它。

-gdspool gdspool_name

要添加Oracle Data Guard代理配置数据库的池。

如果指定的全局数据服务池已包含数据库或其他配置，则GDSCTL将返回错误。

-pwd 密码

GSMUSER的密码。如果未指定-pwd ，则会提示您输入密码。

-region region_name

数据库所属的全局数据服务区域。如果指定区域，则会将所有数据库添加到该区域。如果未指定区域，则会为所有数据库添加UNASSIGNED区域。如果该区域为UNASSIGNED，则必须使用modify database命令更改该区域。

-savename

指定此选项可存储在Global Data Services目录中使用-connect选项指定的网络服务名称，而不是映射到该网络服务名称的连接描述符。

使用说明

  • 在运行add brokerconfig命令之前，必须使用connect命令以具有池管理员权限的用户（GSMUSER数据库帐户）连接到Global Data Services目录数据库。您应该使用CONNECT命令连接到要为其添加代理配置的数据库的GSMUSER。

  • 如果GDS池已具有数据库或其他配置，则会返回错误。如果指定了-region ，则它仅定义主数据库的区域。如果目录中有多个区域，则将取消分配备用数据库的GDS区域属性。用户必须使用修改数据库来指定GDS区域。

例子

将DB1数据库的Oracle Data Guard代理配置添加到全局数据服务池MYREADERFARM和WEST区域。

GDSCTL>添加brokerconfig -connect 192.168.1.1:1521:sid-region west -gdspool myreaderfarm

例外或错误代码

如果您错误地使用此命令，GDSCTL将返回下面列出的错误。

表C-2 GDSCTL添加brokerconfig异常或错误代码

例外 描述

ERROR-44866

池只能包含一个Data Guard代理配置。如果全局数据服务池已包含Oracle Data Guard代理配置，则GDSCTL将返回错误44866，因为在这种情况下必须使用Oracle Data Guard添加数据库。

C.2添加cdb

将cdb添加到分片目录。

句法

add cdb -connect connect_identifier [-pwd gsmrootuser_pwd ] [-savename] [-cpu_threshold cpu ] [-disk_threshold disk ] [ rack_id ] [-force]

选项

表C-3 GDSCTL添加cdb选项

选项 描述
-connect connect_identifier

指定Oracle Net连接描述符或网络服务名称，该名称解析为要添加为分片的数据库的连接描述符。
-pwd gsmrootuser_pwd

输入GSMROOTUSER密码。如果未指定，则提示用户输入密码。
-savename

在分片目录中存储使用-connect选项指定的网络服务名称，而不是映射到该网络服务名称的连接描述符。

-力

如果指定，将重写分片上和分片目录中包含有关此分片的信息的现有GDS和分片配置。
-cpu_threshold cpu

指定CPU利用率百分比阈值。
-disk_threshold disk

指定同步单块读取的平均延迟（以毫秒为单位）。
rack_id

指定机架（硬件机柜）的标识符，或具有类似可用性特征的其他物理分组节点。如果指定，GDS将强制包含复制数据的数据库不会放在同一个机架中。如果不可能，则会引发错误。

使用说明

ADD CDB将有关ADD CDB元数据添加到分片目录中。仅当您打算在ADD SHARD命令中使用-cdb选项将PDB部署为分-cdb才需要此命令。CDB可以支持来自不同分片数据库的多个PDB分片;但是，对于每个CDB，此支持仅限于给定分片数据库中的一个PDB分片。

例子

将一个名为db11的CDB添加到分片目录中。

GDSCTL> add cdb -connect db11 -pwd gsmrootuser_pwd

选项

表C-4 GDSCTL添加凭证选项

选项 描述

指定要添加的凭据的名称。

C.4添加数据库

将数据库添加到全局数据服务区域和全局数据服务池。

句法

add database -connect connect_identifier [-region region_name ] [-gdspool gdspool_name ] [-pwd password ] [-savename] [-cpu_threshold cpu ] [-disk_threshold disk ]

选项

表C-5 GDSCTL添加数据库选项

选项 描述

-connect connect_identifier

指定Oracle Net连接描述符或网络服务名称，该名称解析为要添加的数据库的连接描述符。

-cpu_threshold cpu

指定CPU利用率百分比阈值。

-disk_threshold 磁盘

指定同步单块读取的平均延迟（以毫秒为单位）。

-gdspool gdspool_name

数据库所属的全局数据服务池。

-pwd 密码

GSMUSER的密码。如果未指定-pwd ，则会提示您输入密码。

-region region_name

数据库所属的全局数据服务区域。

-savename

指定此选项可存储在Global Data Services目录中使用-connect选项指定的网络服务名称，而不是映射到该网络服务名称的连接描述符。

使用说明

  • 在运行此命令之前，必须使用connect命令以具有池管理员权限的用户身份连接到Global Data Services目录数据库。

  • 如果-savename 没有指定，那么GDSCTL替换指定用于保存配置到目录之前，完整的连接字符串的网络服务名称是什么。

  • GDSCTL的默认设置是为目录启用autovncr 。如果已为目录禁用autovncr ，则在配置全局数据服务池并将数据库添加到全局数据服务配置之前，运行这些数据库的节点必须是有效节点检查注册（VNCR）列表以进行数据库注册。使用add invitenode（add invitesubnet）命令定义有效节点。

例

将数据库DB1添加到WEST区域和全局数据服务池MYREADERFARM。

GDSCTL>添加数据库-connect 127.0.0.1:1521:db1-region west -gdspool myreaderfarm

使用myalias而不是IP地址连接字符串添加数据库。 

GDSCTL>添加数据库-connect myalias -gdspool myreaderfarm

表C-6 GDSCTL添加数据库异常或错误代码

选项

表C-8 GDSCTL添加gdspool选项

选项 描述

-gdspool gdspool_name_list

逗号分隔的全局数据服务池名称列表。

全局数据服务池在其GDS配置中必须具有唯一名称。如果在创建池时未指定池的名称，则名称默认为oradbpool 。池名称最长可达30个字节，可以是任何有效标识符（按字母顺序排列的字符后跟零个或多个字母数字ASCII字符或下划线（_））。

-users user_list

以逗号分隔的已授予池管理员角色的用户列表。

使用说明

rack_id

-replace old_db_name

替换时，此参数指定旧分片的db_unique_name。ADD SHARD命令的现有参数（例如connect ）必须引用新（替换）分片的属性。

Oracle GoldenGate环境不支持此参数。
-gg_service（http | https）： ogg_host:sm_port/deployment

此参数对于Oracle GoldenGate复制是必需的，并指定将在此分片中管理GoldenGate复制的GoldenGate Admin Server的URI。格式如下：

使用说明

在运行add shard之前，必须通过运行validateShard过程来validateShard ，如使用Oracle Sharding中所述

  • 执行DEPLOY命令后，分片将成为分片数据库的一部分，但-replace的情况除外。

  • 使用ADD SHARD添加到分片配置的任何数据库都不得在另一个配置中作为分片部署（除非指定了-replace ）。重新添加以前部署的分片将导致ADD SHARD命令成功，但在最终运行DEPLOY命令时，分片将无法成功部署并向分片控制器（GSM）注册。

  • ADD SHARD仅向GDS注册数据库（分片）。在新添加的数据库上未配置复制，并且在运行DEPLOY之前，不会将其他数据库中的数据分发给它。

  • 通过Data Guard复制，可以将分片添加为预先存在的Data Guard配置的备用分片。无需重新分片数据。期望添加的分片处于正确的配置状态;应该从主服务器克隆备用服务器并具有相同的DBID。运行DEPLOY ，现有的主数据库和备用数据库相互匹配，使用DBID形成代理配置。如果尚未配置代理，则会对其进行配置，否则将验证它是否已正确配置。一旦配置了代理，Data Guard就可以完成它的工作，如果需要，它应该能够在备用数据库上执行，然后再将它们联机。

  • 见与的Oracle GoldenGate拆分工作 使用的Oracle GoldenGate微服务架构指南了解如何使用的Oracle GoldenGate和Oracle拆分的详细信息在融合中间件。

  • 使用-replace参数时，请参阅使用Oracle Sharding以获取有关其用法的更多信息。

自动VNCR最适用于具有简单专用网络的环境，其中易于配置是最重要的考虑因素。要对哪些主机可能参与GDS配置具有最高级别的控制，请禁用自动VNCR并将每个数据库主机的IP地址显式添加到VNCR配置。

  • 目录或全局服务管理器主机不知道如何将在目标数据库主机上发现的主机名转换为真实IP地址。如果主机文件中的名称或目录或全局服务管理器主机上的DNS具有不同的名称，或者它们在这些主机上不存在，则会发生这种情况。

  • 数据库正在运行Oracle RAC，而其他Oracle RAC实例在不同的子网上运行。这不是Oracle RAC的推荐配置。有关建议的配置，请参阅Oracle RAC文档。使用Oracle RAC，Oracle数据库连接到单个数据库主机以验证目标，并返回包含主机所在的整个子网的子网掩码。如果其他实例位于不同的子网上，则它们没有有效的VNCR条目，并且拒绝注册。

如果未启用—autoVNCR ，或者，如果上述任何一种情况适用，则应使用add invitenode手动添加新主机（添加invitesubnet） 。

GDSCTL>创建目录-database db1 -region west，east -gdspool readerfarm

  • 此命令创建每个区域并将区域添加到目录。如果未指定区域，则会创建名为REGIONORA的默认区域。

  • 对于Data Guard复制， PROTECTMODE参数适用于在ADD SHARDSPACE命令中未指定保护模式而创建的任何分片ADD SHARDSPACE 。

  • 自动VNCR最适用于具有简单专用网络的环境，其中易于配置是最重要的考虑因素。要对哪些主机可能参与GDS配置具有最高级别的控制，请禁用自动VNCR并将每个数据库主机的IP地址显式添加到VNCR配置。

    启用—autovncr ，Oracle会在add shard执行期间验证目标分片的主机名。然后，此主机将作为受邀节点自动添加到目录中的VNCR列表中。此机制与所有网络配置不兼容，在以下情况下可能无效：

    • 目录或全局服务管理器主机不知道如何将在目标分片主机上发现的主机名转换为真实IP地址。如果主机文件中的名称或目录或全局服务管理器主机上的DNS具有不同的名称，或者它们在这些主机上不存在，则会发生这种情况。

    • 目标分片主机具有多个公共网络地址，Oracle选择的地址与分片向全局服务管理器注册时使用的地址不同。如果主机具有多个网卡或已配置虚拟网络接口，则可能发生这种情况。

    • 该分片正在运行Oracle RAC，而其他Oracle RAC实例在不同的子网上运行。这不是Oracle RAC的推荐配置。有关建议的配置，请参阅Oracle RAC文档。使用Oracle RAC，Oracle数据库连接到单个分片主机以验证目标，并返回包含主机所在的整个子网的子网掩码。如果其他实例位于不同的子网上，则它们没有有效的VNCR条目，并且拒绝注册。

    • 见与的Oracle GoldenGate拆分工作 使用的Oracle GoldenGate微服务架构指南了解如何使用的Oracle GoldenGate和Oracle拆分的详细信息在融合中间件。

    如果未启用—autoVNCR ，或者，如果上述任何一种情况适用，则应使用add invitenode手动添加新主机（添加invitesubnet） 。
