4.15 maintQualifier

目的

使用maintQualifier子句指定数据库文件和归档重做日志文件。

句法

maintQualifier :: =


插图描述maintqualifier.eps

（ completedTimeSpec :: = ， deviceSpecifier :: = ）

语义

maintQualifier

语法元素 描述

completedTimeSpec

指定完成备份或复制的时间范围。

另请参见： completedTimeSpec

DEVICE TYPE deviceSpecifier

仅为指定的设备类型分配自动通道。仅当您配置了自动通道且未手动分配通道时，此选项才有效。例如，如果配置自动磁盘和磁带通道，并发出CHANGE...DEVICE TYPE DISK ，然后RMAN仅分配磁盘通道。

另请参见： deviceSpecifier

LIKE ' string_pattern '

通过指定文件名模式来限制数据文件副本。该模式可以包含Oracle模式匹配字符百分号（ % ）和下划线（ _ ）。RMAN仅对名称与模式匹配的文件进行操作。

注意：您不能对LIST使用LIKE选项... ARCHIVELOG命令或备份片段。

GUID guid_name

指定执行指定操作的可插拔数据库（PDB）的GUID。使用唯一GUID标识每个PDB。您可以通过查询dba_pdb_history视图来确定已删除的PDB的GUID。

BACKED UP integer TIMES TO DEVICE TYPE deviceSpecifier

将命令限制为已成功备份integer次或多次的归档日志和备份集到指定的介质。

当BACKED UP选项与DELETE ARCHIVELOG命令一起使用时，RMAN使用BACKED UP设置而不是配置的设置来确定是否可以删除存档日志。也就是说，覆盖了CONFIGURE ARCHIVELOG DELETION POLICY 。使用FORCE和DELETE ARCHIVELOG覆盖此配置以及介质和存储库之间的任何不匹配。

TAG tag_name

按标记指定数据文件副本和备份集。标记名称不区分大小写并以全部大写形式显示。

另请参阅： BACKUP以获取有关如何将标记应用于双工备份集的单个副本的说明，以及有关标记的默认文件名格式的说明
