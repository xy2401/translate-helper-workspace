63 DBMS_EDITIONS_UTILITIES

63.1 DBMS_EDITIONS_UTILITIES概述

DBMS_EDITIONS_UTILITIES包实现了一个接口，该接口为与版本相关的操作提供帮助函数。

63.2 DBMS_EDITIONS_UTILITIES安全模型

该软件包由SYS拥有，并授予PUBLIC执行访问权限。它使用调用者的权限运行，即使用调用者的安全配置文件。

63.4.1 CLEAN_UNUSABLE_EDITIONS程序

删除任何不可用版本中的所有覆盖对象，并删除空的不可用版本。

句法

CLEAN_UNUSABLE_EDITIONS过程立即执行并完整执行。 

DBMS_EDITIONS_UTILITIES.CLEAN_UNUSABLE_EDITIONS（）;

使用说明

执行此过程时， COMPATIBLE参数必须设置为12.2.0或更高。

也可以看看：

  • 有关DROP EDITION信息，请参见Oracle数据库SQL语言参考

  • 有关使用基于版本的重定义的详细信息，请参见“Oracle数据库开发指南”

63.4.2 SET_EDITIONING_VIEWS_READ_ONLY过程

给定模式名称和表名称，此过程将所有版本中的相应编辑视图设置为READ ONLY或READ/WRITE 。

句法

DBMS_EDITIONS_UTILITIES.SET_EDITIONING_VIEWS_READ_ONLY（table_name IN VARCHAR2，owner IN VARCHAR2 DEFAULT NULL，read_only IN BOOLEAN DEFAULT TRUE）;

参数

表63-3 SET_EDITIONING_VIEWS_READ_ONLY过程参数

参数 描述

TABLE_NAME

版本视图的基表

所有者

基表架构。默认（或NULL ）是当前架构。

只读

TRUE ，则将视图设置为只读; FALSE （或NULL ）将视图设置为READ/WRITE 。默认值为TRUE 。

使用说明

用户必须具有以下权限：

  • 表的所有者，或具有ALTER ANY TABLE系统特权

  • 对定义了视图的所有版本的USE对象特权

63.4.3 SET_NULL_COLUMN_VALUES_TO_EXPR过程

此过程使用表达式的值替换替换列中的NULL值。

表达式评估成本推迟到将来的更新和查询。该过程仅适用于基于版本的重新定义（EBR）练习。
