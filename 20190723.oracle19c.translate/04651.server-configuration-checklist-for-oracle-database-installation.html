Oracle Inventory（oraInventory）和OINSTALL组要求

  • 对于升级，Oracle Universal Installer（OUI）会从/etc/oraInst.loc文件中检测现有的oraInventory目录，并使用现有的oraInventory。

  • 对于新安装，如果尚未配置oraInventory目录，则可以在软件安装期间指定oraInventory目录，Oracle Universal Installer将为您设置软件目录。Oracle清单是Oracle软件安装的Oracle基础目录级别之一，并将安装所有者的主要组指定为Oracle清单组。确保您指定的oraInventory路径符合Oracle Optimal Flexible Architecture建议。

Oracle Inventory目录是系统上安装的Oracle软件的中央库存。将Oracle Inventory组作为其主要组的用户将被授予OINSTALL权限以写入中央库存。

OINSTALL组必须是服务器上所有Oracle软件安装所有者的主要组。它应该可由任何Oracle安装所有者写入。
群组和用户 Oracle建议您在开始安装之前创建安全计划所需的组和用户帐户。安装所有者具有资源限制设置和其他要求。组和用户名必须仅使用ASCII字符。
软件二进制文件的挂载点路径 Oracle建议您创建最佳灵活体系结构配置，如适用于您的平台的Oracle数据库安装指南中的“最佳灵活体系结构”附录中所述。
确保Oracle主目录（您为Oracle数据库选择的Oracle主路径）仅使用ASCII字符

ASCII字符限制包括安装所有者用户名，它们用作某些主路径的默认值，以及您可以为路径选择的其他目录名。

取消设置Oracle软件环境变量

如果您已安装现有Oracle软件，并且使用相同的用户来安装此安装，则取消设置以下环境变量： $ORACLE_HOME ， $ORA_NLS10和$TNS_ADMIN 。

如果已将$ORA_CRS_HOME设置为环境变量，则在开始安装或升级之前取消设置它。除非Oracle支持人员指示，否则不要将$ORA_CRS_HOME用作用户环境变量。

设置区域设置（如果需要）

指定要在其中使用Oracle组件的语言和区域或区域设置。区域设置是运行系统或程序的语言和文化环境。NLS（本地语言支持）参数确定服务器和客户端上特定于语言环境的行为。组件的区域设置确定组件的用户界面的语言以及全球化行为，例如日期和数字格式。

检查共享内存文件系统挂载

默认情况下，您的操作系统在/etc/fstab包含一个用于挂载/dev/shm的条目。但是，如果群集验证实用程序（CVU）或Oracle Universal Installer（OUI）检查失败，请确保/dev/shm安装区域的类型为tmpfs并使用以下选项进行挂载：

  • rw和exec权限设置在它上面

  • 没有设置noexec或nosuid

注意：

可能未列出这些选项，因为它们通常被设置为操作系统的默认权限。
