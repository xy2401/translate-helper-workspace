2.3.3配置邮件通知系统

Oracle Health Check Collections Manager提供了一个用户可以订阅的电子邮件通知系统。

设置包括：

  • 配置电子邮件服务器，端口和电子邮件通知的频率。

  • 注册电子邮件地址

注意：

只有分配了Admin角色的用户才能管理Email Notification Server和作业详细信息 。

配置电子邮件通知系统：

  1. 登录到Oracle健康检查收藏管理器，然后单击管理在右上角。

    图2-1 Oracle运行状况检查集合管理器 - 管理


    “图2-1 Oracle运行状况检查集合管理器 - 管理”的说明
  2. 在管理下 ，单击管理电子邮件服务器和作业详细信

    图2-2 Oracle运行状况检查集合管理器 - 配置电子邮件服务器


    “图2-2 Oracle运行状况检查集合管理器 - 配置电子邮件服务器”的说明
    1. 指定有效的电子邮件服务器名称 ， 端口号 ，然后单击“ 设置我的电子邮件服务器设置”
    2. 根据您的需要设置电子邮件通知频率 。
      请参阅同一页面上的通知作业运行详细信息 。

      图2-3 Oracle运行状况检查集合管理器 - 通知作业运行状态详细信息


      “图2-3 Oracle运行状况检查集合管理器 - 通知作业运行状态详细信息”的说明
  3. 返回“ 管理”页面，然后单击“ 管理通知” 。

    图2-4 Oracle运行状况检查集合管理器 - 管理通知


    “图2-4 Oracle运行状况检查集合管理器 - 管理通知”的说明
    1. 如果您是第一次配置，请输入您的电子邮件地址。
      随后访问“ 管理通知”页面会自动显示您的电子邮件地址。
    2. 默认情况下，选中“ 订阅/取消订阅我的邮件通知” 。保持原样。
    3. 在“ 收集通知”下 ，选择要为其接收通知的集合类型。
    4. 选择此选项可在ORAchk CM表空间中的可用空间低于100 MB时接收通知。
    5. 通过点击Test 测试下自己的电子邮件设置验证通知传递。

      如果配置正确，则您必须收到电子邮件。如果您没有收到电子邮件，请咨询管理员。

      以下是示例通知：
      来自：username@example.com发送：2016年1月28日星期四下午12:21收件人：username@example.com主题：来自馆藏管理器的测试邮件测试馆藏管理员电子邮件通知系统
    6. 单击“ 提交”

注意：

管理菜单下的管理通知部分可用于不论角色的所有用户。

如果启用了ACL系统，则注册用户将收到有权访问的系统的通知。如果未配置ACL系统，则所有注册用户都会收到所有通知。

根据您在“ 收集通知”部分下所做的选择，您会收到一封电子邮件，其中包含主题：收集管理器通知，其中包含带有结果的应用程序URL。

图2-5 Oracle Health Check Collections Manager - 示例电子邮件通知


“图2-5 Oracle运行状况检查集合管理器 - 示例电子邮件通知”的说明

在“ 注释”列下，单击“ 单击此处”链接以获取详细信单击相应的URL，进行身份验证，然后查看相应的比较报告。
