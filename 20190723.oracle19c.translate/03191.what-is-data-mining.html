1.1什么是数据挖掘？

了解数据挖掘。

数据挖掘是一种发现以前未知的数据关系的技术。数据挖掘是自动搜索大型数据库以发现超出简单分析的模式和趋势的实践。数据挖掘使用复杂的数学算法来分割数据，并根据过去的事件预测未来事件的可能性。数据挖掘也称为数据中的知识发现（KDD）。

数据挖掘的关键属性是：

  • 自动发现模式

  • 预测可能的结果

  • 创建可操作的信息

  • 专注于大型数据集和数据库

数据挖掘可以回答无法通过简单查询和报告技术解决的问题。

1.1.1自动发现

数据挖掘由使用算法作用于一组数据的模型执行。数据挖掘模型可用于挖掘构建它们的数据，但大多数类型的模型可推广到新数据。将模型应用于新数据的过程称为评分 。

1.1.2预测

许多形式的数据挖掘都是预测性的。例如，模型可以根据教育和其他人口统计因素预测收入。预测具有相关的概率（这种预测有多大可能是真的？）。预测概率也称为置信度 （我对这种预测有多大信心？）。

某些形式的预测数据挖掘会生成规则 ，这些规则意味着给定的结果。例如，规则可以指定具有学士学位并且居住在某个社区的人可能具有大于区域平均水平的收入。规则有相关的支持 （人口中有多少百分比满足规则？）。

1.1.3分组

其他形式的数据挖掘识别数据中的自然分组。例如，模型可以识别具有指定范围内的收入，具有良好驾驶记录并且每年租赁新车的人口部分。

1.1.4可操作的信息

数据挖掘可以从大量数据中获取可操作的信息。例如，城镇规划者可能会使用一种模型来预测基于人口统计的收入，以制定低收入住房计划。汽车租赁代理商可能会使用识别客户细分的模型来设计针对高价值客户的促销。

1.1.5数据挖掘和统计

数据挖掘和数据挖掘之间存在很多重叠统计。事实上，数据挖掘中使用的大多数技术都可以放在统计框架中。但是，数据挖掘技术与传统统计技术不同。

统计模型通常对数据做出强有力的假设，并且基于这些假设，他们对结果做出强有力的陈述。但是，如果假设存在缺陷，那么模型的有效性就会变得有问题。相比之下，数据挖掘中使用的机器学习方法通常会对数据做出微弱的假设。因此，数据挖掘通常不能对结果做出如此强烈的陈述。然而，无论数据如何，数据挖掘都可以产生非常好的结果。

通常，传统的统计方法需要大量的用户交互才能验证模型的正确性。结果，统计方法可能难以自动化。统计方法依赖于检验假设或基于较大人口的较小的代表性样本找到相关性。

数据挖掘需要较少的用户交互和较少的数据知识。用户不需要按摩数据以保证方法对给定数据集有效。数据挖掘技术比传统统计技术更容易自动化。

1.1.6数据挖掘和OLAP

在线分析处理（OLAP）可以定义为快速分析多维数据。OLAP和数据挖掘是不同但互补的活动。

OLAP支持数据汇总，成本分配，时间序列分析和假设分析等活动。但是，大多数OLAP系统都没有归纳推理能力超出了对时间序列预测的支持。归纳推理是从具体例子中得出一般结论的过程，是数据挖掘的一个特征。归纳推理也称为计算学习。

OLAP系统提供数据的多维视图，包括对层次结构的完全支持。这种数据视图是分析企业和组织的自然方式。

数据挖掘和OLAP可以通过多种方式集成。OLAP可用于分析不同粒度级别的数据挖掘结果。数据挖掘可以帮助您构建更有趣和有用的多维数据集。例如，可以将预测数据挖掘的结果作为自定义度量添加到多维数据集。这些措施可以为每个客户提供诸如“可能违约”或“可能购买”的信息。然后，OLAP处理可以聚合和汇总概率。

1.1.7数据挖掘和数据仓库

数据可以是否存储在平面文件，电子表格，数据库表或其他一些存储格式中。数据的重要标准不是存储格式，而是它对要解决的问题的适用性。

适当的数据清理和准备对于数据挖掘非常重要，数据仓库可以促进这些活动。但是，如果数据仓库不包含解决问题所需的数据，则无用。

1.2数据挖掘可以做什么和不做什么？

数据挖掘是一种功能强大的工具，可以帮助您查找数据中的模式和关系。但数据挖掘本身并不起作用。它不会消除了解您的业务，了解您的数据或理解分析方法的需要。数据挖掘可以发现数据中的隐藏信息，但它无法告诉您组织信息的价值。

随着时间的推移，您可能已经意识到了重要的模式。除了寻找通过简单观察无法立即辨别的新模式之外，数据挖掘还可以确认或限定此类经验观察。

重要的是要记住，通过数据挖掘发现的预测关系不是因果关系。例如，数据挖掘可能会确定订阅某些杂志的收入在50,000美元到65,000美元之间的男性可能会购买给定的产品。您可以使用此信息来帮助您制定营销策略。但是，您不能假设通过数据挖掘识别的人口购买产品， 因为他们属于这一群体。

数据挖掘产生概率，而非精确答案。重要的是要记住罕见的事件可能发生;他们只是不经常发生。

1.2.1提出正确的问题

数据挖掘不会在没有指导的情况下自动发现信您通过数据挖掘找到的模式非常不同，具体取决于您如何制定问题。

要获得有意义的结果，您必须学习如何提出正确的问题。例如，您可能会尝试找到过去曾对您的请求做出回应的人的特征，而不是尝试学习如何“改进对直接邮件征集的响应”。

1.2.2了解您的数据

确保有意义的数据挖掘结果，您必须了解您的数据。数据挖掘算法通常对数据的特定特征敏感： 异常值（与数据库中的典型值非常不同的数据值），不相关的列，一起变化的列（例如年龄和出生日期），数据编码以及您选择包含或排除的数据。Oracle Data Mining可以自动执行算法所需的大部分数据准备。但是一些数据准备通常特定于域或数据挖掘问题。无论如何，您需要了解用于构建模型的数据，以便在应用模型时正确解释结果。

1.3.1问题定义

数据挖掘项目的初始阶段侧重于理解项目目标和要求。从业务角度指定问题后，您可以将其制定为数据挖掘问题并制定初步实施计划。

例如，您的业务问题可能是：“我如何向客户销售更多产品？“您可能会将此转化为数据挖掘问题，例如：”哪些客户最有可能购买该产品？“预测谁最有可能购买产品的模型必须建立在描述过去购买过产品的客户的数据之上。在构建模型之前，您必须汇总可能包含已购买产品的客户与未购买产品的客户之间关系的数据。顾客属性可能包括年龄，孩子数，居住年限，所有者/租房者等。

1.3.2数据收集，准备和特征工程

了解如何收集数据，准备数据和工程师功能以解决业务问题。

数据理解阶段涉及数据收集和探索。当您仔细查看数据时，您可以确定它解决业务问题的程度。您决定删除一些数据或添加其他数据。这也是识别数据质量问题和扫描数据模式的时候。

数据准备阶段涵盖了创建所涉及的所有任务用于构建模型的表或视图。数据准备任务可能会多次执行，而不是按照任何规定的顺序执行。任务可以包括列选择和视图创建，以及数据清理和转换。例如，您可以将DATE_OF_BIRTH列转换为AGE ;您可以在INCOME空的情况下插入中位数收入。

此外，您可以添加新的计算属性，以便更接近通常称为特征工程的数据表面的信息。例如，您可以创建一个新属性，而不是使用购买金额：“购买的次数金额在12个月的时间内超过500美元。”经常进行大额购买的客户也可能与回复或不回应优惠的客户有关。

周密的数据准备和创建捕获领域知识的新“工程特性”可以显着改善通过数据挖掘发现的信息。使数据分析师能够在Oracle数据库中执行这些数据汇编，数据准备，数据转换和功能工程是Oracle的一个重要区别。

注意：

Oracle Data Mining支持自动数据准备（ADP）， 这极大地简化了数据准备过程。

1.3.3模型构建和评估

在此阶段，您可以选择并应用各种建模技术并将参数校准为最佳值。如果算法需要数据转换，那么您需要返回上一阶段来实现它们 。

在初步模型构建中，由于最终数据集可能包含数千或数百万行，因此使用减少的数据集通常是有意义的。

在项目的这个阶段，是时候评估模型如何满足最初规定的业务目标（阶段1）。如果该模型应该预测可能购买产品的客户，那么它是否足以区分这两个类别？有足够的电梯？是否存在权衡取舍混淆矩阵可以接受？可以通过添加文本数据来改进模型吗？应该交易数据，如购买 （市场篮子数据）包括在内？应该与假阳性或假阴性相关的成本是否被纳入模型？

1.3.4知识部署

知识部署是在目标环境中使用数据挖掘。在部署阶段，可以从数据中获得洞察力和可操作的信息。

部署可能涉及评分（将模型应用于新数据），提取模型细节（例如决策树的规则，或应用程序，数据仓库基础结构或查询和报告工具中的数据挖掘模型的集成。

由于Oracle Data Mining在Oracle数据库中构建和应用数据挖掘模型，因此可立即获得结果。BI报告工具和仪表板可以轻松显示数据挖掘的结果。此外，Oracle Data Mining支持实时评分：可以挖掘数据，并在单个数据库事务中返回结果。例如，销售代表可以运行模型，该模型预测在线销售交易的上下文中欺诈的可能性。
