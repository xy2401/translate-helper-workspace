9.1关于RDF与Property Graph数据的集成

属性图数据模型比RDF数据模型简单，因为它没有全局资源标识（即没有URI）或形式语义和蕴涵的概念。 此外，属性图允许属性（键值对）与边缘直接关联。相比之下，RDF需要具体化或四元数据模型来将属性与边缘相关联（RDF三元组）。

Oracle Spatial and Graph为存储在Oracle数据库中的属性图数据的RDF视图提供内置支持。这些RDF视图充当属性图数据和RDF数据之间的集成点。属性图数据的RDF视图与其他RDF视图的行为方式相同;您可以对它们运行SPARQL查询，并将它们实现为本机RDF模型。通过以下组件提供对属性图的RDF视图的支持：

  • 用于存储属性图数据[ref to schema]的关系模式的内置R2RML映射。

  • 一个PL / SQL API，用于使用属性图数据的内置R2RML映射创建和维护RDF视图。

在RDF中表示属性图数据时有两个主要考虑因素：

  • 如何从属性图标识符和值生成语法上有效的RDF术语（URI，文字等）

  • 如何表示边缘属性（边的键值对）

Oracle Spatial and Graph使用特定前缀从属性图标识符生成URI，并使用XML Schema类型文字作为属性值。命名图用于模拟边缘属性。

9.3用于创建和维护属性图RDF视图的PL / SQL API

SEM_APIS包中的子程序简化了属性图RDF视图的创建和维护。

这些子程序的参考和使用信息包含在SEM_APIS包子程序章节中。

要从现有模型创建属性图表视图，请使用SEM_APIS.CREATE_PG_RDFVIEW过程。

要删除属性图RDF视图，请使用SEM_APIS.DROP_PG_RDFVIEW 。

应在属性图表上创建索引，以提高RDF视图查询的性能。您可以在这些表上创建任意数量的索引方案，但为方便起见，提供了SEM_APIS.BUILD_PG_RDFVIEW_INDEXES过程。（要删除该过程创建的所有索引，可以使用SEM_APIS.DROP_PG_RDFVIEW_INDEXES过程。）

要返回RDF术语的规范版本的VALUE_ID值（如果术语不存在，则返回NULL），您可以使用SEM_APIS.RES2VID函数。

9.5使用属性图RDF视图时的特殊注意事项

使用属性图RDF视图时，以下特殊注意事项适用。

  • 不支持长度大于4000字节的顶点和边属性值。

  • 在生成边标签URI时，边标签值将替换为IRI安全形式（如W3C R2RML规范中所述）。

  • 在生成顶点和边缘属性名称URI时，顶点和边缘属性名称将替换为IRI安全形式（如W3C R2RML规范中所述）。

  • 字符串值顶点和边属性值中的特殊字符和非ASCII字符将被转义（如W3C N-Triples规范中所述）。
