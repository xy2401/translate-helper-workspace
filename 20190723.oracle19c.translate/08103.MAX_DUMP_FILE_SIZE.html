1.188 MAX_DUMP_FILE_SIZE

MAX_DUMP_FILE_SIZE指定跟踪文件的最大大小（不包括警报日志）。

属性 描述

参数类型

串

句法

MAX_DUMP_FILE_SIZE = { integer [K | M | G] | UNLIMITED }

默认值

无限

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

价值范围

0到无限制，或UNLIMITED

基本

没有

如果您担心跟踪文件可能占用太多空间，则可以更改此限制。

  • MAX_DUMP_FILE_SIZE的数值指定操作系统块中的最大大小。

  • 数字值后跟K或M或G后缀指定文件大小，以千字节，兆字节或千兆字节为单位。

  • 特殊值字符串UNLIMITED表示跟踪文件大小没有上限。因此，转储文件可以与操作系统允许的一样大。

当跟踪文件的大小有限时，如果需要，它可以自动拆分为多个文件，称为段。段将具有与活动跟踪文件相同的文件名，但附加了额外的段号。

跟踪文件最多可以拆分为5个段，每个段的大小通常为跟踪文件限制的1/5。

当所有跟踪文件段的组合大小超过指定的限制时，将删除最旧的段，并创建一个新的空段。因此，跟踪文件始终包含最新的跟踪信息。

请注意，永远不会删除第一个段，因为它可能包含有关进程初始状态的相关信息。
