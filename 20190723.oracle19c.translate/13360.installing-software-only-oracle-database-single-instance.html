为单个实例安装新的Oracle数据库软件

使用此过程概述可帮助您为单个实例部署安装新Oracle Database版本的软件。

注意：

当源和目标Oracle主目录由不同用户拥有时，无法使用数据库升级助手（DBUA）升级数据库。试图这样做会返回错误PRKH-1014 。确保源数据库和目标数据库具有相同的所有者，或执行手动升级。

要为此版本安装新的Oracle数据库软件：

  1. 按照Oracle操作系统特定文档中的说明准备安装Oracle数据库软件。

  2. 启动Oracle Universal Installer，然后选择仅限软件的安装。

    成功完成Oracle数据库软件的安装后，单击“ 退出”以关闭Oracle Universal Installer。

  3. 运行升级前信息工具。该工具使您可以检查DBUA检查的项目类型。该工具可识别问题，并可帮助您解决发现的一些问题。

    默认情况下，该工具位于New_release_Oracle_home /rdbms/admin/preupgrade.jar位置。

    如果使用Oracle Label Security，Oracle Database Vault或两者，则在“ 选择数据库版本”页面上选择Enterprise Edition ，单击“ 选择选项” ，然后从组件列表中启用一个或两个组件。
