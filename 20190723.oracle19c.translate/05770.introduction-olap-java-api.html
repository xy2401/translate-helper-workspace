1 OLAP Java API简介

本章介绍Oracle OLAP Java应用程序编程接口（API）。本章包括以下主题：

  • OLAP Java API概述

  • 通过OLAP Java API访问数据

  • OLAP Java API示例的示例模式

  • OLAP Java API应用程序执行的任务

1.1 OLAP Java API概述

Oracle OLAP Java API是一个应用程序编程接口，可通过OLAP选项访问Oracle数据库中的联机分析处理（OLAP）技术。本主题列出了OLAP Java API客户端应用程序可以执行的操作，描述了OLAP Java API中的类，描述了维度数据模型中的对象，并讨论了用于联机分析处理的组织数据。

有关OLAP技术优势的说明，请参阅“ Oracle OLAP用户指南” 。该文档描述了Oracle OLAP为商业智能和高级分析应用程序分析多维数据提供的功能。它深入描述了维度数据模型，并讨论了与Oracle OLAP相关的数据库管理和管理任务。

1.1.1 OLAP Java API可以做什么

使用OLAP Java API，您可以开发执行以下操作的客户端应用程序。

  • 在与Oracle数据库实例的JDBC连接中建立一个或多个用户会话。多个用户会话可以共享相同的连接和相同的元数据对象缓存。

  • 使用数据库管理OLAP事务。

  • 使用OLAP元数据对象实现维度数据模型。

  • 创建和维护分析工作区。

  • 创建逻辑元数据对象并将它们映射到关系源。

  • 将元数据对象部署为分析工作区或关系表和视图，并将对象提交到数据库。

  • 浏览元数据以发现可供查看或分析的数据。

  • 构造多维数据的分析查询。使最终用户能够根据用户的需要创建指定和操作数据的查询（例如，选择，聚合和计算数据）。

  • 当应用程序用户优化其分析时，修改查询，而不是完全重新定义它们。

  • 检索结构化以便以多维格式显示的查询结果。

有关其中一些操作的详细信息，请参阅“ OLAP Java API应用程序执行的任务 ” 。

1.1.3描述维度数据模型

数据仓库和OLAP应用程序基于数据的多维视图。该视图在包括以下维对象的维数据模型中实现。

多维数据集

多维数据集是具有相同维度集的度量的容器。多维数据集通常对应于单个关系事实表或视图。立方体的度量包含事实，维度使事实数据成形。通常，尺寸形成立方体的边缘，测量数据是立方体的主体。例如，您可以组织销售到多维数据集中的产品单位的数据，该多维数据集的边包含来自时间，产品，客户和渠道维度的成员的值，其主体包含来自销售单位数量的度量值和销售额度量的值。

OLAP Java API中的元数据对象不表示多维数据集边缘的OLAP概念，但边缘通常包含在使用OLAP Java API的应用程序的设计中。每条边包含一个或多个维度的成员值。尽管对立方体上的边数没有限制，但是数据通常被组织用于沿三条边的显示目的，这三条边被称为行边，列边和页边。

措施

度量包含多维数据集中的事实数据。度量值按维度进行组织和标识。措施通常是多方面的。每个度量值由一组唯一的维度成员标识。这组维度成员称为元组 。

尺寸

维度包含唯一值列表，用于标识度量中的数据并对其进行分类。常用尺寸是客户，产品和时间。通常，维度具有一个或多个层次结构，这些层次结构将维度成员组织为父子关系。

通过指定要对数据执行的维度成员，度量和计算，最终用户可以制定业务问题并获取其查询的答案。例如，使用按月对数据进行分类的时间维度，按单位项目对数据进行分类的产品维度，以及包含按月销售的产品单位数量的数据的度量，您可以制定一个查询，询问是否销售产品单位在1月或6月份较高。

层次结构

层次结构是维度的组件，用于将维度成员组织为父子关系。通常，在客户端应用程序的用户界面中，最终用户可以通过在父母和孩子之间向下钻取或向上钻取来扩展或折叠层次结构。父维度成员的度量值是子值的聚合。

维度可以具有多个层次结构。例如，时间维度可以具有日历年度层次结构和会计年度层次结构。层次结构可以是基于级别的，也可以是基于值的。

在基于级别的层次结构中，父级必须处于比该父级子级更高级别的级别。在多维数据集中，父项的度量值通常根据子项的值进行汇总。例如，时间维度可能包含年，季度和月份的级别。月份级别包含基础数据，这是最详细的数据。四分之一的度量值是该季度子项的月份值的汇总，一年的度量值是一年中子项的季度的汇总。通常，每个级别都映射到关系维度表中的不同列。

在基于值的层次结构中，父维度成员和子维度成员通常来自关系表中的同一列。另一列标识成员的父级。例如，值层次结构可以包含公司的所有员工，并为每个拥有一个员工的员工标识经理。包括经理在内的所有员工都来自同一专栏。另一栏将包含员工的经理。

级别

级别是基于级别的层次结构的组件。级别可以与多个层次结构相关联。维度成员只能属于一个级别。

级别通常对应于维度表或视图中的列。基本级别是主键。

属性

属性包含与维度成员相关的信息。最终用户可以使用属性来选择数据。例如，最终用户可以通过使用具有每个产品的描述性名称的属性来选择一组产品。属性包含在维度中。

查询

查询是特定数据集的规范。OLAP Java API中的术语查询是指Source对象，该对象指定一组数据，并且可以包括聚合，计算或使用该数据执行的其他操作。数据及其上的操作定义查询的结果集。在本文档中，通用术语查询是指Source对象。

API在oracle.olapi.syntax包中有一个Query类。Query表示多行，多列结果集，类似于关系表，SQL SELECT语句或OLAP函数。在将维度或度量映射到关系表或视图时，可以使用Query对象。

1.1.5组织OLAP数据

OLAP Java API使Java应用程序（包括applet）可以访问驻留在Oracle数据仓库中的数据。数据仓库是一种关系数据库，专为查询和分析而设计，而不是用于事务处理。仓库数据通常符合星型模式，星型模式是关系数据库的维度数据模型。星型模式由一个或多个事实表和一个或多个维度表组成。事实表具有包含维度表的外键的列。通常，通过提取转换传输（ETT）工具从事务处理数据库创建数据仓库。

要使数据仓库中的数据可供OLAP Java API应用程序访问，数据库管理员必须确保根据Oracle OLAP支持的组织配置数据仓库。星型模式就是这样一个组织，但不是唯一的组织。

在仓库中组织数据后，您可以使用OLAP Java API应用程序设计多维数据集，度量，维度等的OLAP维数据模型，并创建实现该模型的逻辑OLAP元数据对象。您将元数据对象映射到仓库中的数据并构建分析工作区。构建分析工作区使用OLAP元数据对象表示的数据填充OLAP视图和其他存储结构。您还可以使用Analytic Workspace Manager执行相同的任务。

OLAP Java API应用程序可以获取由Analytic Workspace Manager或OLAP Java API创建的OLAP元数据对象。它可以使用元数据对象来创建对仓库中的数据进行操作的查询。

分析工作区中的仓库数据集合是OLAP Java API提供访问权限的数据存储。当然，用户可以访问的数据范围受数据库管理员授予用户的权限的限制。

除了确保已正确准备数据和元数据之外，还必须确保应用程序用户可以与数据存储建立JDBC连接，并且用户具有允许他们访问数据的数据库权限。有关建立连接的信息，请参阅发现元数据 。

1.2通过OLAP Java API访问数据

Oracle OLAP元数据对象组织和描述客户端应用程序可用的数据。元数据对象也包含其他信息，例如数据的数据类型。但是，您无法直接从元数据对象检索数据。要指定所需的数据，必须创建查询。在指定数据时，通常必须指定一个或多个维度成员值。要检索指定的数据，请创建一个Cursor 。本主题简要介绍了这些操作。

查询OLAP元数据对象中包含的数据的另一种方法是通过SQL查询Oracle OLAP为元数据对象创建的视图。有关查询这些视图的信息，请参阅了解OLAP Java API元数据中的 “ 使用OLAP视图 ” 。

1.2.1创建查询

查询由oracle.olapi.data.source.表示oracle.olapi.data.source.Source对象。您从元数据对象获取Source ，并使用该Source对象指定要获取的数据。Source类具有用于选择和执行数据操作的方法。您可以使用这些方法以用户需要的任何方式操作数据。有关Source对象的信息，请参阅了解源对象和使用源方法进行查询 。

1.2.2指定维度成员

Oracle OLAP维度的成员通常组织为一个或多个层次结构。某些层次结构具有基于级别的父子关系，而某些层次结构具有基于值的关系。每个维度成员的值必须是唯一的。

OLAP Java API使用三部分格式来唯一标识维度成员。格式包含维度成员的层次结构，级别和值，从而标识维度中的唯一值。唯一值的第一部分是层次结构对象的名称，第二部分是级别对象的名称，第三部分是级别中成员的值。唯一值的部分由值分隔字符串分隔，默认情况下为双冒号（ :: :）。以下是名为TIME_AWJ的维度中名为CALENDAR_YEAR的层次结构中名为YEAR的级别的唯一成员值的TIME_AWJ 。

CALENDAR_YEAR :: YEAR :: CY2001

唯一值的第三部分是本地值。前面示例中的本地值标识日历年2001。

1.2.3创建游标

要检索Source指定的数据，请创建oracle.olapi.data.cursor.该Source Cursor 。然后，您可以使用此Cursor从数据存储中请求和检索数据。您可以指定Cursor在每次提取操作中检索的数据量（例如，足以填充用户界面中的40个单元格表）。然后，Oracle OLAP可以有效地管理为应用程序检索的数据块的计时，大小调整和缓存，因此您无需这样做。有关Cursor对象的信息，请参阅了解游标类和概念以及检索查询结果 。

1.3 OLAP Java API示例的示例模式

本文档中的OLAP Java API代码示例摘自Oracle技术网（OTN）网站上提供的一组示例程序。一个示例CreateAndBuildAW.java具有创建和构建分析工作区的方法。又如， SpecifyAWValues ，调用的方法CreateAndBuildAW.java和指定的值，如为关系表的列的对的元数据对象映射到数据源的元数据对象和名称的名称。这些示例生成的分析工作区名为GLOBAL_AWJ 。其他示例查询分析工作区。分析工作区中的元数据对象将映射到全局模式中的关系表中的列。

在OTN网站上，您可以下载包含创建全局模式的SQL脚本的文件和包含示例程序的文件。OTN网站位于http://www.oracle.com/technetwork/database/options/olap/index.html 。

要获取任一文件，请在网页的“下载”部分中选择“ 示例代码和架构 ”。要获取示例模式，请选择Global Schema 11g 。要获取示例程序，请选择“文档的示例程序” ，然后选择“下载11g第2版（11.2）的示例程序”以下载包含示例的压缩文件。

示例程序采用可以轻松添加到开发环境的包结构。这些类包括示例程序类扩展的基类，以及它们使用的实用程序类。基类是BaseExample11g.java 。实用程序类包括Context11g.java和CursorPrintWriter.java 。Context11g.java类具有创建与Oracle数据库实例的连接的方法，这些实例存储元数据对象，返回存储的元数据对象，并创建Cursor对象。CursorPrintWriter.java类是一个PrintWriter ，它具有显示Cursor对象内容的方法。

OLAP元数据对象由CreateAndBuildAW.java和SpecifyAWValues程序创建和构建。这些元数据对象包括以下内容：

  • GLOBAL_AWJ，它是包含其他对象的分析工作区。

  • PRODUCT_AWJ，这是产品的维度。它有一个名为PRODUCT_PRIMARY的层次结构。层次结构的最低级别具有产品项标识符，而较高级别具有产品系列，类和总产品标识符。

  • CUSTOMER_AWJ，这是客户的一个维度。它有两个层次结构，名为SHIPMENTS和MARKETS。每个层次结构的最低级别具有客户标识符，更高级别具有仓库，区域和总客户，以及帐户，市场细分和总市场标识符。

  • TIME_AWJ，是时间值的维度。它有一个名为CALENDAR_YEAR的层次结构。最低级别具有月份标识符，其他级别具有季度和年份标识符。

  • CHANNEL_AWJ，这是销售渠道的一个维度。它有一个名为CHANNEL_PRIMARY的层次结构。最低级别具有销售渠道标识符，而较高级别具有总渠道标识符。

  • UNITS_CUBE_AWJ，它是一个包含度量COST，SALES和UNITS的多维数据集。 COST具有产品单位成本的价值。SALES具有产品单位销售的美元金额。UNITS具有销售的产品数量的价值。立方体的大小由所有四个维度决定。多维数据集的聚合方法是SUM ，其中每个父项的每个值都是父项子项的值的总和。

  • PRICE_CUBE_AWJ，它是一个包含度量UNIT_COST和UNIT_PRICE的多维数据集。 UNIT_COST有单位的费用。UNIT_PRICE有单位的价格。多维数据集的大小由PRODUCT_AWJ和TIME_AWJ维度决定。多维数据集的聚合方法是AVG ，其中每个父级的值是父级子级的值的平均值。

有关发现分析工作区的OLAP元数据的程序示例，请参阅发现元数据 。

1.4 OLAP Java API应用程序执行的任务

使用OLAP Java API的客户端应用程序通常执行以下任务：

  1. 连接到数据存储并创建DataProvider和UserSession 。

  2. 创建或发现元数据对象。

  3. 根据需要部署，映射和构建元数据对象。

  4. 指定选择和操作数据的查询。

  5. 检索查询结果。

本主题的其余部分简要介绍了这些任务，本指南的其余部分提供了有关如何完成这些任务的详细信息。

任务1：连接到数据存储并创建DataProvider和UserSession

通过标识有关目标Oracle数据库实例的一些信息并在JDBC连接方法中指定此信息，可以连接到数据存储。建立连接后，您将创建一个DataProvider并使用它和连接来创建UserSession 。有关连接和创建DataProvider和UserSession详细信息，请参阅“ 发现元数据 ”中的“ 连接到Oracle OLAP ” 。

任务2：创建或发现元数据对象

您使用DataProvider获取MdmMetadataProvider 。MdmMetadataProvider允许访问数据存储中的所有元数据对象。接下来，您获得MdmRootSchema通过调用对象getRootSchema的方法MdmMetdataProvider 。MdmRootSchema对象包含数据库中的所有OLAP元数据对象。从MdmRootSchema ，您可以获取当前用户有权访问的模式的MdmDatabaseSchema对象。MdmDatabaseSchema表示由SQL语句SELECT username FROM all_users返回的命名Oracle数据库用户。

从MdmDatabaseSchema ，您可以发现架构拥有的现有元数据对象，也可以创建新的元数据对象。getMeasures和getDimensions等方法获取MdmDatabaseSchema拥有的所有度量或维度。findOrCreateAW和findOrCreateCube等方法获取分析工作区或多维数据集（如果存在），或者创建一个（如果尚不存在）。

从MdmDatabaseSchema包含的顶级元数据对象（例如分析工作区，多维数据集或维度）中，您可以获取它包含的对象。例如，从MdmPrimaryDimension ，您可以获取与其关联的层次结构，级别和属性。确定了用户可用的元数据对象后，您可以向用户显示相关的对象列表，以进行数据选择和操作。

有关元数据对象的说明，请参阅了解OLAP Java API元数据 。有关如何发现可用元数据的信息，请参阅发现元数据 。

任务3：部署，映射和构建对象

如果创建新的MdmCube或MdmPrimaryDimension ，则必须将其部署为分析工作区对象或关系OLAP（Rolap）对象。要部署多维数据集，可以调用MdmCube方法，例如findOrCreateAWCubeOrganization 。要部署dimension ，可以调用MdmPrimaryDimension方法，例如findOrCreateAWPrimaryDimensionOrganization 。

如果创建表示数据的新元数据对象，则必须指定将元数据对象映射到关系源表或视图的Expression ，或Oracle OLAP用于生成数据的Expression 。对于分析工作区包含的对象，可以在映射后构建元数据对象。有关创建元数据，部署，映射和构建元数据对象的信息，请参阅创建元数据和分析工作区 。

任务4：通过查询选择和计算数据

OLAP Java API应用程序可以构建针对数据存储的查询。典型的应用程序用户界面为用户提供了选择数据和指定使用数据执行的操作的方法。然后，数据操作代码将这些指令转换为针对数据存储的查询。查询可以像选择维度成员一样简单，也可以是复杂的，包括几个聚合和计算，涉及由维度成员选择指定的度量值。

表示查询的OLAP Java API对象是Source 。表示数据的元数据对象是MdmSource类的扩展。从MdmSource （例如MdmMeasure或MdmPrimaryDimension ，您可以获取Source对象。用的方法Source对象，就可以产生其他Source指定一个选择的元素的对象Source ，或指定的计算或其它操作上的值执行Source 。

如果要实现简单的用户界面，则可以仅使用Source对象的方法来选择和操作用户在界面中指定的数据。但是，如果要为用户提供多步骤选择过程以及修改查询或撤消其选择中的各个步骤的能力，则应设计并实现Template类。在每个Template的代码中，您使用Source类的方法，但Template类本身允许您动态修改和优化最复杂的查询。此外，您可以编写通用Template类，并在应用程序的各个部分重用它们。

有关使用Source对象的信息，请参阅了解源对象 。有关使用Template对象的信息，请参阅创建动态查询 。

任务5：检索查询结果

当OLAP Java API应用程序的用户正在选择，计算，组合和通常操作数据时，他们还希望查看其工作结果。这意味着应用程序必须从数据存储中检索查询的结果集，并以多维形式显示数据。要通过OLAP Java API检索查询的结果集，请为Source指定查询创建Cursor 。

您还可以获取Oracle OLAP为查询生成的SQL。为此，您SQLCursorManager为Source创建SQLCursorManager而不是创建Cursor 。SQLCursorManager的generateSQL方法返回Source指定的SQL。然后，您可以通过OLAP Java API之外的方法检索数据。

由于OLAP Java API旨在处理数据的多维视图，因此Source可以具有多维结果集。例如， Source可以代表MdmMeasure其由四个尺寸MdmPrimaryDimension对象。每个MdmPrimaryDimension都有一个关联的Source 。您可以通过将维度的Source对象连接到度量的Source来创建查询。生成的查询将度量的Source作为基础，并将维度的Source对象作为输出。

一个Cursor的查询Source具有相同的结构Source 。例如，刚刚提到的Source的Cursor具有作为度量数据的基值。Cursor还有四个输出。输出的值是维度的Source对象的值。

要通过Cursor检索所有数据项，可以循环遍历多维Cursor结构。该设计非常适合于绘制计算机屏幕的标准用户界面对象的要求。它特别适用于以多维格式显示数据。

有关使用Source对象指定查询的详细信息，请参阅了解源对象 。有关使用Cursor对象检索数据的更多信息，请参阅了解游标类和概念 。有关SQLCursorManager类的更多信息，请参阅Oracle OLAP Java API Reference 。
