1.178 LOG_ARCHIVE_TRACE

LOG_ARCHIVE_TRACE启用并控制日志归档和重做传输活动的综合跟踪信息的生成。

属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

没有

价值范围

0,1,2,4,8,16,32,64,128,256,512,1024,2048,4096,8192,16384,32768

基本

没有

Oracle RAC

多个实例可以具有不同的值。

将LOG_ARCHIVE_TRACE设置为非零值时输出的附加跟踪可以出现在跟踪文件中，用于归档过程，RFS过程，LGWR过程，SYNC过程，ASYNC过程，前台过程，MRP过程，恢复过程，日志应用过程，启动过程，关闭过程以及使用重做传输服务的其他过程。

表1-4描述了有效的LOG_ARCHIVE_TRACE值。

表1-4 LOG_ARCHIVE_TRACE值

值 描述 更多信息

0

禁用存档日志跟踪（这是默认值）

1

高级追踪

高级跟踪提供了大多数重做传输过程中的少量跟踪。如果指定的值不是高级别（非1），则即使未特别请求高级别跟踪，该值的高级别跟踪也将输出到跟踪文件。例如，如果将LOG_ARCHIVE_TRACE设置为128，则将获得与FAL引擎相关的所有跟踪消息， 包括来自FAL引擎的高级跟踪。

2

跟踪Data Guard接口

此跟踪级别适用于各种Data Guard配置参数，各种与Data Guard相关的SQL命令，各种Data Guard表访问以及一些Data Guard内部接口。

4

跟踪常见的重做传输服务

此跟踪级别用于通过重做传输提供给各种内部用户（例如RFS进程，LGWR进程和ASYNC进程）提供的许多常见服务。这些包括锁定原语，使用通用VOS（虚拟操作系统）接口，任务调度，进程监视，控制文件访问和其他各种常见服务。

跟踪LogMiner字典

您可以通过将所需跟踪级别的值相加来组合跟踪级别。例如，设置为3将生成级别1和级别2跟踪输出。您可以为主数据库和备用数据库设置不同的值。

当此参数设置为默认值0时，Oracle仍将生成适当的警报和跟踪条目以响应错误条件。如果在ALTER SYSTEM语句中动态更改此参数的值，则更改将立即生效。

来自重做传输进程的许多跟踪消息都包含跟踪消息开头的例程名称。例程名称的第一部分指定该例程的模块。表1-5显示了每个重做传输模块使用的跟踪值。请注意，传输模块的跟踪值可以在发行版之间更改。
