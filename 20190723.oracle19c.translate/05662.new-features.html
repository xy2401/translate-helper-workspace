REST启用SQL支持

与创建数据库链接相反，使用Oracle REST数据服务（ORDS）17.3或更高版本的Oracle Application Express安装现在可以通过REST端点执行任何SQL。开发人员可以通过在共享组件中定义名称，端点URL和身份验证信息，轻松创建REST启用的SQL引用。

Oracle Application Express通过REST将SQL或PL / SQL查询传递给ORDS，并返回自描述的JSON响应。JSON对象包含结果集元数据，结果数据和分页详细信息。

REST启用的SQL引用可用作所有报表类型的基础，例如交互式报表和经典报表，但不能用作交互式网格区域。引用也可以与日历，JET图表，树和PL / SQL进程一起使用。

Oracle数据库链接在每个SQL语句中定义，并通过SQL * Net（或在云环境中通过Internet）工作，并且必须在远程数据库中为每个执行的SQL或PL / SQL打开一个会话。相比之下，REST启用的SQL引用在Oracle Application Express工作区级别定义，并通过HTTP和HTTPS使用JSON，这使它们易于在云环境或Internet上使用。由于ORDS利用远程数据库上的连接池，因此引用也可以更好地扩展。

Web源模块

Oracle Application Express引入了一种称为Web Source Modules的新数据源类型，这是一种声明性方法，用于定义对外部REST API和通用JSON数据源的引用。Web源模块存储有关如何解析响应数据的其他元数据，并将其映射为包含行和列的虚拟表。模块可以包含一个或多个Web Source Operations，它们是对具体外部Web服务的引用。

Web源模块还可以包括后处理SQL，它在Oracle Application Express组件处理之前修改数据。此SQL可用于应用函数，聚合或与本地表连接。

Web源模块可用作所有报告类型的基础，例如交互式报告和经典报告，但不能用作交互式网格区域。这些模块还可以与日历，JET图表，树和PL / SQL进程一起使用。

在早期版本的Oracle Application Express中，可以定义SOAP和REST Web服务，然后在有限的Oracle Application Express组件中使用它们。定义此类服务是手动，耗时且容易出错的。新的Web源模块具有高度声明性，因为它们使用发现来理解和定义Web服务的传入结构。

新的REST研讨会

通过利用ORDS存储库提供REST服务，使用多种工具（包括Application Express，SQL Developer，SQL Plus和SQLcl）在单个位置管理RESTful服务要容易得多。

LISTAGG聚合的DISTINCT选项

LISTAGG聚合函数现在通过使用新的DISTINCT关键字支持重复消除。

LISTAGG聚合函数根据ORDER BY表达式为查询中的每个组排序行，然后将值连接成单个字符串。使用新的DISTINCT关键字，可以在连接成单个字符串之前从指定的表达式中删除重复的值。这消除了在使用聚合LISTAGG函数之前创建复杂查询处理以查找不同值的需要。使用DISTINCT选项，可以直接在LISTAGG函数中完成删除重复值的处理。

结果是更简单，更快速，更高效的SQL。

Broker配置中的简化数据库参数管理

用户现在可以使用SQL * Plus ALTER SYSTEM命令管理所有与Data Guard相关的参数设置，或者使用新的EDIT DATABASE管理 DGMGRL ...SET PARAMETER命令。在DGMGRL接口中进行的参数更改将立即在目标数据库上执行。此外，此新功能允许用户使用ALL限定符修改Data Guard配置中所有数据库的参数，从而无需附加到每个数据库并执行ALTER SYSTEM命令或为多个EDIT的每个数据库设置Broker属性PROPERTY命令。SHOW命令也已更新，以显示目标数据库中参数的当前设置。

此功能消除了数据库的Data Guard参数设置与Data Guard Broker的属性设置之间的任何不一致，通过允许通过SQL * Plus接口进行所有参数管理，简化了数据库参数的数据库管理员管理。

闪回主数据库时闪回备用数据库

闪回数据库将整个数据库移动到较旧的时间点，并使用RESETLOGS打开数据库。在Data Guard设置中，如果主数据库闪回，则备用站点不再与主数据库同步。在以前的版本中，将辅助节点设置为与主节点相同的时间点需要手动过程来闪回备用数据库。引入了一个新参数，该参数使备用数据库能够在主数据库上执行闪回数据库时自动闪回。

通过在主数据库闪回时自动闪回备用数据库，减少了时间，精力和人为错误，从而实现更快的同步和缩短的恢复时间目标（RTO）。

从主站点到备用站点传播还原点

可以在主站点上定义正常还原点或保证还原点，以便在出现任何逻辑损坏问题时实现快速时间点恢复。但是，此还原点存储在控制文件中，不会传播到备用数据库。如果发生故障转移，备用数据库将成为主数据库，并且还原点信息将丢失。此功能可确保还原点从主站点传播到备用站点，以便即使在故障转移事件后还原点也可用。

由于使用在主数据库上创建的还原点更新备用数据库，因此简化了故障转移后还原和恢复过程的复杂性。

Active Data Guard DML重定向

Active Data Guard DML重定向允许在Active Data Guard备用数据库上发出附带的DML。执行DML时，更新将传递到执行它的主数据库，并且事务的重做将更新备用数据库，之后控制权将返回给应用程序。执行DML保留事务的所有ACID属性。

当需要一些写入时，DML重定向允许更多应用程序从Active Data Guard备用数据库中受益。

定期清除闪回日志以提高FRA大小的可预测性

客户拥有许多都使用快速恢复区（FRA）的数据库。他们通常使用recovery_dest_size初始化参数订阅FRA。在空间压力需要闪回日志之前不会清除闪回日志时会出现问题。在许多情况下，唯一的补救措施是关闭闪回日志记录并将其重新打开。此功能使闪回空间的使用从存储管理角度变得可预测，因为闪回不会占用保留所需的空间。此功能还允许用户通过调整闪回保留来控制累积空间压力。

FRA对数据库至关重要，因为它存储备份，联机重做日志，归档重做日志和闪回日志。当FRA变满时，它会影响所有数据库。通过自动确保闪回日志不会过度使用保留所需的空间，可以提高存储管理并提高数据库的运行状况。

对于集中管理的用户，Oracle数据库最初尝试使用SASL bind连接到Active Directory。如果Active Directory服务器拒绝SASL绑定连接，则Oracle数据库会在没有SASL绑定的情况下自动再次尝试连接，但仍使用TLS进行保护。

Active Directory管理员负责配置Active Directory服务器的连接参数，但不需要配置数据库以匹配此新的Active Directory连接增强功能。数据库自动从使用SASL调整为不使用SASL绑定。

统一审计顶级语句

统一审计顶级报表功能使您可以审计数据库中的顶级用户（直接用户）活动，而无需收集间接用户活动审计数据。

您可以使用此功能仅审核顶级用户生成的事件，而无需为间接SQL语句创建审计记录的开销。顶级语句是用户直接发出的SQL语句。这些陈述对于安全性和合规性都很重要。通常，从PL / SQL过程或函数中运行的SQL语句不被视为顶级，因此它们可能与审计目的不太相关。

从Oracle数据库帐户中删除的密码

现在，大多数Oracle数据库提供的仅使用模式的帐户都会删除其密码，以防止用户对这些帐户进行身份验证。

此增强功能不会影响示例模式。示例模式仍然使用其默认密码进行安装。

管理员仍然可以将密码分配给默认的仅架构帐户。Oracle建议稍后将模式更改回仅模式帐户。

此功能的好处是管理员不再需要定期轮换这些Oracle数据库提供的模式的密码。此功能还可以降低使用默认密码侵入这些帐户的攻击者的安全风险。

LOB定位器的基于签名的安全性

从此版本开始，您可以为大对象（LOB）定位器配置基于签名的安全性。

此功能增强了Oracle数据库LOB的安全性，尤其是在分布式环境中使用LOB数据类型（CLOB和BLOB）的实例时。

LOB签名密钥可以位于多租户PDB中，也可以位于独立的非多租户数据库中。您可以通过执行ALTER DATABASE DICTIONARY ENCRYPT CREDENTIALS SQL语句来启用LOB签名密钥凭证的加密 ;否则，凭证以模糊格式存储。如果选择以加密格式存储LOB签名密钥，则数据库或PDB必须具有打开的TDE密钥库。
