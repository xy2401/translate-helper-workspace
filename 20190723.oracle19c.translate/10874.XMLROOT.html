XMLROOT

注意：

不推荐使用XMLROOT函数。它仍然支持向后兼容性。但是，Oracle建议您使用带有版本号的SQL / XML函数XMLSERIALIZE 。有关XMLSERIALIZE函数的更多信息，请参见Oracle XML DB Developer's Guide 。

句法


插图描述xmlroot.eps

目的

XMLROOT允许您通过在现有XML值的XML根信息（prolog）中提供版本和独立属性来创建新的XML值。如果value_expr已经有prolog，则数据库返回错误。如果输入为null，则该函数返回null。

返回的值采用以下形式：

<？xml version =“ version ”[STANDALONE =“{yes | no}”]？>

  • 第一个value_expr指定要为其提供prolog信息的XML值。

  • 在VERSION子句中， value_expr必须解析为表示有效XML版本的字符串。如果为VERSION指定NO VALUE ，则版本默认为1.0。

  • 如果省略可选的STANDALONE子句，或者如果使用NO VALUE指定它，则函数返回的值中不存在独立属性。

例子

以下语句使用DUAL表来说明XMLROOT的语法：

SELECT XMLROOT（XMLType（'<poid> 143598 </ poid>'），VERSION'1.0'，STANDALONE YES）AS“XMLROOT”FROM DUAL; XMLROOT ------------------------------------------------- ------------------------------- <？xml version =“1.0”standalone =“yes”？> <poid> 143598 </ poid>
