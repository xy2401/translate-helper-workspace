9.4管理策略集

无论您是第一次配置Oracle数据库QoS管理系统，还是想要创建新的策略集，都可以使用“创建策略集”向导来创建策略集。

您可以使用“编辑策略集”向导来修改现有的策略集。

注意：

如果您决定创建新的策略集，则必须重新创建所有现有的性能策略和用户添加的性能类。在单击“策略集”向导的最后一页中的“ 提交策略集”之前，不会保存对存储的策略集所做的更改。

  • 编辑策略集
    您可以修改系统的策略集。
  • 将服务器池添加到策略集
    服务器池由集群或数据库管理员使用Oracle Enterprise Manager云控制或服务器控制（SRVCTL）创建。
  • 修改服务器池设置
    要修改服务器池设置，请为策略集配置服务器池指令覆盖。
  • 将数据库服务添加到策略集
    通过创建其他服务，您可以通过限制将新服务用于特定应用程序或用户来更精细地监控工作负载性能。
  • 更新策略集以包含新数据库
    如果将新数据库添加到群集，则可以配置Oracle数据库QoS管理以管理或监视此数据库上的工作负载。

9.4.1编辑策略集

您可以修改系统的策略集。

要修改现有策略集，请执行以下步骤：

  1. 从群集目标菜单中，选择“ 管理” ，然后选择“服务质量管理” ，再选择“ 编辑策略集” 。

    “策略集编辑器”向导已启动。

  2. 在“策略集编辑器”向导的第一页上，您可以查看当前的服务器池设置。

    插图描述edit_policy-general.png

    此页面允许您指定由Oracle数据库QoS管理管理的服务器池。如果未选中服务器池的“管理”框，则Oracle数据库QoS管理将不显示或管理使用该服务器池的任何服务器，数据库或工作负载。“详细信息”列显示任何配置违规详细信息以及要使管理服务器池所采取的更正操作。

    完成后，单击“ 下一步” 。

    将出现“ 策略集编辑器：性能类”页面。
  3. 第二页使您可以为系统创建，编辑，重命名或删除Performance Classes。您可以创建新的Performance Class，也可以使用Edit Performance Class按钮修改现有Performance Class的分类器。

    插图描述edit_policy-perfomance_classes.png

    您可以使用“ 全部展开”链接显示每个性能类的分类器，或者展开单个“性能类”条目以仅显示该性能类的分类器。

    选择要编辑的性能类，然后单击相应的操作按钮（ 编辑性能类 ， 重命名性能类或删除性能类 ）。如果要为此策略集创建性能类，请单击“ 添加性能类” 。

    完成后，单击“ 下一步” 。

  4. 在创建或修改Performance Classes之后，下一步是设置评估分类器的顺序。

    插图描述edit_policy-classifier_ordering.png

    此步骤非常重要，因为分类器确定将工作请求放入哪个Performance Class。选择一个Performance Class并使用分类器右侧的箭头键在列表中上下移动条目。

    当工作请求进入系统时，将根据分类器评估工作请求。计算结果为TRUE的第一个表达式确定工作请求属于哪个Performance Class。您应该将更细粒度的分类器放在顶部，将更通用的分类器放在列表的底部。

  5. 为Performance Classes设置顺序后，下一步是编辑性能策略并设置性能目标。

    说明edit_policy-performance_policies.png的说明

    在此页面上，您可以：

    • 创建新的性能策略

    • 编辑，复制或删除现有的性能策略

    • 指定QoS管理可以自动实现哪些资源分配方法。

    注意：

    无法修改DefaultPolicy性能策略，但可以复制策略并将其用作新性能策略的基础。

  6. 配置性能策略后，必须选择在提交策略集后立即生效的策略。

    插图描述edit_policy-set_policy.png

    选择首选的性能策略，然后单击“ 设置策略”。 完成后，单击“ 下一步” 。

  7. 在将策略集提交到Oracle数据库QoS管理系统之前，系统会要求您查看策略集配置。
    • 如果更改符合您的预期，请单击“ 提交策略集” 。
    • 如果必须修改任何显示的设置，请单击“上一步”或“ 取消”按钮。

    单击“ 提交策略集”后 ，将返回到仪表板。

9.4.3修改服务器池设置

要修改服务器池设置，请为策略集配置服务器池指令覆盖。

配置服务器池指令覆盖时，将使用您为服务器池指定的新参数，而不是在创建服务器池时指定的参数。

注意：

您可以更改列出的服务器池的最小和最大设置，但无法使用此界面创建其他服务器池。提交后，此处所做的任何更改都会在关联的性能策略处于活动状态时更改Oracle Enterprise Manager云控制的“管理服务器池”部分中设置的当前服务器池属性。

  • 您可以使用Oracle Enterprise Manager Cloud Control或SRVCTL的“管理服务器池”页面手动修改服务器池参数。
    启用Oracle数据库QoS管理后，不应使用此方法更改服务器池配置。如果同时使用服务器池指令覆盖并手动更改服务器池配置，则为当前策略集指定的服务器池指令覆盖将覆盖手动设置，并可能导致混淆。

9.4.4将数据库服务添加到策略集

通过创建其他服务，您可以通过限制将新服务用于特定应用程序或用户来更精细地监控工作负载性能。

要将数据库服务添加到策略集，必须为每个新数据库服务创建或修改性能类。
  1. 数据库管理员使用SRVCTL或Oracle Enterprise Manager Cloud Control创建由Oracle Clusterware管理的数据库服务。您可以使用Enterprise Manager Cloud Control的数据库目标页面上的“可用性”菜单为Oracle RAC数据库创建服务。
  2. QoS管理员将一个或多个性能类添加到策略集，或修改现有的性能类以包含新的数据库服务

注意：

数据库服务在Oracle Database QoS Management中显示为db_name : service_name ，因此服务名称在数据库中必须是唯一的。

9.4.5更新策略集以包含新数据库

如果将新数据库添加到群集，则可以配置Oracle数据库QoS管理以管理或监视此数据库上的工作负载。

  1. 数据库管理员首先请求服务器从群集或系统管理员托管群集数据库。
  2. 群集或数据库管理员使用指定的最小节点数创建新服务器池。
    群集或数据库管理员还可以决定授予对现有服务器池的访问权限，而不是创建新的服务器池。
  3. 数据库管理员在分配的服务器池上创建新数据库。
    数据库管理员必须确保正确配置新数据库，以便Oracle Database QoS Management进行管理。
  4. 数据库管理员使用SRVCTL或Oracle Enterprise Manager Cloud Control创建由Oracle Clusterware为新数据库管理的数据库服务。
  5. 数据库管理员从Cloud Control的数据库目标页面启用Oracle数据库QoS管理的数据库。
  6. 使用Cloud Control中的“编辑策略集”链接，QoS管理员将新服务器池添加到策略集，将一个或多个性能类添加到策略集，或修改现有性能类以包括新数据库服务。
  7. 成功提交新策略集后，Oracle数据库QoS管理将监视和管理新数据库。
  8. 您可以使用qosctl qosadmin -updatepolicyset选项qosctl qosadmin -updatepolicyset更新策略集。在以静默方式添加新数据库和/或服务时，或通过运行脚本时，这非常有用。
