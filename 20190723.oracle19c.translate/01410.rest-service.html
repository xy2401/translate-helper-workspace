16.1使用ORDS配置REST服务

Oracle跟踪文件分析器包括REST支持，允许通过HTTPS进行调用和查询。

句法

为了促进此REST支持，Oracle REST数据服务（ORDS）包含在安装中。

tfactl rest [-status | -start | -stop | -upgrade | -uninstall] [ -dir ] [ -port ] [ -user ] [-debug [ -level ]]

注意：

您只能以root用户身份运行REST命令。

参数

表16-1 REST命令参数

参数 描述

-状态

打印当前状态。

-开始

如果尚未运行，则启动Oracle跟踪文件分析器REST服务。

-停

如果正在运行，则停止Oracle跟踪文件分析器REST服务。

-升级

检查是否应升级配置的ORDS API。

如果ORDS API需要升级，则停止ORDS，升级API，然后重新启动ORDS。

-卸载

删除Oracle跟踪文件分析器REST配置。

-dir

用于存储Oracle跟踪文件分析器REST配置详细信息的目录。

默认为用户主目录。

-港口

要运行ORDS的端口。

默认为9090。

-用户

用户启动ORDS为。

默认为GRID所有者。

-debug

启用调试。

-水平
要使用的调试级别，可用级别为：

  • 1 - 致命

  • 2 - 错误

  • 3 - 警告

  • 4 - INFO（默认）

  • 5 - 调试

  • 6 - 追踪
ORDS运行后，您可以使用以下API使用表单请求调用REST：
https：//开头的主机：端口/ ORDS / API
例如：
https：//开头的主机：端口/ ORDS / tfactl /打印/状态

REST身份验证

Oracle跟踪文件分析器REST使用第一方基于cookie的身份验证（基本身份验证）。

Oracle跟踪文件分析器REST应用程序能够使用Web应用程序使用的相同cookie会话对RESTful API进行身份验证和授权。第一方应用程序可以完全访问RESTful API。

在启动期间，Oracle Trace File Analyzer会提示您输入tfaadmin和tfarest用户的密码。

  • 使用tfarest用户进行REST调用

  • 使用tfaadmin进行REST调用并管理REST服务，例如，更改日志记录级别

＃./tfactl rest -start使用ORDS配置TFA REST服务：这可能需要几分钟时间。请耐心等待。将依赖项添加到ORDS将用户添加到ORDS：输入用户tfaadmin的密码：确认用户tfaadmin的密码：输入用户tfarest的密码：确认用户tfarest的密码：启动TFA REST服务成功启动TFA REST服务[PID：32650] URL：https：// myserver：9090 / ords / tfactl / print / status
使用以下URL从浏览器访问Web服务：
https：// host_name ：9090 / ords / tfactl / print / status

您将看到401消息，其中包含登录链接。单击该链接，使用刚刚创建的tfarest凭据登录，然后您将转到REST输出。

或者，您也可以在curl命令中指定凭据。 

#curl -k --user tfarest：mypassword https：// myserver：9090 / ords / tfactl / print / status [{“status”：“CheckOK”，“hostname”：“myserver”，“pid”：2430，“ port“：5000，”version“：”18.2.0.0.0“，”buildId“：”18200020180501035221“，”inventoryStatus“：”COMPLETE“}]

16.2使用Apache Tomcat配置REST服务

Oracle Trace File Analyzer安装包括一个Web应用程序资源（WAR）文件，用于通过Apache Tomcat启用REST服务。

要使用Apache Tomcat启用REST服务：
  1. 将位于TFA_HOME/jlib/tfa.war的WAR文件部署到Tomcat服务器。
  2. 更改tfaadmin用户密码。
    curl -k --user tfaadmin：tfaadmin -X POST“https：// host / tfa / tfactl / user / update”'{“password”：“some_new_password”}'
  3. 更改tfarest用户密码。
    curl -k --user tfarest：tfarest -X POST“https：// host / tfa / tfactl / user / update”'{“password”：“some_new_password”}'
  4. 将Tomcat用户添加到Oracle Trace File Analyzer访问列表。
    tfactl access add -user tomcat_user
