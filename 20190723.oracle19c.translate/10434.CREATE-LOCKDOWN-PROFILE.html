创建锁定概况

目的

使用CREATE LOCKDOWN PROFILE语句创建PDB锁定配置文件。您可以在多租户容器数据库（CDB）中使用PDB锁定配置文件来限制PDB中的用户操作。

创建PDB锁定配置文件后，可以使用ALTER LOCKDOWN PROFILE语句向配置文件添加限制。您可以限制与某些数据库功能，选项和SQL语句关联的用户操作。

将锁定配置文件分配给PDB时，该PDB中的用户无法执行对配置文件禁用的操作。要分配锁定配置文件，请为PDB_LOCKDOWN初始化参数的值设置其名称。您可以为单个PDB或CDB或应用程序容器中的所有PDB分配锁定配置文件，如下所示：

  • 如果在连接到CDB根目录时设置PDB_LOCKDOWN ，则锁定配置文件将应用于CDB中的所有PDB。它不适用于CDB根目录。

  • 如果在连接到应用程序根目录时设置PDB_LOCKDOWN ，则锁定配置文件将应用于应用程序根目录和应用程序容器中的所有PDB。

  • 如果在连接到特定PDB时设置PDB_LOCKDOWN ，则锁定配置文件将应用于该PDB并覆盖CDB或应用程序容器（如果存在）的锁定配置文件。

也可以看看：

  • ALTER LOCKDOWN PROFILE和DROP LOCKDOWN PROFILE

  • 有关PDB锁定配置文件的更多信息，请参见“Oracle数据库安全指南

先决条件

  • 必须从CDB或Application Root发出CREATE LOCKDOWN PROFILE语句。

  • 您必须在发出语句的容器中具有CREATE LOCKDOWN PROFILE系统特权。

  • PDB锁定配置文件名称在发出语句的容器中必须是唯一的。

语义

PROFILE_NAME

您可以使用指定的名称创建新的PDB锁定配置文件。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。锁定配置文件可以从静态或动态基本配置文件派生。

static_base_profile

使用此选项可创建具有基本配置文件的新锁定配置文件。在配置文件创建时生效的基本配置文件的规则将复制到新的锁定配置文件。创建锁定配置文件后对基本配置文件的更改将不适用于锁定配置文件。

dynamic_base_profile

使用此选项可创建新的锁定配置文件，该配置文件将随基本配置文件的更改而更改。新的锁定配置文件将继承基本配置文件的DISABLE规则以及对基本配置文件的后续更改。基本配置文件的规则优先于可能明确添加到锁定配置文件的规则的任何冲突。例如， OPTION_VALUE基本轮廓的条款优先于OPTION_VALUE动态基础轮廓的子句。

例

以下语句使用动态基本配置文件PRIVATE_DBAAS创建PDB锁定配置文件hr_prof ：

CREATE LOCKDOWN PROFILE hr_prof包括PRIVATE_DBAAS;
