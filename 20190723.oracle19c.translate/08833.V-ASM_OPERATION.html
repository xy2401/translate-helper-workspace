7.72 V $ ASM_OPERATION

在Oracle自动存储管理（Oracle ASM）实例中， V$ASM_OPERATION为在Oracle ASM实例中执行的每个活动的Oracle ASM长时间运行操作显示一行。

柱 数据类型 描述

GROUP_NUMBER

数

磁盘组编号（主键）。这是V$ASM_DISKGROUP视图的外键。

操作

CHAR（5）

操作类型：

  • REBAL - 该组的重新平衡待定。磁盘组正在重新平衡。

  • REMIRROR - Remirror正在等待此组。

  • SCRUB - Scrubbing正在等待这一组。

从Oracle Database 12 c开始 ，新查询应使用PASS列而不是此列。

通过

VARCHAR2（9）

操作类型：

  • REBALANCE - 此组的重新平衡待定。磁盘组正在重新平衡。

  • RESYNC - 正在进行重新同步操作，以使一个或多个Oracle ASM磁盘联机

  • COMPACT - Oracle ASM将用户数据更加紧密地联系在一起，从而通过减少搜索距离来提高性能

  • RESILVER - 启用WriteBack FlashCache时，此值显示在Oracle Exadata环境中

  • SCRUBBING - 磁盘组正在擦除。

从Oracle Database 12 c开始 ，新查询应使用此列而不是OPERATION列。

州

VARCHAR2（4）

经营状况：

  • WAIT - 没有为该组运行的操作

  • EST - 根据重新平衡要完成的工作量计算估算值

  • RUN - 为组RUN操作

  • REAP - 正在减少运营

  • DONE - 显示已完成的通行证

  • ERRS - 错误导致操作停止

在工作执行期间，估计值在后台并行计算。因此，从EST到RUN状态的转换可能非常快。

功率

数

由ASM_POWER_LIMIT初始化参数或命令语法指定的操作请求的电源。或者，由清理SQL语法的power选项指定的操作请求的电源。

实际

数

分配给操作的电源

至今

数

操作每分钟移动的分配单元数。或者，到目前为止已清理的分配单元的数量。

EST_WORK

数

估计必须由操作移动的分配单元数。或者，必须由清理操作擦除的估计分配单元数。

EST_RATE

数

操作每分钟移动的估计分配单元数

EST_MINUTES

数

预计剩余操作所需的估计时间（以分钟为单位）

错误代码

VARCHAR2（44）

Oracle外部错误代码;如果没有错误则为NULL

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

对于此视图，该值始终为0 。
