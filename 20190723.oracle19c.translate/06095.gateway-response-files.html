介绍

通常，Oracle Universal Installer以交互模式运行，这意味着它会提示您在图形用户界面（GUI）屏幕中提供信息。或者，您可以在非交互模式下运行Oracle Universal Installer。非交互模式也称为静默模式或静默安装。

在以下情况下，您可能希望使用非交互模式在Microsoft Windows（64位）上安装Oracle Database Gateway：

  • 您需要以无人值守的方式将Oracle组件部署到多个节点。您可以从操作系统调度程序或站点通常使用的其他作业子系统安排非交互式安装模式。

  • 不打算与用户进行任何交互。

  • 无法以交互模式运行Oracle Universal Installer的图形工具。（Oracle Universal Installer始终在Microsoft Windows上可用，但在UNIX系统上不可用。）

本节介绍有关如何使用响应文件以非交互模式运行Oracle Universal Installer的以下主题：

使用响应文件以非交互模式安装Oracle组件

要使用非交互模式，请使用响应文件运行Oracle Universal Installer。响应文件是一个文本文件，其中包含Oracle Universal Installer在安装过程中使用的变量和值。Oracle提供了一组您可以自定义的示例响应文件，或者您可以通过记录安装选择来创建自己的响应文件。

有关响应文件格式的更多信息，请参见适用于Windows和UNIX的Oracle Universal Installer和OPatch用户指南

自定义示例响应文件

Oracle为每种产品和安装类型以及每种配置工具提供响应文件模板。Oracle Gateway的响应文件tg.rsp和netca.rsp位于介质的响应目录中。

注意：

如果将软件复制到硬盘，则响应文件位于Disk1\response目录中。

要复制和修改响应文件：

  1. 将必要的响应文件从安装介质上的\ Response目录复制到硬盘驱动器。
  2. 从“开始”菜单中选择“ 程序” ，然后选择“ Oracle - HOME_NAME” ，“ Oracle安装产品” ，“ Universal Installer概念指南” 。

    适用于Windows和UNIX的Oracle Universal Installer和OPatch用户指南以HTML格式显示。

  3. 按照响应文件和Oracle Universal Installer以及Windows和UNIX的OPatch用户指南中的说明，使用任何文本文件编辑器修改响应文件。
  4. 按照运行Oracle Universal Installer和指定响应文件中的说明运行响应文件 。

创建新的响应文件

在交互模式下运行Oracle Universal Installer时，可以将安装选择记录到响应文件中。您可以通过在记录模式下运行Oracle Universal Installer来完成此操作。完成“ 摘要”页面后，Oracle Universal Installer会立即生成响应文件，因此您无需实际安装网关即可创建响应文件。

如果要在非交互式安装期间使用“记录”模式，则Oracle Universal Installer会将原始源响应文件中指定的变量值记录到新的响应文件中。

注意：

您不能使用记录模式来创建基于的响应文件基本安装类型。

要创建新的响应文件：

  1. 确保要在其上创建响应文件的计算机满足相应网关安装章节中描述的要求。
  2. 在命令提示符下，使用cd命令切换到包含Oracle Universal Installer setup.exe可执行文件的目录。

    在安装介质上， setup.exe位于磁盘1上。如果要从现有网关安装运行Oracle Universal Installer，则可以在ORACLE_BASE \ ORACLE_HOME \oui\bin找到setup.exe 。

  3. 输入以下命令：
    setup -record -destinationFile response_file_name

    将response_file_name替换为新响应文件的完整路径。例如：

    setup -record -destinationFile C：\ response_files \ install_oracle11g
  4. Oracle Universal Installer启动后，输入安装设置，该设置将记录到响应文件中。
  5. 出现“摘要”页面时，请执行以下操作之一：

    • 单击“ 安装”继续安装。

    • 如果只想创建响应文件但不继续安装，请单击“ 取消” 。安装将停止，但您输入的设置将记录到响应文件中。

    稍后，Oracle Universal Installer将使用您在命令行中指定的路径和文件名保存新的响应文件。

  6. 如有必要，请对要运行它的计算机的响应文件进行任何特定于环境的更改。
  7. 按照“ 运行Oracle Universal Installer”和“指定响应文件”部分中的说明运行响应文件 。

运行Oracle Universal Installer并指定响应文件

您在命令行运行Oracle Universal Installer，指定响应文件。Oracle Universal Installer可执行文件setup.exe提供了几个选项。有关这些选项的完整集的帮助信息，请使用-help选项运行setup.exe ，例如：

C：\ ORACLE_BASE \ ORACLE_HOME \ oui \ bin> setup.exe -help

要运行Oracle Universal Installer并指定响应文件：

  1. 启动命令提示符。
  2. 转到安装Oracle Universal Installer的目录。
  3. 从命令行，使用正确的响应文件运行Oracle Universal Installer。例如：

    C:\ORACLE_BASE\ORACLE_HOME \oui\bin> setup.exe [-silent] [-nowelcome] [-nowait] -responseFile filename
