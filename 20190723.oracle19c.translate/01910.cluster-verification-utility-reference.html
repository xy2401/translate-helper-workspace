CVU概述

CVU可以在操作阶段或阶段验证主要群集组件。

组件可以是基本组件，例如可用磁盘空间，也可以是复杂的组件，例如检查Oracle Clusterware的完整性。例如，CVU可以跨Oracle Clusterware层验证多个Oracle Clusterware子组件。此外，CVU还可以检查磁盘空间，内存，进程和其他重要的群集组件。例如，一个阶段可以是数据库安装，CVU可以验证您的系统是否满足Oracle Real Application Clusters（Oracle RAC）安装的标准。其他阶段包括初始硬件设置和通过完全可操作的群集设置建立系统要求。

CVU安装要求

CVU安装要求是：

  • 在运行CVU的节点上，CVU软件至少有30 MB可用空间

  • 每个节点上至少有25 MB可用空间的工作目录。工作目录的默认位置是Linux和UNIX系统上的/tmp ，以及Windows系统上TEMP环境变量中指定的值。您可以通过设置CV_DESTLOC环境变量来指定其他位置。

    使用CVU时，该实用程序会尝试将任何所需信息复制到CVU工作目录。它检查每个节点上是否存在工作目录。如果找不到，则尝试创建一个。确保CVU工作目录存在于群集中的所有节点上，或者在运行CVU以创建该目录的用户的每个节点上建立适当的权限。

CVU使用信息

：CVU包括两个脚本runcluvfy.sh （ runcluvfy.bat在Windows上），您在安装Oracle软件，并在使用前cluvfy （ cluvfy.bat在Windows上），坐落在Grid_home目录和Grid_home /bin分别。runcluvfy.sh脚本包含临时变量定义，使其能够在安装Oracle Grid Infrastructure或Oracle数据库之前运行。安装Oracle Grid Infrastructure后，使用cluvfy命令检查先决条件并执行其他系统准备情况检查。

注意：

Oracle Universal Installer运行cluvfy以检查Oracle软件安装期间的所有先决条件。

在安装Oracle软件之前， runcluvfy.sh从您希望Grid home所在的目录运行runcluvfy.sh ，如下所示：

cd /u01/app/12.2.0/grid ./runcluvfy.sh options

在前面的示例中， options变量表示您选择的CVU命令选项。例如：

$ ./runcluvfy.sh comp nodereach -n node1，node2 -verbose

CVU命令选项包括：

  • –html ：以HTML格式显示CVU输出

  • -verbose ：在输出的详细摘要部分中显示有关所有CVU检查的说明，包括失败和已通过

  • –file file_location ：将输出保存为HTML文件到特定位置

输入CVU命令时，它会提供测试摘要。在预安装期间，Oracle建议您使用CVU命令使用-verbose参数来获取详细输出。-verbose参数生成单个检查的详细输出。在适用的情况下，它以表格布局显示每个节点的结果。

使用cluvfy命令运行CVU命令行工具。使用cluvfy不会对群集环境或已安装的软件产生负面影响。您可以随时运行cluvfy命令，甚至可以在安装Oracle Clusterware之前运行。事实上，CVU旨在帮助您在硬件和操作系统运行时立即为您提供帮助。如果在节点上运行需要Oracle Clusterware的命令，则如果该节点上尚未安装Oracle Clusterware，则CVU会报告错误。

与CVU命令一起使用的节点列表应该是逗号分隔的没有域的主机名列表。CVU在处理节点列表时忽略域。如果CVU命令条目在删除域信息后具有重复的节点条目，则CVU将删除重复的节点条目。

对于网络连接验证，如果未在CVU命令行上指定接口，CVU将发现所有可用的网络接口。对于存储可访问性验证，如果未在命令行上指定特定存储标识，CVU将发现所有受支持存储类型的共享存储。如果可用，CVU还会发现Oracle Clusterware主目录。

CVU输出

CVU输出包含五个不同的部分，包括：

  • 标题 ：包含有关检查阶段或组件执行情况的信息的单行。

  • 配置 ：CVU评估是否可以在所有节点上执行操作。如果您指定的节点已关闭或不满足必要的先决条件（例如无SSH设置），则此处将显示有关这些节点的错误消息。此部分可能并不总是包含在输出中。当正在执行的检查可以在操作中涉及的所有节点上运行时，此部分将不会显示在输出中。

  • 进度消息 ：CVU在循环执行各种检查时显示此部分中的进度消息，这有助于您确定CVU是否挂起。

  • 详细摘要 ：默认情况下，CVU仅在此部分中显示失败的任务或子任务。如果选择-verbose选项，则CVU将显示所有任务和子任务的详细信息。

  • 执行摘要 ：最后，CVU显示整个检查过程的简明摘要，类似于以下示例：

    $ cluvfy stage -pre crsinst -n sales65 CVU操作执行：stage -pre crsinst日期：2016年10月13日下午9:43:39 CVU主页：/ ade / scott_abc4 / oracle /用户：scott $ cluvfy comp baseline -collect all  - n sales65 CVU操作执行：基线日期：2016年10月13日下午9:48:19操作系统：Linux2.6.39-400.211.1.el6uek.x86_64
CVU配置文件

您可以使用CVU配置文件来定义执行CVU的特定输入。配置文件的路径是Grid_home /cv/admin/cvu_config （或Windows平台上的\clusterware\stage\cvu\cv\admin Staging_area \clusterware\stage\cvu\cv\admin ）。您可以使用文本编辑器修改此文件。CVU的输入以键输入的形式定义。修改CVU配置文件时，必须遵循以下规则：

  • 键条目的语法name=value

  • 每个键条目和分配给键的值仅定义一个属性

  • 以数字符号（ # ）开头的行是注释行并被忽略

  • 不遵循语法name=value的行将被忽略

以下是CVU支持的密钥列表：

  • CV_NODE_ALL ：如果设置，则指定未安装Oracle Clusterware时应拾取的节点列表。默认情况下，此条目已注释掉。

  • CV_ORACLE_RELEASE ：如果置位，它指定了特定的Oracle版本（ 10.1 ， 10.2 ， 11.1 ， 11.2 ， 12.1 ，或12.2 ）的量，验证必须执行。如果设置，则不必在适用的任何位置使用-r release选项。默认值为12.2 。

  • CV_RAW_CHECK_ENABLED ：如果设置为TRUE ，则启用对Linux和UNIX系统上共享磁盘的可访问性的检查。此共享磁盘可访问性检查要求您在所有节点上安装cvuqdisk RPM软件包管理器（rpm）。默认情况下，此键设置为TRUE并启用共享磁盘检查。

  • CV_ASSUME_DISTID ：此属性用于CVU无法检测或支持特定平台或分发的情况。Oracle建议您不要更改此属性，因为这可能会导致CVU无法正常运行。

  • CV_XCHK_FOR_SSH_ENABLED ：如果设置为TRUE ，则启用X-Windows检查以验证用户与ssh等效性。默认情况下，此条目已注释掉，并且禁用了X-Windows检查。

  • ORACLE_SRVM_REMOTECOPY ：如果设置，则指定scp或rcp命令的位置以覆盖CVU默认值。默认情况下，此条目已注释掉，CVU使用/usr/bin/scp和/usr/sbin/rcp 。

  • ORACLE_SRVM_REMOTESHELL ：如果设置，则指定ssh命令的位置以覆盖CVU默认值。默认情况下，此条目已注释掉，该工具使用/usr/sbin/ssh 。

  • CV_ASSUME_CL_VERSION ：默认情况下，命令行解析器使用crs activeversion显示命令行语法用法和语法验证。使用此属性传递crs activeversion以外的版本crs activeversion进行命令行语法显示和验证。默认情况下，此条目已注释掉。

如果CVU未找到配置文件中定义的密钥条目，则CVU将搜索与密钥名称匹配的环境变量。如果设置了环境变量，则CVU将使用其值，否则CVU将使用该实体的默认值。

特权和安全

由于root用户缺乏用户等效性，因此大多数CVU命令无法以root身份运行以执行任何远程节点操作，但以下情况除外：

  • cluvfy comp dhcp

  • cluvfy comp dns

但是，使用权限委派，您可以指定-method参数并选择两种方法之一（ sudo或root ）以启用检查并运行需要在远程节点上执行root权限的fixup脚本。系统将提示您输入密码，但密码将在CVU命令运行时动态使用，而不是存储在存储设备上。

在fixup脚本的上下文中指定-method参数是有利的。如果选择权限委派，则可以从本地节点一次运行所有修复脚本。如果不选择权限委派，则必须以root身份登录每个相关节点并运行fixup脚本。

使用CVU帮助

cluvfy命令具有上下文相关帮助，可根据您输入的命令行参数显示其用法。例如，如果输入cluvfy ，则CVU会显示描述阶段和组件语法的高级通用用法文本。以下是上下文帮助命令的列表：

  • cluvfy -help ：CVU显示详细的CVU命令信息。

  • cluvfy -version ：CVU显示Oracle Clusterware的版本。

  • cluvfy comp -list ：CVU显示可以检查的组件列表，以及该实用程序如何检查每个组件的简要说明。

  • cluvfy comp -help ：CVU显示每个有效组件检查的详细语法。

  • cluvfy stage -list ：CVU显示有效阶段列表。

  • cluvfy stage -help ：CVU显示每个有效阶段检查的详细语法。

您还可以将-help选项与任何CVU命令一起使用。例如， cluvfy stage -pre nodeadd -help返回该特定命令的详细信息。

如果输入无效的CVU命令，则CVU会显示该命令的正确用法。例如，如果键入cluvfy stage -pre dbinst ，则CVU会显示dbinst阶段的precheck命令的正确语法。输入cluvfy -help命令以查看详细的CVU命令信息。

使用CVU确定安装先决条件是否完整

您可以使用CVU确定完成安装的系统先决条件。如果要在具有预先存在的Oracle软件安装的系统上安装Oracle Database 12 c软件，请使用此选项。在使用此选项时，请注意以下事项：

  • 您必须将CVU作为计划用于运行安装的用户帐户运行。您不能以root身份运行CVU，并且将CVU作为执行安装的用户以外的其他用户运行不能确保用户和组配置的准确性，以进行安装或其他配置检查。

  • 在完成群集范围状态检查之前，必须为所有群集节点配置SSH。您可以使用安装程序完成SSH配置，也可以在群集中的所有节点之间自行完成SSH配置。您还可以使用CVU生成修订脚本以配置SSH连接。

  • CVU可以通过查找必须完成的预安装步骤来帮助您，但它无法执行预安装任务。

使用以下语法确定完成的安装前步骤以及必须执行的安装前步骤;使用-fixup标志运行该命令会生成一个fixup脚本，以根据需要完成内核配置任务：

$ ./runcluvfy.sh stage -pre crsinst -fixup -n node_list

在前面的语法示例中，将node_list变量替换为集群中节点的名称，以逗号分隔。在Windows上，必须将逗号分隔的节点列表用双引号（ "" ）括起来。

例如，对于具有mountpoint /mnt/dvdrom/以及节点node1 ， node2和node3的集群，请输入以下命令：

$ cd / mnt / dvdrom / $ ./runcluvfy.sh stage -pre crsinst -fixup -n node1，node2，node3

查看CVU报告，并根据需要完成其他步骤。

将CVU与Oracle数据库10g版本1或2一起使用

您可以使用Oracle Database 12 c介质中包含的CVU来检查Oracle Database 10g第 1版（10.1）及更高版本安装的系统要求。要使用CVU检查Oracle Clusterware安装，请将命令-r release_code标志附加到标准CVU系统检查命令。

例如，要在介质挂载点为/mnt/dvdrom且集群节点为node1 ， node2和node3的系统上安装Oracle Clusterware版本10之前执行验证检查，请输入以下命令：

$ cd / mnt / dvdrom $ ./runcluvfy.sh stage -pre crsinst -n node1，node2，node3 -r 10.2

注意：

如果您未指定要检查的发行版本，则CVU会检查12 c版本1（12.1）要求。

进入和退出标准

验证阶段时，CVU使用进入和退出标准。每个阶段都有入口标准，用于定义在启动该阶段之前要执行的一组特定验证任务。除非您满足该阶段的Oracle Clusterware先决条件，否则此检查会阻止您开始安装Oracle Clusterware等阶段。

阶段的退出条件定义了在完成阶段后必须执行的另一组验证任务。检查后确保该阶段的活动已完成。检查后确定阶段特定问题，然后再传播到后续阶段。

详细模式和UNKNOWN输出

虽然默认情况下CVU仅以非详细模式报告测试摘要，您可以使用-verbose参数获取详细输出。-verbose参数生成单个检查的详细输出，并在适用的情况下显示表格布局中每个节点的结果。

如果cluvfy命令响应对于特定节点， UNKNOWN ，则这是因为CVU无法确定检查是通过还是失败。原因可能是可达性的丧失或用户与该节点等效的失败。原因也可能是CVU执行检查时该节点上发生的任何系统问题。

以下是UNKNOWN响应的可能原因列表：

  • 节点已关闭

  • Grid_home /bin或Oracle home目录中缺少CVU所需的可执行文件

  • 运行CVU的用户帐户没有在节点上运行公共操作系统可执行文件的权限

  • 节点缺少操作系统修补程序或所需的程序包

  • 节点已超过最大进程数或最大打开文件数，或者IPC段存在问题，例如共享内存或信号量

CVU节点列表快捷方式

要为CVU提供群集中所有节点的列表，请输入-n all 。CVU尝试按以下顺序获取节点列表：

  1. 如果供应商集群件可用，则CVU使用lsnodes实用程序从供应商集群件中选择所有已配置的节点。

  2. 如果安装了Oracle Clusterware，则CVU使用olsnodes实用程序从Oracle Clusterware中选择所有已配置的节点。

  3. 如果既未安装供应商集群件，也未安装Oracle集群件，则CVU会在配置文件中搜索CV_NODE_ALL键的值。

  4. 如果未安装供应商集群件和Oracle集群件，并且配置文件中不存在名为CV_NODE_ALL密钥，则CVU将搜索CV_NODE_ALL环境变量的值。如果您尚未设置此变量，则CVU会报告错误。

要提供部分节点列表，可以设置环境变量并在CVU命令中使用它。例如，在Linux或UNIX系统上，您可以输入：

setenv MYNODES node1，node3，node5 cluvfy comp nodecon -n $ MYNODES [-verbose]

启用跟踪

除非禁用跟踪，否则CVU会生成跟踪文件。您可以通过将SRVM_TRACE环境变量设置为false或FALSE来禁用跟踪。例如，在tcsh ，诸如setenv SRVM_TRACE FALSE类的条目禁用跟踪。

默认情况下，CVU跟踪文件在ORACLE_BASE/crsdata/ host_name /cvu目录中创建。Oracle数据库自动轮换日志文件，最近创建的日志文件名为cvutrace.log.0 。如果需要，您应该删除不需要的日志文件或将其存档以回收磁盘位置。

Oracle Clusterware存储CVU在ORACLE_BASE/crsdata/ host_name /cvu/cvutrc目录中定期运行时生成的日志文件。

要为跟踪文件使用非默认位置，请将CV_TRACELOC环境变量设置为所需跟踪目录的绝对路径。

群集验证实用程序的已知问题

本节介绍了Cluster Verification Utility（CVU）的以下已知限制：

  • 群集验证实用程序支持的数据库版本

  • Linux Shared Storage Accessibility（ssa）检查报告限制

  • Red Hat Linux上的共享磁盘发现

群集验证实用程序支持的数据库版本

当前的CVU版本仅支持Oracle数据库10g或更高版本，Oracle RAC和Oracle Clusterware; CVU不向后兼容。CVU无法在Oracle Database 10g之前检查或验证Oracle数据库产品的版本。

Linux Shared Storage Accessibility（ssa）检查报告限制

cluvfy的当前版本在Linux上有关于共享存储可访问性检查的以下限制。

  • 支持OCFS2（版本1.2.1或更高版本）。

  • 对于NAS上的共享检查， cluvfy命令要求您具有指定路径的写入权限。如果运行cluvfy命令的用户没有写入权限，则cluvfy路径报告为not shared 。

Red Hat Linux上的共享磁盘发现

要对Red Hat Linux 5.0（或更高版本）和Oracle Linux 5.0（或更高版本）以及SUSE Linux Enterprise Server上的SCSI磁盘执行发现和共享存储可访问性检查，CVU需要CVUQDISK程序包。如果您尝试使用CVU并且Oracle RAC环境中的所有节点上未安装CVUQDISK软件包，则CVU会响应错误。

执行以下过程以安装CVUQDISK程序包：

  1. 以root用户身份登录。

  2. 将程序包cvuqdisk-1.0.9-1.rpm （或更高版本）复制到本地目录。您可以在Oracle Clusterware安装介质中最顶层目录的rpm子目录中找到此rpm。例如，您可以在/ mountpoint /clusterware/rpm/目录中找到cvuqdisk-1.0.9-1.rpm ，其中mountpoint是目录所在磁盘的安装点。 

    #cp / mount_point /clusterware/rpm/cvuqdisk-1.0.9-1.rpm / u01 / oradba

  3. 将CVUQDISK_GRP环境变量设置为应拥有CVUQDISK程序包二进制文件的操作系统组。如果未设置CVUQDISK_GRP ，则默认情况下， oinstall组是所有者的组。 

    #set CVUQDISK_GRP = oinstall

  4. 通过运行命令rpm -q cvuqdisk确定是否安装了以前版本的CVUQDISK软件包。如果找到以前版本的CVUQDISK软件包，则通过运行命令rpm -e cvuqdisk previous_version删除它们，其中previous_version是先前CVUQDISK版本的标识符，如以下示例所示：

    #rpm -q cvuqdisk cvuqdisk-1.0.2-1＃rpm -e cvuqdisk-1.0.2-1

  5. 通过运行命令rpm -iv cvuqdisk-1.0.9-1.rpm安装最新的CVUQDISK软件包。 

    #cd / u01 / oradba＃rpm -iv cvuqdisk-1.0.9-1.rpm
