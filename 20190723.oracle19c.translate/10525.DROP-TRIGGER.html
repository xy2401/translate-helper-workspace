DROP TRIGGER

目的

触发器使用PL / SQL定义。有关创建，更改和删除触发器的完整信息，请参阅Oracle数据库PL / SQL语言参考 。

使用DROP TRIGGER语句从数据库中删除数据库触发器。

也可以看看：

创建触发器和更改触发器

先决条件

触发器必须位于您自己的架构中，或者您必须具有DROP ANY TRIGGER系统特权。要在另一个用户的架构中删除DATABASE上的触发器，您还必须具有ADMINISTER DATABASE TRIGGER系统特权。

句法

drop_trigger :: =


说明drop_trigger.eps的说明

语义

模式

指定包含触发器的架构。如果省略schema ，则Oracle数据库假定触发器位于您自己的模式中。

触发

指定要删除的触发器的名称。Oracle数据库将其从数据库中删除，不会再次触发它。

例子

删除触发器：示例

以下语句删除模式hr的salary_check触发器：

DROP TRIGGER hr.salary_check;
