XDK for Java的XML标准

这里的主题包括DOM，XSLT，JAXB和管道定义语言的XDK标准。

适用于Java的XDK的DOM标准

DOM API包括对DOM Level 3 Validation和DOM Level 3 Load and Save的候选推荐的支持。

注意：

在Oracle数据库10g第2版中，XDK for Java实现了文档对象模型（DOM）级别3.0加载和保存和验证规范的候选推荐版本。Oracle计划生成一个版本或补丁集，其中包括DOM Level 3.0加载和保存和验证建议的实现。为了符合建议，Oracle可能会被迫进行不向后兼容的更改。在此期间，Oracle不保证与DOM加载和保存以及验证实现的向后兼容性。在更新XDK for Java以符合建议后，用于向后兼容的标准Oracle策略将应用于Oracle DOM加载和保存以及验证实现。

DOM Level 3加载和保存

DOM Level 3加载和保存模块使软件开发人员能够在符合要求的产品中加载和保存可扩展标记语言（XML）内容。

LSParser不支持charset-overrides-xml-encoding配置参数。LSParser不支持这些配置参数的可选设置：

  • disallow-doctype（true）

  • ignore-unknown-character-denormalizations（false）

  • 名称空间（false）

  • supported-media-types-only（true）

LSSerializer不支持discard-default-content配置参数。LSSerializer不支持这些配置参数的可选设置：

  • 规范形式（真实）

  • format-pretty-print（true）

  • ignore-unknown-character-denormalizations（false）

  • normalize-characters（true）

DOM 3.0验证

DOM 3.0验证允许用户从XML模式中检索元数据定义，查询DOM操作的有效性，并根据XML模式验证DOM文档或子树。由于验证基于模式，因此在使用这些函数之前，必须将文档类型定义（DTD）转换为模式。

适用于Java的XDK XSLT标准版

XDK XSLT处理器支持XSLT 2.0，XPath 2.0和共享XPath / XQuery数据模型的当前建议。

Oracle XML Development Kit（XDK）支持XSLT 2.0（W3C Recommendation，2007年1月23日）作为基本XSLT处理器 ，具有以下限制：支持xsl:key和xsl:sort行为处于XSLT 1.0级别。

适用于Java的XDK JAXB标准

描述了Java绑定（JAXB）规范的XDK实现不支持的功能。

Java Architecture for XML Binding（JAXB）规范的XDK实现不支持以下功能：

  • Javadoc一代

  • XML Schema组件any和替换组

XDK for Java的管道定义语言标准

介绍了XML Pipeline处理器和W3C Note之间的差异。

两者的不同之处如下：

  • 解析器进程DOMParserProcess和SAXParserProcess包含在XML管道中（第1节）。

  • 仅检查最终目标输出以查看它是否与可用管道输入一起是最新的。XML Pipeline处理器不确定每个进程的中间输出是否是最新的（第2.2节）。

  • 对于select属性，双引号（“...”）之间的任何内容都被视为字符串文字。

  • 如果多一个进程生成相同的信息集，则XML Pipeline处理器会抛出错误（第2.4.2.3节）。

  • 不支持<document>元素（第2.4.2.8节）。
