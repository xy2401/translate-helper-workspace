比较条件

比较条件将一种表达与另一种表达进这种比较的结果可以是TRUE ， FALSE或UNKNOWN 。

比较条件不支持大对象（LOB）。但是，您可以使用PL / SQL程序来比较CLOB数据。

比较数值表达式时，Oracle使用数字优先级来确定条件是否比较NUMBER ， BINARY_FLOAT或BINARY_DOUBLE值。有关数字优先级的信息，请参阅数字优先级。

在比较字符表达式时，Oracle使用数据类型比较规则中描述的规则 。这些规则定义了在比较之前如何对齐表达式的字符集，使用二进制或语言比较（整理），空白填充比较语义的使用，以及对归类键施加的限制所产生的限制，包括报告错误ORA-12742: unable to create the collation key 。

如果非标量类型的两个对象具有相同的命名类型并且它们的元素之间存在一对一的对应关系，则它们是可比较的。此外，用户定义的对象类型的嵌套表，即使它们的元素具有可比性，也必须在其上定义MAP方法，以便在相等或IN条件下使用。

op ANY

一些

“op”必须是=，！=，>，<，<=或> =。

op ANY使用条件op将左侧的值与列表中的每个值或查询返回的每个值进行比较，无论哪个在右侧指定。

如果这些比较中的任何一个返回TRUE，则op ANY返回TRUE 。

如果所有这些比较都返回FALSE ，或者右侧的子查询不返回任何行，则op ANY返回FALSE 。否则，返回值为UNKNOWN 。

op ANY和op SOME是同义词。 

SELECT * FROM employees WHERE salary = ANY（SELECT employees FROM employees WHERE department_id = 30）ORDER BY employee_id;

op ALL

“op”必须是=，！=，>，<，<=或> =。

op ALL使用条件op将左侧的值与列表中的每个值或子查询返回的每个值进行比较，无论哪个在右侧指定。

如果这些比较中的任何一个返回FALSE ，则op ANY返回FALSE 。

如果所有这些比较都返回TRUE ，或者右侧的子查询没有返回任何行，则op ALL返回TRUE 。否则，返回值为UNKNOWN 。

 
SELECT * FROM employees WHERE salary> = ALL（1400,3000）ORDER BY employee_id;
