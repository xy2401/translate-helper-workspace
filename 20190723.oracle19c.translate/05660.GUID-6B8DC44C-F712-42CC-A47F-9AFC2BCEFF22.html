Oracle Data Pump在一个操作中加载分区表数据

Oracle Data Pump可以选择将表的所有分区中的表数据作为一个操作导入，而不是为每个分区单独操作。

GROUP_PARTITION_TABLE_DATA是Import DATA_OPTIONS命令行参数的新值，它通过将表的所有分区中的表数据作为一个操作导入来更改Oracle Data Pump的默认行为。当您不希望将每个表分区作为单独的操作导入的默认导入行为时，此参数很有用。例如，当表可能在加载表时移动到不同的分区时，导入会选择默认值。当导入操作未创建表时，也会使用默认值。

Oracle数据库的自动升级

AutoUpgrade允许您使用单个命令和单个配置文件在命令行升级一个或多个Oracle数据库实例。

AutoUpgrade运行预升级任务，在需要时执行自动修复，处理数据库升级，并通过完成升级后任务完成升级。它包括自动重试和回退，为未来时间点安排升级的选项，以及根据需要设置，更改或删除初始化参数的功能。

AutoUpgrade显着减少了与数据库升级相关的手动操作。它使单个DBA可以同时升级多个数据库。它甚至可以处理由熟练的DBA无法主动管理的数据库的例行升级。通过减少升级所需的工作量，并在升级过程中自动实施建议的实践，AutoUpgrade使数据库升级过程更容易完成，并降低风险。

客户端路由的共置标记

COLOCATION_TAG参数是一个字母数字字符串，可以与TNS连接字符串的CONNECT_DATA参数一起使用。设置COLOCATION_TAG参数时，它会尝试将具有相同COLOCATION_TAG的客户端路由到同一数据库实例。

在同一实例上对会话进行共置可以帮助减少实例间通信，从而提高在同一实例中执行的工作负载的性能。

Grid Infrastructure Management存储库的可选安装

从Oracle Grid Infrastructure 19c开始，Grid Infrastructure Management Repository（GIMR）对于Oracle Standalone Cluster的新安装是可选的。Oracle Domain Services Clusters仍然需要安装GIMR作为服务组件。

GIMR中包含的数据是基于应用机器学习的预防性诊断的基础，可以帮助提高Oracle Real Application Clusters（Oracle RAC）数据库的可用性。对GIMR进行可选安装可以实现更灵活的存储空间管理和更快的部署，尤其是在安装测试和开发系统期间。

支持OCR和表决磁盘的直接文件放置

从Oracle Grid Infrastructure 19c开始，Oracle Standalone Clusters将取消对共享文件系统上直接OCR和Voting磁盘文件放置的支持。对于Oracle域服务群集，要求将Oracle自动存储管理（Oracle ASM）中的OCR和表决文件置于托管在共享文件系统上并用作Oracle ASM磁盘的文件之上。

在Oracle Grid Infrastructure 12c第2版（12.2）中，Oracle宣布它将不再支持直接在共享文件系统上为Oracle Grid Infrastructure放置Oracle Cluster Registry（OCR）和投票文件。这种支持现在已经废除。从Oracle Grid Infrastructure 19c（19.3）开始，使用Oracle Standalone Clusters，您可以再次将OCR和Voting磁盘文件直接放在共享文件系统上。

动态服务后备选项

对于使用“首选”和“可用”设置放置的动态数据库服务，用户现在可以指定此服务在可用时应回退到“首选”实例，并且一旦故障转移到具有最后一个的可用实例下降的首选实例。

动态服务后备选项允许更多地控制放置动态数据库服务，并确保尽可能长时间地在首选实例上提供给定服务。
