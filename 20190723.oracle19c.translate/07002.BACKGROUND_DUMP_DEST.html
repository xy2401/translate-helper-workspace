1.31 BACKGROUND_DUMP_DEST

BACKGROUND_DUMP_DEST指定在Oracle操作期间写入后台进程（LGWR，DBW n等）的调试跟踪文件的路径名（目录或磁盘）。

属性 描述

参数类型

串

句法

BACKGROUND_DUMP_DEST = { pathname | directory }

默认值

依赖于操作系统

可修改

改变系统

可在PDB中修改

没有

价值范围

任何有效的本地路径，目录或磁盘

基本

没有

注意：

在Oracle Database 12 c第1版（12.1.0.1）中不推荐使用BACKGROUND_DUMP_DEST参数。

BACKGROUND_DUMP_DEST指定的目录中的警报日志记录重要的数据库事件和消息。此处记录任何影响数据库实例或全局数据库的内容。警报日志是普通文本文件。它的文件名与操作系统有关。对于支持多个实例的平台，它采用alert_ sid .log形式，其中sid是系统标识符。此文件增长缓慢，但没有限制，因此您可能希望定期删除它。即使数据库正在运行，您也可以删除该文件。

注意：

Oracle Database 11g第1版（11.1）中引入的可诊断性基础结构忽略此参数，该基础结构将跟踪和核心文件放在由DIAGNOSTIC_DEST初始化参数控制的位置。
