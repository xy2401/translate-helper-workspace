仅默认显示的最重要检查

默认情况下，Oracle ORAchk和Oracle EXAchk仅显示报告输出中最关键的检查。

关键检查是Oracle认为具有最严重潜在影响的检查。

所有其他检查仍在报告中运行并可用。您可以通过选择具有以下状态的显示检查控件下的相应选项来查看它们。

要仅运行关键检查，请使用适当的配置文件：
./orachk -profile oratier1 ./exachk -profile exatier1

REST接口

Oracle ORAchk和Oracle EXAchk现在包括完整的REST支持，允许通过HTTPS进行调用和查询。

为了促进REST支持，Oracle REST数据服务（ORDS）包含在安装中。

要启用REST，请通过运行./exachk -ordssetup命令启动ORDS，然后使用-ords选项启动守护程序。
./exachk -d start -ords
通过访问URL启动完整运行状况检查：
https：// host ：7080 / ords / tfaml / orachk / start_client
要运行特定的配置文件
https：// host ：7080 / ords / tfaml / orachk / profile / profile1 ， profile2
要运行特定检查：
https：// host ：7080 / ords / tfaml / orachk / check / check_id ， check_id

任何请求都会返回一个作业ID，您可以使用该ID查询状态或下载结果。

要查询状态：
https：// host ：7080 / ords / tfaml / orachk / status / job_id
要下载结果：
https：// host ：7080 / ords / tfaml / orachk / download / job_id

没有无密码SSH的远程节点连接

如果您不希望使用无密码SSH连接到远程节点，Oracle ORAchk和Oracle EXAchk可以自动生成远程节点的私钥文件。

用于生成私钥的过程如下：

  1. 提示远程节点密码。

  2. 登录远程节点，在远程节点上生成私钥和公钥对。

  3. 将公钥的内容复制到远程节点的.ssh/authorized_keys文件中，然后从远程节点删除公钥。

  4. 将远程节点的私钥复制到本地节点，并使用身份文件进行后续连接。

或者，您可以自己提供私钥文件。

EBS健康检查的弃用

从18.3.0版本开始，不推荐使用Oracle ORAchk和Oracle EXAchk检查EBS功能。

这意味着不再增强针对EBS功能运行运行状况检查，但在Oracle ORAchk和Oracle EXAchk 19.3.0发行版（大约一年）之前，它仍然受支持。

在Oracle ORAchk和Oracle EXAchk 19.3.0版本中，将不支持EBS检查功能，这意味着Oracle将不再修复与EBS检查相关的Oracle ORAchk和Oracle EXAchk中的错误，并且将完全删除EBS检查功能。

使用EBS分析程序运行所有EBS运行状况检查，该分析程序可从My Oracle Support Note 1545562.1获得。

确保您只运行最新检查

从此版本开始，Oracle ORAchk和Oracle EXAchk将仅提供不到一年的检查。

健康检查信息随着时间的推移不断变化。什么是适用的，几个月后可能不适用。这就是Oracle ORAchk和Oracle EXAchk一直有季度发布周期的原因。

较旧的健康检查包含过时信息的风险更大。因此，从此版本开始，Oracle ORAchk和Oracle EXAchk将仅提供不到一年的检查。

如果Oracle ORAchk或Oracle EXAchk超过六个月，那么该工具（根据以前的版本）会显示一条警告消息，建议您升级并提供下载新版本的功能。如果Oracle ORAchk或Oracle EXAchk超过一年，那么该工具将不会运行任何检查。

通过缓存发现数据提高性能

从此版本开始，Oracle ORAchk和Oracle EXAchk将缓存发现数据。

Oracle Exadata系统的内部测试表明，通过将关键检查的执行时间减半，发现数据的缓存可以提高运行时性能。

此版本中包含以下命令选项：

  • -discovery ：缓存发现数据。
  • -checkdiscovery ：可选择验证发现。
  • -usediscovery ：使用发现数据。
  • -refreshdiscovery ：刷新发现数据。
  • -rmdiscovery ：删除发现数据。

此版本中包含以下命令选项：

  • -querypatch all ：查找已安装的所有补丁。
  • -querypatch {bug_num} ：查找是否安装了特定补丁。
  • -rollbackpatch {bug_num} ：回滚特定的补丁。

此版本添加了检查特定的超时值。

如果您发现某些检查仍然超时，则可以使用以下命令覆盖单个检查超时值：export RAT_TIMEOUT={value_in_seconds}

多个自动SRDC的合并

如果Oracle跟踪文件分析器检测到重大事件，则它会等待五分钟检查结果发生的任何其他事件，并收集所有这些事件。

如果在五分钟内在同一数据库上发生多个与SRDC相关的事件，则Oracle跟踪文件分析器会为所有这些事件执行一个合并的SRDC集合。

例如，如果发生ORA-00600事件，30秒后发生ORA-04031事件，则Oracle跟踪文件分析器会执行一个SRDC，收集解决这两个错误所需的所有内容。

SRDC的合并适用于以下内容：

  • ORA-00600
  • ORA-07445
  • ORA-04030
  • ORA-04031

在未来的版本中，特定SRDC的自动收集将扩展到其他事件。
