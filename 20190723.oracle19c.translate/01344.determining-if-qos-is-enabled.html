9.1确定是否启用了Oracle数据库QoS管理

您可以使用分层方法启用Oracle数据库QoS管理。

级别2和级别3取决于之前配置的级别。

  1. 在群集上：对所有操作都是必需的

  2. 在服务器池上：对于该服务器池中的节点上的任何操作都是必需的

  3. 在数据库上：任何操作都需要

如果同一Oracle RAC集群中有多个数据库，则可能并非所有数据库都由Oracle数据库QoS管理进行管理。如果为多租户数据库启用Oracle数据库QoS管理，则会为该容器数据库（CDB）的所有可插拔数据库（PDB）启用Oracle数据库QoS管理。

  • 检查数据库的已启用状态
  • 检查群集的已启用状态
    使用Oracle Enterprise Manager云控制，您可以确定是否为群集启用了Oracle数据库QoS管理。

9.1.1检查数据库的启用状态

要确定您的数据库是否由Oracle Database QoS Management管理，请执行以下步骤：

  1. 登录到Cloud Control并选择要检查的数据库目标。
  2. 从目标菜单中，选择Cluster Database ，然后选择Target Information 。
  3. 在“目标信息”窗口的底部，如果为此数据库启用了Oracle数据库QoS管理，则应启用QoS状态的值。

父主题： 确定是否已启用Oracle数据库QoS管理

9.1.2检查集群的启用状态

使用Oracle Enterprise Manager云控制，您可以确定是否为群集启用了Oracle数据库QoS管理。

要确定群集是否由Oracle Database QoS Management管理，请执行以下步骤：

  1. 登录到Cloud Control并选择要检查的群集目标。
  2. 从目标菜单中，选择“ 管理” ，然后选择“服务质量管理” ，然后选择“ 仪表板” 。
  3. 如果出现提示，请以QoSAdmin用户身份登录。
  4. 在仪表板的“常规”部分中，检查“QoS状态”的值。如果状态值为Enabled，则Oracle Database QoS Management将管理集群。
