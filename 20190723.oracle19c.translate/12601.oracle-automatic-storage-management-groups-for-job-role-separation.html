用于作业角色分离的Oracle自动存储管理组

Oracle Grid Infrastructure操作系统组为其成员提供访问和管理Oracle自动存储管理的特定于任务的系统权限。

  • 适用于Oracle ASM管理的OSASM组（通常为asmadmin ）

    将此组创建为单独的组，以分隔Oracle ASM和Oracle数据库管理员的管理权限组。该组的成员被授予SYSASM系统权限以管理Oracle ASM。在Oracle文档中，其成员被授予权限的操作系统组称为OSASM组，在代码示例中，如果有专门为创建此权限而创建的组，则称为asmadmin 。

    Oracle ASM可以支持多个数据库。如果系统上有多个数据库，并且使用多个OSDBA组以便为每个数据库提供单独的SYSDBA权限，则应创建一个组，其成员被授予OSASM / SYSASM管理权限，并创建一个网格基础结构用户（ grid）不拥有数据库安装，因此您可以将Oracle Grid Infrastructure SYSASM管理权限与数据库管理权限组分开。

    OSASM组的成员可以使用SQL使用操作系统身份验证以SYSASM身份连接到Oracle ASM实例。SYSASM权限允许装入和卸载磁盘组以及其他存储管理任务。SYSASM权限不对RDBMS实例提供访问权限。

    如果未将单独的组指定为OSASM组，但确实为数据库管理定义了OSDBA组，则默认情况下，您定义的OSDBA组也定义为OSASM组。

  • 适用于Oracle ASM的OSOPER组（通常为asmoper ）

    这是一个可选组。如果您希望一组独立的操作系统用户拥有一组有限的Oracle实例管理权限（SYSOPER for ASM权限），包括启动和停止Oracle ASM实例，请创建此组。默认情况下，OSASM组的成员还具有SYSOPER for ASM权限授予的所有权限。
