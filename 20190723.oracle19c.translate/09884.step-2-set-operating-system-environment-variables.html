第2步：设置操作系统环境变量

根据您的平台，您可能必须在启动SQL * Plus之前设置环境变量，或者至少验证它们是否已正确设置。

例如，在大多数平台上，您必须设置环境变量ORACLE_SID和ORACLE_HOME 。此外，必须将PATH环境变量配置为包含ORACLE_HOME / bin目录。某些平台可能需要其他环境变量：

  • 在UNIX和Linux上，通过输入操作系统命令来设置环境变量。

  • 在Windows上，Oracle Universal Installer（OUI）会自动将值分配给Windows注册表中的ORACLE_HOME和ORACLE_SID 。

如果您在安装时未创建数据库，则OUI不会在注册表中设置ORACLE_SID ;在以后创建数据库之后，必须从命令窗口设置ORACLE_SID环境变量。

UNIX和Linux安装附带两个脚本oraenv和coraenv ，可用于轻松设置环境变量。有关更多信息，请参阅适用于Linux和基于UNIX的操作系统的Oracle数据库管理员参考 。

对于所有平台，在具有不同Oracle主目录的实例之间切换时，必须更改ORACLE_HOME环境变量。如果多个实例共享同一个Oracle主目录，则在切换实例时必须仅更改ORACLE_SID 。

示例4-1在UNIX中设置环境变量（C Shell）

setenv ORACLE_SID orcl setenv ORACLE_HOME /u01/app/oracle/product/18.0.0/db_1 setenv LD_LIBRARY_PATH $ ORACLE_HOME / lib：/ usr / lib：/ usr / dt / lib：/ usr / openwin / lib：/ usr / ccs / LIB

示例4-2在UNIX中设置环境变量（Bash Shell）

export ORACLE_SID = orcl export ORACLE_HOME = / u01 / app / oracle / product / 18.0.0 / db_1 export LD_LIBRARY_PATH = $ ORACLE_HOME / lib：/ usr / lib：/ usr / dt / lib：/ usr / openwin / lib：/ usr / CCS / lib目录

示例4-3在Windows中设置环境变量

SET ORACLE_SID = orawin2

示例4-3假定在注册表中设置了ORACLE_HOME和ORACLE_SID ，但是您要覆盖ORACLE_SID的注册表值以连接到其他实例。

在Windows上，您在命令提示符窗口中设置的环境变量值将覆盖注册表中的值。
