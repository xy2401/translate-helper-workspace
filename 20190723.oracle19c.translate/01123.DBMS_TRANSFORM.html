173.1.1 CREATE_TRANSFORMATION过程

此过程创建一个转换，将源类型的对象映射到目标类型的对象。转换表达式可以是SQL表达式或PL / SQL函数。它必须返回目标类型的对象。

句法

DBMS_TRANSFORM.CREATE_TRANSFORMATION（schema VARCHAR2（30），name VARCHAR2（30），from_schema VARCHAR2（30），from_type VARCHAR2（30），to_schema VARCHAR2（30），to_type VARCHAR2（30），transformation VARCHAR2（4000））;

参数

表173-2 CREATE_TRANSFORMATION过程参数

参数 描述

模式

指定转换的架构。

名称

指定转换的名称。

from_schema

指定源类型的架构。

from_type

指定源类型。

to_schema

指定目标类型架构。

输入

指定目标类型。

转型

指定转换表达式，返回目标类型的对象。表达式必须是返回目标类型的对象或目标类型的构造函数表达式的函数。您可以选择不指定转换表达式，而是使用MODIFY_TRANSFORMATION指定目标类型属性的MODIFY_TRANSFORMATION 。

使用说明

  • 转换表达式必须是SQL表达式或PL / SQL函数，返回目标类型的指定属性的类型。

  • 要创建，修改或删除转换，必须在DBMS_TRANSFORM上授予用户执行权限。用户还必须对用户定义的类型具有执行权限，这些类型是转换的源和目标类型。此外，用户还必须对转换函数中使用的任何PLSQL函数具有执行权限。

  • 转换无法写入数据库状态（执行DML）或提交或回滚当前事务。

  • 转换必须是源类型为输入类型的SQL函数，返回目标类型的对象。它也可以是目标类型的SQL表达式，指的是源类型。对源类型的所有引用必须是source.user_data形式。

  • 源类型和目标类型都必须是非标量数据库类型。空转换表达式映射到空目标对象。

要在enqueue和dequeue时使用转换，调用该操作的登录用户必须对转换使用的PLSQL函数具有执行权限。对于传播，队列的拥有模式必须具有这些特权。

173.1.3 MODIFY_TRANSFORMATION过程

此过程修改给定转换的转换表达式。

句法

DBMS_TRANSFORM.MODIFY_TRANSFORMATION（schema VARCHAR2（30），name VARCHAR2（30），attribute_number INTEGER，transformation VARCHAR2（4000））;

参数

表173-4 MODIFY_TRANSFORMATION过程参数

参数 描述

模式

指定转换的架构。

名称

指定转换的名称。

ATTRIBUTE_NUMBER

为其指定新转换表达式的目标类型的属性。将新转换指定为目标类型的单个表达式时，请指定值0。

转型

转换表达式必须是SQL表达式或PL / SQL函数，返回目标类型的指定属性的类型。如果attribute_number为0，则表达式必须是PL / SQL函数，返回目标类型的对象或目标类型的构造函数表达式。

使用说明

  • 如果新变换是目标类型的单个表达式，则可以使用attribute_number为0指定它。还可以为目标类型的每个属性指定新变换。

  • 您可以使用此过程将转换定义为目标类型的每个属性的单独表达式。对于大型变换，这种表示可能更具可读性，并允许对变换应用细粒度控制。如果转换表达式未指定目标类型的某些属性，则在应用转换时将它们计算为null。
