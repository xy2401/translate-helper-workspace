DROP TABLE

目的

使用DROP TABLE语句将表或对象表移动到回收站，或者完全从数据库中删除表及其所有数据。

注意：

除非您指定PURGE子句，否则DROP TABLE语句不会导致将空间释放回表空间以供其他对象使用，并且该空间将继续计入用户的空间配额。

对于外部表，此语句仅删除数据库中的表元数据。它对驻留在数据库之外的实际数据没有影响。

删除属于群集的表时，表将移动到回收站。但是，如果随后删除群集，则会从回收站中清除该表，并且无法再使用FLASHBACK TABLE操作恢复该TABLE 。

删除表会使依赖对象无效并删除表上的对象权限。如果要重新创建表，则必须重新授予表的对象权限，重新创建表的索引，完整性约束和触发器，并重新指定其存储参数。截断没有这些影响。因此，使用TRUNCATE语句删除行比删除和重新创建表更有效。

先决条件

该表必须位于您自己的架构中，或者您必须具有DROP ANY TABLE系统特权。

只有在没有绑定任何会话时，才能在临时表上执行DDL操作（例如ALTER TABLE ， DROP TABLE ， CREATE INDEX ）。通过对表执行INSERT操作，会话将绑定到临时表。会话通过发出TRUNCATE语句或在会话终止时变为未绑定到临时表，或者对于特定于事务的临时表，通过发出COMMIT或ROLLBACK语句。

您可以使用现有的DROP TABLE命令删除专用临时表。删除私有临时表不会提交现有事务。这适用于特定于事务的特定于临时表和特定于会话的私有临时表。请注意，删除的私有临时表不会进入RECYCLEBIN 。

语义

模式

指定包含该表的架构。如果省略schema ，则Oracle数据库假定该表位于您自己的模式中。

表

指定要删除的表的名称。Oracle数据库自动执行以下操作：

  • 表中的所有行都将被删除。

  • 删除所有表索引和域索引，以及在表上定义的任何触发器，无论是谁创建它们或其架构包含它们。如果对table进行了分区，则还会删除任何相应的本地索引分区。

  • 嵌套表和的LOB的所有存储表table被丢弃。

  • 删除范围，哈希或列表分区表时，数据库将删除所有表分区。如果删除复合分区表，则也会删除所有分区和子分区。

  • 当您使用PURGE关键字删除分区表时，该语句将作为一系列子事务执行，每个子事务都会删除分区或子分区及其元数据的子集。将drop操作划分为子事务会优化内部系统资源消耗（例如，库高速缓存）的处理，尤其是对于非常大的分区表的删除。第一个子事务提交后，表将标记为UNUSABLE 。如果任何子事务失败，那么表上允许的唯一操作是另一个DROP TABLE ...PURGE声明。假设您已更正上一个操作遇到的任何错误，此类语句将从先前DROP TABLE语句失败的位置恢复工作。

    通过查询*_TABLES ， *_PART_TABLES ， *_ALL_TABLES或*_OBJECT_TABLES数据字典视图的status列，可以根据需要列出通过此类删除操作标记为UNUSABLE的表。

    也可以看看：

    有关删除分区表的详细信息，请参见“Oracle数据库VLDB和分区指南” 。

  • 对于索引组织表，将删除在索引组织表上定义的任何映射表。

  • 对于域索引，将调用适当的drop例程。有关这些例程的更多信息，请参阅Oracle数据库数据盒开发人员指南 。

  • 如果任何统计类型与表关联，则数据库将统计类型与FORCE子句解除关联，并删除使用统计类型收集的任何用户定义的统计信息。

    也可以看看：

    关于统计类型关联的更多信息，关联统计和DISASSOCIATE STATISTICS

  • 如果表不是集群的一部分，则数据库将分配给表及其索引的所有数据块返回到包含表及其索引的表空间。

    要删除集群及其所有表，请使用带有INCLUDING TABLES子句的DROP CLUSTER语句，以避免单独删除每个表。请参阅DROP CLUSTER 。

  • 如果表是视图的基表，物化视图的容器或主表，或者在存储过程，函数或包中引用它，则数据库使这些依赖对象无效但不删除它们。除非重新创建表或删除并重新创建对象以使它们不再依赖于表，否则不能使用这些对象。

    如果选择重新创建表，则它必须包含最初用于定义实例化视图的子查询和存储过程，函数或包中引用的所有列所选择的所有列。以前授予视图，存储过程，函数或包的对象权限的任何用户都不需要重新授予这些权限。

    如果表是实例化视图的主表，则仍可以查询实例化视图，但除非重新创建表以使其包含由实例化视图的定义查询选择的所有列，否则无法刷新实例化视图。

    如果表具有物化视图日志，则数据库将删除此日志以及与该表关联的任何其他直接路径INSERT刷新信息。

删除表的限制

  • 您不能直接删除嵌套表的存储表。相反，您必须使用ALTER TABLE删除嵌套表列...DROP COLUMN子句。

  • 您不能删除引用分区表的父表。您必须首先删除所有引用分区的子表。

  • 您不能删除使用闪回数据存档进行历史跟踪的表。您必须首先禁用表格使用闪回数据存档。

级联约束

指定CASCADE CONSTRAINTS以删除引用已删除表中的主键和唯一键的所有引用完整性约束。如果省略此子句，并且存在此类参照完整性约束，则数据库将返回错误，并且不会删除该表。

清除

如果要删除表并在一个步骤中释放与其关联的空间，请指定PURGE 。如果指定PURGE ，则数据库不会将表及其依赖对象放入回收站。

注意：

您不能使用PURGE子句回滚DROP TABLE语句，如果使用PURGE子句删除了表，也无法恢复该表。

使用此子句相当于首先删除表，然后从回收站中清除它。此子句允许您在过程中保存一个步骤。如果要防止敏感材料出现在回收站中，它还可以提供增强的安全性。
