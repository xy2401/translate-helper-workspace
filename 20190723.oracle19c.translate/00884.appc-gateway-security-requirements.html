12安全要求

网关架构涉及多个系统，数据库服务器和通信设施，每个系统都具有不同的安全功能和限制。要有效地规划和实施安全方案，除了了解安装安全性要求之外，还必须了解这些功能和限制。

阅读以下主题以了解适用于APPC的Oracle数据库网关的功能和限制：

  • 安全要求概述

  • 验证应用程序登录

  • 定义和控制数据库链接

  • 使用SNA安全验证

  • TCP / IP安全性

安全要求概述

之前实施您的安全方案，您必须了解您环境中的现有安全要求和期望。由于您要启用对不同系统上不同数据库的应用程序访问，因此您可能需要合并多个安全文化。当您将多个不同的系统连接到一个操作整体时，具有最严格安全要求的系统通常会指示其他系统能够做什么和不能做什么。

网关安全包括两个主要问题：

  • 允许访问特定网关实例和OLTP的用户和应用程序

  • 用户和应用程序能够执行的OLTP事务

您可以在网关体系结构中的多个点控制访问。主要选项将在以下部分中讨论。每个OLTP都提供对远程事务程序访问的控制，具有基于用户ID的本机授权机制。 OLTP的产品文档中描述了这些功能。安全要求中的信息包括网关设施如何确定对特定OLTP连接有效的用户ID。

当网关涉及RPC请求时，安全机制对网关遇到的每个系统组件都有效。遇到的第一个系统组件是应用程序工具或第三代语言（3GL）程序。遇到的最后一个系统组件是OLTP。

以下每个部分都标识了该组件中可用的组件和安全性处理类型。每个部分都提供了主要功能和参数的摘要。有关Oracle和非Oracle产品的非网关组件的详细信息，请参阅产品特定的文档。

验证应用程序登录

应用程序必须连接到使用网关之前的Oracle数据库。您使用的登录身份验证类型决定了生成的Oracle用户ID，并且可能会影响网关操作。

可以使用以下类型的身份验证：

  • Oracle身份验证

    使用Oracle身份验证，每个Oracle用户标识都有一个Oracle已知的关联密码。当应用程序连接到服务器时，它会提供用户标识和密码。Oracle确认存在用户标识，并且密码与存储在数据库中的密码相匹配。

  • 操作系统认证

    通过操作系统身份验证，底层服务器操作系统负责身份验证。使用操作系统身份验证访问使用IDENTIFIED EXTERNALLY属性（而不是密码）创建的Oracle用户标识。要使用此类用户标识登录，应用程序会为用户标识提供正斜杠（ / ），并且不提供密码。

    要执行操作系统身份验证，服务器会确定请求者操作系统用户ID，可选地为其添加固定前缀，并将结果用作Oracle用户ID。服务器确认用户ID存在且IDENTIFIED EXTERNALLY已IDENTIFIED EXTERNALLY ，但未进行密码检查。基本假设是用户在登录操作系统时进行身份验证。

    操作系统身份验证并非在所有平台上都可用，并且在某些平台上不可用 Oracle Net（客户端 - 服务器）和多线程服务器配置。请参阅特定于平台的Oracle数据库文档和“ Oracle数据库网络服务管理员指南”，以确定配置中此功能的可用性。

有关验证应用程序登录的详细信息，请参阅“ Oracle数据库管理员指南” 。

定义和控制数据库链接

以下部分讨论了使用TCP / IP或SNA通信协议的网关用户的数据库链接。

链接可访问性

数据库链接的第一个控制点是，如果给定用户可以访问它。一个公共数据库链接可以由任何用户ID使用。私有数据库链接只能由创建它的用户使用。数据库链接可用性取决于其打开网关会话的能力。 Oracle数据库不区分使用类型（例如只读与更新或写入）或可以访问哪些远程对象。这些区别是访问的OLTP的责任。

链接和CONNECT子句

该 CONNECT子句是另一个与安全相关的属性数据库链接。您可以使用CONNECT子句指定显式用户标识和密码，该标识可能与Oracle用户标识和密码不同。首次打开数据库链接连接时，此CONNECT用户标识和密码组合将发送到网关。根据特定于网关的选项，网关可能会将该用户标识和密码发送到要验证的OLTP。

如果使用Oracle身份验证创建了没有CONNECT子句的数据库链接，则在打开连接时会将用户的Oracle用户ID和密码发送到网关。如果用户登录到 Oracle数据库具有操作系统身份验证，然后网关不会从Oracle数据库接收用户ID或密码。经过系统身份验证的Oracle用户不可能使用没有CONNECT子句定义的网关数据库链接。但是，如果您的OLTP提供基于网关的u ser ID映射工具用户连接的LU名称，如果同一网关实例上的所有用户都可以使用相同的OLTP用户ID，则可以进行此类连接。

有关数据库链接的更多信息，请参阅“ Oracle数据库管理员指南” 。

使用SNA安全验证

使用SNA安全验证中的信息仅适用于采用SNA通信协议的网关用户的安全需求。当网关收到启动远程事务程序的RPC请求时，网关尝试启动与OLTP的APPC对话。在对话开始之前，会话必须在平台的逻辑单元之间开始（ LU）和OLTP LU。

APPC SNA通信包（SNAP-IX for Solaris Operating System（SPARC）和SNA Server for AIX-based systems）提供对您的平台的支持。

SNA及其各种访问方法实现，包括VTAM和适用于您的平台的SNA通信包，在会话启动时提供安全验证，允许每个LU对其合作伙伴进行身份验证。在网关和OLTP应用程序开始对话并处理对话级安全数据之前，此验证完全由网络软件执行。如果使用会话级安全性，则必须在其中建立正确的密码信息您的平台的SNA配置文件以及要访问的OLTP中的类似参数结构。有关此主题的详细信息，请参阅相应的通信软件产品文档。

指定SNA会话安全性

该网关初始化文件的PGA_SECURITY_TYPE参数允许您指定确定安全行为的三个选项之一 LU6.2与OLTP分配的会话。这些选项是SNA LU6.2体系结构的一部分，但它们的精确行为可能因特定的OLTP系统而异。

SNA安全选项安全=无

如果指定PGA_SECURITY_TYPE=NONE ，则网关不会处理客户端用户标识和密码。会话分配SNA选项SECURITY=NONE 。

SNA安全选项SECURITY = PROGRAM

如果你指定 PGA_SECURITY_TYPE=PROGRAM ，然后网关使用SNA选项SECURITY=PROGRAM分配对话，并将以下信息发送给 OLTP：

  • 如果使用提示用户ID和密码覆盖，然后发送指定的用户ID和密码，而不管数据库链接规范如何。

  • 如果数据库链接有明确的 CONNECT信息，然后发送指定的用户ID和密码。

  • 如果数据库链接没有CONNECT子句，并且应用程序使用显式用户标识和密码登录到Oracle，则会发送Oracle用户标识和密码。

  • 如果应用程序使用操作系统身份验证登录到Oracle，并且数据库链接缺少显式CONNECT信息，则不会发送任何用户ID和密码。如果未发送用户标识和密码，并且未将OLTP配置为分配默认用户标识，则连接将失败。

通常，SNA选项SECURITY=PROGRAM告诉OLTP使用可用的任何身份验证机制来验证用户ID /密码组合。例如，如果用于z / OS的CICS Transaction Server是OLTP，然后可以使用RACF。但是，情况并非总是如此，因为每个OLTP都可以配置为以其他方式处理入站用户ID。

SNA安全选项SECURITY =相同

我指定你 PGA_SECURITY_TYPE=SAME ，网关使用SNA选项SECURITY=SAME分配会话，并仅向用户ID发送没有密码的用户ID OLTP。在这种情况下，你的 SNA通信包发送网关服务器可执行文件的拥有用户ID， $ORACLE_HOME/bin/pg4asrv作为用户ID。发送的用户标识不是Oracle用户标识。可以使用UNIX ls命令查看此用户标识，并且可以由授权用户使用chown命令进行更改。由于此用户标识对于给定网关实例的所有用户都是相同的，因此该选项的用途有限。

注意：

该您发送的用户ID未被转换为大写 SNA通讯包。如果OLTP在不允许使用小写用户ID的系统上运行（例如，z / OS），则必须在平台上设置大写用户ID，使其成为网关可执行文件的所有者。

SECURITY=SAME与您的平台操作系统身份验证类似。它告诉OLTP用户已在对话的始发端进行了身份验证。服务器端可能存在配置参数或选项，这些参数或选项会影响是否接受SECURITY=SAME会话。正确配置后，OLTP仅确认用户ID本身有效，然后接受连接。和。一样 SECURITY=PROGRAM ，您可以使用许多OLTP中的配置选项更改此设置。

TCP / IP安全性

本节中的安全信息仅适用于使用TCP / IP for IMS Connect功能的Oracle Database Gateway for APPC的用户。当网关收到启动远程事务程序的RPC请求时，网关会尝试启动与IMS Connect的TCP / IP对话。IMS Connect将通过OTMA和XCF联系OLTP（IMS）。有关更多信息，请参阅“IBM I MS Connect指南和参考 ”。当网络使用TCP / IP地址或主机名和端口号从网关连接到IMS Connect时，会发生网关和IMS Connect之间的对话。

注意：

由于网关使用PGAU生成TIP，因此TIP包含SNA信息。使用具有TCP / IP支持的APPC的Oracle数据库网关进行IMS Connect时，需要将SNA名称映射到TCP / IP主机名和端口号，以便网关与IMS Connect通信。使用 pg4tcpmap工具将信息从SNA映射到TCP / IP。有关详细信息，请参阅Oracle Database Gateway for APPC用户指南的第6章“pg4tcpmap命令”。

IMS Connect在会话启动时提供验证，允许每个连接对其伙伴进行身份验证。在IMS的网关和OLTP应用程序开始对话并处理对话级安全数据之前，此验证完全由网络软件执行。如果使用会话级安全性，则必须在您的平台中以及要访问的OLTP中的类似参数结构中建立正确的密码信息。

指定TCP / IP对话安全性

该通过网关初始化文件的PGA_SECURITY_TYPE参数，您可以指定网关为OLTP分配的会话的安全行为。请参阅TCP / IP通信协议的网关初始化参数 。

TCP / IP安全选项SECURITY = NONE

如果指定PGA_SECURITY_TYPE=NONE ，则网关不会处理客户端用户标识和密码。

TPC / IP安全选项SECURITY = PROGRAM

如果你指定 PGA_SECURITY_TYPE=PROGRAM ，然后将以下信息发送给 OLTP：

  • 如果使用提示用户ID和密码覆盖，然后发送指定的用户ID和密码，而不管数据库链接规范如何。

  • 如果数据库链接有明确的 CONNECT信息，然后发送指定的用户ID和密码。

  • 如果数据库链接没有CONNECT子句，并且应用程序使用显式用户标识和密码登录到Oracle，则会发送Oracle用户标识和密码。

  • 如果应用程序使用操作系统身份验证登录到Oracle，并且数据库链接缺少显式CONNECT信息，则不会发送任何用户ID和密码。如果未发送用户标识和密码，并且未将OLTP配置为分配默认用户标识，则连接将失败。

当使用TCP / IP for IMS Connect的Oracle数据库网关APPC与IMS Connect通信时，RACF是唯一可用的身份验证机制。

注意：

你必须指定你的 RACF组名称通过 pg4tcpmap工具，如果您已将PGA安全选项设置为SECURITY=PROGRAM 。有关此问题的更多信息，请参阅“ Oracle Database Gateway for APPC用户指南” 。

网关初始化文件中的密码

初始化参数可能包含敏感信息，例如用户ID或密码。初始化参数存储在纯文本文件中，这可能是不安全的。已向异构服务添加了加密功能，从而可以加密参数值。这是通过dg4pwd实用程序完成的。有关此实用程序的更多信息，请参阅“ Oracle数据库异构连接用户指南” 。
