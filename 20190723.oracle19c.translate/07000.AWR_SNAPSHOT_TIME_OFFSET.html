1.29 AWR_SNAPSHOT_TIME_OFFSET

AWR_SNAPSHOT_TIME_OFFSET指定自动工作负载存储库（AWR）快照开始时间的偏移量。

属性 描述

参数类型

整数

默认值

默认情况下没有偏移。

可修改

改变系统

可在PDB中修改

没有

价值范围

0 - 3599，或特殊值1000000

基本

没有

Oracle RAC

多个实例应使用相同的值

AWR快照通常从小时开始（12:00，1：00,2：00等）开始。此参数允许DBA为AWR快照开始时间指定偏移量。

这是一个有用的参数，可以避免来自多个实例的CPU峰值同时启动其AWR快照。如果您有一个包含许多实例的大型系统（如许多Exadata安装），并且您遇到此类CPU峰值，则此参数非常有用。

参数以秒为单位指定。通常，您将其设置为小于3600的值。如果设置特殊值1000000（1,000,000），则会获得自动模式，其中偏移量基于数据库名称。

当您在同一节点上运行大量实例时，自动模式是获得合理分配偏移时间的有效方法。
