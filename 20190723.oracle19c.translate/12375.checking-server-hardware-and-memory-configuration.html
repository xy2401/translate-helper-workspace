检查服务器硬件和内存配置

使用此过程可收集有关服务器配置的信息。

  1. 使用以下命令报告当前未使用的内存页和交换文件磁盘块的数量：
    ＃sar -rni

    例如：

    ＃sar -r 2 10
    如果系统中安装的物理RAM的大小小于所需的大小，则必须先安装更多内存，然后再继续。
  2. 确定交换空间的使用情况和配置的交换空间的大小：
    ＃/ usr / sbin / swap -s
    如有必要，请参阅操作系统文档以获取有关如何配置其他交换空间的信息。
  3. 确定/tmp目录中可用的空间量：
    #df -kh / tmp

    如果/ tmp目录中的可用空间小于所需的空间，则完成以下步骤之一：

    • 从/ tmp目录中删除不必要的文件以满足磁盘空间要求。

    • 设置Oracle用户的环境时，还要将TMP和TMPDIR环境变量设置为要使用的目录而不是/tmp 。

  4. 确定系统上的可用磁盘交换空间量：
    #df -kh
  5. 确定RAM大小：
    ＃/ usr / sbin / prtconf | grep“内存大小”
  6. 确定系统体系结构是否可以运行该软件：
    ＃/ bin / isainfo -kv
    此命令显示处理器类型。例如：
    64位sparcv9内核模块
    64位amd64内核模块

    如果您没有看到预期的输出，则无法在此系统上安装该软件。
