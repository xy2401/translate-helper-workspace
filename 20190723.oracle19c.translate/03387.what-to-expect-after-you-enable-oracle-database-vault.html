2启用Oracle Database Vault后的预期结果

启用Oracle Database Vault时，会修改多个Oracle数据库安全功能（如默认用户授权）以提供更强的安全限制。

  • 更改的初始化和密码参数设置
    Oracle Database Vault配置修改了多个数据库初始化参数设置，以更好地保护数据库配置。
  • Oracle Database Vault如何限制用户授权
    Oracle数据库配置需要两个额外的管理数据库帐户名。
  • 新的数据库角色强制执行职责分离
    Oracle Database Vault配置实现了职责分离的概念，因此您可以满足法规，隐私和其他合规性要求。
  • 从现有用户和角色撤销的权限
    Oracle Database Vault配置会撤消多个Oracle数据库提供的用户和角色的权限，以便更好地分离职责。
  • 为现有用户和角色预防的权限
    Oracle Database Vault配置会阻止已授予这些权限的所有用户和角色的多个权限，包括用户SYS和SYSTEM 。
  • 修改了非统一审计环境的AUDIT语句设置
    配置Oracle Database Vault并决定不使用统一审计时，Database Vault会配置多个AUDIT语句。

更改的初始化和密码参数设置

Oracle Database Vault配置修改了多个数据库初始化参数设置，以更好地保护数据库配置。

如果这些更改会对您的组织过程或数据库维护过程产生负面影响，请与Oracle支持部门联系以获取解决问题的帮助。

Oracle Database Vault如何限制用户授权

Oracle数据库配置需要两个额外的管理数据库帐户名。

此外，还创建了几个数据库角色。这些帐户是Oracle Database Vault提供的职责分离的一部分。影响多个大型组织的一个常见审计问题是生产实例中的数据库管理员未经授权创建新数据库帐户。安装后，Oracle Database Vault会阻止Oracle Database Vault帐户管理员或授予Oracle Database Vault帐户管理员角色的用户以外的任何人在数据库中创建用户。

新的数据库角色强制执行职责分离

Oracle Database Vault配置实现了职责分离的概念，因此您可以满足法规，隐私和其他合规性要求。

Oracle Database Vault明确区分了数据库内的帐户管理职责，数据安全责任和数据库管理职责。这意味着超级特权用户（例如， DBA ）的概念分为几个新的数据库角色，以确保没有一个用户可以完全控制系统的数据和配置。Oracle Database Vault可以防止特权用户（具有DBA和其他特权角色和系统特权的用户）访问名为realms的数据库的指定保护区域。它还引入了称为Oracle Database Vault Owner（ DV_OWNER ）和Oracle Database Vault Account Manager（ DV_ACCTMGR ）的新数据库角色。这些新的数据库角色将数据安全性和帐户管理与传统的DBA角色分开。您应该将这些角色映射到组织内的不同安全专业人员。

为现有用户和角色预防的权限

Oracle Database Vault配置会阻止已授予这些权限的所有用户和角色的多个权限，包括用户SYS和SYSTEM 。

DV_ACCTMGR角色具有分离职责的这些权限。

  • ALTER PROFILE

  • 更改用户

  • 创建个人资料

  • 创建用户

  • DROP PROFILE

  • DROP USER

为了更好的安全性和维护职责分离标准，请不要让SYS或SYSTEM用户能够创建或管理用户帐户。
