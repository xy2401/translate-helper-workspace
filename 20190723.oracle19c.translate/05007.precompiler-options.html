10.1预编译器命令

预编译器的位置因系统而异。系统或数据库管理员通常定义逻辑或别名，或使用其他特定于系统的方法来使Pro * C / C ++可执行文件可访问。

要运行Pro * C / C ++预编译器，请发出以下命令：

proc 选项 = 值 ...

注意：

选项值始终与选项名称之间用等号分隔，等号周围没有空格。

例如，命令

proc INAME = test_proc

预编译当前目录中的文件test_proc.pc ，因为预编译器假定文件扩展名为pc 。INAME = 参数指定要预编译的源文件。INAME选项不必是命令行中的第一个选项，但如果是，则可以省略选项规范。所以，命令

proc myfile

相当于

proc INAME = myfile

注意：

选项名称和未命名特定操作系统对象的选项值（如文件名）不区分大小写。在本指南的示例中，选项名称以大写形式编写，选项值通常为小写。输入文件名（包括Pro * C / C ++预编译器可执行文件本身的名称）时，请始终遵循操作系统使用的案例约定。

某些平台（如UNIX）在值字符串中的某些字符之前需要“转义字符”。请参阅特定于平台的文档。

10.1.1案例敏感性

通常，您可以使用大写或小写作为预编译器选项名称和值。但是，如果您的操作系统区分大小写（如UNIX），则必须使用正确的大写和小写字母组合指定文件名值，包括Pro * C / C ++可执行文件的名称。

10.2预编译器选项

预编译器选项使您可以控制资源的使用方式，报告错误的方式，输入和输出的格式，以及如何管理游标。

选项的值是文字，表示文本或数字值。例如，对于选项

...INAME = my_test

该值是一个字符串文字，用于指定文件名。

对于选项MAXOPENCURSORS

...MAXOPENCURSORS = 20

值是数字。

某些选项采用布尔值，您可以使用字符串yes或no ， true或false或分别使用整数文字1或0来表示这些值。例如，选项

...SELECT_ERROR = YES

相当于

...SELECT_ERROR =真

要么

...SELECT_ERROR = 1

所有这些都意味着应该在运行时标记SELECT错误。

10.2.2配置文件

配置文件是包含预编译器选项的文本文件。文件中的每个记录（行）只包含一个选项及其关联的值。忽略第一个选项后在一行上输入的任何选项。例如，以下配置文件包含以下行：

FIPS = YES MODE = ANSI CODE = ANSI_C

设置FIPS，MODE和CODE选项的默认值。

pcscfg.cfg每个条目的每行限制为300个字符。要设置长度超过300个字符的值（例如，SYS_INCLUDE路径），请在多行中创建条目。例如，

sys_include = / ade / aime_rdbms_9819 / oracle / precomp / public sys_include = / usr / include，/ usr / lib / gcc-lib / i486-suse-linux / 2.95.3 / include sys_include = / usr / lib / gcc-lib / i386-redhat-linux / 3.2.3 / include sys_include = / usr / lib / gcc-lib / i386-redhat-linux7 / 2.96 / include sys_include = / usr / include

不要在一行的末尾使用括号。行右侧的括号将使所有前一行无效。例如，第三行末尾的括号，

sys_include = / ade / aime_rdbms_9819 / oracle / precomp / public sys_include = / usr / include，/ usr / lib / gcc-lib / i486-suse-linux / 2.95.3 / include sys_include = / usr / lib / gcc-lib / i386-redhat-linux / 3.2.3 / include）sys_include = / usr / lib / gcc-lib / i386-redhat-linux7 / 2.96 / include sys_include = / usr / include

将SYS_INCLUDE设置为

/usr/lib/gcc-lib/i386-redhat-linux/3.2.3/include,/usr/lib/gcc-lib/i386-redhat-linux7/2.96/include,/usr/include

每个安装都有一个系统配置文件。系统配置文件的名称是pcscfg.cfg 。文件的位置是特定于系统的。

注意：

pcscfg.cfg包含变量（如include或LIBPATH默认路径设置。这些路径可以是计算机或操作系统相关的。您必须验证默认路径是否对您的计算机和操作系统有效。如果无效，请用8dot3表示法替换路径。

在pcscfg.cfg文件中，您不能使用空格。例如，如果文件包含以下行：

include =“D：\ Program Files \ Microsoft Visual Studio \ VC98 \ include”

预编译将失败。您可以使用以下内容替换它：

包括= d：\ PROGRA〜1 \毫安〜4 \ VC98 \包括

每个Pro * C / C ++用户都可以拥有一个或多个私有配置文件。必须使用CONFIG = precompiler选项指定配置文件的名称。

注意：

您无法嵌套配置文件。这意味着CONFIG =不是配置文件中的有效选项。

10.2.3期权价值的优先顺序

选项的值以越来越高的优先级确定：

  • 内置于预编译器的值

  • Pro * C / C ++系统配置文件中设置的值

  • Pro * C / C ++用户配置文件中设置的值

  • 在命令行中设置的值

  • 内联值设置

例如，选项MAXOPENCURSORS指定缓存的打开游标的最大数量。此选项的内置预编译器默认值为10。但是，如果在系统配置文件中指定了MAXOPENCURSORS = 32，则默认值现在为32。用户配置文件可以将其设置为另一个值，然后覆盖系统配置值。最后，内联规范优先于所有先前的默认值。

如果多次使用PROC命令行选项，则最后一次出现的最后一个值是PROC用于预编译的值。例如，

$ proc iname = sample.pc ... oname = output1.c ... oname = output2.c ... oname = output3.c

在示例中， output3.c是PROC使用的ONAME值，生成的输出文件名是output3.c 。

如果在配置文件（系统缺省或用户指定）和命令行中都指定了选项，则命令行中指定的值优先。

对于SYS_INCLUDE和INCLUDE选项，行为与环境变量中的定义相同。除非被一行末尾的括号中断，否则将附加值。

如果使用CONFIG = filename指定专用配置文件，则第一个值优先，并忽略命令行中的后续出现。这是命令行最后一个值优先的例外。

某些选项（如USERID）没有预编译器默认值。表10-2中列出了具有这些选项的选项的内置默认值。

注意：

检查系统特定的文档以获取预编译器的默认值;对于您的平台，它们可能已从本章中的值更改。

确定当前值

您可以通过在命令行上使用问号以交互方式确定一个或多个选项的当前值。例如，如果您发出命令

完整的选项集及其当前值将打印到您的终端。（在运行C shell的UNIX系统上，转义'？'用反斜杠。）在这种情况下，值是预编译器中内置的值，由系统配置文件中的任何值覆盖。但是如果你发出命令

proc config = my_config_file.h？

并且当前目录中有一个名为my_config_file.h的文件，列出了所有选项。用户配置文件中的值提供缺失值，并取代Pro * C / C ++预编译器内置的值或系统配置文件中指定的值。

您还可以通过简单地指定选项名称，然后输入=？来确定单个选项的当前值。例如：

打印MAXOPENCURSORS选项的当前默认值。

10.2.5预编译期间发生了什么？

在预编译期间，Pro * C / C ++生成C或C ++代码，用于替换宿主程序中嵌入的SQL语句。生成的代码包含指示主机变量的数据类型，长度和地址的数据结构，以及运行时库SQLLIB所需的其他信息。生成的代码还包含对执行嵌入式SQL操作的SQLLIB例程的调用。

注意：

预编译器不会生成对Oracle调用接口（OCI）例程的调用。

10.2.6期权的范围

预编译单元是包含C代码和一个或多个嵌入式SQL语句的文件。为给定的预编译单元指定的选项仅影响该单元;它们对其他单位没有影响。例如，如果为单元A指定HOLD_CURSOR = YES和RELEASE_CURSOR = YES，而不为单元B指定，则单元A中的SQL语句将使用这些HOLD_CURSOR和RELEASE_CURSOR值运行，但单元B中的SQL语句将使用默认值运行。

10.2.7 Windows平台的Pro * C / C ++预编译器问题

本节重点介绍与Pro * C / C ++ for Windows平台相关的问题。

10.2.7.1配置文件

10.2.7.2代码

CODE选项的默认设置为ANSI_C 。其他操作系统的Pro * C / C ++可能默认设置为KR_C 。

10.2.7.3 DBMS

使用CHAR_MAP=VARCHAR2时，不支持DBMS=V6_CHAR 。而是使用DBMS=V7 。

10.2.7.5 PARSE

PARSE选项的默认设置为NONE 。其他操作系统的Pro * C / C ++可能默认设置为FULL 。

10.4输入选项

您可以在命令行中输入任何预编译器选项。许多也可以使用EXEC ORACLE OPTION语句在预编译器程序源文件中内联输入。

10.4.1在命令行上

您可以使用以下语法在命令行中输入预编译器选项： 

...[OPTION_NAME = value] [OPTION_NAME = value] ...

使用一个或多个空格分隔每个选项=值规范。例如，您可以输入以下内容：

...CODE = ANSI_C MODE = ANSI

10.4.2内联

您可以使用以下语法编写EXEC ORACLE语句来输入内联选项： 

EXEC ORACLE OPTION（OPTION_NAME = value）;

例如，您可以编写以下代码：

EXEC ORACLE选项（RELEASE_CURSOR =是）;
10.4.2.1 EXEC ORACLE的用途

EXEC ORACLE功能对于在预编译期间更改选项值特别有用。例如，您可能希望逐个语句地更改HOLD_CURSOR和RELEASE_CURSOR。

如果操作系统限制了可以在命令行上输入的字符数，则内联或配置文件中的选项也很有用。

10.5.26 HOLD_CURSOR

目的

句法

默认

没有

使用说明

可以在内联或命令行输入。

您可以使用HOLD_CURSOR来提高程序的性能。

执行SQL数据操作语句时，其关联的游标将链接到游标缓存中的条目。游标缓存条目又链接到Oracle私有SQL区域，该区域存储处理语句所需的信息。HOLD_CURSOR控制光标和光标缓存之间的链接发生了什么。

对于使用隐式游标的内联使用，请在执行SQL语句之前设置HOLD_CURSOR。对于使用显式游标的内联使用，请在关闭游标之前设置HOLD_CURSOR。

RELEASE_CURSOR = YES覆盖HOLD_CURSOR = YES且HOLD_CURSOR = NO覆盖RELEASE_CURSOR = NO。有关这两个选项如何交互的信息，请参阅表B-1 。

10.5.36 MAXOPENCURORS

目的

指定预编译器尝试保持高速缓存的并发打开游标数。

句法

MAXOPENCURSORS = 整数

默认

10

使用说明

可以在内联或命令行输入。

您可以使用MAXOPENCURSORS来提高程序的性能。单独预编译时，请使用MAXOPENCURSORS。 MAXOPENCURSORS指定SQLLIB游标缓存的初始大小。

执行隐式语句且HOLD_CURSOR = NO或关闭显式游标时，光标条目将标记为可重用。如果再次发出此语句并且游标条目尚未用于另一个语句，则会重复使用该语句。

由于您的程序需要同时打开游标，因此您可能需要重新指定MAXOPENCURSORS以满足需要。值45到50并不罕见，但请记住，每个游标都需要用户进程内存空间中的另一个私有SQL区域。对于大多数程序，默认值10都足够了。

10.5.55 RELEASE_CURSOR

目的

指定如何在游标缓存中处理SQL语句和PL / SQL块的游标。

句法

RELEASE_CURSOR = {YES |没有}

默认

没有

使用说明

可以在内联或命令行输入。

您可以使用RELEASE_CURSOR来提高程序的性能。

执行SQL数据操作语句时，其关联的游标将链接到游标缓存中的条目。游标缓存条目又链接到Oracle私有SQL区域，该区域存储处理语句所需的信息。RELEASE_CURSOR控制游标缓存和私有SQL区域之间的链接发生了什么。

当RELEASE_CURSOR = YES时，在Oracle执行SQL语句并关闭游标后，预编译器会立即删除该链接。这将释放分配给私有SQL区域的内存并释放解析锁。要确保在关闭游标时释放关联的资源，必须指定RELEASE_CURSOR = YES。

当RELEASE_CURSOR = NO时，保持链接。除非打开游标的数量超过MAXOPENCURSORS的值，否则预编译器不会重复使用该链接。这对于经常执行的SQL语句很有用，因为它可以加速后续执行。无需重新解析语句或为Oracle私有SQL区域分配内存。

对于使用隐式游标的内联使用，请在执行SQL语句之前设置RELEASE_CURSOR。对于使用显式游标的内联使用，请在关闭游标之前设置RELEASE_CURSOR。

RELEASE_CURSOR = YES覆盖HOLD_CURSOR = YES。

10.5.57 SELECT_ERROR

目的

指定当SELECT语句返回多行或者主机数组可容纳的行数时，程序是否生成错误。

句法

SELECT_ERROR = {YES |没有}

默认

是

使用说明

可以在内联或命令行输入。

当SELECT_ERROR = YES时，如果单行SELECT返回太多行，或者数组SELECT返回的行数超过主机数组可容纳的行数，则会生成错误。SELECT的结果是不确定的。

当SELECT_ERROR = NO时，当单行SELECT返回太多行时，或者当数组SELECT返回的行数多于主机数组可容纳的行数时，不会生成错误。

无论是指定YES还是NO，都会从表中选择随机行。确保行的特定排序的唯一方法是在SELECT语句中使用ORDER BY子句。当SELECT_ERROR = NO并且您使用ORDER BY时，Oracle会在您选择进入数组时返回第一行或前n行。当SELECT_ERROR = YES时，无论是否使用ORDER BY，当返回太多行时都会生成错误。

10.5.59 SYS_INCLUDE

目的

指定系统头文件的位置。

句法

SYS_INCLUDE = 路径名 | （path1 ，...， pathn )

默认

系统专用。

使用说明

Pro * C / C ++在特定于平台的标准位置搜索标准系统头文件，例如stdio.h 。例如，在几乎所有UNIX系统上，文件stdio.h都具有完整路径名/usr/include/stdio.h.

但是C ++编译器可以有系统头文件，例如stdio.h ，它们不在标准系统位置。您可以使用SYS_INCLUDE命令行选项指定Pro * C / C ++搜索以查找系统头文件的目录路径列表。例如：

SYS_INCLUDE =（在/ usr /郎/ SC2.0.1 /包括，在/ usr /郎/ SC2.1.1 /包括）

使用SYS_INCLUDE指定的搜索路径会覆盖默认标头位置。

如果PARSE = NONE，则SYS_INCLUDE中指定的值与预编译无关，因为Pro * C / C ++不需要在预编译中包含系统头文件。（当然，您必须包含特定于Oracle的标头，例如sqlca.h 。和系统头文件，以及#include指令，以便编译器进行预处理。）

预编译器按以下顺序搜索目录：

  1. 当前目录

  2. SYS_INCLUDE预编译器选项中指定的系统目录

  3. INCLUDE选项指定的目录，按输入的顺序

  4. 标准头文件的内置目录

由于步骤3，通常不需要为标准头文件指定目录路径，例如sqlca.h和sqlda.h 。

使用SYS_INCLUDE选项指定目录路径的语法是特定于系统的。遵循用于您的操作系统的约定

注意：

对于SYS_INCLUDE选项，选项值的优先级会反转。与其他值被覆盖的选项不同，SYS_INCLUDE会附加以下提到的所有目录文件：

  • 预编译器

  • Pro * C / C ++系统配置文件

  • Pro * C / C ++用户配置文件

  • 命令行

  • 排队

但是，在括号内或不括号内传递值之间存在差异。如果在括号内传递单个值或目录列表，则会覆盖SYS_INCLUDE的现有值。如果将列表作为没有括号的简单值传递，它将补充任何现有值。

10.5.64 UTF16_CHARSET

目的

指定UNICODE（UTF16）变量使用的字符集格式。

句法

UTF16_CHARSET = {NCHAR_CHARSET |值为db_charset}

默认

NCHAR_CHARSET

使用说明

只能在命令行或配置文件中使用，但不能内联使用。

警告：

当目标列为NCHAR时，可能会丢失数据。

10.5.66版本

目的

确定EXEC SQL OBJECT DEREF语句将返回的对象版本。

句法

版本= {最近|最新|任何}

默认

最近

使用说明

可以使用EXEC ORACLE OPTION语句进行内联输入。

RECENT表示如果在当前事务中将对象选择到对象缓存中，则返回该对象。对于以可序列化模式运行的事务，此选项与LATEST具有相同的效果，而不会产生尽可能多的网络往返。大多数应用程序应使用RECENT。

LATEST意味着如果对象不驻留在对象缓存中，则从数据库中检索它。如果它确实驻留在对象缓存中，则会从服务器刷新它。请谨慎使用LATEST，因为它会导致最多的网络往返次数。仅当必须使对象缓存与服务器缓冲区缓存保持一致时，才使用LATEST

ANY表示如果对象已驻留在对象缓存中，则返回该对象。如果没有，请从服务器检索对象。任何网络往返都是最少的。用于访问只读对象的应用程序或用户可以独占访问对象的应用程序。
