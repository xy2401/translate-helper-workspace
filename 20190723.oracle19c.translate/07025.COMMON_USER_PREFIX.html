1.47 COMMON_USER_PREFIX

COMMON_USER_PREFIX指定多租户容器数据库（CDB）中的公共用户，角色和配置文件的名称COMMON_USER_PREFIX的前缀。

属性 描述

参数类型

串

句法

COMMON_USER_PREFIX =前缀

默认值

在CDB根目录中， C##是默认值。在应用程序根目录中，空字符串是默认值。

可修改

没有

可在PDB中修改

是

基本

没有

Oracle RAC

多个实例必须具有相同的值

本地用户，角色和配置文件的名称不得以相同的前缀开头。将用户，角色和配置文件名称的前缀与此参数的值进行不区分大小写的比较。

为COMMON_USER_PREFIX设置值时，Oracle将要求公共用户，角色和配置文件的名称以分配给此参数的字符串开头。

使用此前缀创建的用户，角色和配置文件的名称必须是有效名称。

无论COMMON_USER_PREFIX参数的值如何，本地用户，角色和配置文件的名称都不能以C##开头。类似地，应用程序PDB中的应用程序公共用户和本地用户的名称不能以C##开头。

警告：

您可以更改COMMON_USER_PREFIX参数的值，但只能非常小心地这样做。

如果COMMON_USER_PREFIX设置为空字符串，则Oracle 不会对公共或本地用户，角色和配置文件的名称强制实施任何限制。

将此参数设置为空字符串将导致对公共和本地用户，角色和配置文件的名称没有任何限制，这可能导致在插入PDB时本地用户和公共用户，角色和配置文件的名称之间发生冲突进入不同的CDB，或打开创建普通用户时关闭的PDB。

注意：

COMMON_USER_PREFIX可以在应用程序容器的上下文中设置。

如果将应用程序根目录中的COMMON_USER_PREFIX设置为非空值，则应用程序公共用户应以该前缀开头，并且在应用程序PDB中创建的本地用户不能以该前缀开头。

默认情况下， COMMON_USER_PREFIX是应用程序根目录中的空字符串。

注意：

查询V$PARAMETER或V$SYSTEM_PARAMETER视图时，将为COMMON_USER_PREFIX参数的DEFAULT_VALUE列返回值NONE 。

但是，数据库对CDB根执行默认值C## ，为应用程序根执行默认值空字符串。
