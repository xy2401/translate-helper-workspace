操作：不要在当前消息传递域中执行操作

JMS-00235：无法将持久订阅者名称与unsubscribe方法中的主题链接。

原因： OJMS无法将提供的持久订户名称与主题链接。这会导致取消订阅方法失败。

操作：使用用户可以提供主题名称的其他取消订阅方法。

JMS-00236：OJMS遇到无效的OCI句柄。

原因：使用JDBC OCI驱动程序时，OJMS遇到无效的OCI句柄。

操作：确保在OJMS仍在运行时未关闭基础JDBC连接。

JMS-00237：无法为消息侦听器启动线程。

原因： OJMS无法为消息侦听器启动新线程。

操作：如果线程是由用户提供的线程代理启动的，请检查线程代理代码。

JMS-00238：非法尝试在事务处理的JMS会话上进行恢复

原因：非法尝试在事务处理的JMS会话上调用recover方法

操作：在事务处理的JMS会话上使用rollback方法而不是recover方法。

JMS-00239：非法尝试在XASession上调用{0}方法。

原因：非法尝试在XASession上调用提交或回滚方法

操作：使用JTA提交或回滚分布式事务

JMS-00240：在其他操作后非法尝试调用setClientID。

原因：在对此Connection执行其他操作后，非法尝试调用Connection.setClientID方法

操作：在Connection上的任何其他操作之前设置客户端ID

JMS-00241：当有消费者使用临时目的地时，非法尝试删除临时目的地。

原因：当现有消费者仍在使用临时目的地时，非法尝试删除临时目的地。

操作：在删除临时目标之前关闭使用者

JMS-00242：非法尝试将具有即时可见性和三阶段排队过程的消息排入队列。

原因：非法尝试将具有即时可见性和三阶段排队过程的消息排入队列。

操作：打开系统属性oracle.jms.useTemplobsForEnqueue

JMS-00243：找不到主题{0}

原因：找不到指定的主题

操作：指定有效主题

JMS-00244：{0}是使用Sharded Queue的无效操作。

原因：使用Sharded Queue操作无效

操作：为Sharded Queue指定有效操作

JMS-00245：JMS Streaming支持仅适用于Sharded Queues。

原因：仅使用Sharded Queues支持JMS Streaming

操作：将JMS Streaming API与Sharded Queue一起使用

JMS-00246：JMS Streaming支持不适用于{0}驱动程序。

原因：仅使用瘦驱动程序支持JMS Streaming

操作：将JMS Streaming API与瘦驱动程序一起使用

JMS-00247：JMS Streaming不支持NON_PERSISTENT消息传递。

原因： JMS Streaming不支持NON_PERSISTENT传送模式

操作：指定PERSISTENT消息传递模式

JMS-00248：禁用流式传输时非法尝试使用JMS Streaming API。

原因：禁用流式传输时非法尝试使用JMS Streaming API

操作：打开系统属性oracle.jms.useJmsStreaming

JMS-00249：必须指定表示消息数据的InputStream。

原因： InputStream为null

操作：指定表示消息数据的有效InputStream

JMS-00250：必须指定OutputStream来写入消息数据。

原因： OutputStream为null

操作：指定有效的OutputStream以从接收的消息中写入数据

JMS-00251：使用write方法和Streaming API非法尝试设置消息数据。

原因：使用write方法和Streaming API非法尝试设置消息数据

操作：指定InputStream或write方法以设置消息数据

JMS-00252：当流与dequeue一起使用时，使用{0}非法尝试读取数据。

原因：当出队使用流式传输时，使用读取方法非法尝试读取消息数据

操作：调用receiveData（）以读取消息数据

JMS-00253：具有空JMSMessageID的消息上不允许操作{0}。

原因：对于具有空JMSMessageID的Message，不允许该操作

操作：对具有有效JMSMessageID的Message使用此操作

JMS-00254：流不与dequeue一起使用，请使用标准JMS API来读取消息数据。

原因：流式传输不与出列操作一起使用

操作：使用标准JMS API读取消息数据

JMS-00255：JMS Streaming支持在具有确认模式的会话中可用+

原因：与JMS Streaming一起使用的确认模式无效

操作：有效的确认模式是会话。CLIENT_ACKNOWLEDGE和Session。SESSION_TRANSACTED

JMS-00256：javax.jms的stop（）。连接超时。

原因： javax.jms的stop（）。连接超时

操作：设置默认超时10分钟以禁用注册，并允许onMessage（）处理消息。使用系统属性oracle.jms.notificationTimeOut为此超时设置更高的毫秒值

JMS-00257：接收（长时间超时）javax.jms。MessageConsumer比java.sql配置的网络超时花费的时间更多。连接。}

原因：接收（长时间超时）javax.jms。MessageConsumer超时。

操作：请为系统属性'oracle.jms.networkTimeOut'设置更高的值（以毫秒为单位）或在java.sql上禁用网络超时。通过将系统属性“oracle.jms.setNetworkTimeout”设置为false来进行连接。
