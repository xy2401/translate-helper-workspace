2.1命令模式和语法

通过Oracle连接管理器控制实用程序（CMCTL），您可以启动，配置和更改客户端连接请求的管理方式。

Oracle Connection Manager Control实用程序的基本语法如下：

cmctl command [ argument ]

Oracle Connection Manager Control实用程序支持以下类型的命令：

  • 初始化和终止命令，如STARTUP和SHUTDOWN

  • 更改SET LOG_LEVEL和SET EVENT等命令

  • 显示命令，例如SHOW STATUS和SHOW RULES

  • 网关命令，如SHOW GATEWAYS和RESUME GATEWAYS

注意：

您可以使用SET命令动态更改控制侦听器如何接收客户端连接的配置参数。在Oracle Connection Manager关闭之前，更改仅保持有效。您无法将它们保存到cman.ora文件中。唯一的例外是Oracle Connection Manager密码，您可以使用命令SAVE_PASSWD保存该SAVE_PASSWD 。

您可以在命令模式或批处理模式下使用Oracle Connection Manager Control实用程序。

  • 使用命令模式：

    • 从Oracle Connection Manager控制实用程序：

      在命令行输入cmctl以获取程序提示符，然后发出命令：

      cmctl CMCTL> command

    • 从操作系统：

      从操作系统命令提示符输入整个命令：

      cmctl [ command ] [ argument1 。 。 。 argumentN ] [-c instance_name ]

      以这种方式发出的每个命令都可以附加一个Oracle Connection Manager实例名称作为参数。如果未提供Oracle Connection Manager实例名称，则假定使用默认实例名称。默认名称为cman_ hostname 。如果在先前的CMCTL会话中设置了密码，则系统会提示您输入密码。如果已设置密码，然后从Oracle Connection Manager的Oracle Connection Manager控制实用程序会话发出命令，则必须在会话开始时输入一次密码。

      警告：

      可以选择在命令行上指定密码。但是，这样做会在屏幕上显示密码，并且存在潜在的安全风险。Oracle建议不要在命令行上使用密码选项（ -p ）。

  • 使用批处理模式：

    您可以在标准文本文件中组合命令，然后将它们作为一系列命令运行。要以批处理模式运行，请使用以下语法：

    cmctl @ input_file

2.3.1管理员

ADMINISTER命令只能从Oracle Connection Manager实用程序发出，并允许您选择要管理的Oracle Connection Manager实例。

目的

选择Oracle Connection Manager实例。

先决条件

没有

句法

从Oracle Connection Manager控制实用程序：

CMCTL> ADMINISTER [-c] instance_name

参数

instance_name ：您要管理的Oracle Connection Manager的实例名称。实例在cman.ora文件中定义。

使用说明

您只能在实用程序中发出ADMINISTER命令。您无法从操作系统发出命令。

ADMINISTER使您可以选择要管理的Oracle Connection Manager实例。要启动Oracle Connection Manager实例，必须发出STARTUP命令。

从命令中省略实例名称时，管理的实例默认为本地实例。

使用-c选项何时管理不是本地实例的实例。

只有在安装时或Oracle Connection Manager的上一个会话期间提供密码时，才需要密码。

例

CMCTL> ADMINISTER cman_indl040ad输入CMAN密码： password当前实例cman_indl040ad已启动连接参考（address =（protocol = TCP）（host = indl040ad）（port = 1560））。命令成功完成

2.3.2关闭连接

它是用于终止连接的Oracle Connection Manager实用程序命令，但此时必须运行Oracle Connection Manager。

目的

要终止连接，请使用特定限定符来选择连接。

先决条件

Oracle Connection Manager必须正在运行。

句法

从操作系统：

cmctl CLOSE CONNECTIONS [in state ] [gt time ] [from source ] [to destination ] [for service ] [using gateway_process_id ] [ connect_identifier_list ] [-c cman_name ]

从Oracle Connection Manager控制实用程序：

CMCTL>关闭连接[ 状态 ] [gt 时间 ] [从源 ] [到目的地 ] [用于服务 ] [使用gateway_process_id ] [ connect_identifier_list

参数

state :指定连接状态的以下值之一：

  • idle ：在已建立状态下处于非活动状态的连接。

  • connecting ：正在连接的连接。

  • established ：已连接且正在传输数据的连接。

  • terminating ：断开连接的连接。

如果未指定任何状态，则CLOSE CONNECTIONS默认为所有可能的状态。如果在这些条件下包含时间限定符，则指定的时间是自客户端发起连接以来经过的时间量。

时间 ：时间格式。使用以下格式指定大于指示时间的连接：

gt[ hh : mm :] ss

source ：源地址。使用以下格式之一指定源地址：

  • from IP

  • from hostname

  • from subnet

destination ：目标地址。使用以下格式之一指定目标地址：

  • to IP

  • to hostname

  • to subnet

service ：服务名称。使用service_name参数指定服务，例如sales.us.example.com 。

gateway_process_id ：网关进程标识符是一个数字。使用此编号指定由指示的网关进程代理的连接。要确定网关进程标识符，请使用Oracle Connection Manager控制实用程序show gateways命令。

connect_identifier_list ：连接标识符。在列表中的多个连接标识符之间使用空格。

使用说明

由于CLOSE CONNECTIONS命令终止连接，因此它可能在客户端和服务器端生成错误消息。

IDLE状态限定符始终需要时间限定符。

在没有参数的情况下发出CLOSE CONNECTIONS关闭所有连接。

例子

以下示例关闭任何状态下的连接。连接所用的时间必须大于1小时30分钟。连接源是指定的子网，目标是指定的主机名。

CMCTL> CLOSE CONNECTIONS gt 1:30:00从192.0.2.32/24到host1

以下示例关闭由网关进程0代理的已处于空闲状态超过30分钟的连接：

CMCTL>关闭空闲连接gt 30:00使用0

以下示例关闭连接到服务sales.us.example.com连接：

CMCTL> CLOSE为sales.us.example.com建立了CONNECTIONS

2.3.6 RELOAD

重新加载Oracle Connection Manager Control实用程序命令可动态重新读取参数和规则。

目的

动态重新读取参数和规则。

先决条件

Oracle Connection Manager必须正在运行。

句法

从操作系统：

cmctl RELOAD [-with_ha] [-c instance_name ]

从Oracle Connection Manager控制实用程序：

CMCTL> RELOAD [-with_ha]

参数

-with_ha ：用于重新加载cman.ora而不删除注册

使用说明

使用RELOAD命令修改的配置信息仅适用于新连接。现有连接不受影响。SET RELOAD命令恢复在cman.ora设置的配置，并覆盖SET命令。

RELOAD使用Oracle Connection Manager侦听器重新注册网关，在此期间可能会拒绝某些新连接，直到注册过程完成。

-with_ha选项可与RELOAD一起使用，不会丢弃注册，从而在重新加载期间提供高服务可用性。

例

CMCTL> RELOAD命令成功完成

2.3.12设置事件

目的

记录特定事件的信息。

句法

从操作系统：

cmctl SET EVENT event_group [-c instance_name ]

从Oracle Connection Manager控制实用程序：

CMCTL> SET EVENT event_group {on |关}

参数

event_group ：指定以下事件组之一：

  • init_and_term ：初始化和终止事件组。

  • memory_ops ：内存操作事件组。

  • conn_hdlg ：连接处理事件组。

  • proc_mgmt ：流程管理事件组。

  • reg_and_load ：注册和加载更新事件组。

  • wake_up ：与连接管理器管理（CMADMIN）唤醒队列事件组相关的事件。

  • timer ：网关超时事件组。

  • cmd_proc ：命令处理事件组。

  • relay ：与连接控制相关的事件阻止事件组。

on ：打开事件组。

off ：关闭事件组。

使用说明

SET EVENT命令只接受一个参数。要记录多个事件，必须分别为每个事件发出命令。

例

CMCTL>将事件memory_ops设置为关闭cman11事件memory_ops设置为OFF。命令成功完成

2.3.24显示连接

目的

显示有关特定连接或所有连接的信息。

先决条件

Oracle Connection Manager必须正在运行。

句法

从操作系统：

cmctl SHOW CONNECTIONS [ 信息 ] [ 状态 ] [gt 时间 ] [从源 ] [到目的地 ] [用于服务 ] [使用gateway_process_id ] [ connect_identifier_list ] [-c instance_name ]

从Oracle Connection Manager控制实用程序：

CMCTL>显示连接[ 信息 ] [在状态 ] [gt 时间 ] [从源 ] [到目的地 ] [用于服务 ] [使用gateway_process_id ] [ connect_identifier_list ]

参数

信息 ：指定以下值之一以显示有关连接的信息。信息类别包括连接标识符，源，目标，服务，当前状态，总空闲时间和总耗用时间。

  • count ：满足其他限定符指定条件的连接总数。这是默认值。

  • detail ：有关其他限定符指定的连接的所有信息。

state ：指定以下值之一以指定连接状态：

  • idle ：在已建立状态下处于非活动状态的连接。

  • connecting ：正在连接的连接。

  • established ：已连接且正在传输数据的连接。

  • terminating ：断开连接的连接。

如果未指定任何状态，则SHOW CONNECTIONS默认为所有可能的状态。如果在这些条件下包含时间限定符，则指定的时间是自客户端发起连接以来经过的时间量。

注意：

在Traffic Director模式下，Oracle Connection Manager不支持此参数。

time ：使用以下格式指定大于指示时间的连接：

gt[ hh : mm :] ss

注意：

在Traffic Director模式下，Oracle Connection Manager不支持此参数。

source ：指定以下格式之一以指定源地址：

  • from IP

  • from hostname

  • from subnet

destination ：指定以下格式之一以指定目标地址：

  • to IP

  • to hostname

  • to subnet

service ：使用service_name格式来请求服务：

gateway_process_id ：使用以下格式指定由指示的网关进程代理的连接：

使用gateway_process_id

connect_identifier_list ：使用空格分隔多个连接标识符。

使用说明

连接按网关进程标识符和连接标识符按升序排序。

在没有参数的情况下发出SHOW CONNECTIONS会显示所有连接。

例子

以下命令显示任何状态下连接的详细说明。连接所用的时间必须大于1小时30分钟。连接源是指定的子网，目标是指定的主机名。

CMCTL> SHOW CONNECTIONS gt 1:30:00从192.0.2.32/24到host1

以下命令显示Oracle Connection Manager使用已处于空闲状态超过30分钟的网关进程标识符0代理的连接数：

CMCTL> SHOW空闲CONNECTIONS计数gt 30:00使用0

以下命令显示连接到服务sales.us.example.com的连接的详细说明：

CMCTL> SHOW建立了sales.us.example.com的CONNECTIONS详细信息

流量控制器模式中显示的其他统计信息

在Traffic Director模式下与Oracle Connection Manager的每个连接都显示以下附加统计信息：

  • Source Host Name ：客户端连接的主机名。

  • Source Process Id ：已连接客户端的进程标识。

  • Source Program Name ：已连接的客户端程序的名称。

  • Destination Hostname ：客户端通过Oracle Connection Manager连接的数据库服务器的主机名。

  • State ：具有以下值之一的入站连接的状态

    • THREAD WAIT ：连接正在等待工作线程，在专用线程模式下看不到

    • ACTIVE ：连接正在传输数据，占用线程

    • IDLE ：建立连接，但不活跃，仍然可以占据线程，如果tdm_bind_thread=true在cman.ora

  • Idle time ：连接处于IDLE状态的累计时间（ µs 。

  • Thread Wait time ：连接处于THREAD WAIT状态的累积时间（以µs为单位）。在专用线程模式下始终为0 。

  • Active time ：连接处于ACTIVE状态的累积时间（μs）。

  • PRCP State ：与代理驻留连接池（ PRCP ）相关的入站连接状态，可以是以下值之一

    • WAIT ：连接正在等待PRCP的会话

    • CHECKED-OUT ：连接拿着从出站会话PRCP但不作任何OCI电话

    • ACTIVE ：连接正在从PRCP保持出站会话并忙于OCI呼叫

    • CHECKED-IN ：公布的连接CHECKED-OUT会话回PRCP

    • NO STATE ：没有配置PRCP的服务的客户端具有此状态

  • PRCP Wait time ， PRCP Checked-out time和PRCP Active time ：连接处于CHECKED-OUT PRCP WAIT ， CHECKED-OUT和ACTIVE状态的累计时间（以µs为单位）。在非PRCP服务的情况下，所有这三个状态均为零。

  • Total Session Gets ：来自此连接的PRCP会话获取请求的总数。如果未配置PRCP ，则始终为1 。

  • Session Get Hits ：在所有请求中PRCP存在会话的次数。如果未配置PRCP ，则始终为0 。
