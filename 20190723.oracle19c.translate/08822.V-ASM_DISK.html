7.61 V $ ASM_DISK

V$ASM_DISK为Oracle自动存储管理（Oracle ASM）实例发现的每个磁盘显示一行，包括不属于任何磁盘组的磁盘。

柱 数据类型 描述

GROUP_NUMBER

数

包含磁盘的磁盘组的编号（ V$ASM_DISKGROUP视图的外键）

DISK_NUMBER

数

分配给其磁盘组中的磁盘的编号

COMPOUND_INDEX

数

一个32位数字，由高8位的磁盘组编号和低位24位的磁盘编号组成（用于有效访问视图）

INCARNATION

数

磁盘的化身编号

MOUNT_STATUS

VARCHAR2（7）

磁盘相对于组安装的每实例状态：

  • MISSING - Oracle ASM元数据表示已知该磁盘是Oracle ASM磁盘组的一部分，但未找到存储系统中具有指定名称的磁盘

  • CLOSED - 磁盘存在于存储系统中，但Oracle ASM无法访问

  • OPENED - 磁盘存在于存储系统中，Oracle ASM正在访问该磁盘。这是数据库实例中磁盘的正常状态，它是实例主动使用的磁盘组的一部分。

  • CACHED - 磁盘存在于存储系统中，是Oracle ASM实例访问的磁盘组的一部分。这是Oracle ASM实例中磁盘的正常状态，它是已装入磁盘组的一部分。

  • IGNORED - 磁盘存在于系统中，但由于以下原因之一，Oracle ASM会忽略该磁盘：

    • 磁盘由系统库检测但由于Oracle ASM库发现了同一磁盘而被忽略

    • Oracle ASM已确定磁盘标头声明的成员资格不再有效

  • CLOSING - Oracle ASM正在关闭此磁盘

HEADER_STATUS

VARCHAR2（12）

发现看到的磁盘的每个实例状态：

  • UNKNOWN - 尚未读取Oracle ASM磁盘头

  • CANDIDATE - 磁盘不是磁盘组的一部分，可以使用ALTER DISKGROUP语句添加到磁盘组

  • INCOMPATIBLE - 磁盘头中的版本号与Oracle ASM软件版本不兼容

  • PROVISIONED - 磁盘不是磁盘组的一部分，可以使用ALTER DISKGROUP语句添加到磁盘组。PROVISIONED标头状态与CANDIDATE标头状态不同， PROVISIONED意味着管理员已采取额外的特定于平台的操作，以使磁盘可用于Oracle ASM。

  • MEMBER - 磁盘是现有磁盘组的成员。不应尝试将磁盘添加到其他磁盘组。除非被FORCE选项覆盖，否则ALTER DISKGROUP语句将拒绝这样的添加。

  • FORMER - 磁盘曾经是磁盘组的一部分，但已从组中干净地丢弃。可以使用ALTER DISKGROUP语句将其添加到新磁盘组。

  • CONFLICT - 由于冲突未安装Oracle ASM磁盘

  • FOREIGN - 磁盘包含由Oracle ASM以外的Oracle产品创建的数据。这包括数据文件，日志文件和OCR磁盘。

MODE_STATUS

VARCHAR2（7）

有关允许哪种I / O请求进入磁盘的全局状态：

  • ONLINE - 磁盘在线并正常运行。安装磁盘时会尝试读取和写入。尝试读取作为磁盘发现的一部分。

  • OFFLINE - 磁盘处于脱机状态，不允许访问数据。不尝试读取和写入。脱机磁盘在逻辑上仍然是其磁盘组的一部分。

州

VARCHAR2（8）

磁盘组的全局磁盘状态：

  • UNKNOWN - Oracle ASM磁盘状态未知（通常未安装磁盘）

  • NORMAL - 磁盘处于联机状态且运行正常

  • ADDING - 磁盘正被添加到磁盘组，并且正在等待已安装磁盘组的所有实例的验证

  • DROPPING - 磁盘已手动脱机，磁盘的空间分配或数据访问停止。重新平衡将开始将数据从磁盘重定位到磁盘组中的其他磁盘。完成重新平衡后，磁盘将从组中排出。

  • HUNG - 磁盘丢弃操作无法继续，因为没有足够的空间从被删除的磁盘重新定位数据

  • FORCING - 正在从磁盘组中删除磁盘而不尝试卸载其数据。在可能的情况下，将从冗余副本中恢复数据。

冗余

VARCHAR2（7）

磁盘的硬件冗余：

  • 未知

  • UNPROT

  • 镜子

  • 平价

注意：仅当存在支持返回硬件冗余信息的ASMLIB时，此列才有效。此列与磁盘所属的磁盘组的冗余无关。

TOTAL_MB

数

FREE_MB

数

HOT_USED_MB

COLD_USED_MB

数

名称

FAILGROUP

你做了

产品

制造商名称和产品名称。具有相同产品ID的所有磁盘将具有相同的性能和可靠性特征。

CREATE_DATE

日期

磁盘添加到磁盘组的日期和时间

MOUNT_DATE

日期

第一个实例装入磁盘的日期和时间

REPAIR_TIMER

数

磁盘自动丢弃之前剩余的秒数（如果没有失败，则为0 ）

READS

数

磁盘的I / O读取请求总数

WRITES

数

磁盘的I / O写请求总数

READ_ERRS

数

磁盘的I / O读取请求失败总数

WRITE_ERRS

数

磁盘的I / O写入请求失败总数

READ_TIMEOUT

数

超时的读I / O数

WRITE_TIMEOUT

数

超时的写I / O数

READ_TIME

数

如果TIMED_STATISTICS初始化参数设置为true则为磁盘读取请求的总I / O时间（以秒为单位）（如果设置为false则为0 ）

WRITE_TIME

数

如果TIMED_STATISTICS初始化参数设置为true则为磁盘写入请求的总I / O时间（以秒为单位）（如果设置为false则为0 ）

bytes_read缓存

数

从磁盘读取的总字节数

BYTES_WRITTEN

数

写入磁盘的总字节数

PREFERRED_READ

VARCHAR2（1）

首选读取失败组的状态：

  • U - 磁盘组没有首选读取失败组

  • Y - 磁盘是首选的读取磁盘

  • N - 磁盘不是首选的读取磁盘

对于具有一个或多个首选读取故障组的磁盘组，如果磁盘位于其中一个首选读取故障组中，则此列的值为Y ;否则是N

HASH_VALUE

数

Oracle ASM磁盘的唯一哈希值，使用Oracle ASM磁盘名称（由v$asm_disk的NAME列指定）和Oracle ASM磁盘组名称（由v$asm_diskgroup的NAME列指定）计算v$asm_diskgroup 。

HOT_READS

数

磁盘上热区域的读取次数

HOT_WRITES

数

磁盘上热区域的写入次数

HOT_BYTES_READ

数

从磁盘上的热区读取的字节数

HOT_BYTES_WRITTEN

数

写入磁盘上热区的字节数

COLD_READS

数

从磁盘上的冷区读取的数量

COLD_WRITES

数

磁盘上冷区写入的次数

COLD_BYTES_READ

数

从磁盘上的冷区读取的字节数

COLD_BYTES_WRITTEN

数

写入磁盘上冷区的字节数

VOTING_FILE

VARCHAR2（1）

指示磁盘是否包含投票文件（ Y ）（ N ）

SECTOR_SIZE

数

物理块大小（以字节为单位）

LOGICAL_SECTOR_SIZE

数

以字节为单位显示磁盘的逻辑扇区大小值。这是磁盘可以完成的最小I / O.

如果LOGICAL_SECTOR_SIZE列的值为512 ，而SECTOR_SIZE列中的值为4096 ，则磁盘支持512扇区大小仿真。这意味着该磁盘可用于512个本机磁盘组和4096个本机磁盘组。

FAILGROUP_TYPE

VARCHAR2（7）

失败组的类型：

  • 定期

  • QUORUM

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

对于此视图，该值始终为0 。

THIN_PROVISION_CAPABLE

VARCHAR2（1）

指示磁盘是否支持存储精简配置（ Y ）（ N ）

DATA_INTEGRITY_CAPABLE

VARCHAR2（1）

指示磁盘是否支持数据完整性功能（ Y ）（ N ）

SITE_NAME

VARCHAR2（30）

特定磁盘所属站点的名称

SITE_GUID

VARCHAR2（33）

特定磁盘所属站点的GUID

FAILGROUP_LABEL

VARCHAR2（30）

这是磁盘所属的故障组的建议名称，由discovery返回

SITE_LABEL

VARCHAR2（30）

这是磁盘所属站点的建议名称，由discovery返回

SITE_STATUS

VARCHAR2（11）

在每个伙伴关系和状态表（PST）刷新之后或从磁盘读取PST到内存时计算站点状态。可能的状态包括：

  • UNKNOWN ：这是执行任何检查之前的状态。

  • UNAVAILABLE ：这是站点中所有磁盘都处于脱机状态的状态。

  • COMPROMISED ：当有足够的磁盘或PST副本脱机时的状态，这样该站点就无法再提供数据可用性。

  • AVAILABLE ：当站点内有足够的冗余以提供数据可用性以防其他站点受损或不可用时的状态。

注意：

GROUP_NUMBER和DISK_NUMBER列仅在磁盘是实例当前装入的磁盘组的一部分时才有效。否则， GROUP_NUMBER将为0 ，并且DISK_NUMBER将是相对于也具有组编号0的其他磁盘的唯一值。
