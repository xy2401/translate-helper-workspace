12.2 Oracle JVM支持JNDI的要求

本节介绍Oracle JVM中JNDI支持的实现要求。本节分为以下几个部分：

  • 命名空间

  • Oracle Java目录服务JNDI名称空间提供程序

  • 命名空间浏览器

12.2.1命名空间

命名空间的表示方式与典型的Unix文件系统结构类似。根目录和目录分隔符由斜杠符号（/）表示。根目录由SYS拥有，只有SYS可以在其下创建新的子目录。

在OJDS的安装过程中创建了以下两个目录（ DirContexts ）：

  • /public目录

    /public目录是用于测试的公共区域，任何用户都可以绑定，删除或查找此目录中的对象。

  • /etc目录

    /etc目录是用于部署客户端可能需要的所有生产类型对象的区域，并且不受任何更新或删除的影响。/etc目录只能由SYS用户写入，但所有用户都可以读取。

12.2.2 Oracle Java目录服务JNDI名称空间提供程序

本节介绍以下Oracle Java目录服务概念：

  • StateFactories

  • ObjectFactories

  • OJDS URL支持

  • 客户端类路径

12.2.2.1目录上下文

Oracle Java目录服务（OJDS）必须实现javax.naming.directory.指定的接口javax.naming.directory.DirContext上下文。javax.naming.directory.DirContext上下文， oracle.aurora.jndi.ojds.OjdsServerContext上下文和oracle.aurora.jndi.ojds.OjdsClientContext上下文提供了检查和更新与对象关联的属性的方法，并允许分别搜索目录以进行服务器端和客户端执行。

下表描述了可用于创建上下文或使用上下文的JNDI属性：

包裹名字 描述

指定用于为应用程序创建初始上下文的类。oracle.aurora.jndi.ojds包定义了oracle.aurora.jndi.ojds.OjdsInitialContextFactory与此属性一起使用以创建InitialDirContext 。

12.2.2.2国家工厂

StateFactory将Java对象转换为可以存储在实现JNDI提供程序中的对象。OJDS将所有对象转换为绑定到序列化对象。OJDS遵循的规范java.io.用于此转换的可Serializable接口和Java对象序列化规范。序列化后，对象存储在OJDS持久性存储中。StateFactories不支持外部StateFactories。

12.2.2.3 ObjectFactories

ObjectFactory获取存储在实现JNDI提供程序中的对象，并将它们转换回Java对象。OJDS不支持外部ObjectFactories 。序列化对象是从它们从OJDS持久性存储中检索的二进制形式创建的。对象反序列化后，OJDS通过以下方式之一处理对象：

  • 如果对象是Context ，则设置connection和env字段并返回DirContext 。

  • 如果对象是javax.naming.Reference ，然后您可以使用DirectoryManager.getObjectInstance方法来创建对象。

  • 如果对象既不是Context也不是javax.naming.Reference ，然后将对象原样返回给用户。

指定对象的检索字节必须符合java.io.Serializable接口标准。如果实现对象的类在客户端上发生更改，则对象的反序列化可能会失败。因此，必须小心保持对象字节与反序列化对象字节的类或对象流之间的兼容性。

12.2.2.4 OJDS URL支持

OJDS支持以下列格式指定的URL：

ojds：// jdbc_connection_url /路径... /对象

在上面的语法中：

  • jdbc_connection_url是受支持的JDBC连接URL之一。您必须在URL中指定jdbc_connection_url才能连接到该目录。

    注意：

    OJDS提供程序支持基于JDK的外部客户端的瘦URL和OCI URL。例如，您可以分别对瘦驱动程序和OCI驱动程序使用以下URL：

    瘦：localhost：5521：mysid oci：22.133.242：5521：mysid

    但是，服务器中的OJDS URL支持仅适用于瘦连接类型。您必须为Context.设置一个值Context.SECURITY_PRINCIPAL和Context.SECURITY_CREDENTIALS完成URL连接。

  • path是一个斜杠Foot 1 -separated list，类似于Unix类型的文件系统。这表示目录树中的节点。

  • object是上下文中的实际终端对象名称。如果省略该对象，则路径以斜杠（/）结束。在这种情况下，返回DirContext ，表示该路径为根。

例

以下代码段显示如何使用以用户HR连接的OCI驱动程序在目录/one/two MyObj类型的对象myobj ：

import javax.naming。*; 。。。Hashtable env = new Hashtable（）; env.put（“java.naming.security.principal”，“HR“）; @ env.put（“java.naming.security.credentials”，“ <password> ”）; MyObj obj =（MyObj）（new InitialDirContext（env））。lookupup（ojds：// oci：host1：5521：mysid /一/二/ MyObj中“）;

12.2.2.5客户端类路径

必须将$ORACLE_HOME/jdbc/lib/ojdbc6.jar和$ORACLE_HOME/javavm/lib/aurora.zip jar文件添加到类路径中，以便JDK客户端使用OJDS。
