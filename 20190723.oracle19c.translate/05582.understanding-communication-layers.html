4.1了解客户端/服务器应用程序的Oracle Net Stack通信

数据库服务器是管理数据库的Oracle软件，客户端是从服务器请求信息的应用程序。客户端和服务器通信的方式称为客户端/服务器堆栈。

从客户端通信栈通过网络协议发送的客户端应用程序传递的信息由数据库服务器侧的类似通信栈接收。数据库服务器端的流程与客户端的流程相反，信息通过通信层上升。

4.1.1.2表示层

如果客户端和数据库服务器在不同的操作系统上运行，则会出现字符集差异。表示层解决了任何差异。它针对每个连接进行了优化，以便在需要时执行转换。

客户端/服务器应用程序使用的表示层是 双任务共同（TTC） 。TTC在客户端和数据库服务器上提供不同字符集或格式之间的字符集和数据类型转换。在初始连接时，TTC负责评估内部数据和字符集表示的差异，并确定两台计算机之间是否需要进行转换。

4.1.1.3 Oracle Net Foundation Layer

该 Oracle Net基础层负责建立和维护客户端应用程序与数据库服务器之间的连接，以及在它们之间交换消息。由于透明网络基板（TNS）技术，Oracle Net基础层可以执行这些任务 。TNS为所有行业标准OSI传输和网络层协议提供单一，通用的接口。TNS支持对等应用程序连接，其中两台或多台计算机可以直接相互通信，而无需任何中间设备。

在客户端，Oracle Net基础层接收客户端应用程序请求并解决所有通用计算机级连接问题，例如：

  • 数据库服务器或目标的位置

  • 连接涉及多少个协议

  • 如何根据每个服务器的功能处理客户端和数据库服务器之间的中断

在服务器端，Oracle Net基础层执行与客户端相同的任务。它还与侦听器一起使用以接收传入的连接请求。

除了建立和维护连接之外，Oracle Net基础层还通过命名方法进行通信，以解析名称并使用安全服务来确保安全连接。

4.1.1.4 Oracle协议支持层

Oracle协议支持层位于Oracle Net基础层的最低层。它负责将TNS功能映射到客户端/服务器连接中使用的行业标准协议。该层支持以下网络协议：

  • TCP / IP（ IPv4和IPv6 ）

  • 使用SSL的TCP / IP

  • 命名管道

  • 套接字直接协议（SDP）

  • Exadirect

客户端/服务器连接过程中的所有Oracle软件都需要现有的网络协议堆栈，以便在传输层的两台计算机之间建立计算机级连接。网络协议负责将数据从客户端计算机传输到数据库服务器计算机，此时数据将传递到服务器端Oracle协议支持层。

4.4.1关于TCP / IP协议

TCP / IP（传输控制协议/ Internet协议）是用于通过网络进行客户端/服务器通信的标准通信协议套件。 TCP是管理主机之间数据交换的传输协议。IP是用于分组交换网络的网络层协议。

Oracle Net支持两个版本的IP：IP版本4（IPv4）和 IP版本6（IPv6）。IPv6解决了当前使用的IPv4的缺点。IPv6的主要好处是使用128位地址产生的大地址空间。

4.4.1.1 IPv6地址表示法

Oracle数据库支持RFC 2732指定的标准IPv6地址表示法。的128位IP地址一般表示为8组，每组4个十六进制数字，用冒号（ : ）符号组分隔。例如，以下地址采用有效的IPv6格式：

2001年：将0db8：0000：0000：0000：0000：200C：417A

地址中的每个十六进制数字代表4位，因此地址中的每个组代表16位。以下地址表示2001:0db8:0000:0000的第一个和最后一个主机2001:0db8:0000:0000子网：

2001:0db8:0000:0000 ：0000：0000：0000：0000 2001:0db8:0000:0000 ：FFFF：FFFF：FFFF：FFFF

在简写表示法中，可以使用双冒号（ :: :）分隔符压缩连续的零字段，如以下等效表示法所示：

2001：0db8：0：0 :: 200C：417A 2001：0db8 :: 200C：417A 2001：DB8 :: 200C：417A

4.4.1.1.1 CIDR表示法

无类别域间路由（CIDR）是一种将IP地址分组为独立于地址值的子网的方法。无类路由旨在克服IP类系统中地址空间的耗尽以及路由表大小的无法管理的增长。

CIDR表示网络中第一个地址的网络，以十进制表示网络前缀的比特大小，用斜杠（ / ）分隔。例如， 2001:0db8::/32表示地址的前32位标识网络，而其余位标识网络中的主机。

CIDR对IPv4地址使用类似表示法。例如，在符号192.0.2.0/24中，地址的前24位表示网络前缀。DBMS_NETWORK_ACL_ADMIN程序包提供用于管理访问控制列表的API，支持IPv6和IPv4地址和子网的CIDR表示法。

在URL中，IPv6地址由左括号（ [ ）和右括号（ ] ）括起。例如，IPv6地址[2001:0db8:0:0:8:800:200C:417A]构成以下URL的一部分：

4.4.1.1.3 IPv4映射地址

IPv4映射地址是IPv6地址的子类，其中满足以下条件：

  • 在标准IPv6表示法中，前80位设置为0

  • 在标准IPv6表示法中，第二个16位设置为1

  • 最后32位是IPv4表示法

IPv4映射地址可以将仅IPv4节点的地址表示为IPv6地址。

例4-1以不同的表示法显示了相同的IP地址。第一个地址使用标准IPv6表示法。第二个地址是IPv4映射地址，其中最后32位使用点分十进制IPv4表示法。最后一个地址使用简写表示法来压缩连续的零字段。

4.4.1.2 IPv6接口和地址配置

主机可能具有不同的IPv4和IPv6接口配置。主机可以使用以下配置：

  • 仅限IPv4接口，在这种情况下，主机是仅限IPv4的主机。

  • 仅限IPv6接口，在这种情况下，主机是仅IPv6主机。

  • IPv4和IPv6接口，在这种情况下主机是双栈主机。

单个主机也可以使用不同类型的IP地址。例如，域名服务器可以通过IPv4和IPv6地址或仅IPv6地址来关联双栈主机。不支持的IP地址配置如下：

  • 仅IPv4主机无法使用IPv6地址。

  • 仅IPv6主机不能使用IPv4地址。

4.4.2关于使用SSL协议的TCP / IP

具有安全套接字层（SSL）协议的TCP / IP使客户端上的Oracle应用程序能够通过TCP / IP和SSL与远程数据库进行通信。 SSL在Oracle Wallet中存储身份验证数据，例如证书和私钥。当客户端启动与数据库服务器的连接时，SSL使用证书在两者之间执行握手。在握手期间，会发生以下过程：

  • 客户端和数据库服务器协商由一组身份验证，加密和数据完整性类型组成的密码套件，以应用于它们交换的消息。

  • 根据其配置，数据库服务器在使用客户端公钥加密的消息中将其证书发送到客户端。数据库服务器还可以在同一消息中发送对客户端证书的请求。客户端使用自己的私钥解密此消息，然后验证数据库服务器的证书是否具有证书颁发机构的签名。

  • 如果需要，客户端可以将用户的证书发送到数据库服务器。证书确保用户的信息正确并且公钥实际上属于该用户。

数据库检查用户证书以验证它是否具有证书颁发机构的签名。

4.4.4关于套接字直接协议（SDP）

套接字直接协议（SDP）是InfiniBand网络对等体之间的行业标准有线协议。当通过InfiniBand网络使用时，SDP通过消除数据的中间复制并将大部分消息传递负担从CPU传输到网络硬件来减少TCP / IP开销。

父主题： 了解Oracle协议支持层

4.4.5关于Exadirect协议

Exadirect协议是一种用于低开销数据库访问的创新协议。通过在InfiniBand环境中利用远程直接内存访问（RDMA），使用新传输来改善延迟和吞吐量。

Exadirect协议使用TCP进行控制通信，使用IB RC传输数据。

此版本仅在Oracle Linux上支持Exadirect协议适配器。

父主题： 了解Oracle协议支持层

4.4.6关于Websocket协议

数据库客户端连接支持安全的websocket协议。安全Web套接字连接建立旨在通过HTTPS工作，以支持HTTPS代理和中间代理。此协议以最小的协议开销提供与数据库的本机连接。Secure websocket协议用于在数据库服务器和客户端之间建立连接期间协商SQL*Net协议。安全websocket协议使用TLS并需要钱包进行操作。

Web服务器应支持Web套接字隧道。Web服务器代理模块（在Apache或Oracle HTTP Server（OHS）的情况下， mod_proxy_wstunnel ））代理客户端和后端数据库服务器之间的安全Web套接字数据库连接数据。数据库客户端使用安全websocket协议连接到Web服务器，Web服务器使用websocket协议连接到数据库。
