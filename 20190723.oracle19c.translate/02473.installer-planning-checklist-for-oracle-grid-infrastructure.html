Oracle Grid Infrastructure的安装程序规划核对表

在启动Oracle Universal Installer之前，请查看用于规划Oracle Grid Infrastructure安装的核对表。

表1-8 Oracle Grid Infrastructure安装的Oracle Universal Installer核对表

校验 任务
阅读发行说明

查看适用于您的平台的发行说明，可在以下URL中找到您的发行版：

http://www.oracle.com/technetwork/indexes/documentation/index.html
查看许可信息

您只能使用已购买许可证的Oracle数据库媒体包中的那些组件。有关更多信息，请参阅：

Oracle数据库许可信息用户手册
使用CVU运行OUI并使用修复脚本

Oracle Universal Installer与Cluster Verification Utility（CVU）完全集成，可自动执行许多CVU先决条件检查。运行安装程序时，Oracle Universal Installer会运行所有先决条件检查并创建修订脚本。

您也可以手动运行CVU命令以检查系统准备情况。有关更多信息，请参阅：

“Oracle Clusterware管理和部署指南”
下载并运行Oracle ORAchk以进行运行时和升级检查或运行时运行状况检查

Oracle ORAchk实用程序提供的系统检查有助于防止安装后出现问题。这些检查包括内核要求，操作系统资源分配和其他系统要求。

使用Oracle ORAchk升级准备情况评估获取升级的自动升级特定系统运行状况检查。例如：

./orachk -u -o pre

Oracle ORAchk升级准备评估可自动执行Oracle升级文档中描述的许多手动升级前和升级后检查。

仅在Cygwin环境中的Windows平台上支持Oracle ORAchk。有关更多信息，请参阅：

https://support.oracle.com/rs?type=doc&id=1268927.1
确保在安装期间不运行cron作业 如果安装程序在每日cron作业启动时运行，那么如果您的cron作业正在执行清理，则可能会遇到无法解释的安装问题，并且在安装完成之前删除临时文件。Oracle建议您在运行每日cron作业之前完成安装，或者禁用执行清理的每日cron作业，直到安装完成为止。
获取您的My Oracle Support帐户信息

在安装过程中，您需要My Oracle Support用户名和密码来配置安全更新，下载软件更新和其他安装任务。您可以通过以下URL注册My Oracle Support：

https://support.oracle.com/
检查正在运行的Oracle进程，并在必要时关闭进程

  • 在具有不使用Oracle ASM的独立数据库的节点上：安装Oracle Grid Infrastructure时无需关闭数据库。

  • 在具有使用Oracle ASM的独立Oracle数据库的节点上：停止现有的Oracle ASM实例。安装期间将重新启动Oracle ASM实例。

  • 在Oracle RAC数据库节点上：此安装需要升级Oracle Clusterware，因为运行Oracle RAC需要Oracle Clusterware。作为升级的一部分，随着滚动升级从节点到节点，您必须一次关闭数据库一个节点。
