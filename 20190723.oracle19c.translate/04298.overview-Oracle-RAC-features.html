9.1 Oracle RAC功能概述

UCP JDBC连接池提供与各种Oracle Real Application Clusters（Oracle RAC）数据库功能的紧密集成。这些功能包括快速连接故障转移（FCF），运行时连接负载平衡和连接关联。这些功能需要使用Oracle客户端软件附带的Oracle JDBC驱动程序，Oracle RAC数据库和Oracle Notification Service库（ ons.jar ）。

应用程序使用Oracle RAC功能来最大化连接性能和可用性，并减少因连接问题导致的停机时间。应用程序具有不同的可用性和性能要求，因此应相应地实现Oracle RAC功能。

注意：

从Oracle Database 11g第1版（11.2）开始，单个实例数据库上的Oracle Restart也支持FCF。Oracle Restart以前称为单实例高可用性（SIHA）。

通用高可用性和性能功能

UCP API和连接池属性包括许多不需要Oracle RAC数据库的高可用性和性能功能。这些功能适用于Oracle和非Oracle连接，本指南将对此进行讨论。例如：验证借用的连接;设置超时属性;设置最大重用属性;和连接池管理器操作都用于确保高级别的连接可用性和最佳性能。

注意：

使用Oracle连接时，通用高可用性和性能功能稍微好一些，因为UCP利用了Oracle JDBC内部API。

Oracle RAC的数据库版本兼容性

下表列出了各种Oracle RAC功能支持的数据库版本：

表9-1 Oracle RAC版本兼容性

特征 支持的数据库版本

快速连接故障转移

Oracle Database 10.1.x及更高版本

运行时连接负载平衡

Oracle Database 10.2.x及更高版本

网络会话亲和力

Oracle Database 11.1.x及更高版本

基于交易的亲和力

Oracle Database 10.2.x及更高版本（建议使用Oracle Database 11.1.x）

Oracle RAC的Oracle JDBC驱动程序版本兼容性

Oracle RAC功能支持Oracle JDBC驱动程序10.1.x及更高版本。
