1.2自主健康框架的组成部分

本节介绍作为Oracle Autonomous Health Framework一部分的诊断组件。

  • Oracle ORAchk和Oracle EXAchk简介
    Oracle ORAchk和Oracle EXAchk为Oracle软件和硬件组件堆栈提供了轻量级且非侵入式的运行状况检查框架。
  • 群集运行状况监视器简介
    Cluster Health Monitor是Oracle Grid Infrastructure的一个组件，可持续监视和存储Oracle Clusterware和操作系统资源指标。
  • Oracle Trace File Analyzer简介
    Oracle跟踪文件分析器是一种用于目标诊断收集的实用程序，它还简化了Oracle Clusterware，Oracle Grid Infrastructure和Oracle Real Application Clusters（Oracle RAC）系统的诊断数据收集，以及单实例非群集数据库。
  • Oracle Cluster Health Advisor简介
    Oracle Cluster Health Advisor持续监视群集节点和Oracle RAC数据库的性能和可用性问题前兆，以便在问题变得严重之前提供问题的早期警告。
  • 记忆保护简介
    Memory Guard是一个Oracle Real Application Clusters（Oracle RAC）环境功能，用于监控集群节点，以防止由于内存不足而导致的节点压力。
  • Hang Manager简介
    Hang Manager是Oracle Real Application Clusters（Oracle RAC）环境功能，可自动解决挂起并保持资源可用。
  • Oracle数据库服务质量（QoS）管理简介
    Oracle数据库服务质量（QoS）管理管理跨应用程序共享的资源。
