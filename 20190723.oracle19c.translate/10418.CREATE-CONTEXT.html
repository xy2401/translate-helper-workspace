创建背景

目的

使用CREATE CONTEXT语句：

  • 为上下文创建名称空间（一组应用程序定义的属性，用于验证和保护应用程序）

  • 将命名空间与设置上下文的外部创建的包相关联

您可以使用DBMS_SESSION 。指定包中的SET_CONTEXT过程用于设置或重置上下文的属性。

也可以看看：

  • Oracle数据库安全指南，用于讨论上下文

  • 有关DBMS_SESSION信息，请DBMS_SESSION Oracle Database PL / SQL包和类型参考 。 SET_CONTEXT程序

先决条件

要创建上下文命名空间，您必须具有CREATE ANY CONTEXT系统特权。

请注意，您不能在CREATE CONTEXT命令中使用包名称的同义词。

句法

create_context :: =


说明create_context.eps的说明

语义

或替换

指定OR REPLACE以使用其他包重新定义现有上下文命名空间。

命名空间

指定要创建或修改的上下文命名空间的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。上下文命名空间始终存储在模式SYS 。

也可以看看：

“ 数据库对象命名规则 ”，用于命名上下文命名空间的指南

模式

指定架构拥有package 。如果省略schema ，则Oracle数据库使用当前模式。

包

指定用于设置或重置用户会话命名空间下的上下文属性的PL / SQL包。

为了提供一些设计灵活性，Oracle数据库在您创建上下文时不会验证模式的存在或程序包的有效性。

初始化条款

INITIALIZED子句允许您指定可以初始化上下文命名空间的Oracle数据库以外的实体。

EXTERNALLY

EXTERNALLY表示在建立会话时可以使用OCI接口初始化命名空间。

也可以看看：

Oracle Call Interface Programmer's Guide ，了解使用OCI建立会话的信息

GLOBALLY

GLOBALLY表示当全局用户连接到数据库时，LDAP目录可以初始化命名空间。

会话建立后，只有指定的PL / SQL包才能发出命令来写入命名空间内的任何属性。

也可以看看：

有关建立全局初始化上下文的信息，请参见“Oracle数据库安全指南”

全球访问

此子句指示在整个实例中可以访问namespace中设置的任何应用程序上下文。此设置允许多个会话共享应用程序属性。

例子

创建应用程序上下文：示例

此示例使用PL / SQL程序包emp_mgmt ，该程序包验证并保护人力资源应用程序。有关创建该程序包的示例，请参阅Oracle Database PL / SQL语言参考 。以下语句创建上下文命名空间hr_context并将其与包emp_mgmt相关联：

CREATE CONTEXT hr_context使用emp_mgmt;

您可以使用SYS_CONTEXT函数根据此上下文控制数据访问。例如， emp_mgmt包已将属性department_id定义为特定的部门标识符。您可以通过创建基于department_id值限制访问的视图来保护基表employees ，如下所示：

创建视图hr_org_secure_view AS SELECT * FROM employees WHERE department_id = SYS_CONTEXT（'hr_context'，'department_id'）;
