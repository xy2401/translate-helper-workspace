XMLDATA伪列

Oracle根据XMLSchema信息以及如何指定存储子句，将XMLType数据存储在LOB或对象关系列中。XMLDATA伪列允许您访问基础LOB或对象关系列，以指定其他存储子句参数，约束，索引等。

例

以下语句说明了此伪列的使用。假设您使用一个CLOB列创建一个简单的XMLType表：

XMLTYPE XMLTYPE STORE AS CLOB的CREATE TABLE xml_lob_tab;

要更改基础LOB列的存储特征，可以使用以下语句：

ALTER TABLE xml_lob_tab MODIFY LOB（XMLDATA）（STORAGE（MAXSIZE 2G）CACHE）;

现在假设您已经创建了一个基于XMLSchema的表，就像在SQL语句中使用XML中创建的xwarehouses表一样。然后，您可以使用XMLDATA列设置基础列的属性，如以下语句所示：

ALTER TABLE xwarehouses ADD（UNIQUE（XMLDATA。“WarehouseId”））;
