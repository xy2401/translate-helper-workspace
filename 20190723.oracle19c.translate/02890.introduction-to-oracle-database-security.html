1 Oracle数据库安全性简介

Oracle数据库提供了一组丰富的默认安全功能来管理用户帐户，身份验证，权限，应用程序安全性，加密，网络流量和审计。

  • 关于Oracle数据库安全性
    您可以使用默认的Oracle数据库功能在Oracle数据库安装的多个区域中配置安全性。
  • 其他Oracle数据库安全资源
    除了本指南中描述的安全资源之外，Oracle数据库还提供了其他几种数据库安全产品。

关于Oracle数据库安全性

您可以使用默认的Oracle数据库功能在Oracle数据库安装的多个区域中配置安全性。

您可以配置安全性的区域如下：

  • 用户帐户。 创建用户帐户时，可以通过多种方式保护用户帐户。您还可以创建密码配置文件，以更好地保护您网站的密码策略。管理Oracle数据库用户的安全性 ，介绍如何管理用户帐户。

  • 验证方法。 Oracle数据库提供了多种方法为用户和数据库管理员配置身份验证。例如，您可以在数据库级别，操作系统和网络上对用户进行身份验证。配置身份验证 ，描述Oracle数据库中的身份验证如何工另请参阅使用Microsoft Active Directory配置集中管理的用户 。

  • 特权和角色。 您可以使用特权和角色来限制用户对数据的访问。以下章节描述了如何管理权限和角色：

    • 配置权限和角色授权

    • 执行权限分析以查找权限使用

    • 管理Definer权利和祈求者权利的安全性

    • 管理PL / SQL包和类型中的细粒度访问

    • 在Enterprise Manager中管理多租户环境的安全性

  • 应用安全性。 创建数据库应用程序的第一步是确保它是正确安全的。管理应用程序开发人员的安全性，讨论如何将应用程序安全性纳入应用程序安全策略。

  • 用户会话信息使用应用程序上下 应用程序上下文是保存会话信息的名称 - 值对。您可以检索有关用户的会话信息，例如用户名或终端，并根据此信息限制该用户的数据库和应用程序访问。使用应用程序上下文来检索用户信息 ，介绍如何使用应用程序上下文。

  • 使用虚拟专用数据库在行和列级别上进行数据库访问。 虚拟专用数据库策略动态地将WHERE谓词嵌入到用户发出的SQL语句中。使用Oracle虚拟专用数据库控制数据访问 ，介绍如何创建和管理虚拟专用数据库策略。

  • 对不同类别的数据进行分类和保护。 您可以在数据库中找到包含敏感数据的所有表列（例如信用卡或社会保险号），对此数据进行分类，然后创建一个策略来保护给定类的整个数据。使用透明敏感数据保护 ，介绍如何创建透明敏感数据保护策略。

  • 网络数据加密。 手动加密数据 ，解释了如何使用DBMS_CRYPTO PL / SQL包在数据传输到网络时加密数据，以防止未经授权访问该数据。您可以为服务器和客户端配置本机Oracle Net Services数据加密和完整性 ，这在配置Oracle数据库本机网络加密和数据完整性中有所描述。

  • 瘦JDBC客户端网络配置。 您可以配置瘦Java数据库连接（JDBC）客户端以安全地连接到Oracle数据库。配置瘦JDBC客户端网络 ，提供详细信息。

  • 强认证。 您可以将数据库配置为对支持各种第三方身份验证服务的Oracle身份验证适配器使用强身份验证，包括带有数字证书的SSL。Oracle数据库提供以下强大的身份验证支持：

    • 集中身份验证和单点登录。

    • Kerberos的

    • 远程身份验证拨入用户服务（RADIUS）

    • 安全套接字层（SSL）

    以下章节介绍了强身份验证：

    • 强认证简介

    • 强身份验证管理工具

    • 配置RADIUS身份验证

  • 审核数据库活动。 您可以通用术语审核数据库活动，例如审核所有SQL语句，SQL权限，架构对象和网络活动。或者，您可以以精细的方式进行审核，例如在使用公司网络外部的IP地址时。本章还介绍了如何清除数据库审计跟踪。以下章节介绍如何配置和管理数据库审计。

其他Oracle数据库安全资源

除了本指南中描述的安全资源之外，Oracle数据库还提供了其他几种数据库安全产品。

这些产品如下：

  • Oracle高级安全性。 有关透明数据加密和Oracle数据编辑的信息，请参见“ Oracle数据库高级安全性指南” 。

  • Oracle Label Security。 Oracle Label Security将分类标签应用于数据，允许您过滤用户对行级数据的访问权限。有关Oracle Label Security的详细信息，请参阅“ Oracle Label Security管理员指南” 。

  • Oracle Database Vault。 Oracle Database Vault为您的敏感数据提供细粒度的访问控制，包括保护特权用户的数据。“Oracle Database Vault管理员指南”介绍了如何使用Oracle Database Vault。

  • Oracle Audit Vault和数据库防火墙。 Oracle Audit Vault和Database Firewall从Oracle数据库审计跟踪表，数据库操作系统审计文件和数据库重做日志等来源收集数据库审计数据。使用Oracle Audit Vault和Database Firewall，您可以创建有关可疑活动的警报，并创建有关特权用户更改，架构修改甚至数据级访问历史记录的报告。

  • Oracle企业用户安全性。 Oracle Enterprise User Security使您可以在企业级别管理用户安全性。“Oracle数据库企业用户安全性管理员指南”介绍了如何配置Oracle Enterprise User Security。

  • Oracle Enterprise Manager数据屏蔽和子集包。 Data Masking和Subsetting Pack通过虚拟数据不可逆转地替换原始敏感数据，有助于降低这种风险，从而可以与IT开发人员或离岸业务合作伙伴安全地共享生产数据。有关其他信息，请参见Oracle数据库测试指南

除了这些产品之外，您还可以访问Oracle Technology Network上的安全技术中心，查找有关Oracle数据库安全性的最新信息，例如新产品以及有关安全补丁和警报的重要信息。
