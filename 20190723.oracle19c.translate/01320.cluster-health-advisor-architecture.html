3.1 Oracle Cluster Health Advisor体系结构

Oracle Cluster Health Advisor在群集中的每个节点上作为高可用性群集资源（ ochad运行。

每个Oracle Cluster Health Advisor守护程序（ ochad ）都监视群集节点上的操作系统，并可选择监视节点上的每个Oracle Real Application Clusters（Oracle RAC）数据库实例。

图3-1 Oracle Cluster Health Advisor体系结构


“图3-1 Oracle Cluster Health Advisor体系结构”的说明

ochad守护程序从Cluster Health Monitor接收操作系统度量标准数据，并从内存映射文件中获取Oracle RAC数据库实例度量标准。守护程序不需要连接到每个数据库实例。此数据以及所选模型在Oracle Cluster Health Advisor的运行状况预测引擎中用于节点和每个受监视的数据库实例，以便每分钟多次分析其运行状况。
