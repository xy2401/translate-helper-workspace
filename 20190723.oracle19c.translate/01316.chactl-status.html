D.3 chactl状态

使用chactl status命令检查正在运行的目标的监控状态。

如果未指定任何参数，则chactl status命令将返回所有正在运行的目标的状态。

Oracle Cluster Health Advisor目标的监视状态可以是“ Monitoring 还是“ Not Monitoring 。chactl status命令显示四种类型的结果，具体取决于您是否指定了target和-verbose选项。

该命令的-verbose选项还显示指定目标中包含的目标的监视状态以及每个打印目标的执行模型的名称。chactl status命令仅显示具有正监视状态的目标。只有在命令行中明确指定了相应的目标时， chactl status命令才会显示负监视状态。

句法

chactl status {cluster | database [-db db_unique_name ]} [-verbose] [ -  help]

例子

  • 要显示正在监视的群集节点和数据库的列表：

    #chactl status监控节点rac1Node1，rac1Node2监控数据库SalesDB，HRdb

    注意：

    如果Oracle Cluster Health Advisor正在监视数据库的一个或多个实例，则即使数据库的某些实例未运行，也会显示监视状态的数据库。

  • 要显示Oracle Cluster Health Advisor的状态：

    $ chactl status Cluster Health Advisor服务处于脱机状态。

    在命令行上未指定目标或-verbose选项。Oracle Cluster Health Advisor未在群集的任何节点上运行。

  • 要显示群集节点和数据库的各种Oracle Cluster Health Advisor监视状态，请执行以下操作：

    $ chactl status database -db SalesDB监控数据库SalesDB
    $ chactl status database -db bogusDB Not Monitoring数据库bogusDB
    $ chactl status cluster监控节点rac1，rac2 Not Monitoring节点rac3

    要么

    $ chactl status cluster Cluster Health Advisor处于脱机状态

  • 要显示整个群集的详细Oracle Cluster Health Advisor监视状态，请执行以下操作：

    $ chactl status -verbose监控节点racNd1，racNd2，racNd3，racNd4使用模型MidSparc监控数据库HRdb2，实例HRdb2I1，HRdb2I2在服务器池SilverPool中使用模型M6监控数据库HRdb，实例HRdbI4，HRdbI6在服务器池SilverPool中使用模型M23使用模型TestM13监控数据库testHR，节点racN7上的实例inst3使用模型TestM14监控数据库testHR，节点racN8上的实例inst4

    如果未指定目标且指定了–verbose选项，则chactl status命令将显示数据库实例的状态和模型的名称。
