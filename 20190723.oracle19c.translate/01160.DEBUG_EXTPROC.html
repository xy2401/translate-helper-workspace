210 DEBUG_EXTPROC

DEBUG_EXTPROC包使您可以在会话中启动extproc代理。此实用程序包可以帮助您调试外部过程。

本章包含以下主题：

  • 安全模型

  • 操作说明

  • 规则和限制

  • DEBUG_EXTPROC子程序摘要

210.1 DEBUG_EXTPROC安全模型

您的Oracle帐户必须具有对程序包和CREATE LIBRARY权限的EXECUTE权限。

210.2 DEBUG_EXTPROC操作说明

这些操作说明适用于DEBUG_EXTPROC 。

要安装软件包，请运行脚本DBGEXTP 。 SQL 。

  • 在要调试“extproc”进程的Oracle USER中安装/加载此程序包。

  • 确保您对包DEBUG_EXTPROC具有执行权限

SELECT SUBSTR（OBJECT_NAME，1,20）FROM USER_OBJECTS WHERE OBJECT_NAME ='DEBUG_EXTPROC';

  • 只要您对程序包具有EXECUTE权限，就可以像任何其他用户一样安装此程序包。

    注意：

    这些说明假定您使用调试符号构建了共享库，以帮助进行调试过程。请检查C编译器手册页以获取相应的C编译器开关，以使用调试符号构建共享库。

安装包后，继续：

  • 通过连接到ORACLE通过SQL * Plus或OCI程序启动新的Oracle会话。

  • 执行程序DEBUG_EXTPROC 。 STARTUP_EXTPROC_AGENT在此会话中启动extproc代理程序;例如，执行DEBUG_EXTPROC 。 STARTUP_EXTPROC_AGENT ;不要退出此会话，因为它会终止extproc代理。

  • 确定为此会话启动的extproc代理的PID。

  • 使用调试器（例如，gdb，dbx或本机系统调试程序），加载extproc可执行文件并附加到正在运行的进程。

  • 在函数'pextproc'上设置断点，让调试器继续执行。

  • 现在，在首次执行DEBUG_EXTPROC的同一会话中执行外部过程。 STARTUP_EXTPROC_AGENT

  • 您的调试器现在应该在函数'pextproc'中断。此时，PL / SQL外部函数引用的共享库将被加载并且函数已解析。现在在C函数中设置断点，让调试器继续执行。

由于PL / SQL在运行时加载共享库，因此您使用的调试器可能会也可能不会自动跟踪共享库中的新符号。您可能必须发出一些调试器命令来加载符号（例如，gdb中的'share'）

  • 调试器现在应该在C函数中中断。它假设您已经使用调试符号构建了共享库。

  • 现在继续进行调试。

210.3规则和限制

DEBUG_EXTPROC仅适用于具有可附加到正在运行的进程的调试器的平台。

210.4 DEBUG_EXTPROC子程序摘要

STARTUP_EXTPROC_AGENT过程是唯一的DEBUG_EXTPROC子程序。

210.4.1 STARTUP_EXTPROC_AGENT过程

此过程在会话中启动extproc代理进程。这使您可以获取正在执行的进程的PID。需要此PID才能使用调试器附加到正在运行的进程。

句法

DEBUG_EXTPROC.STARTUP_EXTPROC_AGENT;
