6.1关于数据库存储结构

Oracle数据库由物理和逻辑结构组成。可以从操作系统查看和操作物理结构，例如在磁盘上存储数据的物理文件。

逻辑结构由Oracle数据库创建和识别，操作系统不知道。数据库中的主要逻辑结构（表空间）包含物理文件。应用程序开发人员或用户可能知道逻辑结构，但通常不了解此物理结构。数据库管理员（DBA）必须了解数据库的物理和逻辑结构之间的关系。

从物理角度来看，多租户容器数据库（CDB）与非CDB具有基本相同的结构，除了每个可插拔数据库（PDB）都有自己的一组表空间（包括它自己的SYSTEM和SYSAUX表空间）和数据文件。

6.1.1关于控制文件

控制文件跟踪数据库的物理组件。它是数据库用于查找数据库使用的所有其他文件的根文件。由于控制文件的重要性，Oracle建议对控制文件进行多路复用 ，或者具有多个相同的副本。 对于使用Oracle Database Configuration Assistant（DBCA）创建的数据库，将自动创建控制文件的两个副本并使其保持同步。

如果任何控制文件失败，则您的数据库将变为不可用。但是，如果您有控制文件副本，则可以关闭数据库并从副本重新创建失败的控制文件，然后重新启动数据库。另一个选项是从CONTROL_FILES初始化参数中删除失败的控制文件，并使用其余的控制文件重新启动数据库。

6.1.2关于联机重做日志文件

每个Oracle数据库都有一组两个或多个联机重做日志文件。联机重做日志文件集合统称为数据库的重做日志 。 重做日志由重做条目组成，也称为重做记录 。

联机重做日志存储对数据所做更改的副本。如果故障需要从备份恢复数据文件，则可以从联机重做日志文件中获取已还原数据文件中缺少的最新数据更改，因此工作永远不会丢失。联机重做日志文件用于在硬件，软件或介质发生故障后恢复数据库。为了防止涉及联机重做日志文件本身的故障，Oracle数据库可以复用联机重做日志文件，以便可以在不同磁盘上维护两个或多个相同的联机重做日志文件副本。

数据库的联机重做日志包含一组联机重做日志文件。组由在线重做日志文件及其多路复用副本组成。每个相同的副本都被视为该组的成员。每个组由一个数字定义，例如组1。

图6-2显示了具有三个联机重做日志组和每个组中有两个成员的数据库的配置。对于每个组，成员存储在单独的磁盘上以获得最大可用性。例如，组1的成员是重做日志文件A_LOG1和B_LOG1 。

6.1.3关于存档的重做日志文件

启用联机重做日志的归档后，Oracle数据库会将联机重做日志文件复制到其他位置，然后再覆盖它们。这些复制的文件称为归档重做日志文件 。 您可以存档到多个位置。

这些归档的重做日志文件扩展了可以保存并用于恢复的重做数据量。从备份到当前时间，需要存档的重做日志文件来恢复数据库的备份。可以为数据库启用或禁用存档，但Oracle强烈建议您启用存档。Oracle还建议您配置数据库以将归档重做日志文件写入快速恢复区域。

6.1.4关于回滚段

回滚段是用于在早期版本的Oracle数据库中跟踪数据库的撤消信息的数据库结构。现在，管理撤消信息的首选方法是使用撤消表空间。 有关更多信息，请参阅“ 管理撤消数据 ” 。

注意：

Oracle数据库使用SYSTEM回滚段来执行系统事务。它是在创建数据库时自动创建的，并且在实例启动时始终处于联机状态。它位于SYSTEM表空间中。您不需要执行任何操作来管理SYSTEM回滚段。

6.1.5关于数据文件

数据文件是存储数据库中数据的操作系统文件。数据以Oracle专有格式写入这些文件，其他程序无法读取。 Tempfiles是一类特殊的数据文件，仅与临时表空间相关联。

数据文件可以分解为以下组件：

  • 分割

    段包含特定类型的数据库对象。例如，表存储在表段中，索引存储在索引段中。数据文件可以包含许多段。

  • 程度

    范围是段内的连续数据块集。Oracle数据库以一个范围为单位为段分配空间。当段的现有扩展区已满时，数据库将为该段分配另一个扩展区。

  • 数据块

    数据块 （也称为数据库块 ）是数据库存储的最小I / O单位。范围由几个连续的数据块组成。数据库在创建数据库时使用默认块大小。

    创建数据库后，如果不重新创建数据库，则无法更改默认块大小。但是，可以创建块大小不同于默认块大小的表空间。

段，范围和数据块都是逻辑结构。只有Oracle数据库可以确定文件中有多少数据块。操作系统仅识别文件和操作系统块，而不识别Oracle数据库文件中的数据块数。每个数据块映射到一个或多个操作系统块。

6.1.6关于表空间

数据库分为称为表空间的逻辑存储单元，它将相关的逻辑结构（例如表，视图和其他数据库对象）分组。例如，可以将所有应用程序对象分组到单个表空间中，以简化维护操作。

表空间由一个或多个物理数据文件组成。分配给表空间的数据库对象存储在该表空间的物理数据文件中。

创建Oracle数据库时，已存在一些表空间，例如SYSTEM和SYSAUX 。

表空间提供了一种在存储上物理定位数据的方法。定义包含表空间的数据文件时，请指定这些文件的存储位置。例如，您可以将某个表空间的数据文件位置指定为指定的主机目录（表示某个磁盘卷）或指定的Oracle自动存储管理磁盘组。然后，分配给该表空间的任何模式对象都将位于指定的存储位置。表空间还提供备份和恢复单元。Oracle数据库的备份和恢复功能使您可以在表空间级别进行备份或恢复。

6.1.6.1本地管理的表空间
表空间内的空间管理涉及跟踪可用（空闲）和已用空间，以便在数据插入和删除期间有效分配空间。本地管理的表空间将空间分配信息保留在表空间内，而不是数据字典中，从而提供更好的性能。 默认情况下，Oracle数据库使用自动段管理将所有新创建的表空间设置为本地管理，这一功能可进一步提高性能。
6.1.6.2表空间类型
表空间有三种类型。 例如：

  • 常驻

    您使用永久表空间来存储您的用户和应用程序数据。Oracle数据库使用永久表空间来存储永久数据，例如系统数据。为每个用户分配一个默认的永久表空间。

  • 解开

    以自动撤消管理模式运行的数据库透明地创建和管理撤消表空间中的撤消数据。Oracle数据库使用撤消数据回滚事务，提供读取一致性，帮助恢复数据库，以及启用Oracle闪回查询等功能。数据库实例只能有一个活动的撤消表空间。

  • 临时

    临时表空间用于存储临时数据，这将在SQL语句执行排序操作时创建。创建数据库时，Oracle数据库会获取临时表空间。如果要创建临时表空间组，则可以创建另一个临时表空间。在典型情况下，您不必创建其他临时表空间。如果您有一个非常大的数据库，那么您可以配置其他临时表空间。

    组成临时表空间的物理文件称为tempfiles，而不是数据文件。

    TEMP表空间通常用作未明确分配临时表空间的用户的默认临时表空间。

6.1.6.3表空间状态
您可以设置表空间状态。 例如：

  • 读写

    用户可以在创建表空间后对其进行读写。这是默认值。

  • 只读

    如果创建表空间为只读，则在将状态更改为“读写”之前，无法写入表空间。您不太可能创建只读表空间，但在向其写入不需要修改的数据后，可能会将其更改为该状态。

  • 离线

    如果表空间的状态为“脱机”，则没有用户可以访问它。在对与该表空间关联的数据文件执行维护或恢复之前，您可以将表空间的状态更改为“脱机”。

6.1.6.4自动扩展表空间
您可以将表空间设置为在达到其大小限制时自动将其自身扩展指定的数量。如果未启用autoextend，则会在表空间达到其临界或警告阈值大小时收到警报。 关键和警告阈值参数具有您可以随时更改的默认值。这些参数还会导致为自动扩展表空间生成警报，这些表空间接近其指定的大小限制。您可以通过手动增加表空间大小来响应大小警报。您可以通过增加一个或多个表空间数据文件的大小或通过向表空间添加另一个数据文件来实现此目的。

注意：

虽然将表空间称为自动扩展是很常见的，但自动扩展是数据文件属性，而不是表空间属性。也就是说，当您创建包含表空间的数据文件时，您可以指示这些数据文件是否自动扩展。具有自动扩展数据文件的表空间被视为自动扩展表空间。您可以为自动扩展数据文件指定最大大小。

6.1.6.5加密表空间
加密表空间主要通过数据库以外的方式保护您的数据免受未经授权的访问。 例如，当加密的表空间写入备份介质以便从一个Oracle数据库传输到另一个Oracle数据库或者前往场外设施进行存储时，它们仍然是加密的。此外，加密表空间可保护试图绕过数据库安全功能并直接通过操作系统文件系统访问数据库文件的用户的数据。

您可以加密任何永久表空间以保护敏感数据。加密表空间时，所有表空间块都会加密。支持加密所有段类型，包括表，集群，索引，LOB，表和索引分区等。表空间加密对您的应用程序完全透明，因此无需修改应用程序。

6.1.7关于其他存储结构

Oracle数据库中可以存在的其他存储结构包括初始化参数文件，密码文件和备份文件。

初始化参数文件

初始化参数在启动时由Oracle实例用于确定数据库的运行时属性和资源。在数据库运行时，可以设置或修改某些参数。其他初始化参数需要重新启动数据库才能使更改生效。请参阅“ 查看和修改初始化参数” 。

密码文件

数据库可以使用密码文件来对管理用户进行身份验证 SYSDBA ， SYSOPER和SYSBACKUP 特权。使用任何这些权限远程连接到数据库都需要密码文件。这些特权使DBA能够启动和关闭数据库，备份和恢复数据库，以及执行其他高级管理任务。此密码文件位于数据库本身之外，因此在数据库尚未启动时可以对DBA进行身份验证。（DBA必须在启动数据库之前进行身份验证。）

当您在Oracle数据库安装过程中调用DBCA时，DBCA会创建一个包含一个条目的密码文件： SYS用户。向用户授予SYSDBA ， SYSOPER或SYSBACKUP会自动将该用户添加到密码文件中。

注意：

Oracle数据库还可以使用操作系统身份验证来使用SYSDBA ， SYSOPER和SYSBACKUP权限对用户进行身份验证。操作系统身份验证优先于密码文件身份验证。请参阅“ 启动SQL * Plus并连接到数据库” 。

备份文件

备份文件在技术上不是数据库文件，而是以某种形式的数据库副本，如果故障导致数据丢失，可以用于恢复数据库。

6.5.1关于撤消数据

当事务修改数据时，Oracle数据库会在修改原始数据之前复制它。修改后的数据的原始副本称为撤消数据 。 由于以下原因，保存此信息是必要的：

  • 如果需要回滚，则撤消对数据库所做的任何未提交的更改。可能需要回滚，因为用户想要撤消误导或无意的事务的更改，或者它可以是恢复操作的一部分。

  • 提供读取一致性 ，这意味着每个用户都可以获得一致的数据视图，即使可能正在对数据进行其他更改。使用读取一致性，用户会话不会看到在其他用户会话中进行的未提交的更改（有时称为脏读 ）。例如，如果用户在上午10:00发出查询并且查询持续15分钟，则查询结果将在上午10:00反映数据的整个状态，而不管其他用户在执行更新或插入操作之后查询已开始。

  • 启用某些Oracle闪回功能，例如Oracle闪回查询和Oracle闪回表，使您可以查看或恢复以前某个时间点的数据。

撤消表空间

通过自动撤消管理，撤消数据存储在撤消表空间。撤消表空间具有超出永久表空间的其他属性。可以有多个撤消表空间，但只有一个可以为Oracle实例激活。

使用Database Configuration Assistant（DBCA）创建数据库时，它会创建一个名为UNDOTBS1的自动扩展撤消表空间，最大扩展名为32,767 MB。

撤消保留

Oracle数据库自动确保在提交事务之前，永远不会覆盖活动事务正在使用的撤消数据。提交事务后，可以重用或覆盖该撤消数据占用的空间。在这种情况下，如果撤消表空间中的空间变得稀缺，则可以覆盖撤消数据。

即使在提交了事务之后，保留（不覆盖）其撤消数据，确保Oracle闪回功能的成功以及长时间运行的查询的读取一致性也很有用。为此，数据库维护并自动调整撤消保留期 。保留年龄小于撤消保留期的已承诺撤消数据，供查询或Oracle闪回操作使用。

6.5.2关于管理撤消数据

虽然默认情况下Oracle数据库会自动管理撤消数据和撤消表空间，但如果您的安装使用Oracle闪回功能，则可能必须执行一些撤消管理任务以确保这些操作成功。

导致snapshot too old错误的Oracle闪回操作表明您必须进行干预以确保保留足够的撤消数据以支持这些操作。

以下方法更好地支持Oracle闪回操作：

  • 将自动扩展表空间的最小撤消保留期设置为与预期最长的Oracle闪回操作一样长。

    您可以通过设置UNDO_RETENTION初始化参数来实现此目标。有关详细信息，请参见Oracle数据库管理员指南 。

  • 将撤消表空间更改为固定大小。

    对于自动扩展的撤消表空间，Oracle数据库始终会自动调整撤消保留期，使其略长于运行时间最长的活动查询。但是，此自动调整的保留期可能不足以适应Oracle闪回操作。如果撤消表空间已禁用自动扩展或具有固定大小，则Oracle数据库使用不同的方法来调整撤消保留期以更好地适应Oracle闪回操作。

    要将撤消表空间更改为固定大小，必须选择足够大的表空间大小。如果选择的撤消表空间大小太小，则可能发生以下错误：

    • DML可能会失败，因为没有足够的空间来容纳新事务的撤消数据。

    • 长时间运行的查询可能会失败，一个snapshot too old的错误，这意味着有读一致性还原数据不足。

6.5.4使用Undo Advisor计算最小撤消表空间大小

如果必须将撤消表空间更改为固定大小，请使用Oracle Enterprise Manager Database Express（EM Express）中的“撤消顾问”来帮助确定所需的最小大小。您还可以使用Undo Advisor设置最小撤消保留期。

要将undo表空间配置为具有固定大小，Oracle建议您首先在数据库创建后留出足够的时间来运行完整工作负载，从而允许undo表空间增长到其处理工作负载所需的最小大小。然后，您可以使用Undo Advisor确定配置撤消表空间的最佳大小，以允许将来长时间运行的查询和Oracle闪回操作。

要使用Undo Advisor计算最小撤消表空间大小：

  1. 在EM Express中，转到“撤消管理详细信息”页面。

    请参阅“ 查看撤消信息 ” 。

  2. 在“配置”部分中，查看“撤消顾问”图。

    插图描述exp_undo_advisor.gif

    撤消顾问显示橙色线。您可以在橙色线上单击几个刻度线或点。橙色线上的蓝点标记当前为数据库设置的目标撤消保留时间。

  3. 要确定特定撤消保留时间的撤消表空间的建议最小大小，请选择最接近所需撤消保留时间的橙色线上的点。

    使用Undo Advisor获取特定撤消保留时间所需的最小表空间大小的建议不会更改撤消表空间的大小。您必须按照“ 将表空间更改为固定大小 ”中的说明更改撤消表空间的大小。

6.5.5将撤消表空间更改为固定大小

将撤消表空间更改为固定大小，以防止表空间过大或更好地支持Oracle闪回操作。您可以使用Oracle Enterprise Manager Database Express（EM Express）将撤消表空间更改为固定大小。

注意：

只有bigfile表空间可以更改为固定大小。但是，smallfile表空间的各个数据文件可以更改为固定大小。有关将smallfile表空间的单个数据文件更改为固定大小的说明，请参阅“ 将撤消表空间的数据文件更改为固定大小”。

要将撤消表空间更改为固定大小：

  1. 在EM Express中，转到“撤消管理详细信息”页面，如“ 查看撤消信息 ”中所述 。
  2. 确定所需的最小撤消表空间大小后，单击“表空间”部分中“ 名称”字段后面的链接。

    将出现“表空间”页面，其中显示了撤消表空间。

  3. 选择撤消表空间。然后从“ 操作”菜单中选择“ 调整大小” 。

    将显示“调整大小表空间”页面。

  4. 在“ 文件大小”字段中，输入撤消表空间的计算最小大小。

    请参阅“ 使用撤消顾问计算最小撤消表空间大小 ” 。

  5. 单击确定 。

    出现确认消息。

6.5.6将撤消表空间的数据文件更改为固定大小

您可以将撤消表空间的数据文件更改为固定大小，以防止表空间过大或更好地支持Oracle闪回操作。您可以使用Oracle Enterprise Manager Database Express（EM Express）将撤消表空间的数据文件更改为固定大小。

要将撤消表空间的数据文件更改为固定大小：

  1. 在EM Express中，转到“撤消管理详细信息”页面，如“ 查看撤消信息 ”中所述 。
  2. 确定所需的最小撤消表空间大小后，单击“表空间”部分中“ 名称”字段后面的链接。

    将出现“表空间”页面，其中显示了撤消表空间。

  3. 选择并展开撤消表空间。为撤消表空间选择一个数据文件。然后从“ 操作”菜单中选择“ 调整大小” 。

    将显示“调整大小数据文件”页面。

  4. 在“ 大小”字段中，输入撤消表空间的计算最小大小。

    请参阅“ 使用撤消顾问计算最小撤消表空间大小 ” 。

  5. 单击确定 。

    出现确认消息。

  6. 您可以对撤消表空间的所有数据文件执行这些步骤，以将它们全部更改为固定大小。

6.5.7更改撤消管理分析参数

您可以使用Oracle Enterprise Manager Database Express（EM Express）更改当前分析周期和所需的撤消保留期。

要更改分析期和撤消保留期：

  1. 在EM Express中，转到“撤消管理详细信息”页面，如“ 查看撤消信息 ”中所述 。
  2. 单击更改分析参数 。

    将显示“更改分析参数”页面。

  3. 要更改分析周期，请在“ 分析周期”字段中选择所需的分析周期。
  4. 要更改撤消保留期，请选择以下任一项：

    • 使用必需的撤消保留

    • 以秒为单位指定撤消保留

      在文本框中输入用于保留期的秒数。

6.5.8切换撤消表空间

您可以使用Oracle Enterprise Manager Database Express（EM Express）从一个还原表空间切换到另一个表空间。

要切换撤消表空间：

  1. 在EM Express中，转到“撤消管理详细信息”页面，如“ 查看撤消信息 ”中所述 。
  2. 单击“ 切换撤消表空间” 。

    将显示“切换撤消表空间”页面。

  3. 在“ 切换到撤消表空间”字段中，选择要切换到的撤消表空间的名称。该字段包括数据库的可用撤消表空间的名称。

    例如，如果当前的撤消表空间名为UD1，并且您要切换到名为UD2的撤消表空间，请在“ 切换到撤消表空间”字段中选择“ UD2 ”。

  4. 单击确定。

    出现确认消息。
