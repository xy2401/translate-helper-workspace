创建用于部署Web服务器的用户操作

您可以使用Fleet Patching和Provisioning用户操作来安装和配置任何类型的软件。查看此过程以使用Fleet Patching和Provisioning自动部署Apache Web Server。

Fleet Patching和Provisioning支持创建工作流程来部署和管理所有类型的软件。
  1. 创建脚本以安装Apache Web服务器：
    1. 在Fleet Patching和Provisioning Server上，下载并解压缩Apache Web服务器安装工具包。
    2. 创建脚本以安装，配置和启动Apache Web服务器。
  2. 使用Fleet Patching and Provisioning将脚本注册为用户操作。从Fleet Patching and Provisioning Server运行以下命令：
    rhpctl useraction -useraction apachestart -actionscript /user1/useractions/apacheinstall.sh -post -optype ADD_WORKINGCOPY -onerror ABORT

    该命令为存储在指定目录中的操作脚本添加apachestart用户操作。根据指定的属性，用户操作在ADD_WORKINGCOPY操作之后运行，如果有任何错误则中止。

  3. 创建图像类型并将用户操作与图像类型相关联：
    rhpctl add imagetype -imagetype apachetype -basetype SOFTWARE -useraction“apachestart”

    该命令创建一个名为apachetype的新图像类型，它是基本图像类型SOFTWARE的衍生物，具有关联的用户操作apacherstart 。

  4. 创建图像类型的黄金图像：
    rhpctl import image -image apacheinstall -path / user1 / apache2219_kit / -imagetype apachetype

    该命令在基于先前创建的imagetype的指定路径中创建一个带有Apache Web服务器安装脚本的黄金映像apacheinstall 。

    要查看此映像的属性，请运行rhpctl query image -image apacheinstall命令。
  5. 在目标上部署黄金映像的工作副本：
    rhpctl add workingcopy -workingcopy apachecopy -image apacheinstall -path / user1 / apacheinstallloc -sudouser user1 -sudopath / usr / local / bin / sudo -node node1 -user user1 -useractiondata“/user1/apachehome:1080:2.2.19

    Fleet Patching和Provisioning将软件配置到目标并运行用户操作中指定的脚本apachestart 。您可以使用useractiondata选项提供Apache Web服务器配置详细信息，例如端口号。如果目标是Fleet Patching and Provisioning Client，则无需指定sudo凭据。

您可以使用Fleet Patching and Provisioning创建其他用户操作并自动安装和配置软件之家。
