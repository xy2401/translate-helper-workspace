BFILENAME

句法


说明bfilename.eps的说明

目的

BFILENAME返回与服务器文件系统上的物理LOB二进制文件关联的BFILE定位符。

  • ' directory '是一个数据库对象，用作文件实际所在的服务器文件系统上的完整路径名的别名。

  • ' filename '是服务器文件系统中文件的名称。

必须先创建目录对象并将BFILE值与物理文件关联，然后才能将它们用作SQL或PL / SQL语句， DBMS_LOB包或OCI操作中BFILENAME参数。

您可以通过两种方式使用此功能：

  • 在DML语句中初始化BFILE列

  • 在程序界面中，通过为BFILE定位器赋值来访问BFILE数据

directory参数区分大小写。您必须确保指定与数据字典中存在的目录对象名称完全相同的目录对象名称。例如，如果在CREATE DIRECTORY语句中使用混合大小写和带引号的标识符创建"Admin"目录对象，则在使用BFILENAME函数时，必须将目录对象称为'Admin' 。您必须根据操作系统的大小写和标点符号约定指定filename参数。

  • Oracle Database SecureFiles和Large Objects Developer's Guide and Oracle Call Interface Programmer's Guide ，有关LOB和检索BFILE数据的示例的更多信息

例子

以下示例将一行插入到示例表pm.print_media 。该示例使用BFILENAME函数在目录/demo/schema/product_media标识服务器文件系统上的二进制文件。该示例显示了如何在pm模式中创建目录数据库对象media_dir 。 

CREATE DIRECTORY media_dir AS'/ demo / schema / product_media'; INSERT INTO print_media（product_id，ad_id，ad_graphic）VALUES（3000,31001，BFILENAME（'MEDIA_DIR'，'modem_comp_ad.gif'））;
