209 DBMS_XSTREAM_AUTH

DBMS_XSTREAM_AUTH包提供了用于向XStream管理员授予权限和撤消权限的子程序。

GRANT_ADMIN_PRIVILEGE程序

209.1 DBMS_XSTREAM_AUTH概述

此程序包提供子程序，用于向XStream管理员授予权限以及从XStream管理员撤消权限。

209.2 DBMS_XSTREAM_AUTH安全模型

可以通过将此程序包上的EXECUTE授予选定的用户或角色，或者通过向所选用户或角色授予EXECUTE_CATALOG_ROLE来控制此程序包的安全性。

执行DBMS_XSTREAM_AUTH包中的子程序的用户必须具有SYSDBA管理权限，并且用户必须在连接时使用AS SYSDBA行使权限。

如果程序包中的子程序是从存储过程中运行的，则必须直接为程序包授予运行子程序的用户EXECUTE特权。它不能通过角色授予。

要确保在此程序包中运行子程序的用户具有必要的权限，请以管理用户身份进行连接，以便在使用此程序包时创建用户，授予权限和创建表空间。

209.3 DBMS_XSTREAM_AUTH子程序摘要

该表列出了DBMS_XSTREAM_AUTH子程序并简要描述了它们。

表209-1 DBMS_XSTREAM_AUTH包子程序

子程序 描述

GRANT_ADMIN_PRIVILEGE程序

将用户所需的权限直接授予XStream管理员，或生成授予这些权限的脚本

GRANT_REMOTE_ADMIN_ACCESS过程

通过使用数据库链接连接到被授予者，使远程XStream管理员能够在本地数据库上执行管理操作

209.3.1 GRANT_ADMIN_PRIVILEGE过程

此过程要么直接授予用户成为XStream管理员所需的权限，要么生成授予这些权限的脚本。

也可以看看：

“ GRANT_ADMIN_PRIVILEGE程序 ”

句法

DBMS_XSTREAM_AUTH.GRANT_ADMIN_PRIVILEGE（grantee IN VARCHAR2，privilege_type IN VARCHAR2 DEFAULT'*'，grant_select_privileges IN BOOLEAN DEFAULT FALSE，do_grants IN BOOLEAN DEFAULT TRUE，file_name IN VARCHAR2 DEFAULT NULL，directory_name IN VARCHAR2 DEFAULT NULL grant_optional_privileges IN VARCHAR2 DEFAULT NULL，container IN VARCHAR2 DEFAULT'当前'）;

参数

表209-2 GRANT_ADMIN_PRIVILEGE过程参数

参数 描述

受让人

授予权限的用户

privilege_type

指定以下值之一：

  • 捕获

    指定CAPTURE授予用户管理捕获进程所需的最低权限。

  • 应用

    指定APPLY授予用户管理出站服务器，入站服务器和应用进程所需的最低权限。

  • *

    指定*授予用户管理捕获进程，出站服务器，入站服务器和应用进程所需的最低权限。

grant_select_privileges

如果为TRUE ，则该过程向用户授予一组权限，包括SELECT_CATALOG_ROLE 。

如果为FALSE ，则该过程不会向用户授予该组权限。

SELECT_CATALOG_ROLE使用户能够从数据字典中进行选择。对于XStream受信任用户模型，请将此参数设置为TRUE 。对于XStream不受信任的用户模型，请将此参数设置为FALSE 。

do_grants

如果为TRUE ，则该过程直接向指定的被授予者授予权限，并将该被授予者添加到DBA_XSTREAM_ADMINISTRATOR数据字典视图中，并为LOCAL_PRIVILEGES列和ACCESS_FROM_REMOTE列添加YES 。如果用户在此数据字典视图中已有条目，则该过程不会生成另一个条目，并且不会引发错误。如果为TRUE且任何授权语句失败，则该过程会引发错误。

如果为FALSE ，则该过程不直接向指定的被授予者授予权限，并且不将被授予者添加到DBA_XSTREAM_ADMINISTRATOR数据字典视图中。

当过程生成稍后将运行的文件时，请指定FALSE 。如果指定FALSE且file_name或directory_name参数为NULL ，则该过程会引发错误。

文件名

过程生成的文件的名称。该文件包含授予权限的所有语句。如果指定的目录名中存在具有指定文件名的文件，则grant语句将附加到现有文件。

如果为NULL ，则该过程不生成文件。

目录名

生成文件所在的目录。指定的目录必须是使用SQL语句CREATE DIRECTORY创建的目录对象。如果指定目录，则调用该过程的用户必须对目录对象具有WRITE特权。

如果t他file_name参数为NULL ，则忽略此参数，并且该过程不会产生一个文件。

如果NULL和file_name参数为非NULL ，则该过程会引发错误。

grant_optional_privileges

用于授予被授予者的可选权限的逗号分隔列表，例如DV_XSTREAM_ADMIN和DV_GOLDENGATE_ADMIN权限

容器

如果是CURRENT ，则仅在调用过程的容器中向受让者授予权限。可以在连接到根或PDB时指定CURRENT 。

如果为ALL ，则将权限授予CDB中所有容器中的被授予者以及调用该过程后创建的所有PDB。要指定ALL ，必须在根中调用该过程。

如果是容器名称，则仅在指定容器中向受让者授予权限。要指定root，请在连接到root时使用CDB$ROOT 。要指定PDB，必须在根中调用该过程。

注意：此参数仅适用于CDB。

使用说明

运行该过程的用户必须是可以向其他用户授予权限的管理用户。

具体而言，该过程向指定用户授予以下权限：

  • RESTRICTED SESSION系统特权

  • EXECUTE以下包：

    • DBMS_APPLY_ADM

    • DBMS_AQ

    • DBMS_AQADM

    • DBMS_AQIN

    • DBMS_AQELM

    • DBMS_CAPTURE_ADM

    • DBMS_FLASHBACK

    • DBMS_LOCK

    • DBMS_PROPAGATION_ADM

    • DBMS_RULE_ADM

    • DBMS_TRANSFORM

    • DBMS_XSTREAM_ADM

  • 将消息排入任何队列并将消息从任何队列中出列的权限

  • 管理任何队列的权限

  • 在用户自己的模式和其他模式中创建，更改和执行以下任何类型的对象的权限：

    • 评估背景

    • 规则集

    • 规则

    此外，被授予者可以将这些权限授予其他用户。

  • SELECT_CATALOG_ROLE

  • 与XStream和Oracle Replication相关的数据字典视图的SELECT或READ特权

  • 允许远程XStream管理员通过连接到被授权者通过数据库链接执行管理操作的功能

    通过在此程序包中运行GRANT_REMOTE_ADMIN_ACCESS过程来启用此功能。

    注意：

    • 要详细查看过程运行的所有语句，可以使用该过程生成脚本，然后在文本编辑器中查看脚本。

    • 此过程仅授予配置和管理XStream环境所需的权限。如有必要，您可以向受让人授予其他权限。

209.3.2 GRANT_REMOTE_ADMIN_ACCESS过程

此过程使远程XStream管理员可以通过使用数据库链接连接到被授予者来在本地数据库上执行管理操作。

句法

DBMS_XSTREAM_AUTH.GRANT_REMOTE_ADMIN_ACCESS（grantee IN VARCHAR2）;

参数

表209-3 GRANT_REMOTE_ADMIN_ACCESS过程参数

参数 描述

受让人

允许远程访问的用户。该过程将被授予者添加到DBA_XSTREAM_ADMINISTRATOR数据字典视图中，并为ACCESS_FROM_REMOTE列添加YES 。如果用户在此数据字典视图中已有条目，则该过程不会再生成另一个条目。相反，它将ACCESS_FROM_REMOTE列更新为YES 。

使用说明

通常，如果下游捕获进程捕获源自本地源数据库的更改，则运行该过程并在本地源数据库中指定被授予者。下游捕获数据库的XStream管理员使用此连接管理源数据库。

注意：

此程序包中的GRANT_ADMIN_PRIVILEGE过程运行此过程。

209.3.3 REVOKE_ADMIN_PRIVILEGE程序

此过程直接从用户撤消XStream管理员权限，或生成撤消这些权限的脚本。

句法

DBMS_XSTREAM_AUTH.REVOKE_ADMIN_PRIVILEGE（grantee IN VARCHAR2，privilege_type IN VARCHAR2 DEFAULT'*'，revoke_select_privileges IN BOOLEAN DEFAULT FALSE，do_revokes IN BOOLEAN DEFAULT TRUE，file_name IN VARCHAR2 DEFAULT NULL，directory_name IN VARCHAR2 DEFAULT NULL revoke_optional_privileges IN VARCHAR2 DEFAULT NULL，container IN VARCHAR2 DEFAULT'当前'）;

参数

表209-4 REVOKE_ADMIN_PRIVILEGE过程参数

参数 描述

受让人

撤消权限的用户

privilege_type

指定以下值之一：

  • 捕获

    指定CAPTURE撤消用户管理捕获进程所需的最低权限。

  • 应用

    指定APPLY撤消用户管理出站服务器，入站服务器和应用进程所需的最低权限。

  • *

    指定*撤消用户管理捕获进程，出站服务器，入站服务器和应用进程所需的最低权限。

revoke_select_privileges

如果为TRUE ，则该过程向用户撤消一组权限，包括SELECT_CATALOG_ROLE 。

如果为FALSE ，则该过程不会撤消对用户的权限集。

SELECT_CATALOG_ROLE使用户能够从数据字典中进行选择。

do_revokes

如果为TRUE ，则该过程直接撤消指定用户的权限，并从DBA_XSTREAM_ADMINISTRATOR数据字典视图中删除该用户。如果用户在此数据字典视图中没有记录，则该过程不会从视图中删除记录，也不会引发错误。如果为TRUE且任何revoke语句失败，则该过程会引发错误。如果未授予用户被撤销的权限，则撤消语句将失败。

如果为FALSE ，则该过程不会直接撤消指定用户的权限，也不会从DBA_XSTREAM_ADMINISTRATOR数据字典视图中删除该用户。

当过程生成稍后将运行的文件时，请指定FALSE 。如果指定FALSE并且file_name或directory_name参数为NULL ，则该过程不会引发错误。

文件名

此过程生成的文件的名称。该文件包含撤消权限的所有语句。如果指定的目录名中存在具有指定文件名的文件，则revoke语句将附加到现有文件中。

如果为NULL ，则该过程不生成文件。

目录名

生成文件所在的目录。指定的目录必须是使用SQL语句CREATE DIRECTORY创建的目录对象。如果指定目录，则调用该过程的用户必须对目录对象具有WRITE特权。

如果file_name参数为NULL ，则忽略此参数，并且该过程不会生成文件。

如果NULL和file_name参数为非NULL ，则该过程会引发错误。

revoke_optional_privileges

以逗号分隔的从受让人撤消的可选权限列表，例如DV_XSTREAM_ADMIN和DV_GOLDENGATE_ADMIN权限

容器

如果是CURRENT ，则仅在调用过程的容器中撤消被授予者的特权。可以在连接到根或PDB时指定CURRENT 。

如果为ALL ，则撤销CDB中所有容器中的被授予者的权限。要指定ALL ，必须在根中调用该过程。

如果是容器名称，则仅在指定容器中撤消受让者的权限。要指定root，请在连接到root时使用CDB$ROOT 。要指定PDB，必须在根中调用该过程。

注意：此参数仅适用于CDB。

使用说明

运行此过程的用户必须是可以撤消其他用户权限的管理用户。具体而言，此过程将撤消通过在此程序包中运行GRANT_ADMIN_PRIVILEGE过程而授予的权限。

注意：

要详细查看此过程运行的所有语句，可以使用该过程生成脚本，然后在文本编辑器中查看脚本。

209.3.4 REVOKE_REMOTE_ADMIN_ACCESS过程

此过程通过使用数据库链接连接到被授予者来禁用远程XStream管理员执行管理操作。

注意：

此程序包中的REVOKE_ADMIN_PRIVILEGE过程运行此过程。

也可以看看：

“ REVOKE_ADMIN_PRIVILEGE程序 ”

句法

DBMS_XSTREAM_AUTH.REVOKE_REMOTE_ADMIN_ACCESS（grantee IN VARCHAR2）;

参数

表209-5 REVOKE_REMOTE_ADMIN_ACCESS过程参数

参数 描述

受让人

禁用远程XStream管理员访问的用户。

如果DBA_XSTREAM_ADMINISTRATOR数据字典视图中存在该被授予者的行，则该过程将该被授予者的ACCESS_FROM_REMOTE列更新为NO 。如果在此更新之后，受助者的LOCAL_PRIVILEGES列和ACCESS_FROM_REMOTE列都为NO ，则该过程将从视图中删除被授予者。

如果DBA_XSTREAM_ADMINISTRATOR数据字典视图中不存在被授予者的行，则该过程不会更新视图并且不会引发错误。
