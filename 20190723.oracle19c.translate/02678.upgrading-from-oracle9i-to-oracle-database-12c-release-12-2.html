D从Oracle9 i升级到Oracle Database Release 18c版本18.1

本附录讨论了将Oracle9 i数据库（9.2.0.8）升级到Oracle数据库版本18c，版本18.1以及企业用户安全性。它包括以下部分：

  • 将Oracle Internet Directory从9.2版升级到9.0.4版

  • 将Oracle数据库从9.2.0.8版升级到Oracle数据库版本18c版本18.1

  • 将Oracle数据库从版本10g（10.1）及更高版本升级到Oracle数据库版本18c版本18.1

将Oracle Internet Directory从9.2版升级到9.0.4版

Oracle9 i Database Release 2可以与Oracle Internet Directory 9.2版或9.0.4版一起使用。Oracle数据库12c 分别 1版（12.1）需要Oracle互联网目录9.0.4或更高版本。如果您使用的是Oracle Internet Directory 9.2版，则需要将其升级到9.0.4版。

以下列表讨论了将Oracle Internet Directory 9.2版升级到Oracle Internet Directory 9.0.4版：

  1. 使用Oracle Internet Directory配置助手升级Oracle Internet Directory。如果要在目录中注册Oracle Database 12 c第1版（12.1）实例，则必须执行此操作。
  2. 将用于企业用户安全性的Oracle上下文升级到Identity Management领域（如果它们不是根上下文）。使用Oracle Internet Directory Configuration Assistant命令行实用程序，如下所示：
    oidca模式= CTXTOIMR

    如果要在域中注册Oracle Database 12 c第1版（12.1）实例，则需要执行此步骤。

    您不能将根Oracle Context for Oracle Database 12 c第1版（12.1）数据库用于它，因为它不是Identity Management Realm。

  3. 使用Oracle Internet Directory工具（如ldapmodify和bulkmodify ）将orcluserV2 objectclass添加到现有用户条目。用户需要此objectclass来更改其数据库密码，并对数据库进行kerberos身份验证。
  4. 在包含Oracle9 i数据库和Oracle Database 12 c第1版（12.1）的领域中，使用Oracle Internet Directory 9.0.4版中的基于DAS的工具来创建和管理用户。您可以使用Oracle Internet Directory自助服务控制台或Enterprise Security Manager控制台。不要使用Oracle9 i安装中的Enterprise Security Manager或Enterprise Login Assistant。

将Oracle数据库从9.2.0.8版升级到Oracle数据库版本18c版本18.1

对于升级到Oracle Database 18c版本18.1的每个Oracle9 i数据库（9.2.0.8）实例，请执行以下步骤：

  1. 使用Oracle Wallet Manager禁用数据库钱包的自动登录。
  2. 将数据库专有名称（DN）从初始化参数rdbms_server_dn到安全位置的文件。
  3. 将数据库升级到Oracle Database版本18c，版本18.1。
  4. 根据数据库admin目录的存储位置，将数据库钱包移动到$ORACLE_BASE/admin/ olddbuniquename /wallet或$ORACLE_HOME/admin/ olddbuniquename /wallet 。请注意， $ORACLE_HOME用于新的Oracle数据库版本18c，版本18.1。您可能必须创建wallet目录。
  5. 将旧的$ORACLE_HOME/network/admin/ldap.ora文件$ORACLE_HOME/network/admin/ldap.ora到新的$ORACLE_HOME/ldap/admin/ldap.ora文件中。或者，您可以使用Oracle Net Configuration Assistant创建新的ldap.ora文件。
  6. 使用命令行实用程序mkstore ，使用以下语法将数据库DN（从先前创建的安全目录位置中的文件）放入钱包中：
    mkstore -wrl database_wallet_location -createEntry ORACLE.SECURITY.DN database_DN

    系统将提示您输入钱包密码。

    如果在mkstore命令中出错，则可以使用-modifyEntry选项进行更正。

  7. 使用Database Configuration Assistant在数据库钱包中生成数据库到目录的密码。选择“修改数据库”选项。
  8. 使用Oracle Wallet Manager重新启用数据库钱包的自动登录。
  9. 使用Oracle Net Manager将sqlnet.ora文件中的新钱包位置设置为步骤4中指定的目录。

昵称属性的默认值（例如CN）保持不变。升级过程不会更改默认的昵称属性设置。从Oracle Internet Directory 9.2版升级到版本9.0.4后，如果您无法登录到Oracle数据库版本18c，版本18.1，则必须使用基于DAS的Oracle Internet Directory自助服务控制台重置密码。

将Oracle数据库从10 g版 （10.1）及更高版本升级到Oracle Database Release 18c版本18.1

对于升级到Oracle Database 18c版本18.1的每个Oracle数据库10g第1版（10.2）及更高版本实例，执行以下步骤：

  1. 将数据库升级到Oracle Database版本18c，版本18.1。
  2. 根据数据库admin目录的存储位置，将数据库钱包移动到$ORACLE_BASE/admin/ olddbuniquename /wallet或$ORACLE_HOME/admin/ olddbuniquename /wallet 。请注意， $ORACLE_HOME用于新的Oracle数据库版本18c，版本18.1。您可能必须创建wallet目录。
  3. 将旧的$ORACLE_HOME/network/admin/ldap.ora文件$ORACLE_HOME/network/admin/ldap.ora到新的$ORACLE_HOME/ldap/admin/ldap.ora文件中。或者，您可以使用Oracle Net Configuration Assistant创建新的ldap.ora文件。
  4. 使用Oracle Wallet Manager重新启用数据库钱包的自动登录。
  5. 使用Oracle Net Manager将sqlnet.ora文件中的新钱包位置设置为步骤2中指定的目录。
