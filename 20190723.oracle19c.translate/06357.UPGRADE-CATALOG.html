3.24升级目录

目的

使用UPGRADE CATALOG命令将恢复目录架构从旧版本升级到RMAN客户端所需的版本。

先决条件

必须将RMAN作为恢复目录的所有者连接到必须打开的恢复目录数据库。连接到虚拟专用目录时，不能使用UPGRADE CATALOG命令（请参阅CREATE CATALOG命令）。只能升级基本目录。

恢复目录的版本不得超过RMAN可执行文件所需的版本;否则，RMAN发出错误。RMAN在消息日志中显示升级期间生成的所有错误消息。

Oracle数据库10g R1版本的恢复目录模式需要 CREATE TYPE权限。如果您在10 g R1之前的版本中创建了恢复目录所有者，并且您已授予了当角色未包含CREATE TYPE权限时，此用户的RECOVERY_CATALOG_OWNER角色，则必须在执行升级之前明确地向此用户授予CREATE TYPE 。

如果要将恢复目录升级到Oracle Database 12 c第2版（12.2）或更高版本，则必须运行管理恢复目录特权的dbmsrmansys.sql脚本。此外，如果使用虚拟专用目录，则必须运行升级虚拟专用目录的dbmsrmanvpc.sql脚本。从Oracle Database 12c第1版（12.1.0.2）开始，恢复目录数据库必须使用Oracle数据库企业版。

使用说明

RMAN会提示您连续两次输入UPGRADE CATALOG命令以确认升级。要绕过其他确认步骤，请在第一次运行时输入带有NOPROMPT选项的UPGRADE CATALOG命令。

如果恢复目录已经是最新的，则RMAN允许运行该命令，以便在必要时可以重新创建包。

如果升级到基本恢复目录需要更改现有虚拟专用目录，则RMAN会在下次RMAN连接到该虚拟专用目录时自动进行这些更改。

UPGRADE CATALOG命令不运行脚本来执行升级。相反，RMAN将各种SQL DDL语句发送到恢复目录，以使用新表，视图，列等更新恢复目录模式。
