XDK JavaBeans简介

XDK JavaBeans是一组可扩展标记语言（XML）组件，可以在Java应用程序和applet中使用。

使用XDK JavaBeans的先决条件

描述了使用XDK JavaBeans的先决条件。

本附录假定您熟悉这些技术：

  • JavaBeans 。JavaBeans组件或Bean是可重用的软件组件，可以在构建器工具中以可视方式进行操作。

  • Java数据库连接（JDBC） 。数据库连接包含在XDK JavaBeans中。Bean可以直接连接到启用JDBC的数据库，以检索和存储XML和可扩展样式表语言（XSL）文件。

  • 文档对象模型（DOM） 。DOM是XML文档结构的内存树表示。

  • 简单的XML API（SAX） 。SAX是基于事件的XML解析的标准。

  • XML Schema语言 。有关概述和建议阅读的链接，请参阅使用XML Schema Processor for Java 。

XDK JavaBeans的标准和规范

XDK JavaBeans需要1.2或更高版本的XDK，它们可以与任何版本的JDK 1.2或更高版本一起使用。XDK JavaBeans符合Sun JavaBeans规范，并包含扩展java.beans.所需的BeanInfo类java.beans.SimpleBeanInfo 。

JavaBeans 1.01规范描述了JDK 1.1中存在的JavaBeans。

Java 2平台添加到JavaBeans核心规范中为开发人员提供了创建更复杂的JavaBeans组件的标准方法。

XDK JavaBeans功能

XDK JavaBeans有助于向XML应用程序添加图形用户界面（GUI）。Bean封装包括可以直接从Java集成开发环境（IDE）（如Oracle JDeveloper）访问的文档和描述符。

DOMBuilder

oracle.xml.async.DOMBuilder bean从XML文档构造DOM树。DOMBuilder JavaBean使用bean接口封装DOMParser类的XML解析器，并支持异步解析。通过注册监听器，Java程序可以启动对大型或多个文档的解析，并立即将控制权返回给调用者。

这个bean的一个主要好处是在解析多个文件时提高了效率，特别是在文件很大的情况下。DOMBuilder还可以在交互式可视化应用程序中的后台线程中提供异步解析。如果没有异步解析，GUI将无法使用，直到要解析的文档。使用DOMBuilder ，应用程序调用parse方法然后恢复控制。应用程序可以显示进度条，允许用户取消解析，等等。

XSLTransformer

oracle.xml.async.XSLTransformer JavaBean支持异步转换。它接受XML文档，应用可扩展样式表语言转换（XSLT）样式表，并创建输出文件。它允许您将XML文档转换为几乎任何基于文本的格式，包括XML，HTML和数据定义语言（DDL）。

此bean还可以用作服务器端应用程序或servlet的基础，以将XML文档（例如查询结果的XML表示形式）呈现为HTML，以便在浏览器中显示。

XSLTransformer bean的主要好处是它可以并行转换多个文件。与DOMBuilder一样，您也可以在可视化应用程序中使用它，以避免GUI无响应的长时间。通过实现XSLTransformerListener接口，调用应用程序在转换完成时接收通知。

DBACCESS

oracle.xml.dbaccess.DBAccess bean维护包含多个XML和文本文档的字符大对象（ CLOB ）表。

您可以使用它来存储和检索数据库中的XML文档，但不能在数据库中处理它们。使用DBAccess bean的Java应用程序通过JDBC连接到数据库。存储在XML文档CLOB表是类型的不是XMLType没有自己的实体扩展。

DBAccess bean使您可以执行以下任务：

  • 创建和删除CLOB类型的表。

  • 查询CLOB表的内容。

  • 对存储在CLOB表中的XML文档执行INSERT ， UPDATE和DELETE操作。

XMLDBAccess

oracle.xml.xmldbaccess.XMLDBAccess bean扩展了DBAccess bean以支持存储在XMLType表中的XML文档。该类提供了列出，删除或检索XMLType实例及其表的方法。例如， getXMLXPathTextData()方法从XML文档中检索XPath表达式的值。

DBAccess JavaBean维护可以包含多个XML和文本文档的XMLType表。每个XML或文本文档都存储在表中的一行中。该表是使用此结构化查询语言（SQL）语句创建的：

CREATE TABLE（FILENAME CHAR（）UNIQUE，FILEDATA SYS.XMLType）;

FILENAME字段包含一个唯一的字符串，用作检索，更新或删除行的键。文档文本存储在FILEDATA字段中。

XMLDBAccess bean执行以下任务：

  • 创建和删除XMLType表

  • 列出XMLType列的内容

  • 对存储在XMLType表中的XML文档执行INSERT ， UPDATE和DELETE操作

的xmldiff

在比较XML文档时，逐个字符地比较它们通常是无益的。大多数XML比较都关注结构和重要文本内容的差异，而不是空白区域的差异。

oracle.xml.differ.XMLDiff bean执行以下任务：

  • 构造并比较两个输入XML文档的DOM树，并指出文档是否不同。

  • 提供两个XML文件之间差异的图形显示。具体来说，您可以参考节点插入，删除，修改或移动。

  • 生成一个XSLT样式表，可以将其中一个输入XML文档转换为另一个文档。

XMLDiff bean在管道应用程序中特别有用。例如，应用程序可以更新XML文档，将其与文档的先前版本进行比较，并存储显示它们之间差异的XSLT样式表。

XMLCompress

Oracle XML解析器包含一个压缩程序，可以将XML文档对象序列化为二进制流。

压缩和解压缩XML中对此进行了解释。虽然是一个有用的工具，但使用XML解析器进行压缩有以下缺点：

  • 反序列化XML数据时，必须重新解析。

  • 将XML数据封装在标签中会大大增加其大小。

oracle.xml.xmlcomp.XMLCompress bean是XML压缩功能的封装。它在序列化和反序列化XML时提供了这些优势：

  • 它封装了序列化DOM的方法，该方法生成一个流。

  • XML处理器可以从压缩流中重新生成DOM，而无需重新分析XML。

该bean支持DOMParser或SAXParser解析的输入XML的压缩和解压缩。DOM压缩支持XMLType ， CLOB和BLOB类型的输入。

要使用不同的解析选项，请在输入之前解析文档，然后将XMLDocument对象传递给压缩程序bean。压缩因子是基于输入XML文件和压缩文件的文件大小的粗略值。压缩因子方法的局限性在于只有在使用java.io.执行压缩时才能使用它。File对象作为参数。

XSDValidator

oracle.xml.schemavalidator.XSDValidator bean封装了XSDValidator类，并添加了验证DOM树的功能。

此bean的一个有用功能涉及验证错误。如果应用程序抛出验证错误，则方法getStackList()返回导致无效节点的DOM树路径列表。具有错误的节点在堆栈树的向量中返回，其中堆栈的顶部元素表示根节点。您可以通过从堆栈中提取子节点来获取子节点。要使用getStackList() ，必须使用实例化java.util.Vector和java.util.Stack类。

使用DOMBuilder JavaBean：基本过程

DOMBuilder类根据万维网联盟（W3C）推荐实现XML 1.0解析器。它解析XML文档并构建DOM树。解析在单独的线程中完成。构建树时，必须使用接口DOMBuilderListener进行通知。

在开发使用此bean的应用程序时，必须导入以下子包：

  • oracle.xml.async ，为DOM构建提供异步Java bean

  • oracle.xml.parser.v2 ，它为SAX，DOM和XSLT提供API
