创建序列

目的

使用CREATE SEQUENCE语句创建一个序列 ，该序列是一个数据库对象，多个用户可以从中生成唯一的整数。您可以使用序列自动生成主键值。

生成序列号时，序列会递增，与事务提交或回滚无关。如果两个用户同时递增相同的序列，则每个用户获取的序列号可能有间隙，因为序列号是由另一个用户生成的。一个用户永远无法获取另一个用户生成的序列号。在由一个用户生成序列值之后，该用户可以继续访问该值，而不管该序列是否由另一个用户递增。

序列号是独立于表生成的，因此相同的序列可用于一个或多个表。单个序列号可能会被跳过，因为它们是在最终回滚的事务中生成和使用的。另外，单个用户可能没有意识到其他用户正在从相同的序列中绘制。

创建序列后，可以使用CURRVAL伪列访问SQL语句中的值，该伪列返回序列的当前值，或NEXTVAL伪列，它会递增序列并返回新值。

先决条件

要在自己的模式中创建序列，您必须具有CREATE SEQUENCE系统特权。

要在另一个用户的模式中创建序列，您必须具有CREATE ANY SEQUENCE系统特权。

语义

模式

指定要包含序列的模式。如果省略schema ，则Oracle数据库会在您自己的模式中创建序列。

序列

指定要创建的序列的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

如果没有通过GLOBAL指定INCREMENT BY子句，则创建一个以1开头的递增序列，并且在没有上限的情况下增加1。仅指定INCREMENT BY -1会创建一个以-1开头的降序序列，并且在没有下限的情况下减少。

  • 创建一个序列 无限制的增量，对于升序序列，省略MAXVALUE参数或指定NOMAXVALUE 。对于降序序列，请省略MINVALUE参数或指定NOMINVALUE 。

  • 创建一个序列 停止在预定义的限制，对于升序序列，指定MAXVALUE参数的值。对于降序，请为MINVALUE参数指定值。同时指定NOCYCLE 。一旦序列达到其极限，任何尝试生成序列号都会导致错误。

  • 创建一个序列 达到预定义限制后重新启动，指定MAXVALUE和MINVALUE参数的值。同时指定CYCLE 。

共享

仅当在应用程序根目录中创建序列时，此子句才适用。这种类型的序列称为应用程序公共对象，它可以与属于应用程序根目录的应用程序PDB共享。要确定序列的共享方式，请指定以下共享属性之一：

  • METADATA - 元数据链接共享序列的元数据，但其数据对每个容器都是唯一的。这种类型的序列称为元数据链接的应用程序公共对象 。

  • DATA - 数据链接共享序列，其数据对于应用程序容器中的所有容器都是相同的。其数据仅存储在应用程序根目录中。这种类型的序列称为数据链接的应用程序公共对象 。

  • NONE - 不共享序列。

如果省略此子句，则数据库使用DEFAULT_SHARING初始化参数的值来确定序列的共享属性。如果DEFAULT_SHARING初始化参数没有值，则默认值为METADATA 。

创建序列后，无法更改序列的共享属性。

增量

指定序列号之间的间隔。此整数值可以是任何正整数或负整数，但不能为0。对于升序序列，此值可以包含28个或更少的数字，对于降序序列，该值可以包含27个或更少的数字。该值的绝对值必须小于MAXVALUE和MINVALUE的差值。如果此值为负，则序列下降。如果值为正，则序列上升。如果省略此子句，则间隔默认为1。

从...开始

指定要生成的第一个序列号。使用此子句以大于其最小值的值开始升序，或者以小于其最大值的值开始降序。对于升序序列，默认值是序列的最小值。对于降序序列，默认值是序列的最大值。对于正值，此整数值可以包含28个或更少的数字，对于负值，该数字可以包含27个或更少的数字

注意：

该值不一定是上升或下降循环序列分别在达到其最大值或最小值之后循环的值。

MAXVALUE

指定序列可以生成的最大值。对于正值，此整数值可以包含28个或更少的数字，对于负值，该数字可以包含27个或更少的数字MAXVALUE必须等于或大于START WITH且必须大于MINVALUE 。

NOMAXVALUE

指定NOMAXVALUE以指示升序序列的最大值10 28 -1或降序序列的最大值-1。这是默认值。

MINVALUE

指定序列的最小值。对于正值，此整数值可以包含28个或更少的数字，对于负值，该数字可以包含27个或更少的数字MINVALUE必须小于或等于START WITH且必须小于MAXVALUE 。

NOMINVALUE

指定NOMINVALUE以指示升序序列的最小值1或 - （10 27 -1）表示降序序列。这是默认值。

周期

指定CYCLE以指示序列在达到其最大值或最小值后继续生成值。在升序序列达到其最大值后，它将生成其最小值。在降序序列达到最小值后，它会生成最大值。

NOCYCLE

指定NOCYCLE以指示序列在达到其最大值或最小值后无法生成更多值。这是默认值。

CACHE

指定数据库预分配的序列值的数量，并保留在内存中以便更快地访问。该整数值可以有28个或更少的数字。此参数的最小值为2。对于循环的序列，该值必须小于循环中的值的数量。您不能缓存超过给定循环序列号的值。因此， CACHE允许的最大值必须小于以下公式确定的值：

CEIL（（MAXVALUE  -  MINVALUE）/ ABS（INCREMENT））

如果发生系统故障，则所有未在已提交的DML语句中使用的高速缓存序列值都将丢失。潜在的丢失值数等于CACHE参数的值。

注意：

如果您在Oracle Real Application Clusters环境中使用序列，Oracle建议使用CACHE设置来增强性能。

NOCACHE

指定NOCACHE以指示未预先分配序列的值。如果省略CACHE和NOCACHE ，则数据库默认会缓存20个序列号。

订购

指定ORDER以保证按请求顺序生成序列号。如果使用序列号作为时间戳，则此子句很有用。对于用于生成主键的序列，保证顺序通常并不重要。

NOORDER

如果您不想保证按请求顺序生成序列号，请指定NOORDER 。这是默认值。

保持

如果希望NEXTVAL在重放应用程序连续性期间保留其原始值，请指定KEEP 。仅当运行应用程序的用户是包含序列的架构的所有者时，才会发生此行为。此子句对于在可恢复错误后的重放时提供绑定变量一致性非常有用。有关应用程序连续性的更多信息，请参阅Oracle数据库开发指南 。

NOKEEP

如果您不希望NEXTVAL在重放应用程序连续性期间保留其原始值，请指定NOKEEP 。这是默认值。

注意：

KEEP和NOKEEP子句仅适用于包含序列的模式的所有者。通过授予或撤消SEQUENCE的KEEP SEQUENCE对象权限，您可以控制NEXTVAL在重放应用程序连续性期间是否为其他用户保留其原始值。有关KEEP SEQUENCE对象权限的更多信息，请参阅表18-2 。

规模

使用SCALE启用序列可伸缩性。指定SCALE ，会在序列的开头附加一个数字偏移量，从而删除生成值中的所有重复项。

延伸

如果使用SCALE指定EXTEND ，则生成的序列值全部为长度(x+y) ，其中x是可伸缩偏移的长度（默认值为6）， y是序列中的最大位数(maxvalue/minvalue) 。

当您使用SCALE ，强烈建议您不要在序列上同时使用ORDER 。

NOEXTEND

NOEXTEND是SCALE子句的默认设置。使用NOEXTEND设置，生成的序列值最多与序列中的最大位数(maxvalue/minvalue)一样宽。此设置对于与使用序列填充固定宽度列的现有应用程序集成非常有用。

noScale，则

使用NOSCALE禁用序列可伸缩性。

碎片

有关SHARD子句的完整语义，请参阅ALTER SEQUENCE语句的SHARD子句。

SESSION

指定SESSION以创建会话序列，该序列是一种特殊类型的序列，专门设计用于具有会话可见性的全局临时表。与现有的常规序列（为了进行比较而称为“全局”序列）不同，会话序列仅在会话内返回唯一的序列号范围，而不是跨会话。另一个区别是会话序列不是持久的。如果会话消失，则会话期间访问的会话序列的状态也会消失。

会话序列必须由读写数据库创建，但可以在任何读写或只读数据库（常规数据库临时打开只读或备用数据库）上访问。

使用SESSION子句指定时，将忽略CACHE ， NOCACHE ， ORDER或NOORDER子句。

全球

指定GLOBAL以创建全局或常规序列。这是默认值。
