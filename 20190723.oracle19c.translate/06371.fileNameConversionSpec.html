4.8 fileNameConversionSpec

目的

使用fileNameConversionSpec子句指定一个或多个模式，用于根据旧文件名生成新文件名。用于 BACKUP ， CONVERT ，和 DUPLICATE是生成输出文件名的一种方法。

使用说明

字符串模式的规则以及它们如何影响文件命名等于初始化参数的规则 DB_FILE_NAME_CONVERT 。在括号中，提供偶数个字符串模式。

当RMAN基于旧文件名生成新文件名时，它会将原始文件名与每对字符串模式的第一个成员进行比较。RMAN第一次找到作为原始文件名的子字符串的模式时，RMAN通过将该对的第二个成员替换为匹配的子字符串来生成新文件名。

模式不必在文件名的开头匹配。以下命令将数据文件/disk1/dbs/users.dbf的映像副本创建为/disk1/newdbs/users.dbf ：

备份为COPY DB_FILE_NAME_CONVERT =（'dbs'，'newdbs'）; TABLESPACE用户;

当存在转换的给定文件名的多个可能匹配时，RMAN使用模式列表中的第一个匹配来生成新文件名。以下命令与前一个示例具有相同的效果，因为模式dbs与文件名匹配，因此文件名永远不会与第二个模式/disk1 ：

备份为COPY DB_FILE_NAME_CONVERT =（'dbs'，'newdbs'，'/ disk1'，'/ newdisk'）TABLESPACE用户;

对于CONVERT TABLESPACE ， CONVERT DATABASE和BACKUP AS COPY命令，如果这些操作的源文件是Oracle管理的文件，则不能使用fileNameConversionSpec将源文件名转换为新的输出文件名。对于Oracle管理的文件，无论是在自动存储管理（ASM）还是在普通文件系统存储中，都必须允许数据库生成输出文件的文件名。例如，非ASM存储中的数据文件的OMF文件名可能具有以下形式：

/private/boston/datafile/01_mf_system_ab12554_.dbf

一个 ASM存储中的OMF文件名可能具有以下形式：

+ DISK /波士顿/数据文件/ system.256.4543080

只有数据库才能生成和管理OMF文件名。通常，将不同磁盘组或不同OMF位置的名称替换为OMF文件名不会在新目标中生成有效的文件名。要转换OMF文件名以便存储在另一个OMF位置，请使用带有这些命令的替代方法（如FORMAT子句）来指定新的输出位置，并允许数据库管理特定的输出文件名。

句法

fileNameConversionSpec :: =

语义

语法元素 描述

string_pattern

指定用于转换文件名的字符串对。您可以根据需要使用多对主要和备用替换字符串。例如，您可以将字符串模式设置为如下值：

DB_FILE_NAME_CONVERT =（'str1'，'str2'，'str3'，'str4'...）

在此示例中， str1是与原始文件名匹配的模式， str2是在生成的文件名中替换str1的模式， str3是与原始文件名匹配的模式， str4是在生成的文件名中替换str3的模式。

例子

示例4-17将DB_FILE_NAME_CONVERT与单个转换对配合使用

在此示例中，表空间users包含目录/disk1/oradata/prod/中的数据文件，而表空间tools包含/disk1/oradata/prod/中的数据文件。对于要转换的每个数据文件，如果disk1/oradata/prod是数据文件名的子字符串，则通过将字符串替换为disk2来创建映像副本名称。 

BACKUP AS COPY DB_FILE_NAME_CONVERT =（'disk1 / oradata / prod'，'disk2'）TABLESPACE用户，工具;

示例4-18将DB_FILE_NAME_CONVERT与多个转换对一起使用

此示例创建示例4-17中描述的相同数据文件的映像副本。每对中的第一个字符串指定要在源数据文件的名称中匹配的模式。每对中的第二个字符串是生成图像副本名称时使用的替换模式。 

备份为COPY DB_FILE_NAME_CONVERT =（'/ disk1 / oradata / prod / users'，'/ disk2 / users'，'/ disk1 / oradata / prod / tools'，'/ tmp / tools'）TABLESPACE工具，用户;
