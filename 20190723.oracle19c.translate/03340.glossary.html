词汇表

申请请求者（AR）

由应用程序使用，AR假定客户端组件采用代表应用程序的经典客户端/服务器配置，从而生成所有DRDA协议请求。

应用服务器（AS）

Application Server。假定经典客户端/服务器配置中的服务器组件，充当服务器的DRDA协议前端，处理DRDA请求，执行服务器函数调用并将结果返回给客户端。

AS / 400

Application System / 400，一个IBM平台。

CCSID

编码字符集标识符（IBM / DRDA术语）。16位数字，包括一组特定的编码方案标识符，字符集标识符代码页标识符，以及唯一标识编码图形字符表示的其他信息。

示例包括：500 INTL EBCDIC [CECP：比利时，加拿大（AS / 400 *），瑞士，国际Latin1]，819 ISO 88591 ASCII [ISO 88591：拉丁字母数字1Latin1国家和地区]，850 LATIN1 PCDATA [PC数据：MLP 222拉丁字母数字1Latin1国家和地区]

特殊情况：CCSID 65534（将代码页推迟到较低级别定义）和CCSID 65535（二进制数据）。

数据库请求模块（DBRM）

（IBM / DRDA术语）。这是一个专有的磁盘文件，包含预处理后的嵌入式SQL应用程序的SQL语句。然后，经过后处理的应用程序源将仅包含一个语句引用号，指示要在DBRM中使用的SQL语句。这些语句是外部化的，因此数据库系统可以完全分析和优化SQL的执行。

DB2

IBM DB2的缩写，IBM生成的数据库。

DBCS

在IBM术语中，这是一个双字节字符集，任何字符集的字符代码点长度恰好为2个字节。

专用实例配置

用于DRDA配置的Oracle数据库提供程序，其中为DRDA实例的每个Oracle数据库提供程序创建一个RDB_MAP条目。此方法用于IBM DB2 Database for z / OS DRDA客户端。请参阅多路复用实例配置 。

分布式数据管理（DDM）

DDM体系结构提供分布式数据库使用的整体命令和回复结构。实现应用程序请求者（客户端）和应用程序服务器之间的通信的所有分布式数据库函数所需的命令少于20个。

分布式关系数据库架构

分布式关系数据库体系结构（DRDA）是一种开放的已发布体系结构，可以在不同平台上实现应用程序和数据库系统之间的通信，无论这些应用程序和数据库系统是由相同或不同的供应商提供，以及平台是相同还是不同的硬件/软件架构。DRDA是其他架构和使用它们的环境规则和流程模型的组合。实际包含DRDA的体系结构是分布式数据管理（DDM）和格式化数据对象内容体系结构（FD：OCA）。

DRDA

请参阅分布式关系数据库体系结

DRDA套餐

（IBM / DRDA术语）。“包”是嵌入式SQL应用程序中定义的SQL语句和属性的集合。通过通过DRDA BIND请求命令绑定资源文件（DBRM）来创建包。

动态SQL

包可能包含静态SQL和动态SQL的混合。动态SQL是未在应用程序中执行的SQL（通常被认为是adhoc ，即使它是由应用程序构造的）。静态SQL和动态SQL之间的主要区别在于，静态SQL作为Package的一部分预先加载到数据库中，并且动态SQL必须作为运行时发送到数据库以供执行。从历史上看，Oracle仅实现动态SQL，依靠其光标高速缓存设施的优势来提高SQL执行的速度，这在很大程度上抵消了对静态SQL的需求。请参见静态SQL 。

FD：OCA

请参阅格式化数据对象内容体系结构

格式化数据对象内容体系结构

格式化数据对象内容体系结构（FD：OCA）为DRDA提供数据定义体系结构基础。 DRDA定义的描述符为应用程序请求者和服务器之间常规交换的所有信息提供布局和数据类型信息。描述符组织由DRDA定义，以允许动态定义作为命令或回复数据的一部分流动的用户数据。DRDA还指定描述符只需每个应答集流一次，无论实际返回的行数如何，从而最大限度地减少了线路上的数据流量。

IBM

International Business Machines，该公司负责DB2 Database和DB2组产品。

MBCS

在IBM术语中，这是一个多字节字符集，任何可能包含可变长度字符代码点的字符集。一个例子是Unicode（UTF-8）。另一个例子是将单字节字符集和双字节字符集组合在一起制作多字节字符集。

多路复用实例配置

用于DRDA配置的Oracle数据库提供程序，其中可以使用不同的主机名或IP地址以及未分配的网络端口号指定其他DATA_PORT条目。请参阅专用实例配置 。

Oracle调用接口（OCI）

OCI是一组C语言软件API，提供Oracle数据库的接口。

OCI由过程API组成，这些API执行数据库管理任务以及使用PL / SQL或SQL查询，访问和操作数据。

SBCS

在IBM术语中，这是一个单字节字符集，任何字符集的字符代码点长度恰好为1个字节。

特别注册

在DB2中，特殊寄存器是为应用程序进程定义的存储区域，用于存储可在SQL语句中引用的信息。对特殊寄存器的引用是对当前服务器提供的值的引用。如果值是字符串，则其CCSID是当前服务器的默认CCSID。

静态SQL

包可能包含静态SQL和动态SQL的混合。静态SQL是已作为应用程序的一部分编写的SQL。它的语法和程序中变量（绑定变量）的使用在代码中是固定的，在执行期间不能更改。此外，SQL不是应用程序的一部分。它从源中提取并作为Package的一部分上传，进一步修复其形式并防止恶意修改它的功能。假设预先对静态SQL进行了大量分析，以便实现更高效的执行计划。从历史上看，DB2为主要面向批处理的应用程序发明了静态SQL，因此不受动态构造的影响。请参见动态SQL 。

SQL类型

在IBM的DB2术语中，这是数据类型的接口值。

SQLAM级别

SQLAM代表SQL应用程序管理器。SQLAM Level 8是Oracle的DRDA实现提供的支持级别。

UOW

在IBM术语中，这是一个工作单元，是在事务范围内对数据库进行的所有更改的资源指定。在应用程序的事务期间，DRDA在客户端和数据库之间维护分布式（或远程）工作单元。
