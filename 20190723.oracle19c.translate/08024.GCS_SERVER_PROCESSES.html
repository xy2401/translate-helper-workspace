1.129 GCS_SERVER_PROCESSES

GCS_SERVER_PROCESSES指定后台GCS服务器进程的数量（LMS0，...LMS9和LMSa，......LMSz）用于提供Oracle RAC实例之间的实例间流量。

属性 描述

参数类型

整数

默认值

如果1 - 3 CPUS，则为1

如果是4 - 15个CPU，则为2

如果是16个或更多CPU，则为2 +（CPU / 32）。如果结果包括分数，则忽略该分数。例如，如果您有20个CPU，则2 +（20/32）将等于2个GCS进程。脚1

如果CLUSTER_DATABASE设置为false ，则为0

如果是Oracle ASM，那么1

可修改

没有

可在PDB中修改

没有

价值范围

1到100

基本

没有

Oracle RAC

多个实例可以具有不同的值。

脚注1

在某些操作系统上，RDBMS根据CPU的核心与线程比率优化后台GCS服务器进程的默认分配。

GCS服务器进程仅在Oracle RAC环境中可见。
