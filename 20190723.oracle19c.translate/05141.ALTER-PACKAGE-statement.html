14.3 ALTER PACKAGE声明

ALTER PACKAGE语句显式重新编译包规范，正文或两者。显式重新编译消除了对隐式运行时重新编译的需要，并防止了相关的运行时编译错误和性能开销。

由于包中的所有对象都存储为一个单元，因此ALTER PACKAGE语句将重新编译所有包对象。您不能使用ALTER PROCEDURE语句或ALTER FUNCTION语句单独重新编译作为程序包一部分的过程或函数。

注意：

此声明不会更改现有包的声明或定义。要重新声明或重新定义包，请使用“ CREATE PACKAGE语句 ”或带有OR REPLACE子句的“ CREATE PACKAGE BODY语句 ” 。

语义

alter_package

模式

包含该包的模式的名称。默认值：您的架构。

包裹名字

要重新编译的包的名称。

{版权|不可取的}

指定包是否变成editioned或noneditioned对象如果editioning为架构对象类型稍后启用PACKAGE在schema 。默认值： EDITIONABLE 。有关更改已编辑和未编辑的对象的信息，请参阅“ Oracle数据库开发指南” 。

package_compile_clause

重新编译包规范，正文或两者。

请参阅compile_clause和compiler_parameters_clause语义。

例子

示例14-3重新编译包

该语句显式重新编译hr.emp_mgmt包的规范和主体。

有关创建此包的示例，请参阅“ CREATE PACKAGE语句 ” 。 

ALTER PACKAGE emp_mgmt COMPILE PACKAGE;

如果数据库在重新编译emp_mgmt规范和正文时未遇到编译错误，则emp_mgmt将变为有效。用户hr随后可以调用或引用emp_mgmt规范中声明的所有包对象，而无需运行时重新编译。如果重新编译emp_mgmt导致编译错误，则数据库将返回错误，并且emp_mgmt仍然无效。

该数据库还会使依赖于emp_mgmt所有对象无效。如果您随后引用其中一个对象而未先显式重新编译它，则数据库会在运行时隐式重新编译它。

要在模式hr重新编译emp_mgmt软件包的主体，请发出以下语句：

ALTER PACKAGE hr.emp_mgmt COMPILE BODY;

如果数据库在重新编译包体时未遇到编译错误，则正文将变为有效。用户hr随后可以调用或引用emp_mgmt规范中声明的所有包对象，而无需运行时重新编译。如果重新编译正文会导致编译错误，则数据库将返回错误消息，并且正文仍然无效。

因为此语句重新编译正文而不是emp_mgmt的规范，所以数据库不会使依赖对象无效。
