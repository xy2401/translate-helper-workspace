什么是Oracle数据编辑？

Oracle Data Redaction使您可以屏蔽（编辑）从应用程序发出的查询返回的数据。

您可以使用以下方法之一编辑列数据：

  • 完全编辑。 您编辑列数据的所有内容。返回给查询应用程序用户的编辑值取决于列的数据类型。例如， NUMBER数据类型的列用零（ 0 ）编辑，字符数据类型用单个空格编辑。

  • 部分编辑。 您编辑一部分列数据。例如，您可以使用星号（*）编辑社会保险号，但最后4位数除外。

  • 常用表达。 您可以使用正则表达式查找要编辑的数据模式。例如，您可以使用正则表达式来编辑可能具有不同字符长度的电子邮件地址。它仅适用于字符数据。

  • 随机编辑。 呈现给查询应用程序用户的编辑数据在每次显示时都显示为随机生成的值，具体取决于列的数据类型。

  • 没有编辑。 “无编校类型”选项使您可以测试编校策略的内部操作，而不会对针对其上定义了策略的表的查询结果产生影响。您可以使用此选项在将编校策略定义应用于生产环境之前对其进行测试。

当用户访问数据时（即，在查询执行时），Oracle数据库在运行时应用编校。该解决方案在生产系统中运行良好。在数据被编辑期间，正常执行所有数据处理，并保留后端参照完整性约束。

数据编辑可以帮助您遵守行业法规，如支付卡行业数据安全标准（PCI DSS）和萨班斯 - 奥克斯利法案。

何时使用Oracle数据编辑

必须伪装应用程序和应用程序用户必须访问的敏感数据时，请使用Oracle Data Redaction。

通过数据编辑，您可以使用多种不同的编校样式轻松伪装数据。

Oracle Data Redaction非常适用于必须从返回给某些应用程序用户的个人身份信息（PII）查询结果集中编辑特定字符的情况。例如，您可能希望提供以数字4320结尾的美国社会安全号码***-**-4320 。

Oracle Data Redaction特别适用于呼叫中心应用程序和其他只读应用程序。将Oracle Data Redaction与执行更新的应用程序一起使用时要小心，因为可以将编辑后的数据写回此数据库。

使用Oracle数据编辑的好处

使用Oracle Data Redaction保护数据时，它可提供多种好处。

这些好处如下：

  • 您有不同的编辑风格可供选择。

  • 由于数据在运行时被编辑，因此Data Redaction非常适合数据不断变化的环境。

  • 您可以在一个中心位置创建数据编辑策略，并从那里轻松管理它们。

  • Data Redaction策略使您能够基于SYS_CONTEXT值创建各种函数条件，这些值可以在运行时用于确定数据编辑策略何时应用于应用程序用户查询的结果。

Oracle Data Redaction与数据库应用程序一起使用

Oracle Data Redaction可保护数据库应用程序中显示的敏感数据。

数据编辑对应用程序用户是透明的，因为它保留了原始数据类型和（可选）格式。它对数据库高度透明，因为缓冲区，缓存和存储中的数据保持不变 - 仅在SQL查询结果返回给调用者之前的最后一刻才更改。在使用相同底层数据库的所有应用程序中一致地强制执行编校。您可以通过检查通过SYS_CONTEXT函数传递到数据库的应用程序用户信息来指定哪些应用程序用户应该只看到编辑数据;您可以根据当前数据库或应用程序用户的属性编辑数据;并且您可以在给定的编校策略中实现多个逻辑条件。此外，Data Redaction以最小化性能开销的方式实现。这些特性使Oracle Data Redaction特别适合于共享通用生产数据库的各种应用程序，分析工具，报告工具和监视工具的使用。尽管其主要目标是编写应用程序的生产数据，但Oracle Data Redaction还可与Oracle Enterprise Manager数据屏蔽和子集包结合使用，以保护测试和开发环境中的敏感数据。

具有Ad Hoc数据库查询注意事项的Oracle数据编辑

您可能会遇到这样的情况：为数据库用户执行的即席查询编辑敏感数据很方便。

例如，在支持生产应用程序的过程中，用户可能需要运行临时数据库查询来排除故障并修复应用程序的紧急问题。这与Oracle Data Redaction Use with Database Applications中描述的基于应用程序的方案不同，后者通常生成一组有限的SQL查询，使用已定义的数据库帐户并具有固定权限。

尽管Oracle Data Redaction并非旨在防止数据库直接针对数据库运行即席查询的数据库用户，但它可以提供额外的层来减少意外数据泄露的可能性。由于此类用户可能有权更改数据，更改数据库架构以及完全绕过SQL查询界面，因此恶意用户可能会在某些情况下绕过数据编辑策略。

请记住，Oracle数据库安全工具旨在一起使用以提高整体安全性。通过部署这些工具中的一个或多个作为Oracle Data Redaction的补充，您可以安全地提高整体安全状况。
