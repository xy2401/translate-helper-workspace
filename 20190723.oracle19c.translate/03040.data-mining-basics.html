2.1挖掘功能

介绍数据挖掘功能的概念。

对数据的基本了解使用Oracle Data Mining需要挖掘函数和算法。

每个数据挖掘功能都指定了一类可以建模和解决的问题。数据挖掘功能通常分为两类： 监督和非监督 。有监督和无监督学习的概念来源于科学机器学习，这被称为子区域人工智能。

人工智能是指实现和研究具有自主智能或自身行为的系统。机器学习涉及使设备能够从自己的性能中学习并修改自己的功能的技术。数据挖掘将机器学习概念应用于数据。

2.1.1监督数据挖掘

监督学习也被称为定向学习。学习过程由先前已知的从属属性或目标。定向数据挖掘试图将目标的行为解释为一组独立属性或预测变量的函数。

监督学习通常会导致预测模型。这与无监督学习形成对比，其中目标是模式检测。

监督模型的构建涉及培训 ，该过程软件分析许多已知目标值的情况。在训练过程中，模型“学习”进行预测的逻辑。例如，寻求识别可能响应促销的客户的模型必须通过分析已知在过去已响应或未响应促销的许多客户的特征来进行培训。

2.1.1.1监督学习：测试

建筑（培训）需要单独的数据集并测试一些预测模型。构建数据（训练数据）和测试数据必须具有相同的列结构。通常，一个大型表或视图分为两个数据集：一个用于构建模型，另一个用于测试模型。

将模型应用于测试数据的过程有助于确定基于一个选定样本构建的模型是否可推广到其他数据。特别是，它有助于避免这种现象过度拟合，当模型的逻辑太适合构建数据并且因此具有很小的预测能力时可能发生。

2.1.1.2监督学习：评分

同时应用数据称为评分数据，是实际的应用模型的人口。例如，您可以构建一个模型来标识经常购买某个产品的客户的特征。要获取在某个商店购物并且可能购买相关产品的客户列表，您可以将该模型应用于该商店的客户数据。在这种情况下，商店客户数据是评分数据。

大多数监督学习可以应用于感兴趣的人群。主要的监督采矿技术， 分类和回归 ，都可以用于评分。

Oracle Data Mining不支持另一个受监督函数Attribute Importance的评分操作。这种类型的模型建立在感兴趣的人群上，以获得有关该人口的信息;它们不能应用于单独的数据。属性重要性模型返回并排列在预测目标值时最重要的属性。

2.1.2无监督数据挖掘

无监督学习是非指导性的。依赖属性和独立属性之间没有区别。没有先前已知的结果来指导算法构建模型。

无监督学习可用于描述目的。它也可用于进行预测。

2.1.2.1无监督学习：评分

介绍无监督学习，支持的评分操作和无监督的Oracle数据挖掘功能。

虽然无监督数据挖掘没有指定一个目标，大多数无监督学习可以应用于感兴趣的人群。例如， 聚类模型使用描述性数据挖掘技术，但它们可用于根据聚类分配对案例进行分类。 异常检测 ， 虽然无监督，但通常用于预测数据点在一组案例中是否典型。

2.2算法

一个算法是解决特定问题的数学过程。Oracle Data Mining为每个数据挖掘功能至少支持一种算法。对于某些功能，您可以选择多种算法。例如，Oracle Data Mining支持四种分类算法。

每个数据挖掘模型由特定算法产生。通过使用多个算法可以最好地解决一些数据挖掘问题。这需要开发多个模型。例如，您可能首先使用特征提取模型来创建一组优化的预测变量，然后使用分类模型来对结果进行预测。

2.3数据准备

模型的质量在很大程度上取决于用于构建（训练）它的数据的质量。在任何给定的数据挖掘项目中花费的大部分时间都用于数据准备。必须仔细检查，清理和转换数据，并且必须应用与算法相关的数据准备方法。

由于应用模型的任何数据，无论是用于测试还是用于评分，数据准备过程都变得更加复杂，必须经历与用于训练模型的数据相同的变换。

2.3.1 Oracle数据挖掘简化了数据准备

Oracle Data Mining提供了几个可以显着简化数据准备过程的功能：

  • 嵌入式数据准备： 用于训练模型的转换嵌入在模型中，并在模型应用于新数据时自动执行。如果为模型指定转换，则只需指定一次。

  • 自动数据准备（ADP）：Oracle Data Mining支持自动数据准备模式。当ADP处于活动状态时，Oracle Data Mining会自动执行算法所需的数据转换。转换指令与任何用户指定的转换指令一起嵌入到模型中。

  • 自动管理缺失值和稀疏数据： Oracle Data Mining跨采矿算法使用一致的方法来处理稀疏性和缺失值。

  • 透明度：Oracle Data Mining提供模型详细信息，这些信息是模型内部属性的视图。由于反向转换可以将对转换后的属性值映射到可由用户解释的形式，因此可以深入了解模型的内部细节。在可能的情况下，属性值将反转为原始列值。逆变换也应用于监督模型的目标，因此得分结果与原始目标的单位具有相同的单位。

  • 用于自定义数据准备的工具：Oracle Data Mining在DBMS_DATA_MINING_TRANSFORM PL / SQL包中提供了许多常见的转换例程。您可以使用这些例程，或在SQL中或两者中开发自己的例程。SQL语言非常适合在数据库中实现转换。您可以使用自定义转换说明以及ADP或ADP。

2.3.2案例数据

大多数数据挖掘算法作用于单记录案例数据，其中每个案例的信息存储在单独的行中。案例的数据属性存储在列中。

在事务中组织数据时，一个案例（一个事务）的数据存储在许多行中。交易数据的一个例子是市场篮子数据。除了可以对本机事务数据进行操作的关联规则之外，Oracle数据挖掘算法需要单记录案例组织。

2.3.2.1嵌套数据

Oracle Data Mining支持嵌套列中的属性。事务表可以转换为嵌套列，并包含在单记录案例数据表中。同样，星型模式可以转换为嵌套列。通过嵌套数据转换，Oracle Data Mining可以有效地挖掘源自多个源和配置的数据。

2.3.3文本数据

准备和转换非结构化文本数据以进行数据挖掘。

Oracle Data Mining自动将CLOB列和长VARCHAR2列解释为非结构化文本。此外，您可以将short VARCHAR2 ， CHAR ， BLOB和BFILE列指定为非结构化文本。非结构化文本包括数据项，如网页，文档库，Power Point演示文稿，产品规格，电子邮件，报告中的注释字段以及呼叫中心备注。

Oracle Data Mining使用Oracle Text实用程序和术语加权策略来转换非结构化文本以进行挖掘。在文本转换中，提取文本术语并在文本索引中给出数值。文本转换过程可针对模型和单个属性进行配置。一旦转换，文本可以通过数据挖掘算法挖掘。

2.4数据库内评分

评分是将数据挖掘算法应用于新数据。在传统的数据挖掘中，模型使用远程系统上的专用软件构建，并部署到另一个系统进行评分。这是一个麻烦的，容易出错的过程，容易受到安全违规和数据同步的困扰。

使用Oracle Data Mining，评分简单而安全。评分引擎和数据都驻留在数据库中。评分是SQL语言的扩展，因此挖掘结果可以轻松地合并到应用程序和报告系统中。

2.4.1并行执行和易于管理

所有Oracle Data Mining评分例程都支持并行执行以评估大型数据集。

在数据库评分提供了性能优势。所有Oracle Data Mining评分例程都支持并行执行，可显着减少执行复杂查询和评分大型数据集所需的时间。

数据库内挖掘可最大限度地减少支持数据挖掘计划所需的IT工作量。使用标准数据库技术，可以轻松地在更新的数据上刷新（重新创建）模型并重新部署。部署是立即的，因为评分查询保持不变;只在数据库中替换基础模型。

2.4.2模型应用和动态评分的SQL函数

在Oracle Data Mining中，评分由SQL语言函数执行。了解SQL函数评分涉及的不同方式。

这些函数执行预测，聚类和特征提取。可以通过两种不同的方式调用这些函数：通过应用挖掘模型对象（ 示例2-1 ），或者通过执行分析子句来动态计算挖掘分析并将其应用于数据（ 示例2-2 ）。动态评分消除了对模型的需求，可以补充甚至取代“数据挖掘过程”中描述的更传统的数据挖掘方法。

在示例2-1中 ， PREDICTION_PROBABILITY函数应用示例1-1中创建的模型svmc_sh_clas_sample来对mining_data_apply_v的数据进行mining_data_apply_v 。该功能返回意大利最有可能使用亲和卡的十位客户。
