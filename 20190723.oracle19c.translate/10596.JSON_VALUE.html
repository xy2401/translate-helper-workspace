目的

SQL / JSON函数JSON_VALUE在JSON数据中查找指定的标量JSON值，并将其作为SQL值返回。

“ 数据库全球化支持指南”中的附录C，用于排序规则派生规则，它定义分配给此函数返回的值的排序规则，当它是字符值时

EXPR

使用此子句指定要评估的JSON数据。对于expr ，请指定一个计算结果为文本文字的表达式。如果expr是列，则该列必须是数据类型VARCHAR2 ， CLOB或BLOB 。如果expr为null，则该函数返回null。

如果expr不是使用strict或lax语法的格式良好的JSON数据的文本文本，则该函数默认返回null。您可以使用JSON_value_on_error_clause覆盖此默认行为。请参阅JSON_value_on_error_clause 。

格式JSON

如果expr是数据类型为BLOB的列，则必须指定FORMAT JSON 。

JSON_basic_path_expression

使用此子句指定SQL / JSON路径表达式。该函数使用路径表达式来计算expr并找到匹配或满足路径表达式的标量JSON值。路径表达式必须是文本文字。有关JSON_basic_path_expression的完整语义，请参阅Oracle Database JSON Developer's Guide 。

JSON_value_returning_clause

使用此子句指定此函数返回的值的数据类型和格式。

RETURNING

使用RETURNING子句指定返回值的数据类型。如果省略此子句，则JSON_VALUE返回类型为VARCHAR2(4000) 。

JSON_value_return_type :: =

您可以使用JSON_value_return_type指定以下数据类型：

  • VARCHAR2[( size [BYTE,CHAR])]

    如果指定此数据类型，则此函数返回的标量值可以是字符或数字值。数值将隐式转换为VARCHAR2 。在SQL中的其他位置指定VARCHAR2数据类型时，您需要指定大小。但是，在本节中，您可以省略大小。在这种情况下， JSON_VALUE返回VARCHAR2(4000)类型的值VARCHAR2(4000) 。

    如果返回值大于N字符，请在VARCHAR2(N)之后立即指定可选的TRUNCATE子句，以将返回值截断为N字符。

  • CLOB

    指定此数据类型以返回包含单字节或多字节字符的字符大对象。

  • NUMBER[( precision [, scale ])]

    如果指定此数据类型，则此函数返回的标量值必须是数字值。返回的标量值也可以是JSON布尔值。但请注意，不推荐为JSON布尔值返回NUMBER 。

  • 日期

    如果指定此数据类型，则此函数返回的标量值必须是可以隐式转换为DATE数据类型的字符值。

  • TIMESTAMP

    如果指定此数据类型，则此函数返回的标量值必须是可以隐式转换为TIMESTAMP数据类型的字符值。

  • TIMESTAMP WITH TIME ZONE

    如果指定此数据类型，则此函数返回的标量值必须是可以隐式转换为TIMESTAMP WITH TIME ZONE数据类型的字符值。

  • SDO_GEOMETRY

    此数据类型用于Oracle Spatial和Graph数据。如果指定此数据类型，则expr必须求值为包含GeoJSON数据的文本文字，这是一种用于在JSON中编码地理数据的格式。如果指定此数据类型，则此函数返回的标量值必须是SDO_GEOMETRY类型的对象。

  • JSON_value_return_object_instance

    如果JSON_VALUE以JSON对象为目标，并且您将用户定义的SQL对象类型指定为返回类型，则JSON_VALUE将在object_type_name中返回该对象类型的实例。

    有关示例，请参阅使用JSON_VALUE实例化用户定义的对象类型实例

ASCII

指定ASCII以使用标准ASCII Unicode转义序列自动转义返回值中的任何非ASCII Unicode字符。

JSON_value_on_error_clause

发生以下错误时，使用此子句指定此函数返回的值：

  • 使用严格或宽松的JSON语法， expr不是格式良好的JSON数据

  • 使用SQL / JSON路径表达式计算JSON数据时，会找到非标量值

  • 使用SQL / JSON路径表达式评估JSON数据时找不到匹配项。您可以通过指定JSON_value_on_empty_clause来覆盖此类错误的行为。

  • 返回值数据类型不足以容纳返回值

您可以指定以下子句：

  • NULL ON ERROR - 发生错误时返回null。这是默认值。

  • ERROR ON ERROR - 发生错误时返回相应的Oracle错误。

  • DEFAULT literal ON ERROR - 发生错误时返回literal 。literal的数据类型必须与此函数返回的值的数据类型匹配。

JSON_value_on_empty_clause

如果在使用SQL / JSON路径表达式计算JSON数据时未找到匹配项，请使用此子句指定此函数返回的值。此子句允许您为此类错误指定与使用JSON_value_on_error_clause指定的结果不同的结果。

您可以指定以下子句：

  • NULL ON EMPTY - 未找到匹配项时返回null。

  • ERROR ON EMPTY - 未找到匹配项时返回相应的Oracle错误。

  • DEFAULT literal ON EMPTY - 未找到匹配时返回literal 。literal的数据类型必须与此函数返回的值的数据类型匹配。

如果省略此子句，则JSON_value_on_error_clause确定未找到匹配时返回的值。

JSON_value_on_mismatch_clause

您可以通过两种方式使用JSON_value_on_mismatch_clause ：一般情况或逐个案例。

通常用它来应用于所有错误情况，如额外数据，缺失数据和类型错误。

通过为每种情况指定不同的ON MISMATCH子句来逐个使用它。例如：

IGNORE ON MISMATCH（额外数据）
错误的错误（缺少数据，类型错误）

例子

以下查询返回具有属性名称a的成员的值。由于未指定RETURNING子句，因此值将作为VARCHAR2(4000)数据类型返回：

SELECT JSON_VALUE（'{a：100}'，'$ .a'）AS值FROM DUAL;价值----- 100

以下查询返回具有属性名称a的成员的值。由于指定了RETURNING NUMBER子句，因此该值将作为NUMBER数据类型返回：

SELECT JSON_VALUE（'{a：100}'，'$ .a'RETURNING NUMBER）AS FROM FROM DUAL; VALUE ---------- 100

以下查询返回具有属性名称b的成员的值，该属性名称a具有属性名称a的成员的值：

SELECT JSON_VALUE（'{a：{b：100}}'，'$ .ab'）AS值FROM DUAL;价值----- 100

以下查询返回任何对象中具有属性名称d的成员的值：

SELECT JSON_VALUE（'{a：{b：100}，c：{d：200}，e：{f：300}}'，'$。* .d'）AS值FROM DUAL;价值----- 200

以下查询返回数组中第一个元素的值：

SELECT JSON_VALUE（'[0,1,2,3]'，'$ [0]'）AS值FROM DUAL; VALUE ----- 0

以下查询返回数组中第三个元素的值。该数组是具有属性名称a的成员的值。 

SELECT JSON_VALUE（'{a：[5,10,15,20}}'，'$ .a [2]'）AS值FROM DUAL;价值----- 15

以下查询返回数组中第二个对象中具有属性名称a的成员的值：

SELECT JSON_VALUE（'[{a：100}，{a：200}，{a：300}]'，'$ [1] .a'）AS值FROM DUAL;价值----- 200

以下查询返回数组中任何对象中具有属性名称c的成员的值：

SELECT JSON_VALUE（'[{a：100}，{b：200}，{c：300}]'，'$ [*]。c'）AS值FROM DUAL;价值----- 300

以下查询尝试返回具有属性名lastname的成员的值。但是，这样的成员在指定的JSON数据中不存在，导致不匹配。由于未指定ON ERROR子句，因此该语句使用默认的NULL ON ERROR并返回null。 

SELECT JSON_VALUE（'{firstname：“John”}'，'$ .lastname'）AS“Last Name”FROM DUAL;姓  -  -  -  - -

SELECT JSON_VALUE（'{firstname：“John”}'，'$ .lastname'DEFAULT'找不到姓氏'ON ERROR）AS“姓氏”FROM DUAL;姓氏---------找不到姓氏
