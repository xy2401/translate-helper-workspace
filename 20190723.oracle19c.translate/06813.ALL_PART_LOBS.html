3.5 ALL_PART_LOBS

ALL_PART_LOBS显示有关当前用户可访问的分区LOB的表级信息，包括LOB数据分区的默认属性。

相关观点

  • DBA_PART_LOBS显示有关数据库中所有分区LOB的表级信息。

  • USER_PART_LOBS显示有关当前用户拥有的分区LOB的表级信息。此视图不显示TABLE_OWNER列。

柱 数据类型 空值 描述

TABLE_OWNER

VARCHAR2（128）

不是NULL

包含LOB的分区表的所有者

TABLE_NAME

VARCHAR2（128）

不是NULL

包含LOB的分区表的名称

COLUMN_NAME

VARCHAR2（4000）

LOB列的名称

LOB_NAME

VARCHAR2（128）

不是NULL

分区LOB的名称

LOB_INDEX_NAME

VARCHAR2（128）

不是NULL

分区LOB索引的名称

DEF_CHUNK

数

不是NULL

添加分区时要使用的LOB数据分区的默认值CHUNK

DEF_PCTVERSION

数

不是NULL

添加分区时要使用的LOB数据分区的PCTVERSION默认值

DEF_CACHE

VARCHAR2（10）

指示缓冲区高速缓存中是否以及如何缓存LOB数据：

  • YES - LOB数据放在缓冲区缓存中

  • NO - LOB数据要么不被带入缓冲区高速缓存，要么被带入缓冲区高速缓存并放置在最近最少使用的LRU列表末尾

  • CACHEREADS - LOB数据仅在读取操作期间进入缓冲区高速缓存，但在写入操作期间不进入缓冲区高速缓存

DEF_IN_ROW

VARCHAR2（3）

指示LOB数据<4000字节是否默认内联（在行中）存储（ YES ）或不存储（ NO ）;也就是说，在创建或最后更改LOB列时是否指定了ENABLE STORAGE IN ROW

DEF_TABLESPACE_NAME

VARCHAR2（30）

添加分区时要使用的LOB数据分区的默认表空间

DEF_INITIAL_EXTENT

VARCHAR2（40）

添加分区时要使用的LOB数据分区的缺省值INITIAL ，如果未指定INITIAL值则为DEFAULT

DEF_NEXT_EXTENT

VARCHAR2（40）

添加分区时要使用的LOB数据分区的NEXT默认值，如果未指定NEXT值，则为DEFAULT

DEF_MIN_EXTENTS

VARCHAR2（40）

添加分区时要使用的LOB数据分区的MINEXTENTS默认值，如果未指定MINEXTENTS值，则为DEFAULT

DEF_MAX_EXTENTS

VARCHAR2（40）

添加分区时要使用的LOB数据分区的默认值MAXEXTENTS ;如果未指定MAXEXTENTS值，则为DEFAULT

DEF_MAX_SIZE

VARCHAR2（40）

添加分区时要使用的LOB数据分区的默认值MAXSIZE ;如果未指定MAXSIZE值，则为DEFAULT

DEF_RETENTION

VARCHAR2（7）

添加分区时要使用的LOB数据分区的RETENTION默认值。

SecureFiles的可能值：

  • 没有

  • 汽车

  • MIN

  • MAX

  • 默认

  • 无效

BasicFiles的可能值：

  • 是

  • 没有

DEF_MINRET

VARCHAR2（40）

添加分区时要使用的LOB数据分区的RETENTION MIN默认值，如果未指定RETENTION MIN值，则为DEFAULT

DEF_PCT_INCREASE

VARCHAR2（40）

添加分区时要使用的LOB数据分区的PCTINCREASE缺省值，如果未指定PCTINCREASE值，则为DEFAULT

DEF_FREELISTS

VARCHAR2（40）

添加分区时要使用的LOB数据分区的FREELISTS默认值，如果未指定FREELISTS值，则为DEFAULT

DEF_FREELIST_GROUPS

VARCHAR2（40）

添加分区时要使用的LOB数据分区的FREELIST GROUPS默认值，如果未指定FREELIST GROUPS值，则为DEFAULT

DEF_LOGGING

VARCHAR2（7）

添加分区时要使用的LOB数据分区的默认LOGGING属性：

  • NONE - 未指定

    另请参见： *_LOBS和*_LOB_PARTITIONS视图

  • 是

  • 没有

DEF_BUFFER_POOL

VARCHAR2（7）

添加分区时要使用的LOB数据分区的默认缓冲池：

  • 默认

  • 保持

  • 回收

  • 空值

DEF_FLASH_CACHE

VARCHAR2（7）

添加分区时使用的默认数据库智能闪存缓存提示：

  • 默认

  • 保持

  • 没有

仅限Solaris和Oracle Linux功能。

DEF_CELL_FLASH_CACHE

VARCHAR2（7）

添加分区时要使用的默认单元刷新缓存提示：

  • 默认

  • 保持

  • 没有

另请参阅： Oracle Exadata Storage Server软件文档以获取更多信息

DEF_ENCRYPT

VARCHAR2（4）

添加分区时要使用的LOB数据分区的默认值ENCRYPT 。

SecureFiles的可能值：

  • 是

  • 没有

BasicFiles的可能值：

  • NONE - 不适用

DEF_COMPRESS

VARCHAR2（6）

添加分区时要使用的LOB数据分区的COMPRESS默认值。

SecureFiles的可能值：

  • 低

  • 介质

  • 高

  • NO - 压缩已关闭

BasicFiles的可能值：

  • NONE - 不适用

DEF_DEDUPLICATE

VARCHAR2（15）

添加分区时要使用的LOB数据分区的DEDUPLICATE缺省值。

SecureFiles的可能值：

  • LOB - 重复数据删除

  • NO - 保持重复

BasicFiles的可能值：

  • NONE - 不适用

DEF_SECUREFILE

VARCHAR2（3）

指示LOB是否为SecureFiles（ YES ）（ NO ）

也可以看看：

  • “ DBA_PART_LOBS ”

  • “ USER_PART_LOBS ”
