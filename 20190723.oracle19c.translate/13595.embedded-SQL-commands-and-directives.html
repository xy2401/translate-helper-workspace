E.3.5数据库对象

该 Oracle对象的名称（例如表和列）的长度不得超过30个字符。第一个字符必须是字母，但其余字母可以是字母，数字，美元符号（$），井号（＃）和下划线（_）的任意组合。

但是，如果一个 Oracle标识符用引号（“）括起来，它可以包含任何合法字符组合，包括空格但不包括引号。

Oracle标识符不区分大小写，除非用引号括起来。

E.6 COMMIT（可执行嵌入式SQL）

E.6.1 COMMIT目的

要结束当前事务，请对数据库进行永久性更改，并可选择释放所有资源并断开与Oracle数据库的连接。

E.6.2 COMMIT先决条件

要提交当前事务，不需要任何特权。

要手动提交最初提交的分布式不确定事务，您必须具有FORCE TRANSACTION系统特权。要手动提交最初由其他用户提交的分布式不确定事务，您必须具有FORCE ANY TRANSACTION系统特权。

如果在DBMS MAC模式下使用Oracle，则只有在DBMS标签与事务标签的标签和最初提交事务的用户的创建标签匹配时，或者如果满足以下条件之一时，才能提交不确定事务：

  • 如果交易的标签或用户创建的标签比你的DBMS标签越高，你必须有READUP ，并WRITEUP系统权限。

  • 如果事务的标签或用户的创建标签低于DBMS标签，则必须具有WRITEDOWN系统特权。

  • 如果交易的标签或用户创建的标签不符合你的DBMS标签相媲美，你必须有READUP ， WRITEUP ，并WRITEDOWN的系统权限。

E.6.5 COMMIT使用说明

始终使用COMMIT或ROLLBACK命令和RELEASE选项显式提交或回滚程序中的最后一个事务。如果程序异常终止，Oracle会自动回滚更改。

COMMIT命令对主机变量或程序中的控制流没有影响。有关此命令的更多信息，请参阅定义和控制事务 。

例

此示例说明了嵌入式SQL COMMIT命令的用法：

您可以使用CURRENT OF语法在UPDATE或DELETE语句的WHERE子句中引用游标，然后使用OPEN语句打开游标并将其定位在具有FETCH语句的行上。有关此命令的更多信息，请参阅会议程序要求 。

E.17.5 FETCH使用说明

FETCH语句读取活动集的行，并命名包含结果的输出变量。如果指示符值的关联主机变量为空，则将其设置为-1。如果需要，游标的第一个FETCH语句还会对活动集的行进行排序。

  • 最小数组的大小

  • 可选FOR子句中的： host_integer的值

当然，获取的行数可以进一步受实际满足查询的行数限制。

如果FETCH语句未检索查询返回的所有行，则光标将定位在下一个返回的行上。当检索到查询返回的最后一行时，下一个FETCH语句将导致在SQLCA的SQLCODE元素中返回错误代码。

请注意， FETCH命令不包含AT子句。您必须在DECLARE CURSOR语句中指定光标访问的数据库。

您只能使用FETCH语句向前移动活动集。如果要重新访问以前获取的任何行，则必须重新打开游标并依次获取每一行。如果要更改活动集，则必须在游标的查询中为输入主机变量分配新值，然后重新打开游标。

E.21.4回滚关键字和参数

工作

是可选的，提供ANSI兼容性。

E.21.5 ROLLBACK使用说明

事务（或逻辑工作单元）是Oracle将其视为单个单元的一系列SQL语句。事务以COMMIT ， ROLLBACK或与数据库的连接之后的第一个可执行SQL语句开始。交易以数据库中的COMMIT语句， ROLLBACK语句或断开连接（有意或无意）结束。请注意，Oracle在处理任何数据定义语言语句之前和之后发出隐式COMMIT语句。

使用不带TO SAVEPOINT子句的ROLLBACK命令执行以下操作：

  • 结束交易

  • 撤消当前事务中的所有更改

  • 擦除事务中的所有保存点

  • 释放事务的锁

使用带有TO SAVEPOINT子句的ROLLBACK命令可执行以下操作：

  • 在保存点之后回滚事务的一部分。

  • 丢失在该保存点之后创建的所有保存点。请注意，保留了命名的保存点，因此您可以多次回滚到同一个保存点。之前的保存点也会保留。

  • 释放自保存点以来获取的所有表锁和行锁。请注意，已请求访问保存点之后锁定的行的其他事务必须继续等待，直到提交或回滚事务为止。尚未请求行的其他事务可以立即请求和访问行。

建议您使用COMMIT或ROLLBACK语句显式结束应用程序中的事务。如果您没有显式提交事务并且程序异常终止，则Oracle将回滚最后一个未提交的事务。

E.21.6回滚示例I

以下语句回滚整个当前事务：

EXEC SQL ROLLBACK;

E.21.7回滚示例II

以下语句将当前事务回滚到保存点SP5：

执行SQL ROLLBACK到SAVEPOINT sp5;

E.21.8 ROLLBACK分布式事务

具有分布式选项的Oracle可以执行分布式事务或修改多个数据库上的数据的事务。要提交或回滚分布式事务，您只需像发出任何其他事务一样发出COMMIT或ROLLBACK语句。

如果在分布式事务的提交过程中存在网络故障，则事务的状态可能是未知的或不确定的。在咨询事务中涉及的其他数据库的管理员之后，您可以决定在本地数据库上手动提交或回滚事务。您可以通过发出带有FORCE子句的ROLLBACK语句来手动回滚本地数据库上的事务。

您无法手动将不确定事务回滚到保存点。

带有FORCE子句的ROLLBACK语句仅回滚指定的事务。这样的陈述不会影响您当前的交易。

E.21.9回滚示例III

以下语句手动回滚一个不确定的分布式事务：

EXEC SQL ROLLBACK WORK FORCE '25 .32.87';
