安装后修改Oracle Clusterware二进制文件

安装后，如果需要修改Oracle Clusterware配置，则必须解锁Grid home。查看有关解锁Grid主页的信息。

例如，如果要应用一次性修补程序，或者如果要修改Oracle Exadata配置以在互连上通过RDS运行IPC流量而不是使用默认UDP，则必须解锁Grid home。

警告：

在重新链接可执行文件之前，必须关闭正在重新链接的Oracle主目录中运行的所有可执行文件。此外，关闭与Oracle共享库链接的应用程序。

使用以下步骤解锁住宅：

  1. 转到/u01/app/19.0.0/grid/crs/install目录，使用命令rootcrs.sh -unlock解锁Grid home：

    #cd /u01/app/19.0.0/grid/crs/install#./rootcrs.sh -unlock

  2. 将用户更改为Oracle Grid Infrastructure软件所有者，并使用命令语法make -f /u01/app/19.0.0/grid/rdbms/lib/ins_rdbms.mk target重新链接二进制文件，其中target是要重新链接的二进制文件。例如，在将互连协议从UDP更新到IPC的位置，请输入以下命令：

    #su grid $ make -f /u01/app/19.0.0/grid/rdbms/lib/ins_rdbms.mk ipc_rds ioracle

    注意：

    要重新链接二进制文件，您还可以更改为网格安装所有者并运行命令/u01/app/19.0.0/grid/bin/relink 。

  3. 重新启动Grid home并重新启动群集，如下所示：

    ＃。/ rootcrs.sh -lock #crsctl start crs

    在每个集群成员节点上重复步骤1到3。

注意：

不要删除Grid home中的目录。例如，不要删除目录Grid_home / OPatch。如果删除目录，则Grid Infrastructure安装所有者无法使用OPatch修补Grid home，并且OPatch显示错误消息“ checkdir error: cannot create Grid_home /OPatch ”。
