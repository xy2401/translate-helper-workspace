1全球化支持概述

本章概述了Oracle数据库的全球化支持。本章讨论以下主题：

  • 全球化支持架构

1.1全球化支持架构

Oracle数据库中的全球化支持使您能够以本机语言存储，处理和检索数据。它确保数据库实用程序，错误消息，排序顺序以及日期，时间，货币，数字和日历约定自动适应任何本地语言和区域设置。

过去，Oracle将全球化支持功能称为国家语言支持（NLS）功能。NLS实际上是全球化支持的一个子集。NLS能够选择国家语言并将数据存储在特定字符集中。全球化支持使您能够开发可以在世界任何地方同时访问和运行的多语言应用程序和软件产品。应用程序可以呈现用户界面的内容，并以本机用户的语言和区域设置首选项处理数据。

1.1.1区域设置数据点播

Oracle数据库全球化支持通过Oracle NLS运行时库（NLSRTL）实现。NLS RTL提供了一整套独立于语言的函数，可以执行正确的文本和字符处理以及语言约定操作。这些函数对特定语言和区域的行为由一组在运行时标识和加载的特定于语言环境的数据控制。

特定于语言环境的数据被构建为Oracle数据库支持的每个语言环境的独立数据集。可以独立于其他区域设置数据加载特定区域设置的数据。

这种设计的优点如下：

  • 您可以通过选择所需的语言环境集来管理内存消耗。

  • 您可以添加和自定义特定区域设置的区域设置数据，而不会影响其他区域设置。

1.1.2支持多语言应用的架构

Oracle数据库使多层应用程序和客户端/服务器应用程序能够支持配置数据库的语言。

依赖于语言环境的操作由客户端和数据库服务器上的多个参数和环境变量控制。在数据库服务器上，代表客户端启动的每个会话可以在与其他会话相同或不同的语言环境中运行，并且可以指定相同或不同的语言要求。

Oracle数据库具有一组与会话无关的NLS参数，这些参数在创建数据库时指定。其中两个参数指定数据库字符集和国家字符集，这是可以为NCHAR ， NVARCHAR2和NCLOB数据指定的备用Unicode字符集。参数指定用于在数据库中存储文本数据的字符集。其他参数（如语言和区域）用于评估和检查约束。

如果客户端会话和数据库服务器指定了不同的字符集，则数据库会自动转换字符集字符串。

从全球化支持的角度来看，所有应用程序都被视为客户端，即使它们与Oracle数据库实例在同一台物理计算机上运行。例如，当SQL用户从安装了RDBMS软件的Oracle主目录中拥有Oracle数据库软件的UNIX用户启动时，SQL * Plus通过指定ORACLE_SID参数通过适配器连接到数据库，SQL * Plus被视为客户。它的行为由客户端NLS参数决定。

当中间层是应用程序服务器时，发生被视为客户端的应用程序的另一示例。应用程序服务器生成的不同会话被视为单独的客户端会话。

启动客户端应用程序时，它会从环境设置初始化客户端NLS环境。使用这些设置执行本地执行的所有NLS操作。本地NLS操作的示例是：

  • 显示Oracle Developer应用程序中的格式

  • 使用OCI环境句柄执行NLS OCI功能的用户OCI代码

当应用程序连接到数据库时，将在服务器上创建会话。新会话从初始化参数文件中指定的NLS实例参数初始化其NLS环境。随后可以通过ALTER SESSION语句更改这些设置。该语句仅更改会话NLS环境。它不会更改本地客户端NLS环境。会话NLS设置用于处理在服务器上执行的SQL和PL / SQL语句。例如，使用ALTER SESSION语句将NLS_LANGUAGE初始化参数设置为意大利语：

ALTER SESSION SET NLS_LANGUAGE =意大利语;

输入SELECT语句：

SQL> SELECT last_name，hire_date，ROUND（薪水/ 8,2）薪水来自员工;

请注意，月份名称缩写为意大利语。

建立连接后，如果在客户端定义了NLS_LANG环境设置，则隐式ALTER SESSION语句会立即同步客户端和会话NLS环境。

1.1.3在多语言数据库中使用Unicode

Unicode是通用编码字符集，使您可以使用单个字符集以任何语言存储信息。Unicode为每个角色提供唯一的代码值，无论平台，程序或语言如何。Oracle建议使用AL32UTF8作为数据库字符集。AL32UTF8是Unicode标准的UTF-8编码形式的正确实现。

注意：

从Oracle Database 12 c第2版开始，如果使用Oracle Universal Installer（OUI）或Oracle Database Configuration Assistant（DBCA）创建数据库，则使用的默认数据库字符集是Unicode字符集AL32UTF8 。

Unicode具有以下优点：

  • 简化字符集转换和语言排序功能。

  • 与本机多字节字符集相比，提高了性能。

  • 支持基于Unicode标准的Unicode数据类型。

为了帮助您迁移到Unicode环境，Oracle提供了Unicode数据库迁移助手（DMU）。DMU是一个直观且用户友好的GUI，可通过界面简化迁移过程，最大限度地减少工作负载并确保解决所有迁移问题，同时确保正确有效地执行数据转换。与过去的数据迁移方法相比，DMU具有许多优势，其中一些方法是：

  • 它将指导您完成工作流程。

  • 它提供了处理某些问题的建议，例如清理数据时的失败。

  • 它支持选择性地转换数据。

  • 它提供进度监控。

1.2.1语言支持

Oracle数据库使您能够以本机语言存储，处理和检索数据。可以存储在数据库中的语言都是用脚本编写的语言，这些脚本由Oracle支持的字符集编码。通过使用Unicode数据库和数据类型，Oracle数据库支持大多数当代语言。

可以为一部分语言提供附加支持。例如，数据库可以使用翻译的月份名称显示日期，并且可以根据文化惯例对文本数据进行排序。

当本文档使用术语“ 语言支持”时 ，它指的是附加的语言相关功能，而不是指存储特定语言文本的能力。例如，语言支持包括根据特定的语言环境和文化习俗显示日期或排序文本。此外，对于某些受支持的语言，Oracle数据库为数据库实用程序提供已翻译的错误消息和已翻译的用户界面。

1.2.2领土支持

Oracle数据库支持特定于地理位置的文化约定。默认的本地时间格式，日期格式以及数字和货币约定取决于本地区域设置。设置不同的NLS参数使数据库会话能够使用不同的文化设置。例如，您可以将欧元（ EUR ）设置为主要货币，将日元（ JPY ）设置为给定数据库会话的辅助货币，即使区域定义为AMERICA 。

1.2.10 Unicode支持

Unicode是一种行业标准，它使所有语言的文本和符号能够由计算机一致地表示和操作。

自Oracle 7以来，Oracle数据库一直遵循Unicode标准。随后，Oracle Database 10g第 2版支持Unicode 4.0。Oracle Database 11g发行版支持Unicode 5.0。Oracle Database 12 c第1版支持Unicode 6.2。Oracle Database 12 c第2版（12.2）支持Unicode 7.0。Oracle Database Release 18c及更高版本支持Unicode 9.0。

您可以通过两种方式在Oracle数据库中存储Unicode字符：

  • 您可以创建一个Unicode数据库，使您可以将UTF-8编码的字符存储为SQL CHAR数据类型VARCHAR2 ， CHAR ， LONG （不建议使用）和CLOB 。

  • 您可以使用SQL NCHAR数据类型NVARCHAR2 ， NCHAR和NCLOB支持特定列中的多语言数据。无论如何定义数据库字符集，都可以将Unicode字符存储到NCHAR数据类型的列中。NCHAR数据类型NCHAR Unicode数据类型。

    注意：

    从Oracle Database 12 c第2版（12.2）开始，如果使用Oracle Universal Installer（OUI）或Oracle Database Configuration Assistant（DBCA）创建数据库，则使用的默认数据库字符集是Unicode字符集AL32UTF8。
