本发行版中的Oracle Database PL / SQL语言参考的更改

每个版本都描述了对Oracle Database PL / SQL语言参考的更改。

本前言包含：

  • Oracle Database PL / SQL语言参考的版本19c中的更改

  • Oracle Database PL / SQL语言参考的版本18c中的更改

Oracle Database Release 19c中的更改

新功能

19c没有重要的新功能。但是，多态表功能有所增强。

从Oracle Database 19c开始，如果在同一个包中定义了多态表函数及其实现方法，则USING子句是可选的。

  • PIPELINED子句了解有关多态表函数，语法和语义的更多信息

Oracle Database Release 18c中的更改

对于Oracle Database 18c，Oracle Database PL / SQL语言参考文档记录了这些新功能。

新功能

对象类型的非持久性支持

此功能使您可以标记程序中使用的抽象数据类型，仅用于处理，仅用于存储或混合使用。

非持久性类型的实例不能在磁盘上持久存在。

也可以看看：

  • CREATE TYPE语句[NOT] PERSISTABLE子句，以获取有关语法和语义的更多信息

多态表函数

多态表函数（PTF）是一种新类型的表函数，其返回类型由传递给PTF的参数确定。

表函数是一个返回行集合的函数，可以从SQL查询块的FROM子句中调用。

新的多态表函数（PTF）提供了一个高效且可扩展的框架，以扩展Oracle数据库的分析功能。查询编写器能够在不知道PTF实现细节的情况下调用这些函数，并且PTF不需要知道细节或函数如何执行（例如，串行或并行）或输入行被分区或订购。

因此，当SQL开发人员和数据库管理员希望提供适用于任意输入表或查询的通用扩展时，PTF非常有用。

有关更多信息，请参阅多态表函数概述 。

合格的表达

聚合及其必要的附件，合格的表达式，提高了程序的清晰度和程序员的工作效率。

通过Oracle Database 12c第2版，可以使用表达式提供非标量数据类型的值，例如，通过使用嵌套表的类型构造函数。从Oracle Database Release 18c开始，任何PL / SQL值都可以由表达式（例如，对于记录或关联数组）提供，就像构造函数提供抽象数据类型值一样。在PL / SQL中，我们使用术语“限定表达式”和“聚合”而不是SQL术语“类型构造函数” ，但功能是相同的。

有关语法和语义的更多信息，请参阅限定表达式概述和qualified_expression :: = 。

不推荐的功能

此版本中不推荐使用以下功能，并且可能在将来的版本中不支持这些功能。

命令ALTER TYPE ...不推荐使用INVALIDATE 。请改用CASCADE子句。

不推荐使用ALTER TYPE的REPLACE子句。请改用alter_method_spec子句。或者，您可以使用CREATE OR REPLACE TYPE语句重新创建类型。

有关语法和语义，请参阅ALTER TYPE语句

从Oracle Database 12c第1版（12.1）开始，不推荐使用编译参数PLSQL_DEBUG。

要编译PL / SQL单元以进行调试，请指定PLSQL_OPTIMIZE_LEVEL = 1。

有关编译参数的信息，请参阅PL / SQL单元和编译参数 。

不受支持的功能

对于19c，不支持PL / SQL语言参考中的任何功能。
