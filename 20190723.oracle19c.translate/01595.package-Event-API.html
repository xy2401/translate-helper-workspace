在pull-parse模式下创建一个Event上下文。

使用XmlEvLoadPPDoc加载文档。实际的解析是由对XmlEvNext（）的多次调用驱动的。在每次调用之后，可以通过调用各种XmlEvGet XXX （）函数来检索相关信息。基本属性集与XmlLoadDom相同。应使用XmlEvLoadPPDoc（）调用指定输入源。

这些附加属性应该提供一个终端NULL ：

  • ("expand_entities", boolean) ，当为FALSE ，导致解析的非参数实体引用不被展开。默认情况下会扩展此类引用。

  • ("use_buffer", buffer)是("use_buffer", buffer)的地址，当指定时，将使用缓冲区来收集应该返回给用户的数据。get XXX ()函数将此缓冲区作为数据指针返回。

  • ("use_buffer_len", lengthOfBuffer)是缓冲区中的字节数，缓冲区的实际长度，并且不超过指定的长度。在这种情况下，仅收集部分数据，生成事件的CONT风味。对XmlEventsNext后续调用提供了其他数据。CONT风味事件的序列总是由非CONT事件终止。缓冲区可能仅部分填充。

  • ("get_id_callback", function)是回调函数的地址，用于将文本基本名称转换为8字节ID。一旦提供了这样的功能，就允许用户使用XmlEvGetTagID ， XmlEvGetAttrID ， XmlEvGetTagUriID和XmlEvGetAttrUriID 。

  • ("raw_buffer_len", length)是缓冲区中的字节数。默认情况下，此参数为256K。原始缓冲区用于读取输入数据并执行字符转换，还用于将CRLF和CR转换为LF。

  • ("error_callback", callback)提供回调函数的地址，该函数被调用以表示非法使用该事件的API。

应以下列方式使用这些可选参数：

xmlevctx * XmlEvCreatePPCtx（xmlctx * xctx，xmlerr * xerr，（“expand_entities”， mode ），（“use_buffer”， buffer） ，（“use_buffer_len”， length ），（“get_id_callback”， function ），（“raw_buffer_len”， length ），（“error_callback”， 回调 ））;
