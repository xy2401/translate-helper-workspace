Oracle数据库作为服务呈现给客户端 。数据库可以具有与之关联的一个或多个服务。

图2-3每个数据库的一个服务


“图2-3每个数据库的一个服务”的描述

销售和财务数据库分别由服务名称 sales.us.example.com和finance.us.example.com标识。服务名称是数据库的逻辑表示。实例启动时，它会使用一个或多个服务名称向侦听器注册自己。当客户端程序或数据库连接到侦听器时，它会请求与服务的连接。

服务名称可以标识多个数据库实例，而实例可以属于多个服务。因此，侦听器充当客户端和实例之间的中介，并将连接请求路由到适当的实例。连接到服务的客户端无需指定所需的实例。

2.3.1关于连接描述符

连接描述符由监听器的一个或多个协议地址和tnsnames.ora文件中目标服务的连接信息组成。示例2-1显示了映射到sales数据库的连接描述符。

示例2-1连接描述符

sales =（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（CONNECT_DATA =（SID = sales）（SERVICE_NAME = sales.us.example.com）（INSTANCE_NAME = sales） ））

如例2-1所示，连接描述符包含以下参数：

  • ADDRESS部分包含以下内容：

    • PROTOCOL参数，用于标识侦听器协议地址。协议是tcp for TCP / IP。

    • HOST参数，用于标识主机名。主机是sales-server 。

    • PORT参数，用于标识端口。端口号是1521 ，默认端口号。

    • 可选的HTTPS_PROXY和HTTPS_PROXY_PORT参数，允许数据库客户端连接遍历组织的转发Web代理。这些参数仅适用于PROTOCOL = TCPS的连接描述符。

  • CONNECT_DATA部分包含以下内容：

    • SID参数，用于标识Oracle数据库的系统标识符（SID）。SID是sales 。

    • SERVICE_NAME参数，用于标识服务。目标服务名称是名为sales.us.example.com的数据库服务。

      此连接描述符参数的值来自 SERVICE_NAMES初始化参数（SERVICE_NAMES使用最终的S）在初始化参数文件中。SERVICE_NAMES初始化参数通常是 全局数据库名称 ，包括数据库名称和域名。在该示例中， sales.us.example.com的数据库名称为sales ，域名为us.example.com 。

    • INSTANCE_NAME参数，用于标识数据库实例。实例名称是可选的。

      初始化参数文件中的INSTANCE_NAME参数默认为在安装或数据库创建期间输入的SID。

2.3.1.1关于连接描述符中的IPv6地址

主机可以使用IP版本4（IPv4）和IP版本6（IPv6）接口。解析为IPv6地址的IPv6地址和主机名可在TNS连接地址的HOST参数中使用，该地址可通过表2-1中列出的任何支持的网络命名方法获得。

Oracle Database 12 c中使用IPv6的端到端连接需要以下配置：

  • 客户端TNS连接地址必须连接到IPv6端点上的Oracle Net Listener。

  • 为Oracle Net Listener配置的数据库实例必须侦听IPv6端点上的连接请求。

对于给定的主机名，Oracle Net会尝试连接到域名系统（DNS）名称解析返回的所有IP地址，直到建立成功连接或尝试了所有地址为止。假设在示例2-1中 ， sales-server主机是仅接受客户端连接的纯IPv4主机。DNS将sales-server映射到以下IP地址：

  1. IPv6地址2001:0db8:0:0::200C:417A

  2. IPv4地址192.0.2.213

在这种情况下，Oracle Net首先尝试连接IPv6地址，因为它首先在DNS列表中。在此示例中， sales-server不支持IPv6连接，因此尝试失败。Oracle Net继续连接到IPv4地址，该地址成功。

2.3.2关于协议地址

连接描述符的地址部分是监听器的协议地址 。要连接到数据库服务，客户端首先联系通常驻留在数据库服务器上的侦听器进程。侦听器接收传入的客户端连接请求，并将这些请求发送到数据库服务器。建立连接后，客户端和数据库服务器直接通信。

侦听器配置为接受来自协议地址的客户端的请求 。此地址定义侦听器正在侦听的协议以及任何其他协议特定的信息。例如，侦听器可以配置为侦听以下协议地址：

（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521）））

上面的示例显示了TCP / IP协议地址，该地址指定侦听器的主机和端口号。使用此相同协议地址配置的客户端连接描述符可以向此侦听器发送连接请求。

2.3.3关于服务注册

connect描述符指定客户端寻求建立连接的数据库服务名称。侦听器知道哪些服务可以处理连接请求，因为Oracle数据库会动态地将此信息注册到侦听器。这种注册过程称为服务注册 。注册还为监听器提供有关每个实例可用的数据库实例和服务处理程序的信息。服务处理程序可以是调度程序或专用服务器 。

2.3.3.2指定服务处理程序

始终要使用特定服务处理程序类型的客户端可以使用连接描述符来指定服务处理程序类型。在以下示例中，连接描述符使用(SERVER=shared)在连接到数据库时请求调度程序。数据库可以配置为默认使用专用服务器。 

（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com） （SERVER = shared） ））

当侦听器收到客户端请求时，它会选择一个已注册的服务处理程序。根据所选处理程序的类型，使用的通信协议以及数据库服务器的操作系统，侦听器将执行以下操作之一：

  • 将连接请求直接交给调度员。

  • 使用调度程序或专用服务器进程的位置将重定向消息发送回客户端。然后，客户端直接连接到调度程序或专用服务器进程。

  • 生成专用服务器进程并将客户端连接传递给专用服务器进程。

在侦听器完成客户端的连接操作之后，客户端直接与Oracle数据库通信，而不会让侦听器参与。监听器恢复监听传入的网络会话。

指定服务处理程序时应考虑以下事项：

  • 如果您希望客户端使用专用服务器，则指定(SERVER=dedicated) 。如果未设置SERVER参数，则假定为共享服务器配置。但是，如果没有可用的调度程序，客户端将使用专用服务器。

  • 如果在服务器上启用了数据库驻留连接池，则指定(SERVER=pooled)以从池中获取连接。如果未在服务器上启用数据库驻留连接池，则拒绝客户端请求，并且用户会收到错误消息。

2.4.1关于调度员

共享服务器体系结构使用调度程序进程将客户端连接定向到公共请求队列。来自共享服务器进程池的空闲共享服务器进程从公共队列中获取请求。这种方法使一小部分服务器进程能够为大量客户端提供服务。共享服务器模型相对于专用服务器模型的显着优势是减少了系统资源，从而支持更多用户。

侦听器使用调度程序作为一种服务处理程序 ，它可以将客户端请求定向到该服务处理程序 。当客户端请求到达时，侦听器将执行以下操作之一：

  • 将连接请求直接交给调度员。

  • 向客户端发出重定向消息，其中包含调度程序的协议地址。然后，客户端终止与侦听器的网络会话，并使用重定向消息中提供的网络地址与调度程序建立网络会话。

听众尽可能使用直接手动。例如，当调度程序远离侦听器时，将使用重定向消息。

2.4.2关于专用服务器进程

在一个专用在服务器配置中，侦听器为专用于为客户端提供服务的每个传入客户端连接请求启动单独的专用服务器进程。会话完成后，专用服务器进程终止。由于必须为每个连接启动专用服务器进程，因此此配置可能需要比共享服务器配置更多的系统资源。

专用服务器进程是侦听器在收到客户端请求时启动的一种服务处理程序。要完成客户端/服务器连接，请执行以下操作之一：

  • 专用服务器从侦听器继承连接请求。

  • 专用服务器通知监听器其监听协议地址。侦听器在重定向消息中将协议地址传递给客户端并终止连接。客户端使用协议地址直接连接到专用服务器。

基于操作系统和传输协议选择前述动作之一。

如果客户端和数据库存在于同一台计算机上，则客户端连接可以直接传递到专用服务器进程，而无需通过侦听器。这被称为遗留协议。启动会话的应用程序为连接请求生成专用服务器进程。如果用于启动数据库的应用程序与数据库位于同一台计算机上，则会自动执行此操作。

注意：

为了使远程客户端连接到专用服务器，侦听器和数据库实例必须在同一台计算机上运行。

2.4.3关于数据库驻留连接池

数据库驻留连接池在数据库服务器中为典型的Web应用程序使用方案提供连接池，其中应用程序获取数据库连接，在相对较短的时间内对其进行处理，然后释放它。数据库驻留连接池池“专用”服务器。池化服务器相当于服务器前台进程和数据库会话的组合。数据库驻留连接池使用服务器和侦听器之间的动态注册。它不能使用静态注册。

数据库驻留连接池补充了在中间层进程中的线程之间共享连接的中间层连接池。此外，它还支持在同一中间层主机上甚至跨中间层主机的中间层进程之间共享数据库连接。这样可以显着减少支持大量客户端连接所需的关键数据库资源，从而减少数据库层内存占用并提高中间层和数据库层的可伸缩性。拥有一个随时可用的服务器池可以降低创建和关闭客户端连接的成本。

数据库驻留连接池为池提供池跨客户端应用程序和进程的专用连接 。此功能对于必须维护与数据库的持久连接并优化服务器资源（例如内存）的应用程序非常有用。

从数据库驻留连接池中获取连接的客户端持久连接到后台进程，即连接代理，而不是专用服务器。连接代理实现池功能，并执行从客户端到具有会话的专用服务器池的入站连接的多路复用。

当客户端必须执行数据库工作时，连接代理会从池中选择一个专用服务器并将其分配给客户端。随后，客户端直接连接到专用服务器，直到请求被提供。服务器完成客户端请求的处理后，服务器返回池中，客户端的连接将恢复到连接代理。

2.5了解命名方法

命名方法是客户端应用程序在尝试连接到数据库服务时用于将连接描述符解析为连接描述符的解析方法。用户通过提供连接字符串来发起连接请求。连接字符串包括用户名和密码以及连接标识符 。连接标识符可以是连接描述符，也可以是解析为连接描述符的名称。连接描述符包含：

  • 到服务的网络路由，包括通过协议地址的侦听器的位置

  • 数据库服务名称或Oracle系统标识符（SID）

以下CONNECT命令使用具有完整连接描述符的连接字符串作为连接标识符而不是网络服务名称。该字符串应输入一行。由于页面宽度，它显示在两行上。 

SQL> CONNECT hr @（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server1）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））

最常见的连接标识符之一是网络服务名称 ，即服务的简单名称。以下CONNECT命令使用连接字符串，该字符串使用网络服务名称sales作为连接标识符：

SQL> CONNECT hr @ sales

使用网络服务名称sales ，首先将sales映射到连接描述符，从而进行连接处理。通过命名方法 s访问该映射信息。以下命名方法可用：

  • 本地命名

  • 目录命名

  • 轻松连接命名

  • 外部命名

  • 选择命名方法
  • 使用命名方法建立客户端会话
  • 输入连接字符串

2.5.1选择命名方法

选择用于将名称映射到连接描述符的适当命名方法取决于组织的大小。

  • 对于只有少数数据库的小型组织，请使用Easy Connect命名与数据库服务器的主机名建立TCP / IP连接，或使用本地命名在客户端的tnsnames.ora文件中存储名称。

  • 对于具有多个数据库的大型组织，请使用目录命名将名称存储在集中式目录服务器中。

  • 对于Internet网络，使用本地命名方法配置连接到数据库所需的应用程序Web服务器。

表2-1总结了每种命名方法的相对优缺点，并提供了在网络中使用它们的建议。

表2-1命名方法：优点和缺点

命名方法 描述 优点缺点 推荐用于：

本地命名

优点 ：

  • 提供解析网络服务名称地址的简单方法。

  • 跨运行不同协议的网络解析网络服务名称。

优点 ：

  • 在一个地方集中网络名称和地址，便于管理名称更改和更新。这使管理员无需更改可能是数百甚至数千个客户端的内容。

  • 目录存储其他服务的名称。

  • 工具提供简单的配置。

大型复杂网络（超过20个数据库）经常变化。

  • 客户端和服务器使用TCP / IP进行连接。

  • 不需要需要更高级连接描述符的功能。

优点 ：使管理员能够使用他们熟悉的工具和实用程序将Oracle网络服务名称加载到其本机名称服务中。

缺点 ：需要使用Oracle Net产品无法管理的第三方命名服务。

2.5.2使用命名方法建立客户端会话

使用命名方法建立客户端会话的典型过程如下：

  1. 客户端通过提供连接标识符来发起连接请求。

  2. 连接标识符通过命名方法解析为连接描述符。

  3. 客户端向连接描述符中提供的地址发出连接请求。

  4. 侦听器接收请求并将其定向到相应的数据库服务器。

  5. 数据库服务器接受该连接。

注意：

除连接描述符外，您还可以使用命名方法将连接名称映射到协议地址或协议地址列表。

2.5.3输入连接字符串

启动网络组件后，您应该能够通过网络建立连接。如何建立连接取决于命名方法和用于连接的工具。

连接字符串采用以下格式：

CONNECT username @ connect_identifier

在大多数操作系统上，您可以定义默认连接标识符 。使用默认值时，不需要在连接字符串中指定连接标识符。要定义默认连接标识符，请使用 Linux和UNIX平台上的TWO_TASK环境变量 Microsoft Windows上的LOCAL环境变量或注册表项。

例如，如果将TWO_TASK环境变量设置为sales ，则可以使用CONNECT username而不是CONNECT username @sales从SQL * Plus连接到数据库。Oracle Net检查TWO_TASK变量，并使用值sales作为连接标识符。

2.5.3.1连接标识符和连接描述符语法特征

连接字符串中使用的连接标识符不能包含空格，除非用单引号（'）或双引号（“）括起来。在以下示例中，连接标识符和包含空格的连接描述符包含在单引号中：

CONNECT scott @'（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = sales.us.example.com）））'输入密码： password CONNECT scott @'cn = sales，cn = OracleContext，dc = us，dc = example，dc = com'输入密码： password

如果在连接标识符中使用双引号，则需要单引号。例如：

CONNECT scott @'sales @“good”example.com'输入密码： password

同样，如果在连接标识符中使用单引号，则需要双引号。例如：

CONNECT scott @“cn = sales，cn = OracleContext，ou = Mary's Dept，o = example”输入密码： password

2.6.1关于连接时故障转移

该 连接时故障转移功能使客户端能够连接到另一个侦听器，如果与第一个侦听器的初始连接失败。侦听器协议地址的数量决定了尝试的侦听器数量。如果没有连接时故障转移，Oracle Net将尝试仅与一个侦听器建立连接。

父主题： 使用多个侦听器增强服务可访问性

2.6.2关于透明应用程序故障转移

该 透明应用程序故障转移（TAF） feature是高可用性环境的运行时故障转移，例如Oracle Real Application Clusters。TAF故障转移并重新建立应用程序到服务的连接。它使客户端应用程序能够在连接失败时自动重新连接到数据库，并可选择恢复正在进行的SELECT语句。重新连接从Oracle调用接口（OCI）库中自动发生。

父主题： 使用多个侦听器增强服务可访问性

2.6.3关于客户端负载平衡

该 客户端负载均衡 功能使客户端能够在侦听器之间随机化连接请求。Oracle Net以随机顺序遍历协议地址列表，平衡各种侦听器的负载。如果没有客户端负载平衡，Oracle Net将按顺序逐步执行协议地址列表，直到成功完成。

父主题： 使用多个侦听器增强服务可访问性

2.6.4关于连接负载平衡

该 连接负载平衡功能通过平衡多个调度程序之间的活动连接数来提高连接性能。在单实例环境中，侦听器选择负载最少的调度程序来处理传入的客户端请求。在Oracle Real Application Clusters环境中，连接负载平衡可以平衡多个实例之间的活动连接数。

由于动态服务注册，监听器始终了解所有实例和调度程序，无论其位置如何。根据负载信息，侦听器决定将哪个实例以及（如果配置了共享服务器）发送传入客户端请求的调度程序。

在共享服务器配置中，侦听器按以下顺序选择调度程序：

  1. 最少的节点

  2. 最少的实例

  3. 该实例的最小负载调度程序

在专用服务器配置中，侦听器按以下顺序选择实例：

  1. 最少的节点

  2. 最少的实例

如果数据库服务在多个节点上有多个实例，则侦听器会在最少加载的节点上选择负载最少的实例。
