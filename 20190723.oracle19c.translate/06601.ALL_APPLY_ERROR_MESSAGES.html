2.35 ALL_APPLY_ERROR_MESSAGES

ALL_APPLY_ERROR_MESSAGES显示有关应用进程生成的错误事务中的各个消息的信息，这些应用进程使来自当前用户可访问的队列的消息出列。

对于XStream入站服务器，错误事务中的每条消息都是LCR。

注意：

  • 从内存溢出到硬盘的消息不会出现在此视图中。

  • 此视图不包含与XStream出站服务器相关的信息。

DBA_APPLY_ERROR_MESSAGES显示有关数据库中所有应用进程生成的所有错误事务中的各个消息的信息。

柱 数据类型 空值 描述

MESSAGE_ID

RAW（16）

存储在错误队列中的消息的唯一标识符

LOCAL_TRANSACTION_ID

VARCHAR2（22）

错误事务的本地事务ID

TRANSACTION_MESSAGE_NUMBER

数

引发错误的消息的消息号。消息号是事务中消息的序列号，从1开始。

ERROR_NUMBER

数

事务引发的错误的错误号。仅为引发错误的LCR填充错误编号。对于事务中的其他LCR，此字段为NULL。

错误信息

VARCHAR2（4000）

事务引发的错误的错误消息。仅为引发错误的LCR填充错误消息。对于事务中的其他LCR，此字段为NULL。

SOURCE_OBJECT_OWNER

VARCHAR2（128）

源数据库中对象的所有者

SOURCE_OBJECT_NAME

VARCHAR2（128）

源数据库中对象的名称

OBJECT_OWNER

VARCHAR2（128）

目标表的所有者

OBJECT_NAME

VARCHAR2（128）

目标表的对象名称

首要的关键

VARCHAR2（4000）

表行的主键导致源事务在目标上失败

位置

RAW（64）

LCR的位置

操作

VARCHAR2（100）

LCR中表示的DML或DDL操作

的conflict_type

VARCHAR2（18）

冲突类型。可能的值：

  • INSERT ROW EXISTS ：DML操作是INSERT，并且已存在具有指定键值的行。

  • UPDATE ROW EXISTS ：DML操作是更新。存在具有指定键的行，但某些列的值存在冲突。

  • UPDATE ROW MISSING ：DML操作是UPDATE，不存在具有指定键值的行。

  • DELETE ROW EXISTS ：DML操作是DELETE。存在具有指定键的行，但某些列的值存在冲突

  • DELETE ROW MISSING ：DML操作为DELETE，不存在具有指定键值的行。

APPLIED_STATE

VARCHAR2（7）

冲突适用状态。可能的值：

  • WON ：完整应用了传入的逻辑变更记录。

  • PARTIAL ：传入的逻辑更改记录已应用于一个或多个冲突组。

  • LOST ：未应用传入的逻辑更改记录。

数

RBA

CONFLICT_INFO

VARCHAR2（4000）

标识冲突组信息

SOURCE_PACKAGE_NAME

VARCHAR2（128）

程序复制源的包名称

包裹名字

VARCHAR2（128）

程序复制的目标的程序包名称

信息

CLOB

LCR的内容。内容包括DML LCR中旧值和/或新值的列名和值。对于DDL LCR，内容是DDL SQL的文本。
