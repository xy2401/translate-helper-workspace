130 PRKF-01000至PRKF-01419

PRKF-01000：命令行选项{0}的空值为命令{1}

原因：为此命令行选项指定了空值。

操作：根据特定命令行选项提供值。

PRKF-01001：命令行选项{1}超出了最大字符长度（{0}）

原因：用户输入的字符数多于此命令行选项值的最大长度。

操作：提供不超过最大限制的字符值。

PRKF-01002：输入：{0}

原因：安装选项

行动：没有

PRKF-01003：挂载选项：{0}

原因：安装选项

行动：没有

PRKF-01004：节点：{0}

原因：节点列表

行动：没有

PRKF-01005：服务器池：{0}

原因： Serverpool列表

行动：没有

PRKF-01006：应用程序ID：{0}

原因：应用程序ID

行动：没有

PRKF-01011：文件系统类型“{0}”无效

原因：为文件系统资源指定了无效的文件系统类型。

操作：文件系统类型必须是'ACFS'，'NTFS'，'ZFS'，'JFS'，'EXT3'或'EXT4'

PRKF-01012：“自动启动”选项值“{0}”无效

原因：为命令选项“autostart”指定了无效值。

操作： “自动启动”选项值必须为“始终”，“从不”或“恢复”

PRKF-01013：选项“{0}”和“{1}”发生冲突

原因：虽然相互排斥，但已设置指定的选项。

操作：确保仅设置了其中一个选项。

PRKF-01014：仅允许节点本地文件系统使用应用程序ID。 “{0}”仅在“{1}”或“{2}”存在时有效。

原因：为节点本地文件系统提供了应用程序ID。

操作：确保仅对非节点本地文件系统使用应用程序ID选项。

PRKF-01015：需要为文件系统“{2}”指定选项“{0}”或“{1}”

原因：对于非ACFS文件系统，用户未提供节点列表或服务器池列表。

操作：对于非ACFS文件系统，请确保提供节点列表或服务器池列表。

PRKF-01016：节点列表中的一个或多个节点（“{0}”）为空

原因：节点列表包含空节点名称。

操作：确保没有提供空字符串作为节点列表的一部分。

PRKF-01017：为所有节点添加了为卷设备“{0}”找到的文件系统

原因：具有给定卷设备路径的文件系统已配置为全局文件系统。

操作：更改卷设备路径或不使用'-nodes'和'-serverpools'选项作为命令'srvctl modify filesystem'的一部分。要指定现有全局文件系统的放置位置，请将其删除并重新创建。

PRKF-01018：尝试修改Rapid Home Provisioning Server的端口失败，因为Rapid Home Provisioning Server正在运行。使用-force强制停止和重新启动Rapid Home Provisioning Server。

原因：拒绝修改Rapid Home Provisioning Server端口号的请求，因为此类修改需要停止并重新启动Rapid Home Provisioning Server。

操作：使用'-force'选项强制重新启动Rapid Home Provisioning Server。

PRKF-01019：由于Rapid Home Provisioning Client正在运行，尝试修改Rapid Home Provisioning客户端失败。

原因：拒绝修改Rapid Home Provisioning Client的请求，因为此类修改要求停止并重新启动Rapid Home Provisioning Client。

操作：使用'srvctl stop rhpclient'命令停止Rapid Home Provisioning Client，重新发出'srvctl modify rhpclient'并使用'srvctl start rhpclient'命令重新启动。

PRKF-01020：指定'-clientdata'选项时，不允许其他选项。

原因：在命令'srvctl add gns'期间，指定了' - clientdata'选项以及'-clientdata'不允许的其他选项。

操作：仅指定'-clientdata'选项。

PRKF-01022：“ - {0}”选项需要其中一个选项“ - {1}”，“ - {2}”，“ - {3}”或“ - {4}”。

原因：指定了一个选项，需要同时指定其他可能选项之一。

操作：仅指定一个可能的选项。

PRKF-01023：无法同时指定“ - {0}”和“ - {1}”选项。

原因：指定了无效的选项组合。

操作：仅指定一个指示的选项。

PRKF-01024：指定“-weight”，“ - priority”，“ - port”或“-instance”选项时，不允许使用“ - {0}”选项。

原因：未使用'-weight'，' - priority'，' - port'或'-instance'选项指定'-createsrv'选项。

操作：仅使用'-weight'，' - priority'，' - port'或'-instance'选项指定'-createsrv'，' - target'和'-protocol'选项。

PRKF-01025：指定“ - {0}”选项时，只允许使用“-createsrv”或“-deletesrv”选项。

原因：指定了'-createsrv'或'-deletesrv'以外的选项。

操作：仅使用'-protocol'选项指定'-createsrv'或'-deletesrv'选项。

PRKF-01026：无法使用“-timetolive”选项指定“ - {0}”选项。

原因：使用'-timetolive'选项指定了无效的选项组合。

操作：仅使用'-timetolive'选项指定有效的选项组合。

PRKF-01027：指定“-namettl”选项时，只允许使用“-createtxt”或“-createptr”命令选项。

原因：使用'-namettl'选项指定了'-createtxt'或'-createptr'以外的命令选项。

操作：仅使用'-namettl'选项指定'-createtxt'或'-createptr'命令选项。

PRKF-01028：管理数据库不允许执行此操作

原因：拒绝尝试在管理数据库上执行请求的操作。

操作：在非管理数据库上尝试操作或使用命令'srvctl verb mgmtdb'。

PRKF-01033：为标准ASM指定的无效命令选项{0}

原因：指定了一个只能与Flex ASM一起使用的选项，但当前配置了标准ASM。

操作：重试省略指定的无效选项的命令。

PRKF-01036：如果没有'-srvpool'选项，选项'-eval'不能用于评估管理员管理的数据库的修改

原因：发出了命令'srvctl modify database -db <db_name> -eval'请求而没有'-srvpool'选项。可以为管理员管理的数据库评估的唯一“-eval”选项修改是将该数据库转换为策略管理的数据库，这是使用“-srvpool”选项请求的数据库。

操作：要评估修改对管理员管理的数据库的影响，请提供“-srvpool”选项。

PRKF-01065：如果没有'-node'选项，选项'-eval'不能用于评估数据库的重定位

原因：发出了命令'srvctl relocate database -db <db_name> -eval'请求而没有'-node'选项。

操作：要评估数据库重定位的影响，请指定“-node”选项。

PRKF-01066：无法导出服务器数据，因为GNS未配置域

原因：尝试导出服务器数据失败，因为网格命名服务（GNS）未配置域。

操作：使用命令'srvctl add gns -domain'添加GNS并重试该命令。

PRKF-01067：选项“-delete”只能使用非强制性“-address”选项指定

原因：使用'-delete'选项指定了无效的选项组合。

操作：仅使用“-delete”选项指定“-address”选项，然后重试该命令。

PRKF-01068：命令行选项“{0}”和“{1}”不能一起使用

原因：在srvctl命令上指定了冲突选项。

操作：使用正确的选项重新发出该命令。

PRKF-01070：不允许修改客户端数据操作，因为配置了GNS

原因：尝试修改客户端数据失败，因为配置了GNS。客户端数据只能在GNS客户端群集上进行修改。

操作：删除GNS配置并重试该命令。

PRKF-01071：无法添加Rapid Home Provisioning Server，因为无法找到磁盘组{0}的资源。{1}

原因：找不到指定磁盘组的资源。磁盘组名称拼写错误或磁盘组尚未装入。

操作：更正磁盘组拼写或使用SQL * Plus或ASMCA创建磁盘组。

PRKF-01072：' - file'选项值“{0}”不存在或无法读取

原因： ' - file'选项值指定了不存在或不可读的文件路径名。

操作：确保'-file'选项值对应于现有的可读文件路径名。

PRKF-01073：服务器池列表（“{0}”）中的一个或多个服务器池名称为空

原因：添加文件系统时，指定的服务器池列表包含一个空的服务器池名称。

操作：确保没有提供空字符串作为服务器池列表的一部分。

PRKF-01074：无效的命令行选项。必须使用“-asmlistener”或“-leaflistener”指定“-subnet”选项。

原因：指定了无效的选项组合。

操作：如果使用'-subnet'选项发出命令，请包含'-asmlistener'或'-leaflistener'选项。

PRKF-01075：缺少所需选项'-available'

原因： '-toprefer'选项在命令'srvctl modify service'请求中指定，没有必需的'-available'选项。

操作：确保提供了所需的选项。

PRKF-01076：选项'-eval'不能用于评估添加服务{0}与管理员管理的数据库{1}的影响

原因：在命令'srvctl add service'请求中使用管理员管理的数据库指定了'-eval'选项。

操作：指定策略管理的数据库。

PRKF-01077：无法修改服务器池或节点列表，因为卷设备{0}的文件系统是群集范围的文件系统

原因：尝试修改群集文件系统的服务器池或节点列表属性被拒绝，因为文件系统资源是本地资源。服务器池或节点名称等放置属性仅适用于群集资源。

操作：不要使用命令'srvctl add filesystem {-serverpools <serverpool_list> | -nodes <node_list>}来指定节点名称和服务器池，或删除本地资源并添加群集资源

PRKF-01078：选项'-eval'不能用于评估使用管理员管理的数据库修改服务{0}的效果{1}

原因： '-eval'选项是在命令'srvctl modify service'请求中使用管理员管理的数据库指定的。

操作：指定策略管理的数据库。

PRKF-01079：选项'-eval'不能用于评估使用管理员管理的数据库启动服务{0}的效果{1}

原因：在命令'srvctl start service'请求中使用管理员管理的数据库指定了'-eval'选项。

操作：指定策略管理的数据库。

PRKF-01080：选项'-eval'不能用于评估使用管理员管理的数据库{1}停止服务{0}的效果。

原因：使用管理员管理的数据库在命令'srvctl stop service'请求中指定了'-eval'选项。

操作：指定策略管理的数据库。

PRKF-01083：端口“{0}”已配置为由ONS守护程序用作本地，远程或企业管理器

原因： ONS守护程序已在使用指定的端口。

PRKF-01084：端口“{0}”用于多个ONS端口

原因：指定的端口是作为两个或多个ONS端口提供的。

PRKF-01085：对象“{1}”不支持命令“{0}”

原因：指定了不受支持的命令和对象组合。

操作：使用其中一个支持的用法重新发出命令。

PRKF-01086：命令选项{0}和{1}不能一起使用

原因：提供了冲突的命令行选项。

操作：检查输入的命令行选项，并确保指定了一组中的所有必需选项。

PRKF-01087：指定{1}时缺少必需的选项{0}

原因：缺少指定的命令选项。

操作：使用命令'srvctl'和'-help'选项显示命令的选项详细信息，并确保指定了所有必需的选项。

PRKF-01088：无法使用PQ服务器池{1}添加PQ服务{0}。管理员管理的数据库{2}不支持PQ服务

原因：尝试将PQ服务添加到指定的数据库被拒绝，因为只有策略管理的数据库支持PQ服务。

PRKF-01089：指定的命令选项{0}只能在Flex Cluster中使用

PRKF-01090：现有服务{0}无法注册为主服务{1}的并行查询帮助程序

原因：尝试注册并行查询服务指定了已存在的服务。

PRKF-01091：无法启动未配置为服务的服务器池{2}上的数据库{1}的服务{0}

原因：未将服务配置为在指定的服务器池上运行。

操作：在已配置的服务器池上启动服务，或修改服务以在指定的服务器池上运行。

PRKF-01092：当需要主服务名称{1}时，指定了并行查询服务名称{0}

原因：尝试通过其并行查询帮助程序服务名称而不是主服务名称对数据库服务进行操作。

操作：指定主服务名称。

PRKF-01093：无法停止未配置为服务的服务器池{2}上的数据库{1}的服务{0}

原因：服务未在指定的服务器池上运行。

操作：停止运行它的服务器池上的服务。

PRKF-01094：无法设置并行查询池{0}，因为并行查询帮助程序服务不存在

原因：尝试为没有配置并行查询帮助程序服务的服务设置并行查询池，或者通过'-pqservice'参数删除其配置的并行查询帮助程序服务。

操作：发出命令“srvctl modify service -db <database_name> -service <primary_service_name> -pqservice <pq_service_name> -pqpool <pq_pool_name>”以为指定的并行查询服务器池配置并行查询服务。

PRKF-01098：无需修改

PRKF-01110：此群集上未配置GNS服务器和GNS客户端

PRKF-01116：此命令无法在GNS客户端群集上执行。
