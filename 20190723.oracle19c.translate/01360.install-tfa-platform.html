E.1安装Oracle跟踪文件分析器

了解可以提供给Oracle Trace File Analyzer安装程序脚本以自定义安装的选项。

如果没有参数，Oracle跟踪文件分析器安装脚本将引导您完成安装的面试过程。

Oracle跟踪文件分析器安装脚本将/tfa附加到-tfabase如果尚未存在）。

Oracle Trace File Analyzer安装脚本将日志写入/tmp/tfa_install_ timestamp .log文件，例如/tmp/tfa_install_9263_2018_09_25-07_55_52.log 。

句法

installTFA- platform [[-local] [ -  deferdiscovery] [ -  tfabase install dir ] [ -  javahome path to JRE ] [ -  silent] | -extractto extract dir | -responsefile response file | -noorachk]

参数

表E-1 installTFA- <platform>命令参数

参数 描述

-本地

仅在本地节点上安装。

-deferdiscovery

安装完成后发现Oracle跟踪目录。

-tfabase

安装在指定的目录中。

-javahome

将此目录用于JRE。不要将此选项与-extractto选项一起使用。

-无声

使用此选项不要询问任何安装问题。

-解压到

将Oracle跟踪文件分析器提取到指定的目录（非守护程序模式）。

-tmploc

将安装存档提取到指定的临时目录位置（必须存在）。

-debug

显示调试跟踪，并且在安装失败时不会删除TFA_HOME 。

-perlhome

指定Perl二进制文件的自定义位置。

-responsefile

指定用于驱动Oracle跟踪文件分析器设置参数的响应文件。

-noorachk

指定跳过Oracle ORAchk自动启动。
