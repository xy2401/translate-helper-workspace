13.20 DEFAULT COLLATION条款

排序（也称为排序排序）确定当比较和排序两个字符串时，字符串是否等于，先于或跟随另一个字符串。Oracle数据库归类按照不同语言中使用的排序文本规则排序字符串。

DEFAULT COLLATION子句可以出现在以下SQL语句中：

  • 创建功能声明

  • CREATE PROCEDURE声明

  • CREATE PACKAGE声明

  • CREATE TRIGGER声明

  • CREATE TYPE语句

先决条件

必须将COMPATIBLE初始化参数设置为至少12.2.0，并且必须将MAX_STRING_SIZE设置为EXTENDED才能在这些SQL语句中允许排序规则声明。

句法

default_collation_clause :: =


说明default_collation_clause.eps的说明

collation_option :: =


伪示例说明pseudo_collation.eps

语义

default_collation_clause

default_collation_clause可以出现在包规范，独立类型规范和独立子程序中。

collation_option

过程，函数，包，类型或触发器的默认排序规则必须为USING_NLS_COMP 。 default_collation_clause显式声明PL / SQL单元的默认排序规则为USING_NLS_COMP 。如果没有此子句，该单元将从有效的架构默认排序规则继承其默认排序规则。如果有效架构默认排序规则不是USING_NLS_COMP ，则单位无效。

有效的架构默认排序规则确定如下：

  • 如果设置了会话参数DEFAULT_COLLATION ，则有效架构默认排序规则是此参数的值。可以通过查询SYS_CONTEXT('USERENV', 'SESSION_DEFAULT_COLLATION')来检查参数的值。如果未设置DEFAULT_COLLATION则该函数返回NULL 。可以使用以下语句设置参数DEFAULT_COLLATION的值： ALTER SESSION SET DEFAULT_COLLATION = collation_option ;

  • 如果未设置会话参数DEFAULT_COLLATION ，则有效架构默认排序规则是您在其中创建PL / SQL单元的架构的声明的默认排序规则。可以在静态数据字典* _USERS视图中找到模式的默认排序规则。它可以使用DDL语句CREATE USER和ALTER USER 。

可以使用以下语句取消设置会话参数DEFAULT_COLLATION ： ALTER SESSION SET DEFAULT_COLLATION = NONE;

包体和类型体使用相应规范的默认排序规则。过程，函数和方法中的所有字符数据容器和属性（包括参数和返回值）的行为就像它们的数据绑定排序规则是伪排序USING_NLS_COMP 。

对DEFAULT COLLATION限制

无法为嵌套或打包的子程序或类型方法指定它。

编译语义

如果生成的默认对象排序规则与USING_NLS_COMP不同，则数据库对象将因编译错误而创建为无效。

如果为具有REUSE SETTINGS子句的PL / SQL单元发出ALTER COMPILE语句，则不会更改正在编译的数据库对象的存储默认排序规则。

如果在没有REUSE SETTINGS子句的情况下发出ALTER COMPILE语句，则会丢弃正在编译的数据库对象的存储的默认排序规则，并且在执行语句时对象所有者的有效模式默认排序规则将存储为默认排序规则除非PL / SQL单元包含DEFAULT COLLATION子句，否则为该对象。如果生成的默认排序规则不是USING_NLS_COMP ，则会引发编译错误。

包或类型主体的ALTER COMPILE语句引用相应规范的存储排序规则。
