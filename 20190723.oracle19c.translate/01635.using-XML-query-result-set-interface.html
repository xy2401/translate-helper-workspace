15.1 XML查询结果集接口概述

XML查询结果集接口（RSI）使您能够以XML格式执行查询并将结果作为XML返回，从而避免SQL层和要求在SELECT语义中工作。RSI使用简单的Oracle Text查询和XML结果集描述符，其中命中列表根据结果集描述符以XML格式返回。XML查询RSI使用SDATA部分进行分组和计数。

在应用程序中，搜索结果页面可以包含许多不同的元素，例如前几个文档的元数据，总命中数和每个单词命中数。每次额外调用都需要时间来重新分析查询并查找索引元数据。此外，一些搜索操作（例如迭代查询细化）对于SQL来说很难。如果甚至可以构造一个SQL语句来产生所需的结果，那么这种SQL通常是次优的。

XML查询RSI能够同时生成搜索结果页面所需的各种数据，从而通过共享开销来提高性能。RSI还可以返回难以在SQL中表达的数据视图。

15.2使用XML查询结果集接口

CTX_QUERY.RESULT_SET() 和CTX_QUERY.RESULT_SET_CLOB_QUERY() API使您可以使用单个查询获取查询结果，而不是运行多个CONTAINS()查询以获得相同的结果。这两个API是相同的，除了一个使用VARCHAR2查询参数，另一个使用CLOB查询参数以允许更长的查询。

例如，要显示搜索结果页面，您必须首先获取以下信息：

  • 按日期和相关性排序的前20个热门列表

  • 给定Oracle Text查询的总命中数

  • 按出版日期计算小组

  • 按作者计算

假设以下表定义用于存储要搜索的文档：

create table docs（docid number，author varchar2（30），pubdate date，title varchar2（60），doc clob）;

假设以下Oracle Text Index定义：

在docs上创建索引docidx（doc）indextype是ctxsys.context按作者，pubdate过滤，标题顺序由pubdate过滤;

使用这些定义，您可以发出四个SQL语句来获取显示搜索结果页面所需的四条信息：

15.3使用Oracle Text创建仅XML应用程序

虽然通常使用带有CONTAINS子句的SQL SELECT语句来创建应用程序，但它并不总是最有效的方法。另一种方法是使用基于XML的RSI。优点是在不获取查询的所有结果的情况下很容易获得摘要信息（例如命中总数）。

要使用RSI，请指定结果集描述符（RSD）。RSD声明要返回的信息，其中包括：

  • 查询的总结果计数

  • 一个命中列表

  • SDATA字段的摘要信息

反过来，hitlist包含重复元素，每个元素可能包含：

  • 击中的rowid

  • 来自点击的SDATA字段

15.4结果集描述符的示例

此示例显示如何使用RSD。以下示例请求具有前10个命中（按分数排序）的命中列表以及结果总数的计数。

<ctx_result_set_descriptor> <hitlist start_hit_num =“1”end_hit_num =“10”order =“SCORE DESC”> <rowid /> <sdata name =“title”/> <sdata name =“author”/> <sdata name =“articledate “/> <snippet radius =”20“max_length =”160“starttag =”＆lt; b＆gt;“ ENDTAG = “LT; / B＆gt;” 中/> </ hitlist> <count /> </ ctx_result_set_descriptor>

对于每个匹配，您正在请求rowid （您可以使用它来获取有关该行的更多信息，如果需要）， SDATA字段的内容或title, author,和articledate列以及一个片段（这是一个简短的内容）突出显示关键字的摘要，在本例中为<b> ... </b> ）。

15.5识别配置

Collocates是一组经常在文档中共同出现的单词。它们提供了与指定关键字相关的其他关键字或概念的快速摘要。然后，您可以在查询中使用其他关键字来获取更相关的结果。

您可以根据搜索查询识别并置项。对于查询返回的每个文档，将自动提取搜索关键字周围的文本片段。接下来，通过使用统计测量将这些片段中的单词与查询关键字相关联，并且根据提取的单词在整个文档集中出现的频率，将分数分配给每个返回的共现单词。

使用RSI识别并置。您可以指定查询必须返回的共现单词数。您还可以指定是否标识强调唯一性的常用名词或并置的并置项。也可以返回指定搜索关键字的同义词。

注意：

仅支持BASIC_LEXER.

识别配置：

  1. 为查询创建文档集表。
  2. 在文档集表上创建Oracle Text索引。
  3. 使用XML Query RSI定义和输入标识collocates的查询。包含具有必需属性的collocates元素。

示例15-1识别文档集中的并置项

在此示例中，用于查询数据集中的文档的关键字是“Nobel”。Oracle Text在文档集中搜索此关键字的出现位置。除了结果集之外，使用collocates搜索与'Nobel'共同出现的五个常用词。使用max_words属性标识要生成的并置数。将use_tscore属性设置为TRUE以指定必须为collocates标识公共单词。为了识别并置关键字，要在关键字两侧选择的单词数为10。

以下是用于确定并置的输入RSI描述符：

declare rsd varchar2（32767）; begin ctx_query.result_set（'tdrbnbsan01idx'，'nobel'，<ctx_result_set_descriptor> <collocates radius =“10”max_words =“5”use_tscore =“TRUE”/> </ ctx_result_set_descriptor>'，：rs）;结束; /

以下是查询的输出结果集：

<ctx_result_set> <collocates> <collocation> <word> PRIZE </ word> <score> 82 </ score> </ collocation> <collocation> <word> LAUREATE </ word> <score> 70 </ score> </ collocation> <collocation> <word> NOBELPRIZE </ word> <score> 44 </ score> </ collocation> <collocation> <word> AWARD </ word> <score> 42 </ score> </ collocation> <搭配> <word> ORG </ word> <score> 41 </ score> </ collocation </ collocates> </ ctx_result_set>

对于“诺贝尔奖”，按顺序排名前五位的共同配置者是奖品，桂冠，Nobelprize，奖项和组织。每个单词都分配一个分数，表示发生的频率。在返回任何hitlist元素后，始终返回并置对象。

如果在同一示例中将use_tscore设置为FALSE ，则会识别不常见（唯一）的单词。这是输出结果集：

<ctx_result_set> <collocates> <collocation> <word> MOLA </ word> <score> 110 </ score> </ collocation> <collocation> <word> BISMARCK </ word> <score> 89 </ score> </ collocation> <collocation> <word> COLONNA </ word> <score> 67 </ score> </ collocation> <collocation> <word> LYNEN </ word> <score> 55 </ score> </ collocation> <搭配> <word> TIMBERGEN </ word> <score> 25 </ score> </ collocation> </ collocates> </ ctx_result_set>
