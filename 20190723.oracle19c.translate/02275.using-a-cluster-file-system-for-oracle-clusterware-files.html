使用Oracle Clusterware文件的集群文件系统

从Oracle Grid Infrastructure 19c开始，您可以使用Oracle自动存储管理（Oracle ASM）或经过认证的共享文件系统来存储OCR文件和投票文件。

对于新的Oracle Standalone Cluster安装，您可以使用Oracle ASM或共享文件系统来存储投票文件和OCR文件。对于其他群集类型，必须使用Oracle自动存储管理（Oracle ASM）来存储投票文件和OCR文件。对于Linux 86-64（64位）和Linux Itanium平台，Oracle提供了一个集群文件系统OCFS2。但是，Oracle不建议将OCFS2用于Oracle Clusterware文件。
