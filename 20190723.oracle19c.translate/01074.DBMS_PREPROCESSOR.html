124 DBMS_PREPROCESSOR

DBMS_PREPROCESSOR包提供了一个接口，用于在其后处理的表单中打印或检索PL / SQL单元的源文本。

该软件包包含以下主题：

  • 概观

  • 操作说明

  • 数据结构

  • DBMS_PREPROCESSOR子程序的摘要

124.1 DBMS_PREPROCESSOR概述

首先要了解子程序的三种样式，以了解DBMS_PREPROCESSOR工作原理。

以下是三种子程序样式：

  1. 采用模式名称，单元类型名称和单元名称的子程序。

  2. 采用VARCHAR2字符串的子程序，其中包含任意PL / SQL编译单元的源文本。

  3. 采用VARCHAR2索引表的子程序，该表包含任意PL / SQL编译单元的分段源文本。

第一种样式的子程序用于打印或检索存储的PL / SQL单元的后处理源文本。用户必须具有查看本机原始源文本所需的权限。用户还必须指定定义单元的模式，单元类型和单元名称。如果架构为null，则使用当前用户架构。如果存储单元的状态为VALID且用户具有所需的权限，则保证后处理的源文本与上次编译时的单元的文本相同。

第二或第三种样式的子程序用于在当前用户模式中生成后处理的源文本。源文本作为第二个样式中的单个VARCHAR2字符串传递，或者作为第三个样式中的VARCHAR2索引表传递。源文本可以表示任意PL / SQL编译单元。典型用法是传递匿名块的源文本，并在当前用户模式中生成其后处理的源文本。当源文本超出VARCHAR2长度限制时，第三种样式可能很有用。

124.2 DBMS_PREPROCESSOR操作说明

这些注释解释了DBMS_PREPROCESSOR如何使用三种子程序样式。

  • 对于第一种样式的子程序，存储的PL / SQL单元的状态不需要是VALID 。同样，作为VARCHAR2字符串或VARCHAR2索引表传入的源文本可能包含编译时错误。如果在生成后处理源时发现错误，则错误消息文本也将出现在经过后处理的源文本的末尾。在某些情况下，可以因错误而中止预处理。发生这种情况时，后处理的源文本将显示为不完整，并且相关的错误消息可以帮助指示在预处理期间发生了错误。

  • 对于第二或第三种样式的子程序，源文本可以表示任意PL / SQL编译单元。但是，有效PL / SQL编译单元的源文本不能包含常用的前缀，例如CREATE OR REPLACE 。通常，输入源应该以一种从ALL_SOURCE视图获得的方式进行语法准备。以下列表给出了一些PL / SQL编译单元的有效初始语法示例。 

    匿名块（BEGIN | DECLARE）......包PACKAGE <名称> ...包体PACKAGE BODY <名称> ...程序程序<名称> ...功能功能<名称> ...输入TYPE <name> ...type body TYPE BODY <name> ...触发器（BEGIN | DECLARE）......

    如果源文本表示有效的命名PL / SQL单元，则在生成后处理源文本后将不会创建该单元。

  • 如果从ALL_SOURCE视图获取包装的PL / SQL单元的文本，则关键字WRAPPED始终紧跟单元的名称，如下例所示：

    程序“一些程序”包装了a000000 b2 ......

    如果这样的源文本被提交给一个GET_POST_PROCESSED_SOURCE功能或到一个PRINT_POST_PROCESSED_SOURCE程序 ，例外DBMS_PREPROCESSOR 。 WRAPPED_INPUT被提出。

124.3 DBMS_PREPROCESSOR数据结构

DBMS_PREPROCESSOR包定义TABLE类型。

表类型

SOURCE_LINES_T表类型

124.3.1 DBMS_PREPROCESSOR SOURCE_LINES_T表类型

此表类型存储经过后处理的源文本行。它用于在处理之前和之后保存PL / SQL源文本。它在文本量超过32K的情况下特别有用。

句法

TYPE source_lines_t IS TABLE OF VARCHAR2（32767）INDEX BY BINARY_INTEGER;

124.4.1 GET_POST_PROCESSED_SOURCE函数

此重载函数返回经过后处理的源文本。每种语法形式的不同功能与定义一起呈现。

句法

返回存储的PL / SQL单元的后处理源文本：

DBMS_PREPROCESSOR.GET_POST_PROCESSED_SOURCE（object_type IN VARCHAR2，schema_name IN VARCHAR2，object_name IN VARCHAR2）RETURN source_lines_t;

返回编译单元的后处理源文本：

DBMS_PREPROCESSOR.GET_POST_PROCESSED_SOURCE（source IN VARCHAR2）RETURN source_lines_t;

返回包含编译单元源文本的INDEX-BY表的后处理源文本：

DBMS_PREPROCESSOR.GET_POST_PROCESSED_SOURCE（source IN source_lines_t）RETURN source_lines_t;

参数

表124-2 GET_POST_PROCESSED_SOURCE函数参数

参数 描述

OBJECT_TYPE

必须是PACKAGE ， PACKAGE BODY ， PROCEDURE ， FUNCTION ， TYPE ， TYPE ， BODY或TRIGGER 。区分大小写。

SCHEMA_NAME

架构名称。除非使用带引号的标识符，否则不区分大小写。如果为NULL ，则使用当前架构。

OBJECT_NAME

对象的名称。object_type始终不区分大小写。除非使用带引号的标识符，否则不区分大小写。

资源

编译单元的源文本

source_lines_t

INDEX-BY表，包含编译单元的源文本。源文本是按升序索引顺序的所有非NULL INDEX-BY表元素的串联。

返回值

该函数返回一个INDEX-BY表，其中包含从索引1开始的经过后处理的源文本的行。

使用说明

  • 不删除换行符。

  • 后处理源文本中的每一行都映射到INDEX-BY表中的一行。

  • 在后处理源中，未选中的文本将具有空行。

124.4.2 PRINT_POST_PROCESSED_SOURCE过程

此重载过程调用DBMS_OUTPUT 。 PUT_LINE让您查看经过后期处理的源文本。每种语法形式的不同功能与定义一起呈现。

句法

打印存储的PL / SQL单元的后处理源文本：

DBMS_PREPROCESSOR.PRINT_POST_PROCESSED_SOURCE（object_type IN VARCHAR2，schema_name IN VARCHAR2，object_name IN VARCHAR2）;

打印编译单元的后处理源文本：

DBMS_PREPROCESSOR.PRINT_POST_PROCESSED_SOURCE（source IN VARCHAR2）;

打印包含编译单元源文本的INDEX-BY表的后处理源文本：

DBMS_PREPROCESSOR.PRINT_POST_PROCESSED_SOURCE（source IN source_lines_t）;
