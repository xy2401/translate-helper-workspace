Oracle Database Release 19c，版本19.1中的更改

以下是Oracle Database PLc 19c版本19.1的Oracle Database PL / SQL包和类型参考中的更改。

  • 新功能

  • 不推荐的功能

  • 不受支持的功能

有关此版本中新功能的完整列表，请参阅“ Oracle数据库新增功能指南” 。

新功能

本节列出了Oracle Database 19c版19.1 PL / SQL包和类型参考中的主要新功能。

以下功能是此版本中的新增功能：

  • DBMS_ADDM

    可插拔数据库（PDB）的自动数据库诊断监视器（ADDM）支持。您现在可以使用ADDM分析PDB中的AWR数据，以识别和解决与性能相关的问题。将以下子程序中的两个新参数CDB_TYPE_OVERRIDE和READ_ONLY_TYPE_OVERRIDE添加到DBMS_ADDM程序包：
    • ANALYZE_DB程序
    • ANALYZE_INST过程
    • ANALYZE_PARTIAL过程
    • COMPARE_CAPTURE_REPL AY_REPORT功能
    • COMPARE_DATABASES功能
    • COMPARE_INSTANCES功能
    • COMPARE_REPLAY_REPLA Y_REPORT功能
    • REAL_TIME_ADDM_REPORT功能

  • DBMS_AUTO_INDEX

    新包DBMS_AUTO_INDEX提供了一个界面，用于配置自动索引并生成包含数据库中已执行的自动索引操作的详细信息的报告。

    请参阅“ DBMS_AUTO_INDEX ”

  • DBMS_CREDENTIAL

    您还可以使用DBMS_CREDENTIAL.CREATE_CREDENTIAL过程来管理云对象存储的凭据。impdb的credential参数指定凭证对象的名称，该凭据对象包含访问对象存储桶所需的用户名和密码。您还可以使用数据库属性DEFAULT_CREDENTIAL指定默认凭据。

  • DBMS_HADOOP

    Oracle数据库现在支持混合分区表。混合分区表是一个分区表，其中一些分区驻留在数据库中，一些分区驻留在数据库外部的外部文件中，例如操作系统文件或Hadoop分布式文件系统（HDFS）文件。

    新的过程CREATE_HYBRID_PARTNED_TABLE被添加到DBMS_HADOOP包中。

  • DBMS_INMEMORY_ADMIN

    新功能POPULATE_WAIT被添加到DBMS_INMEMORY_ADMIN包中。此函数确保在允许应用程序访问之前已填充指定的In-Memory对象。

  • DBMS_MEMOPTIMIZE

    以下新子程序作为Memoptimized Rowstore快速摄取功能的一部分添加到DBMS_MEMOPTIMIZE包中：

    • GET_APPLY_HWM_SEQID

    • GET_WRITE_HWM_SEQID

    • WRITE_END

    请参阅“ DBMS_MEMOPTIMIZE ”

  • DBMS_MEMOPTIMIZE_ADMIN

    新包DBMS_MEMOPTIMIZE_ADMIN提供了一个接口，用于将大型池中缓冲的Memoptimized Rowstore数据刷新到磁盘。

    请参见“ DBMS_MEMOPTIMIZE_ADMIN ”

  • DBMS_METADATA

    以下由datapump的元数据层使用的新转换参数将添加到DBMS_METADATA.SET_TRANSFORM_PARAM过程：
    • OMIT_ENCRYPTION_CLAUSE
    • DWCS_CVT_IOTS
    • DWCS_CVT_CONSTRAINTS
    • CONSTRAINT_USE_DEFAULT_INDEXES
    • CONSTRAINT_NAME_FROM_INDEXES

  • DBMS_MVIEW

    Oracle数据库现在支持混合分区表。混合分区表是一个分区表，其中一些分区驻留在数据库中，一些分区驻留在数据库外部的外部文件中，例如操作系统文件或Hadoop分布式文件系统（HDFS）文件。

    新参数SKIP_EXT_DATA将添加到DBMS_MVIEW.REFRESH过程中。

  • DBMS_SQLQ

    新包DBMS_SQLQ提供了一个用于配置SQL语句及其执行计划的隔离阈值的接口。跨越隔离阈值的SQL语句将被隔离，并且不允许在数据库中再次运行。

    请参阅“ DBMS_SQLQ ”

  • DBMS_SQLDIAG新SQL修复顾问函数DBMS_SQLDIAG.SQL_DIAGNOSE_AND_REPAIR

    新的SQL Repair Advisor函数DBMS_SQLDIAG.SQL_DIAGNOSE_AND_REPAIR创建诊断任务，执行它，并接受SQL补丁建议，该SQL补丁建议生成严重错误。此功能可帮助用户诊断和修复问题。

  • DBMS_SODA

    DBMS_SODA是一个实现简单Oracle文档访问（SODA）的PL / SQL包，它允许您将Oracle数据库用作NoSQL文档存储。此版本中添加了其他功能和特性。

  • DBMS_STATS

    新的布尔参数REALTIME_STATS被添加到GET_TABLE_STATS和GET_COLUMN_STATS过程。

  • DBMS_XPLAN

    新的COMPARE_PLANS函数已添加到DBMS_XPLAN包中。此功能将列表中的每个计划与参考计划进行比较并返回报告。

  • 基于密钥的消息传递

    以下新参数添加到GET_QUEUE_PARAMETER ， SET_QUEUE_PARAMETER和UNSET_QUEUE_PARAMETER的程序DBMS_AQADM作为基于密钥的消息功能的一部分包：

    • 分片队列参数
      • SHARD_NUM
      • KEY_BASED_ENQUEUE
      • STICKY_DEQUEUE
    • 基于密钥的参数
      • AQ $ KEY_TO_SHARD_MAP
      • AQ $ GET_KEY_SHARD_INST

    请参阅“ DBMS_AQADM ”

不推荐的功能

Oracle Database 19c版19.1 PL / SQL包和类型参考中不推荐使用的功能列表。

不推荐使用DBMS_SESSION.IS_ROLE_ENABLED函数。

请改用DBMS_SESSION.CURRENT_IS_ROLE_ENABLED或DBMS_SESSION.SESSION_IS_ROLE_ENABLED函数。

不受支持的功能

本节列出了Oracle Database 19c版19.1 PL / SQL包和类型参考中不支持的功能。

此版本不支持以下功能：

  • 支持CONTINUOUS_MINE选项

    Oracle Database 19c中不支持DBMS_LOGMNR.START_LOGMNR DBMS_LOGMNR.START_LOGMNR包的continuous_mine选项，并且该选项不再可用。LogMiner包的CONTINUOUS_MINE功能已过时。它在Oracle Database 12c第2版（12.2）中已弃用。没有替换功能。

    不支持Oracle Streams

    从Oracle Database 19c开始，不支持Oracle Streams功能。使用Oracle GoldenGate替换Oracle Streams的所有复制功能。

  • Oracle多媒体

    从Oracle Database 19c开始，不支持Oracle多媒体功能。将Oracle SecureFiles与开源或第三方产品一起用于元数据提取以及图像处理和转换，而不是Oracle多媒体。

    也可以看看：

    有关更多信息，请参见Oracle数据库升级指南

  • 不支持LOG_ARCHIVE_DEST_n的MAX_CONNECTIONS属性

    Oracle Data Guard重做传输的LOG_ARCHIVE_DEST_n参数的MAX_CONNECTIONS属性已过时。Oracle Database 19c中不支持它。

    MAX_CONNECTIONS属性可能会干扰Oracle Database 11g中引入的新重做传输流机制，并增加了解决间隙所需的时间。为防止出现这些类型的错误，Oracle已取消支持并删除了此属性。

  • 支持扩展数据类型支持（EDS）

    Oracle Database 19c中不支持扩展数据类型支持（EDS）功能。现在，Logical Standby和Oracle GoldenGate都支持EDS功能支持的所有数据类型。

    扩展数据类型支持（EDS）功能为逻辑备用数据库提供了一种机制，以支持缺少基于本机重做的支持的某些Oracle数据类型。例如，EDS用于复制具有SDO_GEOMETRY列的表。但是，从Oracle Database 12c第2版（12.2）开始，没有任何EDS支持的Oracle数据类型本身不受逻辑备用数据库或Oracle GoldenGate支持。Oracle Database 19c（19.1）不支持此功能。
