36.1导出和导入XMLType表概述

Oracle XML DB支持导出和导入存储XML数据的XMLType表和列，无论它是否基于XML模式。

Oracle Data Pump支持从一个数据库到另一个数据库的高速数据和元数据移动。使用Oracle Data Pump有两种模式：可传输表空间模式和非可传输表空间模式。

对于可传输表空间模式，存在关于XMLType数据的此限制：您无法更改XMLType存储模型。

与其他数据库对象一样，XML数据将导出到导出服务器的字符集中。导入期间，数据将转换为导入服务器的字符集。

Oracle Data Pump有两个命令行客户端， expdp和impdp ，分别调用Data Pump Export实用程序和Data Pump Import实用程序。expdp和impdp客户端使用PL / SQL包DBMS_DATAPUMP提供的过程来执行导出和导入命令，并传递在命令行输入的参数。这些参数支持导出和导入完整数据库或数据库子集的数据和元数据。

Data Pump导出和导入实用程序（分别使用命令expdp和impdp调用）与原始Export（ exp ）和Import（ imp ）实用程序具有相似的外观，但它们是完全独立的。

Data Pump Export实用程序（使用expdp)调用expdp)数据和元数据卸载到一组称为转储文件集的操作系统文件中 。转储文件集只能由Data Pump Import实用程序导入（使用impdp调用）。

Oracle XML DB支持导出和导入存储XML数据的XMLType表和列，无论它是否基于XML模式。如果表是基于XML模式的，那么它取决于用于定义其数据的XML模式。在对象关系存储的情况下，此XML模式还可以依赖于用于存储数据的SQL对象类型。

因此，导出具有基于XML模式的XMLType表的用户也会导出以下内容：

  • SQL对象类型（如果使用了对象关系存储）

  • XML模式

  • XML表

无论XMLType存储格式（对象关系或二进制XML）如何，您都可以导出和导入此数据。但是，Oracle Data Pump仅将XML数据导出和导入为文本或二进制XML数据。因此，不会导出用于XMLType对象关系存储的基础表和列。相反，它们被转换为二进制形式，然后作为自描述二进制XML数据导出。

注意：

Oracle Data Pump for Oracle Database 11g第1版（11.1）不支持将XML模式，基于XML模式的XMLType列或二进制XML数据导出到11.1之前的数据库版本。

无论XMLType存储模型如何，转储文件的格式都是文本或自描述的带有令牌映射前导码的二进制XML。默认情况下，使用自描述二进制XML。

由于XMLType数据作为XML数据导出和导入，因此源数据库和目标数据库可以为该数据使用不同的XMLType存储模型。您可以从存储XMLType数据的数据库中导出数据，并将其导入到以不同方式存储XMLType数据的数据库中。

注意：

无论使用哪种XMLType存储模型，都不要使用选项table_exists_action=append从同一个转储文件中多次导入XMLType表。这样做会引发唯一约束违规错误，因为始终使用唯一对象标识符导出和导入XMLType表中的行。

有关table_exists_action信息，请参见Oracle数据库实用程序 。

36.2 Oracle XML DB存储库的导出/导入限制

导出或导入存储基于已注册XML模式的Oracle XML DB存储库资源数据的表时，仅导出该XML数据。导出期间存储库结构将丢失，因此在导入这些表时，它们不会启用层次结构。

您可以导出和导入XMLType表，这些表存储基于已注册XML模式的Oracle XML DB存储库资源的XML数据。

但是，仅导出XML数据。导出期间存储库结构丢失。不为已启用层次结构的表导出文件夹层次结构，行级安全性（RLS）策略和路径索引触发器中的关系。导入这些表时，它们不支持层次结构。

36.3.1执行表模式导出/导入

提供了分别对转储文件执行表模式导出和表模式导入的示例。

XMLType表依赖于用于定义它的XML模式。同样，该XML模式依赖于为其创建或指定的SQL对象类型。导入XMLType表需要存在相应的XML模式和SQL对象类型。

使用TABLE模式导出时，仅导出与表相关的元数据和数据。为了能够成功导入此数据，您必须确保已创建相关的XML架构和对象类型。

此处的示例假定您使用的数据库具有以下功能：

  • 具有模式user23的数据库

  • 表user23.tab41 ，其中XMLType列存储为二进制XML

  • 目录对象dpump_dir ，已为运行expdp或impdp的用户授予了READ和WRITE权限

例36-1显示了使用TABLES参数指定的表模式导出。它将表tab41到转储文件tab41.dmp 。

注意：

在表模式下，如果未在expdp命令中指定模式前缀，则默认情况下将使用导出器的模式。

例36-2显示了表模式导入。它进口表tab41从转储文件tab41.dmp 。

如果tab41已存在名为tab41的表，则指定table_exists_action = append会导致将行附加到该表。无论何时使用参数值append ，数据都会被加载到新的空间中;现有空间永远不会被重用。因此，您可能需要在加载操作后压缩数据。

36.3.2执行模式模式导出/导入

此处的示例执行模式模式导出和导入。执行Schema模式导出时，如果您具有角色EXP_FULL_DATABASE ，则可以导出数据库模式，数据库模式定义以及该数据库模式的系统授权和特权。

此处的示例假定您使用的数据库具有以下功能：

  • 用户x4a创建了一个表po2 。

  • 用户x4a具有注册的XML模式ipo ，其创建了两个有序集合表item_oct2和sitem_nt2 。

用户x4a创建表po2 ，如例36-3所示。

然后填充并导出表po2 ，如例36-4所示 。

例36-4导出以下所有内容：

  • 在注册XML模式ipo期间生成的所有数据类型。

  • XML架构ipo 。

  • 表po2和有序集合表item_oct2和sitem_nt2 ，它们是在XML模式ipo注册期间生成的。

  • 所有这些表中的所有数据。

例36-5将x4a.dmp中的所有数据x4a.dmp另一个数据库，其中用户x4a已经存在。

示例 36-6与示例36-5的作用相同，但它也将数据库架构从用户x4a重新映射到用户quine 。

实施例36-6进口中的所有数据的x4a.dmp （从用户的数据库模式导出x4a ）到数据库模式quine 。要重新映射数据库架构，用户x4a必须已被授予角色IMP_FULL_DATABASE在本地数据库和作用EXP_FULL_DATABASE源数据库上。REMAP_SCHEMA将源模式中的所有对象加载到目标模式中。

注意：

如果将XML模式导入到从中导出的同一数据库中，并且在导入时该XML模式仍在Oracle XML DB中注册，请不要使用remap_schema除非您还指定了impdp参数transform=oid:n 。有关参数transform信息，请参见Oracle数据库实用程序 。
