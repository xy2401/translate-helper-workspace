13.6块

该块对相关的声明和语句进行分组，是PL / SQL源程序的基本单元。

它有一个可选的声明部分，一个必需的可执行部分和一个可选的异常处理部分。声明是块的本地声明，并在块完成执行时停止存在。块可以嵌套。

匿名块是可执行语句。

语义

plsql_block

标签

未声明的标识符，对于块是唯一的。

宣布

启动块的声明部分。

declare_section

包含局部声明，这些声明仅存在于块及其子块中，并且对于封闭块不可见。

declare_section的限制

  • create_package ， create_package_body或compound_trigger_block declare_section不能包含PRAGMA AUTONOMOUS_TRANSACTION 。

  • 甲declare_section在trigger_body或tps_body不能声明的数据类型的变量LONG或LONG RAW 。

也可以看看：

  • 有关create_package更多信息，请参阅“ CREATE PACKAGE语句 ”

  • 有关create_package_body更多信息，请参阅“ CREATE PACKAGE BODY语句 ”

  • 有关compound_trigger_block ， trigger_body和tps_body更多信息，请tps_body “ CREATE TRIGGER语句 ”

subtype_definition

静态表达式可用于子类型声明。有关更多信息，请参见静态表达式 。

亚型

您要定义的用户定义子类型的名称。

base_type

您要定义的子类型的基本类型。 base_type可以是任何标量或用户定义的PL / SQL数据类型说明符，例如CHAR ， DATE或RECORD 。

字符集character_set

指定字符数据类型的子类型的字符集。

对CHARACTER SET character_set的限制

如果base_type不是字符数据类型，请不要指定此子句。

不是NULL

对使用此子类型声明的数据项强加NOT NULL约束。有关此约束的信息，请参阅“ NOT NULL约束 ” 。

约束

指定数值数据类型的子类型的约束。

对约束的限制

如果base_type不是数字数据类型，请不要指定constraint 。

精确

指定数值数据类型的约束子类型的精度。

限制精度

不要指定precision如果base_type不能指定精度。

规模

指定数值数据类型的受约束子类型的比例。

规模限制

不要指定scale ，如果base_type不能指定规模。

RANGE low_value .. high_value

指定数值数据类型的受约束子类型的范围。low_value和high_value必须是数字文字。

对RANGE high_value .. low_value的限制

指定此子句仅当base_type是PLS_INTEGER或子类型PLS_INTEGER （无论是预定义的或用户定义的）。（有关PLS_INTEGER的预定义子类型的摘要，请参阅表3-3 。有关具有范围的用户定义子类型的信息，请参阅“ 约束子类型 ” 。）

身体

开始

启动块的可执行部分，其中包含可执行语句。

例外

启动块的异常处理部分。当PL / SQL引发异常时，块的正常执行停止并控制转移到适当的exception_handler 。异常处理程序完成后，将继续执行块后面的语句。有关异常处理的更多信息，请参阅PL / SQL错误处理 。

exception_handler

请参阅“ 异常处理程序 ” 。

结束

结束块。

名称

END适用的块的名称 - 标签，函数名称，过程名称或包名称。

procedure_call

程序

要调用的过程的名称。

参数 [， 参数 ] ......

要调用的过程的实际参数列表。每个实际参数的数据类型必须与相应形式参数的数据类型兼容。形式参数的模式决定了实际参数的含义：

形式参数模式 实际参数

在

常量，初始化变量，文字或表达式

OUT

其数据类型未定义为NOT NULL变量

IN OUT

变量（通常是字符串缓冲区或数字累加器）

如果过程指定参数的默认值，则可以从参数列表中省略该参数。如果过程没有参数，或者为每个参数指定默认值，则可以省略参数列表或指定空参数列表。
