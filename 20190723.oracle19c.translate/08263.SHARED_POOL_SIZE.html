1.313 SHARED_POOL_SIZE

SHARED_POOL_SIZE指定（以字节为单位）共享池的大小。

属性 描述

参数类型

大整数

句法

SHARED_POOL_SIZE = integer [K | M | G]

默认值

如果设置了SGA_TARGET ：如果未指定参数，则默认值为0 （由Oracle数据库内部确定）。如果指定了参数，则用户指定的值指示内存池的最小值。

如果未设置SGA_TARGET （32位平台）： 64 MB，则向上舍入到最接近的粒度。

如果未设置SGA_TARGET （64位平台）： 128 MB，则向上舍入到最接近的粒度。

有关使用Oracle ASM处理数据库实例时的注意事项，请参阅“ SHARED_POOL_SIZE和自动存储管理 ” 。

可修改

改变系统

可在PDB中修改

是

价值范围

最小：颗粒大小

最大值：取决于操作系统

基本

没有

共享池包含共享游标，存储过程，控制结构和其他结构。如果未设置SGA_TARGET ，则Oracle还会从共享池中分配并行执行消息缓冲区。较大的值可提高多用户系统的性能。较小的值使用较少的内存。

您可以通过查询视图V$SGASTAT来监视共享池的V$SGASTAT 。

注意：

对于可插拔数据库（PDB），此参数是可选的。为PDB设置此参数时，它保证PDB的共享池大小。如果未在PDB级别设置此参数，则PDB对其可以使用的共享池数量没有限制，除了CDB的共享池大小。

为了能够在CDB中使用资源管理器来控制每个PDB可以使用的内存量：

  • 必须在CDB级别（在CDB的根目录中）将NONCDB_COMPATIBLE初始化参数设置为FALSE 。

  • 不得在CDB级别设置MEMORY_TARGET初始化参数。

  • 如果未设置SGA_TARGET初始化参数，但是在CDB级别设置了SHARED_POOL_SIZE初始化参数，则必须满足以下要求：

    • 在PDB中设置的SHARED_POOL_SIZE的值必须小于或等于CDB级别的SHARED_POOL_SIZE值的50％。

    • CDB中所有PDB上的SHARED_POOL_SIZE值之和必须小于或等于CDB级别的SHARED_POOL_SIZE值的50％。

    将PDB中的SHARED_POOL_SIZE设置为不满足这些要求的值时，会收到错误。

如果SGA_TARGET设置为CDB级别，则必须满足以下要求：

  • PDB中DB_CACHE_SIZE加SHARED_POOL_SIZE的值必须小于或等于PDB的SGA_TARGET值的50％。

  • PDB中DB_CACHE_SIZE加SHARED_POOL_SIZE的值必须小于或等于CDB级别的SGA_TARGET值的50％。

  • CDB中所有PDB上的DB_CACHE_SIZE加SHARED_POOL_SIZE的总和必须小于或等于CDB级别的SGA_TARGET值的50％。

如果不满足上述三项要求中的任何一项，您将收到错误消息。

也可以看看：

  • 有关控制PDB内存使用情况的初始化参数的详细信息，请参阅“Oracle Multitenant管理员指南”

  • 有关设置此参数的详细信息，请参见Oracle数据库性能调整指南

SHARED_POOL_SIZE和自动存储管理

在使用Oracle自动存储管理（Oracle ASM）的数据库实例上，需要额外的内存来存储扩展区映射。作为一般准则，您可以聚合以下查询中的值以获取当前数据库存储大小，该大小已在Oracle ASM上或将存储在Oracle ASM中。然后确定使用（或将使用）的冗余类型，并使用聚合值作为输入计算SHARED_POOL_SIZE的值。 

SELECT SUM（BYTES）/（1024 * 1024 * 1024）FROM V $ DATAFILE; SELECT SUM（BYTES）/（1024 * 1024 * 1024）FROM V $ LOGFILE a，V $ LOG b WHERE a.group＃= b.group＃; SELECT SUM（BYTES）/（1024 * 1024 * 1024）FROM V $ TEMPFILE WHERE status ='ONLINE';

此外，请牢记以下准则：

  • 对于使用外部冗余的磁盘组：

    （每100G空间需要1M的额外共享池）+ 2M

  • 对于使用正常冗余的磁盘组：

    （每50G空间需要1M的额外共享池）+ 4M

  • 对于使用高冗余的磁盘组：

    （每33G空间需要1M的额外共享池）+ 6M
