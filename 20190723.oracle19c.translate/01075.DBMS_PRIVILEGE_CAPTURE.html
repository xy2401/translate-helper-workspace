125.1 DBMS_PRIVILEGE_CAPTURE概述

通过数据库特权分析，您可以创建一个策略，记录已授予用户的系统和对象特权的使用情况。然后，您可以确定用户正在使用而不使用的权限。从那里，您可以撤消任何未使用的权限，从而减少用户的超额权限授予数量。

通过分析用户执行特定任务所必需的权限，权限分析策略可帮助您为用户实现最小权限模型。

125.2 DBMS_PRIVILEGE_CAPTURE安全模型

默认情况下，权限分析管理员角色CAPTURE_ADMIN,被授予DBMS_PRIVILEGE_CAPTURE包的EXECUTE权限。

在数据库安装期间， CAPTURE_ADMIN角色被授予DBA角色。

125.4 DBMS_PRIVILEGE_CAPTURE示例

这些示例说明如何使用DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE过程创建各种类型的权限分析，如数据库分析，角色分析和特定于上下文的分析。这些例子还说明了在特定情境分析中结合不同的条件。 

- 创建数据库权限分析策略BEGIN DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE（name =>'all_priv_analysis_pol'，description =>'数据库范围的策略来分析所有权限'，type => DBMS_PRIVILEGE_CAPTURE.G_DATABASE）;结束; - 创建权限分析策略以分析角色中的权限PUBLIC BEGIN DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE（name =>'pub_analysis_pol'，description =>'用于记录PUBLIC使用权限的策略'，type => DBMS_PRIVILEGE_CAPTURE.G_ROLE，roles => role_name_list （'上市'））;结束; - 创建策略以分析来自应用程序模块的权限，“Account  -  Payable”BEGIN DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE（name =>'acc_pay_analysis_pol'，type => DBMS_PRIVILEGE_CAPTURE.G_CONTEXT，condition =>'SYS_CONTEXT（''USERENV''， ''MODULE''）=''应付账款'''）;结束; - 在运行 - 应用程序模块“应付帐款”时创建记录会话用户APPS权限的策略BEGIN DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE（name =>'acc_pay_analysis_pol'，type => DBMS_PRIVILEGE_CAPTURE.G_CONTEXT，condition =>'SYS_CONTEXT（'' USERENV''，''MODULE''）=''应付账款''和SYS_CONTEXT（''USERENV''，''SESSION_USER''）='''APPS'''）;结束;

125.5.1 CAPTURE_DEPENDENCY_PRIVS程序

此过程捕获定义者权限和调用者权限PL / SQL程序单元用于编译的权限。

句法

DBMS_PRIVILEGE_CAPTURE.CAPTURE_DEPENDENCY_PRIVS（）;

参数

此过程没有参数。

使用说明

每次重新运行DBMS_PRIVILEGE_CAPTURE.CAPTURE_DEPENDENCY_PRIVS过程都会从特权分析数据字典视图中删除任何现有记录。然后它根据现有的PL / SQL程序单元重新捕获记录。

125.5.2 CREATE_CAPTURE过程

此过程将创建一个权限分析策略，该策略指定分析权限使用的条件。它还可以选择指定要分析权限使用的角色，以及分析权限使用的条件。

句法

DBMS_PRIVILEGE_CAPTURE.CREATE_CAPTURE（名称IN VARCHAR2，描述IN VARCHAR2 DEFAULT NULL，输入IN NUMBER DEFAULT G_DATABASE，角色IN ROLE_NAME_LIST DEFAULT ROLE_NAME_LIST（），条件IN VARCHAR2 DEFAULT NULL）;

参数

表125-3 CREATE_CAPTURE过程参数

参数 描述

名称

权限分析策略的名称。一个大小不超过30个字符的字符串。

描述

政策说明（最多1024个字符）

类型

权限分析策略的类型。可能的值是：

  • G_DATABASE ：捕获数据库中的所有权限使用，但SYS用户使用的权限除外。

  • G_ROLE ：如果特权是指定角色或角色列表的一部分，则捕获特权的使用。

  • G_CONTEXT ：如果condition参数指定的上下文的计算结果为true，则捕获特权的使用。

  • G_ROLE_AND_CONTEXT ：如果特权是指定角色列表的一部分，并且condition参数指定的condition为真，则捕获特权的使用。

角色

要分析其权限的角色。如果type为G_ROLE或G_ROLE_AND_CONTEXT 。

条件

PL / SQL布尔表达式，最多包含4000个字符。如果type为G_CONTEXT或G_ROLE_AND_CONTEXT 。请注意，布尔表达式只能包含SYS_CONTEXT ，而不能包含其他函数。

使用说明

  • 对CREATE_CAPTURE过程使用基于角色的分析时，即使权限间接授予指定角色，也会分析权限使用。

    例如，假设角色R2包含角色R1，并且R1包含权限P1。如果权限策略仅包括角色R2，则仍会分析对P1权限的任何使用，因为权限P1是角色R2的间接部分。

  • 使用condition参数时，请对PL / SQL表达式使用以下语法：

    condition :: = 谓词 | （ predicate1 ）AND（ predicate2 ）| （ predicate1 ）OR（ predicate2 ）

    哪里，

    predicate :: = sys_context（ namespace ， attribute ）relop constant_value | constant_value和constant_value |之间的sys_context（ 名称空间 ， 属性 ） {constant_value（， constant_value ）*}中的sys_context（namespace，attribute）

    哪里，

    relop :: = = | <| <= | > | > = | <>

  • 权限分析策略无法分析SYS用户权限的使用。

125.5.3 DELETE_RUN过程

此过程将删除权限分析捕获运行。

句法

DBMS_PRIVILEGE_CAPTURE.DELETE_RUN（名称IN VARCHAR2，run_name IN VARCHAR2）;

参数

表125-4 DELETE_RUN过程参数

参数 描述

名称

与捕获运行关联的权限分析策略的名称

run_name

捕获运行的名称

使用说明

  • 您可以通过查询DBA_PRIV_CAPTURES数据字典视图来查找现有权限捕获策略的名称。

  • 删除捕获运行的另一种方法是删除与捕获运行相关联的策略。删除策略会自动删除其关联的捕获运行。

  • 删除捕获运行时，将无法再通过权限捕获数据字典视图访问它。

125.5.4 DISABLE_CAPTURE程序

此过程将停止记录指定权限分析策略的权限使用。禁用策略时，不再记录符合策略条件的权限使用。

句法

DBMS_PRIVILEGE_CAPTURE.DISABLE_CAPTURE（名称IN VARCHAR2）;

参数

表125-5 DISABLE_CAPTURE过程参数

参数 描述

名称

要禁用的权限分析策略的名称

使用说明

首次创建权限分析策略时，默认情况下会禁用该策略。

125.5.5 DROP_CAPTURE程序

此过程将删除权限分析策略以及记录的数据。删除策略后，将删除与策略关联的所有先前记录的权限使用数据。

句法

DBMS_PRIVILEGE_CAPTURE.DROP_CAPTURE（名称IN VARCHAR2）;

参数

表125-6 DROP_CAPTURE过程参数

参数 描述

名称

要删除的权限分析策略的名称

使用说明

  • 您必须先禁用权限分析策略，然后再将其删除。无法删除已启用的策略。

  • 如果存在与此策略关联的捕获运行，则在删除策略时会自动删除它们。

125.5.6 ENABLE_CAPTURE程序

此过程开始记录指定权限分析策略的权限分析，并可选择为此策略提供捕获运行。启用策略后，将记录策略条件下的所有权限使用。

句法

DBMS_PRIVILEGE_CAPTURE.ENABLE_CAPTURE（名称IN VARCHAR2，run_name IN VARCHAR2 DEFAULT NULL）;

参数

表125-7 ENABLE_CAPTURE过程参数

参数 描述

名称

要启用的权限分析策略的名称

run_name

与此策略关联的捕获运行的名称，少于128个字符。用双引号括起异域字符。

使用说明

以下使用说明适用：

  • 首次创建权限分析策略时，默认情况下会禁用该策略。您必须运行ENABLE_CAPTURE才能启用权限分析策略。

  • 您一次只能启用一个权限分析策略。但是，可以与另一个非G_DATABASE特权分析一起启用G_DATABASE类型的数据库范围的特权分析。

  • 您无法多次启用相同的运行。例如， run_01不能再次，如果你想重新启用捕捉使用run_01 。而是创建一个新的运行。

125.5.7 GENERATE_RESULT过程

此过程使用数据填充权限分析数据字典视图。

句法

DBMS_PRIVILEGE_CAPTURE.GENERATE_RESULT（名称IN VARCHAR2，run_name IN VARCHAR2 DEFAULT NULL，DEPENDENCY BOOLEAN DEFAULT NULL）;

参数

表125-8 GENERATE_RESULT过程参数

参数 描述

名称

要为其填充视图的权限分析策略的名称

run_name

与权限分析策略关联的捕获运行的名称。如果省略此参数，则将分析所有已创建运行的记录。指定run_name参数时，仅分析该运行的记录，并且所有其他运行不受影响。

依赖

输入Y （是）或N （否）以指示由DBMS_PRIVILEGE_CAPTURE.CAPTURE_DEPENDENCY_PRIVS过程设置的PL / SQL编译特权是否应包含在报告中。

使用说明

在填充策略的权限分析视图之前，必须禁用权限分析策略。您无法在启用的权限分析策略上调用此子程序。
