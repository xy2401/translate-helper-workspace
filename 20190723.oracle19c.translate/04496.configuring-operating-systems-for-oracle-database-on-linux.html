4在Linux上配置Oracle数据库的操作系统

在开始安装之前，请完成操作系统配置要求并检查Linux操作系统。

在启动Oracle数据库或Oracle Grid Infrastructure安装之前，您必须满足此版本的操作系统分发，软件包和其他软件要求等最低要求。

  • Linux操作系统安装指南
    在继续Oracle安装之前要注意的操作系统准则。
  • 查看操作系统和软件升级最佳实践
    这些主题提供有关升级和迁移的一般规划指南和特定于平台的信息。
  • 查看操作系统安全通用实践
    安全操作系统是一般系统安全性的重要基础。
  • 关于安装修复脚本
    Oracle Universal Installer检测何时不满足安装的最低要求，并创建称为修复脚本的shell脚本，以完成不完整的系统配置步骤。
  • 关于操作系统要求
    根据您要安装的产品，验证您是否安装了所需的操作系统内核和软件包。
  • x86-64 Linux平台的操作系统要求
    x86-64上的此版本支持本节中列出的Linux发行版和软件包。
  • 适用于Linux的其他驱动程序和软件包
    有关可选驱动程序和软件包的信息。
  • 检查Linux的内核和程序包要求
    验证内核和软件包，看它们是否满足最低安装要求。
  • 安装适用于Linux的cvuqdisk RPM
    如果不使用Oracle预安装RPM，并且要使用Cluster Verification Utility，则必须安装cvuqdisk RPM。
  • 确认主机名解析
    检查以确保服务器的主机名是可解析的。
  • 禁用透明HugePages
    Oracle建议您在开始安装之前禁用Transparent HugePages。
  • 在安装期间使用自动SSH配置
    要安装Oracle软件，请在所有集群成员节点之间配置安全shell（SSH）连接。
  • 验证Linux上的磁盘I / O调度程序
    为了获得Oracle ASM的最佳性能，Oracle建议您使用Deadline I / O Scheduler。
