1.22 AUDIT_FILE_DEST

AUDIT_FILE_DEST指定当AUDIT_TRAIL初始化参数设置为扩展的 os ， xml或xml时写入审计跟踪的操作系统目录。

属性 描述

参数类型

串

句法

AUDIT_FILE_DEST = ' directory '

默认值

第一个默认值是：

ORACLE_BASE /admin/ ORACLE_SID /adump

如果第一个默认值不存在或不可用，则使用第二个默认值：

ORACLE_HOME /rdbms/audit

这两个默认值都适用于UNIX系统。其他平台可能具有不同的默认值。

在多租户容器数据库（CDB）中，这两个默认值都将附加可插拔数据库（PDB）的GUID，以存储属于PDB的审计记录。例如，如果PDB的GUID是03E1F908EE04252CE053B280E80AAAA3 ，则第一个默认目录将是：

ORACLE_BASE /admin/ ORACLE_SID /adump/03E1F908EE04252CE053B280E80AAAA3

您可以使用V $ CONTAINERS视图查询PDB的GUID。

可修改

改变系统......DEFERRED

可在PDB中修改

没有

基本

没有

注意：

在已迁移到统一审计的Oracle数据库中，此参数的设置无效。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

如果AUDIT_TRAIL初始化参数设置为xml或xml，则将以XML格式写入审计记录。它也是写入强制审计信息的位置，如果由AUDIT_SYS_OPERATIONS初始化参数指定，则是用户SYS审计记录。

在一个多租户容器数据库 （CDB），此初始化参数的设置范围是CDB。尽管提供了审计跟踪可插拔数据库 （CDB）在CDB中，不能为单个PDB配置此初始化参数。
