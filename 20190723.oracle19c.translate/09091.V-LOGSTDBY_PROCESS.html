8.59 V $ LOGSTDBY_PROCESS

V$LOGSTDBY_PROCESS显示有关Data Guard日志应用服务发生情况的动态信息。

在将归档重做日志逻辑应用到备用数据库期间诊断性能问题时，此视图很有用，对于其他问题可能会有所帮助。此视图仅适用于逻辑备用数据库。

柱 数据类型 描述

SID

数

关联会话的会话ID。这与V$SESSION视图中相应行的SID列匹配。

SERIAL＃

数

关联会话的序列号。同时，（ SID ， SERIAL# ）唯一标识当前数据库实例中的会话。

LOGSTDBY_ID

数

并行查询从站ID

VARCHAR2（24）

类型

VARCHAR2（128）

进程在SQL Apply的上下文中扮演的角色：

  • 协调员

  • 施放

  • 分析仪

  • 读者

  • PREPARER

  • BUILDER

STATUS_CODE

数

识别流程当前操作的操作代码：

  • 16111 - SQL Apply进程正在初始化

  • 16112 - SQL Apply进程正在清理为apply，并且挖掘进程正在基于用户命令停止

  • 16116 - SQL Apply进程空闲

  • 16117 - SQL Apply进程正忙，并没有等待任何有趣的事件

  • 16110 - APPLIER进程已调用用户提供的存储过程，以便在处理DDL语句之前检查它

  • 16113 - APPLIER进程正在将DML更改应用于某些用户对象或序列

  • 16114 - APPLIER进程正在应用DDL更改

  • 16115 - COORDINATOR进程正在从重做流加载LogMiner字典

  • 16243 - BUILDER进程正在分页内存以释放lcr缓存中的空间

  • 16240 - READER进程空闲等待其他日志文件可用

  • 16241 - READER进程空闲，等待日志文件填充日志序列间隙

  • 16242 - READER进程正在处理日志文件

状态

VARCHAR2（256）

该过程的当前动作的描述

HIGH_SCN

数

标识此过程处理的最高重做记录/ LCR

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
