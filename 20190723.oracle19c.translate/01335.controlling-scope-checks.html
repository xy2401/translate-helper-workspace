A.2控制支票范围

使用本节中的命令列表来控制检查范围。

句法

[-b] [-p] [-m] [-u -o pre] [-u -o post] [-clusternodes nodes ] [-failedchecks previous_result ] [-nordbms] [-dbnames db_names ] [-dbnone] [ -dball] [-localonly] [-cells cells ] [-ibswitches switches ] [-profile profile ] [-excludeprofile profile ] [-check check_id ] [-excludecheck check_id ] [-skip_usr_def_checks]

参数

表A-2检查范围

命令 描述

-b

只运行最佳实践检查。

不运行建议的补丁检查。

-p

仅运行补丁检查。

-m

排除对最大可用性体系结构（MAA）记分卡的检查。

-u -o pre

运行Oracle Clusterware和数据库的升级前检查。

-u -o post

运行Oracle Clusterware和数据库的升级后检查。

-clusternodes nodes

指定以逗号分隔的节点名称列表，以仅在节点子集上运行。

-failedchecks previous_result

只运行失败的presious_result检查。

-nordbms

仅在未执行Oracle数据库检查的环境中运行Oracle Grid Infrastructure检查。

-dbnames db_names

指定以逗号分隔的数据库名称列表，以仅在数据库子集上运行。

-dbnone

不提示选择数据库并跳过所有数据库检查。

-dball

不提示选择数据库，并对系统上发现的所有数据库运行数据库检查。

-localonly

仅在本地节点上运行。

- cells

指定以逗号分隔的存储服务器名称列表，以仅对存储服务器的子集运行检查。

-ibswitches switches

指定以逗号分隔的InfiniBand交换机名称列表，以仅对InfiniBand交换机的子集运行检查。

-profile profile

指定以逗号分隔的配置文件列表，以仅运行指定配置文件中的检查。

-excludeprofile profile

指定以逗号分隔的配置文件列表，以排除指定配置文件中的检查。

-check check_id

指定以逗号分隔的检查ID列表，以仅运行列表检查ID中指定的检查。

-excludecheck check_id

指定以逗号分隔的检查ID列表，以排除检查ID列表中指定的检查。

-skip_usr_def_checks

不运行用户定义的xml文件中指定的检查。
