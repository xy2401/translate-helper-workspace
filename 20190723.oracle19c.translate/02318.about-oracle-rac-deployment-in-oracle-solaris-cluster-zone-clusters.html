关于Oracle Solaris Cluster区域群集中的Oracle RAC部署

区域群集由多个Oracle Solaris区域组成，每个区域位于各自独立的服务器上;组成群集的区域链接在一起成为一个虚拟群集。

多个区域群集可以存在于单个全局群集上，从而提供了在全局群集上整合多群集应用程序的方法。可以在物理计算机或虚拟机（如Oracle VM Server for SPARC逻辑域）上配置全局群集的节点。全局集群还可以具有物理节点和虚拟节点的组合。

在区域群集中安装Oracle Real Application Clusters（Oracle RAC）允许您拥有单独的数据库版本或同一数据库的单独部署（例如，一个用于生产，一个用于开发）。使用此体系结构，您还可以将多层解决方案的不同部分部署到不同的虚拟区域群集中。例如，您可以在同一全局集群的不同区域集群中部署Oracle RAC和应用程序服务器。此方法允许您将层和管理域相互隔离，同时利用Oracle Solaris Cluster提供的简化管理。
