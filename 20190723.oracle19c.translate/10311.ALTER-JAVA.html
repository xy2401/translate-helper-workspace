ALTER JAVA

目的

使用ALTER JAVA语句强制解析Java类模式对象或编译Java源模式对象。（在Java类的所有外部引用与其他类关联之前，您无法调用Java类的方法。）

有关解析Java类和编译Java源代码的更多信息，请参见“Oracle数据库Java开发人员指南”

先决条件

Java源代码或类必须位于您自己的架构中，或者您必须具有ALTER ANY PROCEDURE系统特权。您还必须具有Java类的EXECUTE对象特权。

语义

JAVA来源

使用ALTER JAVA SOURCE编译Java源模式对象。

JAVA CLASS

使用ALTER JAVA CLASS解析Java类架构对象。

OBJECT_NAME

指定以前创建的Java类或源模式对象。使用双引号来保留较低或混合大小写的名称。

RESOLVER

RESOLVER子句允许您使用创建Java类或源时指定的映射对来指定如何搜索引用的完全指定的Java名称的模式。

也可以看看：

创建JAVA和“ 解析Java类：示例 ”

解析| COMPILE

RESOLVE和COMPILE是同义关键字。它们允许您指定Oracle数据库应尝试解析主Java类架构对象。

  • 应用于类时，会发生对其他类模式对象的引用名称的解析。

  • 应用于源时，会发生源编译。

invoker_rights_clause

invoker_rights_clause允许您指定类的方法是使用特权以及在定义它的用户的模式中执行还是使用特权和CURRENT_USER模式执行。

此子句还确定Oracle数据库如何在类型的成员函数和过程中解析查询，DML操作和动态SQL语句中的外部名称。

AUTHID CURRENT_USER

指定CURRENT_USER如果希望类的方法与权限执行CURRENT_USER 。此子句是默认值并创建一个调用者权限类 。

此子句还指定查询，DML操作和动态SQL语句中的外部名称在CURRENT_USER的模式中解析。所有其他语句中的外部名称在方法所在的模式中解析。

AUTHID DEFINER

如果希望使用定义类的用户的权限执行类的方法，请指定DEFINER 。

此子句还指定外部名称在方法所在的模式中解析。
