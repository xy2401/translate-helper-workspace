统一审核策略的命令规则支持

您现在可以为统一审核策略创建Oracle Database Vault命令规则。

您现在可以使用命令规则来启用或禁用统一审核策略。通过此功能，您可以直接指定统一策略名称作为命令规则的一部分。此新功能扩展了命令规则的AUDIT和NOAUDIT使用，但是当您为命令规则指定统一审核策略时，必须指定AUDIT POLICY或NOAUDIT POLICY 。

基础架构数据库管理员的Database Vault Operations Control

在多租户数据库中，您现在可以使用Oracle Database Vault阻止普通用户（例如，基础架构DBA）访问自治，常规云或内部部署环境中的可插拔数据库（PDB）中的本地数据。

此增强功能可防止普通用户访问驻留在PDB上的本地数据。它使您能够为业务应用程序存储敏感数据，并允许操作管理数据库基础结构，而无需访问敏感的客户数据。

特权分析现在可在Oracle数据库企业版中使用

权限分析现在作为Oracle数据库企业版的一部分提供。

权限分析对用户和应用程序进行动态分析，以查找已使用和未使用的权限和角色。权限分析通过向您显示每个帐户使用和未使用的权限来减少实施最低权限最佳实践的工作。权限分析具有高度的性能，旨在用于测试，开发和生产数据库。

作为此更改的一部分，权限分析文档已从Oracle Database Vault管理员指南转移到Oracle数据库安全指南 。

Oracle Database Vault模拟模式的增强功能

Oracle Database Vault对此版本的仿真模式进行了一些更改。

  • 模拟模式现在可以从SQL语句中捕获所有必需的域违规。

  • 模拟模式可以捕获完整的调用堆栈信息。

  • 现在，默认的可信路径上下文因子可作为单独的列使用，而不是连接在一起。

从SQL语句捕获所有强制域违规使您可以查看可能需要进行的所有更改。否则，第一个强制域违规可能会掩盖在原始修复完成并运行另一个回归测试之前不会被注意到的其他违规。此增强功能可实现更快的回归测试和应用程

查看完整的调用堆栈可帮助您识别具有违规的原始SQL语句。在许多情况下，类似的SQL语句由应用程序的不同部分调用。此功能可帮助应用程序开发人员快速确定触发违规的应用程序代码。

上下文因素用于为域和命令规则构建可信路径。可信路径有一些常用因素，因此这些因素从最后一个版本中的单个字符串表示提取到它们自己的列中。此增强功能可以更轻松地识别在受信任路径规则集中使用的因子。

新因子功能

从此版本开始，可以使用四个新的因子函数。

因子函数如下：

  • F $ DV $ _CLIENT_IDENTIFIER

  • F $ DV $ _DBLINK_INFO

  • F $ DV $ _module

  • F $ PROXY_USER

能够将数据泵 - 数据库Vault授权授予角色

从此版本开始，您可以授权角色在Oracle Database Vault环境中执行Oracle Data Pump操作。

在以前的版本中，您只能将此授权授予单个用户。此增强功能使管理员可以通过此类授权的角色轻松管理用户。

Oracle Database Vault支持Oracle数据库重放

在此版本中，您现在可以在Oracle Database Vault环境中执行Oracle数据库重放操作。

以下功能支持此功能：

  • DBMS_MACADM PL / SQL程序：

    • DBMS_MACADM.AUTHORIZE_DBCAPTURE

    • DBMS_MACADM.AUTHORIZE_DBREPLAY

    • DBMS_MACADM.UNAUTHORIZE_DBCAPTURE

    • DBMS_MACADM.UNAUTHORIZE_DBREPLAY

  • 数据字典视图：

    • DBA_DV_DBCAPTURE_AUTH

    • DBA_DV_DBREPLAY_AUTH
