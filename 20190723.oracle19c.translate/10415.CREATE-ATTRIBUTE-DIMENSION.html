创建属性维度

目的

使用CREATE ATTRIBUTE DIMENSION语句创建属性维度。属性维指定一个或多个分析视图层次结构的维度成员。它指定它正在使用的数据源及其包含的成员。它指定其成员的级别并确定级别之间的属性关系。

小费：

您可以在Oracle Live SQL网站上查看和运行创建属性维的SQL脚本，网址为https://livesql.oracle.com/apex/livesql/file/index.html 。该网站提供脚本和教程，演示分析视图的创建和使用。

先决条件

要在自己的模式中创建属性维，您必须具有CREATE ATTRIBUTE DIMENSION系统特权。要在另一个用户的模式中创建属性维，您必须具有CREATE ANY ATTRIBUTE DIMENSION系统特权。

语义

或替换

指定OR REPLACE以使用不同的定义替换属性维度的现有定义。

力量和NOFORCE

指定FORCE以强制创建属性维，即使它未成功编译也是如此。如果指定NOFORCE ，则属性维必须成功编译，否则会发生错误。默认为NOFORCE 。

模式

指定要在其中创建属性维度的架构。如果未指定架构，则Oracle数据库会在您自己的架构中创建属性维度。

attr_dimension

指定属性维的名称。

sharing_clause

指定是否将属性维创建为应用程序公共对象。指定METADATA共享属性维度的元数据，但其数据对每个容器都是唯一的。指定DATA共享属性维对象;它的数据对于应用程序容器中的所有容器都是相同的，并且数据仅存储在应用程序根目录中。指定NONE从共享中排除属性维度。

classification_clause

使用classification子句指定CAPTION或DESCRIPTION分类的值，并指定用户定义的分类。分类提供描述性元数据，应用程序可用于提供有关分析视图及其组件的信息。

您可以为同一对象指定任意数量的分类。分类的最大长度为4000字节。

对于CAPTION和DESCRIPTION分类，你可以使用快捷键DDL CAPTION ' caption '和DESCRIPTION ' description '或完全分类的语法。

您可以按语言更改分类值。要为CAPTION或DESCRIPTION分类指定语言，必须使用完整语法。如果未指定语言，则分类的语言值为NULL 。语言值必须为NULL或有效的NLS_LANGUAGE值。

尺寸类型

属性维度可以是STANDARD或TIME类型。STANDARD类型属性维度具有STANDARD类型级别。TIME类型属性维度的每个级别都是TIME类型之一。默认的DIMENSION TYPE是STANDARD 。

attr_dim_using_clause

使用USING关键字，指定表或视图。您可以使用AS关键字为表或视图指定别名。（可选）您可以指定连接路径。当属性维使用在雪花模式中组织的表时，使用连接路径指定连接。

join_path_clause

join path子句指定不同表中列之间的连接条件。对于USING子句中包含的每个连接路径， join_path_name参数指定的连接路径的名称必须是唯一的。

join_condition

连接条件由一个或多个连接条件元素组成; AND操作包含每个附加连接条件元素。

join_condition_element

在连接条件元素中，左侧的列引用必须来自与右侧的列引用不同的表。

attributes_clause

指定一个或多个attr_dim_attribute_clause子句。

attr_dim_attribute_clause

从attr_dim_using_clause源指定一列。除非使用AS关键字指定别名，否则该属性具有列的名称。您可以为每个属性指定分类。

attr_dim_level_clause

在属性维度中指定级别。级别指定提供级别成员的键和可选备用键属性。

如果key属性没有NULL值，那么您可以指定NOT NULL ，这是默认值。如果它确实有一个或多个NULL值，则指定SKIP WHEN NULL 。

等级类型

STANDARD类型属性维度具有STANDARD类型级别。您不需要为STANDARD类型属性维指定LEVEL TYPE 。

在TIME类型属性维度中，您必须指定级别类型。级别的类型可以是时间类型之一。即使级别成员的值不是该类型，您也必须指定时间类型。例如，您可能具有SEASON级别，其值为季节名称。在定义级别时，必须指定任何一种时间级别类型，例如QUARTERS 。应用程序可以将级别类型名称用于其选择的任何目的。

确定

使用DETERMINES关键字，您可以指定此级别确定的属性维度的其他属性。如果属性对于另一个属性的每个值只有一个值，则第一个属性的值确定另一个属性的值。例如，QUARTER_ID属性对MONTH_ID属性的每个值只有一个值，因此您可以在MONTHS级别的DETERMINES短语中包含QUARTER_ID属性。

key_clause

指定一个或多个属性作为级别的键。

alternate_key_clause

指定一个或多个属性作为级别的备用键。

dim_order_clause

指定级别成员的顺序。

all_clause

（可选）为隐式ALL级别指定MEMBER NAME ， MEMBER CAPTION和MEMBER DESCRIPTION值。默认情况下， MEMBER NAME值为ALL。
