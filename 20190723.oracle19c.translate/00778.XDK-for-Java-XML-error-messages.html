一种用于XML的Java错误消息XDK

对于在执行可扩展标记语言（XML）接口期间使用Oracle XML Developer Kit（XDK）for Java的应用程序，列出了错误消息。

XML分析器错误消息

可扩展标记语言（XML）解析器错误消息的范围是XML-20000到XML-20999。

XML-20003：丢失的线串记号字符串 ， 字符串列

原因：在输入数据中找不到预期的令牌。

操作：检查/更新输入数据以修复语法错误。

XML-20004：丢失的线串关键字字符串 ， 字符串列

原因：在输入数据中找不到预期的关键字。

操作：检查/更新输入数据到正确的关键字。

XML-20005：丢失的线串关键字串或字符串 ， 字符串列

原因：在输入数据中找不到预期的关键字。

操作：检查/更新输入数据到正确的关键字。

XML-20006：行字符串 ，列字符串的意外文本;预期的EOF

原因：在根元素的结束标记之后找到了更多文本。

操作：根元素的结束标记只能跟随注释，PI或空格。删除结束标记后的额外文本。

XML-20007：在行字符串 ，列字符串的元素声明中缺少内容模型

原因：元素声明缺少必需的内容模型规范。请参阅XML 1.0第2版中的Production [45]。

操作：将所需的内容规范添加到元素声明中。

XML-20008：行字符串中的内容模型中缺少元素名称，列字符串

原因：元素声明中的内容模型无效，内容粒子需要元素名称。请参阅XML 1.0第2版中的Production [48]。

操作：添加元素名称以语法方式修复内容规范。

XML-20009：行字符串处理指令的目标名称字符串 ，保留列字符串

原因：目标名称"XML: xml"等在XML规范的未来版本中保留用于标准化。请参阅XML 1.0第2版中的Production [17]。

操作：如果PI意图是XML声明，请确保声明发生在文件的最开头。否则，请更改为PI的名称。

XML-20010：在行字符串 ，列字符串中未解析的实体声明中缺少表示法名称

原因：未解析的实体声明中使用的表示法名称与声明的表示法中的名称不匹配。请参阅XML 1.0第2版中的Production [76]。

操作：将符号声明添加到DTD。

XML-20011：在行字符串 ，列字符串的属性列表声明中缺少属性类型

原因：属性类型缺少属性列表声明。必须添加其中一种类型CDATA，ID，IDREF，IDREFS，ENTITY，ENTITIES，NMTOKEN或NMTOKENS。请参阅XML 1.0第2版中的Production [52]，[53]。

操作：检查并更正属性声明。

XML-20012：在行字符串 ，列字符串处缺少空格

原因：缺少所需的空白区域。

操作：添加空格以修复语法错误。

XML-20013：在实体值无效的字符串中的行字符串 ， 字符串列

原因：实体值中使用了无效字符。字符& ， %和"或' （基于值分隔符）无效。请参阅XML 1.0第2版中的Production [9]。

操作：使用实体或字符引用而不是字符例如， &amp;或&#38;可以用来代替& 。

XML-20014： - 不允许在行字符串 ，列字符串的注释中

原因：由于使用“ - ”而在注释中出现语法错误请参阅XML 1.0第2版中的Production [15]。

操作：修复注释，并使用--仅作为注释结束的一部分-->

XML-20015：]>不允许在行字符串 ，列字符串的文本中

原因： ]>文本中不允许。它仅用作CDATA部分的结束标记。请参阅XML 1.0第2版中的Production [14]。

操作：使用&gt;修复文本内容或者char ref > 。

XML-20016：在行字符串 ，列字符串的出现指示符之前不允许有空格

原因：在发生指示符之前的contentspec中不允许有空格。例如， <!ELEMENT x (a,b) *>无效。请参阅XML 1.0第2版中的Production [47]，[48]。

操作：通过删除多余的空间来修复contentspec

XML-20017：发生指示符串在混合内容不允许在线串 ，列串

原因：混合内容声明中不允许出现。例如， <!ELEMENT x（#PCDATA）？>无效。请参阅XML 1.0第2版中的Production [51]。

操作：修复语法以删除发生指示符。

XML-20018：内容列表不允许在行字符串 ，列字符串的混合内容中

原因：混合内容声明中不允许使用内容列表。例如， <!ELEMENT x (#PCDATA | (a,b))>无效。请参阅XML 1.0第2版中的Production [51]。

操作：修复语法以删除内容列表。

XML-20019：在line 字符串中的混合内容的声明重复的元素字符串 ， 字符串列

原因：在混合内容声明中找到了重复的元素名称。例如， <!ELEMENT x (#PCDATA | a | a)>无效。请参阅XML 1.0第2版中的Production [51]

操作：删除重复的元素名称。

XML-20020：根元素字符串与行字符串 ，列字符串处的DOCTYPE名称字符串不匹配

原因：失败：文档类型声明中的名称必须与根元素的元素类型匹配。例如： <?xml版本=“1.0”？> <！DOCTYPE 问候语 [<！元素问候 （#PCDATA）>]> < 敬礼 >你好！</ salutation > 。文档的根元素salutation与DTD（ greeting ）中声明的根元素不匹配。

原因：类型ID的值必须与名称生成匹配。名称在XML文档中不得出现多次作为此类型的值;因此，ID值必须唯一地标识承载它们的元素。
