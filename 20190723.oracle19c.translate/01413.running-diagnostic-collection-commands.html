E.4运行诊断收集命令

运行诊断收集命令以收集诊断数据。

  • tfactl集合
    使用tfactl collection命令停止正在运行的Oracle Trace File Analyzer集合。
  • tfactl dbglevel
    使用tfactl dbglevel命令设置Oracle Grid Infrastructure跟踪级别。
  • tfactl diagcollect
    使用tfactl diagcollect命令执行按需诊断收集。
  • tfactl diagcollect -srdc
    使用tfactl diagcollect -srdc命令运行服务请求数据收集（SRDC）。
  • tfactl目录
    使用tfactl directory命令向目录列表添加目录或从目录列表中删除目录以分析其跟踪或日志文件。
  • tfactl ips
    使用tfactl ips命令收集自动诊断存储库诊断数据。
  • tfactl managelogs
    使用tfactl managelogs命令管理自动诊断存储库日志和跟踪文件。
  • tfactl清除
    使用tfactl purge命令从Oracle Trace File Analyzer存储库中删除早于特定时间的诊断集合。

E.4.3 tfactl diagcollect

使用tfactl diagcollect命令执行按需诊断收集。

Oracle跟踪文件分析器收集器可以执行三种类型的按需收集：

  • 默认集合

  • 事件驱动的支持服务请求数据收集（SRDC）集合

  • 定制集合

先决条件

事件驱动的支持服务请求数据收集（SRDC集合需要Oracle跟踪文件分析器数据库支持工具包中的组件，可从My Oracle Support说明1513912.2获得。

句法

tfactl diagcollect [[ component_name1 ] [ component_name2 ] ... [ component_nameN ] | [-srdc srdc_profile ] | [-defips]] [-sr SR# ] [-node all | local | n1 ， n2 ，...][-tag tagname ] [-z filename ] [-last n h | d | - 从time到time | - time ] [-nocopy] [-notrim] [-silent] [-nocores] [-collectalldirs] [-collectdir dir1 ， dir2 ...][-examples]组件：-ips | -database | -asm | -crsclient | -dbclient | -dbwlm | -tns | -rhp | -procinfo | -afd | -crs | -cha | -wls | -emagent | -oms | -ocm | -emplugins | -em | -acfs | -install | -cfgtools | -os | -ashhtml | -ashtext | -awrhtml | -awrtext

参数

使用减号（ - ）为每个选项添加前缀。

选项 描述

[[ component_name1 ] [ component_name2 ] ... [ component_nameN ] | [-srdc srdc_profile ] | [-defips]]]

指定要为其获取集合的组件列表，或指定SRDC名称，或指定在默认集合中包含ASM，CRS和Oracle数据库的IPS软件包。

[-sr SR# ]

指定Oracle跟踪文件分析器自动上载所有集合的服务请求编号。

-node all | local | n1,n2,...

指定以逗号分隔的节点列表，从中收集诊断信息。默认是全部。

- 标签description

使用此参数可在Oracle Trace File Analyzer存储库中为生成的集合创建子目录。

-z file_name

使用此参数指定输出文件名。

[-last n h | d | - 从time到time | - time ]

  • 指定-last参数以收集具有过去特定小时数（ h ）或天数（ d ）的相关数据的文件。默认情况下，使用带有此参数的命令还会修剪较大的文件，并仅显示指定时间间隔内的文件。

    您还可以使用-since ，它具有与-last相同的功能。包含此选项是为了向后兼容。

  • 指定-from和-to参数（必须同时使用这两个参数）以收集在特定时间间隔内具有相关数据的文件，并在文件较大之前修剪数据。

    支持的时间格式：

    “星期一/日/月亮......：mm：ss”

    “yyyy-mm-dd hh：mm：ss”

    “YYYY-MM-DDTHH：MM：SS”

    “YYYY-MM-DD”

  • 指定-for参数以收集在给定时间内具有相关数据的文件。该文件TFACTL收集将在时间戳之间你以后指定的时间-for被包括在内。没有为此选项进行数据修整。

    支持的时间格式：

    “星期一/日/年”

    “YYYY-MM-DD”

注意：

如果同时指定日期和时间，则必须将这两个值括在双引号（“”）中。如果仅指定日期或时间，则不必将单个值括在引号中。

-nocopy

指定此参数可以停止将生成的跟踪文件集合复制回初始节点。该文件保留在执行节点上的Oracle Trace File Analyzer存储库中。

-notrim

指定此参数可停止修剪收集的文件。

-无声

指定此参数以将诊断集合作为后台进程运行

-nocores

指定此参数以在收集核心文件时停止收集核心文件。

-collectalldirs

指定此参数以从Collect All标记为true的目录中收集所有文件。

-collectdir dir1 ， dir2 ，... dirn

指定逗号分隔的目录列表，集合包括来自这些目录的所有文件，与指定的组件除了类型和时间限制之外。

E.4.5 tfactl目录

使用tfactl directory命令向目录列表添加目录或从目录列表中删除目录以分析其跟踪或日志文件。

另外，使用tfactl directory命令更改目录权限。当自动发现添加目录时，该目录将添加为公共目录。任何具有足够权限来运行tfactl diagcollect命令的tfactl diagcollect收集该目录中的任何文件。这在非root用户或sudo用户运行TFACTL命令时才很重要。

如果目录被标记为私有，那么Oracle Trace File Analyzer，在允许收集任何文件之前：

  • 确定哪个用户正在运行TFACTL命令

  • 验证用户是否有权查看目录中的文件

注意：

用户只能将目录添加到他们具有读访问权限的Oracle跟踪文件分析器。如果配置了自动诊断集合，则Oracle Trace File Analyzer以root身份运行，并且可以收集所有可用文件。

tfactl directory命令包含三个动词，您可以使用它们来管理目录： add ， remove和modify 。

句法

tfactl目录添加directory [-public] [-exclusions | -noexclusions | -collectall] [-node all | n1,n2... ]
tfactl目录删除directory [-node all | n1,n2... ]
tfactl目录修改directory [-private | -public] [-exclusions | -noexclusions | -收集所有]

对于三种语法模型中的每一种，都必须指定Oracle跟踪文件分析器存储集合的目录路径。

参数

参数 描述

-私人的

使用-private参数可以防止无权查看要添加或修改的目录（以及任何子目录）中的文件的Oracle跟踪文件分析器用户运行命令以从指定目录收集文件。

-exclusions

如果文件满足类型，名称和时间范围限制，请使用-exclusions参数指定此目录中的文件是否有资格进行收集。

如果文件满足时间范围限制，请使用-noexclusions参数指定此目录中的文件是否有资格进行收集。

  • Oracle数据库

  • Oracle Grid Infrastructure

  • 操作系统日志

  • 其他一些组件

  • 哪个数据库或实例

注意：

对于OS，CRS，CFGTOOLS，ACFS，ALL或INSTALL文件，仅请求组件，对于Oracle ASM，仅创建实例。没有对这些条目进行验证，因此在输入此数据时请小心。

  • KEYFILE file_spec ：生成远程密钥文件。

  • CORRELATE TYPICAL ：该软件包包括事件转储和在每个事件周围的时间窗口中修改的所有跟踪文件。如果事件共享相关的关联密钥，或者发生在主要事件周围的时间窗口中，则会自动包含更多事件。

  • CORRELATE ALL ：该软件包包括事件转储和在第一个选定事件和最后一个选定事件之间修改的所有跟踪文件。如果事件发生在同一时间范围内，则会自动包含更多事件。

  • MANIFEST file_spec ：生成XML格式包清单文件。

  • KEYFILE file_spec ：生成远程密钥文件。

注意：

如果未指定包内容，例如事件，问题等，则Oracle跟踪文件分析器会创建一个空包。

您可以稍后添加文件和事件。

如果未指定关联级别，则Oracle跟踪文件分析器将使用默认级别。

默认值通常为TYPICAL ，但您可以使用IPS SET CONFIGURATION命令进行更改。

句法

tfactl purge -older n [ h | d ] [-force]

$ tfactl purge -older 30d
要删除超过10小时的文件：
$ tfactl purge -older 10h
