如果Oracle库存不存在，则创建Oracle库存组

作为计划安装的一部分手动创建Oracle Inventory组，尤其是在服务器上安装了多个Oracle软件产品的情况下。

默认情况下，如果oraInventory组不存在，则安装程序将使用安装所有者的主组作为oraInventory组安装的Oracle软件。确保该组可用作所有计划的Oracle软件安装所有者的主要组。
如果oraInst.loc文件不存在，则通过输入类似于以下内容的命令来创建Oracle Inventory组：
＃/ usr / sbin / groupadd -g 54321 oinstall
