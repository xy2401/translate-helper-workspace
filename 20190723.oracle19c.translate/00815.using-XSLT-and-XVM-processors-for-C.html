4使用C的XSLT和XVM处理器

解释了如何为C使用可扩展样式表语言转换（XSLT）和XSLT虚拟机（XVM）处理器。

注意：

使用适用于Oracle XML Developer Kit（XDK）和Oracle XML DB应用程序的统一C应用程序编程接口（API）。不推荐使用较旧的非经过授权的C函数，仅支持向后兼容性。它们将在以后的版本中删除。

使用XML Parser for C中描述了统一的C API。

XSLT XVM处理器

Oracle XVM包包括XSLT编译器和XVM。该软件包实现了1999年11月16日万维网联盟（W3C）建议书中规定的XSLT语言。

实现XSLT编译器和XVM可以将XSLT（版本1.0）编译为字节码格式，该格式由虚拟机执行。XVM是用于运行已编译的XSLT代码的CPU的软件实现。虚拟机假定编译器将XSLT样式表编译为一系列字节码或XSLT CPU的机器指令。字节码程序是与平台无关的2字节单元序列。它可以在不同的XVM上存储，缓存和运行。 XVM使用字节码程序转换XML实例文档。这种方法明确地将编译时与运行时计算分开，并指定了在指令之间交换数据的统一方式。

这种方法的好处是：

  • XSLT样式表可以编译，保存在文件中，并经常重复使用，即使在不同的平台上也是如此。

  • 与其他XSLT处理器相比，XVM速度更快，占用的内存更少。

  • 字节码与语言无关。从C或C ++ XSLT编译器生成的代码之间没有区别。

XVM用法示例

描述了使用包API的典型方案。

  1. 创建和使用XML元上下文对象。

    xctx = XmlCreate（＆err，...）;

  2. 创建和使用XSLT编译器对象。

    comp = XmlXvmCreateComp（xctx）;

  3. 编译XSLT样式表或XPath表达式并存储或缓存生成的字节码。

    code = XmlXvmCompileFile（comp，xslFile，baseuri，flags，＆err）;

    要么

    code = XmlXvmCompileDom（comp，xslDomdoc，flags，＆err）;

    要么

    code = XmlXvmCompileXPath（comp，xpathexp，namespaces＆err）;

  4. 创建和使用XVM对象。当XVM以Stack Overflow消息终止或需要较小的内存占用时，需要显式堆栈大小设置。请参阅XmlXvmCreate() 。 

    vm = XmlXvmCreate（xctx，“StringStack”，32，“NodeStack”，24，NULL）;

  5. 设置输出（可选）。默认是一个流。

    err = XmlXvmSetOutputDom（vm，NULL）;

    要么

    err = XmlXvmSetOutputStream（vm，＆xvm_stream）;

    要么

    err = XmlXvmSetOutputSax（vm，＆xvm_callback，NULL）;

  6. 将样式表字节码设置为XVM对象。可以与其他字节码重复。

    len = XmlXvmGetBytecodeLength（code，＆err）; err = XmlXvmSetBytecodeBuffer（vm，code，len）;

    要么

    err = XmlXvmSetBytecodeFile（vm，xslBytecodeFile）;

  7. 转换实例XML文档或评估已编译的XPath表达式。可以使用相同或其他XML文档重复。

    err = XmlXvmTransformFile（vm，xmlFile，baseuri）;

    要么

    err = XmlXvmTransformDom（vm，xmlDomdoc）;

    要么

    obj =（xvmobj *）XmlXvmEvaluateXPath（vm，code，1,1，node）;

  8. 获取输出树片段（如果在步骤5中设置了DOM输出）。

    node = XmlXvmGetOutputDom（vm）;

  9. 删除对象。

    XmlXvmDestroy（VM）; XmlXvmDestroyComp（COMP）; XmlDestroy（xctx）;

使用XVM处理器命令行实用程序

可以使用命令xvm从命令行访问XVM处理器。 

XVM

用法：

xvm 选项xslfile xmlfile
xvm 选项xpath xmlfile

选项：

-c编译xslfile 。字节码在“xmlfile.xvm”中 。-ct编译xslfile并转换xmlfile 。-t使用xslfile中的字节码转换xmlfile 。-xc编译xpath 。字节码位于“code.xvm”中。-xct编译和XMLFILE计算XPath。-xt使用xmlfile从xpath计算XPath字节码。

例子：

xvm -ct db.xsl db.xml xvm -t db.xvm db.xml xvm -xct“doc / employee [15] / family”db.xml

XSLT处理器使用示例

提供了使用包API的典型方案。

  1. 创建和使用XML元上下文对象。

    xctx = XmlCreate（＆err，...）;

  2. 解析XSLT样式表。

    xslDomdoc = XmlLoadDom（xctx，＆err，“file”，xslFile，“base_uri”，baseuri，NULL）;

  3. 为样式表创建一个XSLT处理器

    xslproc = XmlXslCreate（xctx，xslDomdoc，baseuri，＆err）;

  4. 解析实例XML文档。

    xmlDomdoc = XmlLoadDom（xctx，＆err，“file”，xmlFile，“base_uri”，baseuri，NULL）;

  5. 设置输出（可选）。默认为文档对象模型（DOM）。

    err = XmlXslSetOutputStream（xslproc，＆stream）;

  6. 转换XML文档。可以使用相同或其他XML文档重复此步骤。

    err = XmlXslProcess（xslproc，xmlDomdoc，FALSE）;

  7. 获取输出（如果是DOM）。

    node = XmlXslGetOutput（xslproc）;

  8. 删除对象。

    XmlXslDestroy（xslproc）; XmlDestroy（xctx）;

XPath处理器使用示例

描述了使用包API的典型方案。

跟着这些步骤：

  1. 创建和使用XML元上下文对象。

    xctx = XmlCreate（＆err，...）;

  2. 解析XML文档或从现有DOM获取当前节点。

    node = XmlLoadDom（xctx，＆err，“file”，xmlFile，“base_uri”，baseuri，NULL）;

  3. 创建一个XPath处理器。

    xptproc = XmlXPathCreateCtx（xctx，NULL，node，0，NULL）;

  4. 解析XPath表达式。

    exp = XmlXPathParse（xptproc，xpathexpr，＆err）;

  5. 评估XPath表达式。

    obj = XmlXPathEval（xptproc，exp，＆err）;

  6. 删除对象。

    XmlXPathDestroyCtx（xptproc）; XmlDestroy（xctx）;
