3主动检测和诊断Oracle RAC的性能问题

Oracle Cluster Health Advisor为系统和数据库管理员提供待处理性能问题的早期警告，以及Oracle RAC数据库和集群节点的根本原因和纠正措施。

使用Oracle Cluster Health Advisor可以提高可用性和性能管理。

Oracle Cluster Health Advisor根据默认模型估算观察输入的预期值，默认模型是基于目标系统正常运行周期的经过训练的校准模型。然后，Oracle Cluster Health Advisor根据观察值和预期值之间的差异对每个输入执行异常检测。如果与特定问题相关的足够输入异常，则Oracle Cluster Health Advisor会发出警告并生成即时针对性诊断和纠正措施。

Oracle Cluster Health Advisor在Grid Infrastructure Management Repository（GIMR）中存储分析结果，以及诊断信息，纠正措施和度量标准，以便以后进行分类。Oracle Cluster Health Advisor还使用Oracle Clusterware事件通知协议向Enterprise Manager Cloud Control发送警告消息。

  • Oracle Cluster Health Advisor体系结构
    Oracle Cluster Health Advisor在群集中的每个节点上作为高可用性群集资源（ ochad运行。
  • 使用Oracle Cluster Health Advisor监视Oracle Real Application Clusters（Oracle RAC）环境
    默认情况下，为Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库安装Oracle Grid Infrastructure时，将自动在每个节点上配置Oracle Cluster Health Advisor。
  • 使用Cluster Health Advisor进行健康诊断
    Oracle Cluster Health Advisor自动引发并清除问题，并将历史记录存储在Grid Infrastructure Management Repository（GIMR）中。
  • 为群集部署校准Oracle Cluster Health Advisor模型
    随着默认节点和数据库模型的提供，Oracle Cluster Health Advisor旨在不生成错误警告通知。
  • 查看Oracle Cluster Health Advisor模型的详细信息
    使用chactl query model命令查看模型详细信息。
  • 管理Oracle Cluster Health Advisor存储库
    Oracle Cluster Health Advisor存储库存储群集主机问题，数据库问题和关联度量标准的历史记录以及模型。
  • 查看群集运行状况顾问的状态
    SRVCTL命令是提供对作为高可用性服务管理Oracle Cluster Health Advisor生命周期的完全控制的工具。
