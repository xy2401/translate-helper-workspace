KFOD-00322：未指定凭证文件

原因：之前的'kfod'命令被拒绝，因为它没有包含有效的凭据文件名。

操作：重新执行提供有效凭据文件名的'kfod'命令。有关信息，请参阅命令行帮助。

KFOD-00323：凭证文件中的成员群集版本“ string ”与软件版本“ string ”不匹配。

原因：凭证文件中的成员群集版本与软件版本之间存在版本不匹配。

操作：安装与凭证文件中的版本匹配的成员集群版本。

KFOD-00324：成员群集版本“ string ”与存储服务器版本“ string ”不兼容。

原因：在消息中列出的版本配置的存储服务器不支持成员集群安装或升级。

操作：将成员集群安装或升级到与存储服务器兼容的版本。有关信息，请参阅产品文档。

KFOD-00325：无法验证成员群集' string '\ n string

原因：成员集群安装验证失败。可能的原因是： - 已配置具有相同名称的成员集群。- 已卸载具有相同名称的成员集群，但未在存储服务器上删除成员集群的配置。

操作：通过在存储服务器上执行“asmcmd lscc”命令，验证是否已存在消息中列出名称的成员集群。如果已配置并使用成员集群，请在存储服务器上执行“asmcmd mkcc”命令，并提供未使用的成员集群名称以生成新的凭据文件。使用新凭据文件重试Grid Infrastructure安装。如果已配置成员集群但未使用，请卸载现有成员集群。在存储服务器上执行'asmcmd rmcc'命令以删除成员群集配置。重试Grid Infrastructure安装。如果未配置成员集群或错误仍然存在，请与Oracle支持服务联系。

KFOD-00326：如果存储服务器升级为版本“ string ”，则成员集群“ string ”将不兼容。

原因：存储服务器升级到列出的版本失败，因为列出的成员群集将变得不兼容。

操作：将存储服务器升级到与所有已配置的成员集群兼容的版本。如果无法将存储服务器升级到兼容版本，请使用产品文档中记录的过程强制升级Storage Server。

KFOD-00330：指定了无效的群集名称“ string ”

原因：向'kfod'实用程序提供了无效的群集名称。

操作：指定一个至少包含一个字符但长度不超过15个字符的集群名称。群集名称必须是字母数字，不能以数字字符开头，并且可能包含连字符（ - ）字符。但是，它不能以连字符（ - ）结尾。

KFOD-00331：强制删除成员群集' string '

原因：强制删除邮件中列出的成员集群的配置。

操作：消息中列出的成员集群将无法再访问存储服务器。按照步骤启用成员群集的访问。- 删除所有磁盘组中“+ <diskgroup_name> / <client_cluster_name> /”目录中的所有Oracle Clusterware文件。- 卸载成员集群。- 安装成员集群。
