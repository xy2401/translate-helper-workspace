parallel_clause

目的

parallel_clause允许您并行化数据库对象的创建，并为对象的后续查询和DML操作设置默认的并行度。

您可以在以下语句中指定parallel_clause ：

  • CREATE TABLE ：为TABLE设置并行性（请参阅CREATE TABLE ）。

  • ALTER TABLE （参见ALTER TABLE ）：

    • 改变表的并行性

    • 并行化添加，合并，交换，合并，拆分，截断，删除或移动表分区的操作

  • CREATE CLUSTER和ALTER CLUSTER ：设置或更改集群的并行性（请参阅CREATE CLUSTER和ALTER CLUSTER ）。

  • CREATE INDEX ：为索引设置并行性（请参阅CREATE INDEX ）。

  • ALTER INDEX （参见ALTER INDEX ）：

    • 改变索引的并行性

    • 并行化重建索引或拆分索引分区

  • CREATE MATERIALIZED VIEW ：为物化视图设置并行度（请参阅CREATE MATERIALIZED VIEW ）。

  • ALTER MATERIALIZED VIEW （参见ALTER MATERIALIZED VIEW ）：

    • 更改物化视图的并行度

    • 并行化添加，合并，交换，合并，拆分，截断，删除或移动物化视图分区的操作

    • 并行化添加或移动物化视图子分区的操作

  • CREATE MATERIALIZED VIEW LOG ：为物化视图日志设置并行度（请参阅CREATE MATERIALIZED VIEW LOG ）。

  • ALTER MATERIALIZED VIEW LOG （参见ALTER MATERIALIZED VIEW LOG ）：

    • 更改实例化视图日志的并行度

    • 并行化添加，合并，交换，合并，拆分，截断，删除或移动物化视图日志分区的操作

  • ALTER DATABASE ...RECOVER ：恢复数据库（参见ALTER DATABASE ）。

  • ALTER DATABASE ... standby_database_clauses ：并行化备用数据库上的操作（请参阅ALTER DATABASE ）。

语义

本节描述parallel_clause的语义。有关其他信息，请参阅为特定数据库对象或操作设置或重置并行度的SQL语句。

注意：

parallel_clause的语法取代了早期版本的Oracle中出现的语法。为了向后兼容，仍然支持取代的语法，但可能会导致与记录的行为略有不同。

数据库根据PARALLEL_DEGREE_POLICY初始化参数的设置解释parallel_clause 。当该参数设置为AUTO ，将完全忽略parallel_clause ，并且优化程序确定所有语句的最佳并行度。当PARALLEL_DEGREE_POLICY设置为MANUAL或LIMITED ， parallel_clause解释如下：

NOPARALLEL

指定NOPARALLEL以进行串行执行。这是默认值。

平行

指定PARALLEL执行PARALLEL 。

  • 如果PARALLEL_DEGREE_POLICY设置为MANUAL ，则优化程序计算的并行度等于所有参与实例上可用的CPU数量乘以PARALLEL_THREADS_PER_CPU初始化参数的值。

  • 如果PARALLEL_DEGREE_POLICY设置为LIMITED ，则优化程序将确定最佳并行度。

PARALLEL 整数

integer规范表示并行度 ，它是并行操作中使用的并行线程数。每个并行线程可以使用一个或两个并行执行服务器。

关于parallel_clause的注释

以下说明适用于parallel_clause ：

  • 对已定义触发器或参照完整性约束的表上的DML操作禁用并行性。

  • 对索引组织表的UPDATE或DELETE操作不支持并行性。

  • 在创建表期间指定parallel_clause时，如果表包含LOB或用户定义的对象类型的任何列，则修改LOB或对象类型列的后续INSERT ， UPDATE ， DELETE或MERGE操作将在不通知的情况下串行执行。但是，后续查询将并行执行。

  • 并行提示覆盖的效果parallel_clause 。

  • DML语句和CREATE TABLE ...引用远程对象的AS SELECT语句可以并行运行。但是，远程对象必须位于远程数据库上。引用无法循环回本地数据库上的对象，例如，通过远程数据库上的同义词指向本地数据库上的对象。

  • 可以并行化对具有LOB列的表的DML操作。但是，不支持分区内并行性。
