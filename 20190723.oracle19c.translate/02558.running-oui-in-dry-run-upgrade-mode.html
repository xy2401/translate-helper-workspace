使用Oracle Universal Installer执行Dry-Run升级

在干运行升级模式下运行Oracle Grid Infrastructure安装程序以确定系统是否已准备好进行升级。

在干运行升级期间的任何时候，如果您对要求执行的操作或在干运行升级期间需要提供的输入有疑问，请单击安装程序页面上的“ 帮助”按钮。
在开始干运行升级之前，您应该可以使用网络信息，存储信息以及操作系统用户和组。
  1. 作为grid用户，下载Oracle Grid Infrastructure映像文件并将文件解压缩到Grid主目录。

    例如：

    $ mkdir -p /u01/app/19.0.0/grid $ chown grid：oinstall /u01/app/19.0.0/grid $ cd /u01/app/19.0.0/grid $ unzip -q download_location/grid_home.zip

    download_location/grid_home.zip是下载的Oracle Grid Infrastructure映像文件的路径。

    注意：

    您必须将图像文件解压缩到您希望Grid主页所在的目录中。

  2. 通过运行以下命令以干运行升级模式启动Oracle Grid Infrastructure安装向导：
    $ /u01/app/19.0.0/grid/gridSetup.sh -dryRunForUpgrade
  3. 选择“ 升级Oracle Grid Infrastructure”选项以执行Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）的干运行升级。
  4. 在“节点选择”页面上，选择所有节点。
  5. 根据提示选择安装选项。Oracle建议您配置root脚本自动化，以便在干运行升级期间自动运行rootupgrade.sh脚本。
  6. 自动或手动运行root脚本：
    • 自动运行根脚本：

      如果已配置root脚本自动化，则安装程序将在本地节点上自动运行rootupgrade.sh脚本。

    • 手动运行根脚本

      如果尚未配置root脚本自动化，则在出现提示时，在本地节点上运行rootupgrade.sh脚本。

    如果手动运行root脚本，则仅在本地节点上运行脚本。
  7. 检查gridSetupActions<timestamp>.log日志文件以查找错误并修复日志文件中报告的错误。
  8. 在Finish屏幕上退出安装程序并重新启动它，不带任何标志以启动实际升级。
