Oracle Database Vault环境中的12个 DBA操作

数据库管理员可以在Oracle Database Vault环境中执行操作，例如将Database Vault与Oracle Data Pump等产品一起使用。

  • 将Oracle Database Vault与Oracle Enterprise Manager配合使用
    Oracle Database Vault管理员可以在Oracle Enterprise Manager云控制中执行任务，例如将策略传播到其他数据库。
  • 将Oracle Data Pump与Oracle Database Vault配合使用
    数据库管理员可以授权Oracle Data Pump用户在Database Vault环境中工作。
  • 将Oracle Scheduler与Oracle Database Vault配合使用
    负责调度数据库作业的用户必须具有Oracle Database Vault特定的授权。
  • 将信息生命周期管理与Oracle Database Vault结合使用
    必须为在启用Oracle Database Vault的数据库上执行信息生命周期管理操作的用户授予执行这些操作的权限。
  • 在Oracle Database Vault中使用Oracle数据库重放
    数据库管理员可以授权Oracle Database Replay用户在Database Vault环境中工作。
  • 使用Oracle Database Vault执行预处理程序
    通过外部表执行预处理程序的用户必须具有Oracle Database Vault特定的授权。
  • 使用Database Vault Operations Control限制多租户公共用户对本地PDB数据的访问
    您可以通过CDB根公共用户（例如基础结构数据库管理员）控制PDB访问。
  • Oracle Recovery Manager和Oracle Database Vault
    您可以在Oracle Database Vault环境中使用Recovery Manager（RMAN）。
  • 将XStream与Oracle Database Vault一起使用的权限
    如果要在Oracle Database Vault环境中使用XStream，则必须具有相应的权限。
  • 将Oracle GoldenGate与Oracle Database Vault配合使用的权限
    如果要在Oracle Database Vault环境中使用Oracle GoldenGate，则必须具有相应的权限。
  • 在Oracle Database Vault环境中使用数据屏蔽
    您必须具有在Oracle Database Vault环境中执行数据屏蔽的正确授权。
  • 将独立Oracle数据库转换为PDB并将其插入CDB
    您可以将独立的Oracle Database Release 12c或更高版本的数据库转换为PDB，然后将此PDB插入CDB。
  • 将ORADEBUG实用程序与Oracle Database Vault一起使用
    ORADEBUG实用程序主要由Oracle Support使用，用于诊断Oracle数据库可能出现的问题。

将Oracle Database Vault配置传播到其他数据库

您可以将Database Vault配置（例如域配置）传播到其他受Database Vault保护的数据库。

  1. 以已授予DV_OWNER或DV_ADMIN角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。

  2. 在Database Vault主页的Database Vault Policy Propagation下，选择Database Vault Policy Propagation 。

    “策略传播”子页面中的“可用策略”区域列出了为当前数据库创建的Oracle Database Vault配置的摘要：即为域，命令规则，规则集和安全应用程序角色创建的配置。它没有列出Oracle数据库版本12个C（12.2）分别介绍了Oracle Database Vault的政策。从这里，您可以将这些配置传播到另一个数据库。

  3. 在“可用策略”下，选择要传播到另一个数据库的每个配置。



    说明policy_propagation122.png的说明

  4. 在“目标数据库”下，单击“ 添加”按钮。

  5. 在“搜索并选择：数据库保管库启用的目标数据库”下，搜索目标数据库，然后选择要将传播传播到的每个数据库。然后单击“ 选择”按钮。

  6. 在“目标数据库”下，执行以下操作：

    1. 在“跨目标数据库应用凭据”下，输入包含要传播的配置的Database Vault数据库管理员的用户名和密码。

      此功能将Database Vault管理员的用户名和密码应用于所有选定的目标数据库。

    2. 选择要将传播传播到的每个数据库。

    3. 输入每个数据库的Database Vault管理员用户名和密码。

    4. 单击“ 应用”按钮。

  7. 在“传播选项”页面中，从以下选项中进行选择。

    对种子域，命令规则，规则集等所做的任何更改都不会传播到目标数据库。仅传播自定义创建的数据。

    • 故障时恢复：如果传播操作遇到错误，则回传传播。也就是说，将恢复目标数据库上的原始策略。如果未选择此选项，则目标数据库上的策略传播将继续，并忽略任何错误。

    • 如果存在用户定义的策略，则跳过传播：如果目标数据库已具有用户定义的配置，则不尝试传播操作。如果不选择此选项，则无论目标数据库上是否存在用户定义的策略，都会清除所有现有配置，并将源数据库中的配置应用于目标数据库。

    • 传播数据库保管库度量标准的企业管理器度量标准阈值：如果源数据库设置了Oracle数据库保管库度量标准阈值，则这些阈值也会传播到目标数据库。如果未选择此选项，则仅传播配置，而不传播Oracle Database Vault阈值。

  8. 单击“ 确定”按钮。

  9. 在“确认”窗口中，单击“ 确定” 。

    出现指示成功或失败的消息。如果传播成功，则配置立即在其目标数据库中处于活动状态。

Oracle Database Vault策略的企业管理器云控制警报

要查看Oracle Database Vault警报，您必须被授予DV_OWNER ， DV_ADMIN或DV_SECANALYST角色。

警报如下：

  • Database Vault尝试了Realm Violations。此警报可帮助Oracle Database Vault安全分析师（ DV_SECANALYST角色）监视Database Vault数据库上的违规尝试。此用户可以选择受警报影响的领域，并根据使用错误代码的不同尝试类型过滤这些领域。您可以从“度量标准和策略设置”页面启用此度量标准。默认情况下，每24小时收集一次尝试的域违规。

  • Database Vault尝试命令规则违规。 此警报的功能与Database Vault Attempted Realm Violations的功能相同，只是它侧重于对命令规则的违规。

  • Database Vault领域配置问题。 如果用户配置错误，此度量标准会跟踪并发出警报。安装Oracle数据库保管库时会启用此度量标准，默认情况下，它每小时收集一次数据。

  • Database Vault命令规则配置问题。 此警报的此功能与Database Vault领域配置问题相同，只是它侧重于对命令规则的配置更改。

  • Database Vault策略更改。 此度量标准会针对任何Database Vault策略的任何更改发出警报，例如领域和命令规则的策略。它提供了详细的政策变更报告。

为Database Vault中的Oracle Data Pump常规操作授权用户或角色

您可以授权数据库管理员或角色将Data Pump用于Oracle Database Vault环境中的常规操作。

  1. 以已被授予DV_OWNER或DV_ADMIN角色的用户身份登录数据库实例。
  2. 确保已授予您要授予其权限的用户或角色的EXP_FULL_DATABASE和IMP_FULL_DATABASE角色是使用Oracle Data Pump所必需的。
    SELECT GRANTEE，GRANTED_ROLE来自DBA_ROLE_PRIVS，其中GRANTED_ROLE类似'％FULL％';
  3. 为Oracle Data Pump常规操作授予此用户或角色Oracle Database Vault授权。

    例如，要授权Data Pump用户DP_MGR导出和导入数据库表EMPLOYEES对象：

    EXEC DBMS_MACADM.AUTHORIZE_DATAPUMP_USER（'DP_MGR'，'HR'，'EMPLOYEES'）;

    要将DP_MGR的活动限制为特定模式，请输入以下过程：

    EXEC DBMS_MACADM.AUTHORIZE_DATAPUMP_USER（'DP_MGR'，'HR'）;

    要授权已授予DP_MGR_ROLE角色的用户导出和导入整个数据库的对象，请输入以下内容：

    EXEC DBMS_MACADM.AUTHORIZE_DATAPUMP_USER（'DP_MGR_ROLE'）;

    运行DBMS_MACADM.AUTHORIZE_DATAPUMP_USER过程后，可以通过查询DBA_DV_DATAPUMP_AUTH数据字典视图来检查用户或角色的授权。

  4. 如果用户必须导出整个数据库，则授予用户（或角色） DV_OWNER角色。
    例如，对于角色：
    GRANT DV_OWNER到DP_MGR_ROLE;

从用户或角色撤销Oracle数据泵授权

您可以撤消使用Oracle Data Pump进行常规操作的数据库管理员或角色的授权。

  1. 如果您授予用户或角色DV_OWNER角色，则可以选择撤消DV_OWNER角色。
    REVOKE DV_OWNER来自DP_MGR;
  2. 查询DBA_DV_DATAPUMP_AUTH数据字典视图以查找已授予Oracle Data Pump授权的用户或角色。
    从GRA_DV_DATAPUMP_AUTH中选择GRANTEE，SCHEMA，OBJECT;
  3. 使用从上一步骤收集的信息来构建DBMS_MACADM.UNAUTHORIZE_DATAPUMP_USER命令。

    例如：

    EXEC DBMS_MACADM.UNAUTHORIZE_DATAPUMP_USER（'DP_MGR'，'HR'，'EMPLOYEES'）;

    确保此未授权补充原始授权操作。换句话说，如果您最初对整个数据库授予了DP_MGR授权，则以下命令将不起作用：

关于使用Database Vault Operations Control

您可以自动限制普通用户访问自治，常规云或内部部署环境中的可插拔数据库（PDB）本地数据。

要实现此目的，您可以使用Oracle Database Vault操作控制，该控制适用于普通用户（例如基础架构数据库管理员和应用程序）。

Database Vault操作控制对于数据库管理员必须以高权限用户身份登录CDB根目录但仍无法访问PDB客户数据的情况非常有用。数据库操作控制不会阻止PDB数据库管理员。要阻止这些用户，请在PDB中启用Oracle Database Vault，然后使用域库控制等数据库保险库功能来阻止这些用户。

对于公共用户或应用程序必须执行必须访问PDB上的本地数据的任务的情况，您可以为公共用户和程序包的Database Vault操作控制创建例外列表。您要为例外列表指定的普通用户类型的示例是CTXSYS应用程序帐户，该帐户负责Oracle Text。通过在例外列表中指定包，您可以应用更细粒度的控件，而不是在例外列表中提供对用户的完全访问权限。

使用Database Vault操作控制的一般过程如下：

  1. 启用Database Vault操作控制并为生产环境启用它。
  2. 在此阶段，无论PDB是否已启用Database Vault，Database Vault操作控制都适用于环境中的所有PDB。
  3. 要使特定用户和程序包能够访问PDB的本地模式，请将它们添加到例外列表中。当用户或包不再需要访问权限时，您可以从例外列表中删除它们。例如，如果数据库使用的是Oracle Text，则可以将CTXSYS管理用户帐户和程序包添加到例外列表中。

创建命令规则以控制数据屏蔽特权

在Oracle Database Vault环境中使用数据屏蔽之前，您必须具有管理表，包和触发器的权限。

对于数据屏蔽，用户必须具有屏蔽对象的CREATE TABLE ， SELECT TABLE ， ALTER TABLE和DROP TABLE权限，并且如果要创建任何依赖对象，则用户必须具有相应的权限，例如CREATE PACKAGE ， CREATE TRIGGER ， 等等。

您可以创建命令规则以在粒度级别控制数据屏蔽权限。为此，请创建一个命令规则，该规则可以阻止或允许用户访问必须被数据屏蔽的对象。例如，您可以创建一个名为“允许数据屏蔽”的命令规则，该规则检查用户是否在负责数据屏蔽的用户列表中。如果登录的用户是这些用户之一，则命令规则的计算结果为true，并且允许用户为受保护对象创建数据掩码。

要创建控制数据屏蔽权限的命令规则：

  1. 创建规则集规则。

    例如：

    BEGIN DBMS_MACADM.CREATE_RULE（rule_name =>'是HDRISCOLL或DBA_JSMITH用户'，rule_expr =>'USER IN（'''HDRISCOLL''，''DBA_JSMITH''）'; END; /
  2. 创建规则集，然后将规则添加到其中：
    BEGIN DBMS_MACADM.CREATE_RULE_SET（rule_set_name =>'允许数据屏蔽'，description =>'允许用户访问HDRISCOLL和DBA_JSMITH'，启用=>'Y'，eval_options => 1，audit_options => 1，fail_options => 1，fail_message = >'您无权访问此对象。'，fail_code => 20461，handler_options => 0，is_static => TRUE）;结束; / BEGIN DBMS_MACADM.ADD_RULE_TO_RULE_SET（rule_set_name =>'允许数据屏蔽'，rule_name =>'是HDRISCOLL或DBA_JSMITH用户'），rule_order => 1）;结束; /
  3. 创建命令规则，然后将此规则添加到其中：
    BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'CREATE TABLE'，rule_set_name =>'允许数据屏蔽'，object_owner =>'HR'，object_name =>'EMPLOYEES'，enabled => DBMS_MACUTL.G_YES）;结束; /

将独立Oracle数据库转换为PDB并将其插入CDB

您可以将独立的Oracle Database Release 12c或更高版本的数据库转换为PDB，然后将此PDB插入CDB。

  1. 以已被授予DV_OWNER角色的用户身份连接到root。

    例如：

    sqlplus c ## sec_admin输入密码： 密码
  2. 使用CONTAINER = CURRENT将DV_PATCH_ADMIN角色授予用户SYS 。
    GRANT DV_PATCH_ADMIN TO SYS CONTAINER = CURRENT;
  3. 在根目录中，以SYS用户身份连接SYSOPER系统特权。

    例如：

    将SYS作为SYSOPER连接- 或者，连接SYS @hrpdb AS SYSOPER输入密码： password
  4. 以只读模式重新启动数据库。

    例如：

    SHUTDOWN立即启动安装ALTER DATABASE OPEN READ ONLY
  5. 以具有DV_OWNER角色的用户身份连接到启用Database Vault的数据库。

    例如：

    CONNECT sec_admin @ dv_db
  6. 将DV_PATCH_ADMIN角色授予此数据库上的用户SYS 。
    GRANT DV_PATCH_ADMIN TO SYS;
  7. （可选）运行DBMS_PDB.CHECK_PLUG_COMPATIBILITY函数以确定未插入的PDB是否与CDB兼容。

    运行该功能时，请设置以下参数：

    • pdb_descr_file ：将此参数设置为包含PDB描述的XML文件的完整路径。

    • store_report ：设置此参数以指示在PDB与CDB不兼容时是否要生成报告。将其设置为TRUE以生成报告，或将其设置为FALSE以不生成报告。生成的报告存储在PDB_PLUG_IN_VIOLATIONS临时表中，仅在PDB与CDB不兼容时生成。

    例如，要确定/disk1/usr/dv_db_pdb.xml文件描述的PDB是否与当前CDB兼容，请运行以下PL / SQL块：

    SET SERVEROUTPUT ON DECLARE兼容的CONSTANT VARCHAR2（3）：= CASE DBMS_PDB.CHECK_PLUG_COMPATIBILITY（pdb_descr_file =>'/disk1 / usr / dv_db_pdb.xml'，store_report => TRUE）当为“是”时，“否”结束; BEGIN DBMS_OUTPUT.PUT_LINE（兼容）;结束; /

    如果输出为YES ，则PDB兼容，您可以继续下一步。

    如果输出为NO ，则PDB不兼容。您可以检查PDB_PLUG_IN_VIOLATIONS临时表以查看它不兼容的原因。

  8. 创建描述PDB的XML文件。

    例如：

    BEGIN DBMS_PDB.DESCRIBE（pdb_descr_file =>'/disk1 / oracle / dv_db.xml'）;结束; /
  9. 运行CREATE PLUGGABLE DATABASE语句，并在USING子句中指定XML文件。在需要时指定其他子句。

    例如：

    使用'dv_db.xml'NOCOPY创建可插入数据库dv_db_pdb AS CLONE;
  10. 使用SYSDBA管理权限连接到您刚刚以SYS用户身份创建的PDB。
    CONNECT SYS @ dv_db_pdb AS SYSDBA
  11. 执行noncdb_to_pdb.sql脚本。
    @ $ ORACLE_HOME / RDBMS /管理/ noncdb_to_pdb.sql
  12. 以读/写限制模式打开此PDB。
    ALTER PLUGGABLE DATABASE dv_db_pdb OPEN READ WRITE RESTRICTED;
  13. 运行以下过程以同步PDB：
    EXECUTE DBMS_PDB.SYNC_PDB;
  14. 以已被授予DV_OWNER角色的用户身份连接到root。
    sqlplus c ## sec_admin输入密码： 密码
  15. 使用CONTAINER = CURRENT从用户SYS撤消DV_PATCH_ADMIN角色。
    从SYS CONTAINER RETROKE DV_PATCH_ADMIN = CURRENT;
  16. 使用具有SYSOPER系统特权的用户SYS连接到启用Database Vault的旧数据库。
    作为SYSOPER连接SYS @ dv_db_pdb
  17. 重启此数据库。

    例如：

    关闭IMIMEDIATE STARUP
  18. 从用户SYS撤消DV_PATCH_ADMIN角色。
    从SYS中撤消DV_PATCH_ADMIN;
