使用DBCA选择要与任何Oracle RAC数据库一起使用的存储

您可以选择使用Oracle ASM磁盘组或支持的群集文件系统作为存储。

在“指定数据库存储选项”页上，如果在DBCA中未看到磁盘组，则表示未配置Oracle ASM，或未装入磁盘组。在启动DBCA之前，您可以在Grid Infrastructure主目录中使用ASMCA创建磁盘组。

如果您使用的是Oracle ASM或群集文件系统存储，则还可以在此页面上选择快速恢复区域和大小。如果您使用的是Oracle ASM，则快速恢复区默认为Oracle ASM磁盘组。
