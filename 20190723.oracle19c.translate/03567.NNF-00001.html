NNF-00001：不是TNS地址

原因：在要求读取地址时，名称服务返回的记录不是地址。

操作：通常对用户不可见。有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-00002：传递给函数调用的值无效

原因：传递给接口函数的值无效。

操作：通常对用户不可见。有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-00003：命名适配器string不存在

原因：在当前适配器表中找不到请求的名称服务适配器。

操作：检查所需的适配器的拼写。验证适配器是否在此平台上运行，并将适配器编译到链接到可执行文件的适配器表中。

NNF-00004： string不支持操作

原因：此适配器不支持请求的操作。

操作：验证正在执行的操作是否受限制，例如写入只读名称服务。有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-00005：功能未实现

原因：此实现不支持请求的操作。

操作：如果从生产版本中检索到此信息，请联系全球客户支持。否则，请查看有关此版本中的限制的发行说明。

NNF-00110：不存在任何值

原因：测试程序帮助文本。

行动：没有

NNF-00540：无法插入名称： string

原因：无法将名称插入本机服务。

操作：验证名称是否已存在于本机命名空间中，并为命名空间设置了相应的访问权限。

NNF-01001：ds_initialize：工作空间初始化失败

原因： ds_initialize（）设置的DCE工作空间无法初始化。

操作：验证此平台上是否存在成功进行ds_initialize（）调用所需的条件（例如，是否有足够的内存？）。检查此程序中是否没有其他工作区用户。有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-01002：DCE无法打开配置文件

原因： DCE调用返回错误，无法访问其配置文件。

操作：检查此平台的DCE配置文件是否存在且可访问。在Unix上，这个文件是dce_cf.db。检查对dce_cf_get_cell_name（）的简单调用是否成功。

NNF-01003：DCE找不到单元名称

原因： DCE调用返回错误，指出它在配置文件中找不到单元名称。

操作：检查是否已为DCE单元配置了本机。Chcek表示DCE配置文件没有损坏。在Unix上，此文件是dce_cf.db。检查对dce_cf_get_cell_name（）的简单调用是否成功。

NNF-01004：DCE返回未指定的错误string

原因： DCE调用返回的状态不在此实现的此DCE调用的可能状态值列表中。

操作：验证此平台的DCE版本是否超出此Oracle应用程序支持的版本。如果错误仍然存在，请联系全球客户支持。

NNF-01005：内部XOM类违规

原因： DCE XOM调用返回的状态与DCE文档中的类定义不一致。

操作：通常对用户不可见。有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-01006：属性string不可用

原因：插入调用尝试使用没有转换为DCE属性的属性。

操作：使用其他属性，或将转换插入DCE。

NNF-01007：不支持DCE长字符串

原因：查询返回了一个具有XOM OM_S_LONG_STRING的对象。当前实现不支持此类对象。

操作：有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-01008：Oracle名称语法string XDS属性语法string不正确

原因： XDS对象中指定的语法与给定Oracle名称语法的语法不同。

操作：有关更多详细信息，请启用跟踪并重新执行失败操作。如果错误仍然存在，请联系全球客户支持。

NNF-02001：NIS客户端错误字符串： string

原因：对NIS（YP）功能的调用失败。此消息包含返回的NIS错误代码的文本。

操作：仅在作为另一个错误的一部分返回时才可见。如果错误仍然存在，请联系全球客户支持。

NNF-02002：NIS值不符合适配器架构： string

原因： NIS适配器使用的映射中的值必须符合一组特定的内部格式设置规则。返回的值不符合这些规则。

操作：除非您尝试创建自己的地图，否则不应显示。如果错误仍然存在，请联系全球客户支持。

NNF-02003：未知的NIS错误代码： string

原因： NIS（YP）函数返回的错误代码不在此程序预期的列表中。

操作：验证此平台上的YP功能是否生成错误。通知全球客户支持该错误。

NNF-02004：NIS映射string不存在

原因：请求的属性不存在。

操作：对于未使用的属性，不需要任何操作，因为这是预期的错误。否则，添加属性并重新加载NIS（YP）贴图。

NNF-02005：NIS钥匙string中不存在地图string

原因：找不到请求的名称。

操作：对于未使用的名称，不需要任何操作，因为这是预期的错误。否则，将名称添加到地图并重新加载NIS（YP）地图。

NNF-03002：未知BIND错误代码： string

原因： BIND（DNS）查询响应中的返回码不符合RFC 1035中列出的值。

操作：验证此平台上的BIND函数是否生成错误。通知全球客户支持该错误。

NNF-03003：解析服务器响应的格式错误

原因：从BIND（DNS）API返回的查询响应数据包不符合RFC 1035中列出的格式。

操作：验证此平台上的BIND函数是否生成错误。通知全球客户支持该错误。

NNF-03004：DNS类不匹配（ string ）

原因：查询响应数据包返回的类与接口预期的类不同。

操作：验证此平台上的BIND函数是否生成错误。通知全球客户支持该错误。

NNF-04000：文件已成功转换

原因：执行请求的转换。

行动：正常情况，无需采取任何行动。

NNF-04001：转换输入文件时出错

原因：输入文件包含转换程序无法转换的行。

操作：检查输入文件的语法是否正确。

NNF-04002：Outputfile已存在

原因：当前目录中已存在旧的输出文件

操作：删除现有文件，然后重新启动转换工具

NNF-04003：输入文件中的条目太长

原因： makedbm程序将mapfiles中的条目限制为1017个字符。因此，转换工具不会处理长度超过1017个字符的TNS地址。

行动：缩短违规行为。

NNF-04004：inputfile中的字符串无效

原因：输入文件包含无效行。

操作：检查输入文件的语法是否正确。

NNF-04005：无法打开inputfile

原因：找不到指定的输入文件。

操作：使用正确的输入文件重新启动转换工具。

NNF-04006：无法创建输出文件

原因：无法在当前工作目录中创建输出文件

操作：更改当前目录的权限并重新启动转换工具。

NNF-04007：读取输入文件时出错

原因：读取输入文件时出现问题。这可能是由输入文件中意外的文件结束引起的。

操作：检查文件是否完整以及输入文件的语法是否正确。

NNF-04008：写入outputfile时出错

原因：写入输出文件时发生错误。

操作：检查可用磁盘空间。如果错误仍然存在，请联系全球客户支持。

NNF-04009：用法：tns2nis filename

原因：未正确调用转换工具。文件名应该是tnsnames.ora或native.ora

操作：使用正确的参数重新启动转换工具。
