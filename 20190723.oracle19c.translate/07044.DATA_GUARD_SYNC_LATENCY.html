1.64 DATA_GUARD_SYNC_LATENCY

DATA_GUARD_SYNC_LATENCY控制Log Writer（LGWR）进程在一系列Oracle Data Guard SYNC重做传输模式连接中等待第一个响应之后的秒数。

属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

没有

价值范围

0到LOG_ARCHIVE_DEST_n参数的NET_TIMEOUT属性指定的NET_TIMEOUT

基本

没有

Oracle RAC

必须在所有实例上使用相同的值。

默认值为0 ，这意味着LGWR将等待每个SYNC备用目标的LOG_ARCHIVE_DEST_n参数的NET_TIMEOUT属性指定的NET_TIMEOUT 。

例如，在具有两个在SYNC模式下接收重做的备用数据库的Oracle Data Guard配置中，您将定义LOG_ARCHIVE_DEST_2和LOG_ARCHIVE_DEST_3以使用SYNC传输， DATA_GUARD_SYNC_LATENCY设置为2秒的值。当需要将重做发送到备用数据库时，它将并行发送到LOG_ARCHIVE_DEST_2和LOG_ARCHIVE_DEST_3连接，并且LGWR将等待确认。如果LOG_ARCHIVE_DEST_2以其确认首先收到重做的消息进行响应，则LGWR将不会等待LOG_ARCHIVE_DEST_3响应超过2秒（ DATA_GUARD_SYNC_LATENCY ）。如果LOG_ARCHIVE_DEST_3在这2秒内没有响应，则LGWR与LOG_ARCHIVE_DEST_3断开连接，使其进入错误状态。重做继续像往常一样发送到LOG_ARCHIVE_DEST_2 。在LOG_ARCHIVE_DEST_3的REOPEN属性的持续时间过去之后，LGWR重新连接到LOG_ARCHIVE_DEST_3并继续重做传输。通过这种方式，具有第二个SYNC备用数据库的最大影响将比仅有一个SYNC备用数据库限制为2秒，同时允许Oracle Data Guard配置保持所需的保护级别。

无论DATA_GUARD_SYNC_LATENCY的值如何，LGWR永远不会等待LOG_ARCHIVE_DEST_n参数的NET_TIMEOUT属性的值。
