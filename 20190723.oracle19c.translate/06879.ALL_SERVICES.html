3.72 ALL_SERVICES

ALL_SERVICES显示数据库中的所有服务。该视图不包括标记为删除的行。

相关视图

DBA_SERVICES显示数据库中的所有服务。该视图不包括标记为删除的行。

柱 数据类型 空值 描述

SERVICE_ID

数

该服务的唯一ID

名称

VARCHAR2（64）

描述工作量的名称

NAME_HASH

数

散列服务的简称

网络名字

VARCHAR2（512）

用于连接服务的网络名称

创立日期

日期

服务创建日期

CREATION_DATE_HASH

数

哈希的创作日期

FAILOVER_METHOD

VARCHAR2（64）

TAF仅用于兼容性 - BASIC或NONE

FAILOVER_TYPE

VARCHAR2（64）

对于Java的应用程序连续性， TRANSACTION 。对于TAF， SESSION或SELECT 。

FAILOVER_RETRIES

NUMBER（10）

对于应用程序连续性和TAF，在故障后重新连接时，每次事件重新连接的次数

FAILOVER_DELAY

NUMBER（10）

对于应用程序连续性和TAF，在故障后重新连接时，每次连接重试之间的延迟（以秒为单位）

MIN_CARDINALITY

数

保留供内部使用

MAX_CARDINALITY

数

保留供内部使用

目标

VARCHAR2（12）

运行时负载平衡目标用于创建运行时负载平衡和连接负载平衡建议：

  • 没有

  • SERVICE_TIME - 连接由响应时间平衡

  • THROUGHPUT -连接是通过吞吐平衡

DTP

VARCHAR2（1）

DTP（分布式事务处理）在一个实例上强制执行服务的所有会话。这是11g之前XA的要求，并且是恢复和挂起相同XA分支的要求。

启用

VARCHAR2（3）

保留供内部使用

AQ_HA_NOTIFICATIONS

VARCHAR2（3）

要为OCI连接启用FAN，请将AQ HA通知设置为True 。对于Oracle Database 12 c ，FAN使用ONS（Oracle Notification Service）

CLB_GOAL

VARCHAR2（5）

连接负载平衡目标。使用运行时负载平衡时， GOAL=SERVICE_TIME或THROUGHPUT ，设置为SHORT 。对于每个服务的会话计数，请使用LONG 。

版

VARCHAR2（128）

非NULL值指定使用该服务的后续数据库连接的初始会话版本，但不指定版本。NULL值无效。

COMMIT_OUTCOME

VARCHAR2（3）

此属性用于Transaction Guard。当COMMIT_OUTCOME设置为TRUE ，Transaction Guard将管理所有支持的事务类型的提交状态。该属性基于每个会话，并在连接时设置。

启用该属性时：

  • COMMIT事务的结果是已知的。如果发生中断，应用程序可以使用DBMS_APP_CONT.GET_LTXID_OUTCOME返回上次正在进行的工作的可靠状态。

  • 在登录和每次成功提交时为每个用户会话设置逻辑事务ID（LTXID）。

另请参阅：有关保留提交结果的信息，请参阅“ Oracle数据库开发指南” 。有关逻辑事务ID的信息，请参阅“ Oracle数据库开发指南” 。

RETENTION_TIMEOUT

数

对于Transaction Guard（ COMMIT_OUTCOME设置为TRUE ），此参数确定在数据库中保留提交结果的时间（以秒为单位）

REPLAY_INITIATION_TIMEOUT

数

对于应用程序连续性，指示不会重播请求的时间段（以秒为单位）。时间段从第一次提交请求开始。

默认值为300秒，即5分钟。

SESSION_STATE_CONSISTENCY

VARCHAR2（128）

描述在请求期间如何更改非事务性。仅当failover_type设置为TRANSACTION for Application Continuity时才考虑此参数。会话状态的示例包括NLS设置，优化程序首选项，事件设置，PL / SQL全局变量，临时表，高级队列，LOB和结果缓存。如果请求开始后非事务性值发生更改，则应设置DYNAMIC的默认值。几乎所有应用程序都应使用DYNAMIC模式。如果您不确定，请使用DYNAMIC模式。

GLOBAL_SERVICE

VARCHAR2（3）

指示服务是否为全局服务。全局服务由全局数据服务（GDS）管理，可由包含复制数据的多个数据库提供。可能的值：

  • YES ：表示服务是全局的

  • NO ：表示服务不是全局的

使用GDS时设置此属性。它不能由用户设置。

PDB

VARCHAR2（128）

与给定服务关联的PDB的名称。对于非CDB将包含NULL，或者如果服务未与PDB关联（即，使用此服务连接到CDB将导致用户连接到根。）

管理PDB的服务时，请使用SRVCTL for Oracle RAC和Oracle RAC One Node数据库，如果它是单个实例（不是RAC），则连接到该PDB。PDB属性显示哪个PDB具有该服务。它无法明确设置或修改。

SQL_TRANSLATION_PROFILE

VARCHAR2（261）

非NULL值指定使用该服务的后续数据库连接的初始SQL转换配置文件，并且不指定SQL转换配置文件。NULL值无效。

MAX_LAG_TIME

VARCHAR2（128）

用于提供数据库服务的数据副本可接受的最大复制滞后（以秒为单位）。只能使用全局数据服务（GDS）接口为全局服务指定。不支持在本地数据库中更改此值。

GSM_FLAGS

数

特定于全球数据服务（GDS）的标志。只能使用GDS接口为全局服务指定。不支持在本地数据库中更改这些值。

PQ_SVC

VARCHAR2（64）

关联的并行查询边缘服务的名称

STOP_OPTION

VARCHAR2（13）

停止此服务会话的选项以进行计划维护

FAILOVER_RESTORE

VARCHAR2（6）

指示会话在使用TAF进行故障转移时是否恢复其常用会话状态（如NLS，架构）

DRAIN_TIMEOUT

数

等待会话耗尽的秒数

TABLE_FAMILY_ID

数

与服务关联的分片表系列ID
