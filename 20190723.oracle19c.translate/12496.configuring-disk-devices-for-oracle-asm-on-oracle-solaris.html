在Oracle Solaris上为Oracle ASM配置磁盘设备

完成这些任务以配置磁盘设备以与Oracle自动存储管理（Oracle ASM）配合使用。

  1. 如有必要，请安装要用于磁盘组的磁盘，然后重新启动系统。
  2. 识别属于逻辑卷管理器（LVM）磁盘组的设备：
    此命令显示有关VERITAS Volume Manager磁盘的信息。如果您使用其他LVM，请参阅相应的文档以获取有关确定其管理的磁盘设备的信息。
    #vxdiskconfig＃/ usr / sbin / vxdisk list
    如果此命令显示与磁盘设备关联的磁盘组信息，则该磁盘已经是LVM磁盘组的一部分。不要使用属于LVM磁盘组的磁盘。
  3. 创建或标识要包含在Oracle自动存储管理磁盘组中的磁盘片（分区）：

    1. 列出连接到系统的磁盘：

      ＃/ usr / sbin / format

      此命令的输出类似于以下内容：

      可用的磁盘选择：可用的磁盘选择：0。c0t0d0 <ST34321A cyl 8892 alt 2 hd 15 sec 63> / pci @ 1f，0 / pci @ 1,1 / ide @ 3 / dad @ 0,0 1. c1t5d0 <SUN9。 0G cyl 4924 alt 2 hd 27 sec 133> / pci @ 1f，0 / pci @ 1 / scsi @ 1 / sd @ 5,0

      此命令显示有关连接到系统的每个磁盘的信息，包括设备名称。例如， cxtydz 。

    2. 输入与要使用的磁盘对应的编号。

    3. 使用fdisk命令在磁盘上创建Oracle Solaris分区（如果尚不存在）。

      Oracle Solaris fdisk分区必须从柱面1开始，而不是从柱面0开始。如果创建fdisk分区，则必须在继续之前标记磁盘。

    4. 输入partition命令，然后输入print命令以显示要使用的磁盘的分区表。

    5. 如有必要，从柱面1开始创建单个整盘切片。

      注意：

      要防止自动存储管理覆盖分区表，您不能使用从柱面0开始的切片（例如，切片2）。

    6. 记下要使用的切片编号。

    7. 如果修改了分区表或创建了新分区表，则输入label命令将分区表和标签写入磁盘。

    8. 输入q返回格式菜单。

    9. 如果已完成创建切片，则输入q以退出format实用程序。否则，输入disk命令选择新磁盘，并重复步骤b到g以创建或识别该磁盘上的片。

  4. 如果您计划使用现有切片，请验证它们未作为文件系统挂载：
    #df -h / tmp
    此命令显示有关作为文件系统挂载的磁盘设备上的片的信息。切片的设备名称包括磁盘设备名称后跟切片编号，例如cxtydzsn ，其中sn是切片编号。
  5. 在每个节点上，更改要添加到磁盘组的每个磁盘片的文件的所有者，组和权限：
    #chown grid：asmadmin / dev / rdsk / cxtydzs6＃chmod 660 / dev / rdsk / cxtydzs6

    在此示例中，设备名称指定切片6。

    注意：

    如果在Oracle自动存储管理中使用多路径磁盘驱动程序，请确保仅在磁盘的正确逻辑设备名称上设置权限。
