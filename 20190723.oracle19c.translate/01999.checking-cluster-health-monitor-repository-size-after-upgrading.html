升级后检查群集运行状况监视器存储库大小

如果要将Oracle Grid Infrastructure从使用IPD / OS的先前版本升级到当前版本，请查看Cluster Health Monitor存储库大小（CHM存储库）。

  1. 检查您的CHM存储库需求，并确定是否需要增加存储库大小以维护更大的CHM存储库。

    注意：

    安装Oracle Grid Infrastructure时，将删除以前的IPD / OS存储库。

    默认情况下，无论群集的大小如何，CHM存储库大小都至少为1GB或3600秒（1小时）。

  2. 要扩大CHM存储库，请使用以下命令语法，其中RETENTION_TIME是CHM存储库的大小，以秒为单位：
    oclumon manage -repos changeretentiontime RETENTION_TIME

    例如，要将存储库大小设置为四小时：

    oclumon manage -repos changeretentiontime 14400
    RETENTION_TIME的值必须大于3600（一小时）且小于259200（三天）。如果放大CHM存储库大小，则必须确保在群集的每个节点上选择的存储库大小都有可用的本地空间。如果没有足够的可用空间，则可以将存储库移动到共享存储。
