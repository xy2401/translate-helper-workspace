1.191 MAX_MBPS

MAX_MBPS使您可以设置每个可插拔数据库（PDB）每秒发出的最大兆字节数（MB）。 此参数用于限制PDB I / O.
属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

是

价值范围

0到最大Integer数值。不建议使用非常低的值（例如，低于每秒25 MB）。

基本

没有

Oracle RAC

可以在不同的实例上设置不同的值。

DBWR I / O，控制文件I / O，密码文件I / O和其他关键I / O不受此参数设置的吞吐量限制的限制，但在限制时会考虑它们的I / O.由于这些豁免，PDB的实际I / O速率有时可能超过限制。

仅为多租户容器数据库（CDB）启用此功能。Oracle Exadata不支持该功能。

可以从PDB内部设置此参数。如果参数在CDB $ ROOT中设置，则该CDB中的所有PDB将从CDB $ ROOT继承参数值。无法在非CDB环境中设置此参数。

默认值0表示对PDB中每秒可发出的最大兆字节数（MB）没有设置限制。

如果Oracle进程因IO速率限制而需要等待，则等待事件为resmgr：I / O速率限制。

也可以看看：

  • “ MAX_IOPS ”

  • “ resmgr：I / O速率限制 ”

例子

此示例显示如何使用SQL语句在名为CDB1_PDB1的PDB上设置每秒最多200兆字节：

alter session set container = cdb1_pdb1; alter system set max_mbps = 200;
