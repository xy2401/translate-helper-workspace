13.1什么是SQL * Loader Express模式？

SQL * Loader express模式允许您通过在表列全部是字符，数字或日期时间数据类型时仅指定表名来快速执行加载，并且输入数据文件仅包含分隔的字符数据。

在快速模式下，不使用SQL * Loader控制文件。而是，SQL * Loader使用在ALL_TAB_COLUMNS视图中找到的表列定义来确定输入字段顺序和数据类型。对于大多数其他设置，它假定您可以使用命令行参数覆盖的默认值。

注意：

用于SQL * Loader express模式的唯一有效参数是本章中描述的参数。任何其他参数将被忽略或可能导致错误。

  • 外部表是加载方法。对于某些错误，SQL * Loader express模式会自动从默认外部表加载方法切换到直接路径加载。可能发生这种情况的一个示例是，如果权限违规导致CREATE DIRECTORY SQL命令失败。

    • 没有圈地

    • 左右修剪

  • DEGREE_OF_PARALLELISM参数设置为AUTO 。

  • 日期和时间戳格式使用NLS设置。

  • 使用NLS客户端字符集。

  • 如果新数据中已包含数据，则将新数据附加到表中。

13.2.1.1 SQL * Loader Express模式如何处理字节顺序

通常，SQL * Loader express模式处理字节顺序标记的方式与使用SQL * Loader控制文件执行的加载相同。

综上所述：

  • 对于具有unicode字符集的数据文件，SQL * Loader express模式检查文件开头的字节顺序标记。

  • 对于UTF16数据文件，如果找到字节顺序标记，则字节顺序标记设置数据文件的字节顺序。如果未找到字节顺序标记，则执行SQL * Loader的系统的字节顺序将用于数据文件。

  • 无论字节顺序（字节顺序）是否与运行SQL * Loader express的系统的字节顺序相同，都可以加载UTF16数据文件。

  • 对于UTF8数据文件，将跳过找到的任何字节顺序标记。

  • 如果涉及多个数据文件并且它们使用不同的字节顺序，则终止加载。

13.3.1坏

默认值：默认值取决于是否指定了任何数据文件（使用DATA参数）。请参见SQL * Loader Express模式使用的默认值 。

目的

BAD参数指定坏文件的位置和名称。

语法和说明

BAD = [ 目录 /] [ 文件名 ]

坏文件存储在插入期间导致错误或格式不正确的记录。如果指定BAD参数，则必须提供目录或文件名，或两者都提供。如果未指定BAD参数，并且存在拒绝记录，则使用默认文件名。

directory变量指定写入错误文件的目录。规范可以包括设备名称或网络节点。

filename变量指定在您的平台上被识别为有效的文件名。您必须仅指定名称（和扩展名，如果需要除.bad之外的.bad ）。文件名中的任何空格或标点符号必须用单引号括起来。

directory和filename的值确定如下：

  • 如果使用文件名但没有目录指定BAD参数，则该目录默认为当前目录。

  • 如果使用目录但没有文件名指定BAD参数，则使用指定的目录，并将默认值用于文件名和扩展名。

BAD参数适用于与指定的DATA参数匹配的所有文件（如果已指定）。如果未指定data参数，它将应用于一个数据文件（ table-name.dat ）。

限制

警告：

  • 如果文件名（默认值或您指定的文件名）已存在，则会覆盖该文件名或创建新版本，具体取决于您的操作系统。

  • 如果正在加载多个数据文件，则建议您不要指定BAD参数，也不要仅指定坏文件的目录。

例

以下规范在当前目录中创建名为emp1.bad的错误文件：

> sqlldr hr TABLE = employees BAD = emp1

13.3.2字符

默认值： NLS_LANG环境变量中指定的NLS客户端字符集

目的

CHARACTERSET参数指定用于加载的字符集（默认值除外）。

语法和说明

CHARACTERSET = character_set_name

CHARACTERSET参数指定SQL * Loader输入数据文件的字符集。如果未指定CHARACTERSET参数，则所有数据文件的缺省字符集是会话字符集，该字符集由NLS_LANG环境变量定义。只有字符数据（SQL * Loader数据类型CHAR, VARCHAR, VARCHARC,数字EXTERNAL以及日期时间和间隔数据类型的字段）受数据文件的字符集影响。

character_set_name变量指定字符集名称。通常，指定的名称必须是Oracle数据库支持的字符集的名称。

对于UTF-16 Unicode编码，请使用名称UTF16而不是AL16UTF16。AL16UTF16是UTF-16编码数据支持的字符集名称，仅适用于big-endian字节顺序的UTF-16数据。但是，因为您可以使用创建数据文件的系统的字节顺序来设置数据，所以数据文件中的数据可以是big-endian或little-endian。因此，使用不同的字符集名称（UTF16）。还支持字符集名称AL16UTF16。但是，如果为具有little-endian字节顺序的数据文件指定AL16UTF16，则SQL * Loader会发出警告消息并将数据文件作为little-endian处理。

假定CHARACTERSET参数值对于所有数据文件都相同。

注意：

术语UTF-16是对Unicode的UTF-16编码的一般参考。术语UTF16（无连字符）是字符集的特定名称，是您在使用UTF-16编码时应为CHARACTERSET参数指定的名称。这也适用于UTF-8和UTF8。

限制

  • 没有

例

以下示例指定UTF-8字符集：

> sqlldr hr TABLE = employees CHARACTERSETNAME = utf8

13.3.3 CSV

默认值：如果未在命令行中指定CSV参数，则SQL * Loader express假定正在加载的CSV文件包含没有嵌入字符且没有机箱的数据。

如果在命令行中指定了CSV=WITHOUT_EMBEDDED ，则SQL * Loader express假定正在加载的CSV文件包含没有嵌入字符且可选地用“”括起来的数据。

目的

CSV参数提供的选项允许您指定要加载的逗号分隔值（CSV）格式文件是否包含嵌入了记录终止符的字段。

语法和说明

CSV = [WITH_EMBEDDED | WITHOUT_EMBEDDED]

此参数的有效选项如下：

  • WITH_EMBEDDED - 此选项意味着记录中的字段中可能包含（嵌入）记录终止符。记录终止符是换行符。默认分隔符是TERMINTATED BY ","和OPTIONALLY_ENCLOSED_BY '"' 。必须附上嵌入式记录终止符。

    如果CSV文件包含许多嵌入式记录终止符，则可能会对性能产生负面影响。

  • WITHOUT_EMBEDDED - 此选项表示记录中的字段中不包含（嵌入）记录终止符。记录终止符是换行符。默认分隔符是TERMINTATED BY ","和OPTIONALLY_ENCLOSED_BY '“'。

限制

  • 通常，文件可以并行处理（一次拆分并由多个执行服务器处理）。但对于具有嵌入式记录终止符的CSV格式文件，该文件必须仅由一个执行服务器处理。因此，当CSV=WITH_EMBEDDED时，将禁用数据文件中的并行处理。

例

以下示例将数据文件作为具有嵌入式记录终止符的CSV格式文件进行处理。

> sqlldr hr TABLE = employees CSV = WITH_EMBEDDED

13.3.4数据

默认值：与表名相同的名称，但扩展名为.dat。

目的

DATA参数指定包含要加载的数据的数据文件的名称。

语法和说明

DATA = data-file-name

如果未指定文件扩展名，则默认为。 dat.

文件规范可以包含通配符（仅在文件名和文件扩展名中，而不在设备或目录名中）。星号（*）代表多个字符和一个问号（？）代表一个字符。例如：

DATA ='emp * .dat'DATA ='m？emp.dat”

要列出多个数据文件规范（每个规范都可以包含通配符），文件名必须用逗号分隔。

如果文件名包含任何特殊字符（例如，空格，*，？），则整个名称必须用单引号括起来。

以下是DATA参数可能有效使用的三个示例（仅当文件名包含特殊字符时，才需要使用单引号）：

DATA = 'file1的'， '文件2'， '文件3'， 'file4将'， 'file5'， 'file6'
DATA ='file1'，'file2'DATA ='file3，'file4'，'file5'DATA ='file6'
DATA ='file1'DATA ='file2'DATA ='file3'DATA ='file4'DATA ='file5'DATA ='file6'

限制

警告：

如果正在加载多个数据文件并且您还指定了BAD参数，则建议您仅指定坏文件的目录，而不是文件名。如果指定文件名，并且已存在具有该名称的文件，则会覆盖该文件或创建新版本，具体取决于您的操作系统。

例

假设当前目录包含名称为emp1.dat ， emp2.dat ， m1emp.dat和m2emp.dat数据文件，并发出以下命令：

> sqlldr hr TABLE = employees DATA ='emp *'，'m1emp'

该命令加载emp1.dat ， emp2.dat和m1emp.dat文件。未加载m2emp.dat文件，因为它与任何通配符条件都不匹配。

13.3.5 DATE_FORMAT

默认

13.3.6 DEGREE_OF_PARALLELISM

SQL * Loader express模式的DEGREE_OF_PARALLELISM命令行参数指定用于加载的并行度。

默认

汽车

目的

DEGREE_OF_PARALLELISM参数指定用于加载的并行度。

语法和说明

DEGREE_OF_PARALLELISM = [ degree-num |默认|自动|无]

如果指定了degree-num ，则它必须是从1到n的整数值。

如果指定了DEFAULT ，则使用数据库的默认并行度 （不是默认参数值AUTO ）。

如果使用AUTO ，则Oracle数据库会自动设置负载的并行度。如果DEGREE_OF_PARALLELISM没有指定DEGREE_OF_PARALLELISM参数，这也是默认值。

如果指定了NONE ，则不会并行执行加载。值NONE与值1相同。

也可以看看：

  • 有关并行执行的更多信息，请参见“Oracle数据库VLDB和分区指南”

限制

  • 如果强制load方法为常规路径或直接路径（使用NONE选项），则忽略DEGREE_OF_PARALLELISM参数。每当您指定DEGREE_OF_PARALLELISM参数时，对于任何值，您都会收到一条消息，提醒您注意这一点。

  • 如果加载是默认的外部表加载并且发生错误导致SQL * Loader express模式使用直接路径加载，那么即使您已指定一定程度的并行性或已接受外部作业，也不会并行执行作业表默认为AUTO 。将显示一条消息，提醒您进行此更改。

例

以下示例在不使用并行性的情况下加载数据：

> sqlldr hr TABLE = employees DEGREE_OF_PARALLELISM = NONE

13.3.7直接

默认值：没有默认值

目的

DIRECT参数指定要使用的加载方法，传统路径或直接路径。

语法和说明

DIRECT = [TRUE | FALSE]

值TRUE指定直接路径加载。值FALSE指定传统路径负载。

此参数将覆盖SQL * Loader express模式使用的外部表的默认加载方法。

对于某些错误，SQL * Loader express模式会自动从默认外部表加载方法切换到直接路径加载。可能发生这种情况的一个示例是，如果权限违规导致CREATE DIRECTORY SQL命令失败。

  • BINDSIZE

  • COLUMNARRAYROWS （仅限直接路径加载）

  • DATE_CACHE

  • 错误

  • MULTITHREADING （仅限直接路径加载）

  • NO_INDEX_ERRORS （仅限直接路径加载）

  • RESUMABLE

  • RESUMABLE_NAME

  • RESUMABLE_TIMEOUT

  • ROWS

  • 跳跃

  • streamsize可

限制

  • 没有

例

在以下示例中，SQL * Loader使用直接路径加载方法来加载而不是外部表：

> sqlldr hr TABLE = employees DIRECT = TRUE

13.3.8 DNFS_ENABLE

DNFS_ENABLE SQL * Loader express mode参数允许您在SQL * Loader操作期间启用和禁用对输入数据文件使用Direct NFS Client。

默认

真正

目的

DNFS_ENABLE参数允许您在SQL * Loader操作期间启用和禁用对输入数据文件使用Direct NFS Client。

直接NFS客户端是一种可以由文件服务器实现的API，以便在Oracle访问这些服务器上的文件时提高性能。

语法和说明

语法如下：

DNFS_ENABLE = [TRUE | FALSE]

SQL * Loader在读取超过1 GB的数据文件时默认使用Direct NFS Client接口。对于较小的文件，使用操作系统的I / O接口。要在所有输入数据文件上使用Direct NFS Client，请使用DNFS_ENABLE=TRUE 。

要禁用对所有数据文件使用Direct NFS客户端，请指定DNFS_ENABLE=FALSE 。

DNFS_ENABLE参数可以与DNFS_READBUFFERS参数一起使用，该参数可以指定Direct NFS客户端使用的读取缓冲区的数量。

13.3.9 DNFS_READBUFFERS

DNFS_READBUFFERS SQL * Loader express模式参数允许您控制Direct NFS客户端使用的读取缓冲区的数量。

默认

4

目的

DNFS_READBUFFERS参数允许您控制Direct NFS客户端使用的读取缓冲区数。直接NFS客户端是一种可以由文件服务器实现的API，以便在Oracle访问这些服务器上的文件时提高性能。

语法和说明

语法如下：

DNFS_READBUFFERS = n

使用大于默认值的值可能会补偿Direct NFS客户端文件服务器的I / O不一致，但可能会导致内存使用量增加。

13.3.11 EXTERNAL_TABLE

EXTERNAL_TABLE SQL * Loader express模式参数指示SQL * Loader是否使用外部表选项加载数据。

默认值： EXECUTE

目的

EXTERNAL_TABLE参数指示SQL * Loader是否使用外部表选项加载数据。

语法和说明

EXTERNAL_TABLE = [NOT_USED | GENERATE_ONLY |执行]

有三个可能的值：

  • NOT_USED - 表示使用传统或直接路径模式执行加载。

  • GENERATE_ONLY - 使用SQL * Loader日志文件中的外部表放置加载所需的所有SQL语句。可以编辑和自定义这些SQL语句。通过在SQL * Plus中执行这些语句，可以在不使用SQL * Loader的情况下稍后完成实际加载。

  • EXECUTE - SQL * Loader express模式中的默认值。尝试执行使用外部表进行加载所需的SQL语句。但是，如果任何SQL语句返回错误，则加载尝试将停止。语句在执行时放在日志文件中。这意味着如果SQL语句返回错误，则加载所需的其余SQL语句将不会放在日志文件中。

请注意，外部表选项使用数据库中的目录对象来指示存储所有数据文件的位置，并指示创建输出文件（如坏文件和丢弃文件）的位置。您必须具有对包含数据文件的目录对象的READ访问权限，并且您必须具有对创建输出文件的目录对象的WRITE访问权限。如果数据文件或输出文件的位置没有现有目录对象，则SQL * Loader将生成SQL语句以创建一个。因此，指定EXECUTE选项时，您必须具有CREATE ANY DIRECTORY权限。如果希望在加载结束时删除目录对象，则还必须具有DROP ANY DIRECTORY权限。

注意：

EXTERNAL_TABLE= EXECUTE限定符告诉SQL * Loader创建一个外部表，该表可用于加载数据，然后执行INSERT语句来加载数据。必须将外部表中的所有文件标识为位于目录对象中。SQL * Loader尝试使用已存在且您有权访问的目录对象。但是，如果SQL * Loader找不到匹配的目录对象，则它会尝试创建临时目录对象。如果您没有创建新目录对象的权限，则操作将失败。

要解决此问题，请使用EXTERNAL_TABLE= GENERATE_ONLY来创建SQL * Loader将尝试执行的SQL语句。提取这些SQL语句并将对目录对象的引用更改为您有权访问的目录对象。然后，执行这些SQL语句。

限制

  • 没有

例

sqlldr hr TABLE = employees EXTERNAL_TABLE = NOT_USED

13.3.12 FIELD_NAMES

默认值： NONE

目的

FIELD_NAMES参数用于覆盖数据库表中列的顺序的字段。（默认情况下，SQL * Loader Express使用ALL_TAB_COLUMNS视图中的表列定义来确定输入字段顺序和数据类型。）

此参数何时有用的示例是输入文件中的数据与表中的列的顺序不同。在这种情况下，您可以在数据文件中包含字段名称记录（类似于表的列标题行），并使用FIELD_NAMES参数通知SQL * Loader处理第一个记录中的字段名称以确定顺序的领域。

语法和说明

FIELD_NAMES = [ALL | ALL_IGNORE |第一个| FIRST_IGNORE |没有]

此参数的有效选项如下：

  • ALL - 为每个数据文件处理字段名称记录。

  • ALL_IGNORE - 忽略所有数据文件中的第一个（字段名称）记录并正常处理数据记录。

  • FIRST - 在第一个数据文件中，处理第一个（字段名称）记录。对于所有其他数据文件，没有字段名称记录，因此正常处理数据文件。

  • FIRST_IGNORE - 在第一个数据文件中，忽略第一个（字段名称）记录，并使用表列顺序作为字段顺序。

  • NONE - 任何数据文件中都没有字段名称记录，因此数据文件会正常处理。这是默认值。

限制

  • 如果任何字段名称具有大小写混合大小写或特殊字符（例如，空格），则必须使用OPTIONALLY_ENCLOSED_BY参数或ENCLOSED_BY参数来指示应保留该大小写，并且应将特殊字符作为字段名称的一部分包含在内。

例

如果要将包含列标题的CSV文件加载到表中，并且输入文件中每行中的字段与表中的列的顺序相同，则可以使用以下命令：

> sqlldr hr TABLE = employees CSV = WITHOUT_EMBEDDED FIELD_NAMES = FIRST_IGNORE

13.3.13负荷

默认值：加载所有记录。

目的

LOAD = n

限制

  • 没有

> sqlldr hr TABLE = employees LOAD = 10

对于常规和直接路径负载，成功和不成功的加载尝试都计入总数。因此，如果文件中有15条记录，而记录2和4是坏的，那么实际上只有以下8条记录被加载到表中 - 1,3,5,6,7,8,9和10。

13.3.17沉默

默认值：如果未指定此参数，则不会抑制任何内容。

目的

SILENT参数禁止在SQL * Loader操作期间写入屏幕的某些内容。

语法和说明

语法如下：

SILENT = {HEADER |反馈|错误|丢弃| PARTITIONS |所有}

使用适当的值来抑制以下一个或多个（如果指定了多个选项，则必须用逗号分隔）：

  • HEADER - 禁止通常出现在屏幕上的SQL * Loader标头消息。标头消息仍显示在日志文件中。

  • FEEDBACK - 抑制“达到提交点”消息和通常出现在屏幕上的负载的状态消息。

  • ERRORS - 抑制日志文件中的数据错误消息，这些消息在记录生成导致将其写入错误文件的Oracle错误时发生。仍会显示已拒绝记录的计数。

  • DISCARDS - 禁止写入丢弃文件的每条记录的日志文件中的消息。在快速模式下忽略此选项。

  • PARTITIONS - 禁用在直接加载分区表期间将每分区统计信息写入日志文件。此选项仅在强制直接路径操作中有意义。

  • ALL - 实现所有抑制选项。

例

例如，您可以使用以下命令行参数来抑制通常出现在屏幕上的标题和反馈消息：

> sqlldr hr TABLE = employees SILENT = HEADER，FEEDBACK

13.3.21 TRIM

默认值：传统和直接路径加载的默认值为LDRTRIM 。外部表加载的默认值是LRTRIM 。

目的

TRIM参数指定在加载期间使用的修剪类型。

语法和说明

TRIM = [LRTRIM | NOTRIM | LTRIM | RTRIM | LDRTRIM]

TRIM参数用于指定应从文本字段的开头或文本字段的结尾或两者中修剪空格。空格包括空格和其他非打印字符，例如制表符，换行符和回车符。

NOTRIM表示不会从字段中修剪任何字符。此设置通常可以获得最快的性能。

LRTRIM ， LTRIM和RTRIM用于指示应从字段中修剪字符。LRTRIM意味着修剪前导和尾随空格。LTRIM意味着将修剪前导空格。RTRIM意味着尾随空格被修剪。

LDRTRIM相同NOTRIM除非在以下情况下：

  • 如果该字段是指定了OPTIONALLY _ ENCLOSED _ BY的分隔字段，并且特定实例缺少可选的外壳，则将从左侧修剪空格。

如果为所有空格的字段指定了修剪，则该字段将设置为NULL 。

限制

  • 强制常规路径和强制直接路径加载仅支持LDRTRIM 。无论TRIM指定TRIM参数，对于任何值，您都会收到一条消息，提醒您注意这一点。

  • 如果加载是默认外部表加载并且发生错误导致SQL * Loader express模式使用直接路径加载，则LDRTRM用作修剪方法，即使您指定了不同的方法或已接受外部表默认LRTRIM 。将显示一条消息，提醒您进行此更改。

    要使用NOTRIM ，请使用带有PRESERVE BLANKS子句的控制文件。

例

以下示例读取字段，修剪右侧的所有空格（尾随空格）。

> sqlldr hr TABLE = employees TRIM = RTRIM
