12.3 @@（符号加倍）

句法

@@ { url | file_name [。 ext ]} [ arg ...]

运行脚本。此命令几乎与@（at sign）命令相同。运行嵌套脚本时，它会在与调用脚本相同的路径或URL中查找嵌套脚本。@@ 命令功能与@和START类似。

条款

网址

指定要在指定的Web服务器上运行的脚本的统一资源定位器。SQL * Plus支持HTTP和FTP协议，但不支持HTTPS。此版本不支持http：// username：password @ machine_name.domain ...形式的HTTP身份验证。 

file_name [。 分机 ]

代表嵌套脚本你想跑。如果省略ext ，SQL * Plus将采用默认的命令文件扩展名（通常为SQL）。有关更改默认扩展名的信息，请参阅SET SUF [FIX] {SQL |文字} 。

从脚本中输入@@ file_name.ext时 ，SQL * Plus会从与脚本相同的目录运行file_name.ext 。

以交互方式输入@@ file_name.ext时 ，SQL * Plus会运行file_name 。来自当前工作目录的ext ，或者来自与之相同的url的url 调用。如果SQL * Plus找不到该文件，它将搜索与系统相关的路径以查找该文件。某些操作系统可能不支持路径搜索。有关操作系统环境的特定信息，请参阅为操作系统提供的特定于平台的Oracle文档。 

arg ...

代表您希望传递给参数的数据项在脚本中。 如果输入一个或多个参数，SQL * Plus会将值替换为参数脚本中的（＆1，＆2等）。第一个参数替换＆1的每个匹配项，第二个参数替换＆2的每个匹配项，依此类推。

@@命令使用参数给出的值定义参数。如果在此会话中再次运行脚本，则可以输入新参数或省略参数以使用当前值。有关使用参数的更多信息，请参阅使用替换变量 。

用法

以下所有设置（如COLUMN命令设置）在脚本启动时保持有效。如果脚本更改了任何设置，则新值将在脚本完成后保持有效。

您可以在脚本中包含通常以交互方式输入的任何命令（通常是SQL或SQL * Plus命令）。

如果禁用START命令（请参阅禁用SQL * Plus_ SQL_和PL / SQL命令 ），这也将禁用@@命令。有关更多信息，请参阅SPOOL命令。

SQL * Plus在发出@@命令之前删除SQLTERMINATOR（默认为分号）。解决方法是添加另一个SQLTERMINATOR。参见SET SQLT [ERMINATOR] {; | c | ON | OFF}了解更多信息。

例子

假设您有以下名为PRINTRPT的脚本：

SELECT DEPARTMENT_ID，CITY FROM EMP_DETAILS_VIEW WHERE SALARY> 12000; @ EMPRPT.SQL @@ WKRPT.SQL

当您启动PRINTRPT并且它到达@命令时，它会在当前工作目录中查找名为EMPRPT的脚本并运行它。当PRINTRPT到达@@命令时，它会在与PRINTRPT相同的路径中查找名为WKRPT的脚本并运行它。

假设相同的脚本PRINTRPT位于Web服务器上，并使用START HTTP：// machine_name.domain：port / PRINTRPT运行它。当它到达@命令时，它会在当前工作目录中查找名为EMPRPT的脚本并运行它。当PRINTRPT到达@@命令时，它会在与PRINTRPT，HTTP：// machine_name.domain：port /WKRPT.SQL相同的URL中查找名为WKRPT的脚本并运行它。
