14.20 DROP TYPE声明

DROP TYPE语句删除ADT， VARRAY类型或嵌套表类型的规范和正文。

  • 先决条件

  • 句法

  • 语义

  • 例

  • 相关话题

先决条件

ADT， VARRAY类型或嵌套表类型必须位于架构中，或者您必须具有DROP ANY TYPE系统特权。

句法

drop_type :: =


插图说明drop_type.eps

语义

模式

包含该类型的架构的名称。默认值：您的架构。

TYPE_NAME

要删除的对象，varray或嵌套表类型的名称。您只能删除没有类型或表依赖项的类型。

如果type_name是超类型，则除非您还指定FORCE否则此语句将失败。如果指定FORCE ，则数据库将根据此超类型使所有子类型无效。

如果type_name是统计类型，则除非您还指定FORCE否则此语句将失败。如果指定FORCE ，则数据库首先解除与type_name关联的所有对象的关联，然后删除type_name 。

也可以看看：

  • 有关ASSOCIATE STATISTICS语句的信息，请参见Oracle数据库SQL语言参考

  • 有关DISASSOCIATE STATISTICS语句的信息，请参见Oracle数据库SQL语言参考

如果type_name是与统计信息类型关联的ADT，则数据库首先尝试取消type_name与统计信息类型的关联，然后删除type_name 。但是，如果使用统计信息类型收集了统计信息，则数据库无法将type_name与统计信息类型取消关联，并且此语句将失败。

如果type_name是索引类型的实现类型，则索引类型标记为INVALID 。

如果type_name上定义了公共同义词，则数据库也会删除同义词。

除非指定FORCE ，否则只能删除不具有依赖关系的独立模式对象类型。这是默认行为。

力

即使它具有依赖的数据库对象，也会删除该类型。数据库标记UNUSED所有依赖于要删除的类型的列，并且这些列变得不可访问。

注意：

Oracle建议不要指定FORCE来删除具有依赖项的对象类型。此操作无法恢复，可能会使从属表或列中的数据无法访问。

VALIDATE

使数据库在其任何超类型的可替换列中检查此类型的存储实例。如果未找到此类实例，则数据库将完成放置操作。

该子句仅对子类型有意义。Oracle建议使用此选项安全地删除没有任何显式类型或表依赖性的子类型。

例

例14-37丢弃ADT

此语句删除ADT person_t 。有关创建此ADT的示例，请参阅“ CREATE TYPE语句 ” 。任何依赖于person_t列都标记为UNUSED并且无法访问。 

DROP TYPE person_t FORCE;
