验证要求

在尝试本附录中的任何故障排除步骤之前，请执行以下操作：

  • 检查相应网关安装章节的系统要求部分，以确保系统满足要求并且您已完成所有预安装任务。

  • 在安装之前，请阅读平台上产品的发行说明。发行说明可在Oracle软件安装介质上找到。您可以在Oracle Technology Network网站上找到最新版本的发行说明：

    http://docs.oracle.com/

如果发生安装错误该怎么办

如果在安装过程中遇到错误，则：

  • 不要退出Oracle Universal Installer。

  • 如果在输入有关其中一个安装屏幕的错误信息后单击“ 下一步” ，则单击“上一步”返回到屏幕并更正信息。

  • 如果在Oracle Universal Installer复制或链接文件时遇到错误，请参阅查看安装会话日志 。

  • 如果在配置助手运行时遇到错误，请参阅配置助手故障排除 。

  • 如果无法解决问题，请按照“ 安装失败后清理”中列出的步骤删除失败的安装 。

查看安装会话的日志

中安装时，Oracle Universal Installer会在日志文件中记录它执行的所有操作。如果在安装过程中遇到问题，请查看日志文件以获取有关问题可能原因的信息。 

SYSTEM_DRIVE：\ Program Files \ Oracle \ Inventory \ logs

日志文件名采用以下形式：

installActions date_time .log

例如，如果安装发生在2009年5月14日上午9:00:56，那么日志文件将被命名为：

installActions2009-05-14_09-00-56-am.log

注意：

请勿删除或手动更改Inventory目录或其内容。这样做可能会阻止Oracle Universal Installer查找您在系统上安装的产品。

配置助手故障排除

要解决配置助手运行时发生的安装错误，请执行以下操作：

  • 查看“ 查看安装会话日志”中列出的安装日志文件。

  • 查看位于ORACLE_BASE \ ORACLE_HOME \cfgtoollogs目录中的特定配置助手日志文件。尝试解决导致错误的问题。

  • 如果你看到“致命错误。重新安装“消息，然后通过查看日志文件查找问题的原因。有关详细说明，请参阅致命错误 。

配置助手失败

安装屏幕底部会注明Oracle配置助手故障。配置助手界面显示其他信息（如果有）。配置助手执行状态存储在installActions date_time .log文件中。

执行状态代码列在下表中：

状态 结果代码

配置助手成功

0

配置助手失败

1

配置助手已取消

-1

致命错误

如果在配置助手运行时收到致命错误，则：

  1. 按照在安装失败后清理中所述删除失败的安装 。
  2. 纠正致命错误的原因。
  3. 重新安装Oracle软件。

非交互式安装响应文件错误处理

要确定非交互式安装是成功还是失败，请检查位于SYSTEM_DRIVE :\Program Files\Oracle\Inventory\logs的installActions date_time .log文件。

如有必要，请参阅上一节，以获取有关确定Inventory目录位置的信息。

静默安装失败，如果：

  • 您没有指定响应文件。

  • 您指定了不正确或不完整的响应文件。

  • Oracle Universal Installer遇到错误，例如磁盘空间不足。

Oracle Universal Installer或配置助手在运行时验证响应文件。如果验证失败，则非交互式安装或配置过程结束。 Oracle Universal Installer会处理具有错误上下文，格式或类型的参数的值，就好像文件中未指定任何值一样。

安装失败后清理

如果安装失败，则必须删除Oracle Universal Installer在尝试安装期间创建的文件并删除Oracle主目录。按照删除Oracle Database Gateway中的说明运行deinstall工具以删除网关，删除Oracle目录，并从注册表编辑器键中删除Oracle。稍后，重新安装该软件。
