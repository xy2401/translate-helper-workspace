6.1开发OLAP应用程序

您可以对多维数据集使用任何SQL查询作为报表或仪表板的内容。BI Publisher和Application Express都包含一个查询生成器，您可以使用它来针对关系对象和维度对象开发查询。您还可以从SQL脚本或其他源中剪切和粘贴查询，这是本章中使用的方法。

如果您的目标是创建静态报告和仪表板，则无需进一步阅读。您可以使用自己喜欢的工具立即开始开发OLAP应用程序。本章介绍如何使用动态内容创建应用程序。它侧重于利用多维数据集和维度的独特功能，使用单个查询创建可钻取的报表和图表。您将学习如何创建两种类型的可钻取接口：

  • 选项列表：您可以为每个维度创建下拉列表，以在报表或仪表板中钻取维度。

  • 链接的维度列：在Application Express中，您可以添加指向交叉表的维度列的链接以向下钻取到层次结构的底部，并使用“重置”按钮返回到顶级。

这些用户界面在源查询的WHERE子句中设置绑定变量的值。当用户更改选项列表中的当前选择或单击交叉表中的链接时，该操作会动态更改变量的值。当变量发生变化时，查询条件以及报告或仪表板的内容也会发生变化。

当变量设置层次结构视图的PARENT列的值时，用户可以在父级上钻取以查看其子级。

示例6-1显示了Global示例模式中针对UNITS_CUBE_VIEW的基本SQL查询。查询选择SALES度量和三个使用SALES作为计算基础的计算度量：

  • SALES_PP ：上一期间的销售额。

  • SALES_CHG_PP ：当前期间与前期期间的销售额差异。

  • SALES_PCTCHG_PP ：当前期间与前期期间销售额的百分比差异。

此查询用于本章开发的示例应用程序。产品，客户和时间维度的PARENT列支持在这些应用程序中进行钻取。渠道维度仍然固定在总水平。

6.2.2在Microsoft Word中创建模板

BI Publisher不包含格式化工具。相反，它使您能够使用熟悉的桌面应用程序设计报表。此示例使用Microsoft Word。报告模板可以包含：

  • 您输入的静态文本和图形与任何其他Word文档一样。

  • 动态字段，例如由Word处理的日期和时间或页码。

  • 用于标识数据的XML标记的代码，由BI Publisher处理。当BI Publisher生成报告时，它会使用这些标记标识的数据替换代码。

您可以在Word中格式化报表模板的所有部分，选择字体，文本和背景颜色，表格设计等。

6.2.4在BI Publisher中添加维度选择列表

您可以将维度的选项列表添加到报表中。生成报告时，您可以更改数据选择而不更改查询。要添加选项列表，请执行以下步骤：

  • 创建一个或多个要在菜单中显示的值列表（LOV）。

  • 创建用于显示LOV的菜单。

  • 编辑查询以使用为菜单创建的绑定变量。

以下主题描述了这些步骤：

  • 为BI Publisher报告创建值列表

  • 创建菜单

  • 在BI Publisher中编辑查询

6.2.4.2创建菜单

在BI Publisher中，菜单是一种参数。创建参数会自动创建一个绑定变量，您可以在报表的查询中使用该变量。

要创建菜单：

  1. 选择Parameters，然后单击New 。

    将打开“参数”页面。

  2. 定义参数：

    1. 对于标识符，请输入product等名称。

      这是您将在查询中使用的绑定变量的区分大小写的名称。

    2. 选择适当的数据类型，通常为String。

    3. 对于Default Value，输入LOV查询的WHERE子句中使用的维度键。

      菜单最初显示此键。

    4. 对于参数类型，选择菜单 。

    5. 选择适当的值列表。

    6. 清除所有选项。

  3. 单击保存 。

对其他尺寸重复这些步骤。此示例为Product，Customer和Time创建菜单。

6.2.4.3在BI Publisher中编辑查询

要激活菜单，请在查询中更改WHERE子句以使用绑定变量。绑定变量的值是当前菜单选项。

这是WHERE子句的条件格式：

parent_column =：bind_variable

在此示例中， WHERE子句使用Time，Product和Customer的绑定变量：

WHERE p.parent =：product AND cu.parent =：customer AND t.parent =：time AND ch.level_name ='TOTAL'

要编辑查询：

  1. 在“数据模型”下，选择为此报告定义的数据集。

    将打开“数据集”页面。

  2. 在“SQL查询”框中，编辑WHERE子句以使用参数定义创建的绑定变量。

  3. 单击保存 。

与BI Publisher一样，Application Express使您可以通过添加维度键的选择列表来钻取维度。仪表板用户可以从列表中选择特定项目，并动态更改页面上一个或多个图形和交叉表中显示的数据选择。要实现选择列表，请执行以下步骤：

6.3.2.1创建区域

    “创建区域”向导将打开。

  2. 在Region页面上，选择HTML ，单击Next ，然后再次选择HTML 。

  3. 在“显示属性”页面上，输入描述性标题，然后在页面上为列表选择适当的模板和位置。

    对于此示例，名称为lov_region ，模板为No Template ，位置为Page Template Body（模板内容下方1个项目） 。名称可以显示在呈现的页面上，但在此示例中隐藏了该名称。

  4. 单击创建区域 。

    该区域显示在“区域”下的“页面定义”中。

6.3.2.3创建选择列表

对于选项列表，您将创建一个显示LOV的列表项。

要创建列表项：

  1. 在“项目”下的“页面定义”上，单击“ 创建”图标。

    “创建项目”向导将打开。

  2. 在“项目类型”页面上，选择“ 选择列表”。

  3. 对于Control Type，选择Select List with Submit。

  4. 在“显示位置和名称”页面上：

    • 输入标识维度的名称，例如Customer绑定变量名称的P1_CUSTOMER 。P1是页码， CUSTOMER标识Customer维度。

    • 选择列表位置的新HTML区域。

  5. 在“值列表”页面上，设置以下值：

    • 将LOV命名为为此维创建的值列表，例如CUSTOMER_LOV 。

    • 将空选项显示为否 。

  6. 根据您自己的喜好选择Item属性。

  7. 在“源”页面上，输入默认值的顶部维度键的名称。

    对于全局客户维度，值为TOTAL 。

  8. 单击“ 创建项” 。

对其他列表重复这些步骤。此示例为Product和Customer维度创建列表。

要激活列表项：

  1. 在“分支”下的“页面定义”上，单击“ 创建”图标。

    “编辑分支”向导将打开。

  2. 在“点和类型”页面上，接受默认设置。

  3. 在目标页面上：

    • 在此应用程序中将目标设置为页面 。

    • 将Page设置为包含列表项的页面，在本例中为1 。

    • 选择重置此页面的分页 。

  4. 在“分支条件”页面上，接受默认设置以创建无条件分支。

  5. 单击应用更改 。

    “编辑分支”页面关闭，您将返回“页面定义”。无条件分支列在分支下。

6.3.3.1创建隐藏项

您可以在Application Express中创建提供绑定变量的各种类型的项目。它们存储特定元素的会话状态，在本例中为父维度键的当前选择。

支持钻取的每个维度都需要一个绑定变量。在此示例中，Product和Customer具有使用列表项创建的绑定变量。时间是报告中唯一没有绑定变量的显示维度。由于时间维度列中的链接提供了用于更改会话状态的用户界面，因此Time不需要任何其他图形用户界面。隐藏的项目用于此目的。

要创建隐藏项目：

  1. 打开页面定义。

  2. 在“项目”下，单击“ 创建”图标。

    “创建项目”向导将打开。

  3. 在“项目类型”页面上，选择“ 隐藏” 。

  4. 在“显示位置和名称”页面上：

    • 输入标识维度的名称，例如时间绑定变量名称的P1_TIME 。

    • 选择定义报告的区域。

  5. 在“源”页面上，输入层次结构顶部的维度键。

    TOTAL是全局模式中所有层次结构的顶部。对于此示例，Time设置为CY2006以将选择限制为一年。

  6. 单击“ 创建项” 。

  7. 对于仅支持在列链接上钻取的任何其他维度，请重复这些步骤。

    对于此示例，为Time定义了隐藏项。

6.3.3.2编辑查询以使用绑定变量

要将列链接添加到报表，必须更改SELECT语句的两个区域：

  • 选择列表：Application Express仅管理选择列表中显示的列。您可以选择显示或隐藏列。要定义列链接，请将层次结构视图中的DIM_KEY和PARENT列添加到查询选择列表中。

  • WHERE子句：为“隐藏项”添加绑定变量，就像在“ 编辑查询 ”中为选项列表所做的那样。

例6-3显示了修改后的示例查询。

示例6-3 Application Express中列链接的修订查询

SELECT p.long_description“Product”，cu.long_description“Customer”，t.long_description“Time”，ROUND（f.sales）“Sales”，ROUND（f.sales_pp）“Prior Period”，ROUND（f.sales_chg_pp）“更改“，ROUND（f.sales_pctchg_pp * 100）”“更改百分比”，/ *为列链接添加DIM_KEY和PARENT列* / p.dim_key product_key，p.parent product_parent，cu.dim_key customer_key，cu.parent customer_parent，t。 dim_key time_key，t.parent time_parent / *来自维度视图和多维数据集视图* / FROM product_primary_view p，customer_shipments_view cu，time_calendar_view t，channel_primary_view ch，units_cube_view f / *使用父列和绑定变量进行钻取* / WHERE p.parent = NVL （：P1_PRODUCT，'TOTAL'）和cu.parent = NVL（：P1_CUSTOMER，'TOTAL'）和t.parent = NVL（：P1_TIME，'CY2006'）和ch.level_name ='TOTAL'/ *将维度视图连接到立方体视图* / AND p.dim_key = f.product AND cu.dim_key = f.customer AND t.dim_key = f.time AND ch.dim_key = f.channel
6.3.3.3添加到维度列的链接

当仪表板用户单击交叉表中的链接维度键时，绑定变量的值会更改，从而导致交叉表也发生更改。向下钻取层次结构后，用户可以通过按“重置”按钮将显示恢复为其原始数据选择。要实现这些列链接，必须添加列链接并激活“重置”按钮。

要添加指向维列的链接：

  1. 打开页面定义。

  2. 在“区域”下，单击“ 报告”链接。

    将打开“报告属性”页面。

  3. 在“列属性”下，修改报告显示：

    • 清除要隐藏的列的“显示”复选框，例如DIM_KEY和PARENT列。

    • 设置“排序和排序顺序”复选框以对报告进行适当的排序。在此示例中，排序顺序为Product （1）， Customer （2）和Time （3）。

  4. 单击维列的“ 编辑”图标。

    将打开“列属性”页面。

  5. 在Column Link下，按如下方式定义链接：

    • 链接文本：选择维名称。

    • 页面：输入页码。

    • 名称：按照它们在报告中显示的顺序列出维度。Item是绑定变量的名称。值是正在定义的维度的DIM_KEY列或其他维度的PARENT列。

    图6-11显示了时间维度的链接定义。

  6. 单击应用更改 。

    “列属性”页面将关闭，您将返回“报告属性”页面。

  7. 在其他维度列上定义链接。

  8. 单击应用更改 。

    “报告属性”页面将关闭，您将返回“页面定义”。
