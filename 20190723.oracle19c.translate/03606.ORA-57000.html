97 ORA-57000至ORA-59801

ORA-57000：TimesTen IMDB错误： string

原因： TimesTen操作期间发生错误。

操作：在TimesTen错误文档中查找错误代码以进行诊断。

ORA-57005：TimesTen网格瞬态事务失败： string

原因：事务失败，因为暂时无法访问TimesTen Grid元素。

操作：回滚事务并立即重试。

ORA-57006：TimesTen网格事务失败： string

原因：事务失败，因为暂时无法访问TimesTen Grid元素。

操作：回滚事务，等待元素恢复，然后重试事务。元素恢复可能需要管理员操作。

ORA-57007：TimesTen网格瞬态语句失败： string

原因：语句失败，因为暂时无法访问TimesTen Grid元素。

操作：立即重试当前语句。

ORA-57008：TimesTen Grid语句失败： string

原因： TimesTen Grid元素暂时不可用。

操作：等待元素恢复并重试当前语句。元素恢复可能需要管理员操作。

ORA-59000：并非所有重新平衡阶段都已运行

原因：该组的先前重新平衡未运行所有阶段。

操作：有关详细信息，请参阅GV $ ASM_OPERATION。使用ALTER DISKGROUP REBALANCE命令运行所有重新平衡阶段。

ORA-59001：磁盘' string '必须在磁盘' string '之前处于联机状态。

原因：尝试联机包含另一个脱机磁盘中存在的旧数据副本的磁盘。

操作：按正确顺序联机磁盘。FORCE选项可用于覆盖此检查。

ORA-59002：由于ORA string ，实例string上的ASM磁盘组“ string ”的磁盘重新验证失败。

原因：磁盘重新验证失败可能是由于各种原因。在发现期间发信号通知ORA-15020是可能的。有关详细信息，请参阅警报日志。

操作：确保在发出重试之前，群集中的所有节点都可以看到正在运行的磁盘。

ORA-59003：数据库未准备拆分。

原因：拆分是在数据库上发出的，该数据库未准备好或上次未完成。

操作：在对数据库发出拆分之前准备数据库，或删除现有镜像副本，然后重试拆分镜像操作。

ORA-59004：没有为组string运行重新平衡。

原因：尝试修改未运行的重新平衡的属性。

操作：发出不带MODIFY关键字的ALTER DISKGROUP REBALANCE命令以安排另一个重新平衡。

ORA-59005：此平台不支持Oracle ASM IOServer功能。

原因：此平台不支持ASM IOServer功能。

操作：请参阅文档以获取支持Oracle ASM IOServer功能的平台列表，并考虑转移到其中一个平台。

ORA-59006：拒绝来自陈旧数据库实例的连接请求。

原因：数据库实例尝试从陈旧的IOServer集群进行故障转移。

操作：重新启动数据库实例以启动新连接。

ORA-59007：无法从客户端找到可用的网络

原因：数据库实例未使用任何已配置的ASM网络。

操作：检查数据库实例中的ASM网络配置。

ORA-59008：所有已配置的ASM网络都不可用。

原因：所有已配置的ASM网络都遇到错误。ASM IOServer无法继续。

操作：检查已配置的ASM网络接口的状态。在接口运行后重新启动ASM IOServer实例。

ORA-59009：ASM IOServer实例尚未准备好接受客户端。

原因： ASM IOServer实例尚未准备好接受客户端。

操作：为ASM IOServer实例执行“alter system enable client access”以接受客户端。

ORA-59010：无法SHUTDOWN ASM IOServer实例与连接的客户端string 。

原因：向具有一个或多个连接客户端的ASM IOServer实例发出了SHUTDOWN命令。

操作：停止正在使用此ASM IOServer实例的所有客户端，然后重试SHUTDOWN命令。或者，使用SHUTDOWN ABORT命令强制断开与所有客户端的连接。

ORA-59011：组string操作失败，因为有太多磁盘脱机

原因：磁盘组中有太多磁盘脱机。

操作：确保磁盘处于联机状态并重试该操作。

ORA-59012：磁盘％0！s使用无效的物理扇区大小（％1！S）。

原因：磁盘使用的物理扇区大小值不是标准值，或者当前不受支持。

操作：磁盘不能用于ASM磁盘组。

ORA-59013：磁盘％0！s使用无效的逻辑扇区大小（％1！S）。

原因：磁盘使用的逻辑扇区大小值不是标准的或当前不受支持。

操作：磁盘不能用于ASM磁盘组。

ORA-59014：档案％0！s在磁盘组％1中！s没有准备拆分

原因：指定的文件未准备拆分。

操作：检查附带的错误消息，以获取有关失败原因的更多信息。

ORA-59015：禁用Oracle ASM清理。

原因：尝试执行Oracle ASM清理但已禁用。

操作：启用Oracle ASM清理。

ORA-59016：由于符号无效，无法装入磁盘组string

原因：尝试对名称中包含无效字符的磁盘组执行装入操作。从版本12.2.0.2.0开始，磁盘组名称应仅包含字母数字字符和下划线字符。

操作：使用'renamedg'工具将磁盘组名称更改为有效名称。然后，通过以受限模式挂载磁盘组并使用ASM SQL命令“ALTER DISKGROUP name RENAME DISKS ALL”更改磁盘名称。

ORA-59017：Oracle ASM清理正在停止先前的操作。

原因：如果前一个擦除操作正在停止，则系统无法接受新的清理操作。

操作：等到上一次Oracle ASM清理操作完全停止，然后再启动新操作。

ORA-59018：Oracle ASM清理要求分配单元大小至少为1 MB。

原因：由于分配单元小于1 MB，因此未执行该操作。

操作：在分配单元至少为1 MB的磁盘组上运行该操作。

ORA-59019：无法启动SCRB进程

原因：启动期间ASM清理主站（SCRB）进程失败。

操作：查看警报日志以找出进程启动失败的原因。

ORA-59020：丢弃镜像副本'％0！s'在磁盘组'％1中！s'已经在进行中

原因：已在指定的文件组上发出了丢弃镜像副本。

操作：等待上一个丢弃镜像副本完成。检查ASM警报日志或使用V $ ASM_FILEGROUP视图检查是否删除了相应的文件组。

ORA-59021：删除子文件'％0！s'在磁盘组'％1中！s'是不允许的

原因：由于尝试删除子文件，操作失败。

操作：子文件只能作为丢弃镜像复制操作的一部分一起删除。

ORA-59022：丢弃镜像副本'％0！s'在磁盘组'％1中！由于准备工作尚未完成，因此失败了

原因：镜像副本的准备工作尚未完成

操作：等待准备镜像副本完成。检查ASM警报日志或使用V $ ASM_FILEGROUP视图检查文件组的拆分状态。

ORA-59023：命令需要属性'％0！s'设置为'％1！S'

原因：磁盘组仍在运行必要的后台操作，导致属性name无法设置为value ，这是该命令的先决条件。

操作：确认磁盘组中的所有磁盘都处于联机状态，并等待该属性设置为正确的值。如果磁盘组中的任何磁盘脱机，则可能无法将该属性设置为正确的值。可能还要等一下。

ORA-59024：丢弃镜像副本'％0！s'在磁盘组'％1中！s'因文件拆分完成而失败

原因：镜像副本已完成分割。

行动：没有

ORA-59025：为数据库'％0创建镜像副本！'失败了

原因：尝试操作时存在数据库的另一个镜像副本，或者在与ASM实例通信时发生故障。

操作：如果存在具有PREPARED状态的镜像副本，则使用现有镜像副本创建另一个数据库或删除现有镜像副本。如果镜像副本存在FAILED状态，则删除镜像副本。此后，重试该操作。

ORA-59026：删除文件组'％0！s'在磁盘组'％1中！s'因文件组准备了镜像副本而失败

原因：不允许使用准备好的镜像副本删除文件组。

操作：删除所有准备好的镜像副本，然后重试。

ORA-59027：存在镜像副本时无法修改文件组冗余

原因：存在数据库的镜像副本或磁盘组仍在运行必要的后台操作，从而阻止文件组冗余更改。

操作：使用现有镜像副本创建另一个数据库，或删除现有镜像副本并等待重新平衡操作完成。然后重试该操作。

ORA-59028：数据库'％0上的最后一次镜像拆分操作后，重新平衡未完成！S'

原因：上次镜像拆分操作后的重新平衡仍在运行或尚未安排。

操作：如果已经运行重新平衡，则允许它完成，否则发出新的重新平衡。

ORA-59029：％0中的所有ASM磁盘！已经在线或正在上网。

原因：尝试将已联机或已联机的磁盘联机。

操作：在更改ONLINE语句之前检查磁盘的状态，以排除故障组或磁盘组中的磁盘。

ORA-59030：磁盘组'％0！s'包含文件组'％1！s'有太多的脱机磁盘

原因：镜像拆分操作失败，因为包含文件的磁盘组的联机磁盘数量不足。这可能会阻止打开目标数据库中的某些文件。操作提前失败，使管理员有机会修复错误。

操作：确保磁盘组在线具有所需数量的磁盘，然后重试该命令。

ORA-59031：无法与镜像副本'％0上的其他镜像拆分操作同时运行操作！s'在磁盘组'％1中！S'

原因：另一个冲突的镜像分割操作已在运行。

操作：在镜像拆分操作冲突完成后重试。

ORA-59032：在最后一次镜像拆分或更改冗余操作后，重新平衡未完成

原因：上次镜像拆分操作或更改冗余后的重新平衡操作仍在运行或尚未调度。

操作：如果重新平衡已在运行，则允许其完成;否则，发出重新平衡操作。

ORA-59033：无法启动Oracle ASM实例，因为它无法打开SPFILE'％0！S'

原因：尝试启动Oracle ASM实例失败，因为无法打开ASM服务器参数文件（SPFILE）。SPFILE在指定的磁盘组中配置。磁盘组中的一个或多个磁盘无法访问。这可能是由于远程节点或节点处于非活动状态。

操作：发出'olsnodes'命令以检索集群中的节点列表。启动群集中先前处于非活动状态的所有远程节点。在此节点上发出'crsctl start cluster'命令，并验证Oracle ASM实例是否处于活动状态。

ORA-59035：I / O期间出现HARDCHECK错误

原因： I / O操作期间ASM重新平衡（ARB）进程遇到硬件辅助弹性数据（HARD）检查错误。

操作：有关详细信息，请参阅警报日志。重新连接到数据库或ASM实例。如果进程终止导致实例终止，请重新启动实例。

ORA-59036：脱机磁盘阻止数据重建以实现奇偶校验冗余。

原因：尝试从奇偶校验冗余重建数据失败，因为磁盘组中的某些磁盘处于脱机状态。

操作：使磁盘组中的所有磁盘联机，以便从奇偶校验冗余中重建数据。

ORA-59037：丢弃镜像副本'％0！s'在磁盘组'％1中！'失败了

原因：尝试删除不存在的镜像副本。

操作：检查V $ ASM_DBCLONE_INFO以获取有效镜像列表。确保存在指定的镜像副本，然后重试该操作。

ORA-59038：找到的镜像副本太多

原因：尝试创建已具有最大镜像副本数的数据库的镜像副本。

操作：删除或拆分现有镜像副本并重试操作。

ORA-59039：Oracle ASM清理已在运行。

原因：当另一个清理操作仍在运行时，尝试发出ASM清理操作。

操作：等待运行操作完成，然后重试。或者，使用ALTER DISKGROUP DG SCRUB STOP停止运行操作。

ORA-59040：ASM文件属性已更改。

原因：尝试发出ASM清理操作，并且有一个文件更改了一个属性。

操作：等待ASM清理操作自动重试操作。

ORA-59041：数据库'％0！s'在多个磁盘组中有数据文件

原因：在数据库中发出镜像拆分操作，该数据库的数据文件位于多个磁盘组中。

操作：通过将所有数据文件移动到单个磁盘组或删除其他磁盘组的数据文件，确保所有数据文件都位于同一磁盘组中。然后，重试该操作。

ORA-59042：从'％0创建数据库！s'使用镜像副本'％1！s'在磁盘组'％2中！'失败了

原因：尝试从不存在的镜像副本创建数据库。

操作：连接到数据库并检查V $ ASM_DBCLONE_INFO以获取有效镜像列表。确保已创建指定的镜像副本，然后重试该操作。

ORA-59300：网站名称丢失或无效

原因：该命令未指定有效的站点名称。

操作：指定有效的站点名称。

ORA-59301：SITE令牌使用不正确

原因：为SQL语句中的某些磁盘提供了站点名称，而其他磁盘则缺少站点名称。

操作：在SQL语句中指定所有磁盘的站点名称或不指定站点名称，然后重试该操作。

ORA-59302：属性string必须是string或更高。

原因：尝试将属性设置为低于支持的最小值的版本。

操作：将属性设置为至少支持的最低版本。或者，降低最低支持版本，然后重试。

ORA-59303：正在安装的磁盘组的属性string （ string ）应为string或更高。

原因：尝试装入兼容性低于支持的最小磁盘组。

操作：通过将磁盘组安装在受限模式下或降低最低支持兼容性来提升磁盘组兼容性，然后重试。

ORA-59304：ASM不支持外部和全局用户

原因：提供了ASM不支持的用户类型。

操作：检查Oracle ASM文档，表明ASM不支持外部和全局用户。

ORA-59305：正在安装的磁盘组与此软件版本不兼容。

原因：尝试使用与磁盘组不兼容的软件版本装入磁盘组。

ORA-59306：参数' string '无效。

ORA-59307：磁盘组冗余不是外部的。

ORA-59308：磁盘“ string ”无法分配给站点，因为磁盘组“ string ”包含群集同步服务投票文件。

ORA-59309：现有ASM磁盘已使用故障组名称“ string ”。

原因：指定的故障组名称已在此磁盘组中使用。

操作：指定其他故障组名称。

ORA-59311：指定的ASM实例“ string ”不存在。

原因：指定的ASM实例不存在或已终止。

操作：指定有效的ASM实例名称。

ORA-59312：无法将磁盘“ string ”分配给站点，因为它是Oracle Exadata磁盘。

ORA-59313：块的所有镜像都丢失，可能是由于多个存储故障。

操作：检查系统日志和存储日志，以确定导致丢失块的故障。

ORA-59314：操作不支持文件类型

原因：该操作不支持给定类型的文件。

ORA-59315：文件'％0不支持奇偶校验！''类型'％1！S'。

ORA-59316：由于缺少补丁string ASM软件与磁盘组string不兼容

ORA-59317：客户端％0！s软件与磁盘组％1不兼容！由于缺少补丁％2！小号

ORA-59320：客户端％0！s软件与磁盘组％1不兼容！补丁％2！小号

ORA-59321：ASM软件与磁盘组string补丁string不兼容

ORA-59322：数据库软件与磁盘组string补丁string不兼容

ORA-59325：无法在独占模式下启动多个ASM实例
