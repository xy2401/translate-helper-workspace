256.1 UTL_I18N概述

UTL_I18N服务为使用PL / SQL编写的应用程序提供了额外的全球化功能。

UTL_I18N PL / SQL包由以下类别的服务组成：

  • 各种数据类型的字符串转换函数。

  • 将文本字符串转换为字符引用的函数，反之亦然。

  • 在Oracle，Java和ISO语言和地区之间映射的函数。

  • 在Oracle，Internet Assigned Numbers Authority（IANA）和电子邮件安全字符集之间映射的函数。

  • 一种从Oracle语言名称返回Oracle字符集名称的函数。

  • 一个函数，返回Oracle字符集字符的最大字节数。

  • 执行脚本音译的函数。

  • 返回给定区域支持的ISO货币代码，本地时区和本地语言的函数。

  • 返回最合适的语言排序的函数，所有适用语言排序的列表，以及支持给定语言的本地区域。

  • 在Oracle完整和短语言名称之间映射的函数。

  • 返回给定语言和地域名称的语言翻译的函数。

  • 返回最常用时区列表的函数。

256.2 UTL_I18N安全模型

UTL_I18N包的功能既不读取数据库内容也不修改它们。这些函数仅对其参数进行操作和/或从NLS数据文件中检索静态国际化信息。默认情况下，包的执行权限被授予PUBLIC

256.4.1 ESCAPE_REFERENCE函数

此函数将文本字符串转换为其字符引用副本，以用于不属于当前文档使用的字符集的字符。

字符引用主要用于HTML和XML文档中，以独立于文档的编码来表示字符。字符引用可以以两种形式出现，数字字符引用和字符实体引用。数字字符引用指定字符的Unicode代码点值，而字符实体引用使用符号名称引用相同的字符。例如， &#xe5;是带有上面一个环的小写字母“a”的数字字符引用，而&aring;是同一个字符的字符实体引用。字符实体引用也用于转义特殊字符，例如， &lt;代表<（小于）符号。这是为了避免与标记语言中标记的开头混淆。

句法

UTL_I18N.ESCAPE_REFERENCE（str IN VARCHAR2 CHARACTER SET ANY_CS，page_cs_name IN VARCHAR2 DEFAULT NULL）RETURN VARCHAR2 CHARACTER SET str％CHARSET;

参数

表256-3 ESCAPE_REFERENCE函数参数

参数 描述

海峡

指定输入字符串

page_cs_name

指定文档的字符集。如果page_cs_name为NULL ，则数据库字符集用于CHAR数据，国家字符集用于NCHAR数据。

使用说明

如果用户指定了无效字符集或NULL字符串，则该函数返回NULL字符串。

例子

UTL_I18N.ESCAPE_REFERENCE（'hello <'|| chr（229），'us7ascii'）

这将返回'hello &lt; &#xe5;' 。

参数

表256-12 GET_TRANSLATION函数参数

参数 描述

参数

例子

256.4.12 MAP_CHARSET函数

此函数将字符集映射到另一个字符集。

它映射以下内容：

  • IANA字符集名称的Oracle字符集名称。

  • IANA字符集名称到Oracle字符集名称。

  • Oracle字符集设置为电子邮件安全字符集名称。

句法

UTL_I18N.MAP_CHARSET（charset IN VARCHAR2，上下文IN PLS_INTEGER DEFAULT GENERIC_CONTEXT，标志IN PLS_INTEGER DEFAULT ORACLE_TO_IANA）RETURN VARCHAR2;

参数

表256-13 MAP_CHARSET函数参数

参数 描述

字符集

指定要映射的字符集名称。映射不区分大小写。

上下文

GENERIC_CONTEXT | MAIL_CONTEXT

GENERIC_CONTEXT ：映射在Oracle字符集名称和IANA字符集名称之间。这是默认值。

MAIL_CONTEXT ：映射在Oracle字符集名称和电子邮件安全字符集名称之间。

旗

  • ORACLE_TO_IANA | IANA_TO_ORACLE如果设置了GENERIC_CONTEXT

    ORACLE_TO_IANA ：从Oracle字符集名称映射到IANA字符集名称。这是默认值。

    IANA_TO_ORACLE ：从IANA字符集名称映射到Oracle字符集名称。

  • MAIL_GENERIC | MAIL_WINDOWS如果设置了MAIL_CONTEXT

    MAIL_GENERIC ：在非Windows平台MAIL_GENERIC Oracle字符集名称映射到电子邮件安全字符集名称。

    MAIL_WINDOWS ：在Windows平台上从Oracle字符集名称映射到电子邮件安全字符集名称。

使用说明

电子邮件安全字符集是Oracle字符集，应用程序在提交电子邮件时通常使用该字符集。字符集通常用于将数据库字符集中的内容转换为电子邮件安全内容。要在邮件头中指定字符集名称，您应使用通过使用ORACLE_TO_IANA选项调用MAP_CHARSET函数获得的相应IANA字符集名称，并提供电子邮件安全字符集名称作为输入。

例如，没有电子邮件客户端识别WE8DEC字符集中的消息内容，其对应的IANA名称是DEC-MCS 。如果WE8DEC传递给MAP_CHARSET功能与MAIL_CONTEXT选项，则该函数返回WE8ISO8859P1 。其相应的IANA名称ISO-8859-1被大多数电子邮件客户端识别。

此示例中的步骤如下：

  1. 使用MAIL_CONTEXT | MAIL_GENERIC调用MAP_CHARSET函数MAIL_CONTEXT | MAIL_GENERIC选项，带有数据库字符集名称WE8DEC 。结果是WE8ISO8859P1 。

  2. 将存储在数据库中的内容转换为WE8ISO8859P1 。

  3. 使用ORACLE_TO_IANA | GENERIC_CONTEXT调用MAP_CHARSET函数ORACLE_TO_IANA | GENERIC_CONTEXT选项，带有电子邮件安全字符集WE8ISO8859P1 。结果是ISO-8859-1 。

  4. 提交电子邮件时，在邮件头中指定ISO-8859-1 。

如果找到匹配项，该函数将返回字符集名称。如果未找到匹配项或标志无效，则返回NULL 。

注意：

许多Oracle字符集可以映射到一个电子邮件安全字符集。没有将电子邮件安全字符集映射到Oracle字符集名称的函数。

例子

通用语境

UTL_I18N.MAP_CHARSET（ 'ISO-8859-1'，UTL_I18N.GENERIC_CONTEXT，UTL_I18N.IANA_TO_ORACLE）

这将返回'WE8ISO8859P1' 。

上下文

这将返回'WE8ISO8859P1' 。

256.4.23 VALIDATE_CHARACTER_ENCODING函数

此函数验证VARCHAR2，NVARCHAR2，CLOB和NCLOB数据的字符编码。验证基于VARCHAR2和CLOB数据的数据库字符集以及NVARCHAR2和NCLOB数据的国家字符集。

对于Unicode字符集，例如AL32UTF8，AL16UTF16，AL16UTF16LE，UTF8和UTFE，映射到以下Unicode代码点的任何字节序列都被视为无效：

  • 不成对的代理代码点

  • 非字符代码点

此外，对于AL32UTF8和UTF8字符集，任何不规则或非法的UTF-8字节序列都被视为无效。

VALIDATE_CHARACTER_ENCODING函数已重载。一个函数用于验证VARCHAR2和NVARCHAR2数据，而另一个函数用于验证CLOB和NCLOB数据。

  • 验证VARCHAR2和NVARCHAR2数据

    如果VARCHAR2或NVARCHAR2字节或其字节序列未映射到字符集中定义的任何字符，则该字节序列被视为无效。

  • 验证CLOB和NCLOB数据

    如果一个字节（在单字节数据库字符集的情况下）或一个字节对（在UTF-16编码的情况下与多字节数据库字符集一起使用）对应于编码，则对于字符集，LOB字符被认为是无效的LOB字符不映射到字符集中定义的任何字符。

句法

此函数验证VARCHAR2和NVARCHAR2数据：

UTL_I18N.VALIDATE_CHARACTER_ENCODING（数据输入VARCHAR2字符集ANY_CS）RETURN PLS_INTEGER;

此函数验证CLOB和NCLOB数据：

UTL_I18N.VALIDATE_CHARACTER_ENCODING（lob_loc IN CLOB CHARACTER SET ANY_CS）RETURN PLS_INTEGER;

参数

表256-25 VALIDATE_CHARACTER_ENCODING函数参数

参数 描述

数据

要验证的VARCHAR2或NVARCHAR2数据。

lob_loc

要验证的CLOB或NCLOB数据。

使用说明

此函数返回VARCHAR2或NVARCHAR2数据的第一个无效字节的偏移量。它返回CLOB或NCLOB数据的第一个无效字符的偏移量。如果字符数据中的所有字节都有效，则返回0。如果参数data或lob_loc的值为NULL ，则返回NULL 。

例子

此示例验证NVARCHAR2和CLOB数据的字符编码，其中数据库字符集为AL32UTF8，而国家字符集为AL16UTF16。

CREATE TABLE temp（col1 NVARCHAR2（20），col2 CLOB）;插入临时值（UNISTR（'foo \ D800bar'），UNISTR（'foo \ D800bar'））;承诺; SELECT UTL_I18N.VALIDATE_CHARACTER_ENCODING（col1）invalid_offset_column1，UTL_I18N.VALIDATE_CHARACTER_ENCODING（col2）invalid_offset_column2 FROM temp;

查询返回：

INVALID_OFFSET_COLUMN1 INVALID_OFFSET_COLUMN2 ---------------------- ---------------------- 7 4

这里，代理代码点U+D800无效。数字7作为INVALID_OFFSET_COLUMN1返回，因为对于col1 ， 'foo'在NVARCHAR2中以6字节编码，无效代码点U+D800在偏移7处开始。数字4作为INVALID_OFFSET_COLUMN2返回，因为对于col2 ， 'foo'在CLOB中的3个UTF-16代码点中编码，无效代码点U+D800在偏移4处开始。
