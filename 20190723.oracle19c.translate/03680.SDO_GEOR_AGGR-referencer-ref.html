9 SDO_GEOR_AGGR包参考

SDO_GEOR_AGGR包中包含用于在GeoRaster对象上执行高级大规模镶嵌，追加和虚拟镶嵌操作的子程序（函数和过程）。本章为每个子程序提供一个或多个示例的参考信息。

9.1 SDO_GEOR_AGGR.append

格式

SDO_GEOR_AGGR.append（targetGeoRaster IN OUT SDO_GEORASTER，sourceGeoRaster IN SDO_GEORASTER，sourcePyramidLevel IN NUMBER，appendParam IN VARCHAR2，bgValues IN SDO_NUMBER_ARRAY DEFAULT NULL）;

描述

将源GeoRaster对象追加到目标GeoRaster对象。必要时执行内部整流，公共点规则，间隙填充和色彩平衡。

参数

targetGeoRaster

要更新的GeoRaster对象。不能与sourceGeoRaster相同。（在更新之前，请务必复制此对象。）

sourceGeoRaster

要附加到targetGeoRaster GeoRaster对象。

sourcePyramidLevel

要附加到目标GeoRaster对象金字塔等级0的源GeoRaster对象的金字塔等级。如果为NULL，则使用金字塔等级0。

appendParam

逗号分隔的带引号的关键字=值对的字符串，用于指定操作的参数。它可以包含SDO_GEOR_AGGR.mosaicSubset参考部分中表9-1中的一个或多个关键字，除非在该表中指定始终忽略SDO_GEOR_AGGR.append的关键字。

如果指定了colorBalance的非默认值，则使用目标GeoRaster对象的统计信息作为参考在源GeoRaster对象上执行该值，并忽略以下关键字（如果指定）： maxVal ， minVal ， std和min 。

bgValues

用于填充部分空栅格块的背景值。仅当源GeoRaster对象具有空栅格块且当前操作导致部分为空的栅格块时才有用（请参阅空栅格块 ）。SDO_NUMBER_ARRAY对象中的元素数必须是一个（用于所有波段的相同填充值）或波段维度大小（每个波段的填充值分别不同）。例如， SDO_NUMBER_ARRAY(1,5,10)用1填充第一个波段，用5填充第二个波段，用10填充第三个波段。默认的bgValues为零（0）。

填充值必须是目标单元格深度指定的有效单元格值。

使用说明

注意：

在调用此过程之前，请确保复制targetGeoRaster对象，因为在此过程完成后，对此GeoRaster对象所做的更改可能不可逆。

源和目标GeoRaster对象必须具有相同数量的波段或层。如果源和目标GeoRaster对象的单元格深度不相同，则源GeoRaster对象的单元格值将展开或截断为目标GeoRaster对象的单元格深度。

必须阻止目标GeoRaster对象的栅格数据。

除了更新块的范围和数量以及删除统计信息之外，目标GeoRaster对象的元数据没有变化。

源GeoRaster对象将附加到目标GeoRaster对象金字塔等级0。目标GeoRaster对象的金字塔也会更新。

根据appendParam参数中定义的规则解析源和目标GeoRaster对象的重叠区域和间隙。

源GeoRaster对象可以位于或接触目标GeoRaster对象的任何一侧（也就是说，它不必位于右侧或底侧）。目标GeoRaster对象将相应地自动扩展。

例子

下面的例子追加与GeoRaster对象georid = 2与GeoRaster对象在georid = 1。 

声明gr1 sdo_georaster; gr1 sdo_georaster;开始从georaster_table中选择georaster到gr1，其中georid = 1用于更新;从Georaster_table中选择georaster到gr2，其中georid = 2; sdo_geor_aggr.append（gr1，gr2,0，null）;更新georaster_table set georaster = gr1其中georid = 1;承诺;结束; /

9.2 SDO_GEOR_AGGR.getMosaicExtent

格式

SDO_GEOR_AGGR.getMosaicExtent（inGeoRasters IN SYS_REFCURSOR，outSRID IN NUMBER DEFAULT NULL）RETURN SDO_GEOMETRY;

要么

SDO_GEOR_AGGR.getMosaicExtent（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，outSRID IN NUMBER DEFAULT NULL）RETURN SDO_GEOMETRY;

描述

计算并返回虚拟马赛克或GeoRaster对象集合的空间范围的最小边界矩形（MBR）。

参数

inGeoRasters

光标中的源GeoRaster对象。

georasterTableNames

包含源GeoRaster对象的表的名称（逗号分隔）。

georasterColumnNames

在与georasterTableNames的表名对应的表中以SDO_GEORASTER类型的列命名（逗号分隔）。

outSRID

输出GeoRaster对象的坐标系。必须为null或MDSYS.CS_SRS表的SRID列中的值。如果此参数为空（默认值），则使用4326（WGS 84（经度/纬度）坐标系的EPSG SRID值）。

使用说明

在计算虚拟马赛克或GeoRaster对象集合的空间范围时，此函数尝试使用每个GeoRaster对象的spatialExtent属性。如果spatialExtent属性为null，则根据对象的元数据计算GeoRaster对象的范围。

例子

以下示例显示了当虚拟镶嵌或GeoRaster对象集合是光标时如何获取空间范围。

声明cur sys_refcursor，从georaster_table_1 union选择georaster开始打开cur，georaster_table_2全部选择georaster; extent：= sdo_geor_aggr.getMosaicExtent（cur，26986）;关闭结束; /

以下示例显示如何通过指定表列名称来获取镶嵌范围。

从双重中选择sdo_geor_aggr.getMosaicExtent（'georaster_table_1，georaster_table_2'，'georaster，georaster'，26986）;

9.3 SDO_GEOR_AGGR.getMosaicResolutions

格式

SDO_GEOR_AGGR.getMosaicResolutions（inGeoRasters IN SYS_REFCURSOR，resolutionUnit IN VARCHAR2 DEFAULT NULL）RETURN SDO_RANGE_ARRAY ;;

要么

SDO_GEOR_AGGR.getMosaicResolutions（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，resolutionUnit IN VARCHAR2 DEFAULT NULL）RETURN SDO_RANGE_ARRAY;

描述

返回指定单位中虚拟马赛克或GeoRaster对象集合的分辨率范围。

参数

inGeoRasters

光标中的源GeoRaster对象。

georasterTableNames

包含源GeoRaster对象的表的名称（逗号分隔）。

georasterColumnNames

在与georasterTableNames的表名对应的表中以SDO_GEORASTER类型的列命名（逗号分隔）。

resolutionUnit

返回的分辨率范围的度量单位。如果指定，则它必须是格式为'unit= value '带引号的字符串，其中value是单位名称值（SDO_UNITS_OF_MEASURE表中的有效UNIT_OF_MEAS_NAME值）。如果未指定或为null，则返回的分辨率范围以meter为单位。

使用说明

返回的分辨率范围采用SDO_RANGE_ARRAY格式（SDO_RANGE（ min_x ， max_x ），SDO_RANGE（ min_y ， max_y ））。其中min_x ， max_x是x维度上的最小和最大分辨率， min_y ， max_y是y维度上的最小和最大分辨率。

SDO_RANGE_ARRAY类型定义为SDO_RANGE的VARRAY(1048576) OF SDO_RANGE 。 SDO_RANGE类型定义为：

名字空？输入----------------------------------------- -------- ------- LB号码UB号码

例子

以下示例获取虚拟马赛克或GeoRaster对象集合的空间分辨率，返回值以米为单位。

SELECT sdo_geor_aggr.getMosaicResolutions（'georatser_table_1，georaster_table_2'，'georaster，georaster'，'unit = meter'）FROM DUAL;

9.4 SDO_GEOR_AGGR.getMosaicStatistics

格式

SDO_GEOR_AGGR.getMosaicStatistics（inGeoRasters IN SYS_REFCURSOR，skipFactor IN NUMBER DEFAULT 1，genHistogram IN VARCHAR2 DEFAULT'FALSE'，outStatistics OUT SDO_NUMBER_ARRAYSET，outHistograms OUT SDO_GEOR_HISTOGRAM_ARRAY）;

要么

SDO_GEOR_AGGR.getMosaicStatistics（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，skipFactor IN NUMBER DEFAULT 1，genHistogram IN VARCHAR2 DEFAULT'FALSE'，outStatistics OUT SDO_NUMBER_ARRAYSET，outHistograms OUT SDO_GEOR_HISTOGRAM_ARRAY）;

描述

计算并返回虚拟马赛克或GeoRaster对象集合的估计统计信息。

参数

inGeoRasters

光标中的源GeoRaster对象。

georasterTableNames

包含源GeoRaster对象的表的名称（逗号分隔）。

georasterColumnNames

在与georasterTableNames的表名对应的表中以SDO_GEORASTER类型的列命名（逗号分隔）。

skipFactor

指示在计算期间要使用的每第n个图像的数字。例如，如果skipFactor为3，则源图像中的每个第三个图像都用于统计计算。skipFactor参数可用于减少大图像集的计算时间。默认值为1（即，使用每个图像;不跳过任何图像）。

genHistogram

String（ true或false ）是否返回虚拟镶嵌的估计直方图。默认值为false 。

outStatistics

返回的统计信息按源图像的波段顺序排列。对于每个波段，如果genHistogram为false ，则返回的统计数据为min，max，mean和std;但如果genHistogram为true ，则返回的统计数据为min，max，mean，std，median和mode。

例如，如果源图像有三个波段和

  • 如果genHistogram为false ，则ourStatistics参数的值为： sdo_number_arrayset(sdo_number_array(17.0, 255.0, 154.64, 71.29), sdo_number_array(14.0, 255.0, 155.86, 56.08), sdo_number_array(0.0, 255.0, 98.60, 73.08))

  • 如果genHistogram为true ，则ourStatistics参数的值为： sdo_number_arrayset(sdo_number_array(17.0, 255.0, 154.64, 71.29, 112.0, 255.0), sdo_number_array(14.0, 255.0, 155.86, 56.08, 143.0, 255.0), sdo_number_array(0.0, 255.0, 98.60, 73.08, 61.0, 0.0))

SDO_NUMBER_ARRAYSET类型定义为VARRAY(1048576) OF SDO_NUMBER_ARRAY 。

outHistograms

如果genHistogram是字符串true ，则outHistograms将包含返回的直方图作为SDO_GEOR_HISTOGRAM_ARRAY类型类型的数组，每个项目包含每个波段的直方图。

如果genHistogram是字符串false ，则outHistograms将为null。

使用说明

要计算源图像的统计信息，源图像的统计信息必须存在于GeoRaster对象的元数据中。

源图像必须具有相同数量的波段和相同的celldepth 。

celldepth小于或等于16bit非重叠源图像的返回统计量是准确的计算;否则，返回的统计数据是接近估计值。

例子

以下示例显示了在虚拟镶嵌或GeoRaster对象集合是游标时如何获取统计信息。skipFactor为1， genHistogram参数为false 。 

声明cur sys_refcursor，outStats sdo_number_arrayset：= null; outHists sdo_geor_histogram_array：= null;从georaster_table_1 union开始选择georaster的开放cur，从georaster_table_2选择georaster; sdo_geor_aggr.getMosaicStatistics（cur，1，'false'，outStats，outHists）;关闭结束; /

以下示例显示如何通过指定表列名来获取统计信息。skipFactor为5， genHistogram参数为true 。 

声明outStats sdo_number_arrayset：= null; outHists sdo_geor_histogram_array：= null;开始sdo_geor_aggr.getMosaicStatistics（'georaster_table_1，georaster_table_2'，'georaster，georaster'，5，'true'，outStats，outHists）;结束; /

9.5 SDO_GEOR_AGGR.getMosaicSubset

格式

SDO_GEOR_AGGR.getMosaicSubset（inGeoRasters IN SYS_REFCURSOR，numramidLevel IN NUMBER，outSRID IN NUMBER，outModelCoordLoc IN NUMBER，referencePoint IN SDO_GEOMETRY，cropArea IN SDO_GEOMETRY，polygonClip IN VARCHAR2，boundaryClip IN VARCHAR2，layerNumbers IN VARCHAR2，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit IN VARCHAR2，mosaicParam IN VARCHAR2，rasterBlob IN OUT NOCOPY BLOB，outArea OUT SDO_GEOMETRY，outWindow OUT SDO_NUMBER_ARRAY，storageParam IN VARCHAR2 DEFAULT NULL，bgValues IN SDO_NUMBER_ARRAY DEFAULT NULL）;

要么

SDO_GEOR_AGGR.getMosaicSubset（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，pyramidLevel IN NUMBER，outSRID IN NUMBER，outModelCoordLoc IN NUMBER，referencePoint IN SDO_GEOMETRY，cropArea IN SDO_GEOMETRY，polygonClip IN VARCHAR2，boundaryClip IN VARCHAR2，layerNumbers IN VARCHAR2，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit IN VARCHAR2，mosaicParam IN VARCHAR2，rasterBlob IN OUT NOCOPY BLOB，outArea OUT SDO_GEOMETRY，outWindow OUT SDO_NUMBER_ARRAY，storageParam IN VARCHAR2 DEFAULT NULL，bgValues IN SDO_NUMBER_ARRAY DEFAULT NULL）;

要么

SDO_GEOR_AGGR.getMosaicSubset（inGeoRasters IN SYS_REFCURSOR，numramidLevel IN NUMBER，elevationParam IN VARCHAR2，outSRID IN NUMBER，outModelCoordLoc IN NUMBER，referencePoint IN SDO_GEOMETRY，cropArea IN SDO_GEOMETRY，polygonClip IN VARCHAR2，boundaryClip IN VARCHAR2，layerNumbers IN VARCHAR2，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit IN VARCHAR2，mosaicParam IN VARCHAR2，rasterBlob IN OUT NOCOPY BLOB，outArea OUT SDO_GEOMETRY，outWindow OUT SDO_NUMBER_ARRAY，storageParam IN VARCHAR2 DEFAULT NULL，bgValues IN SDO_NUMBER_ARRAY DEFAULT NULL，referenceImage IN SDO_GEORASTER DEFAULT NULL，referenceValue1 IN SDO_NUMBER_ARRAY DEFAULT NULL，referenceValue2 IN SDO_NUMBER_ARRAY DEFAULT NULL， refHistograms IN SDO_GEOR_HISTOGRAM_ARRAY DEFAULT NULL）;

要么

SDO_GEOR_AGGR.getMosaicSubset（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，pyramidLevel IN NUMBER，elevationParam IN VARCHAR2，outSRID IN NUMBER，outModelCoordLoc IN NUMBER，referencePoint IN SDO_GEOMETRY，cropArea IN SDO_GEOMETRY，polygonClip IN VARCHAR2，boundaryClip IN VARCHAR2，layerNumbers IN VARCHAR2，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit在VARCHAR2中，mosaicParam IN VARCHAR2，rasterBlob IN OUT NOCOPY BLOB，outArea OUT SDO_GEOMETRY，outWindow OUT SDO_NUMBER_ARRAY，storageParam IN VARCHAR2 DEFAULT NULL，bgValues IN SDO_NUMBER_ARRAY DEFAULT NULL，referenceImage IN SDO_GEORASTER DEFAULT NULL，referenceValue1 IN SDO_NUMBER_ARRAY DEFAULT NULL，referenceValue2 IN SDO_NUMBER_ARRAY DEFAULT NULL，refHistograms IN SDO_GEOR_HISTOGRAM_ARRAY DEFAULT NULL）;

描述

对虚拟镶嵌或GeoRaster对象集合执行子集化。它为查询区域动态执行镶嵌操作，并在BLOB中即时返回所需的结果。必要时执行内部整流，公共点规则，间隙填充和色彩平衡。

参数

inGeoRasters

光标中的源GeoRaster对象。

georasterTableNames

包含源GeoRaster对象的表的名称（逗号分隔）。

有关在这些表中定义和使用MIN_X_RES $和MAX_X_RES $列的信息，请参阅使用说明和使用MIN_X_RES $和MAX_X_RES $提高查询性能 。

georasterColumnNames

在与georasterTableNames的表名对应的表中以SDO_GEORASTER类型的列命名（逗号分隔）。

pyramidLevel

用于操作的源GeoRaster对象的金字塔等级。未指定outResolutions参数时使用此参数;否则，源GeoRaster对象的金字塔等级由outResolutions参数确定。

elevationParam

包含高程参数average （平均表面高度）的字符串。此参数必须是包含keyword = value对的带引号的字符串（例如， 'average=800' ）。此参数指定输出GeoRaster对象的高程。如果此参数为null，则假定为average为0。

使用elevationParam参数要求输入的GeoRaster对象具有3D模型SRID和非零平均高度。

outSRID

输出GeoRaster对象的坐标系。必须为null或MDSYS.CS_SRS表的SRID列中的值。

outModelCoordLoc

一个值，指定由单元格表示的区域底部的模型位置： 0表示CENTER ， 1表示UPPERLEFT 。如果为null，则使用CENTER 。

referencePoint

SDO_GEOMETRY类型的点用作马赛克的参考点。如果指定了一个点，则镶嵌图像的左上角与参考点对齐，即， referencePoint与输出的左上角之间的距离将具有整数个像素。如果此参数为null，则参考点隐式使用cropArea ，或者cropArea为null时，输出范围的左上角。

cropArea

作物面积定义。如果SDO_GEOMETRY对象具有非空SRID，则SDO_GEOMETRY对象中指定的坐标位于模型空间中，并且源GeoRaster对象必须进行地理配准;否则，SDO_GEOMETRY对象中指定的坐标位于单元格空间中，源GeoRaster对象可以进行地理配准或非地理配准。如果polygonClip是FALSE的的MBR cropArea用于裁剪镶嵌数据。如果polygonClip为TRUE ，则的几何cropArea用于裁剪镶嵌数据。

polygonClip

字符串TRUE导致cropArea值用于裁剪镶嵌数据;字符串FALSE或空值会导致cropArea的MBR用于裁剪镶嵌数据。

boundaryClip

字符串TRUE或空值导致虚拟马赛克的边界用于剪切cropArea ;字符串FALSE导致虚拟马赛克外但在cropArea内的cropArea用背景值填充。

layerNumbers

标识要在其上执行操作或操作的逻辑层编号的字符串。使用逗号分隔值，使用连字符表示范围（例如，对于第2,3和4层，使用2-4 ）。如果未指定，则镶嵌结果包含与源GeoRaster对象相同数量的波段。

outResolutions

请求输出GeoRaster数据的分辨率。如果为null，则默认为第一个遇到的GeoRaster对象的分辨率。有关详细信息，请参阅用法说明

resolutionUnit

outResolutions参数的单位。如果为null，则默认值为输出SRID的单位。如果指定，则它必须是格式为“unit = value ”的带引号的字符串，其中value是单位名称值（SDO_UNITS_OF_MEASURE表中的有效UNIT_OF_MEAS_NAME值）。如果outResolutions为null，则忽略此参数。

mosaicParam

逗号分隔的带引号的关键字=值对的字符串，用于指定镶嵌参数。它可以包含SDO_GEOR_AGGR.mosaicSubset参考部分中表9-1中的一个或多个关键字。

rasterBlob

BLOB（二进制大对象）保存操作的结果。它必须存在或在执行操作之前已初始化。它通常是临时BLOB。

outArea

描述输出数据范围的几何对象。

outWindow

SDO_NUMBER_ARRAY对象，用于标识单元格空间中输出窗口的左上角和右下角的坐标。对于此功能，输出窗口的左上角始终为（0,0）。

storageParam

指定存储参数的字符串，如存储参数中所述 。如果此参数为null，则生成的GeoRaster对象具有与模型空间中第一个遇到的源GeoRaster对象（如果适用）或单元格空间相同的存储参数（ cellDepth ， interleaving和compression ）。

如果指定了pyramid=true ，则此过程将忽略该值，但对于SDO_GEOR_AGGR.mosaicSubset则不会。

如果指定了bitmapmask=true ，则此过程将忽略它。

bgValues

用于填充部分空的栅格块的背景单元格值（完整的空栅格块在没有填充的情况下保留为空）。它们用于填充由镶嵌操作产生的空白区域，例如剪裁多边形之外的区域或未被任何源图像覆盖的间隙区域。（另请参阅清空栅格块 。）SDO_NUMBER_ARRAY对象中的元素数必须是一个（用于所有波段的相同填充值）或波段维度大小（每个波段的填充值分别不同）。例如， SDO_NUMBER_ARRAY(1,5,10)用1填充第一个波段，用5填充第二个波段，用10填充第三个波段。默认的bgValues为零（0）。

填充值必须是目标单元格深度指定的有效单元格值。

referenceImage

指定色彩平衡操作期间使用的参考图像。如果为null，则默认为最接近输出镶嵌中心的源图像。

referenceValue1

指定颜色平衡操作期间使用的参考值。值的含义由颜色平衡方法确定。

referenceValue2

指定颜色平衡操作期间使用的参考值。值的含义由颜色平衡方法确定。

referenceHistograms

指定色彩平衡操作期间使用的参考直方图。

使用说明

该过程的前两种格式提供了基本的颜色平衡方法，并假设输出GeoRaster对象的高程为0。该过程的最后两种格式使用参考值或参考图像提供更高级的色彩平衡方法，并且它们允许您指定输出GeoRaster对象的高程值。

源GeoRaster对象必须准备好图像或栅格数据，以便可以镶嵌它们。也就是说，虚拟马赛克中的GeoRaster对象必须：

  • 不是地理参考和非参考对象的混合。要么对所有对象进行地理配准，要么没有任何对象进行地理配准。

  • 具有相同数量的层或带。对象的行和列维度大小没有限制。

如果要镶嵌的GeoRaster对象是地理参考的，则根据其地理配准信息将它们放在一起。如果GeoRaster对象未进行地理配准，则根据其ULTCOordinate值将它们放在同一位置。（ULTCoordinate在GeoRaster数据模型中有解释。）

生成的GeoRaster对象的空间参考元数据信息由outSRID和outResolutions参数确定。如果未指定outSRID ，则使用第一个遇到的源GeoRaster对象的SRID。如果未指定outResolutions则使用在指定金字塔等级（ pyramidLevel参数）处遇到的第一个源GeoRaster对象的空间分辨率。必须在所有源图像的元数据中设置空间分辨率。

如果任何源GeoRaster对象具有与outSRID不同的SRID或未被纠正，则会动态重新生成或纠正它，以使镶嵌的GeoRaster对象具有统一的SRID和空间分辨率值。

如果源GeoRaster对象重叠，则重叠区域的数据遵循mosaicParam参数中指定的规则。默认情况下，使用上次遇到的源GeoRaster对象的单元格值。

有助于输出镶嵌的源GeoRaster对象由cropArea参数空间选择， outResolutions参数以分辨率方式选择（分辨率选择）。

  • cropArea参数用于查询源GeoRaster对象的空间范围，以确定cropArea几何体所覆盖的GeoRaster对象;因此，应该在GeoRaster对象的spatialExtent属性上构建一个patial索引。

  • outResolutions参数用于查找源GeoRaster对象，以便outResolutions值在源GeoRaster对象的分辨率范围内。GeoRaster对象的空间分辨率范围由金字塔等级0处的分辨率和GeoRaster对象的顶部金字塔等级处的分辨率确定。它比金字塔等级0低半金字塔，高于顶金字塔等级的半金字塔等级。例如，如果GeoRaster对象在金字塔等级0处具有30米的分辨率并且在顶部金字塔等级5处具有960米的分辨率，则该GeoRaster对象的空间分辨率范围在22.5米（（30 + 30/2）/ 2之间）和1440米（（960 + 960 * 2）/ 2）。

使用outResolutions参数的分辨率选择可以通过在outResolutions参数中列出的表中定义列MIN_X_RES $和MAX_X_RES $（两者都为NUMBER）来georasterTableNames ，其中MIN_X_RES $和MAX_X_RES $指定最小和最大空间分辨率值，分别是源GeoRaster对象。（如果存在大量源GeoRaster对象，则应在MIN_X_RES $和MAX_X_RES $列上创建B树索引。）要利用此功能，必须使用带有georasterTableNames参数的过程格式。它还要求存储在MIN_X_RES $和MAX_X_RES $列中的分辨率值与存储在源GeoRaster对象的spatialExtent属性中的SRID单位具有相同的单位。

可以通过在mosaicParam参数中设置resFilter=false来关闭分辨率选择。关闭分辨率选择后，源GeoRaster对象将仅在空间上进行过滤。

如果源GeoRaster对象具有空栅格块或不覆盖整个区域，空区域填充了指定的值bgValues参数，或0，如果bgValues不指定参数。

为了使用colorBalance在选项mosaicParam ，你应该叫SDO_GEOR.generateStatistics对源图像生成图像的统计数据，并将它们存储在源图像的元数据。

如果所有源GeoRaster对象都是空白且具有相同的blankCellValue值，则生成的rasterBlob将填充blankCellValue值。

有关更多信息，请参阅虚拟马赛克 。

使用说明

该过程的前两种格式提供了基本的颜色平衡方法，并假设输出GeoRaster对象的高程为0。该过程的最后两种格式使用参考值或参考图像提供更高级的色彩平衡方法，并且它们允许您指定输出GeoRaster对象的高程值。

源GeoRaster对象必须准备好图像或栅格数据，以便可以镶嵌它们。也就是说，要镶嵌的GeoRaster对象必须：

  • 不是地理参考和非参考对象的混合。要么对所有对象进行地理配准，要么没有任何对象进行地理配准。

  • 具有相同数量的层或带。对象的行和列维度大小没有限制。

如果要镶嵌的GeoRaster对象是地理参考的，则根据其地理配准信息将它们放在一起。如果GeoRaster对象未进行地理配准，则根据其ULTCOordinate值将它们放在同一位置。（ULTCoordinate在GeoRaster数据模型中有解释。）

生成的GeoRaster对象的空间参考元数据信息由outSRID和outResolutions参数确定。如果未指定outSRID ，则使用第一个遇到的源GeoRaster对象的SRID。如果outResolutions没有指定，在指定的锥体水平（第一个遇到的源GeoRaster对象的空间分辨率pyramidLevel使用参数）。必须在所有源图像的元数据中设置空间分辨率。

如果任何源GeoRaster对象具有与outSRID不同的SRID或未被纠正，则会动态重新生成或纠正它，以使镶嵌的GeoRaster对象具有统一的SRID和空间分辨率值。

如果源GeoRaster对象重叠，则重叠区域的数据遵循mosaicParam参数中指定的规则。默认情况下，使用上次遇到的源GeoRaster对象的单元格值。

有助于输出镶嵌的源GeoRaster对象由cropArea参数空间选择， outResolutions参数以分辨率方式选择（分辨率选择）。

  • cropArea参数用于查询源GeoRaster对象的空间范围，以确定cropArea几何体所覆盖的GeoRaster对象。因此，应该在GeoRaster对象的spatialExtent属性上构建空间索引。

  • outResolutions参数用于查找源GeoRaster对象，以便outResolutions值在源GeoRaster对象的分辨率范围内。GeoRaster对象的空间分辨率范围由金字塔等级0处的分辨率和GeoRaster对象的顶部金字塔等级处的分辨率确定。它比金字塔等级0低半金字塔，高于顶金字塔等级的半金字塔等级。例如，如果GeoRaster对象在金字塔等级0处具有30米的分辨率并且在顶部金字塔等级5处具有960米的分辨率，则该GeoRaster对象的空间分辨率范围在22.5米（（30 + 30/2）/ 2之间）和1440米（（960 + 960 * 2）/ 2）。

使用outResolutions参数的分辨率选择可以通过在georasterTableNames参数中列出的表中定义列MIN_X_RES $和MAX_X_RES $（两者都为NUMBER）来georasterTableNames ，其中MIN_X_RES $和MAX_X_RES $分别指定最小和最大空间分辨率值，源GeoRaster对象。（如果存在大量源GeoRaster对象，则应在MIN_X_RES $和MAX_X_RES $列上创建B树索引。）要利用此功能，必须使用带有georasterTableNames参数的过程格式。它还要求存储在MIN_X_RES $和MAX_X_RES $列中的分辨率值与存储在源GeoRaster对象spatialExtent属性中的SRID单位具有相同的单位。

可以通过在mosaicParam参数中设置resFilter=false来关闭分辨率选择。关闭分辨率选择后，源GeoRaster对象将仅在空间上进行过滤。

如果源GeoRaster对象具有空栅格块或未覆盖整个区域，则镶嵌结果GeoRaster对象可能具有空或部分空的栅格块（请参阅空栅格块 ）。未被任何源GeoRaster对象覆盖的结果栅格块保持为空。任何部分空光栅块被填充在规定的值bgValues参数，或者与0，如果bgValues未指定参数。

如果在storageParam字符串中将bitmapmask参数设置为true ，则将镶嵌位图蒙版镶嵌。默认情况下，位图掩码不是mosaciked。注意，位图掩码也可以被认为是NODATA;如果是，请参阅表9-1中的NODATA关键字。

为了使用colorBalance在选项mosaicParam ，你应该你应该叫SDO_GEOR.generateStatistics对源图像生成图像的统计数据，并将它们存储在源图像的元数据。

如果所有源GeoRaster对象都是空白且具有相同的blankCellValue值，则生成的GeoRaster对象为空并且具有该blankCellValue值;否则，生成的GeoRaster对象不为空。

包含镶嵌操作结果的GeoRaster对象（ georaster参数）不能是任何源GeoRaster对象（执行镶嵌操作的对象）。

镶嵌操作定期执行内部提交操作，因此无法回滚。如果操作中断，则栅格数据表中可能存在悬空栅格块。您可以通过在数据库中维护GeoRaster对象和系统数据来处理悬空栅格块，如在数据库中维护GeoRaster对象和系统数据中所述 。

有关更多信息，请参阅大规模图像 镶嵌和虚拟马赛克 。

9.7 SDO_GEOR_AGGR.validateForMosaicSubset

格式

SDO_GEOR_AGGR.validateForMosaicSubset（inGeoRasters IN SYS_REFCURSOR，outSRID IN NUMBER，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit IN VARCHAR2，resultTableName IN VARCHAR2）;

要么

SDO_GEOR_AGGR.validateForMosaicSubset（georasterTableNames IN VARCHAR2，georasterColumnNames IN VARCHAR2，outSRID IN NUMBER，outResolutions IN SDO_NUMBER_ARRAY，resolutionUnit IN VARCHAR2，resultTableName IN VARCHAR2）;

描述

检查是否可以对虚拟马赛克或大量GeoRaster对象执行镶嵌或子集查询操作。任何验证错误和注释都存储在用户创建的结果表中。

参数

inGeoRasters

光标中的源GeoRaster对象。

georasterTableNames

包含源GeoRaster对象的表的名称（逗号分隔）。

georasterColumnNames

在与georasterTableNames的表名对应的表中以SDO_GEORASTER类型的列命名（逗号分隔）。

outSRID

输出GeoRaster对象的坐标系。必须为null或MDSYS.CS_SRS表的SRID列中的值。

outResolutions

输出GeoRaster数据的分辨率。如果为null，则默认为第一个遇到的源GeoRaster对象的分辨率。

resolutionUnit

outResolutions参数的单位。如果为null，则默认值为输出SRID的单位。如果指定，则它必须是格式为“unit = value ”的带引号的字符串，其中value是单位名称值（SDO_UNITS_OF_MEASURE表中的有效UNIT_OF_MEAS_NAME值）。如果outResolutions为null，则忽略此参数。

resultTableName

验证结果表的名称。此表必须已存在，并且必须具有以下列定义：

time timestamp，type varchar2（16），description varchar2（512），table_name varchar2（32），column_name varchar2（1024），rdt_table_name varchar2（32），raster_id number

使用说明

以下注意事项适用于resultTableName参数值：

  • 如果指定的表不存在，则会生成错误。

  • 如果未指定参数或将其指定为null，则该过程会在找到的第一个验证错误时抛出错误;否则，该过程将所有验证错误放在表中并完成而不会出错。

  • 如果指定的表不为空，则该过程将行附加到表中的现有数据;如果任何列上存在唯一约束，并且新附加的数据与该约束列中的现有数据具有相同的值，则会生成错误。

  • 表的TYPE列包含一个表示问题类型的字符串，例如ERROR （必须修复的内容）或NOTE （可能需要或可能不需要某些操作的信息）。DESCRIPTION列提供有关该问题的详细信息。

此过程执行以下验证检查：

  • 源GeoRaster对象必须具有相同的带尺寸。

  • 源GeoRaster对象必须具有一致的地理配准状态，即全部都是地理配准或全部都没有地理配准。

  • 如果在进行马赛克时要进行重投或整改，则操作必须可行。

例子

以下示例检查是否可以进行马赛克操作。任何验证错误都存储在预定义表MOSAIC_ERROR中。

EXECUTE sdo_geor_aggr.validateForMosaicSubset（'georaster_table_1，georaster_table_2'，'georaster，georaster'，26986，sdo_number_array（30,30），'unit = meter'，'mosaic_error'）; - 检查验证结果：SELECT table_name table，column_name column，rdt_table_name rdt，raster_id rid，type，description FROM mosaic_error ORDER BY time;表格列RDT RID类型说明---------- -------- ----- --- ----- ------------- -------------------------- GEORASTER_1 GEORASTER RDT_1 2 ERROR源georaster对象未进行地理配准。
