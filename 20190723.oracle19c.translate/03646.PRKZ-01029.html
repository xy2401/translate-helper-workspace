137 PRKZ-01029至PRKZ-01233

PRKZ-01029：已禁用HAVIP {0}

原因： '禁用havip'命令失败，因为已禁用HAVIP。

操作：如果这是预期的状态，那么这不是错误。如果不是，则可以通过'srvctl enable havip'命令启用HAVIP。

PRKZ-01030：导出文件系统{0}已被禁用

原因： 'disable exportfs'命令失败，因为它已被禁用。

操作：如果这是预期的状态，那么这不是错误。如果不是，则可以通过'srvctl enable exportfs'命令启用它。

PRKZ-01031：HAVIP {0}已启用

原因： '启用havip'失败，因为已启用HAVIP。

操作：如果这是预期的状态，那么这不是错误。如果不是，则可以通过'srvctl disable havip'命令禁用HAVIP。

PRKZ-01032：已启用导出文件系统{0}

原因： 'enable exportfs'命令失败，因为它已经启用。

操作：如果这是预期的状态，那么这不是错误。如果不是，则可以通过'srvctl disable exportfs'命令禁用它。

PRKZ-01058：地址{0}存在HAVIP

原因：尝试添加HAVIP发现它已经存在

操作：检查输入，如果需要修改HAVIP，请使用命令'srvctl modify havip'。

PRKZ-01059：ID为{0}的HAVIP不存在

原因： 'add exportfs'失败，因为具有指定ID的HAVIP不存在。

操作：指定存在的HAVIP的ID，或使用命令'srvctl add havip -id id -address vip '为指定的ID创建HAVIP。

PRKZ-01060：导出文件系统{0}不存在

原因：具有指定名称的导出文件系统资源不存在。

操作：检查名称，然后使用已配置的导出文件系统名称重试。

PRKZ-01062：无法确定要修改为网络类型“{0}”的子网。网络资源有两个配置了网络类型“{1}”的子网。

原因：命令'srvctl modify network -nettype'失败，因为网络同时具有IPv4和IPv6子网，因此无法确定要修改的子网。

操作：使用命令'srvctl modify network -nettype'指定'-subnet'选项，以指示是否更改IPv4或IPv6子网的类型。

PRKZ-01065：指定的选项组合无效

原因： '-name'和'-id'选项不能组合在一起。

操作：使用'-name'或'-id'选项发出命令。

PRKZ-01066：无效的子网字符串：“{0}”

原因：给定的子网字符串不是点分十进制格式的有效IPv4或IPv6子网号。

操作：确保子网字符串是点分十进制格式的有效IPv4或IPv6子网号。

PRKZ-01067：无效的子网掩码字符串：“{0}”

原因：给定的子网掩码字符串不是点分十进制格式或IPv6前缀长度的有效IPv4子网掩码。

操作：确保子网掩码字符串是点分十进制格式或IPv6前缀长度的有效IPv4子网掩码。

PRKZ-01068：Windows操作系统不支持{0}对象

原因：在命令行上指定的object不是Windows操作系统上的有效对象。

操作：对象rhpserver | rhpclient | havip | exportfs在Windows操作系统上无效。使用有效的对象。

PRKZ-01069：“srvctl modify network -iptype both”不能包含{0}选项

原因：命令'srvctl modify network -iptype''request'包含冲突的选项。

操作：发出命令'srvctl modify network -iptype'与其他网络更改分开。

PRKZ-01070：将网络类型从{0}更改为{1}需要新的VIP地址

原因：当IPv4或IPv6网络类型为动态时，发出命令'srvctl modify nodeapps -nettype mixed'而未指定VIP地址。

操作：发出命令'srvctl modify nodeapps -nettype mixed -address'以指定新的VIP地址。

PRKZ-01071：如果没有'-address'选项，则无法提供'-netnum'选项

原因：发出了命令'srvctl modify havip -netnum'而未指定与新网络子网匹配的VIP地址。

操作：使用'-address'选项和'-netnum'选项重试命令'srvctl modify havip'以指定HAVIP资源的新地址。

PRKZ-01072：SCAN名称“{0}”已在网络{1}上注册

原因：为已在指定网络上注册的单客户端访问名称（SCAN）名称发出了命令'srvctl add scan -scanname'。

操作：使用不同的SCAN名称。

PRKZ-01073：无法将网络类型修改为“混合”，因为未配置GNS。

原因：尝试将网络类型修改为“混合”失败，因为未配置网格命名服务（GNS）。

操作：配置网格命名服务（GNS）并重试“srvctl modify network”命令。

PRKZ-01074：无法修改网络类型，因为指定的子网“{0}”与任何已配置的子网都不匹配。

原因：指定的子网与网络资源中任何已配置的子网都不匹配。

操作：指定已配置的子网以修改其网络类型。

PRKZ-01075：'-iptype'选项不能与'-subnet'一起提供

原因：命令尝试修改网络IP类型及其子网。

操作：将'srvctl modify network'与'-subnet'或'-iptype'一起使用，而不是两者都使用。

PRKZ-01076：无法修改网络类型为“混合”的网络

原因：尝试修改网络类型为“混合”的网络资源的子网或IP类型。

操作：在进行任何其他更改之前，使用'srvctl modify network -nettype'将网络类型修改为STATIC，DHCP或AUTOCONFIG。

PRKZ-01078：ASM客户端群集不支持对象{0}。

原因：在命令行上指定的object不是ASM客户端群集的有效对象。

操作： ASM客户端群集不支持对象ASM，DISKGROUP和VOLUME。使用有效的对象。

PRKZ-01079：'-invitedsubnets'选项的子网值{0}无效

原因：该命令被拒绝，因为使用不正确的语法指定了受邀子网值。

操作：使用无类别域间路由（CIDR）语法（例如192.196.16.0/24）或通配符重新发出指定受邀子网的命令。

PRKZ-01080：' - netnum'和'-vip'选项发生冲突。

原因： 'srvctl start | stop vip'命令被拒绝，因为'-netnum'选项与'-vip'选项一起被指定。

操作：使用命令'srvctl start | stop vip'和'-netnum'或'-vip'，而不是两者。

PRKZ-01082：无法解析VIP名称“{0}”。

原因：指定了无法解析的VIP名称。

操作：确保VIP名称解析为IP地址。

PRKZ-01083：VIP名称“{0}”解析为“{1}”，在任何VIP中均无法找到。

原因：尝试查找VIP名称已解析的指定地址之一的VIP失败。“

操作：指定解析为VIP资源中找到的地址的VIP名称。

PRKZ-01084：添加的指定子网（{0}）的网络类型不能设置为“混合”。

原因：尝试添加IPv4或IPv6子网并将其网络类型直接设置为“混合”。

操作：重新发出该命令，并为-nettype选项使用与“mixed”不同的值。

PRKZ-01085：内部错误：{0}

原因：发生内部错误。包含的值是内部标识符。

操作： Oracle内部错误。联系Oracle支持服务。

PRKZ-01119：无需修改

原因：未指定任何选项进行修改。

操作：使用'modify'命令指定至少一个选项。

PRKZ-01125：未配置Oracle Grid Infrastructure Management存储库（GIMR）客户端。

原因：尝试检索GIMR客户端信息失败，因为GIMR客户端不存在。

操作：添加GIMR客户端，然后重试。

PRKZ-01126：网络存储服务已被禁用。

原因：无法禁用网络存储服务器资源，因为它已被禁用。

行动：没有

PRKZ-01127：网络存储服务已启用。

原因：无法启用网络存储服务器资源，因为它已启用。

行动：没有

PRKZ-01128：无法访问主机{0}。

原因：命令'srvctl modify nodeapps -remoteservers hostname '指定了无法访问的主机名。

操作：确保主机已启动且可在网络上访问，然后重新发出该命令。

PRKZ-01129：主机{0}未知。

原因：命令'srvctl modify nodeapps -remoteservers hostname '指定了指定的主机名，但该主机未知。

操作：使用已知正确的主机名重新发出该命令。

PRKZ-01130：端口{0}无效。

原因：命令'srvctl modify nodeapps -remoteservers hostname ： port '指定了指示的端口，该端口不是有效端口。

操作：使用已知正确的端口重新发出该命令。

PRKZ-01131：仅在ASM客户端群集上支持{0}。

原因：尝试使用指定对象执行命令被拒绝，因为ASM客户端群集不支持指定的对象。

操作：使用有效对象ASM，磁盘组或卷。

PRKZ-01133：成员群集上管理存储库状态的无效查询

原因：拒绝从成员集群查询管理数据库的请求，因为只能在域服务集群上查询管理数据库状态。

操作：检查与管理数据库相关的命令的文档，并从相应的集群执行命令。

PRKZ-01143：无法停用数据库{0}，因为它已启用

原因：尝试将指示的数据库标记为“已停用”已被拒绝，因为数据库当前已启用。

操作：使用命令'srvctl disable database'禁用数据库，然后重试该命令。

PRKZ-01205：使用强制选项删除GNS VIP资源会导致节点{1}上继续可以访问GNS VIP地址{0}

原因：状态消息

行动：不是错误

PRKZ-01214：由于服务{0}已被识别为Java服务，因此在修补过程中无法启用服务。

原因：尝试启用基于Java的服务被拒绝，因为正在修补Oracle Java虚拟机（OJVM），并且在OJVM修补期间必须保持禁用Java服务。

操作：完成所有数据库实例的修补过程。

PRKZ-01217：ASM网络资源不存在。

原因：要操作的ASM网络资源不存在。

操作：可以使用命令'srvctl add asmnetwork'创建ASM网络资源。

PRKZ-01218：ASM网络资源已在节点上启动：{0}

原因：启动asmnetwork命令失败，因为ASM网络资源已启动。

行动：没有

PRKZ-01219：ASM网络资源已在节点上停止：{0}

原因：停止asmnetwork命令失败，因为ASM网络资源已停止。

行动：没有

PRKZ-01233：数据库{0}不是管理员管理的，不支持故障恢复选项。

原因：尝试使用“-failback”选项添加或修改服务被拒绝，因为此选项仅可用于管理员管理的数据库。

操作：重新发出不带'-failback'选项的命令。
