22 JSON的PL / SQL对象类型概述

PL / SQL对象类型允许细粒度编程构造和内存中JSON数据的操作。您可以对其进行内省，修改并将其序列化为文本JSON数据。

主要的PL / SQL JSON对象类型是JSON_ELEMENT_T ， JSON_OBJECT_T ， JSON_ARRAY_T和JSON_SCALAR_T 。另一个较少使用的对象类型是JSON_KEY_LIST ，它是VARCHAR2(4000)的varray。对象类型也称为抽象数据类型（ADT）。

这些JSON对象类型提供存储在数据库中的JSON数据的内存，分层（树状）编程表示。脚1

您可以使用对象类型以编程方式操作内存中的JSON数据，以执行以下操作：

  • 检查现有JSON数据的结构，类型或值。例如，检查给定对象字段的值是否满足某些条件。

  • 转换现有的JSON数据。例如，转换地址或电话号码格式以遵循特定约定。

  • 使用与数据表示的任何特征匹配的编程规则创建JSON数据。例如，如果要表示为JSON对象的产品是易燃的，则包括表示安全信息的字段。

PL / SQL对象类型实例是瞬态的 。要永久存储它们包含的信息，必须将它们序列化为VARCHAR2或LOB数据，然后可以将这些数据存储在数据库表中或编组到数据库客户机（如Java Database Connectivity（JDBC））。

您可以通过解析JSON文本或零碎的方式在内存中构建一个对象类型实例，从空对象或数组实例开始，并向其添加对象成员或数组元素。

未使用的对象类型实例会自动进行垃圾回收;你不能，也不必，释放你不再需要的实例所使用的内存。

通常，在构造PL / SQL对象类型实例并且可能以各种方式以编程方式使用它之后，将序列化为数据类型为VARCHAR2 ， CLOB或BLOB的实例。也就是说，您将内存中JSON数据的瞬态表示转换为数据库中的持久表示。（或者，您可以仅将其序列化为要打印的文本。）

JSON对象类型之间的关系

类型JSON_ELEMENT_T是其他JSON对象类型的超类型：它们中的每一个都将其扩展为子类型。子类型JSON_OBJECT_T和JSON_ARRAY_T分别用于JSON对象和数组。子类型JSON_SCALAR_T用于标量JSON值：字符串，数字，布尔值true和false以及值null 。

您只能通过解析JSON文本来构造JSON_ELEMENT_T类型的实例。解析创建一个JSON_ELEMENT_T实例，它是JSON数据的内存中表示。您不能构造JSON_ELEMENT_T类型的空实例或键入JSON_SCALAR_T 。

类型JSON_OBJECT_T和JSON_ARRAY_T各自具有与该类型相同名称的构造函数，您可以使用它来构造该类型的实例：JSON对象或数组的空（In-Memory）表示。然后，您可以根据需要填充此对象或数组，添加由PL / SQL对象类型实例表示的对象成员或数组元素。

您可以使用PL / SQL函数treat JSON_ELEMENT_T的实例JSON_ELEMENT_T转换为子类型实例。例如， treat(elt AS JSON_OBJECT_T)将实例elt强制转换为JSON对象（ JSON_OBJECT_T实例）。

解析函数

静态函数parse接受VARCHAR2 ， CLOB或BLOB类型的实例作为参数，它将其解析为JSON文本以返回类型为JSON_ELEMENT_T ， JSON_OBJECT_T或JSON_ARRAY_T 。

序列化方法

解析接受输入JSON数据作为文本并返回PL / SQL JSON对象类型的实例。序列化基本上相反：您将它应用于JSON数据的PL / SQL对象表示，并返回该对象的文本表示。序列化方法的名称以前缀to_开头。例如，方法to_string()返回您应用它的JSON对象类型实例的字符串（ VARCHAR2 ）表示。

大多数序列化方法都是成员函数。但是，对于作为CLOB或BLOB实例的序列化，有两种形式的方法：成员函数和成员过程。成员函数不接受任何参数。它创建一个临时LOB作为序列化目标。所述构件过程接受一个LOB IN OUT参数（ CLOB为方法实例to_clob ， BLOB为方法to_blob ）。因此，您可以将要用于序列化表示的LOB（可能为空）传递给它。

Getter和Setter方法

类型JSON_OBJECT_T和JSON_ARRAY_T具有getter和setter方法，它们分别获取和更新给定对象字段或给定数组元素位置的值。

getter方法有两种：

  • 方法get()返回对应用它的原始对象的引用，作为JSON_ELEMENT_T类型的实例。也就是说，应用它的对象通过引用传递 ：如果然后修改返回的JSON_ELEMENT_T实例，则修改将应用于您应用get()的原始对象。

  • 名称前缀为get_ getter方法返回在应用它们的对象或数组中定位的任何数据的副本 。该数据按值传递 ，而不是参考。

    例如，如果将方法JSON_OBJECT_T get_string()应用于JSON_OBJECT_T实例，并将给定字段作为参数传递，则会返回该字段值的字符串副本。如果将JSON_ARRAY_T get_string()应用于JSON_ARRAY_T实例，将给定元素位置作为参数传递，则会返回该数组中该位置的字符串副本。

与序列化方法一样，大多数getter方法都是成员函数。但是get_clob()和get_blob()方法返回给定对象字段的值或给定数组位置的元素作为CLOB或BLOB实例，有两种形式（如序列化方法to_clob()和to_blob() ） ：成员函数和成员过程 。成员函数不接受目标对象字段或数组位置以外的任何参数。它创建并返回一个临时LOB实例。所述构件过程还接受LOB IN OUT参数（ CLOB为get_clob ， BLOB为get_blob ）。因此，您可以将（可能为空的）LOB实例传递给它。

setter方法是put() ， put_null()和（仅适用于JSON_ARRAY_T ） append() 。它们更新应用它们的对象或数组实例，设置目标对象字段或数组元素的值。注意：setter方法修改现有实例 ，而不是返回它的修改副本。

方法append()在数组实例的末尾添加一个新元素。方法put_null()将对象字段或数组元素值设置为JSON null 。

方法put()需要第二个参数（除了对象字段名称或数组元素位置），这是要设置的新值。对于数组， put()也接受可选的第三个参数OVERWRITE 。这是一个BOOLEAN值（默认为FALSE ），表示是否替换给定位置的现有值 。

  • 如果对象已经具有相同名称的字段，则put() 将该值替换为新值。

  • 如果数组在给定位置已经有一个元素，则默认情况下， put()将该元素和任何连续元素向前移动（将它们的位置递增1），以便为放置在给定位置的新元素腾出空间。但是如果存在可选参数OVERWRITE并且为TRUE ，那么给定位置的现有元素将被新元素替换 。

内省方法

类型JSON_ELEMENT_T具有内省方法，可用于确定实例是JSON对象，数组，标量，字符串，数字还是布尔值，或者它是JSON值true ， false还是null 。这些方法的名称以前缀is_开头。它们是谓词，返回BOOLEAN值。

它还具有内省方法get_size()它返回一个的成员的数目JSON_OBJECT_T实例和的元素的数量JSON_ARRAY_T实例（它用于返回1 JSON_SCALAR_T实例）。

类型JSON_ELEMENT_T还具有内省方法is_date()和is_timestamp() ，它们测试实例是否表示日期或时间戳。JSON没有日期或时间戳的本机类型;这些通常表示使用JSON字符串。但是，如果使用SQL数据类型DATE或TIMESTAMP的SQL数据构造JSON_ELEMENT_T实例，则保留此类型信息以用于PL / SQL对象表示。

日期和时间戳数据使用PL / SQL对象类型JSON_SCALAR_T ，其实例不能直接构造。但是，您可以使用方法put()将此值添加到对象（作为字段值）或数组（作为元素put() 。使用方法get()检索它get()返回JSON_SCALAR_T实例。

类型JSON_OBJECT_T和JSON_ARRAY_T具有内省方法get_type() ，它返回目标对象字段或数组元素的JSON类型（作为VARCHAR2实例）。类型JSON_OBJECT_T还具有内省方法has() ，如果对象具有给定名称的字段，则返回TRUE ; get_keys() ，返回PL / SQL对象类型JSON_KEY_LIST的实例，该类型为varCHAR2类型的varray VARCHAR2(4000) 。varray包含给定JSON_OBJECT_T实例中存在的Foot 2 Footref 2字段的名称。

其他方法

类型JSON_OBJECT_T和JSON_ARRAY_T具有以下方法：

  • remove() - 使用给定字段或给定位置的数组元素删除对象成员。

  • clone() - 创建并返回应用该方法的对象或数组的（深层）副本。修改此副本的任何部分对原始对象或数组没有影响。

类型JSON_OBJECT_T具有方法rename_key() ，它重命名给定的对象字段。Footref 2如果提供的新名称已经为现有字段命名，则会引发错误。
