Oracle RAC环境中的Oracle Label Security Policy功能

在一个实例上进行的策略更改可立即用于Oracle Real Application Clusters（Oracle RAC）环境中的其他实例。

没有必要重新启动其他实例来获取更改。

在一个数据库实例上进行的重要更改会自动传播到其他实例。一个例子是创建一个新政策。另一个是改变政策选择。

传播这些变化确保了两个有价值的保护：

  • 该表的所有用户都遵循相同的策略

  • 如果任何实例失败，其他实例继续其工作将使用在该失败之前立即生效的相同策略和参数。因此，如果已启用或禁用策略，则在所有情况下都会将其视为此类策略。

如果管理员使用表D-1中列出的策略函数在一个实例中更改策略信息，Oracle Label Security将存储有关该函数调用更改的相关信息。新信息可立即供Oracle RAC中的其他活动实例使用，从而实现受影响策略的用户之间的一致性。

表D-1保留Oracle RAC环境中的状态的策略功能

政策职能 描述

SA_SYSDBA.CREATE_POLICY

创建新策略

SA_SYSDBA.DROP_POLICY

删除现有政策

SA_SYSDBA.ENABLE_POLICY

启用现有策略

SA_SYSDBA.DISABLE_POLICY

禁用现有策略

SA_SYSDBA.ALTER_POLICY

改变现有政策

Oracle Label Security中的透明应用程序故障转移

会话信息保留在透明应用程序故障转移上。

通过表D-2中列出的会话功能对会话信息的任何更改都将保留在透明应用程序故障转移上。

例如，假设用户Scott使用默认标签Top Secret登录。如果他调用sa_session.set_label（）将其会话标签更改为Secret ，并且发生故障转移到另一个实例，他将看不到任何更改，但他的会话标签仍为Secret 。

保留当前用户会话信息意味着对用户可以看到或影响的数据的访问权限和限制保持不变。尽管存在故障转移，但用户只能查看并影响故障转移之前可访问的表和行。如果不是保留，则故障转移到另一个实例可能导致或使用户能够看到不同的数据集。

只要使用表D-2中列出的某个会话功能，Oracle Label Security就会存储有关该函数调用所更改内容的相关信息。

表D-2会话功能在Oracle RAC环境中保留状态

会话功能 描述

SA_SESSION.SET_LABEL

允许用户设置他或她具有读取权限的新级别和新隔离专区和组

SA_SESSION.SET_ROW_LABEL

允许用户设置将应用于新行的默认行标签

SA_SESSION.SAVE_DEFAULT_LABELS

允许用户将当前会话标签和行标签存储为将来会话的默认值

SA_SESSION.RESTORE_DEFAULT_LABELS

允许用户将当前会话标签和行标签重置为存储的默认设置

SA_SESSION.SET_ACCESS_PROFILE

将Oracle Label Security授权和数据库会话的权限设置为指定用户的权限
