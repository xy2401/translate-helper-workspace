XS_ACL包

XS_ACL程序包创建用于创建和管理访问控制列表（ACL）的过程。

本节包括以下主题：

  • XS_ACL包的安全模型

  • 常量

  • 对象类型，构造函数，同义词和授权

  • XS_ACL子程序摘要

XS_ACL包的安全模型

XS_ACL包是在SYS架构下创建的。

DBA角色被授予ADMIN_ANY_SEC_POLICY权限，该权限允许它跨所有模式管理ACL，安全类和安全策略等模式对象。

如果已为模式授予RESOURCE角色，则用户可以在自己的模式中管理模式对象。RESOURCE角色和XS_RESOURCE应用程序角色包括管理模式中的模式对象所需的ADMIN_SEC_POLICY权限，以及在授予的模式中管理策略工件以在应用程序中实现策略管理。

如果已授予APPLY_SEC_POLICY权限，则用户可以管理架构上的策略实施。使用此权限，用户可以在授予的模式中管理策略实施，以在应用程序内实现策略管理。

CREATE_ACL过程

CREATE_ACL过程创建新的访问控制列表（ACL）。

句法

XS_ACL.CREATE_ACL（名称IN VARCHAR2，ace_list IN XS $ ACE_LIST，sec_class IN VARCHAR2：= NULL，parent IN VARCHAR2：= NULL，inherit_mode IN PLS_INTEGER：= NULL，description IN VARCHAR2：= NULL）;

参数

参数 描述

名称

要创建的ACL的名称。

该名称是模式限定的，例如， SCOTT.ACL1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为ACL1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.ACL1 。

ace_list

ACL中的访问控制条目（ACE）列表。

sec_class

指定ACL范围或类型的安全性类的名称。如果未指定安全性类，则DML类将用作默认安全性类。

该名称是模式限定的，例如， SCOTT.ACL1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为ACL1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.ACL1 。

亲

父ACL名称（如果有）。

该名称是模式限定的，例如， SCOTT.ACL1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为ACL1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.ACL1 。

inherit_mode

如果指定了父ACL，则为继承模式。允许的值为： EXTENDED或CONSTRAINED 。

描述

ACL的可选描述。

例子

以下示例创建名为HRACL的ACL。此ACL包含ace_list包含的ACE。ace_list中使用的权限是HRPRIVS安全性类的一部分。 

DECLARE ace_list XS $ ACE_LIST; BEGIN ace_list：= XS $ ACE_LIST（XS $ ACE_TYPE（privilege_list => XS $ NAME_LIST（'“SELECT”'，'VIEW_SENSITIVE_INFO'），授予=> true，principal_name =>'HRREP'），XS $ ACE_TYPE（privilege_list => XS $ NAME_LIST（'UPDATE_INFO'），grant => true，principal_name =>'HRMGR'））; SYS.XS_ACL.CREATE_ACL（name =>'HRACL'， ace_list => ace_list， sec_class =>'HRPRIVS'， description =>'HR Representative Access'）;结束;

APPEND_ACES程序

APPEND_ACES过程将一个或多个ACE添加到现有ACL。

句法

XS_ACL.APPEND_ACES（acl IN VARCHAR2，ace IN XS $ ACE_TYPE）; XS.ACL.APPEND_ACES（acl IN VARCHAR2，ace_list IN XS $ ACE_LIST）;

参数

参数 描述

ACL

要添加ACE的ACL的名称。

该名称是模式限定的，例如， SCOTT.ACL1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为ACL1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.ACL1 。

高手

要添加到ACL的ACE。

ace_list

要添加到ACL的ACE列表。

例子

以下示例将ACE添加到HRACL ACL。 ACE将SELECT权限授予DB_HR数据库用户。 

DECLARE ace_entry XS $ ACE_TYPE; BEGIN ace_entry：= XS $ ACE_TYPE（privilege_list => XS $ NAME_LIST（'“SELECT”'），grant => true，principal_name =>'DB_HR'，principal_type => XS_ACL.PTYPE_DB）; SYS.XS_ACL.APPEND_ACES（ 'HRACL'，ace_entry）;结束;

DELETE_ACL程序

DELETE_ACL过程删除指定的ACL。

句法

XS_ACL.DELETE_ACL（acl IN VARCHAR2，delete_option IN PLS_INTEGER：= XS_ADMIN_UTIL.DEFAULT_OPTION）;

参数

参数 描述

ACL

要删除的ACL的名称。

该名称是模式限定的，例如， SCOTT.ACL1 。如果缺少名称的模式部分，则假定当前会话模式。例如，在同一个示例中，如果名称指定为ACL1 ，并且当前架构为SCOTT ，则它将解析为SCOTT.ACL1 。

delete_option

要使用的删除选项。对于数据安全策略，以下选项的行为是相同的：

  • DEFAULT_OPTION：

    默认选项仅允许在其他地方未引用ACL时删除ACL。如果在其他地方引用了ACL，则无法删除ACL。

    例如，如果您尝试删除属于数据安全策略的ACL，则删除操作将失败。

  • CASCADE_OPTION ：

    级联选项删除ACL，并删除数据安全策略的数据域约束中的ACL引用。

  • ALLOW_INCONSISTENCIES_OPTION：

    allow inconsistencies选项允许您删除ACL，即使其他实体具有对它的后期绑定引用也是如此。在此模式下，将删除ACL，但不会删除引用。

例子

以下示例使用默认删除选项删除HRACL ACL：

BEGIN SYS.XS_ACL.DELETE_ACL（'HRACL'）;结束;
