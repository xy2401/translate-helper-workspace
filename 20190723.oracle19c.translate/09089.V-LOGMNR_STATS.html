8.57 V $ LOGMNR_STATS

V$LOGMNR_STATS显示当前由活动LogMiner持久会话执行的活动。

柱 数据类型 描述

SESSION_ID

数

LogMiner持久会话的唯一标识符

名称

VARCHAR2（64）

LogMiner统计信息，状态或状态的名称，包括：

  • 重做处理的字节数 - SQL Apply处理的累计字节数

  • 已处理的重做记录 - 由SQL Apply处理的重做记录的计数

  • Txns已传递给客户端 - SQL Apply处理的SQL事务计数

  • 传递DML txns - SQL Apply处理的DML事务计数

  • 传递DDL txns - SQL Apply处理的DDL事务计数

  • 交付的CTAS txns - 由SQL Apply处理的CREATE TABLE AS SELECT （CTAS）事务的计数

  • 递送的递归txns - 由SQL Apply处理的递归事务的计数

  • 回滚txns看到了

  • LCR交付给客户端 - SQL Apply处理的逻辑更改记录（LCR）数

  • 分页的字节数 - 已分页的累计字节数。LogMiner从LCR高速缓存中分页出内存，以适应某些不良行为的工作负载或配置不足的系统。分页的字节数与重做处理的字节数之比应该很低。如果此比率很高（10％或更高），请尝试增加分配给SQL Apply的MAX_SGA 。

  • Microsecs在页面输出中花费了 - LogMiner从LCR缓存中分页内存所花费的时间

  • 字节检查点 - 跟踪检查点的字节数。挖掘引擎采用定期检查点，从而写出与长时间运行的事务有关的逻辑变更记录（LCR）。Bytes Checkpointed的Bytes of Redo Processed的Bytes Checkpointed之比应该很低。高比率（10％或更高）表示工作负载不良。

  • Microsecs在检查点上花费 - 采矿引擎花费检查点所花费的时间，从而写出与长时间运行的事务有关的逻辑变更记录（LCR）。

  • 回滚的字节数 - LogMiner回滚的字节数的累积值。LogMiner有时需要回溯并重新处理重做流的一部分。在这种情况下，它将回滚它已经完成的工作。Bytes Rolled Back的Bytes of Redo Processed的Bytes Rolled Back之比应该很低。如果此比率很高（10％或更高），请减少分配给SQL Apply的PREPARER进程数。

  • Microsecs花在回滚上 - 用于回滚已经应用于逻辑备用数据库的事务的时间

相应的度量值

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
