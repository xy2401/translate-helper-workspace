升级和修补Oracle ASM

本节包含以下主题：

  • 关于Oracle ASM滚动升级

  • 关于Oracle ASM Rolling Patches

注意：

  • 对于Oracle RAC环境，Oracle Clusterware版本号必须至少等于要应用于Oracle数据库的修补程序的版本号。

  • 在将补丁应用于Oracle数据库主目录之前，必须将补丁应用于Oracle Grid Infrastructure主目录。

关于Oracle ASM滚动升级

通过Oracle ASM滚动升级，您可以独立升级或修补集群Oracle ASM节点，而不会影响数据库可用性，从而提供更长的正常运行时间。

滚动升级意味着当群集中的一个或多个节点使用不同的软件版本时，群集Oracle ASM环境的某些功能将继续运行。Oracle建议您在执行Oracle Clusterware滚动升级时执行Oracle ASM滚动升级。

要执行滚动升级，必须准备好您的环境。在启动Oracle ASM滚动升级之前，必须将Oracle Clusterware完全升级到下一个修补程序或发行版。此外，您应该以滚动升级方式准备Oracle Clusterware，以确保高可用性和最长的正常运行时间。

请注意，Oracle ASM随Oracle Clusterware for Oracle 11g第2版（11.2）或更高版本一起升级，因为它们都位于Oracle Grid Infrastructure主目录中。

注意：

  • 滚动升级仅适用于集群Oracle ASM实例，您只能对使用Oracle Database 11g或更高版本的环境执行滚动升级。您无法使用此功能从Oracle数据库10g升级到Oracle Database 11g 。

  • 有关在存在Oracle Exadata存储时执行Oracle ASM实例的滚动升级的信息，请参阅Oracle Exadata文档 。

关于Oracle ASM Rolling Patches

您可以在集群Oracle ASM环境中应用修补程序，以便一次将一个节点更新到最新的修补程序级别，而不会影响Oracle ASM集群或使用Oracle ASM进行存储的数据库集群的整体可用性。

ALTER SYSTEM ROLLING PATCH SQL语句使您可以启动和停止滚动补丁。例如：

SQL> ALTER SYSTEM START ROLLING PATCH; SQL> ALTER SYSTEM STOP ROLLING PATCH;

您可以通过对群集状态执行SYS_CONTEXT SQL查询来确定群集是否处于滚动修补程序模式。添加新状态（In Rolling Patch）以通知用户群集处于滚动补丁模式。

例3-2中的查询显示有关滚动补丁的信息。要运行这些查询，您必须连接到Grid home中的Oracle ASM实例，并且Grid Infrastructure主目录必须配置Oracle RAC环境的Oracle Clusterware选项。

通过查询V$PATCHES视图，可以查看节点和集群上应用的所有补丁ID。

用于滚动补丁的ASMCMD命令包括：

  • showclusterstate

  • showpatches

  • 的showversion

示例3-2确定滚动补丁模式和补丁级别

SELECT SYS_CONTEXT（'SYS_CLUSTER_PROPERTIES'，'CLUSTER_STATE'）FROM DUAL; SELECT SYS_CONTEXT（'SYS_CLUSTER_PROPERTIES'，'CURRENT_PATCHLVL'）FROM DUAL;
