在你开始之前

配置在线事务处理器以允许网关访问需要对OLTP和主机操作系统的某些组件执行操作。虽然主机系统上未安装Oracle软件，但仍需要访问主机系统和OLTP以及一些知识。虽然安装网关包含有关主机系统和OLTP安装步骤的一些信息，但您必须确保具有适用的OLTP和主机系统文档。

OLTP上的某些配置操作可能需要您重新启动OLTP。确保主机系统程序员或DBA在重新启动之前查看OLTP的说明。

使用单个安装和配置网关 Oracle数据库和单个OLTP，执行安装网关软件中描述的过程。

注意：

如果您的网关使用SNA通信协议 ，请按照本章， 配置Oracle网络和使用SNA通信协议的网关配置中的安装和配置说明进行操作。

如果您的网关使用TCP / IP通信协议 ，请按照本章， 配置Oracle网络和使用TCP / IP通信协议的网关配置中的安装和配置说明进行操作。

计划升级或迁移网关

本节仅适用于具有以前版本的Oracle Database Gateway for APPC的客户。如果您之前安装了网关，则需要先执行以下主题中提到的任务，然后才能安装Oracle Database Gateway for APPC的12 c第2版（12.2）。

注意：

阅读本节后，您必须阅读从现有网关迁移，以确定准备升级或迁移网关时必须执行的特定操作。如果要从版本4.01或更早版本迁移到Oracle Database Gateway for APPC 12 c第2版（12.2），那么您将找到与从现有网关迁移中迁移网关相关的内容。

如果您是第一次为APPC安装Oracle Database Gateway，请从“ 预安装步骤 ”开始。

本节包括以下主题：

  • 升级前程序

  • 升级和迁移注意事项

  • 恢复

升级前程序

执行以下步骤以准备将以前版本的Oracle Database Gateway for APPC升级到当前版本：

  1. 备份已更改的PGA已发送文件。

  2. 删除或重命名任何旧网关目录。

升级和迁移注意事项

升级注意事项如下：

  • PGAU 来自Gateway版本8或9的控制文件是向上兼容的，您无需更改它们。

  • 升级后，PG数据字典（PG DD ）包含所有早期条目而不进行修改。添加了新的PGAU控制信息以及一些列以支持新功能，但升级不会更改任何客户条目。

  • 所有必须重新编译Oracle Database Gateway for APPC版本4.0.1或更早版本的TIP，因为以下内容发生了更改：

    • PL / SQL兼容性

    • 网关服务器RPC接口

    • UTL_PG接口

  • 如果您以前使用SNA通信协议在网关上生成的现有TIP并且您希望使用新的TCP / IP功能，则必须由PGAU使用强制NLS_LANGUAGE和Side Profile Settings重新生成TIP。在DEFINE TRANSACTION命令中指定适当的ASCII字符集。

    这是因为网关假定正在使用IMS Connect中的用户出口，这将在ASCII和EBCDIC字符集之间进行转换。

    警告：

    升级了早期版本的PGAU无法访问PG数据字典（PG DD）。

恢复

如果你想恢复以前版本的网关，然后您必须将以下组件恢复到以前的版本：

  • PGAU

  • PG DD

  • 网关服务器

预安装步骤

在安装网关之前，请执行以下预安装程序：

  • 确保您的系统满足系统要求中指定的所有硬件和软件要求 。

  • 确保满足您的安全要求。

    有关OLTP上的连接和数据访问的安全要求的详细信息，请参阅系统要求 。

  • 填写工作表，确定配置系统和所选通信协议（SNA或TCP / IP）所需的唯一参数名称，该通信协议位于配置工作表中 。

  • 确定网关的SID（系统标识符）。 此SID用于配置网关 。

    SID必须是唯一的，不得由任何其他网关使用系统上的Oracle数据库。

  • 仅限SNA ：您必须先安装和配置您的SNA包，然后才能继续安装网关。确保您的系统可以使用适用于您的平台的SNA通信包与OLTP通信。

    有关设置和配置SNA通信包的更多信息，您的平台需要运行适用于APPC的Oracle数据库网关，请参阅以下列表中本指南中的相应章节：

    • 对于基于Intel Pentium的32位系统的Linux ，请参阅在Linux上配置SNA通信包 。

    • 对于基于AIX的系统，请参阅在基于AIX的系统上配置SNA通信包 。

    • 对于Solaris操作系统（SPARC 64位），请参阅在Solaris上配置SNA通信包 。

  • 仅限TCP / IP：您的必须先安装并配置TCP / IP包，然后才能继续安装网关。

    确保您的系统可以使用适用于您的平台的TCP / IP通信包与OLTP进行通信。

如果您需要有关安装Oracle产品和使用Oracle Universal Installer的一般信息，请参阅“ Oracle数据库安装指南” 。

网关安装方法

您可以通过以下任何方式安装网关：

  • 与现有系统相同 Oracle数据库但在不同的目录中。

    本节将讨论此类安装或升级的所有任务。

  • 在与本地Oracle数据库不同的系统上。

  • 与Oracle数据库位于同一系统上，并位于同一Oracle主目录中。请注意，在这种情况下，Oracle数据库和网关必须处于相同的发行版级别。

安装网关软件

有关安装Oracle产品以及如何使用Oracle Universal Installer的一般信息，请参阅“ Oracle数据库安装指南”并首先执行所有必要的任务。

如果服务器版本与网关版本不同， 请不要将网关安装在与Oracle数据库相同的Oracle主目录中。如果网关可执行文件与更高版本的Oracle数据库库重新链接，则需要将网关与可能导致不兼容的Oracle数据库升级隔离开来。

关于卸载工具

安装前，安装介质中提供了卸载工具（ deinstall ），安装后可在Oracle主目录中使用。它位于路径$ORACLE_HOME/deinstall 。

deinstall命令停止Oracle软件，并删除操作系统上的Oracle软件和配置文件。

该命令使用以下语法，其中变量内容以斜体表示：

deinstall -home home的完整路径 [-silent] [-checkonly] [-local] [-paramfile 输入参数属性文件的完整路径 ] [-params name1 = value name2 = value 。 。 。] [-o 保存文件的目录的完整路径 ] [-help | -H]

选项是：

  • -无声

    使用此标志以静默或响应文件模式运行命令。如果使用-silent标志，则必须使用-paramfile标志，并提供一个参数文件，其中包含要卸载或取消配置的Oracle主目录的配置值。

    您可以通过使用-checkonly标志运行deinstall来生成要使用或修改的参数文件。然后， deinstall命令会发现要从中卸载和取消配置的Oracle主目录中的信息。它会生成属性文件，然后您可以将其与-silent选项一起使用。

    您还可以修改位于响应文件夹中的模板文件deinstall.rsp.tmpl 。

  • -checkonly

    使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行该命令不会删除Oracle配置。-checkonly标志生成一个可以与deinstall命令一起使用的参数文件。

  • -本地

    在多节点环境中使用此标志可在群集中取消配置Oracle软件。

    使用此标志运行deconfig时，它会在本地节点（运行deconfig的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

  • -paramfile complete path of input parameter property file

    使用此标志可以使用默认位置以外的参数文件运行deconfig。使用此标志时，请提供参数文件所在的完整路径。

    参数文件的默认位置取决于deconfig的位置：

    • 从安装介质或阶段位置： $ORACLE_HOME/inventory/response

    • 来自Oracle Technology Network的解压缩归档文件： /ziplocation/response

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -params [name1= value name 2= value name3= value . . .]

    将此标志与参数文件一起使用，以覆盖要在已创建的参数文件中更改的一个或多个值。

  • -o complete path of directory for saving response files

    使用此标志提供除保存属性文件（ deinstall.rsp.tmpl ）的默认位置以外的路径。

    参数文件的默认位置取决于deconfig的位置：

    • 安装前从安装介质或阶段位置： $ORACLE_HOME/

    • 来自Oracle Technology Network的解压缩归档文件： /ziplocation/response/

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -help | -H

    使用help选项（ -help或-h ）获取有关命令选项标志的其他信息。

删除Oracle软件

完成以下过程以删除Oracle软件：

  1. 以安装所有者身份登录。
  2. 运行deinstall命令，根据提示提供有关服务器的信息。
