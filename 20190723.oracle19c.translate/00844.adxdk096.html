标有“错误处理程序集”的框中的箭头和标有“SAX回调集”的框中的箭头指向标有“XmlCreate（）”的框。箭头从XmlCreate框导向标记为“XmlLoadSax（）或XmlLoadDom（）”的框。标有“错误回调”的框中的箭头和标有“xml输入文件，缓冲区，数据库，URL，...”的框中的箭头也会导致“XmlLoadSax（）或XmlLoadDom（）”框。标有“OR”的箭头从“XmlLoadSax（）或XmlLoadDom（）”框引出到标有“XmlEvCreatePPCtx（）”的框。错误从“XmlEvCreatePPCtx（）”框导向标记为“XmlEvLoadPPCtx（）”的框。箭头从“XmlEvLoadPPCtx（）”框导向标记为“XmlEvNext（）”的框。箭头从“XmlEvNext（）”框引导到标有“XML Event Get API”的框。箭头从“XML Event Get API”框导向标记为“XmlEvCleanPPCtx（）”的框。箭头也从框“XML Event Get API”返回到框“XmlEvNext（）”。箭头也从框“XmlEvCleanPPCtx（）”导向框“XmlEvLoadPPCtx（）”。箭头也从“XML Event Get API”框导向标记为“Pull Parser Completes XmlEvDestroyPPCtx（）”的框。箭头从“Pull Parser Completes XmlEvDestroyPPCtx（）”框导向标记为“XmlDestroy（）”的框。箭头从标有“XmlFreeDocument（）”的框中指向“XmlDestroy（）”框。箭头从“XmlFreeDocument（）”框返回到“XmlEvCreatePPCtx（）”框。箭头从“XmlLoadSax（）或XmlLoadDom（）”框导向标记为“SAX：callbacks invoked”的框。箭头从“SAX：callbacks invoked”框导向标记为“SAX completed”的框。标有“另一个”的箭头从框“SAX完成”引导到框“XmlLoadSax（）或XmlLoadDom（）”。另外两个箭头从框“SAX完成”到“XmlFreeDocument（）”和“XmlDestroy（）”框。箭头从“XmlLoadSax（）或XmlLoadDom（）”框导向标记为“DOM构造”的框。标有“另一个”的箭头从框“DOM构造”返回到框“XmlLoadSax（）或XmlLoadDom（）”。箭头从“DOM构造”框导向标记为“DOM：query，edit，...”的框，另一个箭头从“DOM：query，edit，...”返回“DOM构造”。箭头从“DOM：查询，编辑，...”框导向标记为“DOM文档”的框。另外两个箭头从框“DOM：查询，编辑，...”开始到框“XmlFreeDocument（）”和“XmlDestroy（）”。虚线框围绕“SAX：调用的回调”和“SAX完成”框。虚线框围绕“DOM构造”，“DOM：查询，编辑，......”和“DOM文档”框。虚线框包围框“XmlEvNext（）”，“XML Event Get API”，“XmlEvCleanPPCtx（）”和“Pull Parser Completes XmlEvDestroyPPCtx（）”。
