8.1.1关于安装

Oracle Data Mining是Oracle Database Enterprise Edition的Oracle Advanced Analytics选件的一个组件。

要安装Oracle数据库，请遵循适用于您的平台的安装说明。在安装期间选择数据仓库配置。

Oracle Data Miner是Oracle Data Mining的图形用户界面，是Oracle SQL Developer的扩展。Oracle技术网上提供了有关下载SQL Developer和安装Data Miner存储库的说明。

要执行数据挖掘活动，您必须能够登录到Oracle数据库和您的数据库用户标识必须包含数据库 例8-7中描述的特权。

8.1.2启用或禁用数据库选项

了解如何在安装后启用或禁用Oracle Advanced Analytics选项。

默认情况下，在Oracle数据库企业版安装期间启用Oracle Advanced Analytics选项。安装后，您可以使用命令行实用程序 chopt启用或禁用数据库选项。有关说明，请参见适用于您的平台的“安装指南”中的“在安装后启用和禁用数据库选项”。

8.1.3数据挖掘的数据库调优注意事项

了解数据挖掘的数据库调优注意事项。

DBA管理支持的生产数据库 Oracle Data Mining必须遵循Oracle数据库管理员指南中所述的标准管理实践。

构建数据挖掘模型和批处理挖掘模型的评分倾向于在系统上放置类似DSS的工作负载。单行评分往往会在系统上放置类似OLTP的工作负载。

数据库内存管理可以对数据挖掘产生重大影响。正确的尺寸程序全局区域（PGA）内存对于模型构建，复杂查询和批处理评分非常重要。从数据挖掘的角度来看， 系统全局区域（SGA）通常不太重要。但是，必须调整SGA的大小以适应实时评分，这会将模型加载到SGA中的共享游标中。在大多数情况下，您可以配置数据库以自动管理内存。为此，请在调整参数MEMORY_TARGET指定总的最大内存大小。通过自动内存管理，Oracle Database可根据需要在SGA和实例PGA之间动态交换内存，以满足处理需求。

大多数数据挖掘算法都可以利用在数据库中启用并行执行。INIT.ORA参数控制并行执行的行为。
