13.7 dropjava工具

dropjava工具与loadjava工具相反。它将命令行文件名和JAR或ZIP文件内容转换为模式对象名，删除模式对象，并删除其对应的摘要表行。您可以在命令行上以任何顺序输入.java ， .class ， .sqlj ， .ser ， .zip ， .jar和资源文件名。

或者，您可以直接为dropjava工具指定架构对象名称。不以.jar ， .zip ， .class ， .java或.sqlj结尾的命令行参数被假定为模式对象名称。如果指定适用于多个模式对象的模式对象名称，则将删除所有模式对象名称。

删除一个类会使依赖它的类无效，递归地向上级联。删除源会删除从中派生的类。

注意：

您必须以与首次加载它们相同的方式删除Java模式对象。如果加载.sqlj源文件并在服务器中进行转换，则必须在同一源文件上运行dropjava工具。如果您在客户端上进行转换并直接加载类和资源，则在相同的类和资源上运行dropjava工具。

您可以从命令行或使用DBMS_JAVA类中的dropjava方法运行dropjava工具。要从Java应用程序中运行dropjava工具，请使用以下命令：

调用dbms_java.dropjava（'... options ...“）;

选项与命令行中指定的选项相同。用空格分隔每个选项。不要使用逗号分隔选项。唯一的例外是-resolver选项。始终与当前会话建立连接。因此，您无法通过-user选项指定其他用户名。

对于-resolver ，您应首先指定所有其他选项，逗号（ , ），然后指定-resolver选项及其定义。不要指定-thin ， -oci ， -user和-password选项，因为它们与loadjava工具的数据库连接相关。输出被定向到stderr 。设置serveroutput并根据需要调用dbms_java.set_output 。

13.7.1 dropjava工具语法

dropjava工具命令的语法是：

dropjava [options] { file .java | 文件 .class | 文件 .sqlj | 文件 .jar | 文件 .zip | 资源文件 } ...-u | -user user / [ password ] [@ database ] [-genmissingjar JARfile ] [-jarasresource]
[-o | -oci | -oci8] [-optionfile file ] [-optiontable table_name ] [-S | -schema schema ] [-stdout] [-s | - 同义词] [-t | -瘦]
[-v | -verbose] [-list] [-listfile]

13.7.3 dropjava工具参数详细信息

本节描述了一些复杂的dropjava工具参数。

文件名

dropjava工具解释大多数文件名，如loadjava工具所做的那样：

  • .class文件

    在文件中查找类名并删除相应的模式对象。

  • .java和.sqlj文件

    在文件中查找第一个类名，并删除相应的模式对象。

  • .jar和.zip文件

    处理已归档的文件名，就像它们已在命令行中输入一样。

如果文件名具有另一个扩展名或没有扩展名，则dropjava工具会将文件名解释为模式对象名，并删除与该名称匹配的所有源，类和资源对象。

如果dropjava工具遇到与架构对象不匹配的文件名，则它会显示一条消息并处理剩余的文件名。

用户

{-user | -u} 用户 / 密码 [@ 数据库 ]

@ database的允许形式取决于您是指定-oci还是-thin ：

  • -oci:@ database是可选的。如果未指定，则dropjava工具将使用用户的默认数据库。如果指定，则database可以是TNS名称或Oracle Net Services名称 - 值列表。

  • -thin:@ database是必需的。格式为host : lport : SID 。

    哪里：

    • host是运行数据库的计算机的名称。

    • lport是已配置为侦听Oracle Net Services连接的侦听器端口。在默认安装中，它是5521 。

    • SID是数据库实例标识符。在默认安装中，它是ORCL 。

以下是dropjava工具命令的示例：

  • 删除从ServerObjects.jar加载的默认数据库中的TEST模式中的所有模式对象：

    dropjava -u HR -schema TEST ServerObjects.jar密码： password

  • 连接JDBC Thin驱动程序，然后从用户的模式中删除一个类和一个资源文件：

    dropjava -thin -u HR @ dbhost：5521：orcl alpha.class beta.props密码： 密码

基于列表的删除

早期版本的dropjava工具要求在运行客户端或服务器端实用程序的计算机上存在类，JAR，源和资源。当前版本的dropjav a有一个选项，使您可以根据类列表删除类，资源或源，这些类可能不存在于客户端计算机或服务器计算机上。此列表可以在命令行上，也可以在文本文件中。例如：

dropjava -list -u HR -v this.is.my.class this.is.your.class密码： 密码

上面的命令会删除命令行中列出的类this.is.my.class和this.is.your.class ，而不会出现在客户端计算机或服务器计算机上。 

dropjava -listfile my.list -u HR -s -v密码： 密码

上述命令根据my.list列出的类列表删除类，资源或源及其同义词，并详细显示。

注意：

'-install'标志忽略了无法修改的系统拥有的模式对象的加载和删除。

这些模式对象是CREATE JAVA COMMAND提供的运行时类和资源。

13.7.4关于使用dropjava工具删除资源

如果要删除直接加载到服务器中的资源，则必须小心。如果您在客户端上翻译它们而不使用-ser2class选项，则包括配置文件。删除由服务器端SQLJ转换程序生成的源或类模式对象或资源模式对象时，将根据适用的.sqlj源文件中的程序包规范找到模式对象。但是，在客户端上生成并直接加载到服务器的资源的完全限定的模式对象名称取决于.jar文件中的路径信息或加载时在命令行上指定的路径信息。如果使用.jar文件加载资源并使用相同的.jar文件来删除资源，那么就没有问题了。但是，如果使用命令行加载资源，则在运行dropjava工具以删除资源时必须小心指定相同的路径信息。
