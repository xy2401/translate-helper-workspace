Oracle Database Release 19.1中的更改

以下是Oracle Spatial and Graph RDF知识图开发人员指南（适用于Oracle数据库发行版19.1）中的更改。

  • 为模式 - 私有语义网络添加了支持
  • 功能名称更改：RDF知识图

父主题： 本指南的此版本中的更改

为模式 - 私有语义网络添加了支持

现在可以在常规用户的模式中创建语义网络。此类网络称为模式专用语义网络，因为关联的数据库对象是在网络所有者的架构中创建的，并且网络所有者具有对这些对象的专有权限。（DBA用户也具有此类权限，网络所有者或DBA可以授予和撤消其他用户的权限。）

在以前的版本中，唯一的方案是MDSYS用户拥有的单个语义网络，并且可以访问整个数据库。该方案仍然受支持，但我们鼓励您使用模式专用语义网络。

有关更多信息，请参阅语义网络 。

父主题： Oracle数据库发行版19.1中的更改

功能名称更改：RDF知识图

以前称为RDF语义图的功能名称现在称为RDF知识图 。

语义网络和语义数据这两个术语在本指南中仍广泛使用。

Oracle Database Release 18.1中的更改

以下是Oracle Spatial和Graph RDF语义图开发人员指南（适用于Oracle数据库发行版18.1）中的更改。

  • 为Oracle Database In-Memory添加了支持
  • 通过复合分区为语义网络添加支持
  • 增强的CLOB支持批量加载操作
  • 对Turtle和Trig RDF格式的本机支持
  • 支持RDF添加到SQL Developer

父主题： 本指南的此版本中的更改

为Oracle Database In-Memory添加了支持

可以轻松地将RDF数据加载到内存中，以利用Oracle Database In-Memory功能。现在可以使用SEM_APIS.ENABLE_INMEMORY过程将语义网络加载到内存中。此外，可以在虚拟模型级别使用内存中虚拟列，以将RDF术语的词值添加到MDSYS.RDF_LINK $表的内存中表示，从而减少评估SPARQL查询所需的联接数。

有关更多信息，请参阅Oracle数据库内存中的RDF支持 。

父主题： Oracle数据库发行版18.1中的更改

通过复合分区为语义网络添加支持

现在可以使用列表哈希复合分区来创建语义网络。通过该方案，语义网络最初由模型id进行列表分区，然后使用RDF谓词ID的散列对每个分区进行子分区。复合分区通过增加并行化和更好的查询优化器统计信息来提高SPARQL查询性能。

有关更多信息，请参阅语义网络 。

父主题： Oracle数据库发行版18.1中的更改

增强的CLOB支持批量加载操作

现在可以使用SEM_APIS.BULK_LOAD_FROM_STAGING_TABLE有效地加载包含具有长文字的RDF四边形（大小超过4000字节的RDF对象值）的临时表。

SEM_APIS.LOAD_INTO_STAGING_TABLE中还添加了新选项，以便在从外部表加载时更好地处理长文字。VC_ONLY选项仅将对象值不大于4000字节的RDF四元组加载到临时表中，而CLOB_ONLY选项仅加载对象值大于4000字节的RDF四元组。这些选项允许非常有效的两阶段批量加载，其中在一个批量加载操作中加载仅VARCHAR数据，而在第二个批量加载操作中加载仅CLOB数据。

父主题： Oracle数据库发行版18.1中的更改

对Turtle和Trig RDF格式的本机支持

现在可以将Turtle和Trig RDF格式直接加载到Oracle数据库中，而无需第三方工具。通过SEM_APIS.UPDATE_MODEL执行的SPARQL LOAD操作现在可以解析并插入以Turtle和Trig格式序列化的RDF数据，以及先前支持的N-Triple和N-Quad格式。

现在可以直接从Turtle或N-Triple格式指定的R2RML映射创建RDF视图。新的R2RML_STRING和R2RML_STRING_FMT参数已添加到SEM_APIS.CREATE_RDFVIEW_MODEL，以便R2RML映射字符串可用于创建RDF视图模型。

支持RDF添加到SQL Developer

您可以使用Oracle SQL Developer创建与RDF相关的对象并使用RDF和OWL功能。

有关更多信息，请参阅SQL Developer中的RDF支持 。
