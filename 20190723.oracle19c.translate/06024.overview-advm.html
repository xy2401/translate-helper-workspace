Oracle ASM动态卷管理器概述

Oracle ASM动态卷管理器（Oracle ADVM）为客户端提供卷管理服务和标准磁盘设备驱动程序接口。

文件系统和其他基于磁盘的应用程序将I / O请求发送到Oracle ADVM卷设备，就像向供应商操作系统上的其他存储设备发送I / O请求一样。

Oracle ADVM卷设备由Oracle ASM动态卷构成。可以在每个Oracle ASM磁盘组中配置一个或多个Oracle ADVM卷设备。Oracle ADVM驱动程序将针对Oracle ADVM卷设备的I / O请求映射到位于Oracle ASM磁盘组中的相应Oracle ASM动态卷和磁盘集中的块。Oracle ADVM卷设备导出Oracle ASM卷管理器功能，并确保在异常系统关闭，Oracle ASM实例故障或系统故障时卷镜像保持一致。

Oracle ADVM通过为分配为Oracle ADVM卷文件的Oracle ASM存储提供磁盘驱动程序接口来扩展Oracle ASM。您可以使用Oracle ADVM创建包含文件系统的虚拟磁盘。Oracle ADVM卷上包含的这些文件系统能够支持Oracle数据库文件之外的文件，例如可执行文件，报告文件，跟踪文件，警报日志和其他应用程序数据文件。由于Oracle ADVM卷实际上是Oracle ASM文件，因此它们需要与Oracle ASM文件相同的管理权限。

Oracle自动存储管理群集文件系统（Oracle ACFS）通过Oracle ADVM接口与Oracle ASM进行通信。通过添加Oracle ADVM，Oracle ASM成为用户数据的完整存储解决方案，可满足数据库和非数据库文件的需求。

要将卷添加到Oracle ASM磁盘组，必须将磁盘组属性COMPATIBLE.ASM和COMPATIBLE.ADVM设置为“ '11.2' 。

注意：

动态卷取代传统的设备分区。每个卷都是单独命名的，可以配置为单个文件系统。可以根据需要从Oracle ASM磁盘组存储创建Oracle ADVM卷，并根据需要动态调整大小。这些属性使Oracle ADVM卷比物理设备和相关的分区方案更灵活。
