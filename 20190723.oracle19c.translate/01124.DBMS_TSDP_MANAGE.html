174 DBMS_TSDP_MANAGE

DBMS_TSDP_MANAGE包提供了一个用于导入和管理数据库中敏感列和敏感列类型的接口，并与DBMS_TSDP_PROTECT包一起用于透明敏感数据保护（TSDP）策略。

DBMS_TSDP_MANAGE仅适用于Enterprise Edition。

174.1 DBMS_TSDP_MANAGE概述

DBMS_TSDP_MANAGE包使您可以管理Oracle数据库中的敏感列和敏感类型。

已识别的敏感列根据敏感类型进行分类。通过使用DBMS_TSDP_PROTECT程序包创建一个策略，该策略基于列类型而不是数据本身来保护给定类的数据，然后您可以统一管理这些类型的安全性并修改设置以适应不断变化的合规性规则。

使用Data Pump执行完全导出时，也可以将策略导出到其他数据库。您无法导出策略本身，但数据库的导出将包括TSDP策略。

174.2 DBMS_TSDP_MANAGE安全模型

所有程序都以调用者的权利执行。DBMS_TSDP_MANAGE包由SYS拥有。

应酌情授予此程序包的EXECUTE特权。通常，应为应用程序数据库管理员授予此程序包的EXECUTE特权，而DBMS_TSDP_PROTECT程序包将由安全管理员管理。

174.3.1 ADD_SENSITIVE_COLUMN过程

此过程将一列添加到敏感列列表中。

句法

DBMS_TSDP_MANAGE.ADD_SENSITIVE_COLUMN（schema_name IN VARCHAR2，table_name IN VARCHAR2，column_name IN VARCHAR2，sensitive_type IN VARCHAR2，user_comment IN VARCHAR2 DEFAULT NULL）;

参数

表174-2 ADD_SENSITIVE_COLUMN过程参数

参数 描述

SCHEMA_NAME

该列所属的模式

TABLE_NAME

包含该列的表

列名

敏感的列名称

sensitive_type

敏感列类型的标识符

USER_COMMENT

关于敏感列的用户评论

例子

在SCOTT.EMP添加一列SAL ：

DBMS_TSDP_MANAGE.ADD_SENSITIVE_COLUMN（schema_name =>'SCOTT'，table_name =>'EMP'，column_name =>'SAL'，sensitive_type =>'SALARY_TYPE'，user_comment =>'薪资栏'）;

174.3.2 ALTER_SENSITIVE_COLUMN过程

此过程会更改敏感列列表中的敏感类型和/或列的注释。

句法

DBMS_TSDP_MANAGE.ALTER_SENSITIVE_COLUMN（schema_name IN VARCHAR2，table_name IN VARCAHR2，column_name IN VARCAHR2，sensitive_type IN VARCAHR2，user_comment IN VARCAHR2 DEFAULT NULL）;

参数

表174-3 ALTER_SENSITIVE_COLUMN过程参数

TABLE_NAME

包含该列的表

列名

敏感的列名称

sensitive_type

USER_COMMENT

174.3.7 IMPORT_DISCOVERY_RESULT过程

此过程可用于从外部源导入敏感列以及关联的敏感类型。外部源可以是Oracle Enterprise Manager Cloud Control中的应用程序数据模型（ADM）实例。

句法

DBMS_TSDP_MANAGE.IMPORT_DISCOVERY_RESULT（discovery_result IN CLOB，discovery_source IN VARCHAR2，强制IN FORCE DEFAULT FALSE）; DBMS_TSDP_MANAGE.IMPORT_DISCOVERY_RESULT（discovery_result在XMLTYPE中，discovery_source IN VARCHAR2，强制IN FORCE DEFAULT FALSE）;

参数

表174-8 IMPORT_DISCOVERY_RESULT过程参数

参数 描述

discovery_result

敏感列的列表，以及XML格式（可能作为CLOB ）的敏感列类型的（可定义的）列表。

discovery_source

进口来源。discovery_sourcename标识导入的敏感列的列表。对于ADM，这应该是ADM名称。

力

指定当发现结果包含已被另一个源标识为敏感列的敏感列时，是否应导入发现结果。

  • FALSE （默认值） - 如果列发生冲突，则不会导入发现结果。没有导入任何列和敏感类型。

  • TRUE - 导入发现结果，并根据传入的发现结果设置冲突列的属性

例子

导入ADM实例的敏感列列表， ADM_Demo ：

DBMS_TSDP_MANAGE.IMPORT_DISCOVERY_RESULT（discovery_results => xml_adm_result，discovery_source =>'ADM_Demo'）;

174.3.8 IMPORT_SENSITIVE_TYPES过程

此过程从源中导入敏感列类型的列表。

句法

DBMS_TSDP_MANAGE.IMPORT_SENSITIVE_TYPES（在CLOB中为sensitive_types，源为IN VARCHAR2）; DBMS_TSDP_MANAGE.IMPORT_SENSITIVE_TYPES（XMLTYPE中的sensitive_types，源IN VARCHAR2）;

参数

表174-9 IMPORT_SENSITIVE_TYPES过程参数

参数 描述

sensitive_types

XML格式的敏感列类型列表（可能作为CLOB ）

资源

进口来源。源标识导入的敏感列类型列表。对于来自Oracle Enterprise Manager Cloud Control实例的应用程序数据模型（ADM），这应该是ADM名称。

例子

导入ADM实例的敏感列类型列表ADM_Demo ：

DBMS_TSDP_MANAGE.IMPORT_SENSITIVE_TYPES（sensitive_types => xml_adm_result，source =>'ADM_Demo'）;

表174-10 REMOVE_DISCOVERY_RESULT过程参数
