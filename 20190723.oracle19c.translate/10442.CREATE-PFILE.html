创建PFILE

目的

使用CREATE PFILE语句将二进制服务器参数文件或当前的In-Memory参数设置导出到文本初始化参数文件中。创建文本参数文件是获取数据库正在使用的当前参数设置列表的便捷方式，它允许您在文本编辑器中轻松编辑文件，然后使用CREATE SPFILE将其转换回服务器参数文件声明。

成功执行此语句后，Oracle数据库会在服务器上创建一个文本参数文件。在Oracle Real Application Clusters环境中，它将包含所有实例的所有参数设置。它还将包含与服务器参数文件中的参数设置在同一行上出现的任何注释。

关于在CDB中创建文本参数文件的注意事项

在多租户容器数据库（CDB）中创建文本参数文件时，当前容器可以是根或PDB。

  • 如果当前容器是根，则数据库将创建一个包含根的参数设置的文本文件。

  • 如果当前容器是PDB，则数据库将创建一个包含PDB参数设置的文本文件。在这种情况下，您必须指定pfile_name 。

先决条件

您必须具有以下系统特权之一才能执行此语句：

  • SYSDBA

  • SYSDG

  • SYSOPER

  • SYSBACKUP

  • SYSASM

  • SYSRAC

您可以在实例启动之前或之后执行此语句。

限制

您不能将OS文件覆盖为SYSDG ， SYSOPER或SYSRAC用户。

句法

create_pfile :: =


说明create_pfile.eps的说明

语义

pfile_name

指定要创建的文本参数文件的名称。如果未指定pfile_name ，则Oracle数据库使用特定于平台的缺省初始化参数文件名。 pfile_name可以包含路径前缀。如果未指定此类路径前缀，则数据库会添加默认存储位置的路径前缀，该位置与平台相关。

spfile_name

指定要从中创建文本文件的二进制服务器参数的名称。

  • 如果指定spfile_name ，则该文件必须存在于服务器上。如果文件不驻留在操作系统上服务器参数文件的缺省目录中，则必须指定完整路径。

  • 如果未指定spfile_name ，则数据库将使用当前与实例关联的spfile，通常是使用启动的spfile。如果没有spfile与实例关联，则数据库将查找特定于平台的缺省服务器参数文件名。如果该文件不存在，则数据库返回错误。

记忆

使用当前系统范围的参数设置指定MEMORY以创建pfile。在Oracle RAC环境中，创建的文件将包含每个实例的参数设置。

例子

创建参数文件：示例

下面的示例创建一个文本参数文件my_init.ora从二进制服务器参数文件s_params.ora ：

CREATE PFILE ='my_init.ora'FROM SPFILE ='s_params.ora';

注意：

通常，您需要在操作系统上指定参数文件的完整路径和文件名。有关路径信息，请参阅Oracle操作系统文档。
