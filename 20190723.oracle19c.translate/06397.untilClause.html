4.22直到条款

目的

使用untilClause子句为各种RMAN操作指定时间，SCN或日志序列号的上限。

使用说明

要指定还原点，请使用TO RESTORE POINT子句。有关示例，请参见“ SET ” 。

句法

untilClause :: =


插图说明untilclause.eps

语义

语法元素 描述

UNTIL SCN integer

将SCN指定为上限非包含限制。

RMAN仅选择可用于还原或恢复但不包括指定SCN的文件（请参阅示例4-41 ）。例如， RESTORE DATABASE UNTIL SCN 1000仅选择可用于恢复到SCN 1000的备份。

UNTIL SEQUENCE integer

将重做日志序列号和线程指定为上限非包含限制。

RMAN仅选择可用于还原或恢复但不包括指定序列号的文件。例如， REPORT OBSOLETE UNTIL SEQUENCE 8000仅报告可用于通过日志序列7999恢复的备份。

THREAD integer

指定重做线程的编号。

UNTIL TIME ' date_string '

将时间指定为上限非包含限制（请参见示例4-42 ）。

RMAN仅选择可用于还原和恢复但不包括指定时间的文件。例如， LIST BACKUP UNTIL TIME 'SYSDATE-7'列出了可用于在一周前恢复到一个点的所有备份。

在RMAN命令中指定日期时，日期字符串必须是：

  • 一个文字字符串，其格式与NLS_DATE_FORMAT设置匹配。

  • DATE类型的SQL表达式，例如， 'SYSDATE-10'或"TO_DATE('01/30/2013', 'MM/DD/YYYY')" 。第二个示例包含其自己的日期格式掩码，因此独立于当前的NLS_DATE_FORMAT设置。

以下是RMAN命令中典型日期设置的示例：

备份归档从时间'星期三-31'直到'时间'SYSDATE-14';恢复数据库直到“到_DATE（'09 / 20/06'，'MM / DD / YY'）”;

注意：基于时间的恢复的粒度取决于重做日志中的时间戳。例如，假设您指定以下命令：

恢复数据库直到'2013-07-26 17:45:00';

如果没有使用时间戳17:45:00写入重做，则恢复继续进行，直到找到更高的下一个重做时间戳。例如，下一个重做时间戳可能是17:45:04 。您可以通过查询V$LOG_HISTORY TABLE的FIRST_TIME和FIRST_CHANGE#列来检查特定SCN的最近时间。

例子

示例4-41对指定的SCN执行不完全恢复

此示例假定已装入数据库，将数据库恢复到（但不包括）指定的SCN：

STARTUP FORCE MOUNT RUN {SET UNTIL SCN 1418901; #set to 1418901通过SCN恢复数据库1418900 RESTORE DATABASE;恢复数据库; ALTER DATABASE OPEN RESETLOGS;
