5.260 DBA_RAT_CAPTURE_SCHEMA_INFO

DBA_RAT_CAPTURE_SCHEMA_INFO显示在工作负载捕获中记录SQL语句时生效的登录架构和当前架构。

在扩展PL / SQL模式下执行工作负载重放时，此视图很有用。此类重放可能包括执行表操作（如SELECT ， UPDATE和DELETE SQL语句。如果当前架构与工作负载捕获时的登录架构不同，那么这些表操作可能是使用当前用户的权限而不是登录用户执行的。在工作负载重放期间，所有操作都使用登录用户的权限执行。因此，如果登录用户没有执行表操作所需的权限，则在重放期间可能会发生错误。

要解决此问题，可以将此视图与DBA_WORKLOAD_CAPTURE_SQLTEXT视图结合使用。加入CAPTURE_ID在该视图中与列CAPTURE_ID列DBA_WORKLOAD_CAPTURE_SQLTEXT确定登录模式和电流模式，它时生效每个SQL语句DBA_WORKLOAD_CAPTURE_SQLTEXT被抓获。检查DBA_WORKLOAD_CAPTURE_SQLTEXT的SQL_TEXT列，以确定SQL语句是否涉及任何表操作，以及这些表操作是否是使用当前用户或登录用户的权限执行的。然后，您可以在执行工作负载重放之前向登录用户授予执行这些表操作所需的权限。

柱 数据类型 空值 描述

CAPTURE_ID

NUMBER（38）

工作负载捕获的内部密钥

SQL_ID

VARCHAR2（13）

库高速缓存中父游标的SQL标识符

LOGIN_SCHEMA

VARCHAR2（128）

登录到工作负载捕获中记录了SQL语句的会话的用户的模式。在会话期间，此值不会更改。

CURRENT_SCHEMA

VARCHAR2（128）

在工作负载捕获中记录SQL语句时，会话的当前活动默认架构。通过使用ALTER SESSION SET CURRENT_SCHEMA语句，此值可能在会话期间ALTER 。它也可能在会话期间发生变化，以反映任何活动定义者权限对象的所有者。
