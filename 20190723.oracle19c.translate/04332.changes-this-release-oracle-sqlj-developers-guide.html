1 Oracle SQLJ Developer's Guide的此发行版中的更改

不推荐的功能

此版本中不推荐使用以下功能，并且可能在以后的版本中不支持该功能：

oracle.sql。*套餐

启动Oracle Database 12c第2版（12.2）时，不推荐使用oracle.sql。*包。

不受支持的功能

Oracle不再支持以下功能：

扩展数据类型支持（EDS）

Oracle Database 19c中不支持扩展数据类型支持（EDS）功能。现在，Logical Standby和Oracle GoldenGate都支持EDS功能支持的所有数据类型。

扩展数据类型支持（EDS）功能为逻辑备用数据库提供了一种机制，以支持缺少基于本机重做的支持的某些Oracle数据类型。例如，EDS用于复制具有SDO_GEOMETRY列的表。但是，从Oracle Database 12c第2版（12.2）开始，没有任何EDS支持的Oracle数据类型本身不受逻辑备用数据库或Oracle GoldenGate支持。Oracle Database 19c（19.1）不支持此功能。

服务器中的SQLJ

从Oracle Database 12c第2版（12.2）开始，Oracle不支持服务器端SQLJ代码。

Oracle支持使用客户端SQLJ。但是，Oracle不支持使用服务器端SQLJ，包括在数据库环境中运行存储过程，函数和触发器。

JPublisher的

Oracle Database 12c第2版（12.2.0.1）中的所有Oracle JPublisher功能都不受支持且不可用。Oracle建议您使用此处列出的替代方法：

  • 要继续使用Web服务标注，Oracle建议您使用Oracle JVM Web服务标注实用程序，该实用程序可替代Web服务标注实用程序。

  • 要替换其他JPublisher自动化功能，包括映射用户定义的SQL类型或SQL类型，包装PL / SQL包和类似功能，Oracle建议开发人员使用显式步骤，例如使用SQLJ预编译器预编译代码，构建Java STRUCT类或使用其他预先构建的选择。
