16.1了解自动诊断信息库

该 自动诊断存储库（ADR） （ADR）是一个系统范围的跟踪和日志记录中央存储库。存储库是基于文件的分层数据存储，用于存储诊断信息，包括网络跟踪和日志记录信息。

ADR主目录是分配给Oracle产品实例的ADR目录的单元。每个数据库实例都有自己的ADR主目录。同样，每个侦听器，Oracle Connection Manager和客户端实例都有自己的ADR主目录。

如果进程失败，则会生成事件。事件转储文件位于ADR_BASE/ADR_HOME/incident/目录中。默认情况下，如果设置了ORACLE_BASE变量，则ADR_BASE为ORACLE_BASE。如果未设置该变量，则ADR_BASE为ORACLE_HOME/log 。ADR_BASE可以设置为任何位置。

可以在进程跟踪文件中找到事件转储文件位置。

ADR主目录的位置由以下路径给出，该路径从ADR基本目录开始：

diag / product_type / product_id / instance_id

16.1.1 ADRCI：ADR命令解释器

ADRCI 是一个命令行工具，是故障可诊断性基础结构的一部分。ADRCI使您可以执行以下操作：

  • 查看ADR中的诊断数据

  • 将事件和问题信息打包到zip文件中以传输到Oracle Support Services

诊断数据包括事件和问题描述，跟踪文件，转储，运行状况监视器报告，警报日志条目等。

ADRCI具有丰富的命令集，可以在交互模式或脚本中使用。此外，ADRCI可以运行ADRCI命令的脚本，就像SQL * Plus使用SQL和PL / SQL命令运行脚本一样。

要使用ADRCI查看跟踪文件， ADRCI在命令行中输入ADRCI 。以下是用于检查客户端的常见ADRCI命令：

客户端

adrci> SHOW ALERT adrci> SHOW BASE -product client adrci> SET BASE -product client adrci> SHOW TRACEFILE adrci> SHOW TRACE trace_file .trc adrci> SHOW SPOOL

在上述命令中， SHOW ALERT命令在文本编辑器（如VI）中显示log.xml文件。 SHOW BASE -product client命令显示SHOW BASE -product client的ADR_BASE目录的值。在SET BASE命令中将该值用于客户端 。

以下是用于检查服务器的常见ADRCI命令：

服务器端

adrci> SHOW BASE adrci> SHOW TRACEFILE adrci> SHOW TRACE trace_file .trc

其他ADRCI命令选项可用于更有针对性的Oracle Net跟踪文件分析。在ADRCI提示符下键入HELP以获取帮助文档。

16.2诊断Oracle Net Services

Oracle Net Services会使用错误编号或消息报告任何潜在的故障，无论是否显着。错误号和消息提供了诊断问题的有用信息，但可能无法始终识别实际问题。本节中的任务有助于确定Oracle Net Services的哪些部分正常运行，而不是无法运行的部分。它们还有助于确定故障属于以下哪个类别：

  • Oracle软件

  • 操作系统层

  • 其他网络层

在大多数情况下，逐步测试各种网络层应该能够发现任何问题。

  • 诊断服务器问题
  • 诊断客户端问题

16.2.1诊断服务器问题

要开始诊断服务器问题，您应该回答以下问题：

  • 是否有其他系统（如工作站或服务器）能够使用Oracle Net连接到服务器？

  • 服务器，数据库或侦听器配置在一段时间内保持不变吗？

如果您对上述任一问题的回答为“是”，请转至“ 诊断客户端问题 ” 。

如果您不确定或对上述任何问题的回答为否，请使用本节中的任务来诊断问题。在服务器上诊断Oracle Net Services涉及以下任务：

  • 任务1，“验证数据库是否正在运行”

  • 任务2，“执行环回测试”

任务1验证数据库是否正在运行

要检查数据库是否已启动，请登录数据库并使用有效的用户名和密码进行连接。例如：

SQLPLUS系统输入密码： password

将显示一条消息，确认您已与数据库连接。如果收到以下错误，请要求数据库管理员为您提供帮助：

  • ORA-1017：无效的U / P.

  • ORA-1034：Oracle不可用

任务2执行环回测试

环回测试使用Oracle Net从数据库服务器返回到自身，绕过进程间通信（IPC）协议。许多网络协议提供了一种测试网络连接的方法。PING实用程序可与TCP / IP网络一起使用。执行成功的环回可验证Oracle Net是否在数据库服务器上运行。

以下过程描述如何从服务器到数据库执行环回测试 ：

  1. 确保listener.ora ， tnsnames.ora和sqlnet.ora文件存在于正确的位置，如“ 使用本地化管理 ”中所述 。

  2. 启动Oracle Net Manager。

    也可以看看：

    “ 使用Oracle Net Manager配置Oracle Net服务 ”

  3. 在导航器中，展开“ 目录”或“ 本地”选项。

  4. 展开“ 服务命名”以查看可用的网络服务和数据库名称。

  5. 选择网络服务名称或数据库服务。

  6. 选择“ 命令” ，然后选择“ 测试网络服务” 。

    测试假定侦听器和数据库正在运行。如果不是，请参阅“ 启动Oracle Net Listener和Oracle数据库服务器 ”以启动组件。

    在测试期间，将出现“连接测试”对话框，提供状态和测试结果。成功的测试会产生以下消息：

    连接测试成功。

    如果测试成功，则继续执行步骤7 。

    如果测试不成功，请执行以下操作：

    1. 确保数据库和侦听器正在运行，然后单击“ 测试” 。

    2. 单击“ 更改登录”以更改连接的用户名和密码，然后单击“ 测试” 。

    • 如果环回测试通过，则转到“ 诊断客户端问题 ” 。

    • 如果环回测试仍然失败，请联系Oracle支持服务。

  7. 单击“ 关闭”以关闭“连接测试”对话框。

16.3.2 ORA-12154：无法解析指定的连接标识符

此消息表示使用连接标识符请求与数据库或其他服务的连接，并且使用配置的命名方法之一无法将指定的连接标识符解析为连接描述符。例如，如果使用的连接标识符类型是网络服务名称，则无法在命名方法存储库中找到网络服务名称，或者无法找到或到达存储库。

执行以下步骤：

  1. 检查sqlnet.ora文件中NAMES.DIRECTORY_PATH参数中列出的命名适配器的类型。如果未配置，则使用adapters命令确定正在使用的适配器。以下示例显示了适配器：

    $ adapter ......安装的Oracle Net命名方法有：本地命名（tnsnames.ora）Oracle目录命名Oracle主机命名NIS命名

    应至少为其中一个命名方法定义连接字符串中给出的网络服务名称。

  2. 检查每个适配器的分辨率路径是否存在问题。例如，如有必要，请使用网络服务的全名确保连接字符串中给出的名称正确且完整。

除了上述步骤之外，还可以根据命名方法执行以下步骤：

  • 使用本地命名方法时出现ORA-12154错误
  • 使用目录命名方法时出现ORA-12154错误

16.4 Oracle Net Services的故障排除建议

诊断网络问题时，以下建议可能有用：

  • 在配置期间使用节点或网络地址而不是服务器计算机的名称。这消除了任何内部查找问题，并使连接速度稍快。

  • 如果您使用的是TCP / IP地址，请使用IP地址而不是主机名。例如，将tnsnames.ora文件中的HOST= server_name行更改为IP地址，例如HOST=192.0.2.5 。

  • 如任务2“执行环回测试”中所述，在服务器上执行环回测试 。如果测试通过，则使用FTP将tnsnames.ora和sqlnet.ora文件发送到客户端。

  • 检查客户端和服务器之间的系统。如果是广域网（WAN），则识别可能无法正常工作的任何中间系统。如果所有计算机都没问题，那么问题可能是时间问题。

  • 验证是否存在计时问题。时序问题与客户端日志文件中的ORA-12535错误相关联。

    要解决计时问题，请尝试使用确切地址而不是名称来加速连接，并增加listener.ora文件中的INBOUND_CONNECT_TIMEOUT_ listener_name参数。此参数的默认值为10秒。

  • 确定哪些Oracle应用程序失败。SQL * Plus可能有效，但CASE工具可能不行。如果确定问题是数据卷问题，则尝试使用基本连接传输大型（5 MB）文件。

以下问题可以帮助诊断问题：

16.6.8.4 ONS节点关闭事件信息的监听器订阅

使用TRACE_FILEAGE_SERVER参数设置此参数时，将根据跟踪文件的使用期限循环跟踪文件。使用第一个文件直到达到年龄限制，然后使用第二个文件，依此类推。达到最后一个文件的期限时，将重复使用第一个文件，依此类推。

    • 3：程序可用于有限的功能。对整体运营并不重要。

    • 4：客户规避的问题。对操作的影响最小（如果有的话）。
