6.1建立连接的先决条件

本章中的任务显示数据库服务器和客户端计算机之间的TCP / IP连接。假设有关数据库服务器和客户端计算机的以下条件：

  • 数据库服务器

    • 服务器正在可以访问客户端的网络上运行

    • 安装了Oracle数据库

    • 已配置侦听器

    • 已安装TCP / IP协议支持

  • 客户电脑

    • 客户端计算机正在可以访问数据库服务器的网络上运行

    • 已安装Oracle客户端

    • 已安装TCP / IP协议支持

在TCP / IP网络中，每台计算机都有一个唯一的IP地址 。名称解析服务（如域名系统（DNS））可用于映射计算机的IP地址及其主机名。如果未使用名称解析服务，则映射通常存储在名为hosts的集中维护文件中。此文件位于Linux上的/etc目录和Microsoft Windows上的\windows\system32\drivers\etc目录中。例如，名为sales-server的数据库服务器计算机的条目可能如下所示：

#IP地址服务器主机名别名192.0.2.203 sales-server sales.us.example.com

6.2确认网络可用性

在使用Oracle Net将客户端计算机连接到数据库服务器之前，请确认客户端计算机可以与数据库服务器计算机成功通信。评估网络连接可以消除基于网络的错误。

以下过程介绍了如何确认网络连接：

  1. 确认数据库服务器计算机可以通过环回测试与自身通信，如下所示：

    1. 要确认硬件连接，请在命令行中输入以下命令：

      ping ip_address

      在上面的命令中， ip_address是数据库服务器计算机的IP地址，如下所示：

      ping 192.0.2.203

    2. 要确认正确配置DNS或主机名，请在命令行中输入以下命令：

      ping host_name

      在上面的命令中， host_name是服务器的主机名。

    3. 要测试服务器的TCP / IP设置，请输入以下命令：

      ping 127.0.0.1 ping6 :: 1

      IP地址127.0.0.1是环回测试的标准IPv4地址。 IP地址:: 1（0：0：0：0：0：0：0：1）是环回测试的标准IPv6地址。

  2. 验证客户端计算机是否可以成功与数据库服务器计算机通信

    验证方法根据网络协议而变化。对于TCP / IP，您可以使用PING，FTP或TELNET实用程序。

    如果客户端计算机无法访问服务器，请验证网络电缆和网络接口卡是否正确连接。请与网络管理员联系以更正这些问题。

6.3启动Oracle Net Listener和Oracle数据库服务器

必须运行Oracle Net Listener和Oracle数据库服务器才能使数据库服务器接收连接。以下过程介绍了如何启动Oracle Net Listener：

  1. 使用监听器控制实用程序启动监听器。从命令行输入以下内容：

    lsnrctl LSNRCTL>  START [ listener_name ]

    在上面的命令中， listener_name是listener.ora文件中定义的侦听器的名称。如果您使用默认名称LISTENER ，则无需标识侦听LISTENER 。

    显示指示侦听器已成功启动的状态消息。

  2. 按如下方式启动数据库：

    1. 使用以下命令启动SQL * Plus而不连接到数据库：

      SQLPLUS / nolog

    2. 使用以下命令以SYSDBA身份连接到数据库：

      SQL> CONNECT username名为sysdba

      系统将提示您输入密码。

      注意：

      为简单起见，此示例不执行已部署系统通常使用的密码管理技术。在生产环境中，请遵循Oracle数据库密码管理准则，并禁用任何示例帐户。有关密码管理指南和其他安全建议，请参阅Oracle数据库安全指南 。

    3. 使用以下命令启动数据库：

      SQL> STARTUP database_name

      在上面的命令中， database_name是数据库的名称。

      也可以看看：

      有关启动数据库的其他信息，请参见“ Oracle数据库管理员指南”

  3. 使用监听器控制实用程序和以下命令确认已完成对侦听器的数据库服务注册 ：

    LSNRCTL> 服务[ listener_name ]

    SERVICES命令列出数据库支持的服务，以及至少一个可用的服务处理程序 。如果未列出数据库服务注册，请输入以下SQL命令：

    SQL> ALTER SYSTEM REGISTER;

    也可以看看：

    “ 监听监听服务 ”以获取有关“ SERVICES命令的其他信息

6.4启动Oracle Connection Manager

如果已安装Oracle Connection Manager，则按如下方式启动Oracle Connection Manager：

  1. 使用以下命令启动Oracle Connection Manager控制实用程序（CMCTL）：

    cmctl  CMCTL> ADMINISTER [ instance_name ]

    在上面的命令中， instance_name是要管理的Oracle Connection Manager的名称。您可以通过查看cman.ora文件来确定名称。该文件位于ORACLE_HOME/network/admin目录中的Oracle Connection Manager计算机上。

    Oracle Connection Manager显示一条状态消息，指示实例的名称，并通知您该实例尚未启动。

    注意：

    如果未提供实例名称作为参数，则为Oracle Connection Manager提供标准主机名。这是默认值。发出ADMINISTER命令后，CMCTL将显示实例名称，如下所示：

    CMAN_ fully_qualified_host_name

  2.

    cmctl> STARTUP

6.5连接数据库

               

                  
有几种方法可以连接到Oracle数据库。表6-1列出了连接数据库的语法。
                  

                  

                     
表6-1连接数据库

                     

                        
                           

                              连接类型
                              连接语法
                              描述
                           

                        
                        
                           

                              
                                 
从命令行

                              		
                              
                                 
从命令行将应用程序连接到数据库服务器的一般形式是：
tool username @ connect_identifier

系统将提示您输入要加密的密码。

                                 
例如：
SQLPLUS system @ sales输入密码： password

                              
                                 
大多数Oracle工具都可以使用操作系统命令行进行连接，有些则提供替代方案。

                              		
                           

                           

                              
                                 
从登录屏幕

                              		
                              
username @ connect_identifier

                              
                                 
某些工具提供登录屏幕作为登录的替代形式。用户可以通过在工具登录屏幕的用户名字段中识别用户名和连接标识符，并在密码字段中输入密码来登录数据库服务器。

                              		
                           

                           

                              
                                 
来自3GL应用程序

                              		
                              
exec sql connect： username ： password 

在上面的连接请求中， ：username和：password是可以在程序中静态或通过提示用户设置的3GL变量。连接到数据库服务器时， ：username变量的值采用以下格式：
用户名@ net_service_name 

：password变量包含要连接的数据库帐户的密码。
                                 

                              		
                              
                                 
使用3GL编写的应用程序（如OCI和预编译器）由中间层和数据库应用程序开发人员用于从客户端程序直接访问数据库。

                              		
                           

                           

                              
                                 
从SQL * Plus中

                              		
                              
SQLPLUS / nolog SQL> CONNECT username@net_service_name

例如：
SQLPLUS / nolog SQL> CONNECT scott @ serverx输入密码： password
在上述命令中， 用户名和密码是数据库用户和密码， net_service_name是网络服务名称。
                                 

                              		
                              
                                 
某些Oracle工具具有用于数据库连接的命令，以允许在不离开工具的情况下指定备用用户名。

                                 
其他Oracle工具使用与其功能或接口特定的略有不同的方法。例如，Oracle CDE工具使用登录按钮，其中包含用户名，密码和远程数据库ID的字段。

6.6使用Easy Connect连接到数据库

               

                  
按照“ 确认网络可用性 ”中所述验证网络连接后，可以使用Easy Connect命名方法连接到数据库。此命名方法为数据库提供了开箱即用的TCP / IP连接。它通过使客户端除了数据库的主机名之外还可以使用可选的端口和服务名称连接到数据库服务器，从而扩展了主机命名方法的功能。以下是使用Easy Connect进行连接的语法：
CONNECT username / password @ host [： port ] [/ service_name ] [： server_type ] [/ instance_name ]

                     
注意：

                     
在Oracle Call Interface文档中， server称为connect_type 。
                     

                  

                  
如果在典型模式下执行Oracle数据库服务器安装，则Oracle实例使用的默认服务名称是数据库名称，并且可以使用以下Easy Connect语法连接到该实例：
SQLPLUS / nolog SQL> CONNECT username @“ host / db_name" SQL>输入密码： password

                     
注意：

                     
从Oracle Database 10g开始 ，Oracle数据库不支持使用Oracle Names。Oracle Database 12 c客户端和数据库无法使用Oracle名称（包括来自LDAP代理的名称）来解析命名。Oracle9 i客户端仍然可以使用Oracle Names来解析Oracle Database 12 c数据库的命名。但是，强烈建议客户迁移到LDAP以利用Oracle Database 12 c的新功能。
