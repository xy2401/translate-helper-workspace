REGEXP_SUBSTR

句法


插图说明regexp_substr.eps

目的

REGEXP_SUBSTR通过让您在字符串中搜索正则表达式模式REGEXP_SUBSTR扩展SUBSTR函数的功能。它也类似于REGEXP_INSTR ，但它不返回子字符串的位置，而是返回子字符串本身。如果您需要匹配字符串的内容但不需要它在源字符串中的位置，则此函数很有用。该函数将字符串作为VARCHAR2或CLOB数据返回到与source_char相同的字符集中。

此函数符合POSIX正则表达式标准和Unicode正则表达式指南。有关更多信息，请参阅Oracle正则表达式支持 。

  • source_char是用作搜索值的字符表达式。它通常是字符列，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB 。

  • pattern是正则表达式。它通常是文本文字，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 。它最多可包含512个字节。如果pattern的数据类型与source_char的数据类型不同，则Oracle数据库将pattern转换为source_char的数据类型。有关可以在pattern指定的运算符的列表，请参阅Oracle正则表达式支持 。

  • position是一个正整数，表示Oracle应该开始搜索的source_char的字符。默认值为1，表示Oracle在source_char的第一个字符处开始搜索。

  • occurrence是一个正整数，表示Oracle应该搜索的source_char哪个pattern出现。默认值为1，表示Oracle搜索第一次出现的pattern 。

    如果occurrence大于1，则数据库将搜索第二次出现，从第一次出现pattern后的第一个字符开始，依此类推。此行为与SUBSTR函数不同，后者开始在第一次出现的第二个字符处搜索第二个匹配项。

  • match_parameter是一个文本文字，可让您更改函数的默认匹配行为。此参数的行为与此函数的行为与REGEXP_COUNT 。有关详细信息，请参阅REGEXP_COUNT 。

  • 对于具有子表达式的pattern ， subexpr是从0到9的非负整数，表示函数返回pattern哪个子表达式。此参数与REGEXP_INSTR函数具有相同的语义。有关更多信息，请参阅REGEXP_INSTR 。

例子

以下示例检查字符串，查找以逗号为界的第一个子字符串。Oracle数据库搜索逗号，后跟一个或多个非逗号字符后跟逗号。Oracle返回子字符串，包括前导和尾随逗号。

SELECT REGEXP_SUBSTR（'500 Oracle Parkway，Redwood Shores，CA'，'，[^，] +，'）“REGEXPR_SUBSTR”FROM DUAL; REGEXPR_SUBSTR -----------------，Redwood Shores，

以下示例检查字符串，查找http://后跟一个或多个字母数字字符的子字符串，以及可选的句点（ . ）。Oracle在http://和斜杠（ / ）或字符串结尾之间搜索至少三次，最多四次出现此子字符串。 

SELECT REGEXP_SUBSTR（'http://www.example.com/products'，'http：//（[[：alnum：]] + \。？）{3,4} /？'）“REGEXP_SUBSTR”FROM DUAL; REGEXP_SUBSTR ---------------------- http://www.example.com/

接下来的两个示例使用subexpr参数返回pattern的特定子表达式。第一个语句返回pattern的第一个子表达式：

SELECT REGEXP_SUBSTR（'1234567890'，'（123）（4（56）（78））'，1,1，'i'，1）“REGEXP_SUBSTR”FROM DUAL; REGEXP_SUBSTR ------------------- 123

下一个语句返回pattern的第四个子表达式：

SELECT REGEXP_SUBSTR（'1234567890'，'（123）（4（56）（78））'，1,1，'i'，4）“REGEXP_SUBSTR”FROM DUAL; REGEXP_SUBSTR ------------------- 78

REGEXP_SUBSTR模式匹配：示例

以下语句创建表regexp_temp并将值插入其中：

CREATE TABLE regexp_temp（empName varchar2（20），emailID varchar2（20））; INSERT INTO regexp_temp（empName，emailID）VALUES（'John Doe'，'johndoe @ example.com'）; INSERT INTO regexp_temp（empName，emailID）VALUES（'Jane Doe'，'janedoe'）;

在以下示例中，语句查询电子邮件列并搜索有效的电子邮件地址：

SELECT empName，REGEXP_SUBSTR（emailID，'[[：alnum：]] + \ @ [[：alnum：]] + \。[[：alnum：]] +'）“有效电子邮件”FROM regexp_temp; EMPNAME有效电子邮件-------- ------------------- John Doe johndoe@example.com Jane Doe

在以下示例中，语句查询电子邮件列并返回有效电子邮件地址的计数：

SELECT empName，REGEXP_SUBSTR（emailID，'[[：alnum：]] + \ @ [[：alnum：]] + \。[[：alnum：]] +'）“有效电子邮件”，REGEXP_INSTR（emailID，'\ w + @ \ w +（\。\ w +）+'）“FIELD_WITH_VALID_EMAIL”FROM regexp_temp; EMPNAME有效电子邮件FIELD_WITH_VALID_EMAIL -------- ------------------- ------------------- --- John Doe johndoe@example.com 1 Jane Doe
