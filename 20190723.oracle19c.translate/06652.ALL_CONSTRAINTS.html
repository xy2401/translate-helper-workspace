2.88 ALL_CONSTRAINTS

ALL_CONSTRAINTS描述当前用户可访问的表的约束定义。

相关观点

  • DBA_CONSTRAINTS描述数据库中的所有约束定义。

  • USER_CONSTRAINTS描述当前用户架构中表的约束定义。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

约束定义的所有者

CONSTRAINT_NAME

VARCHAR2（128）

约束定义的名称

CONSTRAINT_TYPE

VARCHAR2（1）

约束定义的类型：

  • C - 检查表的约束

  • P - 主键

  • U - 唯一键

  • R - 参照完整性

  • V - 使用选中选项，在视图上

  • O - 只读，在视图上

  • H - 哈希表达式

  • F - 涉及REF列的约束

  • S - 补充记录

TABLE_NAME

VARCHAR2（128）

使用约束定义与表（或视图）关联的名称

SEARCH_CONDITION

长

检查约束的搜索条件的文本。仅当行来自当前容器时，此列才返回正确的值。

SEARCH_CONDITION_VC

VARCHAR2（4000）

检查约束的搜索条件的文本。此列可能会截断搜索条件。

R_OWNER

VARCHAR2（128）

在引用约束中引用的表的所有者

R_CONSTRAINT_NAME

VARCHAR2（128）

引用表的唯一约束定义的名称

DELETE_RULE

VARCHAR2（9）

删除参照约束的规则：

  • 级联

  • SET NULL

  • 没有行动

状态

VARCHAR2（8）

约束的执行状态：

  • 启用

  • 禁用

DEFERRABLE

VARCHAR2（14）

指示约束是否可延迟（ DEFERRABLE ）（ NOT DEFERRABLE ）

DEFERRED

VARCHAR2（9）

指示约束是否最初延迟（ DEFERRED ）或不（ IMMEDIATE ）

验证

VARCHAR2（13）

当STATUS = ENABLED ，可能的值为：

  • VALIDATED - 所有数据都遵循约束（即，在启用约束时验证表中的现有数据，以及输入表中的任何后续数据）

  • NOT VALIDATED - 所有数据都可能不遵守约束（即，启用约束时未验证表中的现有数据，但验证了输入表中的后续数据）

当STATUS = DISABLED ，可能的值为：

  • VALIDATED - 所有数据都遵循约束，但约束的唯一索引已被删除。此设置在数据仓库环境中很有用，但有一些限制。有关此设置的详细信息，请参阅Oracle数据库数据仓库指南 。

  • NOT VALIDATED -所有的数据可能不服从约束

GENERATED

VARCHAR2（14）

指示约束的名称是用户生成的（ USER NAME ）还是系统生成的（ GENERATED NAME ）

坏

VARCHAR2（3）

指示此约束是否以模糊方式（ BAD ）指定世纪（NULL）。为避免此歧义导致的错误，请使用年龄为四位数的TO_DATE函数重写约束。

另请参阅： Oracle数据库SQL语言参考和Oracle数据库开发指南中的TO_DATE函数

依靠

VARCHAR2（4）

当VALIDATED = NOT VALIDATED ，此列指示是否要在查询重写（ RELY ）时考虑约束（NULL）。

当VALIDATED = VALIDATED ，此列无意义。

另请参阅： Oracle数据库SQL语言参考中的 约束

ORIGIN_CON_ID

VARCHAR2（256）

数据源自的容器的ID。可能的值包括：

  • 0 ：该值用于非CDB中的行。该值不用于CDB。

  • n ：此值用于包含源自容器ID为n的容器中的数据的行（如果行来自root，则n = 1）
