191 DBMS_XDB_VERSION

Oracle XML DB版本控制接口位于DBMS_XBD_VERSION包中。DBMS_XDB_VERSION函数和过程有助于创建VCR并管理版本历史记录中的版本。

191.1 DBMS_XDB_VERSION安全模型

由XDB拥有， DBMS_XDB_VERSION包必须由SYS或XDB创建。 EXECUTE特权被授予PUBLIC 。此包中的子程序使用当前用户的权限执行。

191.2.1 CHECKIN功能

此功能检入签出的VCR并返回新创建的版本的资源ID。

句法

DBMS_XDB_VERSION.CHECKIN（pathname VARCHAR2）RETURN DBMS_XDB.resid_type;

参数

表191-2 CHECKIN功能参数

参数 描述

路径

签出资源的路径名。

使用说明

这不是自动提交SQL操作。CHECKIN函数不必采用传递给CHECKOUT Procedure操作的相同路径名。但是， CHECKIN函数路径名和CHECKOUT过程路径名必须具有相同的资源才能使操作正常运行。如果资源已重命名，则必须将新名称用于CHECKIN函数，因为旧名称无效或当前与其他资源绑定。如果路径名不存在，则引发异常。如果路径名已更改，则必须使用新路径名来CHECKIN功能资源。

191.2.2检查程序

此过程在更新或删除之前检出VCR。

句法

DBMS_XDB_VERSION.Checkout（pathname VARCHAR2）;

参数

表191-3检查过程参数

参数 描述

路径

要检出的VCR的路径名。

使用说明

这不是自动提交SQL操作。同一个工作区的两个用户无法检查过程在同一时间同一台录像机。如果发生这种情况，一个用户必须回滚。因此，最好在更新资源之前提交CHECKOUT Procedure操作，并避免在事务回滚时丢失更新。如果给定资源不是VCR，如果VCR已经签出，如果资源不存在，则引发异常。

191.2.11 MAKEVERSIONED功能

此函数将路径名称的常规资源转换为版本控制的资源。然后将此新资源置于版本控制之下。所有其他路径名继续引用原始资源。

句法

DBMS_XDB_VERSION.MAKEVERSIONED（pathname VARCHAR2）RETURN DBMS_XDB.resid_type;

参数

表191-12 MAKEVERSIONED函数参数

参数 描述

路径

要置于版本控制下的资源的路径名。

返回值

此函数返回VCR的第一个版本或根的资源ID。

使用说明

如果两个或多个路径名与同一资源绑定，则会创建资源的副本，并且给定的路径名将与新创建的副本绑定。

这不是自动提交SQL操作。如果资源不存在，则引发异常。

  • 此调用对于VCR是合法的，既不会引发异常也不会引发警告。

  • 此调用对于文件夹，版本历史记录，版本资源和ACL是非法的。

  • 不提供对基于Schema的资源的支持。

191.2.12 UNCHECKOUT功能

此函数签入签出资源，并在签出资源之前返回版本的资源ID。

句法

DBMS_XDB_VERSION.UNCHECKOUT（pathname VARCHAR2）RETURN DBMS_XDB.resid_type;

参数

表191-13 UNCHECKOUT函数参数

参数 描述

路径

签出资源的路径名。

使用说明

这不是自动提交SQL操作。UNCHECKOUT函数不必采用与CHECKOUT过程传递给操作的相同路径名。但是， UNCHECKOUT函数路径名和CHECKOUT过程路径名必须具有相同的资源才能使操作正常运行。如果资源已重命名，则必须将新名称用于UNCHECKOUT函数 ，因为旧名称无效或当前与其他资源绑定。如果路径名已更改，则新路径名必须用于UNCHECKOUT功能资源。

例外

如果路径名不存在，则引发异常。
