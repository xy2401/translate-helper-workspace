考虑属于oracle.jdbc包的API，特别是有助于SQL转换的oracle.jdbc元素。要成功迁移JDBC应用程序，了解转换属性，接口和错误转换机制非常重要。

备注

此文件仅用于转换连接建立失败时发生的错误。建立连接后，将绕过此文件，即使它包含建立连接后发生的任何错误的转换详细信息，也不会考虑该文件。属性sqlErrorTranslationFile可以是系统属性或连接属性。当没有有效连接时，此文件的内容用于将Oracle SQLExceptions转换为外部SQLExceptions。

使用说明

什么时候“ ?“占位符与prepareStatement()方法一起使用，驱动程序内部更改” ?“对于Oracle风格的参数，因为服务器端转换器只能使用Oracle风格的标记。这是区分绑定变量所必需的。如果不是，则绑定变量顺序的任何变化都是无法区分的。替换的oracle样式标记遵循以下格式:b< n >其中< n >是增量编号。例如， exec sample_proc(?,?) 变为exec sample_proc(:b1,:b2) 。

为了进一步举例说明，考虑一种供应商格式的场景，其中选择前三行的供应商查询是SELECT * FROM employees WHERE first_name=?AND employee_id =？TOP 3 。查询必须转换为oracle方言。在这种情况下，将在服务器上注册以下转换：

从：

SELECT * FROM employees WHERE first_name =：b1 AND employee_id =：b2 TOP 3

至： 

SELECT * FROM employees WHERE first_name =：b1 AND employee_id =：b2 AND ROWNUM <= 3

有关更多信息，请参见SqlTranslationVersion和“ JDBC应用程序的SQL转换 ” 。

例外

prepareStatement()方法抛出SQLException 。

getSQLTranslationVersions（）

在SQL Translation期间返回查询的所有翻译版本的映射。如果出现异常，并且如果suppressExceptions为true，则映射中的已翻译版本为NULL 。

句法

public Map <SqlTranslationVersion，String> getSqlTranslationVersions（String sql，boolean suppressExceptions）抛出SQL异常;

回报价值

映射查询的所有翻译版本。有关返回版本的更多详细信息，请参阅SqlTranslationVersion enum 。

例外

如果suppressExceptions为false，则在查询转换出现问题时，此方法抛出SQLException 。

错误翻译配置文件

提供XML配置文件（路径）作为的值 oracle.jdbc.sqlErrorTranslationFile属性。此文件包含错误的翻译信息。当无法建立与服务器的连接时，会发生这些错误，因此无法在服务器上进行转换。错误消息的类型定义了阻止建立连接的数据库状态。

配置XML文件的结构在DTD中定义如下：

<！DOCTYPE LocalTranslationProfile [<！ELEMENT LocalTranslationProfile（Exception +）> <！ELEMENT异常（ORAError，ErrorCode，SQLState）> <！ELEMENT ORAError（#PCDATA）> <！ELEMENT ErrorCode（#PCDATA）> <！ELEMENT SQLState（#PCDATA）>]>

哪里，

  • ORAError是一个int值，它指定了oracle错误的错误代码。

  • ErrorCode是一个int值，它指定供应商错误代码， ErrorCode翻译的代码。

  • SQLState是String值，指定供应商SQL状态。
