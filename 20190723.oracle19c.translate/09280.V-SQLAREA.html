9.70 V $ SQLAREA

V$SQLAREA显示共享SQL区域的统计信息，每个SQL字符串包含一行。它提供有关内存，解析和准备执行的SQL语句的统计信息。

柱 数据类型 描述

SQL_TEXT

VARCHAR2（1000）

当前游标的SQL文本的前千个字符

SQL_FULLTEXT

CLOB

当前游标的SQL文本的所有字符

SQL_ID

VARCHAR2（13）

库高速缓存中父游标的SQL标识符

SHARABLE_MEM

数

游标使用的共享内存量。如果存在多个子游标，则为所有子游标使用的所有共享内存的总和。

PERSISTENT_MEM

数

固定用于打开游标生命周期的内存量。如果存在多个子游标，则使用所有子游标的生命周期的固定内存总和。

RUNTIME_MEM

数

执行游标期间所需的固定内存量。如果存在多个子游标，则执行所有子游标期间所需的所有内存的固定总和。

排序

数

为所有子游标执行的排序数的总和

VERSION_COUNT

数

此父级下缓存中存在的子游标数

LOADED_VERSIONS

数

缓存中存在并加载其上下文堆的子游标数

OPEN_VERSIONS

数

当前父级下当前打开的子游标数

USERS_OPENING

数

打开任何子游标的用户数

取

数

与SQL语句关联的提取数

EXECUTIONS

数

所有子游标的总执行次数总计

PX_SERVERS_EXECUTIONS

数

并行执行服务器执行的总执行次数（当语句从未并行执行时为0 ）

END_OF_FETCH_COUNT

数

自光标进入库高速缓存后光标完全执行的次数。当光标部分执行时，此统计信息的值不会增加，因为它在执行期间失败，或者因为在关闭或重新执行游标之前仅获取此游标产生的前几行。根据定义， END_OF_FETCH_COUNT列的值应小于或等于EXECUTIONS列的值。

USERS_EXECUTING

数

在所有子游标上执行语句的用户总数

荷载

数

加载或重新加载对象的次数

FIRST_LOAD_TIME

VARCHAR2（76）

父创建时间的时间戳

废票

数

所有子游标上的失效总数

PARSE_CALLS

数

此父级下所有子游标的所有解析调用的总和

DISK_READS

数

所有子游标上的磁盘读取次数总和

DIRECT_WRITES

数

所有子游标的直接写入次数总和

DIRECT_READS

数

所有子游标的直接读取次数总和

BUFFER_GETS

数

缓冲区总和超过所有子游标

APPLICATION_WAIT_TIME

数

应用程序等待时间（以微秒为单位）

CONCURRENCY_WAIT_TIME

数

并发等待时间（以微秒为单位）

CLUSTER_WAIT_TIME

数

群集等待时间（以微秒为单位）

USER_IO_WAIT_TIME

数

用户I / O等待时间（以微秒为单位）

PLSQL_EXEC_TIME

数

PL / SQL执行时间（以微秒为单位）

JAVA_EXEC_TIME

数

Java执行时间（以微秒为单位）

ROWS_PROCESSED

数

代表此SQL语句处理的总行数

COMMAND_TYPE

数

Oracle命令类型定义

OPTIMIZER_MODE

VARCHAR2（10）

执行SQL语句的模式

OPTIMIZER_COST

数

优化程序给出的此查询的成本

OPTIMIZER_ENV

RAW（2000）

优化器环境

OPTIMIZER_ENV_HASH_VALUE

数

优化程序环境的哈希值

PARSING_USER_ID

数

已解析此父级下第一个光标的用户的用户标识

PARSING_SCHEMA_ID

数

用于解析此子游标的模式ID

PARSING_SCHEMA_NAME

VARCHAR2（128）

用于解析此子游标的模式名称

KEPT_VERSIONS

数

已标记为使用DBMS_SHARED_POOL包保留的子游标数

地址

RAW（4 | 8）

此游标的父句柄的地址

HASH_VALUE

数

库高速缓存中父语句的哈希值

OLD_HASH_VALUE

数

旧的SQL哈希值

PLAN_HASH_VALUE

数

此游标的当前SQL计划的数字表示形式。将一个PLAN_HASH_VALUE与另一个进行比较可以轻松识别两个计划是否相同（而不是逐行比较两个计划）。

FULL_PLAN_HASH_VALUE

数

此游标的完整SQL计划的数字表示形式。将一个FULL_PLAN_HASH_VALUE与另一个进行比较可以轻松识别两个计划是否相同（而不是逐行比较两个计划）。请注意，无法跨数据库版本比较FULL_PLAN_HASH_VALUE 。它不向后兼容。

MODULE

VARCHAR2（64）

包含通过调用DBMS_APPLICATION_INFO.SET_MODULE首次解析SQL语句时执行的模块的名称

MODULE_HASH

数

在MODULE列中命名的模块的哈希值

行动

VARCHAR2（64）

包含通过调用DBMS_APPLICATION_INFO.SET_ACTION首次解析SQL语句时执行的操作的名称

ACTION_HASH

数

在ACTION列中命名的ACTION哈希值

SERIALIZABLE_ABORTS

数

事务序列化失败的次数，产生ORA-08177错误，总计超过所有子游标

OUTLINE_CATEGORY

VARCHAR2（64）

如果在构造光标期间应用了轮廓，则此列将显示该轮廓的类别。否则该列留空。

CPU_TIME

数

此游标用于解析，执行和获取的CPU时间（以微秒为单位）

ELAPSED_TIME

数

此游标用于解析，执行和获取的经过时间（以微秒为单位）。如果游标使用并行执行，则ELAPSED_TIME是查询协调器的累积时间，加上所有并行查询从属进程。

OUTLINE_SID

VARCHAR2（40）

大纲会话标识符

LAST_ACTIVE_CHILD_ADDRESS

RAW（4 | 8）

作为组中最后一个活动子游标的子游标的地址（标识符）（即代表V$SQL中的统计信息更新的子游标）

远程

VARCHAR2（1）

指示光标是否远程映射（ Y ）或不是（ N ）

OBJECT_STATUS

VARCHAR2（19）

光标的状态：

  • VALID - 有效，授权无误

  • VALID_AUTH_ERROR - 有效，授权有授权错误

  • VALID_COMPILE_ERROR - 有效，具有编译错误的授权

  • VALID_UNAUTH - 有效，未经授权

  • INVALID_UNAUTH - 无效，未经授权

  • INVALID - 无效，未经授权但保留时间戳

LITERAL_HASH_VALUE

数

其被替换为系统生成的绑定变量，并且是文字的散列值进行匹配，当CURSOR_SHARING被使用。这不是SQL语句的哈希值。如果未使用CURSOR_SHARING ，则值为0 。

LAST_LOAD_TIME

日期

查询计划加载到库高速缓存的时间

IS_OBSOLETE

VARCHAR2（1）

指示光标是否已过时（ Y ）或未变为（ N ）。如果子游标的数量太大，就会发生这种情况。

IS_BIND_SENSITIVE

VARCHAR2（1）

指示光标是否绑定敏感（ Y ）或不绑定（ N ）。如果优化程序在计算谓词选择性时查看其绑定变量值之一并且绑定变量值的更改可能导致优化程序生成不同的计划，则查询将被视为绑定敏感。

IS_BIND_AWARE

VARCHAR2（1）

指示光标是否为绑定感知（ Y ）或不是（ N ）。如果已将查询标记为使用扩展游标共享，则将其视为绑定感知。该查询已被标记为绑定敏感。

CHILD_LATCH

数

保护光标的子锁存号。此列已过时并维护以便向后兼容。

SQL_PROFILE

VARCHAR2（64）

用于此语句的SQL配置文件（如果有）

SQL_PATCH

VARCHAR2（128）

用于此语句的SQL修补程序（如果有）

SQL_PLAN_BASELINE

VARCHAR2（128）

用于此语句的SQL计划基准（如果有）

PROGRAM_ID

数

程序标识符

PROGRAM_LINE＃

数

程序行号

EXACT_MATCHING_SIGNATURE

数

CURSOR_SHARING参数设置为EXACT时使用的签名

FORCE_MATCHING_SIGNATURE

数

CURSOR_SHARING参数设置为FORCE时使用的签名

LAST_ACTIVE_TIME

日期

查询计划最后一次处于活动状态的时间

BIND_DATA

RAW（2000）

绑定数据

TYPECHECK_MEM

数

Typecheck记忆

IO_CELL_OFFLOAD_ELIGIBLE_BYTES

数

可以由Exadata存储系统过滤的I / O字节数

另请参阅： Oracle Exadata Storage Server软件文档以获取更多信息

IO_INTERCONNECT_BYTES

数

Oracle数据库与存储系统之间交换的I / O字节数

PHYSICAL_READ_REQUESTS

数

受监控SQL发出的物理读取I / O请求数

PHYSICAL_READ_BYTES

数

受监视的SQL从磁盘读取的字节数

PHYSICAL_WRITE_REQUESTS

数

受监控SQL发出的物理写入I / O请求数

PHYSICAL_WRITE_BYTES

数

受监视的SQL写入磁盘的字节数

OPTIMIZED_PHY_READ_REQUESTS

数

受监控SQL发出的来自Database Smart Flash Cache的物理读取I / O请求数

LOCKED_TOTAL

数

子光标被锁定的总次数

PINNED_TOTAL

数

子光标固定的总次数

IO_CELL_UNCOMPRESSED_BYTES

数

卸载到Exadata单元的未压缩字节数（即解压缩后的大小）

另请参阅： Oracle Exadata Storage Server软件文档以获取更多信息

IO_CELL_OFFLOAD_RETURNED_BYTES

数

Exadata单元通过常规I / O路径返回的字节数

另请参阅： Oracle Exadata Storage Server软件文档以获取更多信息

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

IS_REOPTIMIZABLE

VARCHAR2（1）

此列显示匹配此子游标的下一个执行是否将触发重新优化。价值观是：

  • Y ：如果下一次执行将触发重新优化

  • R ：如果子光标包含重新优化信息，但由于光标是在报告模式下编译的，因此不会触发重新优化

  • N ：如果子光标没有重新优化信息

IS_RESOLVED_ADAPTIVE_PLAN

VARCHAR2（1）

此列显示计划的所有自适应部分是否已解析为最终计划。解决方案后， DBMS_XPLAN显示的计划哈希值和计划在执行结束时不会更改。此列的值为：

  • NULL ：如果计划不适应

  • Y ：如果计划完全解决了

  • N ：如果计划尚未完全解决
