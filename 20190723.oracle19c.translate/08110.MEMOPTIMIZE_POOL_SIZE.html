1.195 MEMOPTIMIZE_POOL_SIZE

MEMOPTIMIZE_POOL_SIZE设置memoptimize池的大小，Memoptimized Rowstore使用的系统全局区域（SGA）中的内存区域。

属性 描述

参数类型

大整数

句法

MEMOPTIMIZE_POOL_SIZE = integer [K | M | G]

默认值

0

可修改

没有

可在PDB中修改

没有

价值范围

0到没有最大值

基本

没有

Oracle RAC

可以在不同的实例上使用不同的值。

Memoptimized Rowstore提高了应用程序（如物联网（IoT））的数据查询性能，这些应用程序经常根据主键值查询表。

Memoptimized Rowstore提供了快速查找主要基于主键列查询的表的数据的功能。

此参数指定一个整数值，以指示用于为memoptimize池分配以下结构的SGA数量：

  • 缓冲区缓存区域的大小：这是所有MEMOPTIMIZE FOR READ表的块总数。

  • 散列索引的大小分段数据结构，指向MEMOPTIMIZE FOR READ表的特殊块。

计算正在考虑用于MEMOPTIMIZE FOR READ的表的缓冲区高速缓存要求，并包括对散列索引分段数据结构的额外25％内存要求。

这些结构在实例启动时从SGA分配。

为此参数指定的值将计入SGA_TARGET 。例如，如果将SGA_TARGET设置为10 GB并将MEMOPTIMIZE_POOL_SIZE设置为2 GB，则会将20％的SGA_TARGET设置分配给memoptimize池。

与其他SGA组件（如缓冲区高速缓存和共享池）不同，memoptimize池大小不受自动内存管理控制。当缓冲区缓存或共享池需要更多内存时，数据库不会自动收缩memoptimize池，或者当空间不足时增加memoptimize池。您只能通过手动调整MEMOPTIMIZE_POOL_SIZE初始化参数来增加memoptimize池的大小。
