10 XML C API的包遍历

Package Traversal包含实现遍历规则和行为的API。为方便起见，我们将API分为四种通用接口类型：

  • DocumentTraversal接口

  • NodeFilter接口

  • NodeIterator接口

  • TreeWalker接口

表10-2 XML C实现的NodeFilter遍历方法摘要

功能 摘要

XMLDOM_ACCEPT_NODE_F（）

确定基于节点adn过滤器的过滤操作。

XMLDOM_ACCEPT_NODE_F（）

NodeFilter接口的唯一方法。给定节点和过滤器，确定要执行的过滤操作。

根据需要，此函数指针将传递给节点iterator / tree walker方法。

xmlerr的值是：

  • XMLERR_OK接受节点。为NodeIterator或TreeWalker定义的导航方法将返回此节点。

  • XMLERR_FILTER_REJECT拒绝节点。为NodeIterator或TreeWalker定义的导航方法不会返回此节点。对于TreeWalker ，此节点的子节点也将被拒绝。NodeIterator将此视为XMLDOM_FILTER_SKIP的同义词

  • XMLERR_FILTER_SKIP跳过此单个节点。为NodeIterator或TreeWalker定义的导航方法不会返回此节点。对于NodeIterator和TreeWalker ，仍将考虑此节点的子节点。

句法

#define XMLDOM_ACCEPT_NODE_F（func，xctx，node）xmlerr func（xmlctx * xctx，xmlnode * node）;

XmlDomWalkerGetRoot（）

返回（获取）根节点，或者出错时为NULL 。由于当前节点可以从根节点下与其所属的子树一起移除，因此walker中的当前根节点可能与当前节点无关。TreeWalker迭代基于当前节点。但是，根节点定义迭代的空间。此函数检查根节点是否仍与当前节点的根节点（祖先）关系。如果是，则返回此根节点。否则，它找到当前节点所属树的根，并设置并返回该根作为walker的根节点。如果walker是NULL指针，则返回NULL 。
