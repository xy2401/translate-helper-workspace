数据库开发人员的职责

Oracle开发人员负责创建或维护使用Oracle技术堆栈的应用程序的数据库组件。

Oracle开发人员要么开发新应用程序，要么转换现有应用程序以在Oracle数据库环境中运行因此，开发人员与数据库管理员密切合作，共享知识和信息。

Oracle数据库开发人员可以参与以下任务：

  • 实现应用程序所需的数据模型

  • 创建架构对象

  • 实施数据完整性规则

  • 为新开发项目选择编程环境

  • 编写服务器端PL / SQL或Java子程序以及使用SQL语句的客户端程序代码

  • 使用所选的开发工具创建应用程序界面

  • 建立全球化支持环境以开发全球化应用程序

  • 在不同数据库中实例化应用程序，以便在生产环境中进行开发，测试，培训和部署

应用程序设计和调优原理

Oracle开发人员必须设计，创建和调整数据库应用程序，以便实现安全性和性能目标。

以下应用程序设计和调优原则是有用的指导原则：

  • 了解Oracle数据库的工作原理

    作为开发人员，您希望在最少的时间内针对Oracle数据库开发应用程序，这需要利用数据库体系结构和功能。例如，不了解Oracle数据库数据并发控制和多版本读取一致性可能会使应用程序破坏数据的完整性，运行缓慢并降低可伸缩性。了解Transaction Guard和Application Continuity的工作原理可以避免编写不必要的异常处理代码。

  • 除非您有充分的理由不使用它们，否则请使用绑定变量

    当查询使用绑定变量时，数据库可以编译一次并将查询计划存储在共享池中 。如果再次执行相同的语句，则数据库可以执行软解析并重用该计划。相比之下， 硬解析需要更长时间并使用更多资源。使用绑定变量来允许软解析非常有效，并且是数据库打算让开发人员工作的方式。

  • 在数据库服务器而不是客户端中实现完整性约束

    使用主键和外键可以在多个应用程序中重用数据。在客户端中编写规则意味着其他客户端在针对数据库运行时无权访问这些规则。

  • 使用代表性数据和会话活动构建测试环境

    模拟您的实时生产环境的测试环境提供了多种好处。例如，您可以对应用程序进行基准测试，以确保它可以扩展并运行良好。此外，您可以使用测试环境来衡量更改对数据库的性能影响，并确保升级和修补程序正常工作。

  • 以良好的性能为目标设计数据模型

    通常，尝试使用通用数据模型会导致性能不佳。精心设计的数据模型尽可能高效地回答最常见的查询。例如，数据模型应使用提供最佳性能的索引类型。部署之后的调整是不合需要的，因为逻辑和物理结构的改变可能是困难的或不可能的。

  • 定义明确的绩效目标并保留指标的历史记录

    开发的一个重要方面是确切地确定应用程序的执行和扩展程度。例如，使用包含预期用户负载，每秒事务数，可接受响应时间等的度量标准。良好实践要求您保留性能指标的历史记录。通过这种方式，您可以主动和被动地监控性能。

  • 检测应用程序代码

    良好的开发实践涉及向应用程序添加调试代码。生成跟踪文件的功能对于调试和诊断性能问题很有用。

Oracle预编译器

客户端/服务器程序通常使用Oracle 预编译器编写，Oracle 预编译器是一种编程工具，使您可以在高级程序中嵌入SQL语句。

例如，Oracle Pro * C / C ++预编译器允许您将SQL语句嵌入C或C ++源文件中。Oracle预编译器也可用于COBOL和FORTRAN。

预编译器提供了多种好处，包括：

  • 提高生产力，因为您通常编写的代码少于等效的OCI应用程序

  • 使您可以创建高度自定义的应用程序

  • 允许密切监视资源使用，SQL语句执行和各种运行时指示器

  • 节省时间，因为预编译器（而非您）将每个嵌入式SQL语句转换为对Oracle数据库运行时库的调用

  • 使用对象类型转换器将Oracle数据库对象类型和集合映射到要在Pro * C / C ++应用程序中使用的C数据类型

  • 提供对象类型和集合的编译时类型检查以及从数据库类型到C数据类型的自动类型转换

包含SQL语句的客户端应用程序是宿主程序。该程序是用宿主语言编写的。在宿主程序中，您可以将完整的SQL语句与完整的C语句混合，并在SQL语句中使用C变量或结构。嵌入SQL语句时，必须使用关键字EXEC SQL开始它们，并以分号结束它们。Pro * C / C ++将EXEC SQL语句转换为对运行时库SQLLIB的调用。

许多嵌入式SQL语句与其交互式对应语句的区别仅在于添加了新子句或使用了程序变量。以下示例比较了交互式和嵌入式ROLLBACK语句：

ROLLBACK; - 交互式EXEC SQL ROLLBACK; - 嵌入式

这些语句具有相同的效果，但您可以在交互式SQL环境（例如SQL Developer）中使用第一个语句，在Pro * C / C ++程序中使用第二个语句。

预编译器接受主机程序作为输入，将嵌入式SQL语句转换为标准数据库运行时库调用，并生成可以通常方式编译，链接和运行的源程序。图19-2说明了开发使用预编译器的程序的典型步骤。

客户端API

今天大多数开发人员使用API将SQL嵌入其数据库应用程序中。

例如，两个用于使程序与Oracle数据库通信的流行API是开放式数据库连接（ODBC）和JDBC。 Oracle调用接口（OCI）和Oracle C ++调用接口（OCCI）是另外两种用于客户端编程的常用API。

OCI和OCCI

作为预编译器的替代方案，Oracle提供了OCI和OCCI API。

OCI允许您使用主机编程语言（如C）操作数据库中的数据和模式.OCCI是一种适合与C ++一起使用的面向对象的接口。这两个API使开发人员能够使用本机子程序调用来访问Oracle数据库并控制SQL执行。

在某些情况下，OCI提供比更高级别接口更好的性能或更多功能。OCI和OCCI提供许多功能，包括：

  • 支持Oracle数据库提供的所有SQL DDL ， DML ，查询和事务控制工具

  • 即时客户端，一种在磁盘空间出现问题时部署应用程序的方法

  • 线程管理，连接池，全球化功能以及来自C应用程序的数据的直接路径加载

OCI和OCCI以动态运行时库（OCILIB）的形式提供标准数据库访问和检索功能库。可以在运行时在应用程序中链接此库。因此，您可以以与非数据库应用程序相同的方式编译和链接OCI或OCCI程序，从而避免单独的预处理或预编译步骤。图19-3说明了开发过程。

ODBC和JDBC

ODBC是一种标准API，它使应用程序能够连接到数据库，然后准备和运行SQL语句。

ODBC独立于编程语言，数据库和操作系统。ODBC的目标是使任何应用程序都能访问任何数据库中包含的数据。

数据库驱动程序是位于应用程序和数据库之间的软件。驱动程序将应用程序发出的API调用转换为数据库可以处理的命令。通过使用ODBC驱动程序，应用程序可以访问任何数据源，包括存储在电子表格中的数据。ODBC驱动程序执行ODBC标准和数据库之间的所有映射。

Oracle提供的Oracle ODBC驱动程序使符合ODBC的应用程序能够访问Oracle数据库。例如，用Visual Basic编写的应用程序可以使用ODBC来查询和更新Oracle数据库中的表。

JDBC是一种低级Java接口，它使Java应用程序能够与Oracle数据库进行交互。与ODBC一样，JDBC是独立于供应商的API。 JDBC标准由Sun Microsystems定义，并通过java.sql接口实现。

JDBC标准使各个提供程序能够使用自己的JDBC驱动程序实现和扩展标准。Oracle为客户端编程提供以下JDBC驱动程序：

  • JDBC瘦驱动程序

    此纯Java驱动程序驻留在客户端，无需安装Oracle客户端。它独立于平台，可与applet和应用程序一起使用。

  • JDBC OCI驱动程序

    此驱动程序驻留在客户端，并安装了Oracle客户端。它仅适用于应用程序。JDBC OCI驱动程序（用C和Java编写）将JDBC调用转换为OCI调用。

全球化支持使您能够开发多语言应用程序和软件，可以在世界任何地方同时访问和运行。

编写全球化数据库应用程序的开发人员必须执行以下操作：

全球化支持环境

全球化支持环境包括客户端应用程序和数据库。您可以通过在客户端和服务器上设置参数和环境变量来控制与语言相关的操作，这些变量可能存在于不同的位置。

注意：

在以前的版本中，Oracle将全球化支持功能称为国家语言支持（NLS）功能。NLS实际上是全球化支持的一个子集，它提供了选择国家语言并将数据存储在特定字符集中的能力。

Oracle数据库为以下功能提供全球化支持：

  • 母语和领地

  • 日期，时间，数字和货币的本地格式

  • 日历系统（阳历，日本帝国，泰国佛等）

  • 多个字符集，包括Unicode

  • 字符语义

字符集

全球化支持的一个关键组成部分是字符集 ，它是一种用于在计算机屏幕上显示字符的编码方案。

以下区别在应用程序开发中很重要：

  • 数据库字符集确定可以在数据库中表示哪些语言。字符集在数据库创建时指定。

    注意：

    创建数据库后，更改其字符集通常在时间和资源方面非常昂贵。此操作可能需要通过导出整个数据库并将其导回来转换所有字符数据。

  • 客户端字符集是客户端应用程序输入或显示的数据的字符集。客户端和数据库的字符集可以不同。

可以将一组字符（例如，字母字符，表意符号，符号，标点符号和控制字符）编码为字符集。编码字符集为集合中的每个字符分配唯一的数字代码，称为代码点或编码值。代码点在全局环境中很重要，因为可能需要在不同字符集之间转换数据。

计算机行业使用许多编码字符集。这些集在可用字符数，可用字符，分配给每个字符的代码点等方面有所不同。Oracle数据库支持大多数国家，国际和供应商特定的编码字符集标准。

Oracle数据库支持以下类别的编码字符集：

  • 单字节字符集

    每个字符占用一个字节。US7ASCII是一个7位字符集的例子。一个8位字符集的例子是WE8DEC。

  • 多字节字符集

    每个字符占用多个字节。多字节集通常用于亚洲语言。

  • 统一

    通用编码字符集使您可以使用单个字符集以任何语言存储信息。Unicode为每个角色提供唯一的代码值，无论平台，程序或语言如何。

特定于区域设置的设置

区域设置是运行系统或程序的语言和文化环境。NLS参数确定客户端和数据库上的特定于语言环境的行为。

代表客户端执行语句时，数据库会话使用NLS设置。例如，数据库为客户端生成千位分隔符的正确区域使用。通常，客户端主机上的NLS_LANG环境变量指定服务器会话和客户端应用程序的区域设置。过程如下：

  1. 客户端应用程序启动时，它会从环境设置初始化客户端NLS环境。

    在本地执行的所有NLS操作（例如在Oracle Developer应用程序中显示格式）都使用这些设置。

  2. 客户端在连接时将NLS_LANG定义的信息传递给数据库。

  3. 数据库会话根据客户端传达的设置初始化其NLS环境。

    如果客户端未指定设置，则会话将使用初始化参数文件中的设置。仅当客户端未指定任何NLS设置时，数据库才使用初始化参数设置。如果客户端指定了一些NLS设置，则剩余的NLS设置为默认值。

什么是JSON？

JavaScript Object Notation（JSON）是一种与语言无关的基于文本的数据格式，可以表示对象，数组和标量数据。各种编程语言可以解析和生成JSON数据。

JSON通常用作数据交换语言。它通常用于序列化结构化数据并通过网络进行交换，通常在服务器和Web应用程序之间进行。JSON是运行以HTML嵌入式JavaScript编写的应用程序的Web浏览器中的主要数据格式。

JavaScript对象是零个或多个属性名称对和关联的JSON值的关联数组。JSON对象是一个JavaScript对象文字，它被写为括在括号中的属性列表，其中名称 - 值对用逗号分隔，每个对的名称和值用冒号分隔。对象属性有时称为键 。对象属性名称 - 值对有时称为对象成员 。

JSON的本机数据库支持

JSON广泛存储在缺少关系数据库功能的noSQL数据库中。相比之下，Oracle数据库本身支持JSON，具有事务，索引，声明性查询和视图等功能。

您可以像访问其他数据库数据一样访问存储在数据库中的JSON数据，包括使用OCI，.NET和JDBC。与使用SQL数据类型XMLType存储的XML数据不同，JSON数据使用VARCHAR2 ， BLOB或CLOB存储。通过使用Oracle SQL，您可以在JSON数据上执行以下操作：

  • 使用非JSON关系数据加入JSON数据。

  • 使用SQL函数json_object和json_array从关系数据生成JSON文档。

  • 使用SQL函数json_table将JSON数据json_table为关系格式。

  • 使用is_json创建检查约束以在列中强制执行JSON数据。数据库使用检查约束来确认该列是JSON，以用于特定于JSON的操作，例如简化语法。

  • 将JSON文档作为PL / SQL对象进行操作。

  • 使用SQL函数json_query和json_value接受Oracle JSON路径表达式作为参数，并将其与目标JSON数据进行匹配。

  • 索引JSON数据。

  • 查询存储在外部表中的JSON数据。

  • 使用Oracle GoldenGate复制包含包含JSON数据的列的表。

文本JSON数据始终使用Unicode字符集，UTF8或UTF16。Oracle数据库在分析和查询JSON数据时在内部使用UTF8。如果输入到这种处理或从这种处理输出的数据必须与UTF8的字符集不同，则自动执行适当的字符集转换。

LOB概述

大对象（LOB）数据类型使您能够以二进制或字符格式存储和操作大块非结构化数据。

大对象（LOB）数据类型提供对数据的有效，随机，分段访问。

内部LOB

内部LOB将数据存储在数据库本身而不是外部文件中。

内部LOBS包括以下内容：

  • CLOB （字符LOB），用于在数据库字符集中存储大量文本，如文本或XML文件

  • NCLOB （国家字符集LOB），用于存储Unicode数据

  • BLOB （二进制LOB），它将大量二进制信息存储为比特流，不受字符集转换的影响

数据库以不同于其他数据类型的方式存储LOB。创建LOB列会隐式创建LOB段和LOB索引。包含LOB段和LOB索引的表空间始终存储在一起，可能与包含该表的表空间不同。

注意：

有时，数据库可以在表本身中存储少量LOB数据，而不是在单独的LOB段中。

LOB段将数据存储在称为块的块中 。块是逻辑上连续的数据块集，是LOB的最小分配单位。表中的一行存储一个名为LOB定位符的指针，该指针指向LOB索引。查询表时，数据库使用LOB索引快速定位LOB块。

数据库管理LOB段的读取一致性与其他数据不同。数据库不是使用撤消数据来记录更改，而是将前映像存储在段本身中。当事务更改LOB时，数据库会分配新块并保留旧数据。如果事务回滚，则数据库将更改回滚到索引，该索引指向旧块。

外部LOB

BFILE （二进制文件大对象，或LOB）是外部大对象。

BFILE是外部LOB，因为数据库存储指向操作系统中文件的指针。外部数据是只读的。

BFILE使用目录对象来定位数据。消耗的空间量取决于目录对象名称的长度和文件名的长度。

BFILE不使用与内部LOBS相同的读取一致性机制，因为二进制文件在数据库外部。如果文件中的数据发生更改，则从同一二进制文件重复读取可能会产生不同的结果。

SecureFiles的

SecureFiles LOB存储是两种存储类型之一;另一种类型是BasicFiles LOB存储。

SECUREFILE LOB参数启用高级功能，包括压缩和重复数据删除（高级压缩选项的一部分）和加密（高级安全选项的一部分）。从Oracle Database 12c开始 ，SecureFiles是LOB的默认存储机制。

Oracle Text概述

Oracle Text（Text）是一种与Oracle数据库集成的全文检索技术。Oracle Text索引存储在文件系统，数据库或Web上的任何文档或文本内容。可以基于文本内容，元数据或属性来搜索这些文档。

Oracle Text具有以下优势：

  • Oracle Text允许在单个SQL语句中将文本搜索与常规数据库搜索结合使用。Text索引位于数据库中，Text查询在Oracle Database进程中运行。优化程序可以为任何查询选择最佳执行计划 ，为涉及文本和结构化条件的即席查询提供最佳性能。

  • 您可以将Oracle Text与XML数据一起使用。特别是，您可以将XMLIndex与Oracle Text索引结合使用，同时利用XML和全文索引。

  • Oracle Text SQL API使创建和维护Oracle Text索引和运行搜索变得简单直观。
