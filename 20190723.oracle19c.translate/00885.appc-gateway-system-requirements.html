网络附件要求

网关需要任何网络附件支持适用于您的平台的SNA通信包或用于TCP / IP通信的TCP / IP网络工具。

但是，如果仅使用新的TCP / IP支持IMS Connect功能，则不需要SNA软件包。您的操作系统已安装TCP / IP。

本节中描述的系统软件配置由Oracle支持，前提是底层系统软件产品由其各自的供应商提供支持。与系统软件供应商核实最新支持状态。

操作系统要求

请参阅Oracle数据库安装指南和My Oracle Support上的认证表，获取最新的认证操作系统版本要求列表，以便为Linux 64位系统，基于AIX或Solaris系统运行网关。My Oracle Support Web站点位于

https://support.oracle.com

通信协议要求

每个操作系统都使用特定的通信服务器，如本节所述。

注意：

如果选择对IMS Connect使用TCP / IP支持作为通信协议，则无需使用以下列表中的SNA通信包。每个操作系统都自动安装了TCP / IP协议。如果选择使用TCP / IP协议，则需要将其配置为与网关一起正常工作。请参阅使用TCP / IP通信协议的网关配置 。

适用于基于Intel Pentium的32位系统的Linux通信协议

用于基于Intel Pentium的32位系统的Linux通信协议适用于Linux或更高版本的IBM Communications Server V6.0.1.1，或用于操作系统的TCP / IP通信软件。

基于AIX的系统的通信协议

SNA Server for AIX 6.01或更高版本，或操作系统附带的TCP / IP通信软件。

Solaris通信协议

SNAP-IX版本6或更高版本，或操作系统附带的TCP / IP通信软件。

Oracle数据库要求

所有UNIX平台都需要Oracle数据库，即充当 Oracle数据库是最新的，支持Oracle数据库版本的最新补丁集。

Oracle网络产品要求

Oracle Net自动安装在系统所在的位置安装Oracle数据库并安装在安装网关的系统上。有关详细配置信息，请参阅本指南中的配置Oracle网络 。此外，您可以参考Oracle数据库网络服务管理员指南 。

IBM大型机要求

除了网关和正在使用的平台的其他软件要求之外，以下列表还概述了IBM大型机上所需的其他要求：

  • OLTP for SNA

    OLTP必须支持映射的APPC对话。如果通过网关执行的OLTP事务处理程序进行数据库更新，那么APPC动词CONFIRM ， CONFIRMED ，并SEND_ERR必须由OLTP支持。这些动词实现了APPC SYNCLEVEL 1 。

    由OLTP控制的所有可由通过网关调用的事务程序更新的资源必须定义为OLTP和主机系统的可恢复资源（如果COMMIT / 这些资源需要ROLLBACK功能。例如，由a更新的VSAM文件必须将CICS事务定义为CICS，作为COMMIT / ROLLBACK的可恢复文件来控制更新。

    该网关兼容所有支持SNA的产品版本，如CICS，IMS / TM，IDMS和 z / OS上。

    注意：

    有关已知限制的列表，请在继续安装网关之前阅读“ 已知限制 ”部分。

  • OLTP for TCP / IP

    IMS / TM：需要7.1或更高版本，以及“IBM IMS Connect指南和参考”中列出的任何APAR（修补程序）。

    I MS Connect：需要1.2或更高版本。
