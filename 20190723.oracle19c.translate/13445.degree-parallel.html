设定并行度

平行度 （DOP） 的 程度是与单个操作相关联的并行处理服务器的数量。

并行执行旨在有效地使用多个CPU。Oracle数据库并行执行框架使您能够明确选择特定的并行度，或依靠Oracle数据库自动控制它。

本节包含以下主题：

  • 手动指定并行度

  • 默认的并行度

  • 自动并行度

  • 确定自动DOP中的并行度

  • 控制自动并行度

  • 自适应并行

手动指定并行度

可以从Oracle数据库为表和索引请求特定的并行度（DOP）。

例如，您可以使用以下内容在表级别设置固定DOP：

ALTER TABLE销售PARALLEL 8; ALTER TABLE客户PARALLEL 4;

在此示例中，仅访问sales表的查询请求DOP为8，并且访问customers表的查询请求DOP为4。访问sales表和customers表的查询使用DOP 8进行处理，并可能分配16个并行执行服务器（因为生产者/消费者模型）。无论何时指定不同的DOP，Oracle数据库都使用更高的DOP。

您还可以使用语句级别或对象级别并行提示来请求特定DOP。

仅当PARALLEL_DEGREE_POLICY设置为MANUAL或LIMITED时，表或索引的PARALLEL子句中指定的DOP才会生效。

Oracle Database Resource Manager可以限制语句的实际运行时DOP。

默认的并行度

如果指定了PARALLEL子句但未列出任何并行度（DOP），则该对象将获得默认DOP。

例如，您可以使用以下SQL语句将表设置为默认DOP。

ALTER TABLE销售PARALLEL;

默认并行性使用公式根据系统配置确定DOP，如下所示：

  • 对于单个实例，DOP = PARALLEL_THREADS_PER_CPU x CPU_COUNT

  • 对于Oracle RAC配置，DOP = PARALLEL_THREADS_PER_CPU x sum(CPU_COUNT)

默认情况下， sum(CPU_COUNT)是群集中的CPU总数。但是，如果您使用Oracle RAC服务来限制并行操作可以执行的节点数，则sum(CPU_COUNT)是属于该服务的节点上的CPU总数。例如，在4节点Oracle RAC集群上，每个节点具有8个CPU核心且没有Oracle RAC服务，默认DOP将为2 x（8 + 8 + 8 + 8）= 64。

您还可以使用语句级别或对象级别并行提示来请求默认DOP。

仅当PARALLEL_DEGREE_POLICY设置为MANUAL时，表或索引的PARALLEL子句中指定的默认DOP才会生效。

默认DOP算法旨在使用最大资源，并假设如果可以使用更多资源，则操作会更快完成。默认DOP以单用户工作负载为目标，不建议在多用户环境中使用。

Oracle数据库资源管理器可以限制SQL语句的实际运行时DOP。

自动并行度

自动并行度（Auto DOP）使Oracle数据库能够自动决定语句是否应该并行执行以及它应该使用什么DOP。

以下是启用Auto DOP时并行语句处理的摘要。

  1. 发出SQL语句。

  2. 解析该语句，优化器确定执行计划。

  3. 将检查PARALLEL_MIN_TIME_THRESHOLD初始化参数指定的阈值限制。

    1. 如果预期执行时间小于阈值限制，则SQL语句将以串行方式运行。

    2. 如果预期执行时间大于阈值限制，则语句将根据优化程序计算的DOP并行运行，包括任何已定义资源限制的因子分解。

确定自动DOP中的并行度

通过自动并行度（DOP），优化器可根据该语句的资源要求自动确定语句的DOP。

优化程序使用所有扫描操作的成本（例如全表扫描或索引快速完全扫描）以及执行计划中所有CPU操作的成本来确定必要的DOP。

但是，优化器会限制实际的最大DOP，以确保并行执行服务器不会压倒系统。此限制由参数PARALLEL_DEGREE_LIMIT设置。此参数的默认值为CPU ，这意味着DOP受系统上CPU数量（ PARALLEL_THREADS_PER_CPU * sum(CPU_COUNT) ）的限制，也称为默认DOP。此默认DOP可确保单个用户操作不会使系统崩溃。通过调整此参数设置，您可以控制优化程序可以为SQL语句选择的最大DOP。如果使用Oracle数据库资源管理器限制DOP，优化程序可以进一步限制可以选择的最大DOP。

注意：

PARALLEL_DEGREE_LIMIT的值AUTO与CPU值具有相同的功能。

要计算SQL语句的操作成本，Auto DOP使用有关系统硬件特征的信息。硬件特性包括I / O校准统计数据，因此应收集这些统计数据。

如果未运行I / O校准以收集所需的统计数据，则使用默认校准值来计算操作成本和DOP。

可以使用PL / SQL DBMS_RESOURCE_MANAGER.CALIBRATE_IO过程收集I / O校准统计信息。如果物理硬件没有改变，I / O校准是一次性操作。

控制自动并行度

有几个初始化参数可以控制自动并行度（自动DOP）。这些初始化参数包括PARALLEL_DEGREE_POLICY ， PARALLEL_DEGREE_LIMIT ， PARALLEL_MIN_TIME_THRESHOLD和PARALLEL_MIN_DEGREE 。

初始化参数PARALLEL_DEGREE_POLICY控制是否启用Auto DOP，并行语句排队和内存中并行执行。此参数具有以下可能值：

  • 手册

    此设置禁用自动DOP，并行语句排队和内存中并行执行。它将并行执行的行为恢复为Oracle Database 11g第2版（11.2）之前的行为，这是默认设置。

    对于PARALLEL_DEGREE_POLICY的默认设置MANUAL ，系统仅在对象上显式设置DOP或在SQL语句中指定并行提示时才使用并行执行。使用的DOP正是指定的。不会发生并行语句排队和内存中并行执行。

  • 有限

    此设置为某些语句启用Auto DOP，但禁用并行语句排队和内存中并行执行。自动DOP仅应用于访问使用PARALLEL子句显式声明的表或索引的语句，而不指定显式DOP。指定了DOP的表和索引使用显式DOP设置。

    如果希望Oracle数据库仅针对触及特定对象子集的SQL语句子集自动确定DOP，则将PARALLEL_DEGREE_POLICY设置为LIMITED并设置并行属性，而不在该对象子集上指定显式DOP。

  • 汽车

    此设置为所有语句启用Auto DOP，还启用并行语句排队和内存中并行执行。

    如果希望Oracle数据库自动决定所有SQL语句的DOP，请将PARALLEL_DEGREE_POLICY设置为AUTO 。

  • 自适应

    此设置启用Auto DOP，并行语句排队和内存中并行执行，类似于AUTO值。此外，还启用了性能反馈。

PARALLEL_DEGREE_LIMIT初始化参数指定Auto DOP可在系统范围内使用的最大DOP。要对最大DOP进行更细粒度的控制，可以使用Oracle数据库资源管理器。

PARALLEL_MIN_TIME_THRESHOLD初始化参数指定要为Auto DOP考虑的语句的最小估计执行时间。首先，优化器计算SQL语句的串行执行计划。如果估计的执行时间大于PARALLEL_MIN_TIME_THRESHOLD的值，则该语句将成为Auto DOP的候选者。

PARALLEL_MIN_ DEGREE初始化参数控制由自动并行度计算的最小并行度。但是，如果PARALLEL_MIN_DEGREE的值大于CPU_COUNT的值，或者对象是Oracle拥有的，则PARALLEL_MIN_DEGREE没有影响，例如在字典表上创建的字典表或视图。

您还可以通过指定适当的语句级别或对象级别SQL提示来请求Auto DOP。

自适应并行

随着系统负载的增加，自适应多用户算法降低了并行度。

使用Oracle数据库自适应并行功能时，数据库在SQL执行时使用算法来确定并行操作是应该接收请求的DOP还是降低其DOP以确保系统不会过载。

在通过使用高DOP积极使用并行执行的系统中，当仅少数操作并行运行时，自适应算法调低DOP。虽然算法仍然确保最佳资源利用率，但用户可能会遇到不一致的响应时间。不建议在需要确定性响应时间的环境中仅使用自适应并行功能。通过数据库初始化参数PARALLEL_ADAPTIVE_MULTI_USER控制自适应PARALLEL_ADAPTIVE_MULTI_USER 。

注意：

由于初始化参数PARALLEL_ADAPTIVE_MULTI_USER在Oracle Database 12c第2版（12.2.0.1）中已弃用且未来版本中不支持，因此Oracle建议使用并行语句排队。
