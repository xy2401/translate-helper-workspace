D Oracle接口配置工具（OIFCFG）命令参考

该 Oracle接口配置工具（ OIFCFG）命令行界面可帮助您定义和管理网络接口。您可以使用OIFCFG命令 Oracle Clusterware环境：

  • 分配和释放组件的网络接口

  • 直接组件使用特定的网络接口

  • 检索组件配置信息

本附录包括以下主题：

  • 启动OIFCFG命令行界面

  • OIFCFG用法摘要

启动OIFCFG命令行界面

在调用OIFCFG之前，请确保至少在本地节点上启动了Oracle Clusterware，如果要在命令语法中包含-global选项，则最好在所有节点上启动。

注意：

要更改全局网络接口，Oracle Clusterware必须在所有群集节点上运行。

以安装Oracle Clusterware软件的用户身份从Grid_home /bin/目录运行OIFCFG。例如：

$ ./oifcfg

运行oifcfg -help命令显示OIFCFG的在线帮助。

$ ./oifcfg -help

OIFCFG命令格式

oifcfg iflist [-p] [-n] oifcfg setif {-node nodename | -global} { if_name / subnet ： if_type [， if_type ]} [，...]oifcfg getif [-node nodename | -global] [-if if_name [/ subnet ] [-type if_type ]] oifcfg delif {{-node nodename | -global} [ if_name [/ subnet ]] [-force] | -force} oifcfg [-help]

OIFCFG命令

使用这些命令可以定义和管理网络接口。

您可以输入表D-1中列出的任何OIFCFG命令。

表D-1 OIFCFG命令

命令 描述

oifcfg iflist [-p [-n]]

显示可以使用setif配置的可用接口。iflist命令查询操作系统以查找此节点上存在的网络接口。您可以使用此命令指定两个选项：

  • -p ：显示接口类型的启发式假设（PRIVATE，PUBLIC或UNKNOWN）

  • -n ：显示网络掩码

如果接口具有IPv6网络，则OIFCFG会在单独的行上打印IPv6子网，类似于以下内容：

eth1 fec0 :: 80 PUBLIC 64 eth1 10.229.184.0 PUBLIC 255.255.248.0

oifcfg setif

为接口设置接口类型（公共，集群互连或Oracle ASM）。

oifcfg getif

使用setif命令显示已为其定义接口类型的接口，以及该接口的类型。

注意：如果未使用oifcfg setif -node命令设置接口详细信息，则可以对此命令使用的-node命令参数不返回结果。

oifcfg delif

删除全局或节点特定接口的存储网络配置。您可以使用-force选项而不指定-node或-global选项，以从群集中的所有节点中删除存储的网络配置。

OIFCFG命令参数

本节列出了OIFCFG命令的参数。请注意，某些参数是可选的，具体取决于您运行的命令。

  • -node node_name ： olsnodes命令输出中列出的Oracle Clusterware节点的名称。OLSNODES命令参考描述了olsnodes命令。

  • -global ：网络接口既可以存储为全局接口 （由iflist命令报告），也可以存储为特定于节点的接口 ：

    • 当Oracle Real Application Clusters（Oracle RAC）群集的所有节点具有连接到同一子网的相同接口时，接口将存储为全局接口。全局接口（以及为每个公有子网配置具有相同网络接口的所有节点以及每个私有子网的相同网络接口）不仅是推荐配置，而且也是默认安装配置。

    • 接口可以存储为特定于节点的（本地）接口。

      注意：

      Oracle支持跨节点的不同接口名称的接口名称通配符。Oracle不支持使用-node在不同节点上配置不同的特定接口名称。

  • -if if_name ：在系统中配置接口的名称。

  • subnet ：接口的子网地址。

  • -type if_type ：一个或多个以逗号分隔的接口类型： public ， cluster_interconnect或asm 。

  • -help ：显示OIFCFG命令的在线帮助。

OIFCFG使用说明

  • 网络接口规范采用以下形式：

    if_name / subnet ： if_type

    规范使用以下方法唯一标识网络接口：

    • 接口名称

      使用oifcfg setif命令时，可以指定包含通配符的接口名称，例如星号（*），它匹配任何字符串。但是，您必须使用双引号（ "" ）包含包含通配符的接口名称。

      注意：

      Oracle建议您不要在具有Oracle Database 11g第 2版（11.2.0.2）数据库的群集中使用通配符，因为Oracle通过扩展本地节点上的通配符来解析接口名称，并且每次都会发出PRIF-0029警告你运行oifcfg getif命令。

      如果您使用通配符与11个预-G 2版（11.2.0.2）Oracle数据库，那么这些数据库必须使用CLUSTER_INTERCONNECTS参数，来代替。

    • 关联子网

    • 接口类型

      接口类型指示网络配置的目的。支持的接口类型是：

      • public ：可用于与Oracle RAC实例外部组件（如Oracle Net和虚拟Internet协议（VIP）地址）通信的接口。

      • cluster_interconnect ：用于群集互连的专用接口，用于提供实例或缓存融合脚1通信。

      如果将接口类型设置为cluster_interconnect ，则它会在实例启动时影响实例，并且在重新启动实例之前更改不会生效。

    例如，以下规范将qfe0标识为位于地址204.152.65.0的群集互连：

    qfe0 / 204.152.65.0：cluster_interconnect

  • Oracle Universal Installer使用OIFCFG来识别和显示可用的接口。

  • 更改接口名称的效果取决于您要更改的名称，以及是否还要更改IP地址。如果仅更改接口名称，则分支很小。如果更改存储在Oracle Cluster Registry（OCR）中的公共接口的名称，则必须修改每个节点的nodeapps。因此，必须停止nodeapps才能使此更改生效。

  • 进行全局更改时，必须在群集的所有成员上重新启动Oracle Clusterware。对于本地更改，您只需执行节点重新启动。实例启动时会发生数据库的互连更改。但是，Oracle Clusterware的互连可能不同。

  • 由于在实例启动时选择了互连，因此仅发出OIFCFG命令不会对正在运行的系统产生直接影响。相反，更改将在重新启动可能受命令影响的组件后生效。

  • 使用OIFCFG对群集范围的网络分类所做的更改需要启动所有已知群集节点，以便可以更新其网格即插即用配置文件。

OIFCFG示例

以下示例显示了OIFCFG命令的一些常见用法：

列出网络接口的名称

您可以使用OIFCFG通过执行iflist关键字iflist本地节点上可用的所有接口的接口名称和子网，如下例所示：

oifcfg iflist eth0 172.19.141.0 eth1 172.21.65.0

检索网络信息

您还可以使用getif命令检索特定的OIFCFG信息。

例如，在安装Oracle Clusterware之后，可以通过输入以下命令来验证是否已将公共和群集互连设置为所需的值：

$ oifcfg getif

此命令返回全局public和global cluster_interconnect ，类似于以下内容：

eth0 172.19.141.0 global public eth1 172.21.65.0 global cluster_interconnect

存储全局接口

要存储接口，请使用setif关键字。例如，要将接口eth0与子网172.19.141.0一起存储为全局接口（用作集群中所有Oracle RAC实例和Oracle Clusterware的互连），请使用以下命令：

oifcfg setif -global eth0 / 172.19.141.0：cluster_interconnect

注意：

运行setif命令时，请确保所有节点都在运行，因为Oracle无法在未运行的节点上更新Grid Plug and Play配置文件。

删除存储的接口

使用oifcfg delif命令删除全局或特定于节点的接口的存储配置。可以通过在命令行上提供具有可选子网的接口名称来删除特定节点特定或全局接口。

注意：

如果只输入oifcfg delif ，而在命令行中没有给出其他参数，则OIFCFG将删除集群中所有节点上的所有接口。

例如，以下命令删除子网172.21.65.0的名为eth1的全局接口：

oifcfg delif -global eth1 / 172.21.65.0

以下命令将删除分配有OIFCFG的所有全局接口：

oifcfg delif -global
