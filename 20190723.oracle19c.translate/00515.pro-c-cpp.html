4.5 Pro * C / C ++

Oracle Pro * C / C ++预编译器允许程序员在C和C ++程序中使用用户定义的数据类型。

4.5.1关于Pro * C / C ++

Pro * C开发人员可以使用对象类型转换器将Oracle对象类型和集合映射到要在Pro * C应用程序中使用的C数据类型。

Pro * C提供对象类型和集合的编译时类型检查以及从数据库类型到C数据类型的自动类型转换。

Pro * C包含用于创建和销毁对象的EXEC SQL语法，并提供了两种访问服务器中对象的方法：

  • Pro * C程序中嵌入的SQL语句和PL / SQL函数或过程。

  • 一个接口对象缓存（在“ Oracle调用接口（OCI） ”下描述），其中可以通过遍历指针来访问对象，然后在服务器上进行修改和更新。

4.5.2 Pro * C / C ++中的关联访问

Pro * C / C ++为对象的关联访问提供了以下功能：

  • 支持在对象缓存中分配的对象的临时副本

  • 支持在嵌入式SQL INSERT ， UPDATE和DELETE语句中或SELECT语句的WHERE子句中作为输入主机变量引用的对象的临时副本

  • 支持嵌入式SQL SELECT和FETCH语句中作为输出主机变量引用的对象的临时副本

  • 支持通过DESCRIBE语句引用对象类型的ANSI动态SQL语句，以获取对象的类型和架构信息

4.5.3 Pro * C / C ++中的导航访问

Pro * C / C ++提供以下功能来支持更加面向对象的对象接口：

  • 支持使用嵌入式SQL OBJECT DEREF语句解除引用，固定和锁定对象缓存中的对象

  • 使用嵌入式SQL OBJECT UPDATE ， OBJECT DELETE和OBJECT RELEASE语句，允许Pro * C / C ++用户在更新或删除对象时或在不再需要对象时通知对象缓存

  • 支持使用嵌入式SQL OBJECT CREATE语句在对象缓存中创建新的可引用对象

  • 支持使用嵌入式SQL OBJECT FLUSH语句将对象缓存中的更改刷新到服务器

4.5.4 Oracle类型和C类型之间的转换

Pro * C / C ++提供了易用性增强功能，可简化C和C ++应用程序中OCI类型的使用。

Oracle类型转换程序（OTT）生成的对象的C表示使用不透明的OCI类型，使您无需更改其内部格式，但在C或C ++应用程序中使用起来很麻烦。这些陈述包括OCIString和OCINumber标量的属性和OCITable ， OCIArray和OCIRef收集类型和对象引用类型。

使用Pro * C / C ++增强功能：

  • 可以使用嵌入式SQL OBJECT GET语句检索对象属性并将其隐式转换为C类型。

  • 可以使用嵌入式SQL OBJECT SET语句从C类型设置和转换对象属性。

  • 可以使用嵌入式SQL COLLECTION GET语句将集合映射到主机数组。此外，如果集合由标量类型组成，则OCI类型可以隐式转换为兼容的C类型。

  • 主机数组可用于使用嵌入式SQL COLLECTION SET语句更新集合的元素。与COLLECTION GET语句一样，如果集合由标量类型组成，则C类型将隐式转换为OCI类型。

4.5.5 Oracle类型转换器（OTT）

Oracle Type Translator（OTT）是一个自动生成与对象类型相对应的C语言结构声明的程序。

OTT使得使用Pro * C预编译器和OCI服务器访问包更加容易。
