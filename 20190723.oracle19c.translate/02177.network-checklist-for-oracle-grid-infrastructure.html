Oracle Grid Infrastructure的网络清单

查看此Oracle Grid Infrastructure安装的网络核对表，以确保您具有群集所需的硬件，名称和地址。

在安装期间，您可以指定用作公共，私有或Oracle ASM接口的接口。您还可以指定用于其他目的的接口，例如网络文件系统，并且不可用于Oracle Grid Infrastructure。

如果使用第三方群集软件，则从该软件获取公共主机名信息。

表1-4 Oracle Grid Infrastructure和Oracle RAC的网络配置任务

校验 任务

公共网络硬件

  • 公共网络交换机（建议使用冗余交换机）连接到公共网关和每个集群成员节点的公共接口端口。

  • 以太网接口卡（建议使用冗余网卡，绑定为一个以太网端口名称）。

  • 交换机和网络接口必须至少为1 GbE。

  • 网络协议是传输控制协议（TCP）和因特网协议（IP）。

互连的专用网络硬件

  • 专用网络交换机（建议使用冗余交换机），连接到每个集群成员节点的专用接口端口。

    注意：如果每台服务器都有多个专用网络接口卡，则Oracle Clusterware会使用网格进程间通信（GIPC）和网格基础结构冗余互连（也称为群集高可用性IP（HAIP））自动将这些接口与专用网络相关联。 。

  • 交换机和网络接口适配器必须至少为1 GbE。

  • 互连必须支持用户数据报协议（UDP）。

  • 巨型帧（大于1500位的以太网帧）不是IEEE标准，但如果配置正确，可以减少UDP开销。Oracle建议将Jumbo Frame用于互连。但是，请注意必须对系统进行负载测试，并确保在整个堆栈中启用它们。

Oracle Flex ASM网络硬件

Oracle Flex ASM可以使用与Oracle Clusterware相同的专用网络，也可以使用自己的专用专用网络。每个网络可以分为PUBLIC或PRIVATE + ASM或PRIVATE或ASM。 Oracle ASM网络使用TCP协议。

群集名称和地址

确定并配置群集的以下名称和地址：

  • 群集名称 ：确定群集的名称 ，并准备在安装期间输入该名称。群集名称应具有以下特征：

    所有主机在全球范围内都是唯一的，甚至跨越不同的DNS域。

    至少一个字符长且小于或等于15个字符长。

    根据RFC 1123：连字符（ - ）和单字节字母数字字符（a到z，A到Z和0到9），由用于主机名的相同字符集组成。如果使用第三方供应商集群件，则Oracle建议您使用供应商集群名称。

  • 网格命名服务虚拟IP地址（GNS VIP ）：如果您计划使用GNS，则在DNS中为GNS VIP配置GNS名称和固定地址，并在您委托给GNS VIP的DNS上配置子域以解析群集地址。动态公共网络（DHCP，自动配置）必须使用GNS域委派。

  • 单客户端访问名称（SCAN）和地址

    使用网格命名服务解决方案 ：不要在DNS中配置SCAN名称和地址。 SCAN名称由GNS管理。

    使用手动配置和DNS解析 ：配置SCAN名称以解析域名服务（DNS）上的三个地址。

节点公共，私有和虚拟IP名称和地址

如果您未使用GNS，请为每个节点配置以下内容：

  • 公共节点名称和地址 ，在DNS和/etc/hosts （例如， node1.example.com ，地址192.0.2.10）。公共节点名称应该是每个节点的主要主机名，这是hostname命令显示的名称。

  • 专用节点地址 ，在每个节点的专用接口上配置。

    专用接口使用的私有子网必须连接您打算拥有的所有节点作为集群成员。Oracle建议您为专用网络选择的网络使用RFC 1918定义为私有的地址范围。

  • 公共节点虚拟IP名称和地址 （例如， node1-vip.example.com ，地址192.0.2.11）。

    如果未使用具有GNS和子域委派的动态网络，请确定每个节点的虚拟主机名。虚拟主机名是公共节点名，用于在节点关闭时重新路由发送到节点的客户端请求。Oracle数据库使用VIP进行客户端到数据库的连接，因此VIP地址必须可公开访问。Oracle建议您以hostname -vip格式提供名称。例如： myclstr2-vip 。
