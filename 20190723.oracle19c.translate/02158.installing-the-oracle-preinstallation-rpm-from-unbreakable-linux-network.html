从坚不可摧的Linux网络安装Oracle预安装RPM

使用此过程订阅Oracle Linux通道，并添加分发Oracle预安装RPM的Oracle Linux通道。

  1. 完成默认的Oracle Linux工作站安装。
    您可以从Oracle Software Delivery Cloud下载Oracle Linux：
    https://edelivery.oracle.com/linux
  2. 使用Unbreakable Linux Network（ULN）注册您的服务器。默认情况下，您已注册适用于您的操作系统和硬件的Oracle Linux Latest频道。
  3. 登录坚不可摧的Linux网络：
    https://linux.oracle.com
  4. 单击“ 系统”选项卡，然后在“系统配置文件”列表中，选择已注册的服务器。将打开“系统详细信息”窗口，并显示服务器的预订。
  5. 单击“ 管理订阅”以打开“系统摘要”窗口。
  6. 从“ 可用通道”列表中，选择与Oracle Linux发行版对应的最新和更新修补程序通道。例如，如果您的发行版是Oracle Linux 7 for x86_64，则选择Oracle Linux 7 Latest（x86_64） 。
  7. 单击订阅。
  8. 启动终端会话并以root身份输入以下命令，具体取决于您的平台。例如：
    • Oracle Linux 7：
      #yum install oracle-database-preinstall-19c

      注意：

      如果您希望yum跳过包确认提示，请使用-y选项。

    您应该看到输出，表明您已订阅Oracle Linux通道，并且正在安装这些软件包。

    Oracle Linux自动创建标准（非角色分配）Oracle安装所有者和组，并根据Oracle安装的要求设置其他内核配置设置。
  9. 以root身份输入以下命令以更新sysctl.conf设置：
    #sysctl -p

    注意：

    默认情况下，RPM软件包将Oracle软件用户设置为oracle 。在安装Oracle数据库之前，您可以在/etc/security/limits.d/oracle-database-preinstall-19c.conf文件和其他配置文件中更新Oracle用户名。
  10. 检查RPM日志文件以查看系统配置更改。例如，在Oracle Linux 7上：
    / var / log / oracle-database-preinstall-19c / backup / timestamp /orakernel.log

    在群集中的所有其他服务器上重复这些步骤。
