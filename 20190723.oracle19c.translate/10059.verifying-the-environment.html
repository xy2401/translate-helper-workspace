验证环境

在开始复制之前完成这些步骤。

  1. 确保可以从目标主机访问源数据库主机。您必须能够在两台主机之间进行SSH。

  2. 在目标主机上，使用TNSPING实用程序验证源主机上的侦听器端口是否正常工作。

    例如：
    tnsping source host ：1521
  3. 在目标主机上，使用Easy Connect验证与源数据库的连接：
    host ： port / servicename
    例如：
    sqlplus system @ ip-address ：1521 / proddb

    确保连接字符串不超过64个字符。

  4. 将所需的sqlpatch文件（用于回滚）从源数据库主目录复制到目标数据库。

  5. 确保在源数据库上至少创建了一个存档日志;否则，RMAN重复失败并显示错误。

  6. 如果源数据库使用钱包，则备份基于密码的钱包并将其复制到目标主机上的标准位置。

    例如：
    / opt / oracle / dcs / commonstore / wallets / db_unique_name / db_unique_name /

  7. 确保源数据库中的兼容性参数对于11.2.0.4数据库至少设置为11.2.0.4.0，对于12.1.0.2数据库至少设置为12.1.0.2.0。
