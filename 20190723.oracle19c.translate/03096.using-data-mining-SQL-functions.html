32.2使用数据挖掘SQL函数

了解SQL函数在数据挖掘中的优势。

数据挖掘SQL函数具有以下优点：

  • 可以在现有SQL应用程序的上下文中轻松部署模型。

  • 评分操作利用现有的查询执行功能。这提供了性能优势。

  • 评分结果是流水线的，可以处理行而无需实现。

数据挖掘功能为选择中的每一行生成一个分数。这些函数可以应用挖掘模型模式对象来计算分数，或者它们可以在没有预定义模型的情况下动态评分 ，如“动态评分”中所述 。

32.2.1选择预测因子

数据挖掘功能支持USING子句，该子句指定用于评分的属性。您可以在选择中指定部分或全部属性，并可以指定表达式。以下示例均使用PREDICTION函数查找可能使用关联卡的客户，但每个示例使用不同的预测变量集。

例32-1中的查询使用了所有预测变量。

例32-2中的查询仅使用性别，婚姻状况，职业和收入作为预测因子。

例32-3中的查询使用三个属性和一个表达式作为预测变量。该预测基于性别，婚姻状况，职业以及所有客户处于最高收入范围的假设。

例32-1使用所有预测器

SELECT cust_gender，COUNT（*）AS cnt，ROUND（AVG（age））AS avg_age FROM mining_data_apply_v WHERE PREDICTION（dt_sh_clas_sample USING *）= 1 GROUP BY cust_gender ORDER BY cust_gender; C CNT AVG_AGE  -  ---------- ---------- F 25 38 M 213 43

例32-2使用一些预测器

SELECT cust_gender，COUNT（*）AS cnt，ROUND（AVG（age））AS avg_age FROM mining_data_apply_v WHERE PREDICTION（dt_sh_clas_sample使用cust_gender，cust_marital_status，占用，cust_income_level）= 1 GROUP BY cust_gender ORDER BY cust_gender; C CNT AVG_AGE  -  ---------- ---------- F 30 38 M 186 43

例32-3使用一些预测变量和表达式

SELECT cust_gender，COUNT（*）AS cnt，ROUND（AVG（age））AS avg_age FROM mining_data_apply_v WHERE PREDICTION（dt_sh_clas_sample使用cust_gender，cust_marital_status，占用，'L：300,000及以上'AS cust_income_level）= 1 GROUP BY cust_gender ORDER BY cust_gender ; C CNT AVG_AGE  -  ---------- ---------- F 30 38 M 186 43

32.2.2单记录评分

数据挖掘功能可以为单个记录生成分数，如例32-4和例32-5所示 。

示例 32-4通过应用分类模型NB_SH_Clas_sample返回客户102001的预测。得分为0，表示该客户不太可能使用亲和卡。

示例32-5通过应用文本挖掘模型T_SVM_Clas_sample ，将“Affinity card is great”的预测作为comments属性T_SVM_Clas_sample 。得分为1，表示该客户可能使用亲和卡。

示例32-4评分单个客户或单个文本表达式

SELECT预测（NB_SH_Clas_Sample USING *）FROM sh.customers where cust_id = 102001;预测（NB_SH_CLAS_SAMPLEUSING *）----------------------------------- 0

示例32-5评分单个文本表达式

SELECT PREDICTION（T_SVM_Clas_sample使用'Affinity card很棒'AS评论）FROM DUAL;预测（T_SVM_CLAS_SAMPLEUSING'AFFINITYCARDISGREAT'ASCOMMENTS）------------------------------------------ ----------------------- 1
