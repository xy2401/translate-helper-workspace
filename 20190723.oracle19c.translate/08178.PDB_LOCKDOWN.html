1.257 PDB_LOCKDOWN

PDB_LOCKDOWN确定适用于PDB的PDB锁定配置文件。

属性 描述

参数类型

串

句法

PDB_LOCKDOWN = pdb-lockdown-profile-name

默认值

空值

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

Oracle RAC

所有实例都应具有相同的值

PDB锁定配置文件是一种限制可由连接到给定PDB的用户执行的操作（例如设置某些参数的值和使用某些选项）的机制。您还可以限制允许网络访问的任何程序包的执行，例如， UTL_SMTP 。

您可以使用SQL CREATE LOCKDOWN PROFILE语句创建锁定配置文件。然后，您可以使用SQL ALTER SESSION或ALTER SYSTEM语句设置配置文件。请参阅示例部分。

可以使用ALTER SYSTEM语句设置此参数，并将范围设置为MEMORY ， SPFILE或BOTH 。

PDB的锁定配置文件可由具有通用ALTER SYSTEM或通用SYSDBA权限的普通用户指定。

如果删除PDB锁定配置文件，则为其分配已删除配置文件的任何PDB（通过将删除的配置文件名称存储在PDB_LOCKDOWN参数中）将继续将其PDB_LOCKDOWN参数设置为已删除的锁定配置文件名称。但是，PDB不会受到锁定的锁定配置文件的任何限制。

现在可以在应用程序根目录中创建锁定配置文件，并将其称为应用程序锁定配置文件。

具有通用SYSDBA或通用ALTER SYSTEM权限的CDB通用用户只能将PDB_LOCKDOWN设置为CDB锁定配置文件。同样，具有应用程序通用SYSDBA或应用程序通用ALTER SYSTEM权限的应用程序通用用户只能将PDB_LOCKDOWN设置为应用程序锁定配置文件。

如果PDB_LOCKDOWN设置为应用程序根目录或应用程序PDB中的CDB锁定配置文件，则应用程序通用用户无法覆盖PDB_LOCKDOWN 。

如果PDB中的PDB_LOCKDOWN参数设置为与其祖先不同的锁定配置文件的名称（对于CDB，CDB根，或者对于应用程序PDB，应用程序根），以下将控制所施加的限制之间的交互通过这些资料：

  • 如果PDB（包括应用程序PDB）中的PDB_LOCKDOWN参数设置为CDB锁定配置文件，则会忽略CDB根目录中的PDB_LOCKDOWN参数（以及应用程序PDB，应用程序根目录）指定的锁定配置文件。

  • 如果应用程序PDB中的PDB_LOCKDOWN参数设置为应用程序锁定配置文件，而应用程序根目录或CDB根目录中的PDB_LOCKDOWN参数设置为CDB锁定配置文件，则除了应用程序锁定配置文件中规定的规则之外，还有DISABLE规则来自在其最近的祖先（即应用程序根或CDB根）中设置的CDB锁定配置文件是继承的。

  • 如果有包含CDB锁定简档和应用程序锁定轮廓规则之间的冲突，在CDB锁定轮廓的规则将优先（例如， OPTION_VALUE一个CDB锁定轮廓的子句将优先于OPTION_VALUE应用程序锁定的第轮廓）。

例子

此示例显示SYS用户如何连接到数据库AS SYSDBA并使用CDB根目录中的CREATE LOCKDOWN PROFILE语句来定义新的锁定配置文件。定义新的锁定配置文件后， SYS用户可以使用PDB_LOCKDOWN参数将新的锁定配置文件分配给PDB：

SQL> ALTER SESSION SET CONTAINER = CDB $ ROOT;会话改变了。SQL> CREATE LOCKDOWN PROFILE MYPROFILE;锁定配置文件已创建。SQL> ALTER SESSION SET CONTAINER = CDB1_PDB1;会话改变了。SQL> ALTER SYSTEM SET PDB_LOCKDOWN = MYPROFILE;系统改变了。SQL> SHOW PARAMETER PDB_LOCKDOWN NAME TYPE VALUE ------------------------------------ ------ ----- ------------------------------ pdb_lockdown string MYPROFILE SQL>
