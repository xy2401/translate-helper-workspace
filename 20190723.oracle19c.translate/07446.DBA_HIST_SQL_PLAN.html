5.60 DBA_HIST_SQL_PLAN

DBA_HIST_SQL_PLAN显示工作负载存储库中每个子游标的执行计划信息。

此视图从V$SQL_PLAN捕获信息，并与DBA_HIST_SQLSTAT视图一起使用。

柱 数据类型 空值 描述

DBID

数

不是NULL

数据库ID

SQL_ID

VARCHAR2（13）

不是NULL

库高速缓存中父游标的SQL标识符

PLAN_HASH_VALUE

数

不是NULL

光标的SQL计划的数字表示。将一个PLAN_HASH_VALUE与另一个进行比较可以轻松识别两个计划是否相同（而不是逐行比较两个计划）。

ID

数

不是NULL

分配给执行计划中每个步骤的编号

操作

VARCHAR2（30）

在此步骤中执行的内部操作的名称（例如， TABLE ACCESS ）

OPTIONS

VARCHAR2（30）

“ OPERATION列中描述的OPERATION变体（例如， FULL ）

OBJECT_NODE

VARCHAR2（128）

用于引用对象的数据库链接的名称（表名或视图名称）。对于使用并行执行的本地查询，此列描述了操作输出的使用顺序。

宾语＃

数

表或索引的对象编号

OBJECT_OWNER

VARCHAR2（128）

拥有包含表或索引的模式的用户的名称

OBJECT_NAME

VARCHAR2（128）

表或索引的名称

OBJECT_ALIAS

VARCHAR2（261）

对象的别名

OBJECT_TYPE

VARCHAR2（20）

对象的类型

优化器

VARCHAR2（20）

计划中第一行（语句行）的优化程序的当前模式，例如ALL_ROWS 。当操作是数据库访问（例如， TABLE ACCESS ）时，此列指示是否分析对象。

PARENT_ID

数

对当前步骤的输出进行操作的下一个执行步骤的ID

深度

数

树中操作的深度（或级别）。没有必要发出CONNECT BY语句来获取级别信息，该级别信息通常用于缩进PLAN_TABLE表中的行。根操作（语句）是级别0。

位置

数

具有相同PARENT_ID所有操作的处理PARENT_ID

SEARCH_COLUMNS

数

具有开始和停止键的索引列数（即具有匹配谓词的列数）

成本

数

优化程序基于成本的方法估算的操作成本。对于使用基于规则的方法的语句，此列为null。

CARDINALITY

数

通过基于成本的优化器估计操作产生的行数

BYTES

数

通过基于成本的优化器估计操作产生的字节数

OTHER_TAG

VARCHAR2（35）

描述OTHER列的内容。请参阅EXPLAIN PLAN了解值。

PARTITION_START

VARCHAR2（64）

启动一系列已访问分区的分区

PARTITION_STOP

VARCHAR2（64）

停止一系列访问的分区的分区

PARTITION_ID

数

计算PARTITION_START和PARTITION_STOP列的值对的步骤

其他

VARCHAR2（4000）

用户可能认为有用的执行步骤的其他信息。请参阅EXPLAIN PLAN了解值。

分配

VARCHAR2（20）

存储用于将生产者查询服务器中的行分发到使用者查询服务器的方法

CPU_COST

数

由优化程序基于成本的方法估计的操作的CPU成本。对于使用基于规则的方法的语句，此列为null。

IO_COST

数

由优化器基于成本的方法估计的操作的I / O成本。对于使用基于规则的方法的语句，此列为null。

TEMP_SPACE

数

由优化程序基于成本的方法估计的操作（排序或散列连接）的临时空间使用情况。对于使用基于规则的方法的语句，此列为null。

ACCESS_PREDICATES

VARCHAR2（4000）

用于在访问结构中定位行的谓词。例如，启动或停止索引范围扫描的谓词。

FILTER_PREDICATES

VARCHAR2（4000）

用于在生成行之前过滤行的谓词

投影

VARCHAR2（4000）

操作产生的表达

时间

数

由优化程序基于成本的方法估计的操作的经过时间（以秒为单位）。对于使用基于规则的方法的语句，此列为null。

QBLOCK_NAME

VARCHAR2（128）

查询块的名称

备注

VARCHAR2（4000）

备注

TIMESTAMP

日期

计划生成时间的时间戳

OTHER_XML

CLOB

提供特定于执行计划执行步骤的额外信息。此列的内容使用XML构建，因为它允许存储多条信息，包括以下内容：

  • 用于解析查询的模式的名称

  • 生成解释计划的Oracle数据库的版本号

  • 与执行计划关联的哈希值

  • 大纲的名称（如果有）或用于构建执行计划的SQL配置文件

  • 指示是否使用动态统计数据来制定计划

  • 大纲数据，一组可用于重新生成同一计划的优化程序提示

CON_DBID

数

采样会话的PDB的数据库ID

CON_ID

数

CON_DBID标识的容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

也可以看看：

  • “ V $ SQL_PLAN ”

  • “ DBA_HIST_SQLSTAT ”
