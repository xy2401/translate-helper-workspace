89 DBMS_JOB

DBMS_JOB包计划和管理作业队列中的作业。

注意：

DBMS_JOB包已被DBMS_SCHEDULER包取代，并且在将来的Oracle数据库DBMS_JOB可能会删除对DBMS_JOB支持。特别是，如果要管理作业以管理系统负载，建议您通过撤消用户的程序包执行权限来禁用DBMS_JOB 。

有关更多信息，请参阅Oracle数据库管理员指南中的DBMS_SCHEDULER和“从DBMS_JOB移动到DBMS_SCHEDULER ” 。

本章包含以下主题：

  • 安全模型

  • 操作说明

  • DBMS_JOB子程序摘要

89.1 DBMS_JOB安全模型

使用DBMS_JOB不需要特定的系统特权。没有系统权限可用于管理DBMS_JOB 。除了用户拥有的作业之外，不能更改或删除作业。对所有用户都是如此，包括那些被授予DBA权限的用户。

您可以执行用户拥有的程序或明确授予用户EXECUTE 。但是，无法执行通过角色授予用户执行权限的过程。

请注意，一旦作业启动并运行，就没有简单的方法可以停止作业。

89.2 DBMS_JOB操作说明

这些说明描述了停止工作以及使用Oracle Real Application Clusters。

停止工作

请注意，一旦作业启动并运行，就没有简单的方法可以停止作业。

使用Oracle Real Application Clusters

DBMS_JOB支持多实例作业执行。默认情况下，可以在任何实例上执行作业，但只有一个实例将执行作业。此外，您可以通过将作业绑定到特定实例来强制实例绑定。您可以通过为实例关联参数指定实例编号来实现实例绑定。但请注意，在Oracle数据库10g第1版（10.1）中，不建议使用实例绑定。服务亲和力是首选。此概念在DBMS_SCHEDULER包中实现。

以下过程可用于创建，更改或运行具有实例关联的作业。请注意，不指定关联性意味着任何实例都可以运行该作业。

  • DBMS_JOB.SUBMIT

  • DBMS_JOB.INSTANCE

  • DBMS_JOB.CHANGE

  • DBMS_JOB.RUN

DBMS_JOB.SUBMIT

要将作业提交到作业队列，请使用以下语法：

DBMS_JOB.SUBMIT（作业OUT BINARY_INTEGER，IN VARCHAR2，next_date IN DATE DEFAULT SYSDATE，interval IN VARCHAR2 DEFAULT'NULL'，no_parse IN BOOLEAN DEFAULT FALSE，实例IN BINARY_INTEGER DEFAULT ANY_INSTANCE，强制IN BOOLEAN DEFAULT FALSE）;

使用参数instance和 force控制作业和实例关联。instance的默认值为0（零），表示任何实例都可以执行作业。要在特定实例上运行作业，请指定instance值。如果instance值为负数或NULL. Oracle将显示错误ORA-23319 NULL.

force参数默认为false. 如果force为TRUE,任何正整数都可以作为作业实例。如果force为FALSE,则指定的实例必须正在运行，或者Oracle显示错误号ORA-23428.

DBMS_JOB.INSTANCE

要分配特定实例以执行作业，请使用以下语法：

DBMS_JOB.INSTANCE（JOB IN BINARY_INTEGER，实例IN BINARY_INTEGER，强制IN BOOLEAN DEFAULT FALSE）;

此示例中的FORCE参数默认为FALSE.如果实例值为0（零），则会更改作业关联，并且尽管有force值，任何可用实例都可以执行作业。如果INSTANCE值为正且FORCE参数为FALSE,则仅在指定实例正在运行时更改作业关联，或者Oracle显示错误ORA-23428.

如果force参数为TRUE ，则任何正整数都可以作为作业实例并更改作业关联。如果instance值为负或NULL Oracle将显示错误ORA-23319。

DBMS_JOB.CHANGE

要更改与作业关联的用户可定义参数，请使用以下语法：

DBMS_JOB.CHANGE（JOB IN BINARY_INTEGER，IN VARCHAR2 DEFAULT NULL，next_date IN DATE DEFAULT NULL，interval IN VARCHAR2 DEFAULT NULL，实例IN BINARY_INTEGER DEFAULT NULL，强制IN BOOLEAN DEFAULT FALSE）;

此示例中显示了两个参数instance和force, 。instance的默认值为null表示作业关联性不会更改。

force的默认值为FALSE.甲骨文显示错误ORA-23428如果指定的实例没有运行和错误ORA-23319 ，如果instance数是负的。

DBMS_JOB.RUN

DBMS_JOB.RUN的force参数默认为FALSE.如果force为TRUE,实例关联与在前台进程中运行作业无关。如果force为FALSE ，则作业只能在指定实例中的前台运行。如果force为FALSE并且连接的实例是不正确的实例，Oracle会显示错误ORA-23428 。 

DBMS_JOB.RUN（作业IN BINARY_INTEGER，强制进入BOOLEAN DEFAULT FALSE）;
