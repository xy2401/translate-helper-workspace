13.61共享条款

仅当在应用程序根目录中创建对象时，此子句才适用。此类对象称为应用程序公共对象，可以与属于应用程序根目录的应用程序PDB共享。SHARING子句只能出现在此上下文中的库，独立对象类型，包和独立子程序的声明中。

SHARING子句只能出现在使用以下SQL语句创建应用程序公共对象的上下文中：

  • 创建功能声明

  • CREATE LIBRARY语句

  • CREATE PROCEDURE声明

  • CREATE PACKAGE声明

  • CREATE TRIGGER声明

  • CREATE TYPE语句

语义

sharing_clause

指定如何使用以下共享属性之一共享对象：

  • METADATA - 元数据链接共享元数据，但其数据对每个容器都是唯一的。这种类型的对象称为元数据链接的应用程序公共对象 。

  • NONE - 不共享对象。

如果在应用程序操作期间省略此子句，则数据库使用DEFAULT_SHARING初始化参数的值来确定对象的共享属性。如果DEFAULT_SHARING初始化参数没有值，则默认值为METADATA 。

对SHARING条款的限制

共享子句可能仅在应用程序根目录中的应用程序安装，升级或修补期间出现。您必须发出ALTER PLUGGABLE DATABASE APPLICATION ...BEGIN语句启动操作和ALTER PLUGGABLE DATABASE APPLICATION ...END语句结束操作。在此上下文之外， sharing_clause是非法的，这意味着该对象不会被共享。

创建对象后，无法更改对象的共享属性。
