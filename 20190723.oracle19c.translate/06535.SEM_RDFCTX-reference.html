13 SEM_RDFCTX封装子程序

SEM_RDFCTX包中包含用于管理为文档创建的提取器策略和语义索引的子程序（函数和过程）。

要使用本章中的子程序，您应该了解文档语义索引中的概念和用法信息。

本章提供有关子程序的参考信息，按字母顺序列出。

13.1 SEM_RDFCTX.ADD_DEPENDENT_POLICY

格式

SEM_RDFCTX.ADD_DEPENDENT_POLICY（index_name IN VARCHAR2，policy_name IN VARCHAR2，partition_name IN VARCHAR2 DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

将依赖策略添加到（已创建的）索引或索引分区。

参数

index_name

索引的名称。

policy_name

依赖政策的名称。

partition_name

如果指定的索引是本地的，则为目标分区的名称。（否则，必须为null。）

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

与新依赖策略对应的基本策略必须已经是索引的一部分。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例将新的依赖策略SEM_EXTR_PLUS_GEOONT到索引ArticleIndex 。 

开始sem_rdfctx.add_dependent_policy（index_name =>'ArticleIndex'，policy_name =>'SEM_EXTR_PLUS_GEOONT'）;结束; /

13.2 SEM_RDFCTX.CREATE_POLICY

格式

SEM_RDFCTX.CREATE_POLICY（policy_name IN VARCHAR2，提取器IN mdsys.rdfctx_extractor，preferences IN sys。XMLType DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

要么

SEM_RDFCTX.CREATE_POLICY（policy_name IN VARCHAR2，base_policy IN VARCHAR2，user_models IN SEM_MODELS DEFAULT NULL，user_entailments IN SEM_MODELS DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

创建一个提取器策略。（第一种格式用于基本策略;第二种格式用于依赖于基本策略的策略。）

参数

policy_name

提取器策略的名称。

提取器

RDFCTX_EXTRACTOR类型的子类型的实例，它封装了信息提取器的提取逻辑。

首选项

与策略关联的任何首选项。

base_policy

依赖策略的基本提取器策略。

user_models

依赖策略的用户模型列表。

user_entailments

依赖策略的用户需求列表。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

使用此过程创建的提取器策略确定使用策略创建的语义索引的特征。每个提取器策略直接或间接引用提取器类型的实例。直接引用提取器类型实例的提取器策略可用于组成包含用于本体的其他RDF模型的其他提取器策略。

分配给extractor参数的提取器类型的实例必须是mdsys.rdfctx_extractor类型的直接或间接子类型的mdsys.rdfctx_extractor 。

必须可以由创建策略的用户访问user_models参数中指定的RDF模型。

必须可以由创建策略的用户访问user_entailments参数中指定的RDF蕴涵。请注意，蕴含下的RDF模型不会自动包含在依赖策略中。要包含一个或多个基础RDF模型，您需要在user_models参数中包含模型。

为提取器策略指定的首选项确定用于要编制索引的文档的存储库类型以及其他相关信息。有关更多信息，请参阅索引外部文档 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例使用gatenlp_extractor提取器类型创建提取器策略，该类型包含在Oracle数据库支持中，用于语义索引。

开始sem_rdfctx.create_policy（policy_name =>'SEM_EXTR'，extractor => mdsys.gatenlp_extractor（））;结束; /

以下示例为先前创建的提取器策略创建依赖策略，并将用户定义的RDF模型geo_ontology到依赖策略。 

开始sem_rdfctx.create_policy（policy_name =>'SEM_EXTR_PLUS_GEOONT'，base_policy =>'SEM_EXTR'，user_models => SEM_MODELS（'geo_ontology'））;结束; /

13.4 SEM_RDFCTX.MAINTAIN_TRIPLES

格式

SEM_RDFCTX.MAINTAIN_TRIPLES（index_name IN VARCHAR2，where_clause IN VARCHAR2，rdfxml_content IN sys。XMLType，policy_name IN VARCHAR2 DEFAULT NULL，action IN VARCHAR2 DEFAULT'ADD'，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

向包含从特定文档中提取的信息的图形添加一个或多个三元组。

参数

index_name

文档的语义索引的名称。

where_clause

存储文档的表上的SQL谓词（WHERE子句文本，不带WHERE关键字），用于标识要维护索引的行。

rdfxml_content

三元组，以RDF / XML文档的形式，添加到与文档对应的各个图形中。

policy_name

提取器策略的名称。如果policy_name为null（默认值），则将三元组添加到由索引的默认（或唯一）提取器策略提取的信息中;如果指定策略名称，则会将三元组添加到该策略提取的信息中。

行动

在三元组上执行的维护操作的类型。ADD当前支持的唯一值（默认值），它添加了rdfxml_content参数中指定的三元组。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

从语义索引文档中提取的信息可能是不完整的并且缺乏适当的上下文。此过程使域期望将三元组添加到与特定语义索引文档相关的各个图形，以便所有后续SEM_CONTAINS查询可以在其文档搜索条件中考虑这些三元组。

此过程接受索引名称和WHERE子句文本，以标识要使用其他三元组注释的特定文档。例如，where_clause可能被指定为涉及数字数据的简单谓词，例如'docId IN (1,2,3)' 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例通过将三元组添加到相应的单个图形来使用语义索引ArticleIndex来注释特定文档。 

开始sem_rdfctx.maintain_triples（index_name =>'ArticleIndex'，where_clause =>'docid = 15'，rdfxml_content => sys.xmltype（'<rdf：RDF xmlns：rdf =“http://www.w3.org/1999/ 02/22-rdf-syntax-ns＃“xmlns：rdfs =”http://www.w3.org/2000/01/rdf-schema#“xmlns：pred =”http://myorg.com/pred/ “> <rdf：说明rdf：about =”http://newscorp.com/Org/ExampleCorp“> <pred：hasShortName rdf：datatype =”http://www.w3.org/2001/XMLSchema#string“>示例</ pred：hasShortName> </ rdf：Description> </ rdf：RDF>'））;结束; /

13.5 SEM_RDFCTX.SET_DEFAULT_POLICY

格式

SEM_RDFCTX.SET_DEFAULT_POLICY（index_name IN VARCHAR2，policy_name IN VARCHAR2，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

为使用多个提取器策略配置的语义索引设置默认提取器策略。

参数

index_name

文档的语义索引的名称。

policy_name

提取器策略的名称，用作指定语义索引的默认提取器策略。必须是创建index_name的CREATE INDEX语句的PARAMETERS子句中列出的提取器策略之一。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

为文档创建语义索引时，可以在CREATE INDEX语句的PARAMETERS子句中将多个提取器策略指定为以空格分隔的名称列表。如“ 语义索引文档”中所述，此列表中的第一个策略用作所有未按名称标识提取器策略的SEM_CONTAINS查询的默认提取器策略。您可以使用SEM_RDFCTX.SET_DEFAULT_POLICY过程为索引设置不同的默认策略。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例将CITY_EXTR设置为ArticleIndex索引的默认提取器策略。 

开始sem_rdfctx.set_default_policy（index_name =>'ArticleIndex'，policy_name =>'CITY_EXTR'）;结束; /

13.6 SEM_RDFCTX.SET_EXTRACTOR_PARAM

格式

SEM_RDFCTX.SET_EXTRACTOR_PARAM（param_key IN VARCHAR2，patam_value IN VARCHAR2，param_desc IN VARCHAR2，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

配置Oracle数据库语义索引支持以使用外部信息提取器，例如Calais和GATE。

参数

param_key

要设置的参数的键。

param_value

要设置的参数的值。

param_desc

要设置的参数的简短描述。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

您必须具有SYSDBA角色才能使用此过程。

要使用Calais提取器类型（请参阅配置Calais Extractor类型 ），必须为以下参数指定值：

  • CALAIS_WS_ENDPOINT ：加来的Web服务端点。

  • CALAIS_KEY ：加来的许可证密钥。

  • CALAIS_WS_SOAPACTION ：Calais Web服务的SOAP操作。

要使用文本工程通用体系结构（GATE）提取器类型（请参阅使用文本工程的常规体系结构（GATE） ），必须为以下参数指定值：

  • GATE_NLP_HOST ：GATE NLP侦听器的主机。

  • GATE_NLP_PORT ：GATE NLP侦听器的端口。

除了这些参数之外，您可能还需要为HTTP_PROXY参数指定一个值，以使用防火墙外部的信息提取器或索引文档。

数据库实例仅具有这些参数的一组值，并且它们使用相应的信息提取器用于语义索引的所有实例。如果需要更改任何参数的现有值，可以使用此过程。

有关语义网络类型和选项的信息，请参阅语义网络 。
