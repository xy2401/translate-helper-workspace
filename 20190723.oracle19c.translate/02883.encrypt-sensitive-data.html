关于在数据字典中加密敏感凭证数据

数据字典SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL系统表存储敏感凭证数据，例如用户密码。

SYS.LINK$表存储有关数据库链接的信息。SYS.SCHEDULER$_CREDENTIAL存储有关Oracle Scheduler事件的信息。默认情况下，存储在这些表中的敏感凭据数据会被混淆。

您可以使用ALTER DATABASE DICTIONARY语句手动加密存储在SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL表中的数据。虽然此功能使用透明数据加密（TDE），但您无需具有高级安全选项许可证即可执行加密，但您必须具有SYSKM管理权限。TDE使用AES256（高级加密标准）算法执行加密。加密遵循与使用TDE加密的其他数据相同的行为。

作为最佳安全实践，Oracle建议您加密此敏感凭据数据。要检查数据字典凭据的状态，可以查询DICTIONARY_CREDENTIALS_ENCRYPT数据字典视图。

多租户选项如何影响敏感数据的加密

在多租户环境中，您可以从应用程序根目录以及单个可插拔数据库（PDB）中加密敏感数据字典信息。

在SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL系统表中加密，重新生成密钥或解密敏感凭据数据时，必须在完成此过程后同步受影响的PDB。这样做的说明包含在涵盖这些主题的过程中。

加密系统表中的敏感凭证数据

ALTER DATABASE DICTIONARY语句可以加密SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL系统表中的敏感凭据数据。

在使用ENCRYPT CREDENTIALS子句运行ALTER DATABASE DICTIONARY语句以加密SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL之前，数据库必须具有开放密钥库和加密密钥。凭证数据加密过程对模糊密码进行去模糊处理，然后对其进行加密。加密适用于用户在完成此过程后可能进行的任何密码更改。
  1. 以授予SYSKM管理权限的用户身份连接到数据库实例。
    例如：
    CONNECT hr_admin AS SYSKM输入密码： 密码
    在多租户环境中，连接到应用程序根目录或可插拔数据库（PDB）。
  2. 如有必要，请创建并打开密钥库，然后设置加密密钥。
    您可以查询V$ENCRYPTION_WALLET动态视图以查找密钥库的状态。
    使用ADMINISTER KEY MANAGEMENT语句执行这三个任务。例如：
    ADMINISTER密钥管理CREATE密钥库'的/ etc / ORACLE /钱包/ ORCL' IDENTIFIED BY 密码 ; ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN由“ password ”识别;管理员密钥管理集由“ 密码 ”WITH BACKUP识别的加密密钥;

    在多租户环境中，如果您位于应用程序根目录中，请包含CONTAINER = ALL子句。这适用于处于联合模式的PDB的密钥库操作。对于处于隔离模式的PDB，请从PDB中运行语句。

  3. 运行ALTER DATABASE DICTIONARY语句以加密数据。
    例如：
    ALTER DATABASE DATTIONARY ENCRYPT CREDENTIALS;

    在应用程序根目录中，要将加密应用于关联的PDB，请包含CONTAINER = ALL子句。 

    ALTER DATABASE DATTIONARY ENCRYPT CREDENTIALS CONTAINER = ALL;
  4. 如果从应用程序根目录执行加密，则同步关联的PDB。
    ALTER PLUGGABLE DATABASE APPLICATION APP $ CDB $ SYSTEM SYNC;

在SYS.LINK $系统表中更新敏感凭证数据

您可以使用ALTER DATABASE DICTIONARY语句重新SYS.LINK$数据字典SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL系统表中的敏感凭据数据。

要重新生成此敏感凭据数据，必须使用REKEY CREDENTIALS子句运行ALTER DATABASE DICTIONARY语句。使用列加密的重定密钥操作不会影响其他TDE主加密密钥。
  1. 以授予SYSKM管理权限的用户身份连接到数据库实例。
    例如：
    CONNECT hr_admin AS SYSKM输入密码： 密码
    在多租户环境中，连接到应用程序根目录或可插拔数据库（PDB）。
  2. 如有必要，请创建并打开密钥库，然后设置加密密钥。
    您可以查询V$ENCRYPTION_WALLET动态视图以查找密钥库的状态。
    使用ADMINISTER KEY MANAGEMENT语句执行这三个任务。例如：
    ADMINISTER密钥管理CREATE密钥库'的/ etc / ORACLE /钱包/ ORCL' IDENTIFIED BY 密码 ; ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN由“ password ”识别;管理员密钥管理集由“ 密码 ”WITH BACKUP识别的加密密钥;

    在多租户环境中，如果您位于应用程序根目录中，请包含CONTAINER = ALL子句。

  3. 运行ALTER DATABASE DICTIONARY语句以重新生成数据。
    例如：
    ALTER DATABASE DICTIONARY REKEY CREDENTIALS;

    在应用程序根目录中，要将加密应用于关联的PDB，请包含CONTAINER = ALL子句。 

    ALTER DATABASE DICTIONARY REKEY CREDENTIALS CONTAINER = ALL;
  4. 如果从应用程序根目录执行了重定密钥操作，则同步关联的PDB。
    ALTER PLUGGABLE DATABASE APPLICATION APP $ CDB $ SYSTEM SYNC;

删除系统表中的敏感凭据数据

ALTER DATABASE DICTIONARY语句可以使SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL现有凭据无效， SYS.LINK$这些表的未来凭证条目进行模糊处理。

要删除此凭据数据，必须使用DELETE CREDENTIALS子句运行ALTER DATABASE DICTIONARY语句。此语句主要用于必须从丢失透明数据加密（TDE）密钥库中恢复数据库链接的情况。
  1. 以授予SYSKM管理权限的用户身份连接到数据库实例。
    例如：
    CONNECT hr_admin AS SYSKM输入密码： 密码
    在多租户环境中，连接到应用程序根目录或可插拔数据库（PDB）。
  2. 如有必要，请创建并打开密钥库，然后设置加密密钥。
    您可以查询V$ENCRYPTION_WALLET动态视图以查找密钥库的状态。
    使用ADMINISTER KEY MANAGEMENT语句执行这三个任务。例如：
    ADMINISTER密钥管理CREATE密钥库'的/ etc / ORACLE /钱包/ ORCL' IDENTIFIED BY 密码 ; ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN由“ password ”识别;管理员密钥管理集由“ 密码 ”WITH BACKUP识别的加密密钥;

    在多租户环境中，如果您位于应用程序根目录中，请包含CONTAINER = ALL子句。

  3. 运行ALTER DATABASE DICTIONARY语句以删除密码凭据。
    例如：
    ALTER DATABASE DICTIONARY DELETE CREDENTIALS KEY;

    在应用程序根目录中，要删除关联PDB中的SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL密码凭据，请包含CONTAINER = ALL子句。 

    ALTER DATABASE DICTIONARY DELETE CREDENTIALS CONTAINER = ALL;
  4. 如果您从应用程序根目录执行了凭据删除，则同步关联的PDB。
    ALTER PLUGGABLE DATABASE APPLICATION APP $ CDB $ SYSTEM SYNC;

丢失密钥库后恢复数据库链接的功能

如果无意中丢失了TDE密钥库及其主加密密钥，则会对数据库链接产生不利影响。

当TDE密钥库和主加密密钥丢失时，使用加密密码进行身份验证的现有数据库链接将变得不可用。
  1. 以授予SYSKM管理权限且具有ALTER DATABASE LINK系统特权的用户身份连接到数据库实例。
    例如：
    CONNECT hr_admin AS SYSKM输入密码： 密码
    在多租户环境中，连接到应用程序根目录或可插拔数据库（PDB）。
  2. 从SYS.LINK$系统表中删除加密的凭据。
    ALTER DATABASE DICTIONARY DELETE CREDENTIALS KEY;

    如果要从应用程序根目录执行删除，请包含CONTAINER = ALL子句。 

    ALTER DATABASE DICTIONARY DELETE CREDENTIALS CONTAINER = ALL;
  3. 创建并打开密钥库，然后设置加密密钥。
    例如：
    ADMINISTER密钥管理CREATE密钥库'的/ etc / ORACLE /钱包/ ORCL' IDENTIFIED BY 密码 ; ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN由“ password ”识别;管理员密钥管理集由“ 密码 ”WITH BACKUP识别的加密密钥;

    在多租户环境中，如果您位于应用程序根目录中，请包含CONTAINER = ALL子句。

  4. 加密SYS.LINK$和SYS.SCHEDULER$_CREDENTIAL的密码凭据。
    ALTER DATABASE DATTIONARY ENCRYPT CREDENTIALS;

    如果从应用程序根目录执行加密，则包括CONTAINER = ALL子句。 

    ALTER DATABASE DATTIONARY ENCRYPT CREDENTIALS CONTAINER = ALL;
  5. 使用与数据库链接关联的用户的密码，重置受ALTER DATABASE DICTIONARY DELETE CREDENTIALS KEY语句影响的数据库链接密码。
    例如：
    ALTER DATABASE LINK database_link_name CONNECT TO user_id IDENTIFIED BY password ;

    要查找现有数据库链接及其所有者，请查询DBA_DB_LINKS数据字典视图。
