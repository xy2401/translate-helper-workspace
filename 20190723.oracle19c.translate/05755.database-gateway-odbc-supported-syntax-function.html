B支持的SQL语法和函数

以下主题描述了Oracle Database Gateway for ODBC支持的SQL语法和功能。

支持的SQL语句

Oracle Database Gateway for ODBC支持DELETE ， INSERT ， SELECT和UPDATE语句，但前提是ODBC驱动程序和非Oracle系统可以执行它们， 并且语句包含受支持的Oracle SQL函数。

除了少数例外，网关提供对Oracle DELETE ， INSERT ， SELECT和UPDATE语句的完全支持。

网关不支持Oracle 数据定义语言（DDL）语句。没有形式的Oracle ALTER ， CREATE ， DROP ， GRANT ，或可以使用TRUNCATE语句。相反，对于ALTER ， CREATE ， DROP和GRANT语句，如果需要对非Oracle系统数据库使用DDL语句，请使用网关的传递功能。

注意：

TRUNCATE不能用于传递语句。

删除

完全支持DELETE语句。但是，只能使用非Oracle系统支持的Oracle功能。

插入

完全支持INSERT语句。但是，只能使用非Oracle系统支持的Oracle功能。

选择

完全支持SELECT语句，但有以下例外：

  • CONNECT BY condition

  • NOWAIT

  • START WITH condition

  • WHERE CURRENT OF

  • 更新

UPDATE

完全支持UPDATE语句。但是，只能使用非Oracle系统支持的Oracle功能。此外，您不能在子查询中拥有引用外部查询中相同表名的SQL语句。SET子句不支持子查询。

Oracle函数

在网关将其转换为本机SQL之后，所有函数都由非Oracle系统进行评估。非Oracle系统仅支持一组有限的功能。大多数Oracle函数在此有限集中没有等效函数。因此，尽管后处理由Oracle数据库执行，但Oracle Database Gateway for ODBC不支持许多Oracle功能，可能会影响性能。

如果Oracle Database Gateway for ODBC不支持Oracle SQL函数，则DELETE ， INSERT或UPDATE语句不支持此函数。在SELECT语句中，这些函数由Oracle数据库评估，并在从非Oracle系统返回后进行处理。

如果在DELETE ， INSERT或UPDATE语句中使用了不受支持的函数，则会生成以下Oracle错误：

ORA-02070：数据库db_link_name在此上下文中不支持函数

Oracle Database Gateway for ODBC假定正在使用的ODBC驱动程序提供程序支持以下最小的SQL函数集：

  • AVG( exp )

  • LIKE( exp )

  • 计数（*）

  • MAX( exp )

  • MIN( exp )
