新功能

“新增功能”主题提供有关此版本中新增功能的信息。

以下功能是此版本中的新增功能：

  • Oracle LogMiner

    LogMiner包的CONTINUOUS_MINE功能已过时。它在Oracle Database 12 c第 2版（12.2）中已弃用。没有替换功能。

  • REST API

    在诸如Exalogic弹性云软件（EECS）和Oracle数据仓库云服务（DWCS）等托管云服务中，用户无权访问操作系统。他们只能访问数据库服务。加载数据并将客户从内部部署迁移到此类服务很困难。客户可以将其数据泵转储文件上载到对象库，并使用新的对象库REST API将它们加载到数据库中。 Impdp客户端CLI现在接受名为credential的新命令行参数。

    credential参数是凭证对象的名称，其中包含访问对象库所需的用户名和密码。当用户连接到数据库以运行impdp ，Datapump层将验证凭据是否存在以及用户是否有权读取凭据。如果有任何错误，则会将错误返回给impdp客户端。

  • Oracle Data Pump导出和导入

    • 只要文件设置为read-only ，DBA现在可以导入安装在两个不同数据库上的表空间文件。KEEP_READ_ONLY和NO_BITMAP_REBUILD被添加为TRANSPORTABLE参数值。

    • DBA现在可以更轻松地确定导出将花费多长时间，并发现闭包检查未报告的无法预料的问题。Data Pump Export命令行模式TTS_CLOSURE_CHECK参数用于指示作为Data Pump可传输表空间操作的一部分执行的闭包检查的程度。

    • 引入了一个新的转换参数OMIT_ENCRYPTION_CLAUSE ，该参数使Oracle Data Pump抑制与使用加密列的对象关联的任何加密子句。

    • 当有多个用户在给定数据库环境中执行数据泵作业时，更改的参数MAX_DATAPUMP_JOBS_PER_PDB可以为DBA提供对资源利用率的更多控制。默认值： 100 ，范围： 0到250或Auto - 50％的SESSIONS。

      引入了一个新参数MAX_DATAPUMP_PARALLEL_PER_JOB ，当有多个用户在给定数据库环境中执行数据泵作业时，它为数据库管理员提供了对资源利用率的更多控制。

  • Oracle外部表

    您现在可以使用ORACLE_LOADER访问驱动程序创建分区混合表。

不受支持的功能

此版本不支持以下功能。

此版本不支持以下功能：

  • LogMiner的CONTINUOUS_MINE功能

    Oracle Database 19c（19.1）中不支持dbms_logmnr.start_logmnr软件包的CONTINUOUS_MINE选项，该选项不再可用。

  • Oracle Streams

    从Oracle Database 19c开始，不支持Oracle Streams。Oracle GoldenGate是Oracle数据库的复制解决方案。

Oracle Database 18c中的更改

本主题列出了Oracle Database Utilities for Oracle Database 18c中的更改

Oracle Data Pump，导出和导入

  • 数据泵导入DATA_OPTIONS参数具有新的CONTINUE_LOAD_ON_FORMAT_ERROR选项。如果Data Pump在加载表数据时遇到流格式错误，Data Pump会尝试向前跳过以查找下一行的开头并继续从该位置加载该数据。

  • 出于安全考虑，Data Pump使用不同的密码处理固定用户数据库链接。有关其他信息，请参阅将透明数据加密与其他Oracle功能配合使用

  • 从Oracle Database 18c开始，您可以使用Oracle Data Pump在完整或部分导出和导入操作中包含统一审计跟踪。有关其他信息，请参阅数据泵组件。

  • 从Oracle Database 18c开始， XML_CLOB支持Export DATA_OPTIONS参数的XML_CLOB选项。

Oracle LogMiner

  • 不支持UTL_FILE_DIR初始化参数。它仍然支持向后兼容性，但Oracle建议您改为使用目录对象。此desupport会影响将LogMiner词典提取为平面文件。请参阅将LogMiner字典提取到平面文件 。

DBMS_DATAPUMP PL / SQL包

  • DBMS_DATAPUMP.SET_PARAMETER过程的DATA_OPTIONS参数有一个新选项KU$_DATAOPT_CONT_LOAD_ON_FMT_ERR 。如果遇到表数据中的流格式错误，此选项会告诉Data Pump继续从下一行加载数据。

外部表格

  • 外部表可用作SQL语句中的内联外部表，因此无需在数据字典中创建外部表作为持久数据库对象。
