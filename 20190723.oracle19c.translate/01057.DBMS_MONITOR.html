107 DBMS_MONITOR

DBMS_MONITOR包使您可以使用PL / SQL来控制其他跟踪和统计信息收集。

本章包含以下主题：

  • DBMS_MONITOR子程序摘要

107.1.1 CLIENT_ID_STAT_DISABLE过程

此过程将禁用所有实例的统计信息累积，并从CLIENT_ID_STAT_ENABLE过程启用的V$CLIENT_STATS视图中删除累积结果。

句法

DBMS_MONITOR.CLIENT_ID_STAT_DISABLE（client_id IN VARCHAR2）;

参数

表107-2 CLIENT_ID_STAT_DISABLE过程参数

参数 描述

CLIENT_ID

禁用统计信息聚合的客户端标识符

例子

要禁用累积：

EXECUTE DBMS_MONITOR.CLIENT_ID_STAT_DISABLE（'janedoe'）;

107.1.2 CLIENT_ID_STAT_ENABLE过程

此过程启用给定客户端标识符的统计信息收集。

统计信息收集对于数据库是全局的，并且跨实例启动和重新启动是持久的也就是说，为同一数据库的所有实例启用统计信息，包括重新启动。可以通过V$CLIENT_STATS视图查看统计信息。

句法

DBMS_MONITOR.CLIENT_ID_STAT_ENABLE（client_id IN VARCHAR2）;

参数

表107-3 CLIENT_ID_STAT_ENABLE过程参数

参数 描述

CLIENT_ID

启用统计信息聚合的客户端标识符

例子

要为具有给定客户端ID的客户端启用统计信息累积：

EXECUTE DBMS_MONITOR.CLIENT_ID_STAT_ENABLE（'janedoe'）;

表107-4 CLIENT_ID_TRACE_DISABLE过程参数

EXECUTE DBMS_MONITOR.CLIENT_ID_TRACE_DISABLE（'janedoe'）;

107.1.4 CLIENT_ID_TRACE_ENABLE过程

此过程将为数据库全局跟踪给定客户端标识符。

句法

DBMS_MONITOR.CLIENT_ID_TRACE_ENABLE（client_id IN VARCHAR2，等待IN BOOLEAN DEFAULT TRUE，绑定IN BOOLEAN DEFAULT FALSE，plan_stat IN VARCHAR2 DEFAULT NULL）;

参数

表107-5 CLIENT_ID_TRACE_ENABLE过程参数

参数 描述

CLIENT_ID

启用了SQL跟踪的数据库会话标识符

等待

如果为TRUE ，则跟踪中存在等待信息

结合

如果为TRUE ，则跟踪中存在绑定信息

plan_stat

我们转储行源统计信息的频率。值应该是' NEVER '，' FIRST_EXECUTION '（相当于NULL ）或' ALL_EXECUTIONS '。

使用说明

  • 跟踪将写入多个跟踪文件，因为多个Oracle影子进程可以代表给定的客户端标识符工作。

  • 对所有实例启用跟踪，并在重新启动时保持持久性。

例子

EXECUTE DBMS_MONITOR.CLIENT_ID_TRACE_ENABLE（'janedoe'，TRUE，FALSE）;

107.1.7 SERV_MOD_ACT_STAT_DISABLE过程

此过程将禁用统计信息累积并从V$SERV_MOD_ACT_STATS视图中删除累积结果。

统计信息禁用对于数据库是持久的。也就是说，对同一数据库的实例禁用服务统计信息（以及由于启用而已激活的dblinks ）。

句法

DBMS_MONITOR.SERV_MOD_ACT_STAT_DISABLE（service_name IN VARCHAR2，module_name IN VARCHAR2，action_name IN VARCHAR2 DEFAULT ALL_ACTIONS）;

参数

表107-8 SERV_MOD_ACT_STAT_DISABLE过程参数

参数 描述

服务名称

禁用统计聚合的服务的名称

MODULE_NAME

MODULE名称。服务的附加限定符。这是一个必需的参数。

ACTION_NAME

ACTION名称。 Service和MODULE名称的附加限定符。省略参数（或提供ALL_ACTIONS常量）意味着为给定的Service / MODULE组合的所有Actions启用聚合。在这种情况下，统计信息将在模块级别进行汇总。

使用说明

关于统计信息收集，当您更改模块或操作时，更改将在会话中执行下一个用户调用时生效。例如，如果在会话中将模块设置为“模块1”，并且在会话中的用户调用中将模块重置为“模块2”，则在该用户调用期间模块保持为“模块1”。在会话的下一个用户调用中，模块将更改为“模块2”。

107.1.8 SERV_MOD_ACT_STAT_ENABLE过程

此过程为服务名称， MODULE和ACTION的给定组合启用统计信息收集。

调用此过程可以在同一数据库的所有实例上为服务名称， MODULE名称和ACTION名称的分层组合启用统计信息收集。可以通过V$SERV_MOD_ACT_STATS视图访问统计信息。

句法

DBMS_MONITOR.SERV_MOD_ACT_STAT_ENABLE（service_name IN VARCHAR2，module_name IN VARCHAR2，action_name IN VARCHAR2 DEFAULT ALL_ACTIONS）;

参数

表107-9 SERV_MOD_ACT_STAT_ENABLE过程参数

参数 描述

服务名称

启用统计信息聚合的服务的名称

MODULE_NAME

MODULE名称。服务的附加限定符。这是一个必需的参数。

ACTION_NAME

ACTION名称。 Service和MODULE名称的附加限定符。省略参数（或提供ALL_ACTIONS常量）意味着为给定的Service / MODULE组合的所有Actions启用聚合。在这种情况下，统计信息将在模块级别进行汇总。

使用说明

由于Module / Action值可以是与NULL无法区分的空字符串，因此对给定的Service / Module / Action名称组合启用统计信息聚合会稍微复杂一些。因此，我们采用以下约定：

定义了一个特殊常量（不太可能是真正的动作名称）：

ALL_ACTIONS常量VARCHAR2：='### ALL_ACTIONS';

将ALL_ACTIONS用于操作规范意味着对具有给定模块名称的所有操作启用聚合，而使用NULL （或空字符串）意味着对名称为空字符串的操作启用聚合。

关于统计信息收集，当您更改模块或操作时，更改将在会话中执行下一个用户调用时生效。例如，如果在会话中将模块设置为“模块1”，并且在会话中的用户调用中将模块重置为“模块2”，则在该用户调用期间模块保持为“模块1”。在会话的下一个用户调用中，模块将更改为“模块2”。

例子

要为给定的服务名称和MODULE组合启用统计信息累积：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_STAT_ENABLE（'APPS1'，'PAYROLL'）;

要为给定的服务名称， MODULE和ACTION组合启用统计信息累积：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_STAT_ENABLE（'APPS1'，'GLEDGER'，'DEBIT_ENTRY'）;

如果发出上述两个命令，则统计信息累积如下：

  • 对于APPS1服务，因为每个服务名称的累积是默认值。

  • 对于PAYROLL模块中的所有操作。

  • 对于GLEDGER模块中的DEBIT_ENTRY操作。

107.1.9 SERV_MOD_ACT_TRACE_DISABLE过程

对于给定的Service Name， MODULE和ACTION名称组合，此过程将在ALL启用的实例上禁用跟踪。

句法

DBMS_MONITOR.SERV_MOD_ACT_TRACE_DISABLE（service_name IN VARCHAR2，module_name IN VARCHAR2，action_name IN VARCHAR2 DEFAULT ALL_ACTIONS，instance_name IN VARCHAR2 DEFAULT NULL）;

参数

表107-10 SERV_MOD_ACT_TRACE_DISABLE过程参数

参数 描述

服务名称

禁用跟踪的服务的名称。

MODULE_NAME

MODULE名称。服务的附加限定符

ACTION_NAME

ACTION名称。 Service和MODULE名称的附加限定符。

INSTANCE_NAME

如果设置，则会将跟踪限制为指定的instance_name

使用说明

为module_name参数指定NULL意味着将不再为未设置MODULE属性的会话累积统计信息。

例子

要为名为APPS1的服务启用跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_ENABLE（'APPS1'，DBMS_MONITOR.ALL_MODULES，DBMS_MONITOR.ALL_ACTIONS，TRUE，FALSE，NULL）;

要禁用上一步中指定的跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_DISABLE（'APPS1'）;

要为给定的Service和MODULE组合（所有ACTION ）启用跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_ENABLE（'APPS1'，'PAYROLL'，DBMS_MONITOR.ALL_ACTIONS，TRUE，FALSE，NULL）;

要禁用上一步中指定的跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_DISABLE（'APPS1'，'PAYROLL'）;

107.1.10 SERV_MOD_ACT_TRACE_ENABLE过程

除非指定了instance_name ，否则此过程将对全局服务名称， MODULE和ACTION的给定组合启用SQL跟踪。

句法

DBMS_MONITOR.SERV_MOD_ACT_TRACE_ENABLE（service_name IN VARCHAR2，module_name IN VARCHAR2 DEFAULT ANY_MODULE，action_name IN VARCHAR2 DEFAULT ANY_ACTION，等待IN BOOLEAN DEFAULT TRUE，绑定IN BOOLEAN DEFAULT FALSE，instance_name IN VARCHAR2 DEFAULT NULL，plan_stat IN VARCHAR2 DEFAULT NULL）;

参数

表107-11 SERV_MOD_ACT_TRACE_ENABLE过程参数

参数 描述

服务名称

为其启用SQL跟踪的服务的名称

MODULE_NAME

为其启用SQL跟踪的MODULE名称。服务的可选附加限定符。如果省略，则启用SQL跟踪或给定服务中的所有模块和操作。

ACTION_NAME

为其启用SQL跟踪的ACTION名称。Service和MODULE名称的可选附加限定符。如果省略，则为给定模块中的所有操作启用SQL跟踪。

等待

如果为TRUE ，则跟踪中存在等待信息

结合

如果为TRUE ，则跟踪中存在绑定信息

INSTANCE_NAME

如果设置，则会将跟踪限制为指定的instance_name

plan_stat

我们转储行源统计信息的频率。值应该是' NEVER '，' FIRST_EXECUTION '（相当于NULL ）或' ALL_EXECUTIONS '。

使用说明

  • 该过程为服务， MODULE和ACTION名称的给定组合启用跟踪。规范是严格分层的：必须指定服务名称或服务名称/ MODULE ，或服务名称，MODULE和ACTION名称。省略限定符的行为类似于外卡，因此不指定ACTION意味着所有ACTION 。使用ALL_ACTIONS常量可实现相同的目的。

  • 当应用程序MODULE和可选的已知ACTION遇到较差的服务级别时，此跟踪非常有用。

  • 默认情况下，为数据库全局启用跟踪。提供instance_name参数是为了限制对已知的命名实例的跟踪，例如，显示较差的服务级别。

  • 跟踪信息存在于多个跟踪文件中，您必须使用trcsess工具将其收集到一个文件中。

  • 为module_name参数指定NULL意味着将为未设置MODULE属性的会话累积统计信息。

例子

要为名为APPS1的服务启用跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_ENABLE（'APPS1'，DBMS_MONITOR.ALL_MODULES，DBMS_MONITOR.ALL_ACTIONS，TRUE，FALSE，NULL）;

要为给定的Service和MODULE组合（所有ACTION ）启用跟踪：

EXECUTE DBMS_MONITOR.SERV_MOD_ACT_TRACE_ENABLE（'APPS1'，'PAYROLL'，DBMS_MONITOR.ALL_ACTIONS，TRUE，FALSE，NULL）;

107.1.11 SESSION_TRACE_DISABLE过程

此过程将禁用本地实例上给定数据库会话的跟踪。

句法

DBMS_MONITOR.SESSION_TRACE_DISABLE（session_id IN BINARY_INTEGER DEFAULT NULL，serial_num IN BINARY_INTEGER DEFAULT NULL）;

参数

表107-12 SESSION_TRACE_DISABLE过程参数

参数 描述

SESSION_ID

禁用SQL跟踪的数据库会话标识符

SERIAL_NUM

此会话的序列号

使用说明

如果serial_num是NULL但session_id指定，与给定会话session_id不再追查不论其序列号。如果session_id和serial_num都为NULL ，则不再跟踪当前用户会话。指定NULL session_id和非NULL serial_num是非法的。此外， NULL值是默认值，可以省略。

例子

要为具有给定客户端会话ID的客户端启用跟踪：

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（7,4634，TRUE，FALSE）;

要禁用上一步中指定的跟踪：

EXECUTE DBMS_MONITOR.SESSION_TRACE_DISABLE（7,4634）;;

107.1.12 SESSION_TRACE_ENABLE过程

此过程为本地实例上的给定会话ID启用SQL跟踪

句法

DBMS_MONITOR.SESSION_TRACE_ENABLE（session_id IN BINARY_INTEGER DEFAULT NULL，serial_num IN BINARY_INTEGER DEFAULT NULL，等待IN BOOLEAN DEFAULT TRUE，绑定IN BOOLEAN DEFAULT FALSE，plan_stat IN VARCHAR2 DEFAULT NULL）;

参数

表107-13 SESSION_TRACE_ENABLE过程参数

参数 描述

SESSION_ID

启用了SQL跟踪的客户端标识符。如果省略（或NULL ），则假定用户自己的会话。

SERIAL_NUM

此会话的序列号。如果省略（或NULL ），则仅使用会话ID来确定会话。

等待

如果为TRUE ，则跟踪中存在等待信息

结合

如果为TRUE ，则跟踪中存在绑定信息

plan_stat

我们转储行源统计信息的频率。值应该是' NEVER '，' FIRST_EXECUTION '（相当于NULL ）或' ALL_EXECUTIONS '。

使用说明

该过程为给定的数据库会话启用跟踪，并且对客户端/服务器应用程序仍然有用。仅在调用者连接的实例上启用跟踪，因为数据库会话不跨越实例。此跟踪严格局限于实例。

如果serial_num是NULL但session_id指定，与给定会话session_id是不论其序列号跟踪。如果session_id和serial_num都为NULL ，则跟踪当前用户会话。指定NULL session_id和非NULL serial_num是非法的。此外， NULL值是默认值，可以省略。

例子

要为具有给定客户端会话ID的客户端启用跟踪：

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（7,4634，TRUE，FALSE）;

要禁用上一步中指定的跟踪：

EXECUTE DBMS_MONITOR.SESSION_TRACE_DISABLE（7,4634）;

或

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（5）;

要么

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（5，NULL）;

跟踪会话ID为5的会话

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（）;

要么

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（NULL，NULL）;

跟踪当前用户会话。也，

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（NULL，NULL，TRUE，TRUE）;

跟踪当前用户会话，包括等待和绑定。使用关键字语法也可以表达相同的内容：

EXECUTE DBMS_MONITOR.SESSION_TRACE_ENABLE（binds => TRUE）;
