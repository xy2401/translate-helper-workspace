校验 任务
查看Oracle Inventory（oraInventory）和OINSTALL组要求

Oracle Inventory目录是系统上安装的Oracle软件的中央库存。它应该是所有Oracle软件安装所有者的主要组。将Oracle Inventory组作为其主要组的用户将被授予OINSTALL权限，以读取和写入中央库存。

  • 如果您有现有安装，则OUI会检测/etc/oraInst.loc文件中的现有oraInventory目录，并使用此位置。

  • 如果您是第一次安装Oracle软件，则OUI会创建Oracle基础和中央库存，并使用以下优先级的信息创建Oracle库存：

    • 在为安装所有者用户帐户设置的ORACLE_BASE环境变量中指示的路径中。

    • 在最佳灵活体系结构（OFA）路径（u [01-99] / app / owner ，其中owner是运行安装的用户帐户的名称），如果该用户帐户具有写入该路径的权限。

    • 在用户主目录中的路径/ app / owner ， owner是运行安装的用户帐户的名称。

确保指定为OINSTALL组的组可用作所有计划的Oracle软件安装所有者的主要组。
为标准或角色分配的系统特权创建操作系统组和用户

根据您的安全要求创建操作系统组和用户，如本安装指南中所述。

为Oracle软件安装所有者设置资源限制设置和其他要求。

组和用户名必须仅使用ASCII字符。

注意：

不要删除现有的守护程序用户。如果已删除守护程序用户，则必须将其添加回来。
取消设置Oracle软件环境变量

如果您有现有的Oracle软件安装，并且您使用相同的用户来安装此安装，则取消设置以下环境变量： $ORACLE_HOME ; $ORA_NLS10 ; $TNS_ADMIN 。

如果已将$ORA_CRS_HOME设置为环境变量，则在开始安装或升级之前取消设置它。除非Oracle支持人员指示，否则不要将$ORA_CRS_HOME用作用户环境变量。

配置Oracle软件所有者环境

通过执行以下任务来配置oracle或grid用户的环境：

  • 在shell启动文件中将默认文件模式创建掩码（umask）设置为022。

  • 设置DISPLAY环境变量。
确定安装的root权限委派选项

在安装过程中，系统会要求您以root用户身份运行配置脚本。您可以在出现提示时以root身份手动运行这些脚本，也可以在安装期间使用root权限委派选项提供配置信息和密码。

要自动运行根脚本，请选择“ 自动运行配置脚本” 。在安装过程中要使用自动配置选项，所有集群成员节点的root用户凭据必须使用相同的密码。

  • 使用root用户凭据

    提供集群成员节点服务器的超级用户密码。

  • 使用sudo

    sudo是一个UNIX和Linux实用程序，它允许sudoers list特权的成员以sudoers身份运行各个命令。提供作为sudoers成员的操作系统用户的用户名和密码，并且有权在每个集群成员节点上运行sudo 。

    要启用sudo ，请让具有相应权限的系统管理员配置作为sudoers列表成员的用户，并在安装期间提示时提供用户名和密码。
