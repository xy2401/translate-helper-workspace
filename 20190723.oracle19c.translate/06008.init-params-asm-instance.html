关于Oracle ASM实例的初始化参数文件

在Oracle Restart（独立）配置中安装Oracle ASM时，Oracle Universal Installer（OUI）会为Oracle ASM实例创建单独的服务器参数文件（SPFILE）和密码文件。安装期间，ASM SPFILE存储在磁盘组中。

在集群Oracle ASM环境中安装Oracle ASM时，OUI会在磁盘组中为Oracle ASM创建单个共享SPFILE。

升级Oracle ASM实例时，如果ASM SPFILE最初位于共享文件系统中，则升级后的Oracle ASM实例会将SPFILE保留在同一位置。如果原始Oracle ASM实例使用PFILE，则在升级后，实例将继续使用PFILE。

您可以使用SPFILE或基于文本的初始化参数文件（PFILE）作为Oracle ASM实例参数文件。如果在集群Oracle ASM环境中使用SPFILE，则必须将SPFILE放在磁盘组或集群文件系统中。Oracle建议将Oracle ASM SPFILE放在磁盘组中。您不能使用在现有Oracle ASM SPFILE上创建的新别名来启动Oracle ASM实例

如果不使用共享的Oracle Grid Infrastructure主目录，则Oracle ASM实例可以使用PFILE。适用于数据库初始化参数文件的文件名，默认位置和搜索顺序的相同规则也适用于Oracle ASM初始化参数文件。

当Oracle ASM实例搜索初始化参数文件时，搜索顺序为：

  1. 网格即插即用（GPnP）配置文件中指定的初始化参数文件的位置

  2. 如果尚未在GPnP配置文件中设置位置，则搜索顺序将更改为：

    1. Oracle ASM实例主页中的SPFILE

      例如，SPFILE for Oracle ASM在Linux环境中的Oracle Grid Infrastructure主目录中具有以下默认路径：

      $ ORACLE_HOME / DBS / spfile的+ ASM.ora

    2. Oracle ASM实例主页中的PFILE

注意：

如果您的配置使用Oracle ASM实例的非默认初始化参数，则需要PFILE或SPFILE。

您可以使用SQL * Plus，ASMCA和ASMCMD命令管理Oracle ASM初始化参数文件。

创建，复制SPFILE并将其移动到磁盘组中

如果磁盘组的COMPATIBLE.ASM磁盘组属性设置为11.2或更高，则可以创建，复制或将Oracle ASM SPFILE移动到磁盘组中。

例如，在将实例从Oracle ASM 11g第1版（11.1）升级到Oracle ASM 11g第2版（11.2）之后，可以将Oracle ASM SPFILE放在COMPATIBLE.ASM设置为11.2的磁盘组中。有关磁盘组兼容性属性的信息，请参阅磁盘组兼容性 。

在以下步骤中，假设Oracle ASM 11g第2版（11.2）实例正在使用存储在$ORACLE_HOME/dbs/asmpfile.ora 。您可以使用SQL CREATE SPFILE语句从存储在本地或共享文件系统中的PFILE创建SPFILE。如果PFILE不存在，则可以使用SQL CREATE PFILE语句创建它。

要在磁盘组中创建SPFILE，请执行以下步骤：

  1. 连接到Oracle ASM实例。

    例如：

    $ sqlplus / as sysasm
  2. 使用SQL CREATE SPFILE语句在COMPATIBLE.ASM设置为11.2的磁盘组中CREATE SPFILE 。

    例如，从现有PFILE创建Oracle ASM SPFILE。

    SQL> CREATE SPFILE ='+ DATA / asmspfile.ora'FROM PFILE ='$ ORACLE_HOME / dbs / asmpfile.ora';

    CREATE SPFILE语句还更新网格即插即用（GPnP）配置文件。您可以使用ASMCMD spget命令在GPnP配置文件中检查Oracle ASM SPFILE的位置。见spget 。

    注意：

    如果包含AS COPY选项，则CREATE SPFILE FROM PFILE语句不会更新群集范围的SPFILE位置。

  3. 重新启动Oracle ASM实例，以便实例读取新位置中的SPFILE。

    有关关闭和启动Oracle ASM实例的信息，请参阅关于启动Oracle ASM实例和关于关闭Oracle ASM实例 。

在磁盘组中备份Oracle ASM SPFILE的副本

本节介绍使用ASMCMD命令在另一个磁盘组中备份Oracle ASM SPFILE的步骤。如有必要，可以使用备份副本还原Oracle ASM SPFILE。

源磁盘组和目标磁盘组必须将磁盘组属性COMPATIBLE.ASM设置为11.2或更高。

要使用spcopy命令在另一个磁盘组中复制Oracle ASM SPFILE，请执行以下步骤：

  1. 使用ASMCMD spget命令找到Oracle ASM SPFILE。

    例如：

    ASMCMD [+]> spget + DATA / ASM / ASMPARAMETERFILE / registry.253.849343867
  2. 使用spcopy命令将Oracle ASM SPFILE复制到另一个磁盘组。

    例如：

    ASMCMD [+]> spcopy + DATA / ASM / ASMPARAMETERFILE / registry.253.849343867 + FRA / spfileCopyASM.ora

    不使用-u选项运行spcopy不会更新Oracle ASM SPFILE的位置。您可以使用spset在Grid即插即用（GPnP）配置文件中设置Oracle ASM SPFILE的位置。

  3. 使用ASMCMD ls命令ls FRA磁盘组中包含的Oracle ASM SPFILE文件的所有副本。

    例如：

    ASMCMD [+]> ls -l --absolutepath FRA / ASM / ASMPARAMETERFILE类型Redund Striped Time Sys名称ASMPARAMETERFILE MIRROR COARSE JUN 06 13:00:00 Y + FRA / spfileCopyASM.ora => REGISTRY.253.849533009
  4. 使用spget命令验证Oracle ASM SPFILE文件的当前位置。

    例如：

    ASMCMD [+]> spget + DATA / ASM / ASMPARAMETERFILE / registry.253.849343867

如果磁盘组中的当前Oracle ASM SPFILE文件已损坏或该磁盘组不可访问，则可以使用带-u选项的spset或spcopy使用您拥有的备份副本还原Oracle ASM SPFILE文件以前创建的。

例如：

ASMCMD [+]> spcopy -u + FRA / spfileCopyASM.ora + DATA2 / ASM / spfileASM.ora

关于Oracle ASM初始化参数设置

您必须为Oracle ASM实例设置几个初始化参数。您可以使用Oracle ASM Configuration Assistant（ASMCA）设置这些参数。您还可以在使用SQL ALTER SYSTEM或ALTER SESSION语句创建数据库后设置一些参数。

Oracle ASM参数对大多数环境使用合适的默认值。您不能在数据库实例参数文件中使用带有ASM_*前缀的名称的参数。

关于Oracle ASM的自动内存管理

自动内存管理使用MEMORY_TARGET参数自动管理Oracle ASM和数据库实例的内存相关参数。默认情况下，在Oracle ASM实例上启用自动内存管理，即使未明确设置MEMORY_TARGET参数也是如此。对于大多数环境， MEMORY_TARGET使用的默认值是可接受的。这是您必须为完整的Oracle ASM内存管理设置的唯一参数。Oracle强烈建议您对Oracle ASM使用自动内存管理。

一个Oracle ASM实例可以自动增加设定的值MEMORY_TARGET和MEMORY_MAX_TARGET如果引发ORA-04031错误和自动内存管理功能。如果MEMORY_MAX_TARGET已明确设置的值，那么每ORA-04031升高时， MEMORY_TARGET值增加由现有的10％ MEMORY_TARGET值或128 MB，以较高者为准，但不得低于客户指定更大MEMORY_MAX_TARGET值。如果MEMORY_MAX_TARGET没有明确设置，则两个MEMORY_TARGET和MEMORY_MAX_TARGET增加由现有的10％ MEMORY_TARGET值或128 MB，取大值，为最多五个增加。Oracle ASM实例必须重新启动才能使用新的MEMORY_TARGET和MEMORY_MAX_TARGET设置。

如果没有为MEMORY_TARGET设置值，但是您为其他与内存相关的参数设置了值，则Oracle MEMORY_TARGET根据这些内存参数值在内部计算MEMORY_TARGET的最佳值。您也可以增加MEMORY_TARGET动态，达价值MEMORY_MAX_TARGET参数，就像你可以为数据库实例做。

虽然不建议这样做，但可以通过在Oracle ASM参数文件MEMORY_TARGET的值设置为0或运行ALTER SYSTEM SET MEMORY_TARGET =0语句来禁用自动内存管理。禁用自动内存管理时，Oracle将恢复自动共享内存管理和自动PGA内存管理。要恢复到Oracle数据库10g第2版（10.2）功能以手动管理Oracle ASM SGA内存，还要运行ALTER SYSTEM SET SGA_TARGET=0语句。除非指定，否则Oracle ASM实例中自动内存管理参数的行为与Oracle数据库实例中的行为相同。

注意：

  • 对于Linux环境，如果/dev/shm不可用或尺寸不足，则自动内存管理无法工作。有关更多信息，请参阅适用于Linux和基于UNIX的操作系统的Oracle数据库管理员参考 。有关支持自动内存管理的平台的信息，请参见“ Oracle数据库管理员指南” 。

  • Oracle ASM的最小MEMORY_TARGET为1 GB。如果将MEMORY_TARGET设置MEMORY_TARGET较低，则Oracle会自动将MEMORY_TARGET的值MEMORY_TARGET 1 GB。

  • 有关Oracle Exadata环境中内存初始化参数的建议设置，请参阅Oracle Exadata文档。

ASM_DISKGROUPS

ASM_DISKGROUPS初始化参数指定在发出SQL ALTER DISKGROUP ALL MOUNT语句时，Oracle ASM实例在启动时安装的磁盘组名称列表。

除非指定了NOMOUNT启动选项，否则Oracle ASM实例启动过程将执行ALTER DISKGROUP ALL MOUNT 。

ASM_DISKGROUPS参数的默认值是NULL字符串。

ASM_DISKGROUPS参数是动态的。如果使用服务器参数文件（SPFILE），则在大多数情况下不必手动更改ASM_DISKGROUPS的值。成功创建或装入磁盘组后，Oracle ASM会自动将磁盘组添加到参数中。删除磁盘组时，Oracle ASM还会自动从参数中删除磁盘组。但是，SPFILE不会在手动卸载时更新。

注意：

在Oracle Flex ASM配置中，卸载时不会从SPFILE中删除磁盘组。必须始终自动挂载磁盘组，以便远程实例可以访问它们。远程实例是位于与Oracle ASM实例不同的节点上的Oracle数据库，Oracle IOServer或Oracle ASM代理实例。

以下是动态设置ASM_DISKGROUPS参数的示例：

SQL> ALTER SYSTEM SET ASM_DISKGROUPS = DATA，FRA;

使用文本初始化参数文件（PFILE）时，您可以编辑初始化参数文件以添加任何磁盘组的名称，以便在实例启动时自动装入。您必须删除不再需要自动挂载的任何磁盘组的名称。

以下是初始化文件中ASM_DISKGROUPS参数的示例：

ASM_DISKGROUPS = DATA, FRA

注意：

发行ALTER DISKGROUP ......ALL MOUNT或ALTER DISKGROUP ...ALL DISMOUNT命令不会影响ASM_DISKGROUPS的值。

对于Oracle Database 12 c第1版或更高版本，Oracle ASM配置最多支持511个磁盘组。在12 c版本1之前的Oracle数据库版本的Oracle ASM配置最多只能支持63个磁盘组。

ASM_DISKSTRING

ASM_DISKSTRING初始化参数指定以逗号分隔的字符串列表，这些字符串限制Oracle ASM实例发现的磁盘集。

发现字符串可以包含通配符。仅发现与其中一个字符串匹配的磁盘。无法同时发现两次相同的磁盘。

发现字符串格式取决于Oracle ASM库和正在使用的操作系统。支持模式匹配。有关默认模式匹配的信息，请参阅特定于操作系统的安装指南。

例如，在Linux服务器上，要将发现过程限制为仅包含不使用Oracle ASM过滤器驱动程序（Oracle ASMFD）或ASMLIB的Oracle ASM实例的/dev/rdsk/mydisks目录中的磁盘，请设置ASM_DISKSTRING初始化参数：

的/ dev / rdsk / mydisks / *

星号是必需的。

要将发现过程限制为仅包含名称以disk3或disk4结尾的磁盘，可以在Linux系统上按如下方式设置ASM_DISKSTRING ：

ASM_DISKSTRING = '/dev/rdsk/*disk3' ， '/dev/rdsk/*disk4'

的? 字符用作路径的第一个字符时，会扩展到Oracle主目录。根据操作系统的不同，何时使用? 路径中其他位置的字符，它是一个字符的通配符。

ASM_DISKSTRING参数的默认值是NULL字符串。NULL值会导致Oracle ASM搜索Oracle ASM实例具有读写访问权限的系统中所有磁盘的缺省路径。默认搜索路径是特定于平台的。有关默认搜索路径的详细信息，请参阅特定于操作系统的安装指南。

除非群集中的所有Oracle ASM实例都可以通过其自己的发现字符串之一发现磁盘，否则Oracle ASM无法使用磁盘。每个节点上的名称不必相同，但群集中的所有节点都必须能够发现所有磁盘。这可能需要动态更改初始化参数以启用添加新存储。

ASM_IO_PROCESSES

ASM_IO_PROCESSES初始化参数指定要在Oracle IOServer（IOS）服务器实例中启动的IO工作进程数。

此参数仅适用于在Oracle Grid Infrastructure主目录中运行的Oracle IOServer实例。

在为客户端群集生成凭据文件时，IOServer实例会在Oracle Grid Infrastructure配置中自动启动。您可以配置在客户端群集中运行的IOServer实例的数量。

在大多数情况下，默认值应该有效。但是，在IO负载较重的情况下，可能需要比默认值更高的值。

ASM_POWER_LIMIT

ASM_POWER_LIMIT初始化参数指定磁盘组中磁盘重新平衡的默认功率。

值的范围是0到1024 。默认值为1 。值为0将禁用重新平衡。较高的数值使重新平衡操作能够更快地完成，但可能会导致更高的I / O开销和更多的重新平衡过程。

  • 对于磁盘组ASM兼容性设置为11.2.0.2或更高的磁盘组（例如， COMPATIBLE.ASM = 11.2.0.2 ），重新平衡功率的值的操作范围为0到1024 。

  • 对于磁盘组ASM兼容性设置为小于11.2.0.2的磁盘组，值的操作范围为0到11包括0和11 。如果ASM_POWER_LIMIT的值大于11 ，则对这些磁盘组使用值11 。

您还可以使用SQL ALTER DISKGROUP的POWER子句指定磁盘组中重新平衡操作的DISKGROUP 。REBALANCE声明。POWER子句的允许值范围对于ASM_POWER_LIMIT初始化参数是相同的。如果为ASM兼容性设置为小于11.2.0.2的磁盘组指定的POWER子句的值大于11 ，则会显示警告，并且将使用等于11的POWER值进行重新平衡。

磁盘组中重新平衡操作的功能规范仅影响重新平衡操作，而不影响磁盘组的新分配。

注意：

  • Oracle建议您不要将ASM_POWER_LIMIT初始化参数设置为0 。

  • 您无法在Oracle Exadata或Oracle数据机环境中将ASM_POWER_LIMIT初始化参数设置为0 。

ASM_PREFERRED_READ_FAILURE_GROUPS

ASM_PREFERRED_READ_FAILURE_GROUPS初始化参数值是逗号分隔的字符串列表，用于指定应由给定实例优先读取的故障组。

注意：

ASM_PREFERRED_READ_FAILURE_GROUPS初始化参数已在Oracle ASM 12 c第2版（12.2）中弃用，并且可能在将来的版本中被禁用。

ASM_PREFERRED_READ_FAILURE_GROUPS参数设置是特定于实例的。默认值为NULL 。此参数通常用于集群Oracle ASM实例，其值可以在不同节点上不同。

例如：

diskgroup_name1 。 failure_group_name1 ，...

注意：

在包含跨多个物理上分离的站点的节点的Oracle扩展群集中， PREFERRED_READ.ENABLED磁盘组属性控制是否为磁盘组启用首选读取功能。如果启用了首选读取功能，则此功能使实例能够确定和读取与自身相同的站点上的磁盘，从而提高性能。无论是否已启用PREFERRED_READ.ENABLED ，都可以使用ASM_PREFERRED_READ_FAILURE_GROUPS初始化参数在Oracle ASM实例或集群中的客户端实例上的故障组级别设置首选读取，该参数可用于向后兼容。

ASM_PREFERRED_READ_FAILURE_GROUPS设置的故障组值优先于PREFERRED_READ.ENABLED站点值。客户端实例上的ASM_PREFERRED_READ_FAILURE_GROUPS值优先于Oracle ASM实例上的值。

PROCESSES

PROCESSES初始化参数影响Oracle ASM，但默认值通常是合适的。

但是，如果多个数据库实例连接到Oracle ASM实例，则可以使用以下公式，其中n是连接到Oracle ASM实例的数据库实例的数量。

在非Exadata环境中，建议的设置为：

  • 对于n <10， PROCESSES = 50 * n + 50

  • 对于n > = 10， PROCESSES = 10 * n + 450

在Oracle Exadata环境中，建议的设置为PROCESSES = MAX （450 + 10 * n ，1024）。

关于设置用于Oracle ASM的数据库初始化参数

如果不在数据库实例中使用自动内存管理，则数据库实例的SGA参数设置可能需要稍作修改才能支持Oracle ASM。使用自动内存管理时，本节中讨论的大小调整数据可以仅作为参考信息处理，也可以作为补充信息处理，以帮助确定应用于SGA的适当值。 Oracle强烈建议使用自动内存管理。

以下是SGA调整数据库实例大小的配置指南：

  • PROCESSES初始化参数 - 将16添加到当前值

  • LARGE_POOL_SIZE初始化参数 - 向当前值添加额外的600K

  • SHARED_POOL_SIZE初始化参数 - 聚合以下查询中的值以获取Oracle ASM上或存储在Oracle ASM中的当前数据库存储大小。接下来，确定冗余类型并使用聚合值作为输入计算SHARED_POOL_SIZE 。 

    SELECT SUM（字节）/（1024 * 1024 * 1024）FROM V $ DATAFILE; SELECT SUM（bytes）/（1024 * 1024 * 1024）FROM V $ LOGFILE a，V $ LOG b WHERE a.group＃= b.group＃; SELECT SUM（bytes）/（1024 * 1024 * 1024）FROM V $ TEMPFILE WHERE status ='ONLINE';

    • 对于使用外部冗余的磁盘组，每100 GB空间需要1 MB额外共享池加2 MB

    • 对于使用常规冗余的磁盘组，每50 GB空间需要1 MB的额外共享池和4 MB

    • 对于使用高冗余的磁盘组，每33 GB的空间需要1 MB的额外共享池加上6 MB
