12.1.1.1关于创建应用程序容器

CREATE PLUGGABLE DATABASE ...AS APPLICATION CONTAINER语句创建一个新的应用程序容器。

应用程序容器由应用程序根和一组应用程序PDB组成，这些应用程序PDB存储一个或多个应用程序的数据。应用程序PDB插入应用程序根目录，您可以选择创建应用程序种子，以便快速轻松地创建新的应用程序PDB。应用程序PDB和应用程序根可以共享应用程序公共对象。

应用程序常见对象有三种类型：

  • 元数据链接的应用程序公共对象存储特定对象（例如表）的元数据，以便共享应用程序公共对象的容器具有相同的结构但具有不同的数据。

  • 数据链接的应用程序公共对象在应用程序根目录中定义一次，并在托管应用程序PDB的上下文中作为只读对象共享。

  • 扩展的数据链接应用程序公共对象在应用程序根目录中存储共享数据，但也允许应用程序PDB存储附加到该对象的数据。附加数据是每个应用程序PDB唯一的本地数据。

您可以通过在CREATE PLUGGABLE DATABASE语句中包含AS APPLICATION CONTAINER子句来创建应用程序容器。您可以使用以下技术来创建应用程序容器：

  • 使用PDB种子

  • 克隆现有的PDB或非CDB

  • 重新安置PDB

  • 插入未插入的PDB

要创建应用程序容器，当前容器必须是CDB根目录，并且必须在CREATE PLUGGABLE DATABASE语句中指定AS APPLICATION CONTAINER子句。您必须使用Oracle Managed Files创建应用程序容器。

注意：

如果任何应用程序PDB属于它，则无法拔出或删除应用程序容器。

将现有应用程序迁移到应用程序容器

您可以通过使用现有PDB创建应用程序根目录将应用程序迁移到应用程序根目录。将现有应用程序迁移到应用程序容器时，必须完成其他任务。您插入的PDB必须包含应用程序对象，包括其数据，您必须在DBMS_PDB包中运行过程以指定共享哪些对象。此外，当应用程序根目录中存在应用程序公共用户，角色或配置文件时，您必须在DBMS_PDB包中运行过程以指定它们是通用的。

将应用程序迁移到应用程序根目录后，您可以在应用程序根目录中创建应用程序PDB，并使用现有PDB创建应用程序PDB。

12.1.1.2准备应用程序容器

创建应用程序容器之前必须满足先决条件。

  • CDB必须存在。

  • CDB必须处于读/写模式。

  • 当前用户必须是当前容器为CDB根目录的普通用户。

  • 当前用户必须具有CREATE PLUGGABLE DATABASE系统特权。

  • 您必须为每个应用程序容器确定唯一的应用程序容器名称。每个应用程序容器名称对于单个CDB中的所有容器必须是唯一的，并且每个应用程序容器名称在通过特定侦听器到达其数据库实例的所有CDB的范围内必须是唯一的。

    应用程序容器名称用于区分应用程序容器与CDB中的其他容器。应用程序容器名称遵循与服务名称相同的规则，其中包括不区分大小写。

  • 您必须使用Oracle Managed Files创建包含。

  • 如果要在具有物理备用数据库的Oracle Data Guard配置中创建应用程序容器，则必须在创建应用程序容器之前完成其他任务。

  • 如果要使用安装脚本将现有应用程序迁移到应用程序容器，则必须可以运行脚本。

  • 如果要使用PDB将现有应用程序迁移到应用程序容器，则必须可以将PDB克隆到应用程序根目录或将PDB插入应用程序根目录。

12.1.1.3创建应用程序容器

您可以使用带有AS APPLICATION CONTAINER子句的CREATE PLUGGABLE DATABASE语句创建应用程序容器。

在创建应用程序容器之前，请完成“ 准备应用程序容器 ”中描述的先决条件。
  1. 在SQL * Plus中，确保当前容器是CDB根目录。
  2. 运行CREATE PLUGGABLE DATABASE语句，并包含AS APPLICATION CONTAINER子句。在需要时指定其他子句。
    创建应用程序容器后，它处于装入模式，其状态为NEW 。您可以通过查询V$PDBS视图中的OPEN_MODE列来查看应用程序容器的打开模式。您可以通过查询CDB_PDBS或DBA_PDBS视图的STATUS列来查看应用程序容器的STATUS 。
    为应用程序容器创建新的默认服务。该服务与应用程序容器具有相同的名称，可用于访问应用程序容器。必须正确配置Oracle Net Services才能使客户端访问此服务。
  3. 以读/写模式打开新的应用程序容器。
    必须以Oracle数据库的读/写模式打开新的应用程序容器，才能完成新应用程序容器与CDB的集成。如果您尝试以只读模式打开应用程序容器，则会返回错误。在读/写模式下打开应用程序容器后，其状态为NORMAL 。
  4. 备份应用程序容器。
    除非备份，否则无法恢复应用程序容器。

    注意：

    如果在创建应用程序容器期间返回错误，则正在创建的应用程序容器可能处于UNUSABLE状态。您可以通过查询CDB_PDBS或DBA_PDBS视图来检查应用程序容器的状态，并通过检查警报日志了解有关应用程序容器创建错误的更多信息。只能删除不可用的应用程序容器，并且必须先删除它，然后才能创建与不可用的应用程序容器同名的应用程序容器或PDB。

  5. 如果要将现有应用程序迁移到应用程序容器，请按照“ 将现有应用程序迁移到应用程序容器 ”中的说明进行操作 。
使用应用程序根创建应用程序容器。您可以在应用程序容器中创建应用程序PDB。

示例12-1使用PDB种子创建应用程序容器

此示例假定以下因素：

  • 应用程序容器不需要存储限制。因此，不需要STORAGE子句。

  • 应用程序容器不需要默认表空间。

  • PATH_PREFIX子句不是必需的。

  • 不需要FILE_NAME_CONVERT子句和CREATE_FILE_DEST子句。

    为CDB启用了Oracle Managed Files，或者设置了PDB_FILE_NAME_CONVERT初始化参数。与PDB种子关联的文件将根据Oracle Managed Files配置或初始化参数设置复制到新位置。

  • 没有与将在目标位置创建的新临时文件同名的文件。因此， TEMPFILE REUSE子句。

  • 不需要为PDB_DBA角色授予预定义的Oracle角色。

以下语句从PDB种子创建应用程序容器：

CREATE PLUGGABLE DATABASE salesact AS APPLICATION CONTAINER ADMIN USER salesadm IDENTIFIED BY password ;

12.1.2.1关于拔出应用程序容器

拔出应用程序容器会取消应用程序容器与CDB的关联。

通常，当您要将应用程序容器移动到其他CDB时，请拔出应用程序容器。此外，您可以在不再希望应用程序容器可用时拔下它。

拔出应用程序容器类似于拔出PDB。要拔出应用程序容器，请连接到其CDB根目录并使用ALTER PLUGGABLE DATABASE语句指定XML文件或.pdb文件。指定XML文件（ .xml扩展名）时，它将包含有关应用程序容器拔出后的元数据。SQL语句创建XML文件，它包含在目标CDB上启用CREATE PLUGGABLE DATABASE语句以插入应用程序容器所需的信息。指定.pdb文件时，它包含XML文件的压缩存档，该存档描述应用程序容器和应用程序容器使用的文件（例如数据文件和钱包文件）。使用.pdb文件可以将单个压缩文件（而不是多个文件）复制到新位置，以将应用程序容器插入CDB。

在拔出插头之前，应用程序容器不得插入任何应用程序PDB，并且必须将其关闭。拔下应用程序容器时，未插入的应用程序容器处于装入模式。拔出操作会对应用程序容器的数据文件进行一些更改以记录，例如，应用程序容器已成功拔出。因为它仍然是CDB的一部分，所以未插入的应用程序容器包含在整个CDB的RMAN备份中。这种备份提供了一种方便的方式来存档未插入的应用程序容器，以备将来需要时使用。

要从CDB中完全删除应用程序容器，可以将其删除。拔出的应用程序容器上支持的唯一操作是删除应用程序容器。必须先将应用程序容器从CDB中删除，然后才能将其插回到同一CDB中。应用程序容器只有在插入CDB时才可用。

12.1.2.2拔出应用程序容器

使用ALTER PLUGGABLE DATABASE ...应用程序容器ALTER PLUGGABLE DATABASE ...UNPLUG INTO声明。

先决条件

您必须满足以下先决条件：

  • 当前用户必须具有SYSDBA或SYSOPER管理权限，并且必须在PDB中共同授予或本地授予该权限。用户必须在连接时使用AS SYSDBA或AS SYSOPER行使权限。

  • 应用程序容器必须至少打开一次。

  • 应用程序容器不得将任何应用程序PDB插入其中。

  • 应用程序容器不能将应用程序种子插入其中。

注意：

如果要拔出包含使用透明数据加密加密的数据的应用程序容器，请按照“ Oracle数据库高级安全性指南”中的说明进行操作。

要拔掉应用程序容器：

  1. 在SQL * Plus中，确保当前容器是CDB的根目录。

  2. 关闭应用程序容器。

    在Oracle Real Application Clusters（Oracle RAC）环境中，必须在所有实例上关闭应用程序容器。

  3. 使用UNPLUG INTO子句运行ALTER PLUGGABLE DATABASE语句，并指定要拔出的应用程序容器以及应用程序容器的XML元数据文件或.pdb文件的名称和位置。

示例12-4取消应用程序容器salesact

此ALTER PLUGGABLE DATABASE语句ALTER PLUGGABLE DATABASE应用程序容器salesact并在/oracle/data/ directory创建salesact.xml元数据文件：

ALTER PLUGGABLE DATABASE salesact UNPLUG INTO'/oracle/data/saleact.xml';

12.1.3删除应用程序容器

如果要将应用程序容器从一个CDB移动到另一个CDB或不再需要应用程序容器，可以删除应用程序容器。

删除应用程序容器与删除PDB非常相似。删除应用程序容器时，将修改CDB的控制文件以消除对已删除的应用程序容器的所有引用。存档的重做日志文件和与应用程序容器关联的备份不会被删除，但您可以使用Oracle Recovery Manager（RMAN）将其删除。

删除应用程序容器时，可以使用DROP PLUGGABLE DATABASE语句中的以下子句之一来保留或删除应用程序容器的数据文件：

  • 保留KEEP DATAFILES （默认值）保留数据文件。

    即使指定了KEEP DATAFILES也会删除应用程序容器的临时文件，因为不再需要临时文件。

  • INCLUDING DATAFILES会从磁盘中删除数据文件。

    如果使用SNAPSHOT COPY子句创建了应用程序容器，则在删除应用程序容器时必须指定INCLUDING DATAFILES 。

必须满足以下先决条件：

  • 应用程序容器必须处于已安装模式，或者必须拔下插头。

    请参阅“ 修改一个或多个PDB的开放模式 ” 。

    请参阅“ 拔出应用程序容器 ” 。

  • 当前用户必须具有SYSDBA或SYSOPER管理权限，并且必须在应用程序容器中共同授予或本地授予该权限。用户必须在连接时使用AS SYSDBA或AS SYSOPER行使权限。

  • 应用程序容器不得将任何应用程序PDB插入其中。

  • 应用程序容器不能将应用程序种子插入其中。

注意：

这种操作具有破坏性。

要删除应用程序容器：

  1. 在SQL * Plus中，确保当前容器是CDB根目录。

    请参阅“ 关于当前容器 ”和“ 使用SQL * Plus访问CDB中的容器 ” 。

  2. 运行DROP PLUGGABLE DATABASE语句并指定要删除的应用程序容器。

示例12-5在保留其数据文件的同时删除应用程序容器salesact

DROP PLUGGABLE DATABASE salesact KEEP DATAFILES;

示例12-6删除应用程序容器salesact及其数据文件

DROP PLUGGABLE DATABASE saleact包括DATAFILES;

12.2.1.3创建应用程序种子

您可以通过在CREATE PLUGGABLE DATABASE语句中包含AS SEED子句来创建应用程序种子。

应用程序容器中的应用程序种子类似于CDB中的种子。通过应用程序种子，您可以快速轻松地创建满足应用程序容器要求的应用程序PDB。

在创建应用程序种子之前，请完成“ 准备应用程序种子 ”中描述的先决条件。
  1. 在SQL * Plus中，确保当前容器是应用程序根目录。
  2. 运行CREATE PLUGGABLE DATABASE语句，并包含AS SEED子句，以创建应用程序种子。在需要时指定其他子句。
    创建应用程序种子后，它处于装入模式，其状态为NEW 。您可以通过查询V$PDBS视图中的OPEN_MODE列来查看应用程序种子的打开模式。您可以通过查询CDB_PDBS或DBA_PDBS视图的STATUS列来查看应用程序种子的STATUS 。
    为应用程序种子创建新的默认服务。该服务与应用程序种子具有相同的名称，可用于访问应用程序种子。必须正确配置Oracle Net Services才能使客户端访问此服务。
  3. 以读/写模式打开新的应用程序种子。
  4. 必须在Oracle数据库的读/写模式下打开新的应用程序种子，才能完成新应用程序种子与应用程序容器的集成。如果尝试以只读模式打开应用程序种子，则会返回错误。在读/写模式下打开应用程序种子后，其状态为NORMAL 。
  5. 执行以下一个或多个操作：
    • 如果应用程序种子是从PDB种子创建的，则将容器切换到应用程序种子，并使用带有SYNC子句的ALTER PLUGGABLE DATABASE语句来同步应用程序种子。与应用程序根同步会在应用程序种子中实例化一个或多个应用程序根应用程序。
    • 如果应用程序种子是从应用程序根目录创建的，则将容器切换到应用程序种子，然后运行pdb_to_apppdb.sql脚本将应用程序根目录转换为应用程序PDB。
    通过克隆应用程序PDB创建应用程序种子时，不需要这些操作。
  6. 关闭应用程序种子，然后以打开的只读模式打开它。
  7. 备份应用程序种子。
    除非备份，否则无法恢复应用程序种子。

    注意：

    • 如果在应用程序种子创建期间返回错误，则正在创建的应用程序种子可能处于UNUSABLE状态。您可以通过查询CDB_PDBS或DBA_PDBS视图来检查应用程序种子的状态，并通过检查警报日志了解有关应用程序种子创建错误的更多信息。只能删除不可用的应用程序种子。

    • 当应用程序根目录中的应用程序在应用程序根目录中升级或修补时，应用程序种子必须与应用程序根目录同步以包含更改。

12.2.2.1关于拔出应用程序种子

拔出应用程序种子会取消应用程序种子与应用程序容器的关联。当您不再希望应用程序种子可用时，请拔下应用程序种子。

拔出应用程序种子类似于拔出PDB。要拔出应用程序种子，请连接到其应用程序根目录并使用ALTER PLUGGABLE DATABASE语句指定XML文件或.pdb文件。指定XML文件（ .xml扩展名）时，它将在拔出插件后包含有关应用程序种子的元数据。SQL语句创建XML文件，它包含在目标CDB上启用CREATE PLUGGABLE DATABASE语句以将其作为PDB或应用程序PDB插入所需的信息。指定.pdb文件时，它包含XML文件的压缩归档，该归档描述应用程序种子和应用程序种子使用的文件（例如数据文件和钱包文件）。使用.pdb文件可以将单个压缩文件（而不是多个文件）复制到新位置，以作为PDB或应用程序PDB插入。

在拔掉插头之前，必须关闭应用程序种子。拔下应用程序种子时，未插入的应用程序种子处于装入模式。拔出操作会对应用程序种子的数据文件进行一些更改以记录，例如，应用程序种子已成功拔出。因为它仍然是应用程序容器的一部分，所以未插入的应用程序种子包含在整个CDB的RMAN备份中。这种备份提供了一种方便的方法来存档未插入的应用程序种子，以备将来需要时使用。

要从应用程序容器中完全删除应用程序种子，可以将其删除。拔出的应用程序种子支持的唯一操作是删除应用程序种子。必须先从应用程序容器中删除应用程序种子，然后才能将其插回到同一应用程序容器中。应用程序种子仅在插入应用程序容器时才可用。

12.2.2.2拔掉应用程序种子

要拔掉应用程序种子，请运行ALTER PLUGGABLE DATABASE ...UNPLUG INTO声明。

先决条件

    在Oracle Real Application Clusters（Oracle RAC）环境中，必须在所有实例上关闭应用程序种子。

12.2.3删除应用程序种子

您可以使用DROP PLUGGABLE DATABASE语句删除应用程序种子。您可以在不再需要时删除应用程序种子。

删除应用程序种子时，将修改CDB的控制文件以消除对已删除的应用程序种子的所有引用。存档的重做日志文件和与应用程序种子关联的备份不会被删除，但您可以使用Oracle Recovery Manager（RMAN）将其删除。

删除应用程序种子时，可以使用以下某个子句保留或删除应用程序种子的数据文件：

  • 保留KEEP DATAFILES （默认值）保留数据文件。

    即使指定了KEEP DATAFILES也会删除应用程序种子的临时文件，因为不再需要临时文件。

  • INCLUDING DATAFILES会从磁盘中删除数据文件。

    如果使用SNAPSHOT COPY子句创建应用程序种子，则必须在删除应用程序种子时指定INCLUDING DATAFILES 。

必须满足以下先决条件：

  • 应用程序种子必须处于挂载模式，或者必须拔下插头。

  • 当前用户必须具有SYSDBA或SYSOPER管理权限，并且必须在应用程序容器中共同授予或本地授予该权限。用户必须在连接时使用AS SYSDBA或AS SYSOPER行使权限。

注意：

这种操作具有破坏性。

删除应用程序种子：

  1. 在SQL * Plus中，确保当前容器是应用程序种子所属的应用程序容器的应用程序根目录。

  2. 运行DROP PLUGGABLE DATABASE语句并指定应用程序种子。

示例12-11在保留其数据文件的同时删除应用程序种子salesact $ SEED

DROP PLUGGABLE DATABASE salesact $ SEED KEEP DATAFILES;

示例12-12删除应用程序种子salesact $ SEED及其数据文件
