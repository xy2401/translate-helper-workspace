1 Oracle数据库测试简介

Oracle数据库的Oracle Real Application Testing选项可帮助您安全地确保数据库更改的完整性并管理测试数据。

神谕 Real Application Testing选项使您可以执行Oracle数据库的实际测试。通过在生产部署之前捕获生产工作负载并评估系统更改对这些工作负载的影响，Oracle Real Application Testing可最大程度地降低与系统更改相关的不稳定性风险。SQL性能分析器和数据库重放是关键 Oracle Real Application Testing的组件。根据所测试的系统更改的性质和影响，以及将要执行测试的系统类型，您可以使用其中一个或两个组件来执行测试。

本章包含以下部分：

  • SQL性能分析器

  • 数据库重播

注意：

使用SQL Performance Analyzer和Database Replay需要Oracle Real Application Testing许可选项。有关更多信息，请参见“ Oracle数据库许可信息用户手册” 。

1.1 SQL性能分析器

系统更改（例如升级数据库或添加索引）可能会导致更改SQL语句的执行计划，从而对SQL性能产生重大影响。 在某些情况下，系统更改可能会导致SQL语句退化，从而导致性能下降。在其他情况下，系统更改可能会提高SQL性能。能够准确预测系统更改对SQL性能的潜在影响使您能够在SQL语句退化的情况下预先调整系统，或者在SQL语句性能提高的情况下验证和测量性能增益。

SQL Performance Analyzer通过识别每个SQL语句的性能差异，自动完成评估更改对整个SQL工作负载的总体影响的过程。提供了一份报告，显示由于更改而对工作负载性能的净影响。退步 SQL语句，SQL性能分析器还提供适当的执行计划详细信息以及调优建议。因此，您可以在最终用户受到影响之前纠正任何负面结果。此外，您可以通过显着的时间和成本节省进行验证 - 系统更改为生产环境将导致净改进。

您可以使用SQL Performance Analyzer 分析任何类型的系统更改对SQL性能的影响，包括：

  • 数据库升级

  • 可插拔数据库（PDB）和手动架构合并的数据库整合测试

  • 配置更改操作系统或硬件

  • 架构更改

  • 更改数据库初始化参数

  • 刷新优化程序统计信息

  • 验证SQL调优操作

1.2数据库重放

在进行系统更改（例如硬件和软件升级）之前，通常在测试环境中执行大量测试以验证更改。 但是，尽管进行了测试，新系统在进入生产时经常会出现意外行为，因为测试不是使用实际工作负载来执行的。在测试期间无法模拟实际工作负载是验证系统更改时的最大挑战之一。

数据库重放通过在测试系统上重新创建生产工作负载环境，实现对系统更改的真实测试。使用数据库重放，您可以捕获生产系统上的工作负载，并使用原始工作负载的准确时间，并发性和事务特征在测试系统上重放它。这使您可以全面评估更改的影响，包括不良结果，新争用点或计划回归。提供广泛的分析和报告，以帮助识别任何潜在的问题，例如遇到的新错误和性能差异。

数据库重放在数据库级别捕获外部数据库客户端的工作负载，并且具有可忽略的性能开销。捕获生产工作负载消除了开发模拟工作负载或脚本的需要，从而显着降低成本并节省时间。通过使用数据库重放，现在可以在几天内完成以前使用负载模拟工具花费数月时间的复杂应用程序的实际测试。这使您能够以更高的置信度和更低的风险快速测试变更并采用新技术。

您可以使用数据库重放以测试任何重大系统更改，包括：

  • 数据库和操作系统升级

  • PDB的数据库整合测试和手动架构整合

  • 使用工作负载扩展来编写和试验各种方案

  • 配置更改，例如将数据库从单个实例转换为Oracle Real Application Clusters（Oracle RAC）环境

  • 存储，网络和互连更改

  • 操作系统和硬件迁移
