安装适用于Oracle RAC的新Oracle数据库软件

使用此过程概述可帮助您为Oracle RAC .deployment安装新Oracle Database版本的软件。

注意：

当源和目标Oracle主目录由不同用户拥有时，无法使用数据库升级助手（DBUA）升级数据库。试图这样做会返回错误PRKH-1014。确保源数据库和目标数据库具有相同的所有者，或执行手动升级。

如果要升级Oracle RAC数据库，则必须按所示顺序执行以下步骤：

  1. 升级Oracle Clusterware：

    1. 首先按照适用于您的操作系统的Oracle Grid Infrastructure安装指南中的说明升级Oracle Clusterware。

    2. 挂载Oracle Grid Infrastructure安装介质。

    3. 对要升级的每个节点执行操作系统先决条件检查，以确保它们满足Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）的系统先决条件。

    4. 如有必要，请将早期版本的Oracle Clusterware软件的修补程序升级到最新的修补程序版本。

    5. 确保以拥有Oracle Grid Infrastructure安装的用户身份登录，并运行Oracle Grid Infrastructure安装。根据安装程序的提示提供信息。

    6. 出现提示时，打开一个单独的终端会话，以root身份登录，然后运行root.sh

  2. 升级Oracle Clusterware后，请按照Oracle操作系统特定文档中的说明准备Oracle Database软件安装。

  3. 运行升级前信息工具。您可以预览DBUA检查的项目类型，并识别问题。升级前信息工具可以帮助您解决它找到的一些先决条件问题。

  4. 启动Oracle Universal Installer，然后安装该软件。

    成功完成Oracle数据库软件的安装后，单击“ 退出”以关闭Oracle Universal Installer。

  5. 运行DBUA。

    如果使用Oracle Label Security，Oracle Database Vault或两者，则在“ 选择数据库版本”页面上选择Enterprise Edition ，单击“ 选择选项” ，然后从组件列表中启用一个或两个组件。
