ALTER DATABASE DICTIONARY

目的

加密混淆的数据库链接密码并使用TDE框架来管理加密密钥。

可以为LOB定位符（指向大对象（LOB）值的位置的指针）分配签名以保护LOB。

先决条件

  • TDE密钥库必须存在。DDL首先检查TDE：

    • 密钥库存在。

    • 密钥库是开放的。

    • Master加密密钥存在于TDE密钥库中。

      如果任何检查失败，则DDL失败。发生这种情况时，您必须创建TDE密钥库并提供TDE主密钥。有关更多信息，请参阅数据库安

  • 实例初始化参数COMPATIBLE必须设置为12.2.0.2。

  • 您必须具有SYSKM权限才能执行该命令。

句法

alter_database_dictionary :: =

语义

alter_database_dictionary_encrypt_credentials :: =

此DDL加密数据字典中现有和未来混淆的敏感信息，例如存储在SYS.LINKS$数据库链接密码。

它执行以下操作：

  • 在ENC$插入与SYS.LINK$对应的新条目。

  • 它创建并初始化SGA变量。

  • 在SYS.LINK$模糊密码进行SYS.LINK$模糊SYS.LINK$ 。

  • 使用ENC$为SYS.LINK$生成的加密密钥加密去混淆的密码。

  • 设置标志以指示SYS.LINK$的有效/可用dblink条目。

将此DDL与LOB定位器签名密钥一起使用时，它们始终是加密的。可以为LOB定位符（指向大对象（LOB）值的位置的指针）分配签名以保护LOB。

alter_database_dictionary_rekey_credentials :: =

此DDL用于更改数据加密密钥。它适用于SYS.LINK$和数据字典加密框架下涵盖的任何其他表。

您还可以使用此DDL为LOB定位器重新生成LOB定位器签名密钥。如果数据库处于受限模式，则Oracle数据库将重新生成新的LOB签名密钥，并使用新的加密密钥对其进行加密。如果数据库处于非限制模式，则不会重新生成新的签名密钥，而是Oracle数据库使用新的加密密钥来加密现有的LOB签名密钥。

alter_database_dictionary_delete_credentials_key :: =

此DDL标记加密密码不可用。这意味着SYS.LINK$中的当前密码条目被标记为不可用。它删除用于加密凭据的ENC$中的密钥，并清除SGA变量以防止将来加密。

您还可以使用此DDL删除加密的LOB定位器签名密钥，然后以模糊形式重新生成新的LOB签名密钥。
