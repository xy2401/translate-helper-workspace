3.13发送

目的

使用SEND命令将特定于供应商的字符串发送到媒体管理器支持的一个或多个通道。请参阅媒体管理文档以确定支持哪些命令。

使用说明

除非您指定DEVICE TYPE或CHANNEL ，否则RMAN将使用所有已分配的通道。

在Windows平台上，使用SEND将命令字符串传递到SBT库，而不是在通道分配期间使用PARAMS选项中的ENV设置。在Oracle数据库中，在Windows上并行运行的操作在单个进程中使用多个线程。由一个线程设置的环境变量对所有线程都可见。因此，依赖于特定线程设置的环境变量的并行操作可能无法按预期工作。

句法

发送 :: =


send.eps插图说明

（ deviceSpecifier :: = ）

语义

语法元素 描述

CHANNEL channel_id

指定要使用的通道。您必须在CHANNEL关键字后指定区分大小写的通道ID，即通道的名称。数据库使用通道ID报告I / O错误。

DEVICE TYPE deviceSpecifier

指定存储设备的类型，并将命令发送到指定类型的所有通道。

另请参见： deviceSpecifier

' command '

指定特定于供应商的介质管理命令。

另请参阅：您的媒体管理文档，以确定支持哪些命令。您只能发送媒体管理器支持的命令。字符串的内容不由数据库解释，而是以不变的方式传递给媒体管理子系统。

PARMS ' channel_parms '

指定与媒体管理器通信的通道的参数。

例

示例3-56在Oracle Secure Backup中指定磁带驱动器

此示例使用SEND命令指定用于备份users表空间的磁带机 Oracle Secure Backup。参数OB_DEVICE和磁带机名称之间没有插入OB_DEVICE 。 

RUN {ALLOCATE CHANNEL c1 DEVICE TYPE sbt;发送'OB_DEVICE stape1'; BACKUP TABLESPACE用户; }
