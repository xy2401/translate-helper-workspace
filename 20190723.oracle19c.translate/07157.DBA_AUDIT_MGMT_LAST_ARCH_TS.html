4.112 DBA_AUDIT_MGMT_LAST_ARCH_TS

DBA_AUDIT_MGMT_LAST_ARCH_TS显示有关为审计跟踪清除或清除设置的上次存档时间戳的信息。

注意：

无论是否为数据库启用了Oracle Database 12 c之前的审核或统一审核，此视图都会填充到启用了审核的任何Oracle数据库中。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

柱 数据类型 空值 描述

AUDIT_TRAIL

VARCHAR2（20）

适用于上次存档时间戳的审计跟踪：

  • 标准审计跟踪

  • FGA审计跟踪

  • OS AUDIT TRAIL

  • XML审计跟踪

  • 统一的审计轨道

RAC_INSTANCE

数

不是NULL

适用最后一个存档时间戳的Oracle RAC实例编号; 0意味着不适用

LAST_ARCHIVE_TS

TIMESTAMP（6）带时区

已归档的上一个审核记录或审核文件的时间戳

database_id的

数

不是NULL

要清理的审计记录的数据库ID

CONTAINER_GUID

VARCHAR2（33）

不是NULL

要清理的审计记录的容器的GUID

注意：

在只读数据库（包括Oracle Active Data Guard物理备用数据库）中，调用DBMS_AUDIT_MGMT.SET_LAST_ARCHIVE_TIMESTAMP时不会填充此视图。在这种情况下，如果为数据库实例配置了时间戳，请使用DBMS_AUDIT_MGMT.GET_LAST_ARCHIVE_TIMESTAMP来检查时间戳。
