使用早期Oracle数据库版本的一般限制

您可以将Oracle Database 19c，18c，Oracle Database 12c第1版和第2版以及Oracle Database 11g第2版（11.2.0.3或更高版本）与Oracle Grid Infrastructure 19c一起使用。

请勿使用srvctl ， lsnrctl或其他Oracle Grid Infrastructure主页工具的版本来管理早期版本的数据库。仅使用早期Oracle数据库主目录中的工具管理早期Oracle数据库版本。要确保您使用的工具版本是这些早期版本数据库的正确工具，请从您管理的数据库或对象的Oracle主目录运行工具。

如果数据库版本是Oracle Database 11g第2版或更高版本，则Oracle数据库主目录只能存储在Oracle ASM集群文件系统（Oracle ACFS）中。

在Oracle Flex ASM群集上安装11.2数据库时，必须将Oracle ASM基数设置为All 。

注意：

如果要使用Oracle Grid Infrastructure 19c安装Oracle Database 11g第2版，则在运行Oracle数据库的Oracle Universal Installer（OUI）之前，请仅在本地节点上运行以下命令：

Grid_home / oui / bin / runInstaller -ignoreSysPrereqs -updateNodeList ORACLE_HOME = Grid_home “CLUSTER_NODES = { comma_separated_list_of_nodes }”CRS = true LOCAL_NODE = local_node [-cfs]

仅当Grid_home位于共享位置时，才使用-cfs选项。
