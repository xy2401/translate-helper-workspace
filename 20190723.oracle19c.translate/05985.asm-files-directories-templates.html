关于Oracle ASM文件名

在Oracle ASM中创建的每个文件都提供由Oracle Managed Files生成的文件名，称为完全限定文件名（或系统生成的文件名）。完全限定的文件名表示Oracle ASM文件系统中的完整路径名。完全限定文件名的示例是：

+数据/ ORCL /控制文件/ Current.256.541956473

您可以使用完全限定的文件名来引用（读取或检索）Oracle ASM文件。您还可以使用其他缩写文件名格式（例如别名Oracle ASM文件名）来引用Oracle ASM文件。

在任何创建文件的请求时，Oracle ASM都会生成完全限定的文件名。创建请求不会或不能指定完全限定的文件名。相反，它使用更简单的语法来指定文件，例如别名或磁盘组名称。然后，Oracle ASM根据文件类型创建文件，将其放在正确的Oracle ASM路径中，然后分配适当的完全限定文件名。如果在创建请求中指定别名，Oracle ASM还会创建别名，以便它引用完全限定的文件名。

Oracle ASM文件创建请求可以是单个文件创建请求，也可以是多个文件创建请求。

注意：

  • 您可以在显示Oracle文件名的数据库视图中找到生成的名称，例如V$DATAFILE和V$LOGFILE 。如果以后必须在SQL语句中引用Oracle ASM文件，则可以使用此名称或其缩写形式。与其他Oracle数据库文件名一样，Oracle ASM文件名保存在控制文件和RMAN目录中。

  • 如果指定REUSE关键字，则可以在单文件创建中使用完全限定和数字文件名，如在SQL语句中使用Oracle ASM文件名中所述 。

  • 正斜杠（/）和反斜杠（\）在文件名中是可互换的。文件名不区分大小写，但保留大小写。

单个文件创建表单

单个文件创建表单用于创建单个文件，例如数据文件或控制文件。此类型中的Oracle ASM文件名的格式是别名，例如+data/orcl/ctl.f ，或者是不完整的文件名，例如以加号（ +data ）开头的磁盘组名称。您可以使用在SQL语句中调用文件名的别名或磁盘组名称，例如CREATE TABLESPACE ， CREATE CONTROLFILE 。

多个文件创建表单

多次使用多文件创建表单来创建Oracle ASM文件，例如在为初始化参数DB_CREATE_FILE_DEST分配值时。您还可以多次运行CREATE TABLESPACE语句（没有文件名规范）。每次，Oracle ASM都会创建不同的唯一数据文件名。

使用多文件创建表单时，可以使用不完整的文件名表单，例如以加号（ +data ）开头的磁盘组名称。在这种情况下，您将DB_CREATE_FILE_DEST设置为不完整的文件名（例如， +data ），并且每当运行必须在DB_CREATE_FILE_DEST创建数据库文件的DB_CREATE_FILE_DEST ，将在指定的磁盘组中创建该文件并DB_CREATE_FILE_DEST分配唯一的完全限定名称。您可以在其他*_DEST初始化参数中使用不完整的文件名。

完全合格的文件名表

除了创建磁盘组之外，您可以使用完全限定的表单来引用Oracle ASM操作中的现有Oracle ASM文件。此表单是Oracle ASM文件名，Oracle ASM在创建Oracle ASM文件时自动生成该文件名。

完全限定的文件名在非多租户环境中具有以下形式：

+ diskgroup / dbname / filetype / filetypetag.file.incarnation

对于多租户数据库，完全限定的文件名具有以下形式：

+ diskgroup / cdb / pdb_guid / filetype / filetypetag.file.incarnation

完全限定文件名的字段是：

  • + diskgroup是以加号diskgroup的磁盘组名称。

    您可以将加号（ + ）视为Oracle ASM文件系统的根目录，类似于UNIX或Linux计算机上的斜杠（ / ）。

  • dbname是文件所属的数据库的DB_UNIQUE_NAME 。

  • filetype是Oracle文件类型，可以是表5-3中显示的文件类型之一。

  • filetypetag是有关文件的特定于类型的信息，可以是表5-3中显示的标记之一。

  • file.incarnation是文件/化身对，用于确保唯一性。

  • cdb是cdb的名称

  • pdb_guid是PDB GUID值。

别名Oracle ASM文件名表单

您可以使用Alias Oracle ASM文件名来引用现有Oracle ASM文件和创建新的Oracle ASM文件。别名以磁头组名称开头，后面加一个加号，然后指定名称字符串。别名文件名使用分层目录结构实现，斜杠（/）或反斜杠（\）字符分隔名称组件。您可以在Oracle Managed Files（由系统生成）生成或由用户创建的任何Oracle ASM目录中创建别名。别名必须包含磁盘组名称;您无法在根级别（ + ）创建别名。

使用别名文件名创建Oracle ASM文件时，将使用完全限定名创建文件，并另外创建别名文件名。然后，您可以使用任一名称访问该文件。

别名Oracle ASM文件名与完全限定文件名或数字文件名不同，因为它们不以点对数字结尾。尝试创建以点对数字结尾的别名是错误的，例如格式为USERS.259.685366091 。Oracle ASM别名文件名的示例如下：

+ data / orcl / control_file1 + fra / recover / second.dbf

Oracle数据库按别名文件名引用数据库文件，但前提是您使用别名创建数据库文件。如果您创建没有别名的数据库文件，然后再添加别名，则数据库将通过其完全限定的文件名引用这些文件。以下是数据库如何使用别名文件名的示例：

  • 别名文件名出现在V $视图中。例如，如果创建表空间并使用数据文件的别名文件名，则V$DATAFILE视图将显示别名文件名。

  • 当控制文件指向数据文件和联机重做日志文件时，它可以使用别名文件名。

  • CONTROL_FILES初始化参数可以使用控制文件的别名文件名。Database Configuration Assistant（DBCA）使用别名文件名创建控制文件。

注意：

使用别名文件名创建的文件不被视为Oracle托管文件，如果不再需要，将来可能需要手动删除。

有关别名文件名的详细信息，请参阅管理Oracle ASM文件名的别名 。

在Oracle ASM中创建表空间：使用具有别名的数据文件

示例5-1创建一个undo表空间，其中包含一个具有别名的数据文件，以及由用户定义的模板my_undo_template设置的属性。此示例假定已在磁盘组data创建目录以包含别名并且存在用户定义的模板。

由于在创建数据文件时使用了别名，因此该文件不是Oracle Managed Files（OMF）文件，并且在删除表空间时不会自动删除该文件。要在删除表空间后手动删除文件，请使用以下SQL语句：

ALTER DISKGROUP数据DROP FILE'+ data / orcl / my_undo_ts';

使用模板表单别名Oracle ASM文件名

带模板的别名Oracle ASM文件名仅用于Oracle ASM文件创建操作。它具有以下格式：

+ diskgroup(templatename)/alias

带有模板的别名文件名与别名文件名的行为相同，只是使用带有模板的别名文件名创建的文件接收由命名模板指定的属性。模板必须属于正在创建文件的磁盘组。

带模板的别名Oracle ASM文件名的示例是：

+数据（MyTemplate的）/ CONFIG1

如前例所示，显式指定模板名称会覆盖正在创建的文件类型的系统默认模板。

注意：

使用带有模板的别名文件名创建的文件不被视为Oracle托管文件，如果不再需要，将来可能需要手动删除。

不完整的Oracle ASM文件名表

不完整的Oracle ASM文件名仅用于文件创建操作，并用于创建单个和多个文件。它们仅包含磁盘组名称。Oracle ASM使用系统默认模板来确定Oracle ASM文件镜像和条带化属性。使用的系统模板由正在创建的文件类型确定。例如，如果要为表空间创建数据文件，则使用DATAFILE模板。

使用不完整的Oracle ASM文件名的示例是将DB_CREATE_FILE_DEST初始化参数设置为：

+数据

使用此设置，每次创建表空间时，都会在磁盘组data创建数据文件，并为每个数据文件分配不同的完全限定名称。有关详细信息，请参阅使用磁盘组规范的默认文件位置创建Oracle ASM文件 。

带有模板表单的不完整Oracle ASM文件名

带有模板的不完整Oracle ASM文件名仅用于文件创建操作，并用于创建单个和多个文件。它们由磁盘组名称后跟括号中的模板名称组成。在文件名中显式指定模板时，Oracle ASM使用指定的模板而不是该文件类型的默认模板来确定文件的属性。

将不完整的Oracle ASM文件名与模板一起使用的示例是将DB_CREATE_FILE_DEST初始化参数设置为：

+ data(mytemplate)

使用磁盘组规范的默认文件位置创建Oracle ASM文件

对操作系统文件使用Oracle Managed Files功能，可以将目录指定为创建数据文件，临时文件，重做日志文件和控制文件的默认位置。使用Oracle ASM的Oracle托管文件功能，您可以以不完整的Oracle ASM文件名的形式指定磁盘组，作为创建这些文件的默认位置，以及其他类型的文件，包括归档日志文件。对于操作系统文件，默认磁盘组的名称存储在初始化参数中，只要在文件创建期间未明确指定文件规范（例如， DATAFILE子句），就会使用该名称。

以下初始化参数接受Oracle ASM文件名的多文件创建上下文形式作为目标：

  • DB_CREATE_FILE_DEST

    指定用于创建数据和临时文件的默认磁盘组位置。

  • DB_CREATE_ONLINE_LOG_DEST_ n

    指定用于创建重做日志和控制文件的默认磁盘组位置。

  • DB_RECOVERY_FILE_DEST

    如果指定了此参数DB_CREATE_ONLINE_LOG_DEST_指定DB_CREATE_ONLINE_LOG_DEST_ n和CONTROL_FILES则指定包含控制文件和重做日志文件副本的快速恢复区域的默认磁盘组。

  • CONTROL_FILES

    指定要在其中创建控制文件的磁盘组。

以下初始化参数接受Oracle ASM文件名的多文件创建上下文形式和Oracle ASM目录名作为目标：

  • LOG_ARCHIVE_DEST_ n

    指定默认磁盘组或Oracle ASM目录作为归档重做日志文件的目标

  • LOG_ARCHIVE_DEST

    用于指定默认磁盘组或Oracle ASM目录作为归档重做日志文件的目标的可选参数。仅指定一个目标时使用。

  • STANDBY_ARCHIVE_DEST

    仅与备用数据库相关。它指定默认磁盘组或Oracle ASM目录，该目录是从主数据库到达的存档日志的位置。本书未讨论。此参数已被弃用。请参阅Oracle Data Guard概念和管理 。

以下CREATE TABLESPACE SQL语句说明了如何在默认磁盘组中创建Oracle ASM文件（在本例中为数据文件）。此示例假定DB_CREATE_FILE_DEST初始化参数已设置为+data 。 

SQL> CREATE TABLESPACE mytblspace;

Oracle ASM自动为磁盘组data中的Oracle ASM磁盘上的mytblspace创建和管理数据文件。使用数据文件的默认模板定义的属性存储文件范围。

在SQL语句中使用Oracle ASM文件名

您可以在SQL语句的文件规范子句中指定Oracle ASM文件名。如果您是第一次创建文件，请使用Oracle ASM文件名的创建表单。如果存在Oracle ASM文件，则必须使用文件名的引用形式，如果尝试重新创建该文件，则必须添加REUSE关键字，以便为新文件重用该空间。例如，尝试重新创建控制文件时可能会出现这种情况，如在Oracle ASM中创建控制文件中所示。

如果引用表单与REUSE关键字一起REUSE且文件不存在，则会导致错误。

例5-2是在SQL语句中指定Oracle ASM文件名的示例。在这种情况下，它用于文件创建表单。

创建表空间mytblspace并包含磁盘组data包含的一个大小为200 MB的数据文件。数据文件设置为可自动扩展，具有无限的最大大小。您可以使用AUTOEXTEND子句覆盖此默认值。

有关在SQL语句中使用Oracle ASM文件名的更多示例，请参阅使用SQL * Plus在Oracle ASM中创建数据库文件 。

示例5-2在SQL语句中使用Oracle ASM文件名

CREATE TABLESPACE mytblspace DATAFILE'+ data'SIZE 200M AUTOEXTEND ON;

为Oracle ASM文件名添加别名

您可以使用ALTER DISKGROUP语句的DROP FILE子句从磁盘组中删除Oracle ASM文件及其关联的别名。指定要删除的文件时，必须使用标准文件名或别名。删除别名时，不会删除文件系统上的基础文件。

您可能想要删除文件的一些原因包括：

  • 使用别名创建的文件不是Oracle托管文件。因此，它不会自动删除。

  • 数据库的时间点恢复可能会将数据库还原到创建表空间之前的时间。还原不会删除表空间，但是在还原的数据库中没有对表空间或其数据文件的引用。您可以手动删除数据文件。

示例5-9使用系统生成的名称从磁盘组中删除文件和别名

ALTER DISKGROUP数据DROP FILE'+ data / orcl / datafile / mytable.342.123456789';

管理磁盘组目录

Oracle ASM磁盘组包含系统生成的分层目录结构，用于存储Oracle ASM文件。Oracle ASM分配给每个文件的系统生成的文件名表示此目录层次结构中的路径。以下是系统生成的文件名的示例：

+数据/ ORCL / CONTROLFILE / Current.256.541956473

加号表示Oracle ASM文件系统的根。data目录是data磁盘组中所有文件的父目录。orcl目录是orcl数据库中所有文件的父目录， CONTROLFILE目录包含orcl数据库的所有控制文件。

您可以在此层次结构中创建自己的目录，以存储您创建的别名。因此，除了具有用于Oracle ASM文件的用户友好别名之外，您还可以拥有指向这些名称的用户友好路径。

本节介绍如何使用ALTER DISKGROUP语句为别名创建目录结构。它还描述了如何重命名目录或删除目录。本节包含以下主题：

  • 创建目录

  • 重命名目录

  • 删除目录

创建目录

使用ALTER DISKGROUP语句的ADD DIRECTORY子句为Oracle ASM文件的别名创建分层目录结构。使用斜杠（/）或反斜杠（\）字符分隔目录路径的组件。目录路径必须以磁盘组名称开头，前面带有加号（+），后跟您选择的所有子目录名称。

在尝试在该目录中创建子目录或别名之前，父目录必须存在。必须在磁盘组级别下创建目录。

例5-10为磁盘组data创建了一个分层目录，该目录可以包含例如别名name +data/orcl/control_file1 。

假设目录+data/orcl下没有子目录，则SQL语句在例5-11中失败。

示例5-10创建新目录

ALTER DISKGROUP数据ADD DIRECTORY'+ data / orcl';

示例5-11创建新的子目录

ALTER DISKGROUP数据ADD DIRECTORY'+ data / orcl / first_dir / second_dir';

重命名目录

ALTER DISKGROUP语句的RENAME DIRECTORY子句使您可以重命名目录。系统创建的目录（包含系统生成的名称）无法重命名。无法重命名根级别（磁盘组名称）。

例5-12重命名目录。

示例5-12重命名目录

ALTER DISKGROUP数据RENAME DIRECTORY'+ data / mydir'TO'+ data / yourdir';

删除目录

您可以使用ALTER DISKGROUP语句的DROP DIRECTORY子句删除目录。您无法删除系统创建的目录。除非还指定了FORCE子句，否则不能删除包含别名的目录。

例5-13删除了一个目录及其内容。

示例5-13删除目录

管理磁盘组模板

模板用于设置在Oracle ASM磁盘组中创建的文件的属性。

创建文件时，将根据显式命名的模板或作为文件类型的默认模板的系统模板为该文件设置冗余和条带化属性。

创建磁盘组时，Oracle ASM会为该磁盘组创建一组默认模板。该集由Oracle ASM支持的每种文件类型（数据文件，控制文件，重做日志文件等）的一个模板组成。例如，名为ONLINELOG的模板为写入Oracle ASM磁盘的所有重做日志文件提供默认文件冗余和条带化属性。默认模板设置取决于磁盘组类型。普通冗余磁盘组的数据文件的默认模板设置双向镜像，而高冗余磁盘组中的相应默认模板设置三向镜像。您可以修改这些默认模板。

例如，正常冗余磁盘组的联机重做日志文件（ ONLINELOG模板）的默认冗余是MIRROR 。在示例4-1中 ，此设置意味着当重做日志文件范围的一个副本写入故障组controller1的磁盘时，文件范围的镜像副本将写入故障组controller2的磁盘。要支持普通冗余磁盘组的默认镜像，必须至少定义两个故障组。

将模板添加到磁盘组

要将模板添加到磁盘组，请使用ALTER DISKGROUP语句的ADD TEMPLATE子句。

如果表5-6中未列出新模板的名称，则它不会用作数据库文件类型的默认模板。要使用该模板，您必须在创建文件时引用其名称。

将模板添加到磁盘组时，属性是可选的。如果未指定冗余属性，则默认值为普通冗余磁盘组的MIRROR ，高冗余磁盘组的值为HIGH ，外部冗余磁盘组的UNPROTECTED 。如果未指定条带属性，则该值默认为COARSE 。如果未指定主要或镜像区域属性的值，则该值分别默认为COLD和MIRRORCOLD 。

修改磁盘组模板

ALTER DISKGROUP语句的MODIFY TEMPLATE子句使您可以修改现有系统默认或用户定义的磁盘组模板的属性规范。仅更改指定的模板属性。未指定的属性保留其当前值。修改现有模板时，只有模板创建的新文件才会反映属性更改。现有文件保持其属性。

从磁盘组中删除模板

使用ALTER DISKGROUP语句的DROP TEMPLATE子句从磁盘组中删除一个或多个模板。您只能删除用户定义的模板;你不能删除系统默认模板。
