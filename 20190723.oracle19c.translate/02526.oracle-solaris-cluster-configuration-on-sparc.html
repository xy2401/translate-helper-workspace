关于SPARC上的Oracle Solaris Cluster配置

如果要在SPARC处理器服务器上安装Oracle Grid Infrastructure，请查看以下信息。

如果使用Oracle Solaris Cluster 4.0或更高版本，请在启动Oracle Grid Infrastructure安装和Oracle RAC安装之前参阅Oracle Solaris Cluster文档库。特别是，请参阅“ 适用于Oracle Real Application Clusters的Oracle Solaris Cluster数据服务指南” 。

有关在区域群集中安装Oracle Real Application Clusters（Oracle RAC）的用例，请查看本指南附录“ 在Oracle Solaris Cluster区域群集上部署Oracle RAC”中的信息。

查看UDLM和本机群集成员身份界面的以下附加信息：

  • 对于Oracle Solaris Cluster 3.3，Oracle建议您不要使用UDLM。相反，Oracle建议您使用本机群集成员身份接口功能（本机SKGXN），如果未部署UDLM，则该功能将自动与Oracle Solaris Cluster 3.3一起安装。使用此接口不需要其他软件包。

  • Oracle RAC 12.1发行版或更高版本不支持Oracle UDLM。如果要使用UDLM从先前版本升级，则首先需要迁移到Oracle Solaris Cluster本机SKGXN，然后再进行升级。迁移到Oracle Solaris Cluster本机SKGXN的步骤记录在以下URL中：

    http://docs.oracle.com/cd/E18728_01/html/821-2852/gkcmt.html#scrolltoc

  • 使用Oracle Solaris Zones，一台物理服务器可以托管多个Oracle RAC集群，每个集群都位于一个隔离的区域集群中。根据所使用的成员资格模型，这些区域集群必须各自是自洽的。但是，由于每个区域群集都是隔离环境，因此您可以使用区域群集在一个物理系统上创建ORCLudlm和本机群集成员资格接口Oracle RAC群集的混合。
