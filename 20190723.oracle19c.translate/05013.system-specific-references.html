D系统特定参考

本附录将本指南中对系统特定信息的所有引用集中在一起。

本附录包含此部分：

  • 系统特定信息

D.1系统特定信息

特定于系统的信息在适用于您的平台的Oracle系统特定文档中进行了描述。

D.1.1标准头文件的位置

标准临* C / C ++头文件-的位置sqlca.h ， oraca.h和sqlda.h -is特定系统。对于您的其他系统，请参阅Oracle系统特定的文档。

D.1.2关于为C编译器指定包含文件的位置

使用Pro * C / C ++命令行选项INCLUDE =指定要包含的非标准文件的位置时，还应为C编译器指定相同的位置。您这样做的方式是系统特定的。请参阅“ 包含文件 ” 。

D.1.3 ANSI C支持

使用CODE =选项使Pro * C / C ++生成的C代码与系统的C编译器兼容。请参见“ 功能原型 ” 。

D.1.4结构部件对准

C编译器对齐struct组件的方式各不相同，通常取决于系统硬件。使用sqlvcp()函数确定添加到VARCHAR结构的.arr组件的填充。请参阅“ 查找VARCHAR阵列组件的长度 ”部分 。

D.1.5整数和ROWID的大小

整数数据类型的大小（以字节为单位）和ROWID数据类型的二进制外部大小取决于系统。请参阅“ INTEGER ”和“ ROWID ” 。

D.1.6字节排序

单词中的字节顺序取决于平台。请参阅“未签名 ”部分。

D.1.7关于连接到Oracle服务器

使用Oracle Net驱动程序连接到Oracle服务器涉及特定于系统的网络协议。有关详细信息，请参阅“ 与OCI版本8的接口 ”部分。

D.1.8关于XA库中的链接

您以与系统相关的方式链接到XA库中。有关详细信息，请参阅“ 链接 ”部分以及Oracle安装或用户指南。

D.1.9 Pro * C / C ++可执行文件的位置

Pro * C / C ++预编译器的位置是特定于系统的。有关详细信息，请参阅“ 预编译器命令 ”部分以及您的安装或用户指南。

D.1.10系统配置文件

每个预编译器安装都有一个系统配置文件。预编译器不附带此文件;它必须由系统管理员创建。Pro * C / C ++搜索系统配置文件的位置（目录路径）取决于系统。请参阅“ 预编译期间发生了什么？“了解更多信息。

D.1.11 INCLUDE选项语法

INCLUDE命令行选项的值的语法是特定于系统的。请参阅“ INCLUDE ” 。

D.1.12关于编译和链接

编译和链接Pro * C / C ++输出以获取可执行应用程序始终取决于系统。有关其他信息，请参阅“ 编译和链接 ”部分以及以下部分。

D.1.13用户退出

编译和链接Oracle Forms用户出口是特定于系统的。请参阅用户退出 。
