为直接NFS客户端创建oranfstab文件

Direct NFS使用配置文件oranfstab来确定可用的挂载点。

使用Direct NFS Client为要访问的每个NFS服务器创建一个具有以下属性的oranfstab文件：

  • 服务器

    NFS服务器名称。

  • 本地

    数据库主机上最多有四个路径，由IP地址或名称指定，使用在数据库主机上运行的ifconfig命令显示。

  • 路径

    最多四个到NFS服务器的网络路径，由IP地址或名称指定，如NFS服务器上的ifconfig命令所示。

  • 出口

    NFS服务器的导出路径。

  • 安装

    导出卷的相应本地安装点。

  • mnt_timeout

    指定（以秒为单位）Direct NFS Client在超时之前应等待成功安装的时间。此参数是可选的。默认超时为10分钟（600）。

  • nfs_version

    指定Direct NFS Client使用的NFS协议版本。可能的值包括NFSv3，NFSv4，NFSv4.1和pNFS。默认版本为NFSv3。如果选择NFSv4.x，则必须在oranfstab为nfs_version配置值。如果要将Direct NFS与并行NFS一起使用，请将nfs_version指定为pNFS 。

  • security_default

    指定适用于服务器条目的所有导出的NFS服务器路径的默认安全模式。此参数是可选的。 sys是默认值。请参阅security参数支持的安全级别的security_default参数说明。

  • 安全

    指定安全级别，以使用带有Direct NFS Client的Kerberos身份验证协议启用安全性。可以根据导出安装对指定此可选参数。security_default和security参数支持的安全级别为：

    • sys ：UNIX级别安全性AUTH_UNIX基于用户标识符（UID）和组标识符（GID）值的身份验证。这是安全性参数的默认值。
    • krb5 ：Direct NFS使用普通Kerberos身份验证运行。服务器被认证为它声称的真实服务器。
    • krb5i ：Direct NFS运行Kerberos身份验证和NFS完整性。对服务器进行身份验证，并检查每个邮件传输的完整性。
    • krb5p ：Direct NFS运行Kerberos身份验证和NFS隐私。服务器已通过身份验证，并且所有数据都已完全加密。

    security参数（如果已指定）优先于security_default参数。如果未指定这些参数，则sys是默认身份验证。

    对于NFS服务器Kerberos安全设置，请查看相关的NFS服务器文档。对于Kerberos客户端设置，请查看相关的操作系统文档。

  • dontroute

    指定传出消息不应由操作系统路由，而是使用它们绑定的IP地址发送。

    注意：

    dontroute选项是POSIX选项，有时在同一子网中具有多个路径的Linux系统上不起作用。

  • 管理

    使Direct NFS Client能够使用管理接口进行SNMP查询。如果SNMP在NFS服务器上的单独管理接口上运行，则可以使用此参数。默认值是服务器参数值。

  • 社区

    指定用于SNMP查询的社区字符串。默认值为public 。

以下示例显示了oranfstab三个可能的NFS服务器条目。单个oranfstab可以有多个NFS服务器条目。

示例8-1使用本地和路径NFS服务器条目

以下示例使用local和path。因为它们位于不同的子网中，所以您不必指定dontroute 。 

server：MyDataServer1 local：192.0.2.0 path：192.0.2.1 local：192.0.100.0 path：192.0.100.1 export：/ vol / oradata1 mount：/ mnt / oradata1

示例8-2使用dontroute在同一子网中使用本地和路径

同一子网中的本地和路径，其中指定了dontroute ：

server：MyDataServer2 local：192.0.2.0 path：192.0.2.128 local：192.0.2.1 path：192.0.2.129 dontroute export：/ vol / oradata2 mount：/ mnt / oradata2

示例8-3使用名称代替IP地址，具有多个导出，管理和社区

server：MyDataServer3 local：LocalPath1路径：NfsPath1 local：LocalPath2路径：NfsPath2 local：LocalPath3路径：NfsPath3 local：LocalPath4路径：NfsPath4 dontroute export：/ vol / oradata3 mount：/ mnt / oradata3 export：/ vol / oradata4 mount：/ mnt / oradata4 export：/ vol / oradata5 mount：/ mnt / oradata5 export：/ vol / oradata6 mount：/ mnt / oradata6 management：MgmtPath1 community：private

示例8-4使用直接NFS导出的Kerberos身份验证

security参数会覆盖security_default ：

server：nfsserver local：192.0.2.0 path：192.0.2.2 local：192.0.2.3 path：192.0.2.4 export：/ private / oracle1 / logs mount：/ logs security：krb5 export：/ private / oracle1 / data mount：/ data安全性：krb5p export：/ private / oracle1 / archive mount：/ archive security：sys export：/ private / oracle1 / data1 mount：/ data1 security_default：krb5i
