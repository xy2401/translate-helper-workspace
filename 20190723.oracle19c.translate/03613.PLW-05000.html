PLW-05000：规格和正文之间的NOCOPY资格不匹配

原因：在子程序规范中使用NOCOPY提示声明了指示的参数，但未在正文中声明，或者在体内使用NOCOPY提示声明了参数，但未在规范中声明。

行动：确保规范和机构在使用NOCOPY方面达成一致。如果不纠正这种不匹配，程序将遵守规范中给出的声明，忽略在体内的声明。

PLW-05001：以前使用' string '（在行string ）与此用法冲突

原因：在查找变量，常量或标签的先前声明时，编译器在同一范围内找到了具有相同名称的另一个声明。或者，包规范和正文中子程序的标题不相同。

操作：检查变量，常量或标签名称的拼写。还要检查在同一范围内声明的所有参数和异常的名称。然后，使用重复的名称删除或重命名该对象。或者，更改包规范或正文中打包的子程序的标题，使它们完全相同。

行动：确保规范和机构在使用NOCOPY方面达成一致。如果不纠正这种不匹配，程序将遵守规范中给出的声明，忽略在体内的声明。

PLW-05004：标识符string也在STANDARD中声明或是SQL内置的

原因：指示的标识符也是：o）在包STANDARD中声明，或o）SQL内置函数，或o）伪列。这种情况可能导致名称解析问题，因为将在SQL语句中的本地声明中选择STANDARD / builtin声明;但是，本地声明将在SQL范围之外的STANDARD / builtin声明中选择。（请参阅有关PL / SQL名称解析的文档。）

操作： o）重命名给定的标识符，o）使用包含范围的名称限定对标识符的任何引用，或o）如果在SQL范围中使用标识符，请确保出现预期的解决方案。

PLW-05005：子程序string在行string处返回无值

原因：终止没有返回值的函数。所有PL / SQL函数都需要带有值的RETURN语句。

操作：检查函数源代码以确保每个RETURN语句都返回一个值，并且函数不会通过执行其END语句来终止。请注意，如果函数调用内联并且被调用函数未正确终止，则此消息可能出现在函数的调用站点上。

原因：编译表函数会导致创建内部生成的类型。在创建这些类型期间出现问题，该问题不足以阻止类型的创建，但可能影响表函数的使用。有关详细信息，请参阅随附的错误消息。

操作：修复随附的错误消息中报告的问题。

操作：如果赋值设置的变量的值不会再次使用且没有要考虑的副作用（异常或调用），请删除赋值以获得更好的性能。如果已赋值以提出特定的已知异常，请使用RAISE语句替换赋值。

PLW-07207：BULK COLLECT在没有LIMIT条款的情况下使用

原因：此FETCH语句使用BULK COLLECT而没有附带的LIMIT子句。FETCH的结果集可能很大，可能导致运行时内存不足。

操作：使用LIMIT子句控制内存消耗。
