DROP TYPE

目的

对象类型使用PL / SQL定义。有关创建，更改和删除对象类型的完整信息，请参阅Oracle Database PL / SQL语言参考 。

使用DROP TYPE语句删除对象类型，varray或嵌套表类型的规范和正文。

先决条件

对象类型，varray或嵌套表类型必须位于您自己的架构中，或者您必须具有DROP ANY TYPE系统特权。

句法

drop_type :: =


插图说明drop_type.eps

语义

模式

指定包含类型的架构。如果省略schema ，则Oracle数据库假定该类型位于您自己的模式中。

TYPE_NAME

指定要删除的对象，varray或嵌套表类型的名称。您只能删除没有类型或表依赖项的类型。

如果type_name是超类型，则除非您还指定FORCE否则此语句将失败。如果指定FORCE ，则数据库将根据此超类型使所有子类型无效。

如果type_name是统计类型，则除非您还指定FORCE否则此语句将失败。如果指定FORCE ，则数据库首先解除与type_name关联的所有对象的关联，然后删除type_name 。

ASSOCIATE统计和撇清统计的统计类型的详细信息

如果type_name是已与统计信息类型关联的对象类型，则数据库首先尝试取消将type_name与统计信息类型取消关联，然后删除type_name 。但是，如果使用统计信息类型收集了统计信息，则数据库将无法将type_name与统计信息类型取消关联，并且此语句将失败。

如果type_name是indextype的实现类型，则indextype将标记为INVALID 。

如果type_name上定义了公共同义词，则数据库也将删除同义词。

除非指定FORCE ，否则只能删除作为独立模式对象且没有依赖关系的对象类型，嵌套表或varray类型。这是默认行为。

力

指定FORCE以删除类型，即使它具有依赖的数据库对象。Oracle数据库标记UNUSED所有依赖于要删除的类型的列，并且这些列变得不可访问。

注意：

Oracle建议您不要指定FORCE来删除具有依赖项的对象类型。此操作无法恢复，可能导致从属表或列中的数据无法访问。

VALIDATE

如果在删除类型时指定VALIDATE ，则Oracle数据库会在其任何超类型的可替换列中检查此类型的存储实例。如果未找到此类实例，则数据库将完成放置操作。

该子句仅对子类型有意义。Oracle建议使用此选项安全地删除没有任何显式类型或表依赖性的子类型。

例子

删除对象类型：示例

以下语句删除对象类型person_t 。有关创建此对象类型的示例，请参见Oracle Database PL / SQL语言参考 。任何依赖于person_t的列都标记为UNUSED并且无法访问。 

DROP TYPE person_t FORCE;
