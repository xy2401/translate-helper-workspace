创建Oracle ACFS文件系统和卷

要创建Oracle ACFS卷，请执行以下步骤。

  1. 从Oracle ASM主页访问“ 磁盘组”页面。

  2. 单击“ 磁盘 组”页面的“ 常规”选项卡中的“ 卷”链接。

  3. 在“ 磁盘 组”页面的“ 卷”选项卡中单击“ 创建” 。

    将显示“ 创建 ASM 卷”页面。

    输入卷名，包含卷的磁盘组名称以及卷的初始大小。

    您还可以选择卷的冗余设置以及主扩展区和镜像扩展区的区域设置。

要在磁盘组中的卷上创建Oracle ACFS文件系统，请执行以下步骤。

  1. 访问“ 磁盘 组”页面的“ 卷”选项卡。

  2. 选择磁盘组中的现有卷，然后单击“ 创建ASM群集文件系统” 。

    磁盘组的兼容性参数COMPATIBLE.ASM和COMPATIBLE.ADVM必须设置为11.2或更高。

  3. 在“ 创建 ASM 群集 文件 系统”页面上，输入用于创建文件系统的信息。您可以选择注册和安装文件系统。

    您必须输入卷名称。您可以输入可选的卷标。

    您可以选择在创建文件系统时注册并装入文件系统。从可用目录中选择一个安装点。

    输入提示时运行命令所需的主机凭据。要注册或装入文件系统，您需要root或管理员权限。

    选择目录后，单击“ 显示 命令”以使Oracle Enterprise Manager生成可在操作系统提示符下运行的命令。Linux系统上的命令示例如下：

    / sbin / mkfs -t acfs -b 4k / dev / asm / volume1-361 / sbin / acfsutil registry -f -a / dev / asm / volume1-361 / oracle / acfsmounts / acfs2 / bin / mount -t acfs -rw / dev / asm / volume1-361 / oracle / acfsmounts / acfs2

    您还可以在ASM Cluster File System选项卡上生成用于注册或装载Oracle ACFS文件系统的命令。

查看和修改Oracle ACFS卷和文件系统

要查看和修改有关Oracle ACFS文件系统的信息，请单击Oracle ASM主页中的ASM Cluster File System链接。

Oracle Enterprise Manager中的ASM Cluster File System选项卡列出了与Oracle ASM实例关联的所有Oracle ACFS。

在此页面上，您可以选择装入，卸载，删除，创建快照，查看内容，注册和取消注册所选文件系统。此外，您还可以创建文件系统，装入所有文件系统或卸载所有文件系统。

对于每个Oracle ACFS，这些列提供有关挂载点，可用性，状态，快照，已用（％），已用（GB），大小（GB），已分配空间，冗余，卷和磁盘组的信息。冗余，卷和磁盘组列位于页面的最右侧，未在图中显示。提供挂载点，快照，卷和磁盘组作为进一步信息的链接。

要查看有关特定卷的信息，请单击“ ASM 群集文件 系统”页面上“ 卷”列中的卷名，以显示“ASM 卷”页面的“ 常规”选项卡。

要查看有关文件系统的信息，请单击“ ASM 群集文件 系统”页面上“ 挂载点”列中的链接。

要查看有关Oracle ACFS的磁盘组的信息，请单击“ ASM 群集文件 系统”页面上“ 磁盘组”列中的磁盘组名称。将显示“ 磁盘 组”页面的“ 常规”选项卡。

要查看有关磁盘组中卷的信息，请单击“ 磁盘 组”页面顶部的“ 卷”选项卡。

创建，修改和查看快照

要查看和修改有关现有快照的信息或为文件系统创建快照，请执行以下步骤。

  1. 单击Oracle ASM主页上的ASM Cluster File System选项卡

  2. 单击“ 挂载 点”列中的挂载点链接。

  3. 单击“ 快照”选项卡。

（可选）您可以在“ ASM 群集文件 系统”页面的“ 快照”列中单击现有快照的数字链接以显示“ 快照”页面。

在“ 快照”页面上，您可以创建快照或搜索并显示特定快照。要搜索快照，请在搜索字段中输入带有可选通配符的名称，然后单击“ 搜索” 。

要创建快照，请执行以下步骤。

  1. 单击“ 快照”页面中的“ 创建 ”。

  2. 完成“ 创建 快照”页面上的信息。

    接受默认快照名称，或提供名称。（可选）您可以选择删除最早的快照。

  3. 完成屏幕后，可以单击“ 确定”以运行该命令，或单击“ 显示 命令”以查看生成的命令。

    例如，以下是用于创建快照的生成命令：

    / sbin / acfsutil snap创建“snapshot_20090702_142135”/ oracle / acfsmounts / acfs1

    要运行生成的命令，您需要在主机上具有适当的权限。如果尚未设置主机凭据，Oracle Enterprise Manager会提示您输入主机凭据。

    您还可以从“ 创建 快照”页面打开Telnet会话，以手动输入生成的操作系统命令。

要在快照目录向下钻取，在快照页面上，单击快照名称以显示搜索 和 列表 目录页。

在只读和读写之间转换快照

您可以使用编辑快照在只读和读写之间转换快照属性。要编辑快照，请按照下列步骤操作：

  1. 在Oracle ASM主页中，选择“ASM群集文件系统”选项卡以显示群集文件系统中的装入点。
  2. 单击要编辑快照的文件或目录。

    Enterprise Manager显示ASM Cluster File System页面。

  3. 单击“ 快照”选项卡。
  4. 选择要编辑的快照，然后单击“ 编辑” 。

    将显示“快照”页面。

  5. 更改属性。您可以选择只读或读写 。
  6. 单击确定 。

从现有快照创建子快照

使用“创建子快照”操作可以创建现有Oracle ACFS快照的时间点副本。子快照不会继承对父快照的未来更改。要创建现有快照的子快照，请按照下列步骤操作：

  1. 在Oracle ASM主页中，选择“ASM群集文件系统”选项卡以显示群集文件系统中的装入点。
  2. 单击要在其中创建子快照的文件或目录Mount Point。

    Enterprise Manager显示ASM Cluster File System页面。

  3. 单击“ 快照”选项卡。
  4. 选择要用于创建子快照的父快照，然后单击“ 创建子项” 。

    将显示“ASM群集文件系统上的创建子快照”页面。

  5. “快照名称”字段将自动填充默认的子快照名称。您可以使用您指定的名称替换名称。
  6. 将属性设置为只读或读写 。
  7. （可选）您可以通过打开删除选项打开删除最旧快照的选项。Oracle Enterprise Manager在达到最大限制之前列出剩余快照的数量。
  8. 单击“ 确定”以创建子快照。

    Oracle Enterprise Manager返回到“快照”选项卡，您可以在其中看到表中列出的父名称的新快照。

初始化Oracle ACFS安全性

Oracle ACFS安全性提供统一的安全基础结构，允许用户定义用于访问文件系统对象的安全策略和组。Oracle ACFS加密允许用户以加密格式在磁盘上存储数据。要初始化Oracle ACFS安全性，请按照下列步骤操作：

  1. 在Oracle ASM主页中，选择“ASM群集文件系统”选项卡以显示群集文件系统中的装入点。

    有关Oracle ASM主页以及如何访问该页面的信息，请参阅“ 访问Oracle自动存储管理主页 ” 。

  2. 单击列表中的装入点。

  3. 单击“ 安全/加密”以显示“安全和加密”部分。

  4. 在“ 安全 配置”部分中，可以单击“ 安全 ”下的“ 配置” ， “规则” ，“ 规则集 ”和“域 ”以显示各种选项。

  5. 如果未在群集中初始化Oracle ACFS安全性，则会在页面上显示“ 初始化 安全性”按钮。单击按钮以初始化Oracle ACFS安全性。此操作只需在群集中执行一次。在此页面上，您还可以为文件系统准备安全性。

要准备Oracle ACFS以确保安全性，请执行以下步骤：

  1. 在Oracle ASM主页中，选择“ASM群集文件系统”选项卡以显示群集文件系统中的装入点。

    有关Oracle ASM主页以及如何访问该页面的信息，请参阅“ 访问Oracle自动存储管理主页 ” 。

  2. 单击要为其准备安全性的文件系统的挂载点 。

    Enterprise Manager显示该安装点的ASM Cluster File System页面。

  3. 单击“ 安全/加密”选项卡以显示“安全性概述”部分。此页面显示当前文件系统的安全详细信息。
  4. 从“操作”菜单中，单击“ 准备安全性” 。
