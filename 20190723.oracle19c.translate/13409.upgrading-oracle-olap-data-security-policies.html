关于升级Oracle OLAP数据安全策略

从Oracle Database 12 c开始，安全策略的存储方式不同。如果现有Oracle数据库是11 g版本11.2，则删除安全角色。

在Oracle数据库12c中，甲骨文OLAP使用Oracle真正应用安全（奥拉斯）来存储OLAP数据安全策略，而不是可扩展的数据安全（XDS），它在Oracle数据库11克版本中使用。

当您升级从11版克至新的Oracle数据库版本的Oracle数据库，任何XDS数据安全策略被自动转换为奥拉斯。

注意：

11 g版 Oracle数据库实例中定义的数据安全角色不会自动转换为ORAS。在升级Oracle数据库版本11 克至当前的Oracle数据库版本中，您必须删除11克数据库中定义的任何数据的安全性角色。升级后，您可以使用新版本的Oracle Database Analytic Workspace Manager再次定义数据安全角色。

如果在不删除11g数据安全角色的情况下升级11g数据库，则在以后的Oracle数据库版本中，包含数据安全角色的任何数据安全策略都将无效。
