DROP DISKGROUP

注意：

仅当您使用Oracle ASM并且已启动Oracle ASM实例时，此SQL语句才有效。您必须从Oracle ASM实例中发出此语句，而不是从普通数据库实例发出。有关启动Oracle ASM实例的信息，请参阅“ Oracle自动存储管理管理员指南” 。

目的

DROP DISKGROUP语句允许您删除Oracle ASM磁盘组以及磁盘组中的所有文件。Oracle ASM首先确保磁盘组中没有文件打开。然后它删除磁盘组及其所有成员磁盘并清除磁盘头。

先决条件

您必须具有SYSASM系统特权，并且必须已启动Oracle ASM实例，您才能从该实例发出此语句。必须安装要删除的磁盘组。

句法

drop_diskgroup :: =


说明drop_diskgroup.eps的说明

语义

diskgroup_name

指定要删除的磁盘组的名称。

包括内容

指定INCLUDING CONTENTS以确认Oracle ASM应删除磁盘组中的所有文件。如果磁盘组包含任何文件，则必须指定此子句。

力

此子句清除属于Oracle ASM实例无法装入的磁盘组的磁盘上的标头。任何数据库实例都无法装入磁盘组。

Oracle ASM实例首先确定使用相同存储子系统的任何其他Oracle ASM实例是否正在使用该磁盘组。如果正在使用它，并且磁盘组位于同一个集群中，或者位于同一节点上，则该语句将失败。如果磁盘组位于不同的群集中，则系统会进一步检查以确定磁盘组是否由另一个群集中的任何实例装入。如果它安装在其他位置，则语句失败。但是，后一种检查并不像检查同一群集中的磁盘组那样明确。因此，请谨慎使用本节。

排除内容

指定“ EXCLUDING CONTENTS以确保仅在磁盘组为空时Oracle ASM才会删除磁盘组。这是默认值。如果磁盘组不为空，则将返回错误。
