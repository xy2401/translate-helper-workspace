SQL-02114：无效的SQL游标用法：尝试关闭CLOSEd游标

操作：当MODE = {ANSI | ANSI14}时，在尝试关闭之前验证游标是否已经CLOSEd。如果要重新关闭CLOSEd游标，请指定MODE = {ORACLE | ANSI13}。

SQL-02115：代码解释问题 - 检查COMMON_NAME用法

原因：使用PRO * FORTRAN时，如果未正确指定预编译器选项COMMON_NAME，则会发生此错误。对于其他Oracle预编译器，当预编译器无法生成一段代码时，会发生此错误。

操作：使用Pro * FORTRAN时，使用COMMON_NAME预编译两个或多个源模块时，请确保为每个模块指定不同的公用名。对于其他Oracle预编译器，如果错误仍然存在，请致电客户支持以获取帮助。

SQL-02117：无效的SQL游标用法：尝试打开OPENed游标

原因：尝试打开已使用MODE = {ANSI | ANSI14 | ANSI13}打开的游标。只有当MODE = ORACLE时才能重新打开一个打开的光标。

操作：当MODE = {ANSI | ANSI14 | ANSI13}时，在尝试打开游标之前，请确认游标尚未打开。如果要重新打开OPENed光标，请指定MODE = ORACLE。

SQL-02118：WHERE CURRENT OF操作的行无效

原因：尝试使用UPDATE或DELETE语句中的CURRENT OF子句引用不存在的行。当没有执行FETCH或FETCH返回程序无法捕获的“未找到数据”错误时，会发生这种情况。

操作：检查上一个游标操作是否成功，以及游标的当前行是否有效。可以通过两种方式检查游标操作的结果：使用WHENEVER语句进行隐式检查，或者在SQLCA中显式检查SQLCODE。

SQL-02119：无效的HSTDEF参数

原因：当使用第一个参数时，为hstdef提供了非空ptr作为调用sqlrcn的第二个参数。SQLRCN的第一个或第二个参数必须为null。

操作：将第一个或第二个参数中的空指针传递给SQLRCN。 *注意：此错误* msg *实际上从未使用过。如果发生此错误，则使用错误* number *作为sqlrcn的返回值。

SQL-02120：SQLRCN的第一个和第二个参数都为null

原因： sqlrcn的第一个和第二个参数都为null。HSTDEF必须以OCI lda（第一个参数）的形式或作为HSTDEF指针本身（第二个参数）传递到SQLRCN。

行动：通过OCI LDA或HSTDEF，但不能同时通过。*注意：此错误* msg *实际上从未使用过。如果发生此错误，则使用错误* number *作为sqlrcn的返回值。

SQL-02121：主机名无效

原因：传入SQLFCN调用的主机名未在先前对SQLRCN的调用中使用。

操作：使用相应SQLRCN调用中使用的相同标识符。*注意：此错误* msg *实际上从未使用过。如果发生此错误，则使用错误* number *作为sqlfcn的返回值。

SQL-02122：此数据库连接的OPEN或PREPARE无效

原因：尝试使用当前为另一个数据库连接打开的游标执行OPEN或PREPARE语句，因此不能用于此连接。

操作：关闭光标以使其可用于此连接，或使用不同的光标进行此连接。

SQL-02123：未找到EXEC TOOLS GET CONTEXT语句中引用的上下文

原因： EXEC TOOLS GET CONTEXT语句中给出的上下文名称从未与EXEC TOOLS SET CONTEXT语句一起存储。

操作：使用EXEC TOOLS SET CONTEXT语句保存稍后要检索的任何上下文。

SQL-02124：EXEC TOOLS语句返回的NULL值

原因： EXEC TOOLS为缺少指示符变量的主机变量返回了NULL值。仅当MODE = ANSI时才会返回此错误。当MODE = ORACLE时，虽然主变量的值是不确定的，但不会产生错误。

操作：将指示符变量与可能返回空值的每个主机变量相关联。

SQL-02125：连接错误，无法获取错误文本

原因：没有连接（甚至没有连接到默认主机），因此SQLLIB无法获取发生的Oracle错误的消息文本。但是，SQLLIB返回Oracle错误号，可用于查找消息。

操作：在Oracle7 Server Messages手册中查找相应的消息，并按照提供的原因和操作信息进行操作。

SQL-02126：数组元素的数量不能为负数（即<0）

原因：预编译器在SQLDA的N或F元素中找到了负数的数组元素（SQL描述符区域，它与动态SQL方法4一起使用）。在执行DESCRIBE语句之前，必须将N设置为描述符数组的维度。执行DESCRIBE语句后，必须将N重置为实际的变量DESCRIBEd，该变量存储在F变量中。

操作：检查N或F变量是否设置为非负值。

SQL-02127：预编译器/ SQLLIB版本不匹配

SQL-02128：会话仍然存在，未注销

操作：在调用SQLXDH以删除主机之前，XA必须删除该主机的所有会话或设置“doit_anyway”标志。*注意：此错误* msg *实际上从未使用过。使用错误* number *，作为sqllib xa函数sqlxdh的可能返回值。

SQL-02129：提取的字节数是奇数。

SQL-02146：尝试分配现有描述符

SQL-02147：ANSI动态发生值超出范围

原因：动态GET或SET语句中的value子句小于一个或大于给定描述符分配的MAX。

操作：为VALUE子句提供一个数字，介于1和分配描述符时提供的MAX次数之间。

SQL-02148：ANSI动态计数值超出范围

原因：动态GET或SET语句中的计数小于或大于给定描述符分配的MAX。

操作：为分配描述符时提供的1和MAX次数之间的COUNT子句提供一个数字。

SQL-02149：ANSI动态不足的项目描述符区域

原因：所描述的SQL语句包含的输出项（或列）多于在描述符中分配的输出项。

操作：在分配描述符时增加MAX出现次数。

SQL-02150：在ANSI动态SET语句中遇到零长度

原因： ANSI动态语句中遇到无效的零长度。

操作：将长度设置为非零值

SQL-02151：精确数字SET语句项的无效主机变量

原因：使用具有无效类型或长度的主机变量来获取或设置ANSI动态语句中的精确数字项。

操作：确保您的主机变量为C或C或者COBOL，S9（4）或S9（9）二进制/补偿/显示符号分开的主机变量。

SQL-02152：用于数组GET / SET语句的无效数值变量

原因：对于C或S9（4）或S9（9）二进制或comp，类型不是int或short的主机变量用于精确数字，且数组大小> 1。

操作：当提供与for子句一起使用的精确数字以指示数组时，仅对C或S9（4）或S（9）二进制使用int或short或对COBOL使用comp。具体而言，在此上下文中不允许显示标记前导分离。

SQL-02153：用于GET / SET语句的无效字符变量

原因：在ANSI动态GET或SET描述符语句中，找到类型不是字符，字符串或varchar的主变量作为项目名称（DATA项目除外）。

操作： ANSI动态GET或SET描述符语句中的字符项（DATA项除外）的主机变量类型不能是LONG或LONG VARCHAR类型。将变量重新声明为字符，字符串或varchar类型

SQL-02154：ANSI动态中主机变量的数据类型无效或未设置

原因：遇到具有项目名称DATA的ANSI动态SET语句，但未使用先前设置的有效TYPE且使用了选项模式= ansi

操作：使用mode = ansi时，必须在设置DATA之前设置TYPE。发出具有正确项目TYPE的SET语句，或者对于输出发出DESCRBE来设置项目类型。或者使用mode = oracle，它允许类型默认为bind / define主变量的类型

SQL-02155：日期时间间隔仅对Dynamic = ANSI有效

操作：使用预编译器选项type_code = ansi访问日期时间间隔代码或日期时间间隔精度

SQL-02156：指标或返回长度的类型无效

原因：与项目指示符或返回长度相关联的主变量必须为short或s9（4）comp。如果值语义和标量，它们也可以进入显示标志，引导单独的主变量

操作：使用签名的short指示器，ref指示器和返回的长度，ref返回长度为C的主机变量。对于COBOL，使用s9（4）comp。对于仅使用非数组的GET，您可以使用显示标志引导单独的主机变量

SQL-02157：源或目标lob无效

操作：确保追加调用中使用的lobs为非null。

SQL-02159：超出了允许的最大描述符数
