9.139 V $ XSTREAM_CAPTURE

V$XSTREAM_CAPTURE显示有关将LCR发送到XStream出站服务器的每个捕获过程的信息。

柱 数据类型 描述

SID

数

捕获过程的会话标识符

SERIAL＃

数

捕获进程会话的会话序列号

捕获＃

数

捕获进程号。捕获过程是以cp为前缀的Oracle后台进程

CAPTURE_NAME

VARCHAR2（128）

捕获过程的名称

LOGMINER_ID

数

与捕获过程关联的Oracle LogMiner会话的会话ID

启动时间

日期

上次捕获过程的开始时间

州

VARCHAR2（551）

捕获过程的状态：

  • INITIALIZING - 启动。

  • WAITING FOR DICTIONARY REDO -等待包含与第一SCN的字典建立重做日志文件被添加到捕获过程的会话。在添加了包含字典构建的所有日志文件之前，捕获进程无法开始扫描重做日志文件。

  • DICTIONARY INITIALIZATION - 处理字典构建。

  • MINING （ PROCESSED SCN = scn_value ） - 在SCN scn_value挖掘字典构建。

  • LOADING （ Y步骤X ） - 处理来自字典构建的信息，当前在步骤X处理涉及Y步骤的过程中的信息，其中X和Y是数字。

  • CAPTURING CHANGES -扫描重做日志，以满足捕获流程规则集的改变。

  • WAITING FOR REDO -等待新的重做日志文件被添加到拍摄过程中的会话。捕获过程已完成处理添加到其会话的所有重做日志文件。如果源数据库中没有活动，则可以使用此状态。对于下游捕获进程，如果捕获进程正在等待将新日志文件添加到其会话中，则可以使用此状态。

  • EVALUATING RULE - 根据捕获流程规则集评估更改。

  • CREATING LCR - 将更改转换为LCR。

  • ENQUEUING MESSAGE - 将满足捕获过程规则集的LCR排入捕获进程队列。

  • FOR FLOW CONTROL PAUSED - 由于内存不足或者传播和出站服务器消耗的消息比捕获进程创建消息慢，因此无法将LCR排入队列。此状态表示流量控制，用于在传播或应用落后或不可用时减少捕获的LCR的溢出。

  • WAITING FOR THE BUFFERED QUEUE TO SHRINK -等待缓冲队列更改为更小的尺寸。当存在内存限制或管理员减小其大小时，缓冲队列会收缩。

  • WAITING FOR n SUBSCRIBER （ S ） INITIALIZING -等待接收的LCR从捕捉进程来启动，其中出站服务器n是应用进程的数量。

  • WAITING FOR TRANSACTION -等待的LogMiner提供更多的交易。

  • WAITING FOR INACTIVE DEQUEUERS -等待捕获进程的队列中的用户启动。如果队列中没有活动订户，则捕获进程将停止对LCR进行排队。

  • SUSPENDED FOR AUTO SPLIT / MERGE -等待合并操作来完成。

  • SHUTTING DOWN -停止。

  • ABORTING - 中止。

TOTAL_PREFILTER_EVALUATIONS

预滤器评估总数

数

自上次启动捕获过程以来，LogMiner传递到捕获过程的重做条目总数，以进行详细的规则评估。捕获过程将重做条目转换为消息，并在捕获过程预过滤无法丢弃更改时对消息执行详细的规则评估。

CAPTURE_MESSAGE_CREATE_TIME

日期

最近捕获的消息的创建时间

TOTAL_MESSAGES_CREATED

数

与ELAPSED_LCR_TIME关联的计数以计算费率

TOTAL_FULL_EVALUATIONS

数

与ELAPSED_RULE_TIME关联的计数以计算费率

TOTAL_MESSAGES_ENQUEUED

数

自上次启动捕获过程以来排队的消息总数

数

ENQUEUE_MESSAGE_CREATE_TIME

最后排队消息的创建时间

AVAILABLE_MESSAGE_NUMBER

数

对于本地捕获，最后一个重做SCN刷新到日志文件。对于下游捕获，最后一个SCN通过归档重做日志文件添加到LogMiner。

AVAILABLE_MESSAGE_CREATE_TIME

日期

对于本地捕获，将SCN写入日志文件的时间。对于下游捕获，最新的归档重做日志文件（包含最新的SCN）的时间已添加到LogMiner。

ELAPSED_CAPTURE_TIME

数

自上次启动捕获过程以来，扫描重做日志中的更改的经过时间（以百分之一秒为单位）

ELAPSED_RULE_TIME

数

自上次启动捕获过程以来经过的时间（以百分之一秒为单位）评估规则

ELAPSED_ENQUEUE_TIME

数

自捕获过程最后一次启动以来经过的时间（以百分之一秒为单位）

ELAPSED_LCR_TIME

数

ELAPSED_REDO_WAIT_TIME

数

捕获过程在WAITING FOR REDO状态下花费的经过时间（以百分之一秒为单位）

ELAPSED_PAUSE_TIME

数

经过的流量控制暂停时间（以百分之一秒为单位）

STATE_CHANGED_TIME

日期

捕获过程状态发生变化的时间

SGA_USED

数

捕获进程当前使用的共享内存总量（以字节为单位）超出分配的数量（ SGA_ALLOCATED ）

SGA_ALLOCATED

数

从Streams池为捕获进程分配的共享内存总量（以字节为单位）

BYTES_OF_REDO_MINED

VARCHAR2（64）

自上次启动捕获过程以来挖掘的重做数据总量（以字节为单位）

SESSION_RESTART_SCN

VARCHAR2（64）

捕获进程从上次启动时开始挖掘重做数据的SCN

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

注意：

仅当TIMED_STATISTICS初始化参数设置为true或者STATISTICS_LEVEL初始化参数设置为TYPICAL或ALL ，才会填充ELAPSED_CAPTURE_TIME ， ELAPSED_RULE_TIME ， ELAPSED_ENQUEUE_TIME ， ELAPSED_LCR_TIME和ELAPSED_REDO_WAIT_TIME列。
