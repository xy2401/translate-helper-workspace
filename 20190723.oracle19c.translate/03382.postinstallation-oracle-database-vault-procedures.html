注册Oracle Database Vault后，您可以执行专门的任务，例如在Oracle Real Application Clusters（Oracle RAC）节点上配置它。

  • 在Oracle RAC节点上配置Oracle Database Vault
    为Oracle Real Application Clusters（Oracle RAC）实例配置Oracle Database Vault后，必须配置每个Oracle RAC节点。
  • 将语言添加到Oracle Database Vault
    默认情况下，Oracle Database Vault仅加载英语语言表。
  • 卸载Oracle Database Vault
    您可以从Oracle数据库安装中删除Oracle Database Vault，以用于单实例和Oracle RAC安装。
  • 重新安装Oracle Database Vault
    您可以使用Database Configuration Assistant重新安装Oracle Database Vault，然后注册Database Vault。

在Oracle RAC节点上配置Oracle Database Vault

为Oracle Real Application Clusters（Oracle RAC）实例配置Oracle Database Vault后，必须配置每个Oracle RAC节点。

以下过程假定您为每个节点都有一个单独的Oracle主目录。

  1. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。
    sqlplus sys as sysdba输入密码： password
  2. 在每个Oracle RAC节点上运行以下ALTER SYSTEM语句：
    ALTER SYSTEM SET AUDIT_SYS_OPERATIONS = TRUE SCOPE = SPFILE; - 对于非统一审计环境， ALTER SYSTEM SET OS_ROLES = FALSE SCOPE = SPFILE; ALTER SYSTEM SET RECYCLEBIN ='OFF'SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_LOGIN_PASSWORDFILE ='EXCLUSIVE'SCOPE = SPFILE; ALTER SYSTEM SET SQL92_SECURITY = TRUE SCOPE = SPFILE;
  3. 重启Oracle数据库。
    CONNECT / AS SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP

将语言添加到Oracle Database Vault

默认情况下，Oracle Database Vault仅加载英语语言表。

您可以通过为要添加的每种新语言运行DBMS_MACADM.ADD_NLS_DATA过程来添加更多语言。您可以向Database Vault添加多种语言。

  1. 以已被授予DV_OWNER或DV_ADMIN角色的用户身份登录数据库实例。
  2. 运行以下过程：
    EXEC DBMS_MACADM.ADD_NLS_DATA（' language '）;

    您可以使用任何案例指定language设置。例如：

    EXEC DBMS_MACADM.ADD_NLS_DATA（'french'）; EXEC DBMS_MACADM.ADD_NLS_DATA（'JAPANESE'）;

    更换language与以下支持的语言之一：

    • 英语

    • 德语

    • 西班牙

    • FRENCH

    • 意大利

    • 日本

    • KOREAN

    • 巴西葡萄牙

    • 简体中文

    • 繁体中文

卸载Oracle Database Vault

您可以从Oracle数据库安装中删除Oracle Database Vault，以用于单实例和Oracle RAC安装。

但是，您无法从多租户环境中的数据库卸载Database Vault。此过程仅适用于旧的非CDB Oracle数据库环境。

卸载过程不会影响初始化参数设置，即使是在安装过程中修改的设置，也不会影响Oracle Label Security。

  1. 以具有SYSDBA管理权限的用户SYS或具有ALTER SYSTEM系统特权的用户身份登录数据库实例。

    例如：

    sqlplus psmith - 或者，sqlplus psmith @hrpdb用于可插拔数据库（PDB）输入密码： password
  2. 确保已禁用回收站。
    SHOW PARAMETER RECYCLEBIN
  3. 如果回收站已打开，则使用以下语句之一禁用它：
    ALTER SYSTEM SET RECYCLEBIN = OFF; ALTER SESSION SET recyclebin = OFF SCOPE = SPFILE;
  4. 以已被授予DV_OWNER或DV_ADMIN角色的用户身份进行连接。

    例如：

    CONNECT leo_dvowner - 或者，CONNECT leo_dvowner @hrpdb输入密码： 密码
  5. 运行以下过程以禁用Oracle Database Vault：
    EXEC DBMS_MACADM.DISABLE_DV;
  6. 以SYS身份连接SYSOPER权限，然后重新启动数据库。

    例如：

    将SYS作为SYSOPER连接- 或者，连接SYS @hrpdb AS SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP

    对于Oracle RAC安装，请关闭然后重新启动每个数据库实例，如下所示：

    $ srvctl stop database -db db_name $ srvctl start database -db db_name
  7. 运行dvremov.sql脚本以删除Oracle Database Vault。

    例如：

    $ ORACLE_HOME / RDBMS /管理/ dvremov.sql

之后，您可以通过登录SQL * Plus并输入以下语句来仔细检查Oracle Database Vault是否已真正卸载：

SELECT * FROM V $ OPTION WHERE PARAMETER ='Oracle Database Vault';

如果卸载Oracle Database Vault，则会显示以下输出：

PARAMETER VALUE ----------------------------- ------------------- ---- Oracle Database Vault FALSE

重新安装Oracle Database Vault

您可以使用Database Configuration Assistant重新安装Oracle Database Vault，然后注册Database Vault。

  1. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。
    sqlplus sys as sysdba - 或者，sqlplus sys @hrpdb as sysdba输入密码： password
  2. 启动数据库配置助手（DBCA）。

    • UNIX：在shell窗口中输入以下命令：

      DBCA

    • Windows：使用以下方法之一在Windows上启动DBCA：

      • 单击开始 ，选择程序 （或所有程序 ），然后选择Oracle - HOME_NAME ，然后选择配置和迁移工具 ，再选择数据库配置助手 。

      • 在命令提示符处输入以下命令：

        DBCA
  3. 使用DBCA为新数据库或现有数据库配置Database Vault。
  4. 注册Oracle Database Vault。
