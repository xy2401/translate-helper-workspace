创造JAVA

目的

使用CREATE JAVA语句创建包含Java源，类或资源的模式对象。

也可以看看：

  • Oracle数据库Java开发人员指南 Java概念和有关Java存储过程的信息

  • 有关JDBC的信息，请参见“Oracle数据库JDBC开发人员指南”

先决条件

要在您自己的模式中创建或替换包含Java源，类或资源的模式对象，您必须具有CREATE PROCEDURE系统特权。要在另一个用户的模式中创建或替换此类模式对象，您必须具有CREATE ANY PROCEDURE系统特权。

语义

或替换

指定OR REPLACE以重新创建包含Java类，源或资源的模式对象（如果已存在）。使用此子句可以更改现有对象的定义，而无需删除，重新创建和重新授予先前授予的对象权限。

如果重新定义Java模式对象并指定RESOLVE或COMPILE ，则Oracle数据库将重新编译或解析该对象。无论分辨率或编译是否成功，数据库都会使引用Java模式对象的类无效。

以前被授予重新定义的功能权限的用户仍然可以访问该功能而无需重新授予权限。

解析| COMPILE

RESOLVE和COMPILE是同义关键字。它们指定Oracle数据库应尝试解析在此语句成功时创建的Java模式对象。

  • 应用于类时，会发生对其他类模式对象的引用名称的解析。

  • 应用于源时，会发生源编译。

对RESOLVE和COMPILE的限制

您不能为Java资源指定这些关键字。

NOFORCE

如果已指定RESOLVE或COMPILE并且分辨率或编译失败，请指定NOFORCE以回滚此CREATE命令的结果。如果未指定此选项，则如果解析或编译失败，则Oracle数据库不会执行任何操作，并且仍会保留创建的架构对象。

JAVA SOURCE条款

指定JAVA SOURCE以加载Java源文件。

JAVA CLASS条款

指定JAVA CLASS以加载Java类文件。

JAVA RESOURCE子句

指定JAVA RESOURCE以加载Java资源文件。

被命名的条款

Java源或资源需要NAMED子句。primary_name必须用双引号括起来，并且其长度在数据库字符集中不得超过4000个字节。

  • 对于Java源，此子句指定保存源代码的架构对象的名称。成功的CREATE JAVA SOURCE语句还将创建其他模式对象来保存源定义的每个Java类。

  • 对于Java资源，此子句指定用于保存Java资源的模式对象的名称。

使用双引号保留较低或混合大小写的primary_name 。

如果未指定schema ，则Oracle数据库会在您自己的模式中创建对象。

NAMED Java类的限制

NAMED条款受以下限制：

  • 您不能为Java类指定NAMED 。

  • primary_name不能包含数据库链接。

SCHEMA条款

SCHEMA子句仅适用于Java类。此可选子句指定包含Java文件的对象所在的模式。如果未指定此子句，则Oracle Database会在您自己的架构中创建对象。

共享

仅当在应用程序根目录中创建Java模式对象时，此子句才适用。此类对象称为应用程序公共对象，可以与属于应用程序根目录的应用程序PDB共享。要确定如何共享Java架构对象，请指定以下共享属性之一：

  • METADATA - 元数据链接共享Java模式对象的元数据，但其数据对每个容器都是唯一的。这种类型的Java模式对象称为元数据链接的应用程序公共对象 。

  • NONE - 不共享Java架构对象。

如果省略此子句，则数据库使用DEFAULT_SHARING初始化参数的值来确定Java模式对象的共享属性。如果DEFAULT_SHARING初始化参数没有值，则默认值为METADATA 。

创建后，您无法更改Java模式对象的共享属性。

invoker_rights_clause

使用invoker_rights_clause来指示类的方法是否使用特权以及拥有该类的用户的模式或具有特权以及CURRENT_USER模式的模式执行。

此子句还确定Oracle数据库如何在类型的成员函数和过程中解析查询，DML操作和动态SQL语句中的外部名称。

AUTHID CURRENT_USER

CURRENT_USER表示类的方法以CURRENT_USER的权限执行。此子句是默认值并创建一个调用者权限类 。

此子句还指定查询，DML操作和动态SQL语句中的外部名称在CURRENT_USER的模式中解析。所有其他语句中的外部名称在方法所在的模式中解析。

AUTHID DEFINER

DEFINER表示类的方法使用类所在的模式的所有者的特权执行，并且外部名称在类所在的模式中解析。该子句创建一个definer-rights类 。

RESOLVER条款

RESOLVER子句允许您指定完全限定的Java名称到Java模式对象的映射，其中：

  • match_string是完全限定的Java名称，可以匹配此类Java名称的通配符，或者可以匹配任何名称的通配符。

  • schema_name指定要搜索相应Java模式对象的模式。

  • 短划线（ - ）作为schema_name的替代指示，如果match_string匹配有效的Java名称，则Oracle数据库可以保留未解析的名称。分辨率成功，但该类不能在运行时使用该名称。

此映射与此命令中创建的模式对象的定义一起存储，以便在以后的分辨率中使用（隐式或显式ALTER JAVA ...RESOLVE声明）。

使用条款

USING子句确定Java类或资源的字符数据序列（ CLOB或BFILE ）或二进制数据（ BLOB或BFILE ）。Oracle数据库使用字符序列为Java类或资源定义一个文件，或为Java源定义一个源文件和一个或多个派生类。

BFILE条款

指定操作系统上的先前创建的文件（ directory_object_name ）和包含该序列的服务器文件（ server_file_name ）的目录和文件名。BFILE通常由CREATE JAVA SOURCE解释为字符序列，并由CREATE JAVA CLASS或CREATE JAVA RESOURCE解释为二进制序列。

CLOB | BLOB | BFILE 子查询

指定一个子查询，用于选择指定类型的单个行和列（ CLOB ， BLOB或BFILE ）。列的值构成字符序列。

注意：

在早期版本中， USING子句隐式提供了关键字SELECT 。这已不再是这种情况。但是，仍支持不带关键字SELECT的子查询以实现向后兼容性。

key_for_BLOB

key_for_BLOB子句提供以下隐式查询：

SELECT LOB FROM CREATE $ JAVA $ LOB $ TABLE WHERE NAME ='key_for_BLOB';

对 key_for_BLOB 子句的 限制

为了使用这种情况，表CREATE$JAVA$LOB$TABLE必须存在于当前模式中，并且必须具有BLOB类型的列LOB和类型为VARCHAR2的列NAME 。

AS source_char

指定Java源的字符序列。
