1.73 DB_CACHE_SIZE

DB_CACHE_SIZE指定具有主块大小的缓冲区（由DB_BLOCK_SIZE初始化参数定义的块大小）的DEFAULT缓冲池的大小。

属性 描述

参数类型

大整数

句法

DB_CACHE_SIZE = integer [K | M | G]

默认值

如果设置了SGA_TARGET ：如果未指定参数，则默认值为0 （由Oracle数据库内部确定）。如果指定了参数，则用户指定的值指示内存池的最小值。

如果未设置SGA_TARGET ，则默认值为48 MB或4 MB * CPU数量，以较大者为准

可修改

改变系统

可在PDB中修改

是

基本

没有

该值必须至少为4M * number of cpus （较小的值会自动向上舍入到此值）。用户指定的值大于此值将向上舍入到最接近的粒度大小。值为零是非法的，因为主块大小的DEFAULT内存池需要它，这是SYSTEM表空间的块大小。

注意：

对于可插拔数据库（PDB），此参数是可选的。为PDB设置此参数时，它指定PDB的保证缓冲区高速缓存大小。

为了能够在CDB中使用资源管理器来控制每个PDB可以使用的内存量：

  • 必须在CDB级别（在CDB的根目录中）将NONCDB_COMPATIBLE初始化参数设置为FALSE 。

  • 不得在CDB级别设置MEMORY_TARGET初始化参数。

  • 如果未设置SGA_TARGET初始化参数，但DB_CACHE_SIZE初始化参数设置为CDB级别，则必须满足以下要求：

    • 在PDB中设置的DB_CACHE_SIZE的值必须小于或等于CDB级别的DB_CACHE_SIZE值的50％。

    • CDB中所有PDB上的DB_CACHE_SIZE值之和必须小于或等于CDB级别DB_CACHE_SIZE值的50％。

    将PDB中的DB_CACHE_SIZE设置为不满足这些要求的值时，会收到错误。如果在设置PDB参数后违反了这些要求（例如，如果在CDB级别更改了DB_CACHE_SIZE值，Oracle将调整PDB的值以满足这些要求。

如果SGA_TARGET设置为CDB级别，则必须满足以下要求：

  • PDB中DB_CACHE_SIZE加SHARED_POOL_SIZE的值必须小于或等于PDB的SGA_TARGET值的50％。

  • PDB中DB_CACHE_SIZE加SHARED_POOL_SIZE的值必须小于或等于CDB级别的SGA_TARGET值的50％。

  • CDB中所有PDB上的DB_CACHE_SIZE加SHARED_POOL_SIZE的总和必须小于或等于CDB级别的SGA_TARGET值的50％。

如果不满足上述三项要求中的任何一项，您将收到错误消息。
