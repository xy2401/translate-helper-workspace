1 Oracle Database的WebSphere MQ简介

以下主题概述了消息队列，WebSphere MQ以及访问WebSphere MQ队列时网关的角色。

消息队列简介

消息队列使分布式应用程序能够通过在应用程序之间发送消息来异步通信。

来自发送应用程序的消息存储在队列中，并由接收应用程序检索。应用程序通过向消息排队系统发送请求来通过队列发送或接收消息。发送和接收应用程序可以使用相同或不同的消息排队系统，允许消息排队系统处理从发送方队列到接收方队列的消息转发。

排队的消息可以存储在中间节点，直到系统准备好将它们转发到下一个节点。在目标节点处，消息存储在队列中，直到接收应用程序从队列中检索它们。即使网络或应用程序出现故障，也可以保证邮件传递。这为应用程序之间提供了可靠的通信通道。

底层模型（在不同环境之间存储和转发消息）的复杂性和细节由消息排队系统处理。通过保持这种抽象级别，可以开发分布式应用程序，而无需担心信息传输的细节。

因为发送和接收应用程序彼此独立地操作，所以发送应用程序较少依赖于远程应用程序的可用性，它们之间的网络以及运行接收应用程序的系统。这可以为参与的应用程序提供更高级别的可用性。

应用程序可以配置消息和消息队列操作以在特定模式下操作。例如，发送应用程序可以指定排队的消息应该在系统崩溃后继续存在。作为另一示例，接收应用可以指定来自队列的接收操作的最大等待时段（在接收队列上还没有消息可用的情况下）。

WebSphere MQ简介

WebSphere MQ是基于消息队列客户端和消息队列服务器模型的消息队列系统。

应用程序在队列管理器和队列所在的服务器节点上运行，或者在远程客户机节点上运行。应用程序只能从与其连接的队列管理器所拥有的队列发送或检索消息。

WebSphere MQ术语

此表描述了本指南中使用的WebSphere MQ术语。

表1-1 WebSphere MQ术语

术语 描述

信息队列

应用程序之间交换消息的存储区域。

消息队列接口（MQI）

一个应用程序编程接口（API），用于希望通过WebSphere MQ队列发送或接收消息的应用程序。

WebSphere MQ客户端配置

WebSphere MQ配置，其中队列管理器和消息队列位于与应用程序软件不同的（远程）系统或节点上。客户端应用程序使用IBM软件连接到远程队列管理器，该软件提供连接到远程队列管理器所需的网络软件。

WebSphere MQ服务器配置

WebSphere MQ配置，其中队列管理器和消息队列与应用程序软件位于同一（本地）系统或节点上。客户端应用程序使用MQI连接到本地队列管理器。

队列经理

WebSphere MQ功能，提供应用程序使用的消息队列工具。它管理队列定义，配置表和消息队列。队列管理器还将消息从发送方队列转发到远程接收方队列。

触发器

WebSphere MQ功能，可以在发生消息事件（例如消息到达）时自动启动应用程序。触发器可用于调用程序或事务。例如，触发器可能导致Oracle应用程序调用网关以检索WebSphere MQ消息并对其进行处理。

网关简介

Oracle Database Gateway for WebSphere MQ使Oracle应用程序能够与其他WebSphere MQ应用程序集成。

Oracle应用程序可以将消息发送到其他WebSphere MQ应用程序或从中接收消息。通过网关，Oracle应用程序可以访问WebSphere MQ消息队列远程过程调用（RPC）处理。

网关扩展了Oracle数据库可用的RPC工具，并允许任何客户端应用程序使用PL / SQL访问WebSphere MQ队列中的消息。网关提供由网关转换为MQI调用的PL / SQL过程。这些过程类似于MQI的调用和类型，但它们适用于充分利用与Oracle数据库的事务集成。

通过WebSphere MQ，网关可与各种平台上的任何其他WebSphere MQ系统进行通信，包括大型机，基于UNIX的系统，Microsoft Windows和其他桌面环境。网关不需要远程系统上的任何Oracle软件。该网关与现有WebSphere MQ应用程序集成，无需对这些应用程序进行任何更改，并使用户能够利用他们在这些应用程序中的投资，同时使他们能够最大限度地利用面向消息的系统的优势。

该网关还提供了一种将这些现有WebSphere MQ应用程序与新技术领域（如网络计算）集成的方法。任何Oracle应用程序可以调用PL / SQL程序，包括使用Oracle应用Server 11的3G应用。

开发网关应用

使用Oracle Visual Workbench for Oracle Database Gateway for WebSphere MQ。

如果您正在开发通过网关访问WebSphere MQ的应用程序 ，使用Oracle Visual Workbench for Oracle Database Gateway for WebSphere MQ 。 Oracle Visual Workbench使您能够定义用于访问WebSphere MQ的接口，并定义如何转换从WebSphere MQ队列发送或检索的消息数据。

Visual Workbench为接口和数据转换生成PL / SQL代码。生成的代码称为消息接口包 （MIP）。MIP提供与网关交互的底层代码，执行消息数据转换，并为Oracle应用程序提供易于使用的界面，以便与远程WebSphere MQ应用程序交换消息。

使用网关的优点

这是使用网关访问WebSphere MQ的优点的描述。

使用网关访问WebSphere MQ具有以下优势：

  • 交易支持

    网关和Oracle数据库使WebSphere MQ操作和Oracle数据库更新能够以协调的方式执行。Oracle两阶段提交保护扩展到WebSphere MQ环境，无需任何特殊编程。

  • 快速远程程序

    网关实现的远程过程经过优化，可有效处理WebSphere MQ请求。

    网关和WebSphere MQ的远程过程是在网关中预编译的优化PL / SQL包。由于目标系统上没有其他软件层，因此可以最大限度地减少开销。

  • 位置透明度

    客户端应用程序不必位于特定的操作系统上。例如，您的Oracle应用程序可以将WebSphere MQ消息发送到IBM MVS上的应用程序。如果将接收应用程序移动到其他平台，则无需更改Oracle应用程序的平台。

  • 灵活的界面

    使用Visual Workbench生成的MIP，您可以使用网关与现有的过程逻辑进行交互，或将新的过程逻辑集成到Oracle数据库环境中。

  • Oracle数据库集成

    通过将Oracle数据库与网关集成，您可以从现有和未来的Oracle功能中受益。

  • 广泛的工具选择

    网关支持任何支持PL / SQL的工具或应用程序。这包括使用传统Oracle工具构建的应用程序，例如 Oracle Developer，或为Oracle Application Server 11g支持的Intranet或Internet环境构建的应用程序。该网关还适用于打包的Oracle应用程序，例如 Oracle Financials以及许多第三方工具，如Visual Basic，PowerBuilder和Lotus Notes。

  • 安全

    网关与WebSphere MQ安全授权机制兼容。

组件描述

本主题描述了网关体系结构的组件。

Oracle Applications

Oracle应用程序组件。

Oracle应用程序连接到Oracle数据库。它们通过调用网关RPC将数据发送到WebSphere MQ队列并从WebSphere MQ队列接收数据。

Oracle数据库

Oracle数据库组件。

Oracle应用程序不直接连接到网关，而是通过Oracle数据库间接连接。Oracle数据库使用Oracle Net以正常的Oracle服务器到服务器方式与网关通信。网关是单个进程，不启动后台进程。在UNIX平台上，为每个用户会话启动网关进程。

Oracle Net

Oracle Net组件。

Oracle Net为客户端提供服务器和服务器到网关的通信。它使Oracle应用程序能够与Oracle数据库通信，并使Oracle数据库能够与网关通信。

如果Oracle数据库与网关不在同一系统上，则必须在安装了Oracle数据库的平台上安装正确的Oracle网络软件。

网关

网关组件。

Oracle应用程序使用PL / SQL调用网关实现的RPC。网关过程将这些RPC映射到WebSphere MQ MQI调用以执行相应的WebSphere MQ操作。

通过使用Oracle CREATE DATABASE LINK语句创建的数据库链接名称，可以通过Oracle数据库访问网关。数据库链接是用于标识Oracle数据库的构造。

WebSphere MQ队列管理器

WebSphere MQ队列管理器组件。

WebSphere MQ服务器是WebSphere MQ队列管理器和消息队列所在的位置。WebSphere MQ服务器可能（也可能不）与网关位于同一系统上。

WebSphere MQ应用程序

WebSphere MQ应用程序组件。

WebSphere MQ应用程序通过使用WebSphere MQ MQI调用直接连接到WebSphere MQ队列管理器，以执行相应的WebSphere MQ操作。

网关结构

网关具有与Oracle数据库相同的一些组件。

包括以下组件：

  • 安装网关软件的目录

  • 系统标识符（SID）

  • 一个初始化文件类似于Oracle数据库初始化参数文件

网关没有控制，重做或数据库文件，也没有与Oracle数据库关联的完整子目录和其他文件集。

网关操作

访问网关的每个Oracle数据库用户会话都在运行网关的主机系统上创建独立进程。

网关的启动方式与Oracle数据库的启动方式不同。它没有后台进程，也不需要Oracle Enterprise Manager等管理实用程序。访问网关的每个Oracle数据库用户会话都在运行网关的主机系统上创建独立进程。
