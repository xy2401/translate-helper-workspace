3.20启动

目的

使用STARTUP命令从RMAN环境中启动目标数据库。此命令等同于使用SQL * Plus STARTUP命令。

此外，即使没有服务器参数文件或初始化参数文件，RMAN STARTUP命令也可以在NOMOUNT模式下启动实例。必须还原丢失的服务器参数文件时，此功能非常有用。

也可以看看：

Oracle数据库管理员指南 ，了解如何启动和关闭数据库，以及SQL * Plus用户指南和 SQL * Plus STARTUP语法参考

先决条件

RMAN必须连接到目标数据库。您只能使用此命令启动目标数据库。

使用说明

即使没有服务器参数文件或初始化参数文件，RMAN STARTUP命令也可以在NOMOUNT模式下启动实例。必须还原丢失的服务器参数文件时，此功能非常有用（请参阅示例3-73 ）。

句法

startup :: =


说明startup.eps的说明

语义

语法元素 描述

启动

如果仅指定STARTUP而没有其他选项，则实例将使用默认服务器参数文件启动实例，安装控制文件并打开数据库。

DBA

限制用户访问用户 RESTRICTED SESSION权限。

力

如果数据库已打开，则FORCE在重新打开数据库之前使用SHUTDOWN ABORT语句关闭数据库。如果数据库已关闭，则FORCE将打开数据库。

安装

启动实例，然后安装数据库而不打开它

NOMOUNT

在不挂载数据库的情况下启动实例。如果不存在参数文件，则RMAN使用临时参数文件启动实例。然后，您可以运行RESTORE SPFILE来还原备份服务器参数文件。

PFILE filename

指定目标数据库的基于文本的初始化参数文件的文件名。如果未指定PFILE ，则使用默认初始化参数文件名。

例子

示例3-72指定参数文件时安装数据库

此示例强制SHUTDOWN ABORT ，然后使用受限访问权限安装数据库，并指定非默认初始化参数文件位置：

CONNECT TARGET / STARTUP FORCE MOUNT DBA PFILE = / tmp / initPROD.ora;

示例3-73在没有参数文件的情况下启动实例

假设从文件系统中意外删除了服务器参数文件。以下示例在不使用参数文件的情况下启动实例，然后运行RESTORE SPFILE FROM AUTOBACKUP 。在此示例中，自动备份位置是快速恢复区域，因此不需要SET DBID 。 

连接目标/启动力量NOMOUNT; #RMAN使用虚拟参数文件RESTORE SPFILE TO'启动实例？/dbs/spfileprod.ora'FROM AUTOBACKUP RECOVERY AREA'/ disk2'DB_NAME ='prod';启动力量; #recovery instance with restored server parameter file
