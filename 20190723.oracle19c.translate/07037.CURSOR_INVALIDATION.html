1.59 CURSOR_INVALIDATION

CURSOR_INVALIDATION控制默认情况下是否将延迟游标失效或立即游标失效用于DDL语句。

属性 描述

参数类型

串

句法

CURSOR_INVALIDATION = {DEFERRED | IMMEDIATE}

默认值

即时

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

真正

基本

没有

Oracle RAC

不同的实例可以具有不同的值。

延迟失效减少了游标失效的数量，并随着时间推移重新编译工作负载。请注意，当重新编译工作负载随时间推移时，游标可能会以次优计划运行，直到重新编译为止，并且可能会产生较小的执行时间开销。

在Oracle Database 12 c Release 2（12.2.0.1）之前，使用了立即游标失效。

此参数为DDL语句中的INVALIDATION子句的DEFERRED或IMMEDIATE选项提供系统或会话级别缺省值。

当此参数设置为DEFERRED ，应用程序可以利用减少的游标失效而不进行任何其他应用程序更改。

当此参数设置为IMMEDIATE ，应用程序将遇到与Oracle Database 12 c第1版（12.1）中相同的游标失效行为。
