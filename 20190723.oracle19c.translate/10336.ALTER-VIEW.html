更改视图

目的

使用ALTER VIEW语句显式重新编译无效的视图或修改视图约束。显式重新编译可让您在运行时之前找到重新编译错误。您可能希望在更改其一个基表后显式重新编译视图，以确保更改不会影响视图或依赖于它的其他对象。

您还可以使用ALTER VIEW来定义，修改或删除视图约束。

您不能使用此语句来更改现有视图的定义。此外，如果DDL更改为视图的基表使视图无效，则无法使用此语句编译无效视图。在这些情况下，您必须使用带有OR REPLACE关键字的CREATE VIEW重新定义视图。

发出ALTER VIEW语句时，Oracle数据库会重新编译视图，无论视图是有效还是无效。数据库还会使依赖于视图的任何本地对象无效。

如果更改由一个或多个实例化视图引用的视图，则这些实例化视图将失效。查询重写无法使用无效的实例化视图，因此无法刷新。

先决条件

视图必须位于您自己的架构中，或者您必须具有ALTER ANY TABLE系统特权。

句法

alter_view :: =


插图描述alter_view.eps

（ out_of_line_constraint :: = -part of constraint :: = syntax）

语义

模式

指定包含视图的架构。如果省略schema ，则Oracle数据库假定视图位于您自己的模式中。

视图

指定要重新编译的视图的名称。

修改约束条款

使用MODIFY CONSTRAINT子句更改现有视图约束的RELY或NORELY设置。有关视图约束的一般信息，请参阅“ 视图约束的注意事项 ” 。

修改约束的限制

如果它是参照完整性约束的一部分而不删除外键或更改其设置以匹配view则无法更改唯一键或主键约束的设置。

ADD条款

使用ADD子句添加约束以进行view 。有关视图约束及其限制的信息，请参阅约束 。

DROP条款

使用DROP子句删除现有视图约束。

限制丢弃约束

如果它是视图上的参照完整性约束的一部分，则不能删除唯一或主键约束。

COMPILE

COMPILE关键字指示Oracle数据库重新编译视图。

{只读|读写}

这些条款仅对编辑视图有效。

  • 指定READ ONLY以指示无法更新版本控制视图。

  • 指定READ WRITE以将只读版本视图返回到读/写状态。

指定这些子句时，数据库不会使依赖对象无效，但可能会使游标无效。

版面| NONEDITIONABLE

使用这些子句指定视图是否成为editioned或noneditioned对象，如果editioning为架构对象类型后启用VIEW的schema 。默认为EDITIONABLE 。有关更改已编辑和未编辑的对象的信息，请参阅“ Oracle数据库开发指南” 。

例子

改变视图：示例

要重新编译视图customer_ro （在“ 创建只读视图：示例 ”中创建 ），请发出以下语句：

ALTER VIEW customer_ro COMPILE;

如果Oracle Database在重新编译customer_ro时未遇到编译错误，则customer_ro将变为有效。如果重新编译导致编译错误，则数据库返回错误， customer_ro仍然无效。

Oracle数据库还会使所有依赖对象失效。这些对象包括引用customer_ro任何过程，函数，包体和视图。如果您随后引用其中一个对象而未先显式重新编译它，则数据库会在运行时隐式重新编译它。
