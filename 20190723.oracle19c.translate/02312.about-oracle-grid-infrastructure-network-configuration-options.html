关于Oracle Grid Infrastructure网络配置选项

确保您具有用于群集安装的Oracle Grid Infrastructure所需的网络硬件和Internet协议（IP）地址。

Oracle Clusterware网络

Oracle Clusterware配置至少需要两个接口：

  • 公共网络接口，用户和应用程序服务器连接到该接口以访问数据库服务器上的数据。

  • 用于节点间通信的专用网络接口。

您可以将网络接口配置为在给定网络上使用IPv4协议或IPv6协议。如果使用冗余网络接口（绑定或组合接口），请注意Oracle不支持配置一个接口以支持IPv4地址，另一个支持IPv6地址。您必须使用相同的IP协议配置冗余接口对的网络接口。

群集中的所有节点都必须使用相同的IP协议配置。所有节点都只使用IPv4，或者所有节点仅使用IPv6。您不能将群集中的某些节点配置为仅支持IPv6地址，并且群集中的其他节点配置为仅支持IPv4地址。

VIP代理支持使用无状态地址自动配置协议（RFC 2462）生成IPv6地址，并使用GNS通告这些地址。运行srvctl config network命令以确定是否正在使用动态主机配置协议（DHCP）或无状态地址自动配置。

有关Oracle RAC支持的网络协议和硬件的最新信息，请参阅My Oracle Support上的Certify页面：

https://support.oracle.com
