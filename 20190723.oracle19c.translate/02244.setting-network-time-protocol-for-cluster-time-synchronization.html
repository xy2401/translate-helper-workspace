为集群时间同步设置网络时间协议

使用Oracle Cluster Time Synchronization或网络时间协议（NTP）选项进行时间同步。

Oracle Clusterware要求在所有群集节点上设置相同的时区环境变量。在安装过程中，安装过程会在运行Oracle Universal Installer（OUI）的节点上获取Grid安装所有者的时区（TZ）环境变量设置，并将所有节点上的时区值用作默认TZ环境变量设置适用于Oracle Clusterware管理的所有进程。时区默认用于数据库，Oracle ASM和任何其他托管进程。您有两种时间同步选项：

  • 操作系统配置网络时间协议（NTP），例如chronyd或ntpd

  • Oracle集群时间同步服务

Oracle Cluster Time Synchronization Service专为群集服务器无法访问NTP服务的组织而设计。如果使用NTP，则Oracle Cluster Time Synchronization守护程序（ctssd）将以观察者模式启动。如果您没有NTP守护程序，则ctssd将以活动模式启动并在集群成员之间同步时间，而无需联系外部时间服务器。

注意：

  • 在开始安装Oracle Grid Infrastructure之前，Oracle建议您确保所有节点上的时钟设置为同一时间。

  • 默认情况下，Oracle Linux 7和Red Hat Linux 7上提供的NTP服务是chronyd 。

如果您的服务器上有NTP守护程序，但您无法将它们配置为与时间服务器同步时间，并且您希望使用群集时间同步服务在群集中提供同步服务，则停用并卸载NTP。

当安装程序发现NTP协议未处于活动状态时，群集时间同步服务将以活动模式安装，并在节点之间同步时间。如果发现NTP已配置，则群集时间同步服务将以观察者模式启动，并且群集中的Oracle Clusterware不会执行活动时间同步。

停用chronyd服务

要停用chronyd服务，必须停止现有的chronyd服务，并从初始化序列中禁用它。

在Oracle Linux 7和Red Hat Linux 7上完成此步骤：

  1. 以root用户身份运行以下命令：

    #systemctl stop chronyd #systemctl disable chronyd

安装后确认Oracle Cluster Time Synchronization Service

要确认安装后ctssd处于活动状态，请输入以下命令作为网格安装所有者：

$ crsctl检查ctss
