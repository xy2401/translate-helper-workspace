7.31 V $ ARCHIVE_DEST

对于当前实例， V$ARCHIVE_DEST显示Data Guard配置中的所有目标，包括每个目标的当前值，模式和状态。

柱 数据类型 描述

DEST_ID

数

日志归档目标参数标识符（ 1到31 ）

DEST_NAME

VARCHAR2（256）

记录归档目标参数名称

状态

VARCHAR2（9）

标识目标的当前状态：

  • VALID - 已初始化且可用

  • INACTIVE - 没有目的地信息

  • DEFERRED - 用户手动禁用

  • ERROR - 打开或复制期间出错

  • DISABLED - 错误后DISABLED

  • BAD PARAM - 参数有错误

  • ALTERNATE - 目的地可作为替代

  • FULL - 超出目的地的配额大小

捆绑

VARCHAR2（9）

指定失败将如何影响归档操作：

  • MANDATORY - 需要成功的归档

  • OPTIONAL - 不需要成功存档（取决于LOG_ARCHIVE_MIN_SUCCEED_DEST ）

NAME_SPACE

VARCHAR2（7）

标识参数设置的范围：

  • SYSTEM - 系统定义

  • SESSION - 会话定义

目标

VARCHAR2（7）

指定存档目标是本地数据库还是远程主数据库：

  • PRIMARY - 当地

  • STANDBY - 遥控器

归档器

VARCHAR2（10）

标识与发出查询的数据库相关的归档进程：

  • ARC n

  • 前景

  • LGWR

  • RFS

时间表

VARCHAR2（8）

指示此目标的归档是INACTIVE ， PENDING ， ACTIVE还是LATENT

目的地

VARCHAR2（256）

指定归档重做日志的归档位置

LOG_SEQUENCE

数

标识要归档的上次归档重做日志的序列号

REOPEN_SECS

数

标识错误后的重试时间（以秒为单位）

DELAY_MINS

数

标识归档重做日志自动应用于备用数据库之前的延迟间隔（以分钟为单位）

MAX_CONNECTIONS

数

最大连接数

NET_TIMEOUT

数

日志写入器进程将等待来自日志写入器进程发出的网络操作的网络服务器的状态的秒数

处理

VARCHAR2（10）

标识相对于主数据库的归档进程，即使在备用数据库上发出查询：

  • ARC n

  • 前景

  • LGWR

寄存器

VARCHAR2（3）

指示归档重做日志是否已在远程目标控制文件中注册（ YES ）（ NO ）。如果已注册归档重做日志，则可以使用日志应用服务。

FAIL_DATE

日期

上次错误的日期和时间

FAIL_SEQUENCE

数

上次发生错误时归档的归档重做日志的序列号

FAIL_BLOCK

数

上次发生错误时归档的归档重做日志的块编号

FAILURE_COUNT

数

目标已发生的当前连续归档操作失败的数量

MAX_FAILURE

数

允许您控制日志传输服务尝试重新建立通信并恢复目标失败的归档操作的次数

错误

VARCHAR2（256）

显示错误文本

备用

VARCHAR2（256）

替代目的地，如果有的话

DEPENDENCY

VARCHAR2（256）

保留供将来使用

REMOTE_TEMPLATE

VARCHAR2（256）

指定用于派生要记录的位置的模板

QUOTA_SIZE

数

目标配额，以字节表示

QUOTA_USED

数

当前驻留在指定目标上的所有归档重做日志的大小

MOUNTID

数

实例装入标识符

TRANSMIT_MODE

VARCHAR2（12）

指定网络传输模式：

  • 同步

  • PARALLELSYNC

  • 异步

ASYNC_BLOCKS

数

为ASYNC属性指定的块数

确认

VARCHAR2（3）

指定磁盘I / O模式

类型

VARCHAR2（7）

指示归档日志目标定义是PUBLIC还是PRIVATE 。只能使用ALTER SYSTEM SET或ALTER SESSION SET语句在运行时修改PUBLIC目标。默认情况下，所有存档的日志目标都是PUBLIC 。

VALID_NOW

VARCHAR2（16）

指示目标目前是否对存档操作有效：

  • YES - 此目标的重做日志类型和数据库角色对当前数据库有效

  • WRONG VALID_TYPE - 为此目标指定的重做日志类型对当前数据库角色无效。例如，如果使用VALID_FOR=(STANDBY_LOGFILE,STANDBY_ROLE)属性指定的目标在备用数据库角色中运行但未实现备用重做日志， WRONG VALID_TYPE返回WRONG VALID_TYPE 。

  • WRONG VALID_ROLE - 为此目标指定的数据库角色不是数据库当前正在运行的角色。例如，当使用VALID_FOR=(ONLINE_LOGFILE,STANDBY_ROLE)属性定义的目标在主数据库角色中运行时，将返回WRONG VALID_ROLE 。

  • INACTIVE - 目标处于非活动状态，可能是由于错误

VALID_TYPE

VARCHAR2（15）

重做日志类型或对目标有效的类型：

  • ONLINE_LOGFILE

  • STANDBY_LOGFILE

  • ALL_LOGFILES

VALID_ROLE

VARCHAR2（12）

对目标有效的数据库角色或角色：

  • PRIMARY_ROLE

  • STANDBY_ROLE

  • ALL_ROLES

DB_UNIQUE_NAME

VARCHAR2（30）

唯一数据库名称

校验

VARCHAR2（3）

指示LOG_ARCHIVE_DEST_ n参数上的VERIFY属性值是否已验证（ YES ）或未验证（ NO ）

压缩

VARCHAR2（7）
指示网络压缩是ENABLED还是DISABLED ，或者是否正在使用ZLIB或LZO算法。这些值对应于数据库初始化参数LOG_ARCHIVE_DEST_n的COMPRESSION属性所接受的值。可能的值包括：

  • DISABLE ： DISABLE压缩。

  • ENABLE ：启用压缩。使用ZLIB压缩算法。

  • ZLIB ：使用ZLIB压缩。

  • LZO ：使用LZO压缩。

APPLIED_SCN

数

对于与物理或逻辑备用数据库对应的目标，最后应用的重做的SCN。对于与快照备用数据库对应的目标，在转换为快照备用数据库之前应用的最后一个重做的SCN。此列仅对主数据库或级联备用数据库上的已启用和活动备用数据库目标有效。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

加密

VARCHAR2（7）

指示发送到零数据丢失恢复设备（恢复设备）的重做流的加密是否已启用（ ENABLED ）（ ENABLED DISABLED ）（ ENABLED DISABLED ）
