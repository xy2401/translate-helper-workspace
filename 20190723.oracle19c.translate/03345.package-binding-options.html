适用于DRDA的Oracle数据库提供程序中的B程序包绑定选项

DB2 DSN子命令BIND PACKAGE有几个选项，当Oracle绑定客户端应用程序的包时使用这些选项。

有关BIND PACKAGE子命令的更多详细信息，请参见以下卷： DB2®10for z /OS®命令参考（SC19-2972-05） 。

CONCURRENTACCESSRESOLUTION

始终实现WAITFOROUTCOME语义;这是Oracle如何进行数据锁定的基础，并且无法更改。

CURRENTDATA

支持YES语义。

DBPROTOCOL

支持DRDA语义。

DEFER / NODEFER

支持NODEFER语义。

DYNAMICRULES

支持RUN语义。

默认限定符用法适用于DB2规则定义的每个语句。这相当于STATIC或DYNAMIC SQL语句的SET CURRENT SQLID 。但是，Oracle将对象访问权限限制为当前登录标识，而不是包所有者的标识。要控制对象访问，必须通过显式访问授权（对象授予或角色授予）或通过强制对象访问授权的存储过程来处理登录。

启用/禁用

支持ENABLE语义。

EXTENDEDINDICATOR

支持NO语义。

隔离

支持CS语义。

Oracle和Oracle数据库提供程序（DRDA）不支持大多数隔离模式。可以将Oracles数据隔离模式描述为CS和RR之间的折衷。这是无法更改的，因为它是Oracle实现数据完整性管理的基础。忽略该值时，它将与包定义一起存储以供将来处理。

KEEPDYNAMIC

支持YES语义。

所有者

授权标识必须是有效的Oracle用户标识。

QUALIFIER

限定符名称应该是Oracle中的有效模式名称。

发布

实现COMMIT语义。

使用DRDA时，资源的释放在不同级别执行。在Commit / Rollback中，游标或语句的语义可以请求释放游标和对象锁。其他资源（例如包本身）将保留到会话结束，然后才会取消分配。此行为主要由客户端在运行时指定，而不是由服务器指定。

REOPT

实现AUTO语义。

默认情况下，Oracle会根据语句和主机变量自动评估执行计划; STATIC或DYNAMIC声明都接受相同的处理。

四舍五入

实现HALFEVEN语义。

VALIDATE

实现RUN语义。

忽略此选项，因为DRDA的Oracle数据库提供程序不执行STATIC SQL语句的验证。
