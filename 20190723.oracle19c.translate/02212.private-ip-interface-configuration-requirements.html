专用IP接口配置要求

专用接口的要求取决于您使用的是单个还是多个接口。

单接口专用网络集群的网络要求

  • 每个节点的互连专用接口必须位于同一子网中。

  • 子网必须连接到群集的每个节点。

    例如，如果专用接口的子网掩码为255.255.255.0，则您的专用网络的范围为192.168.0.0--192.168.0.255，您的专用地址必须在192.168.0范围内。[0- 255]。如果专用接口的子网掩码为255.255.0.0，则您的专用地址可以在192.168范围内。[0-255][0-255]

  • 支持IPv4和IPv6地址。

冗余互连使用群集的网络要求

通过冗余互连使用，您可以识别用于群集专用网络的多个接口，而无需使用绑定或其他技术。

定义多个接口时，Oracle Clusterware会创建一到四个高可用性IP（HAIP）地址。Oracle RAC和Oracle自动存储管理（Oracle ASM）实例使用这些接口地址来确保节点之间高度可用的负载平衡接口通信。安装程序启用冗余互连使用以提供高可用性专用网络。默认情况下，Oracle Grid Infrastructure软件使用所有HAIP地址进行专用网络通信，从而为您为专用网络识别的一组接口提供负载平衡。如果专用互连接口出现故障或变为非通信接口，则Oracle Clusterware会将相应的HAIP地址透明地移动到其余功能接口之一。

  • 每个专用接口应位于不同的子网上。

  • 每个集群成员节点必须在每个专用互连子网上都有一个接口，并且这些子网必须连接到集群的每个节点。

    例如，您可以在子网192.168.0和10.0.0上拥有专用网络，但每个集群成员节点必须具有连接到192.168.0和10.0.0子网的接口。

  • 所有指定的互连接口的端点必须在网络上完全可达。应该没有未连接到每个专用网络接口的节点。

    您可以使用ping测试是否可以访问互连接口。

  • 您可以将IPv4和IPv6地址用于与Oracle Clusterware冗余互连的接口。

注意：

在安装过程中，您最多可以为专用网络定义四个接口。安装期间创建的HAIP地址数基于为网络适配器配置的物理和逻辑接口。安装后，您可以定义其他接口。如果将四个以上的接口定义为专用网络接口，请注意Oracle Clusterware一次仅激活四个接口。但是，如果四个活动接口中的一个出现故障，则Oracle Clusterware会将配置为故障接口的HAIP地址转换为定义的一组专用接口中的一个保留接口。
