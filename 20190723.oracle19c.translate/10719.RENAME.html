改名

目的

注意：

您无法回滚RENAME语句。

使用RENAME语句重命名表，视图，序列或私有同义词。

  • Oracle数据库自动将旧对象的完整性约束，索引和授权传递给新对象。

  • Oracle数据库使依赖于重命名对象的所有对象无效，例如视图，同义词以及引用重命名表的存储过程和函数。

先决条件

该对象必须位于您自己的架构中。

句法

rename :: =


说明rename.eps的说明

语义

OLD_NAME

指定现有表，视图，序列或私有同义词的名称。

新名字

指定要为现有对象指定的新名称。新名称不得已由同一名称空间中的另一个模式对象使用，并且必须遵循命名模式对象的规则。

重命名对象的限制

重命名对象受以下限制：

  • 您无法重命名公共同义词。而是删除公共同义词，然后使用新名称重新创建公共同义词。

  • 您不能重命名具有任何从属表或从属有效用户定义对象类型的类型同义词。

例子

重命名数据库对象：示例

以下示例使用示例表hr.departments的副本。要将表departments_new的名称更改为emp_departments ，请发出以下语句：

RENAME departments_new TO emp_departments;

您不能直接使用此语句重命名列。但是，您可以使用ALTER TABLE ... rename_column_clause重命名列。

也可以看看：

rename_column_clause

重命名列的另一种方法是将RENAME语句与带有AS subquery的CREATE TABLE语句一起使用。如果要更改表的结构而不是仅重命名列，则此方法很有用。以下语句重新创建示例表hr.job_history ，将一个列从department_id重命名为dept_id ：

CREATE TABLE临时（employee_id，start_date，end_date，job_id，dept_id）AS SELECT employee_id，start_date，end_date，job_id，department_id FROM job_history; DROP TABLE job_history; RENAME临时TO job_history;

在上面的示例中，表job_history上定义的任何完整性约束都将丢失。您必须使用ALTER TABLE语句在新的job_history表上重新定义它们。
