数据库对象名称和限定符

某些数据库对象由您可以或必须命名的部分组成，例如表或视图中的列，索引和表分区和子分区，表上的完整性约束以及存储在包中的对象，包括过程和存储的功能。本节提供：

  • 命名数据库对象和数据库对象位置限定符的规则

  • 命名数据库对象和限定符的准则

注意：

对于隐式生成的数据库对象和子对象，Oracle使用以“ SYS_ ”开头的系统生成的名称，对于某些Oracle提供的对象，使用以“ ORA_ ”开头的名称。Oracle不鼓励您在显式提供给数据库对象和子对象的名称中使用这些前缀，以避免名称解析可能发生冲突。

数据库对象命名规则

每个数据库对象都有一个名称。在SQL语句中，你代表了带引号的标识符或带引号的标识符的对象的名称。

  • 带引号的标识符以双引号（“）开头和结尾。如果使用带引号的标识符命名架构对象，则每次引用该对象时都必须使用双引号。

  • 未加引号的标识符不包含任何标点符号。

您可以使用带引号或非带引号的标识符来命名任何数据库对象。但是，数据库名称，全局数据库名称，数据库链接名称，磁盘组名称和可插入数据库（PDB）名称始终不区分大小写，并以大写形式存储。如果将此类名称指定为带引号的标识符，则会以静默方式忽略引号。

注意：

Oracle建议不要对数据库对象名称使用带引号的标识符。SQL * Plus接受这些带引号的标识符，但在使用管理数据库对象的其他工具时它们可能无效。

除非另有说明，否则以下规则列表适用于引用和非引用标识符：

  1. 标识符名称的最大长度取决于COMPATIBLE初始化参数的值。

    • 如果COMPATIBLE设置为12.2或更高的值，则名称必须为1到128个字节长，但有以下例外：

      • 数据库的名称限制为8个字节。

      • 磁盘组，可插拔数据库（PDB），回滚段，表空间和表空间集的名称限制为30个字节。

      如果标识符包含由句点分隔的多个部分，则每个属性最长可达128个字节。每个句点分隔符以及任何周围的双引号都计为一个字节。例如，假设您标识了这样的列：

      “ schema ”。“ table ”。“ column ”

      模式名称可以是128个字节，表名称可以是128个字节，列名称可以是128个字节。每个引号和句点都是单字节字符，因此本例中标识符的总长度最多为392个字节。

    • 如果COMPATIBLE设置为低于12.2的值，则名称长度必须为1到30个字节，但以下情况除外：

      • 数据库的名称限制为8个字节。

      • 数据库链接的名称可以长达128个字节。

      如果标识符包含由句点分隔的多个部分，则每个属性最长可达30个字节。每个句点分隔符以及任何周围的双引号都计为一个字节。例如，假设您标识了这样的列：

      “ schema ”。“ table ”。“ column ”

      模式名称可以是30个字节，表名称可以是30个字节，列名称可以是30个字节。每个引号和句点都是单字节字符，因此本例中标识符的总长度最多为98个字节。

  2. 不带引号的标识符不能是Oracle SQL保留字。带引号的标识符可以是保留字，但不建议这样做。

    根据您计划用于访问数据库对象的Oracle产品，名称可能会受到其他特定于产品的保留字的进一步限制。

    注意：

    保留字ROWID是此规则的一个例外。您不能使用引号或非引号的大写单词ROWID作为列名。但是，您可以将大写单词用作不是列名的带引号的标识符，并且可以将带有一个或多个小写字母的单词（例如，“ Rowid ”或“ rowid ”）用作任何带引号的标识符，包括列名。

  3. Oracle SQL语言包含具有特殊含义的其他单词。这些单词包括数据类型，模式名称，函数名称，虚拟系统表DUAL和关键字（SQL语句中的大写单词，如DIMENSION ， SEGMENT ， ALLOCATE ， DISABLE等）。这些话不是保留的。但是，Oracle以特定方式在内部使用它们。因此，如果您将这些单词用作对象和对象部件的名称，那么您的SQL语句可能更难以阅读并可能导致不可预测的结果。

    特别是，不要使用以SYS_或ORA_开头的ORA_作为模式对象名称，也不ORA_ SQL内置函数的名称用于模式对象或用户定义函数的名称。

  4. 您应该在数据库名称，全局数据库名称和数据库链接名称中使用ASCII指令表中的字符，因为这些字符提供了跨不同平台和操作系统的最佳兼容性。您必须在多租户容器数据库（CDB）中仅使用ASCII指令表中的普通用户名和公共角色。

  5. 您可以在密码中包含多字节字符。

  6. 不带引号的标识符必须以数据库字符集中的字母字符开头。带引号的标识符可以以任何字符开头。

  7. 不带引号的标识符只能包含数据库字符集和下划线（_）中的字母数字字符。数据库链接可以包含句点（。）和“at”符号（@）。

    带引号的标识符可以包含任何字符和标点符号以及空格。但是，引号和非引号标识符都不能包含双引号或空字符（ \0 ）。

  8. 在命名空间内，没有两个对象可以具有相同的名称。

    以下架构对象共享一个命名空间：

    • 包

    • 私人同义词

    • 序列

    • 独立程序

    • 独立存储的功能

    • 表

    • 用户定义的运算符

    • 用户定义的类型

    • 查看

    以下每个架构对象都有自己的命名空间：

    • 集群

    • 约束

    • 数据库触发器

    • 外形尺寸

    • 索引

    • 物化视图（创建物化视图时，数据库会创建同名的内部表。此表与架构中的其他表具有相同的命名空间。因此，架构不能包含表和同名的物化视图。）

    • 私人数据库链接

    由于表和序列位于同一名称空间中，因此同一模式中的表和序列不能具有相同的名称。但是，表和索引位于不同的名称空间中。因此，同一模式中的表和索引可以具有相同的名称。

    数据库中的每个模式都有自己的名称空间，用于它包含的对象。这意味着，例如，不同模式中的两个表位于不同的名称空间中，并且可以具有相同的名称。

    以下每个nonschema对象也都有自己的命名空间：

    • 版本

    • 参数文件（ PFILE ）和服务器参数文件（ SPFILE s）

    • 简介

    • 公共数据库链接

    • 公共同义词

    • 表空间

    • 用户角色

    由于这些名称空间中的对象未包含在模式中，因此这些名称空间跨越整个数据库。

  9. 不带引号的标识符不区分大小写。Oracle将它们解释为大写。带引号的标识符区分大小写。

    通过将名称括在双引号中，可以将以下名称赋予同一名称空间中的不同对象：

    “员工”“员工”“员工”

    请注意，Oracle将以下名称解释为相同，因此它们不能用于同一名称空间中的不同对象：

    员工员工“员工”

  10. 当Oracle以大写形式存储或比较标识符时，标识符中每个字符的大写形式是通过应用数据库字符集的大写规则来确定的。不考虑由会话设置NLS_SORT确定的特定于语言的规则。此行为对应于将SQL函数UPPER应用于标识符而不是函数NLS_UPPER 。

    数据库字符集大写规则可以在查看为某种自然语言时产生不正确的结果。例如，德语中使用的小写字母s（“ß”）根据数据库字符集大写规则不具有大写形式。当标识符转换为大写时，它不会被修改，而德语中的预期大写形式是大写字母S（“SS”）的两个字符的序列。类似地，根据数据库字符集大写字母规则，小写字母i的大写形式是大写字母I.但是，土耳其语和阿塞拜疆语中的预期大写形式是带有上方点的大写字母I.

    数据库字符集大写规则确保在会话的任何语言配置中解释标识符相同。如果您希望标识符以某种自然语言正确显示，则可以引用它以保留小写形式，或者您可以在使用该标识符时使用语言正确的大写形式。

  11. 同一表或视图中的列不能具有相同的名称。但是，不同表或视图中的列可以具有相同的名称。

  12. 如果它们的参数的数量和数据类型不同，则同一包中包含的过程或函数可以具有相同的名称。在具有不同参数的同一包中创建具有相同名称的多个过程或函数称为重载过程或函数。

模式对象命名指南

以下是命名对象及其部件的一些有用指南：

  • 使用完整的，描述性的，可发音的名称（或众所周知的缩写）。

  • 使用一致的命名规则。

  • 使用相同的名称来跨表描述相同的实体或属性。

命名对象时，要平衡保持名称简短易用的目标，以使名称尽可能具有描述性。如有疑问，请选择更具描述性的名称，因为数据库中的对象可能会在一段时间内被许多人使用。十年后你的对手可能难以理解一个名为pmdd而不是payment_due_date的表格列。

使用一致的命名规则可帮助用户了解每个表在应用程序中播放的部分。一个这样的规则可能是使用fin_开始属于FINANCE应用程序的所有表的名称。

使用相同的名称来描述表中的相同内容。例如，示例employees和departments表的部门编号列都命名为department_id 。
