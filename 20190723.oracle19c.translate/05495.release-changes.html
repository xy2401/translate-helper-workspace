新功能

以下主要功能是此版本中的新功能。

  • 数据库管理的PDB快照的增强功能

    当初始化参数CLONEDB为TRUE ，数据库管理的快照会创建源PDB的数据文件的稀疏副本。从Oracle Database 19c开始，您可以从打开的读/写源PDB创建数据库管理的快照克隆PDB。在以前的版本中，源PDB需要在任何快照克隆PDB的生命周期内保持只读和不变。

    由于源PDB不需要处于只读模式，因此您可以创建生产数据库的快照，然后从这些快照克隆PDB。此外，快照轮播中的快照可以是稀疏副本，使得轮播比以前的版本消耗更少的存储空间。

    请参阅“ PDB快照的内容 ” 。

  • PDB中的工作负载捕获和重放

    本地用户可以在PDB级别捕获，重放和报告工作负载。

    请参阅“ 关于在CDB中使用可管理性功能 ”和“ Oracle数据库测试指南” 。

  • PDB的ADDM分析

    您可以使用ADDM通过PDB内部的AWR快照分析存储在PDB中的AWR数据。您还可以分析导入到PDB的AWR存储中的非CDB，CDB根或PDB的AWR数据。默认情况下禁用PDB的自动ADDM。您可以通过启用自动AWR快照为PDB启用它。

    请参阅“ 关于在CDB中使用可管理性功能 ”和“ Oracle数据库性能调整指南” 。

  • Database Vault Operations Control用于基础架构数据库管理员

    您可以使用Oracle Database Vault阻止普通用户（例如，基础架构DBA）访问PDB中的本地数据。因此，阻止普通用户访问本地数据。它使您能够为业务应用程序存储敏感数据，并允许操作管理数据库基础结构，而无需访问敏感的客户数据。

    请参阅“ 使用Database Vault Operations Control限制多租户公共用户对本地PDB数据的访问 ” 。

  • 支持同一CDB中的多个PDB分片

    单个CDB可以包含多个PDB作为分片目录数据库。此外，CDB可以包含来自不同分片数据库（SDB）的分片PDB，每个分片数据库由其自己的单独目录数据库管理。

    请参阅使用Oracle Sharding 。

  • 自动PDB重定位

    在Oracle Grid Infrastructure中，您可以使用Fleet Patching和Provisioning自动将PDB从一个CDB重定位到另一个CDB。自动重定位可以更快地修补单个PDB，而不会将其他PDB暴露给补丁中的更改。

    请参见“ Oracle Clusterware管理和部署指南” 。

  • 使用DBCA克隆远程PDB

    您可以使用DBCA以静默方式克隆远程PDB。

    请参阅“ 关于克隆远程PDB ” 。

  • 远程PDB重定位

    您可以使用Database Configuration Assistant（DBCA）将PDB从远程CDB重定位到本地CDB。

    请参阅“ 使用DBCA重定位PDB：示例 ” 。

  • 云对象存储支持数据泵导入

    impdb的credential参数指定凭证对象的名称，该凭据对象包含访问对象存储桶所需的用户名和密码。您还可以使用数据库属性DEFAULT_CREDENTIAL指定默认凭据。

    请参阅“ 在PDB中设置默认凭据 ”和“ 使用数据泵将数据移入CDB ” 。

Oracle Database Release 18c，版本18.1中的更改

以下是Oracle Multitenant管理员指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

  • 新功能

上级主题： Oracle Multitenant管理员指南的此发行版中的更改

新功能

以下功能是此版本中的新增功能：

  • CDB舰队

    CDB机群是不同CDB的集合，可以作为一个逻辑CDB进行管理。

    请参阅“ 管理CDB Fleet ” 。

  • PDB快照轮播

    PDB快照是PDB的时间点副本。您可以使用CREATE PLUGGABLE DATABASE （或ALTER PLUGGABLE DATABASE ）的SNAPSHOT子句手动创建快照，也可以使用EVERY interval子句自动创建快照。 为快照启用PDB后，您可以创建PDB的多个快照（时间点副本）。快照库称为PDB快照轮播 。您可以根据轮播中的任何快照快速克隆新的PDB。通过这种方式，您可以对轮播中的任何快照执行时间点恢复，也可以通过克隆任何快照来快速创建PDB。

    请参阅“ PDB快照轮播的用户界面 ”和“ 管理PDB快照轮播 ” 。

  • 逻辑分区

    容器映射使会话能够发出路由到相应PDB的SQL语句，具体取决于SQL语句中使用的谓词的值。映射表中的分区列不需要与元数据链接表中的列匹配。例如，如果为容器映射pdb_map_tbl启用了表sales ，并且如果sales没有用于分区pdb_map_tbl的列，则具有谓词CONTAINERS(sales)查询仍将路由到映射表中指定的PDB。

    请参阅“ 容器地图 ” 。

  • 可刷新的PDB切换

    可刷新克隆PDB是只读克隆，可以定期与其源PDB同步。您可以反转角色，将源PDB转换为克隆，将克隆转换为源。此技术可用于负载平衡。此外，如果源PDB失败，则可以恢复克隆PDB上的操作，从而无需进行CDB级Oracle Data Guard故障转移。

    请参阅“ 关于可刷新克隆PDB ”和“ 切换可刷新克隆PDB ” 。

  • 锁定配置文件增强功能

    您可以在应用程序容器中创建，更改或删除锁定配置文件。此外，您可以基于静态或动态基本配置文件创建锁定配置文件。

    请参阅“ PDB锁定配置文件概述 ” ， “ 关于限制PDB用户以增强安全性 ”和“ 使用PDB锁定配置文件限制PDB上的操作 ” 。

  • DBCA增强功能

    您可以使用DBCA克隆本地PDB或复制CDB。仅在静默模式下支持复制。

    请参阅“ 关于使用DBCA创建CDB ”和“ 关于克隆本地PDB ” 。

  • 可用于非CDB和重新定位的PDB的备份

    将非CDB克隆为PDB或重定位PDB时，可以使用DBMS_PDB.EXPORTRMANBACKUP过程将RMAN备份元数据导出到PDB字典中。此元数据使源非CDB或PDB的备份可用于恢复和恢复目标PDB。

    请参阅“ PDB创建的一般先决条件 ” 。

  • RMB将PDB复制到另一个CDB

    您可以将PDB从源CDB克隆到打开的读/写的现有CDB。

    请参阅“ 创建PDB的技术 ”

  • 在计划维护期间重新安排会话

    应用程序连续性可以在计划维护期间在应用程序提交连接测试时，在请求边界以及故障转移的良好位置处耗尽数据库会话。重定位对应用程序是透明的。默认情况下，对于在数据库服务和PDB级别调用的所有维护操作，此功能处于启用状态：停止服务，重新定位服务，重新定位PDB以及停止PDB。

    请参阅“ 管理PDB服务 ” ， “ PDB重定位的工作 原理 ”和“ Oracle Real Application Clusters管理和部署指南” 。

  • 在Oracle Data Guard环境中复制PDB

    在主数据库中执行远程克隆或在主数据库中插入PDB时，可以在备用数据库中设置初始化参数，以自动复制新创建的PDB的数据文件。

    请参阅“ 克隆远程PDB：基本步骤 ”和“ 插入未插入的PDB ” 。

  • 并行语句在PDB级别排队

    您可以使用PARALLEL_SERVERS_TARGET初始化参数为PDB配置并行语句队列，就像为非PDB配置一样。在PDB级别，默认值基于PDB的CPU_COUNT设置。在CDB级别，默认值是PARALLEL_MAX_SERVERS初始化参数的值。

    请参阅“ PDB的利用率限制 ” 。

  • 分割镜像克隆PDB

    当PDB驻留在Oracle ASM中时，您可以使用拆分镜像技术来克隆PDB。克隆的PDB独立于原始PDB。主要用例是在Oracle ASM环境中快速配置测试和开发PDB。

    请参阅“ 创建分割镜像克隆PDB ” 。
