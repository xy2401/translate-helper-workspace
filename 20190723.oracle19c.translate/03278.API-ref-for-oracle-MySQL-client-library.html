映射数据类型

“ Oracle调用接口程序员指南”的“内部数据类型”部分介绍了Oracle数据库类型。

MySQL数据类型在MySQL文档中有完整描述。

MySQL C API使用MYSQL_TYPE _symbols处理与MySQL数据库之间的数据。这些类型符号映射到服务器中的MySQL数据类型。

例如， MYSQL_TYPE_VAR_STRING映射到服务器中的VARCHAR 。

错误处理

当错误后调用mysql_errno（）或mysql_error（）方法时，OCI或Oracle服务器生成的所有错误都会传递给应用程序。应用程序收到特定于Oracle的错误。Oracle错误消息比MySQL错误代码更具体，因此更适合解决错误情况。

驱动程序本身生成的错误位于OCI错误空间中为MySQL驱动程序保留的错误范围内。

mysql_sqlstate（）调用尝试尽可能将错误映射到适当的SQLSTATE 。在大多数情况下，它返回HY000 ，它对应于一般错误状态。

可能的SQLSTATE值是：

  • 00000成功

  • HY000所有其他错误

但是，这也意味着必须部分重写期望更具体的SQLSTATE错误的客户端应用程序。

可用的Oracle API支持

Oracle MySQL驱动程序实现了MySQL C API文档中列出的API。请注意以下事项：

  • 一些MySQL函数已经改变了行为，通常是由于Oracle中没有等效行为;说明中注明了更改的行为。

  • 某些MySQL函数不受支持;描述相应地标记它们。驱动程序为这些函数返回错误，并提示应用程序解决不支持的功能。

支持的MySQL API在此处进行功能分组，这里是指向更多信息的链接。但是，我们不提供函数行为和参数的完整文档，而是将其留给原始的MySQL C API文档。

my_init（）

此功能是无操作功能。它由my_sys.h文件中的my_init宏my_sys.h 。所有初始化都由mysql_library_init() 。

回报价值

0

mysql_affected_rows（）函数

返回为INSERT ， UPDATE和DELETE语句处理的行数。

对于UPDATE语句，请注意MySQL的语义不会报告新值与旧值相同的行。相反，Oracle报告行受到影响，即使新值与旧值相同。此函数实现Oracle语义。因此，依赖此调用的现有应用程序可能必须进行编程更改。

对于SELECT语句，返回值为（ my_ulonglong ）-1。

回报价值

由DML语句处理的许多行; >0 。 0表示该语句未进行任何更新。 -1表示该语句是查询（ SELECT ）或错误。

mysql_autocommit（）

将自动提交模式设置为ON或OFF 。

回报价值

0 ，如果自动提交模式成功更改。如果进程中发生错误，则为非零。

mysql_change_user（）

更改用户，包括相同或不同主机上的用户名，密码和数据库。在Oracle Database 12c中 ，不支持更改数据库，因此将忽略为db参数输入的值。

对mysql_change_user()调用将回滚所有活动事务，结束当前会话，然后根据存储在host参数中的信息重新建立新连接。

现有应用程序必须进行必要的应用程序逻辑更改才能在Oracle Database 12 c中实现此行为。

回报价值

如果可以使用提供的用户名和密码的原始主机重新建立连接，则为0 。如果发生错误，则为非零。

mysql_character_set_name（）

在Oracle数据库12c中不支持。依赖于此调用结果的应用程序必须更改其应用程序逻辑。

回报价值

空字符串。

mysql_close（）

关闭连接并释放所有关联的数据结构。

回报价值

没有

mysql_commit（）

提交当前与服务上下文关联的事务。

mysql_commit()调用支持Oracle Database 12 c中的默认模式。因此它忽略了completion type系统变量。

使用此API执行特定于MySQL的completion type操作的现有应用程序必须更改其应用程序逻辑。

回报价值

0如果成功，否则返回非零值。

mysql_connect（）函数

不推荐使用;使用mysql_real_connect（） 。

回报价值

初始化的MYSQL结构。如果发生错误，则为NULL 。

mysql_create_db（）

必须首先调用函数mysql_stmt_bind_param() ，然后调用mysql_stmt_send_long_data() ，然后调用mysql_stmt_execute() 。

可以多次调用该函数以发送字符的部分或列的二进制数据值。

回报价值

0如果数据被成功地发送到服务器，非零值，如果发生错误。

mysql_stmt_sqlstate（）

返回最近准备的语句的SQLSTATE字符串。MySQL中有许多SQLSTATE代码未使用。

回报价值

SQLSTATE代码：“ 00000 ” - 成功，或“ HY0000 ” - 所有其他错误。

了mysql_stmt_affected_rows（）

存储上次查询的结果集。

如果最后一个查询是SELECT ，则返回结果集。如果最后一个语句是非SELECT或错误，则返回NULL结果集。

回报价值

成功时的有效结果集，发生错误时为NULL或非SELECT语句。

了mysql_store_result（）

存储上次查询的结果集。

如果最后一个查询是SELECT ，则返回结果集。

如果最后一个语句是非SELECT或错误，则返回NULL结果集。

回报价值

成功时的有效结果集;否则， NULL表示错误或非SELECT语句。
