8 XML C API的包模式

XML模式验证器的此C实现遵循W3C XML模式规范rev REC-xmlschema-1-20010502。它实现了模式验证器的所需行为，以便将多个模式文档组装到模式中。生成的模式可用于验证特定的实例文档。

对于基于事件的模式验证，请参阅C的Package事件API中记录的方法。

下表总结了通过XML C API的Schema接口可用的方法。

XmlSchemaUnload（）

从验证程序卸载架构文档。所有先前加载的架构文档将保持加载状态，直到它们被卸载。要卸载所有加载的模式文档，请将URI设置为NULL （这相当于XmlSchemaClean ）。请注意，还会卸载与给定模式关联的所有子模式。在此实现中，它仅支持以下方案：

  • 负载，负载，......

  • 加载，加载，加载，卸载，卸载，卸载，清理，然后重复。

它不支持：加载，加载，卸载，加载，....

XmlSchemaValidate（）

根据模式验证元素节点。当前会话中使用的模式包含通过XmlSchemaLoad指定的所有模式文档，并通过实例文档中的schemaLocation或noNamespaceSchemaLocation作为提示提供。调用此例程后，所有加载的模式文档仍保持加载状态，并可由XmlSchemaLoadedList查询。但是，它们将保持不活动状态。在下一个验证会话中，可以通过XmlSchemaLoad指定它们或通过新实例文档中的schemaLocation或noNamespaceSchemaLocation将它们作为提示来激活非活动模式文档。要卸载架构文档及其所有后代（包含或以嵌套方式导入的文档），请使用XmlSchemaUnload 。
