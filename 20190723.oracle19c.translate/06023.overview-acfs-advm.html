Oracle ACFS概述

Oracle自动存储管理群集文件系统（Oracle ACFS）是一种多平台，可扩展的文件系统和存储管理技术，可扩展Oracle自动存储管理（Oracle ASM）功能以支持所有客户文件。

Oracle ACFS支持Oracle数据库文件和应用程序文件，包括可执行文件，数据库数据文件，数据库跟踪文件，数据库警报日志，应用程序报告，BFILE和配置文件。其他支持的文件是视频，音频，文本，图像，工程图和所有其他通用应用程序文件数据。Oracle ACFS符合适用于Linux和UNIX的POSIX标准，适用于Windows的Windows标准。

Oracle ACFS利用Oracle ASM功能，该功能支持：

  • Oracle ACFS动态文件系统调整大小

  • 通过直接访问Oracle ASM磁盘组存储，实现最佳性能

  • 在Oracle ASM磁盘组存储中平衡分配Oracle ACFS，以提高I / O并行性

  • 通过Oracle ASM镜像保护机制实现数据可靠性

Oracle ACFS建立并维护与Oracle ASM实例的通信，以参与Oracle ASM状态转换，包括Oracle ASM实例和磁盘组状态更新以及磁盘组重新平衡。Oracle自动存储管理与Oracle ACFS和Oracle ASM动态卷管理器（Oracle ADVM）提供对所有客户数据的支持，并在Oracle Restart（独立）上的多个供应商平台和操作系统环境中提供一组通用的Oracle存储管理工具和服务和群集配置。

Oracle ACFS与Oracle Clusterware技术紧密结合，直接参与Clusterware集群成员资格状态转换和Oracle Clusterware基于资源的高可用性（HA）管理。此外，Oracle安装，配置，验证和管理工具已更新，以支持Oracle ACFS。

可以使用本机操作系统文件系统工具和标准应用程序编程接口（API）访问和管理Oracle ACFS。Oracle ACFS也可以使用Oracle ASM Configuration Assistant进行管理。可以使用行业标准网络连接存储（NAS）文件访问协议访问Oracle ACFS：网络文件系统（NFS）和通用Internet文件系统（CIFS）。但是，Windows上的CIFS客户端在与Oracle ACFS Linux，Solaris或AIX服务器连接时不能使用ACL，但可以在Windows上将ACL与Oracle ACFS一起使用。

除了共享文件数据外，Oracle ACFS还提供其他存储管理服务，包括支持Oracle Grid Infrastructure集群范围的安装注册表，动态在线文件系统调整大小以及每个文件系统的多个空间高效快照。

Oracle ACFS通过提供以下功能，为整体Oracle存储管理做出贡献：

  • 与Oracle ASM和Oracle Clusterware技术集成的通用独立服务器和集群文件系统解决方案

  • 跨多个供应商平台和操作系统的一组通用文件系统功能，提供本机操作系统或第三方文件系统解决方案的替代方案

  • 独立和群集范围共享Oracle数据库主目录，所有Oracle数据库文件和应用程序数据

  • 统一，一致的共享文件访问和所有客户应用程序文件的群集范围命名

  • 与Oracle Clusterware高可用性资源集成

Oracle ACFS可容纳大容量存储容量和大量群集节点。它有效地管理大量文件系统，文件，并支持具有exabyte功能的文件和文件系统容量的小型和大型文件。Oracle ACFS为具有数百万个文件的大型目录提供优化的快速目录查找。

Oracle ACFS提供对稀疏文件的支持。Oracle ACFS稀疏文件极大地有利于NFS客户端写入操作，这些操作通常由NFS服务器和关联的Oracle ACFS文件系统无序接收。通常，当应用程序写入超出文件末尾时，将分配存储并将零插入超出旧文件末尾和新文件的开头。使用此功能，文件中将保留一个孔而不是插入的零。当读取孔时，Oracle ACFS会在内存中用零填充这些漏洞。稀疏文件功能有利于NFS性能以及有意执行此类写入的其他应用程序的性能和磁盘利用率。此外，对于本质上稀疏的文件，也减少了时间和存储优势，这意味着它们具有大量未使用的空间，例如用于虚拟机的一些图像文件。对于稀疏文件支持， COMPATIBLE.ADVM磁盘组属性必须设置为12.2或更高。

Oracle ACFS文件系统通常安装在所有群集节点上，以便为群集提供单个名称空间，以便每个节点保持与装入的文件系统相同的视图和访问功能。如果成员发生故障，另一个集群成员将代表发生故障的成员快速恢复任何未完成的元数据事务。恢复后，可以恢复其他活动集群成员和任何远程客户端系统的访问。

以下列表提供了有关Oracle ACFS的重要信息：

  • 对于所有应用程序，Oracle ACFS性能最适合较大的write（）大小，例如8 K或更大。

  • 为获得最佳性能，请对Linux系统上磁盘组中的磁盘使用Deadline I / O Scheduler。

  • 在Windows上创建Oracle ACFS文件系统时，请以Windows域用户身份登录。此外，在Windows上的Oracle ACFS文件系统中创建文件时，您应该以Windows域用户身份登录，以确保所有节点都可以访问这些文件。

  • 在Windows平台上跨群集节点使用文件系统时，最佳做法是使用域用户安装文件系统，以确保跨群集节点的安全标识符相同。用于定义文件和目录的访问权限的Windows安全标识符使用标识用户的信息。本地用户仅在本地节点的上下文中已知。Oracle ACFS在第一个文件系统装入期间使用此信息来设置文件系统的默认访问权限。

    Oracle ACFS不支持与Oracle ASM管理相关的任何文件，例如Oracle Grid Infrastructure主目录和Oracle ASM诊断目录中的文件。

  • Oracle ACFS不支持Oracle Cluster Registry（OCR）和投票文件。

  • Oracle ACFS功能要求将ASM和ADVM的磁盘组兼容性属性设置为11.2或更高。

  • 要将Oracle ACFS文件系统用于Oracle数据库主目录，版本级别必须为Oracle 11g第2版（11.2）或更高版本。
