5.60 RC_TEMPFILE

此视图列出有关在恢复目录中注册的所有临时文件的信息。

它对应于V$TEMPFILE视图。如果删除了表空间，则临时文件显示为已删除。

柱 数据类型 描述

DB_KEY

数

目标数据库的主键。使用此列可以与几乎任何其他目录视图连接。

DBINC_KEY

数

目标数据库化身的主键。使用此列与RC_DATABASE_INCARNATION 。

DB_NAME

VARCHAR2（8）

此记录所属的数据库化身的DB_NAME 。

CON_ID

数

当前容器的ID：

  • 0：包含与整个多租户容器数据库（CDB）或传统Oracle数据库（非CDB）相关的数据的行

  • 1：包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

PDB_NAME

VARCHAR2（30）

恢复目录中可插拔数据库（PDB）的名称。

PDB_KEY

数

恢复目录中PDB的主键。

TS#

数

目标数据库中的表空间ID。如果删除并重新创建表空间，则TS#可能在同一个版本中存在多次。

TABLESPACE_NAME

VARCHAR2（30）

表空间名称。如果删除并重新创建表空间，则该名称可能在同一个版本中存在多次。

文件＃

数

临时文件的绝对文件号。临时文件的相同临时文件号可能存在，同时删除并重新创建临时文件。

CREATION_CHANGE＃

数

创建临时文件时的SCN。

CREATION_TIME

日期

创建临时文件的时间。

DROP_CHANGE＃

数

临时文件被删除时记录的SCN。如果发现具有相同FILE#的新临时文件，则对于临时文件，将DROP_CHANGE#设置为CREATION_CHANGE# ;否则，该值设置为RC_CHECKPOINT 。 CKP_SCN 。

DROP_TIME

日期

临时文件被删除的时间。如果发现具有相同FILE#的新临时文件，则将临时文件的DROP_TIME设置为CREATION_TIME ;否则值为RC_CHECKPOINT 。 CKP_TIME 。

BYTES

数

临时文件的大小（以字节为单位）。

积木

数

块中文件的大小。

BLOCK_SIZE

数

临时文件的块大小（以字节为单位）。

名称

VARCHAR2（1024）

临时文件名。

的RFile＃

数

表空间中此临时文件的相对文件号。

AUTOEXTEND

VARCHAR2（3）

如果临时文件是可自动扩展的，则为ON 。否则OFF 。

MAXSIZE

数

文件可以扩展到的块中的最大文件大小。仅在AUTOEXTEND为ON时有效。 AUTOEXTEND为OFF时始终为0 。

NEXTSIZE

数

可在块中扩展的文件的增量大小量。仅在AUTOEXTEND为ON时有效。 AUTOEXTEND为OFF时始终为0 。

BIGFILE

VARCHAR2（3）

如果表空间是bigfile表空间，则为YES ;否则， NO 。

SITE_KEY

数

与此文件关联的Data Guard数据库的主键。Data Guard环境中的每个数据库都具有唯一的SITE_KEY值。您可以使用SITE_KEY与联接RC_SITE以获得DB_UNIQUE_NAME的数据库。

TABLESPACE_CREATION_CHANGE＃

数

创建此表空间的SCN。

TABLESPACE_CREATION_TIME

日期

创建此表空间的时间戳。

TABLESPACE_DROP_CHANGE＃

数

删除此表空间的SCN。

TABLESPACE_DROP_TIME

日期

删除表空间的时间戳。

DB_UNIQUE_NAME

VARCHAR2（512）

此记录所属的数据库化身的DB_UNIQUE_NAME 。Data Guard环境中的所有数据库共享相同的DBID，但DB_UNIQUE_NAME值不同。当特定文件的数据库名称不可知时，此列中的值为null。例如，Oracle Database 11g之前由RMAN版本管理的数据库的行为空。
