5.157 DBA_LOGMNR_SESSION

DBA_LOGMNR_SESSION显示数据库中所有活动的LogMiner持久会话。

通过首次在逻辑备用数据库上启动Data Guard SQL Apply或创建复制捕获来创建持久LogMiner会话。

柱 数据类型 空值 描述

ID

数

不是NULL

唯一会话标识符

名称

VARCHAR2（128）

不是NULL

唯一的会话名称

SOURCE_DATABASE

VARCHAR2（128）

源数据库的全局名称，其归档日志将在此持久LogMiner会话中挖掘

SOURCE_DBID

数

源数据库的数据库ID

SOURCE_RESETLOGS_SCN

数

重置日志与源数据库的化身相关联的SCN，其源存档日志被挖掘

SOURCE_RESETLOGS_TIME

数

与归档日志被挖掘的源数据库的化身相关联的重置日志时间

FIRST_SCN

数

只能使用此持久会话挖掘在此SCN上或之后发生的修改

END_SCN

数

使用此持久会话无法在此SCN上或之后进行任何修改

BRANCH_SCN

数

SCN，根据与源数据库对应的化身，将采用分支。这意味着在此SCN的源数据库上执行了时间点恢复。

WAIT_FOR_LOG

VARCHAR2（3）

指示持久会话是等待RFS注册新的存档日志还是填充间隙（ YES ）或否（ NO ）

HOT_MINE

VARCHAR2（3）

指示实时挖掘是否开启（ YES ）否（ NO ）

SAFE_PURGE_SCN

数

持久会话可以安全地清除到此SCN

CHECKPOINT_SCN

数

SCN，持久LogMiner会话采用最新检查点

PURGE_SCN

数

该会话已被清除至此SCN
