A.1运行通用Oracle ORAchk和Oracle EXAchk命令

Oracle ORAchk和Oracle EXAchk常用的命令选项列表。

句法

[-a] [-v] [-debug] [-nodaemon] [-f] [-upgrade] [-ngradegrade] [-testemail all | “NOTIFICATION_EMAIL = comma-delimited list of email addresses ”] [-sendemail“NOTIFICATION_EMAIL = comma-delimited list of email addresses ”] [-dbserial] [-dbparallel [ n ]] [-dbparallelmax]

参数

表A-1通用命令

选项 描述

-一个

运行所有检查，包括最佳实践检查和推荐的补丁检查。如果未指定任何选项，则默认情况下工具会运行所有检查。

-v

显示Oracle ORAchk和Oracle EXAchk工具的版本。

-debug

在调试模式下运行。

生成的.zip文件包含调试日志和其他对Oracle Support有用的文件。

-nodaemon

不向守护程序发送命令，用法是交互式的。

-F

离线运行。这些工具对已从系统收集的数据执行运行状况检查。

-升级

强制升级正在运行的工具版本。

-noupgrade

即使在RAT_UPGRADE_LOC环境变量中指定的位置下有更高版本，也不会提示升级。

-testemail all | “NOTIFICATION_EMAIL =以comma-delimited list of email addresses ”

发送测试电子邮件以验证电子邮件配置。

-sendemail“NOTIFICATION_EMAIL =以comma-delimited list of email addresses ”

指定以逗号分隔的电子邮件地址列表。

在完成时将生成的HTML报告通过电子邮件发送到指定的电子邮件地址。

-dbserial

以串行方式运行SQL ， SQL_COLLECT和OS状况检查。

-dbparallel [ n ]

使用n个子进程并行运行SQL ， SQL_COLLECT和OS状况检查。

默认值为CPU的25％。

-dbparallelmax

使用最大数量的子进程并行运行SQL ， SQL_COLLECT和OS状况检查。
