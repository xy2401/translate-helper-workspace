关于具有Unbreakable Enterprise Kernel的Oracle Linux

面向Oracle Linux的Unbreakable Enterprise Kernel为从数据中心运行Oracle Linux的客户提供了从上游开发的最新创新。

从Oracle Linux 5 Update 6及更高版本的Oracle Linux内核开始，默认情况下包含并启用了Unbreakable Enterprise Kernel。它基于最近稳定的主线开发Linux内核，还包括与Oracle数据库，Oracle中间件和Oracle硬件工程团队合作开发的优化，以确保最苛刻的企业工作负载的稳定性和最佳性能。

Oracle强烈建议在Oracle Linux环境中部署Unbreakable Enterprise Kernel，尤其是在运行企业应用程序时。但是，使用Unbreakable Enterprise Kernel是可选的。如果您需要严格的Red Hat Enterprise Linux（RHEL）内核兼容性，那么Oracle Linux还包括一个与RHEL Linux内核兼容的内核，直接从RHEL源代码编译。

您可以通过以下URL获取有关适用于Oracle Linux的Unbreakable Enterprise Kernel的更多信息：

http://www.oracle.com/us/technologies/linux

适用于Oracle Linux的Unbreakable Enterprise Kernel是Oracle产品使用的标准内核。Oracle数据库和其他Oracle产品的构建和QA系统仅使用Unbreakable Enterprise Kernel for Oracle Linux。适用于Oracle Linux的Unbreakable Enterprise Kernel也是Oracle Exadata和Oracle Exalogic系统中使用的内核。适用于Oracle Linux的坚不可摧的企业内核用于Oracle参与的Linux上的所有基准测试，以及适用于x86-64的Oracle预安装RPM程序。

Oracle Ksplice是Oracle Linux的一部分，它在运行时更新Linux操作系统（OS）内核，无需重启或中断。Ksplice仅适用于Oracle Linux。
