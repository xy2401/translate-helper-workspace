以下是Oracle Database Oracle9 i Release中提供的XML C API与后续Oracle数据库中可用的当前Unified XML C API之间的映射。

也可以看看：

Oracle XML Developer's Kit程序员指南中的格式模型

C包更改

可以通过oraxml包获得预先存在的C API。它具有以下特点：

  • 规范仅限于xml上下文（ xmlctx ）和xml文档之间的一对一映射。DOM在任何时候都只能访问一个文档，但是多个文档的数据可以是并发的。

  • API并不总是一致的，并不总是遵循xmlctx的声明。

相比之下，新的统一C API解决了这些问题：

  • 多个独立文档共享xmlctx 。

  • 所有API都符合xmlctx的声明。

  • DOM可以同时访问每个文档，直到被XmlDestroy()调用明确销毁。

初始化和解析序列更改

Unified C API中的文档初始化和解析已更改。

示例A-1预先统一的C API的初始化和解析序列，一次一个文档

示例A-3预统一C API，多文档和同时DOM访问的初始化和解析序列

以下伪代码演示了如何使用旧的C API同时进行DOM访问来初始化和解析多个文档。将其与实施例A-4进行对比。
