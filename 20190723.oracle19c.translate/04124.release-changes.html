Oracle Database 19c中的更改

以下是Oracle Database JDBC Developer for Oracle Database 19c 指南中的更改。

新功能

以下功能是此版本中的新增功能：

  • 用于Reactive Streams Ingestion的Java库

    请参阅支持Reactive Streams Ingestion

  • 增强应用程序连续性

    请参阅支持透明应用程序连续性

  • 支持Easy Connect命名语法（EZConnect格式URL）改进

    从Oracle Database Release 19c开始，扩展了简单连接命名语法，连接标识符现在具有以下格式：

    [[protocol：] //] host1 [，host12，host13] [：port1] [，host2：port2] [/ service_name] [：server] [/ instance_name] [？[WALLET_LOCATION = <DIR>] [＆ssl_server_cert_dn = “LongDN”]，...]

    JDBC支持此增强功能，您可以使用以下JDBC URL：

    jdbc：oracle：thin：@tcps：//example1.com:1522 / wkrfs4xeqva1jcu_sqldev_example2.com？WALLET_LOCATION =。

    也可以看看：

    轻松连接命名增强功能
  • 从Oracle Database Release 19c开始，可以在JDBC URL中设置所有JDBC属性。例如，您可以使用以下JDBC URL来启用隐式语句高速缓存大小并将其值设置为100：
    jdbc：oracle：thin：@（description =（address =（protocol = tcps）（port = 1522）（host = example1.com））（connect_data =（service_name = example2.com）））？oracle.jdbc.implicitStatementCacheSize = 100
  • 支持基于主机名的DN验证

    主机名验证是HTTPS协议的一部分，涉及服务器身份检查，以确保客户端与正确的服务器通信，并且未作为中间攻击的一部分重定向。从Oracle Database 19c发行版开始，JDBC驱动程序支持基于主机名的DN验证。

    如果oracle.net.ssl_server_dn_match参数设置为true ，并且如果URL中未提供ssl_server_cert_dn参数值，则驱动程序使用服务器的主机名验证服务器证书中的DN。服务器DN匹配用于SSL握手期间的相互身份验证。您可以通过以下方式进行设置：

    oracle.net.ssl_server_dn_match =真正的系统属性

    注意：

    基于主机名DN验证默认情况下为EZCONNECT格式的URL启用后，如果您指定的协议作为tcps中的URL。

不推荐的功能

不推荐使用以下功能，并且可能在将来的版本中不支持这些功能：

  • 从下列API OraclePreparedStatement和OracleCallableStatement类已经被弃用的增强了对关联数组支持部分：

    • setPlsqlIndexTable

    • setPlsqlIndexTableAtName

    • registerIndexTableOutParameter

    • getOraclePlsqlIndexTable

    • getPlsqlIndexTable

  • oracle.sql包中的具体类

    不推荐使用oracle.sql包中的具体类。使用新的JDBC接口而不是这些类。

    有关这些接口的更多信息，请参见MoS Note 1364193.1。

  • 不推荐使用oracle.jdbc.rowset包。Oracle建议您使用标准JDBC RowSet包来替换此功能。

    也可以看看：

    http://docs.oracle.com/javase/9/docs/api/javax/sql/rowset/package-summary.html

  • defineColumnType方法

    不推荐使用defineColumnType方法的大多数变体。支持的变体适用于：

    • LOB到LONG转换

    • 配置LOB预取大小

    有关更多信息，请参阅JDBC Javadoc。

  • CONNECTION_PROPERTY_STREAM_CHUNK_SIZE属性

    有关更多信息，请参阅JDBC Javadoc。

  • Oracle Update批处理

    Oracle Database 12 c第1版（12.1）中不推荐使用Oracle更新批处理。自Oracle Database 12 c第2版（12.2）以来，Oracle更新批处理是一个无操作代码（无操作）。这意味着，如果在应用程序中使用Oracle Database 19c JDBC驱动程序实施Oracle更新批处理，则不会设置指定的批处理大小，并导致批处理大小为1。使用此批处理设置，您的应用程序一次处理一行。如果您使用的是Oracle Database 19c JDBC驱动程序，Oracle强烈建议您使用标准JDBC批处理。

    有关详细信息，请参阅标准更新批处理部分。

  • EndToEndMetrics相关的API

    自Oracle Database 12 c第 2版（12.2）以来，不推荐使用与EndToEndMetrics相关的API。

    有关更多信息，请参阅JDBC DMS指标 。
