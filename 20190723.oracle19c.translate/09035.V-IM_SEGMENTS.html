8.2 V $ IM_SEGMENTS

V$IM_SEGMENTS显示有关数据库中所有内存V$IM_SEGMENTS信息。

仅显示具有内存中表示的段。如果为内存列存储（IM列存储）标记了一个段但未填充，则此视图中不会显示该段的相应行。

柱 数据类型 描述

所有者

VARCHAR2（128）

细分所有者的用户名

SEGMENT_NAME

VARCHAR2（128）

段的名称（如果有）

PARTITION_NAME

VARCHAR2（128）

对象分区名称（对于非分区对象，设置为NULL）

段类型

VARCHAR2（18）

细分类型：

  • 表

  • 表格分区

  • 表格提交

TABLESPACE_NAME

VARCHAR2（30）

包含段的表空间的名称

INMEMORY_SIZE

数

段的内存中版本的大小（以字节为单位）

BYTES

数

可以在内存中表示的段的磁盘数据字节数（无空间元数据块）

BYTES_NOT_POPULATED

数

未在内存中填充的磁盘段部分的大小（以字节为单位）。

POPULATE_STATUS

VARCHAR2（9）

分部人口状况：

  • STARTED ：表示已启动细分填充任务

  • COMPLETED ：表示没有任何段填充任务处于挂起状态

  • FAILED ：表示段填充任务失败

INMEMORY_PRIORITY

VARCHAR2（8）

表示IM列存储填充的优先级：

  • 低

  • 介质

  • 高

  • 危急

  • 没有

INMEMORY_DISTRIBUTE

VARCHAR2（15）

指示IM列存储在Oracle Real Application Clusters（Oracle RAC）环境中的分布方式：

  • 汽车

  • 通过ROWID RANGE

  • 通过分区

  • 通过提交

INMEMORY_DUPLICATE

VARCHAR2（13）

表示Oracle RAC环境中IM列存储的重复设置：

  • 没有重复

  • 重复

  • 全部重复

INMEMORY_COMPRESSION

VARCHAR2（17）

IM列存储的压缩级别：

  • 没有MEMCOMPRESS

  • 对于DML

  • FOR QUERY [LOW |高]

  • 容量[LOW |高]

INMEMORY_SERVICE

VARCHAR2（12）

指定如何在各种实例上填充启用IM的表：

  • DEFAULT ：Oracle数据库12c第2版（12.2.0.1）之前的行为

  • NONE ：不填充任何实例

  • ALL ：填充所有实例

  • USER_DEFINED ：仅在用户指定的服务处于活动状态的实例上填充。与此对应的服务名称存储在INMEMORY_SERVICE_NAME列中。

INMEMORY_SERVICE_NAME

VARCHAR2（129）

指定应在其上填充启用IM的表的服务名称。仅当相应的INMEMORY_SERVICE列具有值USER_DEFINED时，此列才具有值。

IS_EXTERNAL

VARCHAR2（5）

指示IM段是否用于外部表。可能的值：

  • TRUE ：IM段用于外部表。

  • FALSE ：IM段不适用于外部表。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个多租户容器数据库（CDB）相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
