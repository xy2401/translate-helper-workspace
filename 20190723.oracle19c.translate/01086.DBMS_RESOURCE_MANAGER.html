136.4.2 CALIBRATE_IO过程

此过程可校准存储的I / O功能。校准状态可从V$IO_CALIBRATION_STATUS视图获得，成功校准运行的结果位于DBA_RSRC_IO_CALIBRATE表中。

句法

DBMS_RESOURCE_MANAGER.CALIBRATE_IO（num_physical_disks IN PLS_INTEGER DEFAULT 1，max_latency IN PLS_INTEGER DEFAULT 20，max_iops OUT PLS_INTEGER，max_mbps OUT PLS_INTEGER，actual_latency OUT PLS_INTEGER）;

参数

表136-3 CALIBRATE_IO过程参数

参数 描述

num_physical_disks

数据库存储中的近似物理磁盘数。此参数用于确定校准运行的初始I / O负载。

max_latency

数据库块大小的IO请求的最大容忍延迟（以毫秒为单位）

max_iops

每秒可以维持的最大I / O请求数。I / O请求是随机分布的，数据库块大小的读取。

max_mbps

可持续的I / O最大吞吐量，以每秒兆字节数表示。I / O请求是随机分布的，1兆字节读取。

actual_latency

数据块大小的I / O请求的平均延迟，以max_iops速率表示，以毫秒为单位

使用说明

  • 只有具有SYSDBA权限的用户才能运行此过程。合格用户还必须打开timed_statistics ，并确保为数据文件启用了asynch_io 。这可以通过将filesystemio_options设置为ASYNCH或SETALL来实现。也可以通过以下SQL语句查询asynch_io状态：

    列名格式a50 SELECT名称，asynch_io FROM v $ datafile f，v $ iostat_file i WHERE f.file＃= i.file_no AND filetype_name ='Data File'/

  • 一次只能运行一次校准。如果同时启动另一个校准，它将失败。

  • 对于Oracle Real Application Clusters（Oracle RAC）数据库，将同时从所有实例生成工作负载。

  • 在多租户容器数据库（CDB）中，只能从CDB根目录（ CDB$ROOT ）运行校准。

  • 校准对数据库性能极具破坏性。强烈建议仅在数据库用户可以容忍数据库性能严重恶化时才运行校准。

  • 为获得最佳校准结果，不应运行其他数据库工作负载。

用户评论

分享

指定PDB的资源分配份额。通过为每个PDB指定共享来启用CPU资源管理器和Exadata I / O资源管理器。 shares参数也用于并行语句队列。如果未指定共享，则通过UPDATE_CDB_DEFAULT_DIRECTIVE过程指定的default指令获取缺省值。

指定PDB可以使用的CDB的CPU和Exadata I / O资源的最大百分比。

还可以通过为PDB设置CPU_COUNT参数来限制CPU资源管理器和Exadata I / O资源管理器。

parallel_server_limit

并行服务器，PDB可以使用并行语句排队。或者，您可以在PDB级别设置parallel_servers_target 。

PDB可以为并行执行服务器设置下限，而不是CDB资源计划中指定的限制。在PDB中设置PARALLEL_SERVERS_TARGET初始化参数，并为CDB资源计划中的PDB指定并行执行服务器限制时，将使用下限。

例如，假设在CDB根目录中parallel_servers_target初始化参数设置为100 ，并且对于CDB资源计划中的hrpdb ， parallel_server_limit设置为70。另外，假设hrpdb的parallel_servers_target初始化参数设置为50。在这种情况下，用于hrpdb用于并行处理服务器的上限为50，因为50是比70 CDB资源计划下限hrpdb 。

注意：

136.4.9 CREATE_PENDING_AREA过程

此过程对资源管理器对象进行更改。

对计划模式的所有更改都必须在待处理区域内完成。待处理区域可以被视为计划架构更改的“临时”区域。管理员创建此挂起区域，根据需要进行更改，可能验证这些更改，并且只有在提交完成后，这些更改才会生效。

句法

DBMS_RESOURCE_MANAGER.CREATE_PENDING_AREA;

使用说明

您可以在待处理区域处于活动状态时随时通过从相应的用户视图中进行选择来查看包含更改的当前计划架构。

如果要停止当前更改，您可以随时清除待处理区域。您也可以调用VALIDATE过程来确认您所做的更改是否有效。您不必按给定顺序执行更改以维护一致的条目组。提交待处理区域时，也会隐式执行这些检查。

注意：

Oracle允许“孤儿”消费者群体（换句话说，没有计划指令的消费者群体引用它们）。这是预期管理员可能想要创建当前未使用但将来将使用的消费者组。

对于资源计划，必须遵守以下规则，并在执行验证或提交过程时检查它们：

  • 没有计划架构可能包含任何循环。

  • 计划指令所指的所有计划和消费者群体必须存在。

  • 所有计划必须具有涉及计划或消费者组的计划指令。

  • 对于重点资源分配方法，任何给定级别的所有百分比不得超过100。

  • 当前被活动实例用作顶级计划的计划不得删除。

  • plan指令参数parallel_degree_limit_p1可能只出现在引用消费者组的计划指令中（即不在子计划中）。

  • 来自任何给定计划的计划指令不得超过28个（即，任何计划都不能超过28个孩子）。

  • 任何活动计划架构中的消费者群组不得超过28个。

  • 计划和使用者组使用相同的命名空间;因此，任何计划都不能与任何消费者群体具有相同的名称。

  • 任何活动计划架构中的某处都必须有OTHER_GROUPS的计划指令。这可确保为当前活动计划未涵盖的会话分配OTHER_GROUPS指令指定的资源。

注意：

这些规则不适用于CDB资源计划。

并行服务器限制。设置此项将覆盖utilization_limit设置的并行服务器的限制。

new_utilization_limit

资源限制。对于CPU，这会限制使用者组的CPU利用率。对于并行服务器，这限制了用作百分比的并行服务器parallel_servers_target 。

经过时间将触发switch_group指定的switch_group 。与其他开关指令一样，如果new_switch_for_call为TRUE ，则从通话开始累计经过的时间。否则，在会话长度内累计经过的时间。

如果设置为QUEUE ，则来自使用者组的所有并行语句（无论parallel_degree_policy参数值）都会排队。

默认值为FALSE ，这意味着某些并行语句符合排队条件，具体取决于parallel_degree_policy参数值。

new_session_pga_limit

此使用者组中的会话在终止之前可以分配的最大PGA（MB）。NULL （默认值）表示没有更改。

new_parallel_queue_timeout

指定并行语句在删除之前可能保留在其使用者组的并行语句队列中的时间（以秒为单位）。

此参数的默认操作是ERROR 。可以使用new_pq_timeout_action参数更改此操作。

new_pq_timeout_action

指定由于new_parallel_queue_timeout而从队列中删除并行语句时要采取的操作。

价值观是：

  • CANCEL - SQL语句以ORA-7454错误终止

  • RUN - SQL语句立即运行，如果并行服务器不可用，可能会降级

使用说明

  • 如果未指定UPDATE_PLAN_DIRECTIVE的参数，则它们在数据字典中保持不变。

  • 对于new_max_idle_time和new_max_idle_blocker_time ， PMON将每分钟检查一次这些限制。如果它找到超过其中一个限制的会话，它将强制终止会话并清除其所有状态。

  • 参数new_switch_time_in_call主要用于中间层服务器实现会话池的三层应用程序。通过启用new_switch_time_in_call ，一个客户端的资源使用量不会影响恰好在同一会话上执行的未来客户端的使用者组。

  • 要清除（重置为指令的默认值），请使用值-1 。
