14 SEM_RDFSA包子程序

SEM_RDFSA包中包含子程序（函数和过程），用于使用Oracle Label Security（OLS）为RDF数据提供细粒度的访问控制。

要使用本章中的子程序，您应该了解RDF语义图概述和RDF数据细粒度访问控制中的概念和用法信息。

本章提供有关子程序的参考信息，按字母顺序列出。

14.1 SEM_RDFSA.APPLY_OLS_POLICY

格式

SEM_RDFSA.APPLY_OLS_POLICY（policy_name IN VARCHAR2，rdfsa_options IN NUMBER DEFAULT SEM_RDFSA.SECURE_SUBJECT，table_options IN VARCHAR2 DEFAULT'ALL_CONTROL'，label_function IN VARCHAR2 DEFAULT NULL，谓词IN VARCHAR2 DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL） ;

描述

将OLS策略应用于语义数据存储。

参数

policy_name

现有OLS策略的名称。

rdfsa_options

指定要为RDF数据启用的细粒度访问控制模式的选项。保护RDF数据的默认选项包括为出现三元组主题位置的资源分配敏感度标签。您可以使用rdfsa_options参数覆盖默认值，并指定使用说明中表14-1中定义的常量之一。

table_options

政策执行选项。默认值（ ALL_CONTROL ）是此过程唯一支持的值。

label_function

一个字符串，用于调用函数以返回标签值以用作默认值。

谓词

与基于标签的谓词结合使用的附加谓词。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

必须使用CTXT1作为列名创建使用此过程指定的OLS策略，并且它应使用默认策略选项。有关策略选项的信息，请参阅“ Oracle Label Security管理员指南” 。

此过程在MDSYS模式中定义的多个表上调用sa_policy_admin.apply_table_policy过程。SEM_RDFSA.APPLY_OLS_POLICY过程的参数table_options ， label_function和predicate与sa_policy_admin.apply_table_policy过程中具有相同名称的参数具有相同的语义。

对于rdfsa_options参数，可以为所需选项指定包常量。表14-1列出了这些常量及其说明。

表14-1 rdfsa_options参数的SEM_RDFSA包常量

不变 描述

SEM_RDFSA.SECURE_SUBJECT

为出现三元组主题位置的资源指定敏感度标签。

SEM_RDFSA.SECURE_PREDICATE

为出现三元组谓词位置的资源指定敏感度标签。

SEM_RDFSA.SECURE_OBJECT

为出现三元组对象位置的资源指定敏感度标签。

SEM_RDFSA.TRIPLE_LEVEL_ONLY

应用三级安全性。提供良好的性能，无需为各个资源分配标签。（需要安装My Oracle Support提供的Patch 9819833。）

SEM_RDFSA.OPT_DEFINE_BEFORE_USE

在为资源定义灵敏度标签之前，限制在三元组中使用RDF资源。如果未指定此选项，则用户的初始行标签将在首次使用时用作资源的默认标签。

SEM_RDFSA.OPT_RELAX_TRIPLE_LABEL

放宽三重标签与与其所有组件相关的标签之间存在的主导关系。使用此选项，如果用户具有对所有三元组件的READ访问权限，并且三元标签可能与组件标签没有任何关系，则可以定义三元组。如果没有此选项，三重标签应至少覆盖其所有组件的标签。

您可以在label_function参数中指定一个函数，以便为新插入的三元组生成自定义标签。label函数与MDSYS.RDF_LINK $表关联，此表中的列可以配置为label函数的参数，如以下示例所示：

fgac_admin.new_triple_label（：new.model_id，：new.start_node_id，：new.p_value_id，：new.canon_end_node_id）'

由于OLS策略应用于具有不同结构的多个表，因此分配给predicate参数的任何谓词中唯一有效的列引用是标签列：CTXT1。如果为具有现有数据的语义数据存储启用OLS，则可以指定'OR CTXT1 is null'形式的谓词，以便能够继续使用此数据而不受访问限制。

支持OLS的语义数据存储使用针对在多个模型中组织的所有RDF三元组的敏感性标签。用户通过模型视图和SEM_MATCH查询访问此类三元组受OLS策略的限制。此外，独立于拥有应用程序表的用户，对表中的三列（类型为SDO_RDF_TRIPLE_S）的访问仅限于具有OLS策略的FULL访问权限的用户。

使用相应应用程序表上的INSERT权限将三元组插入到特定的RDF模型中。使用用户的会话上下文（初始行标签）或标签函数生成新三元组的敏感度标签。使用与三元组件关联的标签验证三元组是否存在任何RDF策略违规。虽然可能无法通过应用程序表访问三重信息，但可以查询模型视图以访问三元组，同时强制执行OLS策略限制。如果您具有必要的策略权限（例如writeup，writeacross），则可以更新模型视图中的CTXT1列以重置分配给该三元组的标签。对于涉及三重组件的任何RDF策略违规，新标签会自动进行验证。模型视图的CTXT1列的更新权限被授予模型的所有者，并且此用户可以选择性地将此权限授予其他用户。

如果RDF模型是在具有FULL访问权限的用户之外的模式中创建的，则对模型对象具有必要的权限 - 特别是对应用程序表的读/写访问权限，对模型视图的读访问权以及对CTXT1列的写访问权。模型视图 - 可以授予此类用户进行维护操作。这些操作包括批量加载到模型中，重置分配给三元组的任何敏感标签，以及使用模型创建蕴涵。

要禁用OLS策略，请使用SEM_RDFSA.DISABLE_OLS_POLICY过程。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例使用安全主题和安全谓词选项启用对RDF数据的安全访问。

开始sem_rdfsa.apply_ols_policy（policy_name =>'defense'，rdfsa_options => sem_rdfsa。SECURE_SUBJECT + sem_rdfsa。SECURE_PREDICATE）;结束; /

以下示例通过指定“使用前定义”选项扩展了前面的示例，该选项允许用户仅在使用关联的敏感度标签预定义三重组件（主题，谓词或对象）时才定义三元组。如果插入三元组的用户没有SEM_RDFSA包的执行权限，则此配置有效。

开始sem_rdfsa.apply_ols_policy（policy_name =>'defense'，rdfsa_options => sem_rdfsa。SECURE_SUBJECT + sem_rdfsa。SECURE_PREDICATE + sem_rdfsa。OPT_DEFINE_BEFORE_USE）;结束; /

14.2 SEM_RDFSA.DISABLE_OLS_POLICY

格式

SEM_RDFSA.DISABLE_OLS_POLICY（network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

禁用先前已应用于语义数据存储或在语义数据存储上启用的OLS策略。

参数

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

您可以使用此过程临时禁用已应用于语义数据存储或为语义数据存储启用的OLS策略。禁用策略的用户应具有管理OLS策略所需的权限，并且还应具有应用于RDF数据的OLS策略的访问权限。

分配给各种RDF资源和三元组的敏感性标签将被保留，并且可以重新启用OLS策略以强制执行它们。可以添加具有特定标签的新资源，或者在禁用OLS策略时可以更新现有三元组和资源的标签。

要应用OLS策略，请使用SEM_RDFSA.APPLY_OLS_POLICY过程;要启用已禁用的OLS策略，请使用SEM_RDFSA.ENABLE_OLS_POLICY过程。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例禁用语义数据存储的OLS策略。

开始sem_rdfsa.disable_ols_policy;结束; /

14.6 SEM_RDFSA.SET_PREDICATE_LABEL

格式

SEM_RDFSA.SET_PREDICATE_LABEL（model_name IN VARCHAR2，谓词IN VARCHAR2，label_string IN VARCHAR2，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

为模型级别或整个存储库的谓词设置敏感度标签。

参数

model_name

谓词所属模型的名称，或者如果在所有模型中应用相同标签以使用谓词， RDF$GLOBAL字符串RDF$GLOBAL 。

谓词

应为其分配标签的谓词。

label_string

字符串表示中的OLS行标签。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

如果指定了型号名称，则必须具有对模型的读访问权限并对SEM_RDFSA包执行权限才能执行此操作。如果指定RDF$GLOBAL ，则必须具有应用于RDF数据的OLS策略的FULL访问权限。

如果谓词已存在标签，则必须有权访问指定的标签和OLS策略权限才能覆盖现有标签。必须为RDF数据启用SECURE_PREDICATE选项。

如果使用此操作更新现有谓词标签，则使用此谓词的三元组标签必须全部支配新谓词标签。唯一的例外是为启用OLS的RDF数据选择OPT_RELAX_TRIPLE_LABEL选项。

如果指定RDF$GLOBAL ，则会创建跨模型具有唯一灵敏度标签的全局谓词。如果先前在一个或多个模型中定义了相同的谓词，则全局标签将支配所有此类标签，并且针对给定谓词替换特定于模型的标签。

在设置了谓词的标签之后，只有当三元标签（可以从用户的初始行标签或使用标签函数初始化）支配谓词的敏感性标签时，才能添加带谓词的新三元组。可以使用OPT_RELAX_TRIPLE_LABEL选项放宽此优势关系，在这种情况下，用户应至少具有对谓词的读访问权，以便能够使用谓词定义新的三元组。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例为Contracts模型设置谓词标签，为数据库实例中的所有模型设置另一个谓词标签。 

开始sem_rdfsa.set_predicate_label（model_name =>'contracts'，谓词=>'<http://www.myorg.com/pred/hasContractValue>'，label_string =>'TS：US_SPCL'）;结束; / begin sem_rdfsa.set_predicate_label（model_name =>'rdf $ global'，predicate =>'<http://www.myorg.com/pred/hasStatus>'，label_string =>'SE：US_SPCL：US'）;结束; /

14.7 SEM_RDFSA.SET_RDFS_LABEL

格式

SEM_RDFSA.SET_RDFS_LABEL（label_string IN VARCHAR2，inf_override IN VARCHAR2，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

为RDFS架构元素设置敏感度标签。

参数

label_string

字符串表示形式的OLS行标签，用作所有RDF模式构造的敏感性标签。

inf_override

OLS行标签，用作为推断的三元组生成标签的覆盖。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

此过程设置或重置与RDF模式资源关联的敏感性标签，通常由http://www.w3.org/1999/02/22-rdf-syntax-ns#和http://www.w3.org/2000/01/rdf-schema#识别http://www.w3.org/2000/01/rdf-schema#前缀为其URI。您可以为具有对这些资源的受限访问权限分配敏感度标签，以便可以将具有更高权限的用户限制创建新RDF类和添加新属性等操作。

您必须具有应用于RDF数据的策略的FULL访问权限。

RDF模式元素隐式使用宽松的三元标签选项，因此使用主题，谓词或对象的RDFS和OWL构造的三元组不会被强制使用敏感标签来支配与模式构造关联的标签。因此，能够定义新RDF类和属性的用户必须至少具有对schema元素的读访问权。

当在推断的三元组中引用RDF模式元素时，系统定义的标签生成器和自定义标签生成器在确定推断的三元组的适当标签时考虑推理覆盖标签。如果使用自定义标签生成器，则在涉及RDF架构元素时，将传递此覆盖标签而不是实际标签。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例为所有RDF架构元素设置具有唯一隔离专区的标签。能够定义新RDF类和属性的用户应该拥有该隔离专用成员资格。

开始sem_rdfsa.set_rdfs_label（label_string =>'SE：RDFS：'，inf_override =>'SE：US_SPCL：US'）;结束; /

14.8 SEM_RDFSA.SET_RESOURCE_LABEL

格式

SEM_RDFSA.SET_RESOURCE_LABEL（model_name IN VARCHAR2，resource_uri IN VARCHAR2，label_string IN VARCHAR2，resource_pos IN VARCHAR2 DEFAULT'S'，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

为可以在三元组的主题和/或对象位置中使用的资源设置敏感度标签。

参数

model_name

资源所属模型的名称，或者如果应用相同标签以在所有模型中使用资源， RDF$GLOBAL字符串RDF$GLOBAL 。

resource_uri

可以在一个或多个三元组中用作主题或对象的资源的URI。

label_string

字符串表示中的OLS行标签。

resource_pos

资源在三元组中的位置： S ， O或S,O 。您可以为同一资源指定最多两个单独的标签，一个在资源用于三元组的主题位置时要考虑，另一个在对象位置出现时要考虑。值' S '，' O '或' S,O '分别为主题，对象或主题和对象位置设置资源标签。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

如果指定了型号名称，则必须具有对模型的读访问权限并对SEM_RDFSA包执行权限才能执行此操作。如果指定RDF$GLOBAL ，则必须具有应用于RDF数据的OLS策略的FULL访问权限。

如果谓词已存在标签，则必须有权访问指定的标签和OLS策略权限才能覆盖现有标签。必须为RDF数据启用SECURE_PREDICATE选项。

如果使用此操作更新现有资源标签，则在指定位置使用此资源的三元组的标签必须全部支配新资源标签。唯一的例外是为启用OLS的RDF数据选择OPT_RELAX_TRIPLE_LABEL选项。

如果指定RDF$GLOBAL ，则会创建跨模型具有唯一敏感性标签的全局资源。如果先前在具有相同三重位置的一个或多个模型中定义了相同的资源，则全局标签支配所有此类标签，并且针对该位置中的给定资源替换特定于模型的标签。

设置谓词标签后，只有当三元标签支配资源的敏感性标签时，才能添加使用指定位置资源的新三元组。可以使用OPT_RELAX_TRIPLE_LABEL选项放宽此优势关系，在这种情况下，用户至少应具有对资源的读访问权。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例根据位置为多个资源设置敏感度标签。

14.9 SEM_RDFSA.SET_RULE_LABEL

格式

SEM_RDFSA.SET_RULE_LABEL（rule_base IN VARCHAR2，rule_name IN VARCHAR2，label_string IN VARCHAR2，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

为属于规则库的规则设置敏感度标签。

参数

rule_base

现有RDF规则库的名称。

rule_name

属于规则库的规则的名称。

label_string

字符串表示中的OLS行标签。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

分配给规则的灵敏度标签用于在选择适当的标签生成器选项时为推断的三元组生成标签。

您必须具有访问权限才能访问规则库，并且您必须具有FULL访问权限，并且OLS策略可以为RDFS规则库中的系统定义规则分配标签。

不支持分配给用户定义规则的标签。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例为RDFS规则分配敏感度标签。

开始sem_rdfsa.set_rule_label（rule_base =>'RDFS'，rule_name =>'RDF-AXIOMS'，label_string =>'SE：US_SPCL：'）;结束; /
