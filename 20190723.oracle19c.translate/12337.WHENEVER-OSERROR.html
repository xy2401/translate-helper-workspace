12.52 WHEREVER OSERROR

句法

WHERVER OSERROR {EXIT [ SUCCESS |失败| n | 变量 | ：BindVariable ] [ COMMIT | ROLLBACK] |继续[COMMIT | ROLLBACK | 没有 ]}

施行如果操作，则指定的操作（默认退出SQL * Plus） 系统错误发生（例如文件写入错误）。

条款

[成功|失败| n | 变量 | ：BindVariable ]

一旦检测到操作系统错误，就指示SQL * Plus执行指定的操作。您还可以指定SQL * Plus返回成功或失败代码，操作系统故障代码或您选择的数字或变量。

退出[成功]失败| n | 变量 | ：BindVariable ]

一旦检测到操作系统错误，就指示SQL * Plus退出。您还可以指定SQL * Plus返回成功或失败代码，操作系统故障代码或您选择的数字或变量。有关更多信息，请参阅EXIT 。 

继续

关闭EXIT选项。 

承诺

指示SQL * Plus在退出或继续之前执行COMMIT并将挂起的更改保存到数据库。 

ROLLBACK

指示SQL * Plus在退出或继续之前执行ROLLBACK并放弃对数据库的挂起更改。 

没有

指示SQL * Plus在继续之前不执行任何操作。

用法

如果未输入WHENEVER OSERROR命令，则SQL * Plus的默认行为是继续，并且在发生操作系统错误时不执行任何操作。

如果未输入WHENEVER SQLERROR命令，则SQL * Plus的默认行为是继续，并在发生SQL错误时不执行任何操作。

例子

如果从输出文件读取时发生故障，则执行以下脚本中的命令导致SQL * Plus退出并执行任何挂起的更改：

WHERVER OSERROR EXIT START no_such_file
