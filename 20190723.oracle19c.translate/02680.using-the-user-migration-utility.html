将本地或外部用户迁移到企业用户的好处

从数据库用户模型迁移到企业用户模型可为企业环境中的管理，安全性和可用性挑战提供解决方案。在企业用户模型中，所有用户信息都将移至LDAP目录服务。

企业用户安全性通过提供以下好处，提供了轻松安全地管理企业级用户的能力：

  • 在符合LDAP版本3的目录服务器中集中存储用户凭据，角色和特权

  • 提供基础架构，以使用符合X.509v3的证书实现单点登录，通常在需要端到端SSL的情况下部署

  • 通过更及时的维护和更少的用户密码增强安全性

    企业用户模型中固有的用户信息的集中化使其更易于管理。安全管理员可以立即对用户信息进行必要的维护更改，从而更好地控制对关键网络资源的访问。此外，用户发现企业用户模型更易于使用，因为他们需要记住的密码更少。因此，他们不太可能选择容易猜到的密码或将其写下别人可以复制的密码。

用户迁移实用程序简介

用户迁移实用程序是一个命令行实用程序，使企业用户管理员可以将其用户从本地数据库模型移动到企业用户模型。您可以轻松地将数千个本地和外部数据库用户迁移到LDAP目录中的企业用户环境，以便从中央位置管理它们。该实用程序使用Oracle JDBC OCI驱动程序连接到数据库。

企业用户管理员可以选择迁移数据库中以下用户子集的任意组合：

  • 在命令行或文件中指定的用户列表

  • 所有外部用户

  • 所有全球用户

此外，企业用户管理员可以指定用于确定用户迁移方式的实用程序参数的值，例如

  • 将迁移的用户放在LDAP目录树中的位置

  • 将具有多个帐户的用户映射到单个目录用户条目

以下部分介绍了迁移过程以及用户模式发生的更改。

注意：

迁移外部用户后，其外部身份验证和授权机制将替换为基于目录的机制。如果迁移的用户映射到新创建的目录条目，则会随机生成新密码。

步骤0：关于使用安全外部密码存储区

在运行用户迁移实用程序之前，请将客户端Oracle钱包配置为安全外部密码存储，以便应用程序可以使用存储在钱包中的密码凭据连接到数据库。

在客户端Oracle钱包中存储数据库密码凭证，无需在应用程序代码，批处理作业或脚本中嵌入密码。这样可以降低在脚本和应用程序代码中以明文形式显示密码的风险，并允许您更轻松地管理用户帐户的密码策略，而无需在密码更改时更改应用程序代码或脚本。

有关使用mkstore命令行实用程序将客户端配置为使用外部密码存储库的步骤，请参阅配置客户端以使用外部密码存储库。

注意：

钱包的外部密码存储区与存储公钥基础结构（PKI）凭据的区域是分开的。因此，您无法使用Oracle Wallet Manager来管理钱包的外部密码存储中的凭据。而是使用命令行实用程序mkstore来管理这些凭据。

使用mkstore CreateCredential命令，通过提供<alias, username, password >信息来配置以下用户凭据，在其中将提示您输入每个用户的密码：

  • DBALIAS ， DBADMIN ， password

  • ENTALIAS ， ENTADMIN ， password

配置这些凭据允许您在umu脚本命令行上使用以下参数：

  • DBALIAS = <DB-密码别名>

  • ENTALIAS = <企业密码别名>

迁移对用户旧数据库模式的影响

如果未使用共享模式映射，则用户将保留其旧的数据库模式。如果使用共享模式映射，则会从数据库中删除用户的本地模式，并在执行迁移之前将它们映射到企业用户管理员为此目的创建的共享模式。当迁移的用户在其旧的本地数据库模式中拥有数据库对象时，管理员可以通过将CASCADE参数设置为NO来指定不要删除模式和对象。当CASCADE参数设置为NO ，在旧本地模式中拥有数据库对象的用户不会成功迁移，因此不会删除其对象。

如果某些用户希望将对象保留在其本地数据库模式中并映射到共享模式，则管理员可以在执行批量用户迁移之前手动将这些对象迁移到共享模式。但是，当对象迁移到共享模式时，它们将在共享该新模式的所有用户之间共享。

迁移过程

企业用户在目录中定义和管理，可以使用密码或证书对数据库进行身份验证。使用密码进行身份验证的用户需要Oracle数据库密码，该密码存储在目录中。使用证书进行身份验证的用户必须具有有效的X.509 v3证书。

此实用程序在迁移期间执行以下步骤：

  1. 选择数据库中的用户进行迁移。

  2. 创建相应的用户条目或使用目录中的现有条目。

  3. 创建新的数据库密码并将相应的验证者复制到用于迁移用户的目录。

  4. 将迁移用户条目的模式映射信息放在目录中。 （可选的）

  5. 删除或更改迁移用户的本地数据库模式。 （可选的）

    注意：

    在当前版本中，该实用程序使用基于证书的身份验证迁移用户，并使其可以进行密码身份验证。以前，基于SSL的经过身份验证的用户需要重置其Oracle数据库密码。用户钱包不是作为此过程的一部分创建的。

使用不同的CASCADE选项将用户映射到共享模式

CASCADE参数设置确定在迁移期间映射到共享模式时是否自动删除用户的旧数据库模式。仅当MAPSCHEMA设置为SHARED时才能使用CASCADE 。

本节介绍以下主题：

  • 使用CASCADE将用户映射到共享模式= NO

  • 使用CASCADE = YES将用户映射到共享架构

使用CASCADE将用户映射到共享模式= NO

默认情况下， CASCADE参数设置为NO 。此设置意味着在将迁移用户映射到共享模式时，不会迁移在旧模式中拥有数据库对象的用户。对于不拥有数据库对象的用户，将自动删除旧的数据库模式，并将它们映射到新的共享模式。

使用不同的MAPTYPE选项将用户映射到共享模式

当MAPSCHEMA设置为SHARED ，可以通过指定MAPTYPE参数的值来设置映射类型。此参数采用两个值，即映射类型和映射级别。

映射类型可以在DB ，数据库或DOMAIN中为企业域设置。指定映射类型DB ，映射仅应用于存储共享模式的数据库。将DOMAIN指定为映射类型时，映射将应用于包含存储共享架构的数据库的企业域，并且还适用于该域中的所有数据库。

映射级别可以设置为ENTRY或SUBTREE 。指定ENTRY ，用户将使用其完整可分辨名称（DN）映射到共享模式。这导致每个用户的一个映射。指定SUBTREE ，共享其部分DN的用户组将映射到一起。这导致已经在目录树中的某个公共根下分组的用户组的一个映射。示例A-5显示了使用MAPTYPE参数时要使用的语法。在此示例中，用户scott1 ， scott2和所有外部数据库用户将迁移到c=Users, c=us的目录，同时将所有迁移的用户映射到数据库中的新共享模式。在此示例中，映射将应用于包含数据库的企业域，并且映射将在条目级别执行，从而为每个用户生成映射。

关于使用SUBTREE映射级别选项

如果正在迁移的用户（例如scott ）将在其下的子树中具有将来的用户条目，则从该用户条目（ cn=scott ）到模式创建子树级别映射是有意义的。但是，数据库不会将用户解释为在子树中，因此映射不适用于scott本人。例如，如果要使用DN cn=scott,o=acme迁移用户scott ，并在运行该实用程序时选择SUBTREE作为映射级别，则会从cn=scott,o=acme创建新映射共享架构，但用户scott未映射到该架构。只有在scott目录条目下创建的新用户才会映射到共享模式。因此，仅当用户目录条目放在其他用户目录条目下时才应指定SUBTREE映射级别，这将是一个不寻常的目录配置。

如果希望将任意子树用户映射到只有一个映射条目的单个共享模式，则必须使用Oracle Enterprise Manager创建该映射。
