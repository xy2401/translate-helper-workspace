创造触发器

目的

触发器使用PL / SQL定义。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用CREATE TRIGGER语句创建数据库触发器 ，即：

  • 存储的PL / SQL块，与表，模式或数据库或

  • 匿名PL / SQL块或对PL / SQL或Java中实现的过程的调用

发生指定条件时，Oracle数据库会自动执行触发器。

ALTER TRIGGER和DROP TRIGGER

先决条件

要在您自己的模式中的表或您自己的模式（ SCHEMA ）上的自己的模式SCHEMA CREATE TRIGGER ，您必须具有CREATE TRIGGER系统特权。

在任何模式中的表或任何其他用户的模式（ schema上的任何模式中创建触发器。SCHEMA ），您必须拥有CREATE ANY TRIGGER系统特权。

除了上述特权之外，要在DATABASE上创建触发器，您必须具有ADMINISTER DATABASE TRIGGER系统特权。

要在可插拔数据库（PDB）上创建触发器，当前容器必须是该PDB，并且您必须具有ADMINISTER DATABASE TRIGGER系统特权。有关PDB的信息，请参见“ Oracle数据库管理员指南” 。

除了上述权限之外，要创建交叉触发器，还必须为版本启用。有关为用户启用版本的信息，请参阅“ Oracle数据库开发指南” 。

如果触发器发出SQL语句或调用过程或函数，则触发器的所有者必须具有执行这些操作所需的特权。必须直接向所有者授予这些权限，而不是通过角色获取。

指定OR REPLACE以重新创建触发器（如果已存在）。使用此子句可以在不先删除现有触发器的情况下更改现有触发器的定义。

使用这些子句指定的触发是否是editioned或noneditioned对象，如果editioning为架构对象类型启用TRIGGER的schema 。默认为EDITIONABLE 。有关已编辑和未编辑的对象的信息，请参见“ Oracle数据库开发指南” 。
