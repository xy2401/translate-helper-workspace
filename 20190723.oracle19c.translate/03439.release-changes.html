Oracle Database Release 19c中的更改

以下是Oracle数据库数据仓库指南（适用于Oracle数据库版本19c）中的更改。

新功能

  • LISTAGG函数的DISTINCT关键字使您可以消除指定表达式中的重复项。

    请参阅LISTAGG功能 。

  • 可以基于混合分区表创建物化视图。基于混合分区表的物化视图支持查询重写和刷新。

    请参阅基于混合分区表 创建物化视图 ， 基于混合分区表刷新物化视图 ，以及在混合分区表 存在时关于查询重写 。

  • 通过使用具有非附加事实的物化视图，可以增强包含COUNT(DISTINCT)操作的查询的性能。

    请参阅基于基于位图的COUNT（DISTINCT）函数创建包含基于位图的COUNT（DISTINCT）函数和查询重写和物化视图的实体化视图 。

Oracle Database Release 18c，版本18.1中的更改

以下是Oracle数据库数据仓库指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

新功能

  • 近似查询处理的增强功能

    商业智能（BI）应用程序广泛使用聚合函数（包括分析函数）来提供常见业务查询的答案。通过近似查询处理，您可以立即提高现有分析工作负载的性能，并实现更快的临时数据探索。排名，总和和计数的新近似SQL函数现在可用于前N个样式查询。

    也可以看看：

    关于近似前N个查询处理

  • 并行分区操作的增强功能

    并行分区SQL操作可以显着提高查询性能，从而提高响应时间。并行分区连接通常用于高效且快速地处理大型连接。除了并行分区连接之外，您还可以使用SQL窗口子句启动SQL操作的并行分区执行。

    也可以看看：

    具有窗口函数的并行分区智能操作

  • 分析视图的增强功能

    • 在SELECT语句的WITH或FROM子句中，分析视图的查询现在可以包括FILTER FACT关键字，以在聚合之前过滤分析视图访问的数据。使用新的ADD MEASURES关键字，您可以在SELECT语句中定义分析视图计算度量，以便应用程序现在可以动态定义自己的度量。

      也可以看看：

      使用过滤事实和附加度量的分析视图查询

    • CREATE ATTRIBUTE DIMENSION语句中的JOIN PATH关键字支持使用雪花样式的维度表。CREATE ANALYTIC VIEW语句中的REFERENCES DISTINCT关键字支持使用非规范化事实表，其中维度属性和事实数据位于同一个表中。

    • 分析视图现在支持您可以在计算的度量表达式中使用的各种新函数。新函数包括RANK_ *， PERCENTILE_ *， STATS_ *， COVAR_ *， HIER_DEPTH ， HIER_LEVEL ， HIER_MEMBER_NAME ， HIER_MEMBER_UNIQUE_NAME ， HIER_CAPTION和HIER_DESCRIPTION 。此外，您现在可以在计算度量的定义中使用分层属性。例如，在CASE语句中，您可以根据属性的值指定不同的计算表达式。

Oracle Database 12 c第2版（12.2.0.1）中的更改

以下是Oracle Database 12 c第2版（12.2.0.1）的Oracle数据库数据仓库指南中的更改。

新功能

  • 属性维，层次结构和分析视图

    分析视图提供了一种快速有效的方法来创建存储在现有数据库表和视图中的数据的分析查询。属性维指定维度表或视图列作为属性，并将属性组织为级别。层次结构按层次结构组织层次。分析视图引用层次结构并指定引用事实表中的数据并指定使用数据进行计算的度量。

    请参阅分析视图概述

  • 实时物化视图

    即使物化视图标记为陈旧，实时物化视图也可通过为用户查询提供新数据来进一步提高物化视图的可用性。当查询访问过时的实时物化视图时，Oracle数据库通过使用物化视图中的陈旧数据加上存储在物化视图日志中的增量信息，即时计算新查询结果。

    请参阅使用实时物化视图

  • 物化视图的ON STATEMENT刷新模式

    每次在任何基表上执行DML操作时， ON STATEMENT刷新模式都会刷新实例化视图，而无需提交事务。此模式不要求您在基表上维护物化视图日志。

    请参阅关于物化视图的ON STATEMENT刷新 。

  • 管理物化视图刷新统计信息

    物化视图刷新操作的统计信息被收集并存储在数据库中。您可以使用当前和历史统计信息来分析物化视图刷新性能。

    请参阅监视实体化视图刷新操作 。

  • 支持近似查询处理

    近似查询处理使用返回近似结果的SQL函数提供对探索性查询的极快响应，与精确结果的偏差可忽略不计。包含返回近似结果的SQL函数或近似查询的查询可用于创建物化视图。基于近似查询的物化视图符合查询重写的条件，可以快速刷新。

    请参见关于近似查询处理 。

  • LISTAGG增强功能

    所述LISTAGG函数现在提供对场景控制，其中通过返回串联的字符串LISTAGG函数超过由支持的最大长度VARCHAR2数据类型。

    请参阅LISTAGG功能 。

  • 使用SQL函数改进了对数据错误的处理

    现在，当发生数据类型转换错误时， CAST运算符可以返回用户指定的值。VALIDATE_CONVERSION函数确定转换为指定数据类型是否可以成功。

    请参阅使用SQL处理数据错误 。

  • 使用IM列存储增强查询性能

    内存中表达式（IM表达式）计算并将经常评估的查询表达式的结果填充到内存中列存储（IM列存储）中。将IM表达式填充到IM列存储中可确保后续查询可以使用它们来提供更快的查询响应时间。

    通过内存虚拟列（IM虚拟列），您可以将表中定义的虚拟列填充到IM列存储中。您可以将表中的全部或仅一部分虚拟列填充到内存中，以提高访问这些虚拟列的分析查询的性能。

    请参阅关于使用内存中表达式提高查询性能和关于使用内存中虚拟列来提高查询性能 。

  • IM列存储的自动数据优化（ADO）支持

    ADO可用于管理IM列存储的内容。ADO使用热图统计信息来确保只有最受益的元素存储在IM列存储中。这增强了使用IM列存储的性能优势。

    请参阅关于内存中列存储和自动数据优化 。

  • 在表重新定义期间优化批量更新的性能

    该EXECUTE_UPDATE在程序DBMS_REDEFINITION包可以优化的批量更新的表的性能。

    请参阅关于批量更新和联机表重新定义 。

  • 在线表重新定义期间支持物化视图刷新

    DBMS_REDEFINITION包可用于在联机表重新定义期间逐步刷新快速可刷新的从属物化视图。

    请参阅关于在联机表重新定义期间刷新从属实体化视图 。

  • 支持外部表的分区

    外部表的分区将现有的Oracle分区扩展到外部表。这样可以更好地与外部源集成，例如，将外部表的分区与HIVE表的分区对齐。它还使用现有的分区修剪技术提高了存储在数据库外部的数据的查询性能。

    请参阅分区以便更轻松地访问数据 。

  • 数据库操作监控

    可以监视执行次优的提取，转换和加载（ETL）作业，以帮助识别性能瓶颈。

    请参阅监视ETL操作概述 。

Oracle Database 12 c第1版（12.1.0.2）中的更改

以下是Oracle数据库12 c版本1（12.1.0.2）的Oracle数据库数据仓库指南中的更改。

新功能

  • Oracle内存中的列存储

    Oracle内存中列存储（IM列存储）位于SGA的可选区域中，以压缩列格式存储表，表分区和单个列。IM列存储是对数据库缓冲区高速缓存的补充而不是替代。

    IM列存储主要提高表扫描的性能和WHERE子句谓词的应用。更快的表扫描使优化器更有可能选择布隆过滤器和VECTOR GROUP BY转换。

  • 属性聚类

    通过表的属性聚类，您可以按照有序的方式将数据存储在磁盘上，该方式基于表中某些列的值。通过索引进行表扫描和表数据查找的I / O和CPU成本降低，因为通过表区域映射进行修剪变得更有效。

    也可以看看：

    “ 属性聚类 ”

  • 区域地图

    区域映射可根据磁盘上数据的物理位置自然修剪数据。在全表扫描期间仅访问相关数据块并在索引扫描期间仅访问相关数据行会降低数据访问的I / O和CPU成本。

    也可以看看：

    “ 使用区域地图 ”

  • 内存中聚合

    VECTOR GROUP BY操作提高了将一个或多个相对较小的表连接到较大的表和聚合数据的查询的性能。在数据仓库环境中，通常会为从IM列存储中选择数据的星型查询选择VECTOR GROUP BY聚合。

    VECTOR GROUP BY聚合最小化了将多个维度表连接到一个事实表所涉及的处理。它使用与并行查询相关的基础架构，并将其与CPU高效算法相结合，从而最大限度地提高性能。

  • 自动大表缓存

    自动大表缓存可提高不完全适合缓冲区缓存的大型表的内存查询性能。这些表可以存储在大表缓存中，这是数据库缓冲区缓存的可选，可配置部分。

    也可以看看：

    “ 自动大表缓存以提高内存中并行查询的性能 ”

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle数据库12 c版本1（12.1.0.1）的Oracle数据库数据仓库指南中的更改。

新功能

  • 模式匹配

    SQL已经扩展到支持模式匹配，这使得在序列上检测各种模式变得容易。模式匹配在许多商业应用中很有用，例如库存监控，网络入侵检测和电子商务购买跟踪。

    也可以看看：

    “ 用于模式匹配的SQL ”以获取更多信息

  • 前N个查询的本机SQL支持

    新的row_limiting_clause使您可以限制查询返回的行。您可以指定要返回的偏移量，行数或行百分比。这使您可以实现前N个报告。

    也可以看看：

    “ 限制SQL行 ”以获取更多信息

  • 批量加载操作的在线统计数据收集

    从Oracle Database 12 c开始 ，数据库会自动收集表统计信息作为批量加载操作的一部分。

  • 同步刷新

    一种称为同步刷新的新类型刷新使您可以保持一组表上定义的表和物化视图始终保持同步。它非常适用于数据仓库，其中增量数据的加载受到严格控制并且定期发生。

    也可以看看：

    “ 同步刷新 ”了解更多信息

  • 不合时宜的刷新

    可以使用新类型的刷新来提高物化视图刷新性能和可用性。这种刷新称为异地刷新，因为它在刷新期间使用外部表，在处理具有大量数据更改的情况时特别有效，其中传统的DML语句不能很好地扩展。

    也可以看看：

    “ 更新物化视图 ”以获取更多信息

不受支持的功能

Oracle Database 12 c第1版不支持本文档中先前描述的某些功能。有关不支持的功能的列表，请参见“ Oracle数据库升级指南” 。
