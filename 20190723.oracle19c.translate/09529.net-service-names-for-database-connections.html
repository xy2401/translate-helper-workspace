数据库连接的网络服务名称

连接到任何Oracle RAC实例的客户端在连接描述符中使用SCAN。您还可以使用网络服务名称连接到Oracle RAC。

如果使用DBCA创建作为多租户容器数据库（CDB）的Oracle RAC数据库，则DBCA将创建与数据库同名的数据库服务。DBCA创建的默认数据库服务使Oracle Enterprise Manager能够发现Oracle RAC数据库，不应将其用于客户端连接。使用此数据库服务的客户端可以连接到Oracle RAC CDB的任何数据库实例。但是，如果使用DBCA将可插拔数据库（PDB）添加到现有CDB，则DBCA不会为新PDB创建数据库服务。

网络服务名称不需要运行数据库，数据库实例或侦听器的服务器的完全限定域名。SCAN由DNS或GNS解析，后者向客户端返回三个地址。然后，客户端连续向每个地址提交连接请求，直到建立连接为止。

示例6-2数据库连接的网络服务名称输入示例

此示例显示了tnsnames.ora文件中使用的连接描述符。在这种情况下，连接标识符与集群域mycluster.example.com 。连接描述符不使用指定单个服务器，虚拟Internet协议（VIP）地址或群集节点名称的地址， myscan.mycluster.example.com使用SCAN，即myscan.mycluster.example.com 。 

mycluster.example.com =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（HOST = host = myscan.mycluster.example.com）（PORT = 1522））（CONNECT_DATA =（SERVER = DEDICATED）（SERVICE_NAME = myApp）） ）

Oracle Clusterware将使用网络服务名称mycluster.example.com连接请求解析为运行myApp数据库服务的mycluster数据库的任何数据库实例。运行实例的特定群集节点对客户端不可见。
