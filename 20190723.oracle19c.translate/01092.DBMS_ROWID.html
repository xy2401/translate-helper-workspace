142 DBMS_ROWID

DBMS_ROWID包允许您创建ROWIDs并从PL / SQL程序和SQL语句中获取有关ROWID的信息。您可以找到数据块编号，对象编号和其他ROWID组件，而无需编写代码来解释base-64字符外部ROWID 。 DBMS_ROWID用于从Oracle数据库版本7升级到Oracle数据库版本8。X。

注意：

DBMS_ROWID不能与通用ROWIDs （ UROWIDs ）一起使用。

142.1 DBMS_ROWID安全模型

此程序包以调用用户的权限运行，而不是程序包所有者SYS 。

142.2 DBMS_ROWID类型

有四种DBMS_ROWID类型。

这些是：

  • 扩展和限制类型

  • 验证类型

  • 对象类型

  • 转换类型

扩展和限制类型

类型如下：

  • RESTRICTED ROWID RESTRICTED

  • EXTENDED -extended ROWID

例如：

rowid_type_restricted常量整数：= 0; rowid_type_extended常量：= 1;

注意：

扩展的ROWIDs仅用于Oracle数据库版本8。X i和更高。

142.4 DBMS_ROWID操作说明

这些操作说明适用于DBMS_ROWID 。

  • 此程序包中的某些函数采用单个参数，例如ROWID 。根据需要，这可以是字符或PL / SQL ROWID ，可以是受限制的，也可以是扩展的。

  • 您可以从PL / SQL代码调用DBMS_ROWID函数和过程，也可以在SQL语句中使用这些函数。

    注意：

    ROWID_INFO是一个过程。它只能用于PL / SQL代码。

  • 您可以像使用内置SQL函数一样使用DBMS_ROWID包中的函数;换句话说，您可以在任何可以使用表达式的地方使用它们。在此示例中， ROWID_BLOCK_NUMBER函数仅用于返回EMP表中单个行的块编号：

    SELECT DBMS_ROWID.ROWID_BLOCK_NUMBER（rowid）FROM emp WHERE ename ='KING';

  • 如果Oracle为pragma restrict_references返回错误“ORA：452,0，'子程序'％s'违反其关联的pragma'，则可能意味着违规是由于：

    • 当前程序或功能的问题

    • 在没有编译指示的情况下调用过程或函数，或者调用具有较少限制性编译指示的过程或函数

    • 调用接触程序包中的初始化代码或设置默认值的程序包或函数

142.5 DBMS_ROWID示例

此示例返回ROWID在一排EMP表，提取从数据对象号ROWID ，使用ROWID_OBJECT在功能DBMS_ROWID包，然后显示对象数：

DECLARE object_no INTEGER; row_id ROWID; ...BEGIN SELECT ROWID INTO row_id FROM emp WHERE empno = 7499; object_no：= DBMS_ROWID.ROWID_OBJECT（row_id）; DBMS_OUTPUT.PUT_LINE（'对象。＃是'|| object_no）; ...

142.6.1 ROWID_BLOCK_NUMBER函数

此函数返回输入ROWID的数据库块号。

句法

DBMS_ROWID.ROWID_BLOCK_NUMBER（row_id IN ROWID，ts_type_in IN VARCHAR2 DEFAULT'MINALLFILE'）返回号码;

编译指示

编译指示RESTRICT_REFERENCES（rowid_block_number，WNDS，RNDS，WNPS，RNPS）;

参数

表142-6 ROWID_BLOCK_NUMBER函数参数

参数 描述

ROW_ID

ROWID被解释

ts_type_in

行所属的表空间（bigfile / smallfile）的类型

例子

示例SQL语句从ROWID选择块编号并将其插入另一个表中：

INSERT INTO T2（SELECT dbms_rowid.rowid_block_number（ROWID，'BIGFILE'）from some_table WHERE key_value = 42）;

142.6.2 ROWID_CREATE函数

此功能允许您创建ROWID ，给定组件作为参数。

这对于测试ROWID操作很有用，因为只有Oracle Server才能创建指向数据库中数据的有效ROWID 。

句法

DBMS_ROWID.ROWID_CREATE（rowID_type IN NUMBER，object_number IN NUMBER，relative_fno IN NUMBER，block_number IN NUMBER，row_number IN NUMBER）RETURN ROWID;

编译指示

编译指示RESTRICT_REFERENCES（rowid_create，WNDS，RNDS，WNPS，RNPS）;

参数

表142-7 ROWID_CREATE函数参数

参数 描述

rowid_type

类型（限制或扩展）

对于受限制的ROWID将rowid_type参数设置为0。将其设置为1以创建扩展的ROWID 。

如果将rowid_type指定为0，则忽略所需的object_number参数，并且ROWID_CREATE返回受限制的ROWID 。

object_number

数据对象编号（对于受限制的rowid_object_undefined ）

relative_fno

相对文件号

段号

此文件中的块编号

ROW_NUMBER

返回此块中的行号

例子

创建一个虚拟扩展ROWID ：

my_rowid：= DBMS_ROWID.ROWID_CREATE（1,9999,12,1000,13）;

找出rowid_object函数返回的内容：

obj_number：= DBMS_ROWID.ROWID_OBJECT（my_rowid）;

变量obj_number现在包含9999。

142.6.7 ROWID_TO_ABSOLUTE_FNO函数

此函数从ROWID提取绝对文件编号，其中文件编号对于给定模式和表中的行是绝对的。

模式名称和模式对象的名称（例如表名称）作为此函数的IN参数提供。

句法

DBMS_ROWID.ROWID_TO_ABSOLUTE_FNO（row_id IN ROWID，schema_name IN VARCHAR2，object_name IN VARCHAR2）RETURN NUMBER;

编译指示

编译指示RESTRICT_REFERENCES（rowid_to_absolute_fno，WNDS，WNPS，RNPS）;

参数

表142-12 ROWID_TO_ABSOLUTE_FNO函数参数

参数 描述

ROW_ID

ROWID被解释

SCHEMA_NAME

包含该表的架构的名称

OBJECT_NAME

表名

例子

DECLARE abs_fno INTEGER; rowid_val CHAR（18）; object_name VARCHAR2（20）：='EMP'; BEGIN SELECT ROWID INTO rowid_val FROM emp WHERE empno = 9999; abs_fno：= dbms_rowid.rowid_to_absolute_fno（rowid_val，'SCOTT'，object_name）;

注意：

对于分区对象，名称必须是表名，而不是分区或子/分区名。

142.6.8 ROWID_TO_EXTENDED函数

此函数转换受限制的ROWID ，该ROWID将您指定的架构和表中的行寻址为扩展的ROWID格式。

之后，它可能会从此包中移除到另一个地方。

句法

DBMS_ROWID.ROWID_TO_EXTENDED（old_rowid IN ROWID，schema_name IN VARCHAR2，object_name IN VARCHAR2，conversion_type IN INTEGER）RETURN ROWID;

编译指示

编译指示RESTRICT_REFERENCES（rowid_to_extended，WNDS，WNPS，RNPS）;

参数

表142-13 ROWID_TO_EXTENDED函数参数

参数 描述

old_rowid

要转换的ROWID

SCHEMA_NAME

包含表的模式的名称（可选）

OBJECT_NAME

表名（可选）。

CONVERSION_TYPE

定义了以下常量：

ROWID_CONVERT_INTERNAL （：= 0）

ROWID_CONVERT_EXTERNAL （：= 1）

返回值

ROWID_TO_EXTENDED以扩展字符格式返回ROWID 。如果输入ROWID为NULL ，则该函数返回NULL 。如果提供了零值ROWID （00000000.0000.0000），则返回零值受限ROWID 。

例子

假定有一个称为表RIDS在架构SCOTT ，并且该表包含列ROWID_COL保持ROWIDs （限制），及列TABLE_COL指向在其他表SCOTT架构。您可以使用以下语句将ROWIDs转换为扩展格式：

UPDATE SCOTT.RIDS SET rowid_col = dbms_rowid.rowid_to_extended（rowid_col，'SCOTT“，TABLE_COL，0）;

使用说明

  • 如果模式和对象名称作为IN参数提供，则此函数将验证名为的表上的SELECT权限，并使用表的数据对象编号ROWID提供的受限ROWID转换为扩展的ROWID 。但是， ROWID_TO_EXTENDED返回一个值并不能保证转换后的ROWID实际引用表中的有效行，无论是在调用函数时，还是在实际使用扩展的ROWID时。

  • 如果未提供架构和对象名称（传递为NULL ），则此函数尝试获取由提供的受限ROWID指定的页面。它将存储在此ROWID中的文件编号视为绝对文件编号。如果文件已被删除，这可能会导致问题，并且在迁移之前已重复使用其编号。如果获取的页面属于有效表，则此表的数据对象编号用于转换为扩展的ROWID值。这是非常低效的，并且Oracle建议仅在目标表未知时作为最后的手段执行此操作。用户在使用转换后的值时仍必须知道正确的表名。

  • 如果提供了扩展的ROWID值，则根据从表名参数计算的数据对象编号验证输入扩展ROWID中的数据对象编号。如果两个数字不匹配，则引发INVALID_ROWID异常。如果它们匹配，则返回输入ROWID 。

  • ROWID_TO_EXTENDED不能与分区表一起使用。
