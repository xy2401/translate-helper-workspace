从早期版本升级Oracle Grid Infrastructure

完成此过程以从早期版本升级Oracle Grid Infrastructure（Oracle Clusterware和Oracle自动存储管理）。

在升级过程中的任何时候，如果您对要求执行的操作或升级期间需要提供的输入有疑问，请单击安装程序页面上的“ 帮助”按钮。
在开始升级之前，您应该可以使用网络信息，存储信息以及操作系统用户和组，并且应该准备好运行根脚本。
  1. 作为grid用户，下载Oracle Grid Infrastructure映像文件并将文件解压缩到Grid主目录。

    例如：

    mkdir -p /u01/app/19.0.0/grid chown grid：oinstall /u01/app/19.0.0/grid cd /u01/app/19.0.0/grid unzip -q download_location/grid_home.zip

    download_location/grid_home.zip是下载的Oracle Grid Infrastructure映像文件的路径。

    注意：

    • 您必须将图像软件提取到您希望Grid主页所在的目录中。

    • 仅将Oracle Grid Infrastructure映像文件下载并复制到本地节点。在升级期间，将复制软件并将其安装在群集中的所有其他节点上。

  2. 运行以下命令启动Oracle Grid Infrastructure向导：
    /u01/app/19.0.0/grid/gridSetup.sh
  3. 选择以下配置选项：
    • 升级Oracle Grid Infrastructure：选择此选项可升级Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）。

    注意：

    Oracle Clusterware必须始终是更高版本，因此您无法将Oracle ASM升级到比Oracle Clusterware更新的版本。
  4. 在“节点选择”页面上，选择所有节点。
  5. 根据提示选择安装选项。Oracle建议您配置root脚本自动化，以便在升级期间自动运行rootupgrade.sh脚本。
  6. 自动或手动运行root脚本：
    • 自动运行根脚本：

      如果已配置root脚本自动化，则使用批处理之间的暂停将服务从运行先前版本的节点重定位到新版本。

    • 手动运行根脚本

      如果尚未配置root脚本自动化，则在出现提示时，在群集中要升级的每个节点上运行rootupgrade.sh脚本。

    如果手动运行root脚本，则首先在本地节点上运行脚本。该脚本关闭早期版本的安装，将其替换为新的Oracle Clusterware版本，并启动新的Oracle Clusterware安装。脚本成功完成后，您可以在除一个节点之外的所有节点上并行运行脚本，您选择该节点作为最后一个节点。在除最后一个节点之外的所有节点上成功运行脚本时，请在最后一个节点上运行该脚本。
  7. 由于Oracle Grid Infrastructure主目录位于与以前的Oracle Clusterware和Oracle ASM主目录不同的位置，因此请更新使用驻留在Oracle Clusterware和Oracle ASM主目录中的实用程序，库或其他文件的任何脚本或应用程序。
  8. 按照更新Oracle企业管理器云控制目标参数主题中的说明更新Oracle Enterprise Manager目标参数 。

注意：

  • 升级到Oracle Grid Infrastructure 19c后，如果使用Oracle ASM进行数据库存储，请从$ORACLE_HOME/network/admin/sqlnet.ora文件中删除ADR_BASE=/u01/app/grid条目。

  • 在升级结束时，如果将Oracle Cluster Registry（OCR）备份位置手动设置为早期版本的Oracle Clusterware主目录（CRS主目录），则必须将OCR备份位置更改为新的Oracle Grid Infrastructure主目录（Grid home） ）。如果未手动设置OCR备份位置，则会在升级期间为您更改备份位置。

  • 由于Oracle Clusterware的升级是不合适的升级，因此先前版本的Oracle Clusterware主目录不能是当前版本OCR备份的位置。可以删除旧Oracle Clusterware主目录中的备份。

  • 如果要升级的群集具有存储OCR，OCR备份，Oracle ASM密码，Oracle ASM密码文件备份和Grid Infrastructure Management Repository（GIMR）的单个磁盘组，则Oracle建议您创建单独的磁盘组或使用另一个现有磁盘组并在该磁盘组中存储OCR备份，GIMR和Oracle ASM密码文件备份。
