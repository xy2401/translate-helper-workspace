  • SQLNET.ENCRYPTION_SERVER
    SQLNET.ENCRYPTION_SERVER参数指定充当客户端的客户端或服务器连接到此服务器时的加密行为。
  • SQLNET.ENCRYPTION_CLIENT
    SQLNET.ENCRYPTION_CLIENT参数指定充当客户端的客户端或服务器连接到服务器时的加密行为。
  • SQLNET.CRYPTO_CHECKSUM_SERVER
    SQLNET.CRYPTO_CHECKSUM_SERVER参数指定客户端或充当客户端的其他服务器连接到此服务器时的数据完整性行为。

    SQLNET.CRYPTO_CHECKSUM_CLIENT参数指定当此客户端或服务器充当客户端连接到服务器时所需的数据完整性行为。
  • SQLNET.ENCRYPTION_TYPES_SERVER
    SQLNET.ENCRYPTION_TYPES_SERVER参数指定此服务器按预期用途的顺序使用的加密算法。
  • SQLNET.ENCRYPTION_TYPES_CLIENT
    SQLNET.ENCRYPTION_TYPES_CLIENT参数指定此客户端或充当客户端的服务器使用的加密算法。

    SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER参数按照预期用途指定此服务器或客户端到另一台服务器使用的数据完整性算法。
  • SQLNET.CRYPTO_CHECKSUM_TYPES_CLIENT
    SQLNET.CRYPTO_CHECKSUM_TYPES_CLIENT参数指定此客户端或服务器充当客户端使用的数据完整性算法列表。

关于数据加密和完整性参数

数据加密和完整性参数控制您正在使用的加密算法的类型。

如果未指定Server Encryption，Client Encryption，Server Checksum或Client Checksum的任何值，则相应的配置参数不会出现在sqlnet.ora文件中。但是，默认ACCEPTED 。

对于数据加密和完整性算法，服务器选择其sqlnet.ora文件中列出的第一个算法，该算法与客户端sqlnet.ora文件中列出的算法匹配，或者如果客户端在其sqlnet.ora中未列出任何算法，则选择客户端安装列表中的算法sqlnet.ora文件。如果服务器sqlnet.ora文件中没有条目，则服务器将按顺序搜索其安装的列表以匹配客户端上的项目 - 在客户端sqlnet.ora文件或客户端安装列表中。如果不能进行匹配并且连接的一侧需要算法类型（数据加密或完整性），则连接失败。否则，连接成功，算法类型为inactive 。

数据加密和完整性算法彼此独立地选择。可以在没有完整性的情况下激活加密，并且可以在不加密的情况下激活完整性，如表B-1所示 ：

SQLNET.ENCRYPTION_TYPES_SERVER

SQLNET.ENCRYPTION_TYPES_SERVER参数指定此服务器按预期用途的顺序使用的加密算法。

此列表用于与连接的客户端协商相互可接受的算法。针对可用的客户端算法类型列表检查每个算法，直到找到匹配为止。如果在此侧指定了未安装的算法，则连接将终止，并显示错误消息ORA-12650: No common encryption or data integrity algorithm.

表B-6说明了SQLNET.ENCRYPTION_TYPES_SERVER参数属性。

表B-6 SQLNET.ENCRYPTION_TYPES_SERVER参数属性

属性 描述

句法

SQLNET.ENCRYPTION_TYPES_SERVER = ( valid_encryption_algorithm [, valid_encryption_algorithm ])

有效值

  • AES256 ：AES（256位密钥大小）

  • AES192 ：AES（192位密钥大小）

  • AES128 ：AES（128位密钥大小）

默认设置

如果在本地sqlnet.ora文件中未定义任何算法，则在前面的序列中的协商中使用所有已安装的算法。

使用说明

您可以指定多种加密算法。它可以是单个值，也可以是算法名称列表。例如，可以接受以下任一加密参数：

SQLNET.ENCRYPTION_TYPES_SERVER =（AES256）

SQLNET.ENCRYPTION_TYPES_SERVER =（AES256，AES192，AES128）

SQLNET.ENCRYPTION_TYPES_CLIENT

SQLNET.ENCRYPTION_TYPES_CLIENT参数指定此客户端或充当客户端的服务器使用的加密算法。

此列表用于与连接的另一端协商双方可接受的算法。如果在此侧指定了未安装的算法，则连接将终止于ORA-12650: No common encryption or data integrity algorithm error消息。

表B-7说明了SQLNET.ENCRYPTION_TYPES_CLIENT参数属性。

表B-7 SQLNET.ENCRYPTION_TYPES_CLIENT参数属性

属性 描述

句法

SQLNET.ENCRYPTION_TYPES_CLIENT = ( valid_encryption_algorithm [, valid_encryption_algorithm ])

有效值

  • AES256 ：AES（256位密钥大小）。

  • AES192 ：AES（192位密钥大小）。

  • AES128 ：AES（128位密钥大小）。

默认设置

如果本地sqlnet.ora文件中未定义任何算法，则在协商中使用所有已安装的算法。

SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER

SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER参数按照预期用途指定此服务器或客户端到另一台服务器使用的数据完整性算法。

此列表用于与连接的另一端协商双方可接受的算法。针对可用的客户端算法类型列表检查每个算法，直到找到匹配为止。如果指定的算法未安装在此侧，则连接将终止于 ORA-12650: No common encryption or data integrity algorithm error错误消息。

表B-8说明了SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER参数属性。

表B-8 SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER参数属性

属性 描述

句法

SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER = ( valid_crypto_checksum_algorithm [, valid_crypto_checksum_algorithm ])

有效值

  • SHA512 ：SHA-2，产生512位散列。

  • SHA384 ：SHA-2，产生384位散列。

  • SHA256 ：SHA-2，产生256位散列。这是默认值。

  • SHA1 ：安全散列算法

  • MD5 ：消息摘要5

默认设置

如果本地sqlnet.ora文件中未定义任何算法，则在从SHA256开始的协商中使用所有已安装的算法。
