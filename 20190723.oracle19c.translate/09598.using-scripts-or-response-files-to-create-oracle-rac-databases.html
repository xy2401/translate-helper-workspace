A使用脚本或响应文件创建Oracle RAC数据库

查看非交互式安装的此信息，在此期间您可以使用脚本创建Oracle Real Application Clusters（Oracle RAC）数据库。

注意：

Database Configuration Assistant（DBCA）生成的脚本仅供参考。Oracle强烈建议您使用DBCA创建数据库。

  • 使用DBCA为Oracle RAC生成安装脚本
    查看此主题以生成用于创建Oracle RAC数据库的脚本，使用生成的脚本创建数据库，以及准备数据库以供使用。
  • 关于Oracle RAC的DBCA非交互式（静默）配置
    您可以使用DBCA执行Oracle RAC的非交互式或静默配置。要执行静默配置，您必须已完成Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）安装，从Oracle数据库主目录运行root.sh脚本，并定义Oracle主目录环境变量。
  • 使用DBCA命令进行Oracle RAC的非交互式（静默）配置
    查看此主题以获取使用DBCA创建Oracle RAC数据库的命令语法。
  • 响应文件的工作原理
    响应文件可以帮助您在多台计算机上多次安装Oracle产品。
  • 准备响应文件
    查看此信息以准备响应文件，以便在静默模式或响应文件模式安装期间使用。
  • 使用响应文件运行Oracle Universal Installer
    创建响应文件后，在命令行运行Oracle Universal Installer，指定您创建的响应文件，以执行安装。
  • 使用安装期间创建的响应文件进行安装后配置
    安装后使用响应文件配置Oracle软件。您可以使用在安装期间创建的相同响应文件来完成安装后配置。
  • 使用ConfigToolAllCommands脚本进行安装后配置
    您可以在安装Oracle软件后创建并运行响应文件配置。configToolAllCommands脚本要求用户创建第二个响应文件，其格式与用于安装产品的格式不同。
  • 使用响应文件运行配置助手
    您可以在响应文件或静默模式下运行配置助手，以便在系统上安装Oracle软件后对其进行配置和启动。要在响应文件或静默模式下运行配置助手，您必须复制和编辑响应文件模板。
