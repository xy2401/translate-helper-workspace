2 XML C API的包回调

Callback包定义了为XML回调声明函数（或函数指针）的宏。回调用于错误消息处理，内存分配和释放以及流操作。

下表总结了Callback接口可用的方法。

表2-1 XML C实现的回调方法摘要

XML_ACCESS_CLOSE_F（）

该宏定义了用于访问URL的close函数回调的原型。

XML_ACCESS_OPEN_F（）

该宏定义了用于访问URL的open函数回调的原型。

XML_ACCESS_READ_F（）

该宏定义了用于访问URL的读取函数回调的原型。

XML_ALLOC_F（）

该宏定义了用户提供的低级内存alloc功能的原型。如果未提供分配器，则使用malloc 。此功能不应将内存归零。匹配XML_FREE_F（） 。

XML_ERRMSG_F（）

该宏定义了错误消息处理函数的原型。如果在XML初始化时没有提供错误消息回调，则会将错误打印到stderr 。如果提供了处理程序，则将调用它而不是打印到stderr。

XML_FREE_F（）

该宏定义了用户提供的低级别无内存功能的原型。如果未提供分配器，则使用free（）。匹配XML_ALLOC_F（） 。

XML_STREAM_CLOSE_F（）

这个宏定义了close函数回调的原型，调用它来关闭一个开源并释放它的资源。

XML_STREAM_OPEN_F（）

该宏定义了open函数回调的原型，该函数调用一次以打开输入源。此函数应成功返回XMLERR_OK 。

XML_STREAM_READ_F（）

该宏定义了读取函数回调的原型，称为从开源读取数据到缓冲区，返回读取的字节数（出错时为< 0 ）。eoi标志确定这是否是最后的数据块。

在EOI上，将自动调用close函数。
