6.675 USER_USERS

USER_USERS描述当前用户。

柱 数据类型 空值 描述

用户名

VARCHAR2（128）

不是NULL

用户名

用户身份

数

不是NULL

用户的ID号

帐户状态

VARCHAR2（32）

不是NULL

帐户状态：

  • 打开

  • 过期

  • 到期（GRACE）

  • 锁定（计时）

  • 已锁定

  • 已过期和已锁定（已定时）

  • 过期（GRACE）和锁定（定时）

  • 过期和锁定

  • 过期（GRACE）和锁定

日期

VARCHAR2（30）

不是NULL

临时表的默认表空间的名称或表空间组的名称

LOCAL_TEMP_TABLESPACE

VARCHAR2（30）

用户的默认本地临时表空间

CREATED

日期

不是NULL

用户创建日期

INITIAL_RSRC_CONSUMER_GROUP

VARCHAR2（128）

用户的初始资源使用者组

EXTERNAL_NAME

VARCHAR2（4000）

用户外部名称。对于集中管理的用户，如果数据库用户映射是独占映射，那么这将是用户的目录服务DN。如果此数据库用户是共享架构，则它将是组的DN。

PROXY_ONLY_CONNECT

VARCHAR2（1）

指示用户是否可以直接连接（ N ），或者该帐户是否只能由具有此帐户代理权限的用户（即，已被授予此帐户“connect through”权限的用户）代理（ Y ） 。

有关创建代理用户帐户和授权用户通过它们进行连接的详细信息，请参阅“ Oracle数据库安全指南” 。

共同

VARCHAR2（3）

指示给定用户是否通用。可能的值

  • 如果用户很常见，则为YES

  • NO如果用户是本地的（不常见）

ORACLE_MAINTAINED

VARCHAR2（1）

表示用户是由Oracle提供的脚本（例如catalog.sql还是catproc.sql）创建并维护的。除非运行Oracle提供的脚本，否则不得以任何方式更改此列具有值Y的用户。

遗传

VARCHAR2（3）

指示用户定义是否从其他容器继承（ YES ）（ NO ）

DEFAULT_COLLATION

VARCHAR2（100）

用户架构的默认排序规则

隐

VARCHAR2（3）

指示此用户是否是由隐式应用程序创建的公共用户（ YES ）或否（ NO ）

ALL_SHARD

VARCHAR2（3）

在分片数据库中，此列中的值指示是否已使用启用了分片DDL创建用户。可能的值是：

  • YES ：用户是在启用了分片DDL的情况下创建的。用户存在于所有分片和分片目录中。

  • NO ：创建的用户未启用分片DDL。用户仅存在于创建用户的数据库中。

在非分片数据库中，此列中的值始终为NO 。
