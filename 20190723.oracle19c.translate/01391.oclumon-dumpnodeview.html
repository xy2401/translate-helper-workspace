B.2 oclumon dumpnodeview

使用oclumon dumpnodeview命令以节点视图的形式查看系统监视器服务的日志信息。

使用说明

节点视图是Cluster Health Monitor在某个时间点为节点收集的所有度量标准的集合。Cluster Health Monitor尝试在每个节点上每五秒收集一次度量标准。某些指标是静态的，而其他指标是动态的。

显示详细输出时，节点视图由八个视图组成：

  • SYSTEM ：列出系统指标，例如CPU COUNT，CPU USAGE和MEM USAGE

  • 热门消费者 ：按以下格式列出最常用的流程：

    metric_name ：' process_name （ process_identifier ） utilization '

  • CPUS ：列出每个CPU的统计信息

  • 进程 ：列出进程指标，例如PID，名称，线程数，内存使用情况和文件描述符数

  • 设备 ：列出设备指标，例如磁盘读取和写入速率，队列长度以及每个I / O的等待时间

  • NICS ：列出网络接口卡指标，例如网络接收和发送速率，有效带宽和错误率

  • FILESYSTEMS ：列出文件系统指标，例如总空间，已用空间和可用空间

  • 协议错误 ：列出任何协议错误

生成仅包含SYSTEM和TOP CONSUMERS视图的摘要报告。

句法

oclumon dumpnodeview [-allnodes | -n node1 ...] [-last duration | -s timestamp -e timestamp ] [-i interval ] [-v | [-system] [ -  process] [ -  procag] [ -  device] [ -  filesystem] [ -  nic] [ -  protoerr] [ -  cpu] [ -  topconsumer]] [-format format type ] [-dir directory [-append] ]]

参数

表B-2 oclumon dumpnodeview命令参数

参数 描述 
-allnodes

使用此选项可转储群集中所有节点的节点视图。

 
-n node1 node2

在要以空间分隔的列表中指定要转储节点视图的一个节点或多个节点。

 
- duration “

使用此选项指定以双引号（ "" ）包围的HH24:MM:SS格式给出的时间，以检索最后的度量标准。

例如：
“23点05分00秒”
 
-s“ time_stamp ”-e“ time_stamp ”

使用-s选项指定从中开始一系列查询的时间戳，并使用-e选项指定结束查询范围的时间戳。

以YYYY-MM-DD HH24:MM:SS格式指定时间，用双引号（ "" ）括起来。

例如：
“2011-05-10 23:05:00”

注意：一起指定这两个选项以获取范围。 

-i interval

指定收集间隔，以五秒为增量。

 
-v

显示详细节点视图输出。

 
-system，-process，-device，-filesystem，-nic，-protoerr，-cpu，-topconsumer

转储每个指定的节点视图部件。

 
format type “ format type ”

指定输出格式。

“ format type ”可以是legacy ， tabular或csv 。

对于只有一行的节点视图部分，默认格式通常是表格式的遗留格式。

 
-dir directory

将节点视图转储到指定目录中的文件。

指定-append选项以将当前文件附加到现有文件。如果未指定–append ，则该命令将覆盖现有文件（如果存在）。

例如，命令oclumon dumpnodeview -dir dir_name转储在指定的目录中的数据。

如果此命令运行两次，它将覆盖上一次运行转储的数据。

运行与命令-append ，例如， oclumon dumpnodeview -dir dir_name -append ，追加与前一个在指定的目录中的当前运行的数据。 

-procag

输出节点视图的过程，按类别汇总：

  • DBBG（数据库背景）

  • DBFG（DB前景）

  • CLUST（集群）

  • 其他（其他流程）

注意： -procag目前仅在Linux，Solaris和AIX上可用。 Microsoft Windows系统不支持它。 

-H

显示oclumon dumpnodeview命令的联机帮助。

使用说明

  • 在某些情况下，在命令重放数据之前，数据可能会延迟一段时间。

    例如， crsctl stop cluster -all命令可能导致数据延迟。在运行crsctl start cluster -all ，可能需要几分钟才能在oclumon dumpnodeview显示间隔期间收集的任何数据。

  • 默认设置是不断转储节点视图。要停止连续显示，请在Linux和Microsoft Windows上使用Ctrl + C.

  • 必须运行本地系统监视器服务（ osysmond ）和集群记录器服务（ ologgerd ）才能获取节点视图转储。

  • oclumon dumpnodeview命令仅显示CPU核心的127个CPU，从列表中随机省略CPU。

例B-2 dumpnodeview -n

以下示例转储在过去12小时内收集的node1 ， node2和node3节点视图：

$ oclumon dumpnodeview -n node1 node2 node3 -last“12:00:00”

以下示例显示过去15分钟内以30秒间隔收集的所有节点的节点视图：

$ oclumon dumpnodeview -allnodes -last“00:15:00”-i 30
