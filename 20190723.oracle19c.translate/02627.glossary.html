词汇表

访问控制条目（ACE）

访问控制列表中的一个条目，用于授予或拒绝对给定主体的访问权限。一个或多个ACE列在访问控制列表（ACL）中 ，其中ACE的排序是相关的。

访问控制列表（ACL）

一组访问控制条目，用于确定哪些主体可以访问给定资源。在Oracle Database Real Application Security中，您使用ACL来定义用户权限。

高手

请参阅访问控制条目（ACE） 。

ACL

请参阅访问控制列表（ACL） 。

聚合特权

一个特权包含其他特权。当授予或拒绝聚合特权时，也会授予或拒绝其所有子特权。

应用角色

只能授予应用程序用户或其他应用程序角色的角色。

申请会议

包含仅与应用程序相关的信息的用户会话。与传统的“重量级”数据库会话不同，应用程序会话不拥有自己的数据库资源，如事务和游标。

应用用户

不具有架构的用户帐户，可以通过中间层创建到数据库的应用程序会话 。

列级安全性

将特定权限应用于表列的功能。

自定义权限

Oracle数据库未预定义的权限 。另请参见系统特权 。

数据领域

数据库表中的一组行，通过将其与访问控制列表（ACL）相关联来控制其访问。它由一个或多个对象实例组成 。另请参见动态数据领域约束和静态数据领域约束 。

数据库角色

只能授予数据库用户的角色。它也被称为重量级角色 。另见应用程序角色 。

数据库用户

在数据库中创建并具有架构的用户帐户。它也被称为重量级用户 。另请参见应用程序用户

动态ACL

已与动态数据领域约束关联的访问控制列表。

动态应用角色

仅在某些条件下启用的角色，例如，当用户使用SSL登录时或在指定时间段内登录时。

动态数据领域约束

一个数据域 ，其WHERE谓词是重新运行的每个用户执行对数据的境界约束数据的查询时间。另请参见静态数据领域约束 。

功能安全

用户访问应用程序功能的机制受到控制。例如，对于Oracle Database Real Application Security，请使用checkPrivilege()方法检查行的ACL权限，以确定一个或多个给定ACL上的特定权限是否与该行相关联。有关更多信息，请参阅关于Check Privilege API 。

全局唯一标识符（GUID）

应用程序可用于管理用户会话信息的外部标识。此标识符不保证在所有层中都是唯一的，但包含它的唯一键的数量非常大，以至于它的复制机会很小。另请参见唯一标识符（UID） 。

GUID

请参阅全局唯一标识符（GUID） 。

重量级的角色

传统的数据库角色 。

重量级用户

拥有架构的传统数据库用户帐户。

命名空间

由属性 - 值对组成的容器，反映应用程序会话的状态。

对象实例

单个关系表行，它是数据域的一部分。它由其主键值标识。

密码验证程序

散列版本的明文密码，然后编码为BASE64编码的字符串。

主要

用户或用户集合交替地称为组或角色 。另见应用程序用户和应用程序角色 。

特权

可以授予或拒绝委托人的权利或许可。另请参见聚合权限 ， 自定义权限和系统权限 。

安全等级

可以与ACL关联的命名的权限集合。

静态ACL

已与静态数据领域约束关联的访问控制列表。

静态数据领域约束

一个数据的境界 ，其WHERE谓词存储在缓存中，所以它不是每个用户对数据的境界约束数据的查询时间重新运行。另请参见动态数据领域约束 。

系统特权

Oracle数据库提供的预定义权限 。另请参见自定义权限 。

唯一标识符（UID）

Oracle数据库用于跟踪用户或角色的唯一内部标识符。它用于管理跨数据库企业的用户会话信息。另请参见全局唯一标识符（GUID） 。

UID

请参阅唯一标识符（UID） 。

用户开关

应用程序用户作为另一个用户进行代理的能力。应用程序状态（即名称空间和属性）是从以前的用户维护的，但安全上下文反映了新用户的安全上下文。
