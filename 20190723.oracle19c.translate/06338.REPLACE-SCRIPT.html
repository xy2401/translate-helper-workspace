3.5更换脚本

目的

使用REPLACE SCRIPT命令替换存储在恢复目录中的现有脚本。如果脚本不存在，则REPLACE SCRIPT创建它。

也可以看看：

CREATE SCRIPT以了解如何创建存储的脚本

先决条件

仅在RMAN提示符下执行REPLACE SCRIPT 。RMAN必须连接到目标数据库和恢复目录。必须打开恢复目录数据库。

如果要替换本地脚本，则必须连接到创建脚本时连接到的目标数据库。

存储脚本中的替换变量

RMAN支持在存储的脚本中使用替换变量。 &1表示放置第一个值的位置， &2表示放置第二个值的位置，依此类推。必须引用特殊字符。

替换变量语法是& integer后跟可选的句点，例如&1.3 。可选句点是变量的一部分，并替换为值，从而使替换文本后面紧跟另一个整数。例如，如果将值mybackup传递给包含替换变量&1.3的命令文件，则替换的结果为mybackup3 。要创建结果mybackup.3 ，在&1..3使用两个句点。

使用替换变量创建存储脚本时，必须在创建时提供示例值。您可以在启动RMAN时USING子句提供这些值（请参阅RMAN ），或在提示时输入它们（请参阅示例2-74 ）。

语义

语法元素 描述

全球

将脚本标识为全局。

注意：虚拟专用目录具有对全局脚本的只读访问权限。必须在RMAN连接到基本恢复目录时创建或更新全局脚本。

另请参阅： “ 使用说明 ” ，以解释全局脚本和本地脚本之间的区别

SCRIPT script_name

标识要替换的本地或全局脚本。

COMMENT ' comment '

将解释性注释与恢复目录中存储的脚本相关联。注释显示在LIST SCRIPT NAMES的输出中。
backupCommands

maintenanceCommands

restoreCommands

miscellaneousCommands

指定要包含在存储脚本中的命令。的命令的括号内是允许REPLACE SCRIPT ' script_name ' {...}命令的一个内所支持的相同的命令RUN块。在存储的脚本中允许在RUN命令中有效的任何命令。存储的脚本中不允许使用以下命令： RUN ， @ (at sign)和@@ (double at sign) 。

FROM FILE ' filename '

读取命令序列以从指定文件定义脚本。

该文件类似于有效存储脚本的主体。文件的第一行必须是左大括号（ { ），最后一行必须包含右大括号（ } ）。文件中的RMAN命令必须在存储的脚本中有效。

例

示例3-19替换恢复目录脚本

假设您启动RMAN客户端并以TARGET身份连接到数据库prod ，然后连接到恢复目录。您可以使用CREATE SCRIPT创建名为backup_db的全局脚本，如下所示：

CREATE GLOBAL SCRIPT backup_db COMMENT“从恢复目录备份任何数据库，带有日志”{BACKUP DATABASE; }

然后，使用LIST SCRIPT NAMES列出恢复目录中已知的所有脚本：

RMAN> LIST SCRIPT NAMES;恢复目录中的存储脚本列表全局脚本脚本名称描述-------------------------------------- --------------------------------- backup_db使用日志备份恢复目录中的任何数据库

然后运行以下REPLACE SCRIPT命令，目的是编辑backup_db全局脚本：

RMAN> REPLACE SCRIPT backup_db {BACKUP DATABASE PLUS ARCHIVELOG; }替换脚本backup_db

因为你没有指定GLOBAL关键字，RMAN创建了一个名为本地脚本backup_db除了命名全局脚本backup_db 。LIST SCRIPT NAMES显示恢复目录中记录的全局和本地脚本：

RMAN> LIST SCRIPT NAMES;恢复目录中的存储脚本列表目标数据库的脚本PROD脚本名称描述----------------------------------- ------------------------------------ backup_db全局脚本脚本名称描述-------- -------------------------------------------------- ------------- backup_db使用日志从恢复目录中备份任何数据库

然后，您可以使用DELETE SCRIPT删除名为backup_db的本地脚本，并按如下所示替换全局脚本：

RMAN> DELETE SCRIPT backup_db;已删除的脚本：backup_db RMAN> REPLACE GLOBAL SCRIPT backup_db {BACKUP DATABASE PLUS ARCHIVELOG; }替换了全局脚本backup_db

LIST SCRIPT NAMES命令现在显示目录中只存在一个名为backup_db脚本：

RMAN> LIST SCRIPT NAMES;恢复目录中的存储脚本列表全局脚本脚本名称描述-------------------------------------- --------------------------------- backup_db

PRINT SCRIPT命令确认对全局脚本的更改：

RMAN> PRINT GLOBAL SCRIPT backup_db;打印存储的全局脚本：backup_db {BACKUP DATABASE PLUS ARCHIVELOG; }
