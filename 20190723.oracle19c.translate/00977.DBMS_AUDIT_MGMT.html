26 DBMS_AUDIT_MGMT

DBMS_AUDIT_MGMT包提供子程序来管理审计跟踪记录。这些子程序使审计管理员能够管理审计跟踪。在混合模式环境中，这些审计跟踪包括数据库，操作系统（OS）和XML审计跟踪。在统一的审计环境中，这包括统一的审计跟踪。

26.1 DBMS_AUDIT_MGMT概述

数据库审计有助于满足数据库安全性和合规性要求。在混合模式环境中，审计记录将写入数据库表，操作系统（OS）文件或XML文件，具体取决于AUDIT_TRAIL初始化参数设置。如果已升级到统一审核，则审核记录将写入统一审核跟踪。

在混合模式环境中，当AUDIT_TRAIL设置为DB ，数据库记录将写入AUD$表。在统一审计环境中，审计记录将写入AUDSYS架构中的只读表。可以从UNIFIED_AUDIT_TRAIL数据字典视图中获取此表的内容。当AUDIT_TRAIL设置为OS时，审计记录将写入操作系统文件。当AUDIT_TRAIL设置为XML时，审计记录将以XML格式写入操作系统文件。

使用统一审计工具，所有审计记录都以统一格式写入统一审计跟踪，并通过UNIFIED_AUDIT_TRAIL视图提供。

正确管理审计记录非常重要，以确保有效的性能和磁盘空间管理。DBMS_AUDIT_MGMT子程序使您可以有效地管理审计跟踪记录。

如果尚未迁移到统一审计，则DBMS_AUDIT_MGMT包提供了一个子程序，允许您将数据库审计跟踪表移出SYSTEM表空间。这通过减少SYSTEM表空间的负载来提高整体数据库性能。它还使您能够为审计记录专门提供优化的表空间。

对于混合模式环境， DBMS_AUDIT_MGMT子程序还允许您管理操作系统和XML审计文件。您可以定义属性，例如审计文件的最大大小和年龄。这使您可以检查操作系统和XML审计文件的文件大小。

DBMS_AUDIT_MGMT子程序使您可以对所有审计跟踪类型执行清理操作。可以根据上次存档时间戳值清除审计跟踪。最后一个存档时间戳表示安全存档的最新审计记录的时间戳。

DBMS_AUDIT_MGMT包提供了一个子程序，使审计管理员可以为存档的审计记录设置上次存档时间戳。外部归档系统也可以使用该子程序来设置上一个归档时间戳。

DBMS_AUDIT_MGMT子程序还允许您配置定期删除审计跟踪记录的作业。审计管理员可以控制这些作业应运行的频率。

26.2 DBMS_AUDIT_MGMT已弃用的子程序

Oracle建议您不要在新应用程序中使用已弃用的子程序。对不推荐使用的功能的支持仅用于向后兼容。

从Oracle Database 12 c发行版12.2开始， DBMS_AUDIT_MGMT包中已弃用以下内容：

  • DBMS_AUDIT_MGMT.FLUSH_UNIFIED_AUDIT_TRAIL程序

  • DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_PROPERTY过程的AUDIT_TRAIL_PROPERTY参数的AUDIT_TRAIL_WRITE模式

这些不再是必需的，因为审计记录现在绕过公共日志记录基础结构队列并直接写入新的内部关系表。

26.3 DBMS_AUDIT_MGMT安全模型

所有DBMS_AUDIT_MGMT子程序都要求用户对DBMS_AUDIT_MGMT程序包具有EXECUTE特权。默认情况下， SYSDBA和AUDIT_ADMIN角色对包具有EXECUTE特权。

Oracle强烈建议只有审计管理员才能拥有DBMS_AUDIT_MGMT包的EXECUTE特权， DBMS_AUDIT_MGMT授予AUDIT_ADMIN角色。

26.7.1 ALTER_PARTITION_INTERVAL过程

此过程更改统一审计内部关系表的分区间隔。

句法

DBMS_AUDIT_MGMT.ALTER_PARTITION_INTERVAL（interval_number IN BINARY_INTEGER，interval_frequency IN VARCHAR2）;

参数

表26-8 ALTER_PARTITION_INTERVAL过程参数

参数 描述

interval_number

设置数据库为统一审计内部关系表创建分区的频率。例如，要指定每两天创建一个分区，必须输入2 。

interval_frequency

设置interval_number设置中设置的值的频率。例如，对于每两天创建一个分区，并将interval_number设置为2 ，必须将interval_frequency设置为DAY 。

支持的值为YEAR ， MONTH和DAY 。

使用说明

  • 您选择的间隔频率取决于数据库中生成的审计记录的速率。默认设置为一个月。如果您具有较高的审核记录速率并且使用默认值，则可能会在同一分区中生成太多审核记录。在这种情况下，您应该将间隔频率更改为更频繁的间隔，例如一个月或一天。如果审计记录速率生成不是那么高，那么您可能希望将其保持为一个月的默认值。

例

以下示例将分区间隔设置为每两个月发生一次。

BEGIN DBMS_AUDIT_MGMT.ALTER_PARTITION_INTERVAL（interval_number => 2，interval_frequency =>'MONTH'）;结束;

26.7.2 CLEAN_AUDIT_TRAIL程序

此过程将删除审计跟踪记录。

该CLEAN_AUDIT_TRAIL后过程，通常叫做SET_LAST_ARCHIVE_TIMESTAMP程序已经被用于设置审计记录的最后一个归档时间戳。

句法

DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL（audit_trail_type IN PLS_INTEGER，use_last_arch_timestamp IN BOOLEAN DEFAULT TRUE，container IN PLS_INTEGER DEFAULT CONTAINER_CURRENT，database_id IN NUMBER DEFAULT NULL，container_guid IN VARCHAR2 DEFAULT NULL）;

参数

表26-9 CLEAN_AUDIT_TRAIL过程参数

参数 描述

audit_trail_type

需要执行清理操作的审计跟踪类型。表26-1中列出了审计跟踪类型。

use_last_arch_timestamp

指定是否应使用上次存档的时间戳来确定应删除的记录。

值TRUE表示只应删除在上一个存档时间戳之前创建的审计记录。

值FALSE表示应删除所有审计记录。

默认值为TRUE 。 Oracle建议使用此值，因为这有助于防止意外删除记录。

容器

值： CONTAINER_CURRENT用于连接可插拔数据库（PDB）或CONTAINER_ALL用于可插拔数据库（PDBS）。当CONTAINER设置为CONTAINER_ALL ，这将清除所有PDB中的审计跟踪，否则它仅从连接的PDB中清除。

database_id的

要清理的审计记录的数据库ID（DBID）

container_guid

要清理的审计记录的容器GUID

注意：此参数已弃用，但目前保留用于向后兼容。

使用说明

以下使用说明适用：

  • 清理操作系统（OS）或XML审计文件时，仅清除当前审计目录中由AUDIT_FILE_DEST参数指定的文件。

  • 对于Windows平台，当audit_trail_type参数设置为DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS.时，不会执行清理DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS.这是因为Windows上的操作系统（OS）审核记录将写入Windows事件查看器。

  • 对于Unix平台，对于将操作系统（OS）审核记录写入syslog的情况，不执行清理。当audit_trail_type参数设置为DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS,它仅删除AUDIT_FILE_DEST初始化参数指定的目录下的*.aud文件。

    也可以看看：

    Oracle数据库参考中的“AUDIT_SYSLOG_LEVEL”

  • 当audit_trail_type参数设置为DBMS_AUDIT_MGMT.AUDIT_TRAIL_XML,此过程仅从当前审计目录中删除XML审计文件（ *.xml ）。

    Oracle数据库维护XML审计文件的簿记文件(adx_ $ORACLE_SID .txt ）。清理过程不会删除此文件。

  • 如果在use_last_arch_timestamp参数设置为TRUE时执行统一审计跟踪的清理：

    • 如果为清除操作设置database_id值，则此值将与CLEAN_AUDIT_TRAIL运行时的上一个存档时间戳一起CLEAN_AUDIT_TRAIL 。但是，对于清除溢出操作系统审计文件期间存在的统一审计记录，将忽略database_id值。操作系统审计文件的清理仅基于上一个存档时间戳。如果要将database_id值用于溢出操作系统审计文件中存在的统一审计跟踪记录的清理操作，请在运行DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL之前使用DBMS_AUDIT_MGMT.LOAD_UNIFIED_AUDIT_FILES过程将这些文件的内容加载到数据库表中DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL 。

    • 如果未为清除操作设置database_id值，则CLEAN_AUDIT_TRAIL将使用当前数据库容器的数据库ID。此ID与上一个存档时间戳值一起使用，同时清理数据库表中存在的统一审计记录。但是，对于溢出操作系统审计记录中存在的统一审计记录，不使用此数据库ID，并且这些记录的清除仅基于上一个存档时间戳值。如果要在清除溢出操作系统审计文件中存在的统一审计记录时包括数据库ID值，请在运行DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL之前使用DBMS_AUDIT_MGMT.LOAD_UNIFIED_AUDIT_FILES过程加载这些文件的内容。

  • 如果在use_last_arch_timestamp参数设置为FALSE时执行统一审计跟踪的清理：

    • 如果为清理操作设置database_id值，则使用此值，而CLEAN_AUDIT_TRAIL清除数据库表中存在的统一审计跟踪记录。但是， database_id值不用于清除溢出操作系统文件中存在的统一审计跟踪记录。如果希望在溢出操作系统审计文件中存在的统一审计记录的CLEAN_AUDIT_TRAIL操作中使用database_id值，则在运行DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL之前，使用DBMS_AUDIT_MGMT.LOAD_UNIFIED_AUDIT_FILES过程将这些文件的内容加载到数据库表中。 。

    • 如果在调用CLEAN_AUDIT_TRAIL过程时未设置database_id值，则Oracle数据库将清除所有统一审计记录，而不考虑database_id值，而不管统一审计记录所在的位置（即数据库表或溢出操作系统审计文件）居住。

例子

以下示例调用CLEAN_AUDIT_TRAIL过程来清除在上一个归档时间戳之前更新的操作系统（OS）审计跟踪记录。 

BEGIN DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL（audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS，use_last_arch_timestamp => TRUE）;结束;

26.7.13 INIT_CLEANUP过程

此过程为审计跟踪记录设置审计管理基础结构和默认清除间隔。

INIT_CLEANUP过程当前与AUDIT_TRAIL_OS ， AUDIT_TRAIL_XML和AUDIT_TRAIL_FILES审计跟踪类型无关。这些审计跟踪类型不需要初步设置。

26.7.17 SET_AUDIT_TRAIL_PROPERTY过程

此过程为指定的审计跟踪类型设置审计跟踪属性。

该过程为操作系统（OS）和XML审计跟踪类型设置属性OS_FILE_MAX_SIZE ， OS_FILE_MAX_AGE和FILE_DELETE_BATCH_SIZE 。OS_FILE_MAX_SIZE和OS_FILE_MAX_AGE属性确定在创建新审计跟踪文件之前审计跟踪文件的最大大小和OS_FILE_MAX_AGE 。FILE_DELETE_BATCH_SIZE属性指定在一个批处理中删除的审计跟踪文件的数量。

该过程为数据库审计跟踪类型设置属性DB_DELETE_BATCH_SIZE和CLEANUP_INTERVAL 。DB_DELETE_BATCH_SIZE指定从审计跟踪表中删除记录的批处理大小。这确保了如果清理操作在中途被中断，则该进程在下次调用时不需要重新开始。这是因为最后一个处理批次之前的所有批次都已提交。

CLEANUP_INTERVAL指定调用清除过程的频率（以小时为单位）。

句法

DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_PROPERTY（audit_trail_type IN PLS_INTEGER，audit_trail_property IN PLS_INTEGER，audit_trail_property_value IN PLS_INTEGER）;

参数

表26-23 SET_AUDIT_TRAIL_PROPERTY过程参数

参数 描述

audit_trail_type

需要为其设置属性的审计跟踪类型。表26-1中列出了审计跟踪类型

audit_trail_property

正在设置的审计跟踪属性。表26-2中列出了审计跟踪属性

audit_trail_property_value

使用audit_trail_property指定的属性的值。以下是审计跟踪属性的有效值：

  • OS_FILE_MAX_SIZE的最小值为1，最大值为2000000。默认值为10000。OS_FILE_MAX_SIZE以千字节（KB）为单位。

  • OS_FILE_MAX_AGE的最小值为1，最大值为497。默认值为5。OS_FILE_MAX_AGE以天为单位。

  • DB_DELETE_BATCH_SIZE的最小值为100，最大值为1000000。默认值为10000。DB_DELETE_BATCH_SIZE是以一批中删除的审计记录数量来衡量的。

  • FILE_DELETE_BATCH_SIZE的最小值为100，最大值为1000000。默认值为1000。FILE_DELETE_BATCH_SIZE测量为一批中删除的审计文件数。

  • CLEANUP_INTERVAL的最小值为1，最大值为999。使用INIT_CLEANUP过程设置默认值。CLEANUP_INTERVAL以小时计算。

使用说明

以下使用说明适用：

  • 未明确设置值的审计跟踪属性使用其默认值。

  • 如果为操作系统（OS）或XML审计跟踪类型设置了OS_FILE_MAX_SIZE和OS_FILE_MAX_AGE属性，则会根据首先达到这两个限制中的哪一个来创建新的审计跟踪文件。

    例如，让我们看一个OS_FILE_MAX_SIZE为10000且OS_FILE_MAX_AGE为5的场景。如果操作系统审核文件已超过5天且大小为9000 KB，则会打开新的审核文件。这是因为已达到其中一个限制。

  • 需要为AUDIT_TRAIL_AUD_STD和AUDIT_TRAIL_FGA_STD审计跟踪类型单独设置DB_DELETE_BATCH_SIZE属性。您无法使用AUDIT_TRAIL_DB_STD和AUDIT_TRAIL_ALL审计跟踪类型共同设置此属性。

  • DB_DELETE_BATCH_SIZE属性使您可以控制一批中删除的审核记录的数量。为此参数设置较大的值需要增加撤消日志空间的分配。

  • 需要为AUDIT_TRAIL_OS和AUDIT_TRAIL_XML审计跟踪类型单独设置FILE_DELETE_BATCH_SIZE属性。您无法使用AUDIT_TRAIL_FILES和AUDIT_TRAIL_ALL审计跟踪类型共同设置此属性。

  • 使用FILE_DELETE_BATCH_SIZE属性可以控制一批中删除的审计文件数。设置非常大的值可能会长时间参与GEN0后台进程。

  • 在Oracle Database Standard Edition中，您只能将表空间关联一次以进行统一审计。在为统一审计跟踪生成任何审计记录之前，应执行此关联。默认表空间是SYSAUX 。关联表空间后，无法在Standard Edition上对其进行修改，因为标准版不支持分区功能。

例子

以下示例调用SET_AUDIT_TRAIL_PROPERTY过程为操作系统（OS）审计跟踪设置OS_FILE_MAX_SIZE属性。它将此属性值设置为102400。这意味着每次当前审计文件大小达到100 MB时都会创建一个新的审计文件。

BEGIN DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_PROPERTY（audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS，audit_trail_property => DBMS_AUDIT_MGMT.OS_FILE_MAX_SIZE，audit_trail_property_value => 102400 / * 100MB * /）;结束;

以下示例调用SET_AUDIT_TRAIL_PROPERTY过程为操作系统（OS）审计跟踪设置OS_FILE_MAX_AGE属性。它将此属性值设置为5。这意味着每六天就会创建一个新的审计文件。 

BEGIN DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_PROPERTY（audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_OS，audit_trail_property => DBMS_AUDIT_MGMT.OS_FILE_MAX_AGE，audit_trail_property_value => 5 / *天* /）;结束;

以下示例调用SET_AUDIT_TRAIL_PROPERTY过程为AUDIT_TRAIL_AUD_STD审计跟踪设置DB_DELETE_BATCH_SIZE属性。它将此属性值设置为100000。这意味着在清理操作期间， SYS.AUD$ 100000的批量从SYS.AUD$表中删除审计记录。 

BEGIN DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_PROPERTY（audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_AUD_STD，audit_trail_property => DBMS_AUDIT_MGMT.DB_DELETE_BATCH_SIZE，audit_trail_property_value => 100000 / *删除批量大小* /）;结束;
