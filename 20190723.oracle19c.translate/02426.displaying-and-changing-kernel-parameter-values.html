显示和更改内核参数值

使用这些过程显示为资源控件指定的当前值，并在必要时更改它们：

显示资源控制值

  1. 要显示资源控件的当前值：

    $ id -p //验证项目id uid = 100（oracle）gid = 100（dba）projid = 1（group.dba）$ prctl -n project.max-shm-memory -i project group.dba $ prctl -n project.max-sem-ids -i project group.dba

  2. 要更改当前值，请使用prctl命令。例如：

    • 要将max-shm-memory的值修改为6 GB：

      #prctl -n project.max-shm-memory -v 6gb -r -i project group.dba

    • 要将max-sem-ids的值修改为256：

      #prctl -n project.max-sem-ids -v 256 -r -i project group.dba

    注意：

    使用prctl命令（资源控制）更改系统参数时，无需重新启动系统即可使这些参数更改生效。但是，更改的参数在系统重新启动后不会保留。

修改资源控制值

使用以下过程修改资源控制项目设置，以便在系统重新启动后它们保持不变：

  1. 默认情况下，Oracle实例作为dba组的oracle用户运行。创建名为group.dba项目以用作oracle用户的默认项目。运行id命令验证oracle用户的默认项目：

    #su  -  oracle $ id -p uid = 100（oracle）gid = 100（dba）projid = 100（group.dba）$ exit

  2. 要将最大共享内存大小设置为2 GB，请运行projmod命令：

    #projmod -sK“project.max-shm-memory =（privileged，2G，deny）”group.dba

    或者，将资源控制值project.max-shm-memory=(privileged,2147483648,deny)到Oracle项目的项目条目的最后一个字段。

  3. 检查/etc/project文件的值：

    #cat / etc / project

    输出类似于以下内容：

    system：0 :::: user.root：1 :::: noproject：2 :::: default：3 :::: group.staff：10 :::: group.dba：100：Oracle默认项目:: ：project.max-shm-memory =（特权，2147483648，拒绝）

  4. 要验证资源控制是否处于活动状态，请检查进程所有权，然后运行命令id和prctl ：

    #su  -  oracle $ id -p uid = 100（oracle）gid = 100（dba）projid = 100（group.dba）$ prctl -n project.max-shm-memory -i process $$ process：5754：-bash NAME PRIVILEGE VALUE FLAG ACTION RECIPIENT project.max-shm-memory privileged 2.00GB  -  deny

注意：

最大共享内存的值取决于SGA要求，应设置为大于SGA大小的值。
