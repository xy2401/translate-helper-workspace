1.160 LDAP_DIRECTORY_ACCESS

LDAP_DIRECTORY_ACCESS指定Oracle是否引用Oracle Internet Directory以获取用户身份验证信息。

属性 描述

参数类型

串

句法

LDAP_DIRECTORY_ACCESS = {NONE |密码| SSL}

默认值

没有

可修改

改变系统

可在PDB中修改

没有

基本

没有

如果启用了目录访问，则此参数还指定用户的身份验证方式。

值

  • 没有

    Oracle未提及Oracle Internet Directory以获取企业用户安全性信息。

  • 密码

    Oracle尝试使用存储在数据库钱包中的数据库密码连接到企业目录服务。如果失败，则Oracle Internet Directory连接失败，数据库将无法在企业用户登录时检索企业角色和架构映射。

  • SSL

    Oracle尝试使用SSL连接到Oracle Internet Directory。

将LDAP_DIRECTORY_ACCESS与PDB一起使用

使用默认数据库钱包位置时，PDB可以使用Oracle Internet Directory的密码和SSL身份验证。

由于每个CDB只有一个LDAP_DIRECTORY_ACCESS初始化参数，因此CDB中的所有PDB都将使用参数指定的Oracle Internet Directory身份验证。

对于CDB，默认数据库钱包路径为：

ORACLE_BASE/admin/ db-unique-name / pdb-GUID /wallet （如果设置了ORACLE_BASE ）

要么：

ORACLE_HOME/admin/ db-unique-name / pdb-GUID /wallet （如果未设置ORACLE_BASE ）

例外情况是根数据库，它的默认钱包路径为：

ORACLE_BASE/admin/ db-unique-name /wallet （如果设置了ORACLE_BASE ）

要么：

ORACLE_HOME/admin/ db-unique-name /wallet （如果未设置ORACLE_BASE ）

CDB中的所有PDB都具有相同的数据库唯一名称。通过将钱包放在默认位置，每个PDB都可以拥有自己的标识。请注意，由于CDB只有一个sqlnet.ora文件，因此CDB不支持sqlnet.ora的钱包位置，因为每个PDB必须有自己的钱包。

注意：

Oracle数据库使用Database Configuration Assistant（DBCA）在Oracle Internet Directory中注册。要使用Oracle Internet Directory注册工作，必须使用DBCA注册CDB的所有PDB。

对于非CDB，默认数据库钱包路径为：

ORACLE_BASE/admin/ db-unique-name /wallet （如果设置了ORACLE_BASE ）

要么：

ORACLE_HOME/admin/ db-unique-name /wallet （如果未设置ORACLE_BASE ）
