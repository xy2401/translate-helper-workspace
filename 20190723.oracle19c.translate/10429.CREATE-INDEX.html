创建指数

目的

使用CREATE INDEX语句在以下内容上创建索引：

  • 表的一个或多个列，分区表，索引组织表或集群

  • 表或集群的一个或多个标量类型对象属性

  • 用于索引嵌套表列的嵌套表存储表

索引是一个架构对象，它包含显示在表或集群的索引列中的每个值的条目，并提供对行的直接，快速访问。单个索引条目的最大大小取决于数据库的块大小。

Oracle数据库支持几种类型的索引：

  • 正常指数。 （默认情况下，Oracle数据库会创建B树索引。）

  • 位图索引 ，将与键值关联的rowid存储为位图。

  • 分区索引 ，由分区组成，分区包含表中索引列中显示的每个值的条目。

  • 基于函数的索引 ，基于表达式。它们使您能够构造查询，以评估表达式返回的值，而表达式又可以包含内置函数或用户定义的函数。

  • 域索引 ，是indextype类型的特定于应用程序的索引的indextype 。

注意：

  • 用于讨论索引的Oracle数据库概念

  • 有关与索引大小相关的限制的更多信息，请参阅Oracle数据库参考

  • ALTER INDEX和DROP INDEX

先决条件

要在您自己的架构中创建索引，必须满足下列条件之一：

  • 要编制索引的表或集群必须位于您自己的架构中。

  • 您必须对要编制索引的表具有INDEX对象特权。

  • 您必须具有CREATE ANY INDEX系统特权。

要在另一个模式中创建索引，您必须具有CREATE ANY INDEX系统特权。此外，包含索引的模式的所有者必须具有表空间上的UNLIMITED TABLESPACE系统特权或空间配额，以包含索引或索引分区。

要创建基于函数的索引，除了创建常规索引的先决条件之外，如果索引基于用户定义的函数，那么这些函数必须标记为DETERMINISTIC 。使用索引所有者的凭据执行基于函数的索引，因此索引所有者必须具有该函数的EXECUTE对象特权。

要在自己的模式中创建域索引，除了创建常规索引的先决条件之外，还必须对indextype具有EXECUTE对象特权。如果要在另一个用户的模式中创建域索引，则索引所有者还必须具有indextype及其底层实现类型的EXECUTE对象特权。在创建域索引之前，应首先定义indextype。

语义

独特

指定UNIQUE以指示索引所基于的列（或列）的值必须是唯一的。

对唯一索引的限制

唯一索引受以下限制：

  • 您不能同时指定UNIQUE和BITMAP 。

  • 您不能为域索引指定UNIQUE 。

也可以看看：

“ 唯一约束 ”，用于满足满足唯一约束的条件的信息

位图

指定BITMAP以指示index是与一个位图被创建为每个不同的键，而不是单独索引的每一行。位图索引将与键值关联的rowid存储为位图。位图中的每个位对应于可能的rowid。如果该位置位，则表示具有相应rowid的行包含键值。位图的内部表示最适合具有低级别并发事务的应用程序，例如数据仓库。

注意：

除了位图索引之外，Oracle不会将所有键列都为null的表行编入索引。因此，如果要在表的所有行上建立索引，则必须为索引键列指定NOT NULL约束或创建位图索引。

位图索引的限制

位图索引受以下限制：

  • 创建全局分区索引时，无法指定BITMAP 。

  • 除非索引组织表具有与之关联的映射表，否则无法在索引组织表上创建位图二级索引。

  • 您不能同时指定UNIQUE和BITMAP 。

  • 您不能为域索引指定BITMAP 。

  • 位图索引最多可包含30列。

table_index_clause的 限制

本条款受以下限制：

  • 如果index是本地分区的，则必须对table进行分区。

  • 如果table是索引组织的，则此语句将创建辅助索引。索引包含索引键和索引组织表的逻辑rowid。逻辑rowid排除也是索引键一部分的列。您不能为此二级索引指定REVERSE ，并且索引键和逻辑rowid的组合大小应小于块大小。

  • 如果table是一个临时表，则index也将是暂时的具有相同范围（会话或事务），为table 。以下限制适用于临时表的索引：

    • 您可以指定的index_properties的唯一部分是index_attributes 。

    • 在index_attributes ，您不能指定physical_attributes_clause ， parallel_clause ， logging_clause或TABLESPACE 。

    • 您无法在临时表上创建域索引或分区索引。

  • 您无法在外部表上创建索引。

在扩展数据类型列上创建索引

如果column是扩展数据类型列，则在尝试创建索引时可能会收到“超出最大密钥长度”错误。索引的最大密钥长度取决于数据库块大小和存储在块中的一些其他索引元数据。例如，对于使用Oracle标准8K块大小的数据库，最大密钥长度约为6400字节。

若要解决此问题，您必须使用以下方法之一缩短要索引的值的长度：

  • 创建基于函数的索引，以缩短存储在扩展数据类型列中的值，作为用于索引定义的表达式的一部分。

  • 创建虚拟列以缩短存储在扩展数据类型列中的值，作为用于虚拟列定义的表达式的一部分，并在虚拟列上构建普通索引。使用虚拟列还可以利用常规列的功能，例如收集统计信息以及使用约束和触发器。

对于这两种方法，您可以使用SUBSTR或STANDARD_HASH函数来缩短扩展数据类型列的值以构建索引。这些方法具有以下优点和缺点：

  • 使用SUBSTR函数返回索引键可接受长度的column的子字符串或前缀。此类索引可用于原始列上的相等，IN列表和范围谓词，而无需将SUBSTR列指定为谓词的一部分。有关更多信息，请参阅SUBSTR 。

  • 使用STANDARD_HASH函数可能会创建一个比基于子字符串的索引更紧凑的索引，并且可能导致更少的不必要的索引访问。此类索引可用于原始列上的相等和IN列表谓词，而无需将SUBSTR列指定为谓词的一部分。有关更多信息，请参阅STANDARD_HASH 。

以下示例显示如何在扩展数据类型列上创建基于函数的索引：

CREATE INDEX 索引 ON 表 （SUBSTR（ 列 ，0， n ））;

对于n，请指定足够大的前缀长度，以区分column值。

以下示例说明如何为扩展数据类型列创建虚拟列，然后在虚拟列上创建索引：

ALTER TABLE 表 ADD（ new_hash_column AS（STANDARD_HASH（ column ）））; CREATE INDEX 索引 ON 表 （ new_hash_column ）;

索引列的限制

以下限制适用于索引列：

  • 您无法在类型为用户定义， LONG ， LONG RAW ，LOB或REF列或属性上创建索引，但Oracle数据库支持REF类型列的索引或已使用SCOPE子句定义的属性。

  • 只能在加密列上创建普通（B树）索引，它们只能用于相等搜索。

基于函数的索引的注释

以下说明适用于基于函数的索引：

  • 当您随后查询使用基于函数的索引的表时，Oracle数据库将不会使用索引，除非查询过滤掉空值。但是，Oracle数据库将在查询中使用基于函数的索引，即使WHERE子句中指定的列的顺序与定义基于函数的索引的column_expression中的顺序不同。

    也可以看看：

    “ 基于功能的索引示例 ”

  • 如果索引所基于的函数变为无效或被删除，则Oracle数据库会将索引标记为DISABLED 。如果优化程序选择使用索引，则DISABLED索引上的查询将失败。除非索引也标记为UNUSABLE 且参数SKIP_UNUSABLE_INDEXES设置为true否则DISABLED索引上的DML操作将失败。有关此参数的更多信息，请参阅ALTER SESSION 。

  • 如果在column_expression使用函数，包或类型的公共同义词，并且稍后在表所有者的模式中创建具有相同名称的实际对象，则Oracle数据库将禁用基于函数的索引。当您随后使用ALTER INDEX启用基于函数的INDEX ...ENABLE或ALTER INDEX ...REBUILD ， column_expression使用的函数，包或类型继续解析为公共同义词最初指向的函数，包或类型。它不会解析为新的函数，包或类型。

  • 如果基于函数的索引的定义生成到字符数据的内部转换，则在更改NLS参数设置时请小心。基于函数的索引使用NLS参数的当前数据库设置。如果在会话级别重置这些参数，则使用基于函数的索引的查询可能会返回不正确的结果。排序规则参数有两个例外（ NLS_SORT和NLS_COMP ）。Oracle数据库正确处理转换，即使这些转换已在会话级别重置。

  • 即使明确请求转换，Oracle数据库也无法在所有情况下转换数据。例如，尝试使用TO_NUMBER函数将字符串'105 lbs'从VARCHAR2为NUMBER失败，并显示错误。因此，如果column_expression包含数据转换函数（如TO_NUMBER或TO_DATE ，并且后续INSERT或UPDATE语句包含转换函数无法转换的数据，则索引将导致INSERT或UPDATE语句失败。

  • 如果column_expression包含日期时间格式模型，则定义列的基于函数的索引表达式可能包含与指定格式不同的格式元素。例如，使用yyyy datetime格式元素定义基于函数的索引：

    CREATE INDEX cust_eff_ix ON客户（NVL（cust_eff_to，TO_DATE（'9000-01-01 00:00:00'，' yyyy -mm-dd hh24：mi：ss'）））;

    查询ALL_IND_EXPRESSIONS视图以查看定义列的基于函数的索引表达式使用syyyy datetime格式元素：

    SELECT column_expression FROM all_ind_expressions WHERE index_name ='CUST_EFF_IX'; COLUMN_EXPRESSION ------------------------------------------------- ----------------------------- NVL（“CUST_EFF_TO”，TO_DATE（'9000-01-01 00:00:00'， ' syyyy -mm-dd hh24：mi：ss'））

基于函数的索引的限制

基于函数的索引受以下限制：

  • column_expression引用的函数返回的值受到与B树索引的索引列相同的限制。请参阅“ 索引列的限制 ” 。

  • 必须将column_expression引用的任何用户定义函数声明为DETERMINISTIC 。

  • 对于基于函数的全局分区索引， column_expression不能是分区键。

  • column_expression可以是Column Expressions中描述的任何表达形式。

  • 必须用括号指定所有函数，即使它们没有参数。否则，Oracle Database会将它们解释为列名。

  • 您在column_expression指定的任何函数都必须返回可重复的值。例如，您不能指定SYSDATE或USER函数或ROWNUM伪列。

ASC | DESC

使用ASC或DESC指示是否应按升序或降序创建索引。字符数据的索引按数据库字符集中字符值的升序或降序创建。

Oracle数据库将降序索引视为基于函数的索引。与其他基于函数的索引一样，在您首次分析索引和定义索引的表之前，数据库不使用降序索引。请参阅此语句的column_expression子句。

升序唯一索引允许多个NULL值。但是，在降序唯一索引中，多个NULL值被视为重复值，因此不允许。

对升序和降序指数的限制

您不能为域索引指定这些子句中的任何一个。您不能为反向索引指定DESC 。如果index是位图或者COMPATIBLE初始化参数设置为小于8.1.0的值，则Oracle数据库会忽略DESC 。

index_attributes

指定可选的索引属性。

physical_attributes_clause

使用physical_attributes_clause为索引建立物理和存储特征的值。

如果省略此子句，则Oracle数据库将PCTFREE设置为10，将INITRANS为2。

TABLESPACE

对于tablespace ，请指定用于保存索引，索引分区或索引子分区的表空间的名称。如果省略此子句，则Oracle数据库会在包含索引的架构的所有者的默认表空间中创建索引。

对于本地索引，您可以指定关键字DEFAULT来代替tablespace 。添加到本地索引的新分区或子分区将在与基础表的相应分区或子分区相同的表空间中创建。

index_compression

index_compression子句允许您为索引启用或禁用索引压缩。指定prefix_compression的COMPRESS子句以启用索引的前缀压缩，指定advanced_index_compression的COMPRESS ADVANCED子句以对索引启用高级索引压缩，或指定prefix_compression或advanced_index_compression的NOCOMPRESS子句以禁用索引的压缩。默认值为NOCOMPRESS 。

如果要对分区索引使用压缩，则必须在索引级别创建启用了压缩的索引。随后，您可以为此类分区索引的各个分区启用和禁用压缩设置。您还可以在重建单个分区时启用和禁用压缩。您可以修改现有的非分区索引，以便仅在重建索引时启用或禁用压缩。

prefix_compression

指定COMPRESS以启用前缀 压缩 （也称为密钥压缩），从而消除重复出现的键列值。使用integer指定前缀长度（要压缩的前缀列数）。您可以为至少两列的非唯一索引或唯一索引的索引指定前缀压缩。

  • 对于唯一索引，有效前缀长度值的范围是从1到键列数减1。默认前缀长度是键列数减1。

  • 对于非唯一索引，有效前缀长度值的范围是从1到键列的数量。默认前缀长度是键列的数量。

advanced_index_compression

指定此子句以启用高级索引压缩 。高级索引压缩可显着提高压缩率，同时仍可提供对索引的有效访问。因此，高级索引压缩适用于所有支持的索引，包括那些不适合前缀压缩的索引。

  • COMPRESS ADVANCED LOW - 此级别压缩索引低于HIGH级别，但提供对索引的更快访问。您可以为至少两列的非唯一索引或唯一索引的索引指定此子句。在启用COMPRESS ADVANCED LOW之前，数据库必须为12 。 1 。 0或更高的兼容级别。

  • COMPRESS ADVANCED HIGH - 此级别压缩索引超过LOW级别，但提供对索引的较慢访问。您可以为一列或多列的非唯一索引或唯一索引的索引指定此子句。在启用COMPRESS ADVANCED HIGH之前，数据库必须为12 。 2 。 0或更高的兼容级别。

如果省略LOW和HIGH关键字，则默认值为HIGH 。

  • 您不能为索引组织表指定高级索引压缩。

使用bitmap_join_index_clause定义位图连接索引 。位图连接索引在单个表上定义。对于由维度表列组成的索引键，它存储与该键对应的事实表rowid。在数据仓库环境中，定义索引的表通常称为事实表 ，与此表连接的表通常称为维表 。但是，星型模式不是创建连接索引的必要条件。

从

在FROM子句中，指定连接的表。

哪里

在WHERE子句中，指定连接条件。

如果基础事实表已分区，则还必须指定local_partitioned_index子句之一（请参阅local_partitioned_index ）。

位图连接索引的限制

除了对位图索引的限制（请参阅BITMAP ）之外，以下限制适用于位图连接索引：

  • 您无法在临时表上创建位图连接索引。

  • FROM子句中不会出现两次表。

  • 您无法创建基于函数的连接索引。

  • 维度表列必须是主键列或具有唯一约束。

  • 如果维度表具有复合主键，则主键中的每个列都必须是连接的一部分。

  • 除非事实表是分区的，否则不能指定local_partitioned_index子句。

  • 位图连接索引定义只能引用具有排序规则BINARY ， USING_NLS_COMP ， USING_NLS_SORT或USING_NLS_SORT_CS 。对于任何这些排序规则，将对索引关键字进行排序，并使用BINARY排序规则评估连接条件。有关更多信息，请参见“ Oracle数据库全球化支持指南

您可以指定USABLE和UNUSABLE关键字：

  • 对于索引分区，在on_range_partitioned_table ， on_list_partitioned_table ， on_hash_partitioned_table和on_comp_partitioned_table子句中

  • 对于索引子分区，请在index_subpartition_clause

对于非分区索引，请指定UNUSABLE以在不可用状态下创建索引。必须先重建，删除和重新创建不可用的索引，然后才能使用它。指定USABLE以在可用状态下创建索引。USABLE是默认值。

对于分区索引，请指定USABLE或UNUSABLE ，如下所示：

  • 如果为索引指定UNUSABLE ，则所有索引分区都标记为UNUSABLE 。

  • 如果为索引指定USABLE ，则所有索引分区都标记为USABLE 。

  • 如果未为索引指定USABLE或UNUSABLE ，则所有索引分区都标记为USABLE 。例外是本地部分索引。如果指定了LOCAL和INDEXING PARTIAL条款，并没有指定USABLE或UNUSABLE ，那么每个索引分区标记USABLE如果对应的表分区的索引属性为ON ，或UNUSABLE如果其对应的表分区的索引属性为OFF 。

您可以通过为特定索引分区指定USABLE或UNUSABLE来覆盖前面的条件。

如果基础表是复合分区表，则索引分区和表分区的前述条件将应用于索引子分区和表子分区。

创建分区索引后，您可以选择重建特定索引分区或子分区，以使他们USABLE 。如果要仅在某些索引分区或子分区上维护索引，则这样做非常有用 - 例如，如果要为新分区启用索引访问，而对旧分区不启用索引访问。

当索引或索引的某些分区或子分区创建为UNUSABLE ，不会为不可用对象分配任何段。不可用的索引或索引分区在数据库中不占用空间。

如果索引或索引的某些分区或子分区标记为UNUSABLE ，则仅在以下情况下，优化程序才会将索引视为访问路径：优化程序必须在编译时知道要访问哪些分区，并且所有要访问的分区都必须标记为USABLE 。因此，查询不能包含任何绑定变量。

对可替换列进行索引：示例

您可以为可替换列的声明类型的属性构建索引。此外，您可以使用适当的TREAT函数引用子类型属性。以下示例使用在“可替换表和列示例 ”中创建的表books 。该语句在books表中的所有员工作者的salary属性上创建索引：

CREATE INDEX salary_i ON books（TREAT（作者AS employee_t）.salary）;

TREAT函数的参数中的目标类型必须是添加了被引用属性的类型。在该示例中， TREAT的目标是employee_t ，这是添加salary属性的类型。

如果不满足此条件，则Oracle数据库将TREAT函数解释为任何函数表达式，并将索引创建为基于函数的索引。例如，以下语句在兼职员工的salary属性上创建基于函数的索引，将空值分配给类型层次结构中所有其他类型的实例。 

CREATE INDEX salary_func_i ON人p（TREAT（VALUE（p）AS part_time_emp_t）.salary）;

您还可以使用SYS_TYPEID函数在可替换列的基础上的类型判别列上构建索引。

注意：

Oracle数据库使用类型判别列来评估涉及IS OF type条件的查询。typeid列的基数通常很低，因此Oracle建议您在这种情况下构建位图索引。

以下语句在books表的author列的typeid上创建位图索引：

CREATE BITMAP INDEX typeid_i ON books（SYS_TYPEID（author））;
