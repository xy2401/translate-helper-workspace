1.145 INMEMORY_SIZE

INMEMORY_SIZE设置内存区域的大小，其中包含数据库实例上的IM列存储（IM列存储）。

属性 描述

参数类型

大整数

句法

INMEMORY_SIZE = integer [K | M | G]

默认值

0

可修改

改变系统

可在PDB中修改

是

价值范围

0到其他分配后SGA中剩余的内存量

基本

没有

Oracle RAC

所有实例都应使用相同的值。

默认值为0，表示未使用IM列存储。

设置此参数后，必须重新启动数据库才能启用IM列存储。

可以设置此参数的最小大小为100 MB。

通常，此参数应至少设置为容纳将使用IM列存储的所有表所需的大小。可以将其设置得更高以允许这些表的增长或适应将来将使用IM列存储的其他表。

也可以为每个PDB设置此参数，以限制该PDB的IM列存储的最大大小。注意，PDB值的总和不必等于CDB值，并且PDB值的总和甚至可以大于CDB值。

除非在PDB上专门设置此参数，否则每个PDB都会继承CDB值，这意味着它们可以使用所有可用的IM列存储。

为此参数指定的值将计入SGA_TARGET 。例如，如果将SGA_TARGET设置为10 GB并将INMEMORY_SIZE设置为2 GB，则将SGA_TARGET设置的20％分配给内存区域。

与其他SGA组件（如缓冲区高速缓存和共享池）不同，内存区域大小不受自动内存管理控制。当缓冲区高速缓存或共享池需要更多内存时，数据库不会自动缩小内存区域，或者当空间不足时增加内存区域。您只能通过手动调整INMEMORY_SIZE初始化参数来增加内存区域的大小。

内存中动态扫描需要资源管理器。因此，当您将INMEMORY_SIZE的值从0更改为非零值时，将自动启用资源管理器。不需要特定的资源计划。
