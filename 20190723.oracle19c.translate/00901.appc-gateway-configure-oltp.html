7配置OLTP

以下部分详细介绍了如何配置OLTP。

  • 如果您的通信协议是SNA ：继续为SNA环境 配置OLTP为 SNA环境 配置OLTP 。

  • 如果您的通信协议是TCP / IP：继续为TCP / IP环境 配置OLTP为 TCP / IP环境 配置OLTP 。

    注意：

    在使用TCP / IP支持IMS Connect的网关上，如果需要输入区分大小写，则必须在IMS TRANSACT宏中指定EDIT=ULC 。使用SNA支持时，无需在IMS TRANSACT宏中指定EDIT=ULC 。

为SNA环境配置OLTP

配置OLTP以与Oracle Database Gateway for APPC通信的步骤会有所不同，具体取决于您使用的OLTP以及运行OLTP的平台。适用于z / OS，IMS / TM，APPC / MVS和的CICS Transaction Server z / OS是目前支持的 OLTPs。从以下部分选择适合您的OLTP的说明：

  • “ 为z / OS配置CICS Transaction Server ”

  • “ 配置IMS / TM ”

  • “ 配置APPC / MVS ”

    注意：

    如果这是OLTP的首次配置，则只需执行OLTP的配置步骤。

为z / OS配置CICS Transaction Server

如果您的OLTP是用于z / OS的CICS Transaction Server，则执行以下步骤以将其配置为与网关通信：

  1. 配置MVS 用于将APPC连接到系统的SNA Server的VTAM。至少一个独立的 LU必须可用于网关。

  2. 检查 CICS Transaction Server for z / OS使用的VTAM日志模式表。 （表名在CICS的VTAM APPL定义中的MODETAB参数中指定。）确保具有并行会话和同步级别支持的APPC会话存在条目。

    该 oraplu62.asm文件中 %ORACLE_HOME%\dg4appc\sna目录包含示例模式条目，包括指示模式条目中所需值的注释。

  3. 使用文件传输工具，将以下文件从%ORACLE_HOME%\dg4appc\demo\CICS目录传输到运行CICS Transaction Server for z / OS的z / OS系统：

    • dfhcsdup.jcl - JCL运行CICS DFHCSDUP实用程序

    • pgaflip.asm - CICS FLIP事务的汇编源

    • pgaflip.jcl - 用于汇编和链接CICS FLIP事务的JCL

  4. 使用中的注释 dfhcsdup.jcl文件，定制JCL和输入语句以匹配您的系统设置，并提交它以进行批量执行。执行此步骤将更新Transaction Server for z / OS系统定义。

  5. 使用中的说明 pgaflip.jcl文件注释，定制JCL以匹配您的系统设置，并提交它以进行批量执行。执行此步骤组装并链接 pgaflip.asm文件到您的Transaction Server for z / OS可访问的加载模块库中 CICS启动过程中的DFHRPL DD语句。

  6. 登录到您的CICS Transaction Server for z / OS并输入以下事务：

    CEDA INSTALL GROUP（ORAPGA）

    此事务将安装与Windows上的网关进行APPC通信的CICS连接和会话定义。它还安装了随网关提供的示例CICS程序和事务的定义。

您的CICS Transaction Server for z / OS配置现已完成。

配置IMS / TM

如果您的OLTP是IMS / TM，则执行以下步骤以配置IMS / TM和z / OS以与网关通信：

  1. 配置IMS系统以与APPC一起使用。

  2. 配置MVS 用于SNA APPC连接到Windows的VTAM。除非您使用IMS，否则网关必须至少使用一个独立LU LU6.1适用于LU6.2应用程序的适配器。在这种情况下，你必须有一个依赖的为每个并发会话定义LU。例如，如果要支持10个并发会话，则必须定义10个相关LU。

  3. 检查 IMS / TM使用的VTAM日志模式表。表名由VTAM APPL定义中的MODETAB参数指定。

    对于APPC / IMS，请确保具有同步级支持和并行会话支持的APPC会话存在条目。oralu62.asm 和oraplu62.asm 中的文件 %ORACLE_HOME%\dg4appc\sna目录分别包含单会话和并行会话支持的示例模式条目。样本包含指示模式条目中所需值的注释。

  4. 使用文件传输工具，将以下文件从%ORACLE_HOME%\dg4appc\demo\IMS目录传输到运行IMS / TM的z / OS系统：

    • pgaflip.asm是汇编源代码 IMS FLIP交易;

    • pgaflip.jcl是用于汇编和链接IMS FLIP事务的JCL;

    • imsgen.asm是IMS FLIP事务的IMS阶段1 gen定义。

  5. 将imsgen.asm文件中的语句添加到IMS stage 1 gen并运行IMS stage 1和stage 2 gens。使用联机更改实用程序启用新的事务定义。

  6. 使用pgaflip.jcl文件中的注释，定制JCL以匹配您的系统设置并提交它以进行批量执行。这组装和联系 pgaflip.asm文件放入IMS / TM系统可访问的加载模块库中，并为FLIP事务创建PSB和ACB。

  7. 执行系统上所需的任务，以使新事务可用于IMS / TM。根据您的系统设置，您可能必须重新启动IMS。

    IMS / TM配置现已完成。

配置APPC / MVS

如果您的OLTP是APPC / MVS，请执行以下步骤以配置APPC / MVS以与网关通信：

  1. 配置MVS 用于SNA APPC连接到Windows的VTAM。至少有一个独立的LU必须可供网关使用。

  2. 检查 APPC / MVS使用的VTAM日志模式表。 （表名由APPC / MVS的VTAM APPL定义中的MODETAB参数指定。）确保具有SYNCLEVEL和并行会话支持的APPC会话存在条目。oraplu62.asm 文件中 %ORACLE_HOME%\dg4appc\sna目录包含示例模式条目，包括指示模式条目中所需值的注释。

  3. 分配分区数据集（ 放置样本文件的z / OS系统上的PDS）。PDS应分配RECFM=FB ， RECFM=FB LRECL=80 ，以及适合BLKSIZE在设备类型的BLKSIZE 。一个目录块需要大约两个3390磁盘空间的磁道。Oracle建议命名此分区数据集（PDS） ORAPGA.APPCMVS.SAMPLIB 。

  4. 使用文件传输工具，从中传输以下文件 %ORACLE_HOME%\dg4appc\demo\MVS目录到您在上一步中分配的z / OS PDS，使用以下指定的成员名称：

    • pgaflip.jcl是JCL，用于添加APPC / MVS TP配置文件并定义事务的执行环境。将此文件作为成员PGAFLIPJ存储在z / OS PDS中。

    • pgaflip.rex是APPC / MVS PGAFLIP事务的REXX源。将此文件作为成员PGAFLIP存储在z / OS PDS中。

  5. 使用pgaflip.jcl文件中的注释，定制JCL以匹配您的系统设置并提交它以进行批量执行。执行此步骤定义PGAFLIP事务的APPC / MVS TP配置文件，并将其存储在APPC / MVS配置文件数据集中。确保更改JCL中的数据集名称以匹配在步骤3中分配的z / OS PDS的名称。

APPC / MVS配置现已完成。

  • 现在您已使用SNA协议在网关上完成了网络配置，请继续使用SNA通信协议进行网关配置 。

  • 如果要配置commit-confirm，则可以在配置提交确认中找到此配置的说明。

为TCP / IP环境配置OLTP

这些是配置OLTP以使用TCP / IP for IMS Connect与Oracle Database Gateway for APPC进行通信的步骤。IMS / TM，通过IMS Connect，是此版本网关唯一支持的OLTP。

执行以下步骤以配置IMS / TM和z / OS以与网关通信：

  1. 配置IMS系统。
  2. 配置IMS Connect。

    有关如何配置IMS Connect的信息，请参阅“IBM IMS Connect指南和参考” 。

  3. 使用文件传输工具，将以下文件从%ORACLE_HOME%\dg4appc\demo\IMS目录传输到运行IMS / TM的z / OS系统：

    • pgaflip.asm是汇编源代码 IMS FLIP交易;

    • pgaflip.jcl是用于汇编和链接IMS FLIP事务的JCL;

    • imsgen.asm是IMS FLIP事务的IMS阶段1 gen定义。

  4. 将imsgen.asm文件中的语句添加到IMS stage 1 gen并运行IMS stage 1和stage 2 gens。使用联机更改实用程序启用新的事务定义。
  5. 使用pgaflip.jcl文件中的注释，定制JCL以匹配您的系统设置并提交它以进行批量执行。这组装和联系将pgaflip.asm文件放入IMS / TM系统可访问的加载模块库中，并为FLIP事务创建PSB和ACB。
  6. 执行系统上所需的任务，以使新事务可用于IMS / TM。根据您的系统设置，您可能必须重新启动IMS。

IMS / TM配置现已完成。

  • 此时，请继续使用TCP / IP通信协议进行网关配置，以完成网关及其组件的配置。
