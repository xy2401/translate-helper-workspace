B工作区管理器错误消息

本附录列出了Workspace Manager错误消息，包括每个消息的原因和建议的用户操作。

WM_ERROR_1列'string'的名称超过28个字符

原因：尝试对具有名称超过28个字符的列的表进行版本启用。

操作：确保表的所有列名称不超过28个字符。

工作空间不允许WM_ERROR_2 '字符串' ： '字符串'在'字符串'模式下冻结

原因：在已冻结的工作空间上执行了操作。

操作：在重试操作之前取消冻结工作空间。

WM_ERROR_3无法修改启用版本的表的主键值

原因：在启用版本的表上执行了修改主键约束中的一个或多个值的DML操作。

操作：不要对启用版本的表的主键约束中的列执行DML操作。

WM_ERROR_4此表上有未完成的空头交易。

原因： DisableVersioning失败，因为表上有打开的数据库事务要禁用版本。

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

违反了 WM_ERROR_5完整性约束（ 'string' 。 'string' ） - 找到子记录

原因：尝试使用RESTRICT选项删除或更新参照完整性约束的父表中的记录，并且在完整性约束的子表中存在匹配记录。RESTRICT是参照完整性约束的默认属性，另一个是ON DELETE CASCADE，如果删除父表中的相应行，则删除子表中的相关行。CASCADE选项仅适用于从父表中删除。父表的更新始终遵循RESTRICT选项。

操作：首先从子表中删除所有匹配的记录。

违反了 WM_ERROR_6完整性约束（ 'string' 。 'string' ） - 未找到父键

原因：尝试在参照完整性约束的子表中插入或更新记录，并且在完整性约束的父表中没有匹配的记录。

操作：首先在父表中插入匹配的记录。

WM_ERROR_7在导入平台上找不到WM

原因：导入启用版本的数据库失败，因为导入平台未安装Workspace Manager。

操作：在导入平台上安装Workspace Manager并重试。

WM_ERROR_8导入平台不能有任何版本化表

原因：导入已启用版本的数据库失败，因为导入平台已具有一个或多个启用版本的表。

操作：导入平台可能没有任何支持版本的表。导入平台上需要全新安装Workspace Manager。

WM_ERROR_9导入平台具有非“实时”工作空间或显式保存点

原因：启用版本数据库的导入失败，因为输入平台有两种非LIVE工作区中的层次结构的工作区或在明确的保存点LIVE工作区。

操作：导入平台可能只有LIVE工作区，并且可能没有显式保存点。导入平台上需要全新安装Workspace Manager。

WM_ERROR_10唯一键违规

原因：插入操作失败，因为它违反了表的主键约束。

操作：确保当前工作空间中的插入操作未违反主键。

WM_ERROR_11需要在最新版本上删除

原因：删除操作失败，因为删除是在非LATEST版本的工作区中进行的。

操作：使用GotoWorkspace或GotoSavepoint过程确保当前会话在工作区中的LATEST版本上。

WM_ERROR_12需要在最新版本上插入

原因：插入操作失败，因为插入是在非LATEST版本的工作空间中进行的。

操作：使用GotoWorkspace或GotoSavepoint过程确保当前会话在工作区中的LATEST版本上。

WM_ERROR_13需要在最新版本上进行更新

原因：更新操作失败，因为更新是在非LATEST版本的工作空间中进行的。

操作：使用GotoWorkspace或GotoSavepoint过程确保当前会话在工作区中的LATEST版本上。

WM_ERROR_14 '字符串' 。 'string'尚未启用版本

原因：此操作失败，因为它只能在启用版本的表上调用。

操作：验证表是否已启用版本。

工作空间名称中不允许使用 WM_ERROR_15“/”

原因： CreateWorkspace失败，因为工作区名称包含“/”。

操作：选择不包含“/”的其他工作空间名称。

WM_ERROR_16版本禁用另一个模式中的表需要“WM_ADMIN”系统特权

原因： DisableVersioning失败，因为只有具有WM_ADMIN系统特权的用户才能对另一个模式中的表进行版本禁用。

操作：在尝试对此表进行版本禁用之前，请确保调用用户具有所需的权限。否则，拥有表版本的所有者 - 禁用它。

WM_ERROR_17版本启用另一个模式中的表需要“WM_ADMIN”系统特权

原因： EnableVersioning失败，因为只有具有WM_ADMIN系统特权的用户才能对另一个模式中的表进行版本启用。

操作：在尝试对此表进行版本启用之前，请确保调用用户具有所需的权限。否则，让表的版本所有者启用它。

WM_ERROR_18需要“WM_ADMIN”系统权限或所有权才能更改工作空间属性

原因： AlterWorkspace失败，因为只有具有WM_ADMIN系统特权的用户或工作空间的所有者才能更改工作空间属性。

操作：在尝试更改工作空间属性之前，请确保调用用户具有所需的权限。否则，让工作空间的所有者更改工作空间属性。

WM_ERROR_19冻结工作空间需要“WM_ADMIN”系统特权或所有权

原因： FreezeWorkspace失败，因为只有具有WM_ADMIN系统特权的用户或工作区的所有者才能冻结工作区。

操作：在尝试冻结工作空间之前，确保调用用户具有所需的权限。否则，让工作区的所有者冻结它。

WM_ERROR_20“WM_ADMIN”系统权限或所有权是设置工作空间锁定模式所必需的

原因： SetWorkspaceLockModeOn失败，因为只有具有WM_ADMIN系统特权的用户或工作区的所有者才能设置工作区锁定模式。

操作：在尝试设置工作空间锁定模式之前，请确保调用用户具有所需的权限。否则，让工作空间的所有者设置工作空间锁定模式。

WM_ERROR_21更改保存点属性的权限不足

原因： AlterSavepoint失败，因为只有具有WM_ADMIN系统特权的用户或工作空间或保存点的所有者才能更改保存点属性。

操作：在尝试更改保存点属性之前，请确保调用用户具有所需的权限。否则，让工作区所有者或保存点所有者更改保存点属性。

WM_ERROR_22删除保存点的权限不足

原因： DeleteSavepoint失败，因为只有具有WM_ADMIN系统特权的用户或工作空间或保存点的所有者才能删除保存点。

操作：在尝试删除保存点之前，请确保调用用户具有所需的权限。否则，让工作区所有者或保存点所有者删除保存点。

WM_ERROR_23工作区已存在，名称为： 'string'

原因： CreateWorkspace失败，因为系统中已存在具有相同名称的工作区。Workspace Manager要求工作区名称在整个数据库中是唯一的。

操作：选择其他工作区名称并重试。

WM_ERROR_24无法在隐式保存点上回滚工作空间

原因：在隐式保存点上的非叶工作空间上调用RollbackWorkspace操作。

操作：不要回滚隐式保存点。要删除隐式保存点，请合并或删除后代工作区。

WM_ERROR_25无法从“LIVE”工作区合并表

原因：使用指定为LIVE工作空间的输入工作空间调用MergeTable。LIVE工作空间是工作空间层次结构树中的根工作空间。

操作：不要使用工作空间参数LIVE调用MergeTable。

WM_ERROR_27表无法刷新到“LIVE”工作空间

原因：使用指定为LIVE工作空间的输入工作空间调用RefreshTable。LIVE工作空间是工作空间层次结构树中的根工作空间。

操作：不要使用工作空间参数LIVE调用RefreshTable。

WM_ERROR_28无法在隐式保存点上回滚表

原因：在隐式保存点上的非叶工作空间上调用RollbackTable操作。

操作：不要回滚隐式保存点。要删除隐式保存点，请合并或删除后代工作区。

WM_ERROR_29无法使用RollbackTable回滚此表

原因： RollbackTable失败，因为要回滚的表是参照完整性约束的一部分。

操作：改为使用RollbackWorkspace或RollbackToSP。

WM_ERROR_30无法使用MergeTable合并此表

原因： MergeTable失败，因为要合并的表是参照完整性约束的一部分。

操作：改为使用MergeWorkspace。

WM_ERROR_31必须首先禁用'string'拥有的所有已启用版本的表。

原因：尝试删除拥有一个或多个启用版本的表的数据库用户。

操作：确保在尝试删除数据库用户之前已显式禁用了用户拥有的所有启用版本的表。

WM_ERROR_32索引组织表无法启用版本。

原因： Workspace Manager不支持索引组织表。

操作：确保要启用版本的表不是索引组织的。

WM_ERROR_33尝试'字符串'锁定的行： 'string'在工作区'string'中

原因： DML操作失败，因为该行先前已被锁定。

操作：等待释放行上的锁定或让锁定所有者使用UnlockRows过程来解锁行。查阅表的_LOCK视图以查看此表中的哪些行被锁定。

WM_ERROR_34企图“字符串”通过“串”在工作区中锁定的行：“字符串”

原因： DML操作失败，因为该行先前已被锁定。

操作：等待释放行上的锁定或让锁定所有者使用UnlockRows过程来解锁行。查阅表的_LOCK视图以查看此表中的哪些行被锁定。

WM_ERROR_35尝试锁定工作空间中锁定的行： 'string'

原因：操作失败，因为无法在该行上获取锁定，因为它已被锁定。

操作：等待释放行上的锁定或让锁定所有者使用UnlockRows过程来解锁行。查阅表的_LOCK视图以查看此表中的哪些行被锁定。

WM_ERROR_36尝试锁定由'string'锁定的行

原因：操作失败，因为无法在该行上获取锁定，因为它已被锁定。

操作：等待释放行上的锁定或让锁定所有者使用UnlockRows过程来解锁行。查阅表的_LOCK视图以查看此表中的哪些行被锁定。

WM_ERROR_37尝试修改WM生成的过程

原因：尝试删除或重新创建数据库过程失败，因为该过程是由Workspace Manager创建的。

操作：不要删除或重新创建此过程。

WM_ERROR_38无法禁用非实时工作空间中修改的表格版本

原因： DisableVersioning失败，因为该表已在非LIVE工作区中进行了修改。

操作：删除或合并已修改此表的所有工作区。否则，请使用DisableVersioning的FORCE选项。

WM_ERROR_39无法删除外键关系中涉及的表

原因：尝试删除数据库表失败，因为它涉及与启用版本的表的外键关系。

操作：在尝试删除表之前，请查阅WM_RIC_INFO视图并对外键关系中涉及的表进行版本禁用。

WM_ERROR_40只有特权的授予者可以撤销它

原因：尝试撤消当前用户未授予的权限。

操作：请勿尝试撤消此权限。

WM_ERROR_41无法设置工作区锁定模式

原因： SetWorkspaceLockModeOn失败，因为工作空间包含一个或多个启用版本的表的修改。

操作：使用SetLockingOn设置会话的锁定模式。仅对尚未修改任何启用版本的表的工作空间使用SetWorkspaceLockModeOn。

WM_ERROR_42无法通过' string '拥有的版本启用表

原因： EnableVersioning失败，因为Workspace Manager无法对指定用户拥有的表进行版本启用。

操作：不要对该用户拥有的表调用EnableVersioning。

WM_ERROR_43持续刷新的工作空间必须是叶工作空间。

原因： CreateWorkspace失败，因为要创建的工作空间是不断刷新的工作空间的子项。持续刷新的工作空间带有他们必须是叶子工作空间的限制。

操作：不要从持续刷新的工作区创建工作区。

WM_ERROR_44合并数据的权限不足

原因：合并操作失败，因为用户在调用它的工作空间上没有ACCESS和MERGE权限;或者，在多个工作空间环境中，用户对多根工作空间图的根工作空间上的非根工作空间和ACCESS权限不具有ACCESS和MERGE权限。

操作：使用GetPrivs函数确保调用此操作的用户具有所需的权限。

WM_ERROR_45合并操作需要父工作空间的ACCESS权限

原因：调用的操作失败，因为它在调用它的工作空间的父工作空间上需要ACCESS权限。

操作：使用函数GetPrivs确保调用此操作的用户在父工作空间上具有所需的权限。

WM_ERROR_46提交/回滚在调用CommitResolve之前打开短事务

原因： CommitResolve失败，因为存在打开的数据库事务。

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

WM_ERROR_47提交/回滚在调用CompressWorkspace之前打开短事务

原因： CompressWorkspace失败，因为存在打开的数据库事务。

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

WM_ERROR_48提交/回滚在调用CompressWorkspaceTree之前打开短事务

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

WM_ERROR_49提交/回滚在调用DeleteSavepoint之前打开短事务

原因： DeleteSavepoint失败，因为存在打开的数据库事务。

操作：具有开放数据库事务的用户应发出标准数据

WM_ERROR_50提交/回滚在调用GotoWorkspace之前打开短事务

原因： GotoWorkspace失败，因为存在打开的数据库事务。

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

WM_ERROR_51提交/回滚在调用RollbackResolve之前打开短事务

原因： RollbackResolve失败，因为存在打开的数据库事务。

操作：具有开放数据库事务的用户应发出标准数据库提交或回滚。

WM_ERROR_52只有在调用BeginResolve后才能调用CommitResolve

原因： CommitResolve失败，因为先前未调用BeginResolve。

操作：要解决冲突，首先发出BeginResolve，然后发出ResolveConflicts，最后发出CommitResolve。

WM_ERROR_53 CompressWorkspace操作需要工作空间的ACCESS和MERGE权限

原因：调用的操作失败，因为它在调用它的工作空间上需要ACCESS和MERGE权限。

操作：使用GetPrivs函数确保调用此操作的用户具有工作空间所需的权限。

WM_ERROR_54 CompressWorkspace操作需要工作空间的ACCESS权限

原因：调用的操作失败，因为它在调用它的工作空间上需要ACCESS权限。

操作：使用GetPrivs函数确保调用此操作的用户具有工作空间所需的权限。

原因：操作失败，因为作为输入传入的表不存在或未启用版本。

WM_ERROR_64需要在最新版本上才能创建保存点

WM_ERROR_65设保人和受让人可能不是同一个用户

操作：选择其他工作区。

WM_ERROR_394无法修改虚拟列表达式，因为在列上定义了约束

原因： DDL操作尝试修改已在其上定义参照完整性约束的列的虚拟列表达式。

操作：删除约束或不修改虚拟列表达式。

WM_ERROR_395不支持延迟约束

原因：在包含可延迟约束的表上执行了EnableVersioning或DDL操作。

操作：重新定义约束，使其不再可延迟。

WM_ERROR_396不支持延迟约束

原因：执行CopyWorkspace时，源工作空间和目标工作空间参数都指定为同一工作空间。

操作：为CopyWorkspace操作的源和目标指定不同的工作空间。
