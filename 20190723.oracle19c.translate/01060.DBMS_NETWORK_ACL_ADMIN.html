110.1 DBMS_NETWORK_ACL_ADMIN概述

DBMS_NETWORK_ACL_ADMIN包提供管理网络访问控制列表（ACL）的接口。

ACL用于控制用户通过PL / SQL网络实用程序包（包括UTL_TCP ， UTL_HTTP ， UTL_SMTP和UTL_INADDR）从数据库访问外部网络服务和资源。

110.2 DBMS_NETWORK_ACL_ADMIN不推荐使用的子程序

Oracle建议您不要在新应用程序中使用已弃用的子程序。对不推荐使用的功能的支持仅用于向后兼容

发布Oracle Database 12 c时不推荐使用以下子程序：

  • ADD_PRIVILEGE过程

  • ASSIGN_ACL程序

  • ASSIGN_WALLET_ACL程序

  • CHECK_PRIVILEGE函数

  • CHECK_PRIVILEGE_ACLID函数

  • CREATE_ACL过程

  • DELETE_PRIVILEGE过程

  • DROP_ACL程序

  • UNASSIGN_ACL程序

  • UNASSIGN_WALLET_ACL程序

110.3 DBMS_NETWORK_ACL_ADMIN安全模型

DBMS_NETWORK_ACL_ADMIN程序包的EXECUTE特权DBMS_NETWORK_ACL_ADMIN授予DBA角色和EXECUTE_CATALOG_ROLE 。

110.7.1 ADD_PRIVILEGE过程

此过程添加了授予或拒绝用户的网络访问权限。如果访问控制条目（ACE）不存在，则创建它。

注意：

Oracle Database 12 c中不推荐使用此过程。虽然出于向后兼容性的原因，程序仍在程序包中可用，但Oracle建议使用APPEND_HOST_ACE过程和APPEND_WALLET_ACE过程 。

句法

DBMS_NETWORK_ACL_ADMIN.ADD_PRIVILEGE（acl IN VARCHAR2，主要IN VARCHAR2，is_grant IN BOOLEAN，权限IN VARCHAR2，位置IN PLS_INTEGER DEFAULT NULL，start_date IN TIMESTAMP WITH TIMESTAMP DEFAULT NULL，end_date IN TIMESTAMP WITH TIMESTAMP DEFAULT NULL）;

参数

表110-4 ADD_PRIVILEGE函数参数

参数 描述

ACL

ACL的名称。相对路径将相对于“/ sys / acls”

主要

授予或拒绝特权的委托人（数据库用户或角色）。区分大小写。

is_grant

授予或拒绝特权。

特权

授予或拒绝网络权限

位置

ACE的位置（从1开始）。如果给出非NULL值，则权限将添加到给定位置的新ACE中，并且不应该为具有相同is_grant （grant或deny）的主体提供另一个ACE。如果给出NULL值，则该特权将添加到与主体匹配的ACE和is_grant如果存在），或者如果匹配的ACE不存在则添加到ACL的末尾。

开始日期

访问控制条目（ACE）的开始日期。指定后，ACE仅在指定日期及之后有效。如果将权限添加到现有ACE，则将忽略start_date 。

结束日期

访问控制条目（ACE）的结束日期。指定后，ACE将在指定日期后过期。end_date必须大于或等于start_date 。如果将权限添加到现有ACE，则将忽略end_date 。

使用说明

要删除权限，请使用DELETE_PRIVILEGE过程 。

例子

BEGIN DBMS_NETWORK_ACL_ADMIN.ADD_PRIVILEGE（acl =>'us-example-com-permissions.xml'，principal =>'ST_USERS'，is_grant => TRUE，privilege =>'connect'）END;

110.7.2 APPEND_HOST_ACE程序

此过程将访问控制条目（ACE）附加到网络主机的访问控制列表（ACL）。ACL控制从数据库访问给定主机，ACE指定从指定主体授予或拒绝的权限。

句法

DBMS_NETWORK_ACL_ADMIN.APPEND_HOST_ACE（主机IN VARCHAR2，lower_port IN PLS_INTEGER DEFAULT NULL，upper_port IN PLS_INTEGER DEFAULT NULL，ace IN XS $ ACE_TYPE）;

参数

表110-5 APPEND_HOST_ACE函数参数

参数 描述

主办

主机，可以是主机的名称或IP地址。您可以使用通配符指定域或IP子网。主机或域名不区分大小写。

lower_port

可选TCP端口范围的下限

upper_port

可选TCP端口范围的上限。如果为NULL ，则假定为lower_port 。

高手

ACE

使用说明

  • 将跳过主机ACL中匹配ACE中的重复权限。

  • 要删除ACE，请使用REMOVE_HOST_ACE过程 。

  • 主机的ACL优先于其域的ACL。对于给定的主机，例如www.us.example.com ，以下域以递减优先级列出：

    • www.us.example.com

    • * .us.example.com

    • * .example.com的

    • * .COM

    • *

  • IP地址'ACL优先于其子网的ACL。对于给定的IP地址，例如192.168.0.100 ，以下优先级列出以下子网：

    • 192.168.0.100

    • 192.168.0。*

    • 192.168。*

    • 192。*

    • *

  • 具有“解析”权限的ACE只能附加到没有端口范围的主机ACL。

  • 当具有“连接”权限的ACE附加到具有和不具有端口范围的主机的ACL时，附加到具有端口范围的主机的ACE优先。

  • 指定主机的TCP端口范围时，它不能与主机的其他现有端口范围重叠。

  • 如果ACL与其他主机或钱包共享，则在修改ACL之前将创建ACL的副本。

110.7.6 ASSIGN_ACL程序

此过程将访问控制列表（ACL）分配给主机，域或IP子网，如果指定，则分配TCP端口范围。

注意：

Oracle Database 12 c中不推荐使用此过程。虽然出于向后兼容性的原因，程序仍在程序包中可用，但Oracle建议使用APPEND_HOST_ACE过程和APPEND_WALLET_ACE过程 。

句法

DBMS_NETWORK_ACL_ADMIN.ASSIGN_ACL（acl IN VARCHAR2，host IN VARCHAR2，lower_port IN PLS_INTEGER DEFAULT NULL，upper_port IN PLS_INTEGER DEFAULT NULL）;

参数

表110-9 ASSIGN_ACL功能参数

参数 描述

ACL

ACL的名称。相对路径将相对于“ /sys/acls ”。

主办

要为其分配ACL的主机。主机可以是主机的名称或IP地址。通配符可用于指定域或IP子网。主机或域名不区分大小写。

lower_port

如果不是NULL则TCP端口范围的下限

upper_port

TCP端口范围的上限。如果为NULL ，则假定为lower_port 。

使用说明

  • 只能为任何主机，域或IP子网分配一个ACL，如果指定，则为TCP端口范围。将新访问控制列表分配给网络目标时，Oracle数据库将取消分配分配给同一目标的先前访问控制列表。但是，Oracle数据库不会删除访问控制列表。您可以使用DROP_ACL过程删除访问控制列表。要删除访问控制列表分配，请使用UNASSIGN_ACL过程 。

  • 分配给域的ACL的优先级低于分配给其他ACL的子域，后者的优先级低于分配给各个主机的ACL。因此，对于给定的主机，例如“www.us.example.com”，以下域以递减优先级列出：

    - www.us.example.com

    - * .us.example.com

    - * .example.com

    - * .com

    - *

    同样，分配给子网的ACL的优先级低于分配较小子网的其他ACL，其优先级低于分配给各个IP地址的ACL。因此，对于给定的IP地址，例如“192.168.0.100”，以下优先级列出以下子网：

    - 192.168.0.100

    - 192.168.0。*

    - 192.168*

    - 192。*

    - *

  • 端口范围仅适用于ACL中的“连接”权限分配。 ACL中的“resolve”权限分配仅在将ACL分配给没有端口范围的主机时才有效。

    对于“连接”权限分配，分配给没有端口范围的主机的ACL优先级低于分配给具有端口范围的同一主机的其他ACL。

  • 指定TCP端口范围时， lower_port和upper_port不能为NULL ， upper_port必须大于或等于lower_port 。端口范围不得与已分配的同一主机的任何其他端口范围重叠。

  • 要删除分配，请使用UNASSIGN_ACL过程 。

例子

BEGIN DBMS_NETWORK_ACL_ADMIN.ASSIGN_ACL（acl =>'us-example-com-permissions.xml'，host =>'* .us.example.com'，lower_port => 80）;结束;

110.7.10 CREATE_ACL过程

此弃用过程创建具有初始权限设置的访问控制列表（ACL）。ACL必须至少具有一个权限设置。除非将ACL分配给网络目标，否则ACL不具有访问控制效果。

注意：

Oracle Database 12 c中不推荐使用此过程。虽然出于向后兼容性的原因，程序仍在程序包中可用，但Oracle建议使用APPEND_HOST_ACE过程和APPEND_WALLET_ACE过程 。

句法

DBMS_NETWORK_ACL_ADMIN.CREATE_ACL（acl IN VARCHAR2，描述IN VARCHAR2，主要IN VARCHAR2，is_grant IN BOOLEAN，权限IN VARCHAR2，start_date IN TIMESTAMP WITH TIMEZONE DEFAULT NULL，end_date IN TIMESTAMP WITH TIMEZONE DEFAULT NULL）;

参数

表110-13 CREATE_ACL过程参数

参数 描述

ACL

ACL的名称。相对路径将相对于“/ sys / acls”。

描述

ACL中的描述属性

主要

授予或拒绝特权的委托人（数据库用户或角色）。区分大小写。

is_grant

是否授予特权（拒绝）

特权

要授予或拒绝的网络权限 - 'connect | resolve' （区分大小写）。如果数据库用户使用UTL_TCP ， UTL_HTTP ， UTL_SMTP和UTL_MAIL实用程序包进行连接，则需要将connect权限connect到外部网络主机。要使用UTL_INADDR程序包解析给定主机IP地址的主机名或给定主机名的IP地址，请为数据库用户授予resolve权限。

开始日期

访问控制条目（ACE）的开始日期。指定后，ACE仅在指定日期及之后有效。

结束日期

访问控制条目（ACE）的结束日期。指定后，ACE将在指定日期后过期。end_date必须大于或等于start_date 。

使用说明

要删除访问控制列表，请使用DROP_ACL过程 。

例子

BEGIN DBMS_NETWORK_ACL_ADMIN.CREATE_ACL（acl =>'us-example-com-permissions.xml'，description =>'* .us.example.com'的网络权限'，principal =>'SCOTT'，is_grant => TRUE，privilege = >'connect'）;结束;
