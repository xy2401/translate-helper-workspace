B.1.1捕获进程不支持的数据类型

捕获进程不支持某些数据类型。

捕获过程不会将DML更改的结果捕获到以下数据类型的列：

  • ROWID

  • 嵌套表

  • 以下Oracle提供的类型： ANYTYPE ， ANYDATASET ，URI类型， SDO_TOPO_GEOMETRY ， SDO_GEORASTER和Expression

这些数据类型限制适用于普通（堆组织）表和索引组织表。

注意：

XStream不支持具有不同宽度多字节字符集的数据库中的LONG列。

仅当源数据库兼容级别设置为11.2.0.0或更高版本时，捕获进程才能捕获对SecureFiles LOB列的更改。此外，捕获进程不支持捕获SecureFiles LOB列上基于片段的操作所产生的更改，也不支持捕获SecureFiles归档管理器操作导致的更改。

当捕获进程尝试为DML更改创建行LCR更改为不受支持的数据类型的列时，捕获进程可以忽略对表的更改或引发错误。捕获过程的行为取决于ignore_unsupported_table捕获过程参数的设置。

当捕获进程忽略对表的更改时，它不会捕获更改，并且会在警报日志中记录表名。当捕获过程引发错误时，它会将导致错误的LCR写入其跟踪文件，引发ORA-26744错误，并被禁用。在任何一种情况下，都要修改捕获过程使用的规则，以避免在警报日志中记录消息或捕获进程错误。修改捕获进程的规则后，重新启动捕获进程。

注意：

  • 您可以将规则添加到捕获进程的否定规则集，该规则集指示捕获进程放弃对具有不受支持的数据类型列的表的更改。

  • 捕获进程不支持包含对象类型属性的主键。

  • 如果INSERT操作包含以下任一类型的列，则捕获进程会尝试使用APPEND提示捕获INSERT操作时引发错误：XMLType存储为对象关系或XMLType存储为二进制XML

B.1.2.3捕获进程的DDL更改不受支持

捕获进程不支持某些数据定义语言（DDL）更改。

捕获过程捕获满足其规则集的DDL更改， 但以下类型的DDL更改除外 ：

  • ALTER DATABASE

  • CREATE CONTROLFILE

  • CREATE DATABASE

  • CREATE PFILE

  • CREATE SPFILE

除非DDL语句是CREATE TABLE AS SELECT语句，否则捕获过程可以捕获DDL语句，但不捕获DDL语句的结果。例如，当捕获进程捕获ANALYZE语句时，它不捕获ANALYZE语句生成的统计信息。但是，当捕获进程捕获CREATE TABLE AS SELECT语句时，它会捕获语句本身和所有选定的行（作为INSERT行LCR）。

出站服务器无法应用捕获进程捕获的某些类型的DDL更改。如果出站服务器收到指定无法处理的操作的DDL LCR，则出站服务器将忽略DDL LCR并在其跟踪文件中记录有关它的信息。

B.1.2.4捕获过程忽略的变化

捕获进程忽略某些类型的更改。

捕获过程忽略以下类型的更改：

  • 会话控制语句ALTER SESSION和SET ROLE 。

  • 系统控制语句ALTER SYSTEM 。

  • CALL ， EXPLAIN PLAN和LOCK TABLE语句。

  • 关于视图的GRANT语句。

  • 使用DBMS_REDEFINITION包通过在线重新定义对表或模式所做的更改。捕获进程捕获更改的表支持联机表重定义，但联机重定义之前表的逻辑结构必须与联机重定义后的逻辑结构相同。

  • 更改序列值。例如，如果用户引用NEXTVAL或设置序列，则捕获过程不会捕获由这些操作导致的更改。此外，如果您在多个数据库共享序列，则用于这些数据库中各个行的序列值可能会有所不同。

  • 调用PL / SQL过程，这意味着不会捕获对PL / SQL过程的调用。但是，如果对PL / SQL过程的调用导致更改数据库对象，则如果更改满足捕获过程规则集，则捕获过程可以捕获这些更改。

注意：

  • 如果Oracle提供的与XML相关的包对数据库对象进行了更改，则捕获进程不会捕获这些更改。

  • 如果Oracle提供的与Oracle Text相关的程序包对数据库对象进行了更改，则捕获进程不会捕获这些更改。

也可以看看：

  • 有关与XML相关的包的信息，请参阅Oracle Database PL / SQL包和类型参考

  • 有关Oracle Text相关软件包的信息，请参阅Oracle Text Reference

B.1.2.5 SQL操作的NOLOGGING和UNRECOVERABLE关键字

如果对SQL操作使用NOLOGGING或UNRECOVERABLE关键字，则捕获进程无法捕获SQL操作导致的更改。

因此，请勿使用这些关键字来捕获SQL操作所带来的更改。

如果要为其指定日志记录属性的对象驻留在FORCE LOGGING模式的数据库或表空间中，则Oracle数据库将忽略任何NOLOGGING或UNRECOVERABLE设置，直到数据库或表空间退出FORCE LOGGING模式。您可以通过查询V$DATABASE动态性能视图中的FORCE_LOGGING列来确定数据库的当前日志记录模式。您可以通过查询ALL_TABLESPACES静态数据字典视图中的FORCE_LOGGING列来确定表空间的当前日志记录模式。

注意：

不推荐使用UNRECOVERABLE关键字，并且已将其替换为logging_clause的NOLOGGING关键字。虽然支持UNRECOVERABLE以实现向后兼容，但Oracle强烈建议您在适当时使用NOLOGGING关键字。

有关NOLOGGING和UNRECOVERABLE关键字， FORCE LOGGING模式和logging_clause详细信息，请参阅Oracle数据库SQL语言参考

B.1.2.6直接路径荷载的不可解决的条款

如果在SQL * Loader控制文件中使用UNRECOVERABLE子句进行直接路径加载，则捕获过程无法捕获直接路径加载所导致的更改。

因此，如果捕获进程应捕获直接路径加载导致的更改，则不要使用UNRECOVERABLE子句。

如果将对象加载到处于FORCE LOGGING模式的数据库或表空间，则Oracle数据库会在直接路径加载期间忽略任何UNRECOVERABLE子句，并记录加载的更改。您可以通过查询V$DATABASE动态性能视图中的FORCE_LOGGING列来确定数据库的当前日志记录模式。您可以通过查询DBA_TABLESPACES静态数据字典视图中的FORCE_LOGGING列来确定表空间的当前日志记录模式。

B.1.4使用XStream Out进行下游捕获的操作要求

XStream Out有下游捕获的操作要求。

以下是使用下游捕获的操作要求：

  • 源数据库必须至少运行Oracle Database 10g第2版（10.2）。

  • 下游数据库必须运行Oracle Database 11g第2版（11.2.0.3）或更高版本，并且源数据库必须运行Oracle数据库10g第2版（10.2）或更高版本。

  • 源和下游捕获站点上的操作系统必须相同，但操作系统版本不必相同。此外，下游站点可以使用与源站点不同的目录结构。

  • 源和下游捕获站点上的硬件体系结构必须相同。例如，在64位Sun系统上具有源数据库的下游捕获配置必须具有在64位Sun系统上配置的下游数据库。源和下游站点之间的其他硬件元素（例如CPU数量，内存大小和存储配置）可能不同。

B.3.1出站服务器不支持的数据类型

出站服务器不支持某些数据类型。

出站服务器不处理包含以下数据类型列中DML更改结果的行LCR：

  • ROWID

  • 嵌套表

  • 以下Oracle提供的类型： ANYTYPE ， ANYDATASET ，URI类型， SDO_TOPO_GEOMETRY ， SDO_GEORASTER和Expression

注意：

如果出站服务器尝试处理包含有关不受支持的数据类型列的信息的行LCR，则会引发错误。此外，出站服务器无法处理对以下类型的表的DML更改：

  • 临时表

  • 包含不受支持的数据类型的对象表

如果出站服务器尝试处理此类更改，则会引发错误。当出站服务器为LCR引发错误时，它会将包含LCR的事务移动到错误队列中。

B.3.2出站服务器忽略的DDL更改类型

出站服务器不支持某些类型的DDL更改。

出站服务器不支持以下类型的DDL更改：

  • ALTER MATERIALIZED VIEW

  • ALTER MATERIALIZED VIEW LOG

  • CREATE DATABASE LINK

  • CREATE SCHEMA AUTHORIZATION

  • CREATE MATERIALIZED VIEW

  • CREATE MATERIALIZED VIEW LOG

  • DROP DATABASE LINK

  • DROP MATERIALIZED VIEW

  • DROP MATERIALIZED VIEW LOG

  • FLASHBACK DATABASE

  • 改名

如果出站服务器收到指定无法处理的操作的DDL LCR，则出站服务器将忽略DDL LCR并在出站服务器跟踪文件中记录以下消息，然后记录忽略的DDL文本：

Apply进程忽略以下DDL：

如果应根据出站服务器规则集应用包含更改的DDL LCR，则出站服务器将应用所有其他类型的DDL更改。

注意：

  • 出站服务器处理ALTER object_type object_name RENAME更改，例如ALTER TABLE jobs RENAME 。因此，如果要进行重命名要处理的对象的DDL更改，请使用ALTER object_type object_name RENAME语句而不是RENAME语句。

  • 名称“物化视图”与名称“快照”同义。出站服务器将忽略物化视图上语句的快照等效项。

B.3.3应用不适用于出站服务器的进程特性

某些功能不能与出站服务器一起使用。

以下适用过程功能不能与出站服务器一起使用：

  • 申请处理程序

    您不能为出站服务器指定应用处理程序。如有必要，客户端应用程序可以执行LCR的自定义处理。但是，如果将应用进程配置在与出站服务器相同的数据库中，则可以为这些应用进程指定应用处理程序。此外，您还可以为数据库配置常规应用处理程序。出站服务器忽略常规应用处理程序。

  • 以下适用参数：

    • allow_duplicate_rows

    • commit_serialization

    • compare_key_only

    • disable_on_error

    • 排比

    • preserve_encryption

    • rtrim_on_implicit_conversion

    出站服务器忽略这些应用参数的设置。

    对于出站服务器， commit_serialization参数始终设置为FULL ，并且对于出站服务器， parallelism参数始终设置为1 。

    也可以看看：

    Oracle数据库PL / SQL包和类型参考

  • 应用标签

    出站服务器无法为其处理的更改设置应用标记。

  • 应用数据库链接

    出站服务器无法使用数据库链接。

  • 冲突检测和解决方案

    出站服务器不检测冲突，并且无法为出站服务器设置冲突解决方案。

  • 依赖性调度

    出站服务器不评估依赖关系，因为其并行性必须为1。

  • 替换键列设置

    出站服务器忽略替换键列设置。

  • 由DBMS_APPLY_ADM包中的SET_ENQUEUE_DESTINATION过程指定的入SET_ENQUEUE_DESTINATION指令

    出站服务器无法使用SET_ENQUEUE_DESTINATION过程自动将更改排入Oracle数据库队列。

    也可以看看：

    Oracle数据库PL / SQL包和类型参考

  • 执行DBMS_APPLY_ADM包中的SET_EXECUTE过程指定的指令

    出站服务器忽略执行指令。

    也可以看看：

    Oracle数据库PL / SQL包和类型参考

  • 错误创建和执行

    出站服务器在遇到错误时不会创建错误事务。它在ALL_APPLY视图中记录有关错误的信息，但不会将事务排入错误队列。

B.4.1子集规则的限制

限制适用于子集规则。

以下限制适用于子集规则：

  • 具有子集规则中引用的表名的表必须与子集规则存在于同一数据库中，并且此表必须位于为子集规则中的表引用的相同模式中。

  • 如果子集规则位于捕获进程的正规则集中，则表必须包含子集条件中指定的列，并且每个列的数据类型必须与源数据库中相应列的数据类型匹配。

  • 如果子集规则位于传播的正规则集中，则表必须包含子集条件中指定的列，并且每列的数据类型必须与计算为TRUE行LCR中相应列的数据类型匹配对于子集规则。

  • 不支持为具有以下数据类型的一列或多列的表创建子集规则：LOB， LONG ， LONG RAW ，嵌套表和Oracle提供的类型（包括Any类型，XML类型，空间类型和媒体类型） 。

B.5.1声明性基于规则的转换的不支持的数据类型

除了添加列转换之外，对列进行操作的基于声明规则的转换支持捕获进程支持的相同数据类型。

B.6扩展数据类型的XStream Out限制

某些限制适用于XStream Out中的扩展数据类型。

所述的最大大小VARCHAR2 ， NVARCHAR2 ，和RAW数据类型已在Oracle数据库12c中，当已增加COMPATIBLE初始化参数设置为12.0.0和MAX_STRING_SIZE初始化参数设置为EXTENDED 。 XStream Out支持这些扩展数据类型。

但是，以下限制适用于扩展数据类型：

  • 有关扩展数据类型列的信息可能不包含在数据操作语言（DML）操作的原始LCR中。相反，XStream Out可能会将扩展数据类型列视为与处理LOB列的方式类似。具体而言，其他LCR可能包含扩展数据类型列的信息。

  • 对于扩展数据类型列，XStream规则无法访问LCR中的数据。

  • 无法在子集规则子句中指定扩展数据类型列。

  • 扩展数据类型列不能用于以下类型的基于声明规则的转换：

    • 添加列

    • 保留专栏
