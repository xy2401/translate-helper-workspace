1.76 DB_DOMAIN

在分布式数据库系统中， DB_DOMAIN指定网络结构中数据库的逻辑位置。

属性 描述

参数类型

串

句法

DB_DOMAIN = domain_name

默认值

没有默认值。

可修改

没有

可在PDB中修改

是

价值范围

任何合法的名称组件字符串，以句点分隔，最多128个字符（包括句点）。

基本

是

Oracle RAC

您必须为每个实例设置此参数，并且多个实例必须具有相同的值。

如果此数据库是或将成为分布式系统的一部分，则应设置此参数。该值由全局数据库名称的扩展组件组成，由有效标识符（任何字母数字ASCII字符）组成，以句点分隔。

注意：

Oracle建议您将DB_DOMAIN指定为域中所有数据库的唯一字符串。

此参数允许一个部门创建数据库，而不必担心它可能与另一个部门创建的数据库同名。如果一个销售部门的DB_DOMAIN是JAPAN.EXAMPLE.COM ，那么他们的SALES数据库（ SALES.JAPAN.EXAMPLE.COM ）与DB_NAME = SALES但DB_DOMAIN = US.EXAMPLE.COM另一个数据库唯一区分。

如果省略数据库链接名称中的域，Oracle会通过使用本地数据库的域限定数据库来扩展名称，因为它当前存在于数据字典中，然后将链接名称存储在数据字典中。DB_DOMAIN应以字母字符开头并排除这些字符：
“！@％^＆*（）+ = \\ |`〜[{]};：'\”，<> /？“
