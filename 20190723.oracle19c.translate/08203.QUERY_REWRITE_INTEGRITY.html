1.274 QUERY_REWRITE_INTEGRITY

QUERY_REWRITE_INTEGRITY确定Oracle必须强制执行查询重写的程度。 在最安全的级别，Oracle不使用依赖于非强制关系的查询重写转换。
属性 描述

参数类型

串

句法

QUERY_REWRITE_INTEGRITY = {强制|信任| stale_tolerated}

默认值

强制执行

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

Oracle RAC

多个实例可以具有不同的值。

QUERY_REWRITE_INTEGRITY与QUERY_REWRITE_INTEGRITY化视图以及NOVALIDATE状态中的外键约束相关。

值

  • 强制执行

    Oracle强制执行并保证一致性和完整性。

  • 信任

    Oracle允许使用已声明但未由Oracle强制执行的关系进行重写。

  • STALE_TOLERATED

    Oracle允许使用非强制关系进行重写。即使已知物化视图与基础详细数据不一致，也可以进行重写。必须设置QUERY_REWRITE_INTEGRITY初始化参数stale_tolerated在内存中的列存储（IM列存储）查询外部表之前。

如果外键约束处于NOVALIDATE状态，则在QUERY_REWRITE_INTEGRITY=enforced时QUERY_REWRITE_INTEGRITY=enforced连接消除。这意味着具有RELY NOVALIDATE状态的外键约束的连接查询可能需要更长的时间来解析和执行，因为优化器不信任RELY 。
