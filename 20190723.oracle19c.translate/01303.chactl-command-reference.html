D chactl命令参考

Oracle Cluster Health Advisor命令使Oracle Grid Infrastructure用户能够管理目标上的基本监视功能。

  • chactl监视器
    使用chactl monitor命令可以使用当前集模型开始监视特定Oracle Real Application Clusters（Oracle RAC）数据库的所有实例。
  • chactl unmonitor
    使用chactl unmonitor命令停止监视特定数据库的所有实例。
  • chactl status
    使用chactl status命令检查正在运行的目标的监控状态。
  • chactl配置
    使用chactl config命令列出所有受监视的目标，以及每个目标的当前模型。
  • chactl校准
    使用chactl calibrate命令创建具有更高灵敏度和准确度的新模型。
  • chactl查询诊断
    使用chactl query diagnosis命令可以返回问题和诊断，并建议与特定群集节点或Oracle Real Application Clusters（Oracle RAC）数据库的问题相关的纠正措施。
  • chactl查询模型
    使用chactl query model命令列出所有Oracle Cluster Health Advisor模型或查看有关特定Oracle Cluster Health Advisor模型的详细信息。
  • chactl查询存储库
    使用chactl query repository命令可以查看最大保留时间，目标数以及Oracle Cluster Health Advisor存储库的大小。
  • chactl查询校准
    使用chactl query calibration命令查看有关特定目标的校准数据的详细信息。
  • chactl删除模型
    使用chactl remove model命令从Oracle Cluster Health Advisor存储库中删除Oracle Cluster Health Advisor模型以及模型的校准数据和元数据。
  • chactl重命名模型
    使用chactl rename model命令在Oracle Cluster Health Advisor存储库中重命名Oracle Cluster Health Advisor模型。
  • chactl出口模型
    使用chactl export model命令导出Oracle Cluster Health Advisor模型。
  • chactl导入模型
    使用chactl import model命令导入Oracle Cluster Health Advisor模型。
  • chactl设置maxretention
    使用chactl set maxretention命令设置诊断数据的最长保留时间。
  • chactl resize repository
    使用chactl resize repository命令根据当前保留时间和目标数量调整Oracle Cluster Health Advisor存储库的表空间大小。
