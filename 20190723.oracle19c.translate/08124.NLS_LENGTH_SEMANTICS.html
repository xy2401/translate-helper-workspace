1.207 NLS_LENGTH_SEMANTICS

NLS_LENGTH_SEMANTICS用于指定长度语义。

属性 描述

参数类型

串

句法

NLS_LENGTH_SEMANTICS = string

示例： NLS_LENGTH_SEMANTICS = 'CHAR'

默认值

字节

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

价值范围

BYTE | CHAR

基本

没有

NLS_LENGTH_SEMANTICS的会话级值指定用于VARCHAR2和CHAR表列的默认长度语义，用户定义的对象属性以及在会话中创建的数据库对象中的PL / SQL变量。可以通过列，属性和变量定义中的显式长度语义限定符BYTE和CHAR来覆盖此缺省值。

如果数据库客户端未通过NLS_LENGTH_SEMANTICS客户端环境变量（不适用于JDBC瘦客户端）或ALTER SESSION SET NLS_LENGTH_SEMANTICS语句显式设置NLS_LENGTH_SEMANTICS则NLS_LENGTH_SEMANTICS的实例级值提供会话级别值的缺省值。

NCHAR ， NVARCHAR2 ， CLOB和NCLOB列始终基于字符。

以SYS身份登录的会话不使用NLS_LENGTH_SEMANTICS参数。它们对所有创建的对象使用BYTE长度语义，除非被对象定义中的显式BYTE和CHAR限定符覆盖（SQL DDL语句）。

注意：

Oracle强烈建议您不要在实例或服务器参数文件中将NLS_LENGTH_SEMANTICS参数设置为CHAR 。这可能导致许多现有安装脚本意外创建具有字符长度语义的列，从而导致运行时错误，包括缓冲区溢出。
