关于注册Oracle Label Security

安装Oracle数据库时，默认情况下不启用Oracle Label Security。

您必须在数据库中注册Oracle Label Security。之后，您必须启用默认的Oracle Label Security用户帐户LBACSYS 。注册Oracle Label Security后，您可以在必要时禁用并重新启用它。

如果您使用的是多租户环境，则只在您计划创建Oracle Label Security策略的可插拔数据库（PDB）中注册Oracle Label Security。由于Oracle Label Security不是为保护数据字典对象而设计的，因此无法在根目录中创建策略。

检查Oracle Label Security是否已注册并已启用

您可以查询DBA_OLS_STATUS和CDB_OLS_STATUS数据字典视图，以查找是否已注册并启用了Oracle Label Security。

  1. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。
    sqlplus sys as sysdba输入密码： password
  2. 如果您使用的是多租户环境，请连接到相应的PDB。

    例如，要连接到PDB hrpdb ：

    CONNECT SYS @hrpdb AS SYSDBA输入密码： password

    要查找可用的PDB，请查询DBA_PDBS数据字典视图。要检查当前PDB，请运行show con_name命令。

  3. 运行相应的查询以查找是否已注册并启用Oracle Label Security。
    • 如果要查找非多租户数据库的Oracle Label Security状态，或者仅针对root用户或单个PDB的多租户环境，请查询DBA_OLS_STATUS 。例如：
      SELECT * FROM DBA_OLS_STATUS;姓名状态说明-------------------- ------- -------------------- ----------------- OLS_CONFIGURE_STATUS TRUE确定是否配置了OLS OLS_DIRECTORY_STATUS FALSE确定是否使用OLS启用OID OLS_ENABLE_STATUS TRUE确定OLS是否已启用
    • 如果要在多租户环境中查找所有PDB的Oracle Label Security状态，作为具有管理权限的普通用户，请查询CDB_OLS_STATUS 。例如：
      SELECT * FROM CDB_OLS_STATUS;名称状态描述CON_ID -------------------- ------- ------------------- ----------------- ------ OLS_DIRECTORY_STATUS FALSE确定是否使用OLS 4启用OID OLS_ENABLE_STATUS TRUE确定OLS是否已启用4 OLS_CONFIGURE_STATUS TRUE确定是否配置了OLS 4 OLS_DIRECTORY_STATUS FALSE确定是否使用OLS启用OID 5 OLS_ENABLE_STATUS TRUE确定OLS是否已启用5 OLS_CONFIGURE_STATUS TRUE确定是否配置了OLS 5

从SQL * Plus注册和启用Oracle Label Security

您可以从SQL * Plus注册和启用Oracle Label Security。

  1. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。

    例如：

    sqlplus sys as sysdba输入密码： password
  2. 如果您使用的是多租户环境，请连接到相应的PDB。

    例如，要连接到PDB hrpdb ：

    CONNECT SYS @hrpdb AS SYSDBA输入密码： password

    要查找可用的PDB，请查询DBA_PDBS数据字典视图。要检查当前PDB，请运行show con_name命令。

  3. 注册并启用Oracle Label Security，如下所示。
    EXEC LBACSYS.CONFIGURE_OLS;  - 此过程注册Oracle Label Security。EXEC LBACSYS.OLS_ENFORCEMENT.ENABLE_OLS;  - 此程序启用它。
  4. 用户SYS以SYSOPER权限连接。
    将SYS作为SYSOPER连接- 或者，连接SYS @hrpdb AS SYSOPER输入密码： password
  5. 重启数据库。

    例如：

    关机立即启动

使用DBCA注册和启用Oracle Label Security

您可以使用Database Configuration Assistant注册和启用Oracle Label Security。

  1. 启动数据库配置助手（DBCA）。

    • UNIX ：运行以下命令：

      $ ORACLE_HOME / bin / dbca

    • Windows ：从“ 开始”菜单中，单击“ 所有程序” 。然后单击Oracle - ORACLE_HOME ，然后单击配置和迁移工具 ，再单击Database Configuration Assistant 。

    出现欢迎屏幕。

  2. 单击下一步 。

    出现“操作”屏幕。

  3. 选择配置数据库选项 。单击下一步 。

    出现数据库屏幕。

  4. 从列表中，选择您需要配置和启用OLS的数据库。单击下一步 。

    出现“数据库内容”屏幕。

  5. 选择Oracle Label Security 。单击下一步 。

    出现“连接模式”屏幕。

  6. 选择专用服务器模式或共享服务器模式 。单击完成 。

    将显示一个对话框，通知您该操作将需要重新启动数据库。

  7. 单击确定 。

    将显示确认对话框。

  8. 单击确定 。

    将显示DBCA进度屏幕。

  9. 操作完成后，系统将提示您执行其他操作。单击“ 否”退出DBCA。

管理LBACSYS用户和LBAC_DBA角色的安全指南

作为一种良好的做法，对于日常使用，请将LBAC_DBA数据库角色授予将管理Oracle Label Security的受信任用户。

如果您计划使用Enterprise Manager Cloud Control来管理Oracle Label Security，请确保您已授予LBAC_DBA角色的任何用户也具有SELECT ANY DICTIONARY权限。

Oracle强烈建议您为已被授予LBAC_DBA角色的用户维护两个帐户。一个帐户（主要用户帐户）将在日常使用，另一个帐户将用作备份帐户，以防主帐户的密码丢失并且必须重置。

从Enterprise Manager Cloud Control登录Oracle Label Security

在Enterprise Manager Cloud Control中，您可以使用Oracle Label Security页面来创建和管理Oracle Label Security策略。

  1. 确保已配置计划与Oracle Label Security一起使用的Cloud Control目标数据库。

    有关配置目标数据库的详细信息，请参阅Oracle Enterprise Manager联机帮助。

  2. 将浏览器指向Cloud Control登录页面。

    例如：

    https://myserver.example.com:7799/em
  3. 以SYSMAN用户SYSMAN登录Cloud Control。
  4. 在Cloud Control主页中，从“ 目标”菜单中选择“ 数据库” 。
  5. 在“数据库”页面中，选择要连接的数据库的链接。

    将显示“数据库”主页。

  6. 从Security菜单中，选择Label Security 。

    出现“数据库登录”页面。

  7. 输入以下信息：

    • 用户名：输入已被授予LBAC_DBA数据库角色的用户的用户名，或输入LBACSYS 。

    • 密码：输入密码。

    • 角色：从列表中选择NORMAL 。

    • 另存为：如果您希望下次显示此页面时自动填写这些凭据，请选中此复选框。凭据以安全的方式存储在企业管理器中。访问这些凭据取决于当前登录的用户。

从SQL * Plus登录Oracle Label Security

如果您已被授予LBAC_DBA数据库角色，则可以从SQL * Plus登录Oracle Label Security。

  • 要从SQL * Plus使用Oracle Label Security，请以用户LBACSYS或已被授予LBAC_DBA数据库角色的用户身份进行连接。要查找用户是否已被授予此角色，请查询DBA_ROLE_PRIVS数据字典视图的GRANTEE和GRANTED_ROLE列。

例如：

sqlplus psmith_ols - 或者，sqlplus psmith_ols @hrpdb用于名为hrpdb的PDB输入密码： password

要查找可用的PDB，请查询DBA_PDBS数据字典视图。要检查当前PDB，请运行show con_name命令。
