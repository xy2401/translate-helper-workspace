5.14 RC_BACKUP_FILES

此视图列出了恢复目录中反映的RMAN存储库已知的备份。

此视图对应于V$BACKUP_FILES控制文件视图。

注意：

  • 使用RMAN中的LIST BACKUP和LIST COPY命令访问此信息通常更方便。

  • 必须使用DBMS_RCVMAN.SetDatabase才能使用此视图从恢复目录模式中选择数据库，即使在恢复目录中只注册了一个数据库也是如此。“Oracle数据库备份和恢复用户指南”介绍了如何执行此任务。

柱 数据类型 描述

PKEY

数

备份的主键。

BACKUP_TYPE

VARCHAR2（32）

备份类型： BACKUP SET ， COPY或PROXY COPY 。

文件类型

VARCHAR2（32）

备份文件的类型： DATAFILE ， CONTROLFILE ， SPFILE ， REDO LOG ， ARCHIVED LOG ， COPY （用于映像副本备份）或PIECE （用于备份）。

保持

VARCHAR2（3）

此备份是否设置了覆盖备份保留策略的KEEP属性。值为YES或NO 。

KEEP_UNTIL

日期

此备份被视为过时的日期。

KEEP_OPTIONS

VARCHAR2（13）

影响此备份保留的属性。可能的值是NOLOGS ， BACKUP_LOGS ， LOGS和NULL 。 NOLOGS表示在装入数据库时进行的一致备份。BACKUP_LOGS表示备份是在打开模式下进行的，因此必须应用归档日志备份以使其保持一致。LOGS表示使用LOGS关键字进行的长期备份，现在已弃用。NULL表示此备份没有KEEP选项，并且基于保留策略而变得过时。

状态

VARCHAR2（16）

备份的状态。可能的值有： AVAILABLE ， UNAVAILABLE ，已EXPIRED 。

FNAME

VARCHAR2（1024）

此备份集中包含的文件的此文件名，副本或文件名。例如，对于BACKUP_TYPE为BACKUP SET且FILE_TYPE为DATAFILE ， FNAME是备份中数据文件的名称。另一方面，如果BACKUP_TYPE为BACKUP SET且FILE_TYPE为PIECE ，则FNAME显示备份片段的名称。

标签

VARCHAR2（32）

此备份片或图像副本的标记。如果FILE_TYPE是PIECE或COPY则此列只能具有值。

媒体

VARCHAR2（80）

存储备份的介质的介质ID。如果BACKUP_TYPE为BACKUP SET且FILE_TYPE为PIECE则此列只能具有值。

RECID

数

与该行对应的控制文件记录的ID。

邮票

数

与此行对应的控制文件记录的时间戳。

设备类型

VARCHAR2（255）

存储此备份的设备类型。仅当FILE_TYPE为PIECE或COPY才会填充此列。

BLOCK_SIZE

数

备份或副本的块大小（以字节为单位）。

完成时间

数

此备份完成的时间。仅当FILE_TYPE为PIECE或COPY才会填充此列。

COMPRESSED

VARCHAR2（3）

是否压缩此行表示的备份片。仅当文件类型为PIECE时才填充此列（因为根据定义，无法压缩映像副本）。

已过时

VARCHAR2（3）

此备份或副本是否已过时。可能的值： YES ， NO 。仅当FILE_TYPE为PIECE或COPY才会填充此列。

BYTES

数

此行描述的文件大小。如果BACKUP_TYPE是BACKUP SET ，则表示备份集的总大小。如果FILE_TYPE是PIECE或COPY ，则表示单个文件的大小。如果FILE_TYPE是DATAFILE ， ARCHIVED LOG ， SPFILE或CONTROL FILE ，则该值表示已将多少数据合并到备份集中（但如果在创建备份集时使用了压缩，则相应的备份集可能会更小）。

BS_KEY

数

备份集密钥。仅当BACKUP_TYPE为BACKUP SET时才填充此列。使用此列与RC_BACKUP_SET或RC_BACKUP_PIECE 。

BS_COUNT

数

备份集计数。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_STAMP

数

备份集时间戳。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_TYPE

VARCHAR2（32）

备份集内容的类型（数据文件或归档重做日志文件）。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_INCR_TYPE

VARCHAR（32）

备份集增量类型（已满或未满）。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_PIECES

数

备份集中的件数。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_COPIES

数

此备份集的副本数。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_COMPLETION_TIME

日期

备份集的完成时间。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_STATUS

VARCHAR2（16）

备份集的状态。可能的值为AVAILABLE ， UNAVAILABLE ， EXPIRED或OTHER 。 （OTHER表示并非备份集的所有部分都具有相同的状态，如果某些部分AVAILABLE且其他部分UNAVAILABLE ，则可能会发生这种情况。）仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_BYTES

数

备份集中所有备份片段大小的总和。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_COMPRESSED

VARCHAR2（3）

是否压缩此备份集的备份片段。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_TAG

VARCHAR2（1024）

此备份集的备份片段的标记或标记。如果片段具有不同的标签，则BS_TAGS包含备份集中片段的所有标签的逗号分隔列表。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BS_DEVICE_TYPE

VARCHAR2（255）

存储此备份集的设备类型。如果此备份集的多个副本存在并存储在不同的设备上，则此字段包含所有设备类型的逗号分隔列表。例如，对于磁盘上也备份在磁带上的备份集， BS_DEVICE_TYPE可能包含DISK, SBT_TAPE 。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BP_PIECE＃

数

组成此备份集的备份片数。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

BP_COPY＃

数

此备份集的副本数。仅当BACKUP_TYPE为BACKUP SET时才填充此列。

DF_FILE＃

数

此行描述的数据文件的文件号。仅当FILE_TYPE为DATAFILE时才填充此列。

DF_TABLESPACE

VARCHAR2（30）

此行描述的数据文件的表空间名称。仅当FILE_TYPE为DATAFILE时才填充此列。

DF_RESETLOGS_CHANGE＃

数

RESETLOGS更改此行描述的数据文件。仅当FILE_TYPE为DATAFILE时才填充此列。

DF_CREATION_CHANGE＃

数

此行描述的数据文件的创建更改编号。仅当FILE_TYPE为CONTROLFILE,DATAFILE或SPFILE CONTROLFILE,DATAFILE才会填充此列。

DF_CHECKPOINT_CHANGE＃

数

检查点更改此行描述的数据文件的编号。仅当FILE_TYPE为CONTROLFILE,DATAFILE或SPFILE CONTROLFILE,DATAFILE才会填充此列。

DF_CKP_MOD_TIME

日期

此行描述的数据文件的检查点时间。仅在FILE_TYPE为CONTROLFILE,DATAFILE或SPFILE 。

RL_THREAD＃

数

此行描述的归档重做日志的重做日志线程号。仅在FILE_TYPE为ARCHIVED LOG 。

RL_SEQUENCE＃

数

此行描述的归档重做日志的重做日志序列号。仅在FILE_TYPE为ARCHIVED LOG 。

RL_RESETLOGS_CHANGE＃

数

RESETLOGS更改此行描述的归档重做日志的编号。仅在FILE_TYPE为ARCHIVED LOG 。

RL_FIRST_CHANGE＃

数

此行描述的归档重做日志中的第一个更改编号。仅在FILE_TYPE为ARCHIVED LOG 。

RL_FIRST_TIME

日期

此行描述的归档重做日志中第一次更改的时间。仅在FILE_TYPE为ARCHIVED LOG 。

RL_NEXT_CHANGE＃

数

在此行描述的归档日志之后更改编号。仅在FILE_TYPE为ARCHIVED LOG 。

RL_NEXT_TIME

日期

此行描述的归档日志之后的第一次更改的时间。仅在FILE_TYPE为ARCHIVED LOG 。
