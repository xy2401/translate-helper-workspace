9.16 V $ SESS_TIME_MODEL

V$SESS_TIME_MODEL显示各种操作的会话累计时间。 报告的时间是经过的总时间或CPU时间（以微秒为单位）。任何定时操作都将缓冲最多5秒的时间数据。具体而言，这意味着如果定时操作（例如SQL执行）需要很长时间才能执行，则发布到此视图的数据最多会缺少5秒的操作累计时间。

时间值是8字节整数，因此可以在换行前保存大约580,000年的时间。除非统计信息专门用于后台进程，否则后台处理时间不包括在统计值中。

柱 数据类型 描述

SID

数

会话ID（与V$SESSION相同）

STAT_ID

数

时间统计的统计标识符

STAT_NAME

VARCHAR2（64）

统计名称（ 见表9-1 ）

值

数

会话在此操作中花费的时间（以微秒为单位）

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

表9-1 V $ SESS_TIME_MODEL和V $ SYS_TIME_MODEL统计信息

统计名称 描述

DB时间

执行数据库用户级调用所花费的时间（以微秒为单位）。这不包括在PMON等实例后台进程上花费的时间。

DB CPU

在数据库用户级调用上花费的CPU时间（以微秒为单位）。这不包括在PMON等实例后台进程上花费的CPU时间。

背景经过的时间

数据库后台进程消耗的经过时间（以微秒为单位）。

后台CPU时间

数据库后台进程占用的CPU时间（以微秒为单位）。

序列加载经过的时间

从数据字典中获取下一个序列号所花费的时间。如果序列被缓存，那么这是在缓存用完时补充缓存所花费的时间。在缓存中找到序列号时，不会收取时间。对于非缓存序列，每次nextval调用都需要一些时间。

解析时间已过

解析SQL语句所花费的时间量。它包括软解析和硬解析时间。

硬解析经过的时间

花费在解析SQL语句上花费的时间。

SQL执行已用时间

SQL语句执行所用的时间量。请注意，对于select语句，这还包括执行查询结果提取所花费的时间。

连接管理呼叫已用时间

执行会话连接和断开呼叫所花费的时间。

解析经过的时间失败

执行SQL解析所花费的时间最终会因某些解析错误而失败。

失败的解析（超出共享内存）经过的时间

执行SQL解析所花费的时间最终因错误ORA-04031而失败。

硬分析（共享标准）经过的时间

由于无法在SQL缓存中共享现有游标而导致硬解析时执行SQL硬分析所花费的时间。

硬解析（绑定不匹配）经过的时间

当硬分析由绑定类型或绑定大小与SQL缓存中的现有游标不匹配导致执行SQL硬解析所花费的时间。

PL / SQL执行已用时间

运行PL / SQL解释器所花费的时间量。这不包括递归执行/解析SQL语句所花费的时间或递归执行Java VM所花费的时间。

PL / SQL编译经过的时间

运行PL / SQL编译器所花费的时间量。

入站PL / SQL rpc已用时间

时间入站PL / SQL远程过程调用已执行。它包括递归执行SQL和JAVA所花费的所有时间，因此不容易与“PL / SQL执行已用时间”相关。

Java执行时间

运行Java VM所花费的时间量。这不包括递归执行/解析SQL语句所花费的时间或递归执行PL / SQL所花费的时间。

RMAN cpu时间（备份/恢复）

在RMAN备份和还原操作中花费的CPU时间（以微秒为单位）。

重复绑定经过的时间

为绑定变量提供新值所花费的时间（重新绑定）。

OLAP引擎已用时间

执行OLAP会话事务所花费的时间。这包括在OLAP事务中花费在数据库用户级调用，SQL语句执行和PL / SQL执行上的时间。

OLAP引擎CPU时间

在OLAP会话事务上花费的CPU时间量。这包括在OLAP事务中花费在数据库用户级调用，SQL语句执行和PL / SQL执行上的时间。

表9-1中列出的统计信息之间的关系形成了两棵树，其中树中的子项报告的所有时间都包含在树中的父级中。以下是关系树;数字是给定树中的级别。 

1）后台经过时间2）后台cpu时间3）RMAN cpu时间（备份/恢复）1）DB时间2）DB CPU 2）连接管理调用经过时间2）序列加载经过时间2）sql执行经过时间2）解析时间已过去3）硬解析经过时间4）硬解析（共享标准）经过时间5）硬解析（绑定不匹配）经过时间3）失败解析经过时间4）失败解析（超出共享内存）经过时间2）PL / SQL执行已用时间2）入站PL / SQL rpc已用时间2）PL / SQL编译已用时间2）Java执行已用时间2）重复绑定已用时间

树中父项和子项之间的关系仅表示包含。关于树，请记住以下几点：

  • 儿童不一定与父母相加。

  • 儿童不一定是排他性的（也就是说，他们可能重叠）。

  • 儿童联盟并不一定涵盖整个父母。
