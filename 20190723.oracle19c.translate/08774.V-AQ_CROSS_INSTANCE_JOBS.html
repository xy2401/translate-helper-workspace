7.13 V $ AQ_CROSS_INSTANCE_JOBS

V$AQ_CROSS_INSTANCE_JOBS描述了每个跨进程作业。

每个作业用于将分片的消息从源实例转发到目标实例，以用于分片队列的一组订户。

柱 数据类型 描述

JOB_ID

数

此协调员中的职位ID

SCHEMA_NAME

VARCHAR2（128）

作业的源模式

QUEUE_NAME

VARCHAR2（128）

交叉作业的源队列名称

SHARD_ID

数

源分片ID

START_SUBSHARD_ID

数

启动作业的子硬件ID

DESTINATION_INSTANCE_ID

数

交叉作业的目标实例

COORDINATOR_ID

数

服务于该工作的协调员的索引

DEST_SERVER_PROCESS_ID

数

目标服务器的进程ID

JOB_STATE

VARCHAR2（28）

工作状态：

  • 请求的

  • RUNNING

  • 已停止

  • PAUSED

  • CRASHED

  • INACTIVE

流量控制

数

指示作业是否受流量控制：

  • 0 - 作业不受流量控制

  • 1 - 工作是流量控制的

MSGS_SENT

数

在工作期间发送的消息

BYTES_SENT

数

在工作期间发送的字节数

ACK_LATENCY

数

接收作业的ACK的延迟

工作类型

VARCHAR2（26）

跨实例作业类型。价值观：

  • CROSS_STREAM

    ：此作业类型负责将分片从其所有者实例转发到目标出列实例，以便在目标出列实例中从该分片执行出列的所有订阅者。

  • DEQUEUE_AFFINITY_TO_REMOTE ：此作业类型负责将订阅者的出列关联性从分片的所有者实例切换到远程出列实例。

  • DEQUEUE_AFFINITY_TO_LOCAL ：此作业类型负责将订阅者的出列关联性从远程出列实例切换回分片的所有者实例。

PRIORITY0_CROSS_LWM

数

在DESTINATION_INSTANCE_ID收到最后优先级0子硬币

PRIORITY1_CROSS_LWM

数

最后优先级1 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY2_CROSS_LWM

数

最后优先级2 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY3_CROSS_LWM

数

在DESTINATION_INSTANCE_ID收到最后优先级3子硬币

PRIORITY4_CROSS_LWM

数

最后优先级4 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY5_CROSS_LWM

数

最后优先级5 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY6_CROSS_LWM

数

最后优先级6 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY7_CROSS_LWM

数

最后优先级7 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY8_CROSS_LWM

数

最后优先级8 subhard在DESTINATION_INSTANCE_ID收到

PRIORITY9_CROSS_LWM

数

最后优先级9 subhard在DESTINATION_INSTANCE_ID收到

JOB_START_TIME

TIMESTAMP（3）带时区

这份工作的开始时间

SUBSCRIBER_ID

数

正在为DEQUEUE_AFFINITY_TO_REMOTE和DEQUEUE_AFFINITY_TO_LOCAL作业类型切换其亲缘关系的订户ID

SUBSCRIBER_NAME

VARCHAR2（512）

正在为DEQUEUE_AFFINITY_TO_REMOTE和DEQUEUE_AFFINITY_TO_LOCAL作业类型切换其亲缘关系的订户名称。对于CROSS_STREAM作业类型，此CROSS_STREAM NULL 。

OWNER_INSTANCE_ID

数

分片的所有者实例

QUEUE_ID

数

队列ID

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
