将Oracle Clusterware降级为早期版本

升级成功或失败后，您可以将Oracle Clusterware还原到以前的版本。

降级Oracle Clusterware会将Oracle Clusterware配置恢复到Oracle Grid Infrastructure 19c升级之前的状态。您在Oracle Grid Infrastructure 19c升级期间或之后执行的任何配置更改都将被删除，并且无法恢复。

要将Oracle Clusterware还原到先前版本，请对要降级的版本使用降级过程。

注意：

  • 从Oracle Grid Infrastructure 12c第2版（12.2）开始，您可以按任何顺序降级群集节点。您可以并行降级除1之外的所有群集节点。降级所有其他节点后，必须降级最后一个节点。

  • 升级失败后降级时，如果节点上不存在rootcrs.sh或rootcrs.bat文件，则使用命令perl rootcrs.pl而不是执行脚本。使用位于Oracle主目录中的Perl解释器。

  • Oracle Grid Infrastructure的选项降级
    您可以将Oracle Grid Infrastructure 19c降级到早期版本。
  • Oracle Grid Infrastructure的限制降级
    有关降级Oracle Grid Infrastructure安装的限制和更改，请查看以下信息。
  • 将Oracle Clusterware降级到18c
    升级成功或失败后，将所有Oracle Grid Infrastructure群集类型降级到18c的降级程序。
  • 将Oracle Clusterware降级到12c版本2（12.2）
    在升级成功或失败后，将所有Oracle Grid Infrastructure群集类型降级到12c版本2（12.2）的降级程序。
  • 降级到Oracle Grid Infrastructure 12c第1版（12.1）
    使用此过程降级到Oracle Grid Infrastructure 12c第1版（12.1）。
  • 降级到Oracle Grid Infrastructure 11g第2版（11.2）
    使用此过程降级到Oracle Grid Infrastructure 11g第2版（11.2）。
  • 使用在线中止升级降级Oracle Grid Infrastructure
    如果在设置Oracle Clusterware的活动版本之前升级Oracle Grid Infrastructure失败，请按照以下步骤将Oracle Grid Infrastructure降级到早期版本。
