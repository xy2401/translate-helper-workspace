B网关RPC接口

要使用Oracle Database Gateway for APPC执行远程事务程序，必须执行PL / SQL程序以调用网关功能， 远程过程调用（RPC）。网关功能处理与远程事务程序的网关会话的启动，数据交换和终止。

用于APPC的Oracle数据库网关包括一个工具PGAU，用于生成 PL / SQL包（TIP）根据您以COBOL记录布局和形式提供的定义自动生成 PGDL（程序网关定义语言）。

该网关功能全部通过执行远程过程调用（RPC）。函数从中调用 PL / SQL代码如下：

功能@ DBLINK（PARM1，parm2，...，parmn）;

表B-1描述了此语法中的参数：

表B-1网关功能

项目 描述

功能

是被调用函数的名称。

DBLINK

是Windows系统上网关服务器的预定义数据库链接的名称。

parm1，parm2，parmn

是本附录后面介绍的功能特定参数。

使用函数名@dblink表示法调用PL / SQL代码中的函数是一个远程过程调用。

PGAINIT和PGAINIT_SEC

PGAINIT 和PGAINIT_SEC是远程过程调用，用于启动与指定事务程序的APPC对话。两者之间的区别在于PGAINIT_SEC包括能够将网关会话安全用户ID和密码设置为除当前Oracle用户ID和密码之外的值的附加功能。成功完成任一功能后，会话就可以将数据发送到远程事务程序。

PGAXFER

PGAXFER以在PGAINIT初始化的网关会话上与远程事务程序之间传输数据。该功能基于调用参数发送和/或接收数据项。

表B-4列出了PGAXFER参数的类型，数据类型和说明：

表B-4 PGAXFER参数

参数 类型 数据类型 描述

CONVID

在

RAW（12）

对于使用SNA的网关： PGAINIT函数返回的对话标识符用于标识对话。

对于使用TCP / IP的网关： PGAINIT函数返回的套接字文件描述符用于标识对话。

SENDBUF

在

RAW（32763）

缓冲区包含要发送到远程事务程序的所有数据项。数据项按原样发送，不做任何更改。数据项必须按照远程事务程序期望接收它们的确切顺序出现在缓冲区中。所有数据项的总大小不能超过单个网关发送的最大大小，对于映射的网关会话，这是32,763字节。

SENDBUFL

在

BINARY_INTEGER

SENDBUF包含的数据项的总长度。范围是0 - 32,763字节。当没有要发送的数据项时，使用值“ 0 ”。

SENDLNS

在

RAW（1024）

缓冲区包含最多256 4字节整数值的数组。第一个整数值指定发送缓冲区（ SENDBUF ）中包含的数据项数。在该数据项计数之后是一系列指定数据项长度的整数值。数据项计数与数据项长度值的数量必须完全匹配。此阵列最多可以描述255数据项。所有数据项长度的总和不能超过SENDBUFL的总长度。

RECVBUF

OUT

RAW（32763）

缓冲区包含从远程事务程序接收的所有数据项。数据项按远程事务程序发送它们的确切顺序存储在此缓冲区中。所有数据项的总大小不能超过32,763字节的最大大小。

RECVBUFL

在

BINARY_INTEGER

接收缓冲区的总长度。范围是0 - 32,763字节。当没有要接收的数据项时，使用值“ 0 ”。

RECVLNS

进出

RAW（1024）

缓冲区包含最多256 4字节整数值的数组。第一个整数值指定要接收到接收缓冲区（ RECVBUF ）的数据项的数量。数据项计数之后是一系列整数值，指定要接收的数据项的最大长度。在输出时，这些值将替换为接收的数据项的实际长度。数据项计数与数据项长度值的数量必须完全匹配。此阵列最多可以描述255数据项。所有数据项长度的总和不能超过接收缓冲区的总长度（ RECVBUFL ）。

调用PGAXFER ， SENDBUFL和RECVBUFL任何一个或两个必须非零;换句话说，必须向远程交易程序发送或从远程交易程序接收至少一个数据项。如果在没有要发送或接收的数据项的情况下调用PGAXFER ，则会生成异常。

注意：

在每个PGAXFER调用中，首先进行所有发送处理，然后进行所有接收处理。如果事务以需要多组发送和接收的方式运行，则可以多次调用PGAXFER以适应事务。如果要发送或接收的数据超过32,763个字节，则必须多次调用PGAXFER 。
