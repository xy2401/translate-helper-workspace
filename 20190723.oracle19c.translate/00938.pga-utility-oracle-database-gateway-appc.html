2程序网关管理实用程序

程序网关管理实用程序（PGAU） 是一个实用程序，它帮助PGA管理员或用户定义要与远程事务程序交换的数据。它生成在创建提示 ， 提示内部以及根据您的通信协议， 客户端应用程序开发（仅限SNA）或客户端应用程序开发（仅限TCP / IP ）中讨论的PL / SQL事务接口包（ TIP ） 。

PGAU概述

注意：

如果您之前使用SNA协议在网关上生成了现有的TIP，并且您希望使用新的TCP / IP功能，则必须通过PGAU使用强制NLS_LANGUAGE和Side Profile Settings重新生成TIP。在DEFINE TRANSACTION命令中指定适当的ASCII字符集。

这是因为网关假定正在使用IMS Connect中的适当“用户出口”，这将在适当的ASCII和EBCDIC字符集之间进行转换。

PGAU维护着一个数据字典， PG DD，它是Oracle数据库中表的集合。这些表包含远程事务数据的定义以及如何与远程事务程序交换数据。有关TIP与其各自远程事务程序之间的相关性的讨论，请参阅“ 确保提示和远程事务程序通信 ” 。PG DD内容定义了这种相关性。

该 PGA管理员或用户使用以下内容定义TIP与远程事务程序之间的关联 PGAU命令（也称为“语句”）：

  • PGAU DEFINE DATA语句，描述要交换的数据。

  • PGAU DEFINE CALL语句，描述交换顺序。

  • PGAU DEFINE TRANSACTION语句，它将前面的CALL和DATA命令组合在一起，并描述远程事务程序特有的某些方面，例如其网络名称或位置。

  • PGA管理员或用户在PG DD中定义了TIP /事务关联后用于指定和创建TIP规范的PGAU GENERATE语句。 “ PGAU命令 ”中描述了更改和删除PG DD中定义所需的其他PGAU命令。

PGAU命令统称为程序网关定义语言（PGDL）。对PGDL的任何引用都是本书中定义的PGAU命令集合。

PGAU提供编辑和假脱机功能以及发出SQL命令的功能。

注意：

不要使用PGAU而不是SQL * Plus进行常规数据库管理。

或者， PGAU命令可以在控制文件中提供。该控制文件包含一个或多个用于操作的PGAU命令 PG DD或生成TIP规范。

PGAU在每个操作上发出状态消息。消息文本通过全球化支持消息支持提供。PGAU按顺序处理每个命令。单个命令出错导致PGAU跳过该命令。

要运行PGAU，必须已经创建了PG DD表。请参阅安装指南中与通信协议对应的网关配置部分。

COMMIT / ROLLBACK处理

以下部分提供有关COMMIT/ROLLBACK处理的信息。

  • COMMIT处理

  • ROLLBACK处理

COMMIT处理

PGAU永远不会发行 COMMIT命令。作为用户，这是你的责任， COMMIT时，所有的变化都实现PG DD变化。否则，当您退出PGAU会话时，Oracle会默认发出COMMIT命令。如果不提交PG DD更改，则必须运行退出前的ROLLBACK命令。

ROLLBACK处理

PGAU在每个PGAU命令的开头设置一个保存点，用于改变PG DD和PGAU GROUP的开头。任何PGAU命令或组失败时，PGAU都会回滚到保存点。

你可以编码 COMMIT或 PGAU脚本中的ROLLBACK命令，或PGAU中的交互式命令，但不在GROUP 。

PGAU GENERATE或REPORT命令都不发出COMMIT或ROLLBACK 。

有关将PGAU命令组合在一起以在发生故障时回滚更改的信息，请参阅后面章节中有关PGAU “ GROUP ”命令的讨论。

调用PGAU

在调用PGAU之前，应该已经设置了Oracle数据库。如果不是，请参阅安装指南中有关配置Oracle Database Gateway for APPC的部分。

在执行PGAU之前，必须将ORACLE_HOME环境变量设置为安装网关服务器的目录。

如果要以非英语语言接收PGAU消息，请将LANGUAGE环境变量设置为适当的值。

输入PGAU即可调用 pgau命令。通过使用以下语法在命令行上指定命令字符串，可以直接从操作系统提示符运行PGAU命令的准备脚本：

对于Microsoft Windows：

C：\> pgau @command_file C：\> pgau command = @ command_file C：\> pgau command =“@ command_file”

对于基于UNIX的系统：

$ pgau @command_file $ pgau command = @ command_file $ pgau command =“@ command_file”

默认扩展名为.sql 。如果命令文件名包含非字母数字字符，请使用最后一个表单。

去表演 PG DD维护和要生成PL / SQL包，必须使用PGAADMIN从PGAU连接到Oracle数据库 CONNECT命令。“ PGAU命令 ”部分讨论了如何使用“ CONNECT ”命令。

PGAU中的定义和生成

此版本的PGAU支持在COBOL中定义远程事务数据，以交互方式或在文件中输入。DEFINE和REDEFINE DATA命令支持文件输入，并且可以提供标准COBOL数据分割宏或“副本”。

PGAU和 PG DD支持不同版本的用户数据和远程事务定义。这有助于更改和测试数据格式和事务，而不会影响生产使用。

可能存在任何数据或事务定义的多个版本。您必须确保PG DD中存储和使用的版本与远程事务同步。网关，PGAU和生成的TIP都不提供此同步，但它们将在检测到错误条件时发出消息。

数据定义必须在被调用定义引用之前存在。在被事务定义引用之前，调用定义必须存在。

注意：

您有责任确保PG DD中存储和使用的数据事务定义版本与远程事务同步。网关，PGAU和生成的TIP不提供此同步，但在检测到错误条件时发出消息。

定义和测试TIP的过程

定义和测试给定事务的TIP的一般过程如下：

  1. 使用COBOL数据定义定义输入和输出。
  2. 重新定义上述过程创建的默认数据名和PL / SQL变量名（可选）。
  3. 定义要作为PL / SQL包的一部分生成的PL / SQL FUNCTION调用。
  4. 定义对上述功能进行分组的事务。
  5. 从先前存储的TRANSACTION ， CALL和DATA定义生成TIP规范。
  6. 生成TIP PL / SQL存储过程。
  7. 通过从高级应用程序调用TIP来测试TIP。

有关TIP的更多信息，请参阅创建提示 。

定义名称

定义名称是您通过PGAU指定的唯一标识符。名称是1到30个字节的字符串。如果包含标点符号或空格，则必须在双引号内指定名称。

假定名称在PG DD中是唯一的，除非有意地通过唯一的版本号区分重复的名称。确保名称唯一性是您的责任。

有效字符 PG DD定义名称是：

  • A到Z.

  • a到z

  • 0到9

  • ＃

  • $

  • _（下划线）

请注意，除非重写默认值，否则事务定义名称可能是PL / SQL包名称，而事务调用名称可能是PL / SQL过程名称。因此，选择在语法上对PL / SQL正确的名称，确保它们也是该系统中的唯一名称。作为用户，您有责任确保PL / SQL名称兼容性。

定义版本控制

该 PG数据字典表包含事务和数据结构的描述。可能有多个版本的定义。旧版本无限期保留。

在所有PG DD操作中，定义或包由其名称引用。该名称可以通过特定版本号进行限定。

所有版本号：

  • 由Oracle Sequence Objects提供

  • 纯粹是数字的

  • 必须没有用户更改，后缀或前缀

请参阅APPC数据字典的数据库网关和 pgddcr8.sql文件位于Microsoft Windows上的%ORACLE_HOME%\dg4appc\admin目录或基于UNIX的系统上的$ORACLE_HOME/dg4appc/admin目录中，用于生成版本号的Oracle序列对象的特定名称。

如果指定了显式版本号，则假定它是现有定义的版本号，而不是新定义。在以下情况下使用此类显式引用

  • 从特定的远程事务版本生成TIP

  • 根据特定数据版本定义远程事务

如果未指定显式版本：

  • 在引用定义时，假定最新（最高编号）。对于具有相同定义名称但不是CURRVAL编号的所有行，这是从VERSION列中选择的MAX值。

  • 在添加定义时，假设下一个（ NEXTVAL编号）。

版本号可能不连续。虽然版本号总是在增加，但给定定义的多个版本可能会跳过数字。这是因为对于相同类型的所有定义（ TRANSACTION ， CALL或DATA ）共享序列对象，并且在Oracle数据库事务ROLLBACK情况下不会还原序列对象NEXTVAL 。因此，可以在相同定义的下一版本之前将NEXTVAL分配给不同的定义。

有效定义名称的示例：

DEFINE TRANSACTION | CALL | DATA工资单（新的或最新的定义）payroll_xaction（新的或最新的定义）payroll_xaction VERSION（3）...（现有定义）

PGAU没有尝试同步版本。尽管在定义时确保了依赖项的存在，但是在不参考依赖性的情况下进行删除。例如，生成TIP需要事先定义事务，这需要事先定义调用，这需要事先定义数据。但是，当调用定义仍然引用它时，没有什么能阻止PGAU删除活动数据定义。

关键词

所有PGAU关键字都可以大写或小写指定，而不是保留字。不需要保留，因为所有关键字都已知拼写并出现在可预测的位置，并且因为所有数据都由括号，撇号，引号或空格分隔。

请注意，关键字指定的所有未加引号的值都存储在除非在关键字描述中另有说明，否则PG数据字典为大写。

2 3

ENVIRONMENT ( name )

指定此事务的主机环境的名称，例如IBM370 。如果省略此参数，则假定为IBM370 。IBM370是此版本PGAU支持的唯一环境。

SIDEPROFILE ( name )

对于使用SNA的网关，此参数是可选的，但如果省略，则用户必须指定 LUNAME ， TPNAME和LOGMODE的参数。它指定SNA侧信息配置文件的名称，该配置文件将APPC连接定向到事务管理器。该名称的长度可以是1到8字符。只有在不加引号的情况下，名称值才能是带有“@”，“＃”和“$”字符的字母数字。带引号的值可以包含任何字符，并用引号（“）或撇号（'）分隔。保留所有值的大小写。

是一个可选参数，有两个选项， TRACE和PKGEX 。

TRACE子选项以逗号分隔。

跟踪消息作为诊断工具提供给Oracle支持服务和其他Oracle代表，以帮助他们在生成TIP时诊断客户问题。它们是Oracle保留功能的一部分，其使用，界面和文档可能会在未经通知的情况下由Oracle自行决定更改。提供此信息以便客户记录问题症状。

  • OC - Oracle呼叫

写入消息，跟踪SQL语句处理的Oracle UPI调用结果和PG DD的SELECT 。

  • TG - TIP生成

组

目的

如果任何一个语句失败，可以将多个PGAU命令组合在一起，以便更新PG DD，并回滚组中命令产生的所有更改。

即使组内的所有命令都成功，也不会执行COMMIT处理。您执行COMMIT或者通过编码COMMIT命令在PGAU脚本，外GROUP S，或通过发出COMMIT以交互方式PGAU。

如果组中的任何语句失败，PGAU会在处理组之前向条件发出保存点ROLLBACK 。

句法

GROUP（ pgaustmt1; pgaustmt2; ... pgaustmtN） ;

参数

pgaustmtN:是PGAU DEFINE ， REDEFINE或UNDEFINE语句

例子

GROUP（DEFINE DATA EMPNO PLSDNAME（EMPNO）使用（通过）语言（IBMVSCOBOLII）（01 EMP-NO PIC X（6）。）; DEFINE CALL DB2IMAIN PKGCALL（PGADB2I_MAIN）PARMS（（EMPNO IN），（EMPREC OUT））; DEFINE TRANSACTION DB2I CALL（DB2IMAIN，DB2IDIAG）SIDEPROFILE（CICSPROD）TPNAME（DB2I）LOGMODE（ORAPLU62）SYNCLEVEL（0）NLS_LANGUAGE（“AMERICAN_AMERICA.WE8EBCDIC37C”）;生成DB2I PKGNAME（PGADB2I）OUTFILE（“pgadb2i”）;）;

GROUP使用说明：

  • 不能将非PGAU命令（例如ORACLE或SQL）放在分隔组的括号内。

  • PGAU脚本可以包含多个GROUP 。每个GROUP都可以穿插SQL命令，例如COMMIT或SELECT或者使用PGAU命令，例如GENERATE或REPORT 。

  • 组内第一个失败的PGAU语句导致保存点ROLLBACK在组的开头处条件。组内的所有后续命令都被刷新，不会被检查。PGAU执行将继续执行该组后面的语句。如果该语句是COMMIT ，则在COMMIT失败组之前进行所有PG DD更改。

（定义）

与INFILE参数互斥。它是数据的内联描述。必须以COBOL语法提供描述。此内联描述必须以左括号开头，并以右括号结束。左括号必须是该行上的最后一个非空白字符，并且COBOL数据定义必须在新行上开始，遵循列使用和连续的标准COBOL规则。右括号和终止分号必须位于COBOL数据定义的最后一行之后的单独行上。如果在COBOL中，则规范是COBOL数据项或结构，根据COBOL规则定义。假设边距是标准的，支持显式或隐式延续。包含PL / SQL使用的无效字符（例如“ - ”）的数据名称将转换为最接近的等效项，并根据需要进行截断。

INFILE ( "filespec" )

与(definition)参数互斥。它表示要从filespec,描述的操作系统文件中读取定义filespec,而不是(definition)描述的内联定义。

请注意，“ filespec ”必须用双引号括起来。

例子

REDEFINE DATA使用说明：

  • PLSDNAME ， FIELD或PLSFNAME允许重新定义单个数据项的名称，而(definition)参数重新定义命名数据项的内容。

  • FIELD的存在表示仅更新单个数据字段（由dname ， fname和version)唯一标识的单个PG DD行version) 。缺少FIELD表示由定义输入更新或替换多个数据字段（由dname和version标识的多个PG DD行）。

  • REMOTE_LANGUAGE和LOCAL_LANGUAGE覆盖用于转换任何单个SBCS，DBCS或MBCS字符数据字段的字符集。

  • LANGUAGE (language)和(definition)| INFILE ( "filespec")作为一个组是强制性的。如果要提供数据定义，则必须指定LANGUAGE参数，然后还必须指定内联定义或INFILE 。

  • 存在(definition) | INFILE ("filespec" ）表示由定义输入更新或替换多个数据字段（由dname和version标识的那些PG DD行）。更换可能会导致更少，相同或更多的字段。

  • INFILE ("filespec")是磁盘文件的特定于平台的名称。

  • 只有在使用指定了TRUNC(BIN)编译器选项的COBOL编译远程主机事务时，才应使用COMPOPTS ('TRUNC(BIN)') 。使用此选项时，定义为PIC 9(4)或PIC S9(4)二进制数据项实际上可以包含5位数的值，定义为PIC 9(9)或PIC S9(9)二进制数据项实际上可以包含10位数的值。如果没有COMPOPTS ('TRUNC(BIN)') ，PGAU会为这些数据项生成NUMBER(4,0)或NUMBER(9,0)字段，从而可能会截断值。当COMPOPTS ('TRUNC(BIN)') ，PGAU为这些数据项生成NUMBER(5,0)或NUMBER(10, 0) 10,0 NUMBER(10, 0)字段，避免任何截断值。编写客户端应用程序时必须小心，以确保不会将无效值发送到远程主机事务。对于PIC 9(4)该值必须在0到32767范围内，对于PIC S9(4)该值必须在-32767到+32767范围内，对于PIC 9(9)该值必须在范围0到2,147,483,647 ，对于PIC S9(9)该值必须在-2,147,483,647到+2,147,483,647的范围内。COBOL始终为符号保留二进制字段的高位，因此无符号字段的值范围仅限于有符号字段的值范围的绝对值。有关详细信息，请参阅相应的IBM COBOL编程手册。
