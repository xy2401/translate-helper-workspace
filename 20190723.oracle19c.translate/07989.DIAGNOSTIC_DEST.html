1.103 DIAGNOSTIC_DEST

从Oracle Database 11g第1版（11.1）开始，每个数据库实例的诊断都位于由DIAGNOSTIC_DEST初始化参数指定的专用目录中。

属性 描述

参数类型

串

句法

DIAGNOSTIC_DEST = {pathname |目录 }

默认值

派生自$ORACLE_BASE环境变量的值。如果未设置$ORACLE_BASE ，则从Oracle Universal Installer设置的ORACLE_BASE派生。如果未设置ORACLE_BASE ，则使用$ORACLE_HOME/rdbms/log 。

可修改

改变系统

可在PDB中修改

没有

基本

没有

Oracle RAC

可以在每个实例上设置此参数。Oracle建议群集中的每个实例指定位于共享磁盘上的DIAGNOSTIC_DEST目录位置，并为每个实例指定DIAGNOSTIC_DEST的相同值。

DIAGNOSTIC_DEST指定的目录结构如下：

< diagnostic_dest > / diag / rdbms / < dbname > / < instname >

此位置称为自动诊断存储库（ADR）主页。例如，如果数据库名称为proddb且实例名称为proddb1 ，则ADR主目录将为<diagnostic_dest>/diag/rdbms/proddb/proddb1 。

以下文件位于ADR主目录下：

  • 跟踪文件 - 位于子目录<adr-home>/trace

  • 警报日志 - 位于子目录<adr-home>/alert 。此外， alert.log文件现在是XML格式，符合Oracle ARB日志记录标准。

  • 核心文件 - 位于子目录<adr-home>/cdump

  • 事件文件 - 每个严重错误的发生（例如，ORA-600，ORA-1578，ORA-7445）都会导致创建事件。为每个事件分配一个ID，并为每个事件（错误堆栈，调用堆栈，块转储等）转储存储在其自己的文件中，与进程跟踪文件分开。事件转储文件位于<adr-home>/incident/<incdir#> 。您可以在进程跟踪文件中找到事件转储文件位置。
