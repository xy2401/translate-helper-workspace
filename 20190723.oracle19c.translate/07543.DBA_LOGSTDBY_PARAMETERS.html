5.164 DBA_LOGSTDBY_PARAMETERS

DBA_LOGSTDBY_PARAMETERS显示SQL应用于逻辑备用数据库所使用的参数列表。

此视图仅适用于逻辑备用数据库。

柱 数据类型 空值 描述

名称

VARCHAR2（64）

参数名称：

  • MAX_SGA - 为日志应用服务缓存分配的系统全局区域（SGA）（以兆字节为单位）

  • MAX_SERVERS - SQL Apply服务使用的进程数

  • PREPARE_SERVERS - 控制用于准备更改的并行执行服务器的数量

  • APPLY_SERVERS - 控制用于应用更改的并行执行服务器的数量

  • MAX_EVENTS_RECORDED - DBA_LOGSTDBY_EVENTS视图中存储的事件数

  • RECORD_SKIP_ERRORS - 表示跳过的记录

  • RECORD_SKIP_DDL - 表示跳过的DDL语句

  • RECORD_APPLIED_DDL - 表示已应用的DDL语句

  • RECORD_UNSUPPORTED_OPERATIONS - 显示SQL Apply是否将捕获有关在DBA_LOGSTDBY_EVENTS视图中在主数据库中执行不受支持的操作的事务的信息

  • EVENT_LOG_DEST - 指示SQL Apply记录有趣事件发生的位置

  • LOG_AUTO_DELETE - 显示在逻辑备用数据库中应用日志内容后，SQL Apply是否将自动删除从主数据库接收的远程存档日志。

  • LOG_AUTO_DEL_RETENTION_TARGET - 一旦SQL Apply应用了日志内容，从主数据库收到的远程归档日志将在逻辑备用数据库中保留多少分钟。

  • PRESERVE_COMMIT_ORDER - 显示是否在逻辑备用数据库中以与在主数据库中提交的顺序相同的顺序提交事务

值

VARCHAR2（2000）

参数的值

单元

VARCHAR2（64）

值的单位（如果适用）

设置

VARCHAR2（64）

可能的值如下：

  • SYSTEM - 用户未明确设置参数值。但是，用户可以通过适当调用APPLY_SET过程来更改它。

  • USER - 参数值由用户显式设置

动态

VARCHAR2（64）

如果可以动态设置参数，则为YES （即，无需停止SQL Apply）

如果设置参数需要停止SQL Apply，则为NO

在CDB中，此视图显示在根中查询时的数据。
