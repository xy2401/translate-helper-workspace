C PL / SQL程序限制

本附录描述了PL / SQL语言强加的程序限制。PL / SQL基于编程语言Ada。因此，PL / SQL使用描述性中间属性表达式Ada（DIANA）的变体，这是一种树形结构的中间语言。它使用称为接口定义语言（IDL）的元交换来定义。DIANA由编译器和其他工具在内部使用。

在编译时，PL / SQL源文本被翻译成系统代码。DIANA和子程序或包的系统代码都存储在数据库中。在运行时，它们将加载到共享内存池中。DIANA用于编译依赖的子程序;系统代码只是运行。

在共享内存池中，包规范，ADT规范，独立子程序或匿名块仅限于67108864（2 ** 26）DIANA节点，这些节点对应于标识符，关键字，运算符等标记。这允许大约6,000,000行代码，除非您超出PL / SQL编译器施加的限制，其中一些在表C-1中给出。

要估计程序单元需要多少内存，可以查询静态数据字典视图USER_OBJECT_SIZE 。 PARSED_SIZE列返回“展平”DIANA的大小（以字节为单位）。例如：

创建或替换包pkg1 AS TYPE numset_t是数字表;功能f1（x NUMBER）RETURN numset_t PIPELINED;结束pkg1; / CREATE PACKAGE BODY pkg1 AS  -  FUNCTION f1返回一个元素集合（1,2,3，... x）功能f1（x NUMBER）返回numset_t PIPELINED开始于i IN 1..x LOOP PIPE ROW（i ）;结束循环;返回; END f1;结束pkg1; /

用于格式化下一个查询结果的SQL * Plus命令：

列名称格式A4列类型格式A12列数source_size格式999列数parsed_size格式999列号code_size格式999列错误大小格式999

查询：

SELECT * FROM user_object_size WHERE name ='PKG1'ORDER BY类型;

结果：

名称类型SOURCE_SIZE PARSED_SIZE CODE_SIZE ERROR_SIZE ---- ------------ ----------- ----------- ------ --- ---------- PKG1 PACKAGE 112 498 310 79 PKG1 PACKAGE BODY 233 106 334 0

不幸的是，您无法从解析的大小估计DIANA节点的数量。具有相同解析大小的两个程序单元可能分别需要1500和2000个DIANA节点，因为例如，第二个单元包含更复杂的SQL语句。

当一个PL / SQL块，子程序，程序包，或架构级别的用户定义类型超出大小限制，则得到一个错误，例如PLS-00123: program too large 。通常，包或匿名块会出现此问题。使用包装，最好的解决方案是将其分成更小的包装。使用匿名块，最佳解决方案是将其重新定义为一组子程序，这些子程序可以存储在数据库中。

有关数据类型限制的更多信息，请参阅PL / SQL数据类型 。
