系统管理分片的多表系列支持

Oracle Database 18c的Oracle Sharding功能仅支持每个分片数据库的一个表系列（一组共享相同分片键的相关表）。在Oracle Database 19c中，Oracle Sharding包括对多个表系列的支持，其中来自不同表系列的所有数据都位于相同的块中。此功能仅适用于系统管理的分片数据库。访问不同表系列的不同应用程序现在可以托管在一个分片数据库中。

有一个新的GDSCTL命令， CONFIG TABLE FAMILY ，以及其他几个命令被扩展为支持此功能： ADD SERVICE ， MODIFY SERVICE ， CONFIG SERVICE ， CONFIG CHUNKS ， STATUS ROUTING和验证VALIDATE CATALOG 。

此功能没有引入新的SQL关键字或语句;但是，使用CREATE SHARDED TABLE ， DUPLICATED TABLE和TABLESPACE SET会改变一些限制。

GSMROOTUSER

一个名为GSMROOTUSER的新用户用于以分片配置登录CDB $ ROOT以获取CDB（此用户未在GDS配置中使用）。CDB中与CDB $ ROOT的任何连接现在都将与GSMROOTUSER连接。

弃用和支持

此版本中不推荐使用或不支持以下功能：

  • 支持在GDSCTL命令行中设置密码

父主题： Oracle Database 19c中的更改

支持在GDSCTL命令行中设置密码

为了增强安全性，从Oracle Database 19c开始，不再支持从操作系统提示符调用时从全局数据服务控制实用程序（GDSCTL）命令行指定密码的功能。

此desupport仅适用于从用户命令行提示符调用GDSCTL的密码更改。例如，不支持以下命令：

$ gdsctl add database -connect inst1 -pwd gsm_password

从GDSCTL实用程序本身指定密码仍然有效。例如，以下命令有效：

GDSCTL> add database -connect inst1 -pwd gsm_password

在从操作系统提示符调用的GDSCTL命令中指定密码时，此弃用解决了安全漏洞。

以下功能是此版本中的新增功能：

  • Oracle Sharding

    Oracle Sharding是一种可扩展性和可用性功能，支持在不共享硬件或软件的独立Oracle数据库池中分发和复制数据。在此版本中，全局服务管理器用作将连接请求路由到单个分片的分片导向器，GDS目录扩展为支持包含分片数据库元数据的分片目录，GDSCTL增强了几个新命令以方便分片数据库的创建，监视和生命周期管理。
