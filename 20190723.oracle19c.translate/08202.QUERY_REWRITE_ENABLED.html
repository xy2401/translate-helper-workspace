1.273 QUERY_REWRITE_ENABLED

使用QUERY_REWRITE_ENABLED为数据库启用或禁用全局查询重写。

属性 描述

参数类型

串

句法

QUERY_REWRITE_ENABLED = {false |是的|力量

默认值

如果OPTIMIZER_FEATURES_ENABLE设置为10.0.0或更高，则为true

如果OPTIMIZER_FEATURES_ENABLE设置为9.2.0或更低，则为false

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

Oracle RAC

多个实例可以具有不同的值。

值

  • 假

    Oracle不使用重写。

  • 真正

    Oracle使用重写和无需重写的成本来查询查询，并选择成本较低的方法。

  • 力

    Oracle总是使用重写，并且在执行此操作之前不会评估成本。当您知道查询将始终受益于重写以及何时减少编译时间时，请使用force 。

要利用特定物化视图的查询重写，必须为该物化视图启用查询重写，并且必须启用基于成本的优化。

也可以看看：

  • “Oracle数据库数据仓库指南” ，了解有关实例化视图的查询重写的信息

  • 有关基于成本的优化的信息，请参见 “ Oracle数据库SQL调优指南”和“ OPTIMIZER_MODE ”
