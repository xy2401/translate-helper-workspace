1.3参数文件

参数文件是包含初始化参数列表和每个参数值的文件。您可以在参数文件中指定反映特定安装的初始化参数。

Oracle支持以下两种类型的参数文件：

  • 服务器参数文件

  • 初始化参数文件

1.3.1服务器参数文件

服务器参数文件是二进制文件，充当初始化参数的存储库。

服务器参数文件可以驻留在Oracle数据库服务器执行的计算机上。存储在服务器参数文件中的初始化参数是持久的，因为在实例运行时对参数所做的任何更改都可以在实例关闭和启动时保持不变。

1.3.2初始化参数文件

初始化参数文件是包含初始化参数列表的文本文件。

注意：

有关在初始化参数文件中指定值时要使用的字符集的详细信息，请参阅“ 关于参数值的字符集”。

以下是初始化参数文件中的示例条目：

PROCESSES = 100 OPEN_LINKS = 12 GLOBAL_NAMES = true

初始化参数文件的名称因操作系统而异。例如，它可以是大小写混合或小写，也可以是逻辑名称或名称init.ora的变体。还提供了一个initdw.ora文件，其中包含数据仓库和数据集市的建议参数设置。数据库管理员可以为初始化参数文件选择不同的文件名。

有关操作系统上初始化参数文件的默认位置和文件名，请参阅特定于操作系统的Oracle文档。初始化参数文件由用于启动服务器的客户端工具（例如SQL * Plus）读取。

每个操作系统的Oracle分发介质上都提供了示例初始化参数文件。样本文件足以供初次使用，但您可能希望修改该文件以调整数据库以获得最佳性能。完全关闭并重新启动实例后，任何更改都将生效。

1.3.2.1关于参数值的字符集

参数值中仅支持数据库平台的默认字符集中的字符。对于除IBM z / OS和Fujitsu BS2000之外的所有平台，默认平台字符集为US7ASCII（7位ASCII）。

指定具有不受支持的字符的参数值时，数据库行为未定义。使用ALTER SYSTEM语句在服务器参数文件中设置参数值时，请确保仅使用支持的字符。当指定了包含不受支持的字符的值时，数据库并不总是报告错误。

初始化参数文件是客户端文件。使用时，它位于启动SQL * Plus的主机上以启动数据库实例。必须使用NLS_LANG客户端设置指定的客户端字符集写入初始化参数文件。但是，参数值不应包含数据库的默认字符集不支持的字符。

1.3.2.2在初始化参数文件中指定值

本节介绍在初始化参数文件中设置参数值的几个方面。

1.3.2.2.1初始化参数文件的规则

本节介绍管理初始化参数文件中参数规范的规则。

  • 初始化参数文件应仅包含参数和注释。井号（＃）开始注释行。该行的其余部分被忽略。

  • 您可以按任何顺序指定参数。

  • 仅当主机操作系统上的情况很重要时，文件名中的大小写（上部或下部）才有意义。

  • 要在一行中输入多个参数，请在参数名称和值之间使用空格，如下例所示：

    PROCESSES = 100 CPU_COUNT = 1 OPEN_CURSORS = 10

  • 某些参数（如ROLLBACK_SEGMENTS ）接受多个值。以下任何内容都代表有效的语法。

    • 输入括在括号中的多个值，并用逗号分隔。例如：

      ROLLBACK_SEGMENTS =（SEG1，SEG2，SEG3，SEG4，SEG5）

    • 输入多个值，不带括号和逗号。例如：

      ROLLBACK_SEGMENTS = SEG1 SEG2 SEG3 SEG4 SEG5

    • 输入多个值，每行一个。例如：

      ROLLBACK_SEGMENTS = SEG1 ROLLBACK_SEGMENTS = SEG2 ROLLBACK_SEGMENTS = SEG3 ROLLBACK_SEGMENTS = SEG4 ROLLBACK_SEGMENTS = SEG5

    如果在多行上输入一个参数的值，则条目必须在连续的行上。如果不是，那么第一个条目将无法正确处理。例如，在以下条目中， SEG3和SEG4的设置将覆盖SEG1和SEG2的设置：

    ROLLBACK_SEGMENTS = SEG1 SEG2 OPEN_CURSORS = 10 ROLLBACK_SEGMENTS = SEG3 SEG4

  • 反斜杠（\），也称为转义字符，表示继续参数规范。如果反斜杠继续一行，则续行必须没有前导空格。例如：

    ROLLBACK_SEGMENTS =（SEG1，SEG2，\ SEG3，SEG4，SEG5）

  • 您可以使用IFILE初始化参数将另一个初始化参数文件的内容嵌入到当前初始化参数文件中。

  • 在引号中包含任何包含空格或制表符的参数值。除非另有说明，否则您可以使用单引号或双引号。例如：

    NLS_TERRITORY ='捷克共和国'

    注意：

    在初始化参数文件中按字母顺序列出参数可以帮助您找到它们，并且可以帮助确保每个参数仅指定一次。

  • 在引号中包含任何包含特殊字符的参数值。

1.3.2.2.2在参数值中使用特殊字符

如果参数值包含特殊字符，则特殊字符必须以反斜杠开头，或者整个参数值必须用引号括起来。

例如，您可以使用以下任一方法指定特殊字符：

DB_DOMAIN ='JAPAN.ACME＃。COM'DB_DOMAIN = JAPAN.ACME \＃。COM

1.3.2.2.3使用转义字符

在初始化参数文件中，反斜杠（\）也可以表示行继续。如果反斜杠后跟一个字母数字字符，则反斜杠将被视为输入中的普通字符。

如果后面没有字母数字字符，则反斜杠将被视为反斜杠或延续字符。

1.3.2.2.4使用引号

引号可以在初始化参数文件中以两种方式嵌套。

第一种方法是将嵌套字符串中的引号加倍。例如：

NLS_DATE_FORMAT ='''今天是''MM / DD / YYYY'

第二种方法是交替单引号和双引号。例如：

NLS_DATE_FORMAT ='“今天是”MM / DD / YYYY'
