1.65 DATA_TRANSFER_CACHE_SIZE

DATA_TRANSFER_CACHE_SIZE设置用于接收数据块（通常来自Oracle Data Guard环境中的主数据库）的数据传输高速缓存的大小（以字节为单位），以便在执行RMAN RECOVER ...期间由实例使用RMAN RECOVER ...NONLOGGED BLOCK命令。

属性 描述

参数类型

大整数

句法

DATA_TRANSFER_CACHE_SIZE = integer [K | M | G]

默认值

如果设置了SGA_TARGET ，则如果未指定DATA_TRANSFER_CACHE_SIZE ，则数据传输高速缓存的缺省大小为0（由Oracle数据库内部确定）。如果设置了SGA_TARGET并且指定了DATA_TRANSFER_CACHE_SIZE ，则用户指定的值指示数据传输高速缓存的最小值。

如果未设置SGA_TARGET ，则数据传输缓存将不可用。

可修改

改变系统

可在PDB中修改

没有

价值范围

0 - 512M，向上舍入到下一个颗粒大小

基本

没有

Oracle RAC

多个实例可以具有不同的值。

需要仅为不使用自动内存管理且使用RMAN RECOVER ...数据库设置此初始化参数RMAN RECOVER ...NONLOGGED BLOCK命令。
