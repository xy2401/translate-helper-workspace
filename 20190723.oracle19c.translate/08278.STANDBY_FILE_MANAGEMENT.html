1.327 STANDBY_FILE_MANAGEMENT

STANDBY_FILE_MANAGEMENT启用或禁用自动备用文件管理。

属性 描述

参数类型

串

句法

STANDBY_FILE_MANAGEMENT = {MANUAL |自动}

默认值

手册

可修改

改变系统

可在PDB中修改

没有

基本

没有

启用自动备用文件管理后，将在备用数据库上复制主数据库上的操作系统文件添加和删除。STANDBY_FILE_MANAGEMENT仅适用于物理备用数据库。

值

  • 手册

    禁用自动备用文件管理

  • 汽车

    启用自动备用文件管理

将STANDBY_FILE_MANAGEMENT设置为AUTO会导致Oracle在备用数据库上自动创建文件，并在某些情况下覆盖现有文件。设置STANDBY_FILE_MANAGEMENT和DB_FILE_NAME_CONVERT时必须小心，以免意外覆盖现有的备用文件。

如果备用数据库与主数据库位于同一系统上，请确保主系统和备用系统不指向相同的文件。
