139 PROC-00001至PROC-00111

PROC-00001：无法初始化Oracle Cluster Registry。

原因：传递了无效的初始化级别以初始化Oracle Cluster Registry。

操作：使用有效的init级别初始化Oracle Cluster Registry。

PROC-00002：无法在当前初始化级别执行群集注册表操作。

原因：程序尝试了一个群集注册表操作，这在init级别是不允许的。

操作：使用适当的init级别初始化Oracle Cluster Registry。

PROC-00003：无法执行群集注册表操作，因为上下文无效。

原因：传递的Oracle Cluster Registry上下文无效。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00004：要运行的群集注册表项不存在。

原因：程序尝试对不存在的密钥执行集群注册表操作。

操作：检查密钥是否存在并再次尝试操作。

PROC-00005：用户无权对此密钥执行群集注册表操作。

原因：程序尝试对其没有权限的群集注册表项执行操作。

操作：检查程序是否以正确的用户身份运行。

PROC-00006：用户无权对此密钥执行群集注册表操作。

原因：程序尝试对其没有权限的密钥执行群集注册表操作。

操作：检查程序是否以正确的用户身份运行。

PROC-00007：无法执行群集注册表操作，因为密钥或批处理句柄无效。

原因：传递的群集注册表项或批处理句柄无效。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00008：无法执行群集注册表操作，因为其中一个参数无效。

原因：程序尝试使用无效参数进行集群注册表操作。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00009：无法执行群集注册表操作，因为传递的缓冲区太小。

原因：为缓冲区分配的内存量不足。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00010：当子键存在时，无法删除群集注册表项。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00011：无法执行群集注册表操作，因为密钥为NULL。

原因：传递的群集注册表项为NULL。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00012：堆内存不足。

原因：程序耗尽了堆内存。

操作：联系Oracle支持服务。

PROC-00013：无法执行群集注册表操作，因为其中一个参数为NULL。

原因：程序尝试使用NULL参数进行集群注册表操作。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00014：发生未知错误

原因：这是Oracle Cluster Registry的一般内部错误。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00015：无法初始化群集注册表

原因：无法访问Oracle Cluster Registry（OCR）。可能的原因是：1。群集注册表尚未格式化。2。无法访问群集注册表位置。3。群集注册表位置具有不正确的权限。4。Oracle Clusterware未运行。

行动： 1。运行'ocrcheck'命令以确认OCR已格式化且可用。2。运行'ocrcheck -config'命令以检索已配置的位置，并验证每个位置是否可访问，并通过本机平台或操作系统方法具有正确的权限。如果某个位置是ASM磁盘组，请验证该磁盘组是否存在并已装入。3。运行'crsctl check crs'命令以验证Oracle Clusterware是否正在运行。

PROC-00016：无法检索给定键控句的任何其他子项

原因：该键没有更多枚举的子键。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00017：未实现的功能

原因：当前版本中尚未实现请求的功能。

操作：在即将发布的Oracle版本中再次尝试该功能。

PROC-00018：传递的值大小太大

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00019：无法为密钥创建更多子密钥

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00020：未设置ORACLE_HOME环境

操作：使用root.sh脚本。'ocrconfig -upgrade'命令将初始化OCR。

PROC-00026：访问物理存储时出错

原因：无法访问Oracle Cluster Registry。可能的原因是：1。群集注册表尚未格式化。2。无法访问群集注册表位置。3。群集注册表位置具有不正确的权限。4。Oracle Clusterware未运行。

行动： 1。运行'ocrcheck'命令以确认OCR已格式化且可用。2。运行'ocrcheck -config'命令以检索已配置的位置，并验证每个位置是否可访问，并通过本机平台或OS方法具有正确的权限。如果某个位置是ASM磁盘组，请验证该磁盘组是否存在并已装入。3。运行'crsctl check crs'命令以验证Oracle Clusterware是否正在运行。

PROC-00027：身份验证层出错

原因：执行群集注册表操作时身份验证层中发生错误。

操作：联系Oracle支持服务。

PROC-00028：Oracle Cluster Registry已在当前版本中

原因：尝试升级已升级的群集注册表。

行动：无需采取任何行动。

PROC-00029：记录层错误

原因：执行群集注册表操作时，日志记录层中发生错误。

行动：这是一个内部错误。联系Oracle支持服务。

PROC-00030：Oracle Cluster Registry ID不匹配

原因：客户端看到的群集注册表存储与服务器看到的不同。这可能是由于配置错误造成的。

操作：联系Oracle支持服务。

PROC-00031：群集注册表的存储设备太小

原因：配置的存储设备或要新添加到群集注册表的存储设备太小。

操作：请参阅文档并配置足够大的存储设备以供群集注册表使用。如果已配置群集注册表，请执行'ocrcheck'命令以检索总大小，并确保要添加的设备大小足以供群集注册表使用。

PROC-00032：本地节点上的群集就绪服务未运行

原因：本地节点上的Cluster Ready Services未运行。

操作：使用'crsctl start crs'命令在本地节点上启动Cluster Ready Services。

PROC-00033：未配置Oracle Cluster Registry

原因：节点上不存在Oracle Cluster Registry配置。

操作：安装和配置Oracle Clusterware。

PROC-00034：Oracle Cluster Registry配置不兼容

原因：此节点上的群集注册表配置与群集中的另一个节点不匹配。

操作：在群集中的所有节点上使用“ocrcheck -config”命令检索群集注册表配置。修复此节点上的配置，并使用'ocrconfig -repair'命令使其保持一致。

PROC-00035：由于版本无效，无法执行群集注册表操作

原因：无法对此群集活动版本执行操作

操作：在重试操作之前完成升级

PROC-00036：没有可用空间来执行此操作

原因：没有剩余空间来执行请求的操作。

操作：释放空间并重试操作。

操作：将群集注册表的配置更改为支持的存储类型。有关每个平台的详尽列表，请参阅Oracle文档。

操作：重试该操作。

PROC-00049：节点string上无法访问指定的Oracle Cluster Registry备份位置。

原因： Oracle Cluster Registry备份位置不存在，未正确安装，或者在指示的节点上没有所需的权限。

PROC-00050：要添加的Oracle Cluster Registry位置在节点string上不可访问。

PROC-00053：配置的补丁级别，发布补丁级别和其他活动集群节点上的补丁级别并不完全相同。

原因：由于补丁级别不匹配，尝试的Oracle Clusterware堆栈启动操作失败。出现这种情况的原因有以下几种： - 在修补程序应用程序完成之前启动了Oracle Clusterware。- Oracle Clusterware是从最近修补的Grid Infrastructure主目录以外的位置启动的。

操作：检查Cluster Ready Services（CRS）守护程序日志文件以获取详细信息。1。完成在此节点上应用修补程序。2。确保从最近修补的Grid Infrastructure主目录启动Oracle Clusterware。使用'crsctl query crs releasepatch'和'crsctl query crs softwarepatch'命令验证补丁级别是否匹配。如果问题仍然存在，请与Oracle支持服务联系。

操作：使用“ocrconfig -showbackup”命令检查Oracle Cluster Registry备份的可用性，并还原兼容的备份。如果无法解决问题，请与Oracle支持服务联系。

PROC-00058：指定用于备份Oracle Cluster Registry的文件已存在。

操作：使用提供的其他文件名作为参数运行命令。

PROC-00059：无法访问指定的Oracle Cluster Registry导出或导入文件位置。

原因：指定的Oracle Cluster Registry导出或导入文件位置不存在，未正确装入，没有所需权限或磁盘空间不足。

操作：确保导出或导入文件位置存在，正确安装，权限正确以及磁盘空间足够。如果文件位置是ASM磁盘组，请检查ASM警报日志文件中的相关条目以获取更多详细信息。

PROC-00060：Oracle Cluster Registry备份文件' string '已损坏。

原因： Oracle Cluster Registry备份文件未通过备份后损坏检查。Oracle Cluster Registry损坏或备份引入了错误。

操作：通过执行'ocrconfig -manualbackup'命令再次尝试Oracle Cluster Registry备份。如果问题仍然存在，请与Oracle支持服务联系。
