7使用SPA快速检查

Oracle企业管理器云控制（云控制）包括SQL性能分析器快速检查（SPA快速检查）功能。在某些Cloud Control数据库管理页面上，SPA快速检查可以在您进行更改之前验证系统更改对数据库工作负载的影响。

您可以使用SPA快速检查来验证对以下更改对数据库工作负载的影响：

  • 更改初始化参数的值

  • 收集待处理的优化程序统计信息

  • 实现关键SQL配置文件

注意：

从Cloud Control Release 12.1.0.4 Bundle Patch 8及更高版本开始，可以使用SPA快速检查。

任何运行Oracle数据库10g第2版（10.2）及更高版本的数据库都支持SPA快速检查。但是，Oracle Database 10g第2版（10.2）并未支持所有SPA快速检查功能。

例如，从Oracle Database 11g第1版（11.1）开始，可以使用优化程序挂起的统计信息和自动SQL调优顾问功能，因此只有运行Oracle Database 11g第1版（11.1）和数据库的数据库才支持这些功能的SPA快速检查工作流。后来。

本章介绍如何使用SPA快速检查，并包含以下主题：

  • “ 关于配置SPA快速检查 ”

  • “ 指定SPA快速检查的默认值 ”

  • “ 验证初始化参数更改的影响 ”

  • “ 验证待定优化程序统计信息的影响 ”

  • “ 验证实现关键SQL配置文件的影响 ”

  • “ 验证自动SQL调优顾问的统计结果 ”

7.1关于配置SPA快速检查

在使用SPA快速检查来验证初始化参数更改或收集待处理的优化程序统计信息的影响之前，必须指定SPA快速检查的默认设置。

您将为SPA快速检查指定默认SQL调整集以用作其中一个设置，并且此SQL调优集应包括您尝试调整的数据库应用程序中使用的SQL语句。

注意：

在使用SPA快速检查验证实施一个或多个关键SQL配置文件的影响之前，无需为SPA快速检查设置默认值。

7.2指定SPA快速检查的默认值

您可以在Cloud Control中的SQL Performance Analyzer设置页面上指定SPA快速检查的默认设置。

要指定SPA快速检查的默认设置：
  1. 在Cloud Control的Database Home页面上，从Performance菜单中选择SQL ，然后选择SQL Performance Analyzer Setup 。如果出现“数据库登录”页面，请输入数据库的管理员权限，然后单击“ 登录” 。
    将出现“SQL性能分析器设置”页面。

    sqc_setup_page.png插图的描述
  2. 配置SPA快速检查功能的设置，该功能在某些Cloud Control数据库管理页面上可用。您指定的SQL调优集应该代表您要调整的应用程序的工作负载。
  3. 单击“ 保存”以保存指定的默认SPA快速检查设置。

7.3验证初始化参数更改的影响

在更改会话可修改的初始化参数的值之前，可以使用SPA快速检查来验证该更改对数据库工作负载的影响。会话可修改的参数是初始化参数，其值可以使用ALTER SESSION语句进行更改。

注意：

您可以使用SPA快速检查来验证运行Oracle数据库10g第2版（10.2）及更高版本的数据库中初始化参数更改的影响。

要验证初始化参数更改的影响：
  1. 在Cloud Control的“数据库主页”上，从“ 管理”菜单中选择“ 初始化参数” 。
    将出现“初始化参数”页面。
  2. 使用“初始化参数”页面上的过滤器标识要更改其值的会话可修改初始化参数，然后单击“ 转到”以在页面底部的表中显示该参数。优化器类别中的大多数参数都是会话可修改的。
  3. 在表格中，将参数的当前值更改为您想要使用SPA快速检查验证其影响的新值。
  4. 单击“ 使用SPA验证” 。

    sqc_init_param.gif插图的描述
    信息消息显示在页面顶部，并表示已提交用于验证初始化参数更改影响的SPA任务。
  5. 单击“信息”消息中SPA任务的链接。
    将出现SQL Performance Analyzer主页。
  6. 在页面底部的“SQL性能分析器任务”部分中，选择初始化参数作业的任务，然后单击“ 查看最新报告” 。
    将出现“SQL性能分析器任务报告”页面。
  7. 查看页面底部的表格，了解更改初始化参数值的结果将取决于工作负载中最具影响力的SQL语句。

7.4验证待定优化器统计信息的影响

在收集待处理的优化程序统计信息之前，您可以使用SPA快速检查验证收集这些统计信息对数据库工作负载的影响。

注意：

您可以使用SPA快速检查来验证在运行Oracle Database 11g第1版（11.1）及更高版本的数据库中收集挂起的优化程序统计信息的影响。

要验证收集待处理优化程序统计信息的影响：
  1. 在Cloud Control的Database Home页面上，从Performance菜单中选择SQL ，然后选择Optimizer Statistics 。
    将出现Optimizer Statistics控制台页面。
  2. 在“操作”部分中，单击“ 收集” 。
    将出现“收集优化统计信息”向导。
  3. 在“收集优化程序统计信息：范围”页面底部的“使用SQL性能分析器验证”部分中，启用“ 发布之前验证统计信息对SQL性能的影响（推荐）”选项。在此过程中，数据库全局统计信息收集选项PUBLISH将暂时设置为FALSE。然后单击下一步 。

    sqc_gather_stats.gif插图的描述
  4. 继续完成向导，在Gather Optimizer Statistics：Scope页面上，单击Submit 。
    除了收集挂起的统计信息之外，还会启动一个创建SQL Performance Analyzer任务的作业，该任务验证收集数据库的优化程序统计信息的影响。
  5. 作业启动后，“管理优化程序统计信息”页面上会显示“确认”消息，指出已成功提交“收集优化程序统计信息”作业。单击该消息中的链接。
    将出现SQL Performance Analyzer主页。
  6. 在页面底部的SQL Performance Analyzer任务表中，确保已完成统计信息收集作业。这项工作可能需要几分钟才能完成。然后选择“收集优化程序统计信息”作业的行，并单击“ 查看最新报告” 。
    将出现“SQL性能分析器任务报告”页面。
  7. 查看页面底部的表，以查看将待处理的优化程序统计信息发布到工作负载中影响最大的SQL语句的结果。

7.5验证实现关键SQL配置文件的影响

在为SQL语句实现关键SQL配置文件之前，可以使用SPA快速检查验证使用这些配置文件的影响。您可以在“自动SQL调整结果摘要”页面上验证关键SQL配置文件的影响。关键SQL配置文件经过验证，性能至少提高3倍，如果为自动SQL调优顾问启用了自动实现，则会自动实现这些配置文件。

注意：

您可以使用SPA快速检查来验证在运行Oracle Database 11g第1版（11.1）及更高版本的数据库中实施关键SQL配置文件的影响。

要验证关键SQL配置文件的影响：
  1. 在Cloud Control的“数据库主页”上，从“ 性能”菜单中选择“ 顾问主页” 。
    将显示Advisor Central页面。
  2. 在“顾问”部分中，单击“ SQL顾问” 。
    将出现“SQL Advisors”页面。
  3. 在“SQL Tuning Advisor”部分中，单击“ 自动SQL调整结果” 。
    将出现“自动SQL调整结果摘要”页面。
  4. “任务状态”部分中的“ 密钥SQL配置文件”字段列出了当前自动SQL调整任务的关键SQL配置文件的数量。如果字段中出现值0 ，则不使用（或验证）关键SQL配置文件。如果“ 密钥SQL配置文件”字段中显示大于0的值，请单击该值以验证使用关键SQL配置文件或配置文件的影响。
    自动SQL调整结果详细信息：显示带有密钥SQL配置文件的SQL页面。
  5. 关键SQL配置文件显示在“建议”部分中。单击“使用SPA验证所有配置文件” 。

    sqc_key_sql_profiles.gif插图的描述
    页面顶部显示一个确认语句，指示已提交用于验证SQL配置文件的SPA任务。
  6. 单击Confirmation语句中SPA任务的链接。
    此时将显示SQL Performance Analyzer主页，并且用于验证关键SQL配置文件的SPA任务将显示在页面底部的SQL Performance Analyzer Tasks表中。
  7. 选择任务，然后单击“ 查看最新报告” 。
    将出现“SQL性能分析器任务报告”页面。
  8. 查看页面底部的表格，了解在工作负载中影响最大的SQL语句的“自动SQL调优结果摘要”页面上实施的关键SQL配置文件的实现结果。

7.6验证自动SQL Tuning Advisor的统计结果

您可以使用SPA快速检查验证自动SQL调优顾问的统计结果的影响。

注意：

您可以使用SPA快速检查来验证在运行Oracle Database 11g第1版（11.1）及更高版本的数据库中验证自动SQL调优顾问的统计结果的影响。

验证Automatic SQL Tuning Advisor的统计结果的影响：
  1. 在Cloud Control的“数据库主页”上，从“ 性能”菜单中选择“ 顾问主页” 。
    将显示Advisor Central页面。
  2. 在“顾问”部分中，单击“ SQL顾问” 。
    将出现“SQL Advisors”页面。
  3. 在“SQL Tuning Advisor”部分中，单击“ 自动SQL调整结果” 。
    将出现“自动SQL调整结果摘要”页面。
  4. 如果有任何统计结果可用，它们将显示在页面底部附近的“统计查找摘要”部分中。要验证统计结果在用户模式中的影响，请单击“ 使用SPA验证” 。
    确认声明显示在页面顶部，表示已提交用于验证统计结果的SPA任务。
  5. 单击Confirmation语句中SPA任务的链接。
    此时将显示SQL Performance Analyzer主页，并且用于验证统计结果的SPA任务将显示在页面底部的SQL Performance Analyzer Tasks表中。
  6. 之后在任务中的所有步骤都成功完成，并完成了该表的上次运行状态列时，选择任务，然后单击查看最新报告 。完成任务的所有步骤可能需要几分钟。
    将出现“SQL性能分析器任务报告”页面。
  7. 查看页面底部的表，了解在工作负载中影响最大的SQL语句的“自动SQL调优结果摘要”页面上实现统计信息的结果。
