创建可插入的数据库

目的

使用CREATE PLUGGABLE DATABASE语句创建可插拔数据库（PDB）。

此语句使您可以执行以下任务：

  • 使用种子作为模板创建PDB

    使用create_pdb_from_seed子句通过将多租户容器数据库（CDB）中的种子用作模板来创建PDB。与种子关联的文件将复制到新位置，然后复制的文件将与新PDB关联。

  • 通过克隆现有PDB或非CDB来创建PDB

    使用create_pdb_clone子句通过复制现有PDB或非CDB然后将副本插入CDB来创建PDB。与现有PDB或非CDB关联的文件将复制到新位置，复制的文件将与新PDB关联。

  • 通过将未插入的PDB或非CDB插入CDB来创建PDB

    使用create_pdb_from_xml子句使用XML元数据文件将未插入的PDB或非CDB插入CDB。

  • 通过引用另一个PDB来创建代理PDB。代理PDB提供对引用的PDB的完全功能访问。

    使用create_pdb_clone子句并指定AS PROXY FROM以创建代理PDB。

  • 创建应用程序容器，应用程序种子或应用程序PDB

    使用create_pdb_from_seed ， create_pdb_clone或create_pdb_from_xml clause 。要创建应用程序容器，必须指定AS APPLICATION CONTAINER子句。要创建应用程序种子，必须指定AS SEED子句。

注意：

在CDB中建立新的PDB时，新PDB提供的服务名称可能会与现有服务名称冲突。可以发生冲突的命名空间是允许访问CDB的侦听器的命名空间。在该命名空间内，非CDB的默认服务，CDB的默认服务，PDB的默认服务和用户定义的服务的名称之间可能存在冲突。例如，如果同一计算机系统上的两个或多个CDB使用相同的侦听器，并且新建立的PDB具有与这些CDB中的另一个PDB相同的服务名称，则发生冲突。

创建PDB时，可以为任何潜在的冲突服务名称指定新名称。请参阅service_name_convert子句。如果在创建PDB后发现服务名称冲突，则不得尝试操作导致与现有服务名称冲突的PDB。如果冲突名称是PDB默认服务的名称，则必须重命名PDB。如果冲突名称是PDB中用户创建的服务的名称，则必须删除该服务并使用具有相同目的和属性的非冲突名称创建一个名称。

也可以看看：

  • 有关多租户架构和概念的更多信息，请参阅“Oracle多租户管理员指南” 。

  • ALTER PLUGGABLE DATABASE和DROP PLUGGABLE DATABASE有关修改和删除PDB的信息

先决条件

您必须连接到CDB。 CDB必须处于打开状态并处于READ WRITE模式。

要创建PDB或应用程序容器，当前容器必须是根，并且您必须具有通常授予的CREATE PLUGGABLE DATABASE系统特权。

要创建应用程序种子或应用程序PDB，当前容器必须是应用程序根目录，应用程序容器必须处于打开状态并处于READ WRITE模式，并且您必须具有CREATE PLUGGABLE DATABASE系统特权，可以是通常授予的，也可以是本地授予的应用容器。

要指定create_pdb_clone子句：

  • 如果src_pdb_name引用同一CDB中的PDB，则必须在将在其中创建新PDB的CDB的根目录中以及在克隆的PDB中具有CREATE PLUGGABLE DATABASE系统特权。

  • 如果src_pdb_name引用远程数据库或非CDB中的PDB，则必须在CDB的根目录中具有CREATE PLUGGABLE DATABASE系统特权，其中将创建新的PDB，并且远程用户必须具有CREATE PLUGGABLE DATABASE src_pdb_name引用的PDB或非CDB中的系统特权。

有关PDB创建的先决条件的详细信息，请参阅“ Oracle Multitenant管理员指南” 。

语义

pdb_name

指定要创建的PDB的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。PDB名称的第一个字符必须是字母字符。其余字符可以是字母数字或下划线字符（ _ ）。

PDB名称在CDB中必须是唯一的，并且在通过特定侦听器到达其实例的所有CDB的范围内必须是唯一的。

作为应用容器

指定此子句以创建应用程序容器。

也可以看看：

“Oracle数据库管理员指南” ，了解创建应用程序容器的完整步骤

using_snapshot_clause

指定此子句以从现有PDB快照创建PDB，该快照可通过其名称，SCN或时间戳来标识。

如果另外指定SNAPSHOT COPY，则新PDB将取决于是否存在指定的PDB快照。这将影响您放弃或清除新PDB的能力。

作为种子

指定此子句以创建应用程序种子。数据库为种子分配表单application_container_name $SEED的名称。

应用程序容器最多只能有一个应用程序种子。应用程序种子是可选的，但是，如果它存在，您可以使用它快速创建符合应用程序容器要求的应用程序PDB。应用程序种子可以即时配置从中创建的应用程序PDB。

也可以看看：

有关创建应用程序种子的完整步骤的Oracle数据库管理员指南

create_pdb_from_seed

此子句使您可以通过将CDB中的种子用作模板来创建PDB。

也可以看看：

“Oracle数据库管理员指南” ，了解使用种子创建PDB的完整步骤

ADMIN USER

使用此子句可以创建一个管理用户，该用户可以被授予在PDB上执行管理任务所需的权限。对于admin_user_name ，请指定要创建的用户的名称。使用IDENTIFIED BY子句指定admin_user_name的密码。Oracle数据库在PDB中创建本地用户，并向该用户授予PDB_DBA本地角色。

pdb_dba_roles

此子句允许您向PDB_DBA角色授予一个或多个角色。使用此子句授予具有PDB管理用户所需权限的角色。对于role ，请指定预定义角色。有关预定义角色的列表，请参阅“ Oracle数据库安全指南” 。

您还可以使用GRANT语句在创建PDB PDB_DBA角色授予PDB_DBA角色。在您为PDB_DBA角色授予适当的权限PDB_DBA ， SYS和SYSTEM用户可以在PDB上执行管理任务。

parallel_pdb_creation_clause

该子句指示CDB使用并行执行服务器将新PDB的数据文件复制到新位置。这可能导致更快地创建PDB。

平行

如果指定PARALLEL ，则CDB会自动选择要使用的并行执行服务器的数量。如果COMPATIBLE初始化参数设置为12则这是默认值。 2或更高。

PARALLEL 整数

使用integer指定要使用的并行执行服务器的数量。CDB可以忽略此设置，具体取决于当前数据库负载和可用并行执行服务器的数量。如果指定值0或1，则CDB不会并行创建PDB。这可能导致PDB创建时间更长。

default_tablespace

如果指定此子句，则Oracle数据库会创建一个smallfile表空间并将其设置为PDB的默认永久表空间。 Oracle数据库将默认表空间分配给未指定其他永久表空间的任何非SYSTEM用户。default_tablespace子句与CREATE DATABASE语句具有相同的语义。有关完整信息，请参阅CREATE DATABASE文档中的default_tablespace 。

pdb_storage_clause

使用此子句指定PDB的存储限制。

  • 使用MAXSIZE将PDB中所有表空间可以使用的存储量限制为使用size_clause指定的值。此限制包括属于PDB的表空间的数据文件和临时文件的大小。指定MAXSIZE UNLIMITED以强制执行无限制。

  • 使用MAX_AUDIT_SIZE限制可以通过统一的审计OS溢出（可以使用的存储量.bin格式）文件，在PDB与指定的值size_clause 。指定MAX_AUDIT_SIZE UNLIMITED以强制执行无限制。

  • 使用MAX_DIAG_SIZE限制自动诊断信息库（ADR）中的诊断（跟踪文件和事件转储）的存储量，PDB可以将其用于使用size_clause指定的值。指定MAX_DIAG_SIZE UNLIMITED以强制执行无限制。

如果省略此子句，或指定STORAGE UNLIMITED ，则PDB没有存储限制。这相当于指定STORAGE (MAXSIZE UNLIMITED MAX_AUDIT_SIZE UNLIMITED MAX_DIAG_SIZE UNLIMITED ）。

file_name_convert

使用此子句可确定数据库如何为PDB生成文件名（例如数据文件和钱包文件）。

  • 对于filename_pattern ，指定在与种子关联的文件名称中找到的字符串（使用种子创建PDB时），与源PDB关联（克隆PDB时），或在XML文件中列出（将PDB插入时）一个CDB）。

  • 对于replacement_filename_pattern ，请指定替换字符串。

在生成与新PDB关联的文件的名称时，Oracle数据库将使用replacement_filename_pattern替换filename_pattern 。

文件名模式与Oracle Managed Files管理的文件或目录不匹配。

您可以指定FILE_NAME_CONVERT = NONE ，这与省略此子句相同。如果省略此子句，则数据库首先尝试使用Oracle Managed Files生成文件名。如果您未使用Oracle Managed Files，则数据库使用PDB_FILE_NAME_CONVERT初始化参数来生成文件名。如果未设置此参数，则会发生错误。

service_name_convert

使用此子句根据源PDB的服务名称重命名新PDB的用户定义服务。当新PDB的服务名称与CDB中的现有服务名称冲突时，可能会导致插件违规。此子句使您可以避免这些违规。

  • 对于service_name ，指定在PDB种子中找到的服务的名称（在使用应用程序种子在应用程序容器中创建PDB时）或在源PDB中（在克隆PDB或将PDB插入CDB时）。

  • 对于replacement_service_name ，请指定服务的替换名称。

Oracle数据库将使用正在创建的PDB中的服务的替换服务名称。

您可以指定SERVICE_NAME_CONVERT = NONE ，这与省略此子句相同。

service_name_convert的 限制

service_name_convert子句受以下限制：

  • 您无法更改PDB的默认服务名称。默认服务与PDB具有相同的名称。

  • 使用create_pdb_from_seed子句从CDB种子创建PDB时，不能指定此子句，因为CDB种子没有用户定义的服务。但是，在使用create_pdb_from_seed子句从应用程序种子创建应用程序PDB时，可以指定此子句。

path_prefix_clause

使用此子句可确保与PDB关联的目录对象的文件路径仅限于指定目录或其子目录。此子句还确保与PDB关联的以下文件仅限于指定的目录：PDB的Oracle XML存储库，使用CREATE PFILE语句创建的文件以及Oracle钱包的导出目录。创建PDB后，无法修改此子句的设置。此子句不影响Oracle Managed Files创建的文件。

  • 对于path_name ，请指定操作系统目录的绝对路径名。单引号是必需的，结果是路径名称区分大小写。Oracle数据库使用path_name作为与PDB关联的所有文件路径的前缀。

    请务必指定path_name以便在将相对路径附加到路径名时正确形成结果路径名。例如，在UNIX系统上，请确保使用正斜杠（ / ）结束path_name ，例如：

    PATH_PREFIX ='/ disk1 / oracle / dba / salespdb /'

  • 对于directory_object_name ，请指定CDB根目录中存在的目录对象的名称（ CDB$ROOT ）。目录对象指向用于PATH_PREFIX的绝对路径。

  • 如果指定PATH_PREFIX = NONE ，则与PDB关联的目录对象的相对路径将被视为绝对路径，并且不限于特定目录。

省略path_prefix_clause等同于指定PATH_PREFIX = NONE 。

在为PDB指定path_prefix_clause之后，现有目录对象可能无法按预期工作，因为PATH_PREFIX字符串始终作为前缀添加到PDB中的所有本地目录对象。 path_prefix_clause仅适用于用户创建的目录对象。它不适用于Oracle提供的目录对象。

tempfile_reuse_clause

创建PDB时，Oracle数据库会将临时文件与新PDB关联。根据您创建PDB的方式，临时文件可能已经存在并且可能以前曾使用过。

指定TEMPFILE REUSE以指示数据库格式化TEMPFILE用与新PDB关联的临时文件（如果已存在）。如果指定此子句且临时文件不存在，则数据库将创建临时文件。

如果未指定TEMPFILE REUSE且与新PDB关联的临时文件已存在，则数据库将返回错误并且不会创建PDB。

user_tablespaces_clause

此子句允许您指定要在新PDB中使用的表空间。 SYSTEM ， SYSAUX和TEMP表空间在所有PDB中都可用，并且不能在本节中指定。

您可以使用此子句将多个模式的数据分成不同的PDB。例如，当您将非CDB移动到PDB，并且非CDB具有多个模式，每个模式支持不同的应用程序时，您可以使用此子句将属于每个模式的数据分隔为单独的PDB，假设每个模式在非CDB中使用单独的表空间。

  • 指定tablespace以使表空间在新PDB中可用。您可以在逗号分隔列表中指定多个表空间。

  • 指定ALL以使新PDB中的所有表空间可用。这是默认值。

  • 指定ALL EXCEPT以使新PDB中的所有表空间可用，但指定的表空间除外。

  • 指定NONE以仅在新PDB中提供SYSTEM ， SYSAUX和TEMP表空间。

当CDB的兼容级别为12.2或更高时，此子句排除的表空间在新PDB中脱机创建，并且它们没有与之关联的数据文件。当CDB的兼容级别低于12.2 ，此子句排除的表空间在新PDB中处于脱机状态，属于这些表空间的所有数据文件都是未命名和脱机的。

{SNAPSHOT COPY |没有数据 }

这些子句仅在使用create_pdb_clone子句克隆PDB时适用。克隆非CDB时不适用。默认情况下，数据库根据为克隆PDB指定的设置创建新PDB中可用的每个表空间。这些子句允许您覆盖这些设置，如下所示：

  • SNAPSHOT COPY - 使用存储快照克隆表空间。

  • NO DATA - 克隆表空间的数据模型定义，但不克隆表空间的数据。

{COPY |移动| NOCOPY}

当您使用create_pdb_from_xml子句插入PDB时，这些子句适用。默认情况下，数据库根据为插入PDB指定的设置创建在新PDB中可用的每个表空间。这些子句允许您覆盖这些设置，如下所示：

  • COPY - 将表空间文件复制到新位置。

  • MOVE - 将表空间文件移动到新位置。

  • NOCOPY - 不要将表空间文件复制或移动到新位置。

standbys_clause

使用此子句指定新PDB是否包含在一个或多个备用CDB中。如果在备用CDB中包含PDB，则在备用恢复期间，备用CDB将搜索PDB的数据文件。如果找不到数据文件，则备用恢复将停止，您必须将数据文件复制到正确的位置，然后才能重新启动恢复。

  • 指定cdb_name以在指定的备用CDB中包含新PDB。您可以在逗号分隔列表中指定多个备用CDB名称。

  • 指定ALL以在所有备用CDB中包含新PDB。这是默认值。

  • 指定ALL EXCEPT以在所有备用CDB中包括新PDB，但指定的备用CDB除外。

  • 指定NONE以从所有备用CDB中排除新PDB。当从所有备用CDB中排除PDB时，PDB的数据文件未命名，并在所有备用CDB上标记为脱机。如果在备用数据库上找不到PDB的数据文件，则备用恢复不会停止。如果在创建PDB后实例化新的备用CDB，则必须在新备用CDB上明确禁用PDB以进行恢复。

通过将数据文件复制到正确的位置，使PDB联机并将其标记为已启用以进行恢复，可以在备用CDB上排除备用CDB上的PDB后启用PDB。

logging_clause

使用此子句指定在PDB中创建的表空间的缺省日志记录属性。 logging属性控制是否在重做日志文件（ LOGGING ）中记录某些DML操作（ NOLOGGING ）。默认为LOGGING 。

创建表空间时，可以通过指定CREATE TABLESPACE语句的logging_clause来覆盖缺省日志记录属性。

有关此子句的完整说明，请参阅logging_clause 。

create_file_dest_clause

默认情况下，新创建的PDB从根目录继承其Oracle Managed Files设置。如果root使用Oracle Managed Files，则PDB也使用Oracle Managed Files。PDB与root共享Oracle托管文件的相同基本文件系统目录，并且具有以PDB的GUID命名的自己的子目录。如果根不使用Oracle Managed Files，则PDB也不使用Oracle Managed Files。

此子句允许您覆盖默认行为。您可以为PDB启用或禁用Oracle Managed Files，并为PDB的文件指定不同的基本文件系统目录或Oracle ASM磁盘组。

  • 指定NONE以禁用PDB的Oracle Managed Files。

  • 指定directory_path_name或diskgroup_name以为PDB启用Oracle Managed Files。

    指定directory_path_name以指定PDB文件的基本文件系统目录。指定操作系统目录的完整路径名。该目录必须存在且Oracle进程必须对该目录具有适当的权限。单引号是必需的，结果是路径名称区分大小写。

    指定diskgroup_name以指定PDB文件的默认Oracle ASM磁盘组。

如果指定的值不是NONE ，则数据库会在PDB中使用SCOPE=SPFILE隐式设置DB_CREATE_FILE_DEST初始化参数。

主持人和港口

仅当您要创建计划从代理PDB引用的PDB时，这些子句才有用。这种类型的PDB称为参考PDB。

创建引用的PDB时：

create_pdb_clone

此子句使您可以通过将源克隆到目标PDB来创建新的PDB。源可以是本地CDB中的PDB，远程CDB中的PDB或非CDB。目标PDB是源的克隆。

如果源是本地CDB中的PDB，则可以插入或拔出源PDB。如果源是远程CDB中的PDB，则必须插入源PDB。

如果源是远程CDB中的非CDB或PDB，则包含目标PDB的源和CDB必须满足以下要求：

  • 它们必须具有相同的endian格式。

  • 它们必须具有兼容的字符集和国家字符集，这意味着：

    • 源字符集中的每个字符都可在本地CDB字符集中使用。

    • 源字符集中的每个字符在本地CDB字符集中具有相同的代码点值。

  • 它们必须安装相同的数据库选项集。

PDB中使用源非CDB或PDB的默认临时表空间的用户使用新PDB的默认临时表空间。在非CDB或PDB中使用非默认临时表空间的用户将继续在新PDB中使用相同的本地临时表空间。

您可以使用相同的命令克隆联合PDB或隔离的PDB。唯一的区别是您必须提供的密钥库密码用于不同的密钥库。

快照复制

只有在克隆PDB时才能指定SNAPSHOT COPY 。克隆非CDB时不支持此子句。源PDB可以位于本地CDB或远程CDB中。 SNAPSHOT COPY子句指示数据库使用存储快照克隆源PDB。这减少了创建克隆所需的时间，因为数据库不需要制作源数据文件的完整副本。

当您使用SNAPSHOT COPY子句创建源PDB的克隆并且CLONEDB初始化参数设置为FALSE ，源PDB文件的基础文件系统必须支持存储快照。此类文件系统包括Oracle自动存储管理群集文件系统（Oracle ACFS）和直接NFS客户端存储。

当您使用SNAPSHOT COPY子句创建源PDB的克隆并且CLONEDB初始化参数设置为TRUE ，源PDB的文件的基础文件系统可以是任何本地文件系统，网络文件系统（NFS）或群集已启用Direct NFS的文件系统。但是，只要存在任何克隆，源PDB就必须保持打开的只读模式。

Direct NFS Client使Oracle数据库可以直接访问网络连接存储（NAS）设备，而不是使用操作系统内核NFS客户端。如果PDB文件存储在Direct NFS客户端存储上，则必须满足以下附加要求：

  • 源PDB文件必须位于NFS卷上。

  • 存储凭据必须存储在透明数据加密密钥库中。

  • 存储用户必须具有在承载源PDB文件的卷上创建和销毁快照所需的权限。

  • 必须使用ADMINISTER KEY MANAGEMENT ADD SECRET SQL语句将凭据存储在密钥库中。

当您使用SNAPSHOT COPY子句创建源PDB的克隆时，只要存在任何克隆，以下限制将适用于源PDB：

  • 它不能被拔掉。

  • 它不能丢弃。

使用SNAPSHOT COPY子句创建的PDB克隆无法拔出。他们只能被丢弃。尝试拔出使用SNAPSHOT COPY子句创建的克隆会导致错误。

对于在Oracle Real Application Clusters（Oracle RAC）环境中使用SNAPSHOT COPY子句创建的PDB，必须装入必须访问PDB文件的每个节点。对于在Linux或UNIX平台上运行的Oracle RAC数据库，必须装入基础NFS卷。如果Oracle RAC数据库在Windows平台上运行并使用Direct NFS进行共享存储，则必须使用创建的卷export和mount条目更新所有节点上的oranfstab文件。

使用新的PDB GUID命名和标记存储克隆。您可以查询DBA_PDB_HISTORY视图的CLONETAG列以查看存储克隆的克隆标记。

pdb_refresh_mode_clause

REFRESH MODE子句仅在克隆PDB时适用。源PDB必须位于远程CDB中，即必须使用FROM src_pdb_name @ dblink子句指定源PDB。

此子句允许您指定PDB的刷新模式。您可以使用此子句创建可刷新的PDB 。源PDB中的更改可以手动或自动传播到可刷新的PDB。此操作称为刷新。您可以指定以下刷新模式：

  • MANUAL - 此模式允许您通过发出ALTER PLUGGABLE DATABASE REFRESH语句随时手动刷新可刷新的PDB。

  • EVERY refresh_interval MINUTES或HOURS - 此模式指示数据库在选定时间单位，分钟或小时的每个refresh_interval刷新可刷新的PDB。如果选择MINUTES ，则refresh_interval必须小于3000。如果选择HOURS ，则refresh_interval必须小于2000。此模式还允许您通过发出ALTER PLUGGABLE DATABASE REFRESH语句随时手动刷新PDB。

  • NONE - 如果指定此模式，则克隆PDB不是可刷新的PDB。数据库无法自动刷新PDB，您无法手动刷新PDB。如果指定此模式，则以后无法将PDB更改为可刷新的PDB。这是默认值。

只能在READ ONLY模式下打开可刷新的PDB。必须关闭可刷新的PDB才能进行刷新。如果在尝试执行手动刷新时未关闭，则会发生错误。如果在数据库尝试自动刷新时未关闭，则刷新将推迟到下一次计划刷新。

RELOCATE

使用此子句将PDB从一个CDB重定位到另一个CDB。数据库首先将源PDB克隆到目标PDB，然后删除源PDB。数据库还将与PDB关联的文件移动到新位置。此操作是以最短的停机时间重新定位PDB的最快方法。PDB的停机时间大约是将PDB文件从旧位置复制到新位置所需的时间。源PDB可以在READ WRITE模式下打开，并在重定位操作期间完全正常工作。

pdb_snapshot_clause

如果您希望能够创建PDB快照，请指定此子句。

  • NONE是默认值。这意味着不能创建PDB的快照。

  • MANUAL表示只能手动创建PDB快照。

  • 如果指定了快照间隔，则将以指定的时间间隔自动创建PDB快照。此外，用户还可以手动创建PDB快照

  • 如果以分钟表示，则snapshot_interval必须小于3000。

  • 如果以小时表示，则snapshot_interval必须小于2000。

create_pdb_decrypt_from_xml

您必须具有SYSKM权限才能执行此命令。

对于联合模式下的PDB，以下限制适用：

  • 如果使用受TDE保护的数据库，则必须指定该子句。否则它是可选的。

  • 您无需为隔离的PDB指定子句。

  • 钱包必须在ROOT开放。

  • 在所有情况下都会复制钱包文件： NOCOPY ， COPY和MOVE 。

从XML元数据文件中插入PDB：示例

CREATE PLUGGABLE DATABASE CDB1_PDB2使用'/tmp/cdb1_pdb2.xml'NOCOPY KEYSTORE由keystore_password识别DECRYPT使用transport_secret

创建可插入的数据库CDB1_PDB1_1_C使用'/tmp/cdb1_pdb3.pdb'解密使用transport_secret

对于处于隔离模式的PDB，您无需指定DECRYPT USING transport_secret 。这不是必需的，因为在从XML文件创建未插入的PDB期间复制了钱包文件。如果要从扩展名为.pdb的存档文件创建PDB，则可以在压缩存档中找到PDB的钱包文件。

例子

使用种子创建PDB：示例

以下语句通过将CDB中的种子用作模板来创建PDB salespdb 。创建管理用户salesadm并授予dba角色。分配给没有分配永久表空间的任何非SYSTEM用户的默认表空间是sales 。将使用/disk1/oracle/dbs/pdbseed/替换种子中文件名中的/disk1/oracle/dbs/salespdb/来构造新PDB的文件名。属于sales所有表空间不得超过2G。与salespdb关联的所有目录对象路径的位置仅限于目录/disk1/oracle/dbs/salespdb/ 。 

CREATE PLUGGABLE DATABASE salespdb ADMIN USER salesadm IDENTIFIED BY password ROLES =（dba）DEFAULT TABLESPACE sales DATAFILE'/disk1/oracle/dbs/salespdb/sales01.dbf'SIZE 250M AUTOEXTEND ON FILE_NAME_CONVERT =（'/ disk1 / oracle / dbs / pdbseed / '，'/ disk1 / oracle / dbs / salespdb /'）STORAGE（MAXSIZE 2G）PATH_PREFIX ='/ disk1 / oracle / dbs / salespdb /';

从现有PDB克隆PDB：示例

下面的语句创建一个PDB newpdb克隆PDB salespdb 。PDB newpdb和salespdb在同一个CDB中。由于未明确指定存储限制，因此对newpdb的存储量没有限制。这些文件从/disk1/oracle/dbs/salespdb/复制到/disk1/oracle/dbs/newpdb/ 。与newpdb关联的所有目录对象路径的位置仅限于目录/disk1/oracle/dbs/newpdb/ 。 

CREATE PLUGGABLE DATABASE newpdb FROM salespdb FILE_NAME_CONVERT =（'/ disk1 / oracle / dbs / salespdb /'，'/ disk1 / oracle / dbs / newpdb /'）PATH_PREFIX ='/ disk1 / oracle / dbs / newpdb';

将PDB插入CDB：示例

以下语句将之前未插入的PDB salespdb插入CDB。有关描述salespdb的元数据的详细信息存储在XML文件/disk1/usr/salespdb.xml 。XML文件无法准确描述文件的当前位置。因此， SOURCE_FILE_NAME_CONVERT子句用于指示文件位于/disk2/oracle/dbs/salespdb/ ，而不是/disk1/oracle/dbs/salespdb/ 。NOCOPY子句指示文件已在正确的位置。属于sales所有表空间不得超过2G。目标位置中存在与XML文件中指定的临时文件同名的文件。因此， TEMPFILE REUSE子句是必需的。 

CREATE PLUGGABLE DATABASE salespdb使用'/disk1/usr/salespdb.xml'SOURCE_FILE_NAME_CONVERT =（'/ disk1 / oracle / dbs / salespdb /'，'/ disk2 / oracle / dbs / salespdb /'）NOCOPY STORAGE（MAXSIZE 2G）TEMPFILE REUSE ;
