F设置存档跟踪

Oracle数据库使用LOG_ARCHIVE_TRACE初始化参数来启用和控制日志归档和重做传输活动的综合跟踪信息的生成。

当LOG_ARCHIVE_TRACE设置为非零值时输出的附加跟踪可以出现在跟踪文件中，用于归档过程， RFS过程， LGWR过程， SYNC过程， ASYNC过程，前台进程， MRP过程，恢复过程，日志应用过程，启动进程，关闭进程和使用重做传输服务的其他进程。跟踪信息将写入自动诊断信息库。

也可以看看：

  • 有关LOG_ARCHIVE_TRACE初始化参数及其有效值的完整说明，请参见Oracle数据库参考

  • 有关自动诊断信息库的更多信息，请参见“Oracle数据库管理员指南”

F.1设置LOG_ARCHIVE_TRACE初始化参数

要启用，禁用或修改LOG_ARCHIVE_TRACE参数，请发出ALTER SYSTEM SQL语句。

LOG_ARCHIVE_TRACE参数的格式如下，其中trace_level是一个整数：

LOG_ARCHIVE_TRACE = trace_level

以下是设置LOG_ARCHIVE_TRACE参数的示例：

SQL> ALTER SYSTEM SET LOG_ARCHIVE_TRACE = 15;

您可以通过将LOG_ARCHIVE_TRACE参数的值设置为各个级别的总和来组合跟踪级别。例如，在前面的示例中，将LOG_ARCHIVE_TRACE参数设置为15可设置跟踪级别1,2,4和8。

有关有效LOG_ARCHIVE_TRACE值的完整列表和说明，请参见Oracle数据库参考 。
