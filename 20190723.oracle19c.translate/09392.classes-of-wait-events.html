C.1等待事件的类别

每个等待事件都属于一类等待事件。

以下列表描述了每个等待类。

行政的

DBA命令导致用户等待的等待（例如，索引重建）

应用

用户应用程序代码导致的等待（例如，由行级锁定或显式锁定命令导致的锁定等待）

簇

等待与Oracle Real Application Clusters资源相关的等待（例如，全局缓存资源，例如'gc cr block busy'）

承诺

此等待类仅包含一个等待事件 - 在提交后等待重做日志写入确认（即“日志文件同步”）

并发

等待内部数据库资源（例如，锁存器）

组态

数据库或实例资源配置不当导致的等待（例如，日志文件大小不足，共享池大小）

闲

等待表示会话的等待处于非活动状态，等待工作（例如，'来自客户端的SQL * Net消息'）

网络

与网络消息传递相关的等待（例如，'SQL * Net更多数据到dblink'）

其他

通常不会在系统上发生的等待（例如，'等待EMON生成'）

队列

包含表示在流水线环境中获取其他数据的延迟的事件。花在这些等待事件上的时间表明管道中的效率低下或其他问题。它会影响并行查询或DBMS_PIPE PL / SQL包等功能。

调度

资源管理器相关的等待（例如，'resmgr：变为活动'）

系统I / O.

等待后台进程I / O（例如，DBWR等待'db file parallel write'）

用户I / O.

等待用户I / O（例如'db file sequential read'）
