句法

HO [ST] [ 命令 ]

其中command表示操作系统命令。

执行一个操作系统命令不离开SQL * Plus。

不带命令输入HOST以显示操作系统提示。然后，您可以输入多个操作系统命令。有关返回SQL * Plus的信息，请参阅为您的操作系统提供的特定于平台的Oracle文档。

注意：

使用HOST命令从SQL * Plus会话输入的操作系统命令不会影响当前的SQL * Plus会话。例如，设置操作系统环境变量仅影响随后启动的SQL * Plus会话。

您可以禁用HOST。有关禁用HOST的详细信息，请参阅SQL * Plus安全性 。

用法

在某些操作系统中，您可以使用一个字符代替HOST，例如Windows中的“$”或“！“在UNIX中，或者您可能无法访问HOST命令。请参阅为您的操作系统提供的特定于平台的Oracle文档，或者向DBA咨询以获取更多信息。

在某些平台上，一个可以使用与操作系统相关的HOST返回值创建_RC替换变量。建议您不要在脚本中使用_RC替换变量，因为它不可移植。

SQL * Plus删除了发出HOST命令之前的SQLTERMINATOR（默认为分号）。解决方法是添加另一个SQLTERMINATOR。参见SET SQLT [ERMINATOR] {; | c | ON | OFF}了解更多信息。

例子

要执行UNIX操作系统命令ls * .sql，请输入

HOST ls * .sql

要执行Windows操作系统命令dir * .sql，请输入

HOST目录* .sql
