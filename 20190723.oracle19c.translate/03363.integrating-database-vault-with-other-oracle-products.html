配置企业用户授权

要配置企业用户授权，必须创建Oracle Database Vault规则集以控制用户访问。

  1. 创建允许或禁止用户访问的规则。

    按照创建规则添加到规则集中的说明创建新规则。在“创建规则”页面中，在“规则表达式”字段中输入以下PL / SQL：

    SYS_CONTEXT（'USERENV'，'AUTHENTICATED_IDENTITY'）= 'user_domain_name'

    将user_domain_name替换为域，例如：

    SYS_CONTEXT（'USERENV'，'AUTHENTICATED_IDENTITY'）='myserver.us.example.com '
  2. 将此规则添加到新规则集。

    创建规则集说明了如何创建新规则集，包括如何向其添加现有规则。

  3. 将此规则集添加到要保护的数据的域授权中。

    关于Realm Authorization解释了如何创建领域授权。在“授权规则集”列表中，选择您在步骤2中创建的规则集。之后，领域授权适用于所有用户。

将Oracle Database Vault帐户配置为企业用户帐户

您可以将现有Oracle Database Vault用户帐户配置为企业用户帐户。

  1. 以已被授予CREATE ROLE系统特权的用户身份登录数据库实例。

    例如：

    sqlplus系统输入密码： 密码
  2. 在多租户环境中，连接到相应的可插拔数据库（PDB）。

    例如：

    CONNECT SYSTEM @ hrpdb输入密码： 密码

    要查找可用的PDB，请查询DBA_PDBS数据字典视图。要检查当前PDB，请运行show con_name命令。

  3. 创建了一个全球性的角色DV_OWNER作用，并为全球角色DV_ACCTMGR作用。

    例如：

    创建角色g_dv_owner全球识别;创建角色g_dv_acctmgr全球识别;
  4. 以已被授予DV_OWNER角色的用户身份进行连接。

    例如：

    CONNECT sec_admin_owen - 或者，CONNECT sec_admin_owen @hrpdb输入密码： password
  5. 将DV_OWNER角色授予全局DV_OWNER角色。
    GRANT DV_OWNER TO g_dv_owner;
  6. 以已被授予DV_ACCTMGR角色的用户身份进行连接。

    例如：

    CONNECT dbv_acctmgr - 或者，CONNECT dbv_acctmgr @hrpdb输入密码： password
  7. 将DV_ACCTMGR角色授予全局DV_ACCTMGR角色。
    GRANT DV_ACCTMGR TO g_dv_acctmgr;
  8. 以SYS用户身份连接SYSDBA管理权限。
    将SYS作为SYSDBA连接- 或者，连接SYS @hrpdb AS SYSDBA输入密码： password
  9. 暂时授予将导入Database Vault用户的DV_ACCTMGR用户转换为OID CREATE TABLE权限和SELECT_CATALOG_ROLE角色。
    GRANT CREATE TABLE，SELECT_CATALOG_ROLE到dbv_acctmgr;
  10. 从命令行运行用户迁移实用程序（ UMU ）以将Database Vault帐户导入Oracle Internet Directory（OID）。

    以下示例将数据库保管库帐户leo_dvowner和bea_dvacctmgr导入OID。为DBADMIN设置指定了DV_ACCTMGR用户。 

    $ ORACLE_HOME / rdbms / bin / umu PHASE = ONE DBADMIN = dbv_acctmgr： password ENTADMIN = cn = jane_ent_admin，dc = example，dc = com： password USERS = LIST DBLOCATION = example.com：7777：orcl DIRLOCATION = example.com:636 USERSLIST = leo_dvowner：bea_dvacctmgr MAPSCHEMA = PRIVATE CONTEXT = CONTEXT =“c = Users，c = us”KREALM = EXAMPLE.COM $ ORACLE_HOME / rdbms / bin / umu PHASE = TWO DBADMIN = dbv_acctmgr： password ENTADMIN = cn = jane_ent_admin，dc =例如，dc = com： password DBLOCATION = example.com：7777：orcl DIRLOCATION = example.com：636

    默认情况下，错误将写入$ORACLE_HOME/network/log/umu.log文件。

  11. 从Oracle Internet Directory自助服务控制台（ http:// hostname : port /oiddas/ ），将全局DV_OWNER和DV_ACCTMGR角色（例如， g_dv_owner和g_dv_acctmgr ）授予企业用户Database Vault帐户。

    有关从全局角色创建企业角色然后将此角色授予用户的演示，请参阅“ Oracle数据库企业用户安全性管理员指南”中的创建企业用户的示例。

  12. 从SQL * Plus，作为具有SYSDBA管理权限的用户SYS ，从DV_ACCTMGR用户撤消CREATE TABLE和SELECT_CATALOG_ROLE角色。
    从dbv_acctmgr REVOKE CREATE TABLE，SELECT_CATALOG_ROLE;

Oracle Database Vault与透明数据加密的集成

透明数据加密补充了Oracle Database Vault，因为它在数据离开数据库的安全边界时提供数据保护。

使用透明数据加密，数据库管理员或数据库安全管理员可以简单地加密具有应用程序表中敏感内容的列，或加密整个应用程序表空间，而无需对应用程序进行任何修改。

如果用户通过了身份验证和授权检查，则透明数据加密会自动为用户加密和解密信息。这样，您无需更改应用程序即可实施加密。

授予透明数据加密用户相应权限后，可以照常管理透明数据加密，并与Database Vault免费使用。

Oracle Database Vault如何与Oracle Label Security集成

Oracle Database Vault-Oracle Label Security集成使您可以将OLS标签分配给Database Vault因子标识。

在Oracle Label Security中，您可以限制对数据库表或PL / SQL程序中的记录的访问。例如，Mary可能能够看到受HIGHLY SENSITIVE标签保护的数据， EMPLOYEE表上的Oracle Label Security标签包含应该仅限某些管理员访问的记录。另一个标签可以是PUBLIC，它允许更加开放地访问这些数据。

在Oracle Database Vault中，您可以为数据库会话所源自的网络创建一个名为Network的因子，具有以下标识：

  • Intranet：用于员工在公司内部网站内工作的情况。

  • 远程：用于员工在家中通过VPN连接工作时。

然后，为两者分配最大会话标签。例如：

  • 将Intranet标识分配给HIGHLY SENSITIVE Oracle Label Security标签。

  • 将远程标识分配给PUBLIC标签。

这意味着当Mary使用她的VPN连接在家工作时，她只能访问受PUBLIC身份保护的有限表数据。但是当她在办公室时，她可以访问HIGHLY SENSITIVE数据，因为她使用的是Intranet身份。教程：将Oracle Database Vault与Oracle Label Security集成提供了如何完成此类集成的示例。

在非统一审计环境中，您可以使用标签安全集成审计报告来审计与Oracle Label Security的集成。Oracle Database Vault将审计跟踪写入DVSYS.AUDIT_TRAIL$表。如果启用了统一审核，则可以创建审核策略以捕获此信息，如“ Oracle数据库安全指南”中所述 。

将Oracle Database Vault与Oracle Label Security一起使用的要求

在将Oracle Database Vault与Oracle Label Security一起使用之前，您必须满足特定的要求。

  • Oracle Label Security单独授权。确保您已购买许可证以使用它。

  • 在安装Oracle Database Vault之前，您必须已安装Oracle Label Security。

  • Oracle Label Security的安装过程会创建LBACSYS用户帐户。作为已被授予DV_ACCTMGR角色的用户，请解锁此帐户并为其授予新密码。例如：

    sqlplus bea_dvacctmgr - 或者，sqlplus bea_dvacctmgr @ hrpdb用于PDB输入密码： 密码 ALTER USER LBACSYS ACCOUNT UNLOCK IDENTIFIED BY password ;

    请遵循“ Oracle数据库安全指南”中的准则，将password替换为安全的密码。

  • 如果您计划在Oracle Enterprise Manager中使用LBACSYS用户帐户，则以具有SYSDBA管理权限的SYS用户身份登录Enterprise Manager，并为该用户授予SELECT ANY DICTIONARY和SELECT_CATALOG_ROLE系统权限。

  • 确保已定义适当的Oracle Label Security策略。有关更多信息，请参阅“ Oracle Label Security管理员指南” 。

  • 如果计划将Oracle Label Security策略与Database Vault策略集成，请确保Oracle Label Security的策略名称少于24个字符。您可以通过查询ALL_SA_POLICIES数据字典视图的POLICY_NAME列来检查Oracle Label Security策略的名称。

关于将Oracle Database Vault因素与Oracle Label Security策略配合使用

Oracle Database Vault-Oracle Label Security集成使您可以控制数据库会话的最大安全许可。

Oracle Database Vault通过合并与Oracle Label Security策略关联的Oracle Database Vault因子的标签，通过合并数据库会话中每个标签的最大允许数据来控制数据库会话的最大安全许可。

简而言之，标签充当数据库表行的访问权限的标识符。策略是与管理对表行的访问的标签，规则和授权相关联的名称。

配置使用Oracle Label Security策略的因素

您可以定义有助于Oracle Label Security策略的最大允许数据标签的因子。

  1. 以已授予DV_OWNER或DV_ADMIN角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 使用户LBACSYS帐户成为域的所有者，该域包含已应用标签安全策略的架构。

    这使得LBACSYS帐户可以访问领域中的所有受保护数据，以便它可以正确地对数据进行分类。

    使用Oracle Universal Installer自定义安装选项在Oracle Label Security中创建LBACSYS帐户。在创建用于Oracle Database Vault的Oracle Label Security策略之前，必须使LBACSYS成为您计划使用的LBACSYS的所有者。有关更多信息，请参阅关于领域授权 。

  3. 将架构所有者（已应用标签安全策略）授权为领域参与者或领域所有者。
  4. 在“管理”页面的“数据库Vault组件”下，单击“ OLS集成” 。


    插图描述ols_policies_integration122.png
  5. 在标签安全策略集成页面中：

    • 要使用Database Vault注册新的标签安全策略，请单击“ 创建” 。

    • 要编辑已向Database Vault注册的现有标签安全策略，请从列表中选择该策略，然后单击“ 编辑” 。

  6. 输入以下设置：

    • 标签安全策略：从列表中，选择要使用的Oracle Label Security策略。

    • 算法：可选择更改标签合并算法，以适用于Oracle Label Security合并两个标签的情况。在大多数情况下，您可能希望选择LII - 最小级别/交叉点/交叉点 。此设置是Oracle Label Security管理员在要合并两个标签时使用的最常用方法。当您的应用程序必须确定组合具有不同标签的两个数据集时所需的结果标签时，此设置可提供最佳灵活性。对于必须使用具有不同数据标签的行上的连接执行查询的情况，它也是必需的。

      如果要使用DBMS_MACADM程序包指定合并算法，请参阅表19-2以获取可能的合并算法的完整列表。

    该策略列在“标签安全策略集成”页面中。

  8. 使用策略的标签标记因子标识。

    向因子添加标识可提供详细信息。

注意：

如果未将Oracle Label Security策略与因子关联，则Oracle Database Vault会维护策略的默认Oracle Label Security行为。

    CONNECT bea_dvacctmgr - 或者，CONNECT bea_dvaccmgr @hrpdb输入密码： 密码 ALTER USER LBACSYS ACCOUNT UNLOCK IDENTIFIED BY password ; CONNECT LBACSYS输入密码： 密码
  2. 创建新的Oracle Label Security策略：
    EXEC SA_SYSDBA.CREATE_POLICY（'PRIVACY'，'PRIVACY_COLUMN'，'NO_CONTROL'）;
  3. 为PRIVACY策略创建以下级别：
    EXEC SA_COMPONENTS.CREATE_LEVEL（'PRIVACY'，2000，'S'，'SENSITIVE'）; EXEC SA_COMPONENTS.CREATE_LEVEL（'PRIVACY'，1000，'C'，'CONFIDENTIAL'）;
  4. 创建PII隔间。
    EXEC SA_COMPONENTS.CREATE_COMPARTMENT（'PRIVACY'，100，'PII'，'PERS_INFO'）;
  5. 授予用户mdale和jsmith以下标签：
    EXEC SA_USER_ADMIN.SET_USER_LABELS（'PRIVACY'，'mdale'，'S：PII'）; EXEC SA_USER_ADMIN.SET_USER_LABELS（'PRIVACY'，'jsmith'，'C'）;

    用户mdale被授予更敏感的标签Sensitive，其中包括PII隔间。用户jsmith获取机密标签，该标签不太敏感。

    EXEC DBMS_MACADM.CREATE_RULE_SET（'PII规则集'，'保护特权用户的PII数据'，'Y'，1,0,2，NULL，NULL，0，NULL）;
  3. 为PII规则集创建规则。
    EXEC DBMS_MACADM.CREATE_RULE（'检查OLS因子'，'支配（sa_utl.numeric_label（''PRIVACY''），char_to_label（''PRIVACY''，''S：PII''））=''1'''） ;

  4. 将Check OLS Factor规则添加到PII规则集。
    EXEC DBMS_MACADM.ADD_RULE_TO_RULE_SET（'PII规则集'，'检查OLS因子'）;

步骤4：更新ALTER SYSTEM命令规则以使用规则集

在可以使用规则集之前，必须更新ALTER SYSTEM命令规则，这是一个默认命令规则。

    在默认安装中，ALTER SYSTEM命令规则使用“允许细粒度控制系统参数”规则集并启用。

    此命令将PII规则集添加到ALTER SYSTEM命令规则，将其应用于所有对象所有者和对象名称，并启用命令规则。

将Oracle Database Vault与Oracle Data Guard集成

Oracle Database Vault-Oracle Data Guard集成首先需要主数据库配置，然后是备用数据库配置。

  • 第1步：配置主数据库
    您必须运行DGMGRL和DBCA实用程序，然后运行ALTER SYSTEM语句来配置主数据库。
  • 第2步：配置备用数据库
    您可以在数据库中执行备用数据库配置以用于备用数据库。

父主题： 将Oracle Database Vault与其他Oracle产品集成

第1步：配置主数据库

您必须运行DGMGRL和DBCA实用程序，然后运行ALTER SYSTEM语句来配置主数据库。

  1. 对于Linux和UNIX系统，请确保在要安装Oracle Database Vault的节点上有数据库的/etc/oratab条目。

  2. 如果您使用的是Data Guard Broker，则从命令提示符处禁用配置，如下所示：

    dgmgrl sys输入密码： 密码 DGMGRL>禁用配置;

  3. 运行Database Configuration Assistant（DBCA）并配置数据库选项以将Oracle Database Vault添加到主数据库。

    1. 从命令行输入以下命令以启动DBCA：

      DBCA

    2. 选择正确的数据库类型（ Cluster或Single Instance ），然后单击Next 。

    3. 在Database Operation页面中，选择Configure Database Options ，然后单击Next 。

    4. 选择适当的数据库并单击Next 。

    5. 选择Oracle Label Security ，然后选择Oracle Database Vault并单击Next 。

    6. 输入Database Vault所有者的名称（必需）和Database Vault客户经理（推荐）。

      密码必须至少包含一个字母字符，一个数字和一个特殊字符。

    7. 单击下一步 。

    8. 选择适当的连接模式并单击Next 。

    9. 单击“ 确定”重新启动数据库。

    10. 单击配置其他组件上的确定 。

    此时，主站点上的安装已完成。

  4. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。 

    sqlplus sys as sysdba输入密码： password

  5. 运行以下ALTER SYSTEM语句：

    ALTER SYSTEM SET AUDIT_SYS_OPERATIONS = TRUE SCOPE = SPFILE; ALTER SYSTEM SET OS_ROLES = FALSE SCOPE = SPFILE; ALTER SYSTEM SET RECYCLEBIN ='OFF'SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_LOGIN_PASSWORDFILE ='EXCLUSIVE'SCOPE = SPFILE; ALTER SYSTEM SET SQL92_SECURITY = TRUE SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_OS_AUTHENT = FALSE SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_OS_ROLES = FALSE SCOPE = SPFILE;

  6. 在每个数据库实例上运行ALTER SYSTEM语句以设置参数，如步骤5中所示。

  7. 重启每个数据库实例。

    CONN SYS as SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP

第2步：配置备用数据库

您可以在数据库中执行备用数据库配置以用于备用数据库。

  1. 使用SYSDBA管理权限以SYS用户身份登录数据库实例。
    sqlplus sys as sysdba输入密码： password
  2. 在多租户环境中，连接到相应的PDB。

    例如：

    CONNECT bea_dvacctmgr @hrpdb输入密码： 密码

    要查找可用的PDB，请运行show pdbs命令。要检查当前PDB，请运行show con_name命令。

  3. 挂载备用数据库实例。
    ALTER DATABASE MOUNT STANDBY DATABASE;
  4. 运行以下ALTER SYSTEM语句：
    ALTER SYSTEM SET AUDIT_SYS_OPERATIONS = TRUE SCOPE = SPFILE; ALTER SYSTEM SET OS_ROLES = FALSE SCOPE = SPFILE; ALTER SYSTEM SET RECYCLEBIN ='OFF'SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_LOGIN_PASSWORDFILE ='EXCLUSIVE'SCOPE = SPFILE; ALTER SYSTEM SET SQL92_SECURITY = TRUE SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_OS_AUTHENT = FALSE SCOPE = SPFILE; ALTER SYSTEM SET REMOTE_OS_ROLES = FALSE SCOPE = SPFILE;
  5. 重新启动或装入数据库实例。

    例如：

    关机立即启动
  6. 挂载下一个备用实例。
  7. 按如下方式重新启动托管恢复：
    ALTER DATABASE RECOVER MANAGED STANDBY DATABASE;
  8. 如果您使用的是Data Guard Broker，则从命令行重新启用配置。
    dgmgrl sys输入密码： password DGMGRL> enable configuration;

    此命令将对Oracle Database Vault安装所做的物理备用数据库的更改应用于主数据库。

  9. 在每个物理备用数据库上重复物理备用安装过程。例如，如果有三个物理备用数据库，则运行这些过程或每个备用数据库。
