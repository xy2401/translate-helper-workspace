注意：

本节中的说明使用启用池的数据源来隐式创建和启动连接池。

3.1.1 UCP借用连接概述

UCP API提供了两个启用池的数据源，一个用于借用常规连接，另一个用于借用XA连接。这些数据源提供对UCP JDBC连接池功能的访问，并包含一组用于借用连接的getConnection方法。XA和非XA UCP JDBC连接池中都包含相同的池功能。

UCP JDBC连接池维护可用连接和借用连接。如果应用程序请求借用与可用连接匹配的连接，则从池中重用连接。如果池中没有可用连接与请求的连接匹配，则会创建新连接。可用连接和借用连接的数量受池属性（如池大小，超时间隔和验证规则）的影响。

3.1.2使用启用池的数据源

UCP提供了一个启用池的数据源（ oracle.ucp.jdbc.PoolDataSource ）用于获取与数据库的连接。oracle.ucp.jdbc.PoolDataSourceFactory工厂类提供了一个getPoolDataSource()方法，用于创建启用池的数据源实例。例如：

PoolDataSource pds = PoolDataSourceFactory.getPoolDataSource（）;

启用池的数据源需要连接工厂类才能获得实际的物理连接。连接工厂通常作为JDBC驱动程序的一部分提供，并且可以是数据源本身。UCP JDBC连接池可以使用任何JDBC驱动程序来创建随后由池维护的物理连接。setConnectionFactoryClassName(String)方法用于为启用池的数据源实例定义连接工厂。以下示例使用Oracle的oracle.jdbc.pool.JDBC驱动程序附带的OracleDataSource连接工厂类。如果您使用的是其他供应商的JDBC驱动程序，请参阅供应商的文档以获取相应的连接工厂类。 

pds.setConnectionFactoryClassName（“oracle.jdbc.pool。OracleDataSource“）;

除连接工厂类外，启用池的数据源还需要用于连接数据库的URL，用户名和密码。启用池的数据源实例包括设置每个属性的方法。以下示例使用Oracle JDBC Thin驱动程序语法。如果您使用的是其他供应商的JDBC驱动程序，请参阅供应商的文档以获取要使用的相应URL语法。 

pds.setURL（ “JDBC：预言：瘦：@ //本地主机：1521 / XE”）; pds.setUser（“ user ”）; pds.setPassword（“ password ”）;

最后，启用池的数据源提供了一组getConnection方法。方法包括：

  • getConnection() ：返回与用于连接数据库的用户名和密码关联的连接。

  • getConnection(String username, String password) ：返回与给定用户名和密码关联的连接。

  • 的getConnection（java.util中。Properties labels) ：返回与给定标签匹配的连接。

  • getConnection（String username，String password，java.util。Properties labels) ：返回与给定用户名和密码关联且与给定标签匹配的连接。

应用程序使用getConnection方法从java.sql.类型的池中借用连接句柄java.sql.Connection 。如果池中已存在与请求的连接匹配的连接句柄（相同的URL，用户名和密码），则它将返回给应用程序;否则，将创建一个新连接，并将新连接句柄返回给应用程序。提供了Oracle和MySQL的示例。

3.1.3使用支持池的XA数据源

UCP提供了一个启用池的XA数据源（ oracle.ucp.jdbc.PoolXADataSource ），用于获取可以在分布式事务中登记的XA连接。UCP JDBC XA池具有与非XA UCP JDBC池相同的功能。oracle.ucp.jdbc.PoolDataSourceFactory工厂类提供了一个getPoolXADataSource()方法，用于创建启用池的XA数据源实例。例如：

PoolXADataSource pds = PoolDataSourceFactory.getPoolXADataSource（）;

启用池的XA数据源实例（如非XA数据源实例）需要连接工厂，URL，用户名和密码才能获得实际的物理连接。这些属性的设置方式与非XA数据源实例相同（参见上文）。但是，需要特定于XA的连接工厂类才能获得XA连接。XA连接工厂通常作为JDBC驱动程序的一部分提供，并且可以是数据源本身。以下示例使用Oracle的oracle.jdbc.xa.client.JDBC驱动程序附带的OracleXADataSource XA连接工厂类。如果使用其他供应商的JDBC驱动程序，请参阅供应商的文档以获取适当的XA连接工厂类。 

pds.setConnectionFactoryClassName（“oracle.jdbc.xa.client。OracleXADataSource“）; pds.setURL（”jdbc：oracle：thin：@ // localhost：1521 / XE“）; pds.setUser（” user “）; pds.setPassword（” password “）;

最后，启用池的XA数据源提供了一组getXAConnection方法，这些方法用于从javax.sql.类型的池中借用连接句柄javax.sql.XAConnection 。getXAConnection方法与前面描述的getConnection方法相同。以下示例演示了借用XA连接。 

PoolXADataSource pds = PoolDataSourceFactory.getPoolXADataSource（）; pds.setConnectionFactoryClassName（“oracle.jdbc.xa.client。OracleXADataSource“）; pds.setURL（”jdbc：oracle：thin：@ // localhost：1521 / XE“）; pds.setUser（” <user> “）; pds.setPassword（” <password> “）; XAConnection conn = pds.getXAConnection（）;

3.1.4设置连接属性

Oracle的连接工厂支持配置具有特定功能的连接的属性。启用UCP池的数据源提供setConnectionProperties(Properties)方法，该方法用于设置给定连接工厂的属性。以下示例演示如何设置Oracle JDBC驱动程序的连接属性。如果您使用的是其他供应商的JDBC驱动程序，请参阅特定于供应商的文档，以检查是否支持以这种方式设置属性以及可用的属性：

属性connProps = new Properties（）; connProps.put（“fixedString”，false）; connProps.put（“remarksReporting”，false）; connProps.put（“restrictGetTables”，false）; connProps.put（“includeSynonyms”，false）; connProps.put（“defaultNChar”，false）; connProps.put（“AccumulateBatchResult”，false）; pds.setConnectionProperties（connProps）;

如果在创建和使用池后调用setConnectionProperties ，则UCP JDBC连接池不会删除已创建的连接。

3.1.5使用JNDI借用连接

通过执行JNDI查找启用池的数据源，然后在返回的对象上调用getConnection() ，可以从连接池借用连接。必须首先将启用池的数据源绑定到JNDI上下文和逻辑名称。这假定应用程序包括用于命名和目录服务的服务提供者接口（SPI）实现，其中可以注册和定位对象引用。

以下示例使用Sun的文件系统JNDI服务提供程序，可以从JNDI软件下载页面下载：

http://www.oracle.com/technetwork/java/index.html

该示例演示如何创建初始上下文，然后对绑定到名称MyPooledDataSource的启用池的数据源执行查找。然后，返回的对象用于从连接池借用连接。 

Hashtable env = new Hashtable（）; env.put（上下文。INITIAL_CONTEXT_FACTORY，“com.sun.jndi.fscontext。RefFSContextFactory“）; env.put（Context。PROVIDER_URL，“file：/ tmp”）; ctx = new InitialContext（env）; PoolDataSource jpds =（PoolDataSource）ctx.lookup（MyPooledDataSource）; Connection conn = jpds.getConnection（）;

3.1.6关于连接初始化回调

连接初始化回调使应用程序和框架能够初始化从通用连接池检索的连接。它在池中的每个连接签出时执行，并在故障转移期间每次成功重新连接时执行。

本节讨论以下部分中的初始化回调：

  • 连接初始化回调概述

  • 创建初始化回调

  • 注册初始化回调

  • 删除或取消注册初始化回调

3.1.6.1连接初始化回调概述

如果应用程序因无法更改而无法使用连接标记，则会为此类应用程序提供连接初始化回调。

注册后，每次从池中借用连接时以及在可恢复错误后每次成功重新连接时都会执行初始化回调。在运行时和重放时使用相同的回调可确保在运行时重新建立原始会话建立时使用的完全相同的初始化。如果回调调用失败，则在该连接上禁用重播。

3.1.6.2创建初始化回调

要创建UCP连接初始化回调，应用程序将实现oracle.ucp.jdbc.ConnectionInitializationCallback接口。该接口具有以下方法：

void initialize（java.sql。连接连接）抛出SQLException;

注意：

  • 为每个连接池创建一个回调。

  • 如果为连接池注册了标签回调，则不使用此回调。

例

以下示例演示如何创建简单的初始化回调：

import oracle.ucp.jdbc。ConnectionInitializationCallback; class MyConnectionInitializationCallback实现ConnectionInitializationCallback {public MyConnectionInitializationCallback（）{...public void initialize（java.sql。连接连接）抛出SQLException {//如有必要，重置连接状态（如ALTER SESSION）}}

3.1.6.3注册初始化回调

UCP在oracle.ucp.jdbc.提供了registerConnectionInitializationCallback方法oracle.ucp.jdbc.PoolDataSource接口，用于注册连接初始化回调。 

public void registerConnectionInitializationCallback（ConnectionInitializationCallback cbk）抛出SQLException;

可以在每个连接池实例上注册一个回调。

3.1.6.4删除或取消注册初始化回调

UCP在oracle.ucp.jdbc.提供unregisterConnectionInitializationCallback方法oracle.ucp.jdbc.PoolDataSource接口，用于取消注册连接初始化回调。 

public void unregisterConnectionInitializationCallback（ConnectionInitializationCallback cbk）抛出SQLException;
