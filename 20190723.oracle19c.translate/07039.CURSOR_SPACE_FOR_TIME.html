1.61 CURSOR_SPACE_FOR_TIME

CURSOR_SPACE_FOR_TIME允许您为游标使用更多空间以节省时间。

属性 描述

参数类型

布尔

默认值

假

可修改

没有

可在PDB中修改

没有

价值范围

是的|假

基本

没有

注意：

不推荐使用CURSOR_SPACE_FOR_TIME参数。它仅为了向后兼容而保留。

此参数同时影响共享SQL区域和客户端的私有SQL区域。

大多数用户不需要设置此参数，因为Oracle Database 10g第2版（10.2.0.2）及更高版本中引入了显着增强的并发修改。

值

  • 真正

    共享SQL区域保留在共享池中。因此，只要打开的游标引用它们，共享的SQL区域就不会从池中老化。因为每个活动游标的SQL区域都存在于内存中，所以执行速度更快。但是，共享SQL区域在使用时永远不会留下内存。因此，仅当共享池足够大以同时保存所有打开的游标时，才应将此参数设置为true 。

    此外，设置为true保留在执行之间为每个游标分配的私有SQL区域，而不是在游标执行后丢弃它，从而节省了游标分配和初始化时间。

  • 假

    可以从库高速缓存中释放共享SQL区域，以便为新的SQL语句腾出空间。

注意：

如果此参数设置为true ，则禁用SERIAL_REUSE参数。
