45.1 DBMS_DATA_MINING_TRANSFORM概述

转换是一种SQL表达式，用于修改一列或多列中的数据。

数据通常必须经过某些转换才能用于构建挖掘模型。许多数据挖掘算法都有特定的转换要求。

将要评分的数据必须以与用于创建（训练）模型的数据相同的方式进行转换。

外部或嵌入式转换

DBMS_DATA_MINING_TRANSFORM提供了两种实现转换的方法。对于给定的模型，您可以：

  • 创建转换表达式列表并将其传递给CREATE_MODEL过程

    要么

  • 创建一个实现转换的视图，并将视图名称传递给CREATE_MODEL过程

如果创建转换列表并将其传递给CREATE_MODEL ，则转换表达式将嵌入到模型中，并在应用模型时自动实现。

如果创建视图，则转换表达式在模型外部。无论何时应用模型，都需要重新创建转换。

注意：

嵌入式转换显着增强了模型的可用性，同时简化了模型管理过程。

自动转换

Oracle Data Mining支持自动数据准备（ADP）模式。启用ADP后，将自动嵌入大多数特定于算法的转换。必须在嵌入式转换列表或视图中显式提供任何其他转换。

如果启用了ADP并且您创建了带有转换列表的模型，则会嵌入这两组转换。在执行ADP指定的自动转换之前，模型将从转换列表中执行用户指定的转换。

在转换列表中，您可以有选择地为各个属性禁用ADP。

DBMS_DATA_MINING_TRANSFORM中的转换

本节总结了DBMS_DATA_MINING_TRANSFORM支持的转换。

分档

分箱是指连续或离散值到减少基数的离散值的映射。

  • 监督分箱（分类和数字）

    分箱基于决策树模型确定的数据中的内在关系。

    请参见“ INSERT_BIN_SUPER过程 ” 。

  • 前N个频率分类分箱

    分箱基于每个类别中的案例数量。

    请参阅“ INSERT_BIN_CAT_FREQ过程 ”

  • 等宽数值分档

    分箱基于等范围分区。

    请参见“ INSERT_BIN_NUM_EQWIDTH过程 ” 。

  • 分位数值分档

    分箱基于使用SQL NTILE函数计算的分位数。

    请参见“ INSERT_BIN_NUM_QTILE过程 ” 。

线性归一化

标准化是将连续值缩放到特定范围（通常在0和1之间）的过程。归一化通过减去数字（ 移位 ）并将结果除以另一个数字（ 比例 ）来转换每个数值。 

x_new =（x_old-shift）/ scale

  • 最小 - 最大规范化

    标准化基于最小值和最大值，具有以下偏移和比例：

    shift = min scale = max-min

    请参见“ INSERT_NORM_LIN_MINMAX过程 ” 。

  • 比例标准化

    标准化基于最小值和最大值，具有以下偏移和比例：

    shift = 0 scale = max {abs（max），abs（min）}

    请参见“ INSERT_NORM_LIN_SCALE过程 ” 。

  • Z-Score标准化

    标准化基于平均值和标准偏差，具有以下偏移和比例：

    shift = mean scale = standard_deviation

    请参见“ INSERT_NORM_LIN_ZSCORE过程 ” 。

异常值处理

异常值是一个远离其余数据的数值。异常值可以人为地扭曲数据挖掘的结果。

  • 极值调整

    异常值被替换为不是异常值的最近值。

    请参阅“ INSERT_CLIP_WINSOR_TAIL过程 ”

  • 修剪

    异常值设置为NULL 。

    请参见“ INSERT_CLIP_TRIM_TAIL过程 ” 。

缺少价值待遇

丢失数据可能表示稀疏性，或者可能表示某些值随机丢失。DBMS_DATA_MINING_TRANSFORM支持以下转换，以最大限度地减少缺失值的影响：

  • 缺失的数值被替换为平均值。

    请参见“ INSERT_MISS_NUM_MEAN过程 ” 。

  • 缺少分类值将替换为模式。

    请参见“ INSERT_MISS_CAT_MODE过程 ” 。

注意：

Oracle Data Mining还具有处理缺失数据的默认机制。有关详细信息，请参见Oracle Data Mining用户指南 。

45.2 DBMS_DATA_MINING_TRANSFORM操作说明

DBMS_DATA_MINING_TRANSFORM包提供了一个灵活的框架，用于指定数据转换。如果选择在模型中嵌入转换（首选方法），则创建转换列表对象并将其传递给CREATE_MODEL过程。如果选择在不嵌入的情况下转换数据，则可以创建视图。

在转换列表中指定时，转换表达式由模型执行。在视图中指定时，转换表达式由视图执行。

转换定义

转换定义用于生成转换数据的SQL表达式。例如，规范化数字列的转换定义是该数据的移位和缩放值。

使用DBMS_DATA_MINING_TRANSFORM包，您可以调用程序来计算转换定义，或者您可以自己计算它们，或者您可以同时执行这两种操作。

转换定义表

DBMS_DATA_MINING_TRANSFORM提供INSERT过程，用于计算转换定义并将它们插入转换定义表中。您可以修改转换定义表中的值或自行填充它们。

XFORM例程使用填充的定义表来转换外部视图中的数据。STACK例程使用填充的定义表来构建转换列表。

要基于定义表指定转换，请按照下列步骤操作：

  1. 使用CREATE例程创建转换定义表。

    这些表具有用于保存给定转换类型的转换定义的列。例如， CREATE_BIN_NUM过程创建一个定义表，该表具有用于存储数据值的列和用于存储关联的bin标识符的另一列。

  2. 使用INSERT例程来计算和插入表中的转换定义。

    每个INSERT例程都使用特定技术来计算转换定义。例如， INSERT_BIN_NUM_EQWIDTH过程通过识别最小值和最大值然后以相等的间隔设置bin边界来计算bin边界。

  3. 使用STACK或XFORM例程根据定义表中的信息生成转换表达式：

    • 使用STACK例程将转换表达式添加到转换列表。将转换列表传递给CREATE_MODEL过程 。转换表达式将组合成一个长SQL查询并嵌入到模型中。

    • 使用XFORM例程在视图中执行转换表达式。转换将在模型外部进行，并且只要将模型应用于新数据，就需要重新创建转换。

没有定义表的转换

STACK例程不是将转换表达式添加到转换列表的唯一方法。您还可以在不使用定义表的情况下构建转换列表。

若要指定转换而不使用定义表，请按照下列步骤操作：

  1. 编写用于转换属性的SQL表达式。

  2. 编写用于反转转换的SQL表达式。（请参阅“ DBMS_DATA_MINING_TRANSFORM - 关于转换列表 ”中的 “反向转换和模型透明度 ” 。）

  3. 确定是否为属性禁用ADP。默认情况下，如果为模型指定了属性，则会为该属性启用ADP。（请参阅“ DBMS_DATA_MINING_TRANSFORM - 关于转换列表 ”中的 “禁用自动数据准备 ” 。）

  4. 在调用SET_TRANSFORM过程时指定SQL表达式和ADP指令，该过程将信息添加到转换列表中。

  5. 对要转换的每个属性重复步骤1到4 。

  6. 将转换列表传递给CREATE_MODEL过程 。转换表达式将组合成一个长SQL查询并嵌入到模型中。

    注意：

    使用SET_TRANSFORM指定的SQL表达式必须适合VARCHAR2 。要指定更长的表达式，可以使用SET_EXPRESSION过程 。使用SET_EXPRESSION ，您可以通过将行附加到VARCHAR2数组来构建表达式。

45.2.1 DBMS_DATA_MINING_TRANSFORM - 关于转换列表

转换列表的元素是转换记录 。每个转换记录提供模型所需的所有信息，以管理单个属性的转换。

每个转换记录包括以下字段：

  • attribute_name - 要转换的数据列的名称

  • attribute_subname - 如果attribute_name是嵌套列，则为嵌套属性的名称，否则为NULL

  • expression - 用于转换属性的SQL表达式

  • reverse_expression - 用于反转转换的SQL表达式

  • attribute_spec - 在模型构建期间标识attribute_spec特殊处理。有关详细信息，请参见表45-33 。

逆转换和模型透明度

算法操纵变换的属性以训练和评分模型。但是，转换后的属性对最终用户可能没有意义。例如，如果属性x已被转换为区间1-4，则区域名称1,2,3和4由算法操纵，但是用户可能对关于区间1-4的模型细节不感兴趣。预测数字1 - 4。

要在模型详细信息和预测中返回原始属性值，可以在属性的转换记录中提供反向表达式。例如，如果为属性y指定转换表达式'log(10, y)' ，则可以指定反向转换表达式'power(10, y)' 。

反向转换启用 模型透明度 。它们使内部处理对用户透明。

注意：

STACK过程自动反转规范化转换，但它们不提供反转分级，剪切或缺失值转换的机制。

您可以使用DBMS_DATA_MINING.ALTER_REVERSE_EXPRESSION过程为现有模型指定或更新反向转换表达式。

将转换记录添加到转换列表

转换列表是转换记录的堆栈。添加新转换记录时，会将其附加到堆栈顶部。（有关详细信息，请参阅“ 关于堆叠 ” 。）

使用SET_TRANSFORM将转换记录添加到转换列表时，可以为转换记录中的所有字段指定值。

使用STACK过程将转换记录添加到转换列表时，仅填充转换表达式字段。对于规范化转换，还会填充反向转换表达式字段。

您可以使用STACK过程和SET_TRANSFORM来构建一个转换列表。每个STACK过程调用都会为指定的转换定义表中的所有属性添加转换记录。每个SET_TRANSFORM调用都会为单个属性添加转换记录。

转换列表是通过堆叠转换记录构建的。转换列表从下到上进行评估。每个转换表达式都取决于堆栈中它下面的转换表达式的结果。

堆栈程序

STACK过程根据转换定义表中的信息创建转换记录。例如， STACK_BIN_NUM为定义表中指定的每个属性构建转换记录，以进行数字分箱。STACK过程堆栈转换记录如下：

  • 如果在定义表中指定了属性但未在转换列表中指定属性，则STACK过程创建转换记录，计算反向转换（如果可能），在转换记录中插入转换和反向转换，并将转换记录附加到转型清单的顶部。

  • 如果在转换列表中指定了属性但未在定义表中指定，则STACK过程不执行任何操作。

  • 如果在定义表和转换列表中指定了属性，则STACK过程将转换表达式堆叠在转换记录中转换表达式顶部的定义表中，并更新反向转换。见表45-1和例45-4 。

45.2.3 DBMS_DATA_MINING_TRANSFORM - 嵌套数据转换

CREATE例程创建转换定义表，其中包含两列col和att ，用于标识属性。列col保存数据表中列的名称。如果数据列未嵌套，则att为null，并且属性的名称为col 。如果数据列是嵌套的，则att保存嵌套属性的名称，属性的名称为col.att 。

INSERT和XFORM例程忽略定义表中的att列。INSERT和XFORM例程都不支持嵌套数据。

只有STACK过程和SET_TRANSFORM支持嵌套数据。嵌套数据转换始终嵌入模型中。

在转换记录中指定嵌套属性

转换记录（ TRANSFORM_REC ）包括两个字段attribute_name和attribute_subname ，用于标识属性。字段attribute_name保存数据表中列的名称。如果数据列未嵌套，则attribute_subname为null，并且属性的名称为attribute_name 。如果数据列是嵌套的，则attribute_subname保存嵌套属性的名称，属性的名称为attribute_name.attribute_subname 。

45.6.21 INSERT_NORM_LIN_ZSCORE过程

此过程执行线性规范化，并将转换定义插入转换定义表中。

INSERT_NORM_LIN_ZSCORE计算数据的均值和标准差，并设置shift和scale的值，如下所示：

shift = mean scale = stddev

x_new =（ x_old  -  shift ）/ scale

INSERT_NORM_LIN_ZSCORE将scale的值INSERT_NORM_LIN_ZSCORE入到指定的有效位数，然后将其存储在转换定义表中。
