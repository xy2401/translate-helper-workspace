29.2.1 OCIExtProcAllocCallMemory（）

在外部过程的持续时间内分配N个字节的内存。

目的

在外部过程的持续时间内分配N个字节的内存。

句法

void * OCIExtProcAllocCallMemory（OCIExtProcContext * with_context，size_t amount）;

参数

with_context（IN）

传递给C外部过程的with_context指针。

With_Context Type - PL / SQL外部过程的C可调用接口需要传递with_context参数。此结构的类型是OCIExtProcContext ，它对用户不透明。

用户可以在应用程序中声明with_context参数，如下所示：

OCIExtProcContext * with_context;
金额（IN）

要分配的字节数。

评论

此调用在外部过程调用期间分配内存的amount字节。

从外部过程返回时，PL / SQL将释放此调用分配的所有内存。应用程序不得对OCIExtProcAllocCallMemory()分配的内存使用任何类型的free()函数。使用此函数为函数返回分配内存。

零返回值应视为错误。

返回

指向已分配内存的无类型（不透明）指针。

例

使用OCIExtProcAllocCallMemory（）分配1024字节的内存

text * ptr =（text *）OCIExtProcAllocCallMemory（wctx，1024）

29.2.2 OCIExtProcGetEnv（）

获取OCI环境，服务上下文和错误句柄。

目的

获取OCI环境，服务上下文和错误句柄。

句法

剑OCIExtProcGetEnv（OCIExtProcContext * with_context，OCIEnv ** envh，OCISvcCtx ** svch，OCIError ** errh）;

参数

with_context（IN）

传递给C外部过程的with_context指针。

With_Context Type - PL / SQL外部过程的C可调用接口需要传递with_context参数。此结构的类型是OCIExtProcContext ，它对用户不透明。

用户可以在应用程序中声明with_context参数，如下所示：

OCIExtProcContext * with_context;
envh（OUT）

指向存储OCI环境句柄的变量的指针。

svch（OUT）

指向存储OCI服务句柄的变量的指针。

errh（OUT）

指向存储OCI错误句柄的变量的指针。

评论

此函数的主要目的是允许OCI回调在同一事务中使用数据库。通过此函数获取的OCI句柄应该用于数据库的OCI回调。如果通过标准OCI调用获得这些句柄，则这些句柄使用与数据库的新连接，并且不能用于同一事务中的回调。在一个外部过程中，您可以使用回调或新连接，但不能同时使用两者。

通话示例：

OCIEnv * envh; OCISvcCtx * svch; OCIError * errh; ...OCIExtProcGetEnv（CTX，＆envh，＆svch，＆errh）;

返回

如果调用成功，此函数返回OCI_SUCCESS ;否则，它返回OCI_ERROR 。

29.2.3 OCIExtProcRaiseExcp（）

引发PL / SQL的异常。

目的

引发PL / SQL的异常。

句法

size_t OCIExtProcRaiseExcp（OCIExtProcContext * with_context，int errnum）;

参数

with_context（IN）

传递给C外部过程的with_context指针。

With_Context Type - PL / SQL外部过程的C可调用接口需要传递with_context参数。此结构的类型是OCIExtProcContext ，它对用户不透明。

用户可以在应用程序中声明with_context参数，如下所示：

OCIExtProcContext * with_context;
errnum（IN）

Oracle数据库错误号，用于向PL / SQL发送信号。 errnum值必须为正数，范围为1到32767。

评论

调用此函数会发出PL / SQL异常信号。从此函数成功返回后，外部过程必须启动其退出处理并返回到PL / SQL。一旦向PL / SQL发出异常信号，就不会处理IN / OUT和OUT参数（如果有的话）。

返回

如果调用成功，此函数返回OCIEXTPROC_SUCCESS 。如果调用失败，则返回OCIEXTPROC_ERROR 。
