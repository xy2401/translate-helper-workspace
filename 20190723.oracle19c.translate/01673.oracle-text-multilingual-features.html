D Oracle文本多语言功能

D.1简介

本附录总结了Oracle Text的主要多语言功能。

有关Oracle全球化支持语言和字符集支持的完整列表，请参阅“ Oracle数据库全球化支持指南” 。

注意：

Oracle Text不支持NLS_COMP和NLS_SORT参数。从Oracle Text生成的搜索结果与这些参数的值无关。

在Oracle Database 12 c第2版（12.2）中，无法在具有除BINARY ， USING_NLS_COMP ， USING_NLS_SORT或USING_NLS_SORT_CS之外的声明排序规则的列上创建Oracle Text索引。对于所有受支持的排序规则，Oracle Text行为都是相同的。

D.2.1.1语境索引类型

CONTEXT索引类型完全支持多语言功能，包括使用语言和字符集列。

支持以下词法分析器：

  • AUTO_LEXER

  • MULTI_LEXER

  • USER_LEXER

  • WORLD_LEXER

  • CHINESE_LEXER

  • CHINESE_VGRAM_LEXER

  • JAPANESE_LEXER

  • JAPANESE_VGRAM_LEXER

  • KOREAN_MORPH_LEXER

D.2.1.2 CTXCAT指数类型

CTXCAT支持的多语言功能BASIC_LEXER与索引主题外，并支持以下附加词法分析器：

  • USER_LEXER

  • WORLD_LEXER

CTXCAT还支持以下词法分析器：

  • CHINESE_LEXER

  • CHINESE_VGRAM_LEXER

  • JAPANESE_LEXER

  • JAPANESE_VGRAM_LEXER

  • KOREAN_MORPH_LEXER

D.2.1.3 CTXRULE索引类型

所述CTXRULE索引类型支持的多语言功能BASIC_LEXER包括ABOUT和STEM运算符。它还支持日语，中文和韩语（与SVM_CLASSIFIER一起使用SVM_CLASSIFIER ）。

D.2.2 Lexer类型

Oracle Text支持在索引过程中选择词法分析器，从而支持不同语言的索引。您使用的词法分析器决定了您可以索引的语言。表D-1描述了支持的词法分析器。

D.2.3 Lexer的基本功能

BASIC_LEXER首选项支持以下功能。使用BASIC_LEXER属性启用这些功能。可以一起启用备用拼写，复合和基本字母等功能，以获得更好的搜索结果。

  • 主题索引

  • 替代拼写

  • 基本字母转换

  • 综合

  • 索引源于此

D.2.3.1主题索引

使用具有CONTEXT索引类型的ABOUT运算符启用索引和后续查询文档概念。这些概念源自Oracle Text知识库。英语和法语支持此功能。

CTXCAT索引类型不支持此功能。

D.2.3.2替代拼写

此功能使您可以搜索单词的替代拼写。例如，在德语中启用备用拼写时，对包含groß和gross的 总返回文档的查询。

德语，丹麦语和瑞典语支持此功能。

此外，德语可以根据传统和改革的拼写惯例进行索引。

也可以看看：

“ 替代拼写 ”和“ 新德语拼写 ” 。

D.2.3.3基本字母转换

此功能使您可以查询带有或不带变音标记的单词，如波浪号，重音符号和变音符号。例如，对于西班牙语基本字母索引， energia的查询匹配包含energía和energia的文档。

英语和所有其他受支持的空白分隔语言支持此功能。在英语和法语中，您可以使用基本词法分析器来启用主题索引。

也可以看看：

“ 基础字母转换 ”

D.2.3.4复合材料

通过此功能，您可以搜索包含指定术语的单词作为子复合词。您必须使用词干（$）运算符。德语和荷兰语支持此功能。

例如，在德语中， $ register的查询查找包含Bruttoregistertonne和Registertonne的文档。

D.2.3.5索引词干

此功能使您可以指定用于词干索引的词干分析器。除了正常形式之外，在索引时将标记限制为单个基本形式。指定索引词干可以提高词干查询的查询性能，例如$ computed 。

英语，荷兰语，法语，德语，意大利语和西班牙语支持此功能。

D.2.4多个Lexer功能

MULTI_LEXER词法分析器使您可以索引包含不同语言文档的列。在索引期间，Oracle Text会检查语言列并切换特定于语言的词法分析器来处理文档。在编制索引之前定义每种语言的词法分析器首选项。

多词法分析器使您可以为语言设置不同的首选项。例如，对于德语文档，可以将composite设置为TRUE ，对于荷兰文档，可以将composite设置为FALSE 。

D.2.5世界Lexer功能

与MULTI_LEXER一样， WORLD_LEXER词法分析器使您可以索引包含不同语言的文档。它会自动检测文档的语言，因此不需要您在基表中创建语言列。

WORLD_LEXER处理所有数据库字符集并支持Unicode 5.0标准。要使WORLD_LEXER对使用多种语言的文档有效，必须指定AL32UTF-8或UTF8 Oracle字符集编码。这包括补充或“代理对”字符。

D.3查询

Oracle Text支持使用不同的查询运算符。某些运算符可以设置为根据您的语言运行。本节总结了这些运算符的多语言查询功能。

  • 使用ABOUT运算符查询概念。系统在索引的主题组件中查找概念信息。仅具有CONTEXT索引的英语和法语支持此功能。

  • 使用模糊运算符可以搜索与指定单词具有相似拼写的单词。Oracle Text支持fuzzy英语，法语，德语，意大利语，荷兰语，西班牙语，葡萄牙语，日语，光学字符识别（OCR）和自动语言检测。

  • 使用词干运算符，您可以搜索与指定术语具有相同根的单词。例如， $ sing的词干扩展为单词sang，sung，sing的查询。Oracle Text词干分析器支持以下语言：英语，法语，西班牙语，意大利语，德语，日语和荷兰语。

D.4提供停止列表

停止列表是未编入索引的单词列表。这些通常是在一个语言常用词，如这个 ， 那个 ，并能在英语。

Oracle Text提供英语，中文（繁体和简体），丹麦语，荷兰语，芬兰语，法语，德语，意大利语，葡萄牙语，西班牙语和瑞典语的默认停止列表。 Oracle Text提供的停止列表，列出了各种语言的停止列表。

D.5知识库

Oracle Text知识库是用于主题索引， ABOUT查询和派生文档服务主题的概念的分层树。

Oracle Text仅提供英语和法语的知识库。默认情况下会安装这些知识库。

您可以通过为任何单字节空格分隔语言（包括西班牙语）加载您自己的知识库，将主题功能扩展为英语或法语以外的语言。
