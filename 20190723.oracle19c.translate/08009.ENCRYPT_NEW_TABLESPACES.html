1.117 ENCRYPT_NEW_TABLESPACES

ENCRYPT_NEW_TABLESPACES指定是否加密新创建的用户表空间。

属性 描述

参数类型

串

句法

ENCRYPT_NEW_TABLESPACES = {CLOUD_ONLY |总是| DDL}

默认值

CLOUD_ONLY

可修改

改变系统

可在PDB中修改

是

基本

没有

Oracle RAC

应为所有实例指定相同的值。

可以为ENCRYPT_NEW_TABLESPACES参数指定的值具有以下含义：

  • CLOUD_ONLY ：

    在Oracle Cloud中创建用户表空间时，如果未指定SQL CREATE TABLESPACE语句的ENCRYPTION子句，则使用高级加密标准128（AES 128）对其进行透明加密。在内部部署数据库中创建用户表空间时， CREATE TABLESPACE语句的ENCRYPTION子句确定表空间是否已加密。

    CLOUD_ONLY是默认值。

  • ALWAYS ：

    无论是在Oracle云中还是在内部部署数据库中创建用户表空间，如果在CREATE TABLESPACE语句中未指定ENCRYPTION子句，则表空间将使用AES128进行透明加密。

  • DDL ：

    无论是在Oracle Cloud中还是在内部部署数据库中创建用户表空间， CREATE TABLESPACE语句都将遵循指定的DDL。如果未指定ENCRYPTION子句，则表空间将不会加密。如果USING关键字和算法指定ENCRYPTION子句，则将USING指定的算法来加密表空间。如果指定了ENCRYPTION但未指定算法，则表空间将使用默认的AES128算法进行加密。
