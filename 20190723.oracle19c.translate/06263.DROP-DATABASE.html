2.19 DROP DATABASE

目的

使用DROP DATABASE命令删除目标数据库，如果RMAN连接到恢复目录，则取消注册它。RMAN删除服务器参数文件，所有数据文件，联机重做日志和属于目标数据库的控制文件。默认情况下，RMAN会提示确认。

注意：

此命令不能用于删除配置为创建零数据丢失恢复设备（通常称为恢复设备）备份的受保护数据库。

先决条件

仅在RMAN提示符下执行此命令。您必须连接到目标数据库。目标数据库必须独占挂载而不能打开，并以RESTRICT模式启动。

句法

dropDatabase :: =


说明dropdatabase.eps的说明

语义

语法元素 描述

包括备份

删除从所有已配置的设备类型与目标数据库关联的备份集，代理副本，映像副本和归档重做日志文件。

注意：如果使用恢复目录但在删除数据库时以NOCATALOG模式运行RMAN，则RMAN不会删除恢复目录中已知但目标数据库控制文件中不再存在的任何备份。

NOPROMPT

删除数据库之前不提示确认。

例

示例2-82删除数据库

在此示例中，您要删除在恢复目录中注册的名为test1的测试数据库。启动RMAN客户端，连接到数据库test1作为TARGET ，并连接到恢复目录。然后，运行以下命令以删除目标数据库文件以及与数据库关联的所有备份，副本和归档重做日志文件：
