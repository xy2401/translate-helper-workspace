7.230 V $ FS_FAILOVER_OBSERVERS

V$FS_FAILOVER_OBSERVERS提供有关快速启动故障转移观察器的信息。

如果要查询主数据库，则此视图将返回三行，每行描述一个观察者。但是，只有列NAME具有非空值的行对应于已启动的观察者。如果要查询非主数据库，则不会定义此视图的行为。

柱 数据类型 描述

名称

VARCHAR2（513）

快速启动故障转移观察器名称

注册

VARCHAR2（4）

指示此观察者是否已注册（ YES ）或否（ NO ）。请注意， 仅当HOST不为NULL 才会注册观察者。

主办

VARCHAR2（513）

运行此观察者的主机的名称

ISMASTER

VARCHAR2（4）

指示此观察者是否是主观察者（ YES ）或否（ NO ）

TIME_SELECTED

TIMESTAMP（9）

当观察者成为主观察者时，如果ISMASTER为YES ，则ISMASTER ;否则出现以下常数：

1990-01-01 00：00：00.00

PINGING_PRIMARY

VARCHAR2（4）
可能的值：

  • YES ：Observer当前已连接到主数据库

  • NO ：Observer未连接到主数据库

注意 ：此字段在整个Oracle Real Application Clusters（Oracle RAC）环境中保持一致;也就是说，如果观察者连接到Oracle RAC环境中的主数据库的任何实例，则所有实例都将显示值YES 。

PINGING_TARGET

VARCHAR2（4）
可能的值：

  • YES ：Observer当前连接到目标备用数据库

  • NO ：Observer未连接到目标备用数据库

注意 ：此字段在整个Oracle Real Application Clusters（Oracle RAC）环境中保持一致;也就是说，如果观察者连接到Oracle RAC环境中的目标备用数据库的任何实例，则所有实例都将显示值YES 。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
