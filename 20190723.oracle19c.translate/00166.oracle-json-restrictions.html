一个 Oracle数据库JSON限制

此处列出了与Oracle数据库中Oracle支持JSON数据相关的限制。

除非另有说明，否则如果不遵守指定的限制，则会引发错误。

  • 一般

    • JSON对象或数组的嵌套级别数 ：1000，最大值。

    • JSON字段名称长度： 32767字节，最大值。

  • SQL / JSON函数

    • 返回值长度： 32767字节，最大值。

    • 路径长度： 4K字节，最大值。

    • 路径步数： 65535，最大值。

  • 简化的JSON语法

    • 路径长度： 4K字节，最大值。

    • 路径组件长度： 128字节，最大值。

  • JSON搜索索引

    • 字段名称长度： 255个字节，最大值。如果文档的字段名称长度超过255个字节，则可能无法完全索引。在这种情况下，错误记录在数据库视图CTX_USER_INDEX_ERRORS 。

      注意：

      如果它们索引包含名称长度超过64字节的对象字段的JSON数据，则必须重建在Oracle Database 18c之前创建的任何JSON搜索索引和Oracle Text索引。有关更多信息，请参见Oracle数据库升级指南

      也可以看看：

      Oracle Text Reference

  • JSON数据指南

    • 路径长度： 4000字节，最大值。数据指南忽略长度超过4000字节的路径。

    • 父节点下的子节点数： 1024，最大值。数据指南忽略具有超过1024个子节点的节点。

    • 字段值长度： 32767字节。如果JSON字段的值大于32767字节，则数据指南将长度报告为32767。

    • 零长度字段名称：不支持将零长度（空）对象字段名称（ "" ）与JSON数据指南一起使用。对于包含此类名称的JSON数据，未定义数据引导行为。

  • OSON

    • 所有字段名称，总大小： 65535字节，最大值。

    • 字段名称长度（每个）： 255个字节，最大值。

    • 没有重复字段：如果使用OSON表示具有重复字段名称的JSON对象，则仅存在（保留）这些字段中的一个。

  • PL / SQL getter方法JSON_OBJECT_T.get_keys()为给定的JSON对象返回最多32767个字段名。如果将其应用于具有超过32767个字段的对象，则会引发错误。
