8.1.1在命令行上指定参数

启动SQL * Loader时，指定参数以建立加载操作的各种特征。

您可以用逗号分隔参数，但这不是必需的。

> sqlldr CONTROL = ulcase1.ctl LOG = ulcase1.log用户名：scott密码： password

按位置指定意味着您输入值，但不输入参数名称。在以下示例中，提供了用户名scott ，然后是控制文件的名称ulcase1.ctl 。系统会提示您输入密码：

> sqlldr scott ulcase1.ctl密码： 密码

使用参数名称后，必须为所有后续规范提供参数名称。不允许进一步的位置规范。例如，在以下命令中， CONTROL参数用于指定控制文件名，但随后提供的日志文件名不带LOG参数。即使ulcase1.log的位置正确，这也会导致错误：

> sqlldr scott CONTROL = ulcase1.ctl ulcase1.log

相反，您需要输入以下内容：

> sqlldr scott CONTROL = ulcase1.ctl LOG = ulcase1.log

8.1.3使用SQL * Loader在网络中加载数据

要使用SQL * Loader通过网络连接加载数据，可以在启动SQL * Loader实用程序时在连接字符串中指定连接标识符。

此标识符可以指定与当前用户的ORACLE_SID环境变量设置所标识的当前实例不同的数据库实例。连接标识符可以是Oracle Net连接描述符或映射到连接描述符的网络服务名称（通常在tnsnames.ora文件中定义）。使用连接标识符要求您运行Oracle Net Listener（要启动默认侦听器，请输入lsnrctl start ）。以下示例使用connect标识符inst1为用户scott启动SQL * Loader：

> sqlldr CONTROL = ulcase1.ctl用户名：scott @ inst1密码：密码

本地SQL * Loader客户端连接到由连接标识符inst1 （网络服务名称）定义的数据库实例，并加载数据，如ulcase1.ctl控制文件中所指定。

8.2.1坏

SQL * Loader的BAD命令行参数指定与第一个数据文件规范关联的错误文件的名称或位置，或两者。

默认

数据文件的名称，扩展名为.bad 。

目的

指定与第一个数据文件规范关联的错误文件的名称或位置或两者。

语法和说明

BAD = [ directory /] [ filename ]

坏文件存储在插入期间导致错误或格式不正确的记录。如果指定BAD参数，则必须提供目录或文件名，或两者都提供。如果存在被拒绝的记录，并且您没有为该错误文件指定名称，则该名称默认为扩展名或文件类型为.bad的数据文件的名称。

您为directory提供的值指定了要写入错误文件的目录。规范可以包括设备或网络节点的名称。directory的值确定如下：

  • 如果根本没有指定BAD参数，并且需要一个错误的文件，那么默认目录就是SQL * Loader控制文件所在的目录。

  • 如果使用文件名指定BAD参数，但没有目录，则该目录默认为当前目录。

  • 如果使用目录指定BAD参数但没有文件名，则使用指定的目录，名称默认为数据文件的名称，扩展名或文件类型为.bad 。

您为filename提供的值指定在您的平台上被识别为有效的文件名。您必须仅指定名称（如果要使用.bad以外的名称，则必须指定扩展名）。文件名中的任何空格或标点符号必须用单引号括起来。

在命令行上指定的错误文件将成为与控制文件中的第一个INFILE语句（如果有）相关联的错误文件。您还可以使用BADFILE子句在SQL * Loader控制文件中指定错误文件。如果在控制文件和命令行中都指定了错误文件，则使用命令行值。如果已存在具有该名称的错误文件，则会覆盖该文件，或者创建新版本，具体取决于您的操作系统。

例

以下规范在当前目录中创建名为emp1.bad的错误文件：

BAD = EMP1

8.2.2 BINDSIZE

SQL * Loader的BINDSIZE命令行参数指定绑定数组的最大大小（以字节为单位）。

默认

256000

目的

指定绑定数组的最大大小（以字节为单位）。

BINDSIZE = n

绑定数组是内存中SQL * Loader存储要加载的数据的区域。当绑定数组已满时，数据将传输到数据库。绑定数组大小由参数BINDSIZE和READSIZE 。

BINDSIZE给出的绑定数组的大小会覆盖默认大小（取决于系统）以及由ROWS.确定的任何大小ROWS.

限制

  • BINDSIZE参数仅用于传统路径加载。

例

8.2.5数据

SQL * Loader的DATA命令行参数指定包含要加载的数据的数据文件的名称。

默认

与控制文件同名，但扩展名为.dat 。

目的

DATA参数指定包含要加载的数据的数据文件的名称。

语法和说明

DATA = data_file_name

如果未指定文件扩展名，则默认值为.dat 。

文件规范可以包含通配符（仅在文件名和文件扩展名中，而不在设备或目录名中）。星号（*）代表多个字符和一个问号（？）代表一个字符。例如：

DATA ='emp * .dat'DATA ='m？emp.dat”

要列出多个数据文件规范（每个规范都可以包含通配符），文件名必须用逗号分隔。

如果文件名包含任何特殊字符（例如，空格，*，？），则整个名称必须用单引号括起来。

以下是DATA参数可能有效使用的三个示例（仅当文件名包含特殊字符时，才需要使用单引号）：

DATA ='file1'，'file2'，'file3'，'file4'，'file5'，'file6'DATA ='file1'，'file2'DATA ='file3，'file4'，'file5'DATA ='file6 “

8.2.6 DATE_CACHE

SQL * Loader的DATE_CACHE命令行参数指定日期缓存大小（在条目中）。

默认

已启用（适用于1000元素）。要完全禁用日期缓存功能，请将其设置为0 （零）。

目的

指定日期缓存大小（在条目中）。

日期缓存用于存储从文本字符串到内部日期格式的转换结果。缓存很有用，因为查找日期的成本远低于从文本格式转换为日期格式的成本。如果在日期文件中重复出现相同的日期，则使用日期缓存可以提高直接路径加载的速度。

语法和说明

DATE_CACHE = n

如果需要，每个表都有自己的日期缓存。仅当至少加载了一个需要数据类型转换的日期或时间戳值才能将其存储在表中之前，才会创建日期缓存。

默认情况下启用日期缓存功能。默认日期缓存大小为1000个元素。如果使用默认大小，并且加载的唯一输入值的数量超过1000，则会自动禁用该表的日期缓存功能。但是，如果覆盖缺省值，并指定非零日期高速缓存大小，并且超出该大小，则不会禁用高速缓存。

要调整缓存的大小以用于将来类似的加载，请使用日志文件中包含的日期缓存统计信息（条目，命中和未命中）。

限制

  • 日期缓存功能仅适用于直接路径和外部表加载。

例

以下规范完全禁用日期缓存功能。

DATE_CACHE = 0

8.2.7默认值

SQL * Loader的DEFAULTS命令行参数控制默认表达式的评估和加载。

默认

EVALUATE_ONCE ，除非涉及序列。如果涉及序列，则默认值为EVALUATE_EVERY_ROW 。

目的

控制默认表达式的评估和加载。

DEFAULTS参数仅适用于直接路径负载。

语法和说明

DEFAULTS = {IGNORE | IGNORE_UNSUPPORTED_EVALUATE_ONCE | IGNORE_UNSUPPORTED_EVALUATE_EVERY_ROW | EVALUATE_ONCE | EVALUATE_EVERY_ROW}

每个选项的行为如下：

  • IGNORE ：忽略列上的默认子句。

  • IGNORE_UNSUPPORTED_EVALUATE_ONCE ：在加载开始时评估一次默认表达式。不支持的默认表达式将被忽略。如果未使用DEFAULTS参数，则默认表达式将被计算一次，除非默认表达式引用序列，在这种情况下，将评估每一行。

  • IGNORE_UNSUPPORTED_EVALUATE_EVERY_ROW ：评估每一行中的默认表达式，忽略不支持的默认子句。

  • EVALUATE_ONCE ：在加载开始时评估一次默认表达式。如果未使用DEFAULTS参数，则默认表达式将被计算一次，除非默认值引用序列，在这种情况下，将评估每一行。对不受支持的默认表达式子句发出错误。（这是此参数的默认选项。）

  • EVALUATE_EVERY_ROW ：评估每行中的默认表达式，并针对不受支持的默认值发出错误。

例

此示例显示使用名称test创建表，以及名为test.ctl的SQL * Loader控制文件：

create table test（c0 varchar2（10），c1 number default'100'）; test.ctl：load data infile * truncate into table test fields以'，'尾随nullcols（c0 char）begindata 1结尾

要将NULL加载到c1 ，请发出以下语句：

sqlldr scott / password t.ctl direct = true defaults = ignore

要将默认值100加载到c1 ，请发出以下语句：

8.2.17 EXTERNAL_TABLE

EXTERNAL_TABLE参数指示SQL * Loader是否使用外部表选项加载数据。

默认

不曾用过

语法和说明

EXTERNAL_TABLE = [NOT_USED | GENERATE_ONLY |执行]

可能的值如下：

  • NOT_USED - 默认值。这意味着使用传统或直接路径模式执行负载。

  • GENERATE_ONLY - 使用外部表放置加载所需的所有SQL语句，如控制文件中所述，在SQL * Loader日志文件中。可以编辑和自定义这些SQL语句。通过在SQL * Plus中执行这些语句，可以在不使用SQL * Loader的情况下稍后完成实际加载。

  • EXECUTE - 尝试执行使用外部表进行加载所需的SQL语句。但是，如果任何SQL语句返回错误，则加载尝试将停止。语句在执行时放在日志文件中。这意味着如果SQL语句返回错误，则加载所需的其余SQL语句将不会放在日志文件中。

    如果使用EXTERNAL_TABLE=EXECUTE并在SQL * Loader控制文件中使用SEQUENCE参数，则SQL * Loader将创建数据库序列，使用该序列加载表，然后删除序列。以这种方式进行加载的结果将与使用传统或直接路径进行加载的结果不同。（有关创建序列的更多信息，请参阅Oracle Database SQL语言参考中的 CREATE SEQUENCE 。）

注意：

指定EXTERNAL_TABLE参数后，SQL * Loader控制文件中的任何日期时间数据类型（例如， TIMESTAMP ）将自动转换为CHAR数据类型，并使用外部表date_format_spec子句。请参见date_format_spec 。

请注意，外部表选项使用数据库中的目录对象来指示存储所有输入数据文件的位置，并指示创建输出文件（如坏文件和丢弃文件）的位置。您必须具有对包含数据文件的目录对象的READ访问权限，并且您必须具有对创建输出文件的目录对象的WRITE访问权限。如果数据文件或输出文件的位置没有现有目录对象，则SQL * Loader将生成SQL语句以创建一个。因此，指定EXECUTE选项时，您必须具有CREATE ANY DIRECTORY权限。如果希望在加载结束时删除目录对象，则还必须具有DROP ANY DIRECCTORY权限。

注意：

EXTERNAL_TABLE=EXECUTE限定符告诉SQL * Loader创建一个可用于加载数据的外部表，然后执行INSERT语句来加载数据。必须将外部表中的所有文件标识为位于目录对象中。SQL * Loader尝试使用已存在且您有权访问的目录对象。但是，如果SQL * Loader找不到匹配的目录对象，则它会尝试创建临时目录对象。如果您没有创建新目录对象的权限，则操作将失败。

要解决此问题，请使用EXTERNAL_TABLE=GENERATE_ONLY来创建SQL * Loader将尝试执行的SQL语句。提取这些SQL语句并将对目录对象的引用更改为您有权访问的目录对象。然后，执行这些SQL语句。

使用多表加载时，SQL * Loader执行以下操作：

  1. 在数据库中创建一个表，该表描述将加载到任何表中的输入数据文件中的所有字段。

  2. 创建一个INSERT语句，以从数据的外部表描述中加载此表。

  3. 为控制文件中的每个表执行一个INSERT语句。

要查看此示例，请运行案例研究5，但添加EXTERNAL_TABLE=GENERATE_ONLY参数。为了保证外部表中的唯一名称，SQL * Loader为所有字段使用生成的名称。这是因为字段名称在控制文件中的不同表中可能不唯一。

PARFILE SQL * Loader命令行参数指定在加载多个分区时要使用的内存量。

默认

0 （零）此设置根据PGA_AGGREGATE_TARGET初始化参数的值限制内存使用。当内存使用接近该值时，某些分区的加载会延迟。

目的

指定加载多个分区时要使用的内存量。在您加载的分区数量占用大量内存（甚至可能超过可用内存）的情况下，此参数很有用。（可能会发生此情况，尤其是在压缩数据时）。

达到指定的限制后，某些分区行的加载将延迟，直到内存使用率低于限制。

语法和说明

PARTITION_MEMORY = n

参数值n以千字节为单位。

如果n设置为0 （默认值），则SQL * Loader使用的值是PGA_AGGREGATE_TARGET初始化参数的函数。

如果n设置为-1 （减1），则SQL * Loader在加载许多分区时不会尝试使用更少的内存。

限制

  • 此参数仅对直接路径加载有效。

  • 此参数仅在Oracle Database 12c第1版（12.1.0.2）及更高版本中可用。

例

以下示例将内存使用限制为1 GB。

> sqlldr hr CONTROL = t.ctl DIRECT = true PARTITION_MEMORY = 1000000

默认

8.2.32 SDF_PREFIX

默认

没有默认值。

目的

指定目录前缀，该前缀添加到作为加载操作的一部分打开的LOBFILE和辅助数据文件（SDF）的文件名中。

8.2.33沉默

SILENT SQL * Loader命令行参数禁止在SQL * Loader操作期间写入屏幕的某些内容。

默认

没有默认值。

语法和说明

SILENT = [HEADER |反馈|错误|丢弃| PARTITIONS |所有]

使用适当的值来抑制以下一个或多个（如果指定了多个选项，则必须用逗号分隔）：

  • HEADER ：禁止通常出现在屏幕上的SQL * Loader标头消息。标头消息仍显示在日志文件中。

  • FEEDBACK ：抑制“达到提交点”消息和通常出现在屏幕上的负载的状态消息。但"XX Rows successfully loaded."甚至可以在屏幕上打印。

  • ERRORS ：抑制日志文件中的数据错误消息，这些消息在记录生成导致将其写入错误文件的Oracle错误时发生。仍会显示已拒绝记录的计数。

  • DISCARDS ：禁止写入丢弃文件的每条记录的日志文件中的消息。

  • PARTITIONS ：禁用在直接加载分区表期间将每分区统计信息写入日志文件。

  • ALL ：实现所有抑制值： HEADER, FEEDBACK, ERRORS, DISCARDS,和PARTITIONS. 但"XX Rows successfully loaded."甚至可以在屏幕上打印。

例

您可以使用以下命令行参数来抑制通常出现在屏幕上的标题和反馈消息：

SILENT = HEADER，FEEDBACK

但"XX Rows successfully loaded."甚至可以在屏幕上打印。

8.2.35 SKIP_INDEX_MAINTENANCE

SKIP_INDEX_MAINTENANCE SQL * Loader命令行参数指定是否停止直接路径加载的索引维护。

默认

假

目的

指定是否停止直接路径加载的索引维护。

SKIP_INDEX_MAINTENANCE = [TRUE |假]

8.2.36 SKIP_UNUSABLE_INDEXES

SKIP_UNUSABLE_INDEXES SQL * Loader命令行参数指定是否跳过在Index Unusable状态中遇到的索引并继续加载操作。

默认

初始化参数文件中指定的Oracle数据库配置参数SKIP_UNUSABLE_INDEXES的值。默认数据库设置为TRUE 。

目的

指定是否跳过在Index Unusable状态中遇到的索引并继续加载操作。

语法和说明

SKIP_UNUSABLE_INDEXES = [TRUE |假]

SKIP_UNUSABLE_INDEXES值为TRUE意味着如果遇到索引不可用状态的索引，则跳过该索引并继续加载操作。这允许SQL * Loader在加载开始之前加载具有处于Unusable状态的索引的表。SQL * Loader将维护在加载时不处于Unusable状态的索引。在加载时处于Unusable状态的索引将不会被维护，而是在加载完成时保持Unusable状态。

SQL * Loader和Oracle Database都提供SKIP_UNUSABLE_INDEXES参数。SQL * Loader SKIP_UNUSABLE_INDEXES参数在SQL * Loader命令行中指定。Oracle Database SKIP_UNUSABLE_INDEXES参数在初始化参数文件中指定为配置参数。重要的是要了解它们如何相互影响。

如果在SQL * Loader命令行中为SKIP_UNUSABLE_INDEXES指定值，则它将覆盖初始化参数文件中SKIP_UNUSABLE_INDEXES配置参数的值。

如果未在SQL * Loader命令行中为SKIP_UNUSABLE_INDEXES指定值，则SQL * Loader将使用SKIP_UNUSABLE_INDEXES配置参数的Oracle数据库设置，如初始化参数文件中所指定。如果初始化参数文件未指定SKIP_UNUSABLE_INDEXES的设置，则默认设置为TRUE 。

SKIP_UNUSABLE_INDEXES参数适用于传统和直接路径加载。

限制

  • 唯一且标记为Unusable的索引不允许跳过索引维护。此规则由DML操作强制执行，并由直接路径加载强制执行以与DML一致。

例

如果Oracle数据库初始化参数的值为SKIP_UNUSABLE_INDEXES=FALSE ，则在SQL * Loader命令行上设置SKIP_UNUSABLE_INDEXES=TRUE会覆盖它。因此，如果在设置此参数后遇到Index Unusable状态的索引，则跳过该索引，并继续加载操作。 

SKIP_UNUSABLE_INDEXES = TRUE
