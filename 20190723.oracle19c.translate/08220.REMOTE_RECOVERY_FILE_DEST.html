1.285 REMOTE_RECOVERY_FILE_DEST

REMOTE_RECOVERY_FILE_DEST指定在可插拔数据库（PDB）刷新操作期间从中读取存档日志文件的目录（如果源不可用）。

基本

没有

Oracle RAC

可以在不同实例上访问存档重做日志的目录位置可以不同，因此可以在不同实例上将此参数设置为不同的值。

要使PDB刷新副本与其源同步，可以通过数据库链接从源PDB访问重做。有时，当需要更新刷新副本时，无法访问源PDB所属的源PDB或CDB。在这些情况下，如果设置了此参数，则将尝试从此参数指定的目录中读取存档日志文件。
