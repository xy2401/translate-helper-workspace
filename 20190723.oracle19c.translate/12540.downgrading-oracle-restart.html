降级Oracle Restart

如果在安装期间收到错误，请使用此过程取消配置和降级Oracle Restart，或者对Oracle Restart进行故障排除。

要降级Oracle Restart，必须首先降级Oracle数据库，取消配置Oracle Restart，然后重新配置Oracle Restart。

此外，使用命令flags -deconfig -force运行roothas.sh可以在不删除已安装的二进制文件的情况下取消配置Oracle Restart。如果在Oracle Grid Infrastructure期间遇到针对独立服务器安装的错误，则此功能非常有用。例如，运行root.sh命令时，会发现缺少操作系统软件包。通过运行roothas.sh -deconfig -force ，您可以取消配置Oracle Restart，更正错误原因，然后再次运行root.sh

注意：

在取消配置或降级Oracle Restart之前，停止所有正在运行的数据库，服务和侦听器。
  1. 创建PFILE的备份作为oracle用户：
    CREATE PFILE ='/ u01 / app / oracle / product / 19.0.0 / dbhome_1 / dbs / test_init.ora'FROM SPFILE ='/ u01 / oracle / dbs / test_spfile.ora';
  2. 以root用户身份登录。
  3. 降级Oracle数据库。
  4. 降级Oracle Restart资源：
    db_unique_name downgrade database -d db_unique_name -o $ORACLE_HOME -t to_version
  5. 转到Grid_home /crs/install目录：
    #cd /u01/app/oracle/product/19.0.0/grid/crs/install
  6. 使用-deconfig -force标志运行roothas.sh以取消配置Oracle Restart。
    ＃roothas.sh -deconfig -force
  7. 使用deinstall命令以grid用户卸载Oracle Restart：
    $ cd /u01/app/19.0.0/grid/deinstall/ $ ./deinstall
    deinstall工具将Oracle Grid Infrastructure主页分离，更新OraInventory并清理目录。
  8. 以root用户身份手动运行root.sh ，在早期版本的Oracle Restart home中配置Oracle Restart。
    如果您的系统上没有早期版本的Oracle Restart，则为相应版本的独立服务器安装执行Oracle Grid Infrastructure以安装Oracle Restart。
  9. 将组件作为grid用户添加回Oracle Restart。
    1. 将Oracle数据库添加到Oracle Restart配置：
      $ srvctl add database -db db_unique_name -o $ORACLE_HOME
    2. 将侦听器添加到Oracle Restart配置：
      $ srvctl add listener -l listener_name -o $ORACLE_HOME
    3. 使用srvctl add service命令将每个服务添加到数据库。例如：
      $ srvctl add service -db mydb -service myservice
  10. 将Oracle主目录连接到Oracle Restart配置：
    $ ./runInstaller -silent -ignoreSysPrereqs -attachHome ORACLE_HOME =“/ u01 / app / oracle / product / 19.0.0 / dbhome_1”ORACLE_HOME_NAME =“Ora19c_home”
