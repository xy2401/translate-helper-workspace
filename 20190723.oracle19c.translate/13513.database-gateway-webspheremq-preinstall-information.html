4 Oracle Database Gateway for WebSphere MQ的预安装信息

以下主题将指导您完成Oracle Database Gateway for WebSphere MQ的基本概念和预安装步骤。

预安装任务

Oracle Database Gateway for WebSphere MQ的预安装任务分为以下几部分。

WebSphere MQ软件

本主题说明如何检查WebSphere MQ软件。

执行以下步骤以检查WebSphere MQ软件：

  1. 确定WebSphere MQ队列管理器的运行位置。

    • 本地系统

      如果WebSphere MQ队列管理器在本地系统上运行，则队列管理器在您打算安装网关产品集的同一系统上运行。

    • 远程系统

      如果WebSphere MQ队列管理器在远程系统上运行，则队列管理器在不同的系统上运行，而不是您打算安装网关产品集的系统。

  2. 验证是否已安装WebSphere MQ软件。如果WebSphere MQ服务器软件安装在与网关不同的系统上，则必须在网关系统上安装WebSphere MQ客户机软件。
  3. 标识WebSphere MQ队列管理器的名称。
  4. 确定WebSphere MQ客户机通道定义。

如果队列管理器安装在与网关不同的系统上，则使用WebSphere MQ客户机软件来访问远程队列管理器。此配置需要通道定义。

设置环境变量

在UNIX平台上安装Oracle Database for WebSphere MQ之前，请设置适当的环境变量。

注意：

验证分配给本主题中列出的环境变量的值是否少于42个字符。较长的值可能会在安装过程中产生错误，例如“Word太长”。

ORACLE_HOME

ORACLE_HOME是安装Oracle软件的根目录。

WebSphere Database Gateway for WebSphere MQ无法与其他Oracle产品共享同一Oracle主目录。如果已安装其他Oracle产品，则必须将Oracle Database Gateway for WebSphere MQ安装在不同的ORACLE_HOME目录中。

注意：

不要在包含其他Oracle产品（包括数据库）的ORACLE_HOME目录中安装Oracle Database Gateway for WebSphere MQ。这样的安装可能会覆盖共享组件，导致产品出现故障。

TMP

关于Oracle Universal Installer

Oracle Database Gateway for WebSphere MQ使用Oracle Universal Installer配置环境变量和安装组件。

Oracle Universal Installer将指导您完成安装过程的每个步骤，以便为自定义产品选择配置选项。

Oracle Universal Installer包含执行以下任务的功能：

  • 探索并提供产品的安装选项

  • 检测预设环境变量和配置设置

  • 在安装期间设置环境变量和配置

  • 卸载产品

oraInventory目录

Oracle Universal Installer首次在您的系统上运行时创建oraInventory目录。

oraInventory目录保存Oracle Universal Installer在系统上安装的产品清单以及其他安装信息。如果您以前安装过Oracle产品，那么您可能已经有了oraInventory目录。

  • 指定UNIX组名称时，它将授予该组写入oraInventory目录的权限。如果另一个组尝试运行Oracle Universal Installer，则它们必须具有写入oraInventory目录的权限。如果他们没有权限，安装将失败。

  • 运行Oracle Universal Installer的用户必须具有写入oraInventory目录及其所有文件的权限。这是运行安装程序所必需的。

  • oraInventory的位置在/etc/oratab/oraInst.loc为HP-UX Inanium和基于AIX的系统以及C:\Program Files\Oracle\Inventory\ for Microsoft Windows中定义。

  • oraInventory的位置在/var/opt/oraInst.loc中定义 Sun Solaris。

  • 最新的日志文件是oraInventory_location/logs/installActions.log在基于UNIX的系统上，以及Microsoft Windows的C:\Program Files\Oracle\Inventory\logs\installActions.log 。以前安装会话的日志文件名采用以下格式： installActions datetime .log 。

  • 请勿删除或手动更改oraInventory目录或其内容。这样做可能会阻止Oracle Universal Installer查找您在系统上安装的产品。

启动Oracle Universal Installer

本主题说明如何启动Oracle Universal Installer。

在基于UNIX的系统上，执行以下步骤以启动Oracle Universal Installer，后者将安装Oracle Database Gateway for WebSphere MQ：

  1. 停止所有Oracle进程和服务（例如，Oracle数据库）。

  2. 运行Oracle Universal Installer。

    注意：

    启动Oracle Universal Installer时，请确保您没有以root用户身份登录。如果是，则只有root用户才有权管理Oracle Database的WebSphere MQ。

    1. 以WebSphere MQ用户身份登录Oracle Database Gateway。

    2. 输入以下命令启动Oracle Universal Installer：

      提示> mount_point / runInstaller

在Microsoft Windows上，执行以下步骤以启动Oracle Universal Installer，后者将安装Oracle Database Gateway for WebSphere MQ：

  1. 启动系统并从操作系统Loader选项中选择MS（（nbsp））Windows。以Administrators组成员的身份登录到您的MS（（nbsp））Windows系统。

  2. 如果您是第一次安装网关，请确保在安装网关的磁盘上有足够的空间。

  3. 在安装软件之前，请停止正在运行的所有Oracle NT服务：

    1. 从“ 开始”菜单，转到“ 设置” ，然后转到“ 控制面板” ，再单击“ 服务” 。显示所有NT服务的列表。

    2. 选择Oracle NT服务（这些服务以Oracle开头）。

    3. 单击停止 。

    4. 继续选择并停止所有Oracle NT服务，直到停止所有活动的Oracle NT服务。

  4. 加载安装介质并启动Oracle Universal Installer。

这将启动Oracle Universal Installer，您可以使用它安装Oracle Database Gateway for WebSphere MQ。
