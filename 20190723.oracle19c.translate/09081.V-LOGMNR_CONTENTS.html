8.49 V $ LOGMNR_CONTENTS

V$LOGMNR_CONTENTS包含日志历史信息。要查询此视图，您必须具有LOGMINING权限。

对V$LOGMNR_CONTENTS视图执行SELECT语句时，将按顺序读取存档重做日志文件。重做日志文件中的翻译记录在V$LOGMNR_CONTENTS视图中作为行返回。这将继续，直到满足启动时指定的筛选条件（ EndTime或endScn ）或达到归档日志文件的末尾。

柱 数据类型 描述

SCN

数

进行数据库更改时的系统更改编号（SCN）

START_SCN

数

包含此更改的事务开始时的系统更改编号（SCN）;仅在DBMS_LOGMNR.START_LOGMNR()调用中选择COMMITTED_DATA_ONLY选项时才有意义，否则为NULL。如果查询在不包含事务开始的时间/ SCN范围内运行，则此列也可能为NULL。

COMMIT_SCN

数

提交事务时的系统更改号（SCN）;仅在DBMS_LOGMNR.START_LOGMNR()调用中选择了COMMITTED_DATA_ONLY选项时才有意义

日期

TX_NAME

操作

VARCHAR2（32）

进行更改的用户级SQL操作：

  • 内部 - 变更是由数据库发起的内部操作引起的

  • INSERT - 更改是由insert语句引起的

  • 删除 - 更改是由删除语句引起的

  • 更新 - 更改是由更新语句引起的

  • DDL - 更改是由DDL语句引起的

  • START - 更改是由交易开始引起的

  • COMMIT - 更改是由提交事务引起的

  • SEL_LOB_LOCATOR - Operation是一个返回LOB定位符的SELECT语句

  • LOB_WRITE - 更改是由DBMS_LOB.WRITE的调用引起的

  • LOB_TRIM - 更改是由DBMS_LOB.TRIM的调用引起的

  • SELECT_FOR_UPDATE - 操作是SELECT FOR UPDATE语句

  • LOB_ERASE - 更改是由DBMS_LOB.ERASE的调用引起的

  • MISSING_SCN - LogMiner在重做记录中遇到间隙。这很可能是因为并非所有重做日志都在LogMiner中注册。

  • ROLLBACK - 更改是由事务的完全回滚引起的

  • XML DOC BEGIN - 更改为XMLType列或表的开始

  • XML DOC WRITE - XML文档的数据

  • XML DOC END - XML文档的数据结束

  • UNSUPPORTED - 更改是由LogMiner当前不支持的操作引起的（例如，对嵌套表所做的更改）

此视图中的OPERATION和OPERATION_CODE列可用于顶级用户操作，例如DML和DDL。未记录这些列的值是LogMiner或RDBMS的内部值，不反映用户操作。

OPERATION_CODE

数

操作代码编号：

  • 0 - 内部

  • 1 - 插入

  • 2 - 删除

  • 3 - 更新

  • 5 - DDL

  • 6 - 开始

  • 7 - COMMIT

  • 9 - SELECT_LOB_LOCATOR

  • 10 - LOB_WRITE

  • 11 - LOB_TRIM

  • 25 - SELECT_FOR_UPDATE

  • 29 - LOB_ERASE

  • 34 - MISSING_SCN

  • 36 - ROLLBACK

  • 68 - XML DOC开始

  • 70 = XML DOC WRITE

  • 71 = XML DOC END

  • 255 - 不支持

ROLLBACK

SQL_REDO

VARCHAR2（4000）

重构的SQL语句，等同于进行更改的原始SQL语句。在将SQL_REDO执行到数据库之前，请参阅Oracle数据库实用程序 。

LogMiner不为临时表生成SQL重做。在这种情况下，此列将包含字符串"/* No SQL_REDO for temporary tables */" 。

SQL_UNDO

VARCHAR2（4000）

重构的SQL语句，可用于撤消进行更改的原始语句的效果。DDL语句没有相应的SQL_UNDO 。在将SQL_UNDO执行到数据库之前，请参阅Oracle数据库实用程序 。

LogMiner不会为临时表生成SQL撤消。在这种情况下，此列将包含字符串"/* No SQL_UNDO for temporary tables */" 。

RS_ID

VARCHAR2（32）

记录集ID。元组（ RS_ID ， SSN ）一起唯一地标识逻辑行更改。这通常意味着来自V$LOGMNR_CONTENTS一行，但如果重做或撤消的单个SQL语句太大而无法容纳在相应的SQL_UNDO或SQL_REDO列中， SQL_UNDO可能SQL_REDO 。 RS_ID唯一标识生成该行的重做记录。

SSN

数

SQL序列号。与RS_ID结合使用RS_ID ，它唯一标识逻辑行更改，显示为V$LOGMNR_CONTENTS视图中的一行或多行。

CSF

数

继续SQL标志。可能的值是：

  • 0 - 表示SQL_REDO和SQL_UNDO包含在同一行中

  • 1 - 表示SQL_REDO或SQL_UNDO大小超过4000字节，并在视图返回的下一行中继续

信息

VARCHAR2（64）

有关该行的信息性消息。例如，字符串“USER DDL”表示SQL_REDO列中返回的DDL语句是用户执行的顶级DDL，字符串“INTERNAL DDL”表示SQL_REDO列中返回的DDL语句是由内部执行的RDBMS。

状态

数

值为0表示SQL_REDO和SQL_UNDO列中显示的重构SQL语句是有效的可执行SQL语句。否则，重构的SQL语句不可执行。这可能是因为没有为LogMiner提供数据字典进行分析，或者提供的数据字典没有正在挖掘的对象的定义。

值为5表示此行是对XMLType列或表的更改的一部分，并且XML文档必须在应用之前进行汇编。
