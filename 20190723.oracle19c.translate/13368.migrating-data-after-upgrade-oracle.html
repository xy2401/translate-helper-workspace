8使用Oracle Data Pump迁移数据

使用Oracle Data Pump中的导出和导入实用程序将数据从一个数据库迁移到另一个数据库。

Oracle Data Pump提供高性能的Export（ expdp ）和Import（ impdp ）实用程序。这些实用程序便于升级到Oracle数据库。

数据泵概述和导出/导入以迁移数据

Oracle提供数据泵导出和导入，以将数据从一个Oracle数据库迁移（移动）到另一个Oracle数据库。

使用Oracle Data Pump迁移数据具有以下优势：

  • 支持使用INCLUDE和EXCLUDE参数过滤基于对象和对象类型导出和导入的元数据。

  • 支持卸载/加载数据库部分的不同模式，包括：完整数据库模式，模式模式，表模式，表空间模式和可传输表空间模式。

  • 使您可以使用PARTITION_OPTIONS参数指定在导入操作期间应如何处理分区表。

  • 提供对所有数据类型的支持。

在升级之前使用Oracle Data Pump迁移数据

在安装新的Oracle数据库软件之前，请使用此Oracle Data Pump过程从源数据库导出数据。然后将数据导入目标升级数据库。

  1. 安装新的Oracle数据库软件，并确保将其修补到最新的Oracle软件包修补程序或修补程序集更新（BP或PSU）。Oracle Database的安装步骤将在特定于操作系统的Oracle文档中介绍。

  2. 使用当前数据库附带的导出实用程序从当前数据库导出数据。

    导出期间和之后，当前数据库不能用于更新。如果导出后用户可以使用当前数据库进行更新，那么在使当前数据库可用之前，必须先安装过程，以便在导入完成后将当前数据库中所做的更改复制到新数据库。

    要获得可以更新导出数据库的一致点，请使用以下选项之一：

    • 在参数文件中设置FLASHBACK_TIME=SYSTIMESTAMP ，以便您导出的表中的数据获取的图像表示您要导出的所有表的相同单个时间点的表数据的已提交状态。

    • 使用FLASHBACK_SCN选择Export可用于启用闪回查询实用程序的特定系统更改编号（SCN）。

    使用闪回选项可增加UNDO的使用和保留。

  3. 创建新数据库。如果新数据库位于同一服务器上，并且它与当前数据库具有相同的名称，则在创建新数据库之前关闭当前数据库。

    （选项）您可以从源数据库更改存储参数。您可以在新数据库中预先创建表空间，用户和表，以通过更改存储参数来改善空间使用。使用SQL * Plus预创建表时，可以在原始数据库兼容模式下运行数据库，也可以考虑导入期间发生的特定数据定义转换。

  4. 在新的Oracle数据库环境中启动SQL * Plus，然后启动Oracle数据库实例。

    例如：

    $ SQLPLUS / AS SYSDBA SQL> STARTUP

  5. 如果您有预先创建的项目，请为数据泵导入指定TABLE_EXISTS_ACTION=APPEND选项。

    注意：

    如果在与源数据库相同的服务器上创建新数据库，并且您不想覆盖源数据库数据文件，则必须预先创建表空间并为数据泵导入设置以下参数值：

    • REUSE_DATAFILES=N表示数据泵导入

      （选项）您可以使用REMAP_DATAFILE, REMAP_TABLESPACE和REMAP_TABLE选项重新映射引用，以将转储文件集中的数据库名称导出到导入数据库中的新的非冲突名称。

    • DESTROY=N表示原始导入。

  6. 使用新数据库导入实用程序导入从当前数据库导出的对象。要将导入会话中的信息性消息和错误消息保存到文件，请使用以下参数：

    • 数据泵导入的LOGFILE参数

    • 原始Import的LOG参数

  7. 导入后，检查导入日志文件，以获取有关已成功完成的特定对象的导入的信息。如果出现故障，请检查有关任何失败对象的信息。

  8. 进一步使用导入方案，或使用创建数据库对象的SQL脚本来清理不完整的导入（或者可能启动全新的导入）。

    注意：

    如果数据泵导出或导入作业遇到不可恢复的错误，则可以在纠正导致故障的条件后重新启动作业。作业从故障点自动继续。

  9. 如果在导出后对当前数据库进行了更改，请确保在将这些更改提供给用户之前将这些更改传播到新数据库。请参阅此过程中的步骤1。

  10. 按照第4章“Oracle数据库的升级后任务”中的说明完成升级后所需的升级后任务。

使用网络链接导入完整的Oracle数据库

此数据库导出/导入方法是Oracle Data Pump迁移的替代方法，在迁移到其他存储系统时非常有用。

设置数据库链接并使用数据泵导入实用程序（ impdp ）执行从源数据库到目标数据库的完整数据库导入。使用此方法迁移数据意味着不会写入转储文件，因此您不必复制转储文件。使用不同的存储系统时，此方法特别有用。但是，您必须在Oracle数据库导出期间的早期版本Oracle Data Pump软件限制的范围内工作。

注意：

要避免互操作性错误，请确保在开始升级之前已将适当的补丁集应用于要升级的数据库。

  1. 确保源数据库中的导出用户具有DATAPUMP_EXP_FULL_DATABASE角色。

    您必须在创建数据库链接时指定此导出用户。

  2. 确保目标数据库中的导入用户具有DATAPUMP_IMP_FULL_DATABASE角色。

  3. 创建并测试源数据库和目标数据库之间的数据库链接。

  4. 使用以下命令语法启动数据泵导出，其中import_user是导入用户的用户名， db_link是导出用户拥有的数据库链接的名称：

    impdp import_user NETWORK_LINK = db_link FULL = Y NOLOGFILE = Y;

    注意：

    在导入数据库上运行此命令会隐式触发导出Oracle数据库上的数据泵导出操作（ expdp ）。

  5. 导入操作的日志文件写入DATA_PUMP_DIR目录。您可以通过运行以下命令来发现此目录的位置：

    SQL> select * from dba_directories，其中DIRECTORY_NAME喜欢'DATA_PUMP_DIR';

    在执行此命令时，请注意XDB存储库不会在完整数据库导出和导入操作中移动。LONG列和NESTED列也不会移动。但是，会移动用户创建的XML架构。

注意：

导入操作将在新目标服务器上重新创建用户。dba_users表中用户的创建日期显示实际的导入日期。到期日期更新为creation_date的值加上password_life_time列。目标数据库中dba_users表中的创建日期与源数据库中dba_users表的值不同。

降级Oracle数据库时的数据泵要求

您可以使用Data Pump获取向下兼容的转储文件。

将Oracle Data Pump与降级过程一起使用时，降级的Oracle数据库版本必须不超过要降级的版本的一个版本。

使用以下Data Pump Export方法之一获取向下兼容的转储文件：

  • 使用当前版本Oracle Database home中包含的Data Pump Export实用程序，并将VERSION参数设置为要降级的早期目标的版本。

    除非您使用VERSION参数设置为目标数据库的版本号创建了这些转储文件集，否则Data Pump Import无法读取由Oracle Data Pump版本创建的转储文件集。因此，执行降级的最佳方法是使用Data Pump Export，并将VERSION参数设置为要降级的目标数据库的版本号。

  • 使用带有VERSION参数的NETWORK_LINK参数将数据泵导出实用程序与DOWNGRADE一起使用。

    如果要降级的数据库的兼容级别早于您创建的导出转储文件集的版本，那么如果您希望从的Oracle数据库的兼容级别仍然可以通过数据库链接传输数据出口在两个主要版本号内。例如，如果一个数据库是Oracle Database 18c，则另一个数据库必须是12 c release 1（12.1）或12 c release 2（12.2）。您可以使用Data Pump Export以便在上一次转储文件导出期间将VERSION设置为不兼容的值进行恢复。

    注意：

    如果在安装新版本的Oracle数据库后提高兼容级别，以便它使用新版本中的功能，则使用这些新功能的对象无法降级。例如，如果使用Oracle Database 18c提供的长标识符，则具有这些长标识符的对象不能降级。降级后，当您尝试导入这些对象，并且数据泵导入实用程序尝试重新创建使用长标识符的对象时，您会收到错误。
