15.8更改Oracle Grid Infrastructure跟踪级别

通过启用跟踪级别，您可以收集足够的诊断信息来诊断问题的原因。

Oracle Support会要求您在重现问题时启用某些跟踪级别。

Oracle Trace File Analyzer使您可以轻松启用并禁用正确的跟踪级别。使用dbglevel选项设置跟踪级别。

您可以找到按问题跟踪配置文件分组的所需跟踪级别设置。

设置跟踪级别：

  1. 要设置跟踪配置文件：
    tfactl dbglevel -set profile
  2. 列出所有可用的配置文件：
    tfactl dbglevel -help
  • tfactl dbglevel
    使用tfactl dbglevel命令设置Oracle Grid Infrastructure跟踪级别。

15.8.1 tfactl dbglevel

使用tfactl dbglevel命令设置Oracle Grid Infrastructure跟踪级别。

句法

tfactl [run] dbglevel [{-set | -unset} profile_name dep1 [ dep1 ， dep2 ，... | all] dep2 [ type1 ， type2 ， type3 ，... | all] | {-view | -drop} profile_name | -lsprofiles | -lsmodules | -lscomponents [module_name] | -lsres | -create profile_name [-desc description | [-includeunset] [-includetrace] | -debugstate | - time ] | -modify profile_name [-includeunset] [-includetrace] | -getstate [-module module_name ] | -active [profile_name] | - 描述[profile_name]]]

参数

表15-5 tfactl dbglevel命令参数

参数 描述

PROFILE_NAME

指定配置文件的名称。

活性

显示活动配置文件列表。

组

设置指定配置文件的跟踪或日志级别。

未设置

取消设置指定配置文件的跟踪或日志级别。

视图

显示指定配置文件的跟踪或日志条目。

创建

创建个人资料。

下降

删除指定的配置文件。

修改

修改指定的配置文件。

描述

描述指定的配置文件。

lsprofiles

列出所有可用的配置文件。

lsmodules

列出所有已发现的CRS模块。

lscomponents

列出与CRS模块关联的所有组件。

lsres

列出所有已发现的CRS资源。

，有getstate

显示CRS组件或资源的当前跟踪或日志级别。

模

指定CRS模块。

依赖

指定要考虑，启动或停止依赖关系的依赖关系，或两者。

dependency_type

指定要考虑的依赖项类型。

debugstate

为所有可用级别生成系统状态转储。

includeunset

添加或修改CRS组件或资源的未设置值。

includetrace

添加或修改CRS组件的跟踪值。

警告：

仅在Oracle Support的指导下设置配置文件。
