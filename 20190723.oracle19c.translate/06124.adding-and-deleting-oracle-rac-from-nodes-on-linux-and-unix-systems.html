11在Linux和UNIX系统上从节点添加和删除Oracle RAC

将现有Oracle Real Application Clusters（Oracle RAC）主目录扩展到群集中的其他节点和实例，并从群集中的节点和实例中删除Oracle RAC。

如果您的目标是克隆现有Oracle RAC主目录以在群集中创建多个新的Oracle RAC安装，请使用“在新群集中克隆Oracle RAC到节点”中描述的克隆过程。

本章的主题包括以下内容：

  • 在安装了Oracle Clusterware的情况下将Oracle RAC添加到节点

  • 从群集节点删除Oracle RAC

注意：

  • 在通过运行ocrconfig -showbackup命令添加或删除Oracle RAC之前，请确保您具有Oracle Cluster Registry（OCR）的当前备份。

  • 本章中使用的短语“目标节点”指的是您计划扩展Oracle RAC环境的节点。

在安装了Oracle Clusterware的情况下将Oracle RAC添加到节点

在开始此过程之前，请确保现有节点具有到Grid_home的正确路径，并且$ORACLE_HOME环境变量设置为Oracle RAC主目录。

  • 如果您使用的是本地（非共享）Oracle主目录，则必须将现有节点（此过程中为node1 ）的Oracle RAC数据库主目录扩展到目标节点（此过程中为node3 ）。

    导航到node1上的Oracle_home /addnode目录并运行addnode.sh脚本。

    如果要执行静默安装，请使用以下语法运行addnode.sh脚本：

    $ ./addnode.sh -silent“CLUSTER_NEW_NODES = {node3}”

  • 如果您具有使用Oracle自动存储管理群集文件系统（Oracle ACFS）共享的共享Oracle主目录，请执行以下操作以将Oracle数据库主目录扩展到node3 ：

    1. 通过从Grid_home /bin目录以root身份运行以下命令，在新节点上启动Oracle ACFS资源：

      #srvctl start filesystem -device volume_device [-node node_name ]

      注意：

      确保Oracle ACFS资源（包括Oracle主服务器所在的Oracle ACFS注册表资源和Oracle ACFS文件系统资源）在新添加的节点上处于联机状态。

    2. 从要添加的节点上的Oracle_home /oui/bin目录中安装Oracle RAC的用户运行以下命令，以添加Oracle RAC数据库主目录：

      $ ./runInstaller -attachHome ORACLE_HOME =“ORACLE_HOME”“CLUSTER_NODES = {node3}”LOCAL_NODE =“node3”ORACLE_HOME_NAME =“ home_name ”-cfs

    3. 导航到node1上的Oracle_home /addnode目录，并使用以下语法以安装Oracle RAC的用户身份运行addnode.sh脚本：

      $ ./addnode.sh -noCopy“CLUSTER_NEW_NODES = {node3}”

      注意：

      使用-noCopy选项，因为目标节点上的Oracle主目录已经完全填充了软件。

  • 如果在不是 Oracle ACFS的共享文件系统上具有共享Oracle主目录，则必须首先在目标节点上为Oracle RAC数据库主目录创建装入点，装载并附加Oracle RAC数据库主目录，然后更新Oracle库存，如下：

    1. 在群集中的现有节点上运行srvctl config database -db db_name命令以获取装入点信息。

    2. 在node3以root身份运行以下命令以创建挂载点：

      #mkdir -p mount_point_path

    3. 装载承载Oracle RAC数据库主目录的文件系统。

    4. 从要添加的节点上的Oracle_home /oui/bin目录中安装Oracle RAC的用户运行以下命令，以添加Oracle RAC数据库主目录：

      $ ./runInstaller -attachHome ORACLE_HOME =“ORACLE_HOME”“CLUSTER_NODES = { local_node_name }”LOCAL_NODE =“ node_name ”ORACLE_HOME_NAME =“ home_name ”

    5. 以安装Oracle RAC的用户身份更新Oracle Inventory，如下所示：

      $ ./runInstaller -updateNodeList ORACLE_HOME = mount_point_path “CLUSTER_NODES = { node_list }”

      在上面的命令中， node_list是指安装了Oracle RAC数据库主目录的所有节点的列表，包括要添加的节点。

运行Oracle_home /root.sh上脚本node3为root 。

注意：

Oracle建议您在完成节点添加过程后备份OCR。

将策略管理的Oracle RAC数据库实例添加到目标节点

您必须手动添加撤消和重做日志，除非您将策略管理的数据库存储在Oracle自动存储管理（Oracle ASM）上并启用了Oracle托管文件 。

如果服务器池中有空间添加节点且数据库至少已启动一次，则Oracle Clusterware会将Oracle RAC数据库实例添加到新添加的节点，无需进一步操作。

注意：

在将数据库实例添加到新添加的节点之前，数据库必须至少启动一次。

如果任何服务器池中没有空间，则新添加的节点将移动到空闲服务器池中。使用srvctl modify srvpool命令增加服务器池的基数以容纳新添加的节点，之后节点移出Free服务器池并进入修改后的服务器池，Oracle Clusterware将Oracle RAC数据库实例添加到节点。

将管理员托管的Oracle RAC数据库实例添加到目标节点

注意：

本节中的过程仅适用于管理员管理的数据库。当节点在数据库的服务器池中可用时，策略管理的数据库使用节点。

您可以使用Oracle Enterprise Manager或DBCA将Oracle RAC数据库实例添加到目标节点。

本节介绍如何使用DBCA添加Oracle RAC数据库实例。

这些工具可指导您完成以下任务：

  • 在每个目标节点上创建新的数据库实例

  • 创建和配置高可用性组件

  • 从Oracle主目录为非默认侦听器创建Oracle Net配置

  • 启动新实例

  • 如果在“服务配置”页面上输入了服务信息，则创建和启动服务

将实例添加到目标节点后，应执行任何必要的服务配置过程，如“使用动态数据库服务进行工作负载管理”中所述 。

在交互模式下使用DBCA将数据库实例添加到目标节点

要在交互模式下使用DBCA将数据库实例添加到目标节点，请执行以下步骤：

  1. 确保现有节点将$ORACLE_HOME环境变量设置为Oracle RAC主目录。

  2. 通过在系统提示符下从Oracle_home /bin目录输入dbca启动DBCA。

    DBCA在运行时执行某些CVU检查。但是，您也可以从命令行运行CVU以执行各种验证。

    DBCA显示Oracle RAC的欢迎页面。单击任何DBCA页面上的“ 帮助”以获取其他信息。

  3. 选择Instance Management ，单击Next ，DBCA将显示Instance Management页面。

  4. 选择Add Instance ，然后单击Next 。DBCA显示“群集数据库列表”页面，其中显示数据库及其当前状态，例如ACTIVE或INACTIVE 。

  5. 从“群集数据库列表”页面中，选择要向其添加实例的活动Oracle RAC数据库。单击“ 下一步” ，DBCA将显示“群集数据库实例列表”页面，其中显示您选择的Oracle RAC数据库的现有实例的名称。

  6. 单击“ 下一步”以添加新实例，DBCA将显示“添加实例”页面。

  7. 在“添加实例”页面上，如果DBCA提供的实例名称与现有实例命名方案不匹配，请在此页面顶部的字段中输入实例名称。

  8. 查看“摘要”对话框中的信息，然后单击“ 确定”或单击“ 取消”以结束实例添加操作。DBCA显示进度对话框，显示DBCA执行实例添加操作。

  9. 终止DBCA会话后，运行以下命令以验证目标节点上的管理权限，并获取有关这些权限的详细信息，其中node_list包含添加数据库实例的节点的名称：

    cluvfy comp admprv -o db_config -d Oracle_home -n node_list [-verbose]

  10. 执行任何必要的服务配置过程，如“使用动态数据库服务进行工作负载管理”中所述 。

在无提示模式下使用DBCA将数据库实例添加到目标节点

您可以在静默模式下使用DBCA将实例添加到已扩展Oracle Clusterware主目录和Oracle数据库主目录的节点。

在运行dbca命令之前，请确保已在现有节点上正确设置ORACLE_HOME环境变量。运行DBCA，为表11-1中描述的变量提供值，如下所示：

dbca -silent -nodeList node_name -gdbName gdb_name [-instanceName instance_name -sysDBAUserName sysdba -sysDBAPassword password ]

变量 描述

gdb_name

INSTANCE_NAME

SYSDBA用户的密码。

从群集节点删除Oracle RAC

要从群集节点中删除Oracle RAC，必须先删除数据库实例和Oracle RAC软件，然后才能从群集中删除该节点。

注意：

如果要删除的节点上没有数据库实例，请继续执行“删除Oracle RAC” 。

本节包括从Oracle RAC环境中的群集中删除节点的以下过程：

  • 从Oracle RAC数据库中删除实例

  • 删除Oracle RAC

  • 从群集中删除节点

从Oracle RAC数据库中删除实例

对于策略管理和管理员管理的数据库，删除数据库实例的过程是不同的。

删除策略管理的数据库实例涉及减少数据库实例所在的服务器池中的服务器数。删除管理员管理的数据库实例涉及使用DBCA删除数据库实例。

删除策略管理的数据库

要删除策略管理的数据库，请通过将数据库实例所在的服务器重定位到另一个服务器池来减少数据库实例所在的服务器池中的服务器数。这有效地删除了实例，而无需从节点或节点中删除Oracle RAC软件。

例如，您可以通过在群集中的任何节点上运行以下命令来删除策略管理的数据库：

$ srvctl stop instance -db db_unique_name -node node_name $ srvctl relocate server -servers“ server_name_list ”-serverpool Free

第一个命令停止特定节点上的数据库实例，第二个命令将节点移出其当前服务器池并进入空闲服务器池。

从管理员管理的数据库中删除实例

注意：

在使用SRVCTL从Oracle RAC数据库中删除实例之前，请执行以下操作：

  • 如果已配置服务，则重新定位服务

  • 修改服务，以便每个服务可以在其余一个实例上运行

  • 确保要从管理员管理的数据库中删除的实例既不是任何服务的首选实例，也不是可用的实例

在交互模式下使用DBCA从节点中删除实例

本节中的过程说明如何在交互模式下使用DBCA从Oracle RAC数据库中删除实例。

要在交互模式下使用DBCA删除实例，请执行以下步骤：

  1. 启动DBCA。

    在承载要删除的实例的节点以外的节点上启动DBCA。在此步骤中，应该运行您计划删除的数据库和实例。

  2. 在“DBCA操作”页面上，选择“ 实例管理” ，然后单击“ 下一步” 。DBCA显示“实例管理”页面。

  3. 在“DBCA实例管理”页面上，选择要删除的实例，选择“ 删除实例”，然后单击“ 下一步” 。

  4. 在“群集数据库列表”页面上，选择要从中删除实例的Oracle RAC数据库，如下所示：

    1. 在“群集数据库实例列表”页面上，DBCA显示与您选择的Oracle RAC数据库关联的实例以及每个实例的状态。选择要从中删除实例的群集数据库。

    2. 在“确认”对话框中单击“ 确定 ”以继续删除实例。

      DBCA显示进度对话框，显示DBCA正在删除实例。在此操作期间，DBCA将删除实例和实例的Oracle Net配置。

      单击否并退出DBCA或单击是以执行其他操作。如果单击“ 是” ，则DBCA将显示“操作”页面。

  5. 通过在现有节点上使用SQL * Plus查询GV$LOG视图，验证是否已删除已删除实例的重做线程。如果未禁用重做线程，则禁用该线程。例如：

    SQL> ALTER DATABASE DISABLE THREAD 2;

  6. 通过运行以下命令验证是否已从OCR中删除实例，其中db_unique_name是Oracle RAC数据库的数据库唯一名称：

    $ srvctl config database -db db_unique_name

  7. 如果要删除多个节点，请重复这些步骤以从要删除的所有节点中删除实例。

在静默模式下使用DBCA从节点中删除实例

您可以在静默模式下使用DBCA从节点中删除数据库实例。

运行以下命令，其中变量与表11-1中显示的变量相同，用于添加实例的DBCA命令。仅当您从正在运行DBCA的节点以外的节点中删除实例时才提供节点名称，如以下示例所示，其中password是密码：

dbca -silent -deleteInstance [-nodeList node_name ] -gdbName gdb_name -instanceName instance_name [-sysDBAUserName sysdba -sysDBAPassword password ]

此时，您已完成以下操作：

  • 从其关联的Oracle Net Services侦听器中注销所选实例

  • 从实例的已配置节点中删除选定的数据库实例

  • 删除了Oracle Net配置

  • 从实例的已配置节点中删除了Oracle Flexible Architecture目录结构。

删除Oracle RAC

此过程从要从群集中删除的节点中删除Oracle RAC软件，并更新其余节点上的清单。

  1. 如果要删除的节点上的Oracle RAC主目录中有侦听器，则必须先禁用并停止它，然后再删除Oracle RAC软件。在群集中的任何节点上运行以下命令，指定侦听器的名称和要删除的节点的名称：

    $ srvctl disable listener -l listener_name -n name_of_node_to_delete $ srvctl stop listener -l listener_name -n name_of_node_to_delete

  2. 如果未通过Oracle_home \deinstall目录运行以下命令从要删除的节点共享Oracle主目录，则卸载Oracle主目录：

    deinstall -local

    警告：

    如果共享Oracle主目录，则不要运行此命令，因为它将删除共享软件。相反，继续下一步。

从群集中删除节点

删除数据库实例和Oracle RAC软件后，可以开始从群集中删除节点的过程。您可以通过在要删除的节点上运行脚本来删除Oracle Clusterware安装，然后在其余节点上运行脚本来更新节点列表。
