22.1 SDO_CS.ADD_PREFERENCE_FOR_OP

格式

SDO_CS.ADD_PREFERENCE_FOR_OP（op_id IN NUMBER，source_crs IN NUMBER DEFAULT NULL，target_crs IN NUMBER DEFAULT NULL，use_case IN VARCHAR2 DEFAULT NULL）;

描述

为源坐标系和目标坐标系之间的操作添加首选项。

参数

op_id

操作的ID号。必须是SDO_COORD_OPS表的COORD_OP_ID列中的值（在SDO_COORD_OPS表中描述）。

source_crs

源坐标参考系的SRID。必须为空或SDO_COORD_REF_SYS表的SRID列中的值（在SDO_COORD_REF_SYS表中描述）。

target_crs

目标坐标参考系的SRID。必须为空或SDO_COORD_REF_SYS表的SRID列中的值（在SDO_COORD_REF_SYS表中描述）。

use_case

与此首选项关联的用例的名称。必须为空或SDO_PREFERRED_OPS_USER表的USE_CASE列中的值（在SDO_PREFERRED_OPS_USER表中描述）。

使用说明

如果use_case为null，则与操作关联的转换计划是系统范围的首选项，并且向SDO_PREFERRED_OPS_SYSTEM表（在SDO_PREFERRED_OPS_SYSTEM表中描述）中添加一行（或者如果存在反向操作则添加两行）。如果use_case不为null，则与该操作关联的转换计划是与指定用例关联的首选项，并且向SDO_PREFERRED_OPS_USER表添加一行（如果存在反向操作，则添加两行）（在SDO_PREFERRED_OPS_USER表中描述） 。

要创建连接操作并使其首选系统范围或指定用例，可以使用SDO_CS.CREATE_PREF_CONCATENATED_OP便捷程序。

要撤消源坐标系与目标坐标系之间的操作的首选项，请使用SDO_CS.REVOKE_PREFERENCE_FOR_OP过程。

例子

以下示例添加了操作19977的首选项，以便在为转换指定用例use_case_B时在从SRID 4301到SRID 4326的转换中使用。

EXECUTE SDO_CS.ADD_PREFERENCE_FOR_OP（19977,4301,4326，'use_case_B'）;

22.2 SDO_CS.CONVERT_3D_SRID_TO_2D

格式

SDO_CS.CONVERT_3D_SRID_TO_2D（在数字中为srid3d）返回号码;

描述

将三维SRID值转换为二维SRID值。

参数

srid_3d

三维坐标参照系的SRID。必须是SDO_COORD_REF_SYS表的SRID列中的值（在SDO_COORD_REF_SYS表中描述）。

使用说明

此函数返回与输入坐标参考系统对应的功能等效的二维坐标参考系的SRID值。

  • 如果输入复合3D SRID，则只有一个对应的二维SRID，并返回它。

  • 如果输入了地理3D SRID，则可能存在多个可能的对应二维SRID，并且返回其中一个。

  • 如果输入了二维SRID，则返回它。

有关三维坐标参考系统支持的信息，请参阅三维坐标参考系统支持 。

例子

22.8 SDO_CS.CREATE_OBVIOUS_EPSG_RULES

格式

SDO_CS.CREATE_OBVIOUS_EPSG_RULES（use_case IN VARCHAR2 DEFAULT NULL）;

描述

创建要在某些转换中应用的基本EPSG规则集。

参数

use_case

与创建的EPSG规则的应用程序关联的用例的名称。必须是SDO_PREFERRED_OPS_USER表的USE_CASE列中的值（在SDO_PREFERRED_OPS_USER表中描述）。

使用说明

此过程创建规则以实现特定坐标参考系统之间的主要EPSG定义的转换。对于某些坐标参考系统之间的转换，EPSG指定了应该应用的规则。对于从一个坐标参考系统到另一个坐标参考系统的任何给定转换，EPSG规则可能与默认的Oracle Spatial and Graph规则不同。如果执行此过程，则在任何此类情况下都应用EPSG规则。如果不执行此过程，则在这种情况下使用默认的空间和图形规则。

此过程将许多行插入SDO_PREFERRED_OPS_SYSTEM表（请参阅SDO_PREFERRED_OPS_SYSTEM表 ）。

要删除此过程创建的EPSG规则，从而导致在所有情况下都使用默认的空间和图形规则，请使用SDO_CS.DELETE_ALL_EPSG_RULES过程。

例子

以下示例创建要在某些转换中应用的基本EPSG规则集。

EXECUTE SDO_CS.CREATE_OBVIOUS_EPSG_RULES;

22.9 SDO_CS.CREATE_PREF_CONCATENATED_OP

格式

SDO_CS.CREATE_PREF_CONCATENATED_OP（op_id IN NUMBER，op_name IN VARCHAR2，use_plan IN TFM_PLAN，use_case IN VARCHAR2 DEFAULT NULL）;

描述

创建连接操作，将其与转换计划相关联，并使其在系统范围内或针对指定的用例进行首选。

参数

op_id

要创建的连接操作的ID号。

op_name

与连接操作关联的名称。

use_plan

转型计划。TFM_PLAN对象类型在TFM_PLAN对象类型中进行了说明 。

use_case

此首选连接操作适用的用例。必须是空值或SDO_PREFERRED_OPS_USER表的USE_CASE列中的值（在SDO_PREFERRED_OPS_USER表中描述）。

使用说明

此便捷程序结合了SDO_CS.CREATE_CONCATENATED_OP和SDO_CS.ADD_PREFERENCE_FOR_OP过程的操作。

连接操作是两个或多个原子操作的串联（链接）。

22.20 SDO_CS.GENERATE_SCRIPT_FROM_SRID

格式

GENERATE_SCRIPT_FROM_SRID（s NUM IN NUMBER，偏移IN NUMBER DEFAULT 0，include_units IN NUMBER DEFAULT 1）RETURN CLOB;

描述

返回一个CLOB对象，该对象包含使用指定的SRID值创建坐标系所需的SQL语句。

参数

srid

坐标参照系的SRID。必须是SDO_COORD_REF_SYS表的SRID列中的值（在SDO_COORD_REF_SYS表中描述）。

偏移量

要添加到由生成的脚本创建的坐标系的SRID值的数字。例如，指定SDO_CS.GENERATE_SCRIPT_FROM_SRID( 27700, 50000 )将导致INSERT语句生成的脚本将SRID指定为77700 （即27700 + 50000）。目的可能是确保在您知道当前SRID不等于或大于77700时生成新的唯一SRID。

include_units

数值0表示不在生成的语句中包含度量单位;数值1（默认值）表示在生成的语句中包含度量单位。

几乎在所有情况下都建议使用默认值。例外（如果有的话）应该是罕见的，并且只有在属性存在“非标准”度量单位时才是如此。

使用说明

在使用此函数之前，必须使用SQL * Plus命令SET LONG来增加列输出的最大字节宽度。例如： SET LONG 20000

如果您计划使用输出来帮助您修改坐标系定义或创建新定义 - 而不是仅查看信息 - 您必须根据需要编辑输出以确保语法正确（例如INSERT语句） ）。

该脚本可以在不同的（目标）数据库上运行，也可以在执行此功能的同一数据库上运行。在任何一种情况下，可能存在与所讨论的SRID相关联的现有坐标系;在这种情况下，您可能希望采取措施来处理该场景（例如使用offeet参数）。

不鼓励您更改“标准”坐标系定义。

22.43 SDO_CS.VALIDATE_WKT

格式

SDO_CS.VALIDATE_WKT（srid IN NUMBER）RETURN VARCHAR2;

描述

验证与指定SRID关联的已知文本（WKT）描述。

参数

srid

要验证其着名文本（WKT）描述的坐标系的SRID。SDO_COORD_REF_SYS表中必须存在指定值的条目（在SDO_COORD_REF_SYS表中描述）。

使用说明

如果WKT描述有效，则此函数返回字符串'TRUE'。如果WKT描述无效，则此函数返回格式为'FALSE（<position-number>）'的字符串，其中<position-number>是WKT描述中出现第一个错误的字符位置编号。

检查WKT描述以查看它是否满足知名文本（WKT）中描述的要求。

例子

以下示例验证与SRID 81989000关联的坐标系的WKT描述。结果表明，无效的原因（或无效的第一个原因）开始于WKT描述中的字符位置181。（SRID 81989000与任何已建立的坐标系无关。相反，它是用于插入到MDSYS.CS_SRS表的测试版本中的故意无效坐标系，并且它不包含在Oracle Spatial and Graph附带的MDSYS.CS_SRS表中。）
