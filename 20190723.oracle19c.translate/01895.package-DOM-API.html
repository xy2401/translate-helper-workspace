3为C ++打包DOM API

DOM包中的各种接口表示由W3C指定的DOM级别2核心接口： http://www.w3.org/TR/DOM-Level-2-Core/ ： http://www.w3.org/TR/DOM-Level-2-Core/ 。

关于DOM接口

DOM接口表示为对DOM规范的不同实现的通用引用。它们由Node参数化，Node支持各种特化和实例化。其中，最重要的是xmlnode ，它对应于当前的C实现。

这些泛型引用没有类似NULL的值。任何实现都不应该创建无状态引用。如果需要发出某些事物没有状态的信号，则应该抛出异常。

如果DOM树UNDEFINED_ERR不正确，许多方法可能抛出SYNTAX_ERR异常，如果参数错误或意外的NULL指针，则抛出UNDEFINED_ERR 。如果这些是特定方法可能抛出的唯一错误，则不会反映在方法签名中。

实际DOM树不依赖于上下文（ TCtx ）。但是，在当前基于xmlctx的实现中对DOM树的操作需要访问当前上下文（ TCtx ）。这是通过将上下文指针传递给DOMImplRef的构造函数来完成的。在多线程环境中， DOMImplRef始终在线程上下文中创建，因此具有指向正确上下文的指针。

DOMImplRef了一种创建DOM树的方法。DomImplRef是对实际DOMImplementation对象的引用，该对象是在调用DomImplRef的常规非复制构造函数时DomImplRef的。这在需要共享DOM树的多线程环境中运行良好，并且每个线程都有一个与之关联的单独TCtx。这在单线程环境中同样有效。

DOMString只是Oracle实现支持的编码之一。其他编码的支持是Oracle的扩展。oratext*数据类型用于所有编码。
