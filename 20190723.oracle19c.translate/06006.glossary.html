词汇表

ASMLib程序

ASMLIB是Oracle开发的一个应用程序编程接口（API），用于简化操作系统到数据库的接口，并在基于Linux的系统上利用供应商存储阵列的功能。

Oracle ASM过滤器驱动程序是ASMLIB的推荐替代品。有关Oracle ASM筛选器驱动程序的信息，请参阅“ Oracle ASM筛选器驱动程序 ” 。

集群文件系统（CFS）

集群文件系统（CFS）是分布在多个计算机系统上的文件系统。通常，计算机系统共享对通过存储区域网络（SAN）连接的磁盘的访问。每个计算机系统上的CFS组件使用全局通信机制协调对磁盘的访问。

群集同步服务（CSS）

群集同步服务（CSS）为跨一组节点的协调的进程间通信活动提供基础。CSS服务包括组服务，锁定服务，节点信息和群集配置服务，以及检测加入和离开群集的节点的节点监视服务。

CSS群集

CSS集群是CSS集群成员资格服务定义的集群配置。CSS维护Oracle集群的权威成员资格，并监视加入和离开集群配置的节点。对于给定的节点集，最多只定义一个CSS集群。

磁盘组

Oracle ASM磁盘组是Oracle ASM作为一个单元管理的磁盘集合。在磁盘组中，Oracle ASM公开Oracle Database文件的文件系统接口。存储在磁盘组中的文件内容均匀分布或条带化，以消除热点并在磁盘上提供统一的性能。Oracle ASM文件也可以选择在磁盘组中进行镜像。磁盘组中磁盘的性能与原始设备的性能相当。

直接附加存储（DAS）

直接连接存储（DAS）由直接连接到主机而没有中间网络的存储设备组成。DAS通常比SAN附加存储成本低，但也提供较少的灵活性和功能。DAS的示例包括大多数桌面系统常用的SATA磁盘和许多服务器上的SCSI磁盘。

程度

范围（也称为数据范围）是用于保存文件内容的原始存储。每个范围由特定磁盘上的一个或多个分配单元（AU）组成。

范围图

范围映射是指向文件的所有数据范围的范围指针列表。这是文件的原始存储。每个范围指针都给出数据范围的磁盘和分配单元。为了可靠性，每个扩展指针还包括一个校验字节，以确保它没有被损坏。当使用扩展区映射的内存中副本将文件I / O定向到特定磁盘位置时，这一点尤其重要。

光纤通道

光纤通道是一组标准，用于定义计算机系统和外围设备之间的接口。光纤通道接口是一个串行总线接口，最初设计用于取代SCSI标准。但是，光纤通道和SCSI接口都是彼此独立发展的，并且受益于另一个的存在。纤维是拼写重新而非ER以指示从诸如光纤区分。铜和光纤通常用作光纤通道的介质。

文件系统

文件系统是提供对磁盘的结构化访问的软件组件。文件系统向应用程序呈现对象，例如文件。通常使用标准API来定义对文件的访问，该API定义操作系统调用，例如应用程序用于访问文件的打开/关闭和读/写。文件系统通常作为操作系统的组件提供，但可以作为独立的软件组件提供。

文件

文件是由文件系统呈现给应用程序的对象。文件系统将文件细分为块。文件系统通常将应用程序中显示的内容作为连续块放入磁盘上的非连续位置。文件块到磁盘上的位置的映射保留在所谓的扩展区映射中 。

主机总线适配器（HBA）

主机总线适配器（HBA）或主机适配器将主机系统连接到其他网络和存储设备，例如eSATA和SCSI设备。

逻辑单元号（LUN）

在存储技术中，术语LUN通常用于表示由存储阵列呈现给计算机系统的磁盘。在SCSI术语中，逻辑单元号或LUN是表示呈现给计算机系统的设备地址的数字。

镜像

在存储系统中，镜像是一种通过在多个磁盘上存储数据副本来保护数据完整性的方法。如果磁盘发生故障，则可在第二个或第三个磁盘上获得数据的辅助副本。通常，镜像可以以两种方式之一部署。在最常见的情况下，与RAID存储阵列一样，单个磁盘上的固定数量的物理存储空间均匀地复制到多个磁盘上。通过存储阵列上的软件，多个设备上的物理镜像空间作为单个磁盘呈现给主机。另一种镜像方法是将文件的固定段复制到多个磁盘上。Oracle ASM使用第二种方法实现最大的灵活性。

网络附加存储（NAS）

网络附加存储（NAS）包括一类向主机提供文件服务的系统。操作系统中的设备驱动程序使用NFS或通用Internet文件系统（CIFS）等协议访问数据。从应用程序的角度来看，网络附加存储显示为应用程序所在的主机本地的文件系统。

网络文件系统（NFS）

网络文件系统（NFS）是在许多平台上提供网络连接存储的行业标准。NFS扩展了本地文件系统框架，以便为IP网络上的用户提供设备共享。NFS的用户视图是主机上的远程文件系统显示为本地安装。

Oracle自动存储管理群集

Oracle自动存储管理（Oracle ASM）集群是一组互连节点，每个节点都有一个Oracle ASM实例，使用Oracle Clusterware作为统一集群运行。Oracle ASM集群向一个或多个也在节点上运行的Oracle数据库提供共享存储池。也可以使用Oracle Real Application Clusters对数据库进行集群，但这不是必需的。必须可以从Oracle ASM群集中的所有节点访问提供共享存储池的磁盘。

划分

操作系统通常提供将磁盘分成称为分区的部分的手段。对于应用程序，分区显示为独立磁盘，即使不是这种情况。操作系统提供用于管理磁盘上分区的大小和位置的命令。此命令将分区映射写入磁盘中应用程序不可见的区域。

冗余廉价磁盘阵列（RAID）

冗余廉价磁盘阵列（RAID）是一种通过软件组合多个磁盘并将其作为明显不同磁盘集合呈现给主机的方法。镜像和条带化是用于将磁盘组合为集合的两种方法。RAID可以实现为计算机系统的软件组件或者在存储阵列内部操作的软件。RAID在物理存储方式上运行，而不是在逻辑文件级别进行Oracle ASM镜像和条带化。

存储区域网络（SAN）

存储区域网络（SAN）是专门用于将存储设备（如磁盘阵列和磁带驱动器）连接到计算机系统的网络。SAN中最常用的网络基础设施是光纤通道 。通过允许任何存储设备可以连接到任何计算机系统的通用互连基础设施，SAN提供了极大的灵活性。分区和LUN屏蔽通常用于SAN的访问控制。分区是SAN基础结构确定存储设备对计算机系统的访问权限的机制。LUN掩码是一种机制，其中存储设备确定允许哪些计算机系统连接到设备的访问权限。

小型计算机系统接口（SCSI）

小型计算机系统接口或SCSI是定义计算机系统和外围设备（最常见的是存储设备）之间接口的标准。SCSI标准定义了计算机系统和外围设备之间接口的逻辑和物理方面。

条带化

在存储系统中，条带化是一种在多个磁盘上传播数据的方法，而不是将数据存储在单个磁盘上。通常进行条带化以提高性能。通常，条带化可以以两种方式之一部署。在最常见的情况下，与RAID存储阵列一样，可以存储在单个磁盘上的固定数量的物理存储空间在多个磁盘上统一分条。通过存储阵列上的软件，多个设备上的物理条带空间作为单个磁盘呈现给主机。条带化的另一种方法是在连接到主机的多个磁盘上对文件的固定段进行条带化。Oracle ASM使用第二种方法来获得最佳性能。

体积

在存储领域，卷的含义有许多相关的定义。卷通常被视为表示为逻辑单元号（LUN）的虚拟实体。卷通常表示为来自多个磁盘的碎片的聚合。卷由称为卷管理器的软件组件管理 。

卷管理员

卷管理器是一个软件组件，用于管理磁盘集合到卷中的映射。
