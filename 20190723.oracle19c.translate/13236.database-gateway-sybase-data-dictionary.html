C数据字典

Oracle Database Gateway for Sybase将引用Oracle数据库数据字典表的查询转换为从Sybase系统表中检索数据的查询。您可以通过数据库链接对数据字典表执行查询，方法与查询Oracle数据库中的数据字典表的方式相同。网关数据字典在外观和使用上类似于Oracle数据库数据字典。

  • Sybase系统表

  • 访问网关数据字典

Sybase系统表

Sybase数据字典信息作为Sybase系统表存储在Sybase数据库中。所有Sybase系统表的名称都以“sys”为前缀。Sybase系统表定义数据库的结构。更改数据定义时，Sybase将读取并修改Sybase系统表以添加有关用户表的信息。

访问网关数据字典

访问网关数据字典表或视图与访问Oracle数据库中的数据字典完全相同。您发出SQL 指定数据库链接的SELECT语句。Oracle数据库数据字典视图和列名称用于访问Oracle数据库中的网关数据字典。支持的视图的同义词也是可以接受的。例如，以下语句查询数据字典表ALL_CATALOG以检索Sybase数据库中的所有表名：

SQL> SELECT * FROM“ALL_CATALOG”@SYBS;

发出数据字典访问查询时，网关：

  1. 将请求的表，视图或同义词映射到一个或多个Sybase系统表名称。网关将所有数据字典列名称转换为查询中对应的Sybase列名称。如果映射涉及一个Sybase系统表，则网关会将请求的表名转换为查询中对应的Sybase系统表名。如果映射涉及多个Sybase系统表，则网关使用转换的Sybase系统表名在查询中构造连接。
  2. 将已翻译的查询发送到Sybase。
  3. 可能会转换检索到的Sybase数据，使其具有Oracle数据库数据字典表的外观。
  4. 将数据字典信息从已转换的Sybase系统表传递到Oracle数据库。

    注意：

    查询网关数据字典时返回的值可能与Oracle SQL * Plus DESCRIBE命令返回的值不同。

直接查询Sybase表

允许直接发布到各个Sybase系统表的查询，但它们返回不同的结果，因为Sybase系统表列名称与数据字典视图的名称不同。此外，Sybase系统表中的某些列不能用于数据字典处理。

默认列值

网关数据字典与典型的Oracle数据库数据字典之间存在细微差别。Sybase系统表中缺少的Oracle数据库列将填充零，空格，空值，不适用的值（NA）或默认值，具体取决于列类型。
