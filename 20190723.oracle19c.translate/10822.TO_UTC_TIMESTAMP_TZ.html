TO_UTC_TIMESTAMP_TZ

句法


插图描述to_utc_timestamp_tz.eps

目的

SQL函数TO_UTC_TIMESTAMP_TZ将ISO 8601日期格式字符串作为varchar输入，并返回SQL数据类型TIMESTAMP WITH TIMEZONE的实例。它将输入标准化为UTC时间（协调世界时，以前格林威治标准时间）。与SQL函数TO_TIMESTAMP_TZ不同，新函数假定输入字符串使用ISO 8601日期格式，将时区默认为UTC 0。

如果输入varchar不符合ISO 8601格式，您将看到ORA-40688错误。

此函数的典型用法是将其输出提供给SQL函数SYS_EXTRACT_UTC ，获取UTC时间，然后将其作为SQL绑定变量传递给SQL / JSON条件JSON_EXISTS ，以执行时间戳范围比较。

这是日期和时间的允许语法：

  • 日期（仅）： YYYY - MM - DD

  • 日期与时间： YYYY - MM - DD T hh : mm : ss [ . s [ s [ s [ s [ s [ s ]]]]][ Z |( + | - ) hh : mm ]

哪里：

  • YYYY指定年份 ，为四位小数。

  • MM指定月份 ，为两位小数， 00到12 。

  • DD将日期指定为两位十进制数字00到31 。

  • hh指定小时 ，为两位小数， 00到23 。

  • mm指定分钟 ，为两位小数， 00到59 。

  • ss [ . s [ s [ s [ s [ s ]]]]]指定秒 ，作为两个十进制数字， 00到59 ，可选地后跟一个小数点和1到6个十进制数字（表示秒的小数部分）。

  • Z指定UTC时间（时区0）。（它也可以由+00:00指定，但不能通过–00:00 。）

  • ( + | - )hh : mm指定时区与UTC的差异 。 （+或–是必需的。）

对于时间值，时区部分是可选的。如果不存在则假定UTC时间。

不支持其他ISO 8601日期时语法。特别是：

  • 不支持负日期（公元前1年之前的日期），以连字符开头（例如– 2018–10–26T21:32:52 ）。

  • 需要连字符和冒号分隔符：不支持所谓的“基本”格式YYYYMMDDThhmmss 。

  • 不支持序数日期（年份加上一年中的某一天，日历周加上日期编号）。

  • 不支持使用年份超过四位数。

支持的日期和时间包括以下内容：

  • 2018-10-26T21：32：52

  • 2018-10-26T21：32：52 + 02：00

  • 2018-10-26T19：32：52Z

  • 2018-10-26T19：32：52 + 00：00

  • 2018-10-26T21：32：52.12679

不支持的日期和时间包括以下内容：

  • 2018-10-26T21:32 （如果指定了时间，那么它的所有部分必须存在）

  • 2018-10-26T 25 :32:52+02:00 （小时部，25，超出范围）

  • 18-10-26T21:32 （未完全指定年份）

例子

SELECT TO_UTC_TIMESTAMP_TZ（'1998-01-01'）FROM DUAL; TO_UTC_TIMESTAMP_TZ（'1998-01-01'）---------------------------------------- ----------------------------------- 01-JAN-98 12.00.00.000000000 AM +00：00
从TOAL中选择TO_UTC_TIMESTAMP_TZ（'2000-01-02T12：34：56.789'）; TO_UTC_TIMESTAMP_TZ（'2000-01-02T12：34：56.789'）------------------------------------ --------------------------------------- 02-JAN-00 12.34.56.789000000 PM +00 ：00
SELECT TO_UTC_TIMESTAMP_TZ（'2016-05-05T00：00：00.000Z'）FROM DUAL; TO_UTC_TIMESTAMP_TZ（'2016-05-05T00：00：00.000Z'）----------------------------------- ---------------------------------------- 05-MAY-16 12.00.00.000000000 AM + 00:00
SELECT TO_UTC_TIMESTAMP_TZ（'2016-05-05T02：04：35.4678Z'）FROM DUAL; TO_UTC_TIMESTAMP_TZ（'2016-05-05T02：04：35.4678Z'）----------------------------------- ---------------------------------------- 05-MAY-16 02.04.35.467800000 AM + 00:00
