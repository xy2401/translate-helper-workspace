B.2 UTF-16编码

如表B-1 ，UTF-16所示某些字符的字符代码（附加的中文/日文/韩文字符和专用区＃2）以两个16位为单位表示。这些是补充字符。补充字符由两个16位值组成。第一个16位值的编码范围为0xD800到0xDBFF。第二个16位值的编码范围为0xDC00至0xDFFF。对于补充字符，UTF-16字符代码可以表示超过一百万个字符。没有增补字符，只能表示65,536个字符。Oracle数据库中的AL16UTF16字符集支持增补字符。

B.3 UTF-8编码

UTF-8字符 表B-2中的代码表明满足以下条件：

  • ASCII字符使用1个字节

  • 欧洲（ASCII除外），阿拉伯语和希伯来语字符需要2个字节

  • 印度语，泰语，中文，日语和韩语字符以及某些符号（如欧元符号）需要3个字节

  • 专用区＃1中的字符需要3个字节

  • 补充字符需要4个字节

  • 专用区＃2中的字符需要4个字节

在Oracle数据库中，AL32UTF8字符集支持1字节，2字节，3字节和4字节值。在Oracle数据库中，UTF8字符集支持1字节，2字节和3字节值，但不支持4字节值。
