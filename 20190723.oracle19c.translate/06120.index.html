Pro * COBOL Release 18c，版本18.1中的新功能

以下功能是此版本中的新增功能：

  • 在流量控制器模式下支持Oracle Connection Manager

    Traffic Director模式中的Oracle Connection Manager是一个代理，位于支持的数据库客户端和数据库实例之间，以提高高可用性，连接多路复用和负载平衡。

早期版本中的新功能

本节列出了以前版本中Pro * COBOL引入的新功能。

Pro * COBOL 12.2生产中的功能

  • 支持长标识符（对象长度为128字节）。在以前的版本中，对象长度限制为30个字节。

  • 支持Oracle Instant Client - Basic Light版本。

  • 新命令行选项trim_password ，用于防止由包含尾随空格的密码字符串导致的身份验证问题。

  • 支持以下平台的Micro Focus Visual COBOL 2.2 Update 2编译器：

    • Linux x64

    • Windows 64位和32位

    • Solaris x86（32位和64位）

    • Solaris SPARC（32位和64位）

Pro * COBOL 12.1生产中的功能

  • 支持自动增量列

  • 支持32k列

  • 支持按内存预取

  • 支持SQL计划管理（SPM）

错误修复

以下部分列出了Pro * COBOL中修复的错误。说明后面括号中的数字是指Oracle Bug数据库中的错误号。

在Pro * COBOL版本12.2生产中修复了错误

  • 当timezone.pc文件中的common_parser = yes时，Pro * COBOL不再抛出CSF-S-00000错误（9531787）

  • 当common_parser = yes时，Pro * COBOL在选择列表中使用绑定时不再抛出ORA-01008错误（14127422）

  • 对于具有CASE子句和TIMESTAMP函数的INSERT语句，使用选项USERID和common_parser = yes进行预编译时，Pro * COBOL不再抛出ORA-932错误（14335958,19473788）

  • 使用select / * + index提示* /语句（953338）使用嵌入式PL / SQL进行预编译时，Pro * COBOL不再抛出PCB-S-00576，PLS-103错误

  • Pro * COBOL在回滚期间不再无法设置SQLSTATE，MODE = ANSI且未声明SQLCODE（5891984）

  • Pro * COBOL不再无法生成正确的日志文件（17280039）

  • 在使用88级初始化变量（20194289）预编译程序时，Pro * COBOL不再抛出PCB-S-00400错误

  • 使用common_parser = yes进行预编译时，Pro * COBOL不再获取重复主机变量的SIGSEGV（19473788）

  • 使用common_parser = yes预编译程序时，Pro * COBOL不再抛出PCB-S-00576错误（18800170）

  • Pro * COBOL现在在删除日志和sql文件之前检查预编译期间的致命错误（17871321）

  • 使用PIC N变量的“SELECT INTO”语句时，Pro * COBOL不再抛出ORA-538976288错误（17189633）

  • 当预编译失败时，Pro * COBOL不再创建包含“plan_run = yes”的.sql文件（16240153）

  • 通过改变comp5 = yes修改变量时，Pro * COBOL不再创建超过72列（14708769）

  • Pro * COBOL中的SQLGLS调用现在按预期工作（14640230）

  • 当变量标识符跟随COPY语句时，Pro * COBOL在预编译期间不再返回PCB-S-00400错误（14113014）

  • 使用COPY修改器时，Pro * COBOL不再返回PCB-S-00400错误（13478294）

  • 预编译失败时，Pro * COBOL不再生成UNIX返回码0（10083052）

  • 遇到错误后，Pro * COBOL不再生成.cob文件（9303962）

在Pro * COBOL版本12.1中修复了错误

  • 解析长令牌/ SQL语句时，Pro * COBOL不再崩溃（13006848）

  • Pro * COBOL在预编译包含嵌入式SQL（EXEC SQL）的内联视图的SQL时不再返回PCB-S-400，尽管使用了common_parser = yes（12641413）

  • 使用数字作为组元素的第一个字符时，Pro * COBOL不再抛出PCB-0400错误（10265545）

  • 当设置选项comp1 = integer时，Windows上的Pro * COBOL不再崩溃（10040552）

  • Pro * COBOL不再使用PL / SQL绑定返回ORA-6502（9905110）

  • 当大型源文件包含大量主机变量时，Pro * COBOL不再崩溃（9689604）

  • 在安装补丁9218271（9470397）后，Pro * COBOL现在在COBOL COPY子句后用双点解析语句

  • 当在远程数据库上声明语句然后通过传递字符串而不是bind-var来准备Pro * COBOL时，不再生成非法值（9402996）

  • 设置max_rows_insert时，Pro * COBOL不再因ORA-12899而失败（9381997）

  • 当JUSTFIED子句与主变量一起使用时，Pro * COBOL不再返回PCB-S-00214（9266470）

  • 如果使用SCREEN SECTION，Pro * COBOL不再返回PCB-S-400错误（9151190）

  • 当主机变量用于AT子句时，Pro * COBOL不再返回PCB-W-233（9147830）

  • 当在数组声明的OCCURS子句中使用COPY时，Pro * COBOL不再返回PCB-S-400（9128157）

  • 当COPY语句以双点结束时，Pro * COBOL不再返回PCB-S-400（9055457）

  • Pro * COBOL现在可以正确转换包含列表列表的SQL语句（8932394）

  • 当声明光标启用时，Pro * COBOL不再返回ORA-933（8770900）

  • 在DATA SECTION（8713408）中使用EXE SQL INCLUDE语句而没有结束时段时，Pro * COBOL不再返回PCB-S-00400
