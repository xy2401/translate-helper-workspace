关于用户定义的函数

您可以在PL / SQL，Java或C中编写用户定义的函数，以提供SQL或SQL内置函数中不可用的功能。用户定义的函数可以出现在可以出现表达式的SQL语句中。

例如，用户定义的函数可用于以下内容：

  • SELECT语句的选择列表

  • WHERE子句的条件

  • CONNECT BY ， START WITH ， ORDER BY和GROUP BY子句

  • INSERT语句的VALUES子句

  • UPDATE语句的SET子句

注意：

Oracle SQL不支持使用布尔参数或返回调用函数。因此，如果将从SQL语句调用用户定义的函数，则必须将它们设计为返回数字（0或1）或字符串（“ TRUE ”或“ FALSE ”）。

先决条件

用户定义的函数必须创建为顶级函数或使用包规范声明，然后才能在SQL语句中进行命名。

要在SQL表达式中使用用户函数，您必须拥有或具有用户函数的EXECUTE特权。要查询使用用户函数定义的视图，您必须具有视图的READ或SELECT权限。从视图中选择不需要单独的EXECUTE特权。

名称优先权

在SQL语句中，数据库列的名称优先于没有参数的函数的名称。例如，如果人力资源经理在hr模式中创建以下两个对象：

CREATE TABLE new_emps（new_sal NUMBER，...）;创建功能new_sal返回号码开始...结束 ;

然后在下面的两个语句，提及new_sal指柱new_emps.new_sal ：

SELECT new_sal FROM new_emps; SELECT new_emps.new_sal FROM new_emps;

要访问new_sal函数，请输入：

SELECT hr.new_sal FROM new_emps;

以下是对SQL表达式中允许的用户函数的一些示例调用：

circle_area（radius）payroll.tax_rate（empno）hr.employees.tax_rate（dependent，empno）@remote

例

要调用tax_rate从模式用户功能hr ，执行它反对ss_no和sal列tax_table ，指定以下内容：

SELECT hr.tax_rate（ss_no，sal） INTO income_tax FROM tax_table WHERE ss_no = tax_id;

INTO子句是PL / SQL，允许您将结果放入变量income_tax 。

命名约定

如果只给出了一个可选的模式或包名称，则第一个标识符可以是模式名称或包名称。例如，确定PAYROLL中是否有参考PAYROLL 。 TAX_RATE是架构或包名称，Oracle Database按如下方式进行：

  1. 检查当前架构中的PAYROLL包。

  2. 如果找不到PAYROLL包，则查找包含顶级TAX_RATE函数的模式名称PAYROLL 。如果未找到此类函数，则返回错误。

  3. 如果在当前模式中找到PAYROLL包，则在PAYROLL包中查找TAX_RATE函数。如果未找到此类函数，则返回错误。

您还可以使用为其定义的任何同义词来引用存储的顶级函数。
