78 ORA-12315至ORA-12354

ORA-12315：数据库链接类型对ALTER DATABASE语句无效

原因：您在ALTER DATABASE语句中指定的数据库链接名称不是ROM：链接。使用ALTER DATABASE语句装入或打开辅助数据库时，必须指定ROM：链接。

操作：使用有效的ROM重新发出ALTER DATABASE语句：指向要装入或打开的数据库的链接。如果不存在有效的ROM：链接，请使用CREATE DATABASE LINK命令创建一个。有关使用ROM：链接类型创建数据库链接的更多信息，请参阅“受信任的ORACLE RDBMS安全功能指南”。

ORA-12316：数据库链接的连接字符串中的语法错误

原因： CREATE DATABASE LINK语句中的连接字符串有语法错误。

操作：删除数据库链接并使用有效语法重新创建它。有关CREATE DATABASE LINK语句的连接字符串部分的更多信息，请参见“SQL语言参考手册”。

ORA-12317：登录到数据库（链接名称string ）被拒绝

原因：此错误可能有多种原因。首先，如果辅助数据库中的用户名（和密码，如果使用数据库而不是操作系统身份验证）与主数据库中的用户名（和密码）不同，则会出现此错误。其次，如果辅助数据库中的用户名无效（尚未创建），则可能会出现此错误。第三，如果在数据库链接定义的连接字符串中指定的用户名/密码组合无效（未创建或密码无效），则可能会出现此错误。

操作：在第一种情况下，请确保辅助数据库包含与您在主数据库中使用的用户名（和密码，如果使用数据库身份验证）相同的用户名。通常，您应始终在Trusted ORACLE中使用操作系统身份验证（有关OS身份验证的优点的详细信息，请参阅“受信任的ORACLE RDBMS安全功能指南”）。在第二种情况下，请确保已创建辅助数据库中的用户名。在第三种情况下，请确保已在辅助数据库中创建了连接字符串中指定的用户名。

ORA-12318：已安装数据库（链接名称string ）

原因：您正在尝试装入已由实例装入的辅助数据库。

操作：数据库已装入，因此您无需采取其他操作来装入它。要建立访问权限，请使用ALTER DATABASE linkname OPEN命令打开数据库。

ORA-12319：数据库（链接名称string ）已经打开

原因：您正在尝试打开已打开的辅助数据库。

操作：数据库已打开，您无需采取其他操作来建立访问权限。

ORA-12321：数据库（链接名称string ）未打开且AUTO_MOUNTING = FALSE

原因：您的实例尝试装入的辅助数据库未打开，并且尚未启用自动装入。

操作：使用带有OPEN和MOUNT选项的ALTER DATABASE linkname手动挂载和打开辅助数据库。或者，要允许实例自动装入和打开辅助数据库，请将参数文件中的AUTO_MOUNTING参数设置为TRUE。

ORA-12322：无法装入数据库（链接名称string ）

原因：此消息应附带指示问题原因的其他错误消息。

操作：按照随附的错误消息中列出的步骤解决问题。

ORA-12323：无法打开数据库（链接名称string ）

原因：此消息应附带指示问题原因的其他错误消息。

操作：按照随附的错误消息中列出的步骤解决问题。

ORA-12324：不能在私有数据库链接上使用ROM：link类型

原因：您只能在公共数据库链接上指定ROM：链接类型，而不是私有数据库链接。

操作：确定是否存在指向辅助数据库的现有公用数据库链接。如果不是，并且您希望建立对辅助数据库的公共访问权限，请使用CREATE DATABASE LINK命令创建指向辅助数据库的公共数据库链接。

ORA-12326：数据库string立即关闭;不允许任何操作

原因：您尝试访问的数据库正在关闭，因此您的操作已终止。

操作：等待数据库重新打开，或联系数据库管理员。

ORA-12329：数据库string已关闭;不允许任何操作

原因：您尝试访问的数据库已关闭，因此您的操作已终止。

操作：等待数据库重新打开，或与数据库管理员联系。

ORA-12333：未安装数据库（链接名称string ）

原因：您试图打开尚未装入的数据库。

操作：使用ALTER DATABASE linkname MOUNT命令装入数据库，然后重新尝试打开数据库。

ORA-12334：数据库（链接名称string ）仍处于打开状态

原因：您试图卸载仍处于打开状态的数据库。

操作：使用ALTER DATABASE linkname CLOSE命令关闭数据库，然后重新尝试卸载数据库。

ORA-12335：数据库（链接名称string ）未打开

原因：您试图关闭未打开的数据库。

行动：数据库已关闭;你可以继续拆卸它。

ORA-12336：无法登录数据库（链接名称string ）

原因：当您正在安装或卸载同一数据库时，您正在登录。

操作：检查数据库是否处于挂载或打开状态，并在数据库可访问后再次尝试登录。

ORA-12341：超出最大打开次数

原因：参数文件中OPEN_MOUNTS参数上指定的数量超过允许的最大值（255）。

操作：更改此参数的值，以便它反映可能的打开挂载到辅助数据库的实际数量。这必须小于255。

ORA-12342：打开的挂载超出了OPEN_MOUNTS参数设置的限制

原因：当前打开的安装数超过了您在OPEN_MOUNTS参数上指定的值。

操作：增加OPEN_MOUNTS参数的值，以使其适应辅助数据库的最大可能打开次数。

ORA-12345：用户string缺少数据库链接中的CREATE SESSION权限（链接名称string ）

原因：此消息可能有多种原因：首先，如果指定的第二个数据库中的用户名未被授予CREATE SESSION系统特权，则会收到此消息。其次，如果未在数据库链接定义的连接字符串中指定的用户名被授予CREATE SESSION系统特权，您将收到此消息。

操作：您采取的操作取决于消息的原因：在第一种情况下，请确保在辅助数据库中的用户名被授予CREATE SESSION系统特权。在第二种情况下，请确保在辅助数据库中为数据库链接定义的连接字符串中指定的用户名授予CREATE SESSION系统特权。

ORA-12350：仍然挂载了正在删除的数据库链接

原因：尝试删除仍挂载和/或打开的ROM：数据库链接。

操作：关闭并卸载数据库，然后重新发出drop语句。

ORA-12351：无法使用具有远程对象引用的远程对象创建视图

原因：您尝试创建一个引用远程对象的视图，该远程对象又引用另一个数据库上的对象。由于您尝试创建的视图引用了远程对象，因此该对象无法引用另一个数据库上的对象。

操作：在视图中选择要引用的其他对象，或更改远程对象以使其不引用其他数据库。

ORA-12352：对象string 。 string @ string无效

原因：尝试引用（编译）辅助数据库的对象，但该对象无效，系统无法验证或重新编译它，因为它位于辅助数据库中。

操作：手动重新编译辅助数据库中的无效对象。

ORA-12353：辅助存储对象无法引用远程对象

原因：您试图从远程视图中选择或执行引用另一个数据库上的对象的远程过程。由于远程视图或过程位于辅助数据库上，因此无法对另一个数据库执行其他引用。

操作：选择其他对象以引用或更改远程视图或过程，以使其不引用其他数据库。

ORA-12354：被删除的辅助对象

原因：您试图访问正在被删除的辅助数据库上的对象（例如，表或视图）。

操作：重复操作。如果再次收到此消息，请尝试从辅助数据库访问该对象。如果收到内部错误或跟踪文件，请与Oracle WorldWide技术支持联系。
