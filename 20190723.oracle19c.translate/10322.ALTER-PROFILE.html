ALTER PROFILE

目的

使用ALTER PROFILE语句在概要文件中添加，修改或删除资源限制或密码管理参数。

使用ALTER PROFILE语句对配置文件所做的ALTER仅在后续会话中影响用户，而不会影响当前会话中的用户。

也可以看看：

创建个人资料的信息，创建一个配置文件

先决条件

您必须具有ALTER PROFILE系统特权。

要指定CONTAINER子句，必须连接到多租户容器数据库（CDB）。要指定CONTAINER = ALL ，当前容器必须是根。要指定CONTAINER = CURRENT ，当前容器必须是可插拔数据库（PDB）。

语义

ALTER PROFILE语句中的关键字，参数和子句的含义与CREATE PROFILE语句中的含义相同。

您无法从DEFAULT配置文件中删除限制。

有关更多信息，请参阅CREATE PROFILE和下一节中的示例。

例子

使密码不可用：示例

以下语句使new_profile配置文件的密码（在“ 创建配置文件：示例 ”中创建 ）在90天内无法重用：

ALTER PROFILE new_profile LIMIT PASSWORD_REUSE_TIME 90 PASSWORD_REUSE_MAX UNLIMITED;

设置默认密码值：示例

以下语句将app_user配置文件的PASSWORD_REUSE_TIME值（在“ 设置配置文件资源限制：示例 ”中创建 ） DEFAULT为DEFAULT配置文件中的已定义值：

ALTER PROFILE app_user LIMIT PASSWORD_REUSE_TIME DEFAULT PASSWORD_REUSE_MAX UNLIMITED;

限制登录尝试和密码锁定时间：示例

以下语句将配置文件app_user与FAILED_LOGIN_ATTEMPTS设置为5并将PASSWORD_LOCK_TIME设置为1进行更改：

ALTER PROFILE app_user LIMIT FAILED_LOGIN_ATTEMPTS 5 PASSWORD_LOCK_TIME 1;

此声明会导致分配了app_user配置文件的任何用户帐户在连续五次登录尝试失败后被锁定一天。

更改密码生命周期和宽限期：示例

以下语句将配置文件app_user2 PASSWORD_LIFE_TIME修改为90天，将PASSWORD_GRACE_TIME为5天：

ALTER PROFILE app_user2 LIMIT PASSWORD_LIFE_TIME 90 PASSWORD_GRACE_TIME 5;

限制帐户不活动：示例

以下语句将配置文件app_user2 INACTIVE_ACCOUNT_TIME修改为连续30天：

ALTER PROFILE app_user2 LIMIT INACTIVE_ACCOUNT_TIME 30;

如果该帐户已在一定天数内处于非活动状态，那么这些天数将计入新的30天限制。

限制并发会话：示例

此语句为app_user配置文件定义了5个并发会话的新限制：

ALTER PROFILE app_user LIMIT SESSIONS_PER_USER 5;

如果app_user配置文件当前未定义SESSIONS_PER_USER的限制，则前面的语句将限制为5添加到配置文件。如果配置文件已定义限制，则前面的语句将其重新定义为5。分配了app_user配置文件的任何用户随后限制为5个并发会话。

删除配置文件限制：示例

此语句从app_user配置文件中删除IDLE_TIME限制：

ALTER PROFILE app_user LIMIT IDLE_TIME DEFAULT;

分配了app_user配置文件的任何用户在其后续会话中都将使用DEFAULT配置文件中定义的IDLE_TIME限制。

限制配置文件空闲时间：示例

此语句定义DEFAULT配置文件的2分钟空闲时间限制：

ALTER PROFILE默认LIMIT IDLE_TIME 2;

此IDLE_TIME限制适用于以下用户：

  • 未明确分配任何配置文件的用户

  • 明确分配了未定义IDLE_TIME限制的配置文件的用户

此语句定义app_user2配置文件的无限空闲时间：

ALTER PROFILE app_user2 LIMIT IDLE_TIME UNLIMITED;

随后，任何分配了app_user2配置文件的用户都被允许无限制的空闲时间。
