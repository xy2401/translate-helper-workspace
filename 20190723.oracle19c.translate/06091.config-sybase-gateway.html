配置网关初始化参数文件

执行以下任务以配置。 网关初始化参数文件。

  1. 选择网关的系统标识符
  2. 自定义初始化参数文件

选择网关的系统标识符

该网关系统标识符 （SID）是标识网关实例的字母数字字符串。对于要访问的每个Sybase数据库，您需要一个网关实例，因此需要一个网关SID。SID用作初始化参数文件的文件名的一部分。默认SID是 dg4sybs 。

您可以定义网关SID，但使用默认的dg4sybs更容易，因为您不需要更改初始化参数文件名。但是，如果要访问两个Sybase数据库，则需要两个网关SID，每个网关用于网关的每个实例。如果您只有一个Sybase数据库并且希望有时使用一组网关参数设置来访问它，而其他时候使用不同的网关参数设置，那么您将需要多个网关SID用于单个Sybase数据库。

自定义初始化参数文件

启动网关时，初始化参数文件必须可用。在安装过程中，将创建以下默认初始化参数文件：

 ORACLE_HOME \ dg4sybs \ admin \ initdg4sybs.ora

其中ORACLE_HOME是安装网关的目录。

此初始化文件用于默认网关SID。如果未使用dg4sybs作为网关SID，则必须使用在上一步中选择的SID重命名初始化参数文件选择网关的系统标识符 。此默认初始化参数文件足以启动网关，验证安装是否成功以及运行演示脚本。

可以使用许多初始化参数来修改网关行为。有关可以设置的初始化参数的完整列表，请参阅初始化参数 。对初始化参数所做的更改仅在下一个网关会话中生效。最重要的参数是HS_FDS_CONNECT_INFO ，它描述了与非Oracle系统的连接。

默认初始化参数文件已有此参数的条目。HS_FDS_CONNECT_INFO的语法如下：

HS_FDS_CONNECT_INFO = host_name ： port_number / database_name

是Sybase数据库服务器的端口号。

为网关配置Oracle Net

网关要求Oracle Net与Oracle数据库通信。配置网关后，执行以下任务以配置Oracle Net以使用网关：

  1. 为网关配置Oracle Net Listener
  2. 停止并启动Gateway的Oracle Net Listener

为网关配置Oracle Net Listener

Oracle Net Listener侦听来自Oracle数据库的传入请求。要使Oracle Net Listener监听网关，必须将有关网关的信息添加到Oracle Net Listener配置文件listener.ora 。默认情况下，此文件位于ORACLE_HOME \network\admin ，其中ORACLE_HOME是安装网关的目录。

必须将以下条目添加到listener.ora文件中：

  • Oracle Net Listener侦听的Oracle Net地址列表

  • Oracle Net Listener响应传入连接请求而启动的网关的可执行文件名

ORACLE_HOME \dg4sybs\admin目录中提供了listener.ora条目（ listener.ora.sample ）的示例，其中ORACLE_HOME是安装网关的目录。

listener.ora文件条目的语法

Oracle数据库使用Oracle Net和任何支持的协议适配器与网关通信。以下是Oracle Net Listener使用TCP / IP协议适配器侦听的地址的语法：

LISTENER =（ADDRESS =（PROTOCOL = TCP）（HOST = host_name ）（PORT = port_number ））

创建数据库链接

连接到Oracle数据库的任何Oracle客户端都可以通过网关访问Sybase数据。Oracle客户端和Oracle数据库可以驻留在不同的计算机上。网关仅接受来自Oracle数据库的连接。

首次在Oracle会话中使用时，通过数据库链接建立与网关的连接。在此上下文中，连接是指Oracle数据库与网关之间的连接。在Oracle会话结束之前，连接将保持建立状态。另一个会话或用户可以访问相同的数据库链接，并获得与网关和Sybase数据库的不同连接。

数据库链接在网关会话期间处于活动状态。如果要在会话期间关闭数据库链接，可以使用ALTER SESSION语句执行此操作。

要访问Sybase服务器，必须创建数据库链接。公共数据库链接是最常见的数据库链接。

SQL> CREATE PUBLIC DATABASE LINK dblink CONNECT TO 2“ user” IDENTIFIED BY “password” USING'tns_name_entry ';

哪里：

变量 描述

DBLINK

是完整的数据库链接名称。

tns_name_entry

指定tnsnames.ora文件中指定的标识网关的Oracle Net连接描述符

创建数据库链接后，您可以验证与Sybase数据库的连接，如下所示：

SQL> SELECT * FROM DUAL @dblink;

配置两阶段提交

网关支持以下事务功能：

  • COMMIT_CONFIRM

  • 只读

  • SINGLE_SITE

使用HS_TRANSACTION_MODEL初始化参数设置事务模型。 默认情况下，网关以COMMIT_CONFIRM事务模式运行。当事务更新Sybase数据库时，网关将成为提交点站点。在验证事务中的所有Oracle数据库是否已成功准备事务之后，Oracle数据库将在Sybase数据库中提交工作单元。只有一个网关实例可以作为提交点站点参与Oracle两阶段提交事务。

也可以看看：

“Oracle数据库异构连接用户指南” ，了解有关两阶段提交过程的信息。

要启用COMMIT_CONFIRM事务模式，请执行以下任务：

  1. 创建恢复帐户和密码
  2. 创建事务日志表

名为HS_TRANSACTION_LOG的日志表是记录两阶段提交事务的地方。

创建恢复帐户和密码

对于网关来恢复分布式事务，a 必须在Sybase数据库中设置恢复帐户和密码。默认情况下，两者都有帐户的用户名和密码是RECOVER 。可以使用网关初始化参数HS_FDS_RECOVERY_ACCOUNT更改帐户的名称。可以使用网关初始化参数HS_FDS_RECOVERY_PWD更改帐户密码。

注意：

Oracle建议您不要使用默认值RECOVER作为用户名和密码。此外，将纯文本作为用户名和密码存储在初始化文件中并不是一个好的安全策略。有一个名为dg4pwd的实用程序应该用于加密。有关更多详细信息，请参见“ Oracle数据库异构连接用户指南 ”中的第4.2.3节“加密初始化参数” 。

  1. 在Sybase数据库中设置用户帐户。用户名和密码都必须是有效的Sybase用户名和密码。
  2. 在初始化参数文件中，设置以下网关初始化参数：

    • HS_FDS_RECOVERY_ACCOUNT指向您设置用于恢复的Sybase用户帐户的用户名。

    • HS_FDS_RECOVERY_PWD为您设置用于恢复的Sybase用户帐户的密码。

      也可以看看：

      有关编辑初始化参数文件的信息，请自定义初始化参数文件。有关HS_FDS_RECOVERY_ACCOUNT和HS_FDS_RECOVERY_PWD ，请参阅初始化参数 。

创建事务日志表

配置网关进行两阶段提交时，a 必须在Sybase数据库中创建表以记录事务。网关使用事务日志表来检查网关在Sybase数据库中启动并在表中注册的失败事务的状态。

注意：

事务日志表的更新不能是Oracle分布式事务的一部分。

注意：

事务日志表中的信息是恢复过程所必需的 不得改变。桌子 必须仅由网关使用，访问或更新。

该表名为HS_TRANSACTION_LOG,由两列组成， GLOBAL_TRAN_ID ，数据类型CHAR(64) NOT NULL和TRAN_COMMENT ，数据类型CHAR(255) 。

通过使用HS_FDS_TRANSACTION_LOG初始化参数指定其他名称，可以使用除HS_TRANSACTION_LOG之外的日志表的其他名称。

也可以看看：

初始化有关HS_FDS_TRANSACTION_LOG初始化参数的信息的参数。

在创建恢复帐户和密码中创建的用户帐户中创建事务日志表。由于事务日志表用于记录网关事务的状态，因此该表必须驻留在发生Sybase更新的数据库中。此外，必须在恢复帐户的所有者下创建事务日志表。

注意：

要使用事务日志表，必须为该网关的用户授予对该表的特权。

要创建事务日志表，请使用位于ORACLE_HOME \dg4sybs\admin目录中的dg4sybs_tx.sql脚本，其中ORACLE_HOME是安装网关的目录。使用isql在MS-DOS提示符下执行脚本，如下所示：

> isql -Urecovery_account -Precovery_account_password [-Sserver] -idg4sybs_tx.sql

为数据字典支持创建Sybase视图

要启用Oracle数据字典转换支持，请使用位于ORACLE_HOME \dg4sybs\admin目录中的dg4sybs_cvw.sql脚本，其中ORACLE_HOME是安装网关的目录。您必须在要通过网关访问的每个Sybase数据库上运行此脚本。使用isql执行脚本，如下所示：

> isql -Usa_user -Psa_pwd [-Sserver] [-Ddatabase] -e -i dg4sybs_cvw.sql

其中sa_user和sa_pwd分别是Sybase系统管理员用户标识和密码。

加密网关初始化参数值

网关使用用户标识和密码来访问远程数据库中的信息。必须在网关初始化文件中定义一些用户标识和密码，以处理资源恢复等功能。在当前具有安全意识的环境中，具有可在初始化文件中访问的纯文本密码被认为是不安全的。dg4pwd加密实用程序已添加为异构服务的一部分，以帮助提高安全性。此网关可以访问此实用程序。包含敏感值的初始化参数可以以加密形式存储。

配置网关以访问多个Sybase数据库

配置网关以访问多个Sybase数据库的任务类似于为单个数据库配置网关的任务。配置示例假定以下内容：

  • 使用默认SID dg4sybs安装和配置网关。

  • 为一个名为db1 Sybase数据库配置网关。

  • 正在添加具有IP地址204.179.79.15的主机上名为db2和db3两个Sybase数据库。

多个Sybase数据库示例：配置网关

为每个Sybase数据库选择一个系统ID

每个Sybase数据库都需要一个单独的网关实例。每个实例都需要自己的网关系统ID（SID）。对于此示例，为访问Sybase数据库的实例选择网关SID：

  • dg4sybs2用于访问数据库db2的网关。

  • dg4sybs3用于访问数据库db3的网关。

创建两个初始化参数文件

通过复制原始初始化参数文件： ORACLE_HOME \dg4sybs\admin\initdg4sybs.ora两次为网关的每个实例创建一个初始化参数文件，用db2的网关SID命名一个，用db3的网关SID命名另一个：

> cd ORACLE_HOME \ dg4sybs \ admin> copy initdg4sybs.ora initdg4sybs2.ora> copy initdg4sybs.ora initdg4sybs3.ora

更改新文件中HS_FDS_CONNECT_INFO参数的值。

对于initdg4sybs2.ora ，输入以下内容：

HS_FDS_CONNECT_INFO = 204.179.79.15：5000 / DB2

对于initdg4sybs3.ora ，输入以下内容：

HS_FDS_CONNECT_INFO = 204.179.79.15：5000 / DB3

注意：

如果同一Sybase数据库有多个网关SID，因为要在不同时间使用不同的网关参数设置，请执行相同的过程。您可以创建多个初始化参数文件，每个文件具有不同的SID和不同的参数设置。

多个Sybase数据库示例：配置Oracle Net Listener

将条目添加到listener.ora

在Oracle Net Listener配置文件中添加两个新条目， listener.ora 。即使多个网关实例访问同一数据库，您也必须为每个网关实例提供一个条目。

以下示例首先显示原始安装网关的条目，然后显示新条目：

SID_LIST_LISTENER =（SID_LIST =（SID_DESC =（SID_NAME = dg4sybs）（ORACLE_HOME = oracle_home_directory ）（PROGRAM = dg4sybs））（SID_DESC =（SID_NAME = dg4sybs2）（ORACLE_HOME = oracle_home_directory ）（PROGRAM = dg4sybs））（SID_DESC =（SID_NAME = dg4sybs3） ）（ORACLE_HOME = oracle_home_directory ）（PROGRAM = dg4sybs）））

其中， oracle_home_directory是网关所在的目录。

多个Sybase数据库示例：为Gateway Access配置Oracle数据库

将两个连接描述符条目添加到tnsnames.ora文件中。即使网关实例访问同一数据库，您也必须为每个网关实例创建一个条目。

此示例描述如何在Oracle数据库上为多个网关实例配置Oracle Net。它首先显示原始安装网关的条目，然后是新网关实例的两个条目：

old_db_using =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（PORT = port_number）（HOST = host_name））（CONNECT_DATA =（SID = dg4sybs））（HS = OK）） new_db2_using =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（PORT = port_number）（HOST = host_name））（CONNECT_DATA =（SID = dg4sybs2））（HS = OK）） new_db3_using =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（PORT = port_number）（HOST = host_name ））（CONNECT_DATA =（SID = dg4sybs3））（HS = OK））

PORT的值是正在侦听网关的Oracle Net Listener的TCP / IP端口号。该数字可以在Oracle Net Listener使用的listener.ora文件中找到。HOST的值是运行网关的计算机的名称。该名称也可以在Oracle Net Listener使用的listener.ora文件中找到。

多个Sybase数据库示例：访问Sybase数据

输入以下内容dg4sybs2网关创建数据库链接：

SQL> CREATE PUBLIC DATABASE LINK SYBS2 CONNECT TO 2“ user2” IDENTIFIED BY “password2” USING'new_db2_using';

输入以下内容dg4sybs3网关创建数据库链接：

SQL> CREATE PUBLIC DATABASE LINK SYBS3 CONNECT TO 2 “user3” IDENTIFIED BY “password3” USING'new_db3_using';

创建数据库链接后，您可以验证与新Sybase数据库的连接，如下所示：

SQL> SELECT * FROM ALL_USERS @ SYBS2;
SQL> SELECT * FROM ALL_USERS @ SYBS3;
