8 CAT-01000至CAT-01509

CAT-01000：无效的命令行选项{0}

原因：提供的命令行选项无效。

操作：查看命令用法并使用有效选项。

CAT-01001：命令行选项{0}的参数数量不正确

原因：提供的参数数对于指定的命令行选项无效。

操作：查看命令用法并使用正确的语法。

CAT-01002：命令行选项{0}的输入参数无效

原因：提供的参数对于指定的命令行选项无效。

操作：查看命令用法并使用正确的输入参数。

CAT-01003：意外错误：{0}

原因：由于意外错误导致命令执行失败。

操作：如果错误仍然存在，请联系Oracle支持服务。

CAT-01004：群集就绪服务未在本地节点上运行

原因：由于Cluster Ready Services未在本地节点上运行，因此命令执行失败。

操作：在本地节点上启动Cluster Ready Services并重试该命令。

CAT-01005：{0}不是命令行选项{0}的有效参数。

原因：提供的参数对于指定的命令行选项无效。

操作：查看命令用法并使用正确的语法。

CAT-01006：{0}是命令行选项{0}的必需参数。

原因：指定的命令行选项缺少参数。

操作：查看命令用法并使用正确的语法。

CAT-01007：命令行选项{0}需要{0}的值。

原因：没有指定命令行选项的值的参数。

操作：查看命令用法并使用正确的语法。

CAT-01008：服务质量管理（QoS）服务器未联机。

原因：尝试切换活动策略发现服务质量管理（QoS）服务器资源“qosmserver”未联机。

操作：使用'srvctl start qosmserver'命令启动服务质量管理（QoS）服务器资源'qosmserver'，然后重试原始命令。

CAT-01009：服务质量管理（QoS）服务器资源“qosmserver”不存在。

原因：尝试切换活动策略发现服务质量管理（QoS）服务器资源“qosmserver”不存在。

操作：使用'srvctl add qosmserver'命令创建服务质量管理（QoS）服务器资源。

CAT-01100：无效的用户{0}

原因：身份验证失败，因为QoS管理用户名不存在。

操作：指定现有QoS管理用户名。

CAT-01101：用户{0}的身份验证无效

原因：指定用户的身份验证失败。

操作：输入指定用户的有效密码。

CAT-01102：用户{0}已存在。

原因：添加新QoS管理用户的请求失败，因为用户名已存在。

操作：指定新的QoS管理用户名。

CAT-01103：用户{0}不存在。

原因：修改或删除QoS Management用户的请求失败，因为用户名不存在。

操作：指定现有QoS管理用户名。

CAT-01104：不允许操作; {0}是必需用户。

原因：删除用户的请求失败，因为无法删除所需的用户。

操作：指定先前添加的用户名。

CAT-01105：密码不匹配。

原因：提供的密码彼此不匹配。

操作：输入匹配的密码。

CAT-01114：用户{0}的凭据不存在。

原因：在随附的system-jazn-data.xml文件中找不到指定用户的凭据。

操作：联系Oracle支持服务。

CAT-01115：由于OC4J资源正在运行，因此无法执行操作

原因：由于OC4J资源正在运行，因此无法执行操作。

操作：停止OC4J资源，然后重试该操作。

CAT-01121：生成QoS管理用户的意外错误：\ n {0}

原因：未正确生成QoS管理用户。

操作：更正随附消息中指示的问题，然后重试。

CAT-01123：当前用户无权执行命令行选项{0}。

原因：执行qosctl的当前用户无权执行指示的命令行选项。

操作：重试执行qosctl作为Oracle Grid Infrastructure用户。

CAT-01201：参数数量不正确

原因：指定了错误数量的命令行参数。

操作：查看命令用法并使用正确的输入参数。

CAT-01202：无效参数：{0}

原因：指定了无效的命令行参数。

操作：查看命令用法并使用正确的输入参数。

CAT-01203：用户{0}无权执行此命令。

原因：指定的用户没有足够的权限来执行该命令。

操作：使用附带消息中指示的足够权限指定凭据存储中的现有用户。

CAT-01204：用户名{0}不存在。

原因：在凭证存储中找不到指定的用户。

操作：在凭证存储中指定现有用户。

CAT-01205：凭据存储APPQOSADMIN不存在。

原因：找不到凭据存储类型APPQOSADMIN。

操作：联系Oracle支持服务以获取帮助。

CAT-01206：策略名称{0}不存在。

原因：当前策略集中不存在指定的策略名称。

操作：指定当前策略集中存在的策略名称。

CAT-01207：服务质量管理（QoS）服务器未联机。

原因：尝试切换活动策略发现服务质量管理（QoS）服务器资源“qosmserver”未联机。

操作：使用'srvctl start qosmserver'命令启动服务质量管理（QoS）服务器资源'qosmserver'，然后重试原始命令。

CAT-01208：服务质量管理（QoS）服务器资源“qosmserver”不存在。

原因：尝试切换活动策略发现服务质量管理（QoS）服务器资源“qosmserver”不存在。

操作：使用'srvctl add qosmserver'命令创建服务质量管理（QoS）服务器资源。

CAT-01209：服务质量管理（QoS）服务器未在“qosmserver”资源中运行。

原因：尝试切换活动策略发现服务质量管理（QoS）服务器未在“qosmserver”资源中运行。

操作：联系Oracle支持服务。

CAT-01210：QoS管理服务器有多个实例。

原因：找到了多个服务质量管理（QoS）服务器。

操作：联系Oracle支持服务。

CAT-01211：日志级别{0}无效。

原因：跟踪日志记录期间遇到错误，因为环境变量QOS_TRACE_LEVEL未设置为有效的日志级别。

操作：在QOS_TRACE_LEVEL环境变量中指定有效的日志级别。有效值为OFF，SEVERE，WARNING，INFO，CONFIG，FINE，FINER，FINEST和ALL。如果问题仍然存在，请与Oracle支持服务联系。

CAT-01300：start datetime参数无效。

原因：指定的开始日期时间未使用格式YYYY-MM-DD HH：mm。

操作：更正start datetime参数。

CAT-01301：结束日期时间参数无效。

原因：指定的结束日期时间未使用格式YYYY-MM-DD HH：mm。

操作：更正结束日期时间参数。

CAT-01302：开始日期时间是结束日期时间。

原因：指定的开始日期时间是在结束日期时间之后。

操作：指定结束日期时间之前的开始日期时间。

CAT-01303：缺少输出目录。

原因：未提供输出目录名称。

操作：指定输出目录名称。

CAT-01305：输出目录{0}不为空。

原因：输出目录不为空。

操作：指定其他输出目录或删除该目录中的文件。

CAT-01503：未执行操作“{0}”。

原因：由于连接问题或数据差异，请求的操作被拒绝。

操作：检查日志，如果问题是由数据差异引起的，请发出'vmcpu list'命令。

CAT-01504：缺少必需选项-hostname。\ n用法：vmcpu delete -hostname hostname 。

原因：尝试从VM主机删除CPU被拒绝，因为未指定强制选项-hostname。

操作：重新发出提供-hostname选项的'vmcpu delete'命令。

CAT-01505：带有'add'或'update'的'vmcpu'命令应该只有这三个必需选项-hostname，-cores和-strands。

原因：尝试添加或更新VM主机的cpu被拒绝，因为指定的选项不合适。

操作：重新发出命令，仅提供必需的选项-hostname，-cores和-strands。

CAT-01506：核心数不应超过股数。

原因：核心数量多于股数。

操作：重新发出提供不少于核心数的链的命令。

CAT-01507：缺少必需的“{0}”选项。

原因：请求的操作被拒绝，因为未提供指示的强制选项。

操作：重新发出提供“自述文件”文件中指示的强制选项的命令。

CAT-01508：提供给{0}选项的值不是正整数。

原因：尝试添加或更新VM主机的cpu被拒绝，因为为strand或cores指定的值不是正数。

操作：重新发出提供正数作为链或核的值的命令。

CAT-01509：绞线数不是核心数的倍数。

原因：尝试添加或更新VM主机的cpu被拒绝，因为为strand count指定的值不是为core count指定的值的倍数。

操作：重新发出命令，将链计数指定为核心计数的倍数。
