XDK for UNIX上的C ++组件依赖项

本节中描述的C ++库位于$ORACLE_HOME/lib 。

C和C ++组件的XDK包含在库中：

libxml11.a

除了在UNIX上使用XDK for C Component Dependencies中描述的XDK for C组件之外 ，该库还包括XML类生成器，该生成器基于输入文档类型定义（DTD）或XML模式创建C ++源文件。

UNIX上的C组件依赖关系XDK中的 表3-1描述了XDK for C组件（UNIX）所依赖的Oracle CORE和全球化支持库。C和C ++的库依赖关系是相同的。

在UNIX上设置和测试XDK for C ++编译时环境

描述了如何设置和测试XDK C ++编译时UNIX环境。

C和C ++头文件都位于$ORACLE_HOME/xdk/include 。表26-1介绍了C ++头文件。在UNIX上设置和测试XDK C编译时环境的 表3-4介绍了C头文件。必须先设置运行时环境，然后才能编译C ++代码。

表26-1 XDK for C ++编译时环境中的头文件

头文件 描述

oraxml.hpp

包括Oracle9i XML ORA数据类型和libxml.a包含的公共ORA应用程序编程接口（API）（仅用于向后兼容）。

oraxmlcg.h

包含C ++类生成器的C API（仅用于向后兼容）。

oraxsd.hpp

包括Oracle9i XML架构定义（XSD）验证器数据类型和API（仅用于向后兼容）

xml.hpp

无论您是通过Oracle Call Interface（OCI）还是独立使用统一文档对象模型（DOM）API，都可以透明地处理它们

xmlotn.hpp

包括公共API，无论您是独立编译还是使用OCI和Unified DOM

xmlctx.hpp

包括初始化和异常处理公共API
在UNIX上测试XDK for C ++编译时环境

测试编译时环境的最简单方法是在示例程序上运行make实用程序。

演示程序位于示例介质而不是Oracle数据库CD上。安装这些程序后，它们位于$ORACLE_HOME/xdk/demo/cpp 。

通过在系统提示符下执行以下命令来构建和运行示例程序：

cd $ ORACLE_HOME / xdk / demo / cpp make

在UNIX上验证XDK for C ++组件版本

如何确定您拥有的XDK版本。

要获取您正在使用的XDK版本，请更改为$ORACLE_HOME/lib并在系统提示符下运行此命令：

字符串libxml10.a | grep -i开发人员

为C ++组件配置XDK的Windows环境

这里的主题包括组件依赖项，环境变量，测试运行时环境，设置和测试编译时环境以及Visual C / C ++。

XDK for Windows上的C ++组件依赖项

本节中描述的C ++库位于%ORACLE_HOME%\lib 。

C和C ++组件的XDK包含在此Windows库中：

libxml10.dll

Windows上C组件依赖关系的XDK 表3-5描述了Windows的C组件所依赖的Oracle公共Oracle运行时环境（CORE）和全球化支持库。C和C ++的库依赖关系是相同的。

在Windows上为C ++环境变量设置XDK

与XDK一起使用所需的Windows环境变量对于C和C ++是相同的。

请参阅“ 在Windows上为C环境变量设置XDK”中的 表3-6 。

在Windows上测试XDK for C ++ Runtime Environment

您可以通过运行UNIX的任何XDK C实用程序来测试您的环境。这些实用程序没有C ++版本。

在Windows上测试XDK for C运行时环境的 表3-7中描述了这些实用程序。

在Windows上设置和测试XDK for C ++编译时环境

描述了如何设置和测试XDK C ++编译时Microsoft Windows环境。

UNIX上设置和测试XDK for C ++编译时环境一节中的表26-1描述了在Windows上编译C组件所需的头文件。UNIX和Windows安装上的相对文件名相同。

在Windows上，头文件位于%ORACLE_HOME%\xdk\include 。必须先设置运行时环境，然后才能编译代码。

在Windows上测试XDK for C ++编译时环境

您可以通过编译演示程序来测试编译时环境，如果已安装Oracle数据库示例介质，则演示程序位于%ORACLE_HOME%\xdk\demo\cpp 。

设置C ++编译器路径的过程与在Windows上设置C编译器路径的XDK中描述的过程相同。编辑Make.bat文件的过程与在Windows上编辑Make.bat文件中描述的过程相同。
