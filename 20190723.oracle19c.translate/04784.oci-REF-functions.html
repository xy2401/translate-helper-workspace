28.6.1 OCIRefAssign（）

将一个REF分配给另一个，这样两者都引用同一个对象。

目的

将一个REF分配给另一个，这样两者都引用同一个对象。

句法

sword OCIRefAssign（OCIEnv * env，OCIError * err，const OCIRef * source，OCIRef ** target）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

来源（IN）

从中复制的REF 。

目标（IN / OUT）

REF要复制到。

评论

将source REF复制到target REF ;然后两者都引用同一个对象。如果target REF指针为NULL （ *target == NULL ），则OCIRefAssign()在复制操作之前为OCI对象缓存中的target REF分配内存。

28.6.2 OCIRefClear（）

清除或NULL给定REF 。

目的

清除或NULL给定REF 。

句法

void OCIRefClear（OCIEnv * env，OCIRef * ref）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

参考（IN / OUT）

REF清除。

评论

如果REF不再指向对象，则认为它是NULL REF 。逻辑上， NULL REF是一个悬空的REF 。

请注意， NULL REF仍然是有效的SQL值，并且不是SQL NULL 。它可以用作NOT NULL列的有效非NULL常量REF值或表中行的属性。

如果将NULL指针值作为REF传递，则此函数不可操作。

28.6.3 OCIRefFromHex（）

将给定的十六进制字符串转换为REF 。

目的

将给定的十六进制字符串转换为REF 。

句法

sword OCIRefFromHex（OCIEnv * env，OCIError * err，const OCISvcCtx * svc，const OraText * hex，ub4 length，OCIRef ** ref）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

svc（IN）

OCI服务上下文句柄，如果使用此服务上下文初始化结果ref 。

十六进制（IN）

之前由OCIRefToHex()输出的十六进制文本字符串，用于转换为REF 。

长度（IN）

十六进制文本字符串的长度。

参考（IN / OUT）

转换为十六进制字符串的REF 。如果*ref在输入时为NULL ，则REF空间在对象缓存中分配;否则，重用给定REF占用的内存。

评论

此功能可确保生成的REF格式正确。它不能确保生成的REF指向的对象存在。

28.6.5 OCIRefIsEqual（）

比较两个REF以确定它们是否相等。

目的

比较两个REF以确定它们是否相等。

句法

boolean OCIRefIsEqual（OCIEnv * env，const OCIRef * x，const OCIRef * y）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

x（IN）

REF进行比较。

y（IN）

REF进行比较。

评论

当且仅当两个REF都引用相同的对象（无论是持久的还是瞬态的）时，它们是相等的。

注意：

此函数认为两个NULL REF 不相等。

返回

如果两个REF相等，则为TRUE 。

FALSE ，如果两个REF不相等，或者x为NULL ，或者y为NULL.

28.6.6 OCIRefIsNull（）

测试REF是否为NULL 。

目的

测试REF是否为NULL 。

句法

boolean OCIRefIsNull（OCIEnv * env，const OCIRef * ref）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

参考（IN）

REF测试NULL 。

评论

当且仅当以下情况时， REF为NULL ：

  • 它应该是引用持久对象，但对象的标识符是NULL

  • 它应该引用一个瞬态对象，但它当前没有指向一个对象

    注意：

    如果REF指向的对象不存在，则REF是悬空REF 。

返回

如果给定的REF为NULL ，则返回TRUE ;否则，它返回FALSE 。

28.6.7 OCIRefToHex（）

将REF转换为十六进制字符串。

目的

将REF转换为十六进制字符串。

句法

sword OCIRefToHex（OCIEnv * env，OCIError * err，const OCIRef * ref，OraText * hex，ub4 * hex_length）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

参考（IN）

REF要转换为十六进制字符串;如果ref是NULL REF （即OCIRefIsNull （ ref ）== TRUE ），则返回零hex_length值。

十六进制（OUT）

缓冲区，其大小足以包含生成的十六进制字符串;字符串的内容对调用者是不透明的。

hex_length（IN / OUT）

输入时，指定hex缓冲区的大小; on output，指定以十六进制格式返回的hex进制字符串的实际大小。

评论

将给定的REF转换为十六进制字符串，并返回字符串的长度。结果字符串对调用者是不透明的。

返回

如果给定的缓冲区不足以容纳结果字符串，则此函数返回错误。
