11解决记忆压力

Memory Guard通过在节点因内存不足而受到压力时阻止节点被驱逐，持续监视并确保集群节点的可用性。

  • 记忆保护概述
    Memory Guard自动监视群集节点，以防止由于内存不足而导致的节点压力。
  • 内存保护架构
    Memory Guard实现为作为由Cluster Ready Services（CRS）管理的J2EE容器中的MBean运行的守护程序。
  • 在Oracle Real Application Clusters（Oracle RAC）环境中启用Memory Guard
    为Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库安装Oracle Grid Infrastructure时，会自动启用Memory Guard。
  • 在Oracle Real Application Clusters（Oracle RAC）部署中使用Memory Guard
    Memory Guard在打开时自动检测和监控Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库。
