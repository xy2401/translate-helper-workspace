表16-59包含acfsutil sec init命令可用的选项。

表16-59 acfsutil sec init命令的选项

选项 描述

-u admin

指定第一个安全管理员用户名。指定的用户必须是现有操作系统（OS）用户和-g选项指定的操作系统组的成员。

在Windows上，必须使用domain_name \ username形式的完全限定用户名指定安全管理员用户username 。

-g admin_sec_group

指定管理员的安全组的名称。指定的组必须是现有的操作系统（OS）组。

在Windows上，必须使用domain_name \ groupname形式的完全限定的域组名指定组名。如果domain_name \ groupname包含空格，则将该字符串括在引号（“”）中。

acfsutil sec init命令创建安全凭证所需的存储，并将操作系统用户标识为第一个安全管理员。该命令还标识作为指定安全组的操作系统组。作为安全管理员的所有用户必须是指定安全组的成员。安全管理员对所有Oracle ACFS文件系统都很常见。

如果要设置OS用户和OS组，请参阅特定于操作系统的（OS）文档以获取信息。

acfsutil sec init命令运行一次，为每个集群设置Oracle ACFS安全性，并且可以从集群中的任何节点运行。其他安全命令也可以从群集中的任何节点运行。

只有root用户或Windows Administrator用户才能运行此命令。用户为安全管理员指定密码。安全管理员密码必须符合以下格式：

  • 最大字符数为20。

  • 最小字符数为8。

  • 密码必须至少包含一位数字。

  • 密码必须至少包含一个字母。

新安全管理员可以使用acfsutil sec admin password命令更改password 。有关信息，请参阅“ acfsutil sec admin密码 ” 。

允许安全管理员浏览Oracle ACFS文件系统中的所有目录，无论它们是否具有基础操作系统权限以及是否有任何领域检查允许它。此异常使安全管理员可以在使用Oracle ACFS安全领域保护文件时检查文件的位置。但是，如果没有相应的操作系统和安全领域权限，安全管理员无法查看单个文件的内容。

目的

将Oracle ACFS安全元数据加载到由装入点标识的文件系统中。

语法和说明

acfsutil sec load -h acfsutil sec load -m mount_point -p file

acfsutil sec load -h显示帮助文本并退出。

表16-60包含acfsutil sec load命令可用的选项。

表16-60 acfsutil sec load命令的选项

选项 描述

-m mount_point

指定安装文件系统的目录。

-p file

指定现有已保存安全性元数据文件的名称。

acfsutil sec load命令将保存的XML文件中的安全元数据加载到指定的Oracle ACFS文件系统中。 acfsutil sec load仅恢复用户创建的安全策略;该命令不会将文件添加到领域。

acfsutil sec load和acfsutil sec save可以一起用于将用户创建的策略从一个文件系统复制到另一个文件系统。例如，如果要在一个文件系统上安装要在其他文件系统上复制的安全策略，则在源文件系统上使用acfsutil sec save来创建XML备份文件。接下来，在其他目标文件系统上使用acfsutil sec load以加载已保存的安全元数据并创建相同的策略。创建策略后，您可以根据要对这些文件施加的策略，通过在不同域中添加目录和文件，选择在该文件系统上的不同目录和文件上应用策略。

要运行acfsutil sec load命令，目标安装点必须具有为安全性而准备的文件系统，并且不包含任何用户创建的安全对象。

如果安装在目标安装点上的文件系统包含安全对象，则必须运行acfsutil sec prepare -u以删除文件系统上所有先前创建的安全对象。成功运行acfsutil sec prepare -u ，必须运行acfsutil sec prepare文件系统以确保安全性。成功运行acfsutil sec prepare ，可以在文件系统上运行acfsutil sec load 。有关在文件系统上准备安全性或从文件系统中删除安全性的信息，请参阅“ acfsutil sec prepare ” 。

acfsutil sec load命令不会从备份文件加载系统安全域。使用acfsutil sec prepare命令创建系统安全域; acfsutil sec load不会重新创建这些领域。有关系统创建的安全领域的信息，请参阅“ acfsutil sec prepare ” 。

只有安全管理员才能运行此命令。

acfsutil sec准备

目的

准备Oracle ACFS文件系统以获取安全功能。

语法和说明

acfsutil sec prepare -h acfsutil sec prepare [-u] -m mount_point

acfsutil sec prepare -h显示帮助文本并退出。

表16-61包含acfsutil sec prepare命令可用的选项。

表16-61 acfsutil sec prepare命令的选项

选项 描述

-m mount_point

指定安装文件系统的目录。

-u

退出指定安装点的安全性。

此命令从文件系统中删除安全性，并将文件系统恢复到在文件系统上运行acfsutil sec prepare之前的状态。

此命令从域中删除所有域安全文件和目录，然后从文件系统中销毁所有Oracle ACFS安全规则，规则集和域。但是， .Security目录及其内容（包括日志文件和安全元数据备份文件）不会被删除。

如果要删除加密并且正在使用安全性，则必须在取消加密之前运行此命令。要取消加密，请参阅“ acfsutil encr set ” 。

必须在任何领域管理命令之前运行acfsutil sec prepare命令。此命令准备指定的Oracle ACFS文件系统以确保安全性，并且默认情况下会打开文件系统的安全性。

运行acfsutil sec prepare -u ，请确保在acfsutil sec prepare完成之前acfsutil运行任何其他Oracle ACFS安全命令。

如果在群集上初始化审核，则此命令还会在文件系统上启用Oracle ACFS安全审核源。启用此审计源时执行的操作与直接运行acfsutil audit enable命令时执行的操作相同。有关更多信息，请参阅“ acfsutil audit enable ” 。

此命令创建/ mount_point /.Security ， / mount_point /.Security/backup和/ mount_point /.Security/realm/logs目录，其中mount_point是命令行中指定的选项。

此命令创建以下系统安全域：

  • SYSTEM_Logs

    这是一个系统创建的领域，用于保护目录中的Oracle ACFS安全日志文件.Security/realm/logs/目录。

  • SYSTEM_Audit

    这是一个系统创建的领域，用于保护审计跟踪文件。如果已初始化审计，则会创建此领域。如果尚未初始化审计，则会在通过acfsutil audit enable命令为acfsutil enable audit时创建审计。此领域保护审计跟踪文件，以便审计经理可以读取和写入，审计员可以读取该文件，并且没有其他人可以访问。此领域还保护审计跟踪文件，以便审计管理器无法删除（不运行acfsutil audit purge命令），截断，覆盖或chmod文件。

  • SYSTEM_SecurityMetadata

    这是一个系统创建的领域，用于保护目录中的Oracle ACFS元数据XML文件.Security/backup/目录。

  • SYSTEM_Antivirus

    这是一个系统创建的领域，允许访问在Oracle ACFS文件系统上运行的防病毒软件。对于每个受领域保护的文件或目录，在执行授权检查时评估SYSTEM_Antivirus以确定SYSTEM_Antivirus是否允许访问该文件或目录。

    要允许防病毒进程访问受域保护的文件或目录，必须使用acfsutil sec realm add命令将LocalSystem或SYSTEM组添加到acfsutil ，如例16-58所示 。如果其他防病毒进程以Administrator身份运行，则必须将用户Administrator添加到SYSTEM_Antivirus领域，以允许访问受领域保护的文件和目录。

    如果未安装防病毒产品，请不要将任何用户或组添加到SYSTEM_Antivirus领域。由于添加到SYSTEM_Antivirus领域的用户或组具有READ和READDIR访问权限，因此限制添加到此领域的用户或组。当域的用户或组可以使用基于时间的规则访问域受保护的文件或目录时，可以限制时间窗口。如果您可以识别扫描文件的防病毒安装的进程名称，则还可以使用基于应用程序的规则。

    SYSTEM_Antivirus领域只能对文件或目录执行以下操作： OPEN ， READ ， READDIR和设置时间属性。要删除或删除文件或目录，您可能需要禁用安全性来清除受感染的文件。

    此领域仅适用于Windows系统。

  • SYSTEM_BackupOperators

    这是一个系统创建的领域，使您可以授权可以备份领域安全文件和目录的用户。您可以向此领域添加用户，组，规则集和命令规则，以便为备份领域安全的文件和目录提供细粒度授权。必须将用户添加到此域以备份域安全文件和目录。

    将组添加到此系统领域时要小心。将组添加到此系统域后，添加的组的所有用户都可以覆盖领域保护以访问文件。

要访问系统安全领域中的文件，应使用acfsutil sec admin add命令将用户指定为安全管理员。

您可以使用acfsutil sec realm add命令将用户，组，规则集和命令规则添加到系统创建的领域，与用户创建的领域相同。但是，不建议将文件和目录添加到系统领域。您可以使用acfsutil sec realm delete命令从系统创建的域中删除对象。

安全管理员无法使用acfsutil sec admin destroy命令删除系统创建的安全领域。只有在使用-u选项执行acfsutil sec prepare命令时从文件系统中退出安全性时，才会删除这些域。

如果文件系统中存在任何快照，则不允许使用acfsutil sec prepare –u命令。

只有安全管理员才能运行acfsutil sec prepare命令。

表16-63安全领域命令规则

规则 描述

所有

防止文件和目录上的所有文件系统操作。

APPENDFILE

禁止添加到文件末尾。限制包括在当前文件大小内开始但在文件末尾之后继续的写入。

chgrp命令

防止更改文件或目录上的组所有权。

CHMOD

防止更改文件或目录的权限。

CHOWN

防止更改文件或目录的所有者信息。

CREATEFILE

防止在目录中创建新文件。

防止从目录中删除文件。

延伸

拒绝对acfsutil的文件和目录进行任何更改，除了由acfsutil tag和acfsutil encr等命令导致的扩展属性的更改。

IMMUTABLE不会拒绝对atime属性进行任何更改。用户访问文件时， atime属性会更改。

可以设置为存档安全领域中的文件和目录。

LINKFILE

限制创建文件的硬链接。

MKDIR

防止在目录中创建新目录。

MMAPREAD

使用Linux上的mmap()或在Windows上使用CreateFileMapping然后使用MapViewOfFile() ，保护文件不被内存映射以进行读取操作。

MMAPWRITE

保护文件不被写入操作的内存映射。设置MMAPWRITE还可以保护文件不被映射以进行读取，因为操作系统会为读取和写入映射文件。

打开文件

防止打开文件。

OVERWRITE

防止文件中的现有内容被write操作覆盖，该操作的开始和结束偏移量在当前文件大小范围内。

如果文件上的操作是truncate后跟append ，则OVERWRITE不保护文件。要为append和overwrite操作提供额外保护，请使用WRITE命令规则。

READDIR

限制目录列表，安全管理员组使用除外。

读

防止读取文件的内容。READ还可以使用mmap(2)防止read操作。

改名

防止重命名文件或目录。

RMDIR

防止删除目录。

SYMLINK

限制在受安全领域保护的目录中创建符号链接。创建符号链接时，源文件是否受安全领域保护无关紧要。

截短

限制文件的截断。

写

保护文件免受write系统调用。WRITE还可以防止append和overwrite操作，以及使用mmap(2) write操作。

文件仍可以使用其他文件操作进行修改。要保护文件免受其他修改，还可以使用TRUNCATE和DELETEFILE命令规则。

只有安全管理员才能运行此命令。

acfsutil sec rule创建

表16-72包含acfsutil sec rule create命令可用的选项。

表16-72 acfsutil sec rule create命令的选项

选项 描述

规则

指定规则的名称。如果名称包含空格，请用引号（ " " ）括起来。

-m mount_point

指定安装文件系统的目录。

-t rule_type rule_value

指定规则类型和规则值。规则类型可以是application ， hostname ， time或username 。规则值取决于规则的类型。本节描述了有效的规则类型和值。

-o option

指定以-o选项。指定的选项可以是ALLOW或DENY 。默认值为DENY 。

acfsutil sec rule create命令在挂载点指定的Oracle ACFS文件系统中创建新规则。可以将新规则添加到规则集中，并将该规则集添加到安全领域。

最多可以创建500个Oracle ACFS安全规则。

规则类型和关联的规则值是：

  • 应用

    此规则类型指定允许或拒绝访问受域保护的对象的应用程序的名称。

  • 主机名

    此规则类型指定用户从中访问受域保护的对象的计算机的名称。使用此规则可以允许或拒绝来自节点的访问。hostname应该是群集节点名之一，而不是可以将Oracle ACFS文件系统作为网络文件系统（NFS）装载安装的任何其他外部节点。

  • 时间

    此规则类型以start_time ， end_time格式指定时间间隔。此时间间隔指定对领域的访问。通过在一个领域中设置此规则，仅允许在一天的特定时间段内对域保护的对象进行访问。时间基于主持人的当地时间。

  • 用户名

    此规则类型指定要从领域添加或删除的用户的名称。您可以使用此选项拒绝属于属于域的安全组的任何用户的访问权限。

只有安全管理员才能运行此命令。
