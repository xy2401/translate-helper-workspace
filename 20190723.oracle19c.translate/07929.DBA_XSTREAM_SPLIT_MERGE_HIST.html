6.159 DBA_XSTREAM_SPLIT_MERGE_HIST

DBA_XSTREAM_SPLIT_MERGE_HIST显示有关过去XStream自动拆分和合并操作的信息。

柱 数据类型 空值 描述

ORIGINAL_CAPTURE_NAME

VARCHAR2（128）

不是NULL

原始捕获过程的名称

CLONED_CAPTURE_NAME

VARCHAR2（128）

克隆的捕获进程的名称

ORIGINAL_QUEUE_OWNER

VARCHAR2（128）

原始捕获进程使用的队列的所有者

ORIGINAL_QUEUE_NAME

VARCHAR2（128）

原始捕获进程使用的队列名称

CLONED_QUEUE_OWNER

VARCHAR2（128）

克隆捕获进程使用的队列的所有者

CLONED_QUEUE_NAME

VARCHAR2（128）

克隆捕获进程使用的队列名称

ORIGINAL_CAPTURE_STATUS

VARCHAR2（8）

原始捕获过程的状态：

  • 禁用

  • 启用

  • ABORTED

CLONED_CAPTURE_STATUS

VARCHAR2（8）

克隆捕获过程的状态：

  • 禁用

  • 启用

  • ABORTED

ORIGINAL_XSTREAM_NAME

VARCHAR2（128）

直接从原始捕获进程接收数据库更改的原始XStream组件的名称。该组件是一个progagation或一个本地应用程序。

CLONED_XSTREAM_NAME

VARCHAR2（128）

克隆的XStream组件的名称，该组件直接从克隆的捕获进程接收数据库更改。该组件是一个progagation或一个本地应用程序。

XSTREAM_TYPE

VARCHAR2（11）

ORIGINAL_XSTREAM_NAME和CLONED_XSTREAM_NAME组件的类型：

  • 传播

  • 应用

RECOVERABLE_SCRIPT_ID

RAW（16）

要拆分或合并操作的脚本的唯一ID

SCRIPT_STATUS

VARCHAR2（12）

可恢复脚本的状态：

  • GENERATING

  • 不执行

  • EXECUTING

  • EXECUTED

  • 错误

ACTION_TYPE

VARCHAR2（7）

脚本执行的操作类型：

  • 分裂

  • 合并

  • 监控

ACTION_THRESHOLD

VARCHAR2（40）

对于SPLIT操作，由split_threshold捕获进程参数设置的阈值。对于MERGE操作，由merge_threshold捕获进程参数设置的阈值。

状态

VARCHAR2（16）

行动现状：

  • NOTHING TO SPLIT - 没有准备拆分或不需要拆分

  • 关于分裂

  • SPLITTING - 分裂正在进行中

  • SPLIT DONE - 完成拆分

  • NOTHING TO MERGE合并 - 还没准备合并

  • 关于合并

  • MERGING -合并正在进行中

  • 合并MERGE DONE - 合并完成

  • ERROR - 拆分或合并期间返回错误

  • NONSPLITTABLE - 原始捕获NONSPLITTABLE ，因为它被禁用，它有多个发布者到其队列，或者它只有一个目标用于捕获的消息

STATUS_UPDATE_TIME

TIMESTAMP（6）

上次更新状态的时间

CREATION_TIME

TIMESTAMP（6）

行动开始的时间

落后

数

克隆捕获进程滞后于原始捕获进程的时间（以秒为单位）

JOB_OWNER

VARCHAR2（128）

执行拆分或合并操作的作业的所有者

JOB_NAME

VARCHAR2（128）

执行拆分或合并操作的作业的名称

ERROR_NUMBER

数

如果捕获过程中止，则出现错误编号

错误信息

VARCHAR2（4000）

如果捕获过程已中止，则会显示错误消息
