A.6控制守护进程的行为

使用本节中的命令列表来控制守护程序的行为。

句法

[-id id ] -set daemon_option [-id id ] -unset daemon_option |所有[-id id ] -get parameter |全部[-d start] [-d start -debug] [-d stop] [-d stop_client] [-d status] [-d info] [-id id ] -d nextautorun [-initsetup] [-initrmsetup] [ -initcheck]

参数

表A-5守护程序选项

选项 描述

[-id id ] -set daemon_option

（可选）使用id和set命令设置特定的守护程序使用情况配置文件。

[-id id ] -unset daemon_option |所有

取消设置参数。

与–id id配合使用可设置守护程序特定于配置文件的值。

[-id id ] -get parameter |所有

显示指定参数的值或所有参数。

与–id id配合使用可设置守护程序特定于配置文件的值。

-d开始

启动守护进程。

-d start -debug

以调试模式启动守护程序。

-d停止

停止守护进程。

-d stop_client

强制正在运行的守护程序客户端停止。

-d状态

检查守护程序的当前状态。

-d信息

显示有关守护程序的详细信息。

详细信息包括安装以及守护程序何时启动。

[-id id ] -d nextautorun

显示有关下次计划的自动运行何时发生的详细信息

-initsetup

设置在节点启动时启动守护程序的守护程序自动重新启动功能。

-initrmsetup

删除自动重启功能。

-initcheck

检查是否已设置自动重启功能。
