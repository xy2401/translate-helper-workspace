255 UTL_HTTP

UTL_HTTP包从SQL和PL / SQL生成超文本传输协议（HTTP）标注。您可以使用它通过HTTP访问Internet上的数据。

当程序包使用HTTPS从Web站点获取数据时，它需要Oracle Wallet Manager，可以由Oracle Wallet Manager或orapki实用程序创建。非HTTPS提取不需要Oracle钱包。

255.1 UTL_HTTP概述

使用UTL_HTTP包，您可以编写与Web（HTTP）服务器通信的PL / SQL程序。UTL_HTTP还包含一个可以在SQL查询中使用的函数。

该软件包支持通过安全套接字层协议（SSL）的HTTP，也称为HTTPS。它还通过在钱包中发送客户端证书以通过远程Web服务器进行身份验证来支持SSL客户端身份验证。

使用支持这些协议的HTTP代理服务器也支持其他与Internet相关的数据访问协议（如文件传输协议（FTP）或Gopher协议）。

255.2 UTL_HTTP安全模型

这个包是一个调用者的权利包。调用用户将需要在分配给他想要连接的远程网络主机的访问控制列表中授予的connect权限，以及use-client-certificates或use-passwords权限以使用远程Web服务器对自己进行身份验证使用存储在Oracle钱包中的凭据。

255.6.2 REQUEST_CONTEXT_KEY类型

此类型用于表示请求上下文的键。

请求上下文是一个上下文，它包含一个私有钱包和cookie表来发出HTTP请求。与包中维护的会话范围不同，此私有钱包和cookie表将不会与数据库会话中的其他HTTP请求共享。

句法

SUBTYPE request_context_key IS PLS_INTEGER;

使用说明

为了提供增强的安全性， UTL_HTTP允许PL / SQL程序创建请求上下文。请求上下文是一个私有上下文，它包含一个钱包和一个cookie表，在发出HTTP请求和接收HTTP响应时，这些表不会与同一数据库会话中的其他程序共享。当PL / SQL程序需要使用包含敏感信息（如身份验证凭据）的钱包或cookie时，它们应使用请求上下文。

255.6.5连接类型

根据HTTP 1.1协议规范，使用PL / SQL记录类型表示在HTTP请求完成后保持持久的网络连接的远程主机和TCP / IP端口。持久网络连接可以由对同一主机和端口的后续HTTP请求重用。随后的HTTP请求可以更快地完成，因为避免了网络连接等待时间。 connection_table是PL / SQL connection表。

对于与Web服务器的直接HTTP持久连接， host和port字段包含Web服务器的主机名和TCP / IP端口号。未设置proxy_host和proxy_port字段。对于以前用于使用代理连接到Web服务器的HTTP持久连接， proxy_host和proxy_port字段包含代理服务器的主机名和TCP / IP端口号。未设置主机和端口字段，这表示持久连接在连接到代理服务器时未绑定到任何特定目标Web服务器。与代理服务器的HTTP持久连接可用于访问使用代理的任何目标Web服务器。

SSL字段指示是否在HTTP持久连接中使用安全套接字层（SSL）。HTTPS请求是通过SSL发出的HTTP请求。对于使用代理连接的HTTPS（SSL）持久连接，主机和端口字段包含目标HTTPS Web服务器的主机名和TCP / IP端口号，并且将始终设置字段。使用代理服务器的HTTPS Web服务器的HTTPS持久连接只能重复用于向同一目标Web服务器发出另一个请求。

句法

255.7.2 UTL_HTTP简单HTTP 提取

您不应期望REQUEST或REQUEST_PIECES成功联系URL，除非您可以通过在同一台计算机上使用浏览器（并具有相同的权限，环境变量等）来联系该URL。

如果REQUEST或REQUEST_PIECES失败（例如，如果它引发异常，或者它返回HTML格式的错误消息，但您认为URL参数是正确的），则尝试使用浏览器联系同一URL以验证网络可用性从你的机器。您可能在浏览器中设置了一个代理服务器，需要使用可选的proxy参数为每个REQUEST或REQUEST_PIECES调用设置。

注意：

UTL_HTTP还可以使用环境变量来指定其代理行为。例如，在UNIX上，将环境变量http_proxy设置为URL将该服务用作HTTP请求的代理服务器。将环境变量no_proxy设置为域名不会将HTTP代理服务器用于该域中的URL。在Oracle数据库服务器中执行UTL_HTTP程序包时，环境变量是在启动数据库实例时设置的变量。

参数

表255-18 BEGIN_REQUEST函数参数

参数 描述

网址

HTTP请求的URL

方法

发送请求的HTTP协议版本。协议版本的格式为HTTP/major-version.minor-version ，其中major-version和minor-version为正数。如果此参数设置为NULL ，则UTL_HTTP使用它支持的最新HTTP协议版本来发送请求。该软件包支持的最新版本为1.1，可以升级到更高版本。默认值为NULL 。

    req：= UTL_HTTP.BEGIN_REQUEST（url => the_url，method =>'POST'）; UTL_HTTP.SET_HEADER（r => req，name =>'Content-Type'，value =>'application / x-www-form-urlencoded'）; UTL_HTTP.SET_HEADER（r => req，name =>'Content-Length'，value =>'<以字节为单位的数据长度>'）; UTL_HTTP.WRITE_TEXT（r => req，data =>'p1 = value1＆p2 = value2 ...“）; resp：= UTL_HTTP.GET_RESPONSE（r => req）; ...

    程序员必须确定特定的Web服务器是否可以接受以这种方式提供的数据。

  • 在通过HTTPS访问Web服务器之前，必须先设置Oracle钱包。有关如何设置Oracle钱包的信息，请参阅SET_WALLET过程 。要使用SSL客户端身份验证，客户端证书应存储在钱包中，并且调用者必须在钱包上具有use-client-certificates特权。请参阅“ Oracle数据库安全指南 ”中的“管理对外部网络服务的细粒度访问 ”以授予权限。

  • 要直接连接到远程Web服务器，或通过HTTP代理间接connect ， UTL_HTTP必须分别具有到远程Web服务器主机或代理主机的connect ACL权限。

255.9.5 CLOSE_PERSISTENT_CONNS过程

此过程将关闭当前数据库会话中由UTL_HTTP程序包维护的一组HTTP持久连接。此过程使用模式匹配方法来确定要关闭的持久连接。

UTL_HTTP.CLOSE_PERSISTENT_CONNS（host =>'foobar'）;

UTL_HTTP.CLOSE_PERSISTENT_CONNS（proxy_host =>'foobar'，proxy_port => 80）;

以下对该过程的调用将关闭所有持久连接：

UTL_HTTP.CLOSE_PERSISTENT_CONNS;

使用说明

HTTP客户端必须始终让远程Web服务器知道它正在发送的请求主体的长度。如果事先已知数据量，则可以在请求中设置Content-Length标头，其中内容的长度以字节而不是字符来度量。如果事先不知道请求主体的长度，则可以使用HTTP 1.1分块传输编码格式发送请求主体。请求体以块的形式发送，其中每个块的长度在块发送之前发送。当设置Transfer-Encoding：chunked头时， UTL_HTTP包在请求主体上透明地执行分块传输编码。请注意，某些基于HTTP-1.1的Web服务器或CGI程序不支持或接受HTTP 1.1分块传输编码格式的请求正文编码。有关详细信息，请参阅SET_HEADER过程。

如果发送Content-Length标头，则应注意标头中指定的长度应该是文本请求主体从数据库字符集转换为请求主体字符集后的字节长度。当两个字符集中的任何一个是多字节字符集时，请求主体字符集中的请求主体的精确字节长度不能预先知道。在这种情况下，您可以显式执行字符集转换，确定结果的字节长度，使用WRITE_RAW过程发送Content-Length标头和结果，以避免自动字符集转换。或者，如果删除Web服务器或CGI程序允许，您可以使用HTTP 1.1分块传输编码格式发送请求正文，其中UTL_HTTP透明地处理块的长度。
