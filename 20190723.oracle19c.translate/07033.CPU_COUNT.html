1.55 CPU_COUNT

CPU_COUNT指定可供Oracle数据库使用的CPU数。

属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

是

价值范围

0到无限制

基本

没有

在具有多个CPU线程的CPU上，它指定可用CPU线程的总数。Oracle数据库的各种组件都是根据CPU的数量配置的，例如优化器，并行查询和资源管理器。

如果CPU_COUNT设置为0 （其默认设置），则Oracle数据库会持续监视操作系统报告的CPU数并使用当前计数。如果CPU_COUNT设置为0以外的值，则Oracle数据库将使用此计数而不是实际的CPU数，从而禁用动态CPU重新配置。

当资源管理器管理CPU（设置了RESOURCE_MANAGER_PLAN ）时，数据库的CPU利用率仅限于CPU_COUNT CPU线程。此功能称为实例锁定。如果在CDB级别启用了资源管理器，则PDB的CPU利用率仅限于PDB的CPU_COUNT 。

注意：

将CPU_COUNT设置为大于当前CPU数的值会导致错误。但是，如果将CPU_COUNT设置为大于初始化参数文件中当前CPU数的值，则CPU_COUNT将限制为当前CPU数。
