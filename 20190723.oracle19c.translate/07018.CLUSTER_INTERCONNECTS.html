1.42 CLUSTER_INTERCONNECTS

可以在Oracle Real Application Clusters环境中使用CLUSTER_INTERCONNECTS来指示可用于数据库流量的群集互连。

属性 描述

参数类型

串

句法

CLUSTER_INTERCONNECTS = ifn [: ifn ] ...

默认值

没有默认值。

可修改

没有

可在PDB中修改

没有

价值范围

一个或多个IP地址，以冒号分隔

基本

没有

使用此参数可覆盖为数据库通信配置的默认互连，该通信存储在群集注册表中。对于具有降低的可用性要求和高互连带宽需求的数据仓库系统，此过程也可能很有用。

CLUSTER_INTERCONNECTS专门覆盖以下内容：

  • oifcfg在OCR中存储的网络分类。

  • Oracle选择的默认互连。

如果要对互连进行负载平衡，则Oracle建议您在操作系统级别使用链接绑定，即使在同一服务器上有两个数据库，以便多个互连使用相同的地址。请注意，多个专用地址提供负载平衡，但除非已绑定，否则不提供故障转移。如果使用CLUSTER_INTERCONNECTS在init.ora中指定多个地址，而不是在操作系统级别绑定多个地址，则通常会降低可用性，因为每个网络接口卡故障都会关闭该实例。

有关绑定接口的信息，请参阅供应商文档。某些供应商绑定体系结构可能需要使用此参数。

如果Oracle Real Application Clusters节点上有多个数据库实例，并且想要为每个实例使用特定接口，则可以将CLUSTER_INTERCONNECTS初始化参数设置为每个数据库实例的IP地址。例如：

hr1.init.ora.cluster_interconnects =“192.0.2.111”oltp3.init.ora.cluster_interconnects =“192.0.2.112”

如果Oracle RAC互连配置为在与Oracle Clusterware互连不同的接口上运行，则此配置可能会导致可用性降低，因为如果Oracle RAC互连在Oracle Clusterware NIC保持运行时发生故障，则可能会延迟故障转移或实例驱逐。
