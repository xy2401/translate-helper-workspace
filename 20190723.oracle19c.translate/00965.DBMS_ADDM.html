14.2.1 ANALYZE_DB过程

此过程创建ADDM任务，以便在数据库分析模式下进行分析并执行它。

句法

DBMS_ADDM.ANALYZE_DB（task_name IN OUT VARCHAR2，begin_snapshot IN NUMBER，end_snapshot IN NUMBER，read_only_type_override IN VARCHAR2，db_id IN NUMBER：= NULL）;

参数

表14-2 ANALYZE_DB过程参数

参数 描述

任务名称

要创建的任务的名称。

begin_snapshot

启动分析周期的快照编号。

end_snapshot

结束分析周期的快照编号。

read_only_type_override
覆盖CDB ADDM确定的分析类型。可能的值是：

  • READ-WRITE - 数据保护配置中的常规数据库或主数据库

  • READ-ONLY - 数据库以只读模式打开，例如活动数据保护备用数据库

  • AUTO -允许ADDM来决定的类型CDB重写根据数据

DB_ID

要分析的数据库的数据库ID。默认情况下，这是当前连接的数据库。

返回值

在task_name参数中返回已创建任务的名称。它可能与作为输入给出的值不同（仅在名称已被其他任务使用的情况下）。

例子

要在数据库分析模式下创建ADDM任务并执行它，其名称在变量tname ：

var tname VARCHAR2（60）; BEGIN：tname：='my_database_analysis_mode_task'; DBMS_ADDM.ANALYZE_DB（：tname，1,2）;结束

要查看报告：

SET LONG 100000 SET PAGESIZE 50000 SELECT DBMS_ADDM.GET_REPORT（：tname）FROM DUAL;

请注意，报表的返回类型是CLOB ，格式设置为适合80行的大小。

14.2.13 GET_ASH_QUERY函数

该函数返回一个字符串，其中包含ASH查询的SQL文本，该查询标识ASH中的行，并对查找产生影响。

对于大多数类型的发现，这确定了与该发现相对应的ASH中的确切行。对于某些类型的发现，查询是近似值，不应用于精确识别发现的影响或发现的特定活动。

句法

DBMS_ADDM.GET_ASH_QUERY（task_name IN VARCHAR2，finding_id IN NUMBER）RETURN VARCHAR2;

参数

参数 描述

任务名称

发现

返回值

19年2月14日 REAL_TIME_ADDM_REPORT功能

此函数在过去五分钟内为ADDM相关活动生成实时ADDM报告。在Oracle Real Application Clusters（Oracle RAC）环境中，该函数假定可以在GV$执行SQL。

句法

DBMS_ADDM.REAL_TIME_ADDM_REPORT（）cdb_type_override IN VARCHAR2，read_only_type_override IN VARCHAR2，RETURN CLOB;

参数

表14-20 REAL_TIME_ADDM_REPORT函数参数

参数 描述

cdb_type_override
覆盖ADDM为进行分析而确定的CDB类型。可能的值是：

  • PDB内的AUTONOMOUS OLTP

  • AUTONOMOUS DATA WAREHOUSE PDB内的自主数据仓库（ADWH）

  • PDB

    - 常规PDB

  • CDB ROOT

    - CDB的根源

  • NON-CDB

    - 不是CDB或PDB的系统

  • 汽车

    - 允许ADDM根据数据决定要覆盖的CDB类型

read_only_type_override
覆盖CDB ADDM确定的分析类型。可能的值是：

  • READ-WRITE - 数据保护配置中的常规数据库或主数据库

  • READ-ONLY - 数据库以只读模式打开，例如活动数据保护备用数据库

  • AUTO -允许ADDM来决定的类型CDB重写根据数据

返回值

CLOB包含实时ADDM报告
