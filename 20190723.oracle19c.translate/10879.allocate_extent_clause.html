allocate_extent_clause

目的

使用allocate_extent_clause子句为数据库对象显式分配新范围。

使用此子句显式分配扩展区不会更改NEXT和PCTINCREASE存储参数的值，因此不会影响Oracle数据库隐式分配的下一个扩展区的大小。有关NEXT和PCTINCREASE存储参数的信息，请参阅storage_clause 。

您可以在以下SQL语句中分配扩展区：

  • ALTER CLUSTER （参见ALTER CLUSTER ）

  • ALTER INDEX ：为索引，索引分区或索引子分区分配扩展区（请参阅ALTER INDEX ）

  • ALTER MATERIALIZED VIEW ：为物化视图，其分区或子分区之一或索引组织的物化视图的溢出段分配范围（请参阅ALTER MATERIALIZED VIEW ）

  • ALTER MATERIALIZED VIEW LOG （参见ALTER MATERIALIZED VIEW LOG ）

  • ALTER TABLE ：为TABLE分配扩展区，表分区，表子分区，索引组织表的映射表，索引组织表的溢出段或LOB存储段（请参阅ALTER TABLE ）

语义

本节介绍allocate_extent_clause的参数。有关其他信息，请参阅为特定数据库对象设置或重置这些参数的SQL语句。

您不能在同一语句中指定allocate_extent_clause和deallocate_unused_clause 。

尺寸

以字节为单位指定范围的大小。integer的值可以是0到2147483647。要指定更大的扩展区大小，请使用此范围内的整数（ K ， M ， G或T指定扩展区大小（以KB为单位，兆字节，千兆字节或太字节）。

对于表，索引，物化视图或物化视图日志，如果省略SIZE ，则Oracle数据库将根据对象的存储参数值确定大小。但是，对于群集，Oracle不会评估群集的存储参数，因此如果您不希望Oracle使用默认值，则必须指定SIZE 。

DATAFILE“ 文件名 ”

在表，集群，索引，物化视图或物化视图日志的表空间中指定一个数据文件以包含新范围。如果省略DATAFILE ，则Oracle选择数据文件。

INSTANCE 整数

仅在使用Oracle Real Application Clusters时才使用此参数。

指定INSTANCE integer使新范围可用于与指定实例关联的空闲列表组。如果实例编号超过最大空闲列表组数，则Oracle将指定的数字除以最大数字，并使用余数来标识要使用的空闲列表组。实例由其初始化参数INSTANCE_NUMBER的值标识。

如果省略此参数，则会将空间分配给表，集群，索引，物化视图或物化视图日志，但不会从任何特定的空闲列表组中提取空间。相反，Oracle使用主freelist并根据需要分配空间。

注意：

如果使用自动段空间管理，则allocate_extent_clause的INSTANCE参数可能不会为指定实例保留新分配的空间，因为自动段空间管理不会在扩展区和实例之间保持严格的关联。
