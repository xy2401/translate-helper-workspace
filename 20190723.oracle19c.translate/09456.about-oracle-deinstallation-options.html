关于Oracle卸载选项

您可以使用deinstall命令在Oracle数据库主目录中停止和删除Oracle数据库软件和组件。

您可以使用deinstall删除以下软件：

  • Oracle数据库

  • Oracle Grid Infrastructure，包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）

  • Oracle Real Application Clusters（Oracle RAC）

  • Oracle数据库客户端

安装后，Oracle主目录中提供了deinstall命令。它位于$ORACLE_HOME/deinstall目录中。

deinstall使用Oracle主目录中的信息并使用您提供的信息创建响应文件。您可以使用-checkonly选项运行deinstall命令来使用先前生成的响应文件。您还可以编辑响应文件模板。

如果运行deinstall以删除Oracle Grid Infrastructure安装，则卸载程序会提示您以root用户身份运行deinstall命令。对于群集的Oracle Grid Infrastructure，脚本为rootcrs.sh ，对于独立服务器（Oracle Restart）的Oracle Grid Infrastructure，脚本为roothas.sh 。

注意：

  • 您必须从同一版本运行deinstall命令才能删除Oracle软件。请勿从更高版本运行deinstall命令以从早期版本中删除Oracle软件。例如，不要从19c Oracle主目录运行deinstall命令，以从现有的11.2.0.4 Oracle主目录中删除Oracle软件。

  • 与Oracle数据库12c的第1版（12.1.0.2）开始， roothas.sh脚本替换roothas.pl在Oracle网格基础架构回家甲骨文重新启动脚本， rootcrs.sh脚本替换rootcrs.pl脚本在网格家适用于集群的Oracle Grid Infrastructure。

如果Oracle主目录中的软件未运行（例如，安装失败后），则deinstall无法确定配置，您必须以交互方式或在响应文件中提供所有配置详细信息。

此外，在为Oracle Grid Infrastructure安装运行deinstall之前：

  • 卸载Oracle自动存储管理群集文件系统（Oracle ACFS）并禁用Oracle自动存储管理动态卷管理器（Oracle ADVM）。

  • 如果正在使用网格命名服务（GNS），请通知DNS管理员从DNS中删除子域名条目。

deinstall删除的文件

当您运行deinstall ，如果中央库存（ oraInventory ）除了您要解除配置和删除的主页之外不包含其他已注册的主目录，则deinstall将删除Oracle数据库安装所有者的Oracle基本目录中的以下文件和目录内容：

  • 管理

  • cfgtoollogs

  • 检查站

  • 诊断

  • ORADATA

  • fast_recovery_area

Oracle强烈建议您使用Optimal Flexible Architecture（OFA）配置来配置安装，并保留Oracle基础和Oracle主路径以专用于Oracle软件。如果Oracle基础中拥有Oracle软件的用户帐户拥有这些位置中的任何用户数据，则deinstall删除此数据。

警告：

deinstall删除Oracle数据库配置文件，用户数据和快速恢复区（FRA）文件，即使它们位于Oracle基本目录路径之外。
