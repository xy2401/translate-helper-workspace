38 DBMS_CREDENTIAL

DBMS_CREDENTIAL包提供了一个接口，用于验证和模拟EXTPROC标注功能，以及来自SCHEDULER外部作业，远程作业和文件监视器。

38.1 DBMS_CREDENTIAL概述

凭据是包含用户名/密码对的数据库对象，用于验证和模拟EXTPROC标注功能，以及来自SCHEDULER远程作业，外部作业和文件监视器。

它们是使用CREATE_CREDENTIAL过程创建的。该过程还允许您为针对Windows服务器执行的远程外部作业指定Windows域。

38.2 DBMS_CREDENTIAL安全模型

每个Oracle凭证都有一个唯一的凭据名称，您可以通过PL / SQL别名库将凭证通过其唯一的凭证名称与EXTPROC相关联。

为了将凭证与PL / SQL别名库和外部过程相关联，您必须具有CREATE AND / OR REPLACE LIBRARY特权或CREATE AND/OR REPLACE FUNCTION / PROCEDURE特权以及别名的DLL或共享对象的读取权限要关联的库，以便您可以创建和/或替换函数或过程以使用别名库。

经过身份验证后， EXTPROC必须根据提供的用户凭据中定义的客户端身份代表客户端执行操作。如果未经过身份验证， EXTPROC必须返回错误消息。

要创建或更改凭据，您必须具有CREATE CREDENTIAL权限。如果您尝试在自己以外的模式中创建或更改凭证，则必须具有CREATE ANY CREDENTIAL权限。

38.3 DBMS_CREDENTIAL操作说明

由于现有的CREATE OR REPLACE LIBRARY语句和CREATE OR REPLACE FUNCTION/PROCEDURE不支持CREDENTIAL子句，因此该模型需要在CREATE OR REPLACE LIBRARY和CCREATE OR REPLACE FUNCTION/PROCEDUREREATE语句中进行语法和语义更改。

例如：

创建或替换LIBRARY测试AS'$ ORACLE_HOME / bin / test.so'CREDENTIAL ricky_cred;创建或替换函数ftest1（x VARCHAR2，y BINARY_INTEGER）返回BINARY_INTEGER作为语言C库测试名称“否定”参数（x STRING，y INT）;

CREDENTIAL子句中定义的凭据名称是数据库对象的名称。因此，请勿使用单引号或双引号将凭证名称括起来。

在外部作业上使用的凭证示例：

BEGIN DBMS_SCHEDULER.CREATE_JOB（job_name =>'example_job'，job_type =>'EXECUTABLE'，job_action =>'/ bin / ls'，credential_name =>'ricky_cred'）;结束; /

38.4.1 CREATE_CREDENTIAL过程

此过程在名为Oracle凭据的数据库对象中创建存储的用户名/密码对。您还可以使用此过程来管理用于访问存储在云对象存储中的文件的凭据。

句法

DBMS_CREDENTIAL.CREATE_CREDENTIAL（credential_name IN VARCHAR2，username IN VARCHAR2，password IN VARCHAR2，database_role IN VARCHAR2 DEFAULT NULL windows_domain IN VARCHAR2 DEFAULT NULL，注释IN VARCHAR2 DEFAULT NULL，启用IN BOOLEAN DEFAULT TRUE）;

参数

表38-2 CREATE_CREDENTIAL过程参数

参数 描述

credential_name

凭证的名称。它可以选择以模式为前缀。这不能设置为NULL 。它被转换为大写，除非用双引号括起来。

用户名

如果选择此凭据，则登录操作系统或远程数据库以运行作业的用户名。这不能设置为NULL 。

密码

如果选择此凭据，则登录远程操作系统以运行作业的密码。它区分大小写。

DATABASE_ROLE

使用此凭据的数据库作业是否应尝试使用管理权限登录。值： SYSDBA ， SYSDG ， SYSADMIN或SYSBACKUP 。

windows_domain

对于Windows远程可执行目标，这是指定用户所属的域。域将自动转换为大写。

评论

可用于向用户描述凭据的文本字符串。调度程序不使用此字段。

启用

确定是否启用凭据

使用说明

  • 凭据驻留在特定模式中，并且可由具有CREATE CREDENTIAL或CREATE ANY CREDENTIAL系统特权的任何用户CREATE 。要在除您自己的模式之外的模式中创建凭证，您必须具有CREATE CREDENTIAL或CREATE ANY CREDENTIAL权限。

  • 用户名区分大小写。它不能包含双引号或空格。

  • 尝试使用现有凭据名称创建凭据会返回错误。要更改现有凭据，用户必须首先使用DROP_CREDENTIAL过程删除现有凭据。

  • 尝试删除已由别名库引用的现有凭据会返回错误。要在不进行任何检查的情况下删除现有凭据，用户必须将DROP_CREDENTIAL Procedure的force参数设置为TRUE 。

  • 您还可以通过UPDATE_CREDENTIAL过程更改凭证。

例子

创建基本凭据

CONN scott输入密码： password BEGIN  - 基本凭证。DBMS_CREDENTIAL.CREATE_CREDENTIAL（credential_name =>'TIM_HALL_CREDENTIAL'，username =>'tim_hall'，password =>'password'）;结束

创建Windows凭据

CONN scott输入密码： password -  Credential包括Windows域BEGIN DBMS_CREDENTIAL.CREATE_CREDENTIAL（credential_name =>'TIM_HALL_WIN_CREDENTIAL'，用户名=>'tim_hall'，密码=>'密码'，windows_domain =>'localdomain'）;结束

显示有关凭据的信息

使用[DBA|ALL|USER] _CREDENTIALS视图显示有关凭据的信息。

38.4.2 DISABLE_CREDENTIAL程序

此过程禁用Oracle凭据。

句法

DBMS_CREDENTIAL.DISABLE_CREDENTIAL（credential_name IN VARCHAR2，强制输入BOOLEAN DEFAULT FALSE）;

参数

表38-3 DISABLE_CREDENTIAL过程参数

参数 描述

credential_name

凭证的名称。它可以选择以模式为前缀。这不能设置为NULL 。它被转换为大写，除非用双引号括起来。

力

如果为FALSE ，则不会禁用凭据，前提是它不依赖于任何现有的调度程序作业或PL / SQL库。如果观察到依赖关系，则返回错误。如果为TRUE ，则无论是否有任何调度程序作业或PL / SQL库引用它，都将禁用该凭据。

使用说明

  • 凭据驻留在特定模式中，任何具有CREATE CREDENTIAL或CREATE ANY CREDENTIAL系统特权的用户都可以禁用凭据。要禁用除您自己之外的架构中的凭据，您必须具有CREATE ANY CREDENTIAL权限。

  • 可以将OS用户的凭证视为作为特定用户的操作系统的入口点。允许管理员（或凭据所有者）作为外部作业，数据库作业，文件传输，外部过程和文件监视的特定用户，快速，轻松，可逆地禁止从数据库到操作系统的所有登录。要启用现有的已禁用凭据，您需要使用ENABLE_CREDENTIAL过程 。

  • 如果凭据的属性（Windows域，用户名，密码，其启用/禁用位）发生更改，则库可能会变为无效。

38.4.3 DROP_CREDENTIAL过程

此过程将删除Oracle凭据。

句法

DBMS_CREDENTIAL.DROP_CREDENTIAL（credential_name IN VARCHAR2，强制IN BOOLEAN DEFAULT FALSE）;

参数

表38-4 DROP_CREDENTIAL过程参数

参数 描述

credential_name

凭证的名称。它可以选择以模式为前缀。这不能设置为NULL 。

力

如果设置为FALSE ，则任何EXTPROC别名库都不能引用凭证，否则会引发错误。如果设置为TRUE ，则无论是否存在引用它的extproc别名库，都会删除凭据。引用已删除凭据的EXTPROC别名库将变为无效。

使用说明

只有凭证的所有者或具有CREATE ANY CREDENTIAL系统特权的用户才可以删除凭证。

例子

EXEC DBMS_CREDENTIAL.DROP_CREDENTIAL（'TIM_HALL_CREDENTIAL'，FALSE）; EXEC DBMS_CREDENTIAL.DROP_CREDENTIAL（'TIM_HALL_WIN_CREDENTIAL'，FALSE）;

38.4.4 ENABLE_CREDENTIAL程序

此过程启用Oracle凭据。

句法

DBMS_CREDENTIAL.ENABLE_CREDENTIAL（credential_name IN VARCHAR2）;

参数

表38-5 ENABLE_CREDENTIAL过程参数

参数 描述

credential_name

凭证的名称。它可以选择以模式为前缀。这不能设置为NULL 。它被转换为大写，除非用双引号括起来。

使用说明

  • 凭据驻留在特定模式中，任何具有CREATE CREDENTIAL OR CREATE ANY CREDENTIAL系统特权的用户都可以禁用凭据。要禁用除您自己以外的模式中的凭据，您必须具有CREATE CREDENTIAL OR CREATE ANY CREDENTIAL权限。

  • 可以将OS用户的凭证视为作为特定用户的操作系统的入口点。允许禁用凭证将允许管理员（或凭证所有者）以特定用户（外部作业，文件传输，外部过程，文件监视）快速，轻松且可逆地禁止从数据库到OS的所有登录。要禁用现有凭据，您需要使用DISABLE_CREDENTIAL过程 。

  • 如果凭据的属性（Windows域，用户名，密码，其启用/禁用位）发生更改，则库可能会变为无效。

38.4.5 UPDATE_CREDENTIAL过程

此过程更新现有Oracle凭据。

句法

DBMS_CREDENTIAL.UPDATE_CREDENTIAL（credential_name IN VARCHAR2，属性IN VARCHAR2，值IN VARCHAR2）;

参数

表38-6 UPDATE_CREDENTIAL过程参数

参数 描述

credential_name

凭证的名称。它可以选择以模式为前缀。这不能设置为NULL 。它被转换为大写，除非用双引号括起来。

属性

要更新的属性的名称： USERNAME ， PASSWORD ， WINDOWS_DOMAIN ， DATABASE_ROLE或COMMENTS

值

所选属性的新值

使用说明

  • 凭据驻留在特定模式中，并且可由具有CREATE CREDENTIAL或CREATE ANY CREDENTIAL系统特权的任何用户CREATE 。要在除您自己的模式之外的模式中创建凭证，您必须具有CREATE ANY CREDENTIAL权限。

  • 用户名区分大小写。它不能包含双引号或空格。

  • 引用更新的凭据的EXTPROC别名库将变为无效。如果更改凭证的属性（Windows域，用户名，密码，其启用/禁用位），则库将变为无效。

例子

更新基本凭据

CONN scott输入密码： password BEGIN  - 基本凭证。DBMS_CREDENTIAL.UPDATE_CREDENTIAL（credential_name =>'TIM_HALL_CREDENTIAL'，attribute =>'password'，value =>'password2'）; DBMS_CREDENTIAL.UPDATE_CREDENTIAL（credential_name =>'TIM_HALL_CREDENTIAL'，attribute =>'username'，value =>'tim_hall'）;结束;

更新Windows凭据

CONN scott输入密码： password -  Credential包括Windows域BEGIN DBMS_CREDENTIAL.UPDATE_CREDENTIAL（credential_name =>'TIM_HALL_WIN_CREDENTIAL'，用户名=>'tim_hall'，密码=>'密码'，windows_domain =>'localdomain'）;结束
