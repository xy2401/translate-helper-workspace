目的

PREDICTION_COST返回选择中每行的成本。成本是指最低成本类或指定class 。成本以BINARY_DOUBLE形式返回。

PREDICTION_COST可以执行分类或异常检测。对于分类，返回的成本是指预测的目标类别。对于异常检测，返回的成本是指1 （对于典型行）或0 （对于异常行）的分类。

您可以将PREDICTION_COST与PREDICTION函数结合使用，以获得预测和预测成本。

cost_matrix_clause

成本是最小化最有害的错误分类的偏向因素。例如，误报可能被认为比假阴性更昂贵。成本在成本矩阵中指定，该矩阵可与模型关联或在VALUES子句中内联定义。所有分类算法都可以使用成本来影响评分。

决策树是唯一可以使用成本来影响模型构建的算法。用于构建决策树模型的成本矩阵也是模型的默认评分成本矩阵。

以下成本矩阵表指定1的错误分类比错误分类0成本高五倍。 

ACTUAL_TARGET_VALUE PREDICTED_TARGET_VALUE COST ------------------- ---------------------- ------ ---- 0 0 0 0 1 1 1 0 5 1 1 0

在cost_matrix_clause ：

  • COST MODEL表明应该通过考虑与模型相关联的评分成本矩阵来执行评分。如果成本矩阵不存在，则该函数返回错误。

  • COST MODEL AUTO表示存在成本矩阵未知。如果存在成本矩阵，则该函数使用它来返回最低成本预测。否则，该函数返回最高概率预测。

  • VALUES子句为class_value指定内联成本矩阵。例如，您可以指定1的错误分类比错误分类0成本高5倍，如下所示：

语法选择

PREDICTION_COST可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，也可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态挖掘数据。选择语法或分析语法 ：

  • 语法 - 使用第一种语法使用预定义的模型对数据进行评分。提供执行分类或异常检测的模型的名称。

  • 分析语法 - 使用分析语法在没有预定义模型的情况下对数据进行评分。分析语法使用mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见“ analytic_clause :: = ” 。）

    • 对于分类，请指定FOR expr ，其中expr是一个表达式，用于标识具有字符数据类型的目标列。

    • 对于异常检测，请指定关键字OF ANOMALY 。

在对分区模型进行评分时， PREDICTION_COST函数的语法可以使用可选的GROUPING提示。请参阅GROUPING提示 。

mining_attribute_clause

mining_attribute_clause标识要用作评分预测变量的列属性。使用分析语法调用函数时，这些预测变量也用于构建瞬态模型。mining_attribute_clause行为与PREDICTION函数的描述相同。（参见“ mining_attribute_clause :: = ” 。）
