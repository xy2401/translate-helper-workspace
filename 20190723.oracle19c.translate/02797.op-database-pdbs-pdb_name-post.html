克隆或拔出PDB的操作

岗位

/数据库/ PDBS / {pdb_name} /

克隆或拔掉PDB。这些操作对指定的可插拔数据库有重大影响。拔出操作将删除PDB。请注意，在克隆可插拔数据库时，源数据库的状态将在操作期间暂时设置为OPEN READ ONLY并重置为其原始状态。仅在数据库和Oracle REST数据服务位于同一主机上的拓扑中才支持使用Oracle透明数据加密。此服务需要在池配置中设置db.cdb.adminUser凭据。

  • clonePDBName（可选）：string

  • 方法：字符串
    允许值： [ "CLONE", "UNPLUG" ]

  • pdb_name（可选）：字符串
    在克隆的情况下使用源PDB，或者在拔出插头的情况下拔出PDB。此属性是可选的，URI中定义的名称优先。

    与克隆操作相关。对于临时文件重用是真的。

  • tdeKeystorePath（可选）：string
    如果tdeExport标志设置为true，则需要TDE密钥库路径。仅可用于拔出操作。
  • tdePassword（可选）：字符串
    仅用于拔出操作的TDE密码。当tdeExport标志设置为true时，这是必需的。
  • tdeSecret（可选）：字符串
    如果tdeExport标志设置为true，则需要TDE密钥。仅可用于拔出操作。
  • tempSize（可选）：字符串
    与克隆操作相关。临时表空间的大小。请参阅“数据库SQL语言参考”文档中的size_clause说明。
  • totalSize（可选）：字符串
    与克隆操作相关。表空间的大小。请参阅“数据库SQL语言参考”文档中的size_clause说明。
  • unlimitedStorage（可选）：布尔值
    与克隆操作相关。适用于无限存储。即使设置为true，如果方法是CLONE，也必须在请求中指定totalSize和tempSize。
