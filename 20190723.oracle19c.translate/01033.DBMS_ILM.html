83.1 DBMS_ILM概述

要实施ILM策略，可以使用Oracle数据库中的热图来跟踪数据访问和修改。您还可以使用自动数据优化（ADO）在数据库中的不同存储层之间自动压缩和移动数据。DBMS_ILM包支持立即评估或执行ADO相关任务。Ť

他的软件包支持以下两种调度ADO操作的方法。

  • 数据库用户计划在一组对象上立即执行ADO策略。

  • 数据库用户查看对一组对象的ADO策略的评估结果。然后，用户向该集添加或删除对象，并再次查看ADO策略评估的结果。用户重复此步骤以确定ADO执行的对象集。然后，用户可以调度ADO操作以立即在这组对象上执行。

以下过程支持两种使用模式。在描述这些过程之前，我们将ADO任务的概念介绍为有助于跟踪ADO策略的特定评估或（评估和执行）的实体。特定的ADO任务可能处于以下状态之一。

  • 待用

  • 活性

  • 已完成

83.5.1 ADD_TO_ILM程序

此过程将通过参数指定的对象添加到特定ADO任务，并评估此对象上的ADO策略。

该过程只能在处于非活动状态的ADO任务上执行。可以使用适当的视图查看对此对象的ADO策略评估结果，具体取决于角色和访问权限（ USER_ILMTASKS或DBA_ILMTASKS ， USER_ILMEVALUATIONDETAILS或DBA_ILMEVALUATIONDETAILS ， USER_ILMRESULTS或DBA_ILMRESULTS ）。

句法

DBMS_ILM.ADD_TO_ILM（task_id IN NUMBER，所有者IN VARCHAR2，object_name IN VARCHAR2，subobject_name IN VARCHAR2 DEFAULT NULL）;

参数

83.5.3 EXECUTE_ILM过程

此过程执行ADO任务。

此过程有两个重载。第一个重载为一组对象执行ADO任务，而不先前对它们进行评估。第二个重载执行特定对象的ADO策略。

句法

DBMS_ILM.EXECUTE_ILM（task_id OUT NUMBER，ilm_scope IN NUMBER DEFAULT SCOPE_SCHEMA，execution_mode IN NUMBER DEFAULT ILM_EXECUTION_ONLINE）; DBMS_ILM.EXECUTE_ILM（所有者IN VARCHAR2，object_name IN VARCHAR2，task_id OUT NUMBER，subobject_name IN VARCHAR2 DEFAULT NULL，policy_name IN VARCHAR2 DEFAULT ILM_ALL_POLICIES，execution_mode IN NUMBER DEFAULT ILM_EXECUTION_ONLINE）;

参数

表83-6 EXECUTE_ILM过程参数

参数 描述

TASK_ID

标识特定的ADO任务

ilm_scope

确定为ADO执行考虑的对象集。默认设置是仅考虑架构中的对象。

execution_mode

是否在线执行ADO任务（ ILM_EXECUTION_ONLINE ）或脱机（ ILM_EXECUTION_OFFLINE ）

所有者

对象的所有者

OBJECT_NAME

对象的名称

subobject_name

子对象的名称（分区表的分区名称）

POLICY_NAME

要在对象上评估的ADO策略的名称。如果应评估对象上的所有ADO策略，则应使用包常量ILM_ALL_POLICIES 。

使用说明

  • 想要更多控制何时执行ADO以及谁不想等到下一个维护窗口的用户可以使用EXECUTE_ILM过程。

  • 该过程像DDL一样执行，因为它在ADO任务之前和之后自动提交并创建相关作业。

83.5.5 PREVIEW_ILM程序

此过程评估使用ILM_SCOPE参数指定的对象上的ADO策略。

它返回一个数字作为task_id ，用于标识特定的ADO任务。这可用于根据角色和访问权限在适当的视图中查看策略评估的结果（ USER_ILMTASKS或DBA_ILMTASKS ， USER_ILMEVALUATIONDETAILS或DBA_ILMEVALUATIONDETAILS ， USER_ILMRESULTS或DBA_ILMRESULTS ）。

PREVIEW_ILM过程使ADO任务处于非活动状态。预览结果后，可以向此任务添加或删除对象。

句法

DBMS_ILM.PREVIEW_ILM（task_id OUT NUMBER，ilm_scope IN NUMBER DEFAULT SCOPE_SCHEMA）;

参数

表83-8 PREVIEW_ILM过程参数

参数 描述

TASK_ID

标识特定的ADO任务

ilm_scope

标识执行范围。应该是常量中描述的SCOPE_DATABASE或SCOPE_SCHEMA
