1简介

作为Oracle数据库管理员（DBA），您负责Oracle数据库的性能。调整数据库以达到理想的性能水平可能是一项艰巨的任务，尤其是对于Oracle数据库新手的DBA。“Oracle数据库2天+性能调优指南”是一个快速入门指南，介绍如何使用Oracle诊断包，Oracle调优包和Oracle企业管理器云控制（云控制）提供的功能执行日常数据库性能调整任务。

本章包含以下部分：

  • 关于本指南

  • 常见的Oracle DBA任务

  • 用于调整数据库的工具

  • 访问数据库主页

1.1关于本指南

在使用本指南之前，您必须执行以下操作：

  • 熟悉Oracle数据库管理。有关更多信息，请参见“ Oracle数据库管理员指南” 。

  • 获取“ 调整数据库的工具”中描述的必要产品和工具。

Oracle Database 2 Day +性能调优指南是面向任务的。目标是描述需要执行调整任务的原因和时间。

本指南不是对所有Oracle数据库概念的详尽讨论。有关此类信息，请参阅Oracle数据库概念 。

本指南未介绍Oracle数据库的基本管理任务。有关此类信息，请参见“ Oracle数据库管理员指南”。

本指南中使用的主要接口是企业管理器云控制控制台。本指南不是对所有Oracle数据库性能调优功能的详尽讨论。它不包括可用的应用程序编程接口（API），它们提供与本指南中介绍的相似的调整选项。有关此类信息，请参见“ Oracle数据库性能调整指南”和“ Oracle数据库SQL调整指南” 。

1.2常见的Oracle DBA任务

作为Oracle DBA，您可以参与以下任务：

  • 安装Oracle软件

  • 创建Oracle数据库

  • 将数据库软件升级到新版本

  • 启动和关闭数据库

  • 管理数据库的存储结构

  • 管理用户帐户和安全性

  • 管理模式对象，例如表，索引和视图

  • 必要时进行数据库备份并执行数据库恢复

  • 主动监控数据库的状况，并根据需要采取预防或纠正措施

  • 监视和调整数据库性能

本指南介绍了如何完成前面列表中的最后两个任务。

1.3调整数据库的工具

本指南旨在让您快速有效地调整和优化Oracle数据库的性能。

要实现本指南的目标，您必须获得以下产品，工具，功能和实用程序：

  • Oracle Database 19c，版本19.1（企业版）

    Oracle数据库在群集和单服务器配置上提供企业级性能，可伸缩性和可靠性。它包含本指南中使用的许多性能功能。

  • Oracle企业管理器云控制

    管理数据库的主要工具是企业管理器云控制（云控制），这是一个基于Web的界面。安装Oracle软件，创建或升级数据库以及配置网络后，可以使用Cloud Control管理数据库。此外，Cloud Control还为性能顾问和数据库实用程序（如SQL * Loader和Recovery Manager（RMAN））提供了一个界面。

  • Oracle诊断包

    Oracle Diagnostics Pack提供了一个完整，经济高效且易于使用的解决方案，通过提供独特的功能（如自动识别性能瓶颈，指导性问题解决和全面的系统监控）来管理Oracle数据库环境的性能。本指南中使用的Oracle Diagnostics Pack的主要功能包括自动工作负载存储库（AWR），自动数据库诊断监视器（ADDM）和活动会话历史记录（ASH）。

  • Oracle Tuning Pack

    Oracle Tuning Pack可自动执行数据库应用程序调整过程，从而显着降低数据库管理成本，同时提高性能和可靠性。本指南中使用的Oracle Tuning Pack的主要功能包括：

    • SQL Tuning Advisor

      此功能使您能够以特定建议或建议的形式提交一个或多个SQL语句作为输入和接收输出，以及如何调整语句，以及每个建议的基本原理及其预期收益。建议涉及对象统计信息的收集，新索引的创建，SQL语句的重构或SQL配置文件的创建。

    • SQL Access Advisor

      此功能使您可以通过推荐适当的物化视图集并查看给定SQL工作负载的日志，索引和分区来优化SQL查询的数据访问路径。

  • Oracle Real Application Testing

    Oracle Real Application Testing包含以下主要功能：

    • 数据库重播

      此功能使您能够捕获生产系统上的数据库工作负载，并在测试系统上以与Oracle数据库相同或更高版本上的生产系统完全相同的时序和并发性重放它。

    • SQL性能分析器

      此功能使您可以通过识别已退化，改进或保持不变的SQL语句来评估系统更改对SQL性能的影响。

    也可以看看：

    “Oracle数据库测试指南”，以获取有关如何使用数据库重放和SQL性能分析器功能的信息

注意：

上述列表中的某些产品和工具（包括Oracle Diagnostics Pack和Oracle Tuning Pack）需要单独的许可证。有关更多信息，请参阅Oracle数据库许可信息 。

1.4访问数据库主页

数据库主页是Oracle企业管理器云控制（云控制）中的主数据库管理页面。登录到Cloud Control后，导航到要在Cloud Control中管理的目标数据库的Database Home页面。

要在Cloud Control中访问数据库主页：

  1. 启动云控制。

    用于访问Cloud Control的URL具有以下语法：

    http:// hostname.domain:portnumber /em

  2. 在“欢迎”页面中，输入您的Cloud Control用户名和密码，然后单击“ 登录” 。
  3. 从“ 目标”菜单中，选择“ 数据库” 。

    将显示“数据库”页面。

  4. 在“数据库”页面中，选择“ 搜索列表”以显示可用目标数据库的列表。
  5. 在“名称”列中，选择要观察或修改的目标数据库。

    将显示目标数据库的主页。第一次从某些菜单中选择选项时，例如“性能”菜单，将显示“数据库登录”页面。

  6. 在目标数据库的登录页面中，以具有相应权限的用户身份登录。例如，以具有SYSDBA权限的用户SYS身份登录：

    • 用户名：输入SYS 。

    • 密码：输入SYS用户的密码。

    • 连接为：从“角色”列表中，选择“ SYSDBA” 。
