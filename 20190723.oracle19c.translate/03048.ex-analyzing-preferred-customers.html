27.3示例：分析首选客户

本节中的示例显示有关使用关联卡或可能使用关联卡的客户的信息。

示例27-2查找有关首选客户的人口统计信息

此查询返回典型亲和卡持有者的性别，年龄和居住时间。异常检测模型SVMO_SH_Clas_sample ，对于典型情况返回1 ，对于异常返回0 。人口统计数据仅针对典型客户进行预测;异常值不包括在样本中。 

SELECT cust_gender，round（avg（age））age，round（avg（yrs_residence））yrs_residence，count（*）cnt FROM mining_data_one_class_v WHERE 预测（SVMO_SH_Clas_sample使用*）= 1 GROUP BY cust_gender ORDER BY cust_gender; CUST_GENDER AGE YRS_RESIDENCE CNT ------------ ---------- ------------- ---------- F 40 4 36 M 45 5 304

示例27-3动态识别与首选客户相似的客户

此查询可识别目前没有亲和卡，但共享亲和卡持有者的许多特征的客户。PREDICTION和PREDICTION_PROBABILITY函数使用OVER子句而不是预定义的模型来对客户进行分类。预测和概率是动态计算的。 

SELECT cust_id，pred_prob FROM（SELECT cust_id，affinity_card，PREDICTION（FOR TO_CHAR（affinity_card）USING *）OVER（）pred_card，  PRATINGIC_PROBABILITY（FOR TO_CHAR（affinity_card），1 USING *）OVER（）pred_prob FROM mining_data_build_v）WHERE affinity_card = 0 AND pred_card = 1 ORDER BY pred_prob DESC; CUST_ID PRED_PROB ---------- --------- 102434 .96 102365 .96 102330 .96 101733 .95 102615 .94 102686 .94 102749 .93。。。。选择102580 .52 102269 .52 102533 .51 101604 .51 101656 .51 226行。

示例27-4预测新客户成为首选客户的可能性

此查询计算首次成为首选客户（亲和卡持有者）的概率。该查询可以在销售点实时执行。

这位新客户是一位44岁的美国高管，拥有学士学位，每年收入超过30万美元。他已婚，住在一个3口之家，过去6年一直住在同一个住所。该客户成为典型的亲和卡持有者的概率仅为5.8％。

SELECT PREDICTION_PROBABILITY（SVMO_SH_Clas_sample，1使用44岁AS，6岁yrs_residence，'巴赫。'AS教育，'结婚'AS cust_marital_status，'执行。'AS职业'，美利坚合众国'AS country_name，'M'AS cust_gender，'L：300,000及以上'AS cust_income_level，'3'AS houshold_size）prob_typical FROM DUAL; PROB_TYPICAL ------------ 5.8
