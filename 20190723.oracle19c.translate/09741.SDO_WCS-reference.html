36 SDO_WCS包（网络覆盖服务）

MDSYS.SDO_WCS包中包含与Web Coverage Service（WCS）的Oracle Spatial和Graph支持相关联的子程序。

它假定您熟悉Web Coverage Service（WCS）支持中描述的概念和技术。

36.1 SDO_WCS.CreateTempTable

格式

SDO_WCS.CreateTempTable（schema_name IN VARCHAR2 DEFAULT USER，tablespace_name IN VARCHAR2 DEFAULT NULL）;

描述

在涉及重新投影或转换时，在GetCoverage处理期间创建必要的临时表。

参数

架构名称

将在其中创建临时表的模式的名称。如果未指定，则在调用用户架构时创建表。

tablespace_name

临时表的表空间的名称。如果未指定，则使用用户的默认表空间。

使用说明

每个临时GeoRaster对象都存储在与原始GeoRaster对象相同的模式中。将GetCoverage操作（WCS）响应发送到客户端后，将删除在处理请求期间创建的任何临时GeoRaster。

必须为其GeoRaster对象将作为coverage发布的所有模式调用此过程。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例在调用程序的架构中创建WCS临时表。

CALL SDO_WCS.createTempTable（）;

以下示例在SCOTT模式中创建WCS临时表。调用用户必须具有在SCOTT模式中创建表的权限。

CALL SDO_WCS.createTempTable（'SCOTT'）;

36.2 SDO_WCS.DropTempTable

格式

SDO_WCS.DropTempTable（schema_name IN VARCHAR2 DEFAULT USER）;

描述

从架构中删除WCS临时表。（WCS GeoCoverage处理需要WCS临时表。）

参数

schema_name

包含要删除的WCS临时表的数据库模式的名称。如果未指定，则使用调用程序的模式。

使用说明

如果schema_name不是调用用户，则调用用户必须具有从该模式中删除表的权限。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例从调用程序的模式中删除WCS临时表。

BEGIN SDO_WCS.DropTempTable（）;结束;

以下示例从SCOTT模式中删除WCS临时表。调用用户必须具有删除SCOTT模式中的表的权限。

BEGIN SDO_WCS.DropTempTable（'SCOTT'）;结束;

36.3 SDO_WCS.GrantPrivilegesToWCS

格式

SDO_WCS.GrantPrivilegesToWCS（table_name IN VARCHAR2，wcs_schema IN VARCHAR2，可更新IN VARCHAR2 DEFAULT'FALSE'）;

描述

当WCS架构与用户架构（即具有GeoRaster对象的架构）不同时，授予对所有GeoRaster对象和GeoRaster表的栅格数据表的选择和更新权限到WCS架构。

参数

table_name

包含将作为coverage发布的GeoRaster对象的表的名称。

wcs_schema

要用作WCS架构的架构的名称。

更新

字符串TRUE或FALSE （默认值），指示是否授予更新权限。

使用说明

如果用户架构（具有GeoRaster对象的架构作为coverage发布）和WCS架构相同，请不要使用此过程。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例将所有GeoRaster对象的读取权限和当前架构中名为GEORASTER_TABLE的表的栅格数据表授予名为WCS_1的WCS架构。

BEGIN SDO_WCS.grantPrivilegesToWCS（'GEORASTER_TABLE'，'WCS_1'）;结束; /

父主题： SDO_WCS包（Web Coverage服务）

36.4 SDO_WCS.Init

格式

SDO_WCS.Init（）;

描述

创建元数据表和序列，以存储对作为WCS coverage发布的GeoRaster对象的引用。调用模式成为WCS模式，并在部署Spatial Web Services Web应用程序的Java EE容器中进行配置。

参数

（没有。）

使用说明

在发布任何coverage之前以及在WebLogic Server中配置WCS数据源之前，必须调用此过程一次。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例创建必需的WCS元数据表。然后，它在SCOTT模式的GEORASTER_TABLE.RASTER列中发布GeoRaster对象的coverage。

BEGIN SDO_WCS.Init（）; SDO_WCS.PublishCoverage（ 'SCOTT'， 'GEORASTER_TABLE'， '光栅'）;结束; /

36.5 SDO_WCS.PublishCoverage

格式

SDO_WCS.PublishCoverage（georaster IN SDO_GEORASTER，可更新IN VARCHAR2 DEFAULT'FALSE'）;

要么

SDO_WCS.PublishCoverage（user_name IN VARCHAR2，table_name IN VARCHAR2，column_name IN VARCHAR2，updateable IN VARCHAR2 DEFAULT'FALSE'）;

描述

将GeoRaster对象发布为WCS coverage。

参数

georaster

要作为WCS coverage发布的GeoRaster对象。

user_name

拥有包含GeoRaster列的表的用户（架构）的名称。

table_name

要发布其GeoRaster对象的GeoRaster表的名称。

column_name

table_name GeoRaster列的名称。

更新

如果要通过事务请求编辑已发布的coverage，则包含字符串TRUE ;如果发布的coverage不能通过事务请求进行编辑，则包含字符串FALSE （默认值）。

使用说明

为发布的每个GeoRaster分配唯一的coverage ID。GetCapabilities操作（WCS）响应将显示新生成的覆盖ID。 GeoRaster对象只能发布一次。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例从SCOTT模式中名为GEORASTER_TABLE的表中发布特定的GeoRaster对象（ where ID=1 ）作为coverage。 

DECLARE gr1 SDO_GEORASTER; BEGIN SELECT栅格INTO gr1 FROM SCOTT.GEORASTER_TABLE，其中ID = 1; SDO_WCS.PublishCoverage（GR1）;结束; /

以下示例在SCOTT模式中的IMAGE表的RASTER列中发布所有 GeoRaster对象。 

CALL SDO_WCS.publishCoverage（'SCOTT'，'IMAGE'，'RASTER'）;

36.6 SDO_WCS.RevokePrivilegesFromWCS

格式

SDO_WCS.RevokePrivilegesFromWCS（table_name IN VARCHAR2，wcs_schema IN VARCHAR2）;

描述

从指定的wcs_schema用户撤消对调用用户架构中指定表的选择和更新权限。

参数

table_name

以前使用SDO_WCS.GrantPrivilegesToWCS过程授予权限的GeoRaster表的名称。

wcs_schema

从中撤消权限的WCS架构的名称。不得与包含GeoRaster对象的用户架构相同。

使用说明

当WCS架构与GeoRaster对象的架构相同时，请勿使用此过程。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例从名为WCS_1的WCS模式中撤消对GEORASTER_TABLE及其与之关联的所有栅格数据表的读取和更新权限。

BEGIN SDO_WCS.RevokePrivilegesFromWCS（'GEORASTER_TABLE'，'WCS_1''）;结束; /

36.7 SDO_WCS.UnpublishCoverage

格式

SDO_WCS.UnpublishCoverage（coverage_id IN VARCHAR2）;

描述

取消发布保险。

参数

coverage_id

一个字符串，用于指定GetCapabilities响应的coverage列表中的coverage ID。

使用说明

GetCapabilities操作（WCS）响应包含来自给定WCS服务器实例的所有coverage ID的列表。运行此过程后，指定的coverage ID将不会出现在GetCapabilities响应中。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例取消发布覆盖范围为Id C0001的coverage。

DECLARE BEGIN SDO_WCS.UnpublishCoverage（'C0001'）;结束; /

36.8 SDO_WCS.ValidateCoverages

格式

SDO_WCS.ValidateCoverages（）RETURN MDSYS.SDO_WCS_INVALID_COVERAGE;

描述

验证WCS元数据表中的GeoRaster对象，并返回所有无效WCS元数据条目的coverage ID。

参数

（没有。）

使用说明

必须由WCS用户调用此过程。

删除GeoRaster对象时，对它们的引用可能会保留在WCS元数据表中。此类引用是无效的元数据条目。

此函数返回MDSYS.SDO_WCS_INVALID_COVERAGE对象，该对象定义为： TABLE OF VARCHAR2(4000)

返回对象的每个元素都是WCS元数据表中的coverage Id，其中未找到相应的GeoRaster对象。DescribeCoverage操作（WCS）和GetCoverage操作（WCS）不处理无效的元数据条目。

有关WCS支持的信息，请参阅Web Coverage Service（WCS）支持 。

例子

以下示例迭代所有无效的coverage ID并取消发布它们，只留下WCS元数据中的有效coverage。

对于i IN的DECLARE BEGIN（select * from table（SDO_WCS.ValidateCoverages（）））LOOP SDO_WCS.UnpublishCoverage（i.column_value）;结束循环;结束; /
