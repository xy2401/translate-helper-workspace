Oracle Net Services配置文件和参数

如果使用Easy Connect之外的命名方法，则可能需要额外配置Oracle Net Services。

Oracle数据库服务器和客户端的网络元素已针对大多数环境进行了预配置。默认情况下启用Easy Connect命名方法，不需要存储库。

有关Oracle RAC数据库的Oracle Net Services配置文件和参数的信息，请查看这些主题：

  • 数据库服务注册的数据库初始化参数
    Oracle Database 19c数据库服务自动向LOCAL_LISTENER和REMOTE_LISTENER参数中指定的侦听器注册。
  • 网络服务名称和tnsnames.ora文件
    安装过程在每个节点上创建一个tnsnames.ora文件。此文件充当网络服务名称的存储库。每个网络服务名称都与连接标识符相关联。连接标识符是将用户定义的名称映射到连接描述符的标识符。
  • DBCA创建的网络服务名称
    DBCA为连接创建网络服务名称。
  • 监听器配置和listener.ora文件
    在Oracle RAC环境中，Oracle建议您让Oracle代理管理Oracle数据库的Oracle侦听器。
  • 网络服务配置文件（sqlnet.ora）
    安装数据库软件后，Oracle Universal Installer启动Oracle Net Configuration Assistant（NETCA）。NETCA创建Oracle Net Services概要文件或sqlnet.ora文件。
