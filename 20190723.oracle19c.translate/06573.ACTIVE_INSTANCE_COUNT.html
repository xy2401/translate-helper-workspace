1.7 ACTIVE_INSTANCE_COUNT

ACTIVE_INSTANCE_COUNT使您可以将双实例集群中的一个实例指定为主实例，将另一个实例指定为辅助实例。此参数在具有两个以上实例的群集中没有任何功能。

属性 描述

参数类型

整数

默认值

没有默认值。

可修改

没有

可在PDB中修改

没有

价值范围

1或> =群集中的实例数。（1以外的值对任何实例的活动或备用状态都没有影响。）

基本

没有

Oracle RAC

您必须为每个实例设置此参数，并且多个实例必须具有相同的值。

注意：

不推荐使用ACTIVE_INSTANCE_COUNT参数。它仅为了向后兼容而保留。

将此参数设置为1时，启动的第一个实例将成为主实例并接受客户端连接。第二个实例作为辅助实例启动，只有在第一个实例失败时才能接受客户端连接。在这种情况下，辅助实例成为主要实例。

当故障实例再次启动时，它将作为辅助实例启动，并且除非当前主实例失败，否则不会接受客户端连接。

注意：

此参数仅在只有两个实例的集群中起作用。
