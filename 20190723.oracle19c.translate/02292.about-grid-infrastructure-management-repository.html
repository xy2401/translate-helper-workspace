关于Grid Infrastructure Management Repository

每个Oracle域服务集群都包含一个Grid Infrastructure Management Repository（GIMR），但GIMR配置对于Oracle Standalone Cluster是可选的。

网格基础结构管理存储库（GIMR）或管理数据库（MGMTDB）是一个多租户数据库，具有用于每个集群的GIMR的可插拔数据库（PDB）。GIMR存储有关群集的以下信息：

  • Cluster Health Monitor收集的实时性能数据

  • Cluster Health Advisor收集的故障，诊断和指标数据

  • 有关Oracle Clusterware收集的所有资源的群集范围事件

  • 服务质量管理（QoS）收集的工作负载性能和CPU架构数据

  • Oracle Fleet补丁和配置所需的元数据

从Oracle Grid Infrastructure 19c开始，为Oracle Standalone Cluster部署配置GIMR是可选的。Oracle独立群集在Oracle ASM磁盘组或共享文件系统上本地托管GIMR;这个GIMR是一个带有单个可插拔数据库（PDB）的多租户数据库。

全局GIMR在Oracle域服务集群中运行。Oracle Domain Services Cluster在单独的Oracle ASM磁盘组中本地托管GIMR。Oracle成员集群数据库使用位于Oracle域服务集群上的远程GIMR。在远程群集上托管GIMR可减少在群集上运行额外基础架构存储库的开销。Oracle域服务集群的GIMR是一个多租户数据库，其中包含一个PDB，并为添加的每个成员集群添加了额外的PDB。

配置Oracle域服务群集时，安装程序会提示为GIMR配置单独的Oracle ASM磁盘组，默认名称为MGMT 。
