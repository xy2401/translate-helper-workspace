14使用ASMCA管理Oracle ACFS和Oracle ADVM

Oracle ASM Configuration Assistant（ASMCA）提供用于管理Oracle自动存储管理群集文件系统（Oracle ACFS）和Oracle ASM动态卷管理器（Oracle ADVM）的实用程序。

Oracle ASM Configuration Assistant（ASMCA）支持安装和配置Oracle ASM实例，Oracle ASM磁盘组，Oracle ASM动态卷管理器（Oracle ADVM）卷和Oracle自动存储管理群集文件系统（Oracle ACFS）文件系统。此外，您还可以使用ASMCA命令行界面。

本章讨论以下主题：

  • 用于管理Oracle ACFS和Oracle ADVM的ASMCA GUI工具

  • 用于管理Oracle ACFS和Oracle ADVM的ASMCA命令行界面

有关使用ASMCA管理Oracle ASM实例和磁盘组的信息，请参阅使用ASMCA管理Oracle ASM 。

注意：

某些ASMCA功能和菜单选项仅在特定环境中可用，例如Oracle RAC配置。有关特定ASMCA页面的详细信息，请访问“ 帮助”按钮提供的在线帮助。

注意：

Oracle ASM Configuration Assistant使您可以创建或配置Oracle ADVM卷。

右键单击Oracle ASM Configuration Assistant屏幕中的选定卷以显示配置选项菜单。

菜单包括以下选项：

  • 启用禁用的卷并禁用启用的卷

  • 查看状态详情

  • 调整尚未在Oracle ACFS文件系统上装入的卷

  • 删除卷

可以选择创建卷，启用所有卷以及禁用所有卷。

单击“ 创建”以显示用于创建卷的对话框。您必须为从磁盘组列表中选择的现有Oracle ASM磁盘组提供唯一的卷名。您还必须指定卷的大小。您可以选择显示用于创建卷的高级选项，以便您指定冗余级别和条带化特征。

必须将兼容性参数COMPATIBLE.ASM和COMPATIBLE.ADVM设置为11.2或更高，以使磁盘组包含Oracle ADVM卷。要使用Oracle ACFS加密，复制，安全性或标记，为文件系统创建卷的磁盘组必须具有ASM和ADVM兼容属性，设置为11.2.0.2或更高版本。

在AIX上创建Oracle ADVM卷之前，请确保已创建必要的用户授权。

使用ASMCA管理Oracle ACFS文件系统

Oracle ASM Configuration Assistant使您可以创建或配置Oracle ACFS文件系统。

要配置现有文件系统，请右键单击Oracle ASM Configuration Assistant屏幕中的选定文件系统以显示配置选项菜单。

菜单包括以下选项：

  • 查看状态详情

  • 装载和卸载文件系统

  • 调整文件系统的大小

  • 删除文件系统

  • 显示或执行安全命令

  • 创建或删除快照

某些命令需要root权限，例如挂载文件系统。ASMCA生成命令，您可以以root身份手动运行。

在Windows上创建Oracle ACFS文件系统时，请将ASMCA作为Windows域用户运行，该用户同时也是计算机的管理员。

“创建”，“全部安装”，“全部卸载”以及“安全和加密”命令都有按钮。

Oracle ASM Configuration Assistant使您可以创建Oracle ACFS文件系统。

选择Create ，然后从列表中选择要创建的Oracle ACFS文件系统的类型和现有的Oracle ADVM卷。指定安装点字段;挂载点必须是现有目录。必须安装文件系统才能使其可用。您可以选择启用“ 自动 挂载”选项。您还可以在“ 描述”字段中提供文件系统的描述 。

必须在操作系统提示符下以root用户或Windows Administrator手动运行mount命令。

您可以选择为Oracle数据库创建Oracle ACFS文件系统。

也可以看看：

使用ASMCA管理Oracle ACFS的安全性和加密

Oracle ASM Configuration Assistant使您可以为Oracle ACFS文件系统配置安全性和加密。

您必须初始化安全系统，作为配置Oracle ACFS文件系统安全性的第一步。您还必须初始化加密系统，作为加密Oracle ACFS文件系统的第一步。您不必在同一文件系统上同时使用Oracle ACFS安全性和加密。如果您决定同时使用安全性和加密，则必须先初始化并设置加密，然后才能在安全领域启用加密。

在显示的对话框中，输入信息以指定安全管理员和安全管理员的操作系统组。您可以选择创建受密码保护的钱包。完成对话框中的输入字段后，单击“ 显示 命令”以显示必须在操作系统提示符下以root用户或管理员用户身份运行的命令。例如：

创建用于数据库使用的Oracle ACFS文件系统

要创建数据库使用Oracle ACFS文件系统，你可以选择从磁盘组中配置选项菜单数据库 使用 ，以显示该对话框输入框的创建 ACFS。

注意：

为旨在存储数据库文件的Oracle ACFS文件系统创建Oracle ADVM卷时，请使用ASMCMD volcreate命令或SQL ALTER DISKGROUP ADD VOLUME SQL语句而不是ASMCA工具来确保将列条带设置为1。

在此对话框中，您必须输入：

  • 卷名

    这是您要创建的Oracle ADVM卷的名称。

  • 登山点

    这是要安装数据库主目录的文件系统的安装点。包含数据库主目录的文件系统不应位于Oracle Grid Infrastructure基础（ ORACLE_BASE for grid）目录下。

  • 大小（千兆字节）

    默认值为7 GB，建议的最小大小。

  • 所有者姓名

    这是安装数据库并在数据库主目录中拥有该软件的用户的操作系统名称。

  • 业主组

    这是数据库主目录的操作系统组。

安装点必须是现有目录。必须安装文件系统才能使其可用。

选择“ 自动运行配置命令”以自动运行ASMCA配置命令。要使用此选项，您必须在ASMCA设置页面上提供root凭据。

mount命令也可以在操作系统提示符下以root身份或Windows Administrator手动运行。

也可以看看：

  • volcreate有关信息volcreate命令

  • 管理磁盘组中的Oracle ADVM卷以获取有关ALTER DISKGROUP ADD VOLUME语句的信息

  • 有关Oracle ADVM卷的信息，请参见Oracle ASM Dynamic Volume Manager概述

  • 关于Oracle ACFS和Oracle数据库主目录 ，了解有关挂载点和数据库主目录的信息

  • mount以获取有关在Linux上安装Oracle ACFS文件系统的信息

创建Oracle ACFS快照

—createACFSSnapshot创建Oracle自动存储管理群集文件系统（Oracle ACFS）快照。

句法

asmca -silent -createACFSSnapshot（-acfsMountPoint mount_path ）（ - 快照名称snapshot_name ）[ - 快照模式{r | w}] [-parentSnapshotName parent_snapshot_name ]

表14-1包含—createACFSSnapshot命令可用的选项。

表14-1 -createACFSSnapshot命令的选项

选项 描述

-acfsMountPoint mount_path

指定要用作装入点的现有路径。这在Oracle RAC环境中是必需的。

-snapshotName anapshot_name

指定Oracle ACFS快照的名称。

-snapshotMode r|w

指定Oracle ACFS快照的模式，即只读（ r ）或读写（ w ）。

-parentSnapshotName parent_snapshot_name

指定Oracle ACFS父快照的名称。

创建一个卷

-createVolume创建Oracle ADVM卷。

句法

asmca -silent -createVolume {-volumeName volume_name -volumeDiskGroup diskgroup -volumeSizeGB size_GB [-volumeRedundancy {INHERIT | HIGH |正常| EXTERNAL}] ......} [-sysAsmPassword sysasm_password ]

表14-3包含 - createVolume命令可用的选项。

表14-3 -createVolume命令的选项

选项 描述

-volumeName volume_name

指定要创建的卷的名称。

-volumeDiskGroup diskgroup

指定要在其中创建卷的磁盘组的名称。

-volumeSizeGB size_GB

以千兆字节为单位指定卷的大小。

-volumeRedundancy { INHERIT | HIGH | NORMAL | EXTERNAL }

指定卷的冗余设置。

有关多个命令通用的其他选项，请参阅表9-1 。

例14-1使用asmca -silent -createVolume

创建Oracle ACFS文件系统

- createACFS创建Oracle自动存储管理群集文件系统（Oracle ACFS）。

此命令不会装入Oracle ACFS文件系统。有关装入Oracle ACFS文件系统的信息，请参阅Linux环境的“ mount ”和Windows环境的“ acfsmountvol ” 。

句法

asmca -silent -createACFS {-acfsVolumeDevice volume_device} {-acfsMountPoint mount_path} [-acfsUserName acfs_username] [-acfsUserGroup acfs_usergroup] [-sysAsmPassword sysasm_password]

表14-4包含 - createACFS命令可用的选项。

表14-4 -createACFS命令的选项

选项 描述

-acfsVolumeDevice volume_device

指定Oracle ADVM卷设备的名称。

-acfsMountPoint mount_path

指定要用作装入点的现有路径。这在Oracle RAC环境中是必需的。

-acfsUserName acfs_username

指定Oracle ACFS用户名。

-acfsUserGroup acfs_usergroup

指定Oracle ACFS组名称。

有关多个命令通用的其他选项，请参阅表9-1 。
