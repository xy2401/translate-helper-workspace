DGM-16952：DGMGRL无法确定配置详细信息

原因：找不到代理配置。

操作： 1）无法联系Data Guard代理进程（DMON）。验证DMON进程是否正在运行。2）目前没有代理配置。你必须创建一个。3）您已连接到已禁用或已删除的代理配置中的备用数据库。当备用数据库处于此状态时，备用数据库上不提供代理配置的详细信息。连接到主数据库并重新发出该命令。4）您已连接到代理配置中必须恢复或重新实例化的数据库。处于此状态时，代理配置的详细信息不可用。恢复或重新安排数据库并重新发出该命令，或连接到主数据库并重新发出该命令。

DGM-16953：XML文档太长

原因： XML文档的长度超出了内部限制。

行动：这是一个内部错误。联系Oracle支持服务。

DGM-16959：代理服务器进程没有响应

操作： Data Guard代理进程（DMON）不可用。验证DMON进程是否正在运行并重试该命令。

DGM-16960：状态不可用

DGM-16979：无法登录主数据库或备用数据库

原因：观察者用于登录主数据库和快速启动故障转移目标备用数据库的用户名或密码没有有效的SYSDBA或SYSDG凭据。

操作：确保主数据库和快速启动故障转移目标数据库正在使用远程登录密码文件。确保两个数据库上的SYSDBA或SYSDG密码相同，并且CONNECT命令中使用了SYSDBA或SYSDG密码。再次启动观察者。

DGM-16991：两个或多个资源的名称为“ string ”

DGM-17002：无法删除主数据库

DGM-17152：文件操作选项中指定的值“ string ”无效

DGM-17175：无法将会话容器设置为string 。

原因： DGMGRL CLI无法将会话容器设置为错误消息文本中标识的可插入数据库。

操作：检查可插拔数据库是否存在，或查找可能已记录在数据库警报日志中的错误，并在重试命令之前更正这些错误。

DGM-17176：无法查询PDB_PLUG_IN_VIOLATIONS。

原因： DGMGRL CLI无法查询PDB_PLUG_IN_VIOLATIONS表。

操作：检查容器是否设置为CDB $ ROOT并且容器已打开。还要检查数据库警报日志中的任何错误并在重试命令之前更正它们。

DGM-17290：检测到角色更改。此数据库可能不再是主数据库。

原因：在此配置中对备用数据库执行了角色更改。

操作：检查是否已对此配置中的备用数据库执行了角色更改。如果已执行角色更改，则需要从新主数据库的副本恢复或重新创建此数据库。如果尚未执行角色更改并且您要打开此数据库，请启用配置，然后打开数据库。

DGM-17295：参数“ string ”未更新。

原因：指定的参数对应于Oracle Database 19c之前的成员上由Data Guard代理管理的属性。

操作：使用带有SET PROPERTY子句的EDIT命令。

DGM-17297：此命令需要与数据库建立遗留连接。

原因：命令失败，因为未向数据库创建遗留连接。需要遗留连接，因为必须重新启动数据库实例。

操作：使用遗留连接连接到数据库，然后重试该命令。

DGM-17298：未指定DIRECTORY-LOCATION参数。

原因：命令失败，因为未指定DIRECTORY-LOCATION参数且数据库未建立本地归档目标。

操作：为DIRECTORY-LOCATION参数指定一个值，然后重试该命令。

DGM-17299：没有可用的本地档案目的地。

原因：命令失败，因为没有可用的本地存档目标。本地存档需要编号为10或更低的LOG_ARCHIVE_DEST_n初始化参数。

操作：如果可能，清除编号为10或更低的LOG_ARCHIVE_DEST_n初始化参数，然后重试该命令。

DGM-17300：未指定SIZE参数。

原因：该命令失败，因为未设置DB_RECOVERY_FILE_DEST_SIZE初始化参数时未指定SIZE参数。

操作：指定SIZE参数的值并重试该命令。

DGM-17301：服务器参数（SPFILE）未用于RAC数据库。

原因：该命令失败，因为数据库是RAC数据库并且未配置服务器参数文件。

操作：为数据库创建服务器参数文件，重新启动数据库的所有实例以使用服务器参数文件，然后重试该命令。

DGM-17328：必须指定DIRECTORY_LOCATION参数才能启用闪回数据库。

原因：该命令失败，因为如果未设置DB_RECOVERY_FILE_DEST初始化参数，则无法启用闪回数据库。

操作：指定DIRECTORY_LOCATION参数并重试该命令。
