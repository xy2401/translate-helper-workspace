Oracle ASM升级通知

了解Oracle ASM升级选项和限制。

  • 您可以通过执行单个节点或群集中的一组节点的滚动升级来升级Oracle自动存储管理（Oracle ASM）11g第2版（11.2）及更高版本，而无需关闭Oracle RAC数据库。但是，如果在使用Oracle ASM的群集上有独立数据库，则必须在升级之前关闭独立数据库。

  • 在Oracle Grid Infrastructure 11g第2版（11.2）中更改了Oracle ASM主目录的位置，以便Oracle ASM与Oracle Grid Infrastructure主目录（Grid home）中的Oracle Clusterware一起安装。

  • 不同版本的两个节点无法在群集中运行。从Oracle Grid Infrastructure 11g第2版（11.2）或Oracle Grid Infrastructure 12c第1版（12.1）升级到更高版本时，如果在滚动升级期间出现中断，则在重新启动升级时，请确保先启动发布Oracle Grid Infrastructure并使Oracle ASM集群恢复滚动迁移模式。
