2.12在REST上使用Oracle ORAchk或Oracle EXAchk

Oracle ORAchk和Oracle EXAchk包括完整的REST支持，允许通过HTTPS进行调用和查询。

  • 启用REST
    为了促进REST支持，Oracle REST数据服务（ORDS）包含在安装中。
  • start_client

  • start_client
    使用POST请求使用特定参数运行正常运行状况检查。
  • 轮廓
    使用GET请求为指定的配置文件运行运行状况检查运行。
  • 校验
    使用GET请求为指定的检查ID运行运行状况检查运行。
  • 状态
    使用GET请求报告指定作业ID的状态。
  • 下载
    使用GET请求下载指定作业ID的收集结果。
  • checktfaupload

  • checktfafaileduploads
    如果任何Oracle跟踪文件分析器服务上载失败，请使用GET请求报告。
  • gettfaupload

2.12.1启用REST

为了促进REST支持，Oracle REST数据服务（ORDS）包含在安装中。

  1. 要启用REST，请设置ORDS，然后在出现提示时提供用户密码。
    ./orachk -ordssetup ./exachk -ordssetup
  2. 使用-ords选项启动守护程序：
    ./orachk -d start -ords ./exachk -d start -ords

    注意：

    只有root用户才能使用ORDS支持启动守护程序。

    由于ORDS支持需要守护程序，因此ORDS仅在具有守护程序支持的平台（Linux）上可用。

2.12.9 checktfafaileduploads

2.12.16删除ORDS设置

  1. 要完全删除ORDS设置：
    ./orachk -ordsrmsetup ./exachk -ordsrmsetup

    如果守护程序正在运行，则命令选项-ordsrmsetup停止守护程序，然后停止ORDS服务。
