  • 审计表升级和存档要求

审计表升级和存档要求

对于使用Oracle Label Security和Oracle Database Vault的12.1之前的Oracle数据库版本，必须在升级之前运行OLS预处理脚本。

如果要从使用Oracle Label Security（OLS）和Oracle Database Vault的Oracle Database 12.1之前的数据库升级，则必须首先运行OLS预处理脚本olspreupgrade.sql来处理aud$ table内容。OLS升级将aud$表从SYSTEM模式移动到SYS模式。olspreupgrade.sql脚本是此移动所需的预处理脚本。

警告：

升级前运行olspreupgrade.sql脚本对于升级使用Oracle Label Security和Oracle Database Vault的Oracle Database 12.1之前的数据库是必需的。升级到Oracle Database 12.1版之后，您无需执行OLS预处理过程即可修补或升级数据库。

olspreupgrade.sql脚本在SYS模式中创建临时表PREUPG_AUD$ ，并将SYSTEM.aud$记录移动到SYS.PREUPG_AUD$ 。作为安全措施，Oracle建议您在运行olspreupgrade.sql脚本之前归档审计跟踪。如果数据库上安装了Oracle Label Security，并且您要从早期版本升级，则必须在升级之前运行OLS预处理脚本。

Oracle Database Vault和Oracle Database 11.2版的升级

如果在要升级的早期版本中安装了Oracle Label Security，则将DV_PATCH_ADMIN角色授予SYS。

如果Oracle Database Vault未随版本11.2数据库一起安装，则可以跳过本节中的步骤2,3,6和7。

要在升级之前在版本11.2数据库上运行OLS预处理脚本，请执行以下操作：

  1. 将以下脚本脚本从新安装的Oracle主目录复制到y0u要升级的数据库的Oracle主目录：

    • ORACLE_HOME /rdbms/admin/olspreupgrade.sql

    • ORACLE_HOME /rdbms/admin/emremove.sql

    • ORACLE_HOME /rdbms/admin/catnoamd.sql

  2. 启动SQL * Plus并以DVOWNER身份连接到要升级的数据库。

  3. 运行以下语句：

    SQL> GRANT DV_PATCH_ADMIN到SYS;

  4. 在系统提示符下，将SYS连接为SYSDBA：

    将SYS作为SYSDBA连接

  5. 运行Data Vault的预处理脚本

    ORACLE_HOME /rdbms/admin/olspreupgrade.sql ORACLE_HOME /rdbms/admin/emremove.sql ORACLE_HOME /rdbms/admin/catnoamd.sql

    在预处理脚本运行时，您可以继续在数据库上运行应用程序。

  6. 在olspreupgrade.sql成功完成运行后，启动SQL * Plus并以DVOWNER连接到数据库。

  7. 运行以下SQL语句：

    SQL> REVOKE来自SYS的DV_PATCH_ADMIN;

将DV_PATCH_ADMIN角色授予SYS for Oracle Database Vault

如果启用了Oracle Database Vault，则要对Oracle Data Vault执行检查，升级过程需要运行三个SQL脚本。

SYS用户需要以下脚本的DV_PATCH_ADMIN角色：

OLS_SYS_MOVE运行olspreupgrade.sql ：

CHECK.OLS_SYS_MOVE.MIN_VERSION_INCLUSIVE = 10.2 CHECK.OLS_SYS_MOVE.MAX_VERSION_EXCLUSIVE = 12.1

EM_PRESENT运行emremove.sql ：

CHECK.EM_PRESENT.MIN_VERSION_INCLUSIVE =无CHECK.EM_PRESENT.MAX_VERSION_EXCLUSIVE = 12.1.0.1

AMD_EXISTS CHECK运行catnoamd.sql ：

CHECK.AMD_EXISTS.MIN_VERSION_INCLUSIVE = NONE CHECK.AMD_EXISTS.MAX_VERSION_EXCLUSIVE = NONE
