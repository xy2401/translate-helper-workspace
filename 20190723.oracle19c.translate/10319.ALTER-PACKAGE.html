改变包装

目的

包使用PL / SQL定义。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用ALTER PACKAGE语句显式重新编译包规范，正文或两者。显式重新编译消除了对隐式运行时重新编译的需要，并防止了相关的运行时编译错误和性能开销。

由于包中的所有对象都存储为一个单元，因此ALTER PACKAGE语句将所有包对象重新编译在一起。您不能使用ALTER PROCEDURE语句或ALTER FUNCTION语句单独重新编译作为程序包一部分的过程或函数。

注意：

此声明不会更改现有包的声明或定义。要重新声明或重新定义包，请使用带有OR REPLACE子句的CREATE PACKAGE或CREATE PACKAGE BODY语句。

先决条件

要修改程序包，程序包必须位于您自己的架构中，或者您必须具有ALTER ANY PROCEDURE系统特权。

句法

alter_package :: =

语义

模式

指定包含该包的模式。如果省略schema ，则Oracle数据库假定程序包在您自己的模式中。

包裹名字

指定要重新编译的包的名称。

package_compile_clause

有关此子句的语法和语义以及有关创建和编译包的完整信息，请参阅Oracle数据库PL / SQL语言参考 。

版面| NONEDITIONABLE

使用这些子句指定包是否成为editioned或noneditioned对象，如果editioning为架构对象类型后启用PACKAGE的schema 。默认为EDITIONABLE 。有关更改已编辑和未编辑的对象的信息，请参阅“ Oracle数据库开发指南” 。
