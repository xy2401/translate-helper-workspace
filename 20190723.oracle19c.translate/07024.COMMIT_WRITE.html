1.46 COMMIT_WRITE

COMMIT_WRITE是一个高级参数，用于控制如何将重做事务提交写入重做日志。

属性 描述

参数类型

串

句法

COMMIT_WRITE ='{IMMEDIATE | BATCH}，{WAIT | NOWAIT}'

默认值

如果未显式指定此参数，则数据库提交行为默认为在将控制权返回给客户端之前将提交记录写入磁盘。

如果仅指定了IMMEDIATE或BATCH ，但未指定WAIT或NOWAIT ，则假定为WAIT模式。

如果仅指定了WAIT或NOWAIT ，但未指定IMMEDIATE或BATCH ，则假定为IMMEDIATE模式。

请注意， NOWAIT选项可能会导致在数据库收到提交消息后但在写入重做日志记录之前发生的故障。这可能错误地向事务指示其更改是持久的。此外，如果数据库意外关闭，它可能会违反ACID（原子性，一致性，隔离性，持久性）事务的持久性。

可修改

是（在会话级和系统级）。在ALTER SYSTEM或ALTER SESSION语句中为COMMIT_WRITE提供的值必须用逗号分隔。

可在PDB中修改

是

价值范围

单引号，逗号分隔的IMMEDIATE或BATCH列表，以及WAIT或NOWAIT 。

基本

没有

Oracle RAC

每个实例都可以有自己的设置

IMMEDIATE和BATCH选项控制Log Writer如何批量重做。WAIT和NOWAIT选项控制何时将提交的重做刷新到重做日志。

注意：

不推荐使用COMMIT_WRITE参数。它仅为了向后兼容而保留。它由COMMIT_LOGGING和COMMIT_WAIT参数替换。
