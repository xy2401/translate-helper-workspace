4协议地址配置

了解如何为Oracle数据库实例和客户端配置连接。

网络对象由协议地址标识。建立连接时，请求的客户端和接收者（侦听器或Oracle连接管理器）配置有相同的协议地址。客户端使用该地址将连接请求发送到特定网络对象位置，并且接收者“侦听”该地址上的请求，并基于其与客户端信息匹配的地址信息授予连接。

  • 协议地址
    协议地址由ADDRESS和ADDRESS_LIST元素组成。
  • 协议参数
    侦听器和Oracle Connection Manager由协议地址标识。
  • 推荐的端口号
    Oracle建议您使用客户端和Oracle Connection Manager连接的默认端口号。
  • 端口号限制
    了解端口号的限制。如果需要配置侦听器以使用系统端口号（在1到1024范围内），请使用此处描述的过程配置侦听器。

4.1协议地址

协议地址由ADDRESS和ADDRESS_LIST元素组成。

  • 地址
    ADDRESS网络参数指定ADDRESS_LIST或DESCRIPTION参数下的协议地址。
  • ADDRESS_LIST
    ADDRESS_LIST网络参数指定共享公共特征的协议地址的数量。

父主题： 协议地址配置

4.1.1地址

ADDRESS网络参数指定ADDRESS_LIST或DESCRIPTION参数下的协议地址。

目的

定义协议地址。

使用说明

将此参数放在ADDRESS_LIST或DESCRIPTION参数下。DESCRIPTION用于tnsnames.ora或listener.ora文件。

例

（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521））

4.1.2 ADDRESS_LIST

ADDRESS_LIST网络参数指定共享公共特征的协议地址的数量。

目的

定义共享共同特征的协议地址列表。

使用说明

指定多个地址时，此参数不是必需的。

例

（ADDRESS_LIST =（LOAD_BALANCE = on）（ADDRESS =（PROTOCOL = tcp）（HOST = sales-server）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp）（HOST = hr-server）（PORT = 1521）） ）

4.4端口号限制

了解端口号的限制。如果需要配置侦听器以使用系统端口号（在1到1024范围内），请使用此处描述的过程配置侦听器。

Oracle允许端口号从1到65535。但是，通常会保留低于1024的端口号（ 众所周知的端口或系统端口 ）。通常，只有特权进程才能侦听1024以下端口上的TCP连接。

如果需要将侦听器配置为侦听小于1024的端口号，请完成以下过程：

注意：

这个程序是一个指导原则。您的操作系统可能需要不同的过程。

  1. 使用Oracle Net Configuration Assistant或Oracle Net Manager为侦听器配置协议地址和其他配置参数。

  2. 以具有侦听器的计算机上的root用户身份登录。

  3. 为侦听器可执行文件（ tnslsnr ）和从属共享库设置文件所有权和访问权限，以便只能由root用户修改这些文件。

  4. 从root目录开始，确保在这些文件的路径名中找到的各个目录的权限共享相同的所有权和访问权限。

  5. 以root用户身份启动侦听器。

  6. 在系统提示符下输入以下命令：

    tnslsnr listener_name -user user -group group

    在上面的命令中，使用以下选项：

    表4-3 tnslsnr实用程序选项

    选项 描述

    LISTENER_NAME

    指定要配置的侦听器的名称。如果省略，则使用默认名称LISTENER 。

    用户

    指定在不需要超级用户（ root ）权限时希望侦听器使用其权限的用户。执行特权操作后，侦听器会不可逆转地放弃root权限。

    组

    指定在不需要超级用户（ root ）组权限时希望侦听器使用其权限的组。执行特权操作后，侦听器会不可逆转地放弃root组权限。

    在此步骤中，侦听器从root更改为您指定的用户和组权限。除了监听配置的端点所需的系统调用之外，所有操作都使用指定的用户和组权限完成。侦听器将恢复为root用户以侦听保留的地址，例如小于1024的TCP端口。

    在侦听器开始侦听listener.ora文件中配置的所有端点之后，它会不可逆地切换到指定的用户和组。在那时候。监听器放弃它最初拥有的root权限。-user和-group命令行参数仅接受以数字形式指定的用户和组标识符。

    例如，要运行具有root权限的名为mylsnr的侦听器，并使用具有37555的用户标识符（UID）的Oracle用户的权限，以及具有组标识符（GID）16的OSDBA组dba成员资格，在操作系统提示符处输入以下命令：

    tnslsnr mylsnr -user 37555 -group 16

  7. 侦听器启动后，您可以使用侦听器控件实用程序对其进行管理。

    警告：

    • Oracle建议运行侦听器进程的用户是oracle用户，或具有侦听器进程通常在操作系统上运行的特权的类似特权用户。

    • 不要让侦听器进程以root用户身份运行。以超级用户身份运行进程是一个安全漏洞。
