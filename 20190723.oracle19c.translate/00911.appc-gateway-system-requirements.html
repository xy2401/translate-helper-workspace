3系统要求

以下主题描述了网关的系统要求。

  • 硬件要求

  • 软件要求

硬件要求

以下各节介绍了此版本的Microsoft Windows网关的硬件要求。

处理器要求

有关为您的系统运行网关的认证硬件平台和操作系统版本要求的最新列表，请参阅适用于Microsoft Windows的Oracle数据库安装指南和My Oracle Support上的认证表。可以在以下位置找到My Oracle Support Web站点：

https://support.oracle.com

用于APPC的Oracle数据库网关需要基于Pentium处理器的Intel或100％兼容的个人计算机，该处理器可以运行Microsoft Windows 2000及更高版本。

内存要求

对于大多数安装，Oracle建议为运行Oracle Database Gateway for APPC的系统至少提供256 MB的实内存。

每次并发使用网关至少需要10 MB。

以下因素会影响网关的虚拟内存要求服务器进程：

  • 每个用户打开的并发网关连接数;

  • 在网关和RTP之间传输的数据项的数量;

  • 其他因素，如配置的网络缓冲区大小;

  • 网关安装期间包含的Oracle Net协议适配器。

网络附件要求

网关需要任何网络附件支持适用于您的平台的SNA Server或用于TCP / IP通信的TCP / IP网络工具。

但是，如果您仅使用新的TCP / IP支持IMS Connect功能，则不需要SNA软件包。Windows操作系统安装了TCP / IP，您需要对其进行配置。

磁盘空间要求

网关磁盘空间要求至少为700 MB。

软件要求

Oracle支持这些要求中描述的系统软件配置，前提是底层系统软件产品由其各自的供应商提供支持。与系统软件供应商核实最新支持状态。

操作系统要求

有关为Windows英特尔系统运行网关的最新认证操作系统版本要求列表，请参阅适用于Microsoft Windows的Oracle数据库安装指南和My Oracle Support上的认证表。可以在以下位置找到My Oracle Support Web站点：

https://support.oracle.com

通信协议要求

需要Microsoft Host Integration Server 2000，或Windows的IBM Communication Server V6.1.1（或更高版本）。在本文档中，这些通信服务器中的任何一个通常称为SNA Server。

注意：

如果您选择使用IMS Connect作为通信协议的TCP / IP支持，则不需要使用SNA Server。您的操作系统自带安装的TCP / IP协议。如果您选择使用TCP / IP协议，则需要将其配置为与网关一起正常工作，然后执行此操作的说明位于使用TCP / IP通信协议的网关配置中 。

Oracle数据库要求

Microsoft Windows平台要求 Oracle数据库是最新的，支持Oracle数据库版本的最新补丁集。

Oracle网络产品要求

Oracle Net自动安装在系统所在的位置安装Oracle数据库并安装在安装网关的系统上。有关配置信息，请参阅本指南中的配置Oracle网络 。此外，您可以参考Oracle数据库网络服务管理员指南 。

IBM大型机要求

除了网关和您的特定平台的其他软件要求之外，以下列表还概述了IBM大型机上所需的其他要求：

OLTP for SNA

OLTP必须支持映射的APPC对话。如果通过网关执行的OLTP事务处理程序进行数据库更新，那么APPC动词CONFIRM ， CONFIRMED ，并SEND_ERR必须由OLTP支持。这些动词实现了APPC SYNCLEVEL 1 。

由OLTP控制的所有可由通过网关调用的事务程序更新的资源必须定义为OLTP和主机系统的可恢复资源（如果COMMIT / 这些资源需要ROLLBACK功能。例如，由a更新的VSAM文件必须将CICS事务定义为CICS，作为COMMIT / ROLLBACK的可恢复文件来控制更新。

该网关与支持SNA的产品的所有受支持版本兼容，例如CICS，IMS / TM和 z / OS上。

重要提示：有关已知限制列表，请参阅“ 已知的限制 ”一节已知的限制与安装网关的继续之前。

OLTP for TCP / IP

IMS / TM：需要7.1或更高版本，以及“IBM IMS Connect指南和参考”中列出的任何APAR（修补程序）。

I MS Connect：需要1.2或更高版本。
