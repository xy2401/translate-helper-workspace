1.105调度员

DISPATCHERS在共享服务器体系结构中配置调度程序进程。

属性 描述

参数类型

串

句法

DISPATCHERS = ' dispatch_clause '

句法

dispatch_clause :: =

(PROTOCOL = protocol ) |

(ADDRESS = address ) |

(DESCRIPTION = description )

[options_clause]

句法

options_clause :: =

(DISPATCHERS = integer |

SESSIONS = integer |

CONNECTIONS = integer |

MULTIPLEX = {1 | ON |是的| TRUE |

0 |关闭|没有| FALSE |两个| IN | OUT} |

LISTENER = tnsname |

SERVICE = service |

INDEX = integer )

默认值

如果SHARED_SERVERS大于0 ，则DISPATCHERS默认为'(PROTOCOL=tcp)' ，从而导致创建1个TCP / IP调度程序。

可修改

改变系统

可在PDB中修改

没有

基本

没有

解析软件支持名称 - 值语法，以与位置无关，不区分大小写的方式启用属性规范。例如：

DISPATCHERS ='（PROTOCOL = TCP）（DISPATCHERS = 3）'

可以使用完整属性名称或以前3个字符开头的任何子字符串来指定属性。例如， SESSIONS可以指定为SES ， SESS ， SESSI等。

仅指定以下属性之一： PROTOCOL ， ADDRESS或DESCRIPTION 。如果指定ADDRESS或DESCRIPTION ，则可以指定其他网络属性。这样做支持多宿主主机。

dispatch_clause

  • 协议

    调度程序为其生成侦听端点的网络协议。

  • 地址

    调度程序侦听的端点的网络协议地址。

  • 描述

    调度程序侦听的端点的网络描述，包括协议地址。

options_clause

  • DISPATCHERS

    最初的调度员数量。默认值为1。

  • SESSIONS

    每个调度程序允许的最大网络会话数。默认值是特定于操作系统的。大多数操作系统的默认值为16 KB。

  • CONNECTIONS

    允许每个调度程序的最大网络连接数。默认值是特定于操作系统的。

  • MULTIPLEX

    启用Oracle Connection Manager会话多路复用功能。

    • 值1 ， ON ， YES ， TRUE和BOTH表示为传入和传出网络连接启用了网络会话复用。

    • 值IN表示为传入网络连接启用了网络会话多路复用。

    • 值OUT表示为传出网络连接启用了网络会话多路复用。

    • 值0 ， NO ， OFF和FALSE表示对传入和传出网络连接都禁用了网络会话多路复用。这是默认值。

  • LISTENER

    指定调度程序将注册的Oracle Net侦听器的地址或地址列表的网络名称。

    LISTENER属性有助于管理多宿主主机。此属性指定调度程序将注册的相应侦听器。LISTENER属性优先于LOCAL_LISTENER和REMOTE_LISTENER参数。请参阅“ LOCAL_LISTENER ”和“ REMOTE_LISTENER ” 。

  • 服务

    指定客户端可以通过其连接到调度程序的一个或多个名称。SERVICE属性优先于SERVICE_NAMES参数。

  • 指数

    在ALTER SYSTEM SET DISPATCHERS语句中使用此属性可指示要修改的调度程序配置。（如果在初始化参数文件中指定INDEX ，则Oracle数据库会忽略它。）在ALTER SYSTEM语句中， INDEX指定参数值初始化的顺序。值的范围从0 （对于第一个调度程序配置）到一个小于您定义的调度程序配置总数的值。

    例如，如果在初始化参数文件中指定3个调度程序配置，则可以通过在ALTER SYSTEM语句中指定INDEX = 2来修改第三个调度程序配置。您还可以通过指定INDEX = 3在ALTER SYSTEM语句中添加另一个调度程序配置。

    如果未在ALTER SYSTEM语句中指定INDEX ，则必须指定PROTOCOL ， ADDRESS或DESCRIPTION属性，并且如果存在与此PROTOCOL ， ADDRESS或DESCRIPTION匹配的调度程序配置，则将修改该配置。否则，将添加新配置。
