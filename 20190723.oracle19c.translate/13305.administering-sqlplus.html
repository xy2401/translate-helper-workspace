4.1.1使用安装文件

当您启动SQL * Plus时，它会运行glogin.sql站点配置文件安装文件，然后运行login.sql用户配置文件安装文件。

使用站点配置文件

全局站点配置文件是$ORACLE_HOME/sqlplus/admin/glogin.sql 。如果此位置已存在站点配置文件，则在安装SQL * Plus时会覆盖该配置文件。如果删除了SQL * Plus，则还会删除站点配置文件。

使用用户配置文件

用户配置文件是login.sql 。SQL * Plus在当前目录中查找此文件，然后在SQLPATH环境变量指定的目录中查找。此环境变量的值是以冒号分隔的目录列表。SQL * Plus按照SQLPATH环境变量中列出的顺序在这些目录中搜索login.sql文件。

login.sql文件中设置的选项会覆盖glogin.sql文件中设置的选项。

Oracle数据库提供了PRODUCT_USER_PROFILE表，您可以使用该表来禁用指定的SQL和SQL * Plus命令。选择安装预配置数据库的安装类型时，将自动创建此表。

要重新创建PRODUCT_USER_PROFILE表，请在SYSTEM模式中运行$ORACLE_HOME/sqlplus/admin/pupbld.sql脚本。例如，运行以下命令，其中SYSTEM_PASSWORD是SYSTEM用户的密码：

您还可以使用$ORACLE_HOME/bin/pupbld shell脚本在SYSTEM模式中手动重新创建PRODUCT_USER_PROFILE表。此脚本提示输入SYSTEM密码。要运行pupbld无交互脚本，设置SYSTEM_PASS环境变量系统用户名和密码。

4.1.4.1安装SQL * Plus命令行帮助

有三种方法可以安装SQL * Plus命令行帮助：

  • 完成安装预配置数据库的安装。

    在安装过程中安装预配置的数据库时，SQL * Plus会自动在SYSTEM模式中安装SQL * Plus命令行帮助。

  • 使用$ORACLE_HOME/bin/helpins shell脚本在SYSTEM模式中手动安装命令行帮助。

    helpins脚本会提示输入SYSTEM密码。要在不进行交互的情况下运行此脚本，请将SYSTEM_PASS环境变量设置为SYSTEM用户名和密码。例如：

    • Bourne，Bash或Korn shell：

      $ SYSTEM_PASS = SYSTEM / system_password ;导出SYSTEM_PASS

    • C shell：

      ％setenv SYSTEM_PASS SYSTEM / system_password

  • 使用$ORACLE_HOME/sqlplus/admin/help/helpbld.sql脚本在SYSTEM模式中手动安装命令行帮助。

    例如，运行以下命令，其中system_password是SYSTEM用户的密码：

    $ sqlplus SQL> CONNECT SYSTEM输入密码： system_password SQL> @？/sqlplus/admin/help/helpbld.sql？/ sqlplus / admin / help helpus.sql

    注意：

    在创建新表之前， helpins shell脚本和helpbld.sql脚本都会删除现有的命令行帮助表。

4.1.4.2删除SQL * Plus命令行帮助

要从SYSTEM模式手动删除SQL * Plus命令行帮助表，请运行$ORACLE_HOME/sqlplus/admin/help/helpdrop.sql脚本。为此，请运行以下命令，其中system_password是SYSTEM用户的密码：

$ sqlplus SQL> CONNECT SYSTEM输入密码： system_password SQL> @？/sqlplus/admin/help/helpdrop.sql

4.2.1使用SQL * Plus中的系统编辑器

如果在SQL * Plus提示符下运行ED或EDIT命令，则系统将启动操作系统编辑器，例如ed ， emacs ， ned或vi 。然而 PATH环境变量必须包含编辑器可执行文件所在的目录。

HISTORY命令使用户能够从当前会话的历史列表中运行，编辑或删除以前使用的SQL * Plus，SQL或PL / SQL命令。您可以启用或禁用当前SQL * Plus会话中发出的命令的历史记录。

启动编辑器时，当前的SQL缓冲区将放置在编辑器中。退出编辑器时，更改的SQL缓冲区将返回到SQL * Plus。

您可以通过定义SQL * Plus _EDITOR变量来指定应该从哪个编辑器开始。您可以在glogin.sql站点配置文件或login.sql用户配置文件中定义此变量。或者，您可以在SQL * Plus会话期间定义它。例如，要将默认编辑器设置为vi ，请运行以下命令：

SQL> DEFINE _EDITOR = vi

如果未设置_EDITOR变量，则使用EDITOR或VISUAL环境变量的值。如果设置了两个环境变量，则使用EDITOR变量的值。如果未指定_EDITOR ， EDITOR和VISUAL ，则默认编辑器为vi 。定义的编辑器用于HISTORY命令的编辑选项。请考虑以下SQL语句：

SQL> hist 1 select * from dual; 2 desc dual

要在vi编辑器中打开上述SQL语句中的第一个条目，请使用以下命令：

SQL> hist 1编辑

当您启动编辑器时，SQL * Plus使用afiedt.buf临时文件将文本传递给编辑器。您可以使用SET EDITFILE命令指定其他文件名。例如：

SQL> SET EDITFILE /tmp/myfile.sql

SQL * Plus不会删除临时文件。

4.2.2从SQL * Plus运行操作系统命令

使用HOST命令或感叹号（！）作为SQL * Plus提示符后面的第一个字符导致后续字符传递给子shell。SHELL环境变量设置用于运行操作系统命令的shell。默认shell是Bourne shell。如果无法运行shell，则SQL * Plus会显示错误消息。

要返回SQL * Plus，请运行exit命令或按Ctrl + D.

例如，要运行单个命令，请使用以下命令语法：

SQL>！命令

在此示例中， command表示要运行的操作系统命令。

要从SQL * Plus运行多个操作系统命令，请运行HOST或！命令。按Enter返回操作系统提示符。

4.2.3中断SQL * Plus

运行SQL * Plus时，可以通过按Ctrl + C来停止滚动记录显示并终止SQL语句。

4.2.4使用SPOOL命令

SPOOL命令生成的文件的缺省文件扩展名为.lst 。要更改此扩展名，请指定包含句点（。）的假脱机文件。例如：

SQL> SPOOL query.txt

4.3 SQL * Plus限制

本节介绍以下SQL * Plus限制：

  • 调整Windows大小

  • 退货代码

  • 隐藏密码

4.3.1调整Windows大小

SQL * Plus LINESIZE和PAGESIZE系统变量的默认值不会自动调整窗口大小。如果更改窗口大小，则必须设置LINESIZE和PAGESIZE系统变量。

4.3.2返回代码

操作系统返回码仅使用一个字节，这不足以返回Oracle错误代码。返回码的范围是0到255。

4.3.3隐藏密码

如果在命令行上传递密码或将SYSTEM_PASS环境变量设置为SYSTEM用户的用户名和密码，则ps命令的输出可能会显示此信息。要防止未经授权的访问，请仅在SQL * Plus提示时输入SYSTEM密码。

要自动运行脚本，请考虑使用不需要存储密码的身份验证方法。例如，对Oracle数据库进行外部身份验证登录。如果您的安全性较低，则必须考虑使用脚本文件中的操作系统管道将密码传递给SQL * Plus。例如：

$ echo system_password | sqlplus SYSTEM @MYSCRIPT

或者，运行以下命令：

$ sqlplus << EOF SYSTEM / system_password SELECT ...退出EOF

在前面的示例中， system_password是SYSTEM用户的密码。
