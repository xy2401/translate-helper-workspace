4.4启用Java客户端

要在客户端和服务器之间运行Java，您必须执行以下活动：

  1. 在客户端上安装Java SE
  2. 设置环境变量

4.4.1在客户端上安装Java SE

客户端需要Java Development Kit（JDK）1.6或更高版本。要确认您使用的JDK版本，请在命令行上运行以下命令：

$ java java /usr/local/jdk1.6.0_26/bin/java $ javac /usr/local/jdk1.6.0_26/bin/javac $ java -version java version“1.6.0_26”

4.4.2设置环境变量

在客户端上安装JDK后，将目录路径添加到以下环境变量：

  • $ JAVA_HOME

    必须将此变量设置为已安装的JDK库的顶级目录。

  • $ PATH

    此变量必须包含$JAVA_HOME/bin 。

  • $ LD_LIBRARY_PATH

    该变量必须包含$JAVA_HOME/lib 。

Java客户端所需的JAR文件

要确保Java客户端与服务器成功通信，请在CLASSPATH包含以下文件：

注意：

对于在不同平台上运行的Oracle JVM， CLASSPATH要求的细节可能会有所不同。您必须确保存在CLASSPATH所有元素（如Oracle JVM实用程序的脚本中所定义）。

  • 对于JDK 8，包括$JAVA_HOME/lib/dt.jar

  • 对于JRE 8，包括$JAVA_HOME/lib/rt.jar

  • 对于与JDBC的任何交互，包括$ORACLE_HOME/jdbc/lib/ojdbc8.jar

  • 对于使用SSL的任何客户端，包括$ORACLE_HOME/jlib/jssl-1_2.jar和$ORACLE_HOME/jlib/javax-ssl-1_2.jar

  • 对于使用Java Transaction API（JTA）功能的任何客户端，包括$ORACLE_HOME/jlib/jta.jar

  • 对于使用Java命名和目录接口（JNDI）功能的任何客户端，请包含$ORACLE_HOME/jlib/jndi.jar

客户端上的服务器应用程序开发

如果您在客户端上开发和编译服务器应用程序并希望使用服务器上加载的相同Java Archive（JAR）文件，则在CLASSPATH包含$ORACLE_HOME/lib/aurora.zip 。运行Java客户端不需要这样做。
