3.2使用Directory Server进行集中管理

当Oracle网络使用目录服务器时，目录服务器将全局数据库链接作为网络中Oracle数据库的网络服务名称进行管理。用户和程序可以使用全局链接来访问相应数据库中的对象。

Oracle Net Services使用集中式目录服务器作为存储连接标识符的主要方法之一。客户端在其连接字符串中使用连接标识符，目录服务器将连接标识符解析为传递回客户端的连接描述符。此功能称为目录命名 。这种类型的基础架构降低了管理和配置网络资源的成本。

Oracle Internet Directory提供了一种用于管理和配置分布式Oracle网络的集中式机制。目录服务器可以替换客户端和服务器端本地化的tnsnames.ora文件。

  2. 目录服务器解析连接标识符并检索客户端的连接描述符。

  3. 客户端使用连接描述符将连接请求发送到侦听器。

每个条目由a唯一标识 专有名称（DN） 。DN指示条目在目录服务器层次结构中的确切位置。db1的DN是dn:cn=db1,cn=OracleContext,dc=jp,dc=example,dc=com 。销售的DN是dn:cn=sales,cn=OracleContext,dc=jp,dc=example,dc=com 。DN的格式将DIT的最低组成部分放在左侧，然后逐渐向上移动DIT。

每个DN由一系列的序列组成相对专有名称（RDN）。RDN由一个 属性 ，例如cn ，以及值，例如db1或sales 。db1的RDN是cn=db1 ，销售的RDN是cn=sales 。属性及其值唯一标识条目。

3.2.1.1域组件命名空间的完全限定名称

对于域组件名称空间，为客户端定义的默认目录条目必须采用以下格式之一：

DC [特区] [...]OU，DC [特区] [...]

在前面的语法中，[dc]表示可选的域组件，[...]表示其他域组件条目，ou表示组织单元条目。

客户端在连接标识符中使用的完全限定名称必须采用以下格式之一：

cn.dc [.DC] [...]CN [.ou] @dc [.DC] [...]

在前面的语法中，[cn]表示Oracle Net条目。

示例3-1使用具有组织单位的完全限定名称

考虑一个目录服务器，其中包含数据库对象销售条目，其DN为cn=sales,cn=OracleContext,dc=jp,dc=example,dc=com 。在此示例中，客户端需要sales.jp.example.com的连接标识符。

考虑一个包含数据库对象销售的类似条目，其DN为cn=sales,cn=OracleContext, ou=mktg ,dc=jp,dc=example,dc=com 。

由于域组件必须与组织单元分开，因此客户端必须使用cn.ou@dc.dc.dc格式。客户端要求连接标识符为sales.mktg@jp.example.com 。

3.2.1.2 X.500命名空间的完全限定名称

对于X.500名称空间，为客户端定义的默认目录条目必须采用以下格式之一：

[ou]，o [ou]，o，c

在前面的格式中，[ou]表示可选的组织单位名称，o表示组织，c表示国家/地区。

客户端用作连接标识符的完全限定名称必须采用以下格式之一：

cn [.ou] .o cn [.ou] .oc

在前面的格式中，cn表示Oracle Net条目。

例如，如果目录包含DN为cn=sales,cn=OracleContext,ou=mktg,o=example,c=jp数据库对象sales ，则客户端需要sales.mktg.example.jp的连接标识符。图3-6说明了这个例子。

3.2.1.4使用条目的完全限定名称

考虑相同的目录结构， 如图3-7所示，但是客户端的Oracle主目录配置了默认领域Oracle Context of cn=OracleContext,o=example,c=jp 。

由于客户端配置的默认Oracle上下文与目录服务器中的销售位置不匹配，因此使用销售的连接字符串不起作用。相反，客户必须专门识别销售地点，这可以通过以下方式之一完成：

  • 条目的完整DN可以在连接字符串中使用，例如：

    CONNECT username @“cn = sales，cn = OracleContext，o = example，c = us”输入密码： password

    JDBC Thin驱动程序仅在使用完整DN时才支持完全限定命名。但是，许多应用程序不支持使用DN。

  • 该条目可以由完全限定的DNS样式名称引用，该名称由目录命名适配器映射到LDAP目录中数据库对象的完整x.500 DN，例如：

    CONNECT username @ sales.example.us输入密码： password

注意：

JDBC OCI Driver支持完全限定的命名。仅当使用完整DN时， JDBC Thin Driver才支持完全限定命名。有关其他信息，请参阅Oracle数据库JDBC开发人员指南 。

3.2.2了解Oracle上下文

在图3-8中 ，条目db1和sales位于cn=OracleContext 。此条目是一个特殊的RDN，称为 Oracle Context 。Oracle Context下的条目支持各种启用目录的功能，包括目录命名。

图3-8 DIT中的Oracle Context


“图3-8 DIT中的Oracle上下文”的描述

在目录配置期间，您可以设置默认的Oracle Context。客户端使用此Oracle Context作为在目录服务器中查找连接标识符的默认位置。对于Oracle Internet Directory，位于DIT根目录的Oracle上下文（DN为dn:cn=OracleContext ）指向身份管理领域中的默认Oracle上下文。身份管理领域是由相同管理策略管理的身份集合。此Oracle Context称为领域Oracle Context 。除非配置为使用其他Oracle Context，否则客户端将使用此特定于域的Oracle Context。

默认的Oracle Context会影响连接字符串。例如，如果客户端必须经常访问db1和sales条目，那么合理的默认Oracle Context将是dc=jp,dc=example,dc=com 。如果客户端目录条目与服务所在的目录条目不匹配，则客户端必须在连接字符串中指定条目的完全限定名称，如“ 使用目录命名的客户端连接 ”中所述 。

注意：

RDN cn=OracleContext不必在连接字符串中明确指定。

3.2.3了解网络服务别名条目

通过目录命名，您可以创建除数据库服务和网络服务名称条目之外的网络服务别名条目。网络服务别名是网络服务名称或数据库服务的备用名称。网络服务别名条目没有连接描述符信息。相反，它引用了作为别名的条目的位置。当客户端请求网络服务别名的目录查找时，该目录确定该条目是网络服务别名并完成查找，就好像它是引用的条目一样。例如，在图3-9中 ，为db1的数据库服务创建了db1alias的网络服务别名。当db1alias用于连接数据库服务时，如CONNECT username @db1alias ，它实际上将解析并使用db1的连接描述符信息。

图3-9 Directory Server中的Net Service别名db1alias


“图3-9 Directory Server中的Net Service别名db1alias”的说明

网络服务别名有多种用途。如图3-9所示，网络服务别名可用作客户端通过其他名称引用网络服务名称的方法。另一个用途是在一个Oracle Context中为不同的Oracle Context中的数据库服务或网络服务名称提供网络服务别名。这使得数据库服务或网络服务名称可以在目录服务器中定义一次，但是由使用其他Oracle上下文的客户端引用。

在图3-10中 ，数据库服务db1位于dc=jp,dc=example,dc=com 。在dc=us,dc=example,dc=com创建名为db1的网络服务别名。这使得日本和美国的客户端可以使用连接字符串CONNECT username @db1 ，而不是美国的客户端需要指定CONNECT username @db1.jp.example.com 。

3.2.4谁可以在Directory Server中添加或修改条目

在安装期间或之后配置数据库服务条目。然后，您可以使用Oracle Enterprise Manager云控制或 Oracle Net Manager修改数据库服务条目的Oracle Net属性。您还可以使用这些工具创建网络服务名称和网络服务别名条目。

要使用这些配置工具，必须存在包含根Oracle Context和身份管理领域的DIT结构。目录管理员使用Oracle Internet Directory Configuration Assistant创建此结构。对于某些部署，目录管理员可能需要创建其他Oracle上下文。其他Oracle上下文通常用于细分大型站点，或将生产环境与测试环境分开。

某些组使用某些工具，您必须是该组的成员才能使用这些工具，如下所述：

  • 使用Database Configuration Assistant创建数据库服务条目：

    • OracleDBCreators组 （ cn=OracleDBCreators,cn=OracleContext... ）

    • OracleContextAdmins组（ cn=OracleContextAdmins,cn=Groups,cn=OracleContext... ）

  • 使用Oracle Net Manager创建网络服务名称或网络服务别名：

    • OracleNetAdmins组（ cn=OracleNetAdmins,cn=OracleContext... ）

    • OracleContextAdmins组

OracleNetAdmins组由其自己拥有。OracleNetAdmins组的成员可以创建，修改和读取对Oracle Net对象和属性的访问权限。他们还可以添加或删除组中的成员，并添加或删除组作为OracleNetAdmins组的所有者。

OracleNetAdmins组的任何成员都可以添加或删除OracleNetAdmins组中的其他成员。如果您希望其他组添加或删除OracleNetAdmins成员，则可以将OracleNetAdmins组的owner属性更改为另一个组。所有者不能是单个用户条目，但必须是组条目，组条目是由LDAP模式对象类GroupOfUniqueNames和orclPriviledgeGroup 。

OracleContextAdmins组是Oracle Context的超级用户组。OracleContextAdmins组的成员可以将所有受支持的条目类型添加到Oracle Context。

创建Oracle Context的目录用户将自动添加到这些组中。目录管理员可以将其他用户添加到这些组中。

3.2.5使用目录命名的客户端连接

大多数需要在目录服务器中执行名称查找的客户端使用匿名身份验证访问目录服务器。要执行查找，目录服务器必须允许匿名身份验证。目录服务器通常默认允许匿名身份验证，但是，某些目录服务器（如早期版本的Oracle Internet Directory）需要目录配置才能允许匿名访问。

要查找条目，客户端必须能够找到该条目所在的目录服务器。客户端以两种方式之一找到目录服务器：

  • 动态使用DNS。在这种情况下，目录服务器位置信息在中央域名服务器中存储和管理。客户端在请求处理时从DNS检索此信息。

  • 静态地位于目录服务器使用文件ldap.ora ，由Oracle Internet Directory Configuration Assistant创建并存储在客户端主机上。

找到目录后，客户端将从根Oracle Context定向到域Oracle Context。

客户端使用连接标识符以与使用其他命名方法相同的方式连接到数据库。连接标识符可以是数据库服务，网络服务名称或网络服务别名。如果默认的Oracle Context是实体所在的位置，则可以通过它们的通用名称（相对名称）来引用它们。如果不是，则连接标识符需要完全限定名称或可分辨名称。

3.2.6.1性能考虑因素

连接标识符存储在目录服务器中，供所有客户端访问。根据客户端的数量，目录服务器上可能会有很大的负载。

在连接标识符查找期间，将在特定的Oracle上下文中搜索名称。用户期望性能相对较快，因此数据库连接时间不受影响。由于查找的范围，如果查找花费的时间超过一秒，用户可能会开始注意到连接时间较慢。

您可以通过更改网络拓扑或实施复制来解决性能问题。

3.2.6.2.1认证方法

客户端使用不同的身份验证方法，具体取决于要执行的任务。

  • 对目录服务器中的信息执行查找的客户端通常使用匿名身份验证。在Oracle Database 12c中 ，可以将客户端配置为在名称查找期间对其LDAP绑定进行身份验证。需要保护其网络服务数据或禁用匿名绑定到目录的站点必须将其客户端配置为在名称查找期间使用钱包进行身份验证。这些客户端需要在sqlnet.ora文件中设置以下两个参数：

    names.ldap_authenticate_bind = TRUE wallet_location = location_value

  • 管理目录服务器条目的客户端使用目录服务器进行身份验证。Database Configuration Assistant或Oracle Net Manager可用于添加或修改条目。只有具有适当权限的已认证用户才能修改条目。使用以下身份验证方法之一：

    • 简单认证

      客户端通过DN和密码向目录服务器标识自己。服务器验证客户端发送的DN和密码是否与目录服务器中存储的DN和密码匹配。

    • 强认证

      客户端通过安全套接字层（SSL）提供的公钥加密向目录服务器标识自己。在公钥加密中，消息的发送者使用接收者的公钥加密消息。在传递时，接收者使用接收者的私钥解密该消息。

3.2.6.2.2访问控制列表

身份验证与访问控制列表（ACL）一起使用，以确定客户端是否可以在目录服务器中读取，修改或添加信息。ACL指定以下内容：

  • 用户可以访问的条目。

  • 用于访问条目的身份验证方法。

  • 访问权限，或用户可以对对象执行的操作，例如读取或写入。

为一组用户建立ACL。在Oracle Context创建期间，将创建OracleDBCreators ， OracleNetAdmins和OracleContextAdmins组。

使用Oracle Net Configuration Assistant创建Oracle Context的用户将自动添加为这些组的第一个成员。

组 ACL要求

3.2.7使用Microsoft Active Directory支持目录命名的限制

除了Oracle Internet Directory之外， Microsoft Active Directory还提供了目录命名支持具有以下限制：

  • Oracle仅在Microsoft Windows操作系统上提供对Microsoft Active Directory的支持。因此，客户端计算机和数据库服务器必须在Microsoft Windows操作系统上运行才能访问或创建Microsoft Active Directory中的条目。

  • Microsoft Active Directory不支持以下功能：

    • 多个Oracle上下文

      Microsoft Active Directory只能支持一个Oracle Context。

    • 网络服务别名

      您无法在Microsoft Active Directory中创建网络服务别名。但是，您可以创建网络服务名称。
