2.1关于安装Oracle C ++调用接口

OCCI作为Oracle数据库的一部分安装。要确定其他配置要求，请参阅特定于您的平台的“ Oracle数据库安装指南”和“ Oracle数据库客户端安装指南” 。

2.2关于升级注意事项

要使用此版本中提供的新功能，在从早期Oracle客户端版本升级时，必须重新编译并重新链接所有OCCI应用程序，包括通过对象类型转换程序实用程序生成的类。

2.3关于确定Oracle数据库版本

当应用程序使用具有不同服务器版本或客户端补丁集的多个单独代码路径时，您可以在编译期间和运行时验证这些选项。

2.4关于Instant Client

通过消除对ORACLE_HOME的需求，Instant Client功能使部署基于OCCI的客户应用程序变得非常容易和快速。存储空间要求是额外的好处; Instant Client共享库占用完整客户端安装所需的磁盘空间的四分之一。

2.4.1关于Instant Client的优点

  • 安装只涉及复制四个文件。

  • 客户端的存储空间要求很低

  • 部署的应用程序不会丢失任何功能或性能

  • 使用ISV应用程序简化包装

OCCI Instant Client功能简化了OCCI的安装。尽管OCCI是独立的ORACLE_HOME的即时客户端模式的设置，依赖于应用程序ORACLE_HOME设置，可以将其设置为适当的值继续操作。Instant Client模式的激活仅取决于加载Instant Client数据共享库的能力。特别是，此功能允许与使用ORACLE_HOME作为其数据的Oracle应用程序进行互操作，但使用较新版本的Oracle Client。

2.4.2关于安装Instant Client

OCCI只需要由操作系统的动态加载程序加载四个共享库（或动态链接库，因为它们在某些操作系统上被调用）。使用Oracle Database 12 c第1版（12.1）库名称;库名称的数字部分更改为与将来的版本号保持一致。

为了清晰和易于开发，从Oracle Database 12 c第1版（12.1）开始更改库结构。客户端共享库libclntsh.so.12.1依赖于libclntshcore.so.12.1 。libclntshcore.so.12.1库包含NLS和CORE功能。这两个库必须安装在同一目录中。

  • （OCI即时客户端数据共享库 libociei.so在Linux和UNIX和oraociei11.dll在Windows上）;正确安装此文件可确定您是否在Instant Client模式下运行

  • 客户端代码库 （Linux和UNIX上的libclntsh.so.11.1和Windows上的oci.dll ）

  • 安全库 （ libnnz11.so在Linux和UNIX和orannzsbb11.dll在Windows上）

  • OCCI库 （ libocci.so.11.1在Linux和UNIX和oraocci11.dll在Windows上）

注意：

libclntshcore.so.12.1文件现在必须与数据共享库驻留在同一个库中。

Instant Client库也可从Oracle技术网（OTN）网站获得：

如果可以通过操作系统库路径变量（Linux和UNIX上的LD_LIBRARY_PATH和Windows上的PATH上的目录访问这四个库，则OCCI将以Instant Client模式运行。在此模式下，不依赖于ORACLE_HOME并且OCCI不需要ORACLE_HOME中提供的其他任何代码和数据文件。

  2. 将操作系统共享库路径环境变量（Linux和UNIX上的LD_LIBRARY_PATH和Windows上的PATH ）设置为步骤1中使用的目录instantclient_12_1 。

2.4.2.1.1关于Instant Client SDK

Instant Client也可以作为SDK包下载。SDK包含所有必需的头文件和用于在Instant Client环境中开发OCCI应用程序的makefile。一旦开发，这些应用程序可以部署在任何客户端环境中。SDK具有以下附加功能：

  • 它包含C ++演示程序。

  • 它包括在Windows上链接应用程序所需的库，并提供Make.bat文件来构建演示。

  • Makefile demo.mk用于构建Linux和UNIX的演示。在链接应用程序之前， instantclient_12_1目录必须位于LD_LIBRARY_PATH 。这些程序需要在instantclient_12_1目录中分别为客户端代码库和OCCI库libclntsh.so.12.1和libocci.so.12.1符号链接。演示Makefile， demo.mk ，在链接步骤之前生成这些。这些符号链接也可以在shell脚本中创建：

    cd instantclient_12_1 ln -s libclntsh.so.11.1 libclntsh.so ln -s libocci.so.11.1 libocci.so

  • SDK还包含对象类型转换器（OTT）实用程序及其生成应用程序头文件的类。

2.4.2.2关于完整客户端安装

如果通过选择“ 管理”选项执行完整的客户端安装，

  • 在Linux或UNIX平台上，可以从$ ORACLE_HOME /instantclient目录中复制libociei.so库。可以在完整的Oracle安装中从$ ORACLE_HOME /lib目录复制所有其他库。

  • 在Windows上，可以从ORACLE_HOME \instantclient目录复制oraociei11.dll库。可以从ORACLE_HOME \bin目录复制所有其他Windows库。

2.4.2.3运行Oracle Universal Installer

如果未安装数据库，则可以通过从Oracle Universal Installer中选择Instant Client选项来安装这些库。完成这些步骤后，您就可以开始运行OCCI应用程序了。

  1. 将Instant Client共享库安装到诸如instantclient_12_1的目录中。
  2. 将操作系统共享库路径环境变量设置为步骤1中的目录。例如，在Linux或UNIX上，将LD_LIBRARY_PATH设置为instantclient_12_1 。在Windows上，设置PATH以找到instantclient_12_1目录。
2.4.2.4关于Instant Client CD

您还可以从Instant Client CD安装Instant Client。您必须在空目录或其他系统上安装Instant Client。

操作系统Library Path变量上应该只有一组Oracle库;如果您有多个Instant Client库的目录或副本，则操作系统库路径中只应有一个目录。

同样，如果您在同一系统的ORACLE_HOME上也有安装，请不要将ORACLE_HOME /lib和Instant Client目录放在操作系统库路径上，而不管它们在库路径中的显示顺序如何。ORACLE_HOME /lib目录（用于非Instant Client操作）或Instant Client目录（用于Instant Client操作）中只应有一个位于操作系统Library Path变量上。

2.4.3关于使用Instant Client

Instant Client功能专为运行生产应用程序而设计。对于开发，请使用Instant Client SDK或完整安装来访问OCCI头文件，makefile，演示程序等。

2.4.4在UNIX上修补Instant Client共享库

Windows平台上不提供此功能。

由于Instant Client主要是部署功能，因此其设计目标之一是减少必要文件的数量和大小。因此，Instant Client部署不包括用于修补共享库的所有文件。您应该在基于ORACLE_HOME的OPATCH上使用OPATCH实用程序来修补Instant Client共享库。OPATCH实用程序将用于Linux和UNIX的ORACLE_HOME安装的修补信息存储在libclntsh.so.11.1中。可以使用genezi实用程序检索此信息：

genezi -v

如果未在部署Instant Client的系统上安装genezi实用程序，则可以从ORACLE_HOME系统的ORACLE_HOME /bin目录中复制它。

在ORACLE_HOME环境中应用修补程序后，将“ 关于安装Instant Client ”中列出的文件复制到Instant Client目录。您可以生成Instant Client *.zip文件，而不是复制单个文件，如“ 重新生成数据共享库和Zip文件 ”中所述 。然后，您可以将zip文件复制到目标系统并解压缩，而不是复制单个文件。

2.4.6关于Instant Client的数据库连接名称

所有不需要使用ORACLE_HOME或TNS_ADMIN来定位配置文件（如tnsnames.ora或sqlnet.ora Oracle网络命名方法都可以在Instant Client模式下工作。

可以使用以下格式指定createConnection（）调用中的connectString参数：

  • 作为SQL Connect URL字符串，格式如下：

    // host ：[ port ] [/ service name ]

    如：

    // myserver111：5521 / bjava21

  • 作为Oracle Net关键字 - 值对 。例如：

    （DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = myserver111）（PORT = 5521））（CONNECT_DATA =（SERVICE_NAME = bjava21）））

  • 作为在为LDAP服务器发现配置站点时通过目录命名解析的连接名称 。

  • 作为tnsnames.ora文件中的条目 。

    如果未设置TNS_ADMIN环境变量，并且使用了诸如inst1类的TNSNAMES条目，则必须设置ORACLE_HOME变量，并且配置文件应位于$ ORACLE_HOME /network/admin目录中。

    如果设置了TNS_ADMIN环境变量，则需要TNS_ADMIN定位配置文件的命名方法才能继续工作。

    在这种情况下， ORACLE_HOME变量仅用于查找Oracle Net配置文件，OCCI客户端代码库的其他组件不使用ORACLE_HOME的值。

通过将环境变量（Linux和UNIX上为TWO_TASK ，Windows上为LOCAL ）设置为前面描述的值之一，可以支持createConnection（）的空connectString参数。

2.4.7设置OCCI Instant Client的环境变量

ORACLE_HOME环境变量不再确定全球化支持，CORE和错误消息文件的位置。仅OCCI应用程序不应要求设置ORACLE_HOME 。但是，如果已设置，则不会对OCCI的操作产生影响。OCCI始终从数据共享库获取其数据。如果数据共享库不可用，则仅使用ORACLE_HOME并假定ORACLE_HOME安装。设置后， ORACLE_HOME应该是标识目录的有效操作系统路径名。

Instant Client模式ORA_NLS忽略环境变量ORA_NLS33 ， ORA_NLS32和ORA_NLS 。

在Instant Client模式下，如果未设置ORA_TZFILE变量，则使用数据共享库中较大的默认timezlrg_ n .dat文件（其中n是文件的版本号）。如果使用数据共享库中较小的timezone _ n .dat文件，则将ORA_TZFILE环境变量设置为文件名，而不包含任何绝对路径名或相对路径名，如例2-3所示。

如果OCCI未在Instant Client模式下运行，因为数据共享库不可用，则ORA_TZFILE变量（如果已设置）将命名完整的路径名。

如果使用TNSNAMES条目，则TNS_ADMIN目录必须包含TNSNAMES配置文件。如果未设置TNS_ADMIN ，则ORACLE_HOME /network/admin目录必须包含Oracle Net Services配置文件。

示例2-3如何设置ORA_TZFILE环境变量

在Linux和UNIX上：

setenv ORA_TZFILE timezone_ n .dat

在Windows上：

设置ORA_TZFILE timezone_ n .dat

2.5关于Instant Client Light（英文）

Instant Client Light（英语）通过另一个63 MB进一步减少了客户端安装对Instant Client的安装空间要求。具体而言，当完整的Instant Client共享库libociei.so占用67个时，在UNIX和UNIX上安装Instant Client Light（英语）共享库libociicus.so和在oraociicus11.dll for Windows上，在UNIX平台上占用4 MB。 MB的磁盘空间。

顾名思义，Instant Client Light（英文）适用于需要英文错误消息并使用US7ASCII ， WE8DEC或Unicode字符集的应用程序。Instant Client Light（英语）对NLS_LANG设置的TERRITORY字段也没有限制。因此，满足这些字符和区域标准的应用程序如果在Instant Client Light（英语）环境中运行，则可以显着减少其占用空间。

2.5.1关于Instant Client Light的全球化设置（英语）

Instant Client Light（英文）支持以下字符集：

  • 单字节字符集包括US7ASCII ， WE8DEC ， WE8MSWIN1252和WE8ISO8859P1 。

  • Unicode字符集包括UTF8 ， AL16UTF16和AL32UTF8 。

如果应用程序尝试在客户端或数据库上尝试使用此处未列出的字符集或国家字符集，则Instant Client Light（英语）将返回错误消息。此处列出的可能错误消息仅提供英文版：

  • ORA-12734 Instant Client Light：不支持的客户端国家字符集（NLS_LANG值设置）

  • ORA-12735 Instant Client Light：不支持的客户端字符集（NLS_LANG值集）

  • ORA-12736 Instant Client Light：不支持的服务器国家字符集（NLS_LANG值设置）

  • ORA-12737 Instant Client Light：不支持的服务器字符集（NLS_LANG值设置）

设置NLS_LANG参数时，请使用以下命令：

美国领土 。 字符集

territory是可以通过NLS_LANG指定的任何有效区域，而charset是本节中列出的字符集。

2.6关于在Microsoft Visual C ++中使用OCCI

Oracle数据库12 c解除1（12.1）包括用于开发与Microsoft Visual C ++版本10.0（.NET 2010 SP1 10.0）时，Microsoft Visual C ++版本11.0（.NET 2012 11.0）时，Microsoft Visual C ++ 12.0版应用OCCI库（.NET 2013 12.0）和英特尔12.1 C编译器与Microsoft Visual Studio 2010 STL。不再支持Microsoft Visual C ++ 8.0和9.0版。

Microsoft Visual C ++ 10.0版库安装在以下默认位置：

ORACLE_BASE \ ORACLE_HOME \ bin \ oraocci12.dll ORACLE_BASE \ ORACLE_HOME \ oci \ lib \ msvc \ oraocci12.lib

这两个文件的副本也安装在以下目录中：

ORACLE_BASE \ ORACLE_HOME \ oci \ lib \ msvc \ vc10

Microsoft Visual C ++ 2012 OCCI库安装在以下默认位置：

ORACLE_BASE \ ORACLE_HOME \ oci \ lib \ msvc \ vc11

在使用MSVC ++ 2012开发OCCI应用程序时，请确保从此目录中正确选择OCCI库以进行链接和执行。

Microsoft Visual C ++ 2013 OCCI库安装在以下默认位置：

ORACLE_BASE \ ORACLE_HOME \ oci \ lib \ msvc \ vc12

使用MSVC ++ 2013开发OCCI应用程序时，请确保从此目录中正确选择OCCI库以进行链接和执行。

应用程序应链接到相应的OCCI库。您必须确保相应的DLL位于Windows系统PATH中。

链接到MSVCRTD.DLL （Microsoft C-Runtime的调试版本， /MDd编译器标志）的应用程序应链接到这些特定的OCCI库： oraocci12d.lib和oraocci12d.dll 。

所有Instant Client程序包都包含与Microsoft Visual C ++ 10.0版兼容的OCCI DLL版本。
