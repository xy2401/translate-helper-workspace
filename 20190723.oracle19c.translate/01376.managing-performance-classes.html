9.5管理性能类

每个策略集都包含一个或多个性能类。每个性能类都定义了服务器上的一种工作负载。

  • 创建性能类
  • 删除性能类
    您可以删除不再需要的Performance Class。
  • 重命名性能类
    您可以使用“编辑策略集”向导重命名性能类。
  • 编辑现有的性能类
    编辑Performance Classes时，您可以为系统创建，编辑，重命名或删除性能类。
  • 指定分类器的评估顺序
    分类器生成布尔表达式，每次工作请求进入系统时都会对其进行评估。

9.5.1创建性能类

要创建或编辑Performance Classes及其使用的分类器，请执行以下任务：

  1. 启动“策略集编辑器”向导。从Oracle Enterprise Manager Cloud Control中的群集目标页面中，选择“ 管理” ，然后选择“服务质量管理” ，再选择“ 编辑策略集” 。
  2. 转到向导中的第二个屏幕。

    在“编辑策略集：性能类”页面上，将显示可用的“性能类”。如果这是第一次配置系统，则会显示每个数据库服务的性能类以及默认性能类。


    说明cc12_policy_edit_2.gif的说明

    要为工作请求指定分类器，必须至少指定一个服务名称。如果指定多个服务，则使用逗号分隔的列表。或者，您也可以指定以下任何过滤器：

    • 逗号分隔的模块名称列表以及工作请求是否使用此列表中的模块

    • 以逗号分隔的操作列表，以及工作请求是否在此列表中执行操作

    • 逗号分隔的用户名列表，以及工作请求是否在此列表中使用用户名

    • 以逗号分隔的程序列表，以及工作请求是否在此列表中运行程序

  3. 单击“ 添加性能类” ，将显示“性能类创建”页面。在“性能类名称”文本字段中，输入“性能类”的名称。

    说明cc12_add_perf_class_1.gif的说明
  4. 在“分类器”部分中，输入信息以定义用于对工作请求进行分类的规则。首先选择数据库服务，然后为使用指定数据库服务的工作请求关联的模块 ， 操作 ， UserName或程序名指定匹配值（In Set）。您还可以为这些属性指定排除值（Not In Set）。

    如果要为Performance Class添加多个分类器，请单击Add Classifier按钮并输入相应的信息。

    注意：

    在评估Performance Class的分类器时，使用AND操作将所有指定的值与工作请求属性进行比较;如果为Performance Class指定多个分类器，则使用OR运算组合该Performance Class的每个分类器评估的结果。

  5. 为Performance Class定义所有分类器后，单击Next按钮，直到到达向导结束。查看您指定的信息，然后单击“ 提交策略集” 。

9.5.2删除性能等级

您可以删除不再需要的Performance Class。

  1. 启动“编辑策略”向导。
  2. 转到向导中的第二个屏幕。
    在“编辑策略集：性能类”页面上，将显示可用的“性能类”。
  3. 选择“性能类”，然后单击“ 删除性能类” 。
  4. 前进到“编辑策略集”向导的末尾，然后单击“ 提交策略集”以使更改成为永久更改。

9.5.3重命名性能类

您可以使用“编辑策略集”向导重命名性能类。

  1. 启动“编辑策略”向导。
  2. 转到向导中的第二个屏幕。
    在“编辑策略集：性能类”页面上，将显示可用的“性能类”。
  3. 选择“性能类”，然后单击“ 重命名性能类” 。
  4. 在“重命名性能类”页上，输入“性能类”的新名称，然后单击“ 确定” 。
  5. 前进到“编辑策略集”向导的末尾，然后单击“ 提交策略集”以使更改成为永久更改。

9.5.4编辑现有的性能类

编辑Performance Classes时，您可以为系统创建，编辑，重命名或删除性能类。

您可以使用“编辑性能类”按钮修改现有性能类的分类器。

  1. 启动“策略集编辑器”向导。
    1. 从Oracle Enterprise Manager Cloud Control中的群集目标页面中，选择“ 管理” 。
    2. 选择服务质量管理 。
    3. 选择“ 编辑策略集” 。
  2. 转到向导中的第二个屏幕。
    在“编辑策略集：性能类”页面上，将显示可用的“性能类”。
  3. 选择要修改的Performance Class，然后单击Edit Performance Class 。
  4. 完成更改后，单击“ 确定” 。
  5. 前进到策略集编辑器的最后一页，然后单击“ 提交策略集” 。

9.5.4.1添加分类器

要将分类器添加到Performance Class，请执行以下步骤：

  1. 启动“策略集编辑器”向导。
  2. 转到向导的第二个屏幕，即Edit Policy Set：Performance Classes页面。
  3. 选择要修改的Performance Class，然后单击Edit Performance Class 。
  4. 在“编辑性能类”页面的“分类器”部分中，单击“ 添加分类器”按钮并输入相应的信息。完成后，单击“ 确定” 。
  5. 然后，您将前进到“编辑策略集”向导的末尾，然后单击“ 提交策略集”以使更改成为永久更改。

9.5.4.2更改分类器

要修改Performance Class的一个或多个分类器，请执行以下步骤：

  2. 转到向导的第二个屏幕，即Edit Policy Set：Performance Classes页面。

要删除Performance Class的一个或多个分类器，请执行以下步骤：

9.5.5指定分类器的评估顺序

分类器生成布尔表达式，每次工作请求进入系统时都会对其进行评估。

评估为TRUE的第一个分类器确定该工作请求的Performance Class。要确保将工作请求放入正确的Performance Classes中，您必须小心指定评估分类器的顺序。

要设置分类器的评估顺序，请执行以下步骤：

  1. 启动“策略集编辑器”向导。
  2. 继续执行向导中的第三页，标题为“编辑策略集：分类器排序”。
  3. 使用分类器右侧的箭头键在列表中向上或向下移动分类器。首先评估列表顶部的Performance Classes的分类器。如果工作请求与该Performance Class的分类器不匹配，则继续评估列表中的下一个Performance Class，直到不再进行进一步的评估。如果工作请求与Performance Class的分类器匹配，则工作请求将与该Performance Class相关联，并且评估将停止。

    为了正确分类工作请求，您应该将具有最严格分类器的Performance Classes放在列表顶部，并将具有最多松散分类器的Performance Classes放在列表底部附近。具有最常规分类器的Default_pc性能类应始终位于列表的底部。

  4. 然后，您将前进到“编辑策略集”向导的末尾，然后单击“ 提交策略集”以使更改成为永久更改。
