1.107 DML_LOCKS

DML_LOCKS指定DML锁的最大数量 - 对于事务中修改的每个表一个。

属性 描述

参数类型

整数

默认值

派生：4 * TRANSACTIONS

可修改

没有

可在PDB中修改

没有

价值范围

20到无限;设置为0将禁用队列

基本

没有

Oracle RAC

您必须为每个实例设置此参数，并且所有实例必须具有正值或全部必须为0。

DML锁是在正在进行DML操作（插入，更新，删除）的表上获得的锁。DML_LOCKS值应等于所有用户当前引用的表的锁总数。例如，如果三个用户正在修改一个表中的数据，则需要三个条目。如果三个用户正在修改两个表中的数据，那么将需要六个条目。

默认值假定每个事务引用的平均四个表。对于某些系统，此值可能不够。

队列是共享内存结构，用于序列化对数据库资源的访问。如果将DML_LOCKS的值设置为0 ，则禁用队列并略微提高性能。但是，将DML_LOCKS设置为0时，应注意以下限制：

  • 您不能使用DROP TABLE ， CREATE INDEX语句

  • 您不能使用显式锁定语句，例如LOCK TABLE IN EXCLUSIVE MODE

  • 企业管理器无法在DML_LOCKS设置为0任何实例上运行

Oracle在并行DML期间比在串行执行期间拥有更多锁。因此，如果您的数据库支持大量并行DML，则可能需要增加此参数的值。
