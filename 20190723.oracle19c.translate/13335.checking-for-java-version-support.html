检查Java版本支持

通常情况。您在要升级的Oracle主目录中启动AutoUpgrade JAR文件。但是，Oracle主目录必须包含Java 8。

AutoUpgrade实用程序是Java 8发行版JAR文件，在Oracle Database 12c第1版（12.1.0.2）之后的版本中可用于Oracle数据库Oracle主目录中。Oracle主目录中Java版本的路径是Oracle-home /jdk/bin/java ，其中Oracle-home是源数据库Oracle主目录（您正在升级的早期Oracle数据库版本）。例如：

$ ORACLE_HOME / jdk / bin / java -version

结果应该是Java 8版本。例如：

[oracle @ sales] $ $ ORACLE_HOME / jdk / bin / java -version java version“1.8.0_152”Java（TM）SE运行时环境（版本1.8.0_152-b16）Java HotSpot（TM）64位服务器VM（构建） 25.152-b16，混合模式）

升级Oracle Database 11g第2版（11.2.0.4）或更早版本时，可以使用AutoUpgrade。但是，由于Oracle数据库Oracle Oracle数据库11g第2版及更早版本中不提供Java 8，因此必须从Target Oracle主目录（要升级的Oracle主目录）中的Java发行版启动AutoUpgrade，或者安装源Oracle Oracle中的Java 8。

对autoupgrade.jar -version成功响应应类似于以下内容：

[oracle @ sales~] $ $ ORACLE_HOME / jdk / bin / java -jar autoupgrade.jar -version build.version 20190201 build.date 2019/02/01 11:50:22 build.label RDBMS_PT.AUTOUPGRADE_LINUX.X64_190131.1800
