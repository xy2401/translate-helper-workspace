60 DBMS_DISTRIBUTED_TRUST_ADMIN

DBMS_DISTRIBUTED_TRUST_ADMIN过程维护可信服务器列表。使用这些过程来定义服务器是否可信。如果数据库不受信任，Oracle将拒绝来自数据库的当前用户数据库链接。

本章包含以下主题：

  • 概观

  • 安全模型

  • 例子

  • DBMS_DISTRIBUTED_TRUST_ADMIN子程序的摘要

60.1 DBMS_DISTRIBUTED_TRUST_ADMIN概述

Oracle使用本地可信服务器列表以及存储在企业LDAP目录服务中的企业域成员资格列表来确定另一个数据库是否可信。LDAP目录服务条目使用Oracle Enterprise Manager中的Enterprise Security Manager工具进行管理。

如果符合以下条件，Oracle会认为另一个数据库是“受信任的”：

  1. 它与目标数据库位于目录服务中的同一企业域中。

  2. 企业域在目录服务中标记为受信任。

  3. 它在本地可信服务器列表中未列为不受信任。如果所涉及的两个数据库彼此信任，则仅从另一个数据库接受当前用户数据库链接。

无论目录服务中列出了什么，您都可以在“可信任服务器列表”中本地列出数据库服务器。但是，如果列出的数据库与本地数据库不在同一个域中，或者该域不受信任，则该条目将不起作用。

此功能是Oracle Advanced Security Option的企业用户安全功能的一部分。

60.2 DBMS_DISTRIBUTED_TRUST_ADMIN安全模型

要执行DBMS_DISTRIBUTED_TRUST_ADMIN ，必须将EXECUTE_CATALOG_ROLE角色授予DBA。要从视图TRUSTED_SERVERS选择，必须将SELECT_CATALOG_ROLE角色授予DBA。

了解所有服务器是受信任还是不受信任非常重要。如果数据库已信任所有数据库，或者该数据库已受信任，则使用ALLOW_SERVER过程信任特定服务器不会产生任何影响。同样，如果数据库已经不信任任何数据库或者该数据库已经不受信任，则使用DENY_SERVER过程拒绝特定服务器不会产生任何影响。

DENY_ALL和ALLOW_ALL过程DENY_ALL删除使用ALLOW_SERVER过程或DENY_SERVER过程显式允许或拒绝的所有条目（换句话说，服务器名称）。

60.3 DBMS_DISTRIBUTED_TRUST_ADMIN示例

如果尚未使用DBMS_DISTRIBUTED_TRUST_ADMIN包更改信任列表，则默认情况下，如果该域在目录服务中列为受信任域，则您信任该域中的所有数据库：

SELECT * FROM TRUSTED_SERVERS; TRUST NAME --------- ---------------------值得信赖

由于所有服务器当前都是受信任的，因此您可以执行DENY_SERVER过程并指定特定服务器不受信任：

EXECUTE DBMS_DISTRIBUTED_TRUST_ADMIN.DENY_SERVER（'SALES.US.AMERICAS.ACME_AUTO.COM'）; PL / SQL过程成功完成。SELECT * FROM TRUSTED_SERVERS; TRUST NAME --------- --------------------------------------- --------不受信任的SALES.US.AMERICAS.ACME_AUTO.COM

通过执行DENY_ALL过程 ，您可以选择不信任任何数据库服务器：

EXECUTE DBMS_DISTRIBUTED_TRUST_ADMIN.DENY_ALL; PL / SQL过程成功完成。SELECT * FROM TRUSTED_SERVERS; TRUST NAME --------- --------------------------------------- --------不受信任的全部

ALLOW_SERVER过程可用于指定要信任一个特定数据库：

EXECUTE DBMS_DISTRIBUTED_TRUST_ADMIN.ALLOW_SERVER（'SALES.US.AMERICAS.ACME_AUTO.COM'）; PL / SQL过程成功完成。SELECT * FROM TRUSTED_SERVERS; TRUST NAME --------- --------------------------------------- --------- Trusted SALES.US.AMERICAS.ACME_AUTO.COM

60.4 DBMS_DISTRIBUTED_TRUST_ADMIN子程序摘要

此表列出并简要描述了DBMS_DISTRIBUTED_TRUST_ADMIN子程序。

表60-1 DBMS_DISTRIBUTED_TRUST_ADMIN包子程序

子程序 描述

ALLOW_ALL程序

清空列表并插入一行，指示应信任所有服务器

ALLOW_SERVER程序

即使列表中指示了deny所有服务器，也允许允许特定服务器访问

DENY_ALL程序

清空列表并插入一行，指示所有服务器都应该不受信任

DENY_SERVER程序

即使在列表中指示了all all，也允许拒绝特定服务器访问

60.4.1 ALLOW_ALL程序

此过程清空“可信服务器列表”，并指定允许在企业目录服务中属于同一域中的受信任域的所有服务器以及属于同一域的所有服务器。

视图TRUSTED_SERVERS将显示“ TRUSTED ALL ”，表示数据库信任企业目录服务当前信任的所有服务器。

句法

DBMS_DISTRIBUTED_TRUST_ADMIN.ALLOW_ALL;

使用说明

ALLOW_ALL仅适用于在企业目录服务和同一企业域中列为受信任的服务器。

60.4.2 ALLOW_SERVER过程

此过程可确保指定的服务器被视为受信任（即使您之前已指定“ deny all ”）。

句法

DBMS_DISTRIBUTED_TRUST_ADMIN.ALLOW_SERVER（服务器IN VARCHAR2）;

参数

表60-2 ALLOW_SERVER过程参数

参数 描述

服务器

要信任的服务器的唯一，完全限定名称。

使用说明

如果可信服务器列表包含条目“ deny all ”，则此过程添加一个规范，指示要信任特定数据库（例如， DBx ）。

如果可信服务器列表包含条目“ allow all ”，并且列表中没有“ deny DBx ”条目，则执行此过程不会导致更改。

如果可信服务器列表包含条目“ allow all ”，并且列表中有“ deny DBx ”条目，则删除该条目。

60.4.3 DENY_ALL程序

此过程清空可信服务器列表并指定拒绝所有服务器访问。

视图TRUSTED_SERVERS将显示“ UNTRUSTED ALL ”，表示当前没有服务器受信任。

句法

DBMS_DISTRIBUTED_TRUST_ADMIN.DENY_ALL;

60.4.4 DENY_SERVER程序

此过程可确保指定的服务器被视为不受信任（即使您之前已指定allow all ）。

句法

DBMS_DISTRIBUTED_TRUST_ADMIN.DENY_SERVER（服务器IN VARCHAR2）;

参数

表60-3 DENY_SERVER过程参数

参数 描述

服务器

不受信任的服务器的唯一，完全限定名称。

使用说明

如果可信服务器列表包含条目allow all ，则此过程添加一个条目，指示不信任指定的数据库（例如， DBx ）。

如果可信服务器列表包含条目“ deny all ”，并且列表中没有“ allow DBx ”条目，则此过程不会导致更改。

如果可信服务器列表包含条目“ deny all ”，并且如果存在“ allow DBx ”条目，则此过程将导致该条目被删除。
