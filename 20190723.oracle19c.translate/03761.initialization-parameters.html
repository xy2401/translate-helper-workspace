初始化参数

使用初始化参数配置网关。

这是通过创建初始化文件并在此文件中设置所需参数来完成的。

异构服务初始化参数与Oracle数据库初始化参数不同。异构服务初始化参数在异构服务初始化文件中设置，而不是在Oracle数据库初始化参数文件（ init.ora文件）中设置。每个网关实例都有一个异构服务初始化文件。

加密初始化参数

已向异构服务添加了加密功能，可以加密参数值。

初始化参数可能包含敏感信息，例如用户ID或密码。初始化参数存储在纯文本文件中并且不安全。已向异构服务添加了加密功能，可以加密参数值。这是通过dg4pwd实用程序完成的。要使用此功能，需要将初始化文件中的参数值设置为不带引号的星号（*）。例如：

HD_FDS_CONNECT_INFO = *

将值设置为此安全标记后，所有异构服务代理都知道实际值将存储在相关的加密密码文件中。此文件的名称是init sid .pwd ，其中sid是用于网关的Oracle系统标识符。此文件由包含初始化文件的当前目录中的dg4pwd实用程序创建。运行dg4pwd实用程序会提示dg4pwd实用程序参数的实际值，该实用程序将加密并存储在密码文件中。加密的初始化参数被隐式视为PRIVATE参数，不会上传到服务器。

  • 使用dg4pwd实用程序
    dg4pwd实用程序用于加密通常以纯文本格式存储在初始化参数文件中的初始化参数。

使用dg4pwd实用程序

dg4pwd实用程序用于加密通常以纯文本格式存储在初始化参数文件中的初始化参数。

该实用程序通过读取当前目录中的初始化参数文件并查找具有该值的安全标记的参数来工作。安全标记是未加引号的星号（*）。这表示此参数的值将以加密形式存储在密码文件中。以下是设置为此值的初始化参数的示例：

HS_FDS_CONNECT_INFO = *

要加密HS_FDS_CONNECT_INFO初始化参数，请执行以下步骤：

  1. 编辑当前目录中的HS_FDS_CONNECT_INFO初始化参数文件，将值设置为不带引号的星号（*）安全标记。

  2. 运行dg4pwd实用程序，在命令行上指定网关SID，并使用可选的用户ID指定加密信息的其他所有者。

  3. dg4pwd实用程序读取初始化参数文件，并提示您输入要加密的实际值。

该命令的语法如下：

dg4pwd [ sid ] { userid }

在上面的语法示例中， [ sid ]是网关的SID ， { userid }是用于加密内容的可选用户ID。如果未指定用户标识，则使用当前用户的标识。使用此ID加密值。为了解密这些值，代理必须以该用户身份运行。以下示例假定网关SID为SYBASE：

dg4pwd SYBASE ORACLE网关密码实用程序构建网关SID SYBASE的密码文件输入HS_FDS_CONNECT_INFO SYBASE_password的值

在前面的示例中，读取初始化参数文件initSYBASE.ora 。参数HS_FDS_CONNECT_INFO被标识为需要加密。输入值（例如， SYBASE_password ）并按Enter键。如果更多参数需要加密，则实用程序会依次提示输入这些密码。加密数据存储在与初始化文件相同的目录中。在使用Oracle数据库网关之前，任何需要加密的初始化参数都应加密。

网关初始化参数

所有网关通用的通用初始化参数。

网关初始化参数可以分为两组。一个是所有网关共用的一组通用初始化参数，另一个是特定于各个网关的一组初始化参数。以下通用初始化参数是本文档中讨论的唯一初始化参数：

  • HS_BULK
  • HS_CALL_NAME
  • HS_COMMIT_POINT_STRENGTH
  • HS_DB_DOMAIN
  • HS_DB_INTERNAL_NAME
  • HS_DB_NAME
  • HS_DESCRIBE_CACHE_HWM
  • HS_FDS_ARRAY_EXEC
  • HS_FDS_CONNECT_INFO
  • HS_FDS_DEFAULT_SCHEMA_NAME
  • HS_FDS_SHAREABLE_NAME
  • HS_FDS_TRACE_LEVEL
  • HS_LANGUAGE
  • HS_LONG_PIECE_TRANSFER_SIZE
  • HS_NLS_DATE_FORMAT
  • HS_NLS_DATE_LANGUAGE
  • HS_NLS_NCHAR
  • HS_NLS_NUMERIC_CHARACTERS
  • HS_NLS_TIMESTAMP_FORMAT
  • HS_NLS_TIMESTAMP_TZ_FORMAT
  • HS_OPEN_CURSORS
  • HS_ROWID_CACHE_SIZE
  • HS_RPC_FETCH_REBLOCKING
  • HS_RPC_FETCH_SIZE
  • HS_TIME_ZONE

设置任何这些参数时，请勿使用PRIVATE关键字。使用PRIVATE关键字可防止将参数上载到服务器，并可能导致SQL处理错误。请勿使用SET命令将这些参数设置为环境变量。
