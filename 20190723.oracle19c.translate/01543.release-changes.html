Oracle Database Release 19c中的更改

以下是Oracle数据库备份和恢复用户指南（适用于Oracle数据库版本19c）中的更改。

  • PDB的恢复目录支持

    当目标数据库是可插拔数据库（PDB）时，您可以连接到恢复目录。

    请参见“ 将目标连接到PDB ” 。

  • 自动删除闪回日志

    超出保留期的闪回日志会自动删除。自动删除闪回日志简化了快速恢复区域的管理，提高了闪回数据库的性能。

    请参阅“ 在快速恢复区中管理闪回日志的空间 ” 。

Oracle Database 12c第2版（12.2.0.1）中的更改

以下是Oracle Database 12c第2版（12.2.0.1）的Oracle数据库备份和恢复用户指南中的更改。

新功能

  • 支持PDB中的还原点

    您可以创建PDB还原点，该点是特定于单个可插拔数据库（PDB）的还原点。为特定PDB创建PDB还原点可确保与多租户容器数据库（CDB）中的其他PDB定义的还原点没有名称冲突。

    看到：

    • 多租户环境中的还原点概述

    • 创建CDB还原点

    • 创建PDB还原点

    • 使用V $ RESTORE_POINT视图列出还原点

  • 闪回数据库支持PDB

    您可以为单个PDB执行闪回数据库操作，而不会影响多租户容器数据库（CDB）中的其他PDB。这使您只能将指定的PDB（而不是整个CDB）倒回到先前的时间点。特定PDB上的闪回数据库操作不会影响CDB中的其他PDB。

    在Oracle Database 12c第1版（12.1）中，要在PDB时间点恢复（PITR）中执行CDB的闪回数据库操作，您必须在PDB中为所有PITR执行脱机的数据文件。此限制现已删除。您可以通过PDB PITR或PDB闪回在CDB上执行闪回数据库操作，并将COMPATIBLE设置为12.2或更高。

    看到：

    • 为CDB和PDB执行闪回数据库的基本概念

    • 为整个CDB执行闪回数据库操作

    • 为PDB执行闪回数据库操作

    • 使用SQL * Plus执行闪回数据库

  • 恢复表和表分区的增强功能

    在执行表恢复之前，RMAN会检查目标主机上是否有足够的空间来创建恢复期间使用的辅助实例。您还可以将表和表分区恢复为与其最初存在的源模式不同的模式。

    看到：

    • RMAN执行以恢复表和表分区的步骤

    • 关于将表和分区恢复到新架构

    • 示例：将表恢复为新架构

  • 跨平台传输的增强功能

    您可以执行PDB跨平台传输到新CDB。加密的表空间可以跨平台传输。您还可以跨平台和网络还原和恢复文件。

    看到：

    • 执行封闭式PDB的跨平台传输

    • 使用不一致的备份执行PDB的跨平台传输

    • 使用备份集将只读表空间传输到不同平台的步骤

    • 将不一致的表空间传输到不同平台的步骤

    • 通过网络执行数据文件的跨平台传输

  • 数据库复制的增强功能

    您可以使用DUPLICATE命令创建Oracle Data Guard远程同步实例。

    请参阅复制整个数据库 。

  • 应用程序容器的备份和恢复支持

    您可以使用RMAN执行应用程序容器的备份，完全恢复和时间点恢复。这包括应用程序根和一个或多个应用程序PDB。

    看到：

    • 备份应用程序容器

    • 执行应用程序容器的完全恢复

    • 执行应用程序PDB的时间点恢复

  • 稀疏数据库的备份和恢复

    RMAN使您可以在COMPATIBLE初始化参数设置为12.2或更高的情况下为稀疏数据库执行备份，恢复和基于备份的复制。您还可以备份和恢复单个稀疏数据文件，表空间，CDB和PDB。

    看到：

    • 关于稀疏备份

    • 使用RMAN备份稀疏数据库

    • 执行稀疏数据库的完全恢复

    • 执行稀疏数据库的时间点恢复

    • 复制稀疏数据库

Oracle Database 12 c第1版（12.1.0.1）中的更改

以下是Oracle Database 12 c第1版（12.1.0.1）的备份和恢复用户指南中的更改。

新功能

以下是此版本中的新功能：

  • 支持多租户容器数据库和可插拔数据库

    RMAN提供多租户容器数据库（CDB）的备份和恢复，这些数据库在Oracle Database 12 c第1版（12.1）中引入。此支持包括指定可插拔数据库（PDB）的备份和时间点恢复。

    看到：

    • “ 备份CDB和PDB ”

    • “ 执行CDB和PDB的时间点恢复 ”

    • “ 执行CDB的崩溃和实例恢复 ”

  • SYSBACKUP权限

    SYSBACKUP管理权限包含备份和恢复所需的权限，包括连接到已关闭数据库的权限。系统管理员可以向执行备份和恢复的用户授予SYSBACKUP而不是SYSDBA ，从而减少SYSDBA权限的扩散。与SYSDBA ， SYSBACKUP不包括数据访问权限，例如SELECT ANY TABLE 。

    看到：

    “ 使用RMAN建立数据库连接 ”

  • 存储快照优化

    通过Storage Snapshot Optimization，您可以使用第三方技术获取数据库的存储快照，而无需将数据库置于BACKUP模式。然后，您可以使用快照来恢复全部或部分数据库。

    看到：

    • “ 使用第三方快照技术进行备份 ”

    • “ 使用存储快照优化进行恢复 ”

  • SQL接口改进

    现在，您可以在RMAN中发出大多数SQL命令，而不必使用带有SQL关键字的命令。对于RMAN和SQL中存在且具有非常不同用途的一些命令，您可以指定SQL关键字以消除歧义。您不再需要将SQL命令括在引号中，这大大简化了SQL命令本身需要引号时的语法。SQL ALTER命令替换RMAN命令。

    新的RMAN DESCRIBE命令提供SQL * Plus DESCRIBE命令的功能。

    看到：

    Oracle数据库备份和恢复参考

  • 多节备份改进

    RMAN为增量备份和映像副本提供多节备份支持。尽可能将未使用的块压缩和块更改跟踪与多节增量备份结合使用。这可以提高备份和还原性能。

    看到：

    • “ 指定多节增量备份 ”

    • “ 使用映像副本进行多节备份 ”

  • 通过网络恢复和恢复文件

    您现在可以使用物理备用数据库中的备份集还原或恢复数据库，数据文件，表空间或控制文件。RMAN通过网络将备份集传输到目标主机。当您要同步备用数据库和主数据库时，这在Data Guard环境中很有用。

    看到：

    “ 通过网络恢复和恢复文件 ”

  • 主动数据库复制改进

    RMAN现在可以使用备份集执行活动数据库复制。当分配了足够的辅助通道时，辅助实例连接到目标实例并通过网络检索备份集，从而减少目标实例上的处理负载。在复制过程中可以使用未使用的块压缩，从而减少通过网络传输的备份的大小。您可以指定要使用的二进制压缩级别。您还可以在执行活动数据库复制时加密备份并使用多节备份。

    看到：

    “ 关于使用RMAN进行活动数据库复制 ”

  • 跨平台备份和还原改进

    您可以使用完整备份集和增量备份集跨平台传输数据。增量备份可以减少跨平台数据迁移期间的整体应用程序停机时间。

    看到：

    跨平台传输数据

  • 从RMAN备份恢复表和表分区

    RMAN可以将表和表分区从先前创建的RMAN备份恢复到指定的时间点。

    看到：

    从RMAN备份恢复表和表分区

  • 统一审计和RMAN

    统一审计将所有Oracle数据库审计记录合并为一个单独的审计跟踪。要使用统一审计，必须先将数据库升级到Oracle Database 12 c第1版（12.1），然后再迁移数据库以使用统一审计。

    看到：

    • 有关迁移数据库以使用统一审计的详细信息，请参见“ Oracle数据库升级指南”

    • 有关如何查找RMAN统一审核信息的详细信息，请参见“ Oracle数据库安全指南”

  • DUPLICATE增强功能

    您可以指定在创建后不能使用RESETLOGS打开重复数据库。如果要更改重复数据库的初始化参数，或者打开重复数据库，可能会在重复数据库中启动与源数据库冲突的服务，则可能不希望打开重复数据库。

    看到：

    “ 指定重复数据库的状态 ”
