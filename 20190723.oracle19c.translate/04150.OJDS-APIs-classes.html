12.4.1 oracle.aurora.jndi.ojds。OjdsClientContext

该类实现了javax.naming.directory.DirContext接口。它建立与数据库的连接，并执行支持OJDS所需的所有功能。它支持[DirContext]描述的所有方法，但以下方法除外：

  • 的getSchema

  • getSchemaClassDefintion

  • 的modifyAttributes

  • 搜索

在基于JAVA JDK的客户端上创建InitialDirContext时，将自动创建此类。它提供客户端应用程序和OJDS之间的通信和对象传输。

必须将以下JNDI属性设置为特定值才能完成连接：

  • java.naming.factory.initial到oracle.aurora.jndi.ojds.OjdsIntialContextFactory

  • java.naming.security.principal到连接模式的名称

  • java.naming.security.credentials到架构密码

  • java.naming.provider.url到有效的OJDS URL

您可以设置这些属性，如以下代码段所示：

Hashtable env = new Hashtable（）; env.put（“java.naming.factory.initial”，“oracle.aurora.jndi.ojds。OjdsInitialContextFactory“）; env.put（Context。SECURITY_PRINCIPAL，“HR”）; env.put（上下文。SECURITY_CREDENTIALS，“ <密码> ”）; env.put（上下文。PROVIDER_URL， “ojds：//薄：本地主机：5521：J3”）;

12.4.2 oracle.aurora.jndi.ojds。OjdsServerContext

该类实现了javax.naming.directory.DirContext接口。它使用内部数据库连接与OJDS持久性存储进行通信。它支持[DirContext]描述的所有方法，但以下方法除外：

  • 的getSchema

  • getSchemaClassDefintion

  • 的modifyAttributes

  • 搜索

在数据库驻留应用程序中创建InitialDirContext时，将自动创建此类。它使用数据库内部JDBC驱动程序与OJDS持久性存储进行通信。

对于这四个环境属性OjdsClientContext是被忽略的OjdsServerContext因为应用程序运行作为login模式。使用kprb [JDBC]内部驱动程序建立连接。如果Java存储过程需要在服务器外部进行访问，则必须使用OJDS URLContext作为java.naming.provider.url属性的值。

12.4.3 oracle.aurora.jndi.ojds。OjdsInitialContextFactory

该类实现了javax.naming.spi.InitialContextFactory接口。JNDI InitialContext或InitialDriContext类根据执行环境创建OjdsClientContext或OjdsServerContext 。

12.4.4 oracle.aurora.jndi.ojds。OjdsURLContextFactory

该类支持OJDS样式的URL。根据提供给URL的方法，此方法可以返回DirContext或存储在OJDS中的对象的实例。

12.4.5 oracle.aurora.jndi.ojds。OjdsURLContext

该类是oracle.aurora.jndi.ojds.的扩展oracle.aurora.jndi.ojds.OjdsClientContext。 它支持从OJDS URL提取连接信息并建立与OJDS的连接。它支持与oracle.aurora.jndi.ojds.相同的接口oracle.aurora.jndi.ojds.OjdsClientContext类。

您必须设置以下参数才能使用OjdsURLContext类：

  • javax.naming.security.principal到连接模式

  • javax.naming.security.credentials到连接模式的密码

  • javax.naming.factory.initial到oracle.aurora.jndi.ojds.OjdsInitialContextFactory

您可以设置这些属性，如以下代码段所示：

Hashtable env = new Hashtable（）; env.put（“java.naming.factory.initial”，“oracle.aurora.jndi.ojds。OjdsInitialContextFactory“）; env.put（Context。SECURITY_PRINCIPAL，“HR”）; env.put（上下文。SECURITY_CREDENTIALS，“ <密码> ”）; DirContext dir =（new InitialContext（env））。lookup（“ojds：// thin：localhost：5521：j3 / public ./mydir”）;
