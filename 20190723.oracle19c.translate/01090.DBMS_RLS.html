140 DBMS_RLS

DBMS_RLS包中包含细粒度的访问控制管理界面，用于实现虚拟专用数据库（VPD）。

DBMS_RLS仅适用于Enterprise Edition。

140.1 DBMS_RLS概述

支持细粒度访问控制的功能基于动态谓词，其中安全规则未嵌入到视图中，但是在DML语句中引用基表或视图时，在语句分析时获取。

表/视图或同义词的动态谓词由PL / SQL函数生成，该函数通过PL / SQL接口与安全策略相关联。例如：

DBMS_RLS.ADD_POLICY（'hr'，'employees'，'emp_policy'，'hr'，'emp_sec'，'select'，'user_ctx'，'time'）;

每当在HR模式下的EMPLOYEES表在查询或子查询（ SELECT ）中引用时，服务器就会调用EMP_SEC函数（在HR模式下）。此函数返回特定于EMP_POLICY策略的当前模式的EMP_POLICY 。策略函数可以基于函数调用期间可用的会话环境变量生成谓词。这些变量通常以应用程序上下文的形式出现。策略可以指定安全相关列和这些语句类型的任意组合： INDEX ， SELECT ， INSERT ， UPDATE或DELETE 。

然后，服务器生成一个包含文本的瞬态视图：

SELECT * FROM hr.employees WHERE P1

这里， P1 （例如， SAL > 10000，甚至是子查询）是从EMP_SEC函数返回的谓词。服务器将EMPLOYEES表视为视图，并像普通视图一样进行视图扩展，但视图文本是从瞬态视图而不是数据字典中获取的。

如果谓词包含子查询，则策略函数的所有者（定义者）用于解析子查询中的对象并检查这些对象的安全性。换句话说，对受策略保护的对象具有访问权限的用户不需要了解有关策略的任何信息。不需要为任何基础安全策略授予对象权限。此外，用户不需要对策略函数EXECUTE特权，因为服务器使用函数definer的权限进行调用。

注意：

瞬态视图可以保留父对象的可更新性，因为它是从仅具有谓词的单个表或视图派生的;也就是说，没有JOIN ， ORDER BY ， GROUP BY等等。

DBMS_RLS还提供了删除或启用安全策略的接口。例如，您可以使用以下PL / SQL语句删除或启用EMP_POLICY ：

DBMS_RLS.DROP_POLICY（'hr'，'employees'，'emp_policy'）; DBMS_RLS.ENABLE_POLICY（'hr'，'employees'，'emp_policy'，TRUE）;

140.2 DBMS_RLS安全模型

使用子查询创建瞬态视图时，将执行安全检查。拥有策略函数的模式（生成动态谓词）是瞬态视图定义的安全检查和对象查找。

140.4 DBMS_RLS操作说明

DBMS_RLS过程会导致当前DML事务（如果有）在操作之前提交。但是，如果这些过程位于DDL事件触发器内，则不会首先导致提交。对于DDL事务， DBMS_RLS过程是DDL事务的一部分。

例如，您可以为CREATE TABLE创建一个触发器。在触发器内，您可以通过ALTER TABLE添加列，并且可以通过DBMS_RLS添加策略。所有这些操作都与CREATE TABLE处于同一事务中，即使每个操作都是DDL语句。仅当触发器成功完成时， CREATE TABLE才会成功。

可以从中获取当前游标和相应谓词的视图 V$VPD_POLICIES 。

同义词只能引用视图或表。

140.5 DBMS_RLS规则和限制

VPD支持使用长标识符。适用于对象（表名，策略名和子程序名）和视图的参数（如object_schema ， object_name和policy_name的最大长度为128个字节。

作为第二个示例，以下命令创建另一个适用于托管的同一对象的策略，因此用户只能根据其订户ID访问数据。由于它被定义为SHARED_STATIC策略类型，因此服务器将首先尝试在SGA缓存中查找谓词。如果搜索失败，服务器将仅调用策略函数subfun 。 

BEGIN DBMS_RLS.ADD_POLICY（object_schema =>'hr'，object_name =>'employee'，policy_name =>'hosting_policy'，function_schema =>'hr'，policy_function =>'subfun'，policy_type => dbms_rls。SHARED_STATIC）;结束;
