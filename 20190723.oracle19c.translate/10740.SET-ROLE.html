设置角色

目的

当用户登录到Oracle数据库时，数据库将启用显式授予用户的所有权限以及用户默认角色中的所有权限。在会话期间，用户或应用程序可以多次使用SET ROLE语句来启用或禁用当前为会话启用的角色。

您不能一次启用超过148个用户定义的角色。

注意：

  • 对于大多数角色，除非直接或通过其他角色授予您角色，否则无法启用或禁用角色。但是，可以通过其关联的PL / SQL包授予和启用安全应用程序角色。有关安全应用程序角色的信息，请参阅USING包的CREATE ROLE语义和Oracle数据库安全指南 。

  • 只有在调用堆栈上没有定义者的权限单位时， SET ROLE才会成功。如果调用堆栈上至少有一个DR单元，则发出SET ROLE命令会导致ORA - 06565 。有关定义者权限单元的更多信息，请参见Oracle数据库PL / SQL语言参考 。

  • 要从PL / SQL运行SET ROLE命令，必须使用动态SQL，最好是EXECUTE IMMEDIATE语句。有关此语句的更多信息，请参见Oracle Database PL / SQL语言参考 。

您可以通过检查SESSION_ROLES数据字典视图来查看当前启用的角色。

也可以看看：

  • CREATE ROLE创建角色的信息

  • 有关更改用户默认角色的信息，请输入ALTER

  • 有关SESSION_ROLES会话参数的信息，请参见Oracle数据库参考

先决条件

您必须已经被授予您在SET ROLE语句中命名的角色。

句法

set_role :: =


插图描述set_role.eps

语义

角色

指定要为当前会话启用的一个或多个角色。未指定的所有角色将在当前会话中禁用，或者在当前会话中发出另一个SET ROLE语句。

在IDENTIFIED BY password子句中，指定角色的密码。如果角色具有密码，则必须指定密码才能启用该角色。

设置角色的限制

您无法指定全局标识的角色。登录时默认启用全局角色，以后无法重新启用。

所有条款

指定ALL以启用为当前会话授予您的所有角色，但EXCEPT子句中可选择的那些角色除外。

EXCEPT子句中列出的角色必须是直接授予您的角色。它们不能是通过其他角色授予您的角色。

如果您在EXCEPT子句中列出了直接和通过其他角色授予您的角色，则该角色将凭借其已被授予的角色保持启用状态。

对ALL条款的限制

以下限制适用于ALL子句：

  • 您不能使用此子句来启用具有直接授予您的密码的角色。

  • 您不能使用此子句来启用安全应用程序角色，该角色只能由使用授权包的应用程序启用。有关创建安全应用程序角色的信息，请参阅“ Oracle数据库安全指南” ;有关教程 ，请参阅“ Oracle数据库2天+安全指南” 。

没有

指定NONE以禁用当前会话的所有角色，包括DEFAULT角色。

例子

设置角色：示例

要为当前会话启用密码warehouse标识的角色dw_manager ，请发出以下语句：

SET ROLE dw_manager由仓库识别;

要启用为当前会话授予您的所有角色，请发出以下语句：

SET ROLE ALL;

要启用除dw_manager之外授予您的所有角色，请发出以下语句：

SET ROLE ALL除了dw_manager;

要禁用为当前会话授予您的所有角色，请发出以下语句：

没有设定;
