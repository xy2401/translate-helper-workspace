3.12跑

目的

使用RUN命令将系列RMAN命令分组到要按顺序执行的块中。在读取RUN块的右括号时，RMAN将作业命令列表编译为一个或多个作业步骤，然后立即执行这些步骤。

先决条件

仅在RMAN提示符下执行此命令。您必须在job命令列表的前面加上一个左括号（ { ），并用一个右括号（ } ）终止它。

使用说明

您可以使用RUN创建一个范围，在该范围内脚本可以覆盖默认配置。例如，您可以使用SET命令使用ALLOCATE CHANNEL和RELEASE CHANNEL命令以及其他参数覆盖已组态的通道（如示例3-54所示）。执行在列出的命令后RUN块，所述内分配通道RUN块被释放并且设置返回到它们的值。

如例3-55所示，您必须在RUN块中使用EXECUTE SCRIPT命令。

例子

示例3-54覆盖已配置的设置

假设您配置的设备配置如下：

RMAN> SHOW DEVICE TYPE;具有db_unique_name PROD1的数据库的RMAN配置参数是：配置设备类型磁盘并行1备份类型到备份; #default CONFIGURE DEVICE TYPE SBT_TAPE PARALLELISM 1 BACKUP TYPE TO BACKUPSET; #default

您想要备份到非默认目录。您可以在作业中覆盖它，而不是更改配置，如下所示：

RUN {ALLOCATE CHANNEL c1 DEVICE TYPE DISK FORMAT“/ disk2 /％U”; BACKUP DATABASE PLUS ARCHIVELOG; }

示例3-55执行RMAN脚本

假设您使用CREATE SCRIPT命令创建名为backup_db的备份脚本。此示例执行存储的脚本：

RUN {EXECUTE SCRIPT backup_db; }
