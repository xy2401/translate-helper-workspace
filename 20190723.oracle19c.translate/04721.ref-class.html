13.25参考级

CRE编程语言中SQL REF值的映射，它是对数据库中SQL结构类型值的引用。

每个REF值都具有其引用的对象的唯一标识符。可以使用SQL REF值代替它引用的SQL结构类型;它可以用作表中的列值或结构化类型中的属性值。

因为SQL REF值是指向SQL结构类型的逻辑指针，所以Ref对象默认也是逻辑指针;因此，将SQL REF值检索为Ref对象不会实现客户端上结构化类型的属性。

NULL Ref对象上唯一有效的方法是isNull（）和operator =（） 。

Ref对象可以保存到持久存储中，并通过operator *（） ， operator - >（）或ptr（）方法取消引用。T必须是从PObject派生的类。在以下部分中， T*和PObject *可互换使用。

表13-35 Ref Class使用的枚举值

  • OCCI_LOCK_NONE清除Ref对象上的锁定设置。

  • OCCI_LOCK_X表示应该锁定对象，并且如果对象被另一个会话锁定，则等待锁可用。

  • OCCI_LOCK_X_NOWAIT表示该对象应该被锁定，如果被另一个会话锁定，则返回错误。

  • OCCI_MAX_PREFETCH_DEPTH表示应该以最大深度进行提取。

13.25.15 setPrefetch（）

设置复杂对象检索的预取选项。此方法指定应该预取从此对象通过Ref （传递闭包）可到达的所有对象的深度。如果仅预取选定的属性类型，则必须使用该方法的第一个版本。此方法指定应该遵循它所引用的对象的哪些Ref属性来预取对象（复杂对象检索）以及应该遵循这些链接的深度级别。

句法 描述 
void setPrefetch（const string＆typeName，unsigned int depth）;

使用类型名称和深度设置复杂对象检索的预取选项。

 
void setPrefetch（unsigned int depth）;

仅使用深度设置复杂对象检索的预取选项。

 
void setPrefetch（const string＆schName，const string＆typeName，unsigned int depth）;

使用模式，类型名称和深度设置复杂对象检索的预取选项。

 
void setPrefetch（const UString＆schName，const UString＆typeName，unsigned int depth）;

使用模式，类型名称和深度以及UString支持设置复杂对象检索的预取选项。

参数 描述 
的typeName

要预取的Ref属性的类型。

 
schName

要预取的Ref属性的模式名称。

 
深度

应遵循链接的深度级别;可以使用表13-35中定义的PrefetchOption 。
