219.2.1 GET_CLIENT_HOSTNAME函数

此函数返回客户端的主机名。

句法

OWA_SEC.GET_CLIENT_HOSTNAME RETURN VARCHAR2;

返回值

主机名。

219.2.2 GET_CLIENT_IP函数

此函数返回客户端的IP地址。

句法

OWA_SEC.GET_CLIENT_IP返回OWA_UTIL.IP_ADDRESS;

返回值

IP地址。owa_util.ip_address datatype是PL / SQL表，其中前四个元素包含IP地址的四个数字。例如，如果IP地址为123.45.67.89且变量ipaddr为owa_util.ip_address数据类型，则该变量将包含以下值：

ipaddr（1）= 123 ipaddr（2）= 45 ipaddr（3）= 67 ipaddr（4）= 89

219.2.3 GET_PASSWORD函数

此函数返回用户用于登录的密码。

句法

OWA_SEC.GET_PASSWORD RETURN VARCHAR2;

返回值

密码。

使用说明

出于安全原因，仅当使用自定义身份验证时，此函数才返回true值。如果在不使用自定义身份验证时调用此函数，则该函数将返回未定义的值。因此，不公开数据库密码。

219.2.4 GET_USER_ID函数

此函数返回用户用于登录的用户名。

句法

OWA_SEC.GET_USER_ID RETURN VARCHAR2;

返回值

用户名。

219.2.5 SET_AUTHORIZATION过程

在OWA_CUSTOM包的初始化部分中调用的此过程设置PL / SQL网关的授权方案。

这实现了您的authorize函数，该函数在运行请求的过程之前授权用户。authorize功能的位置取决于您选择的方案。

句法

OWA_SEC.SET_AUTHORIZATION（INT INTEGER中的方案）;

参数

表219-2 SET_AUTHORIZATION过程参数

参数 描述

方案

授权方案。它是SET_AUTHORIZATION的以下方案SET_AUTHORIZATION ：

  • OWA_SEC.NO_CHECK - 指定PL / SQL应用程序不进行任何自定义身份验证。这是默认值。

  • OWA_SEC.GLOBAL - 定义为所有用户和所有过程调用的授权函数。这是OWA_CUSTOM 。 AUTHORIZE “ sys ”模式中的函数 。

  • OWA_SEC.PER_PACKAGE - 定义在调用包或匿名过程中的过程时调用的authorize函数。如果程序在包中，那么包。调用用户模式中的AUTHORIZE函数以授权用户。如果过程不在包中，则调用用户模式中的匿名authorize功能。

  • OWA_SEC.CUSTOM - 为每个用户实现不同的授权功能。功能OWA_CUSTOM 。 AUTHORIZE调用用户模式中的函数以授权用户。如果用户的架构不包含OWA_CUSTOM .AUTHORIZE函数，则PL / SQL Gateway会在“sys”架构中查找它。

    自定义authorize功能具有以下签名：

    功能授权返回布尔;

如果函数返回TRUE ，则验证成功。如果返回FALSE ，则验证失败。如果未定义authorize功能，则网关返回错误并失败。
