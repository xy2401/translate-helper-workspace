CHR

句法


插图描述chr.eps

目的

CHR在数据库字符集中返回具有等效于n的二进制值的字符作为VARCHAR2值，或者，如果指定USING NCHAR_CS ，则返回国家字符集。

对于单字节字符集，如果n > 256，则Oracle Database返回n mod 256的二进制等价物。对于多字节字符集， n必须解析为整个代码点。无效的代码点无效，指定无效代码点的结果是不确定的。

此函数将NUMBER值或任何可以隐式转换为NUMBER值作为参数，并返回一个字符。

注意：

使用CHR函数（带或不带可选的USING NCHAR_CS子句）会导致代码在基于ASCII和EBCDIC的机器体系结构之间无法移植。

例子

以下示例在基于ASCII的计算机上运行，数据库字符集定义为WE8ISO8859P1：

选择CHR（67）|| CHR（65）|| CHR（84）“Dog”FROM DUAL;狗猫

要在具有WE8EBCDIC1047字符集的基于EBCDIC的机器上生成相同的结果，必须按如下方式修改前面的示例：

选择CHR（195）|| CHR（193）|| CHR（227）“Dog”FROM DUAL;狗猫

对于多字节字符集，这种串联会产生不同的结果。例如，给定一个多字节字符，其十六进制值为a1a2 （ a1表示第一个字节， a2表示第二个字节），必须为n指定“ a1a2 ”或41378的十进制等效值：

SELECT CHR（41378）FROM DUAL;

您不能指定与a2的十进制等效值连接的a1的十进制等效值，如下例所示：

SELECT CHR（161）|| CHR（162）FROM DUAL;

但是，您可以连接整个多字节代码点，如下例所示，它连接十六进制值为a1a2和a1a3的多字节字符：

SELECT CHR（41378）|| CHR（41379）FROM DUAL;
