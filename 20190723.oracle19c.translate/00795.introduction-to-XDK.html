1 Oracle XML Developer's Kit简介

引入了Oracle XML Developer's Kit（XDK）。

XDK概述

Oracle XML Developer's Kit（XDK）是一组通用的组件，使您能够构建和部署处理可扩展标记语言（XML）的C，C ++和Java软件程序。您可以将这些组件组装到满足业务需求的XML应用程序中。

注意：

如果您正在将XDK与PL / SQL一起使用并从Oracle Database 8.1或9.2迁移，Oracle强烈建议您使用数据库字符集AL32UTF8。否则，当PL / SQL处理包含转义实体的XML数据时，可能会出现问题。

Oracle XML Developer's Kit（XDK）支持Oracle XML DB ，这是一组用于在Oracle数据库中存储和处理XML的技术。您可以将XDK与Oracle XML DB一起使用来构建在Oracle数据库中运行的应用程序。您也可以独立于Oracle XML DB使用XDK。

生成的XML中的日期和时间戳采用XML Schema指定的格式。请参阅Oracle XML DB Developer's Guide 。

XDK完全受Oracle支持，并附带商业再发行许可证。Oracle数据库的标准安装包括XDK。

XML解析器

XML解析器读取XML文档并确定数据的结构和属性。它将数据分成几部分并将它们提供给其他XDK组件。

XML解析器可以使用这些API以编程方式访问已解析的XML数据：

  • SAX

    使用SAX API按元素串行访问数据元素。您可以使用SAX解析器注册事件处理程序，并在遇到某些事件时调用回调方法。

  • DOM

    使用DOM API将XML文档表示为内存中的树并对其进行操作或导航。

XDK包括用于Java，C和C ++的XML解析器。每个解析器都包含对DOM和SAX API的支持。

Java的XML解析器支持用于XML处理的Java API 1.2版（JAXP），这是一种标准API，可以独立于处理器实现使用DOM，SAX，XML Schema和XSLT。因此，您可以在不影响程序的情况下更改XML处理器的实现。

XML压缩器已集成到Java的XML解析器中。它使用DOM和SAX接口提供元素级XML压缩和解压缩。XML压缩器压缩XML文档而不会丢失DOM树的结构和层次信息。解析XML文档后，可以使用DOM或SAX将其序列化为二进制流，然后再重新构建。

您可以使用XML压缩器来减少XML消息有效负载的大小，从而提高吞吐量。在应用程序中用作内部XML文档访问时，它可以显着减少内存使用，同时保持快速访问。

XSLT处理器

XSLT是一种样式表语言，使处理器能够将一个XML文档转换为另一个XML文档。XSLT文档是一个样式表，其中包含管理此类转换的模板规则。XDK支持在任何操作系统上对数据库内外的XML数据进行XSLT转换。

每个Oracle XML解析器都包含一个集成的XSLT处理器，用于使用XSLT样式表转换XML数据。使用XSLT处理器，您可以将XML文档转换为XML，可扩展超文本标记语言（XHTML）或几乎任何其他文本格式。

XML模式处理器

由W3C创建的XML Schema语言描述了XML本身的XML文档的内容和结构。

XML模式包含定义XML应用程序有效性的规则 - 这是它相对于DTD的主要优势。

XML模式指定一组内置数据类型（例如字符串，浮点数和日期）。用户可以从内置数据类型派生自己的数据类型。例如，架构可以将日期限制为2000年之后的日期，或指定合法值列表。

XDK包括用于Java，C和C ++的XML Schema处理器。

XML类生成器

XML类生成器将解析的XML模式或DTD作为输入，并生成Java或C ++源类文件作为输出。XDK包括Java Architecture for XML Binding（JAXB）类生成器和C ++类生成器。

JAXB是一个Java API和一组工具，它们将XML数据映射到Java对象，反之亦然。因为JAXB以Java格式向Java程序提供XML文档，所以您可以编写处理XML数据的Java程序，而无需使用SAX解析器或编写回调方法。每个Java对象都派生自输入XML文档中的模式组件的实例。JAXB不直接支持DTD，但您可以将DTD转换为JAXB可以使用的XML架构。用于C ++的XML类生成器直接支持DTD和XML模式。

作为如何使用JAXB的示例，您可以编写一个Java程序，该程序使用生成的Java类逐步构建XML文档。假设您编写了一个XML模式供人力资源部门和Java程序使用，该Java模式响应更改其个人数据的用户。该程序可以使用JAXB以零散的方式构建XML确认文档，XSLT处理器可以将其转换为XHTML并传送到浏览器。

XML管道处理器

XML Pipeline Definition Language是一个XML词汇表，用于描述XML资源之间的处理关系。Oracle XML Pipeline处理器符合XML Pipeline Definition Language 1.0标准。

XML Pipeline处理器将XML管道文档（定义进程之间的关系）作为输入，并根据派生的依赖项执行管道进程。例如，输入文档可以指定程序必须首先验证输入XML文档，然后，如果它是有效的，则转换它。

XML管道处理器通过用简单的声明性XML语法替换自定义Java代码来帮助Java开发人员构建XML处理应用程序。

Oracle XML SQL实用程序（XSU）是一组Java类库，可用于将SQL查询的结果呈现为规范XML或将XML文档中的数据加载到现有数据库模式或视图中。

您可以使用XSU执行以下任务：

  • 自动并动态地将任意SQL查询的结果呈现为规范XML。

    XSU支持对结构丰富，用户定义的对象类型和对象视图（包括XMLType 。XSU将关系数据转换为XML，如下所示：

    • 列成为顶级元素。

    • 标量值成为具有纯文本内容的元素。

    • 对象类型成为具有显示为子元素的属性的元素。

    • 集合映射到元素列表。

  • 将XML文档中的数据加载到现有数据库架构或视图中。

TransX实用程序概述

Oracle TransX实用程序使您可以使用多语言XML数据填充数据库。该实用程序使用对开发人员和翻译人员而言直观的数据格式，并使用比以前的技术更不容易出错的验证功能。

TransX实用程序是XSU的一个应用程序，它将已转换的种子数据和消息加载到数据库模式中。为了使用多种语言的数据填充数据库，TransX实用程序提供了您必须使用XSU开发的功能。

Oracle Data Provider for .NET

Oracle Data Provider for .NET（ODP.NET）是Oracle数据库的.NET数据提供程序的实现。它使用Oracle本机API从任何.NET应用程序提供对Oracle数据和功能的快速，可靠的访问。它使用并继承Microsoft .NET Framework类库中的类和接口。

您可以使用ODP.NET和XDK从关系和对象关系表和视图中提取数据作为XML文档。您还可以在数据库服务器上使用XML文档进行插入，更新和删除操作。ODP.NET通过Oracle XML DB在数据库中本机支持XML。

ODP.NET支持具有以下功能的XML：

  • 将XML数据本机存储在数据库服务器中作为Oracle本机类型XMLType 。

关于安装XDK

Oracle数据库的标准安装包括XDK（其所有组件）。

警告：

使用Oracle XML Developer Kit（XDK）的组件构建软件程序可以实现一些功能强大但具有潜在危险的功能，例如外部实体扩展和递归扩展。有关如何安全使用XDK的信息，请参阅Oracle XML Developer's Kit的安全注意事项 。

本节假定：

  • 您从从Oracle技术网（OTN）下载的CD-ROM或存档中安装了Oracle数据库。

    Oracle数据库CD-ROM默认安装XDK。

  • 您从Oracle数据库示例介质安装了XDK演示程序。

示例1-1显示了安装Oracle数据库和XDK演示程序后Oracle数据库主目录的外观。

包含XDK的目录称为XDK home 。将环境变量$XDK_HOME （UNIX）或%XDK_HOME% （Windows）的值设置为Oracle主目录中的xdk目录。例如，您可以使用UNIX上的csh使用以下命令设置XDK主目录：

setenv XDK_HOME $ ORACLE_HOME / xdk
