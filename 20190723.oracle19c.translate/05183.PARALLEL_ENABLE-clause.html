13.47 PARALLEL_ENABLE条款

启用并行执行功能，使其可以安全地用于并行DML评估的从属会话。

表示该函数可以从并行查询操作的并行执行服务器运行。

PARALLEL_ENABLE子句可以出现在以下SQL语句中：

  • 创建功能声明

  • CREATE PACKAGE声明

  • CREATE TYPE BODY声明

语义

parallel_enable_clause

parallel_enable_clause只能在函数中出现一次。

该函数不能使用会话状态，例如包变量，因为这些变量不一定在并行执行服务器之间共享。

仅将可选的PARTITION argument BY子句与具有REF CURSOR数据类型的函数一起使用。此子句允许您从REF CURSOR参数定义函数输入的分区。当函数用作查询的FROM子句中的表函数时，对函数的输入进行分区会影响查询的并行化方式。

任何

表示可以在并行执行服务器之间随机分区数据

注意：

您可以仅使用ANY将弱游标变量参数分区为表函数，而不是使用RANGE ， HASH或VALUE 。

范围或哈希

将数据分区到由函数的REF CURSOR参数返回的指定列中。

streaming_clause

可选的streaming_clause允许您对并行处理进行排序或聚类。

订购| CLUSTER BY

ORDER BY或CLUSTER BY表示并行执行服务器上的行必须是本地排序的，并且具有与column列表指定的键值相同的键值。

值

指定直接键分区，用于执行MapReduce工作负载时使用的表函数。该column的数据类型必须为NUMBER 。 VALUE在可用的reducer上统一分配行处理。

如果列的减速器数量多于可用的减速器数量，则PL / SQL使用模数运算将列中的减速器数量映射到正确的范围。

在计算处理相应行的reducer数时，PL / SQL将负值视为零，并将正小数值舍入为最接近的整数。

EXPR

expr标识指定了分区的表函数的REF CURSOR参数名称，并在其列中指定并行查询执行中每个从属的排序或集群。

对parallel_enable_clause的限制

您不能为嵌套函数指定parallel_enable_clause 。
