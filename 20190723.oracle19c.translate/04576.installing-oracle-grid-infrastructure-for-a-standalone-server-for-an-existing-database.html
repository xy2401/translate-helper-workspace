为现有数据库的独立服务器安装Oracle Grid Infrastructure

按照本节中的高级说明为独立服务器安装Oracle Grid Infrastructure，并为现有Oracle数据库配置它。

Oracle Restart可以管理来自同一版本的资源，并且最多可以发布一个低于Oracle Restart的版本。例如，您可以为独立服务器19c（Oracle Restart）安装Oracle Grid Infrastructure，以便为Oracle Database 19c和Oracle Database 18c提供服务。早期的Oracle数据库版本可以在同一台服务器上共存，而无需由Oracle Restart管理。
要为已安装的数据库的独立服务器安装Oracle Grid Infrastructure，请执行以下操作：

  1. 在与数据库相同的主机上，为独立服务器安装Oracle Grid Infrastructure，然后选择为独立服务器配置Oracle Grid Infrastructure（Oracle Restart）作为安装选项。请参阅“ Oracle数据库安装指南 ”中的“使用新数据库安装为独立服务器安装Oracle Grid Infrastructure ” 。

    用于独立服务器组件的Oracle Grid Infrastructure安装在Oracle Grid Infrastructure Oracle主目录（Grid home）中，该主目录位于与现有Oracle数据库主目录不同的位置。

  2. 如果您有现有的Oracle数据库，请使用srvctl命令将其注册为Oracle Restart的高可用性：

    $ cd $ ORACLE_HOME / bin $ srvctl add database -db dbname -o oracle_home_path -dbtype SINGLE
