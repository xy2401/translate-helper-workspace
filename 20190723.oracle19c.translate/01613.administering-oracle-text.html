17.1 Oracle Text用户和角色

虽然任何用户都可以创建Oracle Text索引并输入CONTAINS查询，但Oracle Text为管理提供了CTXSYS用户，为应用程序开发人员提供了CTXAPP角色。

本节包含以下部分：

  • CTXSYS用户

  • CTXAPP角色

  • 向用户授予角色和权限

17.1.1 CTXSYS用户

CTXSYS用户在安装期间创建，可以：

  • 查看所有索引

  • 同步所有索引

  • 运行知识库扩展编译器ctxkbtc,

  • 查询所有系统定义的视图

  • 使用CTXAPP角色执行用户的所有任务

注意：

在早期版本的Oracle Text中， CTXSYS具有SYSDBA权限，只有CTXSYS用户可以执行某些功能，例如修改系统定义的首选项或设置系统参数。

从Oracle Database Release 19c开始， CTXSYS用户是仅模式用户。要使用CTXSYS模式，请运行以下语句：

connect / as sysdba; alter session set CURRENT_SCHEMA = CTXSYS;

17.1.2 CTXAPP角色

CTXAPP角色是系统定义的角色，使用户能够：

  • 创建和删除Oracle Text首选项

  • 使用Oracle Text PL / SQL包

17.1.3向用户授予角色和权限

系统使用标准SQL模型向用户授予角色。要向用户授予Oracle Text角色，请使用GRANT语句。

此外，为了允许应用程序开发人员调用Oracle Text PL / SQL包中的过程，您必须向每个用户显式授予Oracle Text包的EXECUTE特权。

17.2 DML队列

当您对基表中的文档进行插入，更新或删除时，数据操作语言（DML）队列存储对等待编制索引的文档的请求。将索引与CTX_DDL同步时。 SYNC_INDEX,请求将从此队列中删除。

您可以使用CTX_PENDING和CTX_USER_PENDING视图查询挂起的插入，更新和删除操作。

您可以使用CTX_INDEX_ERRORS或CTX_USER_INDEX_ERRORS视图查询插入，更新和删除错误。

17.4 CTX_REPORT包

使用CTX_REPORT包生成索引和查询的报告。这些报告可以帮助您微调或排除应用程序故障。

也可以看看：

有关此程序包的更多信息，请参阅Oracle Text Reference

CTX_REPORT包中包含以下过程：

CTX_REPORT.DESCRIBE_INDEX和CTX_REPORT.DESCRIBE_POLICY

这些过程创建描述现有索引或策略的报告，包括索引元数据的设置，索引对象，对象属性的设置，以及（对于CTX_REPORT.DESCRIBE_INDEX ）索引分区信息（如果有）。这些过程对于诊断与索引相关的问题特别有用。

CTX_REPORT.CREATE_INDEX_SCRIPT和CTX_REPORT.CREATE_POLICY_SCRIPT

CREATE_INDEX_SCRIPT创建一个SQL * Plus脚本，该脚本可以创建给定Oracle Text索引的副本。如果您有索引但是没有用于创建此索引的原始脚本（如果有），并且您希望能够重新创建索引，请使用此选项。例如，如果您不小心删除了脚本， CREATE_INDEX_SCRIPT可以重新创建它。同样，如果您具有来自其他用户的继承索引但不是创建它们的脚本，则CREATE_INDEX_SCRIPT可能很有用。

CREATE_POLICY_SCRIPT与CREATE_POLICY_SCRIPT作用相同CREATE_INDEX_SCRIPT,只是它使您能够重新创建策略而不是索引。

CTX_REPORT.INDEX_STATS

INDEX_STATS生成有关索引的各种计算统计信息，例如索引的文档数，索引中的唯一标记数，标记的平均大小以及索引的碎片信息。优化停止列表是INDEX_STATS.的一个使用INDEX_STATS.

CTX_REPORT.QUERY_LOG_SUMMARY

此过程将创建已记录查询的报告，您可以使用该报告执行简单分析。通过查询分析，您可以找到：

  • 进行了哪些查询

  • 哪些查询成功

  • 哪些查询不成功

  • 每个查询的次数

您可以通过各种方式组合这些因素，例如确定应用程序发出的50个最常见的不成功查询。

CTX_REPORT.TOKEN_INFO

TOKEN_INFO可帮助您诊断查询问题。例如，使用它来检查索引数据是否已损坏，并找出哪些文档产生了意外或不良令牌。

CTX_REPORT.TOKEN_TYPE

TOKEN_TYPE是一个查找函数，主要用作其他函数的输入（ CTX_DDL.OPTIMIZE_INDEX ， CTX_REPORT.TOKEN_INFO等）。

17.5 Oracle Enterprise Manager中的文本管理器

Oracle Enterprise Manager提供了Text Manager，用于配置，维护和管理Oracle Text索引。使用Text Manager，您可以执行Oracle Text索引的所有基本配置和管理任务。您可以监视单个Oracle数据库实例或Oracle Real Application Clusters环境的Oracle Text索引的总体运行状况。文本管理器提供关键信息摘要，使您可以深入了解所需的详细程度，解决问题并了解您需要采取的任何操作。

“文本索引”页面显示正在进行的作业，在过去七天内计划的作业或遇到问题的作业。在此页面中，您可以转到Job Scheduler以查看此数据库实例的所有作业的摘要以及管理所选作业。Oracle Enterprise Manager中的联机帮助提供了使用每个文本管理器功能的详细信息和过程。

本节包含以下部分：

  • 使用文本管理器

  • 查看Oracle Text Index的常规信息

  • 检查Oracle Text Index Health

注意：

您无法使用文本管理器创建Oracle Text索引。使用CREATE INDEX语句中所述创建Oracle文本索引与Oracle全文索引下创建Oracle文本索引 。

17.5.1使用文本管理器

在“文本管理器”主页面上，您可以从“操作”列表中对所选索引执行以下操作：

  • 同步

  • 优化

  • 重建

  • 恢复失败的操作

  • 显示日志

  • 显示错误

您还可以为指定的索引计划作业。

要访问文本管理器：

  1. 使用有权访问Cloud Control的用户帐户登录数据库。例如，使用SYS或SYSTEM以及在数据库安装期间指定的密码。
  2. 在“ 数据库主页”上，单击“ 架构”选项卡。
  3. 在“ 文本管理器”组中，选择“ 文本索引”。

    “ 文本索引”页面显示此数据库实例的Oracle Text索引列表。

从“文本索引”页面选择Oracle Text索引时，编辑和操作选项可用于该索引。例如，要配置搜索属性，请单击所选索引的“ 编辑 ”。在“编辑文本索引”页面上，可以设置“外卡最大术语”，“模糊分数”和“模糊扩展数”等属性。您还可以更改索引和分区名称，并指定URL_DATASTORE.设置URL_DATASTORE.

17.5.2查看Oracle Text Index的常规信息

使用“查看文本索引”页面可以查看有关特定索引的常规信息，例如索引类型，并行度，同步模式，通配符限制，模糊分数，模糊数值结果和数据存储。还提供有关索引上任何分区的信息。

要查看Oracle Text索引的常规信息，请在“ 文本索引”页面的索引列表中，单击索引的名称。将打开“ 查看文本索引”页面，并选中“ 常规”选项卡。从这里，您可以选择执行维护任务的操作。

17.5.3检查Oracle Text Index Health

在Text Manager中，Text Indexes页面显示数据库实例的Oracle Text索引。使用该页面可帮助您了解确保整个应用程序正常运行所必需的关键操作。

使用“文本索引”页面查看：

  • 过去七天内提交的索引和工作的状态。

  • 包含无效分区的Oracle Text索引的数量，因此无效。还会显示所有Oracle Text索引的无效分区数（如果有）。

  • 处于进行中状态的索引和分区数。

  • 所有分区有效且没有活动正在进行的索引数。

  • 为此数据库实例找到的Oracle Text索引的总和。

  • 每个Oracle Text索引的索引类型，所有者，未同步的文档数，文档总数以及碎片百分比。

从列表中选择Oracle Text索引后，可以使用选项进行编辑或执行操作。

17.7跟踪Oracle Enterprise Manager中的数据库功能使用情况

在Oracle Enterprise Manager中，数据库功能使用情况统计信息提供了各种数据库功能使用频率的近似值。跟踪此信息对于应用程序开发和审计非常有用。

要访问数据库功能使用，请在Oracle Enterprise Manager中单击“ 服务器”选项卡，然后选择“ 数据库配置”下的“ 数据库 功能使用 ” 。

数据库功能使用捕获Oracle Text的以下信息：

  • 索引使用情况统计信息：数据库中CONTEXT, CTXCAT,和CTXRULE索引类型的现有索引数

  • SQL运算符使用情况统计信息：用户是否曾使用过CONTAINS, CATSEARCH,和MATCHES运算符

  • 软件包使用情况统计信息：使用以下软件包的频率（如果有）和时间：

    • CTX_ADM

    • CTX_CLS

    • CTX_DDL

    • CTX_DOC

    • CTX_OUTPUT

    • CTX_QUERY

    • CTX_REPORT

    • CTX_THES

    • CTX_ULEXER

注意：

功能使用情况跟踪统计信息可能不是100％准确。

17.9在Oracle Database Vault环境中配置Oracle Text

在Oracle Database Vault环境中，如果您具有DV_ACCTMGR角色，则可以创建CTXSYS用户。

要创建CTXSYS用户，请运行@$ORACLE_HOME/ctx/admin/catctx_user.sql SQL脚本。然后，以SYS用户身份连接并运行@$ORACLE_HOME/ctx/admin/catctx_schema.sql SQL脚本。

注意：

如果SYS用户也具有DV_ACCTMGR角色，则可以运行@$ORACLE_HOME/ctx/admin/catctx.sql SQL脚本，该脚本同时安装catctx_user.sql和catctx_schema.sql脚本。

17.10 Oracle Database Vault领域中不支持的Oracle文本操作

Oracle Database Vault领域限制领域内的DDL操作。因此，一旦将您添加到领域但未在领域中获得授权，则无法创建，更改或删除Oracle Text索引。您也不能使用CTX_DDL包中包含的任何DDL操作。

无法在域中创建的索引上的DDL错误消息和查询错误消息可能表示权限不足是原因。权限不足的消息特定于域内不允许的DDL操作。
