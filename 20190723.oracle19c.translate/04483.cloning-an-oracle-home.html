克隆Oracle数据库主页

克隆Oracle主目录涉及创建Oracle主目录的副本，然后针对新环境进行配置。

如果您正在执行多个Oracle数据库安装，那么您可能希望使用克隆来创建每个Oracle主目录，因为从现有Oracle数据库安装中复制文件所需的时间比创建新版本的文件要少。如果要克隆的Oracle主目录已应用了修补程序，则此方法也很有用。克隆Oracle主目录时，新的Oracle主目录具有修补程序更新。

执行以下步骤以克隆Oracle主目录。

注意：

在克隆过程中，Oracle Universal Installer（OUI）会提示您运行需要root权限的脚本。
  1. 验证要克隆的Oracle数据库的安装是否成功。

    您可以通过查看安装会话的installActionsdate_time.log文件来执行此操作，该文件通常位于/u01/app/oracle/oraInventory/logs目录中。

    如果安装修补程序，请使用以下命令检查其状态：

    $ cd $ ORACLE_HOME / OPatch

    在$PATH包含$ORACLE_HOME/OPatch

    $ opatch lsinventory
  2. 停止与Oracle主目录相关的所有进程。
  3. 使用Oracle主目录（但不是Oracle基础）目录创建ZIP或TAR文件。

    例如，如果源Oracle安装位于路径/u01/app/oracle/product/19.0.0/dbhome_1 ，则使用以下命令压缩dbhome_1目录：

    ＃zip -r dbhome_1.zip /u01/app/oracle/product/19.0.0/dbhome_1

    您也可以使用TAR命令。例如：

    #tar -cvf dbhome_1.tar /u01/app/oracle/product/19.0.0/dbhome_1

    不要包含Oracle基目录下的admin ， fast_recovery_area和oradata目录。当您在那里创建新数据库时，稍后将在目标安装中创建这些目录。

  4. 将ZIP或TAR文件复制到目标计算机的根目录。如果使用文件传输协议（FTP），则仅以二进制模式传输ZIP或TAR文件。
  5. 使用以下命令提取ZIP或TAR文件内容：
    #unzip -d / dbhome_1.zip #tar-xvf dbhome_1.tar
  6. 如有必要，请将Oracle Database主目录的所有权更改为属于Oracle Inventory组（ oinstall ）的Oracle数据库安装所有者用户（ oracle ）。
    #chown -R oracle：oinstall /u01/app/oracle/product/19.0.0/dbhome_1
  7. 在目标计算机上，将目录更改为解压缩的Oracle主目录，并删除解压缩的$ORACLE_HOME/network/admin目录中存在的所有.ora （* .ora ）文件。
  8. 从解压缩的Oracle主目录中删除不必要的文件。

    解压缩的Oracle主目录包含仅与源Oracle主目录相关的文件。以下示例说明如何从解压缩的Oracle主目录中删除这些不必要的文件：

    从network/admin目录中删除.ora文件，并从dbs目录中删除旧的数据库条目。 

    #cd $ ORACLE_HOME＃rm -rf network / admin / * old_database_entries rm dbs / old_database_entries
  9. 从$ORACLE_HOME/clone/bin目录中，运行clone.pl压缩的Oracle主目录的clone.pl文件。

    使用以下语法（您还可以在语法中包含一个或多个扩展的Oracle数据库组）：

    $ ORACLE_HOME /的Perl /斌/ perl的$ ORACLE_HOME /克隆/斌/ clone.pl ORACLE_BASE = “ target_oracle_base ” ORACLE_HOME =“ target_oracle_home ” OSDBA_GROUP = OSDBA_privileged_group OSOPER_GROUP = OSOPER_privileged_group OSBACKUPDBA_GROUP = OSBACKUPDBA_privileged_group OSDGDBA_GROUP = OSDGDBA_privileged_group OSKMDBA_GROUP = OSKMDBA_privileged_group OSRACDBA_GROUP = OSRACDBA_privileged_group -defaultHomeName

    例如：

    $ ORACLE_HOME / perl / bin / perl $ ORACLE_HOME / clone / bin / clone.pl ORACLE_BASE =“/ u01 / app / oracle”ORACLE_HOME =“/ u01 / app / oracle / product / 19.0.0 / dbhome_1”OSDBA_GROUP = dba OSOPER_GROUP = oper OSBACKUPDBA_GROUP = backupdba OSDGDBA_GROUP = dgdba OSKMDBA_GROUP = kmdba OSRACDBA_GROUP = racdba -defaultHomeName

    注意：

    • 在此命令中，如果未提供操作系统组的参数，则clone.pl将使用源主clone.pl的操作系统组值。

    • 有关命令选项标志的更多信息，请运行$ORACLE_HOME/perl/bin/perl $ORACLE_HOME/clone/bin/clone.pl -help命令。

    • clone.pl脚本在Oracle Database 19c中已弃用，可在将来的发行版中删除。因此，Oracle建议您使用数据库安装程序中提供的纯软件安装选项，而不是clone.pl来克隆数据库。

    OUI启动，然后将克隆操作记录在cloneActionstimestamp.log文件中。此日志文件通常位于/u01/app/oracle/oraInventory/logs目录中。
  10. 使用以下命令运行Net Configuration Assistant以配置新数据库的连接信息：
    $ cd $ ORACLE_HOME / bin $ ./netca
  11. 使用以下命令运行Database Configuration Assistant以为新克隆的oracle主目录创建新的Oracle数据库：
    $ cd $ ORACLE_HOME / bin $ ./dbca
