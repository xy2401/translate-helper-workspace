1.93 DB_UNIQUE_NAME

DB_UNIQUE_NAME指定数据库的全局唯一名称。

属性 描述

参数类型

串

句法

DB_UNIQUE_NAME = database_unique_name

默认值

数据库实例： DB_NAME的值

Oracle自动存储管理实例： +ASM

可修改

没有

可在PDB中修改

没有

基本

是

Oracle RAC

多个实例必须具有相同的值。

在同一个DB_DOMAIN具有相同DB_NAME数据库（例如，为报告或物理备用数据库创建的数据库的副本）必须具有唯一的DB_UNIQUE_NAME 。每个数据库的DB_UNIQUE_NAME在企业中必须是唯一的。

DB_UNIQUE_NAME的值最多为30个字符，不区分大小写。以下字符在数据库名称中有效：字母数字字符，下划线（_），数字符号（＃）和美元符号（$）。

注意：

DB_UNIQUE_NAME由Oracle实例中的多个组件用于默认文件名或文件路径，例如DG_BROKER_CONFIG_FILE n初始化参数的默认值。在这些情况下，在UNIX平台上，将从路径或文件名中删除美元符号（$）字符，因为Oracle使用该字符来定义路径或文件名中环境变量替换的开始。

注意：

作为其操作的一部分，某些数据库工具或实用程序会创建唯一标识数据库的字符串。该字符串可以包括数据库的DB_UNIQUE_NAME以及数据库的其他标识信息，例如数据库SID。 Oracle数据库将一些标识符限制为30个字符，因此使用简短的DB_UNIQUE_NAME可以帮助防止ORA-00972 “标识符太长”消息来自创建包含DB_UNIQUE_NAME的字符串的数据库工具和实用程序。
