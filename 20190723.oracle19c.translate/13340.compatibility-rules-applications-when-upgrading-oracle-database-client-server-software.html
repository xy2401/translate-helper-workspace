升级Oracle数据库客户端或服务器软件时的应用程序兼容性规则

升级Oracle数据库客户端或服务器软件时适用兼容性规则。

兼容性规则基于您正在执行的软件升级类型以及客户端/服务器配置的类型。

注意：

本节使用“软件升级和客户端和服务器配置”中介绍的术语。

  • 升级Oracle数据库服务器软件的规则
    根据数据库环境升级Oracle数据库服务器软件时，适用不同的规则。
  • 升级Oracle数据库客户端软件
    将服务器和客户端软件保持在相同的版本号可确保应用程序的最大稳定性。

升级Oracle数据库服务器软件的规则

根据数据库环境升级Oracle数据库服务器软件时，适用不同的规则。

  • 如果您不更改客户端环境，则不需要重新链接
    查看这些方案以确定升级后是否必须重新链接应用程序。
  • 应用程序可以针对较新或较旧的Oracle数据库服务器版本运行
    如果针对数据库服务器运行预编译器或OCI应用程序，则Oracle建议数据库服务器软件的发行版等于或晚于客户端软件版本。

如果您不更改客户端环境，则不需要重新链接

查看这些方案以确定升级后是否必须重新链接应用程序。

如果您的客户端和服务器位于不同的计算机上，或者位于同一台计算机上的不同Oracle主目录中，并且您在不更改客户端软件的情况下升级Oracle数据库服务器软件，则无需预编译，编译或重新链接应用程序。

在这组场景中，使用Oracle数据库的客户端软件与服务器软件位于不同的位置，客户端软件继续运行，没有升级的直接影响。

但是，如果您的应用程序使用与Oracle数据库服务器相同的Oracle主目录，则服务器升级也会升级客户端软件，并且必须遵循升级Oracle数据库客户端软件的规则。

注意：

当您对两个二进制文件使用相同的Oracle主目录时，可以升级Oracle数据库服务器软件，但不能安装新的预编译器或OCI客户端软件。在这种情况下，客户端软件不会升级。但是，Oracle不建议使用此配置。

应用程序可以针对较新或较旧的Oracle数据库服务器版本运行

如果针对数据库服务器运行预编译器或OCI应用程序，则Oracle建议数据库服务器软件的发行版等于或晚于客户端软件版本。

此推荐配置不是严格要求的。

例如：如果您的客户端软件是Oracle 12 c第 2版（12.2.0.1），那么如果您在客户端上针对服务器运行预编译器应用程序，Oracle建议您的服务器软件是Oracle 12 c第 2版（12.2）或更高版本。

升级Oracle数据库客户端软件

将服务器和客户端软件保持在相同的版本号可确保应用程序的最大稳定性。

使用此信息来规划Oracle数据库客户端安装。根据应用程序的链接方式，升级Oracle数据库客户端软件时会应用不同的规则。

Oracle建议您升级客户端软件以匹配当前的服务器软件。例如，当您将Oracle数据库升级到新的Oracle数据库版本时，Oracle建议您还将Oracle数据库客户端软件升级到新版本。最新的Oracle数据库客户端软件可以提供仅在以后版本中提供的附加功能和性能增强功能。

  • 关于基于映像的Oracle数据库客户端安装
    从Oracle Database 19c开始，通过基于映像的安装简化了Oracle Database Client软件的安装和配置。
  • 关于将应用程序与较新的库链接
    您可以将预编译器应用程序生成的代码与OCI链接到客户端库的版本，该版本等于或晚于服务器版本。
  • 静态链接的应用程序必须始终重新链接
    静态链接的代码可能与升级后的ORACLE_HOME中的错误消息不兼容。
  • 关于重新链接动态链接的应用程序
    Oracle Database 10g第1版（10.1）及更高版本中的动态链接OCI应用程序与当前版本向上兼容。

关于基于映像的Oracle数据库客户端安装

从Oracle Database 19c开始，通过基于映像的安装简化了Oracle Database Client软件的安装和配置。

要安装Oracle数据库客户端，请创建新的Oracle主目录，将映像文件解压缩到新创建的Oracle主目录中，然后运行安装向导以注册Oracle数据库产品。

您必须将映像软件（ client_home.zip ）解压缩到您希望Oracle Database Client主目录所在的目录中，然后运行安装向导以启动Oracle Database Client安装和配置。Oracle建议您创建的Oracle主目录路径符合Oracle Optimal Flexible Architecture建议。

使用基于映像的安装，您可以安装Oracle数据库客户端管理员安装类型的32位和64位配置。

与Oracle数据库和Oracle Grid Infrastructure映像文件安装一样，Oracle数据库客户端映像安装可简化Oracle数据库客户端安装并确保最佳实践部署。Oracle Database Client安装二进制文件继续以传统格式提供为非映像zip文件。

关于将应用程序与较新的库链接

您可以将预编译器应用程序生成的代码与OCI链接到客户端库的版本，该版本等于或晚于服务器版本。

您可以将OCI应用程序与OCI运行时库的版本链接，该版本等于或晚于开发应用程序的OCI库的版本。

父主题： 升级Oracle数据库客户端软件

静态链接的应用程序必须始终重新链接

静态链接的代码可能与升级后的ORACLE_HOME中的错误消息不兼容。

您必须为主要和次要版本重新链接静态链接的OCI应用程序。静态链接的Oracle客户端库代码可能与升级后的ORACLE_HOME中的错误消息不兼容。例如，如果使用其他参数更新错误消息，则它将与静态链接的代码不兼容。

父主题： 升级Oracle数据库客户端软件

关于重新链接动态链接的应用程序

Oracle Database 10g第1版（10.1）及更高版本中的动态链接OCI应用程序与当前版本向上兼容。

Oracle客户端动态库与以前版本的库向上兼容。Oracle Universal Installer为解析为当前版本的库的先前版本创建符号链接。因此，与先前版本的Oracle客户端动态库动态链接的应用程序不需要重新链接以与当前版本的Oracle客户端库一起运行。

注意：

如果应用程序与运行时库搜索路径（例如Linux上的-rpath ）链接，则应用程序仍可以使用与其链接的Oracle客户端库的版本运行。您必须重新链接应用程序才能与当前版本的Oracle客户端库一起运行。

如果应用程序与延迟选项链接（例如，静态链接的应用程序），则必须重新链接。

如果应用程序来自Oracle Database 10g第1版（10.1）之前的版本，则必须重新链接。
