7.1 Oracle Net Listener配置文件概述

存储在listener.ora文件中的Oracle Net Listener配置包含以下元素：

  • 监听器的名称

  • 侦听器正在接受连接请求的协议地址

  • 侦听器允许向数据库注册的有效节点

  • 数据库服务

  • 控制参数

动态服务注册 ，无需静态配置支持的服务。但是，如果计划使用Oracle Enterprise Manager云控制，则需要静态服务配置。

默认情况下， listener.ora文件位于ORACLE_HOME/network/admin目录中。listener.ora文件也可以存储在以下位置：

  • 由TNS_ADMIN环境变量或注册表值指定的目录。

  • 在Linux和UNIX操作系统上，它是全局配置目录。例如，在Oracle Solaris操作系统上，目录为/var/opt/oracle 。

    也可以看看：

    • “Oracle数据库全球数据服务概念和管理指南” ，了解有关全局服务管理的信息

    • Oracle操作系统特定的文档

  • 在只读Oracle主模式下， listener.ora文件的默认位置是ORACLE_BASE_HOME /network/admin 。

  • 在只读Oracle主模式下，默认为ORACLE_HOME位置的参数更改为默认值 ORACLE_BASE_HOME位置。

可以在一个listener.ora文件中配置多个侦听器，每个侦听器都具有唯一的名称。可以进行多个侦听器配置，因为每个顶级配置参数都具有侦听器名称的后缀或者是侦听器名称本身。

注意：

  • 在一个listener.ora文件中配置多个侦听器通常很有用。但是，Oracle建议在大多数客户环境中为每个节点仅运行一个侦听器。

  • Oracle Net Services支持listener.ora文件中的IFILE参数，最多可嵌套三层。该参数手动添加到文件中。以下是语法示例：

    IFILE = / tmp / listener_em.ora IFILE = / tmp / listener_cust1.ora IFILE = / tmp / listener_cust2.ora

    有关其他信息，请参阅Oracle数据库参考 。

    • 
                     
  • RATE_LIMIT
    listener.ora文件的RATE_LIMIT配置参数指示特定侦听端点是速率限制的。

7.3.2 RATE_LIMIT

                  

                     
listener.ora文件的RATE_LIMIT配置参数指示特定侦听端点是速率限制的。
                     

                     

                        
目的

                     

                     
                     
指示特定侦听端点是速率限制的。

                     

                        
使用说明

                     

                     
                     
该参数在侦听器端点配置的ADDRESS部分中指定。
                     

                     

                        
句法

                     

                     
LISTENER =（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1521）（RATE_LIMIT = yes））
    
                        
  • 
                           
    当端点的RATE_LIMIT参数设置为yes ，该端点将包含在CONNECTION_RATE_ listener_name参数配置的全局速率的强制中。在RATE_LIMIT设置为yes每个端点上单独强制执行全局速率限制。
                           
    
                        
    • 
                        
  • 
                           
    由Oracle Clusterware管理的侦听器的动态端点将RATE_LIMIT参数设置为yes 。
                           
    
                        
    • 
                        
  • 
                           
    当RATE_LIMIT参数设置为大于0的值时，将在该端点级别强制执行速率限制。
                           
    
                        
    • 
                     

                     

                        
例子

                     

                     
                     
以下示例使用CONNECTION_RATE_ listener name和RATE_LIMIT参数。
                     

                     
例1
CONNECTION_RATE_LISTENER = 10 LISTENER =（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1521）（RATE_LIMIT = yes））（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1522）（RATE_LIMIT =是））（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1523）））
在前面的示例中，为每个端点单独强制实施新连接的全局速率。通过端口1521的连接每秒限制为10，并且通过端口1522的连接也被限制为每秒10。通过端口1523的连接不受限制。

                     
例2
LISTENER =（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1521）（RATE_LIMIT = 5））（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1522）（RATE_LIMIT = 10） ）（ADDRESS =（PROTOCOL = tcp）（HOST =）（PORT = 1523）））
在前面的示例中，连接速率在端点级别强制执行。每秒最多通过端口1521处理5个连接。通过端口1522的连接限制是每秒10。通过端口1523的连接不受限制。

                     

                        
注意：

                        
在前面的配置中未指定全局CONNECTON_RATE_ listener_name参数。如果已指定，则忽略端口1521和1522上的限制，而是使用全局值。

7.5 Oracle Net Listener的ADR诊断参数

               

                  
快速捕获关键错误的诊断数据并将其存储在ADR for Oracle Net侦听器中。

                  
自Oracle Database 11g以来，Oracle数据库包含一个高级故障诊断基础结构，用于预防，检测，诊断和解决问题。问题是严重错误，例如由数据库代码错误，元数据损坏和客户数据损坏引起的错误。

                  
发生严重错误时，会为其分配一个事件编号，并立即捕获错误的诊断数据（如跟踪和转储），并使用事件编号进行标记。然后将数据存储在自动诊断存储库（ADR）中，该存储库是数据库外部的基于文件的存储库。

                  
本节包括启用ADR时使用的参数。默认情况下启用ADR。启用ADR时，将忽略listener.ora文件中列出的非ADR参数。

    • 
                     
  • DIAG_ADR_ENABLED_listener_name
    DIAG_ADR_ENABLED_ listener_name是listener.ora文件的诊断参数。它表示ADR是否已启用。
                     
    • 
                     
  • LOG_FILE_NUM_listener_name
    LOG_FILE_NUM_ listener_name是listener.ora文件的诊断参数，用于指定日志文件段的数量。
                     
    • 
                     
  • LOG_FILE_SIZE_listener_name
    listener.ora文件的LOG_FILE_SIZE_ listener_name诊断参数指定每个日志文件段的大小。
                     
    • 
                     
  • LOGGING_listener_name
    listener.ora文件的LOGGING_ listener_name诊断参数将打开或关闭日志记录。
                     
    • 
                     
  • TRACE_LEVEL_listener_name
    listener.ora文件的TRACE_LEVEL_ listener_name诊断参数会在特定级别打开侦听器跟踪，或将其关闭。
                     
    • 
                     
  • TRACE_TIMESTAMP_listener_name
