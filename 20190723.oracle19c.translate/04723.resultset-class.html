13.27 ResultSet类

ResultSet提供对通过执行Statement生成的数据表的访问。按行顺序检索表行。在一行中，可以按任何顺序访问列值。

ResultSet维护指向其当前数据行的游标。最初，光标位于第一行之前。next方法将光标移动到下一行。

get xxx ()方法检索当前行的列值。您可以使用列的索引号检索值。列从1开始编号。对于get xxx ()方法，OCCI尝试将基础数据转换为指定的C ++类型并返回C ++值。SQL类型使用ResultSet::get xxx ()方法映射到C ++类型。

ResultSet列的数量，类型和属性由getColumnListMetaData（）方法返回的MetaData对象提供。

  • STREAM_DATA_AVAILABLE指示应用程序应调用getCurrentStreamColumn（）方法并读取流。

13.27.39 next（）

此方法从先前执行的查询中获取指定数量的行numRows ，并报告此获取的Status ，如表13-38中所定义。

最多numRows数据记录将填充setDataBuffer（）调用指定的缓冲区。要确定返回的确切记录数，请使用getNumArrayRows（）方法。

指定数据缓冲区，其中将获取或绑定数据。buffer参数是指向用户分配的数据缓冲区的指针。必须在length参数中指定当前的数据length 。数据量不应超过size参数。最后， type是数据的数据类型。只能使用非OCCI和非C ++特定类型，例如STL字符串。不能使用像Bytes和Date这样的OCCI类。

如果setDataBuffer（）用于获取数组提取的数据，则每个结果集只应调用一次。假设每行的数据位于buffer (i- 1)位置，其中i是行号。类似地，假设数据的长度为(length+(i-1)).

有关使用32K length参数的此方法版本的更多信息，请参阅Oracle数据库SQL语言参考 。

句法 描述 
void setDataBuffer（unsigned int colIndex，void * buffer，Type type，sb4 size = 0，ub2 * length = NULL，sb2 * ind = NULL，ub2 * rc = NULL）;

使用ub2 length缓冲区。这将VARCHAR2和NVARCHAR2长度限制为4,000字节， RAW数据类型限制为2,000字节。 

void setDataBuffer（unsigned int colIndex，void * buffer，Type type，sb4 size = 0，ub4 * length = NULL，sb2 * ind = NULL，ub2 * rc = NULL）;

参数 描述
