1.1概述

您必须为Oracle数据库设置Oracle数据库环境变量，参数和用户设置才能工作。本章介绍Oracle数据库的各种设置。

在Oracle数据库文件和程序中，有一个问号（？）表示ORACLE_HOME环境变量的值。例如，Oracle Database将以下SQL语句中的问号扩展到Oracle主目录的完整路径：

SQL> ALTER TABLESPACE TEMP添加数据文件'？/dbs/temp02.dbf'SIZE 200M

类似地，at符号（@）表示ORACLE_SID环境变量。例如，要指示属于当前实例的文件，请运行以下命令：

SQL> ALTER TABLESPACE tablespace_name ADD DATAFILE tempfile @ .dbf

1.2.1 Oracle数据库环境变量

下表描述了Oracle数据库使用的一些环境变量。

表1-1 Oracle数据库环境变量

变量 定义

NLS_LANG

功能：指定客户端环境的语言，区域和字符集。NLS_LANG指定的客户端字符集必须与终端或终端仿真器的字符集匹配。如果需要，可以在启动非交互式批处理程序以匹配此程序处理的文件和脚本的字符集之前，将NLS_LANG临时重置为另一个字符集。NLS_LANG指定的字符集可以与数据库字符集不同，在这种情况下，字符集会自动转换。

有关此变量的参数列表，请参阅“ Oracle数据库全球化支持指南” 。

语法： language_territory.characterset

示例： french_france.we8iso8859p15

ORA_NLS10

功能：指定存储语言，地域，字符集和语言定义文件的目录。

语法： directory_path

示例： $ORACLE_HOME/nls/data

ORA_TZFILE

功能：指定时区文件的完整路径和文件名。Oracle数据库服务器始终使用大时区文件（ $ORACLE_HOME/oracore/zoneinfo/timezlrg_ number .dat ）。如果要在客户端使用小时区文件，则必须将此环境变量设置为小时区文件的完整路径（ $ORACLE_HOME/oracore/zoneinfo/timezone_ number .dat ）。如果在客户端使用小时区文件，则必须确保您访问的数据库仅包含小时区文件识别的时区区域中的数据。

语法： directory_path

示例： $ORACLE_HOME/oracore/zoneinfo/timezlrg_11.dat

ORACLE_BASE

功能：指定最佳Flexible Architecture兼容安装的Oracle目录结构的基础。

语法： directory_path

示例： /u01/app/oracle

ORACLE_HOME

功能：指定包含Oracle软件的目录。

语法： directory_path

示例： $ORACLE_BASE/product/19.0.0/dbhome_1

ORACLE_PATH

功能：指定Oracle应用程序（如SQL * Plus）使用的文件的搜索路径。如果未指定文件的完整路径，或者文件不在当前目录中，则Oracle应用程序使用ORACLE_PATH来定位文件。

语法：以冒号分隔的目录列表：

directory1 ： directory2 ： directory3

示例： /u01/app/oracle/product/19.0.0/dbhome_1/bin:

ORACLE_SID

功能：指定Oracle系统标识符。

语法：一串必须以字母开头的数字和字母。Oracle建议系统标识符最多包含8个字符。有关此环境变量的更多信息，请参阅“ Oracle数据库安装指南” 。

示例： SAL1

ORACLE_TRACE

功能：在安装期间启用shell脚本跟踪。如果将其设置为T ，则许多Oracle shell脚本使用set -x命令，该命令在运行时打印命令及其参数。如果将其设置为任何其他值或没有值，则脚本不使用set -x命令。

语法： T或不T

示例： T

ORAENV_ASK

功能：控制oraenv或coraenv脚本是否提示或不提示输入ORACLE_SID环境变量的值。如果将其设置为NO ，则脚本不会提示输入ORACLE_SID环境变量的值。如果将其设置为任何其他值或没有值，则脚本会提示输入ORACLE_SID环境变量的值。

语法： NO或NO

示例： NO

SQLPATH

功能：指定SQL * Plus搜索login.sql文件的目录或目录列表。

语法：以冒号分隔的目录列表： directory1:directory2:directory3

示例： /home:/home/oracle:/u01/oracle

TNS_ADMIN

功能：指定包含Oracle Net Services配置文件的目录。

语法： directory_path

示例： $ORACLE_HOME/network/admin

TWO_TASK

功能：指定要在连接字符串中使用的默认连接标识符。如果设置了此环境变量，则不要在连接字符串中指定连接标识符。例如，如果TWO_TASK环境变量设置为sales ，则可以使用以下命令连接到数据库：

SQL> CONNECT username输入密码： 密码

语法：任何连接标识符。

值范围：可以使用命名方法解析的任何有效连接标识符，例如tnsnames.ora文件或目录服务器。

示例： PRODDB_TCP

NLS_OS_CHARSET

功能：指定与UNIX语言环境字符集对应的Oracle字符集名称，其中文件名和用户名由操作系统编码。如果UNIX区域设置字符集与Oracle客户端字符集不同NLS_OS_CHARSET,则必须设置环境变量NLS_OS_CHARSET, 。这两个字符集可能不同，例如，如果NLS_LANG设置为用于编码SQL脚本的特定字符集，则将在SQL * Plus会话中执行。通常，Oracle客户端字符集和操作系统字符集是相同的， NLS_OS_CHARSET不能设置NLS_OS_CHARSET 。

语法： characterset

示例： WE8ISO8859P1

注意：

  • 要防止冲突，请不要定义名称与Oracle数据库服务器进程名称相同的环境变量。例如ARCH ， PMON和DBWR 。

  • 如果缺少Oracle安装所有者（ oracle用户）环境变量，或者将它们设置为不正确的路径，则在Oracle数据库启动期间可能会遇到未定义的错误。确保在启动数据库之前正确设置了所有环境变量。有关更多信息，请参阅My Oracle Support说明373303.1：

    https://support.oracle.com/epmos/faces/DocumentDisplay?id=373303.1

警告：

在按照My Oracle Support说明373303.1中的建议实施任何更改之前，您必须咨询Oracle支持。

1.2.3设置共同环境

本节介绍如何使用oraenv或coraenv脚本设置公共操作系统环境，具体取决于默认shell：

  • 对于Bourne，Bash或Korn shell，请使用oraenv命令。

  • 对于C shell，请使用coraenv命令。

oraenv和coraenv脚本文件

oraenv和coraenv脚本是在安装期间创建的。这些脚本根据oratab文件的内容设置环境变量，并提供：

  • 使用数据库更改更新所有用户帐户的核心方法

  • 用于在oratab文件中指定的数据库之间切换的机制

您可能会发现自己经常在开发系统中添加和删除数据库，或者您的用户可能在同一系统上安装的多个不同Oracle数据库之间切换。您可以使用oraenv或coraenv脚本来确保更新用户帐户并在数据库之间切换。

注意：

不要从Oracle软件所有者（通常是oracle ）用户的shell启动脚本调用oraenv或coraenv脚本。由于这些脚本会提示输入值，因此它们可以阻止dbstart脚本在系统启动时自动启动数据库。

oraenv或coraenv脚本通常从用户的shell启动文件中调用（例如， .profile或.login ）。它设置ORACLE_SID和ORACLE_HOME环境变量，并在PATH环境变量设置中包含$ORACLE_HOME/bin目录。在数据库之间切换时，用户可以运行oraenv或coraenv脚本来设置这些环境变量。

注意：

要运行其中一个脚本，请使用适当的命令：

  • coraenv脚本：

    ％source / usr / local / bin / coraenv

  • oraenv脚本：

    $。在/ usr / local / bin目录/ oraenv的

本地bin目录

包含oraenv ， coraenv和dbhome脚本的目录称为本地bin目录。所有数据库用户必须具有此目录的读取权限。包括用户的本地bin目录PATH环境变量设置的PATH 。在安装后运行root.sh脚本时，脚本会提示您输入本地bin目录的路径，并自动将oraenv ， coraenv和dbhome脚本复制到您指定的目录。默认的本地bin目录是/usr/local/bin 。如果不运行root.sh脚本，则可以手动将oraenv或coraenv和dbhome脚本从$ORACLE_HOME/bin目录复制到本地bin目录。

1.3.2 DISK_ASYNCH_IO初始化参数（HP-UX）

DISK_ASYNCH_IO初始化参数确定数据库文件是驻留在原始磁盘还是文件系统上。异步I / O仅适用于自动存储管理磁盘组，该磁盘组使用原始分区作为数据库文件的存储选项。DISK_ASYNCH_IO参数可以设置为TRUE或FALSE具体取决于文件所在的位置。默认情况下，该值设置为TRUE 。

注意：

当数据库文件驻留在文件系统上时， DISK_ASYNCH_IO参数必须设置为FALSE 。仅当数据库文件驻留在原始分区上时，此参数必须设置为TRUE 。

1.3.3 PROCESSOR_GROUP_NAME初始化参数

PROCESSOR_GROUP_NAME指定运行实例的处理器组的名称。此参数指示Oracle数据库仅在作为指定操作系统处理器组的一部分的处理器上运行。对于NUMA系统，从与该处理器组中的CPU相关联的NUMA节点分配所有系统全局区域（SGA）和程序全局区域（PGA）。

只有Linux x86-64和Oracle Solaris 11 SRU 4及更高版本支持PROCESSOR_GROUP_NAME参数。

在Linux x86-64上，命名的CPU子集是通过称为控制组（cgroups）的Linux功能创建的。Cgroups在Linux内核版本2.6.24中引入。它是通过为组指定名称和一组CPU来创建的。当进程映射到cgroup时，它仅使用与cgroup关联的CPU。

在Oracle Solaris 11 SRU 4上，通过称为资源池的功能创建指定的CPU子集。每个资源池由一个名称和一组CPU组成。当进程映射到资源池时，它使用与资源池关联的CPU。

注意：

Oracle建议仅为使用专用连接代理的数据库设置PROCESSOR_GROUP_NAME参数。USE_DEDICATED_BROKER初始化参数用于配置专用连接代理。
