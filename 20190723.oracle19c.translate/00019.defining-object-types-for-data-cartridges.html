3.1数据磁带的对象和对象类型

在Oracle对象关系数据库管理系统（ORDBMS）中，您使用对象类型来模拟现实世界的实体。对象类型具有反映实体结构的属性，以及实现实体操作的方法。使用内置类型或其他对象类型定义属性。方法是用PL / SQL或外部语言（如C）编写的函数或过程，并存储在数据库中。

对象类型的典型用途是在数据库中的某些数据部分上施加结构。例如，盒式磁带可以使用名为DataStream的对象类型将大量数据存储在字符LOB（大型对象的数据类型）中。此对象类型具有标识符，名称，日期等属性。例3-1中的语句定义了DataStream数据类型：

方法是一个过程或函数，它是对象类型定义的一部分，可以对对象类型数据属性进行操作。这些方法称为成员方法 ，当您将它们指定为对象类型的组件时，它们将使用关键字MEMBER 。DataStream类型定义声明了三种方法。前两个DataStreamMin和DataStreamMax分别计算存储在字符LOB中的数据流中的最小值和最大值。第三种方法DataStreamToInt是一种map方法 ，用于控制数据流类型实例之间的比较。

3.2将对象标识符分配给数据盒中的对象类型

CREATE TYPE语句具有可选的关键字OID ，它将用户指定的对象标识符（ OID ）与类型定义相关联。任何创建在几个database.s中使用的对象类型的人都需要

每种类型都有一个OID 。如果创建对象类型但未指定OID ，Oracle会生成OID并将其分配给该类型。Oracle在内部使用OID进行与该类型相关的操作。如果您计划跨数据库共享该类型的实例以执行导出/导入和分布式查询等操作，则对类型使用相同的OID非常重要。

请注意，在带有OID CREATE TYPE ， OID类型本身分配OID 。具有指定类型的列的表中的每一行都具有行特定的OID 。

考虑创建一个SpecialPerson类型，然后在名为SpecialPersonTable1和SpecialPersonTable2表中的两个不同数据库中实例化此类型。RDBMS必须知道两个实例中的SpecialPerson类型是相同的类型，因此必须在两个数据库中使用相同的OID定义类型。如果未使用CREATE TYPE指定OID ，则RDBMS会自动创建唯一标识符。为对象类型指定OID的语法在例3-3中 。

在示例3-4中 ， SELECT语句生成OID ， CREATE TYPE语句在创建名为mytype的对象类型时使用OID 。请务必使用SELECT语句为要创建的每个对象类型生成不同的OID ，因为这是保证每个OID有效且全局唯一的唯一方法。

3.4数据磁带中的对象比较

SQL对对象执行比较操作。一些比较是显式的，使用比较运算符（=，<，>，<>，<=，> = ,!=）和BETWEEN和IN谓词。其他比较是隐式的，如GROUP BY ， ORDER BY ， DISTINCT和UNIQUE子句。

对象的比较使用对象类型的特殊成员函数：map方法和order方法。要执行对象比较，必须在CREATE TYPE和CREATE TYPE BODY语句中实现map方法或order方法。在示例3-7中 ， DataStream类型的类型主体实现了map成员函数。

map成员函数的定义依赖于DataStream类型的id属性的存在，以将实例映射到整数。只要DataStream类型的对象之间需要进行比较操作，系统就会隐式调用map函数DataStreamToInt() 。

对象类型rational_type没有像DataStream这样的简单id属性。相反，它的map成员函数很复杂，如例3-8所示 。因为map函数可以返回任何内置类型，所以rational_type可以返回值或键入REAL 。

如果尚未为对象类型定义映射或顺序函数，则它只能支持相等比较。Oracle SQL通过对该类型的属性进行逐字段比较来执行比较。
