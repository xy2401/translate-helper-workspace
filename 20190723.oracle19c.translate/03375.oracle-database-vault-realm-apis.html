14 Oracle Database Vault领域API

DBMS_MACADM PL / SQL包使您可以配置Oracle Database Vault领域。

只有已被授予DV_OWNER或DV_ADMIN角色的用户才能使用这些过程。有关可以与这些过程一起使用的常量，请参阅表20-1以获取更多信息。

  • ADD_AUTH_TO_REALM程序
    ADD_AUTH_TO_REALM过程授权用户或角色作为所有者或参与者访问领域。在多租户环境中，您可以对常见域和本地域进行身份验证。
  • ADD_OBJECT_TO_REALM程序
    ADD_OBJECT_TO_REALM过程注册一组对象以进行域保护。
  • CREATE_REALM过程
    CREATE_REALM过程创建一个领域。在多租户环境中，您可以创建公共领域和本地领域。
  • DELETE_AUTH_FROM_REALM过程
    DELETE_AUTH_FROM_REALM过程删除用户或角色访问领域的授权。
  • DELETE_OBJECT_FROM_REALM过程
    DELETE_OBJECT_FROM_REALM过程从域保护中删除一组对象。
  • DELETE_REALM过程
    DELETE_REALM过程删除领域，包括其相关配置信息，该信息指定谁被授权以及哪些对象受到保护。
  • DELETE_REALM_CASCADE过程
    DELETE_REALM_CASCADE过程删除领域，包括其相关的Database Vault配置信息，该信息指定授权的人员和受保护的对象。
  • RENAME_REALM程序
    RENAME_REALM过程重命名领域;名称更改将在使用领域的任何位置生效。
  • UPDATE_REALM过程
    UPDATE_REALM过程更新领域。
  • UPDATE_REALM_AUTH过程
    UPDATE_REALM_AUTH过程更新用户或角色的授权以访问领域。

ADD_AUTH_TO_REALM程序

ADD_AUTH_TO_REALM过程授权用户或角色作为所有者或参与者访问领域。在多租户环境中，您可以对常见域和本地域进行身份验证。

有关域授权的详细信息，请参阅关于域授权 。

（可选）您可以指定在允许启用授权之前必须检查的规则集。

句法

DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name IN VARCHAR2，grantee IN VARCHAR2，rule_set_name IN VARCHAR2，auth_options IN NUMBER auth_scope IN NUMBER DEFAULT）;

参数

表14-1 ADD_AUTH_TO_REALM参数

参数 描述

realm_name

境界名称。

要查找当前数据库实例中的现有域，请查询DBA_DV_REALM视图中描述的DBA_DV_REALM视图 。

受让人

要授权为所有者或参与者的用户或角色名称。

要查找当前数据库实例中的现有用户和角色，请查询Oracle Database Reference中描述的DBA_USERS和DBA_ROLES视图。

要查找特定用户或角色的授权，请查询DVA_DV_REALM_AUTH视图中描述的DVA_DV_REALM_AUTH视图 。

要查找权限管理中使用的现有安全应用程序角色，请查询DBA_DV_ROLE视图。这两者都在Oracle Database Vault数据字典视图中描述。

rule_set_name

可选的。规则集在运行时检查。仅当规则集的计算结果为TRUE才会启用领域授权。

要查找可用的规则集，请查询DBA_DV_RULE_SET视图中描述的DBA_DV_RULE_SET视图 。

auth_options

可选的。指定以下选项之一以授权该领域：

  • DBMS_MACUTL.G_REALM_AUTH_PARTICIPANT ：参与者。此帐户或角色提供访问，操作和创建域的受保护对象的系统或直接权限，前提是这些权限已使用标准Oracle数据库权限授予过程授予。（默认）

  • DBMS_MACUTL.G_REALM_AUTH_OWNER ：所有者。此帐户或角色具有与领域参与者相同的授权，以及授予或撤消领域受保护对象的领域安全角色和特权的授权。

有关参与者和所有者的更多信息，请参阅关于领域授权 。

auth_scope

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）在当前PDB中本地授权域

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）授权应用程序根目录中的域

例子

以下示例将用户SYSADM授权为Performance Statistics Realm中的参与者。由于默认设置是将用户授权为参与者，因此可以省略auth_options参数。 

BEGIN DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name =>'Performance Statistics Realm'，grantee =>'SYSADM'）;结束; /

此示例将用户SYSADM设置为Performance Statistics Realm的所有者。 

BEGIN DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name =>'Performance Statistics Realm'，grantee =>'SYSADM'，auth_options => DBMS_MACUTL.G_REALM_AUTH_OWNER）;结束; /

下一个示例在允许用户SYSADM充当Performance Statistics Realm的所有者之前触发Check Conf Access规则集。 

BEGIN DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name =>'Performance Statistics Realm'，grantee =>'SYSADM'，rule_set_name =>'Check Conf Access'，auth_options => DBMS_MACUTL.G_REALM_AUTH_OWNER）;结束; /

此示例显示如何通常授予公共用户C##HR_ADMIN访问公共领域HR Statistics Realm的权限。运行此过程的用户必须位于CDB根目录中，并且规则集必须是驻留在应用程序根目录中的公共规则集。 

BEGIN DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name =>'HR Statistics Realm'，grantee =>'C ## HR_ADMIN'，rule_set_name =>'Check Access'，auth_options => DBMS_MACUTL.G_REALM_AUTH_OWNER，auth_scope => DBMS_MACUTL.G_SCOPE_COMMON）;结束; /

此示例显示如何在本地授予公共用户C##HR_CLERK访问公共领域HR Statistics Realm的权限。运行此过程的用户必须位于授权适用的同一PDB中。查找现有PDB查询DBA_PDBS数据字典视图。规则集必须是本地规则集。 

BEGIN DBMS_MACADM.ADD_AUTH_TO_REALM（realm_name =>'HR Statistics Realm'，grantee =>'C ## HR_CLERK'，rule_set_name =>'Check Access'，auth_options => DBMS_MACUTL.G_REALM_AUTH_OWNER，auth_scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

CREATE_REALM过程

CREATE_REALM过程创建一个领域。在多租户环境中，您可以创建公共领域和本地领域。

创建领域后，使用以下过程完成领域定义：

  • ADD_OBJECT_TO_REALM过程为域注册一个或多个对象。

  • ADD_AUTH_TO_REALM过程授权领域的用户或角色。

句法

DBMS_MACADM.CREATE_REALM（realm_name IN VARCHAR2，描述IN VARCHAR2，启用IN VARCHAR2，audit_options IN NUMBER，realm_type IN NUMBER DEFAULT，realm_scope IN NUMBER DEFAULT pl_sql_stack IN BOOLEAN DEFAULT）;

参数

表14-3 CREATE_REALM参数

参数 描述

realm_name

领域名称，混合大小最多128个字符。

要查找当前数据库实例中的现有域，请查询DBA_DV_REALM视图中描述的DBA_DV_REALM视图 。

描述

领域的目的描述，混合大小最多1024个字符。

启用

指定以下选项之一以设置领域的状态：

  • DBMS_MACUTL.G_YES或'y'启用领域检查（默认）

  • DBMS_MACUTL.G_NO或'n'禁用所有领域检查，包括在模拟日志中捕获违规

  • DBMS_MACUTL.G_SIMULATION或's'使SQL语句能够执行，但在模拟日志中捕获违规

audit_options

指定以下选项之一以审核域：

  • DBMS_MACUTL.G_REALM_AUDIT_OFF ：禁用领域审核（默认）

  • DBMS_MACUTL.G_REALM_AUDIT_FAIL ：发生域违规时创建审核记录（例如，当未经授权的用户尝试修改受DBMS_MACUTL.G_REALM_AUDIT_FAIL的对象时）

  • DBMS_MACUTL.G_REALM_AUDIT_SUCCESS ：为受领域保护的对象的授权活动创建审计记录

  • DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS ：为受领域保护的对象上的授权和未授权活动创建审核记录

realm_type

指定以下选项之一：

  • 0 ：禁用强制域检查。

  • 1 ：启用领域对象的强制领域检查。只有领域所有者或领域参与者才能访问领域中的对象。不是领域所有者或参与者的对象所有者和对象特权用户将无法访问。

有关必需领域的详细信息，另请参阅强制领域以限制用户对领域内对象的访问 。

realm_scope

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）如果域必须在当前PDB中是本地的。

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果域必须位于应用程序根目录中。此设置会复制所有关联PDB中的域。

如果在应用程序根目录中创建公共领域并希望它对关联的PDB可见，则必须同步应用程序。例如：

ALTER PLUGGABLE DATABASE APPLICATION saas_sales_app SYNC;

pl_sql_stack

启用模拟模式时，指定是否记录失败操作的PL / SQL堆栈。输入TRUE以记录PL / SQL堆栈，输入FALSE以不记录。默认值为FALSE 。

例子

以下示例说明如何创建已启用的域，将审计设置为跟踪失败和成功访问，使用强制域检查，以及记录PL / SQL堆栈。

BEGIN DBMS_MACADM.CREATE_REALM（realm_name =>'Performance Statistics Realm'，description =>'Realm to measure performance'，enabled => DBMS_MACUTL.G_YES，audit_options => DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS，realm_type => 1，pl_sql_stack =>真正）;结束; /

此示例显示如何创建前面示例的变体，但作为位于应用程序根目录中的公共领域。创建此领域的用户必须是普通用户，并且必须在CDB根目录中执行该过程。

BEGIN DBMS_MACADM.CREATE_REALM（realm_name =>'Performance Statistics Realm'，description =>'Realm to measure performance'，enabled => DBMS_MACUTL.G_YES，audit_options => DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS，realm_type => 1，realm_scope => DBMS_MACUTL.G_SCOPE_COMMON）;结束; /

此示例显示如何创建前面示例的本地版本n。创建此领域的用户必须位于领域所在的PDB中。要查找现有PDB，请查询DBA_PDBS数据字典视图。 

BEGIN DBMS_MACADM.CREATE_REALM（realm_name =>'Performance Statistics Realm'，description =>'Realm to measure performance'，enabled => DBMS_MACUTL.G_YES，audit_options => DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS，realm_type => 1，realm_scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

  • DBMS_MACUTL.G_REALM_AUDIT_SUCCESS ：为受领域保护的对象上的授权活动创建审核记录。

  • DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS ：为受领域保护的对象上的授权和未授权活动创建审核记录

audit_options的默认值是先前设置的值，您可以通过查询DBA_DV_REALM数据字典视图找到该DBA_DV_REALM 。

realm_type

如果未指定realm_type参数，则Oracle Database Vault不会更新当前的realm_type设置。

指定以下选项之一：

  • 0 ：将领域设置为常规领域，没有强制领域检查。

  • 1 ：启用领域对象的强制领域检查。只有领域所有者或领域参与者才能访问领域中的对象。不是领域所有者或参与者的对象所有者和对象特权用户将无法访问。

有关必需领域的详细信息，另请参阅强制领域以限制用户对领域内对象的访问 。

pl_sql_stack

启用模拟模式时，指示是否已记录PL / SQL堆栈以进行失败操作。TRUE表示已记录PL / SQL堆栈; FALSE表示尚未记录PL / SQL堆栈。

例

BEGIN DBMS_MACADM.UPDATE_REALM（realm_name =>'Sector 2 Performance Statistics Realm'，description =>'Realm to measure Sector 2 applications'，enabled => DBMS_MACUTL.G_YES，audit_options => DBMS_MACUTL.G_REALM_AUDIT_FAIL + DBMS_MACUTL.G_REALM_AUDIT_SUCCESS），realm_type => 1）;结束; /

UPDATE_REALM_AUTH过程

UPDATE_REALM_AUTH过程更新用户或角色的授权以访问领域。

句法

DBMS_MACADM.UPDATE_REALM_AUTH（realm_name IN VARCHAR2，grantee IN VARCHAR2，rule_set_name IN VARCHAR2，auth_options IN NUMBER，auth_scope IN NUMBER DEFAULT）;

参数

表14-10 UPDATE_REALM_AUTH参数

参数 描述

realm_name

境界名称。

要查找当前数据库实例中的现有域，请查询DBA_DV_REALM视图中描述的DBA_DV_REALM视图 。

受让人

用户或角色名称。

要查找当前数据库实例中的可用用户和角色，请查询Oracle Database Reference中描述的DBA_USERS和DBA_ROLES视图。

要查找特定用户或角色的授权，请查询DVA_DV_REALM_AUTH视图中描述的DVA_DV_REALM_AUTH视图 。

要查找权限管理中使用的现有安全应用程序角色，请查询DBA_DV_ROLE视图中描述的DBA_DV_ROLE视图 。

rule_set_name

可选的。要在运行时检查的规则集。仅当规则集的计算结果为TRUE才会启用领域授权。

要查找可用的规则集，请查询DBA_DV_RULE_SET视图。要查找与规则集关联的规则，请查询DBA_DB_RULE_SET_RULE视图。这两者都在Oracle Database Vault数据字典视图中描述。

auth_options

可选的。指定以下选项之一以授权该领域：

  • DBMS_MACUTL.G_REALM_AUTH_PARTICIPANT ：参与者。此帐户或角色提供访问，操作和创建域的受保护对象的系统或直接权限，前提是这些权限已使用标准Oracle数据库权限授予过程授予。

  • DBMS_MACUTL.G_REALM_AUTH_OWNER ：所有者。此帐户或角色具有与领域参与者相同的授权，以及授予或撤消领域受保护对象的领域安全角色和特权的授权。领域可以拥有多个所有者。

auth_options值的缺省值是先前设置的值，您可以通过查询DBA_DV_REALM_AUTH数据字典视图找到该DBA_DV_REALM_AUTH 。

realm_auth

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）如果域在当前PDB中本地授权

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果域在应用程序根目录中被授权

例

BEGIN DBMS_MACADM.UPDATE_REALM_AUTH（realm_name =>'Sector 2 Performance Statistics Realm'，grantee =>'SYSADM'，rule_set_name =>'Check Conf Access'，auth_options => DBMS_MACUTL.G_REALM_AUTH_OWNER）;结束; /
