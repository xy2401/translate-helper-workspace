176 DBMS_TTS

DBMS_TTS包检查可传输集是否是自包含的。所有违规都插入到临时表中，可以从视图TRANSPORT_SET_VIOLATIONS选择。

176.1 DBMS_TTS安全模型

只有具有execute_catalog_role用户才能执行此过程。此角色最初仅分配给用户SYS。

176.2 DBMS_TTS例外

DBMS_TTS包为缺失或无效的可传输表空间创建例外。 

ts_not_found EXCEPTION; PRAGMA exception_init（ts_not_found，-29304）; ts_not_found_num NUMBER：= -29304; invalid_ts_list EXCEPTION; PRAGMA exception_init（invalid_ts_list，-29346）; invalid_ts_list_num NUMBER：= -29346; sys_or_tmp_ts EXCEPTION; PRAGMA exception_init（sys_or_tmp_ts，-29351）; sys_or_tmp_ts_num NUMBER：= -29351;

176.3 DBMS_TTS操作说明

对于可传输的表空间，禁用和启用的引用完整性约束的处理方式不同。

  • 禁用的参照完整性约束不违反可传输性规则，并在导入阶段被删除。

  • 如果启用的引用完整性约束引用可传输集外部的表空间中的表，则会违反可传输性规则。

176.4 DBMS_TTS子程序摘要

表中列出的两个过程旨在由数据库管理员调用。

表176-1 DBMS_TTS包子程序

子程序 描述

DOWNGRADE程序

降低可传输的表空间相关数据

TRANSPORT_SET_CHECK程序

检查一组表空间（要传输）是否是自包含的

176.4.1降级程序

此过程降级可传输的表空间相关数据。

句法

DBMS_TTS.DOWNGRADE;

176.4.2 TRANSPORT_SET_CHECK程序

此过程检查一组表空间（要传输）是否是自包含的。在调用此过程之后，用户可以从视图中选择以查看违规列表（如果有）。

句法

DBMS_TTS.TRANSPORT_SET_CHECK（ts_list IN CLOB，incl_constraints IN BOOLEAN DEFAULT FALSE，full_check IN BOOLEAN DEFAULT FALSE）;

参数

表176-2 TRANSPORT_SET_CHECK过程参数

参数 描述

ts_list

一个或多个表空间的列表，以逗号分隔。

incl_constraints

如果要在检查表空间集是否为自包含时计算引用完整性约束，则为TRUE 。（ incl_constraints参数是默认值，因此如果仅使用ts_list参数调用TRANSPORT_SET_CHECK它将起作用。）

full_check

指示是否需要完整或部分依赖性检查。如果为TRUE,处理所有IN和OUT指针（依赖项）并将它们捕获为违规（如果它们在可传输集中不是自包含的）。对于TSPITR，如果需要严格版本的transportable，则该参数应设置为TRUE 。默认情况下，该参数设置为FALSE 。它只会将OUT指针视为违规。

例子

如果视图不返回任何行，则表空间集是自包含的。例如，

SQLPLUS> EXECUTE DBMS_TTS.TRANSPORT_SET_CHECK（'foo，bar'，TRUE）; SQLPLUS> SELECT * FROM TRANSPORT_SET_VIOLATIONS;
