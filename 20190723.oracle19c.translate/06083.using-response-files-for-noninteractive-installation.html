一个使用响应文件进行非交互式安装

以下主题介绍如何使用响应文件安装和配置Oracle产品。

  • 介绍

  • 创建oraInst.loc文件

  • 准备响应文件

  • 以静默或抑制模式运行Oracle Universal Installer

介绍

通过在启动Oracle Universal Installer时指定响应文件，可以完全或部分自动完成Oracle软件的安装和配置。Oracle Universal Installer使用响应文件中包含的值来提供部分或全部Oracle Universal Installer提示的答案：

  • 如果包含响应文件中所有提示的响应，并在启动Oracle Universal Installer时指定-silent选项，则Oracle Universal Installer将以静默方式运行。在静默模式安装期间，Oracle Universal Installer不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。

  • 如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则Oracle Universal Installer将以抑制模式运行。在抑制模式安装期间，Oracle Universal Installer仅显示未指定所有必需信息的屏幕。您还可以在响应文件或命令行选项中使用变量来禁止不提示输入信息的其他安装程序屏幕，例如“欢迎”屏幕或“摘要”屏幕。

下表描述了您可能希望以静默模式或抑制模式运行Oracle Universal Installer的几个原因：

模式 用途

无声

如果您想要使用静音模式：

  • 完成无人参与安装，您可以使用操作系统实用程序（如at

  • 在多个系统上完成几个类似的安装，无需用户交互

  • 在没有安装X Window System软件的系统上安装软件

Oracle Universal Installer在您用于启动它的终端中显示进度信息，但它不显示任何Oracle Universal Installer屏幕。

抑制

如果要在多个系统上完成类似的Oracle软件安装，请使用抑制模式，为某些（但不是全部）Oracle Universal Installer提示提供默认答案。

如果未在响应文件中指定特定安装程序屏幕所需的信息，Oracle Universal Installer将显示该屏幕。它会抑制您提供了所有必需信息的屏幕。

安装概述

要在静默或抑制模式下使用Oracle Universal Installer安装和配置Oracle产品，请按照下列步骤操作：

  1. 创建oraInst.loc文件。
  2. 准备响应文件。
  3. 以静默或抑制模式运行Oracle Universal Installer。

以下各节介绍了这些步骤。

创建oraInst.loc文件

如果计划在静默或禁止模式下使用Oracle Universal Installer安装Oracle产品，则必须手动创建oraInst.loc文件（如果该文件尚不存在）。此文件指定Oracle Inventory目录的位置，Oracle Universal Installer在该目录中创建系统上安装的Oracle产品的清单。

注意：

如果以前在系统上安装了Oracle软件，则oraInst.loc文件可能已存在。如果文件存在，则无需创建文件。

要创建oraInst.loc文件，请按照下列步骤操作：

  1. 将用户切换到root：
    
                     
    2. 
                     
  2. 在Solaris（SPARC）上，如果它不存在，请创建/var/opt/oracle目录：
    #mkdir / var / opt / oracle
    
                     
    3. 
                     
  3. 根据您的操作系统更改目录，如下所示：
    
                           
    AIX：
    #cd / etc
    Solaris（SPARC）：
    #cd / var / opt / oracle
    
                     
    4. 
                     
  4. 输入以下命令以在oraInst.loc文件上设置适当的所有者，组和权限：
    #chown oracle：oinstall oraInst.loc #chmod 664 oraInst.loc
    
                     
    5.

编辑响应文件模板

                  

                     

                        
Oracle为每种产品和安装类型以及每种配置工具提供响应文件模板。Oracle Database Gateway， tg.rsp和netca.rsp的响应文件位于介质的response目录中。
                        

                        

                           
注意：

                           
如果将软件复制到硬盘，则响应文件位于Disk1/response目录中。
                           

                        

                        
准备响应文件：

                     

                     
                     
    
                        
  1. 将响应文件从响应文件目录复制到系统上的目录：
    $ cp / directory_path / response / response_file .rsp local_directory

    在此示例中， directory_path是CD-ROM安装点目录或DVD上的目录。如果已将软件复制到硬盘驱动器，则可以根据需要编辑response目录中的文件。
                              
    
                           
    
                        
    2. 
                        
  2. 在文本编辑器中打开响应文件：
    $ vi / local_dir / response_file .rsp
    
                        
    3. 
                        
  3. 按照文件中的说明编辑文件。
    
                              
    
                                 
    注意：
    
                                 
    如果未正确配置响应文件，Oracle Universal Installer或配置助手将失败。有关对失败的静默模式安装进行故障排除的详细信息，请参阅静默模式响应文件错误处理 。
                                 
    
                              
    
                           
    
                        
    4. 
                        
  4. 将文件的权限更改为700：
    $ chmod 700 / local_dir / response_file .rsp
    
                        
    5. 
                     

                  

               
记录响应文件

                  

                     

                        
此方法对于自定义或纯软件安装最有用。

                        
您可以在交互模式下使用Oracle Universal Installer来记录可以编辑的响应文件，然后用于完成静默模式或抑制模式安装。在记录响应文件时，您可以完成安装，也可以在开始将软件复制到系统之前从“摘要”页面退出Oracle Universal Installer。

                        
要记录新的响应文件：

                     

                     
                     
    
                        
  1. 完成相应主题中列出的预安装任务。
    
                              
    当您运行Oracle Universal Installer来记录响应文件时，它会检查系统以验证它是否满足安装软件的要求。因此，Oracle建议您在完成安装时完成所有必需的安装前任务并记录响应文件。
    
                           
    
                        
    2. 
                        
  2. 如果您之前未在此系统上安装Oracle软件，请创建oraInst.loc文件，如上一节中所述。
    3. 
                        
  3. 确保Oracle软件所有者用户（通常是oracle ）有权创建或写入您在运行Oracle Universal Installer时指定的Oracle主路径。
    4. 
                        
  4. 要记录响应文件，请输入类似于以下内容的命令以启动Oracle Universal Installer：
    
                              
    
                                 
    注意：
    
                                 
    不要指定响应文件的相对路径。如果指定相对路径，Oracle Universal Installer将失败。
    
                              
    $ / directory_path / runInstaller -record -destinationFile filename

    在前面的示例中：
    
                              
      
                                 
    • 
                                    
      directory_path是CD-ROM安装点目录，DVD上目录的路径或硬盘驱动器上Disk1目录的路径
      
                                 
      • 
                                 
    • 
                                    
       -record参数指定您要记录在响应文件中输入的响应
      
                                 
      • 
                                 
    • 
                                    
      filename是要记录的响应文件的完整路径和文件名
      
                                 
      • 
                              
    
                           
    
                        
    5. 
                        
  5. 在每个安装程序屏幕上，指定所需信息。
    6. 
                        
  6. 当Oracle Universal Installer显示“摘要”屏幕时，请执行以下操作之一：
    
                              
      
                                 
    • 
                                    
      单击“ 安装”以创建响应文件，然后继续安装。
                                    
      
                                 
      • 
                                 
    • 
                                    
      单击取消 ，然后单击是以创建响应文件，但在不安装软件的情况下退出Oracle Universal Installer。
                                    
      
                                 
      • 
                              
    
                              
    响应文件保存在您使用-destinationFile选项指定的位置。
                              
    
                           
    
                        
    7. 
                        
  7. 如果未完成安装，请使用在“指定文件位置”屏幕上指定的路径删除Oracle Universal Installer创建的Oracle主目录。
    8. 
                        
  8. 在其他系统上使用录制的响应文件之前，请使用文本编辑器编辑文件并进行所需的更改。
    
                              
    编辑时，请使用文件中的注释作为指导。
    
                           
    
                        
    9. 
                     

                  

               

            
以静默或抑制模式运行Oracle Universal Installer

               

                  

                     
要以静默或抑制模式运行Oracle Universal Installer，请按照下列步骤操作：

                  

                  
                  
    
                     
  1. 完成相应主题中列出的预安装任务。
    2. 
                     
  2. 以Oracle软件所有者用户（通常是oracle ）身份登录。
    3. 
                     
  3. 要以静默或抑制模式启动Oracle Universal Installer，请输入类似于以下内容的命令：
    $ $ / directory_path / runInstaller -silent -noconfig -responseFile filename
    
                              
    注意：
    
                              
    不要指定响应文件的相对路径。如果指定相对路径，Oracle Universal Installer将失败。
    
                           
    
                           
      
                              
      • 
                           
    
                           
    在这个例子中：
    
                           
      
                              
    • 
                                 
      directory_path是安装介质装入点目录，DVD上目录的路径或硬盘驱动器上Disk1目录的路径。
                                 
      
                              
      • 
                              
    • 
                                 
       -silent表示您要以静默方式运行Oracle Universal Installer。
                                 
      
                              
      • 
                              
    • 
                                 
      -noconfig禁止在安装期间运行配置助手，而是执行纯软件安装。
                                 
      
                              
      • 
                              
    • 
                                 
      filename是您配置的安装响应文件的完整路径和文件名。
                                 
      
                              
      • 
                           
    
                           
    
                              
    注意：
    
                              
    有关runInstaller命令的其他选项的更多信息，请输入以下命令：
    $ / directory_path / runInstaller -help
    
                        
    
                     
    4.
