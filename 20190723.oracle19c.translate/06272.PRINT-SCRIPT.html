2.28打印脚本

目的

使用PRINT SCRIPT命令将本地或全局存储的脚本打印到标准输出或文件。

先决条件

仅在RMAN提示符下执行PRINT SCRIPT 。RMAN必须连接到目标数据库和恢复目录。必须打开恢复目录数据库。

如果指定的脚本是本地脚本，则必须将RMAN连接到创建或替换脚本时连接到的目标数据库。

使用说明

如果未指定GLOBAL ，则RMAN将查找要打印的本地或全局脚本script_name 。如果找到本地脚本，则打印它。如果未找到本地脚本，但找到了全局脚本script_name ，则会打印全局脚本。

句法

printScript :: =


printcript.eps插图的描述

语义

语法元素 描述

全球

将脚本标识为全局。

如果指定了GLOBAL ，则恢复目录中必须存在具有指定名称的全局脚本。否则，RMAN返回错误RMAN-06004 。

另请参阅： “ 使用说明 ” ，以解释全局脚本和本地脚本之间的区别

SCRIPT_NAME

指定要打印的脚本的名称。当名称包含空格或保留字时，必须在脚本名称周围使用引号。

TO FILE ' filename '

将输出发送到指定的文件而不是标准输出。

例子

示例2-114将脚本打印到文件

此示例将脚本打印到文件/tmp/global_backup_db.rman ：

RMAN> PRINT GLOBAL SCRIPT global_backup_db TO FILE“/tmp/global_backup_db.rman”;

示例2-115将脚本打印到屏幕

此示例将存储的脚本打印到标准输出（并包括示例输出）：

RMAN> PRINT SCRIPT backup_whole;打印存储脚本：backup_whole {BACKUP INCREMENTAL LEVEL 0 TAG backup_whole FORMAT“/ disk2 / backup /％U”DATABASE PLUS ARCHIVELOG; }
