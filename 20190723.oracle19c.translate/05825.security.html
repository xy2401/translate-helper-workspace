8安全

Oracle OLAP使用Oracle数据库的标准安全机制来保护您的数据。

本章包含以下主题：

  • Oracle数据库中多维数据的安全性

  • 设置对象安全性

  • 在维度和多维数据集上创建数据安全策略

  • 创建OLAP数据安全角色

8.1 Oracle数据库中多维数据的安全性

贵公司的数据是宝贵的资产。信息必须是安全，私密和受保护的。分析数据特别容易受到攻击，因为它组织严密，易于导航，并归纳为有意义的测量单位。

使用Oracle OLAP时，数据将存储在数据库中。它具有Oracle数据库的安全优势，可以提高业界的安全性。您无需通过将数据传输到独立数据库来公开数据。您无需在单独的系统上管理安全性。而且，您不需要通过将数据存储在比Oracle数据库更不安全的环境中来破坏数据。

8.1.1安全管理

因为您只有一个系统需要管理，所以您不必复制以下基本安全任务：

  • 创建用户帐户

  • 创建和管理密码保护规则

  • 保护网络连接

  • 检测并消除安全漏洞

  • 保护系统免受入侵者的侵害

数据安全的基础是管理用户帐户和角色。用户使用用户名和密码打开与Oracle数据库的连接，并且他们可以访问同一会话中的维度和关系对象。

8.1.2安全类型

默认情况下，用户对分析工作区或其他用户的架构中的任何其他数据类型没有访问权限。所有者或管理员必须授予他们或他们所属的角色任何访问权限。

Oracle OLAP提供两种类型的安全性：对象安全性和数据安全性。

  • 对象安全性提供对维度对象的访问。必须先设置对象安全性，然后其他用户才能访问它们。使用SQL GRANT和REVOKE实现对象安全性。

  • 数据安全性可在细胞级别上提供对数据的细粒度控制。这种安全性是可选的。仅当您要限制对多维数据集的特定区域的访问时，才必须定义数据安全策略。使用Oracle Real Application Security实现数据安全性。

    注意：

    只有模式的所有者才能创建数据安全策略和OLAP数据安全角色。数据安全策略和OLAP数据安全角色仅适用于架构中的对象。

您可以在Analytic Workspace Manager中管理数据安全性和对象安全性。对于对象安全性，您还可以选择使用SQL GRANT和REVOKE 。

8.1.3关于权限

使用对象安全性和数据安全性，您可以授予和撤消以下权限：

  • 更改 ：更改多维数据集或维度的定义。用户需要此权限才能创建和修改维模型。

  • 删除 ：删除旧维度成员。用户需要此权限才能刷新维度。

  • 插入 ：添加新维度成员。用户需要此权限才能刷新维度。

  • 选择 ：查询多维数据集或维度。用户需要此权限才能查询多维数据集或维度的视图或使用CUBE_TABLE函数。CUBE_TABLE是一个返回维对象值的SQL函数。

  • 更新 ：更改多维数据集的数据值或维度成员的名称。用户需要此权限才能刷新维度或多维数据集。

用户通过使用Analytic Workspace Manager创建和管理维对象，或使用SQL查询它们来行使这些权限。它们不直接在多维数据集和维度上发出SQL INSERT和UPDATE等命令。

8.1.4分层安全

对于维对象，您可以在以下级别管理安全性：

  • 维度成员

  • 尺寸

  • 立方体

  • 分析工作区

  • 视图

  • 物化视图

权限是分层的，例如，具有软件产品上的SELECT数据安全性的用户还必须在PRODUCT维度和全局分析工作区上具有SELECT对象安全性。用户还需要对维对象的视图具有SELECT权限。

您可以使用与数据库中任何其他视图和实体化视图相同的方式管理维度和维度视图的物化视图。

8.2设置对象安全性

您可以使用SQL或Analytic Workspace Manager来设置对象安全性。结果是一样的。以下主题描述了这些方法。

  • 使用SQL设置对象安全性

  • 使用Analytic Workspace Manager设置对象安全性

8.2.1使用SQL设置对象安全性

您可以使用SQL GRANT和REVOKE命令在维对象上设置和撤消对象权限。

8.2.1.1在分析工作区上设置对象安全性

分析工作区上的对象权限只是打开容器。您必须为多维数据集和维度授予对象权限，以便用户能够访问它们。表名与分析工作区名称相同，但添加了AW$前缀。

以下命令使Scott能够将Global分析工作区AW$GLOBAL附加到会话：

在全球范围内进行GRANT SELECT TO scott;
8.2.1.2在维度上设置对象安全性

您可以为各个维度授予权限，以使用户能够查询维度成员和属性。要让用户查询多维数据集，他们必须拥有多维数据集的每个维度的权限。

权限适用于整个维度。但是，您可以在维度上设置细粒度访问以限制权限，如“ 在维度和多维数据集上创建数据安全策略 ”中所述 。

    分析工作区上的权限不会自动扩展到分析工作区中包含的多维数据集和维度。

8.2.2.3在多维数据集上设置对象安全性

在多维数据集上设置对象安全性之前，请在多维数据集使用的维上设置对象安全性。在Analytic Workspace Manager中，按照以下步骤在多维数据集上设置对象安全性：

8.3在维度和多维数据集上创建数据安全策略

通过数据安全策略，您可以为选定的维成员授予数据库用户和角色权限。例如，您可以将区域销售经理限制为仅针对其自己的区域而非所有地理区域的数据。您可以在维，多维数据集或两者上创建数据安全策略：

  • 只有架构的所有者才能为架构中的维和多维数据集创建数据安全策略。

  • 在维度上创建数据安全策略时，策略将扩展到具有该维度的所有多维数据集。您无需为每个多维数据集重新创建策略。

  • 在多维数据集上创建数据安全策略时，可以为多维数据集的每个维度选择成员。该策略仅适用于该多维数据集。

  • 在维度和多维数据集上创建数据安全策略时，用户对策略重叠的数据范围最窄的部分具有特权。

授予数据权限

您可以将策略应用于一个或多个数据库用户和角色。您还可以将策略应用于OLAP数据安全角色。OLAP数据安全角色是一组数据库用户和角色，您可以在Analytic Workspace Manager中管理这些用户和角色，以便在安全策略中使用。您可以在Analytic Workspace Manager中创建OLAP数据安全角色和数据安全策略。

按标准选择数据

定义数据安全策略时，您可以根据维层次结构选择特定维度成员或满足特定标准的成员。通过使用标准而不是硬编码特定维度成员，选择在数据刷新后仍然有效。添加成员后，无需修改选择。例如，当旧产品被淘汰并且新产品被添加到PRODUCT维度时，授予所有硬件产品SELECT权限的安全策略仍然有效。

注意：

你必须拥有 OLAP_XS_ADMIN角色，用于管理Analytic Workspace Manager中的数据安全策略。

要创建数据安全策略：

  1. 展开维度或多维数据集的文件夹。

  2. 右键单击“数据安全”，然后选择“ 创建数据安全策略

    将出现“创建数据安全策略”对话框。

  3. 在“常规”选项卡上，在“数据安全策略名称”字段中输入描述性名称。

  4. 可选 ：在“描述”字段中输入描述。

  5. 对于维度，请选择要用于选择可查看维度成员的方法，“ 成员选择”或“ OLAP DML表达式” 。相关选项卡变为活动状态。

    对于多维数据集，该方法是成员选择。

  6. 单击“ 添加用户或角色” 。

    将出现“添加用户或角色”对话框。

  7. 选择数据库用户和角色以及OLAP数据安全角色以使用此策略。然后单击“ 确定”关闭对话框。

    现在，“常规”选项卡上的表中列出了所选的数据库用户和角色以及OLAP数据安全角色。

  8. 选择要授予每个用户或角色的权限。您无法为OLAP数据安全角色分配权限，因为权限是其定义的一部分。

  9. 对于多维数据集，请填写“成员选择”选项卡

    对于维度，请完成“成员选择”选项卡或“OLAP DML表达式”选项卡，具体取决于先前选择的方法。

  10. 单击“ 创建”以保存数据安全策略。

    数据安全策略显示在维度或多维数据集的Data Security文件夹的导航树中。

  11. 在维或多维数据集以及分析工作区上授予这些用户和角色对象权限。

8.4创建OLAP数据安全角色

您可以创建OLAP数据安全角色，以管理要为其分配相同数据访问权限的一组用户。然后，您可以在管理数据安全策略时使用数据安全角色，而不是定义每个用户的权限。OLAP数据安全角色与数据库角色类似，只是它们仅在OLAP数据安全性的上下文中运行，并且它们可以由具有较低功能的数据库权限的用户创建。只有模式的所有者才能在模式中创建数据安全角色。

注意：

您必须具有OLAP_XS_ADMIN权限才能在Analytic Workspace Manager中管理数据安全策略。

要创建OLAP数据安全角色：

  1. 在导航树中，右键单击“数据安全角色”，然后选择“ 创建数据安全角色” 。

    将出现“创建数据安全角色”对话框。

  2. 在“常规”选项卡上，在“数据安全性角色名称”字段中输入描述性名称。
  3. 可选 ：在“描述”字段中输入描述。
  4. 单击“ 添加用户或角色” 。

    将出现“添加用户或角色”对话框。

  5. 选择要包含在此OLAP数据安全角色中的用户和角色。然后单击“ 确定”关闭此对话框。

    现在，“常规”选项卡上的表中列出了所选用户和角色。

  6. 选择要授予每个用户或角色的权限。
  7. 单击“ 创建”以保存OLAP数据安全角色。

    新的OLAP数据安全角色将显示在“数据安全角色”文件夹的导航树中。
