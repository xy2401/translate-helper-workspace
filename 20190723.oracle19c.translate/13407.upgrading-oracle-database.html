4升级Oracle数据库

Oracle提供了一套全面的工具，用于以最短的停机时间升级Oracle数据库，以及将应用程序迁移到新版本。

警告：

如果您保留旧的Oracle软件，则永远不要使用旧的Oracle软件启动升级的数据库。仅使用新Oracle数据库安装中的可执行文件启动数据库。

话题：

  • 备份Oracle数据库以进行升级
    在尝试升级之前，请使用此过程备份现有Oracle数据库。
  • 使用并行升级实用程序升级（catctl.pl和dbupgrade Shell命令）
    本节介绍如何使用并行升级实用程序（ catctl.pl ）使用并行处理，包含和排除列表以及管理升级的其他功能来运行手动升级。
  • 使用Oracle数据库升级助手（DBUA）进行升级
    数据库升级助手（DBUA）提供图形用户界面，指导您完成Oracle数据库的升级。DBUA适用于CDB和非CDB数据库系统。
  • 非CDB Oracle数据库的升级方案
    查看这些主题以了解非CDB Oracle数据库的升级方案和过程
  • Windows非CDB Oracle数据库的手动升级示例
    这些示例显示了完成Oracle Database 11g第2版（11.2.0.4）升级到Oracle Database 19c的预升级检查，升级和升级后检查的步骤。
  • 多租户架构Oracle数据库的手动升级方案
    要准备手动升级，请查看使用多租户体系结构部署的Oracle数据库的手动升级方案和过程。
  • 数据字典升级和升级状态显示的改进
    Oracle Database 12 c包括对升级过程的改进，以及升级数据库的升级状态。
  • 关于Oracle数据库早期版本中的Dbupgrade脚本和catupgrd.sql
    catupgrd.sql脚本的功能由并行升级实用程序catctl.pl以及dbupgrade和dbupgrade.cmd脚本替换。
  • 关于传输和升级数据库（完全可传输导出/导入）
    您可以使用基于文件或非基于文件的模式来传输数据。
  • Oracle数据库升级故障排除
    使用这些故障排除提示可以解决升级数据库时可能遇到的错误或问题。
  • 重新启动Oracle数据库升级
    使用这些选项重新运行升级。
  • 从使用-p失败的特定阶段重新启动升级
    修复错误后，使用此选项完成升级。
