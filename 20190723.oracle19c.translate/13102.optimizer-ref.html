OTHER_TAG

VARCHAR2（255）

描述OTHER列的内容。价值观是：

  • SERIAL （空白）：串行执行。目前，在这种情况下，不会在OTHER列中加载SQL。

  • SERIAL_FROM_REMOTE (S -> R) ：远程站点的串行执行。

  • PARALLEL_FROM_SERIAL (S -> P) ：串行执行。步骤的输出被分区或广播到并行执行服务器。

  • PARALLEL_TO_SERIAL (P -> S) ：并行执行。步骤输出返回到串行QC过程。

  • PARALLEL_TO_PARALLEL (P -> P) ：并行执行。步骤的输出被重新分配给第二组并行执行服务器。

  • PARALLEL_COMBINED_WITH_PARENT (PWP) ：并行执行;步骤输出在同一并行过程中进入下一步。没有与父进程的进程间通信。

  • PARALLEL_COMBINED_WITH_CHILD (PWC) ：并行执行。步骤的输入来自同一并行过程中的先前步骤。没有来自孩子的进程间通信。

PARTITION_START

VARCHAR2（255）

启动一系列已访问分区的分区。它可以采用以下值之一：

n表示SQL编译器已识别出起始分区，其分区号由n给出。

KEY表示在运行时通过分区键值识别起始分区。

ROW REMOVE_LOCATION表示数据库在运行时从每个检索到的记录的位置计算起始分区（与停止分区相同）。记录位置由用户或全局索引获得。

INVALID表示访问的分区范围为空。

PARTITION_STOP

VARCHAR2（255）

停止一系列访问的分区的分区。它可以采用以下值之一：

n表示SQL编译器已识别出停止分区，其分区号由n给出。

KEY表示在运行时通过分区键值识别停止分区。

ROW REMOVE_LOCATION表示数据库在运行时从每个检索到的记录的位置计算停止分区（与起始分区相同）。记录位置由用户或全局索引获得。

INVALID表示访问的分区范围为空。

分区（ROWID）

使用行的rowid将表或索引分区为UPDATE / DELETE ，将行映射到查询服务器。

分区（关键）

根据使用一组列对表或索引进行分区，将行映射到查询服务器。用于部分分区连接， PARALLEL INSERT ， CREATE TABLE AS SELECT分区表的CREATE TABLE AS SELECT以及CREATE PARTITIONED GLOBAL INDEX 。

HASH

使用连接键上的哈希函数将行映射到查询服务器。用于PARALLEL JOIN或PARALLEL GROUP BY 。

范围

使用排序键的范围将行映射到查询服务器。在语句包含ORDER BY子句时使用。

循环赛

随机将行映射到查询服务器。

广播

将整个表的行广播到每个查询服务器。当一个表与另一个表相比非常小时，用于并行连接。

质量控制（订单）

QC从第一个查询服务器到最后一个查询服务器按顺序使用输入。在语句包含ORDER BY子句时使用。

QC（随机）

QC随机消耗输入。在语句没有ORDER BY子句时使用。
