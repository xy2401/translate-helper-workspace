CLSCH-01001：Oracle集群运行状况分析服务（OCHAD）在主机{0}上启动。

原因： Oracle Cluster Health Analysis Service（OCHAD）已成功启动。

行动：没有

CLSCH-01002：Oracle集群运行状况分析服务（OCHAD）在主机{0}上中止。{1}中的详细信息。

原因： Oracle Cluster Health Analysis Service（OCHAD）由于内部错误而中止。检查OCHAD日志文件以确定原因。

操作：使用“srvctl status cha -node node name ”命令确定Oracle Cluster Health Analysis Service（OCHAD）是否在消息中列出的节点上处于联机状态。如果OCHAD处于脱机状态，请使用'srvctl stop cha -node node name -f'命令然后使用'srvctl start cha -node node name '命令重新启动OCHAD。如果问题仍然存在，请与Oracle支持服务联系。

CLSCH-01003：Oracle集群运行状况分析服务（OCHAD）无法在主机{0}上进行校准。{1}中的详细信息。

原因： Oracle Cluster Health Analysis Service（OCHAD）由于内部错误而无法校准。

操作：检查OCHAD校准日志文件以确定原因。如果问题仍然存在，请与Oracle支持服务联系。

CLSCH-01004：Oracle集群运行状况分析服务（OCHAD）中止对主机{2}上的{0} {1}的监视。{3}中的详细信息。

原因： Oracle Cluster Health Analysis Service（OCHAD）因内部错误而中止对目标的监视。

操作：联系Oracle支持服务。

CLSCH-01005：可用保留时间低于最小{0}小时。Oracle Cluster Health Analysis Service（OCHAD）正在停止对所有数据库的监视。

原因： Oracle Cluster Health Analysis Service（OCHAD）已停止监视所有数据库，因为可用保留时间低于允许的最短保留时间。

操作：使用命令“chactl resize repository”重新启动监视较少的数据库或增加存储库大小，然后重新启动监视数据库。要重新启动监视数据库，请使用命令“chactl monitor database”。

CLSCH-02002：没有校准数据。校准失败。

原因：在指定的时间范围内没有可用的校准数据。

操作：使用有效时间范围重新发出'oclumon calibrate'命令。有关该命令的信息，请参阅产品文档或联机帮助。

CLSCH-02004：Oracle Cluster Health Analysis Service（OCHAD）无法从本地实例“{0}”检索校准数据。

原因：连接到或从本地数据库实例检索数据时发生错误。

操作：执行以下步骤：1）检查随附的错误消息。如果可能，请更正报告的问题。2）使用'srvctl status database'命令检查目标数据库的本地实例的状态。如果目标数据库脱机，请发出'srvctl start database'命令。从正在运行目标数据库实例的节点重新发出'oclumon calibrate'命令。3）如果您无法确定或更正问题，请联系Oracle支持服务。

CLSCH-02005：由于Grid Infrastructure Management Repository错误，Oracle Cluster Health Analysis Service（OCHAD）失败。

原因：发生了网格基础结构管理存储库错误。

操作：执行以下步骤：1）检查随附的错误消息。如果可能，请更正报告的问题。2）发出'srvctl status mgmtdb'命令以确定Grid Infrastructure Management Repository的状态。如果Grid Infrastructure Management Repository处于脱机状态，请发出'srvctl start mgmtdb'命令以启动它。如果无法启动，请检查数据库实例警报日志以确定发生的情况。3）如果您无法确定或更正问题，请联系Oracle支持服务。
