1.185 MAX_DATAPUMP_JOBS_PER_PDB

MAX_DATAPUMP_JOBS_PER_PDB确定每个PDB的最大并发Oracle Data Pump作业数。

属性 描述

参数类型

串

句法

MAX_DATAPUMP_JOBS_PER_PDB = {AUTO |整数 }

默认值

100

可修改

改变系统

可在PDB中修改

是

价值范围

0到250或AUTO

基本

没有

Oracle RAC

必须在所有实例上使用相同的值。

默认值不适用于所有数据库。数据库管理员必须确定默认值是否适用于其数据库。

当此参数的值为AUTO ，Oracle Data Pump将其实际值派生为SESSIONS初始化参数的50％。

过大的值可能导致Oracle Data Pump占用过多的系统资源，而过小的值可能会阻止用户执行其Oracle Data Pump任务。

Oracle Data Pump使用的主要资源是数据库的系统全局区域（SGA）中的共享池。并行作业会增加会话数，并根据作业增加使用的PQ从站数。
