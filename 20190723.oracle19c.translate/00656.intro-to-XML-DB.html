Oracle XML DB包括以下功能：

  • 用于XML数据的抽象SQL数据类型XMLType 。

  • 用于XML内容的企业级Oracle数据库功能：可靠性，可用性，可伸缩性和安全性。特定于XML的内存管理和优化。

  • 访问和更新XML数据的行业标准方法。您可以使用FTP，HTTP（S）和WebDAV将XML内容移入和移出Oracle数据库。行业标准API使用Java，C和PL / SQL提供对XML内容的编程访问和操作。

  • 使用SQL和XQuery访问XML数据时存储，查询，更新和转换XML数据的方法。

  • 对SQL数据执行XML操作的方法。

  • Oracle XML DB存储库：一个简单，轻量级的存储库，您可以使用文件/文件夹/ URL隐喻来组织和管理数据库内容，包括XML内容。

Oracle XML DB功能部分基于相关XML标准的Oracle XML Developer的Kit C实现，例如XML Parser，XSLT Virtual Machine，XML DOM和XML Schema Validator。

1.2.2数据二元性：XML和关系

Oracle XML DB提供了对称的双重数据视图：XML和关系。

Oracle XML DB的一个关键特性是，它允许您使用XML数据，就好像它是关系数据和关系数据一样，就像它是XML数据一样。在处理XML内容时，您可以利用关系模型的强大功能，并且在处理关系内容时可以利用XML的灵活性。您可以针对特定业务问题的不同方面使用最合适的工具。

这种二元性意味着相同的数据可以作为表中的行公开，并使用SQL进行操作，或者作为XML文档中的节点公开，并使用XQuery，DOM或XSL转换进行操作。访问和处理技术独立于底层存储方法。

这些功能可以解决常见的业务问题：

  • 您可以直接从SQL查询生成XML数据。您可以使用数据库驻留的XSLT处理器将XML数据转换为其他格式，例如HTML。

  • 您可以使用SQL查询，联机分析处理（OLAP）和业务智能/数据仓库操作来访问XML内容，而无需在不同数据格式之间进行转换。

  • 您可以对XML内容执行文本和空间数据操作。

  • 如果您的数据不是XML，请使用XMLType视图
    XMLType视图为您提供了一种以XML格式包装现有关系数据或对象关系数据的方法。

XML的层次性质为传统的关系数据库带来了一些挑战：

  • 在关系数据库中， 表行隐喻定位内容。主键外键关系有助于定义内容之间的关系。使用table-row-column隐喻访问和更新内容。

  • 另一方面，XML使用分层技术来实现相同的功能。URL用于查找XML文档。基于URL的标准（如XLink）用于定义XML文档之间的关系。W3C建议（如XPath）用于访问和更新XML文档中包含的内容。URL和XPath表达式都基于层次结构隐喻。URL使用文件夹层次结构中的路径来标识文档，而XPath使用通过XML文档的节点层次结构的路径来访问XML文档的一部分。

Oracle XML DB通过引入允许使用以XML为中心的隐喻的SQL函数和方法来解决这些挑战，例如XQuery和XPath表达式，用于查询和更新XML文档。

1.4.1 XMLType数据类型

使用XMLType ，XML开发人员可以在关系数据库的上下文中利用XML标准的强大功能，SQL开发人员可以在处理XML数据时利用关系数据库的强大功能。

XMLType是XML数据的抽象本机SQL数据类型。它提供了PL / SQL和Java构造函数，用于从VARCHAR2 ， CLOB ， BLOB或BFILE实例创建XMLType实例。它为各种XML操作提供PL / SQL方法。

您可以像使用任何其他SQL数据类型一样使用XMLType 。例如，您可以在关系表中创建XMLType表或视图或XMLType列。

您可以在PL / SQL存储过程中使用XMLType来获取参数，返回值和变量。

您还可以使用Java和C语言的应用程序编程接口（API）来操作XMLType数据，包括Java数据库连接（JDBC），XQuery for Java（XQJ）和Oracle Data Provider for .NET（ODP.NET）。

XMLType是Oracle数据库对象类型，因此您还可以创建XMLType对象实例的表。默认情况下， XMLType表或列可以包含任何格式良好的XML文档。

您可以约束XMLType表或列以符合XML模式，在这种情况下，数据库确保只有针对XML模式验证的XML数据才会存储在列或表中。无效文件被排除在外。

1.4.2 XMLType存储模型

XMLType是一种抽象数据类型，它提供了不同的存储模型，以最好地适应您的数据及其使用。作为抽象数据类型，您的应用程序和数据库查询可以获得灵活性：所有XMLType操作都可以使用相同的接口。

XMLType表和列可以通过以下方式存储：

  • 二进制XML存储（默认） - XMLType数据以专为XML数据设计的解析后二进制格式存储。二进制XML是紧凑的，解析后的，XML模式感知的XML数据。这也称为解析后持久性 。

  • 对象关系存储 - XMLType数据存储为一组对象。这也称为结构化存储和基于对象的持久性 。

注意：

从Oracle Database 12c第1版（12.1.0.1）开始， 不推荐使用 XMLType的非结构化（ CLOB ）存储模型。请改用二进制XML存储。

1.4.3 Oracle XML DB中的XML模式支持

支持万维网联盟（W3C）XML模式建议书是Oracle XML DB中的一项关键功能。

XML Schema指定XML文档的结构，内容和某些语义。它在http://www.w3.org/TR/soap12-part0/中有详细描述。

W3C模式工作组发布了一个特定的XML模式，通常称为模式的模式 ，它提供XML模式语言的定义或词汇表。XML模式定义 （ XSD Foot 2 ），也称为XML模式 ，是一种XML文档，它符合模式模式定义的词汇表。

XML模式使用模式为模式定义的词汇表来创建XML模式类型定义和元素声明的集合，这些声明包含用于描述新XML文档类的内容和结构的词汇表，符合该XML的XML实例文档架构。

注意：

本手册使用术语“XML模式”（小写“s”）来引用符合W3C XML Schema（大写“S”）建议书的任何XML模式。由于XML模式用于定义XML文档类，因此术语“实例文档”通常用于描述符合特定XML模式的XML文档。

XML Schema语言提供了强大的元素和属性类型。它定义了许多标量数据类型。可以使用诸如继承和扩展之类的面向对象技术来扩展此基本数据类型集以定义更复杂的类型。XML Schema词汇表还包括可用于定义复杂类型，替换组，重复集，嵌套，排序等的构造。Oracle XML DB支持XML Schema Recommendation定义的所有构造，但redefines除外。

XML模式通常用作检查（验证）XML实例文档是否符合其规范的机制。Oracle XML DB包含XMLType方法和SQL函数，可用于根据XML模式验证XML文档。

在Oracle XML DB中，您可以为所有数据使用标准数据模型，无论其结构如何。您可以使用XML Schema自动创建用于存储XML数据的数据库表。基于XML模式的数据维护DOM保真度并允许重要的数据库优化。

可以使用Oracle XML DB XMLType存储模型存储基于XML模式的数据：二进制XML存储或对象关系存储。基于非模式的XML数据只能使用二进制XML存储来存储。

您还可以将现有的关系数据和对象关系数据包装为XMLType视图，这些视图可以选择基于XML模式。您可以从传入的XML文档映射到XMLType存储，使用已注册的XML模式指定映射。

1.4.4 Oracle XML DB中的DTD支持

与DTD相比，XML模式通常是定义XML文档结构的更强大的方法。但是，您可以在某种程度上使用Oracle XML DB来使用DTD。

与XML模式一样，A DTD是一组规则，用于定义XML文档的允许结构。DTD是从SGML派生其格式的文本文件。它们可以通过使用DTD元素DOCTYPE或通过DOCTYPE引用使用外部文件与XML文档相关联。

Oracle XML DB使用XML Schema而不是DTD来定义XMLType存储的结构化映射，但XML处理器仍然可以访问和解释您的DTD。

注意：

您可以使用DTD获取其中定义的XML实体。实体是DTD中使用的唯一信息。 Oracle XML DB不使用DTD中的结构和类型信息。

内联DTD定义

当XML实例文档具有内联DTD定义时，在文档解析期间使用该定义。此时完成任何DTD验证和实体声明处理。但是，一旦解析，实体引用将替换为实际值，并且原始实体引用将丢失。

外部DTD定义

如果Oracle XML DB存储在Oracle XML DB存储库中，则它们支持外部DTD定义。需要处理包含外部DTD定义（如/public/flights.dtd的XML文档的应用程序必须首先确保DTD文档存储在路径为/public/flights.dtd Oracle XML DB中。

1.4.7对Oracle XML DB的编程访问（Java，PL / SQL和C）

可以从C，PL / SQL和Java访问所有Oracle XML DB功能。

您可以构建基于Web的应用程序，以各种方式利用Oracle XML DB，包括：

  • 使用servlet和Java Server Pages（JSP）。典型的API使用Java数据库连接（JDBC）访问数据。

  • 使用可扩展样式表语言（XSL）和XML服务器页面（XSP）。典型的API以XML文档的形式访问数据，这些数据使用文档对象模型（DOM）API实现进行处理。

Oracle XML DB支持这种样式的应用程序开发。它提供DOM API的Java，PL / SQL和C实现。使用JDBC的应用程序（例如基于servlet的应用程序）需要事先了解它们正在处理的数据结构。Oracle JDBC驱动程序允许您访问和更新XMLType表和列，并调用访问Oracle XML DB存储库的PL / SQL过程。使用DOM的应用程序（例如基于XSLT转换的应用程序）通常需要较少的数据结构知识。基于DOM的应用程序使用字符串名称来标识内容，并且必须动态遍历DOM树以查找所需信息。为此，Oracle XML DB支持使用DOM API来访问和更新XMLType列和表。编程到DOM API比通过JDBC编程更灵活，但在运行时可能需要更多资源。

1.4.8 Oracle XML DB存储库：概述

Oracle XML DB Repository是Oracle数据库的一个组件，它允许您使用文件/文件夹/ URL隐喻处理XML数据。

Oracle XML DB Repository包含资源 ，可以是文件夹 （目录，容器）或文件。

资源（无论是文件夹还是文件）具有以下属性：

  • 它由路径和名称标识。

  • 它有内容 （数据），可以是XML数据，但不一定是。

  • 除了内容之外，它还有一组系统定义的元数据 （属性），例如Owner和CreationDate 。Oracle XML DB使用此信息来管理资源。

  • 它也可能具有用户定义的元数据 。与系统定义的元数据一样，这是不属于内容但与其相关联的信息。

  • 它有一个关联的访问控制列表 ，用于确定谁可以访问资源以及执行哪些操作。

虽然Oracle XML DB Repository专门处理XML内容，但您可以使用存储库存储除XML之外的其他类型的数据。您可以使用存储库访问存储在Oracle数据库中的任何数据。

您可以通过以下方式访问存储库中的数据：

  • SQL - 使用视图RESOURCE_VIEW和PATH_VIEW

  • 标准协议 - FTP，HTTP（S）和WebDAV

  • PL / SQL - 使用PL / SQL包DBMS_XDB_REPOS

  • Java - 使用适用于Java的Oracle XML DB资源API

除了提供用于访问和操作存储库数据的API之外，Oracle XML DB还为以下存储库服务提供API，这些服务基于IETF WebDAV：

  • 版本控制 - 使用PL / SQL包DBMS_XDB_VERSION

  • ACL安全性 - 使用访问控制列表（ACL）

  • 文件夹 - 使用存储库路径名称

1.5 Oracle XML DB支持的标准

Oracle XML DB支持所有主要的XML，SQL，Java和Internet标准。

这些包括以下内容：

  • W3C XML Schema 1.0建议书。您可以注册XML模式，根据XML模式验证存储的XML内容，或将存储在服务器中的XML约束为XML模式。

  • W3C XQuery 1.0推荐和W3C XPath 2.0推荐。您可以使用XQuery和XPath从HTTP（S）请求或SQL中搜索或遍历存储在数据库中的XML。

  • ANSI / ISO / IEC 9075-14：2011，信息技术 - 数据库语言 - SQL - 第14部分：与XML相关的规范（SQL / XML）。

  • W3C DOM推荐级别1.0和2.0核心。您可以将存储在服务器中的XML检索为XML DOM，以进行动态访问。

  • Java数据库连接（JDBC）API。提供对XML数据的Java访问。

  • XQuery API for Java（XQJ）。使用XQuery提供对XML数据的Java访问。

  • W3C XSL 1.0建议书。您可以使用XSLT在服务器上转换XML文档。

  • 协议支持。您可以使用Oracle Net或标准协议（如HTTP（S），FTP和IETF WebDAV）从Oracle XML DB存储或检索XML数据。

  • Java Servlet 2.2版（除了：servlet WAR文件， web.xml ，不完全支持;只有一个ServletContext ;当前支持一个web-app ;不支持有状态servlet）。

  • Web服务：SOAP 1.1。您可以从SOAP请求访问存储在服务器中的XML。您可以使用WSDL和UDDI使用Oracle XML DB和Oracle融合中间件构建，发布或查找Web服务。您可以对Oracle数据库中存储的XML使用Oracle数据库高级队列IDAP（排队操作的SOAP规范）。

  • W3C XML链接语言（Xlink）1.0推荐。您可以在XML文档之间定义各种类型的链接。

  • W3C XML指针语言（XPointer）推荐和XPointer框架。您可以在单个信息集中包含多个XML文档或片段的内容。
