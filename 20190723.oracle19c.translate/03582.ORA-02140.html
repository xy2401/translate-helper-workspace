ORA-04092：无法在触发器中string

原因：触发器尝试提交或回滚。

操作：重写触发器，使其不提交或回滚。

ORA-04093：触发器中不允许引用LONG类型的列

原因：触发器尝试引用触发表中的长列。

行动：不要引用长列。

ORA-04094：表string 。 string是约束，触发器可能不会修改它

原因：触发器尝试修改限制父SQL语句的某些引用约束的表。

行动：不适用

ORA-04095：触发' string '已存在于另一个表中，无法替换它

原因：无法替换已替换的表上已存在的触发器。

操作：删除具有相同名称的触发器并重新创建它。

ORA-04096：触发' string '有一个WHEN子句太大，限制2K

原因：出于字典存储原因，触发器的'when'子句限制为2K。创建的触发器超出了该大小。

操作：使用较小的'when'子句。注意，触发器主体可用于执行与'when'子句相同的限制动作。

ORA-04097：尝试删除或更改触发器时DDL冲突

原因：尝试在触发器或触发器表上同时执行两个DDL操作。

操作：调查触发器的新状态并重试DDL操作（如果仍然合适）。

ORA-04098：触发' string 。 string '无效且重新验证失败

原因：尝试检索触发器以执行并发现无效。这也意味着触发器的编译/授权失败。

操作：选项用于解决编译/授权错误，禁用触发器或删除触发器。

ORA-04099：触发器' string '有效但不以编译形式存储

原因：尝试检索触发器以执行，并且发现触发器有效，但未存储。这可能意味着升级是从未存储的触发器版本不正确地完成的。

操作：更改编译触发器以创建存储形式的触发器。此外，您可能希望检查是否已完成正确的升级。
