2.1字符集

PL / SQL要处理或存储在数据库中的任何字符数据都必须表示为字节序列。单个字符的字节表示称为字符代码 。一组字符代码称为字符集 。

每个Oracle数据库都支持数据库字符集和国家字符集。PL / SQL也支持这些字符集。本文档解释了PL / SQL如何使用数据库字符集和国家字符集。

2.1.3关于数据绑定整理

排序规则（也称为排序排序）是一组规则，用于在比较和排序两个字符串时确定字符串是否等于，先于或跟随另一个字符串。

不同的排序规则对应于不同口语的规则。对排序规则敏感的操作是比较文本并需要排序规则来控制比较规则的操作。相等运算符和内置函数INSTR是对结果敏感操作的示例。

从Oracle Database 12c第2版（12.2）开始，新架构提供了对应用于字符数据操作的排序规则的控制。在新架构中，归类成为字符数据的属性，类似于数据类型。您现在可以为列声明排序规则，并且引用该列的所有排序规则敏感的SQL操作都会自动应用此排序规则。数据绑定的排序规则功能使用与ISO / IEC SQL标准兼容的语法和语义。

PL / SQL语言对数据绑定的排序规则体系结构的支持有限。假设PL / SQL表达式中处理的所有数据都具有兼容性排序USING_NLS_COMP 。此伪归类指示对归类敏感的运算符的行为方式与先前Oracle数据库发行版中的行为相同。也就是说，会话参数NLS_COMP和NLS_SORT值确定要使用的排序规则。但是，在PL / SQL中动态嵌入或构造的所有SQL语句都完全支持新的体系结构。

在表，视图，物化视图，包，存储过程，存储函数，触发器和类型中添加了一个名为default collation的新属性。单元的默认排序规则确定数据容器的排序规则，例如列，变量，参数，文字和返回值，这些容器在该单元中没有自己的显式排序规则声明。包，存储过程，存储函数，触发器和类型的默认排序规则必须为USING_NLS_COMP 。

有关语法和语义，请参阅DEFAULT COLLATION子句 。

2.2.2.3.2引用的用户定义标识符

带引号的用户定义标识符用双引号括起来。

在双引号之间，允许使用数据库字符集中的任何字符，但双引号，换行符和空字符除外。例如，这些标识符是可接受的：

2.2.3文字

文字是既不由标识符表示也不从其他值计算的值。

例如， 123是整数文字， 'abc'是字符文字，但1+2不是文字。

PL / SQL文字包括所有SQL文字（在Oracle数据库SQL语言参考中描述）和BOOLEAN文字（SQL没有）。BOOLEAN文字是预定义的逻辑值TRUE ， FALSE或NULL 。 NULL表示未知值。

注意：

与Oracle数据库SQL语言参考一样 ，本文档可互换地使用术语字符文字和字符串 。

在PL / SQL中使用字符文字时，请记住：

  • 字符文字区分大小写。

    例如， 'Z'和'z'是不同的。

  • 空白字符很重要。

    例如，这些文字是不同的：

    'abc''abc''abc''abc''ab c'

  • PL / SQL没有行继续符，表示“此字符串在下一个源代码行继续。”如果在下一个源行上继续一个字符串，则该字符串包含一个换行符。

    例如，这个PL / SQL代码：

    BEGIN DBMS_OUTPUT.PUT_LINE（'此字符串在这里打破。“）;结束; /

    打印这个：

    这个字符串打破了。

    如果您的字符串不适合源代码行，并且您不希望它包含换行符，则使用连接运算符（ || ）构造该字符串。

    例如，这个PL / SQL代码：

    BEGIN DBMS_OUTPUT.PUT_LINE（'此字符串'||'不包含换行符。“）;结束; /

    打印这个：

    此字符串不包含换行符。

    有关串联运算符的详细信息，请参阅“ 串联运算符 ” 。

  • '0'到'9'不等于0到9的整数文字。

    但是，因为PL / SQL将它们转换为整数，所以可以在算术表达式中使用它们。

  • 带有零个字符的字符文字的值为NULL ，称为空字符串 。

    但是，此NULL值不是BOOLEAN值NULL 。

  • 普通字符文字由数据库字符集中的字符组成。

    有关数据库字符集的信息，请参阅“ Oracle数据库全球化支持指南” 。

  • 国家字符文字由国家字符集中的字符组成。

    有关国家字符集的信息，请参阅“ Oracle数据库全球化支持指南” 。

2.2.5.1单行注释

单行注释以--开头，并延伸到行尾。

警告：

不要在PL / SQL块中放置单行注释，以便Oracle预编译器程序动态处理。Oracle预编译器程序忽略行尾字符，这意味着单行注释在块结束时结束。

在测试或调试程序时，您可以通过将其作为注释来禁用一行代码。例如：

2.2.5.2多行注释

多行注释以/*开头，以*/结尾，并且可以跨越多行。

您可以使用多行注释分隔符来“注释掉”代码段。这样做时，请注意不要导致嵌套的多行注释。一条多行注释不能包含其他多行注释。但是，多行注释可以包含单行注释。例如，这会导致语法错误：

/ * IF 2 + 2 = 4那么some_condition：= TRUE; / *我们希望这个THEN始终执行* / END IF; * /

这不会导致语法错误：

/ * IF 2 + 2 = 4那么some_condition：= TRUE; -我们希望这个THEN始终执行END IF; * /

2.5标识符的范围和可见性

标识符的范围是PL / SQL单元的区域，您可以从中引用标识符。标识符的可见性是PL / SQL单元的区域，您可以从中引用标识符而不限定它。标识符是声明它的PL / SQL单元的本地标识符。如果该单元具有子单元，则标识符对它们是全局的 。

如果子单元重新声明全局标识符，则在子单元内，两个标识符都在范围内，但只有本地标识符可见。要引用全局标识符，子单元必须使用声明它的单元的名称对其进行限定。如果该单元没有名称，则子单元不能引用全局标识符。

PL / SQL单元不能引用在同一级别的其他单元中声明的标识符，因为这些标识符既不是本地的也不是全局的。

您不能在同一个PL / SQL单元中两次声明相同的标识符。如果这样做，则在引用重复标识符时会发生错误。

您可以在两个不同的单元中声明相同的标识符。标识符表示的两个对象是不同的。改变一个不会影响另一个。

在同一范围内，为标签和子程序指定唯一名称，以避免混淆和意外结果。

2.7表达式

表达式是一个或多个值，运算符和SQL函数的组合，它们计算为值。

表达式始终返回单个值。最简单的表达式，按照日益复杂的顺序，是：

  1. 单个常量或变量（例如， a ）

  2. 一元运算符及其单个操作数（例如， -a ）

  3. 二元运算符及其两个操作数（例如， a+b ）

操作数可以是变量，常量，文字，运算符，函数调用或占位符或其他表达式。因此，表达式可以是任意复杂的。有关表达式语法，请参阅Expression 。

操作数的数据类型确定表达式的数据类型。每次计算表达式时，都会生成该数据类型的单个值。该结果的数据类型是表达式的数据类型。

2.7.9静态表达式

  • 如果表达式是NULL文字，则表达式是静态的。

2.9.4条件编译指令限制

条件编译指令受这些语义限制的约束。

条件编译指令不能出现在模式级用户定义类型的规范中（使用“ CREATE TYPE语句 ” 创建 ）。此类型规范指定类型的属性结构，该结构确定依赖类型的属性结构和从属表的列结构。

警告：

使用条件编译指令更改类型的属性结构可能导致依赖对象“失去同步”或依赖表变得不可访问。Oracle建议您仅使用“ ALTER TYPE语句 ” 更改类型的属性结构。ALTER TYPE语句将更改传播到依赖对象。

如果在模式级类型规范中使用条件编译指令，则编译器会引发错误PLS-00180: preprocessor directives are not supported in this context.

由于所有条件编译器构造都由PL / SQL预处理器处理，因此SQL Parser对存储的PL / SQL单元或匿名块中第一个条件编译指令的位置施加以下限制：

  • 在包规范，包体，类型主体，模式级函数和模式级过程中，在条件编译指令有效之前，必须在单元名称的标识符后面出现至少一个非空白空间PL / SQL标记。 。

    注意：

    • PL / SQL注释“ - ”或“/ *”被计为空白标记。

    • 如果令牌在PL / SQL中无效，则发出PLS-00103错误。但是如果使用违反此规则的条件编译指令，则会产生ORA错误。

    例2-61和例2-62显示第一个条件编译指令出现在定义单元标识符后面的第一个PL / SQL标记之后。

  • 在触发器或匿名块中，第一个条件编译指令不能出现在关键字DECLARE或BEGIN ，以先到者为准。

SQL解析器还强加了此限制：如果匿名块使用占位符，则占位符不能出现在条件编译指令中。例如：

开始：n：= 1; - 有效使用占位符$ IF ...$ THEN ：n ：= 1; - 无效使用占位符$ END
