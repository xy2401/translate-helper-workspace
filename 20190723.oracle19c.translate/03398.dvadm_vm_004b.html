说明dvadm_vm_004b.png的描述

此图说明了领域授权如何为两个领域所有者，Order Entry领域所有者和人力资源领域所有者工作。订单输入领域所有者具有OE_ADMIN角色，人力资源领域拥有HR_ADMIN角色。这两个领域所有者都具有以下数据库权限：

  • 创建任何表

  • DROP ANY TABLE

  • 选择任何表格

  • GRANT / REVOKE

当这些领域所有者在其各自的Schemas-OE模式中为Order Entry领域所有者和人力资源领域所有者的HR模式执行操作时，Oracle Database Vault领域授权允许他们执行它们所属的任何数据库特权。分配。

但是，他们不允许在彼此的领域中执行这些操作。例如，Order Entry领域所有者无法在HR模式中创建表，人力资源领域所有者也无法从OE模式中删除表。
