7.179 V $ DIAG_INFO

V$DIAG_INFO使用NAME=VALUE对描述自动诊断存储库（ADR）功能的状态。

柱 数据类型 描述

INST_ID

数

实例ID

名称

VARCHAR2（64）

标识反映ADR状态的一段数据，例如是否已启用，目录和文件的位置以及存在的问题（事件和问题）。

可能的值包括：

  • Diag Enabled ：表示是否启用ADR

  • 通过运行状况Health Monitor ADR Base ：显示不同的目录（ADR基础，ADR主页，然后是ADR主目录中的子目录）

  • Default Trace File ：指定当前进程的当前默认跟踪文件

  • Active Problem Count和Active Incident Count ：指定多少问题/事件有这ADR， 要么发生在过去24小时或有表明它是一个持续的错误（如磁盘损坏）一块元数据集的

值

VARCHAR2（512）

描述NAME列中标识的数据的当前状态

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
