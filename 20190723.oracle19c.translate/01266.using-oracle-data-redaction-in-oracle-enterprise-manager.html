关于在Oracle Enterprise Manager中使用Oracle数据编辑

Oracle Enterprise Manager Cloud Control提供统一的用户界面，用于创建和管理Oracle Data Redaction策略。

您可以执行以下操作：

  • 创建和管理自定义Oracle Data Redaction格式，以前称为数据编辑快捷方式。（命令行无法使用此功能。）

注意：

确保您拥有适用于Oracle Enterprise Manager的最新插件。如果您具有Oracle数据库插件版本13.1.1.0.0，则可以在Oracle Enterprise Manager中创建命名的数据编辑策略表达式。

企业管理器中敏感列类型的管理

敏感列类型将表列敏感信息分类为敏感信息类型，例如信用卡号。

敏感列类型使用列名，列注释和使用正则表达式定义的数据模式的组合将列标记为特定的敏感信息类型。

在创建Oracle Data Redaction策略时，将根据所选的敏感列类型筛选编校格式，从而节省时间和精力。例如，如果要编辑的数据库表列包含美国社会保险号，并且在将该列添加到Oracle Data Redaction策略时为该列选择了SOCIAL_SECURITY_NUMBER敏感列类型，则可以使用默认的编校格式进行编辑过滤列数据，仅显示相关的编校格式。

使用Enterprise Manager创建自定义Oracle数据编校格式

您可以使用Enterprise Manager Cloud Control创建和保存自定义Oracle Data Redaction格式。

  1. 以用户SYSTEM或SYSMAN身份登录Oracle Enterprise Manager云控制。
    URL如下：
    https:// host : port /em
  2. 从“ 目标”菜单中，选择“ 数据库” 。
  3. 选择“ 搜索列表” ，然后单击数据库目标的名称。
  4. 在数据库目标的主页上，从“ 安全性”菜单中选择“ 数据编辑” 。
  5. 如果系统提示您，请登录数据库。
    确保以具有DBMS_REDACT PL / SQL包的EXECUTE特权的用户身份登录数据库。
  6. 选择“ 格式”选项卡。
  7. 执行以下操作之一：
    • 要创建新的编校格式，请单击“ 创建” 。
    • 要创建基于默认格式的编校格式，请选择格式，然后单击“ 创建类似” 。
    如果选择“ 创建” ，则会出现以下对话框：

    em_red_format_create.png插图的描述
  8. 提供要创建的编校格式的名称和说明。

    如果要将编校格式映射到特定的敏感列类型（以便创建的编校格式可用于编辑与敏感列类型关联的列的数据），则选择敏感列类型的值。

    选择格式用于编辑列数据的函数。对于编校功能 ，请选择如下：

    • 如果格式应该编辑整个列数据，则为FULL 。

    • PARTIAL如果格式应该只编辑列数据的一部分。确保提供函数属性以及要使用编校格式的数据类型。

    • REGEX，如果格式应该基于正则表达式编辑数据。确保提供功能属性。

    • RANDOM，如果格式应使用随机生成的值以随机方式编辑数据

    • 如果格式将仅用于测试编校策略的定义，而不是编辑任何列数据，则为NONE

  9. 单击“ 确定”以创建并保存自定义编校格式。
    此格式现在可用于创建编校策略。

使用Enterprise Manager创建Oracle数据编校策略

您可以使用Enterprise Manager Cloud Control创建Oracle Data Redaction策略。

  1. 以用户SYSTEM或SYSMAN身份登录Oracle Enterprise Manager云控制。
    URL如下：
    https:// host : port /em
  2. 从“ 目标”菜单中，选择“ 数据库” 。
  3. 选择“ 搜索列表” ，然后单击要为其创建Oracle Data Redaction策略的数据库目标的名称。
  4. 在数据库目标的主页上，从“ 安全性”菜单中选择“ 数据编辑” 。
  5. 如果系统提示您，请登录数据库。
    确保以具有DBMS_REDACT PL / SQL包的EXECUTE特权的用户身份登录数据库。
  6. 在“策略”选项卡的“策略”部分中，选择“ 创建” 。

    如果这是您第一次创建数据编辑策略，则会出现数据编辑：设置以启用列敏感类型发现对话框。此功能允许对数据编辑策略使用列敏感类型发现。

    为此，Enterprise Manager在DBSNMP架构中创建GET_COL_DATA_SENSITIVE_TYPES过程。要在创建Oracle数据编校策略时对选定列执行敏感类型发现，用户必须具有DBSNMP.GET_COL_DATA_SENSITIVE_TYPES过程的EXECUTE特权。如果数据库受Oracle Database Vault保护，则确保必须创建数据编辑策略的任何用户都是保护DBSNMP架构的域的DBSNMP 。

  7. 如果出现“数据编辑：设置为启用列敏感类型发现”对话框，并且当前登录用户没有正确的要求，请选择对DBSNMP.GET_COL_DATA_SENSITIVE_TYPES具有EXECUTE特权的用户的凭据。然后单击OK 。
  8. 在“创建数据编辑策略”页面上，输入以下信息：

    • 架构 ：输入（或搜索）包含要编辑的数据的架构的名称。

    • 表/视图 ：输入（或搜索）包含要编辑的列的表或字段。

    • 策略名称 ：为策略输入a，例如emp_wages_pol 。

    • 默认表达式 ：输入默认表达式。默认设置为1 = 1，这意味着将始终强制执行策略。如果您不熟悉策略表达式的组件，请单击“ 策略表达式”字段旁边的铅笔图标以使用“策略表达式构建器”。选择“ 策略”时 ，选择所需条件，然后单击“ 添加” 。如果要手动编辑策略表达式，请单击“ 编辑” 。构建所需的策略表达式后，单击“ 确定” 。策略表达式生成器如下所示：


      插图说明em_red_pol_exp_bldr.png
  9. 在“对象列”部分中，单击“ 添加”以将表或视图列添加到编校策略。

    出现类似于以下内容的对话框：


    em_red_pol_attr.png插图的描述

    编校策略仅应用于添加到其中的表或视图列。

  10. 从“ 列”菜单中，选择要应用编校策略的表或视图列。
    “ 列”菜单右侧是一个图标，您可以单击该图标以查看所选列的内容。
    如果列包含敏感数据并已映射到敏感列类型，则从“ 敏感列类型”菜单中选择已映射到的敏感列类型。如果“ 敏感列类型”菜单中的搜索模式匹配，则默认情况下会选择敏感列类型。例如，对于列出信用卡号的列，如果匹配，则菜单将列出Undefined和CREDIT_CARD_TYPE 。如果未创建敏感列类型，则默认的“ 敏感列类型”菜单列表仅为“ 未定义” 。
  11. 从“ 编辑格式”菜单中，选择要使用的编校格式。
    下拉列表中填充了Oracle数据库提供的编校格式，以及您创建和保存的自定义编校格式。

    如果您不想使用预定义的编校格式（即Oracle数据库提供的编校格式或您创建的自定义编校格式），而是希望在创建编校策略时指定编校细节，选择CUSTOM for Redaction Format 。

    “添加”对话框会调整以适应您选择的编校格式和功能的类型。例如，如果选择CUSTOM编校格式和REGEX编校功能，则会在对话框中显示“功能属性”区域。

  12. 从“ 编辑功能”菜单中，选择要用于编辑列数据的功能。
    如果要编辑整个列数据，请选择FULL;如果要仅编辑部分列数据，请选择PARTIAL;如果要根据正则表达式编辑列数据，请选择REGEX;如果要编辑列数据，请选择RANDOM以随机方式，使用随机生成的值，如果您只想测试编校策略的定义，则为NONE ，而不是编辑任何列数据。请注意，所有编校功能可能不适用于特定的编校格式。下拉列表仅显示适用于所选编校格式的编校功能。
    如果在上一步中选择了CUSTOM作为编校格式 ，而在编辑功能中选择了 PARTIAL或REGEX ，请确保指定了功能属性。
  13. 单击确定 。
  14. 对于要添加到编校策略的所有列，从步骤8开始重复这些步骤。
  15. 在“创建数据编校策略”页面上，单击“ 确定”以创建数据编校策略。

    创建Oracle Data Redaction策略时，默认情况下会启用该策略。

使用Enterprise Manager编辑Oracle数据编校策略

您可以使用Enterprise Manager Cloud Control编辑Oracle Data Redaction策略。

  1. 以用户SYSTEM或SYSMAN身份登录Oracle Enterprise Manager云控制。
    URL如下：
    https:// host : port /em
  2. 从“ 目标”菜单中，选择“ 数据库” 。
  3. 选择“ 搜索列表” ，然后搜索并单击要为其编辑要编辑的Oracle数据编辑策略的数据库目标的名称。
  4. 在数据库目标的主页上，从“ 安全性”菜单中选择“ 数据编辑” 。
  5. 如果系统提示您，请登录数据库。
    确保以具有DBMS_REDACT PL / SQL包的EXECUTE特权的用户身份登录数据库。
  6. 在策略选项卡的策略部分，选择要编辑的新版本策略，然后单击编辑。 。
    em_red_pol_edit.png插图的描述
  7. 在“编辑数据编校策略”页面上，选择编辑策略表达式，向编校策略添加新列，修改作为策略一部分的列的编校详细信息，或从编校策略中删除列。

    您可以执行以下操作：

    • 要向编校策略添加新列，请在“对象列”部分中单击“ 添加”，选择要添加的表或视图列，然后指定编校详细信息。

    • 要修改作为策略一部分的列的编校详细信息，请选择该列，单击“ 修改”，然后编辑编校详细信息。

    • 要从编校策略中删除列，请选择该列，然后单击“ 删除” 。

  8. 在“编辑数据编校策略”页面上，在编辑所需字段后，单击“ 确定”以保存并启用编辑的编校策略。

在企业管理器中创建命名数据编校策略表达式

您可以使用Enterprise Manager Cloud Control创建和应用命名的Oracle Data Redaction策略表达式。

  1. 以用户SYSTEM或SYSMAN身份登录Oracle Enterprise Manager云控制。
    URL如下：
    https:// host : port /em
  2. 从“ 目标”菜单中，选择“ 数据库” 。
  3. 选择“ 搜索列表” ，然后单击要为其创建Oracle Data Redaction策略的数据库目标的名称。
  4. 在数据库目标的主页上，从“ 安全性”菜单中选择“ 数据编辑” 。
  5. 如果系统提示您，请登录数据库。
    确保以具有DBMS_REDACT PL / SQL包的EXECUTE特权的用户身份登录数据库。
  6. 在Oracle Data Redaction页面上，选择“ 表达式”选项卡。
  7. 单击“ 创建”

    将出现“创建”对话框。


    插图说明em_red_pol_exp_create.png
  8. 在“创建”对话框中，输入以下信息：
    • 表达式名称 ：输入策略表达式的名称。现有策略表达式列在“数据编辑”页面上。
    • 描述 ：输入策略的简短描述。
    • 表达式 ：输入表达式。对于更复杂的表达式，例如应用或免除特定用户的策略，请单击“ 表达式”字段右侧的“策略表达式生成器”图标。在Policy Expression Builder中单击OK以创建表达式。
  9. 在“创建”对话框中单击“ 确定 ”。

    创建策略表达式后，它将在“数据编辑”页面中列出，并准备与数据编辑策略关联。

  10. 在“数据编辑”页面中，选择“ 策略”选项卡。
  11. 在“策略”下，选择用于编辑要应用策略表达式的列的策略的行，然后单击“ 编辑” 。
  12. 在“对象列”下，选择所需的列，然后单击“ 修改”按钮。
  13. 在“修改”对话框中，从“ 表达式名称”列表中选择表达式。
  14. 单击“ 确定” ，然后在“编辑数据编校策略”对话框中再次单击“ 确定 ”。
