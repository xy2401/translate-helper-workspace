适用于Windows的F Pro * COBOL

本节概述了使用Pro * COBOL for Windows操作系统构建Oracle数据库应用程序。

本节包含以下主题：

  • 编译和链接Pro * COBOL应用程序

  • 示例程序

F.1编译和链接Pro * COBOL应用程序

Pro * COBOL支持用于32位Windows的MERANT Micro Focus NetExpress 5.0版。本节介绍如何使用MERANT Micro Focus编译器编译和链接Pro * COBOL应用程序。

您可以通过两种方式构建和执行MERANT Micro Focus COBOL应用程序：

  • 如何使用IDE （仅使用NetExpress）

  • 如何使用动画制作产品 （NetExpress以外的产品）

  • COBOL和CBLLINK命令 （所有产品）

  • COBSQL命令

在每个中都可以使用COBSQL实用程序，具有以下优点：

  • Pro * COBOL由MERANT Micro Focus编译器运行，不需要作为单独的步骤运行。

  • 动画是使用.pco源文件而不是Pro * COBOL生成的.cbl文件完成的。

  • MAKEYSYN指令是自动提供的，无需手动指定。

F.1.1如何使用IDE

Pro * COBOL生成的程序可以在MERANT Micro Focus NetExpress IDE中编译和执行。只需将Pro * COBOL生成的.cbl文件添加到Net Express项目中即可。为避免在Oracle库中调用例程时可能出现的不一致，应使用该指令编译程序：

MAKESYN“COMP-5”=“COMP”

可以在源文件的构建设置，项目设置中或通过源文件开头的$SET行指定此伪指令。选择Rebuild或Rebuild All时 ，IDE会生成一个可执行文件，可以运行或动画。

F.1.2如何使用动画制作产品

可以在MERANT Micro Focus COBOL调试器Animator V2中编译和执行程序。

要在Oracle库中调用例程时避免潜在的不一致，请选择菜单选项Compiler Directives，然后输入：

此步骤是必需的，因为MERANT Micro Focus COBOL以Big Endian格式存储二进制数。Oracle库期望二进制数以Little Endian格式（机器格式）存储。

MAKESYN“COMP-5”=“COMP”

F.1.3 COBOL和CBLLINK命令

COBOL和CBLLINK可用于以两种方式构建程序，具体取决于Pro * COBOL运行时是静态链接还是在运行时通过DLL访问。

对于动态链接，命令是：

COBOL sample1 / MAKESYN“COMP-5”=“COMP”; CBLLINK样本1

对于静态链接，命令是：

COBOL sample1 / LITLINK / MAKESYN“COMP-5”=“COMP”; CBLLINK sample1 ORACLE_BASE \ ORACLE_HOME \ precomp \ lib \ orasql12.lib

以前的命令生成sample1.exe ，可以像任何其他Windows程序一样执行。

注意：

MERANT Micro Focus COBOL必须与Pro * COBOL安装在同一系统上才能成功执行该文件。

F.1.4 COBSQL命令

COBSQL可用于简化预处理和调试。要使用COBSQL ，请为COBOL编译器指定以下指令：

预处理（COBSQL）COBSQLTYPE = ORACLE8 ENDP

或简短形式：

P（COBSQL）CSQLT = ORA8 ENDP

对于版本8.0之前的Pro * COBOL版本，应将COBSQLTYPE设置为ORACLE或ORA 。该指令可以在源文件的开头，COBOL命令行，NetExpress的程序构建设置或项目设置中使用$SET行设置，或者使用Animator的SQL编译器指令设置。在编译时，COBSQL将Pro * COBOL作为后台任务运行，并将其输出传递给COBOL编译器以及使Animator使用.pco文件而不是.cbl文件跟踪执行所需的其他信息。

使用COBSQL ，无需直接处理.cbl文件。而是将.pco文件添加到NetExpress项目，或使用Animator打开它。

F.2示例程序

Oracle提供了示例程序来演示Pro * COBOL与Oracle数据库功能的使用。有关这些程序的列表，请参阅“ 示例文件 ” 。

本节介绍如何使用基本预编译，编译和链接命令来构建示例程序。本节还介绍了运行Pro * COBOL示例程序所需的准备工作。

F.2.1建立演示表

要运行Pro * COBOL示例程序，您必须拥有一个包含用户名scott和密码tiger的数据库帐户。如果数据库中不存在此帐户，请在运行示例程序之前创建一个帐户。

scott帐户必须包含emp和dept表。如果该帐户不包含这些表，请使用demobld.sql脚本创建它们。

要运行demobld.sql脚本：

  1. 启动SQL * Plus。

  2. 使用密码tiger以用户名scott连接到数据库。

  3. 运行demobld.sql脚本。例如：

    SQL> @ORACLE_BASE \ ORACLE_HOME \ sqlplus \ demo \ demobld.sql

F.2.2构建示例程序

Pro * COBOL提供makeit.bat文件，该文件将在下一节中列出，用于构建MERANT Micro Focus COBOL示例文件：

批处理文件位于ORACLE_BASE\ORACLE_HOME \precomp\demo\procob2 。

要构建示例程序：

  1. 使用任何示例文件运行批处理文件。不要包含文件扩展名。例如：

    C：\ ORACLE \ ORA90 \ PRECOMP \ DEMO \ PROCOB2> makeit sample1

  2. 如果在构建示例程序时遇到错误，请确保所有路径和文件名都反映系统的配置。

运行示例程序的命令假定以下是当前工作目录：

ORACLE_BASE \ ORACLE_HOME \ precomp \ demo \ procob2

您可能需要修改示例链接脚本以反映系统的配置。有关详细信息，请参阅“ 编译和链接Pro * COBOL应用程序 ” 。

makeit.bat包含以下内容：

procob iname =％1.pco ireclen = 132 cobol％1 / anim / litlink makesyn“COMP-5”=“COMP”; cbllink％1 / M％1 ORACLE_BASE \ ORACLE_HOME \ precomp \ lib \ orasql12.lib
