1.172 LOG_ARCHIVE_DEST_ n

LOG_ARCHIVE_DEST_ n初始化参数最多定义31个（其中n = 1,2,3，... 31）目标，每个目标必须指定LOCATION或SERVICE属性以指定归档重做数据的位置。

基本

是

除LOCATION或SERVICE属性之外的所有其他属性都是可选的。请注意，无论是指定LOCATION属性还是SERVICE属性，它都必须是属性列表中提供的第一个属性。

如果您选择不输入任何属性，则可以通过输入以下内容指定NULL字符串：

LOG_ARCHIVE_DEST_ n ='';

您可以设置LOG_ARCHIVE_DEST_ n初始化参数的属性，以控制重做传输服务如何将重做数据从生产或主数据库目标传输到另一个（备用）数据库目标的不同方面。您可以查询V$ARCHIVE_DEST视图以查看每个目标的当前属性设置（ n ）。

注意：

不推荐使用此参数的某些属性，但为了向后兼容性而保留这些属性。请参见“ LOG_ARCHIVE_DEST_n的不推荐使用的属性 ” 。

对于您定义的每个LOG_ARCHIVE_DEST_n初始化参数，必须指定相应的LOG_ARCHIVE_DEST_STATE_n参数。LOG_ARCHIVE_DEST_STATE_n （其中n是1到31之间的整数）初始化参数指定当前是启用还是禁用相应的目标。

“ LOG_ARCHIVE_DEST_STATE_n ”

目标LOG_ARCHIVE_DEST_11到LOG_ARCHIVE_DEST_31不支持SYNC ， ARCH ， LOCATION ， MANDATORY或ALTERNATE属性，也不能指定为ALTERNATE属性的目标。

只有当COMPATIBLE初始化参数设置为11.2.0或更高时，才能使用LOG_ARCHIVE_DEST_11到LOG_ARCHIVE_DEST_31 。

值

也可以看看：

有关本节中列出的所有值的详细说明，请参阅Oracle Data Guard概念和管理

  • AFFIRM和NOAFFIRM

    控制重做传输目标是否在将其写入备用重做日志之前或之后确认接收到的重做数据。默认值为NOAFFIRM 。

  • ALTERNATE=LOG_ARCHIVE_DEST_ n

    指定原始目标失败时要使用的备用存档目标。没有默认值;如果未指定备用目标，则如果原始目标失败，则归档不会自动更改为其他目标。

  • 异步

    在该事务可以提交之前，不需要在具有该属性的目的地处接收由事务生成的重做数据。如果既未指定SYNC也未指定ASYNC则这是默认行为。

  • 压缩

    指示是启用还是禁用或禁用网络压缩，或者是否使用ZLIB或LZO算法。可能的值包括：

    • DISABLE ： DISABLE压缩。

    • ENABLE ：启用压缩。使用ZLIB压缩算法。

    • ZLIB ：使用ZLIB压缩。

    • LZO ：使用LZO压缩。

    要使用COMPRESSION属性，必须使用“高级压缩”选项。

  • DB_UNIQUE_NAME = 姓名

    指定此目标的数据库的唯一名称。你必须指定一个名字;没有默认值。

  • DELAY [= 分钟 ]

    指定在备用站点上存档重做数据从主数据库检索重做时以及将存档重做日志文件应用于备用数据库或从其中级联的任何备用数据库之间的最小时间延迟。如果指定DELAY属性而没有时间间隔，则默认值为30分钟。

  • ENCRYPTION = DISABLE | ENABLE

    控制是否启用或禁用发送到零数据丢失恢复设备（恢复设备）的重做流的加密。该属性的默认值为DISABLE 。

    此属性不能与COMPRESSION ， LOCATION或SYNC属性一起使用。

    要使用ENCRYPTION属性，必须在受保护的数据库11.2.0.4 COMPATIBLE初始化参数设置为11.2.0.4或更高版本。

    也可以看看：

    零数据丢失恢复设备管理员指南 ，了解有关恢复设备的介绍性信息

  • GROUP = group

    GROUP属性用于指定特定日志归档目标集合中的成员身份。组编号为1到8。默认组（ GROUP=0 ）的特殊之处在于无法分配。默认组将填充未明确分配给组的所有目标。默认组以外的所有组都必须包含一组远程目标。没有组（默认组除外）可以包含本地目标。

  • LOCATION = local_disk_directory或USE_DB_RECOVERY_FILE_DEST

    指定本地文件系统目标或将用作快速恢复区域的目录，文件系统或自动存储管理磁盘组。您必须至少为一个目标指定此属性。您可以使用LOCATION属性指定本地磁盘目录或快速恢复区域。您必须为每个目标包含LOCATION或SERVICE属性，以指定归档重做数据的位置。

  • 强制性

    指定在可以重用本地联机重做日志文件之前，必须成功将重做数据传输到目标。如果未指定MANDATORY属性，则目标是可选的。

  • MAX_FAILURE

    控制重做传输服务尝试重新建立通信的连续次数，并在主数据库放弃目标之前将重做数据传输到失败的目标。有关如何在Oracle Database 12 c第2版（12.2.0.1）和Oracle Database 12 c第1版（12.1.0.2）中以不同方式处理此属性的使用说明，请参阅Oracle Data Guard概念和管理中的MAX_FAILURE描述。

  • NET_TIMEOUT = seconds

    指定在终止网络连接之前，主系统上的日志写入程序进程等待来自SYNC（NSS n ）进程的状态的秒数。默认值为30秒。

  • NOREGISTER

    指示存档的重做日志文件的位置不会记录在相应的目标位置。

  • PRIORITY = priority

    PRIORITY属性用于指定一组日志归档目标中的首选项。优先级编号为1到8。较低的值表示较高的优先级。最低优先级（ PRIORITY=8 ）在某种意义上是特殊的，即如果该优先级处于活动状态，那么该优先级的所有目的地都将被激活。如果任何更高优先级的目标返回服务，则该目标将变为活动状态，并且所有低优先级目标将变为非活动状态，因为它们将通过级联或远程同步从其他一个重做目标接收重做。

  • REOPEN [= seconds ]

    指定归档程序进程（ARC n ）或日志写入程序进程（LGWR）应再次尝试访问先前失败的目标之前的最小秒数。默认值为300秒。

  • SERVICE = net_service_name

    指定有效的Oracle Net服务名称（ SERVICE= net_service_name ），该名称标识将向其发送重做数据的远程Oracle数据库实例。每个目标必须指定LOCATION或SERVICE属性。没有默认的网络服务名称。

  • 同步

    事务生成的重做数据必须已由每个已启用的目标接收，该目标在该事务可以提交之前具有此属性。

  • TEMPLATE = filename_template _％t_％s_％r

    指定在重做传输目标上创建的归档重做日志文件的路径名和文件名模板，该重做日志文件包含指定此属性的数据库中的重做数据。此属性将覆盖重做传输目标上的LOG_ARCHIVE_FORMAT初始化参数的值。此属性没有默认值。

  • VALID_FOR =（redo_log_type，database_role）

    确定重做传输服务何时可以根据以下因素将重做数据传输到目标：

    • redo_log_type - 在线重做日志文件，备用重做日志文件或两者当前是否在此目标的数据库上存档

    • database_role数据库当前是以主角色还是备用角色运行

没有。此属性仅与已弃用的ARCH属性一起使用。

此外，请注意ASYNC和SYNC属性的以下更改：

  • 不再需要ASYNC属性上的BLOCKS关键字。

    不再需要设置此关键字，因为Data Guard根据需要动态地将块计数向上或向下调整到适当数量的块。

  • 不再需要SYNC属性上的PARALLEL和NOPARALLEL关键字。
