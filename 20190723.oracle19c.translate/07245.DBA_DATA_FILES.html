4.198 DBA_DATA_FILES

DBA_DATA_FILES描述数据库文件。

柱 数据类型 空值 描述

文件名

VARCHAR2（513）

数据库文件的名称

FILE_ID

数

数据库文件的文件标识号

TABLESPACE_NAME

VARCHAR2（30）

文件所属的表空间的名称

BYTES

数

文件大小（以字节为单位）

积木

数

Oracle块中文件的大小

状态

VARCHAR2（9）

文件状态： AVAILABLE或INVALID （ INVALID表示文件号未被使用，例如，表空间中的文件被删除）

RELATIVE_FNO

数

相对文件号

自动扩展

VARCHAR2（3）

自动扩展指标

MAXBYTES

数

最大文件大小（字节）

MAXBLOCKS

数

块中的最大文件大小

INCREMENT_BY

数

用作自动扩展增量的Oracle块数

USER_BYTES

数

可用于用户数据的文件大小。文件的实际大小减去USER_BYTES值用于存储文件相关的元数据。

USER_BLOCKS

数

数据可以使用的块数

ONLINE_STATUS

VARCHAR2（7）

文件的在线状态：

  • SYSOFF

  • 系统

  • 离线

  • 线上

  • 恢复

LOST_WRITE_PROTECT

VARCHAR2（7）

丢失文件的写保护状态。可能的值：

  • ENABLED ：表示正在收集丢失的写入数据

  • OFF ：表示未收集丢失的写入数据

  • SUSPEND ：表示当前未收集丢失的写入数据，但可以在以后启用。文件处于ENABLED时收集的丢失写入数据仍保留在丢失的写入数据库中，但未进行检查或更新。

如果为单个数据文件启用了丢失的写保护，则不必为同一表空间中的另一个数据文件启用它。

如果为表空间启用了丢失写保护，则会为该表空间的所有数据文件启用它，包括稍后添加的数据文件。

您可以通过查询DBA_TABLESPACES视图中的LOST_WRITE_PROTECT列来检查表空间的丢失写保护状态。
