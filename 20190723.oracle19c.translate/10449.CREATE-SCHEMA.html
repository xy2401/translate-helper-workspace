创建模式

目的

使用CREATE SCHEMA语句创建多个表和视图，并在单个事务中在您自己的模式中执行多个授权。

要执行CREATE SCHEMA语句，Oracle数据库将执行每个包含的语句。如果所有语句都成功执行，则数据库将提交事务。如果任何语句导致错误，则数据库将回滚所有语句。

注意：

该语句实际上并不创建模式。Oracle Database在您创建用户时自动创建架构（请参阅CREATE USER ）。此语句允许您使用表和视图填充模式，并为这些对象授予权限，而无需在多个事务中发出多个SQL语句。

先决条件

CREATE SCHEMA语句可以包括CREATE TABLE ， CREATE VIEW和GRANT语句。要发出CREATE SCHEMA语句，您必须具有发出包含语句所需的特权。

语义

模式

指定架构的名称。模式名称必须与Oracle数据库用户名相同。

create_table_statement

指定要作为此CREATE SCHEMA语句的一部分发出的CREATE TABLE语句。不要以分号（或其他终止符）结束此语句。

GRANT

CREATE SCHEMA语句仅支持标准SQL定义的这些语句的语法，而不支持Oracle数据库支持的完整语法。

列出CREATE TABLE ， CREATE VIEW和GRANT语句的顺序并不重要。CREATE SCHEMA语句中的语句可以引用您在同一CREATE SCHEMA语句中的其他语句中创建的现有对象或对象。

授予模式权限的限制

对CREATE SCHEMA的CREATE TABLE语句允许使用parallel_clause的语法，但在创建对象时不使用并行性。

创建架构：示例

下面的语句创建一个名为架构oe的样品订单输入用户oe ，创建表new_product ，创建视图new_product_view ，并授予SELECT的对象权限new_product_view到样品人力资源使用者hr 。 

CREATE SCHEMA授权oe CREATE TABLE new_product（颜色VARCHAR2（10）PRIMARY KEY，数量NUMBER）CREATE VIEW new_product_view AS SELECT颜色，数量FROM new_product WHERE color ='RED'GRANT选择ON new_product_view TO hr;
