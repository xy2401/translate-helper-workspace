网络接口硬件最低要求

查看这些要求以确保您拥有Oracle Grid Infrastructure群集的最低网络硬件技术。

每个节点的公共网络

公共网络为客户端提供数据库服务访问。公共网络必须满足这些最低要求：

  • 适配器：每个节点必须至少有一个公共网络适配器或网络接口卡（NIC）。

    Oracle支持使用链路聚合，绑定，中继或组合网络来提高带宽和高可用性。

  • 协议：每个公共接口必须支持TCP / IP。

每个节点的专用网络

专用网络（也称为互连）是仅集群成员节点可以访问的网络。他们使用交换机进行连接。专用网络必须满足以下最低要求：

  • 适配器：每个节点必须至少有一个专用网络适配器或网络接口卡（NIC）。

    Oracle建议您使用冗余互连使用配置互连，其中多个网络适配器配置了链路本地范围内的地址，以便为互连提供高可用性IP（HAIP）地址。您可以在安装期间或使用Oracle Interface Configuration Tool（OIFCFG）安装后配置冗余互连使用，以提供更高的带宽和高可用性。

    Oracle还支持使用链路聚合，绑定，中继或组合网络，以提高带宽和高可用性。

  • 协议：使用支持TCP / IP的高速网络适配器和交换机或使用Infiniband的可靠数据报套接字（RDS）的用户数据报协议（UDP）。

    交换机：必须使用支持TCP / IP的互连交换机。 Oracle建议您使用专用交换机。最小交换机速度为1千兆以太网。

局域网技术

Oracle不支持互连的令牌环或交叉电缆。Oracle支持Jumbo Frames和Infiniband。在互连上使用Infiniband时，Oracle支持使用RDS协议。

如果您使用共享物理适配器进行共享以太网VLAN部署，请确保应用标准以太网设计，部署和监视最佳实践，以防止由于常见共享以太网交换机网络事件导致的群集中断和性能下降。

存储网络

Oracle自动存储管理和Oracle Real Application Clusters需要网络连接存储。

Oracle自动存储管理（Oracle ASM）：用于Oracle Clusterware文件的网络接口也用于Oracle ASM。

第三方存储：Oracle建议您为存储配置其他接口。
