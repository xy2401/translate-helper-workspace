在SQL语句中使用XML

本节介绍了在数据库中使用XMLType数据的一些方法。

XMLType表

示例架构oe包含一个表warehouses ，其含有XMLType柱warehouse_spec 。假设您要使用warehouse_spec信息创建单独的表。以下示例使用一个CLOB列创建一个非常简单的XMLType表：

创建表xwarehouses OF XMLTYPE XMLTYPE STORE as CLOB;

您可以使用XMLType语法插入到这样的表中，如下一个语句所示。（此示例中插入的数据对应于样本表oe.warehouses的warehouse_spec列中的数据，其中warehouse_id = 1。）

您可以在基于XMLSchema的表上定义约束。为此，您使用XMLDATA伪列来引用Warehouse XML元素中的相应属性：

ALTER TABLE xwarehouses ADD（PRIMARY KEY（XMLDATA。“WarehouseId”））;

由于xwarehouses的数据是以对象方式存储的，因此Oracle xwarehouses查询重写为此XMLType表，以便在可能的情况下转到底层存储。因此，以下查询将使用前面示例中主键约束创建的索引：

SELECT * FROM xwarehouses x WHERE EXISTSNODE（VALUE（x），'/ Warehouse [WarehouseId =“1”]'，'xmlns：who =“http://www.example.com/xwarehouses.xsd”'）= 1; SELECT * FROM xwarehouses x WHERE EXTRACTVALUE（VALUE（x），'/ Warehouse / WarehouseId'，'xmlns：who =“http://www.example.com/xwarehouses.xsd”'）= 1;

XMLType列

使用XMLType类型的warehouse_spec列创建了示例表oe.warehouses 。本节中的示例使用两种不同类型的存储创建oe.warehouses表的缩写形式。

第一个示例创建一个表，其中XMLType表存储为CLOB 。此表不需要XMLSchema，因此内容结构未预先确定：

CREATE TABLE xwarehouses（warehouse_id NUMBER，warehouse_spec XMLTYPE）XMLTYPE warehouse_spec STORE AS CLOB（TABLESPACE示例存储（INITIAL 6144）CHUNK 4000 NOCACHE LOGGING）;

以下示例创建一个类似的表，但将XMLType数据存储在对象关系XMLType列中，该列的结构由指定的XMLSchema确定：

CREATE TABLE xwarehouses（warehouse_id NUMBER，warehouse_spec XMLTYPE）XMLTYPE warehouse_spec STORE AS OBJECT RELATIONAL XMLSCHEMA“http://www.example.com/xwarehouses.xsd”ELEMENT“Warehouse”;
