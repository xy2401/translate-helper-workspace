A.1在云控制中使用IM列存储的会议先决条件

在启用数据库以使用IM列存储之前，请确保COMPATIBLE设置为12.1.0.0或更高版本。

要设置兼容级别，请按照下列步骤操作：

  1. 在Enterprise Manager的Database Home页面中，从Administration菜单中选择Initialization Parameters以导航到Initialization Parameters页面。

    您可以使用此页面设置或更改兼容级别。

  2. 搜索COMPATIBLE初始化参数。

    参数的类别是Miscellaneous。

  3. 将值更改为12.1.0.0 ，然后单击“ 应用” 。

    Cloud Control会提示您重新启动数据库。重新启动数据库后，新值将生效。

要设置或更改IM列存储的大小，请按照下列步骤操作：

  1. 在Enterprise Manager的Database Home页面中，从Administration菜单中选择Initialization Parameters以导航到Initialization Parameters页面。

  2. 搜索参数INMEMORY_SIZE 。参数的类别是In-Memory。

  3. 更改值，然后单击“ 应用” 。

    您可以将值设置为高于最小大小100 MB的任何值。

    然后，系统将提示您重新启动数据库。

A.2使用内存列存储中央主页监视数据库对象的内存支持

使用内存中列存储中心主页来监视对数据库对象（如表，索引，分区和表空间）的内存内支持。您可以查看对象的内存中功能并监视其内存中使用情况统计信息。

您可以在内存列存储中心主页上完成以下操作：

  • 内存中对象访问热图显示IM列存储中的前100个对象及其相对大小，并显示对象的访问频率，由不同颜色表示。要激活热图，必须在初始化参数文件中打开热图的选项。通常，在激活地图之前有一天的等待期。您可以使用日期选择器选择热图中显示的对象的日期范围。您还可以使用滑块控制颜色的粒度。

  • 使用“配置”部分可以查看状态设置，例如内存中查询，内存中强制和默认内存中子句。单击“编辑”以导航到“初始化参数”页面，您可以在其中更改此部分中显示的值和设置。使用“性能”部分查看“活动会话”的度量标准。

  • 使用“对象摘要”部分可以查看压缩因子和有关填充对象使用的内存的数据。通过从页面上的View In-Memory Enabled Object Statistics链接向下钻取，弹出窗口中提供了In-Memory Enabled Object Statistics。

  • 使用“内存中对象分发”部分可以按百分比查看内存中使用的各种对象的分布。该部分包括一个图表，显示分区，子分区，非分区表和非分区物化视图的分布。每个的数值显示在图表上方。

  • 使用“内存中对象搜索”部分搜索为内存使用指定的对象。输入要搜索的参数后，单击“搜索”。结果表显示了找到的每个对象的名称及其大小，内存大小，磁盘大小，内存中百分比及其内存中参数。您还可以搜索内存中或非内存中的访问对象。如果启用了热图，则“内存对象搜索”框的“视图”字段的下拉列表中将显示“访问的对象”选项。选择“访问的对象”时，可以根据前100个对象进行过滤，这些对象的访问数据可以是内存中的，也可以不是内存中的。您可以选择时间范围并搜索该范围内的对象。如果选择“所有内存中的对象”选项，则可以根据内存大小查看内存中前100个对象的列表。

如果您在Oracle RAC环境中工作，则可以通过在热图上方和右侧的实例选择框中选择实例来快速在实例之间移动。

A.3创建表或分区时指定内存中的详细信息

您可以在创建表或分区时指定IM列存储详细信息。

  1. 从Schema菜单中，选择Database Objects ，然后选择Tables选项。
  2. 单击“ 创建”以创建表。

    将显示“创建表”页面。选择“内存中列存储”选项卡以指定表的内存中选项。

  3. 选择覆盖下表中指定列的列级内存详细信息（如果需要）。
  4. （可选）您可以单击“ 分区”选项卡。
  5. 使用向导根据需要创建表分区。

    若要为分区指定IM列存储详细信息，请从“分区”选项卡中的表中选择它，然后单击“ 高级选项” 。

  6. 在表级别，列级别和分区级别输入所有必需的IM列存储详细信息后，单击“ 显示SQL”以查看生成的SQL。单击“ 确定”以创建表。

A.4查看或编辑表的IM列存储详细信息

您可以查看或编辑表的IM列存储详细信息。

  1. 从Schema菜单中，选择Database Objects ，然后选择Tables选项。
  2. 搜索所需的表，然后单击“ 查看”以查看其详细信息。
  3. 单击“ 编辑”以启动“编辑表”页面。

    或者，在“搜索”页面上，单击“ 编辑” 。单击“ 内存中列存储”选项卡以指定表的内存中选项。

  4. 编辑所需的详细信息。
  5. 单击“ 应用”

A.5查看或编辑分区的IM列存储详细信息

您可以查看或编辑分区的IM列存储详细信息。

  1. 从Schema菜单中，选择Database Objects ，然后选择Tables选项。
  2. 搜索包含所需分区的表，选择它，然后单击“ 查看” 。
  3. 单击“ 编辑”以启动“编辑表”页面。

    或者，在“表搜索”页面上，单击“ 编辑” 。

  4. 单击“ 分区”选项卡，然后选择所需的分区。
  5. 单击高级选项 。
  6. 编辑所需的详细信息。
  7. 单击“ 确定”返回“分区”选项卡。
  8. 对表的所有所需分区进行类似更改后，单击“ 应用” 。

A.6在创建表空间期间指定IM列存储详细信息

您可以在创建表空间时指定IM列存储详细信息。

  1. 从“ 管理”菜单中，选择“ 存储” ，然后选择“ 表空间” 。
  2. 单击“ 创建”以创建表空间。
  3. 输入“常规”选项卡上显示的详细信息。
  4. 单击“ 内存中列存储”选项卡。
  5. 输入表空间的所有必需IM列存储详细信息。
  6. 单击“ 显示SQL” 。
  7. 在“显示SQL”页面中，单击“ 返回” 。
    出现另一页。
  8. 单击确定 。
  9. 单击“ 确定”以创建表空间。

表空间的IM列存储设置适用于在表空间中创建的任何新表。如果表必须覆盖表空间的配置，则必须在单个表级别指定IM列存储配置详细信息。

A.7查看和编辑表空间的IM列存储详细信息

您可以查看或编辑表空间的IM列存储详细信息。

  1. 从“ 管理”菜单中，选择“ 存储” ，然后选择“ 表空间”选项。
  2. 搜索所需的表空间，选择它，然后单击“ 查看” 。
  3. 单击“ 编辑”以启动“编辑表空间”页面，然后单击“ 内存中列存储”选项卡。
  4. 编辑所需的详细信息并单击“ 应用”

A.8在物化视图创建期间指定IM列存储详细信息

您可以在创建实例化视图时指定IM列存储详细信息。

  1. 从Schema菜单中，选择Materialized Views ，然后选择Materialized Views选项。
  2. 单击“ 创建”以创建实例化视图。
  3. 输入物化视图名称，并指定其查询。
  4. 单击“ 内存中列存储”选项卡以指定实例化视图的IM列存储选项。
  5. 输入所有必需的IM列存储详细信息后，单击“ 显示SQL” 。单击Show SQL页面中的Return ，然后在结果页面中单击OK 。
  6. 单击“ 确定”以创建实例化视图。

A.9查看或编辑物化视图的IM列存储详细信息

您可以查看或编辑实例化视图的IM列存储详细信息。

  1. 从Schema菜单中，选择Materialized Views ，然后选择Materialized Views选项。
  2. 搜索所需的物化视图，然后单击“ 查看”以查看其详细信息。
  3. 单击“ 编辑”以启动“编辑实体化视图”页面。
  4. 单击“ 内存中列存储”选项卡以指定实例化视图的IM列存储选项。
  5. 编辑所需的详细信息，然后单击“ 应用”
