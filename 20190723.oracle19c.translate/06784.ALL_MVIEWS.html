2.228 ALL_MVIEWS

ALL_MVIEWS描述了当前用户可访问的所有物化视图。

相关观点

  • DBA_MVIEWS描述了数据库中的所有物化视图。

  • USER_MVIEWS描述了当前用户拥有的所有物化视图。

柱 数据类型 空值 描述

所有者

VARCHAR2（128）

不是NULL

用于创建物化视图的模式

MVIEW_NAME

VARCHAR2（128）

不是NULL

物化视图的名称

CONTAINER_NAME

VARCHAR2（128）

不是NULL

保存物化视图数据的容器的名称。通常这与MVIEW_NAME相同。对于在Oracle8 i之前创建的物化视图，Oracle数据库附加了6字节前缀SNAP$_ 。如果MVIEW_NAME字节数超过19个，则Oracle数据库会将名称截断为19个字节，并可能会添加一个4字节的序列号作为后缀，以生成无歧义的CONTAINER_NAME 。

QUERY

长

定义物化视图的查询

QUERY_LEN

NUMBER（38）

定义查询的长度（以字节为单位）

UPDATABLE

VARCHAR2（1）

指示物化视图是否可更新（ Y ）（ N ）

update_log中

VARCHAR2（128）

对于可更新的物化视图，更新日志的文件名

MASTER_ROLLBACK_SEG

VARCHAR2（128）

主站点或主物化视图站点的回滚段

MASTER_LINK

VARCHAR2（128）

主站点或主物化视图站点的数据库链接

REWRITE_ENABLED

VARCHAR2（1）

指示是否启用重写（ Y ）（ N ）

REWRITE_CAPABILITY

VARCHAR2（9）

指示实例化视图是否符合重写条件，如果符合，则必须遵循哪些规则：

  • NONE - 物化视图不能用于重写，因为不允许或禁止重写

  • TEXTMATCH - 定义物化视图的查询包含对查询重写使用的限制

  • GENERAL - 定义实体化视图的查询不包含对查询重写使用的限制，因此Oracle数据库可以应用任何支持的重写规则

REFRESH_MODE

VARCHAR2（6）

物化视图的刷新模式：

  • DEMAND - 只要调用适当的刷新过程，Oracle数据库就会刷新此物化视图

  • COMMIT - Oracle数据库在其中一个物化视图的主服务器上的事务提交时刷新此物化视图

  • NEVER - Oracle数据库永远不会刷新此物化视图

REFRESH_METHOD

VARCHAR2（8）

用于刷新实例化视图的默认方法（可以通过API覆盖）：

  • COMPLETE -物化视图是从主人完全刷新

  • FORCE - 如果可能，Oracle数据库执行快速刷新，否则完全刷新

  • FAST - Oracle数据库执行增量刷新，应用与上次刷新后主服务器中的更改相对应的更改

  • NEVER - 用户指定Oracle数据库不应刷新此物化视图

BUILD_MODE

VARCHAR2（9）

指示在创建期间如何填充实例化视图：

  • IMMEDIATE - 在创作过程中由大师填充

  • DEFERRED - 在创建过程中未填充。必须稍后由用户明确填充。

  • PREBUILT - 在创建期间填充现有表。Oracle数据库不知道此预构建表的内容与物化视图的主数据之间的关系。

FAST_REFRESHABLE

VARCHAR2（18）

指示实例化视图是否有资格进行增量（快速）刷新。Oracle数据库基于物化视图定义查询静态计算此值：

  • NO - 物化视图不能快速刷新，因此很复杂

  • DML - 仅对DML操作支持快速刷新

  • DIRLOAD_DML - 直接加载和DML操作都支持快速刷新

  • DIRLOAD_LIMITEDDML - 直接加载和DML操作的子集支持快速刷新

LAST_REFRESH_TYPE

VARCHAR2（8）

用于最近刷新的方法：

  • COMPLETE -最近更新完成

  • FAST - 最近刷新很快（增量）

  • NA - 物化视图尚未刷新（例如，如果它已创建为DEFERRED ）

LAST_REFRESH_DATE

日期

物化视图最近刷新的日期（如果尚未填充，则为空白）

LAST_REFRESH_END_TIME

日期

物化视图上最近刷新的结束时间（如果尚未填充，则为空白）

泄气

VARCHAR2（19）

物化视图的内容与物化视图主人的内容之间的关系：

  • FRESH - 物化视图是其主人当前状态的读一致视图

  • STALE - 物化视图已过期，因为其中一个或多个主数据已更改。如果物化视图在STALE之前是FRESH ，那么它是对其主人的前状态的读一致视图。

  • NEEDS_COMPILE - 物化视图所依赖的某个对象已更改。ALTER MATERIALIZED VIEW...需要COMPILE语句来验证此物化视图并计算内容的陈旧性。

  • UNUSABLE -物化视图不是它的主人在任何时间点的读一致性视图

  • UNKNOWN - Oracle数据库不知道物化视图是否在任何时间点处于其主控的读一致视图中（在预构建表上创建的物化视图就是这种情况）

  • UNDEFINED - 物化视图具有远程主控。对于这种物化观点，没有定义陈旧性的概念。

AFTER_FAST_REFRESH

VARCHAR2（19）

指定在对此物化视图应用快速刷新时将发生的过期值。其值与STALENESS列的值相同，加上值NA ，当快速刷新不适用于此物化视图时使用该值。

UNKNOWN_PREBUILT

VARCHAR2（1）

指示物化视图是否为预建（ Y ）或不是（ N ）

UNKNOWN_PLSQL_FUNC

VARCHAR2（1）

指示实例化视图是否包含PL / SQL函数（ Y ）（ N ）

UNKNOWN_EXTERNAL_TABLE

VARCHAR2（1）

指示实例化视图是否包含外部表（ Y ）或不包含外部表（ N ）

UNKNOWN_CONSIDER_FRESH

VARCHAR2（1）

指示物化视图是否被视为新鲜（ Y ）或不（ N ）

UNKNOWN_IMPORT

VARCHAR2（1）

指示物化视图是否已导入（ Y ）（ N ）

UNKNOWN_TRUSTED_FD

VARCHAR2（1）

指示物化视图是否使用可信约束进行刷新（ Y ）或不使用（ N ）

COMPILE_STATE

VARCHAR2（19）

物化视图对其所依赖的对象的有效性：

  • VALID - 物化视图已经过验证且没有错误，并且自上次验证以来，它所依赖的对象没有发生变化

  • NEEDS_COMPILE - 物化视图所依赖的某个对象已更改。ALTER MATERIALIZED VIEW...需要COMPILE语句来验证此物化视图。

  • ERROR - 物化视图已通过一个或多个错误验证

USE_NO_INDEX

VARCHAR2（1）

指示是否使用USING NO INDEX子句（ Y ）创建USING NO INDEX化视图，或者使用默认索引（ N ）创建USING NO INDEX化视图。USING NO INDEX子句禁止创建默认索引。

STALE_SINCE

日期

从物化视图变为陈旧的时间

NUM_PCT_TABLES

数

PCT详细信息表的数量

NUM_FRESH_PCT_REGIONS

数

新的PCT分区数量

NUM_STALE_PCT_REGIONS

数

陈旧的PCT分区数量

SEGMENT_CREATED

VARCHAR2（3）

指示是否使用SEGMENT CREATION DEFERRED子句创建了物化视图。如果创建物化视图的段，则值为YES;如果不是，则为NO。

EVALUATION_EDITION

VARCHAR2（128）

解析表达式列中引用的已编辑对象的版本的名称

UNUSABLE_BEFORE

VARCHAR2（128）

最旧版本的名称，其中物化视图的子查询的存储结果可用于查询重写。在指定版本之前的版本中，实体化视图数据的存储结果被视为不可用。如果未指定此类版本，则此值为NULL。

UNUSABLE_BEGINNING

VARCHAR2（128）

最旧版本的名称，其中物化视图的子查询的存储结果不能用于查询重写。对于指定版本和此版本的任何后代中的查询重写，数据不可用。如果未指定此类版本，则此值为NULL。

DEFAULT_COLLATION

VARCHAR2（100）

物化视图的默认排序规则

ON_QUERY_COMPUTATION

VARCHAR2（1）

指示物化视图是否是实时物化视图（ Y ）或不是（ N ）
