创建类型身体

目的

使用PL / SQL定义类型主体。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用CREATE TYPE BODY定义或实现对象类型规范中定义的成员方法。您可以使用CREATE TYPE和CREATE TYPE BODY语句创建对象类型。CREATE TYPE语句指定对象类型的名称，其属性，方法和其他属性。CREATE TYPE BODY语句包含实现该类型的方法的代码。

对于未指定call_spec,的对象类型规范中指定的每个方法call_spec,必须在对象类型主体中指定相应的方法主体。

注意：

如果创建SQLJ对象类型，则将其指定为Java类。

也可以看看：

  • 有关创建类型规范的信息，请参见CREATE TYPE

  • ALTER TYPE有关修改类型规范的信息

先决条件

CREATE TYPE规范中对象类型的每个成员声明必须在CREATE TYPE或CREATE TYPE BODY语句中具有相应的构造。

要在自己的模式中创建或替换类型主体，您必须具有CREATE TYPE或CREATE ANY TYPE系统特权。要在另一个用户的模式中创建对象类型，您必须具有CREATE ANY TYPE系统特权。要替换另一个用户的架构中的对象类型，您必须具有DROP ANY TYPE系统特权。

语义

或替换

指定OR REPLACE以重新创建类型主体（如果已存在）。使用此子句可以更改现有类型主体的定义，而无需先删除它。

以前在重新创建的对象类型主体上授予权限的用户可以使用和引用对象类型主体，而无需再次授予权限。

您可以使用此子句将新成员子程序定义添加到使用ALTER TYPE添加的规范中...REPLACE声明。

[版权所有|不可挽回的]
