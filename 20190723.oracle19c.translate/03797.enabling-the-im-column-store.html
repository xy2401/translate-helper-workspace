要启用或禁用IM列存储，请为INMEMORY_SIZE初始化参数指定值。

3.1启用IM列存储的概述

默认情况下， INMEMORY_SIZE初始化参数设置为0 ，这意味着禁用了IM列存储。

要启用IM列存储，请在重新启动数据库实例之前将初始化参数INMEMORY_SIZE设置为非零值。您可以使用ALTER SYSTEM语句动态增加INMEMORY_SIZE大小设置。

默认情况下，必须使用CREATE或ALTER语句的INMEMORY子句为表，表空间或物化视图指定IM列存储中的INMEMORY 。

3.2估计IM列存储的所需大小

根据您的要求估计IM列存储的大小，然后调整IM列存储的大小以满足这些要求。应用压缩可以减少内存大小。

IM列存储所需的内存量取决于存储在其中的数据库对象以及应用于每个对象的压缩方法。为INMEMORY对象选择压缩方法时，请将性能优势与可用内存量进行平衡：

  • 要最大限度地减小内存大小，请选择FOR CAPACITY HIGH或FOR CAPACITY LOW压缩方法。但是，这些选项在查询执行期间需要额外的CPU来解压缩数据。

  • 要获得最佳查询性能，请选择FOR QUERY HIGH或FOR QUERY LOW压缩方法。但是，这些选项会占用更多内存。

调整IM列存储大小时，请考虑以下准则：

  1. 对于要填充到IM列存储中的每个对象，估计它消耗的内存量。

    Oracle Compression Advisor使用MEMCOMPRESS子句估计您可以实现的压缩率。顾问程序使用DBMS_COMPRESSION接口。

  2. 将各个金额添加到一起。

    注意：

    填充后， V$IM_SEGMENTS显示磁盘上对象的实际大小及其在IM列存储中的大小。您可以使用此信息计算已填充对象的压缩率。但是，如果对象在磁盘上压缩，则此查询不会显示正确的压缩率。

  3. 如果您配置了内存优化算术 ，并且如果内存中表使用FOR QUERY LOW压缩，则添加大约15％来计算NUMBER列的双重存储。

  4. 添加空间以考虑数据库对象的增长，并在DML操作之后存储行的更新版本。

    动态调整大小的最小量为128 MB。

3.3为数据库启用IM列存储

在可以将表或实例化视图填充到IM列存储中之前，必须为数据库启用IM列存储。

在此上下文中，“数据库”是非CDB，CDB或PDB。在CDB中，CDB根目录中的INMEMORY_SIZE设置确定IM列存储的总大小。默认情况下，所有PDB都可以访问IM列存储。
在单个PDB中，您可以通过将INMEMORY_SIZE设置为其他值来限制对共享内存区域的访问。例如，在具有100个PDB的CDB中，您可以在CDB级别将INMEMORY_SIZE设置为20G ，然后在一个PDB中将INMEMORY_SIZE设置为12G ，在第二个PDB中设置为8G ，在剩余的PDB中设置为0 。

先决条件

此任务假定以下内容：

  • 数据库已打开。

  • COMPATIBLE初始化参数设置为12.1.0或更高。

  • INMEMORY_SIZE初始化参数设置为0 （默认值）。

要启用IM列存储：

  1. 在SQL * Plus或SQL Developer中，以具有管理权限的用户身份登录数据库。

  2. 将INMEMORY_SIZE初始化参数设置为非零值。

    最小设置为100M 。

    使用ALTER SYSTEM语句在服务器参数文件（SPFILE）中设置此初始化参数时，必须指定SCOPE=SPFILE 。

    例如，以下语句将内存区域大小设置为10 GB：

    ALTER SYSTEM SET INMEMORY_SIZE = 10G SCOPE = SPFILE;

  3. 关闭数据库，然后重新打开它。

    您必须重新打开数据库才能在SGA中初始化IM列存储。

  4. （可选）检查当前为IM列存储分配的内存量：

    SHOW PARAMETER INMEMORY_SIZE

    注意：

    启用IM列存储后，您可以动态增加其大小，而无需重新打开数据库。

示例3-1启用IM列存储

假设INMEMORY_SIZE初始化参数设置为0 。以下SQL * Plus示例将INMEMORY_SIZE设置为10 GB，关闭数据库实例，然后重新打开数据库以使更改生效：

3.4动态增加IM列存储的大小

如果IM列存储需要更多内存，则可以动态增加其大小。

IM列存储的大小不能动态减少。如果将INMEMORY_SIZE设置为小于其当前设置的值，则必须在ALTER SYSTEM语句中指定SCOPE=SPFILE 。如果通过指定SCOPE=SPFILE设置此参数，则必须重新启动数据库才能使更改生效。

先决条件

要动态增加IM列存储的大小，您必须满足以下先决条件：

  • 必须启用列存储。

  • 兼容级别必须为12.2.0或更高。

  • 必须使用SPFILE启动数据库实例。

  • IM列存储的新大小必须至少比当前INMEMORY_SIZE设置大128 MB。

  1. 在SQL * Plus或SQL Developer中，使用管理权限登录数据库。
  2. （可选）检查当前为IM列存储分配的内存量：
    SHOW PARAMETER INMEMORY_SIZE
  3. 使用指定SCOPE=BOTH或SCOPE=MEMORY的ALTER SYSTEM语句将INMEMORY_SIZE初始化参数设置为大于IM列存储的当前大小的值。

    动态设置此参数时，必须将其设置为高于其当前值的值，并且SGA中必须有足够的可用内存，以便将IM列存储的大小动态增加到新值。

    例如，以下语句将INMEMORY_SIZE动态设置为500M ：
    ALTER SYSTEM SET INMEMORY_SIZE = 500M SCOPE = BOTH;

3.5禁用IM列存储

您可以通过将INMEMORY_SIZE初始化参数设置为零来禁用IM列存储，然后重新打开数据库。

假设

此任务假定在打开的数据库中启用了IM列存储。

要禁用IM列存储：

  1. 在服务器参数文件（SPFILE）中将INMEMORY_SIZE初始化参数设置为0 。

  2. 关闭数据库。

  3. 启动数据库实例，然后打开数据库。
