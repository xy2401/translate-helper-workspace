创建个人资料

注意：

Oracle建议您使用数据库资源管理器而不是此SQL语句来建立资源限制。数据库资源管理器提供了更灵活的方法来管理和跟踪资源使用。有关数据库资源管理器的更多信息，请参阅Oracle数据库管理员指南。

目的

使用CREATE PROFILE语句创建配置文件 ，该配置文件是对数据库资源的一组限制。如果将配置文件分配给用户，则该用户不能超过这些限制。

要为用户指定资源限制，您必须：

  • 使用ALTER SYSTEM语句或初始化参数RESOURCE_LIMIT动态启用资源限制。此参数不适用于密码资源。始终启用密码资源。

  • 使用CREATE PROFILE语句创建定义限制的配置文件

  • 使用CREATE USER或ALTER USER语句将配置文件分配给用户

在多租户环境中，可以将不同的配置文件分配给根和PDB中的公共用户。当普通用户登录到PDB时，其设置适用于会话的配置文件取决于设置是密码相关还是资源相关。

  • 密码相关的配置文件设置是从分配给根目录中的公共用户的配置文件中提取的。例如，假设您将公共配置文件c##prof （其中FAILED_LOGIN_ATTEMPTS设置为1）分配给根目录中的公共用户c##admin 。在PDB中，为用户分配了本地配置文件local_prof （其中FAILED_LOGIN_ATTEMPTS设置为6.）普通用户c##admin只有在尝试登录分配了loc_prof的PDB loc_prof允许一次失败的登录尝试。

  • 在PDB中分配给用户的配置文件中指定的与资源相关的配置文件设置可以在不咨询分配给根中的公共用户的配置文件中的资源相关设置的情况下使用。例如，如果在local_prof中分配给用户c##admin的配置文件local_prof将SESSIONS_PER_USER设置为2，那么当c##admin登录到PDB loc_prof时，只允许2个并发会话分配给他，无论在根目录中分配给他的配置文件中此设置的值如何。

要创建配置文件，您必须具有CREATE PROFILE系统特权。

语义

轮廓

指定要创建的配置文件的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。使用配置文件限制用户可用于单个呼叫或单个会话的数据库资源。

在非CDB中，配置文件名称不能以C##或c##开头。

在CDB中，配置文件名称的要求如下：

  • 公共配置文件的名称必须以与COMMON_USER_PREFIX初始化参数指定的前缀不区分大小写的字符开头。默认情况下，前缀为C## 。

  • 本地配置文件的名称不得以与COMMON_USER_PREFIX初始化参数指定的前缀不区分大小写的字符开头。无论COMMON_USER_PREFIX的值COMMON_USER_PREFIX ，本地配置文件的名称都不能以C##或c##开头。

注意：

如果COMMON_USER_PREFIX的值是空字符串，则对常见或本地配置文件名称没有要求，但有一个例外：本地配置文件的名称永远不能以C##或c##开头。Oracle建议不要使用空字符串值，因为当PDB插入不同的CDB时，或者打开在创建普通用户时关闭的PDB时，它可能会导致本地和公共配置文件的名称之间发生冲突。

Oracle数据库通过以下方式强制执行资源限制：

  • 如果用户超过CONNECT_TIME或IDLE_TIME会话资源限制，则数据库将回滚当前事务并结束会话。当用户进程下一次发出调用时，数据库将返回错误。

  • 如果用户尝试执行超出其他会话资源限制的操作，则数据库将中止该操作，回滚当前语句，并立即返回错误。然后，用户可以提交或回滚当前事务，然后必须结束会话。

  • 如果用户尝试执行超过单个调用限制的操作，则数据库将中止操作，回滚当前语句，并返回错误，使当前事务保持不变。

注意：

  • 对于限制时间的所有参数，您可以使用几天的天数，以天为单位。例如，1小时是1/24，1分钟是1/1440。

  • 无论是否启用资源限制，您都可以为用户指定资源限制。但是，在启用限制之前，Oracle数据库不会强制执行限制。

无限

使用资源参数指定时， UNLIMITED指示分配了此配置文件的用户可以使用无限量的此资源。使用password参数指定时， UNLIMITED表示没有为参数设置限制。

默认

如果要在此配置文件中省略此资源的限制，请指定DEFAULT 。分配了此配置文件的用户受DEFAULT配置文件中指定的此资源的限制。DEFAULT配置文件最初定义了无限的资源。您可以使用ALTER PROFILE语句更改这些限制。

未明确分配配置文件的任何用户都必须遵守DEFAULT配置文件中定义的限制。此外，如果明确分配给用户的配置文件省略某些资源的限制或为某些限制指定DEFAULT ，则用户受DEFAULT配置文件定义的那些资源的限制。

resource_parameters

SESSIONS_PER_USER

指定要限制用户的并发会话数。

CPU_PER_SESSION

指定会话的CPU时间限制，以百分之一秒表示。

CPU_PER_CALL

指定调用的CPU时间限制（解析，执行或获取），以百分之一秒表示。

CONNECT_TIME

指定会话的总运行时间限制，以分钟为单位。

空闲时间

指定会话期间连续非活动时间的允许时间段，以分钟为单位。长时间运行的查询和其他操作不受此限制。

LOGICAL_READS_PER_SESSION

指定在会话中读取的允许数据块数，包括从内存和磁盘读取的块。

LOGICAL_READS_PER_CALL

指定为调用处理SQL语句（解析，执行或获取）的读取所允许的数据块数。

PRIVATE_SGA

指定会话可在系统全局区域（SGA）的共享池中分配的专用空间量。有关该子句的信息，请参阅size_clause 。

注意：

仅当您使用共享服务器体系结构时，此限制才适用。SGA中会话的私有空间包括私有SQL和PL / SQL区域，但不包括共享SQL和PL / SQL区域。

COMPOSITE_LIMIT

指定会话的总资源成本，以服务单位表示。Oracle数据库将总服务单位计算为CPU_PER_SESSION ， CONNECT_TIME ， LOGICAL_READS_PER_SESSION和PRIVATE_SGA的加权和。

password_parameters

使用以下子句设置密码参数。设置时间长度的参数 - 即除FAILED_LOGIN_ATTEMPTS和PASSWORD_REUSE_MAX之外的所有密码参数 - 都以天数来解释。出于测试目的，您可以为这些参数指定分钟（ n / 1440）或甚至秒（ n / 86400）。您也可以为此目的使用小数值（例如.0833大约一小时）。最小值为1秒。最大值是24855天。对于FAILED_LOGIN_ATTEMPTS和PASSWORD_REUSE_MAX ，您必须指定一个整数。

FAILED_LOGIN_ATTEMPTS

指定在锁定帐户之前登录用户帐户的连续失败尝试次数。如果省略此子句，则默认值为10次。

PASSWORD_LIFE_TIME

指定可以使用相同密码进行身份验证的天数。如果还为PASSWORD_GRACE_TIME设置了值，则如果在宽限期内未更改PASSWORD_GRACE_TIME ，则密码将过期，并且拒绝其他连接。如果省略此子句，则默认值为180天。

也可以看看：

有关将PASSWORD_LIFE_TIME设置为较低值的信息，请PASSWORD_LIFE_TIME “Oracle数据库安全指南”

PASSWORD_REUSE_TIME 和PASSWORD_REUSE_MAX

这两个参数必须相互结合设置。PASSWORD_REUSE_TIME指定无法重复使用密码的天数。PASSWORD_REUSE_MAX指定在重用当前密码之前所需的密码更改次数。要使这些参数生效，必须为它们指定一个值。

  • 如果为这两个参数指定了值，则在密码更改为在PASSWORD_REUSE_TIME指定的天数内为PASSWORD_REUSE_MAX指定的次数之前，用户无法重复使用PASSWORD_REUSE_TIME 。

    例如，如果将PASSWORD_REUSE_TIME指定为30并将PASSWORD_REUSE_MAX指定为10，则如果密码已更改10次，则用户可在30天后重复使用密码。

  • 如果为这些参数中的任何一个指定值并为另一个指定UNLIMITED ，则用户永远不能重用密码。

  • 如果为任一参数指定DEFAULT ，则Oracle数据库使用DEFAULT配置文件中定义的值。默认情况下， DEFAULT配置文件中的所有参数都设置为UNLIMITED 。如果尚未在DEFAULT配置文件中更改UNLIMITED的默认设置，则数据库会将该参数的值视为UNLIMITED 。

  • 如果将这两个参数都设置为UNLIMITED ，则数据库将忽略它们。如果省略这两个参数，则这是默认值。

PASSWORD_LOCK_TIME

指定在指定的连续失败登录尝试次数后帐户将被锁定的天数。如果省略此子句，则默认值为1天。

PASSWORD_GRACE_TIME

指定宽限期开始后发出警告并允许登录的天数。如果省略此子句，则默认值为7天。

INACTIVE_ACCOUNT_TIME

指定不允许登录到用户帐户的连续天数，之后将锁定帐户。最小值为15天。没有最大价值。如果省略此子句，则默认值为UNLIMITED 。

PASSWORD_VERIFY_FUNCTION

PASSWORD_VERIFY_FUNCTION子句允许将PL / SQL密码复杂性验证脚本作为参数传递给CREATE PROFILE语句。Oracle数据库提供了一个默认脚本，但您可以创建自己的例程或使用第三方软件。

  • 对于function ，请指定密码复杂性验证例程的名称。该函数必须存在于SYS模式中，并且您必须具有该函数的EXECUTE特权。

  • 指定NULL以指示未执行密码验证。

如果为任何密码参数指定expr ，则表达式可以是除标量子查询表达式之外的任何形式。

密码参数限制

将配置文件分配给外部用户或全局用户时，密码参数不会对该用户生效。

集装箱条款

当您连接到CDB时， CONTAINER子句适用。但是，没有必要指定CONTAINER子句，因为它的默认值是唯一允许的值。

  • 要创建公共配置文件，您必须连接到根。您可以选择指定CONTAINER = ALL ，这是连接到根目录时的默认值。

  • 要创建本地配置文件，您必须连接到PDB。您可以选择指定CONTAINER = CURRENT ，这是连接到PDB时的默认值。

例子

创建配置文件：示例

以下语句创建配置文件new_profile ：

CREATE PROFILE new_profile LIMIT PASSWORD_REUSE_MAX 10 PASSWORD_REUSE_TIME 30;

设置配置文件资源限制：示例

以下语句创建配置文件app_user ：

CREATE PROFILE app_user LIMIT SESSIONS_PER_USER UNLIMITED CPU_PER_SESSION UNLIMITED CPU_PER_CALL 3000 CONNECT_TIME 45 LOGICAL_READS_PER_SESSION DEFAULT LOGICAL_READS_PER_CALL 1000 PRIVATE_SGA 15K COMPOSITE_LIMIT 5000000;

如果将app_user配置文件分配给用户，则后续会话中的用户将受到以下限制：

  • 用户可以拥有任意数量的并发会话。

  • 在单个会话中，用户可以消耗无限量的CPU时间。

  • 用户进行的单次调用不会消耗超过30秒的CPU时间。

  • 单个会话不能持续超过45分钟。

  • 在单个会话中，从内存和磁盘读取的数据块数量受DEFAULT配置文件中指定的限制。

  • 用户进行的单次调用无法从内存和磁盘中读取超过1000个数据块。

  • 单个会话在SGA中不能分配超过15千字节的内存。

  • 在单个会话中，总资源成本不能超过500万个服务单位。计算总资源成本的公式由ALTER RESOURCE COST语句指定。

  • 由于app_user配置文件省略了IDLE_TIME和密码限制的限制，因此用户受DEFAULT配置文件中指定的这些资源的限制。

设置配置文件密码限制：示例

以下语句创建app_user2配置文件，其中设置了密码限制值：

创建配置文件app_user2 LIMIT FAILED_LOGIN_ATTEMPTS 5 PASSWORD_LIFE_TIME 60 PASSWORD_REUSE_TIME 60 PASSWORD_REUSE_MAX 5 PASSWORD_VERIFY_FUNCTION ora12c_verify_function PASSWORD_LOCK_TIME 1/24 PASSWORD_GRACE_TIME 10 INACTIVE_ACCOUNT_TIME 30;

此示例使用默认的Oracle数据库密码验证功能ora12c_verify_function 。有关使用此验证功能或设计自己的验证功能的信息，请参阅Oracle数据库安全指南 。
