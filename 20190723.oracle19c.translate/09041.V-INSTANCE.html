8.8 V $ INSTANCE

V$INSTANCE显示当前实例的状态。

柱 数据类型 描述

INSTANCE_NUMBER

数

用于实例注册的实例编号（对应于INSTANCE_NUMBER初始化参数）

另请参阅： “ INSTANCE_NUMBER ”

INSTANCE_NAME

VARCHAR2（16）

实例的名称

主机名

VARCHAR2（64）

主机名称

版

VARCHAR2（17）

数据库版本

VERSION_LEGACY

VARCHAR2（17）

Oracle Database 18c之前使用的旧数据库版本。此列显示与VERSION列相同的值

VERSION_FULL

VARCHAR2（17）

Oracle Database 18c中引入了具有新Oracle数据库版本方案的版本字符串。

启动时间

日期

实例启动的时间

状态

VARCHAR2（12）

实例的状态：

  • STARTED - STARTUP NOMOUNT

  • MOUNTED - STARTUP MOUNT或ALTER DATABASE CLOSE

  • OPEN - STARTUP或ALTER DATABASE OPEN

  • OPEN MIGRATE - ALTER DATABASE OPEN { UPGRADE | DOWNGRADE }

平行

VARCHAR2（3）

指示实例是否以群集数据库模式（ YES ）安装（ NO ）

线＃

数

由实例打开的重做线程

归档器

VARCHAR2（7）

自动存档状态：

  • 已停止

  • 已启动

  • FAILED - Archiver上次无法归档日志，但会在5分钟内再次尝试

LOG_SWITCH_WAIT

VARCHAR2（15）

日志切换正在等待的事件：

  • 存档日志

  • 清除日志

  • CHECKPOINT

  • NULL - ALTER SYSTEM SWITCH LOGFILE挂起，但当前在线重做日志中有空间

LOGINS

VARCHAR2（10）

指示实例是否在不受限制的模式，允许登录的所有用户（ ALLOWED ，或在受限模式，允许数据库管理员的登录只（ RESTRICTED ）

SHUTDOWN_PENDING

VARCHAR2（3）

指示关闭是否挂起（ YES ）或否（ NO ）

DATABASE_STATUS

VARCHAR2（17）

数据库的状态：

  • 活性

  • 悬

  • 实例恢复

INSTANCE_ROLE

VARCHAR2（18）

指示实例是活动实例（ PRIMARY_INSTANCE ）还是非活动辅助实例（ SECONDARY_INSTANCE ），如果实例已启动但未安装，则指示UNKNOWN

ACTIVE_STATE

VARCHAR2（9）

静默状态：

  • NORMAL - 数据库处于正常状态。

  • QUIESCING - ALTER SYSTEM QUIESCE RESTRICTED已发布：在此实例中不处理新的用户事务，查询或PL / SQL语句。在ALTER SYSTEM QUIESCE RESTRICTED语句之前发出的用户事务，查询或PL / SQL语句不受影响。DBA事务，查询或PL / SQL语句也不受影响。

  • 已发出QUIESCED - ALTER SYSTEM QUIESCE RESTRICTED ：不处理用户事务，查询或PL / SQL语句。DBA事务，查询或PL / SQL语句不受影响。不处理在ALTER SYSTEM QUIESCE RESTRICTED语句之后发出的用户事务，查询或PL / SQL语句。

单个ALTER SYSTEM QUIESCE RESTRICTED语句将ALTER SYSTEM QUIESCE RESTRICTED Oracle RAC环境中的所有实例。发出此语句后，某些实例可能会在其他实例之前进入静默状态;当所有实例进入静止状态时，系统处于静默状态。

BLOCKED

VARCHAR2（3）

指示是否所有服务都被阻止（ YES ）或否（ NO ）

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

INSTANCE_MODE

VARCHAR2（11）

显示当前实例的实例模式。

可能的值：

  • REGULAR ：常规的Oracle RAC实例。此值也始终用于任何非Oracle RAC实例。

  • READ MOSTLY ：一个执行极少数据库写入的Oracle RAC实例

  • READ ONLY ：只读Oracle RAC实例

版

VARCHAR2（7）

数据库的版本。

可能的值包括：

  • CORE EE ：核心企业版

  • EE ：企业版

  • PO ：个人版

  • XE ：Express Edition

家庭

VARCHAR2（80）

仅限内部使用。

DATABASE_TYPE

VARCHAR2（15）

数据库类型：

  • RAC ：如果数据库是可能具有多个实例的常规Oracle RAC数据库。

  • RACONENODE ：如果数据库是Oracle RAC，但只允许一个实例随时运行 - RAC One Node模式。

  • SINGLE ：如果数据库作为单个实例运行。

  • UNKNOWN ：如果无法确定数据库的类型。当数据库使用CRS注册为数据库资源但CRS服务无法返回有效的数据库类型信息时，可能会发生这种情况。通常，这表示CRS服务已关闭或处于故障状态。
