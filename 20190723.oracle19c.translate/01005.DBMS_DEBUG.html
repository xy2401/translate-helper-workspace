54 DBMS_DEBUG

不推荐使用DBMS_DEBUG 。请改用DBMS_DEBUG_JDWP 。

有关更多信息，请参阅DBMS_DEBUG_JDWP 。

DBMS_DEBUG是Oracle服务器中PL / SQL调试器层Probe的PL / SQL接口。

该软件包主要用于实现服务器端调试器，它提供了一种调试服务器端PL / SQL程序单元的方法。

注意：

术语“ 程序单元”是指任何类型的PL / SQL程序（过程，函数，包，包体，触发器，匿名块，对象类型或对象类型主体）。

54.1 DBMS_DEBUG概述

要调试服务器端代码，您必须具有两个数据库会话：一个会话以在调试模式下运行代码（目标会话），另一个会话用于监督目标会话（调试会话）。

通过使用DBMS_DEBUG进行初始化调用，目标会话可用于调试。这标记会话，以便PL / SQL解释器以调试模式运行并生成调试事件。生成调试事件时，会从会话中发布它们。在大多数情况下，调试事件需要返回通知：解释程序暂停等待回复。

同时，调试会话还必须使用DBMS_DEBUG初始化自身：这告诉它要监督哪个目标会话。然后，调试会话可以调用DBMS_DEBUG入口点以读取从目标会话发布的事件并与目标会话通信。

以下子程序在目标会话（要调试的会话）中运行：

  • SYNCHRONIZE函数

  • DEBUG_ON程序

  • DEBUG_OFF程序

DBMS_DEBUG不提供PL / SQL编译器的接口，但它确实依赖于编译器可选生成的调试信息。如果没有调试信息，则无法检查或修改参数或变量的值。

54.2 DBMS_DEBUG常量

断点状态可能具有以下值： breakpoint_status_unused -breakpoint未使用。

否则，状态是以下值的掩码：

  • breakpoint_status_active -a line breakpoint

  • breakpoint_status_disabled -breakpoint目前已被禁用

  • breakpoint_status_remote -a shadow breakpoint（远程断点的本地表示）

54.5 DBMS_DEBUG操作说明

有两种方法可以确保生成调试信息：通过会话切换或通过单独的重新编译。

要设置会话切换，请输入以下语句：

ALTER SESSION SET PLSQL_DEBUG = true;

这指示编译器为会话的剩余部分生成调试信息。它不会重新编译任何现有的PL / SQL。

要为现有PL / SQL代码生成调试信息，请使用以下语句之一（第二个重新编译包或键入主体）：

更改[程序|功能|包装| TRIGGER | TYPE] <name> COMPILE DEBUG;改变[包装] TYPE] <name> COMPILE DEBUG BODY;

控制口译员

解释器在以下时间暂停执行：

  1. 在启动解释器时，可以在执行之前安装任何延迟断点。

  2. 在包含已启用断点的任何行。

  3. 在任何有趣事件发生的行。有趣事件集由传递给DBMS_DEBUG的标志指定。在breakflags参数中CONTINUE 。

会话终止

会话终止没有事件。因此，调试会话负责检查并确保目标会话尚未结束。调用DBMS_DEBUG 。目标会话结束后的SYNCHRONIZE会导致调试会话挂起，直到超时为止。

延期运营

该图表明可以在具有目标会话之前设置断点。这是真的。在这种情况下，Probe会缓存断点请求，并在首次同步时将其发送到目标会话。但是，如果以这种方式延迟断点请求，则：

  • SET_BREAKPOINT不设置断点号（如果需要，可以稍后从SHOW_BREAKPOINTS获取）。

  • SET_BREAKPOINT不验证断点请求。如果请求的源行不存在，则在同步时会以无提示方式发生错误，并且不会设置断点。

诊断输出

要调试Probe， DBMS_DEBUG某些调用有诊断参数。这些参数指定是否将诊断输出放在RDBMS跟踪文件中。如果禁用输出到RDBMS跟踪文件，则这些参数无效。

OER断点

在PL / SQL程序中声明的异常称为用户定义的异常。此外，还有从Oracle内核返回的Oracle错误（OER）。为了将这两种机制联系在一起，PL / SQL提供了exception_init编译指示，它将用户定义的异常转换为OER，以便可以使用PL / SQL处理程序，以便PL / SQL引擎可以将OER返回给Oracle内核。截至目前的版本，有关OER的唯一可用信息是其编号。如果两个用户定义的异常对同一个OER是exception_init，则它们是无法区分的。

命名空间

服务器上的程序单元位于不同的名称空间中。设置断点时，请指定所需的命名空间。

  1. Namespace_cursor包含游标（匿名块）。

  2. Namespace_pgkspec_or_toplevel包含：

    • 包装规格。

    • 未嵌套在其他包，过程或函数中的过程和函数。

    • 对象类型。

  3. Namespace_pkg_body包含包体和类型主体。

  4. Namespace_trigger包含触发器。

54.7.18 PING程序

此过程ping目标会话以防止它超时。在目标会话中暂停执行时使用此过程，例如在断点处。

如果timeout_behaviour设置为retry_on_timeout则不需要此过程。

句法

DBMS_DEBUG.PING;

例外

如果没有目标程序或者目标会话当前没有等待来自调试会话的输入，Oracle将显示no_target_program异常。

使用说明

通过调用set_timeout_behaviour:向目标会话注册目标会话的超时选项set_timeout_behaviour:

  • retry_on_timeout - 重试。超时无效。这就像将超时设置为无限大的值。

  • continue_on_timeout - 使用相同的事件标志继续执行。

  • nodebug_on_timeout - 关闭调试模式（换句话说，调用debug_off），然后继续执行。除非通过调用debug_on重新初始化，否则此目标会话不会生成更多事件。

  • abort_on_timeout - 使用abort_execution标志继续执行，该标志应该导致程序立即终止。会话保持在调试模式。

retry_on_timeout CONSTANT BINARY_INTEGER：= 0;

continue_on_timeout CONSTANT BINARY_INTEGER：= 1;

nodebug_on_timeout CONSTANT BINARY_INTEGER：= 2;

abort_on_timeout CONSTANT BINARY_INTEGER：= 3;
