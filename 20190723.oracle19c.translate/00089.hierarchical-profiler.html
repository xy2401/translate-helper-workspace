15使用PL / SQL分层分析器

您可以使用PL / SQL分层分析器来识别PL / SQL应用程序中的瓶颈和性能调优机会。

分析器报告由函数调用组织的PL / SQL程序的动态执行配置文件，并分别考虑SQL和PL / SQL执行时间。无需特殊来源或编制时准备;任何PL / SQL程序都可以进行分析。

本章介绍PL / SQL分层分析器，并说明如何使用它来收集和分析PL / SQL程序的概要文件数据。

PL / SQL分层分析器概述

非等级（ 平面 ）剖面仪记录程序在每个子程序中花费的时间 - 每个子程序的功能时间或自身时间 。功能时间很有帮助，但往往不够。例如，知道程序在子程序INSERT_ORDER花费40％的时间是有帮助的，但知道哪些子程序经常调用INSERT_ORDER以及程序在INSERT_ORDER下花费的总时间（包括其子代子程序）更有帮助。分层分析器提供此类信息。

PL / SQL分层分析器：

  • 报告PL / SQL程序的动态执行配置文件，由子程序调用组织

  • 分别考虑SQL和PL / SQL执行时间

  • 不需要特殊的源代码或编译时准备

  • 通过集成开发环境（IDE）工具（如SQL Developer和第三方工具）存储数据库表（ 分层分析器表 ）以生成自定义报告的结果

  • 提供子程序级执行摘要信息，例如：

    • 子程序的调用次数

    • 在子程序本身花费的时间 （ 功能时间或自我时间 ）

    • 在子程序本身及其后代子程序中花费的时间 （ 子树时间 ）

    • 详细的父子信息，例如：

      • 给定子程序的所有呼叫者（父母）

      • 给定子程序调用的所有子程序（子项）

      • 从y调用子程序x时花了多少时间

      • 子程序x的调用次数来自y

PL / SQL层次分析器由DBMS_HPROF包实现，有两个组件：

  • 数据采集

    数据收集组件是PL / SQL虚拟机的固有部分。DBMS_HPROF包提供API以打开和关闭分层分析，并将原始分析器输出写入文件或原始分析器数据表。

  • 分析仪

    分析器组件处理原始分析器输出并生成分析结果。分析器组件分析：

    • 原始分析器数据位于原始分析器数据文件和原始分析器数据表中，分为HTML CLOB报告，分析报告文件和分层分析器分析表。

    注意：

    要直接从原始探查器输出生成简单的HTML报告，而不使用分析器，可以使用plshprof命令行实用程序。

收集个人资料数据

要从PL / SQL分层探查器的PL / SQL程序收集配置文件数据，请按照下列步骤操作：

  1. 确保您拥有以下权限：

    • DBMS_HPROF包的EXECUTE特权

    • 对调用DBMS_HPROF时指定的目录的WRITE特权。 START_PROFILING

  2. 使用DBMS_HPROF 。 START_PROFILING PL / SQL API在会话中启动分层探查器数据收集。
  3. 运行PL / SQL程序足够长的时间以获得足够的代码覆盖率。

    要获得最准确的测量经过时间，请避免运行PL / SQL程序的系统上的不相关活动。

  4. 使用DBMS_HPROF 。 STOP_PROFILING PL / SQL API停止分层探查器数据收集。

创建分层Profiler表

以下步骤说明了如何在表15-3中创建分层探查器表以及持久存储概要文件数据所需的其他数据结构，运行DBMS_HPROF程序包所需的权限以及生成自定义报告：

  1. 表15-3中的分层分析器表和持久存储配置文件数据所需的其他数据结构可以通过以下方式创建。
    1. 调用DBMS_HPROF 。 CREATE_TABLES过程。
    2. 运行脚本dbmshptab 。 sql （位于目录rdbms/admin ）。

      注意：

      运行脚本dbmshptab 。 sql删除任何以前创建的分层探查器表。

      注意：

      dbmshptab 。 sql （位于目录rdbms/admin ）已被弃用。此脚本包含删除表和序列以及弃用说明的语句。

  2. 确保您拥有以下权限：

    • DBMS_HPROF包的EXECUTE特权

    • 对DBMS_HPROF目录的READ特权。 analyze指定

  3. 使用PL / SQL API DBMS_HPROF 。 analyze以分析单个原始分析器输出文件，并将结果存储在分层分析器表中。

    （有关原始探查器输出文件的示例，请参阅了解原始探查器输出中的 test trc 。）

    有关DBMS_HPROF更多信息。 analyze ，请参阅Oracle Database PL / SQL包和类型参考 。

  4. 使用分层探查器表生成自定义报告。

示例15-3调用DBMS_HPROF.analyze

DECLARE runid NUMBER; BEGIN runid：= DBMS_HPROF.analyze（LOCATION =>'PLSHPROF_DIR'， FILENAME =>'test.trc'）; DBMS_OUTPUT.PUT_LINE（'runid ='|| runid）;结束; /

例15-3中的匿名块：

  • 调用函数DBMS_HPROF 。 analyze功能，其中：

    • 分析原始探查器输出文件test的配置文件数据。 trc （来自Understanding Raw Profiler Output ），它位于映射到目录对象PLSHPROF_DIR的目录中，并将数据存储在表15-3中的分层分析器表中。

    • 返回可用于查询表15-3中的分层分析器表的唯一标识符。（通过查询这些分层的分析器表，您可以生成自定义报告。）

  • 将唯一标识符存储在它打印的变量runid 。

重载的子程序是具有相同名称的不同子程序。

假设程序声明了三个名为compute的子程序 - 第50行第一个，第76行第二个，第100行第三个子程序。在DBMSHP_FUNCTION_INFO表中，每个子程序都在FUNCTION列中进行了compute 。要区分这三个子程序，请使用LINE#列（第一个子程序为50，第二个子程序为76，第三个子程序为100）或HASH列（每个子程序具有唯一值）。

在两个不同运行的配置文件数据中，函数的LINE#和HASH值可能不同。如果在函数定义之前插入或删除行，则函数的LINE#值会更改。只有当函数的签名发生变化时， HASH值才会改变;例如，如果更改参数列表。

差异报告惯例

差异报告使用以下约定：

  • 在报告标题中， Delta表示差异或变化 。

  • 正值表示从第一次运行到第二次运行的数量增加（ 回归 ）。

  • 差值的负值表示从第一次运行到第二次运行的数量减少（ 改善 ）。

  • 函数名后的符号#表示仅在一次运行中调用该函数。

功能级差异报告

对于每个函数，每个函数级差异报告包括从第一次运行到第二次运行的这些值的变化：

  • 功能时间（在功能本身中花费的时间，也称为“自我时间”）

  • 后代时间（在函数后代中花费的时间）

  • 子树时间（在函数 - 函数时间的子树中花费的时间加上后代时间）

  • 对函数的调用次数

  • 平均功能时间

    平均功能时间是功能时间除以对功能的调用次数。

  • 功能名称

    函数名称超链接到函数的“父母和子女差异报告”。
