SYS_CONTEXT

句法


说明sys_context.eps的说明

目的

SYS_CONTEXT返回当前时刻与上下文namespace关联的parameter值。您可以在SQL和PL / SQL语句中使用此函数。SYS_CONTEXT必须在本地执行。

对于namespace和parameter ，您可以指定解析为指定名称空间或属性的字符串的字符串或表达式。如果为namespace和parameter指定了文字parameter ，并且在SQL语句中显式使用SYS_CONTEXT而不是在SQL语句中提到的PL / SQL函数中 - 那么Oracle数据库每个SQL语句只评估一次SYS_CONTEXT调用SYS_CONTEXT函数的每个调用站点的执行。

必须已经创建了上下文namespace ，并且还必须使用DBMS_SESSION设置关联的parameter及其值。 set_context过程。namespace必须是有效的标识符。parameter名称可以是任何字符串。它不区分大小写，但长度不能超过30个字节。

返回值的数据类型是VARCHAR2 。返回值的默认最大大小为256个字节。您可以通过指定可选的length参数来覆盖此默认值，该参数必须是NUMBER或可以隐式转换为NUMBER 。有效值范围是1到4000个字节。如果指定了无效值，则Oracle数据库会忽略它并使用默认值。

Oracle提供以下内置命名空间：

  • USERENV - 描述当前会话。表7-11中列出了命名空间USERENV的预定义参数。

  • SYS_SESSION_ROLES - 指示当前是否为会话启用了指定的角色。

以下语句返回登录数据库的用户的名称：

CONNECT OE输入密码： 密码 SELECT SYS_CONTEXT（'USERENV'，'SESSION_USER'）FROM DUAL; SYS_CONTEXT（'USERENV'，'SESSION_USER'）--------------------------------------- OE

表7-11命名空间USERENV的预定义参数

参数 回报价值

行动

标识模块中的位置（应用程序名称），并通过DBMS_APPLICATION_INFO程序包或OCI设置。

AUDITED_CURSORID

返回触发审计的SQL的游标ID。此参数在细粒度审计环境中无效。如果在此类环境中指定它，则Oracle数据库始终返回null。

AUTHENTICATED_IDENTITY

返回身份验证中使用的标识。在下面的列表中，用户的类型后跟返回的值：

  • 经过Kerberos身份验证的企业用户：kerberos主体名称

  • Kerberos认证的外部用户：kerberos主要名称;与架构名称相同

  • 经过SSL身份验证的企业用户：用户PKI证书中的DN

  • 经过SSL身份验证的外部用户：用户PKI证书中的DN

  • 经过密码验证的企业用户：昵称;与登录名相同

  • 经过密码验证的数据库用户：数据库用户名;与架构名称相同

  • 经过OS身份验证的外部用户：外部操作系统用户名

  • Radius身份验证的外部用户：架构名称

  • 带DN的代理：客户端的Oracle Internet Directory DN

  • 带证书的代理：客户端的证书DN

  • 代理用户名：数据库用户名，如果客户端是本地数据库用户;如果客户端是企业用户，则为昵称。

  • SYSDBA / SYSOPER使用密码文件：登录名

  • 使用OS身份验证的SYSDBA / SYSOPER：操作系统用户名

AUTHENTICATION_DATA

用于验证登录用户的数据。对于X.503证书认证会话，此字段以HEX2格式返回证书的上下文。

注意：您可以使用语法的length参数更改AUTHENTICATION_DATA属性的返回值。最高可达4000。这是USERENV的唯一属性，Oracle数据库为此实现了此类更改。

身份验证方法

返回身份验证方法。在下面的列表中，用户的类型后跟返回的方法：

  • 使用密码文件的密码验证企业用户，本地数据库用户或具有SYSDBA或SYSOPER管理权限的用户;使用密码的用户名代理：PASSWORD

  • 使用密码文件的密码验证企业用户，本地数据库用户或具有SYSDBA或SYSOPER管理权限的用户;使用密码的用户名代理：PASSWORD_GLOBAL

  • 经过Kerberos身份验证的企业用户或外部用户（没有管理权限）：KERBEROS

  • 经过Kerberos身份验证的企业用户（具有管理权限）：KERBEROS_GLOBAL

  • 经过Kerberos身份验证的外部用户（具有管理权限）：KERBEROS_EXTERNAL

  • 经过SSL身份验证的企业或外部用户（没有管理权限）：SSL

  • 经过SSL身份验证的企业用户（具有管理权限）：SSL_GLOBAL

  • 经过SSL身份验证的外部用户（具有管理权限）：SSL_EXTERNAL

  • Radius身份验证的外部用户：RADIUS

  • 经过OS身份验证的外部用户或与SYSDBA或SYSOPER管理权限一起使用：操作系统

  • 使用证书，DN或用户名而不使用密码的代理：无

  • 后台进程（作业队列从属进程）：JOB

  • 并行查询从属进程：PQ_SLAVE

对于非管理连接，当身份验证方法为PASSWORD，KERBEROS或SSL时，可以使用IDENTIFICATION_TYPE区分外部用户和企业用户。对于管理连接， AUTHENTICATION_METHOD足以用于PASSWORD，SSL_EXTERNAL和SSL_GLOBAL身份验证方法。

BG_JOB_ID

当前会话的作业ID（如果它是由Oracle数据库后台进程建立的）。如果会话未由后台进程建立，则为空。

CDB_NAME

如果在连接到多租户容器数据库（CDB）时进行查询，则返回CDB的名称。否则，返回null。

CLIENT_IDENTIFIER

返回应用程序通过DBMS_SESSION.SET_IDENTIFIER过程，OCI属性OCI_ATTR_CLIENT_IDENTIFIER或Oracle动态监视服务（DMS）设置的标识符。各种数据库组件使用此属性来标识作为同一数据库用户进行身份验证的轻量级应用程序用户。

CLIENT_INFO

返回最多64个字节的用户会话信息，可由应用程序使用DBMS_APPLICATION_INFO程序包存储。

CLIENT_PROGRAM_NAME

用于数据库会话的程序的名称。

CON_ID

如果在连接到CDB时查询，则返回当前容器ID。否则，返回0。

CON_NAME

如果在连接到CDB时查询，则返回当前容器名称。否则，返回DB_NAME初始化参数中指定的数据库名称。

CURRENT_BIND

用于细粒度审计的绑定变量。您只能在事件处理程序内为细粒度审计功能指定此属性。

CURRENT_EDITION_ID

当前版本的标识符。

CURRENT_EDITION_NAME

当前版本的名称。

CURRENT_SCHEMA

当前活动的默认架构的名称。通过使用ALTER SESSION SET CURRENT_SCHEMA语句，此值可能在会话期间发生ALTER 。这也可能在会话期间发生变化，以反映任何活动定义者权限对象的所有者。直接在视图定义的主体中使用时，将返回执行使用视图的游标时使用的默认模式;它不会将游标中使用的视图视为定义者的权限。

注意 ：Oracle建议不要在除登录触发器之外的所有类型的存储PL / SQL单元中发出SQL语句ALTER SESSION SET CURRENT_SCHEMA 。

CURRENT_SCHEMAID

当前活动的默认架构的标识符。

CURRENT_SQL

CURRENT_SQL n

CURRENT_SQL返回触发细粒度审计事件的当前SQL的前4K字节。CURRENT_SQL n属性返回后续的4K字节增量，其中n可以是1到7的整数，包括1和7。CURRENT_SQL1返回4K到8K的字节; CURRENT_SQL2返回字节8K到12K，依此类推。您只能在事件处理程序内为细粒度审计功能指定这些属性。

CURRENT_SQL_LENGTH

触发细粒度审计或行级安全性（RLS）策略函数或事件处理程序的当前SQL语句的长度。您只能在事件处理程序内为细粒度审计功能指定此属性。

当前用户

权限当前处于活动状态的数据库用户的名称。在附加或分离Real Application Security会话期间，这可能会在数据库会话期间发生更改，或者反映任何活动定义者权限对象的所有者。当没有定义者的权限对象处于活动状态时， CURRENT_USER返回与SESSION_USER相同的值。当直接在视图定义的主体中使用时，这将返回正在执行使用视图的游标的用户;它不会将游标中使用的视图视为定义者的权限。对于企业用户，返回架构。如果Real Application Security用户当前处于活动状态，则返回用户XS$NULL 。

另请参阅： Oracle Database 2 Day +安全指南 ，了解有关用户XS$NULL更多信息

CURRENT_USERID

特权当前处于活动状态的数据库用户的标识符。

DATABASE_ROLE

使用带有USERENV名称空间的SYS_CONTEXT函数的数据库角色。该角色是以下之一： PRIMARY ， PHYSICAL STANDBY ， LOGICAL STANDBY ， SNAPSHOT STANDBY 。

DB_DOMAIN

DB_DOMAIN初始化参数中指定的数据库的域。

DB_NAME

DB_NAME初始化参数中指定的数据库的名称。

DB_SUPPLEMENTAL_LOG_LEVEL

如果启用了补充日志记录，则返回包含已启用的补充日志记录级别列表的字符串。可能的值有： ALL_COLUMN ， FOREIGN_KEY ， MINIMAL ， PRIMARY_KEY ， PROCEDURAL和UNIQUE_INDEX 。如果未启用补充日志记录，则返回null。

DB_UNIQUE_NAME

DB_UNIQUE_NAME初始化参数中指定的数据库的名称。

DBLINK_INFO

返回数据库链接会话的源。具体来说，它返回一个表单的字符串：

SOURCE_GLOBAL_NAME = dblink_src_global_name ，DBLINK_NAME = dblink_name ，SOURCE_AUDIT_SESSIONID = dblink_src_audit_sessionid

哪里：

  • dblink_src_global_name是源数据库的唯一全局名称

  • dblink_name是源数据库上数据库链接的名称

  • dblink_src_audit_sessionid是源数据库上使用dblink_name启动与远程数据库的连接的会话的审计会话ID

ENTRYID

当前审核条目号。审计entryid序列在细粒度审计记录和常规审计记录之间共享。您不能在分布式SQL语句中使用此属性。只有通过标准或细粒度审计的审计处理程序才能看到正确的审计条目标识符。

ENTERPRISE_IDENTITY

返回用户的企业范围标识：

  • 对于企业用户：Oracle Internet Directory DN。

  • 对于外部用户：外部标识（Kerberos主体名称，Radius模式名称，OS用户名，证书DN）。

  • 对于本地用户和SYSDBA / SYSOPER登录： NULL 。

属性的值因代理方法而异：

  • 对于具有DN的代理：客户端的Oracle Internet Directory DN

  • 对于具有证书的代理：外部用户的客户端证书DN;全球用户的Oracle Internet Directory DN

  • 对于具有用户名的代理：如果客户端是企业用户，则为Oracle Internet Directory DN;如果客户端是本地数据库用户，则为空。

FG_JOB_ID

如果从使用DBMS_JOB程序包创建的作业中查询：如果客户端前台进程已建立，则返回当前会话的作业ID。如果会话未由前台进程建立，则为空。

否则：返回0。

GLOBAL_CONTEXT_MEMORY

返回全局访问的上下文在系统全局区域中使用的数字。

GLOBAL_UID

从Oracle Internet Directory for Enterprise User Security（EUS）登录中返回全局用户ID;对所有其他登录名返回null。

主办

IDENTIFICATION_TYPE

返回在数据库中创建用户架构的方式。具体来说，它反映了CREATE / ALTER USER语法中的IDENTIFIED子句。在下面的列表中，模式创建期间使用的语法后跟返回的标识类型：

  • IDENTIFIED BY password ： LOCAL

  • EXTERNALLY EXTERNAL IDENTIFIED ： EXTERNAL

  • IDENTIFIED GLOBALLY ： GLOBAL SHARED

  • IDENTIFIED GLOBALLY AS DN ： GLOBAL PRIVATE

  • GLOBAL EXCLUSIVE专属全球用户映射。

  • GLOBAL SHARED用于共享用户映射。

INSTANCE_NAME

IP地址

IS_APPLY_SERVER

IS_DG_ROLLING_UPGRADE

语言

LDAP_SERVER_TYPE

MODULE

NETWORK_PROTOCOL
