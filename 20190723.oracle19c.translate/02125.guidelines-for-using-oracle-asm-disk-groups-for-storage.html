使用Oracle ASM磁盘组进行存储的准则

规划您希望如何配置Oracle ASM磁盘组以进行部署。

在Oracle Grid Infrastructure安装期间，您可以创建一个或两个Oracle ASM磁盘组。安装Oracle Grid Infrastructure之后，您可以使用Oracle自动存储管理配置助手（ASMCA），SQL * Plus或自动存储管理命令行实用程序（ASMCMD）创建其他磁盘组。

选择在Oracle Grid Infrastructure安装期间创建第二个磁盘组。第一个磁盘组存储Oracle Cluster Registry（OCR），投票文件和Oracle ASM密码文件。第二个磁盘组存储Grid Infrastructure Management Repository（GIMR）数据文件和Oracle Cluster Registry（OCR）备份文件。Oracle强烈建议您将OCR备份文件存储在与存储OCR文件的磁盘组不同的磁盘组中。此外，为存储的性能，可用性，大小和可管理性建议使用GIMR的第二个磁盘组。

注意：

  • 您必须在安装Oracle Grid Infrastructure时指定Grid Infrastructure Management Repository（GIMR）位置。您以后无法将GIMR从一个磁盘组迁移到另一个磁盘组。

  • 对于Oracle域服务群集，您必须配置两个单独的Oracle ASM磁盘组，一个用于OCR和投票文件，另一个用于GIMR。

如果在安装Oracle Grid Infrastructure之后安装Oracle数据库或Oracle RAC，则可以将相同的磁盘组用于数据库文件，OCR和投票文件，也可以使用不同的磁盘组。如果在安装Oracle RAC之前或创建数据库之前创建多个磁盘组，则可以执行以下操作之一：

  • 将数据文件放在与Oracle Clusterware文件相同的磁盘组中。

  • 对数据文件和恢复文件使用相同的Oracle ASM磁盘组。

  • 为每种文件类型使用不同的磁盘组。

如果仅为存储创建一个磁盘组，则OCR和表决文件，数据库文件和恢复文件将包含在一个磁盘组中。如果为存储创建多个磁盘组，则可以将文件放在不同的磁盘组中。

对于Oracle Database 11g第2版（11.2）及更高版本，Oracle数据库配置助手（DBCA）不具备为Oracle ASM创建磁盘组的功能。
