12为XML C API打包XmlDiff

包XmlDiff的方法允许您比较和修改XML文档。XmlDiff（）和XmlPatch（）方法通常等同于UNIX命令diff和patch ，另外还针对XML进行了优化和识别 。

下表总结了通过XmlDiff包为XML C API提供的方法。

的xmldiff（）

确定两个XML文档之间的更改。

XmlDiff（）以符合Xdiff XML模式的XML格式捕获两个文档之间的diff ;您可以自定义此输出。

这些输入文档可以指定为DOM树，文件，URI， orastream等。如果它们不作为DOM树提供，则将创建两个输入的DOM树。创建diff文档的DOM，并返回doc节点。

如果调用者将输入提供为DOM，则不会释放DOM的内存。

两个文档的数据（DOM）编码必须与xctx的数据编码xctx 。diff DOM将以xctx指定的数据编码创建。

有四种算法可以在XmlDiff（）中运行：全局，本地，带散列的全局和带散列的本地。在四种情况下， diff可能不同。

全局算法将使用插入，追加，删除和更新操作生成最小diff 。它需要比本地算法更多的内存和时间。本地算法可能不会生成最小差异，但是比全局算法更快并且使用更少的空间。

散列可以与全局和本地算法一起使用。如果散列与全局算法一起使用，它将显着加速差异计算，但可能会降低差异的质量。使用局部算法，它可以提高diff的质量。

您必须指定使用散列的深度。在散列中，每个元素节点的散列值与以该节点为根的整个子树的摘要相关联。在计算diff不会在指定的散列级别深度之外调查树。

具有或不具有散列的全局算法的输出满足“在docorder中操作”的要求（节点必须以与文档树的前序遍历相同的顺序出现），但是本地算法的输出不是。

XmlDiff（）将在xdiff文档中使用的名称空间前缀可能与第一个或第二个doc的名称空间前缀相同，具体取决于处理时首先看到的前缀。NS URI将适当地绑定到输出中的前缀。如果此NS在两个doc中都没有前缀，则会生成一个新前缀并绑定到xdiff doc.的NS doc.

XmlHash（）

计算XML文档或DOM中的节点的哈希值。

如果两个XML子树的哈希值相等，则相应的子树等于非常高的概率。使用消息摘要算法5（MD5）计算散列值，消息摘要算法5是一种广泛使用的具有128位散列值的加密散列函数，因此两个不同输入可能映射到相同MD5摘要的概率非常小。

源可以指定为文件，URL等。它也可以是DOM中的Document节点或任何其他DOM节点，并且必须使用inputSource参数指定。如果inputSource是非Document DOM节点，则inputSourceExtra必须指向DOM的Document节点。

XmlPatch（）

XmlPatch（）将符合Xdiff模式的更改应用于输入文档。输入文档和diff文档可以指定为DOM树，文件，URI或缓冲区。

如果它们不作为DOM提供，则为输入和diff文档构建DOM。

输入和差异文档的数据（DOM）编码必须与xctx的数据编码xctx 。修补后的DOM将采用xctx指定的数据编码。

快照模型中仅支持简单的XPath。XPath应该用缩写语法识别带有posistion谓词的节点，例如/a[1]/b[2] 。XmlDiff（）生成的XPath满足此要求。此外，'在docorder中运行'条件必须为TRUE ;节点必须以与文档树的前序遍历相同的顺序出现。全局（有或没有散列）满足此要求。本地没有。

编程接口应指定diff doc中使用的输出模型。oracle-xmldif应该是顶级xdiff元素的第一个子元素。它还应该使用标志来指定操作是否按文档顺序（ TRUE或FALSE ），并且输出模型是快照还是当前。
