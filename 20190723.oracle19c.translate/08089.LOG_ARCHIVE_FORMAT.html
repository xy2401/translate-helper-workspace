1.175 LOG_ARCHIVE_FORMAT

归档重做日志文件时，使用LOG_ARCHIVE_FORMAT指定默认文件名格式。

属性 描述

参数类型

串

句法

LOG_ARCHIVE_FORMAT = filename

默认值

依赖于操作系统

可修改

没有

可在PDB中修改

没有

价值范围

任何解析为有效文件名的字符串

基本

没有

Oracle RAC

多个实例可以具有不同的值，但建议使用相同的值。

只有在ARCHIVELOG模式下使用重做日志时， LOG_ARCHIVE_FORMAT才适用。归档重做日志文件时，使用文本字符串和变量指定默认文件名格式。从此格式生成的字符串将附加到LOG_ARCHIVE_DEST参数中指定的字符串。

以下变量可以采用以下格式：

%s日志序列号

%S日志序列号，填零

%t线程号

%T线号，零填充

%a激活ID

%d数据库ID

%r resetlogs ID，用于确保跨多个数据库版本为归档日志文件构造唯一名称

对变量使用大写字母（例如， %S ）会使值固定为长度，并用零填充到左侧。指定存档重做日志文件名格式的示例如下：

LOG_ARCHIVE_FORMAT ='log％t_％s_％r.arc'

归档日志文件名必须包含每个元素%s （序列）， %t （线程）和%r （重置日志ID），以确保所有归档日志文件名都是唯一的。如果在参数文件中设置了LOG_ARCHIVE_FORMAT初始化参数，则确保参数值包含%s ， %t和%r元素。否则，实例启动时会显示以下错误：

ORA-19905：log_archive_format必须包含％s，％t和％r

LOG_ARCHIVE_DEST和LOG_ARCHIVE_FORMAT都不必是完整的文件或目录说明符本身;在将变量替换为LOG_ARCHIVE_FORMAT并将两个参数连接在一起后，它们只需要形成一个有效的文件路径。

在这些情况下， LOG_ARCHIVE_FORMAT被忽略：

  • 对于转到快速恢复区域的归档日志文件

  • 当LOG_ARCHIVE_DEST[_n]指向Oracle ASM磁盘组的根时（例如， +DATA ）。必须为要遵守的参数指定磁盘组的目录（例如，+ DATA / logs）。

在忽略LOG_ARCHIVE_FORMAT情况下，将使用Oracle ASM文件名。有关Oracle ASM文件名的详细信息，请参见“ Oracle自动存储管理管理员指南” 。
