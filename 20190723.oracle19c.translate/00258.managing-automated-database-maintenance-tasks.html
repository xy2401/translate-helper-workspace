26管理自动数据库维护任务

Oracle数据库已自动执行通常由数据库管理员执行的几项常见维护任当预期系统负载较轻时，执行这些自动维护任务。您可以启用和禁用单个维护任务，并可以配置这些任务的运行时间以及分配的资源分配。

注意：

本章介绍如何使用PL / SQL包管理自动维护任务。更简单的方法是使用Oracle企业管理器云控制（云控制）的图形界面。

使用Cloud Control管理自动维护任务：

  1. 访问数据库主页。

  2. 从Administration菜单中，选择Oracle Scheduler ，然后选择Automated Maintenance Tasks 。

  3. 在“自动维护任务”页面上，单击“ 配置” 。

  • 关于自动维护任务
    自动维护任务是定期自动启动以对数据库执行维护操作的任务。一个示例是收集查询优化器的架构对象的统计信息的任务。
  • 关于维护Windows
    维护窗口是连续的时间间隔，在此期间运行自动维护任务。维护窗口是属于名为MAINTENANCE_WINDOW_GROUP的窗口组的Oracle Scheduler窗口。
  • 配置自动维护任务
    要在任何维护窗口子集中启用或禁用特定维护任务，可以使用DBMS_AUTO_TASK_ADMIN PL / SQL程序包。
  • 配置维护Windows
    您可能希望将预定义的维护时段调整为适合您的数据库环境的时间或创建新的维护时段。您可以使用DBMS_SCHEDULER PL / SQL包自定义维护窗口。
  • 为自动维护任务配置资源分配
    您可以减少或增加自动维护任务的资源分配。
  • 自动维护任务参考
    Oracle数据库具有预定义的维护窗口它还具有数据字典视图，您可以查询有关自动维护的信息。

26.1关于自动维护任务

自动维护任务是定期自动启动以对数据库执行维护操作的任务。一个示例是收集查询优化器的架构对象的统计信息的任务。

自动维护任务在维护窗口中运行， 维护窗口是预定义的时间间隔，旨在在系统负载较低的时间段内发生。您可以根据数据库的资源使用模式自定义维护窗口，或禁用某些默认窗口运行。您还可以创建自己的维护窗口。

Oracle Database具有以下预定义的自动维护任务：

  • 自动优化程序统计信息收集 -收集数据库中没有统计信息或仅有过时统计信息的所有模式对象的优化程序统计信息。SQL查询优化器使用此任务收集的统计信息来提高SQL执行的性能。

26.2关于维护Windows

维护窗口是连续的时间间隔，在此期间运行自动维护任务。维护窗口是属于名为MAINTENANCE_WINDOW_GROUP的窗口组的Oracle Scheduler窗口。

“计划程序”窗口可以是简单的重复间隔（例如“午夜至早上6点，每周六”之间），也可以是更复杂的间隔（例如“每月最后一个工作日的午夜至早上6点之间，不包括公司假期“）。

当维护窗口打开时，Oracle数据库会为计划在该窗口中运行的每个维护任务创建Oracle Scheduler作业。为每个作业分配一个在运行时生成的作业名称。所有自动维护任务作业名称ORA$AT开头。例如，自动段顾问的作业可能被称为ORA$AT_SA_SPC_SY_26 。自动维护任务作业完成后，将从Oracle Scheduler作业系统中删除该作业。但是，仍可在“计划程序”作业历史记录中找到该作业。

注意：

要查看作业历史记录，您必须以SYS用户身份登录。

在维护时间非常长的情况下，除自动SQL调优顾问之外的所有自动维护任务每四小时重新启动一次。无论窗口大小如何，此功能都可确保定期运行维护任务。

26.3配置自动维护任务

要在任何维护窗口子集中启用或禁用特定维护任务，可以使用DBMS_AUTO_TASK_ADMIN PL / SQL程序包。

  • 为所有维护Windows启用和禁用维护任务
    通过单个操作，您可以为所有维护窗口禁用或启用特定的自动维护任务。
  • 启用和禁用特定维护Windows的维护任务
    默认情况下，所有维护任务都在所有预定义维护窗口中运行您可以禁用特定窗口的维护任务。

父主题： 管理自动数据库维护任务

26.3.1为所有维护Windows启用和禁用维护任务

通过单个操作，您可以为所有维护窗口禁用或启用特定的自动维护任务。

您可以通过一次操作为所有维护窗口禁用特定的自动维护任务。您可以通过调用DBMS_AUTO_TASK_ADMIN PL / SQL包的DISABLE过程而不提供window_name参数来实现。例如，您可以完全禁用Automatic SQL Tuning Advisor任务，如下所示：

BEGIN dbms_auto_task_admin.disable（client_name =>'sql tuning advisor'，operation => NULL，window_name => NULL）;结束; /

要再次启用此维护任务，请使用ENABLE过程，如下所示：

BEGIN dbms_auto_task_admin.enable（client_name =>'sql tuning advisor'，operation => NULL，window_name => NULL）;结束; /

DBA_AUTOTASK_CLIENT数据库字典视图中列出了用于client_name参数的任务名称。

要为所有窗口启用或禁用所有自动维护任务，请在不带参数的情况下调用ENABLE或DISABLE过程。 

EXECUTE DBMS_AUTO_TASK_ADMIN.DISABLE;

26.3.2启用和禁用特定维护Windows的维护任务

26.4配置维护Windows

您可能希望将预定义的维护时段调整为适合您的数据库环境的时间或创建新的维护时段。您可以使用DBMS_SCHEDULER PL / SQL包自定义维护窗口。

  • 修改维护窗口
    DBMS_SCHEDULER PL / SQL包包含用于修改窗口属性的SET_ATTRIBUTE过程。
  • 创建新的维护窗口
    要创建新的维护窗口，必须创建Oracle Scheduler窗口对象，然后将其添加到窗口组MAINTENANCE_WINDOW_GROUP 。
  • 删除维护窗口
    要删除现有维护窗口，请将其从MAINTENANCE_WINDOW_GROUP窗口组中删除。

父主题： 管理自动数据库维护任务

26.4.1修改维护窗口

DBMS_SCHEDULER PL / SQL包包含用于修改窗口属性的SET_ATTRIBUTE过程。

例如，以下脚本将维护窗口SATURDAY_WINDOW的持续时间更改为4小时：

BEGIN dbms_scheduler.disable（name =>'SATURDAY_WINDOW'）; dbms_scheduler.set_attribute（name =>'SATURDAY_WINDOW'，attribute =>'DURATION'，value => numtodsinterval（4，'hour'））; dbms_scheduler.enable（name =>'SATURDAY_WINDOW'）;结束; /

请注意，必须使用DBMS_SCHEDULER.DISABLE子程序在对窗口进行更改之前禁用该窗口，然后在完成后使用DBMS_SCHEDULER.ENABLE重新启用该窗口。如果在当前打开时更改窗口，则更改将在下次打开窗口时生效。

26.4.2创建新的维护窗口

要创建新的维护窗口，必须创建Oracle Scheduler窗口对象，然后将其添加到窗口组MAINTENANCE_WINDOW_GROUP 。

您使用DBMS_SCHEDULER 。 CREATE_WINDOW包过程创建窗口，以及DBMS_SCHEDULER 。 ADD_GROUP_MEMBER过程将新窗口添加到窗口组。

以下示例创建名为EARLY_MORNING_WINDOW的维护窗口。这个窗口每天早上5点到早上6点运行一小时

BEGIN DBMS_SCHEDULER.CREATE_WINDOW（window_name =>'EARLY_MORNING_WINDOW'，持续时间=> NUMTODSINTERVAL（1，'小时'），resource_plan =>'DEFAULT_MAINTENANCE_PLAN'，repeat_interval =>'FREQ = DAILY; BYHOUR = 5; BYMINUTE = 0; BYSECOND = 0 “）; DBMS_SCHEDULER.ADD_GROUP_MEMBER（group_name =>'MAINTENANCE_WINDOW_GROUP'，member =>'EARLY_MORNING_WINDOW'）;结束; /

26.4.3删除维护窗口

要删除现有维护窗口，请将其从MAINTENANCE_WINDOW_GROUP窗口组中删除。

窗口继续存在，但不再运行自动维护任务。分配给此窗口的任何其他Oracle Scheduler作业将继续照常运行。

以下示例从窗口组中删除EARLY_MORNING_WINDOW ：

BEGIN DBMS_SCHEDULER.REMOVE_GROUP_MEMBER（group_name =>'MAINTENANCE_WINDOW_GROUP'，member =>'EARLY_MORNING_WINDOW'）;结束; /

26.5配置自动维护任务的资源分配

    要在维护窗口中更改自动维护任务的资源分配，必须更改分配给该窗口的资源计划中的子计划ORA$AUTOTASK的资源百分比。

26.5.1关于自动维护任务的资源分配

默认情况下，所有预定义的维护窗口都使用资源计划DEFAULT_MAINTENANCE_PLAN 。自动维护任务在其子计划ORA$AUTOTASK下运行。该子计划在维护任务中平均分配其总资源分配部分。

DEFAULT_MAINTENANCE_PLAN定义以下资源分配：

消费者群体/子计划 1级 最大利用率限制

ORA $ AUTOTASK

5％

90

OTHER_GROUPS

20％

-

SYS_GROUP

75％

-

在此计划中， SYS_GROUP使用者组中的任何会话都将获得优先权。（此组中的会话是由用户帐户SYS和SYSTEM创建的会话。）然后， SYS_GROUP的会话未使用的任何资源分配将由属于计划中其他使用者组和子计划的会话共享。在该分配中，5％用于维护任务，20％用于用户会话。ORA$AUTOTASK的最大利用率限制为90。因此，即使CPU处于空闲状态，该组/计划也不能分配超过90％的CPU资源。

要减少或增加对自动维护任务的资源分配，请对DEFAULT_MAINTENANCE_PLAN进行调整。有关详细信息，请参阅“ 更改自动维护任务的资源分配 ” 。

请注意，与任何资源计划一样，消费者组或子计划未使用的分配部分可用于其他使用者组或子计划。另请注意，在使用100％的CPU之前，数据库资源管理器不会根据资源计划开始限制资源分配。

注意：

虽然DEFAULT_MAINTENANCE_PLAN是默认值，但您可以将任何资源计划分配给任何维护窗口。如果确实更改了维护ORA$AUTOTASK资源计划，请确保在新计划中包含子计划ORA$AUTOTASK 。

26.5.2更改自动维护任务的资源分配

要在维护窗口中更改自动维护任务的资源分配，必须更改分配给该窗口的资源计划中的子计划ORA$AUTOTASK的资源百分比。

（默认情况下，每个预定义维护窗口的资源计划为DEFAULT_MAINTENANCE_PLAN 。）您还必须在窗口的资源计划中调整一个或多个其他子计划或使用者组的资源分配，以使计划顶级的资源分配总计达到100％。有关更改资源分配的信息，请参阅使用Oracle数据库资源管理器管理资源 。
