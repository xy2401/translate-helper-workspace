舰队修补和配置的身份验证

要使用Fleet Patching和Provisioning REST API，请执行以下步骤：

  1. 注册REST用户

    要注册与CRS用户关联的REST用户，请运行以下命令：

    rhpctl register user -user <restUserName> -restuser -email <useremail> -rhpuser <crsUser>

    响应上述命令，系统将提示您输入密码。

    注意：

    CRS用户是您用于设置Fleet Patching and Provisioning或具有类似角色和权限的任何用户的用户。
    哪里：

    • <restUserName>选项是用于对发送到Fleet Patching and Provisioning Server的每个REST请求进行身份验证的名称。

    • <useremail>选项是启用通知时将发送通知的电子邮件地址。

    • <crsUser>选项是具有类似角色和权限的用户的名称，用于执行Fleet Patching和Provisioning操作。

    例：

    要使用与CRS用户关联的电子邮件ID rest@email.com注册名为rest的REST用户，请以CRS用户身份登录到Fleet Patching and Provisioning服务器，然后运行以下命令：

    $ $ ORACLE_HOME / bin / rhpctl注册用户-user rest -restuser -email rest@email.com -rhpuser $ USER

  2. 在Fleet Patching and Provisioning服务器上启用HTTPS。 ：

    对于Fleet Patching和Provisioning REST API，证书直接从Fleet Patching and Provisioning服务器获取。默认情况下，在Fleet Patching and Provisioning服务器上启用HTTPS。

    如果未在Fleet Patching and Provisioning服务器上启用HTTPS，请登录Fleet Patching and Provisioning服务器并执行以下步骤：

    1. 停止Fleet Patching和Provisioning服务器。

      $ $ ORACLE_HOME / bin / srvctl停止rhpserver

    2. 在Fleet Patching and Provisioning服务器上启用HTTPS。

      $ $ ORACLE_HOME / bin / srvctl修改rhpserver -enableHTTPS YES

    3. 启动Fleet Patching和Provisioning服务器。

      $ $ ORACLE_HOME / bin / srvctl启动rhpserver

  3. 如果启用了HTTPS，则导出Fleet Patching和Provisioning REST API证书。

    如果以CRS User或具有类似角色或权限的任何用户身份登录到Fleet Patching and Provisioning服务器，则可以执行此任务。

    1. 将目录更改为用户具有导出证书的写入权限的路径。

      $ cd~

    2. 运行以下命令：

      $ $ ORACLE_HOME / bin / crskeytoolctl -printrootcert

      群集根公共证书将打印到文件： <ClusterGUID.pem> 。

      注意：

      每次启动Fleet Patching and Provisioning服务器时都会创建证书。每次启动Fleet Patching and Provisioning服务器时，所有客户端都必须更新其证书。

  4. 验证Fleet Patching和Provisioning REST API的工作情况。

    要验证Fleet Patching和Provisioning REST API的工作情况，您将需要将在所有REST请求中使用的Fleet Patching和Provisioning服务器主机名，以及注册用户的用户和密码（如步骤2中所述 ）。

    1. 使用以下命令将环境变量CURL_CA_BUNDLE设置为导出的pem证书的绝对路径：

      $ export CURL_CA_BUNDLE = / home / crsUser / <clusterGUID> .pem

      注意：

      要删除环境变量，请运行以下命令：

      $ unset CURL_CA_BUNDLE

    2. 如果要查找运行Fleet Patching和Provisioning Server的节点的名称，请运行以下命令：

      $ $ ORACLE_HOME / bin / srvctl status rhpserver

      作为响应，您将获得节点的名称以及Fleet Patching和Provisioning服务器的状态。

    3. 使用以下cURL命令查询工作副本。这将提示您输入用户密码：

      curl -u <restUser> http：// <RHPServerHostName>：8894 / rhp-restapi / rhp / workingcopies

      例：

      要获取工作副本的名称，请登录Fleet Patching and Provisioning服务器并运行以下命令：

      $ curl -u rest https：// $ HOSTNAME：8894 / rhp-restapi / rhp / workingcopies

      这将提示您输入密码：

      输入用户“rest”的主机密码：

      输入密码，作为响应，您将获得Fleet Patching and Provisioning服务器中可用的工作副本的名称。如果没有工作副本，那么您将获得一个空项目数组作为响应：

      {"items":[ ]} 。
