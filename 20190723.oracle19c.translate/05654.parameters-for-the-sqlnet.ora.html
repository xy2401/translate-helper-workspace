5.1配置文件配置文件概述

sqlnet.ora文件是配置文件配置文件。它驻留在客户端计算机和数据库服务器上。使用此文件存储和实现配置文件。可以使用sqlnet.ora文件中的访问控制参数配置数据库服务器。这些参数指定是否允许客户端基于协议进行访问。

sqlnet.ora文件使您可以执行以下操作：

  • 指定要附加到非限定名称的客户端域

  • 优先考虑命名方法

  • 启用日志记录和跟踪功能

  • 通过特定进程路由连接

  • 配置外部命名的参数

  • 配置Oracle Advanced Security

  • 使用特定于协议的参数来限制对数据库的访问

默认情况下， sqlnet.ora文件位于ORACLE_HOME/network/admin目录中。sqlnet.ora文件也可以存储在TNS_ADMIN环境变量指定的目录中。

注意：

  • sqlnet.ora文件中的设置适用于多租户容器数据库环境中的所有可插拔数据库（PDB）。

  • Oracle Net Services支持sqlnet.ora文件中的IFILE参数，最多可嵌套三层。该参数手动添加到文件中。以下是语法示例：

    IFILE = / tmp / listener_em.ora IFILE = / tmp / listener_cust1.ora IFILE = / tmp / listener_cust2.ora

    有关其他信息，请参阅Oracle数据库参考 。

  • 在只读Oracle主模式下， sqlnet.ora文件的默认位置是ORACLE_BASE_HOME /network/admin 。

  • 在只读Oracle主模式下，默认为ORACLE_HOME位置的参数更改为默认值 ORACLE_BASE_HOME位置。

5.2.15 NAMES.LDAP_PERSISTENT_SESSION

目的

指定LDAP命名适配器是否应在名称查找完成后保持LDAP服务器的会话处于打开状态。

如果Oracle钱包存储在Microsoft Windows注册表中且钱包的密钥（ KEY)为SALESAPP ，则加密钱包的存储位置为HKEY_CURRENT_USER\SOFTWARE\ORACLE\WALLETS\SALESAPP\EWALLET.P12 。解密钱包的存储位置是HKEY_CURRENT_USER\SOFTWARE\ORACLE\WALLETS\SALESAPP\CWALLET.SSO.

句法

5.4.13 TRACE_FILENO_CLIENT

目的

指定客户端跟踪的跟踪文件数。

使用说明

使用TRACE_FILELEN_CLIENT参数设置此参数时， 将以循环方式使用跟踪文件。首先填充第一个文件，然后填充第二个文件，依此类推。填写完最后一个文件后，将重新使用第一个文件，依此类推。

使用TRACE_FILEAGE_CLIENT参数设置此参数时，将根据其年龄循环跟踪文件。使用第一个文件直到达到年龄限制，然后使用第二个文件，依此类推。达到最后一个文件的期限时，将重复使用第一个文件，依此类推。

如果使用TRACE_FILELEN_CLIENT和TRACE_FILEAGE_CLIENT参数设置此参数，则在达到大小限制或年龄限制时，将循环跟踪文件。

跟踪文件名通过序列号彼此区分。例如，如果使用sqlnet.trc的缺省跟踪文件，并且此参数设置为3，则跟踪文件将命名为sqlnet1.trc ， sqlnet2.trc和sqlnet3.trc 。

此外，跟踪文件中的跟踪事件前面是文件的序列号。未启用ADR时使用此参数。

默认

没有

例

TRACE_FILENO_CLIENT = 3

5.4.15 TRACE_UNIQUE_CLIENT

目的

指定是否为每个客户端跟踪会话创建唯一跟踪文件。

使用说明

当该值设置为on ，进程标识符将附加到每个跟踪文件的名称，从而使多个文件共存。例如，跟踪文件名为sqlnet pid .trc如果默认的跟踪文件名创建sqlnet.trc使用。当该值设置为off ，来自新客户端跟踪会话的数据将覆盖现有文件。未启用ADR时使用此参数。

默认

上

值

on或off

例

TRACE_UNIQUE_CLIENT =上
