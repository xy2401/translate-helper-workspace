NLSSORT

句法


说明nlssort.eps的描述

目的

NLSSORT返回字符值char的排序规则键以及显式或隐式指定的排序规则。排序规则键是一个字节字符串，用于根据指定的排序规则对char进行排序。核对键的属性是当根据它们的二进制顺序进行比较时为给定的核对生成的两个这样的键的相互排序与根据给定的核对进行比较时的源字符值的相互排序相同。

char和' nlsparam '都可以是CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2任何数据类型。

' nlsparam '的值必须具有该形式

'NLS_SORT = 整理 '

collation是语言排序规则或BINARY的名称。 NLSSORT使用指定的排序规则生成排序规则键。如果省略' nlsparam '，则此函数使用参数char的派生归类。如果指定BINARY ，则此函数返回char值本身强制转换为RAW并可能截断，如下所述。

如果指定' nlsparam '，则可以在语言排序规则名称后附加后缀_ai以请求不区分重音的排序规则，或者_ci请求不区分大小写的排序规则。有关重音和不区分大小写排序的详细信息，请参阅“ Oracle数据库全球化支持指南” 。不建议对ORDER BY查询子句使用不区分重音或不区分大小写的排序规则，因为它会导致不确定的排序顺序。

返回的归类键是RAW数据类型。由给定排序规则的给定char值产生的排序规则键的长度可能超过NLSSORT返回的RAW值的最大长度。在这种情况下， NLSSORT的行为取决于初始化参数MAX_STRING_SIZE的值。如果MAX_STRING_SIZE = EXTENDED ，则返回值的最大长度为32767字节。如果排序规则键超出此限制，则该功能将失败，并显示错误“ORA-12742：无法创建排序规则键”。如果短输入字符串包含高百分比的Unicode字符且分解率非常高，则也可能会报告此错误。

如果MAX_STRING_SIZE = STANDARD ，则返回值的最大长度为2000字节。如果要返回的值超出限制，则NLSSORT计算char的最大前缀或初始子字符串的排序规则键，以便计算结果不超过最大长度。对于单语校对，例如FRENCH ，前缀长度通常为1000个字符。对于多语言归类，例如GENERIC_M ，前缀通常为500个字符。对于Unicode归类算法（UCA）归类，例如UCA0610_DUCET ，前缀通常为285个字符。确切的长度可以更低或更高，具体取决于整理和char包含的char 。

MAX_STRING_SIZE = STANDARD时的行为意味着如果两个字符值的校对键（ NLSSORT结果）被比较以找到语言顺序，则它们被认为是相等的，即使它们在前缀上没有差异，即使它们在某些其他字符位置可能不同。由于隐式使用NLSSORT函数来查找比较条件的语言排序， BETWEEN条件， IN条件， ORDER BY ， GROUP BY和COUNT(DISTINCT) ，这些操作可能返回仅对长字符值近似的结果。如果要保证这些操作的结果是准确的，则迁移数据库以使用MAX_STRING_SIZE = EXTENDED 。

有关MAX_STRING_SIZE初始化参数的更多信息，请参阅“ 扩展数据类型 ” 。

此函数不直接支持CLOB数据。但是， CLOB可以通过隐式数据转换作为参数传递。

例子

此函数可用于根据语言排序顺序而不是字符串的二进制值指定排序和比较操作。以下示例创建一个包含两个值的测试表，并显示NLSSORT函数如何排序返回的值：

CREATE TABLE test（名称VARCHAR2（15））;插入测试值（'Gaardiner'）;插入测试值（'Gaberd'）;插入测试值（'Gaasten'）; SELECT * FROM测试ORDER BY名称; NAME --------------- Gaardiner Gaasten Gaberd SELECT * FROM test ORDER BY NLSSORT（name，'NLS_SORT = XDanish'）;姓名--------------- Gaberd Gaardiner Gaasten

以下示例显示如何在比较操作中使用NLSSORT函数：

SELECT * FROM test WHERE name>'Gaberd'ORDER BY name;没有选择行SELECT * FROM test WHERE NLSSORT（name，'NLS_SORT = XDanish'）> NLSSORT（'Gaberd'，'NLS_SORT = XDanish'）ORDER BY name;姓名--------------- Gaardiner Gaasten

如果您经常在具有相同语言排序顺序的比较操作中使用NLSSORT ，那么请考虑这种更有效的替代方法：将NLS_COMP参数（对于数据库或当前会话）设置为LINGUISTIC ，并将会话的NLS_SORT参数设置为所需的排序顺序。默认情况下，Oracle数据库将对当前会话期间的所有排序和比较操作使用该排序顺序：

ALTER SESSION SET NLS_COMP ='LINGUISTIC'; ALTER SESSION SET NLS_SORT ='XDanish'; SELECT * FROM test WHERE name>'Gaberd'ORDER BY name;姓名--------------- Gaardiner Gaasten
