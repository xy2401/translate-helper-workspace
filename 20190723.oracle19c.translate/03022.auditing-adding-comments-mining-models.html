34.5审核并向挖掘模型添加注释

挖掘模型对象支持SQL COMMENT和AUDIT语句。

34.5.1向挖掘模型添加注释

注释可用于将描述性信息与数据库对象相关联。您可以使用a将注释与挖掘模型相关联 SQL COMMENT语句。 

对MINING模型的评论schema_name.model_name IS string ;

注意：

要向另一个模式中的模型添加注释，您必须具有COMMENT ANY MINING MODEL系统特权。

要删除注释，请将其设置为空的''字符串。

以下语句在您自己的模式中向模型DT_SH_CLAS_SAMPLE添加注释。 

关于采矿模型的评论dt_sh_clas_sample IS'决策树模型预测促销响应';

您可以通过查询目录视图USER_MINING_MODELS来查看USER_MINING_MODELS 。

SELECT model_name，mining_function，algorithm，comments FROM user_mining_models; MODEL_NAME MINING_FUNCTION ALGORITHM评论----------------- ---------------- ------------- -  ----------------------------------------------- DT_SH_CLAS_SAMPLE CLASSIFICATION DECISION_TREE决策树模型预测促销响应

要从数据库中删除此注释，请发出以下语句：

关于采矿模型的评论dt_sh_clas_sample'';

34.5.2审计挖掘模型

Oracle数据库审计系统是一种功能强大，高度可配置的工具，用于跟踪生产环境中架构对象的操作。审计系统可用于跟踪数据挖掘模型的操作。

注意：

要审核挖掘模型，您必须具有AUDIT_ADMIN角色。

“ Oracle数据库安全指南”中介绍了统一审核。但是，默认情况下不启用完整的统一审核系统。“ Oracle数据库升级指南”中提供了有关迁移到统一审核的说明。
