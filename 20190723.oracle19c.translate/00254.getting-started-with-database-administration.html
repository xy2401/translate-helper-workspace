1.2.3任务3：规划数据库

作为数据库管理员，您必须规划数据库的逻辑存储结构，整体数据库设计以及数据库的备份策略。

规划数据库的逻辑存储结构将如何影响系统性能和各种数据库管理操作非常重要。例如，在为数据库创建任何表空间之前，您应该知道表空间将包含多少数据文件，每个表空间中将存储哪种类型的信息，以及将物理存储数据文件的磁盘驱动器。在规划数据库结构的整体逻辑存储时，请考虑实际创建和运行数据库时此结构将产生的影响。考虑数据库的逻辑存储结构将如何影响：

  • 运行Oracle数据库的计算机的性能

  • 数据访问操作期间数据库的性能

  • 数据库的备份和恢复过程的效率

规划数据库对象的关系设计以及每个对象的存储特征。通过在创建每个对象与其物理存储之间规划关系，可以直接影响数据库作为一个单元的性能。一定要计划数据库的增长。

在分布式数据库环境中，此规划阶段非常重要。频繁访问数据的物理位置会显着影响应用程序性能。

在规划阶段，为数据库开发备份策略。您可以更改数据库的逻辑存储结构或设计，以提高备份效率。

讨论关系数据库和分布式数据库设计超出了本书的范围。如果您不熟悉此类设计问题，请参阅公认的行业标准文档。

Oracle数据库结构和存储以及架构对象提供有关为数据库创建逻辑存储结构，对象和完整性约束的特定信息。

1.3.3.3第2步：设置操作系统环境变量

1.3.3.5.1 SQL * Plus CONNECT命令的语法

使用SQL * Plus CONNECT命令初始连接到Oracle实例或重新连接到Oracle实例。

句法

CONN [ECT] [ 登录 ] [AS {SYSOPER | SYSDBA | SYSBACKUP | SYSDG | SYSKM | SYSRAC}]

logon语法如下：

{ username | /} [ @connect_identifier ] [edition = { edition_name | DATABASE_DEFAULT}]

当您提供username ，SQL * Plus会提示输入密码。键入密码时不会回显密码。

下表描述了CONNECT命令的语法组件。

语法组件 描述

/

调用连接请求的外部身份验证。在此类型的身份验证中不使用数据库密码。最常见的外部身份验证形式是操作系统身份验证，其中数据库用户通过使用特定主机用户帐户登录到主机操作系统进行身份验证。也可以使用Oracle钱包或网络服务执行外部身份验证。有关更多信息，请参见Oracle数据库安全指南另请参阅“ 使用操作系统身份验证 ” 。

AS {SYSOPER | SYSDBA | SYSBACKUP | SYSDG | SYSKM | SYSRAC}

表示数据库用户正在使用管理权限进行连接。只有某些预定义的管理用户或已添加到密码文件的用户可以使用这些权限进行连接。有关详细信息，请参阅“ 管理权限 ” 。

用户名

有效的数据库用户名。数据库通过将username与数据字典进行匹配并提示输入用户密码来验证连接请求。

connect_identifier （1）

Oracle Net连接标识符，用于远程连接。确切的语法取决于Oracle Net配置。如果省略，SQL * Plus将尝试连接到本地实例。

公共连接标识符是网络服务名称 。这是Oracle Net连接描述符（网络地址和数据库服务名称）的别名。别名通常在本地计算机上的tnsnames.ora文件中解析，但可以通过其他方式解析。

有关连接标识符的更多信息，请参见“ Oracle数据库网络服务管理员指南” 。

connect_identifier （2）

作为替代方案，连接标识符可以使用简单的连接语法。轻松连接为远程数据库提供开箱即用的TCP / IP连接，而无需在客户端（本地）计算机上配置Oracle Net Services。

连接标识符的简单连接语法如下（必须包含封闭的双引号）：

“ host [： port ] [/ service_name ] [： server ] [/ instance_name ]”

哪里：

  • host是托管远程数据库的计算机的主机名或IP地址。

    支持IP版本4（IPv4）和IP版本6（IPv6）地址。IPv6地址必须用方括号括起来。有关IPv6寻址的信息，请参见“ Oracle数据库网络服务管理员指南” 。

  • port是host上的Oracle Net侦听器侦听数据库连接的TCP端口。如果省略，则假定为1521。

  • service_name是要连接的数据库服务名称。如果远程主机上的Net Services侦听器配置指定默认服务，则可以省略它。如果未配置默认服务，则必须提供service_name 。每个数据库通常都提供一个标准服务，其名称等于全局数据库名称，该名称由DB_NAME和DB_DOMAIN初始化参数组成，如下所示：

    DB_NAME.DB_DOMAIN

    如果DB_DOMAIN为null，则标准服务名称就是DB_NAME 。例如，如果DB_NAME是orcl且DB_DOMAIN是us.example.com ，则标准服务名称是orcl.us.example.com 。

    有关详细信息，请参阅“ 使用数据库服务管理应用程序工作负载 ” 。

  • server是服务处理程序的类型。可接受的值dedicated ， shared ，和pooled 。如果省略，则侦听器选择默认类型的服务器：如果已配置，则为共享服务器，否则为专用服务器。

  • instance_name是要连接的实例。您可以指定服务名称和实例名称，这通常仅适用于Oracle Real Application Clusters（Oracle RAC）环境。对于Oracle RAC或单实例环境，如果仅指定实例名称，则连接到默认数据库服务。如果listener.ora文件中没有配置默认服务，则会生成错误。您可以从INSTANCE_NAME初始化参数中获取实例名称。

有关轻松连接的详细信息，请参阅“ Oracle数据库网络服务管理员指南” 。

edition = { edition_name | DATABASE_DEFAULT }

指定新数据库会话启动的版本。如果指定版本，则它必须存在，并且您必须具有USE特权。如果未指定此子句，则会话将使用数据库缺省版本。

1.5.2.4 SYSBACKUP，SYSDG，SYSKM和SYSRAC

创建Oracle数据库时，将自动创建以下用户以便于分离数据库管理员的职责： SYSBACKUP ， SYSDG ， SYSKM和SYSRAC 。

这些用户通过以下方式分离职责：

  • SYSBACKUP可以从RMAN或SQL * Plus中促进Oracle Recovery Manager（RMAN）备份和恢复操作。

  • SYSDG有助于Data Guard操作。用户可以使用Data Guard Broker或DGMGRL命令行界面执行操作。

  • SYSKM促进透明数据加密密钥库操作。

  • SYSRAC通过Clusterware代理代表Oracle RAC实用程序（如SRVCTL）连接到数据库， SYSRAC促进Oracle Real Application Clusters（Oracle RAC）操作。

    无法向数据库用户授予SYSRAC管理权限，密码文件不支持该权限。SYSRAC管理权限仅由Oracle Clusterware的Oracle代理程序使用，以使用操作系统身份验证连接到数据库。

这些帐户中的每一个都为具有相同名称的新管理权限提供指定用户。具体来说， SYSBACKUP帐户为SYSBACKUP管理权限提供指定用户。SYSDG帐户为SYSDG管理权限提供指定用户。SYSKM帐户为SYSKM管理权限提供指定用户。

创建用户并授予该用户执行日常管理任务时要使用的适当管理权限。这样，您就可以单独管理每个用户帐户，并且每个用户帐户都可以拥有不同的密码。不要将SYSBACKUP ， SYSDG或SYSKM用户帐户用于这些目的。

要使用这些管理权限之一，用户必须通过指定权限连接到数据库时行使权限，例如AS SYSBACKUP ， AS SYSDG或AS SYSKM 。如果身份验证成功，则用户将连接到具有启用管理权限的会话的数据库。在这种情况下，会话用户是相应的管理用户帐户。例如，如果用户bradmin与AS SYSBACKUP管理权限连接，则会话用户为SYSBACKUP 。

注意：

  • 无法删除这些用户帐户。
  • 这些用户帐户是仅模式帐户，也就是说，它们是在没有密码的情况下创建的。您可以在需要对这些用户帐户进行身份验证时为其分配密码。

orapwd DESCRIBE FILE = 文件名

1.7.3共享和禁用数据库密码文件

使用初始化参数REMOTE_LOGIN_PASSWORDFILE来控制是否在多个Oracle数据库实例之间共享数据库密码文件。您还可以使用此参数禁用密码文件身份验证。

要共享密码文件或禁用密码文件身份验证：

  • 设置REMOTE_LOGIN_PASSWORDFILE初始化参数。

您可以将REMOTE_LOGIN_PASSWORDFILE初始化参数设置为以下值之一：

  • none ：将此参数设置为none会导致Oracle数据库的行为就像密码文件不存在一样。也就是说，不允许通过非安全连接进行特权连接。

  • exclusive :(默认值） exclusive密码文件只能与一个数据库一起使用。只能修改exclusive文件。使用exclusive密码文件可以添加，修改和删除用户。它还允许您使用ALTER USER命令更改SYS ， SYSBACKUP ， SYSDG或SYSKM的密码。

    当exclusive密码文件存储在Oracle ASM磁盘组上时，它可以由单实例数据库或Oracle Real Application Clusters（Oracle RAC）数据库的多个实例使用。

    当exclusive密码文件存储在操作系统上时，它只能与一个数据库的一个实例一起使用。

  • shared ： shared密码文件可以由运行在同一服务器上的多个数据库使用，也可以由Oracle RAC数据库的多个实例使用，即使它存储在操作系统上也是如此。shared密码文件是只读的，无法修改。因此，您无法将用户添加到shared密码文件。任何尝试这样做或使用管理权限更改SYS或其他用户的密码都会产生错误。当REMOTE_LOGIN_PASSWORDFILE设置为exclusive时，必须将所有需要管理权限的用户添加到密码文件中。添加完所有用户后，可以将REMOTE_LOGIN_PASSWORDFILE更改为shared ，然后共享该文件。

    如果使用单个密码文件管理多个数据库，则此选项很有用。

    您无法为Oracle ASM密码文件指定shared 。

如果REMOTE_LOGIN_PASSWORDFILE设置为exclusive或shared且密码文件丢失，则相当于将REMOTE_LOGIN_PASSWORDFILE设置为none 。

1.7.4保持管理员密码与数据字典同步

如果将REMOTE_LOGIN_PASSWORDFILE初始化参数从none更改为exclusive或shared ，则必须确保存储在数据字典中的密码和存储在非SYS管理用户的密码文件中的密码，例如SYSDBA ， SYSOPER ， SYSBACKUP ， SYSDG和SYSKM用户是一样的。

注意：

从Oracle Database 12 c第2版（12.2）开始， SYS用户的身份验证仅使用密码文件而不使用数据字典。

要同步非SYS管理用户（例如SYSDBA ， SYSOPER ， SYSBACKUP ， SYSDG和SYSKM用户）的SYSKM ，必须首先撤消然后向这些用户重新授予权限，如下所示：

  1. 查找已被授予SYSDBA权限的所有用户。
    SELECT USERNAME FROM V $ PWFILE_USERS WHERE USERNAME！='SYS'和SYSDBA ='TRUE';
  2. 撤消然后将SYSDBA权限重新授予这些用户。
    从非SYS用户撤消SYSDBA;向非SYS用户授予SYSDBA;
  3. 查找已被授予SYSOPER权限的所有用户。
    SELECT USERNAME FROM V $ PWFILE_USERS WHERE USERNAME！='SYS'和SYSOPER ='TRUE';
  4. 撤消并向这些用户重新授予SYSOPER权限。
    从非SYS用户撤消SYSOPER;向非SYS用户授予SYSOPER;
  5. 查找已被授予SYSBACKUP权限的所有用户。
    SELECT USERNAME FROM V $ PWFILE_USERS WHERE USERNAME！='SYS'和SYSBACKUP ='TRUE';
  6. 撤消并向这些用户SYSBACKUP授予SYSBACKUP权限。
    从非SYS用户撤消SYSBACKUP; GRANT SYSBACKUP到非SYS用户 ;
  7. 查找已被授予SYSDG权限的所有用户。
    SELECT USERNAME FROM V $ PWFILE_USERS WHERE USERNAME！='SYS'和SYSDG ='TRUE';
  8. 撤消并向这些用户SYSDG授予SYSDG权限。
    从非SYS用户撤消SYSDG;向非SYS用户授予SYSDG;
  9. 查找已被授予SYSKM权限的所有用户。
    SELECT USERNAME FROM V $ PWFILE_USERS WHERE USERNAME！='SYS'和SYSKM ='TRUE';
  10. 撤消并向这些用户SYSKM授予SYSKM权限。
    从非SYS用户撤消SYSKM; GRANT SYSKM给非SYS用户 ;

1.7.5将用户添加到数据库密码文件

将SYSDBA ， SYSOPER ， SYSBACKUP ， SYSDG或SYSKM管理权限授予用户时，该用户的名称和权限信息将添加到数据库密码文件中。

只要该用户至少具有这些权限中的一个，用户的名称就会保留在密码文件中。如果撤消所有这些权限，则Oracle数据库会从密码文件中删除该用户。

    注意：

    REMOTE_LOGIN_PASSWORDFILE是静态初始化参数，因此无法在不重新启动数据库的情况下进行更改。

1.7.6授予和撤销行政特权

使用GRANT语句授予管理权限。使用REVOKE语句撤消管理权限。

要将SYSDBA ， SYSOPER ， SYSBACKUP ， SYSDG或SYSKM管理权限授予用户：

  • 运行GRANT语句。

例如：

授予SYSDBA给mydba;

要撤消用户的管理权限：

  • 运行REVOKE语句。

例如：

从mydba撤消SYSDBA;

如果在授予管理权限的GRANT语句中指定WITH ADMIN OPTION则忽略WITH ADMIN OPTION ，并应用以下规则：

  • 当前以SYSDBA身份连接的SYSDBA可以向其他用户授予任何管理权限，并从其他用户撤消任何管理权限。

  • 当前以SYSOPER身份连接的用户无法向其他用户授予任何管理权限， 也无法撤消其他用户的任何管理权限。

  • 当前以SYSBACKUP连接的用户可以授予或撤消其他用户的SYSBACKUP管理权限。

  • 当前以SYSDG连接的用户可以授予或撤消其他用户的SYSDG管理权限。

  • 当前以SYSKM连接的用户可以授予或撤消其他用户的SYSKM管理权限。

无法为角色授予管理权限，因为角色仅在数据库启动后可用。不要将数据库管理权限与操作系统角色混淆。
