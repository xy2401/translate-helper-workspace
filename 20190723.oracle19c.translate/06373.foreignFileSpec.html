4.10 foreignFileSpec

目的

使用foreignFileSpec子句提供执行跨平台还原操作所需的信息。此信息包括包含跨平台备份的备份集的名称以及必须还原的对象或文件的详细信息。

使用说明

指定foreignFileSpec是执行跨平台还原操作所必需的。即使源平台和目标平台使用不同的endian格式，也可以恢复表空间。

语法元素 描述

ALL FOREIGN DATAFILES

指定必须还原跨平台备份集中包含的所有数据文件。使用FROM BACKUPSET和ALL FOREIGN DATAFILES指定包含要还原的数据文件的备份集的名称。

FORMAT formatSpec

指定目标数据库中已还原数据文件的模式。

新的

指定必须使用Oracle Managed File（OMF）名称将数据文件还原到DB_FILE_CREATE_DEST初始化参数指定的位置。

DUMP FILE 'filename ”

指定必须还原包含数据泵导出转储文件的备份集。使用FROM BACKUPSET子句指定包含转储文件的备份集的名称。转储文件包含跨平台传输的表空间的元数据。您需要此元数据才能将已还原的表空间插入目标数据库。

filename表示还原导出转储文件的文件的名称。

DATAPUMP DESTINATION ' datapump_destination '

指定必须将还原导出转储文件的位置。如果省略此子句，则导出转储文件将还原到与操作系统相关的默认位置。

使用此子句时，必须以读写模式打开数据库，否则会发生错误。

NOIMPORT

指定必须还原导出转储文件，但不将其导入目标数据库。

由于未导入导出转储文件，因此表空间未插入目标数据库。您需要使用数据泵导入手动将转储文件导入目标数据库以插入表空间。

外国数据库

指定必须从跨平台备份集还原整个数据库。使用FROM BACKUPSET指定包含了跨平台的数据库备份的备份集。

使用FOREIGN DATABASE ，不能装入数据库，否则会发生错误。

FOREIGN PLUGGABLE DATABASE pdb_name

指定必须从跨平台备份集还原由pdb_name指定的可插入数据库（PDB）。使用FROM BACKUPSET指定包含跨平台PDB备份的备份集。

不得安装CDB，否则会发生错误。

FOREIGN TABLESPACE tablespace_name

指定必须从跨平台备份集还原的表空间。表空间的名称必须是源数据库中的原始表空间名称。使用FROM BACKUPSET指定包含跨平台备份的备份集的名称。

语法元素 描述

BACKUPSET filename

指定包含必须还原的跨平台备份的备份集。

大多数备份集只包含一个备份。但是，您可以通过配置MAXPIECESIZE来指定每个备份片的最大大小，从而创建包含多个备份片的备份。当备份集包含多个备份FROM BACKUPSET ， FROM BACKUPSET子句必须包含备份集中所有备份片的逗号分隔列表。备件的顺序必须是从第一件到最后一件。如果未遵循此顺序，则还原操作将失败。有关包含多个备份片段的RESTORE命令，请参见例3-38 。

如果跨平台备份包含多个备份集，则必须对作为跨平台备份的一部分创建的每个备份集使用单独的BACKUPSET子句。有关包含多个备份集的RESTORE命令，请参见示例3-37 。
