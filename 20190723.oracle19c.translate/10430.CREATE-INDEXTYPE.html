CREATE INDEXTYPE

目的

使用CREATE INDEXTYPE语句创建indextype ，该类型是指定管理域（特定于应用程序）索引的例程的对象。Indextypes与表，视图和其他模式对象驻留在同一名称空间中。此语句将indextype名称绑定到实现类型，实现类型又指定并引用用户定义的索引函数和实现indextype的过程。

有关实现索引类型的更多信息，请参见Oracle数据库数据盒开发人员指南

先决条件

要在自己的模式中创建索引类型，您必须具有CREATE INDEXTYPE系统特权。要在另一个模式中创建索引类型，您必须具有CREATE ANY INDEXTYPE系统特权。在任何一种情况下，您都必须具有实现类型和支持的运算符的EXECUTE对象特权。

indextype支持一个或多个运算符，因此在创建索引类型之前，必须首先设计要支持的运算符，并为这些运算符提供功能实现。

语义

模式

指定indextype所在的模式的名称。如果省略schema ，则Oracle Database会在您自己的模式中创建indextype。

INDEXTYPE

指定要创建的索引类型的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

FOR Clause

使用FOR子句指定indextype支持的运算符列表。

  • 对于schema ，请指定包含运算符的模式。如果省略schema ，则Oracle假定运算符在您自己的模式中。

  • 对于operator ，请指定indextype支持的运算符的名称。

    本节中列出的所有运算符都必须是有效的运算符。

  • 对于parameter_type ，列出运算符的参数类型。

using_type_clause

USING子句允许您指定为新索引类型提供实现的类型。

对于implementation_type ，请指定实现相应Oracle数据盒式磁带接口（ODCI）的类型的名称。

  • 您必须指定实现ODCI中的例程的有效类型。

  • 实现类型必须与indextype位于同一模式中。

与本地分区

使用此子句指示indextype可用于在范围，列表，散列和间隔分区表上创建本地域索引。您可以通过多种方式将此子句与storage_table_clause结合使用（请参阅storage_table_clause ）。

  • 建议的方法是WITH SYSTEM MANAGED STORAGE TABLES指定WITH LOCAL PARTITION 。此组合使用系统管理的存储表（这是首选的存储管理），并允许您在范围，列表，散列和间隔分区表上创建本地域索引。在这种情况下， RANGE关键字是可选的并被忽略，因为如果指定WITH LOCAL PARTITION WITH SYSTEM MANAGED STORAGE TABLES则不再需要它。

  • 您可以指定WITH LOCAL RANGE PARTITION （包括RANGE关键字）并省略storage_table子句。用户管理的存储表支持范围分区表上的本地域索引，以实现向后兼容性。Oracle不建议使用此组合，因为它使用效率较低的用户管理存储表。

如果完全省略此子句，则不能随后使用此indextype在范围，列表，散列或间隔分区表上创建本地域索引。

storage_table_clause

使用此子句指定如何管理在此索引类型上构建的索引的存储表和分区维护操作：

  • 指定WITH SYSTEM MANAGED STORAGE TABLES以指示统计数据的存储将由系统管理。您在statistics_type指定的类型应该将统计信息相关信息存储在由系统维护的表中。此外，必须已创建或更改指定的索引类型以支持WITH SYSTEM MANAGED STORAGE TABLES子句。

  • 指定WITH USER MANAGED STORAGE TABLES以指示存储用户定义的统计信息的表将由用户管理。这是默认行为。

array_DML_clause

使用此子句让indextype支持ODCIIndexInsert方法的数组接口。

type 和 varray_type

如果要编制索引的列的数据类型是用户定义的对象类型，则必须指定此子句以标识Oracle应用于保存type列值的varray varray_type 。如果indextype支持类型列表，则可以指定相应的varray类型列表。如果省略type或varray_type schema ，则Oracle假定该类型位于您自己的模式中。

如果要索引的列的数据类型是内置系统类型，则为indextype指定的任何varray类型优先于系统定义的ODCI类型。
