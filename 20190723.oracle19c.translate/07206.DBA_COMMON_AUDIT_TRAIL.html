4.161 DBA_COMMON_AUDIT_TRAIL

DBA_COMMON_AUDIT_TRAIL显示所有标准和细粒度的审计跟踪条目，以XML格式编写的强制和SYS审计记录。

注意：

此视图仅在未启用统一审核的Oracle数据库中填充。在Oracle数据库中启用统一审计时，审计记录将填充在新审计跟踪中，并可从UNIFIED_AUDIT_TRAIL查看。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

柱 数据类型 空值 描述

AUDIT_TYPE

VARCHAR2（22）

审计跟踪类型：

  • 标准审计

  • 标准XML审计

  • 细粒度审计

  • 细粒度XML审计

  • SYS XML Audit

  • 强制XML审计

SESSION_ID

数

Oracle会话的数字ID

PROXY_SESSIONID

数

代理会话序列号，如果企业用户已通过代理机制登录

STATEMENTID

数

语句运行的数字ID;声明可能会导致多个审计记录

ENTRYID

数

会话中审计跟踪条目的数字标识

EXTENDED_TIMESTAMP

VARCHAR2（128）

应用执行上下文标识

VARCHAR2（4000）

用户外部名称

数

SCN

数

COMMENT_TEXT

VARCHAR2（4000）

审计跟踪条目的文本注释，提供有关审计的语句的更多信息

还指示用户如何进行身份验证：

  • DATABASE - 身份验证是通过密码完成的

  • NETWORK - 身份验证由Oracle Net Services或Advanced Networking Option完成

  • PROXY - 客户端已被其他用户验证。代理用户的名称遵循方法类型。

  • F - 失败

  • B - 两者
