17支持可插拔数据库

多租户架构使Oracle数据库能够包含一个可移植的模式，模式对象和非复制对象集合，这些对象在Oracle客户端中显示为单独的数据库。

多租户容器数据库（CDB）是包含一个或多个可插拔数据库（PDB）的Oracle数据库。

OCI客户端可以使用已将可插入数据库属性设置为相关PDB的服务连接到PDB。

通常，无论是连接到可插拔数据库还是普通数据库，OCI调用的行为都是相同的。需要特别考虑CDB的OCI呼叫和功能将在以下章节中介绍：

这些增强功能是Oracle Database 12 c第2版（12.2）及更高版本现在已删除的限制的结果。

以下增强功能是Oracle Database 12 c第2版（12.2）及更高版本现在删除的限制的结果。

  • 客户端结果缓存与可插拔数据库的连接一起使用。

17.2 OTER会话集合的 OCI增强功能

删除了有关使用ALTER SESSION SET CONTAINER语句的一些OCI限制。

客户端必须是Oracle Database Release 12 c第2版（12.2）才能执行这些操作。如果尝试以下任何操作，则返回较低版本的客户端错误。

  • ALTER SESSION SET CONTAINER语句支持在OCI中使用TIMESTAMP WITH TIMEZONE或TIMESTAMP WITH LOCAL TIMEZONE数据类型的应用程序。当应用程序在具有不同数据库时区设置或不同数据库时区文件版本设置的可插拔数据库之间切换时，将保留这些类型的语义。

  • ALTER SESSION SET CONTAINER语句支持使用此命令在任何两个字符集不同的可插拔数据库之间切换OCI连接。

17.3一般情况下使用多租户容器数据库（CDB）对OCI API调用的限制

描述一般限制。

  • 当连接到CDB$ROOT以外的任何容器时尝试登录OCI_PRELIM_AUTH模式将导致ORA-24542错误。

  • 尝试在连接到CDB$ROOT以外的任何容器时发出OCIDBStartup()导致ORA-24543错误。

  • 当连接到CDB$ROOT以外的任何容器时尝试发出OCIDBShutdown()导致ORA-24543错误。当OCIDBShutdown()被发布连接到CDB$ROOT ，它会关闭整个实例。

  • CDB不支持OCI连续查询通知（CQN）。

  • 与早于12.1或更高版本的客户端库链接并连接到可插拔数据库的OCI应用程序在作为普通（非常见）用户连接时将无法使用快速应用程序通知（FAN）高可用性（HA）功能。作为一种解决方法，此类应用程序应作为普通用户进行连接。对于12.1或更高版本的OCI客户端，此限制不存在。

17.4使用ALTER SESSION SET CONTAINER对OCI呼叫的限制

描述具体的限制。

ALTER SESSION SET CONTAINER语句可用于将OCI连接从一个可插拔数据库切换到另一个可插拔数据库。但是，使用ALTER SESSION SET CONTAINER语句在可插拔数据库之间切换的应用程序需要确保它们的使用与下面描述的OCI限制一致。

  • 对于OCI可迁移会话（例如在登录期间使用OCI_MIGRATE模式创建的会话），不允许使用ALTER SESSION SET CONTAINER语句，并且该组合会导致ORA-65135错误。

  • OCI连接池（旧的OCI连接池API）不支持ALTER SESSION SET CONTAINER语句，并且该组合导致ORA-65135错误。

  • 与OCI会话切换（其中多个OCI用户句柄共享相同的OCI服务器句柄）不支持ALTER SESSION SET CONTAINER语句。

  • 如果客户端最初连接到具有EXTENDED MAX_STRING_SIZE设置的容器，然后在同一会话中切换到具有STANDARD MAX_STRING_SIZE设置的容器（使用ALTER SESSION SET CONTAINER语句），则后续的OCIStmtExecute()调用将导致如果尝试使用大小超过4000字节的任何绑定变量，则ORA-14697错误。

  • 尝试使用OCIStmtFetch()或OCIStmtFetch2()在与执行它的容器不同的容器的上下文中从OCI语句句柄获取将导致ORA-65108错误。

  • 如果在OCI中完成ALTER SESSION SET CONTAINER语句，则禁用OCI客户端结果缓存。

  • 使用ALTER SESSION SET CONTAINER语句在可插拔数据库之间切换连接的应用程序不支持快速应用程序通知（FAN）和运行时连接负载平衡通知。

  • ALTER SESSION SET CONTAINER语句将当前事务（如果有）设置为只读以及任何尝试执行任何OCI事务调用（ OCITransStart() ， OCITransDetach() ， OCITransCommit() ， OCITransRollback() ， OCITransPrepare() ， OCITransMultiPrepare() ， OCITransForget() ）将在新容器中返回错误。要发出任何这些调用，您需要切换回原始容器。

  • 如果在不正确的容器（与完成相应的OCISubscriptionRegister()调用的容器不同OCISubscriptionUnRegister()的上下文中尝试进行OCISubscriptionRegister()调用，则返回ORA-24950 。

  • 甲OCIDescribeAny()与呼叫OCI_PTYPE_DATABASE描述了其中连接被连接到数据库中。完成ALTER SESSION SET CONTAINER语句后，如果应用程序想要查看当前数据库描述，则需要重新发出OCIDescribeAny()调用。

  • 调用任何OCI不支持用于操作来自不同容器的对象的任何数据，集合或对象函数。

  • 一个OCIObjectFlush()调用仅在对象实例用的创建容器支撑OCIObjectNew()调用。

  • Oracle建议在使用ALTER SESSION SET CONTAINER语句切换容器之前调用OCIObjectFlush() 。请注意，如果尚未启动事务，则OCIObjectFlush()调用将启动事务。

  • 在切换容器之后完成的OCIObjectFlush()调用可能会返回错误，如果事务已经由同一会话在另一个容器上先前启动（作为显式DML的结果或作为OCIObjectFlush()调用的结果）。

  • OCIObjectFlush()调用仅刷新在发出OCIObjectFlush()调用的容器的上下文中弄脏的对象。

  • ALTER SESSION SET CONTAINER语句中的各种会话属性可能会更改。如果应用程序缓存这些属性，则在ALTER SESSION SET CONTAINER语句之后它们的设置可能不再相同。可以使用OCIAttrGet()调用获取并且可以在ALTER SESSION SET CONTAINER语句上更改的属性示例包括以下内容：

    • OCI_ATTR_CURRENT_SCHEMA

    • OCI_ATTR_INITIAL_CLIENT_ROLES

    • OCI_ATTR_EDITION

    • OCI_ATTR_MAX_OPEN_CURSORS

17.5使用ALTER SESSION SWITCH CONTAINER SWITCH SERVICE对OCI呼叫的限制

描述使用ALTER SESSION SWITCH CONTAINER SWITCH SERVICE语句对OCI调用的限制。

从Oracle Database 12c第2版（12.2）开始，添加了ALTER SESSION SWITCH CONTAINER SWITCH SERVICE语句，如果新服务的配置与连接到原始服务时的配置不同，则可能导致服务属性发生更改。但是，OCI不会根据此声明的新设置更改其处理;相反，它将继续使用原始设置，如连接到原始服务时。例如，FAN开/关，TAF开/关设置等将保持与首次连接到原始服务时相同。这被认为是切换后的期望行为，因为ALTER SESSION SWITCH CONTAINER SWITCH SERVICE语句的正常使用情况是在多租户环境中，在切换后使用相同的应用程序。
