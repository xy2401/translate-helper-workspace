使用响应文件运行安装后配置

您可以使用响应文件同时在一个或多个服务器上完成安装后任务。

完成此过程以使用executeConfigTools命令和响应文件运行配置助手。

  1. 编辑响应文件并为配置指定所需的密码。您可以使用安装期间创建的响应文件，位于$ORACLE_HOME/install/response/ product_timestamp.rsp 。例如：

    对于Oracle数据库（ oracle用户）

    oracle.install.asm。SYSASMPassword = 密码 oracle.install.config.emAdminPassword = 密码

    对于Oracle Grid Infrastructure（ grid用户）

    grid.install.asm。SYSASMPassword = 密码 grid.install.config.emAdminPassword = 密码
  2. 将目录更改为包含安装软件的Oracle主目录。例如：

    对于Oracle Grid Infrastructure：

    cd Grid_home

    其中， Grid_home是Oracle Grid Infrastructure主目录的路径/u01/app/oracle/product/19.0.0/grid

    对于Oracle数据库：

    cd $ ORACLE_HOME
  3. 使用以下语法运行配置脚本：

    对于Oracle Grid Infrastructure：

    $ ./gridSetup.sh -executeConfigTools -responseFile Grid_home / install / response / product_timestamp.rsp

    对于Oracle数据库：

    $ ./runInstaller -executeConfigTools -responseFile $ ORACLE_HOME / install / response / product_timestamp.rsp

    对于Oracle数据库，您还可以运行位于$ORACLE_HOME/inventory/response/目录中的响应文件：

    $ ./runInstaller -executeConfigTools -responseFile $ORACLE_HOME/inventory/response/ db_install.rsp

    安装后配置工具以图形用户界面模式运行安装程序，显示安装后配置的进度。指定[-silent]选项以在静默模式下运行安装后配置。

    例如，对于Oracle Grid Infrastructure：

    $ ./gridSetup.sh -executeConfigTools -responseFile /u01/app/oracle/product/19.0.0/grid [-silent]

    对于Oracle数据库：

    $ ./runInstaller -executeConfigTools -responseFile /u01/app/oracle/product/19.0.0/dbhome_1/inventory/response/ db_2016-01-09_01-03-36PM.rsp [-silent]
