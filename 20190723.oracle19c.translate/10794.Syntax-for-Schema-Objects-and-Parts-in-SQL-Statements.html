SQL语句中的模式对象和部件的语法

本节介绍如何在SQL语句的上下文中引用模式对象及其部分。本节向您展示：

  • 引用对象的一般语法

  • Oracle如何解析对象的引用

  • 如何引用除您自己以外的模式中的对象

  • 如何引用远程数据库中的对象

  • 如何引用表和索引分区和子分区

您可以在将引用的组件分隔到对象的句点周围包含空格，但通常省略它们。

Oracle数据库如何解析模式对象引用

当您在SQL语句中引用对象时，Oracle会考虑SQL语句的上下文并将对象定位在相应的命名空间中。找到对象后，Oracle会执行该对象上的语句指定的操作。如果在相应的命名空间中找不到命名对象，则Oracle会返回错误。

以下示例说明了Oracle如何解析对SQL语句中对象的引用。考虑这个语句，它将一行数据添加到由名称departments标识的表中：

插入部门VALUES（280，'ENTERTAINMENT_CLERK'，206,1700）;

根据声明的上下文，Oracle确定departments可以是：

  • 您自己的架构中的表

  • 您自己的架构中的视图

  • 表或视图的私有同义词

  • 公共同义词

在考虑模式之外的名称空间之前，Oracle总是尝试在您自己的模式中解析名称空间内的对象引用。在此示例中，Oracle尝试按如下方式解析名称departments ：

  1. 首先，Oracle尝试在包含表，视图和私有同义词的自己的模式中定位名称空间中的对象。如果对象是私有同义词，则Oracle将找到同义词所代表的对象。此对象可以位于您自己的架构，另一个架构或另一个数据库中。该对象也可以是另一个同义词，在这种情况下，Oracle会找到此同义词所代表的对象。

  2. 如果对象位于命名空间中，则Oracle会尝试对该对象执行该语句。在此示例中，Oracle尝试将数据行添加到departments 。如果对象的语句类型不正确，则Oracle会返回错误。在此示例中， departments必须是表，视图或解析为表或视图的专用同义词。如果departments是序列，则Oracle返回错误。

  3. 如果对象不在目前为止搜索的任何名称空间中，则Oracle搜索包含公共同义词的名称空间。如果对象位于该命名空间中，则Oracle会尝试对其执行语句。如果对象的语句类型不正确，则Oracle会返回错误。在此示例中，如果departments是序列的公共同义词，则Oracle会返回错误。

如果公共同义词具有任何依赖表或用户定义类型，则无法在与依赖对象相同的模式中创建与同义词同名的对象。

如果同义词没有任何依赖表或用户定义类型，则可以在与依赖对象相同的模式中创建具有相同名称的对象。Oracle使任何依赖对象无效，并在下次访问时尝试重新验证它们。

对远程数据库中对象的引用

要引用除本地数据库之外的数据库中的对象，请使用对象名称以及该数据库的数据库链接名称。数据库链接是一种模式对象，它使Oracle连接到远程数据库以访问其中的对象。本节告诉您：

  • 如何创建数据库链接

  • 如何在SQL语句中使用数据库链接

创建数据库链接

使用语句CREATE DATABASE LINK创建数据库链接 。该语句允许您指定有关数据库链接的此信息：

  • 数据库链接的名称

  • 数据库连接字符串以访问远程数据库

  • 用于连接远程数据库的用户名和密码

Oracle将此信息存储在数据字典中。

数据库链接名称

创建数据库链接时，必须指定其名称。数据库链接名称与其他类型的对象的名称不同。它们可以长达128个字节，可以包含句点（。）和“at”符号（@）。

您为数据库链接指定的名称必须与数据库链接所引用的数据库的名称以及该数据库在数据库名称的层次结构中的位置相对应。以下语法图显示了数据库链接名称的形式：

用户名和密码

Oracle使用用户名和密码连接到远程数据库。数据库链接的用户名和密码是可选的。

数据库连接字符串

数据库连接字符串是Oracle Net用于访问远程数据库的规范。有关编写数据库连接字符串的信息，请参阅特定网络协议的Oracle Net文档。数据库链接的数据库连接字符串是可选的。

对数据库链接的引用

只有在使用Oracle分布式功能时，才能使用数据库链接。发出包含数据库链接的SQL语句时，可以使用以下其中一种形式指定数据库链接名称：

  • 存储在数据字典中的完整数据库链接名称，包括database ， domain和可选的connection_qualifier组件。

  • partial数据库链接名称是database和可选的connection_qualifier组件，但不是domain组件。

Oracle在连接到远程数据库之前执行以下任务：

  1. 如果语句中指定的数据库链接名称是partial，则Oracle会扩展名称以包含在数据字典中存储的全局数据库名称中找到的本地数据库的域。（您可以在GLOBAL_NAME数据字典视图中查看当前的全局数据库名称。）

  2. Oracle首先在您自己的模式中搜索私有数据库链接，其名称与语句中的数据库链接相同。然后，如有必要，它将搜索具有相同名称的公共数据库链接。

    • Oracle始终从第一个匹配的数据库链接（私有或公共）确定用户名和密码。如果第一个匹配的数据库链接具有关联的用户名和密码，则Oracle使用它。如果它没有关联的用户名和密码，则Oracle会使用您当前的用户名和密码。

    • 如果第一个匹配的数据库链接具有关联的数据库字符串，则Oracle使用它。否则，Oracle会搜索下一个匹配（公共）数据库链接。如果未找到匹配的数据库链接，或者没有匹配的链接具有关联的数据库字符串，则Oracle将返回错误。

  3. Oracle使用数据库字符串来访问远程数据库。访问远程数据库后，如果GLOBAL_NAMES参数的值为true ，则Oracle验证数据库链接名称的database.domain部分是否与远程数据库的完整全局名称匹配。如果此条件为真，则Oracle使用在步骤2中选择的用户名和密码继续连接。如果没有，Oracle会返回错误。

  4. 如果使用数据库字符串，用户名和密码的连接成功，则Oracle会尝试使用解析对象引用和引用本节前面讨论的其他模式中的对象的规则来访问远程数据库上的指定对象。

通过将初始化参数GLOBAL_NAMES或ALTER SYSTEM或ALTER SESSION语句的GLOBAL_NAMES参数设置为FALSE ，可以禁用数据库链接名称的database.domain部分必须与远程数据库的完整全局名称匹配的要求。

对分区表和索引的引用

可以对表和索引进行分区。分区时，这些模式对象由许多称为分区的部分组成，所有部分都具有相同的逻辑属性。例如，表中的所有分区共享相同的列和约束定义，并且索引中的所有分区共享相同的索引列。

分区扩展和子分区扩展名称允许您在一个分区或子分区上执行一些分区级和子分区级操作，例如从分区或子分区中删除所有行。如果没有扩展名，则此类操作将要求您指定谓词（ WHERE子句）。对于范围和列表分区表，尝试使用谓词表示分区级操作可能很麻烦，尤其是当范围分区键使用多个列时。对于散列分区和子分区，使用谓词仍然比较困难，因为这些分区和子分区基于系统定义的散列函数。

分区扩展名称允许您像使用表一样使用分区。此方法的一个优点是，对范围分区表最有用的是，您可以通过向这些视图（或来自）其他用户或角色授予（或撤销）特权来构建分区级访问控制机制。要将分区用作表，请通过从单个分区中选择数据来创建视图，然后将该视图用作表。

在partition_extended_name ， subpartition_extended_name和partition_extension_clause ， PARTITION FOR和SUBPARTITION FOR子句允许您在不使用其名称的情况下引用分区。它们对任何类型的分区都有效，对间隔分区特别有用。将数据插入表中时，会根据需要自动创建间隔分区。

对于相应的partition_key_value或subpartition_key_value ，为每个分区键列指定一个值。对于多列分区键，请为每个分区键指定一个值。对于复合分区，请为每个分区键指定一个值，然后为每个子分区键指定一个值。所有分区键值都以逗号分隔。对于间隔分区，您只能指定一个partition_key_value ，它必须是有效的NUMBER或datetime值。您的SQL语句将对包含您指定的值的分区或子分区进行操作。

扩展名称的限制

目前，分区扩展和子分区扩展表名的使用具有以下限制：

  • 无远程表：分区扩展或子分区扩展表名称不能包含数据库链接（dblink）或转换为具有dblink的表的同义词。要使用远程分区和子分区，请在远程站点上创建使用扩展表名语法的视图，然后引用远程视图。

  • 无同义词：必须使用基表指定分区或子分区扩展名。您不能使用同义词，视图或任何其他对象。

  • PARTITION FOR和SUBPARTITION FOR子句对视图的DDL操作无效。

  • 在PARTITION FOR和SUBPARTITION FOR子句中，不能为partition_key_value或subpartition_key_value指定关键字DEFAULT或MAXVALUE或绑定变量。

  • 在PARTITION和SUBPARTITION子句中，您不能为分区或子分区名称指定绑定变量。

例

在以下语句中， sales是分区表，其分区为sales_q1_2000 。您可以创建单个分区sales_q1_2000的视图，然后将其用作表格。此示例从分区中删除行。 

创建视图Q1_2000_sales AS SELECT * FROM sales PARTITION（SALES_Q1_2000）;从Q1_2000_sales删除WHERE amount_sold <0;
