C.2确定恢复目录模式版本

使用SQL查询确定恢复目录架构的版本。

要确定恢复目录模式的当前版本：

  1. 使用SQL * Plus作为目录所有者连接到恢复目录数据库。例如，输入：
    ％sqlplus rco @ catdb
  2. 查询rcver目录表。例如，运行此查询：
    SQL> SELECT * FROM rcver;版本------------ 12.01.00.01

C.3 RMAN兼容性矩阵

RMAN兼容性有一定的规则。

RMAN兼容性规则包括：

  • RMAN客户端的版本必须等于目标数据库的版本。

  • 辅助数据库的版本必须等于RMAN客户端的版本。

  • 恢复目录模式的版本必须大于或等于RMAN客户端的版本。

  • 如果恢复目录是虚拟专用目录 （请参阅CREATE CATALOG ），则连接到它的RMAN客户端的版本必须是Oracle数据库10g第1版（10.1.0.6）或Oracle数据库10g第2版（10.2.0.3） 。Oracle 9 i RMAN客户端无法连接到虚拟专用目录。即使基本目录具有虚拟专用目录用户，此版本限制也不会影响RMAN客户端与Oracle Database 11 g基本恢复目录的连接。

  • 在使用Oracle9 i RMAN客户端备份Oracle数据库10g或更高版本时，不能在数据文件备份集中包含使用COMPATIBLE=10.0.0创建的控制文件。解决方法是打开控制文件自动ON 。

  • 任何Oracle数据库版本都可以恢复任何以前的Oracle数据库版本创建的备份集和副本。

C.4恢复目录导出和导入的跨版本兼容性

数据泵恢复目录的导出通常用作备份其内容的方法。在计划使用Data Pump Export进行恢复目录的逻辑备份时，必须牢记与跨Oracle Oracle数据库的数据库导出使用相关的某些兼容性问题。

无法将更高版本数据库的导出导入到在早期版本下运行的数据库中。必须使用用于恢复目录的最早版本的Oracle数据库中的导出实用程序导出恢复目录数据。

例如，要从Oracle Database 11g第2版（11.2）数据库导出恢复目录数据并将其导入Oracle数据库10g第1版（10.1.0.5）数据库以进行灾难恢复，必须使用Oracle中的导出实用程序数据库10g第1版（10.1.0.5）执行导出操作。否则，导入操作失败。

C.5 RMAN兼容性：场景

本节包含用于使用各种Oracle数据库版本的生产数据库的RMAN客户端版本和恢复目录数据库版本的示例。

假设您维护以下版本的生产数据库：

  • Oracle Database 11g第1版（11.2）

  • Oracle Database 12 c第1版（12.1）

  • Oracle Database 12 c第2版（12.2）

  • Oracle数据库发行版18c，版本18.1

您希望在单个恢复目录数据库中记录有关这些数据库的RMAN存储库数据。根据RMAN兼容性矩阵 ，您可以使用单个恢复目录数据库，其版本为Oracle Database Release 18c，其目录模式的版本为Oracle Database Release 18c，适用于所有目标数据库。

确保用于备份每个目标数据库的RMAN客户端版本满足以下要求：

  • 使用Oracle Database 11g第1版（11.2）RMAN客户端备份Oracle Database 11g第1版（11.2）数据库。

  • 使用Oracle Database 12 c第1版（12.1）RMAN客户端备份Oracle Database 12 c第1版（12.1）数据库。

  • 使用Oracle Database 12 c第2版（12.2）RMAN客户端备份Oracle Database 12 c第2版（12.2）数据库。

  • 使用Oracle数据库版本18c版本18.1 RMAN客户端备份Oracle Database Release 18c数据库。
