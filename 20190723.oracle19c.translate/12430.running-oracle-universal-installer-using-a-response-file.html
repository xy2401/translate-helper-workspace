使用响应文件运行Oracle Universal Installer

创建响应文件后，在命令行运行Oracle Univeral Installer，指定您创建的响应文件，以执行安装。

在命令行运行Oracle Universal Installer，指定您创建的响应文件。Oracle Universal Installer可执行文件runInstaller提供了几个选项。有关这些选项的完整集的帮助信息，请使用-help选项运行runInstaller命令。例如：

$ directory_path / runInstaller -help

一段时间后，帮助信息会显示在窗口中。

要使用响应文件运行安装程序：

  1. 完成正常安装的预安装任务

  2. 以软件安装所有者用户身份登录。

  3. 如果要完成响应文件模式安装，请为运行安装的用户设置操作系统DISPLAY环境变量。

    注意：

    如果要完成静默模式安装，则不必设置DISPLAY环境变量。

  4. 要以静默或响应文件模式启动安装程序，请输入类似于以下内容的命令：

    $ / directory_path / runInstaller [-silent] [-noconfig] \ -responseFile responsefilename

    注意：

    不要指定响应文件的相对路径。如果指定相对路径，则安装程序将失败。

    在这个例子中：

    • directory_path是复制安装二进制文件的目录的路径。

    • -silent以静默方式运行安装程序。

    • -noconfig禁止在安装期间运行配置助手，而是执行纯软件安装。

    • responsefilename是您配置的安装响应文件的完整路径和文件名。

  5. 如果这是您第一次在系统上安装Oracle软件，则Oracle Universal Installer会提示您运行orainstRoot.sh脚本。

    以root用户身份登录并运行orainstRoot.sh脚本：

    $ su root密码：＃/ u01 / app / oraInventory / orainstRoot.sh

    注意：

    您不必手动创建oraInst.loc文件。运行orainstRoot.sh脚本就足够了，因为它指定了Oracle Inventory目录的位置。

  6. 安装完成后，以root用户身份登录并运行root.sh脚本。例如

    $ su root密码：＃/ oracle_home_path /root.sh
