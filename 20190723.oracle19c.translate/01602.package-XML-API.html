  • memory_free是一个低级别的内存释放功能，如果不使用free 。匹配memory_alloc函数。

  • memory_context是传递给alloc和free函数的用户定义的内存上下文。它的定义和使用完全取决于用户;它只是在这里设置并传递给回调。

XDK还有其他属性：

  • input_buffer_size是基本的I / O缓冲区大小。默认值为256K;范围是4K到4MB。根据编码，可能需要1,2或3个这些缓冲区。请注意，大小是字符，而不是字节。如果缓冲区保存Unicode数据，则它将是两倍大。

  • memory_block_size是高级内存包将从低级分配器请求的块的大小;它是内存分配的基本单位。默认值为64K;范围是16K到256K。

XmlLoadDom（）

从输入源加载（解析）XML文档并创建DOM。成功时返回根文档节点，失败时返回NULL （设置了err）。

该函数有两个固定的参数，xmlctx和一个错误返回码，然后是零或更多（属性，值）对，然后是NULL 。

SOURCE输入源由以下互斥属性之一设置（选择一个）：

  • （“ uri ”，文档URI）[编译器编码]

  • （“ file ”，文件文件系统路径）[编译器编码]

  • （“ buffer ”， buffer地址，“ buffer_length ”，缓冲区中的＃个字节）

  • （“ stream ”，流对象的地址，“ stream_context" ，指向流对象上下文的指针）

  • （“ stdio ”， FILE* stream）

PROPERTIES附加特性：

  • （“ dtd ”，DTD节点）文档的DTD

  • （“ base_uri ”，文档基URI）用于从URI以外的其他来源加载的文档，设置有效的基URI。需要文档的基URI才能解析相对URI包含，导入等。

  • （“ input_encoding ”，编码名称）强制输入编码[name]

  • （“ default_input_encoding ”， encoding_name ）默认输入编码，用于假设文档不是自描述的（没有BOM，协议头， XMLDecl等）

  • （“ schema_location ”， string ）此文档的架构的schemaLocation 。用于在将文档加载到数据库时计算最佳布局

  • （“ validate ”，boolean）当为TRUE ，打开DTD验证;默认情况下，仅检查格式良好。请注意，架构验证是一个单独的野兽。

  • （“ discard_whitespace ”，boolean）当为TRUE ，将丢弃输入文档中元素（换行符和缩进）之间的空格。默认情况下，保留所有输入字符。

  • （“ dtd_only ”，boolean）当为TRUE ，解析外部DTD，而不是完整的XML文档。

  • （“ stop_on_warning ”，boolean）当为TRUE ，警告被视为与错误相同并导致解析，验证等，立即停止。默认情况下，会发出警告但游戏仍在继续。

  • （“ warn_duplicate_entity ”，boolean）当为TRUE ，多次声明的实体将导致发出警告。默认是接受第一个声明并静默忽略其余声明。

  • （“ no_expand_char_ref ”，boolean）当为TRUE ，导致字符引用在DOM数据中保持未展开状态。通常，字符引用被它们代表的字符替换。但是，保存文档时，这些字符实体不会再次出现。为了确保它们保持通过负载和保存是不扩展它们。

  • （“ no_check_chars ”，boolean）当为TRUE ，省略XML测试[2] Char生成：所有输入字符都将被接受为有效

XmlSaveDom（）

将文档或子树序列化到给定目标并返回写入的字节数;如果没有提供目的地，只返回格式化的大小但不输出。

如果指定了输出编码，则将在输出上重新编码文档;否则，它将采用现有的编码方式。

顶级是缩进的step *级别空格，下一级是*（level + 1）空格，依此类推。

保存到缓冲区时，如果缓冲区溢出，则返回0并将err设置为XMLERR_SAVE_OVERFLOW 。

DESTINATION输出目的地由以下互斥属性之一设置（选择一个）：

  • （ “ ur I”，文件URI） POST ， PUT ？[编译器编码]

  • （“文件”，文件文件系统路径）[编译器编码]

  • （“ buffer ”， buffer地址，“b uffer_length ”，缓冲区中的＃个字节）

  • （“ stream ”，流对象的地址，“ stream_context ”，指向流对象上下文的指针）

特性附加特性：

  • （“ output_encoding ”，编码名称）文档的最终编码名称。除非指定，否则保存的文档将与xmlctx的编码相同。

  • （“ indent_step ”，unsigned）空格用于缩进每个输出级别。默认值是4 ， 0表示没有缩进。

  • （“ indent_level ”，unsigned）初始缩进级别。默认值为0 ，表示没有缩进，向左刷新。

  • （“ xmldecl" ，boolean）在输出文档中包含XMLDecl 。通常为竞争文档输出X MLDecl （根节点是DOC）。

  • （“ bom ”，boolean）在输出文档中输入BOM。通常只有某些编码（UTF-16）需要BOM，其他编码（UTF-8）可选。导致输出可选的BOM。

  • （“ prune ”，boolean）修剪输出，如unix'find'命令;不会下降到孩子，只打印给定的一个节点。
