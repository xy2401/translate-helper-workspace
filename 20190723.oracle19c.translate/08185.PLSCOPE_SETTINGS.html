1.262 PLSCOPE_SETTINGS

PLSCOPE_SETTINGS控制PL / SQL源代码标识符数据的编译时间收集，交叉引用和存储。

属性 描述

参数类型

串

句法

PLSCOPE_SETTINGS ='value_clause [，value_clause]'

value_clause :: =

{IDENTIFIERS |声明}：{ALL |没有| PUBLIC（仅限IDENTIFIERS）| SQL（仅限IDENTIFIERS）| PLSQL（仅限IDENTIFIERS）}

默认值

'标识符：NONE'

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

值

  • 标识符：ALL

    启用所有源代码标识符数据的收集。

  • 标识符：NONE

    禁用所有源代码标识符数据的集合。

  • 身份证明者：公共

    允许收集所有PUBLIC用户标识符数据（ DEFINITION除外）

  • 标识符：SQL

    启用所有SQL标识符数据的集合。

  • 标识符：PLSQL

    启用所有PLSQL标识符数据的收集。

  • 声明：ALL

    启用PL / SQL中使用的所有SQL语句的集合。

  • 声明：NONE

    禁用所有语句的集合。

可以在会话，系统或每个库单元（ ALTER COMPILE ）的基础上设置PLSCOPE_SETTINGS 。通过查询*_PLSQL_OBJECT_SETTINGS视图， PLSCOPE_SETTINGS任何库单元的PLSCOPE_SETTINGS的当前设置。可以使用*_IDENTIFIERS视图访问通过设置此参数收集的任何标识符数据。

如果未指定STATEMENTS设置，并且指定了IDENTIFIERS但设置为NONE以外的值，则STATEMENTS默认设置为ALL ，它等于：
IDENTIFIERS：[ALL | PLSQL | PLSQL | PUBLIC]

例子

以下参数设置会导致收集PL / SQL和SQL标识符和语句：

PLSCOPE_SETTINGS ='IDENTIFIERS：ALL，STATEMENTS：ALL'

以下参数设置仅导致收集SQL标识符和语句：

PLSCOPE_SETTINGS ='IDENTIFIERS：SQL，语句：全部'

以下参数设置仅导致收集PL / SQL标识符：

PLSCOPE_SETTINGS ='IDENTIFIERS：PLSQL，声明：无'

以下参数设置不会收集标识符数据，并丢弃任何现有标识符数据：

PLSCOPE_SETTINGS ='IDENTIFIERS：NONE'
