将Oracle ORAchk和Oracle EXAchk的运行状况检查结果集成到企业管理器和其他第三方工具中。

    将Oracle ORAchk和Oracle EXAchk的运行状况检查结果集成到Oracle Enterprise Manager中。

    将Oracle ORAchk和Oracle EXAchk的运行状况检查结果集成到各种第三方日志监控和分析工具中，例如Elasticsearch和Kibana。

2.10.1将运行状况检查结果与Oracle Enterprise Manager集成

Oracle Enterprise Manager Cloud Control版本13.1和13.2支持通过Oracle Enterprise Manager ORAchk Healthchecks插件与Oracle ORAchk和Oracle EXAchk集成。Oracle Engineered System Healthchecks插件支持与EXAchk for Oracle Enterprise Manager Cloud Control 12 c发行版12.1.0.5及更早版本的集成。

2.10.2将运行状况检查结果与第三方工具集成

将Oracle ORAchk和Oracle EXAchk的运行状况检查结果集成到各种第三方日志监控和分析工具中，例如Elasticsearch和Kibana。

图2-26第三方工具集成


“图2-26第三方工具集成”的说明
Oracle ORAchk和Oracle EXAchk在输出上载目录中创建JSON输出结果，例如：
Report_Output_Dir /upload/mymachine_orachk_results.json Report_Output_Dir /upload/mymachine_orachk_exceptions.json
Report_Output_Dir /upload/mymachine_exachk_results.json Report_Output_Dir /upload/mymachine_exachk_exceptions.json

  1. 运行–syslog选项以将JSON结果写入syslog守护程序。

    例如：
    ./orachk-syslog
    ./exachk -syslog

  2. 通过运行以下命令验证syslog配置：

    Oracle ORAchk和Oracle EXAchk使用消息级别： CRIT ， ERR ， WARN和INFO 。

    $ logger -p user.crit crit_message $ logger -p user.err err_message $ logger -p user.warn warn_message $ logger -p user.info info_message

  3. 验证配置的消息位置，例如/var/adm/messages是否写入每条测试消息。

2.10.3将运行状况检查结果与自定义应用程序集成

Oracle ORAchk和Oracle EXAchk将来自多个实例的集合结果上载到单个数据库中，以便更轻松地使用整个企业的检查结果。

使用Oracle Health Check Collections Manager或您自己的自定义应用程序来使用运行状况检查结果。

  1. 将集合结果上载到集合末尾的下表中：

    表2-3将收集结果上载到数据库中

    表 什么得到了上传

    rca13_docs

    完整的压缩收集结果。

    auditcheck_result

    健康检查结果。

    auditcheck_patch_result

    补丁检查结果。

    如果安装Oracle Health Check Collections Manager，则这些表由安装脚本创建。

  2. 如果未创建表，则使用以下DDL语句：
