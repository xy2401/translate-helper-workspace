Oracle Database Release 19c中的更改

以下是Oracle Database 19c的Oracle数据库SQL语言参考中的更改。

新功能

以下功能是版本19c中的新功能：

新功能

更精细的粒度补充记录

细粒度的补充日志记录为部分数据库复制用户提供了一种方法，即使在数据库或模式级别启用了补充日志记录，也可以禁用不感兴趣的表的补充日志记录。

在只有数据库中的某些表需要补充日志记录的情况下使用此功能，从而显着减少资源使用和重做生成的开销。

要使用此功能，请将新的supplemental_subset_replication_clause添加到ALTER DATABASE和ALTER PLUGGABLE DATABASE语句的supplemental_db_logging子句中。

LOB定位器的基于签名的安全性

从此版本开始，您可以为大对象（LOB）定位器配置基于签名的安全性。LOB签名密钥可以位于多租户PDB中，也可以位于独立的非多租户数据库中。

您可以通过执行ALTER DATABASE DICTIONARY ENCRYPT CREDENTIALS SQL语句来启用LOB签名密钥凭据的ALTER DATABASE DICTIONARY ENCRYPT CREDENTIALS 。否则，凭据以模糊格式存储。如果选择以加密格式存储LOB签名密钥，则数据库或PDB必须具有打开的TDE密钥库。

使用REST API支持云对象存储

Oracle Data Pump可以从位于受支持的对象库中的文件导入数据。

您可以在命令行上指定数据库凭据，并使用ALTER DATABASE语句的新property_clause将缺省凭据存储在数据库中。

系统管理分片的多表系列支持

此功能仅适用于系统管理的分片数据库。您可以使用CREATE SHARDED TABLE语句CREATE SHARDED TABLE在一个分片数据库上访问不同表系列的不同应用程序。

跨越碎片生成唯一序列号

对于具有唯一约束的非主键列，您可以跨分片生成全局唯一的序列号，而无需自行管理它们。分片数据库为您管理这些序列号。

使用CREATE SEQUENCE语句的SHARD子句或ALTER SEQUENCE语句在分片中生成唯一的序列号。

内存外部表的大数据和性能增强

CREATE TABLE和ALTER TABLE语句的inmemory_clause支持支持ORACLE_HIVE和ORACLE_BIGDATA驱动程序类型的规范。

您可以在非分区表的inmemory_clause子句中指定INMEMORY以支持这些驱动程序类型。

基于位图的计数Dictinct SQL函数

您可以使用五个新的位向量函数来加速SQL查询中的COUNT DISTINCT操作：

  • BITMAP_BUCKET_NUMBER
  • BITMAP_BIT_POSITION
  • BITMAP_CONSTRUCT_AGG
  • BITMAP_OR_AGG
  • BITMAP_COUNT

Memoptimized Rowstore - 快速摄取

使用CREATE TABLE或ALTER TABLE的memoptimize_write_clause启用快速摄取。快速摄取优化了物联网（IoT）应用程序的高频单行数据插入的内存处理，通过使用大缓冲池在将插入写入磁盘之前存储插入。

JSON对象映射

此功能支持将JSON数据映射到用户定义的SQL对象类型和集合，以及从用户定义的SQL对象类型

您可以使用SQL / JSON函数json_value将JSON数据转换为SQL对象类型的实例。在相反的方向，您可以使用SQL / JSON函数json_object或json_array从SQL对象类型的实例生成JSON数据。

JSON_MERGEPATCH函数

您现在可以使用新的SQL函数json_mergepatch声明方式更新JSON文档。您可以使用单个语句将一个或多个更改应用于多个文档。

此功能提高了JSON更新操作的灵活性。

JSON语法简化

使用函数json_object为SQL / JSON路径表达式和SQL / JSON生成提供语法简化。新的SQL查询子句NESTED提供了使用带有LEFT OUTER JOIN json_table的简单替代方法。

JSON_SERIALIZE和JSON数据指南支持GeoJSON数据

您可以使用新的SQL函数json_serialize将JSON数据序列化为文本或UTF编码的BLOB数据。

SQL聚合函数json_dataguide现在可以检测文档中的GeoJSON地理数据。您可以使用它来创建一个视图，将SQL数据类型SDO_GEOMETRY等数据SDO_GEOMETRY 。

混合分区表

您可以创建混合分区表，其中某些分区驻留在Oracle数据库段中，而某些分区驻留在外部文件和源中。可以根据需要将内部和外部分区集成到单个分区表中。

在CREATE TABLE或ALTER TABLE的table_partition_description子句中指定INTERNAL或EXTERNAL 。

奇偶校验保护文件

您可以为一次写入文件配置单一奇偶校验，例如存档日志和不需要高级冗余并节省空间的备份集。

在ALTER DISKGROUP语句的redundancy_clause中指定PARITY 。

LISTAGG聚合的DISTINCT选项

LISTAGG聚合函数现在通过使用新的DISTINCT关键字支持重复消除。

统一审计顶级声明

如果要审核用户直接发出的SQL语句，请在CREATE AUDIT POLICY语句（Unified Auditing）中指定ONLY TOPLEVEL子句。

透明在线转换支持非OMF模式下的自动重命名

如果在ALTER TABLESPACE ENCRYPTION语句中省略了FILE_NAME_CONVERT ，Oracle将在内部为辅助文件选择一个名称，稍后将其重命名为原始名称。

ALTER SYSTEM子句FLUSH PASSWORDFILE_METADATA_CACHE

命令ALTER SYSTEM FLUSH PASSWORDFILE_METADATA_CACHE刷新存储在SGA中的密码文件元数据高速缓存，并通知数据库发生了更改。

不推荐的功能

此版本中不推荐使用以下功能，并且可能在将来的版本中不支持这些功能：

不推荐使用SQLNET.ENCRYPTION_WALLET_LOCATION参数。

不受支持的功能

Oracle Database Release 19c中不支持以下功能：

  • 不支持Oracle多媒体。

  • Oracle Steams不受支持。

有关版本19c的不支持功能的完整列表，请参阅“ Oracle数据库升级指南” 。
