17数据仓库的运输

17.1数据仓库运输概述

运输是将数据从一个系统移动到另一个系统的操作。在数据仓库环境中，最常见的传输要求是从以下位置移动数据：

  • 源系统到登台数据库或数据仓库数据库

  • 数据仓库的临时数据库

  • 数据集市的数据仓库

运输通常是ETL过程中较简单的部分之一，并且可以与过程的其他部分集成。例如，如数据仓库中的提取 ，分布式查询技术提供了一种提取和传输数据的机制。

17.2数据仓库中的运输机制简介

在仓库中传输数据有三种基本选择：

  • 使用平面文件的运输

  • 分布式运营的运输

  • 使用可传输表空间的传输

17.2.1使用平面文件进行运输

传输数据最常见的方法是使用FTP或其他远程文件系统访问协议等机制传输平面文件。使用数据仓库中的提取中讨论的技术将数据从源系统卸载或导出到平面文件中 ，然后使用FTP或类似机制将数据传输到目标平台。

由于源系统和数据仓库通常使用不同的操作系统和数据库系统，因此使用平面文件通常是在异构系统之间以最少的转换交换数据的最简单方法。但是，即使在同构系统之间传输数据时，平面文件通常也是数据传输最有效和最易于管理的机制。

17.2.2通过分布式操作进行的运输

具有或不具有网关的分布式查询可以是用于提取数据的有效机制。这些机制还将数据直接传输到目标系统，从而在一个步骤中提供提取和转换。根据对时间和系统资源的可容忍影响，这些机制非常适合于提取和转换。

与平面文件传输相反，传输的成功或失败立即被分布式查询或事务的结果识别。

17.2.3使用可传输表空间进行传输

Oracle可传输表空间是在两个Oracle数据库之间移动大量数据的最快方法。在引入可传输表空间之前，最具可伸缩性的数据传输机制依赖于移动包含原始数据的平面文件。这些机制要求将数据从源数据库卸载或导出到文件中。然后，在传输之后，将这些文件加载或导入到目标数据库中。可传输的表空间完全绕过卸载和重新加载步骤。

使用可传输的表空间，Oracle数据文件（包含表数据，索引和几乎所有其他Oracle数据库对象）可以直接从一个数据库传输到另一个数据库。此外，与导入和导出一样，可传输表空间除了传输数据外，还提供传输元数据的机制。

可传输表空间有一些限制：源系统和目标系统必须运行Oracle8 i （或更高版本），必须使用兼容的字符集，并且在Oracle Database 10g之前必须在同一操作系统上运行。有关如何在操作系统之间传输表空间的详细信息，请参见“ Oracle数据库管理员指南” 。

可传输表空间在数据仓库中最常见的应用是将数据从登台数据库移动到数据仓库，或将数据从数据仓库移动到数据集市。

17.2.3.1使用可传输表空间将数据传输到数据仓库：示例

假设您有一个包含销售数据的数据仓库，以及每月刷新的多个数据集市。还假设您要将一个月的销售数据从数据仓库移动到数据集市。

使用以下步骤创建可传输的表空间：

  1. 将要传输的数据放入其自己的表空间中
  2. 导出元数据
  3. 将数据文件和导出文件复制到目标系统
  4. 导入元数据

将要传输的数据放入其自己的表空间中

必须将当前月份的数据放入单独的表空间中才能进行传输。在此示例中，您有一个表空间ts_temp_sales ，它包含当前月份数据的副本。使用CREATE TABLE ...AS SELECT语句，可以将当前月份的数据有效地复制到此表空间：

CREATE TABLE temp_jan_sales NOLOGGING TABLESPACE ts_temp_sales AS SELECT * FROM sales WHERE time_id BETWEEN '31 -DEC-1999'AND'01-FEB-2000';

执行此操作后，表空间ts_temp_sales将设置为只读：

ALTER TABLESPACE ts_temp_sales READ ONLY;

除非没有修改表空间的活动事务，否则无法传输表空间。将表空间设置为只读会强制执行此操作。

表空间ts_temp_sales可以是一个表空间，专门用于临时存储数据以供可传输表空间功能使用。在“ 将数据文件和导出文件复制到目标系统 ”之后 ，可以将此表空间设置为读/写，如果需要，可以删除表temp_jan_sales ，或者可以将表空间重新用于其他传输或其他传输目的。

在给定的可传输表空间操作中，传输给定表空间中的所有对象。虽然在此示例中仅传输一个表，但表空间ts_temp_sales可能包含多个表。例如，数据集市可能不仅刷新了新月的销售交易，而且还刷新了客户表的新副本。这两个表都可以在同一个表空间中传输。此外，此表空间还可以包含其他数据库对象，例如索引，这些对象也将被传输。

此外，在给定的可传输表空间操作中，可以同时传输多个表空间。这样可以更轻松地在数据库之间移动大量数据。但请注意，可传输表空间功能只能传输一组表空间，这些表空间包含一组完整的数据库对象，而不依赖于其他表空间。例如，索引不能在没有表的情况下传输，也不能在没有表的其余部分的情况下传输分区。您可以使用DBMS_TTS包来检查表空间是否可传输。

在此步骤中，您已将1月销售数据复制到单独的表空间中;但是，在某些情况下，可以利用可传输的表空间功能，甚至无需将数据移动到单独的表空间。如果销售表已在数据仓库中按月分区，并且每个分区都在其自己的表空间中，则可以直接传输包含January数据的表空间。假设January分区sales_jan2000位于表空间ts_sales_jan2000 。然后可能会传输表空间ts_sales_jan2000 ，而不是在ts_temp_sales创建January销售数据的临时副本。

但是，必须满足相同的条件才能传输表空间ts_sales_jan2000 ，这是特殊创建的表空间所需的。首先，必须将此表空间设置为READ ONLY 。其次，由于分区表的单个分区无法在没有分区表的其余部分传输的情况下进行传输，因此必须将January分区交换为单独的表（使用ALTER TABLE语句）来传输January数据。EXCHANGE操作非常快，但是1月数据将不再是基础sales表的一部分，因此在导出元数据之后将这些数据交换回sales表之前，用户可能无法使用它。完成“ 将数据文件和导出文件复制到目标系统 ”步骤后，可以将1月数据交换回sales表。

导出元数据

导出实用程序用于导出描述传输的表空间中包含的对象的元数据。对于我们的示例场景，Export命令可以是：

EXP TRANSPORT_TABLESPACE = y TABLESPACES = ts_temp_sales FILE = jan_sales.dmp

此操作生成导出文件jan_sales.dmp 。导出文件很小，因为它只包含元数据。在这种情况下，导出文件包含描述表temp_jan_sales ，例如列名，列数据类型以及目标Oracle数据库为了访问ts_temp_sales的对象而需要的所有其他信息。

将数据文件和导出文件复制到目标系统

使用平面文件的任何传输机制将组成ts_temp_sales的数据文件以及导出文件jan_sales.dmp到数据集市平台。复制数据文件后，如果需要，可以将表空间ts_temp_sales设置为READ WRITE模式。

导入元数据

将文件复制到数据集市后，应将元数据导入数据集市：

IMP TRANSPORT_TABLESPACE = y DATAFILES ='/ db / tempjan.f'TABLESPACES = ts_temp_sales FILE = jan_sales.dmp

此时，可以在数据集市中访问表空间ts_temp_sales和表temp_sales_jan 。您可以将此新数据合并到数据集市的表中。

您可以通过以下两种方式之一将temp_sales_jan表中的数据插入到数据集市的sales表中：

INSERT / * + APPEND * / INTO销售SELECT * FROM temp_sales_jan;

执行此操作后，您可以删除temp_sales_jan表（甚至整个ts_temp_sales表空间）。

或者，如果数据集市的销售表按月分区，则新传输的表空间和temp_sales_jan表可以成为数据集市的永久部分。temp_sales_jan表可以成为数据集市sales表的分区：

ALTER TABLE sales ADD PARTITION sales_00jan VALUES少于（TO_DATE（'01 -feb-2000'，'dd-mon-yyyy'））; ALTER TABLE sales EXCHANGE PARTITION sales_00jan WITH TABLE temp_sales_jan包含带有验证的索引;
17.2.3.2可移动表空间的其他用途

前面的示例说明了在数据仓库中传输数据的典型方案。但是，可传输表空间可用于许多其他目的。在数据仓库环境中，可传输表空间应被视为实用程序（非常类似于Import / Export或SQL * Loader），其目的是在Oracle数据库之间移动大量数据。与并行数据移动操作结合使用时，例如CREATE TABLE ...AS SELECT和INSERT ......AS SELECT语句，可传输表空间为快速传输数据提供了一个重要的机制，用于多种用途。
