12 Oracle数据库安装后任务

安装Oracle数据库后完成配置任务。

安装Oracle数据库后，您需要完成一些配置任务。此外，Oracle建议您在安装后立即完成其他任务。在使用这些产品之前，还必须完成特定于产品的配置任务。

注意：

本章仅介绍基本配置。有关更详细的配置和调整信息，请参阅特定于产品的管理和调整指南。

  • 必需的安装后任务
    完成初始安装后，下载并应用软件版本所需的修补程序。
  • 推荐的安装后任务
    Oracle建议您在安装后完成这些任务。
  • 关于Oracle数据库的默认SGA权限的更改
    从Oracle Database 12c第2版（12.2.0.1）开始，默认情况下，读取和写入系统全局区域（SGA）的权限仅限于Oracle软件安装所有者。

    使用以下步骤检查Oracle数据库安装的内容和目录位置：
  • 安装后启用和禁用Oracle数据库选项
    chopt工具在安装后更改数据库选项。
  • 启动Oracle Enterprise Manager Database Express
    使用以下步骤登录Oracle Enterprise Manager Database Express（EM Express）。
  • 关于使用Oracle Fleet补丁和配置部署Oracle数据库
    您可以使用Oracle Fleet Patching and Provisioning（Oracle FPP）来配置Oracle数据库软件。
  • 创建快速恢复区
    在Oracle Restart安装期间，您只能创建一个磁盘组。在Oracle Clusterware安装期间，您可以创建多个磁盘组。如果计划为独立服务器或Oracle RAC数据库添加Oracle数据库，则应为数据库文件创建快速恢复区域。
  • 克隆Oracle数据库主页
    克隆Oracle主目录涉及创建Oracle主目录的副本，然后针对新环境进行配置。
