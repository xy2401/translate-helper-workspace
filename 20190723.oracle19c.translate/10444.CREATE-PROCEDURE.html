创建程序

目的

使用PL / SQL定义过程。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用CREATE PROCEDURE语句创建独立存储过程或调用规范。

过程是一组可以按名称调用的PL / SQL语句。调用规范 （有时称为调用规范）声明Java方法或第三代语言（3GL）例程，以便可以从SQL和PL / SQL调用它。调用规范告诉Oracle数据库在调用时调用哪个Java方法。它还告诉数据库为参数和返回值进行的类型转换。

存储过程在开发，完整性，安全性，性能和内存分配方面具有优势。

也可以看看：

  • 有关存储过程的详细信息，请参阅“Oracle数据库开发指南” ，包括如何调用存储过程以及有关注册外部过程的信息。

  • CREATE FUNCTION用于特定于函数的信息，这些函数在很多方面与过程类似。

  • 有关创建包的信息，请参阅CREATE PACKAGE 。CREATE PROCEDURE语句将过程创建为独立模式对象。您还可以创建一个过程作为包的一部分。

  • ALTER PROCEDURE和DROP PROCEDURE上修改和删除一个独立的程序信息。

  • 有关共享库的更多信息，请参见CREATE LIBRARY 。

先决条件

要在您自己的模式中创建或替换过程，您必须具有CREATE PROCEDURE系统特权。要在另一个用户的模式中创建或替换过程，您必须具有CREATE ANY PROCEDURE系统特权。

要调用调用规范，您可能需要其他权限，例如，对于C调用规范，C库的EXECUTE对象权限。

要在Oracle预编译器程序中嵌入CREATE PROCEDURE语句，必须使用关键字END-EXEC后跟特定语言的嵌入式SQL语句终止符来终止该语句。

语义

或替换

指定OR REPLACE以重新创建过程（如果已存在）。使用此子句可以更改现有过程的定义，而无需删除，重新创建和重新授予先前在其上授予的对象权限。如果重新定义过程，则Oracle Database会重新编译它。

以前被授予重新定义过程特权的用户仍然可以访问该过程而无需重新授予特权。

如果任何基于函数的索引依赖于该过程，则Oracle Database会将索引标记为DISABLED 。
