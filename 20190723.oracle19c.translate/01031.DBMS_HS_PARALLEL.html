81 DBMS_HS_PARALLEL

DBMS_HS_PARALLEL PL / SQL包支持异构目标访问的并行处理。此程序包旨在提高从大型外表检索数据时的性能。

本章讨论以下主题：

  • 使用DBMS_HS_PARALLEL

  • DBMS_HS_PARALLEL子程序摘要

81.1使用DBMS_HS_PARALLEL子程序

DBMS_HS_PARALLEL使用CURRENT_USER的授权ID进行编译，该ID使用调用者的权限。换句话说，此包中的所有过程都使用调用用户的权限执行。

81.2 DBMS_HS_PARALLEL子程序摘要

此表按字母顺序列出DBMS_HS_PARALLEL子程序并简要描述它们。

表81-1 DBMS_HS_PARALLEL包子程序

子程序 描述

CREATE_OR_REPLACE_VIEW

创建（或替换）要引用的只读视图，以便并行检索远程表中的数据。

CREATE_TABLE_TEMPLATE

根据从远程表收集的信息写出CREATE TABLE模板。您可以使用该信息添加任何最佳的Oracle CREATE TABLE子句。

DROP_VIEW

删除CREATE_OR_REPLACE_VIEW过程创建的视图和内部对象。如果CREATE_OR_REPLACE_VIEW过程尚未创建视图，则会返回错误消息。

LOAD_TABLE

将数据从远程表并行加载到本地Oracle表。如果本地Oracle表尚不存在，则会自动创建。

81.2.1 CREATE_OR_REPLACE_VIEW过程

此过程创建（或替换）要引用的只读视图，以便并行检索远程表中的数据。

句法

 CREATE_OR_REPLACE_VIEW（remote_table，database_link，oracle_view，parallel_degree）

参数

表81-2 CREATE_OR_REPLACE_VIEW过程参数

参数 描述

remote_table

远程数据库表的名称。它被指定为[remote_schema_name.]remote_table_name 。

database_link

远程数据库链接名称。该调用只能应用于异构服务数据库链接。

oracle_view

Oracle视图的名称。它被指定为[schema_name.]oracle_view_name 。默认架构名称是当前用户。如果未指定oracle_view参数，则远程表名称将用作视图名称。

parallel_degree

根据范围分区号（如果适用）或CPU数计算操作的并行进程数。值的范围是2到16 。

使用说明

  • 创建指定的Oracle视图，此视图的未来引用利用内部数据库对象来并行检索远程非Oracle表数据。如果Oracle视图已存在，则会引发以下Oracle错误消息：

    ORA-00955：名称已被现有对象使用

  • 此视图创建为只读视图。如果您尝试插入并更新视图，则会引发以下Oracle错误消息：

    ORA-01733：此处不允许使用虚拟列

  • 如果远程表或数据库链接不存在，则会引发以下Oracle错误消息之一：

    ORA-00942：表或视图不存在或ORA-02019：找不到远程数据库的连接描述

  • 您需要CREATE VIEW ， CREATE TABLE ， CREATE TYPE ， CREATE PACKAGE和CREATE FUNCTION权限才能执行CREATE_OR_REPLACE_VIEW过程。

  • 如果遇到以下任一Oracle错误消息，请在Oracle初始化参数文件中增加PROCESSES和SESSIONS参数：

    ORA-12801: error signaled in parallel query server P003或ORA-00018: maximum number of session exceeded ORA-12801: error signaled in parallel query server P003 ORA-00018: maximum number of session exceeded

  • 由于CREATE_OR_REPLACE_VIEW过程会创建一些内部对象，因此请使用DROP_VIEW过程删除视图和内部对象。SQL DROP VIEW语句仅删除视图而不删除内部对象。

81.2.2 CREATE_TABLE_TEMPLATE过程

此过程基于从远程表收集的信息写出CREATE TABLE模板。您可以使用该信息添加任何最佳的Oracle CREATE TABLE子句。

句法

CREATE_TABLE_TEMPLATE（remote_table，database_link，oracle_table，create_table_template_string）

参数

表81-3 CREATE_TABLE_TEMPLATE过程参数

参数 描述

remote_table

远程数据库表的名称。它被指定为[remote_schema_name.]remote_table_name 。

database_link

远程数据库链接名称。该调用只能应用于异构服务数据库链接。

oracle_table

将加载数据的本地Oracle表的名称。它被指定为[schema_name.]oracle_table_name 。默认架构名称是当前用户。如果未指定oracle_table参数，则远程表名将用作本地Oracle名称。

create_table_template_string

返回过程时包含Oracle CREATE TABLE SQL模板。

81.2.3 DROP_VIEW程序

此过程将删除CREATE_OR_REPLACE_VIEW过程创建的视图和内部对象。如果CREATE_OR_REPLACE_VIEW过程尚未创建视图，则会返回错误消息。

句法

DROP_VIEW（oracle_view）

参数

表81-4 DROP_VIEW过程参数

参数 描述

oracle_view

CREATE_OR_REPLACE_VIEW过程创建的Oracle视图的名称。如果CREATE_OR_REPLACE_VIEW过程尚未创建视图，则会返回错误。

81.2.4 LOAD_TABLE程序

此过程并行地将数据从远程表加载到本地Oracle表。如果本地Oracle表尚不存在，则会自动创建。

句法

LOAD_TABLE（remote_table，database_link，oracle_table，truncate，parallel_degree，row_count）

参数

表81-5 LOAD_TABLE过程参数

参数 描述

remote_table

远程数据库表的名称。它被指定为[remote_schema_name.]remote_table_name

database_link

远程数据库链接名称。该调用只能应用于异构服务数据库链接。

oracle_table

将加载数据的本地Oracle表的名称。它被指定为[schema_name.]oracle_table_name 。默认架构名称是当前用户。如果未指定oracle_table参数，则远程表名将用作本地Oracle名称。

截短

确定在加载数据之前是否截断Oracle表。值为TRUE或FALSE 。默认值为TRUE ，这意味着首先截断Oracle表。设置为FALSE ，在加载数据之前不会截断Oracle表。

parallel_degree

根据范围分区号（如果适用）或CPU数计算操作的并行进程数。值的范围是2到16 。

ROW_COUNT

包含刚加载表操作时添加的行数。

使用说明

  • 此过程仅将远程表数据加载到Oracle本地表中。它不会创建键，索引，约束或任何其他依赖项（如触发器）。建议您在加载表数据后创建这些依赖项，因为性能会大大提高。您需要根据您对远程表数据和依赖关系的了解，决定是在加载数据之前还是之后创建依赖关系。

  • 如果本地表不存在，则LOAD_TABLE过程基于远程表的精确列匹配创建简单（非分区）本地表，之后将数据插入到本地表中。

  • 如果远程表或数据库链接不存在，则返回错误消息。

  • 如果本地表与远程表不兼容，则返回错误消息。

  • 您需要CREATE TABLE ， CREATE TYPE ， CREATE PACKAGE和CREATE FUNCTION权限才能执行LOAD_TABLE过程。

  • 如果遇到以下任一Oracle错误消息，请在Oracle初始化参数文件中增加PROCESSES和SESSIONS参数：

    ORA-12801: error signaled in parallel query server P003或ORA-00018: maximum number of session exceeded ORA-12801: error signaled in parallel query server P003 ORA-00018: maximum number of session exceeded

  • 并行处理需要以下其中一项：

    • 远程表是范围分区的。

    • 可以使用数字列的直方图信息。

    • 有一个数字索引或主键。

  • 要删除本地表，请使用DROP TABLE SQL语句。
