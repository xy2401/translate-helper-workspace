17 Oracle Messaging Gateway简介

Messaging Gateway管理包DBMS_MGWADM提供了一个界面，用于创建Messaging Gateway代理，管理代理，创建消息系统链接，注册非Oracle队列以及设置传播作业。

17.1 Oracle Messaging Gateway概述

Messaging Gateway支持基于非Oracle消息传递系统和Oracle数据库高级队列的应用程序之间的通信。

Oracle Database Advanced Queuing提供两个Oracle Database Advanced Queuing队列之间的传播 ，以启用电子商务（HTTP到IDAP ）。Messaging Gateway将此扩展到基于非Oracle消息传递系统的应用程序。

由于Messaging Gateway与Oracle数据库高级队列和Oracle数据库集成在一起，因此可以提供可靠的消息传递。Messaging Gateway保证在Oracle Database Advanced Queuing和支持持久性的非Oracle消息传递系统之间只传递一次消息。PL / SQL接口提供易于学习的管理API ，尤其适用于已经熟练使用Oracle数据库高级队列的开发人员。

此版本的Messaging Gateway支持Oracle数据库高级队列与基于WebSphere MQ 9.0.5.0和TIB / Rendezvous 8.2的应用程序的集成。

17.2 Oracle Messaging Gateway功能

Messaging Gateway提供以下功能：

  • 扩展Oracle数据库高级队列消息传播

    Messaging Gateway在Oracle Database Advanced Queuing和非Oracle消息传递系统之间传播消息。非Oracle邮件系统应用程序可以接收Oracle Database Advanced Queuing应用程序发送的消息。相反，Oracle数据库高级排队应用程序可以使用非Oracle消息传递系统应用程序发布的消息。

  • 支持Java消息服务 （JMS）消息传递系统

    Messaging Gateway在Oracle Java消息服务（Oracle JMS）和WebSphere MQ Java消息服务（WebSphere MQ JMS）之间传播消息。

  • 本机消息格式支持

    Messaging Gateway支持消息传递系统的本机消息格式。Oracle数据库高级队列消息可以具有RAW或任何Oracle 对象类型负载。WebSphere MQ消息可以是文本消息或字节消息。TIB / Rendezvous消息可以是除嵌套数据类型MSG和具有无符号整数的数据类型之外的任何TIB / Rendezvous有线数据类型。

  • 邮件转换

    Messaging Gateway有助于Oracle Database Advanced Queuing消息和非Oracle消息系统消息之间的消息转换。消息通过Messaging Gateway提供的自动例程或您提供的自定义消息转换功能进行转换 。

    注意：

    • Messaging Gateway不支持JMS和非JMS消息传递系统之间的消息传播。

    • MGW在12 c版本2（12.2）中不支持Oracle数据库AQ分片队列

  • 与Oracle数据库集成

    Messaging Gateway通过类似于Oracle Database Advanced Queuing的PL / SQL接口进行管理。配置信息存储在Oracle数据库表中。消息传播由Oracle数据库服务器的外部进程执行。

  • 保证邮件传递

    如果传播源和传播目标上的消息传递系统都支持事务，则Messaging Gateway会保证持久消息只传播一次。如果消息不是持久性的，或者传播源或传播目的地的消息传递系统不支持事务，则保证最多一次传播。

  • 安全支持

    Messaging Gateway支持Oracle数据库和非Oracle消息传递系统的客户端身份验证。

    Messaging Gateway还允许对Messaging Gateway代理进行的IBM WebSphere MQ和WebSphere MQ JMS连接提供安全套接字层（SSL）支持。

  • 多代理支持

    Messaging Gateway支持给定数据库的多个代理。用户可以根据功能，组织或工作负载对传播作业进行分区，并将它们分配给不同的Messaging Gateway代理。这允许Messaging Gateway在Oracle RAC环境中进行扩展，并启用传播作业分组和隔离。

17.3.1管理包DBMS_MGWADM

Messaging Gateway管理包DBMS_MGWADM提供了一个接口，用于创建命名的Messaging Gateway代理，管理代理，创建消息系统链接，注册非Oracle队列以及设置传播作业。

无论Messaging Gateway代理是否正在运行，用户都会调用程序包中的过程进行配置更改。如果Messaging Gateway代理正在运行，则程序包中的过程会向代理发送配置更改通知。尽管某些更改要求代理在生效之前关闭并重新启动，但代理会动态更改其配置以进行大多数配置更改。程序包中的所有过程都是序列化的，以保证Messaging Gateway代理以与生成顺序相同的顺序接收和处理通知。

17.3.2 Oracle Messaging Gateway代理

Messaging Gateway代理作为Oracle数据库服务器的外部进程运行，并处理传播作业。通过调用DBMS_MGWADM包中的STARTUP和SHUTDOWN过程来启动和关闭它。

Messaging Gateway代理包含多线程传播引擎和一组用于消息传递系统的驱动程序。传播引擎公平地调度传播作业并同时处理传播作业。代理中的轮询线程会定期轮询已启用的传播作业的源队列，并在消息可用时唤醒工作线程以处理传播作业。非Oracle消息传递系统的驱动程序作为所有消息传递操作的消息传递系统的客户端运行。

17.3.3 Oracle数据库

作为Oracle数据库功能，Messaging Gateway提供了Oracle数据库高级队列和非Oracle消息传递系统之间的消息传播机制。Oracle Database Advanced Queuing作为传播源或传播目标在每个传播作业中都包含在内。

Messaging Gateway通过PL / SQL管理包DBMS_MGWADM管理。 Messaging Gateway的所有配置信息和执行状态信息都存储在Oracle数据库中，可以通过数据库视图访问。

Messaging Gateway代理作为Oracle数据库服务器的外部过程运行。因此，它仅在其关联的数据库服务器正在运行时运行。

17.3.4非Oracle消息系统

Messaging Gateway代理通过邮件系统链接连接到非Oracle邮件系统。消息传递系统链接是Messaging Gateway代理和非Oracle消息传递系统之间的通信通道。用户可以使用管理包DBMS_MGWADM来配置指向相同或不同非Oracle消息传递系统的多个链接。

非Oracle消息传递系统中的队列（例如WebSphere MQ队列，TIB / Rendezvous主题和WebSphere MQ JMS目标（队列和主题））都可以充当Messaging Gateway的传播源和目标。他们被称为外国队列。必须通过管理包注册作为源队列，目标队列或异常队列的消息传播所涉及的所有外部队列。外部队列的注册不会在非Oracle消息传递系统中创建物理队列，而只是记录有关队列的信息，例如用于访问它的消息传递系统链接，其本机名称及其域（队列或主题） 。必须通过非Oracle消息传递系统的管理界面创建物理队列。

17.4传播处理概述

必须定义传播作业，以便将消息从一个消息传递系统传播到另一个消息传递系统。传播作业定义源队列，目标队列以及影响传播作业处理的各种其他属性。

如果传播源是队列（点对点），则Messaging Gateway代理会将队列中的所有消息移动到目标。如果传播源是主题（ 发布/订阅 ），则Messaging Gateway代理会在传播源主题上创建订阅。在创建订阅后，代理会移动发布到主题的所有消息。

启用传播作业时将对其进行处理。禁用传播作业会停止传播处理，但不会停止邮件订阅。

当Messaging Gateway代理处理传播作业时，它会使来自源队列的消息出列，并将消息排入目标队列。随着每个消息的传播，它将从源消息传递系统中的本机格式转换为目标消息传递系统中的本机格式。Messaging Gateway在简单和常用消息格式之间提供自动消息转换。您可以通过提供自己的消息转换功能来自定义消息转换。

当Messaging Gateway代理无法将消息从源格式转换为目标格式时，如果存在异常队列 ，代理会将消息从源队列移动到异常队列，并继续处理传播作业。

如果Messaging Gateway代理在处理传播作业时遇到故障，则在停止重试之前，它会以指数退避方案（从2秒到30分钟）重试最多16次。

为了保证可靠的消息传递，Messaging Gateway需要在支持事务和持久消息的消息传递系统中记录队列。Messaging Gateway代理使用日志记录队列来存储传播作业的处理状态，以便它可以从故障中恢复传播处理。

17.5 Oracle数据库AQ缓冲消息和消息传递网关

Messaging Gateway不支持缓冲消息的传播。在出站传播中，Messaging Gateway代理仅从AQ队列中取消持久性消息。在入站传播中，Messaging Gateway代理始终将持久性消息排入AQ队列。
