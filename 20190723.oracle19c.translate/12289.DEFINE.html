12.17定义

句法

DEF [INE] [ 变量 ] | [ variable = text ]

指定一个用户或预定义变量并为其分配CHAR值，或列出单个变量或所有变量的值和变量类型。

条款

变量

表示要分配或列出其值的用户或预定义变量。

文本

表示要分配给变量的CHAR值。如果文本包含标点符号或空格，请将文本用单引号括起来。 

变量 = 文本

定义（命名）替换变量并为其赋值 CHAR值。

输入DEFINE后面变量列表中选择值和变量的类型。 输入DEFINE，不带子句列出所有替换变量的值和类型。

用法

定义的变量保留其值，直到您：

  • 输入引用变量的新DEFINE命令

  • 进入UNDEFINE 命令引用变量

  • 输入ACCEPT 命令引用变量

  • 引用COLUMN的NEW_VALUE或OLD_VALUE子句中的变量命令然后在SELECT中引用该列命令

  • 退出SQL * Plus

无论何时运行存储的查询或脚本，SQL * Plus都会替换每个变量的值替代变量引用变量 （形式和变量或&& 变量 ）。直到取消定义变量 SQL * Plus将不提示您输入变量的值，在这个环节。

如果已定义变量的值扩展到多行（使用SQL * Plus命令延续字符），则SQL * Plus将使用空格替换每个延续字符和回车符。例如，SQL * Plus解释

DEFINE TEXT ='ONE-TWO-THREE'

如

DEFINE TEXT ='一两三'

您应该避免使用可能与传递给它们的值相同的名称定义变量，因为可能会出现意外结果。如果为定义的变量提供的值与变量名匹配，则使用匹配变量的内容而不是提供的值。

SQL * Plus启动时预定义了一些变量。输入DEFINE以查看其定义。

例子

要将值MANAGER分配给变量POS，请键入：

DEFINE POS =经理

如果执行包含对＆POS的引用的命令，则SQL * Plus会将值MANAGER替换为＆POS，并且不会提示您输入POS值。

要将CHAR值20分配给变量DEPARTMENT_ID，请键入：

DEFINE DEPARTMENT_ID = 20

即使输入数字20，SQL * Plus也会为DEPARTMENT_ID分配一个CHAR值，该值包含两个字符，即2和0。

要列出DEPARTMENT_ID的定义，请输入

DEFINE DEPARTMENT_ID
DEFINE DEPARTMENT_ID =“20”（CHAR）

此结果显示DEPARTMENT_ID的值为20。

12.17.1预定义变量

在SQL * Plus安装期间定义了九个变量。这些变量仅通过具有预定义值而与用户定义的变量不同。

表12-3 SQL * Plus安装中预定义的变量

变量名 包含

_CONNECT_IDENTIFIER

用于建立连接的连接标识符（如果可用）。

_日期

当前日期或用户定义的固定字符串。

_编辑

指定EDIT使用的编辑器命令。

_O_VERSION

已安装的Oracle数据库的当前版本。

_O_RELEASE

已安装的Oracle数据库的完整版本号。

_特权

当前连接的权限级别。

_SQLPLUS_RELEASE

已安装的SQL * Plus组件的完整版本号。

_用户

用于建立连接的用户名。
_SQL_ID 执行SQL语句的sql_id。

_CONNECT_IDENTIFIER

包含连接标识符中的INSTANCE_NAME, SERVICE_NAME或ORACLE_SID 。如果用户在连接期间未提供连接标识符，则_CONNECT_IDENTIFIER包含ORACLE_SID 。

_日期

包含当前日期作为动态变量或固定字符串。当前日期是默认日期，并使用NLS_DATE_FORMAT的值进行格式化。

因为_DATE可以用作普通替换变量，所以用户可以将其放在TTITLE中。如果_DATE是动态的并且在TTITLE中使用它将具有所有正常的变量语义。如果它与和号一起使用，则该值将设置为执行TTITLE命令的时间。如果在没有＆符号前缀的情况下使用它，则将针对每个页面重新评估它。对于在TTITLE中具有_DATE的长报告或对＆_DATE的多个引用，可以为每次出现的变量显示不同的时间。

在TTITLE中使用_DATE的用户几乎肯定会想要使用＆符号＆_DATE，以便报表的每个页面具有完全相同的时间戳。当前日期格式包含“秒”组件时尤其如此。

DEFINE（没有参数）或使用＆_DATE取消引用将给出当前日期。

_DATE值可以是UNDEFINED，或设置为具有显式DEFINE _DATE的固定字符串。

您可以使用以下命令重新启用默认动态日期行为：

DEFINE _DATE =“”（空字符串）

_DATE使时间值可以包含在SQL * Plus提示中。

_编辑

指定EDIT使用的默认编辑器命令。

在Windows上运行SQL * Plus期间系统，它设置为记事本。在UNIX操作系统上，它设置为UNIX环境变量EDITOR的值（如果存在），否则设置为Ed。

你可以使用DEFINE 命令重新定义_EDITOR ，保存首选文本编辑器的名称。例如，要将EDIT使用的编辑器定义为vi，请输入以下命令：

DEFINE _EDITOR = vi

_O_VERSION

包含以下形式的已安装Oracle数据库的当前版本：

Oracle Database 19 c企业版19.0.0.0.0版 - 生产

版本19.1.0.0.0

_O_RELEASE

包含以下格式的已安装Oracle数据库的完整版本号：

18.01亿

_特权

包含指示当前连接的权限级别的值。它包含以下值之一：

  • 作为SYSASM

  • AS SYSBACKUP

  • AS SYSDBA

  • AS SYSDG

  • 作为SYSOPER

  • AS SYSRAC

  • 用于正常用户连接或无连接时的空字符串。

AS SYSASM，AS SYSBACKUP，AS SYSDBA，AS SYSDG，AS SYSOPER和AS SYSRAC是数据库管理员级别权限。

有关AS SYSDBA和AS SYSOPER权限的信息，请参阅GRANT 。

_SQLPLUS_RELEASE

包含以下格式的已安装SQL * Plus组件的完整版本号：

18.01亿

_用户

包含连接到当前连接的用户名。

_SQL_ID

包含当前执行的SQL或PL / SQL语句的sql_id。

您可以使用DEFINE命令查看每个变量的值。

可以像任何其他替换变量一样访问和重新定义这些变量。它们可以在TTITLE，'＆'替换变量或SQL * Plus命令行提示符中使用。

您可以使用DEFINE命令以与查看其他DEFINE定义相同的方式查看这九个预定义变量的定义。您还可以使用DEFINE命令重新定义它们的值，或者可以使用UNDEFINE命令删除它们的定义并使它们不可用。

要查看特定变量定义，请输入

DEFINE 变量

其中variable是要查看其定义的替换变量的名称。

要查看所有预定义和用户定义的变量定义，请输入

限定

显示所有预定义和所有用户定义的变量定义。

您可以使用UNDEFINE删除替换变量定义并使其不可用。

使用预定义变量的示例

要更改SQL * Plus提示以显示连接标识符，请输入：

SET SQLPROMPT'_CONNECT_IDENTIFIER>'

查看预定义的值 _SQLPLUS_RELEASE替换变量，输入

DEFINE _SQLPLUS_RELEASE

将显示预定义变量_SQLPLUS_RELEASE的值。

DEFINE _SQLPLUS_RELEASE =“1801000000”（CHAR）
