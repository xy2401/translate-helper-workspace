6.52 DBA_THRESHOLDS

DBA_THRESHOLDS描述了所有阈值。

柱 数据类型 空值 描述

METRICS_NAME

VARCHAR2（64）

指标名称

WARNING_OPERATOR

VARCHAR2（12）

警告阈值的关系运算符：

  • GT

  • EQ

  • LT

  • LE

  • 通用电器

  • CONTAINS

  • NE

  • 不要检查

  • DO_NOT_CHECK

WARNING_VALUE

VARCHAR2（256）

警告阈值

CRITICAL_OPERATOR

VARCHAR2（12）

关键阈值的关系运算符：

  • GT

  • EQ

  • LT

  • LE

  • 通用电器

  • CONTAINS

  • NE

  • 不要检查

  • DO_NOT_CHECK

CRITICAL_VALUE

VARCHAR2（256）

临界阈值

OBSERVATION_PERIOD

数

观察期长度（以分钟为单位）

CONSECUTIVE_OCCURRENCES

数

发出警报之前发生的次数

INSTANCE_NAME

VARCHAR2（16）

实例名称;数据库范围的警报为NULL

OBJECT_TYPE

VARCHAR2（64）

对象类型：

  • 系统

  • 服务

  • EVENT_CLASS

  • TABLESPACE

  • 文件

OBJECT_NAME

VARCHAR2（513）

设置阈值的对象的名称

状态

VARCHAR2（7）

指示阈值是否适用于有效对象（ VALID ）（ INVALID ）

非表空间度量标准的阈值只能在ROOT设置并作为整体应用于CDB。PDB中可能存在的任何预先存在的非表空间阈值在DBA_THRESHOLDS视图中的状态为INVALID 。您可以使用DBMS_SERVER_ALERT.SET_THRESHOLD API删除这些阈值设置。

有关DBMS_SERVER_ALERT.SET_THRESHOLD API的详细信息，请参阅Oracle数据库PL / SQL包和类型参考 。
