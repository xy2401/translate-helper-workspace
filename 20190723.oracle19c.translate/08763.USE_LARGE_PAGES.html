1.351 USE_LARGE_PAGES

USE_LARGE_PAGES用于管理数据库对SGA内存的大页面的使用。

属性 描述

参数类型

串

句法

USE_LARGE_PAGES = {TRUE | FALSE |只要 }

默认值

真正

可修改

没有

可在PDB中修改

没有

基本

没有

Oracle RAC

多个实例可以使用不同的值

此参数不会影响进程专用内存分配。USE_LARGE_PAGES仅适用于Linux操作系统。

值

可以为USE_LARGE_PAGES指定这些值：

  • 真正

    指定如果在系统上配置了大页面，则实例可以使用大页面。

    在Oracle Database 11g第2版（11.2.0.2）中，如果系统上没有配置足够的大页面，则将使用常规大小的页面来分配SGA内存。这可能导致空闲大页面未使用，操作系统可以分配大量内存来创建页表，以将SGA映射到Oracle进程的物理页面。这可能会导致ORA-04030错误并严重影响实例性能。

    在Oracle Database 11g第2版（11.2.0.3）中，Oracle在大页面中分配尽可能多的SGA，如果它用完了，它将使用常规大小的页面分配剩余的SGA。这可能导致实例为SGA创建额外的共享内存段，但总SGA大小将保持不变。在此支持的混合页面模式分配中，数据库将在切换到常规大小的页面之前耗尽可用的大页面。

  • 假

    指定实例不使用大页面。建议不要使用此设置，因为它可能会导致实例性能严重下降。

  • 只要

    指定如果大页面不能用于整个SGA内存，则实例将无法启动。

注意：

USE_LARGE_PAGES设置为FALSE时，在Oracle ASM实例会自动MEMORY_TARGET启用。
