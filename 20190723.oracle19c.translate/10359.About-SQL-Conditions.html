关于SQL条件

条件可以有多种形式，如以下语法所示。

condition :: =


插图condition.eps的描述

如果已安装Oracle Text，则可以使用属于该产品的内置运算符创建条件，包括CONTAINS ， CATSEARCH和MATCHES 。有关这些Oracle Text元素的更多信息，请参阅Oracle Text Reference 。

以下部分描述了各种形式的条件。只要condition出现在SQL语句中，就必须使用适当的条件语法。

您可以在这些语句的WHERE子句中使用条件：

  • 删除

  • 选择

  • UPDATE

您可以在SELECT语句的任何子句中使用条件：

  • 哪里

  • START WITH

  • CONNECT BY

  • HAVING

虽然Oracle数据库不正式支持这种数据类型，但可以说条件属于逻辑数据类型。

以下简单条件始终计算为TRUE ：

1 = 1

以下更复杂的条件将salary值添加到commission_pct值（将值0替换为null）并确定总和是否大于常量25000的数字：

NVL（薪水，0）+ NVL（薪水+（薪水*佣金_pct，0）> 25000）

逻辑条件可以将多个条件组合成单个条件。例如，您可以使用AND条件组合两个条件：

（1 = 1）和（5 <7）

以下是一些有效条件：

name ='SMITH'emberss.department_id = departments.department_id hire_date> '01 -JAN-08'work_id IN（'SA_MAN'，'SA_REP'）工资BETWEEN 5000和10000 commission_pct IS NULL AND salary = 2100

Oracle数据库不接受所有SQL语句的所有部分中的所有条件。有关该语句中条件限制的信息，请参阅本书中专门讨论特定SQL语句的部分。

条件优先

优先级是Oracle数据库评估同一表达式中不同条件的顺序。在评估包含多个条件的表达式时，Oracle会在评估优先级较低的条件之前评估具有较高优先级的条件。Oracle在表达式中从左到右评估具有相同优先级的条件，但以下情况除外：

  • 对于使用AND连接的多个条件，无法保证从左到右的评估

  • 对于使用OR连接的多个条件，无法保证从左到右的评估
