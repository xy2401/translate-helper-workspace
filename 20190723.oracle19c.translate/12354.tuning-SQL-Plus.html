8调整SQL * Plus

本章提供有关如何调整SQL * Plus以获得更好性能的信息。它讨论了以下主题：

  • 关于跟踪报表

  • 关于收集时间统计

  • 跟踪并行和分布式查询

  • 早期数据库中的执行计划输出

  • 关于SQL * Plus脚本调整

有关调整Oracle数据库的信息，请参阅“ Oracle数据库性能调整指南” 。

8.1关于跟踪报表

您可以自动获取有关SQL优化程序使用的执行路径和语句执行统计信息的报告。在成功的SQL DML（即SELECT，DELETE，UPDATE和INSERT）语句之后生成报告。它对监控和监控非常有用调整这些陈述的表现。

如果动态采样有效，则DML的SQL * Plus报表输出可能不同。

8.1.1控制Autotrace报告

您可以通过设置来控制报告 AUTOTRACE系统变量。

自动跟踪设置 结果

SET AUTOTRACE OFF

没有生成AUTOTRACE报告。这是默认值。

在EXPLAIN上设置AUTOTRACE

AUTOTRACE报告仅显示优化程序执行路径。

设置自动化统计数据

AUTOTRACE报告仅显示SQL语句执行统计信息。

SET AUTOTRACE ON

AUTOTRACE报告包括优化程序执行路径和SQL语句执行统计信息。

设置AUTOTRACE TRACEONLY

与SET AUTOTRACE ON类似，但禁止打印用户的查询输出（如果有）。如果启用了STATISTICS，则仍会获取查询数据，但不会打印。

要使用此功能，您必须创建一个 PLAN_TABLE表在你的架构，然后有授予您的PLUSTRACE角色。授予PLUSTRACE角色需要DBA权限。有关如何授予角色以及如何创建PLAN_TABLE表的信息，请参阅Oracle数据库SQL语言参考 。

有关角色和PLAN_TABLE的更多信息，请参阅“ Oracle数据库SQL语言参考”和SET命令的AUTOTRACE变量。

注意：

SQL * Plus AUTOTRACE不支持使用ALTER SESSION SET CONTAINER选项切换容器。在这种情况下收集的统计数据可能不一致。

有关使用AUTOTRACE收集统计信息的信息，请参阅TIMING命令和关于跟踪语句 。

8.4早期数据库中的执行计划输出

Oracle Database 9 i Release 2（9.2）或更早版本的执行计划输出是不同的。

执行计划的每一行都有一个连续的行号。SQL * Plus还显示父操作的行号。

执行计划由按以下顺序显示的四列组成：

列名称 描述

ID_PLUS_EXP

显示每个执行步骤的行号。

PARENT_ID_PLUS_EXP

显示每个步骤与其父级之间的关系。此列对大型报告很有用。

PLAN_PLUS_EXP

显示报告的每个步骤。

OBJECT_NODE_PLUS_EXP

显示使用的数据库链接或并行查询服务器

可以使用COLUMN命令更改列的格式。例如，要停止显示PARENT_ID_PLUS_EXP列，请输入

COLUMN PARENT_ID_PLUS_EXP NOPRINT

使用EXPLAIN PLAN命令生成执行计划输出。

当你追踪一个在并行或分布式查询中的语句，执行计划显示基于成本的优化器对行数（基数）的估计。通常，每个节点的成本，基数和字节表示累积结果。例如，连接节点的成本不仅要考虑完成连接操作的成本，还要考虑访问该连接中的关系的全部成本。

标有星号（*）的行表示并行或远程操作。每个操作都在报告的第二部分中解释。有关并行和分布式操作的详细信息，请参阅分布式事务概念

此报告的第二部分包含按以下顺序显示的三列

列名称 描述

ID_PLUS_EXP

显示每个执行步骤的行号。

OTHER_TAG_PLUS_EXP

描述OTHER_PLUS_EXP列中SQL语句的功能。

OTHER_PLUS_EXP

显示并行服务器或远程数据库的查询文本。

可以使用COLUMN命令更改列的格式。

8.5关于SQL * Plus脚本调整

大多数性能优势来自调优脚本中执行的SQL查询。这是通过SQL * Plus的AUTOTRACE命令等工具完成的。它涉及重构查询以充分利用Oracle数据库SQL优化器。有关调整SQL语句的信息，请参阅“ Oracle数据库性能调整指南” 。

调整SQL * Plus特定命令所带来的性能提升较小，但对某些应用程序可能很重要。下列系统变量和命令可以影响SQL * Plus的性能。

8.5.1列NOPRINT

COLUMN NOPRINT 关闭屏幕输出和打印列标题和为列选择的所有值。

最好从SELECT中删除不需要的列，然后使用COLUMN NOPRINT来停止显示。从查询中删除列意味着SQL引擎不需要处理它，或者需要将列数据传输回SQL * Plus。

8.5.2设置APPINFO关闭

设置自动通过注册脚本 DBMS_APPLICATION_INFO包。将APPINFO设置为OFF可防止管理员监视脚本的性能和资源使用情况。

如果正在调用许多SQL脚本，则关闭APPINFO会停止对数据库DBMS_APPLICATION_INFO程序包的内部SQL * Plus调用。

8.5.3设置ARRAYSIZE

设定数量 SQL * Plus将一次从数据库中获取的行。有效值为1到5000。

设置ARRAYSIZE的有效性取决于Oracle数据库填充网络数据包的程度以及网络延迟和吞吐量。在最新版本的SQL * Plus和Oracle数据库中，ARRAYSIZE可能收效甚微。超大尺寸可以轻松占用更多SQL * Plus内存，这可能会降低整体性能。

8.5.4设置定义关闭

SET DEFINE OFF禁用解析要替换的命令替换变量及其值。

8.5.5设置FLUSH OFF

设置冲洗使操作系统能够缓冲输出。ON禁用缓冲和刷新输出到屏幕。将FLUSH设置为ON或OFF的任何好处取决于您的操作系统和数据。收益可能是微不足道的。

仅在运行不需要用户交互的脚本时才使用OFF，并且在脚本完成运行之前不需要查看其输出。

8.5.6设置LINESIZE

SET LINESIZE设置总数在开始换行之前，SQL * Plus在一行上显示的字符。

保持LINESIZE尽可能小，以避免额外的内存分配和内存复制。

但是，如果LINESIZE太小，则不能彼此相邻的列将放在单独的行上。这可能会显着降低性能。

8.5.7设置LONGCHUNKSIZE

SET LONGCHUNKSIZE套装 SQL * Plus用于检索BLOB的增量大小 ，BFILE，CLOB ， 长 ，NCLOB 或XMLType 值。

如果正在获取LONGS或LOB，请尝试不同的大小。

8.5.8 SET PAGESIZE

集每页上的行数输出

增加PAGESIZE以避免频繁打印标题，或将其设置为0以防止显示标题。

8.5.9 SET SERVEROUTPUT

SET SERVEROUTPUT OFF抑制显示输出（ DBMS_OUTPUT.PUT_LINE）SQL * Plus中的存储过程或PL / SQL块。

将SERVEROUTPUT设置为OFF将停止对用户SQL语句之后完成的DBMS_OUTPUT包的内部SQL * Plus调用。

8.5.10设置SQLPROMPT

设置SQL * Plus命令提示符。

使用默认提示“SQL>”可在每次显示提示时停止发生变量替换。

8.5.11 SET TAB

确定SQL * Plus如何格式化终端输出中的空白区域。

设置TAB ON会导致在终端输出中压缩多个空格。除非这显着减少了书面数据，否则所需的处理可能会略微超过任何好处。

8.5.12 SET TERMOUT

组 TERMOUT OFF会禁止显示，以便您可以从脚本中输出假脱机，而无需在屏幕上看到它。

如果不需要假脱机到文件和写入终端，请在SQL脚本中使用SET TERMOUT OFF来禁用终端输出。

8.5.13设置TRIMSPOOL ON时设置TRIMOUT

组 TRIMOUT ON或SET TRIMSPOOL ON删除每个显示或假脱机行末尾的尾随空白。

将这些变量设置为ON可以减少写入的数据量。但是，如果LINESIZE是最佳的，则将变量设置为OFF可能会更快。 SQL * Plus输出行在整个查询处理例程中都是空白的，因此删除空格可能需要额外的工作。

8.5.14未定

删除显式（使用DEFINE命令）或隐式（使用START命令的参数或COLUMN NEW_VAL | OLD_VAL）定义的替换变量。

使用UNDEFINE命令删除不必要的替换变量。这可以减少使用'＆'，new_value或old_value变量的任何操作所花费的时间。
