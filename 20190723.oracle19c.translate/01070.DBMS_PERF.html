120.1 DBMS_PERF概述

DBMS_PERF包提供用于生成数据库性能报告的接口。所有子程序都返回一个活动报告，这些报告可以在系统级别，会话级别或SQL级别生成。

120.2 DBMS_PERF安全模型

DBMS_PERF包需要DBA角色。

120.3 DBMS_PERF子程序摘要

此表列出了DBMS_PERF子程序并简要描述了它们。

120.3.1 REPORT_PERFHUB功能

此函数在指定的时间段内生成整个数据库系统的复合活动性能报告。

句法

DBMS_PERF.REPORT_PERFHUB（is_realtime IN NUMBER DEFAULT NULL，outer_start_time IN DATE DEFAULT NULL，outer_end_time IN DATE DEFAULT NULL，selected_start_time IN DATE DEFAULT NULL，selected_end_time IN DATE DEFAULT NULL，inst_id IN NUMBER DEFAULT NULL，dbid IN NUMBER DEFAULT NULL，monitor_list_detail IN NUMBER DEFAULT NULL，workload_sql_detail IN NUMBER DEFAULT NULL，addm_task_detail IN NUMBER DEFAULT NULL，report_reference IN VARCHAR2 DEFAULT NULL，report_level IN VARCHAR2 DEFAULT NULL，输入IN VARCHAR2 DEFAULT'ACTIVE'，base_path IN VARCHAR2 DEFAULT NULL）;返回CLOB;

参数

表120-2 REPORT_PERFHUB函数参数

参数 描述

is_realtime

如果是1，则实时。如果为NULL （默认值）或0 ，则为历史模式。

outer_start_time

时间选择器中显示的外部时间的开始时间。如果为NULL （默认值）：

  • 如果is_realtime = 0 （历史），则在outer_end_time之前24小时。

  • 如果is_realtime = 1 （实时模式），则在outer_end_time之前1小时。

outer_end_time

时间选择器中显示的外部时间段的结束时间。如果为NULL （默认值），则为最新的AWR快照。

  • 如果is_realtime = 0 （历史）， is_realtime最新的AWR快照

  • 如果is_realtime = 1 （实时模式），这是当前时间（并且忽略任何输入）

selected_start_time

选择的开始时间段。如果为NULL （默认）

  • 如果is_realtime = 0 ，则在selected_end_time之前1小时

  • 如果is_realtime = 1 ，则在selected_end_time之前5分钟

selected_end_time

结束时间选择。如果为NULL （默认）

  • 如果is_realtime = 0 ，那么最新的AWR快照

  • If is_realtime = 1 ，那么当前时间

INST_ID

要为其检索数据的实例ID

  • 如果为-1 ，则为当前实例

  • 如果指定了number，则为该实例

  • 如果为NULL （默认值），则为所有实例

DBID

要查询的DBID 。

  • 如果为NULL ， DBID当前DBID 。

  • 如果is_realtime = 1 ，则DBID必须是本地DBID 。

monitor_list_detail

SQL监视器列表中的前N个，用于检索SQL监视器详细信息。

  • 如果为NULL （默认值），则检索前10名

  • 如果为0 ，则不检索监控列表详细信息

workload_sql_detail

Workload Top SQL列表中的前N个用于检索监视器详细信息，

  • 如果为NULL （默认值），则检索前10名

  • 如果为0 ，则不检索监控列表详细信息

addm_task_detail

要检索的最多N个最新ADDM任务

  • 如果为NULL （默认值），则检索可用数据但不超过N.

  • 如果为0 ，则不检索任何ADDM任务详细信息

report_reference

从SQL * Plus使用时必须为NULL 。

report_level

'typical'将获得性能中心的所有标签

类型

报告类型：

  • 'ACTIVE' （默认）

  • 'xml'返回XML

BASE_PATH

HTML资源的URL路径，因为flex HTML需要访问外部文件。这仅适用于type =' ACTIVE' ，通常不使用。默认值将从OTN检索所需的文件。

使用说明

  • 选择时间段后，将根据性能主题区域收集和显示性能信息。

  • 时间段可以是实时的或历史的。

  • 选择实时数据时，会显示更精细的数据，因为数据点每分钟都可用。

  • 选择历史数据时，会显示更详细的数据（按不同的度量标准细分），但数据点将平均到自动工作负载存储库（AWR）间隔（通常为一小时）。

  • Performance Hub中提供了不同的选项卡，具体取决于实时模式的is_real-time是1还是历史模式的0。

120.3.2 REPORT_SESSION函数

此函数为特定数据库会话生成性能报告，其中会话由inst_id ， sid和serial_num标识。

如果缺少任何这些参数，则报告将用于当前会话。

会话级性能报告包含以下选项卡：

  • 摘要 - 此选项卡包含会话的关键标识符和属性以及其活动数据的摘要。它还包含由实时SQL监视监视的该会话执行的SQL，PLSQL块和数据库操作（DBOP）列表。

  • 活动 - 此选项卡显示按此等会话的等待类细分的活动。用于此图表的数据是从活动会话历史记录（ASH）中获取的。

  • 度量标准 - 此选项卡显示所选会话的某些关键度量标准的图表，并且仅在历史模式下可用。显示的一些指标包括CPU使用率，PGA使用率，IO吞吐量和IO请求。

句法

DBMS_PERF.REPORT_SESSION（inst_id IN NUMBER DEFAULT NULL，sid IN NUMBER DEFAULT NULL，serial IN NUMBER DEFAULT NULL，is_realtime IN NUMBER DEFAULT NULL，outer_start_time IN DATE DEFAULT NULL，outer_end_time IN DATE DEFAULT NULL，selected_start_time IN DATE DEFAULT NULL，selected_end_time IN DATE DEFAULT NULL，dbid IN NUMBER DEFAULT NULL，monitor_list_detail IN NUMBER DEFAULT NULL，report_reference IN VARCHAR2 DEFAULT NULL，report_level IN VARCHAR2 DEFAULT NULL，输入IN VARCHAR2 DEFAULT'ACTIVE'，base_path IN VARCHAR2 DEFAULT NULL）RETURN CLOB;

120.3.3 REPORT_SQL函数

此函数为其sql_id标识的特定SQL语句生成活动性能报告。

SQL级性能报告包含以下选项卡：

  • 摘要 - 此选项卡包含SQL语句的概述，其中包含SQL文本，用户名，执行它的会话以及相关信息等关键属性。它还包含一个Plans选项卡，显示在内存和AWR中找到的此SQL语句的每个不同计划的统计信息和活动。

  • 活动 - 此选项卡显示按此SQL语句的等待类细分的活动。用于此图表的数据是从活动会话历史记录（ASH）中获取的。

  • 执行统计信息 - 此选项卡显示此语句的每个不同计划的统计信息和活动，以及计划的图形和表格表示。

  • 受监视的SQL - 此选项卡中列出了由实时SQL监视监视的此SQL语句的所有执行。

  • 计划控制 - 此选项卡显示有关此SQL语句的SQL配置文件和SQL计划基准的信息。

  • 历史统计 - 此选项卡仅在历史模式下可用。它包含统计信息，例如执行次数，I / O数，处理的行数以及不同执行计划随时间产生的其他信息。此信息从AWR中检索。

句法

DBMS_PERF.REPORT_SQL（sql_id IN varchar2默认为null，is_realtime IN数字默认为null，outer_start_time IN日期默认为null，outer_end_time IN日期默认为null，selected_start_time IN日期默认为null，selected_end_time IN日期默认为null，inst_id IN数字默认为null，dbid IN数字默认值null，monitor_list_detail IN number default null，report_reference IN varchar2 default null，report_level IN varchar2 default null，type IN varchar2 default'ACTIVE'，base_path IN varchar2 default null）;返回CLOB;
