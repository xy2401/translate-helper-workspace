advmutil export create

目的

创建导出并分配其初始存储。

语法和说明

advmutil export create -h advmutil export create -n export_name -c cluster_name -p storage_repos {-s file_size | -F}

下表包含advmutil export create命令可用的选项。

表16-111 advmutil export create命令的选项

选项 描述

-n export_name

指定导出名称。

-c cluster_name

指定成员群集名称。

-p storage_repos

指定存储后备文件的存储库的位置。

-s file_size

以n {K | M | G | T | P | E}的格式指定文件大小。单位是K（千字节），M（兆字节），G（千兆字节），T（太字节），P（千兆字节）或E（艾字节）。

-F

指定重用现有存储库。

advmutil export create管理集群的导出。它负责创建导出并分配其初始存储。创建导出时，出于性能原因分配完全分配（非稀疏）文件。

创建导出时， -n选项提供的名称也将成为成员集群上卷的名称。如果提供了-n expvol1 ，则会导致在成员集群上创建设备/dev/asm/expvol1-000 。

-p选项必须为存储库指定Oracle ACFS文件系统。如果存在Oracle ACFS集群件（CRS）资源，则会将其修改为依赖于注册到集群的任何传输VIP资源。

如果备份文件已存在于存储库中，则必须提供-f选项。创建后备文件后，Oracle ACFS会将其列为Oracle ACFS远程服务（使用Remote标志）正在使用中，并阻止对该文件执行各种操作。

出口快照仅限于DSC。在成员集群上安装的Oracle ACFS文件系统的成员集群上创建的快照只能在成员集群上访问。

此命令仅适用于域服务群集（DSC）。

例16-107使用advmutil export create

在以下示例中，将在cluster1上创建export1 ，其中存储库位于/repository1并且分配的大小为512 M.

＃/ sbin / advmutil export create -n export1 -c cluster1 -p / repository1 -s 512M Creating File Backed Volume。已创建的集群“cluster1”的卷“export1”的导出和后备存储。

要将Oracle ACFS远程服务存储库与DSC存储库的Oracle ACFS快照一起使用，必须在DSC上执行一些手动步骤。捕获DSC存储库时，将捕获存储库中的所有后备文件。

在以下示例中，快照将导出到同一成员集群。

  1. 创建一个支持文件。

    ＃/ sbin / advmutil export create -c mc1 -p / repo -n vol1 -s 2G

  2. 捕捉备份文件。

    $ / sbin / acfsutil snap create -w snap1 / repo

  3. 在存储库的根目录中创建快照链接：

    $ / sbin / acfsutil snap link -s snap1 / repo / snap1_link

  4. 使用-f选项导出快照。

    ＃/ sbin / advmutil export create -f -n vol1 -p / repo / snap1_link -c mc1

在以下示例中，快照将导出到其他成员集群。

  1. 创建一个支持文件。

    ＃/ sbin / advmutil export create -c mc1 -p / repo -n vol1 -s 2G

  2. 捕捉备份文件。

    $ / sbin / acfsutil snap create -w snap3 / repo

  3. 在存储库的根目录中创建快照链接：

    $ / sbin / acfsutil snap link -s snap3 / repo / snap3_link

  4. 使用ln链接到新成员集群存储库中的文件。

    请注意，f903 ...是旧成员集群的GUID，而54633 ...是新成员集群的GUID。

    $ ln -sf / repo / 54633dd2059d6f2cbff1217a6054564b / vol1 / repo / snap3_link / f9038b6ff0d64f36fff4d345803dfacd / vol1

  5. 将新快照添加到新成员集群。

    ＃/ sbin / advmutil export create -f -n vol1 -p // repo / -c mc2

advmutil出口清单

目的

显示有关导出和成员集群的信息。

语法和说明

advmutil export list -h advmutil export list [-c cluster_name ] [-o options ] [-g group_type ] [-a] [-n]

下表包含advmutil export list命令可用的选项。

表16-112 advmutil export list命令的选项

选项 描述

-c cluster_name

指定成员集群的名称。

-o options

指定选项列表，逗号分隔，不带空格。选项包括clientDev ， clusterGUID ， clusterName ， storageRepo ， exportName ， filePath ， state ， exportSize ， active ， domainNode和minor 。

-g group_type

按clusterName ， clusterGUID或storageRepo指定组类型。

-一个

仅显示活动导出。

指定不打印列标题。

advmutil export list是一个强大的导出管理查询命令。它包含一个显示基本信息的默认视图，但也可用于按不同的组排序，以及在不同的群集上显示不同的信息。

默认情况下， advmutil export list按群集名称排序（ clusterName组排序选项）。导出也可以按存储库（ storageRepo ）或群集GUID（ clusterGUID ）进行排序。

导出的默认视图应使用适当的标头和信息队列轻松读取。您可以使用-n选项在没有标题的单行上显示所有输出，以便可以在脚本中使用该信息。

默认视图包括clientDev选项，该选项显示成员集群系统上设备的路径名。可以向成员集群管理员提供此默认视图，以描述管理员管理其系统所需的信息。

此命令仅适用于域服务群集（DSC）。

advmutil export resize

目的

将存储添加到底层后备文件以进行导出。

语法和说明

advmutil export resize -h advmutil export resize -c cluster_name -p storage_repos -n export_name -s file_size

下表包含advmutil export resize命令可用的选项。

表16-116 advmutil export resize命令的选项

选项 描述

-c cluster_name

指定成员群集名称。

-p storage_repos

指定存储后备文件的存储库的位置。

-n export_name

指定导出名称。

-s file_size

以n {K | M | G | T | P | E}的格式指定文件大小。单位是K（千字节），M（兆字节），G（千兆字节），T（太字节），P（千兆字节）或E（艾字节）。

advmutil export resize将存储添加到底层后备文件以进行导出。支持文件增长后，成员群集上的设备也会调整大小。要使用新大小，管理员必须在使用新大小设备的成员集群装入文件系统上运行acfsutil size 。（可选）如果在与此设备关联的已装载成员集群Oracle ACFS文件系统上配置了Oracle ACFS自动调整大小功能，则在需要时使用新存储。

要收缩设备，必须在成员集群上运行acfsutil size以减小文件系统的大小。此操作还会缩小关联的后备文件的大小。

您可以在成员集群上运行advmutil volinfo以显示基础卷的大小。但是，这可能与使用此设备的当前Oracle ACFS文件系统的大小不同。

此命令仅适用于域服务群集（DSC）。

advmutil运输创造

目的

在域服务群集（DSC）上创建传输。

语法和说明

advmutil transport create -h advmutil transport create -c cluster_name [-b binding [-n network_num ] | [-i resource_id ] [-f]

下表包含advmutil transport create命令可用的选项。

表16-118 advmutil transport create命令的选项

选项 描述

-c cluster_name

指定传输所属的成员群集名称。

-b binding

指定传输VIP，主机名或网络地址。

-n network_num

指定指定VIP的网络号。默认值为1 。

-i resource_id

指定现有备用资源标识符。

-F

强制接受指定的现有绑定。

advmutil transport create在域服务集群（DSC）上创建传输。

可以使用crsctl query member_cluster_configuration命令发现群集名称。传输VIP可以是IPv4或IPv6地址，并且可以是可解析的DNS条目或IP地址。

创建启用了其他功能的传输（例如多路径和故障转移）时，必须为单个成员群集启用多个传输。您可以通过多次运行advmutil transport create命令来创建多个传输，并指定其他VIP地址。

成员集群可以共享VIP。您可以通过在为新成员集群注册传输时指定公共VIP名称来完成此操作。但是，所有成员集群必须至少运行一次advmutil transport create命令才能指定其传输。

使用格式为cluster_name_transport_type_sequence＃的标识符为每个传输创建传输ID，其中n是从0开始的数字，并且对于每个传输增加1。

此命令仅适用于域服务群集。

例16-114使用advmutil transport创建

在以下示例中，使用网络2为VIP havip1为成员集群cluster1创建传输。 

＃/ sbin / advmutil transport create -c cluster1 -b havip1 -n 2
