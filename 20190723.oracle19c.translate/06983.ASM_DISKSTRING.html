1.18 ASM_DISKSTRING

ASM_DISKSTRING指定Oracle ASM用于限制考虑进行发现的磁盘集的操作系统相关值。

属性 描述

参数类型

串

句法

ASM_DISKSTRING = discovery_string [, discovery_string ] ...

默认值

空字符串; Oracle自动存储管理发现查找Oracle自动存储管理实例具有读/写访问权限的操作系统特定位置中的所有磁盘。

可修改

ALTER SESSION，ALTER SYSTEM

可在PDB中修改

是

Oracle RAC

多个实例可以具有不同的值。不同的节点可能会以不同的名称看到相同的磁盘;但是，每个实例必须能够使用其ASM_DISKSTRING来发现与群集中其他节点相同的物理介质。

注意：

此参数只能在Oracle自动存储管理（Oracle ASM）实例中指定。

将新磁盘添加到磁盘组时，安装了磁盘组的每个Oracle ASM实例都必须能够使用ASM_DISKSTRING的值发现新磁盘。

在大多数情况下，默认值就足够了。使用限制性更强的值可以减少Oracle ASM执行发现所需的时间，从而缩短磁盘组安装时间或将磁盘添加到磁盘组的时间。一个 ”？“在字符串的开头扩展到Oracle主目录。根据操作系统，可以使用通配符。在添加磁盘之前可能需要动态更改ASM_DISKSTRING ，以便发现新磁盘。

如果新值不能用于发现已装入的磁盘组中的磁盘，则将拒绝尝试动态修改ASM_DISKSTRING并保留旧值。
