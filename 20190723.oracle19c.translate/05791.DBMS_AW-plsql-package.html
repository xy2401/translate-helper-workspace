B.1管理分析工作区

要与Oracle OLAP交互，必须将分析工作区附加到会话。安装DBMS_AW PL / SQL包后，可以在SQL * Plus中执行此任务。例如，您可以使用以下命令附加具有只读访问权限的分析工作区。

EXECUTE dbms_aw.aw_attach（' awname '）;

每个分析工作区都与分析工作区列表相关联。包含OLAP引擎代码的只读工作区EXPRESS.AW始终附加在列表的最后。创建工作区时，默认情况下它首先附加在列表中。

您可以使用FIRST和LAST等关键字在列表中重新定位分析工作区。例如，以下命令显示如何将名为MYAW.TEST2的分析工作区从第二个位置移动到列表中的第一个位置。 

EXECUTE dbms_aw.execute（'AW LIST'）; TEST1 R / O UNCHANGED GLOBAL.TEST1 TEST2 R / O UNCHANGED GLOBAL.TEST2 EXPRESS R / O UNCHANGED SYS.EXPRESS EXECUTE dbms_aw.aw_attach（'test2'，FALSE，FALSE，'FIRST'）; EXECUTE dbms_aw.execute（'AW LIST'）; TEST2 R / O UNCHANGED GLOBAL.TEST2 TEST1 R / O UNCHANGED GLOBAL.TEST1 EXPRESS R / O UNCHANGED SYS.EXPRESS

在SQL * Plus中，您可以重命名工作空间并复制工作空间。如果您具有附带读/写访问权限的分析工作区，则可以更新工作区并将更改保存在存储工作空间的永久数据库表中。您必须执行SQL COMMIT以在数据库中保存工作区更改。

以下命令在名为test3 ，update test3的新工作空间中复制工作空间test2中的对象和数据，并将更改提交到数据库。 

EXECUTE dbms_aw.aw_copy（'test2'，'test3'）; EXECUTE dbms_aw.aw_update（'test3'）; 承诺;

B.2在SQL语句中嵌入OLAP DML

使用DBMS_AW包，您可以在分析工作区内执行所有OLAP处理。您可以从旧工作区，关系表或平面文件导入数据。您可以定义OLAP对象并执行复杂计算。

注意：

如果使用DBMS_AW程序包从头开始创建分析工作区，则可能无法使用OLAP实用程序，例如Analytic Workspace Manager和DBMS_AW Aggregate Advisor，它们需要特定结构的分析工作区。

B.2.1执行OLAP DML命令的方法

DBMS_AW包提供了执行ad hoc OLAP DML命令的几个过程。使用EXECUTE或INTERP_SILENT程序或INTERP或INTERCLOB函数，可以执行单个OLAP DML命令或一系列由分号分隔的命令。

您使用哪些过程取决于您希望如何直接输出以及输入和输出缓冲区的大小。例如， EXECUTE程序指导输出到打印机缓冲器，所述INTERP_SILENT过程抑制输出，并且INTERP函数返回会话日志。

DBMS_AW包还提供用于评估OLAP表达式的函数。EVAL_TEXT函数返回文本表达式的结果， EVAL_NUMBER返回数值表达式的结果。

注意：

不要将DBMS_AW函数EVAL_NUMBER和EVAL_TEXT与SQL函数OLAP_EXPRESSION 。有关更多信息，请参阅 “ OLAP_EXPRESSION ” 。

B.2.2在OLAP DML命令中使用引号的指南

SQL处理器在将它们发送到Oracle OLAP进行处理之前，全部或部分地评估嵌入式OLAP DML命令。在DBMS_AW过程的olap-commands参数中格式化OLAP DML命令时，请遵循以下准则：

  • 无论你通常会使用一个单引号（ ' ）在OLAP DML命令，使用两个单引号（ '' ）。SQL处理器在将OLAP DML命令发送到Oracle OLAP之前删除其中一个单引号。

  • 在OLAP DML中，双引号（ " ）表示注释的开头。

B.3使用Sparsity Advisor

数据可以在分析工作区中以多种不同的形式存储，具体取决于它是密集的，稀疏的还是非常稀疏的。稀疏Advisor是一组子程序DBMS_AW ，你可以用它来分析关系源数据并将其存储在分析工作区得到的建议。

B.3.1分析工作区中的数据存储选项

分析工作区以多维格式分析和操作数据，为每个维度成员组合分配一个单元格。单元格可以包含数据值，也可以包含NA （null）。无论其内容如何，单元格大小都由数据类型定义，例如， DECIMAL变量中的每个单元格都是8个字节。

变量可以是密集的（它们包含30％或更多具有数据值的单元格）或稀疏（小于30％的数据值）。大多数变量稀疏，许多变量非常稀疏。

虽然数据也可以存储在用于分析的多维格式中，但其他方法可用于存储稀疏变量，从而更有效地利用磁盘空间并提高性能。稀疏数据可以存储在使用复合维度定义的变量中。复合具有作为其成员的维度 - 值组合（称为元组 ），其中存在数据。将数据值添加到由复合体标注的变量时，该操作会触发复合元组的创建。复合是一个或多个稀疏数据变量的索引，用于以紧凑的形式存储稀疏数据。非常稀疏的数据可以存储在使用压缩复合定义的变量中，该复合使用不同的算法来存储来自常规复合的数据。

B.3.2选择最佳数据存储方法

与维度数据相比，关系数据以非常紧凑的格式存储在表中，行仅用于实际数据值。在设计分析工作区时，您可能难以手动识别源数据中的稀疏性并确定最佳存储方法。Sparsity Advisor分析关系表中的源数据并推荐存储方法。建议可以包括复合的定义和数据变量的分区。

Sparsity Advisor包含以下程序和功能：

  • SPARSITY_ADVICE_TABLE程序
  • ADD_DIMENSION_SOURCE过程
  • ADVISE_SPARSITY程序
  • ADVISE_DIMENSIONALITY功能
  • ADVISE_DIMENSIONALITY程序

Sparsity Advisor还提供公共表类型，用于存储有关所分析事实的维度的信息。三个对象用于定义表类型：

  • DBMS_AW $ _COLUMNLIST_T

B.3.3使用Sparsity Advisor

按照以下步骤使用Sparsity Advisor：

  1. 调用SPARSITY_ADVICE_TABLE创建一个表，用于存储Sparsity Advisor的评估。
  2. 将由一个或多个列相关的每个维度调用ADD_DIMENSION_SOURCE到正在评估的事实表。

    您提供的有关这些维度的信息存储在DBMS_AW$_DIMENSION_SOURCES_T变量中。

  3. 调用ADVISE_SPARSITY来评估事实表。

    其建议存储在SPARSITY_ADVICE_TABLE创建的表中。您可以使用这些建议来自行判断在分析工作区中定义变量，也可以继续执行以下步骤。

  4. 调用ADVISE_DIMENSIONALITY过程以获取推荐的复合，分区和变量定义的OLAP DML对象定义。

    要么

    使用ADVISE_DIMENSIONALITY函数获取建议复合的OLAP DML对象定义以及特定分区的变量定义的维度顺序。

B.4使用Aggregate Advisor

该管理分析工作区内的聚合数据可能会对性能产生重大影响。要确定要预聚合的最佳维成员组合集，可以使用DBMS_AW包中的ADVISE_REL和ADVISE_CUBE过程。这些过程一起称为Aggregate Advisor。

根据您指定的百分比， ADVISE_REL建议一组要预聚合的维成员。ADVISE_CUBE过程为多维数据集的每个维度建议一组成员。

B.4.1工作区内的聚合设施

存储聚合数据的说明在称为aggmap的分析工作空间对象中指定。OLAP DML AGGREGATE命令使用aggmap预聚合数据。在查询数据时， AGGREGATE函数会动态聚合任何未预聚合的数据。

在静态和动态聚合之间选择平衡取决于许多因素，包括磁盘空间，可用内存以及针对数据运行的查询的性质和频率。在权衡这些因素后，您可以获得一定比例的数据进行预聚合。

确定要预聚合的数据百分比后，可以使用Aggregate Advisor。这些过程分析层次结构中维度成员的分布，并标识要预先聚合的最佳维度成员集。
