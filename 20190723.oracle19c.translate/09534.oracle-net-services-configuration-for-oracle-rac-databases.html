Oracle RAC数据库的Oracle Net Services配置

用户可以使用客户端和服务器配置访问Oracle RAC数据库，也可以通过一个或多个中间层访问Oracle RAC数据库，无论是否有连接池。

连接到Oracle数据库时，可以使用连接描述符或网络服务名称。对于Oracle RAC数据库，还可以使用单客户端访问名称（SCAN）连接到Oracle RAC数据库的任何可用实例。

  • Oracle RAC数据库的数据库服务
    每个数据库由一个或多个服务表示。服务由服务名称标识，例如sales.example.com 。客户端使用服务名称来标识它必须访问的数据库。
  • 命名方法和连接描述符
    每个网络服务名称都与连接描述符相关联。连接描述符提供数据库的位置和数据库服务的名称。
  • 轻松连接命名方法
    Easy Connect命名方法无需在tnsnames.ora文件或TCP / IP环境的其他存储库中查找服务名称。
  • 了解SCAN
    SCAN是在域名服务（DNS）或网格命名服务（GNS）中注册到至少一个和最多三个IP地址的域名。
  • 关于使用SCAN连接到Oracle RAC数据库
    Oracle建议您配置Oracle RAC数据库客户端以使用SCAN连接到数据库，而不是配置tnsnames.ora文件。
  • 关于Oracle RAC数据库的侦听器配置
    Oracle数据库通过本地侦听器接收连接请求。
  • 关于Oracle RAC数据库的服务注册
    Oracle Database 19c数据库服务自动向数据库初始化参数LOCAL_LISTENER和REMOTE_LISTENER指定的侦听器注册。
