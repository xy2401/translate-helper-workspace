184.3.28 MODIFY_SNAPSHOT_SETTINGS程序

此过程控制快照生成的三个方面。

  • INTERVAL设置会影响自动捕获快照的频率。

  • RETENTION设置会影响工作负载存储库中保留快照的时间。

  • 为每个Top条件捕获的SQL数。如果用户手动为Top N SQL指定值，则AWR SQL集合将使用用户指定的数字进行自动和手动快照。

有两个重载。第一个采用NUMBER ，第二个采用VARCHAR2作为topnsql参数。差异在参数描述下描述。

句法

DBMS_WORKLOAD_REPOSITORY.MODIFY_SNAPSHOT_SETTINGS（保留在NUMBER DEFAULT NULL，间隔IN NUMBER DEFAULT NULL，topnsql IN NUMBER DEFAULT NULL，dbid IN NUMBER DEFAULT NULL）; DBMS_WORKLOAD_REPOSITORY.MODIFY_SNAPSHOT_SETTINGS（保留在NUMBER DEFAULT NULL，间隔IN NUMBER DEFAULT NULL，topnsql IN VARCHAR2，dbid IN NUMBER DEFAULT NULL）;

参数

表184-33 MODIFY_SNAPSHOT_SETTINGS过程参数

参数 描述

保留

新保留时间（以分钟为单位）。指定的值必须在MIN_RETENTION （1天）到MAX_RETENTION （100年）的范围内。

如果指定了ZERO ，则将永久保留快照。系统定义的大值将用作保留设置。

如果指定NULL ，则保留旧的保留值。

注意：保留设置必须大于或等于“ SYSTEM_MOVING_WINDOW ”基线的窗口大小。如果保留需要小于窗口大小，则可以使用MODIFY_BASELINE_WINDOW_SIZE过程调整窗口大小。

间隔

每个快照之间的新间隔设置，以分钟为单位。指定的值必须在MIN_INTERVAL （10分钟）到MAX_INTERVAL （1年）的范围内。

如果指定了ZERO ，则将禁用自动和手动快照。系统定义的大值将用作保留设置。

如果指定NULL ，则保留当前值。

topnsql

  • 如果NUMBER ：前N个SQL大小。每个SQL条件要刷新的Top SQL数（经过时间，CPU时间，解析调用，可共享内存，版本计数）。此设置的值不受统计信息/刷新级别的影响，并将覆盖AWR SQL集合的系统默认行为。该设置的最小值为30，最大值为50,000。指定NULL将保持当前设置。

  • 如果允许VARCHAR2 ：Users指定以下值：（ DEFAULT ， MAXIMUM ， N ），其中N是要为每个SQL条件刷新的Top SQL的数量。指定DEFAULT会将统计级别TYPICAL和Top 100的统计级别ALL恢复为Top 30的默认行为。指定MAXIMUM将使系统捕获游标缓存中的完整SQL集。指定数字N相当于使用NUMBER类型设置Top N SQL。为此参数指定NULL将保持当前设置。

DBID

AWR中的数据库标识符，用于修改快照设置。如果指定NULL ，则将使用本地dbid。默认为NULL 。

例子

此示例将interval设置更改为一小时，并将本地数据库的retention设置更改为两周：
