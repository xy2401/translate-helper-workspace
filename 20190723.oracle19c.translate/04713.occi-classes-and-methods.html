13.1.1使用OCCI类

OCCI类在oracle::occi命名空间中定义。oracle::occi命名空间中的OCCI类名称可以通过以下三种方式之一引用：

  • 对每个OCCI类名使用范围解析运算符（ :: :)。

  • 对每个OCCI类名使用using声明。

  • 对所有OCCI类名使用using指令。

使用OCCI的范围解析运算符

范围解析运算符（ :: ）来显式地指定oracle::occi命名空间和OCCI类名。要使用范围解析运算符声明myConnection ，一个Connection对象，您将使用以下语法：

oracle :: occi :: Connection myConnection;

在OCCI中使用声明

当OCCI类名可以在编译单元中使用而没有冲突时， using声明。要在oracle::occi命名空间中声明OCCI类名，可以使用以下语法：

使用oracle :: occi :: Connection;

Connection现在引用oracle::occi::Connection ， myConnection可以声明为Connection myConnection; 。

在OCCI中使用指令

当所有OCCI类名都可以在编译单元中使用而没有冲突时， using指令。要在oracle::occi命名空间中声明所有OCCI类名，可以使用以下语法：

使用oracle :: occi;

然后，就像using声明一样，以下声明现在将引用OCCI类Connection作为Connection myConnection; 。

在OCCI中使用高级队列

高级队列类Producer ， Consumer ， Message ， Agent ， Listener ， Subscription和NotifyResult在oracle::occi::aq命名空间中定义。

13.1.2 OCCI对Windows NT和z / OS的支持

在Windows上构建OCCI应用程序时，必须添加WIN32COMMON的预处理器定义。

以下全局方法用于访问Windows NT和z / OS上的ResultSet类和Statement类中的Ref的集合。方法名称更改时，参数数量及其类型保持不变。

  • 在Windows NT和z / OS上使用getVectorOfRefs()而不是getVector() 。

  • 在Windows NT和z / OS上使用setVectorOfRefs()而不是setVector() 。

Windows NT和z / OS上的应用程序应仅调用这些新方法来检索和插入引用集合。未在Windows NT或z / OS上运行的应用程序可以使用任一组访问器。但是，Oracle建议对Ref的任何向量操作使用新方法。

本节包括以下主题： 使用参考集合 。

13.1.2.1使用参考集合

可以使用以下类的方法获取和插入Refs的集合：

13.1.2.1.1 ResultSet类

获取Refs的集合

使用以下版本的getVectorOfRefs（）返回一列引用：

void getVectorOfRefs（ResultSet * rs，unsigned int index，vector <Ref <T >>＆vect）;
13.1.2.1.2陈述类

获取Refs的集合

使用getVectorOfRefs（）从列返回引用的集合：

void getVectorOfRefs（Statement * stmt，unsigned int index，vector <Ref <T >>＆vect）;

插入参考集合

使用setVectorOfRefs（）将引用集合插入到列中：

template <class T> void setVectorOfRefs（Statement * stmt，unsigned int paramIndex，const vector <Ref <T >>＆vect，const string＆sqltype）;

插入Refs集合：多字节支持

多字节支持需要以下方法：

void setVectorOfRefs（Statement * stmt，unsigned int paramIndex，const vector <Ref <T>>＆vect，const string＆schemaName，const string＆typeName）;

插入Refs集合：UString（UTF16）支持

UString支持需要以下方法：

template <class T> void setVectorOfRefs（Statement * stmt，unsigned int paramIndex，const vector <Ref <T >>＆vect，const UString＆schemaName，const UString＆typeName）;
