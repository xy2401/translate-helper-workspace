31.2 OCI语言环境功能

列出并描述OCI语言环境功能。

表31-2列出了本节中描述的OCI语言环境功能。

Oracle语言环境由语言，区域和字符集定义组成。区域设置确定约会，例如日期和月份名称，日期，时间，数字和货币格式。全球化应用程序遵循用户的语言环境设置和文化习俗。例如，当语言环境设置为德语时，用户希望以德语显示日期和月份名称。

功能 目的

返回指定Oracle数据库字符集名称的Oracle数据库字符集ID

从NLS_LANG返回字符集ID或从NLS_NCHAR返回国家字符集ID

31.2.1 OCINlsCharSetIdToName（）

从指定的字符集ID返回Oracle数据库字符集名称。

目的

从指定的字符集ID返回Oracle数据库字符集名称。

句法

剑OCINlsCharSetIdToName（void * hndl，OraText * buf，size_t buflen ub2 id）;

参数

hndl（IN / OUT）

OCI环境或用户会话句柄。如果句柄无效，则该函数返回OCI_INVALID_HANDLE 。

buf（OUT）

指向目标缓冲区。如果函数返回OCI_SUCCESS ，则该参数包含字符集名称的NULL终止字符串。

buflen（IN）

目标缓冲区的大小。建议的大小为OCI_NLS_MAXBUFSZ以保证存储Oracle数据库字符集名称。如果目标缓冲区的大小小于字符集名称的长度，则该函数返回OCI_ERROR 。

id（IN）

Oracle数据库字符集ID。

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。

31.2.2 OCINlsCharSetNameToId（）

返回指定Oracle数据库字符集名称的Oracle数据库字符集ID。

目的

返回指定Oracle数据库字符集名称的Oracle数据库字符集ID。

句法

ub2 OCINlsCharSetNameToId（void * hndl，const OraText * name）;

参数

hndl（IN / OUT）

OCI环境或用户会话句柄。如果句柄无效，则函数返回零。

名称（IN）

指向NULL终止的Oracle数据库字符集名称的指针。如果字符集名称无效，则该函数返回零。

返回

如果指定的字符集名称和OCI句柄有效，则为字符集ID。否则，它返回0 。

31.2.3 OCINlsEnvironmentVariableGet（）

从NLS_LANG返回字符集ID或从NLS_NCHAR返回国家字符集ID。

目的

从NLS_LANG返回字符集ID或从NLS_NCHAR返回国家字符集ID。

句法

sword OCINlsEnvironmentVariableGet（void * val，size_t size，ub2 item，ub2 charset，size_t * rsize）;

参数

val（IN / OUT）

返回全球化支持环境变量的值，例如NLS_LANG字符集ID或NLS_NCHAR字符集ID。

尺寸（IN）

指定给定输出值的大小，该值仅适用于字符串数据。每条信息的最大长度是 OCI_NLS_MAXBUFSZ个字节。对于数字数据，将忽略此参数。

项目（IN）
指定从全球化支持环境变量获取的其中一个值：

  • OCI_NLS_CHARSET_ID ： ub2数据类型中的NLS_LANG字符集ID;如果未设置NLS_LANG ，则返回默认字符集ID（ASCII）。

  • OCI_NLS_NCHARSET_ID ： ub2数据类型中的NLS_NCHAR字符集ID;如果未设置NLS_NCHAR ，则返回OCI_NLS_CHARSET_ID的值。

字符集（IN）

指定检索到的字符串数据的字符集ID。如果为0 ，则使用NLS_LANG值。OCI_UTF16ID是此参数的有效值。对于数字数据，将忽略此参数。

rsize（OUT）

返回值的长度，以字节为单位。

评论

为了允许该功能的未来增强（检索来自环境变量的其他值）的输出的数据类型val是一个指针void 。字符串数据不会以NULL结尾。

请注意，该函数不接受环境句柄，因此它返回的字符集ID和国家字符集ID是NLS_LANG和NLS_NCHAR指定的值，而不是OCI环境句柄中保存的值。要获取OCI环境句柄使用的字符集ID，请分别为OCI_ATTR_ENV_CHARSET_ID和OCI_ATTR_ENV_NCHARSET_ID调用OCIAttrGet() 。

返回

OCI_SUCCESS ;或OCI_ERROR 。

31.2.4 OCINlsGetInfo（）

将区域设置信息从OCI环境或用户会话句柄复制到指定大小的目标缓冲区指向的数组中。

目的

从OCI环境或用户会话句柄获取指定大小的目标缓冲区指向的数组的区域设置信息。

句法

剑OCINlsGetInfo（void * hndl，OCIError * errhp，OraText * buf，size_t buflen，ub2 item）;

参数

hndl（IN / OUT）

在对象模式下初始化的OCI环境或用户会话句柄。

errhp（输入/输出）

OCI错误句柄。如果有错误，则将其记录在errhp ，并且该函数返回NULL指针。可以通过调用OCIErrorGet()获取诊断信息。

buf（OUT）

指向目标缓冲区的指针。返回的字符串由NULL字符终止。

buflen（IN）

目标缓冲区的大小。每条信息的最大长度为OCI_NLS_MAXBUFSZ字节。

OCI_NLS_MAXBUFSIZE ：调用OCINlsGetInfo （）时，必须分配缓冲区以存储返回的信息。缓冲区大小取决于您要查询的项目以及用于存储信息的编码。开发人员不需要知道使用JA16SJIS编码将January存储在日语中需要多少字节。OCI_NLS_MAXBUFSZ属性保证缓冲区足以容纳OCINlsGetInfo （）返回的最大项。

31.2.5 OCINlsNumericInfoGet（）

将数字语言信息从OCI环境或用户会话句柄复制到输出编号变量。

目的

从OCI环境或用户会话句柄获取数字语言信息，并将其放入输出数字变量中。

句法

剑OCINlsNumericInfoGet（void * hndl，OCIError * errhp，sb4 * val，ub2 item）;

参数

hndl（IN / OUT）

OCI环境或用户会话句柄。如果句柄无效，则返回OCI_INVALID_HANDLE 。

errhp（输入/输出）

OCI错误句柄。如果有错误，则将其记录在errhp ，并且该函数返回NULL指针。可以通过调用OCIErrorGet()获取诊断信息。

val（OUT）

指向输出数字变量的指针。如果函数返回OCI_SUCCESS ，则该参数包含请求的全球化支持数字信息。

项目（IN）

它指定从OCI环境句柄获取的项目，并且可以是以下值之一：

  • OCI_NLS_CHARSET_MAXBYTESZ ：OCI环境或会话句柄字符集的最大字符字节大小

  • OCI_NLS_CHARSET_FIXEDWIDTH ：固定宽度字符集的字符字节大小; 0表示可变宽度字符集

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。
