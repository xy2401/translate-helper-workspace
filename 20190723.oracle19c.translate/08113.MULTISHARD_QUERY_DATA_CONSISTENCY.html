1.198 MULTISHARD_QUERY_DATA_CONSISTENCY

MULTISHARD_QUERY_DATA_CONSISTENCY允许您为MULTISHARD_QUERY_DATA_CONSISTENCY查询指定数据一致性设置。

属性 描述

参数类型

串

句法

MULTISHARD_QUERY_DATA_CONSISTENCY = {STRONG | SHARD_LOCAL | DELAYED_STANDBY_ALLOWED}

默认值

强大

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

Oracle RAC

在不同的Oracle RAC实例上，此参数的值可能不同。

您可以使用MULTISHARD_QUERY_DATA_CONSISTENCY来避免在跨分片执行MULTISHARD_QUERY_DATA_CONSISTENCY片查询时SCN同步的成本，这些分片可以是全局分布的。

可以在系统级别或会话级别设置MULTISHARD_QUERY_DATA_CONSISTENCY 。

可以为MULTISHARD_QUERY_DATA_CONSISTENCY设置的MULTISHARD_QUERY_DATA_CONSISTENCY是：

  • STRONG ：使用此设置，跨所有分片执行SCN同步，并且数据在所有分片中保持一致。此设置提供全局一致的读取功能。这是默认值。

  • SHARD_LOCAL ：使用此设置，不会跨所有分片执行SCN同步。每个分片中的数据都是一致的。此设置提供最新数据。

  • DELAYED_STANDBY_ALLOWED ：使用此设置，不会跨所有分片执行SCN同步。每个分片中的数据都是一致的。此设置允许在可能的情况下从Data Guard备用数据库中提取数据（例如，取决于负载平衡），并且可以从备用数据库返回过时数据。
