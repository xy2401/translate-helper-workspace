1.1连接池概述

连接池是数据库连接对象的缓存。对象表示应用程序可用于连接到数据库的物理数据库连接。在运行时，应用程序从池请求连接。如果池包含可以满足请求的连接，则它将返回与应用程序的连接。如果未找到任何连接，则会创建新连接并将其返回给应用程序。应用程序使用连接对数据库执行某些操作，然后将对象返回池。然后，该连接可用于下一个连接请求。

连接池可以促进连接对象的重用，并减少创建连接对象的次数。连接池显着提高了数据库密集型应用程序的性能，因为创建连接对象在时间和资源方面都很昂贵。网络通信，读取连接字符串，身份验证，事务登记和内存分配等任务都会影响创建连接对象所需的时间和资源。此外，由于已创建连接，因此应用程序等待较少的时间来获取连接。

连接池通常提供用于优化池性能的属性。属性控制行为，例如池中允许的最小和最大连接数，或者连接在返回池之前保持空闲的时间。配置最佳的连接池可以平衡快速响应时间与维护池中连接所花费的内存。通常需要尝试不同的设置，直到达到特定应用的最佳平衡。

1.2使用连接池的好处

数据库密集型应用程序通常从连接池中获益最多。作为策略，只要知道数据库使用会影响应用程序性能，应用程序就应该使用连接池。

连接池具有以下优点：

  • 减少创建新连接对象的次数。

  • 促进连接对象重用。

  • 加快获取连接的过程。

  • 减少手动管理连接对象所需的工作量。

  • 最大限度地减少过时连接的数量。

  • 控制维护连接所花费的资源量。

1.3通用连接池概述

UCP提供用于缓存JDBC连接的连接池实现。数据库密集型的Java应用程序使用连接池来提高性能并更好地利用系统资源。

UCP JDBC连接池可以使用任何JDBC驱动程序来创建随后由池维护的物理连接。可以配置池并提供一整套属性，这些属性用于根据应用程序的性能和可用性要求优化池行为。对于更高级的应用程序，UCP提供了一个可用于管理池实例的池管理器。

该池还利用了Oracle提供的许多高可用性和性能功能 Real Application Clusters（Oracle RAC）数据库。这些功能包括快速连接故障转移（FCF），运行时连接负载平衡（RLB）和连接关联。

注意：

从Oracle Database 11g第2版开始，单个实例数据库上的Oracle Restart也支持FCF。Oracle Restart也称为独立服务器的Oracle Grid Infrastructure。

1.3.1概念架构

应用程序使用启用UCP池的数据源从UCP JDBC连接池实例获取连接。PoolDataSource数据源用于获取常规连接（ java.sql.Connection ）和PoolXADataSource数据源用于获取XA（eXtended API）连接（ javax.sql.XAConnection ）。XA和非XA UCP JDBC连接池中都包含相同的池功能。

启用池的数据源依赖于连接工厂类来创建池维护的物理连接。应用程序可以选择使用能够创建Connection或XAConnection对象的任何工厂类。启用池的数据源提供了一种设置连接工厂类的方法，以及设置工厂类用于连接到数据库的数据库URL和数据库凭据的方法。

应用程序从池中借用连接句柄以执行数据库的工作。完成工作后，将关闭连接并将连接句柄返回到池中，并且可以再次使用。下图显示了应用程序与UCP JDBC连接池之间交互的概念视图。

1.3.2连接池属性

UCP JDBC连接池属性是通过启用池的数据源上可用的方法配置的。池属性用于控制池大小，处理过时连接，并自动决定连接在返回池之前可以保留多长时间。池属性的最佳设置取决于应用程序和硬件资源。通常，在应用程序获取连接所花费的时间与维护特定池大小所需的内存量之间存在权衡。在许多情况下，需要进行实验以找到最佳平衡以实现特定应用的所需性能。

1.3.3连接池管理器

UCP包括连接池管理器，该管理器由需要对连接池进行管理控制的应用程序使用。管理器用于显式控制池的生命周期并对池执行维护。管理器还为应用程序提供了通过管理控制台公开池及其可管理性的机会。

1.3.4高可用性和性能方案

UCP JDBC连接池提供了许多用于确保高连接可用性和性能的功能。许多这些功能（例如刷新池或验证连接）都是通用的，可以跨驱动程序和数据库实现工作。其中一些功能（如运行时连接负载平衡和连接关联性）需要使用Oracle JDBC驱动程序和Oracle RAC数据库。

1.3.5支持用于Reactive Streams摄取的Java库

Oracle Database Release 19c引入了一个Java库，它为反应流提取提供支持，使客户能够有效地将数据流式传输到Oracle数据库。新的Java库使Java应用程序能够不断地从大量客户端接收和接收数据。使用Oracle数据库的直接路径加载方法来插入数据，它使摄取过程无阻塞且速度极快。

此Java库使用现有UCP API的扩展，这使得提取过程能够提供数据库的多个高可用性和可伸缩性功能，例如支持表分区，Oracle RAC连接关联和分片。
