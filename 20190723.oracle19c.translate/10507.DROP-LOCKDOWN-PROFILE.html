DROP LOCKDOWN PROFILE

目的

使用DROP LOCKDOWN PROFILE语句从数据库中删除PDB锁定配置文件。已分配已删除配置文件的PDB将继续分配配置文件，但不受受删除配置文件强加的限制的约束。

如果CDB，应用程序根目录或PDB的PDB_LOCKDOWN初始化参数具有已删除锁定配置文件的值，则删除该配置文件时所施加的限制将被禁用。但是， PDB_LOCKDOWN初始化参数的值将保持不变，直到您明确取消设置它。

也可以看看：

  • 创建锁定配置文件和更改锁定配置文件

  • 有关PDB锁定配置文件的更多信息，请参见“Oracle数据库安全指南

先决条件

  • 您必须从CDB Root或Application Root发出此语句。

  • 您必须在要发出语句的容器中具有DROP LOCKDOWN PROFILE系统特权。

句法

drop_lockdown_profile :: =


说明drop_lockdown_profile.eps的描述

语义

PROFILE_NAME

指定要删除的PDB锁定配置文件的名称。

您可以通过查询DBA_LOCKDOWN_PROFILES数据字典视图找到现有PDB锁定配置文件的名称。

也可以看看：

有关DBA_LOCKDOWN_PROFILES数据字典视图和PDB_LOCKDOWN初始化参数的更多信息，请DBA_LOCKDOWN_PROFILES Oracle数据库参考

例

以下语句删除了PDB锁定配置文件hr_prof ：

DROP LOCKDOWN PROFILE hr_prof;
