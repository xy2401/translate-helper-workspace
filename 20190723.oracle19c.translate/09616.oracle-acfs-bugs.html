Oracle ASM集群文件系统（Oracle ACFS）已知错误

这些是Oracle Database 19c中的Oracle自动存储管理（Oracle ASM）群集文件系统（Oracle ACFS）错误。

  • 错误29391849

父主题： 打开影响所有平台的错误

错误29391849

升级到Oracle Grid 18.x或19.x完成后，后续升级到Oracle Grid 18.y或19.y会导致Oracle自动存储管理群集文件系统（Oracle ACFS）在第一个节点上脱机正在升级。在全新安装的Oracle Grid 19c或18c上不会发生此问题。

解决方法：

  • 要避免此问题：
    1. 完成从Oracle Grid 12c第1版（12.1.0.2）或Oracle Grid 12c第2版（12.2）到Oracle Grid 18.x或19.x的升级。
    2. 在每个节点上重新启动Cluster Ready Services（CRS），最后重新启动主节点。例如：
      1. 运行acfsutil clusterinfo | grep -i master 。
      2. 以任何顺序在每个非主节点上重新启动CRS。
      3. 最后在主节点上重启CRS。
    3. 然后，从Oracle Grid 18.x或19.x升级到Oracle Grid 18.y或19.y.
  • 如果您已完成从Oracle Grid 12.1.0.2或12.2到Oracle Grid 18.x或19.x的升级，并已开始升级到Oracle Grid 18.y或19.y，并且您看到了Oracle自动存储管理动态卷管理器（Oracle ADVM）卷在已升级的节点上脱机，然后：
    1. 将该节点回滚到Oracle Grid 18.x或19.x.
    2. 在每个节点上重新启动CRS，最后重新启动主节点。
      1. 运行acfsutil clusterinfo | grep -i master 。
      2. 以任何顺序在每个非主节点上重新启动CRS。
      3. 最后在主节点上重启CRS。
    3. 然后，从Oracle Grid 18.x或19.x升级到Oracle Grid 18.y或19.y.
