6在Windows上配置SNA通信包

APPC的Oracle数据库网关使用SNA高级程序编程通信（APPC / LU6.2）协议与OLTP进行通信。

可以使用以下任一工具提供Microsoft Windows上的APPC支持：

  • Microsoft Host Integration Server 2000（HIS），或

  • IBM Communications Server v6.1.1（或更高版本）。

使用SNA安全验证

当网关收到启动远程事务程序（RTP）的远程过程调用（RPC）请求时，网关尝试启动与OLTP的APPC对话。在开始对话之前，必须在Windows逻辑单元之间启动会话 （ LU）和OLTP LU。

SNA及其各种访问方法实现（包括HIS和IBM Communication Server和VTAM）在会话启动时提供安全验证，允许每个LU对其伙伴进行身份验证。在网关和OLTP应用程序开始对话并处理对话级安全数据之前，此验证完全由网络软件执行。如果使用会话级安全性，则必须在Windows SNA Server定义中以及要访问的OLTP中的类似参数结构中建立正确的密码信息。有关此主题的详细信息，请参阅合适的通信软件产品文档。

处理入站连接

许多 OLTP提供用于操纵入站（客户端）APPC会话请求的安全行为的选项。有关此主题的详细信息，请参阅合适的OLTP文档。

请注意 CICS，网关不支持一个安全选项：

  • ATTACHSEC=PERSISTENT ，在 CICS CONNECTION定义需要网关中尚不可用的功能。

但是，网关完全支持ATTACHSEC=LOCAL ， ATTACHSEC=IDENTIFY ， ATTACHSEC=VERIFY和ATTACHSEC=MIXIDPE 。

配置Microsoft Host Integration Server

以下部分讨论有关SNA Server以及如何配置Microsoft Host Integration Server的一般信息。

注意：

如果您使用IBM Communication Server为网关配置Windows系统，请继续配置IBM Communications Server 。

注意：

虽然Microsoft Host Integration Server已替换Microsoft SNA服务器管理器，但通信包的配置保持不变。

这意味着配置Microsoft SNA服务器管理器所示的以下步骤也适用于Microsoft Host Integration Server。您将在配置期间实际看到的图形窗口将标记为“Microsoft Host Integration Server”窗口，而不是“SNA Server Manager”。

独立对抗的LU

Oracle建议为Oracle数据库网关APPC提供独立的LU，因为它们支持多个并行会话或会话。这意味着多个Oracle客户端应用程序可以通过独立LU与同一OLTP同时处于活动状态。

从属LU仅支持单个活动会话。CP（节点的控制点，在这种情况下是Windows的SNA Server）在已经活动的对话后面从网关服务器排队其他对话请求。换句话说，对话LU是单线程的。

如果正确定义了从属LU，则不需要对Oracle数据库网关进行APPC配置的更改，也不需要对主机事务或OLTP如何启动进行任何更改。

依赖LU的操作影响是第一个客户端应用程序可以通过数据库网关与OLTP发起对话。当该事务处于活动状态时（可能是几秒到几分钟到几小时，具体取决于客户端应用程序和事务的设计方式），启动与同一OLTP实例的对话的任何其他客户端应用程序在等待前一个对话后都会挂起。

如果生产应用程序在任何时候确实只使用一个会话或事务，那么应该没有任何影响。

但是，测试或其他应用程序开发可能需要额外的并发对话或事务。每个都要求在远程主机上定义其他从属LU，以及其他SNA Server配置条目，这些条目定义Windows系统上的其他从属LU。该启动对话的提示必须通过不同的侧面信息配置文件或通过覆盖LU名称来指定不同的合作伙伴LU。请参阅Oracle Database Gateway for APPC用户指南的第2章“过程网关管理实用程序”中的PGAU DEFINE TRANSACTION ， SIDEPROFILE和LUNAME参数。

示例SNA服务器定义的位置

%ORACLE_HOME%\dg4appc\sna子目录包含使用SNACFG命令创建的一组示例网关SNA Server定义。该 snacfg.ctl文件包含SNA Server的示例定义。

在构建SNA Server定义之前，请检查snacfg.ctl文件以确定所需的定义，其内容及其相互关系。文件格式是面向文本的，每个定义的每个字段都有清晰的标记。您可以打印在Microsoft Host Integration Server（以前称为SNA Server Manager）会话中使用您的定义时要使用的文件的副本。

在Microsoft Host Integration Server上创建SNA服务器定义

本节介绍在Microsoft Host Integration Server（以前称为。）中创建SNA定义的过程 SNA服务器管理器）。本节中描述的所有任务都是在Microsoft Host Integration Server中执行的。

服务器选择

选择适当的文件夹以确保为该服务器创建的定义。启动HIS Manager时，会出现一个对话框。

选择本地系统下的Servers文件夹，然后选择本地SNA Server。从该服务器的服务列表中，选择SNA服务。

链接服务定义

您必须为SNA Server安装和配置链接服务才能使用工作站中安装的网络适配器，如下所示：

  1. 从“ 插入”菜单中，选择“ 链接服务” 。
  2. 从“ 插入链接服务”对话框中，从选择列表中选择要使用的链接服务，然后单击“ 添加” 。例如，选择DLC 802.2链接服务 ，然后单击添加 。

现在，将显示“ 链接服务属性”对话框。请注意，此对话框的内容取决于所选的链接服务。在此示例中，使用DLC 802.2链接服务属性对话框。

从“适配器”下拉菜单中选择合适的网络适配器，然后单击“ 确定” 。在“ 插入链接服务”对话框中，单击“ 完成” 。系统现在更新您的网络绑定。

连接定义

要创建连接定义，请执行以下步骤：

  1. 您必须创建连接定义以定义HIS用于执行SNA通信的设备。从“ 插入”菜单中，选择“ 连接” 。将出现“ 连接属性”对话框。

  2. 选择“ 常规”选项卡。输入连接名称。这是HIS用于命名连接的名称。此示例将连接命名为TOKEN1 。从“链接服务”下拉菜单中，选择连接的链接服务。可以将所有其他设置保留为默认值。

  3. 在“ 连接属性”框中选择“ 地址”选项卡。输入远程网络地址和远程SAP地址 。

  4. 选择“ 系统标识”选项卡。在“本地节点名称”下，输入“ 网络名称” ，“ 控制点名称 ”和“ 本地节点ID” 。

  5. 在“远程节点名称”下，输入“ 网络名称” ，“ 控制点名称 ”和（可选）“ 远程节点ID” 。 XID类型应设置为格式3。

  6. 接下来，在“ 连接属性”框中选择“ DLC”选项卡。在此示例中，正在使用802.2 DLC （令牌环）。对于802.2 DLC ，所有默认值通常都是可接受的。如果您需要更改任何值，请立即执行此操作。

  7. 现在，设置了所有连接属性。单击“ 确定”继续。

本地LU定义

您必须创建本地逻辑单元（LU）定义。本地LU定义描述了网关通过其与OLTP系统通信的SNA LU。执行以下步骤以创建本地LU定义：

  1. 从“ 插入”菜单中，选择“ 本地APPC LU” 。将出现“ 本地LU属性”对话框。

  2. 选择“ 常规”选项卡。输入LU别名 ， 网络名称和LU名称 。确保未选中“ APPC SyncPoint支持”框。

  3. 选择“ 本地APPC LU属性”框的“ 高级”选项卡。检查默认传出本地APPC LU池的成员复选框。将LU 6.2类型设置为独立以允许并行会话。

  4. 现在，本地LU属性都已设置。单击“ 确定”继续。

模式定义

此定义描述了在LU之间建立会话时使用的SNA模式条目。此处定义的模式必须与目标系统上定义的模式匹配。执行以下步骤以创建模式定义：

  1. 从“ 插入”菜单中，选择“ APPC模式定义” 。将出现“ APPC模式属性”对话框。

  2. 选择“ 常规”选项卡。输入模式名称 。必须为OLTP通信软件定义您指定的模式名称。在咨询负责配置OLTP通信软件的人员之后，请选择模式名称以及其他模式参数。

  3. 接下来，选择“ 限制”选项卡。输入并行会话限制 ， 最小争用获胜者限制 ， 合作伙伴最小争用获胜者限制和自动激活限制值。

  4. 并行会话限制确定网关实例与OLTP之间允许的最大并发会话数。这相当于通过网关实例的最大并发活动RTP调用数。

  5. 现在，选择“ 特征”选项卡。输入起搏发送计数 ， 起搏接收计数 ， 最大发送RU大小和最大接收RU大小 。为获得最佳性能，请选中“ 高优先级模式”复选框。

    起搏和RU尺寸参数与性能相关，应根据您的应用进行调整。对于大多数安装，示例中设置的值就足够了。

  6. 在此框中输入所要求的值后，将设置所有APPC模式属性。单击框底部的“ 确定 ”继续。

远程LU定义

该定义描述了网关与之通信的OLTP系统的SNA LU。您必须为远程OLTP系统创建远程LU定义。确定与LU关联的链接（在示例中，它是TOKEN1 ）。执行以下步骤以创建远程LU定义：

  1. 从“ 插入”菜单中，选择“ APPC远程LU” 。将出现“ 远程APPC LU属性”对话框。

  2. 选择“ 常规”选项卡。使用“连接”下拉菜单选择用于访问此LU的连接。输入LU别名 ， 网络名称 ， LU名称和未解释的LU名称 。

    您应该联系负责SNA网络的人员，以确定正确的LU和网络名称。请注意，您可以使用LU别名来定义只有SNA Server知道的名称，即使远程LU名称更改，该名称也可以保持不变。这有助于减少网络更改发生时所需的维护量。

  3. 现在，选择选项选项卡。选中Supportorts Parallel Sessions复选框。使用隐式传入模式下拉菜单选择模式。设置所需的任何安全选项。

  4. 在“ 远程APPC LU属性”框中填写合适的值后，将设置APPC LU属性。单击“ 确定”继续。

CPI-C符号目的地名称

创建本地和远程伙伴定义和模式定义后，您可以创建CPI-C符号目标名称，也称为附带信息。辅助信息用于识别目标要通过网关访问的OLTP系统。

  1. 从“ 插入”菜单中，选择“ APPC CPIC符号名称” 。将出现“ CPIC名称属性”对话框。

  2. 选择“ 常规”选项卡。输入辅助信息的名称 。从“模式名称”下拉列表中，选择合适的模式。

  3. 现在，选择合作伙伴信息选项卡。选择Application TP并输入TP名称。如果您计划为OLTP访问多个事务程序定义一个CPI-C符号目标名称，则此时可以输入虚拟TP名称。在运行时，网关会覆盖TP名称。

    输入合作伙伴LU名称别名 。

  4. 单击“ 确定”以保存辅助信息。

Microsoft Host Integration Server的配置现已完成。继续“ 测试连接 ” 。
