使用新数据库安装为独立服务器安装Oracle Grid Infrastructure

完成以下步骤以为独立服务器安装Oracle Grid Infrastructure，然后创建由Oracle Restart管理的数据库。

为独立服务器安装Oracle Grid Infrastructure，该服务器安装Oracle Restart和Oracle ASM，并创建一个磁盘组。

在开始安装之前，您应该可以使用网络信息，存储信息以及操作系统用户和组。您还应该准备好运行根脚本或提供自动化根脚本的信息。

  1. 以Oracle Restart软件所有者用户（ oracle ）身份登录。
  2. 下载Oracle Grid Infrastructure以获取独立服务器安装映像文件，创建网格主目录，并在此网格主目录中提取映像文件。

    例如：

    $ mkdir -p /u01/app/oracle/product/19.0.0/grid $ chgrp oinstall /u01/app/oracle/product/19.0.0/grid $ cd /u01/app/oracle/product/19.0.0/ grid $ unzip -q /tmp/grid_home.zip

    注意：

    确保您创建的Grid主目录路径符合Oracle Optimal Flexible Architecture建议。此外，仅在您创建的此Grid主目录中解压缩安装映像文件。
  3. 配置和配置共享磁盘以与Oracle ASM筛选器驱动程序（Oracle ASMFD）一起使用。
    1. 以root身份登录并将$ ORACLE_HOME设置为网格主页位置，并将$ ORACLE_BASE设置为临时位置。
      su root #set ORACLE_HOME = / u01 / app / oracle / product / 19.0.0 / grid #set ORACLE_BASE = / tmp

      您将$ ORACLE_BASE设置为临时位置，以避免在安装Oracle Grid Infrastructure之前在Grid主目录中创建诊断或跟踪文件。

    2. 使用ASMCMD afd_label命令配置磁盘设备以与Oracle ASM过滤器驱动程序一起使用。例如：
      #cd /u01/app/oracle/product/19.0.0/grid/bin#./asmcmd afd_label DATA1 / dev / rdsk / cXtYdZsA --init＃。/ asmcmd afd_label DATA2 / dev / rdsk / cXtYdZsB --init＃。 / asmcmd afd_label DATA3 / dev / rdsk / cXtYdZsC --init
    3. 使用ASMCMD afd_lslbl命令验证设备是否已标记为与Oracle ASMFD一起使用。例如：
      ＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsA＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsB＃。/ asmcmd afd_lslbl / dev / rdsk / cXtYdZsC
    4. 取消设置$ ORACLE_BASE。
      #unset ORACLE_BASE
  4. 以Oracle Restart软件所有者用户amd运行gridSetup.sh以启动Oracle Grid Infrastructure安装向导：
    $ Grid_home /gridSetup.sh

    Grid_home是Oracle Grid Infrastructure主目录。

    注意：

    您可以将gridSetup.sh命令与-applyRU和-applyOneOffs选项一起使用，以在Oracle Grid Infrastructure安装或升级期间安装版本更新（RU）和一次性修补程序。
  5. 在Select Configuration Option屏幕中，选择为独立服务器配置Oracle Grid Infrastructure（Oracle Restart）选项以安装和配置Oracle Restart和Oracle ASM。单击下一步。
  6. 在安装期间，安装在Oracle ASM上并在Oracle ASMFD上使用字符串AFD:*注册的磁盘路径将列为缺省数据库存储候选磁盘。
  7. 根据需要使用其他磁盘组配置Oracle ASM。

    • 默认磁盘组名称为DATA。您可以为磁盘组输入新名称，或使用默认名称。

    • 您创建的任何其他磁盘设备必须由执行网格安装的用户拥有。

  8. 如果要使用Oracle ASM筛选器驱动程序（Oracle ASMFD）来管理Oracle ASM磁盘设备，请在“创建ASM磁盘组”屏幕上选择“ 配置Oracle ASM筛选器驱动程序 ”。
    在Linux上，如果要使用Oracle ASM筛选器驱动程序（Oracle ASMFD）来管理Oracle ASM磁盘设备，则必须在启动Oracle Grid Infrastructure安装之前卸载Oracle ASM库驱动程序（Oracle ASMLIB）。
  9. 根据需要响应配置提示以配置Oracle Grid Infrastructure。单击帮助以获取信息。
  10. 提供信息以自动执行根脚本，或在OUI提示时以root身份运行脚本。

    如果为运行根脚本配置自动化，并且根脚本失败，则可以手动修复问题，然后单击“ 重试”以再次运行根脚本。

  11. 启动Oracle数据库安装，并为Oracle数据库文件存储选择Oracle ASM磁盘组。有关安装期间的帮助 ，请单击Oracle Universal Installer页面上的“ 帮助” ，以获取更多信息。
