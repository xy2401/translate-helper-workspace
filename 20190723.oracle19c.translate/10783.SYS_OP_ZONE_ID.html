SYS_OP_ZONE_ID

句法


说明sys_op_zone_id.eps的说明

目的

SYS_OP_ZONE_ID将rowid作为参数，并返回区域ID。 rowid标识表中的行。区域ID标识包含该行的称为区域的连续磁盘块集。该函数返回NUMBER值。

使用CREATE MATERIALIZED ZONEMAP语句创建区域映射时，将使用SYS_OP_ZONE_ID函数。必须在区域映射的定义子查询的SELECT和GROUP BY子句中指定SYS_OP_ZONE_ID 。

对于rowid ，请指定区域映射的事实表的ROWID伪列。

使用schema和table指定事实表的模式和名称，或使用t_alias指定事实表的表别名。这些参数的规范取决于区域映射的定义子查询中的FROM子句：

  • 如果FROM子句指定事实表的表别名，则还必须在SYS_OP_ZONE_ID指定表别名（ t_alias ）。

  • 如果FROM子句未指定事实表的表别名，则使用table指定事实表的名称。如果事实表位于您自己的模式之外，则可以使用schema限定符。如果省略schema ，则数据库假定事实表在您自己的模式中。如果FROM子句仅指定一个表（事实表），则无需指定schema或table 。

可选的scale参数表示区域图的比例。没有必要指定此参数，因为默认情况下， SYS_OP_ZONE_ID使用正在创建的区域映射的比例。如果您指定了scale ，则它必须与正在创建的区域地图的比例相匹配。有关指定区域地图比例的信息，请参阅CREATE MATERIALIZED ZONEMAP的SCALE子句。

例子

以下示例在创建跟踪事实表sales的列time_id的基本区域映射时使用SYS_OP_ZONE_ID函数。区域图的比例是默认值10。因此， SYS_OP_ZONE_ID函数将默认为比例值10。 

CREATE MATERIALIZED ZONEMAP sales_zmap AS SELECT SYS_OP_ZONE_ID（rowid），MIN（time_id），MAX（time_id）FROM sales GROUP BY SYS_OP_ZONE_ID（rowid）;

以下示例与上一示例类似，不同之处在于创建的区域映射的比例指定为8。因此， SYS_OP_ZONE_ID函数将默认为比例值8。 

CREATE MATERIALIZED ZONEMAP sales_zmap SCALE 8 AS SELECT SYS_OP_ZONE_ID（rowid），MIN（time_id），MAX（time_id）FROM sales GROUP BY SYS_OP_ZONE_ID（rowid）;

以下示例返回错误，因为正在创建的区域映射的比例指定为8，这与SYS_OP_ZONE_ID函数中指定的scale参数12不匹配。 

CREATE MATERIALIZED ZONEMAP sales_zmap SCALE 8 AS SELECT SYS_OP_ZONE_ID（rowid，12），MIN（time_id），MAX（time_id）FROM sales GROUP BY SYS_OP_ZONE_ID（rowid，12）;

以下示例创建连接区域映射。事实表是sales ，维度表是products和customers 。因为表的别名s为事实表中所指定的FROM子句中，表的别名s也被指定SYS_OP_ZONE_ID功能。 

CREATE MATERIALIZED ZONEMAP sales_zmap AS SELECT SYS_OP_ZONE_ID（s.rowid），MIN（prod_category），MAX（prod_category），MIN（country_id），MAX（country_id）FROM sales s，products p，customers c WHERE s.prod_id = p.prod_id（ +）AND s.cust_id = c.cust_id（+）GROUP BY SYS_OP_ZONE_ID（s.rowid）;
