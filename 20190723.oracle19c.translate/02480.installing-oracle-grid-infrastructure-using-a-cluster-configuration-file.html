使用群集配置文件安装Oracle Grid Infrastructure

在安装Oracle Grid Infrastructure期间，您可以选择手动提供群集配置信息，也可以使用群集配置文件。

群集配置文件是您可以在启动gridSetup.sh之前创建的文本文件，该文件为安装程序提供配置群集所需的群集节点地址。

如果您打算在测试群集上执行重复安装，或者您打算在许多节点上执行安装，Oracle建议您考虑使用群集配置文件。Grid_home /install/response/sample.ccf目录中提供了一个示例集群配置文件。

要手动创建集群配置文件，请启动文本编辑器，并创建一个文件，该文件为每个集群成员节点提供公共和虚拟IP地址的名称，格式如下：

node1 node1-vip node2 node2-vip 。。。

指定不同的节点，用空格或冒号（:)分隔它们。

例如：

mynode1 mynode1-vip mynode2 mynode2-vip

或者，例如：

mynode1：mynode1-vip mynode2：mynode2-vip
