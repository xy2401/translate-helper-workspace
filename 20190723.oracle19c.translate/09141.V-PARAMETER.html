8.108 V $ PARAMETER

V$PARAMETER显示有关当前对会话有效的初始化参数的信息。 新会话从V$SYSTEM_PARAMETER视图显示的实例范围值继承参数值。
柱 数据类型 描述

NUM

数

参数号

名称

VARCHAR2（80）

参数名称

类型

数

参数类型：

  • 1 - 布尔值

  • 2 - 字符串

  • 3 - 整数

  • 4 - 参数文件

  • 5 - 保留

  • 6 - 大整数

值

VARCHAR2（4000）

会话的参数值（如果在会话中修改）;否则，实例范围的参数值

DISPLAY_VALUE

VARCHAR2（4000）

参数值采用用户友好的格式。例如，如果VALUE列显示大整数参数的值262144 ，则DISPLAY_VALUE列将显示值256K 。

默认值

VARCHAR2（255）

此参数的默认值。如果未为参数显式指定值，则此参数的值。

ISDEFAULT

VARCHAR2（9）

指示参数是否设置为默认值（ TRUE ）或参数文件中是否指定了参数值（ FALSE ）。

对于未在init.ora或服务器参数文件（SPFILE）中指定的参数，数据库将ISDEFAULT列的值设置为TRUE 。

ISSES_MODIFIABLE

VARCHAR2（5）

指示是否可以使用ALTER SESSION （ TRUE ）更改参数（ FALSE ）

ISSYS_MODIFIABLE

VARCHAR2（9）

指示是否可以使用ALTER SYSTEM更改参数以及更改何时生效：

  • IMMEDIATE - 无论用于启动实例的参数文件类型如何，都可以使用ALTER SYSTEM更改参数。更改立即生效。

  • DEFERRED - 无论用于启动实例的参数文件类型如何，都可以使用ALTER SYSTEM更改参数。更改将在后续会话中生效。

  • FALSE - 除非使用服务器参数文件启动实例，否则无法使用ALTER SYSTEM更改参数。更改将在后续实例中生效。

ISPDB_MODIFIABLE

VARCHAR2（5）

指示是否可以在PDB内修改参数（ TRUE ）（ FALSE ）

ISINSTANCE_MODIFIABLE

VARCHAR2（5）

对于可以使用ALTER SYSTEM更改的参数，指示参数的值是否可以针对每个实例（ TRUE ）不同，或者参数是否必须对所有Real Application Clusters实例具有相同的值（ FALSE ）。如果ISSYS_MODIFIABLE FALSE ，则此列始终为FALSE 。

ISMODIFIED

VARCHAR2（10）

指示实例启动后是否已修改参数：

  • MODIFIED - 已使用ALTER SESSION修改参数

  • SYSTEM_MOD - 已使用ALTER SYSTEM修改参数（导致所有当前登录的会话值被修改）

  • FALSE - 实例启动后未修改参数

ISADJUSTED

VARCHAR2（5）

指示Oracle是否将输入值调整为更合适的值（例如，参数值应为prime，但用户输入非素数，因此Oracle将值调整为下一个素数）

ISDEPRECATED

VARCHAR2（5）

指示参数是否已弃用（ TRUE ）（ FALSE ）

ISBASIC

VARCHAR2（5）

指示参数是否为基本参数（ TRUE ）（ FALSE ）

描述

VARCHAR2（255）

参数说明

UPDATE_COMMENT

VARCHAR2（255）

与最新更新相关的评论

HASH

数

参数名称的哈希值

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

例子

以下查询返回ALLOW_GLOBAL_DBLINKS初始化参数的默认值：

SQL> SELECT name，default_value FROM v $ parameter 2 WHERE name ='allow_global_dblinks';名称  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  - - -  -  -  -  -  默认值  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  - - -------------------- allow_global_dblinks FALSE SQL>

以下查询显示ALLOW_GLOBAL_DBLINKS初始化参数在PDB中不可修改：

SQL> SELECT name，ispdb_modifiable FROM v $ parameter 2 WHERE name ='allow_global_dblinks';名称  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  -  - - ---------- ISPDB ----- allow_global_dblinks FALSE SQL>
