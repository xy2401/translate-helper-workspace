降级到Oracle Grid Infrastructure 12c第1版（12.1）

使用此过程降级到Oracle Grid Infrastructure 12c第1版（12.1）。

  1. 删除Oracle Grid Infrastructure 12c第2版（12.2）管理数据库：
    dbca -silent -deleteDatabase -sourceDB -MGMTDB
  2. 使用命令语法rootcrs.sh -downgrade以任何顺序在所有节点上降级Oracle Grid Infrastructure。例如：
    ＃/01 /app/12.2.0/grid/crs/install/rootcrs.sh -downgrade

    从具有Oracle Grid Infrastructure安装用户写入权限的目录运行此命令。您可以在所有群集节点上并行运行降级脚本，但只能运行一个。

  3. 降级所有其他节点后降级最后一个节点：
    ＃/01 /app/12.2.0/grid/crs/install/rootcrs.sh -downgrade
  4. 删除Oracle Grid Infrastructure 12c第2版（12.2）Grid home作为活动的Oracle Clusterware主目录：

    1. 在rootupgrade.sh脚本已成功运行的任何集群成员节点上，以Oracle Grid Infrastructure安装所有者身份登录。

    2. 使用以下命令启动安装程序，其中/u01/app/12.2.0/grid是新（已升级）Grid /u01/app/12.2.0/grid的位置：

      cd /u01/app/12.2.0/grid/oui/bin ./runInstaller -nowait -waitforcompletion -ignoreSysPrereqs -updateNodeList -silent CRS = false ORACLE_HOME = / u01 / app / 12.2.0 / grid“CLUSTER_NODES = node1，node2， node3 “-doNotUpdateNodeList

      如果Grid home是共享主-cfs则添加标志-cfs 。

  5. 将Oracle Grid Infrastructure 12c第1版（12.1）Grid home设置为活动的Oracle Clusterware主目录：

    1. 在rootupgrade脚本已成功运行的任何集群成员节点上，以Oracle Grid Infrastructure安装所有者身份登录。

    2. 使用以下命令启动安装程序，其中为ORACLE_HOME提供的路径是早期Oracle Clusterware安装中主目录的位置。 

      $ cd /u01/app/12.1.0/grid/oui/bin $ ./runInstaller -nowait -waitforcompletion -ignoreSysPrereqs -updateNodeList -silent CRS = true ORACLE_HOME = / u01 / app / 12.1.0 / grid“CLUSTER_NODES = node1， node2，node3 “
  6. 在所有节点上启动12.1 Oracle Clusterware堆栈。
    crsctl开始crs
  7. 在任何节点上，删除MGMTDB资源，如下所示：
    121_Grid_home / bin / srvctl删除mgmtdb
  8. 如果要降级到Oracle Grid Infrastructure 12c第1版（12.1.0.2），请运行以下命令以配置网格基础结构管理数据库：

    1. 从12.1.0.2 Oracle主目录以静默方式运行DBCA，并按如下方式创建管理数据库容器数据库（CDB）：

      12102_Grid_home / bin / dbca -silent -createDatabase -createAsContainerDatabase true -templateName MGMTSeed_Database.dbc -sid -MGMTDB -gdbName _mgmtdb -storageType ASM -diskGroupName ASM_DG_NAME -datafileJarLocation 12102_Grid_home / assistantants / dbca / templates -characterset AL32UTF8 -autoGeneratePasswords -skipUserTemplateCheck

    2. 从12.1.0.2 Oracle主目录以静默方式运行DBCA，并创建管理数据库可插拔数据库（PDB），如下所示：

      12102_Grid_home / bin / dbca -silent -createPluggableDatabase -sourceDB -MGMTDB -pdbName cluster_name -createPDBFrom RMANBACKUP -PDBBackUpfile 12102_Grid_home /assistants/dbca/templates/mgmtseed_pdb.dfb -PDBMetadataFile 12102_Grid_home /assistants/dbca/templates/mgmtseed_pdb.xml -createAsClone true -internalSkipGIHomeCheck
  9. 如果要降级到Oracle Grid Infrastructure 12c第1版（12.1.0.1），请从12.1.0.1 Oracle主目录以静默方式运行DBCA，并按如下方式创建管理数据库：
    12101_Grid_home / bin / dbca -silent -createDatabase -templateName MGMTSeed_Database.dbc -sid -MGMTDB -gdbName _mgmtdb -storageType ASM -diskGroupName ASM_DG_NAME -datafileJarLocation 12101_Grid_home / assistantants / dbca / templates -characterset AL32UTF8 -autoGeneratePasswords
  10. 通过从121_Grid_home /bin/mgmtca位置运行Configuration Assistant来配置管理数据库。
