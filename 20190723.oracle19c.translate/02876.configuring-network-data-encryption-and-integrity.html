Oracle数据库本机网络加密和完整性的工作原理

Oracle数据库提供本机数据网络加密和完整性，以确保数据在网络中传输时的安全性。

安全密码系统的目的是将明文数据转换为基于密钥的难以理解的密文 ，这样在不知道正确密钥的情况下将密文转换回其相应的明文是非常困难的（计算上不可行的）。

在对称密码系统中，相同的密钥用于加密和解密相同的数据。Oracle数据库提供高级加密标准（AES）对称密码系统，用于保护Oracle Net Services流量的机密性。

高级加密标准

Oracle数据库支持联邦信息处理标准（FIPS）加密算法，高级加密标准（AES）。

所有美国政府组织和企业都可以使用AES来保护网络上的敏感数据。此加密算法定义了三种标准密钥长度，即128位，192位和256位。所有版本均以外部密码块链接（CBC）模式运行。

Oracle数据库支持Academia，Research Institute和Agency（ARIA）算法。

该算法承认韩国研究人员在设计算法方面的合作努力。

ARIA定义了三种标准密钥长度，即128位，192位和256位。所有版本均以外部密码密码块链接（CBC）模式运行。

GOST

Oracle数据库支持GOsudarstvennyy STandart（GOST）算法。

GOST算法由欧亚标准化，计量和认证委员会（EACS）创建。

GOST定义了256位的密钥大小。在Oracle数据库中，使用外部密码块链接（CBC）模式。

种子

Oracle数据库支持韩国信息安全局（KISA）加密算法SEED。

SEED定义了128位的密钥大小。标准的扩展定义了192位和256位的其他密钥大小，但Oracle数据库不支持这些扩展。在Oracle数据库中，SEED以外部密码块链接（CBC）模式运行。

三重DES支持

Oracle数据库支持三重DES加密（3DES），它通过三次DES算法加密消息数据。

3DES提供了高度的消息安全性，但性能下降。性能损失的大小取决于执行加密的处理器的速度。与标准DES算法相比，3DES加密数据块通常需要三倍的时间。

3DES有两键和三键版本，有效密钥长度分别为112位和168位。两个版本都在外部密码块链接（CBC）模式下运行。

Oracle数据库和安全网络服务提供的DES40算法是DES的变体，其中秘密密钥被预处理以提供40个有效密钥位。它旨在为美国和加拿大以外的客户提供基于DES的加密，此时美国的出口法律更具限制性。目前DES40，DES和3DES都可以出口。仍然支持DES40为国际客户提供向后兼容性。

Oracle数据库还提供针对两种形式的主动攻击的保护。

攻击类型 说明

数据修改攻击

未授权方拦截传输中的数据，更改数据并重新传输数据是数据修改攻击。例如，拦截100美元的银行存款，将金额更改为10,000美元，并重新传输更高的金额是数据修改攻击。

数据完整性算法支持

消息摘要5（MD5）算法或安全散列算法（SHA-1和SHA-2）的密钥序列实现可防止这些攻击。

这两种哈希算法都会创建一个校验和，如果数据以任何方式被更改，它就会发生变化。此保护独立于加密过程运行，因此您可以启用或不启用加密来启用数据完整性。

基于Diffie-Hellman的密钥协商

您可以使用Diffie-Hellman密钥协商算法来保护多用户环境中的数据。

在多用户环境中，安全密钥分发很困难。Oracle数据库使用众所周知的Diffie-Hellman密钥协商算法来执行加密和数据完整性的安全密钥分发。

使用加密来保护加密数据的安全性时，必须经常更改密钥以最大限度地减少受损密钥的影响。因此，Oracle数据库密钥管理功能会在每个会话中更改会话密钥。

您可以使用Authentication Key Fold-in来解决Diffie-Hellman密钥协商算法密钥协商中可能存在的第三方攻击（历史上称为中间人攻击 ）。它通过将仅由客户端和服务器知道的共享秘密与由Diffie-Hellman协商的原始会话密钥相结合，显着增强了会话密钥。

客户端和服务器使用Diffie-Hellman生成的会话密钥开始通信。当客户端向服务器进行身份验证时，它们会建立一个只有双方都知道的共享密钥。Oracle数据库结合了共享密钥和Diffie-Hellman会话密钥，以生成更强大的会话密钥，旨在抵御中间人攻击。

注意：

身份验证密钥折叠功能是Oracle数据库的嵌入功能，无需系统或网络管理员进行配置。

关于激活加密和完整性

在任何网络连接中，客户端和服务器都可以支持多种加密算法和完整性算法。

建立连接后，服务器将从sqlnet.ora文件中指定的算法中选择要使用的算法（如果有）。服务器在客户端和服务器上搜索可用算法之间的匹配，并在其自己的列表中选择也出现在客户端列表中的第一个算法。如果连接的一侧未指定算法列表，则可以接受安装在该侧的所有算法。如果任何一方指定未安装的算法，则连接失败并显示错误消息ORA-12650 。

通过修改客户端和网络上的服务器上的sqlnet.ora文件来定义加密和完整性参数。

您可以选择配置任何或所有可用的加密算法，以及其中一种或两种可用的完整性算法。每个连接会话仅使用一种加密算法和一种完整性算法。

注意：

Oracle数据库选择在客户端和服务器上启用的第一个加密算法和第一个完整性算法。Oracle建议您按照您希望协商的顺序选择算法和密钥长度，首先选择最强的密钥长度。

    Oracle Net Manager可用于为加密和完整性配置参数指定四个可能的值。
  • 拒绝配置参数
    REJECTED值禁用安全服务，即使另一方需要此服务。
  • 接受的配置参数
    如果另一方需要或请求服务，则ACCEPTED值启用安全服务。
  • 请求的配置参数
    如果另一方允许此服务，则REQUESTED值启用安全服务。
  • 需要的配置参数
    REQUIRED值启用安全服务或排除连接。

拒绝配置参数

REJECTED值禁用安全服务，即使另一方需要此服务。

在此方案中，连接的此端指定不允许安全服务。如果另一端设置为REQUIRED ，则连接将终止，并显示错误消息ORA-12650 。如果另一端设置为REQUESTED ， ACCEPTED或REJECTED ，则连接会继续而不会出现错误且未启用安全服务。

父主题： 关于协商加密和完整性

接受的配置参数

如果另一方需要或请求服务，则ACCEPTED值启用安全服务。

在这种情况下，连接的这一侧不需要安全服务，但如果另一端设置为REQUIRED或REQUESTED ，则启用它。如果另一端设置为REQUIRED或REQUESTED ，并且找到加密或完整性算法匹配，则连接将继续，不会出现错误并启用安全服务。如果另一端设置为REQUIRED且未找到算法匹配，则连接将终止，并显示错误消息ORA-12650 。

如果另一端设置为REQUESTED并且未找到算法匹配，或者另一端设置为ACCEPTED或REJECTED ，则连接将继续而不会出现错误并且未启用安全服务。

父主题： 关于协商加密和完整性

请求的配置参数

如果另一方允许此服务，则REQUESTED值启用安全服务。

在这种情况下，连接的这一侧指定需要安全服务但不是必需的。如果另一方指定了ACCEPTED ， REQUESTED或REQUIRED ，则启用安全服务。另一方必须有匹配的算法，否则服务未启用。如果另一方指定REQUIRED并且没有匹配的算法， 则连接失败。

需要的配置参数

REQUIRED值启用安全服务或排除连接。

在此方案中，连接的此端指定必须启用安全服务。如果另一方指定REJECTED或另一方没有兼容算法，则连接失败 。

关于同时为不同用户启用Oracle本机加密和SSL身份验证

                        

                           
默认情况下，Oracle数据库不允许同时为不同用户提供Oracle本机加密和安全套接字层（SSL）身份验证。

                           
同时使用Oracle本机加密（也称为高级网络选项（ANO）加密）和SSL身份验证称为双重加密。假设客户端将SQLNET.ENCRYPTION_CLIENT参数设置为required ，并且服务器将SQLNET.ENCRYPTION_SERVER参数设置为required 。如果使用TCPS侦听器，则拒绝连接并且ORA-12696: Double Encryption Turned On, login disallowed将返回ORA-12696: Double Encryption Turned On, login disallowed错误。
                           

                           
但是，在某些情况下，必须同时配置TCP和TCPS侦听器，以便某些用户可以使用用户名和密码连接到服务器，而其他用户可以使用SSL证书对服务器进行验证。在这些情况下，您必须配置基于密码的身份验证和SSL身份验证。在以前的版本中一种解决方法是设置SQLNET.ENCRYPTION_SERVER参数requested 。如果您的要求是将SQLNET.ENCRYPTION_SERVER设置为required ，则可以将SQLNET.ENCRYPTION_CLIENT和SQLNET.ENCRYPTION_SERVER的IGNORE_ANO_ENCRYPTION_FOR_TCPS参数设置为TRUE 。默认情况下，它设置为FALSE 。

                           
将IGNORE_ANO_ENCRYPTION_FOR_TCPS设置为TRUE强制客户端忽略为所有传出TCPS连接的SQLNET.ENCRYPTION_CLIENT参数设置的值。在配置TCPS和ANO时，不是返回ORA-12696错误， SQLNET.ENCRYPTION_SERVER当使用TCPS客户端和这两个参数之间存在冲突时，此参数允许数据库忽略SQLNET.ENCRYPTION_CLIENT或SQLNET.ENCRYPTION_SERVER设置。设置为required 。

同时为不同用户配置Oracle本机加密和SSL身份验证

                        

                           
使用IGNORE_ANO_ENCRYPTION_FOR_TCPS参数可以同时使用Oracle本机加密和安全套接字层（SSL）身份验证。
                           

                           
在服务器上，必须在sqlnet.ora文件中设置IGNORE_ANO_ENCRYPTION_FOR_TCPS ，在客户端上，可以在sqlnet.ora文件或tnsnames.ora文件中设置它。
                           

                           
                           
    
                              
  1. 登录数据库服务器
    2. 
                              
  2. 转到sqlnet.ora文件的位置。
    默认情况下， sqlnet.ora位于ORACLE_BASE /network/admin目录中。sqlnet.ora文件也可以存储在TNS_ADMIN环境变量指定的目录中。
                                 
    
                              
    3. 
                              
  3. 在sqlnet.ora ，检查是否required或requested当前的SQLNET.ENCRYPTION_SERVER设置。
    4. 
                              
  4. 如果SQLNET.ENCRYPTION_SERVER设置为required，则将SQLNET.IGNORE_ANO_ENCRYPTION_FOR_TCPS添加到sqlnet.ora ，然后将其设置为TRUE 。
    IGNORE_ANO_ENCRYPTION_FOR_TCPS = TRUE
    
                              
    5. 
                              
  5. 保存并退出sqlnet.ora 。
    6. 
                              
  6. 登录客户端。
    对于客户端，您可以在sqlnet.ora文件或tnsnames.ora文件中设置该值。
                                 
    
                                 
      
                                    
    • 在sqlnet.ora设置值：检查SQLNET.ENCRYPTION_CLIENT参数是否设置为required 。如果是SQLNET.ENCRYPTION_CLIENT ，则编辑sqlnet.ora文件以进行以下设置：
      IGNORE_ANO_ENCRYPTION_FOR_TCPS = TRUE
      • 
                                    
    • 在tnsnames.ora设置值：默认情况下， tnsnames.ora与sqlnet.ora位于同一位置。如果在sqlnet.ora SQLNET.ENCRYPTION_CLIENT设置为required ，则在TNS_ALIAS设置的SECURITY部分中，设置IGNORE_ANO_ENCRYPTION_FOR_TCPS=TRUE 。例如：
      test_ssl =（DESCRIPTION =（ADDRESS =（PROTOCOL = tcps）（HOST =）（PORT = 1750））（CONNECT_DATA =（SID = ^ ORACLE_SID ^））（SECURITY =（IGNORE_ANO_ENCRYPTION_FOR_TCPS = TRUE）））
