词汇表

ADT

抽象数据类型。

API

请参阅应用编程接口 。

应用程序接口

应用程序访问操作系统和其他服务的调用约定。

近似CSCN

提交已将消息排入提交时队列的事务时，基于数据库的当前SCN的近似系统更改编号值。

异步

如果多任务系统的执行可以在后台独立进行，则它是异步的。在异步过程完成之前，可以启动其他进程。与同步相反。

BFILE

存在于驻留在操作系统中的数据库表空间外部的外部二进制文件。

二进制大对象

一种大型对象数据类型，其内容由二进制数据组成。此数据被视为原始数据，因为数据库无法识别其结构。

BLOB

见二元大对象 。

广播

一种发布/订阅模式，其中消息 生产者不知道任何消息使用者的身份。该模式类似于无线电或电视台。

缓冲队列

缓冲队列支持使用缓冲属性（缓冲消息）对消息进行排队，并在内存中实现它们。如果较新消息需要专用于缓冲消息的内存，或者缓冲消息已超过内存中规定的持续时间，则该缓冲消息将交换到基础队列表。缓冲消息的内存来自一个称为流池的独立池。如果数据库被退回，则无法恢复缓冲的消息。没有缓冲属性集的消息在基础持久队列中排队为持久消息。

典范

某种通常或标准的状态或方式。

人物大对象

大对象数据类型，其值由与数据库字符集对应的字符数据组成。Oracle Text搜索引擎可以索引和搜索字符大对象。

CLOB

见角色大物体 。

连接工厂

ConnectionFactory封装了一组由管理员定义的连接配置参数。客户端使用它来创建与Java消息服务提供者的连接。

提交时间队列

消息按其近似CSCN值排序的队列。

消费者

可以使消息出列的用户或应用程序。

数据操纵语言

数据操作语言（DML）语句操纵数据库数据。例如，查询，插入，更新和删除表的行都是DML操作;锁定表或视图以及检查SQL语句的执行计划也是DML操作。

数据库配置助手

用于创建和删除数据库以及管理数据库模板的Oracle数据库工具。

DBCA

请参见Database Configuration Assistant 。

出队

从队列中检索消息

DML

请参阅数据操作语言 。

排队

将消息放入队列中。JMS相当于enqueue的发送 。

异常队列

如果由于某种原因无法检索和处理消息，则将消息传送到异常队列 。

IDAP

请参阅Internet数据访问演示 。

索引组织表

与将数据存储为无序集合的普通表不同，索引组织表的数据存储在按主键排序的B树索引结构中。除了存储索引组织表行的主键列值之外，B树中的每个索引条目也存储非键列值。

互联网数据访问演示

Oracle Database Advanced Queuing操作的简单对象访问协议 （SOAP）规范。IDAP定义SOAP请求主体的XML消息结构。使用HTTP（S）在因特网上传输IDAP结构的消息 。

进程间通信

一个进程与另一个进程之间的数据交换，可以在同一台计算机内或通过网络进行。它意味着一种保证响应请求的协议。

IOT

请参阅索引组织表 。

IPC

请参阅进程间通信 。

Java数据库连接

用于从Java程序连接到关系数据库的行业标准Java接口，由Sun Microsystems定义。

Java消息服务

由Sun Microsystems，Oracle，IBM和其他供应商定义的消息传递标准。JMS是一组接口和关联语义，用于定义JMS客户端如何访问企业消息传递产品的工具。

Java命名和目录接口

Sun的编程接口，用于将Java程序连接到命名和目录服务。

Java虚拟机

Java解释器，将已编译的Java字节码转换为平台的机器语言并运行它。JVM可以在客户端，浏览器，中间层，Intranet，应用程序服务器（如Oracle Application Server 10g ）或数据库服务器（如Oracle Database 10g）上运行 。

JDBC

请参阅Java数据库连接 。

JDBC驱动程序

特定于供应商的Java Database Connectivity层，提供对特定数据库的访问。Oracle数据库提供了三个JDBC驱动程序 - Thin， OCI和KPRB。

JMS

请参阅Java消息服务 。

JMS连接

客户端与其JMS提供程序的活动连接，通常是客户端和提供程序的服务守护程序之间的开放TCP / IP套接字（或一组打开的套接字）。

JMS消息

JMS消息由标头，一个或多个可选属性和消息有效负载组成。

JMS会话

用于生成和使用消息的单线程上下文。

JMS主题

相当于其他Oracle Database Advanced Queuing接口中的多消费者队列。

JNDI

请参阅Java命名和目录接口 。

Jnnn

作业队列过程

JServer等

在Oracle数据库的内存空间中运行的Java虚拟机。

JVM

请参阅Java虚拟机

大物件

由SQLILE ， BLOB ， CLOB和NCLOB对象组成的SQL数据类型。

LDAP

请参阅轻量级目录访问协议

轻量级目录访问协议

标准的可扩展目录访问协议。它是LDAP客户端和服务器用于通信的通用语言。支持行业标准目录产品的设计约定框架，例如Oracle Internet Directory。

高球

看大对象

当地消费者

本地使用者将消息从生成器排入消息的同一队列中取消。

逻辑变化记录

具有描述数据库更改的特定格式的对象，由捕获进程或用户应用程序从重做日志捕获。捕获进程仅将包含逻辑更改记录（LCR）的消息排入ANYDATA队列。为了提高性能，这些LCR始终存储在缓冲队列中 。

信息

插入队列并从队列中检索的最小信息单元。消息由控制信息（元数据）和有效负载（数据）组成。

组播

一种发布/订阅模式，其中消息 生产者知道每个消费者的身份。此模式也称为点对多点。

民族性大物

大对象数据类型，其值由对应于数据库国家字符集的字符数据组成。

NCLOB

见国家大字体 。

非持久

非持久队列将消息存储在内存中。它们通常用于提供异步机制，以向当前连接的所有用户发送通知。Oracle数据库高级队列10g第2版（10.2）中不推荐使用非持久队列。Oracle建议您使用缓冲消息传递。

非事务性

允许一次只有一条消息的入队和出列。

对象类型

对象类型封装了数据结构以及操作数据所需的功能和过程。使用CREATE TYPE语句定义对象类型时，可以创建与现实世界对象对应的抽象模板。

OCI

请参阅Oracle Call Interface 。

Oracle JMS

请参阅Oracle Java消息服务 。

OLTP

请参阅在线事务处理 。

在线交易处理

在线交易处理系统经过优化，可实现快速可靠的交易处理。与数据仓库系统相比，大多数OLTP交互涉及相对较少的行，但涉及较大的一组表。

Oracle调用接口

应用程序编程接口，支持Oracle数据库中的数据和模式操作。

Oracle Java消息服务

Oracle Java消息服务（Oracle JMS）基于Java消息服务 （JMS）标准为Oracle数据库高级排队提供Java API 。Oracle JMS支持标准JMS接口，并具有支持Oracle数据库高级队列管理操作和其他Oracle数据库高级队列功能的扩展，这些功能不属于该标准。

制片人

可以将消息排入队列的用户或应用程序。

传播

将消息从一个队列复制到另一个队列（本地或远程）队列。

发布/订阅

一种消息传递，其中生产者将消息排入一个或多个多消费者队列，然后该消息由多个订阅者出列。发布的消息可以具有称为广播的广泛传播模式或称为多播的更窄目标模式。

QMNC

队列监视器协调员。它根据系统负载动态生成从属qXXX。奴隶做各种背景任务。

QMNn

队列监控过程。

队列

消息传递系统用于存储消息的抽象存储单元。

队列表

存储队列的数据库表。每个队列表都包含一个默认的异常队列 。

接受者

由enqueuer或队列管理员授权检索消息的代理。enqueuer可以明确指定可以将消息作为消息的接收者检索的消费者。队列管理员可以指定可以从队列中检索消息的默认收件人列表。默认列表中指定的收件人称为订户 。如果在未指定收件人的情况下将邮件排队，则会将邮件发送给所有订阅者。队列中的特定消息可以指向特定接收者，特定接收者可以是或可以不是队列的订户，从而覆盖订户列表。

如果仅指定了收件人的名称，则收件人必须从邮件入队的队列中将邮件出列。如果使用协议值0指定收件人的名称和地址，则该地址应该是同一数据库中的另一个队列的名称或Oracle数据库的另一个安装。如果收件人的名称为NULL ，则该消息将传播到该地址中的指定队列，并且可以由该地址中指定的队列的任何订户出列。如果协议字段非零，则系统不会解释名称和地址，并且消息可以由特殊使用者出列 。

远程消费者

远程消费者从队列是从哪里队列不同出队的消息被排队。

结果缓存

结果缓存可以定义为一组参数值的评估结果的缓存。在规则引擎的情况下，结果缓存指的是在给定某些输入参数的情况下与规则集，规则和独立表达式评估有关的结果的缓存

规则

布尔表达式，定义订阅者订阅消息的兴趣。表达式使用类似于SQL查询的WHERE子句的语法，并且可以包括以下条件：消息属性（当前优先级和相关标识符），用户数据属性（仅对象有效负载）和函数。如果与订户关联的规则对邮件的评估结果为TRUE ，则即使邮件没有指定的收件人 ，也会将邮件发送给该订户。

规则引擎

评估规则的Oracle数据库软件。规则是数据库对象，使客户端能够在事件发生且满足条件时执行操作。规则类似于SQL查询的WHERE子句中的条件。用户创建的应用程序和Oracle数据库功能（例如Oracle数据库高级队列）都可以是规则引擎的客户端。

模式

数据库对象的集合，包括表，视图，序列，存储过程，同义词，索引，集群和数据库链接等逻辑结构。模式具有控制它的用户的名称。

发送

JMS相当于入队 。

的servlet

作为网络服务的一部分运行并响应来自客户端的请求的Java程序。它通常是HTTP服务器。

SGA

请参阅系统全局区域 。

分片队列

单个逻辑队列，通过系统维护的分区划分为多个独立的物理队列。

简单对象访问协议

使用World Wide Web Consortium定义的使用XML over HTTP调用代码的最小约定。

肥皂

请参阅简单对象访问协议 。

订户

队列管理员授权从队列中检索消息的代理。

系统全球区

一组共享内存结构，包含一个Oracle数据库实例的数据和控制信息。SGA和Oracle数据库进程构成Oracle数据库实例。每当启动实例时，Oracle数据库都会自动为SGA分配内存，并且当您关闭实例时，操作系统会回收内存。每个实例都有一个且只有一个SGA。

同步

如果它们取决于诸如公共定时信号之类的特定事件的发生，则两个或更多个过程是同步的。异步的反面。

交易

允许同时将多个消息排队或出列为组的一部分。

转型

从一种Oracle数据类型到另一种Oracle数据类型的映射，由SQL函数表示，该函数将源数据类型作为输入并返回目标数据类型的对象。可以在enqueue期间指定转换，以在将消息插入队列之前将消息转换为正确的类型。可以在出队期间指定它以所需格式接收消息。如果使用远程使用者指定，则在将消息传播到目标队列之前对其进行转换。

用户队列

用于正常消息处理的队列 。

VARRAY

一组有序的数据元素。给定数组的所有元素都具有相同的数据类型。每个元素都有一个索引，它是一个与元素在数组中的位置相对应的数字。数组中元素的数量是数组的大小。Oracle数据库允许数组具有可变大小。

通配符

与字符串比较中的任何字符匹配的特殊字符或字符序列。

工作流程

项目或业务事务中所有活动之间的关系集，从开始到结束。活动与不同类型的触发关系相关。活动可以由外部事件或其他活动触发。
