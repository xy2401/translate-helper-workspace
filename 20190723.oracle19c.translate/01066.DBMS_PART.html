116.1 DBMS_PART安全模型

DBMS_PART是一个调用者权限包，以用户的权限运行。

116.2 DBMS_PART操作说明

以下操作说明适用于DBMS_PART。

  • DBMS_PART忽略它在清理过程中DBMS_PART所有错误。

  • 要显示消息， PL/SQL procedure executed successfully需要至少一次清理操作才能成功。

116.3.1 CLEANUP_GIDX过程

作为使用异步全局索引维护的先前分区维护操作的结果，全局索引可以包含指向不再存在的数据段的条目。这些陈旧的索引行在表或索引的任何操作期间都不会导致任何正确性问题或损坏，无论这些是查询，DML，DDL还是分析。此过程将识别并清除这些全局索引，以确保存储和性能方面的效率。

句法

DBMS_PART.CLEANUP_GIDX（schema_name_in IN VARCHAR2 DEFAULT NULL，table_name_in IN VARCHAR2 DEFAULT NULL）;

参数

表116-2 CLEANUP_GIDX功能参数

参数 描述

schema_name_in

非NULL进程仅对给定模式中的表进行索引

table_name_in

非NULL进程仅在给定模式中的给定表上建立索引（如果table_name_in为非NULL则schema_name_in必须为非NULL ）

116.3.2 CLEANUP_ONLINE_OP过程

执行ALTER TABLE ...时有许多可能的失败点ALTER TABLE ...移动分区......ONLINE操作。此过程主动清除此类失败的在线移动操作，而不是等待后台进程（ SMON ）这样做。

句法

DBMS_PART.CLEANUP_ONLINE_OP（schema_name IN VARCHAR2 DEFAULT NULL，table_name IN VARCHAR2 DEFAULT NULL，partition_name IN VARCHAR2 DEFAULT NULL）;

参数

表116-3 CLEANUP_ONLINE_OP函数参数

参数 描述

SCHEMA_NAME

架构名称

TABLE_NAME

架构名称

的partition_name

分区名称

使用说明

  • 如果指定了schema_name ， table_name和partition_name则会清除指定分区的失败联机移动操作。

  • 如果指定了schema_name和table_name则会清除指定表的所有分区的所有失败的联机移动操作。

  • 如果仅指定了schema_name则会清除架构中所有失败的联机移动操作。

  • 如果没有提供参数，我们将清除系统中所有失败的在线移动操作。

  • 所有其他情况引发ORA-20000以通知用户无效输入作为参数。
