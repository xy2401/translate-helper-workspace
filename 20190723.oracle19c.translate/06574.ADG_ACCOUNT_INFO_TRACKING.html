1.8 ADG_ACCOUNT_INFO_TRACKING

ADG_ACCOUNT_INFO_TRACKING控制Oracle Active Data Guard备用数据库上用户的登录尝试。它扩展了对用户帐户安全信息的控制。

属性 描述

参数类型

串

句法

ADG_ACCOUNT_INFO_TRACKING = {LOCAL |全球}

默认值

本地

可修改

没有

可在PDB中修改

是

基本

没有

Oracle RAC

必须在所有实例上使用相同的值。

通过在备用数据库的内存中视图中维护用户帐户信息的本地副本，将ADG_ACCOUNT_INFO_TRACKING设置为LOCAL （默认值）继续强制执行默认行为。登录失败仅在每个数据库的基础上进行本地跟踪，并在达到失败最大值时拒绝登录。

通过在所有Data Guard数据库（主数据库和备用数据库）中维护用户帐户信息的单个全局副本，将参数设置为GLOBAL会触发更安全的行为。Data Guard配置中所有数据库的登录失败计入最大计数，并且在达到计数时将拒绝任何位置的登录。此设置可提高跨生产数据库和所有Active Data Guard备用数据库的登录攻击的安全性。
