关于Oracle Grid Infrastructure软件的Oracle主目录

查看有关在每个群集节点上创建Oracle主目录位置的信息。

Oracle家庭网格基础架构软件（Grid home）应位于与任何其他Oracle软件的Oracle主目录路径不同的路径中。Grid home的Optimal Flexible Architecture指南是以/pm/v/u的形式创建路径，其中p是字符串常量， m是唯一的固定长度密钥（通常是两位数字）， v是该版本的软件，以及u是Oracle网格基础架构软件（安装所有者grid用户）。在用于群集安装的Oracle Grid Infrastructure期间，Grid home的路径将更改为root用户，因此任何其他用户都无法在该路径中读取，写入或执行命令。例如，要以标准安装点路径格式u[00-99][00-99]/app/ release /grid创建Grid主目录，其中release是Oracle Grid Infrastructure软件的版本号，请创建以下路径：

/u01/app/19.0.0/grid

在安装过程中，以网格家整个路径的所有权更改为root （ /u01 ， /u01/app ， /u01/app/19.0.0 ， /u01/app/19.0.0/grid ）。如果未创建Grid home的唯一路径，则在Grid安装后，您可能会遇到其他安装的权限错误，包括同一路径下的任何现有安装。为避免将应用程序目录放在root所有权下的挂载点中，您可以为Grid home创建和选择以下路径：

/u01/19.0.0/grid

警告：

对于群集安装的Oracle Grid Infrastructure，请注意Oracle Grid Infrastructure二进制主目录的以下限制（Oracle Grid Infrastructure的网格主目录）：

  • 它不能放在Oracle基本目录之下，包括Oracle Grid Infrastructure安装所有者的Oracle基本目录。

  • 它不能放在安装所有者的主目录中。这些要求特定于群集安装的Oracle Grid Infrastructure。

用于独立服务器的Oracle Grid Infrastructure（Oracle Restart）可以安装在Oracle数据库安装的Oracle基础之下。
