操作：使用有效的错误号。

PLS-00702：PRAGMA EXCEPTION_INIT的第二个参数必须是数字文字

原因：传递给EXCEPTION_INIT pragma的第二个参数不是数字文字（例如变量）。对于Oracle错误，第二个参数必须是-9999 .. -1（不包括-100）范围内的数字文字，对于用户定义的错误，必须在-20000 ... -20999范围内。

操作：使用有效的错误号替换第二个参数。

PLS-00703：列表中多个命名参数实例

原因：子程序调用中的两个或多个实际参数引用相同的形式参数。

操作：删除重复的实际参数。

PLS-00704：' string '必须声明为异常

原因：传递给EXCEPTION_INIT编译指示的exception_name参数拼写错误或未引用合法声明的异常。或者，pragma是错误的;它必须出现在异常声明后的某个声明部分中。

操作：检查exception_name参数的拼写。然后，检查异常声明，确保异常名称和关键字EXCEPTION拼写正确。还要确保在异常声明之后的某个声明部分中出现了pragma。

PLS-00705：在需要返回类型的表达式中使用的异常' string '

原因：表达式中引用了异常。异常具有名称但不具有值，因此无法为表达式提供值。例如，以下RETURN语句是非法的：FUNCTION credit_limit（cust_no INTEGER）RETURN NUMBER IS限制NUMBER; over_limit EXCEPTION; ...开始 ...RETURN over_limit; - 非法END;

操作：检查表达式中标识符的拼写，然后重写表达式，使其不引用异常。

原因：异常名称被错误地用于限定对组件的引用。例如，当使用点表示法指定记录中的字段时，可能已编码了异常名称而不是记录名称。

操作：使用有效前缀（例如，包，记录或模式的名称）重写组件引用。
