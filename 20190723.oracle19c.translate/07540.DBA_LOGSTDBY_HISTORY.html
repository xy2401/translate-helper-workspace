5.161 DBA_LOGSTDBY_HISTORY

DBA_LOGSTDBY_HISTORY显示Data Guard配置中的切换和故障转移的历史记录。

它通过在所有角色转换中显示在本地系统上处理或创建的完整重做日志流序列来实现此目的。（在角色转换之后，将启动新的日志流，并且新的主数据库会增加日志流序列号。）。此视图仅适用于逻辑备用数据库。

柱 数据类型 空值 描述

STREAM_SEQUENCE＃

数

列出在本地系统上创建或应用的所有日志流的序列号。

注意 ：值为0表示未知序列顺序;这是为将来的日志流保留的。

状态

VARCHAR2（16）

日志流处理的描述：

  • Past - 日志流已处理完毕

  • Immediate Past - 这是最近处理的日志流;它的状态从过渡Current到Past

  • Current - 当前正在处理日志流

  • Immediate Future - 这是要处理的下一个日志流;它的地位正在从Future转变为Current

  • Future - 将处理日志流

资源

VARCHAR2（5）

描述日志流的启动方式：

  • RFS - RFS流程创建了日志流

  • User - 用户注册了日志流的初始日志文件

  • Synch - 用户发出ALTER DATABASE START LOGICAL STANDBY APPLY NEW PRIMARY DDL语句

  • Redo - 日志流信息记录在重做日志中

DBID

数

创建日志流的主数据库的数据库标识符

FIRST_CHANGE＃

数

当前日志文件中的最低系统更改编号（SCN）

最后一次变更＃

数

当前日志文件中的最高系统更改编号（SCN）

第一次

日期

当前日志文件中第一个SCN条目（ FIRST_CHANGE# ）的时间

上次

日期

当前日志文件中最后一个SCN条目的时间（ LAST_CHANGE# ）

DGNAME

VARCHAR2（255）

生成日志流的主数据库的唯一数据库名称（ DB_UNIQUE_NAME ）。请参阅V$DATAGUARD_CONFIG以显示Data Guard配置中定义的所有数据库DB_UNIQUE_NAME值。

MERGE_CHANGE＃

数

在关联的重做日志流的上下文中，应该用于闪回故障主节点（创建日志流）或闪回故障转移后的旁观者逻辑备用数据库的SCN。使用从主数据库接收的归档日志的本地副本，可以在所有数据库中安全地合并关联日志流的重做的SCN。为了在故障转移后应用超出此范围的更改，您需要从故障转移目标中获取并挖掘重做日志。

PROCESSED_CHANGE＃

数

严格限制SCN上限，SQL Apply在切换到新日志流之前已应用重做记录（因为它已被激活并成为主数据库，或者在旁观者逻辑备用数据库的情况下切换到新的日志流以容纳新的主数据库）。
