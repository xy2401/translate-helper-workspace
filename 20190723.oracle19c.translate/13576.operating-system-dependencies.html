A.1.1区分大小写

虽然标准的FORTRAN字符集不包括小写字母字符，但许多编译器允许它们使用标识符，注释和引用的文字。

Pro * FORTRAN预编译器不区分大小写;但是，有些编译器是。如果编译器区分大小写，则必须以相同的大写/小写格式声明和引用变量。检查您的FORTRAN编译器用户指南。

A.1.2编码区

您必须在第7列到第72列中编写EXEC SQL和EXEC ORACLE语句（忽略第73到80列）。其他列用于以下目的：第1列表示注释行，第1列到第5列包含可选语句标签，第6列表示连续行。

在某些系统上，支持终端格式 ;也就是说，条目不限于某些列。检查Oracle系统特定的文档。

一行中只能出现一个语句。

A.1.3延续线

您可以根据FORTRAN的规则将SQL语句从一行继续到下一行。要编写连续行，请在第6列中放置非零非空字符。在本手册中，数字用作连续字符，如以下示例所示：

*检索员工数据。EXEC SQL SELECT EMPNO，ENAME，JOB，SAL 1 INTO：MYEMPNO，：MYENAME，：MYJOB，：来自EMP 3的MYSAL 2，其中DEPTNO =：MYDEPTNO

您还可以将字符串文字从一行继续到下一行。通过第72列对文字进行编码，然后在下一行中编写一个连续字符和文字的其余部分。一个例子如下：

*执行动态SQL语句。EXEC SQL EXECUTE IMMEDIATE'更新EMP设置COMM = 500 WHERE 1 DEPTNO = 20'

大多数FORTRAN实现允许多达19个连续行。查看您的FORTRAN语言用户指南。

A.1.4 FORTRAN版本

Pro * FORTRAN预编译器支持操作系统的FORTRAN标准实现（通常为FORTRAN 77）。检查Oracle系统特定的文档。

如何声明和命名主机变量取决于您使用的FORTRAN编译器。有关声明和命名主机变量的详细信息，请查看FORTRAN用户指南。

A.1.6命名

主机变量名称必须仅包含字母和数字，并且必须以字母开头。它们可以是任意长度，但只有前31个字符才有意义。有些编译器禁止长度超过6个字符的变量名，或者在第6个字符后忽略字符。检查您的FORTRAN编译器用户指南。

A.1.7包含声明

您可以包含任何文件。预编译Pro * FORTRAN程序时，每个EXEC SQL INCLUDE语句将替换为语句中指定的文件的副本。

如果您的系统使用文件扩展名但未指定文件扩展名，则Pro * FORTRAN预编译器将采用源文件的默认扩展名（通常为FOR或F）。默认扩展名取决于系统。检查Oracle系统特定的文档。

如果系统使用目录，则可以通过指定预编译器选项INCLUDE = path来为INCLUDEd文件设置目录路径 。必须使用INCLUDE为非标准文件指定目录路径，除非它们存储在当前目录中。指定目录路径的语法是特定于系统的。检查Oracle系统特定的文档。

A.1.8 MAXLITERAL默认值

使用MAXLITERAL预编译器选项，您可以指定预编译器生成的字符串文字的最大长度，以便不超过编译器限制。MAXLITERAL默认值为1000，但您可能必须指定较低的值。

例如，如果FORTRAN编译器无法处理长度超过512个字符的字符串文字，请指定“MAXLITERAL = 512”。检查您的FORTRAN编译器用户指南。有关MAXLITERAL选项的更多信息，请参见MAXLITERAL 。

A.3.1 SQLADR

如果FORTRAN编译器生成CHARACTER变量的描述符并将描述符地址（而不是数据地址）传递给SQLADR，则不能将CHARACTER变量与SQLADR一起使用。检查您的FORTRAN编译器用户指南。在这种情况下，SQLADR获取错误的地址。相反，使用LOGICAL * 1变量，因为它们总是具有简单的地址。

但是，如果编译器提供了访问数据地址的内置函数，则可以（谨慎地）使用带有CHARACTER变量的SQLADR。例如，如果编译器提供名为％REF的函数，而X是CHARACTER变量，则按如下方式调用SQLADR：

*使用％REF内置功能。CALL SQLADR（％REF（X），...）
