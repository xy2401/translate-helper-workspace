DROP PACKAGE

目的

包使用PL / SQL定义。有关创建，更改和删除程序包的完整信息，请参阅Oracle Database PL / SQL语言参考 。

使用DROP PACKAGE语句从数据库中删除存储的包。该声明删除了包的正文和规范。

注意：

不要使用此语句从包中删除单个对象。相反，使用带有OR REPLACE子句的CREATE PACKAGE和CREATE PACKAGE BODY语句重新创建没有对象的包。

先决条件

程序包必须位于您自己的架构中，或者您必须具有DROP ANY PROCEDURE系统特权。

句法

drop_package :: =


说明drop_package.eps的说明

语义

身体

指定BODY仅删除包的主体。如果省略此子句，则Oracle数据库将删除程序包的正文和规范。

如果仅删除包的主体但不删除其规范，则数据库不会使依赖对象无效。但是，在重新创建包体之前，不能调用包规范中声明的过程或存储函数之一。

模式

指定包含该包的模式。如果省略schema ，则数据库假定程序包在您自己的模式中。

包

指定要删除的包的名称。

Oracle数据库使依赖于包规范的任何本地对象无效。如果随后引用其中一个对象，则数据库会尝试重新编译该对象，如果尚未重新创建已删除的包，则会返回错误。

如果任何统计类型与包关联，则数据库将使用FORCE子句取消关联统计类型，并删除使用统计类型收集的任何用户定义的统计信息。

例子

删除包：示例

以下语句删除了emp_mgmt包的规范和主体，使依赖于规范的所有对象无效。有关创建此程序包的示例，请参阅Oracle Database PL / SQL语言参考 。 

DROP PACKAGE emp_mgmt;
