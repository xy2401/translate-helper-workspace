将早期Oracle数据库版本与Oracle Grid Infrastructure一起使用

有关将早期Oracle数据库版本与Oracle Grid Infrastructure 19c安装一起使用的信息，请查看以下主题：

  • 使用早期Oracle数据库版本的一般限制
    您可以将Oracle Database 19c，18c，Oracle Database 12c第1版和第2版以及Oracle Database 11g第2版（11.2.0.3或更高版本）与Oracle Grid Infrastructure 19c一起使用。
  • 在Oracle ACFS上配置早期版本Oracle数据库
    查看此信息以在Oracle自动存储管理群集文件系统（Oracle ACFS）上配置11.2版Oracle数据库。
  • 使Oracle ASM适用于早期的Oracle数据库版本
    要将Oracle ASM与Oracle Database 12c第2版（12.2）之前的Oracle数据库版本一起使用，必须固定所有群集节点。
  • 使用ASMCA管理磁盘组以进行早期数据库发布
    在Oracle Grid Infrastructure安装上安装早期Oracle数据库和Oracle RAC数据库时，使用Oracle ASM Configuration Assistant（ASMCA）创建和修改磁盘组。
  • 使用正确的LSNRCTL命令
    要使用lsnrctl命令管理Oracle Database 19c本地和扫描侦听器，请将$ORACLE_HOME环境变量设置为Oracle Grid Infrastructure主目录（Grid home）的路径。
