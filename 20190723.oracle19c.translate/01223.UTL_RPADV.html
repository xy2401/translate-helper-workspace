266 UTL_RPADV

UTL_RPADV程序包提供子程序，用于收集和分析分布式数据库环境中Oracle Replication组件的统计信息。此程序包使用Oracle Replication Performance Advisor来收集统计信息。

本章包含以下主题：

  • UTL_RPADV概述

  • DBMS_COMPARISON安全模型

  • UTL_RPADV操作说明

  • UTL_RPADV子程序摘要

266.1 UTL_RPADV概述

通过此程序包，您可以收集和分析有关性能或Oracle Replication组件的统计信息。您可以按需收集统计信息，也可以创建持续监视Oracle Replication性能的监视作业。

在Oracle Database 11g第2版（11.2）数据库上使用此程序包时，它可以监视Oracle数据库10g第2版（10.2）及更高版本的数据库。它无法在10.2版之前监视数据库。

266.2 DBMS_COMPARISON安全模型

可以通过将此程序包上的EXECUTE授予选定的用户或角色，或者通过向所选用户或角色授予EXECUTE_CATALOG_ROLE来控制此程序包的安全性。

如果程序包中的子程序是从存储过程中运行的，则必须直接为程序包授予运行子程序的用户EXECUTE特权。它不能通过角色授予。

要确保在此程序包中运行子程序的用户具有必要的权限，请在使用此程序包时配置Oracle Replication管理员并以Oracle Replication管理员身份进行连接。

266.3 UTL_RPADV操作说明

要使用此程序包，必须以Oracle Replication管理员身份连接到Oracle数据库，并在ORACLE_HOME中的rdbms/admin目录中运行utlrpadv.sql脚本。

utlrpadv.sql脚本创建以下表：

  • STREAMS $ _PA_COMPONENT表

  • STREAMS $ _PA_COMPONENT_LINK表

  • STREAMS $ _PA_COMPONENT_PROP表

  • STREAMS $ _PA_COMPONENT_STAT表

  • STREAMS $ _PA_CONTROL表

  • STREAMS $ _PA_DATABASE表

  • STREAMS $ _PA_DATABASE_PROP表

  • STREAMS $ _PA_MONITORING表

  • STREAMS $ _PA_PATH_BOTTLENECK表

  • STREAMS $ _PA_PATH_STAT表

  • STREAMS $ _PA_SHOW_COMP_STAT表

  • STREAMS $ _PA_SHOW_PATH_STAT表

Oracle Replication Performance Advisor在运行时填充这些表。

266.4.1 ALTER_MONITORING程序

此过程会更改当前用户提交的监视作业。

句法

UTL_RPADV.ALTER_MONITORING（区间IN NUMBER DEFAULT NULL，top_event_threshold IN NUMBER DEFAULT NULL，bottleneck_idle_threshold IN NUMBER DEFAULT NULL，bottleneck_flowctrl_threshold IN NUMBER DEFAULT NULL，retention_time IN NUMBER DEFAULT NULL）;

参数

表266-14 ALTER_MONITORING过程参数

参数 描述

间隔

每个Performance Advisor运行之间的时间量（以秒为单位）。最长为3600秒。

如果为NULL ，则不更改当前值。

top_event_threshold

确定是否收集最高等待事件统计信息的百分比。

对于收集等待事件统计信息的过程，等待事件的百分比必须大于此参数中指定的值。例如，如果指定了15 ，则仅收集值大于15％的等待事件。

如果为NULL ，则不更改当前值。

bottleneck_idle_threshold

确定Oracle Replication组件会话是否有资格根据其IDLE百分比进行瓶颈分析的百分比。

IDLE百分比必须小于或等于此参数中指定的值，以使Oracle Replication组件会话有资格进行瓶颈分析。例如，如果指定了50 ，则只有50％或更少时间闲置的组件才有资格进行瓶颈分析。

如果为NULL ，则不更改当前值。

bottleneck_flowctrl_threshold

确定Oracle Replication组件会话是否有资格根据其FLOW CONTROL百分比进行瓶颈分析的百分比。

FLOW CONTROL百分比必须小于或等于此参数中指定的值，以便Oracle Replication组件会话有资格进行瓶颈分析。例如，如果指定了50 ，则只有50％或更少时间暂停流量控制的组件才有资格进行瓶颈分析。

如果为NULL ，则不更改当前值。

保留时间

保留监测结果的小时数。

如果为NULL ，则不更改当前值。

如果为FALSE ，则图例不包含在输出中。

同时使用bgn_run_id和end_run_id指定要显示的Oracle Replication Performance Advisor运行范围。正数显示较早前的统计数据。负数表示后来向后运行的统计数据。

例如，如果bgn_run_id设置为1且end_run_id设置为10 ，则该过程将显示前十个Oracle Replication Performance Advisor运行的统计信息。

但是，如果bgn_run_id设置为-1并且end_run_id设置为-10 ，则该过程将显示最近运行的Oracle Replication Performance Advisor的统计信息。

266.4.6 START_MONITORING程序

此过程启动监视作业。

此过程运行COLLECT_STATS过程以收集有关分布式数据库环境中的Oracle Replication组件和子组件的统计信息。

注意：

这个程序提交。

也可以看看：

  • “ COLLECT_STATS程序 ”

句法

UTL_RPADV.START_MONITORING（job_name IN VARCHAR2 DEFAULT'STREAMS $ _MONITORING_JOB'，client_name IN VARCHAR2 DEFAULT NULL，query_user_name IN VARCHAR2 DEFAULT NULL，interval IN NUMBER DEFAULT 60，top_event_threshold IN NUMBER DEFAULT 15，bottleneck_idle_threshold IN NUMBER DEFAULT 50，bottleneck_flowctrl_threshold IN NUMBER DEFAULT 50， retention_time IN NUMBER DEFAULT 24）;

参数

表266-20 START_MONITORING过程参数

参数 描述

JOB_NAME

要创建的监视作业的名称。

客户名称

客户的名称。

query_user_name

将查询结果表的用户。

此过程向指定用户授予权限，以使用户能够查询结果表。

间隔

每个Performance Advisor运行之间的时间量（以秒为单位）。最长为3600秒。

指定的时间间隔用于COLLECT_STATS过程中的interval参数。

top_event_threshold

确定是否收集最高等待事件统计信息的百分比。

对于收集等待事件统计信息的过程，等待事件的百分比必须大于此参数中指定的值。例如，如果指定了15 ，则仅收集值大于15％的等待事件。

bottleneck_idle_threshold

确定Oracle Replication组件会话是否有资格根据其IDLE百分比进行瓶颈分析的百分比。

IDLE百分比必须小于或等于此参数中指定的值，以使Oracle Replication组件会话有资格进行瓶颈分析。例如，如果指定了50 ，则只有50％或更少时间闲置的组件才有资格进行瓶颈分析。

bottleneck_flowctrl_threshold

确定Oracle Replication组件会话是否有资格根据其FLOW CONTROL百分比进行瓶颈分析的百分比。

FLOW CONTROL百分比必须小于或等于此参数中指定的值，以便Oracle Replication组件会话有资格进行瓶颈分析。例如，如果指定了50 ，则只有50％或更少时间暂停流量控制的组件才有资格进行瓶颈分析。

保留时间

保留监测结果的小时数。

使用说明

以下是START_MONITORING过程的使用说明：

  • 运行过程的用户的要求

  • 完整监控工作名称

  • 监控工作的限制

运行过程的用户的要求

运行START_MONITORING过程的用户必须满足以下要求：

  • 用户必须能够访问包含Oracle Replication组件的每个数据库的数据库链接。

  • 必须使用DBMS_XSTREAM_AUTH.GRANT_ADMIN_PRIVILEGE过程为用户授予权限，并且每个数据库链接必须连接到已使用DBMS_XSTREAM_AUTH.GRANT_ADMIN_PRIVILEGE过程授予权限的远程数据库的用户。

完整监控工作名称

提交监视作业时，将连接客户端名称和作业名称以形成完整的监视作业名称。在运行START_MONITORING过程时，使用client_name参数指定客户端名称，使用job_name参数指定作业名称。Oracle Enterprise Manager提交的监视作业的客户端名称始终为EM 。

监控工作的限制

以下限制适用于监视作业：

  • 完整监视作业名称的长度限制为30个字节。

  • 即使监视作业是由不同的模式提交的，两个监视作业也不能具有相同的完整监视作业名称。名称检查不区分大小写。例如， strmSTREAMS$_MONITORING_JOB和STRMSTREAMS$_MONITORING_JOB被认为是同一名称。

  • Oracle Enterprise Manager每个数据库最多只能有一个监视作业。

  • 每个模式最多只能有一个监视作业。
