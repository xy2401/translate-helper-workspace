适用于DRDA的Oracle数据库网关概述

通过Oracle数据库网关DRDA，您可以使用一组应用程序访问Oracle数据和DB2数据，同时继续使用现有IBM应用程序访问DB2数据。网关使您能够：

  • 集成异构数据库管理系统，使它们显示为单个同构数据库系统。

  • 从Oracle应用程序读取和写入DB2 UDB for z / OS，DB2 Universal Database™for iSeries™（DB2 UDB for iSeries）和DB2 Universal Database中的数据。

Oracle Database 12 c第2版（12.2）为下一代Oracle数据库网关奠定了基础，该数据库网关通过利用Oracle数据库异构服务提供增强的集成功能。

作为Oracle数据库的集成组件， 异构服务可以利用Oracle数据库强大的SQL解析和分布式优化功能。此集成还确保网关可以立即利用对Oracle数据库未来版本所做的任何增强。有关Oracle异构服务的详细信息，请参阅“ Oracle数据库异构连接用户指南” 。

与先前版本相比，网关与Oracle Database 12 c第2版（12.2）的集成度更高，从而提高了性能和增强功能，同时仍提供Oracle和非Oracle数据的透明集成。例如，连接初始化信息在本地Oracle数据库中可用，从而减少了往返次数和通过网络发送的数据量。SQL执行也更快，因为应用程序发出的语句会被解析和翻译一次，然后可以被多个应用程序重用。

网关功能

Oracle Database Gateway for DRDA使您可以将异构系统集成到单个无缝环境中。如果数据从DRDA数据库移动到Oracle数据库，则不需要更改应用程序设计或功能。网关处理应用程序和数据库之间的数据类型和SQL函数的所有差异。因此，最终用户和应用程序员不需要知道数据的物理位置或存储特性。

这种透明性不仅可以使您无缝集成异构数据，还可以简化网关实施，应用程序开发和维护。网关功能如下：

所有级别的透明度

适用于DRDA的Oracle数据库网关为您提供以下级别的透明度：

  • 地点

    用户可以按名称访问表，而不需要知道表的物理位置。

  • 网络

    该网关利用Oracle Net技术，允许用户跨多个网络访问数据，而无需考虑网络架构。 TCP / IP 协议受支持。此版本支持Oracle数据库和网关之间以及网关和DB2服务器之间的IPV4和IPV6。

  • 操作系统

    用户可以访问存储在多个操作系统下的数据，而无需了解保存数据的操作系统。

  • 数据存储

    无论数据库或文件格式如何，网关都能够访问数据。

  • 访问方法

    您可以对任何数据存储使用单一的SQL方言，从而无需为特定于数据库的访问方法或SQL实现进行编码。

扩展数据库服务

以下是一些通过网关提供的Oracle数据库服务：

  • SQL函数

    您的应用程序可以使用Oracle SQL访问所有数据。网关与Oracle数据库集成的方法可确保每个数据库版本的最新功能始终可立即用于网关。

  • 分布式功能

    异构数据可以无缝集成，因为Oracle分布式功能，例如 JOIN和 UNION可以应用于非Oracle数据，无需任何特殊编程或映射。

  • 分布式查询优化

    Oracle数据库可以利用其高级查询优化技术来确保对任何数据有效地执行SQL语句。本地和数据的数据分布和存储特性远程数据同样被考虑。

  • 两阶段提交保护

    Oracle数据库两阶段提交机制通过确保跨越数据存储的事务仍被视为单个工作单元，提供跨数据存储的一致性。除非可以在将受影响的所有数据存储中提交更改，否则不会在任何数据存储中提交（或永久存储）更改。

  • 存储过程和数据库触发器

    可以使用相同的Oracle存储过程和数据库触发器来访问所有数据，从而确保在整个企业中统一实施业务规则。

扩展的高级网络，Internet和Intranet支持

网关与Oracle数据库的集成将Oracle Internet和Oracle Net软件的优势扩展到非Oracle数据，并扩展了Oracle客户端/服务器和服务器/服务器连接软件的优势。这些功能包括：

  • 应用服务器支持

    任何可以访问Oracle数据库中的数据的Internet或Intranet应用程序也可以合并来自可通过网关访问的数据存储的信息。Web浏览器可以使用任何支持Oracle软件的应用程序服务器产品连接到Oracle数据库。

  • 含蓄协议转变

    Oracle和Oracle Net可以一起工作协议转换器，允许应用程序透明地访问不支持客户端网络协议的平台上的其他数据存储。Oracle数据库可以使用 TCP / IP 与网关和另一个数据存储进行通信。

  • 高级安全性

    在传输到客户端期间，可以保护非Oracle数据免受未经授权的访问或篡改。您可以通过使用独立于硬件和协议无关的加密和校验和服务高级安全性。

  • 无线通信

    Oracle Mobile 代理是Oracle业界领先的移动技术，可以与Oracle数据库或可通过网关访问的任何数据库进行无线通信。这使您的现场人员可以直接从移动便携式计算机访问企业数据。

动态字典映射

网关的简单设置不需要任何额外的映射。在应用程序可以访问任何信息之前，必须告知应用程序数据的结构，例如表的列及其长度。许多产品要求管理员在存储在集线器中的单独数据字典中手动定义该信息。然后，应用程序使用中心字典而不是每个数据库的本机字典来访问信息。这种方法需要您进行大量的手动配置和维护。作为管理员，无论何时结构都必须更新集线器中的数据字典远程表已更改。

Oracle Database Gateway for DRDA不需要高效复制。网关使用每个数据库的现有本机词典。您的应用程序使用专为每个数据库设计的字典访问数据，这意味着不需要创建或维护冗余字典。

SQL

Oracle Database Gateways允许您使用统一的SQL集访问任何数据，从而简化了应用程序的开发和维护。对位置，存储特征或表结构的更改不需要对应用程序进行任何更改。 ANSI和支持ISO标准SQL以及强大的Oracle扩展。

数据定义语言

Oracle Applications可以使用本机数据定义语言（DDL）语句在目标数据存储中创建表。

数据控制语言

您可以从Oracle环境发出本机数据控制语言（DCL）语句，允许集中管理异构数据存储的用户权限和访问级别。

Passthrough和Native DB2 SQL

可以通过网关传递本机DB2 SQL的执行，以直接针对DB2执行。这使应用程序能够将语句（例如DB2 CREATE TABLE ）发送到网关，以便在目标DB2系统上执行 。

存储过程

通过该网关，您可以利用Oracle和非Oracle存储过程，充分利用您在分布式多数据库环境中的投资。Oracle存储过程可以轻松访问多个数据存储，而无需对异构数据访问进行任何特殊编码。

Oracle存储过程

Oracle存储过程使您可以访问和更新使用存储在Oracle数据库中的集中业务规则的DB2数据。使用Oracle存储过程可以通过最小化网络流量来提高数据库性能。应用程序可以发送单个EXECUTE命令来开始整个PL / SQL例程，而不是通过网络发送单个SQL语句。

本机DB2存储过程

网关可以使用标准Oracle PL / SQL执行DB2存储过程 。 Oracle应用程序执行DB2存储过程，就像它是Oracle一样远程程序。

语言

任何支持Oracle数据库的应用程序或工具都可以通过Oracle网关访问30多个不同的数据源。即使数据以传统的专有格式存储，也可以使用Oracle公司和第三方供应商提供的各种开放系统工具。支持数百种工具，包括即席查询工具，Web浏览器，交钥匙应用程序和应用程序开发工具。

Oracle数据库技术和工具

该网关集成到Oracle数据库技术中，该技术提供全局查询优化，多站点事务的事务协调以及对所有人的支持 Oracle Net配置。支持Oracle数据库的工具和应用程序可用于通过网关访问异构数据。

在SQL * Plus

您可以使用SQL * Plus在数据库之间移动数据。该产品使您能够将数据从部门数据库复制到企业Oracle数据库实例。

两阶段提交和多站点事务

网关可以作为合作伙伴参与多站点事务和两阶段提交。如何发生这取决于底层数据源的功能，这意味着网关可以实现为以下任何一种：

  • 完整的两阶段提交合作伙伴

  • 提交点网站

  • 单站点更新合作伙伴

  • 只读合作伙伴

实现网关的决定因素是目标数据库的锁定和事务处理功能。

默认情况下，Oracle Database Gateway for DRDA配置为提交点站点，即commit confirm protocol 。可选 ，如果选择通过网关强制执行只读功能，则可以将网关配置为只读。不支持其他协议。

网站自治

所有Oracle数据库产品（包括网关）都提供站点自治权。例如，管理数据源仍然是原始系统管理员的责任。站点自治也起作用，网关产品不会覆盖由数据源或操作环境建立的安全措施。

迁移与共存

通过网关集成数据源不需要对数据源上的应用程序进行任何更改。结果是Oracle数据库技术是非侵入式的，提供了共存和简单的迁移路径。

安全

网关不绕过现有的安全机制。网关安全性与已在数据源的操作环境中使用的安全机制共存。

从功能上讲，网关安全性与Oracle数据库的安全性相同，如“ Oracle数据库安全指南”中所述 。Oracle数据库安全性映射到数据源的数据字典。

DRDA UDB服务器加密支持

此版本的Gateway for DRDA提供完整的UDB服务器加密支持。有关各种选项，请参阅以下新参数：

  • HS_FDS_AUTHENTICATE_USER

  • HS_FDS_ENCRYPT_SESSION

  • HS_FDS_TRUSTSTORE_FILE

  • HS_FDS_TRUSTSTORE_PASSWORD

    DRDA服务器必须位于主机可通过网络访问的系统上。

履行

当网关安装在主机上时，它与主机上的Oracle数据库实例具有一些相同的组件。网关具有以下组件：

  • 基本文件目录，类似于与Oracle实例ORACLE_HOME环境变量关联的目录

  • 网关系统标识符（SID），与Oracle实例ORACLE_SID相当

  • Oracle Net支持Oracle数据库与用于DRDA的Oracle数据库网关之间的通信

网关没有：

  • 控制，重做日志或数据库文件

  • 与已安装的Oracle数据库关联的完整子目录和辅助文件集

由于网关没有后台进程且不需要管理实用程序（例如Oracle Enterprise Manager），因此您无需启动网关产品。访问特定网关的每个Oracle数据库用户会话都会在主机上创建一个独立的进程。此过程运行网关会话并执行与DRDA服务器通信的网络操作。

网关如何工作

网关没有自己的数据库功能。相反，它提供了一个接口 Oracle数据库可以将部分或全部SQL操作定向到DRDA数据库。

使用数据库链接向Oracle数据库标识支持DRDA服务器的网关。数据库链接是用于标识其他Oracle数据库的相同构造。DRDA服务器上的表在SQL中引用为：

TABLE_NAME @ dblink_name

要么

owner.table_name@dblink_name

如果您创建同义词或者在Oracle数据库中查看视图，然后您可以使用简单名称来引用DRDA服务器上的表，就好像该表是Oracle数据库的本地表一样。

当Oracle数据库遇到对DRDA服务器上的表的引用时，SQL语句的适用部分将发送到网关进行处理。与SQL语句关联的任何主机变量都绑定到网关，因此绑定到DRDA服务器。

网关负责将这些SQL语句发送到DRDA服务器以执行以及部署和返回响应。响应是数据或消息。Oracle数据类型和DRDA数据类型之间的任何转换都由网关执行。Oracle数据库和应用程序都只读取和处理Oracle数据类型。

SQL差异

并非所有SQL实现都是相同的。Oracle数据库支持比当前通过网关访问的数据库更大的内置函数集。Oracle数据库和网关一起工作将SQL转换为与特定DRDA服务器兼容的表单。

在此转换期间，Oracle数据库函数可以转换为特定DRDA服务器可识别的函数。例如，Oracle数据库NVL函数将转换为DB2 VALUE函数。

或者，Oracle数据库包含DRDA服务器无法执行的功能，并在从DRDA数据库中提取行后执行这些功能。此处理仅适用于SELECT语句。Oracle数据库和网关无法对UPDATE ， INSERT或DELETE语句执行此类操作，因为这样做会更改事务语义。

Oracle工具和网关

使用Oracle Database Gateway运行读取和写入存储在DRDA数据库中的数据的应用程序（如Oracle数据库工具）。

虽然用于DRDA的Oracle数据库网关不提供新的应用程序或开发工具，但它扩展了现有Oracle数据库工具的范围，以将数据包含在支持DRDA的非Oracle数据库中。

将Oracle Database Gateway与其他Oracle产品一起使用可以极大地扩展独立网关的功能。

在SQL * Plus

使用SQL * Plus和用于DRDA的Oracle数据库网关创建分布式数据库系统，提供易于使用的传输工具，用于在分布式数据库之间移动数据。一种可能的用途是将公司Oracle数据库中的数据分发到部门DRDA数据库。您还可以将公司DRDA数据库中的数据分发到部门Oracle数据库。

异构服务架构

适用于DRDA的Oracle数据库网关利用Oracle数据库中的Oracle异构服务组件。异构服务是下一代Oracle数据库网关的构建块。

有关异构服务的详细信息，请参阅“ Oracle数据库异构连接用户指南” 。

性能增强

用于DRDA的Oracle数据库网关包含多项内部性能增强功能。与版本10网关相比，该产品显示了各种工作负载的所有相关地址空间的响应时间和CPU利用率的重大改进。您站点的实际性能改进可能会有所不同，具体取决于您的安装类型和工作负载。

获取重新阻止

该应用程序的数组大小 SELECT在应用程序和Oracle数据库之间有效。但是，阵列块大小和Oracle数据库与网关之间的块提取由两个异构服务初始化参数控制： HS_RPC_FETCH_SIZE和HS_RPC_FETCH_REBLOCKING 。这些参数在网关初始化文件中指定。有关更多信息，请参阅“ Oracle数据库异构连接用户指南” 。

支持Oracle数据库Passthrough

您可以使用Oracle数据库DBMS_HS_PASSTHROUGH.EXECUTE_IMMEDIATE方法通过网关传递DRDA服务器中可用的命令或语句。

通过Passthrough检索结果集

用于DRDA的Oracle数据库网关提供了一种从通过passthrough发出的SELECT语句中检索结果集的工具。有关其他信息，请参阅“ 通过直通检索结果集 ” 。

支持TCP / IP

此版本的网关仅支持网关和DRDA服务器之间的TCP / IP通信协议。请参阅POWER Systems（64位）上的IBM AIX上的Oracle Database Gateway安装和配置指南，Linux x86-64，SPARC上的Oracle Solaris（64位），x86-64（64位）上的Oracle Solaris和HP- UX Itanium或适用于Microsoft Windows的Oracle Database Gateway安装和配置指南，具体取决于您的平台。

原生语义

此版本的网关支持DRDA服务器有选择地启用或禁用各种SQL功能的后处理功能。有关详细信息，请参阅“本机语义 ” 。

结果集中支持的列

用于DRDA的Oracle数据库网关支持最多1000列结果集。

EXPLAIN_PLAN改进

EXPLAIN_PLAN表包含通过网关从Oracle数据库传递到DRDA服务器的实际SQL语句。

异构数据库集成

网关支持 ANSI标准SQL支持对DRDA数据库的读/写访问。即使您的数据存在于不同应用程序的不同平台上，新应用程序也可以使用所有数据，而不管位置如何。

对现有系统的最小影响

网关不需要在OS / 390（MVS），AS / 400，基于UNIX或Microsoft Windows目标系统上安装其他Oracle软件。它使用的数据库接口由IBM提供，并内置于DRDA数据库产品和这些平台上已存在的网络设施。

配置IBM系统以进行DRDA访问通常包括定义所涉及的网络资源以及建立特定于目标数据库的访问安全性定义。

应用程序可移植性

网关与异构数据库交互的能力使得开发可以用于Oracle和IBM数据库以及Oracle为其提供网关的任何其他数据库的单组可移植应用程序成为可能。

远程数据访问

由于网关架构允许每个组件之间的网络连接，因此最大化了位置灵活性。应用程序可以使用Oracle客户端 - 服务器功能通过Oracle Net连接到远程Oracle数据库。 Oracle数据库可以使用数据库链接连接到远程网关。网关通过网络设施连接到DRDA服务器。

远程访问的好处是：

  • 提供将适当资源分配给给定任务的方法

    例如，您可以将应用程序开发从昂贵的处理器转移到具有成本效益的工作站或微型计算机上。

  • 扩展可用数据源的数量

    如果没有远程访问，则仅限于本地环境中可用的数据。通过远程访问，只有您的网络限制您的数据源。

  • 提供为给定用户定制应用程序环境的方法

    例如，一些用户更喜欢块模式终端环境，而另一些用户更喜欢位映射的图形驱动终端环境。远程访问可以满足这两个要求，因为您不受数据位置强加的接口环境的限制。

支持分布式应用程序

由于网关使您的应用程序可以直接访问DRDA数据，因此无需将大量数据库数据上载和下载到其他处理器。相反，您可以在需要时访问数据，而无需在计算机之间移动数据，并且可能存在未同步和不一致的数据。避免大规模数据复制还可以降低所有系统的聚合磁盘存储要求。

但是，如果您的系统设计需要在网络中的计算机之间移动数据，则SQL * Plus和网关可以简化数据传输。使用单个SQL * Plus命令，您可以将整个数据集从网络的一个节点移动到另一个节点，从一个数据库移动到另一个数据库。

您可以通过网关传递特定于DRDA数据库的命令和语句，以便由DRDA数据库执行。例如，您可以通过网关传递本机DB2 SQL以执行DB2。您还可以执行在非Oracle数据库中定义的存储过程。

应用程序开发和最终用户工具

通过该网关，Oracle扩展了可用于访问IBM数据库的应用程序开发和最终用户工具的范围。这些工具通过减少原型，开发和维护时间来提高应用程序开发和用户生产力。当前的Oracle数据库用户无需学习一组新工具来访问存储在DRDA数据库中的数据。相反，他们可以使用一组工具访问Oracle数据库和DRDA数据。

利用Oracle提供的网关和应用程序开发工具，您可以开发一组应用程序来访问Oracle数据库和DRDA数据。用户可以使用Oracle提供的决策支持工具来访问Oracle数据库和DRDA数据。这些工具可以在通过其连接的远程机器上运行 Oracle Net到Oracle数据库。

在设计应用程序时，请记住网关是为检索和相对较轻的事务负载而设计的。网关目前不是一个繁重的事务处理系统。

密码加密实用程序

用于DRDA的Oracle数据库网关包括一个实用程序，用于支持在网关初始化文件中加密纯文本密码。有关POWER系统（64位）上的IBM AIX的Oracle Database Gateway安装和配置指南，Linux x86-64，SPARC上的Oracle Solaris（64位），x86-64上的Oracle Solaris，请参阅第15章“安全注意事项”。有关详细信息，请参阅（64位）和 适用于Microsoft Windows的 HP-UX Itanium或Oracle Database Gateway安装和配置指南 。

支持DB2 UDB for z / OS存储过程

用于DRDA的Oracle数据库网关支持本机存储过程。

IBM DB2通用数据库支持

适用于DRDA的Oracle Database Gateway支持IBM DB2 通用数据库。

DB2 z / OS ASCII和UNICODE表支持

用于DRDA的Oracle数据库网关支持DB2 z / OS的EBCDIC，ASCII和UNICODE表。在表创建期间定义字符集选择。

只读支持

当前版本允许将网关配置为只读网关。在此模式下，用户将无法在DRDA数据库中修改数据或调用远程过程。

支持图形和多字节数据

当前版本的网关添加了对DB2 GRAPHIC和VARGRAPHIC数据类型的支持。有关更多信息，请参阅开发应用程序

支持英特尔硬件上的DB2通用数据库

当前版本的网关增加了对在Intel Windows和Linux上运行的DRDA服务器的支持。

DB2通用数据库的数据字典支持

当前版本的网关还为DB2 UDB提供了Oracle数据字典支持。
