7.64 V $ ASM_DISKGROUP

V$ASM_DISKGROUP为节点上的Oracle ASM实例发现的每个Oracle自动存储管理（Oracle ASM）磁盘组显示一行。

柱 数据类型 描述

GROUP_NUMBER

数

分配给磁盘组的群集范围编号（主键）

名称

VARCHAR2（30）

磁盘组的名称

SECTOR_SIZE

数

物理块大小（以字节为单位）

LOGICAL_SECTOR_SIZE

数

此列表示磁盘组的逻辑扇区大小值（以字节为单位）。这是可在此磁盘组中完成的最小I / O.任何小于逻辑扇区大小的I / O都会在代码中产生断言。

如果LOGICAL_SECTOR_SIZE列的值为512 ，而SECTOR_SIZE列中的值为4096 ，则磁盘组支持512扇区大小仿真。这意味着I / O操作可以是512或4096对齐。例如，这意味着您可以在此磁盘组中具有文件块大小为512字节的重做日志文件（可以读取和写入），以及创建文件块大小为4096字节的新重做日志文件。

BLOCK_SIZE

数

Oracle ASM元数据块大小（以字节为单位）

ALLOCATION_UNIT_SIZE

数

分配单元的大小（以字节为单位）

州

VARCHAR2（11）

磁盘组相对于实例的状态：

  • CONNECTED - 数据库实例正在使用磁盘组

  • BROKEN - 数据库实例失去与挂载磁盘组的Oracle ASM实例的连接

  • UNKNOWN - Oracle ASM实例从未尝试装入磁盘组

  • DISMOUNTED - 成功安装后，Oracle ASM实例彻底卸载了磁盘组

  • MOUNTED - 实例已成功将磁盘组提供给其数据库客户端

  • QUIESCING - CRSCTL实用程序尝试卸载包含Oracle Cluster Registry（OCR）的磁盘组。在群集就绪服务（CRS）退出之前，无法卸载磁盘组，因为磁盘组包含OCR。

类型

VARCHAR2（6）

磁盘组的冗余类型：

  • EXTERN

  • 正常

  • 高

TOTAL_MB

数

磁盘组的总容量（以兆字节为单位）

FREE_MB

数

磁盘组的未使用容量（以兆字节为单位）

HOT_USED_MB

数

热区域中使用的兆字节数

COLD_USED_MB

数

寒冷地区使用的兆字节数

REQUIRED_MIRROR_FREE_MB

数

在给定磁盘组中为了在一个或多个磁盘故障后恢复冗余而需要可用的空间量。此列中显示的空间量会考虑镜像效果。

USABLE_FILE_MB

数

可以安全使用的可用空间量，将镜像考虑在内，并且能够在磁盘发生故障后恢复冗余

OFFLINE_DISKS

数

磁盘组中当前处于脱机状态的磁盘数

兼容性

VARCHAR2（60）

安装此磁盘组的Oracle ASM实例所需的最低软件版本

DATABASE_COMPATIBILITY

VARCHAR2（60）

数据库实例使用此磁盘组中的文件所需的最低软件版本

VOTING_FILES

VARCHAR2（1）

指示磁盘是否包含投票文件（ Y ）（ N ）

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

对于此视图，该值始终为0 。

注意：

GROUP_NUMBER ， TOTAL_MB和FREE_MB列仅在实例装入磁盘组时才有意义。否则，它们的值将为0 。
