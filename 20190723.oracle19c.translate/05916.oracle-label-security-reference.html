ALL_SA_AUDIT_OPTIONS查看

ALL_SA_AUDIT_OPTIONS数据字典视图根据SA_AUDIT_ADMIN.AUDIT过程设置显示当前用户的Oracle Label Security审核选项。

请参见SA_AUDIT_ADMIN.AUDIT 。

此视图显示审计是否配置为每个会话（ BY SESSION ）或每个访问（ BY ACCESS ）以及成功或不成功操作生成审计记录。可能的值如下：

  • 短划线（ - ）表示未设置审核选项。

  • S字符表示审计选项BY SESSION设置。

  • A字符表示审计选项是BY ACCESS设置的。

  • 每个审计选项都有两个可能的设置， WHENEVER SUCCESSFUL和WHENEVER NOT SUCCESSFUL ，用斜杠（ / ）分隔。

例如，在以下输出中，将使用BY ACCESS审核类型审核用户jjones ，以获取涉及特定于策略的特权的成功操作。使用BY SESSION审核类型审核用户rlayton ：审核记录是针对失败的尝试删除策略以及成功尝试设置用户授权而编写的。 

SELECT * FROM DBA_SA_AUDIT_OPTIONS; POLICY_NAME USER_NAME APY REM SET_ PRV ----------- ------------ --- ---- ---- --- HR_OLS_POL JJONES  -  /  -   -  / -   -  /  -  A /  -  HR_OLS_POL RLAYTON  -  /  -   -  / SS /  -   -  /  -

  • G3是G2的子组， G2是G1的子组。
