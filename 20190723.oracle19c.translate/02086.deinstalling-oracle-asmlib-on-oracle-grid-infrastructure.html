在Oracle Grid Infrastructure上卸载Oracle ASMLIB

如果安装了Oracle ASM库驱动程序（Oracle ASMLIB）但您没有将其用于设备路径持久性，则卸载Oracle ASMLIB。

  1. 以root身份登录。
  2. 停止Oracle ASM以及节点上任何正在运行的数据库实例：
    $ srvctl stop instance -d db_unique_name -node node_name $ srvctl stop asm -node node_name
    要停止节点上的最后一个Oracle Flex ASM实例，请停止Oracle Clusterware堆栈：
    Grid_home / bin / crsctl停止crs
  3. 停止Oracle ASMLIB：
    /etc/init.d/oracleasm禁用
  4. 删除oracleasm库和工具RPM：
    #rpm -e oracleasm-support
    #rpm -e oracleasmlib
  5. 检查是否还有任何oracleasm RPM：
    #rpm -qa | grep oracleasm
  6. 如果剩余任何oracleasm配置文件，请将其删除：
    #rpm -qa | grep oracleasm | xargs rpm -e
    现在删除了Oracle ASMLIB和相关的RPM。

启动Oracle Clusterware堆栈。（可选）您可以在启动Oracle Clusterware堆栈之前安装和配置Oracle ASM筛选器驱动程序（Oracle ASMFD）。
