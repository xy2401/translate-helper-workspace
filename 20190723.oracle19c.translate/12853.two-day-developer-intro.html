1.2.1关于模式对象

Oracle数据库中的每个对象只属于一个模式，并且具有该模式的唯一名称。

模式可以包含的一些对象是：

  • 表

    表是Oracle数据库中数据存储的基本单位。表包含所有用户可访问的数据。每个表都包含表示各个数据记录的 行 。行由表示记录字段的列组成。

  • 索引

    索引是可选对象，可以提高从表中检索数据的性能。索引在表的一列或多列上创建，并自动在数据库中维护。

  • 查看

    您可以创建一个视图，将来自多个不同表的信息组合到一个演示文稿中。视图可以依赖来自表和其他视图的信息。

  • 序列

    当表的所有记录必须是不同的时，您可以使用序列为数字列生成唯一整数的序列列表，每个整数表示一个记录的ID。

  • 同义词

    同义词是架构对象的别名。为了安全和方便，您可以使用同义词;例如，隐藏对象的所有权或简化SQL语句。

  • 存储的子程序

    存储的子程序（也称为模式级子程序 ）是存储在数据库中的过程和函数。可以从访问数据库的客户端应用程序调用它们。

    触发器是存储的子程序，当特定表或视图中发生指定事件时，这些子程序由数据库自动运行。触发器可以限制对特定数据的访问并执行日志记录。

  • 包

    包是一组相关的子程序，以及它们使用的显式游标和变量，作为一个单元存储在数据库中，以便继续使用。与存储的子程序一样，可以从访问数据库的客户端应用程序调用包子程序。

通常，应用程序使用的对象属于同一模式。

1.2.2.3关于结构化查询语言（SQL）

SQL是一种声明性或非过程性语言;也就是说，它描述了要做什么，而不是如何做。您可以指定所需的结果集（例如，当前员工的名称），但不指定如何获取它。

1.2.2.4关于过程语言/ SQL（PL / SQL）

过程语言/ SQL（PL / SQL） （发音为PL sequel ）是SQL的本机Oracle数据库扩展。它通过添加程序元素（例如条件控制和循环）来弥合声明性和命令式程序控制之间的差距。

在PL / SQL中，您可以声明常量和变量，过程和函数，这些类型的类型和变量以及触发器。您可以处理异常（运行时错误）。您可以创建存储在数据库中的PL / SQL单元 - 过程，函数，包，类型和触发器，以供使用任何Oracle数据库编程接口的应用程序重用。

PL / SQL源程序的基本单元是块，它对相关的声明和语句进行分组。块具有可选的声明部分，必需的可执行部分和可选的异常处理部分。

Oracle Data Provider for .NET（ODP.NET）提供从.NET应用程序到Oracle数据库的快速有效的ADO.NET数据访问。ODP.NET允许开发人员利用Oracle数据库中存在的高级Oracle数据库功能，包括SecureFiles，XML DB和高级队列。

适用于Visual Studio的Oracle开发人员工具（ODT）

用于Visual Studio的Oracle Developer Tools（ODT）是一组与Visual Studio环境集成的应用程序工具。这些工具提供对Oracle功能的图形用户界面访问，使用户能够执行各种应用程序开发任务，并提高开发效率和易用性。Oracle Developer Tools支持使用Visual Basic，C＃和其他.NET语言编写和实现.NET存储过程。

.NET存储过程

Oracle Database Extensions for .NET是Windows上Oracle Database的数据库选项。它使用Visual Basic .NET或Visual C＃在Oracle Database for Microsoft Windows中构建和运行.NET存储过程或函数成为可能。

在将.NET过程和函数构建到.NET程序集中之后，可以使用Oracle部署向导for .NET（Oracle Studio Tools for Visual Studio的一个组件）将它们部署到Oracle数据库中。

适用于ASP.NET的Oracle提供程序

Oracle Providers for ASP.NET为ASP.NET开发人员提供了一种在Oracle数据库中存储Web应用程序通用状态的简便方法。这些提供程序以现有Microsoft ASP.NET提供程序为模型，共享类似的架构和编程接口，为.NET开发人员提供熟悉的界面。Oracle支持成员资格，配置文件，角色和其他提供程序。
