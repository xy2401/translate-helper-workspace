ALTER TABLESPACE SET

注意：

只有在使用Oracle Sharding时，此SQL语句才有效。有关Oracle Sharding的更多信息，请参阅Oracle Database Administrator's Guide 。

目的

使用ALTER TABLESPACE SET语句更改现有表空间集的属性。属性更改将应用于表空间集中的所有表空间。

也可以看看：

创建TABLESPACE SET和DROP TABLESPACE SET

先决条件

您必须作为SDB用户连接到分片目录数据库。

如果您具有ALTER TABLESPACE系统特权，则可以执行任何ALTER TABLESPACE SET操作。如果您具有MANAGE TABLESPACE系统特权，则只能执行以下操作：

  • 获取在线或离线设置的表空间中的所有表空间

  • 开始或结束备份

  • 使表空间中的所有表空间都设置为只读或读写

  • 设置表空间中所有表空间的默认日志记录模式设置为LOGGING或NOLOGGING

  • 将所有表空间放在以强制日志记录模式设置的表空间中，或将它们置于强制日志记录模式之外

  • 调整表空间集的所有数据文件的大小

  • 启用或禁用表空间集的所有数据文件的自动扩展

在将表空间设置为只读之前，必须满足以下条件：

  • 表空间集中的表空间必须在线。

  • 表空间集不得包含任何活动的回滚段。此外，由于无法访问只读表空间集的回滚段，因此Oracle建议您在将表空间设置为只读之前删除回滚段。

  • 表空间集不得涉及开放备份，因为备份结束时会更新表空间集中所有数据文件的头文件。

句法

alter_tablespace_set :: =


说明alter_tablespace_set.eps的说明

alter_tablespace_attrs :: =


说明alter_tablespace_attrs.eps的描述

（请参阅ALTER TABLESPACE的以下子句： default_tablespace_params :: = ， size_clause :: = ， datafile_tempfile_clauses :: = ， tablespace_logging_clauses :: = ， tablespace_state_clauses :: = ， autoextend_clause :: = ， alter_tablespace_encryption :: = ）

语义

tablespace_set

指定要更改的表空间的名称。

alter_tablespace_attrs

使用此子句更改表空间集中所有表空间的属性。

alter_tablespace_attrs的子条款与ALTER TABLESPACE语句具有相同的语义，但有以下例外：

  • 您不能为表空间集指定以下子条款：

    • MINIMUM EXTENT size_clause

    • SHRINK SPACE [ KEEP size_clause ]

    • tablespace_group_clause

    • flashback_mode_clause

    • tablespace_retention_clause

  • 对于datafile_tempfile_clauses ，表空间集仅支持以下子条款：

    • RENAME DATAFILE

    • DATAFILE { ONLINE | OFFLINE }

  • 对于tablespace_state_clauses ，表空间集不支持PERMANENT和TEMPORARY子条款。

例子

更改表空间集：示例

以下语句将表空间集ts1中的所有表空间置于强制日志记录模式：

ALTER TABLESPACE SET ts1 FORCE LOGGING;
