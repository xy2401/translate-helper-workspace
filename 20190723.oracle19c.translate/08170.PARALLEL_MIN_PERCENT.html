1.251 PARALLEL_MIN_PERCENT

PARALLEL_MIN_PERCENT允许您指定并行执行所需的并行执行进程数的最小百分比。

属性 描述

参数类型

整数

默认值

0

可修改

改变会话

可在PDB中修改

没有

价值范围

0到100

基本

没有

Oracle RAC

多个实例可以具有不同的值。

当未启用并行语句排队时（当PARALLEL_DEGREE_POLICY设置为manual或limited ），此参数控制并行操作的行为。它确保操作始终获得并行执行服务器的最小百分比或错误输出。设置此参数可确保除非有足够的资源可用，否则不会执行并行操作。默认值0表示没有设置最小进程百分比。

请考虑以下设置：

PARALLEL_MIN_PERCENT = 50 PARALLEL_MIN_SERVERS = 5 PARALLEL_MAX_SERVERS = 10

如果10个并行执行进程中有8个忙，则只有2个进程可用。如果您随后请求并行度为8的查询，则不会满足最小50％。

您可以将此参数与PARALLEL_ADAPTIVE_MULTI_USER 。在多用户环境中，单个用户或应用程序可以将PARALLEL_MIN_PERCENT设置为最小值，直到系统上有足够的资源可用并返回可接受的并行度。
