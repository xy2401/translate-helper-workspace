23 SDO_CSW包（Web服务目录服务）

MDSIS.SDO_CSW包中包含与支持Web服务（CSW）相关的各种处理操作的子程序。

要使用本章中的子程序，您必须了解有关Web 目录服务（CSW）支持中 Web的目录服务的概念和用法信息。

本章的其余部分提供了有关子程序的参考信息，按字母顺序列出。

  • SDO_CSW.CREATE_SPATIAL_IDX
  • SDO_CSW.CREATE_XQFT_IDX
  • SDO_CSW.INITIALIZE_CSW
  • SDO_CSW.SYNC_INDEX

父主题： 参考信息

23.1 SDO_CSW.CREATE_SPATIAL_IDX

格式

SDO_CSW.CREATE_SPATIAL_IDX（所有者IN VARCHAR2，csw_table_name IN VARCHAR2，srid IN NUMBER）;

描述

如果尚未存在空间索引，则在CSW表上创建空间索引。

参数

所有者

表和空间索引的CSW模式的名称。

csw_table_name

包含CSW目录数据的表的名称。

srid

与CSW目录数据相关联的坐标系（或SRID：空间参照系）。它应该是CSW表的几何列中的SRID。示例： 4326 （EPSG SRID值相当于Oracle SRID 8307）。

您可以指定2D或3D SRID值，但CSW表的几何列必须具有指定SRID的适当维数。

使用说明

如果csw_table_name上已存在空间索引，则此过程不执行任何操作。如果空间索引已存在但需要为CSW重新创建，则必须删除索引，然后使用此过程创建索引。USER_SDO_GEOM_METADATA视图将填充CSW表信息。

此过程使用CSW表的XMLType列中的SDO_GEOMETRY对象。这些几何转换为由SRID参数表示的坐标参考系。

创建的空间索引将具有csw_table_name _IDX形式的名称。

有关支持Web服务的目录服务的信息，请参阅Web 服务目录服务（CSW）支持 。

例子

以下示例在SCOTT.MY_CSW_CATALOG TABLE表上创建空间索引，并将SRID 4326与空间数据相关联。

DECLARE BEGIN sdo_csw.create_spatial_idx（'SCOTT'，'MY_CSW_CATALOG_TABLE'，4326）;结束; /

23.2 SDO_CSW.CREATE_XQFT_IDX

格式

SDO_CSW.CREATE_XQFT_IDX（所有者IN VARCHAR2，csw_table_name IN VARCHAR2;

描述

如果没有此类索引，则在CSW表上创建XQFT（XML Search-XPath查询全文）索引。

参数

所有者

CSW表和索引的数据库模式的名称。

csw_table_name

包含CSW目录数据的表的名称。

使用说明

如果XQFT指标已经存在于csw_table_name ，如果你只需要通过与当前CSW数据同步来更新索引，则应该使用SDO_CSW.SYNC_INDEX过程，因为该过程只需不到SDO_CSW.CREATE_XQFT_IDX执行时间。

如果XQFT指标上已存在csw_table_name ，这个过程删除现有的一个，并重新创建XQFT指数。它还在内部调用CTX_DDL.DROP_PREFERENCE和CTX_DDL.DROP_SECTION_GROUP过程。

此过程在内部调用CTX_DDL.CREATE_PEREFERENCE过程以创建名为CSWLEX的BASIC_LEXER词法分析器。

创建的索引将具有csw_table_name _XQFT_IDX形式的名称。

有关支持Web服务的目录服务的信息，请参阅Web 服务目录服务（CSW）支持 。

例子

以下示例在SCOTT.MY_CSW_CATALOG_TABLE表上创建XQFT索引。

DECLARE BEGIN sdo_csw.create_xqft_idx（'SCOTT'，'MY_CSW_CATALOG_TABLE'）;结束; /

23.3 SDO_CSW.INITIALIZE_CSW

格式

SDO_CSW.INITIALIZE_CSW（所有者IN VARCHAR2，csw_version IN VARCHAR2，csw_xsd_id IN NUMBER，csw_table_name IN VARCHAR2，srid IN NUMBER，generate_index IN NUMBER）;

描述

初始化Web（CSW）版本2.0.2服务数据库实例的Oracle Spatial和Graph Catalog Services。这使得当前用户的模式成为CSW模式，如果CSW目录表尚不存在，则创建CSW目录表，并准备索引，如使用说明中所述。

参数

所有者

拥有创建的表和索引的数据库模式的名称。

csw_version

CSW版本号。示例： 2.0.2 。

csw_xsd_id

MDSYS.SDO_XSD_TABLE表中的XSD的ID。可能的值：DCMI为1 ，ISO 19139为2 。

csw_table_name

用于保存CSW目录数据的表的名称。

srid

与CSW数据相关联的坐标系（或SRID：空间参照系）。它应该是CSW表的几何列中的SRID。示例： 4326 （EPSG SRID值相当于Oracle SRID 8307）。

您可以指定2D或3D SRID值，但CSW表的几何列必须具有指定SRID的适当维数。

generate_index

确定是否在csw_table_name表中的数据上构建索引。 0 （零）不会在表上创建任何索引; 1 （或任何正的非零值）在表上构建所有适当的索引。

使用说明

此过程允许您创建用户表，指定数据的SRID值，并控制是否在该表上构建索引。它还会在USER_CSW_SERVICE_INFO视图中填充CSW元数据，以将架构注册为CSW架构。如果owner.csw_table_name尚不存在，则此过程使用以下列创建该表：

（Metadata_Id VARCHAR（4000）PRIMARY KEY，Record_Instance_Xml XMLType，Record_Geometry SDO_GEOMETRY，XMLTYPE COLUMN Record_Instance_Xml STORE AS BINARY XML）

如果CSW目录表（和任何索引）已存在，则此过程也可用于通过将SRID参数设置为NULL并将GENERATE_INDEX参数设置为0来注册CSW目录表。然后，它仅填充USER_CSW_SERVICE_INFO视图中的CSW元数据，以将架构注册为CSW架构。

有关支持Web服务的目录服务的信息，请参阅Web 服务目录服务（CSW）支持 。

例子

以下示例使用过程的第一种格式。它使SCOTT成为CSW模式，将SRID 4326与空间数据相关联，并在MY_CSW_CATALOG_TABLE表中的数据上构建索引。

DECLARE BEGIN sdo_csw.initialize_csw（'SCOTT'，'2.0.2'， - 必须是2.0.2 1， - 对于DCMI'MY_CSW_CATALOG_TABLE'，4326,1）;结束; /

23.4 SDO_CSW.SYNC_INDEX

格式

SDO_CSW.SYNC_INDEX（所有者IN VARCHAR2，csw_table_name IN VARCHAR2;

描述

通过将索引与表中的数据同步，更新CSW用户数据表上的XQFT（XML Search-XPath查询全文）索引。

参数

所有者

表和索引的数据库模式的名称。

csw_table_name

包含CSW目录数据的表的名称。

使用说明

当CSW用户数据表上有重要的插入，更新或删除操作时，应该运行此过程，因为现有的XQFT nidex会自动更新。

此过程将更新名称格式为csw_table_name _XQFT_IDX 。

例子

以下示例更新MDMETT.CSW_TABLE表上的XQFT索引。

DECLARE BEGIN sdo_csw.sync_index（'MDMETT'，'CSW_TABLE'）;结束; /
