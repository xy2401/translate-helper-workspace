SET CONSTRAINT [S]

目的

使用SET CONSTRAINTS语句为特定事务指定是在每个DML语句（ IMMEDIATE ）之后还是在提交事务时（ DEFERRED ）检查可延迟约束。您可以使用此语句为约束名称列表或ALL约束设置模式。

SET CONSTRAINTS模式持续到事务的持续时间或直到另一个SET CONSTRAINTS语句重置模式。

注意：

您还可以使用带有SET CONSTRAINTS子句的ALTER SESSION语句来设置所有可延迟约束。这相当于在当前会话中的每个事务开始时发出SET CONSTRAINTS语句。

您不能在触发器定义中指定此语句。

SET CONSTRAINTS可以是分布式语句。当发出SET CONSTRAINTS ALL语句时，将通知正在处理事务的现有数据库链接，并且一旦新链接启动事务就会通知新链接已发出。

先决条件

要指定何时检查可延迟约束，您必须对应用约束的表具有READ或SELECT权限，除非该表位于您的模式中。

语义

约束

指定一个或多个完整性约束的名称。

所有

指定ALL以设置此事务的所有可延迟约束。

即时

指定IMMEDIATE以在执行每个受约束的DML语句时立即检查指定的约束。Oracle数据库首先检查在事务中先前延迟的任何约束，然后继续立即检查该事务中任何其他语句的约束，只要所有已检查的约束一致且未发出其他SET CONSTRAINTS语句。如果任何约束未通过检查，则发出错误信号。此时， COMMIT语句会导致整个事务撤消。

在事务结束时立即制定约束是检查COMMIT是否成功的一种方法。通过将约束设置为IMMEDIATE作为事务中的最后一个语句，可以避免意外回滚。如果任何约束未通过检查，则可以在提交事务之前更正错误。

DEFERRED

指定DEFERRED以指示在提交事务时检查由延迟约束指定的条件。

注意：

通过发出SET CONSTRAINTS ALL IMMEDIATE语句，可以在提交之前验证可延迟约束是否成功。

设置约束：示例

以下语句设置在每个DML语句之后立即检查此事务中的所有可延迟约束：
