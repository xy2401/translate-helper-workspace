DROP USER

目的

使用DROP USER语句删除数据库用户，并可选择删除用户的对象。

在Oracle自动存储管理（Oracle ASM）群集中，经过身份验证的AS SYSASM用户可以使用此子句从当前节点的Oracle ASM实例本地的密码文件中删除用户。

删除用户时，Oracle数据库还会从回收站中清除该用户的所有架构对象。

注意：

不要尝试删除用户SYS或SYSTEM 。这样做会破坏您的数据库。

也可以看看：

创建用户和ALTER USER以获取有关创建和修改用户的信息

先决条件

您必须具有DROP USER系统特权。在Oracle ASM群集中，必须通过AS SYSASM进行身份验证。

句法

drop_user :: =


说明drop_user.eps的描述

语义

用户

指定要删除的用户。除非您指定CASCADE或除非您首先显式删除用户的对象，否则Oracle数据库不会删除其模式包含对象的用户。

删除用户的限制

您不能删除其架构包含使用闪回数据存档进行历史跟踪的表的用户。您必须首先禁用表格使用闪回数据存档。

级联

在删除用户之前，指定CASCADE以删除用户架构中的所有对象。您必须指定此子句以删除其架构包含任何对象的用户。

  • 如果用户的模式包含表，则Oracle数据库会删除表并自动删除对引用这些表上的主键和唯一键的其他模式中的表的任何参照完整性约束。

  • 如果此子句导致删除表，则数据库还会删除在这些表的列上创建的所有域索引，并调用适当的删除例程。

    也可以看看：

    有关这些例程的更多信息，请参见Oracle数据库数据盒开发人员指南

  • Oracle数据库使其他模式中的以下对象无效，但不会丢弃：

    • 已删除用户架构中对象的视图或同义词

    • 查询已删除用户架构中的对象的存储过程，函数或包

  • Oracle数据库不会删除基于已删除用户架构中的表的其他架构中的实体化视图。但是，由于基表不再存在，因此无法再刷新其他模式中的实例化视图。

  • Oracle数据库删除用户架构中的所有触发器。

  • Oracle数据库不会删除用户创建的角色。

注意：

Oracle数据库也随FORCE删除了用户拥有的所有类型。请参阅DROP TYPE的FORCE关键字。

例子

删除数据库用户：示例

如果用户Sidney的架构不包含任何对象，那么您可以通过发出以下语句来删除sidney ：

DROP USER sidney;

如果Sidney的模式包含对象，则必须使用CASCADE子句删除sidney和对象：

DROP USER sidney CASCADE;
