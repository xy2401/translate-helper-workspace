4创建和管理模式对象

要创建，更改和删除模式对象，可以使用数据定义语言（DDL）语句。

  • 关于数据定义语言（DDL）语句
    数据定义语言（DDL）语句创建，更改和删除模式对象。在DDL语句之前和之后，Oracle数据库发出隐式COMMIT语句;因此，您无法回滚DDL语句。
  • 创建和管理表
    表是Oracle数据库中数据存储的基本单位。表包含所有用户可访问的数据。每个表都包含表示各个数据记录的行。行由表示记录字段的列组成。
  • 创建和管理视图
    视图将查询结果显示为表格。在大多数可以使用表格的地方，您可以使用视图。当您需要频繁访问存储在多个不同表中的信息时，视图非常有用。
  • 创建和管理序列
    序列是模式对象，您可以从中生成唯一的顺序值，这在您需要唯一主键时非常有用。序列通过伪列CURRVAL和NEXTVAL使用 ，它们分别返回序列的当前值和下一个值。
  • 创建和管理同义词
    同义词是另一个模式对象的别名。使用同义词的一些原因是安全性（例如，隐藏对象的所有者和位置）和便利性。

4.1关于数据定义语言（DDL）语句

数据定义语言（DDL）语句创建，更改和删除模式对象。在DDL语句之前和之后，Oracle数据库发出隐式COMMIT语句;因此，您无法回滚DDL语句。

注意：

创建模式对象时，必须遵守Oracle数据库SQL语言参考中的模式对象命名规则。

在SQL * Plus环境中，您可以在SQL>提示符后输入DDL语句。

在SQL Developer环境中，您可以在Worksheet中输入DDL语句。或者，您可以使用SQL Developer工具来创建，更改和删除对象。

一些创建模式对象的DDL语句具有可选的OR REPLACE子句，该子句允许语句将现有模式对象替换为具有相同名称和类型的另一个模式对象。当SQL Developer为其中一个语句生成代码时，它始终包含OR REPLACE子句。

要在SQL Developer中查看DDL语句的效果，可能必须在Connections框架中选择新创建的对象的模式对象类型，然后单击“ 刷新”图标。

4.2.3.1关于约束

约束限制列可以具有的值。尝试以违反约束的方式更改数据会导致错误并回滚更改。如果现有数据违反约束，则尝试向填充表添加约束会导致错误。

可以启用和禁用约束。默认情况下，它们是在启用状态下创建的。

约束类型是：

  • Not Null ，阻止值为null

    在EMPLOYEES表中，列LAST_NAME具有NOT NULL约束，该约束强制执行每个员工必须具有姓氏的业务规则。

  • 唯一 ，可防止多行在同一列或列组合中具有相同的值，但允许某些值为null

    在EMPLOYEES表中，EMAIL列具有UNIQUE约束，该约束强制执行业务规则，即员工不能拥有电子邮件地址，但不能拥有与其他员工相同的电子邮件地址。

  • 主键 ，它是NOT NULL和UNIQUE的组合

    在EMPLOYEES表中，EMPLOYEE_ID列具有PRIMARY KEY约束，该约束强制执行每个员工必须具有唯一员工标识号的业务规则。

  • 外键 ，它要求一个表中的值与另一个表中的值匹配

    在EMPLOYEES表中，JOB_ID列具有引用JOBS表的FOREIGN KEY约束，该约束表强制实施员工不能拥有JOB表中不存在的JOB_ID的业务规则。

  • 检查 ，这要求值满足指定条件

    EMPLOYEES表没有CHECK约束。但是，假设EMPLOYEES需要一个新列EMPLOYEE_AGE，并且每个员工必须至少为18。约束CHECK (EMPLOYEE_AGE >= 18)强制执行业务规则。

    小费：

    仅当其他约束类型无法提供必要的检查时才使用检查约束。

  • REF ，它进一步描述了REF列与它引用的对象之间的关系

    REF列引用另一个对象类型或关系表中的对象。

    有关REF约束的信息，请参阅Oracle数据库概念 。

4.2.3.2教程：向现有表添加约束

本教程介绍如何使用SQL Developer工具和ALTER TABLE语句向现有表添加约束。

要向现有表添加约束，请使用SQL Developer工具或DDL语句ALTER TABLE 。本主题说明如何使用这两种方法向“ 创建表 ”中创建的表添加约束。

本教程有几个步骤。第一个过程使用“编辑表”工具将“非空”约束添加到PERFORMANCE_PARTS表的NAMES列。其余过程说明如何使用其他工具添加约束;但是，您可以使用“编辑表”工具添加相同的约束。

注意：

在本教程的任何步骤之后，您可以查看表具有的约束：

  1. 在Connections框架中，选择表的名称。

  2. 在右侧框架中，单击“ 约束 ”选项卡。

有关查看表属性和数据的更多信息，请参阅“ 教程：使用SQL Developer查看EMPLOYEES表属性和数据 ” 。

要使用“编辑表”工具添加“非空”约束：

  1. 在Connections框架中，展开hr_conn 。

  2. 在架构对象类型列表中，展开“ 表” 。

  3. 在表列表中，右键单击PERFORMANCE_PARTS 。

  4. 在选项列表中，单击“ 编辑” 。

  5. 在“编辑表”窗口中，单击“ 名称 ”列。

  6. 选择属性Not Null 。

  7. 单击确定 。

    Not Null约束将添加到PERFORMANCE_PARTS表的NAME列。

以下过程使用ALTER TABLE语句将Not Null约束添加到PERFORMANCE_PARTS表的WEIGHT列。

要使用ALTER TABLE语句添加Not Null约束：

  1. 如果存在具有选项卡hr_conn的窗格，请选择它。否则，单击图标SQL Worksheet ，如“ 在SQL Developer中运行查询 ”中所示 。

  2. 在“工作表”窗格中，键入以下语句：

    ALTER TABLE PERFORMANCE_PARTS MODIFY WEIGHT NOT NULL;

  3. 单击图标Run Statement 。

    该语句运行，将Not Null约束添加到PERFORMANCE_PARTS表的WEIGHT列。

以下过程使用“添加唯一”工具向SCORES表添加“唯一”约束。

要使用“添加唯一”工具添加“唯一”约束：

  1. 在Connections框架中，展开hr_conn 。

  2. 在架构对象类型列表中，展开“ 表” 。

  3. 在表的列表中，右键单击“ SCORES” 。

  4. 在选项列表中，选择“ 约束” 。

  5. 在选项列表中，单击“ 添加唯一” 。

  6. 在“添加唯一性”窗口中：

    1. 对于Constraint Name，输入SCORES_EVAL_PERF_UNIQUE 。

    2. 对于第1列，从菜单中选择EVALUATION_ID 。

    3. 对于第2列，从菜单中选择PERFORMANCE_ID 。

    4. 单击“ 应用”

  7. 在“确认”窗口中，单击“ 确定” 。

    名为SCORES_EVAL_PERF_UNIQUE的唯一约束将添加到SCORES表中。

以下过程使用“添加主键”工具将主键约束添加到PERFORMANCE_PARTS表的PERFORMANCE_ID列。

要使用“添加主键”工具添加主键约束，请执行以下操作：

  1. 在Connections框架中，展开hr_conn 。

  2. 在架构对象类型列表中，展开“ 表” 。

  3. 在表列表中，右键单击PERFORMANCE_PARTS 。

  4. 在选项列表中，选择“ 约束” 。

  5. 在选项列表中，单击“ 添加主键” 。

  6. 在“添加主键”窗口中：

    1. 对于主键名称，输入PERF_PERF_ID_PK 。

    2. 对于第1列，从菜单中选择PERFORMANCE_ID 。

    3. 单击“ 应用”

  7. 在“确认”窗口中，单击“ 确定” 。

    名为PERF_PERF_ID_PK主键约束将添加到PERFORMANCE_PARTS表的PERFORMANCE_ID列中。

以下过程使用ALTER TABLE语句将主键约束添加到EVALUATIONS表的EVALUATION_ID列。

要使用ALTER TABLE语句添加主键约束：

  1. 如果存在具有选项卡hr_conn的窗格，请选择它。否则，单击图标SQL Worksheet ，如“ 在SQL Developer中运行查询 ”中所示 。

    该语句运行，将主键约束添加到EVALUATIONS表的EVALUATION_ID列。

4.2.7管理索引

您可以在表的一列或多列上创建索引，以加快该表上的SQL语句执行速度。正确使用时，索引是减少磁盘输入/输出（I / O）的主要方法。

在表上定义主键时：

  • 如果现有索引以主键列开头，则Oracle数据库将该现有索引用于主键。现有索引不必是唯一的。

    例如，如果定义主键（A，B），Oracle数据库将使用现有索引（A，B，C）。

  • 如果没有现有索引以主键列开始且约束是立即的，则Oracle数据库会在主键上创建唯一索引。

  • 如果现有索引不以主键列开头且约束可延迟，则Oracle数据库会在主键上创建非唯一索引。

例如，在“ 教程：向现有表添加约束 ”中 ，您将主键约束添加到EVALUATIONS表的EVALUATION_ID列。因此，如果在SQL Developer Connections框架中选择EVALUATIONS表并单击Indexes选项卡，则Indexes窗格将在EVALUATION_ID列上显示Unique索引。

4.4创建和管理序列

序列是模式对象，您可以从中生成唯一的顺序值，这在您需要唯一主键时非常有用。序列通过伪列CURRVAL和NEXTVAL使用 ，它们分别返回序列的当前值和下一个值。

创建序列后，必须使用NEXTVAL初始化它以获取其第一个值。只有在初始化序列后， CURRVAL才会返回其当前值。

HR模式有三个序列：DEPARTMENTS_SEQUENCE，EMPLOYEES_SEQUENCE和LOCATIONS_SEQUENCE。

小费：

当您计划使用序列填充表的主键时，请为序列指定一个反映此用途的名称。（本主题使用命名约定TABLE_NAME _SEQUENCE。）

EVALUATIONS表在例4-1中创建。

要创建序列，请使用SQL Developer工具Create Sequence或DDL语句CREATE SEQUENCE 。等效的DDL语句是：

CREATE SEQUENCE evaluations_sequence INCREMENT BY 1 START WITH 1 ORDER;

要使用“创建数据库序列”工具创建EVALUATIONS_SEQUENCE：

  1. 在Connections框架中，展开hr_conn 。

  2. 在架构对象类型列表中，右键单击“ 序列” 。

  3. 在选项列表中，单击“ 新建序列” 。

  4. 在“ 创建序列”窗口的“ 名称”字段中，键入EVALUATIONS_SEQUENCE超过默认值“SEQUENCE1”。

  5. 如果未显示“ 属性”窗格，请单击“ 属性 ”选项卡。

  6. 在“ 属性”窗格中：

    1. 在“ 增量 ”字段中，键入1 。

    2. 在“ 开始于”字段中，键入1 。

    3. 对于其余字段，请接受默认值。

    4. 单击确定 。

      创建序列EVALUATIONS_SEQUENCE。其名称显示在Connections框架中的Sequences下。

4.4.2丢弃序列

要删除序列，请使用SQL Developer Connections框架和Drop工具，或使用DDL语句DROP SEQUENCE 。

该语句删除了EVALUATIONS_SEQUENCE序列：

DROP SEQUENCE EVALUATIONS_SEQUENCE;

警告：

不要删除序列EVALUATIONS_SEQUENCE - 您需要它来执行例5-3 。如果要练习删除序列，请创建其他序列然后删除它们。

要使用Drop工具删除序列：

  1. 在Connections框架中，展开hr_conn 。
  2. 在架构对象类型列表中，展开“ 序列” 。
  3. 在序列列表中，右键单击要删除的序列的名称。
  4. 在选项列表中，单击“ 删除” 。
  5. 在“下拉”窗口中，单击“ 应用” 。
  6. 在“确认”窗口中，单击“ 确定” 。

4.5创建和管理同义词

同义词是另一个模式对象的别名。使用同义词的一些原因是安全性（例如，隐藏对象的所有者和位置）和便利性。

便利的例子是：

  • 使用简短的同义词（如SALES ）作为长对象名称，例如ACME_CO 。 SALES_DATA

  • 使用重命名对象的同义词，而不是在使用它的应用程序中更改该对象名称

    例如，如果您的应用程序使用名为DEPARTMENTS的表，并且其名称更改为DIVISIONS ，则可以为该表创建DEPARTMENTS同义词并继续按其原始名称引用它。

  • 创建同义词
    要创建同义词，请使用SQL Developer工具Create Database Synonym或DDL语句CREATE SYNONYM 。
  • 删除同义词
    要删除同义词，请使用SQL Developer Connections框架和Drop工具，或使用DDL语句DROP SYNONYM 。

4.5.1创建同义词

要创建同义词，请使用SQL Developer工具Create Database Synonym或DDL语句CREATE SYNONYM 。

以下教程介绍了如何使用“创建数据库同义词”工具为EMPLOYEES表创建同义词EMP。等效的DDL语句是：

为员工创建SYNONYM EMPL;

使用“创建数据库同义词”工具创建同义词EMP：

  1. 在Connections框架中，展开hr_conn 。

  2. 在架构对象类型列表中，右键单击“ 同义词” 。

  3. 在选项列表中，单击“ 新建同义词” 。

  4. 在“ 新同义词”窗口中：

    1. 在“ 同义词名称”字段中，键入EMPL 。

    2. 在“ 对象所有者”字段中，从菜单中选择HR 。

    3. 在“ 对象名称”字段中，从菜单中选择“ EMPLOYEES ”。

      同义词是指特定的模式对象;在这种情况下，表EMPLOYEES。

    4. 单击“ 应用”

  5. 在“ 确认”窗口中，单击“ 确定” 。

    创建同义词EMPL。要查看它，请展开Connections框架中的同义词 。您现在可以使用EMPL而不是EMPLOYEES。
