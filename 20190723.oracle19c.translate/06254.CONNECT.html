2.10连接

目的

使用CONNECT命令在RMAN与目标，辅助或恢复目录数据库之间建立连接。

先决条件

指定和验证与数据库的RMAN连接的方式与SQL * Plus与数据库的连接相同。唯一的区别是与目标或辅助数据库的RMAN连接需要SYSBACKUP或SYSDBA权限。

也可以看看：

  • Oracle数据库管理员指南 ，了解使用SQL * Plus时的数据库连接选项

  • “Oracle数据库备份和恢复用户指南”了解如何使用SYSBACKUP管理权限

警告：

良好的安全性要求不要在命令行上以纯文本形式输入密码。仅在RMAN提示请求时在RMAN中输入密码。请参阅Oracle数据库安全指南以了解密码保护。

RMAN用于命令行连接选项

只能在RMAN尚未连接到这些命令指定的数据库的情况下，才能在RMAN提示符下运行CONNECT TARGET ， CONNECT CATALOG和CONNECT AUXILIARY命令。要连接到其他目标，目录或辅助数据库，您必须启动新的RMAN会话。

使用说明

如果满足以下所有条件，则默认情况下RMAN会话以NOCATALOG模式运行：

  • 启动RMAN时未指定CATALOG或NOCATALOG 。

  • 您还没有在RMAN会话中运行CONNECT CATALOG 。

  • 您运行需要RMAN存储库连接的BACKUP等命令（如示例2-61所示）。

连接到CDB和PDB

RMAN为在多租户环境中执行备份和恢复操作提供全面支持。多租户架构使Oracle数据库可以作为多租户容器数据库（CDB）运行，该数据库包括零个，一个或多个客户创建的可插拔数据库（PDB）。建立与CDB或PDB的RMAN连接的过程类似于非CDB的过程。您可以使用操作系统身份验证或密码文件身份验证。

也可以看看：

  • 有关多租户架构概念的信息，请参见Oracle数据库概念

  • 有关管理多租户环境的信息，请参见“Oracle数据库管理员指南”

要连接到CDB，请以具有SYSDBA或SYSBACKUP权限的公共用户身份连接到root。

要连接到PDB，您可以以普通用户或本地用户身份进行连接。创建连接的用户必须具有“ connectStringSpec ”中描述的权限。

要在整个CDB或根目录上执行操作，请将TARGET连接到根目录。创建连接的用户必须是具有SYSDBA或SYSBACKUP权限的普通用户。要对多个PDB执行操作，请将TARGET连接到根目录。创建连接的用户必须是具有SYSDBA或SYSBACKUP权限的普通用户。要在PDB上执行操作，您可以作为TARGET连接到根或PDB。

语义

语法元素 描述

连接辅助

在RMAN和辅助数据库实例之间建立连接。

辅助实例与TRANSPORT TABLESPACE和DUPLICATE命令一起使用，并在RMAN TSPITR期间使用。

连接目录

在RMAN和恢复目录数据库之间建立连接。

如果恢复目录是虚拟专用目录（请参阅CREATE CATALOG ），则连接到此目录的RMAN客户端必须处于修补程序级别10.1.0.6或10.2.0.3。Oracle9 i RMAN客户端无法连接到虚拟专用目录。此版本限制不会影响RMAN客户端与Oracle Database 11 g基本恢复目录的连接，即使它具有某些虚拟专用目录用户也是如此。

RMAN发布了一个如果在RMAN处于默认NOCATALOG模式时尝试在RMAN会话中使用CONNECT CATALOG命令， RMAN-06445出现RMAN-06445错误（请参阅“ 使用说明 ” ）。

注意：必须在Data Guard环境中将RMAN与恢复目录一起使用。

连接目标

建立RMAN与目标数据库之间的连接。

注意： RMAN可以在Data Guard环境中作为TARGET连接到物理备用数据库。如果对具有恢复目录未知的DB_UNIQUE_NAME的数据库运行CONNECT TARGET ，但DBID标识已注册的数据库，则RMAN会自动并隐式地将数据库注册到恢复目录中。

    connectStringSpec

指定数据库的连接信息。

例子

示例2-60连接到没有恢复目录的目标数据库

此示例以NOCATALOG模式启动RMAN，然后使用Oracle Net服务名称prod1连接到目标数据库。 sbu是被授予SYSBACKUP权限的用户。 

％rman NOCATALOG RMAN> CONNECT TARGET“sbu @ prod1 AS SYSBACKUP”;目标数据库密码：连接到目标数据库的密码 ：PROD1（DBID = 39525561）

示例2-61在默认NOCATALOG模式下连接到目标数据库

此示例在不指定CATALOG或NOCATALOG情况下启动RMAN，然后使用操作系统身份验证通过操作系统身份验证连接到目标数据库。由于未运行CONNECT CATALOG命令，因此在运行BACKUP命令时，RMAN默认为NOCATALOG模式。 

％rman RMAN> CONNECT TARGET / RMAN> BACKUP DATABASE;

此时，在RMAN会话中，您无法运行CONNECT CATALOG因为会话已默认为NOCATALOG模式。尝试连接到此会话中的目录会收到错误：

RMAN> CONNECT CATALOG rco @ catdb RMAN-00571：======================================= ==================== RMAN-00569：===============错误信息堆栈跟踪======= ======== RMAN-00571：====================================== =====================  RMAN-06445：使用NOCATALOG后无法连接到恢复目录

示例2-62连接到目标，恢复目录和辅助数据库

此示例使用操作系统身份验证连接到目标数据库，并使用密码文件连接到恢复目录和辅助数据库。 sbu是被授予SYSBACKUP权限的用户。RMAN提示输入密码。 

％rman RMAN> CONNECT TARGET;连接到目标数据库：PROD（DBID = 39525561）RMAN> CONNECT CATALOG rco @ catdb;恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> CONNECT AUXILIARY“sbu @ dupdb AS SYSBACKUP”;辅助数据库密码：连接到辅助数据库的密码 ：DUPDB（未安装）

示例2-63使用操作系统身份验证连接到根目录

此示例使用操作系统身份验证连接到CDB中的根。默认情况下，使用SYSDBA权限建立连接。 

％rman RMAN> CONNECT TARGET /

示例2-64以本地用户身份连接到PDB

此示例连接到PDB hr_pdb作为在hr_pdb PDB上被授予SYSBACKUP权限的本地用户sbu_pdb 。 hrpdb是与PDB hr_pdb对应的网络服务名称。 

％rman RMAN> CONNECT TARGET“sbu_pdb @hrpdb AS SYSBACKUP”;
