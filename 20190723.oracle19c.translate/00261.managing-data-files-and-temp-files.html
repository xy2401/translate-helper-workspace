14.1.2.2确定DB_FILES初始化参数的值

启动Oracle数据库实例时， DB_FILES初始化参数指示为数据文件信息保留的SGA空间量，从而指示可为该实例创建的最大数据文件数。

此限制适用于实例的生命周期。您可以更改DB_FILES的值（通过更改初始化参数设置），但在关闭并重新启动实例之前，新值不会生效。

确定DB_FILES的值时，请考虑以下因素：

  • 如果DB_FILES的值太低，则无法在未先关闭数据库的情况下添加超出DB_FILES限制的数据文件。

  • 如果DB_FILES的值太高，则会不必要地消耗内存。

14.1.2.3将数据文件添加到表空间时考虑可能的限制

将数据文件添加到表空间时需要考虑一些限制。

您可以将数据文件添加到传统的smallfile表空间，但受以下限制：

  • 操作系统通常会对进程可以同时打开的文件数量施加限制。达到打开文件的操作系统限制时，无法创建更多数据文件。

  • 操作系统对数据文件的数量和大小施加限制。

  • 数据库对任何实例打开的任何Oracle数据库的数据文件数量施加了最大限制。此限制是特定于操作系统的。

  • 您不能超过DB_FILES初始化参数指定的数据文件数。

  • 发出CREATE DATABASE或CREATE CONTROLFILE语句时， MAXDATAFILES参数指定控制文件的数据文件部分的初始大小。但是，如果您尝试添加一个数字大于MAXDATAFILES但小于或等于DB_FILES的新文件，控制文件将自动扩展，以便数据文件部分可以容纳更多文件。

14.1.2.4考虑数据文件数量的性能影响

表空间中包含的数据文件数量，最终包含在数据库中，可能会对性能产生影响。

Oracle数据库允许数据库中的数据文件多于操作系统定义的限制。数据库DBW n进程可以打开所有在线数据文件。Oracle数据库能够将打开的文件描述符视为缓存，当打开的文件描述符数达到操作系统定义的限制时自动关闭文件。这可能会对性能产生负面影响。如果可能，请调整打开文件描述符的操作系统限制，使其大于数据库中的联机数据文件数。

14.3.1启用和禁用数据文件的自动扩展

您可以创建数据文件或更改现有数据文件，以便在数据库中需要更多空间时自动增加它们的大小。文件大小以指定的增量增加到指定的最大值。

将数据文件设置为自动扩展可提供以下优势：

  • 当表空间空间不足时，减少了立即干预的需要

  • 确保应用程序不会因为分配扩展区失败而暂停或暂停

使用以下SQL语句创建数据文件时，可以通过指定AUTOEXTEND ON子句来指定自动文件扩展名：

  • CREATE DATABASE

  • ALTER DATABASE

  • CREATE TABLESPACE

  • ALTER TABLESPACE

要启用或禁用数据文件的自动扩展：

  1. 通过查询DBA_DATA_FILES视图并检查AUTOEXTENSIBLE列来确定数据文件是否可自动扩展。

  2. 使用带有AUTOEXTEND子句的ALTER DATABASE语句启用或禁用现有数据文件的自动文件扩展名，或手动调整数据文件大小。对于bigfile表空间，请使用带有AUTOEXTEND子句的ALTER TABLESPACE语句。

以下示例为添加到users表空间的数据文件启用自动扩展：

ALTER TABLESPACE用户添加数据文件'/u02/oracle/rbdb1/users03.dbf'SIZE 10M AUTOEXTEND ON NEXT 512K MAXSIZE 250M;

NEXT的值是扩展时添加到文件的增量的最小大小。MAXSIZE的值是文件可以自动扩展的最大大小。

下一个示例禁用数据文件的自动扩展。

ALTER DATABASE DATAFILE'/u02/oracle/rbdb1/users03.dbf'AUTOEXTEND OFF;

14.3.2手动调整数据文件大小

您可以使用ALTER DATABASE语句手动增加或减少数据文件的大小。

因此，您可以在不添加更多数据文件的情况下为数据库添加更多空间。如果您担心达到数据库中允许的最大数据文件数，这将非常有用。

对于bigfile表空间，可以使用ALTER TABLESPACE语句调整数据文件的大小。您不能将数据文件添加到bigfile表空间。

手动减小数据文件的大小使您可以回收数据库中未使用的空间。这对于纠正空间要求估计中的错误很有用。

在以下示例中，假设数据文件/u02/oracle/rbdb1/stuff01.dbf已扩展到250M。但是，由于其表空间现在存储较小的对象，因此可以减小数据文件的大小。

以下语句减小了数据文件/u02/oracle/rbdb1/stuff01.dbf的大小：

ALTER DATABASE DATAFILE'/u02/oracle/rbdb1/stuff01.dbf'RESIZE 100M;

注意：

并不总是可以将文件的大小减小到特定值。可能是文件包含超出指定减小大小的数据，在这种情况下数据库将返回错误。

14.4.1关于更改数据文件可用性

您可以通过使其脱机或使其联机来更改单个数据文件或临时文件的可用性。脱机数据文件对数据库不可用，并且在它们重新联机之前无法访问。

更改数据文件可用性的原因包括：

  • 您想要执行数据文件的脱机备份。

  • 您想要重命名或重定位脱机数据文件。您可以先使数据文件脱机或使表空间脱机。

  • 数据库在写入数据文件时会出现问题，并自动使数据文件脱机。之后，在解决问题后，您可以手动将数据文件重新联机。

  • 数据文件丢失或损坏。必须先将其脱机才能打开数据库。

只读表空间的数据文件可以脱机或联机，但将文件联机不会影响表空间的只读状态。在表空间返回读/写状态之前，您无法写入数据文件。

注意：

您可以通过使表空间本身脱机来使表空间的所有数据文件暂时不可用。您必须将这些文件保留在表空间中以使表空间重新联机，尽管您可以按照类似于“ 重命名和重定位数据文件 ”中所示的过程重新定位或重命名它们。

有关更多信息，请参阅“使表空间脱机 ” 。

要使数据文件脱机或使其联机，您必须具有ALTER DATABASE系统特权。要使用ALTER TABLESPACE语句使所有数据文件或临时文件脱机，您必须具有ALTER TABLESPACE或MANAGE TABLESPACE系统特权。在Oracle Real Application Clusters环境中，数据库必须以独占模式打开。

14.4.2在ARCHIVELOG模式下使数据文件联机或脱机

要使单个数据文件联机或使单个数据文件脱机，请发出ALTER DATABASE语句并包含DATAFILE子句。

以下语句将指定的数据文件联机：

ALTER DATABASE DATAFILE'/u02/oracle/rbdb1/stuff01.dbf'ONLINE;

要使同一文件脱机，请发出以下语句：

ALTER DATABASE DATAFILE'/u02/oracle/rbdb1/stuff01.dbf'OFFLINE;

注意：

要使用此形式的ALTER DATABASE语句，数据库必须处于ARCHIVELOG模式。此要求可防止您意外丢失数据文件，因为在NOARCHIVELOG模式下使数据文件脱机可能会导致文件丢失。

父主题： 更改数据文件可用性

14.4.3在NOARCHIVELOG模式下使数据文件脱机

要在数据库处于NOARCHIVELOG模式时使数据文件脱机，请将ALTER DATABASE语句与DATAFILE和OFFLINE FOR DROP子句一起使用。

  • OFFLINE关键字使数据库将数据文件标记为OFFLINE ，无论它是否已损坏，以便您可以打开数据库。

  • FOR DROP关键字标记数据文件以供后续删除。这样的数据文件不能再恢复在线状态。

    注意：

    此操作实际上不会删除数据文件。它保留在数据字典中，您必须使用以下方法之一自行删除它：

    • 一个ALTER TABLESPACE ...DROP DATAFILE语句。

      在OFFLINE FOR DROP ，此方法仅适用于字典管理的表空间。

    • DROP TABLESPACE ...INCLUDING CONTENTS AND DATAFILES声明

    • 如果上述方法失败，则执行操作系统命令删除数据文件。这是最不理想的方法，因为它在数据字典和控制文件中留下对数据文件的引用。

以下语句使指定的数据文件脱机并将其标记为要删除：

ALTER DATABASE DATAFILE'/u02/oracle/rbdb1/users03.dbf'DELLINE FOR DROP;

14.4.4更改表空间中所有数据文件或临时文件的可用性

ALTER TABLESPACE语句的子句允许您更改表空间中所有数据文件或临时文件的联机或脱机状态。

具体而言，影响在线/离线状态的语句为：

  • ALTER TABLESPACE ...DATAFILE { ONLINE | OFFLINE }

  • ALTER TABLESPACE ...TEMPFILE { ONLINE | OFFLINE }

您只需输入表空间名称，而不是单个数据文件或临时文件。所有数据文件或临时文件都会受到影响，但表空间本身的联机/脱机状态不会更改。

在大多数情况下，无论何时挂载数据库，都可以发出前面的ALTER TABLESPACE语句，即使它未打开也是如此。但是，如果表空间是SYSTEM表空间，撤消表空间或默认临时表空间，则不能打开数据库。ALTER DATABASE DATAFILE和ALTER DATABASE TEMPFILE语句也有ONLINE/OFFLINE子句，但是在这些语句中，您必须输入表空间的所有文件名。

语法与ALTER TABLESPACE...不同ALTER TABLESPACE...ONLINE|OFFLINE语句改变表空间可用性，因为这是一个不同的操作。ALTER TABLESPACE语句使数据文件脱机以及表空间，但它不能用于更改临时表空间或其临时文件的状态。

您可以使用ALTER DATABASE MOVE DATAFILE SQL语句重命名或重定位在线数据文件。此语句使您可以在数据库打开且用户访问数据文件时重命名或重定位数据文件。

重命名或重定位在线数据文件时，会更改数据库控制文件中记录的数据文件指针。这些文件也在操作系统级别进行物理重命名或重定位。

您可以重命名或重定位在线数据文件，因为您希望在执行以下任务之一时允许用户访问数据文件：

  • 将数据文件从一种类型的存储移动到另一种类型

  • 将不经常访问的数据文件移动到较低成本的存储

  • 将表空间设置为只读并将其数据文件移动到一次写入存储

  • 将数据库移动到Oracle自动存储管理（Oracle ASM）

当您运行ALTER DATABASE MOVE DATAFILE语句并且目标位置中存在具有相同名称的文件时，您可以指定REUSE选项以覆盖现有文件。如果未指定REUSE ，并且目标位置中存在具有相同名称的文件，则不会覆盖现有文件，并且该语句将返回错误。

默认情况下，当您运行ALTER DATABASE MOVE DATAFILE语句并为数据文件指定新位置时，该语句将移动数据文件。但是，您可以指定KEEP选项以将数据文件保留在旧位置并将其复制到新位置。在这种情况下，数据库仅在语句成功完成时才使用新位置中的数据文件。

使用ALTER DATABASE MOVE DATAFILE语句重命名或重定位数据文件时，Oracle数据库会在执行操作时创建数据文件的副本。确保在操作期间有足够的磁盘空间用于原始数据文件和副本。

您可以通过查询DBA_DATA_FILES视图来查看每个数据文件的名称，位置和在线状态。

注意：

  • 如果指定的数据文件脱机， ALTER DATABASE MOVE DATAFILE语句将引发错误。

  • 如果使用备用数据库，则可以在主数据库和备用数据库上独立执行联机移动数据文件操作（物理或逻辑）。在主数据库上移动数据文件时，备用数据库不受影响，反之亦然。有关更多信息，请参阅Oracle Data Guard概念和管理 。

  • 闪回操作不会将移动的数据文件重定位到其先前的位置。如果将数据文件从一个位置在线移动到另一个位置，然后将数据库闪回到移动前的某个时间点，则数据文件将保留在新位置，但数据文件的内容将更改为闪回中指定的时间。有关闪回数据库操作的详细信息，请参见“ Oracle数据库备份和恢复用户指南” 。

  • 在Windows平台上重定位数据文件时，即使省略了KEEP选项，原始数据文件也可能保留在旧位置。在这种情况下，数据库仅在语句成功完成时才使用新位置中的数据文件。如有必要，您可以在操作完成后手动删除旧数据文件。

    请参阅“ 使用SQL * Plus启动数据库 ” 。

14.6删除数据文件

使用ALTER TABLESPACE语句的DROP DATAFILE和DROP TEMPFILE子句删除单个数据文件或临时文件。

数据文件必须为空。（如果没有从中分配任何扩展区，则认为数据文件为空。）删除数据文件或临时文件时，将从数据字典和控制文件中删除对数据文件或临时文件的引用，并从文件系统或Oracle自动存储管理（Oracle ASM）磁盘组中删除物理文件。

以下示例删除Oracle ASM磁盘组DGROUP1别名example_df3.f标识的数据文件。数据文件属于example表空间。 

ALTER TABLESPACE示例DROP DATAFILE'+ DGROUP1 / example_df3.f';

下一个示例删除临时文件lmtemp02.dbf ，该文件属于lmtemp表空间。 

ALTER TABLESPACE lmtemp DROP TEMPFILE'/u02/oracle/data/lmtemp02.dbf';

这相当于以下声明：

ALTER DATABASE TEMPFILE'/u02/oracle/data/lmtemp02.dbf'DROP INCLUDING DATAFILES;

注意：

如果存在使用临时文件的会话，并且您尝试删除临时文件，则会返回错误，并且不会删除临时文件。在这种情况下，临时文件将脱机，并且临时文件脱机时尝试使用临时文件的查询将失败。

有关ALTER TABLESPACE语法详细信息，请参见Oracle数据库SQL语言参考 。

删除数据文件的限制

删除数据文件和临时文件的限制如下：

  • 数据库必须是开放的。

  • 如果数据文件不为空，则无法删除。

    如果必须删除非空的数据文件并且通过删除模式对象而无法使其为空，则必须删除包含数据文件的表空间。

  • 您不能删除表空间中的第一个或唯一的数据文件。

    因此， DROP DATAFILE不能与bigfile表空间一起使用。

  • 您不能将数据文件放在从管理的字典迁移到本地管理的只读表空间中。支持从所有其他只读表空间中删除数据文件。

  • 您不能删除SYSTEM表空间中的数据文件。

  • 如果本地管理的表空间中的数据文件处于脱机状态，则无法将其删除。

14.7验证数据文件中的数据块

要将数据库配置为使用校验和来验证数据块，请将初始化参数DB_BLOCK_CHECKSUM设置为TYPICAL （默认值）。

此设置使DBW n进程和直接加载程序计算每个块的校验和，并在将块写入磁盘时将校验和存储在块头中。

读取块时会验证校验和，但仅当DB_BLOCK_CHECKSUM为TRUE且块的最后一次写入存储校验和时才会验证校验和。如果检测到损坏，数据库将返回消息ORA-01578并将有关损坏的信息写入警报日志。

可以使用ALTER SYSTEM语句动态更改DB_BLOCK_CHECKSUM参数的值。无论此参数的设置如何，校验和始终用于验证SYSTEM表空间中的数据块。

14.8.3第三方文件传输

虽然DBMS_FILE_TRANSFER包中的过程通常作为本地过程调用被调用，但它们也可以作为远程过程调用来调用。远程过程调用允许您复制数据库中的文件，即使您连接到其他数据库也是如此。

例如，通过执行以下远程过程调用，即使连接到另一个数据库，也可以在数据库DB上创建文件的副本：

DBMS_FILE_TRANSFER.COPY_FILE@DB（...）

使用远程过程调用使您可以在两个数据库之间复制文件，即使您未连接到任一数据库也是如此。例如，您可以连接到数据库A ，然后将文件从数据库B传输到数据库C在此示例中，数据库A是第三方，因为它既不是传输文件的源也不是目标。

第三方文件传输可以推送和拉取文件。继续前面的示例，如果您具有从A到B或C的数据库链接，并且该数据库具有指向其他数据库的数据库链接，则可以执行第三方文件传输。数据库A不需要与B和C的数据库链接。

例如，如果您有从A到B数据库链接，以及从B到C另一个数据库链接，那么您可以在A处运行以下过程以将文件从B传输到C ：

DBMS_FILE_TRANSFER.PUT_FILE@B（...）

此配置推送文件。

或者，如果您有从A到C数据库链接，以及从C到B另一个数据库链接，那么您可以在数据库A上运行以下过程将文件从B传输到C ：

DBMS_FILE_TRANSFER.GET_FILE@C（...）

此配置提取文件。

14.9.2.1.1 FMON

FMON是FILE_MAPPING初始化参数设置为true时由数据库启动的后台进程。当发生变化时，FMON会构建地图信息并刷新映射信息。

FMON负责：

  • 构建映射信息，存储在SGA中。此信息由以下结构组成：

    • 档

    • 文件系统范围

    • 分子

    • 子元素

    这些结构在“ 映射结构 ”中进行了解释。

  • 由于以下原因而发生更改时刷新映射信息：

    • 对数据文件的更改（大小）

    • 添加或删除数据文件

    • 存储配置的更改（不常见）

  • 将映射信息保存在数据字典中，以维护在启动和关闭操作中持久保存的信息视图

  • 在实例启动时将映射信息恢复到SGA中。这避免了在每次实例启动时对映射信息进行潜在昂贵的完全重建的需要。

您可以使用使用DBMS_STORAGE_MAP程序包调用的过程来帮助控制此映射。

14.9.2.1.2外部过程（FMPUTL）

FMON产生一个名为FMPUTL的外部非Oracle数据库进程，该进程直接与供应商提供的映射库进行通信。

此过程通过I / O堆栈的所有级别获取映射信息，假设所有级别都存在映射库。在某些平台上，外部进程要求SETUID位设置为ON因为需要root权限才能映射I / O映射堆栈的所有级别。

外部进程负责发现映射库并将其动态加载到其地址空间中。

14.9.2.1.3映射库

Oracle数据库使用映射库来发现特定映射库拥有的元素的映射信息。

通过这些映射库，可以传递有关各个I / O堆栈元素的信息。此信息用于填充用户可以查询的动态性能视图。

映射库由供应商提供。但是，Oracle目前提供EMC存储的映射库。可用于数据库服务器的映射库在名为filemap.ora的特殊文件中标识。

14.9.2.2映射结构

以下是组成映射信息的主要结构：

  • 档

    文件映射结构为文件提供一组属性，包括文件大小，组成文件的文件系统扩展区的数量以及文件类型。

  • 文件系统范围

    注意：

  • 分子

    元素映射结构是描述I / O堆栈中的存储组件的抽象映射结构。元素可以是镜像，条带，分区，RAID5，连接元素和磁盘。这些结构是映射构建块。

  • 子元素

    子元素映射结构描述了元素与I / O映射堆栈中的下一个元素之间的链接。此结构包含子元素编号，大小，子元素所在的元素名称以及元素偏移量。
