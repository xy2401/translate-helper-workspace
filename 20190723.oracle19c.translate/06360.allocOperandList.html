4.1 allocOperandList

目的

使用allocOperandList子句来控制通道上的选项，这是RMAN和数据库实例之间的连接。

句法

allocOperandList :: =


说明allocoperandlist.eps的说明

（ connectStringSpec :: = ， formatSpec :: = ， sizeSpec :: = ， toDestSpec :: = ）

语义

allocOperandList

语法元素 描述

CONNECT connectStringSpec

指定RMAN进行备份或还原操作的数据库实例的连接字符串。使用此参数可以在Oracle RAC配置中的不同实例之间分配工作。

如果未指定此参数，并且未指定AUXILIARY选项，则RMAN将对命令行CONNECT参数或CONNECT命令指定的目标数据库实例执行所有操作。通常，不要将CONNECT参数与AUXILIARY选项一起使用。

另请参见： connectStringSpec和RMAN

辅助格式

指定在辅助实例上创建的映像副本的格式。

RMAN必须通过CONNECT AUXILIARY连接到辅助实例，并且可以访问辅助通道。

   formatSpec

指定辅助实例上的映像副本名称的模式。

该路径必须在辅助主机上有效。

另请参见： formatSpec用于有效的替换变量

新

在由指定的目录中创建映像副本 DB_CREATE_FILE_DEST辅助实例的初始化参数。

映像副本名称采用Oracle托管文件格式。

FORMAT formatSpec

指定用于在此通道上创建的名称备份片段或图像副本的格式。例4-1说明了这种技术。

如果您分配多个磁盘通道并希望每个通道写入其他目录，则FORMAT参数很有用。CONFIGURE CHANNEL或ALLOCATE CHANNEL指定的FORMAT参数在语义上等同于BACKUP命令中指定的FORMAT参数（而不是forRecoveryOfSpec中的DATAFILECOPY FORMAT参数）。如果在BACKUP命令中指定FORMAT参数，则它将覆盖CONFIGURE CHANNEL或ALLOCATE CHANNEL指定的FORMAT参数。

如果未指定FORMAT ，则RMAN默认使用%U ，这保证了备份文件名称的唯一名称。如果配置了快速恢复区域，则RMAN会在默认磁盘位置创建备份文件。否则，默认磁盘位置是特定于操作系统的（例如， ?Solaris上的/dbs ）。

您最多可以指定四个FORMAT字符串。仅当BACKUP COPIES ， SET BACKUP COPIES或CONFIGURE ...时，RMAN才使用第二个，第三个和第四个值CONFIGURE ...BACKUP COPIES生效。在选择用于每个备份片段的格式时，RMAN使用副本1的第一个格式值，副本2的第二个格式值，依此类推。如果格式值的数量超过副本数，则不使用额外的格式。如果格式值的数量小于副本数，则RMAN将重用格式值，从第一个开始。

由于通道对应于目标数据库上的服务器会话，因此FORMAT字符串必须使用目标主机的约定，而不是客户端主机。例如，如果RMAN客户端在Windows主机上运行且目标数据库在Linux主机上运行，则FORMAT字符串必须符合Linux文件系统或原始设备的命名约定。

另请参阅： formatSpec了解可用的FORMAT参数

到达 TO DESTINATION 到达 TO DESTINATION

指定创建备份的目录。此参数对磁盘而不是SBT通道有效。备份文件在Oracle Managed Files（OMF）目录中创建。仅当指定的TO DESTINATION不存在备份时，备份才会跳过文件。

MAXOPENFILES integer

控制BACKUP命令在任何给定时间可以打开的最大输入文件数（默认值为8）。使用此参数可以防止将大量文件备份到单个备份集时，“打开文件太多”错误消息。

MAXPIECESIZE sizeSpec

指定在此通道上创建的每个备份片的最大大小。例4-2说明了这种技术。指定大小（以字节为单位），千字节（ K ），兆字节（ M ）或千兆字节（ G ）。默认设置以字节为单位，向下舍入为千字节。例如，如果将MAXPIECESIZE设置为5000，则RMAN将最大块大小设置为4千字节，即5000字节的较低千字节边界。

注意：您不能使用BACKUP ...SECTION SIZE的MAXPIECESIZE SECTION SIZE 。

PARMS ' channel_parms '

指定sbt通道的参数。例4-3说明了这种技术。如果已指定DEVICE TYPE DISK请勿使用此特定于端口的字符串。

您可以将以下格式用于通道参数：

  • 'ENV=( var1=val1 , var2=val2 ,...)“

    指定所需的一个或多个环境变量与此RMAN客户端对应的服务器会话中的媒体管理器。由于RMAN是客户端程序，因此可以使用ENV参数来设置代表RMAN客户端执行操作的服务器会话特定变量，例如， PARMS 'ENV=(OB_DEVICE_1=tape1)' 。

  • 'SBT_LIBRARY= lib_name '

    如果媒体管理库支持它（例如，OSB云模块），则必须使用SBT_PARMS参数指定环境变量。在所有其他场景中，您使用ENV变量。

    指定在此sbt通道上使用的媒体库，例如， PARMS="SBT_LIBRARY=/oracle/lib/mmv.so" 。默认库是特定于操作系统的（例如，Linux上的libobk.so和Windows上的ORASBT.DLL 。

在Windows上，使用ENV设置的参数对所有通道都可见。如果通道清除或覆盖另一个通道正在使用的环境变量，则可能会导致不确定行为。此限制是由Windows上的Oracle体系结构引起的。建议您使用SEND命令为SBT库设置环境变量，而不是在PARAMS选项中使用ENV 。

另请参阅： Oracle数据库备份和恢复用户指南 ，了解如何集成媒体管理库

RATE sizeSpec

设置RMAN在此通道上每秒读取的最大字节数（默认值），千字节（ K ），兆字节（ M ）或千兆字节（ G ）。此参数设置读取的字节数的上限，以便RMAN不会占用太多磁盘带宽并降低性能。

SEND ' command '

将特定于供应商的命令字符串发送到所有已分配的通道。例如，您可以使用SEND 'OB_MEDIA_FAMILY datafile_mf'指定Oracle Secure Backup介质系列。

在Windows上，要将命令字符串发送到SBT库，建议您使用SEND选项而不是在PARAMS选项中使用ENV 。

另请参阅：您的媒体管理器文档，以确定是否支持此功能以及何时使用它。

toDestSpec

本小节为备份片或映像副本指定目录或自动存储管理（ASM）磁盘组。有关语法图，请参阅toDestSpec :: = 。

例子

示例4-1指定磁盘备份的默认位置

此示例分配指定非默认格式的磁盘通道，然后将数据库备份到非默认位置。

RUN {ALLOCATE CHANNEL d1 DEVICE TYPE DISK FORMAT ='/ disk1 / bkup_％U';备份数据库; }

示例4-2设置备份件的最大尺寸

此示例手动分配SBT通道，该通道指定Oracle Secure Backup磁带驱动器，并进行整个数据库备份。MAXPIECESIZE参数指定写入磁带的备份片不能超过800 MB。

RUN {ALLOCATE CHANNEL c1 DEVICE TYPE sbt PARMS'SBT_LIBRARY = / usr / local / oracle / backup / lib / libobk.so，ENV =（OB_DEVICE_1 = stape1）'MAXPIECESIZE 800M;备份数据库; }

例4-3设置SBT通道参数

此示例将默认SBT通道配置为使用名为stape1的Oracle Secure Backup磁带驱动器，并使用默认通道进行数据库备份：

配置默认设备类型为sbt; CONFIGURE CHANNEL DEVICE TYPE sbt PARMS'ENV =（OB_DEVICE_1 = stape1）';备份数据库;

稍后您决定备份数据库以驱动stape2 。以下示例使用手动分配的SBT通道将数据库备份到stape2 。 

RUN {ALLOCATE CHANNEL st2 DEVICE TYPE sbt PARMS'ENV =（OB_DEVICE_1 = stape2）';备份数据库; }
