87 DBMS_IOT

DBMS_IOT包创建一个表，使用ANALYZE命令可以在其中放置对索引组织表的链接行的引用。DBMS_IOT还可以创建一个异常表，在enable_constraint操作期间可以向其中放置对违反约束的索引组织表的行的引用。

数据库安装期间未加载DBMS_IOT 。要安装DBMS_IOT,运行ADMIN目录中提供的dbmsiotc.sql 。

本章包含以下主题：

  • DBMS_IOT子程序摘要

    注意：

    通过在Oracle Database 8.1中引入IOT的逻辑rowid，您不再需要使用此包中包含的过程，这些过程仅为了向后兼容而保留。但是，运行Oracle Database Release 8.0的服务器需要它。

87.1.1 BUILD_CHAIN_ROWS_TABLE过程

此过程创建一个表，使用ANALYZE命令可以在其中放置对索引组织表的链接行的引用。

句法

DBMS_IOT.BUILD_CHAIN_ROWS_TABLE（所有者IN VARCHAR2，iot_name IN VARCHAR2，chainrow_table_name IN VARCHAR2默认'IOT_CHAINED_ROWS'）;

参数

表87-2 BUILD_CHAIN_ROWS_TABLE过程参数

参数 描述

所有者

索引组织表的所有者。

iot_name

索引组织的表名。

chainrow_table_name

链式行表的目的名称。

使用说明

您应该为每个索引组织表创建一个单独的链式行表，以容纳其主键。

例子

CREATE TABLE l（char（16），b char（16），c char（16），d char（240），PRIMARY KEY（a，b，c））ORGANIZATION INDEX pctthreshold 10 overflow; EXECUTE DBMS_IOT.BUILD_CHAIN_ROWS_TABLE（'SYS'，'L'，'LC'）;

使用以下列创建链式行表：

列名是否为空？键入------------------------------ -------- ---- OWNER_NAME VARCHAR2（30）TABLE_NAME VARCHAR2 （30）CLUSTER_NAME VARCHAR2（30）PARTITION_NAME VARCHAR2（30）SUBPARTITION_NAME VARCHAR2（30）HEAD_ROWID ROWID TIMESTAMP DATE CHAR（16）B CHAR（16）C CHAR（16）

87.1.2 BUILD_EXCEPTIONS_TABLE过程

此过程创建一个异常表。

在执行以下SQL语句期间，可以将违反约束的索引组织表的行放入此表中：

  • ALTER TABLE ...ENABLE CONSTRAINT ......EXCEPTIONS INTO

  • ALTER TABLE ...ADD CONSTRAINT ......EXCEPTIONS INTO

句法

DBMS_IOT.BUILD_EXCEPTIONS_TABLE（所有者IN VARCHAR2，iot_name IN VARCHAR2，exceptions_table_name IN VARCHAR2默认'IOT_EXCEPTIONS'）;

参数

表87-3 BUILD_EXCEPTIONS_TABLE过程参数

参数 描述

所有者

索引组织表的所有者。

iot_name

索引组织的表名。

exceptions_table_name

异常表的目的名称。

使用说明

您应该为每个索引组织表创建一个单独的异常表以容纳其主键。

例子

EXECUTE DBMS_IOT.BUILD_EXCEPTIONS_TABLE（'SYS'，'L'，'LE'）;

前面的索引组织表的异常表，包含以下列：

列名是否为空？键入------------------------------ -------- ---- ROW_ID VARCHAR2（30）OWNER VARCHAR2 （30）TABLE_NAME VARCHAR2（30）CONSTRAINT VARCHAR2（30）A CHAR（16）B CHAR（16）C CHAR（16）
