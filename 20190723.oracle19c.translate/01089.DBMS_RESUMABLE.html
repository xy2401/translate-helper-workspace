139 DBMS_RESUMABLE

使用DBMS_RESUMABLE程序包，您可以在执行很长时间后暂停空间不足或达到空间限制的大型操作，修复问题并使语句恢复执行。通过这种方式，您可以编写应用程序而无需担心遇到与空间相关的错误。

本章包含以下主题：

  • 操作说明

  • DBMS_RESUMABLE子程序摘要

139.1 DBMS_RESUMABLE操作说明

暂停语句时，应将暂停记录在警报日志中。您还应该在语句暂停时注册要执行的过程。使用视图，您可以监视语句的进度并指示语句当前是在执行还是暂停。

挂起语句会自动导致暂停事务。因此，所有事务资源都在语句暂停和恢复期间保留。当错误条件消失时，挂起的语句将自动恢复执行。可恢复的空间分配可以在执行期间暂停和恢复多次。

暂停超时间隔与可恢复空间分配相关联。在超时间隔（默认为两小时）暂停的可恢复空间分配唤醒并向用户返回异常。可以使用DBMS_RESUMABLE.ABORT()过程强制挂起的语句抛出异常。

139.2.1中止程序

此过程中止暂停的可恢复空间分配。

参数session_id是执行语句的会话ID。对于并行DML / DDL， session_id是参与并行DML / DDL的任何会话ID。此操作保证成功。可以在AFTER SUSPEND触发器的内部或外部调用该过程。

句法

DBMS_RESUMABLE.ABORT（session_id IN NUMBER）;

参数

表139-2 ABORT过程参数

参数 描述

SESSION_ID

可恢复空间分配的会话标识符。

使用说明

要调用ABORT过程，您必须是具有session_id,的会话的所有者session_id,具有ALTER SYSTEM权限或者是DBA。

此函数返回具有session_id.的会话的可恢复空间分配的当前超时值session_id.

DBMS_RESUMABLE.GET_SESSION_TIMEOUT（session_id IN NUMBER）返回号码;

参数

表139-3 GET_SESSION_TIMEOUT函数参数

参数 描述

返回值

数

具有session_id.的会话的可恢复空间分配的当前超时值session_id.超时以秒为单位返回。

139.2.3 GET_TIMEOUT功能

此函数返回当前会话的可恢复空间分配的当前超时值。

句法

DBMS_RESUMABLE.GET_TIMEOUT返回号码;

返回值

表139-5 GET_TIMEOUT函数返回值

回报价值 描述

数

当前会话的可恢复空间分配的当前超时值。返回值以秒为单位。

使用说明

如果当前会话未启用可恢复，则GET_TIMEOUT函数返回-1。

139.2.4 SET_SESSION_TIMEOUT过程

此过程为session_id的会话设置可恢复空间分配的超时。

新的超时设置立即应用于会话。如果session_id不存在，则不执行任何操作。

句法

DBMS_RESUMABLE.SET_SESSION_TIMEOUT（session_id IN NUMBER，超时IN NUMBER）;

参数

超时

139.2.6 SPACE_ERROR_INFO函数

此函数查找错误堆栈中与空间相关的错误。

如果找不到与空格有关的错误，则返回FALSE.否则，返回TRUE并返回有关导致空间错误的特定对象的信息。

句法

DBMS_RESUMABLE.SPACE_ERROR_INFO error_type OUT VARCHAR2，object_type OUT VARCHAR2，object_owner OUT VARCHAR2，table_space_name OUT VARCHAR2，object_name OUT VARCHAR2，sub_object_name OUT VARCHAR2）RETURN BOOLEAN;

参数

表139-8 SPACE_ERROR_INFO函数参数

参数 描述

ERROR_TYPE

空间错误类型。它将是以下之一：

  • 没有更多的空间

  • 达到最大限度

  • 空间配额超出

OBJECT_TYPE

对象类型。它将是以下之一：

  • 表

  • 指数

  • 簇

  • 表空间

  • ROLLBACK SEGMENT

  • UNDO SEGMENT

  • LOB SEGMENT

  • TEMP SEGMENT

  • 索引分区

  • 表格分区

  • LOB PARTITION

  • 表格下载

  • INDEX SUBPARTITION

  • LOB SUBPARTITION

如果不适用，该类型也可以为NULL 。

OBJECT_OWNER

对象的所有者。如果无法确定，则为NULL 。

table_space_name

对象所在的表空间。如果无法确定，则为NULL 。

OBJECT_NAME

回滚段，临时段，表，索引或集群的名称。

sub_object_name

LOB, TABLE,或INDEX. NULL的分区名称或子分区名称INDEX. NULL如果无法确定，则为INDEX. NULL 。
