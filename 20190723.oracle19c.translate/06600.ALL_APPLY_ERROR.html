2.34 ALL_APPLY_ERROR

ALL_APPLY_ERROR显示有关应用进程生成的错误事务的信息，这些错误事务使来自当前用户可访问的队列的消息出列。

相关观点

  • DBA_APPLY_ERROR显示有关数据库中所有应用进程生成的错误事务的信息。

  • USER_APPLY_ERROR显示有关当前用户可见的应用进程生成的错误事务的信息。此视图不显示SOURCE_ROOT_NAME列。

柱 数据类型 空值 描述

APPLY_NAME

VARCHAR2（128）

处理事务的本地数据库中的应用进程的名称

QUEUE_NAME

VARCHAR2（128）

事务从中出列的本地数据库中的队列名称

QUEUE_OWNER

VARCHAR2（128）

事务出列的本地数据库中队列的所有者

LOCAL_TRANSACTION_ID

VARCHAR2（22）

错误事务的本地事务ID

SOURCE_DATABASE

VARCHAR2（128）

事务发起的数据库

SOURCE_TRANSACTION_ID

VARCHAR2（128）

源数据库中的原始事务ID

SOURCE_COMMIT_SCN

数

源数据库中事务的原始提交系统更改号（SCN）

MESSAGE_NUMBER

数

引发错误的事务中消息的标识符

ERROR_NUMBER

数

事务引发的错误的错误号

错误信息

VARCHAR2（4000）

事务引发的错误的错误消息

RECIPIENT_ID

数

应用交易的原始用户的用户标识

收件人姓名

VARCHAR2（128）

应用事务的原始用户的名称

MESSAGE_COUNT

数

错误事务中的消息总数

ERROR_CREATION_TIME

日期

创建错误的时间

SOURCE_COMMIT_POSITION

RAW（64）

交易的原始提交位置

ERROR_TYPE

VARCHAR2（11）

如果应用进程可以访问错误事务中的所有LCR，则为NULL。

如果应用进程无法访问错误事务中的所有LCR，则EAGER ERROR 。此错误类型通常表示应用进程正在大型事务中应用LCR。当ERROR_TYPE为EAGER ERROR ，请使用Oracle Database XStream Guide中的说明管理错误事务。

RECORD LCR表示已按用户指定的错误处理配置记录了单个LCR

RECORD TXN NO LCRS表示标识的事务遇到错误，并且只有事务ID记录为用户指定的错误处理配置所请求的

RECORD TXN WITH LCRS表示遇到的已识别事务和错误。整个事务按用户指定的错误处理配置的请求记录。

UNHANDLED ERRORS NO LCR表示标识的事务遇到错误，并且没有为此处理程序指定错误处理。此交易不记录LCR。

DISCARDED用于标记记录和丢弃的LCR。

SOURCE_ROOT_NAME

VARCHAR2（128）

源根数据库的全局名称

ERROR_POSITION

RAW（64）

发生错误的LCR位置
