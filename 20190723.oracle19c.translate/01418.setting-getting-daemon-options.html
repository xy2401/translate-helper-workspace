2.5.3设置和获取守护程序的选项

在启动守护程序之前设置守护程序选项。启动守护程序后，可以随时重置守护程序选项。

要设置守护程序选项：

  1. 使用–set选项设置守护程序选项。
    设置选项如下：
    $ ./orachk -set“ option_1 = option_1_value ”
    $ ./exachk -set“ option_1 = option_1_value ”
    使用以分号分隔的name = value格式设置多个选项，如下所示：
    $ ./orachk -set“ option_1 = option_1_value ; option_2 = option_2_value ; option_n = option_n_value ”
    $ ./exachk -set“ option_1 = option_1_value ; option_2 = option_2_value ; option_n = option_n_value ”
  • AUTORUN_SCHEDULE
    使用AUTORUN_SCHEDULE守护程序选项计划定期重复运行状况检查。
  • AUTORUN_FLAGS
    AUTORUN_FLAGS守护程序选项确定运行状况检查的方式。
  • NOTIFICATION_EMAIL
    设置NOTIFICATION_EMAIL守护程序选项以向您指定的收件人发送电子邮件通知。
  • collection_retention
    设置collection_retention守护程序选项以清除早于指定天数的运行状况检查集合结果。
  • PASSWORD_CHECK_INTERVAL
    PASSWORD_CHECK_INTERVAL守护程序选项定义守护程序验证在第一次启动守护程序时输入的密码的频率（以小时为单位）。
  • 为守护程序设置多个选项配置文件
    每个服务器只使用一个守护程序进程。不要在群集中的多个数据库上启动单个守护程序，也不要在同一数据库上启动多个守护程序。
  • 获取守护程序的现有选项
    查询为守护程序选项设置的值。

2.5.3.1 AUTORUN_SCHEDULE

使用AUTORUN_SCHEDULE守护程序选项计划定期重复运行状况检查。

要安排定期运行状况检查运行：

  1. 设置AUTORUN_SCHEDULE选项，如下所示：
    AUTORUN_SCHEDULE = hour minute day month day_of_week
    哪里：

    • minute是0-59（可选。如果省略，则使用0）

    • hour是0-23

    • day是1-31

    • month是1-12

    • day_of_week是0-6，其中0 =星期日，6 =星期六

    使用星号（*）作为通配符指定以逗号分隔的多个值。

    表2-1 AUTORUN_SCHEDULE

    例 结果

    “AUTORUN_SCHEDULE = 0,15,30,45 * * * *”

    每15分钟运行一次。

    “AUTORUN_SCHEDULE = * * * *”

    每小时运行一次。

    “AUTORUN_SCHEDULE = 3 * * 0”

    每周日凌晨3点运行。

    “AUTORUN_SCHEDULE = 2 * * 1,3,5”

    周一，周三和周五凌晨2点运行。

    “AUTORUN_SCHEDULE = 4 1 * *”

    每个月的第一天凌晨4点运行。

    “AUTORUN_SCHEDULE = 8,20 * * 1,2,3,4,5”

    每周一，周二，周三，周四和周五上午8点和晚上8点运行。
例如：
$ ./orachk -set“AUTORUN_SCHEDULE = 3 * * 0”
$ ./exachk -set“AUTORUN_SCHEDULE = 3 * * 0”

2.5.3.2 AUTORUN_FLAGS

AUTORUN_FLAGS守护程序选项确定运行状况检查的方式。

要配置运行状况检查的运行方式：

  1. 设置AUTORUN_FLAGS选项如下：
    AUTORUN_FLAGS = flags

    其中， flags可以是有效命令行标志的任意组合。

    表2-2 AUTORUN_FLAGS

    例 结果

    “AUTORUN_FLAGS = -profile dba”

    仅运行dba配置文件检查。

    “AUTORUN_FLAGS = -profile sysadmin -tag syadmin”

    仅运行dba配置文件检查并使用值sysadmin标记输出。

    -excludeprofile ebs

    运行除ebs配置文件中的检查之外的所有检查。

例如：
$ ./orachk -set“AUTORUN_FLAGS = -profile sysadmin -tag sysadmin”
$ ./exachk -set“AUTORUN_FLAGS = -profile sysadmin -tag sysadmin”

2.5.3.3 NOTIFICATION_EMAIL

设置NOTIFICATION_EMAIL守护程序选项以向您指定的收件人发送电子邮件通知。

配置电子邮件通知：

每次运行状况检查运行完成或守护程序遇到问题时，守护程序都会通知收件人。
  1. 指定以逗号分隔的电子邮件地址列表，如下所示：
    $ ./orachk -set“NOTIFICATION_EMAIL = some.person @ acompany.com，another.person @acompany.com”
    $ ./exachk -set“NOTIFICATION_EMAIL = some.person @ acompany.com，another.person @ acompany.com”
  2. 使用–testemail选项测试电子邮件通知配置，如下所示：
    $ ./orachk -testemail all
    $ ./exachk -testemail全部

运行第一次运行状况检查后，守护程序会通过附加的报告输出通知收件人。

对于在第一次电子邮件通知之后运行的后续运行状况检查，守护程序会将最近运行之间的差异摘要通过电子邮件发送给NOTIFICATION_EMAIL列表中指定的所有收件人。

2.5.3.4 collection_retention

设置collection_retention守护程序选项以清除早于指定天数的运行状况检查集合结果。

配置收集保留期：

  1. 设置collection_retention选项，如下所示：
    collection_retention = number_of_days

    如果未设置此选项，则守护程序不会清除过时集合。

  2. 根据以下内容将collection_retention选项设置为适当的天数：

    • 预定馆藏的频率

    • 收集结果的大小

    • 可用磁盘空间

例如：
$ ./orachk -set“collection_retention = 60”
$ ./exachk -set“collection_retention = 60”

2.5.3.5 PASSWORD_CHECK_INTERVAL

PASSWORD_CHECK_INTERVAL守护程序选项定义守护程序验证在第一次启动守护程序时输入的密码的频率（以小时为单位）。

如果由于密码更改而找到无效密码，则守护程序将停止，在守护程序日志中生成一个条目，然后向NOTIFICATION_EMAIL选项中指定的收件人发送电子邮件通知消息。

配置密码验证频率：

  1. 设置PASSWORD_CHECK_INTERVAL选项，如下所示：
    PASSWORD_CHECK_INTERVAL = number_of_hours

    如果未设置PASSWORD_CHECK_INTERVAL选项，则守护程序无法主动检查密码有效性，并且在密码更改后下次守护程序尝试运行时将失败。使用PASSWORD_CHECK_INTERVAL选项可以使您采取纠正措施并使用正确的密码重新启动守护程序，而不是使集合失败。

  2. 根据以下内容将PASSWORD_CHECK_INTERVAL选项设置为适当的小时数：

    • 预定馆藏的频率

    • 密码更改政策

例如：
$ ./orachk -set“PASSWORD_CHECK_INTERVAL = 1”
$ ./exachk -set“PASSWORD_CHECK_INTERVAL = 1”

2.5.3.6为守护进程设置多个选项配置文件

每个服务器只使用一个守护程序进程。不要在群集中的多个数据库上启动单个守护程序，也不要在同一数据库上启动多个守护程序。

如果守护程序检测到本地运行的另一个Oracle ORAchk或Oracle EXAchk守护程序进程，则守护程序无法启动。

使用同一个守护程序定义多个不同的运行配置文件。通过定义多个不同的运行配置文件，您可以使用不同的守护程序选项（例如，不同的计划，电子邮件通知和自动运行标志）运行多个不同的运行状况检查。守护程序管理所有配置文件。

要为守护程序设置多个选项配置文件：

  1. 在–set选项之前使用–id id选项定义守护程序选项配置文件。

    其中， id是配置文件的名称

    $ ./orachk -id id -set“ option = value ”
    $ ./exachk -id id -set“ option = value ”

例如，如果数据库管理员想要在dba配置文件中运行检查，并且系统管理员想要在sysadmin配置文件中运行检查，则使用profiles选项配置守护程序。

定义数据库管理员配置文件如下：
$ ./orachk -id dba -set“NOTIFICATION_EMAIL = dba @ example.com; \ AUTORUN_SCHEDULE = 4,8,12,16,20 * * *; AUTORUN_FLAGS = -profile dba -tag dba; \ collection_retention = 30”创建的notification_email for ID [dba]为ID创建autorun_schedule [dba]为ID创建autorun_flags [dba]为ID [dba]创建collection_retention
$ ./exachk -id dba -set“NOTIFICATION_EMAIL = dba @ example.com; \ AUTORUN_SCHEDULE = 4,8,12,16,20 * * *; AUTORUN_FLAGS = -profile dba -tag dba; \ collection_retention = 30”创建的notification_email for ID [dba]为ID创建autorun_schedule [dba]为ID创建autorun_flags [dba]为ID [dba]创建collection_retention
定义系统管理员配置文件如下：
$ ./orachk -id sysadmin -set“NOTIFICATION_EMAIL = sysadmin @ example.com; \ AUTORUN_SCHEDULE = 3 * * 1,3,5; AUTORUN_FLAGS = -profile sysadmin -tag sysadmin; \ collection_retention = 60”为ID [sysadmin创建] notification_email ]为ID [sysadmin]创建了autorun_schedule为ID [sysadmin]创建了autorun_flags为ID [sysadmin]创建了collection_retention
$ ./exachk -id sysadmin -set“NOTIFICATION_EMAIL = sysadmin @ example.com; \ AUTORUN_SCHEDULE = 3 * * 1,3,5; AUTORUN_FLAGS = -profile sysadmin -tag sysadmin; \ collection_retention = 60”为ID创建notification_email [sysadmin ]为ID [sysadmin]创建了autorun_schedule为ID [sysadmin]创建了autorun_flags为ID [sysadmin]创建了collection_retention

2.5.3.7获取守护程序的现有选项

查询为守护程序选项设置的值。

要查询值，请使用
[-id ID ] -get option | all

哪里：

  • ID是守护程序选项配置文件

  • option是您要检索的特定守护程序选项

  • all返回所有选项的值

要获取守护程序的现有选项：

  1. 要获取特定的守护程序选项：
    例如：
    $ ./orachk -get NOTIFICATION_EMAIL ID：orachk.default -------------------------------------- ---- notification_email = some.body@example.com
    $ ./exachk -get NOTIFICATION_EMAIL ID：exachk.default -------------------------------------- ---- notification_email = some.body@example.com
  2. 要查询多个守护程序选项配置文件：
    例如：
    $ ./orachk -get NOTIFICATION_EMAIL ID：orachk.default -------------------------------------- ---- notification_email = some.body@example.com ID：dba ---------------------------------- -------- notification_email = dba@example.com ID：sysadmin -------------------------------- ---------- notification_email = sysadmin@example.com
    $ ./exachk -get NOTIFICATION_EMAIL ID：exachk.default -------------------------------------- ---- notification_email = some.person@example.com ID：dba ---------------------------------- -------- notification_email = dba@example.com ID：sysadmin -------------------------------- ---------- notification_email = sysadmin@example.com
  3. 要将请求限制为特定守护程序选项配置文件，请使用–id ID -get option选项：

    例如：

    要获取名为dba的守护程序配置文件的NOTIFICATION_EMAIL ：

    $ ./orachk -id dba -get NOTIFICATION_EMAIL ID：dba ------------------------------------- ----- notification_email = dba@example.com
    $ ./exachk -id dba -get NOTIFICATION_EMAIL ID：dba ------------------------------------- ----- notification_email = dba@example.com
  4. 要设置所有选项，请使用–get all选项：
    例如：
    $ ./orachk -get all ID：orachk.default -------------------------------------- ---- notification_email = some.body@example.com autorun_schedule = 3 * * 0 collection_retention = 30 password_check_interval = 1
    $ ./exachk -get all ID：exachk.default -------------------------------------- ---- notification_email = some.body@example.com autorun_schedule = 3 * * 0 collection_retention = 30 password_check_interval = 1
  5. 要查询所有守护程序选项配置文件：
    例如：
    $ ./orachk -get all ID：orachk.default -------------------------------------- ---- notification_email = some.body@example.com autorun_schedule = 3 * * 0 collection_retention = 30 password_check_interval = 12 ID：dba ---------------------- -------------------- notification_email = dba@example.com autorun_schedule = 4,8,12,16,20 * * * autorun_flags = -profile dba  -  tag dba collection_retention = 30 password_check_interval = 1 ID：sysadmin ------------------------------------------ notification_email = sysadmin@example.com autorun_schedule = 3 * * 1,3,5 autorun_flags = -profile sysadmin -tag sysadmin collection_retension = 60 password_check_interval = 1
    $ ./exachk -get all ID：exachk.default -------------------------------------- ---- notification_email = some.body@example.com autorun_schedule = 3 * * 0 collection_retention = 30 password_check_interval = 1 ID：dba ---------------------- -------------------- notification_email = dba@example.com autorun_schedule = 4,8,12,16,20 * * * autorun_flags = -profile dba  -  tag dba collection_retention = 30 password_check_interval = 1 ID：sysadmin ------------------------------------------ notification_email = sysadmin@example.com autorun_schedule = 3 * * 1,3,5 autorun_flags = -profile sysadmin -tag sysadmin collection_retension = 60 password_check_interval = 1
  6. 要获取为守护程序配置文件设置的所有选项，例如，名为dba的守护程序配置文件：
    $ ./orachk -id dba -get all ID：dba ------------------------------------- ----- notification_email = dba@example.com autorun_schedule = 4,8,12,16,20 * * * autorun_flags = -profile dba  -  tag dba collection_retention = 30 password_check_interval = 1
