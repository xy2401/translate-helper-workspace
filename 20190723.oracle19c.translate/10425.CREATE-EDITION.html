创造版

目的

此声明将创建一个新版本作为现有版本的子版本。版本可以在数据库中具有两个或更多版本的相同可编辑对象。创建版本时，它会立即继承其父版本的所有可编辑对象。以下对象类型是可编辑的：

  • 代名词

  • 视图

  • 功能

  • 程序

  • 包装（规格和正文）

  • 类型（规格和正文）

  • 图书馆

  • 触发

可编辑对象是启用版本的模式中的上述可编辑对象类型之一的对象。在数据库中拥有这些对象的多个版本的能力极大地促进了在线应用程序升级。

注意：

上面未列出的所有数据库对象类型都不可编辑。对数据库中的所有版本都可以立即看到对不可编辑的对象类型的更改。

每个新创建或升级的Oracle数据库都有一个名为ORA$BASE默认版本，它作为使用CREATE EDITION语句创建的第一版的父版本。随后，您可以使用ALTER DATABASE DEFAULT EDITION语句将用户定义的版本指定为数据库默认版本。

也可以看看：

  • “Oracle数据库开发指南” ，可以更全面地讨论可编辑对象类型和版本

  • ALTER DATABASE “ DEFAULT EDITION子句 ”，用于指定将版本指定为数据库的默认版本的信息

先决条件

要创建版本，您必须具有CREATE ANY EDITION系统特权，可以直接授予或通过角色授予。要将版本创建为另一版本的子版本，您必须具有父版本的USE对象特权。

句法

create_edition :: =


说明create_edition.eps的说明

语义

版

指定要创建的版本的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

要查看为数据库创建的版本，请查询DBA_OBJECTS或ALL_OBJECTS数据字典视图的EDITION_NAME列。

创建版本时，系统会自动在您创建的版本上授予USE对象权限WITH GRANT OPTION 。

注意：

Oracle强烈建议您不要使用前缀ORA ， ORACLE ， SYS ， DBA和DBMS命名版本，因为这些前缀保留供内部使用。

作为儿童的条款

如果使用此子句，则新版本将创建为parent_edition的子级。如果省略此子句，则新版本将作为叶版本的子项创建。在创建时，新版本从其父版本继承了所有已编辑的对象。

对版本的限制

一个版本只能有一个子版本。如果为parent_edition指定已具有子版本的版本，则会返回错误。

例子

以下非常简单的示例旨在显示创建和使用版本的语法。有关使用版本的实际示例，请参阅Oracle数据库开发指南 。

在以下语句中，用户HR被赋予创建和使用版本所需的权限：

授予创建任何版本，降低人力资源的任何版本;格兰特成功了。ALTER USER hr ENABLE EDITIONS;用户改动了。

HR为测试目的创建了新版TEST_ED ：

CREATE EDITION test_ed;

然后， HR会在默认版本ORA$BASE创建一个版本视图ed_view用于测试目的，首先验证当前版本是否为默认版本：

SELECT SYS_CONTEXT（'userenv'，'current_edition_name'）FROM DUAL; SYS_CONTEXT（'USERENV'，'CURRENT_EDITION_NAME'）---------------------------------------- ---------------------------------------- ORA $ BASE 1行被选中。创建版本视图e_view AS SELECT last_name，first_name，email FROM employees;查看已创建。DESCRIBE e_view名称是否为空？输入----------------------------------------- -------- ---------------------------- LAST_NAME NOT NULL VARCHAR2（25）FIRST_NAME VARCHAR2（20）EMAIL NOT NULL VARCHAR2（25）

该视图然后在现实化TEST_ED版时HR使用TEST_ED版和重新创建以不同的形式的视图：

ALTER SESSION SET EDITION = TEST_ED;会话改变了。创建或替换版本视图e_view AS SELECT last_name，first_name，email，salary FROM employees;查看已创建。

TEST_ED版本中的视图还有一个列：

DESCRIBE e_view名称是否为空？输入----------------------------------------- -------- ---------------------------- LAST_NAME NOT NULL VARCHAR2（25）FIRST_NAME VARCHAR2（20）EMAIL NOT NULL VARCHAR2（25）SALARY NUMBER（ 8,2）

ORA$BASE版本中的视图与测试环境保持隔离：

ALTER SESSION SET EDITION = ora $ base;会话改变了。DESCRIBE e_view;名字空？输入----------------------------------------- -------- ---------------------------- LAST_NAME NOT NULL VARCHAR2（25）FIRST_NAME VARCHAR2（20）EMAIL NOT NULL VARCHAR2（25）

即使视图在测试环境中被删除，它仍然保留在ORA$BASE版本中：

ALTER SESSION SET EDITION = TEST_ED;会话改变了。DROP VIEW e_view;视图丢了。ALTER SESSION SET EDITION = ORA $ BASE;会话改变了。DESCRIBE e_view;名字空？输入----------------------------------------- -------- ---------------------------- LAST_NAME NOT NULL VARCHAR2（25）FIRST_NAME VARCHAR2（20）EMAIL NOT NULL VARCHAR2（25）

当完成需要TEST_ED版本的升级测试时，可以删除该版本：

DROP EDITION TEST_ED;
