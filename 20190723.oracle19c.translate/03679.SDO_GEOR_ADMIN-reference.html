8.1 SDO_GEOR_ADMIN.checkSysdataEntries

格式

SDO_GEOR_ADMIN.checkSysdataEntries（）返回SDO_STRING2_ARRAY;

描述

检查USER_SDO_GEOR_SYSDATA视图是否存在任何无效条目。

参数

没有。

使用说明

此函数返回一组逗号分隔的GeoRaster系统数据条目列表，这些条目无效。它会检查以下错误：

  • RDT名称不是唯一的。

  • GeoRaster表不存在。

  • GeoRaster列不存在。

  • GeoRaster对象不存在。

  • GeoRaster对象是非空的或非空的，但RDT不存在。

  • 存在重复的GeoRaster对象（即，RDT和栅格ID的一个或多个非唯一组合）。

在GeoRaster系统数据视图（xxx_SDO_GEOR_SYSDATA）中描述了USER_SDO_GEOR_DATA和ALL_SDO_GEOR_SYSDATA视图。

例子

以下示例检查USER_SDO_GEOR_SYSDATA视图是否存在无效条目。

SELECT * FROM THE（SELECT SDO_GEOR_ADMIN.checkSysdataEntries FROM DUAL）; COLUMN_VALUE ------------------------------------------------- ------------------------------- RDT名称“RDT1”不是唯一的GeoRaster对象GEOR_TEST1。表格1。GEOR：RDT = RDT2 RID = 3与不存在的RDT表相关联！GeoRaster列GEOR_TEST1的规范。TABLE1.c1不正确。GeoRaster对象GEOR_TEST1。TABLE1.geor：RDT = dt3 RID = 2不存在！GeoRaster表GEOR_TEST1.t1不存在！

参数

没有。

  • 系统数据输入错误，GeoRaster对象不存在。

  • GeoRaster对象是非空的或非空的，但RDT不存在。

  • 存在重复的GeoRaster对象（即，RDT和栅格ID的一个或多个非唯一组合）。

  • 有一对未注册的（GeoRaster列，GeoRaster对象）。

例子

以下示例检查GeoRaster系统数据条目和GeoRaster数据。它假定您作为MDSYS用户连接。

8.9 SDO_GEOR_ADMIN.listDanglingRasterData

格式

SDO_GEOR_ADMIN.listDanglingRasterData（）返回SDO_STRING2_ARRAYSET;

描述

检查GeoRaster系统数据条目和GeoRaster数据，并列出所有悬空栅格数据。

参数

没有。

使用说明

对于不存在的GeoRaster对象或未在SYSDATA表中引用的GeoRaster对象，可能存在栅格数据表（RDT）行。与这些行相关联的栅格块被称为悬空块。悬空栅格块会导致RDT中浪费的磁盘空间，尽管只要在RDT上定义了必要的主键，它们就不会出现问题。要在当前模式或所有模式中查找这些悬空块，请调用SDO_GEOR_ADMIN.listDanglingRasterData函数。

在调用此函数之前，应调用SDO_GEOR_ADMIN.registerGeoRasterObjects以注册所有现有的GeoRaster对象。

要从RDT中删除悬空栅格块数据，请删除与SDO_GEOR_ADMIN.listDanglingRasterData函数发现的问题关联的行。

此函数返回一个逗号分隔的悬挂栅格数据列表数组。该列表包含以下信息：

  • 架构名称（仅当您作为MDSYS用户连接时）

  • RDT名称

  • 光栅ID

例子

以下示例列出了当前架构中的所有悬空栅格数据。

SELECT * FROM THE（SELECT SDO_GEOR_ADMIN.listDanglingRasterData FROM DUAL）; COLUMN_VALUE ------------------------------------------------- ------------------------------- SDO_STRING2_ARRAY（'RDT11'，'3'）

8.13 SDO_GEOR_ADMIN.maintainSysdataEntries

格式

SDO_GEOR_ADMIN.maintainSysdataEntries（）返回SDO_STRING2_ARRAY;

描述

检查USER_SDO_GEOR_SYSDATA视图是否存在任何无效条目，并根据需要采取纠正措施。

参数

没有。

使用说明

此函数执行与SDO_GEOR_ADMIN.checkSysdataEntries函数相同的检查，并采取适当的纠正操作（如果有）。对于以下每个错误，该函数执行以下操作：

  • RDT名称不是唯一的。如果您以MDSYS之外的其他用户身份连接，则不执行任何操作;如果您以用户MDSYS身份连接，则会重命名重复的RDT，以使其名称唯一。

  • GeoRaster表不存在。该条目已删除。

  • GeoRaster列不存在。该条目已删除。

  • GeoRaster对象不存在。该条目已删除。

  • GeoRaster对象是非空的或非空的，但RDT不存在。该条目已删除。

  • 存在重复的GeoRaster对象（即，RDT和栅格ID的一个或多个非唯一组合）。该条目已删除。

在GeoRaster系统数据视图（xxx_SDO_GEOR_SYSDATA）中描述了USER_SDO_GEOR_DATA和ALL_SDO_GEOR_SYSDATA视图。

例子

以下示例检查USER_SDO_GEOR_SYSDATA视图是否存在无效条目，并根据需要执行更正操作。

DECLARE ret SDO_STRING2_ARRAY; BEGIN ret：= sdo_geor_admin。MAINTAINSYSDATAENTRIES; for i in 1..ret.count loop dbms_output.put_line（ret（i））;结束循环;结束; / RDT名称GEOR_TEST1。RDT2重命名为GEOR_TEST1。RDT1！sysdata条目（SCHEMA = GEOR_TEST1 RDT = dt1 RID = 1）将被删除！PL / SQL过程成功完成。
