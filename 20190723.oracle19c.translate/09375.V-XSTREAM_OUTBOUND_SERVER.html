9.141 V $ XSTREAM_OUTBOUND_SERVER

V$XSTREAM_OUTBOUND_SERVER显示有关出站服务器的统计信息。出站服务器将LCR发送到XStream客户端应用程序。

注意：

当COMMITTED_DATA_ONLY列是YES在V$XSTREAM_OUTBOUND_SERVER视图中， V$STREAMS_APPLY_SERVER视图提供有关出站服务器进程关于由出站服务器中使用的应用服务器后台进程的其他信息，和信息。

柱 数据类型 描述

SID

数

出站服务器会话的会话ID

SERIAL＃

数

出站服务器会话的序列号

SPID

VARCHAR2（12）

将LCR发送到客户端应用程序的操作系统进程的进程标识号

服务器名称

VARCHAR2（128）

出站服务器的名称

启动时间

日期

客户端应用程序附加到出站服务器的时间

州

VARCHAR2（19）

出站服务器的状态

当COMMITTED_DATA_ONLY列显示YES ，可能存在以下状态：

  • INITIALIZING - 启动出站服务器。

  • IDLE - 由于没有LCR发送到XStream客户端应用程序，因此无法正常工作。

  • GET TRANSACTIONS - 从出站服务器的应用协调器接收事务。

  • SEND TRANSACTION - 将事务发送到XStream客户端应用程序。

  • WAIT FOR NEXT CHUNK -等待一个大事务下一组的LCR的。

  • TRANSACTION CLEANUP - 清理已应用的事务，包括从出站服务器的队列中删除LCR。

  • WAIT FOR CLIENT -等待XStream的客户端应用程序请求更多的LCR。

当COMMITTED_DATA_ONLY列显示NO ，可能存在以下状态：

  • INITIALIZING - 启动出站服务器。

  • INITIALIZING RULE EVALUATION CONTEXT - 初始化上下文以评估出站服务器的规则。

  • IDLE - 由于没有LCR发送到XStream客户端应用程序，因此无法正常工作。

  • BROWSING LCR - 浏览出站服务器的队列以获取下一个LCR。

  • EVALUATING RULES - 根据规则集评估LCR。

  • DEQUEUING LCR - 从出站服务器的队列中取出LCR。

  • SENDING LCR - 将LCR发送到XStream客户端应用程序。

  • WAITING FOR CAPTURE TO TERMINATE -等待拍摄过程中被禁用。

  • SUSPENDED DUE TO A DROPPED SUBSCRIBER -由于连接的用户下降了暂停。例如，可以在拆分或合并操作期间删除订户。

  • SUSPENDED FOR AUTO SPLIT / MERGE正在执行暂停，因为自动分割或合并操作- 。

  • WAITING ON EMPTY QUEUE -等待从捕获过程更LCR的。

  • WAITING FOR CLIENT -等待XStream的客户端应用程序请求更多的LCR。

  • WAITING FOR CAPTURE TO INITIALIZE -等待捕获过程完成数据字典建立。

  • WAITING TO ATTACH TO CAPTURE -等待出站服务器连接到捕获过程。

当状态引用捕获进程时，捕获进程会捕获出站服务器的更改。当状态引用传播时，它是出站服务器将LCR发送到XStream客户端应用程序。

XIDUSN

数

事务ID撤消当前正在处理的事务的段号。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

XIDSLT

数

当前正在处理的交易的交易ID槽号。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

XIDSQN

数

当前正在处理的事务的事务ID序列号。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

COMMITSCN

数

提交当前正在处理的事务的SCN。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

TOTAL_TRANSACTIONS_SENT

数

自上次客户端应用程序连接到出站服务器以来，出站服务器发送到XStream客户端应用程序的事务总数。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

MESSAGE_SEQUENCE

数

出站服务器正在处理的当前LCR的编号。每个事务开始时，此值将重置为1。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

TOTAL_MESSAGES_SENT

数

自上次客户端应用程序连接到出站服务器以来出站服务器发送到XStream客户端应用程序的LCR总数

SEND_TIME

日期

出站服务器将最后一个LCR发送到XStream客户端应用程序的时间

LAST_SENT_MESSAGE_NUMBER

数

出站服务器向XStream客户端应用程序发送的最后一个LCR的消息号

LAST_SENT_MESSAGE_CREATE_TIME

日期

源出口服务器向客户端应用程序发送的最后一个LCR的源数据库的创建时间

ELAPSED_SEND_TIME

数

自上次客户端应用程序连接到出站服务器以来，将LCR发送到XStream客户端应用程序的时间（以百分之一秒为单位）

COMMIT_POSITION

RAW（64）

提交当前正在处理的交易的位置。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

LAST_SENT_POSITION

RAW（64）

发送到XStream客户端应用程序的最后一个LCR的位置。仅当COMMITTED_DATA_ONLY列显示YES才会填充此列。当COMMITTED_DATA_ONLY列显示NO ，此列为NULL 。

BYTES_SENT

数

自上次客户端应用程序连接到出站服务器以来出站服务器发送到XStream客户端应用程序的总字节数

COMMITTED_DATA_ONLY

CHAR（3）

如果出站服务器只能将已提交事务中的LCR发送到XStream客户端应用程序，则为YES 。已提交的事务是一个没有回滚的已组装的非交叉事务。

如果出站服务器可以在尚未提交到XStream客户端应用程序的事务中发送LCR，则为NO 。此模式仅供Oracle内部使用。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
