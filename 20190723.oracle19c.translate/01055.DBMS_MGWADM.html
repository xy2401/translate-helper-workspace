105.1 DBMS_MGWADM安全模型

具有管理权限的用户可以在DBMS_MGWADM运行所有过程。

105.2 DBMS_MGWADM已弃用的子程序

Oracle建议您不要在新应用程序中使用已弃用的过程。对不推荐使用的功能的支持仅用于向后兼容。

Oracle Database 11g第1版已弃用以下子程序，但已被改进的技术取代：

  • ADD_SUBSCRIBER过程 - 改为使用CREATE_JOB过程

  • ALTER_PROPAGATION_SCHEDULE过程 - 改为使用ALTER_JOB过程

  • ALTER_SUBSCRIBER过程 - 改为使用ALTER_JOB过程

  • DB_CONNECT_INFO过程 - 改为使用ALTER_AGENT过程

  • 对于MQSERIES_BASE_JAVA_INTERFACE ，这是使用WebSphere MQ管理工具创建的物理WebSphere MQ队列的名称。

  • 对于JMS_CONNECTION接口和JMS_QUEUE_CONNECTION接口，这是使用WebSphere MQ管理工具创建的物理WebSphere MQ队列的名称。

  • 对于JMS_TOPIC_CONNECTION接口，它指定WebSphere MQ JMS主题的名称。必须使用WebSphere MQ管理工具创建该主题的订户使用的物理WebSphere MQ队列。默认情况下，使用的物理队列是SYSTEM.JMS.D.SUBSCRIBER.QUEUE 。

使用说明

除非另有说明，否则Messaging Gateway使用命名属性，如下所示：

  • 具有MGWPROP$_前缀的名称是保留的。它们用于特殊用途，在用作普通属性名称时无效。

  • 属性名称只能在属性列表中存在一次;也就是说，列表只能包含给定名称的一个值。该名称不区分大小写。

  • 通常，属性列表与顺序无关，属性名称可以按任何顺序出现。alter属性列表是一个例外。

  • 您可以使用新属性列表来更改现有属性列表。每个新属性都通过以下方式之一修改原始列表：添加新属性，修改属性，删除属性或删除所有属性。

从第一个元素到最后一个元素按顺序处理alter list。因此，元素在alter list中出现的顺序是有意义的，尤其是在使用alter list从现有列表中删除属性时。

属性名称和值用于确定该元素如何影响原始列表。以下规则适用：

  • 添加或修改属性

    MGW_PROPERTY.NAME = property_name MGW_PROPERTY.VALUE = property_value

    如果给定名称的属性已存在，则将当前值替换为新值;否则新属性将添加到列表的末尾。

  • 删除属性

    MGW_PROPERTY.NAME ='MGWPROP $ _REMOVE'MGW_PROPERTY.VALUE = name_of_property_to_remove

    如果原始列表中不存在属性名称，则不执行任何操作。

  • 删除所有属性

    MGW_PROPERTY.NAME ='MGWPROP $ _REMOVE_ALL'MGW_PROPERTY.VALUE =未使用

105.5.1 ADD_SUBSCRIBER过程

此过程添加一个订户，用于使用来自源队列的消息以传播到目标。

注意：

由于技术的改进，该子程序已被弃用（请参阅CREATE_JOB过程 ），并且仅出于向后兼容性的原因而保留。

句法

DBMS_MGWADM.ADD_SUBSCRIBER（subscriber_id IN VARCHAR2，propagation_type IN BINARY_INTEGER，queue_name IN VARCHAR2，destination IN VARCHAR2，rule IN VARCHAR2 DEFAULT NULL，transformation IN VARCHAR2 DEFAULT NULL，exception_queue IN VARCHAR2 DEFAULT NULL选项IN SYS.MGW_PROPERTIES DEFAULT NULL）;

参数

表105-20 ADD_SUBSCRIBER过程参数

参数 描述

subscriber_id

指定标识此订户的用户定义名称

propagation_type

指定消息传播的类型。DBMS_MGWADM.OUTBOUND_PROPAGATION用于Oracle数据库高级队列到非Oracle传播。DBMS_MGWADM.INBOUND_PROPAGATION用于非Oracle到Oracle数据库高级队列传播

queue_name

指定要向其添加此订户的源队列。此参数的语法和解释取决于为propagation_type指定的值。

目的地

指定传播此订户使用的消息的目标队列。此参数的语法和解释取决于为propagation_type指定的值。

规则

指定订户用于从源队列中出列消息的可选订阅规则。如果不需要规则，则为NULL 。此参数的语法和解释取决于为propagation_type指定的值。

转型

指定在Oracle Database Advanced Queuing有效内容和Messaging Gateway定义的ADT之间进行转换所需的转换。所需的转换类型取决于为propagation_type指定的值。

如果为NULL ，则Messaging Gateway必须支持Oracle数据库高级队列有效内容类型。

exception_queue

指定用于异常消息记录目的的队列。此队列必须与传播源位于同一邮件系统上。如果为NULL,则不使用异常队列，如果出现问题，传播将停止。此参数的语法和解释取决于为propagation_type.指定的值propagation_type.

源队列和异常队列不能是同一个队列。

选项

可选订户属性。如果没有则为NULL 。通常，这些是消息传递系统支持的较少使用的配置属性。

OUTBOUND_PROPAGATION订阅者

用于出站传播的订户的参数解释如下：

  • queue_name指定作为传播源的本地Oracle数据库高级队列队列。这必须具有schema.queue的语法。

  • destination指定传播消息的外部队列。这必须具有registered_queue@message_link的语法。

  • 如果使用本机Oracle数据库高级队列接口，则rule指定可选的Oracle Database Advanced Queuing订户规则;如果使用Oracle JMS接口，则指定JMS选择器。如果为NULL ，则不使用规则或选择器。

  • transformation指定用于将Oracle Database Advanced Queuing有效内容转换为Messaging Gateway定义的ADT的转换。

    消息传递网关传播使用转换将来自Oracle数据库高级队列队列的消息排队，以将Oracle数据库高级队列有效负载转换为Messaging Gateway定义的已知ADT。然后，该消息在基于Messaging Gateway ADT的外部消息传递系统中排队。

  • exception_queue指定发生异常时将消息移动到的本地Oracle数据库高级队列队列的名称。这必须具有schema.queue的语法。

如果使用本机Oracle数据库高级队列接口，则在调用此过程时，无论Messaging Gateway是否正在运行，都将将订户添加到Oracle Database Advanced Queuing队列。本地订户的格式为sys.aq$_agent('MGW_ subscriber_id ' ， NULL, NULL) 。

如果使用Oracle JMS接口，则Messaging Gateway代理将创建名为MGW_ subscriber_id的JMS持久subscriber_id 。如果在调用此过程时代理未运行，则下次代理启动时将创建持久订阅者。

异常队列有以下警告：

  • 用户负责创建要用作异常队列的Oracle数据库高级队列队列。

  • 源和异常队列的有效内容类型必须匹配。

  • 必须将异常队列创建为DBMS_AQADM.NORMAL_QUEUE的队列类型，而不是DBMS_AQADM.EXCEPTION_QUEUE 。排队限制阻止Messaging Gateway传播使用EXCEPTION_QUEUE类型的Oracle数据库高级队列队列作为Messaging Gateway异常队列。

INBOUND_PROPAGATION订阅者

用于入站传播的订户的参数解释如下：

  • queue_name指定作为传播源的外部队列。这必须具有registered_queue@message_link的语法。

  • destination指定传播消息的本地Oracle数据库高级队列队列。这必须具有schema.queue的语法。

  • rule指定对外部邮件系统有效的可选订户规则。如果不需要规则，则为NULL 。

  • transformation指定用于将Messaging Gateway定义的ADT转换为Oracle Database Advanced Queuing有效内容类型的转换。

    消息传递网关传播使来自外部消息传递系统的消息出列，并将消息正文转换为Messaging Gateway定义的已知ADT。当消息排入Oracle数据库高级队列队列时，转换用于将Messaging Gateway ADT转换为Oracle数据库高级队列有效内容类型。

  • exception_queue指定发生异常时将消息移动到的外部队列的名称。这必须具有registered_queue@message_link的语法。

是否需要订户取决于非Oracle消息传递系统的要求。如果需要持久订户，则它将由Messaging Gateway代理创建。如果在调用此过程时代理未运行，则在下一次启动代理时，将在非Oracle消息传递系统上创建订户。

异常队列有以下警告：

  • 异常队列必须是已注册的非Oracle队列。

  • 源和异常队列必须使用相同的消息传递系统链接。

当Messaging Gateway代理崩溃或发生某些其他异常事件时， CLEAN_STARTUP_STATE操作CLEAN_STARTUP_STATE Messaging Gateway恢复到已知状态，并且无法重新启动Messaging Gateway。只有在Messaging Gateway代理程序已启动但似乎已崩溃或长时间没有响应时，才应执行此操作。

当MGW_GATEWAY视图显示AGENT_STATUS值不是NOT_STARTED或START_SCHEDULED,可能需要CLEAN_STARTUP_STATE操作START_SCHEDULED,并且AGENT_PING值在延长的时间段内是UNREACHABLE的。

如果AGENT_STATUS值为BROKEN ，则在解决问题并使用CLEAN_STARTUP_STATE操作重置代理状态之前，无法启动Messaging Gateway代理。BROKEN状态可以指示Messaging Gateway启动作业检测到Messaging Gateway代理已在运行。这种情况在正常使用情况下不应发生。

清理任务包括：

  • 删除用于启动外部Messaging Gateway代理进程的Scheduler作业。

  • 将某些配置信息设置为已知状态。例如，将代理状态设置为NOT_STARTED.

如果执行此命令失败：

  • 代理状态为NOT_STARTED或START_SCHEDULED.

  • 在调用此过程之前未进行任何关闭尝试，除非代理状态为STARTING.

  • 已成功联系Messaging Gateway代理。

    假设代理处于活动状态，此过程失败。如果代理在多次尝试后没有响应，则执行清理任务。此过程至少需要几秒钟，最多可能需要一分钟。在特定清理操作适当且必要的条件下，这是预期的行为。

注意：

在成功执行CLEAN_STARTUP_STATE操作后，终止可能仍在运行的任何Messaging Gateway代理进程。这应该在调用DBMS_MGWADM之前DBMS_MGWADM 。 STARTUP以启动Messaging Gateway。该过程通常命名为extprocmgwextproc 。

CLEAN_LOG_QUEUES

sarg未使用且必须为NULL 。

Messaging Gateway代理将清除所有已配置的邮件系统链接的日志队列。代理将暂时停止所有传播活动，然后从所有链接的日志队列中删除所有过时和错误的日志记录。如果Messaging Gateway代理未运行，则该过程将失败。

每次启动Messaging Gateway代理时，都会自动执行此清理操作。

注意：

CLEAN_LOG_QUEUES操作仅在代理启动时执行。如果在代理程序运行时调用此过程，则Messaging Gateway代理会忽略它。

RESET_SUB_MISSING_LOG_REC

sarg指定要重置的Messaging Gateway作业名称（或订户ID）。它不能为NULL 。

Messaging Gateway代理恢复由于缺少日志记录而失败的Messaging Gateway传播作业。代理将重置源和目标日志记录。如果Messaging Gateway代理未运行，则该过程将失败。

注意：

如果源队列中的消息已传播到目标队列，则此操作可能会导致重复的消息。

RESET_SUB_MISSING_MESSAGE

sarg指定要重置的Messaging Gateway作业名称（或订户ID）。它不能为NULL 。

Messaging Gateway代理恢复由于缺少持久性源消息而失败的Messaging Gateway传播作业。代理会将消息视为非持久消息，并继续处理该传播作业。如果Messaging Gateway代理未运行，则该过程将失败。
