Oracle Database 19.1中的更改

以下是Oracle Spatial和Graph GeoRaster开发人员指南（适用于Oracle数据库19.1）中的更改。

  • 必须在架构级别启用GeoRaster
  • SDO_GEOR_RA支持BLOB中的结果数据

父主题： Oracle Spatial和Graph GeoRaster开发人员指南的此发行版中的更改

必须在架构级别启用GeoRaster

必须为将使用GeoRaster的每个模式启用Oracle Spatial and Graph的GeoRaster功能。在以前的版本中，您为整个数据库启用了GeoRaster（通过执行名为MDSYS.enableGeoRaster的过程）。

有关信息和说明，请参阅在架构级别启用GeoRaster 。另请参见以下新的PL / SQL相关子程序：

  • SDO_GEOR_ADMIN.enableGeoRaster （为当前架构启用GeoRaster）

  • SDO_GEOR_ADMIN.isGeoRasterEnabled （检查是否为当前架构启用了GeoRaster）

父主题： Oracle数据库19.1中的更改

SDO_GEOR_RA支持BLOB中的结果数据

SDP_GEOR_RA包中的所有子程序都支持将结果数据放入BLOB以支持动态栅格代数。这些子程序包括一个或多个格式，其中包含名为rasterBlob的参数。

有关信息，请参阅SDO_GEOR_RA包参考章节中的各个子程序主题。

Oracle数据库18.1中的更改

以下是Oracle Spatial和Graph GeoRaster开发人员 Oracle数据库18.1 指南中的更改。

  • 新的SDO_GEOR_GDAL包
  • GeoRaster PL / SQL API更改
  • GeoRaster Java API更改
  • 外部JPEG 2000图像文件已删除4GB限制
  • 支持混合大小写用户和模式名称

父主题： Oracle Spatial和Graph GeoRaster开发人员指南的此发行版中的更改

新的SDO_GEOR_GDAL包

新的SDO_GEOR_GDAL PL / SQL包将部分GDAL集成到Oracle数据库服务器中。它提供服务器端栅格数据加载，导出和数据库内地形分析和可视化功能。它还通过GDAL API实现并简化了C / C ++插件的开发。

有关此程序包的说明以及有关其子程序的参考信息，请参阅SDO_GEOR_GDAL程序包参考 。

父主题： Oracle数据库18.1中的更改

GeoRaster PL / SQL API更改

以下更改与GeoRaster PL / SQL子程序有关。

  • 几个SDO_GEOR_IP子程序具有BLOB作为输出的新格式，以支持即时图像处理和可视化： SDO_GEOR_IP.equalize ， SDO_GEOR_IP.filter ， SDO_GEOR_IP.normalize ， SDO_GEOR_IP.piecewiseStretch ， SDO_GEOR_IP.stretch 。

  • 许多SDO_GEOR_RA子程序具有新格式，以支持将结果数据放入BLOB而不是GeoRaster对象。这些新的BLOB相关格式包括参数rasterBlob ， outArea和outWindow 。所有这些子程序的参考信息都在SDO_GEOR_RA包参考中 。

  • SDO_GEOR_IP.dodge的新格式将图像闪烁到参考图像。

  • 新的SDO_GEOR_AGGR.getMosaicStatistics过程为虚拟马赛克生成统计数据和直方图。

  • SDO_GEOR.getRasterSubset的新流水线表函数返回单元格值表，以便用户可以更轻松地利用SQL分析。

父主题： Oracle数据库18.1中的更改

GeoRaster Java API更改

以下更改与GeoRaster Java API有关。

  • 除了SDO_GEOR_GDAL包之外，为新的PL / SQL函数和过程添加了Java API。

  • 新的全局图像处理功能被添加到Java API中，包括自动线性拉伸，手动线性拉伸，分段拉伸，标准化和均衡。这些功能允许基于GeoRaster对象统计的图像处理，以实现对大图像和虚拟马赛克的一致可视化增强。

    GeoRaster Viewer已得到增强，可在Java API中使用新的全局图像处理集。

有关GeoRaster Java API的参考信息包含在Oracle Spatial and Graph Java API Reference中 。

父主题： Oracle数据库18.1中的更改

外部JPEG 2000图像文件已删除4GB限制

外部JPEG 2000图像文件的大小限制为4GB，因此可以将这些大型JPEG 2000图像直接加载到数据库中而无需解压缩。

父主题： Oracle数据库18.1中的更改

支持混合大小写用户和模式名称

支持混合大小写的用户名和模式名称以与GeoRaster一起使用。
