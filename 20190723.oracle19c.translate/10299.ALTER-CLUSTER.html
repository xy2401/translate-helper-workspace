ALTER CLUSTER

目的

使用ALTER CLUSTER语句重新定义集群的存储和并行特征。

注意：

您不能使用此语句更改群集键中列的数量或名称，也不能更改存储群集的表空间。

也可以看看：

有关创建集群的信息，请参阅CREATE CLUSTER ;有关从集群中删除表的信息，请参阅DROP CLUSTER和DROP TABLE ;有关将表添加到集群的信息，请参阅CREATE TABLE ... physical_properties

先决条件

群集必须位于您自己的架构中，或者您必须具有ALTER ANY CLUSTER系统特权。

语义

模式

指定包含群集的架构。如果省略schema ，则Oracle数据库假定集群在您自己的模式中。

簇

指定要更改的集群的名称。

physical_attributes_clause

使用此子句可更改群集的PCTUSED ， PCTFREE和INITRANS参数的值。

使用STORAGE子句更改群集的存储特征。

也可以看看：

  • physical_attributes_clause有关参数的信息

  • storage_clause获取该子句的完整描述

对物理属性的限制

您无法更改群集的存储参数INITIAL和MINEXTENTS的值。

尺寸

整数

使用SIZE子句指定将存储在分配给群集的数据块中的群集密钥数。

对SIZE的限制

您只能为索引的群集更改SIZE参数，而不能为哈希群集更改。

也可以看看：

创建CLUSTER以获取SIZE参数和“ 修改集群：示例 ”的说明

修改分区

指定MODIFY PARTITION partition allocate_extent_clause以显式为群集分区分配新范围。此操作仅对范围分区的散列簇有效。对于partition ，请指定群集分区名称。

allocate_extent_clause

指定allocate_extent_clause以明确为群集分配新范围。此操作仅对索引集群和非分区散列集群有效。

当你明确分配的程度与allocate_extent_clause ，Oracle数据库不计算集群的存储参数，并确定要分配的下一个扩展新的大小（当你创建一个表作为它）。因此，如果您不希望Oracle数据库使用默认值，请指定SIZE 。

也可以看看：

allocate_extent_clause获取此子句的完整描述

deallocate_unused_clause

使用deallocate_unused_clause显式释放群集末尾的未使用空间，并使释放的空间可用于其他段。

也可以看看：

deallocate_unused_clause获取此子句的完整描述和“ 释放未使用的空间：示例 ”

parallel_clause

指定parallel_clause以更改群集上查询的默认并行度。

也可以看看：

有关此子句的完整信息，请参阅CREATE TABLE文档中的parallel_clause

例子

以下示例修改了在CREATE CLUSTER “ Examples ”中创建的集群。

修改群集：示例

下一个声明改变personnel集群：

ALTER CLUSTER人员SIZE 1024 CACHE;

Oracle数据库为每个群集键值分配1024个字节，并启用缓存属性。假设数据块大小为2千字节，则此群集中的未来数据块在每个数据块中包含2个群集密钥，或2千字节除以1024个字节。

解除分配未使用空间：示例

以下语句从language群集中释放未使用的空间，保留30千字节的未使用空间以供将来使用：

ALTER CLUSTER语言DEALLOCATE UNUSED KEEP 30 K;

改变集群：示例

以下语句创建一个具有默认密钥大小（600）的集群：

创建集群EMP_DEPT（DEPTNO NUMBER（3））SIZE 600 TABLESPACE用户存储（初始200K，下一个300K MINEXTENTS 2 PCTINCREASE 33）;

以下语句查询USER_CLUSTERS以显示集群元数据：

SELECT CLUSTER_NAME，TABLESPACE_NAME，KEY_SIZE，CLUSTER_TYPE，AVG_BLOCKS_PER_KEY，MIN_EXTENTS，USER_CLUSTERS的MAX_EXTENTS; CLUSTER_NAME TABLESPACE_NAME KEY_SIZE CLUST AVG_BLOCKS_PER_KEY MIN_EXTENTS MAX_EXTENTS --------------- ---------------------------- -  ---------- ----- ------------------ ----------- ---- ------- EMP_DEPT USERS 600 INDEX 1 2147483645

以下语句修改群集密钥大小：

ALTER CLUSTER EMP_DEPT SIZE 1024;

以下语句显示已修改集群的元数据：

SELECT CLUSTER_NAME，TABLESPACE_NAME，KEY_SIZE，CLUSTER_TYPE，AVG_BLOCKS_PER_KEY，MIN_EXTENTS，USER_CLUSTERS的MAX_EXTENTS; CLUSTER_NAME TABLESPACE_NAME KEY_SIZE CLUST AVG_BLOCKS_PER_KEY MIN_EXTENTS MAX_EXTENTS --------------- ---------------------------- -  ---------- ----- ------------------ ----------- ---- ------- EMP_DEPT USERS 1024 INDEX 1 2147483645

以下语句从EMP_DEPT集群中释放未使用的空间，保留30千字节的未使用空间以供将来使用：

ALTER CLUSTER EMP_DEPT DEALLOCATE UNUSED KEEP 30 K;
