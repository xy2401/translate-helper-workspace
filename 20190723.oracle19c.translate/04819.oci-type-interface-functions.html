30.2.1 OCITypeAddAttr（）

将属性添加到先前使用typecode OCI_TYPECODE_OBJECT构造的对象类型。

目的

将属性添加到先前使用typecode OCI_TYPECODE_OBJECT构造的对象类型。

句法

sword OCITypeAddAttr（OCISvcCtx * svchp，OCIError * errhp，OCIType * type，const text * a_name，ub4 a_length，OCIParam * attr_info）;

参数

svchp（IN）

OCI服务上下文。

属性的名称。

属性名称的长度，以字节为单位。

attr_info（IN）

有关属性的信息。它是通过使用OCIAttrSet()调用在OCIParam分配OCIParam参数句柄和设置类型信息而获得的。

30.2.2 OCITypeBeginCreate（）

开始了瞬态类型的构建过程。类型是匿名的（没有名称）。

目的

开始了瞬态类型的构建过程。类型是匿名的（没有名称）。

句法

sword OCITypeBeginCreate（OCISvcCtx * svchp，OCIError * errhp，OCITypeCode tc，OCIDuration dur，OCIType ** type）;

参数

svchp（IN）

OCI服务上下文。

errhp（IN / OUT）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

tc（IN）

类型的类型代码。类型代码可以对应于对象类型或内置类型。

目前，用户定义类型的允许值为：

  • 对象类型的OCI_TYPECODE_OBJECT （结构化）

  • OCI_TYPECODE_VARRAY表示VARRAY集合类型

  • 嵌套表集合类型的OCI_TYPECODE_TABLE

对于对象类型，请调用OCITypeAddAttr()以添加每个属性类型。对于集合类型，请调用OCITypeSetCollection() 。随后，调用OCITypeEndCreate()来完成创建过程。

内置类型代码的允许值在“ Typecodes ”中指定。有关内置类型（精度，数字比例， VARCHAR2的字符集信息等）的附加信息（如果有）必须在随后调用OCITypeSetBuiltin() 。最后，您必须使用OCITypeEndCreate()来完成创建过程。

dur（IN）

类型的分配持续时间。它是以下之一：

  • 之前创建的用户持续时间。它可以使用OCIDurationBegin()创建。

  • 预定义的持续时间，例如OCI_DURATION_SESSION 。

类型（OUT）

正在构造的OCIType （类型描述符）。

评论

要创建持久性命名类型，请使用SQL语句CREATE TYPE 。瞬态类型没有身份。它们是纯粹的价值观。

30.2.5 OCITypeSetCollection（）

设置集合类型信息。

目的

只有在使用集合类型代码构造类型时才能进行此调用。

句法

sword OCITypeSetCollection（OCISvcCtx * svchp，OCIError * errhp，OCIType * type，OCIParam * collelem_info，ub4 coll_count）;

参数

svchp（IN）

OCI服务上下文。

errhp（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

类型（IN / OUT）

正在构造的类型描述符。

collelem_info（IN）

collelem_info提供有关集合元素的信息。它是通过使用OCIAttrSet()调用在OCIParam分配OCIParam参数句柄和设置类型信息而获得的。

coll_count（IN）

集合中元素的数量。传递0表示嵌套表（无界限）。
