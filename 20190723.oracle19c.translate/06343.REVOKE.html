3.10 REVOKE

目的

使用REVOKE命令撤消先前使用GRANT命令授予的恢复目录特权。

先决条件

仅在RMAN提示符下执行此命令。

使用说明

假设虚拟专用目录用户被授予REGISTER DATABASE特权，该特权隐式授予任何已注册数据库的CATALOG FOR DATABASE特权。该用户注册多个数据库。如果您从此用户REVOKE REGISTER DATABASE权限，则此用户将保留已注册数据库的CATALOG FOR DATABASE权限。CATALOG权限包括注册和取消注册指定的数据库。

要阻止此用户访问任何数据库的元数据或注册其他数据库，请为该用户执行REVOKE ALL PRIVILEGES 。要撤消此用户注册的数据库子集的CATALOG权限，请REVOKE CATALOG FOR DATABASE集中的每个数据库执行REVOKE CATALOG FOR DATABASE 。

句法

语义

语法元素 描述
CATALOG FOR DATABASE { databasename | integer }

从指定用户撤消指定数据库的恢复目录访问权限。

您可以通过数据库名称或DBID指定数据库。如果在恢复目录中注册具有此名称的多个数据库时指定数据库名称，则RMAN将返回错误。在这种情况下，请按DBID指定数据库。
CATALOG FOR PLUGGABLE DATABASE { database_name | GUID guid_name }

从指定用户撤消指定可插拔数据库（PDB）的恢复目录访问权限。

您可以通过PDB的名称或GUID指定PDB。如果在恢复目录中注册了具有此名称的多个PDB时指定了PDB名称，则RMAN将返回错误。在这种情况下，使用GUID指定PDB。

注册数据库

撤消指定用户在此恢复目录中注册新数据库的能力（请参阅示例3-47 ）。

所有特权

撤消指定用户的所有CATALOG和REGISTER权限。

FROM userid

指定要从中撤消权限的用户的名称。

例子

示例3-47从虚拟专用目录用户撤消权限

假设您将RMAN连接到基本恢复目录作为恢复目录所有者rco 。作为基本目录所有者，您可以按如下方式使用RMAN GRANT命令为bckop2提供在其虚拟专用目录中注册任何数据库的功能，但只允许bckop3访问数据中心中的一部分数据库：

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码： 密码连接到恢复目录数据库RMAN> GRANT REGISTER DATABASE to bckop2; RMAN>数据库的GRANT CATALOG prod to bckop3; RMAN>数据库的GRANT CATALOG prodb to bckop3; RMAN>退出;

稍后，您希望限制用户BCKOP2的权限，以便此用户无法注册新数据库，因此您将以rco连接到基本目录并执行REVOKE命令。BCKOP2保留此用户注册的数据库的目录特权。 

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码： 密码连接到恢复目录数据库RMAN> REVOKE REGISTER DATABASE FROM bckop2;
