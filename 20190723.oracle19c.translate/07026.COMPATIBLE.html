1.48兼容

COMPATIBLE使您可以使用新版本的Oracle，同时确保将数据库降级到早期版本的能力。

属性 描述

参数类型

串

句法

COMPATIBLE = release_number

默认值

18.0.0

可修改

没有

可在PDB中修改

没有

价值范围

11.2.0到默认版本

基本

是

Oracle RAC

多个实例必须具有相同的值。

设置COMPATIBLE可确保新功能不会将数据格式或结构写入与早期版本不兼容的磁盘，从而防止将来降级。需要更高的COMPATIBLE值才能正常工作的功能可能会受到限制或禁用，以确保可以降级。

COMPATIBLE参数指定数据库磁盘格式必须兼容的Oracle版本号。可以将数据库降级到COMPATIBLE参数或任何更高版本中指定的版本。

注意：

COMPATIBLE参数的值可以增加到更高版本，但永远不能降低到更低版本。

注意：

使用备用数据库时，此参数在主数据库和备用数据库上必须具有相同的值。

通常，用户在升级其Oracle软件时保持COMPATIBLE不变。升级后，用户将运行新版本的Oracle软件几周，以确保新版本正常运行。之后，用户可以选择将COMPATIBLE更新到最新版本以利用新功能。

将COMPATIBLE参数的值设置为12.2.0.0.0 ，SQL标识符的最大大小将从30个字节增加到128个字节。

注意：

COMPATIBLE参数必须指定为至少三个十进制数，每对由一个点分隔，例如12.0.0 。
