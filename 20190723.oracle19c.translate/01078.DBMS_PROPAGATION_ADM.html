128 DBMS_PROPAGATION_ADM

DBMS_PROPAGATION_ADM程序包是一组Oracle Replication程序包之一，它提供了用于配置从源队列到目标队列的传播的管理接口。

本章包含以下主题：

  • 概观

  • 安全模型

  • DBMS_PROPAGATION_ADM子程序摘要

128.1 DBMS_PROPAGATION_ADM概述

该软件包提供了启动，停止和配置传播的接口。

128.2 DBMS_PROPAGATION_ADM安全模型

可以通过将此程序包上的EXECUTE授予选定的用户或角色，或者通过向所选用户或角色授予EXECUTE_CATALOG_ROLE来控制此程序包的安全性。

如果程序包中的子程序是从存储过程中运行的，则必须直接为程序包授予运行子程序的用户EXECUTE特权。它不能通过角色授予。

使用DBMS_PROPAGATION_ADM程序包管理Oracle Replication配置时，需要为用户授予Oracle Replication管理员的权限。

128.3.1 ALTER_PROPAGATION过程

此过程添加，更改或删除传播的规则集。

句法

DBMS_PROPAGATION_ADM.ALTER_PROPAGATION（propagation_name IN VARCHAR2，rule_set_name IN VARCHAR2 DEFAULT NULL，remove_rule_set IN BOOLEAN DEFAULT FALSE，negative_rule_set_name IN VARCHAR2 DEFAULT NULL，remove_negative_rule_set IN BOOLEAN DEFAULT FALSE）;

参数

表128-2 ALTER_PROPAGATION过程参数

参数 描述

propagation_name

您正在改变的传播的名称。您必须指定现有的传播名称。不要指定所有者。

rule_set_name

传播的正规则集的名称。正规则集包含指示传播传播消息的规则。

如果要为传播使用正规则集，则必须以[ schema_name .] rule_set_name形式指定现有规则集。例如，要在名为prop_rules的hr模式中指定正规则集，请输入hr.prop_rules 。如果未指定架构，则当前用户是默认用户。

如果指定的规则集不存在，则返回错误。您可以使用DBMS_RULE_ADM包创建规则集并向其添加规则。

如果指定NULL并且remove_rule_set参数设置为FALSE ，则该过程将保留任何现有的正规则集。如果指定NULL并且remove_rule_set参数设置为TRUE ，则该过程将删除任何现有的正规则集。

remove_rule_set

如果为TRUE ，则该过程将删除指定传播的正规则集。如果删除传播的正规则集，并且传播没有负规则集，则传播将传播所有消息。

如果为传播删除正规则集，并且传播存在否定规则集，则传播将传播其队列中未被否定规则集丢弃的所有消息。

如果为FALSE ，则该过程保留指定传播的正规则集。

如果rule_set_name参数为非NULL ，则应将此参数设置为FALSE 。

negative_rule_set_name

为传播设置的否定规则的名称。否定规则集包含指示传播丢弃消息的规则。

如果要对传播使用否定规则集，则必须以[ schema_name .] rule_set_name形式指定现有规则集。例如，要在名为neg_rules的hr模式中指定负规则集，请输入hr.neg_rules 。如果未指定架构，则当前用户是默认用户。

如果指定的规则集不存在，则返回错误。您可以使用DBMS_RULE_ADM包创建规则集并向其添加规则。

如果指定NULL并且remove_negative_rule_set参数设置为FALSE ，则该过程将保留任何现有的否定规则集。如果指定NULL并且remove_negative_rule_set参数设置为TRUE ，则该过程将删除任何现有的否定规则集。

如果为传播指定正规则集和负规则集，则始终首先评估否定规则集。

remove_negative_rule_set

如果为TRUE ，则该过程将删除指定传播的否定规则集。如果删除传播的否定规则集，并且传播没有正规则集，则传播将传播所有消息。

如果删除传播的否定规则集，并且传播存在正规则集，则传播将传播其队列中未被正规则集丢弃的所有消息。

如果为FALSE ，则该过程保留指定传播的否定规则集。

如果negative_rule_set_name参数为非NULL ，则此参数应设置为FALSE 。

128.3.2 CREATE_PROPAGATION过程

此过程创建传播并指定源队列，目标队列和传播的任何规则集。传播将本地源队列中的消息传播到目标队列。目标队列可能与源队列位于同一数据库中，也可能不在同一数据库中。

句法

DBMS_PROPAGATION_ADM.CREATE_PROPAGATION（propagation_name IN VARCHAR2，source_queue IN VARCHAR2，destination_queue IN VARCHAR2，destination_dblink IN VARCHAR2 DEFAULT NULL，rule_set_name IN VARCHAR2 DEFAULT NULL，negative_rule_set_name IN VARCHAR2 DEFAULT NULL，queue_to_queue IN BOOLEAN DEFAULT NULL，original_propagation_name IN VARCHAR2 DEFAULT NULL，auto_merge_threshold IN NUMBER DEFAULT空值）;

参数

表128-3 CREATE_PROPAGATION过程参数

参数 描述

propagation_name

您正在创建的传播的名称。不允许NULL设置。不要指定所有者。

注意：创建传播后，不能更改propagation_name设置。

source_queue

源队列的名称，指定为[ schema_name .] queue_name 。当前数据库必须包含源队列。

例如，要在strmadmin架构中指定名为streams_queue的源队列，请为此参数输入strmadmin.streams_queue 。如果未指定架构，则当前用户是默认用户。

destination_queue

目标队列的名称，指定为[ schema_name .] queue_name 。

例如，要在strmadmin架构中指定名为streams_queue的目标队列，请为此参数输入strmadmin.streams_queue 。如果未指定架构，则当前用户是默认用户。

destination_dblink

传播将使用的数据库链接的名称。数据库链接是从包含源队列的数据库到包含目标队列的数据库。

如果为NULL ，则源队列和目标队列必须位于同一数据库中。

注意：不允许使用连接限定符。

rule_set_name

传播的正规则集的名称。正规则集包含指示传播传播消息的规则。

如果要为传播使用正规则集，则必须以[ schema_name .] rule_set_name形式指定现有规则集。例如，要在名为prop_rules的hr模式中指定正规则集，请输入hr.prop_rules 。如果未指定架构，则当前用户是默认用户。

如果指定的规则集不存在，则返回错误。您可以使用DBMS_RULE_ADM包创建规则集并向其添加规则。

如果指定NULL ，并且传播不存在负规则集，则传播将传播其队列中的所有消息。

如果指定NULL ，并且传播存在负规则集，则传播将传播其队列中未被否定规则集丢弃的所有消息。

negative_rule_set_name

为传播设置的否定规则的名称。否定规则集包含指示传播丢弃消息的规则。

如果要对传播使用否定规则集，则必须以[ schema_name .] rule_set_name形式指定现有规则集。例如，要在名为neg_rules的hr模式中指定负规则集，请输入hr.neg_rules 。如果未指定架构，则当前用户是默认用户。

如果指定的规则集不存在，则返回错误。您可以使用DBMS_RULE_ADM包创建规则集并向其添加规则。

如果指定NULL ，并且传播不存在正规则集，则传播将传播其队列中的所有消息。

如果指定NULL ，并且传播存在正规则集，则传播将传播其队列中未被正规则集丢弃的所有消息。

如果为传播指定正规则集和负规则集，则始终首先评估否定规则集。

queue_to_queue

如果为TRUE或NULL ，则传播是队列传播的队列。队列到队列传播始终具有自己的传播作业，并且当目标队列是Oracle Real Application Clusters（Oracle RAC）数据库中的缓冲队列时，使用服务进行自动故障转移。

如果为FALSE ，则传播是队列到dblink的传播。队列到数据链路的传播可以与使用相同数据库链接的其他传播共享传播作业，并且不支持Oracle RAC环境中的自动故障转移。

original_propagation_name

如果正在创建的传播是拆分和合并操作的一部分，请指定原始传播名称。拆分操作以新名称克隆原始传播。将克隆的传播复制回原始流时，原始传播的名称很重要。

如果正在创建的传播不是拆分和合并操作的一部分，请指定NULL 。

auto_merge_theshold

如果满足以下两个条件，请指定正数：

  • 正在创建的传播是拆分和合并操作的一部分。

  • 流将自动合并回原始流。

如果满足以下任一条件，请指定NULL ：

  • 正在创建的传播不是拆分和合并操作的一部分。

  • 正在创建的传播是拆分和合并操作的一部分，但拆分的流不会自动合并回原始流。

使用说明

此过程开始传播，可能会创建传播作业。如果此过程创建传播作业，则它会为传播作业建立默认计划。每个传播作业都是Oracle Scheduler作业。您可以使用Oracle Scheduler调整传播作业的计划。

拥有源队列的用户是传播消息的用户。此用户必须具有传播邮件所需的权限。

128.3.3 DROP_PROPAGATION过程

此过程将删除传播并删除源队列中目标队列的所有消息。此过程还会删除从源队列到目标队列的传播计划。

句法

DBMS_PROPAGATION_ADM.DROP_PROPAGATION（propagation_name IN VARCHAR2，drop_unused_rule_sets IN BOOLEAN DEFAULT FALSE）;

参数

表128-4 DROP_PROPAGATION过程参数

参数 描述

propagation_name

要丢弃的传播的名称。您必须指定现有的传播名称。不要指定所有者。

drop_unused_rule_sets

如果为TRUE ，则如果任何其他Oracle Replication客户端（包括捕获进程，传播，应用进程和消息传递客户端）未使用这些规则集，则该过程将删除指定传播使用的任何规则集（正数和负数）。如果此过程删除了规则集，则此过程还会删除规则集中不在另一个规则集中的任何规则。

如果为FALSE ，则该过程不会删除指定传播使用的规则集，并且规则集会保留其规则。

使用说明

使用此过程删除传播时，将从Oracle Replication规则的数据字典视图中删除有关为传播创建的规则的信息。即使规则不在传播的任一规则集中，也会删除有关此类规则的信息。

以下是Oracle Replication规则的数据字典视图：

  • ALL_STREAMS_GLOBAL_RULES

  • DBA_STREAMS_GLOBAL_RULES

  • ALL_STREAMS_SCHEMA_RULES

  • DBA_STREAMS_SCHEMA_RULES

  • ALL_STREAMS_TABLE_RULES

  • DBA_STREAMS_TABLE_RULES

    注意：

    删除传播时，如果没有其他传播正在使用传播作业，则传播使用的传播作业将自动删除。

128.3.4 START_PROPAGATION过程

此过程开始传播。

句法

DBMS_PROPAGATION_ADM.START_PROPAGATION（propagation_name IN VARCHAR2）;

参数

表128-5 START_PROPAGATION过程参数

参数 描述

propagation_name

您正在开始的传播的名称。您必须指定现有的传播名称。不要指定所有者。

使用说明

传播状态持续记录。因此，如果状态为ENABLED ，则在数据库实例启动时启动传播。

128.3.5 STOP_PROPAGATION过程

此过程会停止传播。

句法

DBMS_PROPAGATION_ADM.STOP_PROPAGATION（propagation_name IN VARCHAR2，force IN BOOLEAN DEFAULT FALSE）;

参数

表128-6 STOP_PROPAGATION过程参数

参数 描述

propagation_name

您要停止的传播的名称。您必须指定现有的传播名称。不要指定所有者。

力

如果为TRUE ，则该过程将停止传播并清除传播的统计信息。

如果为FALSE ，则该过程将停止传播而不清除传播的统计信息。

使用说明

传播状态持续记录。因此，如果状态为DISABLED或ABORTED ，则在数据库实例启动时不会启动传播。
