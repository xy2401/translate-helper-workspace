C数据字典

数据字典信息作为系统表存储在非Oracle系统中，并通过ODBC应用程序编程接口（API）访问。

以下主题说明如何访问非Oracle数据字典，使用支持的视图和表以及数据字典映射：

访问非Oracle数据字典

访问非Oracle数据字典表或视图与访问Oracle数据库中的数据字典完全相同。您发出指定数据库链接的SELECT语句。Oracle数据字典视图和列名称用于访问非Oracle数据字典。支持的视图的同义词也是可以接受的。

例如，以下语句查询数据字典表ALL_USERS以检索非Oracle系统中的所有用户：

SQL SELECT * FROM all_users @ sid1;

发出数据字典访问查询时，ODBC代理：

  1. 将请求的表，视图或同义词映射到一个或多个ODBC API（请参阅数据字典映射 ）。代理将所有数据字典列名称转换为查询中对应的非Oracle列名称。
  2. 将API序列发送到非Oracle系统。
  3. 可能转换检索到的非Oracle数据，使其具有Oracle数据字典表的外观。
  4. 将数据字典信息从非Oracle系统表传递给Oracle。

    注意：

    查询Oracle Database Gateway for ODBC数据字典时返回的值可能与Oracle SQL * Plus DESCRIBE命令返回的值不同。

Oracle Database Gateway for ODBC支持的视图和表

Oracle Database Gateway for ODBC仅支持表，Oracle Database Gateway for ODBC数据字典映射中显示的视图和表。

如果使用不受支持的视图，则会收到Oracle错误消息，指出未选择任何行。

如果要使用SELECT...查询数据字典视图SELECT...FROM DBA_* ，首先以Oracle用户SYSTEM或SYS 。否则，您收到以下错误消息：

ORA-28506：存储在％s中的％s的数据字典转换中的解析错误

使用Oracle Database Gateway for ODBC，当您没有这些非Oracle对象的访问权限时，对支持的数据字典表和以字符ALL_开头的视图的查询可能会从非Oracle系统返回行。使用Oracle数据字典查询Oracle数据库时，仅为允许访问的对象返回行。
