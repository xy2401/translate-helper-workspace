7 Oracle数据库和Oracle Grid Infrastructure支持的存储选项

在安装规划过程中查看支持的存储选项。

  • Oracle Grid Infrastructure支持的存储选项
    下表显示了Oracle Grid Infrastructure二进制文件和文件支持的存储选项：
  • Oracle ACFS和Oracle ADVM
    Oracle自动存储管理群集文件系统（Oracle ACFS）扩展了Oracle ASM技术，以支持单实例和群集配置中的所有应用程序数据。
  • Oracle Grid Infrastructure和Oracle RAC的存储注意事项
    对于所有安装，您必须选择用于Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）和Oracle Real Application Clusters（Oracle RAC）数据库的存储选项。
  • 使用Oracle ASM磁盘组进行存储的准则
    规划您希望如何配置Oracle ASM磁盘组以进行部署。
  • 使用Oracle ASM的网络文件系统的准则
    在Oracle Grid Infrastructure安装期间，您可以选择在网络文件系统上配置Oracle ASM。
  • 将逻辑卷管理器与Oracle Grid Infrastructure和Oracle RAC配合使用
    Oracle Grid Infrastructure和Oracle RAC仅支持群集感知卷管理器。
  • 关于数据文件的NFS存储
    有关NFS存储配置指南，请查看本节。
  • 关于直接NFS客户端安装到NFS存储设备
    Direct NFS Client直接在Oracle软件中集成NFS客户端功能，以优化Oracle和NFS服务器之间的I / O路径。这种集成可以显着提高性能。
