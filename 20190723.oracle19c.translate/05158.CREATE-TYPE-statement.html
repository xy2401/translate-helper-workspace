14.13 CREATE TYPE语句

CREATE TYPE语句指定CREATE TYPE的名称及其属性，方法和其他属性。

CREATE TYPE语句创建或替换其中一个的规范：

  • 抽象数据类型（ADT）

  • 独立变量数组（varray）类型

  • 独立嵌套表类型

  • 不完整的对象类型

    不完整类型是由转发类型定义创建的类型。它被称为不完整，因为它有一个名称，但没有属性或方法。它可以被其他类型引用，允许您定义相互引用的类型。但是，必须先完全指定类型，然后才能使用它来创建表或对象列或嵌套表类型的列。

CREATE TYPE BODY语句包含实现该类型的方法的代码。

注意：

  • 如果创建的类型的规范仅声明属性但没有声明方法，则无需指定类型体。

  • 使用CREATE TYPE语句创建的独立集合类型与使用PL / SQL块或包中的关键字TYPE定义的集合类型不同。有关后者的信息，请参阅“ 收集变量声明 ” 。

  • 使用CREATE TYPE语句，您可以创建嵌套表和VARRAY类型，但不能创建关联数组。在PL / SQL块或包中，您可以定义所有三种集合类型。

先决条件

要在架构中创建类型，您必须具有CREATE TYPE系统特权。要在另一个用户的模式中创建类型，您必须具有CREATE ANY TYPE系统特权。您可以显式获取这些权限，也可以通过角色授予它们。

要创建子类型，您必须具有UNDER ANY TYPE系统特权或超类型的UNDER对象特权。

必须显式授予该类型的所有者EXECUTE对象特权才能访问该类型定义中引用的所有其他类型，或者必须为类型所有者授予EXECUTE ANY TYPE系统特权。所有者无法通过角色获得这些权限。

如果类型所有者打算授予其他用户访问该类型的权限，则必须使用GRANT OPTION或具有ADMIN OPTION的EXECUTE ANY TYPE系统特权为所有者授予所有者EXECUTE对象特权。否则，类型所有者没有足够的权限将该类型的访问权限授予其他用户。

语义

create_type

或替换

如果存在，则重新创建类型，然后重新编译它。

在重新定义之前被授予该类型特权的用户仍然可以访问该类型而无需重新授予特权。

如果任何基于函数的索引依赖于类型，则数据库会将索引标记为DISABLED 。

[版权所有|不可挽回的]

指定类型是否为editioned或noneditioned对象，如果editioning为架构对象类型启用TYPE的schema 。默认值： EDITIONABLE 。有关已编辑和未编辑的对象的信息，请参见“ Oracle数据库开发指南” 。

plsql_type_source

模式

包含该类型的架构的名称。默认值：您的架构。

TYPE_NAME

ADT的名称，嵌套表类型或VARRAY类型。

如果创建类型会导致编译错误，则数据库将返回错误。您可以使用SQL * Plus命令SHOW ERRORS查看关联的编译器错误消息。

数据库隐式为您创建的每个用户定义类型定义构造函数方法。构造函数是系统提供的过程，在SQL语句或PL / SQL代码中用于构造类型值的实例。构造函数方法的名称是用户定义类型的名称。您还可以使用constructor_spec语法创建用户定义的构造constructor_spec 。

ADT构造函数方法的参数是ADT的数据属性。它们的出现顺序与ADT的属性定义顺序相同。嵌套表或varray构造函数的参数是嵌套表或varray的元素。

力

如果type_name存在且具有类型依赖项，但不具有表依赖项，则FORCE强制该语句替换该类型。（如果type_name具有表依赖项，则该语句在有或没有FORCE情况下失败。）

注意：

如果类型t1具有类型依赖t2 ，并且类型t2具有表依赖项，则类型t1也具有表依赖项。

属性

ADT属性的名称。ADT属性是一个数据项，其名称和类型说明符构成ADT的结构。您必须为每个ADT指定至少一个属性。该名称在ADT中必须是唯一的，但可以在其他ADT中使用。

如果要创建子类型，则属性名称不能与超类型链中声明的任何属性或方法名称相同。

数据类型

ADT属性的数据类型。此数据类型必须存储在数据库中;即，预定义数据类型或用户定义的独立集合类型。

数据类型的限制

  • 您不能对属性强加NOT NULL约束。

  • 您不能指定ROWID ， LONG或LONG RAW类型的属性。

  • 您不能为ADT指定UROWID的数据类型。

  • 如果指定REF类型的对象，则目标对象必须具有对象标识符。

  • 如果要创建用作表的嵌套表或varray列的集合类型，则不能指定ANYTYPE ， ANYDATA或ANYDATASET类型的属性。

object_type_def

创建ADT。形成数据结构的变量称为属性 。定义ADT行为的成员子程序称为方法 。

宾语

关键字OBJECT是必需的。

[NOT]最终，[NOT]无法实现，[NOT]持久

在语法的模式级别，这些子句指定该类型的继承属性。

[NOT]最终

使用[ NOT ] FINAL子句指示是否可以为此类型创建任何其他子类型：

  • （默认）如果不能为此类型创建其他子类型，请指定FINAL 。

  • 如果可以在此类型下创建更多子类型，请指定NOT FINAL 。

[NOT]不可思议

使用[ NOT ] INSTANTIABLE子句指示是否可以构造此类型的任何对象实例：

  • （默认）如果可以构造此类型的对象实例，请指定INSTANTIABLE 。

  • 如果此ADT不存在缺省或用户定义的构造函数，请指定NOT INSTANTIABLE 。您必须为具有不可实例化方法的任何类型以及任何没有属性的类型指定这些关键字，无论是在此语句中继承还是指定。

[NOT]坚持不懈

使用[ NOT ] PERSISTABLE子句指示对象类型的实例是否可持久化。

只有PERSISTABLE类型可以存储在表中。

  • （默认）如果所有对象类型属性都是可持久的，则可以指定PERSISTABLE 。不允许使用不可持久的属性创建可持久的对象类型。

  • 如果对象类型属性是可持久的或不可持久的，则可以指定NOT PERSISTABLE 。

对[NOT] PERSISTABLE的限制

您不能在子类型定义中指定[NOT] PERSISTABLE子句。子类型的持久性属性是从其超类型继承的。

varray_type_spec

将类型创建为有序元素集，每个元素具有相同的数据类型。

对varray_type_spec的限制

您可以为程序目的创建VARRAY类型的XMLType或LOB类型，例如，在PL / SQL或视图查询中。但是，不支持此类varray的数据库存储，因此您无法创建对象表或此类VARRAY类型的列。

[NOT]坚持不懈

（datatype [NOT NULL]）

使用[ NOT ] PERSISTABLE子句指示集合类型（ VARRAY或嵌套表）的实例是否可持久化。

示例14-24创建子类型

此语句显示如何创建示例oe架构中的子类型corporate_customer_typ 。

示例14-25创建类型层次结构

这些语句创建类型层次结构。

类型employee_t从类型person_t继承name和ssn属性，另外还有department_id和salary属性。类型part_time_emp_t继承所有从属性employee_t ，并通过employee_t ，那些person_t ，另外有一个num_hrs属性。类型part_time_emp_t默认为final，因此不能在其下创建其他子类型。 

CREATE TYPE person_t AS OBJECT（名称VARCHAR2（100），ssn NUMBER）NOT FINAL; CREATE TYPE employee_t UNDER person_t（department_id NUMBER，salary NUMBER）NOT FINAL; CREATE TYPE part_time_emp_t UNDER employee_t（num_hrs NUMBER）;

您可以使用类型层次结构来创建具有可替换列的可替换表和表。

示例14-30创建成员方法

此示例调用方法构造函数col.get_square 。

首先创建类型：

CREATE TYPE demo_typ2 AS OBJECT（a1 NUMBER，MEMBER FUNCTION get_square RETURN NUMBER）;

接下来，使用ADT列创建表，并将一些数据插入表中：

CREATE TABLE demo_tab2（col demo_typ2）; INSERT INTO demo_tab2 VALUES（demo_typ2（2））;

此示例更改employee_t类型的定义以将其与construct_emp函数关联。
