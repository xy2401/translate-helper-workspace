COLUMN_AUTH_INDICATOR功能

COLUMN_AUTH_INDICATOR函数检查指定的表列是否在特定表行上被授权。如果数据安全策略授权当前应用程序用户访问当前行的列值，或者该列未受任何数据安全策略保护，则返回1。如果应用程序用户未获得授权，则返回0 。

句法

COLUMN_AUTH_INDICATOR（col）RETURN BOOLEAN;

参数

参数 描述

关口

表或视图中的列。

此参数不接受对象类型列或表达式。

例

SELECT po_number，project_id，region，DECODE（COLUMN_AUTH_INDICATOR（price），0，'xxxxxx'，1，price）price FROM purchaseorder WHERE po_number BETWEEN 10000和10003;

XS_SYS_CONTEXT函数

XS_SYS_CONTEXT函数可以快速访问当前应用程序会话中的会话属性，而不会产生因使用PL / SQL API而产生的开销。所述SYS_XS_CONTEXT函数定义将镜像的SYS_CONTEXT功能，并且可以被描述为应用会话对应于SYS_CONTEXT 。 XS_SYS_CONTEXT返回请求的命名空间和属性。如果它们不存在，则返回NULL 。

ORA_CHECK_ACL功能

ORA_CHECK_ACL函数根据ACL列表检查应用程序用户是否具有查询的应用程序权限。当应用程序用户在启用了数据安全策略的表上运行查询时，Oracle数据库会自动使用此功能。如果已将指定的应用程序权限授予应用程序用户，则ORA_CHECK_ACL将返回1 。如果它们未被授予应用程序用户，则返回0 。

句法

ORA_CHECK_ACL（acls IN RAW，（特权IN VARCHAR（128））+）返回NUMBER;

参数

参数 描述

访问控制列表

RAID的8个字节的ACL ID列表。允许的最大acls数为250。

特权

正在检查应用程序权限名称。允许的最大应用程序权限数为100。

例子

以下示例使用ORA_CHECK_ACL检查应用程序用户是否已在ACL1 ACL中被授予P1和P2应用程序权限。

SELECT ORA_CHECK_ACL（TO_ACLID（'ACL1'），'P1'，'P2'）INTO ACLRESULT FROM DUAL;

ORA_GET_ACLIDS功能

ORA_GET_ACLIDS函数返回与当前应用程序用户的启用数据安全策略的表的对象实例关联的ACL IDS列表。Oracle数据库评估每个动态数据领域约束规则，因为如果已授予对当前行的访问权限， ORA_GET_ACLIDS捕获与匹配数据领域约束关联的所有ACL标识符。如果数据领域约束来自主 - 详细信息关系中的详细信息表，则ORA_GET_ACLIDS将从主表和详细信息表中检索ACL标识符。如果已将多个数据安全策略应用于表，则ORA_GET_ACLIDS将返回与每个策略关联的ACL。

句法

ORA_GET_ACLIDS（table_alias IN VARCHAR2，（特权IN VARCHAR（128））+）RETURN RAW;

参数

参数 描述

table_alias

来自子句的查询中的表或视图对象别名。

确保该表已启用XDS。为此，请查询DBA_XS_APPLIED_POLICIES数据字典视图。

如果指定已解析为启用XDS的表的视图，并且视图中有多个启用XDS的表，则Oracle数据库仅返回其中一个表。

特权

与返回的ACL标识符关联的应用程序特权名称。允许的最大应用程序权限数为100。

例

从SCOTT.DEPT t中选择ORA_GET_ACLIDS（t，'SELECT'，'VIEW_LOC'）;

ORA_CHECK_PRIVILEGE功能

ORA_CHECK_PRIVILEGE函数检查是否已将指定的权限授予应用程序用户。如果已为应用程序用户授予指定的权限，则ORA_CHECK_PRIVILEGE将返回1 。此功能仅适用于系统特权，例如CREATE_SESSION 。如果未向应用程序用户授予系统特权，则返回0 。

句法

ORA_CHECK_PRIVILEGE（（privs IN VARCHAR（128））+）返回NUMBER;

参数

参数 描述

PRIVS

正在检查的权限名称。允许的最大权限数为100。

例子

以下示例使用ORA_CHECK_PRIVILEGE检查应用程序用户是否已被授予CREATE_SESSION系统特权。 

从ORAL中选择ORA_CHECK_PRIVILEGE（'CREATE_SESSION'）;

TO_ACLID函数返回提供给它的ACL名称的ACL ID。
