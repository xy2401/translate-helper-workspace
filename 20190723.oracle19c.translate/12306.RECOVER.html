12.34恢复

句法

RECOVER { general | 管理 |开始备份|结束备份}

其中general子句具有以下语法：

[AUTOMATIC] [FROM location ] {{ full_database_recovery | partial_database_recovery | LOGFILE filename } [{TEST |允许整数 CORRUPTION | parallel_clause } [TEST |允许整数 CORRUPTION | parallel_clause ] ...]|继续[默认] |取消}

其中full_database_recovery子句具有以下语法：

[STANDBY] DATABASE [{UNTIL {CANCEL |时间日期 | CHANGE integer } |使用备份控制文件| SNAPSHOT TIME date} ...]

partial_database_recovery子句具有以下语法：

{TABLESPACE 表空间 [， tablespace ] ...| DATAFILE { filename | filenumber } [， filename | filenumber ] ...| STANDBY {TABLESPACE 表空间 [， 表空间 ] ......| DATAFILE { filename | filenumber } [， filename | filenumber ] ...}UNTIL [一致] CONTROLFILE}

parallel子句具有以下语法：

{NOPARALLEL | PARALLEL [ 整数 ]}

其中managed子句具有以下语法：

MANAGED STANDBY DATABASE recover_clause | cancel_clause | finish_clause

recover_clause具有以下语法：
{{DISCONNECT [FROM SESSION] | {TIMEOUT 整数 | NOTIMEOUT}} | {NODELAY |默认延迟| DELAY 整数 } |下一个整数 | {EXPIRE 整数 |没有EXPIRE} | parallel_clause |使用当前的日志文件| UNTIL CHANGE 整数 |通过{[THREAD integer ] SEQUENCE 整数 |所有ARCHIVELOG | {ALL |最后| NEXT} SWITCHOVER}} ...

cancel_clause具有以下语法：

CANCEL [IMMEDIATE] [等待| NOWAIT]

其中finish_clause具有以下语法：

[DISCONNECT [FROM SESSION]] [ parallel_clause ] FINISH [SKIP [STANDBY LOGFILE]] [WAIT | NOWAIT]

parallel_clause具有以下语法：

{NOPARALLEL | PARALLEL [ 整数 ]}

在一个或多个表空间，一个或多个数据文件或整个数据库上执行介质恢复。有关RECOVER命令的更多信息，请参阅Oracle数据库管理员指南 ， Oracle数据库SQL语言参考中的ALTER DATABASE RECOVER命令以及Oracle数据库备份和恢复用户指南 。

条款

自动

自动生成继续恢复操作所需的下一个归档重做日志文件的名称。Oracle数据库使用LOG_ARCHIVE_DEST（或LOG_ARCHIVE_DEST_ 1）和LOG_ARCHIVE_FORMAT参数（或其默认值）生成目标重做日志文件名。如果找到该文件，则应用该文件中包含的重做。如果找不到该文件，SQL * Plus会提示您输入文件名，并将生成的文件名显示为建议。

如果未指定AUTOMATIC或LOGFILE，则SQL * Plus会提示您输入文件名，并提示生成的文件名。您可以接受生成的文件名，也可以使用完全限定的文件名替换它。如果您知道归档文件名与Oracle数据库将生成的文件名不同，则可以使用LOGFILE子句指定文件名来节省时间。

从 地点

指定从中读取归档重做日志文件组的位置。location的值必须是完全指定的文件位置。如果省略此参数，则SQL * Plus假定归档重做日志文件组位于初始化参数LOG_ARCHIVE_DEST或LOG_ARCHIVE_DEST_1指定的位置。如果已使用SET LOGSOURCE设置文件，请不要指定FROM。

full_database_recovery

使您可以指定完整数据库的恢复。

partial_database_recovery

使您可以指定单个表空间和数据文件的恢复。

 LOGFILE

通过应用指定的重做日志文件继续进行介质恢复。在交互式恢复模式（AUTORECOVERY OFF）中，如果输入了错误的日志名称，则会显示错误日志名称的错误，并提示您输入新的日志名称。

测试

指定试用恢复以检测可能的问题。正常应用重做，但没有更改写入磁盘，并且在试用恢复结束时回滚更改。如果已还原备份，则只能使用TEST子句进行试用恢复。如果日志文件损坏，则指定允许恢复继续时可以容忍的损坏块数。在正常恢复期间， 整数不能超过1。 

允许整数 CORRUPTION

如果日志文件损坏，则指定允许恢复继续时可以容忍的损坏块数。

并行_clause

使您可以指定在恢复操作期间使用的并行处理程度。

继续

在中断禁用线程后继续多实例恢复。

继续默认

如果未指定其他日志文件，则使用Oracle数据库自动生成的重做日志文件继续恢复。这相当于指定AUTOMATIC，但Oracle数据库不提示输入文件名。

取消

终止基于取消的恢复。

 SNAPSHOT TIME日期

使用存储快照优化通过存储快照恢复数据库。

备用数据库

使用从主数据库复制的控制文件和归档重做日志文件恢复备用数据库。必须装入备用数据库但不能打开。

数据库

恢复整个数据库。

直到取消

指定基于取消的不完整恢复。通过提示归档重做日志文件的建议文件名来进行恢复，并在指定CANCEL而不是文件名时完成恢复。

直到时间

指定不完整的基于时间的恢复。使用单引号，格式如下：

'YYYY-MM-DD：HH24：MI：SS'
直到变化

指定基于更改的不完整恢复。 integer是您希望恢复的上一次更改之后的系统更改编号（SCN）的编号。例如，如果要将数据库还原到SCN为9的事务，则应指定UNTIL CHANGE 10。 

使用备份控制文件

指定使用控制文件的备份而不是当前控制文件。

 TABLESPACE

恢复特定的表空间。 tablespace是当前数据库中表空间的名称。您可以在一个语句中恢复最多16个表空间。 

 DATAFILE

恢复特定的数据文件。您可以指定任意数量的数据文件。

支持 TABLESPACE

使用从主数据库和控制文件复制的归档重做日志文件，在备用数据库中重建丢失或损坏的表空间。

支持 DATAFILE

使用从主数据库和控制文件复制的归档重做日志文件，在备用数据库中重建丢失或损坏的数据文件。

 UNTIL与CONTROLFILE一致

指定旧备用数据文件或表空间的恢复使用当前备用数据库控制文件。

并行[ 整数 ]

SQL * Plus选择的并行度等于所有参与实例上可用的CPU数量乘以PARALLEL_THREADS_PER_CPU初始化参数的值。

PARALLEL关键字会覆盖RECOVERY_PARALLELISM初始化参数。有关PARALLEL关键字的更多信息，请参阅“ Oracle Real Application Clusters管理和部署指南”指南。

使用integer指定并行度，这是并行操作中使用的并行线程数。每个并行线程可以使用一个或两个并行执行过程。 

 NOPARALLEL

指定串行恢复处理。这是默认值。

管理备用数据库

指定持续待机恢复模式。此模式假定备用数据库是整个备用数据库体系结构的活动组件。主数据库主动将其重做日志文件存档到备用站点。当这些归档重做日志到达备用站点时，它们可供受管备用恢复操作使用。持续待机恢复仅限于媒体恢复。

有关此子句参数的详细信息，请参阅“ Oracle数据库备份和恢复用户指南” 。 

断开

表示托管重做进程（MRP）应将归档重做文件应用为分离的后台进程。这样做会使当前会话可用。

超时

以分钟为单位指定持续恢复操作的等待时间。恢复过程等待整数分钟，以使请求的归档日志重做可用于写入备用数据库。如果重做日志文件在该时间内不可用，则恢复过程将终止并显示错误消息。然后，您可以再次发出该语句以返回持续待机恢复模式。

如果未指定此子句，或者指定NOTIMEOUT，则数据库将保持持续待机恢复模式，直到您使用RECOVER CANCEL子句重新发出语句或直到实例关闭或失败。

NODELAY

将延迟归档日志立即应用于备用数据库，覆盖主数据库上LOG_ARCHIVE_DEST_n参数中的任何DELAY设置。如果省略此子句，则根据参数设置延迟归档日志的应用。如果未在参数中指定DELAY，则立即应用存档日志。

默认延迟

在应用归档重做日志之前，等待LOG_ARCHIVE_DEST_n初始化参数中指定的默认分钟数。

DELAY 整数

在应用归档重做日志之前等待整数分钟。 

下一个整数

归档后，应尽快应用指定数量的归档重做日志。它会暂时覆盖主数据库上LOG_ARCHIVE_DEST_n参数中的任何DELAY设置，以及早期SQL * Plus RECOVER命令或ALTER DATABASE RECOVER命令中设置的任何延迟值。

EXPIRE 整数

指定自动终止托管恢复的当前时间的分钟数。

没有出口

禁用先前指定的EXPIRE 整数选项。 

使用当前的日志文件

在备用联机日志填充时从备份联机日志中恢复重做，而不要求它们首先在备用数据库中归档。

UNTIL CHANGE 整数

处理受管恢复，但不包括指定的系统更改编号（SCN）。

THROUGH THREAD 整数 SEQUENCE 整数

根据存档日志线程编号和序列号终止托管恢复。应用相应的存档日志后，托管恢复将终止。如果省略，THREAD默认为1。

通过所有ARCHIVELOG

继续托管备用，直到所有归档日志都已恢复。您可以使用此语句覆盖先前语句中发出的THROUGH THREAD 整数 SEQUENCE 整数子句。如果省略THROUGH子句，则这是默认值。 

通过所有开关

保持托管备用恢复在所有切换操作中运行。

通过最后一个开关

在最终的重做结束归档指示符之后终止托管备用恢复。

通过下一个开关

恢复下一个重做结束归档指示器后终止托管备用恢复。

取消 （ 管理条款）

应用当前归档重做文件后终止托管备用恢复。会话控制在恢复过程终止时返回。

取消立即

应用当前归档重做文件后，或在读取下一个重做日志文件后，以先到者为准，终止托管备用恢复。会话控制在恢复过程终止时返回。

取消立即等待

应用当前归档重做文件后或在读取下一个重做日志文件之后终止受管备用恢复，以先到者为准。会话控制在托管备用恢复终止时返回。

无法从发出RECOVER MANAGED STANDBY DATABASE语句的同一会话中发出CANCEL IMMEDIATE。

取消立即行动

应用当前归档重做文件后，或在读取下一个重做日志文件后，以先到者为准，终止托管备用恢复。会话控制立即返回。

取消NOWAIT

在读取下一个重做日志文件后终止托管备用恢复，并立即返回会话控制。

完

恢复备用数据库的当前备用联机日志文件。如果主数据库失败，则此子句可能很有用。它会覆盖为archivelog指定的任何延迟，以便立即应用日志。

如果您还指定了TIMEOUT，DELAY，EXPIRE或NEXT子句，则无法发出FINISH。

用法

您必须启用OSDBA角色。通过多线程服务器连接时，不能使用RECOVER命令。

要在整个数据库（所有表空间）上执行介质恢复，必须装入和关闭数据库，并且所有需要恢复的表空间必须在线。

要在表空间上执行介质恢复，必须装入或打开数据库，并且表空间必须处于脱机状态。

要对数据文件执行介质恢复，数据库可以保持打开状态并在损坏的数据文件脱机的情况下装入（除非该文件是SYSTEM表空间的一部分）。

在使用RECOVER命令之前，您必须从先前的备份中恢复已损坏的数据文件的副本。确保您可以访问所有存档和联机重做日志文件，这些文件可以追溯到备份时。

在恢复期间需要另一个日志文件时，会出现一个提示，指出所需文件的名称。该名称源自初始化参数LOG_ARCHIVE_DEST和LOG_ARCHIVE_FORMAT中指定的值。如有必要，应将恢复所需的归档重做日志文件的副本还原到LOG_ARCHIVE_DEST中指定的目标。您可以通过使用SET LOGSOURCE命令设置LOGSOURCE变量来覆盖初始化参数。

在恢复期间，您可以通过按回车键接受建议的日志名称，通过输入CANCEL而不是日志名称来取消恢复，或者在提示输入AUTO以自动选择文件而无需进一步提示。

如果已启用自动恢复（即SET AUTORECOVERY ON），则恢复将继续，而不会提示您使用文件名。应用每个日志文件时会显示状态消息。完成正常介质恢复后，将返回完成状态。

例子

要恢复整个数据库，请输入

恢复数据库

要在指定时间内恢复数据库，请输入

恢复数据库直到时间01-JAN-2001：04：32：00

要从数据库中恢复两个表空间ts_one和ts_two，请输入

RECOVER TABLESPACE ts_one，ts_two

要从数据库恢复数据文件data1.db，请输入

RECOVER DATAFILE'data1.db'
