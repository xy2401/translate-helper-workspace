6.215 PLAN_TABLE

PLAN_TABLE自动创建为全局临时表，以保存所有用户的EXPLAIN PLAN语句的输出。

PLAN_TABLE是EXPLAIN PLAN语句插入描述执行计划的行的默认样本输出表。

虽然为每个用户自动设置了PLAN_TABLE表，但您可以使用SQL脚本utlxplan.sql在架构中手动创建本地PLAN_TABLE 。

如果需要在PLAN_TABLE任何行上添加或更改注释，请使用UPDATE语句修改PLAN_TABLE的行。

成本

优化程序的查询方法估计的操作成本。未确定表访问操作的成本。此列的值没有任何特定的度量单位;它只是用于比较执行计划成本的加权值。此列的值是CPU_COST和IO_COST列的函数。

BYTES

NUMBER（38）

通过查询优化方法估计操作访问的字节数

OTHER_TAG

VARCHAR2（255）

描述OTHER列的内容：

  • SERIAL - 串行执行。目前，在这种情况下，不会在OTHER列中加载SQL。

  • SERIAL_FROM_REMOTE - 远程站点的串行执行。

  • PARALLEL_FROM_SERIAL - 串行执行。步骤的输出被分区或广播到并行执行服务器。

  • PARALLEL_TO_SERIAL - 并行执行。步骤的输出返回到串行查询协调器（QC）进程。

  • PARALLEL_TO_PARALLEL - 并行执行。步骤的输出被重新分配给第二组并行执行服务器。

  • PARALLEL_COMBINED_WITH_PARENT - 并行执行;步骤输出在同一并行过程中进入下一步。没有与父进程的进程间通信。

  • PARALLEL_COMBINED_WITH_CHILD - 并行执行。步骤的输入来自同一并行过程中的先前步骤。没有来自孩子的进程间通信。

PARTITION_START

VARCHAR2（255）

启动一系列访问分区的分区：

  • number - SQL编译器已识别启动分区，其分区号由number给出

  • KEY - 将在运行时通过分区键值识别启动分区

  • ROW REMOVE_LOCATION - 将在运行时从正在检索的每个记录的位置计算启动分区（与停止分区相同）。记录位置由用户或全局索引获得。

  • INVALID - 访问的分区范围为空

PARTITION_STOP

VARCHAR2（255）

停止一系列访问分区的分区：

  • number - SQL编译器已识别停止分区，其分区号由number给出

  • KEY - 将在运行时通过分区键值识别停止分区

  • ROW REMOVE_LOCATION - 停止分区（与起始分区相同）将在运行时从每个检索记录的位置计算。记录位置由用户或全局索引获得。

  • INVALID - 访问的分区范围为空

PARTITION_ID

NUMBER（38）

已计算PARTITION_START和PARTITION_STOP列的值对的步骤

其他

长

特定于用户可能会发现有用的执行步骤的其他信息（请参阅OTHER_TAG列）

OTHER_XML

CLOB

提供特定于执行计划执行步骤的额外信息。此列的内容使用XML构建，因为可以在那里存储多条信息。这包括：

  • 用于解析查询的模式的名称

  • 生成解释计划的Oracle数据库的版本号

  • 与执行计划关联的哈希值

  • 大纲的名称（如果有）或用于构建执行计划的SQL配置文件

  • 指示是否使用动态统计数据来制定计划

  • 大纲数据，一组可用于重新生成同一计划的优化程序提示

分配

VARCHAR2（30）

用于将行从生产者查询服务器分发到使用者查询服务器的方法

CPU_COST

NUMBER（38）

通过查询优化器的方法估计的操作的CPU成本。此列的值与操作所需的机器周期数成比例。对于使用基于规则的方法的语句，此列为NULL。

IO_COST

NUMBER（38）

通过查询优化器的方法估计的操作的I / O开销。此列的值与操作读取的数据块数成比例。对于使用基于规则的方法的语句，此列为NULL。

TEMP_SPACE

NUMBER（38）

由查询优化器的方法估计的操作使用的临时空间（以字节为单位）。对于使用基于规则的方法的语句，或对于不使用任何临时空间的操作，此列为NULL。

ACCESS_PREDICATES

VARCHAR2（4000）

用于在访问结构中定位行的谓词。例如，启动或停止索引范围扫描的谓词。

FILTER_PREDICATES

VARCHAR2（4000）

用于在生成行之前过滤行的谓词

投影

VARCHAR2（4000）

操作产生的表达

时间

NUMBER（38）

通过查询优化估计的操作的经过时间（以秒为单位）。对于使用基于规则的方法的语句，此列为NULL。

QBLOCK_NAME

VARCHAR2（128）

查询块的名称（由系统生成或由具有QB_NAME提示的用户定义）
