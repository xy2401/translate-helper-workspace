19.1关于ADR命令解释器（ADRCI）实用程序

ADRCI是一个命令行工具，是Oracle Database 11g中引入的故障可诊断性基础结构的一部分。

ADRCI使您能够：

  • 查看自动诊断信息库（ADR）中的诊断数据。

  • 查看Health Monitor报告。

  • 将事件和问题信息打包到zip文件中以传输到Oracle Support。

诊断数据包括事件和问题描述，跟踪文件，转储，运行状况监视器报告，警报日志条目等。

ADR数据由ADR目录上的操作系统权限保护，因此无需登录ADRCI。

ADRCI具有丰富的命令集，可以在交互模式或脚本中使用。

注意：

管理诊断数据的更简单和推荐的方法是使用Oracle企业管理器支持工作台（Support Workbench）。ADRCI提供了支持Workbench的大多数功能的命令行替代方案，并添加了列出和查询跟踪文件等功能。

有关Support Workbench的完整信息，请参见Oracle数据库管理员指南 。

19.2定义

这些是用于ADRCI和Oracle数据库故障可诊断性基础结构的术语的定义。

特别：

自动诊断存储库（ADR）

自动诊断存储库（ADR ）是一个基于文件的存储库，用于存储数据库诊断数据，例如跟踪，转储，警报日志，运行状况监视器报告等。它具有跨多个实例和多个产品的统一目录结构。从版本11 g开始，数据库，Oracle自动存储管理（Oracle ASM）和其他Oracle产品或组件将所有诊断数据存储在ADR中。每个产品的每个实例都将诊断数据存储在其自己的ADR主目录下（请参阅ADR Home ）。例如，在具有共享存储和Oracle ASM的Oracle Real Application Clusters（Oracle RAC）环境中，每个数据库实例和每个Oracle ASM实例在ADR中都有一个主目录。 ADR的统一目录结构使客户和Oracle支持人员能够跨多个实例和多个产品关联和分析诊断数据。

问题

问题是数据库中的严重错误。严重错误包括内部错误，如ORA-00600和其他严重错误，如ORA-07445 （操作系统异常）或ORA-04031 （共享池中的内存不足）。ADR中会跟踪问题。每个问题都有一个问题密钥和一个唯一的问题ID。 （请参阅问题密钥 。）

事件

事件是一次出现问题。多次出现问题时，会为每次出现创建一个事件。在ADR中跟踪事件。每个事件都由数字事件ID标识，该ID在ADR中是唯一的。发生事件时，数据库会在警报日志中生成一个条目，向Oracle Enterprise Manager发送事件警报，以转储文件（事件转储）的形式收集有关事件的诊断数据，使用事件ID标记事件转储，并将事件转储存储在为该事件创建的ADR子目录中。

诊断和解决严重错误通常始于事件警报。您可以使用ADRCI命令获取ADR中所有事件的列表。每个事件仅映射到一个问题。

事件受洪水控制，因此单个问题不会产生太多事件和事件转储。有关事件泛滥控制的更多信息，请参见“ Oracle数据库管理员指南” 。

问题关键

每个问题都有一个问题键 ，它是一个包含错误代码（如ORA 600 ）的文本字符串，在某些情况下还包含一个或多个错误参数。如果两个事件的问题键匹配，则认为它们具有相同的根本原因。

事件包

事件包 （包）是关于一个或多个问题的事件的数据的集合。在将事件数据发送到Oracle Support之前，必须使用事件打包服务（IPS）将其收集到包中。创建一个包之后，可以将外部文件添加到包，从包中取出选定的文件，或擦洗 （编辑）选择的文件包中的，以除去敏感数据。

包是仅逻辑构造，直到您从包内容创建物理文件。也就是说，事件包作为ADR中的元数据集合开始。在添加和删除包内容时，仅修改元数据。准备好将数据上载到Oracle Support时，可以使用ADRCI创建物理包，该数据包将数据保存到zip文件中。然后，您可以将zip文件上载到Oracle Support。

最终处理

在ADRCI可以从逻辑包生成物理包之前，必须最终确定包。这意味着调用其他组件将任何相关的诊断数据文件添加到此包中已有的事件中。Finalizing还会添加最近的跟踪文件，警报日志条目，Health Monitor报告，SQL测试用例和配置信息。生成物理包时会自动运行此步骤，也可以使用ADRCI实用程序手动运行此步骤。手动完成程序包后，您可以查看已添加的文件，然后删除或编辑包含敏感信息的任何文件。

ADR主页

ADR主目录是所有诊断数据跟踪，转储，警报日志等的根目录，用于特定Oracle产品或组件的特定实例。例如，在具有Oracle ASM的Oracle RAC环境中，每个数据库实例和每个Oracle ASM实例都具有ADR主目录。所有ADR家庭共享相同的分层目录结构。每个ADR主目录中的一些标准子目录包括警报（用于警报日志），跟踪（用于跟踪文件）和事件（用于事件信息）。所有ADR家庭都位于ADR基本目录中。（参见ADR基础 。）

某些ADRCI命令可以同时与多个ADR家庭一起使用。当前ADRCI HOMEPATH确定在其中搜索诊断数据时发出的命令ADRCI的ADR家。有关更多信息，请参阅Homepath 。

ADR基地

为了允许跨多个ADR家庭的诊断数据相关，ADR家庭在称为ADR基础的相同根目录下组合在一起。例如，在Oracle RAC环境中，ADR基础可能位于共享磁盘上，并且每个Oracle RAC实例的ADR主目录可位于此ADR基础之下。

数据库实例的ADR基地的位置由DIAGNOSTIC_DEST初始化参数设置。如果省略此参数或为null，则数据库将其设置为默认值。有关详细信息，请参见Oracle数据库管理员指南 。

当多个数据库实例共享Oracle主目录时，无论它们是多个单个实例还是Oracle RAC数据库的实例，并且当这些实例中的一个或多个将ADR基址设置在不同位置时，最后一个启动实例将确定默认ADR基础对于ADRCI。

HOMEPATH

所有ADRCI命令都在当前 ADR家中的诊断数据上运行。任何时候都可以有多个ADR家庭。一些ADRCI命令（如SHOW INCIDENT ）搜索并显示从当前所有的ADR舍诊断数据，而其他命令要求只有一个ADR家是电流，并且如果多于一个是当前显示错误消息。

该ADRCI HOMEPATH决定ADR家是电流。它通过指向ADR基础层次结构中的目录来实现。如果它指向单个ADR主目录，则该ADR主目录是当前唯一的ADR主目录。如果主路径指向层次结构中位于ADR主目录级别之上的目录，则指向的目录下方的所有ADR主目录都将成为当前目录。

ADRCI启动时，默认情况下，主路径为空。这意味着ADR基础下的所有ADR家庭都是最新的。

SHOW HOME和SHOW HOMEPATH命令显示当前ADR HOME的列表， SET HOMEPATH命令设置主路径。

19.4在使用ADRCI命令之前设置ADRCI主路径

诊断问题时，您可能希望使用来自多个数据库实例或组件的诊断数据，或者您可能希望关注来自一个实例或组件的诊断数据。

要处理来自多个实例或组件的诊断数据，必须确保所有这些实例或组件的ADR回家是最新的 。要仅处理来自一个实例或组件的诊断数据，必须确保只有该实例或组件的ADR主目录是最新的。您可以通过设置ADRCI主路径来控制当前的ADR家庭。

如果多个家庭是当前的，这意味着主路径指向ADR目录结构中包含多个ADR主目录的目录。要专注于单个ADR主目录，必须将目录层次结构中较低的主路径设置为单个ADR主目录。

例如，如果具有数据库名称orclbi的Oracle RAC数据库有两个实例，其中实例具有SID或orclbi1和orclbi2 ，并且Oracle RAC使用共享Oracle主目录，则存在以下两个ADR主目录：

/ diag / rdbms / orclbi / orclbi1 / / diag / rdbms / orclbi / orclbi2 /

在所有ADRCI命令和输出中，ADR主目录路径（ADR家庭）始终相对于ADR基础表示。因此，如果ADR基地目前是/ u01 / app / oracle，这两个ADR家庭的绝对路径如下：

/ u01 / app / oracle / diag / rdbms / orclbi / orclbi1 / / u01 / app / oracle / diag / rdbms / orclbi / orclbi2 /

您可以使用SET HOMEPATH命令将一个或多个ADR主目录设置为当前。如果ADR base是/ u01 / app / oracle并且您想将主路径设置为/ u01 / app / oracle / diag / rdbms / orclbi / orclbi2 /，则使用以下命令：

adrci>设置homepath diag / rdbms / orclbi / orclbi2

当ADRCI启动时，默认情况下主路径为空，这意味着ADR基础下的所有ADR家庭都是当前的。因此，在之前引用的示例中，两个Oracle RAC实例的ADR主页都是最新的。

adrci> show homes ADR Homes：diag / rdbms / orclbi / orclbi1 diag / rdbms / orclbi / orclbi2

在这种情况下，您运行的任何ADRCI命令（假设该命令支持多个当前ADR主页）都可以使用来自两个ADR主目录的诊断数据。如果要将主路径设置为/ diag / rdbms / orclbi / orclbi2，则只有具有SID或orclbi2的实例的ADR主orclbi2最新的。 

adrci> set homepath diag / rdbms / orclbi / orclbi2 adrci> show homes ADR Homes：diag / rdbms / orclbi / orclbi2

在这种情况下，您运行的任何ADRCI命令都只能使用来自此单个ADR主目录的诊断数据。

19.5查看警报日志

警报日志既写为XML格式的文件，也写为文本文件。您可以使用任何文本编辑器查看文件的任一格式，也可以运行ADRCI命令以查看省略XML标记的XML格式的警报日志。

要使用ADRCI查看警报日志：

  1. 以交互模式启动ADRCI。

    有关说明 ，请参阅启动ADRCI和获取帮助 。

  2. （可选）使用SET HOMEPATH命令选择（使当前）单个ADR主页。

    您可以先使用SHOW HOMES命令查看当前ADR家庭列表。有关详细信息，请参阅Homepath并在使用ADRCI命令之前设置ADRCI主路径 。

  3. 在ADRCI提示符下，输入以下命令：
    显示警告

    如果当前有多个ADR主页，则会提示您从列表中选择单个ADR主目录。在默认编辑器中显示警报日志，省略XML标记。

  4. 退出编辑器以返回ADRCI命令提示符。

19.8.1关于包装事件

                  

                     
包装事件分为三个步骤。

                     

                        
第1步：创建逻辑事件包。

                     

                     
                     

                        
事件包（包）表示为逻辑，因为它仅作为自动诊断存储库（ADR）中的元数据存在。在从逻辑包生成物理包之前，它没有任何内容。为逻辑包分配了一个包编号，您可以在后续命令中通过该编号引用它。

                        
您可以将逻辑包创建为空包，也可以基于事件编号，问题编号，问题密钥或时间间隔创建包。如果将包创建为空包，则可以在步骤2中向其中添加诊断信息。

                        
基于事件创建程序包意味着包括诊断数据转储，运行状况监视器报告等等。基于问题编号或问题密钥创建包意味着在包中包含引用该问题编号或问题密钥的事件的诊断数据。基于时间间隔创建包意味着包括在时间间隔中发生的事件的诊断数据。

                     

                     
                     

                        
步骤2：将诊断信息添加到事件包中

                     

                     
                     

                        
如果基于事件编号，问题编号，问题密钥或时间间隔创建了逻辑包，则此步骤是可选的。您可以向包中添加其他事件，也可以将ADR中的任何文件添加到包中。如果您创建了一个空包，则必须使用ADRCI命令将事件或文件添加到包中。

                     

                     
                     

                        
第3步：生成物理事件包

                     

                     
                     

                        
当您提交生成物理包的命令时，ADRCI会收集所有必需的诊断文件，并将它们添加到指定目录中的zip文件中。您可以生成完整的zip文件或增量zip文件。增量文件包含自从为同一逻辑包创建上一个zip文件以来添加或更改的所有诊断文件。只有在创建完整文件后才能创建增量文件，并且可以根据需要创建任意数量的增量文件。为每个zip文件分配一个序列号，以便可以按正确的顺序分析文件。

                        
Zip文件根据以下方案命名：
packageName _ mode _ sequence .zip
哪里：

                        
    
                           
  • 
                              
    packageName由问题键的一部分后跟一个时间戳组成
    
                           
    • 
                           
  • 
                              
    mode为COM或INC ，表示完整或增量
    
                           
    • 
                           
  • 
                              
    sequence是一个整数
    
                           
    • 
                        

                        
例如，如果为2006年9月6日下午4:53创建的逻辑包生成完整的zip文件，然后为同一逻辑包生成增量zip文件，则可以创建名称类似于以下：
ORA603_20060906165316_COM_1.zip ORA603_20060906165316_INC_2.zip

    例如，以下命令根据事件3创建包：
    IPS创建包装事件3
    ADRCI生成类似于以下内容的输出：

    分配给此逻辑包的包编号为10。

以下是IPS CREATE PACKAGE命令的变体：
IPS创建包
这会创建一个空包。在生成诊断数据之前，必须使用IPS ADD INCIDENT或IPS ADD FILE命令将诊断数据添加到包中。
                           
IPS CREATE PACKAGE PROBLEM problem_ID

这将创建一个包，并包含引用指定问题ID的事件的诊断信息。 （问题ID是整数。）您可以获取问题ID，用于从显示的报告事故SHOW INCIDENT -MODE BRIEF命令。由于可能存在许多具有相同问题ID的事件，因此ADRCI会向包中添加针对此问题ID发生的前三个事件（ 早期事件 ）和最后三个事件（ 延迟事件 ）的诊断信息，不包括任何事件超过90天。
                           

                           

                              
注意：

                              
早期和晚期事件的数量以及90天的年龄限制是可以更改的默认值。请参阅IPS SET CONFIGURATION 。

                           

                           
ADRCI还可以添加其他事件，这些事件在时间或其他标准中与已添加的事件密切相关。
IPS CREATE PACKAGE PROBLEMKEY“ problem_key ”
这将创建一个包，并包含引用指定问题键的事件的诊断信息。您可以从SHOW INCIDENT命令显示的报告中获取问题密钥。由于可能存在许多具有相同问题密钥的事件，因此ADRCI仅向包中添加前三个早期事件的诊断信息以及此问题密钥的最后三个晚期事件，不包括超过90天的事件。
                           

                           

                              
注意：

                              
早期和晚期事件的数量以及90天的年龄限制是可以更改的默认值。请参阅IPS SET CONFIGURATION 。

                           

                           
ADRCI还可以添加其他事件，这些事件在时间或其他标准中与已添加的事件密切相关。

                           
如果问题键包含空格或引号，则必须用单引号（'）或双引号（“）括起来。
IPS CREATE PACKAGE SECONDS 秒

这将创建一个包，并包含从sec秒到现在发生的所有事件的诊断信息。 sec必须是整数。
                           
IPS创建包装时间' start_time '到' end_time '
这将创建一个包，并包含在指定时间范围内发生的所有事件的诊断信息。 start_time和end_time必须采用' YYYY-MM-DD HH24:MI:SS.FF TZR '格式。这是NLS_TIMESTAMP_TZ_FORMAT初始化参数的有效格式字符串。时间的分数（ FF ）部分是可选的，并且HH24:MI:SS分隔符可以是冒号或句点。
                           

                           
例如，以下命令创建一个包含在2010年7月24日到7月30日期间发生的事件的包：
IPS CREATE PACKAGE TIME'2010-07-24 00:00:00 -07：00'至'2010-07-30 23.59.59 -07：00'

19.9.8跑

                  

                     
运行ADR命令解释器（ADRCI）脚本。

                     

                        
语法和说明

                        
运行script_name

                        
@ script_name

                        
@@ script_name

                        
变量script_name是包含要运行的ADRCI命令的文件。除非提供完整路径名，否则ADRCI会在当前目录中查找脚本。如果文件名没有文件扩展名，则ADRCI使用默认扩展名.adi 。
                        

                        
run和@命令是同义词。@@命令类似于run和@ 。但是，在脚本中使用时， @@使用调用脚本的路径来查找script_name ，而不是当前目录。
                        

                        
在使用run命令之前，不需要安装ADR主页。
                        

                     

                     
                     

                        
例
运行my_script @my_script

语法和说明

                     

                     
                     
选择{* | [ field1 ，[ field2 ，...]}FROM {incident | problem} [WHERE predicate_string ] [ORDER BY field1 [， field2 ，...][ASC | DSC | DESC]] [GROUP BY field1 [， field2 ，...]][ 有了having_predicate_string ]

指示是否查询事件或问题。

使用类似SQL的谓词字符串仅显示谓词为true的事件或问题。谓词字符串必须用双引号括起来。

显示按给定顺序按字段排序的结果，以及升序（ ASC ）和降序（ DSC或DESC ）。指定ORDER BY子句时，默认情况下结果以升序显示。
                                    

                                 		
                              

                              

                                 
                                    
GROUP BY field1 , field2 , ...

                                 		
                                 
                                    
显示按指定字段分组的结果。

-rt | -t

                                 		
                                 
                                    
按时间戳排序跟踪文件名。 -t按时间戳按升序对文件名进行排序， -rt按相反顺序对它们进行排序。请注意，文件名仅相对于其目录进行排序。列出跟踪文件的多个目录会对每个目录应用单独的排序。
                                    

                                    
使用此选项时，每个文件名旁边都会列出时间戳。

                                 		
                              

                              

                                 
                                    
-i inc1 inc2 ...
                                    

                                 		
                                 
                                    
仅选择为给定事件ID生成的跟踪文件。

                                 		
                              

                              

                                 
                                    
-path path1 path2 ...
                                    

                                 		
                                 
                                    
仅查询给定路径名下的跟踪文件。

                                 		
                              

                           
                        

                     

                     
                     

                        
例子

                     

                     
                     

                        
此示例显示当前ADR主目录下的所有跟踪文件：
show tracefile
此示例显示所有mmon跟踪文件，按时间顺序按相反顺序排序：
show tracefile％mmon％-rt
此示例显示路径/ home / steve / temp下事件1和4的所有跟踪文件：
show tracefile -i 1 4 -path / home / steve / temp

19.9.29 SPOOL

                  

                     
将ADRCI输出定向到文件。

                     

                        
语法和说明
SPOOL filename [[APPEND] | [OFF]]
filename是输出定向的文件名。如果未给出完整路径名，则在当前ADRCI工作目录中创建该文件。如果未指定文件扩展名，则使用默认扩展名.ado 。APPEND输出附加到文件末尾。否则，文件将被覆盖。使用OFF关闭假脱机。
                        

                        
在使用它之前，此命令不需要设置ADR主页。

SPOOL myfile
SPOOL myfile.ado APPEND
SPOOL OFF
SPOOL

19.10 ADRCI故障排除

               

                  
描述一些常见的ADRCI错误消息。

                  

                     
以下是一些常见的ADRCI错误消息及其可能的原因和补救措施：

                  

                  
                  

                     
没有设置ADR基础

                  

                  
                  

                     
原因 ：您可能已启动ADRCI，其ORACLE_HOME环境变量的值为null或无效。
                     

                     
操作 ：退出ADRCI ，设置ORACLE_HOME环境变量，然后重新启动ADRCI。有关详细信息，请参阅“ ADR Base ” 。
                     

                  

                  
                  

                     
DIA-48323：指定的路径名字符串必须位于当前ADR主目录中

                  

                  
                  

                     
原因 ：不允许ADR主目录之外的文件作为此命令的事件文件。
                     

                     
操作 ：使用ADR主目录中的事件文件重试。
                     

                  

                  
                  

                     
DIA-48400：ADRCI初始化失败

                  

                  
                  

                     
原因 ：ADR Base目录不存在。
                     

                     
操作 ：检查DIAGNOSTIC_DEST初始化参数的值，并确保它指向包含至少一个ADR主目录的ADR基目录。如果缺少DIAGNOSTIC_DEST或null，请在ORACLE_HOME / log中检查有效的ADR基目录层次结构。
                     

                  

                  
                  

                     
DIA-48431：必须至少指定一个ADR主路径

                  

                  
                  

                     
原因 ：该命令要求至少一个ADR主页为当前。
                     

                     
操作 ：使用SET HOMEPATH命令使一个或多个ADR之家处于当前状态。
                     

                  

                  
                  

                     
DIA-48432：ADR主路径字符串无效

                  

                  
                  

                     
原因 ：提供的ADR主页无效，可能是因为该路径不存在。
                     

                     
操作 ：检查提供的ADR主路径是否存在。
                     

                  

                  
                  

                     
DIA-48447：输入路径[ 路径 ]不包含任何ADR家庭

                  

                  
                  

                     
原因 ：使用SET HOMEPATH设置ADR主页时，必须提供相对于当前ADR基础的路径。
                     

                     
操作 ：如果新的所需ADR主页不在当前ADR基础内，首先使用SET BASE设置ADR base，然后使用SHOW HOMES检查新ADR基础下的ADR主页。接下来，如有必要，使用SET HOMEPATH设置新的ADR主页。
                     

                  

                  
                  

                     
DIA-48448：此命令不支持多个ADR家庭

                  

                  
                  

                     
原因 ：当前ADRCI会话中有多个当前ADR家庭。
                     

                     
操作 ：使用SET HOMEPATH命令生成单个ADR原点电流。
