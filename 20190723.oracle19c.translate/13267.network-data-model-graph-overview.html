5.1网络建模简介

在许多应用程序中，功能或对象被建模为网络中的节点和链接。网络模型包含逻辑信息，例如节点和链路之间的连接关系，链路的方向以及节点和链路的成本。

利用逻辑网络信息，您可以分析网络并回答问题，其中许多问题与路径计算和跟踪有关。例如，对于生化途径，您可以找到两种化合物之间的所有可能的反应途径;或者对于道路网络，您可以找到以下信息：

  • 两个城市之间的最短（距离）或最快（旅行时间）路径是多少？

  • 距离特定机场最近的酒店是什么？如何到达那里？

除了逻辑网络信息之外，诸如节点位置和链路几何形状的空间信息可以与网络相关联。此信息可以帮助您对逻辑信息（例如路径的成本）进行建模，因为其物理长度可以直接根据其空间表示来计算。

5.2.1让空间执行大多数操作

要通过让Spatial和Graph执行大多数必要操作来创建网络，请按照下列步骤操作：

  1. 使用CREATE_ <network-type> _NETWORK形式的名称过程创建网络，其中<network-type>反映您要创建的网络类型：

    • SDO_NET.CREATE_SDO_NETWORK用于具有非LRS SDO_GEOMETRY对象的空间网络

    • 具有LRS SDO_GEOMETRY对象的空间网络的SDO_NET.CREATE_LRS_NETWORK

    • SDO_NET.CREATE_TOPO_NETWORK用于与拓扑几何形状（SDO_TOPO_GEOMETRY）对象的空间网络

    • SDO_NET.CREATE_LOGICAL_NETWORK用于不包含空间信息的逻辑网络

    这些过程中的每一个都创建必要的网络数据模型图表（在网络数据模型图表中描述），并将具有适当网络元数据信息的行插入到xxx_SDO_NETWORK_METADATA视图中（在xxx_SDO_NETWORK_METADATA视图中描述）。

  2. 将数据插入到节点和链接表中，并在必要时插入到路径和路径链接表中。（ 网络数据模型图表中描述了节点，链接，路径和路径链接表 。）
  3. 使用SDO_NET.VALIDATE_NETWORK函数验证网络。
  4. 对于空间（SDO或LRS）网络，将适当的信息插入USER_SDO_GEOM_METADATA视图，并在几何列上创建空间索引。

    如果计划将视图用作节点，链接或路径表，则必须在USER_SDO_GEOM_METADATA视图中插入有关节点，链接或路径表的信息时指定TABLE_NAME列值的视图名称。

5.2.2自己进行操作

要通过自己执行必要的操作来创建网络，请按照下列步骤操作：

5.3网络数据模型图概念

网络是一种数学图形，使用连接来捕获对象之间的关系。

连接可以基于空间接近度，也可以不基于空间接近度。例如，如果两个城镇位于湖的两侧，那么如果您想从一个城镇开车到另一个城镇，那么基于空间接近度的最短路径（穿过湖中央的直线）是不相关的。相反，要找到最短的行驶距离，您需要有关道路和交叉口的连接信息以及各个链路的“成本”。

网络由一组节点和链接组成。每个链接（有时也称为边或段）指定两个节点。

可以引导网络（即，默认情况下，开始和结束节点确定链路方向）或无向引导（即，可以在任一方向上遍历链路）。

以下是与网络数据模型图相关的一些关键术语：功能

  • 节点 （也称为顶点）是链接可以相互连接的点。一个 隔离节点是未包含在任何链接中的节点。（如果删除包含该节点的所有链接，则非隔离节点将被隔离。）

  • 链接表示两个节点之间的关系。在有向网络中，任何链路都可以是无向的 （即，能够从起始节点到末端节点或从末端节点到起始节点遍历）或定向 （即，只能从路径遍历）。启动节点到结束节点）。在无向网络中，所有链路都是无向的。

  • 网络元素是节点或链接。

  • 路径是节点和链路的交替序列，以节点开始和结束，并且通常没有节点和链接出现多次。（允许在路径中重复节点和链接，但在大多数网络应用程序中很少见。）

  • 子路径是沿路径的部分路径，可以作为网络分析操作的结果创建，也可以由用户显式创建。子路径在子路径中进行了解释和说明。

  • 一个 逻辑网络包含连接信息，但没有几何信息。这是用于网络分析的模型。根据应用程序，逻辑网络可以被视为有向图或无向图。

  • 一个 空间网络包含连接信息和几何信息。在空间网络中，节点和链路是没有LRS信息的SDO_GEOMETRY几何对象（ SDO网络 ）或具有LRS信息（ LRS网络 ）或SDO_TOPO_GEOMETRY对象（ 拓扑几何网络 ）。

    在LRS网络中，每个节点包括几何ID值和度量值，并且每个链接包括几何ID值以及开始和结束度量值;并且每种情况下的几何ID值是指具有LRS信息的SDO_GEOMETRY对象。根据应用，空间网络可以是定向的或不定向的。

  • 一个 feature是与节点或链接相关联的网络应用程序中的关注对象。功能和要素图层中介绍了要素和要素图层类型

  • 成本是非负数字属性，可以与用于计算最小成本路径的链接或节点相关联， 最小成本路径是从起始节点到结束节点具有最小总成本的路径。您可以在网络元数据中指定单个成本因子，例如链接的行驶时间或行驶距离，以及检查成本的网络分析功能将使用此指定的成本因子。

  • 持续时间是非负数字属性，可以与链接或节点关联，以指定链接或节点的持续时间值。持续时间值可以指示分钟数或任何其他用户确定的重要性。您可以在网络元数据中指定单个持续时间因子，例如链接的驱动时间;但是，如果使用持续时间而不是成本来指示已用时间，则检查成本的网络分析功能将不考虑指定的持续时间因子。

  • State是一个字符串属性， ACTIVE或INACTIVE ，它与链接或节点相关联，以指定网络分析功能是否考虑链接或节点。例如，如果节点的状态是INACTIVE ，则在计算两个节点之间的最短路径时，将忽略来自该节点的任何链接。创建链接或节点时，默认情况下状态为ACTIVE ，但您可以将状态设置为INACTIVE 。

  • Type是一个字符串属性，可以与链接或节点关联，以指定链接或节点类型的用户定义值。

  • 临时链接，节点和路径仅存在于网络内存对象中，并且在写入网络内存对象时不会写入数据库。例如，在网络分析和编辑会话期间，您可以创建临时节点来表示最短路径计算的街道地址，但在保存编辑操作的结果时不保存这些临时节点。

  • 可达节点是可以从给定节点到达的所有节点。到达节点是可以到达给定节点的所有节点。

  • 节点的程度是节点的链接数（即，事件）。in-degree是入站链接的数量， out-degree是出站链接的数量。

  • 一个 连接组件是一组直接或间接连接的网络节点。如果节点A可以到达节点B，则它们必须属于相同的连接组件。如果没有连接两个节点，则可以断定它们之间没有可能的路径。此信息可用作过滤器以避免不必要的路径计算。

  • 一个连接图的生成树是连接图的所有节点的树（即，没有循环的图）。（生成树中将忽略链接的方向。）最低成本生成树是连接所有节点并具有最小总成本的生成树。

  • 分区网络是包含多个分区的网络。对大型网络进行分区只能将必要的分区按需加载到内存中，从而提供更好的整体性能。

    网络分区是子网络，每个子网络覆盖整个网络的节点和链路的子集。网络分区是按需分析的基本处理单元。它们是通过将网络中的每个节点分配给一个分区ID来创建的。网络分区信息存储在分区表中。

  • 按需加载（按需加载分析）是一种将大型网络划分为可管理分区并且仅在分析期间加载所需分区的方法，从而将内存限制作为考虑因素并提供更好的整体性能。

  • 分区BLOB是网络分区的二进制表示。它们提供更快的分区加载时间。它们存储在分区BLOB表中。

  • 按需加载分区缓存是内存占位符，用于在网络分析期间加载到内存中的网络分区。您可以配置分区缓存。

  • 用户定义的数据是用户想要与网络表示关联的信息（与连接无关）。用户定义的数据可以在节点，链接，路径和子路径级别定义，并存储在节点，链接，路径和子路径表中的列中。

5.4.1道路网示例

在典型的道路网络中，道路的交叉点是节点，两个交叉点之间的道路段是链路。道路的空间表示与网络中的节点和链路本身并不相关。例如，如果该形状点不与交叉点相关联，则道路的空间表示中的形状点（反映道路中的急转弯）不是网络中的节点;并且单个空间对象可以构成网络中的若干链路（例如与三条交叉道路相交的直线段）。道路网络的一个重要操作是找到从起点到终点的路径，最小化行程时间或距离。路径计算可能存在其他约束，例如使路径经过特定地标或避开特定交叉点。

5.4.2地铁（列车）网络示例

任何主要城市的地铁网络可能最好被建模为逻辑网络，假设停靠站和轨道线的精确空间表示是不重要的。在这样的网络中，系统上的所有站点构成网络的节点，并且如果列车直接在这两个站点之间行进，则链路是两个站点之间的连接。使用列车网络的重要操作包括查找可从指定站点到达的所有站点，查找两个指定站点之间的站点数量，以及查找两个站点之间的行程时间。

5.4.3多模式网络和时间实例

多模式网络是由多种模式组成的网络，例如由驾驶和步行路线组成的网络。它们通常被建模为单独的网络（特定模式）并被视为聚合网络，因此可以表示和计算单模式以及多模式的路由。通常，多模网络通过不同模式的调度“连接”，并且在这种情况下，它们也是时间网络。一个例子是计算行程，步行到最近的公交车站，乘坐最快的公交车路线，在离目的地最近的车站下车，然后步行到目的地。您还可以添加驾驶或飞行等模式。

时态建模和分析为网络建模和分析增加了时间（时间）维度。时间因素在静态（非时间）网络之上提供成本和/或约束。一个例子是考虑交通模式（与时间相关的旅行时间成本）而不是静态旅行时间成本。

许多城市交通网络由多种模式组成，例如公共汽车，地铁和通勤铁路线，可以跨越模式进行转移（例如，从公共汽车到地铁）。每种运输模式在较大的运输网络内具有组件网络。组件网络可以使用节点和链接建模，跨模式的传输可以建模为连接可以进行传输的停靠点的链接。

这种多式联运网络的一个重要特征是其基于计划的运行。在执行常见的网络操作（例如计算从起点到终点的最快路线）时，必须考虑调度信息和跨模式的可能传输。停靠点处的调度信息可以表示为表示这些停靠点的节点处的用户定义数据。在多模式网络中使用调度信息的操作的示例是（A）在指定的开始时间找到从起点到终点的最快路线（最小行驶时间），以及（B）在开始时找到最新的出发时间指向到达指定到达时间的终点。

5.5网络层次结构

某些网络应用程序需要不同抽象级别的表示。例如，两个主要进程可能表示为在最高抽象级别之间具有链接的节点，并且每个主要进程可能具有若干从属进程，这些进程在下一级别表示为节点和链接。

通过网络层次结构，您可以通过为每个节点分配层次结构级别来表示具有多个抽象级别的网络。（链接未分配层次结构级别，链接可以位于同一层次结构级别或不同级别的节点之间。）层次结构中最低（最详细）的级别为级别1，连续的较高级别编号为2,3，依此类推。

网络层次结构的相邻级别的节点具有父子关系。较高级别的每个节点可以是较低级别的一个或多个节点的父节点 。较低级别的每个节点可以是较高级别的一个节点的子节点 。同级节点是具有相同父节点的节点。

链接也可以具有父子关系。但是，由于链接未分配给层次结构级别，因此链接父子关系和网络层次结构级别之间不一定存在关系。 同级链接是具有相同父链接的链接。

通过网络元数据中的CHILD_NETWORK和HIERARCHY_TABLE_NAME列定义父子网络关系。

5.8.2 NFE要素类

要素类根据属性值，形状和样式描述一组要素。每个要素类仅属于一个要素图层，因此要素类中的要素组也属于同一要素图层。

例如，电网可能包括两个功能层：变压器和导体。这些要素图层中的每一个都可能包含两个要素类：

  • 变压器功能层可能包括HT变压器（高压变压器）和LT变压器（低压变压器），其中每种功能类型的变压器都具有指定的输入和输出电压以及张力类型。两种要素类型都与点（它们的形状）相关联。每种要素类型都有一个不同的关联图标，可用于图表。

  • 导体特征层可能包括HT导体（高压导体）和LT导体（低压导体），其中每种特征类型中的导体都具有指定的属性值。两种要素类型都与线条（形状）相关联。每种要素类型都有一个不同的关联图标，可用于图表。

5.8.3 NFE规则

NFE模型中的规则是与特征连接相关的约束。规则仅适用于从头开始创建的模型，即不适用于通过现有网络创建的模型。规则可以自定义，并且可以支持任何类型的连接约束。

规则通常可以分为基数规则和连接规则：

  • 基数规则用于具有特定特征（即属于同一要素类）的点要素，以指定点要素可接受的传入和传出连接的最大和最小数量。

  • 连接规则用于指示如果两个要素具有某些特定特征（要素图层，要素类和属性）和特定交互，则是否可以连接这两个要素。只要点要素具有可用空间以支持所需的相应传入和传出连接，连接规则将允许与点要素的连接，如其相关的基数规则所指示的。

    连接规则是连接NFE中功能的唯一方法，它们始终是正面的;也就是说，它们允许连接，但不能否认它们。元素之间缺少连接规则表示元素无法连接。

线点连接规则

线点连接规则指出，只要线要素和点要素以某种方式相互作用，它们就可以连接。

线点连通性规则及其基数规则可以表示如下：“具有与条件P1匹配的要素类C1的要素层L1的任何线要素可以连接到具有要素类C2的要素层L2的点要素匹配条件P2。Point Feature的基数表示允许最多X个传入连接和最多Y个传出连接。“

规则实现始终使用决策处理程序来管理功能之间的连接。NFE提供了一种默认实现，用于连接线点和线路规则的功能。执行规则时，默认的决策处理程序实现将尝试尽可能多地连接与规则匹配的元素，但是在某些情况下，您希望对连接的控制比默认情况下规则提供的更多，在这种情况下，您必须修改默认决策处理器实现..

使用水网络示例，假设两个阀门可以在空间点与管道相互作用。每个阀门有四个出口。（A）所有四个管端都可以连接到单个阀的四个出口，或者（B）两个管端可以连接到一个阀的两个相对的出口，另外两个管端可以连接到两个相对的出口。另一个阀门。默认的决策处理程序实现将尝试第一种方法（所有四个管道末端连接到单个阀门）;但是如果你想在两个可用的阀门中分配四个管道，你可以使用第二种方法实现自定义决策处理器（两个管道末端连接到第一个阀门，另外两个管道末端连接到第二个阀门）。

规则实例

当由于某些规则的应用而连接功能时，连接功能组称为规则实例 ，即允许连接的规则实例。规则实例在规则实例表中维护。

仅当不存在依赖规则实例时（即，不依赖于任何现有规则实例的连接），才能删除或修改规则定义

如果删除了规则实例中涉及的要素元素，则可以自动删除或不自动删除规则实例。如果要素元素删除导致元素数量低于规则实例的最小数量（取决于规则类型），则删除规则实例。但是，如果删除要素元素不会导致元素数量低于规则实例的最小数量，则会从规则实例中删除要素元素，但不删除规则实例本身（但它是改性）。

5.10.1对网络进行分区

您可以使用SDO_NET.SPATIAL_PARTITION过程对网络进行分区，指定每个分区中的最大节点数。分区结果存储在自动生成的分区表中，并且分区元数据信息被插入到网络元数据中。（作为使用该过程的替代方法，您可以通过创建和填充分区表来对网络进行分区。）您可以使用其他SDO_NET子程序来查询分区元数据。

一个好的分区策略是最小化分区之间的链接数量，这减少了需要加载的分区数量以及需要重新加载相同分区的可能次数。此外，太小的分区在分析期间需要过多的分区装载和卸载。

假设1 GB内存，建议的每个分区的最大节点数介于5,000和10,000之间。考虑到可用内存，分析类型和网络大小，您可以调整数量并查看最适合您的应用程序的数量。您还应该考虑配置分区缓存大小。

父主题： 使用按需加载进行网络分析

5.10.2生成分区BLOB

要增强网络加载的性能，您可以选择将分区作为BLOB存储在网络分区BLOB表中。此信息需要存储在网络元数据视图中，以便利用更快的分区加载时间。请注意，如果更新了网络或分区信息，则还需要重新生成分区BLOB。

分区BLOB是包含网络分区信息的二进制数据流，例如节点数，链接数，每个节点的属性，每个链接的属性等。如果存在分区BLOB，则Spatial和Graph使用它在加载操作期间读取信息，而不是执行耗时的数据库查询。

要生成分区BLOB，请使用SDO_NET.GENERATE_PARTITION_BLOBS过程。分区BLOB及其元数据存储在分区BLOB表中 。

5.10.3配置分区缓存

在执行网络分析之前，您可以通过修改XML配置文件以覆盖默认配置来配置网络分区缓存以优化性能。您可以指定以下内容：

  • 缓存大小：分区缓存中的最大节点数

  • 分区源：来自网络表或分区BLOB

  • 驻留分区：不会从缓存中刷新的分区ID，但在加载后将保留在内存中

  • 缓存刷新策略： CachingHandler实现的类名

    默认缓存策略是LeastRecentlyUsed ，它在缓存已满时刷新最旧的分区内存。您可以通过实现CachingHandler接口来指定其他缓存策略。

5.10.5使用链接级别进行优先级建模

尽管按需加载方法降低了分析大型网络时内存限制的影响，但分析操作有时仍然非常慢。例如，对角线穿过整个网络的两个节点的最短路径分析可能需要遍历网络中的几乎每个链路，并且这将花费大量时间用于具有多于例如两百万个节点的网络。

为了进一步缩短网络分析时间，您可以在不同的链路级别上执行分析。链接级别是分配给链接的正整数，表示此链接的首选级别。链接级别越高，偏好越高。例如，道路网络可以包括两个链路级别，1级用于本地道路，2级用于高速公路。在网络分析期间，高速公路优先于当地道路，最小链路级别为1。（如果没有为链接分配链接级别，则链接将使用默认链接级别1。）

链接级别具有隐式继承属性，这意味着较高链接级别的网络必须是较低链接级别的网络的子网。也就是说，链路级别2是链路级别1的子网，链路级别3是链路级别2的子网，依此类推。

您可以在加载网络或分区时指定链接级别，从而导致加载该级别和更高级别的链接。使用道路网络示例，本地道路的链路级别1和高速公路的链路级别2，在装载操作上指定链路级别1加载链路级别1和2（即本地道路和高速公路）的链路，但指定链路级别2在加载操作中仅加载高速公路链接。如果只想使用高速公路链接执行分析，可以通过为加载操作指定链接级别2来优化性能。

5.10.6预先计算的分析结果

某些分析操作（例如连通组件分析）可能非常耗时。要提高运行时性能，可以调用SDO_NET.FIND_CONNECTED_COMPONENTS过程，该过程计算网络中的连接组件并将结果存储在连接组件表中 。

在运行时，在调用最短路径分析或可达性分析之前，您可以通过查询连接的组件表来检查感兴趣的节点是否属于同一个连接的组件。如果预先计算的组件信息不存在，则最短路径和可达性分析可能需要很长时间才能发现两个节点实际上没有连接。

本节中的表格与使用网络特征编辑（NFE）的特征操作有关（请参阅使用网络特征编辑（NFE）的特征建模 ）。这些表用于描述每个NFE模型。

在大多数情况下，使用PL / SQL函数SDO_NFE.CREATE_MODEL_STRUCTURE或Java API创建新的NFE模型时，会自动创建或更新这些表。但您也可以手动创建表或视图，并将它们注册到所需的模型。

  • 自动创建的点默认属性表
    自动创建的点默认属性表包含有关自动创建的点的属性的默认值的信息，作为执行连接线 - 线规则的结果。
  • 连接线 - 线规则表
    连接线路规则表包含有关适用于NFE模型中涉及的功能的连接线路规则定义的信息。
  • 连接线点规则表
    连接线点规则表包含有关适用于NFE模型中涉及的功能的连接线点规则定义的信息。
  • 要素类表
    要素类与要素相关。要素类表包含要素图层中所有要素类的记录，并且必须包含下表中描述的列。
  • 要素类属性约束表
    要素类属性约束表包含有关要素类属性的用户限制的信息，这些要素属性是为要素类所属的要素图层定义的相同属性。
  • 要素类默认预定义连接点表
    要素类默认预定义连接点表包含有关要素类描述的线要素的此类默认连接点要素的信息。
  • 要素类关系表
    要素类关系表包含有关要素和要素类之间关系的信息，即哪个要素属于哪个要素类。
  • 特征规则关系表
    特征规则关系表包含将规则中涉及的每个特征元素与导致其生成或连接的规则相关联的信息。
  • 功能用户数据表
    要素用户数据表包含目录类型的要素类属性的信息。
  • 功能用户数据目录表
    您可以从目录中为要素类属性指定值;也就是说，要素类中的属性类型可以是目录类型。功能用户数据目录表保留了可用于此目的的目录的注册表。
  • 功能用户数据目录值表
    功能用户数据目录值表包含功能用户数据目录表中定义的目录所拥有的值列表。
  • 点基数规则表
    点基数规则表包含特定点要素在NFE模型中必须支持的最大和最小入站和出站连接的配置。
  • 规则决策处理程序表
    规则决策处理程序表包含有关在执行连接规则（行 - 行或行 - 点）时作为决策处理程序执行的Java类和/或PL / SQL过程的名称的信息。
  • 规则实例表
    规则实例表包含有关在NFE模型中应用线路或线路点连接规则生成的规则实例的信息。

父主题： 网络数据模型图表

以下性能注意事项适用于PL / SQL和Java API：

  • 如果您计划仅分析或编辑空间网络的非空间方面，则可以在执行分析或编辑之前，在USER_SDO_NETWORK_METADATA视图（在xxx_SDO_NETWORK_METADATA视图中描述） 中将 NETWORK_CATEGORY列值设置为LOGICAL ，然后更改值，从而获得更好的性能之后回到SPATIAL 。

    例如，您可以在找到两个节点之间的最短路径时使用此技术，因为最短路径计算会考虑成本值。但是，在设置空间几何对象或链接的结束度量值时，无法使用此技术。

  • 如果您不打算修改任何网络对象（即，如果您计划仅执行网络分析操作或检索网络信息），则可以通过将网络内存对象创建为只读（即，通过指定不允许更新）。

5.14.1通过在网络元数据中指定所有者来进行跨模式访问

要使非所有者用户（具有适当的权限）能够访问网络，您可以在网络元数据中指定网络所有者。对于将被允许访问网络的每个非所有者用户，请按照下列步骤操作：

  1. 确保用户具有对必要的网络数据模型图表的SELECT或READ权限访问权限。如果用户没有此访问权限，请以网络所有者身份进行连接并授予权限。例如，以User1身份连接并执行以下语句：

    GRANT选择ON network1_node $ TO user2; GRANT选择ON network1_link $ TO user2; GRANT选择ON network1_path $ TO user2; GRANT选择ON network1_plink $ TO user2;

  2. 以非所有者用户身份进行连接。例如，以User2身份连接。

  3. 使用网络所有者的架构名称在USER_SDO_NETWORK_METADATA视图中限定网络的网络数据模型图表（在xxx_SDO_NETWORK_METADATA视图中说明）。例如，如果此视图中尚未定义网络，请在连接为User2时输入以下内容：

    INSERT INTO user_sdo_network_metadata（network，network_category，geometry_type，node_table_name，node_geom_column，link_table_name，link_geom_column，link_direction，path_table_name，path_geom_column，path_link_table_name）VALUES（'NETWORK1'，'SPATIAL'，'SDO_GEOMETRY'，'USER1.NETWORK1_NODE $'，'GEOMETRY' ，'USER1.NETWORK1_LINK $'，'GEOMETRY'，'DIRECTED'，'USER1.NETWORK1_PATH $'，'GEOMETRY'，'USER1.NETWORK1_PLINK $'）;

    如果已在此视图中定义网络，请更新定义以使用模式名称限定每个表名。例如：

    UPDATE USER_SDO_NETWORK_METADATA SET node_table_name ='USER1.NETWORK1_NODE $'，link_table_name ='USER1.NETWORK1_LINK $'，path_table_name ='USER1.NETWORK1_PATH $'，path_link_table_name ='USER1.NETWORK1_PLINK $'WHERE network ='NETWORK1';

在这种情况下，User2现在可以将NETWORK1读入内存。

5.14.2使用视图进行跨模式访问

要使非所有者用户（具有适当的权限）能够访问网络或网络的特定部分，您可以创建视图。对于将被允许访问网络的每个非所有者用户，请按照下列步骤操作：

  1. 确保用户具有对必要的网络数据模型图表的SELECT或READ权限访问权限。如果用户没有此访问权限，请以网络所有者身份进行连接并授予权限。例如，以User1身份连接并执行以下语句：

    GRANT选择ON network1_node $ TO user2; GRANT选择ON network1_link $ TO user2; GRANT选择ON network1_path $ TO user2; GRANT选择ON network1_plink $ TO user2;

  2. 以非所有者用户身份进行连接。例如，以User2身份连接。

  3. 在每个必要的网络数据模型图节点上创建视图，每个视图选择关联表中的所有列。使用网络所有者的架构名称限定表名称。例如，在连接为User2时：

    CREATE VIEW network1_node $ AS select * from user1.network1_node $; CREATE VIEW network1_link $ AS select * from user1.network1_link $; CREATE VIEW network1_path $ AS select * from user1.network1_path $; CREATE VIEW network1_plink $ AS select * from user1.network1_plink $;

    注意：
