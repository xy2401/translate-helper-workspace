描述

Oracle ODBC驱动程序使应用程序能够从Windows客户端以及使用Microsoft开放式数据库连接（ODBC）API读取和写入Oracle数据库的UNIX客户端连接到Oracle数据库。

Oracle ODBC驱动程序分发工具包包含动态链接库和共享库（适用于UNIX平台），帮助文件（在UNIX和Windows平台上），许可证副本和此产品说明。要使用启用ODBC的应用程序，除Oracle ODBC驱动程序外，还需要以下软件：

  • Oracle Client＆Net 12.2版

  • Oracle数据库服务器

Oracle ODBC驱动程序符合ODBC 3.52规范。

ODBC 11.2.0.1.0

描述版本11.2.0.1.0的新功能。

  • Oracle ODBC驱动程序已得到增强，可预取LONG和LONG RAW数据，以提高ODBC应用程序的性能。为此，必须在Windows上的注册表中设置LONG数据的最大大小（ MaxLargeData ）（还需要在DSN中添加注册表项MaxLargeData= ），并在UNIX平台上手动设置odbc.ini文件。此增强功能最多可将Oracle ODBC驱动程序的性能提高10倍，具体取决于用户设置的MaxLargeData大小。MaxLargeData的默认值为0。可由用户设置的MaxLargeData的最大值为64 KB（65536字节）。即使MaxLargeData的值设置为某个大于65536的值，所获取的数据也只是65536字节。如果用户在数据库中具有LONG和LONG RAW数据（大于65536字节），则MaxLargeData应设置为0（默认值），这将导致单行提取并且可以获取完整的LONG数据。如果用户在非轮询模式下传递的缓冲区小于MaxLargeData大小，则如果数据库中的LONG数据大小大于缓冲区大小，则会发生数据截断错误。（增强请求7006879）。

  • 现在，Oracle ODBC驱动程序支持OCIDescribeAny()调用（以获取元数据），以便在应用程序大量调用通过odbc.ini名为UseOCIDescribeAny的选项返回REF CURSORS小型打包过程时提高性能。要在UNIX平台上使用OCIDescribeAny() ，请在odbc.ini文件中将UseOCIDescribeAny设置为T （ True ），并在Windows上通过DSN中的注册表设置。 （增强请求7704827）。

ODBC 11.1.0.1.0

描述版本11.1.0.1.0的新功能。

  • 添加了对禁用RULE提示的支持。Oracle ODBC驱动程序现在允许用户指定选项以选择是否在目录API中使用RULE Hint。已完成更改以提高目录API的ODBC驱动程序的性能。该选项的默认值为TRUE ，这意味着默认情况下RULE Hint不会在目录API中使用。（Bug4150034）。

  • 添加了对绑定号码作为FLOAT支持。当列包含FLOAT数据时，通过将NUMBER列的列绑定作为FLOAT引入，可以加快使用绑定变量作为FLOAT的查询执行速度。 （Bug4608183）。

  • 添加了对OCI语句缓存功能的支持，该功能为每个会话提供和管理语句缓存。通过实现对OCI语句缓存选项的支持，Oracle ODBC驱动程序将看到性能的提高，用户必须在同一连接中多次解析同一语句。语句缓存标志的默认值为FALSE 。

  • 通过保存不必要的内存调用来更改结果集缓存的行为，以提高ODBC驱动程序性能。

ODBC 10.2.0.1.0

描述10.2.0.1.0版的新功能。

  • 添加了对命名参数的支持，按名称绑定参数。Oracle ODBC驱动程序现在允许应用程序在过程调用中按名称指定存储过程的参数。命名参数仅用于对存储过程的调用，不能在其他SQL语句中使用。（Bug3617324）

  • 更改了描述某些包中存在的存储过程的元数据的行为。已完成更改以提高存储过程调用的ODBC驱动程序的性能。现在，如果存储过程在某个包中，则将使用PL / SQL过程描述元数据。（Bug4030664）

  • 版本10.2.0.1中引入了对Linux和Solaris平台的Oracle ODBC驱动程序的支持。从版本10.2.0.4开始，ODBC驱动程序也可用于AIX平台。

ODBC 10.1.0.2.0

描述版本10.1.0.2.0的新功能。

  • 在带有参数标记的INSERT / UPDATE语句中添加了对NCHAR列的支持（错误2827132）。以前，驱动程序没有注意到该列是NCHAR类型，并且在OCI级别绑定时未能设置正确的属性。作为副作用， SQLDescribeParam调用现在返回INSERT / UPDATE语句中参数的有用信息，但不会在其他语句类型中返回。

  • 增加了对大于4 GB的LOB的支持。要求Oracle客户端和数据库都是Oracle数据库10g （10.1）或更高版本。

  • 在Oracle Database 10g及更高版本的服务器中添加了对新BINARY_FLOAT和BINARY_DOUBLE数据类型的支持。

  • 在SQL语句中添加了对MONTHNAME和DAYNAME函数的支持。

安装后任务

完成这些安装后任务。

  • 在UNIX平台上配置Oracle ODBC驱动程序。

    您可以通过运行install-home /odbc/utl/odbc_update_ini.sh来配置Oracle ODBC驱动程序。

    实用程序odbc_update_ini.sh采用四个命令行参数：

    • arg-1 ：已安装unixODBC DM完整路径。

    • arg-2 ：驱动程序安装位置的完整路径（可选）;如果未传递此参数，则将驱动程序路径设置为运行该实用程序的目录。

    • arg-3 ：驱动程序名称（可选）;如果未传递此参数，则将驱动程序名称设置为Oracle 12c ODBC driver 。

    • arg-4 ：数据源名称（可选）;如果没有传递任何值，则DSN设置为OracleODBC-12c 。

  • 更新并验证环境变量的值，例如： PATH ， LD_LIBRARY_PATH ， LIBPATH和TNS_ADMIN 。

卸载ODBC驱动程序

完成以下任务以从UNIX平台卸载Oracle ODBC驱动程序。

  • 更新~/.odbc.ini文件：

    • 从[ODBC Data Sources]删除DSN条目（例如， OracleODBC-12c ）。

    • 删除相应DSN的完整DSN信息。

  • 更新ODBCDM_HOME /etc/odbcinst.ini文件：

    • 删除Oracle 12c ODBC驱动程序的驱动程序信息。

  • 删除UNIX平台的Oracle ODBC驱动程序

    • 删除libsqora.so.12.1

  • 重置环境变量，例如： PATH ， LD_LIBRARY_PATH ， LIBPATH和TNS_ADMIN 。
