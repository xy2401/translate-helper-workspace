Oracle数据库如何使用监听器文件（listener.ora）

listener.ora文件是listener.ora器的配置文件。它可以包括接受连接请求的协议地址，数据库列表以及它正在侦听的其他服务，以及监听器使用的控制参数。

您可以使用服务器控制实用程序（SRVCTL）命令或使用NETCA修改Oracle Clusterware和Oracle RAC使用的侦听器的配置。不需要手动编辑listener.ora文件。

每个侦听器都配置有一个或多个指定其侦听端点的协议地址。侦听器代理使用侦听器动态更新端点。从Oracle Database 11g第2版开始， listener.ora文件现在只包含IPC密钥和以下信息：

（ADDRESS =（PROTOCOL = TCP）（HOST =）（PORT = 1521））

在前面的示例中，协议ADDRESS隐式引用到本地节点的HOST端点。listener.ora文件在Oracle RAC数据库的每个节点上都是相同的。侦听端点（例如端口号）将与侦听器动态注册。

在安装Oracle RAC之前，在Oracle Grid Infrastructure安装期间，NETCA在名为LISTENER的Grid主目录中创建并启动默认侦听LISTENER 。侦听器配置有默认协议侦听地址。侦听器配置为响应指向安装期间指定的一个协议地址的连接请求。

在Oracle RAC安装期间，Oracle RAC数据库使用Grid主目录中的侦听器，并配置有关Oracle RAC数据库的服务信息。数据库服务自动向侦听器注册其信息，例如服务名称，实例名称和负载信息。动态服务注册消除了对数据库服务的静态配置的需要。但是，如果计划使用Oracle Enterprise Manager，则需要静态服务配置。
