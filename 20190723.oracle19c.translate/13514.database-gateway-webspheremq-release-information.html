大型有效负载支持

DG4MQ 11 g支持大型有效负载或长度超过32767字节的消息。

有关更多信息，请参阅随putlongsample.sql getlongsample.sql安装的putlongsample.sql和getlongsample.sql示例程序。

数据库链接和别名库

通过数据库链接建立与网关的连接。

从DG4MQ 10 g版本2及更高版本开始，此数据库链接不再与每个DG4MQ网关过程调用相关联（例如， PGM.MQPUT@dblink ）。从10 g版本2及更高版本开始，需要在MQOPEN使用的MQOD数据类型中仅定义一次，并且此数据库链接在MQOPEN调用返回的对象句柄中注册。有关详细信息，请参阅随网关安装的示例程序。默认情况下，在执行DG4MQ部署脚本时，将使用您的默认SID创建公共数据库链接dg4mqdepdblink 。

已知的限制

本主题描述了此版本中的已知限制。

已知此版本存在以下限制。

自定义LOG_DESTINATION

自定义网关初始化文件以进行Microsoft Windows平台的网关跟踪时存在一个已知问题。自定义LOG_DESTINATION的路径名时，分隔符必须是双反斜杠。例如：

LOG_DESTINATION = C：\\预言\\产品\\ 12.2 \\ dg4mqs \\ dg4mq \\登录\\ dg4mqs.log

注意：

如果没有为Microsoft Windows平台定义LOG_DESTINATION ，则使用默认名称，并在ORACLE_HOME\dg4mq\trace目录中创建日志

自定义部署脚本pgmobj.sql

为Microsoft Windows平台自定义网关部署脚本pgmobj.sql时存在一个已知问题。定义libdg4mq的路径名时，分隔符必须是反斜杠。例如，创建或替换库libdg4mq为：

创建或替换库libdg4mq为'C：\ oracle \ product \ 12.2 \ dg4mqs \ bin \ oradg4mqs.dll'事务性

要么

创建或替换库libdg4mq为'$ ORACLE_HOME \ bin \ oradg4mqs.dll'事务性

CALLBACK链接

Oracle Database Gateway for WebSphere MQ不支持CALLBACK链接。尝试与网关的CALLBACK链接将返回以下错误消息：

ORA-02025：SQL语句中的所有表必须位于远程数据库中
