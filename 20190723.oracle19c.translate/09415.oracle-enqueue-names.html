D Oracle入队名称

本附录列出了Oracle排队。队列是共享内存结构（锁），用于序列化对数据库资源的访问。 它们可以与会话或事务相关联。排队名称显示在DBA_LOCK和DBA_LOCK_INTERNAL数据字典视图的LOCK_TYPE列中。

资源唯一地标识可以由实例（本地资源）内的不同会话或实例之间（全局资源）锁定的对象。尝试锁定资源的每个会话都将在资源上加入队列 。

注意：

队列的名称及其定义可能会随着版本的不同而变化。

也可以看看：

“ DBA_LOCK_INTERNAL ”和“ DBA_LOCK ”

Oracle排队的是：

  • BL，缓冲区缓存管理

  • BR，备份/恢复

  • CF，Controlfile事务

  • CI，跨实例调用调用

  • CU，Bind Enqueue

  • DF，数据文件

  • DL，直接装载机索引创建

  • DM，数据库安装

  • DR，分布式恢复过程

  • DW，SecureFiles

  • DX，分布式事务

  • FP，文件对象

  • FS，文件集

  • HW，高水位锁

  • IN，实例编号

  • IR，实例恢复

  • IS，实例状态

  • 四，库缓存失效

  • JI，Enqueue在AJV快照刷新期间使用

  • JQ，工作队列

  • KK，重做日志“踢”

  • KP，Oracle Data Pump启动和关闭过程中的争用

  • KO，多个对象检查点

  • L [AP]，库缓存锁

  • LS，Log Start或Switch

  • MM，Mount Definition

  • MR，媒体恢复

  • N [AZ]，库缓存引脚

  • PE，ALTER SYSTEM SET PARAMETER = VALUE

  • PF，密码文件

  • PI，平行奴隶

  • PR，流程启动

  • PS，并行从动同步

  • Q [AZ]，行缓存

  • RO，对象重用

  • RT，重做线程

  • RW，Row Wait

  • SC，系统变更编号

  • SM，SMON

  • SN，序列号

  • SQ，序列号入队

  • SR，同步复制

  • SS，排序段

  • ST，空间管理交易

  • SV，序列号值

  • TA，交易恢复

  • TC，线程检查点

  • TE，扩展表

  • TM，DML入队

  • TO，临时表对象入队

  • TS，临时段（也是TableSpace）

  • TT，临时表

  • TX，交易

  • UL，用户定义的锁

  • 联合国，用户名

  • 美国，撤消细分，序列化

  • WL，正在写重做日志

  • XA，实例属性锁定

  • XI，实例注册锁

  • ZA，移动审计表时的独占锁定
