POWERMULTISET_BY_CARDINALITY

句法


插图描述powermultiset_by_cardnlty.eps

目的

POWERMULTISET_BY_CARDINALITY将嵌套表和基数作为输入，并返回嵌套表的嵌套表，该表包含指定基数的嵌套表的所有非空子集（称为子矩阵）。

  • expr可以是任何求值为嵌套表的表达式。

  • cardinality可以是任何正整数。

  • 如果expr解析为null，则Oracle数据库返回NULL 。

  • 如果expr解析为空的嵌套表，则Oracle返回错误。

  • 嵌套表的元素类型必须具有可比性。有关非标量类型可比性的信息，请参阅“ 比较条件 ” 。

    注意：

    PL / SQL不支持此功能。

例子

首先，创建一个数据类型，该数据类型是cust_address_tab_type数据类型的嵌套表：

CREATE TYPE cust_address_tab_tab_typ AS TABLE OF cust_address_tab_typ; /

接下来，复制所有嵌套表行中的元素，以将嵌套表行的基数增加到2：

更新customers_demo SET cust_address_ntab = cust_address_ntab MULTISET UNION cust_address_ntab;

现在，使用POWERMULTISET_BY_CARDINALITY函数从customers_demo表中选择嵌套表列cust_address_ntab ：

SELECT CAST（POWERMULTISET_BY_CARDINALITY（cust_address_ntab，2）AS cust_address_tab_tab_typ）FROM customers_demo;
