2.7缩短的班级名称

每个Java源，类和资源都存储在服务器中自己的架构对象中。架构对象的名称源自完全限定名称，该名称包括相关路径或包信息。点用斜线代替。

但是，架构对象名称最多只能包含30个字符，并且所有字符必须合法且可转换为数据库字符集中的字符。如果任何完全限定名称超过30个字符或包含非法或不可转换的字符，则Oracle数据库会将其转换为短名称或别名，以用作架构对象的名称。Oracle数据库会跟踪名称以及如何在它们之间进行转换。如果完全限定名称不超过30个字符且没有非法或不可转换的字符，则将其用作架构对象名称。

由于Java类和方法的名称可能超过最大SQL标识符长度，因此Oracle数据库在内部使用缩写名称进行SQL访问。Oracle数据库在DBMS_JAVA包中提供LONGNAME()函数，用于检索任何截断名称的原始Java类名。 

FUNCTION longname（短名称VARCHAR2）RETURN VARCHAR2

此函数返回Java模式对象的完全限定名称，该名称使用其别名指定。以下是用于显示无效类的完全限定名称的语句示例：

SELECT dbms_java。  longname（object_name）FROM user_objects WHERE object_type ='JAVA CLASS'和status ='INVALID';

您还可以使用DBMS_JAVA包的SHORTNAME()函数为数据库指定全名。该函数将全名作为输入并返回相应的短名称。通过查询USER_OBJECTS视图，此函数可用于验证是否已成功加载类。 

FUNCTION shortname（longname VARCHAR2）RETURN VARCHAR2
