47.1 DBMS_DB_VERSION概述

DBMS_DB_VERSION包指定Oracle版本号和其他对基于Oracle版本的简单条件编译选择有用的信息。

Oracle Database 12c第2版的软件包如下所示。

包DBMS_DB_VERSION是VERSION CONSTANT PLS_INTEGER：= 12; -  RDBMS版本号RELEASE CONSTANT PLS_INTEGER：= 2; -  RDBMS版本号ver_le_9_1 CONSTANT BOOLEAN：= FALSE; ver_le_9_2 CONSTANT BOOLEAN：= FALSE; ver_le_9 CONSTANT BOOLEAN：= FALSE; ver_le_10_1 CONSTANT BOOLEAN：= FALSE; ver_le_10_2 CONSTANT BOOLEAN：= FALSE; ver_le_10 CONSTANT BOOLEAN：= FALSE; ver_le_11_1 CONSTANT BOOLEAN：= FALSE; ver_le_11_2 CONSTANT BOOLEAN：= FALSE; ver_le_11 CONSTANT BOOLEAN：= FALSE; ver_le_12_1 CONSTANT BOOLEAN：= FALSE; ver_le_12_2 CONSTANT BOOLEAN：= TRUE; ver_le_12 CONSTANT BOOLEAN：= TRUE; END DBMS_DB_VERSION;

布尔常量遵循命名约定。每个常量都给出一个布尔表达式的名称。例如：

  • VER_LE_9_1表示版本<= 9且版本<= 1

  • VER_LE_10_2表示版本<= 10并且发布<= 2

  • VER_LE_10表示版本<= 10

这些布尔常量的典型用法是：

$ IF DBMS_DB_VERSION.VER_LE_10 $ THEN版本10及更早版本代码$ ELSIF DBMS_DB_VERSION.VER_LE_11 $ THEN版本11代码$ ELSE版本12及更高版本代码$ END

此代码结构将保护对版本12的代码的任何引用。它还会阻止控制包常量DBMS_DB_VERSION 。在版本10下编译程序时引用VER_LE_11 。类似的观察也适用于版本11。即使静态常量VER_LE_11未在版本10数据库中定义，此方案仍然有效，因为如果DBMS_DB_VERSION条件编译会保护$ELSIF免于评估。 VER_LE_10为TRUE 。

47.3 DBMS_DB_VERSION示例

此示例使用条件编译来保护新功能。

创建或替换程序磨刀石IS  - 注意条件编译结构 - 可以中断常规的PL / SQL语句。- 您可以在任何地方找到条件编译指令 - 常规语句中有空格。SUBTYPE my_real IS $ IF DBMS_DB_VERSION.VER_LE_9 $ THEN NUMBER $ ELSE BINARY_DOUBLE $ END; t CONSTANT my_real：= $ IF DBMS_DB_VERSION.VER_LE_9 $ THEN 0.499975 $ ELSE 0.499975d $ END; t2 CONSTANT my_real：= $ if if DBMS_DB_VERSION.VER_LE_9 $ THEN 2.0 $ ELSE 2.0d $ END; x CONSTANT my_real：= $ IF DBMS_DB_VERSION.VER_LE_9 $ THEN 1.0 $ ELSE 1.0d $ END; y CONSTANT my_real：= $ IF DBMS_DB_VERSION.VER_LE_9 $ THEN 1.0 $ ELSE 1.0d $ END; z MY_REAL;程序P（x in my_real，y in my_real，z OUT NOCOPY my_real）IS x1 my_real; y1 my_real; BEGIN x1：= x; y1：= y; x1：= t *（x1 + y1）; y1：= t *（x1 + y1）; z：=（x1 + y1）/ t2;结束P; BEGIN P（x，y，z）; DBMS_OUTPUT.PUT_LINE（'z ='|| z）;结束磨刀石; /
