使用DBCA时的群集检测和节点选择

当您启动DBCA时，如果它从中央Oracle Inventory检测到Oracle主机已启用Oracle RAC，它将自动显示Oracle RAC选项。

如果DBCA未自动显示Oracle RAC选项，则DBCA无法检测群集上是否安装了Oracle主目录。在这种情况下，请检查OUI清单是否正确位于目录/etc/oraInst.loc ，并且oraInventory文件未损坏。此外，使用以下CVU命令语法执行集群件诊断：

/u01/app/19.0.0/grid/bin/cluvfy/cluvfy.sh stage -post crsinst -n nodelist

例如，使用mountpoint /u01/app/19.0.0/grid和节点node1和node2 ，运行以下命令：

$ /u01/app/19.0.0/grid/bin/cluvfy stage -post crsinst -n node1，node2

请注意，使用DBCA时，如果群集安装中的节点未显示在“节点选择”页面上，请运行lsinventory命令以执行库存诊断。还可以使用CVU执行集群件诊断。

在“管理选项”页面上，您可以使用Oracle Enterprise Manager Database Express或Oracle Enterprise Manager Cloud Control来管理数据库。对于Oracle RAC数据库，Oracle Enterprise Manager Database Express配置为使用单客户端访问名称（SCAN）连接到群集。
