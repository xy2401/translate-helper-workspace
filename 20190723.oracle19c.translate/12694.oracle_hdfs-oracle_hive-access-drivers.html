17.1指定属性的语法规则

使用SQL CREATE TABLE ACCESS PARAMETERS子句和配置文件中的关键字 - 值对设置属性。

语法必须遵守以下规则：

  • 每个关键字 - 值对的格式是关键字 ，冒号或等号以及值 。以下是有效的关键字 - 值对：

    keyword = value关键字：值

    值是从分隔符后面的第一个非空白字符到行尾的所有内容。分隔符和值之间的空格将被忽略。保留值的尾随空格。

  • 属性定义可以在一行或多行上。

  • 行终止符是换行符，回车符或回车符，后跟换行符。

  • 当属性定义跨越多行时，则在行终止符前面加上反斜杠（转义字符），最后一行除外。在此示例中， Keyword1属性的值为Value part 1 Value part 2 Value part 3 。 

    关键字1 =值部分1 \值部分2 \值部分3

  • 您可以通过剥离前导空格的每个物理行并连接这些行来创建逻辑行 。解析器从逻辑行中提取属性名称和值。

  • 您可以通过在带有反斜杠（转义字符）的字符前面嵌入特殊字符，从而在属性名称或属性值中嵌入特殊字符，以指示替换。表17-1介绍了特殊字符。

17.4.1 com.oracle.bigdata.colmap

将源数据中的列映射到Oracle外部表中的列。您可以定义一对或多对列映射。当源字段名称超过Oracle列名称的最大长度时，或者要在外部表中使用不同的列名称时，请使用此属性。

默认值

外部表中与Hive列同名的列

句法

具有关键字 - 值对的JSON文档如下图所示：

colmap：


插图描述colmap.eps

colmap_entry：


插图描述colmap_entry.eps

语义

“col”： 名字

"col" ：关键字必须小写并用引号括起来。

name ：Oracle外部表中列的名称。它区分大小写，必须用引号括起来。

“字段”： 姓名

"field" ：关键字必须小写并用引号括起来。

name ：数据源中字段的名称。它不区分大小写，但必须用引号括起来。请参见指定属性的语法规则 。

例子

com.oracle.bigdata.colmap：[{“col”：“KOL1”，“field”：“PROJECT_NAME”}，{“col”：“KOL2”，“field”：“wsdl_name”}，{“col”： “KOL3”，“field”：“method”}]

语义

自动

17.4.9 com.oracle.bigdata.rowformat

提供访问驱动程序从文件中的记录中提取字段所需的信息。

重要：

com.oracle.bigdata.rowformat与使用“type ORACLE_LOADER”的传统外部表的访问参数语法无关。在两个子句中都有关键字，如FIELDS ， TERMINATED和其他关键字，但命名的共性是巧合的，并不意味着常见的功能。com.oracle.bigdata.rowformat访问参数无需更改即传递给默认的Hive serde。从行中提取列的Hive serde是故意限制的。复杂案件由专门的serdes处理。

默认值

DELIMITED

句法

具有关键字 - 值对的JSON文档如下图所示。

rowformat：


插图描述rowformat.eps

语义

DELIMITED

描述用于分隔记录中字段的字符：

  • FIELDS TERMINATED BY ：分隔记录中每个字段的字符。当分隔字符出现在字段值中时，可选的ESCAPED BY字符位于分隔字符之前。

  • COLLECTION ITEMS TERMINATED BY ：标记数组元素结尾的字符。当列是集合或嵌套记录时使用。在这种情况下，结果值将是JSON数组。

  • MAP KEYS TERMINATED BY ：标记MAP字段中条目结尾的字符。当列是集合或嵌套记录时使用。结果值是JSON对象。

17.4.10 com.oracle.bigdata.tablename

Hive参数com.oracle.bigdata.tablename标识包含源数据的Hive表。

默认值

DEFAULT.external_table_name

句法

[hive_database_name。]表名

语义

hive_database_name和table_name的最大长度为128个UTF-8字符（512字节）。

hive_database_name ：源数据所在的Hive数据库。DEFAULT是初始Hive数据库的名称。

table_name ：包含数据的Hive表。如果省略table_name ，则ORACLE_HIVE搜索与外部表同名的Hive表。表名不区分大小写。

例

此设置指示源数据位于Hive ORDER_DB数据库中名为ORDER_SUMMARY的表中：

com.oracle.bigdata.tablename ORDER_DB.ORDER_SUMMARY
