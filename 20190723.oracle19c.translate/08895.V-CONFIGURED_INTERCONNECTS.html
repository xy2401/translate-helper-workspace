7.142 V $ CONFIGURED_INTERCONNECTS

V$CONFIGURED_INTERCONNECTS显示Oracle知道的所有互连。此视图试图回答Oracle在何处找到有关特定互连的信息的问题。

柱 数据类型 描述

名称

VARCHAR2（15）

互连的名称（例如eth0）

IP地址

VARCHAR2（64）

互连的IP地址

IS_PUBLIC

VARCHAR2（3）

如果值为YES ，则界面为公众所知。

如果值为NO ，则认为接口是私有的。请注意，如果还指定了CLUSTER_INTERCONNECTS初始化参数，则预期互连是私有的。Oracle期望集群流量仅在专用互连上运行。

如果该值为空，则不知道接口是公共接口还是私有接口。

Oracle建议您在Oracle Cluster Registry（OCR）中设置Oracle Real Application Clusters（Oracle RAC）通信的接口。

资源

VARCHAR2（31）

指示从以下位置拾取此接口的位置：

  • Oracle Cluster Registry - 在OCR中配置了接口，Oracle数据库在OCR中找到了该接口

  • 依赖于操作系统的软件 - Oracle数据库会自动检测到这一点

  • CLUSTER_INTERCONNECTS参数 - 已设置此初始化参数

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
