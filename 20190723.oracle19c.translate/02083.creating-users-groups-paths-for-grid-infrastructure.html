为Oracle Grid Infrastructure创建组，用户和路径

以root身份登录，并使用以下说明查找或创建Oracle Inventory组，并为Oracle Grid Infrastructure创建软件所有者，并为Oracle主目录创建目录。

Oracle软件安装需要安装所有者，Oracle Inventory组，它是所有Oracle安装所有者的主要组，并且至少需要一个组指定为系统特权组。与系统管理员一起查看组和用户选项。如果您具有系统管理权限，请查看本节中的主题并根据需要配置操作系统组和用户。

  • 确定是否存在Oracle Inventory和Oracle Inventory Group
    确定您是否具有现有Oracle中央库存，并确保为所有Oracle软件安装使用相同的Oracle Inventory。此外，请确保您打算用于安装的所有Oracle软件用户都有权写入此目录。
  • 如果Oracle库存不存在，则创建Oracle库存组
    如果oraInst.loc文件不存在，则创建Oracle Inventory组。
  • 关于Oracle安装所有者帐户
    根据要用于安装的组和用户管理计划，为安装选择或创建Oracle安装所有者。
  • Oracle软件安装所有者的限制
    查看为拥有Oracle软件而创建的用户的以下限制。
  • 识别Oracle软件所有者用户帐户
    首次在系统上安装Oracle软件时，必须至少创建一个软件所有者用户帐户。使用现有的Oracle软件用户帐户，或为您的安装创建Oracle软件所有者用户帐户。
  • 关于网格用户的Oracle Base Directory
    查看有关在每个群集节点上创建Oracle基本目录的信息。
  • 关于Oracle Grid Infrastructure软件的Oracle主目录
    查看有关在每个群集节点上创建Oracle主目录位置的信息。
  • 关于创建Oracle Home和Oracle Base Directory
    在每个群集节点上创建Grid home和Oracle基本主目录。
