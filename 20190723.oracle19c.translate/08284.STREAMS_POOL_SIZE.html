1.332 STREAMS_POOL_SIZE

STREAMS_POOL_SIZE值有助于确定Streams池的大小。

属性 描述

参数类型

大整数

句法

STREAMS_POOL_SIZE = integer [K | M | G]

默认值

0

可修改

改变系统

可在PDB中修改

没有

价值范围

最小值： 0 （大于零的值向上舍入到最接近的粒度）

最大值：取决于操作系统

基本

没有

当SGA_TARGET初始化参数设置为非零值时，Oracle的自动共享内存管理功能可管理Streams池的大小。如果STREAMS_POOL_SIZE初始化参数也设置为非零值，则自动共享内存管理将此值用作Streams池的最小值。

如果SGA_TARGET设置为非零值STREAMS_POOL_SIZE指定STREAMS_POOL_SIZE或设置为空值，则自动共享内存管理将使用0（零）字节作为Streams池的最小值。

如果STREAMS_POOL_SIZE初始化参数设置为非零值，并且SGA_TARGET参数设置为0（零），则Streams池大小是STREAMS_POOL_SIZE参数指定的值（以字节为单位）。

如果STREAMS_POOL_SIZE和SGA_TARGET初始化参数都设置为0（零），则默认情况下，在数据库中第一次请求Streams池内存时，将从内存中传输等于共享池10％的内存量。缓冲区缓存到Streams池。使用Streams池的产品和功能包括Oracle GoldenGate，XStream，Oracle Advanced Queuing和Oracle Data Pump。

Streams池是共享资源，进程可以从Streams池使用的内存量由应用程序确定。可以为Oracle GoldenGate或XStream控制捕获或应用参数MAX_SGA_SIZE 。对于Oracle Advanced Queuing，请使用dbms_aqadm软件包中的过程来控制所需的Streams池的数量。
