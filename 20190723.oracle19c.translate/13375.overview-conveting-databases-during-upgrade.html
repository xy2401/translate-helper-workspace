升级期间转换数据库的概述

在Oracle数据库升级期间，可以使用四种方法转换数据库。

下表列出了可用于转换升级的方法，包括对可用性问题的引用。它还提供了描述如何执行每种升级方法的文档的参考。

表1-3升级过程中转换数据库的方法

方法 描述 参考

Oracle Data Guard瞬态待机（物理备用）数据库

使用现有物理备用数据库通过将其临时转换为逻辑备用数据库，然后将其转换回物理备用数据库来执行数据库升级。

Oracle数据库升级指南 ，“关于使用备用数据库升级”
Oracle GoldenGate同步生产和备用数据库，实现零停机升级 使用Oracle GoldenGate进行软件升级和Oracle数据库数据迁移过程，以执行同步方法以在升级期间维护可用性：

  • 使用RMAN还原和升级以使用现有备份设置运行早期版本软件的备用数据库

  • 将备用数据库升级到新的Oracle数据库版本

  • 移动整个数据库并使用以下工具将备用数据库与生产数据库同步：

    • Oracle Data Pump

    • 可移动表空间（TTS）

    • CREATE TABLE AS SELECT（CTIS）创建新表并使用指定查询中的行填充它们。

    • INSERT AS SELECT（IAS）创建非分区表

  • 使用数据加载/卸载将数据加载到新数据库版本中，并从旧数据库版本卸载数据

Oracle企业管理器云控制

与Oracle数据库12c的开始，Oracle提供了执行数据库升级，云控制支持。此选项要求您购买企业管理器生命周期管理包。

Oracle企业管理器云控制中的联机帮助

注意：

在Oracle数据库升级之前，Oracle Grid Infrastructure（Oracle Clusterware和Oracle自动存储管理）的升级将单独执行。在升级Oracle数据库安装之前，必须完成Oracle Grid Infrastructure升级。随Oracle Database一起安装的其他功能可能还有其他升级要求。
