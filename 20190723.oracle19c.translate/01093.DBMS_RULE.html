143.3.2评估程序

此过程评估指定规则集中的规则，该规则集使用为指定事件指定的评估上下文。

此过程已超载。true_rules和maybe_rules参数与true_rules_iterator和maybe_rules_iterator参数互斥。此外，具有true_rules和maybe_rules参数的过程包括stop_on_first_hit参数，但另一个过程不包括。

句法

DBMS_RULE.EVALUATE（rule_set_name IN VARCHAR2，evaluation_context IN VARCHAR2，event_context IN SYS.RE $ NV_LIST DEFAULT NULL，table_values IN SYS.RE $ TABLE_VALUE_LIST DEFAULT NULL，column_values IN SYS.RE $ COLUMN_VALUE_LIST DEFAULT NULL，variable_values IN SYS.RE $ VARIABLE_VALUE_LIST DEFAULT NULL，attribute_values IN SYS.RE $ ATTRIBUTE_VALUE_LIST DEFAULT NULL，
skip_rules IN SYS.RE $ RULE_NAME_LIST DEFAULT NULL，
掺入数量，
result_cache IN BOOLEAN DEFAULT FALSE，stop_on_first_hit IN BOOLEAN DEFAULT FALSE，simple_rules_only IN BOOLEAN DEFAULT FALSE，true_rules OUT SYS.RE $ RULE_HIT_LIST，maybe_rules OUT SYS.RE $ RULE_HIT_LIST）; DBMS_RULE.EVALUATE（rule_set_name IN VARCHAR2，evaluation_context IN VARCHAR2，event_context IN SYS.RE $ NV_LIST DEFAULT NULL，table_values IN SYS.RE $ TABLE_VALUE_LIST DEFAULT NULL，column_values IN SYS.RE $ COLUMN_VALUE_LIST DEFAULT NULL，variable_values IN SYS.RE $ VARIABLE_VALUE_LIST DEFAULT NULL，attribute_values IN SYS.RE $ ATTRIBUTE_VALUE_LIST DEFAULT NULL，
skip_rules IN SYS.RE $ RULE_NAME_LIST DEFAULT NULL，
掺入数量，
simple_rules_only IN BOOLEAN DEFAULT FALSE，true_rules_iterator OUT BINARY_INTEGER，maybe_rules_iterator OUT BINARY_INTEGER）;

参数

表143-3评估过程参数

参数 描述

rule_set_name

以[ schema_name .] rule_set_name形式设置的规则的名称。例如，要评估hr模式中名为hr_rules的规则集中的所有规则，请为此参数输入hr.hr_rules 。如果未指定架构，则使用当前用户的架构。

evaluation_context

[ schema_name .] evaluation_context_name形式的评估上下文名称。如果未指定架构，则使用当前用户的名称。

仅评估使用指定评估上下文的规则。

event_context

标识导致评估的事件的名称 - 值对列表

table_values

使用在创建评估上下文时指定的表别名包含表行的数据。列表中的每个表别名必须是唯一的。

column_values

包含表行的部分数据。它不能包含表的列值，其值已在table_values指定。

variable_values

包含变量数据的列表。

显式变量值的唯一获取方法是在此列表中指定其值。

如果未在列表中指定隐式变量值，则调用用于获取隐式变量值的函数。如果在列表中指定了隐式变量值，则使用此值并且不调用该函数。

ATTRIBUTE_VALUES

包含变量的部分数据。它必须不包含其值已在variable_values指定的variable_values属性值。

stop_on_first_hit

如果为TRUE ，则规则引擎会在找到TRUE规则后立即停止评估。

如果TRUE ，也没有TRUE规则，那么规则引擎，只要它认定，可以评估的规则将停止评估TRUE赋予了更多的数据。

如果为FALSE ，则规则引擎即使在找到TRUE规则后也会继续评估规则。

simple_rules_only

如果为TRUE ，则仅考虑那些足够简单以便快速评估（不发布SQL）的规则进行评估。

如果为FALSE ，则评估所有规则。

true_rules

将EVALUATE过程的输出接收到RE$RULE_HIT_LIST类型的RE$RULE_HIT_LIST 。

如果没有规则评估为TRUE ，则true_rules为空。

如果至少有一个规则的计算结果为TRUE且stop_on_first_hit为TRUE ，则true_rules包含一个计算结果为TRUE规则。

如果stop_on_first_hit为FALSE ，则true_rules包含评估为TRUE所有规则。

maybe_rules

如果可以完全评估所有规则，而不需要任何其他数据，则maybe_rules为空。

如果stop_on_first_hit为TRUE ，那么如果在给定更多数据的情况下至少有一个规则可以评估为TRUE ，并且没有规则评估为TRUE ，那么maybe_rules包含一个可以评估为TRUE规则。

如果stop_on_first_hit为FALSE ，那么maybe_rules包含在给定更多数据的情况下可以评估为TRUE所有规则。

true_rules_iterator

包含用于访问TRUE规则的迭代器

maybe_rules_iterator

包含用于访问规则的迭代器，该规则在给定附加数据或发出SQL的能力下可能为TRUE

skip_rules

要在此评估中跳过的规则列表。

DOP

并行度

RESULT_CACHE

如果为TRUE ，则将创建结果缓存。如果使用true_rules_iterator或maybe_rules_iterator调用true_rules_iterator maybe_rules_iterator ，则不启用result_cache 。

使用说明

注意：

使用不同于指定的评估上下文的规则集中的规则不被考虑用于评估。

使用为table_values ， column_values ， variable_values和attribute_values指定的数据评估规则集中的规则。这些值必须引用指定评估上下文中的表和变量。否则，会引发错误。

调用者可以使用stop_on_first_hit指定，如果找到第一个TRUE规则或第一个MAYBE规则（如果没有TRUE规则），则必须停止评估。

调用者还可以使用simple_rules_only指定是否只应考虑足够简单以便快速评估的规则（这意味着没有SQL）进行评估。这使评估更快，但导致无法在没有SQL的情况下评估的规则作为MAYBE规则返回。

支持部分评估。只能使用某些表，列，变量或属性的数据调用EVALUATE过程。在这种情况下，由于缺少数据而无法评估的规则将作为MAYBE规则返回，除非根据规则中的一个或多个简单表达式的值确定它们为TRUE或FALSE 。例如，给定变量"x"属性"ab"的值为1 ，具有以下规则条件的规则可以返回为TRUE ，而不包含表"tab"的值：

（：xab = 1）或（tab.c> 10）

评估结果如下：

  • TRUE规则，它是根据给定数据计算为TRUE的规则列表。这些规则在OUT参数true_rules中返回，该参数返回评估为TRUE所有规则，或者在OUT参数true_rules_iterator中返回，该参数返回每次评估为TRUE每个规则。

  • MAYBE规则，这是由于以下原因之一而无法评估的规则列表：

    • 该规则指的是不可用的数据。例如，指定了变量属性"xab" ，但没有为变量"x" ，属性"a"或属性"ab" 。

    • 该规则不够简单，无法快速评估（没有SQL）， simple_rules_only被指定为TRUE ，或者部分数据可用。

    也许规则在OUT参数maybe_rules中返回，它返回所有评估为MAYBE的规则，或者返回OUT参数maybe_rules_iterator ，它返回每个评估为MAYBE规则。

调用者可以指定过程是否返回所有的计算结果的规则TRUE和MAYBE的事件或为评估规则迭代器TRUE和MAYBE 。真正的规则迭代器使客户端能够一次获取评估为TRUE每个规则，并且可能的规则迭代器使客户端能够获取每次评估为MAYBE每个规则。

如果使用迭代器，则使用DBMS_RULE包中的GET_NEXT_HIT函数来检索从迭代器求值为TRUE或MAYBE的下一个规则。Oracle建议您关闭迭代器，如果不再需要它来释放迭代器使用的资源（如内存）。可以通过以下方式关闭迭代器：

  • DBMS_RULE包中的CLOSE_ITERATOR过程在指定了迭代器的情况下运行。

  • 迭代器返回NULL因为没有更多规则评估为TRUE或MAYBE 。

  • 迭代器运行的会话结束。

要运行DBMS_RULE.EVALUATE过程，用户必须满足以下至少一个要求：

  • 对规则集具有EXECUTE_ON_RULE_SET权限

  • 具有EXECUTE _ ANY _ RULE _ SET系统权限

  • 成为规则集所有者

    注意：

    规则引擎不会调用任何操作。每个返回的规则都可以返回一个操作上下文，但规则引擎的客户端必须调用任何必要的操作。

143.3.7 GET_NEXT_HIT函数

此函数返回从真正的规则迭代器计算为TRUE下一个规则，或从可能的规则迭代器返回评估为MAYBE下一个规则。如果没有更多规则评估为TRUE或MAYBE ，则该函数返回NULL 。

句法

DBMS_RULE.GET_NEXT_HIT（迭代器IN BINARY_INTEGER）返回SYS.RE $ RULE_HIT;

参数

表143-8 GET_NEXT_HIT函数参数

参数 描述

迭代器

从中检索评估为TRUE或MAYBE的规则的迭代器

使用说明

此过程需要一个打开的迭代器，该迭代器由之前在同一会话中调用DBMS_RULE.EVALUATE返回。运行此过程的用户不需要对要评估的规则集具有任何权限。

当迭代器返回NULL ，它会自动关闭。如果不再需要打开的迭代器，则使用DBMS_RULE包中的CLOSE_ITERATOR过程将其关闭。

注意：

如果在调用返回迭代器的DBMS_RULE.EVALUATE过程之后修改了要评估的规则集，则此函数会引发错误。对规则集的修改包括向规则集添加规则，更改规则集中的现有规则，从规则集中删除规则以及删除规则集。
