预编译器的位置因系统而异。通常，系统管理员或DBA定义环境变量，逻辑或别名，或使用其他特定于操作系统的方法来使预编译器可执行文件可访问。

INAME选项指定要预编译的源文件。例如，Pro * COBOL命令

procob INAME = test

预编译当前目录中的test.pco文件，因为预编译器假定文件扩展名为.pco 。你不需要使用指定INAME时的文件扩展名，除非扩展名是非标准的。

输入和输出文件名不需要附带各自的选项名称INAME和ONAME。如果未指定选项名称，则预编译器假定命令行上指定的第一个文件名是输入文件名，第二个文件名是输出文件名。

因此，Pro * FORTRAN命令

profor MODE = ANSI myfile.pfo DBMS = V7 myfile.f

相当于

profor MODE = ANSI INAME = myfile.pfo DBMS = V7 ONAME = myfile.f

未命名特定操作系统对象（如文件名）的选项名称和选项值不区分大小写。在本指南的示例中，选项名称以大写字母书写，选项值通常以小写字母表示。文件名（包括预编译器可执行文件本身的名称）始终遵循执行它的操作系统使用的大小写约定。

6.2预编译期间发生了什么？

在预编译期间，Oracle预编译器生成主机语言代码，用于替换宿主程序中嵌入的SQL语句。生成的代码包括数据结构，其中包含每个主机变量的数据类型，长度和地址，以及Oracle运行时库SQLLIB所需的其他信息。生成的代码还包含对执行嵌入式SQL操作的SQLLIB例程的调用。

生成的代码还包括对执行嵌入式SQL操作的SQLLIB例程的调用。请注意，预编译器不会生成对Oracle调用接口（OCI）例程的调用。

预编译器不会生成对Oracle调用接口（OCI）例程的调用。

预编译器可以发出警告和错误消息。这些消息具有前缀PCC-，并在Oracle数据库错误消息中进行了描述。

6.3预编译器选项

预编译时可以使用许多有用的选项。它们允许您控制资源的使用方式，报告错误的方式，输入和输出的格式，以及如何管理游标。要指定预编译器选项，请使用以下语法：

<OPTION_NAME> = <值>

选项的值是字符串文字，表示文本或数字值。例如，对于选项

...INAME = my_test

该值是一个字符串文字，指定文件名，但是对于该选项

...MAXOPENCURSORS = 20

值是数字。

某些选项采用布尔值，您可以使用字符串YES或NO，TRUE或FALSE或整数文字1或0来表示布尔值。例如，选项

...SELECT_ERROR = YES

相当于

...SELECT_ERROR = TRUE

要么

...SELECT_ERROR = 1

选项值始终与选项名称以等号分隔，在等号周围不留空格，因为空格分隔各个选项。例如，您可以在命令行上指定选项AUTO_CONNECT ，如下所示：

...AUTO_CONNECT = YES

如果缩写是明确的，您可以缩写选项的名称。例如，您不能使用缩写MAX，因为它可能代表MAXLITERAL或MAXOPENCURSORS。

可在线获得有关预编译器选项的便捷参考。要查看在线显示，请在操作系统提示符下输入不带参数的预编译器命令。显示屏提供每个选项的名称，语法，默认值和用途。标有星号（*）的选项可以在内联和命令行中指定。

6.3.1默认值

许多选项都有默认值，由以下因素确定：

  • 内置于预编译器的值

  • 系统配置文件中设置的值

  • 用户配置文件中设置的值

  • 内联规范中设置的值

例如，选项MAXOPENCURSORS指定缓存的打开游标的最大数量。此选项的内置预编译器默认值为10。但是，如果在系统配置文件中指定了MAXOPENCURSORS = 32，则默认值为32。用户配置文件可以将其设置为另一个值，然后覆盖系统配置值。

然后，如果在命令行上设置了此选项，则新的命令行值优先。最后，内联规范优先于所有先前的默认值。有关更多信息，请参阅“ 配置文件 ”。

6.3.2确定当前值

您可以通过在命令行上使用问号以交互方式确定一个或多个选项的当前值。例如，如果您发出Pro * COBOL命令

procob？

终端上会显示完整的选项集以及当前值。在这种情况下，值是预编译器中内置的值，由系统配置文件中的任何值覆盖。但是，如果您发出以下命令

procob CONFIG = my_config_file.cfg？

并且当前目录中有一个名为my_config_file.cfg的文件， my_config_file.cfg文件中的选项与其他默认值一起列出。用户配置文件中的值提供缺失值，它们取代预编译器中内置的值或系统配置文件中指定的值。

您还可以通过简单地指定选项名称后跟“=？”来确定单个选项的当前值。“如同

procob MAXOPENCURSORS =？

注意：

：对于一些操作系统，“？“可能需要以”转义“字符开头，例如反斜杠（\）。例如，你可能需要使用“procob”？而不是“procob？”而不是“procob？”。“列出Pro * COBOL选项设置。

6.3.3区分大小写

通常，您可以使用大写或小写作为命令行选项名称和值。但是，如果您的操作系统区分大小写（如UNIX），则必须使用大写和小写字母的正确组合指定文件名值，包括预编译器可执行文件的名称。

配置文件是包含预编译器选项的文本文件。文件中的每个记录（行）都包含一个选项及其关联的值。例如，配置文件可能包含这些行

FIPS = YES MODE = ANSI

设置FIPS和MODE选项的默认值。

6.4输入选项

可以在命令行上输入所有预编译器选项，也可以从配置文件中输入（ CONFIG除外）。许多选项也可以内联输入。在给定的运行期间，预编译器可以接受来自所有三个源的选项。

6.4.1在命令行上

您可以使用以下命令在命令行上输入预编译器选项句法：

... [option_name = value] [option_name = value] ...

用一个或多个空格分隔每个选项。例如，您可以输入以下选项：

...错误=没有LTYPE =短

6.4.2内联

您可以使用以下语法编写EXEC ORACLE语句来输入内联选项 ：

EXEC ORACLE OPTION（option_name = value）;

例如，您可以编写以下语句：

EXEC ORACLE选项（RELEASE_CURSOR = YES）;

输入内联的选项将覆盖在命令行中输入的相同选项。

6.4.3优点

EXEC ORACLE功能对于在预编译期间更改选项值特别有用。例如，您可能希望HOLD_CURSOR语句地更改HOLD_CURSOR和RELEASE_CURSOR值。 性能调优向您展示了如何使用内联选项来优化运行时性能。

如果操作系统限制了可以在命令行中输入的字符数，则可以在内部指定选项，并且可以在配置文件中存储内联选项，这将在下一节中讨论。

6.4.4 EXEC ORACLE的范围

EXEC ORACLE语句保持有效，直到文本上被另一个指定相同选项的EXEC ORACLE语句取代。在以下示例中， HOLD_CURSOR=NO保持有效，直到被HOLD_CURSOR=YES:取代HOLD_CURSOR=YES:

6.4.5从配置文件

Oracle预编译器可以使用包含预设命令行选项的配置文件。默认情况下，使用称为系统配置文件的文本文件。但是，你可以在命令行上指定任何几个备用文件，称为用户配置文件 。

6.4.6优点

配置文件提供了几个优点。系统配置文件允许您标准化所有项目的一组选项。用户配置文件允许您为每个项目自定义一组选项。使用配置文件，您无需在命令行中输入长字符串选项。此外，如果您的系统限制命令行的长度，配置文件允许您指定比命令行可以容纳的更多选项。

6.4.7关于使用配置文件

配置文件中的每个记录（行）都包含一个命令行选项。例如，配置文件可能包含以下行，这些行设置FIPS ， MODE和SQLCHECK选项的默认值：

FIPS = YES MODE = ANSI SQLCHECK = SEMANTICS

每个Oracle预编译器都可以拥有自己的系统配置文件。文件的名称和位置是特定于语言和系统的。如果找不到该文件，则会收到警告，但预编译器会继续处理。

给定语言只有一个系统配置文件，但您可以创建任意数量的用户配置文件。您使用新的命令行选项 CONFIG指定特定用户配置文件的名称和位置，如下所示：

...CONFIG = <文件名>

您无法嵌套配置文件。因此，您无法在配置文件中指定CONFIG选项。此外，您不能指定内联CONFIG 。

6.4.8关于设置选项值

许多预编译器运行时选项都有内置的默认值，可以在配置文件或命令行中重置。命令行设置会覆盖用户配置文件设置，这会覆盖系统配置文件设置。

6.5选择范围

一个预编译单元是包含主机语言代码和一个或多个嵌入式SQL语句的文件。为给定的预编译单元指定的选项仅影响该单元;它们对其他单位没有影响。

例如，如果为单元A而不是单元B指定HOLD_CURSOR=YES和RELEASE_CURSOR=YES ，则单元A中的SQL语句将使用这些HOLD_CURSOR和RELEASE_CURSOR值运行，但单元B中的SQL语句将使用默认值运行。但是，连接到Oracle时生效的MAXOPENCURSORS设置在该连接的生命周期内保持有效。

内联选项的范围是位置，而不是逻辑。也就是说，内联选项会影响源文件中跟随它的SQL语句，而不会影响程序逻辑流。除非您重新指定选项，否则选项设置将一直有效，直到文件结束。

6.7.2 ASSUME_SQLCODE

目的

指示Oracle预编译器假定声明了SQLCODE，无论它是在Declare Section中声明还是在正确类型中声明。ASSUME_SQLCODE = YES导致版本1.6及更高版本的Oracle预编译器在这方面的行为与版本1.5类似。

句法

ASSUME_SQLCODE={YES|NO }

默认

没有

使用说明

无法输入内联。

当ASSUME_SQLCODE=NO ，当且仅当满足以下条件中的至少一个时， SQLCODE ASSUME_SQLCODE=NO识别为状态变量：

  • 它在Declare Section中声明，具有完全正确的数据类型。

  • 预编译器找不到其他状态变量。

如果预编译器在声明段找到SQLSTATE声明（ 完全正确类型的课程），或者找到一个INCLUDE SQLCA的，它不会擅自SQLCODE声明。

当ASSUME_SQLCODE=YES ，并且当SQLSTATE和SQLCA（仅限Pro * FORTRAN）被声明为状态变量时，预编译器假定SQLCODE被声明是在声明部分中声明还是在正确类型中声明。这导致版本1.6.7及更高版本在这方面的行为类似于版本1.5。

6.7.6 CLOSE_ON_COMMIT

目的

指定是否在提交语句上关闭游标。

句法

CLOSE_ON_COMMIT = {是|没有}

默认

没有

使用说明

只能在命令行或配置文件中使用。

如果MODE的指定级别高于CLOSE_ON_COMMIT，则MODE优先。例如，默认值为MODE = ORACLE，CLOSE_ON_COMMIT = NO。如果用户在命令行上指定MODE = ANSI，则任何游标将在提交时关闭。

当CLOSE_ON_COMMIT = NO（当MODE = ORACLE）时，发出COMMIT或ROLLBACK将仅关闭使用FOR UPDATE子句声明的游标或在CURRENT OF子句中引用的游标。其他不受COMMIT或ROLLBACK语句影响的游标，如果已经打开，则保持打开状态。但是，当CLOSE_ON_COMMIT = YES（当MODE = ANSI时），发出COMMIT或ROLLBACK将关闭所有游标。

目的

此属性确定当池中的所有其他物理连接都忙时，应用程序是否必须重复尝试进行物理连接，并且物理连接的总数已达到其最大值。

句法

CNOWAIT =范围是1到65535。

默认

0表示未设置。

使用说明

如果物理连接不可用且无法再打开物理连接，则在设置此属性时会引发错误。否则，呼叫将等待，直到获得另一个连接。默认情况下，不设置CNOWAIT，因此线程将等待，直到它可以获取空闲连接，而不是返回错误。

6.7.10代码

目的

指定Pro * C / C ++预编译器生成的C函数原型的格式。（ 函数原型声明了一个函数及其参数的数据类型。）预编译器为SQL库例程生成函数原型，以便您的C编译器可以解析外部引用。CODE选项可让您控制原型设计。

句法

CODE = {ANSI_C | KR_C | CPP}

默认

KR_C

使用说明

可以在命令行中输入，但不能内联。

ANSI C标准X3.159-1989提供功能原型设计。当CODE = ANSI_C时，Pro * C / C ++生成完整的函数原型，符合ANSI C标准。一个例子如下：

extern void sqlora（long *，void *）;

预编译器还可以生成其他ANSI批准的构造，例如const类型限定符。

当CODE = KR_C（默认值）时，预编译器会注释掉生成的函数原型的参数列表，如下所示：

extern void sqlora（/ * _ long *，void * _ * /）;

如果您的C编译器不符合X3.159标准，请指定CODE = KR_C。

当CODE = CPP时，预编译器生成C ++兼容代码。

6.7.11 COMMON_NAME

目的

仅对于Pro * FORTRAN， COMMON_NAME选项指定用于命名内部FORTRAN COMMON块的前缀。您的主机程序不直接访问COMMON块。但是，它们允许同一预编译单元中的两个或多个程序单元包含SQL语句。

句法

COMMON_NAME= blockname

默认

输入文件名称中的前五个字符

使用说明

Pro * FORTRAN预编译器使用称为块数据子程序的特殊程序文件为输入文件中的所有SQL变量建立COMMON块。块数据子程序定义了两个COMMON块 - 一个用于CHARACTER变量，另一个用于非CHARACTER变量 - 并使用DATA语句初始化变量。

块数据子程序的格式如下：

BLOCK DATA <subprogram_name>变量声明COMMON语句DATA语句END

您的主机程序不直接访问COMMON块。但是，它们允许同一预编译文件中的两个或多个程序文件包含SQL语句。

要命名COMMON块，预编译器使用输入文件的名称和后缀C，D和I.最多使用文件名的前五个字符。例如，如果输入文件的名称是ACCTSPAY ，则预编译器将COMMON块命名为ACCTSC ， ACCTSD和ACCTSI 。

但是，预编译器可以为不同输出文件中定义的COMMON块提供相同的名称，如下图所示：

ACCTSPAY.PFO ===> ACCTSPAY中的ACCTSC，ACCTSD，ACCTSI.FOR ACCTSREC.PFO ===> ACCTSREC.FOR中的ACCTSC，ACCTSD，ACCTSI

如果要将ACCTSPAY和ACCTSREC链接到可执行程序，链接器将看到三个而不是六个COMMON块。

要解决此问题，可以重命名输入文件，也可以通过指定COMMON_NAME内联或命令行来覆盖默认的COMMON块名称，如下所示：

COMMON_NAME = <block_name>

其中block_name是合法的COMMON块名称。例如，如果指定COMMON_NAME=PAY ，则预编译器将其COMMON块PAYC和PAYI 。最多使用block_name中的前五个字符。

例如，如果指定COMMON_NAME=PAY ，则预编译器将其COMMON块PAYC和PAYI 。最多使用block_name中的前5个字符。

如果指定COMMON_NAME内联，则其EXEC ORACLE OPTION语句必须位于FORTRAN PROGRAM， SUBROUTINE或FUNCTION语句之前。

您可能要重写默认COMMON如果他们用自己的用户自定义的冲突块名COMMON块名称。但是，首选做法是重命名用户定义的COMMON块。

COMMON_NAME如果指定MULTISUBPROG是没有必要的。

6.7.32 HOLD_CURSOR

目的

指定如何在游标缓存中处理SQL语句和PL / SQL块的游标。

句法

HOLD_CURSOR = {YES | NO}

默认

没有

使用说明

您可以使用HOLD_CURSOR来提高程序的性能。有关更多信息，请参阅性能调整

执行SQL数据操作语句时，其关联的游标将链接到游标缓存中的条目。游标缓存条目又链接到Oracle私有SQL区域，该区域存储处理语句所需的信息。HOLD_CURSOR控制光标和光标缓存之间的链接发生了什么。

对于使用隐式游标的内联使用， HOLD_CURSOR在执行SQL语句之前设置HOLD_CURSOR 。对于使用显式游标的内联使用， HOLD_CURSOR在打开游标之前设置HOLD_CURSOR 。

请注意， RELEASE_CURSOR=YES会覆盖HOLD_CURSOR=YES ，而HOLD_CURSOR = NO会覆盖RELEASE_CURSOR=NO 。有关这两个选项如何交互的信息，请参阅表C-1 。

6.8条件预编译

条件预编译包括（或排除）主机程序中基于特定条件的代码段。例如，您可能希望在UNIX下进行预编译时包含一段代码，而在VMS下进行预编译时则需要包含另一部分代码。条件预编译允许您编写可在不同环境中运行的程序。

条件的条件部分由定义环境和要采取的操作的语句标记。您可以在这些部分中编写主机语言语句和EXEC SQL语句。以下语句允许您对预编译执行条件控制：

 EXEC ORACLE DEFINE符号; - 定义一个符号 EXEC ORACLE IFDEF符号; - 如果定义了符号 EXEC ORACLE IFNDEF符号; - 如果未定义符号 EXEC ORACLE ELSE; -  除此以外 EXEC ORACLE ENDIF; - 结束此控制块

必须使用您的宿主语言的语句终止符终止所有EXEC ORACLE语句。例如，在Pro * COBOL中，条件语句必须以“ END-EXEC ” END-EXEC 。在Pro * FORTRAN中，它必须以返回字符终止。

6.8.1一个例子

在以下示例中，仅在定义符号site2时才预编译SELECT语句：

EXEC ORACLE IFDEF site2; EXEC SQL SELECT DNAME INTO：dept_name FROM DEPT WHERE DEPTNO =：dept_number; EXEC ORACLE ENDIF;

条件块可以嵌套，如以下示例所示：

EXEC ORACLE IFDEF外部; EXEC ORACLE IFDEF内部; ...EXEC ORACLE ENDIF; EXEC ORACLE ENDIF;

您可以通过将它放在IFDEF和ENDIF之间而不是定义符号来“注释掉”主机语言或嵌入式SQL代码。

6.8.2定义符号

您可以通过两种方式定义符号。要么包括声明

EXEC ORACLE DEFINE符号;

在您的主机程序中或使用语法在命令行上定义符号

...INAME =文件名...DEFINE =符号

符号不区分大小写。

在系统上安装Oracle预编译器时，会为您预定义某些特定于端口的符号。例如，预定义的操作系统符号包括CMS，MVS，MS-DOS，UNIX和VMS。

6.9单独的预编译

使用Oracle预编译器，您可以单独预编译多个主机程序模块，然后将它们链接到一个可执行程序中。这支持模块化编程，当程序的功能组件由不同的程序员编写和调试时，这是必需的。各个程序模块不需要用同一种语言编写。

6.9.1指南

以下指南将帮助您避免一些常见问题。

引用游标

游标名称是SQL标识符，其范围是预编译单元。因此，游标操作不能跨越预编译单元（文件）。也就是说，您不能在一个文件中声明游标并在另一个文件中打开或从中获取。因此，在进行单独的预编译时，请确保对给定游标的所有定义和引用都在一个文件中。

指定MAXOPENCURORS

预编译连接到Oracle的程序模块时，请为MAXOPENCURSORS指定一个足以支持任何程序模块的值。如果将其用于其他程序模块，则忽略MAXOPENCURSORS。在运行时仅使用对连接有效的值。

使用单个SQLCA

如果只想使用一个SQLCA，则必须在其中一个程序模块中全局声明它。

6.9.2限制

对显式游标的所有引用必须位于同一程序文件中。您不能对其他模块中DECLAREd的游标执行操作。有关游标的更多信息，请参阅使用嵌入式SQL 。

此外，任何包含SQL语句的程序文件都必须具有本地SQL语句范围内的SQLCA。

6.10编译和链接

要获取可执行程序，必须编译预编译器生成的源文件，然后将生成的对象模块与SQLLIB和系统特定的Oracle库所需的任何模块相链接。此外，如果要嵌入OCI调用，请确保在OCI运行时库（OCILIB）中进行链接。

链接器解析对象模块中的符号引用。如果这些引用冲突，则链接失败。当您尝试将第三方软件链接到预编译程序时，可能会发生这种情况。并非所有第三方软件都与Oracle兼容，因此您可能会遇到问题。请与Oracle客户服务部门联系，了解该软件是否受支持。

编译和链接是系统相关的。有关说明，请参阅特定于系统的Oracle手册。

有关OCI句柄和描述符属性的更多信息，请参阅句柄和描述符属性。

6.10.1系统相关

编译和链接是系统相关的。例如，在某些系统上，必须在编译宿主语言程序时关闭编译器优化。有关说明，请参阅特定于系统的Oracle文档。

6.10.2多字节全球化支持兼容性

使用多字节全球化支持功能时，必须将目标文件链接到当前版本的SQLLIB运行时库。此版本中的多字节全球化支持功能由SQLLIB运行时库支持，而不是由Oracle Server支持。然后，可以使用Oracle数据库的任何版本执行生成的应用程序。
