1.45 COMMIT_WAIT

COMMIT_WAIT是一个高级参数，用于控制何时将提交的重做刷新到重做日志。

属性 描述

参数类型

串

句法

COMMIT_WAIT = {NOWAIT |等待| FORCE_WAIT}

默认值

没有默认值。

可修改

是（在会话级和系统级）

可在PDB中修改

是

基本

没有

Oracle RAC

每个实例都可以有自己的设置

请注意， NOWAIT选项可能会导致在数据库收到提交消息后但在写入重做日志记录之前发生的故障。这可能错误地向事务指示其更改是持久的。此外，如果数据库意外关闭，它可能会违反ACID（原子性，一致性，隔离性，持久性）事务的持久性。

如果参数设置为FORCE_WAIT ，则使用默认行为（使用wait立即刷新重做日志缓冲区）。如果这是系统设置，则将忽略会话级别和事务级别选项。如果这是会话级别设置，则将忽略事务级别选项。如果COMMIT_WAIT在设置为FORCE_WAIT ，则FORCE_WAIT选项不再有效。
