file_specification

目的

使用file_specification表单之一将file_specification指定为数据文件或临时文件，或将一个或多个文件组指定为重做日志文件组。如果要将文件存储在Oracle自动存储管理（Oracle ASM）磁盘组中，则可以进一步将该文件指定为磁盘组文件。

file_specification可以出现在以下语句中：

  • CREATE CONTROLFILE （参见CREATE CONTROLFILE ）

  • CREATE DATABASE （参见CREATE DATABASE ）

  • ALTER DATABASE （参见ALTER DATABASE ）

  • CREATE TABLESPACE （参见CREATE TABLESPACE ）

  • ALTER TABLESPACE （参见ALTER TABLESPACE ）

  • ALTER DISKGROUP （参见ALTER DISKGROUP ）

先决条件

您必须具有发出文件规范所在语句所需的权限。

语义

本节介绍file_specification的语义。有关其他信息，请参阅指定数据文件，临时文件，重做日志文件或Oracle ASM磁盘组或磁盘组文件的SQL语句。

datafile_tempfile_spec

如果数据库存储位于文件系统或Oracle ASM磁盘组中，请使用此子句指定数据文件和临时文件的属性。

redo_log_file_spec

如果数据库存储位于文件系统或Oracle ASM磁盘组中，请使用此子句指定重做日志文件的属性。

文件名

对存储在文件系统中的文件使用filename 。filename可以指定新文件或现有文件。对于新文件：

  • 如果您未使用Oracle Managed Files，则必须同时指定filename和SIZE子句，否则语句将失败。指定不带大小的文件名时，Oracle会尝试重用现有文件，并在文件不存在时返回错误。

  • 如果您正在使用Oracle管理文件，那么filename是可选的，因为是规范的其余条款。在这种情况下，Oracle数据库会为该文件创建一个唯一的名称，并将其保存在由以下初始化参数之一指定的目录中：

    • DB_RECOVERY_FILE_DEST （用于日志文件和控制文件）

    • DB_CREATE_FILE_DEST初始化参数（适用于任何类型的文件）

    • DB_CREATE_ONLINE_LOG_DEST_ n初始化参数，对于日志文件优先于DB_CREATE_FILE_DEST和DB_RECOVERY_FILE_DEST 。

对于现有文件，请指定数据文件，临时文件或重做日志文件成员的名称。filename只能包含7位ASCII或EBCDIC字符集中的单字节字符。多字节字符无效。

文件名可以包含路径前缀。如果未指定此类路径前缀，则数据库会添加默认存储位置的路径前缀，该位置与平台相关。

重做日志文件组可以包含一个或多个成员（副本）。必须根据操作系统的约定完全指定每个filename 。

数据库解释filename的方式还取决于您是否使用SIZE和REUSE子句指定它。

  • 如果仅指定filename ，或者使用REUSE子句但没有SIZE子句，则该文件必须已存在。

  • 如果使用SIZE指定filename但没有REUSE ，则该文件必须是新文件。

  • 如果使用SIZE和REUSE指定filename ，则该文件可以是new或existing。如果文件存在，则以新大小重复使用。如果它不存在，则数据库将忽略REUSE关键字并创建指定大小的新文件。

ASM_filename

对存储在Oracle ASM磁盘组中的文件使用ASM_filename形式。您可以使用此语法创建或引用数据文件，临时文件和重做日志文件。

所有形式的ASM_filename以加号（+）开头，后跟磁盘组的名称。您可以通过查询V$ASM_DISKGROUP视图来确定所有Oracle ASM磁盘组的名称。

fully_qualified_file_name

在Oracle ASM磁盘组中创建文件时，该文件将接收系统生成的完全限定的Oracle ASM文件名。只有在引用现有Oracle ASM文件时才能使用此表单。因此，如果在文件创建期间使用此表单，则还必须指定REUSE 。

  • db_name是DB_UNIQUE_NAME初始化参数的值。此名称等同于文件所在的数据库的名称，但该参数区分主数据库和备用数据库（如果两者都存在）。

  • file_type和file_type_tag表示数据库文件的类型。表8-1列出了所有文件类型及其对应的Oracle ASM标记。

  • filenumber和incarnation_number是系统生成的标识符，以保证唯一性。

您可以通过查询适合于文件类型的动态性能视图来确定Oracle ASM文件的完全限定名称（例如，数据文件为V$DATAFILE ，控制文件为V$CONTROLFILE ，等等）。您还可以通过查询V$ASM_FILE视图获取完全限定名称的filenumber和incarnation_number部分。

numeric_file_name

数字Oracle ASM文件名类似于完全限定的文件名，除了它仅使用唯一的filenumber.incarnation_number字符串。您只能使用此表单来引用现有文件。因此，如果在文件创建期间使用此表单，则还必须指定REUSE 。

incomplete_file_name

仅在文件创建期间使用不完整的Oracle ASM文件名。如果仅指定磁盘组名称，则Oracle ASM会使用适当的默认模板作为文件类型。例如，如果要在CREATE TABLESPACE语句中创建数据文件，Oracle ASM将使用默认的DATAFILE模板来创建Oracle ASM数据文件。如果使用模板指定磁盘组名称，则Oracle ASM将使用指定的模板来创建该文件。在这两种情况下，Oracle ASM还会创建完全限定的文件名。

TEMPLATE_NAME

模板是命名的属性集合。您可以创建模板并将其应用于磁盘组中的文件。您可以通过查询V$ASM_TEMPLATE数据字典视图来确定所有Oracle ASM模板名称的名称。有关创建Oracle ASM模板的说明，请参阅diskgroup_template_clauses 。

您只能在文件创建期间指定template 。它出现在ASM_filename图的不完整和别名名称中：

  • 如果在磁盘组名称后立即指定template ，则Oracle ASM使用指定的模板创建文件，并为该文件提供完全限定的文件名。

  • 如果在指定别名后指定模板，则Oracle ASM使用指定的模板创建文件，为文件提供完全限定的文件名，并创建别名，以便随后可以使用它来引用该文件。如果您指定的别名引用现有文件，则Oracle ASM将忽略模板规范，除非您还指定了REUSE 。

alias_file_name

别名是Oracle ASM文件的用户友好名称。您可以在文件创建或引用期间使用别名文件名。您可以使用别名指定模板，但仅限于文件创建期间。要确定Oracle ASM文件的别名，请查询V$ASM_ALIAS数据字典视图。

如果在文件创建期间指定别名，请参阅diskgroup_directory_clauses和diskgroup_alias_clauses以获取有关指定完整别名的说明。

SIZE条款

以字节为单位指定文件的大小。使用K ， M ， G或T指定大小，以千字节，兆字节，千兆字节或太字节为单位。

  • 对于撤消表空间，必须为每个数据文件指定SIZE子句。对于其他表空间，如果文件已存在，或者您正在创建Oracle托管文件，则可以省略此参数。

  • 如果在创建Oracle托管文件时省略此子句，则Oracle会创建一个100M文件。

  • 表空间的大小必须比其中包含的对象的大小总和大一个块。

BLOCKSIZE条款

指定BLOCKSIZE以覆盖依赖于操作系统的扇区大小。如果省略此子句，则数据库将使用依赖于操作系统的扇区大小作为块大小。

将重做日志文件添加到512字节扇区磁盘或具有512字节仿真的4KB扇区磁盘时，新文件的块大小必须是原始平台基本块大小或4KB。

  • 如果将重做日志文件添加到512字节扇区磁盘，则必须指定512或1024（或1K）作为块大小，具体取决于您的平台。

  • 如果将重做日志文件添加到4KB扇区磁盘（本机），则必须指定4096或4K作为块大小。

  • 如果将重做日志文件添加到具有512字节仿真的4KB扇区磁盘，则可以指定512,1024（或1K）或4096（或4K）作为块大小，具体取决于您的平台。

日志组中的所有日志必须具有相同的块大小。在不同磁盘上创建的两个日志组可以具有不同的块大小。但是，混合配置会在每个日志切换时引入开销。Oracle建议您使用相同的块大小创建所有日志文件。

当使用4K扇区大小时，此子句很有用，但您希望优化磁盘空间使用而不是性能。在这种情况下，您可以通过指定BLOCKSIZE 512或对于HP-UX BLOCKSIZE 1024来覆盖操作系统扇区大小。

REUSE

指定REUSE以允许Oracle重用现有文件。

  • 如果该文件已存在，则Oracle将重用文件名并应用新大小（如果指定SIZE ）或保留原始大小。

  • 如果该文件不存在，则Oracle忽略此子句并创建该文件。

对REUSE条款的限制

除非指定了filename否则无法指定REUSE 。

每当Oracle使用现有文件时，该文件的先前内容都将丢失。

“ 添加数据文件：示例 ”和“ 添加日志文件：示例 ”

autoextend_clause

autoextend_clause对数据文件和临时文件有效，但对重做日志文件无效。使用此子句可启用或禁用新数据文件或现有数据文件或临时文件的自动扩展。如果省略此子句，则：

  • 对于Oracle托管文件：

    • 如果指定SIZE ，则Oracle数据库会创建一个指定大小的文件，并禁用AUTOEXTEND 。

    • 如果未指定SIZE ，则数据库将创建启用了AUTOEXTEND的100M文件。当需要自动扩展时，数据库将文件扩展为原始大小或100MB，以较小者为准。您可以通过指定NEXT子句来覆盖此默认行为。

  • 对于用户管理的文件，无论是否指定了SIZE ，Oracle都会创建一个禁用AUTOEXTEND的文件。

上

指定ON以启用autoextend。

关闭

指定OFF以关闭自动扩展（如果已打开）。关闭autoextend时， NEXT和MAXSIZE的值设置为零。如果在后续语句中重新打开autoextend，则必须重置这些值。

下一个

使用NEXT子句指定在需要更多扩展区时自动分配的下一个磁盘空间增量的大小（以字节为单位）。默认值是一个数据块的大小。

MAXSIZE

使用MAXSIZE子句指定允许自动扩展数据文件的最大磁盘空间。

无限

如果您不想限制Oracle可以分配给数据文件或临时文件的磁盘空间，请使用UNLIMITED子句。

对 autoextend_clause的 限制

您不能在CREATE CONTROLFILE语句或ALTER DATABASE CREATE DATAFILE子句中将此子句指定为datafile_tempfile_spec一部分。

例子

指定日志文件：示例

以下语句创建一个名为payable的数据库，该数据库具有两个重做日志文件组，每个组包含两个成员和一个数据文件：

CREATE DATABASE应付LOGFILE GROUP 1（'diska：log1.log'，'diskb：log1.log'）SIZE 50K，GROUP 2（'diska：log2.log'，'diskb：log2.log'）SIZE 50K DATAFILE'diskc ：dbone.dbf'SIZE 30M;

LOGFILE子句中的第一个文件规范指定GROUP值为1的重做日志文件组。该组的成员名为' diska:log1.log '和' diskb:log1.log '，每个大小为50千字节。

LOGFILE子句中的第二个文件规范指定GROUP值为2的重做日志文件组。该组的成员名为“ diska:log2.log ”和“ diskb:log2.log ”，大小也为50千字节。

DATAFILE子句中的文件规范指定名为“ diskc:dbone.dbf ”的数据文件，大小为30兆字节。

每个文件规范都指定了SIZE参数的值并省略了REUSE子句，因此这些文件都不存在。Oracle必须创建它们。

添加日志文件：示例

以下语句将另一个带有两个成员的重做日志文件组添加到payable数据库：

ALTER DATABASE应付ADD LOGFILE GROUP 3（'diska：log3.log'，'diskb：log3.log'）SIZE 50K REUSE;

ADD LOGFILE子句中的文件规范指定GROUP值为3的新重做日志文件组。这个新组的成员名为' diska:log3.log '和' diskb:log3.log '，每个大小为50千字节。由于文件规范指定了REUSE子句，因此每个成员都可以（但不必）已经存在。

以下语句在迁移目标磁盘4k_disk_a和4k_disk_b上添加了日志文件组5和成员日志文件。执行此语句后，您可以使用switch_logfile_clause将具有512字节块大小的磁盘上的现有日志文件切换为具有4K块大小的日志。 

ALTER DATABASE添加LOGFILE GROUP 5（'4k_disk_a：log5.log'，'4k_disk_b：log5.log'）SIZE 100M BLOCKSIZE 4096 REUSE;

指定数据文件：示例

以下语句创建一个名为stocks的表空间，其中包含三个数据文件：

CREATE TABLESPACE股票DATAFILE'stock1.dbf'尺寸10M，'stock2.dbf'尺寸10M，'stock3.dbf'尺寸10M;

数据文件的文件规范指定名为' diskc:stock1.dbf '，' diskc:stock2.dbf '和' diskc:stock3.dbf '的文件。

添加数据文件：示例

以下语句更改stocks表空间并添加新数据文件：

ALTER TABLESPACE库存ADD DATAFILE'stock4.dbf'SIZE 10M REUSE;

文件规范指定名为“ stock4.dbf ”的数据文件。如果文件名不存在，那么Oracle只会忽略REUSE关键字。

ALTER DATABASE testdb DATAFILE'+ dgroup_01 / testdb / datafile / system.261.1'ONLINE;
