前言

本指南介绍了如何配置服务器以准备安装和配置Oracle Grid Infrastructure安装（Oracle Clusterware和Oracle自动存储管理）。

它还说明了如何配置服务器和存储以准备Oracle Real Application Clusters（Oracle RAC）安装。

听众

本指南提供了网络和系统管理员的配置信息，以及在Oracle Grid Infrastructure中为集群安装安装和配置Oracle Clusterware和Oracle自动存储管理的数据库管理员（DBA）的数据库安装信息。

对于具有安装Oracle RAC的专业系统角色的用户，本书旨在供系统管理员，网络管理员或存储管理员用于配置系统以准备用于群集安装的Oracle Grid Infrastructure，并完成所有配置需要操作系统root权限的任务。成功完成Oracle Grid Infrastructure安装和配置后，系统管理员只需提供配置信息并授予数据库管理员访问权限，以便在Oracle RAC安装期间以root用户身份运行脚本。

本指南假定您熟悉Oracle数据库概念。

设置Java Access Bridge以实现Java辅助功能

安装Java Access Bridge，以便Microsoft Windows系统上的辅助技术可以使用Java Accessibility API。

Java Access Bridge是一种技术，它使实现Java Accessibility API的Java应用程序和applet对Microsoft Windows系统上的辅助技术可见。

有关使用Java Access Bridge所需的最低支持辅助技术版本的信息，请参阅Java平台标准版可访问性指南 。另请参阅本指南以获取安装和测试说明以及有关如何使用Java Access Bridge的说明。
