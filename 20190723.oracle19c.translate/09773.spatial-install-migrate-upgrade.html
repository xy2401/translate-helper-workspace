A安装，迁移，兼容性和升级

本附录提供有关Oracle Spatial和Graph的安装，迁移，兼容性和升级的信息。

如果要升级到当前的Oracle数据库版本，则Oracle Spatial and Graph会自动升级为操作的一部分。有关升级过程的信息，请参见“ Oracle数据库升级指南” 。

如果需要将Spatial和Graph降级到以前的Oracle Database版本，请按照“ Oracle数据库升级指南”中的将数据库降级回以前的Oracle Database版本的说明进行操作。

如果需要将空间数据从一个数据库迁移到另一个数据库，请参阅将空间数据从一个数据库迁移到另一个数据库

如果使用Oracle Spatial和Graph GeoRaster，请参阅确保安装或升级后GeoRaster正常工作 。

如果您使用空间和图形Web要素服务（WFS）或Web服务目录服务（CSW）支持，并且如果您拥有使用一个或多个SYS.XMLTABLEINDEX索引编制索引的先前版本的数据，请参阅索引维护之前和升级后（WFS和CSW） 。

A.1将空间数据从一个数据库迁移到另一个数据库

迁移空间数据是指将Oracle Spatial和Graph几何对象从一个Oracle数据库移动或复制到另一个Oracle数据库。（数据库可以具有相同或不同的Oracle数据库版本号。）

如果要使用数据库链接，导出/导入，Oracle Data Pump或可传输表空间迁移空间数据，则MDSYS模式必须已存在于目标数据库中。

在有限的情况下，您可能还需要使用SDO_MIGRATE.TO_CURRENT子程序（在SDO_MIGRATE包（升级）中描述）。请参阅该子程序的使用说明。

A.2安装或升级后确保GeoRaster正常工作

要使用Oracle Spatial and Graph的GeoRaster功能，必须启用它并确保正确安装Oracle XML DB存储库。

注意：

升级后，验证GeoRaster对象和数据是否有效以及是否为使用该功能的所有模式启用了GeoRaster，如“ Oracle Spatial and Graph GeoRaster开发人员指南 ”中的“在数据库中维护GeoRaster对象和系统数据”中所述。

  • 在架构级别启用GeoRaster
  • 确保安装Oracle XML DB存储库

A.2.1在模式级别启用GeoRaster

必须为将使用GeoRaster功能的每个数据库架构启用GeoRaster。

默认情况下，在最初安装Oracle Spatial and Graph后，将禁用GeoRaster功能。GeoRaster只能在模式范围内启用（即不适用于整个数据库），并且必须为将使用GeoRaster功能的每个模式启用GeoRaster。

要启用GeoRaster，请对将启用GeoRaster 的每个架构执行以下步骤：

  1. 以该架构的用户身份连接到数据库。例如：
    CONNECT SCOTT / <password-for-scott>
  2. 输入以下语句：
    执行sdo_geor_admin.enableGeoRaster;
  3. 通过检查以下语句是否返回TRUE验证现在是否已启用GeoRaster：
    SELECT sdo_geor_admin.isGeoRasterEnabled FROM DUAL;
  4. 确保此模式的用户具有CREATE TRIGGER权限（GeoRaster正常工作所需的权限）。如果用户没有CREATE TRIGGER权限（或者您不知道），请以DBA用户身份连接并向用户授予该权限。例如：
    CONNECT / AS SYSDBA GRANT创建触发器TO scott;

如果的GeoRaster表已创建并用数据填充，那么数据库升级后的GeoRaster自动为表的模式启用，您不需要为架构重新启用的GeoRaster。（只需确保为用户授予CREATE TABLE权限。）

A.2.2确保安装Oracle XML DB存储库

有关更多信息，请参阅Oracle Spatial and Graph GeoRaster开发人员指南中第1章开头附近的安装和升级说明。

要使用Oracle Spatial and Graph的GeoRaster功能，必须正确安装Oracle XML DB存储库。（通常，在安装Oracle Spatial and Graph之前，应确保已安装Oracle XML DB存储库。）

如果已安装Oracle Spatial and Graph（例如在升级期间）但未安装Oracle XML DB存储库，则需要安装Oracle XML DB存储库并重新加载相关的GeoRaster PL / SQL包。在这种情况下，请按照下列步骤操作

  1. 安装Oracle XML DB存储库。

    有关安装Oracle XML DB存储库的信息，请参阅Oracle XML DB Developer's Guide 。

  2. 转到$ORACLE_HOME /md/admin目录。

  3. 以SYS AS SYSDBA身份连接到数据库。

  4. 输入以下SQL语句：

    ALTER SESSION SET CURRENT_SCHEMA = MDSYS; @ prvtgrs.plb @ sdogrxml.sql

A.3升级前后的索引维护（WFS和CSW）

从版本11.2开始，不推荐使用SYS.XMLTABLEINDEX索引类型，因此Spatial WFS和CSW createXMLTableIndex方法创建类型为XDB.XMLINDEX的索引，而不是像以前版本中那样创建类型SYS.XMLTABLEINDEX。但是，如果您拥有使用一个或多个SYS.XMLTABLEINDEX索引编制索引的先前版本的数据，则必须在升级之前删除关联的索引，并在升级后重新创建索引，如下所示：

  1. 使用Oracle Database Release 11.1，调用dropXMLTableIndex方法（在oracle.spatial.csw.CSWAdmin或oracle.spatial.wfs.WFSAdmin ，视适用情况而定，取决于应用程序）删除关联的索引。

  2. 将数据库从版本11.1升级到版本11.2。

  3. 使用Oracle Database Release 11.2，调用createXMLTableIndex （在oracle.spatial.csw.CSWAdmin或oracle.spatial.wfs.WFSAdmin ，根据应用程序的不同，创建在步骤1中删除的索引。

有关空间和图形Web要素服务（WFS）支持的信息，请参阅Web要素服务（WFS）支持 。

有关Web的空间和图形目录服务（CSW）支持的信息，请参阅Web服务目录服务（CSW）支持 。

A.4增加纵坐标阵列的大小以支持非常大的几何

如果需要支持超过1,048,576个纵坐标的几何，则必须按照本节中的说明进行操作。但是，这样做涉及大量额外工作（运行脚本，迁移现有空间数据），某些数据库停机时间以及一些注意事项和限制。因此，除非您需要，否则不应执行本节中的操作。

要增加纵坐标数组的大小以支持最多10M纵坐标的几何，请按照下列步骤操作：

  1. 确保没有用户使用任何空间表或Oracle Spatial and Graph或Locator功能。
  2. 以具有SYSDBA权限的SYS用户身份（SYS AS SYSDBA）连接到数据库，并在出现提示时输入SYS帐户密码。
  3. 如果包含SDO_GEOMETRY列的表上存在任何sdoupggeom.sql化视图，请在执行下一步（运行sdoupggeom.sql ） 之前删除所有此类sdoupggeom.sql 。

    升级操作完成后，您可以重新创建这些物化视图。

  4. 输入以下语句：

    • Linux： @$ORACLE_HOME/md/admin/sdoupggeom.sql

    • Windows： @%ORACLE_HOME%\md\admin\sdoupggeom.sql

    sdoupggeom.sql脚本的一个操作是自动迁移所有空间数据以适应新的SDO_ORDINATE_ARRAY定义。此脚本可能需要很长时间才能完成，并且时间量将取决于数据库中存在的空间数据量。

执行这些步骤后，以下注意事项和限制适用：

  • 使用旧SDO_ORDINATE_ARRAY定义创建的任何现有可传输表空间都不起作用。

  • 如果使用具有旧SDO_ORDINATE_ARRAY定义的数据库上的原始导出实用程序创建导出文件，并且如果需要将该文件导入到使用新SDO_ORDINATE_ARRAY定义的数据库中，则必须使用原始导入实用程序指定TOID_NOVALIDATE标志，如下例所示：

    imp scott / <password> file = states.dmp tables = states TOID_NOVALIDATE = MDSYS.SDO_GEOMETRY，MDSYS.SDO_ORDINATE_ARRAY，MDSYS.SDO_ELEM_INFO_ARRAY

  • 如果计划在源数据库上执行sdoupggeom.sql之后使用Oracle Data Pump导入数据，则还必须在导入操作后在目标（目标）数据库上运行sdoupggeom.sql 。
