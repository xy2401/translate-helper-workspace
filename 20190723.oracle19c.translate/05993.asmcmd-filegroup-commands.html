ASMCMD文件组管理命令

本主题提供ASMCMD命令的摘要，使您可以管理Oracle ASM文件组和配额组。

下表列出了ASMCMD文件组和配额组命令以及简要说明。

表10-78 ASMCMD文件组命令摘要

命令 描述

chfg

根据XML配置文件或脚本修改磁盘组中的文件组。

chqg

修改磁盘组中的配额组。

lsfg

列出磁盘组中的文件组。

lsqg

列出磁盘组中的配额组。

mkfg

根据XML配置文件或脚本在磁盘组中创建文件组。

mkqg

将配额组添加到磁盘组。

mvfg

将磁盘组中的文件组移动到指定的配额组。

mvfile

将文件移动到存储文件的同一磁盘组中的指定文件组。

rmfg

从磁盘组中删除现有文件组。

rmqg

从磁盘组中删除配额组。

也可以看看：

  • 有关Oracle文件组的更多信息，请管理Oracle ASM Flex Disk Groups

  • 有关包含文件组和配额组信息的V$ASM视图的信息，请参见Oracle数据库参考 。

  • 有关使用SQL语句管理Oracle ASM文件组和配额组的信息，请参见“Oracle数据库SQL语言参考 ”

chfg

目的

根据XML配置文件或脚本更改磁盘组中文件组的属性。

语法和说明

chfg { config_file.xml | ' contents_of_xml_script '}

下表列出了chfg命令的选项。

表10-79 chfg命令的选项

选项 描述

config_file.xml

包含文件组更改的XML文件的名称。除非指定了路径，否则chfg将在ASMCMD的当前工作目录中搜索XML文件

contents_of_xml_script.xml

用单引号括起来的XML脚本的内容。

XML配置文件或脚本的根元素是<file_group>标记，并具有以下属性：

  • name ：文件组名

  • dg ：文件组所属的磁盘组名称

您可以使用以下标记设置属性：

  • name ：属性名称

  • value ：物业价值

  • file_type ：属性文件类型（可选）

  • template ：Oracle ASM模板名称（可选）

以下是chfg的XML配置文件的chfg 。配置文件名为改变文件组filegroup1磁盘组dg_data1 。文件组与配额组quotagroup1相关联。 

<filegroup name =“filegroup1”dg =“dg_data1”> <p name =“quota_group”value =“quotagroup1”/> </ filegroup>

有关Oracle ASM文件组的信息，请参阅管理Oracle ASM Flex磁盘组 。

例

以下示例显示了使用XML配置文件和XML脚本使用chfg命令。

例10-83使用chfg命令

ASMCMD [+]> chfg config_file.xml ASMCMD [+]> chfg'<filegroup name =“filegroup1”dg =“dg_data1”> <p name =“redundancy”value =“high”/> </ filegroup>'

属性

chqg命令修改指定磁盘组的指定配额组中的指定属性。有关Oracle ASM文件和配额组的信息，请参阅“ 管理Oracle ASM Flex磁盘组 ” 。

以下是chqg命令的示例。第一个示例修改data磁盘组中配额组quota_grp1的quota属性。第二个示例修改data磁盘组中配额组quota_grp2的quota属性。

mkfg

目的

根据XML配置文件或脚本在磁盘组中创建文件组。

语法和说明

mkfg { config_file.xml | ' contents_of_xml_script '}

下表列出了mkfg命令的选项。

表10-83 mkfg命令的选项

选项 描述

config_file.xml

包含文件组配置信息的XML文件的名称。除非指定了路径，否则mkfg将在ASMCMD的当前工作目录中搜索XML文件

contents_of_xml_script.xml

用单引号括起来的XML脚本的内容。

XML配置文件或脚本的根元素是<file_group>标记，并具有以下属性：

  • name ：文件组名

  • dg ：文件组所属的磁盘组名称

  • client_type ：数据库，集群，卷

  • client_name ：数据库，CDB，PDB，群集或卷名

您可以使用以下标记设置属性：

  • name ：属性名称

  • value ：物业价值

  • file_type ：属性文件类型（可选）

  • template ：Oracle ASM模板名称（可选）

以下是mkfg的XML配置文件的mkfg 。配置文件创建一个名为文件组filegroup1磁盘组dg_data1为数据库命名sample 。文件组与配额组quotagroup1相关联。除了具有高冗余的控制文件外，此文件组中的文件的冗余设置为镜像。所有其他属性都设置为默认值。 

<filegroup name =“filegroup1”dg =“dg_data1”client_type =“database”client_name =“sample”> <p name =“redundancy”value =“mirror”/> <p name =“redundancy”value =“high”file_type =“controlfile”/> <p name =“quota_group”value =“quotagroup1”/> </ filegroup>

有关Oracle ASM文件组的信息，请参阅“ 管理Oracle ASM Flex磁盘组 ” 。

例

以下示例显示了使用XML配置文件和XML脚本使用mkfg命令。

例10-87使用mkfg命令

ASMCMD [+]> mkfg config_file.xml ASMCMD [+]> mkfg'<filegroup name =“filegroup1”dg =“dg_data1”client_type =“database”client_name =“sample”> </ filegroup>'

rmfg

目的

从磁盘组中删除现有文件组。

语法和说明

rmfg [-r] disk_group file_group

下表列出了rmfg命令的选项。

表10-87 rmfg命令的选项

选项 描述

-r

指定递归操作。

DISK_GROUP

指定磁盘组。

file_group

指定文件组。

-r选项指定递归删除，包括内容。文件组包含的磁盘组中的所有文件将在删除文件组的同时删除。您必须指定此选项才能删除包含任何文件的文件组。如果省略此选项且文件组不为空，则ASMCMD将返回错误，但不会删除文件组。

以下示例显示了rmfg命令的rmfg 。示例中的第一个命令从DATA1磁盘组中删除文件组FG1 。第二个命令从DATA2磁盘组中删除文件组FG2及其内容。

例10-91使用rmfg命令

ASMCMD [+]> rmfg DATA1 FG1 ASMCMD [+]> rmfg -r DATA2 FG2
