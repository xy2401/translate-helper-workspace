目的

PREDICTION_SET返回一组预测，其中包含选择中每行的概率或成本。返回值是与字段名对象的VARRAY PREDICTION_ID和PROBABILITY或COST 。预测标识符是Oracle NUMBER ;概率和成本字段是BINARY_DOUBLE 。

PREDICTION_SET可以执行分类或异常检测。对于分类，返回值是指预测的目标类。对于异常检测，返回值是指1 （对于典型行）或0 （对于异常行）的分类。

bestN和cutoff

您可以指定bestN和cutoff来限制函数返回的预测数。默认情况下， bestN和cutoff都为null，并返回所有预测。

  • bestN是最可能或最便宜的N预测。如果多个预测共享第N个概率或成本，则该函数选择其中一个。

  • cutoff是一个值阈值。仅返回概率大于或等于cutoff或成本小于或等于cutoff预测。要仅按cutoff过滤，请为bestN指定NULL 。如果函数使用带COST MODEL AUTO的cost_matrix_clause ，则忽略cutoff 。

您可以使用cutoff指定bestN ，以返回大于或等于cutoff的N最可能的预测。如果使用成本，请指定带有cutoff bestN ，以返回小于或等于截止值的N最低成本预测。

cost_matrix_clause

您可以指定cost_matrix_clause作为偏差因子，以最大限度地减少最有害的错误分类。 cost_matrix_clause行为与“ PREDICTION_COST ”中描述的相同。

语法选择

PREDICTION_SET可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，也可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态挖掘数据。选择语法或分析语法 ：

  • 语法 - 使用第一种语法使用预定义的模型对数据进行评分。提供执行分类或异常检测的模型的名称。

  • 分析语法 - 使用分析语法在没有预定义模型的情况下对数据进行评分。分析语法使用mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见“ analytic_clause :: = ” 。）

    • 对于分类，请指定FOR expr ，其中expr是一个表达式，用于标识具有字符数据类型的目标列。

    • 对于异常检测，请指定关键字OF ANOMALY 。

在对分区模型进行评分时， PREDICTION_SET函数的语法可以使用可选的GROUPING提示。请参阅GROUPING提示 。

mining_attribute_clause

mining_attribute_clause标识要用作评分预测变量的列属性。使用分析语法调用函数时，这些预测变量也用于构建瞬态模型。mining_attribute_clause行为与PREDICTION函数的描述相同。（参见“ mining_attribute_clause ” 。）
