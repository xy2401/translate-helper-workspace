163.1 DBMS_SQLTUNE概述

DBMS_SQLTUNE包提供了许多相互关联的功能区域。

本节包含以下主题：

  • DBMS_SQLTUNE SQL调优顾问子程序

  • DBMS_SQLTUNE SQL配置文件子程序

  • DBMS_SQLTUNE SQL调整集子程序

SQL Tuning Advisor

SQL Tuning Advisor是一套顾问程序之一，是一组识别并帮助解决数据库性能问题的专家系统。具体而言，SQL Tuning Advisor可自动调整有问题的SQL语句。它将一个或多个SQL语句作为输入，并提供有关如何调整语句的精确建议。顾问程序以SQL操作的形式提供建议，用于调优SQL及其预期的性能优势。

DBMS_SQLTUNE SQL Tuning Advisor子程序组提供面向任务的界面，使您可以访问顾问程序。您可以按给定的顺序调用以下子程序，以使用SQL Tuning Advisor的一些功能：

  1. CREATE_TUNING_TASK函数创建调整任务以调整一个或多个SQL语句。

  2. EXECUTE_TUNING_TASK函数和过程执行先前创建的调整任务。

  3. REPORT_TUNING_TASK函数显示调整任务的结果。

  4. 您可以使用SCRIPT_TUNING_TASK函数创建SQL * Plus脚本，然后可以执行该脚本以实现一组Advisor建议

SQL配置文件子程序

SQL Tuning Advisor可能会建议创建SQL配置文件以提高语句的性能。SQL配置文件包含特定于该语句的辅助统计信息。查询优化器会对基数，选择性和成本进行估算，这些估计有时会大量关闭，从而导致执行计划不佳。SQL配置文件通过使用采样和部分执行技术收集其他信息来调整这些估计值来解决此问题。

DBMS_SQLTUNE SQL配置文件子程序组提供了一种机制，用于向优化程序提供针对一个特定SQL语句的统计信息，并通过为其提供最准确的统计信息，帮助优化程序为该语句做出正确的决策。例如：

  • 您可以使用ACCEPT_SQL_PROFILE过程和函数来接受SQL Tuning Advisor建议的SQL配置文件。

  • 您可以使用ALTER_SQL_PROFILE过程更改现有SQL配置文件的STATUS ， NAME ， DESCRIPTION和CATEGORY属性。

  • 您可以使用DROP_SQL_PROFILE过程删除SQL配置文件。

SQL调优集

SQL调优集存储SQL语句以及以下信息：

  • 执行上下文，例如解析模式名称和绑定值

  • 执行统计信息，例如平均运行时间和执行计数

  • 执行计划，是数据库为运行SQL语句而执行的操作序列

  • 行源统计信息，例如为计划内执行的每个操作处理的行数

您可以通过从多个源过滤或排序SQL语句来创建SQL调优集：

  • 使用SELECT_CURSOR_CACHE函数的共享SQL区域

  • 使用SELECT_WORKLOAD_REPOSITORY函数从自动工作负载存储库中排名靠前的 SQL语句

  • 使用SELECT_SQLSET函数的其他SQL调优集

  • SQL Performance Analyzer任务比较结果使用SELECT_SQLPA_TASK函数

  • 使用SELECT_SQL_TRACE函数的 SQL跟踪文件

  • 用户定义的工作负载

完整的DBMS_SQLTUNE SQL调整集子程序组有助于实现此功能。例如：

  • CREATE_SQLSET过程和函数在数据库中创建SQL调优集对象。

  • LOAD_SQLSET过程使用一组选定的SQL填充SQL调优集。

  • CAPTURE_CURSOR_CACHE_SQLSET过程在指定的时间间隔内从共享SQL区域收集SQL语句，尝试构建数据库工作负载的真实图像。

注意：

当处理SQL调优集，您可以使用DBMS_SQLSET作为替代DBMS_SQLTUNE 。

导入和导出SQL调优集和SQL配置文件

使用DBMS_SQLTUNE子程序使用通用编程模型将SQL配置文件和SQL调优集从一个系统移动到另一个系统。在这两种情况下，您都在源数据库上创建一个临时表，并使用相关数据填充此临时表。然后，按照您选择的方法（例如Oracle Data Pump或数据库链接）将该临时表移动到目标系统，在该系统中，它用于以原始形式重新构建对象。以下步骤通过此包中包含的子程序实现：

  1. 要在源系统上创建登台表，请调用CREATE_STGTAB_SQLPROF过程或CREATE_STGTAB_SQLSET过程 。

  2. 要使用源系统中的信息填充登台表，请调用PACK_STGTAB_SQLPROF过程或PACK_STGTAB_SQLSET过程 。

  3. 将登台表移动到目标系统。

  4. 要在新系统上重新创建对象，请调用UNPACK_STGTAB_SQLPROF过程或UNPACK_STGTAB_SQLSET过程 。

自动调整任务功能

自动化系统任务SYS_AUTO_SQL_TUNING_TASK由数据库创建，作为目录脚本的一部分。此任务自动从AWR中选择一组高负载SQL，并在此SQL上运行SQL Tuning Advisor。自动化任务执行与任何其他SQL调整任务相同的综合分析。

您可以通过DBMS_AUTO_SQLTUNE.REPORT_AUTO_TUNING_TASK API获取有关自动SQL调整任务活动的报告。

实时SQL监控

实时SQL监控使DBA或性能分析人员能够在执行时监视长时间运行的SQL语句的执行情况。在语句执行期间，几乎实时更新游标统计信息（例如CPU时间和IO时间）和执行计划统计信息（例如输出行数，内存和使用的临时空间）。V$SQL_MONITOR和V$SQL_PLAN_MONITOR视图公开这些统计信息。此外， DBMS_SQLTUNE还提供REPORT_SQL_MONITOR和REPORT_SQL_MONITOR_LIST函数来报告监视信息。

注意：

DBMS_SQL_MONITOR还包含REPORT_SQL_MONITOR和REPORT_SQL_MONITOR_LIST函数。

调整备用数据库工作负载

在某些情况下，备用数据库除了具有数据保护角色外，还可以承担报告角色。备用数据库可以有自己的查询工作负载，其中一些可能需要调优。您可以在备用数据库上发出SQL Tuning Advisor语句，该数据库是只读的。备用到主数据库链接使DBMS_SQLTUNE能够将数据写入主数据库并从主数据库读取数据。有资格调整备用工作负载的过程包括database_link_to参数。

163.2 DBMS_SQLTUNE安全模型

该程序包可供PUBLIC使用，并执行自己的安全检查。

请注意以下事项：

  • 由于SQL Tuning Advisor依赖于顾问程序框架，因此所有调优任务接口（ * _TUNING_TASK ）都需要ADVISOR权限。

  • SQL调优集子程序（ * _SQLSET ）需要以下任一特权：

    • ADMINISTER SQL TUNING SET

      您只能创建和修改您拥有的SQL调优集。

    • 管理任何SQL TUNING SET

      您可以操作所有SQL调优集，甚至是其他用户拥有的调优集。

  • 在早期版本中，需要三个不同的权限来调用涉及SQL配置文件的子程序：

    • 创建任何SQL配置文件

    • 更改任何SQL配置文件

    • DROP ANY SQL PROFILE

    已弃用上述权限，而使用ADMINISTER SQL MANAGEMENT OBJECT 。

163.3 DBMS_SQLTUNE数据结构

DBMS_SQLTUNE包中的SELECT_ *子程序返回SQLSET_ROW类型的对象。

对象类型

  • SQLSET_ROW对象类型

163.3.1 SQLSET_ROW对象类型

SQLSET_ROW对象为用户建模SQL调整集的内容。

逻辑上，SQL调优集是SQLSET_ROW对象的集合。每个SQLSET_ROW包含一个SQL语句及其执行上下文，统计信息，绑定和计划。SELECT_ *子程序每个都将数据源建模为SQLSET_ROW对象的集合，每个对象由（ sql_id ， plan_hash_value ）唯一标识。类似地， LOAD_SQLSET过程将行类型为SQLSET_ROW的游标作为输入，根据用户请求的策略SQLSET_ROW处理每个SQLSET_ROW 。

几个子程序包接受SQL调优集或数据源内容的基本过滤器。这些过滤器根据定义的SQLSET_ROW中的属性表示。

句法

此过程使用一组选定的SQL语句填充SQL调优集。您可以多次调用该过程来添加新的SQL语句或替换现有语句的属性。

也可以看看：

DBMS_SQLTUNE SQL调整设置该组中其他子程序的子程序

句法

DBMS_SQLTUNE.LOAD_SQLSET（sqlset_name IN VARCHAR2，populate_cursor IN sqlset_cursor，load_option IN VARCHAR2：='INSERT'，update_option IN VARCHAR2：='REPLACE'，update_condition IN VARCHAR2：= NULL，update_attributes IN VARCHAR2：= NULL，ignore_null IN BOOLEAN：= TRUE ，commit_rows IN POSITIVE：= NULL，sqlset_owner IN VARCHAR2：= NULL）;

参数

表163-26 LOAD_SQLSET过程参数

参数 描述

sqlset_name

指定要加载的SQL调整集的名称。

populate_cursor

指定要加载的SQL调整集的游标引用。

load_option

指定将哪些语句加载到SQL调优集中。可能的值是：

  • INSERT （默认） - 仅添加新语句。

  • UPDATE - 更新现有的SQL语句并忽略任何新语句。

  • MERGE - 插入新语句并更新现有语句的信息。

update_option

指定现有SQL语句的更新方式。

仅当使用UPDATE或MERGE作为选项指定load_option才会考虑此参数。可能的值是：

  • REPLACE （默认） - 使用新统计信息，绑定列表，对象列表等更新语句。

  • ACCUMULATE - 尽可能组合属性（例如，诸如elapsed_time统计信息），否则用提供的值替换现有值（例如，模块和操作）。可以累积的SQL语句的属性有： elapsed_time ， buffer_gets ， direct_writes ， disk_reads ， row_processed ， fetches ， executions ， end_of_fetch_count ， stat_period和active_stat_period 。

update_condition

指定何时执行更新。

该过程仅在满足指定条件时执行更新。条件可以指数据源或目标。条件必须使用以下前缀来引用源或目标中的属性：

  • OLD - 引用SQL调优集（目标）中的语句属性。

  • NEW - 引用输入语句（source）中的语句属性。

update_attributes方法

指定在合并或更新期间要更新的SQL语句属性的列表。

可能的值是：

  • NULL （默认值） - 指定除执行上下文之外的输入游标的内容。换句话说，它等同于没有执行上下文的ALL ，例如模块和操作。

  • BASIC - 仅指定统计信息和绑定。

  • TYPICAL - 指定具有SQL计划的BASIC （没有行源统计信息）且没有对象引用列表。

  • ALL - 指定所有属性，包括执行上下文属性，例如模块和操作。

  • 要更新的逗号分隔属性名称列表：

    • EXECUTION_CONTEXT

    • EXECUTION_STATISTICS

    • SQL_BINDS

    • SQL_PLAN

    • SQL_PLAN_STATISTICS （类似于SQL_PLAN ，添加了行源统计信息）

ignore_null

指定在新值为NULL时是否更新属性。

如果为TRUE ，则当新值为NULL时，过程不会更新属性。也就是说，除非有意，否则不要覆盖NULL值。

commit_rows

指定是否在DML之后提交语句。

如果提供了值，则在插入每个指定数量的语句后提交加载。如果提供NULL ，则在操作结束时，加载仅提交一次。

通过为此参数提供值，您可以在DBA_SQLSET视图中监视SQL调整集加载操作的DBA_SQLSET 。加载新的SQL语句时， STATEMENT_COUNT值会增加。

sqlset_owner

定义SQL调优集的所有者或当前架构所有者（或当前所有者的NULL ）。

例外

  • 当sqlset_name无效或相应的SQL调优集不存在，或者populate_cursor不正确且无法执行时，此过程将返回错误。

  • 提供无效过滤器时也会引发异常。过滤器可能无效，因为它们不解析（例如，它们引用不在sqlset_row属性），或者因为它们违反了用户的权限。

使用说明

输入populate_cursor中的行必须是SQLSET_ROW类型。

例子

在此示例中，您将使用所有共享SQL区域语句创建并填充SQL调优集，其中经过时间为5秒或更长时间，不包括属于SYS模式的语句（以模拟应用程序用户工作负载）。您选择SQL语句的所有属性并使用默认模式在调优集中加载它们，默认模式仅加载新语句，因为SQL调优集为空。

假设您现在希望使用工作负载存储库（AWR）中存储的内容来扩充此信息。您使用'ACCUMULATE' update_option 'ACCUMULATE'作为update_option填充调优集，因为假设自快照拍摄以来当前在缓存中的游标已经老化。
