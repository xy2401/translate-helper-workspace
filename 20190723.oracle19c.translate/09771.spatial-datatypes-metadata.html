2.2 SDO_GEOMETRY对象类型

使用Spatial和Graph，空间对象的几何描述存储在单个行中，在用户定义的表中的对象类型SDO_GEOMETRY的单个列中。

任何具有SDO_GEOMETRY类型列的表必须具有另一列或一组列，用于为该表定义唯一主键。这种表有时被称为空间表或空间几何表。

Oracle Spatial and Graph将对象类型SDO_GEOMETRY定义为：

CREATE TYPE sdo_geometry AS OBJECT（SDO_GTYPE NUMBER，SDO_SRID NUMBER，SDO_POINT SDO_POINT_TYPE，SDO_ELEM_INFO SDO_ELEM_INFO_ARRAY，SDO_ORDINATES SDO_ORDINATE_ARRAY）;

Oracle Spatial and Graph还定义SDO_POINT_TYPE，SDO_ELEM_INFO_ARRAY和SDO_ORDINATE_ARRAY类型，这些类型在SDO_GEOMETRY类型定义中使用，如下所示：

CREATE TYPE sdo_point_type AS OBJECT（X NUMBER，Y NUMBER，Z NUMBER）; CREATE TYPE sdo_elem_info_array编号为NUMBER的AS VARRAY（1048576）; CREATE TYPE sdo_ordinate_array编号为NUMBER的AS VARRAY（1048576）;

由于最大SDO_ORDINATE_ARRAY大小为1,048,576个数字，因此SDO_GEOMETRY对象中的最大顶点数取决于每个顶点的维数：两个维度为524,288，三维为349,525，四维为262,144。

2.2.1 SDO_GTYPE

该 SDO_GTYPE属性指示几何的类型。有效的几何类型对应于OGIS Simple Features for SQL规范的几何对象模型中指定的几何类型（Surfaces除外）。数值与OGIS规范中给出的值不同，但在适用的名称和语义之间存在直接对应关系。

SDO_GTYPE值为DLTT格式的4位数，其中：

  • D标识尺寸数量（2,3或4）

  • L标识三维线性参考系统（LRS）几何的线性参考测量维度，即，哪个维度（3或4）包含测量值。对于非LRS几何体，请指定0。有关线性参考系统（LRS）的信息，请参阅线性参考系统 。

  • TT标识几何类型（00到09，保留10到99以备将来使用）。

2.2.2 SDO_SRID

该 SDO_SRID属性可用于标识与几何体关联的坐标系（空间参照系）。如果SDO_SRID为null，则没有坐标系与几何关联。如果SDO_SRID不为null，则它必须包含SDO_COORD_REF_SYS表的SRID列中的值（在SDO_COORD_REF_SYS表中描述），并且必须将此值插入USER_SDO_GEOM_METADATA视图的SRID列（在几何元数据视图中描述）。

如果将在该列上构建空间索引，则几何列中的所有几何都必须具有相同的SDO_SRID值。

有关坐标系的信息，请参阅坐标系（空间参考系） 。

2.2.3 SDO_POINT

该 SDO_POINT属性是使用SDO_POINT_TYPE对象类型定义的，该对象类型具有属性X，Y和Z，全部类型为NUMBER。 （SDO_POINT_TYPE定义显示在SDO_GEOMETRY对象类型中 。）如果SDO_ELEM_INFO和SDO_ORDINATES数组都为null，并且SDO_POINT属性为非null，则X，Y和Z值将被视为点几何的坐标。否则，Spatial和Graph将忽略SDO_POINT属性。您应该将点几何存储在SDO_POINT属性中以获得最佳存储空间;如果图层中只有点几何，强烈建议您将点几何存储在SDO_POINT属性中。

2.2.4 SDO_ELEM_INFO

该 SDO_ELEM_INFO属性使用不同长度的数字数组定义。此属性使您可以了解如何解释存储在SDO_ORDINATES属性中的纵坐标（在SDO_ORDINATES中描述）。

每个三元组数字解释如下：

  • SDO_STARTING_OFFSET - 指示SDO_ORDINATES数组中存储此元素的第一个纵坐标的偏移量。偏移值从1开始，而不是从0开始。因此，第一个元素的第一个纵坐标为SDO_GEOMETRY.SDO_ORDINATES（1）。如果存在第二个元素，则其第一个纵坐标将位于SDO_GEOMETRY.SDO_ORDINATES（ n ），其中n反映SDO_ORDINATE_ARRAY定义中的位置（例如，19表示第19个数字， 如图2-4中的带有孔的多边形） ）。

  • SDO_ETYPE - 表示元素的类型。有效值如表2-2所示。

    SDO_ETYPE值1,2,1003和2003被视为简单元素 。它们由SDO_ELEM_INFO数组中的单个三元组条目定义。对于SDO_ETYPE值1003和2003，第一个数字表示外部 （1）或内部 （2）：

    1003：外部多边形环（必须以逆时针顺序指定）

    2003：内部多边形环（必须按顺时针顺序指定）

    注意：

    不鼓励在单个几何中使用3作为多边形环元素的SDO_ETYPE值。仅当您不知道简单多边形是外部还是内部时，才应指定3，然后应使用SDO_MIGRATE.TO_CURRENT过程将表或层升级到当前格式，如SDO_MIGRATE包（升级）中所述 。

    您不能在单个几何中混合使用1位和4位SDO_ETYPE值。

    SDO_ETYPE值4,1005,2005,1006和2006被视为复合元素 。它们包含至少一个标题三元组，其中包含属于复合元素的一系列三元组值。对于4位SDO_ETYPE值，第一个数字表示外部 （1）或内部 （2）：

    1005：外部多边形环（必须以逆时针顺序指定）

    2005：内部多边形环（必须按顺时针顺序指定）

    1006：由一个或多个多边形环组成的外表面

    2006年：内部表面采用坚固的元素

    1007：实体元素

    复合元素的元素是连续的。复合元素中子元素的最后一个点是下一个子元素的第一个点。这一点不再重复。

  • SDO_INTERPRETATION - 表示两种情况之一，具体取决于SDO_ETYPE是否为复合元素。

    如果SDO_ETYPE是复合元素（4,1005或2005），则此字段指定后续三元组值是元素的一部分。

    如果SDO_ETYPE不是复合元素（1,2,1003或2003），则解释属性确定如何解释此元素的纵坐标序列。例如，线串或多边形边界可以由一系列连接的直线段或圆弧组成。

    表2-2中给出了有效SDO_ETYPE和SDO_INTERPRETATION值对的描述。

如果几何包含多个元素，则元素的最后一个纵坐标始终小于下一个元素的起始偏移量。几何中的最后一个元素由纵坐标从其起始偏移量到SDO_ORDINATES变长度数组的末尾描述。

2

3

2.7.6定向点

定向点是一种特殊类型的点几何，包括表示点的位置和虚拟终点的坐标，用于指示可用于在该点旋转符号或从该点扩展标签的方向矢量。定向点的主要用途是在地图可视化和显示应用程序中包含符号，例如用于指示高速公路的屏蔽符号。

要指定定向点：

  • 对于点或多点几何体，请使用SDO_GTYPE值（在SDO_GTYPE中说明）。

  • 为SDO_POINT属性指定空值。

  • 在SDO_ELEM_INFO数组中（在SDO_ELEM_INFO中说明），指定另一个三元组，第二个和第三个值（SDO_ETYPE和SDO_INTERPRETATION）为1和0。例如，3,1,0的三元组表示该点是一个定向点，SDO_ORDINATES数组中的第三个数字是反映任何符号的方向向量的终点的第一个坐标或x轴值。或标签。

  • 在SDO_ORDINATES数组中（在SDO_ORDINATES中解释），从点指定方向矢量的终点坐标，值介于-1和1之间。假定取向起始点为（0,0），并将其转换为与其对应的物理点的位置。

2.7.7类型0（零）元素

类型0（零）元素用于模拟Oracle Spatial和Graph不支持的几何类型，例如曲线和样条线。类型0元素的SDO_ETYPE值为0。（有关SDO_ETYPE的信息，请参阅SDO_ELEM_INFO 。）Oracle Spatial and Graph不对0类元素编制索引，空间函数和过程会忽略它们。

类型为0的元素的几何必须包含至少一个非零元素，即SDO_ETYPE值不为0的元素。非零元素应该是不支持的几何的近似值，因此它必须具有：

  • 与Spatial和Graph支持的几何类型关联的SDO_ETYPE值

  • SDO_INTERPRETATION值对SDO_ETYPE值有效（ 参见表2-2 ）

    （类型0元素的SDO_INTERPRETATION值可以是任何数值，应用程序负责确定值的有效性和重要性。）

非零元素由Spatial和Graph索引，它将由空间索引返回。

必须将包含类型0元素的几何的SDO_GTYPE值设置为非零元素的几何类型的值。

NURBS（非均匀有理B样条曲线）允许表示具有任意形状的自由形状。NURBS表示允许控制曲线的形状，因为控制点和节点引导曲线的形状，并且它们允许用很少的数据表示复杂的形状。有关NURBS曲线的说明以及定义NURBS曲线几何的要求，请参阅Oracle Spatial and Graph中的NURBS曲线支持 。

在例2-15中所示的几何体的SDO_GEOMETRY定义中：

  • SDO_GTYPE = 2002。前2表示二维，第二2表示单行字符串。

2.11计量单位支持

如果地理参考坐标系（SDO_SRID值）与输入几何或几何相关联，则涉及测量的几何函数允许可选unit参数指定指定距离或区域的测量单位。

unit参数对于具有空SDO_SRID值的几何（即，正交笛卡尔系统）无效。有关支持坐标系的信息，请参阅坐标系（空间参考系） 。

默认度量单位是与地理坐标系统相关联的度量单位。大多数坐标系的测量单位是仪表，在这些情况下，距离的默认单位是米，区域的默认单位是平方米。但是，通过使用unit参数，您可以让Spatial和Graph自动转换并返回对应用程序用户更有意义的结果，例如，以英里为单位显示到餐馆的距离。

unit参数必须用单引号括起来，并且包含字符串unit=和SDO_UNITS_OF_MEASURE表中的有效UNIT_OF_MEAS_NAME值（在SDO_UNITS_OF_MEASURE表中描述）。例如，以下示例中的“unit = KM”（使用坐标系转换示例中的 示例6-17中的数据和定义）指定km作为度量单位：

SELECT c.name，SDO_GEOM.SDO_LENGTH（c.shape，m.diminfo，'unit = KM'）FROM cola_markets_cs c，user_sdo_geom_metadata m WHERE m.table_name ='COLA_MARKETS_CS'AND m.column_name ='SHAPE';

Spatial和Graph使用SDO_UNITS_OF_MEASURE表中的信息（在SDO_UNITS_OF_MEASURE表中描述）来确定哪些单元名称有效以及在不同单元之间进行比较或转换时使用的比率。为方便起见，您还可以使用以下旧版视图查看角度，面积和距离度量单位：

  • MDSYS.SDO_ANGLE_UNITS（在MDSYS.SDO_ANGLE_UNITS视图中描述）

  • MDSYS.SSDO_AREA_UNITS（在MDSYS.SDO_AREA_UNITS视图中描述）

  • MDSYS.SSDO_DIST_UNITS（在MDSYS.SDO_DIST_UNITS视图中描述）
