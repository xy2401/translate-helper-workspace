9.58 V $ SQL_PLAN

V$SQL_PLAN包含库高速缓存中加载的每个子游标的执行计划信息。

柱 数据类型 描述

地址

RAW（4 | 8）

此游标的父句柄的地址

HASH_VALUE

数

库高速缓存中父语句的哈希值。ADDRESS和HASH_VALUE这两列可用于与V$SQLAREA以添加特定于游标的信息。

SQL_ID

VARCHAR2（13）

库高速缓存中父游标的SQL标识符

PLAN_HASH_VALUE

数

此游标的当前SQL计划的数字表示。将一个PLAN_HASH_VALUE与另一个进行比较可以轻松识别两个计划是否相同（而不是逐行比较两个计划）。

FULL_PLAN_HASH_VALUE

数

此游标的完整SQL计划的数字表示形式。将一个FULL_PLAN_HASH_VALUE与另一个进行比较可以轻松识别两个计划是否相同（而不是逐行比较两个计划）。请注意，无法跨数据库版本比较FULL_PLAN_HASH_VALUE 。它不向后兼容。

CHILD_ADDRESS

RAW（4 | 8）

子光标的地址

CHILD_NUMBER

数

使用此执行计划的子游标的编号。列ADDRESS ， HASH_VALUE和CHILD_NUMBER可以使用加入V$SQL添加的儿童特定游标的信息。

TIMESTAMP

日期

生成执行计划的日期和时间

操作

VARCHAR2（120）

在此步骤中执行的内部操作的名称（例如， TABLE ACCESS ）

OPTIONS

VARCHAR2（120）

“ OPERATION列中描述的OPERATION变体（例如， FULL ）

OBJECT_NODE

VARCHAR2（160）

用于引用对象的数据库链接的名称（表名或视图名称）。对于使用并行执行的本地查询，此列描述了操作输出的使用顺序。

宾语＃

数

表或索引的对象编号

OBJECT_OWNER

VARCHAR2（128）

拥有包含表或索引的模式的用户的名称

OBJECT_NAME

VARCHAR2（128）

表或索引的名称

OBJECT_ALIAS

VARCHAR2（261）

对象的别名

OBJECT_TYPE

VARCHAR2（80）

对象的类型

优化器

VARCHAR2（80）

计划中第一行（语句行）的优化程序的当前模式，例如ALL_ROWS 。当操作是数据库访问（例如， TABLE ACCESS ）时，此列指示是否分析对象。

ID

数

分配给执行计划中每个步骤的编号

PARENT_ID

数

对当前步骤的输出进行操作的下一个执行步骤的ID

深度

数

树中操作的深度（或级别）。没有必要发出CONNECT BY语句来获取级别信息，该级别信息通常用于缩进PLAN_TABLE表中的行。根操作（语句）是级别0。

位置

数

具有相同PARENT_ID所有操作的处理PARENT_ID

SEARCH_COLUMNS

数

具有开始和停止键的索引列数（即具有匹配谓词的列数）

成本

数

优化程序基于成本的方法估算的操作成本。对于使用基于规则的方法的语句，此列为null。

CARDINALITY

数

通过基于成本的优化器估计操作产生的行数

BYTES

数

通过基于成本的优化器估计操作产生的字节数

OTHER_TAG

VARCHAR2（140）

描述OTHER列的内容。请参阅EXPLAIN PLAN了解值。

PARTITION_START

VARCHAR2（256）

启动一系列已访问分区的分区

PARTITION_STOP

VARCHAR2（256）

停止一系列访问的分区的分区

PARTITION_ID

数

计算PARTITION_START和PARTITION_STOP列的值对的步骤

其他

VARCHAR2（4000）

用户可能认为有用的执行步骤的其他信息。请参阅EXPLAIN PLAN了解值。

分配

VARCHAR2（80）

存储用于将生产者查询服务器中的行分发到使用者查询服务器的方法

CPU_COST

数

由优化程序基于成本的方法估计的操作的CPU成本。对于使用基于规则的方法的语句，此列为null。

IO_COST

数

由优化器基于成本的方法估计的操作的I / O成本。对于使用基于规则的方法的语句，此列为null。

TEMP_SPACE

数

由优化程序基于成本的方法估计的操作（排序或散列连接）的临时空间使用情况。对于使用基于规则的方法的语句，此列为null。

ACCESS_PREDICATES

VARCHAR2（4000）

用于在访问结构中定位行的谓词。例如，启动或停止索引范围扫描的谓词。

FILTER_PREDICATES

VARCHAR2（4000）

用于在生成行之前过滤行的谓词

投影

VARCHAR2（4000）

操作产生的表达

时间

数

由优化程序基于成本的方法估计的操作的经过时间（以秒为单位）。对于使用基于规则的方法的语句，此列为null。

QBLOCK_NAME

VARCHAR2（128）

查询块的名称

备注

VARCHAR2（4000）

备注

OTHER_XML

CLOB

提供特定于执行计划执行步骤的额外信息。此列的内容使用XML构建，因为可以在那里存储多条信息。这包括：

  • 用于解析查询的模式的名称

  • 生成解释计划的Oracle数据库的版本号

  • 与执行计划关联的哈希值

  • 大纲的名称（如果有）或用于构建执行计划的SQL配置文件

  • 指示是否使用动态统计数据来制定计划

  • 大纲数据，一组可用于重新生成同一计划的优化程序提示

  • 描述计划表中的行与自适应计划的子计划之间的关系的附加数据。请注意，在Oracle Database 12 c中 ，计划表和V$SQL_PLAN中将有额外的行。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
