8安装适用于SQL Server的Oracle Database Gateway

本节提供有关Oracle Database Gateway for SQL Server的硬件和软件要求以及安装过程的信息。

要安装SQL Server的Oracle数据库网关，请按照下列步骤操作：

  1. 确保系统满足Oracle Database Gateway for SQL Server的 系统要求部分中指定的所有硬件和软件要求部分Oracle Database Gateway for SQL Server的 系统要求 。

  2. 运行Oracle Universal Installer。

    有关运行Oracle Universal Installer的更多信息，请参见步骤通过Oracle Universal Installer部分逐步完成Oracle Universal Installer。

    Oracle Universal Installer是一个菜单驱动的实用程序，它通过提示您指导您完成网关的安装行动项目。操作项及其显示顺序取决于您的平台。

    有关Oracle Database Gateway for SQL Server的安装过程的说明，请参阅表8-2 。

检查硬件要求

要确保系统满足最低要求，请按照下列步骤操作：

  1. 确定物理RAM大小。例如，对于使用Microsoft Windows 2000的计算机，请在控制面板中打开“ 系统” ，然后选择“ 常规”选项卡。如果系统中安装的物理RAM的大小小于所需的大小，则必须先安装更多内存，然后再继续。
  2. 确定配置的交换空间的大小（也称为页面文件大小）。例如，对于使用Microsoft Windows 2000的计算机，请在控制面板中打开“ 系统 ”，选择“ 高级”选项卡，然后单击“ 性能选项” 。

    如有必要，请参阅操作系统文档以获取有关如何配置其他交换空间的信息。

  3. 确定系统上的可用磁盘空间量。例如，对于使用Microsoft Windows 2000的计算机 ，请打开“ 我的电脑” ，右键单击要安装Oracle软件的驱动器，然后选择“ 属性” 。
  4. 确定temp目录中可用的磁盘空间量。这相当于可用磁盘空间总量减去安装Oracle软件所需的数量。

    如果temp目录中可用的磁盘空间少于125 MB，则首先删除所有不必要的文件。如果临时磁盘空间仍小于125 MB，则将TEMP或TMP环境变量设置为指向其他硬盘驱动器。例如，对于使用Microsoft Windows 2000的计算机，请打开“ 系统”控制面板，选择“ 高级”选项卡，然后单击“ 环境变量” 。

当Oracle Universal Installer确认安装完成时，请验证安装过程是否成功。为此，请阅读安装日志文件的内容，该文件位于C：\ Program Files \ Oracle \ Inventory \ logs目录中。

默认文件名是InstallActions YYYY-MM-DD_HH-mm-SS-AM/ PM .log ，其中：

  • YYYY是一年
  • MM是月
  • DD是白天
  • HH是小时
  • mm是分钟
  • SS是秒
  • AM/PM是白天或晚上

日志文件名中的每个变量都代表产品的安装日期和时间。
