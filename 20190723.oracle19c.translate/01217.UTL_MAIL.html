260 UTL_MAIL

UTL_MAIL包是一个用于管理电子邮件的实用程序，其中包括常用的电子邮件功能，例如附件，CC和BCC。

本章包含以下主题：

  • 安全模型

  • 操作说明

  • 规则和限制

  • UTL_MAIL子程序摘要

260.1 UTL_MAIL安全模型

默认情况下不安装UTL_MAIL ，因为SMTP_OUT_SERVER配置要求及其涉及的安全风险。在安装UTL_MAIL ，您应该采取措施防止SMTP_OUT_SERVER定义的端口被数据传输淹没.

此包现在是一个调用者权限包，并且调用用户将需要在分配给他想要连接的远程网络主机的访问控制列表中授予的连接权限。

注意：

有关管理细粒度访问的详细信息，请参阅“ Oracle数据库安全指南” 。

260.2 UTL_MAIL操作说明

您必须安装UTL_MAIL并定义SMTP_OUT_SERVER.

  • 要安装UTL_MAIL：
    sqlplus sys / <pwd> SQL> @ $ ORACLE_HOME / rdbms / admin / utlmail.sql SQL> @ $ ORACLE_HOME / rdbms / admin / prvtmail.plb

  • 您在init定义SMTP_OUT_SERVER参数。 ora rdbms初始化文件。但是，如果SMTP_OUT_SERVER ，则会调用默认值DB_DOMAIN ，保证将其定义为正确执行。

260.3 UTL_MAIL规则和限制

仅在ASCII（美国信息交换标准码）和EBCDIC（扩展二进制编码十进制交换码）代码的上下文中使用UTL_MAIL 。

260.4.1发送程序

此过程将电子邮件打包为适当的格式，查找SMTP信息，并将邮件传递到SMTP服务器以转发给收件人。

它隐藏了SMTP API并公开了一个单行的电子邮件工具，以方便使用。

句法

UTL_MAIL.SEND（发件人IN VARCHAR2 CHARACTER SET ANY_CS，收件人IN VARCHAR2 CHARACTER SET ANY_CS，cc IN VARCHAR2 CHARACTER SET ANY_CS DEFAULT NULL，bcc IN VARCHAR2 CHARACTER SET ANY_CS DEFAULT NULL，subject IN VARCHAR2 CHARACTER SET ANY_CS DEFAULT NULL，message IN VARCHAR2 CHARACTER SET ANY_CS，mime_type IN VARCHAR2 DEFAULT'text / plain; charset = us-ascii'，priority IN PLS_INTEGER DEFAULT 3，reply to IN VARCHAR2 CHARACTER SET ANY_CS DEFAULT NULL）;

参数

表260-2 SEND过程参数

参数 描述

寄件人

发件人的电子邮件地址

收件人

收件人的电子邮件地址，以逗号分隔

CC

CC收件人的电子邮件地址（以逗号分隔），默认为NULL

BCC

BCC收件人的电子邮件地址，以逗号分隔，默认为NULL

学科

要包含为电子邮件主题字符串的字符串，默认为NULL

信息

短信正文

MIME_TYPE

Mime类型的消息，默认为'text/plain; charset=us-ascii'

优先

消息优先级，映射到X优先级字段。 1是最高优先级，5是最低优先级。默认值为3。

回复

定义向谁发送回复电子邮件
