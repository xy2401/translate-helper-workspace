14.1自动收集诊断

本节介绍自动诊断收集概念。

如果Oracle Trace File Analyzer检测到任何问题，则会执行以下操作：

  • 运行必要的诊断并在出现问题时收集所有相关的日志数据

  • 在问题发生时修剪日志文件，以便Oracle跟踪文件分析器仅收集诊断所需的内容

  • 从集群中的所有节点收集并打包所有已修剪的诊断，整合单个节点上的所有内容

  • 在Oracle Trace File Analyzer存储库中存储诊断集合

  • 向您发送问题的电子邮件通知以及准备上载到Oracle Support的诊断收集的详细信息

图14-1自动诊断集合


“图14-1自动诊断集合”的说明

Oracle Trace File Analyzer使用泛洪控制机制。重复错误不会使系统充满自动收集。

识别事件会触发集合的起点，五分钟后，Oracle Trace File Analyzer会开始收集诊断数据。五分钟后开始是一起捕捉任何其他相关事件。如果事件在五分钟后仍然发生，则诊断收集继续等待。Oracle跟踪文件分析器等待30秒，没有事件发生，最多再过5分钟。

如果事件在10分钟后仍然发生，则会发生诊断收集。一个新的收集点开始。

收集完成后，Oracle跟踪文件分析器会将包含集合位置的电子邮件通知发送给相关收件人。

如果您的环境可以与oracle.com建立连接，则可以使用Oracle Trace File Analyzer将集合上载到服务请求。

$ tfactl set autodiagcollect = ON|OFF

默认情况下，自动集合为ON 。

14.2配置电子邮件通知详细信息

配置Oracle跟踪文件分析器，以在自动收集完成后向已注册的电子邮件地址发送电子邮件。

要发送电子邮件，请配置运行Oracle跟踪分析器的系统。您必须使用用户电子邮件地址配置通知才能使其正常工作。

要配置电子邮件通知详细信

  1. 要设置用于特定ORACLE_HOME的通知电子邮件，请在命令中包含操作系统所有者：
    tfactl set notificationAddress = os_user ： email
    例如：
    tfactl set notificationAddress = oracle：some.body@example.com
  2. 设置要用于任何ORACLE_HOME的通知电子邮件：
    tfactl set notificationAddress = email
    例如：
    tfactl set notificationAddress = another.body@example.com
  3. 使用tfactl set smtp配置SMTP服务器。

    出现提示时设置SMTP参数。

    表14-2 tfactl diagnosetfa命令参数

    参数 描述

    smtp.host

    指定SMTP服务器主机名。

    smtp.port

    指定SMTP服务器端口。

    smtp.user

    指定SMTP用户。

    smtp.password

    指定SMTP用户的密码。

    smtp.auth

    将Authentication标志设置为true或false。

    smtp.ssl

    将SSL标志设置为true或false。

    smtp.from

    指定来自邮件ID。

    smtp.to

    指定以逗号分隔的收件人邮件ID列表。

    smtp.cc

    指定以逗号分隔的CC邮件ID列表。

    smtp.bcc

    指定以逗号分隔的BCC邮件ID列表。

    smtp.debug

    将Debug标志设置为true或false。

    注意：

    您可以使用tfactl print smtp查看当前的SMTP配置详细信息。

  4. 通过使用tfactl sendmail email_address发送测试电子邮件来验证SMTP配置。

    当Oracle跟踪文件分析器检测到发生重大错误时，它将发送电子邮件通知，如下所示：

    图14-2电子邮件通知


    “图14-2电子邮件通知”的说明
  5. 收到通知电子邮件后，请执行以下操作：
    1. 要查找根本原因，请检查引用的集合详细信息。
    2. 如果您可以解决问题，请解决问题的根本原因。
    3. 如果您不知道问题的根本原因，请使用Oracle Support记录SR，并上载集合详细信息。

14.3收集Oracle Cluster Health Advisor检测到的问题

配置Oracle Cluster Health Advisor以自动收集ABNORMAL事件的诊断信息，并发送电子邮件通知。

  1. 要为ABNORMAL事件配置Oracle Cluster Health Advisor自动收集：
    tfactl set chaautocollect = ON
  2. 通过Oracle Trace File Analyzer启用Oracle Cluster Health Advisor通知：
    tfactl set chanotification = on
  3. 配置要发送到的Oracle Cluster Health Advisor通知的电子邮件地址：
    tfactl set notificationAddress = chatfa：john.doe@acompany.com
