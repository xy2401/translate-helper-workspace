6个 JSON数据的字符集和字符编码

JSON数据始终使用Unicode字符集。在这方面，JSON数据比XML数据更易于使用。这是JSON数据交换格式（RFC 4627）的重要组成部分。对于Oracle数据库处理的JSON数据，将自动执行任何所需的字符集转换。

Oracle数据库在处理JSON数据（解析，查询）时在内部使用UTF-8。如果输入到此类处理的数据或从其输出的数据必须与UTF-8的字符集不同，则相应地执行字符集转换。

字符集转换可能会影响性能。在某些情况下，它可能是有损的。将输入数据转换为UTF-8是无损操作，但是对于无法在输出字符集中表示的字符，转换为输出可能导致信息丢失 。

如果您的JSON数据以Unicode格式存储在数据库中，则无需进行字符集转换即可进行存储或检索。如果数据库字符集是AL32UTF8（Unicode UTF-8）或者您的JSON数据存储在具有字符集AL16UTF16的CLOB实例中，则会出现这种情况。Oracle建议您尽可能使用AL32UTF8作为数据库字符集 。

无论数据库字符集如何，使用数据类型BLOB存储的JSON数据都不会进行字符集转换以进行存储或检索。可以使用数据类型BLOB将JSON数据存储为AL32UTF8，AL16UTF16或AL16UTF16LE。

如果使用SQL / JSON函数或PL / SQL方法转换 JSON数据，并使用数据类型BLOB返回转换结果，则结果将编码为AL32UTF8。即使输入BLOB数据使用另一种Unicode编码，也是如此。

例如，如果使用函数json_query从BLOB输入中提取某些JSON数据并将结果作为BLOB返回，则使用AL32UTF8返回它。

如果应用SQL / JSON函数或PL / SQL方法指定VARCHAR2或CLOB的返回数据类型且数据库字符集不是Unicode，则可能发生有损字符集转换。如果输入JSON数据存储在BLOB实例中，那么即使它最终再次写为BLOB ，如果其中一些临时更改为VARCHAR2或CLOB则生成的BLOB数据可能会遭受有损转换。
