管理磁盘组属性

磁盘组属性是绑定到磁盘组的参数，而不是Oracle ASM实例。创建或更改磁盘组时，可以设置某些磁盘组属性。其他磁盘属性只能在创建磁盘组时设置，或者仅在更改磁盘组时设置。

本节包含以下主题：

  • 关于查看和设置磁盘组属性

  • 磁盘组属性摘要

除了本节中列出的磁盘组属性外，还会将模板属性分配给磁盘组。有关模板属性的信息，请参阅管理磁盘组模板 。

关于查看和设置磁盘组属性

您可以设置和显示磁盘组属性设置。

您可以使用V$ASM_ATTRIBUTE视图和ASMCMD lsattr命令显示磁盘组属性。

可以使用ALTER DISKGROUP或CREATE DISKGROUP SQL语句的ATTRIBUTE子句以及ASMCMD setattr和mkdg命令设置磁盘组属性。

ACCESS_CONTROL.ENABLED

此属性配置Oracle ASM文件访问控制，只能在更改磁盘组时设置。

有关ACCESS_CONTROL.ENABLED属性的信息，请参阅为Oracle ASM文件访问控制设置磁盘组属性 。

ACCESS_CONTROL.UMASK

此属性配置Oracle ASM文件访问控制，只能在更改磁盘组时设置。

有关ACCESS_CONTROL.UMASK属性的信息，请参阅为Oracle ASM文件访问控制设置磁盘组属性 。

AU_SIZE

此属性指定分配单元（AU）大小，并且只能在创建磁盘组时设置。例4-1显示了如何使用CREATE DISKGROUP SQL语句指定AU_SIZE 。

有关分配单元的信息，请参阅关于Oracle ASM分配单元 。

COMPATIBLE.ASM

此属性指定磁盘组的Oracle ASM兼容级别。创建磁盘组时可以设置兼容性属性，更改磁盘组时可以设置高级属性，但不能反转兼容性属性。有关COMPATIBLE.ASM属性的信息，请参阅磁盘组兼容性属性 。

COMPATIBLE.RDBMS

此属性指定磁盘组的数据库兼容级别。创建磁盘组时可以设置兼容性属性，更改磁盘组时可以设置高级属性，但不能反转兼容性属性。有关COMPATIBLE.RDBMS属性的信息，请参阅磁盘组兼容性属性 。

COMPATIBLE.ADVM

此属性指定磁盘组的Oracle自动存储管理群集文件系统（Oracle ACFS）和Oracle ASM动态卷管理器（Oracle ADVM）兼容级别。创建磁盘组时可以设置兼容性属性，更改磁盘组时可以设置高级属性，但不能反转兼容性属性。有关COMPATIBLE.ADVM属性的信息，请参阅磁盘组兼容性属性和Oracle ASM Dynamic Volume Manager概述 。

CONTENT.CHECK

CONTENT.CHECK属性在执行磁盘组的数据重定位操作时启用或禁用内容检查。

属性值可以设置为true （启用）或false （禁用）。

启用CONTENT.CHECK ，Oracle ASM重定位过程（重新平衡，重新同步或重新同步）会检测数据复制操作期间的损坏，并通过使用未损坏的镜像副本替换这些损坏来执行自动块损坏恢复（如果有的话）。

内容检查过程检测并修复I / O操作成功但是块内容无效的情况下的损坏。该过程还对所有支持的文件执行硬件辅助弹性数据（HARD）检查，并对数据文件执行块头检查。

只能在更改磁盘组时设置此属性。

内容类型

此属性标识磁盘组类型： data ， recovery或system 。类型值确定Oracle ASM将最近邻居磁盘放置在Oracle ASM镜像数据副本的故障组中的位置。Oracle ASM使用此属性值使存储介质中的双重故障导致不同内容类型的磁盘组不可用的可能性降低。通过指定此属性，Oracle ASM可以更好地部署可用存储中的所有磁盘组。

默认值是指定距离最近的邻居磁盘1的data 。recovery值指定距离最近的相邻磁盘3的距离， system的值指定距离5。

可以在创建或更改磁盘组时指定该属性。如果使用ALTER DISKGROUP设置或更改CONTENT.TYPE属性，则在显式运行磁盘组重新平衡之前，新配置不会生效。

CONTENT.TYPE属性仅对设置为正常或高冗余的磁盘组有效。必须将COMPATIBLE.ASM属性设置为11.2.0.3或更高才能启用磁盘组的CONTENT.TYPE属性。

可以使用ASMCMD mkdg或SQL CREATE和ALTER DISKGROUP语句设置此属性。

例如：

SQL> CREATE DISKGROUP数据正常冗余FAILGROUP控制器1 DISK'/ devices / diska1'名称diska1，...ATTRIBUTE'compatibility.asm'= '12 .1.0.1'，'content.type'='恢复'，...SQL> ALTER DISKGROUP数据SET ATTRIBUTE'content.type'='data';

此属性主要用于Oracle Exadata存储。

有关磁盘重新同步的信息，请参阅Oracle ASM快速镜像重新同步 。

CONTENT_HARDCHECK.ENABLED

CONTENT_HARDCHECK.ENABLED磁盘组属性在执行用于重新平衡磁盘组的数据复制操作时启用或禁用硬件辅助弹性数据（HARD）检查。

属性值可以设置为true或false 。只能在更改磁盘组时设置此属性。

  • 当CONTENT.CHECK磁盘组属性设置为enabled（ true ）时，将忽略CONTENT_HARDCHECK.ENABLED的设置，并检查用户数据的内容，包括HARD检查。

  • 当CONTENT.CHECK属性设置为disabled（ false ）且CONTENT_HARDCHECK.ENABLED属性设置为enabled（ true ）时，仅执行HARD检查。

  • 当CONTENT.CHECK属性设置为disabled（ false ）且CONTENT_HARDCHECK.ENABLED属性设置为disabled（ false ）时，不执行任何检查。

DISK_REPAIR_TIME

此属性指定在启动放置操作之前修复磁盘并使其恢复联机的时间间隔。此属性只能在更改磁盘组时设置，并且仅适用于普通和高冗余磁盘组。

有关磁盘修复时间的信息，请参阅Oracle ASM快速镜像重新同步 。

FAILGROUP_REPAIR_TIME

此属性指定磁盘组中故障组的默认修复时间。如果Oracle ASM确定整个故障组已失败，则使用故障组修复时间。默认值为24小时（ 24h ）。如果为磁盘指定了修复时间，例如SQL ALTER DISKGROUP OFFLINE DISK语句的DROP AFTER子句，则该磁盘修复时间将覆盖故障组修复时间。

此属性只能在更改磁盘组时设置，并且仅适用于普通和高冗余磁盘组。

有关故障组修复时间的信息，请参阅Oracle ASM快速镜像重新同步 。

LOGICAL_SECTOR_SIZE

LOGICAL_SECTOR_SIZE磁盘组属性指定磁盘组的逻辑扇区大小。可以在创建或更改磁盘组时设置该值。

LOGICAL_SECTOR_SIZE的值指定磁盘组上允许的最小I / O单位。该值必须小于或等于SECTOR_SIZE磁盘组属性的大小。

必须将COMPATIBLE.ASM设置为12.2或更高才能使用LOGICAL_SECTOR_SIZE磁盘组属性。

有关设置LOGICAL_SECTOR_SIZE磁盘组属性的信息，请参阅指定逻辑和物理扇区大小 。

有关SECTOR_SIZE磁盘组属性的信息，请参阅SECTOR_SIZE 。

PHYS_META_REPLICATED

此磁盘组属性跟踪磁盘组的复制状态。当磁盘组的Oracle ASM兼容性提高到12.1或更高时，将复制每个磁盘的物理元数据。此元数据包括磁盘标头，可用空间表块和分配表块。复制是在线异步执行的。如果已复制磁盘组中每个磁盘的物理元数据，则Oracle ASM将此属性值设置为true 。

此磁盘组属性仅在磁盘组中定义，并且Oracle ASM磁盘组兼容性（ COMPATIBLE.ASM ）设置为12.1和更高版本。此属性是只读的，仅供参考。用户无法设置或更改其值。值为true或false 。

PREFERRED_READ.ENABLED

在包含跨多个物理上分离的站点的节点的Oracle扩展群集中， PREFERRED_READ.ENABLED磁盘组属性控制是否为磁盘组启用首选读取功能。

如果启用了首选读取功能，则此功能使实例能够确定和读取与自身相同的站点上的磁盘，从而提高性能。

Oracle扩展集群中的每个Oracle ASM实例都具有确定其所在站点的信息。如果启用了首选读取功能，则根据站点信息，如果磁盘与实例位于同一站点，则确定磁盘的首选状态为true。否则，如果磁盘不与Oracle ASM实例位于同一站点，则磁盘的首选状态将设置为false。

对于扩展群集，默认情况下会启用首选读取设置，并为每个磁盘组的PREFERRED_READ.ENABLED磁盘组属性设置TRUE 。对于未扩展的群集（仅一个物理站点），将禁用首选读取。首选读取状态适用于扩展，普通，高和Flex冗余磁盘组。

Oracle Database 12 c第2版（12.2）是必需的， COMPATIBLE.ASM必须设置为12.2或更高才能使用PREFERRED_READ.ENABLED 。

无论是否已启用PREFERRED_READ.ENABLED ，都可以使用ASM_PREFERRED_READ_FAILURE_GROUPS初始化参数在Oracle ASM实例或集群中的客户端实例上的故障组级别设置首选读取，该参数可用于向后兼容。

要确定磁盘是否具有首选读取状态，可以检查V$ASM_DISK视图的PREFERRED_READ列。

SCRUB_ASYNC_LIMIT

SCRUB_ASYNC_LIMIT磁盘组属性设置异步清理的最大允许未完成并行读取请求。

值的范围是1到1024 。默认值为1 。

必须将COMPATIBLE.ASM设置为12.2或更高才能使用此磁盘组属性。

有关清理操作的信息，请参阅清理磁盘组 。

SECTOR_SIZE

SECTOR_SIZE磁盘组属性指定磁盘组的扇区大小。可以在创建或更改磁盘组时设置该值。

SECTOR_SIZE的值用于新创建的重做日志和Oracle ACFS文件系统的块大小。可以更改该值以便于将磁盘迁移到不同的扇区大小。该值必须大于或等于LOGICAL_SECTOR_SIZE磁盘组属性的大小。

必须将COMPATIBLE.ASM设置为12.2或更高才能使用SECTOR_SIZE的12.2功能，其中包括在创建磁盘组后更改扇区大小的功能。

有关设置SECTOR_SIZE磁盘组属性的信息，请参阅指定逻辑和物理扇区大小 。

有关LOGICAL_SECTOR_SIZE磁盘组属性的信息，请参阅LOGICAL_SECTOR_SIZE 。

STORAGE.TYPE

此磁盘组属性指定磁盘组中磁盘的类型。可能的值是AXIOM ， ZFSSA和OTHER 。如果该属性设置为AXIOM或ZFSSA ，则磁盘组中的所有磁盘必须属于该类型。如果该属性设置为OTHER ，则磁盘组中可以包含任何类型的磁盘。

如果STORAGE.TYPE磁盘组属性设置为AXIOM或ZFSSA ，则可以为Pillar Axiom或ZFS存储启用混合列压缩（HCC）功能。

注意：

必须将Pillar Axiom存储添加为SCSI /光纤通道。

要设置STORAGE.TYPE属性， COMPATIBLE.ASM和COMPATIBLE.RDBMS磁盘组属性必须设置为11.2.0.3或更高版本。要获得对ZFS存储的最大支持，请将COMPATIBLE.ASM和COMPATIBLE.RDBMS磁盘组属性设置为11.2.0.4或更高版本。有关磁盘组兼容性属性的信息，请参阅磁盘组兼容性 。

可以在创建磁盘组或更改磁盘组时设置STORAGE.TYPE属性。客户端连接到磁盘组时，无法设置该属性。例如，在磁盘组上启用Oracle ADVM卷时，无法设置该属性。

在设置属性之前，该属性在V$ASM_ATTRIBUTE视图或ASMCMD lsattr命令中不可见。

THIN_PROVISIONED

此属性启用或禁用在磁盘组重新平衡完成后丢弃未使用的存储空间的功能。属性值可以是true或false 。默认值为false 。

支持精简资源调配的存储供应商产品能够重用已丢弃的存储空间，以实现更高效的整体物理存储利用率。

注意：

只有Oracle Grid Infrastructure 12.2及更高版本的Linux上的Oracle ASM过滤器驱动程序（Oracle ASMFD）才支持THIN_PROVISIONED属性。有关Oracle ASMFD的信息，请参阅管理Oracle ASM筛选器驱动程序 。有关Oracle ASMFD的其他信息，例如支持的操作系统平台（认证矩阵）或配置问题，请参阅My Oracle Support （ https://support.oracle.com ）上提供的My Oracle Support文章。

有关磁盘组重新平衡的信息，请参阅调整重新平衡操作 。

VAM_MIGRATION_DONE

虚拟分配元数据（VAM）支持灵活的存储分配。必须先启用VAM_MIGRATION_DONE磁盘组属性，并在将NORMAL或HIGH冗余磁盘组迁移到FLEX冗余磁盘组之前完成迁移，而不使用限制性安装。
