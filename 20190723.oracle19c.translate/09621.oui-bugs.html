父主题： 打开影响所有平台的错误

错误18336219

Oracle数据库安装程序不会检查“指定管理选项”屏幕上为ASMSNMP指定的密码是否正确。如果继续进行配置并指定了错误的密码，则Oracle Enterprise Manager Cloud Control无法发现详细信息并监视Oracle ASM实例。

解决方法＃1：

确保在Oracle Database安装程序的“指定管理选项”屏幕中指定了正确的密码（先前在Oracle Grid Infrastructure期间为群集安装指定的密码）。

解决方法＃2：

在Oracle Enterprise Manager Cloud Control门户上，导航到Oracle ASM凭据屏幕并更新ASMSNMP的密码。在Oracle Enterprise Manager Cloud Control上保存密码后，Oracle ASM监控即开始工作。

错误23006768

在配置为使用Oracle域服务集群（DSC）的Oracle ASM服务的Oracle成员集群数据库上安装Oracle RAC数据库时，如果为ASM或ASM & Private选择的网络ASM & Private使用的类型不同作为DSC的Oracle ASM网络，数据库实例终止时出现以下错误：

IOS命中ORA-00600：内部错误代码，参数：[kfias_creg！净]

解决方法：

在安装Oracle成员集群数据库期间，选择ASM或ASM & Private的网络接口，以使其与DSC的Oracle ASM网络位于同一网络上。

错误27080535

在为具有Oracle Management Server配置的独立服务器主目录卸载Oracle Grid Infrastructure时，不会删除ORACLE_BASE/admin/emca的emConfig.txt文件。

解决方法：

要删除emConfig.txt文件，请运行以下命令：

rm -rf $ ORACLE_BASE / admin / emca / emConfig.txt

在最后一次ORACLE_HOME卸载期间，要删除ORACLE_BASE ，请在卸载工具退出后运行以下命令：

rm -rf $ ORACLE_BASE

错误27120934

在使用网格设置向导降级Oracle Clusterware后，从Oracle Database 19c降级到Oracle Clusterware 12.1版或11.2版，19c Oracle Grid Infrastructure主目录的Oracle Grid Infrastructure Management Repository（GIMR）的未使用数据文件仍可能存在于磁盘组。

解决方法：

在使用网格设置向导启动降级过程之前，使用以下命令删除GIMR数据库：

<Active_GI_HOME> / bin / dbca -silent -deleteDatabase -sourceDB -MGMTDB
