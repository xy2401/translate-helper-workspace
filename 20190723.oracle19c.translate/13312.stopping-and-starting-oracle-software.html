2.1.1停止和启动Oracle数据库和Oracle自动存储管理实例

本节介绍如何停止和启动Oracle数据库和Oracle自动存储管理实例，并包含以下主题：

  • 停止Oracle数据库或Oracle自动存储管理实例

  • 重新启动Oracle数据库或Oracle自动存储管理实例

2.1.1.1停止Oracle数据库或Oracle自动存储管理实例

警告：

在停止使用Oracle自动存储管理实例管理其存储的所有Oracle数据库实例之前，请不要停止Oracle自动存储管理实例。

要停止Oracle数据库或Oracle自动存储管理实例：

  1. 运行以下命令以标识必须关闭的实例的SID和Oracle主目录：

    在Oracle Solaris上：

    $ cat / var / opt / oracle / oratab

    在其他操作系统上：

    $ cat / etc / oratab

    oratab文件包含类似于以下内容的行，用于标识系统上每个数据库或Oracle自动存储管理实例的SID和相应的Oracle主目录：

    $ ORACLE_SID：$ ORACLE_HOME：<N / Y>

    注意：

    Oracle建议您使用加号（+）作为Oracle自动存储管理实例的SID中的第一个字符。

  2. 根据默认shell运行oraenv或coraenv脚本，以设置必须关闭的实例的环境变量：

    • Bourne，Bash或Korn shell：

      $。在/ usr / local / bin目录/ oraenv的

    • C shell：

      ％source / usr / local / bin / coraenv

    出现提示时，指定实例的SID 。

  3. 运行以下命令以关闭实例：
    $ sqlplus SQL> CONNECT SYS AS SYSDBA输入密码： sys_password SQL> SHUTDOWN NORMAL

    实例关闭后，您可以退出SQL * Plus。

2.1.1.2重新启动Oracle数据库或Oracle自动存储管理实例

警告：

如果数据库实例使用Oracle自动存储管理进行存储管理，则必须在启动数据库实例之前启动Oracle自动存储管理实例。

要重新启动Oracle数据库或Oracle自动存储管理实例：

  1. 如果需要，请重复步骤1和2 ，以设置ORACLE_SID和ORACLE_HOME环境变量，以标识要启动的实例的SID和Oracle主目录。
  2. 运行以下命令以启动实例：
    $ sqlplus SQL> CONNECT SYS AS SYSDBA输入密码： sys_password SQL> STARTUP

    实例启动后，您可以退出SQL * Plus。

2.2关于自动化数据库关闭和启动

Oracle建议您将系统配置为在系统启动时自动启动Oracle数据库，并在系统关闭时自动将其关闭。自动化数据库启动和关闭防止错误的数据库关闭。

要自动执行数据库启动和关闭，请使用位于$ORACLE_HOME/bin目录中的dbstart和dbshut脚本。脚本引用oratab文件中的相同条目，这些条目应用于同一组数据库。例如，您不能让dbstart脚本自动启动sid1 ， sid2和sid3 ，并且只sid1 dbshut脚本关闭sid1 。但是，您可以指定dbshut脚本关闭一组数据库，而根本不使用dbstart脚本。为此，请在系统关闭文件中包含dbshut条目，但不要在系统启动文件中包含dbstart条目。

要使用dbstart和dbshut脚本自动执行数据库启动和关闭，请执行以下操作：

  1. 以root用户身份登录。
  2. 编辑平台的oratab文件。

    要打开该文件，请使用以下命令之一：

    • 在Oracle Solaris上：

      #vi / var / opt / oracle / oratab

    • 在POWER Systems（64位）和Linux上的IBM AIX上：

      #vi / etc / oratab

    oratab文件中的数据库条目以以下格式显示：

    $ ORACLE_SID：$ ORACLE_HOME：<N | Y>

    在此示例中，值Y和N指定是希望脚本启动还是关闭数据库。对于要为其自动关闭和启动的每个数据库，首先确定该数据库的实例标识符（SID），该标识符由第一个字段中的SID标识。然后，将每个字段的最后一个字段更改为Y

    您可以将dbstart设置为自动启动单实例数据库，该数据库使用Oracle Clusterware自动启动的自动存储管理安装。这是自动存储管理群集的默认行为。为此，您必须更改数据库的oratab条目和自动存储管理安装，以分别使用值为W和N的第三个字段。这些值指定dbstart仅在启动自动存储管理实例后自动启动数据库。
