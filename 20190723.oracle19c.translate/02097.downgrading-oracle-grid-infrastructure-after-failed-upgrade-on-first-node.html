使用在线中止升级降级Oracle Grid Infrastructure

如果在设置Oracle Clusterware的活动版本之前升级Oracle Grid Infrastructure失败，请按照以下步骤将Oracle Grid Infrastructure降级到早期版本。

仅在升级失败之前运行此过程以降级Oracle Clusterware，然后root脚本crsctl set crs activeversion在最后一个节点上运行crsctl set crs activeversion命令。如果需要避免整个群集的停机，请使用此过程降级Oracle Grid Infrastructure。此过程将群集降级到先前版本。由于Oracle ASM和数据库操作在此状态下受限，因此建议尽快将群集从此状态移出。按升级失败后降级Oracle Grid Infrastructure中记录的过程完成Oracle Grid Infrastructure的降级 。
  1. 关闭第一个节点上的Oracle Grid Infrastructure堆栈：
    crsctl停止crs
  2. 从运行早期版本的Grid Infrastructure堆栈的任何节点，取消设置Oracle ASM滚动迁移模式，如下所示：
    1. 以grid用户身份登录，并在Oracle ASM实例上以SYSASM用户身份运行以下命令：
      SQL> ALTER SYSTEM停止滚动移动;
  3. 如果要从11.2.0.4或12.1.0.1升级，请在群集中的所有节点上应用最新的可用修补程序。如果升级前版本为12.1.0.2或更高版本，则不需要修补程序。
    1. 在除第一个节点之外的所有其他节点上，运行早期版本的Grid Infrastructure堆栈，使用opatchauto过程应用最新的补丁。
    2. 在早期版本Grid Infrastructure堆栈停止的第一个节点上，使用opatch apply过程应用最新的修补程序。

      有关最新可用修补程序的列表，请参阅以下链接中的My Oracle Support：

      https://support.oracle.com/

      1. 从早期版本解锁Grid Infrastructure主页：

        rootcrs.pl -unlock -crshome pre-upgrade-grid-home

        pre-upgrade-grid-home是之前发布的Grid home。

      2. 应用补丁：

        opatch应用-local -oh pre-upgrade-grid-home

      3. 从早期版本重新锁定网格主页：

        rootcrs.pl -lock
    3. 从运行早期版本的Grid Infrastructure堆栈的任何其他节点，取消设置Oracle ASM滚动迁移模式，如步骤2中所述。
  4. 在运行Oracle Grid Infrastructure而非第一个节点的任何节点上，从早期版本的Grid主页运行命令：
    clscfg -nodedowngrade -h hostname

    hostname是第一个节点的主机名。

  5. 从后来的版本Grid home中，运行命令以降级Oracle Clusterware：
    rootcrs.sh -downgrade -online

    如果rootcrs.sh不存在，则使用rootcrs.pl 。

  6. 在早期版本Grid home的第一个节点上启动Oracle Grid Infrastructure堆栈：
    crsctl开始crs

注意：

您可以按任何顺序降级群集节点。
