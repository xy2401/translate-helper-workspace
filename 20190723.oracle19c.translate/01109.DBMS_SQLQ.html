161 DBMS_SQLQ

DBMS_SQLQ包提供了用于为SQL语句的执行计划配置隔离阈值的接口。如果任何Resource Manager阈值等于或小于SQL语句的隔离配置中指定的隔离阈值，则如果SQL语句使用其隔离配置中指定的执行计划，则不允许运行SQL语句。

本章包含以下主题：

  • DBMS_SQLQ概述

  • DBMS_SQLQ子程序摘要

161.1 DBMS_SQLQ概述

DBMS_SQLQ包提供了用于为SQL语句的执行计划配置隔离阈值的接口。如果任何Resource Manager阈值等于或小于SQL语句的隔离配置中指定的隔离阈值，则如果SQL语句使用其隔离配置中指定的执行计划，则不允许运行SQL语句。

您可以使用DBMS_SQLQ包子程序：

  • 为SQL语句的执行计划创建隔离配置，并为其资源使用指定隔离阈值

  • 查询隔离配置中指定的隔离阈值

  • 删除隔离配置

  • 将隔离配置从一个数据库传输到另一个数据库

161.2.1 ALTER_QUARANTINE程序

此过程为隔离区配置中的资源指定SQL语句执行计划的隔离阈值。

句法

DBMS_SQLQ.ALTER_QUARANTINE（quarantine_name IN VARCHAR2，parameter_name IN VARCHAR2，parameter_value IN VARCHAR2）;

参数

表161-2 ALTER_QUARANTINE过程参数

参数 描述

quarantine_name

隔离配置的名称。

PARAMETER_NAME

需要指定隔离阈值的资源的名称。您可以指定以下任何一个值：

  • CPU_TIME ：CPU时间

  • ELAPSED_TIME ：经过的时间

  • IO_MEGABYTES ：I / O，以兆字节为单位

  • IO_REQUESTS ：物理I / O请求数

  • IO_LOGICAL ：逻辑I / O请求数

  • ENABLED ：标志以启用或禁用隔离配置。指定YES启用它， NO指定禁用它。默认值为YES 。

  • AUTOPURGE ：标志以启用或禁用隔离配置的自动清除。如果设置为YES ，则隔离配置将在53周后自动清除（如果未使用）。如果将其设置为NO ，则永远不会清除隔离配置。默认值为YES 。

PARAMETER_VALUE

parameter_name指定的资源的隔离阈值。

例子

在以下示例中，为CPU时间指定的隔离阈值为5秒，隔离配置SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4已用时间为10秒。 

BEGIN DBMS_SQLQ.ALTER_QUARANTINE（QUARANTINE_NAME =>'SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4'，PARAMETER_NAME =>'CPU_TIME'，PARAMETER_VALUE =>'5'）; DBMS_SQLQ.ALTER_QUARANTINE（QUARANTINE_NAME =>'SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4'，PARAMETER_NAME =>'ELAPSED_TIME'，PARAMETER_VALUE =>'10'）;结束; /

使用隔离配置中指定的执行计划执行SQL语句时，如果CPU时间的资源管理器阈值为5秒或更短，或者经过的时间为10秒或更短，则不允许运行SQL语句。

161.2.2 CREATE_QUARANTINE_BY_SQL_ID函数

此函数为基于SQL ID的SQL语句的执行计划创建隔离配置。

句法

DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_ID（sql_id IN VARCHAR2，plan_hash_value IN NUMBER DEFAULT NULL）RETURN VARCHAR2;

参数

表161-3 CREATE_QUARANTINE_BY_SQL_ID函数参数

参数 描述

SQL_ID

SQL语句的SQL ID。

plan_hash_value

SQL语句的执行计划的哈希值。默认值为NULL 。

如果为NULL ，则隔离配置将应用于SQL语句的所有执行计划。

回报价值

隔离配置的名称。

例子

以下示例为SQL语句为8vu7s907prbgr的SQL语句创建隔离配置。隔离配置适用于SQL语句的所有执行计划。 

DECLARE quarantine_config VARCHAR2（30）; BEGIN quarantine_config：= DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_ID（SQL_ID =>'8vu7s907prbgr'）;结束; /

以下示例为具有SQL ID为8vu7s907prbgr的SQL语句的哈希值为3488063716的执行计划创建隔离配置。 

DECLARE quarantine_config VARCHAR2（30）; BEGIN quarantine_config：= DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_ID（SQL_ID =>'8vu7s907prbgr'，PLAN_HASH_VALUE =>'3488063716'）;结束; /

161.2.3 CREATE_QUARANTINE_BY_SQL_TEXT函数

此函数为基于SQL文本的SQL语句的执行计划创建隔离配置。

句法

DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_TEXT（sql_text IN CLOB，plan_hash_value IN NUMBER DEFAULT NULL）RETURN VARCHAR2;

参数

表161-4 CREATE_QUARANTINE_BY_SQL_TEXT函数参数

参数 描述

SQL_TEXT

SQL语句。

plan_hash_value

SQL语句的执行计划的哈希值。默认值为NULL 。

如果为NULL ，则隔离配置将应用于SQL语句的所有执行计划。

回报价值

隔离配置的名称。

例子

以下示例创建一个隔离配置，该配置适用于SQL语句'select count(*) from emp'所有执行计划'select count(*) from emp' 。 

DECLARE quarantine_config VARCHAR2（30）; BEGIN quarantine_config：= DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_TEXT（SQL_TEXT => to_clob（'从emp选择计数（*）））;结束; /

以下示例为执行计划创建隔离配置，其SQL语句的哈希值为3488063716 ，其SQL文本为'select count(*) from emp' 。 

DECLARE quarantine_config VARCHAR2（30）; BEGIN quarantine_config：= DBMS_SQLQ.CREATE_QUARANTINE_BY_SQL_TEXT（SQL_TEXT => to_clob（'从emp'选择计数（*）），PLAN_HASH_VALUE =>'3488063716'）;结束; /

161.2.4 CREATE_STGTAB_QUARANTINE过程

此过程创建一个用于存储隔离配置的临时表，以便可以从当前数据库导出临时表并将其导入另一个数据库，从而使隔离配置可以跨数据库使用。

句法

DBMS_SQLQ.CREATE_STGTAB_QUARANTINE（staging_table_name IN VARCHAR2，staging_table_owner IN VARCHAR2 DEFAULT NULL，tablespace_name IN VARCHAR2 DEFAULT NULL）;

参数

表161-5 CREATE_STGTAB_QUARANTINE过程参数

参数 描述

staging_table_name

登台表的名称。

staging_table_owner

登台表的模式所有者的名称。默认值为NULL ，这意味着执行此过程的数据库用户将设置为登台表所有者。

TABLESPACE_NAME

需要在其中创建登台表的表空间的名称。默认值为NULL ，表示在数据库的默认表空间中创建登台表。

例子

以下示例在数据库的缺省表空间中创建登台表TBL_STG_QUARANTINE ，并将其表所有者设置为执行此过程的数据库用户。 

BEGIN DBMS_SQLQ.CREATE_STGTAB_QUARANTINE（STAGING_TABLE_NAME =>'TBL_STG_QUARANTINE'）;结束; /

161.2.5 DROP_QUARANTINE过程

此过程将删除隔离配置。

句法

参数

表161-6 DROP_QUARANTINE过程参数

参数 描述

quarantine_name

要删除的隔离配置的名称。

例子

BEGIN DBMS_SQLQ.DROP_QUARANTINE（'SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4'）;结束; /

161.2.6 GET_PARAM_VALUE_QUARANTINE函数

此函数返回隔离配置中指定的资源的隔离阈值。

句法

DBMS_SQLQ.GET_PARAM_VALUE_QUARANTINE（quarantine_name IN VARCHAR2，parameter_name IN VARCHAR2）RETURN VARCHAR2;

参数

表161-7 GET_PARAM_VALUE_QUARANTINE函数参数

参数 描述

quarantine_name

隔离配置的名称。

PARAMETER_NAME

需要检索隔离阈值的资源。

回报价值

返回隔离配置中指定的资源的隔离阈值。

例子

以下示例返回隔离配置中指定的CPU时间的隔离阈值，该隔离阈值为SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4 。 

DECLARE quarantine_config_setting_value VARCHAR2（30）; BEGIN quarantine_config_setting_value：= DBMS_SQLQ.GET_PARAM_VALUE_QUARANTINE（QUARANTINE_NAME =>'SQL_QUARANTINE_3z0mwuq3aqsm8cfe7a0e4'，PARAMETER_NAME =>'CPU_TIME'）;结束; /

161.2.7 PACK_STGTAB_QUARANTINE函数

此功能将一个或多个隔离配置添加到临时表中。

句法

DBMS_SQLQ.PACK_STGTAB_QUARANTINE（staging_table_name IN VARCHAR2，staging_table_owner IN VARCHAR2 DEFAULT NULL，name IN VARCHAR2 DEFAULT'％'，sql_text IN VARCHAR2 DEFAULT'％'，启用IN VARCHAR2 DEFAULT NULL）返回数字;

参数

表161-8 PACK_STGTAB_QUARANTINE函数参数

参数 描述

staging_table_name

需要添加隔离配置的登台表的名称。

staging_table_owner

登台表的模式所有者的名称。默认值为NULL ，这意味着执行此过程的数据库用户将设置为登台表所有者。

名称

隔离配置的名称。它的值区分大小写，并且它接受通配符。

SQL_TEXT

SQL语句文本。它的值区分大小写，并且它接受通配符。

启用

指示是应启用还是禁用隔离配置的标志。如果设置为YES ，则启用隔离配置，否则禁用隔离配置。默认值为NULL ，这意味着默认情况下禁用隔离配置。

回报价值

添加到登台表的隔离配置数。

例子

以下示例将名称以SQL_QUARANTINE_开头的所有隔离配置添加到登台表TBL_STG_QUARANTINE 。

DECLARE quarantine_configs NUMBER; BEGIN quarantine_configs：= DBMS_SQLQ.PACK_STGTAB_QUARANTINE（STAGING_TABLE_NAME =>'TBL_STG_QUARANTINE'，NAME =>'SQL_QUARANTINE_％'）;结束; /

161.2.8 UNPACK_STGTAB_QUARANTINE功能

DBMS_SQLQ.UNPACK_STGTAB_QUARANTINE（staging_table_name IN VARCHAR2，staging_table_owner IN VARCHAR2 DEFAULT NULL，name IN VARCHAR2 DEFAULT'％'，sql_text IN VARCHAR2 DEFAULT'％'，启用IN VARCHAR2 DEFAULT NULL）返回数字;

参数 描述

staging_table_name

需要在数据库中创建隔离配置的登台表的名称。

staging_table_owner

登台表的模式所有者的名称。默认值为NULL ，这意味着执行此过程的数据库用户将设置为登台表所有者。

名称

隔离配置的名称。它的值区分大小写，并且它接受通配符。

SQL_TEXT

SQL语句文本。它的值区分大小写，并且它接受通配符。

启用

指示是应启用还是禁用隔离配置的标志。如果其值为YES ，则启用隔离配置，否则禁用隔离配置。默认值为NULL ，这意味着默认情况下禁用隔离配置。

回报价值
