ALTER DATABASE LINK

目的

当连接或身份验证用户的密码更改时，使用ALTER DATABASE LINK语句修改固定用户数据库链接。

注意：

  • 您不能使用此语句更改与数据库链接关联的连接或身份验证用户。要更改user ，必须重新创建数据库链接。

  • 您不能使用此语句来更改连接或身份验证用户的密码。为此，必须使用ALTER USER语句，然后使用ALTER DATABASE LINK语句ALTER DATABASE LINK 。

  • 此语句仅对固定用户数据库链接有效，对连接用户或当前用户数据库链接无效。有关这两种类型的数据库链接的详细信息，请参阅CREATE DATABASE LINK 。

先决条件

要更改专用数据库链接，您必须具有ALTER DATABASE LINK系统特权。若要更改公用数据库链接，您必须具有ALTER PUBLIC DATABASE LINK系统特权。

语义

ALTER DATABASE LINK语句仅用于使用连接和身份验证用户的当前密码更新固定用户数据库链接。因此，在CREATE DATABASE LINK语句中有效且未出现在上面语法图中的任何子句在ALTER DATABASE LINK语句中无效。此语句中允许的所有子句的语义与CREATE DATABASE LINK这些子句的语义相同。有关此信息，请参阅CREATE DATABASE LINK 。

例子

以下语句显示ALTER DATABASE LINK语句的有效变体：

ALTER DATABASE LINK private_link CONNECT TO hr由hr_new_password识别; ALTER PUBLIC DATABASE LINK public_link CONNECT TO scott IDENTIFIED by scott_new_password ; ALTER SHARED PUBLIC DATABASE LINK shared_pub_link CONNECT TO scott IDENTIFIED by scott_new_password由hr_new_password识别的hr认证; ALTER SHARED DATABASE LINK shared_pub_link CONNOT TO scott IDENTIFIED by scott_new_password ;
