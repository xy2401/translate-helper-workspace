C UTL_PG接口

用于APPC的Oracle数据库网关需要使用 RAW数据类型，用于传输数据和从中传输数据 PL / SQL没有任何改动 Oracle Net。这是必要的，因为只有PL / SQL应用程序具有关于发送到远程事务程序和从远程事务程序接收的数据的格式的信息。Oracle Net仅包含有关运行PL / SQL应用程序和网关服务器的系统的信息。如果允许Oracle Net对PL / SQL和网关之间的数据进行转换，数据可能以错误的格式结束。

话题：

  • UTL_PG功能

  • NUMBER_TO_RAW和RAW_TO_NUMBER个参数值

注意：

IBM VS COBOL II编译器已被不支持。但是，字符串“ IBMVSCOBOLII ”仍用作编译器名称参数的值，以表示您选择使用的任何COBOL编译器。IBMVSCOBOLII使用值IBMVSCOBOLII ，并且不会对任何特定版本的编译器创建依赖关系。

UTL_PG功能

UTL_PG包是PL / SQL的扩展，它提供了一整套功能，用于将COBOL数字格式转换为Oracle数字，将Oracle数字转换为COBOL数字格式。

UTL_PG转换格式RAW在此版本中不可移植。另外，不支持在一个系统上生成转换格式RAW并转移到另一个系统。

本节中列出的函数以标准PL / SQL方式调用：

package_name.function_name（参数）

特别是对于UTL_PG例程，这是：

UTL_PG.function_name（参数）

对于下面列出的每个函数，提供函数名称，参数及其数据类型以及返回值数据类型。除非另有说明，否则参数为IN ，而不是OUT ，参数。

wmsgbsiz

是警告消息块声明的大小（以字节为单位）。它是一个BINARY_INTEGER设置为wmsgblk的字节长度。警告消息块的长度必须至少为512且不超过8192个字节。在声明wmsgblk ，根据请求的转换的性质，为每个返回的警告计划大约512个字节。
