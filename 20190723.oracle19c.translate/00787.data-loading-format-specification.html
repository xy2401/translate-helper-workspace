23数据加载格式（DLF）规范

给出了数据加载格式（DLF）1.0版的描述，DLF是用于描述TransX实用程序加载到数据库中的翻译消息和种子数据的标准格式。

相关话题

  • 使用TransX Utility

DLF简介

DLF定义了使用TransX实用程序加载数据的标准格式。它旨在取代使用SQL脚本加载数据。DLF提供以下优势：

  • 格式验证。验证可减少翻译和加载过程中的错误。

  • 便于使用。用户不必维护每个数据文件的字符编码以与数据文件中使用的语言相对应。

DLF基于XML 1.0规范。

注意：

TransX作为经过身份验证的用户运行。请务必检查您的数据文件，并仅从受信任的来源加载数据文件。

DLF的命名约定

本节介绍本文档中使用的命名约定。

元素和属性

描述了本文档中使用的元素和属性的命名约定。

  • 标准英文字母

  • 仅限小写字母

  • 连字符（ - ）可用于连接

  • 必须始终一致地定义属性名称

  • 必须尽可能遵循行业标准术语

值

除了用于列名的某些属性值之外，值区分大小写。所有预定义的属性值都是小写的。本规范未定义任何元素值。

文件扩展名

本规范不推荐使用文件扩展名。本规范的未来版本可能会建议文档使用符合8.3标准的扩展。

DLF的一般结构

数据加载格式是XML，因此它以XML声明开头。在XML声明之后出现DLF文档本身，包含在元素<table> 。

DLF文档由以下必需部分组成：

  • <lookup-key>元素包含一个列名列表，用于确定数据库中的现有行是否与<dataset>元素中包含的数据集定义中的行重复。

  • <columns>元素包含有关<dataset>元素的元数据，例如列的名称，数据类型和属性。

  • <dataset>元素包含每行的<row>元素，后者又包含一个<col>元素，该元素对应于数据库列中加载的一段数据。通过这种方式，DLF文档看起来类似于在数据库中打印数据时熟悉的表格格式，并且允许轻松编辑。

DLF提供了一个可选部分，它包含在<translation>元素中。此部分可能位于所需部分之前。

此外，DLF还提供有关TransX实用程序处理的信息。此类信息包括但不限于此：

  • <query>元素用于从SQL查询中检索要加载到列的值。

  • sequence属性用于从数据库中的序列对象检索要加载到列的值。

  • constant属性用于指定列的常量值。

  • language 属性用于指定要加载到列的语言标识符。

DLF规格

这里的主题包括DLF中的XML声明，实体引用，元素和属性。

DLF中的XML声明

可扩展标记语言（XML）声明启动XML实体。它表示XML版本。

它还可以声明文件的编码，如下例所示：

<？xml version =“1.0”encoding =“iso-8859-1”？>

与所有XML文件一样，DLF文件的默认编码假定为Unicode的8位编码（UTF-8），它是7位ASCII字符集的超集，或者是Unicode的16位编码（UTF-16），概念上是2字节的通用字符集（UCS-2），代码点代表点高于65,535。因此，对于这些字符集，编码声明不是必需的。此外，所有XML解析器都支持这些字符集。如果编码是UTF-16，则文件的第一个字符必须是Unicode Byte-Order-Mark，＃xFEFF，表示文件的字节顺序。

Oracle XML解析器支持的其他字符集包括所有Oracle字符集和常用的Internet Assigned Numbers Authority（IANA）字符集和Java编码。可以在解析器文档中找到这些字符集的名称。如果文档没有外部编码信息源（例如来自执行环境或网络协议），则必须使用编码声明声明这些声明。因此，Oracle建议您使用Unicode字符编码，以便可以省去编码声明。建议的做法是以UTF-8编码文档并使用此声明：

<？xml版本=“1.0”？>
