Oracle Database Vault跟踪事件的级别

您可以将多个级别用于Oracle Database Vault跟踪事件。

这些级别如下：

  • Low将所有失败的Oracle Database Vault授权的信息打印到跟踪文件。此类跟踪文件包括失败的域授权，失败的因子加载，失败的规则集评估等。它对Oracle数据库性能的影响很小。

  • 高打印跟踪记录，包括成功和失败的授权。由于此类跟踪跟踪所有授权，因此开销大于低级跟踪的开销。此外，跟踪文件通常更大。

  • 最高将PL / SQL堆栈和函数调用堆栈打印到跟踪文件，以及在高级别跟踪的内容（如表E-1中所述 ）。它对Oracle数据库性能的影响最大。

为当前数据库会话启用跟踪事件

您可以使用ALTER SESSION SET EVENTS SQL语句为当前数据库会话启用跟踪事件。

  1. 以已被授予DV_ADMIN角色和ALTER SESSION系统特权的用户身份登录数据库实例。

    例如：

    sqlplus leo_dvowner输入密码： 密码已连接。
  2. 输入ALTER SESSION SET EVENTS SQL语句，将跟踪设置为低，高或最高，如Oracle Database Vault跟踪事件级别中所述。

    • 要为影响较小的失败操作启用跟踪，请输入以下语句之一：

      ALTER SESSION SET EVENTS'TRACE [DV] DISK = LOW'; ALTER SESSION SET EVENTS'47998 TRACE NAME CONTEXT FOREVER，LEVEL 1';

    • 要为具有高影响的失败和成功操作启用跟踪，请输入以下语句之一：

      ALTER SESSION SET EVENTS'TRACE [DV] DISK = HIGH'; ALTER SESSION SET EVENTS'47998 TRACE NAME CONTEXT FOREVER，LEVEL 3';

    • 要使用影响最大的函数和PL / SQL调用堆栈打开失败和成功操作的跟踪，请输入以下语句之一：

      ALTER SESSION SET EVENTS'TRACE [DV] DISK = HIGHEST'; ALTER SESSION SET EVENTS'47998 TRACE NAME CONTEXT FOREVER，LEVEL 4';

为所有数据库会话启用跟踪事件

您可以使用ALTER SYSTEM SET EVENTS SQL语句为所有数据库会话启用Database Vault跟踪事件。

  1. 以已被授予DV_ADMIN角色和ALTER SYSTEM系统特权的用户身份登录数据库实例。

    例如：

    sqlplus leo_dvowner输入密码： 密码已连接。
  2. 使用在为当前数据库会话启用跟踪事件的步骤2中显示的语法输入ALTER SYSTEM SET EVENTS SQL语句。

    例如：

    ALTER SYSTEM SET EVENTS'TRACE [DV] DISK = LOW';
  3. 重启数据库。

    例如：

    关机立即启动

为所有数据库会话启用跟踪事件的另一种方法是init.ora下行添加到init.ora文件，然后重新启动数据库：

event =“47998永远跟踪名称上下文，级别[ trace_level ]”

将trace_level替换为以下值之一：

  • 1表示最低级别的跟踪

  • 3为高水平

  • 4为最高级别

例如：

event =“47998永远跟踪名称上下文，级别[1]”

多租户环境中跟踪事件的管理

您应该了解在多租户环境中如何启用跟踪事件。

  • 当前会话的跟踪事件：在多租户环境中，从根或可插拔数据库（PDB）运行ALTER SESSION SET EVENTS SQL语句可以跟踪当前用户会话。如果从一个PDB切换到另一个PDB（通过使用ALTER SESSION SET CONTAINER语句），则仍然为新PDB启用跟踪。您无法在多租户容器数据库（CDB）中为单个PDB启用跟踪;它适用于所有PDB和根。请记住，必须具有ALTER SESSION SET CONTAINER系统特权才能从一个PDB移动到另一个PDB。

  • 跟踪所有数据库会话的事件：在多租户环境中，从根或特定PDB运行ALTER SYSTEM SET EVENTS语句可以跟踪容器数据库中的所有PDB。

禁用当前数据库会话的跟踪事件

您可以使用ALTER SESSION SET EVENTS SQL语句禁用当前数据库会话的Database Vault跟踪。

  1. 以已被授予DV_ADMIN角色和ALTER SESSION系统特权的用户身份登录数据库实例。

    例如：

    sqlplus leo_dvowner输入密码： 密码已连接。
  2. 输入以下两个SQL语句以禁用跟踪：
    ALTER SESSION SET EVENTS'TRACE [DV] OFF'; ALTER SESSION SET EVENTS'47998跟踪名称上下文';

    或者，您也可以使用ALTER SYSTEM语句：

    ALTER SYSTEM SET EVENTS'TRACE [DV] OFF'; ALTER SYSTEM SET EVENTS'47998跟踪名称上下文';

禁用所有数据库会话的跟踪事件

您可以使用ALTER SYSTEM SET EVENTS SQL语句禁用所有数据库会话的Database Vault跟踪。

  1. 以已被授予DV_ADMIN角色和ALTER SYSTEM系统特权的用户身份登录数据库实例。

    例如：

    sqlplus leo_dvowner输入密码： 密码已连接。
  2. 使用“ 禁用当前数据库会话的跟踪事件”中的步骤2中显示的语法输入ALTER SYSTEM SET EVENTS SQL语句。

    例如：

    ALTER SYSTEM SET EVENTS'TRACE [DV] OFF';
  3. 重启数据库。

    例如：

    关机立即启动

可以为所有数据库会话禁用跟踪事件的另一种方法是init.ora下行添加到init.ora文件，然后重新启动数据库：

event =“47998跟踪名称上下文关闭”

确保init.ora文件没有任何冲突的47998行，例如event="47998 trace name context forever, level [1]" 。

在多租户环境中禁用跟踪事件

您应该了解在多租户环境中如何启用跟踪事件。

  • 当前会话的跟踪事件：在多租户环境中，从根或PDB运行ALTER SESSION SET EVENTS SQL语句会禁用当前用户会话的跟踪。如果从一个PDB切换到另一个PDB（通过使用ALTER SESSION SET CONTAINER语句），则仍然会禁用新PDB的跟踪。您无法禁用CDB中单个PDB的跟踪;它适用于所有PDB和根。请记住，必须具有ALTER SESSION SET CONTAINER系统特权才能从一个PDB移动到另一个PDB。

  • 跟踪所有数据库会话的事件：在多租户环境中，从根或特定PDB运行ALTER SYSTEM SET EVENTS语句会禁用对CDB中所有PDB的跟踪。

一般诊断提示

Oracle提供了诊断领域，因子和规则集中的问题的一般提示。

这些指导原则如下：

  • 对于领域保护，请验证用户是否具有可能影响该命令的基础系统或对象特权（直接授予或通过角色授予）。

  • 如果领域授权不起作用，请验证是否正确设置了帐户角色。

  • 对于因子和规则集中使用的PL / SQL表达式，将这些表达式中使用的PL / SQL包函数的EXECUTE特权直接授予帐户，并确定结果是否正确。

  • 使用审核报告来诊断问题。有关更多信息，请参阅Oracle Database Vault审核报告 。

重置DV_OWNER用户密码

您可以使用DV_OWNER备份帐户重置DV_OWNER用户密码。

要重置DV_OWNER用户密码，必须暂时撤消此用户的DV_OWNER角色，重置密码，然后将该角色重新授予用户。
  1. 以DV_OWNER用户帐户的备份用户身份登录数据库实例。

    例如：

    sqlplus dbv_owner_backup输入密码： 密码
  2. 从丢失密码的DV_OWNER用户撤消DV_OWNER角色。

    例如：

    从sec_admin_owen撤消DV_OWNER;
  3. 以已被授予DV_ACCTMGR角色的用户身份进行连接。

    例如：

    CONNECT accts_admin_ace输入密码： 密码
  4. 重置DV_OWNER用户的密码。
    ALTER USER sec_admin_owen IDENTIFIED BY password ;

    请遵循“ Oracle数据库安全指南”中的准则，将password替换为安全的密码。

  5. 作为备份DV_OWNER用户连接。
    CONNECT dbv_owner_backup输入密码： 密码
  6. 将DV_OWNER角色授予DV_OWNER用户。
    授予DV_OWNER至sec_admin_owen;

注意：

确保备份DV_OWNER帐户安全存储，以备再次使用时使用。

重置DV_ACCTMGR用户密码

您可以使用DV_ACCTMGR备份帐户重置DV_ACCTMGR用户密码。

要重置DV_ACCTMGR用户密码，您可以使用备份DV_ACCTMGR帐户重置此用户的密码。
  1. 以DV_ACCTMGR用户帐户的备份用户登录数据库实例。

    例如：

    sqlplus dbv_acctmgr_backup输入密码： 密码
  2. 重置DV_ACCTMGR用户的密码。

    例如：

    ALTER USER accts_admin_ace IDENTIFIED BY password ;

    请遵循“ Oracle数据库安全指南”中的准则，将password替换为安全的密码。

注意：

确保备份DV_ACCTMGR帐户安全存储，以备再次使用时使用。
