父主题： 配置命令规则

关于命令规则

命令规则保护影响一个或多个数据库对象的Oracle数据库SQL语句。

这些语句可以包括SELECT ， ALTER SYSTEM ，数据库定义语言（DDL）和数据操作语言（DML）语句。

要自定义和强制执行命令规则，请将其与规则集关联，规则集是一个或多个规则的集合。命令规则在运行时强制执行。命令规则会影响尝试使用它保护的SQL语句的任何人，无论对象存在的领域如何。

除了基本的Oracle Database DDL和DML语句之外，您还可以使用命令规则来保护各种SQL语句。例如，您可以保护与Oracle闪回技术一起使用的语句。

除了将命令规则与命令关联之外，命令规则还具有以下属性：

  • 命令规则保护的SQL语句

  • 命令规则影响的对象的所有者

  • 命令规则影响的数据库对象

  • 是否启用了命令规则

  • 关联的规则集

命令规则可以分类如下：

  • 具有系统范围的命令规则。 对于此类型，在大多数情况下，您只能为每个数据库实例创建一个命令规则。

  • 特定于模式的命令规则。 特定于模式的命令规则的示例是DROP TABLE语句的命令规则。您只能为每个架构创建一个CONNECT命令规则。

  • 特定于对象的命令规则。 例如，使用命令规则定义中包含的特定表为DROP TABLE语句创建命令规则。

当用户执行受命令规则影响的语句时，Oracle Database Vault会首先检查领域授权。如果未发现域违规并且启用了关联的命令规则，则Database Vault将评估关联的规则集。如果所有规则集都评估为TRUE ，则该语句被授权进一步处理。如果任何规则集的计算结果为FALSE ，则不允许执行该语句并引发命令规则违规。

您可以定义一个命令规则，该命令规则使用CONNECT事件的因子来在通常的步骤 - 用户身份验证过程，因子初始化和Oracle Label Security集成完成后允许或拒绝会话。例如，您可以配置一个命令规则，允许在工作时间之后授权BIZAPP模式中的DDL语句（如CREATE TABLE ， DROP TABLE和ALTER TABLE ，但不能在工作时间授权。

您可以运行在Oracle Database Vault中创建的命令规则的报告。

您无法创建阻止SYS执行SYS拥有的过程的命令规则。

多租户环境中的命令规则

在多租户环境中，您可以在CDB根目录或应用程序根目录中创建公共和本地命令规则。

通用命令规则只能与通用领域，规则集和规则相关联。本地命令规则只能与本地域，规则集和规则相关联。

要将这些命令规则应用于整个多租户环境，必须从CDB根目录或应用程序根目录执行命令规则过程，作为已被授予DVADM或DVOWNER角色的普通用户。在CDB根目录中创建的公共命令规则将应用于该CDB环境中的所有PDB。在应用程序根目录中创建的常用命令规则将仅应用于与此应用程序根目录关联的PDB。要将命令规则传播到与CDB根或应用程序根关联的PDB，必须同步PDB。例如，要将名为saas_sales_app的应用程序根同步到其应用程序PDB：

ALTER PLUGGABLE DATABASE APPLICATION saas_sales_app SYNC;

要将CDB根目录中的公共命令规则同步到PDB：

ALTER PLUGGABLE DATABASE APPLICATION APP $ CDB $ SYSTEM SYNC;

您可以通过查询USER_ROLE_PRIVS数据字典视图来检查用户的角色。要查找有关命令规则的信息，请查询DBA_DV_COMMAND_RULE数据字典视图。

CONNECT命令规则

DBMS_MACADM.CREATE_CONNECT_CMD_RULE过程创建用户特定的CONNECT命令规则。

此类命令规则指定用户，关联的规则集，启用状态，以及多租户环境，执行CONNECT命令规则的位置。您可以启用或禁用CONNECT命令规则，也可以将其设置为使用模拟模式。在模拟模式下，对命令规则的违反将记录在指定的日志表中，其中包含足以描述错误的信息，例如使用的用户名或SQL语句。

在多租户环境中，您可以在应用程序根目录或特定PDB中创建CONNECT命令规则。关联的规则集必须与CONNECT命令规则一致：如果CONNECT命令规则位于应用程序根目录中，则规则集和规则也必须位于应用程序根目录中。您作为普通用户从CDB根运行CONNECT命令规则过程。如果CONNECT命令规则对于可插拔数据库（PDB）是本地的，则必须在该PDB中运行CONNECT命令规则创建命令，并且规则集和规则必须是本地的。

以下示例显示了CONNECT命令规则定义，该定义为HR用户创建本地的已启用CONNECT命令规则。与此命令规则关联的规则集是当前PDB的本地规则集。

BEGIN DBMS_MACADM.CREATE_CONNECT_COMMAND_RULE（rule_set_name =>'Enabled'，user_name =>'HR'，enabled => DBMS_MACUTL.G_YES，scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

ALTER SESSION和ALTER SYSTEM命令规则

您可以创建不同类型的ALTER SESSION和ALTER SYSTEM命令规则，为这些SQL语句提供细粒度控制。

创建这些类型的命令规则的过程如下：

  • DBMS_MACADM.CREATE_COMMAND_RULE创建ALTER SESSION和ALTER SYSTEM命令规则，这些规则使用来自相应SQL语句的子句，例如ADVISE ， CLOSE DATABASE LINK ， COMMIT IN PROCEDURE ， SET for ALTER SESSION ，或ARCHIVE_LOG ， CHECK DATAFILES ， CHECKPOINT和SET for ALTER SYSTEM 。

  • DBMS_MACADM.CREATE_SESSION_EVENT创建特定于ALTER SESSION SET EVENTS SQL语句的命令规则

  • DBMS_MACADM_CREATE_SYSTEM_EVENT创建特定于ALTER SYSTEM SET EVENTS SQL语句的命令规则。

要创建这些命令规则，请使用相应的Database Vault过程在创建语句中指定子句的参数（如果适用）。如果ALTER SESSION或ALTER SYSTEM命令规则使用SET EVENTS设置，则可以使用特殊参数指定事件，组件和操作。

例如，对于ALTER SYSTEM命令规则，您可以从ALTER SYSTEM SQL语句指定SECURITY子句及其RESTRICTED SESSION参数。要指定RESTRICTED SESSION是TRUE还是FALSE ，您必须创建Database Vault规则和规则集，该规则集可以测试此序列号的有效性。

要了解此概念的工作原理，请首先创建以下规则和规则集，这些规则和规则集旨在检查RESTRICTED SESSION参数是否设置为TRUE ：

EXEC DBMS_MACADM.CREATE_RULE（'RESTRICTED SESSION TRUE'，'UPPER（PARAMETER_VALUE）=''TRUE'''）; BEGIN DBMS_MACADM.CREATE_RULE_SET（rule_set_name =>'检查RESTRICTED SESSION为TRUE'，description =>'检查受限会话是否为真'，启用=> DBMS_MACUTL.G_YES，eval_options => DBMS_MACUTL.G_RULESET_EVAL_ALL，audit_options => DBMS_MACUTL.G_RULESET_AUDIT_FAIL + DBMS_MACUTL .G_RULESET_AUDIT_SUCCESS，fail_options => DBMS_MACUTL.G_RULESET_FAIL_SILENT，fail_message =>'RESTRICTED SESSION不为TRUE'，fail_code => 20461，handler_options => DBMS_MACUTL.G_RULESET_HANDLER_FAIL，handler =>''，is_static => false）;结束; / EXEC DBMS_MACADM.ADD_RULE_TO_RULE_SET（检查RESTRICTED SESSION为TRUE'，'RESTRICTED SESSION TRUE'）;

使用规则和规则集，您就可以创建一个ALTER SYSTEM命令规则来检查RESTRICTED SESSION参数：

BEGIN DBMS_MACADM.CREATE_COMMAND_RULE（command =>'ALTER SYSTEM'，rule_set_name =>'Check RESTRICTED SESSION为TRUE'，object_owner =>'％'，object_name =>'％'，enabled => DBMS_MACUTL.G_YES，clause_name =>'SECURITY '，parameter_name =>'RESTRICTED SESSION'，scope => DBMS_MACUTL.G_SCOPE_LOCAL）;结束; /

在这个例子中：

  • rule_set_name检查RESTRICTED SESSION是设置为TRUE还是FALSE 。在多租户环境中，必须在与命令规则相同的位置创建规则集和规则：在应用程序根目录中或在PDB中本地。

  • 对于这种ALTER SESSION或ALTER SYSTEM命令规则， object_owner和object_name必须始终设置为% 。

  • enabled允许您启用或禁用命令规则，或使用模拟模式将违反命令规则的日志记录到指定的日志表。日志数据描述错误，例如使用的用户名或SQL语句。

  • clause_name指定ALTER SYSTEM SQL语句的SECURITY子句

  • parameter_name指定SECURITY子句中的RESTRICTED SESSION参数

  • scope将命令规则设置为当前PDB的本地规则。关联的规则集和规则也必须是当前PDB的本地规则。如果要在应用程序根目录中创建命令规则，则作为普通用户，可以将scope设置为DBMS_MACUTL.G_SCOPE_COMMON并从应用程序根目录运行该过程（及其随附的规则集和规则创建过程）。

创建命令规则

您可以在Oracle Database Vault Administrator中创建命令规则。

  1. 以已授予DV_OWNER或DV_ADMIN角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 在“管理”页面的“数据库Vault组件”下，单击“ 命令规则” 。
  3. 在“命令规则”页面中：

    • 要创建新的命令规则，请单击“ 创建”以显示“创建命令规则”页面。



      插图描述create_command_rule122.png
  4. 在“创建命令规则”页面中，输入以下设置：

    • 命令：选择要为其创建命令规则的SQL语句或操作。此属性是必需的。

      如果计划为统一审计策略对象创建命令规则，请确保指定AUDIT POLICY或NOAUDIT POLICY ，而不是AUDIT或NOAUDIT 。

    • 状态：选择“已启用” ，“ 已禁用 ”或“ 模拟” ，这将在运行时应用于命令规则。此属性是必需的。

    • 适用对象所有者：从列表中选择命令规则影响的对象的所有者。您可以使用通配符%来选择所有所有者。（但是，您不能使用带有文本的通配符，例如EM%来选择名称以EM开头的所有所有者。）对于在特定模式中的对象上运行的所有SQL语句，此属性是必需的。有关受支持的SQL语句的列表，请参阅可由命令规则保护的 SQL语句。

      请注意， SELECT ， INSERT ， UPDATE ， DELETE和EXECUTE语句不允许选择所有（％）或SYS和DVSYS模式。

    • 适用对象名称：输入命令规则影响的数据库对象的名称，或指定％以选择所有数据库对象，包括表，过程，视图，统一审计策略等。如果从“对象所有者”列表中选择了对象所有者，则此属性是必需的。

      您可以对命令规则影响的对象运行Oracle Database Vault报告。有关更多信息，请参阅命令规则相关报告和数据字典视图 。

    • 规则集：从列表中，选择要与命令规则关联的规则集。此属性是必需的。

      如果规则集的计算结果为true，则SQL语句将成功。如果计算结果为false，则语句将失败，然后Oracle Database Vault会引发命令规则违规。（您可以使用Oracle Database Vault Reports中讨论的命令规则配置问题报告来跟踪规则违规。）与规则集关联的任何审核和自定义事件处理都作为命令规则处理的一部分发生。

      有关规则集的详细信息，请参阅配置规则集。

  5. 单击确定 。

命令规则如何工作

命令规则遵循一组步骤来检查其关联的组件。

Realms Work如何描述当数据库帐户发出影响领域内对象的SELECT ，DDL或DML语句时会发生什么。

发出SELECT ，DDL或DML语句时会发生以下操作：

  1. Oracle Database Vault会查询需要应用的所有命令规则。

    对于SELECT ，DDL和DML语句，可能会应用多个命令规则，因为对象所有者和对象名称支持通配符表示法。

    您可以将规则集与命令规则和领域授权相关联。Oracle Database Vault首先评估领域授权规则集，然后评估适用于正在评估的命令类型的规则集。

  2. 对于适用的每个命令规则，Oracle Database Vault会评估其关联的规则集。

  3. 如果任何适用的命令规则的关联规则集返回false或错误，则Oracle Database Vault会阻止该命令执行。否则，该命令被授权进一步处理。针对审计和事件处理程序的规则集的配置指示发生的审计或自定义处理。

    命令规则覆盖对象权限。也就是说，如果对象受命令规则保护，即使对象的所有者也无法访问该对象。您可以禁用命令规则或命令的规则集。如果禁用命令规则，则命令规则不会执行它旨在处理的检查。如果禁用规则集，则规则集始终计算为TRUE 。但是，如果要禁用特定命令的命令规则，则应禁用命令规则，因为规则集可能与其他命令规则或域授权相关联。

设计命令规则的指南

Oracle提供了设计命令规则的指南。

  • 创建更精细的命令规则，因为它们更容易维护。

    例如，如果要阻止在特定模式对象上发生SELECT语句，则设计多个命令规则以停止那些特定模式对象上的SELECT语句，而不是创建一般命令规则来阻止模式级别中的SELECT语句。

  • 在为CONNECT事件设计规则时，请注意包含不会无意中锁定任何所需用户连接的逻辑。如果任何帐户被意外锁定，请询问已被授予DV_ADMIN或DV_OWNER角色的用户登录并更正导致锁定问题的规则。CONNECT命令规则不适用于具有DV_OWNER和DV_ADMIN角色的用户。这可以防止不正确配置的CONNECT命令规则导致完全锁定。

    如果帐户已被锁定，您可以禁用Oracle Database Vault，更正导致锁定问题的规则，然后重新启用Oracle Database Vault。即使禁用了Oracle Database Vault，您仍然可以使用Database Vault Administrator和Database Vault PL / SQL包。有关禁用和重新启用Database Vault的说明，请参阅禁用和启用Oracle Database Vault。

  • 有时，您必须暂时放松管理任务的已启用命令规则。而不要禁用命令规则，有安全管理器（与该帐户DV_ADMIN或DV_OWNER登录，设置规则设置为启用 ，启用审核成功或失败，名为默认规则集角色）启用，并设置任务完成后，命令规则返回其原始规则集。（请注意，在统一审核环境中，此设置不起作用。相反，您必须创建统一的审核策略。“Oracle数据库安全指南”介绍了如何为Database Vault创建统一的审核策略。）

  • 在设计命令规则时，请小心考虑自动化过程，例如备份，这些过程可能会被无意中禁用。您可以通过创建允许在一系列Oracle Database Vault因素为真时（例如，正在使用的程序），正在使用的帐户或客户端所在的计算机或网络上的命令来执行这些任务的帐户程序正在运行。

  • 您可以使用模拟模式测试命令规则的开发阶段，该模式启用命令规则，但会将有关它的详细信息写入日志文件。

命令规则如何影响性能

命令规则的性能取决于与命令规则关联的规则集中的规则的复杂性。

例如，假设规则集调用PL / SQL函数，该函数需要5秒才能运行。在这种情况下，使用该规则集的命令规则将花费5秒来授予对命令语句运行的访问权限。

您可以通过运行Oracle Enterprise Manager（包括Oracle Enterprise Manager云控制，默认情况下随Oracle Database安装），自动工作负载存储库（AWR）和TKPROF等工具来检查系统性能。

也可以看看：

  • Oracle数据库性能调优指南 ，了解如何监控数据库性能

  • Oracle数据库SQL调优指南，用于监控单个SQL和PL / SQL语句的性能
