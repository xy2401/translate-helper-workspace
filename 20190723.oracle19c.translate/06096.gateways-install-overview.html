1 Oracle Database Gateway安装概述

以下主题描述了在Microsoft Windows（64位）上安装Oracle数据库网关，以及在安装软件之前应考虑的问题。

网关安装配置

您可以使用以下任一配置安装Oracle Database Gateway：

  1. 在与现有Oracle数据库相同的计算机上，但在不同的Oracle主目录中。

  2. 在没有Oracle数据库的系统上。

  3. 与Oracle数据库在同一台计算机上，位于同一Oracle主目录中。请注意，在这种情况下，Oracle数据库和网关必须处于相同的发行版级别。

网关安装方法

以下是安装Oracle数据库网关的安装方法：

  • 交互式安装方法

  • 使用响应文件的自动安装方法

交互式安装方法

使用交互方法安装Oracle Database Gateway时，Oracle Universal Installer会显示一系列屏幕，使您可以指定所有必需的信息。

使用响应文件的自动安装方法

通过创建响应文件并在启动Oracle Universal Installer时指定此文件，可以自动执行部分或全部Oracle Database Gateway安装。

有关这些模式以及如何使用响应文件完成安装的更多信息，请参阅使用响应文件进行非交互式安装 。

硬件和软件认证

本指南发布时，本安装指南中包含的特定于平台的硬件和软件要求是最新的。但是，由于在本指南发布后可能会对新平台和操作系统软件版本进行认证，因此请查看My Oracle Support Web站点上的认证表，以获取最新的认证硬件平台和操作系统版本列表。My Oracle Support Web站点位于以下Web站点：

https://support.oracle.com

多个Oracle Homes支持

该产品支持多个Oracle家庭。这意味着您可以在同一系统上，在不同的Oracle主目录中多次安装此版本或以前版本的软件。

在具有现有Oracle安装的系统上安装软件

您必须在新的Oracle主目录中安装此产品。您无法将一个Oracle Database Gateways版本的产品安装到另一个版本的Oracle主目录中。例如，您无法将12 c Release 1（12.1）软件安装到现有的Oracle 10 g R2 Oracle主目录中。如果尝试在包含早期Oracle发行版软件的Oracle主目录中安装此发行版，则安装将失败。

如果每个安装都安装在单独的Oracle主目录中，则可以在同一系统上多次安装此版本。

将Windows用户帐户用作Oracle Home用户

使用Windows，您可以登录具有管理员权限的用户来安装Oracle数据库软件。您还可以在安装期间指定Oracle Home User（基于低权限，非管理用户帐户）。

以下是Windows用户帐户：

  • Windows本地用户帐户

  • Windows域用户帐户

  • Windows托管服务帐户（MSA）

  • Windows内置帐户

也可以看看：

“ 适用于Microsoft Windows的Oracle数据库平台指南 ”中的“在Windows上使用Oracle Home用户”

支持Windows组托管服务帐户和虚拟帐户

从Oracle Database 12 c第2版（12.2）开始，组托管服务帐户（gMSA）和虚拟帐户允许您安装Oracle数据库，并创建和管理没有密码的数据库服务。用户名不会出现在登录屏幕上。gMSA是域级帐户，域中的多个服务器可以使用此帐户来运行服务。

虚拟帐户是自动管理的。

另请参阅： 适用于Microsoft Windows的Oracle®数据库平台指南以获取更多信息

Oracle数据库网关升级

Oracle数据库网关不支持升级。

访问安装软件

您可以使用以下方法之一访问Oracle Database Gateway软件：

  • 从OTN网站下载Oracle软件

  • 复制Oracle软件

从OTN网站下载Oracle软件

您可以从OTN下载安装文件并将其解压缩到系统上的本地目录。

要下载安装文件：

  1. 使用任何浏览器访问OTN软件下载页面：
    http://www.oracle.com/technetwork/indexes/downloads/index.html
  2. 导航到要安装的产品的每个下载页面。
  3. 在每个下载页面上，通过添加每个所需文件的文件大小来标识所需的磁盘空间。文件大小旁边列出了文件大小。
  4. 选择具有足够可用空间的文件系统来存储和扩展文件。在大多数情况下，可用磁盘空间必须至少是每个压缩文件大小的两倍。
  5. 在刚刚选择的文件系统上，为计划安装的每个产品（例如Dg_1创建父目录，以保存安装目录。
  6. 将所有安装文件下载到刚刚创建的目录中。
  7. 验证您下载的文件大小与OTN上的相应文件大小相同。
  8. 解压缩刚刚创建的每个目录中的文件。
  9. 解压缩所需的安装文件后，请转至“ 运行Oracle Universal Installer”部分。

复制Oracle软件

在安装Oracle Database Gateway之前，您可能希望将软件复制到本地目录。这使安装过程运行得更快。

要将安装介质的内容复制到本地目录：

  1. 在硬盘上创建一个目录。例如：
    d：\安装\ DISK1
  2. 将安装介质的内容复制到刚刚创建的目录中。
  3. 复制完所有必需的安装文件后，请转至运行Oracle Universal Installer部分 。

运行Oracle Universal Installer

在大多数情况下，您使用Oracle Universal Installer提供的图形用户界面（GUI）来安装网关。但是，您也可以使用Oracle Universal Installer完成非交互式安装，而无需使用GUI。

有关非交互式安装和其他高级安装主题的信息，请参阅使用非交互式安装的响应文件

启动安装程序并安装软件，如下所示：

  1. 如果要从本地目录进行安装，请双击位于为下载或复制的安装文件创建的目录中的setup.exe 。

  2. 从安装介质安装时，将自动显示自动运行屏幕。如果未显示自动运行屏幕，则：

    1. 从“ 开始”菜单中，选择“运行” 。

    2. 输入以下内容：

      DRIVE _ LETTER ：\ autorun \ autorun.exe

      在“自动运行”屏幕中，选择“ 安装/卸载产品” 。

  3. 使用以下准则完成安装：

    • 按照安装程序窗口中显示的说明进行操作。如果需要其他信息，请单击“ 帮助” 。

    • 如果在安装或链接软件时遇到错误，请参阅Oracle Database Gateway故障排除以获取有关故障排除的信息。

  4. 安装完成后，单击“ 退出” ，然后单击“ 是”退出安装程序。

在群集环境中安装和配置

Oracle Database Gateway可以安装在现有Oracle数据库主目录中，也可以安装在所有节点上的单独网关主目录中。Oracle OUI可以在所有节点或选择性节点上安装Oracle Database Gateway。

Oracle建议不要使用Oracle Database Gateway主目录中的侦听器。而是在Grid home中配置监听器。默认情况下，在群集配置期间创建本地侦听器，该侦听器在网格基础结构主目录之外运行，并侦听节点虚拟IP（VIP）的指定端口（默认为1521）。

支持单客户端访问名称（SCAN）

Oracle Database 11g第2版及更高版本的客户端使用支持单客户端访问名称（SCAN）连接到数据库。它为连接到Oracle RAC的客户端提供单一名称，即使您在群集中添加或删除节点，该名称也不会在群集的整个生命周期中发生更改。与SCAN连接的客户端可以使用简单的连接字符串，例如瘦JDBC URL或EZConnect，并实现负载平衡和客户端连接故障转移。

除了三个SCAN侦听器（每个虚拟IP地址一个）之外，在托管数据库实例的每个节点上都有一个节点侦听器。使用两层侦听器（SCAN侦听器和节点侦听器）的目的是分离Oracle RAC中侦听器的两个功能，首先是负载平衡连接，其次是产生和遗留会话。SCAN侦听器将接收来自客户端的连接请求，由GNS（网格命名服务）随机分发。然后，SCAN侦听器将使用负载平衡度量标准将请求智能地重定向到最能够提供所请求服务的节点上的节点侦听器。数据库实例向SCAN侦听器注册为远程侦听器，并将节点侦听器注册为本地侦听器。

无法使用SCAN配置Oracle数据库网关，SCAN是数据库连接到网关的单个名称。有两个原因。网关不适用于远程侦听器。与您可以指定REMOTE_LISTENER以设置为SCAN侦听器的数据库不同，Oracle Database Gateway中不支持它。这对于SCAN侦听器将连接路由到节点侦听器至关重要。其次，网关不向群集注册，以便将其作为群集资源进行管理。

对于网关，当Oracle数据库和Oracle数据库网关在同一群集上运行时，SCAN不是很有用。可以在安装数据库的每个节点上安装和配置Oracle数据库网关，并且可以配置数据库，使每个实例连接到在同一节点上运行的网关。

本地监听器

应使用本地侦听器配置Oracle Database Gateway服务。它是本地监听器产生网关进程。这意味着监听器应该知道要生成哪个网关进程。使用Grid infrastructure home中的listener.ora添加Gateway SID。如果数据库的SCAN侦听器已在该节点上运行，则可以使用相同的listener.ora文件来配置本地侦听器。

负载平衡和透明应用程序故障转移（TAF）

Oracle Database Gateway本身不支持客户端（使用数据库主目录中的tnsnames）或服务器端（使用SCAN）负载平衡。网关级别的负载平衡不适用，因为网关进程当前专用于单个会话。但是，通过为每个数据库实例关联不同的Oracle Database Gateway实例，可以实现节点级负载平衡，即通过数据库上的负载平衡选择负载最小的节点。

Oracle Database Gateway支持连接故障转移功能。如果为Gateway配置客户端连接负载平衡，则其工作方式与故障转移类似。

每当发生数据库故障转移时，即会话从一个数据库实例迁移到另一个数据库实例（在新节点上），迁移的会话将使用该新节点中的网关实例。

默认情况下，Oracle Call Interface（OCI）应用程序可以使用三种类型的Oracle Net故障转移功能：

  • session ：设置为故障转移会话。如果用户连接丢失，则会自动为备份上的用户创建新会话。此类故障转移不会尝试恢复选择操作。

  • select ：设置为允许具有打开游标的用户在失败后继续获取它们。但是，此模式涉及正常选择操作中客户端的开销。

  • none ：这是默认值。没有使用故障转移功能。这也可以明确指定，以防止发生故障转移。

要使故障转移起作用，需要使用多个侦听器地址配置Database home中的tnsnames.ora 。

如果实例在连接后失败，则TAF应用程序将故障转移到另一个节点的侦听器，并保留正在进行的任何SELECT语句。

在以下用于负载平衡的tnsnames.ora示例中，仅用作故障转移，仅当gateway1-server上的网关不可用时，数据库才连接到主机gateway2-server的网关：

dg4sybs.us.example.com =（DESCRIPTION =（LOAD_BALANCE = on）（FAILOVER = on）（ADDRESS =（PROTOCOL = tcp）（HOST = gateway1-server）（PORT = 1521））（ADDRESS =（PROTOCOL = tcp） （HOST = gateway2-server）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = dg4sybs.us.example.com）（HS = OK）（FAILOVER_MODE =（TYPE = select）（METHOD = basic））））
