了解SQLRF中的图形语法图

在你开始之前

这个15分钟的教程向您展示了如何阅读SQL参考手册和其他Oracle数据库手册中的图形语法图。

背景

在Backus-Naur Form（BNF）中指定了SQL语句的语法规则和功能。BNF只是一种描述语法的方法。

每个图形语法图都有一个文本描述，显示为图形下方的链接。

你需要什么

  • 理解BNF表示法中使用的符号。请参阅最后提供的链接。

BNF pdb_refresh_mode_clause的规范

pdb_refresh_mode_clause的BNF规范：

刷新模式（（手动）|（每次“refresh_interval”（HOURS | MINUTES））|（NONE））

如何阅读BNF规范

您从左到右阅读BNF表示。表示遵循以下符号规则：

  1. Oracle关键字和命令是大写的，并按原样指定。在pdb_refresh_mode_clause中，Oracle关键字是REFRESH，MODE，EVERY，HOURS，MINUTES，NONE和SWITCHOVER。
  2. 括号用于对项目进行分组。分组使语法更易于阅读。例如，MANUAL，EVERY和NONE选项分组在括号内。选项HOURS和MINUTES的分组清楚地表明它们属于每个选项。
  3. 方括号（[]）包含可选项。
  4. 由竖线分隔的分组中的项目意味着在给定时间仅需要其中一个项目。
  5. 用户定义的输入（如参数和子句）用双引号括起来。例如，pdb_refresh_mode_clause中的“refresh_interval”。
  6. 标点符号用双引号括起来。双引号表示标点符号被解释为字符串。
  7. 标点符号前面的正斜杠字符（“/”）表示标点符号不会被解释为字符串。例如，在句法元素之后的/“，”表示句法元素可以重复。

阅读pdb_refresh_mode_clause的语法图

语法图沿箭头方向从左向右读取。规则：

  1. Oracle关键字和命令在矩形内部以大写形式显示。它们的输入应完全如矩形所示。在pdb_refresh_mode_clause中，这些是REFRESH，MODE，EVERY，HOURS，MINUTES，NONE和SWITCHOVER。
  2. 所需的关键字如REFRESH和MODE单独出现在主路径上，即您正在行进的水平线。
  3. 所需参数MANUAL，EVERY和NONE出现在与主路径相交的备用路径的垂直列表中。您必须选择其中一个选项才能完成该声明。
  4. EVERY选项需要参数refresh_interval。它在椭圆形内呈小写字母。

以下陈述有效：

  • 刷新模式手册
  • 刷新模式每个“refresh_interval”小时
  • 刷新模式每次“refresh_interval”MINUTES
  • 刷新模式无

阅读语法图 - 标点符号和循环

标点符号，运算符和分隔符包含在BNF规范中的双引号内，并显示在图形语法图中的圆圈内。

示例：ALTER TABLE语句的enable_disable_clause。

BNF规格：

（ENABLE | DISABLE）[VALIDATE | NOVALIDATE]（（UNIQUE“（”“column”/“，”“）”）| PRIMARY KEY | CONSTRAINT“constraint_name”）[“using_index_clause”] [“exceptions_clause”] [CASCADE] [ （KEEP | DROP）INDEX]

注意逗号前的正斜杠，这意味着它之前的项可以重复。

图形语法图的文本描述

HTML输出中的每个图形语法图都在图形语法图下方的链接中包含图形的文本描述。

文本描述与BNF规范类似，但有两点不同：

1。大括号包含物品，其中只需要一件物品。

pdb_refresh_mode_clause的文字说明：

刷新模式{手动|每个refresh_interval {HOURS | MINUTES} |没有 }

2。在句法元素之后的省略号（“...”）表示该元素可以重复。

enable_disable_clause的文字说明：

{ENABLE | DISABLE} [VALIDATE | NOVALIDATE] {UNIQUE（列[，列] ...）|主要关键| CONSTRAINT constraint_name} [using_index_clause] [exceptions_clause] [CASCADE] [{KEEP | DROP} INDEX]
