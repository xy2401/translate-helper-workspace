2.1什么是数据泵输出？

Data Pump Export（以下简称为Export）是一个实用程序，用于将数据和元数据卸载到一组称为转储文件集的操作系统文件中。

转储文件集只能由Data Pump Import实用程序导入。转储文件集可以在同一系统上导入，也可以移动到另一个系统并加载到那里。

转储文件集由一个或多个包含表数据，数据库对象元数据和控制信息的磁盘文件组成。文件以专有的二进制格式编写。在导入操作期间，数据泵导入实用程序使用这些文件来查找转储文件集中的每个数据库对象。

由于转储文件是由服务器而不是客户端编写的，因此数据库管理员（DBA）必须创建目录对象，以定义写入文件的服务器位置。

通过数据泵导出，您可以指定作业应移动数据和元数据的子集（由导出模式确定）。这是使用数据过滤器和元数据过滤器完成的，这些过滤器通过导出参数指定。

2.2.2.1完整模式

您可以使用数据泵通过使用FULL参数执行完整数据库导出。

在完整数据库导出中，将卸载整个数据库。此模式要求您具有DATAPUMP_EXP_FULL_DATABASE角色。

在完全模式导出期间使用可传输选项

如果指定TRANSPORTABLE=ALWAYS参数以及FULL参数，则Data Pump将执行完全可传输导出。完全可传输导出导出创建数据库完整副本所需的所有对象和数据。使用混合的数据移动方法：

  • 驻留在可传输表空间中的对象只将其元数据卸载到转储文件集中;将数据文件复制到目标数据库时，数据本身会被移动。必须复制的数据文件列在导出操作的日志文件末尾。

  • 驻留在非可传输表空间中的对象（例如， SYSTEM和SYSAUX ）使用直接路径卸载和外部表将其元数据和数据卸载到转储文件集中。

执行完全可运输的出口具有以下限制：

  • 执行完全可传输导出的用户需要DATAPUMP_EXP_FULL_DATABASE权限。

  • 执行导出的用户的默认表空间不能设置为正在传输的表空间之一。

  • 如果要导出的数据库包含加密表空间或包含加密列的表（透明数据加密（TDE）列或SecureFiles LOB列），则还必须提供ENCRYPTION_PASSWORD参数。

  • 如果源数据库中存在加密表空间，则源数据库和目标数据库必须位于具有相同字节序的平台上。

  • 如果源平台和目标平台具有不同的字节顺序，则必须转换正在传输的数据，使其采用目标平台的格式。您可以使用DBMS_FILE_TRANSFER包或RMAN CONVERT命令来转换数据。

  • 完全可传输的导出不可重新启动。

  • 具有存储选择用于导出的所有对象必须将其所有存储段完全置于管理，不可传输的表空间（ SYSTEM/SYSAUX ）内，或者完全位于用户定义的可传输表空间内。单个对象的存储不能跨越两种表空间。

  • 使用完全可传输导出通过网络传输数据库时，如果审计跟踪信息本身存储在用户定义的表空间中，则无法对存储在管理表空间（例如SYSTEM和SYSAUX ）中的表启用审计。

  • 如果源数据库和目标数据库都在运行Oracle Database 12 c ，那么要执行完全可传输导出，Data Pump VERSION参数必须至少设置为12.0。或者COMPATIBLE数据库初始化参数必须至少设置为12.0或更高版本。

运行版本11.2.0.3的源数据库支持完全可传输导出。为此，请将Data Pump VERSION参数设置为至少12.0，如以下语法示例所示，其中user_name是执行完全可传输导出的用户：

> expdp user_name FULL = y DUMPFILE = expdat.dmp DIRECTORY = data_pump_dir TRANSPORTABLE = always VERSION = 12.0 LOGFILE = export.log

2.2.2.2模式模式

您可以使用SCHEMAS参数指定使用Data Pump导出模式。架构导出是默认的导出模式。

如果您具有DATAPUMP_EXP_FULL_DATABASE角色，则可以指定模式列表，可选地包括模式定义本身以及这些模式的系统特权授予。如果您没有DATAPUMP_EXP_FULL_DATABASE角色，则只能导出自己的架构。

SYS架构不能用作导出作业的源架构。

除非在要导出的模式列表中也指定了引用的模式，否则不会导出跨模式引用。例如，不会导出在其中一个指定模式中的表上定义但位于未明确指定的模式中的触发器。此外，不会导出指定模式中的表所依赖的外部类型定义。在这种情况下，期望在导入时类型定义已存在于目标实例中。

2.2.2.3表格模式

您可以使用Data Pump通过使用TABLES参数指定表来执行表模式导出。

在表模式下，仅卸载指定的一组表，分区及其依赖对象。创建表所需的任何对象（例如拥有的模式或列的类型）必须已存在。

如果使用TABLES参数指定TRANSPORTABLE=ALWAYS参数，则仅卸载对象元数据。要移动实际数据，请将数据文件复制到目标数据库。这导致更快的导出时间。如果要在版本或平台之间移动数据文件，则Oracle Recovery Manager（RMAN）需要处理数据文件。

您必须具有DATAPUMP_EXP_FULL_DATABASE角色才能指定不在您自己的架构中的表。请注意，列的类型定义不会以表格模式导出。期望在导入时类型定义已存在于目标实例中。此外，与模式导出一样，不会导出跨模式引用。

要恢复表和表分区，还可以使用RMAN备份和RMAN RECOVER TABLE命令。在此过程中，RMAN创建（并可选地导入）包含已恢复对象的数据泵导出转储文件。有关跨平台传输数据的详细信息，请参阅“ Oracle数据库备份和恢复指南” 。

执行表模式导出具有以下限制：

  • 使用带有TABLES参数的TRANSPORTABLE=ALWAYS参数时，如果要导出的表包含加密列（透明数据加密（TDE）列或SecureFiles LOB列），则还必须使用ENCRYPTION_PASSWORD参数。

2.2.2.4表空间模式

您可以使用Data Pump通过使用TABLESPACES参数指定表来执行表空间导出。

2.2.2.5可传输表空间模式

您可以使用Oracle Data Pump通过使用TRANSPORT_TABLESPACES参数执行可传输的表空间导出。

在可传输表空间模式下，仅导出指定的一组表空间中的表（及其依赖对象）的元数据。表空间数据文件在单独的操作中复制。然后，执行可传输的表空间导入以导入包含元数据的转储文件，并指定要使用的数据文件。

可传输表空间模式要求指定的表完全自包含。也就是说，在表空间集中定义的所有表（及其索引）的所有存储段也必须包含在集合中。如果存在自包含违规，则Export会在不实际执行导出的情况下识别所有问题。

将导出和导入指定表空间中的表列的类型定义。拥有这些类型的模式必须存在于目标实例中。

停止后，无法重新启动可传输的表空间导出。而且，它们的并行度不能大于1。

注意：

您无法导出可传输的表空间，然后将它们导入到较低版本级别的数据库中。目标数据库必须与源数据库处于相同或更高版本级别。

使用Oracle Data Pump执行可传输表空间导出具有以下限制：

  • 如果要导出的任何表空间包含带有加密列的表，透明数据加密（TDE）列或SecureFiles LOB列，则还必须提供ENCRYPTION_PASSWORD参数。

  • 如果导出的任何表空间都是加密的，那么使用ENCRYPTION_PASSWORD是可选的，但建议使用。如果在这种情况下省略ENCRYPTION_PASSWORD ，则会显示以下警告消息：

    ORA-39396：警告：使用无密码的可传输选项导出加密数据

    此警告指出，为了成功导入此类可传输表空间作业，目标数据库钱包必须包含执行导出时源数据库中使用的相同数据库主密钥的副本。在导出和导入期间使用ENCRYPTION_PASSWORD参数可以消除此要求。

2.2.3网络注意事项

启动Data Pump Export实用程序时，可以在连接字符串中指定连接标识符。此标识符可以指定与当前Oracle系统ID（SID）标识的当前实例不同的数据库实例。

连接标识符可以是Oracle * Net连接描述符或映射到连接描述符的网络服务名称（通常在tnsnames.ora文件中定义）。使用连接标识符要求您运行Oracle Net Listener（要启动默认侦听器，请输入lsnrctl start ）。以下是此类连接的示例，其中inst1是连接标识符：

expdp hr @ inst1 DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp TABLES =员工

然后导出会提示您输入密码：

密码： 密码

本地Export客户端连接到由连接标识符inst1 （网络服务名称）定义的数据库实例，从inst1检索数据，并将其写入hr.dmp上的转储文件inst1 。

启动“导出”实用程序时指定连接标识符与使用NETWORK_LINK参数执行导出操作不同。当您启动导出操作并指定连接标识符时，本地导出客户端将连接到由连接标识符标识的数据库实例，从该数据库实例检索数据，并将其写入该数据库实例上的转储文件集。然而，当您使用NETWORK_LINK参数执行导出时，将使用数据库链接执行导出。（数据库链接是两个物理数据库服务器之间的连接，允许客户端将它们作为一个逻辑数据库进行访问。）

2.3导出操作期间的过滤

Data Pump Export提供数据和元数据过滤功能。此功能可帮助您限制导出的信息类型。

  • 数据过滤器
    特定于数据的过滤是通过QUERY和SAMPLE参数实现的，这些参数指定了要导出的表行的限制。
  • 元数据过滤器
    元数据过滤通过EXCLUDE和INCLUDE参数实现。EXCLUDE和INCLUDE参数是互斥的。

父主题： 数据泵导出

2.3.1数据过滤器

特定于数据的过滤是通过QUERY和SAMPLE参数实现的，这些参数指定了要导出的表行的限制。

由于元数据过滤，数据过滤也可以间接发生，元数据过滤可以包括或排除表对象以及任何关联的行数据。

每个数据过滤器可以在作业中的每个表中指定一次。如果使用相同名称的不同过滤器同时应用于特定表和整个作业，则为特定表提供的过滤器参数优先。

父主题： 导出操作期间的筛选

2.3.2元数据过滤器

元数据过滤通过EXCLUDE和INCLUDE参数实现。EXCLUDE和INCLUDE参数是互斥的。

元数据过滤器标识要在“导出”或“导入”操作中包含或排除的一组对象。例如，您可以请求完整导出，但不包括包规范或包体。

要正确使用过滤器并获得预期的结果，请记住已识别对象的相关对象与标识的对象一起处理。 例如，如果过滤器指定要在操作中包含索引，则还将包括该索引的统计信息。同样，如果过滤器排除了表，则过滤器也会排除表上的索引，约束，授权和触发器。

如果为对象类型指定了多个过滤器，则会对它们应用隐式AND操作。也就是说，与作业相关的对象必须传递应用于其对象类型的所有过滤器。

可以在作业中多次指定相同的元数据过滤器名称。

要查看有效的对象类型的列表，查询以下观点： DATABASE_EXPORT_OBJECTS全模式， SCHEMA_EXPORT_OBJECTS的架构模式， TABLE_EXPORT_OBJECTS表和表空间模式。OBJECT_PATH列中列出的值是有效的对象类型。例如，您可以执行以下查询：

2.4导出命令行模式中可用的参数

本节介绍Data Pump Export的命令行模式中可用的参数。

    了解如何在命令行模式下使用Oracle Data Pump Export参数，包括区分大小写，引号，转义字符以及有关如何使用示例的信息。
  • ABORT_STEP
    数据泵导出命令行实用程序ABORT_STEP参数在初始化后停止作业。
  • ACCESS_METHOD
    Data Pump Export命令行实用程序ACCESS_METHOD参数指示Export使用特定方法卸载数据。
  • 连接
    Data Pump Export命令行实用程序ATTACH参数将客户机会话附加到现有导出作业，并自动将您置于交互式命令界面中。
  • 簇
    Data Pump Export命令行实用程序CLUSTER参数确定Data Pump是否可以在其他Oracle RAC实例上使用Oracle RAC，资源和启动工作程序。
  • 压缩
    Data Pump Export命令行实用程序COMPRESSION参数指定在写入转储文件集之前要压缩的数据。
  • COMPRESSION_ALGORITHM
    Data Pump Export命令行实用程序COMPRESSION_ALGORITHM参数指定压缩转储文件数据时要使用的压缩算法。
  • 内容
    Data Pump Export命令行实用程序CONTENT参数使您可以过滤Export卸载的内容：仅数据，仅元数据或两者。
  • DATA_OPTIONS
    Data Pump Export命令行实用程序DATA_OPTIONS参数指定在导出操作期间处理某些类型数据的方式。
  • 目录
    Data Pump Export命令行实用程序PARALLEL参数指定Export可以写入转储文件集和日志文件的默认位置。
  • DUMPFILE
    Data Pump Export命令行实用程序DUMPFILE参数指定导出作业的转储文件的名称和（可选）目录对象。
  • ENABLE_SECURE_ROLES
    Data Pump Export命令行实用程序ENABLE_SECURE_ROLES参数可防止在导出期间无意中使用受保护的角色。

指定导出参数

对于可以指定多个值的参数，可以使用逗号或空格指定值。例如，您可以指定TABLES=employees,jobs或TABLES=employees jobs 。

对于您输入的每个参数，您必须输入等号（ = ）和值。数据泵没有其他方式可以知道先前的参数规范已经完成，并且新的参数规范正在开始。例如，在以下命令行中，即使NOLOGFILE是有效参数，Export NOLOGFILE将该字符串解释为DUMPFILE参数的另一个转储文件名：

expdp DIRECTORY = dpumpdir DUMPFILE = test.dmp NOLOGFILE TABLES =员工

此命令导致创建两个转储文件test.dmp和nologfile.dmp 。

要避免此结果，请指定NOLOGFILE=YES或NOLOGFILE=NO 。

指定参数值时的区分大小写

对于作为参数值输入的表空间名称，模式名称，表名称等，Oracle Data Pump默认情况下将输入为小写或混合大小写的值更改为大写。例如，如果输入TABLE=hr.employees ，则将其更改为TABLE=HR.EMPLOYEES 。要保持大小写，必须将值括在引号内。例如， TABLE="hr.employees"将保留所有小写的表名。您输入的名称必须与存储在数据库中的名称完全匹配。

在数据泵命令行上使用引号

某些操作系统将引号视为特殊字符。因此，这些操作系统不会将引号传递给应用程序，除非引号前面有转义字符，例如反斜杠（ \ ）。这个要求在命令lin和参数文件中都是正确的。某些操作系统可能需要在命令行上围绕包含特殊字符的整个参数值附加一组单引号或双引号。

TABLES = \“MixedCaseTableName \”

2.4.3 ACCESS_METHOD

Data Pump Export命令行实用程序ACCESS_METHOD参数指示Export使用特定方法卸载数据。

默认值： AUTOMATIC

目的

指示导出使用特定方法卸载数据。

语法和说明

ACCESS_METHOD = [AUTOMATIC | DIRECT_PATH | EXTERNAL_TABLE | INSERT_AS_SELECT]

提供了ACCESS_METHOD参数，以便在默认方法由于某种原因不起作用时可以尝试替代方法。可以为网络导出指定所有方法。如果无法使用指定的访问方法卸载表的数据，则数据将显示表的错误并继续下一个工作项。

可用选项如下：

  • AUTOMATIC - 数据泵确定卸载每个表的数据的最佳方法。Oracle建议您尽可能使用AUTOMATIC因为它允许Data Pump自动选择最有效的方法。

  • DIRECT_PATH - 数据泵为每个表使用直接路径卸载。

  • EXTERNAL_TABLE - Data Pump使用SQL CREATE TABLE AS SELECT语句使用存储在转储文件中的数据创建外部表。SELECT子句从要卸载的表中读取。

  • INSERT_AS_SELECT - Data Pump执行SQL INSERT AS SELECT语句以从远程数据库卸载数据。此选项仅适用于网络模式导出。

限制

  • 要将ACCESS_METHOD参数与网络导出一起使用，您必须使用Oracle Database 12 c第2版（12.2.0.1）或更高版本。

  • Data Pump Export的ACCESS_METHOD参数对可传输表空间作业无效。

例

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = expdat.dmp SCHEMAS = hr ACCESS_METHOD = EXTERNAL_TABLE

2.4.7 COMPRESSION_ALGORITHM

Data Pump Export命令行实用程序COMPRESSION_ALGORITHM参数指定压缩转储文件数据时要使用的压缩算法。

默认值： BASIC

目的

指定压缩转储文件数据时要使用的压缩算法。

语法和说明

COMPRESSION_ALGORITHM = {BASIC |低| MEDIUM |高}

参数选项定义如下：

  • BASIC ：提供压缩比和速度的良好组合;使用的算法与以前版本的Oracle Data Pump中的算法相同。

  • LOW ：对出口吞吐量的影响最小。此选项适用于CPU资源是限制因素的环境。

  • MEDIUM ：适用于大多数环境。与BASIC选项一样，此选项提供压缩比和速度的良好组合，但它使用与BASIC不同的算法。

  • HIGH ：最适合在较慢的网络上复制转储文件的情况，其中限制因素是网络速度。

您可以通过CPU使用情况和压缩率（压缩输出的大小占未压缩输入的百分比）来表征压缩算法的性能。这些度量根据输入的大小和类型以及所使用的压缩算法的速度而变化。压缩比通常从低到高增加，可能会消耗更多的CPU资源。

建议您对环境中的数据运行具有不同压缩级别的测试。根据您的环境，工作负载特征以及数据大小和类型选择压缩级别是确保导出的转储文件集压缩级别满足性能和存储要求的唯一方法。

限制

  • 要使用此功能，必须将数据库兼容性设置为12.0.0或更高版本。

  • 此功能要求您启用Oracle Advanced Compression选项。

例1

此示例执行HR模式的模式模式卸载，并使用具有低压缩级别的压缩算法仅压缩表数据。使用此命令选项可以减少使用的CPU资源，但代价是压缩率低于最佳值。 

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp COMPRESSION = DATA_ONLY COMPRESSION_ALGORITHM = LOW

例2

此示例执行HR模式的模式模式卸载，并使用基本压缩级别压缩元数据和表数据。省略COMPRESSION_ALGORITHM参数等同于将BASIC指定为值。 

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp COMPRESSION = ALL COMPRESSION_ALGORITHM = BASIC

  • 要指定ALL ， DATA_ONLY或METADATA_ONLY选项， COMPATIBLE初始化参数必须至少设置为11.0.0。

2.4.15 ENCRYPTION_MODE

Data Pump Export命令行实用程序ENCRYPTION_MODE参数指定执行加密和解密时要使用的安全性类型。

默认值：默认模式取决于使用的其他加密相关参数。如果仅指定了ENCRYPTION参数且Oracle加密钱包已打开，则默认模式为TRANSPARENT 。如果仅指定了ENCRYPTION参数并且钱包已关闭，则会返回错误。

如果指定了ENCRYPTION_PASSWORD参数且钱包已打开，则默认为DUAL 。如果指定了ENCRYPTION_PASSWORD参数并且钱包已关闭，则默认值为PASSWORD 。

目的

指定执行加密和解密时要使用的安全性类型。

语法和说明

ENCRYPTION_MODE = [DUAL |密码|透明]

DUAL模式创建一个转储文件集，以后可以透明地或通过指定创建双模加密转储文件集时使用的密码导入。稍后导入在DUAL模式下创建的转储文件集时，可以使用通过ENCRYPTION_PASSWORD参数指定的钱包或密码。DUAL模式最适合使用钱包在现场导入转储文件集的情况，但也可能需要在钱包不可用的地点导入。

PASSWORD模式要求您在创建加密转储文件集时提供密码。导入转储文件集时，需要提供相同的密码。PASSWORD模式要求您还指定ENCRYPTION_PASSWORD参数。PASSWORD模式最适合将转储文件集导入另一个或远程数据库但在传输过程中必须保持安全的情况。

TRANSPARENT模式使您可以创建加密的转储文件集，无需数据库管理员（DBA）的任何干预，前提是所需的钱包可用。因此，不需要ENCRYPTION_PASSWORD参数。事实上，如果在TRANSPARENT模式下使用该参数，则会导致错误。此加密模式最适合将转储文件集导入到导出它的同一数据库中的情况。

限制

  • 要使用DUAL或TRANSPARENT模式， COMPATIBLE初始化参数必须至少设置为11.0.0。

  • 使用ENCRYPTION_MODE参数时，还必须使用ENCRYPTION或ENCRYPTION_PASSWORD参数。否则，返回错误。

  • 使用ENCRYPTION= ENCRYPTED_COLUMNS_ONLY ，不能使用ENCRYPTION_MODE参数。否则，返回错误。

  • 此参数仅在Oracle Database 11g或更高版本的企业版中有效。

  • Data Pump加密功能要求启用Oracle Advanced Security选项。有关Oracle Advanced Security选项的许可要求的信息 ，请参阅Oracle数据库许可信息 。

例

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = hr_enc4.dmp ENCRYPTION = all ENCRYPTION_PASSWORD = secretwords ENCRYPTION_ALGORITHM = AES256 ENCRYPTION_MODE = DUAL

目的

指定用于加密导出转储文件中的加密列数据，元数据或表数据的密码。使用此参数可防止对加密转储文件集的未授权访问。

注意：

自Oracle Database 11g第 1版（11.1）起，数据泵加密功能已更改。在版本11.1之前， ENCRYPTION_PASSWORD参数仅应用于加密列。但是，从版本11.1开始，新的ENCRYPTION参数提供了加密其他类型数据的选项。作为此更改的结果，如果现在指定ENCRYPTION_PASSWORD而不指定ENCRYPTION和特定选项，则写入转储文件的所有数据都将被加密（相当于指定ENCRYPTION=ALL ）。要仅重新加密加密列，除了ENCRYPTION_PASSWORD之外，您现在必须指定ENCRYPTION=ENCRYPTED_COLUMNS_ONLY 。

语法和说明

ENCRYPTION_PASSWORD = password

提供的password值指定用于重新加密加密表列，元数据或表数据的密钥，以便它们不会在转储文件集中写为明文。如果导出操作涉及加密表列，但未提供加密密码，则加密列将作为明文写入转储文件集并发出警告。

您输入的密码将回显到屏幕。如果您在输入时不想显示屏幕上显示的密码，请使用ENCRYPTION_PWD_PROMPT参数。

加密密码允许的最大长度通常为128个字节。但是，如果ENCRYPTION=ENCRYPTED_COLUMNS_ONLY且VERSION参数或数据库兼容性设置为小于12.2，则限制为30个字节。

对于导出操作，如果ENCRYPTION_MODE参数设置为PASSWORD或DUAL ，则此参数是必需的。

注意：

在最初创建具有加密列的表时，使用数据泵ENCRYPTION_PASSWORD参数指定的密钥与使用ENCRYPT关键字指定的密钥之间没有连接或依赖关系。例如，假设一个表创建如下，其中一个加密列的键为xyz ：

  • ENCRYPTION_PASSWORD参数是在完全可传输导出中传输包含具有加密列的表的加密表空间和表空间所必需的。

  • Data Pump加密功能要求启用Oracle Advanced Security选项。有关Oracle Advanced Security选项的许可要求的信息 ，请参阅Oracle数据库许可信息 。

  • 如果请求的加密模式为TRANSPARENT ，则ENCRYPTION_PASSWORD参数无效。

  • 如果ENCRYPTION_MODE设置为DUAL ， ENCRYPTION_PASSWORD使用ENCRYPTION_PASSWORD参数，必须启用Oracle Advanced Security透明数据加密（TDE）。有关TDE的更多信息，请参见“ Oracle数据库高级安全指南” 。

2.4.20排除

Data Pump Export命令行实用程序EXCLUDE参数使您可以通过指定要从导出操作中排除的对象和对象类型来过滤导出的元数据。

默认值：没有默认值

目的

允许您通过指定要从导出操作中排除的对象和对象类型来过滤导出的元数据。

语法和说明

EXCLUDE = object_type [： name_clause ] [，...]

object_type指定要排除的对象的类型。要查看有效值列表object_type ，查询以下观点： DATABASE_EXPORT_OBJECTS全模式， SCHEMA_EXPORT_OBJECTS的架构模式， TABLE_EXPORT_OBJECTS表和表空间模式。OBJECT_PATH列中列出的值是有效的对象类型。

给定导出模式的所有对象类型都包含在导出中，但EXCLUDE语句中指定的对象类型除外。如果排除了某个对象，则也会排除所有相关对象。例如，排除表也会排除表中的所有索引和触发器。

name_clause是可选的。使用此参数可以选择对象类型中的特定对象。它是一个SQL表达式，用作对该类型的对象名称的过滤器。它由一个SQL运算符和要与之比较指定类型的对象名称的值组成。name_clause仅适用于其实例具有名称的对象类型（例如，它适用于TABLE ，但不适用于GRANT ）。它必须用冒号与对象类型分开，并用双引号括起来，因为需要使用单引号来分隔名称字符串。例如，您可以设置EXCLUDE=INDEX:"LIKE 'EMP%'"以排除名称以EMP开头的所有索引。

您为name_clause提供的name_clause必须完全匹配，包括上限和下限，数据库中的现有对象。例如，如果您提供的name_clause是针对名为EMPLOYEES的表，则必须使用全部大写的名为EMPLOYEES的现有表。如果您提供name_clause作为Employees或employees或任何其他与现有表不匹配的变体，则找不到该表。

如果未提供name_clause ，则排除指定类型的所有对象。

您可以指定多个EXCLUDE语句。

根据您的操作系统，为此参数指定值时使用引号也可能需要使用转义字符。Oracle建议您将此参数放在参数文件中，这样可以减少命令行上可能需要的转义字符数。

如果您指定的object_type是CONSTRAINT ， GRANT或USER ，那么请注意这些效果，如以下段落所述。

排除约束

无法明确排除以下约束：

  • 表成功创建和加载所需的约束;例如，索引组织表的主键约束，或具有REF列的表的REF SCOPE和WITH ROWID约束

例如，以下EXCLUDE语句解释如下：

  • EXCLUDE=CONSTRAINT排除所有约束，但成功创建和加载表所需的任何约束除外。

  • EXCLUDE=REF_CONSTRAINT排除参照完整性（外键）约束。

不包括拨款和用户

指定EXCLUDE=GRANT排除对所有对象类型和系统权限授予的对象授予。

指定EXCLUDE=USER仅排除用户的定义，而不排除用户模式中包含的对象。

要排除特定用户和该用户的所有对象，请指定如下命令，其中hr是要排除的用户的模式名称。 

expdp FULL = YES DUMPFILE = expfull.dmp EXCLUDE = SCHEMA：“='HR'”

在此示例中，指定了导出模式FULL 。如果未指定模式，则使用默认模式。默认模式是SCHEMAS 。但是，如果使用默认模式，则在此示例中，默认模式会导致错误，因为如果使用SCHEMAS ，则该命令指示您希望同时导出和排除模式。

如果您尝试使用诸如EXCLUDE=USER:"='HR'"语句排除CREATE USER hr ，则仅排除CREATE USER hr DDL语句中使用的信息，并且您可以获得意外结果。

限制

  • EXCLUDE和INCLUDE参数是互斥的。

例

以下是使用EXCLUDE语句的示例。 

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = hr_exclude.dmp EXCLUDE = VIEW，PACKAGE，FUNCTION

此示例导致模式模式导出（默认导出模式），其中导出除其视图，包和函数之外的所有hr模式。

2.4.34并行

导出命令行实用程序PARALLEL参数指定代表导出作业运行的最大活动执行进程数。

默认值： 1

目的

指定代表导出作业运行的最大活动执行进程数。此执行集由工作进程和并行I / O服务器进程组合而成。在并行查询操作中充当查询协调器的主控制进程和工作进程不计入此总数。

此参数使您可以在资源消耗和已用时间之间进行权衡。

语法和说明

PARALLEL = integer

您为integer指定的integer应小于或等于转储文件集中的文件数（或者您应在转储文件规范中指定%U或%L替换变量）。由于每个活动工作进程或I / O服务器进程一次只写入一个文件，因此文件数量不足会产生不利影响。例如，某些工作进程在等待文件时可能处于空闲状态，从而降低了作业的整体性能。更重要的是，如果并行I / O服务器进程的协作组的任何成员无法获取输出文件，则导出操作会因ORA-39095错误而停止。通过使用数据泵导出实用程序附加到作业，在交互模式下使用ADD_FILE命令添加更多文件，以及在停止作业的情况下，重新启动作业，可以纠正这两种情况。

要在作业执行期间增加或减少PARALLEL的值，请使用交互式命令模式。减少并行性不会导致与作业相关的工作进程减少;它减少了在任何给定时间执行的工作进程数。此外，任何正在进行的工作必须在减少生效之前达到有序完成点。因此，可能需要一段时间才能看到降低值的任何影响。在作业退出之前，不会删除空闲工作程序。

如果有可以并行执行的工作，那么增加并行性会立即生效。

在Oracle RAC环境中导出期间使用PARALLEL

在Oracle Real Application Clusters（Oracle RAC）环境中，如果导出操作的PARALLEL=1 ，则所有Data Pump进程都驻留在启动作业的实例上。因此，目录对象可以指向该实例的本地存储。

如果导出操作将PARALLEL设置为大于1的值，则数据泵进程可以驻留在作业启动的实例之外的实例上。因此，目录对象必须指向可由Oracle RAC的所有实例访问的共享存储。

限制

  • 此参数仅在Oracle Database 11g或更高版本的企业版中有效。

  • 要并行导出表或表分区（使用PQ从属），您必须具有DATAPUMP_EXP_FULL_DATABASE角色。

例

以下是使用PARALLEL参数的示例：

> expdp hr DIRECTORY = dpump_dir1 LOGFILE = parallel_export.log JOB_NAME = par4_job DUMPFILE = par_exp％u.dmp PARALLEL = 4

这会导致hr模式的模式模式导出（默认），其中最多可以在目录对象dpump_dir1指向的路径中创建四个文件。

2.4.36查询

Data Pump Export命令行实用程序QUERY参数使您可以指定用于筛选导出数据的查询子句。

默认值：没有默认值

目的

使您可以指定用于过滤导出数据的查询子句。

语法和说明

QUERY = [ schema 。] [ table_name ：] query_clause

query_clause通常是用于细粒度行选择的SQL WHERE子句，但可以是任何SQL子句。例如，您可以使用ORDER BY子句来加速从堆组织表到索引组织表的迁移。如果未提供架构和表名，则查询将应用于导出作业中的所有表（并且必须对其有效）。特定于表的查询会覆盖应用于所有表的查询。

当要将查询应用于特定表时，冒号必须将表名与查询子句分开。可以指定多个特定于表的查询，但每个表只能指定一个查询。

如果与QUERY参数一起指定了NETWORK_LINK参数，则必须使用NETWORK_LINK值明确限定query_clause中指定的远程（源）节点上的任何对象。否则，Data Pump假定对象位于本地（目标）节点上;如果不是，则返回错误，并从远程（源）系统导入表失败。

例如，如果指定NETWORK_LINK=dblink1 ，则QUERY参数的query_clause必须指定该链接，如以下示例所示：

QUERY =（hr.employees：“WHERE last_name IN（SELECT last_name FROM hr.employees@dblink1）”）

根据您的操作系统，当您为此参数指定使用引号的值时，还可能要求您使用转义字符。Oracle建议您将此参数放在参数文件中，这样可以减少命令行上可能需要的转义字符数。 。

要在特定于表的查询中指定除您自己之外的模式，必须授予您对该特定表的访问权限。

限制

  • QUERY参数不能与以下参数一起使用：

    • CONTENT = METADATA_ONLY

    • ESTIMATE_ONLY

    • TRANSPORT_TABLESPACES

  • 为表指定QUERY参数时，Data Pump使用外部表卸载目标表。外部表使用SQL CREATE TABLE AS SELECT语句。QUERY参数的值是CREATE TABLE语句的SELECT部分中的WHERE子句。如果QUERY参数包含对具有与正在卸载的表匹配的列的另一个表的引用，并且如果在查询中使用这些列，则您将需要使用表别名来区分正在卸载的表中的列和列中的列具有相同名称的SELECT语句。Data Pump用于卸载表的表别名是KU$ 。

    例如，假设你要导出的一个子集sh.sales基于信用额度在客户表sh.customers表。在以下示例中， KU$用于限定QUERY参数中的cust_id字段以卸载sh.sales 。因此，Data Pump仅为信用额度超过10,000美元的客户导出行。 

    QUERY ='sales：'WHERE EXISTS（SELECT cust_id FROM customers c WHERE cust_credit_limit> 10000 AND ku $ .cust_id = c.cust_id）“'

    在以下查询中， KU$不用于表别名。结果是所有行都被卸载：

    QUERY ='sales：“WHERE EXISTS（SELECT cust_id FROM customers c WHERE cust_credit_limit> 10000 AND cust_id = c.cust_id）”'

  • QUERY字符串允许的最大长度为4000字节，其中包括引号。此限制意味着允许的实际最大长度为3998字节。

例

> expdp hr PARFILE = emp_query.par

emp_query.par文件的内容如下：

2.4.37 REMAP_DATA

Data Pump Export命令行实用程序REMAP_DATA参数使您可以指定重映射函数，该函数将指定列的原始值作为源，并返回将替换转储文件中原始值的重映射值。

默认值：没有默认值

目的

REMAP_DATA参数允许您指定重映射函数，该函数将指定列的原始值作为源，并返回将替换转储文件中原始值的重映射值。此选项的常见用途是在从生产系统移动到测试系统时屏蔽数据。例如，一列敏感的客户数据（如信用卡号）可以替换为REMAP_DATA函数生成的数字。这将允许数据保留其基本格式和处理特性，而不会将私人数据暴露给未经授权的人员。

相同的函数可以应用于被转储的多个列。当您希望保证在引用约束中重新映射子列和父列时的一致性时，这非常有用。

语法和说明

REMAP_DATA = [ schema 。] tablename 。 column_name ：[ schema 。] pkg 。 功能

每个语法元素在语法中出现的顺序的描述如下：

schema ：包含要重新映射的表的模式。默认情况下，这是执行导出的用户的架构。

tablename ：将重新映射其列的表。

column_name ：要重新映射其数据的列。

schema ：包含您创建的包含重映射功能的PL / SQL包的模式。默认情况下，这是执行导出的用户的架构。

pkg ：您创建的包含重映射功能的PL / SQL包的名称。

function ：PL / SQL中将被调用以重新映射指定表的每一行中的列表的函数的名称。

限制

  • source参数和返回值的数据类型和大小必须与表中指定列的数据类型和大小相匹配。

  • 除自治事务外，重映射函数不应执行提交或回滚。

  • 不支持使用同义词作为REMAP_DATA参数的值。例如，如果hr模式中的regions表具有regn的同义词，则如果将regn指定为REMPA_DATA规范的一部分，则会返回错误。

  • 不支持重新映射远程表的LOB列数据。

  • REMAP_DATA不支持以下类型的列：用户定义类型，用户定义类型的属性，LONG，REF，VARRAY，嵌套表，BFILE和XMLtype。

例

以下示例假定已创建名为remap的包，其中包含名为minus10和plusx函数。这些函数更改employees表中employee_id和first_name的值。 

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = remap1.dmp TABLES = employees REMAP_DATA = hr.employees.employee_id：hr.remap.minus10 REMAP_DATA = hr.employees.first_name：hr.remap.plusx

默认值：没有默认值

目的

用于指定与CLUSTER参数一起使用的服务名称。

语法和说明

  • 如果在实例A上启动数据泵作业，并指定CLUSTER=YES （或接受默认值，即Y ），并且未指定SERVICE_NAME参数，则Data Pump将在所有实例上创建工作程序：A，B，C和D，取决于指定的并行度。

  • 如果在实例D上启动数据泵作业，并指定CLUSTER=YES和SERVICE_NAME=my_service ，则可以在实例A，B，C和D上启动工作程序。即使实例D不在my_service ，也包括在内它是作业开始的实例。

  • 如果在实例A上启动数据泵作业，并指定CLUSTER=NO ，则忽略您指定的任何SERVICE_NAME参数。所有进程都从实例A开始。

  • 默认情况下，数据库中的表名称存储为大写。如果您具有大小写或小写的表名，并且希望保留表名的区分大小写，则必须将名称括在引号中。名称必须与存储在数据库中的表名完全匹配。

    某些操作系统要求命令行上的引号前面有转义字符。以下示例显示了在不同的导出模式下如何保持区分大小写。

    • 在命令行模式下：

      TABLES = '\ “的Emp \”'

    • 在参数文件模式中：

      TABLES = ' “的Emp”'

    TABLES =（emp＃，dept，mydata）

    TABLES =（'“emp＃”'，dept，mydata）

    注意：

    某些操作系统使用单引号作为转义字符，而不是双引号，而其他操作系统则相反。请参阅Oracle操作系统特定的文档。不同的操作系统对表命名也有其他限制。

在表模式导出期间使用可传输选项

要在表模式导出期间使用可传输选项，请使用TABLES参数指定TRANSPORTABLE=ALWAYS参数。指定表，分区或子分区的元数据将导出到转储文件。要移动实际数据，请将数据文件复制到目标数据库。

  • 表格使用的类型不以表格模式导出。此限制意味着如果您随后导入转储文件，并且目标数据库中尚不存在该类型，则表创建将失败。

  • 不支持使用同义词作为TABLES参数的值。例如，如果hr模式中的regions表具有regn的同义词，则使用TABLES=regn 。如果您尝试使用同义词，则会返回错误。

  • 如果在导出时也设置了TRANSPORTABLE=ALWAYS则只能从一个表中指定分区。

TRANSPORT_FULL_CHECK = [是|没有]

如果TRANSPORT_FULL_CHECK=YES ，则Export会验证可传输集内的这些对象与可传输集外的对象之间是否存在依赖关系。该检查解决了双向依赖关系。例如，如果表位于可传输集内，但其索引不在，则返回失败，并终止导出操作。同样，如果索引位于可传输集中，则也会返回失败，但表不是。

如果TRANSPORT_FULL_CHECK=NO则Export仅验证可传输集中是否存在依赖于可传输集外部对象的对象。此检查解决了单向依赖关系。例如，表不依赖于索引，但索引依赖于表，因为没有表的索引没有意义。因此，如果可传输集包含表而不包含其索引，则此检查成功。但是，如果可传输集包含索引而不包含表，则终止导出操作。

还执行了其他检查。例如，export始终验证在TRANSPORT_TABLESPACES指定的表空间集中定义的所有表（及其索引）的所有存储段实际上都包含在表空间集中。

有两个当前命令行参数控制完全关闭检查：

TTS_FULL_CHECK = [YES | NO] TRANSPORT_FULL_CHECK = [YES | NO]
[TTS|TRANSPORT]_FULL_CHECK=YES被解释为TTS_CLOSURE_CHECK=FULL 。 [TTS|TRANSPORT]_FULL_CHECK=NO被解释为TTS_CLOSURE_CHECK=ON.

例

> expdp hr DIRECTORY = dpump_dir1 DUMPFILE = tts.dmp TRANSPORT_TABLESPACES = tbs_1 TRANSPORT_FULL_CHECK = YES LOGFILE = tts.log

指定在表模式导出（使用TABLES参数指定）或完整模式导出（使用FULL参数指定）期间是否应使用可传输选项。

2.5.11 STOP_JOB

Data Pump Export交互式命令模式STOP_JOB参数将停止当前作业。它会立即停止作业，或者在有序关闭后停止作业，然后退出导出。
