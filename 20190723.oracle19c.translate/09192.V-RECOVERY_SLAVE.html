8.160 V $ RECOVERY_SLAVE

V$RECOVERY_SLAVE用于跟踪数据库介质恢复过程，以监控其性能统计信息并分析介质恢复会话。

柱 数据类型 描述

开始时间

日期

恢复过程的开始时间

类型

VARCHAR2（64）

正在执行的恢复过程类型：

  • MEDIA RECOVERY应用SLAVE

  • MEDIA RECOVERY Merge Slave

  • 系列媒体恢复

项目

VARCHAR2（32）

正在测量的项目。

当TYPE为MEDIA RECOVERY Apply SLAVE ，可能的值为：

  • 恢复ID

  • 进程ID

  • 重做缓存已满的数量

  • 重做缓存副本数量

  • CV缓存的数量

  • 简历应用OK

  • CV Applied Stuck

  • 简历应用修复

  • CV应用腐败

  • CV Applied Ckpt

  • CV应用重新申请

  • 总CV处理大小

  • 总CV应用

  • 缓冲区缓存已满的数量

  • 缓冲区重试次数

  • 发布的最大读数数量

  • Unrcv条件的数量

  • 刷新的涌入缓冲区数量

  • 收获请求数量

  • Reap Wait IO的数量

  • 收获数量没有缓冲区

  • 等待全部读取次数

  • 缓冲区的数量

  • 缓冲平时

  • 阅读发布时间

  • 发出的读取请求数

当TYPE为MEDIA RECOVERY Merge Slave ，可能的值为：

  • 恢复ID

  • 进程ID

  • 重做缓存已满的数量

  • 重做缓存副本数量

  • CV缓存的数量

  • Total Redo Read Bytes

  • 总CV被解析

当TYPE为SERIAL MEDIA RECOVERY ，可能的值包括来自其他两个TYPE的值。

单位

VARCHAR2（32）

每个项目的计量单位

至今

数

到目前为止完成的工作量

总

数

预期的工作总量

评论

VARCHAR2（248）

其他注释，可能会显示当前恢复会话的恢复ID和进程ID

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
