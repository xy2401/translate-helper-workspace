Oracle自动存储管理概述

Oracle ASM是Oracle推荐的存储管理解决方案，可替代传统的卷管理器，文件系统和原始设备。

Oracle ASM是Oracle数据库文件的卷管理器和文件系统 ，支持单实例Oracle数据库和Oracle Real Application Clusters（Oracle RAC）配置。

Oracle ASM使用磁盘组来存储数据文件; Oracle ASM磁盘组是Oracle ASM作为一个单元管理的磁盘集合。在磁盘组中，Oracle ASM公开Oracle Database文件的文件系统接口。存储在磁盘组中的文件内容均匀分布，以消除热点并在磁盘上提供统一的性能。性能与原始设备的性能相当。

当数据库继续访问磁盘组中的文件时，您可以在磁盘组中添加或删除磁盘。在磁盘组中添加或删除磁盘时，Oracle ASM会自动重新分配文件内容，并在重新分发内容时无需停机。

Oracle ASM卷管理器功能提供灵活的基于服务器的镜像选项。Oracle ASM普通和高冗余磁盘组分别支持双向和三向镜像。您可以使用外部冗余来启用独立磁盘冗余阵列（RAID）存储子系统来执行镜像保护功能。

Oracle ASM还使用Oracle托管文件（OMF）功能来简化数据库文件管理。OMF自动在指定位置创建文件。当删除表空间或文件时，OMF还会在放弃空间的同时命名文件并将其删除。

Oracle ASM通过将数据存储整合到少量磁盘组中来减少管理数据库存储的管理开销。较少数量的磁盘组合并了多个数据库的存储，并提供了改进的I / O性能。

Oracle ASM文件可以与其他存储管理选项共存，例如原始磁盘和第三方文件系统。此功能简化了Oracle ASM与预先存在的环境的集成。

Oracle ASM具有易于使用的管理界面，例如SQL * Plus，Oracle ASM命令行实用程序（ASMCMD）命令行界面和Oracle ASM配置助手（ASMCA）。

关于Oracle ASM实例

Oracle ASM实例基于与Oracle数据库实例相同的技术构建。

Oracle ASM实例具有与Oracle数据库类似的系统全局区域（SGA）和后台进程。但是，由于Oracle ASM执行的任务少于数据库，因此Oracle ASM SGA比数据库SGA小得多。此外，Oracle ASM对服务器的性能影响最小。Oracle ASM实例装载磁盘组以使Oracle ASM文件可用于数据库实例; Oracle ASM实例不装入数据库。

在Oracle数据库安装在单独的Oracle主目录之前，Oracle ASM安装在Oracle Grid Infrastructure主目录中。Oracle ASM和数据库实例需要对磁盘组中的磁盘进行共享访问。Oracle ASM实例管理磁盘组的元数据，并向数据库实例提供文件布局信息。

Oracle ASM元数据是Oracle ASM用于控制磁盘组的信息，元数据驻留在磁盘组中。Oracle ASM元数据包括以下信息：

  • 属于磁盘组的磁盘

  • 磁盘组中可用的空间量

  • 磁盘组中文件的文件名

  • 磁盘组数据文件范围的位置

  • 重做日志，记录有关原子更改元数据块的信息

  • Oracle ADVM卷信息

可以使用Oracle Clusterware对Oracle ASM实例进行集群;每个群集节点都有一个Oracle ASM实例。如果同一节点上的不同数据库有多个数据库实例，则数据库实例在该节点上共享同一个Oracle ASM实例。

如果标准Oracle ASM集群中的节点上的Oracle ASM实例失败，则该节点上的所有数据库实例也将失败。但是，在Oracle Flex ASM配置中，Oracle 12 c数据库实例不会失败，因为它们可以在另一个节点上远程访问另一个Oracle ASM实例。

与文件系统驱动程序故障不同，Oracle ASM实例故障不需要重新启动操作系统。在Oracle RAC环境中，幸存节点上的Oracle ASM和数据库实例会自动从节点上的Oracle ASM实例故障中恢复。

关于Oracle ASM磁盘组

磁盘组由多个磁盘组成，是Oracle ASM管理的基本对象。

每个磁盘组都包含管理磁盘组中的空间所需的元数据。磁盘组组件包括磁盘，文件和分配单元。

文件从磁盘组分配。任何Oracle ASM文件都完全包含在单个磁盘组中。但是，磁盘组可能包含属于多个数据库的文件，而单个数据库可以使用来自多个磁盘组的文件。对于大多数安装，您只需要少量磁盘组，通常为两个，少于三个。

关于镜像和故障组

镜像通过在多个磁盘上存储数据副本来保护数据完整性。

创建磁盘组时，可以根据以下三个冗余级别之一指定Oracle ASM磁盘组类型：

  • 正常用于双向镜像

  • 高于 3向镜像

  • 外部不使用Oracle ASM镜像，例如在配置硬件RAID以实现冗余时

冗余级别控制在不卸载磁盘组或丢失数据的情况下可容忍的磁盘故障数。磁盘组类型确定Oracle在磁盘组中创建文件的镜像级别。

Oracle ASM镜像比传统的RAID镜像更灵活。对于指定为NORMAL冗余的磁盘组，您可以为每个文件指定冗余级别。例如，两个文件可以共享同一个磁盘组，其中一个文件是镜像的，而另一个文件则不是。

当Oracle ASM为镜像文件分配扩展区时，Oracle ASM会分配主副本和镜像副本。Oracle ASM选择将镜像副本存储在与主副本不同的故障组中的磁盘。故障组用于放置数据的镜像副本，以便每个副本位于不同故障组的磁盘上。故障组中所有磁盘的同时故障不会导致数据丢失。

您在创建Oracle ASM磁盘组时定义磁盘组的故障组。创建磁盘组后，您无法更改磁盘组的冗余级别。如果省略故障组规范，则Oracle ASM会自动将每个磁盘置于其自己的故障组中，但Oracle Exadata单元上包含磁盘的磁盘组除外。正常冗余磁盘组至少需要两个故障组。高冗余磁盘组至少需要三个故障组。具有外部冗余的磁盘组不使用故障组。

关于Oracle ASM磁盘

Oracle ASM磁盘是配置到Oracle ASM磁盘组的存储设备。

Oracle ASM磁盘的示例包括：

  • 存储阵列中的磁盘或分区

  • 整个磁盘或磁盘的分区

  • 逻辑卷

  • 网络附加文件（NFS）

将磁盘添加到磁盘组时，可以分配Oracle ASM磁盘名称，或者Oracle ASM自动分配Oracle ASM磁盘名称。此名称与操作系统使用的路径名称不同。在群集中，可以在不同节点上为磁盘分配不同的操作系统设备名称，但磁盘在所有节点上具有相同的Oracle ASM磁盘名称。在群集中，必须可以从共享磁盘组的所有实例访问Oracle ASM磁盘。

Oracle ASM按比例在磁盘组中的所有磁盘上传播文件。此分配模式将每个磁盘维持在相同的容量级别，并确保磁盘组中的所有磁盘具有相同的I / O负载。由于Oracle ASM在磁盘组中的所有磁盘之间进行负载平衡，因此不同的Oracle ASM磁盘不应共享同一物理驱动器。

关于Oracle ASM分配单元

每个Oracle ASM磁盘都分为分配单元（AU）。

分配单元是磁盘组内的基本分配单位。文件范围由一个或多个分配单元组成。Oracle ASM文件由一个或多个文件范围组成。

创建磁盘组时，可以使用AU_SIZE磁盘组属性设置Oracle ASM分配单元大小。值可以是1,2,4,8,16,32或64 MB，具体取决于特定磁盘组兼容级别。较大的AU大小通常为使用大型顺序读取的数据仓库应用程序提供性能优势。

关于Oracle ASM文件

存储在Oracle ASM磁盘组中的文件称为Oracle ASM文件。

每个Oracle ASM文件都包含在单个Oracle ASM磁盘组中。Oracle数据库在文件方面与Oracle ASM进行通信。这类似于Oracle数据库在任何文件系统上使用文件的方式。您可以将各种文件类型存储在Oracle ASM磁盘组中，包括：

  • 控制文件

  • 数据文件，临时数据文件和数据文件副本

  • SPFILEs

  • 联机重做日志，存档日志和闪回日志

  • RMAN备份

  • 灾难恢复配置

  • 更改跟踪位图

  • 数据泵转储集

Oracle ASM自动生成Oracle ASM文件名，作为文件创建和表空间创建的一部分。Oracle ASM文件名以加号（ + ）开头，后跟磁盘组名称。您可以为Oracle ASM文件指定用户友好的别名，并为别名创建分层目录结构。

了解Oracle ASM范围

Oracle ASM文件的内容作为存储在磁盘组中各个磁盘上的扩展区的集合或集合存储在磁盘组中。

每个范围都驻留在单个磁盘上。范围包括一个或多个分配单元（AU）。为了适应越来越大的文件，Oracle ASM使用可变大小的扩展区。

可变大小的扩展区支持更大的Oracle ASM数据文件，减少非常大的数据库的SGA内存需求，并提高文件创建和打开操作的性能。初始扩展区大小等于磁盘组分配单元大小，并且在预定义阈值时增加4或16倍。本主题描述了各种范围大小。

对于AU大小小于4 MB的磁盘组：

  • 范围大小始终等于前20000范围集（0 - 19999）的磁盘组AU大小。

  • 范围大小等于下一个20000范围集（20000 - 39999）的4 * AU大小。

  • 对于下一个20000及更高范围集（40000+），范围大小等于16 * AU大小。

对于AU大小大于或等于4 MB且磁盘组RDBMS兼容性大于或等于11.2.0.4的磁盘组，大小范围的计数（磁盘组AU大小，4 * AU大小或16 * AU size）使用应用程序块大小计算，以支持最大文件大小。

当特定磁盘组兼容性属性设置为11.1或更高时，范围大小调整功能对于新创建和调整大小的数据文件是自动的。有关兼容性属性的信息，请参阅磁盘组兼容性 。

了解Oracle ASM条带化

Oracle ASM条带化有两个主要目的：平衡磁盘组中所有磁盘的负载并减少I / O延迟。

粗粒度条带化为磁盘组提供负载平衡，而细粒度条带化通过更广泛地扩展负载来减少某些文件类型的延迟。

为了条带化数据，Oracle ASM将文件分成条带，并在磁盘组中的所有磁盘上均匀分布数据。在任何配置中，细粒度条带大小始终等于128 KB;这为小型I / O操作提供了较低的I / O延迟。粗粒度条带大小始终等于AU大小（不是数据范围大小）。

图1-5和图1-6是Oracle ASM文件条带化的图示。在两个插图中，对于由8个磁盘组成的磁盘组，分配单元大小已设置为1 M（ AU_SIZE = 1M ）。该实例是Oracle ASM 11g第2版（11.2），ASM和RDBMS的磁盘组兼容性属性已设置为11.2 ，因此在前20,000个扩展区之后，图形中会显示可变扩展区。对于前20,000个范围，范围大小为1 M并等于一个分配单元（AU）。对于接下来的20,000个范围，范围大小为4 M且等于4个AU。

要识别文件的条带块，它们已被标记为A ..X（24个字母）使用不同的字体连续A系列X直到识别出所有块。

在图1-5中 ，文件以128 K块（条带标记为A ..X）每个128 K块存储在一个扩展区中，从磁盘1中的第一个扩展区开始，然后是磁盘2中的第一个扩展区，然后以循环模式继续通过所有磁盘，直到整个文件被条带化。如此示例所示，条带化块首先填满每个磁盘的第一个范围，然后填充每个磁盘的第二个范围，依此类推，直到整个文件被条带化。

关于Oracle ASM文件模板

模板是属性值的集合，用于在创建Oracle ASM文件时为其指定磁盘区域，文件镜像和条带化属性。

创建文件时，可以包含模板名称，并根据单个文件而不是文件类型分配所需的属性。

为每种Oracle文件类型提供了默认模板，但您可以自定义模板以满足独特要求。每个磁盘组都有一个与每种文件类型关联的默认模板。

关于发现磁盘

磁盘发现过程用于查找Oracle ASM可以访问的磁盘的操作系统名称。

磁盘发现查找包含要装入的磁盘组的所有磁盘。发现的磁盘集还包括可以添加到磁盘组的磁盘。

Oracle ASM实例需要ASM_DISKSTRING初始化参数值来指定其发现字符串。仅发现Oracle ASM实例有权打开的路径名。发现字符串的确切语法取决于各种因素，例如平台以及是否使用Oracle Exadata磁盘。操作系统接受的路径名始终可用作发现字符串。

关于安装和卸载磁盘组

在数据库实例可以访问磁盘组中的文件之前，必须由本地Oracle ASM实例装入磁盘组。

挂载磁盘组需要发现所有磁盘并找到正在挂载的磁盘组中的文件。

您可以显式卸载磁盘组。如果在任何磁盘组文件打开时尝试在没有force选项的情况下卸载磁盘组，Oracle会报告错误。磁盘故障可能超过Oracle ASM冗余设置。如果发生这种情况，则强制卸除磁盘组。如果强制卸载磁盘组，则数据库无法访问磁盘组中的文件。

关于添加和删除磁盘

您可以将磁盘添加到现有磁盘组以添加空间并提高吞吐量。

指定的发现字符串标识您可以添加的磁盘。必须使用其ASM_DISKSTRING初始化参数，由每个Oracle ASM实例发现您添加的磁盘。添加磁盘后，Oracle ASM重新平衡操作会将数据移动到新磁盘上。要最小化重新平衡I / O，同时添加多个磁盘会更有效。

如果磁盘组出现故障，则可以从磁盘组中删除磁盘或重新占用容量。使用Oracle ASM磁盘名称删除磁盘，而不是发现字符串设备名称。如果写入磁盘时发生错误，则Oracle ASM会自动删除磁盘。

关于在线存储重新配置和动态重新平衡

重新平衡磁盘组会在磁盘之间移动数据，以确保每个文件均匀分布在磁盘组中的所有磁盘上。

当所有文件均匀分散后，所有磁盘均匀填充到相同的百分比;这确保了负载平衡。重新平衡不会根据I / O统计信息重新定位数据，也不会根据I / O统计信息启动重新平衡。Oracle ASM重新平衡操作由磁盘组中磁盘的大小控制。

Oracle ASM会在存储配置更改后自动启动重新平衡，例如添加，删除或调整磁盘大小时。power setting参数确定重新平衡操作发生的速度。

您可以手动启动重新平衡以更改正在运行的重新平衡的功率设置。如果正在运行重新平衡的实例停止，则会自动重新启动重新平衡。数据库可以在重新平衡操作期间保持运行。

您可以通过设置ASM_POWER_LIMIT初始化参数来最小化对数据库性能的影响。
