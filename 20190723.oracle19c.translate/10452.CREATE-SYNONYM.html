创建SYNONYM

目的

使用CREATE SYNONYM语句创建同义词 ，它是表，视图，序列，运算符，过程，存储函数，包，物化视图，Java类架构对象，用户定义对象类型或其他同义词的替代名称。同义词会对其目标对象设置依赖关系，并在更改或删除目标对象时变为无效。

同义词提供数据独立性和位置透明性。无论哪个用户拥有表或视图，无论哪个数据库拥有表或视图，同义词都允许应用程序无需修改即可运行。但是，同义词不能替代数据库对象的权限。在用户可以使用同义词之前，必须向用户授予适当的权限。

您可以在以下DML语句中引用同义词： SELECT ， INSERT ， UPDATE ， DELETE ， FLASHBACK TABLE ， EXPLAIN PLAN和LOCK TABLE 。

您可以在以下DDL语句中引用同义词： AUDIT ， NOAUDIT ， GRANT ， REVOKE和COMMENT 。

先决条件

要在您自己的架构中创建私有同义词，您必须具有CREATE SYNONYM系统特权。

要在另一个用户的模式中创建私有同义词，您必须具有CREATE ANY SYNONYM系统特权。

要创建PUBLIC同义词，您必须具有CREATE PUBLIC SYNONYM系统特权。

语义

或替换

指定OR REPLACE以重新创建同义词（如果已存在）。使用此子句可以更改现有同义词的定义，而无需先删除它。

更换同义词的限制

您不能将OR REPLACE子句用于具有任何从属表或从属有效用户定义对象类型的类型同义词。

[版权所有|不可挽回的]

使用这些子句指定同义词是否是editioned或noneditioned对象，如果editioning为架构对象类型启用SYNONYM的schema 。对于私有同义词，默认为“可以EDITIONABLE 。对于公共同义词，默认值为NONEDITIONABLE 。有关已编辑和未编辑的对象的信息，请参见“ Oracle数据库开发指南” 。

上市

指定PUBLIC以创建公共同义词。所有用户都可以访问公共同义词。但是，每个用户必须对基础对象具有适当的权限才能使用同义词。

解析对象的引用时，Oracle数据库仅在对象未以模式开头且未跟随数据库链接时才使用公共同义词。

如果省略此子句，则同义词是私有的。私有同义词名称在其架构中必须是唯一的。只有当这些用户对基础数据库对象具有适当的权限并指定模式以及同义词名称时，私有同义词才能由所有者以外的用户访问。

关于公共同义词的说明

以下说明适用于公共同义词：

  • 如果创建公共同义词并且随后具有从属表或从属有效的用户定义对象类型，则无法在与依赖对象相同的模式中创建与同义词同名的另一个数据库对象。

  • 注意不要创建与现有模式同名的公共同义词。如果这样做，那么使用该名称的所有PL / SQL单元都将失效。

模式

指定架构以包含同义词。如果省略schema ，则Oracle数据库会在您自己的模式中创建同义词。如果已指定PUBLIC则无法为同义词指定架构。

代名词

指定要创建的同义词的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

注意：

同义词名称的最大长度遵循以下规则：

  • 如果COMPATIBLE初始化参数设置为值12 。 2或更高，则同义词名称的最大长度为128个字节。该数据库允许您创建和删除长度为129到4000字节的同义词。但是，除非这些较长的同义词名称代表Java名称，否则它们将无法在任何其他SQL命令中使用。

  • 如果COMPATIBLE初始化参数设置为低于12的值。 2 ，则同义词名称的最大长度为30个字节。该数据库允许您创建和删除长度为31到128个字节的同义词。但是，除非这些较长的同义词名称代表Java名称，否则它们将无法在任何其他SQL命令中使用。

较长的同义词名称将转换为模糊的较短字符串，以便存储在数据字典中。

共享

仅当在应用程序根目录中创建同义词时，此子句才适用。这种类型的同义词称为应用程序公共对象，它可以与属于应用程序根目录的应用程序PDB共享。要确定共享同义词的方式，请指定以下共享属性之一：

  • METADATA - 元数据链接共享同义词的元数据，但其数据对每个容器都是唯一的。这种类型的同义词被称为元数据链接的应用程序公共对象 。

  • NONE - 不共享同义词。

如果省略此子句，则数据库使用DEFAULT_SHARING初始化参数的值来确定同义词的共享属性。如果DEFAULT_SHARING初始化参数没有值，则默认值为METADATA 。

创建后，您无法更改同义词的共享属性。

FOR Clause

指定为其创建同义词的对象。要为其创建同义词的架构对象可以是以下类型：

  • 表或对象表

  • 视图或对象视图

  • 序列

  • 存储过程，功能或包

  • 物化视图

  • Java类架构对象

  • 用户定义的对象类型

  • 代名词

架构对象当前不需要存在，您无需具有访问该对象的权限。

对FOR条款的限制

架构对象不能包含在包中。

模式

指定对象所在的模式。如果不使用schema限定对象，则数据库假定模式对象位于您自己的模式中。

如果要为远程数据库上的过程或函数创建同义词，则必须在此CREATE语句中指定schema 。或者，您可以在对象所在的数据库上创建本地公共同义词。但是，数据库链接必须包含在对过程或函数的所有后续调用中。

DBLINK

您可以指定完整或部分数据库链接，以便为对象所在的远程数据库上的模式对象创建同义词。如果指定dblink并省略schema ，则同义词指的是数据库链接指定的模式中的对象。Oracle建议您在远程数据库中指定包含该对象的模式。

如果省略dblink ，则Oracle数据库假定该对象位于本地数据库中。

数据库链接的限制

您不能为Java类同义词指定dblink 。

例子

创建SYNONYM：示例

要定义同义词offices的桌子locations在架构hr ，声明如下：

为hr.locations创建SYNONYM办公室;

要在remote数据库上的模式hr为employees表创建PUBLIC同义词，可以发出以下语句：

创建公共SYNONYM emp_table FOR hr.employees@remote.us.example.com;

如果底层对象包含在另一个模式中，则同义词可以与基础对象具有相同的名称。

Oracle数据库同义词解析：示例

Oracle数据库尝试在PUBLIC同义词级别解析之前解析对模式级别对象的引用。例如，模式oe和sh都包含名为customers表。在下一个示例中，用户SYSTEM为oe.customers创建名为customers的PUBLIC同义词：

为客户创建公共SYNONYM客户;

如果用户sh然后发出以下语句，则数据库返回sh.customers的行sh.customers ：

SELECT COUNT（*）FROM customers;

要从oe.customers检索行oe.customers ，用户sh必须在customers oe.customers加上模式名称。（用户sh必须对oe.customers有选择权限。） 

SELECT COUNT（*）FROM oe.customers;

如果用户hr的架构不包含名为customers的对象，并且hr对oe.customers具有select权限，则hr可以使用公共同义词customers访问oe架构中的customers表：

SELECT COUNT（*）FROM customers;
