D数据类型转换

你必须转换数据类型和当您使用PGAU工具生成TIP时以及使用时开发自定义TIP时，数据格式正确 PL / SQL以及UTL_RAW和UTL_PG函数。

阅读以下主题以了解与TIP相关的数据类型转换：

  • 长度检查

  • 转变

长度检查

PGAU生成的TIP在发送和接收的每个参数结束时执行长度检查。

表D-1提供了PGAU生成的长度参数列表：

表D-1长度参数

参数 描述

预期长度

在生成TIP时由PGAU计算。

转换长度

由每个转换字段的TIP求和。

发送长度

传输的发送数据长度是否也等于发送参数的实际长度。

接收长度

是传输的接收数据长度。

当发送参数的转换长度不等于其预期长度时，会引发异常。如果执行太多或太少的发送字段转换，则会发生这种情况。

当接收参数的转换长度不等于其接收长度时，会引发异常。当执行的转换太少或太多时，会导致这些长度异常。

当接收参数的预期长度不等于其转换或接收长度并启用数据转换跟踪时，将发出警告。当预期有最大长度记录时会发生这种情况，但会传输较短的记录并正确转换。

参数长度超过32K

PGAU生成TIP，支持超过32K字节的单个数据参数的传输。

PGAU会自动包含此支持 PGAU GENERATE处理检测超过32K的数据参数的最大长度。

这种支持是由放在数据中的数据定义驱动的 PG DD并且不能由用户选择。要包含支持，数据定义必须实际或可能超过32K。 要删除支持，必须将参数长度减小到小于32K， REDEFINE数据，然后再次GENERATE提示。

这种支持测试跨越32K缓冲区边界的场位置在转换跨越这些边界的那些场之前和之后。在重复组的情况下，对于重复组，可以是多个字段，对于简单线性记录，可以是很少的字段。

每个测试和相应的缓冲区管理逻辑都会增加开销。

注意：

REDEFINE子句的目标不能驻留在先前处理的缓冲区中。包含此类REDEFINE子句的字段的运行时TIP处理会得到不可预测的结果。

转变

该 PG DD和 PGAU支持生成的TIP COBOL，在定义数据时指定为IBMVSCOBOLII 。

用法（PASS）

当指定USAGE(PASS) PGAU DEFINE DATA语句，支持以下数据类型和格式转换：

  • PIC X.

  • PIC G

PIC X数据类型转换

PGAU TIP将COBOL X数据类型转换为具有相同字符长度的PL / SQL CHAR数据类型。还执行全球化支持字符集转换。

注意 ： COBOL缺少专门为可变长度数据指定的数据类型。它在COBOL中表示为包含PIC 9长度字段，后跟PIC X字符字段的子组。例如：

10 NAME 。

1 5 LENGTH PIC S9(4) 。

15 LETTERS PIC X(30) 。

鉴于此上下文，不能保证S9(4)字段后跟X字段的所有实例始终是可变长度数据。而不是PGAU TIP将上述COBOL组NAME转换为VARCHAR ，TIP代替构造嵌套的PL / SQL记录，如下所示：

TYPE NAME_typ是RECORD（LENGTH NUMBER（4,0），LETTERS CHAR（30））;
TYPE ...是RECORD（...NAME NAME_typ，...

这是客户端应用程序的责任（基于远程主机数据的特定知识）提取NAME.LENGTH从人物NAME.LETTERS和结果分配给一个PL / SQL VARCHAR ，如果VARCHAR需要。

用法（ASIS）

                  

                     

                        
在PGAU DEFINE DATA语句中指定USAGE(ASIS) ，不执行转换。因此，每个这样的字段被简单地复制到相同字节长度的PL / SQL RAW。不进行转换，翻译或重新格式化。
                        

                     

                     
                  

               
用法（SKIP）

                  

                     

                        
什么时候 USAGE(SKIP)在PGAU DEFINE DATA语句中指定，不执行数据交换。跳过数据就好像它不存在一样。因此，这些字段不是从中选择的 PG DD，未反映在TIP逻辑中，并假定与远程主机交换的数据流不存在。“ SKIP ”的目的是在PG DD中定义，但不活动，可能是因为远程主机已删除该字段或尚未包含该字段。即使远程主机上不存在某些字段， SKIP允许使用现有数据定义。

重复的名称

                     

                     
                     

                        
COBOL允许在记录中重复定义相同的组或元素名称，而更高级别组的上下文用于唯一限定名称。但是，由于PGAU生成的TIP声明了PL / SQL记录变量，这些变量引用了从属组和字段的嵌套PL / SQL记录，因此这种嵌套的PL / SQL记录类型可能具有重复的名称。

                        
给定以下COBOL定义，请注意ZIP在COBOL中是唯一限定的，但相应的PL / SQL声明将具有ZIP的重复嵌套记录类型。
01 EMPREC。05 HIREDATE PIC X（8）。05生日图片X（8）。05技能PIC X（12）发生4。05 EMPNO PIC 9（4）。05 EMPNAME。10 FIRST-NAME PIC X（10）。10 LAST-NAME PIC X（15）。05家庭地址。 10 STREET PIC X（20）。10 CITY PIC X（15）。10状态PIC XX。 10个ZIP。15 FIRST-F PIC X（5）。15 LAST-FOUR PIC X（4）。05 DEPT PIC X（45）。05 OFFICE-ADDRESS。10 STREET PIC X（20）。10 CITY PIC X（15）。10状态PIC XX。 10个ZIP。15 FIRST-F PIC X（5）。15 LAST-FOUR PIC X（4）。05 JOBTITLE PIC X（20）。

PGAU避免声明重复的嵌套记录类型，并生成以下PL / SQL：
SKILL_Key BINARY_INTEGER; TYPE SKILL_Tbl是BINARY_INTEGER的CHAR（12）INDEX表; TYPE EMPNAME_Typ是RECORD（FIRST_NAME CHAR（10），LAST_NAME CHAR（15））; TYPE ZIP_Typ是RECORD（FIRST_FIVE CHAR（5），LAST_FOUR CHAR（4））; TYPE HOME_ADDRESS_Typ是RECORD（STREET CHAR（20），CITY CHAR（15），STATE CHAR（2），ZIP ZIP_Typ）; TYPE OFFICE_ADDRESS_Typ是RECORD（STREET CHAR（20），CITY CHAR（15），STATE CHAR（2），ZIP ZIP_Typ）; TYPE EMPREC_Typ是RECORD（HIREDATE CHAR（8），BIRTHDATE CHAR（8），SKILL SKILL_Tbl，EMPNO NUMBER（4,0），EMPNAME EMPNAME_Typ，HOME_ADDRESS HOME_ADDRESS_Typ，DEPT CHAR（45），OFFICE_ADDRESS OFFICE_ADDRESS_Typ，JOBTITLE CHAR（20））;
但是，如果多个嵌套组具有相同的名称但具有不同的子字段（请参阅下面的ZIP ）：
05家庭地址。10 STREET PIC X（20）。10 CITY PIC X（15）。10状态PIC XX。 10个ZIP。15 LEFTMOST-FOUR PIC X（4）。15 RIGHMOST-FURE PIC X（5）。05 DEPT PIC X（45）。05 OFFICE-ADDRESS。10 STREET PIC X（20）。10 CITY PIC X（15）。10状态PIC XX。10个ZIP。15 FIRST-F PIC X（5）。15 LAST-FOUR PIC X（4）。05 JOBTITLE PIC X（20）。
         
PGAU为名称，数据类型或选项中子字段不同的每个声明更改PL / SQL嵌套记录类型的名称。请注意第二个声明（ ZIP_Typ02 ）附加的“ 02 ”及其在OFFICE_ADDRESS引用。
TYPE EMPNAME_Typ是RECORD（FIRST_NAME CHAR（10），LAST_NAME CHAR（15））; TYPE ZIP_Typ是RECORD（LEFTMOST_FOUR CHAR（4），RIGHTMOST_FIVE CHAR（5））; TYPE HOME_ADDRESS_Typ是RECORD（STREET CHAR（20），CITY CHAR（15），STATE CHAR（2），ZIP ZIP_Typ）; TYPE ZIP_Typ02是RECORD（FIRST_FIVE CHAR（5），LAST_FOUR CHAR（4））; TYPE OFFICE_ADDRESS_Typ是RECORD（STREET CHAR（20），CITY CHAR（15），STATE CHAR（2），ZIP ZIP_Typ02）; TYPE EMPREC_Typ是RECORD（HIREDATE CHAR（8），BIRTHDATE CHAR（8），SKILL SKILL_Tbl，EMPNO NUMBER（4,0），EMPNAME EMPNAME_Typ，HOME_ADDRESS HOME_ADDRESS_Typ，DEPT CHAR（45），OFFICE_ADDRESS OFFICE_ADDRESS_Typ，JOBTITLE CHAR（20））;
