5配置Oracle网络

本节中的说明介绍了如何配置网络，无论您的网关是使用SNA还是TCP / IP通信协议。

网关必须定义为必须定义Oracle Net Listener和服务名称才能访问网关。为此，请执行以下步骤：

  1. 将网关条目添加到listener.ora文件：

    • 如果您使用SNA ：

      （SID_DESC =（SID_NAME = PGA ）（ORACLE_HOME = C：\ oracle \ pga \ 12.2 ）（PROGRAM = pg4asrv））

      其中： C:\oracle\pga\12.2是您的网关Oracle主页， PGA是网关SID名称。

    • 或者，如果您使用的是TCP / IP ：

      （SID_DESC =（SID_NAME = PGA ）（ORACLE_HOME = C：\ oracle \ pga \ 12.2 ）（PROGRAM = pg4t4ic））

      其中： C:\oracle\pga\12.2是您的网关Oracle主页， PGA是网关SID名称。

  2. 将网关的服务名称添加到 Oracle数据库所在系统上的tnsnames.ora文件。服务名称在。中指定 USING参数为从Oracle数据库访问网关而定义的数据库链接。例如，如果您使用IPC协议适配器并且网关sid是PGA，则将以下条目添加到tnsnames.ora：
    pgaipc =（DESCRIPTION =（ADDRESS =（PROTOCOL = ipc）（KEY = key ））（CONNECT_DATA =（SID = PGA））（  HS =））

    在此示例中， key是IPC 密钥定义在 IPC的listener.ora文件协议。您只能使用IPC协议 Oracle数据库和网关位于同一系统上。

    如果你正在使用 TCP / IP协议适配器，如果您的网关sid是PGA，则添加以下条目 tnsnames.ora ：

    pgatcp =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（Host = gateway ）（Port = port ））（CONNECT_DATA =（SID = PGA））（  HS =））

    在此示例中， port是在中定义的TCP端口 TCP的listener.ora文件 protocol和gateway是gateway所在系统的TCP / IP主机名。

    注意：

    在以下情况下：

    • 如果您的网关和Oracle数据库不在同一系统上，

    • 或者，如果网关和Oracle数据库在同一系统上但是 Oracle数据库侦听器与网关侦听器不同，

    那么你必须定义Oracle数据库 PGAU通过添加服务名称网关所在系统上的tnsnames.ora。例如：

    ora_server =（DESCRIPTION =（ADDRESS =（PROTOCOL = TCP）（PORT = port ）（HOST = ora_srv ））（CONNECT_DATA =（SID = ora_server ）））

    在这个例子中，

    • port是Oracle数据库中定义的TCP端口 TCP协议的listener.ora;

    • ora_srv是Oracle数据库所在系统的TCP / IP主机名;和

    • ora_server是Oracle数据库的SID。

    确保启动已定义的侦听器。有关配置网络的详细信息，请参阅“ Oracle数据库网络服务管理员指南” 。

    要查看示例listener.ora文件，请参阅SNA协议的网关初始化参数中的 “ 使用SNA的网关的示例listener.ora文件 ”和“ 使用SNA的网关的 示例tnsnames.ora文件 ” （如果您的通信协议是SNA） 。

    要查看示例listener.ora文件，请参阅TCP / IP通信协议的网关初始化参数中的 “ 使用TCP / IP的网关的 示例listener.ora文件 ”和“ 使用TCP / IP的网关的示例tnsnames.ora文件 ” （如果您的通信协议是TCP / IP）。
