34.2.1升级前步骤

之前升级时，必须删除在其中创建的任何数据挖掘模型 Java和在中创建的任何挖掘活动 Oracle Data Miner Classic（早期版本的 Oracle Data Miner）。

警告：

在Oracle Database 12c中 ，Oracle Data Mining不支持Java API，Oracle Data Miner Classic无法针对Oracle Database 12 c运行。

34.2.1.1删除用Java创建的模型

如果10 g或11 g数据库包含使用Java创建的模型，请在升级数据库之前使用DBMS_DATA_MINING.DROP_MODEL例程删除模型。

34.2.1.2删除Oracle Data Miner Classic中创建的挖掘活动

如果您的数据库包含挖掘活动在Oracle Data Miner Classic中，删除挖掘活动并在升级数据库之前删除存储库。跟着这些步骤：

  1. 使用Data Miner Classic用户界面删除挖掘活动。

  2. 在SQL * Plus或SQL Developer中，删除这些表：

    DM4J $ ACTIVITIES DM4J $结果DM4J $ TRANSFORMS

    以及这些观点：

    DM4J $ MODEL_RESULTS_V DM4J $ RESULTS_STATE_V

完成这些步骤后，数据库中的任何模式中都必须没有前缀为DM4J$表或视图。

34.2.2升级Oracle数据挖掘

了解如何升级Oracle Data Mining。

在你之后完成“升级前步骤” ，无论您是从11 g还是从10 g版本升级 ，所有模型和挖掘元数据都与Oracle数据库升级过程完全集成。

升级的模型继续像以前的版本一样工作。您在升级后的环境中创建的升级模型和新模型都可以使用新版本中引入的新挖掘功能。

要升级数据库，可以使用数据库升级助手（DBUA），也可以使用导出/导入实用程序执行手动升级。

34.2.2.1使用数据库升级助手升级Oracle数据挖掘

神谕数据库升级助手提供了一个图形用户界面，可以指导您以交互方式完成升级过程。

在Windows平台上，请按照以下步骤启动升级助手：

  1. 转到Windows“ 开始”菜单，然后选择Oracle主目录。

  2. 选择“ 配置和迁移工具”菜单。

  3. 启动升级助手 。

在Linux平台上，运行DBUA实用程序以升级Oracle数据库。

34.2.2.1.1从版本10 g升级

在 Oracle Data Mining 10g，数据挖掘元数据和PL / SQL包存储在DMSYS模式中。在Oracle Data Mining 11 g和12 c中 ， DMSYS不再存在;数据挖掘元数据对象存储在SYS 。

当Oracle Database 10g升级到12 c时 ，所有数据挖掘元数据对象和PL / SQL包都将从DMSYS迁移到SYS 。成功迁移后，将删除DMSYS模式及其关联对象。删除DMSYS后， SYS.DBA_REGISTRY视图不再将Oracle Data Mining列为组件。

升级到Oracle Database 12 c后 ，您无法再切换到数据挖掘评分引擎（ DMSE ）。评分引擎不会在Oracle数据库11 g或12c的存在。

34.2.2.1.2从版本11 g升级

如果你将Oracle Database 11g升级到Oracle Database 12c ，并且数据库先前已从Oracle Database 10g升级，然后DMSYS架构可能仍然存在。如果升级过程检测到DMSYS ，则会在升级期间显示警告消息并丢弃DMSYS 。

34.2.2.2使用导出/导入升级数据挖掘模型

如果必要的，你可以您可以使用自动化程度较低的方法来升级数据挖掘模型。您可以导出在先前版本的Oracle数据库中创建的模型，并将它们导入Oracle Database 12 c的实例中。

警告：

不要导入使用Java创建的数据挖掘模型。Oracle Database 12 c不支持它们。

34.2.2.2.1导出/导入发布10 g数据挖掘模型

按照导出和导入数据挖掘模型的说明进行操作。

要将模型从Oracle Database 10g实例导出到转储文件，请按照“导出和导入挖掘模型”中的说明进行操作 。在从转储文件导入模型之前，运行DMEIDMSYS脚本以在Oracle Database 12 c中创建DMSYS模式。 

SQL> CONNECT / as sysdba; SQL> @ ORACLE_HOME \ RDBMS \ admin \ dmeidmsys.sql SQL> EXIT;

注意：

TEMP表空间必须已存在于Oracle Database 12 g数据库中。DMEIDMSYS脚本使用TEMP和SYSAUX表空间来创建DMSYS模式。

要将转储文件导入Oracle Database 12 c数据库：

％ ORACLE_HOME \ bin \ impdp system \ < password > dumpfile = < dumpfile_name > directory = < directory_name > logfile = < logfile_name > .....SQL> CONNECT / as sysdba; SQL> EXECUTE dmp_sys.upgrade_models（）; SQL> ALTER SYSTEM FLUSH SHARED_POOL; SQL> ALTER SYSTEM FLUSH BUFFER_CACHE; SQL> EXIT;

upgrade_models脚本将所有数据挖掘元数据对象和PL / SQL包从DMSYS迁移到SYS ，然后在升级模型之前删除DMSYS 。

ALTER SYSTEM声明

您可以通过发出ALTER SYSTEM FLUSH FLASH_CACHE语句来刷新数据库智能闪存缓存。刷新数据库如果您需要测量重写查询的性能或来自相同起点的一组查询，则智能闪存缓存非常有用。

34.2.2.2.2导出/导入版本11 g数据挖掘模型

为了从Oracle数据库11 g的实例中导出模型到转储文件，请按照指示导入和导出挖掘模型 。

警告：

不要导入使用Java创建的数据挖掘模型。Oracle Database 12 c不支持它们。

要将转储文件导入Oracle Database 12 c数据库：

％ ORACLE_HOME \ bin \ impdp system \ < password > dumpfile = < dumpfile_name > directory = < directory_name > logfile = < logfile_name > .....SQL> CONNECT / as sysdba; SQL> EXECUTE dmp_sys.upgrade_models（）; SQL> ALTER SYSTEM flush shared_pool; SQL> ALTER SYSTEM flush buffer_cache; SQL> EXIT;

ALTER SYSTEM声明

您可以通过发出ALTER SYSTEM FLUSH FLASH_CACHE语句来刷新数据库智能闪存缓存。刷新数据库如果您需要测量重写查询的性能或来自相同起点的一组查询，则智能闪存缓存非常有用。

34.2.3升级后步骤

执行步骤以查看已升级的数据库。

升级数据库后，请检查已升级数据库中的DBA_MINING_MODELS视图。必须在此视图中列出新升级的挖掘模型。

验证升级并确认无需降级后，必须将初始化参数COMPATIBLE设置为12.1 。

注意：

CREATE MINING MODEL pr ivilege必须授予Data Mining 用于创建挖掘模型的用户帐户。

34.2.4降级Oracle数据挖掘

之前将Oracle Database 12 c数据库降级回以前的版本，确保不存在奇异值分解模型或期望最大化模型。这些算法仅在Oracle Database 12 c中可用。在降级之前，使用DBMS_DATA_MINING.DROP_MODEL例程删除这些模型。如果不这样做，数据库降级过程将终止。

在SYS发出以下SQL语句以验证降级：

SQL> SELECT o.name FROM sys.model $ m，sys.obj $ o WHERE m.obj＃= o.obj＃AND m.version = 2;
