7.201 V $ ENCRYPTION_WALLET

V$ENCRYPTION_WALLET显示有关钱包状态和透明数据加密钱包位置的信息。在多租户容器数据库（CDB）中，此视图显示从CDB$ROOT查询时所有可插拔数据库（PDB）的钱包信息。从PDB查询时，此视图仅显示该PDB的钱包详细信息。

柱 数据类型 描述

WRL_TYPE

VARCHAR2（20）

钱包资源定位器的类型（例如， FILE ）

WRL_PARAMETER

VARCHAR2（4000）

钱包资源定位器的参数（例如，钱包或密钥库的绝对目录位置，如果WRL_TYPE = FILE ）

状态

VARCHAR2（30）

钱包的状态：

  • 关闭

  • 无法使用

    此状态表示密钥库在WALLET_ROOT初始化参数指定的位置中不可用

  • 打开

  • OPEN_NO_MASTER_KEY

  • OPEN_UNKNOWN_MASTER_KEY_STATUS

    当数据库处于挂载状态时，它无法检查是否设置了硬件密钥库的主密钥，因为数据字典不可用。在这种情况下，将显示此状态。

WALLET_TYPE

VARCHAR2（20）

显示正在使用的密钥库的类型， HSM或SOFTWARE_KEYSTORE 。

如果密钥库是使用mkstore实用程序创建的，则WALLET_TYPE是UNKNOWN 。

WALLET_ORDER

VARCHAR2（9）

可能的值：

  • SINGLE - 仅配置单个钱包时，这是列中的值。

  • PRIMARY - 当配置了多个钱包时，此值表示钱包是主要的（保留当前主密钥）。

  • SECONDARY - 当配置了多个钱包时，此值表示钱包是次要的（保留旧密钥）。

主密钥的查找首先在主密钥库中进行，然后在辅助密钥库中进行（如果需要）。

如果只使用一种类型的密钥库（硬件安全模块或软件密钥库），则会出现SINGLE 。

如果使用这两种类型，则此列中的值显示将查找每个密钥库的顺序。

KEYSTORE_MODE

VARCHAR2（8）

显示密钥库模式：

  • NONE ：从CDB$ROOT查询此列时，或者当数据库是非CDB时，可以看到此值。密钥库模式不适用于这些情况。

  • UNITED ：PDB配置为使用CDB$ROOT的钱包。要在此配置中打开钱包，必须使用CDB$ROOT钱包的密码。

  • ISOLATED ：PDB配置为使用自己的钱包。要在此配置中打开钱包，必须使用隔离钱包的密码。

FULLY_BACKED_UP

VARCHAR2（9）

指示是否已备份密钥库中的所有密钥

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
