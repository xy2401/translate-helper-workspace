24原始出口

原始Export实用程序（ exp ）将数据从Oracle数据库写入二进制格式的操作系统文件。

此文件存储在数据库外部，可以使用原始导入实用程序将其读入另一个Oracle数据库。

注意：

从Oracle Database 11g开始，不支持原始导出以供一般使用。Oracle Database 11g中唯一支持使用原始导出是将XMLType数据向后迁移到Oracle数据库10g第 2版（10.2）或更早版本。因此，Oracle建议您使用新的数据泵导出和导入实用程序，但以下情况除外，这些情况需要原始导出和导入：

  • 您想要导入使用原始导出实用程序（ exp ）创建的文件。

24.1什么是出口实用程序？

Export实用程序为您在Oracle数据库之间传输数据对象提供了一种简单的方法，即使它们驻留在具有不同硬件和软件配置的平台上也是如此。

对Oracle数据库运行导出时，将提取对象（如表），然后提取其相关对象（如索引，注释和授权）（如果有）。

导出文件是Oracle二进制格式转储文件，通常位于磁盘或磁带上。转储文件可以使用FTP传输或物理传输（在磁带的情况下）到不同的站点。然后，这些文件可以与Import实用程序一起使用，以在未通过网络连接的系统上的数据库之间传输数据。除正常备份过程外，这些文件还可用作备份。

导出转储文件只能由Oracle Import实用程序读取。导入实用程序的版本不能早于用于创建转储文件的导出实用程序的版本。

您还可以显示导出文件的内容，而无需实际执行导入。为此，请使用Import SHOW参数。有关更多信息，请参阅SHOW 。要从ASCII固定格式或分隔文件加载数据，请使用SQL * Loader实用程序。

24.2.1运行catexp.sql或catalog.sql

要使用Export，必须在创建数据库或迁移到较新版本后运行脚本catexp.sql或catalog.sql （运行catexp.sql ）。

catexp.sql或catalog.sql脚本只需在数据库上运行一次。该脚本执行以下任务以准备数据库以进行导出和导入操作：

  • 在数据字典中创建必要的导出和导入视图

  • 创建EXP_FULL_DATABASE和IMP_FULL_DATABASE角色

  • 为EXP_FULL_DATABASE和IMP_FULL_DATABASE角色分配所有必要的权限

  • 将EXP_FULL_DATABASE和IMP_FULL_DATABASE分配给DBA角色

  • 记录已安装的catexp.sql的版本

EXP_FULL_DATABASE和IMP_FULL_DATABASE角色功能强大。在向用户授予这些角色时，数据库管理员应谨慎使用。

24.2.2确保出口业务的足够磁盘空间

在运行“导出”之前，请确保有足够的磁盘或磁带存储空间来写入导出文件。

如果没有足够的空间，则Export会因写入失败错误而终止。

您可以使用表格大小来估计所需的最大空间。您可以在Oracle数据字典的USER_SEGMENTS视图中找到表大小。以下查询显示所有表的磁盘使用情况：

SELECT SUM（BYTES）FROM USER_SEGMENTS WHERE SEGMENT_TYPE ='TABLE';

查询结果不包括用于存储在LOB（大对象）或VARRAY列或分区表中的数据的磁盘空间。

24.2.3验证导出和导入操作的访问权限

要使用Export，您必须具有Oracle数据库的CREATE SESSION特权。

此权限属于在数据库创建期间建立的CONNECT角色。要导出其他用户拥有的表，必须启用EXP_FULL_DATABASE角色。此角色授予所有数据库管理员（DBA）。

如果您没有EXP_FULL_DATABASE角色中包含的系统特权，则无法导出其他用户的架构中包含的对象。例如，即使您为其创建了同义词，也无法在其他用户的架构中导出表。

无法导出多个系统模式，因为它们不是用户模式;它们包含Oracle管理的数据和元数据。未导出的模式示例包括SYS ， ORDSYS和MDSYS 。

24.3.3参数文件

您可以在参数文件中指定所有有效参数及其值。

将参数存储在文件中允许轻松修改或重用它们，并且是调用Export的推荐方法。如果对不同的数据库使用不同的参数，则可以有多个参数文件。

使用任何平面文件文本编辑器创建参数文件。命令行选项PARFILE= filename告诉Export从指定文件而不是从命令行读取参数。例如：

参数文件规范的语法是以下之一：

PARAMETER = value PARAMETER =（ value ）PARAMETER =（ value1，value2，... ）

以下示例显示了部分参数文件列表：

FULL = y FILE = dba.dmp GRANTS = y INDEXES = y CONSISTENT = y

注意：

参数文件的最大大小可能受操作系统的限制。参数文件的名称取决于操作系统的文件命名约定。

您可以在参数文件前面加上井号（＃）来添加注释。导出忽略井号（＃）右侧的所有字符。

您可以在命令行上输入参数的同时指定参数文件。实际上，您可以在两个位置指定相同的参数。PARFILE参数的位置和命令行上的其他参数确定哪些参数优先。例如，假设参数文件params.dat包含参数INDEXES=y并且使用以下行启动Export：

exp PARFILE = params.dat INDEXES = n

在这种情况下，因为INDEXES=n出现在 PARFILE=params.dat ， INDEXES=n会覆盖参数文件中INDEXES参数的值。

24.4导出模式

导出实用程序支持四种操作模式。

特别：

  • 完整：导出完整数据库。只有具有EXP_FULL_DATABASE角色的用户才能使用此模式。使用FULL参数指定此模式。

  • 表空间：允许特权用户将一组表空间从一个Oracle数据库移动到另一个Oracle数据库。使用TRANSPORT_TABLESPACE参数指定此模式。

  • 用户：允许您导出属于您的所有对象（例如表，授权，索引和过程）。以用户模式导入的特权用户可以导入指定用户集的模式中的所有对象。使用OWNER参数在“导出”中指定此模式。

  • 表：允许您导出特定的表和分区。特权用户可以通过指定包含它们的架构来限定表。对于未指定模式名称的任何表，Export默认为导出器的模式名称。使用TABLES参数指定此模式。

有关在每种模式下导出和导入的对象列表，请参阅表24-1 。

注意：

原始导出实用程序不会导出任何使用延迟段创建创建的表，并且没有为其创建段。创建段的最常用方法是将一行存储到表中，但其他操作（如ALTER TABLE ALLOCATE EXTENTS也将创建一个段。如果表中存在某个段并且导出了表，则SEGMENT CREATION DEFERRED子句不包含在由原始Import实用程序执行的CREATE TABLE语句中。

您可以使用常规路径导出或直接路径导出以除表空间模式之外的任何模式导出。常规路径导出和直接路径出口之间的差异描述常规路径导出与直接路径导出 。

24.4.1表级和分区级导出

您可以导出表，分区和子分区。

在所有模式中，分区数据以一种格式导出，以便可以选择性地导入分区或子分区。

  • 表级导出
    导出指定表中的所有数据。
  • 分区级导出
    仅导出指定源分区或子分区中的数据。

父主题： 导出模式

24.4.1.1表级导出

导出指定表中的所有数据。

在表级导出中，您可以导出整个表（分区或非分区）及其索引和其他依赖于表的对象。如果表已分区，则还会导出其所有分区和子分区。这适用于直接路径导出和常规路径导出。您可以在任何导出模式下执行表级导出。

父主题： 表级和分区级导出

24.4.1.2分区级导出

仅导出指定源分区或子分区中的数据。

在分区级别导出中，您可以导出表的一个或多个指定分区或子分区。您只能在表模式下执行分区级导出。

有关如何指定表级别和分区级别导出的信息，请参阅TABLES 。

24.5.11.1完整数据库导出和导入要考虑的要点

完整数据库导出和导入是复制或清理数据库的好方法。

但是，为避免出现问题，请务必牢记以下几点：

  • 完全导出不会导出模式SYS拥有的触发器。您必须在完全导入之前或之后手动重新创建SYS触发器。Oracle建议您在导入后重新创建它们，以防它们定义会阻碍导入进度的操作。

  • 完整导出也不会导出默认配置文件。如果已修改源数据库中的默认配置文件（例如，通过添加模式SYS拥有的密码验证功能），则必须手动预创建该功能，并在导入完成后修改目标数据库中的默认配置文件。

  • 如果可能，在开始之前，请创建导出数据库的物理副本以及要导入的数据库。这可以确保任何错误都是可逆的。

  • 在开始导出之前，建议生成包含以下信息的报告：

    • 表空间和数据文件的列表

    • 回滚段列表

    • 按用户计算每个对象类型（如表，索引等）

    通过此信息，您可以确保已创建表空间并且导入成功。

  • 如果您是从出口创造一个完全新的数据库，然后记得要创建一个额外的回滚段SYSTEM ，并使它在你的初始化参数文件（可init 。 ora)继续进行导入操作前。

  • 执行导入时，请确保指向正确的实例。这非常重要，因为在某些UNIX系统上，只需输入子shell即可更改执行导入操作的数据库。

  • 除非您确定已创建所有表空间，否则请勿对具有多个数据库的系统执行完全导入。完全导入使用与导出的数据库相同的数据文件名创建任何未定义的表空间。这可能会导致以下情况出现问题：

    • 如果数据文件属于任何其他数据库，则它们将被破坏。如果导出的数据库位于同一系统上，则尤其如此，因为其数据文件将由要导入的数据库重用。

24.5.19查询

使用此参数可以在执行表模式导出时从一组表中选择行的子集。

默认

没有。

目的

query参数的值是一个字符串，其中包含将应用于TABLES参数中列出的所有表（或表分区）的SQL SELECT语句的WHERE子句。

例如，如果用户scott只想导出那些职位为SALESMAN且薪水低于1600的员工，那么他可以执行以下操作（此示例基于UNIX）：

exp scott TABLES = emp QUERY = \“WHERE job = \'SALESMAN \'和sal \ <1600 \”

注意：

由于QUERY参数的值包含空格，因此大多数操作系统要求将整个字符串WHERE job=\'SALESMAN\'和sal\<1600放在双引号中或通过某种方法标记为文字。操作系统保留字符也需要以转义字符开头。有关系统上特殊字符和保留字符的信息，请参阅Oracle操作系统特定的文档。

执行此查询时，Export构建类似于以下内容的SQL SELECT语句：

SELECT * FROM emp WHERE job ='SALESMAN'和sal <1600;

为QUERY参数指定的值将应用于TABLES参数中列出的所有表（或表分区）。例如，以下语句将卸载与查询匹配的emp和bonus中的行：

exp scott TABLES = emp，奖金QUERY = \“WHERE job = \'SALESMAN \'和sal \ <1600 \”

同样，Export执行的SQL语句类似于以下内容：

SELECT * FROM emp WHERE job ='SALESMAN'和sal <1600; SELECT * FROM bonus WHERE job ='SALESMAN'和sal <1600;

如果表缺少QUERY子句中指定的列，则将生成错误消息，并且不会为违规表导出任何行。

24.5.26.1表名限制

这是对Export实用程序的表名限制的说明。

以下限制适用于表名：

    某些操作系统要求命令行上的引号前面有转义字符。以下是如何在不同的导出模式中保留区分大小写的示例。

    • 在命令行模式下：

      TABLES = '\ “的Emp \”'

    • 在交互模式下：

      待出口的表格（T）：“Emp”

    • 在参数文件模式中：

      TABLES = ' “的Emp”'

  • 除非表名用引号括起，否则在命令行中指定的表名不能包含井号（＃）。类似地，在参数文件中，如果表名包含井号（＃），则导出实用程序会将该行的其余部分解释为注释，除非表名用引号括起。

    TABLES =（emp＃，dept，mydata）

    但是，给定以下行，Export实用程序将导出所有三个表，因为emp#用引号括起来：

    TABLES =（“emp＃”，dept，mydata）

    注意：

    某些操作系统需要单引号而不是双引号，反之亦然。不同的操作系统对表命名也有其他限制。

例如，如果将不存在的表指定为表模式导出的一部分，则导出实用程序将导出所有其他表。然后它发出警告并成功终止。

24.9常规路径输出与直接路径输出

特别：

  • 常规路径导出

  • 直接路径导出

24.12.1.1字符集排序顺序对转换的影响

创建公共SYNONYM emp for scott.emp;连接blake / paper;在插入ON BEGIN NULL之前创建TRIGGER t_emp;结束; CREATE VIEW emp AS SELECT * FROM dual;

如果导出前面示例中的数据库，则触发器中对emp的引用将引用blake的视图而不是scott的表。当Import尝试重新建立t_emp触发器时，这将导致错误。

24.14.13支持细粒度访问控制

描述了对细粒度访问控制策略的支持。

您可以导出启用了细粒度访问控制策略的表。这样做时，请考虑以下事项：

  • 从包含此类表的导出文件导入的用户必须具有相应的权限（特别是DBMS_RLS包的EXECUTE特权，以便可以恢复表的安全策略）。如果没有正确权限的用户尝试导出启用了细粒度访问策略的表，则只会导出导出器有权读取的那些行。

  • 只有用户SYS或启用了EXP_FULL_DATABASE角色或已被授予EXEMPT ACCESS POLICY用户才能对具有细粒度访问控制的表执行直接路径导出。

24.15可移动表空间

可传输表空间功能使您可以将一组表空间从一个Oracle数据库移动到另一个Oracle数据库。

您无法导出可传输的表空间，然后将它们导入到较低版本级别的数据库中。目标数据库必须与源数据库处于相同或更高版本级别。

要移动或复制一组表空间，必须使表空间为只读，复制这些表空间的数据文件，并使用导出和导入来移动存储在数据字典中的数据库信息（元数据）。必须将数据文件和元数据导出文件都复制到目标数据库。可以使用任何复制平面二进制文件的工具来完成这些文件的传输，例如操作系统复制工具，二进制模式FTP或CD-ROM上的发布。

复制数据文件并导出元数据后，您可以选择将表空间置于读/写模式。

导出和导入提供以下参数以启用可传输表空间元数据的移动。

  • TABLESPACES

  • TRANSPORT_TABLESPACE

有关在导出操作期间使用这些参数的详细信息，请参阅TABLESPACES和TRANSPORT_TABLESPACE 。

24.17.3如何使用导出和导入对数据库迁移进行分区

描述如何使用“导出”和“导入”对数据库迁移进行分区。

要以分区方式执行数据库迁移，请执行以下步骤：

  1. 对于数据库中的所有顶级元数据，请发出以下命令：

    1. exp FILE = full FULL = y CONSTRAINTS = n TRIGGERS = n ROWS = n INDEXES = n

    2. imp FILE =满FULL = y

  2. 对于数据库中的每个模式n ，发出以下命令：

    1. exp OWNER=schema n FILE=schema n

    2. imp FILE=schema n FROMUSER=schema n TOUSER=schema n IGNORE=y

所有出口都可以并行完成。当进口full 。 dmp完成后，所有剩余的导入也可以并行完成。

24.18使用不同版本的导出和导入

描述与使用不同版本的Export和Oracle数据库相关的兼容性问题。

无论何时在Oracle数据库的不同版本之间移动数据，都适用以下基本规则：

  • 导入实用程序和要导入数据的数据库（目标数据库）必须是同一版本。例如，如果您尝试使用导入实用程序9.2.0.7导入9.2.0.8数据库，则可能会遇到错误。

  • Export实用程序的版本必须等于源数据库或目标数据库的版本，以较早者为准。

    例如，要创建导入到更高版本数据库的导出文件，请使用与源数据库等同的Export实用程序版本。相反，要创建导入到早期版本数据库的导出文件，请使用与目标数据库版本相同的Export实用程序版本。

    • 通常，您可以使用任何Oracle8发行版中的导出实用程序从Oracle9 i服务器导出并创建Oracle8导出文件。

24.18.1使用不同版本的导出和导入时的限制

描述在使用不同版本的导出和导入时适用的限制。

特别：

  • 导出转储文件只能由导入实用程序读取，因为它们以特殊的二进制格式存储。

  • 可以将任何导出转储文件导入到Oracle数据库的更高版本中。
