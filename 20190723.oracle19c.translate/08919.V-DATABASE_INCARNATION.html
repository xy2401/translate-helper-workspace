7.158 V $ DATABASE_INCARNATION

V$DATABASE_INCARNATION显示有关所有数据库化身的信息。

每当使用RESETLOGS选项打开数据库时，Oracle都会创建新的版本。V$DATABASE视图中还包含有关当前和前一个版本的记录。

柱 数据类型 描述

INCARNATION＃

数

控制文件中分支记录的记录ID

RESETLOGS_CHANGE＃

数

Resetlogs系统更改编号（SCN）用于当前行的化身

RESETLOGS_TIME

日期

Resetlogs当前行的化身的时间戳

PRIOR_RESETLOGS_CHANGE＃

数

重置日志SCN为前一个版本

PRIOR_RESETLOGS_TIME

日期

重置日志前一个版本的时间戳

状态

VARCHAR2（7）

化身状态：

  • ORPHAN - 孤儿化身

  • CURRENT - 数据库的当前版本

  • PARENT - 当前化身的父母

RESETLOGS_ID

数

当前行的化身的分支ID（由用户管理的恢复/ RMAN还原使用，以获取跨化身的归档日志的唯一名称）

PRIOR_INCARNATION＃

数

父母化身记录ID，如果非零

FLASHBACK_DATABASE_ALLOWED

VARCHAR2（26）

指示闪回数据库是否可以在执行中的SCN或时间戳中执行。值为YES意味着您可以闪回到该化身中的某个点。值NO表示您不能闪回到道成肉身。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
