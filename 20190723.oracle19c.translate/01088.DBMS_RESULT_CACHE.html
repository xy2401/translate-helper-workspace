138 DBMS_RESULT_CACHE

DBMS_RESULT_CACHE包提供了一个接口，允许DBA管理SQL结果缓存和PL / SQL函数结果缓存使用的共享池的那部分。

这两个缓存都使用相同的基础结构。因此，例如， DBMS_RESULT_CACHE 。 BYPASS确定是绕过两个缓存还是使用两个缓存，以及DBMS_RESULT_CACHE 。 FLUSH刷新SQL查询的所有缓存结果和PL / SQL函数的所有缓存结果。

138.3.5 BYPASS程序

此过程为结果缓存设置旁路模式。

它设置以下旁路模式之一：

  • 打开旁路模式时，意味着不再使用缓存结果，并且不会在缓存中保存新结果。

  • 当旁路模式关闭时，缓存恢复正常操作。

句法

DBMS_RESULT_CACHE.BYPASS（bypass_mode IN BOOLEAN，session IN BOOLEAN）;

参数

表138-6 BYPASS程序参数

参数 描述

bypass_mode

  • TRUE =>结果缓存使用率缓存

  • FALSE =>结果缓存使用率已打开

会议

  • TRUE =>适用于当前会话

  • FALSE （默认值）=>适用于所有会话

使用说明

此操作是特定于数据库实例的。

例子

当需要在正在运行的系统中热修补PL / SQL代码时，可以使用此操作。如果将代码补丁应用于结果缓存函数直接或传递依赖的PL / SQL模块，则不会自动刷新与结果缓存函数关联的缓存结果（如果实例未重新启动/退回）。这必须手动实现。

要在修补过程中确保正确性，请执行以下步骤：

  1. 将结果缓存置于旁路模式，并刷新现有结果。

    BEGIN DBMS_RESULT_CACHE.BYPASS（TRUE）; DBMS_RESULT_CACHE.FLUSH;结束; /

    如果在Oracle Real Application Clusters环境中，则必须在每个实例上执行此步骤。

  2. 应用PL / SQL代码补丁。

  3. 通过关闭缓存旁路模式继续使用结果缓存。

    BEGIN DBMS_RESULT_CACHE.BYPASS（FALSE）;结束; /

    如果在Oracle Real Application Clusters环境中，则必须在每个实例上执行此步骤。

138.3.6 FLUSH功能和程序

此函数和过程尝试从结果缓存中删除所有对象，并根据参数保留或释放内存并保留或清除统计信息。

句法

DBMS_RESULT_CACHE.FLUSH（retainMem IN BOOLEAN DEFAULT FALSE，retainSta in BOOLEAN DEFAULT FALSE）RETURN BOOLEAN;
DBMS_RESULT_CACHE.FLUSH（retainMem IN BOOLEAN DEFAULT FALSE，retainSta IN BOOLEAN DEFAULT FALSE）;

参数

表138-7 FLUSH功能和程序参数

参数 描述

retainMem

  • TRUE =>保留缓存中的空闲内存

  • FALSE （默认值）=>释放系统的可用内存

retainSta

  • TRUE =>保留现有的缓存统计信息

  • FALSE （默认值）=>清除现有缓存统计信息

返回值

如果成功删除所有对象，则为TRUE 。

138.3.10状态功能

此函数检查结果缓存的状态。

句法

DBMS_RESULT_CACHE.STATUS RETURN VARCHAR2;

返回值

以下值之一：

  • STATUS_DISA - 缓存不可用

  • STATUS_ENAB - 缓存可用

  • STATUS_BYPS ：缓存已暂时STATUS_BYPS用。

  • STATUS_SYNC - 缓存可用，但与Oracle RAC节点同步
