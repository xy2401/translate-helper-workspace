Oracle Grid Infrastructure的服务器配置清单

使用此核对表可以检查Oracle Grid Infrastructure安装的最低服务器配置要求。

表1-3 Oracle Grid Infrastructure的服务器配置清单

校验 任务
分配给临时文件系统的磁盘空间

临时磁盘空间（ /tmp ）目录中至少有1 GB的空间。
交换相对于RAM的空间分配

4 GB到16 GB之间：等于RAM
超过16 GB：16 GB
注意：如果为Linux服务器启用HugePages，则应在计算交换空间之前从可用RAM中扣除分配给HugePages的内存。

HugePages内存分配

将内存分配给足够大的HugePages，以便计划在群集上运行的所有数据库的系统全局区域（SGA），并为Grid Infrastructure Management Repository提供系统全局区域。
软件二进制文件的挂载点路径 Oracle建议您创建最佳灵活体系结构配置，如适用于您的平台的Oracle Grid Infrastructure安装和升级指南中的“最佳灵活体系结构”附录中所述。
确保Oracle主目录（您为Oracle数据库选择的Oracle主路径）仅使用ASCII字符

ASCII字符限制包括安装所有者用户名，它们用作某些主路径的默认值，以及您可以为路径选择的其他目录名。

设置区域设置（如果需要）

指定要在其中使用Oracle组件的语言和区域或区域设置。区域设置是运行系统或程序的语言和文化环境。NLS（本地语言支持）参数确定服务器和客户端上特定于语言环境的行为。组件的区域设置确定组件的用户界面的语言以及全球化行为，例如日期和数字格式。

设置群集时间同步的网络时间协议

Oracle Clusterware要求在所有群集节点上设置相同的时区环境变量。

确保使用操作系统配置的网络时间协议（NTP）或Oracle Cluster Time Synchronization Service在所有群集节点上设置时区同步。

检查共享内存文件系统挂载

默认情况下，您的操作系统在/etc/fstab包含一个用于挂载/dev/shm的条目。但是，如果您的群集验证实用程序（CVU）或Oracle Universal Installer（OUI）检查失败，请确保/dev/shm安装区域的类型为tmpfs并使用以下选项进行安装：

  • rw和exec权限设置在它上面

  • 没有设置noexec或nosuid

注意：您的操作系统通常将这些选项设置为默认权限。如果它们是由操作系统设置的，则它们不会在挂载选项中列出。
