准备升级现有Oracle Clusterware安装

如果您已安装现有Oracle Clusterware，则可以通过执行非现场升级来升级现有群集。您无法执行就地升级。

以下主题列出了升级Oracle Grid Infrastructure之前可以执行的步骤：

  • Oracle Grid Infrastructure的升级核对表
    在升级现有Oracle Grid Infrastructure之前，请查看此核对表。正在升级群集，直到所有群集成员节点都在运行新安装，并且新的群集件将成为活动版本。
  • 在升级Oracle Grid Infrastructure之前检查完成
    升级Oracle Grid Infrastructure之前，请完成以下任务。
  • 将Oracle Clusterware文件从NFS移动到Oracle ASM
    您可以将Oracle Cluster Registry（OCR）和投票文件从网络文件系统（NFS）移动到Oracle自动存储管理（Oracle ASM）磁盘组。
  • 运行Oracle ORAchk升级准备评估
    在升级Oracle Grid Infrastructure之前，请下载并运行Oracle ORAchk升级准备情况评估。
  • 使用CVU验证Oracle Clusterware升级的准备情况
    Oracle建议您使用Cluster Verification Utility（CVU）来帮助确保升级成功。
  • 使用Dry-Run升级模式检查系统升级准备情况
    使用Oracle Grid Infrastructure安装向导gridSetup.sh干运行升级模式来检查Oracle Clusterware升级的准备情况。
