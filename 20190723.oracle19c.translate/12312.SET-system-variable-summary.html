12.41.1 SET APPI [NFO] {ON | 关闭 | 文字 }

设置自动通过注册脚本 DBMS_APPLICATION_INFO包。

这样可以让DBA监视每个脚本的性能和资源使用情况。注册名称显示在的MODULE列中 V $ SESSION和 V $ SQLAREA虚拟表。您还可以使用DBMS_APPLICATION_INFO.READ_MODULE过程读取已注册的名称。

ON注册由@，@@或START命令调用的脚本。OFF禁用脚本注册。而是注册文本的当前值。 text指定在没有运行脚本或APPINFO为OFF时要注册的文本，这是默认值。文本的默认值为“SQL * Plus”。如果为文本输入多个单词，则必须将它们用引号括起来。文本的最大长度受DBMS_APPLICATION_INFO程序包的限制。

注册名称的格式为nn @ xfilename ，其中： nn是脚本的深度级别;脚本名称被截断时x为'<'，否则为空;和filename是脚本名称，可能被截断为DBMS_APPLICATION_INFO包接口允许的长度。

12.41.2 SET ARRAY [SIZE] { 15 | n }

设定数量 SQL * Plus将一次从数据库中获取的行。

有效值为1到5000。较大的值会提高获取许多行的查询和子查询的效率，但需要更多内存。超过大约100的值几乎不会增加性能。除了提高效率之外，ARRAYSIZE对SQL * Plus操作的结果没有影响。

12.41.3 SET AUTO [COMMIT] {ON | 关闭 | IMM [EDIATE] | n }

控制Oracle数据库在SQL或PL / SQL命令之后提交对数据库的挂起更改的时间 。

在Oracle数据库执行每个成功的INSERT，UPDATE或DELETE或PL / SQL块之后，ON将提交对数据库的挂起更改。OFF禁止自动提交，因此您必须手动提交更改（例如，使用SQL命令COMMIT）。IMMEDIATE的功能与ON相同。 n条提交未决更改数据库Oracle数据库执行经过n成功SQL插入，更新，或删除的命令，或PL / SQL块。 n不能小于零或大于2,000,000,000。成功完成n INSERT，UPDATE或DELETE命令或PL / SQL块，提交，回滚或SET AUTOCOMMIT命令后，语句计数器将重置为零。

当SQL * Plus退出时，SET AUTOCOMMIT不会改变提交行为。默认情况下会提交任何未提交的数据。

注意：

对于此功能，PL / SQL块被视为一个事务，无论其中包含的SQL命令的实际数量如何。

12.41.6 SET AUTOT [RACE] {ON | 关闭 | TRACE [仅]} [EXP [LAIN]] [STAT [ISTICS]]

显示一个报告成功SQL的执行情况 DML语句（例如SELECT，INSERT，UPDATE，DELETE或MERGE）。

该报告可以包括执行统计和查询执行路径。

如果动态采样有效，则DML的SQL * Plus报表输出可能不同。

OFF不显示跟踪报告。ON显示跟踪报告。TRACEONLY显示跟踪报告，但不打印查询数据（如果有）。EXPLAIN通过执行EXPLAIN PLAN来显示查询执行路径。 STATISTICS显示SQL语句统计信息。有关EXPLAIN PLAN的更多信息，请参阅EXPLAIN PLAN。

使用ON或TRACEONLY没有显式选项默认为EXPLAIN STATISTICS。

TRACEONLY选项可用于抑制大型查询的查询数据。如果指定了STATISTICS，则SQL * Plus仍会从服务器获取查询数据，但不会显示数据。

语句成功完成后将打印AUTOTRACE报告。

当SQL * Plus生成STATISTICS报告时，将自动创建与数据库的第二个连接。当STATISTICS选项设置为OFF或您注销SQL * Plus时，此连接将关闭。

AUTOTRACE报告的格式可能会有所不同，具体取决于您连接的服务器的版本和服务器的配置。连接到Oracle数据库10g （版本10.1）或更高版本时，支持AUTOTRACE PLAN的附加信息和表格输出。连接到早期数据库时，将使用旧表单或AUTOTRACE报告。

启用FIPS标记时，AUTOTRACE不可用。

有关AUTOTRACE的更多信息，请参见关于跟踪语句 。
