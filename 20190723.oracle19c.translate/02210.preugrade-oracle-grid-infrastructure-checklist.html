Oracle数据库升级指南
补丁集（推荐） 为现有安装安装最新的修补程序集版本。在升级Oracle Grid Infrastructure之前，请查看My Oracle Support说明2180188.1以获取最新补丁的列表。

安装用户帐户 确认您计划使用的安装所有者与拥有要升级的安装的安装所有者相同。

Oracle ASM的实例名称

Oracle自动存储管理（Oracle ASM）实例必须使用标准Oracle ASM实例名称。

单实例数据库的默认ASM SID是+ ASM。
群集名称和站点名称 群集名称必须具有以下特征：

  • 至少一个字符但长度不超过15个字符。

  • 连字符（ - ）和单字节字母数字字符（a到z，A到Z和0到9）。

  • 它不能以数字字符开头。

  • 它不能以连字符（ - ）开头或结尾。
操作系统 确认您使用的是受支持的操作系统，内核版本以及新Oracle Grid Infrastructure安装所需的所有操作系统软件包。
标准Oracle Grid Infrastructure的网络地址

对于标准Oracle Grid Infrastructure安装，请确认以下网络配置：

  • 私有和公共IP地址位于不相关的独立子网中。私有子网应位于专用私有子网中。

  • 公共和虚拟IP地址（包括SCAN地址）位于同一子网中（子网网络的子网掩码允许的地址范围）。

  • 私有IP地址和公共IP地址都不使用链路本地子网（169.254。*。*）。
原始或块设备上的OCR 将OCR文件从RAW或块设备迁移到Oracle ASM或支持的文件系统。不支持直接使用RAW和Block设备。

运行ocrcheck命令以确认Oracle Cluster Registry（OCR）文件的完整性。如果此检查失败，则在继续之前修复OCR。

检查GIMR的空间

升级到Oracle Grid Infrastructure 19c时，仅当源Grid Home配置了GIMR时才会创建新的GIMR。按Oracle Clusterware存储空间要求中的说明分配额外的存储空间。

从Oracle Grid Infrastructure 12c第2版（12.2）升级时，GIMR将保留其内容。
Oracle ASM密码文件

从Oracle Grid Infrastructure 12c第1版（12.1），Oracle Grid Infrastructure 12c第2版（12.2）或Oracle Grid Infrastructure 18c升级到Oracle Grid Infrastructure 19c时，将Oracle ASM密码文件从文件系统升级到Oracle ASM，然后再继续使用以下ASMCMD命令进行升级：

ASMCMD [+]> pwcopy --asm current_location_of_ASM_password_file_in_OS_directory + target_disk_group_name / orapwASM

从Oracle Grid Infrastructure 11g第2版（11.2）升级到Oracle Grid Infrastructure 19c时，请在升级后将Oracle ASM密码文件从文件系统移至Oracle ASM。

注意：

在移动密码文件之前，将compatible.asm设置为至少12.1.0.2。
CVU升级验证 使用群集验证实用程序（CVU）帮助您进行系统检查，以准备开始升级。
取消设置环境变量

在执行升级的用户时，取消设置环境变量$ORACLE_HOME和$ORACLE_SID 。

检查是否未设置ORA_CRS_HOME环境变量。除了在Oracle Support的明确指示下，不要将ORA_CRS_HOME用作环境变量。

有关要取消设置的环境变量的完整列表，请参阅升级Oracle Grid Infrastructure之前的检查完成 。

使用干运行升级检查系统升级准备情况 在干运行升级模式下运行Oracle Grid Infrastructure安装向导gridSetup.sh ，以执行Oracle Grid Infrastructure升级的系统准备情况检查。
RACcheck升级准备评估 下载并运行RACcheck升级准备情况评估，以获取针对Oracle Grid Infrastructure升级的自动升级特定运行状况检查。请参阅My Oracle Support说明1457357.1，该说明可从以下URL获得：

https://support.oracle.com/rs?type=doc&id=1457357.1
在升级之前备份Oracle软件 在对Oracle软件进行任何更改之前，Oracle建议您创建Oracle软件和数据库的备份。
HugePages内存分配 将内存分配给足够大的HugePages，以便计划在群集上运行的所有数据库的系统全局区域（SGA），并为Grid Infrastructure Management Repository提供系统全局区域。
升级前删除Oracle ACFS文件系统的加密

为避免数据损坏，请确保在升级之前删除Oracle ACFS文件系统的加密。
