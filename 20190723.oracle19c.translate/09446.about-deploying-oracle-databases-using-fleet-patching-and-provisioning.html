关于使用Oracle Fleet修补和配置部署Oracle数据库

Oracle Fleet Patching and Provisioning（Oracle FPP）提供了一种标准化方法，用于在单个群集中的任意数量的节点上配置，修补和升级软件家庭。

注意：

从Oracle Database 19c开始，Rapid Home Provisioning功能称为Oracle Fleet Patching and Provisioning或Oracle FPP。

通过Oracle Fleet Patching和Provisioning，您可以创建，存储和管理Oracle家庭模板，作为Oracle数据库，网格基础架构，中间件和应用程序的映像（称为黄金映像）。这些黄金映像在Oracle Fleet Patching and Provisioning Server上维护，可以在信息技术产业中的任意数量的节点上提供和维护映像的工作副本。

Oracle Fleet补丁和配置

使用Oracle FPP部署Oracle软件具有以下优势：

  • 使您能够创建和管理Oracle Real Application Clusters（Oracle RAC），单实例和Oracle Real Application Clusters One节点（Oracle RAC One Node）数据库。您可以部署具有或不具有Oracle Grid Infrastructure的单实例数据库。Oracle Multitenant也受支持。

  • 通过黄金映像和已部署软件的管理系列确保标准化并实现高度自动化。

  • 支持变更管理。借助标准化的Oracle家庭，管理员可以更好地控制托管的Oracle软件，并可以通过单一位置轻松管理软件的大规模部署，修补和升级，以进行变更管理。

  • 最大限度地减少修补和升级期间的停机时间，简化回滚，并使大型系统的配置更容易，更高效。

  • 确保轻松维护和修补数据库软件。

  • 减少修补软件映像的累积时间，因为单个Oracle主目录可用于许多数据库实例。
