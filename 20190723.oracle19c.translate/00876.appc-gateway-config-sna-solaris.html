8在Solaris上配置SNA通信包

Oracle Database Gateway for APPC使用SNA高级程序编程通信（APPC / LU6.2）协议与OLTP进行通信。 Solaris操作系统（SPARC 64位）上的APPC支持由。提供 SNAP-IX产品。

以下主题描述如何在Solaris系统上配置SNAP-IX以运行Oracle Database Gateway for APPC。

处理入站连接

许多 OLTP提供用于操纵入站（客户端）APPC会话请求的安全行为的选项。有关此问题的详细信息，请参阅OLTP的相应文档。

请注意 CICS，网关提供以下支持：

  • 网关完全支持ATTACHSEC=LOCAL ， ATTACHSEC=IDENTIFY ， ATTACHSEC=VERIFY和ATTACHSEC=MIXIDPE 。

  • ATTACHSEC=PERSISTENT ，在。中指定 CICS CONNECTION定义需要网关中尚不可用的功能。

在你开始之前

您必须指定系统特有的参数才能正确配置SNAP-IX版本6。在开始之前，请从网络管理员处请求这些参数。

SNAP-IX配置工具

所有SNAP-IX产品配置均使用xsnaadmin程序完成。此工具是一个X-Windows应用程序，它提供了一个图形界面，用于查看和修改当前SNAP-IX配置以及主机SNA节点的当前运行状态。

为网关创建SNAP-IX配置文件

Oracle Database Gateway for APPC需要一组存储的定义，称为侧信息配置文件，用于支持网关和网关服务器之间的连接。每个配置文件由配置文件名称和配置文件类型组成，配置文件类型是描述配置文件的一组字段。给定配置文件类型中的字段通常是操作参数值和与配置文件相关的其他SNA配置文件的名称的混合。APPC的每个功能部分，例如模式，远程事务程序名称和逻辑单元（LU），由不同的配置文件类型描述。

独立对抗的LU

网关配置可以适应独立或依赖逻辑单元。如果您选择使用从属LU，或仅限于使用从属LU，则网关可以正常运行。如果正确定义了从属LU，则无需更改网关的配置，也不需要对网关服务器进行任何更改。但是，Oracle建议您为网关使用独立的LU，因为它们支持多个并行会话或会话。这意味着多个Oracle客户端应用程序可以通过独立LU与同一网关服务器同时处于活动状态。

与独立LU相比，从属LU仅支持单个活动会话。CP（节点的控制点）将来自已经活动的对话后面的网关的每个附加会话请求排队。换句话说，对话LU是单线程的。

依赖LU的操作影响是第一个客户端应用程序可以通过网关与网关服务器发起对话，但是该会话处于活动状态（可能是几秒，几分钟或几小时，具体取决于客户端应用程序和事务的方式设计），启动与同一网关服务器的会话的任何其他客户端应用程序似乎挂起，因为它等待前一个会话。

如果生产应用程序在任何时候确实只使用一个对话，那么应该没有问题。但是，在某些时候，您可能需要额外的并发对话来进行测试或进行其他应用程序开发。进行多个会话需要在远程主机上定义其他从属LU。必须将其他配置条目添加到SNAP-IX。应定义附加的侧信息配置文件以使用新的从属LU。应创建网关实例并将其配置为使用这些新的辅助信息配置文件。

为网关创建SNA定义

SNAP-IX定义存储在以下文件中，这些文件位于/etc/opt/sna目录中：

  • SNA节点定义：sna_node.cfg

  • SNA域定义： sna_domn.cfg

这些文件是使用。创建和维护的 xsnaadmin工具。SNA定义的维护通常由具有管理权限的用户完成。以下信息适用于为网关创建SNA定义的用户。在阅读本节之前，您必须具备一些SNA知识。

样品SNAP-IX定义

$ORACLE_HOME/dg4appc/sna子目录包含一组用于网关的示例SNAP-IX定义文件，这些文件是使用 xsnaadmin 。这些示例文件是sna_domn.cfg和sna_node.cfg 。SNA定义非常特定于主机和SNA网络。因此，如果没有为本地主机和SNA网络定制，所提供的样本定义将无法工作。

配置SNAP-IX

本节介绍使用xsnaadmin为SNAP-IX创建SNA定义的过程。所有配置都使用xsnaadmin的各种下拉菜单和面板xsnaadmin 。以下配置说明遵循提供的示例。请为您的本地主机和SNA网络定制各种SNA值。

启动xsnaadmin

使用以下命令调用xsnaadmin 。必须正确设置DISPLAY环境变量。如果从本地控制台运行xsnaadmin ，则应已设置DISPLAY 。如果从远程X显示器运行xsnaadmin ，则将DISPLAY设置为该显示器的主机名或IP地址。 

$ DISPLAY = < your_display >：0 $ export DISPLAY $ xsnaadmin＆

在启动xsnaadmin ，将打开主屏幕并显示本地SNA节点的当前配置。

配置SNA节点

要配置SNA节点，您需要执行以下操作：

  1. 从“服务”菜单中，选择“ 配置节点参数” 。
  2. 在“节点参数”对话框中，根据需要输入APPN支持类型，控制点名称，控制点别名和节点ID。控制点名称由SNA网络名称和本地主机的CP名称组成。
  3. 单击确定 。
添加端口

要添加新端口，请从“服务”菜单中选择“ 连接”和“新端口” 。

  1. 在“添加到节点名”对话框中，选择“端口类型”，然后单击“ 确定” 。
  2. 在SAP对话框中，输入端口名称和网卡号。端口名称用于逻辑命名您正在使用的物理网卡，用于将服务访问端口绑定到SNA协议的卡。通常，您可以接受对话框中提供的值。但是，如果需要其他网卡，请输入dmesg命令报告的卡号。
  3. 单击确定 。
创建一个链接站

定义端口后，您必须创建链接站。Link Station表示网关服务器的远程主机的SNA节点。但在创建Link Station之前，必须按照以下过程中的说明创建远程节点定义：

  1. 从“服务”菜单中，选择“ APPC”和“添加远程节点” 。

  2. 在对话框中，输入远程节点的SNA CPNAME，然后单击“ 确定” 。

现在您可以按如下方式创建Link Station：

  1. 从“服务”菜单中，选择“ 连接”和“新建链接站” 。在对话框中，选择先前定义的端口，然后单击“ 确定” 。
  2. 在“链接站”对话框中，输入“链接站”的名称，选择“SNA端口名称”，“链接激活类型”和“LU流量类型”。要获得最大的灵活性，请选择任意选项。
  3. 对于独立LU流量，请指定远程节点名称。单击“ 远程节点”并选择先前创建的节点，然后单击“ 确定” 。选择“远程”节点的类型，通常是“网络”节点。
  4. 对于从属LU流量，请指定远程节点的角色，通常为“主机”，本地节点ID和可选的远程节点ID。
  5. 指定联系信息。联系信息包含远程主机的MAC地址以及SAP编号。
  6. 单击“ 高级”以获取Link Station的其他参数。令牌环参数对话框显示链接站的其他参数。这些参数会影响初始XID联系和重新传输时间和限制。您通常不需要更改默认值。
  7. 单击确定 。
创建本地LU

完成远程节点定义后，为本地主机创建本地LU名称，如下所示：

  1. 从“服务”菜单中，选择“ APPC”和“新建本地LU” 。在本地LU对话框中，输入本地LU的名称和别名。此名称必须与UNIX主机的远程网关服务器主机上的VTAM定义相对应。
  2. 单击确定 。
创建合作伙伴LU

现在定义一个Partner LU，它代表网关服务器用于通信的LU。

  1. 从“服务”菜单中，选择“远程节点上的 APPC和新伙伴LU和伙伴LU” 。
  2. 在“合作伙伴LU”对话框中，输入合作伙伴LU名称和特征。合作伙伴LU名称包含SNA网络名称以及远程LU的LU名称。启用并行会话支持。该位置是远程节点名称的名称。您可以单击“ 位置 ”以获取列表。
  3. 单击确定 。
创建模式和CPI-C配置文件

在创建了本地和远程LU定义后，创建必要的模式和CPI-C定义。

  1. 从“服务”菜单中，选择“ APPC和模式” 。在“模式”对话框中，单击“ 新建”以添加新模式。
  2. 在“模式”对话框中，输入“模式名称”和其他会话参数。网关模式的推荐名称是CICSPGA 。请与Remote Host系统管理员联系以获取适当的模式参数。
  3. 单击确定 。
  4. 现在已经定义了模式，创建CPI-C侧信息配置文件，网关将其用作连接名称。从菜单中选择APPC和CPI-C 。
  5. 在“CPI-C目标名称”对话框中，单击“ 新建”以添加新的“配置文件”。
  6. 在CPI-C目标对话框中，输入配置文件名称，本地LU名称，伙伴TP，伙伴LU和模式以及安全选项。伙伴TP名称是主机事务程序的名称或要在TIP中覆盖的虚拟值。
  7. 对于本地LU，您可以指定特定的LU或选择默认的LU。对于合作伙伴LU，请输入完整的LU名称或先前创建的别名。
  8. 输入ORAPLU62作为模式名称。选择要使用的这些会话的安全类型。这会影响会话授权的完成方式。
  9. 单击确定 。

测试连接

在继续进行网关配置任务之前，请确保您的连接正常。通过启动SNAP-IX节点然后启动各个链接站来执行此操作。
