1.85 DB_LOST_WRITE_PROTECT

DB_LOST_WRITE_PROTECT启用或禁用丢失的写入检测。

属性 描述

参数类型

串

句法

DB_LOST_WRITE_PROTECT = {NONE |典型的|完全}

默认值

没有

可修改

改变系统

可在PDB中修改

没有

基本

没有

Oracle RAC

在Oracle RAC实例中，参数值是系统范围的。

当I / O子系统确认块写入完成时，会发生数据块丢失写入，而实际上写入不会发生在持久存储中。

当主数据库上的参数设置为TYPICAL时，实例会在重做日志中记录缓存读取/写入表空间的缓存读取，这对于检测丢失的写入是必需的。

当主数据库上的参数设置为FULL时，实例日志将读取只读表空间和读/写表空间。

在介质恢复期间，在备用数据库或主数据库上将参数设置为TYPICAL或FULL时，实例将执行丢失写入检测。

在主数据库或备用数据库上将参数设置为NONE时，不会启用丢失写入检测功能。
