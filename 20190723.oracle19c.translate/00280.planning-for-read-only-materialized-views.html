38规划只读物化视图

在开始规划只读物化视图环境之前，了解与物化视图相关的概念和体系结构非常重要。在了解了只读物化视图的概念和体系结构之后，规划只读物化视图环境时需要考虑重要因素。

  • 主表的注意事项
    对于主表，您必须考虑主键，外键和数据类型。
  • 规划主数据库和物化视图数据库
    在只读物化视图环境中规划数据库包括准备物化视图和配置物化视图日志。

38.1主表的注意事项

对于主表，您必须考虑主键，外键和数据类型。

  • 主键和主表
    如果可能，每个主表都应该有一个主键。
  • 外键和主表
    在使用外键引用约束复制表时，Oracle建议您始终索引外键列并复制这些索引，除非父表中不允许更新和删除。索引不会自动复制。
  • 主表的数据类型注意事项
    数据类型和主表有几个注意事项。
  • 不支持的表类型
    物化视图不能基于某些类型的表。

父主题： 规划只读实体化视图

38.1.1主键和主表

如果可能，每个主表都应该有一个主键。

如果主键不可用，则每个主表必须具有一组列，这些列可用作表的每一行的唯一标识符。如果计划在复制环境中使用的表没有主键或一组唯一列，则相应地更改这些表。此外，如果您计划基于主表创建任何主键物化视图，则该主服务器必须具有主键。

父主题： 主表的注意事项

38.1.2外键和主表

在使用外键引用约束复制表时，Oracle建议您始终索引外键列并复制这些索引，除非父表中不允许更新和删除。索引不会自动复制。

父主题： 主表的注意事项

38.1.3主表的数据类型注意事项

数据类型和主表有几个注意事项。

您可以基于包含使用以下数据类型的列的主表创建只读实例化视图：

  • VARCHAR2

  • NVARCHAR2

  • 数

  • 日期

  • TIMESTAMP

  • TIMESTAMP WITH TIME ZONE

  • TIMESTAMP LOCAL TIME ZONE

  • INTERVAL YEAR TO MONTH

  • INTERVAL DAY TO SECOND

  • 生的

  • ROWID

  • CHAR

  • NCHAR

  • 带有BASICFILE存储的CLOB

  • NCLOB与BASICFILE存储

  • 带有BASICFILE存储的BLOB

  • XMLType存储为CLOB

  • 用户定义的类型，不使用类型继承或类型演变

  • Oracle提供的类型不使用类型继承或类型演变

注意：

不推荐使用存储为CLOB XMLType 。

您不能在WHERE化视图的定义查询的WHERE子句中引用LOB列。

您可以创建使用用户定义类型的实体化视图，包括列对象，对象表， REF ，varrays和嵌套表。

您无法基于具有使用以下数据类型的列的主表创建实例化视图：

  • 浮动

  • BINARY_FLOAT

  • BINARY_DOUBLE

  • 长

  • LONG RAW

  • 带有SECUREFILE存储的CLOB

  • NCLOB与SECUREFILE存储

  • 带有SECUREFILE存储的BLOB

  • BFILE

  • XMLType存储对象关系或二进制XML

  • Expression类型

  • 用户定义的类型，使用类型继承或类型evolution

  • Oracle提供的类型继承或类型演变的类型

您应该将LONG数据类型转换为具有BASICFILE存储的LOB。

物化视图不能基于某些类型的表。

38.2.4为物化视图做准备

物化视图复制遇到的大多数问题都是由于没有正确准备环境。

在创建物化视图环境之前，请确保满足以下先决条件：

  • 确保存在所需的模式。

  • 确保存在所需的数据库链接。

  • 确保授予所需的权限。

  • 确保退出足够的作业进程。

  • 物化视图数据库中的必需模式
    包含远程数据库中的实例化视图的模式必须与包含master数据库中的主表的模式相对应。
  • 物化视图所需的数据库链接
    物化视图的定义查询可以使用一个或多个数据库链接来引用远程表数据。
  • 所需特权
    物化视图的创建者和所有者必须能够发出物化视图的定义SELECT语句。
  • 足够的工作流程
    分配足够的作业流程来处理复制环境的自动化非常重要。作业进程自动刷新实例化视图。

父主题： 规划主数据库和物化视图数据库

38.2.4.1物化视图数据库中的必需模式

包含远程数据库中的实例化视图的模式必须与包含master数据库中的主表的模式相对应。

因此，请确定包含要使用实例化视图进行复制的主表的模式。在master数据库中标识目标模式后，在远程数据库中创建具有相同名称的相应帐户。例如，如果所有主表都在ny.example.com数据库的sales模式中，则在物化视图数据库sf.example.com创建相应的sales模式。

38.2.4.2物化视图所需的数据库链接

物化视图的定义查询可以使用一个或多个数据库链接来引用远程表数据。

在创建实体化视图之前，您计划使用的数据库链接必须可用。此外，数据库链接用于访问远程数据库的帐户定义了Oracle数据库创建的安全上下文，并随后刷新了实例化视图。

为了确保正确的行为，物化视图的定义查询必须使用在其定义中包含嵌入式用户名和密码的数据库链接;创建实例化视图时，不能使用公共数据库链接。具有嵌入式名称和密码的数据库链接始终使用指定的帐户建立与远程数据库的连接。此外，链接使用的远程帐户必须具有访问实例化视图的定义查询中引用的数据所需的SELECT权限。

在创建实体化视图之前，必须创建多个管理数据库链接。具体来说，您应该创建从物化视图数据库到主数据库的PUBLIC数据库链接。这样做可以更轻松地定义私有数据库链接，因为您不需要在每个链接中包含USING子句。

例如，以下语句创建从实例化视图数据库到主数据库的公共数据库链接：

使用'orc1.example.com'创建公共数据库链接orc1.example.com;

创建管理数据库链接后，必须创建专用数据库链接，将物化视图数据库中的每个复制的物化视图模式连接到主数据库中的相应模式。确保在物化视图数据库的每个专用数据库链接中嵌入关联的主数据库帐户信息。例如，物化视图数据库中的hr模式应具有指向使用hr用户名和密码连接的主数据库的专用数据库链接。

例如，以下语句创建从实例化视图数据库到主数据库的专用数据库链接：

创建数据库链接orc1.example.com连接到myuser IDENTIFIED BY 密码 ;

38.2.4.4足够的工作流程

分配足够的作业流程来处理复制环境的自动化非常重要。作业进程自动刷新实例化视图。

根据物化视图复制的性质，每个物化视图数据库通常具有一个到主数据库的预定链接，并且至少需要一个作业过程。物化视图数据库通常需要一到三个作业进程，具体取决于用户定义的作业和计划的链接。此外，每个并行度至少需要一个作业流程。

或者，如果您的用户负责通过应用程序界面手动刷新实体化视图，则无需创建计划链接，并且物化视图数据库只需要少一个作业流程。

使用JOB_QUEUE_PROCESSES初始化参数定义作业进程。此初始化参数是可修改的。因此，您可以在实例运行时对其进行修改。Oracle数据库自动确定作业进程的间隔。也就是说，Oracle数据库确定作业进程何时“唤醒”以执行作业。

38.2.5创建物化视图日志

在主表上创建物化视图日志，以便可以快速刷新基于主表的物化视图。

在为远程物化视图数据库创建实例化视图之前，请确保在master数据库中创建必要的实例化视图日志。每个主表都需要物化视图日志，该表支持至少一个具有快速刷新的物化视图。

要创建物化视图日志，您需要以下权限：

  • CREATE ANY TABLE

  • CREATE ANY TRIGGER

  • SELECT （在物化视图日志的主表上）

  • COMMENT ANY TABLE

要创建物化视图日志：

  1. 以具有所需权限的用户身份连接到master数据库，以在目标表上创建实例化视图日志。
  2. 运行CREATE MATERIALIZED VIEW LOG语句。

    在对象表上创建实例化视图日志时，必须通过指定WITH OBJECT ID子句来记录对象标识符，但如果对象标识符是基于主键，则还可以指定记录主键。

示例38-1创建物化视图日志

创建物质化的视图登录hr.employees;

示例38-2在对象表上创建实体化视图日志

以下SQL语句创建categories_typ用户定义的类型：

CREATE TYPE oe.category_typ AS OBJECT（category_name VARCHAR2（50），category_description VARCHAR2（1000），category_id NUMBER（2））; /

基于此类型创建对象表时，可以指定对象标识符应由系统生成或基于主键：

CREATE TABLE oe.categories_tab_sys OF oe.category_typ（category_id PRIMARY KEY）OBJECT ID SYSTEM GENERATED; CREATE TABLE oe.categories_tab_pkbased OF oe.category_typ（category_id PRIMARY KEY）OBJECT ID PRIMARY KEY;

例如，以下语句为categories_tab_sys对象表创建实例化视图日志，并指定记录对象标识符列：

使用OBJECT ID创建物料化视图oe.categories_tab_sys;

以下语句为categories_tab_pkbased对象表创建实例化视图日志，并指定将主键列与对象标识符列一起记录：

创建物化视图登录oe.categories_tab_pk基于OBJECT ID，PRIMARY KEY;

示例38-3创建实体化视图时记录列

创建具有PRIMARY KEY的oe.orders的物料化视图（customer_id，order_total）;

                     
                     

                        
示例38-4记录现有物化视图的列

                        
您可以通过发出以下语句将orders.customer_id和orders.order_total列添加到oe.orders表的oe.orders化视图日志中：
ALTER MATERIALIZED VIEW LOG ON oe.orders ADD（customer_id，order_total）;

                     
                     

                        
示例38-5记录列对象的属性

                        
如果使用用户定义的数据类型，则可以在实例化视图日志中记录列对象的属性。例如， oe.customers表具有cust_address.postal_code属性，可以通过发出以下语句将其记录在物化视图日志中：
ALTER MATERIALIZED VIEW LOG ON oe.customers ADD（cust_address.postal_code）;
