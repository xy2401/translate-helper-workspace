Oracle数据库中的更改

以下是Oracle Database SecureFiles和Large Objects Developer's Oracle数据库指南中的更改。

  • 新功能

  • 不推荐的功能

  • 不受支持的功能

新功能

19c版本的新功能

  • Oracle数据库允许使用LOB_SIGNATURE_ENABLE初始化参数为大对象（LOB）定位器配置基于签名的安全性

    管理LOB签名以获取更多信息。

18c版本的新功能

  • Oracle数据库允许LOB和LOB相关功能与Oracle Database In-Memory一起使用。

    有关更多信息，请参见Oracle Database In-Memory指南 。

  • Oracle数据库将对LOB的Exadata支持扩展到压缩LOB 。

    在行条款中启用或禁用存储

  • Oracle数据库改进了多个持久和临时LOB操作的性能。

不推荐的功能

Oracle®DatabaseSecureFiles和Large Objects 18c中已弃用的功能列表

Oracle数据库版本18c中不推荐使用Oracle多媒体，在将来的版本中可能不支持Oracle多媒体。Oracle建议您尽快停止使用已弃用的功能。

Oracle®DatabaseSecureFiles和Large Objects 12c，Release 2（12.2）中不推荐使用的功能列表

Oracle Database 12c第2版（12.2）中不推荐使用以下功能列表，并且可能在将来的发行版中不支持这些功能。

  • DBMS_LOB.LOADFROMFILE过程。

    请改用DBMS_LOB.LoadClobFromFile或DBMS_LOB.LoadBlobFromFile 。

  • LOB缓冲子系统API

    从Oracle 12 c第2版（12.2）开始，不推荐使用以下函数：

    • OCILobEnableBuffering（）

    • OCILobDisableBuffering（）

    • OCILobFlushBuffer（）

    代替使用这些LOB缓冲功能，使用预取LOB数据，长度和卡盘大小中描述的LOB预取功能。

  • DBMS_XSLPROCESSOR.CLOB2FILE过程。

    请改用DBMS_LOB.CLOB2FILE过程。

  • DBMS_LOB.LOADFROMFILE过程。

    请改用DBMS_LOB.LoadClobFromFile或DBMS_LOB.LoadBlobFromFile 。

  • DBMS_XSLPROCESSOR.CLOB2FILE过程。

    请改用DBMS_LOB.CLOB2FILE过程。
