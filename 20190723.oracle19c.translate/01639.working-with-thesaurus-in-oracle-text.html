13.1 Oracle Text同义词库功能概述

查询应用程序的用户查找有关给定主题的信息可能不知道在引用该主题的文档中使用了哪些单词。

Oracle Text使您能够创建区分大小写或不区分大小写的语句，这些语句定义单词和短语之间的同义词和层次关系。然后，您可以通过扩展查询来检索包含相关文本的文档，以包括同义词库中定义的类似或相关术语。

您可以创建同义词库并将其加载到系统中。

本节包含以下主题。

  • Oracle Text Thesaurus创建和维护

  • 使用区分大小写的同义词库

  • 使用不区分大小写的同义词库

  • 默认词库

  • 提供的词库

注意：

Oracle Text同义词库格式和功能符合ISO-2788和ANSI Z39.19（1993）标准。

13.1.1 Oracle Text Thesaurus创建和维护

如果您具有CTXAPP角色，则可以创建，修改，删除，导入和导出同义词和同义词条目。

本节包含以下主题。

  • CTX_THES包：要以编程方式维护和浏览同义词库，可以使用CTX_THES PL / SQL包。使用此包，您可以浏览术语和层次关系，添加和删除术语，添加和删除同义词关系，以及导入和导出同义词库表中的同义词。

  • 同义词运算符：要根据加载的同义词库扩展查询术语，可以使用CONTAINS子句中的同义词库运算符。例如，使用SYN运算符将诸如dog之类的术语扩展为其同义词：

    “顺式（狗）”

  • ctxload实用程序：您可以使用ctxload实用程序将纯文本文件中的同义词加载到同义词库表中，并将表中的同义词转储到输出（或转储）文件中。

    您可以打印同义词库转储文件，您可以将它们用作其他应用程序的输入，并且可以使用它们将同义词库加载到同义词库表中（当您希望使用现有同义词库作为新同义词库的基础时非常有用）。

    警告：

    为确保良好的安全性，Oracle建议您使用交互模式输入ctxload的密码，该模式会提示您输入用户密码。Oracle强烈建议您不要在命令行上输入密码。

    注意：

    您还可以使用PL / SQL包CTX_THES过程IMPORT_THESAURUS和EXPORT_THESAURUS以编程方式导入和导出同义词库表中的同义词。

    有关这些过程的详细信息，请参阅Oracle Text Reference 。

13.1.2使用区分大小写的同义词库

在区分大小写的同义词库中，术语（单词和短语）与您输入时完全相同。例如，如果以混合大小写形式输入术语（使用CTX_THES包或同义词库加载文件），则同义词库将以混合大小写形式存储该条目。

注意：

要充分利用区分大小写的同义词库导致的查询扩展，您的索引也必须区分大小写。

加载同义词库时，可以使用-thescase参数指定区分大小写的同义词库。

使用CTX_THES.CREATE_THESAURUS或CTX_THES.IMPORT_THESAURUS,创建同义词库时CTX_THES.IMPORT_THESAURUS,可以指定区分大小写的同义词库。

此外，当您在查询中指定区分大小写的同义词库时，同义词库查找将完全按照您在查询中输入的查询条件使用查询术语。因此，使用区分大小写的同义词的查询可以在查询扩展中提供更高级别的精度，这有助于在且仅当您具有区分大小写的索引时进行查找。

例如，区分大小写的同义词库是使用不同的条目创建的，用于表示土耳其 （国家）和火鸡 （鸟类）的不同含义。使用同义词库，对土耳其的查询扩展为仅包括与土耳其相关的条目。

13.1.3使用不区分大小写的同义词库

在不区分大小写的同义词库中，术语以全部大写形式存储，无论它们最初输入的情况如何。

默认情况下， ctxload程序在不区分大小写的模式下加载同义词库。

使用CTX_THES创建同义词库时。 CREATE_THESAURUS或CTX_THES.IMPORT_THESAURUS,默认情况下CTX_THES.IMPORT_THESAURUS,同义词库创建为不区分大小写。

此外，在查询中指定不区分大小写的同义词库时，查询术语将转换为全部大写以进行同义词库查找。因此，Oracle Text无法区分具有不同含义的术语。

例如，创建一个不区分大小写的同义词库，其中包含术语TURKEY （国家或鸟类）的两个不同含义的不同条目。使用词库，对于无论是土耳其或土耳其查询转换为土耳其词库查找，然后扩大到包括与两种意义相关联的所有条目。

13.1.4默认同义词库

如果未在查询中按名称指定同义词库，则默认情况下，同义词库运算符使用名为DEFAULT的同义词库。但是，Oracle Text不提供DEFAULT叙词表。

因此，如果要对同义词库运算符使用默认同义词库，则必须创建名为DEFAULT的同义词库。您可以通过Oracle Text支持的任何同义词库创建方法创建同义词库：

  • CTX_THES 。 CREATE_THESAURUS （PL / SQL）

  • CTX_THES 。 IMPORT_THESAURUS （PL / SQL）

  • ctxload实用程序

13.1.5提供的词库

虽然Oracle Text不提供默认同义词库，但Oracle Text确实提供了一个同义词库，以您使用ctxload,加载的文件的形式ctxload,您可以使用它创建一个通用的英语同义词库。

您可以使用同义词库加载文件为Oracle Text创建默认同义词库，也可以将其用作针对特定主题或主题范围定制的词库的基础。

  • 提供的同义词库结构和内容：提供的同义词库类似于传统的同义词库，例如Roget的同义词库，因为它提供了同义和语义相关术语的列表。

    它通过将术语组织成层次结构来提供额外的价值，该层次结构定义了较窄的术语与其更广泛术语之间的现实世界，实际关系。

    另外，在层次结构的不同区域中的术语之间建立交叉引用。

  • 提供的同义词库位置：同义词库加载文件的确切名称和位置取决于操作系统;但是，该文件通常名为dr0thsus （具有适当的文本文件扩展名），通常位于以下目录结构中：

13.2.1定义同义词

如果您有计算机科学术语的词库，那么您可以将术语XML的同义词定义为可扩展标记语言。 此同义词允许对这些术语中的任何一个进行查询以返回相同的文档。 

XML
SYN可扩展标记语言

您可以使用SYN运算符将XML扩展为其同义词：

'SYN（XML）'

扩展到：

'XML，可扩展标记语言'

13.2.2定义层次关系

如果您的文档集包含新闻文章，则可以使用同义词库来定义地理术语的层次结构。请考虑以下描述加利福尼亚州的地理层次结构：

加州NT北加州NT旧金山NT圣何塞NT中央谷NT弗雷斯诺NT南加州NT洛杉矶

您可以使用NT运算符扩展加利福尼亚州的查询：

'NT（加州）'

扩展到：

'加利福尼亚州，北加州，旧金山，圣何塞，中央山谷，弗雷斯诺，南加州，洛杉矶'

生成的命中列表显示与加利福尼亚州和城市相关的所有文档。

13.3在查询应用程序中使用同义词库

定义自定义同义词库时，可以更智能地处理查询。由于您的应用程序的用户可能不知道哪些单词代表某个主题，因此您可以为可能的查询术语定义同义词或更窄的术语。您可以使用同义词库运算符将查询扩展为您的同义词库术语。

有两种方法可以使用自定义同义词库来增强查询应用程序，以便您可以更智能地处理查询。每种方法都有其优点和缺点。

  • 加载自定义同义词库并使用同义词库运算符输入查询

  • 使用自定义同义词库扩充知识库（仅限英语），并使用ABOUT运算符扩展查询。

13.4加载自定义同义词库并发布基于同义词库的查询

您可以构建和加载自定义同义词库。

此方法的优点是您可以在索引后修改同义词库。

此方法的局限性在于您必须在查询中使用同义词扩展运算符。长查询会导致同义词库扩展中的额外开销，并减慢查询速度。

13.5.1优点

                  

                     
在编制索引之前使用现有知识库编译自定义同义词库可以使用ABOUT运算符实现更快，更简单的查询。文档服务还可以充分利用自定义信息来创建主题摘要和要点。
                     

                  

               
13.5.2限制

                  

                     
使用ABOUT运算符需要索引中的主题组件，这需要稍多的磁盘空间。在索引文档之前，还必须定义同义词库。如果更改同义词库，则必须重新编译同义词库并重新索引文档。

13.7使用ctxload加载同义词库的示例

               

                  
假设医学词库位于med.thes文件中， medthes使用ctxload将同义词库加载为medthes ，如下所示：
ctxload -thes -thescase y -name medthes -file med.thes -user ctxsys
当您输入ctxload命令行时，系统会提示您输入用户密码。为了获得最佳安全性，请不要在命令行输入密码。或者，您可以省略-user并让ctxload提示您输入用户名和密码。
                  

               

            
13.8使用CTX_THES.IMPORT_THESAURUS PL / SQL过程加载同义词库的示例

               

                  
这个例子创建一个区分大小写的词库命名mythesaurus并导入词库内容myclob到Oracle文本词库表：
宣告myclob clob;开始myclob：= to_clob（'peking SYN beijing BT capital country NT beijing tokyo'）; ctx_thes.import_thesaurus（'mythesaurus'，myclob，'Y'）;结束;
词库的格式被导入（ myclob在这个例子中）应该是相同的作为格式ctxload效用。如果要导入的同义词库的格式不正确，则IMPORT_THESAURUS会引发异常。
                  

               

            
13.9编译已加载的同义词库

               

                  
要将加载的medthes同义词库链接到知识库，请使用ctxkbtc ，如下所示：
ctxkbtc -user ctxsys -name medthes
当您输入ctxkbtc命令行时，系统会提示您输入用户密码。与ctxload,为了获得最佳安全性，请不要在命令行输入密码。
                  

                  

                     
警告：

                     
为确保完善的安全措施，Oracle建议您在交互模式下输入ctxload和ctxkbtc的密码。此模式会提示您输入用户密码。Oracle强烈建议您不要在命令行上输入密码。

13.10.1添加特定于语言的知识库

                  

                     

                        
您可以通过为任何单字节空白分隔语言（包括西班牙语）加载您自己的知识库，将主题功能扩展为英语或法语以外的语言。

                        
主题功能包括主题索引， ABOUT查询，主题突出显示，以及使用CTX_DOC.生成主题， CTX_DOC.和主题摘要CTX_DOC.

                        
您可以通过添加用户定义的知识库来扩展主题功能。例如，您可以从西班牙语词库创建西班牙语知识库。

                        
要加载特定于语言的知识库：

                     

                     
                     
    
                        
  1. 使用ctxload.加载自定义同义词库ctxload.
    2. 
                        
  2. 设置NLS_LANG以使语言部分成为目标语言。charset部分必须是单字节字符集。
    3. 
                        
  3. 使用ctxkbtc编译加载的同义词库，然后在出现提示时输入-user的密码。此语句根据加载的同义词库编译特定于语言的知识库。
    ctxkbtc -user ctxsys -name my_lang_thes

13.10.2添加知识库的限制

                  

                     
以下是添加知识库的限制：

                     
    
                        
  • 
                           
    Oracle仅提供英语和法语的知识库。您必须为任何其他语言提供自己的同义词库。
    
                        
    • 
                        
  • 
                           
    您只能为具有单字节字符集的语言添加知识库。您无法为只能以多字节字符集表示的语言创建知识库。如果数据库是多字节通用字符集（如UTF-8），则在编译同义词库时，仍必须将NLS_LANG参数设置为兼容的单字节字符集。
                           
    
                        
    • 
                        
  • 
                           
    添加知识库最适合以空格分隔的语言。
    
                        
    • 
                        
  • 
                           
    每种NLS_LANG语言只允许一个知识库。
                           
    
                        
    • 
                        
  • 
                           
    获取分层查询反馈信息（例如，更广泛的术语，更窄的术语和相关术语）在英语和法语之外的语言中不起作用。在其他语言中，知识库完全来自您的同义词。在这种情况下，Oracle建议您直接从您的同义词中获取分层信息。
