5 PDB创建概述

CDB支持多种创建PDB的技术。

创建的PDB自动包括完整数据字典，包括元数据和CDB根目录中系统提供的对象的内部链接。您必须从单个根定义每个PDB： CDB根或应用程序根 。

每个PDB都有一个全局唯一标识符（GUID）。PDB GUID主要用于为存储PDB文件的目录生成名称，包括Oracle Managed Files目录和非Oracle Managed Files目录。

本章包含以下主题：

  • 创建PDB的技术
    您可以使用各种技术创建PDB，所有这些技术都需要CREATE PLUGGABLE DATABASE语句。
  • PDB存储
    但是，如果选择创建PDB，则必须确定将存储数据的表空间和文件。
  • 服务名称转换
    PDB创建的一个重要方面是管理数据库服务的重命名。
  • 创建PDB的条款摘要
    使用CREATE PLUGGABLE DATABASE语句创建PDB时，可以根据不同因素使用各种子句。
  • PDB创建的一般先决条件
    在创建PDB之前，您必须满足某些先决条件。

5.1.1当前容器和PDB创建

您可以使用CREATE PLUGGABLE DATABASE语句来创建PDB，应用程序容器，应用程序种子和应用程序PDB。

创建PDB时，当前容器-CDB根或应用程序根目录确定PDB的关联。创建PDB和应用程序PDB的SQL语句是相同的。例如，在CDB根目录中运行CREATE PLUGGABLE DATABASE语句时，PDB属于CDB根目录。在应用程序根目录中运行CREATE PLUGGABLE DATABASE语句时，应用程序PDB属于应用程序根目录。

当CDB根目录是当前容器时，通过使用AS APPLICATION CONTAINER子句运行CREATE PLUGGABLE DATABASE语句来创建应用程序根目录。在将PDB克隆，重定位或插入应用程序容器时，PDB的应用程序名称和版本必须与应用程序名称和应用程序容器的版本匹配。

父主题： 创建PDB的技术

5.1.2从非CDB创建PDB的选项

您有多个选项可以将非CDB移动到PDB中。

您可以通过以下方式完成此任务：

  • 克隆非CDB

    这是使用非CDB创建PDB的最简单方法，但它需要将非CDB的文件复制到新位置。

    CDB和非CDB都必须运行Oracle Database 12c第1版（12.1.0.2）或更高版本。如果您当前的非CDB在Oracle Database 12c第1版（12.1.0.2）之前使用Oracle数据库版本，则必须将非CDB升级到更高版本才能使用此技术。

  • 使用DBMS_PDB包生成XML元数据文件

    XML元数据文件描述了非CDB的数据库文件，以便您可以将其插入CDB。

    此方法比通过克隆非CDB创建PDB需要更多步骤，但它使您能够使用非CDB创建PDB，而无需在某些情况下移动非CDB文件。

    要使用此技术，非CDB必须运行Oracle Database 12c或更高版本。如果您当前的非CDB使用Oracle Database 12c之前的版本，则必须升级到更高版本。

  • 从非CDB导出数据并使用Oracle Data Pump将其导入PDB

    导入时，请在用户名后指定PDB的连接标识符。例如，如果PDB的连接标识符是hrpdb ，则在运行Oracle Data Pump Import实用程序时输入以下内容：

    impdp user_name @hrpdb ...

    如果非CDB的Oracle数据库版本是Oracle Database 11g第2版（11.2.0.3）或更高版本，则可以使用完全可传输导出/导入来移动数据。将非CDB从Oracle Database 11g第2版（11.2.0.3）或更高版本的Oracle Database 11g数据库传输到Oracle Database 12c或更高版本时， VERSION Data Pump导出参数必须设置为12.0.0.0.0或更高版本。

    如果非CDB的Oracle数据库版本在Oracle Database 11g第2版（11.2.0.3）之前，那么您可以使用可传输的表空间来移动数据，或者可以执行完整的数据库导出/导入。

  • 使用GoldenGate将数据从非CDB复制到PDB

    当PDB赶上非CDB时，您将故障转移到PDB。

    请参阅Oracle GoldenGate文档。

5.2.1存储限制

CREATE PLUGGABLE DATABASE语句的可选STORAGE子句指定PDB的存储限制。

STORAGE子句指定以下限制：

  • 属于PDB的所有表空间可以使用的存储量

    使用MAXSIZE和size子句指定限制，或将MAXSIZE设置为UNLIMITED以指示无限制。

  • PDB中统一审计OS溢出（.bin格式）文件可以使用的存储量

    使用MAX_AUDIT_SIZE和size子句指定限制，或将MAX_AUDIT_SIZE设置为UNLIMITED以指示无限制。

  • PDB可以使用的自动诊断信息库（ADR）中的诊断（跟踪文件和事件转储）的数量

    使用MAX_DIAG_SIZE和size子句指定限制，或将MAX_DIAG_SIZE设置为UNLIMITED以指示无限制。

如果设置了STORAGE UNLIMITED ，或者没有STORAGE子句，则PDB没有存储限制。

5.2.2默认表空间

CREATE PLUGGABLE DATABASE语句的DEFAULT TABLESPACE子句指定新PDB的默认表空间。

Oracle数据库将为没有指定不同永久表空间的任何非SYSTEM用户分配默认表空间。

从PDB种子或应用程序种子创建PDB并指定DEFAULT TABLESPACE子句时，Oracle数据库会创建一个smallfile表空间并将其设置为PDB的默认表空间。使用除使用PDB种子或应用程序种子之外的方法创建PDB（例如克隆PDB或插入未插入的PDB）时，默认表空间必须是源PDB中已存在的表空间。

5.2.3用户表空间

CREATE PLUGGABLE DATABASE语句的USER_TABLESPACES子句指定新PDB中可用的表空间。

您可以使用此子句将多个模式的数据分成不同的PDB。例如，当您将非CDB移动到PDB，并且非CDB具有多个模式，每个模式支持不同的应用程序时，您可以使用此子句将属于每个模式的数据分离到单独的PDB中。此技术假定每个模式在非CDB中使用单独的表空间。

您可以使用此子句指定以下选项之一：

  • 列出要包含的一个或多个表空间。

  • 指定ALL （默认值）以包括所有表空间。

  • 指定ALL EXCEPT以包括所有表空间，但列出的表空间除外。

  • 指定NONE以排除所有表空间。

  • 如果用户表空间的创建模式必须与Oracle提供的表空间（例如SYSTEM和SYSAUX ）的创建模式不同，请在USER_TABLESPACES子句中指定以下内容之一：

    • COPY ：表空间的文件将复制到新位置。

    • MOVE ：表空间的文件被移动到一个新位置。

    • NOCOPY ：不复制或移动表空间的文件。

    • SNAPSHOT COPY ：使用存储快照克隆表空间。

    • NO DATA ：克隆表空间的数据模型定义，但不克隆表空间的数据。

当CDB的兼容级别为12.2.0或更高时，此子句排除的表空间在新PDB中脱机创建，并且它们没有与之关联的数据文件。当CDB的兼容级别低于12.2.0时，此子句排除的表空间在新PDB中处于脱机状态，属于这些表空间的所有数据文件都是未命名和脱机的。

本节不适用于SYSTEM ， SYSAUX或TEMP表空间。不要在这个子句的表空间列表中包含这些表空间。

以下是使用USER_TABLESPACES子句的示例。

示例5-4包含一个表空间的USER_TABLESPACES子句

假设从中创建PDB的非CDB或PDB包括以下表空间： tbs1 ， tbs2和tbs3 。此USER_TABLESPACES子句包含tbs2表空间，但不包括tbs1和tbs3表空间。 

USER_TABLESPACES =（ 'TBS2'）

示例5-5包含表空间列表的USER_TABLESPACES子句

假设从中创建PDB的非CDB或PDB包括以下表空间： tbs1 ， tbs2 ， tbs3 ， tbs4和tbs5 。此USER_TABLESPACES子句包含tbs1 ， tbs4和tbs5表空间，但不包括tbs2和tbs3表空间。 

USER_TABLESPACES =（ 'TBS1'， 'tbs4'， 'tbs5'）

示例5-6 USER_TABLESPACES子句包括除列出的所有表空间之外的所有表空间

假设从中创建PDB的非CDB或PDB包括以下表空间： tbs1 ， tbs2 ， tbs3 ， tbs4和tbs5 。此USER_TABLESPACES子句包含tbs2和tbs3表空间，但不包括tbs1 ， tbs4和tbs5表空间。 

USER_TABLESPACES = ALL除外（'tbs1'，'tbs4'，'tbs5'）

示例5-7不同创建模式下的USER_TABLESPACES

此示例显示了一个完整的CREATE PLUGGABLE DATABASE语句，该语句插入非CDB并且仅包含来自非CDB的tbs3用户表空间。该示例将Oracle提供的表空间（例如SYSTEM和SYSAUX ）的文件复制到新位置，但移动tbs3用户表空间的文件。

5.2.4 PDB文件位置

在CREATE PLUGGABLE DATABASE语句中，您可以指定新PDB使用的文件的位置。

术语“文件名”表示文件的名称和位置。CREATE PLUGGABLE DATABASE语句具有以下子句，指示要创建的新PDB的文件名：

  • FILE_NAME_CONVERT子句指定创建PDB后PDB文件的名称。

    当文件尚未到达其最终目的地时，请使用此子句，并且您希望在创建PDB期间复制或移动它们。您可以在任何CREATE PLUGGABLE DATABASE语句中使用此子句。

  • CREATE_FILE_DEST子句指定PDB文件的缺省Oracle Managed Files文件系统目录或Oracle ASM磁盘组。

    使用此子句为新PDB启用Oracle Managed Files，而不依赖于CDB根目录中指定的任何Oracle Managed Files缺省位置。您可以在任何CREATE PLUGGABLE DATABASE语句中使用此子句。

必要时，可以在同一个CREATE PLUGGABLE DATABASE语句中使用这两个子句。此外，以下初始化参数可以控制新PDB文件的位置：

  • 根目录中设置的DB_CREATE_FILE_DEST初始化参数

    此初始化参数指定CDB的Oracle Managed Files的默认位置。在PDB中设置此参数时，它指定PDB的Oracle Managed Files的默认位置。

  • PDB_FILE_NAME_CONVERT初始化参数

    处理CREATE PLUGGABLE DATABASE语句时，此初始化参数将现有文件的名称映射到新文件名。

5.2.4.1 FILE_NAME_CONVERT子句

如果PDB不使用Oracle Managed Files，则CREATE PLUGGABLE DATABASE语句的FILE_NAME_CONVERT子句指定如何使用现有文件的名称生成文件名（例如数据文件）。

您可以使用此子句指定以下选项之一：

  • 一个或多个文件名模式和替换文件名模式，格式如下：

    ' string1 '，' string2 '，' string3 '，' string4 '，...

    该字符串2文件名模式替换字符串1文件名模式，以及串，4文件名模式替换STRING3文件名模式。您可以根据需要使用多对文件名模式和替换文件名模式字符串。

    如果指定奇数个字符串（最后一个字符串没有相应的替换字符串），则返回错误。不要指定多个匹配单个文件名或目录的模式/替换字符串。

  • NONE当没有文件应该被复制或PDB创建过程中移动。省略FILE_NAME_CONVERT子句与指定NONE相同。

您可以在任何CREATE PLUGGABLE DATABASE语句中使用FILE_NAME_CONVERT子句。

如果未在CREATE PLUGGABLE DATABASE语句中指定FILE_NAME_CONVERT子句，则Oracle Managed Files或PDB_FILE_NAME_CONVERT初始化参数指定如何生成文件的名称。如果同时使用Oracle Managed Files和PDB_FILE_NAME_CONVERT初始化参数，则Oracle Managed Files优先。指定时， FILE_NAME_CONVERT子句优先。

FILE_NAME_CONVERT子句中指定的文件名模式与Oracle Managed Files管理的文件或目录不匹配。

例5-8 FILE_NAME_CONVERT子句

此FILE_NAME_CONVERT子句使用/oracle/dbs目录中的文件名为/oracle/pdb5目录中的新PDB生成文件名。 

FILE_NAME_CONVERT =（'/ oracle / dbs /'，'/ oracle / pdb5 /'）

5.2.4.2 CREATE_FILE_DEST子句

CREATE PLUGGABLE DATABASE语句的CREATE_FILE_DEST子句为PDB启用Oracle Managed Files，并为PDB文件指定缺省文件系统目录或Oracle ASM磁盘组。

PDB数据文件和临时文件仅限于指定的目录及其子目录。如果将文件系统目录指定为此子句中的缺省位置，则该目录必须存在。此外，运行CREATE PLUGGABLE DATABASE语句的用户必须具有在指定目录中创建文件的相应权限。或者，您可以指定CDB根目录中存在的目录对象的名称（ CDB$ROOT ）。目录对象指向CREATE_FILE_DEST使用的文件系统目录。

如果在CDB根目录中设置了CDB的默认Oracle Managed Files位置，则CREATE_FILE_DEST设置将覆盖CDB根目录的设置，并且指定的CREATE_FILE_DEST设置用于PDB。

如果指定了CREATE_FILE_DEST=NONE ，则禁用PDB的Oracle Managed Files。

当CREATE_FILE_DEST子句设置为NONE以外的值时， DB_CREATE_FILE_DEST初始化参数将在PDB中隐式设置， SCOPE=SPFILE 。

如果CDB根目录使用Oracle Managed Files，并且未指定此子句，则PDB将从CDB根目录继承Oracle Managed Files默认位置。

注意：

从Oracle Database 12c第1版（12.1.0.2）开始提供此功能。

例5-9 CREATE_FILE_DEST子句

此CREATE_FILE_DEST子句指定/ oracle / pdb2 /作为新PDB的缺省Oracle Managed Files文件系统目录。

CREATE_FILE_DEST ='/ oracle / pdb2 /'

5.2.4.3 PDB文件位置的限制

CREATE PLUGGABLE DATABASE语句的PATH_PREFIX子句确保与PDB关联的所有目录对象路径都限制在指定目录或其子目录中。

此子句还确保与PDB关联的以下文件仅限于指定的目录：PDB的Oracle XML存储库，使用CREATE PFILE语句创建的文件以及Oracle钱包的导出目录。如果要确保PDB的文件驻留在特定目录及其子目录中，请使用此子句。

您可以使用此子句指定以下选项之一：

  • 绝对路径，用作与PDB关联的所有文件路径的前缀。

  • CDB根目录中存在的目录对象的名称（ CDB$ROOT ）。目录对象指向用于PATH_PREFIX的绝对路径。

  • NONE表示文件路径没有限制。省略PATH_PREFIX子句与指定NONE相同。

创建PDB后，无法修改其PATH_PREFIX设置。

您可以在任何CREATE PLUGGABLE DATABASE语句中使用PATH_PREFIX子句。

例5-10 PATH_PREFIX子句

此PATH_PREFIX子句确保与PDB关联的所有文件路径都限制在/disk1/oracle/dbs/salespdb/目录中。 

PATH_PREFIX ='/ disk1 / oracle / dbs / salespdb /'

请务必指定路径名，以便在附加文件名时正确形成路径名。例如，在UNIX系统上，请确保使用正斜杠（/）结束路径名。

注意：

  • 在为PDB指定PATH_PREFIX子句之后，现有目录对象可能无法按预期工作，因为PATH_PREFIX字符串始终作为前缀添加到PDB中的所有本地目录对象。

  • PATH_PREFIX子句不会影响Oracle Managed Files创建的文件。

  • PATH_PREFIX子句仅适用于用户创建的目录对象。它不适用于Oracle提供的目录对象。

  • PATH_PREFIX子句不适用于数据文件或临时文件。如果您使用的是Oracle Managed Files，则使用CREATE_FILE_DEST子句限制数据文件和临时文件的位置。

5.3服务名称转换

PDB创建的一个重要方面是管理数据库服务的重命名。

当新PDB的服务名称与CDB中的现有服务名称冲突时，可能会导致插件违规。CREATE PLUGGABLE DATABASE语句的SERVICE_NAME_CONVERT子句根据源PDB的服务名称重命名新PDB的用户定义服务。使用此子句，您可以重命名服务并避免插件违规。

您可以使用此子句指定以下选项之一：

  • 一个或多个服务名称和替换服务名称，格式如下：

    ' string1 '，' string2 '，' string3 '，' string4 '，...

    string2服务名称替换string1服务名称， string4服务名称替换string3服务名称。您可以根据需要使用多对服务名称和替换服务名称。

    如果指定奇数个字符串（最后一个字符串没有相应的替换字符串），则返回错误。

  • NONE时，没有服务名称需要进行重命名。省略SERVICE_NAME_CONVERT子句与指定NONE相同。

您可以在任何CREATE PLUGGABLE DATABASE语句中使用SERVICE_NAME_CONVERT子句， CREATE PLUGGABLE DATABASE PDB种子创建PDB的CREATE PLUGGABLE DATABASE语句除外。PDB种子不能具有用户定义的服务。但是，您可以将此语句用于CREATE PLUGGABLE DATABASE语句，该语句从应用程序容器中的应用程序种子创建应用程序PDB。

注意：

本节不适用于PDB的默认服务。默认服务与PDB具有相同的名称。

示例5-11 SERVICE_NAME_CONVERT子句

此SERVICE_NAME_CONVERT子句使用salesrep服务重命名为salesperson 。 

SERVICE_NAME_CONVERT =（'salesrep'，'salesperson'）
