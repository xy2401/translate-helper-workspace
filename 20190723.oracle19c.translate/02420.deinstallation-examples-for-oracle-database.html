Oracle数据库的卸载示例

使用这些示例可帮助您了解如何运行deinstall命令。

从$ORACLE_HOME/deinstall目录运行deinstall 。卸载开始时不会提示您输入Oracle主路径。 

$ ./deinstall

您可以通过使用-checkonly标志运行deinstall来生成卸载响应文件。或者，您可以使用位于$ORACLE_HOME/deinstall/response/deinstall.rsp.tmpl的响应文件模板。如果您有响应文件，则使用可选标志-paramfile来提供响应文件的路径。

在以下示例中， deinstall命令位于路径/u01/app/oracle/product/19.0.0/dbhome_1/deinstall 。它在软件所有者位置/home/usr/oracle使用名为my_db_paramfile.tmpl的响应文件：

$ cd /u01/app/oracle/product/19.0.0/dbhome_1/deinstall $ ./deinstall-paramfile /home/usr/oracle/my_db_paramfile.tmpl

要删除Oracle Grid Infrastructure主目录，请使用Oracle Grid Infrastructure主目录中的deinstall命令。

在此示例中，Oracle Grid Infrastructure主目录为/u01/app/19.0.0/grid

$ cd /u01/app/19.0.0/grid / deinstall $ ./deinstall-paramfile /home/usr/oracle/my_grid_paramfile.tmpl
