14存储的PL / SQL单元的SQL语句

本章介绍如何使用创建，更改和删除存储的PL / SQL单元的SQL语句。

创建[或REPLACE]语句

这些SQL语句中的每一个都在模式级别创建一个PL / SQL单元并将其存储在数据库中：

  • 创建功能声明

  • CREATE LIBRARY语句

  • CREATE PACKAGE声明

  • CREATE PACKAGE BODY声明

  • CREATE PROCEDURE声明

  • CREATE TRIGGER声明

  • CREATE TYPE语句

  • CREATE TYPE BODY声明

每个CREATE语句都有一个可选的OR REPLACE子句。指定OR REPLACE以重新创建现有PL / SQL单元 - 即更改其声明或定义而不删除它，重新创建它，以及重新授予先前在其上授予的对象权限。如果重新定义PL / SQL单元，数据库将重新编译它。

警告：

在替换现有PL / SQL单元之前， CREATE OR REPLACE语句不会发出警告。

这些CREATE语句都不会出现在PL / SQL块中。

更改语句

要重新编译现有的PL / SQL单元而不重新创建它（不更改其声明或定义），请使用以下SQL语句之一：

  • 更改功能声明

  • ALTER LIBRARY声明

  • ALTER PACKAGE声明

  • ALTER PROCEDURE声明

  • ALTER TRIGGER声明

  • ALTER TYPE语句

使用ALTER语句的原因是：

  • 显式重新编译已变为无效的存储单元，从而消除了对隐式运行时重新编译的需要，并防止了相关的运行时编译错误和性能开销。

  • 使用不同的编译参数重新编译存储的单元。

  • 启用或禁用触发器。

  • 指定其架构对象类型尚未在其架构中可编辑的存储单元的EDITIONABLE或NONEDITIONABLE属性。

ALTER TYPE语句还有其他用途。

DROP语句

要从数据库中删除现有PL / SQL单元，请使用以下SQL语句之一：

  • DROP FUNCTION声明

  • DROP LIBRARY声明

  • DROP PACKAGE声明

  • DROP PROCEDURE声明

  • DROP TRIGGER声明

  • DROP TYPE语句

  • DROP TYPE BODY声明
