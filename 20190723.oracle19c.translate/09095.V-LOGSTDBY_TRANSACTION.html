8.63 V $ LOGSTDBY_TRANSACTION

V$LOGSTDBY_TRANSACTION显示SQL Apply正在处理的所有事务。

此视图中显示的事务标识符是从重做流中挖掘的事务标识符，对应于在主数据库中分配的事务标识符，并且不对应于在逻辑备用数据库中处于活动状态的事务。有关逻辑备用数据库中活动的事务（包括作为SQL Apply的一部分创建的事务）的信息，请在逻辑备用数据库中查询V$TRANSACTION视图。

柱 数据类型 描述

PRIMARY_XIDUSN

数

撤消事务的段号

PRIMARY_XIDSLT

数

交易的槽号

PRIMARY_XIDSQN

数

交易的序号

PRIMARY_XID

RAW（8）

交易ID

PRIMARY_START_SCN

数

启动系统更改编号（SCN）库

PRIMARY_START_TIME

日期

开始时间

PRIMARY_PARENT_XIDUSN

数

撤消父事务的段号

PRIMARY_PARENT_XIDSLT

数

父事务的插槽号

PRIMARY_PARENT_XIDSQN

数

父事务的序号

PRIMARY_PARENT_XID

RAW（8）

父事务的事务ID（PDML）

类型

VARCHAR2（32）

类型：

  • PL/SQL - 事务是作为受支持的PL / SQL过程的一部分完成的

  • Direct Path Load - 事务是直接路径加载

  • CTAS - Transaction包含至少一个CREATE TABLE ...AS SELECT操作

  • DDL - Transaction包含一个或多个DDL操作

  • PDML Child - Transaction是一个子事务

  • DML - Transaction仅包含DML操作

MINING_STATUS

VARCHAR2（32）

采矿状况：

  • ACTIVE - LogMiner仍在挖掘事务。此交易的至少一部分已准备好应用或已应用。

  • COMPLETE -交易完成并准备应用。LogMiner已完成挖掘。

SRC_CON_ID

数

包含生成此事务更改的源数据库的PDB ID（ DBA_PDBS视图中的PDB_ID列）。

APPLY_STATUS

VARCHAR2（6）

  • ACTIVE - 已将事务分配给应用服务器。它处于以下状态之一：

    - 交易正在积极应用

    - 交易由应用服务器持有，等待某些事件发生

    - 交易由申请服务器持有，等待此交易的后续部分

  • NONE - 尚未将事务分配给应用服务器

SID

数

应用服务器会话的会话ID;如果APPLY_STATUS为NONE APPLY_STATUS空

SERIAL＃

数

应用服务器会话的序列号;如果APPLY_STATUS为NONE APPLY_STATUS空

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
