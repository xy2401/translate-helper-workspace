格式模型

格式模型是一个字符文字，用于描述存储在字符串中的日期时间或数字数据的格式。格式模型不会更改数据库中值的内部表示形式。将字符串转换为日期或数字时，格式模型将确定Oracle数据库如何解释字符串。在SQL语句中，您可以使用格式模型作为TO_CHAR和TO_DATE函数的参数来指定：

  • Oracle用于从数据库返回值的格式

  • 您为Oracle指定要存储在数据库中的值的格式

例如：

  • 字符串HH24:MI:SS 17:45:29 '的日期时间格式模型是' HH24:MI:SS '。

  • 字符串11-Nov-1999 '的日期时间格式模型是' DD-Mon-YYYY '。

  • 字符串' $2,304.25 '的数字格式模型是' $9,999.99 '。

有关数字和日期时间格式模型元素的列表，请参阅表2-15和表2-17 。

某些格式的值由初始化参数的值确定。对于此类格式，您可以使用初始化参数NLS_TERRITORY隐式指定这些格式元素返回的字符。您可以使用ALTER SESSION语句ALTER SESSION的默认日期格式。

数字格式模型

您可以在以下函数中使用数字格式模型：

  • 在TO_CHAR函数中将NUMBER ， BINARY_FLOAT或BINARY_DOUBLE数据类型的值TO_CHAR为VARCHAR2数据类型

  • 在TO_NUMBER函数中，将CHAR或VARCHAR2数据类型的值转换为NUMBER数据类型

  • 在TO_BINARY_FLOAT和TO_BINARY_DOUBLE函数TO_BINARY_DOUBLE CHAR和VARCHAR2表达式转换为BINARY_FLOAT或BINARY_DOUBLE值

所有数字格式模型都会将数字四舍五入到指定的有效位数。如果值在小数点左边的位数比格式中指定的位数更多，那么井号（＃）将替换该值。当您使用具有限制性数字格式字符串的TO_CHAR时，通常会发生此事件，从而导致舍入操作。

  • 如果正NUMBER值非常大且无法以指定格式表示，则无穷大符号（〜）将替换该值。同样，如果负NUMBER值非常小并且无法用指定的格式表示，则负无穷大符号将替换值（ - 〜）。

  • 如果BINARY_FLOAT或BINARY_DOUBLE值转换为CHAR或NCHAR ，并且输入为无穷大或NaN （不是数字），则Oracle始终返回井号以替换该值。但是，如果省略格式模型，则Oracle会返回Inf或Nan作为字符串。

日期时间格式模型

您可以在以下函数中使用datetime格式模型：

  • 在TO_* datetime函数中，将格式不是默认格式的字符值转换为日期时间值。（ TO_ * datetime函数是TO_DATE ， TO_TIMESTAMP和TO_TIMESTAMP_TZ 。）

  • 在TO_CHAR函数中，将日期时间值TO_CHAR默认格式的字符值（例如，从应用程序打印日期）

日期时间格式模型的总长度不能超过22个字符。

默认日期时间格式使用NLS会话参数NLS_DATE_FORMAT ， NLS_TIMESTAMP_FORMAT和NLS_TIMESTAMP_TZ_FORMAT显式指定，或者使用NLS会话参数NLS_TERRITORY隐式NLS_TERRITORY 。您可以使用ALTER SESSION语句ALTER SESSION的默认日期时间格式。

日期时间格式元素

日期时间格式模型由表2-17中列出的一个或多个日期时间格式元素组成。

  • 对于输入格式模型，格式项不能出现两次，并且无法组合表示类似信息的格式项。例如，您不能在相同的格式字符串中使用“SYYYY”和“BC”。

  • 第二列指示是否可以在TO_* datetime函数中使用format元素。所有格式元素都可以在TO_CHAR函数中使用。

  • 以下日期时间格式元素可用于时间戳和间隔格式模型，但不能用于原始DATE格式模型： FF ， TZD ， TZH, TZM和TZR 。

  • 许多日期时间格式元素用空格或前导零填充到特定长度。有关更多信息，请参阅格式模型修改器FM 。

注意：

由于以下原因，Oracle建议您使用4位年份元素（ YYYY ）而不是较短的年份元素：

  • 4位数年份元素消除了歧义。

  • 较短的年份元素可能会影响查询优化，因为查询编译时不知道年份，并且只能在运行时确定。

日期格式元素中的大写字母

拼写单词，缩写或罗马数字中的大写在相应的格式元素中大写。例如，日期格式模型'DAY'产生大写单词，如'MONDAY'; 'Day'产生'Monday';并且'day'产生'星期一'。

日期时间格式模型中的标点符号和字符文字

您可以在日期格式模型中包含这些字符：

  • 标点符号，如连字符，斜杠，逗号，句点和冒号

  • 字符文字，用双引号括起来

这些字符出现在返回值中与格式模型中显示的位置相同的位置。

Oracle数据库可以灵活地将字符串转换为日期。例如，当使用TO_DATE函数时，包含标点符号的格式模型匹配缺少部分或全部这些字符的输入字符串，前提是输入字符串中的每个数字元素都包含允许的最大位数 - 例如，两位数字' 05'代表'MM'或'2007'代表'YYYY'。以下语句不会返回错误：

SELECT TO_CHAR（TO_DATE（'0207'，'MM / YY'），'MM / YY'）FROM DUAL; TO_CH ----- 02/07

但是，以下格式字符串确实会返回错误，因为FX（格式完全）格式修饰符需要表达式和格式字符串的完全匹配：

SELECT TO_CHAR（TO_DATE（'0207'，'fxmm / yy'），'mm / yy'）FROM DUAL; SELECT TO_CHAR（TO_DATE（'0207'，'fxmm / yy'），'mm / yy'）FROM DUAL; *第1行的错误：ORA-01861：文字与格式字符串不匹配

允许任何非字母数字字符匹配格式模型中的标点字符。例如，以下语句不会返回错误：

SELECT TO_CHAR（TO_DATE（'02＃07'，'MM / YY'），'MM / YY'）FROM DUAL; TO_CH ----- 02/07

Oracle根据ISO标准计算日期时间格式元素IYYY，IYY，IY，I和IW返回的值。有关这些值与日期时间格式元素YYYY，YYY，YY，Y和WW返回的值之间的差异的信息，请参阅“ Oracle数据库全球化支持指南”中有关全球化支持的讨论。

RR日期时间格式元素

RR datetime格式元素类似于YY日期时间格式元素，但它为在其他世纪存储日期值提供了额外的灵活性。RR日期时间格式元素允许您通过仅指定年份的最后两位数来存储21世纪的20世纪日期。

如果将TO_DATE函数与YY日期时间格式元素一起使用，则返回的年份始终与当前年份的前2位数字相同。如果使用RR datetime格式元素，则返回值的世纪根据指定的两位数年份和当前年份的最后两位数而变化。

那是：

  • 如果指定的两位数年份是00到49，那么

    • 如果当前年份的最后两位数字是00到49，那么返回的年份与当前年份的前两位数字相同。

    • 如果当前年份的最后两位数字是50到99，那么返回年份的前2位数字比当前年份的前2位数字大1。

  • 如果指定的两位数年份是50到99，那么

    • 如果当前年份的最后两位数字是00到49，那么返回年份的前2位数字比当前年份的前2位数字少1。

    • 如果当前年份的最后两位数字是50到99，那么返回的年份与当前年份的前两位数字相同。

SELECT TO_CHAR（TO_DATE（'27 -OCT-98'，'DD-MON-RR'），'YYYY'）“年”来自DUAL;年---- 1998 SELECT TO_CHAR（TO_DATE（'27 -OCT-17'，'DD-MON-RR'），'YYYY'）“年”来自DUAL;年---- 2017年

SELECT TO_CHAR（TO_DATE（'27 -OCT-98'，'DD-MON-RR'），'YYYY'）“年”来自DUAL;年---- 1998 SELECT TO_CHAR（TO_DATE（'27 -OCT-17'，'DD-MON-RR'），'YYYY'）“年”来自DUAL;年---- 2017年

请注意，无论是在2000年之前还是之后发布，查询都会返回相同的值。RR datetime format元素允许您编写SQL语句，这些语句将从前两位数不同的年份返回相同的值。

格式模型修饰符

在TO_CHAR函数中的格式模型中使用的FM和FX修饰符控制空白填充和精确格式检查。

修改器可以多次出现在格式模型中。在这种情况下，每次后续发生都会切换修改器的效果。它的效果在第一次出现后对模型的部分启用，然后在第二次出现之后的部分被禁用，然后为第三次之后的部分重新启用，依此类推。

调频

填充模式。Oracle使用尾随空白字符和前导零来将格式元素填充到恒定宽度。宽度等于相关格式模型的最大元素的显示宽度：

  • 数字元素用前导零填充到元素允许的最大值的宽度。例如， YYYY元素填充为四位数（长度为“9999”）， HH24为两位数（长度为“23”）， DDD填充为三位数（长度为“366”）。

  • 字符元素MONTH ， MON ， DAY和DY用尾随空格填充到有效名称中最长的完整月份名称，最长的缩写月份名称，最长的完整日期名称或最长的缩写日期名称的宽度由NLS_DATE_LANGUAGE和NLS_CALENDAR参数的值确定。例如，当NLS_DATE_LANGUAGE为AMERICAN且NLS_CALENDAR为GREGORIAN （默认值）时， MONTH的最大元素为SEPTEMBER ，因此MONTH格式元素的所有值都填充为九个显示字符。NLS_DATE_LANGUAGE和NLS_CALENDAR参数的值在TO_CHAR和TO_* datetime函数的第三个参数中指定，或者从当前会话的NLS环境中检索它们。

  • 字符元素RM用尾随空格填充长度为4，即'viii'的长度。

  • 其它字符元件和拼写出数字（ SP ， SPTH和THSP后缀）不填充。

FM修饰符在TO_CHAR函数的返回值中抑制上述填充。

FX

格式准确。此修饰符指定TO_DATE函数的字符参数和日期时间格式模型的精确匹配：

  • 字符参数中的标点符号和引用文本必须与格式模型的相应部分完全匹配（例外情况除外）。

  • 角色参数不能有额外的空白。如果没有FX ，Oracle会忽略额外的空白。

  • 字符参数中的数字数据必须与格式模型中的相应元素具有相同的位数。没有FX ，字符参数中的数字可以省略前导零。

    启用FX ，您也可以使用FM修改器禁用此前导零的检查。

如果字符参数的任何部分违反任何这些条件，则Oracle会返回错误消息。

以下语句使用日期格式模型返回字符表达式：

SELECT TO_CHAR（SYSDATE，'fmDDTH'）|| 'of'|| TO_CHAR（SYSDATE，'fmMonth'）|| '，'|| TO_CHAR（SYSDATE，'YYYY'）“Ides”来自DUAL; Ides ------------------ 2008年4月3日

SELECT TO_CHAR（SYSDATE，'DDTH'）|| 'of'|| TO_CHAR（SYSDATE，'月'）|| '，'|| TO_CHAR（SYSDATE，'YYYY'）“Ides”来自DUAL; Ides ----------------------- 2008年4月的03RD

SELECT TO_CHAR（SYSDATE，'fmDay'）||来自DUAL的特别''菜单';菜单-----------------周二的特别节目

返回值的格式：示例

您可以使用格式模型指定Oracle用于将数据库中的值返回给您的格式。

以下语句选择Department 80中员工的工资，并使用TO_CHAR函数将这些工资转换为字符值，格式为数字格式模型' $99,990.99 '：

SELECT last_name employee，TO_CHAR（salary，'$ 99,990.99'）FROM employees WHERE department_id = 80;

由于这种格式模型，Oracle会返回带有前导美元符号的工资，每三位数字的逗号和两位小数。

以下语句选择雇用Department 20中每个员工的日期，并使用TO_CHAR函数将这些日期转换为字符串，格式为日期格式模型' fmMonth DD, YYYY '：

SELECT last_name employee，TO_CHAR（hire_date，'fmMonth DD，YYYY'）雇用员工WHERE department_id = 20;

使用此格式模型，Oracle返回的租用日期没有空白填充（由fm指定），当天的两位数和年份中包含的世纪。

提供正确的格式模型：示例

插入或更新列值时，指定的值的数据类型必须与列的列数据类型相对应。您可以使用格式模型指定要从一种数据类型转换为列所需的另一种数据类型的值的格式。

例如，插入DATE列的值必须是DATE数据类型的值或默认日期格式的字符串（Oracle隐式地将默认日期格式的字符串转换为DATE数据类型）。如果值是另一种格式，则必须使用TO_DATE函数将值转换为DATE数据类型。您还必须使用格式模型来指定字符串的格式。

以下语句使用TO_DATE函数更新Hunold's雇用日期，格式为掩码'YYYY MM DD'，将字符串'2008 05 20'转换为DATE值：

UPDATE员工SET hire_date = TO_DATE（'2008 05 20'，'YYYY MM DD'）WHERE last_name ='Hunold';

字符串到日期转换规则

将字符串值转换为日期值时，以下附加格式规则适用（除非您在格式模型中使用FX或FXFM修饰符来控制精确格式检查）：

  • 如果指定了数字格式元素的所有数字（包括前导零），则可以从日期字符串中省略格式字符串中包含的标点符号。例如，为两位数格式元素（如MM，DD和YY）指定02而不是2。

  • 您可以省略日期字符串中格式字符串末尾的时间字段。

  • 您可以在日期字符串中使用任何非字母数字字符来匹配格式字符串中的标点符号。

  • 如果日期时间格式元素与日期字符串中的相应字符之间的匹配失败，则Oracle会尝试其他格式元素，如表2-20所示。
