升级Oracle ASM后的建议任务

升级Oracle ASM后，Oracle建议您执行重置Oracle ASM密码和配置磁盘组等任务。

  • 在ASM磁盘组中创建共享密码文件
    如果您提前COMPATIBLE.ASM磁盘组属性，则创建共享密码文件。
  • 重置Oracle ASM密码以强制区分大小写

    您可以跨软件版本推进Oracle数据库和Oracle ASM磁盘组兼容性设置。

    Oracle ASM管理员可以将某些磁盘指定为读取I / O操作的首选读取磁盘。

重置Oracle ASM密码以强制区分大小写

要利用强制区分大小写的密码，必须在数据库升级过程中重置现有用户的密码。

在早于Oracle Database 11g第1版（11.1）的版本中，密码不区分大小写。您可以强制实施密码的区分大小写。例如，如果密码为hpp5620QR或hPp5620Qr ，则密码hPP5620qr失败。

对于新的Oracle ASM实例，没有其他任务或管理要求。对于升级的Oracle ASM实例，必须使用ALTER USER语句重置每个用户密码。

注意：

如果默认的Oracle数据库安全设置已就绪，则密码必须至少为八个字符，并且不允许使用welcome和oracle等密码。有关更多信息，请参见Oracle数据库安全指南

推进Oracle ASM和Oracle数据库磁盘组兼容性

您可以跨软件版本推进Oracle数据库和Oracle ASM磁盘组兼容性设置。

警告：

如果您提前COMPATIBLE.RDBMS属性，则无法恢复到先前的设置。在推进COMPATIBLE.RDBMS属性之前，请确保在提升属性值之前，将所有使用磁盘组的数据库的COMPATIBLE初始化参数的值设置为COMPATIBLE.RDBMS的新设置。

提高兼容性使新功能仅在新版本中可用。但是，这样做会使磁盘组与较旧版本的软件不兼容。提高磁盘兼容性是一项不可逆转的操作。

使用compatible.rdbms和compatible.asm属性分别指定数据库实例和Oracle ASM实例访问磁盘组所需的最低软件版本。例如，以下ALTER DISKGROUP语句提升了磁盘组asmdg2的Oracle ASM兼容性：

ALTER DISKGROUP asmdg2 SET ATTRIBUTE'compatibility.asm'= '12 .2'

在这种情况下，磁盘组只能由版本12.2或更高版本的Oracle ASM软件管理，而版本11.2或更高版本的任何数据库客户端都可以使用该磁盘组。

设置Oracle ASM首选读取失败组

Oracle ASM管理员可以将某些磁盘指定为读取I / O操作的首选读取磁盘。

当ASM管理员定义Oracle ASM首选读取故障组时，Oracle ASM可以从最近的首选读取磁盘中读取范围，而不是始终读取主副本。

也可以看看：

  • “Oracle Clusterware管理和部署指南”，以获取有关在扩展集群中指定故障组设置的信息

  • “Oracle自动存储管理管理员指南” ，获取有关Oracle ASM首选读取失败组的完整信息，并指定新的ASM_PREFERRED_READ_FAILURE_GROUPS初始化参数以列出包含群集中每个节点的首选读取磁盘的故障组名称
