原因：不允许将具有PL / SQL类型属性的记录集合作为TABLE运算符的参数。

操作：使用其属性属于SQL可以理解的类型的记录集合。

ORA-39966：此表单仅在WITH子句上下文中允许

原因：仅为SYS_PLSQL_CPU，SYS_PLSQL_COUNT或SYS_PLSQL_IO成本核算运算符指定了入口点编号。仅当这些运算符应用于WITH子句中出现的函数时，才允许这样做。

操作：指定所需的功能，包或类型名称。还可以指定可选的模式名称。

ORA-39967：在移动或重新键入数据文件时无法使用此PL / SQL过程。

原因：某些数据文件正在主数据库或备用数据库上移动或重新键入。

操作：检查数据文件的状态并再次尝试此命令。

ORA-39968：参数PLSCOPE_SETTINGS的值无效。值必须来自'IDENTIFIERS：[ALL | NONE | PLSQL | SQL | PUBLIC]，语句：[ALL | NONE]'。

ORA-39969：绑定变量应该是受支持的PL / SQL类型

原因：在DBMS_SQL.BIND_VARIABLE_PKG调用中使用了错误类型的变量。在DBMS_SQL.BIND_VARIABLE_PKG调用中，变量的类型应在包规范中声明，并且应该是记录或受支持的集合类型之一。

操作：使用绑定变量，其类型是此操作支持的类型之一，其类型在包规范中声明。

ORA-39970：光标执行后不允许绑定调用

原因：执行游标后使用绑定调用。绑定只能在解析游标之后但在执行之前完成。

操作：在游标执行之前移动绑定调用。

ORA-39971：当无法访问PATH_PREFIX信息时，无法对此可插拔数据库使用此PL / SQL过程

原因：出于安全考虑，某些可插拔数据库已指定PATH_PREFIX，但PL / SQL过程无法检查PATH_PREFIX，因为根未打开或出于某些其他安全原因。

操作：打开根或直接连接到root并再次尝试此命令。

ORA-39972：不能在SQL语句中使用具有PL / SQL唯一属性的对象类型

原因：在SQL语句中使用了使用PL / SQL唯一属性创建的对象类型。

操作：在SQL语句中仅使用SQL允许的类型属性。
