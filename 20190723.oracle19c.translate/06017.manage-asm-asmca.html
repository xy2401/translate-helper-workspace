9使用ASMCA管理Oracle ASM

Oracle ASM Configuration Assistant（ASMCA）提供用于管理Oracle ASM的实用程序。

Oracle ASM Configuration Assistant（ASMCA）支持安装和配置Oracle ASM实例，Oracle ASM磁盘组，Oracle ASM动态卷管理器（Oracle ADVM）卷和Oracle自动存储管理群集文件系统（Oracle ACFS）文件系统。此外，您还可以使用ASMCA命令行界面。

本章讨论以下主题：

  • ASMCA GUI工具入门

  • 使用ASMCA管理实例和磁盘组

  • ASMCA命令行界面

有关使用ASMCA管理Oracle ACFS和Oracle ADVM的信息，请参阅使用ASMCA 管理Oracle ACFS和Oracle ADVM 。

注意：

某些ASMCA功能和菜单选项仅在特定环境中可用，例如Oracle RAC配置。有关特定ASMCA页面的详细信息，请访问“ 帮助”按钮提供的在线帮助。

ASMCA GUI工具入门

要启动ASMCA， asmca在Oracle Grid Infrastructure主目录内的命令提示符下输入asmca 。例如：

$ asmca

GUI工具显示并尝试连接到使用设置为+ASM的Oracle系统标识符（SID）标识的Oracle ASM实例。

如果ASMCA检测到未运行的Oracle ASM实例，则ASMCA会提示您是否要启动该实例。

要启动命令行ASMCA，请参阅ASMCA命令行界面 。

注意：

某些ASMCA功能和菜单选项仅在特定环境中可用，例如Oracle RAC配置。有关特定ASMCA页面的详细信息，请访问“ 帮助”按钮提供的在线帮助。

使用ASMCA创建Oracle ASM实例

如果在主机上找不到Oracle ASM实例，则Oracle ASM Configuration Assistant允许您创建Oracle ASM实例。

在“ 创建 ASM”页面上，输入Oracle ASM实例的管理员的SYS密码以及可以监视此Oracle ASM实例的ASMSNMP用户的密码。ASMSNMP是一个权限较低的用户，主要用于监视Oracle ASM实例。

您可以选择用于注册此Oracle ASM实例的侦听器。单击“ ASM 参数”以更新Oracle ASM实例的初始化参数。

如果可以在“ 创建 ASM”页面中单击“ ASM 参数” ，则可以在Oracle ASM Configuration Assistant页面上更新Oracle ASM实例的初始化参数。

使用ASMCA管理磁盘组

Oracle ASM Configuration Assistant使您可以使用“ 配置 ASM 磁盘 组”选项卡配置或创建Oracle ASM磁盘组。

磁盘组选项卡显示有关磁盘组的选定详细信息，例如名称，大小，可用空间，可用空间，冗余级别和磁盘组的状态。

您可以右键单击屏幕中的选定磁盘组以显示配置选项菜单。菜单选项使您能够：

  • 将磁盘添加到磁盘组并从磁盘组中删除磁盘

  • 添加，删除和列出配额组。

  • 添加，删除和移动文件组。

  • 编辑磁盘组属性

  • 管理磁盘组的模板

  • 在磁盘组上创建Oracle ACFS文件系统以供数据库使用

  • 装载和卸载磁盘组

  • 删除磁盘组

  • 删除并清除Oracle ASM筛选器驱动程序标签。

还有其他选项可用于创建磁盘组，装入所有磁盘组以及卸载所有磁盘组。

Oracle ASM Configuration Assistant使您可以通过单击Oracle ASM Configuration Assistant磁盘组页面上的“ 创建 ”来创建 Oracle ASM磁盘组。

在“ 创建磁盘组”页面上，您可以输入磁盘组的名称，选择冗余级别，更改磁盘发现路径以在不同目录中查找磁盘，识别磁盘组中的磁盘，输入磁盘组的名称故障组，并指定故障组是否为仲裁类型。

仲裁失败组是一种特殊类型的故障组，这些故障组中的磁盘不包含用户数据，在确定与用户存储有关的冗余要求时，不考虑仲裁失败组。但是，安装磁盘组时会出现仲裁失败组。

如果已为系统配置了Oracle ASM筛选器驱动程序，则可以将Oracle ASM筛选器驱动程序标签添加到磁盘或从磁盘清除Oracle ASM筛选器驱动程序标签。

您可以选择在创建磁盘组时显示高级选项。高级选项部分允许您设置其他选项，例如磁盘组兼容性设置和分配单元（AU）大小。完成创建磁盘组规范后，单击“ 确定” 。

注意：

磁盘组兼容性设置只能进行。提升磁盘组兼容性设置后，无法恢复到较低的兼容性设置。

注意：

如果磁盘组类型为FLEX或EXTEND ，则Free（GB）和Usable（GB）空间的0 。在这种情况下， 0值表示没有给出值，而不是Free（GB）和Usable（GB）空间中的零量。

  • 管理Oracle ASM筛选器驱动程序以获取有关Oracle ASM筛选器驱动程序的信息

  • 磁盘组兼容性和磁盘组兼容性 概述，以获取有关磁盘组兼容性属性和提高磁盘组兼容性的信息

配置Oracle ASM实例

- configureASM配置或创建Oracle ASM实例。

句法

asmca -silent -configureASM [-sysAsmPassword sysasm_password] [-asmsnmpPassword asm_monitor_password] [{-param PARAMNAME = paramvalue，...}] [{-diskString discovery_path {-diskGroupName diskgroup { -disk disk_path [-diskName disk ] | -diskList disk_list [-diskSize disk_size_MB ] [-failuregroup failure_group ] [-force | -noforce] [-quorum | -noquorum]} [-redundancy {HIGH |正常| EXTERNAL}]}] [-au_size value ] [-compatible.asm value ] [-compatible.rdbms value ] [-compatible.advm value ]}

表9-2包含 - configureASM命令可用的选项。

表9-2 -configureASM命令的选项

选项 描述

-param paramname = paramvalue ，...

指定具有Oracle ASM实例值的初始化参数列表。

-diskString discovery_path

指定用于查找磁盘的Oracle ASM发现字符串。

-diskGroupName diskgroup

指定要创建的Oracle ASM磁盘组的名称。

-disk disk_path

指定磁盘设备的完整路径。

-diskName disk

指定用户提供的磁盘名称。

-diskList disk_list

指定要添加到磁盘组的以逗号分隔的磁盘名称列表。

-diskSize disk_size_MB

指定用于Oracle ASM存储的磁盘大小。

-failuregroup failure_group

指定故障组名称。

-force | -noforce

指定强制使用磁盘的位置。

-quorum | -noquorum

指定仲裁或标准故障组。

-redundancy { HIGH | NORMAL | EXTERNAL }

指定Oracle ASM磁盘组的冗余设置。
