113 DBMS_OUTLN

DBMS_OUTLN包与OUTLN_PKG同义，包含与存储的轮廓管理相关的子程序的功能接口。

注意：

在将来的版本中将支持存储的大纲，以支持SQL计划管理。在Oracle Database 11g第1版（11.1）中，存储的大纲继续像过去的版本一样运行。但是，Oracle强烈建议您对新应用程序使用SQL计划管理。SQL计划管理创建SQL计划基准，与存储的轮廓相比，它提供了卓越的SQL性能和稳定性。如果您有现有的存储大纲，请考虑使用DBMS_SPM软件包的LOAD_PLANS_FROM_CURSOR_CACHE函数或LOAD_PLANS_FROM_SQLSET函数将它们迁移到SQL计划基准。迁移完成后，您应该禁用或删除存储的大纲。

113.1 DBMS_OUTLN概述

存储的大纲是与给定SQL语句的执行计划有关的存储数据。它使优化器能够重复创建与最初与大纲一起生成的计划等效的执行计划。存储在轮廓中的数据部分地由用于实现计划稳定性的一组提示组成。

在将来的版本中将取消支持存储的大纲，以支持SQL计划管理。从11g R1开始，存储的轮廓将继续像过去的版本一样运行，但Oracle强烈建议您对新应用程序使用SQL计划管理。SQL计划管理创建SQL计划基准，与存储的轮廓相比，它提供了卓越的SQL性能和稳定性。如果您有现有的存储大纲，请考虑使用LOAD_PLANS_FROM_CURSOR_CACHE函数或DBMS_SPM软件包的LOAD_PLANS_FROM_SQLSET函数将它们迁移到SQL计划基准。迁移完成后，您应该禁用或删除存储的大纲。

113.2 DBMS_OUTLN安全模型

DBMS_OUTLN包含仅适用于适当用户的管理过程。除非DBA明确这样做，否则EXECUTE特权不会扩展到一般用户社区。

可用于大纲管理目的的PL / SQL函数只能由具有过程（或包）的EXECUTE特权的用户EXECUTE 。

113.3.1 CLEAR_USED程序

此过程清除轮廓“已使用”标志。

句法

DBMS_OUTLN.CLEAR_USED（名称IN VARCHAR2）;

参数

表113-2 CLEAR_USED过程参数

参数 描述

名称

大纲名称。

113.3.2 CREATE_OUTLINE过程

此过程通过从散列值和子编号标识的共享游标中重新分析SQL语句来生成大纲。

句法

DBMS_OUTLN.CREATE_OUTLINE（hash_value IN NUMBER，child_number IN NUMBER，类别IN VARCHAR2 DEFAULT'DEFAULT'）;

参数

表113-3 CREATE_OUTLINE过程参数

参数 描述

HASH_VALUE

标识目标共享游标的哈希值。

CHILD_NUMBER

目标共享游标的子编号。

类别

用于创建轮廓的类别（可选）。

113.3.3 DROP_BY_CAT过程

此过程会删除属于特定类别的轮廓。虽然除非另有说明，否则将轮廓放入DEFAULT类别，用户可以选择将其轮廓分组为称为类别的组。

句法

DBMS_OUTLN.DROP_BY_CAT（cat VARCHAR2）;

参数

表113-4 DROP_BY_CAT过程参数

参数 描述

猫

要删除的大纲类别。

使用说明

此过程在一次调用中清除一类轮廓。

例子

此示例删除DEFAULT类别中的所有轮廓：

DBMS_OUTLN.DROP_BY_CAT（ 'DEFAULT'）;

113.3.4 DROP_UNUSED过程

此过程将删除从未在SQL语句的编译中应用的大纲。

句法

DBMS_OUTLN.DROP_UNUSED;

使用说明

您可以将DROP_UNUSED用于应用程序生成的轮廓，以便一次性使用由动态SQL创建的SQL语句。从不使用这些大纲并占用宝贵的磁盘空间。

113.3.5 EXACT_TEXT_SIGNATURES过程

此过程将概要签名更新为基于精确文本匹配计算的签名。

句法

DBMS_OUTLN.EXACT_TEXT_SIGNATURES;

使用说明

此过程仅适用于将大纲降级到8.1.6或更早版本。

113.3.6 UPDATE_BY_CAT过程

此过程将一个类别中所有轮廓的类别更改为新类别。

句法

DBMS_OUTLN.UPDATE_BY_CAT（oldcat VARCHAR2默认为'DEFAULT'，newcat VARCHAR2默认为'DEFAULT'）;

参数

表113-5 UPDATE_BY_CAT过程参数

参数 描述

oldcat

当前的轮廓类别。

newcat

新类别的轮廓。

113.3.7 UPDATE_SIGNATURES过程

此过程将大纲签名更新为当前版本的签名。

句法

DBMS_OUTLN.UPDATE_SIGNATURES;

使用说明

如果已导入早期版本中生成的轮廓，则应执行此过程，以确保签名与当前版本的计算算法兼容。
