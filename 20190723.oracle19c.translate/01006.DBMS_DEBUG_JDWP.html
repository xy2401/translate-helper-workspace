55 DBMS_DEBUG_JDWP

DBMS_DEBUG_JDWP提供了通过Java调试线协议（JDWP）启动和控制PL / SQL存储过程和Java存储过程的调试的接口。

本章包含以下主题：

  • DBMS_DEBUG_JDWP概述

  • DBMS_DEBUG_JDWP安全模型

  • DBMS_DEBUG_JDWP子程序摘要

55.1 DBMS_DEBUG_JDWP概述

Oracle支持通过JDWP调试PL / SQL存储过程和Java存储过程。使用DBMS_DEBUG_JDWP包，您可以：

  • 检索当前会话的会话ID和序列号

  • 通过JDWP将数据库会话连接到调试器

  • 设置NLS参数以更改通过JDWP表示PL / SQL程序值的格式

55.2 DBMS_DEBUG_JDWP安全模型

调用DBMS_DEBUG_JDWP子程序的调试用户需要具有将目标数据库会话连接到调试器的权限。

表55-1系统权限

系统权限 描述

调试连接会议

允许用户将其当前会话连接到调试器

调试连接任何

允许用户将任何登录用户的会话连接到调试器

此外，还可以授予用户以下用户权限以调试另一个用户的会话。

表55-2用户权限

用户权限 描述

DEBUG CONNECT ON USER < user >

允许用户将任何指定用户的登录会话连接到调试器，以调试其他用户的会话或他自己的会话

55.3.1 CONNECT_TCP程序

此过程将指定的会话连接到在host:port处等待的调试器。

句法

DBMS_DEBUG_JDWP.CONNECT_TCP（主机IN VARCHAR2，端口IN VARCHAR2，session_id IN PLS_INTEGER：= NULL，session_serial IN PLS_INTEGER：= NULL，debug_role IN VARCHAR2：= NULL，debug_role_pwd IN VARCHAR2：= NULL，option_flags IN PLS_INTEGER：= 0，extensions_cmd_set IN PLS_INTEGER ：= 128）;

参数

表55-4 CONNECT_TCP参数

参数 描述

主办

调试器正在等待的主机名

港口

调试器正在等待的端口号

SESSION_ID

会话ID

session_serial

会话编号

debug_role

调试角色

debug_role_pwd

调试密码

option_flags

价值观：

  • 1 ：在下一个客户端/服务器请求开始之前，不会挂起程序。这可以用于隐藏最终用户的启动顺序，最终用户可能只想查看自己的代码

  • 2 ：即使会话似乎连接到调试器，也强制连接。这应该最好只在一些人为交互确认步骤发生后指定;即，如果没有此选项的尝试引发ORA-30677，则如果用户确认，则使用此位设置重试。

这些可以一起添加以选择多个选项。

extensions_cmd_set

Oracle JDWP扩展命令集的ID

表55-5 CONNECT_TCP例外

例外 描述

ORA-01031

权限不足

ORA-30677

ORA-30682

ORA-30683

使用说明

  • 要将当前会话连接到调试器，可以将NULL传递给session_id和session_serial参数。

  • 要连接其他会话，您需要找到其ID和序列。这些可在v$plsql_debuggable_sessions视图中找到。Oracle Enterprise Manager的“实例管理器”选项是向用户显示这些值的用户界面的一个示例。您还可以使用CURRENT_SESSION_ID函数和CURRENT_SESSION_SERIAL函数为您自己的会话找到这些值。

  • debug_role和debug_role_pwd参数允许用户将任何角色命名为“调试角色”，在检查连接会话的权限和检查调试会话中对象的可用权限时，可以使用权限检查。角色及其密码都在此处作为字符串而不是标识符传递，因此不应使用双引号但案例很重要。如果原始角色名称不是双引号，则应在此处以大写形式指定。

  • ORA-30677表示所请求的会话已在调试中。在这种情况下建议用户被要求确认他希望从现有连接中窃取会话，然后可以使用显式断开连接调用或使用connect_force_connect选项位来允许连接第二次尝试成功。请注意，如果当前通过调试器暂停会话，则使用connect_force_connect位将避免允许会话自由运行 - 换句话说，此位允许您将会话从一个调试器窃取到另一个调试器而不会实际干扰会话状态。

55.3.4断开程序

此过程将指定的会话与其连接的任何调试器断开连接。

句法

DBMS_DEBUG_JDWP.DISCONNECT（session_id IN PLS_INTEGER：= NULL，session_serial IN PLS_INTEGER：= NULL）;

参数

表55-6 DISCONNECT过程参数

参数 描述

SESSION_ID

会话ID

session_serial

会话编号

使用说明

  • 如果要断开的会话是当前会话，则断开调试器后将允许会话自由运行。否则，会话将被终止。

  • 此呼叫需要与connect相同的权限，除非断开当前会话，并且在呼叫时有效用户与不需要权限的会话的登录用户相同。

55.3.5 GET_NLS_PARAMETER函数

此函数获取指定NLS参数的值，该参数影响PL / SQL程序的NUMBER ， DATE ， TIME (WITH TIME ZONE)和TIMESTAMP (WITH TIME ZONE)运行时值转换为字符串的格式，因为它们通过JDWP。

这些值对于当前会话是私有的，但是对于调试器机制而言是私有的，与用于在调试的程序本身内转换值的值分开。

句法

DBMS_DEBUG_JDWP.GET_NLS_PARAMETER（名称IN VARCHAR2）RETURN VARCHAR2;

参数

表55-7 GET_NLS_PARAMETER函数参数

参数 描述

名称

指定的NLS参数

使用说明

  • 当通过JDWP读取或分配任何变量值时，或者在会话中首次调用GET_NLS_PARAMETER Function或SET_NLS_PARAMETER Procedure ，调试器机制会生成当时的NLS_LANGUAGE ， NLS_TERRITORY ， NLS_CALENDAR ， NLS_DATE_LANGUAGE ， NLS_NUMERIC_CHARACTERS ， NLS_TIMESTAMP_FORMAT的私有副本， NLS_TIMESTAMP_TZ_FORMAT ， NLS_TIME_FOMAT和NLS_TIME_TZ_FORMAT值。可以使用此GET_NLS_PARAMETER Function调用读取这些私有副本，并使用以下对SET_NLS_PARAMETER过程的调用进行更改 。

  • 一旦建立了调试器的NLS参数的私有副本，使用ALTER SESSION语句对当前会话中的NLS参数所做的ALTER SESSION将不会影响通过JDWP看到的值的格式。要修改用于JDWP的NLS参数，必须使用SET_NLS_PARAMETER Procedure 。SET_NLS_PARAMETER Procedure同样的原因，使用SET_NLS_PARAMETER Procedure对调试器的NLS参数的私有副本所做的更改将对调试的程序本身没有影响。

  • 始终使用NLS_TIMESTAMP_FORMAT格式化日期值以供JDWP使用。会话中使用的DATE (NLS_DATE_FORMAT)的默认格式通常不会显示实际存在于值中的时间信息，并且出于调试目的，始终显示该信息似乎是有益的。

55.3.6 PROCESS_CONNECT_STRING过程

此过程以两种方式将会话连接到调试器，因此您不必直接修改应用程序代码。

这两种方式是：

  • 在运行OCI程序时使用ORA_DEBUG_JDWP环境变量

  • 在通过PL / SQL Web Gateway运行应用程序时，设置名为OWA_DEBUG_<dad>的Web浏览器cookie

句法

DBMS_DEBUG_JDWP.PROCESS_CONNECT_STRING（connect_string IN VARCHAR2，connect_string_type IN PLS_INTEGER）;

参数

表55-8 PROCESS_CONNECT_STRING过程参数

参数 描述

CONNECT_STRING

ORA_DEBUG_JDWP环境变量或OWA_DEBUG_<dad> cookie值，包含JDWP连接信息，例如要连接的调试器的主机和端口号

connect_string_type
可以有以下两个值：

  • 如果从ORA_DEBUG_JDWP环境中检索连接字符串值， ORA_DEBUG_JDWP

  • 如果值来自OWA_DEBUG_<dad> cookie， OWA_DEBUG_<dad> 2
