目的

SQL / JSON函数JSON_OBJECTAGG是一个聚合函数。它将属性键值对作为输入。通常，属性键，属性值或两者都是SQL表达式的列。此函数为每个键值对构造一个对象成员，并返回包含这些对象成员的单个JSON对象。

[KEY] 字符串 VALUE expr

使用此子句指定属性键 - 值对。

  • KEY是可选的，并且是为了语义清晰度而提供的。

  • 使用string将属性键名称指定为区分大小写的文本文字。

  • 使用expr指定属性值。对于expr ，您可以指定任何计算结果为SQL数字文字，文本文字，日期或时间戳的表达式。日期和时间戳数据类型将按照ISO 8601日期格式作为JSON字符串打印在生成的JSON对象或数组中。如果expr求值为数字文字，则结果属性值为JSON数值;否则，结果属性值是用双引号括起来的区分大小写的JSON字符串值。

格式JSON

使用此可选子句指示输入字符串是JSON，因此不会在输出中引用。

JSON_on_null_clause

当expr求值为null时，使用此子句指定此函数的行为。

  • NULL ON NULL - 当指定NULL ON NULL时，JSON NULL值将用作给定键的值。

  • ABSENT ON NULL - 如果指定此子句，则该函数将忽略JSON对象中的属性键值对。

JSON_agg_returning_clause

使用此子句指定此函数返回的字符串的数据类型。您可以指定以下数据类型：

  • VARCHAR2[( size [BYTE,CHAR])]

    在SQL中的其他位置指定VARCHAR2数据类型时，您需要指定大小。但是，在本节中，您可以省略大小。

  • CLOB返回包含单字节或多字节字符的字符大对象。

  • BLOB返回AL32UTF8字符集的二进制大对象。

如果省略此子句，或者如果指定VARCHAR2但省略size值，则JSON_OBJECTAGG将返回VARCHAR2(4000)类型的字符串VARCHAR2(4000) 。

有关上述数据类型的更多信息，请参阅“ 数据类型 ” 。

严格

指定STRICT子句以验证JSON生成函数的输出是否正确JSON。如果检查失败，则会引发语法错误。

有关示例，请参阅JSON_OBJECT 。

用独特的钥匙

指定WITH UNIQUE KEYS以保证生成的JSON对象具有唯一键。

例子

以下示例构造一个JSON对象，其成员包含部门名称和部门编号：

SELECT JSON_OBJECTAGG（KEY department_name VALUE department_id）“部门编号”FROM部门WHERE department_id <= 30;部门编号------------------------------------------------ ---- {“管理”：10，“营销”：20，“采购”：30}
