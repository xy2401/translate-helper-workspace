使用CVU数据库升级验证命令选项

使用Cluster Verification Utility（CVU）命令cluvfy stage -pre dbinst -upgrade检查Oracle RAC安装是否已准备好进行升级。

使用-predbinst -upgrade选项运行cluvfy会执行系统检查，以确认群集是否处于从现有Oracle RAC安装升级的正确状态。

该命令使用以下语法，其中变量内容以斜体表示：

cluvfy stage -pre dbinst -upgrade -src_dbhome src_RAChome [-dbname db_names_list ] -dest_dbhome dest_RAChome -dest_version dest_version [-fixup] [-fixupnoexec] [ -  method root | -method sudo -user user_name [-location dir_path ]] [-verbose]

在设置Oracle RAC数据库之前，该命令会对集群中的所有节点执行相应的检查。下表介绍了该命令的选项。

表C-1 Oracle RAC数据库的群集验证实用程序命令选项

选项 描述
-src_dbhome src_RAChome 要升级的源Oracle RAC主目录的位置，其中src_RAChome是要升级的主src_RAChome的路径。
-dest_dbhome dest_RAChome 用于升级的Oracle RAC主目录的位置，其中dest_RAChome是Oracle RAC主dest_RAChome的路径。
-dest_version dest_version 升级版本号，包括任何补丁集。版本号必须包含五个数字，指定发布到特定于平台的修补程序级别。例如：12.2.0.1.0。
—user user_name 用户名以访问具有root权限的所有节点。
-location dir_path sudo可执行文件的完整文件系统路径。
-dbname db_names_list 要升级的数据库的唯一名称列表。
-verbose 显示单个检查的详细输出。
-修理 如果指定，则在验证失败时，执行修复操作。
-fixupnoexec 如果指定，则在验证失败时，将生成修正数据，并显示手动执行生成的修正的指令。
