ORA-00000：正常，成功完成

原因：正常退出。

行动：没有

ORA-00001：违反了唯一约束（ string 。 string ）

原因： UPDATE或INSERT语句尝试插入重复键。对于在DBMS MAC模式下配置的受信任Oracle，如果存在不同级别的重复条目，则可能会看到此消息。

操作：删除唯一限制或不插入密钥。

ORA-00017：会话请求设置跟踪事件

原因：请求当前会话通过另一个会话设置跟踪事件。

行动：这在内部使用;不需要采取任何行动

ORA-00018：超出最大会话数

原因：所有会话状态对象都在使用中。

操作：增加SESSIONS初始化参数的值。

ORA-00019：超出会话许可证的最大数量

原因：所有许可证都在使用中。

操作：增加LICENSE MAX SESSIONS初始化参数的值。

ORA-00020：超出了最大进程数（ string ）

原因：所有进程状态对象都在使用中。

操作：增加PROCESSES初始化参数的值。

ORA-00021：附加到其他进程的会话;无法切换会话

原因：用户会话当前由其他人使用。

操作：不要切换到附加到其他进程的会话。

ORA-00022：会话ID无效;拒绝访问

原因：指定的会话不存在或调用者没有访问权限。

操作：指定您有权访问的有效会话ID，即您拥有它或具有CHANGE_USER权限。

ORA-00023：会话引用处理私有内存;无法分离会话

原因：当包含对进程专用内存的引用时，尝试分离当前会话。

操作：如果会话具有开放式网络连接，非常大的上下文区域或操作系统权限，则会话可能包含对进程内存（PGA）的引用。为了允许分离，可能需要关闭会话的数据库链接和/或游标。始终不允许使用操作系统权限分离会话。

ORA-00024：在单进程模式下不允许来自多个进程的登录

原因：尝试从ORACLE的不同进程多次登录以单进程模式启动。

操作：从其他进程注销。

ORA-00025：无法分配string

原因：内存不足。

操作：使用较大的sga堆重新启动。

ORA-00026：会话ID丢失或无效

操作：使用有效的会话ID重试。

原因：尝试使用ALTER SYSTEM KILL SESSION来终止当前会话。

行动：没有

原因：特权用户已终止您的会话，您不再登录到数据库。

操作：如果您希望继续工作，请再次登录。

ORA-00029：会话不是用户会话

原因：当前操作中指定的会话ID不是用户会话（例如，递归等）。

操作：使用用户会话ID重试。

ORA-00030：用户会话ID不存在。

原因：用户会话ID不再存在，可能是因为会话已注销。

操作：使用有效的会话ID。

ORA-00031：标记为kill的会话

原因：在ALTER SYSTEM KILL SESSION命令中指定的会话不能立即被终止（因为它在网络操作中回滚或被阻止），但它已被标记为kill。这意味着它将在当前不间断操作完成后尽快被杀死。

操作：要杀死的会话不需要任何操作，但在此会话中进一步执行ALTER SYSTEM KILL SESSION命令可能会导致会话更快被杀死。

原因：会话创建调用中指定的会话迁移密码无效（可能太长）。

操作：使用有效密码重试（少于30个字符）。

ORA-00033：当前会话具有空迁移密码

原因：尝试分离或克隆当前会话，并且它具有空的迁移密码。这是不允许的。

操作：使用非空迁移密码创建会话。

操作：在此会话中启用PL / SQL提交，或者在当前会话中禁用PL / SQL时不尝试使用提交或回滚。

原因：尝试在由其他用户拥有的服务器组中创建不可迁移的会话。

操作：服务器组由第一个用户拥有，该用户以非可迁移模式登录服务器组中的服务器。所有后续的非可迁移模式登录都必须由拥有服务器组的用户进行。要在不可迁移模式下使用其他用户登录，必须更改服务器组的所有权。这可以通过注销所有当前会话并从服务器组中的所有现有服务器分离，然后让新用户登录成为新所有者来完成。
