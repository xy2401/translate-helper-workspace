确认Oracle Clusterware功能

安装Oracle Grid Infrastructure后，确认已安装并正确运行Oracle Clusterware安装。

安装后，以root身份登录，并使用以下命令语法确认您的Oracle Clusterware安装已安装并正确运行：

crsctl检查集群-all

例如：

$ crsctl check cluster -all ******************************************** ****************** node1：CRS-4537：群集就绪服务在线CRS-4529：群集同步服务在线CRS-4533：事件管理器在线**** ************************************************** ******** node2：CRS-4537：群集就绪服务在线CRS-4529：群集同步服务在线CRS-4533：事件管理器在线************** ************************************************ node3： CRS-4537：群集就绪服务在线CRS-4529：群集同步服务在线CRS-4533：事件管理器在线************************ **************************************

注意：

安装完成后，请不要手动删除或运行在Oracle Clusterware启动时删除/tmp/.oracle或/var/tmp/.oracle或其文件的cron作业。如果删除这些文件，则Oracle Clusterware可能会遇到间歇性挂起，并且您将遇到错误CRS-0184: Cannot communicate with the CRS daemon 。
