4.7.1 JDBC对Oracle对象数据的访问

JDBC（Java数据库连接）是Oracle服务器的一组Java接口。

Oracle提供了对象和JDBC之间的紧密集成。您可以非常灵活地将SQL类型映射到Java类。

Oracle JDBC：

  • 允许通过动态SQL访问Java程序中的对象和集合类型（在数据库中定义）。

  • 通过默认或可自定义的映射将数据库中定义的类型转换为Java类。

JDBC规范的2.0版支持对象关系构造，例如用户定义（对象）类型。JDBC将Oracle对象实现为特定Java类的实例。使用JDBC访问Oracle对象涉及为Oracle对象创建Java类并填充这些类。你可以：

  • 让JDBC将对象实现为STRUCT 。在这种情况下，JDBC会为属性创建类并为您填充它们。

  • 手动指定Oracle对象和Java类之间的映射;也就是说，为对象数据定制Java类。然后，驱动程序将填充您指定的自定义Java类，这会对Java类施加一组约束。要满足这些约束，您可以选择根据SQLData接口或ORAData接口定义类。

4.7.2数据映射策略

Oracle SQLJ支持对象类型的强类型或弱类型Java表示，引用类型（ REF ）以及要在迭代器或宿主表达式中使用的集合类型（varrays和嵌套表）。

强类型表示使用与特定对象类型， REF类型或集合类型对应的自定义Java类 ，并且必须实现接口oracle 。 sql 。ORAData 。Oracle JVM Web服务调用实用程序可以自动生成此类自定义Java类。

弱类型表示使用类oracle 。 sql 。STRUCT （对象）， oracle 。 sql 。REF （供参考）或oracle 。 sql 。ARRAY （收藏品）。

4.7.3 Java对象存储

Oracle JVM Web服务调用实用程序使您可以构造映射到现有SQL类型的Java类。然后，您可以使用JDBC从Java应用程序访问SQL类型。

你也可以向另一个方向走。也就是说，您可以创建映射到现有Java类的SQL类型。此功能使您可以提供持久性 Java对象的存储。这种SQL类型称为语言Java的SQL类型，或 SQLJ对象类型。它们可以用作对象表中的对象，属性，列或行的类型。您可以通过对象引用或外键导航访问此类型的对象 - Java对象，并且可以从SQL查询和操作此类对象。

使用CREATE TYPE语句创建SQLJ类型，就像使用其他用户定义的SQL类型一样。对于SQLJ类型，在CREATE TYPE语句中添加了两个特殊元素：

  • 一个EXTERNAL NAME短语，用于标识每个SQLJ属性和方法的Java副本以及与SQLJ类型本身对应的Java类

  • 一个USING子句，用于指定如何将SQLJ类型表示给服务器。USING子句指定用于检索SQLJ类型和存储类型的接口。

4.7.3.2关于映射的附加说明

以下是将Java类映射到SQL类型时要考虑的其他注意事项：

  • 你可以映射一个 SQLJ静态函数到Java类中的用户定义构造函数。此函数的返回值是用户定义的类型，其中函数是本地定义的。

  • Java静态变量映射到SQLJ静态方法，返回由EXTERNAL NAME标识的相应静态变量的值。属性的EXTERNAL NAME子句是可选的，带有SQLData或ORAData表示。

  • SQL表示形式的SQLJ类型中的每个属性都必须映射到Java字段，但不是每个Java字段都必须映射到相应的SQLJ属性：您可以从映射中省略Java字段。

  • 您可以省略类：您可以将SQLJ类型映射到Java类层次结构中的非根类，而无需将SQLJ类型映射到根类和插入的超类。这样做可以隐藏超类，同时仍然包含从它们继承的属性和方法。

    但是，必须保留类层次结构中的节点与SQLJ类型层次结构中的节点之间的结构对应关系。换句话说，对于通过继承关联并分别映射到两个SQL类型s_A和s_B两个Java类j_A和j_B ，对于继承中的每个节点，从s_A到s_B的继承路径上必须只有一个对应的节点从j_A到j_B路径。

  • 只要不违反前一段中的限制，就可以将Java类映射到多个SQLJ类型。换句话说，映射到同一Java类的两个SQLJ类型不能具有共同的超类型祖先。

  • 如果所有Java类都未映射到SQLJ类型，则可能将SQLJ对象类型的属性设置为未映射的Java类的对象。具体而言，发生在继承层次结构中属性映射到的类的上方或下方的类。如果对象的类是属性的类型/类的超类，则会引发错误。如果它是属性的类型/类的子类，则该对象将映射到其层次结构中存在SQL映射的最具体类型

4.7.3.3 SQLJ类型演变

您可以使用ALTER TYPE语句来删除SQLJ类型。

ALTER TYPE语句使您可以通过添加或删除属性或方法来演化类型。

当演变SQLJ类型时，将执行附加验证以检查类和类型之间的映射。如果类和演进类型匹配，则类型标记为有效。否则，该类型将标记为挂起验证。

被标记为待定验证与被标记为无效的不同。例如，仍可以使用ALTER TYPE和GRANT语句操作待处理验证的TYPE 。

如果具有SQL表示的类型标记为挂起评估，则仍可以使用不需要方法调用的任何DML或SELECT语句来访问该类型的表。

但是，您不能对具有可序列化表示的类型的表执行DML或SELECT语句，并且已将其标记为挂起验证。可序列化类型的数据只能通过方法调用进行导航访问。对于待处理验证的类型，这些是不可能的。但是，您仍然可以重新进化类型，直到它通过验证。

请参阅“ 类型演变 ” 。

4.7.3.4约束

对于具有SQL表示的SQLJ类型，可以为普通对象类型定义相同的约束。

约束是在表上定义的，而不是在类型上定义的，并且在列级别定义。具有SQL表示的SQLJ类型支持以下约束：

  • 独特的约束

  • 首要的关键

  • 检查约束

  • 对属性的NOT NULL约束

  • 参考约束

对于具有SQL表示的SQLJ类型，也支持对列可替代性的IS OF TYPE约束。参见“ 约束可替代性 ” 。

4.7.3.6插入Java对象

您可以将Java对象插入表中。

在包含SQLJ类型列的表中插入行需要调用类型的构造函数来创建该类型的Java对象。

  • 使用隐式的，系统生成的构造函数，或定义映射到Java类中的用户定义构造函数的静态函数。

4.7.3.7更新SQLJ对象

您可以通过多种方式更新SQLJ对象。

SQLJ对象可以通过以下方式更新：

  • 使用UPDATE语句修改一个或多个属性的值

  • 调用一个更新属性并返回SELF的方法，即返回对象本身并进行更改。

例如，假设raise()是一个成员函数，它将salary字段/属性增加指定的量并返回SELF 。以下语句为对象表employee_objtab的每个雇员提高了1000 ：

更新employee_objtab SET c = c.raise（1000）;

可以使用与普通对象类型相同的语法将SQLJ类型的列设置为NULL或另一列。例如，以下语句将列d分配给列c ：

更新employee_reltab SET c = d;

4.7.5 JDeveloper

Oracle JDeveloper是一个功能齐全的跨平台集成开发环境，用于创建与Oracle应用服务器和数据库良好集成的多层Java应用程序。

Oracle JDeveloper使您能够基于行业标准模型开发，调试和部署Java客户端应用程序，动态HTML应用程序，Web和应用程序服务器组件，JavaBean组件和数据库存储过程。

JDeveloper也是ADF和TopLink的集成开发环境。

4.7.5.1应用程序开发框架（ADF）

ADF是用于构建可伸缩的企业Java EE应用程序的框架。开发人员可以使用ADF构建应用程序，其中应用程序数据持久保存到Oracle Object表以及其他架构对象。

4.7.5.2 TopLink

TopLink是一个框架，用于将Java对象映射到各种持久性技术（包括数据库），并提供利用Oracle Objects构建应用程序的工具。
