产品集

产品安装介质中包含以下产品组件：

  • 适用于APPC的Oracle数据库网关，12 c第2版（12.2）

  • Oracle Net，12 c第2版（12.2）

变化和增强

以下部分描述了网关11g版本特有的更改和增强功能：

  • 网关密码加密工具

  • 部分IPv6支持

网关密码加密工具

网关密码加密工具（ g4drpwd ）已被现在属于异构服务的通用功能所取代。有关详细信息，请参阅“Oracle数据库网关安装和配置指南（适用于Microsoft Windows）”中的第15章“安全注意事项”。

部分IPv6支持

Oracle数据库与网关之间存在完整的IPv6支持，但网关与IMS Connect之间尚不支持IPv6。

已知的限制

以下部分列出了针对APPC和PGAU的Oracle数据库网关的已知限制。

网关的已知限制

下列已知此版本的Oracle Database Gateway for APPC中存在限制。

数字数据和子句不支持多字节字符集

自3.4版以来，Oracle数据库网关APPC已支持COBOL PIC G数据类型的多字节字符集数据。但是，允许的非数字字符数据（如$，（，），+， - ，。） 仍必须指定DISPLAY数据类型和PIC 9编辑掩码 EBCDIC。非数字字符数据不受MBCS转换的约束。

CICS事务不允许PF密钥仿真

使用Oracle Database Gateway for APPC执行CICS事务时，无法模拟CICS PF密钥。

不支持APPC PIP数据

您无法在此版本的Oracle Database Gateway for APPC中定义和传输APPC PIP数据。

不支持浮点数据类型转换

Oracle Database Gateway for APPC不支持浮点数据类型转换。

交易程序负责所有数据压缩和解压缩

适用于APPC的Oracle数据库网关不提供压缩和解压缩工具的出口。网关和事务之间交换的所有数据必须采用未压缩格式。

不支持IBM VS COBOL II编译器

IBM VS COBOL II编译器已被不支持。但是，字符串“ IBMVSCOBOLII ”仍用作编译器名称参数的值，以表示您选择使用的任何COBOL编译器。IBMVSCOBOLII使用值IBMVSCOBOLII ，并且不会对任何特定版本的编译器创建依赖关系。

PGAU的已知限制

PGAU COBOL COPY REPLACE的限制

当COBOL输入到 PGAU DEFINE DATA语句包含COPY REPLACE子句，只进行第一次替换。
