执行Oracle Grid Infrastructure的滚动升级

查看此信息以执行Oracle Grid Infrastructure的滚动升级。

  • 从早期版本升级Oracle Grid Infrastructure
    完成此过程以从早期版本升级Oracle Grid Infrastructure（Oracle Clusterware和Oracle自动存储管理）。
  • 在节点无法访问时完成Oracle Clusterware升级
    如果某些节点在升级过程中无法访问，则无法完成升级，因为升级脚本（ rootupgrade.sh ）未在无法访问的节点上运行。由于升级不完整，Oracle Clusterware仍保留在先前版本中。
  • 强制升级后加入无法访问的节点
    强制群集升级后，使用此过程连接不可访问的节点。
  • 更改第一个节点以进行安装和升级
    如果第一个节点无法访问，则可以强制另一个节点成为安装或升级的第一个节点。
