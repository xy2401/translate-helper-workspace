目的

FIRST_VALUE是一个分析函数。它返回有序值集中的第一个值。如果集合的第一个值是零，则函数返回NULL ，除非您指定IGNORE NULLS 。此设置对数据密集化很有用。

注意：

此语法的两种形式具有相同的行为。顶部分支是ANSI格式，Oracle建议ANSI兼容。

{ RESPECT | IGNORE } NULLS确定expr空值是否包含在计算中或从计算中消除。默认值为RESPECT NULLS 。如果指定IGNORE NULLS ，则FIRST_VALUE返回集合中的第一个非空值，如果所有值都为null，则返回NULL 。有关数据密集化的示例，请参阅“ 使用分区外连接：示例 ” 。

您不能通过使用FIRST_VALUE或expr任何其他分析函数来嵌套分析函数。但是，您可以为expr使用其他内置函数表达式。有关有效形式的expr信息，请参阅“ 关于SQL表达式 ” 。

例子

以下示例为Department 90中的每个员工选择薪水最低的员工的姓名。

SELECT employee_id，last_name，salary，hire_date，FIRST_VALUE（last_name）OVER（ORDER BY salary ASC ROWS UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 102 De Haan 17000 13-JAN-01 De Haan 101 Kochhar 17000 21-SEP-05 De Haan 100 King 24000 17-JUN-03 De Haan

该示例说明了FIRST_VALUE函数的非确定性。Kochhar和DeHaan的工资相同，相邻的行也是如此。Kochhar首先出现，因为子查询返回的行由hire_date排序。但是，如果子查询返回的行按hire_date按降序排序，如下一个示例所示，则函数返回不同的值：

SELECT employee_id，last_name，salary，hire_date，FIRST_VALUE（last_name）OVER（ORDER BY salary ASC ROWS UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER by hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 101 Kochhar 17000 21-SEP-05 Kochhar 102 De Haan 17000 13-JAN-01 Kochhar 100 King 24000 17-JUN-03 Kochhar

以下两个示例显示如何通过对唯一键进行排序来使FIRST_VALUE函数具有确定性。通过使用salary和唯一键employee_id在函数内进行排序，无论子查询中的顺序如何，都可以确保相同的结果。 

SELECT employee_id，last_name，salary，hire_date，FIRST_VALUE（last_name）OVER（ORDER BY salary ASC， employee_id ROWS UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 101 Kochhar 17000 21-SEP-05 Kochhar 102 De Haan 17000 13-JAN-01 Kochhar 100 King 24000 17-JUN-03 Kochhar SELECT employee_id，last_name， salary，hire_date，FIRST_VALUE（last_name）OVER（ORDER BY salary ASC， employee_id ROWS UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 101 Kochhar 17000 21-SEP-05 Kochhar 102 De Haan 17000 13-JAN-01 Kochhar 100 King 24000 17-JUN-03 Kochhar

以下两个示例显示，当您使用逻辑偏移量（ RANGE而不是ROWS ）时， FIRST_VALUE函数是确定性的。当找到ORDER BY表达式的重复项时， FIRST_VALUE是expr的最小值：

SELECT employee_id，last_name，salary，hire_date，FIRST_VALUE（last_name）OVER（ORDER BY salary ASC RANGE UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 102 De Haan 17000 13-JAN-01 De Haan 101 Kochhar 17000 21-SEP-05 De Haan 100 King 24000 17-JUN-03 De Haan SELECT employee_id ，last_name，salary，hire_date，FIRST_VALUE（last_name）OVER（按薪水排序ASC RANGE UNBOUNDED PRECEDING）AS fv FROM（SELECT * FROM employees WHERE department_id = 90 ORDER BY hire_date DESC ）; EMPLOYEE_ID LAST_NAME SALARY HIRE_DATE FV ----------- ------------------------- --------- -  --------- ------- 102 De Haan 17000 13-JAN-01 De Haan 101 Kochhar 17000 21-SEP-05 De Haan 100 King 24000 17-JUN-03 De Haan
