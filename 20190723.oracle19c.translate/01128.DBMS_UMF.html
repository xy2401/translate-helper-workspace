178 DBMS_UMF

DBMS_UMF包提供了用于为Oracle数据库部署远程管理框架（RMF）的接口。RMF用于收集Oracle数据库的性能统计信息。

也可以看看：

有关为Oracle数据库配置RMF的详细信息，请参见“Oracle数据库性能调整指南” 。

本章包含以下主题：

  • DBMS_UMF子程序摘要

178.1 DBMS_UMF子程序摘要

本主题按字母顺序列出DBMS_UMF子程序，并简要介绍它们。

178.1.1 CONFIGURE_NODE过程

此过程配置需要向RMF拓扑注册的节点。必须在需要配置的节点上执行此过程。

句法

DBMS_UMF.CONFIGURE_NODE（node_name IN VARCHAR2 DEFAULT NULL，dblink_to_target IN VARCHAR2 DEFAULT NULL）;

参数

表178-2 CONFIGURE_NODE过程参数

参数 描述

NODE_NAME

节点的名称。

必须为RMF拓扑中的每个节点分配唯一的名称。如果在此过程中未为节点提供名称，则默认情况下DB_UNIQUE_NAME初始化参数DB_UNIQUE_NAME值指定为节点的名称。

dblink_to_target

从此节点到目标节点的数据库链接。

安全模型

只有数据库用户SYS和SYS$UMF才能执行此过程。

178.1.2 CREATE_LINK程序

此过程在RMF拓扑中的两个节点之间创建数据库链接。必须仅在目标节点上执行此过程。

句法

DBMS_UMF.CREATE_LINK（topology_name IN VARCHAR2，node_a_name IN VARCHAR2，node_b_name IN VARCHAR2，dblink_a_to_b IN VARCHAR2，dblink_b_to_a IN VARCHAR2）;

参数

表178-3 CREATE_LINK过程参数

参数 描述

topology_name

RMF拓扑的名称。

node_a_name

第一个节点的名称。

node_b_name

第二个节点的名称。

dblink_a_to_b

从第一个节点到第二个节点的数据库链接。

dblink_b_to_b

从第二个节点到第一个节点的数据库链接。

安全模型

只有数据库用户SYS和SYS$UMF才能执行此过程。

178.1.6 ENABLE_SERVICE过程

此过程在RMF拓扑中的节点上启用服务，例如AWR服务。必须仅在目标节点上执行此过程。

句法

DBMS_UMF.ENABLE_SERVICE（topology_name IN VARCHAR2，node_name IN VARCHAR2，service_type IN NUMBER）;

参数

表178-7 ENABLE_SERVICE过程参数

参数 描述

topology_name

RMF拓扑的名称。

NODE_NAME

需要启用特定服务的节点的名称。

服务类型

标识服务的数字常量。此参数唯一允许的值是UMF_SERVICE_TYPE_AWR ，它是自动工作负载存储库（AWR）服务。

安全模型

只有数据库用户SYS和SYS$UMF才能执行此过程。

178.1.13 REGISTER_NODE函数和过程

此功能和过程使用RMF拓扑注册节点。必须仅在RMF拓扑中的目标节点上执行此过程和函数。

句法

DBMS_UMF.REGISTER_NODE（topology_name IN VARCHAR2，node_name IN VARCHAR2，dblink_to_node IN VARCHAR2 DEFAULT NULL，dblink_from_node IN VARCHAR2 DEFAULT NULL，as_source IN VARCHAR2 DEFAULT'TRUE'，as_candidate_target IN VARCHAR2 DEFAULT'FALSE'）; DBMS_UMF.REGISTER_NODE（topology_name IN VARCHAR2，node_name IN VARCHAR2，dblink_to_node IN VARCHAR2 DEFAULT NULL，dblink_from_node IN VARCHAR2 DEFAULT NULL，as_source IN VARCHAR2 DEFAULT'TRUE'，as_candidate_target IN VARCHAR2 DEFAULT'FALSE'）; node_id OUT VARCHAR2）; DBMS_UMF.REGISTER_NODE（topology_name IN VARCHAR2，node_name IN VARCHAR2，dblink_to_node IN VARCHAR2 DEFAULT NULL，dblink_from_node IN VARCHAR2 DEFAULT NULL，as_source IN VARCHAR2 DEFAULT'TRUE'，as_candidate_target IN VARCHAR2 DEFAULT'FALSE'）返回号码;

参数

表178-12 REGISTER_NODE功能和过程参数

参数 描述

topology_name

RMF拓扑的名称。

NODE_NAME

要注册的节点的名称。

dblink_to_node

从目标到节点的数据库链接的名称。

dblink_from_node

从节点到目标的数据库链接的名称。

as_source

如果节点是源，则设置为TRUE ，否则设置为FALSE 。

as_candidate_target

设置为TRUE ，如果节点是候选目的地，否则设置为FALSE 。

NODE_ID

过程返回的节点ID。

回报价值

返回已注册节点的节点ID。

安全模型

只有数据库用户SYS和SYS$UMF才能执行此过程。

178.1.14 SWITCH_DESTINATION过程

此过程使候选目标成为RMF拓扑中的新目标。必须仅在候选目标节点上执行此过程。

句法

DBMS_UMF.SWITCH_DESTINATION（topology_name IN VARCHAR2，force_switch IN BOOLEAN DEFAULT TRUE）;

参数

表178-13 SWITCH_DESTINATION过程参数

参数 描述

topology_name

RMF拓扑的名称。

force_switch

如果设置为FALSE ，则在以下情况下执行此过程将失败：

  • 候选目的地是只读数据库，因此它不能成为新目的地。

  • 候选目标没有到拓扑中的一个或多个源的数据库链接。

  • 候选目的地无法从旧目的地获取最新的AWR数据。

如果设置为TRUE ，则在以下情况下执行此过程将失败：

  • 候选目的地是只读数据库，因此它不能成为新目的地。

安全模型

只有数据库用户SYS和SYS$UMF才能执行此过程。
