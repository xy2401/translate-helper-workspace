关于Oracle Grid Infrastructure干运行升级模式

Oracle Grid Infrastructure干运行升级模式使您可以检查系统是否准备好进行升级。

从Oracle Grid Infrastructure 19c开始，Oracle Grid Infrastructure安装程序使您可以执行干运行升级，以检查系统是否准备好进行升级。要执行Oracle Grid Infrastructure干运行升级，请创建具有必要用户组权限的新Grid主机，将Oracle Grid Infrastructure 19c黄金映像提取到新的Grid主目录，然后使用—dryRunForUpgrade标志启动安装程序。

注意：

安装程序在干运行升级模式下不执行实际升级。如果干运行成功，您可以从任何群集节点重新启动安装程序（无任何标记）以升级Oracle Grid Infrastructure。
安装程序在干运行升级模式下执行以下任务：

  • 验证新版本的存储和网络配置

  • 检查系统是否满足新版本的软件和硬件要求

  • 在开始升级之前检查补丁要求并应用必要的补丁

  • 在gridSetupActions<timestamp>.log日志文件中写入系统配置问题或错误

Grid基础架构干运行升级流程与常规升级类似，但安装程序不运行任何配置工具。
