9.134 V $ XML_AUDIT_TRAIL

V$XML_AUDIT_TRAIL显示以XML格式文件编写的标准，细粒度，SYS和强制审计记录。

注意：

此视图仅在未启用统一审核的Oracle数据库中填充。在Oracle数据库中启用统一审计时，审计记录将填充在新审计跟踪中，并可从UNIFIED_AUDIT_TRAIL查看。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

柱 数据类型 描述

AUDIT_TYPE

数

审计行的类型：

  • 1 =标准XML审计

  • 2 =细粒度XML审计

  • 4 = SYS XML审计

  • 8 =强制XML审计

SESSION_ID

数

Oracle会话的数字ID

PROXY_SESSIONID

数

代理会话序列号，如果企业用户已通过代理机制登录

STATEMENTID

数

语句运行的数字ID（语句可能导致多个审计记录）

ENTRYID

数

会话中审计跟踪条目的数字标识

EXTENDED_TIMESTAMP

TIMESTAMP（6）带时区

审计操作的时间戳（用户登录条目的时间戳由AUDIT SESSION创建）

GLOBAL_UID

VARCHAR2（32）

用户的全局用户标识符（如果用户已以企业用户身份登录）

DB_USER

VARCHAR2（128）

审核其操作的用户的数据库用户名

CLIENTIDENTIFIER

VARCHAR2（64）

Oracle会话中的客户端标识符

EXT_NAME

VARCHAR2（1024）

用户的外部名称

OS_USER

VARCHAR2（128）

操作系统登录已审核其操作的用户的用户名

OS_HOST

VARCHAR2（128）

客户端主机系统名称

OS_PROCESS

VARCHAR2（16）

操作系统进程标识Oracle服务器进程

终奌站

VARCHAR2（30）

用户终端的标识符

INSTANCE_NUMBER

数

INSTANCE_NUMBER初始化参数指定的实例编号

OBJECT_SCHEMA

VARCHAR2（128）

审计对象的所有者

OBJECT_NAME

VARCHAR2（128）

受操作影响的对象的名称

POLICY_NAME

VARCHAR2（128）

细粒度审计政策的名称

NEW_OWNER

VARCHAR2（128）

“ NEW_NAME列中指定的对象的所有者

新名字

VARCHAR2（128）

重命名后对象的新名称或基础对象的名称（例如， CREATE INDEX owner.obj_name ON new_owner.new_name ）

行动

数

操作类型的数字代码

STATEMENT_TYPE

数

行动描述

TRANSACTIONID

RAW（8）

访问或修改对象的事务的标识符

RETURNCODE

数

操作生成的Oracle错误代码。如果操作成功则为零。

SCN

数

查询的系统更改编号（SCN）

COMMENT_TEXT

VARCHAR2（4000）

对标准审核条目的文本注释。还指示用户如何进行身份验证 - 该方法可以是以下之一：

  • DATABASE - 身份验证是通过密码完成的

  • NETWORK - 身份验证由Oracle Net Services或Advanced Networking Option完成

  • PROXY - 客户端已被其他用户验证。代理用户的名称遵循方法类型。

AUTH_PRIVILEGES

VARCHAR2（32）

在标准审计跟踪条目中记录的GRANT和REVOKE语句中授予和撤销的权限

GRANTEE

VARCHAR2（128）

授予或撤销权限的用户

PRIV_USED

数

行动中使用的特权的数字代码（如果有）

SES_ACTIONS

VARCHAR2（16）

标准审计记录的会话摘要。每个操作类型一个12个字符的字符串，顺序如下：更改，审核，注释，删除，授予，索引，插入，锁定，重命名，选择，更新，闪回。

值： - =无，S =成功，F =失败，B =两者

OS_PRIVILEGE

VARCHAR2（7）

会话中使用的操作系统特权（ SYSDBA或SYSOPER ）（如果有）。如果没有使用权限，则它将为NONE 。

ECONTEXT_ID

VARCHAR2（64）

应用执行上下文标识

SQL_BIND

VARCHAR2（4000）

语句中使用的绑定变量列表

SQL_TEXT

VARCHAR2（4000）

触发审核事件的语句或命令

OBJ_EDITION_NAME

VARCHAR2（128）

包含审计对象的版本的名称

DBID

数

审计数据库的数据库标识符

RLS_INFO

VARCHAR2（4000）

存储由分隔符分隔的虚拟专用数据库（VPD）策略名称和谓词。

要将输出格式化为单个行，请使用DBMS_AUDIT_UTIL.DECODE_RLS_INFO_ATRAIL_XML函数。

当前用户

VARCHAR2（128）

语句执行的有效用户

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

注意：

仅当AUDIT_TRAIL初始化参数设置为xml, extended或AUDIT_SYS_OPERATIONS初始化参数设置为TRUE xml, extended才会填充SQL_BIND和SQL_TEXT列。
