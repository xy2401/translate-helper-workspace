12 Oracle数据编辑功能和功能

Oracle Data Redaction提供了多种编辑不同类型数据的方法。

  • 完整数据编辑以编辑所有数据
    完整数据编辑会编辑指定表或视图列的全部内容。
  • 部分数据编辑以编辑数据部分
    在部分数据编辑中，您可以编辑显示输出的部分内容。
  • 用于编辑数据模式的正则表达式
    正则表达式基于模式搜索来编辑列数据值中的特定数据。
  • 使用空值进行编辑
    您可以创建一个Oracle Data Redaction策略，通过将其替换为空值来编辑列数据。
  • 随机数据编辑生成随机值
    在随机数据编辑中，通过将其替换为随机值来编辑整个值。
  • 基于数据类型的完全，部分和随机编校的比较
    完整，部分和随机数据编校样式以不同方式影响Oracle内置，ANSI，用户定义和Oracle提供的类型。
  • 没有用于测试目的的编辑
    您可以创建不执行编校的数据编辑策略。
  • 命名数据编辑策略表达式的集中管理
    您可以创建一个可在多个表和视图的列中使用的命名策略表达式库。

完整数据编辑以编辑所有数据

完整数据编辑会编辑指定表或视图列的全部内容。

默认情况下，输出显示如下：

  • 字符数据类型：输出文本是单个空格。

  • 数字数据类型：输出文本为零（ 0 ）。

  • 日期时间数据类型：输出文本设置为2001年1月的第一天，显示为01-JAN-01 。

完全编校是默认值，只要数据编辑策略指定列但省略了function_type参数设置，就会使用完全编校。运行DBMS_REDACT.ADD_POLICY过程时，要为完全编校设置function_type参数设置，请输入以下设置：

function_type => DBMS_REDACT.FULL

您可以使用DBMS_REDACT.UPDATE_FULL_REDACTION_VALUES过程将完整的编辑输出更改为不同的值。

部分数据编辑以编辑数据部分

在部分数据编辑中，您可以编辑显示输出的部分内容。

您可以在实际数据中设置开始编校的位置，从该位置开始编辑的字符数以及要使用的编校字符。这种类型的编辑对于您希望查看数据以某种方式进行编辑的人员显而易见的情况非常有用。通常，您对信用卡或ID号使用此类型的编辑。

请注意，部分数据编校要求您的数据宽度保持固定。如果要编辑包含可变长度字符串值的列，则必须使用正则表达式。

要指定部分编校，必须将DBMS_REDACT.ADD_POLICY过程function_type参数设置为DBMS_REDACT.PARTIAL并使用function_parameters参数定义部分编校行为。

部分数据编校的显示输出可以如下：

  • 字符数据类型：当部分编辑时，可以编辑值为987-65-4320的社会安全号码（表示为字符数据类型中的带连字符的字符串），以便显示如下面的示例所示。右边的代码指定了如何编辑字符数据：它指定了实际数据的预期输入格式，用于显示编辑输出的格式，开始编辑的开始位置，要使用的字符用于编辑，以及编辑多少个字符。第一个示例使用预定义格式（在以前的版本中称为快捷方式）用于字符数据类型社会保险号，第二个示例使用星号（ * ）替换前五个数字，同时在数字之间保留连字符（ - ）。 

    XXX-XX-4320 function_parameters => DBMS_REDACT.REDACT_US_SSN_F5，***  -  **  -  4320 function_parameters =>'VVVFVVFVVVV，VVV-VV-VVVV，*，1,5'，

  • 号码数据类型：部分编辑的NUMBER数据类型社会保险号码987654328可能如下所示。两者都编辑前五位数字。第一个示例使用为NUMBER数据类型中的社会安全号码设计的预定义格式，第二个示例使用数字9替换前五个数字，从第一个数字开始。 

    XXXXX4328 function_parameters => DBMS_REDACT.REDACT_NUM_US_SSN_F5,999994328 function_parameters =>'9,1,5'，

用于编辑数据模式的正则表达式

正则表达式基于模式搜索来编辑列数据值中的特定数据。

例如，你可以编校的电子邮件地址的用户名，以便只有域显示（例如，更换hpreston的电子邮件地址hpreston@example.com与[redacted] ，使其显示为[redacted]@example.com ）。要执行编校，请将DBMS_REDACT.ADD_POLICY过程function_type参数设置为DBMS_REDACT.REGEXP或DBMS_REDACT.REGEXP_WIDTH ，然后使用以下参数构建正则表达式：

  • 字符串搜索模式（即要搜索的值），例如：

    regexp_pattern =>'（。+）@（。+ \。[A-ZA-Z] {2,4}）”

    此设置将查找以下形式的模式：

    one_or_more_characters @ one_or_more_characters 。 2-4_characters_in_range_A-Z_or_a-Z

  • 替换字符串，它替换regexp_pattern设置匹配的值。替换字符串可以包括对主正则表达式模式的子表达式的反向引用。以下示例将@符号（来自regexp_pattern设置）之前的数据替换为文本[redacted] regexp_pattern [redacted] 。\2设置指的是第二个匹配组，即(.+ \。[A-ZA-Z] {2,4})来自regexp_pattern设置。 

    regexp_replace_string =>'[编辑] @ \ 2'

  • 字符串搜索字符串的起始位置，例如数据的第一个字符，例如：

    regexp_position => DBMS_REDACT.RE_BEGINNING

  • 要执行的搜索和替换操作的类型，例如第一次出现，每五次出现或所有出现，例如：

    regexp_occurrence => DBMS_REDACT.RE_ALL

  • 搜索和替换操作的默认匹配行为，例如搜索是否区分大小写（ i将其设置为区分大小写）：

    regexp_match_parameter =>'我

除了默认参数之外，您还可以使用一组预定义格式，这些格式使您可以将常用的正则表达式用于电话号码，电子邮件地址和信用卡号。

使用空值进行编辑

您可以创建一个Oracle Data Redaction策略，通过将其替换为空值来编辑列数据。

此功能使您可以使用DBMS_REDACT.NULLIFY函数隐藏表或视图列中的所有敏感数据，并将其替换为空值。您可以使用DBMS_REDACT.ADD_POLICY或DBMS_REDACT.ALTER_POLICY过程的function_type参数来设置此功能。

例如：

function_type => DBMS_REDACT.NULLIFY

随机数据编辑生成随机值

在随机数据编辑中，通过将其替换为随机值来编辑整个值。

每次应用程序用户运行查询时，查询结果集中显示的编辑值会随机更改。

如果您不希望数据被编辑，那么这种类型的编辑很有用。它对数字和日期时间数据类型特别有效，因为很难区分随机数据和实际数据。

随机值的显示输出会根据编辑列的数据类型而更改，如下所示：

  • 字符数据类型：随机输出是字符的混合（例如， HTU[G{\pjkEWcK ）。它对CHAR和VARCHAR2数据类型的行为有所不同，如下所示：

    • CHAR数据类型：编辑的输出始终与列的字符集具有相同的字符集。编辑输出的字节长度始终与列定义长度（即创建表时提供的列长度）相同。例如，如果列是CHAR(20) ，则在用户查询的编辑输出中提供20个随机字符的字符串。

    • VARCHAR2数据类型：对于VARCHAR数据类型的随机编辑，编辑的输出始终与列的字符集具有相同的字符集。编辑输出的长度基于列中实际数据的长度而受到限制。不会显示超过实际数据长度的字符。例如，如果VARCHAR2(20)且被编辑的行包含长度为12的实际数据，则在该行的用户查询的编辑输出中提供12个随机字符（不是20）的字符串。

  • 数字数据类型：通过将其替换为以实际数据的绝对值为模的随机非负数来编辑每个实际数字值。此编辑导致随机数不超过实际数据的精度。例如，该号码987654321可以通过与任何数字的替换来删节12345678 ， 13579 ， 0 ，或987654320 ，但不被任何数目的替换它987654321 ， 99987654321 ，或-1 。数-123可以通过与数字替换来删节122 ， 0 ，或83 ，但不是由与任何数目的替换它123 ， 1123 ，或-2 。

    上述唯一的例外是当实际值是介于-1和9之间的整数时。在这种情况下，实际数据通过将其替换为模10（10）的随机非负整数来进行编辑。

  • 日期时间数据类型：使用随机数据编辑方式编辑日期数据类型的值时，Oracle数据库会显示随机日期，这些日期始终与实际数据的日期不同。

使用随机编辑的设置如下：

function_type => DBMS_REDACT.RANDOM

没有用于测试目的的编辑

您可以创建不执行编校的数据编辑策略。

这对于具有编辑基表的情况很有用，但您希望特定应用程序用户拥有始终显示实际数据的视图。您可以创建编辑表的新视图，然后为此视图定义数据编辑策略。该策略仍存在于基表上，但只要使用DBMS_REDACT.NONE function_type设置在视图上创建策略，当应用程序使用视图进行查询时，不会执行任何编校。

命名数据编辑策略表达式的集中管理

您可以创建一个可在多个表和视图的列中使用的命名策略表达式库。

通过命名策略表达式，您可以集中管理数据库中的所有策略表达式。

修改策略表达式时，更改将反映在使用该表达式的所有表列中。策略表达式优先于数据编辑策略中的expression设置。要创建策略表达式，必须使用DBMS_REDACT.CREATE_POLICY_EXPRESSION过程，并将策略表达式应用于列，使用DBMS_REDACT.APPLY_POLICY_EXPR_TO_COL 。此功能可根据不同的运行时条件灵活地编辑表或视图中的不同列。

例如，考虑涉及客户服务应用程序的用例。客户致电客户服务中心请求最近购买的退货。呼叫中心的1级支持代表必须首先验证订单ID，客户名称和客户地址，然后才能启动退货。在此过程中，1级支持代表无需查看客户的信用卡号。因此，当支持代表在呼叫中心应用程序中查询客户详细信息时，将对信用卡列进行编辑。当启动退货时，退货部门的销售代表可能需要查看信用卡号以处理退货。但是，销售代表无需查看信用卡的到期日期。因此，当销售代表在同一应用程序中查询客户详细信息时，信用卡号可见，但到期日期将被编辑。

在此用例中，必须根据登录用户的身份以不同方式编辑客户详细信息表中的不同列。Oracle Data Redaction通过使用命名的数据编辑策略表达式简化了此用例的实现。通过此类策略表达式，您可以在同一表或视图中的不同列上定义和关联不同的策略表达式。此外，您可以集中管理数据库中的命名策略表达式。对命名策略表达式所做的任何更新都会立即传播到所有关联的表或视图列。
