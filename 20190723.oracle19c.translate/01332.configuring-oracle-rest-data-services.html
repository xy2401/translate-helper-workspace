2.11.1使用包含的ORDS配置REST

通过设置shell环境变量来覆盖默认的ORDS配置。

  • 默认情况下，Oracle REST数据服务（ORDS）使用7080 - 7085范围内可用的任何端口。如果此范围内没有可用的端口，则ORDS退出并提示您设置RAT_ORDS_PORT环境变量。如果RAT_ORDS_PORT已经设置，然后ORDS使用在指定的端口RAT_ORDS_PORT环境变量。

  • 默认情况下，ORDS是使用管理员用户ordsadmin设置的。您可以通过在RAT_ORDSADMIN_USER环境变量中指定其他用户来覆盖它。

  • 根据Oracle ORAchk和Oracle EXAchk，ORDS作为名为ordsorachk或ordsexachk的nologin用户ordsexachk 。如果您使用已在运行的ORDS，则该用户与运行ORDS的用户相同。

  • 如果安装了Oracle Trace File Analyzer，则ORDS从TFA_HOME选择JAVA_HOME 。如果未安装Oracle跟踪文件分析器，则ORDS会选择默认的JAVA_HOME 。您需要使用JDK8。但是，您可以通过设置RAT_JAVAEXE环境变量来覆盖。

2.11.2使用现有ORDS安装配置REST

  1. 要将orachk.jar文件添加到现有的ords.war文件中：
    ./orachk -ordssetup ords_war_dir -configdir config_dir ./exachk -ordssetup ords_war_dir -configdir config_dir

    哪里，

    ords_war_dir是包含ords.war文件的目录

    config_dir是一个可选目录，您可以指定该目录来存储ORDS配置文件。如果未指定可选目录，则配置文件将存储在orda_war_dir目录中。

    运行-ordssetup命令后停止并重新启动ORDS：

    • 将orachk.jar文件添加到现有的ords.war文件中

    • 将用户ordsadmin添加到ords.war文件，并将ORAchk admin权限授予ordsadmin

  2. 要启动Oracle ORAchk或Oracle EXAchk守护程序：
    ./orachk -d start -ords ords_war_dir ./exachk -d start -ords ords_war_dir
完成后，打开ords_war_dir /log/ords_setup.log文件以查看REST URL详细信息。
