配置Oracle自动存储管理存储

确定存储要求和Oracle ASM磁盘组选项。

  • 确定Oracle自动存储管理的存储要求
    要确定使用Oracle ASM的存储要求，必须确定所需的设备数量和可用磁盘空间量。
  • Oracle Clusterware存储空间要求
    使用此信息可确定基于冗余类型的最小磁盘数和最小磁盘空间要求，以便为各种Oracle Cluster部署安装Oracle Clusterware文件。
  • 关于Grid Infrastructure Management Repository
    每个Oracle域服务集群都包含一个Grid Infrastructure Management Repository（GIMR），但GIMR配置对于Oracle Standalone Cluster是可选的。
  • 使用现有Oracle ASM磁盘组
    使用Oracle Enterprise Manager Cloud Control或Oracle ASM命令行工具（ asmcmd ）标识现有磁盘组，并确定磁盘组中是否有足够的可用空间。
  • 关于升级现有Oracle自动存储管理实例
    Oracle自动存储管理（Oracle ASM）升级在Oracle Grid Infrastructure升级期间执行。
  • 选择要与Oracle ASM磁盘组一起使用的磁盘
    如果您确定系统上不存在合适的磁盘组，请安装或识别要添加到新磁盘组的相应磁盘设备。
  • 指定Oracle ASM磁盘发现字符串
    初始化Oracle ASM实例时，Oracle ASM会发现并检查您在ASM_DISKSTRING初始化参数中使用值指定的路径中的所有磁盘的内容。
  • 在NAS设备上创建文件以与Oracle自动存储管理一起使用
    如果您拥有经过认证的NAS存储设备，则可以在NFS安装目录中创建零填充文件，并将这些文件用作Oracle ASM磁盘组中的磁盘设备。
