151 DBMS_SHARED_POOL

DBMS_SHARED_POOL包提供对共享池的访问，共享池是存储游标和PL / SQL对象的共享内存区域。DBMS_SHARED_POOL使您可以显示共享池中对象的大小，并标记它们是为了保持还是不保留，以减少内存碎片。

本章包含以下主题：

  • 概观

  • 操作说明

  • DBMS_SHARED_POOL子程序的摘要

151.1 DBMS_SHARED_POOL概述

加载大型PL / SQL对象时，此处提供的过程可能很有用。加载大型PL / SQL对象时，用户响应时间会受到影响，因为需要从共享池中老化的大量较小对象才能腾出空间（由于内存碎片）。在某些情况下，可能没有足够的内存来加载大对象。

DBMS_SHARED_POOL对于频繁执行的触发器也很有用。您可能希望将已编译的触发器保留在共享池中的常用表上。

此外， DBMS_SHARED_POOL支持序列。当序列从共享池中老化时，序列号将丢失。DBMS_SHARED_POOL对于将序列保留在共享池中非常有用，从而防止丢失序列号。

151.2 DBMS_SHARED_POOL操作说明

要创建DBMS_SHARED_POOL，请运行DBMSPOOL.SQL脚本。运行DBMSPOOL.SQL后，将自动执行PRVTPOOL.PLB脚本。这些脚本不是作为标准数据库创建的一部分运行的。

151.3.1 ABORTED_REQUEST_THRESHOLD程序

此过程设置共享池的中止请求阈值。

句法

DBMS_SHARED_POOL.ABORTED_REQUEST_THRESHOLD（threshold_size NUMBER）;

参数

表151-2 ABORTED_REQUEST_THRESHOLD过程参数

参数 描述

threshold_size

请求的大小（以字节为单位），该请求不会尝试释放共享池中的未固定（非“非保持”）内存。threshold_size的范围是5000到2 GB（含）。

例外

如果阈值不在有效范围内，则会引发异常。

使用说明

通常，如果在空闲列表中无法满足请求，则RDBMS会尝试通过从LRU列表中释放对象并定期检查以查看是否可以满足请求来回收内存。完成此步骤后，RDBMS执行了近似等效的' ALTER SYSTEM FLUSH SHARED_POOL '。

因为这会影响系统上的所有用户，所以此过程thresh_hold对该进程的影响“本地化”，从而无法找到大小超过thresh_hold大小的共享池内存。此用户在不尝试搜索LRU列表的情况下获得“内存不足”错误。

151.3.2保持程序

此过程将对象保留在共享池中。一旦对象保留在共享池中，它就不会从池中老化。这对于经常使用的大型对象可能很有用。当大对象进入共享池时，可能需要对几个对象进行老化以创建足够大的连续区域。

句法

DBMS_SHARED_POOL.KEEP（名称VARCHAR2，标志CHAR DEFAULT'P'）;
DBMS_SHARED_POOL.KEEP（schema VARCHAR2，objname VARCHAR2，namespace NUMBER，堆NUMBER，edition_name VARCHAR2 DEFAULT NULL）;
DBMS_SHARED_POOL.KEEP（散列VARCHAR2，命名空间NUMBER，堆NUMBER）;

参数

表151-3 KEEP过程参数

参数 描述

名称

要保持的对象的名称。

旗

一个字符串，指示要保留名称的对象类型。字符串不区分大小写。

此参数是可选的。

如果未指定参数，则包假定第一个参数是包/过程/函数的名称，并将解析该名称。

模式

用户名或对象所属的架构。

objname表

要保留的对象的名称。

命名空间

一个数字，指示必须在其中搜索对象的库高速缓存命名空间。

堆

要保持的堆积。例如，如果要保留堆0和堆6。

 
EDITION_NAME

表示目标对象所在的版本的名称。此参数是可选的。

哈希

对象的16字节哈希值。

例外

如果找不到命名对象，则会引发异常。

使用说明

有两种对象：

  • 由name指定的PL / SQL对象，触发器，序列和类型

  • SQL游标对象，由两部分编号指定（表示共享池中的位置）。

例如：

DBMS_SHARED_POOL.KEEP（ 'scott.hispackage'）

这保留了由SCOTT拥有的包HISPACKAGE 。 PL / SQL对象的名称遵循命名对象的SQL规则（例如，允许使用分隔标识符和多字节名称）。游标可以由DBMS_SHARED_POOL保存。 KEEP('0034CDFF, 20348871','C') 0034CDFF作为ADDRESS和20348871的HASH_VALUE 。请注意，完整的十六进制地址必须在前8个字符中。

151.3.3 MARKHOT程序

此过程将库高速缓存对象标记为热对象。

句法

DBMS_SHARED_POOL.MARKHOT（schema VARCHAR2，objname VARCHAR2，namespace NUMBER DEFAULT 1，global BOOLEAN DEFAULT TRUE，edition_name VARCHAR2 DEFAULT NULL）; DBMS_SHARED_POOL.MARKHOT（散列VARCHAR2，命名空间NUMBER DEFAULT 1，全局BOOLEAN DEFAULT TRUE）;

参数

表151-4 MARKHOT过程参数

参数 描述

模式

用户名或对象所属的架构

objname表

对象的名称

命名空间

指示要在其中搜索对象的库高速缓存命名空间的数字。视图，如USER_OBJECTS和DBA_OBJECTS ，反映名称空间的数列，如大多数字典表，如OBJ$ 。

全球

如果为TRUE （默认值），则在所有Oracle RAC实例上将对象标记为hot

哈希

对象的16字节哈希值

EDITION_NAME

表示目标对象所在的版本的名称。

此参数是可选的。

例外

ORA-06502 ：如果由于输入错误而无法找到命名对象，则引发异常

ORA-04043 ：如果找不到命名对象（错误的命名空间或散列输入），则引发异常

使用说明

如果包或类型的规范标记为hot或unhot，则相应的包或类型主体将隐式标记为hot或unhot。

用户可以检查列V$DB_OBJECT_CACHE.PROPERTY ，以查看该对象是否已标记为热。PROPERTY的值是：
  • HOTCOPYnnn - 一个整数标识符为'nnn'的热拷贝对象。例如， HOTCOPY5 ， HOTCOPY94和HOTCOPY125 。
  • HOTCOPY - 如上所述，但标识符未知
  • HOT -The“根” KGL对象已被标记为热
  • NULL - 一个普通对象

151.3.4清洗程序

此过程将清除指定对象或对象的指定堆。

句法

DBMS_SHARED_POOL.PURGE（名称VARCHAR2，标记CHAR DEFAULT'P'，堆积NUMBER DEFAULT 1）; DBMS_SHARED_POOL.PURGE（schema VARCHAR2，objname VARCHAR2，namespace NUMBER，堆NUMBER，edition_name VARCHAR2 DEFAULT NULL）; DBMS_SHARED_POOL.PURGE（散列VARCHAR2，命名空间NUMBER，堆NUMBER）;

参数

表151-5 PURGE过程参数

参数 描述

名称

要清除的对象的名称。

此标识符的值是v$sqlarea视图中address和hash_value列的串联。这由SIZES程序显示。

目前，可能不会清除TABLE和VIEW对象。

旗

（可选）如果未指定，则程序包假定第一个参数是程序包/过程/函数的名称并解析名称。

设置为“P”或“p”以完全指定输入是包/过程/函数的名称。

设置为“T”或“t”以指定输入是类型的名称。

设置为“R”或“r”以指定输入是触发器的名称。

设置为“Q”或“q”以指定输入是序列的名称。

如果第一个参数是游标地址和哈希值，则该参数应设置为除“P”或“p”或“Q”或“q”或“R”或“r”或“T”之外的任何字符。或't'。

堆

堆积被清除。例如，如果要清除堆0和堆6：

1 << 0 | 1 << 6 =>十六进制0x41 =>十进制65，因此指定heaps => 65。默认值为1 ，即堆0，表示将清除整个对象

模式

用户名或对象所属的架构

objname表

要清除的对象的名称

命名空间

参数是一个数字，指示要在其中搜索对象的库高速缓存命名空间

哈希

对象的16字节哈希值

EDITION_NAME

目标对象所在的版本的名称。

此参数是可选的。

例外

ORA-6570 ：如果找不到命名对象，则引发异常

ORA-6570 ：物体不能被清除，标记为永久保存

使用说明

PURGE支持KEEP程序支持的所有对象。

151.3.5尺寸程序

此过程显示shared_pool中的对象大于指定的大小。还给出了对象的名称，可以将其用作KEEP或UNKEEP调用的参数。

句法

DBMS_SHARED_POOL.SIZES（minsize NUMBER）;

参数

表151-6 SIZES过程参数

参数 描述

MINSIZE

对象必须占用共享池的大小（以千字节为单位），以便显示它。

使用说明

在使用此过程之前发出SQLDBA或SQLPLUS 'SET SERVEROUTPUT ON SIZE XXXXX'命令，以便显示结果。
