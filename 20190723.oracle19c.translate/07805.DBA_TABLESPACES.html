6.49 DBA_TABLESPACES

DBA_TABLESPACES描述数据库中的所有表空间。

相关视图

USER_TABLESPACES描述了当前用户可访问的表空间。此视图不显示PLUGGED_IN列。

EXTENT_MANAGEMENT

VARCHAR2（10）

指示表空间中的扩展区是字典管理（ DICTIONARY ）还是本地管理（ LOCAL ）

ALLOCATION_TYPE

VARCHAR2（9）

对表空间有效的扩展区分配类型：

  • 系统

插入

VARCHAR2（3）

指示表空间是否已插入（ YES ）或否（ NO ）

SEGMENT_SPACE_MANAGEMENT

VARCHAR2（6）

指示表空间中的空闲和已用段空间是使用空闲列表（ MANUAL ）还是位图（ AUTO ）进行管理

DEF_TAB_COMPRESSION

VARCHAR2（8）

指示是否启用（ ENABLED ）默认表压缩（ DISABLED ）

注意：启用默认表压缩表示除非另行指定，否则将在启用表压缩的情况下创建表空间中的所有表。

保留

VARCHAR2（11）

撤消表空间保留：

  • GUARANTEE - 表空间是一个撤消表空间，其中RETENTION指定为GUARANTEE

    GUARANTEE的RETENTION值表示撤销表空间中所有还原段中的未过期撤消应保留，即使这意味着需要在这些段中生成撤消的前向操作失败。

  • NOGUARANTEE - 表空间是一个撤消表空间，其中RETENTION指定为NOGUARANTEE

  • NOT APPLY - 表空间不是撤消表空间

BIGFILE

VARCHAR2（3）

指示表空间是bigfile表空间（ YES ）还是smallfile表空间（ NO ）

PREDICATE_EVALUATION

VARCHAR2（7）

ENCRYPTED

指示表空间是否加密（ YES ）或否（ NO ）

COMPRESS_FOR

VARCHAR2（30）

什么样的操作的默认压缩：

  • 存档高

“ QUERY LOW ，“ QUERY HIGH ，“ ARCHIVE LOW ”和“ ARCHIVE HIGH值与“混合列压缩”相关联，“混合列压缩”是Oracle数据库企业版的一项功能，它依赖于底层存储系统。有关更多信息，请参阅Oracle数据库概念

DEF_INMEMORY

VARCHAR2（8）

指示此表空间中的表是否默认启用（ ENABLED ）或禁用（ DISABLED ）内存中列存储（IM列存储）

DEF_INMEMORY_PRIORITY

VARCHAR2（8）

指示此表空间的内存中列存储（IM列存储）填充的默认优先级。可能的值：

  • 低

  • 介质

  • 高

  • 危急

  • 没有

  • 空值

DEF_INMEMORY_DISTRIBUTE

VARCHAR2（15）

指示默认情况下如何在Oracle Real Application Clusters（Oracle RACE）环境中为此表空间分发IM列存储：

  • 汽车

  • 通过ROWID RANGE

  • 通过分区

  • 通过提交

DEF_INMEMORY_COMPRESSION

VARCHAR2（17）

指示此表空间的IM列存储的默认压缩级别：

  • 没有MEMCOMPRESS

  • 对于DML

  • FOR QUERY [LOW |高]

  • 容量[LOW |高]

  • 空值

DEF_INMEMORY_DUPLICATE

VARCHAR2（13）

表示Oracle RAC环境中IM列存储的重复设置：

  • 没有重复

  • 重复

  • 全部重复

共享

VARCHAR2（12）

表空间类型：

  • SHARED ：对于共享表空间

  • LOCAL_ON_LEAF ：用于叶（只读）实例的本地临时表空间

  • LOCAL_ON_ALL ：用于所有实例类型的本地临时表空间

DEF_INDEX_COMPRESSION

VARCHAR2（8）

指示是否启用（ ENABLED ）默认索引压缩（ DISABLED ）

注意：启用默认索引压缩表示除非另行指定，否则将在启用索引压缩的情况下创建表空间中的所有索引。

INDEX_COMPRESS_FOR

VARCHAR2（13）

有效值为：

  • 高级低

  • 高级

  • 空值

不允许其他值。

DEF_CELLMEMORY

VARCHAR2（14）

这指定了CELLMEMORY属性的默认值，即表空间中创建的表将继承，除非明确地重写该行为

此列适用于Oracle Exadata。

DEF_INMEMORY_SERVICE

VARCHAR2（12）

指示默认情况下此表空间在各种实例上如何填充IM列存储。可能的值是：

  • DEFAULT ：在使用PARALLEL_INSTANCE_GROUP初始化参数指定的所有实例上填充数据。如果未设置该参数，则会在所有实例上填充数据。这是默认值。

  • NONE ：未在任何实例上填充数据。

  • ALL ：无论PARALLEL_INSTANCE_GROUP初始化参数的值如何，都会在所有实例上填充数据。

  • USER_DEFINED ：仅在用户指定的服务处于活动状态的实例上填充数据。与此对应的服务名称存储在DEF_INMEMORY_SERVICE_NAME列中。

DEF_INMEMORY_SERVICE_NAME

VARCHAR2（1000）

指示默认情况下应为此表空间填充IM列存储的服务的服务名称。仅当相应的DEF_INMEMORY_SERVICE为USER_DEFINED时，此列才具有值。在所有其他情况下，此列为空。

LOST_WRITE_PROTECT

VARCHAR2（7）

表空间丢失的写保护设置。可能的值：

  • ENABLED ：表示正在收集丢失的写入数据

  • OFF ：表示未收集丢失的写入数据

  • SUSPEND ：表示当前未收集丢失的写入数据，但可以在以后启用。文件处于ENABLED时收集的丢失写入数据仍保留在丢失的写入数据库中，但未进行检查或更新。

如果为表空间启用了丢失写保护，则会为该表空间的所有数据文件启用它，包括稍后添加的数据文件。

如果为单个数据文件启用了丢失的写保护，则不必为同一表空间中的另一个数据文件启用它。

您可以通过查询检查数据文件丢失的写保护状态LOST_WRITE_PROTECT列DBA_DATA_FILES视图。
