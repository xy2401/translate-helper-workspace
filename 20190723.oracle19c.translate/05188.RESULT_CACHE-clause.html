13.56 RESULT_CACHE条款

表示将功能结果存储到服务器结果缓存中。

RESULT_CACHE子句可以出现在以下SQL语句中：

  • 创建功能声明

  • CREATE TYPE BODY声明

句法

result_cache_clause :: =


说明result_cache_clause.eps的描述

语义

result_cache_clause

result_cache_clause只能在函数中出现一次。

RESULT_CACHE

要使函数结果缓存，请在函数定义中包含RESULT_CACHE子句。如果在定义函数之前声明该函数，则还必须在函数声明中包含RESULT_CACHE选项。

对RESULT_CACHE的限制

  • 具有OUT或IN OUT参数的函数不允许使用RESULT_CACHE 。

  • 对于具有（或包含）这些类型的IN或RETURN参数的函数，不允许使用RESULT_CACHE ：

    • BLOB

    • CLOB

    • NCLOB

    • REF CURSOR

    • 采集

    • 宾语

    • 包含不受支持的返回类型的记录或PL / SQL集合

  • 在匿名块中的函数不允许RESULT_CACHE 。

  • 在管道化表函数和嵌套函数上不允许使用RESULT_CACHE 。

依靠

指定函数结果所依赖的数据源。每个data_source都是数据库表或视图的名称。

注意：

  • 不推荐使用此子句。从Oracle Database 12c开始，数据库会检测在结果缓存函数运行时查询的所有数据源，并且RELIES_ON子句不执行任何操作。

  • 您不能在匿名块中声明的函数中使用RELIES_ON子句。
