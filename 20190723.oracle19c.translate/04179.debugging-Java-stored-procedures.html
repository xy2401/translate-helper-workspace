6.4调试Java存储过程

Oracle数据库提供了用于调试Java存储过程的Java调试线协议（JDWP）接口。Java Development Kit（JDK）1.3及更高版本支持JDWP。

以下是JDWP接口支持的一些功能：

  • 倾听连接

  • 调试时更改变量的值

  • 评估任意Java表达式，包括方法评估

  • 在一行或一个方法中设置或清除断点

  • 单步执行代码

  • 设置或清除字段访问或修改观察点

注意：

Oracle JDeveloper提供了与这些调试功能的用户友好集成。其他独立的集成开发环境（IDE）供应商也可以将自己的调试器与Oracle数据库集成。

本节讨论以下主题：

  • 调试Java存储过程的先决条件

  • 使用jdb调试器调试Java存储过程

  • 使用JDeveloper调试Java存储过程

6.4.1调试Java存储过程的先决条件

在调试Java存储过程之前，请确保满足以下先决条件：

  • 必须将Java代码部署到数据库，并且可以选择使用调试信息进行编译。

  • 您的数据库用户帐户必须具有以下权限：

    • DEBUG CONNECT SESSION权限

    • DEBUG CONNECT ANY特权

    • DEBUG CONNECT ON USER <user>权限

    • 要调试的存储过程的DEBUG对象特权

  • 您必须以下列方式将jdwp特权添加到访问控制列表（ACL）：

    SQL> begin 2 DBMS_NETWORK_ACL_ADMIN.APPEND_HOST_ACE（3 host => <host_name>，4 ace => xs $ ace_type（privilege_list => xs $ name_list（'jdwp'），5 principal_name => <user_name>，6 principal_type => xs_acl。 ptype_db））; 7结束; 8 /

6.4.2使用jdb调试器调试Java存储过程

可以使用-listen <port>命令启动jdb会话。如果以这种方式启动会话，则jdb将使用标准连接器等待正在运行的虚拟机（VM）在指定端口进行连接。

注意：

在调试Java存储过程时， jdb无法启动JVM会话，只能等待VM连接。

执行以下步骤以调试在Oracle JVM中运行的Java程序：

  1. 在debugging terminal中执行以下命令：

    jdb -listen 4000

    下图显示了启动jdb会话的调试终端：


    说明start_jdb.png的描述

  2. 使用SQL客户端等Oracle客户端发出连接调试器的命令。您可以通过以下两种方式进行连接：

    • 从执行Java存储过程的同一会话发出调试器连接命令。例如，如果您使用的是SQL * Plus，则发出以下命令以打开与JDWP会话的指定计算机和端口的TCP / IP连接：

      EXEC DBMS_DEBUG_JDWP.CONNECT_TCP（<host_ip>，<port>）

      下图显示了运行连接调试终端命令的客户端：


      说明debug_1.png的说明

      在调试器接受连接后，在调试器会话中发出断点并在Oracle客户端中调用Java存储过程。调试器现在将在您指定的第一个断点处暂停。

    • 在另一个会话中发出debugger connection命令并指定另外两个参数，如以下示例所示：

      EXEC DBMS_DEBUG_JDWP.CONNECT_TCP（<host_ip>，<port>，<session_id>，<session_serial>）

      在上面的命令中， session_id和session_serial标识数据库会话，其中执行Java存储过程，用户希望将其连接到调试器。要将另一个会话连接到调试器，用户必须对该会话的登录用户具有DEBUG CONNECT用户权限，或者具有DEBUG CONNECT ANY系统权限。

  3. 成功建立连接后，可以使用以下语法在调试终端中添加断点：

    停在<ClassName>：<LineNumber>

    下图显示了如何向调试终端添加断点：


    addbreakpoint.png插图的描述

  4. 在步骤2中使用的Oracle客户端中，按以下方式调用Java程序的SQL包装器：

    调用<SQLWrapperName>。<MethodName>

  5. 以下是一些jdb命令，可用于调试调试终端中的代码：

    • 对于一步一步： step

    • 要检查变量值的值： print<ClassName>:<VariableName>

    • 继续： cont

    • 要明确断点： clear

6.4.3使用JDeveloper调试Java存储过程

您可以使用JDeveloper无缝地调试Java存储过程和PL / SQL程序。在本地调试PL / SQL程序和Java存储过程时，直接从JDeveloper调用启动调试。JDeveloper执行以下活动：

  1. 它会自动启动您要调试的程序（也称为debuggee）
  2. 它将调试器附加到该程序。

远程调试和本地调试PL / SQL程序和Java存储过程之间的主要区别在于如何启动调试会话。对于远程调试，您必须手动启动要使用Oracle客户端（如SQL * Plus），使用DBMS_SCHEDULER程序包创建的作业，OCI程序或触发器触发来调试的程序。然后，您必须建立从要调试的数据库程序（debuggee）到JDeveloper调试器的连接。启动调试对象并将JDeveloper调试器连接到它后，远程调试与本地调试非常相似。

注意：

您可以选择关闭JIT以获得更好的调试体验。
