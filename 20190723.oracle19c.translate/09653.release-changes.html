本版本中针对Oracle Data Guard概念和管理的更改

本前言列出了Oracle Data Guard概念和管理中的更改。

Oracle Database Release 19c中的更改

这些是Oracle Data Guard概念和 Oracle Database Release 19c 管理中的更改。

新功能

  • 通过自动将还原点从主数据库复制到备用数据库，可以简化将物理备用数据库闪回到主数据库上捕获的时间点的过程。请参阅将还原点从主节点复制到备用节点 。

  • 在主数据库上执行闪回或时间点恢复时，处于装入模式的备用数据库可以自动遵循在主数据库上执行的相同恢复过程。请参阅主RESETLOGS操作后自动闪回已安装的待机状态 。

  • 您可以在Active Data Guard备用数据库上同时启用Oracle Database In-Memory列存储和Data Guard多实例重做应用。

  • 可以在Active Data Guard备用实例上执行DML操作。在ADG备用数据库上运行无效的PL / SQL对象时，将自动重新编译该对象。请参阅在Active Data Guard备用数据库上执行DML操作以及在Active Data Guard备用数据库上 运行顶级PL / SQL操作 。

不受支持的功能

  • 不支持LOG_ARCHIVE_DEST_ n初始化参数的MAX_CONNECTIONS属性。

  • 不支持扩展数据类型支持（EDS）。现在，逻辑备用数据库或Oracle GoldenGate本身支持所有EDS支持的Oracle数据类型。

Oracle Database Release 18c，版本18.1中的更改

这些是Oracle Data Guard概念和 Oracle Database Release 18c，版本18.1中的更改管理中的更改。

  • 现在，在角色更改期间，Oracle Active Data Guard备用数据库上将维护数据库缓冲区高速缓存状态。请参阅涉及物理备用数据库的角色转换 。

  • 现在可以在Oracle Active Data Guard备用数据库上动态创建全局临时表。请参阅Active Data Guard实例上的全局临时表 。

  • 新的初始化参数ADG_ACCOUNT_INFO_TRACKING扩展了对用户帐户安全性的控制，以防止跨生产数据库和所有Oracle Active Data Guard备用数据库的登录攻击。请参见Oracle数据库参考 。

  • 新视图V$DATAGUARD_PROCESS （替换V$MANAGED_STANDBY ）提供了可以查询的信息，以验证重做是从主数据库传输并应用于备用数据库。请参阅创建物理备用任务7：验证物理备用数据库是否正常运行 。

  • 私有临时表（也称为本地临时表）的元数据可以存储在内存中。这允许在只读数据库上启用专用临时表，从而允许报告应用程序在Oracle Active Data Guard备用数据库上运行。请参阅Active Data Guard实例上的私有临时表 。

  • 数据库nologging已经扩展了两种新模式： Standby Nologging for Load Performance Standby Nologging for Data Availability 。这些模式为在Oracle Active Data Guard环境中使用提供了更好的支持，而不会显着增加重做生成的数量。请参阅启用适当的日志记录模式 。

  • 现在可以使用一个RMAN命令RECOVER STANDBY DATABASE在网络上刷新RECOVER STANDBY DATABASE 。请参阅使用一个命令滚动待机 。

  • 已对Data Guard代理对使用DBMS_ROLLING PL / SQL程序包执行的升级的支持进行了DBMS_ROLLING 。请参阅Data Guard Broker对DBMS_ROLLING升级的支持 。

  • 现在，多实例重做应用支持块更改跟踪。请参阅设置多实例重做应用 。
