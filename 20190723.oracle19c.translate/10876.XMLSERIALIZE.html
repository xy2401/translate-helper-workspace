XMLSERIALIZE

句法


插图描述xmlserialize.eps

目的

XMLSerialize创建包含value_expr内容的字符串或LOB。

  • 如果指定DOCUMENT ，则value_expr必须是有效的XML文档。

  • 如果指定CONTENT ，则value_expr不必是单根XML文档。但它必须是有效的XML内容。

  • 指定的datatype可以是字符串类型（ VARCHAR2或VARCHAR ，但不是NVARCHAR2 ）， BLOB或CLOB 。默认值为CLOB 。

  • 如果datatype为BLOB ，则可以指定ENCODING子句以在prolog中使用指定的编码。xml_encoding_spec是XML编码声明（ encoding="..." ）。

  • 指定VERSION子句以使用您在XML声明中提供的版本string_literal （ <?xml version =“...”......？> ）。

  • 指定NO INDENT以从输出中INDENT所有无关紧要的空格。指定INDENT SIZE = N ，其中N是整数，对于使用N空格的相对缩进进行漂亮打印的输出。如果N为0 ，则漂亮打印在每个元素后面插入一个换行符，将每个元素单独放在一行上，但省略输出中所有其他无关紧要的空格。如果没有SIZE规范存在INDENT ，则使用2空格缩进。如果省略此子句，那么行为（漂亮打印与否）是不确定的。

  • HIDE DEFAULTS和SHOW DEFAULTS仅适用于基于XML模式的数据。如果指定SHOW DEFAULTS并且输入数据缺少XML架构为其定义默认值的任何可选元素或属性，则这些元素或属性将以其默认值包含在输出中。如果指定HIDE DEFAULTS ，则输出中不包含此类元素或属性。HIDE DEFAULTS是默认行为。

例子

以下语句使用DUAL表来说明XMLSerialize的语法：

SELECT XMLSERIALIZE（CONTENT XMLTYPE（'<Owner> Grandco </ Owner>'））AS xmlserialize_doc FROM DUAL; XMLSERIALIZE_DOC ---------------- <所有者> Grandco </ Owner>
