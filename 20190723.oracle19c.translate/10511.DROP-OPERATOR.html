DROP OPERATOR

目的

使用DROP OPERATOR语句删除用户定义的运算符。

也可以看看：

  • 创建操作员和更改操作员以获取有关创建和修改操作员的信息

  • “ 用户定义的运算符 ”和“ Oracle数据库数据盒带开发人员指南” ，以获取有关运算符的更多信息

  • 有关删除用户定义的索引类型的运算符的信息，请使用ALTER INDEXTYPE

先决条件

运算符必须位于您的模式中，或者您必须具有DROP ANY OPERATOR系统特权。

句法

drop_operator :: =


说明drop_operator.eps的描述

语义

模式

指定包含运算符的模式。如果省略schema ，则Oracle数据库假定运算符在您自己的模式中。

操作者

指定要删除的运算符的名称。

力

指定FORCE以删除运算符，即使它当前正被一个或多个模式对象引用，例如索引类型，包，函数，过程等。数据库标记任何此类依赖对象INVALID 。如果没有FORCE ，如果任何模式对象引用它，则不能删除运算符。

例子

删除用户定义的运算符：示例

以下语句删除运算符eq_op ：

DROP OPERATOR eq_op;

由于未指定FORCE子句，如果indextype引用此运算符的任何绑定，则此操作将失败。
