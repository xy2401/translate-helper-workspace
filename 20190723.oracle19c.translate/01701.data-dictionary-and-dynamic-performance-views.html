6数据字典和动态性能视图

每个Oracle数据库的中央只读参考表和视图集合统称为数据字典 。动态性能视图是在数据库打开和使用时不断更新的特殊视图。

本章包含以下部分：

  • 数据字典概述

  • 动态性能视图概述

  • 数据库对象元数据

数据字典概述

Oracle数据库的一个重要部分是它的数据字典，它是一组只读表，提供有关数据库的管理元数据。

数据字典包含以下信息：

  • 数据库中每个模式对象的定义，包括列的缺省值和完整性约束信息

  • 为架构对象分配和当前使用的空间量

  • Oracle数据库用户的名称，授予用户的权限和角色以及与用户相关的审核信息

数据字典是每个Oracle数据库的数据管理的核心部分。例如，数据库执行以下操作：

  • 访问数据字典以查找有关用户，架构对象和存储结构的信息

  • 每次发出DDL语句时修改数据字典

由于Oracle数据库将数据字典数据存储在表中，就像其他数据一样，用户可以使用SQL 查询数据。例如，用户可以运行SELECT语句来确定它们的权限，哪些表存在于它们的模式中，哪些列在这些表中，是否在这些列上构建索引，等等。

数据字典的内容

数据字典由基表和视图组成。

这些对象定义如下：

  • 基表

    这些存储有关数据库的信息。只有Oracle数据库应该写入并读取这些表。用户很少直接访问基表，因为它们已标准化，大多数数据以隐藏格式存储。

  • 查看

    它们使用连接和WHERE子句将基表数据解码为有用信息，例如用户名或表名，以简化信息。视图包含数据字典中所有对象的名称和描述。某些视图可供所有数据库用户访问，而其他视图仅供管理员使用。

使用前缀DBA_的视图

带有前缀DBA_视图显示整个数据库中的所有相关信息。DBA_视图仅供管理员使用。

以下示例查询显示有关数据库中所有对象的信息：

选择所有者，OBJECT_NAME，OBJECT_TYPE来自DBA_OBJECTS ORDER by OWNER，OBJECT_NAME;

包含前缀ALL_的视图

带有前缀ALL_视图指的是用户对数据库的整体视角。

除了用户拥有的架构对象之外，这些视图还通过公共或显式授予的权限和角色返回有关用户可以访问的架构对象的信息。

例如，以下查询返回有关您有权访问的所有对象的信息：

选择所有者，OBJECT_NAME，OBJECT_TYPE来自ALL_OBJECTS ORDER by OWNER，OBJECT_NAME;

由于ALL_视图遵循当前启用的角色集，因此查询结果取决于启用的角色，如以下示例所示：

SQL> SET ROLE ALL;角色集。SQL> SELECT COUNT（*）FROM ALL_OBJECTS; COUNT（*）---------- 68295 SQL> SET ROLE NONE;角色集。SQL> SELECT COUNT（*）FROM ALL_OBJECTS; COUNT（*）---------- 53771

应用程序开发人员应该了解在存储过程中使用ALL_视图时角色的影响，默认情况下不会启用角色。

使用前缀USER_的视图

典型数据库用户最感兴趣的USER_是前缀为USER_ 。

这些观点：

  • 请参阅数据库中用户的私有环境，包括用户创建的模式对象的元数据，用户提供的授权等等

  • 仅显示与用户相关的行，返回ALL_视图中的信息子集

  • 列与其他视图相同，只是隐含了列OWNER

  • 为方便起见，可以缩写PUBLIC同义词

例如，以下查询返回架构中包含的所有对象：

SELECT OBJECT_NAME，OBJECT_TYPE FROM USER_OBJECTS ORDER BY OBJECT_NAME;
双表

DUAL是数据字典中的一个小表，Oracle数据库和用户编写的程序可以引用它来保证已知结果。

当值必须仅返回一次（例如，当前日期和时间）时，双表非常有用。所有数据库用户都可以访问DUAL 。

DUAL表有一列名为DUMMY ，一行包含值X以下示例查询DUAL以执行算术运算：

SQL> SELECT（（3 * 4）+5）/ 3 FROM DUAL; （（3 * 4）+5）/ 3 ----------- 5.66666667

存储数据字典

数据字典基表是在任何Oracle数据库中创建的第一个对象。

数据库的所有数据字典表和视图都存储在SYSTEM表空间中。由于SYSTEM表空间在数据库打开时始终处于联机状态，因此数据库打开时数据字典始终可用。

Oracle数据库如何使用数据字典

Oracle数据库用户帐户SYS拥有数据字典的所有基表和用户可访问的视图。

在数据库操作期间，Oracle数据库读取数据字典以确定存在架构对象以及用户是否具有对它们的适当访问权限。Oracle数据库不断更新数据字典，以反映数据库结构，审计，授权和数据的变化。

例如，如果用户hr创建名为interns的表，则数据库会向数据字典中添加新行，以反映新表，列，段，范围以及hr对表的权限。下次查询字典视图时，此新信息可见。

数据字典的基表中的数据是Oracle数据库运行所必需的 。只有Oracle数据库才能编写或更改数据字典信息。没有Oracle数据库用户应该永远改变包含在行或模式对象SYS模式，因为这种活动可能损害数据的完整性 。安全管理员必须严格控制此中央帐户。

警告：

更改或操纵数据字典表中的数据会永久地和不利地影响数据库操作。

数据字典缓存

大部分数据字典信息都在数据字典高速缓存中，因为数据库不断要求信息验证用户访问并验证模式对象的状态。

缓存通常包含解析信息。描述表及其列的COMMENTS列不会缓存在字典高速缓存中，但可以缓存在数据库缓冲区高速缓存中 。

其他程序和数据字典

其他Oracle数据库产品可以引用现有视图并创建自己的其他数据字典表或视图。

Oracle建议编写引用数据字典的程序的应用程序开发人员使用公共同义词而不是基础表。同义词不太可能在版本之间发生变化。

动态性能视图概述

在整个操作过程中，Oracle数据库维护一组记录当前数据库活动的虚拟表。

这些视图是动态的，因为它们在数据库打开和使用时不断更新。视图有时称为V $视图，因为它们的名称以V$开头。

动态性能视图包含以下信息：

  • 系统和会话参数

  • 内存使用和分配

  • 文件状态（包括RMAN备份文件）

  • 工作和任务的进展

  • SQL执行

  • 统计和指标

动态性能视图具有以下主要用途：

  • Oracle Enterprise Manager使用视图来获取有关数据库的信息。

  • 管理员可以使用视图进行性能监视和调试。

动态性能视图的内容

动态性能视图称为固定视图，因为数据库管理员无法更改或删除它们。但是，数据库管理员可以查询和创建表的视图，并将这些视图的访问权限授予其他用户。

SYS拥有动态性能表，其名称以V_$开头。将在这些表上创建视图，然后在前缀为V$公共同义词上创建视图。例如， V$DATAFILE视图包含有关数据文件的信息。V$FIXED_TABLE视图包含有关所有动态性能表和视图的信息。

对于几乎每个V$视图，都存在相应的GV$视图。在Oracle Real Application Clusters（Oracle RAC）中，查询GV$视图会从所有限定数据库实例中检索V$视图信息。

使用Database Configuration Assistant（DBCA）创建数据库时，Oracle会自动创建数据字典。Oracle数据库自动运行catalog.sql脚本，该脚本包含动态性能视图的视图和公共同义词的定义。您必须运行catalog.sql才能创建这些视图和同义词。

存储动态性能视图

动态性能视图基于从数据库内存结构构建的虚拟表。

视图不是存储在数据库中的传统表。由于数据是动态更新的，因此无法保证视图的读取一致性。

由于动态性能视图不是真表，因此数据取决于数据库和数据库实例的状态 。例如，您可以在启动但未装入数据库时查询V$INSTANCE和V$BGPROCESS 。但是，在装入数据库之前，您无法查询V$DATAFILE 。

DBMS_METADATA包提供用于提取数据库对象的完整定义的接口。
