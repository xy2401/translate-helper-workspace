1.6 Oracle数据库中的Java编程

Oracle为企业应用程序开发人员提供了用于创建，部署和管理Java应用程序的端到端Java解决方案。整个解决方案包括客户端和服务器端编程接口，支持Java开发的工具以及与Oracle数据库集成的JVM。所有这些产品都完全兼容Java标准。本节包括以下主题：

  • Java数据库应用程序开发

  • Java编程环境用法

  • Java存储过程

  • PL / SQL集成和Oracle RDBMS功能

  • 开发工具

  • Internet协议版本6支持

1.6.1数据库应用程序开发中的Java

Java在数据库应用程序开发中最重要的特性是：

  • 使用Java SE API和JDBC设计数据绑定过程和函数。

  • 使用标准和第三方Java库扩展数据库的范围和功能。例如，使用数据库中的驱动程序访问第三方数据库并访问Hadoop / HDFS。

  • 为JDBC级别的对称数据访问提供Java2平台标准版（J2SE）应用程序的灵活分区。

  • 通过以下方式桥接SQL和Java2平台企业版（J2EE）世界：

    • 调用Web组件，例如JSP和servlet

    • 使用Web服务标注桥接SQL和Web服务

  • 使用Oracle JVM作为ERP集成中心。

  • 使缓存无效。

1.6.3 Java存储过程

Java的存储过程是在服务器上编写和部署的Java程序，从服务器运行，就像PL / SQL存储过程一样。您可以使用SQL * Plus等产品直接调用它，也可以使用触发器间接调用它。您可以从任何Oracle Net客户端访问它，例如OCI和PRO *，或JDBC或SQLJ。

此外，您可以使用Java开发功能强大的服务器端程序，这些程序可以独立于PL / SQL。 Oracle数据库提供标准Java编程语言的完整实现和完全兼容的JVM。

1.6.4 PL / SQL集成和Oracle RDBMS功能

您可以从PL / SQL从Java和Java程序调用现有的PL / SQL程序。该解决方案可保护和利用您的PL / SQL和Java代码，并开辟基于Java的Internet计算的优势和机遇。

Oracle数据库提供了两种不同的Java API，用于访问SQL数据，JDBC和SQLJ。这两个API都可以在客户端上使用，并且JDBC API也可以在服务器上使用。因此，您可以在客户端和服务器上部署应用程序。

1.6.4.1 JDBC驱动程序

JDBC是一种数据库访问协议，使您可以连接到数据库并运行SQL语句和查询到数据库。核心Java类库提供以下JDBC API： java.sql和javax.sql 。但是，JDBC旨在使供应商能够提供为特定数据库提供必要专业化的驱动程序。神谕提供以下不同的JDBC驱动程序：

司机 描述

JDBC瘦驱动程序

您可以使用JDBC Thin驱动程序编写访问Oracle SQL数据的纯Java应用程序和applet。JDBC Thin驱动程序特别适用于基于Web的应用程序和applet，因为您可以从Web页面动态下载它，类似于任何其他Java applet。

JDBC OCI驱动程序

JDBC OCI驱动程序访问特定于Oracle的本机代码，即非Java代码，以及客户端或中间层上的库，与JDBC Thin驱动程序相比提供了性能提升，但代价是显着更大的大小和客户端安装。

JDBC服务器端内部驱动程序

当Java代码在服务器上运行时，Oracle数据库使用服务器端内部驱动程序。它允许在服务器上的Oracle JVM中运行的Java应用程序使用JDBC访问本地定义的数据，即同一系统和同一进程中的数据。它提供了性能提升，因为它能够直接使用底层的Oracle RDBMS库，而不需要Java代码和SQL数据之间的中间网络连接的开销。通过在服务器上支持相同的Java-SQL接口，Oracle数据库不需要您在部署时重做代码。

1.6.4.2 SQLJ

Oracle与其他供应商合作，包括IBM，Tandem和Sybase，开发了一种在SQL程序中嵌入SQL语句的标准方法。这项工作产生了一个新标准ANSI x.3.135.10-1998，用于比JDBC更简单，更高效的编程API。用户将应用程序写入此更高级别的API，然后使用预处理器将程序转换为使用JDBC调用的标准Java源代码。在运行时，程序可以使用标准JDBC驱动程序与多供应商数据库进行通信。

SQLJ提供了一种简单但功能强大的方法来开发从Java访问数据库的客户端应用程序。您可以在存储过程和触发器中使用SQLJ。此外，您可以将SQLJ程序与JDBC结合使用。

SQLJ转换器是一个Java程序，它将Java源代码中的嵌入式SQL转换为纯粹的基于JDBC的Java代码。

1.6.6 Internet协议版本6支持

从Oracle Database 11g第2版开始，Oracle JVM支持数据库中Java代码的URL和机器名中的Internet协议版本6（IPv6）地址，这些地址将解析为IPv6地址。IPv6是由Internet工程任务组（IETF）设计的新的网络层协议，用于替换当前版本的Internet协议，Internet协议版本4（IPv4）。IPv6的主要优点是大地址空间，源自128位地址。IPv6还在诸如路由，网络自动配置，安全性，服务质量等方面改进了IPv4。

以下系统属性使您可以配置IPv6首选项：

java.net.preferIPv4Stack

如果操作系统上有IPv6，则底层本机套接字将是IPv6套接字。这使Java应用程序能够连接并接受来自IPv4和IPv6主机的连接。如果您的应用程序优先使用IPv4套接字，则可以将此属性设置为true 。如果将该属性设置为true ，则表示该应用程序无法与IPv6主机通信。

java.net.preferIPv6Addresses

即使操作系统上有IPv6，但出于向后兼容性原因，地址也会映射到IPv4。例如，依赖于仅访问IPv4服务的应用程序将受益于此类映射。如果要更改首选项以使用IPv4地址上的IPv6地址，则可以将java.net.preferIPv6Addresses属性设置为true 。这允许在预期应用程序连接到IPv6服务的环境中测试和部署应用程序。

注意：

在数据库初始化期间启用Java时，将加载IPv6支持所需的所有新System类。因此，如果您的应用程序没有包含在软件代码中的任何地址，那么您无需更改代码即可使用IPv6功能。
