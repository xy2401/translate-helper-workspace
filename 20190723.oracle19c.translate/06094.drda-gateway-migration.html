16从早期版本迁移

本节介绍如何从现有安装迁移到Oracle Database Gateway for DRDA的新实例。执行以下步骤以从现有版本迁移到适用于DRDA的新版Oracle Database Gateway：

  1. 安装新版本

  2. 使用新网关初始化参数文件

  3. 更新初始化参数

  4. 绑定网关包

  5. 安装或升级数据字典视图

安装新版本

如安装Oracle数据库网关DRDA中所述，在单独的目录中安装新版本的网关。

注意：

不要在先前存在的网关安装上安装网关。这破坏了现有的安装。

使用新网关初始化参数文件

此版本的DRDA数据库网关具有全新的架构。大多数先前的参数已经过时。您不应该使用旧的初始化文件作为基础并尝试修改它。相反，您应该使用作为安装的一部分生成的新初始化作为基础。有关参数的语法，请参阅初始化参数。

迁移到此版本的Database Gateway for DRDA的现有TG4DB2客户需要使用此版本的网关提供恢复用户的密码HS_FDS_RECOVERY_PWD 。

更新初始化参数

迁移到新版Oracle Database Gateway for DRDA的下一步包括更新初始化参数。

更改参数

DRDA_CONNECT_PARM的使用在此版本中已更改。有关参数的语法，请参阅初始化参数。

过时的参数

下列参数是过时的11g版本。从配置文件中删除它们：

  • 模式

  • 的server_path

  • DRDA_OVERRIDE_FROM_CODEPAGE

  • DRDA_OVERRIDE_TO_CODEPAGE

  • ERROR_LOGGING

  • 使用error_reporting

  • ERRORTAG

  • GATEWAY_SID

  • GROUP_BY_OFF

  • GTWDEBUG

  • INCREMENT_CURSORS

  • DRDA_CALLDESC_STMT

  • DRDA_CALLDESC_PROC

绑定网关包

升级到11g版本时，如果更改了以下任何初始化参数，则必须重新绑定网关包：

  • DRDA_DISABLE_CALL

  • DRDA_ISOLATION_LEVEL

  • DRDA_PACKAGE_COLLID

  • DRDA_PACKAGE_CONSTOKEN

  • DRDA_PACKAGE_NAME

  • DRDA_PACKAGE_OWNER

  • DRDA_PACKAGE_SECTIONS

安装或升级数据字典视图

为了正确运行网关，必须在任何将由网关访问的DB2系统上安装数据字典视图。如果要升级网关，则必须将数据字典视图替换为最新网关附带的视图。新视图是最新版本网关正常运行所必需的。它们包含必要的向后功能以与先前版本兼容。有关创建数据字典视图的更多信息，请参阅为DRDA配置Oracle数据库网关 。
