14.9 CREATE PACKAGE声明

CREATE PACKAGE语句创建或替换存储包的规范，存储包是数据库中作为一个单元存储的相关过程，函数和其他程序对象的封装集合。

包规范声明了这些对象。随后指定的包体定义了这些对象。

先决条件

要在架构中创建或替换包，您必须具有CREATE PROCEDURE系统特权。要在另一个用户的模式中创建或替换包，您必须具有CREATE ANY PROCEDURE系统特权。

要在Oracle数据库预编译器程序中嵌入CREATE PACKAGE语句，必须使用关键字END-EXEC后跟特定语言的嵌入式SQL语句终止符来终止该语句。

语义

create_package

或替换

如果包存在，则重新创建它，然后重新编译它。

在重新定义包之前被授予权限的用户仍然可以访问包而无需重新授予权限。

如果任何基于函数的索引依赖于包，则数据库会将索引标记为DISABLED 。

[版权所有|不可挽回的]

指定包是否是editioned或noneditioned对象如果editioning为架构对象类型启用PACKAGE在schema 。默认值： EDITIONABLE 。有关已编辑和未编辑的对象的信息，请参见“ Oracle数据库开发指南” 。

plsql_package_source

模式

包含该包的模式的名称。默认值：您的架构。

包裹名字

存储在数据库中的包。有关命名约定，请参阅“ 标识符 ” 。

package_item_list

定义包中的每个类型，并声明包中的每个游标和子程序。除了多态表函数之外，每个声明都必须在包体中具有相应的定义。相应声明和定义的标题必须逐字逐句匹配，白色空间除外。包多态表函数必须在与其实现包相同的包中声明。

对package_item_list的限制

PRAGMA AUTONOMOUS_TRANSACTION不能出现在这里。

例

示例14-19为emp_mgmt包创建规范

此语句创建emp_mgmt包的规范。 

创建或替换包装emp_mgmt AS FUNCTION hire（last_name VARCHAR2，job_id VARCHAR2，manager_id NUMBER，salary NUMBER，commission_pct NUMBER，department_id NUMBER）返回号码;功能create_dept（department_id NUMBER，location_id NUMBER）返回号码;程序remove_emp（employee_id NUMBER）;程序remove_dept（department_id NUMBER）; PROCEDURE increase_sal（employee_id NUMBER，salary_incr NUMBER）;程序increase_comm（employee_id NUMBER，comm_incr NUMBER）; no_comm EXCEPTION; no_sal EXCEPTION; END emp_mgmt;

emp_mgmt包的规范声明了这些公共程序对象：

  • 函数hire和create_dept

  • 过程remove_emp ， remove_dept ， increase_sal和increase_comm

  • no_comm和no_sal例外

有权访问该程序包的用户可以使用所有这些对象。创建包后，您可以开发调用任何这些公共过程或函数的应用程序，或者引发包的任何公共异常。

在调用此程序包的过程和函数之前，必须在程序包主体中定义这些过程和函数。有关创建emp_mgmt包主体的CREATE PACKAGE BODY语句的emp_mgmt ，请参阅“ CREATE PACKAGE BODY语句 ” 。
