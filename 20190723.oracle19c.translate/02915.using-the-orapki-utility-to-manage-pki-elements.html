orapki实用程序orapki管理公钥基础结构（PKI）元素，例如钱包和证书撤销列表。

orapki实用语法

orapki实用程序语法指定Oracle钱包，证书吊销列表或PKI数字证书。

orapki命令行实用程序的语法如下：

orapki module命令 - 参数值

在本规范中， module可以是wallet （Oracle钱包）， crl （证书撤销列表）或cert （PKI数字证书）。可用命令取决于您使用的module 。

例如，如果您正在使用wallet ，则可以使用add命令将证书或密钥add到钱包。以下示例将位于/private/lhale/cert.txt的用户证书添加到位于$ORACLE_HOME/wallet/ewallet.p12 ：

orapki wallet add -wallet $ ORACLE_HOME / wallet / ewallet.p12 -user_cert -cert /private/lhale/cert.txt

为测试目的创建签名证书

orapki实用程序提供了一种方便，轻量级的方法来创建用于测试目的的签名证书。

  • 要创建用于测试目的的签名证书，请使用以下命令：

    orapki cert create [-wallet wallet_location ] -request certificate_request_location -cert certificate_location -validity number_of_days [-summary]

此命令从证书请求创建签名证书。 -wallet参数指定包含用于对证书请求进行签名的用户证书和私钥的钱包。 -validity参数指定此证书有效的从当前日期开始的天数。此命令必须指定证书和证书请求。

查看证书

创建证书后，您可以使用orapki实用程序进行查看。

  • 要查看证书，请使用以下命令：

    orapki cert display -cert certificate_location [-summary | -完成]

使用此命令可以查看使用orapki创建的测试证书。您可以选择-summary或-complete ，这就决定了该命令将多少细节显示。如果选择-summary ，该命令将显示证书及其到期日期。如果选择-complete ，它将显示其他证书信息，包括序列号和公钥。

控制MD5和SHA-1证书使用

您可以使用sqlnet.ora文件来控制是否接受MD5和SHA-1签名证书。

要控制是否接受MD5和SHA-1签名证书，可以编辑sqlnet.ora文件以启用或禁用它们。
  1. 登录Oracle数据库所在的服务器。
  2. 编辑sqlnet.ora文件。

    默认情况下， sqlnet.ora文件位于$ORACLE_HOME/dbs目录中，或位于TNS_ADMIN环境变量设置的位置。

  3. 设置以下参数：
    • ACCEPT_MD5_CERTS控制MD5证书的使用。默认值为FALSE 。此参数替换ORACLE_SSL_ALLOW_MD5_CERT_SIGNATURES环境变量。
    • ACCEPT_SHA1_CERTS控制SHA-1证书的使用。默认值为TRUE 。 。

创建PKCS＃12钱包

您可以使用orapki实用程序创建PKCS＃12 Oracle钱包。

  • 要创建Oracle PKCS＃12钱包（ ewallet.p12 ），请使用orapki wallet create命令。 

    orapki钱包创建-wallet wallet_location [-pwd 密码 ]

如果在命令行上未指定密码，此命令将提示您输入并重新输入钱包密码。它在为-wallet指定的位置创建一个钱包。

注意：

出于安全原因，Oracle建议您不要在命令行中指定密码。您应该仅在提示时提供密码。

创建自动登录电子钱包

您可以使用orapki实用程序创建自动登录钱包。

  • 要创建一个自动登录钱包（ cwallet.sso ），它不需要密码来打开钱包，请使用orapki wallet create命令：

    orapki wallet创建-wallet wallet_location -auto_login_only

您可以在不使用密码的情况下修改或删除钱包。文件系统权限为此类自动登录钱包提供必要的安全性。

您无法将本地自动登录钱包移动到另一台计算机。它们必须在创建它们的主机上使用。

即使本地自动登录钱包不需要打开密码，您也必须提供相关PKCS＃12钱包的密码才能修改或删除钱包。对PKCS＃12钱包的任何更新也会更新相关的自动登录钱包。

创建与PKCS＃12钱包相关联的自动登录电子钱包

您可以创建与PKCS＃12钱包关联的自动登录钱包。

自动登录钱包不需要密码即可打开。

但是，您必须提供相关PKCS＃12钱包的密码才能修改或删除钱包。对PKCS＃12钱包的任何更新也会更新相关的自动登录钱包。

  • 要创建与PKCS＃12钱包（ ewallet.p12 ）关联的自动登录钱包（ cwallet.sso ），请使用以下命令：

    orapki钱包创建-wallet wallet_location -auto_login [-pwd 密码 ]

此命令创建一个启用了自动登录的钱包（ cwallet.sso ），并将其与PKCS＃12钱包（ ewallet.p12 ）相关联。如果在命令行中未指定密码，则该命令会提示您输入PKCS＃12钱包的密码。

如果wallet_location已包含PKCS＃12钱包，则会为其启用自动登录。您必须提供现有PKCS＃12钱包的密码才能为其启用自动登录。

如果wallet_location不包含PKCS＃12钱包，则创建新的PKCS＃12钱包。您必须为新PKCS＃12钱包指定密码。

如果要关闭PKCS＃12钱包的自动登录功能，请使用Oracle Wallet Manager。

创建一个本地的自动登录电子钱包和创建它的用户

orapki实用程序可以创建一个自动登录钱包，该钱包是创建它的用户的计算机本地的。

转换Oracle Wallet以使用AES256算法

默认情况下，具有ADMINISTER KEY MANAGEMENT或ALTER SYSTEM语句的Oracle钱包使用3DES加密。

您可以使用orapki convert命令将钱包转换为使用AES256算法，该算法比3DES算法更强大。请注意，如果您使用orapki而不是ADMINISTER KEY MANAGEMENT或ALTER SYSTEM语句创建了钱包，则默认情况下它使用AES256算法。

  • 要将钱包算法从3DES更改为AES256：

    orapki钱包转换-wallet wallet_location [-pwd 密码 ] [-compat_v12]

compat_v12设置执行从3DES到AES256的转换。

将证书申请添加到Oracle Wallet

您可以使用orapki实用程序向Oracle钱包添加证书和证书请求。

  • 要将证书请求添加到Oracle钱包，请使用orapki wallet add命令。 

    orapki wallet add -wallet wallet_location -dn user_dn -keySize 512 | 1024 | 2048

此命令为具有指定专有名称（ user_dn ）的用户的钱包添加证书请求。该请求还指定了所请求证书的密钥大小（512,1024或2048位）。要对请求进行签名，请使用export选项将其导出。

将可信证书添加到Oracle Wallet

您可以使用orapki实用程序将可信证书添加到Oracle钱包。

  • 要将可信证书添加到Oracle钱包，请使用以下命令：

    orapki wallet add -wallet wallet_location -trusted_cert -cert certificate_location

此命令将指定位置（ -cert certificate_location ）的受信任证书添加到钱包。在添加用户证书之前，必须在用户证书的证书链中添加所有可信证书，否则添加用户证书的命令将失败。

将根证书添加到Oracle Wallet

您可以使用orapki实用程序将根证书添加到Oracle钱包。

  • 要将根证书添加到Oracle钱包，请使用以下命令：

    orapki wallet add -wallet wallet_location -dn certificate_dn -keySize 512 | 1024 | 2048 -self_signed -validity number_of_days

此命令创建新的自签名（根）证书并将其添加到钱包。-validity参数（必需）指定此证书有效的从当前日期开始的天数。您可以为此根证书（ -keySize ）指定-keySize或2048位的密钥大小。

将PKCS＃11信息添加到Oracle Wallet

包含PKCS＃11信息的钱包可以像任何Oracle钱包一样使用。

私钥存储在硬件设备上。加密操作也在设备上执行。

  • 使用以下命令将PKCS＃11信息添加到钱包：

    orapki钱包p11_add -wallet wallet_location -p11_lib pkcs11Lib [-p11_tokenlabel tokenLabel ] [-p11_tokenpw tokenPassphrase ] [-p11_certlabel certLabel ] [-pwd 密码 ]

在本规范中：

  • wallet指定钱包位置。

  • p11_lib指定PKCS＃11库的路径。这包括库文件名。

  • p11_tokenlabel指定设备上使用的令牌或智能卡。当设备上有多个令牌时使用此选项。令牌标签使用供应商工具设置。

  • p11_tokenpw指定用于访问令牌的密码。令牌密码使用供应商工具设置。

  • p11_certlabel用于在令牌上指定证书标签。当令牌包含多个证书时使用此选项。证书标签使用供应商工具设置。

  • pwd用于指定钱包密码。

使用orapki从Oracle Wallets导出证书和证书请求

您可以使用orapki实用程序从Oracle钱包导出证书和证书请求。

  • 要从Oracle钱包导出证书，请使用以下命令：

    orapki wallet export -wallet wallet_location -dn certificate_dn -cert certificate_filename

此命令将具有主题的可分辨名称（ -dn ）的证书从钱包-cert到由-cert指定的-cert 。

要从Oracle钱包导出证书请求，请使用以下命令：

orapki wallet export -wallet wallet_location -dn certificate_request_dn -request certificate_request_filename

此命令将证书请求与主题的可分辨名称（ -dn ）从钱包导出到-request指定的文件。

示例F-1使用自签名证书创建钱包并导出证书

  1. 创建一个钱包。

    例如：

    c -wallet / private / user / orapki_use / root

    钱包是在/private/user/orapki_use/root位置创建的。

  2. 将自签名证书添加到钱包。

    orapki wallet add -wallet / private / user / orapki_use / root -dn'CN = root_test，C = US' -keysize 2048 -self_signed -validity 3650

    这将创建一个有效期为3650天的自签名证书。主题的可分辨名称是CN=root_test,C=US 。证书的密钥大小为2048位。

    orapki钱包显示-wallet / private / user / orapki_use / root

    这用于查看钱包中包含的证书。

    orapki wallet export -wallet / private / user / orapki_use / root -dn'CN = root_test，C = US'-cert /private/user/orapki_use/root/b64certificate.txt

    这b64certificate.txt自签名证书导出到文件b64certificate.txt 。请注意，使用的可分辨名称与步骤2中的相同。

示例：创建电子钱包和用户证书

orapki实用程序可以创建钱包和用户证书。

示例F-2说明了与创建用户证书相关的其他任务。

以下步骤说明了创建钱包，创建证书请求，导出证书请求，从测试请求创建签名证书，查看证书，向钱包添加可信证书以及向钱包添加用户证书。

示例F-2创建钱包和用户证书

  1. 创建启用了自动登录的钱包。

    例如：

    orapki钱包创建-wallet / private / user / orapki_use / server -auto_login

    这将在/private/user/orapki_use/server创建一个启用了自动登录的钱包。

  2. 将证书申请添加到钱包。

    orapki wallet add -wallet /private/user/orapki_use/server/ewallet.p12 -dn'CN = server_test，C = US' -keysize 2048

    这会将证书请求添加到已创建的钱包（ ewallet.p12 ）。主题的可分辨名称是CN=server_test,C=US 。指定的密钥大小为2048位。

  3. 将证书请求导出到文件。

    orapki wallet export -wallet / private / user / orapki_use / server -dn'CN = server_test，C = US'-request /private/user/orapki_use/server/creq.txt

    这creq.txt证书请求导出到指定的文件，在本例中为creq.txt 。

  4. 从请求中创建签名证书以进行测试。

    orapki cert create -wallet / private / user / orapki_use / root -request /private/user/orapki_use/server/creq.txt -cert /private/user/orapki_use/server/cert.txt-validity 3650

    这将创建一个有效期为3650天的证书cert.txt 。证书是根据上一步生成的证书请求创建的。

  5. 查看证书。

    orapki证书显示-cert /private/user/orapki_use/server/cert.txt-complete

    这将显示上一步中生成的证书。-complete选项使您可以显示其他证书信息，包括序列号和公钥。

  6. 将可信证书添加到钱包。

    orapki wallet add -wallet /private/user/orapki_use/server/ewallet.p12-trusted_cert -cert /private/user/orapki_use/root/b64certificate.txt

    这会将可信证书b64certificate.txt到ewallet.p12钱包中。在添加用户证书之前，必须在用户证书的证书链中添加所有可信证书。

  7. 将用户证书添加到钱包。

    orapki wallet add -wallet /private/user/orapki_use/server/ewallet.p12 -user_cert -cert /private/user/orapki_use/server/cert.txt

    此命令将用户证书cert.txt到ewallet.p12钱包。

orapki钱包添加

orapki wallet add命令将证书请求和证书添加到Oracle钱包。

句法

要添加证书请求：

orapki wallet add -wallet wallet_location -dn user_dn -keySize 512 | 1024 | 2048

  • wallet指定要添加证书请求的钱包的位置。

  • dn指定证书所有者的可分辨名称。

  • keySize指定证书的密钥大小。

  • 要对请求进行签名，请使用export选项将其导出。请参阅orapki钱包出口

要添加受信任的证书：

orapki wallet add -wallet wallet_location -trusted_cert -cert certificate_location

  • trusted_cert将使用-cert指定的位置的可信证书添加到钱包。

要添加根证书：

orapki wallet add -wallet wallet_location -dn certificate_dn -keySize 512 | 1024 | 2048 -self_signed -validity number_of_days

  • self_signed创建根证书。

  • validity是强制性的使用它指定从当前日期开始，该根证书有效的天数。

要添加用户证书：

orapki wallet add -wallet wallet_location -user_cert -cert certificate_location

  • user_cert将使用-cert参数指定的位置的用户证书添加到钱包。在将用户证书添加到钱包之前，必须添加构成证书链的所有可信证书。如果在添加用户证书之前钱包中未安装所有受信任的证书，则添加用户证书将失败。
