12.13栏目

句法

COL [UMN] [{ column | expr } [ 选项 ...]]

其中option表示以下子句之一：

ALI [AS] 别名 CLE [AR] ENTMAP {ON | OFF} FOLD_A [FTER] FOLD_B [EFORE] FOR [MAT] 格式 HEA [DING] 文本 JUS [TIFY] {L [EFT] | C [ENTER] | R [IGHT]} LIKE { expr | 别名 } NEWL [INE] NEW_V [ALUE] 变量 NOPRI [NT] | PRI [NT] NUL [L] 文本 OLD_V [ALUE] 变量 ON |关闭WRA [PPED] | WOR [D_WRAPPED] | TRU [NCATED]

指定显示给定的属性列，如

  • 列标题的文本

  • 列的对齐方式标题

  • NUMBER数据的格式

  • 包裹列数据

还列出单个列或所有列的当前显示属性。

输入COLUMN 后跟列或expr ，没有其他条款列出当前仅显示指定列或表达式的属性。输入COLUMN 没有条款列出所有当前列显示属性。

条款

{ column | expr }

标识SQL SELECT中的数据项（通常是列的名称） 列命令引用的命令。如果在COLUMN命令中使用表达式，则必须完全按照SELECT命令中的显示输入expr 。例如，如果SELECT命令中的表达式为+ b，则不能在COLUMN命令中使用b + a或（a + b）来引用SELECT命令中的表达式。

如果从不同的表中选择具有相同名称的列，则该列名称的COLUMN命令将应用于这两列。也就是说，LAST_NAME列的COLUMN命令适用于您在此会话中引用的所有名为LAST_NAME的列。COLUMN忽略SELECT命令中的表名前缀。此外，除非名称放在双引号中，否则将忽略空格。

要以不同方式格式化列，请为SELECT命令本身中的每个列分配唯一的别名（不要使用COLUMN命令的ALIAS子句），并为每个列的别名输入COLUMN命令。

ALI [AS] 别名

分配一个规定列的别名，可用于引用BREAK，COMPUTE和其他COLUMN命令中的列。 

明确]

重置的显示属性柱默认值。

要重置所有列的属性，请使用CLEAR COLUMNS命令。CLEAR COLUMNS也清除了ATTRIBUTE 该专栏。 

ENTMAP {ON | OFF}

允许为HTML输出中的选定列打开或关闭实体映射。通过此功能，您可以在数据列中包含HTML超链接，同时仍然可以在同一报表的其他列中映射实体。通过关闭包含HTML超链接的列的实体映射，可以在报告中正确解释HTML锚标记分隔符<，>，“和＆”。否则，它们将被它们各自的实体替换，＆lt;，＆gt;，＆quot;和＆amp;，阻止Web浏览器正确解释HTML。

列标题中的实体以及列中出现的任何COMPUTE标签或输出将根据列的ENTMAP值进行映射或不映射。

COLUMN ENTMAP的默认设置是MARKUP HTML ENTMAP选项的当前设置。

有关MARKUP HTML ENTMAP选项的更多信息，请参阅SET MARKUP选项 。 

FOLD_A [压脚提升]

在列标题之后和列中的每一行之后插入回车符。SQL * Plus不会在SELECT列表中的最后一列之后插入额外的回车符。除非PREFORMAT设置为ON，否则FOLD_A [FTER]在SET MARKUP HTML ON模式下不起作用。

FOLD_B [EFORE]

在列标题之前和列的每一行之前插入回车符。SQL * Plus不会在SELECT列表中的第一列之前插入额外的回车符。除非PREFORMAT设置为ON，否则FOLD_A [FTER]在SET MARKUP HTML ON模式下不起作用。

FOR [MAT] 格式

指定列的显示格式。格式规范必须是文本常量，例如A10或$ 9,999。

字符列

CHAR，NCHAR的默认宽度 ， VARCHAR2（VARCHAR）和NVARCHAR2 （NCHAR VARYING）列是数据库中列的宽度。SQL * Plus将这些数据类型格式化为左对齐格式。如果值不适合列宽，则SQL * Plus会根据SET WRAP的设置包装或截断字符串。

一个很长的 ，BLOB ，BFILE，CLOB ，NCLOB 或XMLType列的宽度默认为SET的值 LONGCHUNKSIZE或LONG LONG，无论哪个更小。

改变一个的宽度数据类型为n ，使用FORMAT A n 。（A代表字母。）如果指定的宽度小于列标题，则SQL * Plus会截断标题。

SQL * Plus在2000字节后截断或包装XMLType列。为避免这种情况，您需要为XMLType列设置显式的COLUMN格式。COLUMN格式每行最多可达60000。

日期列

SQL * Plus中未格式化的DATE列的默认宽度和格式是从NLS_DATE_FORMAT参数派生的。NLS_DATE_FORMAT设置由NLS地域参数确定。例如，NLS区域的默认格式America是DD-Mon-RR，默认宽度为A9。可以在数据库参数文件，环境变量或等效的特定于平台的机制中设置NLS参数。也可以使用ALTER SESSION命令为每个会话指定它们。有关DATE格式和NLS参数的详细信息，请参阅Oracle数据库SQL语言参考 。

你可以改变使用SQL SELECT语句中的SQL函数TO_CHAR的任何DATE列的格式。您可能还希望使用显式COLUMN FORMAT命令来调整列宽。

当您使用像TO_CHAR这样的SQL函数时，Oracle数据库会自动启用非常宽的列。默认列宽度还可能取决于SQL * Plus和数据库中使用的字符集。为了在使用多个字符集时最大化脚本可移植性，Oracle数据库建议对所选的每个列使用COLUMN FORMAT。

要将DATE列的宽度更改为n ，请使用带有FORMAT A n的COLUMN命令。如果指定的宽度小于列标题，则标题将被截断。

NUMBER列

对于数字列，COLUMN FORMAT设置优先于SET NUMFORMAT设置，这些设置优先于SET NUMWIDTH设置。

请参见SET NUMF [ORMAT]格式和SET NUM [WIDTH] {10 | n} 。

要更改NUMBER列的宽度，请使用 FORMAT后跟表12-1中指定的元素。

HEA [DING] 文字

定义一列标题。如果不使用HEADING子句，则列的标题默认为column或expr 。如果文字包含空格或标点字符，您必须用单引号或双引号括起来。每次出现的 HEADSEP字符（默认情况下，“|”）开始一个新行。

例如，

COLUMN LAST_NAME标题'员工|姓名'

会生成一个两行列标题。

参见SET HEADS [EP] {| c | ON | OFF}有关更改HEADSEP字符的信息。 

JUS [TIFY] {L [EFT] | C [ENTER] |对]}

对齐标题。如果不使用JUSTIFY子句，则NUMBER列的标题默认为RIGHT，其他列类型的标题默认为LEFT。

LIKE { expr | 别名 }

副本另一列或表达式的显示属性（其属性已使用另一个COLUMN命令定义）。LIKE仅复制当前COLUMN命令中未由另一个子句定义的属性。 

新队]

启动显示列值之前的新行。NEWLINE与FOLD_BEFORE具有相同的效果。除非PREFORMAT设置为ON，否则NEWL [INE]在SET MARKUP HTML ON模式下不起作用。

NEW_V [ALUE] 变量

指定用于保存列的变量值 。您可以在中引用变量 TTITLE 命令。使用NEW_VALUE显示列值或该日期在最高标题。您必须在带有SKIP PAGE操作的BREAK命令中包含该列。变量名称不能包含井号（＃）。

NEW_VALUE对主/详细报告很有用其中每个页面都有一个新的主记录。对于主/详细报告，您还必须在ORDER BY子句中包含该列。请参阅此命令说明末尾的示例。

使用NEW_V [ALUE]指定的变量在执行TTITLE之前展开。结果字符串存储为TTITLE文本。在报告的每个页面的后续执行期间，变量的扩展值本身可以被解释为具有意外结果的变量。

您可以通过不使用要在报表的每个页面上替换的NEW_V [ALUE]变量的＆前缀来避免在TTITLE命令中进行此双重替换。如果要使用替换变量在TTITLE中插入不变的文本，请将其括在引号中，以便仅替换一次。

有关在底部标题中显示列值的信息，请参阅下面的OLD_V [ALUE]变量。有关在标题中引用变量的更多信息，请参阅TTITLE命令。有关格式化和有效格式模型的信息，请参阅上面的FOR [MAT]格式。 

NOPRI [NT] |打印]

控制列的打印（列标题和所有选定的值）。NOPRINT 关闭屏幕输出和打印列。打印圈列打印。

NUL [L] 文字

控制SQL * Plus显示的文本给定列中的空值。默认为空格。SET NULL控制为所有列的所有空值显示的文本，除非通过COLUMN命令的NULL子句覆盖特定列。当选择NULL值时，变量的类型总是变为CHAR，因此SET NULL文本可以存储在其中。 

OLD_V [ALUE] 变量

指定用于保存列值的变量。 您可以在中引用变量 BTITLE 命令。使用OLD_VALUE在底部标题中显示列值。您必须在带有SKIP PAGE操作的BREAK命令中包含该列。

OLD_VALUE对主/详细报告很有用其中每个页面都有一个新的主记录。对于主/详细报告，您还必须在ORDER BY子句中包含该列。

在执行BTITLE之前，将扩展使用OLD_V [ALUE]指定的变量。结果字符串存储为BTITLE文本。在报告的每个页面的后续执行期间，变量的扩展值本身可以被解释为具有意外结果的变量。

您可以通过不使用要在报表的每个页面上替换的OLD_V [ALUE]变量的＆前缀来避免BTITLE命令中的这种双重替换。如果要使用替换变量在BTITLE中插入不变的文本，请将其括在引号中，以便仅替换一次。

有关在顶部标题中显示列值的信息，请参阅NEW_V [ALUE]变量。有关在标题中引用变量的更多信息，请参阅TTITLE命令。 

ON |关闭

控制列的显示属性的状态。关闭禁用列的属性而不影响属性的定义。上恢复属性。

WRA [PPED] | WOR [D_WRAPPED] | TRU [NCATED]

指定SQL * Plus如何处理对于列太宽的数据类型或DATE字符串。WRAPPED包装列边界内的字符串，在需要时开始新行。启用WORD_WRAP时，SQL * Plus左对齐每个新行，跳过所有前导空格（例如，返回，换行符，制表符和空格），包括嵌入的换行符。不会跳过不在行边界上的嵌入空格。截断的截断显示第一行末尾的字符串。

无法使用WORD_WRAPPED选项格式化NCLOB，BLOB，BFILE或多字节CLOB列。如果使用COLUMN WORD_WRAPPED格式化NCLOB，BLOB，BFILE或多字节CLOB列，则列数据的行为就像应用了COLUMN WRAPPED一样。

用法

您输入的COLUMN命令可以控制多个SQL SELECT命令的列的显示属性。

您可以为一列或多列输入任意数量的COLUMN命令。为每个列设置的所有列属性在会话的其余部分保持有效，直到您将列关闭，或直到您使用CLEAR COLUMN命令。

当你输入多个对于同一列的COLUMN命令，SQL * Plus统一应用它们的子句。如果多个COLUMN命令将同一子句应用于同一列，则输入的最后一个将控制输出。

例子

要使LAST_NAME列宽20个字符并在两行显示EMPLOYEE NAME作为列标题，请输入

COLUMN LAST_NAME FORMAT A20 HEADING'员工|姓名'

要格式化SALARY列，使其显示数百万美元，舍入为美分，使用逗号分隔数千，并在值为零时显示$ 0.00，输入

COLUMN SALARY FORMAT $ 9,999,990.99

要将别名NET分配给包含long表达式的列，以美元格式显示结果，并为空值显示<NULL>，可以输入

COLUMN SALARY + COMMISSION_PCT + BONUS-EXPENSES-INS-TAX ALIAS NET COLUMN NET FORMAT $ 9,999,999.99 NULL'<NULL>'

请注意，该示例将此列规范划分为两个命令。第一个定义别名NET，第二个使用NET定义格式。

另请注意，在第一个命令中，您必须完全按照在SELECT命令中输入的方式输入表达式。否则，SQL * Plus无法将COLUMN命令与相应的列匹配。
