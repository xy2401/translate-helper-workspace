词汇表

ADP

请参阅自动数据准备 。

聚合

将数据值合并为较少数量的值的过程。例如，每天收集的销售数据可以总计为周级别。

算法

解决问题的一系列步骤。参见数据挖掘算法 。Oracle Data Mining API支持以下算法： MDL ， Apriori ， 决策树 ， k-Means ， Naive Bayes ， GLM ， O-Cluster ， 支持向量机 ， 期望最大化和奇异值分解 。

算法设置

为模型构建指定特定于算法的行为的设置。

异常检测

检测异常值或非典型病例。Oracle Data Mining将异常检测实现为一类SVM。

应用

对数据进行评分的数据挖掘操作。评分是将模型应用于新数据以预测结果的过程。

先验

使用频繁项集来计算关联的算法。

协会

一种识别项目之间关系的机器学习技术。

关联规则

一种挖掘函数，用于捕获事务之间的项目共现。典型的规则是形式A - > B的含义，这意味着项集A的存在意味着项集B的存在具有一定的支持和信心。规则的支持是项目集A和B所在的交易数量与交易总数的比率。规则的置信度是项目集A和B所在的交易数量与项目集A所在的交易数量之比。Oracle Data Mining将Apriori算法用于关联模型。

属性

属性是预测模型中的预测器或描述性模型中的描述性信息项。数据属性是用于构建模型的数据列。数据属性经历转换，以便它们可以被模型用作分类或数字。分类和数字是模型属性 。另见目标 。

属性重要性

一种挖掘函数，用于衡量属性的重要性并预测指定的目标。训练数据表的不同属性的度量使用户能够选择被发现与挖掘模型最相关的属性。较小的属性集导致更快的模型构建;结果模型可能更准确。Oracle Data Mining使用Minimum Description Length来发现重要属性。有时也称为特征选择或关键字段 。

自动数据准备

可以使用自动数据准备（ADP）创建挖掘模型，ADP根据算法的要求转换构建数据，并将转换指令嵌入到模型中。只要将模型应用于新数据，就会执行嵌入式转换。

装袋

在训练样本上组合独立训练的模型（装袋是自举聚合）。

分级

请参阅离散化 。

建立数据

用于构建（训练）模型的数据。也称为训练数据 。

案件

收集的有关特定交易或相关值集的所有数据。数据集是案例的集合。案例也称为记录或示例 。在最简单的情况下，案例对应于表中的行。

案例表

单记录格式的表或视图。每个案例的所有数据都包含在一行中。案例表可以包括案例ID列，其保存每行的唯一标识符。挖掘数据必须作为案例表提供。

分类属性

一个属性，其值对应于离散类别。例如， state是具有离散值（CA，NY，MA）的分类属性。分类属性可以是非有序的（名义上的）状态或性别，也可以是有序的（有序的），如高温，中等或低温。

形心

请参阅群集质心 。

分类

一种挖掘功能，用于使用根据已知目标值的记录建立的模型来预测新记录的分类目标值。Oracle Data Mining支持以下分类算法：朴素贝叶斯，决策树，广义线性模型和支持向量机。

剪裁

看到修剪 。

群集质心

对于每个属性，为分配给集群的训练数据中的案例的均值（如果属性是数字）或模式（如果属性是分类的）编码的向量。群集质心通常被称为“质心”。

集群

用于在数据中查找自然发生的分组的挖掘函数。更准确地说，给定一组数据点，每个数据点具有一组属性，以及它们之间的相似性度量，聚类是将数据点分组到不同的聚类中的过程，使得同一聚类中的数据点彼此更相似并且不同群集中的数据点彼此不太相似。Oracle Data Mining支持三种算法，用于聚类， k-Means ， 正交分区聚类和期望最大化 。

混淆矩阵

测量模型从测试任务中做出的预测的正确性。混淆矩阵的行索引对应于在测试数据中观察和提供的实际值 。列索引对应于通过将模型应用于测试数据而产生的预测值 。对于任何一对实际/预测索引，该值表示在该配对中分类的记录数。

当预测值等于实际值时，模型会产生正确的预测。所有其他条目表示错误。

成本矩阵

n n表，定义与预测相关的成本与实际值。成本矩阵通常用于分类模型，其中n是目标中不同值的数量，列和行用目标值标记。行是实际值;列是预测值。

反

目标的负面实例。除了一类支持向量机之外，分类模型还需要反例。

数据挖掘

数据挖掘是自动搜索大型数据库以发现超出简单分析的模式和趋势的实践。数据挖掘使用复杂的数学算法来分割数据并评估未来事件的概率。数据挖掘也称为数据中的知识发现 （KDD）。

数据挖掘模型实现数据挖掘算法以解决给定数据集的给定类型的问题。

数据挖掘算法

用于生成数据挖掘模型的特定技术或过程。算法使用特定数据表示和特定挖掘函数 。

Oracle Data Mining支持的算法包括Naive Bayes ， 支持向量机 ， 广义线性模型和决策树，用于分类; 支持向量机和广义线性模型的回归; k-Means ， O-Cluster和期望最大化聚类; 最小描述属性重要性的长度 ;用于特征提取的非负矩阵分解和奇异值分解 ;用于关联的Apriori和用于异常检测的一类支持向量机 。

数据挖掘服务器

Oracle数据库的组件，用于实现数据挖掘引擎和持久性元数据存储库。您必须在执行数据挖掘任务之前连接到数据挖掘服务器。

数据集

一般来说，是一组数据。数据集是案例的集合。

描述模型

描述性模型有助于理解底层流程或行为。例如，关联模型可以描述消费者购买模式。另见挖掘模型 。

离散

离散化（分箱）将相关值组合在单个值（或箱）下。这减少了列中不同值的数量。较少的垃圾箱导致模型构建更快。许多Oracle数据挖掘算法（例如NB）可能会受益于在模型构建，测试，计算提升和应用（评分）之前离散化的输入数据。不同的算法可能需要不同类型的分箱。Oracle Data Mining支持有监督的分 箱 ，分类属性的前N个频率 分级和数字属性的等宽分级和分位数分箱 。

基于距离（聚类算法）

基于距离的算法依赖于距离度量（函数）来测量数据点之间的相似性。根据使用的距离度量将数据点分配给最近的集群。

决策树

决策树是分类系统或监督模型的表示。树的结构是一系列问题;问题的答案跟踪树下到叶子的路径，从而产生预测。

决策树是表示导致类或价值的一系列问题的一种方式。决策树的顶级节点称为根节点;终端节点称为叶节点。决策树通过将数据迭代分割成离散组来增长，其目标是最大化每个分组处的组之间的距离。

决策树模型的一个重要特征是它们是透明的;也就是说，有一些规则可以解释分类。

另见规则 。

等宽分级

等宽分级通过将值的范围划分为相等大小的指定数量的区间来确定数字属性的区间。

期望最大化

期望最大化是一种概率聚类算法，可创建数据的密度模型。密度模型允许改进的方法来组合源自不同域的数据（例如，销售交易和客户人口统计，或结构化数据和文本或其他非结构化数据）。

爆炸

对于分类属性 ，请使用多个二进制分类列替换多值分类列。要展开该属性，请为该属性所采用的每个不同值创建一个新的二进制列。在新列中，1表示属性的值取值列的值; 0，它没有。例如，假设分类属性采用值{1,2,3}。要展开此属性，请创建三个新列col_1 ， col_2和col_3 。如果属性取值为1，则col_1值为1;其他两列中的值为0。

特征

数据中属性的组合，这些属性具有特殊意义并且可以捕获数据的重要特征。请参阅特征提取 。

另请参见文本功能 。

特征提取

通过分解原始数据创建一组新功能。通过特征提取，您可以使用许多通常远小于原始属性数的要素来描述数据。另请参见非负矩阵分解和奇异值分解 。

广义线性模型

线性建模的统计技术。广义线性模型（GLM）包括并扩展了一类简单的线性模型。Oracle Data Mining支持GLM分类的逻辑回归和GLM回归的线性回归。

GLM

请参见广义线性模型 。

k -Means

基于距离的聚类算法，将数据划分为预定数量的聚类（假设存在足够的不同情况）。基于距离的算法依赖于距离度量（函数）来测量数据点之间的相似性。根据使用的距离度量将数据点分配给最近的集群。Oracle Data Mining提供了k -Means的增强版本。

电梯

使用模型比使用偶然获得的预测结果有多好的度量。例如，假设有2％的客户通过邮寄目录进行购买;假设当您使用模型选择目录收件人时，10％进行购买。然后模型的升力为10/2或5。提升也可以用作比较不同数据挖掘模型的度量。由于使用具有实际结果的数据表计算升力，因此升力比较模型对预测结果的数据执行情况。提升表示在给定实际结果的情况下，模型在随机选择上改进预测的程度。Lift允许用户推断模型对新数据的执行方式。

血统

在模型构建过程的数据准备阶段期间对数据集执行的转换序列。

线性回归

Oracle Data Mining支持的GLM回归算法。

逻辑回归

Oracle Data Mining支持的GLM分类算法。

最小描述长度

给出一个数据样本和有效的替代理论列举来解释数据，最好的理论是最小化总和的理论。

  • 理论描述的长度（以位为单位）

  • 在理论的帮助下编码时数据的长度（以位为单位）

最小描述长度原则用于选择对属性重要性中的目标值区分影响最大的属性 。

挖掘功能

数据挖掘的主要子域，具有共同的高级特征。Oracle Data Mining API支持以下挖掘功能： 分类 ， 回归 ， 属性重要性 ， 特征提取 ， 聚类和异常检测 。

采矿模型

一个第一类模式对象，它指定Oracle数据库中的数据挖掘模型 。

缺失价值

随机丢失的数据值。该值可能会丢失，因为它不可用，未知或因为丢失了。Oracle Data Mining将具有简单数据类型（非嵌套）的列中的缺失值解释为随机丢失。Oracle Data Mining将嵌套列中的缺失值解释为稀疏性。

数据挖掘算法在处理缺失值的方式上有所不同。有几种典型的方法可以处理它们：忽略它们，省略任何包含缺失值的记录，用模式或均值替换缺失值，或从现有值中推断缺失值。另请参见稀疏数据 。

模型

模型使用算法来实现给定的挖掘函数。模型可以是监督模型或无监督模型 。模型可用于直接检查，例如，检查从关联模型生成的规则，或对数据进行评分（预测结果）。在Oracle数据库中，数据挖掘模型实现为挖掘模型模式对象。

多记录案例

数据表中的每个案例都存储在多行中。也称为交易数据 。另见单记录案例 。

朴素贝叶斯

一种基于贝叶斯定理的分类算法。Naive Bayes假设每个属性在条件上独立于其他属性：给定目标的特定值，每个预测变量的分布独立于其他预测变量。

嵌套数据

Oracle Data Mining支持名称/值对的嵌套列中的事务数据 。表达一对多关系的多维数据可以加载到嵌套列中，并与案例表中的单记录案例数据一起挖掘。

NMF

请参阅非负矩阵分解 。

非负矩阵分解

一种特征提取算法，通过创建用户定义的特征数来分解多变量数据，从而减少原始数据的表示。

正常化

归一化包括将数值转换为特定范围，例如[-1.0,1.0]或[0.0,1.0]，使得x_new = (x_old-shift)/scale 。规范化仅适用于数字属性。Oracle Data Mining提供执行最小 - 最大规范化 ， 缩放规范化和z分数规范化的转换 。

数字属性

值为数字的属性。数值可以是整数或实数。数值属性值可以作为连续值进行操作。另请参见分类属性 。

O型集群

一类支持向量机

支持向量机的版本用于解决异常检测问题。该算法执行没有目标的分类。

正交分区聚类

Oracle专有的聚类算法，可创建基于网格的分层聚类模型，即在输入属性空间中创建轴并行（正交）分区。该算法递归地运行。生成的层次结构表示不规则网格，将属性空间细分为簇。

局外人

不是来自典型数据群或极值的数据值。在正态分布中，异常值通常与平均值至少相差三个标准差。

积极的目标价值

在二进制分类问题中，您可以将两个类别（目标值）中的一个指定为正数，另一个指定为负数。当Oracle Data Mining计算模型的提升时，它会计算一组测试实例中的正目标值的密度，模型会以给定的置信度预测正值。

预测模型

预测模型是一个方程或一组规则，可以从其他已知值（自变量或输入）预测未见或未测量的值（因变量或输出）。通过从所研究的过程收集的挖掘数据来建议等式或规则的形式。一些训练或估计技术用于估计等式或规则的参数。预测模型是监督模型 。

预报器

用作构建模型的监督算法的输入的属性。

准备好的数据

适用于使用指定算法建模的数据。数据准备通常占据数据挖掘项目花费的大部分时间。自动数据准备通过自动为算法准备数据，极大地简化了模型开发和部署。

主成分分析

主成分分析是作为奇异值分解算法的特殊评分方法实现的。

先验概率

该组先验概率指定原始源数据中各种类的示例的分布。也称为先验 ，这些可能与为模型构建提供的数据集中观察到的分布不同。

先验

参见先验概率 。

分位数分箱

数字属性被分成区域，使得每个区间包含大致相同数量的区间。

随机抽样

一个示例，其中数据集的每个元素都有相同的选择机会。

重新编码

字面上“改变或重新安排代码”。根据给定业务问题的要求，重新编码可用于准备数据，例如：

  • 缺少值处理：缺失值可能由NULL以外的其他值指示，例如“0000”或“9999”或“NA”或其他字符串。处理缺失值的一种方法是将例如“0000”重新编码为NULL 。然后，Oracle Data Mining算法和数据库将该值识别为缺失。

  • 更改变量的数据类型：例如，将“Y”或“是”更改为1，将“N”或“否”更改为0。

  • 建立一个临界值：例如，将所有低于20,000美元的收入重新编码为相同的值。

  • 组项目：例如，将美国各个州划分为区域。“新英格兰地区”可能包括ME，VT，NH，MA，CT和RI;为了实现这一点，将这五个州重新编号为NE（新英格兰）。

回归

奇异值分解

一种特征提取算法，使用正交线性投影捕获数据的基础方差。奇异值分解可以很好地扩展到非常大的数据大小（行和属性），并且具有强大的数据压缩功能。

请参见奇异值分解 。

稀疏数据

在任何给定情况下，只有一小部分属性为非零或非空的数据。市场购物篮数据和非结构化文本数据通常很少。Oracle Data Mining将嵌套数据解释为稀疏数据 。另见缺失值 。

分裂

将数据集划分为多个不相交的子集。例如，在分类问题中，数据集通常被划分为训练数据集和测试数据集。

分层样本

将数据集划分为不相交的子集（层），然后从每个子集中随机抽取样本。当目标值的分布大大偏离时，使用该技术。例如，对营销活动的响应可能在1％或更少的时间内具有正目标值。分层样本为数据挖掘算法提供了足够的正例，以学习区分正目标值和负目标值的因素。另见随机样本 。

监督分级

一种智能分箱形式，其中箱边界是从数据的重要特征导出的。有监督的分箱构建单预测器决策树以找到关于目标的有趣的箱边界。有监督的分箱可用于数字或分类属性。

监督学习

见监督模型 。

监督模型

使用已知因变量构建的数据挖掘模型，也称为目标。分类和回归技术是监督挖掘的例子。见无监督模型 。也称为预测模型 。

支持向量机

一种算法，它使用机器学习理论来最大化预测准确性，同时自动避免过度拟合数据。支持向量机可以使用稀疏数据进行预测，即，在具有大量预测变量列和相对较少行的域中，如生物信息学数据的情况。支持向量机可用于分类，回归和异常检测。

SVM

请参阅支持向量机 。

目标

在监督学习中，要预测的已识别属性。有时称为目标值或目标属性 。另见属性 。

文字功能

捕获文档或文档类重要属性的单词组合。文本功能通常是关键字，单词的频率或其他文档派生的功能。文档通常包含大量单词和少量特征。

文本挖掘

使用文本功能完成传统数据挖掘。文本功能通常是关键字，单词的频率或其他文档派生的功能。一旦获得文本特征，就像对待任何其他数据一样挖掘它们。Oracle Data Mining和Oracle Text都支持文本挖掘。

前N个频率分级

这种类型的分箱箱属于分类属性。基于从数据计算的值的出现频率来计算每个属性的bin定义。用户指定特定数量的区间，例如N.区间bin_1，...，bin_N中的每一个对应于具有顶部频率的值。bin bin_N + 1对应于所有剩余值。

交易数据

一个案例的数据包含在几行中。一个例子是市场购物篮数据，其中一个案例代表一个包含多个商品的购物篮。Oracle Data Mining支持属性名称/值对的嵌套列中的事务数据。另请参见嵌套数据 ， 多记录案例和单记录案例 。

转型

应用于数据的函数，导致数据的新表示。例如，离散化和标准化是对数据的转换。

修剪

一种最小化异常值影响的技术。在进一步计算中忽略修剪值的意义上，修剪会删除分布尾部中的值。通过将尾部设置为NULL来实现修剪。

非结构化数据

图像，音频，视频，地理空间映射数据和文档或文本数据统称为非结构化数据。Oracle Data Mining支持挖掘非结构化文本数据。

无监督模型

在没有已知正确结果的指导（监督）的情况下建立的数据挖掘模型。在监督学习中，在目标属性中提供了这个正确的结果。无监督学习没有这样的目标属性。聚类和关联是无监督挖掘功能的示例。见监督模型 。

极值调整

一种最小化异常值影响的技术。Winsorizing涉及将特定属性的尾值设置为某个指定值。例如，对于90％Winsorization，最低5％的值设置为等于第6百分位数中的最小值，而高5％设置为等于第95百分位数中的最大值。
