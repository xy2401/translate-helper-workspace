A.7跟踪文件属性更改

使用Oracle ORAchk和Oracle EXAchk -fileattr选项和命令标志来记录和跟踪文件属性设置，并比较快照。

句法

[-fileattr start] [-fileattr check] [-fileattr remove] [-fileattr [start | check] -includedir directories ] [-fileattr [start | check] -excludediscovery] [-fileattr check -baseline baseline snapshot path ] [ - fileattr check -fileattronly]

表A-6 Oracle ORAchk和Oracle EXAchk文件属性跟踪选项列表

选项 描述

-fileattr start

获取已发现目录的文件属性快照，并将快照存储在输出目录中。

默认情况下，此选项会获取Oracle Grid Infrastructure主目录和所有已安装的Oracle数据库主目录的快照。如果用户不拥有特定目录，则该工具不会拍摄目录的快照。

-fileattr检查

获取已发现目录的新快照，并将其与先前的快照进行比较。

-fileattr删除

删除文件属性快照和相关文件。

-fileattr [start | check] -includedir directories

指定以逗号分隔的目录列表以检查文件属性。

例如：
./orachk -fileattr start -includedir“/ root / home，/ etc”
./orachk -fileattr check -includedir“/ root / home，/ etc”

-fileattr [start | check] -excludediscovery

排除发现的目录。

例如：
./orachk -fileattr start -includedir“/ root / home，/ etc”-excludediscovery

-fileattr check -baseline baseline snapshot path

使用您指定的快照作为快照比较的基准。提供要用作基准的快照的路径。

基线是您要在以后比较的起始文件属性。将当前文件属性与基线进行比较，并报告增量。

例如：
./orachk -fileattr check -baseline“/ tmp / Snapshot”

-fileattr check -fileattronly

仅执行文件属性检查，然后退出Oracle ORAchk。

例如：
./orachk -fileattr check -fileattronly
