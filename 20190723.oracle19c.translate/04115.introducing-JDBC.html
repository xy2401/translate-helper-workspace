1 JDBC简介

Java数据库连接（ JDBC）是一种Java标准，它提供了从Java连接到关系数据库的接口。JDBC标准是通过标准java.sql接口定义和实现的。这使各个提供程序能够使用自己的JDBC驱动程序实现和扩展标准。JDBC基于X / Open SQL调用级接口（CLI）。JDBC 4.0符合SQL 2003标准。

1.1 Oracle JDBC驱动程序概述

除了支持标准JDBC应用程序编程接口（API）之外，Oracle驱动程序还具有支持Oracle特定数据类型和增强性能的扩展。

Oracle提供以下JDBC驱动程序：

  • 瘦司机

    该 JDBC Thin驱动程序是纯Java，Type IV驱动程序，可以在应用程序中使用。它独立于平台，在客户端不需要任何其他Oracle软件。JDBC Thin驱动程序使用Oracle Net Services与服务器通信以访问Oracle数据库。

    JDBC Thin驱动程序通过提供Oracle Net Services的实现来实现与数据库的直接连接 Java套接字。该驱动程序支持TCP / IP协议，并且需要数据库服务器上的TCP / IP套接字上的TNS侦听器。

    注意：

    除非您具有仅由特定驱动程序支持的功能，否则Oracle建议您使用Thin驱动程序。

  • Oracle调用接口（OCI）驱动程序

    它在客户端用于Oracle客户端安装。它只能用于应用程序。

    该 JDBC OCI驱动程序是与Java应用程序一起使用的Type II驱动程序。它需要特定于平台的OCI库。它支持所有已安装的Oracle Net适配器，包括进程间通信（IPC），命名管道，TCP / IP和Internetwork Packet Exchange / Sequenced Packet Exchange（IPX / SPX）。

    JDBC OCI驱动程序使用Java和C的组合编写，使用本机方法调用C-entry点，将JDBC调用转换为对OCI的调用。这些调用使用Oracle Net Services与数据库通信。

    JDBC OCI驱动程序在安装它的客户端计算机上使用OCI库，C-entry点，Oracle Net，核心库和其他必需文件。

    OCI是一种API，使您能够创建使用第三代语言的本机过程或函数调用的应用程序来访问Oracle数据库并控制SQL语句处理的所有阶段。

  • 服务器端瘦驱动程序

    它在功能上类似于客户端瘦驱动程序。但是，它用于在数据库服务器上运行的代码，并且需要访问同一服务器或任何层上的远程服务器上的另一个会话。

    该 JDBC服务器端Thin驱动程序提供与在客户端运行的JDBC Thin驱动程序相同的功能。但是，JDBC服务器端Thin驱动程序在Oracle数据库中运行，并访问远程数据库或同一数据库上的不同会话，以便与数据库中的Java一起使用。

    此驱动程序在以下方案中很有用：

    • 从充当中间层的Oracle数据库实例访问远程数据库服务器

    • 从另一个内部访问Oracle数据库会话，例如从Java存储过程访问

    从客户端应用程序或从服务器内部使用JDBC Thin驱动程序不会影响代码。

  • 服务器端内部驱动程序

    它用于在数据库服务器上运行并访问同一会话的代码。也就是说，代码运行并访问单个Oracle会话中的数据。

    该 JDBC服务器端内部驱动程序支持在Oracle数据库中运行的任何Java代码，例如在Java存储过程中，并访问同一数据库。它允许Oracle Java虚拟机（Oracle JVM）直接与SQL引擎通信，以便与数据库中的Java一起使用。

    JDBC服务器端内部驱动程序，Oracle JVM，数据库和SQL引擎都在同一地址空间内运行，因此，网络往返问题无关紧要。程序通过使用函数调用来访问SQL引擎。

    注意：

    服务器端内部驱动程序不支持Statement类的cancel和setQueryTimeout方法。

    JDBC服务器端内部驱动程序与客户端驱动程序完全一致，并支持相同的功能和扩展。

1.2选择合适的驱动程序

为应用程序或applet选择JDBC驱动程序时，请考虑以下事项：

  • 通常，除非您需要特定于OCI的功能，例如支持非TCP / IP网络，否则请使用JDBC Thin驱动程序。

  • 如果您想要最大的可移植性和性能，请使用JDBC Thin驱动程序。您可以使用JDBC Thin驱动程序从应用程序连接到Oracle数据库。

  • 如果要在安全套接字层（SSL）/传输层安全性（TLS）上使用轻量级目录访问协议（LDAP），请使用JDBC Thin驱动程序。

  • 如果要为Oracle客户端环境编写客户端应用程序并且需要特定于OCI驱动程序的功能（例如支持非TCP / IP网络），请使用JDBC OCI驱动程序。

  • 对于在数据库服务器中运行并需要访问同一数据库实例中的远程数据库或另一个会话的代码，请使用JDBC服务器端Thin驱动程序。

  • 如果您的代码在数据库服务器内运行并且需要在会话中本地访问数据，那么使用JDBC服务器端内部驱动程序来访问该服务器。

1.4环境和支持

本节简要讨论以下主题：

  • 支持的JDK和JDBC版本

1.4.1支持的JDK和JDBC版本

在Oracle Database 12 c第2版（12.2.0.1）中，所有JDBC驱动程序都与JDK 8兼容。通过ojdbc8.jar文件提供对JDK 8的支持。

相关话题

  • Oracle JDBC驱动程序的版本兼容性
  • Oracle JDBC驱动程序的版本兼容性

1.4.2 JNI和Java环境

JDBC OCI驱动程序使用标准Java本机接口（JNI）来调用OCI C库。您可以将JDBC OCI驱动程序与Java虚拟机（JVM）一起使用，尤其是与Microsoft和IBM JVM一起使用。

1.4.3 JDBC和IDE

该神谕 JDeveloper Suite为开发人员提供了一组集成的产品，用于构建，调试和部署基于组件的Internet数据库应用程序。Oracle JDeveloper环境包含对JDBC的集成支持，包括JDBC Thin驱动程序和本机OCI驱动程序。Oracle JDeveloper的数据库组件使用JDBC驱动程序来管理客户端上运行的应用程序与服务器之间的连接。
