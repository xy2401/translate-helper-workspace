词汇表

添加剂

描述可以通过添加来概括的事实 （或度量 ）。附加事实是最常见的事实类型。例子包括销售，成本和利润。与非 添加剂和半添加剂对比。

顾问

请参见SQL Access Advisor 。

骨料

汇总数据。例如，特定产品的单位销售额可以按日，月，季度和年销售额汇总。

聚合

将数据值合并为单个值的过程。例如，可以每天收集销售数据，然后将其汇总到周级别 ，将周数据汇总到月级别，依此类推。然后可以将数据称为聚合数据。术语聚合与摘要同义，聚合数据与摘要数据同义。

分析视图

一种封装事实数据的聚合，计算和连接的视图。分析视图使用维模型组织数据。它们允许您轻松地向数据集添加聚合和计算，并在可以使用相对简单的SQL查询的视图中显示数据。

祖先

高于层次结构中给定值的任何级别的值。例如，在时间维度中 ，值1999可能是值Q1-99和Jan-99的祖先。

属性

一个或多个级别的描述性特征。例如，服装制造商的产品尺寸可能包含一个名为item的级别 ，其中一个属性为color。属性表示允许最终用户基于类似特征选择数据的逻辑分组。

请注意，在关系建模中，属性被定义为实体的特征。在Oracle数据库10g中 ，属性是维度中的一列，用于表征单个级别的每个元素 。

属性维度

指定数据源以及属性维度属性的数据源列。它指定其成员的级别并确定级别之间的属性关系。属性维度由层次结构和分析视图使用。

基数

从OLTP角度来看，这指的是表中的行数。从数据仓库的角度来看，这通常是指列中不同值的数量。对于大多数数据仓库 DBA来说，更重要的问题是基数的程度 。

儿童

层次结构中给定值下的级别的值。例如，在时间维度中 ，值Jan-99可能是值Q1-99的子值。如果子值属于多个层次结构，则值可以是多个父项的子项。

净化

解决不一致问题和修复源数据异常的过程，通常作为ETL过程的一部分。

通用仓库元数据（CWM）

Oracle数据仓库和决策支持使用的存储库标准。CWM存储库模式是其他产品可以共享的独立产品 - 每个产品仅拥有其创建的CWM存储库中的对象。

交叉产品

用于组合多个集合中的元素的过程。例如，给定的两列中，每个元件的第一列中的一个与第二列中的每个元素匹配。一个简单的例子说明如下：

Col1 Col2 Cross Product ---- ---- ------------- ac ac bd ad bc bd

如果连接分组集，则会执行交叉产品，如数据仓库中的聚合SQL所述。

数据库

专为特定业务线（如销售，营销或财务）设计的数据仓库 。在从属数据集市中，数据可以从企业范围的数据仓库派生。在独立的数据集市中，可以直接从源收集数据。

数据源

将数据提供给仓库的数据库，应用程序，存储库或文件。

数据仓库

一种关系数据库，专为查询和分析而非事务处理而设计。数据仓库通常包含从事务数据派生的历史数据，但它可以包含来自其他源的数据。它将分析工作负载与事务工作负载分开，使企业能够整合来自多个来源的数据。

除了关系数据库之外，数据仓库环境通常还包括ETL解决方案，分析SQL引擎，客户端分析工具以及管理收集数据并将其交付给业务用户的过程的其他应用程序。

基数度

列的唯一值数除以表中的总行数。在决定构建哪些索引时，这一点尤其重要。您通常希望在高度基数列上使用低基数列和B树索引的位图索引。作为一般规则，小于1％的基数是位图索引的良好候选者。

非规范化

允许表中冗余的过程。与标准化对比。

派生事实（或衡量标准）

使用数学运算或数据转换从现有数据生成的事实 （或度量 ）。示例包括平均值，总数，百分比和差异。

详情

见： 事实表 。

细节表

见： 事实表 。

尺寸

术语维度通常以两种方式使用：

  • 用于指定数据集成员的任何特征的通用术语。面向销售的数据仓库中最常见的三个维度是时间，地理位置和产品。大多数维度都有层次结构。

  • 数据库中定义的对象，用于使查询能够导航维度。在Oracle数据库10g中 ，维度是一个数据库对象，用于定义列集对之间的层次（ 父 / 子 ）关系。在Oracle Express中，维度是一个由值列表组成的数据库对象。

维度表

维度表描述企业的业务实体，表示为分层的分类信息，如时间，部门，位置和产品。维度表有时称为查找表或引用表。

维度值

列表中构成维度的一个元素 。例如，计算机公司可能在产品维度中具有名为LAPPC和DESKPC的维度值。地理维度中的值可能包括波士顿和巴黎。时间维度中的值可能包括MAY96和JAN97。

钻头

从一个项目导航到一组相关项目。钻取通常涉及在层次结构中的一个级别 （或多个级别）上下导航。在选择数据时，可以在向下钻取时展开层次结构，并在向上钻取时折叠层次结构。

深入研究

展开视图，包括孩子是与相关值父母在价值观层次 。

向上钻

折叠与层次结构中的父值关联的后代值列表。

元件

对象或过程。例如， 维度是对象， 映射是流程，两者都是元素。

企业数据仓库

数据仓库，其中原始数据合并在一个存储位置，并用作数据仓库体系结构的中心。

实体

实体用于数据库建模。在关系数据库中，它通常映射到表。

ELT

ELT代表提取 ，装载， 转换和运输 。这是旧ETL的更现代版本。

ETL

ETL代表提取 ， 转换和加载。ETL指的是访问和操作源数据并将其加载到数据仓库中所涉及的方法。执行这些过程的顺序各不相同。

注意，有时使用ETT（提取，转换， 运输 ）和ETM（提取，转换，移动）代替ETL。

萃取

将数据从源中取出作为ETL初始阶段的一部分的过程。

事实

可以检查和分析的数据，通常是数字和附加的 。例子包括销售，成本和利润。事实和衡量标准是同义词;事实是更常用于关系环境，测量更常用于多维环境。使用数学运算或数据变换从现有数据生成派生事实（或度量） 。

事实表

星型模式中包含事实的表。事实表通常具有两种类型的列：包含事实的列和具有维度表外键的列。事实表的主键通常是由其所有外键组成的复合键。

事实表可能包含详细级别事实或已聚合的事实（包含聚合事实的事实表通常称为汇总表）。事实表通常包含具有相同聚合级别的事实。

快速刷新

仅将数据更改应用于实例化视图的操作 ，因此无需从头开始重建实例化视图。

文件到表的映射

将数据从平面文件映射到仓库中的表。

等级制度

使用有序级别作为组织数据的手段的逻辑结构。层次结构可用于定义数据聚合 ;例如，在时间维度中 ，可以使用层次结构将数据从月级别到季度级别聚合到年级别。可以在Oracle中将层次结构定义为维度对象的一部分。无论层次结构中的级别是否表示聚合总计，层次结构也可用于定义导航钻取路径。

层次结构也可以是数据字典对象，它是一种定义属性维度级别之间的层次关系的视图类型。属性维和层次结构提供分析视图的维成员。

水平

层次结构中的位置。例如，时间维度可能具有表示月，季度和年级别数据的层次结构。

级别值表

一个数据库表，用于存储作为维度和层次结构的一部分创建的级别的值或数据。

制图

源和目标对象之间的关系和数据流的定义。

物化观点

预先计算的表，包括来自事实和可能的维度表的聚合或连接数据。也称为摘要或聚合表。

物化视图日志

记录有关给定物化视图的详细信息的日志。如果要使用快速刷新，则需要物化视图日志，但分区更改跟踪刷新除外。

测量

看看事实 。

元数据

描述数据和其他结构的数据，例如对象，业务规则和流程。例如， 数据仓库的模式设计通常作为元数据存储在存储库中，元数据用于生成用于构建和填充数据仓库的脚本。存储库包含元数据。

示例包括：对于数据，定义用于生成和填充数据仓库的目标 转换的源 ;用于信息，存储在关系建模工具中的表，列和关联的定义;对于商业规则，在销售1,000件商品后折扣10％。

模型

表示要制作的东西的对象。代表性的风格，计划或设计。模型也可以是定义数据仓库结构的元 数据 。

非加

描述无法通过添加进行总结的事实 （或度量 ）。一个例子包括平均值。与添加剂和半添加剂对比。

正常化

在关系数据库中，通过将数据分成多个表来删除数据冗余的过程。与非规范化对比。

通过将数据分成多个表来删除数据冗余的过程。

OLTP

请参阅： 在线事务处理（OLTP） 。

在线交易处理（OLTP）

在线交易处理。OLTP系统经过优化，可实现快速可靠的事务处理。与数据仓库系统相比，大多数OLTP交互将涉及相对较少的行，但涉及较大的一组表。

并行执行

分解任务，以便多个流程完成部分工作。当多个CPU同时执行其部分时，可以实现非常大的性能提升。

排比

分解任务，以便多个流程完成部分工作。当多个CPU同时执行其部分时，可以实现非常大的性能提升。

亲

一个值 在电平以上的给定值的层次结构 。例如，在时间维度中 ，值Q1-99可能是子值Jan-99的父Jan-99 。

划分

使用非常大的表和索引可能很困难且耗时。为了提高可管理性，您可以将表和索引分解为称为分区的较小部分。

分区变更跟踪（PCT）

一种跟踪分区和子分区级别的物化视图的陈旧性的方法。

模式匹配

一种使用MATCH_RECOGNIZE子句识别行序列中的模式的方法。

旋转

转换 ，其中输入流中的每个记录都转换为数据仓库中相应表中的许多记录。从非关系数据库中获取数据时，这一点尤为重要。

查询重写

使用物化视图 （预先计算）快速回答查询的机制。

刷新

物化视图的更改机制，以反映新数据。

改写

请参阅： 查询重写 。

模式

相关数据库对象的集合。关系模式按数据库用户ID分组，包括表，视图和其他对象。本指南中使用了示例模式sh 。两种特殊类型的模式是雪花模式和星型模式 。

半加成

描述可以通过沿着一些但不是所有维度添加来概括的事实 （或度量 ）。例子包括人数和现有库存。与添加剂和非 添加剂对比。

切片和骰子

这是一个涉及数据检索和操作的非正式术语。我们可以将数据仓库描绘为数据立方体，其中立方体的每个轴代表一个维度 。“切片”数据是通过为某些或所有维度指定度量和值来检索多维数据集的一块（一个切片）。当我们检索数据切片时，我们也可以移动并重新排序其列和行，就像我们将切片切成许多小块一样。具有良好切片和切割的系统可以轻松浏览大量数据。

雪花模式

一种星型模式 ，其中每个维度表部分或完全标准化。

资源

数据库，应用程序，文件或其他存储工具，从中导出数据仓库中的数据 。

源系统

数据库，应用程序，文件或其他存储工具，从中导出数据仓库中的数据 。

源表

源数据库中的表。

SQL Access Advisor

SQL Access Advisor通过为给定工作负载建议正确的物化视图集，物化视图日志，分区和索引，帮助您实现性能目标。它是Oracle Enterprise Manager中的GUI，具有与DBMS_ADVISOR包类似的功能。

集结区

在进入仓库之前处理数据的地方。

暂存文件

在进入仓库之前处理数据时使用的文件。

明星查询

事实表和许多维度表之间的连接。使用主键到外键连接将每个维度表连接到事实表，但维度表不会相互连接。

星型模式

一种关系模式，其设计表示多维数据模型 。星型模式由一个或多个事实表和一个或多个通过外键相关的维表组成。

学科领域

表示或区分组织或知识领域的部分的分类系统。通常开发数据集市以支持诸如销售，营销或地理的主题领域。

摘要

请参阅： 物化视图 。

摘要顾问

由SQL Access Advisor替换。

目标

保存ETL过程任何部分的中间或最终结果。整个ETL过程的目标是数据仓库 。

第三范式（3NF）

一种经典的关系数据库建模技术，通过规范化最小化数据冗余。

第三种常规形式架构

使用与OLTP系统中通常相同的规范化的模式 。有时为大型数据仓库选择第三种常规表单模式，尤其是具有大量数据加载要求的环境，用于为数据集市提供数据并执行长时间运行的查询。与雪花模式和星型模式进行比较。

转型

操纵数据的过程。任何超越复制的操作都是一种转变。示例包括清理 ，聚合和集成来自多个源表的数据 。

运输

将复制或转换的数据从源移动到数据仓库的过程 。与转换比较。

唯一标识符

一个标识符，其目的是在出现在多个位置时区分同一项目。

更新窗口

可用于更新仓库的时间长度。例如，您可能需要8个小时才能更新仓库。

更新频率

使用新信息更新数据仓库的频率。例如，仓库可能每晚从OLTP系统更新。

验证

验证元数据定义和配置参数的过程。

版本

能够为新的需求和更改创建数据仓库项目的新版本。
