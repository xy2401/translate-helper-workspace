关于Oracle数据库FIPS 140-2设置

联邦信息处理标准（FIPS）标准140-2是美国政府标准，定义了加密模块安全要求。

FIPS 140-2加密库旨在通过网络保护静止和传输的数据。

Oracle数据库将这些加密库用于安全套接字层（SSL），透明数据加密（TDE）和DBMS_CRYPTO PL / SQL包。

要验证认证的当前状态，您可以在美国国家标准与技术研究院的计算机安全资源中心（CSRC）网站地址中找到相关信息：

http://csrc.nist.gov/groups/STM/cmvp/validation.html

您可以通过搜索Validated FIPS 140 Cryptographic Modules找到特定于FIPS的信息。成功通过认证后，此站点上提供的安全策略包括安全配置主机操作系统的要求。

配置FIPS 140-2以实现透明数据加密和DBMS_CRYPTO

DBFIPS_140初始化参数配置FIPS模式。

  1. 要将透明数据加密和DBMS_CRYPTO PL / SQL程序包单元配置为在FIPS模式下运行，请将DBFIPS_140初始化参数设置为TRUE 。
    此参数的效果取决于平台。
  2. 重启数据库。

表E-1描述了DBFIPS_140参数如何影响各种平台。

表E-1 DBFIPS_140初始化参数如何影响平台

平台 将DBFIPS_140设置为TRUE或FALSE的效果

Intel x86_64上的Linux或Windows

  • TRUE ：TDE和DBMS_CRYPTO程序单元使用Micro Edition Suite（MES）4.0.5.1 FIPS模式

  • FALSE ：TDE和DBMS_CRYPTO程序单元使用Intel Performance Primitives（IPP）

SPARC T系列或Intel x86_64上的Solaris 11.1+

  • TRUE ：TDE和DBMS_CRYPTO程序单元使用MES 4.1.2 FIPS模式

  • FALSE ：TDE和DBMS_CRYPTO程序单元使用Solaris加密框架（SCF）/ UCrypto（单独验证FIPS 140）

其他操作系统或硬件

  • TRUE ：TDE和DBMS_CRYPTO程序单元使用MES 4.1.2 FIPS模式

  • FALSE ：TDE和DBMS_CRYPTO程序单元使用MES 4.1.2非FIPS模式

请注意，将DBFIPS_140设置为TRUE并因此在FIPS模式下使用底层库会在首次加载库时产生一定的开销。这是由于签名的验证和库上自检的执行。加载库后，对性能没有其他影响。

为安全套接字层配置SSLFIPS_140参数

在fips.ora文件中将SSLFIPS_140参数设置为TRUE可将安全套接字层（SSL）适配器配置为在FIPS模式下运行。

  1. 确保fips.ora文件位于$ORACLE_HOME /ldap/admin目录中，或者位于FIPS_HOME环境变量指向的位置。
  2. 在fips.ora文件中，设置SSLFIPS_140 。

    例如，要将SSLFIPS_140设置为TRUE ：

    SSLFIPS_140 = TRUE

    默认情况下，此参数设置为FALSE 。您必须在客户端和服务器上将其设置为TRUE才能进行FIPS模式操作。

  3. 对任何数据库服务器或客户端在任何Oracle数据库主目录中重复此过程。

将SSLFIPS_140设置为TRUE ，安全套接字层加密操作将在FIPS模式下的嵌入式RSA / Micro Edition Suite（MES）库中进行。当硬件加速可用并在主机硬件和软件中正确配置时，CPU会加速这些加密操作。

如果将SSLFIPS_140设置为FALSE ，则安全套接字层加密操作将在非 -FIPS模式下在嵌入式RSA / Micro Edition Suite（MES）库中进行，与TRUE设置一样，如果可能，将加速操作。

对于本机加密，登陆RSA / Micro Edition Suite（MES）并加速的加密操作的这种行为与上述类似，不同之处在于它由sqlnet.ora的FIPS_140设置决定（而不是fips.ora中的SSL_FIPS140设置） fips.ora ）。

注意：

SSLFIPS_140参数替换Oracle Database 10g第 2版（10.2）中使用的SQLNET.SSLFIPS_140参数。您必须在fips.ora文件中设置参数，而不是sqlnet.ora文件。

经批准的FIPS 140-2 SSL密码套件

密码套件是一组在网络节点之间交换消息的身份验证，加密和数据完整性算法。

例如，在SSL握手期间，两个节点协商以查看在来回传输消息时它们将使用哪个密码套件。

只有以下密码套件才能获得FIPS验证：

  • SSL_DH_anon_WITH_3DES_EDE_CBC_SHA

  • SSL_RSA_WITH_AES_256_CBC_SHA

  • SSL_RSA_WITH_AES_128_CBC_SHA

  • SSL_RSA_WITH_AES_256_GCM_SHA384

  • SSL_RSA_WITH_3DES_EDE_CBC_SHA

Oracle Database SSL密码套件自动设置为FIPS认可的密码套件。如果您希望配置特定的密码套件，可以通过编辑sqlnet.ora或listener.ora文件中的SSL_CIPHER_SUITES参数来实现。 

SSL_CIPHER_SUITES =（SSL_cipher_suite1 [，SSL_cipher_suite2 [..]]）

您还可以使用Oracle Net Manager在服务器和客户端上设置此参数。

安装后检查FIPS 140-2

配置FIPS 140-2设置后，必须验证操作系统中的权限。

权限如下：

  • 根据系统安全策略，对所有Oracle可执行文件设置执行权限，以防止未经授权的用户执行Oracle加密库。

  • 设置所有Oracle可执行文件的读写权限，以防止任何用户意外或故意阅读或修改Oracle加密库。

为了符合FIPS 140-2 Level 2要求，在安全策略中，包括防止未经授权的用户读取，修改或执行Oracle Cryptographic Libraries进程及其在操作系统中使用的内存的过程。

验证FIPS 140-2连接

要检查是否为SSL启用了FIPS模式，可以在sqlnet.ora文件中启用跟踪。

您可以在跟踪文件中找到FIPS自检消息。
  1. sqlnet.ora添加到sqlnet.ora以启用跟踪：
    trace_directory_server = trace_directory trace_file_server = trace_file trace_level_server = trace_level

    例如：

    trace_directory = / private / oracle / owm trace_file_server = fips_trace.trc trace_level_server = 6
  2. 要检查是否为TDE和DBMS_CRYPTO启用了FIPS模式，请登录SQL * Plus并运行以下命令：
    显示参数DBFIPS_140

跟踪级别6是检查FIPS自检结果所需的最低跟踪级别。
