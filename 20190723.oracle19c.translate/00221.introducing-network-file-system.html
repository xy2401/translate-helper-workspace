18网络文件系统简介（NFS）

NFS协议是一种广泛使用的文件系统协议，用于跨网络访问存储。

注意：

NFS客户端可以通过将它们安装在客户端计算机上来访问通过OFS服务器导出的Oracle对象

话题：

  • 通过NFS服务器访问存储的先决条件

  • NFS安全性

通过NFS服务器访问存储的先决条件

通过NFS服务器访问存储的先决条件如下：

  • 必须在使用OFS之前创建DBFS文件系统。

  • 您应该能够挂载数据库导出的文件系统。

  • 必须使用KERNEL模块配置NFS服务器。

    注意：

    通过适用于Linux的FUSE驱动程序支持KERNEL模块。

NFS安全性

从Oracle Database 12 c Release 2（12.2.0.1）开始，OFS将使用OS身份验证模型来授权NFS客户端用户。如果用户正在访问本地节点（运行Oracle实例的位置），则通过为每个对象设置的Unix访问控制列表来控制对文件系统中每个文件的访问 。在Linux上，OFS使用FUSE从OS内核或NFS客户端接收文件系统请求。如果除root用户和oracle用户之外的OS用户需要访问文件系统，则需要在/etc/fuse.conf配置文件中设置user_allow_other参数。

注意：

用户还可以配置Oracle密码以登录Oracle客户端工具（如SQL* Plus来执行SQL。

如果网络不安全，建议客户设置Kerberos以使用OS NFS对用户进行身份验证。

注意：

  • 从NFS版本4开始可以使用Kerberos身份验证。如果通过NFS版本3导出OFS，则使用AUTH_SYS执行身份验证。

  • 对于本地节点，无论OFS如何导出（NFS版本3或NFS版本4），都使用AUTH_SYS执行身份验证。

Kerberos的

Kerberos是广泛使用的安全机制，提供所有三种安全性：

  • 认证

  • 完整性检查

  • 隐私

Kerberos使用加密技术，密钥分发中心（KDC）和仲裁程序在开放网络上执行安全身份验证。Kerberos基础结构由Kerberos软件，安全认证服务器，集中式帐户和密码存储以及配置为通过Kerberos协议进行身份验证的系统组成。OS NFS服务器使用kerberos主体名称作为用户名来处理完整的身份验证和完整性检查。执行身份验证后，将根据通过VFS I / O请求传递的用户名来处理传递给Oracle内核的请求。

话题：

  • 在Linux中配置Kerberos服务器

在Linux中配置Kerberos服务器

在Linux系统中配置Kerberos服务器的步骤如下：

  1. 在Linux系统中安装Kerberos软件。

  2. 使用以下命令检查守护程序是否正在运行。
    ＃/ sbin / chkconfig krb5kdc on＃/ sbin / chkconfig kadmin on
  3. 如果守护程序未运行，请使用以下命令手动启动守护程序：
    ＃/ etc / rc.d / init.d / krb5kdc start＃/ etc / rc.d / init.d / kadmin start

  4. 使用kadmin.local命令添加用户kadmin.local 。

    例：
    kadmin.local：addprinc <scott>
