OCIFileObject

OCIFileObject数据结构保存有关文件打开方式以及打开文件后访问方式的信息。当此结构由OCIFileOpen()初始化时，它将成为一个标识符，通过该标识符可以对该文件执行操作。它是对打开文件进行操作的每个函数的必要参数。此数据结构对OCIFile客户端不透明。它是由初始化OCIFileOpen（）和通过封端（）OCIFileClose 。

29.6.1 OCIFileClose（）

关闭以前打开的文件。

目的

关闭以前打开的文件。

句法

sword OCIFileClose（void * hndl，OCIError * err，OCIFileObject * filep）;

参数

hndl（IN）

OCI环境或用户会话句柄。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。可以通过调用OCIErrorGet()获取诊断信息。

filep（IN / OUT）

指向要关闭的文件标识符的指针。

评论

一旦这个函数返回OCI_SUCCESS，该OCIFileObject结构指向filep被破坏。因此，在此函数返回OCI_SUCCESS后，您不应尝试访问此结构。

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。

29.6.6 OCIFileOpen（）

打开一个文件。

目的

打开一个文件。

句法

剑OCIFileOpen（void * hndl，OCIError * err，OCIFileObject ** filep，OraText * filename，OraText * path，ub4 mode，ub4 create，ub4 type）;

参数

hndl（IN）

OCI环境或用户会话句柄。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。可以通过调用OCIErrorGet()获取诊断信息。

filep（IN / OUT）

文件标识符。

文件名（IN）

文件名为NULL -terminated字符串。

路径（IN）

文件的路径为NULL -terminated字符串。

模式（IN）

打开文件的模式。有效模式是

OCI_FILE_READ_ONLY

OCI_FILE_WRITE_ONLY

OCI_FILE_READ_WRITE

创建（IN）

指示如果文件不存在，是否要创建该文件。有效值为：

OCI_FILE_TRUNCATE - 创建文件，无论文件是否存在。如果文件存在，则覆盖现有文件。

OCI_FILE_EXCL - 如果文件存在则失败;否则，创建一个文件。

OCI_FILE_CREATE - 如果文件存在则打开文件，如果不存在则创建它。

OCI_FILE_APPEND - 在写入之前将文件指针设置为文件的末尾。该标志可以与逻辑运算符OR一起使用，也可以与OCI_FILE_CREATE一起OCI_FILE_CREATE 。

类型（IN）

文件类型。有效值为：

OCI_FILE_TEXT

OCI_FILE_BIN

OCI_FILE_STDIN

OCI_FILE_STDOUT

OCI_FILE_STDERR

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。

29.6.7 OCIFileRead（）

从文件读入缓冲区。

目的

从文件读入缓冲区。

句法

sword OCIFileRead（void * hndl，OCIError * err，OCIFileObject * filep，void * bufp，ub4 bufl，ub4 * bytesread）;

参数

hndl（IN）

OCI环境或用户会话句柄。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。可以通过调用OCIErrorGet()获取诊断信息。

filep（IN / OUT）

唯一引用该文件的文件标识符。

bufp（IN）

指向读取数据的缓冲区的指针。假定分配的内存的长度为bufl 。

bufl（IN）

缓冲区的长度，以字节为单位。

bytesread（OUT）

读取的字节数。

评论

尽可能多的字节被读入用户缓冲区。读取在用户缓冲区已满或者到达文件结尾时结束。

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。

29.6.8 OCIFileSeek（）

更改文件中的当前位置。

目的

更改文件中的当前位置。

句法

剑OCIFileSeek（void * hndl，OCIError * err，OCIFileObject * filep，uword origin，ubig_ora offset，sb1 dir）;

参数

hndl（IN）

OCI环境或用户会话句柄。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。可以通过调用OCIErrorGet()获取诊断信息。

filep（IN / OUT）

唯一引用该文件的文件标识符。

来源（IN）

寻求的起点。使用以下值之一：

OCI_FILE_SEEK_BEGINNING （开头）

OCI_FILE_SEEK_CURRENT （当前位置）

OCI_FILE_SEEK_END （文件结束）

偏移量（IN）

读取开始的原点的字节数。

目录（IN）

从原点走向的方向。

注意：

方向可以是OCIFILE_FORWARD或OCIFILE_BACKWARD 。

评论

此功能允许搜索超过文件末尾。从这样的位置读取会导致报告文件结束条件。写入这样的位置不适用于所有文件系统。这是因为某些系统不允许文件动态增长。它们要求文件预先分配固定大小。请注意，此函数执行对字节位置的搜索。

返回

OCI_SUCCESS ; OCI_INVALID_HANDLE ;或OCI_ERROR 。
