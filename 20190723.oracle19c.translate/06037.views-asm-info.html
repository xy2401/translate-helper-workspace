6使用视图显示Oracle ASM信息

动态视图显示有关Oracle ASM的重要信息。

本章包含有关使用动态视图显示Oracle自动存储管理（Oracle ASM）信息的信息。

  • 查看Oracle ASM文件访问控制信息

也可以看看：

  • Oracle Database Reference ，了解有关所有动态性能视图的信息

  • 使用ASMCMD管理Oracle ASM以获取显示Oracle ASM信息的ASMCMD ls*命令。

包含Oracle ASM信息的视图

表6-1中列出的视图提供了有关Oracle ASM的信息。

有关显示有关Oracle ASM的所有动态性能视图的信息，请参阅Oracle数据库参考

注意：

V$ASM_FILE视图中的REDUNDANCY_LOWERED列不再提供有关冗余减少的文件的信息。不推荐使用此列，它始终显示值U

表6-1 Oracle ASM动态视图

视图 描述

V$ASM_ALIAS

对于Oracle ASM实例装入的每个磁盘组中的每个别名，都包含一行。

V$ASM_ATTRIBUTE

为每个定义的属性显示一行。除了CREATE DISKGROUP和ALTER DISKGROUP语句指定的属性外，视图还可以显示自动创建的其他属性。仅对COMPATIBLE.ASM设置为11.1或更高的磁盘组显示属性。

V$ASM_AUDIT_CLEAN_EVENTS

显示有关审计跟踪清除或清除事件历史记录的信息。

V$ASM_AUDIT_CLEANUP_JOBS

显示有关已配置的审计跟踪清除作业的信息。

V$ASM_AUDIT_CONFIG_PARAMS

显示有关当前配置的审计跟踪属性的信息。

V$ASM_AUDIT_LAST_ARCH_TS

显示有关为审计跟踪清除或清除设置的上次存档时间戳的信息。

V$ASM_CLIENT

在Oracle ASM实例中，使用Oracle ASM实例管理的磁盘组标识数据库。

在Oracle数据库实例中，如果数据库具有任何打开的Oracle ASM文件，则包含有关Oracle ASM实例的信息。

V$ASM_DBCLONE_INFO

在Oracle ASM实例中，显示源数据库，克隆数据库及其文件组之间的关系。

V$ASM_DISK

对于Oracle ASM实例发现的每个磁盘，包含一行，包括不属于任何磁盘组的磁盘。

每次查询时，此视图都会执行磁盘发现。由于执行发现是非常耗费资源的，因此不建议将此视图用于监视脚本。

V$ASM_DISK_IOSTAT

显示有关每个Oracle ASM客户端的磁盘I / O统计信息的信息。

在Oracle数据库实例中，仅显示该实例的行。

V$ASM_DISK_STAT

包含与V$ASM_DISK相同的列，但为了减少开销，在查询时不执行发现。V$ASM_DISK_STAT仅返回有关存储系统中已装入磁盘组的任何磁盘的信息。

建议使用V$ASM_DISK_STAT视图来监视脚本。

要显示有关所有磁盘的信息，请改用V$ASM_DISK 。

V$ASM_DISKGROUP

描述磁盘组（数量，名称，大小相关信息，状态和冗余类型）。

每次查询时，此视图都会执行磁盘发现。由于执行发现是非常耗费资源的，因此不建议将此视图用于监视脚本。

V$ASM_DISKGROUP_STAT

包含与V$ASM_DISKGROUP相同的列，但为了减少开销， V$ASM_DISKGROUP_STAT在查询时不执行发现。

建议使用V$ASM_DISKGROUP_STAT视图来监视脚本。

要显示有关所有磁盘的更完整信息，请使用V$ASM_DISKGROUP 。

V$ASM_ESTIMATE

显示Oracle ASM磁盘组重新平衡和重新同步操作的执行计划中涉及的工作估计。

V$ASM_FILE

Oracle ASM实例装入的每个磁盘组中的每个Oracle ASM文件都包含一行。

V$ASM_FILEGROUP

对于Oracle ASM实例装入的每个磁盘组中的每个文件组，都包含一行。

V$ASM_FILEGROUP_FILE

对于与Oracle ASM实例装入的每个磁盘组中的文件组关联的每个文件，包含一行。

V$ASM_FILEGROUP_PROPERTY

包含与文件组关联的每个属性的一行或Oracle ASM实例装入的每个磁盘组中每个文件组的每种文件类型。

V$ASM_OPERATION

在Oracle ASM实例中，对于在Oracle ASM实例中执行的每个活动的Oracle ASM长时间运行操作，都包含一行。

在Oracle数据库实例中，不包含任何行。

V$ASM_QUOTAGROUP

对于节点上的Oracle ASM实例发现的每个Oracle自动存储管理（Oracle ASM）配额组，包含一行。

V$ASM_TEMPLATE

对于Oracle ASM实例装入的每个磁盘组中的每个模板，都包含一行。

V$ASM_USER

包含已连接数据库实例的有效操作系统用户名和文件所有者的名称。

V$ASM_USERGROUP

包含每个Oracle ASM文件访问控制组的创建者。

V$ASM_USERGROUP_MEMBER

包含每个Oracle ASM文件访问控制组的成员。

查询V$ASM视图时，磁盘组编号的值不是静态值。安装磁盘组时，会选择磁盘组编号。此数字可能会在磁盘组安装中发生更改。磁盘组编号不会记录在任何持久性结构中，但可以在V$ASM视图的GROUP_NUMBER列中查看当前值。

查看Oracle ASM文件访问控制信息

您可以在V$ASM_USER ， V$ASM_USERGROUP ， V$ASM_USERGROUP_MEMBER和V$ASM_FILE视图的列中查看有关Oracle ASM文件访问控制的信息。

有关Oracle ASM文件访问控制的更多信息，请参阅管理磁盘组的Oracle ASM文件访问控制 。

示例6-6使用V $ ASM_USER查看Oracle ASM文件访问控制信息

此示例显示有关V$ASM_USER视图中显示的Oracle ASM文件访问控制用户的信息。 

SELECT dg.name AS diskgroup，u.group_number，u.user_number，u.os_id，u.os_name FROM V $ ASM_DISKGROUP dg，V $ ASM_USER u WHERE dg.group_number = u.group_number AND dg.name ='DATA'; DISKGROUP GROUP_NUMBER USER_NUMBER OS_ID OS_NAME --------------- ------------ ----------- -----  - ----- DATA 1 1 1001 oracle1 DATA 1 2 1002 oracle2 DATA 1 3 1003 grid

SELECT dg.name AS diskgroup，ug.group_number，ug.owner_number，u.os_name，ug.usergroup_number，ug.name FROM V $ ASM_DISKGROUP dg，V $ ASM_USER u，V $ ASM_USERGROUP ug WHERE dg.group_number = ug.group_number AND dg.name ='DATA'和ug.owner_number = u.user_number; DISKGROUP GROUP_NUMBER OWNER_NUMBER OS_NAME USERGROUP_NUMBER NAME ----------------- ------------ ------------ --- ------------ ---------------- -------- DATA 1 3 grid 1 asm_data

SELECT dg.name AS diskgroup，um.group_number，um.member_number，u.os_name，um.usergroup_number，ug.name FROM V $ ASM_DISKGROUP dg，V $ ASM_USER u，V $ ASM_USERGROUP_MEMBER um，V $ ASM_USERGROUP ug WHERE dg.group_number = um.group_number AND dg.group_number = ug.group_number AND dg.group_number = u.group_number AND dg.name ='DATA'AND um.member_number = u.user_number AND um.usergroup_number = ug.usergroup_number; DISKGROUP GROUP_NUMBER MEMBER_NUMBER OS_NAME USERGROUP_NUMBER NAME --------------- ------------ ------------- ---- -------------- ---------------- -------- DATA 1 1 oracle1 1 asm_data DATA 1 2 oracle2 1 asm_data
