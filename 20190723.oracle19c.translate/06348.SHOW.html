3.15 SHOW

目的

使用SHOW命令显示用于为一个或多个数据库设置当前RMAN配置的CONFIGURE命令。RMAN默认配置以#default为后缀。

先决条件

仅在RMAN提示符下执行此命令。必须满足以下任一条件：

  • RMAN必须连接到目标数据库，该数据库必须安装或打开。

  • 必须将RMAN连接到恢复目录，并且必须已运行SET DBID 。

句法

show :: =


说明show.eps的描述

（ deviceSpecifier :: = ）

forDbUniqueNameOption :: =


说明fordbuniquenameoption.eps的说明

语义

语法元素 描述

所有

显示所有用户输入的CONFIGURE命令和默认配置。

ARCHIVELOG备份副本

显示当前配置的归档重做日志备份双工程度。

ARCHIVELOG删除政策

显示CONFIGURE ARCHIVELOG DELETION POLICY设置。

AUXNAME

显示CONFIGURE AUXNAME设置。

BACKUP OPTIMIZATION

显示CONFIGURE BACKUP OPTIMIZATION设置： ON或OFF （默认值）。

[AUXILIARY] CHANNEL

显示CONFIGURE CHANNEL设置。您可以指定普通频道或AUXILIARY频道。

FOR DEVICE TYPE deviceSpecifier

指定通道的设备类型。例如， SHOW CHANNEL FOR DEVICE TYPE DISK仅显示磁盘通道的通道设置。

压缩算法

显示配置的备份压缩算法。

CONTROLFILE AUTOBACKUP

显示CONFIGURE CONTROLFILE AUTOBACKUP设置： ON或OFF 。

格式

显示已配置设备的控制文件自动备份文件的格式。

DATAFILE BACKUP COPIES

显示CONFIGURE ...BACKUP COPIES设定的数据文件： 1 ， 2 ， 3 ，或4 。

DB_UNIQUE_NAME

显示恢复目录已知的DB_UNIQUE_NAME值。

[DEFAULT] DEVICE TYPE

显示已配置的设备类型和并行度设置。如果指定了DEFAULT ，则SHOW显示默认设备类型和设置。

加密

当与ALGORITHM或FOR {DATABASE | TABLESPACE}一起使用时，显示数据库中的数据库或表空间当前配置的加密设置。 FOR {DATABASE | TABLESPACE} 。

算法

显示编写加密备份集时用于加密的已配置默认算法。可能的值列在V$RMAN_ENCRYPTION_ALGORITHMS 。

对于数据库

显示数据库的当前加密设置。

对于TABLESPACE

显示每个表空间的当前加密设置。

排除

仅显示您排除的表空间。

MAXSETSIZE

显示CONFIGURE MAXSETSIZE设置。

RETENTION POLICY

显示当前目标数据库的CONFIGURE RETENTION POLICY的设置。

SNAPSHOT CONTROLFILE NAME

显示CONFIGURE SNAPSHOT CONTROLFILE设置。

forDbUniqueNameOption

显示唯一命名数据库的恢复目录中的配置，即使RMAN未作为TARGET连接到此数据库也是如此。您可以使用db_unique_name指定数据库，也可以对所有唯一命名的数据库使用ALL 。

数据库的唯一名称是其DB_UNIQUE_NAME初始化参数设置的值。FOR DB_UNIQUE_NAME子句可用于显示Data Guard环境中备用数据库的配置。

RMAN必须连接到恢复目录。必须将RMAN连接到已装入的目标数据库，或者必须使用SET DBID标识目标数据库。例如，您可以使用目标数据库的DBID运行SET DBID ，然后显示恢复目录中已知的所有备用数据库的配置（请参阅示例2-56 ）。

另请参阅： forDbUniqueNameOption ，以获取此子句中选项的说明

RMAN OUTPUT

显示RMAN输出日志记录的当前配置值。
