4.1 Oracle错误代码故障排除

本节包含有关您可能遇到的特定Oracle错误消息的疑难解答信息，例如：

  • ORA-01045：用户GSMADMIN_INTERNAL缺少CREATE SESSION权限;登录被拒绝
  • ORA-12514：TNS：监听器当前不知道连接描述符中请求的服务
  • ORA-12516：TNS：侦听器找不到具有匹配协议栈的可用处理程序
  • ORA-12541：TNS：没有倾听者
  • GSM-45034：未建立与GDS目录的连接
  • GSM-45054：GSM错误或NET-40006：无法启动GSM

父主题： 全局数据服务故障排除

4.1.1 ORA-01045：用户GSMADMIN_INTERNAL缺少CREATE SESSION权限;登录被拒绝

用户GSMADMIN_INTERNAL是内部唯一用户，永远不应解锁或用于任何数据库登录。除非Oracle技术支持人员指示，否则不应对Global Data Services架构对象进行任何直接修改。

父主题： Oracle错误代码故障排除

4.1.2 ORA-12514：TNS：监听器当前不知道连接描述符中请求的服务

全球服务可能会下降。验证池数据库已启动且服务是否已启动。

可以禁用全局服务。确保池数据库已启动且服务已启用并已启动。

GDS池数据库可能已关闭。确保GDS池数据库已启动且服务已启用并已启动。

父主题： Oracle错误代码故障排除

4.1.3 ORA-12516：TNS：侦听器找不到具有匹配协议栈的可用处理程序

GDS池数据库的本地侦听器可能已关闭。确保GDS池数据库本地侦听器正在运行。

父主题： Oracle错误代码故障排除

4.1.4 ORA-12541：TNS：没有听众

所有全球服务经理都可能失败。验证全局服务管理器是否正在运行。

父主题： Oracle错误代码故障排除

4.1.5 GSM-45034：未建立与GDS目录的连接

GDS目录数据库或其侦听器可能已关闭。验证GDS目录数据库及其本地侦听器是否正在运行。

父主题： Oracle错误代码故障排除

4.1.6 GSM-45054：GSM错误或NET-40006：无法启动GSM

GDS目录数据库或其侦听器可能已关闭。验证GDS目录数据库及其本地侦听器是否正在运行。

4.2.1没有全局服务管理器运行时连接到GDS配置数据库

您需要多个地址列表;第一个列表应该是区域全局服务管理器侦听器，第二个列表包含伙伴区域的全局服务管理器侦听器，第三个列表包含本地侦听器。

您可以随时通过全局服务管理器进行连接，并且只有在所有全局服务管理器侦听器关闭时才会故障转移到本地侦听器。

模板：

（DESCRIPTION =（FAILOVER = on）（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（global_protocol_address_information））（ADDRESS =（global_protocol_address_information））（ADDRESS =（global_protocol_address_information）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（ global_protocol_address_information））（ADDRESS =（global_protocol_address_information））（ADDRESS =（global_protocol_address_information）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（local_protocol_address_information））（ADDRESS =（local_protocol_address_information）））（CONNECT_DATA =（SERVICE_NAME = global_service_name）（ REGION = REGION_NAME）））

例：

（DESCRIPTION =（FAILOVER = on）（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（HOST = gsmhost1）（PORT = 1523）（PROTOCOL = TCP））（ADDRESS =（HOST = gsmhost2）（PORT = 1523）（协议） = TCP））（ADDRESS =（HOST = gsmhost3）（PORT = 1523）（PROTOCOL = TCP）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（HOST = gsmhost4）（PORT = 1523）（PROTOCOL = TCP） ）（ADDRESS =（HOST = gsmhost5）（PORT = 1523）（PROTOCOL = TCP））（ADDRESS =（HOST = gsmhost6）（PORT = 1523）（PROTOCOL = TCP）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS） =（HOST = server1）（PORT = 1521）（PROTOCOL = TCP）））（CONNECT_DATA =（SERVICE_NAME = sales_read_service.dbpoolora.oradbcloud）（REGION = WEST）））

注意：

对于启用了Oracle RAC的GDS数据库，第三个地址列表包含本地Oracle RAC数据库的SCAN侦听器。

4.2.2没有全局服务管理器运行时连接到目录数据库

即使全局服务管理器关闭，本地侦听器也可以访问GDS目录数据库。

当全局服务管理器未运行时，目录数据库上的任何数据库管理/维护活动可能需要此访问权限。

4.5在全局服务管理器上启用和禁用跟踪

您可以使用set trace_level命令启用跟踪。 

GDSCTL>设置trace_level -gsm gsm_name支持

SUPPORT选项提供跟踪Oracle支持服务的故障排除信息。其他选项是ADMIN和USER 。

要禁用跟踪：

GDSCTL>设置trace_level -gsm gsm_name OFF

4.7在GDS环境中使用SYS_CONTEXT参数

对于使用与全局服务的连接建立的会话，名称空间USERENV的某些参数的值与连接到同一数据库上的本地服务时设置的值不同。设置全局服务的不同值以使数据库池作为具有许多实例的单个数据库显示给客户端。这样做是为了提供与12c之前客户端的向后兼容性，这些客户端期望服务的多个实例仅存在于Oracle RAC数据库中。

当客户端连接到全局服务时，GDS会在会话上下文中以不同方式设置以下内容。

  • DB_UNIQUE_NAME和DB_DOMAIN设置为<gdspool_name>。<config_name>

  • INSTANCE设置为系统生成的数字<inst_num>，在GDS配置中是唯一的

  • INSTANCE_NAME设置为<gdspool_name>％<virtual_instance_num>

  • SERVICE_NAME设置为<region_name>％<service_name>
