关于安装后配置文件

运行静默或响应文件安装时，在响应文件中提供有关服务器的信息，否则在图形用户界面安装期间手动提供这些信息。

但是，响应文件不包含配置助手在完成软件安装后需要的用户帐户密码。配置助手使用名为configToolAllCommands的脚本启动。您可以使用密码响应文件以响应文件模式运行此脚本。该脚本使用密码连续运行配置工具以完成配置。

如果保留用于克隆安装的密码文件，则Oracle强烈建议您将密码文件存储在安全位置。此外，如果必须停止安装以修复错误，则可以使用configToolAllCommands和密码响应文件运行配置助手。

configToolAllCommands密码响应文件具有以下语法选项：

  • oracle.crs for Oracle Grid Infrastructure组件或oracle.server for Oracle Database组件，配置助手配置

  • variable_name是配置文件变量的名称

  • value是用于配置的所需值。

命令语法如下：

internal_component_name |变量名=值

例如：

oracle.crs | S_ASMPASSWORD =密码

数据库配置助手需要SYS，SYSTEM和DBSNMP密码才能与Oracle DBCA一起使用。您可能需要指定以下附加密码，具体取决于您的系统配置：

  • 如果数据库使用Oracle自动存储管理（Oracle ASM）进行存储，则必须为S_ASMSNMPPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

  • 如果使用一个或多个可插拔数据库（PDB）创建多租户容器数据库（CDB），则必须为S_PDBADMINPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

Oracle强烈建议您使用密码响应文件来维护安全性：

  • 响应文件的权限应设置为600。

  • 响应文件的所有者应该是安装所有者用户，并将该组设置为中央库存（ oraInventory ）组。
