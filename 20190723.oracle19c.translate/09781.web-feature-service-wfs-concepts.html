16 Web功能服务（WFS）支持

Oracle Spatial and Graph包括Web Feature Service（WFS）支持。

注意：

在使用WFS之前，请确保您了解空间Web服务简介中描述的概念，并且已执行该章中所述的任何必要的配置工作。

如果您拥有使用一个或多个SYS.XMLTABLEINDEX索引编制索引的先前版本的数据，则必须在升级之前删除关联的索引，并在升级后重新创建索引，如升级前后的索引维护中所述（ WFS和CSW） 。

16.1 WFS引擎

本主题描述Web Feature Service引擎，包括它与客户端和数据库服务器的关系。

WFS作为Java Web应用程序实现，可以部署在WebLogic 12.1.3或更高版本中。所需的Java版本是JDK 1.8或更高版本。

WFS有一个元数据层，它在数据库中存储了回复WFS请求所需的元数据。元数据包括空间列，可以使用Oracle Spatial和Graph接口查询和处理这些列。元数据还存储要素的非空间和空间属性的关联，以及Web要素服务为其客户提供的服务。

图16-1显示了WFS架构。

图16-1 Web功能服务体系结构


“图16-1 Web要素服务体系结构”的说明

如图16-1所示：

  • WFS服务器是中间层的一部分。

  • WFS客户端可以通过SOAP，KVP或XML格式的请求和响应与WFS服务器通信。

  • WFS服务器通过对数据库的JDBC调用来执行空间数据和元数据访问。

  • 该数据库包括带有WFS元数据和空间数据的Oracle Spatial and Graph，以及用于管理操作的PL / SQL包（请参阅SDO_WFS_PROCESS包（WFS处理）和SDO_WFS_LOCK包（WFS） ）。

    每个数据库可以有一个WFS架构，它对应于WebLogic Server中的一个数据源。

父主题： Web功能服务（WFS）支持

16.2.1编辑WFSConfig.xml文件

在WFS管理控制台的“ 配置文件”选项卡中，“取消注释”并根据需要进行修改。考虑以下：

  • 日志记录级别可以是OFF，SEVERE，WARNING，INFO（默认值），CONFIG，FINE，FINER，FINEST或ALL。默认值为INFO。

    size_limit指定要写入任何一个文件的近似最大量（以MB为单位）。如果该值为零，则没有限制。默认值为10

    file_count指定要循环的输出文件的数量。默认值为10。

  • 代理配置允许您控制Spatial Web Services服务器生成的URL。一个有用的场景是HTTP代理服务器接收来自用户的请求并将它们传递给WebLogic Server，以控制和保护对专用网络上的服务器的访问

  • （可选）取消注释wfs_query_timeout元素以指定查询超时值，该值在调用服务器端锁定API时使用。该值可以是非负整数，其单位为秒。默认值为10秒。

  • （可选）取消注释wfs_lock_expiry元素以配置默认WFS锁定到期值，如果未在GetFeatureWithLock或LockFeature请求中显式指定锁定到期值，则为WFS锁定的到期时间（以分钟为单位）。默认值为4分钟。

  • （可选）取消注释gml_consider_coordinate_axis_sequence_ordering并将其设置为值1，如果应检查空间参照系（SRS）是否需要在生成GML时对x，y坐标反转纵坐标。

  • wfs_xsd_loc_url ， wfs_ex_xsd_loc_url和gml3_xsd_loc_url元素已弃用，不再使用。

16.2.2 WFS引擎的数据源设置

要设置WFS引擎的数据源，请执行以下步骤。

  1. 登录WebLogic Server控制台。

  2. 在左窗格中，对于“ 域结构” ，选择“ Data Sources 。

  3. 单击“ 新建” ，然后单击“ 通用数据源”

  4. 在“创建新JDBC数据源”页面上，输入：

    1. 名称 ：任何没有空格的名称

    2. JNDI名称 ：任何以jdbc/开头的名称

    3. 数据库类型 ： Oracle

  5. 对于Database Driver ，选择Oracle Driver (Thin) for Instance connections ;对于版本 ，请选择Any 。

  6. Skip支持全局事务。

  7. 在“连接属性”页面上，输入：

    1. 数据库名称

    2. 主机名

    3. 港口

    4. 数据库用户名

    5. 密码

  8. 在下一页上，测试连接。

  9. 如果连接成功，请为此部署选择适当的服务器名称： AdminServer或受管服务器名称。

    每个数据库只能为WFS配置一个模式，这对应于WebLogic Server中的一个数据源。但是，您可以在WebLogic中为WFS配置多个数据源。可以通过不同的URL访问每个数据源，其中URL的最后部分对应于WebLogic Server中配置的数据源名称。以下是WLS数据源名称wfsdata1的示例链接：

    http：// localhost：80 / oraclespatial / wfs / wfsdata1？service = WFS＆version = 1.0.0＆request = GetCapabilities
    10.

16.3管理要素类型

               

                  
WFS支持具有空间和非空间属性的要素类型。

                  
要素类型将数据库表的内容公开为要素实例。功能类型非常适合使用Oracle Spatial and Graph管理其地理空间数据并使用Oracle数据库管理其他业务数据的用户。Spatial和Graph WFS实现提供了访问数据的方法，尤其是在使用Web服务实现的面向服务的体系结构（SOA）系统中。

16.4能力文件（WFS）

               

                  
WFS服务器生成功能文档以响应GetCapabilities请求。它显示已发布的要素类型（例如道路或河流）和支持的操作（例如插入和删除）。

                  
WFS服务器使用功能模板，并将有关功能类型和操作的信息添加到此模板以创建功能文档。

                  
客户端使用HTTP GET / KVP协议来访问此功能文档：
http：// < hostname：port > / oraclespatial / wfs / < 数据源名称 > /？请求=＆的GetCapabilities服务= WFS
在上面的格式中：

                  
    
                     
  • 
                        
    hostname是运行应用程序服务器的系统的主机名。
                        
    
                     
    • 
                     
  • 
                        
    port是运行应用程序服务器的端口号。
                        
    
                     
    • 
                     
  • 
                        
    oraclespatial是安装Spatial and Graph Web服务应用程序的默认上下文根。
                        
    
                     
    • 
                     
  • 
                        
    data source name在应用程序服务器上配置的数据源，用于访问空间数据。可以在WLS实例中配置多个数据源，WFS可以通过专用URL公开每个数据源的数据

WFS管理控制台使用您的WebLogic Server凭据。

                  
在使用任何Oracle Spatial Service页面之前，请从所有可用数据源名称列表中选择数据源。（当前选择的数据源显示在右上角，您可以随时在其中进行更改。）

                  
WFS选项卡或菜单允许您管理WFS功能类型。它具有配置文件（适用于WFSConfig.xml ），功能，模板，发布功能，测试和日志的子选项卡。您可以：

    • 
                     
  • 
                        
    使用“配置文件”选项卡更新WFSConfig.xml。
    
                     
    • 
                     
  • 
                        
    使用“功能模板”选项卡更新GetCapabilities模板。
    
                     
    • 
                     
  • 
                        
    使用“测试”选项卡基于空间和非空间约束获取或查询要素。
    
                     
    • 
                     
  • 
                        
    使用“测试”选项卡创建新的要素实例（插入操作）。
    
                     
    • 
                     
  • 
                        
    使用“测试”选项卡删除要素实例。
    
                     
    • 
                     
  • 
                        
    使用“测试”选项卡更新要素实例。
    
                     
    • 
                     
  • 
                        
    使用“日志”选项卡检查WFS日志文件。
    
                        
    “日志”选项卡允许您选择要显示的文件。您可以单击“ 刷新”以查看自加载屏幕以来生成的新日志消息。您还可以下载zip文件格式的特定日志文件。

16.7诊断WFS问题

               

                  
WFS日志文件提供诊断信息。

                  
这些日志文件位于配置文件夹的log目录中。在WFS管理控制台中 ，您可以使用WFS的“日志”选项卡查看和下载WFS日志文件。
                  

                  
本主题介绍了某些类型的日志消息以及如何处理它们。

                  

                     
“未找到DataSource jdbc / wfs_admin_ds” -  GetCapabilities响应错误消息

                     
响应可能类似于以下内容（为了便于阅读而重新格式化）：
<？xml version ='1.0'coding ='UTF-8'？> <ows：ExceptionReport xmlns：ows =“http://www.opengis.net/ows/2.0”xmlns：xsi =“http://www.w3.org/2001/XMLSchema-instance”xsi：schemaLocation =“ http://www.opengis.net/ows/2.0 http://schemas.opengis.net/ows/2.0/owsExceptionReport.xsd“version =”2.0.1“> <ows：异常exceptionCode =”NoApplicableCode“locator = “未找到DataSource jdbc / wfs_admin_ds”/> </ ows：ExceptionReport>
这意味着未配置WFS数据源或WebLogic Server无法连接到数据库。有关配置WFS数据源的信息，请参阅WFS引擎的数据源设置 。
                     

                  

                  
                  

                     
指示WFS元数据的异常未正确填充

                     
如果您怀疑WFS元数据可能未正确填充，则以具有足够权限的用户身份进行连接，请检查：

                     
    
                        
  • 
                           
    功能类型已发布并显示在MDSYS.WFS_FeatureType $表中。（此表由WFS服务器使用。）
    
                        
    • 
                        
  • 
                           
    如果使用自定义SRS名称前缀，则正确填充MDSYS.WFS_Srs $表。
    
                        
    • 
                     

                  

                  
                  

                     
新发布的要素类型未出现在任何回复中

                     
如果任何已发布的要素类型未出现在任何响应中，请确保在发布新要素类型时使用了SDO_WFS_PROCESS.InsertFtMDUpdated 。
                     

                     
您还可以使用SDO_WFS_PROCESS.Publish_FeatureTypes_In_Schema过程在WFS架构中发布所有要素类型。

16.8将WFS与Oracle Workspace Manager一起使用

               

                  
您可以使用Oracle Workspace Manager对具有关系功能的WFS表进行版本启用。

                  
为此，首先使用SDO_WFS_LOCK.RegisterFeatureTable过程注册WFS表;然后执行DBMS_WM.EnableVersioning过程。（有关Workspace Manager的信息，包括DBMS_WM PL / SQL包的参考文档，请参阅Oracle Database Workspace Manager开发人员指南 。）
                  

                  
您可以使用WFS-T（Web要素服务事务）接口创建工作空间并对这些工作空间执行事务性WFS更改。但是，要使用WFS-T以外的接口，必须使用在WFS表上启用数据库事务的SQL * Plus会话。这些数据库事务包括以下内容：

                  
    
                     
  • 
                        
    更新和删除WFS表上的操作
    
                     
    • 
                     
  • 
                        
    工作区维护操作，例如刷新工作区或合并工作区
    
                     
    • 
                  

                  
要在WFS表上启用数据库事务，请调用SDO_WFS_LOCK.EnableDBTxns过程（记录在SDO_WFS_LOCK包（WFS）中 ）。执行此过程后，WFS表上允许数据库事务，并为WFS事务维护WFS-T语义，直到会话结束。
