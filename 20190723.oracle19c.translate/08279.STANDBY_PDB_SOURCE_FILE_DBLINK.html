1.328 STANDBY_PDB_SOURCE_FILE_DBLINK

STANDBY_PDB_SOURCE_FILE_DBLINK指定将用于尝试从数据库链接指向的源PDB复制数据文件的数据库链接的名称。

Oracle RAC

可以在不同的实例上设置不同的值。

在Oracle Database 18c之前的Oracle数据库版本中，如果通过从同一CDB（本地克隆）克隆PDB来创建PDB，则在备用数据库上会自动从源PDB复制数据文件。但是，当PDB创建为远程克隆时，用户负责将数据文件复制到备用数据库上的Oracle Managed Files（OMF）位置。

为了解决这一缺陷， STANDBY_PDB_SOURCE_FILE_DBLINK参数指定将用于尝试从数据库链接指向的源PDB复制数据文件的数据库链接的名称。仅当数据库链接指向源PDB且源PDB以只读模式打开时，才会执行文件复制。否则，用户仍负责将数据文件复制到备用数据库上的OMF位置。

此参数还可用于在备用数据库中不存在源PDB文件的情况下尝试复制文件。例如，可以使用standbys = NONE创建源PDB。在这种情况下，如果存在设置为主数据库的数据库链接，则STANDBY_PDB_SOURCE_FILE_DBLINK参数可以使备用数据库从主数据库中复制文件。
