目的

SQL / JSON函数JSON_ARRAY将一系列SQL标量表达式或一个集合类型实例VARRAY或NESTED TABLE作为其输入。

它将每个表达式转换为JSON值，并返回包含这些JSON值的JSON数组。

如果ADT的成员是一个集合而不是类型映射，则为ADT创建一个JSON对象，并为集合成员添加嵌套的JSON数组。

如果集合包含ADT实例，则类型映射将创建JSON对象的JSON数组。

JSON_ARRAY_content

使用此子句定义JSON_ARRAY函数的输入。

JSON_ARRAY_element

  • EXPR

    对于expr ，您可以指定任何SQL表达式，其值为JSON对象，JSON数组，数字文字，文本文字，日期，时间戳或null。此函数将数字文字转换为JSON数字值，将文本文字转换为JSON字符串值。日期和时间戳数据类型将按照ISO 8601日期格式作为JSON字符串打印在生成的JSON对象或数组中。

  • format_clause

    您可以指定FORMAT JSON以指示输入字符串是JSON，因此不会在输出中引用。

JSON_on_null_clause

当expr求值为null时，使用此子句指定此函数的行为。

  • NULL ON NULL - 如果指定此子句，则该函数返回JSON null值。

  • ABSENT ON NULL - 如果指定此子句，则该函数将省略JSON数组中的值。这是默认值。

JSON_returning_clause

使用此子句指定返回值的类型。其中一个：

  • VARCHAR2将大小指定为字节数或字符数。默认值为字节。如果省略此子句，或指定子句而未指定size值，则JSON_ARRAY返回VARCHAR2(4000)类型的字符串。有关更多信息，请参阅VARCHAR2数据类型 。请注意，在SQL中的其他位置指定VARCHAR2数据类型时，您需要指定大小。但是，在JSON_returning_clause您可以省略大小。

  • CLOB返回包含单字节或多字节字符的字符大对象。

  • BLOB返回AL32UTF8字符集的二进制大对象。

严格

指定STRICT子句以验证JSON生成函数的输出是否正确JSON。如果检查失败，则会引发语法错误。

有关示例，请参阅JSON_OBJECT 。

例子

以下示例从JSON对象，JSON数组，数字文本，文本文字和null构造JSON数组：

SELECT JSON_ARRAY（JSON_OBJECT（'percentage'VALUE .50），JSON_ARRAY（1,2,3），100，'California'，null NULL ON NULL）“JSON数组示例”FROM DUAL; JSON数组示例----------------------------------------------- --------------------------------- [{“percentage”：0.5}，[1,2,3]， 100， “加州”，空]
