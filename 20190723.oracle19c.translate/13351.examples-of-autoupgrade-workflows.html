AutoUpgrade工作流程的示例

要指导升级，请查看与升级用例匹配的AutoUpgrade工作流示例。

这些示例以典型的工作流程序列呈现。

  • 在同一服务器上使用源和目标数据库主目录自动升级
    当您的Oracle数据库源和目标Oracle主目录位于同一物理服务器上，并且两个Oracle主目录都已启用且可用时，请使用此示例。
  • 使用不同服务器上的源和目标数据库主目录自动升级
    在自动升级Oracle数据库源和目标Oracle主目录位于不同物理服务器上时，请使用此示例。

在同一服务器上使用源和目标数据库主目录自动升级

当您的Oracle数据库源和目标Oracle主目录位于同一物理服务器上，并且两个Oracle主目录都已启用且可用时，请使用此示例。

上下文：源和目标主目录位于同一服务器上。

要开始分析，请输入以下命令。

java -jar autoupgrade.jar -config config.txt -mode analyze

该命令生成一个报告，指示命令找到的任何错误情况。查看错误情况。相关时，请自定义AutoUpgrade创建的修正以解决错误情况。

要开始部署升级，请输入以下命令：

java -jar autoupgrade.jar -config config.txt -mode deploy

使用不同服务器上的源和目标数据库主目录自动升级

在自动升级Oracle数据库源和目标Oracle主目录位于不同物理服务器上时，请使用此示例。

上下文：源和目标Oracle主目录位于不同的物理服务器上。

要开始分析，请输入以下命令。

java -jar autoupgrade.jar -config config.txt -mode analyze

该命令生成一个报告，指示命令找到的任何错误情况。查看错误情况。相关时，请自定义AutoUpgrade创建的修正以解决错误情况。

由于源数据库和目标Oracle数据库Oracle主目录位于不同的服务器上，因此您需要为每个服务器运行修正。

  1. 在源服务器上运行fixup：

    java -jar autoupgrade.jar -config config.txt -mode fixups

  2. 执行相关任务以将源Oracle数据库从源服务器移动到目标服务器。例如，使用Oracle Data Pump或Oracle Recovery Manager（RMAN）等技术将数据库文件从源服务器复制到目标服务器。

  3. 在目标服务器上，以升级模式启动数据库，然后在upgrade模式下运行AutoUpgrade：

    java -jar autoupgrade.jar -config config.txt -mode upgrade
