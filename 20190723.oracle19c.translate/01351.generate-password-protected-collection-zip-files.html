2.13生成受密码保护的收集zip文件的命令行选项

使用本节中的命令列表来加密和解密诊断集合zip文件。

表2-4加密和解密诊断收集zip文件

选项 描述

./orachk -d start -encryptzip

./exachk -d start -encryptzip

使用–encryptzip选项启动守护程序。

守护程序在启动时会提示输入密码。然后，守护程序使用该密码加密后续按需和计划运行集合。

注意：

当-encryptzip通过，甲骨文ORAchk和Oracle EXAchk成功后加密诊断收集zip文件中删除集合目录。

./orachk [-option value ] -encryptzip

./exachk [-option value ] -encryptzip

加密运行结果。

提示输入密码，并使用该密码加密在运行结束时创建的集合。

您可以使用-encryptzip与产生集合其他Oracle ORAchk和Oracle EXAchk选项。

例如：
./orachk -profile profile-name -encryptzip ./orachk -profile sysadmin -encryptzip
./orachk -check check-id -encryptzip ./orachk -check D47661C55B1A291AE0431EC0E50A5C53

注意：

当-encryptzip通过，甲骨文ORAchk和Oracle EXAchk成功后加密诊断收集zip文件中删除集合目录。

./orachk zip_file

./exachk zip_file

加密已生成的集合。

提示输入密码，加密使用该密码指定的zip文件，然后将集合重命名为，例如orachk_ host _ db _encrypted_ date _ time .zip 。

注意：

当-encryptzip通过，甲骨文ORAchk和Oracle EXAchk成功后加密诊断收集zip文件中删除集合目录。

./orachk -decryptzip zip_file

./exachk -decryptzip zip_file

解密加密的集合。

提示输入密码，解密用该密码指定的zip文件，然后将集合重命名为，例如orachk_ host _ db _ date _ time .zip 。
