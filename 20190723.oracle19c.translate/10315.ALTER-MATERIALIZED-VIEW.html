改变物质化的视图

目的

物化视图是包含查询结果的数据库对象。查询的FROM子句可以命名表，视图和其他物化视图。这些源对象统称为主表 （复制术语）或详细信息表 （数据仓库术语）。此引用使用术语主表来保持一致性。包含主表的数据库称为主数据库 。

使用ALTER MATERIALIZED VIEW语句可以通过以下一种或多种方式修改现有的物化视图：

  • 改变其存储特性

  • 更改其刷新方法，模式或时间

  • 改变它的结构，使它成为一种不同类型的物化视图

  • 启用或禁用查询重写

注意：

支持使用关键字SNAPSHOT代替MATERIALIZED VIEW以实现向后兼容。

也可以看看：

  • 有关创建实体化视图的更多信息，请创建MATERIALIZED VIEW

  • “Oracle数据库管理员指南” ，了解有关复制环境中的实例化视图的信息

  • Oracle数据库数据仓库指南 ，了解有关数据仓库环境中物化视图的信息

先决条件

物化视图必须位于您自己的架构中，或者您必须具有ALTER ANY MATERIALIZED VIEW系统特权。

要为查询重写启用实例化视图：

  • 如果物化视图中的所有主表都在您的模式中，那么您必须具有QUERY REWRITE权限。

  • 如果任何主表位于另一个模式中，则您必须具有GLOBAL QUERY REWRITE权限。

  • 如果物化视图位于另一个用户的模式中，那么您和该模式的所有者都必须具有相应的QUERY REWRITE权限，如前两个项中所述。此外，实例化视图的所有者必须具有对SELECT化视图所有者不拥有的任何主表的SELECT访问权限。

要在evaluation_edition_clause或unusable_editions_clause指定版本，您必须具有该版本的USE特权。

语义

模式

指定包含实例化视图的模式。如果省略schema ，则Oracle数据库假定物化视图位于您自己的模式中。

materialized_view

指定要更改的实例化视图的名称。

physical_attributes_clause

为PCTFREE ， PCTUSED和INITRANS参数指定新值（或者，在USING INDEX子句中USING ，仅为INITRANS参数指定）和物化视图的存储特性。有关PCTFREE ， PCTUSED和INITRANS参数的信息，请参阅ALTER TABLE ;有关存储特性的信息，请参阅storage_clause 。

modify_mv_column_clause

使用此子句可加密或解密实例化视图的此列。有关此子句的信息，请参阅CREATE TABLE子句encryption_spec 。

table_compression

使用table_compression子句指示Oracle数据库是否压缩数据段以减少磁盘和内存使用。有关此子句的完整语义，请参阅CREATE TABLE的table_compression子句。

inmemory_table_clause

使用inmemory_table_clause启用或禁用内存中列存储（IM列存储）的inmemory_table_clause化视图或其列，或更改实例化视图或其列的内存中属性。此子句与ALTER TABLE语句具有相同的语义。有关此子句的完整语义，请参阅ALTER TABLE的inmemory_table_clause 。

LOB_storage_clause

LOB_storage_clause允许您指定新LOB的存储特征。 LOB存储对物化视图的行为与对表的行为完全相同。有关LOB存储参数的信息，请参阅LOB_storage_clause （在CREATE TABLE ）。

modify_LOB_storage_clause

modify_LOB_storage_clause允许您修改LOB属性LOB_item或LOB对象属性的物理属性。LOB存储的修改对物化视图的行为与对表的行为完全相同。

也可以看看：

ALTER TABLE的modify_LOB_storage_clause ，以获取有关可以修改的LOB存储参数的信息

alter_table_partitioning

物化视图的分区子句的语法和一般功能与分区表的相同。请参阅ALTER TABLE文档中的alter_table_partitioning 。

改变物化视图分区的限制

您不能在任何partitioning_clauses指定LOB_storage_clause或modify_LOB_storage_clause 。

注意：

如果要使实例化视图的内容与主表的内容保持同步，则Oracle建议您在删除或截断表分区后手动执行依赖于表的所有实例化视图的完整刷新。

修改分区不可用的本地索引

使用此子句标记UNUSABLE与分区关联的所有本地索引partition 。

修改分区重建不可用的本地索引

使用此子句可重建与分区关联的不可用本地索引partition 。

parallel_clause

parallel_clause允许您更改实例化视图的默认并行度。

有关此子句的完整信息，请参阅CREATE TABLE文档中的parallel_clause 。

logging_clause

指定或更改实例化视图的日志记录特征。有关此子句的完整说明，请参阅logging_clause 。

allocate_extent_clause

allocate_extent_clause允许您为实例化视图显式分配新范围。有关此子句的完整说明，请参阅allocate_extent_clause 。

deallocate_unused_clause

使用deallocate_unused_clause显式释放实例化视图末尾的未使用空间，并使释放的空间可用于其他段。有关此子句的完整说明，请参阅deallocate_unused_clause 。

shrink_clause

使用此子句可压缩实例化视图段。有关此子句的完整信息，请参阅CREATE TABLE文档中的shrink_clause 。

CACHE | NOCACHE

对于将频繁访问的数据， CACHE指定在执行全表扫描时，为此表检索的块放置在缓冲区高速缓存中最近使用的LRU列表末尾。此属性对小型查找表很有用。NOCACHE指定将块放置在LRU列表的最近最少使用的末尾。有关此子句的更多信息，请参阅有关CREATE TABLE的文档中的“ CACHE | NOCACHE | CACHE READS ” 。

alter_iot_clauses

使用alter_iot_clauses更改索引组织的物化视图的特征。alter_iot_clauses的组件的关键字和参数具有与ALTER TABLE相同的语义，并具有以下限制。

改变指数组织物化观点的限制

你不能指定mapping_table_clause或prefix_compression的条款index_org_table_clause 。

也可以看看：

有关创建索引组织的物化视图的信息，请参阅 CREATE MATERIALIZED VIEW index_org_table_clause

使用INDEX条款

使用此子句可更改Oracle数据库用于维护物化视图数据的索引的INITRANS和STORAGE参数的值。

对USING INDEX条款的限制

您不能在本节中指定PCTUSED或PCTFREE参数。

修改scoped_table_ref_constraint

使用MODIFY scoped_table_ref_constraint子句将REF列或属性重新映射到新表或新列的别名。

Resive REF列的限制

您只能在每个ALTER MATERIALIZED VIEW语句中重新扫描一个REF列或属性，这必须是此语句中的唯一子句。

alter_mv_refresh

使用alter_mv_refresh子句更改默认方法和模式以及自动刷新的默认时间。如果修改了物化视图的主表的内容，则必须更新物化视图中的数据，以使物化视图准确反映当前在其主表中的数据。此子句允许您计划时间并指定Oracle数据库刷新实例化视图的方法和模式。

也可以看看：

  • 此子句仅设置默认刷新选项。有关实际实现刷新的说明，请参阅“ Oracle数据库管理员指南”和“ Oracle数据库数据仓库指南” 。

  • Oracle数据库数据仓库指南 ，了解如何使用刷新统计信息来监视物化视图刷新操作的性能

快速条款

为快速刷新方法指定FAST ，该方法根据主表发生的更改执行刷新。更改存储在与主表关联的物化视图日志中（对于传统的DML更改）或直接加载器日志（对于直接路径INSERT操作）。

对于传统的DML更改和直接路径INSERT操作，其他条件可能会限制物化视图的合格性以进行快速刷新。

在ALTER MATERIALIZED VIEW语句中将刷新方法更改为FAST时，Oracle数据库不会执行此验证。如果物化视图不符合快速刷新条件，则当您尝试刷新此视图时，Oracle数据库会返回错误。

完整的条款

为完整刷新方法指定COMPLETE ，该方法通过执行实例化视图的定义查询来实现。如果指定完整刷新，则即使可以快速刷新，Oracle数据库也会执行完全刷新。

FORCE条款

指定FORCE如果，当刷新时，您希望Oracle数据库进行快速刷新，如果一个是可能的，或者一个完整的刷新，否则。

ON COMMIT条款

如果要在Oracle数据库提交在实例化视图的主表上运行的事务时进行刷新，请指定ON COMMIT 。

您不能同时指定ON COMMIT和ON DEMAND 。如果指定ON COMMIT ，则不能同时指定START WITH或NEXT 。

对ON COMMIT的限制

仅对实体化连接视图和单表实现聚合视图支持此子句。

根据需求条款

如果希望通过调用三个DBMS_MVIEW刷新过程之一来DBMS_MVIEW刷新DBMS_MVIEW化视图，请指定ON DEMAND 。如果省略ON COMMIT和ON DEMAND ，则ON DEMAND是默认值。

您不能同时指定ON COMMIT和ON DEMAND 。 START WITH和NEXT优先于ON DEMAND 。因此，在大多数情况下，指定START WITH或NEXT时指定ON DEMAND没有意义。

从条款开始

指定START WITH date以指示第一个自动刷新时间的日期。

下一条款

指定NEXT以指示用于计算自动刷新之间间隔的日期表达式。

START WITH和NEXT值都必须评估到将来的某个时间。如果省略START WITH值，则Oracle数据库通过根据NEXT化视图的创建时间计算NEXT表达式来确定第一个自动刷新时间。如果指定START WITH值但省略NEXT值，则Oracle Database仅刷新实例化视图一次。如果省略START WITH和NEXT值，或者如果完全省略alter_mv_refresh ，则Oracle数据库不会自动刷新alter_mv_refresh化视图。

与主要条款

指定WITH PRIMARY KEY以将rowid实体化视图更改为主键物化视图。主键物化视图允许重新组织物化视图主表，而不会影响物化视图继续快速刷新的能力。

要指定此子句，主表必须包含已启用的主键约束，并且必须在其上定义记录主键信息的物化视图日志。

evaluation_edition_clause

使用此子句可更改实例化视图的评估版。该子句在CREATE MATERIALIZED VIEW和ALTER MATERIALIZED VIEW语句中具有相同的语义。有关此子句的完整信息，请参阅CREATE MATERIALIZED VIEW文档中的evaluation_edition_clause 。

关于更改物化视图评估版的注意事项

更改实体化视图的评估版时，以下注释适用：

  • 如果更改了提交时刷新实例化视图的评估版，则Oracle数据库将执行物化视图的完整刷新，除非您指定CONSIDER FRESH 。

  • 如果更改按需刷新实体化视图的评估版，则Oracle数据库会将实例化视图的过期状态设置为STALE除非您指定CONSIDER FRESH 。

  • 对于提交时刷新和按需刷新实例化视图：如果更改评估版并指定CONSIDER FRESH ，则Oracle数据库不会更新实例化视图的过期状态，也不会重建实例化视图。因此，您可以指定CONSIDER FRESH来指示尽管评估版已更改，但subquery将产生的结果没有差异。如果物化视图过时并且在发出此语句之前需要快速刷新或完全刷新，则状态将不会更改，并且物化视图可能包含错误数据。

{ENABLE |关于查询计算的DISABLE}

此子句允许您控制实例化视图是实时物化视图还是常规物化视图。

  • 指定ENABLE ON QUERY COMPUTATION通过ENABLE QUERY COMPUTATION将常规物化视图转换为实时物化视图。

  • 指定DISABLE ON QUERY COMPUTATION通过DISABLE QUERY COMPUTATION将实时物化视图转换为常规物化视图。

该子句在CREATE MATERIALIZED VIEW和ALTER MATERIALIZED VIEW语句中具有相同的语义。有关此子句的完整信息，请参阅{ENABLE |关于CREATE MATERIALIZED VIEW文档中的QUERY COMPUTATION的DISABLE} 。

启用物化视图的限制

启用实体化视图受以下限制：

  • 如果物化视图处于无效或不可用状态，则无论是否为ENABLE模式，它都不符合查询重写的条件。

  • 如果物化视图是从视图中完全或部分创建的，则无法启用查询重写。

  • 仅当物化视图中的所有用户定义函数都为DETERMINISTIC才能启用查询重写。

禁用条款

如果您不希望DISABLE化视图符合查询重写的条件，请指定DISABLE 。如果物化视图处于无效状态，则无论是否禁用，它都不符合查询重写的条件。但是，可以刷新禁用的物化视图。

unusable_editions_clause

使用此子句指定实例化视图不符合查询重写条件的版本。该子句在CREATE MATERIALIZED VIEW和ALTER MATERIALIZED VIEW语句中具有相同的语义。有关此条款的完整信息，请参阅unusable_editions_clause在文档中CREATE MATERIALIZED VIEW 。

使用物化视图进行查询重写的游标以及在无法使用的版本中编译的游标将无效。

COMPILE

指定COMPILE以显式重新验证实例化视图。如果物化视图所依赖的对象被删除或更改，则物化视图仍可访问，但对于查询重写无效。您可以使用此子句显式重新验证实例化视图，使其符合查询重写的条件。

如果物化视图无法重新验证，则无法刷新或用于查询重写。

也可以看看：

“ 编译物化视图：示例 ”

考虑新鲜

此子句允许您在对主表进行更改后管理实例化视图的过期状态。CONSIDER FRESH指示Oracle数据库考虑新的物化视图，因此有资格在TRUSTED或STALE_TOLERATED模式下进行查询重写。

警告：

CONSIDER FRESH子句还指示Oracle数据库在发布CONSIDER FRESH子句之前不再在物化视图日志中应用任何行或将分区更改跟踪更改应用于物化视图。换句话说，挂起的更改将被忽略并删除，不会应用于实体化视图。这可能导致物化视图包含比基表更多或更少的数据。

由于Oracle数据库无法保证物化视图的新鲜度，因此不支持在ENFORCED模式下进行查询重写。此子句还将物化视图的陈旧状态设置为UNKNOWN 。 STALENESS状态显示在ALL_MVIEWS ， DBA_MVIEWS和USER_MVIEWS数据字典视图的STALENESS列中。

如果对其任何主表的内容进行了更改，则物化视图将过时。该子句指示Oracle数据库假定物化视图是新鲜的，并且没有进行此类更改。因此，对于物化视图，将清除对待刷新的表的实际更新。

自动刷新：示例

以下语句将sales_by_month_by_state化视图（在“ 创建实体化聚合视图：示例 ”中创建）的默认刷新方法更改为FAST ：

ALTER MATERIALIZED VIEW sales_by_month_by_state REFRESH FAST;

物化视图的下一次自动刷新将是快速刷新，前提是它是一个简单的物化视图，其主表具有在物化视图创建或上次刷新之前创建的物化视图日志。

由于REFRESH子句未指定START WITH或NEXT值，因此当创建或最后更改sales_by_month_by_state化视图时，Oracle数据库将使用REFRESH子句建立的刷新间隔。

以下语句为sales_by_month_by_state化视图的自动刷新之间建立了新的间隔：

ALTER MATERIALIZED VIEW sales_by_month_by_state REFRESH NEXT SYSDATE + 7;

由于REFRESH子句未指定START WITH值，因此在创建或最后更改sales_by_month_by_state化视图时指定的START WITH和NEXT值建立的时间发生下一次自动刷新。

在下一次自动刷新时，Oracle数据库刷新实例化视图，评估NEXT表达式SYSDATE +7以确定下一个自动刷新时间，并继续每周自动刷新一次实例化视图。由于REFRESH子句未明确指定刷新方法，因此Oracle数据库继续使用CREATE MATERIALIZED VIEW或最新ALTER MATERIALIZED VIEW语句的REFRESH子句指定的refresh方法。

考虑新鲜：例子

以下语句指示Oracle数据库应将物化视图sales_by_month_by_state视为新的。即使您对sales_by_month_by_state的主表执行了分区维护操作，此语句也允许sales_by_month_by_state符合TRUSTED模式下的查询重写sales_by_month_by_state ：

ALTER MATERIALIZED VIEW sales_by_month_by_state CONSIDER FRESH;

作为上述语句的结果，自上次刷新实例化视图以来对基表执行的任何分区维护操作都不会应用于实例化视图。例如，如果在下次刷新实例化视图之前使用CONSIDER FRESH ，则基表中分区中数据的添加，删除或更改将不会反映在实例化视图中。有关更多信息，请参阅CONSIDER FRESH 。

完成刷新：示例

以下语句指定新的刷新方法，新的NEXT刷新时间以及emp_data化视图的自动刷新之间的新间隔（在“ 物化视图的周期性刷新：示例 ”中创建 ）：

ALTER MATERIALIZED VIEW emp_data REFRESH完成START TRUNC（SYSDATE + 1）+ 9/24 NEXT SYSDATE + 7;

START WITH值将物化视图的下一次自动刷新设置为明天上午9:00。此时，Oracle数据库执行物化视图的完整刷新，评估NEXT表达式，然后每周刷新物化视图。

启用查询重写：示例

以下语句在物化视图emp_data上启用查询重写，并隐式重新验证它：

ALTER MATERIALIZED VIEW emp_data ENABLE QUERY REWRITE;
