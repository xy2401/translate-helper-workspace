词汇表

动作脚本

用于定义资源的启动，停止和检查操作的脚本。启动资源时启动操作，停止资源的停止操作以及检查资源运行状态时的检查操作。

管理员管理

数据库管理员定义数据库资源应在哪些服务器上运行，并根据需要手动放置资源。这是以前版本中使用的管理策略。

代理人

包含代理框架和用户代码以管理资源的程序。

代理框架

AC库，使用户能够插入特定于应用程序的代码来管理自定义应用程序。

可用性指令

Oracle Clusterware在服务器离开或加入群集时重新配置系统的说明。

基数

同时运行资源的服务器数。

客户端集群

在多集群环境中，客户端集群将其名称通告给服务器集群 。

簇

多个互连的计算机或服务器，看起来好像是最终用户和应用程序的一个服务器。

群集感知

任何旨在使用集群件部署的应用程序。

集群管理员

可以根据设置的策略和权限管理群集的某个部分的管理员。

群集配置策略

策略管理群集的文档，其中只包含系统中定义的每个服务器池的一个定义。

群集配置策略集

定义集群中配置的所有服务器池的名称的文档，其中包含一个或多个配置策略。任何时候只有一个策略可以生效，但管理员可以根据业务需求和系统需求，在一天中的不同日期或时间设置不同的策略。

群集健康监视器

检测并分析操作系统和群集资源相关的降级和故障。

集群资源

一种了解群集环境并受跨节点切换和故障转移的资源。

群集时间同步服务

一种时间同步机制，可确保群集中所有节点的所有内部时钟同步。

群集验证实用程序（CVU）

一种工具，用于验证各种Oracle RAC组件，例如共享存储设备，网络配置，系统要求，Oracle Clusterware，组和用户。

数据库云

由全局服务和负载管理框架集成到一个虚拟服务器中的一组数据库，提供一个或多个全局服务，同时确保资源的高性能，可用性和最佳利用率。

依赖

两个或多个资源之间的关系。以及他们之间的互动。

磁盘组

Oracle ASM磁盘组是Oracle ASM作为一个单元管理的磁盘集合。在磁盘组中，Oracle ASM公开Oracle Database文件的文件系统接口。存储在磁盘组中的文件内容均匀分布或条带化，以消除热点并在磁盘上提供统一的性能。Oracle ASM文件也可以选择在磁盘组中进行镜像。磁盘组中磁盘的性能与原始设备的性能相当。

动态主机配置协议（DHCP）

设备（DHCP客户端）用于获取用于在Internet协议网络中操作的配置信息的网络应用程序协议。该协议减少了系统管理工作量，允许在很少或没有人工干预的情况下将设备添加到网络中。

修复脚本

Oracle Universal Installer检测何时未完成最低安装要求，并创建称为修复脚本的shell脚本程序，以解决许多不完整的系统配置要求。如果Oracle Universal Installer检测到未完成的任务，则会提示您创建修订脚本，然后在单独的终端会话中运行修复脚本。您还可以使用-fixup标志生成具有某些CVU命令的修正脚本。

黄金图像

仅限软件的副本，安装在Oracle主目录中，其中不包含特定于主页的详细信息。这使您可以轻松地将Oracle主目录的映像复制到新文件系统上的新主机，以用作活动的可用Oracle主目录。

黄金图像通常可以对应于任何应用程序，但在本文档的上下文中，黄金图像对应于Oracle数据库。此外，黄金映像不是回家，这意味着您在部署黄金映像时不必运行Oracle Universal Installer。

网格主页客户端

Grid Home客户端是一个远程Oracle Grid Infrastructure集群，它订阅Fleet Patching和Provisioning Server以供应黄金映像 。

Oracle Grid Infrastructure

为企业网格体系结构提供基础结构的软件。在集群中，该软件包括Oracle Clusterware和Oracle ASM。对于独立服务器，此软件包括Oracle Restart和Oracle ASM。 Oracle Database 12 c将这些基础架构产品组合到一个名为Oracle Grid Infrastructure home （ Grid_home ）的软件安装中。

网格命名服务（GNS）

通用服务，通过将它们映射到区域内的IP地址来解析委派的普通DNS区域中的主机名称。GNS支持为Oracle RAC数据库节点使用动态主机配置协议（DHCP）地址，从而简化部署。GNS还解析从SCAN侦听器传回的主机名。

枢纽锚

Oracle Flex Cluster中的Hub节点，用作一个或多个其他受支持节点的集群成员身份的连接点。这些节点使用单个Hub锚点交换心跳。

集线器节点

Oracle Flex Cluster中的节点，与其他服务器紧密连接，可直接访问共享存储。

图片

Oracle数据库主页软件的主副本，驻留在Fleet Patching和Provisioning Server上。

IPv4的

Internet协议版本4。IPv4是IP协议的当前标准。IPv4使用32位（四字节）地址，通常以点分十进制表示法表示。每个八位字节的十进制值用句点分隔，如192.168.2.22中所示。

IPv6的

Internet协议版本6。该协议旨在取代IPv4。在IPv6中，IP地址通常由以冒号分隔的八个十六进制值字段表示，如2001：0DB8：0000：0000：0000：0000：1428：57AB。在某些情况下，可以压缩具有0值的字段，如2001年：DB8 :: 1428：57AB。

当地资源

在群集节点上运行但不知道节点范围之外的任何资源的资源。

OCR作家

每个CRSD进程也充当OCR服务器。群集中的一个CRSD进程是OCR服务器，它对磁盘组或文件或块或原始设备执行I / O.

Oracle自动存储管理（Oracle ASM）

Oracle ASM管理Oracle Clusterware和Oracle RAC在磁盘组中使用的存储磁盘。默认情况下，Oracle Clusterware使用Oracle ASM存储OCR和投票文件。

Oracle集群注册表（OCR）

Oracle RAC配置信息存储库，用于管理有关群集节点列表和实例到节点映射信息的信息。OCR还管理有关自定义应用程序的Oracle Clusterware资源配置文件的信息。

Oracle Clusterware

允许连接服务器的组（集群）作为一个单元运行或控制的软件。

Oracle Clusterware堆栈

Oracle Clusterware堆栈包括Oracle Cluster Ready Services，事件管理器，群集同步服务和Oracle ASM（如果使用）。

Oracle Flex Cluster

由集线器节点和其他受支持节点组成的大型集群，其中集线器节点使用当前成员资格算法形成集群，而其他节点将成员资格连接到单个集线器节点。

政策管理

数据库管理员指定运行数据库资源的服务器池（不包括Generic或Free）。Oracle Clusterware将数据库资源放在服务器上。

Fleet Patching和Provisioning Server

Fleet Patching和Provisioning Server是服务器群集上的一个服务器，可以容纳黄金映像的存储库。

读者节点

读取器节点是运行Oracle RAC数据库实例的节点。

资源

由Oracle Clusterware管理的数据库，应用程序或进程。

资源组

应用程序的容器，由Oracle Clusterware作为单个实体管理和监视的各种逻辑相关资源组成。

资源状态

任何给定时间的特定资源的状态，用于确定其对群集的可用性。

资源类型

定义资源是群集资源还是本地资源 。

服务器分类

一种通过使用一组服务器属性将集群中的服务器标识为不同类别的方法。管理员可以配置服务器池以限制它们接受的类别。通过提供SERVER_CATEGORY参数的属性来创建服务器类别。

服务器集群

在多集群环境中，服务器群集是运行网格命名服务（GNS）进程的群集。

服务器池

将群集中的服务器逻辑划分为托管应用程序，数据库或两者的组。

单客户端访问名称（SCAN）

单个名称，解析为公共网络中的三个IP地址。

开始努力评估

Oracle Clusterware在启动资源时经历的过程。在此过程中，Oracle Clusterware会考虑资源概要文件中包含的资源依赖性。

州

一组描述特定资源条件的值。

工作副本

黄金映像的Oracle ACFS快照，可以是纯软件安装的Oracle主目录，也可以是实例化的已配置的Oracle主目录。

区域数据

保存在Oracle Cluster Registry中的GNS守护程序记录。
