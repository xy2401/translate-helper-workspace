22 Oracle Database Vault策略API

您可以使用DBMS_MACADM PL / SQL包来管理Oracle Database Vault策略。

只有已被授予DV_OWNER或DV_ADMIN角色的用户才能使用这些过程。

  • ADD_CMD_RULE_TO_POLICY过程
    ADD_COMMAND_RULE_TO_POLICY过程使您可以将现有命令规则添加到Oracle Database Vault策略。
  • ADD_OWNER_TO_POLICY过程
    ADD_OWNER_TO_POLICY过程使您可以将现有数据库用户作为所有者添加到Oracle Database Vault策略。
  • ADD_REALM_TO_POLICY过程
    ADD_REALM_TO_POLICY过程使您可以将现有域添加到Oracle Database Vault策略。
  • CREATE_POLICY过程
    CREATE_POLICY过程使您可以创建Oracle Database Vault策略。
  • DELETE_CMD_RULE_FROM_POLICY过程
    DELETE_CMD_RULE_FROM_POLICY过程使您可以从Oracle Database Vault策略中删除现有命令规则。
  • DELETE_OWNER_FROM_POLICY过程
    DELETE_OWNER_FROM_POLICY过程使您可以从Oracle Database Vault策略中删除所有者。
  • DELETE_REALM_FROM_POLICY过程
    DELETE_REALM_FROM_POLICY过程使您可以从Oracle Database Vault策略中删除现有领域。
  • DROP_POLICY过程
    DROP_POLICY过程使您可以删除现有的Oracle Database Vault策略。
  • RENAME_POLICY程序
    UPDATE_POLICY_DESCRIPTION过程使您可以重命名现有的Oracle Database Vault策略。
  • UPDATE_POLICY_DESCRIPTION过程
    UPDATE_POLICY_DESCRIPTION过程使您可以更新Oracle Database Vault策略中的description字段。
  • UPDATE_POLICY_STATE过程
    UPDATE_POLICY_STATE过程使您可以更新Oracle Database Vault策略中的policy_state字段。

ADD_CMD_RULE_TO_POLICY过程

ADD_COMMAND_RULE_TO_POLICY过程使您可以将现有命令规则添加到Oracle Database Vault策略。

当命令规则处于任何状态时，您可以将命令规则添加到策略。例如，您可以将已禁用的命令规则添加到已启用的策略。在这种情况下，禁用的命令规则将在添加到策略时自动启用。命令规则只能添加到一个策略中。换句话说，您无法将相同的命令规则分配给多个策略。

句法

DBMS_MACADM.ADD_CMD_RULE_TO_POLICY（policy_name IN VARCHAR2，命令IN VARCHAR2，object_owner IN VARCHAR2，object_name IN VARCHAR2，clause_name IN VARCHAR2 DEFAULT，parameter_name IN VARCHAR2 DEFAULT，event_name IN VARCHAR2 DEFAULT，component_name IN VARCHAR2 DEFAULT，action_name IN VARCHAR2 DEFAULT，范围IN NUMBER DEFAULT） ;

参数

表22-1 ADD_CMD_RULE_TO_POLICY参数

参数 描述

POLICY_NAME

政策名称。要查找当前数据库实例中的现有Database Vault策略，请查询DBA_DV_POLICY视图中描述的DBA_DV_POLICY视图 。

命令

命令规则名称

要查找当前数据库实例中的现有Database Vault命令规则，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图 。

OBJECT_OWNER

应用命令规则的数据库模式

要查找此命令规则的现有对象所有者，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

OBJECT_NAME

受命令规则保护的对象

要查找此命令规则的现有对象，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

clause_name

对于ALTER SYSTEM和ALTER SESSION命令规则，SQL语句中用于创建命令规则的子句

要查找此命令规则的现有子句，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

PARAMETER_NAME

对于ALTER SYSTEM和ALTER SESSION命令规则，来自clause_name参数的参数。

要查找此命令规则的现有参数，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

要查找此命令规则的现有组件名称，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

要查找此命令规则的现有操作名称，请查询DBA_DV_COMMAND_RULE视图中描述的DBA_DV_COMMAND_RULE视图

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）如果命令规则在当前PDB中是本地的

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果命令规则适用于所有PDB

ADD_OWNER_TO_POLICY过程

将所有者添加到已启用的策略时，会立即进行更改。您添加到策略的用户数没有限制。

句法

DBMS_MACADM.ADD_OWNER_TO_POLICY（policy_name IN VARCHAR2，owner_name IN VARCHAR2）;

参数

BEGIN DBMS_MACADM.ADD_OWNER_TO_POLICY（policy_name =>'HR_DV_Policy'，owner_name =>'PSMITH'）;结束; /

ADD_REALM_TO_POLICY过程使您可以将现有域添加到Oracle Database Vault策略。

您可以将已禁用的域添加到已启用的策略中。在这种情况下，域在添加时会自动启用。领域只能添加到一个策略中。换句话说，您不能将同一领域分配给多个策略。

句法

DBMS_MACADM.ADD_REALM_TO_POLICY（policy_name IN VARCHAR2，realm_name IN VARCHAR2）;

realm_name

CREATE_POLICY过程

CREATE_POLICY过程使您可以创建Oracle Database Vault策略。

创建策略后，必须至少向策略添加一个领域和一个命令规则。（可选）您可以将这些域和命令规则设置为单独实施，或使用策略使用的实施。

不需要策略的所有者，但如果您未将所有者分配给策略，则已授予DV_OWNER或DV_ADMIN角色的用户必须管理该策略。

创建策略后，使用以下过程完成策略定义：

  • ADD_REALM_TO_POLICY将域添加到策略中。

  • ADD_CMD_RULE_TO_POLICY将命令规则添加到策略。

  • ADD_OWNER_TO_POLICY使指定的数据库用户可以管理策略。

句法

DBMS_MACADM.CREATE_POLICY（policy_name IN VARCHAR2，描述IN VARCHAR2 DEFAULT，policy_state IN NUMBER，pl_sql_stack IN BOOLEAN DEFAULT）;

参数

表22-4 CREATE_POLICY参数

参数 描述

POLICY_NAME

策略名称，大小写混合最多128个字符

要查找当前数据库实例中的现有策略，请查询DBA_DV_POLICY视图中描述的DBA_DV_POLICY视图 。

描述

策略用途说明，混合大小最多4000个字符。

policy_state

指定策略的启用方式。可能的值是：

  • DBMS_MACADM.G_ENABLED （ 1 ），在您创建策略后启用该策略

  • DBMS_MACADM.G_DISABLED （ 0 ），在您创建策略后禁用该策略

  • DBMS_MACADM.G_SIMULATION （ 2 ），它将策略设置为模拟模式。在模拟模式下，对策略中使用的域或命令规则的任何违规都记录在指定的日志表中，其中包含足以描述错误的信息，例如使用的用户名或SQL语句。

  • DBMS_MACADM.G_PARTIAL （ 3 ），它将策略设置为部分模式。在部分模式下，可以单独更改与策略关联的域的强制状态或命令规则。

有关模拟模式的更多信息，请参见关于模拟模式

pl_sql_stack

启用模拟模式时，指定是否记录失败操作的PL / SQL堆栈。输入TRUE以记录PL / SQL堆栈，输入FALSE以不记录。

例

以下示例创建一个使用部分状态的策略，并启用PL / SQL堆栈的捕获。稍后，当向该策略添加领域或命令规则时，它们的强制状态将能够单独更改。

BEGIN DBMS_MACADM.CREATE_POLICY（policy_name =>'HR Database Vault Policy'，description =>'保护HR模式的策略'，policy_state => DBMS_MACADM.G_ENABLED，pl_sql_stack => TRUE）;结束; /

DELETE_CMD_RULE_FROM_POLICY过程

DELETE_CMD_RULE_FROM_POLICY过程使您可以从Oracle Database Vault策略中删除现有命令规则。

无论策略的状态如何，您都可以随时从策略中删除命令规则。从策略中删除命令规则时，命令规则规则保持不变。也就是说，如果启用了策略，并且从策略中删除了命令规则，则在从策略中删除命令规则后，该命令规则仍将处于启用状态。
