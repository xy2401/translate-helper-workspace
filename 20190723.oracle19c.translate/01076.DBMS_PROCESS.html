126 DBMS_PROCESS

DBMS_PROCESS程序包提供了一个用于管理预先生成的服务器的接口。

本章包含以下主题：

  • DBMS_PROCESS概述

  • DBMS_PROCESS安全模型

  • DBMS_PROCESS子程序摘要

126.1 DBMS_PROCESS概述

默认情况下，Oracle数据库可以预先设置前台进程以提高客户端连接的性能。

预先确定的流程是指已生成但尚未召开会话的流程。当用户连接到数据库或需要服务进程时，该进程根据需要执行进一步的初始化。

要管理前台进程，请使用DBMS_PROCESS包。此程序包中的过程配置连接池的前台进程数，启动连接池和停止连接池。

也可以看看：

  • 有关管理预创建进程的详细说明，请参见“Oracle数据库管理员指南”

  • 有关USE_DEDICATED_BROKER参数的信息，请USE_DEDICATED_BROKER Oracle数据库参考 ，该参数确定如何生成专用服务器。TRUE使侦听器能够将连接发送到生成专用服务器的专用连接代理。FALSE设置侦听器直接生成专用服务器。

126.2 DBMS_PROCESS安全模型

必须授予您SYSDBA管理权限才能使用DBMS_PROCESS程序包。

126.3 DBMS_PROCESS子程序摘要

此表列出了DBMS_PROCESS子程序并简要描述了它们。

表126-1 DBMS_PROCESS程序包子程序

子程序 描述

CONFIGURE_POOL过程

126.3.1 CONFIGURE_POOL过程

此过程配置预先生成的进程的最小数量，批处理中预先生成的进程数以及前台连接池的预生成进程的初始数。

句法

DBMS_PROCESS.CONFIGURE_POOL（pool_name IN VARCHAR2 DEFAULT“SYS_DEFAULT_FOREGROUND_POOL”，min_count IN NUMBER DEFAULT 10，batch_count IN NUMBER DEFAULT 20，init_count IN NUMBER DEFAULT 0）;

参数

表126-2 CONFIGURE_POOL过程参数

参数 描述

POOL_NAME

前台池的名称

min_count

计数之后将在池中生成一批预先生成的进程（默认为10;最大进程数为64,000）

batch_count

要在池中生成的预先生成的服务器批次（默认为20）。min_count和batch_count值的min_count不能超过PROCESSES参数值。

init_count

在启动时或在连接风暴之前产生的预先生成的服务器集合，分批生成（默认为0）。不要超过PROCESSES参数值。

例外

表126-3 CONFIGURE_POOL过程异常

错误 描述

ORA-456

Prespawn未启用

ORA-457

没有找到Prespawn池

ORA-458

Prespawn池已经开始了

ORA-459

Prespawn游泳池尚未开始

ORA-460

进程池无效配置值

使用说明

  • 要查找有关前台进程的信息，请查询V$PROCESS_POOL动态视图。

例

BEGIN DBMS_PROCESS.CONFIGURE_POOL（pool_name =>'hrdb_pool'，min_count => 40，batch_count => 20，init_count => 10）;结束; /

126.3.2 START_POOL过程

此过程启动预先确定的前台进程池。

句法

DBMS_PROCESS.START_POOL（pool_name IN VARCHAR2 DEFAULT“SYS_DEFAULT_FOREGROUND_POOL”）;

参数

表126-4 START_POOL过程参数

参数 描述

POOL_NAME

预先确定的前台进程池的名称。要查找现有池，请查询V$PROCESS_POOL动态视图。

例外

表126-5 START_POOL过程异常

错误 描述

ORA-457

没有找到Prespawn池

ORA-458

Prespawn池已经开始了

ORA-459

Prespawn游泳池尚未开始

例

BEGIN DBMS_PROCESS.START_POOL（pool_name =>'hrdb_pool'，END;
