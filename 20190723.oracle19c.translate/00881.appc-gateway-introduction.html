1 APPC的Oracle数据库网关简介

用于APPC的Oracle数据库网关（网关）使用户能够在远程启动事务程序在线事务处理器（OLTP）。APPC的Oracle数据库网关可以使用SNA通信协议与OLTP建立连接。网关还可以使用TCP / IP进行IMS Connect，以通过TCP / IP与OLTP建立通信。该网关为Oracle应用程序提供了对IBM大型机数据和服务的无缝访问远程过程调用（RPC）处理。网关可以直接或通过TP监视器（如CICS）访问任何能够使用CPI-C API的应用程序。

网关概述

用于APPC的Oracle数据库网关扩展了 RPC Oracle数据库提供的工具。网关允许任何客户端应用程序使用 PL / SQL请求执行远程交易程序 （RTP）驻留在主机上。网关为使用该系统的系统提供RPC处理 SNA APPC（高级程序到程序通信）协议和使用的系统 IMS Connect协议的TCP / IP。此体系结构允许分别有效访问IBM大型机和IMS上可用的数据和事务。

网关不需要远程主机系统上的Oracle软件。因此，网关在远程主机上使用很少或没有编程工作的现有事务。

注意：

对于使用SNA的网关，仅使用通用和标准协议APPC，使网关能够访问多个系统。该网关几乎可以与任何支持APPC的系统通信，包括任何平台上的IBM Corporation CICS，IBM Corporation IMS和APPC / MVS。这些事务监视器提供对各种系统的访问，允许网关访问许多系统数据存储，包括VSAM，DB2（静态SQL），IMS等。

网关可以直接或通过TP监视器（如CICS）访问任何能够使用CPI-C API的应用程序。

网关的特点

Oracle Database Gateway for APPC具有以下优势：

  • IMS Connect的TCP / IP支持

    此版本的网关包括对IMS Connect的TCP / IP支持，为用户提供SNA或TCP / IP通信协议之间的选择。IMS Connect是一种IBM产品，它使TCP / IP客户端能够触发IMS事务的执行。网关可以使用TCP / IP通信协议来访问IMS Connect，从而触发IMS事务的执行。如果您选择使用TCP / IP，则此配置没有SNA参与。

    与此网关的新功能相关的是 pg4tcpmap工具。此版本的网关包括一个工具，其目的是将您的Side Profile Name中的信息映射到TCP / IP和IMS Connect。有关网关映射工具的详细信息，请参阅本指南中的Oracle Database Gateway for APPC用户指南的第6章和使用TCP / IP通信协议的网关配置 。

  • 快速界面

    当您远程执行程序时，网关已针对最小网络流量进行了优化。网关的接口是优化的 PL / SQL存储过程规范称为事务接口包（TIP）。该规范预编译在 Oracle数据库。由于远程主机系统上没有其他软件层，因此仅在程序执行时才会产生开销。

  • 平台独立

    客户端应用程序不必是特定于操作系统的。例如，您的应用程序可以调用CICS Transaction Server上的程序 z / OS上。如果将程序移动到pSeries上的CICS区域，则无需更改应用程序。

  • 应用透明度

    调用执行远程事务程序的应用程序的用户不知道请求被发送到主机。

  • 灵活的界面

    您可以将网关用作现有过程逻辑的接口，或将新的过程逻辑集成到Oracle数据库环境中。

  • Oracle数据库集成

    通过将Oracle数据库与网关集成，您可以从现有和未来的Oracle功能中受益。例如，可以从Oracle存储过程或数据库触发器调用网关。

  • 交易支持

    网关和Oracle数据库允许以协调的方式执行远程事务更新和Oracle数据库更新。

  • 选择范围广工具

    网关支持任何支持PL / SQL的工具或应用程序。

  • PL / SQL代码生成器

    该 Oracle Database Gateway for APPC提供了强大的开发环境，包括：

    • 用于存储与远程事务相关的信息的数据字典

    • 生成PL / SQL事务接口包或TIP的工具

    • 用于查看存储在网关字典中的信息的报告实用程序

    • 一整套跟踪和调试工具

    • 一组示例，用于演示如何针对DB2，IMS和CICS等数据存储使用该产品。

  • 网站自治和安全

    网关提供站点自治，允许您执行诸如验证用户之类的任务。它还提供与大型机计算机上运行的任何安全包兼容的基于角色的安全性。

  • 自动转换

    通过提示，执行以下转换：

    • ASCII往返 EBCDIC

    • 远程事务程序数据类型与PL / SQL数据类型之间的数据类型

    • 许多语言的国家语言支持

条款

本指南中使用了以下术语和定义。有关网关及其组件和功能的术语和定义的完整列表，请参阅网关术语 。

Oracle数据库

这是与网关通信的任何Oracle数据库实例，用于执行远程过程调用 RTP。该 Oracle数据库可以与网关位于同一系统上，也可以位于不同的系统上。如果它在不同的系统上，那么两个系统都需要Oracle Net。有关网关体系结构的视图，请参见图1-2 。

在线事务处理器（OLTP）

OLTP是其他供应商提供的在线事务处理器，包括用于z / OS的CICS Transaction Server，IMS / TM和 z / OS上。

程序网关管理实用程序（PGAU）

PGAU是用于定义和生成的工具 PL / SQL事务接口包（TIP）。有关PGAU的更多信息，请参阅“ Oracle数据库网关中的APPC用户指南”中的第2章“程序网关管理实用程序” 。

数据字典（PG DD）

此网关组件是远程主机事务（RHT）定义和数据定义的存储库。生成TIP时，PGAU访问PG DD中的定义。PG DD具有数据类型依赖性，因为它支持PGAU，并不打算由客户直接访问。有关PG DD表的列表，请参阅APPC用户指南的Oracle数据库网关中的附录A，APPC数据字典的程序网关。

RPC

RPC是一种编程调用，它响应来自另一个系统的请求在一个系统上执行程序逻辑。有关更多信息，请参阅网关术语 ，并参阅APPC用户指南的Oracle Database Gateway中的附录B，网关RPC接口。

RTP

远程事务程序是客户编写的事务，在OLTP的控制下运行，用户使用PL / SQL过程远程调用。要通过网关执行远程事务程序，必须使用RPC执行PL / SQL程序来调用网关功能。

提示（交易界面包）

一个 TIP是Oracle 应用程序和远程事务程序之间存在的PL / SQL包。TIP是一组PL / SQL存储过程，它通过网关调用远程事务程序。TIP使用PL / SQL和UTL_RAW / UTL_PG函数执行远程主机数据的转换和重新格式化。

网关的实施

无论您的通信协议是SNA还是IMS Connect的TCP / IP支持，网关的基本结构都是相同的。网关与UNIX上的Oracle数据库实例具有一些相同的组件。它有以下组件：

  • 主目录，类似于与Oracle实例ORACLE_HOME环境变量关联的目录

  • 系统标识符，标识为sid或ORACLE_SID

  • 初始化参数文件，类似于Oracle数据库 init sid 。 ora文件。

网关没有：

  • 控制，重做日志或数据库文件

  • 与已安装的Oracle数据库关联的完整子目录和辅助文件集

由于网关没有后台进程且不需要管理实用程序（如Oracle Enterprise Manager），因此无需启动网关。每访问特定网关的Oracle数据库用户会话在UNIX上创建一个独立的进程，该进程又运行网关服务器并执行SNA或TCP / IP功能以与OLTP通信。

与网关通信

之间的所有沟通用户或客户端程序和网关通过事务接口包处理 （ TIP），在Oracle数据库上执行。TIP是一个标准 PL / SQL包提供以下功能：

  • 声明可以与远程事务程序交换的PL / SQL变量

  • 调用处理通信的网关包以开始对话，交换数据和终止对话

  • 处理之间的所有数据类型转换 PL / SQL数据类型和目标程序数据类型

程序网关管理实用程序（ PGAU），随网关自动生成提示规范。

使用数据库链接将网关标识到Oracle数据库。数据库链接与用于标识其他Oracle数据库的构造相同。网关中的功能参考 PL / SQL为：

函数名@ dblink_name

以下部分描述RPC函数如何使用SNA或TCP / IP通信协议在网关上执行。

  • 远程事务启动

  • 数据交换

  • 远程交易终止

远程事务启动

该 TIP使用其中一个网关功能PGAINIT启动与远程主机系统的连接。

通信协议是SNA ： PGAINIT 提供所需的输入 SNA参数用于启动与目标事务程序的对话。这些参数通过SNA网络发送，该网络将会话标识符返回给PGAINIT 。将来对目标程序的任何调用都使用会话标识符作为INPUT参数。

当通信协议是TCP / IP时 ： PGAINIT 提供所需的TCP / IP参数作为输入。使用 pg4tcpmap工具来映射参数。这些参数通过TCP / IP网络发送，以启动与目标事务程序的对话。TCP / IP网络返回一个套接字文件描述符到PGAINIT 。 PGAXFER和PGATERM将来对目标程序的任何调用PGATERM使用套接字文件描述符作为输入参数。

有关pg4tcpmap工具的功能和用法的更多信息，请参阅“TCP / IP通信协议的网关初始化参数”和“ Oracle数据库网关APPC用户指南”中的第6章。

数据交换

建立会话后，调用数据库网关函数 PGAXFER可以以输入和输出变量的形式交换数据。PGAXFER向目标事务程序发送和接收缓冲区。网关将缓冲区视为仅RAW字节流。居住在中的TIP Oracle数据库负责在将缓冲区发送到网关之前将应用程序P L / SQL数据类型转换为RAW 。它还负责在将结果返回给应用程序之前将RAW转换为PL / SQL数据类型。

远程交易终止

当与远程程序的通信完成时，网关功能PGATERM终止网关和远程主机之间的对话。

当通信协议是SNA时， PGATERM使用会话标识符作为INPUT参数来请求会话终止。

当通信协议是TCP / IP时， PGATERM使用TCP / IP的套接字文件描述符作为INPUT参数来请求会话终止。

使用SNA的网关的事务类型

适用于APPC的Oracle数据库网关支持从远程主机系统读取数据和向远程主机系统写入数据的三种类型的事务：

  • 单次

    在一个一次性事务，应用程序初始化连接，交换数据并终止连接，所有这些都在一次调用中完成。

  • 一贯

    在一个持久性事务，可以在终止对话之前进行多次与远程事务交换数据的调用。

  • Multiconversational

    在一个多会话事务，数据库网关服务器可用于在一次调用远程事务程序时交换多个记录。

有关事务类型的更多信息，请参阅“ APPC用户指南”的第4章“ Oracle数据库网关的客户端应用程序开发”中的“远程主机事务类型” 。

以下列表展示了它的强大功能适用于APPC的Oracle数据库网关：

  • 您可以在大型机上启动CICS事务，以从PC应用程序的VSAM文件中检索数据。

  • 您可以修改和监视远程过程控制系统的操作。

  • 您可以启动在DB2中执行静态SQL的IMS / TM事务。

  • 你可以发起一个 CICS事务，在单个调用中返回大量记录。

使用TCP / IP的网关的事务类型

该使用TCP / IP for IMS Connect的Oracle Database Gateway for APPC支持三种类型的事务套接字连接：

  • 交易套接字

    套接字连接跨越单个事务。

  • 持久套接字

    套接字连接跨多个事务。

  • 非持久套接字

    套接字连接跨越由一个输入和一个输出组成的单个交换。

    注意：

    如果您计划实现会话事务，请不要使用非持久套接字类型，因为会发生多个连接和断开连接。
