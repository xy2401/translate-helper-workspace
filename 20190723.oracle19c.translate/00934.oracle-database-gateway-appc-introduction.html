1 APPC的Oracle数据库网关简介

Oracle Database Gateway for APPC使用户能够在远程启动事务程序在线事务处理器（OLTP）。APPC的Oracle数据库网关可以使用SNA通信协议与OLTP建立连接。网关还可以使用TCP / IP进行IMS Connect建立通信 IMS / TM通过TCP / IP。该网关为Oracle应用程序提供了对IBM大型机数据和服务的无缝访问远程过程调用（RPC）处理。

有关经过认证的硬件平台和操作系统版本的最新列表，请参阅Oracle数据库安装指南和My Oracle Support网站上的认证表。可以在以下位置找到My Oracle Support Web站点：

https://support.oracle.com

网关概述

用于APPC的Oracle数据库网关扩展了 RPC Oracle数据库提供的工具。网关允许任何客户端应用程序使用 PL / SQL请求执行远程交易程序（RTP）驻留在主机上。网关为使用该系统的系统提供RPC处理 SNA高级程序到程序通信（APPC）协议和使用的IMS / TM系统 IMS Connect的TCP / IP支持。该体系结构允许有效访问IBM大型机上可用的数据和事务 IMS，分别。

网关不需要远程主机系统上的Oracle软件。因此，网关在远程主机上使用很少或没有编程工作的现有事务。

对于仅使用SNA的网关：

通用和标准协议APPC的使用允许网关访问多个系统。该网关几乎可以与任何支持APPC的系统进行通信，包括任何平台上的IBM公司的CICS和IBM公司的IMS和APPC / MVS。这些事务监视器提供对各种系统的访问，允许网关访问许多系统数据存储，包括VSAM，DB2（静态SQL），IMS等。

网关可以直接或通过TP监视器（如CICS）访问任何能够使用CPI-C API的应用程序。

网关的特点

Oracle Database Gateway for APPC具有以下优势：

  • 快速界面

    网关经过优化，可以在最小的网络流量下实现程序的远程执行。网关的接口是优化的预编译的PL / SQL存储过程规范（称为TIP或事务接口包） Oracle数据库。由于远程主机上没有其他软件层，因此仅在程序执行时才会产生开销。

  • 地点透明度

    客户端应用程序不必是特定于操作系统的。例如，您的应用程序可以调用CICS Transaction Server for z / OS中的程序。如果将程序移动到AIX上的CICS区域，则无需更改应用程序。

  • 应用透明度

    调用执行远程事务程序的应用程序的用户不知道请求被发送到主机。

  • 灵活的界面

    您可以使用网关与现有过程逻辑接口，或将新过程逻辑集成到Oracle数据库环境中。

  • Oracle数据库集成

    Oracle数据库与网关的集成使网关能够从现有和未来的Oracle数据库功能中受益。例如，可以从Oracle存储过程或数据库触发器调用网关。

  • 交易支持

    网关和Oracle数据库允许以协调的方式执行远程传输更新和Oracle数据库更新。

  • 选择范围广工具

    网关支持任何支持PL / SQL的工具或应用程序。

  • PL / SQL代码生成器

    该 Oracle Database Gateway for APPC提供了强大的开发环境，包括：

    • 用于存储与远程交易相关的信息的数据字典

    • 生成PL / SQL事务接口包或TIP的工具

    • 报告实用程序，用于查看存储在网关字典中的信息

    • 一整套跟踪和调试工具

    • 一组广泛的示例，用于演示如何针对数据存储（如DB2，IMS和CICS）使用该产品。

  • 网站自治和安全

    网关提供站点自治，允许您执行诸如对用户进行身份验证之类的操作。它还提供与大型机计算机上运行的任何安全包兼容的基于角色的安全性。

  • 自动转换

    通过提示，执行以下转换：

    • ASCII往返 EBCDIC

    • 远程事务程序与PL / SQL数据类型之间的数据类型

    • 多种语言的国家语言支持

  • 全球化支持

  • IMS Connect的TCP / IP支持

    此版本的网关包括对IMS Connect的TCP / IP支持，使用户可以选择是使用SNA还是TCP / IP通信协议。IMS Connect是一种IBM产品，它允许TCP / IP客户端触发IMS事务的执行。网关可以使用TCP / IP通信协议来访问IMS Connect，从而触发IMS事务的执行。此配置没有SNA参与。

    与此网关功能相关的是：

    • 该网关映射工具。此版本的网关包括一个工具（ pg4tcpmap ）其目的是将信息从SNA侧配置文件名称映射到TCP / IP主机名和端口号。

      注意：

      当您的通信协议是 TCP / IP，仅支持IMS作为OLTP。

条款

本指南中使用了以下术语和定义：

网关初始化文件

这个文件被称为 init sid .ora ，它包含控制网关操作的参数。如果您使用的是SNA协议，请参阅适用于POWER Systems（64位）上的IBM AIX的Oracle数据库网关APPC安装和配置指南的附录A“SNA协议的网关初始化参数” ，Linux x86-64，Oracle 适用于Microsoft Windows的 SPARC（64位）上的Solaris 和适用于Microsoft Windows的APPC安装和配置指南的 HP-UX Itanium或Oracle数据库网关，以获取更多信息。如果您的协议是TCP / IP，请参阅适用于POWER系统（64位）上的IBM AIX的APPC安装和配置指南的Oracle数据库网关中的附录B“TCP / IP通信协议的网关初始化参数” ，Linux x86- 64，SPARC上的Oracle Solaris（64位）和 适用于Microsoft Windows的APPC安装和配置指南的 HP-UX Itanium或Oracle数据库网关 。

网关远程过程

Oracle Database Gateway for APPC提供了预构建的远程过程。一般来说， 使用以下三个远程过程：

  • PGAINIT ，初始化交易

  • PGAXFER ，传输数据

  • PGATERM ，终止交易

请参阅本指南中的网关RPC接口 以及适用于POWER系统上的IBM AIX的APPC安装和配置指南的Oracle数据库网关第1章中的“远程过程调用函数” （64位），Linux x86-64，Oracle Solaris on有关网关远程过程的详细信息，请参见 SPARC（64位）和 适用于Microsoft Windows的APPC安装和配置指南的 HP-UX Itanium或Oracle数据库网关 。

dg4pwd

dg4pwd是一个实用程序，用于加密通常存储在网关初始化文件中的密码。密码以加密形式存储在密码文件中，使信息更安全。请参阅适用于POWER系统（64位）上的IBM AIX的Oracle数据库网关安装和配置指南的安全要求一章中的“网关初始化文件中的密码” ，Linux x86-64，SPARC上的Oracle Solaris（64- Bit）和 适用于Microsoft Windows的APPC安装和配置指南的 HP-UX Itanium和Oracle数据库网关，以获取有关dg4pwd实用程序如何工作的详细信息。

pg4tcpmap工具

仅当网关对IMS Connect使用TCP / IP支持时，此工具才适用。其功能是将SNA参数（例如Side Profile Name）映射到TCP / IP参数（例如OLTP主机名，IMS Connect端口号和IMS目标ID）。

PGA（程序网关管理）

PGA是本指南中对包含Oracle数据库网关APPC的所有或大多数组件的一般参考。当对特定产品或组件的引用太窄时使用该术语。

PGDL（程序网关定义语言）

PGDL是用于为PGAU定义事务和数据的语句集合。

PL / SQL存储过程规范（PL / SQL包）

这是一个存储在Oracle数据库中的预编译PL / SQL过程。

UTL_RAW PL / SQL包（UTL_RAW函数）

网关的这个组件代表PL / SQL RAW变量和远程主机数据的一系列数据转换函数。执行的转换类型取决于远程主机数据的语言。有关详细信息，请参阅本指南中的数据类型转换 。

UTL_PG PL / SQL包（UTL_PG函数）

网关的这个组件代表一系列COBOL数字数据转换功能。有关支持的数值数据类型转换，请参阅本指南的UTL_PG接口中的“ NUMBER_TO_RAW和RAW_TO_NUMBER参数值 ” 。

Oracle数据库

这是与网关通信的任何Oracle数据库实例，用于执行RPC RTP。 Oracle数据库可以与网关位于同一系统上，也可以位于不同的系统上。如果它在不同的系统上，那么两个系统都需要Oracle Net。有关网关体系结构的视图，请参见图1-2 。

OLTP（在线交易处理器）

OLTP是其他供应商提供的众多在线事务处理器中的任何一种，包括用于z / OS和IMS / TM的CICS Transaction Server。

注意：

当您的通信协议是 TCP / IP，仅支持IMS作为OLTP。

PGAU（程序网关管理实用程序）

PGAU是用于定义和生成的工具 PL / SQL事务接口包（TIP）。有关PGAU的更多信息，请参阅本指南中的Procedural Gateway Administration Utility 。

PG DD（程序网关数据字典）

网关的这个组件是远程主机事务定义和数据定义的存储库。生成TIP时，PGAU访问PG DD中的定义。PG DD具有数据类型依赖性，因为它支持PGAU，并且不打算由客户直接访问。有关PG DD表的列表，请参阅本指南中的APPC数据字典的数据库网关 。

RPC（远程过程调用）

RPC是一种编程调用，它响应来自另一个系统的请求在一个系统上执行程序逻辑。请参阅适用于POWER系统（64位）上的IBM AIX的Oracle数据库网关APPC安装和配置指南附录C中的“网关远程过程” ，Linux x86-64，SPARC上的Oracle Solaris（64位）和HP -UX Itanium或Oracle Database Gateway for APPC安装和配置指南（适用于Microsoft Windows ）以及本指南中的Gateway RPC Interface以获取更多信息。

RTP（远程交易程序）

远程事务程序是客户编写的事务，在OLTP的控制下运行，用户使用PL / SQL过程远程调用。要通过网关执行远程事务程序，必须使用RPC执行PL / SQL程序来调用网关功能。

提示（交易界面包）

一个 TIP是Oracle 应用程序和远程事务程序之间存在的PL / SQL包。事务接口包或TIP是一组通过网关调用RTP的PL / SQL存储过程。TIP使用PL / SQL和UTL_RAW/UTL_PG函数执行远程主机数据的转换和重新格式化。

网关的体系结构

Oracle Database Gateway for APPC的体系结构由几个组件组成：

  1. Oracle数据库

    有关建立网关与Oracle数据库关系的各种方法的说明，请参阅安装指南中与通信协议对应的配置部分。

    Oracle数据库也可用于非网关应用程序。

  2. 网关

    必须在可运行该服务器的服务器上安装Oracle Database Gateway for APPC 所需的操作系统版本。

  3. 一个OLTP

    必须可以使用SNA或TCP / IP通信协议从网关访问OLTP。多 Oracle数据库可以访问同一个网关。可以将单个系统网关安装配置为访问多个OLTP。

    • 对于使用TCP / IP的网关：唯一的通过TCP / IP支持的OLTP是IMS通过IMS Connect。使用TCP / IP协议，系统必须可以访问OLTP。多 Oracle数据库可以访问同一个网关。可以将单个系统网关安装配置为访问多个OLTP。可以从IMS Connect访问多个IMS系统。如果您有许多可用的IMS Connect系统，则其中任何一个都可以连接到一个或多个IMS系统。

以下部分描述了TIP如何工作，首先建立与远程主机的连接，然后从目标事务程序交换数据，最后终止对话。

  • 远程事务启动

  • 数据交换

远程事务启动

该 TIP使用其中一个网关功能PGAINIT启动与远程主机的连接。

通信协议是SNA： PGAINIT 提供所需的输入 SNA参数用于启动与目标事务程序的对话。这些参数通过SNA网络发送，该网络将会话标识符返回给PGAINIT 。对目标程序的未来调用使用会话标识符作为输入参数。

当通信协议是TCP / IP时 ： PGAINIT 提供作为输入，所需的TCP / IP参数。这些参数通过TCP / IP网络发送，以启动与目标事务程序的对话。TCP / IP网络返回一个套接字文件描述符到PGAINIT 。未来的电话，如 PGAXFER和 PGATERM ，使用相同的套接字文件描述符作为输入参数。

数据交换

建立会话后，调用数据库网关函数 PGAXFER可以以输入和输出变量的形式交换数据。PGAXFER向目标事务程序发送和接收缓冲区。网关将缓冲区视为仅RAW字节流。驻留在的TIP Oracle数据库负责在将缓冲区发送到网关之前将应用程序的P L / SQL数据类型转换为RAW。它还负责在将结果返回给应用程序之前将RAW转换为PL / SQL数据类型。

远程交易终止

当与远程程序的通信完成时，网关功能PGATERM终止网关和远程主机之间的对话。

当通信协议是SNA时 ： PGATERM使用会话标识符作为输入参数来请求会话终止。

当通信协议是TCP / IP时 ： PGATERM使用TCP / IP的套接字文件描述符作为输入参数来请求对话终止。

使用SNA的网关概述

如果您使用的是SNA通信协议，请阅读以下部分，以了解网关如何与Oracle数据库和大型机通信，以及网关独有的事务类型和编写TIP。

使用SNA的网关的事务类型

适用于APPC的Oracle数据库网关支持从远程主机系统读取数据和向远程主机系统写入数据的三种类型的事务：

  • 一次性

    在一个一次性事务，应用程序在一次调用中启动连接，交换数据和终止连接。

  • 一贯

    在一个持久性事务，可以在终止对话之前执行与远程事务交换数据的多次调用。

  • 多对话

    在一个多会话事务，数据库网关服务器可用于在一次调用远程事务程序时交换多个记录。

有关事务类型的详细信息，请参阅本指南的客户端应用程序开发（仅限SNA）中的“ 远程主机事务类型 ” 。

以下列表演示了权力的例子适用于APPC的Oracle数据库网关：

  • 您可以在大型机上启动CICS事务，以从PC应用程序的VSAM文件中检索数据。

  • 您可以修改和监视远程过程控制计算机的操作。

  • 您可以启动在DB2中执行静态SQL的IMS / TM事务。

  • 你可以发起一个 CICS事务，在单个调用中返回大量记录。

与Oracle数据库（SNA）的简单网关通信

本节介绍大型机与主机之间的简单通信使用SNA通信协议的网关上的Oracle数据库。Oracle Database Gateway for APPC允许您编写自己的过程，以便开始在Oracle数据库和IBM大型机上的各种程序（包括IBM CICS，IMS和APPC / MVS）之间传输信息。

有关适用于APPC的Oracle数据库网关的通信功能的说明，请参阅Microsoft Windows上的%ORACLE_HOME%\dg4appc\demo\CICS\pgacics.sql或基于UNIX的平台上的$ORACLE_HOME/dg4appc/demo/CICS/pgacics.sql 。这是Oracle数据库和CICS Transaction Server for z / OS之间的示例通信。执行这个简单 PL / SQL过程pgacics.sql ，导致Oracle数据库调用使用的数据库网关 SNA与之交谈 CICS中的FLIP事务。在使用SNA在网关和大型机之间进行通信的步骤中详细描述了这些步骤。请注意，在配置网关时，您已经编译并链接了存储过程。

使用TCP / IP的网关的事务类型

该使用TCP / IP支持IMS Connect的Oracle Database Gateway for APPC支持三种类型的事务套接字连接：

  • 交易套接字。

    套接字连接跨越单个事务。

  • 持久套接字。

    套接字连接跨多个事务。

  • 非持久套接字。

    套接字连接跨越由一个输入和一个输出组成的单个交换。

    注意：

    如果计划实现对话事务，请不要使用非持久套接字类型，因为会发生多个连接和断开连接。

    有关这些参数的功能和用法的更多信息，请参阅本指南的PG4TCPMAP命令（仅限TCP / IP）中有关pg4tcpmap命令的部分。
