57 DBMS_DESCRIBE

您可以使用DBMS_DESCRIBE包来获取有关PL / SQL对象的信息。指定对象名称时， DBMS_DESCRIBE将返回一组带有结果的索引表。执行全名转换，并检查最终对象的安全检查。

57.1 DBMS_DESCRIBE概述

此程序包提供与Oracle Call Interface OCIDescribeAny调用相同的功能。

也可以看看：

Oracle调用接口程序员指南

57.2 DBMS_DESCRIBE安全模型

该程序包可供PUBLIC使用，并根据所描述的模式对象执行自己的安全检查。

57.3 DBMS_DESCRIBE类型

DBMS_DESCRIBE包声明了两种PL / SQL表类型，用于保存DESCRIBE_PROCEDURE在其OUT参数中返回的数据。

类型是：

TYPE VARCHAR2_TABLE是VARCHAR2（30）索引表BINARY_INTEGER; TYPE NUMBER_TABLE是BINARY_INTEGER的数字索引表;

57.4 DBMS_DESCRIBE例外

57.6.1 DESCRIBE_PROCEDURE程序

过程DESCRIBE_PROCEDURE提供了PL / SQL存储过程的简要说明。

它采用存储过程的名称并返回有关该过程的每个参数的信息。

句法

DBMS_DESCRIBE.DESCRIBE_PROCEDURE（object_name IN VARCHAR2，reserved1 IN VARCHAR2，reserved2 IN VARCHAR2，过载OUT NUMBER_TABLE，位置OUT NUMBER_TABLE，level OUT OUT VARCHAR2_TABLE，数据类型OUT NUMBER_TABLE，default_value OUT NUMBER_TABLE，in_out OUT NUMBER_TABLE，长度OUT NUMBER_TABLE，精度OUT NUMBER_TABLE，缩小OUT NUMBER_TABLE，基数OUT NUMBER_TABLE，备用OUT NUMBER_TABLE include_string_constraints OUT BOOLEAN DEFAULT FALSE）;

参数

表57-3 DBMS_DESCRIBE.DESCRIBE_PROCEDURE参数

参数 描述

OBJECT_NAME

所描述的过程的名称。

此参数的语法遵循SQL中用于标识符的规则。名称可以是同义词。此参数是必需的，可能不为null。名称的总长度不能超过197个字节。错误指定的OBJECT_NAME可能会导致以下异常之一：

ORA-20000 - 指定了一个包。您只能指定存储过程，存储函数，打包过程或打包函数。

ORA-20001 - 您指定的过程或函数在给定的包中不存在。

ORA-20002 - 您指定的对象是远程对象。此过程当前无法描述远程对象。

ORA-20003 - 您指定的对象无效且无法描述。

ORA-20004 - 使用语法错误指定了对象。

reserved1 reserved2

保留供将来使用 - 必须设置为NULL或空字符串。

超载

分配给过程签名的唯一编号。

如果过程重载，则此字段为每个过程版本保留不同的值。

位置

参数在参数列表中的位置。

位置0返回函数返回类型的值。

水平

如果参数是复合类型（例如record），则此参数返回数据类型的级别。有关示例的ODESSP调用说明，请参阅“ Oracle调用接口程序员指南” 。

argument_name

与您描述的过程关联的参数的名称。

数据类型

正在描述的参数的Oracle数据类型。数据类型及其数字类型代码为：

没有参数的程序的0占位符1 VARCHAR，VARCHAR，STRING 2 NUMBER，INTEGER，SMALLINT，REAL，FLOAT，DECIMAL 3 BINARY_INTEGER，PLS_INTEGER，POSITIVE，NATURAL 8 LONG 11 ROWID 12 DATE 23 RAW 24 LONG RAW 58 OPAQUE TYPE 96 CHAR（ ANSI固定字符，字符106 MLSLABEL 121对象122嵌套表123 VARRAY 178时间179时区与时区180 TIMESTAMP 181 TIMESTAMP与时区231 TIMESTAMP与本地时区250 PL / SQL RECORD 251 PL / SQL表252 PL/SQL BOOLEAN

默认值

如果描述的参数具有默认值，则为1;否则，该值为0。

进出

描述参数的模式：

0 IN 1 OUT 2 IN OUT

长度

对于%rowtype形式参数，返回长度约束，否则返回0。如果include_string_constraints参数设置为TRUE, the参数的类型合适，则会传回参数的形式长度约束。这些是字符串类型：1; 8; 23; 24; 96

精确

如果描述的参数是数据类型2（ NUMBER ），则此参数是该数字的精度。

规模

如果描述的参数是数据类型2（ NUMBER ），则此参数是该数字的比例。

基数

如果描述的参数是数据类型2（ NUMBER ），则此参数是该数字的基数。

备用

保留用于将来的功能。

include_string_constraints

默认值为FALSE 。如果参数设置为TRUE ，则如果参数的形式类型约束属于适当的类型，则会将其传回。这些是字符串类型：1; 8; 23; 24; 96

返回值

DESCRIBE_PROCEDURE中的所有值都在其OUT参数中返回。这些数据类型是PL / SQL表，以容纳可变数量的参数。
