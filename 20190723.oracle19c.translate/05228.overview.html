1 PL / SQL概述

PL / SQL是SQL的Oracle过程扩展，是一种可移植的高性能事务处理语言。本概述解释了其优点，并简要介绍了其主要特性和架构。

话题

  • PL / SQL的优点

  • PL / SQL的主要特点

  • PL / SQL的体系结构

1.1 PL / SQL的优点

PL / SQL与其他编程语言相比具有几个优点。

PL / SQL具有以下优点：

  • 与SQL紧密集成

  • 高性能

  • 高生产力

  • 可移植性

  • 可扩展性

  • 可管理性

  • 支持面向对象的编程

1.1.1与SQL紧密集成

PL / SQL与SQL（最广泛使用的数据库操作语言）紧密集成。

例如：

  • PL / SQL允许您使用所有SQL数据操作，游标控制和事务控制语句，以及所有SQL函数，运算符和伪列。

  • PL / SQL完全支持SQL数据类型。

    您无需在PL / SQL和SQL数据类型之间进行转换。例如，如果PL / SQL程序从SQL类型VARCHAR2的列中检索值，则它可以将该值存储在VARCHAR2类型的PL / SQL变量中。

    您可以为PL / SQL数据项提供数据库表的列或行的数据类型，而无需显式指定该数据类型（请参阅“ 使用％TYPE属性 ”和“ 使用％ROWTYPE属性 ” ）。

  • PL / SQL允许您运行SQL查询并一次处理一个结果集的行 （请参阅“一次处理一行查询结果集 ” ）。

  • 可以在SQL SELECT语句的WITH子句中声明和定义PL / SQL函数（请参阅Oracle数据库SQL语言参考 ）。

PL / SQL支持静态和动态SQL。 静态SQL是SQL，其全文在编译时是已知的。动态SQL是SQL，其全文在运行时才知道。动态SQL使您可以使应用程序更加灵活和通用。有关更多信息，请参阅PL / SQL静态SQL和PL / SQL动态SQL 。

1.1.2高性能

PL / SQL允许您向数据库发送一个语句块，从而显着减少应用程序和数据库之间的流量。

绑定变量

当您在PL / SQL代码中直接嵌入SQL INSERT ， UPDATE ， DELETE ， MERGE或SELECT语句时，PL / SQL编译器将WHERE和VALUES子句中的变量转换为绑定变量（有关详细信息，请参阅“ 名称解析”）在静态SQL语句中 “ ）。每次运行相同的代码时，Oracle数据库都可以重用这些SQL语句，从而提高性能。

使用动态SQL时，PL / SQL不会自动创建绑定变量，但您可以通过动态SQL明确指定它们来使用它们（有关详细信息，请参阅“ EXECUTE IMMEDIATE语句 ” ）。

子程序

PL / SQL子程序以可执行的形式存储，可以重复调用。由于存储的子程序在数据库服务器中运行，因此通过网络进行的单个调用可以启动大型作业。这种分工可以减少网络流量并缩短响应时间。存储的子程序在用户之间进行缓存和共享，从而降低了内存需求和调用开销。有关子程序的详细信息，请参阅“ 子程序 ” 。

优化

PL / SQL编译器有一个优化器，可以重新排列代码以获得更好的性能。有关优化程序的更多信息，请参阅“ PL / SQL优化程序 ” 。

1.1.3高生产率

PL / SQL具有许多可以节省设计和调试时间的功能，并且在所有环境中都是相同的。

PL / SQL允许您编写用于操作数据的紧凑代码。就像PERL这样的脚本语言可以读取，转换和写入文件中的数据一样，PL / SQL也可以查询，转换和更新数据库中的数据。

如果您学习将PL / SQL与一个Oracle工具结合使用，则可以将您的知识转移到其他Oracle工具。有关PL / SQL功能的概述，请参阅“ PL / SQL的主要功能 ” 。

1.1.4便携性

PL / SQL是Oracle开发的可移植和标准语言。

您可以在运行Oracle数据库的任何操作系统和平台上运行PL / SQL应用程序。

1.1.5可扩展性

PL / SQL存储的子程序通过在数据库服务器上集中应用程序处理来提高可伸缩性。

共享服务器的共享内存功能使Oracle数据库可以在单个节点上支持数千个并发用户。有关子程序的详细信息，请参阅“ 子程序 ”

为了进一步扩展，您可以使用Oracle Connection Manager来复用网络连接。有关Oracle Connection Manager的信息，请参阅Oracle Database Net Services Reference

1.1.6可管理性

PL / SQL存储的子程序增加了可管理性，因为您只能在数据库服务器上维护子程序的一个副本，而不是每个客户端系统上的一个副本。

任意数量的应用程序都可以使用子程序，您可以更改子程序，而不会影响调用它们的应用程序。有关子程序的详细信息，请参阅“ 子程序 ” 。

1.1.7支持面向对象的编程

PL / SQL允许定义可用于面向对象设计的对象类型。

PL / SQL支持使用“ 抽象数据类型 ”的面向对象编程。

1.2 PL / SQL的主要特点

PL / SQL将SQL的数据操作功能与过程语言的处理能力相结合。

当您可以解决SQL问题时，您可以从PL / SQL程序发出SQL语句，而无需学习新的API。

与其他过程编程语言一样，PL / SQL允许您声明常量和变量，控制程序流，定义子程序和捕获运行时错误。

您可以将复杂问题分解为易于理解的子程序，您可以在多个应用程序中重复使用这些子程序。

1.2.1错误处理

PL / SQL使检测和处理错误变得容易。

发生错误时，PL / SQL会引发异常。正常执行停止并控制传输到PL / SQL块的异常处理部分。您不必检查每个操作以确保它成功，就像在C程序中一样。

有关更多信息，请参阅PL / SQL错误处理 。

1.2.2块

PL / SQL源程序的基本单元是块 ，它对相关的声明和语句进行分组。

PL / SQL块由关键字DECLARE ， BEGIN ， EXCEPTION和END 。这些关键字将块分为声明部分，可执行部分和异常处理部分。只需要可执行部分。块可以有标签。

声明是块的本地声明，并且在块完成执行时不再存在，有助于避免变量和子程序的混乱命名空间。

块可以嵌套：因为块是可执行语句，所以只要允许可执行语句，它就可以出现在另一个块中。

您可以将块提交给交互式工具（例如SQL * Plus或Enterprise Manager），也可以将其嵌入Oracle预编译器或OCI程序中。交互式工具或程序运行块一次。该块未存储在数据库中，因此，它被称为匿名块 （即使它具有标签）。

每次将匿名块加载到内存时都会编译它，并且它的编译有三个阶段：

  1. 语法检查：检查PL / SQL语法，并生成解析树。

  2. 语义检查：在解析树上键入检查和进一步处理。

  3. 代码生成

注意：

匿名块是SQL语句。

有关语法详细信息，请参阅“ 阻止 ” 。

1.2.3变量和常数

PL / SQL允许您声明变量和常量，然后在任何可以使用表达式的地方使用它们。

当程序运行时，变量的值可以改变，但常量的值不能改变。

有关更多信息，请参阅“ 声明 ”和“ 将值赋给变量 ” 。

1.2.4子程序

PL / SQL 子程序是可以重复调用的命名PL / SQL块。

如果子程序具有参数，则每个调用的值可能不同。PL / SQL有两种类型的子程序，过程和函数。函数返回结果。

有关PL / SQL子程序的更多信息，请参阅PL / SQL子程序 。

PL / SQL还允许您调用用其他语言编写的外部程序。

有关更多信息，请参见“ 外部子程序 ” 。

1.2.5包

包是一个模式对象，它对逻辑上相关的PL / SQL类型，变量，常量，子程序，游标和异常进行分组。

包被编译并存储在数据库中，其中许多应用程序可以共享其内容。您可以将包视为应用程序。

您可以编写自己的包 - 有关详细信息，请参阅PL / SQL包 。您还可以使用Oracle数据库提供的许多特定于产品的软件包。有关这些的信息，请参阅Oracle Database PL / SQL包和类型参考 。

1.2.6触发器

触发器是一个命名的PL / SQL单元，它存储在数据库中并运行以响应数据库中发生的事件。

您可以指定事件，触发器在事件之前或之后触发，以及触发器是针对每个事件还是针对受事件影响的每一行运行。例如，您可以创建一个每次INSERT语句影响EMPLOYEES表时运行的触发器。

有关触发器的更多信息，请参阅PL / SQL触发器 。

1.2.8数据抽象

数据抽象使您可以使用数据的基本属性，而无需过多地参与细节。

您可以先设计数据结构，然后设计操作它的算法。

1.2.8.1游标

游标是指向私有SQL区域的指针，该区域存储有关处理特定SQL语句或PL / SQL SELECT INTO语句的信息。

您可以使用游标一次检索一个结果集的行。您可以使用游标属性来获取有关游标状态的信息 - 例如，语句到目前为止已经影响了多少行。

有关游标的更多信息，请参阅“ 游标概述 ” 。

1.2.8.2复合变量

复合变量具有内部组件，您可以单独访问它们。

您可以将整个复合变量作为参数传递给子程序。PL / SQL有两种复合变量，集合和记录。

在集合中 ，内部组件始终具有相同的数据类型，并且称为元素 。您可以通过其唯一索引访问每个元素。列表和数组是集合的经典示例。

在记录中 ，内部组件可以是不同的数据类型，并称为字段 。您可以按名称访问每个字段。记录变量可以保存表行或表行中的某些列。

有关复合变量的更多信息，请参阅PL / SQL集合和记录 。

1.2.8.3使用％ROWTYPE属性

%ROWTYPE属性允许您声明一条记录，该记录表示数据库表或视图的完整行或部分行。

对于完整行或部分行的每一列，记录都有一个具有相同名称和数据类型的字段。如果行的结构发生变化，则记录的结构会相应地发生变化。

有关%ROWTYPE语法和语义的详细信息，请参阅“ ％ROWTYPE属性 ” 。有关其用法的更多详细信息，请参阅“ 使用％ROWTYPE属性声明项目 ” 。

1.2.8.4使用％TYPE属性

%TYPE属性允许您声明与先前声明的变量或列具有相同数据类型的数据项（不知道该类型是什么）。

如果引用项的声明发生更改，则引用项的声明会相应更改。在声明用于保存数据库值的变量时， %TYPE属性特别有用。有关%TYPE语法和语义的详细信息，请参阅“ ％TYPE属性 ” 。有关其用法的更多详细信息，请参阅“ 使用％TYPE属性声明项目 ” 。

1.2.8.5抽象数据类型

抽象数据类型（ADT）由数据结构和操纵数据的子程序组成。

形成数据结构的变量称为属性 。操纵属性的子程序称为方法 。

ADT存储在数据库中。ADT的实例可以存储在表中并用作PL / SQL变量。

ADT可以通过将大型系统分离为可以重用的逻辑组件来降低复杂性。

在静态数据字典视图*_OBJECTS ，所述OBJECT_TYPE一个ADT的是TYPE 。在静态数据字典视图*_TYPES ，ADT的TYPECODE是OBJECT 。

有关ADT的详细信息，请参阅“ CREATE TYPE语句 ” 。

注意：

ADT也称为用户定义类型和对象类型 。

1.2.9控制声明

控制语句是SQL最重要的PL / SQL扩展。

PL / SQL有三类控制语句：

  • 条件选择语句 ，允许您为不同的数据值运行不同的语句。

    有关更多信息，请参阅“ 条件选择语句 ” 。

  • 循环语句 ，允许您使用一系列不同的数据值重复相同的语句。

    有关更多信息，请参阅“ LOOP语句 ” 。

  • 顺序控制语句 ，允许您转到指定的带标签的语句，或者不执行任何操作。

    有关更多信息，请参阅“ 顺序控制语句 ” 。

1.2.10条件编译

条件编译允许您在不删除源文本的情况下自定义PL / SQL应用程序中的功能。

例如，您可以：

  • 使用具有最新数据库版本的新功能，并在较旧的数据库版本中运行应用程序时禁用它们。

  • 在开发环境中激活调试或跟踪语句，并在生产站点上运行应用程序时隐藏它们。

有关更多信息，请参阅“ 条件编译 ” 。

1.3.1 PL / SQL引擎

PL / SQL编译和运行时系统是一个编译和运行PL / SQL单元的引擎。

引擎可以安装在数据库中或应用程序开发工具中，例如Oracle Forms。

在任一环境中，PL / SQL引擎都接受任何有效的PL / SQL单元作为输入。引擎运行过程语句，但将SQL语句发送到数据库中的SQL引擎， 如图1-1所示。

“图1-1 PL / SQL引擎”的描述

通常，数据库处理PL / SQL单元。

当应用程序开发工具处理PL / SQL单元时，它会将它们传递给本地PL / SQL引擎。如果PL / SQL单元不包含SQL语句，则本地引擎处理整个PL / SQL单元。如果应用程序开发工具可以从条件和迭代控制中受益，这将非常有用。

例如，Oracle Forms应用程序经常使用SQL语句来测试字段条目的值并进行简单的计算。通过使用PL / SQL而不是SQL，这些应用程序可以避免对数据库的调用。

注意：

不推荐使用编译参数PLSQL_DEBUG ，该参数指定是否编译PL / SQL单元以进行调试。要编译PL / SQL单元以进行调试，请指定PLSQL_OPTIMIZE_LEVEL=1 。

表1-2中的参数的编译时值与每个存储的PL / SQL单元的元数据一起存储，这意味着您可以在显式重新编译单元时重用这些值。（使用“ CREATE [OR REPLACE]语句之一” 创建 存储的PL / SQL单元 。匿名块不是存储的PL / SQL单元。）

要显式重新编译存储的PL / SQL单元并重用其参数值，必须使用包含COMPILE子句和REUSE SETTINGS子句的ALTER语句。所有ALTER语句都有此子句。有关ALTER语句的列表，请参阅“ ALTER语句 ” 。
