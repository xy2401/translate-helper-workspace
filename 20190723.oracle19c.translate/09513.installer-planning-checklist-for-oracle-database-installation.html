使用CVU运行OUI并使用修复脚本

Oracle Universal Installer与Cluster Verification Utility（CVU）完全集成，可自动执行许多CVU先决条件检查。运行安装程序时，Oracle Universal Installer会运行所有先决条件检查并创建修订脚本。您可以在不启动安装的情况下运行OUI直到“摘要”屏幕。

您也可以手动运行CVU命令以检查系统准备情况。有关更多信息，请参阅：

“Oracle Clusterware管理和部署指南”
更新并运行Oracle ORAchk以进行运行时和升级检查或运行时运行状况检查

Oracle建议您更新到最新版本的Oracle ORAchk。

Oracle ORAchk实用程序提供的系统检查有助于防止安装前后出现问题。这些检查包括内核要求，操作系统资源分配和其他系统要求。

使用Oracle ORAchk升级准备情况评估获取自动升级特定的系统运行状况检查，以便升级到11.2.0.3,11.2.0.4,12.1.0.1,12.1.0.2,12.2,18c和19c。例如：

  • 在执行全新数据库安装之前：

    ./orachk -profile预安装

  • 要将现有数据库升级到更高版本或版本：

    ./orachk -u -o pre

Oracle ORAchk升级准备评估可自动执行Oracle升级文档中描述的许多手动升级前和升级后检查。有关Oracle ORAchk支持的更多信息，请查看My Oracle Support说明1268927.1。

https://support.oracle.com/rs?type=doc&id=1268927.2
验证是否已安装Oracle Grid Infrastructure

如果要使用Oracle ASM或Oracle Restart，请在安装和创建数据库之前为独立服务器安装Oracle Grid Infrastructure。否则，要使用Oracle ASM，您必须完成Oracle Grid Infrastructure安装，然后使用Oracle Restart手动注册数据库。

对于Oracle Real Application Clusters（Oracle RAC）安装，请确保已为群集安装和配置Oracle Grid Infrastructure。

检查正在运行的Oracle进程，并在必要时关闭

  • 在不使用Oracle ASM的独立数据库上：安装Oracle Grid Infrastructure时无需关闭数据库。

  • 在使用Oracle ASM的独立数据库上：在安装期间重新启动Oracle ASM实例。

  • 在Oracle RAC数据库节点上：此安装需要升级Oracle Clusterware，因为运行Oracle RAC需要Oracle Clusterware。作为升级的一部分，随着滚动升级从节点到节点，您必须一次关闭数据库一个节点。
确保在安装期间不运行cron作业 如果安装程序在每日cron作业启动时运行，那么如果您的cron作业正在执行清理，则可能会遇到无法解释的安装问题，并且在安装完成之前删除临时文件。Oracle建议您在每天运行cron作业之前完成安装，或者每天禁用

查看内存分配和自动内存管理功能

您可以在Oracle数据库安装期间或之后启用自动内存管理。如果在安装后启用自动内存管理，则必须关闭并重新启动数据库。

如果数据库实例的总物理内存大于4 GB，则在数据库安装和创建期间无法选择Oracle自动内存管理选项。而是使用自动共享内存管理。自动共享内存管理根据需要自动在各种组件之间分配可用内存，从而允许系统最大限度地使用所有可用的SGA内存。

有关更多信息，请参阅：

Oracle数据库管理员指南

确定安装的超级用户（ root ）权限委派选项

在数据库或网格基础结构安装期间，系统会要求您以root用户身份运行配置脚本。

您可以在出现提示时以root身份手动运行这些脚本，也可以使用root权限委派选项提供配置信息和密码。

要自动运行root脚本，请选择安装期间自动运行配置脚本 。

  • 使用root用户凭据

    提供集群成员节点服务器的超级用户密码。

  • 使用Sudo

    须藤是UNIX和Linux的工具，它允许的sudoers文件列表的权限成员运行单独的命令为root 。提供作为sudoers成员的操作系统用户的用户名和密码，并且有权在每个集群成员节点上运行Sudo。

    要启用Sudo，请让具有相应权限的系统管理员配置作为sudoers列表成员的用户，并在安装期间提示时提供用户名和密码。
