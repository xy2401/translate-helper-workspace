词汇表

应用实例

在Oracle Real Application Clusters（Oracle RAC）数据库环境中，apply实例是将归档重做数据应用于备用数据库的一个实例。

经纪人

一种分布式管理框架，可自动化并简化创建，控制和监视Oracle Data Guard配置所需的大多数复杂操作。

经纪人配置

Oracle Data Guard配置的主数据库和备用数据库（包括重做传输服务和日志应用服务）的逻辑分组。

另请参见 Data Guard配置 。

旁观者备用数据库

备用数据库，它不是切换或故障转移操作的目标或直接参与切换或故障转移操作。

配置对象

命名的数据库对象集合。它是Oracle Data Guard配置的抽象。

数据库守卫

数据库防护控制是否可以对逻辑备用数据库中的表进行修改。

数据库对象

与Oracle Data Guard配置中的主数据库或备用数据库对应的命名对象。代理使用此对象来管理和控制单个数据库的状态，并将属性与数据库关联。

Data Guard命令行界面

Oracle Data Guard命令行界面（DGMGRL）使您能够从DGMGRL提示或脚本中控制和监视Oracle Data Guard配置。

Data Guard配置

一种分布式计算系统，可防止或最大限度地减少因计划外事件（例如，人为错误，环境灾难或数据损坏）导致的损失以及计划停机时间（例如日常维护任务）。

另请参见 代理配置 。

Data Guard环境

主数据库，备用数据库和远程同步实例的物理配置。环境取决于许多因素，包括：

  • 与主数据库关联的备用数据库和远程同步实例的数量

  • 数据库使用的主机系统数

  • 数据库使用的计算机的目录结构

  • 网络配置

  • 重做运输服务

  • 记录应用服务

Oracle Data Guard环境可由DBA手动管理，使用Enterprise Manager或Data Guard命令行界面（DGMGRL）或所有这些的组合自动管理。

默认状态

启用配置的代理管理时，数据库对象将在其中运行的初始运行时状态。实际的默认状态可能因数据库运行的角色（主要或备用）而异。

另见 预期状态 。

故障转移

故障转移将备用数据库更改为主数据库的角色，并禁用旧的主数据库。

另请参见 快速启动故障转移和手动故障转移 。

远同步实例

远程Oracle Data Guard目标，它接受来自主数据库的重做，并在整个Oracle Data Guard配置中重新分配该重做。它类似于物理备用数据库，因为它管理控制文件，将重做接收到备用重做日志（SRL），并将这些SRL归档到本地存档重做日志（ARL）。但与备用数据库不同，远程同步实例不管理数据文件，无法打开访问权限，也无法运行重做应用。

快速启动故障转移

主数据库不可用时自动启用故障转移。启用快速启动故障切换后，代理会确定是否需要进行故障切换，并自动，快速且可靠地故障切换到指定的备用数据库，而无需执行任何手动步骤来调用故障切换。

另请参见 手动故障转移 。

实例对象

命名对象;数据库对象是一个或多个命名实例对象的集合。代理使用此对象来管理和控制与实例关联的数据库的状态，并将特定于实例的属性与此数据库实例相关联。

预期的状态

启用代理管理时数据库对象的运行时状态。

另请参见 默认状态 。

熄灭计算

指操作自动化的计算设备或软件，人类管理员几乎不需要干预。

术语“熄灯”源于计算中心位于一个房间并包含许多服务器，这些服务器保持锁定和密钥以及黑暗中。在正常操作下，人工管理员没有输入房间，房间内的所有操作都是自动化的。

逻辑备用数据库

逻辑备用数据库采用标准的Oracle归档重做日志文件，将它们转换回SQL事务，然后将它们应用于开放备用数据库。尽管可以与最终用户访问同时应用更改，但通过重新生成的SQL事务维护的表允许对逻辑备用数据库的用户进行只读访问。由于数据库已打开以允许应用重构的SQL事务，因此它与主数据库在物理上不同。数据库表可以与其主数据库对等体具有不同的索引和物理特性，但这些表必须从应用程序访问角度保持逻辑一致性，以履行其作为备用数据源的角色。

手动故障转移

首先确定故障转移的DBA启动故障转移，然后通过单击企业管理器中的FAILOVER或发出DGMGRL FAILOVER命令来调用故障转移。“ 手动 ”一词用于将此类故障转移与快速启动故障转移进行对比。

另请参见 快速启动故障转移 。

观察者

DGMGRL客户端，持续监视主数据库和目标备用数据库，评估是否需要进行故障转移，并在条件允许时启动快速启动故障转移。

物理备用数据库

物理备用数据库是主数据库的精确副本。当主数据库处于打开和活动状态时，物理备用数据库使用重做应用来应用从主数据库接收的重做数据。重做应用可以在已装入的物理备用数据库实例上运行。如果已购买Oracle Active Data Guard选件的许可证，则Redo Apply也可以在打开的物理备用数据库实例上运行。有关更多详细信息，请参阅Oracle Data Guard概念和管理 。

主数据库

生成数据库，从中创建和维护一个或多个备用数据库。每个备用数据库都与一个且仅一个主数据库相关联。但是，单个主数据库可以支持多个备用数据库。Oracle Data Guard代理监视器（DMON）使用主数据库维护二进制配置文件的主副本，确保每个备用数据库的文件副本在进行更改时保持最新。

代理使用DB_UNIQUE_NAME初始化参数中的值来引用此数据库，该参数被定义为全局唯一。

只读模式

可以打开允许查询但不允许修改的数据库的模式。

物理备用数据库可以只读方式打开，以便可以执行查询。如果已购买Oracle Active Data Guard选件的许可证，则可以在Redo Apply处于活动状态时打开物理备用数据库。此功能称为实时查询。有关更多详细信息，请参阅Oracle Data Guard概念和管理 。

重做申请

物理备用数据库通过Redo Apply与主数据库保持同步，该数据库恢复从主数据库接收的重做数据并将重做应用于物理备用数据库。

复职

恢复是将故障转移操作后已禁用的数据库（包括旧主数据库）转换为新主数据库的可用备用数据库的过程。必须在数据库上启用闪回数据库才能恢复它。

快照备用数据库

快照备用数据库是完全可更新的备用数据库，它是从物理备用数据库创建的。在快照备用数据库上，接收重做数据，但在将快照备用数据库转换回物理备用数据库之前不会应用重做数据。

SQL Apply

逻辑备用数据库通过SQL Apply与主数据库保持同步，SQL Apply将从主数据库接收的重做数据转换为SQL语句，然后在备用数据库上执行SQL语句。

备用数据库

使用主数据库备份创建的主数据库的副本。通过随时间从主数据库到每个备用数据库应用归档重做数据，备用数据库与主数据库保持同步。备用数据库可以接管主数据库的处理，提供几乎连续的数据库可用性。备用数据库具有自己的服务器参数文件，控制文件和数据文件。它还具有代理配置文件的副本，在主数据库实例中运行的DMON进程的指导下保持最新。

代理通过其全局唯一名称引用备用数据库，该名称存储在其DB_UNIQUE_NAME初始化参数中。

另请参见 逻辑备用数据库 ， 物理备用数据库和快照备用数据库 。
