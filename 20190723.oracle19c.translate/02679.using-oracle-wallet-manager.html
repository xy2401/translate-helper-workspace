什么是Oracle钱包管理器？

您可以使用Oracle Wallet Manager管理Oracle客户端和服务器上的公钥安全凭证。它创建的钱包可以由Oracle数据库，Oracle应用服务器和Oracle身份管理基础架构读取。

Oracle Wallet Manager使钱包所有者能够管理和编辑Oracle钱包中的安全凭证。钱包是受密码保护的容器，用于存储身份验证和签名凭据，包括SSL所需的私钥，证书和可信证书。您可以使用Oracle Wallet Manager执行以下任务：

  • 创建钱包

  • 生成证书请求

  • 打开钱包以访问基于PKI的服务

  • 使用符合公钥加密标准＃11（ PKCS＃11 ）规范的API将凭据保存到硬件安全模块

  • 将钱包上传到LDAP目录（并从中下载）

  • 导入第三方PKCS＃12 -format钱包

  • 将Oracle钱包导出到第三方环境

钱包密码管理

神谕钱包受密码保护。Oracle Wallet Manager包含一个增强型钱包密码管理模块，可执行密码管理策略指南，包括以下内容：

  • 最小密码长度（8个字符）

  • 最大密码长度不限

  • 需要字母数字字符组合

强大的钱包加密

Oracle钱包管理器存储与X.509证书关联的私钥并使用AES加密。

Microsoft Windows注册表钱包存储

Oracle钱包管理器允许您在Windows文件管理系统或Microsoft Windows系统注册表的用户配置文件区域中存储多个Oracle钱包。将您的钱包存储在注册表中可提供以下好处：

  • 更好的访问控制：存储在注册表的用户配置文件区域中的钱包只能由关联用户访问。因此，系统的用户访问控制因此成为钱包的访问控制。此外，当用户退出系统时，有效地排除了对该用户钱包的访问。

  • 更轻松的管理：钱包与特定的用户配置文件相关联，因此不需要管理文件权限，并且在删除用户配置文件时会自动删除存储在配置文件中的钱包。您可以使用Oracle Wallet Manager在注册表中创建和管理钱包。

支持的选项如下：

  • 从注册表中打开钱包

  • 将钱包保存到注册表

  • 另存为注册表位置

  • 从注册表中删除钱包

  • 从文件系统打开钱包并将其保存到注册表

  • 从注册表中打开钱包并将其保存到文件系统

使用电子钱包管理器创建的电子钱包文件所需的ACL设置

在Microsoft Windows系统上，从Oracle Database 12c （版本12.1）开始，您可能需要手动设置文件系统ACL，例如，授予对使用Wallet Manager创建的文件系统中的钱包的访问权限。由于Oracle数据库服务现在在低权限用户下运行，因此除非文件系统访问控制列表（ACL）授予对文件的访问权限，否则Oracle数据库服务可能无法访问该文件。虽然Oracle安装以某种方式配置ACL，以确保您不必手动更改ACL以用于典型用途，但可能需要手动更改ACL。

向后兼容性

Oracle Wallet Manager是向后兼容版本8.1.7。

公钥加密标准（PKCS）支持

RSALaboratories是RSA Security，Inc。的一个部门，与业界，学术界和政府的代表合作开发了一系列称为公钥加密标准的基本加密标准，简称PKCS。这些标准建立了使用公钥技术保护内部网和互联网数据的计算机系统之间的互操作性。

Oracle Wallet Manager以PKCS＃12格式存储X.509证书和私钥 ，并根据PKCS＃10规范生成证书请求。这些功能使Oracle钱包结构可与受支持的第三方PKI应用程序互操作，并提供跨操作系统的钱包可移植性。

Oracle Wallet Manager钱包可以在使用符合PKCS＃11规范的API的硬件安全模块上存储凭据。当选择PKCS11作为钱包类型创建钱包时，存储在该钱包中的所有密钥都将保存到硬件安全模块或令牌中。这种硬件设备的示例包括智能卡， PCMCIA卡 ，智能软盘或存储私钥或执行加密操作（或两者）的其他便携式硬件设备。

注意：

要在64位Solaris操作系统上使用Oracle Wallet Manager与PKCS＃11集成，请在命令行中输入以下内容：

owm -pkcs11

多证书支持

Oracle Wallet Manager使您可以在每个钱包中存储多个证书 ，支持以下任何Oracle PKI证书用法 ：

  • SSL身份验证

  • S / MIME签名

  • S / MIME加密

  • 代码签名

  • CA证书签名

您创建的每个证书请求都会生成一个唯一的私钥/公钥对。私钥保留在钱包中，公钥随请求一起发送给证书颁发机构。当该证书颁发机构生成您的证书并对其进行签名时，您只能将其导入到具有相应私钥的钱包中。

如果钱包还包含单独的证书请求，则与该请求相对应的私钥/公钥对当然不同于第一证书请求的对。将此单独的证书请求发送到证书颁发机构可以获得单独的签名证书，您可以将其导入到同一个钱包中

可以多次向证书颁发机构发送单个证书请求以获取多个证书。但是，钱包中只能安装与该证书请求对应的一个证书。

Oracle Wallet Manager使用X.509版本3 KeyUsage扩展来定义Oracle PKI证书使用情况（ 表6-1 ）。单个证书不能应用于所有可能的证书使用。表6-2和表6-3显示了合法使用组合。

LDAP目录支持

Oracle Wallet Manager可以将钱包上传到LDAP兼容目录并从中检索它们。将钱包存储在集中的LDAP兼容目录中，用户可以从多个位置或设备访问它们，确保一致和可靠的用户身份验证，同时在整个钱包生命周期中提供集中的钱包管理。为防止用户意外覆盖功能钱包，只能上载包含已安装证书的钱包。

必须先在LDAP目录中定义和配置目录用户条目，然后才能使用Oracle Wallet Manager为用户上载或下载钱包。如果目录包含Oracle8 i （或之前）用户，则会在首次使用时自动升级以使用钱包上载和下载功能。

Oracle Wallet Manager通过使用与LDAP目录的简单基于密码的连接来下载用户钱包。但是，对于上载，如果打开的钱包包含具有SSL Oracle PKI证书使用的证书，则它使用SSL连接。如果钱包中不存在SSL证书，则使用基于密码的身份验证。

注意：

目录密码和钱包密码是独立的，可以是不同的。Oracle建议保持这些密码始终不同，其中任何一个密码都不能在逻辑上从另一个密码派生。

创建Oracle Wallet的一般过程

Oracle钱包提供了一个必要的存储库，您可以在其中安全地存储用户证书以及验证同行证书所需的信任点 。

以下步骤概述了完整的钱包创建过程：

  1. 使用Oracle Wallet Manager创建新钱包。
  2. 生成证书请求。使用Oracle Wallet Manager创建新钱包时，该工具会自动提示您创建证书申请。
  3. 将证书请求发送到您要使用的CA.您可以将证书请求文本复制并粘贴到电子邮件中，也可以将证书请求导出到文件中。证书请求成为您钱包的一部分。它必须保留在那里，直到您删除其相关的证书。
  4. 当CA发送您签名的用户证书及其关联的可信证书时 ，您可以按以下顺序导入这些证书。可以同时导入PKCS＃7格式的用户证书和可信证书。

    • 首先将CA的可信证书导入您的钱包。如果新用户证书已由其中一个CA（默认情况下已存在于Oracle Wallet Manager中的可信证书）颁发，则此步骤可以是可选的。

    • 成功导入可信证书后，将CA发送给您的用户证书导入到钱包中。

  5. （可选）设置钱包的自动登录功能。

    通常，大多数钱包都需要此功能，该功能可以在没有密码的情况下启用基于PKI的服务访问。它是数据库服务器和客户端钱包所必需的。它仅适用于在启动时获取钱包密码的产品。

完成上述过程后，您将拥有一个包含用户证书及其关联信任点的钱包。

创建Oracle Wallet密码的必备指南

由于Oracle钱包包含可用于向多个数据库验证用户身份的用户凭据，因此选择强钱包密码尤为重要。猜测钱包密码的恶意用户可以访问钱包所有者可以访问的所有数据库。

密码必须包含至少八个字符，这些字符由字母字符和数字或特殊字符组成。

注意：

强烈建议您避免根据用户名，电话号码或政府识别号码选择容易猜到的密码。这可以防止潜在的攻击者使用个人信息来推断用户的密码。用户定期更改密码也是一种谨慎的安全措施，例如每月一次或每季一次。

更改密码时，必须重新生成自动登录钱包。

创建新的Oracle Wallet

您可以使用Oracle Wallet Manager创建PKCS＃12钱包（标准默认钱包类型），将凭证存储在文件系统的目录中。它还可用于创建PKCS＃11钱包，用于在服务器的硬件安全模块上存储凭据，或在客户端的令牌上存储私钥。以下部分介绍如何使用Oracle Wallet Manager创建两种类型的钱包。

本节介绍以下主题：

  • 创建Oracle Wallet以存储硬件安全模块凭据

创建标准Oracle Wallet

除非您有硬件安全模块（PKCS＃11设备），否则您应该使用标准钱包将凭证存储在文件系统的目录中。

要创建标准Oracle钱包，请执行以下任务：

  1. 启动Oracle Wallet Manager。

    • （UNIX）在命令行中，输入以下命令：

      OWM

    • （Windows）选择开始 ， 程序 ， Oracle-HOME_NAME ， 集成管理工具 ， 电子钱包管理器

  2. 从电子钱包菜单中，选择新建 。

    如果尚未为钱包创建目录，则会提示您创建此目录。默认情况下，wallet目录在/oracle/owm/wallets/ user_name目录中创建。

    将出现“新建钱包”对话框。

  3. 输入以下信息：

    • 密码和确认密码：在这两个字段中输入密码。确保遵循“ 创建Oracle Wallet密码的必需指南 ”中的准则 。此密码可防止未经授权使用您的凭据。

    • 钱包类型：选择标准 。

  4. 单击确定 。

    将显示警报，并通知您已创建新的空钱包。它会提示您决定是否要添加证书申请。

  5. 当系统提示您创建证书请求时，请选择以下选项之一：

    • 是：请参阅“ 添加证书申请 ” 。

    • 否：如果选择“ 否” ，则返回Oracle Wallet Manager主窗口。您刚刚创建的新钱包显示在左侧窗格中。证书的状态为[Empty] ，钱包显示其默认的可信证书。

  6. 从电子钱包菜单中，选择保存在系统默认值以保存新钱包。

    如果您没有权限将钱包保存在系统默认值中，则可以将其保存到其他位置。必须在客户端和服务器的SSL配置中使用此位置。

    窗口底部的消息确认钱包已成功保存。

创建Oracle Wallet以存储硬件安全模块凭据

要创建Oracle钱包以在符合PKCS＃11的硬件安全模块上存储凭据，请执行以下操作：

  1. 启动Oracle Wallet Manager。

    • （UNIX）在命令行中，输入以下命令：

      OWM

    • （Windows）选择开始 ， 程序 ， Oracle-HOME_NAME ， 集成管理工具 ， 电子钱包管理器

  2. 从电子钱包菜单中，选择新建 。

    将出现“新建钱包”对话框。

  3. 输入以下信息：

    • 密码和确认密码：在这两个字段中输入密码。确保遵循“ 创建Oracle Wallet密码的必需指南 ”中的准则 。此密码可防止未经授权使用您的凭据。

    • 钱包类型：选择PKCS11 。

  4. 单击“ 确定”继续。

    出现PKCS11信息窗口。

  5. 在“PKCS11信息”窗口中，输入以下信息：从“ 选择硬件供应商”列表中，选择供应商名称。

    • 硬件供应商：选择供应商名称。SafeNET和nCipher硬件已通过认证，可与Oracle钱包进行互操作。

    • PKCS11库文件名字段，输入存储PKCS11库的目录的路径，或单击“ 浏览”以通过搜索文件系统查找它。

    • 智能卡密码：输入此密码。与钱包密码不同的智能卡密码存储在钱包中。

  6. 单击确定 。

    将显示警报，并通知您已创建新的空钱包。它会提示您决定是否要添加证书申请。

  7. 当系统提示您创建证书请求时，请选择以下选项之一：

    • 是：请参阅“ 添加证书申请 ” 。

    • 否：如果选择“ 否” ，则返回Oracle Wallet Manager主窗口。您刚刚创建的新钱包显示在左侧窗格中。证书的状态为[Empty] ，钱包显示其默认的可信证书。

  8. 从电子钱包菜单中，选择保存在系统默认值以保存新钱包。

关于为Oracle钱包使用自动登录

该钱包的自动登录功能是能够启用基于PKI的服务访问，而无需人工干预来提供必要的密码。启用自动登录会创建钱包的模糊副本，然后自动使用该副本，直到为该钱包禁用自动登录功能。

自动登录钱包受文件系统权限的保护。为钱包启用自动登录后，只有创建它的操作系统用户才能通过Oracle Wallet Manager对其进行管理。

如果要对多个Oracle数据库进行单点登录访问，则必须启用自动登录：默认情况下，此类访问通常是禁用的。有时，混淆的自动登录钱包称为“SSO钱包”，因为它们支持单点登录功能。

将用户证书导入Oracle Wallet

当证书颁发机构授予您证书时，它可能会向您发送一封电子邮件，其中包含您的证书（BASE64）表格或附加为二进制文件。您可以使用以下方法导入用户证书：

  • 从证书颁发机构的文本电子邮件中导入用户证书

  • 从文件导入用户证书

注意：

证书颁发机构可以将您的证书发送到PKCS＃7证书链或作为单独的X.509证书。Oracle Wallet Manager可以导入这两种类型。

PKCS＃7证书链是证书的集合，包括用户证书和所有支持的可信CA和subCA证书。

相反，X.509证书文件包含没有支持证书链的单个证书。

但是，在您可以导入任何此类个人证书之前，签名者的证书必须是钱包中的可信证书。

导入第三方创建的证书和钱包

第三方证书是根据未使用Oracle Wallet Manager生成的证书请求创建的证书。从Oracle意义上讲，这些第三方证书实际上是钱包，因为它们不仅仅包含用户证书;它们还包含该证书的私钥。此外，它们还包括可信证书链，用于验证证书是否由可信赖的实体创建。

Oracle Wallet Manager通过以PKCS＃12格式导入这些钱包，只需一步即可使这些钱包可用，其中包括前面描述的所有三个元素：用户证书，私钥和可信证书。它支持以下PKCS＃12格式证书：

  • Netscape Communicator 4.x及更高版本

  • Microsoft Internet Explorer 5.x及更高版本

Oracle Wallet Manager遵循PKCS＃12标准，因此任何符合PKCS＃12标准的工具导出的证书都可以与Oracle Wallet Manager一起使用。

此类第三方证书无法存储到现有的Oracle钱包中，因为它们缺少私钥和受信任的权限链。因此，每个此类证书都作为独立的PKCS＃12文件导出和检索，即作为其自己的钱包。

导入使用第三方工具创建的用户证书

一旦第三方生成钱包，您需要导入它以使用它，如本节所述。

要导入使用第三方工具创建的证书，请执行以下操作：

    执行导出产品中指示的操作以在导出中包含私钥，并指定新密码以保护导出的证书。还包括所有相关的信任点。（在PKCS＃12下，除了签名者自己的证书之外，浏览器不一定导出可信证书。您可能需要添加其他证书以向同行进行身份验证。您可以使用Oracle Wallet Manager导入可信证书。）

    生成的文件包含证书，私钥和信任点，是允许使用第三方证书的新钱包。

  2. 将钱包放在容易找到的位置，方法是将其复制到正确的系统和目录中。

    要由特定应用程序或服务器（例如Web服务器或LDAP服务器）使用，必须精确定位钱包。每个应用程序都有自己的期望，它将搜索哪个目录以查找所需的钱包。

注意：

只要相关应用程序启动或需要证书，就需要输入密码。要自动进行此类访问，请参阅“ 使用Oracle钱包自动登录以启用无需人工干预的访问 ” 。
