<XSQL：动作>

描述了元素<xsql:action> 。

目的

调用用Java实现的用户定义的操作处理程序，用于执行自定义逻辑并在XSQL页面中包含自定义XML数据。使用此操作调用的Java类必须实现oracle.xml.xsql.XSQLActionHandler接口。

使用<xsql:action>执行内置操作处理程序未处理的任务。自定义操作可以向数据页提供任意XML内容并执行任意处理。

使用说明

XSQL页面处理器以这种方式处理页面中的操作：

  1. 使用默认构造函数构造操作处理程序类的实例。

  2. 通过调用init(Element actionElt, XSQLPageRequest context)方法，使用action元素对象和页面处理器上下文初始化处理程序实例。

  3. 调用允许处理程序处理动作handleAction(Node result) 。

句法

此操作的语法如下所示，其中handler是一个必需的属性，其值是被调用操作的完全限定Java类名， yourpackage是Java包， YourCustomHandler是Java类：

<xsql：action handler =“ yourpackage。YourCustomHandler “/>

某些操作处理程序希望文本内容或元素内容出现在<xsql:action>元素中。在这种情况下，使用如下语法：

<xsql：action handler =“yourpackage。YourCustomHandler“> Some_text </ xsql：action>

您也可以使用以下语法：

<xsql：action handler =“yourpackage。YourCustomHandler“> <some> <other /> <elements /> <here /> </ some> </ xsql：action>

属性

唯一必需的属性是handler ，但您可以为handler提供其他属性。例如，如果你的yourpackage.YourCustomHandler需要名为param1和param2属性，然后使用以下语法：

<xsql：action handler =“yourpackage。YourCustomHandler“param1 =”xxx“param2 =”yyy“>

例子

<XSQL：如果-PARAM>

描述了元素<xsql:if-param> 。

目的

如果指定的条件为true，则允许您包含嵌套在其中的元素和操作。如果条件为true，则所有嵌套的XML内容和操作都包含在页面中。如果条件为false，则不包括任何嵌套的XML内容或操作（因此不会执行任何嵌套操作）。

通过提供所需的name属性来指定要评估的参数值。支持简单的参数名称和数组参数名称。

注意：

如果测试的参数不存在，则测试评估为false。

句法

该操作的语法是这样的，其中some_name是name属性的值， test_condition恰好是表33-5中列出的条件之一 ：

<xsql：if-param name =“ some_name ” test_condition > element_or_action </ xsql：if-param>

将顶级XSLT样式表参数设置为值。该值可以由静态文本和其他参数值的组合提供，也可以由SQL SELECT语句的结果提供。stylesheet参数在处理当前页面期间使用的任何样式表上设置。

使用说明

如果使用SQL语句选项，则从结果集中提取单行，并为参数分配第一列的值。此用法要求通过在其出现的XSQL页面的文档元素上提供connection="connname"属性来提供数据库连接。

要根据单个SQL语句的结果设置多个样式表参数值，请不要使用name属性。您可以使用names属性并提供一个或多个样式表参数名称的空格或逗号分隔列表。

句法

此操作的语法如下，其中paramname是参数的名称，其中value是值：

<xsql：set-stylesheet-param name =“ paramname ”value =“ value ”/>

或者，您可以使用此语法，其中SQL_statement是SQL SELECT语句， paramname是参数的名称：

<xsql：set-stylesheet-param name =“paramname”> SQL_statement </ xsql：set-stylesheet-param>

name或names属性是必需的。value属性和包含的SQL语句是互斥的。

描述了元素<xsql:update-request> 。

目的

接受从XML文档或HTML表单发布的数据，并使用XML SQL实用程序（XSU）从目标表或视图以规范形式更新XML文档的内容。

通过将XSU与XSLT相结合，您可以将XML转换为给定表所期望的规范格式。之后，您可以使用XSU更新生成的规范XML。对于指定的数据库表，规范XML表单由针对表的SELECT *查询的一行XML输出给出。

句法

此操作的语法是：

<xsql：update-request table =“ table_name ”/>
