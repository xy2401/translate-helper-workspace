9.7.1查看所有性能类的性能度量标准

您可以在“性能类服务质量”页面上查看系统的性能指标。

配置Oracle数据库QoS管理后，Oracle Database QoS Management需要很短的时间来收集性能数据并评估系统性能。经过这段时间后，您可以查看系统的性能指标。要查看当前性能指标，请执行以下步骤：

  1. 以集群管理员身份登录Oracle Enterprise Manager Cloud Control。转到群集目标页面。

  2. 从群集目标菜单中，选择“ 管理” ，然后选择“服务质量管理” ，然后选择“ 查看性能类服务质量” 。

  3. Performance Class Quality of Service页面显示三个图表，用于衡量正在监控的每个Performance Class的当前性能：

9.7.2查看各个性能类的性能指标

配置Oracle数据库服务质量管理并且过了很短的时间后，您可以查看特定性能类的性能指标。

要查看性能类的当前性能指标，请执行以下步骤：

  1. 以集群管理员身份登录Oracle Enterprise Manager Cloud Control。
  2. 选择为QoS Management配置的群集目标页面。
  3. 从群集目标菜单中，选择“ 管理” ，然后选择“服务质量管理” ，然后选择“ 查看性能类服务质量” 。
  4. 在任何图形的右侧，在图例框中，单击与要查看其性能指标的性能类对应的链接。
    查看单个性能类的性能指标时，您可以看到另外两个图：

    • 资源使用时间

    • 资源等待时间

9.7.3配置服务质量管理事件的警报

要求对服务质量管理仪表板进行持续的手动监控是不方便或有效的。相反，您可以使用企业管理器云控制通知系统报告持续用户指定时间的负面性能满意度指标（PSM）。

通过设置警报阈值和通知，可以根据每个性能等级的指定持续时间来警告警告和关键级别。警报是针对提供受监视服务的数据库配置的
  1. 启动企业管理器云控制。
  2. 导航到要为其配置警报的数据库。
  3. 从Cluster Database菜单中选择Monitoring ，然后选择| 指标和集合设置 。

    说明cc13_metrics_and_collections_db.png的说明
  4. 在“ 度量标准”选项卡下，编辑“ QoS管理 - 性能满意度量标准 ”类别中的度量标准 。

    说明modify_qos_mgmt_performance_metric.png的说明

    例如，如果服务质量管理使用数据库服务sales_svc_pc ，则在负PSM持续时间（秒）下，对于该服务，您可以在持续时间超过120秒时配置警告警报，如果持续时间更长，则可以配置严重警报超过180秒。

如果检测到违规，配置的警报将显示在Enterprise Manager的“数据库主页”页面上。
