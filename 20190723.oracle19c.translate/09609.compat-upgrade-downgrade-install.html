核心转储ORA-00600 [8153]数据库升级后遇到错误

本主题描述数据库升级后可能发生的ORA-00600错误。

数据库升级到Oracle Database 12c第2版（12.2）或Oracle Database 19c后，会遇到核心转储ORA-00600 [8153]错误，然后降级回原始版本（参考错误20898997）。

在将数据库升级到Oracle Database 12c第2版（12.2）或Oracle Database 19c之前，始终应用此错误的修补程序。

从Oracle Database 11g第2版（11.2.0.4）升级到19c升级到Oracle数据库会导致升级期间出现错误

本主题描述从Oracle Database 11g第2版（11.2.0.4）升级到Oracle Database 19c时可能发生的ORA-08102错误。

在某些情况下，当更改表WRH$_SEG_STAT以添加一些列时，您可能会遇到ORA-08102错误（参考错误28668676）。例如：
ALTER TABLE WRH $ _SEG_STAT ADD（im_membytes NUMBER DEFAULT 0）; ALTER TABLE WRH $ _SEG_STAT ADD（im_membytes NUMBER DEFAULT 0）*第1行的错误：ORA-08102：未找到索引键，obj＃16200，文件2，块4765（2）

从bug中下载c18.sql和c1202000.sql并重新启动升级过程。

将Oracle数据库12.1非CDB插入Oracle数据库12.2本地撤消CDB

本主题描述从Oracle Database版本12.1发布到版本12.2时未插入撤消表空间。

尝试将Oracle数据库12c第1版（12.1）非CDB插入Oracle数据库12c版本2（12.2）或更高版本的CDB并启用本地撤消时，未插入撤消表空间。

解决方法：

没有。
