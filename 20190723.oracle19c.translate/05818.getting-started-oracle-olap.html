2 Oracle OLAP入门

本章介绍了使用Oracle OLAP应采取的初步步骤。它假定您已安装Oracle Database 12 c企业版。OLAP选项作为Oracle数据库的基本安装的一部分自动安装。

注意：

要立即开始查询维对象，请安装Global分析工作区，如“ 安装示例模式 ”中所述 。然后按照查询维对象中的说明进行操作。

您应该创建适合维度对象使用的撤消，永久和临时表空间。按照“ 存储管理 ”中的建议操作。

2.3向DBA和应用程序开发人员授予权限

必须在Oracle数据库中创建或管理维对象的任何人都需要必要的权限。这些权限与查询维度对象中存储的数据所需的权限不同。安全系统在讨论安全 。

DBA和应用程序开发人员需要以下角色和特权。

要在用户自己的架构中创建维对象：

  • OLAP_USER角色

  • CREATE SESSION权限

要在不同的模式中创建维对象：

  • OLAP_DBA角色

  • CREATE SESSION权限

管理数据安全性：

  • OLAP_XS_ADMIN角色

要在用户自己的架构中创建多维数据集物化视图：

  • CREATE MATERIALIZED VIEW权限

  • CREATE DIMENSION权限

  • ADVISOR特权

要在不同的模式中创建多维数据集物化视图：

  • CREATE ANY MATERIALIZED VIEW权限

  • CREATE ANY DIMENSION权限

  • ADVISOR特权

用户还需要在存储维对象的表空间上具有无限制的配额。应该专门为OLAP使用定义表空间，如管理Oracle OLAP中所述 。

如果源表位于不同的模式中，则维对象的所有者必须对这些表具有READ或SELECT对象特权。

2.4 Analytic Workspace Manager入门

在本节中，您将学习如何安装Analytic Workspace Manager软件并建立与Oracle数据库的连接。

2.4.1安装Analytic Workspace Manager

Analytic Workspace Manager分布在Oracle数据库客户端安装盘上。

如果要在与数据库相同的系统上进行安装，请选择“ 自定义”安装并将其安装到与数据库相同的Oracle主目录中。从组件列表中选择OLAP Analytic Workspace Manager和Worksheet 。

如果要在远程系统上进行安装，请选择“ 管理员”或“ 自定义”安装。管理员选择会自动在客户端上安装Analytic Workspace Manager。

也可以看看：

客户端平台的安装指南。

2.4.2打开Analytic Workspace Manager

使用适用于您的平台的相应过程。

在Windows上，打开Analytic Workspace Manager：

  • 从“开始”菜单中，选择Oracle - Oracle_home ，然后选择Integrated Management Tools ，然后选择OLAP Analytic Workspace Manager和Worksheet 。

在Linux上，打开Analytic Workspace Manager：

  • 从shell命令行输入以下命令：

    $ ORACLE_HOME / OLAP / AWM / awm.sh

如果Analytic Workspace Manager无法访问Internet，则属性查看器会显示指向多个有用站点的链接。它还显示异常，因为Analytic Workspace Manager无法显示OLAP主页。要连接到Internet，通常需要标识代理服务器。

  3. 输入代理服务器的端口号（如果它不是默认端口80）。

  4. 单击“ 确定”以保存这些设置。下次启动Analytic Workspace Manager时将显示OLAP主页。

2.4.3定义数据库连接

您可以定义与用于OLAP的每个数据库的连接。定义连接后，导航树中会列出数据库实例，供您随时访问。

要定义数据库连接：

  1. 右键单击导航树中的顶级Databases文件夹，然后从快捷菜单中选择New Database Connection 。

  2. 完成“新建数据库连接”对话框。

2.4.4打开数据库连接

要连接到数据库：

  1. 单击导航树中数据库连接旁边的加号图标（+）。

  2. 在“连接到数据库”对话框中提供数据库用户名和密码。

2.4.5显示分析工作区附件模式

您可以在打开分析工作区时指定分析工作区附件模式。模式如下：

  • 只读

    在此模式下，用户可以查看分析工作区对象和数据，但无法创建或更改对象。用户可以通过复制或将其保存为模板来导出对象。任意数量的用户都可以在只读模式下打开分析工作区。

  • 读写

    在此模式下，用户可以查看分析工作区对象和数据，以及创建或更改对象。用户可以导出或导入对象。只有一个用户可以在读写模式下打开分析工作区，但任何数量的其他用户都可以在只读模式下打开它。这是默认模式。

  • 读写独家

    在此模式下，用户具有与“读写”模式相同的访问权限，但没有其他人可以打开分析工作区。如果其他用户打开了分析工作区，则此模式不可用。

要指定显示附件模式：

  1. 从“工具”菜单中，选择“ 配置” 。

    将打开“配置”对话框。

  2. 选择“ 显示分析工作区附件选项” 。单击确定 。

2.4.6安装插件

插件扩展了Analytic Workspace Manager的功能。插件作为JAR文件分发。任何Java开发人员都可以创建插件。开发人员应提供有关插件的功能和使用方法的信息。

如果您有一个或多个插件，则必须将其位置标识到Analytic Workspace Manager。

要使用插件：

  1. 创建用于存储插件的本地目录。

  2. 将JAR文件复制到该目录。

  3. 打开Analytic Workspace Manager。

  4. 从工具菜单中选择配置 。

    将打开“配置”对话框。

  5. 选择“ 启用插件”并标识插件目录。单击确定 。

  6. 关闭并重新打开Analytic Workspace Manager。

    导航器中提供了插件提供的功能。

要查看当前安装的插件列表：

  • 在“帮助”菜单上，单击“ 关于” ，然后单击“ 插件” 。

可以从Oracle技术网（OTN）下载某些Analytic Workspace Manager插件。

从OTN下载插件：

  • 在Web浏览器中，转到Oracle OLAP下载页面

    http://www.oracle.com/technetwork/database/options/olap/olap-downloads-098860.html

2.5从Oracle OLAP 10 g升级元数据

通过将对象另存为XML模板并将XML导入到其他模式，可以将Oracle OLAP 10 g分析工作区升级到OLAP 11 g或12 c 。升级过程可以保持原始分析工作区的可访问性和未更改状态。

先决条件：

  • OLAP 10 g分析工作区可以使用OLAP标准表单元数据。

  • 原始关系源数据必须可用于加载到新的分析工作区。如果数据位于不同的模式中或表名不同，则必须在升级后将维对象重新映射到新的关系源。

  • 您可以在与OLAP 10 g分析工作区相同的架构中创建OLAP 12 c分析工作区。但是，如果选择在其他架构中创建OLAP 12 c分析工作区，则必须为新用户授予适当的权限，如“ 向DBA和应用程序开发人员授予权限 ”中所述 。

要升级OLAP 11 g分析工作区，请执行以下操作：

  1. 打开Oracle Database 12 c第1版的分析工作区管理器。

  2. 如有必要，使用分析工作区创建与数据库实例的新数据库连接。请参阅“ 定义数据库连接 ” 。

  3. 打开数据库连接。在“连接到数据库”对话框中，为“多维数据集类型”选择“ OLAP 11g / 12c ”。请参阅“ 打开数据库连接 ” 。

  4. 展开导航树，直到出现分析工作区的名称。

  5. 右键单击分析工作区，然后选择“ 为11g分析工作区创建12c升级模板” 。将XML模板保存到文件中。

    如果任何子对象（例如级别和层次结构）具有相同的名称，则会出现“12 c分析工作区的创建12c升级模板”对话框。

    重复的对象名称会自动更改以进行升级。您现在无法编辑名称，但可以稍后更改。

  6. 单击“ 关闭”以关闭对话框。

  7. 右键单击树中的连接，然后选择“ 断开数据库” 。

  8. 再次右键单击该连接，然后选择“ 连接数据库” 。

  9. 在“连接到数据库”对话框中，使用新用户名登录，并为多维数据集类型选择OLAP 11g / 12c 。

  10. 展开树，右键单击新架构下的“ 分析工作区 ”，然后选择“ 从模板创建分析工作区” 。

  11. 打开先前创建的升级模板。

    如果任何对象（例如多维数据集，维度或分析工作空间）复制了架构中已存在的对象名称，则会出现“分析工作区模板导入”对话框中的正确重复名称。

  12. 输入新名称以解决任何冲突，然后单击“ 确定” 。

  13. 在加载数据之前，您可能希望浏览维对象并对对象名，多维数据集分区或聚合策略进行任何更改。

  14. 将数据加载到新的分析工作区中，如“将数据加载到多维数据集 ”中所述 。选择所有对象进行维护。
