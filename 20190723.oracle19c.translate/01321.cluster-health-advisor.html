1.2.4 Oracle Cluster Health Advisor简介

Oracle Cluster Health Advisor持续监视群集节点和Oracle RAC数据库的性能和可用性问题前兆，以便在问题变得严重之前提供问题的早期警告。

Oracle Cluster Health Advisor已集成到Oracle企业管理器云控制（EMCC）事件管理器中。

Oracle Cluster Health Advisor执行以下操作：

  • 检测节点和数据库性能问题

  • 提供预警警报和纠正措施

  • 支持现场校准以提高灵敏度

在Oracle Database 12c第2版（12.2.0.1）中，Oracle Cluster Health Advisor支持监视Oracle Real Application Clusters（Oracle RAC）的两个关键子系统：数据库实例和主机系统。Oracle Cluster Health Advisor确定并跟踪受监视系统的运行状况。它定期从受监控系统中采集各种关键测量数据。

已经对超过一百个数据库和集群节点问题进行了建模，并且已经确定了指示这些问题的发展或存在的特定操作系统和Oracle数据库度量标准。该信息用于构建基于目标系统的正常操作时段的训练的校准模型。

Oracle Cluster Health Advisor每分钟多次运行分析。Oracle Cluster Health Advisor根据默认模型估计观察输入的预期值。然后，Oracle Cluster Health Advisor根据观察值和预期值之间的差异对每个输入执行异常检测。如果与特定问题相关的足够输入异常，则Oracle Cluster Health Advisor会发出警告并生成即时针对性诊断和纠正措施。

Oracle Cluster Health Advisor模型是保守的，可以防止出现错误警告通知。但是，对于关键生产系统，默认配置可能不够灵敏。因此，Oracle Cluster Health Advisor提供现场模型校准功能，以使用实际生产工作负载数据构成其默认设置的基础，并提高节点和数据库模型的准确性和灵敏度。

Oracle Cluster Health Advisor在Grid Infrastructure Management Repository（GIMR）中存储分析结果，以及诊断信息，纠正措施和度量标准，以便以后进行分类。Oracle Cluster Health Advisor还使用Oracle Clusterware事件通知协议向Enterprise Manager Cloud Control发送警告消息。

您还可以使用Oracle Cluster Health Advisor诊断和分类过去的问题。您可以通过Oracle Enterprise Manager云控制（EMCC）事件管理器或命令行界面CHACTL指定过去的日期。通过在Grid Infrastructure Management Repository（GIMR）中配置Oracle Cluster Health Advisor表空间的保留设置，管理Oracle Cluster Health Advisor查看过去问题的功能。默认保留期为72小时。
