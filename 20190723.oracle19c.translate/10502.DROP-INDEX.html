DROP INDEX

目的

使用DROP INDEX语句从数据库中删除索引或域索引。

删除全局分区索引，范围分区索引或散列分区索引时，也会删除所有索引分区。如果删除复合分区索引，则也会删除所有索引分区和子分区。

此外，当您删除域索引时：

  • Oracle数据库调用适当的例程。

  • 如果任何统计信息与域索引相关联，则Oracle数据库会将统计信息类型与FORCE子句解除关联，并删除使用统计信息类型收集的用户定义统计信息。

    也可以看看：

    • Oracle Database Data Cartridge开发人员指南 ，了解有关例程的信息

    • CREATE INDEX和ALTER INDEX以获取有关创建和修改索引的信息

    • 有关域索引的更多信息，请参阅CREATE INDEX的domain_index_clause

    • 关于统计类型关联的更多信息，关联统计和DISASSOCIATE STATISTICS

先决条件

索引必须位于您自己的架构中，或者您必须具有DROP ANY INDEX系统特权。

句法

drop_index :: =


说明drop_index.eps的描述

语义

模式

指定包含索引的架构。如果省略schema ，则Oracle数据库假定索引位于您自己的模式中。

指数

指定要删除的索引的名称。删除索引时，分配给索引的所有数据块都将返回到包含索引的表空间。

线上

指定ONLINE以指示在删除索引时将允许对表或分区执行DML操作。

力

FORCE仅适用于域索引。即使indextype例程调用返回错误或索引标记为IN PROGRESS此子句也会删除域索引。如果没有FORCE ，如果其索引类型例程调用返回错误或索引标记为IN PROGRESS ，则不能删除域索引。

{DEFERRED | IMMEDIATE}无效

此子句允许您控制数据库何时在删除索引时使相关游标无效。它具有与ALTER INDEX语句相同的语义，并添加了以下内容：当您删除带有DEFERRED INVALIDATION的索引时，Oracle数据库将立即使其计划中引用已删除索引的任何DML语句或查询无效。

见{DEFERRED | IMMEDIATE}有关此子句的完整语义的ALTER INDEX文档中的INVALIDATION 。

删除索引的限制

以下限制适用于删除索引：

  • 如果索引或其任何索引分区标记为IN_PROGRESS则无法删除域索引。

  • 删除域索引，集群索引或队列表上的索引时，无法指定ONLINE子句。

例子

删除索引：示例

此语句删除名为ord_customer_ix_demo的索引，该索引是在“ 压缩索引：示例 ”中创建的 ：

DROP INDEX ord_customer_ix_demo;
