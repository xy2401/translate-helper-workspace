了解操作系统权限组

查看此信息以获取Oracle数据库或Oracle自动存储管理（Oracle ASM）管理所需的系统特权。

作为管理员，您经常执行特殊操作，例如关闭或启动数据库或配置存储。由于只有负责这些管理决策的管理员必须执行这些操作，因此Oracle数据库或Oracle自动存储管理（Oracle ASM）管理的系统特权需要安全的身份验证方案。

特殊操作系统组的成员身份使管理员可以通过操作系统而不是用户名和密码对Oracle数据库或Oracle ASM进行身份验证。这称为操作系统身份验证 。群集中的每个Oracle数据库都可以拥有自己的操作系统权限组，因此可以为群集上的每个Oracle数据库分离操作系统身份验证。由于群集上只能安装一个Oracle Grid Infrastructure，因此Oracle ASM只能有一组操作系统权限组。

在安装Oracle Grid Infrastructure和Oracle Database期间，您需要提供操作系统组的组名。指定这些操作系统组的逻辑角色是为Oracle数据库和Oracle ASM的管理系统特权授予操作系统组认证。

在Oracle RAC集群中，系统特权组的组ID号（GID）在每个集群成员节点上必须相同。可以将一个操作系统组指定为其成员被授予Oracle Database和Oracle ASM的所有系统特权的逻辑组，包括安装所有者的OINSTALL系统特权。您还可以将逻辑系统特权委派给两个或更多实际操作系统组。Oracle建议您为每个逻辑系统特权指定单独的操作系统组。这使您可以向数据库管理员授予一个或多个管理员系统特权子集。然后，这些数据库管理员可以执行标准数据库管理任务，而无需SYSDBA系统特权。

OINSTALL

为oinstall

为安装所有者安装系统特权，其中包括写入每个服务器的中央oraInventory目录的权限，以及授予Oracle二进制安装所有者用户的其他特权。

OSDBA

DBA

Oracle数据库的SYSDBA系统特权，包括数据库的所有系统特权。

OSOPER

OPER

SYSOPER启动和关闭Oracle数据库的系统特权。

OSBACKUPDBA

backupdba

SYSBACKUP Oracle数据库的备份和恢复系统特权。

OSDGDBA

dgdba

SYSDG系统特权来管理和监视Oracle Data Guard。

OSKMDBA

kmdba

SYSKM系统特权，用于Oracle Wallet Manager等应用程序的加密密钥管理。

OSASM

asmadmin

集群上Oracle ASM的SYSASM系统特权，包括Oracle ASM存储的所有系统特权。

OSMER for ASM

asmoper

SYSOPER在集群上启动和关闭Oracle ASM的系统特权。

OSDBA for ASM

asmdba

SYSDBA for ASM系统特权，用于获取Oracle ASM管理的文件的读写访问权限。所有Oracle数据库软件所有者必须是该组的成员。

OSRACDBA

racdba

SYSRAC权限在Oracle RAC集群上执行Oracle数据库的日常管理。所有Oracle数据库软件所有者必须是该组的成员。
