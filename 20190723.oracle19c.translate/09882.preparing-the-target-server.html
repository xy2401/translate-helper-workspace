2准备目标服务器

本章介绍如何使用Database Configuration Assistant（DBCA）在目标服务器中创建目标数据库。

在源服务器和目标服务器上运行PING命令以检查两个服务器是否都可访问。

  • 创建目标数据库
    在目标系统上创建一个新数据库。

创建目标数据库

在目标系统上创建一个新数据库。

新的目标数据库最初只包含SYSTEM ， SYSAUX ， UNDO ，临时表空间和用户表空间。您可以使用DBCA创建目标数据库。

创建目标数据库时，请考虑以下事项：

  • 虽然原始源数据库中的所有用户表空间都被传输并插入到新的目标数据库中，但最初目标数据库必须包含将要传输的用户表空间的占位符表空间。最初在目标数据库中创建的用户表空间的大小可能很小;目标表空间不必与源数据库中的大小匹配。在从源系统传输表空间之前，将从目标数据库中删除占位符表空间。

  • SYSTEM ， SYSAUX ， UNDO和临时表空间的大小必须与源数据库上的表空间相同或更大。

  • 新目标数据库中每个日志文件组的日志文件大小和成员数应与源数据库相同或大于源数据库。

  • 源数据库和目标数据库必须使用相同的字符集和相同的国家字符集。通过发出以下查询来检查源数据库字符集：
    SQL> select * from database_properties其中property_name类似'％CHARACTERSET';

  • 源数据库中使用的数据库选项和组件应安装在目标数据库上。

    • 查询V$OPTION视图以获取当前安装的数据库选项。

    • 查询DBA_REGISTRY以获取当前安装的数据库组件。
