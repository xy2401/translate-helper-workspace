8.178 V $ RMAN_OUTPUT

V$RMAN_OUTPUT显示RMAN报告的消息。

这是内存中的视图，不会记录在控制文件中。视图可以容纳32768行。

柱 数据类型 描述

SID

数

正在运行此RMAN操作的会话的会话ID

RECID

数

记录相应V$RMAN_STATUS行的ID

邮票

数

相应V$RMAN_STATUS行的时间戳

SESSION_RECID

数

会话的记录ID（对应的V$RMAN_STATUS行， ROW_LEVEL = 0 ）

SESSION_STAMP

数

会话的时间戳（对应的V$RMAN_STATUS行， ROW_LEVEL = 0 ）

OUTPUT

VARCHAR2（130）

RMAN报告的输出文本

RMAN_STATUS_RECID

数

拥有V$RMAN_STATUS记录ID

RMAN_STATUS_STAMP

数

拥有V$RMAN_STATUS记录戳记

SESSION_KEY

数

会话标识符

GUID Foot 1

数

执行RMAN操作的可插拔数据库（PDB）的guid。V$RMAN_OUTPUT捕获rman输出。

如果PDB sysdba执行了备份，则PDB的guid将显示在V$RMAN_OUTPUT 。如果root进行了备份，那么root的guid将显示在V$RMAN_OUTPUT 。根

root用户可以看到所有V$RMAN_OUTPUT行（即所有PDB拥有的行），但PDB用户只能看到PDB的V$RMAN_OUTPUT行（即，没有来自root或其他PDB的行）。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID

脚注1此列从Oracle Database 19c版本19.1开始提供。
