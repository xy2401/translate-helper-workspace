17删除Oracle Database Gateway

以下主题描述如何从Oracle主目录中删除Oracle Database Gateway。

  • 关于卸载工具

  • 删除Oracle软件

关于卸载工具

安装前，安装介质中提供了卸载工具（ deinstall ），安装后可在Oracle主目录中使用。它位于路径$ORACLE_HOME/deinstall 。

deinstall命令停止Oracle软件，并删除操作系统上的Oracle软件和配置文件。

该命令使用以下语法，其中变量内容以斜体表示：

deinstall -home home的完整路径 [-silent] [-checkonly] [-local] [-paramfile 输入参数属性文件的完整路径 ] [-params name1 = value name2 = value 。 。 。] [-o 保存文件的目录的完整路径 ] [-help | -H]

选项是：

  • -无声

    使用此标志以静默或响应文件模式运行命令。如果使用-silent标志，则必须使用-paramfile标志，并提供一个参数文件，其中包含要卸载或取消配置的Oracle主目录的配置值。

    您可以通过使用-checkonly标志运行deinstall来生成要使用或修改的参数文件。然后， deinstall命令会发现要从中卸载和取消配置的Oracle主目录中的信息。它会生成属性文件，然后您可以将其与-silent选项一起使用。

    您还可以修改位于响应文件夹中的模板文件deinstall.rsp.tmpl 。

  • -checkonly

    使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行该命令不会删除Oracle配置。-checkonly标志生成一个可以与deinstall命令一起使用的参数文件。

  • -本地

    在多节点环境中使用此标志可在群集中取消配置Oracle软件。

    使用此标志运行deconfig时，它会在本地节点（运行deconfig的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

  • -paramfile complete path of input parameter property file

    使用此标志可以使用默认位置以外的参数文件运行deconfig。使用此标志时，请提供参数文件所在的完整路径。

    参数文件的默认位置取决于deconfig的位置：

    • 从安装介质或阶段位置： $ORACLE_HOME/inventory/response

    • 来自OTN的解压缩归档文件： /ziplocation/response

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -params [name1= value name 2= value name3= value . . .]

    将此标志与参数文件一起使用，以覆盖要在已创建的参数文件中更改的一个或多个值。

  • -o complete path of directory for saving response files

    使用此标志提供除保存属性文件（ deinstall.rsp.tmpl ）的默认位置以外的路径。

    参数文件的默认位置取决于deconfig的位置：

    • 安装前从安装介质或阶段位置： $ORACLE_HOME/

    • 来自OTN的解压缩归档文件： /ziplocation/response/

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -help | -H

    使用help选项（ -help或-h ）获取有关命令选项标志的其他信息。

删除Oracle软件

完成以下过程以删除Oracle软件：

  1. 以安装所有者身份登录。
  2. 运行deinstall命令，根据提示提供有关服务器的信息。
