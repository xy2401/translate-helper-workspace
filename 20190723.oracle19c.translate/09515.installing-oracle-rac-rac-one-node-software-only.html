安装Oracle RAC和Oracle RAC One节点数据库软件

安装Oracle RAC或Oracle RAC One Node软件。

Oracle Real Application Clusters数据库安装分为两步。此过程描述了第一步 - 安装Oracle RAC软件。
如果您有现有的Oracle安装，请记下版本号，修补程序和其他配置信息，并查看现有安装的升级过程。在继续安装之前，请查看“Oracle数据库升级指南”。
确保您具有su或sudo凭据，因为您将需要提供root凭据以自动运行根脚本或在安装期间以root用户身份运行脚本。
  1. 确保您可以使用SSH访问其他节点。安装程序要求允许运行Oracle RAC安装的用户帐户设置无密码SSH。安装程序可以自动为您设置，或者系统管理员可以在安装开始之前为您进行设置。
  2. 打开要从中运行安装程序的终端，然后以要拥有Oracle数据库安装的用户帐户（例如， oracle ）身份登录。

    如果您无法关闭stty命令，或者有其他限制阻止安装程序中的自动SSH配置，则必须确保在继续开始安装之前配置并启用了SSH。

  3. 下载Oracle数据库安装映像文件（ db_home.zip ）并将文件db_home.zip压缩到新的Oracle主目录中。例如：
    $ mkdir -p /u01/app/oracle/product/19.0.0/dbhome_1 $ chgrp oinstall /u01/app/oracle/product/19.0.0/dbhome_1 $ cd /u01/app/oracle/product/19.0.0/ dbhome_1 $ unzip -q /tmp/db_home.zip

    注意：

    Oracle建议您创建的Oracle主目录路径符合Oracle Optimal Flexible Architecture建议。此外，仅在您创建的此Oracle主目录中解压缩安装映像文件。
  4. 从Oracle主目录中，启动Oracle数据库软件安装：
    $ cd /u01/app/oracle/product/19.0.0/dbhome_1 $ ./runInstaller

    注意：

    仅从Oracle主目录运行runInstaller命令。不要运行驻留在$ORACLE_HOME/oui/bin/或任何其他位置的runInstaller命令来安装Oracle Real Application Clusters或Oracle Grid Infrastructure。
  5. 在“选择配置选项”屏幕中，选择“ 仅设置软件”选项。单击下一步 。
  6. 在“选择数据库安装选项”屏幕中，选择“ Oracle Real Application Clusters数据库安装”选项。单击下一步 。
  7. 在“节点选择”屏幕中，选择要安装Oracle RAC或Oracle RAC One Node软件的所有节点。单击下一步 。
  8. 在“选择数据库版本”屏幕中，选择“ 企业版”选项。单击下一步 。
  9. 在“指定安装位置”屏幕中，提供Oracle RAC软件安装的Oracle基础位置。
    Oracle基本目录必须与Grid主目录不同。单击下一步 。
  10. 在“特权操作系统组”屏幕中，接受Oracle数据库管理的默认操作系统组名称，然后单击“ 下一步” 。
  11. 选择“ 自动运行配置脚本 ”选项。输入root用户或sudo帐户的凭据，然后单击“ 下一步” 。
    或者，您可以在安装程序提示时在安装过程结束时以root用户身份手动运行脚本。
  12. “执行先决条件检查”屏幕显示先决条件检查的结果。如果任何检查的状态为“失败”且不可修复，则必须手动更正这些问题。解决问题后，可以单击“再次检查”按钮让安装程序重新检查要求并更新状态。根据需要重复，直到所有检查的状态为Succeeded。单击下一步 。
  13. 查看“摘要”屏幕的内容，然后单击“ 安装” 。
    安装程序显示进度指示器，使您可以监视安装过程。
  14. 如果未配置根脚本的自动化，则需要以root用户身份运行root.sh脚本，如“执行配置脚本”窗口中所指定。直到你已经运行了不要单击OK root.sh所有节点上脚本遵医嘱。
    您可以在群集中的所有节点上同时运行root.sh脚本，以进行Oracle RAC安装或升级。
安装Oracle RAC软件后，从ORACLE_HOME/bin/dbca目录运行Database Configuration Assistant（DBCA）以创建和配置Oracle RAC数据库。
