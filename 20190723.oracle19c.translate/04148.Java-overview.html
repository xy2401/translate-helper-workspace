1.1 Java概述

Java已经成为面向对象编程语言的首选。Java的一些重要概念包括：

  • Java虚拟机（JVM），它提供了平台独立性的基础

  • 自动存储管理技术，例如垃圾收集

  • 语言语法类似于C语言的语法

结果是一种面向对象且对应用程序编程有效的语言。

1.1.1.1课程

所有面向对象的编程语言都支持类的概念。与表定义一样，类为共享共同特征的对象提供模板。每个类都可以定义以下内容：

  • Fie lds

    字段是存在于特定类的每个对象或实例中的变量，或者是全局且对所有实例通用的变量。实例字段类似于关系表行的列。该类定义了每个字段的字段和类型。

    您可以将Java中的字段声明为静态。声明为static的类的字段是全局的，并且对于该类的所有实例都是通用的。对于Java运行时的给定实例化中的静态字段，在任何给定时间只有一个值。未声明为static的字段将在类的每个实例中创建为不同的值。

    public ， private ， protected和default访问修饰符定义应用程序中字段的范围。该 Java语言规范（JLS）定义了所有字段的数据可见性规则。这些规则定义了在什么情况下您可以访问这些字段中的数据。

    在图1-1所示的示例中，员工标识符被定义为private ，表示其他对象无法直接访问此字段。在该示例中，对象可以通过调用getId()方法来访问id字段。

  • 方法

    方法是与类相关的过程。像字段一样，方法可以声明为static ，在这种情况下可以全局调用。如果方法未声明为静态，则意味着该方法是实例方法，并且只能作为对象的操作进行调用，其中对象是类的实例。

    与字段类似，方法可以声明为public ， private ， protected或默认访问。此声明定义了可以调用方法的范围。

Java对象是类的实例，类似于关系表行。对象是数据值的集合，其各个元素由类的非静态字段定义描述。

1.1.1.3继承

继承是面向对象编程语言的一个重要特性。它使类能够包含其他类的属性。继承属性的类称为子类或子类，从中继承属性的类称为父类或超类。此功能还有助于重用已定义的代码。

在图1-1所示的示例中，您可以创建一个继承Employee类属性的FullTimeEmployee类。继承的属性取决于为每个字段和超类方法声明的访问修饰符。

1.1.1.4接口

Java仅支持单继承，也就是说，每个类只能继承一个类的字段和方法。如果需要从多个源继承属性，那么Java提供了接口的概念，这是一种多重继承的形式。接口类与类相似。但是，它们只定义方法的签名而不是它们的实现。接口中声明的方法在类中实现。当类实现多个接口时，会发生多重继承。

1.1.1.5封装

封装描述了对象隐藏其数据和方法与世界其他地方的能力，并且是面向对象编程的基本原则之一。在Java中，类封装了保存对象状态的字段，以及定义对象操作的方法。封装使您可以编写可重用的程序。它还允许您仅限制对已声明为public的对象的那些功能的访问。所有其他字段和方法都是private ，可用于内部对象处理。

在图1-1所示的示例中， id字段是private ，对它的访问仅限于定义它的对象。其他对象可以使用getId()方法访问此字段。使用封装，您可以通过将getId()方法声明为private或不定义getId()方法来拒绝访问id字段。

1.1.1.6多态性

多态性是不同对象对同一消息的不同响应的能力。在面向对象的编程语言中，您可以使用相同的名称定义一个或多个方法。这些方法可以执行不同的操作并返回不同的值。

在图1-1的示例中，假设不同类型的员工必须能够响应他们迄今为止的薪酬。对不同类型的员工进行不同的薪酬计算：

  • 全职员工有资格获得奖金。

  • 非豁免员工获得加班费。

在过程语言中，您编写了一个switch语句，其中定义了不同的可能情况，如下所示：

switch：（employee.type）{case：Employee return employee.salaryToDate; case：FullTimeEmployee返回employee.salaryToDate + employee.bonusToDate ...}

如果添加新类型的员工，则必须更新switch语句。此外，如果修改数据结构，则必须修改使用它的所有switch语句。在面向对象的语言（如Java compensationToDate() ，如果Employee类的每个子类都包含超出Employee类中已定义的信息，则可以为Employee类的每个子类实现一个方法compensationToDate() 。例如，您可以为非豁免员工实施compensationToDate()方法，如下所示：

public float compensationToDate（）{return（super.compensationToDate（）+ this.overtimeToDate（））; }

对于全职员工， compensationToDate()方法可以实现如下：

通过使用方法名称，您可以调用不同类的方法并获取所需的结果，而无需指定员工的类型。您不必编写特定方法来处理全职员工和兼职员工。

此外，您可以创建一个Contractor类，该类不从Employee继承属性并在其中实现compensationToDate()方法。计算到目前为止的工资总额的程序将迭代工资单上的所有人员，无论他们是全职员工还是兼职员工或承包商，并将每个人调用compensationToDate()方法返回的值相加。您可以安全地对各个compensationToDate()方法或类进行更改，并且知道方法的调用方将正常工作。

1.1.2 Java语言的主要特性

Java语言提供了某些关键功能，使其成为开发服务器应用程序的理想选择。这些功能包括：

  • 简单

    Java比用于创建服务器应用程序的大多数其他语言更简单，因为它一致地强制执行对象模型。大型标准类库为所有平台上的Java开发人员提供了强大的工具。

  • 可移植性

    Java可跨平台移植。可以在Java中编写与平台相关的代码，编写跨系统无缝移动的程序也很简单。

    “ Java虚拟机 ”

  • 自动存储管理

    程序运行时，JVM会自动执行所有内存分配和释放。Java程序员无法为新对象显式分配内存，也无法为不再引用的对象释放内存。相反，它们依赖于JVM来执行这些操作。释放内存的过程称为垃圾收集。

  • 强打字

    在使用字段之前，必须声明字段的类型。Java中的强类型使得为Java和PL / SQL应用程序之间的中介语调用提供合理且安全的解决方案成为可能，并在同一应用程序中集成Java和SQL调用。

  • 没有指针

    尽管Java在语法上与C语言非常相似，但它不支持直接指针或指针操作。您通过引用而不是按值传递除基本类型之外的所有参数。结果，保留了对象标识。Java不提供对指针的低级别直接访问，从而消除了内存损坏和泄漏的任何可能性。

  • 异常处理

    Java异常是对象。Java要求开发人员声明任何特定类中的方法可以抛出哪些异常。

  • 灵活命名空间

    Java定义了类并将它们放在一个镜像结构中，该结构镜像了Internet的域命名空间。您可以分发Java应用程序并避免名称冲突。Java扩展，例如 Java命名和目录接口（JNDI）为要联合的多个名称服务提供框架。Java的命名空间方法足够灵活，Oracle可以使用模式的概念来解析类名，完全符合JLS。

  • 安全

    Java字节码和JVM规范的设计允许内置机制来验证Java二进制代码的安全性。Oracle数据库与安全管理器实例一起安装，当与Oracle数据库安全性结合使用时，确定谁可以调用任何Java方法。

  • 连接关系数据库的标准

    Java数据库连接（JDBC）和SQLJ使Java代码能够访问和操作关系数据库中的数据。Oracle提供的驱动程序允许独立于供应商的可移植Java代码访问关系数据库。

1.1.3 Java虚拟机

与其他高级计算机语言一样，Java源代码编译为低级机器指令。在Java中，这些指令被称为字节码，因为每条指令都有一个字节的统一大小。大多数其他语言（如C）会编译为特定于机器的指令，例如特定于Intel或HP处理器的指令。

编译时，Java代码将转换为标准的，与平台无关的字节码集，由Java虚拟机（JVM）执行。JVM是一个单独的程序，针对运行Java代码的特定平台进行了优化。

图1-2说明了Java如何保持平台独立性。每个平台都安装了一个特定于操作系统的JVM。Java字节码通过JVM解释为适当的平台相关操作。

1.1.4 Java类层次结构

Java在大型类层次结构中定义类。层次结构的顶部是Object类。当你走过超类的继承链时，Java中的所有类都在某种程度上从Object类继承。当我们说B类继承自A类时，B类的每个实例都包含B类中定义的所有字段，以及A类中定义的所有字段。

图1-4说明了一个通用的Java类层次结构。FullTimeEmployee类包含Employee类中定义的id和lastName字段，因为它继承自Employee类。此外， FullTimeEmployee类还添加了另一个字段bonus ，该字段仅包含在FullTimeEmployee 。

您可以在类A或类B中定义的类B实例上调用任何方法。在该示例中， FullTimeEmployee实例可以调用仅在FullTimeEmployee类中定义的方法和在Employee类中定义的方法。
