B DBMS_JAVASCRIPT包

DBMS_JAVASCRIPT包提供了一个PL / SQL过程，用于从其参数标识的位置运行DBMS驻留的JavaScript代码。

本章包含以下主题：

  • DBMS_JAVASCRIPT概述

  • DBMS_JAVASCRIPT安全模型

B.1 DBMS_JAVASCRIPT概述

DBMS_JAVASCRIPT包是AUTHID CURRENT_USER包，并提供单个过程RUN ，用于运行JavaScript代码。RDBMS用户将他们的JavaScript代码保存在RDBMS中，作为存储在各自模式中的Java资源对象。

您可以使用以下两种方法之一在模式中创建脚本：

  • 使用loadjava工具或调用dbms_java.loadjava过程

  • 使用CREATE [OR REPLACE] JAVA RESOURCE命令

B.1.1运行程序

此过程调用Java代码，该代码定位Java资源，其名称作为script_name参数提供。然后，它通过Nashorn JavaScript编译器和运行时引擎提交其内容，以进行语法和语义分析，编译和评估。

在执行之前，Nashorn代码被编译为Java字节码。

句法

procedure run（script_name varchar2）;

您可以通过以下方式调用此过程：

调用dbms_javascript.run（'my_script.js'）;

注意：

  • 要运行此过程，必须授予DBJAVASCRIPT角色。

  • DBMS_JAVASCRIPT.RUN过程不运行JavaScript文本，也不查找文件系统中的脚本。Java资源名称是此过程的唯一有效输入。

B.2 DBMS_JAVASCRIPT安全模型

为了能够运行JavaScript代码，必须为RDBMS用户授予角色DBJAVASCRIPT。

DBMS_JAVASCRIPT.RUN过程的参数必须是一个字符串，用于标识先前加载到用户自己的模式中的有效Java资源。该过程拒绝以下参数：

  • 开放编码的脚本

  • 文件路径

  • 网址

这些限制对于防止用户编写PL / SQL代码很重要，PL / SQL代码从可能与JavaScript注入攻击有关的片段构建脚本。DBMS_JAVASCRIPT.RUN过程调用的Nashorn引擎的脚本模式扩展名被无条件禁用，包括SYS用户在内的任何人都无法运行它们。引擎实例化和使用依赖于oracle.DbmsJavaScriptUser permission授予。此外，所有与Nashorn相关的API都位于Oracle JVM的受限制的包空间中，并且当脚本尝试加载Java类时，将无条件地应用类过滤。类过滤实现以下目的：

  • 拒绝任何未通过SecurityManager管理的Java包访问检查的类

  • 无条件地限制包java.security ， java.lang.reflect和oracle.aurora.rdbms.security以供任何用户加载

DBA使用DBJAVASCRIPT角色来管理数据库中的JavaScript使用。该角色被授予唯一的授权，这是一个oracle.DbmsJavaScriptUser权限。
