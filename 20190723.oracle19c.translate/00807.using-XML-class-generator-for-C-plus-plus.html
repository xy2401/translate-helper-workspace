32使用XML类生成器进行C ++

这里的主题解释了如何为C ++使用可扩展标记语言（XML）类生成器。

访问XML C ++类生成器

XML C ++类生成器随Oracle数据库一起提供。

使用XML C ++类生成器

XML C ++类生成器从XML文档类型定义（DTD）或XML模式创建源文件。它为每个定义的元素生成一个类。然后，这些类在C ++程序中用于构造符合DTD或XML模式的XML文档。

当应用程序想要基于商定的DTD或XML架构将XML消息发送到另一个应用程序，或者作为Web表单的后端来构造XML文档时，这非常有用。使用这些类，C ++应用程序可以构造，验证和打印符合输入的XML文档。

类生成器与C ++的Oracle XML解析器一起使用，它解析输入并将解析的文档传递给类生成器。

外部DTD解析

XML C ++类生成器还可以通过使用用于C ++例程xmlparsedtd()的Oracle XML解析器直接解析外部DTD，而无需完整（虚拟）文档。提供的命令行程序xmlcg有一个-d选项，用于解析外部DTD。

使用XML C ++类生成器命令行实用程序

可以通过调用bin/xmlcg将独立类生成器调用为可执行文件。

您可以从命令行调用C ++类生成器：

xmlcg [options] input_file

选项 描述

-o directory

默认输入文件编码。

显示此用法帮助。

显示类生成器版本验证器选项。

是名字

Input是具有给定名称的XML Schema文件。生成name .cpp并name .h 。

input_file是带有<!的解析XML文档的名称<!DOCTYPE>定义，或解析的DTD，或XML Schema文档。XML文档必须具有关联的DTD。

XML C ++类生成器的DTD输入是包含DTD的XML文档，或者是外部DTD。文档正文本身被忽略 - 只有DTD是相关的，尽管文档必须符合DTD。

如果使用了无效选项，或者未提供任何输入，则输出使用消息。

输出两个源文件， name .h头文件和C ++文件， name .cpp 。它们以DTD文件命名。

输出文件通常用于生成XML文档。

为每个类（元素）提供了构造函数，允许以这些方式创建对象：

  • 最初为空，然后在初始创建后添加子项或数据

  • 使用初始完整的子集或初始数据创建

为#PCDATA （和Mixed）元素提供了一种方法来设置数据，并在适当的时候设置元素的属性。

输入到XML C ++类生成器

输入是包含DTD的XML文档。文件正文本身被忽略了;虽然虚拟文档必须符合DTD，但只有DTD是相关的。底层XML解析器仅接受文档和关联外部实体的文件名。

XML C ++类生成器示例3：CG示例程序

提供了示例程序CG， CG.cpp 。

它执行以下操作：

  1. 初始化XML解析器。

  2. 加载DTD（通过解析包含DTD的文件 - 忽略虚拟文档部分）。

  3. 使用生成的类创建一些对象。

  4. 调用验证函数，验证构造的类是否与DTD匹配。

  5. 将构造的文档写入Sample.xml 。
