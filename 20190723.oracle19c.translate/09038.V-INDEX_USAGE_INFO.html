8.4 V $ INDEX_USAGE_INFO

V$INDEX_USAGE_INFO跟踪自上次刷新以来的索引使用情况。每15分钟发生一次冲洗。每次刷新后， ACTIVE_ELEM_COUNT将重置为0， LAST_FLUSH_TIME将更新为当前时间。

柱 数据类型 描述

INDEX_STATS_ENABLED

数

指示是否启用索引使用情况统计信息。可能的值：

  • 0 ：禁用索引统计信息

  • 1 ：启用索引统计

INDEX_STATS_COLLECTION_TYPE

数

指示用于索引使用情况统计信息的集合类型。可能的值：

  • 0 ：表示ALL集合类型。使用此类型的集合，将为每个具有索引访问权限的执行收集统计信息。选择此统计信息收集类型可能会对性能产生一些影响。

  • 1 ：表示SAMPLED集合类型。使用此类集合，将根据采样收集统计信息（在收集统计信息时仅考虑少数执行）。这是默认的统计信息收集类型。

    使用SAMPLED集合类型收集的索引统计信息不如使用ALL集合类型收集的索引统计信息准确。

ACTIVE_ELEM_COUNT

数

自上次刷新以来的活动索引数

ALLOC_ELEM_COUNT

数

分配的索引条目数

MAX_ELEM_COUNT

数

可以跟踪的最大活动索引数

FLUSH_COUNT

数

自数据库启动以来成功刷新的次数

TOTAL_FLUSH_DURATION

数

自数据库启动以来完成索引使用情况统计信息刷新所花费的累计已用时间

LAST_FLUSH_TIME

TIMESTAMP（3）

最后一次冲洗的时间

STATUS_MSG

VARCHAR2（256）

状态消息（如果有）。此处报告了冲洗错误。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此行用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
