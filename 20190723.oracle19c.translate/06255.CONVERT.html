2.11 CONVERT

目的

使用CONVERT命令将表空间，数据文件或数据库转换为目标平台的格式，以便为跨不同平台的传输做准备。

在Oracle Database 10g及更高版本中，在以下方案中需要CONVERT DATAFILE或CONVERT TABLESPACE ：

  • 在其中具有值的平台之间传输数据文件 V$TRANSPORTABLE_PLATFORM.ENDIAN_FORMAT不同。

  • 无论ENDIAN_FORMAT是相同还是不同，都在平台之间传输具有还原段（通常为SYSTEM和UNDO表空间，以及使用回滚段的表空间）的表空间。通常，只有在转换整个数据库时才会转换SYSTEM和UNDO表空间。

  • 在其他特定于平台的数据文件上执行任何所需的转换，例如转换到HP Tru64操作系统或从HP Tru64操作系统转换时。

CONVERT一个用途是将表空间传输到存储在Oracle自动存储管理（Oracle ASM）中的数据库中。本机操作系统命令（如Linux cp和Windows COPY无法读取或写入Oracle ASM磁盘组。

先决条件

CONVERT命令必须支持源平台和目标平台。查询V$TRANSPORTABLE_PLATFORM以确定支持的平台。仅当源和目标平台都包含在此视图中时，才支持跨平台表空间传输。

源数据库和目标数据库都必须在初始化参数COMPATIBLE设置为10.0.0或更高版本的情况下运行。请注意以下兼容性先决条件：

  • 如果COMPATIBLE小于11.0.0，那么只读表空间或现有传输的表空间必须至少进行一次读/写操作才能传输到不同的平台。您可以打开一个表空间读/写，然后立即再次将其设置为只读。

  • 如果COMPATIBLE为11.0.0或更高，则前面的读/写表空间限制不适用。但是，任何现有的已传输表空间必须在传输之前以COMPATIBLE设置为10.0进行读/写。

CONVERT TABLESPACE先决条件

当作为TARGET连接到源数据库并转换源平台上的表空间时，您只能使用CONVERT TABLESPACE 。

必须安装或打开源数据库。要转换的表空间在转换时必须是只读的。转换源数据库上的表空间时，目标数据库的状态无关紧要。

CONVERT DATAFILE先决条件

当作为TARGET连接到目标数据库并在目标平台上转换数据文件副本时，您只能使用CONVERT DATAFILE 。

如果运行CONVERT DATABASE ON DESTINATION生成的CONVERT DATAFILE脚本，则必须使用NOMOUNT选项启动目标数据库实例。如果未运行CONVERT DATABASE ON DESTINATION生成的CONVERT DATAFILE脚本，则可以启动，装入或打开目标数据库。

在目标数据库上转换数据文件副本时，源数据库的状态无关紧要。但是，如果在目标数据库上运行CONVERT DATAFILE脚本作为数据库转换的一部分，并且脚本直接访问源数据库上的数据文件（例如，通过NFS安装），那么源数据库必须是只读。

转换目标主机上的表空间时，必须使用CONVERT DATAFILE而不是CONVERT TABLESPACE因为目标数据库在转换期间无法将数据文件与表空间关联。转换表空间所需的数据文件后，可以将它们传输到目标数据库。

CONVERT DATABASE先决条件

当作为TARGET连接到源数据库时，您只能使用CONVERT DATABASE ， 源数据库必须以只读方式打开。即使您运行CONVERT DATABASE ON DESTINATION ，目标数据库的状态在执行CONVERT DATABASE时也无关紧要。

由于CONVERT DATABASE使用与CONVERT TABLESPACE和CONVERT DATAFILE相同的机制来转换数据文件，因此表空间和数据文件的使用说明和限制也适用。

CONVERT DATABASE的主要附加先决条件是源平台和目标平台必须共享相同的endian格式。例如，您可以将数据库从Microsoft Windows传输到Linux for x86（小端），或者从HP-UX传输到AIX（都是big-endian），但不能从Solaris传输到Linux x86。但是，您可以手动在目标平台上创建新数据库，并使用CONVERT TABLESPACE或CONVERT DATAFILE从源数据库传输单个表空间。

即使源平台和目标平台的字节序格式相同，可传输数据库的数据文件也必须在源主机或目标主机上进行转换。与跨平台传输表空间不同，如果端序格式相同，则不需要进行转换，传输整个数据库需要转换某些类型的块（例如还原段中的块）以确保与目标平台的兼容性。

使用说明

CONVERT不会更改输入文件，因为转换不会在适当的位置执行。而是，RMAN将转换后的文件写入指定的输出目标。

数据类型限制

CONVERT 不执行存储在以下数据类型中的数据的字节序转换：

  • 生的

  • LONG RAW

  • BLOB

  • ANYTYPE / ANYDATA/ANYDATASET

  • 用户定义的类型或Oracle抽象类型（例如ORDImage媒体类型），包含任何上述数据类型的属性

注意：

虽然转换了BFILE数据类型中的文件定位器，但不转换BFILE指向的外部文件。

要在以特定于平台的格式存储数据的基础类型上构建的数据库之间传输对象，请使用数据泵导入和导出实用程序。

在Oracle Database 10g之前，可变宽度字符集（如UTF8中的CLOB以依赖于endian的固定宽度格式存储。CONVERT命令不会对这些CLOB执行转换。相反，RMAN捕获每个LOB列的endian格式并将其传播到目标数据库。SQL层随后对此数据的读取将根据任一端格式正确解释数据，并在表空间可写时以与字节无关的方式将其写出。在Oracle数据库10g及更高版本中创建的CLOB存储在与平台无关的字符集AL16UTF16 。

兑换

此子句指定要转换的对象：数据文件，表空间或数据库。

语法元素 描述

数据库

将数据库的所有数据文件转换为目标平台的格式，并确保创建其他所需的数据库文件。

在一个多租户容器数据库（CDB），转换CDB中的所有数据文件。您连接到根以转换整个CDB。请参阅“ 连接到CDB和PDB ” 。

注意：不支持通过连接到PDB然后使用CONVERT DATABASE命令CONVERT DATABASE PDB。

您使用CONVERT DATABASE将整个数据库从源平台传输到目标平台。源平台和目标平台必须具有相同的endian格式。

根据具体情况，您可以在源平台或目标平台上使用CONVERT DATABASE （请参阅示例2-68 ）。数据库的以下部分不直接传输：

  • 不传输源数据库中的重做日志和控制文件。RMAN在传输期间为目标数据库创建新的控制文件和重做日志，并在创建新数据库后执行OPEN RESETLOGS 。转换后的数据库的控制文件不包含源数据库中的RMAN存储库。源数据库中的备份不适用于已转换的数据库。

  • BFILE不运输。CONVERT DATABASE输出提供使用BFILE数据类型的对象列表，但您必须手动复制BFILE并在目标平台上修复它们的位置。

  • 不传输本地管理的临时表空间的数据文件。通过运行传输脚本在目标平台上重新创建临时表空间。

  • 不传输外部表和目录。CONVERT DATABASE输出显示受影响对象的列表，但您必须在目标平台上重新定义这些对象。有关管理外部表和目录的详细信息，请参见“ Oracle数据库管理员指南” 。

  • 密码文件不会被传输。如果密码文件与源数据库一起使用，则CONVERT DATABASE的输出包括所有用户名及其相关权限的列表。使用此信息在目标数据库上创建新密码文件。有关管理密码文件的详细信息，请参见“ Oracle数据库管理员指南” 。

使用CONVERT DATABASE ，RMAN会检测到以下问题，并且在修复之前不会继续：

  • 数据库具有活动或不确定的事务。

  • 数据库已保存还原段。

  • 数据库COMPATIBILITY设置低于10。

  • 当数据库COMPATIBILITY设置为10或更高时，某些表空间尚未打开读/写。

数据库根

在一个 CDB，将根数据文件转换为目标平台的格式，并确保创建其他所需的数据库文件。按照“ 连接到CDB和PDB ”中的说明连接到根目录并转换数据文件。

有关转换数据库的一般信息，请参阅DATABASE参数的先前描述。

TABLESPACE tablespace_name convertOptionList

指定的平台必须是V$TRANSPORTABLE_PLATFORM的PLATFORM_NAME列中列出的PLATFORM_NAME 。必须使用源或目标平台的确切名称作为CONVERT命令的参数。以下SQL语句查询支持的Linux平台：

例子

示例2-65在源平台上转换表空间

假设您必须将源数据库prodlin中的表空间finance和hr转换为目标数据库prodsun的平台格式。finance表空间包括数据文件/disk2/orahome/fin/fin01.dbf和/disk2/orahome/fin/fin02.dbf 。hr表空间包括数据文件/disk2/orahome/fin/hr01.dbf和/disk2/orahome/fin/hr02.dbf 。

prodlin数据库在Linux主机lin01上运行。您查询V$DATABASE并发现该平台名称是Linux IA (32-bit)并使用little-endian格式。prodsun数据库在Solaris主机sun01上运行。您查询V$TRANSPORTABLE_PLATFORM并发现Solaris主机的PLATFORM_NAME是Solaris[tm] OE (64-bit) ，它使用大端格式。

您计划转换源主机上的表空间，并将转换后的数据文件存储在主机lin01上的/tmp/transport_to_solaris/ 。该示例假定您已在源数据库上将COMPATIBLE设置为10.0或更高。

在源主机lin01 ，启动RMAN客户端并运行以下命令，其中SBU是具有SYSBACKUP权限的任何用户：

CONNECT TARGET“sbu @ prodlin AS SYSBACKUP”目标数据库密码：连接到目标数据库的密码 ：PRODLIN（DBID = 39525561）ALTER TABLESPACE finance READLYLY; ALTER TABLESPACE hr阅读; CONVERT TABLESPACE finance，hr TO PLATFORM'Solaris [tm] OE（64-bit）'FORMAT'/ tmp / transport_to_solaris /％U';

结果是/tmp/transport_to_solaris/目录中的一组转换数据文件，其中数据采用右端序列，用于Solaris 64位平台。

从这一点开始，您可以按照表格空间传输的其余大纲进行操作。使用Data Pump Export实用程序创建结构信息文件，将结构信息文件和转换后的数据文件从/tmp/transport_to_solaris/到目标主机上的所需目录，然后使用Data将表空间插入新数据库泵进口实用程序。

示例2-66在目标平台上转换数据文件

此示例假定您要将来自主机sun01上的数据库prodsun的finance和hr表空间转换为目标主机lin01上的数据库prodlin可用的格式。您将未转换的数据文件临时存储在目标主机lin01上的目录/tmp/transport_from_solaris/ ，并使用CONVERT DATAFILE执行转换。将数据文件传输到目标数据库时，它们存储在/disk2/orahome/dbs 。

该示例假定您已执行以下步骤以准备表空间传输：

  • 您使用数据泵导出实用程序来创建结构信息文件（在我们的示例中命名为expdat.dmp ）。

  • 您使finance和hr表空间在源数据库上是只读的。

  • 您使用操作系统实用程序将expdat.dmp和未转换的数据文件复制到/tmp/transport_from_solaris目录中的目标主机lin01 。数据文件存储为：

    • /tmp/transport_from_solaris/fin/fin01.dbf

    • /tmp/transport_from_solaris/fin/fin02.dbf

    • /tmp/transport_from_solaris/hr/hr01.dbf

    • /tmp/transport_from_solaris/hr/hr02.dbf

  • 您在V$TRANSPORTABLE_PLATFORM查询了源平台的名称，发现PLATFORM_NAME是Solaris[tm] OE (64-bit) 。

执行转换时请注意以下注意事项：

  • 按文件名识别数据文件，而不是按表空间名称识别。在插入数据文件之前，本地实例无法知道预期的表空间名称。

  • FORMAT参数控制转换的数据文件的名称和位置。

  • 在目标主机上进行转换时，必须使用FROM参数指定源平台。否则，RMAN假定源平台也是执行转换的主机的平台。

您启动RMAN客户端并连接到目标数据库prodlin作为TARGET 。 sbu是被授予SYSBACKUP权限的用户。以下CONVERT命令将要传输的数据文件转换为目标主机格式，并将结果存入/disk2/orahome/dbs ：

CONNECT TARGET“sbu @ prodlin AS SYSBACKUP”目标数据库密码：连接到目标数据库的密码 ：PRODLIN（DBID = 39525561）CONVERT DATAFILE'/tmp/transport_from_solaris/fin/fin01.dbf'，'/ tmp / transport_from_solaris / fin / fin02。 dbf'，'/ tmp / transport_from_solaris / hr / hr01.dbf'，'/ tmp / transport_from_solaris / hr / hr02.dbf'DB_FILE_NAME_CONVERT'/ tmp / transport_from_solaris / fin'，'/ disk2 / orahome / dbs / fin'，' / tmp / transport_from_solaris / hr'，'/ disk2 / orahome / dbs / hr'来自PLATFORM'Solaris [tm] OE（64位）';

结果是以下数据文件已转换为Linux格式：

  • /disk2/orahome/dbs/fin/fin01.dbf

  • /disk2/orahome/dbs/fin/fin02.dbf

  • /disk2/orahome/dbs/hr/hr01.dbf

  • /disk2/orahome/dbs/hr/hr02.dbf

从这一点开始，请遵循表空间传输的其余大纲。使用数据泵导入将转换的表空间插入新数据库，并使表空间可读/写（如果适用）。

示例2-68将数据库传输到其他平台

CONVERT DATABASE的参数取决于您是否计划转换源或目标平台上的数据文件。有关源和目标平台上的转换过程和扩展示例的说明，请参阅“ Oracle数据库备份和恢复用户指南” 。在尝试数据库转换之前，请完整阅读该讨论。

假设您要将Linux主机上的数据库prod传输到Windows主机。您决定转换源主机而不是目标主机上的数据文件。以下示例将RMAN连接到Linux主机上的PROD数据库，并使用CONVERT DATABASE NEW DATABASE转换数据文件并生成传输脚本：

CONNECT TARGET“sbu @ lin01 AS SYSBACKUP”目标数据库密码：连接到目标数据库的密码 ：PROD（DBID = 39525561）CONVERT DATABASE NEW DATABASE'prodwin'TRANSPORT SCRIPT'/ tmp / convertdb / transportscript'TO PLATFORM'Microsoft Windows IA（32 -bit）'DB_FILE_NAME_CONVERT'/ disk1 / oracle / dbs'，'/ tmp / convertdb';

在以下变体中，您希望将在Linux主机上运行的数据库传输到Windows主机，但是您希望转换目标主机而不是源主机上的数据文件。 sbu是被授予SYSBACKUP权限的用户。以下示例将RMAN连接到Linux主机上的prod数据库，并执行CONVERT DATABASE ON DESTINATION PLATFORM ：

CONNECT TARGET“sbu @ lin01 AS SYSBACKUP”目标数据库密码：连接到目标数据库的密码 ：PROD（DBID = 39525561）CONVER DATABASE ON DESTINATION PLATFORM CONVERT SCRIPT'/tmp/convertdb/convertscript.rman'Transport SCRIPT'/ tmp / convertdb / transportscript.sql'NED DATABASE'prodwin'FORMAT'/ tmp / convertdb /％U';

在Linux数据库上执行的CONVERT DATABASE ON DESTINATION PLATFORM命令生成一个转换脚本，该脚本可以在Windows主机上运行，以将数据文件转换为Windows格式。CONVERT DATABASE命令还会生成传输脚本。

示例2-69将数据库传输到其他平台和存储类型

在此方案中，您在名为sun01的Solaris主机上有一个数据库prod ，您希望将其移动到名为aix01的AIX主机。Solaris数据文件存储在非ASM文件系统中，但您希望将数据文件存储在AIX主机上的ASM中。

以下示例连接到sun01并运行CONVERT DATABASE以生成必要的脚本：

CONNECT TARGET“sbu @ sun01 AS SYSBACKUP”目标数据库密码：连接到目标数据库的密码 ：PROD（DBID = 39525561）CONVER DATABASE ON DESTINATION PLATFORM CONVERT SCRIPT'/tmp/convert_newdb.rman'TRATER SCRIPT'/tmp/transport_newdb.sql'新数据库'prodaix'DB_FILE_NAME_CONVERT'/ u01 / oradata / DBUA / datafile'，'+ DATA';

convert脚本包含以下形式的语句，其中your_source_platform代表您的源平台：

CONVERT DATAFILE '/u01/oradata/DBUA/datafile/o1_mf_system_2lg3905p_.dbf' FROM PLATFORM 'your_source_platform'格式'+ DATA / o1_mf_system_2lg3905p_.dbf';

要减少转换的停机时间，您可以使用NFS而不是通过网络复制数据文件或还原备份。例如，您可以在AIX主机上将Solaris文件系统挂载为/net/solaris/oradata 。在这种情况下，您将编辑转换脚本以引用NFS挂载的目录作为要转换的源数据文件的位置，将命令放入以下格式：

CONVERT DATAFILE '/net/solaris/oradata/DBUA/datafile/o1_mf_system_2lg3905p_.dbf' FROM PLATFORM 'your_source_platform'格式'+ DATA / o1_mf_system_2lg3905p_.dbf';

然后，将RMAN连接到目标数据库实例，在本例中为主机aix01上的实例，并转换数据文件。在转换期间，主机sun01上的数据库仍处于打开只读模式。然后，将SQL * Plus连接到aix01上的数据库实例，并运行传输脚本以创建数据库。
