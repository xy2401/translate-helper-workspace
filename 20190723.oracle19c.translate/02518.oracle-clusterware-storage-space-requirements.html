Oracle Clusterware存储空间要求

使用此信息可确定基于冗余类型的最小磁盘数和最小磁盘空间要求，以便为各种Oracle Cluster部署安装Oracle Clusterware文件。

Oracle Cluster部署类型所需的Oracle Clusterware可用存储空间总量

在安装Oracle独立群集期间，如果为Grid Infrastructure Management Repository（GIMR）创建MGMT磁盘组，则安装程序要求您使用具有至少35 GB可用空间的磁盘组。

注意：

从Oracle Grid Infrastructure 19c开始，为Oracle Standalone Cluster部署配置GIMR是可选的。升级到Oracle Grid Infrastructure 19c时，仅当源Grid Home配置了GIMR时才会创建新的GIMR。

根据要安装的群集配置，Oracle Clusterware空间要求因不同的冗余级别而异。下表列出了每个群集配置和冗余级别的空间要求。

注意：

DATA磁盘组存储OCR和表决文件， MGMT磁盘组存储GIMR和Oracle Clusterware备份文件。
