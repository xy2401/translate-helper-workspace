1支持迁移的工具和产品简介

在将应用程序迁移到Oracle数据库之前，您必须了解Oracle数据库概念中描述的几个关键点。

在讨论以数据库为中心的企业的迁移时，请务必记住，数据库模式和数据的实际迁移只是流程的一部分。核心业务解决方案的迁移通常涉及多个数据库和应用程序，它们协同工作以提供推动组织收入的产品和服务。有关准备迁移计划的详细信息，请参阅“ Oracle SQL Developer用户指南” 。

用于迁移支持的Oracle数据库功能

Oracle Database 12c引入了大量功能，这些功能共同增强了非Oracle数据库应用程序到Oracle数据库的迁移过程。

SQL转换框架

将非Oracle数据库迁移到Oracle数据库的关键部分涉及将非Oracle SQL语句转换为Oracle数据库可接受的SQL语句。应用程序的非Oracle SQL语句的转换是一个手动且繁琐的过程。为了最大限度地减少工作量，或者无需转换这些语句，Oracle Database 12 c引入了一项名为SQL Translation Framework的新功能。SQL Translation Framework从客户端应用程序接收这些SQL语句，然后在运行时进行翻译。

SQL Translation Profile在数据库中注册SQL Translater，以便它可以处理非Oracle客户端应用程序的SQL转换。如果在执行SQL语句时发生错误，则SQL转换程序可以将Oracle错误代码和ANSI SQLSTATE转换为应用程序所需的特定于供应商的值。然后将翻译后的语句保存在SQL翻译配置文件中，由用户自行决定进行检查和编辑。

SQL Translation Framework的优点如下：

  • SQL语句，Oracle错误代码和ANSI SQLSTATE是自动的。

  • 翻译是集中的和可检查的。

  • 用户可以选择提取翻译并将其插回到应用程序中。

支持MySQL应用程序

用于MySQL的Oracle数据库驱动程序简化了最初开发用于MySQL数据库的应用程序的迁移。此功能有两个主要优点：

  • 它使企业能够重用相同的应用程序来使用存储在MySQL数据库和Oracle数据库中的数据

  • 它降低了将MySQL应用程序迁移到Oracle数据库的成本和复杂性

Oracle数据库支持客户端接口中具有相同语义的所有MySQL功能。

SQL语句转换的限制

在翻译SQL语句时，SQL Translation具有以下限制：

  • SQL Translation忽略以下SQL构造：

    • 不使用表的ENGINE规范;只有一个存储引擎，即Oracle 。

    • ENUM和SET类型用作VARCHAR2 。如果在数值上下文中检索这些值，则不会将这些值转换为索引值。

  • SQL Translation在尝试处理以下SQL构造时会生成错误;必须重新编码应用程序。

    • Oracle不支持空间数据类型，例如GEOMETRY ， POINT ， LINESTRING ， POLYGON ， GEOMETRYCOLLECTION ， MULTILINESTRING ， MULTIPOINT和MULTIPOLYGON 。

      Oracle不支持特定于MySQL的NLS命令。

  • 以下SQL命令提供特定于Oracle的输出或具有Oracle特定的效果：

    • SHOW DATABASES只显示一个数据库，即oracle 。

    • SHOW ENGINES仅显示Oracle引擎。

    • CREATE PROCEDURE必须遵循Oracle Database 12 c中的 Oracle PL / SQL规范。

  • 以下数据类型在Oracle数据库中的行为与本机数据库中的预期行为不同：

    • ENUM数据类型的列创建为VARCHAR2(4000) 。没有进行插入验证。

    • SET数据类型的列创建为VARCHAR2(64) 。没有进行插入验证。

有关更多详细信息，请参阅Oracle客户端库驱动程序和Oracle MySQL客户端库驱动程序的API参考 。

支持身份列

Oracle Database 12 c实现符合ANSI标准的IDENTITY列。使用标识列的数据库系统的迁移得到简化，可以利用这一新功能。

此功能通过增强DEFAULT或DEFAULT ON NULL语义来实现自动增量，以供SEQUENCE.NEXTVAL和SYS_GUID ，支持内置函数和隐式返回默认值。

隐含的陈述结果

从Oracle Database 12 c第2版（12.2）开始，Oracle隐式地向客户端应用程序返回在存储过程中执行的SQL语句的结果，绕过显式使用REF CURSOR 。此功能消除了重写客户端代码的开销。

隐式语句结果使用户能够编写存储过程，其中每个预期查询（ FOR关键字之后的语句）是OPEN游标变量的一部分。当代码从其他供应商环境迁移到Oracle数据库时，PL / SQL层会添加等效功能，并使SELECT语句能够将结果传递给客户端。然后，存储过程可以使用DBMS_SQL.RETURN_RESULT过程将结果直接返回给客户端。可以调用SQL * Plus FORMAT命令及其变体来自定义输出。

有关DBMS_SQL程序包的信息，请参阅Oracle Database PL / SQL程序包和类型参考 。有关如何使用格式输出的信息，请参阅SQL * Plus用户指南和参考 。

ODBC支持隐式结果

从Oracle Database 12 c开始 ，ODBC应用程序通过新函数SQLMoreResults()为隐式结果提供支持。ODBC驱动程序已得到增强，可以使用以下新的OCI API来增强迁移过程：

  • OCIStmtGetNextResult()函数

  • OCI_ATTR_IMPLICIT_RESULT_COUNT属性

  • OCI_RESULT_TYPE_SELECT属性

对隐式结果的ODBC支持允许迁移使用捆绑在存储过程中的多个结果集的Sybase和SQL Server应用程序。Oracle通过将语句或过程发送到服务器来实现此目的，其中非Oracle SQL被转换为Oracle语法。

增强的SQL到PL / SQL绑定处理

在早期版本的Oracle数据库中，SQL表达式无法调用具有非SQL数据类型的形式参数或返回类型的PL / SQL函数。

从Oracle Database 12 c开始 ，PL / SQL匿名块，SQL CALL语句或SQL查询可以调用具有以下类型参数的PL / SQL函数：

  • 布尔

  • 记录在包规范中的记录

  • 在包规范中声明的集合

SQL TABLE运算符也得到了增强，因此您可以查询本地作用域类型的PL / SQL集合作为TABLE运算符的参数。这里，集合可以是由PLS_INTEGER索引的嵌套表类型， VARRAY或PL / SQL索引表。

此功能扩展了TABLE运算符的灵活性，并使非Oracle存储过程代码轻松迁移到PL / SQL。

ODBC驱动程序支持应用程序迁移

ODBC驱动程序支持使用SQL Translation Framework将第三方应用程序迁移到Oracle数据库。这使得非Oracle数据库SQL语句可以针对Oracle数据库运行。在开始将第三方ODBC应用程序迁移到Oracle数据库之前，请参阅“ 如何使用SQL Translation Framework ” 。

要将此功能与ODBC应用程序一起使用，必须将作为SQL Translation Framework设置的一部分创建的服务名称指定为.odbc.ini文件中的ServerName=条目。

如果您需要支持将Oracle错误（ORA错误）转换为本机数据库，那么一旦您的应用程序开始针对Oracle数据库运行，您必须在.odbc.ini文件中启用SQLTranslateErrors=T条目。有关此主题的更多信息，请参见“ ODBC应用程序的SQL转换 ” 。

其他支持迁移的Oracle产品

Oracle建议使用多个Oracle产品作为整体迁移策略的一部分。

OEM调整和性能包

对于每种类型的迁移，应用程序中使用的一些SQL语句必须更改，并且必须重新构建一些索引。Oracle SQL Tuning和Performance Pack为应用程序迁移的优化步骤提供了指导。

Oracle GoldenGate

Oracle GoldenGate是一个全面的软件包，用于在异构数据环境中实现数据复制。该产品集支持运营和分析企业系统之间的高可用性解决方案，实时数据集成，事务变更数据捕获，数据复制，转换和验证。

Oracle GoldenGate支持在整个企业的多个异构平台之间交换和处理事务级别的数据。其模块化体系结构可灵活地提取和复制所选数据记录，事务更改以及跨各种拓扑的DDL（数据定义语言）更改。

迁移非常大的数据库时，将数据从一个数据库复制到另一个数据库的实际过程非常耗时。在此期间，企业必须继续使用旧解决方案提供服务，旧解决方案会更改某些数据。必须捕获这些运行时更改并将其传播到Oracle数据库。Oracle GoldenGate捕获这些更改并启用并行测试，以确保新解决方案按计划执行。

Oracle数据库网关

Oracle数据库网关可满足不同数据访问的需求。在异构分布式环境中，网关可以与Oracle应用程序中的任意数量的非Oracle系统集成。它们支持与IBM DB2，Microsoft SQL Server和Excel等数据存储，IBM CICS等事务管理器以及IBM WebSphere MQ等消息队列系统的集成。

有关Oracle数据库网关的更多信息，请http://www.oracle.com/technetwork/database/gateways/index.html
