C.1整理派生

确定SQL操作的字符结果的排序规则的过程称为排序规则派生。此类操作可以是运算符，列引用，字符文字，绑定变量引用，函数调用，CASE表达式或查询子句。

SQL表达式中的每个字符值都具有派生的排序规则和派生的强制级别 。

操作结果的派生校对和强制级别基于操作参数的校对和强制级别。操作的派生相关字符参数是用于派生运算符结果的排序规则的参数。运算符可以具有零个或多个与派生相关的字符参数，以及零个或多个其他字符参数，例如不直接与派生相关参数交互的标志或其他控制信息。如果参数的值包含在结果中，则在某些变换之后或不经历任何变换时，参数被视为与派生相关 。

作为格式模型，模式，标志字符串或系统信息虚拟表中的键的参数不被视为与派生相关的参数。例如，内置函数TO_CHAR(arg1,arg2,arg3)没有与派生相关的参数，因为主参数arg1不是字符数据类型。两个字符参数arg2和arg3不是与派生相关的参数，因为它们只定义主参数arg1转换的格式和参数。

没有派生相关参数的操作结果的派生校对和强制性级别与将字符文字放在操作位置的表达式中时相同。

以下是返回字符值并具有派生相关参数的操作的排序规则派生规则。这些规则在表达式树中递归应用。这些规则基于SQL标准版本ISO / IEC 9075-2：1999。

使用派生相关字符参数arg1，arg2，...，argn的操作结果的派生排序规则是：

  • 如果至少有一个参数的强制级别为0，那么所有具有强制级别0的参数必须具有相同的排序规则，这是结果的派生排序规则。结果的强制性等级为0。如果两个具有强制级别0的参数具有不同的排序规则，则会报告错误。

  • 否则，如果至少有一个参数具有强制级别1，则表达式结果具有强制级别1，并且不为其分配排序规则。

  • 否则，如果LCL是参数的数字最低强制级别 ，那么：

    • 如果LCL的所有参数具有相同的排序规则，则此排序规则是结果的派生排序规则，结果的强制级别为LCL。

    • 否则，表达式的结果具有强制级别1，并且不为其分配排序规则。

注意：

集合运算符具有表达式列表的参数。对于集合运算符，对表达式列表的每个参数的相应元素分别执行排序规则派生。例如，在查询中：

SELECT expr1，expr2 FROM t1 UNION SELECT expr3，expr4 FROM t2

对于结果集的第一列和第二列，分别派生排序规则。对于第一列，排序规则派生规则应用于expr1和expr3 。对于第二列，规则应用于expr2和expr4 。

绑定变量引用的排序推导

在OCI中，您可以使用OCI_ATTR_COLLATION_ID属性的值为查询或DML语句中的绑定变量传递排序OCI_ATTR_COLLATION_ID 。可以使用OCIAttrSet()函数在任何支持的归类ID的绑定变量句柄上设置OCI_ATTR_COLLATION_ID属性。允许使用命名归类和伪归类的ID。在这种情况下，绑定变量引用的派生强制性级别为0。

当OCI_ATTR_COLLATION_ID属性值在绑定变量句柄上设置为OCI_COLLATION_NONE （默认值）时，绑定变量的排序规则为USING_NLS_COMP ，绑定变量引用的派生强制级别为4。

OCI不检查排序规则是否对绑定变量的给定数据类型有效。如果为非字符数据类型变量设置了OCI_ATTR_COLLATION_ID属性值，则数据库服务器将忽略该值。

PL / SQL表达式中当前忽略了绑定变量的排序规则。出于向前兼容性原因，不应为传递给匿名PL / SQL块的绑定变量设置OCI_ATTR_COLLATION_ID属性，除非在SQL语句中仅引用变量。

C.2整理确定

排序确定是在执行排序规则敏感操作期间选择要应用的正确排序规则的过程。对结果敏感的操作可以是SQL运算符，条件，内置函数调用，CASE表达式或查询子句。

对于12.2之前的Oracle数据库版本，操作应用的排序规则仅由NLS_SORT和NLS_COMP会话参数确定。

注意：

NLS_UPPER, NLS_LOWER, NLS_INITCAP,和NLSSORT的可选第二个参数是例外。

从Oracle Database 12.2开始，操作应用的排序规则由其参数的派生数据绑定排序规则确定。将伪排序规则确定为要使用的排序NLS_SORT ，将检查NLS_SORT和NLS_COMP会话参数以提供要应用的实际命名排序规则。

注意：

归类确定不必适用于应用归类推导的相同操作。例如， TO_CHAR函数不是归类敏感的，因此不需要归类确定。但是， TO_CHAR函数返回需要排序规则声明的字符结果，因此排序规则派生适用于它。相反， INSTR函数需要匹配字符并需要为此匹配操作确定的排序规则。但是， INSTR函数的结果是一个数字，因此不需要整理派生。

操作的确定相关字符参数是用于确定操作要使用的排序规则的参数。校对敏感操作可以具有一个或多个确定相关字符参数和零个或多个其他字符参数，例如不直接与确定相关参数交互的标志或其他控制信息。

如果在操作评估期间比较其值，则认为该参数与确定相关。作为格式模型，标志字符串或系统信息虚拟表中的键的参数不被视为与确定相关的参数。但是，模式参数可以是与确定相关的参数。例如， LIKE谓词的三个参数中的两个 - 参数和模式 - 是与确定相关的参数。第三个参数 - 转义字符 - 不被认为是与决定相关的参数。另一个例子是内置函数REGEXP_COUNT ，它有四个参数 - source_char ， pattern ， position和match_param 。与决定相关的参数是source_char和pattern ，它们包含要比较的字符串。非确定相关的字符参数是position ，它是数字， match_param ，它提供匹配操作的参数。

以下是用于确定用于具有确定相关字符参数arg1，arg2，...，argn的操作的排序规则的排序规则确定规则。这些规则基于SQL标准版本ISO / IEC 9075-2：1999。

  • 如果operation是相等条件并且用于强制执行外键约束，则要使用的排序规则是所引用的主键或唯一键列的声明排序规则。此声明的排序规则必须与外键列的声明排序规则相同。

  • 否则，如果至少一个参数的派生强制级别为0，则所有具有强制级别0的参数必须具有相同的排序规则，并且操作将使用此排序规则。如果两个具有强制级别0的参数具有不同的排序规则，则会报告错误。

  • 否则，如果至少有一个参数的派生强制级别为1，则报告错误。

  • 否则，如果LCL是参数的数字最低强制级别 ，那么：

    • 如果具有LCL的所有参数具有相同的排序规则，则操作将使用该排序规则。

    • 否则，报告错误。

当确定的排序规则是伪排序规则时，受影响的操作必须引用会话或数据库设置NLS_SORT或NLS_COMP或两者来确定要应用的实际命名排序规则。数据库设置用于虚拟列中的表达式， CHECK约束和细粒度审计（FGA）规则。

涉及CLOB或NCLOB数据类型值的操作的排序规则确定规则必须导致伪排序USING_NLS_COMP ，否则报告错误。

注意：

某些条件，集合运算符和查询子句具有表达式列表的参数。在这种情况下，对表达式列表中的每个自变量的相应比较元素执行核对确定。例如，在条件中：

（expr1，expr2）IN（SELECT expr3，expr4 FROM t1）

对于比较元素对，分别确定校对。首先，将核对确定规则应用于expr1和expr3 。然后，规则应用于expr2和expr4 。当评估条件时，使用第一确定的核对将expr1的值与expr3的值进行比较， expr4使用第二确定的核对将expr2的值与expr4值进行比较。同样，在查询中：

SELECT expr1，expr2 FROM t1 MINUS SELECT expr3，expr4 FROM t2

核对确定规则首先应用于expr1和expr3 ，然后应用于expr2和expr4 。当评估MINUS运算符时，使用第一确定的核对将expr1的值与expr3的值进行比较， expr4使用第二确定的核对将expr2的值与expr4值进行比较。

在查询中：

SELECT * FROM t1 ORDER BY expr1，expr2，expr3

首先使用派生的collation expr1对expr1进行排序，然后通过使用派生的collation expr2对expr2值进行排序，然后使用派生的collation expr3对expr3值进行排序来打破tie。ORDER BY列表中的每个位置都被视为行值的单独比较运算符。
