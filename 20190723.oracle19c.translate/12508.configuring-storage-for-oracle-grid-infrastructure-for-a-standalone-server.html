9为独立服务器配置Oracle Grid Infrastructure的存储

完成这些过程以将Oracle Grid Infrastructure用于独立服务器，其中包括Oracle自动存储管理（Oracle ASM）。

用于独立服务器的Oracle Grid Infrastructure（也称为Oracle Restart）为单实例Oracle数据库提供系统支持。Oracle ASM是Oracle数据库文件的卷管理器和文件系统，支持单实例Oracle数据库和Oracle Real Application Clusters（Oracle RAC）配置。Oracle自动存储管理还支持满足您的应用程序需求的通用文件系统，包括Oracle数据库二进制文件。Oracle自动存储管理是Oracle推荐的存储管理解决方案。它提供了传统卷管理器和文件系统的替代方案。

注意：

  • 如果要使用Oracle ASM或Oracle Restart，则必须在安装和创建数据库之前为独立服务器安装Oracle Grid Infrastructure。否则，您必须使用Oracle Restart手动注册数据库。

  • 您既不能在Oracle Grid Infrastructure集群成员节点上安装Oracle Restart，也不能将Oracle Restart服务器添加到Oracle Grid Infrastructure集群成员节点。Oracle Restart支持在一台服务器上支持单实例数据库，而Oracle Grid Infrastructure for Cluster支持群集上的单实例或Oracle RAC数据库。

  • 配置Oracle自动存储管理存储
    确定存储要求和Oracle ASM磁盘组选项。
  • 使用Oracle ASMFD配置存储设备路径持久性
    Oracle ASM筛选器驱动程序（Oracle ASMFD）维护存储文件路径持久性，并有助于保护文件免遭意外覆盖。
  • 在Oracle Solaris上为Oracle ASM配置磁盘设备
    完成这些任务以配置磁盘设备以与Oracle自动存储管理（Oracle ASM）配合使用。
  • 为Oracle自动存储管理创建DAS或SAN磁盘分区
    您可以将直接连接存储（DAS）和存储区域网络（SAN）磁盘与Oracle ASM一起使用。
  • 创建Oracle数据库文件的目录
    您可以将Oracle数据库和恢复文件存储在与配置文件不同的文件系统中。
  • 在NAS设备上创建文件以与Oracle自动存储管理一起使用
    如果您拥有经过认证的NAS存储设备，则可以在NFS安装目录中创建零填充文件，并将这些文件用作Oracle ASM磁盘组中的磁盘设备。
