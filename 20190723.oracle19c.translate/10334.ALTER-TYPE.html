更改类型

目的

对象类型使用PL / SQL定义。因此，本节提供了一些常规信息，但有关语法和语义的详细信息，请参阅Oracle Database PL / SQL语言参考 。

使用ALTER TYPE语句添加或删除成员属性或方法。您可以更改对象类型的现有属性（ FINAL或INSTANTIABLE ），并可以修改该类型的标量属性。

您还可以使用此语句重新编译类型的规范或正文，或通过添加新的对象成员子程序规范来更改对象类型的规范。

先决条件

对象类型必须位于您自己的架构中，并且您必须具有CREATE TYPE或CREATE ANY TYPE系统特权，或者您必须具有ALTER ANY TYPE系统特权。

句法

alter_type :: =


说明alter_type.eps的描述

（ alter_type_clause ：有关此子句的语法，请参阅Oracle Database PL / SQL语言参考 。）

语义

模式

指定包含该类型的架构。如果省略schema ，则Oracle数据库假定该类型位于当前模式中。

TYPE_NAME

指定对象类型的名称，嵌套表类型或varray类型。

对 type_name的 限制

您无法改进已编辑的对象类型。如果满足以下任一条件，则ALTER TYPE语句将失败并显示ORA-22348：

  • 类型是已编辑的对象类型， ALTER TYPE语句没有type_compile_clause 。您可以使用ALTER TYPE语句重新编译已编辑的对象类型，但不能用于任何其他目的。

  • 该类型具有一个依赖项，它是一个已编辑的对象类型，而ALTER TYPE语句具有一个CASCADE子句。

有关type_compile_clause和CASCADE子句的更多信息，请参阅Oracle Database PL / SQL语言参考 。
