1.162 LICENSE_MAX_SESSIONS

LICENSE_MAX_SESSIONS指定允许的最大并发用户会话数。

属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

没有

价值范围

0到会话许可证的数量

基本

没有

Oracle RAC

多个实例可以具有不同的值，但是安装数据库的所有实例的总数应小于或等于为该数据库许可的会话总数。

注意：

Oracle不再按并发会话数提供许可。因此，不推荐使用LICENSE_MAX_SESSIONS和LICENSE_SESSIONS_WARNING初始化参数。

达到LICENSE_MAX_SESSIONS指定的限制时，只有具有RESTRICTED SESSION权限的用户才能连接到数据库。无法连接的用户会收到一条警告消息，指示系统已达到最大容量。

零值表示未强制执行并发使用（会话）许可。如果将此参数设置为非零数字，则可能还需要设置LICENSE_SESSIONS_WARNING （请参阅“ LICENSE_SESSIONS_WARNING ” ）。

不要同时启用并发使用许可和用户许可。将LICENSE_MAX_SESSIONS或LICENSE_MAX_USERS设置为零。
