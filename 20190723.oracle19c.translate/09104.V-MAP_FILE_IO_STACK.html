8.70 V $ MAP_FILE_IO_STACK

V$MAP_FILE_IO_STACK显示文件的存储容器的分层排列。视图中的每一行代表层次结构中的一个级别。

柱 数据类型 描述

FILE_MAP_IDX

数

文件索引（对应于V$MAP_FILE FILE_MAP_IDX ）

深度

数

I / O堆栈中的元素深度

ELEM_IDX

数

与元素对应的索引

CU_SIZE

数

连续驻留在元素上的文件的连续逻辑块集（以HKB为单位）

STRIDE

数

文件中与此元素相邻的连续单元（CU）之间的HKB数。用于RAID5和条带文件。

NUM_CU

数

此元素上由文件中的STRIDE HKB分隔的相邻单元的数量。在RAID5中，连续单元的数量还包括奇偶校验条带。

ELEM_OFFSET

数

HKB单位的元素抵消

FILE_OFFSET

数

从文件开头到连续单元的第一个字节的HKB单位的偏移量

数据类型

VARCHAR2（15）

数据类型：

  • 数据

  • 平价

  • 数据和奇偶校验

PARITY_POS

数

平价的位置。仅适用于RAID5。需要此列来区分奇偶校验和数据部分。

PARITY_PERIOD

数

平价期。仅适用于RAID5。

ID

数

唯一标识符

PARENT_ID

数

父标识符

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
