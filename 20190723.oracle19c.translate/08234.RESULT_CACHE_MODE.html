1.293 RESULT_CACHE_MODE

RESULT_CACHE_MODE指定将ResultCache运算符拼接到查询的执行计划中的时间。

属性 描述

参数类型

串

句法

RESULT_CACHE_MODE = {手动| FORCE}

默认值

手册

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

值

  • 手册

    仅在查询注释（即提示）时才添加ResultCache运算符。

  • 力

    ResultCache运算符被添加到所有SELECT语句的根（当它有效时）。

    注意：

    建议不要使用FORCE模式，因为数据库和客户端将尝试缓存所有查询，这可能会产生显着的性能和锁存开销。此外，由于调用非确定性PL / SQL函数的查询也会被缓存，因此以这种基于广泛的方式启用结果缓存可能会导致对结果进行重大更改。

对于FORCE设置，如果语句包含NO_RESULT_CACHE提示，则提示优先于参数设置。
