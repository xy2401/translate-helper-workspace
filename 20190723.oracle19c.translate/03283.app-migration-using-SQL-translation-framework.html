5示例：使用SQL Translation Framework进行应用程序迁移

请考虑迁移Sybase JDBC应用程序的示例，以及迁移报告中包含的信息：如何使用它来调整迁移以获得最佳结果。

迁移Sybase JDBC应用程序

图5-1说明了用于查询Sybase数据库的编码应用程序如何使用SQL Translation Framework来查询存储在Oracle数据库中的信息。

图5-1针对Oracle数据库运行的Sybase应用程序


“图5-1针对Oracle数据库运行的Sybase应用程序”的说明

应用概述

此示例中使用的Sybase数据库有三个表和五个过程，包括以下功能：

  • IDENTITY列

  • INSERT语句INSERT到具有IDENTITY列的表中

  • VARCHAR列，大小超过4000字符

  • 从过程返回的多个隐式结果集

Java应用程序使用JDBC连接到此Sybase数据库。

设置迁移

迁移过程分为四个阶段 - 捕获，转换，生成和数据移动。最佳做法是完成迁移过程的每个阶段，查看“摘要”页面上的任何问题，然后继续下一阶段。通过迁移向导，您可以依次完成每个步骤，然后返回到向导以完成更多步骤。为此，在完成每个阶段后，选中“ 继续到摘要页面”复选框，然后单击“ 下一步” 。

执行以下步骤以设置迁移：

  1. 下载JDBC驱动程序JTDS 1.2。

  2. 将JTDS添加为第三方JDBC驱动程序，如下所示：

    1. 从工具菜单中选择首选项 。

    2. 从右侧面板的Database选项中选择Third Party JDBC Driver ， 如图5-2所示。

      图5-2设置JTDS JDBC驱动程序

  3. 单击添加条目 。

    将显示“选择路径条目”框。

  4. 选择jtds-1.2.jar文件，然后单击Select 。

  5. 单击确定 。

  6. 连接到要迁移信息的Oracle数据库。

  7. 使用以下命令验证连接是否正在使用Oracle Database 12c JDBC驱动程序：

    显示jdbc

  8. 使用以下命令在Oracle数据库中为迁移存储库创建新用户migrep ：

    GRANT CONNECT，RESOURCE，创建视图到migrep INDENTIFIED BY migrep; ALTER USER migrep QUOTA UNLIMITED对用户;

  9. 以migrep用户身份连接到数据库，并将迁移存储库与用户关联， 如图5-3所示。

    图5-3将用户与迁移存储库关联

  10. 创建与Sybase数据库的连接，在本例中为simpledemo12c ， 如图5-4所示。

    图5-4创建与Sybase数据库的连接

捕获阶段在此时保存所选数据库的快照。仅捕获对象定义，而不捕获实际的表数据。可以在“迁移项目”导航器中查看此捕获的快照。

请注意，快照不是与数据库的连接，它只允许您浏览迁移存储库中保存的信息。

设置迁移首选项

在开始转换阶段之前，您必须设置迁移首选项。执行以下步骤以实现此目的：
  1. 从“工具”菜单中，依次选择“首选项” ，“ 迁移 ”和“ 转换器” 。选择Generate Compound Triggers选项。

    图5-11设置迁移首选项

  2. 从Tools菜单中，选择Preferences ，然后选择Migration ，然后选择Generation Options 。在“迁移”选项中选择“ 使用所有Oracle Database 12c功能” 。

    图5-12设置迁移首选项

转换迁移

执行以下步骤以开始迁移过程的转换阶段：
  1. 右键单击Capture Model节点，选择Convert ， 如图5-13所示。

    图5-13启动迁移过程的转换阶段

    迁移向导在转换阶段打开， 如图5-14所示。

    图5-14转换迁移的数据

  2. 选择继续到摘要页面，然后单击下一步 。
  3. 单击完成 。

在转换阶段，对象名称将解析为有效的Oracle名称。数据类型转换为Oracle数据库类型，T-SQL定义的对象（如存储过程，视图等）将转换为Oracle PL / SQL。创建转换后的模型，可以在“迁移项目”导航器中进行浏览。转换后的程序可以在转换后的模型中查看。

请注意，转换后的模型不是实际的Oracle数据库，而是Oracle数据库的原型。该信息仍存储在Migration Repository表中。

生成迁移

迁移生成阶段在目标Oracle数据库中创建对象。将创建一个脚本，并通过以下两种方式针对选定的Oracle连接运行该脚本：

  • 在offline模式下，脚本在SQL工作表中打开，您必须选择连接并手动运行它。

  • 在online模式下，必须在向导中提供目标连接，向导将自动运行脚本。

以下步骤演示了如何在offline模式下执行迁移过程的生成阶段：

  1. 在“迁移项目”面板中右键单击“ 转换的数据库对象” ，然后选择“ 生成目标” 。
  2. 在迁移向导中选择offline作为数据库模式， 如图5-15所示。

    单击下一步 。

    图5-15选择数据库模式

  3. 在目标Oracle数据库中选择一个连接， 如图5-16所示。

    图5-16为目标用户创建Oracle数据库连接dbo_simpledemo12c

    不在此步骤中选择的连接下创建数据库对象。但是，此连接必须具有足够的权限才能创建其他用户和对象。

移动数据

执行以下步骤将数据移动到Oracle数据库：

  1. 右键单击“ 转换的数据库对象”节点，选择“ 移动数据” ， 如图5-18所示。

    单击下一步 。

    图5-18将数据从Sybase数据库移动到Oracle数据库

  2. 在“移动数据”屏幕中选择online作为数据移动模式。

    如果迁移过程涉及大量数据，则可以选择offline作为数据移动模式。

  3. 单击下一步 。将出现“摘要”屏幕。
  4. 单击完成 。

    您可以浏览数据库对象以验证数据是否已移至Oracle数据库。

生成迁移报告

Oracle SQL Developer提供了许多有关迁移过程的报告，以帮助确定要解决的任务和问题。在“迁移项目”导航器中单击或双击迁移的项目。右侧面板上将显示一个报告，其中包含多个选项卡和子报告， 如图5-19所示。

图5-19生成迁移报告

Analysis报告提供有关已迁移数据库大小的信息，如对象数，行大小等， 如图5-20所示。

图5-20迁移分析报告

“目标状态”报告提供有关目标数据库中已迁移对象的状态的信息。首先，选择具有足够权限的目标连接以查看其他模式对象的状态，然后选择刷新 。转换后的模型中存在但目标Oracle数据库中缺少的对象被列为缺失对象。这些对象可以是有效的也可以是无效的。

图5-21目标状态报告

“数据质量”选项卡提供有关目标Oracle数据库与源数据库相比的行数的信息。执行以下步骤来比较数据库：

  1. 选择已转换的模型，源连接和目标连接。
  2. 单击“ 分析” 。
  3. 单击刷新 。

    这将对源和目标数据库中的每个表执行count(*)函数。因此，建议不要对生产数据执行此操作。
