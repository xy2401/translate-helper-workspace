从PGQL结果集创建顶点/边集

您可以从给定的PGQL结果集创建顶点或边集。在以下示例中，PGQL查询选择具有大于24的属性年龄值的所有顶点，然后getVertices()调用创建包含此类顶点的顶点集。 

resultSet = g.queryPgql（“SELECT x MATCH（x）WHERE x.age> 24”）vertexSet = g.getVertices（new ResultSetVertexFilter（resultSet，“x”））

当在客户端 - 服务器模式下运行时，此操作在PGX服务器上执行，并且无需将结果集提取到客户端以创建集合。这为任何大小的结果集带来了性能优势，并且能够处理在客户端计算机上处理可能很麻烦的非常大的结果集。

按名称共享图表

图表可以通过名称共享，允许您将图表标记为共享，并从图形名称的任何会话中检索它们。您可以通过调用graph.publish()将图表标记为与其他会话共享，其他会话可以通过session.getGraph("<name>")获取对已发布图表的引用。您可以通过调用session.getGraphs()来查看已发布的图表。共享图的所有者可以通过调用graph.rename("<name>")来修改其名称。

除了图形之外，还可以通过调用property.publish()来共享图形的各个属性。

此功能的一个含义是，所有图形名称现在必须在会话中全局唯一。

在PATH子句中使用子查询

您可以在PATH定义的WHERE子句中表示子查询。以下示例定义了一个以不是图中最旧的顶点结尾的路径：

PATH p AS（a） - >（b）WHERE EXISTS（SELECT * MATCH（x）WHERE x.age> b.age）SELECT ...

当前实现的一个限制是在WHERE子句下只允许简单约束（最多访问一个模式变量的约束）。这将不支持查询是：

PATH p AS（a） - >（b）在哪里存在（选择*匹配（a） - >（c） - >（b））SELECT ...

版本19c的其他新增和更改的PGQL功能

其他值得注意的新功能和更改功能包括以下内容。

  • 引入了用于指定输入图的FROM子句。
  • FROM子句在PgxGraph#queryPgql(String) ， PgxGraph#preparePgql(String)和PgxGraph#explainPgql(String)是可选的。
  • FROM子句在PgxSession#queryPgql(String) ， PgxSession#preparePgql(String)和PgxSession#explainPgql(String) 。
  • 使用OraclePgqlStatement和OraclePgqlPreparedStatement对Oracle数据库执行PGQL查询时，不支持FROM子句。
  • PGQL 1.0的WHERE子句已被拆分为MATCH子句和WHERE子句。
  • MATCH子句包含图形模式（顶点，边和路径），而PGQL 1.1中的WHERE子句仅包含过滤谓词。
  • 公共路径表达式现在有一个可选的WHERE子句，用于指定过滤器表达式（例如， PATH connects_to AS () -[e]-> () WHERE e.status = 'OPEN' SELECT * MATCH ... ）。
  • 已删除通过WITH ...的内联过滤器谓词。
  • OO样式的函数调用语法（例如， x.label()或x.id()已被函数式语法（例如， label(x)或id(x) ）替换。
  • OraclePgqlResultSet现在扩展了oracle.pgql.lang.ResultSet因此可以使用通用接口处理内存分析器和Oracle数据库的PGQL结果。

OraclePgqlResultSet的弃用

从19c开始，是oracle.pg.rdbms.不推荐使用OraclePgqlResultSet接口，可能会在将来的发行版中将其删除。而是使用标准化的接口oracle.pgql.lang.ResultSet从PGQL结果集中检索值。

这两个接口之间的一个区别是oracle.pgql.lang.ResultSet不提供API来检索顶点和边缘对象。使用这些接口的现有代码应更改为项目ID，而不是OracleVertex和OracleEdge对象。

例如，更改投影顶点和边的代码片段：

OraclePropertyGraph opg = ...; OraclePgqlStatement ops = OraclePgqlExecutionFactory.createStatement（opg）; OraclePgqlResultSet rs = ops.executeQuery（“SELECT v，e MATCH（v） -  [e]  - >（）WHERE ...”，“”）; Iterator <OraclePgqlResult> itr = rs.getResults（）。iterator（）; OraclePgqlResult opr = itr.next（）; OracleVertex v = opr.getVertex（1）; OracleEdge e = opr.getEdge（2）;

以下内容：

OraclePropertyGraph opg = ...; OraclePgqlStatement ops = OraclePgqlExecutionFactory.createStatement（opg）; ResultSet rs = ops.executeQuery（“SELECT id（v）AS vid，id（e）AS eid MATCH（v） -  [e]  - >（）WHERE ...”，“”）; rs.next（）; OracleVertex v = opg.getVertex（rs.getLong（“vid”））; OracleEdge e = opg.getEdge（rs.getLong（“eid”））;

仅使用ResultSet API将确保在删除OraclePgqlResultSet时应用程序不会中断。
