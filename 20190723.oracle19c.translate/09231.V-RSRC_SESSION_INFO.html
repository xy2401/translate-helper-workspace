8.194 V $ RSRC_SESSION_INFO

V$RSRC_SESSION_INFO显示每个会话的资源管理器统计信息。

柱 数据类型 描述

SID

数

会话标识符

CURRENT_CONSUMER_GROUP_ID

数

会话当前所属的使用者组的对象ID;如果会话尚未登录，则为NULL

CURRENT_CONSUMER_GROUP

VARCHAR2（32）

会话当前所属的使用者组的名称

ORIG_CONSUMER_GROUP_ID

数

消费者组映射所在的消费者组的对象ID;如果会话尚未登录，则为NULL

该组可能不是当前组，因为当前计划中的SWITCH_GROUP指令可能已更改会话的当前组。该组可能不等于MAPPED_CONSUMER_GROUP因为MAPPED_CONSUMER_GROUP可能不是当前计划的一部分。

MAPPING_ATTRIBUTE

VARCHAR2（32）

用于将会话映射到其原始使用者组的会话属性;如果未使用映射，则为NULL

另请参阅： “ DBA_RSRC_GROUP_MAPPINGS ”以获取更多详细信息

MAPPED_CONSUMER_GROUP

VARCHAR2（32）

会话最初映射到的消费者组;如果未使用映射，则为NULL

这可能与原始使用者组不对应，因为映射的组可能不是当前计划的一部分。

另请参阅： “ DBA_RSRC_GROUP_MAPPINGS ”以获取更多详细信息

州

VARCHAR2（32）

会议的当前状态：

  • NOT MANAGED - 会话尚未登录，或者当前的Resource Manager计划不需要在此时管理会话

  • RUNNING - 会话当前正在CPU上运行

  • WAITING FOR CPU - 会话已准备好运行。它正在等待CPU量程运行。

  • QUEUED - 会话排队，因为达到了活动会话限制

  • IDLE - 会话空闲

  • IDLE BLKR - 会话空闲并阻止另一个会话

  • WAITING - 会话目前正处于等待状态

    另请参阅：等待类型的“ V $ SESSION_WAIT ”

  • WAITING_FOR_IO - 会话正在等待提交I / O请求

  • UNBOUND - 会话不受任何进程的约束

  • EXITING - 会话即将终止

活性

VARCHAR2（5）

指示会话当前是否处于活动状态（ TRUE ）（ FALSE ）。这包括满足以下条件之一时：

  • 会话是最重要的

  • 会话正在进行中

  • 会话正在使用临时空间对象

  • 会话保持用户入队

CURRENT_IDLE_TIME

数

在此使用者组中，会话空闲的状态数（状态为IDLE或IDLE BLKR ）;如果当前资源管理器计划不需要更新此统计信息，则为NULL。会话变为活动状态时，此值将重置为零。

CURRENT_CPU_WAIT_TIME

数

由于当前使用者组中的资源管理（状态为WAIT FOR CPU ），会话等待CPU的时间（以毫秒为单位）;如果当前资源管理器计划不需要更新此统计信息，则为NULL。这不包括由于锁存或入队争用，I / O等待等等。如果使用SWITCH_TIME_IN_CALL ，则会在每次调用结束时重置。

CPU_WAIT_TIME

数

由于资源管理，会话等待CPU（在其生命周期内）的累计时间量（以毫秒为单位）。这不包括由于锁存或入队争用，I / O等待等等。

CURRENT_CPU_WAITS

数

由于此消费者组中的资源管理，会话必须等待CPU的次数;如果当前资源管理器计划不需要更新此统计信息，则为NULL。这不包括由于锁存或入队争用，I / O等待等等。如果使用SWITCH_TIME_IN_CALL ，则会在每次调用结束时重置。

CPU_WAITS

数

由于资源管理，会话必须等待CPU（在其生命周期内）的累计次数。这不包括由于锁存或入队争用，I / O等待等等。

CURRENT_CONSUMED_CPU_TIME

数

当前使用者组中会话消耗的CPU时间（以毫秒为单位）;如果当前资源管理器计划不需要更新此统计信息，则为NULL。如果使用SWITCH_TIME_IN_CALL ，则会在每次调用结束时重置。

CONSUMED_CPU_TIME

数

会话消耗的累计CPU时间（在其生命周期内）（以毫秒为单位）

CURRENT_ACTIVE_TIME

数

会话在当前使用者组中处于活动状态的时间量（以毫秒为单位）;如果当前资源管理器计划不需要更新此统计信息，则为NULL。活动时间是在执行呼叫时运行和等待所花费的时间。它不包括会话等待CPU资源的时间。如果使用SWITCH_TIME_IN_CALL ，则在每次调用结束时重置。

ACTIVE_TIME

数

会话消耗的累计活动时间（以毫秒为单位）（在其生命周期内）

CURRENT_QUEUED_TIME

数

来自会话的当前请求已排队的时间量（以毫秒为单位）（状态为QUEUED ）。如果会话当前没有排队的请求，则此数字将为零。

QUEUED_TIME

数

会话在QUEUED状态（其生命周期）中花费的总时间（以毫秒为单位）

CURRENT_YIELDS

数

在当前使用者组中，会话必须将CPU产生到其他会话（由于量子到期）的次数;如果当前资源管理器计划不需要更新此统计信息，则为NULL。如果使用SWITCH_TIME_IN_CALL ，则在每次调用结束时重置。

产额

数

由于量子到期（在其生命周期内）会话必须为其他会话产生CPU的累积次数

CURRENT_UNDO_CONSUMPTION

数

会话消耗的当前量（以KB为单位）;如果当前资源管理器计划没有UNDO_POOL指令， UNDO_POOL

MAX_UNDO_CONSUMPTION

数

会话生存期内消耗的最大撤消量（以KB为单位）。此值可能不会更新，因为当前的资源管理器计划可能没有UNDO_POOL指令。

SQL_CANCELED

数

由于超出资源管理器计划的SWITCH_TIME限制而取消会话中运行的SQL查询的SWITCH_TIME 。

QUEUE_TIME_OUTS

数

会话请求超时的次数，因为它们的排队时间超过了资源管理器计划的限制

ESTIMATED_EXECUTION_LIMIT_HIT

数

由于优化程序执行查询的估计时间超过MAX_EST_EXEC_TIME限制，因此未运行会话请求的次数

CURRENT_IO_SERVICE_TIME

数

当前SQL操作的当前I / O等待时间（以毫秒为单位）

IO_SERVICE_TIME

数

会话的累计I / O等待时间（在其生命周期内）（以毫秒为单位）

CURRENT_IO_SERVICE_WAITS

数

当前I / O按会话等待当前SQL操作

IO_SERVICE_WAITS

数

累积I / O按会话等待（在其生命周期内）

CURRENT_SMALL_READ_MEGABYTES

数

当前SQL操作的会话读取的单个块兆字节数

SMALL_READ_MEGABYTES

数

会话读取的单个块兆字节总数（在其生命周期内）

CURRENT_LARGE_READ_MEGABYTES

数

会话为当前SQL操作读取的多块兆字节数

LARGE_READ_MEGABYTES

数

会话读取的多块兆字节总数（在其生命周期内）

CURRENT_SMALL_WRITE_MEGABYTES

数

会话为当前SQL操作写入的单个块兆字节数

SMALL_WRITE_MEGABYTES

数

会话写入的单个块兆字节总数（在其生命周期内）

CURRENT_LARGE_WRITE_MEGABYTES

数

会话为当前SQL操作写入的多块兆字节数

LARGE_WRITE_MEGABYTES

数

会话写入的多块兆字节总数（在其生命周期内）

CURRENT_SMALL_READ_REQUESTS

数

当前SQL操作的会话的单个块读取请求数

SMALL_READ_REQUESTS

数

会话的单个块读取请求总数（在其生命周期内）

CURRENT_SMALL_WRITE_REQUESTS

数

当前SQL操作的会话的单个块写入请求数

SMALL_WRITE_REQUESTS

数

会话的单个块写入请求总数（在其生命周期内）

CURRENT_LARGE_READ_REQUESTS

数

当前SQL操作的会话的多块读取请求数

LARGE_READ_REQUESTS

数

会话的多块读取请求总数（在其生命周期内）

CURRENT_LARGE_WRITE_REQUESTS

数

会话对当前SQL操作的多块写入请求数

LARGE_WRITE_REQUESTS

数

会话的多块写入请求总数（在其生命周期内）

CURRENT_PQ_ACTIVE_TIME

数

当前活动并行语句为当前SQL操作执行的时间量，不包括语句排队的时间量（以毫秒为单位）。如果并行语句排队，则值为0。

PQ_ACTIVE_TIME

数

并行语句在会话生命周期内执行的累计时间（以毫秒为单位）

DOP

数

活动或排队并行语句的并行度，如果会话中有任何并行度

PQ_SERVERS

数

如果会话处于活动状态并运行并行查询，则活动并行服务器的数量。如果查询排队，则显示此查询尝试运行的并行服务器数。

ESTIMATED_EXECUTION_TIME

数

并行语句的估计执行时间，由优化程序估计（以毫秒为单位）。您可以将此值与CURRENT_PQ_ACTIVE_TIME进行比较，以估计并行语句将运行多长时间。

CURRENT_PQ_QUEUED_TIME

数

当前并行语句在当前SQL操作中排队的时间（以毫秒为单位）。如果会话没有排队的并行语句，则该值为0。

PQ_QUEUED_TIME

数

会话在其生命周期中在PQ_QUEUED状态下花费的PQ_QUEUED时间（以毫秒为单位）

PQ_QUEUED

数

会话中并行语句排队的次数

PQ_QUEUE_TIME_OUTS

数

会话中并行语句因其队列时间超出资源管理器计划的PARALLEL_QUEUE_TIMEOUT限制而超时的次数

PQ_ACTIVE

VARCHAR2（5）

指示会话是否正在主动运行并行语句（ TRUE ）（ FALSE ）。

PQ_STATUS

VARCHAR2（44）

在此会话中运行的并行语句的状态：

  • Active ：并行语句当前处于活动状态并正在运行

  • Queued ：并行语句排队，不是下一个可能的候选者

  • Queue head ：并行语句由于全局系统范围限制而排队，是下一个要出列并运行的并行语句

  • Queue head - waiting on CG limit ：并行语句由于使用者组限制而排队，并且是下一个从会话的使用者组运行的语句

  • Queue head - waiting on service availability ：并行语句排队，因为没有足够的并行服务器可用于运行此会话的服务。此会话是会话服务的下一个会话

  • Queue head - waiting on PDB limit ：并行语句由于PDB限制而排队，并且是从会话的PDB下一个要运行的语句

  • NULL ：会话当前不涉及并行执行

CURRENT_LOGICAL_IOS

数

当前SQL操作的会话的逻辑I / O请求数

LOGICAL_IOS

数

此会话生命周期中的逻辑I / O请求总数

CURRENT_ELAPSED_TIME

数

会话当前SQL操作的经过时间

ELAPSED_TIME

数

此会话在其生命周期中的所有SQL操作所用的总时间

LAST_ACTION

VARCHAR2（48）

资源管理器对此SQL操作执行的最新操作。其值为以下之一：

  • CANCEL_SQL

  • KILL_SESSION

  • LOG_ONLY

  • 切换到<CG NAME>

对于最后一个值，<CG NAME>是SQL操作切换到的使用者组的名称。如果资源计划已被更改，则<CG NAME>是消费者组的ID。

LAST_ACTION_REASON

VARCHAR2（30）

资源管理器对此SQL操作执行的最新操作的原因。其值为以下之一：

  • SWITCH_CPU_TIME

  • SWITCH_IO_REQS

  • SWITCH_IO_MBS

  • SWITCH_ELAPSED_TIME

  • SWITCH_IO_LOGICAL

LAST_ACTION_TIME

日期

资源管理器对此SQL操作执行的最近操作的时间

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
