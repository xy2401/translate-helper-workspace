3.7重置数据库

目的

使用 RESET DATABASE TO INCARNATION命令将RMAN存储库中目标数据库的化身重置为先前的数据库化身。您只需在以下方案中使用此命令：

  • 在当前RESETLOGS时间戳之前，使用RESTORE或RECOVER将数据库返回到SCN。

  • 您使用FLASHBACK DATABASE将数据库回滚到孤立的数据库化身。

也可以看看：

“Oracle数据库备份和恢复用户指南” ，了解有必要使用RESET DATABASE命令的情况

先决条件

仅在RMAN提示符下执行RESET DATABASE TO INCARNATION 。RMAN必须连接到目标数据库。

如果RMAN以NOCATALOG模式运行，则必须装入目标数据库。装入的控制文件必须包含指定数据库化身的记录。

如果RMAN在CATALOG模式下运行，则可以装入或卸载目标数据库。如果装入了数据库，则控制文件必须包含指定数据库化身的记录。

使用说明

当您在使用RMAN NOCATALOG模式下， RESET DATABASE TO INCARNATION命令是跨RMAN会话持久性。

CDB的使用说明

可插拔数据库（PDB）的初始化身编号为0。后续的化身编号是唯一的，但并不总是顺序的。PDB化身是多租户容器数据库（CDB）的子化身，表示为( database_incarnation, pdb_incarnation ) 。例如，如果CDB处于化身5 ，并且PDB处于化身3 ，那么PDB的完全指定的化身编号是(5, 3) 5,3 (5, 3) 。

语义

语法元素 描述

数据库

指定整个数据库。

首要的关键

将当前的化身更改为由数据库化身的DBINC记录的主键指定的非当前数据库化身。化身键用于唯一地标记和标识重做流。

运行LIST INCARNATION OF DATABASE以获取可能的键值。发出RESET DATABASE TO INCARNATION ，可以运行RMAN命令，如FLASHBACK DATABASE ， RESTORE和RECOVER 。
