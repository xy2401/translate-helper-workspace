CP

目的

使您可以在Oracle ASM磁盘组之间以及磁盘组和操作系统之间复制文件。

语法和说明

cp src_file [--target target_type] [--service service_name] [ -  port port_num] [ connect_str ：] tgt_file

下表包含cp命令的语法选项。

表10-45 cp命令的选项

选项 描述

--target target_type

指定ASMCMD必须连接到的实例的目标类型以进行复制操作。有效选项是ASM ， IOS或APX 。

--service service_name

如果不是默认的+ ASM，则指定Oracle ASM实例名称。

--port port_num

指定侦听器端口号。默认值为1521。

connect_str

指定远程实例的连接字符串。

SRC_FILE

要复制的源文件的名称。

tgt_file

创建的目标文件名或别名目录名的用户别名。

cp无法在两个远程实例之间复制文件。本地Oracle ASM实例必须是操作的源或目标。

您可以使用cp命令：

  • 将文件从磁盘组复制到操作系统

  • 将文件从磁盘组复制到磁盘组

  • 将文件从操作系统复制到磁盘组

某些文件类型不能是cp命令的源或目标。这些文件类型包括OCR和SPFILE文件类型。要备份，复制或移动Oracle ASM SPFILE，请使用spbackup ， spcopy或spmove命令。要复制OCR备份文件类型，源必须来自磁盘组。

如果文件存在于Oracle ASM磁盘组中，则复制文件的格式可在Little-Endian和Big-Endian系统之间移植。编写文件时，Oracle ASM会自动转换格式。要从Oracle ASM磁盘组复制非Oracle ASM文件，您可以将该文件复制到另一个endian平台，然后使用一个常用的实用程序来转换该文件。

本地实例副本不需要connect_str ，这是默认情况。对于远程实例副本，必须在非回显提示中指定连接字符串和Oracle ASM提示输入密码。connect_str的格式如下：

user @ host .SID

connect_str参数中需要user ， host和SID 。默认端口号为1521 ，可以使用--port选项进行更改。

连接权限（ SYSASM或SYSDBA ）由启动ASMCMD时使用--privilege选项确定。

src_file必须是完全限定的文件名或Oracle ASM别名。

在ASMCMD复制过程中，Oracle ASM服务器以以下格式创建Oracle托管文件（OMF）文件：

diskgroup / db_unique_name / file_type / file_name .# .＃

其中db_uniq_name设置为ASM ， #是一串数字。在复制过程中，ASMCMD cp为目标创建目录结构，并从该位置创建用户别名到创建的实际OMF文件。

例子

以下是cp命令的示例。第一个示例显示data磁盘组中的文件副本到操作系统上的文件。第二个示例显示操作系统上的文件副本到data磁盘组。第三个示例显示如何将data磁盘组中的文件复制到远程服务器。系统会提示您输入密码。

目的

表10-46列出了du命令的语法选项。

表10-46 du命令的选项

选项 描述

R M

目的

删除指定的Oracle ASM文件和目录。

语法和说明

rm [-f | -r] pattern [ pattern ...]

表10-52列出了rm命令的语法选项。

表10-52 rm命令的选项

选项 描述

-r

递归删除文件和子目录。

-F

删除文件和子目录，而不提示确认。

图案

文件，目录或通配符模式的名称。

如果pattern是文件或别名，则rm命令只有在当前未使用时才能删除该文件或别名。如果pattern是一个目录，那么rm命令只有在它为空时才会删除它（除非使用了-r标志）并且它不是系统生成的目录。如果pattern是别名，则rm命令将删除别名和别名引用的文件。要仅删除别名并保留别名引用的文件，请使用rmalias命令。

注意：

删除系统创建的目录中的所有文件时，将删除该目录。如果父目录为空，则还会删除所有父目录。

pattern可以包含通配符。请参阅“ 通配符 ” 。

如果使用通配符，则rm命令将删除除非空目录之外的所有匹配项，除非使用-r标志。要递归删除，请使用-r标志。使用-r选项，您可以删除非空目录，包括其中的所有文件和目录以及其下的整个目录树。如果使用-r标志或通配符，则rm命令会在继续之前提示您确认删除，除非指定-f标志。如果通配符与别名或具有别名的系统生成的文件匹配，则别名和它引用的系统生成的文件都将被删除。使用-r标志删除与通配符模式匹配的别名时，别名或具有别名的系统生成的文件必须存在于运行rm命令的目录中。

例如，如果您有一个用户别名， +data/dir1/file.alias指向+data/orcl/DATAFILE/System.256.146589651 ，则运行rm -r +data/dir1命令将删除+data/dir1/file.alias和+data/orcl/DATAFILE/System.256.146589651 。

例

以下是rm命令的示例。第一个示例删除myexamples.bak文件。第二个示例删除subdir2目录及其内容。
