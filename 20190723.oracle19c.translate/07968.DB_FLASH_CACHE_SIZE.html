1.81 DB_FLASH_CACHE_SIZE

DB_FLASH_CACHE_SIZE指定数据库智能闪存缓存（闪存缓存）的大小。此参数只能在实例启动时指定。

属性 描述

参数类型

大整数

句法

DB_FLASH_CACHE_SIZE = 整数 [K | M | G] [, 整数 [K | M | G]]...

默认值

0

可修改

改变系统

可在PDB中修改

没有

价值范围

最低： 0

最大值：取决于操作系统

基本

没有

对于使用DB_FLASH_CACHE_FILE指定的每个闪存设备，最多可以指定16个文件大小。例如，如果有三个闪存原始设备，则可以按如下方式指定每个设备的大小：

db_flash_cache_file = / dev / raw / sda，/ dev / raw / sdb，/ dev / raw / sdc db_flash_cache_size = 32G，32G，64G

如果闪存缓存由一个闪存缓存设备组成，则可以在数据库启动后为该闪存缓存设备（禁用闪存缓存）动态更改此参数为0 。然后，您可以通过在启动数据库时将设备的此参数设置回原始值来重新启用闪存缓存。不支持动态调整DB_FLASH_CACHE_SIZE大小或重新启用不同大小的闪存缓存。

如果您的闪存缓存包含多个闪存缓存设备，则可以在数据库启动后将特定闪存缓存设备的参数动态更改为0 （将其关闭）。然后，您可以通过将设备的此参数设置回数据库启动时的原始值（重新打开）来重新启用该闪存缓存设备。

例如，要关闭/dev/raw/sdb闪存缓存设备：

db_flash_cache_file = / dev / raw / sda，/ dev / raw / sdb，/ dev / raw / sdc db_flash_cache_size = 32G，0,64G

并且，要重新打开/dev/raw/sdb闪存缓存设备：

db_flash_cache_file = / dev / raw / sda，/ dev / raw / sdb，/ dev / raw / sdc db_flash_cache_size = 32G，32G，64G
