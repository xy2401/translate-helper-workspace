36.2.1 OCISodaBulkInsert（）

将一组文档插入到集合中。

目的

将文档数组插入集合中。

句法

剑OCISodaBulkInsert（OCISvcCtx * svchp，OCISodaColl *集合，OCISodaDoc ** documentarray，ub4 arraylen，OCISodaOutputOptions * opoptns，OCIError * errhp，ub4模式）;

参数

参数 描述
svchp（IN）

服务上下文。
集合（IN）

现有集合的集合句柄。
文档阵列（IN）

要插入到集合中的文档句柄数组。
阿拉伦（IN）

documentarray参数中提供的文档句柄数组的长度。

opoptns（OUT）
输出选项处理。有效属性是：

  • OCI_ATTR_SODA_DOC_COUNT. - 插入的文件数量。

注意 ：如果发生错误， OCI_ATTR_SODA_DOC_COUNT包含错误发生前插入的文档数。

errhp（OUT）

错误句柄。
模式（IN）
指定执行模式。有效模式是：

  • OCI_DEFAULT - 是默认模式。这意味着在没有特殊模式的情况下执行操作。

  • OCI_SODA_ATOMIC_COMMIT - 当操作在此模式下执行并且成功完成时，则在完成后提交当前事务。这将提交所有未完成的更改，而不仅仅是SODA操作所做的更改。如果操作失败，则仅回滚对SODA操作所做的更改;在调用SODA操作之前所做的任何未提交的更改都不会回滚。

返回

OCI_SUCCESS ，如果文档数组插入指定的集合成功。

OCI_ERROR ，如果文档阵列插入失败。OCIError参数具有必要的错误信息。

使用说明

没有。

例子

有关示例，请参阅使用SODA for C将文档插入集合

36.2.2 OCISodaBulkInsertAndGet（）

将一个文档数组插入一个集合中，并返回一个结果文档数组，每个文档包含除内容之外的所有文档组件。

目的

将文档数组插入集合，并返回结果文档数组，每个文档包含除内容之外的所有文档组件。在插入期间由SODA生成的组件（例如密钥（如果集合具有自动分配的密钥），最后修改的时间戳，创建的时间戳和版本）与每个结果文档一起返回。

句法

剑OCISodaBulkInsertAndGet（OCISvcCtx * svchp，OCISodaColl *集合，OCISodaDoc ** documentarray，ub4 arraylen，OCISodaOutputOptions * opoptns，OCIError * errhp，ub4模式）;

参数

参数 描述
svchp（IN）

服务上下文。
集合（IN）

现有集合的集合句柄。
documentarray（IN / OUT）

要插入到集合中的文档句柄数组。

注意 ：由于使用相同的参数返回结果文档，因此需要保存对每个输入文档句柄的引用，以便稍后可以使用OCIHandleFree()方法将其与每个结果文档句柄一起OCIHandleFree() 。如果不保存对每个输入文档句柄的引用，则会覆盖对结果文档的引用，并且永远不会正确释放输入文档句柄（导致内存泄漏）。

阿拉伦（IN）

documentarray参数中提供的文档句柄数组的长度。

opoptns（OUT）
输出选项处理。有效属性是：
  • OCI_ATTR_SODA_DOC_COUNT ：

    插入的文件数量。如果发生错误， OCI_ATTR_SODA_DOC_COUNT包含错误发生之前插入的文档数。

errhp（OUT）

错误句柄。
模式（IN）
指定执行模式。有效模式是：

  • OCI_DEFAULT - 是默认模式。这意味着在没有特殊模式的情况下执行操作。

  • OCI_SODA_ATOMIC_COMMIT - 当操作在此模式下执行并且成功完成时，则在完成后提交当前事务。这将提交所有未完成的更改，而不仅仅是SODA操作所做的更改。如果操作失败，则仅回滚对SODA操作所做的更改;在调用SODA操作之前所做的任何未提交的更改都不会回滚。

返回

OCI_SUCCESS ，如果文档数组插入指定的集合成功。

OCI_ERROR ，如果文档阵列插入失败。OCIError参数具有必要的错误信息。

使用说明

没有。

36.2.3 OCISodaBulkInsertAndGetWithCtnt（）

仅使用其内容将一个文档数组插入到集合中，并返回包含除内容之外的所有文档组件的结果文档数组。

目的

要仅使用其内容将文档数组插入集合中，并返回结果文档数组，每个文档包含除内容之外的所有文档组件。

注意：

这是一种仅在不构造文档句柄的情况下插入JSON文档的便捷方法。更通用的方法是OCISodaBulkInsertAndGet 。

句法

剑OCISodaBulkInsertAndGetWithCont（OCISvcCtx * svchp，OCISodaColl *集合，void ** contentStrings，ub4 * contentStringLengths，OraText **键，ub4 * keyLengths，ub4 arrayLength，ub4 docFlags，OCISodaDoc **文档，OCISodaOutputOptions * opoptns，OCIError * errhp ub4模式） ;

参数

参数 描述
svchp（IN）

服务上下文。
集合（IN）

现有集合的集合句柄。
contentStrings（IN）

包含文档的JSON内容的字符串数组。

注意 ：此方法仅适用于JSON文档。

contentStringLengths（IN）

一组内容字符串长度。
键（IN）

插入文档的键数组。如果集合具有客户端分配的密钥，则提供密钥。否则，设置为null。
keyLengths（IN）

一系列密钥长度。仅当集合具有客户端分配的密钥时才需要设置。否则，必须设置为null。
arrayLength（IN）

contentStrings和keys数组的长度。

docFlags（IN）
编码标志用于指示所提供文档内容的编码。有效标志选项包括：
  • OCI_DEFAULT - 如果您使用此参数值，那么您实际上声明文档内容位于由环境句柄定义的字符集中（或环境变量NLS_LANG ，如果没有为句柄设置）。如果给定文档不是这种情况，那么尝试编写文档的结果是不可预测的。

  • OCI_SODA_DETECT_JSON_ENC - 自动检测文档内容的编码为UTF-8，UTF-16 LE或UTF-16 BE。如果您使用此参数值，那么您实际上声明文档内容是UTF-8，UTF-16 LE或UTF-16 BE。如果给定文档不是这种情况，那么尝试编写文档的结果是不可预测的。

文件（OUT）

返回包含除内容之外的所有文档组件的结果文档数组。每个结果文档都包含组件，例如密钥（如果集合具有自动分配的密钥），最后修改的时间戳，创建的时间戳以及SODA插入命令生成的版本。
opoptns（OUT）
指定输出选项句柄。有效属性是：

  • OCI_ATTR_SODA_DOC_COUNT. - 插入的文件数量。

注意 ：如果发生错误， OCI_ATTR_SODA_DOC_COUNT包含错误发生前插入的文档数。

errhp（OUT）

错误句柄。
模式（IN）
指定执行模式。有效模式是：

  • OCI_DEFAULT - 是默认模式。这意味着在没有特殊模式的情况下执行操作。

  • OCI_SODA_ATOMIC_COMMIT - 当操作在此模式下执行并且成功完成时，则在完成后提交当前事务。这将提交所有未完成的更改，而不仅仅是SODA操作所做的更改。如果操作失败，则仅回滚对SODA操作所做的更改;在调用SODA操作之前所做的任何未提交的更改都不会回滚。

返回

OCI_SUCCESS ，如果文档插入指定的集合是成功的。

OCI_ERROR ，如果文档插入失败。OCIError参数具有必要的错误信息。

使用说明

没有。

例子

有关示例，请参阅使用SODA for C将文档插入集合 。

  • OCI_DEFAULT - 是默认模式。这意味着在没有特殊模式的情况下执行操作。

返回

OCI_SUCCESS ，如果密钥设置成功。

使用说明

没有。
