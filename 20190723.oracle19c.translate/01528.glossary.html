词汇表

活动数据库重复

通过网络创建的重复数据库 ，无需还原目标数据库的备份。此技术是基于备份的复制的替代方法。

祖先的化身

父化身是数据库化身 ， 当前化身从OPEN RESETLOGS操作后分支出来。父母化身的父母是祖先的化身。任何祖先化身的父母也是祖先的化身。

应用容器

插入应用程序根目录的一组命名的应用程序PDB 。

申请PDB

插入应用程序容器的 PDB。

应用程序根

应用程序容器中的根容器 。每个应用程序容器只有一个应用程序根。应用程序根目录可以包含公共对象，并使用CREATE PLUGGABLE DATABASE语句CREATE PLUGGABLE DATABASE 。

档案备份

一种免于正常备份和恢复策略的数据库备份。通常，这些备份存档在单独的存储介质上并保留很长时间。

存档的重做日志

当数据库处于ARCHIVELOG模式时，在线重做日志组的已填充成员的副本。在LGWR进程使用重做记录填充每个联机重做日志后，归档进程会将日志复制到一个或多个重做日志归档目标。此副本是存档的重做日志。RMAN不区分原始归档重做日志和归档重做日志的映像副本;两者都被视为图像副本。

归档重做日志删除策略

可配置的持久性RMAN策略，用于管理何时可以删除存档的重做日志。您可以使用CONFIGURE ARCHIVELOG DELETION POLICY命令配置策略。

归档重做日志故障转移

RMAN功能，即使某些归档日志目标缺少日志或具有损坏块的日志，RMAN也可以完成备份。例如，如果您备份RMAN确定已损坏的快速恢复区域中的日志，则RMAN可以搜索其他归档位置中的日志，如果它们完好无损，则可以将其备份。

ARCHIVELOG模式

Oracle数据库将填充的联机重做日志复制到磁盘的数据库模式。在创建数据库时或使用ALTER DATABASE ARCHIVELOG语句指定模式。

另请参阅： 归档重做日志 ， NOARCHIVELOG模式

归档

将填充的联机重做日志文件复制到脱机日志归档目标的操作。联机重做日志的脱机副本称为归档重做日志 。您必须以ARCHIVELOG模式运行数据库以存档重做日志。

异步I / O.

当RMAN正在读取或写入数据时，服务器进程可以在等待I / O完成时开始I / O然后执行其他工作。在等待第一个I / O完成之前，RMAN还可以开始多个I / O操作。

自动频道分配

RMAN无需使用ALLOCATE CHANNNEL命令即可执行备份和还原任务的能力。您可以使用CONFIGURE命令指定磁盘和磁带通道。然后，您可以在RMAN命令提示符下发出BACKUP和RESTORE命令，而无需手动分配通道。RMAN使用执行命令所需的任何已配置通道。

自动诊断存储库（ADR）

系统管理的存储库，用于存储和组织数据库跟踪文件和其他诊断数据。ADR提供了数据库遇到的所有严重错误的全面视图，并维护问题诊断及其最终解决所需的所有相关数据。存储库包含描述事件，跟踪，转储，警报消息，数据修复记录， 数据完整性检查记录，SQL跟踪信息，核心转储等的数据。

初始化参数DIAGNOSTIC_DEST指定ADR基址的位置，该目录包含一个或多个ADR主目录。产品或产品实例使用每个ADR主目录将诊断数据存储在明确定义的子目录中。例如，Oracle数据库实例的诊断数据存储在其ADR主目录中，其中包括警报消息的alert子目录，跟踪文件的trace子目录等。查找跟踪文件和警报日志的最简单方法是运行以下SQL查询： SELECT NAME, VALUE FROM V$DIAG_INFO 。

自动存储管理（ASM）

垂直集成文件系统和专为Oracle数据库文件构建的卷管理器。ASM将存储设备整合到易于管理的磁盘组中，并提供镜像和条带化等优势，而无需第三方逻辑卷管理器。

自动撤消管理模式

数据库的一种模式，其中撤消数据存储在专用的撤消表空间中 。您必须执行的唯一撤消管理是创建撤消表空间。所有其他撤消管理都是自动执行的。

辅助渠道

连接到辅助实例的RMAN 通道 。使用ALLOCATE CHANNEL或CONFIGURE CHANNEL命令的AUXILIARY关键字指定AUXILIARY CONFIGURE CHANNEL 。

辅助数据库

（1）使用RMAN DUPLICATE命令从目标数据库备份创建的数据库。

（2）临时数据库，在表空间时间点恢复（TSPITR）期间恢复到新位置，然后以新实例名称启动。TSPITR辅助数据库包含恢复集和辅助集 。

辅助目的地

在可传输的表空间操作中，可以存储磁盘上的位置，其中可以存储辅助设置文件（例如参数文件），数据文件（正在传输的表空间的数据文件除外），控制文件和辅助实例的联机重做日志。

辅助实例

与辅助数据库关联的Oracle实例，或表空间时间点恢复（TSPITR）或可传输表空间操作中使用的临时实例。

辅助集

在TSPITR中，不在恢复集中但必须在辅助数据库中恢复以使TSPITR操作成功的文件集。在可传输的表空间操作中，辅助集包括表空间传输所需的数据文件和其他文件，但这些文件本身不是恢复集的一部分。

备用

（1）数据的备份副本 ，即数据库，表空间，表，数据文件，控制文件或归档重做日志。备份可以是物理的（在数据库文件级别）或逻辑的（在数据库对象级别）。可以使用RMAN备份一个或多个数据文件，控制文件或归档重做日志文件来创建物理备份。您可以使用Data Pump Export创建逻辑备份。

（2）在RMAN上下文中， BACKUP命令的输出。备份的输出格式可以是备份集 ， 代理副本或映像副本 。由数据库归档的日志被视为副本而不是备份。

备份和恢复

保护数据库免受因介质故障或用户错误导致的数据丢失所涉及的一组概念，过程和策略。

基于备份的复制

通过还原和恢复目标数据库的备份创建的重复 数据库 。此技术是活动数据库复制的替代方法。

备份控制文件

控制文件的备份。您可以备份控制文件与RMAN backup命令或SQL语句ALTER DATABASE BACKUP CONTROLFILE TO ' filename ' 。

备份加密

使用V$RMAN_ENCRYPTION_ALGORITHMS列出的算法对备份集进行加密。 RMAN可以透明地加密写入备份集的数据，并在RESTORE操作中需要时解密这些备份集。RMAN提供三种加密模式：透明，密码保护和双模式。

备份模式

发出ALTER TABLESPACE时启动的数据库模式（也称为热备份模式 ） ... 在进行在线备份之前， BEGIN BACKUP或ALTER DATABASE BEGIN BACKUP命令。当您发出ALTER TABLESPACE时，您将表空间从备份模式中删除... END BACKUP或ALTER DATABASE END BACKUP命令。

在联机表空间中进行数据文件的用户管理备份时，必须将表空间置于备份模式以防止出现块断裂的可能性。在备份模式下，对数据库的更新会产生超过通常的重做次数。每次缓冲区高速缓存中的块变脏时，除了记录对数据的更改外，数据库还必须将已更改块的映像写入重做日志文件。RMAN 不要求你把数据库备份模式。

另见： 腐败块

备份优化

一种配置，使RMAN能够自动跳过已备份的文件的备份。使用CONFIGURE命令启用和禁用备份优化。

备用件

用于存储RMAN 备份集的物理文件格式。每个逻辑备份集包含一个或多个物理备份片。

备份保留策略

用户定义的策略，用于确定必须保留备份和归档日志的时间以进行介质恢复。您可以根据备份冗余或恢复窗口定义保留策略。RMAN保留满足当前保留策略所需的数据文件备份，以及完全恢复这些数据文件备份所需的任何归档重做日志。

备份集

备份一个或多个数据文件，控制文件，服务器参数文件和归档重做日志文件。每个备份集由一个或多个二进制文件组成。每个二进制文件称为备份文件。备份片段以专有格式编写，只能由RMAN创建或恢复。

备份集由RMAN BACKUP命令生成。备份集通常只包含一个备份片。仅当使用ALLOCATE CHANNEL或CONFIGURE CHANNEL命令的MAXPIECESIZE选项限制备份块大小时， MAXPIECESIZE才会在多个备份块之间划分备份集的内容。

另请参见： 未使用的块压缩 ， 多路复用 ， RMAN

备份撤消优化

排除撤消不需要恢复RMAN备份，因为它描述并包含已提交的事务。例如，用户更新USERS表空间中的salaries表。更改将写入USERS表空间，而数据的前映像将写入UNDO表空间。UNDO表空间的后续RMAN备份可能不包括工资更改的撤消。备份撤消优化是内置的RMAN行为，无法禁用。

备份窗口

备份活动必须完成的一段时间。

基础恢复目录

整个恢复目录架构。基本恢复目录与虚拟专用目录不同 ， 虚拟专用目录是恢复目录的子集。

二进制压缩

RMAN将压缩算法应用于备份集中的数据的技术。

阻止变更跟踪

一种数据库选项，可使Oracle数据库跟踪受每个数据库更新影响的数据文件块。跟踪信息存储在块改变跟踪文件中。启用块更改跟踪时，RMAN使用更改跟踪文件中已更改块的记录，通过仅读取已知已更改的块而不是完整读取数据文件来提高增量备份性能。

阻止更改跟踪文件

RMAN用于记录更改的块以提高增量备份性能的二进制文件。使用ALTER DATABASE语句创建并重命名此文件。

阻止媒体恢复

使用Recovery Manager RECOVER ...恢复数据文件中的指定块RECOVER ...BLOCK命令。阻止介质恢复使受影响的数据文件保持联机状态，并仅恢复和恢复损坏或损坏的块。

打破镜子

终止磁盘镜像过程，以便镜像不再保持最新状态。

CDB

包含至少一个PDB的Oracle数据库安装。 CDB还包含一个根和一个种子。每个Oracle数据库都是CDB 或非CDB 。

CDB还原点

多租户容器数据库（CDB）中 SCN的别名或时间点。CDB中的所有PDB都可以访问CDB还原点。

渠道

RMAN通道表示与备份设备之间的一个数据流。通道可以是DISK通道（用于执行磁盘I / O）或SBT通道（用于通过第三方介质管理软件执行I / O）。每个分配的通道启动一个新的Oracle数据库会话。然后，会话将执行备份，还原和恢复操作。

渠道并行

为RMAN操作分配多个通道。

检查站

一种数据结构，用于在数据库的重做线程中定义SCN。检查点记录在控制文件和每个数据文件头中，是恢复的关键要素。

校验

数据库根据存储在数据或重做块中的所有字节计算的数字。如果启用了DB_BLOCK_CHECKSUM初始化参数，则数据库将计算每个数据文件或联机重做日志块的校验和，并在写入磁盘时将其存储在块头中。数据库可以使用校验和值来检查一致性。

循环再利用记录

控制文件记录，包含RMAN用于备份和恢复操作的信息。这些记录按逻辑环排列。当所有可用记录槽已满时，Oracle会扩展控制文件以为新记录腾出空间或覆盖最旧记录。CONTROL_FILE_RECORD_KEEP_TIME初始化参数控制在覆盖记录之前必须保留多少天。CONTROL_FILE_RECORD_KEEP_TIME的默认值为7天。

另请参见： 非圆形重用记录

关闭备份

数据库关闭时拍摄的一个或多个数据库文件的备份。通常，封闭备份是整个数据库备份。如果一致地关闭数据库，则备份中的所有文件都是一致的。否则，备份不一致。

普通用户

未使用的块压缩

RMAN通过跳过数据块来减小数据文件备份集大小的功能。RMAN总是跳过从未使用过的块。在Oracle数据库备份和恢复参考中的BACKUP AS BACKUPSET条目中描述的某些条件下，RMAN还会跳过以前使用的当前未使用的块。

用户管理的备份

使用非RMAN方法进行的备份，例如，使用操作系统实用程序。例如，您可以通过在Linux上运行cp命令或在Windows上运行COPY命令来进行用户管理的备份。用户管理的备份也称为操作系统备份 。

用户管理的备份和恢复

不使用RMAN的Oracle数据库的备份和恢复策略。该术语相当于操作系统备份和恢复 。您可以使用操作系统实用程序（例如，UNIX中的cp命令）备份和还原数据库文件，并使用SQL * Plus RECOVER命令进行恢复。

验证

在RMAN上下文中，检查数据库文件是否存在块损坏或检查备份集以确定是否可以还原的测试。RMAN可以检查物理和逻辑块损坏。

虚拟私人目录

基本恢复目录中的元数据的子集，数据库用户被授予该访问权限。基本恢复目录的所有者可以向其他数据库用户授予或撤消对恢复目录的受限访问权限。每个受限用户都具有对其自己的虚拟专用目录的完全读/写访问权限。

卷影复制服务（VSS）

Windows服务器平台上的基础结构，使请求者，编写者和提供者能够参与创建称为卷影副本的一致快照。VSS服务使用定义良好的COM接口。请参阅适用于Microsoft Windows的Oracle数据库平台指南，以了解如何将RMAN与VSS一起使用。

整个数据库备份

控制文件的备份以及属于数据库的所有数据文件。

零数据丢失恢复设备（恢复设备）

企业级云计算工程系统，为所有Oracle数据库的备份提供单一的集中存储库。您可以使用RMAN命令将目标数据库备份到恢复设备。
