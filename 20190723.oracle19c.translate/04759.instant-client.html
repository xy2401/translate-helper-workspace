2.1关于Oracle Instant Client

Oracle Instant Client提供必要的Oracle数据库客户端文件，以创建和运行OCI，OCCI，ODBC和JDBC OCI应用程序。还提供有用的命令行实用程序，包括SQL * Plus，SQL * Loader和Oracle Data Pump。Oracle Instant Client消除了客户端计算机上对Oracle主目录的需求，从而简化了应用程序的部署。

与在完整客户端安装中运行的同一应用程序相比，在Instant Client模式下运行的OCI应用程序的存储空间要求显着降低。最小安装的Instant Client共享库可占用完整客户端安装的不到10％的空间。最后，Instant Client易于安装。

为何使用Instant Client？

  • 安装涉及解压缩一些文件或安装RPM软件包。

  • Oracle数据库客户端所需文件数量和总磁盘存储量显着减少。

  • 使用Instant Client部署的应用程序不会丧失功能或性能。

  • 独立软件供应商很容易打包应用程序。

Instant Client安装中包含README文件。它描述了生成Instant Client的版本，日期和时间以及操作系统。

默认情况下，OCI应用程序在运行时库搜索路径中查找OCI数据共享库libociei.so （或Windows上的Oraociei18.1.dll ）（例如Linux上的LD_LIBRARY_PATH或Windows上的PATH ）以确定是否应用程序应该在Instant Client模式下运行。如果找不到此库，则OCI会尝试加载Instant Client Light数据共享库（请参阅表2-3和表2-4 ）， libociicus.so （或Windows上的Oraociicus18.1.dll ）。如果找到Instant Client Light库，则应用程序将在Instant Client Light模式下运行。否则，将假定基于Oracle主目录的完整安装。

2.2从ZIP文件，RPM和Oracle Universal Installer安装

可以通过多种方式安装Oracle Instant Client。

可以通过从Oracle Universal Installer中选择Instant Client选项，或者从Oracle Technology Network网站上的OCI页面下载并安装ZIP文件或RPM来安装Instant Client库： Oracle Instant Client

从Oracle技术网站下载并安装Instant Client库

以下步骤适用于Linux，Unix和Windows环境。

  1. 从Oracle技术网 ，下载并将所需的ZIP包从OTN解压缩到一个空目录，例如instantclient_18_1 ，用于Oracle数据库版本18c，版本18.1。所有Instant Client ZIP安装都需要Basic或Basic Light软件包。

  2. 或者，要安装RPM文件，请按照以下说明操作：

    1. 通过运行su并输入超级用户密码成为root用户。

    2. 切换到要安装的目录。

    3. 卸载任何早期安装。

      #rpm -e package-name

    4. 用yum安装每个包。Yum将自动安装所需的依赖项，例如libaio 。 

      #yum remove package-name #yum install package-name.rpm

      或者，使用rpm安装每个包。

      #rpm -ivh package-name .rpm

    5. 如果要节省磁盘空间，请删除.rpm文件。

    6. 退出root shell。无需重启。

  3. 将操作系统共享库路径环境变量设置为步骤1中的目录。例如，对于Linux或UNIX上的ZIP文件，请将LD_LIBRARY_PATH设置为instantclient_18_1 。
    export LD_LIBRARY_PATH = / opt / oracle / instantclient_18_1：$ LD_LIBRARY_PATH
    使用Linux RPMs，将LD_LIBRARY_PATH设置为：
    export LD_LIBRARY_PATH = / usr / lib / oracle / 18.1 / client64 / lib：$ LD_LIBRARY_PATH

    在Windows上，将PATH设置为instantclient_18_1目录。例如，在Windows 7上，更新控制面板中的PATH - >系统 - >高级系统设置 - >高级 - >环境变量 - >系统变量 - > PATH。

  4. 如有必要，请设置NLS_LANG环境变量以指定应用程序打开的客户端应用程序和数据库连接使用的语言和区域，以及客户端的字符集，即客户端程序输入或显示的数据的字符集。NLS_LANG在UNIX平台上设置为环境变量，并在Windows平台上的注册表中设置。有关设置NLS_LANG环境变量的更多信息，请参见“ Oracle数据库全球化支持指南” 。

  5. 如果您打算将可选的Oracle客户端配置文件（如tnsnames.ora ， sqlnet.ora ， ldap.ora或oraaccess.xml与Instant Client放在一起，请将它们放在Instant Client的network/admin子目录中。这是与Instant Client链接的应用程序的默认Oracle客户端配置目录。或者，Oracle客户端配置文件可以放在另一个可访问的目录中。然后将环境变量TNS_ADMIN设置为该目录名称。

完成上述步骤后，您就可以运行OCI应用程序了。

当可以通过操作系统Library Path变量访问OCI共享库时，OCI应用程序使用Instant Client。通过这种方式，不依赖于Oracle主目录，并且OCI不需要Oracle主目录中提供的任何其他代码和数据文件。

从Oracle Universal Installer安装Instant Client

对于Oracle Universal Installer ，请调用Oracle Universal Installer并选择Instant Client选项，并将Instant Client共享库安装到Oracle数据库版本18c版本18.1的空目录，例如instantclient_18_1 。

请参阅从Oracle技术网网站下载和安装Instant Client库中有关设置环境变量的步骤3到步骤5。

如果您完成了完整的客户端安装（通过选择Oracle Universal Installer中的Admin选项），则完整客户端安装中Instant Client共享库的位置为：

在Linux或UNIX上：

libociei.so库位于$ORACLE_HOME/instantclient

libclntsh.so.18.1 ， libclntshcore.so.18.1和libnnz18.so位于$ORACLE_HOME/lib

在Windows上：

oraociei18.dll库位于ORACLE_HOME\instantclient

oci.dll ， ociw32.dll和orannzsbb18.dll在ORACLE_HOME\bin

要使用Instant Client启用运行OCI应用程序，请将前面的库复制到其他目录，并设置操作系统共享库路径以找到此目录。

注意：

必须从同一Oracle主目录复制所有库，并且必须将它们放在同一目录中。符号链接到Instant Client库的协同定位不能替代库的物理共址。

操作系统Library Path变量上应该只有一组Oracle库。也就是说，如果您有多个包含Instant Client库的目录，则操作系统库路径中只应有一个此类目录。

同样，如果在同一系统上执行Oracle基于主目录的安装，则不应在操作系统库路径上同时具有ORACLE_HOME/lib和Instant Client目录，而不管它们在库路径中的显示顺序如何。也就是说， ORACLE_HOME/lib目录（用于非Instant Client操作）或Instant Client目录（用于Instant Client操作）应位于操作系统Library Path变量上，但不能同时位于两者上。

要启用OCCI和JDBC OCI等其他功能，您必须复制一些其他文件。要启用OCCI，您必须安装OCCI库（ libocci.so.18.1在Linux或UNIX和oraocci18.dll的即时客户端目录在Windows上）。对于JDBC OCI驱动程序，除了三个OCI共享库，还必须下载OCI JDBC库（例如libocijdbc18.so在Linux或UNIX和ocijdbc18.dll在Windows上）。将所有库放在Instant Client目录中。

注意：

在Sparc64等混合平台上，要在Instant Client模式下操作JDBC OCI驱动程序，请将libociei.so库从ORACLE_HOME/instantclient32目录复制到Instant Client目录。将JDBC OCI Instant Client所需的所有其他Sparc64库从ORACLE_HOME/lib32目录复制到Instant Client目录。

2.3 Oracle Instant Client的环境变量

ORACLE_HOME环境变量不再确定NLS，CORE和错误消息文件的位置。

仅OCI应用程序不应要求设置ORACLE_HOME 。但是，如果设置，则不会影响OCI。 OCI始终从数据共享库获取其数据。如果数据共享库不可用，则仅使用ORACLE_HOME并假定ORACLE_HOME安装。虽然不需要设置ORACLE_HOME ，但如果设置了ORACLE_HOME ，则必须将其设置为标识目录的有效操作系统路径名。

如果要加载动态用户回调库，则如本指南所指定，回调包必须驻留在ORACLE_HOME/lib （Windows上的ORACLE_HOME\bin ）中。在这种情况下设置ORACLE_HOME 。

Instant Client模式ORA_NLS_PROFILE33忽略环境变量ORA_NLS10和ORA_NLS_PROFILE33 。

Oracle数据库12开始c解除2（12.2），使用即时客户端库时，即时客户端可以使用ORA_TZFILE环境变量时该环境变量被设置为从文件系统读取时区的文件。否则，如果未设置ORA_TZFILE变量，则使用数据共享库中较大的默认timezlrg_ n .dat文件。如果要从数据共享库中使用较小的timezone_ n .dat文件，则将ORA_TZFILE环境变量设置为文件名，而不包含任何绝对路径名或相对路径名。必须将该文件复制到（UNIX / LINUX）的子目录oracore/zoneinfo或即时客户端目录中的（Microsoft Windows）的oracore\zoneinfo 。您可以将genezi实用程序与-v选项一起使用，以验证时区文件和位置是否有效，或运行SQL * Plus。

在Linux或UNIX上：

export ORA_TZFILE = timezone_ n .dat

在Windows上：

设置ORA_TZFILE = timezone_ n .dat

在这些示例中， n是时区数据文件版本号。

确保为（UNIX / LINUX）或子目录oracore\zoneinfo创建子目录oracore/zoneinfo for（Microsoft Windows） oracoei或oraociicus DLL所在的位置（通常位于instantclient_12_2目录中）并将timezone_ n .dat文件复制到此目录。

要确定打包在Instant Client数据共享库中的小型和大型时区文件的版本，请输入以下命令以运行genezi实用程序：

genezi -v

如果OCI未在Instant Client模式下运行（因为数据共享库不可用），则ORA_TZFILE变量（如果已设置）将命名完整路径名，就像在以前的Oracle数据库发行版中一样。

如果使用TNSNAMES本地命名参数，那么，如前所述， TNS_ADMIN目录必须包含TNSNAMES配置文件。如果未设置TNS_ADMIN ，则ORACLE_HOME/network/admin目录必须包含Oracle Net Services配置文件。

2.5适用于Oracle Instant Client的SDK

软件开发工具包（SDK）是一组开发工具，可以从Oracle Technology Network网站上的Instant Client链接URL下载创建应用程序：

Oracle Instant Client

  • Instant Client SDK包具有C和C ++头文件以及用于在Instant Client环境中开发OCI和OCCI应用程序的makefile。开发的应用程序可以部署在任何客户端环境中。

  • SDK包含C和C ++演示程序。

2.6关于Oracle Instant Client Light

Instant Client Light版本的Instant Client进一步减少了客户端安装的磁盘空间要求。

通过删除除英语以外的语言的错误消息文件并且仅保留约250个支持的字符集定义，可以减小库的大小。

此Instant Client Light版本适用于使用US7ASCII，WE8DEC，WE8ISO8859P1，WE8MSWIN1252或Unicode字符集的应用程序。NLS_LANG设置的LANGUAGE和TERRITORY字段没有限制，因此Instant Client Light可以使用任何语言和区域设置进行操作。由于Instant Client Light仅提供英语错误消息，因此即使NLS_LANG设置为AMERICAN以外的语言，客户端生成的错误消息（如网络连接错误）也始终以英语报告。如果在数据库实例的Oracle主目录中安装了相应的已翻译消息文件，则数据库端生成的错误消息（如SQL语句中的语法错误）采用所选语言。

2.6.3安装Oracle Instant Client Light

如何安装Instant Client Light。

Instant Client Light可以通过以下方式之一安装：

  • 安装在Oracle技术网（OTN）上

    以下URL是Oracle Technology Network网站上的Instant Client页面： Oracle Instant Client

    对于Instant Client Light，请将basiclite.zip包下载并解压缩到空的instantclient_12_2目录中。

  • 从Instant Client管理员安装中安装

    从ORACLE_HOME/instantclient/light子目录中，复制libociicus.so （或Windows上的oraociicus12.dll ）。LD_LIBRARY_PATH上的Instant Client目录（Windows上的PATH ）应包含Instant Client Light数据共享库libociicus.so （Windows上的oraociicus12.dll ），而不是更大的OCI Instant Client数据共享库libociei.so （ oraociei12.dll在Windows上的oraociei12.dll ）。

  • 从Oracle Universal Installer安装中安装

    当您从Oracle Universal Installer中选择Instant Client选项时， libociei.so （或Windows上的oraociei12.dll ）安装在安装的基本目录中，这意味着这些文件放在LD_LIBRARY_PATH （Windows上的PATH ）上。

    Instant Light客户端数据共享库libociicus.so （或Windows上的oraociicus12.dll ）安装在基本目录的light子目录中，默认情况下不启用。因此，要在Instant Client Light模式下运行，必须删除或重命名OCI数据共享库libociei.so （或Windows上的Oraociei12.dll ），并且必须将Instant Client Light库从light子目录复制到基本目录的安装。

    例如，如果Oracle Universal Installer已在LD_LIBRARY_PATH （Windows上为PATH ）的my_oraic_12_2目录中安装了Instant Client，则使用以下命令序列在Instant Client Light模式下运行：

    cd my_oraic_12_2 rm libociei.so mv light / libociicus.so。

    注意：

    要确保安装中不存在不兼容的二进制文件，请始终将Instant Client文件复制并安装到空目录中。

2.7关于在Linux或UNIX上修补Oracle Instant Client共享库

由于Instant Client是部署功能，因此运行OCI应用程序所需的文件数量和大小（客户端占用空间）已减少。

因此，在Instant Client部署中无法使用修补Instant Client共享库所需的所有文件。修补需要基于Oracle Home的完整客户端安装。使用opatch实用程序进行修补。

在Oracle主环境中应用修补程序后，将关于Oracle Instant Client中列出的文件复制到即时客户端目录，如从ZIP文件安装，从RPM和Oracle Universal Installer中所述 。

您可以为OCI和OCCI，JDBC和SQL * Plus生成Instant Client zip和RPM文件，而不是复制单个文件，如数据共享库和Zip和RPM文件的再生中所述。然后，您可以将zip和RPM文件复制到目标系统并按照从ZIP文件安装，从RPM和Oracle Universal Installer中的说明解压缩它们。

opatch实用程序将ORACLE_HOME安装的修补信息存储在libclntsh.so 。可以通过以下命令检索此信息：

genezi -v

如果Instant Client部署系统没有genezi实用程序，则可以从ORACLE_HOME /bin 目录中复制它。

注意：

opatch实用程序在Windows上不可用。

2.8数据共享库和Zip和RPM文件的重新生成

Oracle Database 12 c第1版（12.1）中更改了重新生成数据共享库以及zip和RPM文件的过程。

单独的make文件目标用于单独或全部创建数据共享库，zip和RPM文件。目前， ilibociei仅构建zip和RPM文件。数据共享库的重新生成需要编译器和链接器，这可能并非在所有安装中都可用。以下部分显示用于重新生成数据共享库，zip和RPM文件的make file目标。

注意：

重新生成的Instant Client二进制文件仅包含安装在Oracle Client Administrator主目录中的Instant Client文件，从中进行重新生成。因此，再生环境中存在的错误消息，字符集编码和时区文件是唯一打包在数据共享库中的文件。错误消息，字符集编码和时区文件取决于为安装Oracle客户端管理员主目录而选择的国家/地区语言。

Windows平台上不提供数据共享库以及zip和RPM文件的重新生成。

本节包括以下主题：

  • 重新生成数据共享库libociei.so

  • 重新生成数据共享库libociicus.so

  • 在一步中重新生成数据共享库libociei.so和libociicus.so

  • 为基本包重新生成Zip和RPM文件

  • 为基本光包重新生成Zip和RPM文件

  • 为JDBC包重新生成Zip和RPM文件

  • 重新生成ODBC包的Zip和RPM文件

  • 为SQL * Plus程序包重新生成Zip和RPM文件

  • 为工具包重新生成Zip和RPM文件

  • 为所有包重新生成Zip和RPM文件

2.8.1重新生成数据共享库libociei.so

可以使用ORACLE_HOME的管理员安装中的以下命令重新生成OCI Instant Client数据共享库（ libociei.so ）：

cd $ ORACLE_HOME / rdbms / lib make -f ins_rdbms.mk igenlibociei

新的重新生成的libociei.so放在ORACLE_HOME /instantclient目录中。位于同一目录中的原始现有libociei.so将重命名为libociei.so0 。

2.8.2重新生成数据共享库libociicus.so

要重新生成Instant Client Light数据共享库（ libociicus.so ），请使用以下命令：

mkdir -p $ ORACLE_HOME / rdbms / install / instantclient / light cd $ ORACLE_HOME / rdbms / lib make -f ins_rdbms.mk igenlibociicus

新重新生成的libociicus.so放在ORACLE_HOME / instantclient / light目录中。位于同一目录中的原始现有libociicus.so将重命名为libociicus.so0 。

2.8.3一步重新生成数据共享库libociei.so和libociicus.so

要重新生成数据共享库libociei.so和libociicus.so，请使用以下命令：

mkdir -p $ ORACLE_HOME / rdbms / install / instantclient / light cd $ ORACLE_HOME / rdbms / lib make -f ins_rdbms.mk igenliboci

新重新生成的libociei.so位于ORACLE_HOME / instantclient目录中。位于同一目录中的原始现有libociei.so将重命名为libociei.so0 。

新重新生成的libociicus.so放在ORACLE_HOME / instantclient / light目录中。位于同一目录中的原始现有libociicus.so将重命名为libociicus.so0 。

2.8.4为基本包重新生成Zip和RPM文件

要重新生成基本包的zip和RPM文件，请使用以下命令：

cd $ ORACLE_HOME / rdbms / lib make -f ins_rdbms.mk ic_basic_zip
