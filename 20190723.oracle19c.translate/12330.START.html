12.45开始

句法

STA [RT] { url | file_name [。 ext ]} [ arg ...]

运行指定的SQL * Plus语句脚本。可以从本地文件系统或Web服务器调用该脚本。

条款

网址

指定要在指定的Web服务器上运行的脚本的统一资源定位器。SQL * Plus支持HTTP和FTP协议，但不支持HTTPS。此版本不支持http：// username：password @ machine_name.domain ...形式的HTTP身份验证。 

file_name [。 分机 ]

该脚本你想执行。该文件可以包含可以交互运行的任何命令。

如果未指定扩展名，则SQL * Plus将采用默认的命令文件扩展名（通常为SQL）。看到 SET SUF [FIX] {SQL |文字}关于改变默认扩展的信息。

输入START file_name时 。 ext ，SQL * Plus搜索具有您在当前默认目录中指定的文件名和扩展名的文件。如果SQL * Plus找不到这样的文件，SQL * Plus将搜索与系统相关的路径以查找该文件。某些操作系统可能不支持路径搜索。有关操作系统环境的特定信息，请参阅为操作系统提供的特定于平台的Oracle文档。 

arg ...

您希望传递给参数的数据项在脚本中。 如果输入一个或多个参数，SQL * Plus会将值替换为参数脚本中的（＆1，＆2等）。第一个参数替换＆1的每个匹配项，第二个参数替换＆2的每个匹配项，依此类推。

START命令使用参数值定义参数;如果在此会话中再次启动脚本，则可以输入新参数或省略使用旧值的参数。

有关使用参数的详细信息，请参阅定义替换变量和使用替换变量 。

用法

以下所有设置（如COLUMN命令设置）在脚本启动时保持有效。如果脚本更改了任何设置，则此新值将在脚本完成后保持有效

@（在标志处） 和@@（双重标志） 命令功能类似于开始 。禁用产品用户配置文件中的START命令也会禁用@和@@命令。有关这些命令的更多信息，请参阅@（at sign）和@@（sign at sign） 。有关更多信息，请参阅禁用SQL * Plus_ SQL_和PL / SQL命令 。

该退出或脚本中的QUIT命令终止SQL * Plus。

例子

名为PROMOTE且扩展名为SQL的文件（用于提升员工）可能包含以下命令：

SELECT FIRST_NAME，LAST_NAME，JOB_ID，SALARYFROM EMP_DETAILS_VIEWWHERE JOB_ID ='＆1'AND SALARY>＆2;

要运行此脚本，请输入

START PROMOTE ST_MAN 7000

或者如果它位于Web服务器上，请在以下格式中输入命令：

START HTTP：// machine_name .domain： port /PROMOTE.SQL ST_MAN 7000

在哪里machine_name 。 domain必须由host.domain名称替换， port由脚本所在的Web服务器使用的端口号替换。

执行以下命令：

SELECT LAST_NAME，LAST_NAME来自EMP_DETAILS_VIEW WHERE JOB_ID ='ST_MAN'AND SALARY> 7000;

并显示结果。
