Oracle数据库安装的存储核对表

表1-5 Oracle数据库的存储核对表

校验 任务
Oracle软件的最小本地磁盘存储空间

对于Linux x86-64：
对于独立服务器安装，Oracle Grid Infrastructure至少为6.5 GB。
Oracle数据库企业版至少7.2 GB。
Oracle Database Standard Edition 2至少7.2 GB。

注意：

Oracle建议您分配大约100 GB的空间，以便在现有Oracle主目录之上应用任何将来的修补程序。有关特定的与修补程序相关的磁盘空间要求，请参阅修补程序文档。
选择数据库文件存储选项

确保您具有以下可用存储选项之一：

  • 文件系统安装在服务器上。Oracle建议您选择的文件系统与操作系统或Oracle软件使用的文件系统分开。选项包括以下内容：

    • 逻辑卷管理器（LVM）卷或RAID设备上的文件系统

    • 安装在经过认证的网络连接存储（NAS）设备上的网络文件系统（NFS）

  • Oracle自动存储管理（Oracle ASM）。

    Oracle ASM作为Oracle Grid Infrastructure安装的一部分安装。如果您计划使用Oracle ASM进行存储，则应在安装和创建数据库之前安装Oracle Grid Infrastructure。
确定您的恢复计划 如果要在安装期间启用恢复，请准备选择以下选项之一：

  • 文件系统 ：在安装期间在文件系统上配置快速恢复区域

  • Oracle自动存储管理 ：使用Oracle ASMCA配置快速恢复区磁盘组。

有关配置恢复的详细信息，请查看本文档的存储配置部分。
