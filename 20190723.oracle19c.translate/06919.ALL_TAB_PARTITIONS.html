3.111 ALL_TAB_PARTITIONS

ALL_TAB_PARTITIONS显示DBMS_STATS程序包为当前用户可访问的分区生成的分区级分区信息，分区存储参数和分区统计信息。

相关观点

  • DBA_TAB_PARTITIONS显示数据库中所有分区的此类信息。

  • USER_TAB_PARTITIONS显示当前用户拥有的所有分区对象的分区的此类信息。此视图不显示TABLE_OWNER列。

注意：

仅当您使用DBMS_STATS包收集有关表的统计信息时，才会填充标有星号（ * ）的列。

注意：

对于包含列描述中的双星号（ ** ）的以下列，以下情况如下：

该列可以显示有关段级属性（对于简单分区表）或元数据（对于复合分区表）的信息。在简单的分区表中，分区在物理上包含数据库中的数据（段）。在复合分区表中，分区是元数据，数据本身存储在子分区中。

SEGMENT_CREATED

VARCHAR2（4）

指示简单分区表的分区的实际段创建属性，或复合分区表的子分区的缺省值（如果指定），否则为NONE 。**

对于简单的分区表，此列指示是否创建了段（ YES ）或否（ NO ）。

对于复合分区表，此列指示是否显式指定了默认段创建属性。可能的值：

  • NONE - 在分区级别未指定默认段创建属性。此值仅对复合分区显示，并被视为未指定的值。

  • YES - 在分区级别明确指定了立即段创建，并将其用作其所有子分区的默认值。

  • NO - 延迟段创建在分区级别明确指定，并将用作其所有子分区的默认值。

分度

VARCHAR2（4）

指示简单分区表的分区的实际索引属性，或复合分区表的子分区的缺省值（如果指定），否则为NONE。**

可能的值：

  • NONE - 分区是复合分区，未指定默认设置用于索引。

    此值仅对复合分区显示，并被视为未指定的值。当用户向表中添加子分区时，由于未指定分区的默认值，因此ALL_PART_TABLES.DEF_INDEXING值将用于新创建的子分区。

  • ON - INDEXING已开启。

  • OFF - INDEXING关闭。

只读

VARCHAR2（4）

表示分区的默认设置：

  • YES ：分区的默认设置是只读的。

  • NO ：分区的默认设置是读/写。

  • NONE ：没有为分区指定默认设置。

  • 没有重复

  • 重复

CELLMEMORY

VARCHAR2（24）

存储单元闪存缓存中的列式压缩值。可能的值：

  • ENABLED ：Oracle Exadata Storage将自动决定是否以柱状形式进行缓存

  • DISABLED ：禁止Oracle Exadata存储以列式形式缓存

  • NO CACHECOMPRESS ：Oracle Exadata存储将以HCC格式缓存（无重新压缩）

  • FOR QUERY ：Oracle Exadata存储将以INMEMORY查询高格式重新压缩和缓存

  • FOR CAPACITY ：Oracle Exadata存储将以INMEMORY容量低格式重新压缩和缓存

此列适用于Oracle Exadata。

INMEMORY_SERVICE

VARCHAR2（12）

指示如何在各种实例上填充IM列存储。可能的值是：

  • DEFAULT ：在使用PARALLEL_INSTANCE_GROUP初始化参数指定的所有实例上填充数据。如果未设置该参数，则会在所有实例上填充数据。这是默认值。

  • NONE ：未在任何实例上填充数据。

  • ALL ：无论PARALLEL_INSTANCE_GROUP初始化参数的值如何，都会在所有实例上填充数据。

  • USER_DEFINED ：仅在用户指定的服务处于活动状态的实例上填充数据。与此对应的服务名称存储在INMEMORY_SERVICE_NAME列中。

INMEMORY_SERVICE_NAME

VARCHAR2（100）

指示应在其上填充IM列存储的服务的服务名称。仅当相应的INMEMORY_SERVICE为USER_DEFINED时，此列才具有值。在所有其他情况下，此列为空。
