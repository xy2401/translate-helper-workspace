关于Oracle安装所有者帐户

根据要用于安装的组和用户管理计划，为安装选择或创建Oracle安装所有者。

在以下情况下，您必须为安装创建软件所有者：

  • 如果Oracle软件所有者用户不存在;例如，如果这是系统上第一次安装Oracle软件。

  • 如果存在Oracle软件所有者用户，但您希望使用具有不同组成员身份的其他操作系统用户将Oracle Grid Infrastructure管理权限与Oracle数据库管理权限分开。

在Oracle文档中，创建仅拥有Oracle Grid Infrastructure软件安装的用户称为网格用户（ grid ）。该用户拥有Oracle Clusterware和Oracle自动存储管理二进制文件。创建为拥有所有Oracle安装或一个或多个Oracle数据库安装的用户称为Oracle用户（ oracle ）。您只能拥有一个Oracle Grid Infrastructure安装所有者，但您可以让不同的Oracle用户拥有不同的安装。

Oracle软件所有者必须将Oracle Inventory组作为其主要组，以便每个Oracle软件安装所有者都可以写入中央库存（oraInventory），以便正确设置OCR和Oracle Clusterware资源权限。数据库软件所有者还必须具有OSDBA组和（如果您创建它们）OSOPER，OSBACKUPDBA，OSDGDBA，OSRACDBA和OSKMDBA组作为辅助组。
