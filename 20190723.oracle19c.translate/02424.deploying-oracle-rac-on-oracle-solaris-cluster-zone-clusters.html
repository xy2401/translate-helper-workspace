C在Oracle Solaris Cluster区域群集上部署Oracle RAC

Oracle Solaris Cluster提供了创建高可用性区域群集的功能。在区域群集中安装Oracle Real Application Clusters（Oracle RAC）允许您拥有单独的数据库版本或同一数据库的单独部署（例如，一个用于生产，一个用于开发）。

本附录列出了Oracle Solaris Cluster区域群集中Oracle RAC部署的用例，还提供了指向部署任务的文档资源的链接。

  • 关于Oracle Solaris Cluster区域群集中的Oracle RAC部署
    区域群集由多个Oracle Solaris区域组成，每个区域位于各自独立的服务器上;组成群集的区域链接在一起成为一个虚拟群集。
  • Oracle Solaris Cluster区域群集中Oracle RAC部署的先决条件
    查看在Oracle Solaris Cluster区域群集中部署Oracle Real Application Clusters（Oracle RAC）的先决条件。
  • 在全局区域中部署Oracle RAC
    此部署方案由多个服务器组成，您可以在全局区域中安装Oracle Real Application Clusters（Oracle RAC）。
  • 在区域群集中部署Oracle RAC
    在此部署方案中，您可以在区域群集中安装Oracle Real Application Clusters（Oracle RAC）。区域群集是Oracle Solaris非全局区域的群集。
