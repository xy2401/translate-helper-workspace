149 DBMS_SFW_ACL_ADMIN

DBMS_SFW_ACL_ADMIN包提供用于管理和管理“数据库服务防火墙”功能的访问控制策略的接口。每个策略都由一个访问控制列表（ACL）表示，该列表包含允许访问特定数据库服务的主机。本地侦听器和服务器进程根据ACL验证所有入站客户端连接。

本章包含以下主题：

  • DBMS_SFW_ACL_ADMIN安全模型

  • DBMS_SFW_ACL_ADMIN操作说明

  • DBMS_SFW_ACL_ADMIN示例

  • DBMS_SFW_ACL_ADMIN子程序的摘要

149.1 DBMS_SFW_ACL_ADMIN安全模型

该软件包归DBSFWUSER架构所有。此程序包中的过程只能由DBSFWUSER用户运行。

149.2 DBMS_SFW_ACL_ADMIN操作说明

这些操作说明适用于DBMS_SFW_ACL_ADMIN 。

  • ACL包含“访问控制条目”的条目，称为“ace”。

  • 您可以使用IP_ADD_ACE过程和IP_ADD_PDB_ACE过程向ACL添加条目。调用这些过程后，调用COMMIT_ACL过程将更新的ACL发送给侦听器。同样，如果使用IP_REMOVE_ACE过程 ， IP_REMOVE_ACL过程 ， IP_REMOVE_PDB_ACE过程或IP_REMOVE_PDB_ACL过程从ACL中删除条目，则需要调用COMMIT_ACL过程以更新侦听器上的ACL。

  • 必须使用新的FIREWALL端点属性和新的LOCAL_REGISTRATION_ADDRESS_ listenerName参数启用访问控制。配置可以在listener.ora手动完成，也可以通过Server Control Utility（SRVCTL）完成。有关配置说明，请参阅“ Oracle数据库网络服务参考”和“ Oracle Real Application Clusters管理和部署指南” 。

  • 您可以查询IP_ACL表以显示已提交的ACL。但是要查看已发送到本地侦听器的ACL，您必须查询V$IP_ACL或GV$IP_ACL视图。IP_ACL表可以包含不在[G]V$IP_ACL的ACL，因为这些ACL的数据库服务在提交时未运行。当服务正在运行时，您可以再次调用COMMIT_ACL过程以将IP_ACL表中的已提交ACL发送到本地侦听器。

    在Oracle RAC环境中， GV$IP_ACL可用于查询数据库集群中的ACL， V$IP_ACL用于查询已连接实例中的ACL。

149.3 DBMS_SFW_ACL_ADMIN示例

这三个示例显示了如何使用DBMS_SFW_ACL_ADMIN来管理和管理访问控制策略。

以下示例将三个访问控制条目添加到ACL并提交它们。

##连接到DBSFWUSER SQL> connect dbsfwuser / Welcome1已连接。##为数据库服务创建ACL SVC1 SQL> exec dbms_sfw_acl_admin.ip_add_ace（'svc1'，'192.168.12.1'）; PL / SQL过程成功完成。SQL> exec dbms_sfw_acl_admin.ip_add_ace（'svc1'，'192.168.12.2'）; PL / SQL过程成功完成。SQL> exec dbms_sfw_acl_admin.ip_add_ace（'svc1'，'test02.example.com'）; PL / SQL过程成功完成。##将ACL提交到DB ACL表。##这会将运行服务的ACL发送到所有本地监听器SQL> exec dbms_sfw_acl_admin.commit_acl; PL / SQL过程成功完成。

以下示例检索从上一示例提交的ACL。

SQL> select * from ip_acl; SERVICE_NAME HOST ----------------------- -------------------“SVC1。EXAMPLE.COM“192.168.12.1”SVC1。EXAMPLE.COM“192.168.12.2”SVC1。EXAMPLE.COM“TEST02。EXAMPLE.COM ##查看发送到本地监听器的ACL ##注意：ACL仅发送到正在运行的服务SQL> select * from v $ ip_acl; SERVICE_NAME HOST CON_ID ---------------------- ----------------- -------- SVC1。EXAMPLE.COM 192.168.12.1 1 SVC1。EXAMPLE.COM 192.168.12.2 1 SVC1。EXAMPLE.COM TEST02。EXAMPLE.COM 1

以下示例使用各种主机格式添加可插拔数据库“PDB1”的访问控制条目。

SQL> exec dbms_sfw_acl_admin.ip_add_pdb_ace（'pdb1'，'192.168.12.3'）; PL / SQL过程成功完成。SQL> exec dbms_sfw_acl_admin.ip_add_pdb_ace（'pdb1'，'192.168.12.0/23'）; PL / SQL过程成功完成。SQL> exec dbms_sfw_acl_admin.ip_add_pdb_ace（'pdb1'，'192.168.12。*'）; PL / SQL过程成功完成。SQL> exec dbms_sfw_acl_admin.commit_acl; PL / SQL过程成功完成。

149.4 DBMS_SFW_ACL_ADMIN子程序摘要

该表列出了DBMS_SFW_ACL_ADMIN子程序并简要描述了它们。

表149-1 DBMS_SFW_ACL_ADMIN包子程序

子程序 描述

COMMIT_ACL过程

提交对ACL表的更改，并将更改传播到数据库实例的本地侦听器。

IP_ADD_ACE程序

将访问控制条目添加到数据库服务的ACL。

IP_ADD_PDB_ACE过程

为可插拔数据库（PDB）中的所有数据库服务的ACL添加访问控制条目。

IP_REMOVE_ACE程序

从ACL中删除数据库服务的条目。

IP_REMOVE_ACL程序

从数据库服务的ACL中删除所有条目。

IP_REMOVE_PDB_ACE过程

从可插拔数据库（PDB）中的所有数据库服务的ACL中删除访问控制条目。

IP_REMOVE_PDB_ACL过程

从可插拔数据库（PDB）中的所有数据库服务中删除ACL中的所有条目。

149.4.1 COMMIT_ACL过程

此过程提交对ACL表的更改。它还将更改传播到数据库实例的本地侦听器。

如果已更改数据库服务的访问条目，但在调用COMMIT_ACL过程时数据库服务未运行，则这些更改将提交到ACL表，但不会将它们发送到本地侦听器。要将条目发送到侦听器，请启动数据库服务，然后再次调用COMMIT_ACL过程。

操作成功完成后，此过程将返回。

句法

DBMS_SFW_ACL_ADMIN.COMMIT_ACL;

参数

没有

149.4.2 IP_ADD_ACE程序

此过程为数据库服务的ACL添加访问控制条目。

句法

DBMS_SFW_ACL_ADMIN.IP_ADD_ACE（p_service_name IN VARCHAR2，p_host IN VARCHAR2）;

参数

表149-2 IP_ADD_ACE过程参数

参数 描述

p_service_name

要为其添加访问控制条目的数据库服务的名称。

p_host

允许访问服务的客户端主机。该值可以是主机名，IPv4地址或IPv6地址。还允许使用IPv4和CIDR格式的通配符“*”。

149.4.3 IP_ADD_PDB_ACE过程

此过程为可插拔数据库（PDB）中的所有数据库服务的ACL添加访问控制条目。

句法

DBMS_SFW_ACL_ADMIN.IP_ADD_PDB_ACE（p_pdb_name IN VARCHAR2，p_host IN VARCHAR2）;

参数

表149-3 IP_ADD_PDB_ACE过程参数

参数 描述

p_pdb_name

PDB的名称。

p_host

允许访问PDB中的数据库服务的客户机的主机。该值可以是主机名，IPv4地址或IPv6地址。还允许使用IPv4和CIDR格式的通配符“*”。

149.4.4 IP_REMOVE_ACE过程

此过程从ACL中删除数据库服务的条目。

句法

DBMS_SFW_ACL_ADMIN.IP_REMOVE_ACE（p_service_name IN VARCHAR2，p_host IN VARCHAR2）;

参数

表149-4 IP_REMOVE_ACE过程参数

参数 描述

p_service_name

要从中删除访问控制条目的数据库服务的名称。

p_host

要从ACL中删除的主机。该值可以是主机名，IPv4地址或IPv6地址。还允许使用IPv4和CIDR格式的通配符“*”。

这必须与现有值完全匹配。您可以查询IP_ACL表以获取数据库服务的条目列表。

参数

表149-5 IP_REMOVE_ACL过程参数

要清除其ACL的数据库服务的名称。
