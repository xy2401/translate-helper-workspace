CRSCTL概述

CRSCTL是您与Oracle Clusterware之间的接口，用于解析和调用Oracle Clusterware API的Oracle Clusterware API。

CRSCTL提供了群集感知命令，您可以使用该命令在群集上执行检查，启动和停止操作。您可以从群集中另一个节点上的群集中的任何节点或群集中的所有节点上运行这些命令，具体取决于操作。

您可以使用CRSCTL命令在Oracle Clusterware上执行多个操作，例如：

  • 启动和停止Oracle Clusterware资源

  • 启用和禁用Oracle Clusterware守护程序

  • 检查群集的运行状况

  • 管理代表第三方应用程序的资源

  • 将智能平台管理接口（IPMI）与Oracle Clusterware集成，以提供故障隔离支持并确保集群完整性

  • 调试Oracle Clusterware组件

群集（群集感知）命令

您可以在一个节点上运行群集命令，以在群集中的另一个节点上执行操作。这些被称为远程操作。这简化了管理，例如，您不再需要登录每个节点来检查所有节点上的Oracle Clusterware的状态。

集群化命令完全独立于操作系统;它们依赖于OHASD（Oracle高可用性服务守护程序）。如果此守护程序正在运行，则可以执行远程操作，例如启动，停止和检查远程节点的状态。

群集命令包括以下内容：

  • crsctl检查集群

  • crsctl启动集群

  • crsctl停止集群

动态调试使用crsctl set log

设置Oracle Clusterware的日志级别。

句法

crsctl set log module_name “ component1 = level1 ，...”

使用说明

  • 您可以为以下模块的各种组件设置日志级别：

    • mdns ：组播域名服务器
    • gpnp ：网格即插即用服务
    • css ：群集同步服务
    • crf ：Cluster Health Monitor
    • crs ：群集就绪服务
    • ctss ：集群时间同步服务
    • evm ：活动经理
    • gipc ：网格进程间通信

    使用crsctl lsmodules命令获取每个模块的组件列表。

  • 您可以在用双引号（“”）括起的逗号分隔列表中指定多个组件名称/日志级别对。

  • 在Oracle Clusterware 11g第 2版（11.2.0.3）及更高版本中使用等号（=）分隔组件名称/日志级别对。以前的Oracle Clusterware版本使用冒号（:)。

  • log_level是1到5的数字，用于设置组件或资源的日志级别，其中1表示日志输出量最少，5表示最详细的日志输出。默认日志级别为2。

例

要为CRS模块的CRSRTI和CRSCOMM组件设置日志级别，请执行以下操作：

$ crsctl set log crs“CRSRTI = 1，CRSCOMM = 2”

启用Oracle Clusterware模块的调试
您可以通过设置环境变量或运行crsctl set log命令来启用Oracle Clusterware模块及其组件和资源的调试。

使用以下语法为各种Oracle Clusterware模块启用调试：

crsctl set {log | trace} module_name “ component ： debugging_level [，...]”

以root用户身份运行crsctl set命令，并提供以下信息：

  • module_name ：指定以下模块之一的名称：

    • mdns ：组播域名服务器
    • gpnp ：网格即插即用服务
    • css ：群集同步服务
    • crf ：Cluster Health Monitor
    • crs ：群集就绪服务
    • ctss ：集群时间同步服务
    • evm ：活动经理
    • gipc ：网格进程间通信

  • component ：指定其中一个模块的组件名称。有关所有组件的列表，请参阅表E-91 。

  • debugging_level ：指定1到5之间的数字，表示要返回debug命令的详细程度，其中1表示调试输出量最少，5表示调试输出最详细。默认调试级别为2。

以下示例显示如何为各种模块启用调试：

  • 要启用Oracle Clusterware的调试：

    #crsctl set log crs“CRSRTI：1，CRSCOMM：2”

  • 要启用OCR调试：

    #crsctl set log crs“CRSRTI：1，CRSCOMM：2，OCRSRV：4”

  • 要启用EVM的调试：

    #crsctl set log evm“EVMCOMM：1”

  • 启用资源调试

    #crsctl set log res“resname：1”

要获取可用于调试的组件列表，请使用以下语法运行crsctl lsmodules命令：

$ crsctl lsmodules {mdns | gpnp | css | crf | crs | ctss | evm | GIPC}

注意：

您不必是root用户来运行crsctl lsmodulues命令。

启用Oracle Clusterware资源的调试

您可以使用以下语法运行crsctl set log命令，以启用对Oracle Clusterware资源的调试：

crsctl set log res“ resource_name = debugging_level ”

以root用户身份运行crsctl set log命令，并提供以下信息：

  • resource_name ：要调试的资源的名称。

  • debugging_level ： 1到5 ，表示要返回debug命令的详细程度，其中1表示调试输出量最少，5表示调试输出最详细。

    注意：

    解决了需要进行其他调试的问题后，可以将调试级别重置为默认值，如下所示：
    #crsctl set log crs resource_name = default

要获取可用于调试的资源列表，请运行crsctl status resource命令。

例

要在node1上为VIP资源生成调试日志，请发出以下命令：

#crsctl set log res“ora.node1.vip:1”
