16 DBMS_ADVISOR

DBMS_ADVISOR是服务器可管理性顾问套件的一部分，这是一组专家系统，用于识别和帮助解决与数据库服务器组件相关的性能问题。

一些顾问有自己的包。对于这些顾问程序，Oracle建议您使用特定于顾问程序的程序包而不是DBMS_ADVISOR 。以下每个顾问都有自己的包，根据其特定功能进行定制：

  • 自动数据库诊断监视器（ DBMS_ADDM ）

  • SQL性能分析器（ DBMS_SQLPA ）

  • SQL修复顾问（ DBMS_SQLDIAG ）

  • SQL调优顾问（ DBMS_SQLTUNE ）

  • 压缩顾问（ DBMS_COMPRESSION.GET_COMPRESSION_RATIO ）

SQL Access Advisor和Segment Advisor是唯一具有DBMS_ADVISOR常见用例的顾问DBMS_ADVISOR 。 Undo Advisor和Compression Advisor不支持DBMS_ADVISOR子程序。

16.3.1 ADD_SQLWKLD_REF过程

此过程在当前SQL Access Advisor任务和SQL Workload对象之间建立链接。

注意：

从Oracle Database 11g开始，不推荐使用此过程。

该链接允许顾问任务访问有趣的数据以进行分析。该链接还提供稳定的数据视图。在建立SQL Access Advisor任务与SQL Workload对象之间的连接后，将保护工作负载不被删除或修改。

对于所有基于SQL调优集的顾问程序运行，用户应使用ADD_STS_REF而不是ADD_SQLWKLD_REF 。此功能仅用于向后兼容。

句法

DBMS_ADVISOR.ADD_SQLWKLD_REF（task_name IN VARCHAR2，workload_name IN VARCHAR2，is_sts IN NUMBER：= 0）;

参数

表16-2 ADD_SQLWKLD_REF过程参数

参数 描述

任务名称

唯一标识现有任务的SQL Access Advisor任务名称。

workload_name

要链接的工作负载对象的名称。将对象链接到任务后，它将变为只读且无法删除。对工作负载对象的链接数没有限制。要删除工作负载对象的链接，请使用DELETE_REFERENCE过程。

is_sts

指示工作负载源的类型。可能的值是：

  • 0 - SQL工作负载对象

  • 1 - SQL调优集

例子

DECLARE task_id NUMBER; task_name VARCHAR2（30）; workload_name VARCHAR2（30）; BEGIN task_name：='我的任务'; workload_name：='我的工作量'; DBMS_ADVISOR.CREATE_TASK（DBMS_ADVISOR.SQLACCESS_ADVISOR，task_id，task_name）; DBMS_ADVISOR.ADD_SQLWKLD_REF（task_name，workload_name，1）;结束; /

16.3.2 ADD_SQLWKLD_STATEMENT过程

此过程向指定的工作负载添加单个语句。

注意：

从Oracle Database 11g开始，不推荐使用此过程。

句法

DBMS_ADVISOR.ADD_SQLWKLD_STATEMENT（workload_name IN VARCHAR2，模块IN VARCHAR2，操作IN VARCHAR2，cpu_time IN NUMBER：= 0，elapsed_time IN NUMBER：= 0，disk_reads IN NUMBER：= 0，buffer_gets IN NUMBER：= 0，rows_processed IN NUMBER：= 0 ，optimizer_cost IN NUMBER：= 0，执行IN NUMBER：= 1，优先级IN NUMBER：= 2，last_execution_date IN DATE：='SYSDATE'，stat_period IN NUMBER：= 0，用户名IN VARCHAR2，sql_text IN CLOB）;

参数

表16-3 ADD_SQLWKLD_STATEMENT过程参数

参数 描述

workload_name

唯一标识现有工作负载的工作负载名称。

模

可选的业务应用程序模块，它将与SQL语句关联。

行动

与SQL语句关联的可选应用程序操作。

CPU_TIME

SQL语句消耗的总CPU时间（以秒为单位）。

ELAPSED_TIME

SQL语句消耗的总耗用时间（以秒为单位）。

DISK_READS

SQL语句使用的总磁盘读取操作数。

BUFFER_GETS

SQL语句使用的总缓冲区 - 获取操作。

rows_processed

SQL语句处理的平均行数。

optimizer_cost

优化程序计算的成本值。

处决

SQL语句的总执行次数。该值应大于零。

优先

SQL语句的相对优先级。该值必须为以下值之一：1- HIGH ，2 MEDIUM或3- LOW 。

last_execution_date

SQL语句上次执行的日期和时间。如果值为NULL ，则数据库使用当前日期和时间。

stat_period

计算语句统计信息的时间间隔（以秒为单位）。

用户名

执行SQL语句的数据库用户。由于用户名是Oracle标识符，因此username值必须与存储在服务器中的username完全相同。例如，如果用户SCOTT是执行用户，则必须以全大写字母提供用户标识符SCOTT 。它不会将用户scott或Scott识别为SCOTT的匹配项。

SQL_TEXT

完整的SQL语句。要提高建议的质量，SQL语句不应包含绑定变量。

使用说明

当活动任务当前引用工作负载时，您无法修改或删除该工作负载。如果任务未处于初始状态，则认为该任务处于活动状态。有关将任务设置为初始状态的说明，请参阅RESET_TASK过程 。

ADD_SQLWKLD_STATEMENT过程接受调用者可能忽略的几个参数。数据库仅在disk_reads实际分析时使用disk_reads ， buffer_gets和optimizer_cost参数对工作负载数据进行排序。因此，仅当order_list任务参数引用特定统计信息时，才需要实际值。

要确定在向工作负载添加新SQL语句时要提供的统计信息，请检查或设置任务参数order_list 。order_list参数接受键的任意组合：

  • CPU_TIME

  • ELAPSED_TIME

  • BUFFER_GETS

  • optimizer_cost

  • DISK_READS

  • 处决

  • 优先

optimizer_cost键是典型的priority设置，表示SQL Access Advisor按priority和optimizer_cost对工作负载数据进行排序，并首先处理最高成本语句。您添加到工作负载的任何语句都必须包含适当的priority和optimizer_cost值。所有其他统计信息可以默认或设置为零。

对于order_list任务参数引用的统计键，实际参数值应该相当准确，因为它们将与工作负载中的其他语句进行比较。如果调用者无法估计值，则选择将确定其相对于工作负载中其他语句的重要性的值。例如，如果当前语句被视为业务中最关键的查询，则适当的值将大于工作负载中找到的相同统计信息的所有其他值。

例子

DECLARE workload_name VARCHAR2（30）; BEGIN workload_name：='我的工作量'; DBMS_ADVISOR.CREATE_SQLWKLD（workload_name，'My Workload'）; DBMS_ADVISOR.ADD_SQLWKLD_STATEMENT（workload_name，'MONTHLY'，'ROLLUP'，100,400,5041,103,640445,680000,2,1，SYSDATE，1，'SH'，'SELECT AVG（amount_sold）FROM sh.sales'）;结束; /

16.3.3 ADD_STS_REF过程

此过程在当前SQL Access Advisor任务和SQL调优集之间建立链接。

该链接使顾问任务能够访问数据以进行分析。该链接还提供稳定的数据视图。在建立SQL Access Advisor任务和SQL调优集之间的连接后，将保护STS不被删除或修改。

对任何基于STS的顾问程序运行使用ADD_STS_REF 。使用ADD_SQLWKLD_REF和参数IS_STS=1的旧方法仅支持向后兼容性。此外， ADD_STS_REF函数接受SQL调优集所有者名称，而ADD_SQLWKLD_REF则不接受。

句法

DBMS_ADVISOR.ADD_STS_REF（task_name IN VARCHAR2 NOT NULL，sts_owner IN VARCHAR2，workload_name IN VARCHAR2 NOT NULL）;

参数

表16-4 ADD_STS_REF过程参数

参数 描述

任务名称

唯一标识现有任务的SQL Access Advisor任务名称。

sts_owner

SQL调优集的所有者。此参数的值可能为NULL ，在这种情况下，顾问程序假定SQL调优集由当前登录的用户拥有。

workload_name

要链接的工作负载的名称。工作负载由一个或多个SQL语句以及完全描述每个语句的统计信息和属性组成。数据库将工作负载存储为SQL调优集。

句法

DBMS_ADVISOR.CANCEL_TASK（task_name IN VARCHAR2）;

参数

表16-5 CANCEL_TASK过程参数

参数 描述

任务名称

唯一标识现有任务的有效Advisor任务名称。

使用说明

取消命令将任务恢复到取消操作开始之前的状态。因此，无法恢复已取消的任务或数据对象。

DECLARE task_id NUMBER; task_name VARCHAR2（30）; BEGIN task_name：='我的任务'; DBMS_ADVISOR.CREATE_TASK（DBMS_ADVISOR.SQLACCESS_ADVISOR，task_id，task_name）; DBMS_ADVISOR.UPDATE_TASK_ATTRIBUTES（task_name，'New Task Name'）; DBMS_ADVISOR.UPDATE_TASK_ATTRIBUTES（'新任务名'，NULL，'新描述'）;结束; /
