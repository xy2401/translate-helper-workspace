6 AMDU-00200到AMDU-00503

AMDU-00200：无法从偏移量[ number ]处的磁盘N number读取[ number ]个字节

原因：报告的磁盘读取操作失败。

操作：确保磁盘正常运行。

AMDU-00201：磁盘N number ：' string '

原因：报告的磁盘发生错误。此消息指示AMDU报告的磁盘编号的磁盘路径名称。

操作：有关详细信息，请参阅其他消息。

AMDU-00202：无法重新打开磁盘N number

原因：尝试重新打开磁盘失败。

操作：确保磁盘正常运行。

AMDU-00203：文件已删除。无法提取文件string 。

原因：指定的ASM文件不存在。

操作：更正为提取指定的ASM文件编号或名称。

AMDU-00204：磁盘N number位于当前安装的磁盘组string 。

原因：指定的ASM文件不存在。

操作：确保正确指定了ASM文件编号或名称。

AMDU-00205：深度发现期间磁盘N number打开失败。

原因：在发现过程中尝试打开磁盘失败。

操作：确保磁盘正常运行。

AMDU-00206：浅发现失败的string使用ASMLib string 。

原因：尝试发现ASMLIB磁盘失败。

操作：联系Oracle支持服务。

AMDU-00207： string浅发现失败。

原因：尝试发现磁盘失败。

操作：联系Oracle支持服务。

AMDU-00208：找不到文件目录块。无法提取文件string 。

原因：指定文件的文件目录块超出范围。

操作：更正为提取指定的ASM文件编号或名称。

AMDU-00209：找到损坏的块：磁盘N number AU [ number ]块[ number ]类型[ number ]

原因：报告块的块验证失败

操作：联系Oracle支持服务。

AMDU-00210：磁盘组string找不到磁盘

原因：未找到指定磁盘组的磁盘。

操作：确保正确输入磁盘组名称并且可以发现成员磁盘。

AMDU-00211：磁盘组string中的磁盘不一致。

原因：磁盘发现在多个磁盘组中发现了具有相同名称的磁盘。

操作：指定磁盘发现字符串，以避免使用非唯一名称对磁盘进行doscovery。

AMDU-00212：扫描磁盘组string未出现在报告中。

原因： AMDU无法在报告文件中找到有关指定磁盘组的信息。

操作：确保-directory选项指向具有报告文件的目录，并指定指定磁盘组的转储。

AMDU-00213：找不到组string磁盘N number

原因： AMDU遇到报告文件中不存在的磁盘的映射文件条目。mapfile可能已损坏。

操作：重新创建原始转储。如果操作仍然失败，请联系Oracle支持服务。

AMDU-00214：未为磁盘组string创建映像文件。

原因：原始的AMDU转储请求指定了-noimage。

操作：重新创建包含图像文件的AMDU转储。

AMDU-00215：对文件string number块进行镜像误比较

原因：镜像块的比较显示不匹配。

操作：联系Oracle支持服务。

AMDU-00216：0xbadfda7a写入文件string number块

原因： AMDU无法从磁盘读取多个块。它写了“0xbadfda7a”来代替这些块。

操作：联系Oracle支持服务。

AMDU-00217：无效的块num1 = [ string ]，num2 = [ string ]，error = [ string ]

原因：块报告验证因报告块而失败

操作：联系Oracle支持服务。

AMDU-00218：找不到投票文件。无法提取文件string 。

原因：指定的磁盘组中不存在投票文件。

操作：确保正确指定了磁盘组名称。

AMDU-00219：找到不一致的块：磁盘N number AU [ number ]块[ number ]类型[ number ]。

原因：装入磁盘组时块验证失败。

操作：卸载磁盘组并再次尝试操作。

AMDU-00220：请求提取文件string被拒绝

原因：尝试提取无法提取的文件类型。

操作：确保指定的文件编号是针对允许提取的文件类型。

AMDU-00500：操作系统相关操作： string失败，状态为： string

原因：存在操作系统调用错误。

操作：联系Oracle支持服务。

AMDU-00501：操作系统故障消息： string

原因：存在操作系统调用错误。

操作：联系Oracle支持服务。

AMDU-00502：发生故障： string

原因：存在操作系统调用错误。

操作：联系Oracle支持服务。

AMDU-00503：附加信息： string

原因：存在操作系统调用错误。

操作：联系Oracle支持服务。
