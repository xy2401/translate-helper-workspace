Oracle Enterprise Manager Cloud Control支持多租户环境安全性的管理。

在多租户环境中，您可以使用Oracle Enterprise Manager Cloud Control来创建，管理和监视根和相关可插拔数据库（PDB）的公共用户和角色。

企业管理器使您可以在根目录和指定的PDB之间轻松切换。

在Enterprise Manager中登录多租户环境

在多租户环境中，您可以登录CDB或PDB，并从PDB切换到其他PDB或根目录。

本节包含以下主题：

  • 登录CDB或PDB
    根据您在登录时请求的功能，将自动显示企业管理器数据库登录页面的不同变体。
  • 切换到不同的PDB或根目录
    在Oracle Enterprise Manager中，您可以从一个PDB切换到另一个PDB，也可以切换到根。

登录CDB或PDB

根据您在登录时请求的功能，将自动显示企业管理器数据库登录页面的不同变体。

要以CDB管理员（对CDB目标具有CONNECT权限的企业管理器用户）登录多租户环境，以使用CDB范围的功能：

  1. 以用户SYSTEM或SYSMAN身份登录Oracle Enterprise Manager云控制。

    URL如下：

    https：// host ： port / em
  2. 导航到“数据库”页面。
  3. 选择要访问的数据库。

    将显示数据库主页。

  4. 选择要执行的操作的菜单项，例如选择“ 管理” ，然后选择“ 安全” ，然后选择“ 用户”以对用户进行身份验证。

    出现“数据库登录”页面。以下示例显示了CDB的“数据库登录”页面（因为数据库名称显示为CDB$ROOT ）。由于此名称，此页面通俗地称为多租户环境的root的数据库登录页面。数据库字段指的是当前数据库;如果您选择了PDB，那么PDB的名称将出现在此字段中。


    插图说明em_login.gif
  5. 使用适当的凭据登录。

    请记住，只有普通用户才能登录到根目录，并且普通用户的名称以C##或c##开头。普通用户和本地用户都可以登录到PDB，具体取决于他们的权限。

切换到不同的PDB或根目录

在Oracle Enterprise Manager中，您可以从一个PDB切换到另一个PDB，也可以切换到根。

  1. 在页面的左上角，找到数据库链接。

    在数据库链接中，将显示当前容器名称。以下示例显示当前数据库是CDB本身（ CDB$ROOT ），通俗地称为根。


    插图说明em_database_breadcrumb.gif
  2. 选择容器右侧的菜单图标，然后从此菜单中选择要访问的数据库。

    如果未显示菜单项，则导航到显示它的页面，例如数据库主页。

  3. 当您决定要执行哪个活动（例如创建用户）时，请使用适当的权限登录。

    如果您尝试在未事先使用适当权限进行身份验证的情况下执行活动，则系统将提示您使用相应的权限登录。

在Enterprise Manager中管理公共和本地用户

在多租户环境中，Oracle Enterprise Manager使您可以创建，编辑和删除公共和本地用户。

本节包含以下主题：

  • 在企业管理器中创建通用用户帐户

  • 在Enterprise Manager中编辑公共用户帐户
    您可以从根目录编辑公共用户帐户。
  • 在Enterprise Manager中删除通用用户帐户
    您可以从CDB根目录中删除普通用户。
  • 在Enterprise Manager中创建本地用户帐户
    本地用户是仅存在于特定PDB中且无法访问多租户环境中的任何其他PDB的用户。
  • 在Enterprise Manager中编辑本地用户帐户
    您可以从本地用户所在的PDB编辑本地用户。
  • 在Enterprise Manager中删除本地用户帐户
    您可以从本地用户所在的PDB中删除本地用户。

在企业管理器中创建通用用户帐户

普通用户是存在于根中的用户，可以访问CDB中的PDB。

  1. 在企业管理器数据库主页中，以具有公共CREATE USER和SET CONTAINER权限的普通用户身份登录到root用户。
  2. 从Administration菜单中，选择Security ，然后选择Users 。

    如果出现提示，请输入您的登录信息之后，将显示“用户”页面。

  3. 单击“ 创建”

    将出现“创建用户”页面。


    插图说明em_com_user_create2.gif
  4. 选择用于创建公共用户的选项并授予此用户权限。

    确保用C##或c##作为用户名的c## 。

  5. 单击确定或应用 。

    普通用户在根目录中创建，并将显示在任何关联PDB的“用户”页面中。

在Enterprise Manager中编辑公共用户帐户

您可以从根目录编辑公共用户帐户。

  1. 在企业管理器数据库主页中，以具有公共CREATE USER和SET CONTAINER权限的普通用户身份登录到root用户。
    • 如果要登录到根目录，请确保您是具有常见CREATE USER和SET CONTAINER权限的普通CREATE USER 。
    • 如果要登录PDB，请确保您具有该PDB的CREATE USER权限。
  2. 从Administration菜单中，选择Security ，然后选择Users 。

    如果出现提示，请输入您的登录信息之后，将显示“用户”页面。在根目录中，仅列出普通用户。在PDB中，列出了普通用户和本地用户。

  3. 选择要编辑的普通用户，然后单击“ 编辑” 。

    将出现“编辑用户”页面。对于root中的普通用户，您可以修改普通用户的所有设置。对于PDB中的普通用户，您无法更改用户密码，默认表空间和临时表空间。您所做的设置仅适用于当前的PDB。以下屏幕显示了如何在PDB中显示常用用户“编辑用户”页面。


    插图说明em_com_user_edit.gif
  4. 根据需要修改普通用户。
  5. 单击“ 应用”

在Enterprise Manager中删除通用用户帐户

您可以从CDB根目录中删除普通用户。

  1. 在企业管理器数据库主页中，以具有公共CREATE USER和SET CONTAINER权限的普通用户身份登录到root用户。
    您无法从PDB中删除普通用户。
  2. 从Administration菜单中，选择Security ，然后选择Users 。

    如果出现提示，请输入您的登录信息之后，将显示“用户”页面，仅列出普通用户。

  3. 选择要删除的普通用户，然后单击“ 删除” 。
  4. 确认您要删除普通用户。

在Enterprise Manager中创建本地用户帐户

本地用户是仅存在于特定PDB中且无法访问多租户环境中的任何其他PDB的用户。

  1. 在Enterprise Manager数据库主页中，以具有本地CREATE USER权限的本地用户或公共用户身份登录到root CREATE USER 。
  2. 从Administration菜单中，选择Security ，然后选择Users 。

    如果出现提示，请输入您的登录信息之后，将显示“用户”页面，仅显示当前PDB的本地用户。

  3. 单击“ 创建”

    将出现“创建用户”页面。

  4. 选择创建本地用户的选项并授予此用户权限。

    确保不要在用户名前添加C##或c## 。

  5. 单击确定 。

    在当前PDB中创建本地用户。

在企业管理器中创建公共角色

常用角色可用于为普通用户分配公共权限。

这些角色在多租户环境的所有容器中都有效。
  1. 在企业管理器数据库主页中，以具有公共CREATE ROLE和SET CONTAINER权限的普通用户身份登录到root。
  2. 从Administration菜单中，选择Security ，然后选择Roles 。

    如果出现提示，请输入您的登录信息然后，将显示“创建角色”页面。

  3. 单击“ 创建”

    将出现“创建角色”页面。


    插图说明em_com_role_create.gif
  4. 选择创建公共角色的选项并授予此角色权限。

    确保使用C##或c##作为角色名称的c## 。

  5. 单击确定 。

    常用角色在根目录中创建。

在Enterprise Manager中编辑公共角色

您可以从根目录编辑公共角色。

  1. 从Enterprise Manager数据库主页，登录到根或PDB。如果要登录到根目录，请确保您是具有常见CREATE ROLE和SET CONTAINER权限的普通用户。如果要登录PDB，请确保您具有该PDB的CREATE ROLE权限。
  2. 从Administration菜单中，选择Security ，然后选择Roles 。

    如果出现提示，请输入您的登录信息之后，将显示“角色”页面。在根目录中，仅显示常见角色。在PDB中，显示了常见和本地角色。

  3. 选择要编辑的公共角色，然后单击“ 编辑” 。

    将出现“编辑角色”页面。对于root中的普通用户，您可以修改普通用户的所有设置。

    对于PDB中的常见角色，您只能更改角色的身份验证，并为此用户授予不同的角色，系统权限，对象权限和使用者组权限。这些设置仅适用于当前的PDB。

  4. 根据需要修改普通用户。
  5. 单击“ 应用”

撤消企业管理器中的公共权限授予

  6. 单击“ 确定”按钮。
