14.2关于将Oracle数据库用作Web服务使用者

除了启用联合数据之外，您还可以扩展关系数据库的存储，索引和搜索功能，以包括半结构化和非结构化数据（包括Web服务）。通过调用Web服务，数据库可以跟踪，聚合，刷新和查询按需生成的动态数据，例如股票价格，货币汇率和天气信息。

将Oracle数据库用作服务使用者的一个示例是从预定义的数据库作业调用外部Web服务以从多个供应商检索库存信息，然后更新本地库存数据库。另一个例子是Web爬虫，可以安排数据库作业来整理来自多个来源的产品和价格信息。

本节包括以下主题：

  • 如何使用Oracle JVM Web服务调用实用程序

  • Web服务数据源（虚拟表支持）

  • Oracle数据库作为Web服务使用者的功能

14.2.1关于使用Oracle JVM Web服务调用实用程序

从Oracle Database 12 c第2版（12.2.0.1）开始，您可以使用Oracle JVM Web服务调用实用程序从Oracle数据库中调用网络中运行的Web服务的操作。此实用程序接受以WSDL格式指定的SOAP Web服务或以WADL格式指定的REST Web服务。

使用此实用程序之前执行以下操作：

  • 设置JAVA_HOME环境变量。

  • 使用以下命令创建OJVMWCU_INSTALL模式：

    创建由<ANY_PASSWROD>标识的用户OJVMWCU_INSTALL

    注意：

    • 必须在运行install_ojvmwcu.sql脚本之前创建OJVMWCU_INSTALL模式。install_ojvmwcu.sql脚本检查数据库中是否存在OJVMWCU_INSTALL模式。如果没有，则显示一条消息，表明架构不存在并停止运行。

    • 仅为使用Oracle JVM Web服务调出实用程序创建OJVMWCU_INSTALL模式，不应将其用于任何其他目的。

  • 运行install_ojvmwcu.sql脚本，然后运行grant_ojvmwcu.sql脚本。这两个脚本文件都存在于ORACLE_HOME/javavm/ojvmwcu/install目录中。

    grant_ojvmwcu.sql脚本将用户名作为参数，并且必须以SYSDBA形式调用它。例如： sqlplus / as sysdba @ grant_ojvmwcu.sql scott

14.2.1.1 Oracle JVM Web服务调用实用程序的体系结构

Oracle JVM Web服务调出实用程序包含以下两个阶段：

  • 客户端存根生成

  • Oracle JVM特有的工件生成

客户端存根生成

Oracle JVM Web服务调用实用程序使用JAX-WS库并从“Oracle JVM Web服务调用实用程序的输入”部分中指定的输入生成Java客户机存根，以访问SOAP Web服务。对于REST服务，Oracle JVM Web服务调用实用程序使用第三方工具wadl2java ，您必须单独下载该工具。

注意：

如果使用REST服务，则在下载wadl2java工具后，必须执行以下任一操作：

  • 将WADL_HOME环境变量设置为wadl2java工具目录的路径

  • 使用–t命令行选项指定wadl2java工具目录的路径

Oracle JVM特定的工件生成

要从PL / SQL访问Web服务，您需要一个静态Java方法和一个PL / SQL包装函数，用于Web服务支持的每个操作。Oracle JVM Web服务调用实用程序为Web服务中的每个受支持的操作创建静态方法，并通过解释不同的注释从生成的客户端类中提取操作的详细信息。提取的信息包括WebService，Webmethods，WebServiceClient和WebEndpoint。使用此信息，该实用程序生成相应的静态方法，使得每个操作具有与发布的Web服务中的相应操作相同的输入参数和返回类型。然后，它将对应于每个支持的操作的所有静态方法添加到Java类。

然后，Oracle JVM Web服务调用实用程序创建与生成的Java类中的每个静态方法相对应的PL / SQL包装函数，并将这些函数打包到具有Web服务名称的PL / SQL包中。它还生成PL / SQL包装器，用于授予和撤消在Oracle JVM中运行Java类的基本权限。

14.2.1.4从Oracle JVM Web服务调用实用程序调用安全Web服务

Oracle JVM Web服务调用实用程序为基于SSL的Web服务提供支持。此实用程序还为使用基本HTTP身份验证保护的Web服务提供支持。如果您使用的是基于SSL的Web服务，则必须在运行此实用程序之前将SSL证书添加到密钥库，或使用-ts命令行选项传递信任库路径。在Web调用之前，必须使用wrappers.sql文件中的grabAndSaveCertificate <WebServiceName> (host, port)过程来设置密钥存储路径的路径。如果您使用的是使用基本身份验证保护的Web服务，请在此实用程序中使用-Xauthfile<auth_file>命令行选项。auth_file参数包含以下格式的授权信息：

HTTP：//用户名：密码@ <网络的serviceURL>的WSDL

在Web调用之前，必须使用wrappers.sql文件中的setwsCred<WebServiceName>(usr,pwd)来设置Web服务凭据。

14.2.2 Web服务数据源（虚拟表支持）

要访问从单个或多个Web服务调用返回的数据，请使用Web服务数据源创建虚拟表。此表允许您查询一组返回的行，就像它是一个表一样。

客户端调用Web服务，结果存储在数据库的虚拟表中。您可以将结果集从函数传递给函数。这使您可以设置转换序列，而不包含保存中间结果的表。要减少内存使用量，可以在函数内一次返回几个结果集行。

通过将Web服务与表函数一起使用，您可以将来自单个或多个Web服务的一系列输入值作为实际表进行操作。在以下示例中，内部SELECT语句创建行，其列用作调用CALL_WS Web服务调用的参数。 

SELECT column1 ， cloumn2 ，...FROM TABLE（WS_TABFUN（CURSOR（SELECT s FROM table_name ）））WHERE ...

前面示例中的表表达式可用于其他SQL查询，用于构造视图等。

14.2.3 Oracle数据库作为Web服务使用者的功能

将Oracle数据库用作Web服务使用者可提供以下功能：

  • 从Java使用Web服务

    为Web服务调用提供易于使用的界面，从而使开发人员免受低级SOAP编程的影响。在数据库中运行的Java类可以使用以前加载的Java代理类或通过动态调用直接调用外部Web服务。

  • 从SQL和PL / SQL中使用Web服务

    允许任何启用SQL的工具或应用程序透明且轻松地使用来自外部Web服务的动态数据。在将Web服务方法公开为Java存储过程之后，Java存储过程之上的PL / SQL包装器会隐藏SQL客户端的所有Java和SOAP编程详细信息。

  • 使用Web服务数据源

    通过将外部Web服务转换为SQL数据源，使外部Web服务显示为常规SQL表，从而实现应用程序和数据集成。此表函数表示调用外部Web服务的输出，可以在SQL查询中使用。
