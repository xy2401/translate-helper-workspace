Oracle Database Vault中的安全应用程序角色是什么？

在Oracle Database Vault中，您可以创建使用Oracle Database Vault规则集启用的安全应用程序角色。

通过自定义PL / SQL过程启用常规Oracle数据库安全应用程序角色。您使用安全应用程序角色来阻止用户从应用程序外部访问数据。这会强制用户在已授予角色的应用程序权限的框架内工作。

在多租户环境中，您只能在PDB中创建安全应用程序角色，而不能在CDB根目录或应用程序根目录中创建。

为规则集上的角色建立数据库访问权限的优点是，您可以将数据库安全策略存储在一个中心位置，而不是将其存储在所有应用程序中。基于规则集的角色提供了一致且灵活的方法来实施角色提供的安全策略。这样，如果必须更新应用程序角色的安全策略，则可以在一个位置（规则集）中执行此操作。此外，无论用户如何连接到数据库，结果都是相同的，因为规则集绑定到角色。您需要做的就是创建角色，然后将其与规则集相关联。关联的规则集验证尝试启用该角色的用户。

创建Oracle Database Vault安全应用程序角色

您可以在Database Vault Administrator中创建Database Vault安全应用程序角色。

  1. 以已授予DV_OWNER或DV_ADMIN角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 创建一个包含至少一个规则的规则集，以设置允许或禁止用户启用该角色的条件。

    为规则集创建基础规则时，请记住该规则应验证尝试启用该角色的用户。

  3. 在“管理”页面的“数据库Vault组件”下，单击“ 安全应用程序角色” 。
  4. 在“安全应用程序角色”页面中，单击“ 创建


    插图描述create_secure_app_role122.png
  5. 在“创建安全应用程序角色”页面中，输入以下设置：

    • 角色名称：使用不超过30个字符输入名称，不能有空格。确保此名称遵循Oracle数据库SQL语言参考中描述的CREATE ROLE语句创建角色的标准Oracle命名约定。此属性是必需的。

    • 状态：选择“已启用”或“ 已禁用”以在运行时启用或禁用安全应用程序角色。此属性是必需的。

      • 已启用：允许角色可供使用。也就是说，允许用户调用DBMS_MACSEC_ROLES.SET_ROLE函数来尝试启用该角色。请注意，是否启用角色取决于关联规则集的评估结果。

      • 已禁用：禁用角色可供使用。DBMS_MACSEC_ROLES.SET_ROLE函数将无法启用该角色。

    • 规则集：从列表中，选择要与安全应用程序角色关联的规则集。此属性是必需的。

      调用DBMS_MACSEC_ROLES.SET_ROLE ，如果规则集的计算结果为true，则Oracle Database Vault将启用数据库会话的角色。如果规则集的计算结果为false，则表明该角色未启用。

  6. 单击确定 。

对安全应用程序角色的修改

只有在Oracle Database Vault中创建现有安全应用程序角色时，才能对其进行修改。

您无法修改在Oracle Database Vault外部创建的安全应用程序角色或数据库角色。如果要修改现有Oracle数据库角色以使其可以与Oracle Database Vault一起使用，请在Oracle Database Vault中创建新的安全应用程序角色，然后将现有角色授予安全应用程序角色。例如，在SQL * Plus中：

授予myExistingDBrole to myDVrole;

创建新的安全应用程序角色后，必须修改代码才能使用此新角色。您可以在应用程序代码中使用DBMS_MACSEC_ROLES.SET_ROLE来完成此任务。

Oracle Database Vault安全应用程序角色的安全性

具有数据库管理权限的用户可能会尝试使用DROP ROLE语句删除Oracle Database Vault安全应用程序角色。

每当创建Oracle Database Vault安全应用程序角色时，Database Vault都会将安全应用程序角色添加到Oracle Database Vault领域。这可以防止数据库管理员使用DROP ROLE语句删除安全应用程序角色。

Oracle Database Vault安全应用程序角色的工作原理

创建和设置安全应用程序角色后，将开始Oracle Database Vault安全应用程序角色的处理流程。

  1. 在Oracle Database Vault Administrator中创建或更新角色，或者使用DBMS_MACADM包中的安全应用程序角色特定功能。

    有关更多信息，请参阅DBMS_MACADM安全应用程序角色过程 。

  2. 使用DBMS_MACSEC_ROLES.SET_ROLE函数修改应用程序以调用该角色。

    有关更多信息，请参见SET_ROLE过程 。

  3. 然后，Oracle Database Vault会评估与安全应用程序角色关联的规则集。

    如果规则集的计算结果为true，则Oracle Database Vault将为当前会话启用该角色。如果规则集的计算结果为false，则不会启用该角色。在任何一种情况下，Oracle Database Vault都会处理与安全应用程序角色关联的规则集的关联审计和自定义事件处理程序。

第3步：创建规则集及其规则

规则集和规则将限制谁可以修改OE.ORDERS表中的OE.ORDERS 。

  1. 以已授予DV_OWNER或DV_ADMIN角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 在“管理”页面中，选择“ 规则集” 。

    将出现“规则集”页面。

  3. 单击“ 创建”

    将出现“创建规则集”页面。

  4. 输入以下信息：

    • 名称：输入Can Modify Orders 。

    • 描述：输入Rule set to control who can modify orders in the OE.ORDERS table 。

    • 状态：选择已启用 。

    • 评估选项：选择全部为真。

  5. 保留其余设置及其默认值，然后单击“ 下一步”转到“与规则关联”页面。
  6. 单击“ 创建规则”，然后在“创建规则”对话框中，输入以下设置：

    • 名称： Check IP Address

    • 表达式： DVF.F$CLIENT_IP = ' your_IP_address '

    对于“检查IP地址”规则，请将your_IP_address替换为计算机的IP地址。在实际场景中，您将创建一个表达式，其中包含应被允许访问的用户的所有IP地址。

    此规则使用默认因子Client_IP。如果已删除此因子，则可以使用以下规则表达式：

    UPPER（SYS_CONTEXT（'USERENV'，'IP_ADDRESS'））=' your_IP_address '
  7. 单击确定 。
  8. 再次单击“ 创建规则” ，在“创建规则”对话框中，输入以下设置：

    • 名称： Check Session User

    • 表达式： DVF.F$SESSION_USER IN ('EABEL','AHUTTON')

    此规则使用默认因子Session_User。如果已删除或修改此因子，则可以使用以下规则表达式：

    UPPER（SYS_CONTEXT（'USERENV'，'SESSION_USER'））IN（'EABEL'，'AHUTTON'）
  9. 单击确定 。
  10. 单击“完成” ，然后单击“ 完成” 。

步骤6：测试Database Vault安全应用程序角色

有了所有组件，您就可以测试Database Vault安全应用程序角色。

  1. 在SQL * Plus中，以eabel用户eabel直接连接到数据库。
    connect eabel @ orcl输入密码： 密码

    将orcl替换为数据库实例的名称。

  2. 设置ORDERS_MGMT角色。
    EXEC DBMS_MACSEC_ROLES.SET_ROLE（'ORDERS_MGMT'）;

    通常，您可以将此调用嵌入用户登录的应用程序中。

  3. 从OE.ORDERS表中选择。
    SELECT COUNT（*）FROM OE.ORDERS;

    应出现以下输出：

    COUNT（*）---------- 105

    由于用户eabel从正确的IP地址直接登录到数据库并被列为有效的会话用户，因此她可以从OE.ORDERS表中进行选择。如果用户ahutton以相同的方式登录到SQL * Plus，她也可以从OE.ORDERS表中进行选择。

  4. 在不指定数据库实例的情况下重新连接为用户eabel ，然后再次尝试从OE.ORDERS表中进行选择。
    CONNECT eabel输入密码： 密码 EXEC DBMS_MACSEC_ROLES.SET_ROLE（'ORDERS_MGMT'）;

    应出现以下输出：

    第1行出错：ORA-47305：SET ROLE上的规则集违规（可以修改订单）......

    下一个：

    SELECT COUNT（*）FROM OE.ORDERS;

    应出现以下输出：

    第1行的错误：ORA-00942：表或视图不存在

    即使用户eabel是有效用户，她也违反了规则集中的“检查IP地址”规则，因此她无法启用ORDERS_MGMT角色。识别IP地址的唯一方法是通过指定数据库实例进行连接，如eabel在步骤1中eabel那样。（有关其工作原理的说明，请参阅步骤3中的步骤9 ：在配置因子 中将域因子标识映射到Client_IP因子 。）

  5. 以用户ldoran连接。
    CONNECT ldoran - 或者，CONNECT ldoran @hrpdb输入密码： password
  6. 输入以下语句：
    EXEC DBMS_MACSEC_ROLES.SET_ROLE（'ORDERS_MGMT'）; SELECT COUNT（*）FROM OE.ORDERS;

    由于用户ldoran不是有效用户，因此无法启用ORDERS_MGMT角色。因此，她无法从OE.ORDERS表中进行选择。
