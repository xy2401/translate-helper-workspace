171.1 DBMS_TRACE概述

DBMS_TRACE提供子程序来启动和停止会话中的PL / SQL跟踪。Oracle在程序执行时收集跟踪数据并将其写入数据库表。

典型的会议涉及：

  1. （可选）限制对特定子程序的跟踪并选择跟踪级别。

    跟踪大型程序中的所有子程序和异常会产生大量难以管理的数据。

  2. 启动PL / SQL中的会话（跟踪DBMS_TRACE 。 SET_PLSQL_TRACE ）。

  3. 运行要跟踪的应用程序。

  4. 停止PL / SQL跟踪会话（ DBMS_TRACE 。 CLEAR_PLSQL_TRACE ）。

使用Trace收集数据后，可以查询包含性能数据的数据库表，并以与分析Profiler中的性能数据相同的方式对其进行分析。

171.2 DBMS_TRACE安全模型

必须在SYS下创建此包。

171.4 DBMS_TRACE限制

您不能在共享服务器环境中使用PL / SQL跟踪。

171.5 DBMS_TRACE操作说明

某些操作说明适用于DBMS_TRACE 。

这些在以下部分中描述：

控制数据量

分析大型应用程序可能会产生大量数据。您可以通过启用特定程序单元来跟踪数据收集来控制收集的数据量。

您可以通过编译它来启用程序单元。这可以通过以下两种方式之一完成：

alter session set plsql_debug = true;创建或替换... / *创建库单元 - 将生成调试信息* /

要么：

/ *使用调试选项重新编译特定的库单元* / alter [PROCEDURE |功能| PACKAGE BODY] <libunit-name>编译调试;

注意：

您不能将第二种方法用于匿名块。

您可以通过在SET_PLSQL_TRACE过程的TRACE_LEVEL参数中包含TRACE_LIMIT来仅保留最近的8,192条记录（大约）来限制数据库中使用的存储量。

创建数据库表以收集DBMS_TRACE输出

你必须创建DBMS_TRACE包写入输出的数据库表。否则，不会收集数据。要创建这些表，请运行脚本TRACETAB.SQL 。此脚本创建的表由SYS拥有。

收集跟踪数据

您可以跟踪的PL / SQL功能在脚本DBMSPBT.SQL中描述。一些关键的跟踪功能是：

  • 追踪电话

  • 跟踪异常

  • 跟踪SQL

  • 追踪线

DBMS_TRACE的其他功能还允许暂停和恢复跟踪，并限制输出。

追踪电话

有两种级别的呼叫跟踪：

  • 级别1：跟踪所有呼叫。这对应于常量TRACE_ALL_CALLS 。

  • 级别2：仅对已启用的程序单元进行跟踪调用。这对应于常量TRACE_ENABLED_CALLS 。

无法检测远程过程调用（RPC）的启用情况;因此，RPC仅跟踪级别1。

跟踪异常

可以使用两级异常跟踪：

  • 第1级：跟踪所有异常。这对应于TRACE_ALL_EXCEPTIONS 。

  • 级别2：仅在启用的程序单元中引发的跟踪异常。这对应于TRACE_ENABLED_EXCEPTIONS 。

跟踪SQL

有两种级别的SQL跟踪：

  • 级别1：跟踪所有SQL。这对应于常量TRACE_ALL_SQL 。

  • 级别2：仅在启用的程序单元中跟踪SQL。这对应于常量TRACE_ENABLED_SQL 。

追踪线

有两种级别的线追踪：

  • 第1级：跟踪所有行。这对应于常量TRACE_ALL_LINES 。

  • 级别2：仅在启用的程序单元中跟踪行。这对应于常量TRACE_ENABLED_LINES 。

在跟踪行时，Oracle会在每次行号更改时向数据库添加记录。这包括由于过程调用和返回而导致的行号更改。

注意：

对于所有类型的跟踪，级别1将覆盖级别2。例如，如果启用了级别1和级别2，则级别1优先。

收集的数据

如果仅对启用的程序单元请求跟踪，并且如果未启用当前程序单元，则不会写入跟踪数据。

跟踪呼叫时，将跟踪呼叫和返回。如果被调用的例程或调用例程被“启用”，则检查是否“已启用”跟踪。

呼叫跟踪将始终输出呼叫者和被呼叫者的程序单元类型，程序单元名称和行号。它将输出调用者的堆栈深度。如果启用了呼叫者，则还将输出呼叫者的姓名。如果启用了被调用者，则还将输出被调用者的名称。

异常跟踪会写出行号。引发异常会显示有关异常是用户定义还是预定义的信息。它还显示了预定义异常情况下的异常编号。引发异常的位置和跟踪它们的处理程序。“启用”跟踪检查是针对引发异常的位置和处理异常的位置独立完成的。启用NO_TRACE_HANDLED_EXCEPTIONS会将数据收集限制为未处理的异常

对DBMS_TRACE.SET_PLSQL_TRACE和DBMS_TRACE.CLEAR_PLSQL_TRACE所有调用DBMS_TRACE.SET_PLSQL_TRACE在数据库中放置一条特殊的跟踪记录。因此，始终可以确定何时更改跟踪设置。

跟踪控制

除了确定收集哪些项目之外，您还可以暂停和恢复跟踪过程。在跟踪暂停的时间和恢复时间之间不收集任何信息。常量TRACE_PAUSE和TRACE_RESUME用于实现此目的。生成跟踪记录以指示跟踪已暂停/恢复。

通过使用常量TRACE_LIMIT也可以仅保留运行的最后8,192个跟踪事件。这允许在不填满数据库的情况下打开跟踪。当跟踪停止时，将保存最后的8,192条记录。限制是近似值，因为未在每个跟踪记录中检查。至少会生成所请求的跟踪记录数;最多可生成1,000条记录。至少会生成所请求的跟踪记录数;最多可生成1,000条记录。可以更改8,192记录限制。将事件10940设置为级别n会将记录限制更改为1024 * n 。

启用NO_TRACE_ADMINISTRATIVE可防止生成PL/SQL Trace Tool started ， Trace flags changed ， PL/SQL Virtual Machine started以及PL/SQL Virtual Machine stopped等此类管理事件记录。
