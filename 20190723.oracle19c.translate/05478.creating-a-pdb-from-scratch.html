6.2创建PDB

使用CREATE PLUGGABLE DATABASE语句，您可以从PDB种子创建PDB，并且可以从应用程序种子或PDB种子创建应用程序PDB。

先决条件

在从PDB种子（ PDB$SEED ）创建PDB或从应用程序种子或PDB种子创建应用程序PDB之前，请完成“ PDB创建的一般先决条件”中描述的先决条件 。

要创建PDB：

  1. 在SQL * Plus中，确保当前容器是CDB根目录或应用程序根目录。

    当前容器是CDB根时，使用PDB种子的文件在CDB中创建PDB。

    当前容器是应用程序根时，将使用应用程序种子的文件在应用程序容器中创建应用程序PDB。如果应用程序容器中没有应用程序种子，则使用PDB种子的文件在应用程序容器中创建应用程序PDB。

  2. 运行CREATE PLUGGABLE DATABASE语句，并为PDB指定本地管理员。在需要时指定其他子句。

    创建PDB后，它处于挂载模式，其状态为NEW 。您可以通过查询V$PDBS视图中的OPEN_MODE列来查看PDB的打开模式。您可以通过查询CDB_PDBS或DBA_PDBS视图的STATUS列来查看PDB的STATUS 。

    为PDB创建新的默认服务。该服务与PDB具有相同的名称，可用于访问PDB。必须正确配置Oracle Net Services才能使客户端访问此服务。

  3. 以读/写模式打开新PDB。

    您必须在Oracle数据库的读/写模式下打开新的PDB，才能完成新PDB与CDB的集成。如果尝试以只读模式打开PDB，则会返回错误。在读/写模式下打开PDB后，其状态为NORMAL 。

  4. 备份PDB。

    除非备份PDB，否则无法恢复PDB。

创建具有指定本地管理员名称的本地用户，并在PDB中本地授予PDB_DBA公共角色。如果在PDB创建期间未授予此用户管理员权限，则使用SYS和SYSTEM常用用户管理PDB。

注意：

如果在创建PDB期间返回错误，则正在创建的PDB可能处于UNUSABLE状态。您可以通过查询CDB_PDBS或DBA_PDBS视图来检查PDB的状态，并通过检查警报日志了解有关PDB创建错误的更多信息。只能删除不可用的PDB，并且必须先删除它，然后才能创建与不可用的PDB同名的PDB。

6.3.1使用无子句创建PDB：示例

此示例显示了创建PDB的最简单方法。

此示例假定以下因素：

  • PDB不需要存储限制。因此，不需要STORAGE子句。

  • PDB不需要默认表空间。

  • PATH_PREFIX子句不是必需的。

  • 不需要FILE_NAME_CONVERT子句和CREATE_FILE_DEST子句。

    为CDB启用了Oracle Managed Files，或者设置了PDB_FILE_NAME_CONVERT初始化参数。与PDB种子或应用程序种子关联的文件将根据Oracle Managed Files配置或初始化参数设置复制到新位置。

  • 没有与将在目标位置创建的新临时文件同名的文件。因此， TEMPFILE REUSE子句。

  • 不需要为PDB_DBA角色授予预定义的Oracle角色。

以下语句创建PDB：

CREATE PLUGGABLE DATABASE salespdb ADMIN USER salesadm IDENTIFIED BY password ;

6.3.2创建PDB并向PDB管理员授予预定义的Oracle角色：示例

此示例使用ROLES参数授予预定义角色。

此示例假定以下因素：

  • PDB不需要存储限制。因此，不需要STORAGE子句。

  • PDB不需要默认表空间。

  • PATH_PREFIX子句不是必需的。

  • 不需要FILE_NAME_CONVERT子句和CREATE_FILE_DEST子句。

    已启用Oracle Managed Files，或者已设置PDB_FILE_NAME_CONVERT初始化参数。与PDB种子或应用程序种子关联的文件将根据Oracle Managed Files配置或初始化参数设置复制到新位置。

  • 没有与将在目标位置创建的新临时文件同名的文件。因此， TEMPFILE REUSE子句。

  • 应在本地为PDB_DBA角色授予以下预定义Oracle角色： DBA 。

以下语句创建PDB：

CREATE PLUGGABLE DATABASE salespdb ADMIN USER salesadm IDENTIFIED BY password ROLES =（DBA）;

6.3.3使用多个子句创建PDB：示例

此示例使用STORAGE ， DEFAULT TABLESPACE ， PATH_PREFIX和FILE_NAME_CONVERT子句创建PDB。

此示例假定以下因素：

  • 必须对PDB强制执行存储限制。因此，需要STORAGE子句。具体而言，属于PDB的所有表空间不得超过2千兆字节。

  • 对于未指定其他永久表空间的任何非管理用户，都需要默认的永久表空间。具体来说，此示例创建一个名为sales的默认永久表空间，具有以下特征：

    • 表空间的单个数据文件是sales01.dbf ，该语句在/disk1/oracle/dbs/salespdb目录中创建它。

    • SIZE子句指定表空间的初始大小为250兆字节。

    • AUTOEXTEND子句启用文件的自动扩展。

  • 必须将路径前缀添加到PDB目录对象路径。因此， PATH_PREFIX子句是必需的。在此示例中，路径前缀/disk1/oracle/dbs/salespdb/被添加到PDB的目录对象路径中。

  • 将不使用CREATE_FILE_DEST子句，未启用Oracle Managed Files，并且未设置PDB_FILE_NAME_CONVERT初始化参数。因此，需要FILE_NAME_CONVERT子句。在系统上指定PDB种子或应用程序种子的数据文件的位置。在此示例中，Oracle数据库将文件从/disk1/oracle/dbs/pdbseed到/disk1/oracle/dbs/salespdb 。

  • 没有与将在目标位置创建的新临时文件同名的文件。因此， TEMPFILE REUSE子句。

  • 不需要为PDB_DBA角色授予预定义的Oracle角色。

以下语句创建PDB：

CREATE PLUGGABLE DATABASE salespdb ADMIN USER salesadm IDENTIFIED BY password STORAGE（MAXSIZE 2G）DEFAULT TABLESPACE sales DATAFILE'/disk1/oracle/dbs/salespdb/sales01.dbf'STPE 250M AUTOEXTEND ON PATH_PREFIX ='/ disk1 / oracle / dbs / salespdb /' FILE_NAME_CONVERT =（'/ disk1 / oracle / dbs / pdbseed /'，'/ disk1 / oracle / dbs / salespdb /'）;
