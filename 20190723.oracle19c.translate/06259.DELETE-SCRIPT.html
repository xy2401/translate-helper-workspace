2.16删除脚本

目的

使用DELETE SCRIPT命令从恢复目录中删除本地或全局存储的脚本。

先决条件

仅在RMAN提示符下执行DELETE SCRIPT 。必须将RMAN连接到恢复目录和目标数据库。必须打开恢复目录数据库。

使用说明

存储的脚本可以是本地脚本或全局脚本。仅为当前目标数据库创建本地脚本，而全局脚本可用于在恢复目录中注册的任何数据库。

如果指定了GLOBAL ，则恢复目录中必须存在具有此名称的全局脚本;否则，RMAN返回错误RMAN-06710 。如果未指定GLOBAL ，则RMAN将查找在当前目标数据库上定义的具有指定名称的本地存储脚本。如果在目标数据库上未定义此类脚本，则RMAN将检查具有此名称的全局存储脚本，如果存在则将其删除。

句法

deleteScript :: =


说明deletescript.eps的说明

语义

语法元素 描述

全球

将脚本标识为全局。

如果您尝试删除全局脚本，则不得将RMAN连接到虚拟专用目录。虚拟目录用户无法修改全局脚本，尽管他们可以执行它们。

另请参阅： “ 使用说明 ” ，以解释全局脚本和本地脚本之间的区别

SCRIPT script_name

指定要删除的脚本的名称。当名称包含空格或保留字时，必须在脚本名称周围使用引号。
