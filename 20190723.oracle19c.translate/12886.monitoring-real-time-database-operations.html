5.1关于监控数据库操作

Cloud Control中的SQL Monitoring页面显示可用于监视数据库操作执行时的性能的信息，查看有关用于最近完成的操作的时间和资源的详细信息，以及跟踪和报告数据库操作。数据库操作是由最终用户或应用程序代码（例如SQL语句或PL / SQL函数，批处理作业或提取，转换和加载（ETL）处理）定义的一组数据库任务。您可以定义，监视和报告数据库操作。

5.1.1数据库操作的类型

数据库操作要么简单，要么是复合操简单的数据库操作是单个SQL语句或PL / SQL过程或函数。当SQL语句并行运行时，或者在单次执行中消耗了至少5秒的CPU或I / O时间时，将自动监视简单操作。

复合数据库操作是数据库会话中两个定义的时间点之间的活动。您可以使用PL / SQL过程开始和结束复合操作，如“ 创建数据库操作 ”中所示 。一次只能有一个复合数据库操作可以在数据库会话中运行。

5.1.2监控数据库操作的目的

通常，数据库操作监视对以下用户很有用：

  • DBA的职责包括识别昂贵的（高响应时间）SQL语句和PL / SQL函数

  • 管理数据仓库或OLTP系统中的批处理作业的DBA

  • 需要监视与特定操作相关的活动的应用程序或数据库开发人员，例如，数据泵操作

监视数据库操作对于执行以下任务很有用：

  • 跟踪和报告

    跟踪需要首先定义复合数据库操作。操作开始时，数据库基础结构将确定代表操作跟踪的内容。例如，您的调优任务可能涉及确定哪些SQL语句代表特定批处理作业运行，它们的执行统计信息是什么，执行操作时数据库中发生了什么，等等。在Cloud Control中，您可以查看复合数据库操作的报告以及组成复合数据库操作的简单数据库操作。您可以将报告保存到磁盘。

  • 监控执行进度

    此任务涉及监视当前正在执行的数据库操作。当您调查操作需要很长时间才能完成的原因时，此信息特别有用。

  • 监视资源使用情况

    您可能希望检测SQL执行何时使用过多的CPU，发出过多的I / O，或者需要很长时间才能完成。使用Oracle数据库资源管理器（资源管理器），您可以为每个使用者组配置阈值，以指定组中所有SQL执行的最大资源使用情况。当SQL操作达到指定阈值时，资源管理器可以将操作切换到优先级较低的使用者组，终止会话或取消并隔离SQL操作。在Cloud Control中，您可以检查这些SQL操作。

    也可以看看：

    有关使用资源管理器设置资源限制的消费者组切换的详细信息，请参见“Oracle数据库管理员指南”

  • 调整响应时间

    调整数据库操作时，通常需要缩短响应时间。数据库操作性能问题通常主要是SQL性能问题。

5.1.3启用数据库操作监控

当STATISTICS_LEVEL初始化参数设置为TYPICAL （默认值）或ALL时，默认情况下启用实时数据库操作监视。由于数据库操作监视是Oracle Tuning Pack的一项功能，因此CONTROL_MANAGEMENT_PACK_ACCESS参数必须设置为DIAGNOSTIC+TUNING （默认值）。

5.1.4数据库操作的属性

数据库操作具有唯一标识它的属性。这些属性如下：

  • 数据库操作名称

    这是用户创建的名称，例如daily_sales_report 。有关命名数据库操作的示例，请参阅“ 创建数据库操作 ” 。

  • 数据库操作执行ID

    同一数据库操作的两次或多次出现可以同时运行，每个都在不同的数据库会话中，具有相同的名称但具有不同的执行ID。此数字ID唯一标识同一数据库操作的不同执行。

    开始数据库操作时，数据库会自动创建执行ID。您还可以指定自己的执行ID。

5.2创建数据库操作

您可以使用DBMS_SQL_MONITOR程序包子程序创建复合数据库操作。

以下示例创建名为DBOP_EXAMPLE的数据库操作。该示例开始数据库操作。它有一个PL / SQL程序，按城市选择客户的最大销售额。然后它有一个SQL语句，用于从拥有至少两个客户的城市中选择所有客户的最大销售额。最后，它结束了数据库操作。

示例5-1创建数据库操作

VAR eid NUMBER EXEC：eid：= DBMS_SQL_MONITOR.BEGIN_OPERATION（'DBOP_EXAMPLE'）;声明 -  v1号码; -  CURSOR c1 IS SELECT cust_city FROM（SELECT COUNT（*）cnt，cust_city FROM sh.customers GROUP BY cust_city ORDER BY 1 desc）; - 开始为我在c1循环 -  v1：= 0; -  SELECT MAX（amount_sold）INTO v1 FROM sh.sales WHERE cust_id IN（select cust_id FROM sh.customers WHERE cust_city = i.cust_city）; -  DBMS_OUTPUT.PUT_LINE（'Amount：'|| v1）; - 结束循环; -  结束; / SELECT MAX（asld）FROM（SELECT MAX（amount_sold）asld，cust_id FROM sh.sales WHERE cust_id IN（SELECT cust_id FROM sh.customers WHERE cust_city IN（SELECT cust_city FROM（SELECT count（*）cnt，cust_city FROM sh.customers GROUP BY cust_city HAVING COUNT（*）> 1）））GROUP BY cust_id）/ EXEC DBMS_SQL_MONITOR.END_OPERATION（'DBOP_EXAMPLE'，：eid）;

5.3监控云控制中的数据库操作

Cloud Control的“监视的SQL执行”页面显示当前正在运行或已完成的数据库操作的表。

通过从Top 100 By列表中进行选择，您可以按操作的各个方面（例如Last Active Time，Duration或CPU Time）对表的行进行排序。通过“类型”列表中的选择，您可以显示所有操作或过滤行以仅显示SQL，PL / SQL或数据库操作。

该表包含有关操作的数据，例如操作的状态和类型，操作ID，消耗的数据库时间，SQL文本（如果适用）等。某些列中的值是指向其他页面的链接或当光标指向它们时显示信息。例如，ID列中的值是指向“受监视的SQL执行详细信息”页面的链接。另一个示例是，当光标指向“数据库时间”列中的栏时，将显示一条上下文消息，其中显示等待类，时间量和数据库时间百分比等信息。例如，指向“数据库时间”列中的栏会显示一条消息，例如CPU: 2.8m (92%) 。从表中选择行时，表上方的“执行详细信息”，“SQL详细信息”和“会话详细信息”控件将变为活动状态。

注意：

从Oracle Database 19c开始，没有管理权限的数据库用户也可以通过导航到Cloud Control的Monitored SQL Executions页面来查看其SQL语句的执行计划和性能指标。
