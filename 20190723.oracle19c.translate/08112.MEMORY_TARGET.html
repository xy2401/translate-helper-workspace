1.197 MEMORY_TARGET

MEMORY_TARGET指定Oracle系统范围的可用内存。 数据库将内存调整为MEMORY_TARGET值，根据需要减小或放大SGA和PGA。
属性 描述

参数类型

大整数

句法

MEMORY_TARGET = integer [K | M | G]

默认值

0 （对DEFERRED模式自动调整请求禁用SGA自动调整，但允许进行IMMEDIATE模式自动调整请求）

可修改

改变系统

可在PDB中修改

没有

价值范围

152 MB到MEMORY_MAX_TARGET

基本

没有

MEMORY_TARGET应设置为高于或等于SGA和PGA当前大小的总和。

在基于文本的初始化参数文件，如果省略MEMORY_MAX_TARGET ，并包括一个价值MEMORY_TARGET ，则数据库会自动设置MEMORY_MAX_TARGET到的值MEMORY_TARGET 。如果省略行MEMORY_TARGET ，包括一个值MEMORY_MAX_TARGET时， MEMORY_TARGET参数的默认值为零。启动后，你就可以动态地改变MEMORY_TARGET为非零值，只要不超过价值MEMORY_MAX_TARGET 。

总内存使用量可能超过MEMORY_TARGET的值。例如，存储器被分配给PL / SQL表，并且无论上述值的可变数组MEMORY_TARGET只要存储器可在操作系统级别。

在默认值字段中， IMMEDIATE模式自动调整请求是必要的，以避免ORA-04031错误。DEFERRED和IMMEDIATE模式反映在V$MEMORY_RESIZE_OPS视图的OPER_MODE列中。

注意：

默认值SGA_MAX_SIZE取决于值MEMORY_TARGET和MEMORY_MAX_TARGET 。
