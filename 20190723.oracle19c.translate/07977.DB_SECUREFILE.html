1.91 DB_SECUREFILE

DB_SECUREFILE指定是否将LOB文件视为SecureFiles。

属性 描述

参数类型

串

句法

DB_SECUREFILE = {NEVER |允许|首选|总是|忽视 }

默认值

如果COMPATIBLE初始化参数设置为11.2.0.1,11.2.0.2或11.2.0.3，则为PERMITTED;如果COMPATIBLE初始化参数设置为12.0.0.0或更高，则为PREFERRED

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

基本

没有

值

  • 决不

    任何指定为SecureFiles的LOB都将创建为BasicFiles LOB。所有特定于SecureFiles的存储选项和功能（例如，压缩，加密，重复数据删除）都将导致异常。BasicFiles LOB默认值将用于未指定的存储选项。

  • 许可

    允许将LOB创建为SecureFiles。

  • 具体

    除非在LOB存储子句中显式指定BASICFILE或表空间是手动段空间管理表空间，否则所有LOB都将创建为SecureFiles。设置PREFERRED ，将忽略BASICFILE以其他方式从分区或列级别LOB存储继承的情况; LOB将改为创建为SecureFiles。

  • 总是

    尝试将所有LOB创建为SecureFiles LOB，但创建不在自动段空间管理（ASSM）表空间中作为BasicFiles LOB的任何LOB，除非明确指定了SECUREFILE 。将忽略指定的任何BasicFiles LOB存储选项，并且SecureFiles LOB默认值将用于未指定的所有存储选项。

  • 忽视

    SECUREFILE关键字和所有SecureFiles选项都将被忽略。

如果COMPATIBLE初始化参数未设置为11.1.0或更高，则LOB不会被视为SecureFiles。

如果有一个带有两个分区的LOB列（一个具有为其启用ASSM的表空间，另一个具有未启用ASSM的表空间），则具有启用ASSM的表空间的分区中的LOB将被视为SecureFiles另一个分区中的LOB将被视为BasicFiles。

如果应用程序没有与SecureFiles的认证，该值DB_SECUREFILE初始化参数可以设置为PERMITTED获得的Oracle数据库11 克默认行为。
