D SQL * Plus Instant Client

SQL * Plus Instant Client是一个独立的产品，具有以下所有功能 SQL * Plus命令行。它连接到现有的远程Oracle数据库，但不包括自己的数据库。它易于安装，并且占用的磁盘空间明显少于完整磁盘空间使用SQL * Plus命令行需要Oracle Database Client安装。

SQL * Plus Instant Client可在支持的平台上使用 OCI即时客户端。有关OCI Instant Client的详细信息，请参阅关于OCI Instant Client。

至安装SQL * Plus Instant Client，需要两个包：

  • SQL * Plus Instant Client包。

  • Basic OCI Instant Client软件包或轻量级OCI Instant Client软件包。

D.1关于选择要安装的SQL * Plus Instant Client

SQL * Plus Instant Client可以通过两种方式安装：

  • 下载来自Oracle技术网（OTN）的软件包。

  • 复制来自Oracle Database 19 c Client Administrator安装的软件包中的相同文件。

这俩 SQL * Plus和OCI包必须来自相同的Oracle数据库版本。

D.1.1基本Instant Client

使用Basic OCI包的SQL * Plus Instant Client适用于Oracle数据库支持的任何NLS_LANG设置。它支持Oracle数据库中可用的所有字符集和语言设置。

D.1.2轻量级即时客户端

SQL * Plus Instant Client 使用轻量级OCI包仅显示英文错误消息，仅支持特定字符集。它比使用Basic OCI包的SQL * Plus Instant Client要小得多。

有效值轻量级Instant Client的NLS_LANG参数为：

  • language可以是Oracle数据库支持的任何有效语言，但是，错误消息仅以英语报告。

  • 区域可以是Oracle数据库支持的任何有效区域。

  • charset是以下字符集之一：

    • US7ASCII

    • WE8DEC

    • WE8MSWIN1252

    • WE8ISO8859P1

    • UTF8

    • AL16UTF16

    • AL32UTF8

例如：

NLS_LANG = AMERICAN_AMERICA.UTF8

有关NLS设置的详细信息，请参阅设置全球化支持环境和NLS_LANG环境变量 。

D.1.2.1具有不支持的字符集的轻量级SQL * Plus即时客户端错误

试图开始具有不受支持的字符集的SQL * Plus Instant Client将失败，并显示以下错误：

错误5初始化SQL * Plus NLS初始化错误

D.2关于通过从OTN下载来安装SQL * Plus Instant Client

OTN下载对于Linux是RPM包。UNIX和Windows的OTN下载是zip文件。

永远不应在ORACLE_HOME上安装SQL * Plus Instant Client软件包。

D.2.1从Linux RPM软件包安装SQL * Plus Instant Client

  1. 下载RPM包包含SQL * Plus Instant Client软件包，以及OTN Instant Client页面中的OCI软件包。两个软件包必须来自同一版本。
  2. 使用rpm -i进行RPM软件包的初始安装，或使用rpm -u升级到更新版本的软件包。
  3. 配置SQL * Plus Instant Client。请参见关于配置SQL * Plus Instant Client 。

D.2.2从UNIX或Windows Zip文件安装SQL * Plus Instant Client

  1. 下载zip文件包含SQL * Plus Instant Client软件包，以及OTN Instant Client页面中的OCI软件包。两个软件包必须来自同一版本。
  2. 创建一个新目录，例如UNIX上的/ home / instantclient12_2或Windows上的c：\ instantclient12_2 。
  3. 将两个包解压缩到新目录中。
  4. 配置SQL * Plus Instant Client。请参见关于配置SQL * Plus Instant Client 。

D.3从19c客户端版本媒体安装SQL * Plus Instant Client

  1. 在Oracle Database 19 c Client Release介质上运行安装程序并选择管理员选项。
  2. 在UNIX和Linux上创建新目录，例如/ home / instantclient_19_1 ，在Windows上创建c：\ instantclient_19_1 。
  3. 将SQL * Plus Instant Client和OCI Instant Client文件复制到新目录。必须从同一ORACLE_HOME复制所有文件。

    请参阅在UNIX或Linux上 安装SQL * Plus Instant Client或在Windows上安装SQL * Plus Instant Client以获取要复制的文件的列表。

  4. 配置SQL * Plus Instant Client。请参见关于配置SQL * Plus Instant Client 。

D.3.1在UNIX或Linux上安装SQL * Plus Instant Client

使用Basic OCI软件包安装SQL * Plus Instant Client 在UNIX和Linux上，复制以下文件：

$ ORACLE_HOME / instantclient / libociei.so $ ORACLE_HOME / lib / libclntsh.so.19.1 $ ORACLE_HOME / lib / libnnz19.so $ ORACLE_HOME / lib / libsqlplus.so $ ORACLE_HOME / lib / libsqlplusic.so $ ORACLE_HOME / bin / sqlplus

要在UNIX和Linux上使用轻量级OCI软件包安装SQL * Plus Instant Client，请复制以下文件：

$ ORACLE_HOME / instantclient / light / libociicus.so $ ORACLE_HOME / lib / libclntsh.so.19.1 $ ORACLE_HOME / lib / libnnz19.so $ ORACLE_HOME / lib / libsqlplus.so $ ORACLE_HOME / lib / libsqlplusic.so $ ORACLE_HOME / bin / sqlplus

D.3.2在Windows上安装SQL * Plus Instant Client

使用Basic OCI软件包安装SQL * Plus Instant Client 在Windows上，复制以下文件：

ORACLE_HOME \ instantclient \ oraociei12.dll ORACLE_HOME \ bin \ oci.dll ORACLE_HOME \ bin \ orannzsbb12.dll ORACLE_HOME \ bin \ orasqlplusic12.dll ORACLE_HOME \ bin \ sqlplus.exe

要在Windows上使用轻量级OCI软件包安装SQL * Plus Instant Client，请复制以下文件：

ORACLE_HOME \ instantclient \ light \ oraociicus12.dll ORACLE_HOME \ bin \ oci.dll ORACLE_HOME \ bin \ orannzsbb12.dll ORACLE_HOME \ bin \ orasqlplusic12.dll ORACLE_HOME \ bin \ sqlplus.exe

D.4关于配置SQL * Plus Instant Client

SQL * Plus Instant Client可执行文件只能与OCI Instant Client的匹配版本一起使用。

请注意，不需要设置ORACLE_HOME或ORACLE_SID环境变量。

D.4.1在Linux上配置SQL * Plus Instant Client（来自RPM）

从OTN下载的RPM安装到/ usr文件系统中的Oracle特定子目录中。子目录结构允许多个版本的Instant Client可用。

  1. 将包含Instant Client库的目录的名称添加到LD_LIBRARY_PATH。删除所有其他Oracle目录。

    例如，要在Bourne或Korn shell中的Solaris上设置LD_LIBRARY_PATH：

    LD_LIBRARY_PATH = / usr / lib / oracle / 19.1 / client / lib：$ {LD_LIBRARY_PATH} export LD_LIBRARY_PATH
  2. 确保从RPM安装的sqlplus可执行文件是第一个在PATH中找到的。为了测试中，输入sqlplus中应返回的/ usr / bin中/ sqlplus的 。如果没有，请从PATH中删除任何其他Oracle目录，或将/ usr / bin放在PATH中的其他SQL * Plus可执行文件之前，或使用绝对或相对路径启动SQL * Plus。

    例如，要在bash shell中设置PATH：

    PATH = / usr / bin：$ {PATH}导出PATH

    如果安装多个版本的SQL * Plus，则可能需要将符号链接/ usr / bin / sqlplus更改为与LD_LIBRARY_PATH中的库匹配的SQL * Plus版本。对于19.1， / usr / bin / sqlplus是/usr/lib/oracle/19.1/client/bin/sqlplus中 SQL * Plus二进制文件的符号链接。

  3. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。有关更多信息，请参阅区域设置数据

    例如：

    NLS_LANG = AMERICAN_AMERICA.UTF8导出NLS_LANG

D.4.2在Linux（从客户端媒体或Zip文件）和UNIX上配置SQL * Plus Instant Client

  1. 将包含Instant Client文件的目录的名称添加到相应的共享库路径LD_LIBRARY_PATH，LIBPATH或SHLIB_PATH。删除所有其他Oracle目录。

    例如，在Bourne或Korn shell中的Solaris上：

    LD_LIBRARY_PATH = / home / instantclient_19_1：$ {LD_LIBRARY_PATH} export LD_LIBRARY_PATH
  2. 将包含Instant Client文件的目录添加到PATH环境变量中。如果未设置，则必须使用绝对路径或相对路径来启动SQL * Plus。从PATH中删除任何其他Oracle目录。例如：
    PATH = / home / instantclient_19_1：$ {PATH}导出PATH
  3. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。有关更多信息，请参阅区域设置数据

    例如：

    NLS_LANG = AMERICAN_AMERICA.UTF8导出NLS_LANG

D.4.3在Windows上配置SQL * Plus Instant Client

可以使用Windows命令提示符中的SET命令配置环境，也可以通过在“系统属性”中设置“环境变量”使其成为永久环境。

例如，要使用“系统属性”在Windows 2000中设置环境变量，请从“控制面板”中打开“系统”，单击“高级”选项卡，然后单击“环境变量”。

  1. 将包含Instant Client文件的目录添加到PATH系统环境变量中。从PATH中删除任何其他Oracle目录。

    例如，将c：\ instantclient12_2添加到PATH的开头。

  2. 设置您的语言环境所需的Oracle全球化变量。如果未设置变量，则将假定默认语言环境。有关更多信息，请参阅区域设置数据例如，要为日语环境设置NLS_LANG，请创建设置为JAPANESE_JAPAN.JA16EUC的用户环境变量NLS_LANG。

    如果已安装轻量级Instant Client，请参阅轻量级Instant Client以获取有关支持的NLS_LANG设置的信息。

D.5关于使用SQL * Plus Instant Client连接数据库

SQL * Plus Instant Client始终与任何数据库服务器“远程”。要连接到数据库，必须使用Oracle Net连接标识符指定数据库。

使用Easy Connection标识符连接到mymachine上运行的MYDB数据库中的HR模式的示例是：

SQLPLUS HR / your_password @ \ “MYDOMAIN // mymachine上 ： 端口 / MYDB \”。

或者，您可以使用网络服务名称：

sqlplus hr / your_password @MYDB

网络服务名称可以存储在许多地方，包括LDAP。建议使用LDAP来利用Oracle Database 12c的新功能。有关更多信息，请参阅tnsnames.ora文件中的本地命名参数 。

如果要使用在本地Oracle Net tnsnames.ora文件中配置的网络服务名称，请将环境变量TNS_ADMIN设置为包含tnsnames.ora文件的目录。例如，在UNIX上，如果您的tnsnames.ora文件位于/ home / user1中，并且它定义了网络服务名称MYDB2：

TNS_ADMIN = / home / user1导出TNS_ADMIN sqlplus hr @ MYDB2

如果未设置TNS_ADMIN，则检查依赖于操作系统的目录集以查找tnsnames.ora 。此搜索路径包括查找由network / admin / tnsnames.ora的ORACLE_HOME环境变量指定的目录。这是为SQL * Plus Instant Client设置ORACLE_HOME环境变量的唯一原因。如果在运行Instant Client应用程序时设置了ORACLE_HOME，则必须将其设置为存在的目录。

此示例假定已设置ORACLE_HOME环境变量，并且$ ORACLE_HOME / network / admin / tnsnames.ora或ORACLE_HOME \ network \ admin \ tnsnames.ora文件定义网络服务名称MYDB3：

sqlplus hr @ MYDB3

可以将TWO_TASK（在UNIX上）或LOCAL（在Windows上）环境变量设置为连接标识符。这样就无需在SQL * Plus或SQL * Plus Instant Client中建立连接时显式输入连接标识符。此UNIX示例连接到称为MYDB4的数据库：

TNS_ADMIN = / home / user1 export TNS_ADMIN TWO_TASK = MYDB4 export TWO_TASK sqlplus hr

在Windows上，可以在“系统属性”中设置TNS_ADMIN和LOCAL。请参阅在Windows上配置SQL * Plus Instant Client 。

D.6 AS SYSDBA或AS SYSOPER与SQL * Plus Instant Client的连接

要连接AS SYSDBA或AS SYSOPER以执行DBA任务，您需要使用数据库的orapwd实用程序在数据库服务器上设置Oracle密码文件。配置完成后，SQL * Plus Instant Client连接字符串可能如下所示：

sqlplus sys @ MYDB AS SYSDBA

有关Oracle密码文件的信息，请参阅使用密码文件验证 。

D.7关于卸载Instant Client

SQL * Plus Instant Client软件包可以与OCI Instant Client分开删除。卸载SQL * Plus Instant Client软件包后，剩余的OCI Instant Client库支持自定义编写的OCI程序或第三方数据库实用程序以连接到数据库。

D.7.1卸载SQL * Plus Instant Client

  1. 对于Linux上从RPM软件包安装，请仅在SQL * Plus Instant Client软件包上使用rpm -e

    要么

    对于UNIX和Windows上的安装以及来自客户端发行介质的Linux上的安装，请手动删除以下SQL * Plus特定文件：

    表D-4 SQL * Plus包中的Instant Client文件

    UNIX 视窗 描述

    SQLPLUS

    SQLPLUS.EXE

    SQL * Plus可执行文件

    libsqlplus.so

    不适用

    SQL * Plus库

    libsqlplusic.so

    orasqlplusic12.dll

    SQL * Plus数据共享库
  2. 重置环境变量并在必要时删除tnsnames.ora 。

D.7.2卸载Complete Instant Client

  1. 对于Linux上从RPM软件包安装，使用rpm -qa查找SQL * Plus Instant Client和Basic OCI软件包名称并运行rpm -e以删除它们

    要么

    对于UNIX和Windows上的安装以及来自客户端发行介质的Linux上的安装，请手动删除包含SQL * Plus可执行文件和Oracle库的目录。

    有关您复制到目录的文件的列表，请参阅从12c客户端版本介质安装SQL * Plus Instant Client 。

  2. 重置环境变量，如PATH，SQLPATH，LD_LIBRARY_PATH和TNS_ADMIN。
  3. 如有必要，删除tnsnames.ora 。
