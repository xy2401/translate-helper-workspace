1.1 Oracle自治健康框架问题和解决方案空间

Oracle Autonomous Health Framework可帮助监控，诊断和预防可用性和性能问题。

系统管理员可以在安装，修补和升级期间以交互方式使用Oracle Autonomous Health Framework中的大多数组件。数据库管理员可以使用Oracle Autonomous Health Framework来诊断操作运行时问题并减轻这些问题的影响。

  • 可用性问题
    可用性问题是运行时问题，威胁到软件堆栈的可用性。
  • 性能问题
    性能问题是威胁系统性能的运行时问题。

1.1.1可用性问题

可用性问题是运行时问题，威胁到软件堆栈的可用性。

可用性问题可能来自软件问题（Oracle数据库，Oracle Grid Infrastructure，操作系统）或底层硬件资源（CPU，内存，网络，存储）。

Oracle Autonomous Health Framework中的组件解决了以下可用性问题：

服务器可用性问题示例

服务器可用性问题可能导致服务器从群集中逐出，并关闭服务器上运行的所有数据库实例。

此类问题的示例如下：

  • 问题：服务器耗尽空闲物理内存导致的内存压力导致操作系统Swapper进程长时间运行将内存移动到磁盘。交换可防止时间关键的集群进程运行并最终导致节点被驱逐。

    解决方案： Memory Guard会提前检测内存压力并导致工作耗尽以释放内存。

  • 问题：专用互连上的网络拥塞可能导致时间关键的节点间或存储I / O具有过多的延迟或丢弃的数据包。这种类型的故障通常会累积，可以及早发现，纠正或缓解。

    解决方案：如果服务器配置中的更改导致此问题，则群集验证实用程序（CVU）会在问题持续一个多小时后检测到它。但是，Oracle Cluster Health Advisor会在几分钟内检测到该问题并提供纠正措施。

  • 问题：由拉线或网络接口卡（NIC）引起的专用互连网络故障可能会立即导致被逐出的节点。

    解决方案：虽然无法及早检测到这些类型的网络故障，但可以通过使用Cluster Health Monitor和Oracle Trace File Analyzer来查明故障时间和所涉及的网络接口，从而缩小原因。

数据库可用性问题示例

数据库可用性问题可能导致Oracle数据库或数据库的某个实例无响应，从而无法供用户使用。

此类问题的示例如下：

  • 问题：失控查询或挂起可以将关键数据库资源（例如锁，锁存器或CPU）拒绝到其他会话。拒绝关键数据库资源会导致数据库或数据库实例对应用程序无响应。

    解决方案： Hang Manager检测并自动解决这些类型的挂起。此外，Oracle Cluster Health Advisor还会检测，识别并通知数据库管理员此类挂起并提供适当的更正操作。

  • 问题：拒绝服务（DoS）攻击，漏洞或简单的软件错误可能导致数据库或数据库实例无响应。

    解决方案： Oracle ORAchk提供的已知问题及其解决方案的主动建议可以防止此类事件发生。如果不阻止这些问题，那么除了Cluster Health Monitor收集的数据之外，Oracle Trace File Analyzer自动收集日志可以加快纠正这些问题。

  • 问题：配置更改可能导致难以排除故障的数据库中断。例如，对oracle.bin文件的不正确权限可能会阻止创建会话进程。

    解决方案：使用Cluster Verification Utility和Oracle ORAchk加速识别和纠正这些类型的问题。您可以使用Oracle ORAchk生成差异报告，以查看两个报告的基线比较和差异列表。您还可以查看Cluster Verification Utility创建的配置报告，以验证您的系统是否满足Oracle安装的条件。

1.1.2性能问题

性能问题是威胁系统性能的运行时问题。

性能问题可能由软件问题（错误，配置问题，数据争用等）或客户端问题（需求，查询类型，连接管理等）引起。

服务器和数据库性能问题交织在一起，难以分离。根据它们的来源对它们进行分类更容易：数据库服务器或客户端。

数据库服务器性能问题的示例

  • 问题：偏离配置中的最佳实践可能会导致数据库服务器性能问题。

    解决方案： Oracle ORAchk会在Oracle ORAchk定期运行时检测配置问题，并通知数据库管理员相应的更正设置。

  • 问题：瓶颈资源或构造不良的SQL语句可能导致数据库服务器性能问题。

    解决方案： Oracle数据库服务质量（QoS）管理标记这些问题，并在问题使服务级别协议（SLA）面临风险时生成通知。Oracle Cluster Health Advisor会检测问题何时超出正常运行条件，并通知数据库管理员采取纠正措施。

  • 问题：会话可能导致其他会话减慢等待阻塞会话释放其资源或完成其工作。

    解决方案： Hang Manager检测到这些会话链并自动终止root持有者会话以缓解瓶颈。

  • 问题：未解决的已知问题或未修补的错误可能导致数据库服务器性能问题。

    解决方案：可以通过自动Oracle ORAchk报告检测这些问题，并使用相关修补程序或解决方法进行标记。Oracle ORAchk定期得到增强，可以在现有产品或新产品领域中包含新的关键问题。

数据库客户端引起的性能问题示例

  • 问题：当服务器托管的数据库实例多于其资源且客户端负载可以管理时，性能会因等待CPU，I / O或内存而受到影响。

    解决方案： Oracle ORAchk和Oracle数据库QoS管理检测这些问题何时是错误配置的结果，例如超额订购CPU，内存或后台进程。Oracle ORAchk和Oracle数据库QoS管理系统会通过纠正措施通知您。

  • 问题：配置错误的参数（如SGA和PGA分配，会话或进程数，CPU计数等）可能导致数据库性能下降。

    解决方案： Oracle ORAchk和Oracle Cluster Health Advisor分别检测设置和结果，并通过建议的纠正措施自动通知您。

  • 问题：客户端连接的激增可能会超出服务器或数据库容量，从而导致超时错误和其他性能问题。

    解决方案： Oracle数据库QoS管理和Oracle Cluster Health Advisor自动检测性能下降。此外，Oracle数据库QoS管理和Oracle Cluster Health Advisor会通过纠正措施通知您，以缓解瓶颈并恢复性能。
