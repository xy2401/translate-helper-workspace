3.1什么是数据泵导入？

数据泵导入（以下简称为“导入”）是一种用于将导出转储文件集加载到目标系统的实用程序。

转储文件集由一个或多个包含表数据，数据库对象元数据和控制信息的磁盘文件组成。文件以专有的二进制格式编写。在导入操作期间，数据泵导入实用程序使用这些文件来查找转储文件集中的每个数据库对象。

Import也可用于直接从源数据库加载目标数据库，而无需插入转储文件。这称为网络导入。

通过数据泵导入，您可以指定作业是否应从转储文件集或源数据库（在网络导入的情况下）移动数据和元数据的子集（由导入模式确定）。这是使用数据过滤器和元数据过滤器完成的，这些过滤器通过导入命令实现。

3.2.2.1关于数据泵导入模式

了解数据泵导入模式在导入过程中的运行方式。

您为导入指定的数据泵导入模式适用于操作源。如果指定NETWORK_LINK参数，则该源是转储文件集或其他数据库。

当导入操作的源是转储文件集时，指定模式是可选的。如果未指定模式，则“导入”会尝试在运行导出操作的模式下加载整个转储文件集。

使用适当的参数在命令行上指定模式。

注意：

导入由完整模式导出创建的转储文件时，导入操作会尝试从源数据库复制SYS帐户的密码。此副本有时会失败（例如，如果密码位于共享密码文件中）。如果失败，则导入完成后，必须将目标数据库中SYS帐户的密码设置为您选择的密码。

3.2.2.2完全导入模式

使用FULL参数指定完全导入。

在完全导入模式下，源（转储文件集或其他数据库）的整个内容将加载到目标数据库中。此模式是基于文件的导入的默认模式。如果源是另一个包含您自己的模式的数据库，那么您必须具有DATAPUMP_IMP_FULL_DATABASE角色。

不为非特权用户导入跨模式引用。例如，不导入在导入用户的模式中的表上定义但位于另一个用户模式中的触发器。

目标数据库上需要DATAPUMP_IMP_FULL_DATABASE角色。如果NETWORK_LINK参数用于完全导入，则源数据库上需要DATAPUMP_EXP_FULL_DATABASE角色

在完全模式导入期间使用可传输选项

您可以在完整模式导入期间使用可传输选项来执行完全可传输导入。

基于网络的完全可传输导入需要使用FULL=YES ， TRANSPORTABLE=ALWAYS和TRANSPORT_DATAFILES= datafile_name参数。

基于文件的完全可传输导入仅需要使用TRANSPORT_DATAFILES= datafile_name参数。数据泵导入推断出TRANSPORTABLE=ALWAYS和FULL=Y参数的存在。

执行完全可传输导入时有几个要求：

  • 您还必须指定NETWORK_LINK参数，或者必须在导出期间使用transportable选项创建正在导入的转储文件集。

  • 如果使用网络链接，则NETWORK_LINK参数上指定的数据库必须是Oracle Database 11g第 2版（11.2.0.3）或更高版本，并且Data Pump VERSION参数必须至少设置为12。（在非网络导入中，从转储文件中隐式确定VERSION=12 ）

  • 如果源平台和目标平台具有不同的字节顺序，则必须转换正在传输的数据，使其采用目标平台的格式。您可以使用DBMS_FILE_TRANSFER包或RMAN CONVERT命令来转换数据。

  • 如果源平台和目标平台不具有相同的字节顺序，则在网络模式或转储文件模式下不支持加密表空间的完全可传输导入

3.2.2.3架构模式

使用SCHEMAS参数指定模式导入。

在模式导入中，仅加载由指定模式拥有的对象。源可以是完整，表，表空间或模式模式导出转储文件集或其他数据库。如果您具有DATAPUMP_IMP_FULL_DATABASE角色，则可以指定模式列表，并且除了这些模式中包含的对象之外，还会在数据库中创建模式本身（包括系统特权授予）。

除非将其他架构重新映射到当前架构，否则不会为非特权用户导入跨架构引用。例如，不导入在导入用户的模式中的表上定义但位于另一个用户的模式中的触发器。

3.2.2.4表格模式

使用TABLES参数指定表模式导入。

使用TABLES参数指定表模式导入。在表模式下，仅加载指定的一组表，分区及其依赖对象。源可以是完整，架构，表空间或表模式导出转储文件集或其他数据库。您必须具有DATAPUMP_IMP_FULL_DATABASE角色才能指定不在您自己的架构中的表。

通过使用TABLES参数指定TRANPORTABLE=ALWAYS参数，可以在表模式导入期间使用transportable选项。如果您使用此选项，则还必须使用NETWORK_LINK参数。

要恢复表和表分区，还可以使用RMAN备份和RMAN RECOVER TABLE命令。在此过程中，RMAN创建（并可选地导入）包含已恢复对象的数据泵导出转储文件。

3.2.2.6可传输表空间模式

使用TRANSPORT_TABLESPACES参数指定可传输表空间导入。

在可传输表空间模式下，使用数据库链接（使用NETWORK_LINK参数指定）或通过指定包含元数据的转储文件来加载来自另一个数据库的元数据。必须从源系统提供由TRANSPORT_DATAFILES参数指定的实际数据文件，以便在目标数据库中使用，通常是将它们复制到目标系统。

执行可传输作业时，最佳做法是在源系统上保留数据文件的副本，直到在目标系统上成功完成导入作业。如果导入作业由于某种原因失败，您仍将拥有未损坏的数据文件副本。

此模式需要DATAPUMP_IMP_FULL_DATABASE角色。

注意：

您无法导出可传输的表空间，然后将它们导入到较低版本级别的数据库中。目标数据库必须与源数据库处于相同或更高版本级别。

3.2.3网络注意事项

启动数据泵导入实用程序时，可以在连接字符串中指定连接标识符。

连接标识符可以指定与当前Oracle系统ID（SID）标识的当前实例不同的数据库实例。连接标识符可以是Oracle * Net连接描述符或映射到连接描述符的网络服务名称（通常在tnsnames.ora文件中定义）。使用连接标识符要求您运行Oracle Net Listener（要启动默认侦听器，请输入lsnrctl start ）。以下是此类连接的示例，其中inst1是连接标识符：

impdp hr @ inst1 DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp TABLES =员工

然后导入会提示您输入密码：

密码： 密码

本地Import客户端连接到由连接标识符inst1 （网络服务名称）标识的数据库实例，并将数据从转储文件hr.dmp到inst1 。

启动“导入”实用程序时指定连接标识符与使用NETWORK_LINK参数执行导入操作不同。当您启动导入操作并指定连接标识符时，本地Import客户端将连接到由connect标识符标识的数据库实例，并将数据从命令行上指定的转储文件导入该数据库实例。

然而，当您使用NETWORK_LINK参数执行导入时，导入是使用数据库链接执行的，并且不涉及转储文件。（数据库链接是两个物理数据库服务器之间的连接，允许客户端将它们作为一个逻辑数据库进行访问。）

3.3导入操作期间的过滤

Data Pump Import提供数据和元数据过滤功能，可以帮助您限制导入的信息类型。

  • 数据过滤器
    特定于数据的过滤是通过QUERY和SAMPLE参数实现的，这些参数指定了要导入的表行的限制。
  • 元数据过滤器
    元数据过滤通过EXCLUDE和INCLUDE参数实现。

父主题： 数据泵导入

3.3.1数据过滤器

特定于数据的过滤是通过QUERY和SAMPLE参数实现的，这些参数指定了要导入的表行的限制。

由于元数据过滤，数据过滤也可以间接发生，元数据过滤可以包括或排除表对象以及任何关联的行数据。

每个数据过滤器只能为每个表指定一次，每个作业只能指定一次。如果使用相同名称的不同过滤器同时应用于特定表和整个作业，则为特定表提供的过滤器参数优先。

父主题： 导入操作期间的筛选

3.3.2元数据过滤器

元数据过滤通过EXCLUDE和INCLUDE参数实现。

Data Pump Import提供了比原始Import实用程序提供的更大的元数据过滤功能。EXCLUDE和INCLUDE参数是互斥的。

元数据过滤器标识要包含在数据泵操作中或从数据泵操作中排除的一组对象。例如，您可以请求完全导入，但不包括包规范或包体。

要正确使用过滤器并获得预期的结果，请记住已识别对象的相关对象与标识的对象一起处理 。例如，如果过滤器指定要将包包含在操作中，则还将包括对该包的授予。同样，如果过滤器排除了表，则过滤器也会排除表上的索引，约束，授权和触发器。

如果为对象类型指定了多个过滤器，则会对它们应用隐式AND操作。也就是说，参与作业的对象必须传递应用于其对象类型的所有过滤器。

可以在作业中多次指定相同的过滤器名称。

要查看有效的对象类型的列表，查询以下观点： DATABASE_EXPORT_OBJECTS全模式， SCHEMA_EXPORT_OBJECTS的架构模式， TABLE_EXPORT_OBJECTS表和表空间模式。OBJECT_PATH列中列出的值是有效的对象类型。请注意，完整对象路径名由导出模式确定，而不是由导入模式确定。

3.4.3 ACCESS_METHOD

Data Pump Import命令行模式ACCESS_METHOD参数指示Import使用特定方法加载数据

默认值： AUTOMATIC

目的

指示Import使用特定方法加载数据。

语法和说明

ACCESS_METHOD = [AUTOMATIC | DIRECT_PATH | EXTERNAL_TABLE | CONVENTIONAL_PATH | INSERT_AS_SELECT]

提供了ACCESS_METHOD参数，以便在默认方法由于某种原因不起作用时可以尝试替代方法。如果无法使用指定的访问方法加载表的数据，则数据将显示表的错误并继续下一个工作项。

可用选项包括：

  • AUTOMATIC ：此访问方法是默认方法。Data Pump确定为每个表加载数据的最佳方式。Oracle建议您尽可能使用AUTOMATIC ，因为它使Data Pump能够自动选择最有效的方法。

  • DIRECT_PATH ：Data Pump对每个表使用直接路径加载。

  • EXTERNAL_TABLE ：Data Pump在存储在转储文件中的数据上创建外部表，并使用SQL INSERT AS SELECT语句将数据加载到表中。Data Pump将APPEND提示应用于INSERT语句。

  • CONVENTIONAL_PATH ：Data Pump在存储在转储文件中的数据上创建外部表，并一次从外部表中读取行。每次读取一行时，Data Pump都会执行一个insert语句，将该行加载到目标表中。此方法需要很长时间才能加载数据，但这是加载直接路径和外部表无法加载的数据的唯一方法。

  • INSERT_AS_SELECT ：Data Pump通过执行SQL INSERT AS SELECT语句来加载表，该语句从远程数据库中选择数据并将其插入目标表。此选项仅适用于网络模式导入。它用于在数据通过网络移动时禁用DIRECT_PATH 。

限制

  • 网络模式导入的有效选项是AUTOMATIC ， DIRECT_PATH和INSERT_AS_SELECT 。

  • 从转储文件导入时唯一有效的选项是AUTOMATIC ， DIRECT_PATH ， EXTERNAL_TABLE和CONVENTIONAL_PATH

  • 要将ACCESS_METHOD参数与网络导入一起使用，您必须使用Oracle Database 12 c第2版（12.2.0.1）或更高版本

  • Data Pump Import的ACCESS_METHOD参数对可传输表空间作业无效。

例

以下示例允许Data Pump同时为预先存在的表SALES多个分区加载数据。 

impdp hr SCHEMAS = hr DIRECTORY = dpump_dir1 LOGFILE = schemas.log DUMPFILE = expdat.dmp ACCESS_METHOD = CONVENTIONAL

3.4.4 ATTACH

Data Pump Import命令行模式ATTACH参数将客户端会话附加到现有导入作业，并自动将您置于交互式命令模式。

目的

将客户端会话附加到现有导入作业，并自动将您置于交互式命令模式。

ATTACH [= [ schema_name 。] job_name ]

此示例假定hr模式中存在名为import_job的作业。

  • DISABLE_APPEND_HINT ：指定您不希望导入操作在加载数据对象时使用APPEND提示。禁用APPEND提示可用于解决重复数据。例如，当有一DISABLE_APPEND_HINT要加载的数据对象存在于数据库中时，可以使用DISABLE_APPEND_HINT ，而某些其他应用程序可以同时访问一个或多个数据对象。

    DISABLE_APPEND_HINT ：更改默认行为，以便APPEND提示不用于加载数据对象。未设置时，默认设置是使用APPEND提示加载数据对象。

  • GROUP_PARTITION_TABLE_DATA ：告诉Oracle Data Pump将表的所有分区中的表数据作为一个操作导入。默认行为是将每个表分区作为单独的操作导入。导入选择默认值。例如，当设置此分区时，作为导入的一部分，作为加载表的一部分，表可能会移动到不同的分区，Oracle Data Pump将表数据分组到一个分区中。Oracle Data Pump还将表的所有分区组合为一个由Import操作创建的表的操作。

  • REJECT_ROWS_WITH_REPL_CHAR ：指定您希望导入操作拒绝任何遇到数据丢失的行，因为在字符集转换期间使用了默认替换字符。

    如果未设置REJECT_ROWS_WITH_REPL_CHAR ，则默认行为是使用替换字符加载已转换的行。

  • SKIP_CONSTRAINT_ERRORS ：影响在加载数据对象（表，分区或子分区）时如何处理非延迟约束违规。

    如果遇到延迟约束违规，则SKIP_CONSTRAINT_ERRORS对负载没有影响。延迟约束违规总是导致整个负载回滚。

    SKIP_CONSTRAINT_ERRORS选项指定您希望导入操作继续进行，即使遇到非延迟约束违规也是如此。它会记录导致非延迟约束违规的所有行，但不会停止遇到违规的数据对象的负载。

    SKIP_CONSTRAINT_ERRORS ：当遇到非延迟约束违规时，阻止回滚整个数据对象。

    如果未设置SKIP_CONSTRAINT_ERRORS ，则默认行为是回滚遇到非延迟约束违规的数据对象的整个负载。

  • TRUST_EXISTING_TABLE_PARTITIONS ：告诉Data Pump将分区数据并行加载到现有表中。

    在启动表数据导入之前使用数据泵从导出数据库中的定义创建表时，请使用此选项。通常，当元数据是静态时，您可以将此参数用作迁移的一部分，并且可以在脱机数据库之前移动它以迁移数据。单独移动元数据可最大限度地减少停机如果使用此选项，并且数据库的其他属性相同（例如，字符集），则导出数据库中的数据将转到导入数据库中的相同分区。

    您可以在数据泵外部创建表。但是，如果使用Data Pump创建表作为单独的选项，则分区属性和分区名称必须与导出数据库相同。

  • VALIDATE_TABLE_DATA ：指示数据泵验证表数据列中的数字和日期数据类型。

    如果导入遇到无效数据，则会向.log文件写入ORA-39376错误。错误文本包括列名称。默认是不进行验证。如果数据泵转储文件的来源不受信任，请使用此选项。

  • ENABLE_NETWORK_COMPRESSION ：用于网络导入，其中Data Pump ACCESS_METHOD参数设置为DIRECT_PATH以加载远程表数据。

    指定ENABLE_NETWORK_COMPRESSION ，Data Pump会在远程节点上通过网络将数据压缩到目标数据库之前对其进行压缩。如果远程数据库和本地数据库之间的网络连接速度很慢，则此选项很有用，因为它减少了通过网络发送的数据量。

    如果Data Pump使用DIRECT_PATH进行网络导入，则设置ACCESS_METHOD=AUTOMATIC可使Data Pump在导入期间自动设置ENABLE_NETWORK_COMPRESSION 。

    如果Data Pump正在从转储文件导入数据，如果远程数据库早于Oracle Database 12 c Release 2（12.2），或者正在使用INSERT_AS_SELECT语句从远程数据库加载数据，则忽略ENABLE_NETWORK_COMPRESSION选项。

  • CONTINUE_LOAD_ON_FORMAT_ERROR ：当加载表数据时遇到流格式错误时，指示数据泵跳转到下一个粒度的开头。

    流格式错误通常是损坏的转储文件的结果。如果Data Pump遇到流格式错误，并且原始导出数据库不可用于再次导出表数据，则可以使用CONTINUE_LOAD_ON_FORMAT_ERROR 。如果Data Pump跳过数据，则不会导入源数据库中的所有数据，这可能会跳过数百或数千行。

限制

  • 如果使用DISABLE_APPEND_HINT ，则加载数据对象可能需要更长时间。

  • 如果使用SKIP_CONSTRAINT_ERRORS ，并且如果数据对象在加载时在其上定义了唯一索引或约束，则APPEND提示不用于加载该数据对象。因此，使用SKIP_CONSTRAINT_ERRORS选项时，加载此类数据对象可能需要更长时间。

  • 即使指定了SKIP_CONSTRAINT_ERRORS ，除非使用外部表访问方法加载数据对象，否则不会使用它。

例

此示例显示启用了SKIP_CONSTRAINT_ERRORS的仅数据表模式导入：

> impdp hr TABLES = employees CONTENT = DATA_ONLY DUMPFILE = dpump_dir1：table.dmp DATA_OPTIONS = skip_constraint_errors

如果在此导入操作期间遇到任何非延迟约束违规，则会记录它们。进口继续完成。

Data Pump Import命令行模式DUMPFILE参数指定Export创建的转储文件集的名称和（可选）目录对象。

默认值： expdat.dmp

目的

指定名称，如果选择，还指定由Export创建的转储文件集的目录对象或默认凭据。

语法和说明

DUMPFILE = [ directory_object ：] file_name [，...]

要么

DUMPFILE = [DEFAULT_CREDENTIAL：] URI_file [，...]

如果已由DIRECTORY参数建立了DIRECTORY则该directory_object是可选的。如果确实提供了值，则它必须是已存在且您有权访问的目录对象。指定为DUMPFILE参数一部分的数据库目录对象将覆盖DIRECTORY参数指定的值。

file_name是转储文件集中文件的名称。文件名也可以是包含替换变量%U模板。导入过程检查与模板匹配的每个文件，以查找属于转储文件集的所有文件，直到找不到匹配项。如果DUMPFILE参数中定义的文件规范包含整个集合，则导入的文件中包含足够的信息以定位整个集合。这些文件不需要在导出时使用相同的名称，位置或顺序。

可能的替代变量如下表所述。

替代变量 描述

％U

 如果使用%U ，则%U扩展为从01开始的2位递增整数。

%l ， %L

 指定系统生成的唯一文件名。

文件名可以包含替换变量（ %L ），这意味着可能会生成多个文件。替换变量在生成的文件名中扩展为2位，固定宽度的递增整数，从01开始，到99结束，与（ %U ）相同。此外，替换变量在生成的文件名中扩展为3位到10位，可变宽度，递增的整数，从100开始，到2147483646结束。宽度字段由整数中的位数确定。

例如，如果当前整数为1，则exp%Laa%L.dmp将解析为以下顺序
exp01aa01.dmp exp02aa02.dmp
2位数的增量继续增加，最多为99。然后，下一个文件名替换3位增量：
exp100aa100.dmp exp101aa101.dmp

3位数的增量一直持续到999。然后，下一个文件名替换4位增量。替换继续达到允许的最大数量替换，即2147483646。

3.4.12 ENCRYPTION_PASSWORD

Data Pump Import命令行模式ENCRYPTION_PASSWORD参数指定用于访问转储文件集中的加密列数据的密码。这可以防止对加密转储文件集的未授权访问。

默认值：没有默认值;该值是用户提供的。

目的

指定用于访问转储文件集中的加密列数据的密码。这可以防止对加密转储文件集的未授权访问。

在完全可传输的导出或导入操作期间，传输与加密表空间关联的密钥以及具有加密列的表也需要此参数。

您输入的密码将回显到屏幕。如果您在输入时不想显示屏幕上显示的密码，请使用ENCRYPTION_PWD_PROMPT参数。

语法和说明

ENCRYPTION_PASSWORD = password

如果在导出操作上指定了加密密码，则在导入操作中需要此参数。指定的密码必须与导出操作中指定的密码相同。

限制

  • 此参数仅在Oracle Database 11g或更高版本的企业版中有效。

  • Data Pump加密功能要求您启用Oracle Advanced Security选项。有关Oracle Advanced Security选项的许可要求的信息 ，请参阅Oracle数据库许可信息 。

  • 如果转储文件集是使用透明加密模式创建的，则ENCRYPTION_PASSWORD参数无效。

  • 基于网络的完全可传输导入需要ENCRYPTION_PASSWORD参数，其中源数据库具有加密表空间或具有加密列的表。

  • 所有列的加密属性必须在导出的表定义和目标表之间匹配。例如，假设您有一个表EMP ，其中一个列名为EMPNO 。以下两种情况都会导致错误，因为源表中EMP列的加密属性与目标表中EMP列的加密属性不匹配：

    • 导出EMP表并加密EMPNO列，但在导入表之前，从EMPNO列中删除加密属性。

    • 导出EMP表时不会对EMPNO列进行加密，但在导入表之前，您需要在EMPNO列上启用加密。

例

在以下示例中，必须指定加密密码123456 ，因为它是在创建dpcd2be1.dmp转储文件时指定的。 

> impdp hr TABLES = employee_s_encrypt DIRECTORY = dpump_dir DUMPFILE = dpcd2be1.dmp ENCRYPTION_PASSWORD = 123456

在导入操作期间，导出操作期间加密的employee_s_encrypt表中的任何列都将在导入之前被解密。

3.4.15排除

Data Pump Import命令行模式EXCLUDE参数使您可以通过指定要从导入作业中排除的对象和对象类型来过滤导入的元数据。

默认值：没有默认值

目的

允许您通过指定要从导入作业中排除的对象和对象类型来过滤导入的元数据。

语法和说明

EXCLUDE = object_type [： name_clause ] [，...]

object_type指定要排除的对象的类型。要查看有效值列表object_type ，查询以下观点： DATABASE_EXPORT_OBJECTS全模式， SCHEMA_EXPORT_OBJECTS的架构模式， TABLE_EXPORT_OBJECTS表和表空间模式。OBJECT_PATH列中列出的值是有效的对象类型。

对于给定的导入模式，包括源（及其依赖项）中包含的所有对象类型，但EXCLUDE语句中指定的对象类型除外。如果排除了某个对象，则也会排除其所有依赖对象。例如，排除表也将排除表中的所有索引和触发器。

name_clause是可选的。它允许在对象类型中细粒度选择特定对象。它是一个SQL表达式，用作对类型的对象名称的过滤器。它由SQL运算符和要与之比较指定类型的对象名称的值组成。name_clause仅适用于其实例具有名称的对象类型（例如，它适用于TABLE和VIEW ，但不适用于GRANT ）。必须使用冒号将其与对象类型分开并用双引号括起来，因为需要使用单引号来分隔名称字符串。例如，您可以设置EXCLUDE=INDEX:"LIKE 'DEPT%'"以排除名称以dept开头的所有索引。

您为name_clause提供的name_clause必须完全匹配，包括上限和下限，数据库中的现有对象。例如，如果您提供的name_clause是针对名为EMPLOYEES的表，则必须使用全部大写的名为EMPLOYEES的现有表。如果name_clause作为Employees或employees或任何其他变体提供，则无法找到该表。

可以指定多个EXCLUDE语句。

根据您的操作系统，为此参数指定值时使用引号可能还需要使用转义字符。Oracle建议您将此参数放在参数文件中，这样可以减少命令行上可能需要的转义字符数。

如以下各节所述，您应该了解指定某些对象的排除效果，特别是CONSTRAINT ， GRANT和USER 。

排除约束

不能排除以下约束：

  • 成功创建和加载表所需的约束（例如，索引组织表的主键约束或REF SCOPE和具有REF列的表的WITH ROWID约束）。

这意味着以下EXCLUDE语句将被解释如下：

  • EXCLUDE=CONSTRAINT排除所有约束，但成功创建和加载表所需的任何约束除外。

  • EXCLUDE=REF_CONSTRAINT排除参照完整性（外键）约束。

  • 网络导入不支持使用演进类型。

  • 如果正在执行导入作业的USERID在目标数据库上具有DATAPUMP_IMP_FULL_DATABASE角色，则该用户还必须在源数据库上具有DATAPUMP_EXP_FULL_DATABASE角色。

3.4.29并行

Data Pump Import命令行模式PARALLEL参数设置可以并行加载的最大工作进程数。

默认值： 1

目的

指定代表导入作业运行的最大活动执行进程数。

语法和说明

PARALLEL = integer

您为integer指定的值指定代表导入作业运行的最大活动执行进程数。此执行集由工作进程和并行I / O服务器进程组合而成。并行I / O操作中作为并行执行协调器的主控制进程，空闲工作进程和工作进程不计入此总数。此参数使您可以在资源消耗和已用时间之间进行权衡。

如果导入源是由文件组成的转储文件集，则多个进程可以从同一文件读取，但性能可能受I / O争用的限制。

要在作业执行期间增加或减少PARALLEL的值，请使用交互式命令模式。

在网络模式导入期间使用PARALLEL

在网络模式导入期间， PARALLEL参数定义可以分配给作业的最大工作进程数。要在网络导入模式下了解PARALLEL参数的效果，了解Data Pump定义的“table_data对象”的概念非常重要。当Data Pump移动数据时，它会将以下项目视为单独的“table_data对象”：

  • 一个完整的表（一个没有分区或子分区的表）

  • 分区，如果表已分区但未分区

  • 子分区，如果表是子分区的

例如：

  • 非分区表scott.non_part_table有一个table_data对象：

    scott.non_part_table

  • 分区表scott.part_table （具有分区p1和分区p2 ）具有两个table_data对象：

    scott.part_table：P1

    scott.part_table:p 2 scott.part_table:p

  • 子分区表scott.sub_part_table （具有分区p1和p2 ，以及子分区p1s1 ， p1s2 ， p2s1和p2s2 ）具有四个table_data对象：

    scott.sub_part_table：p1s1

    scott.sub_part_table：p1s2

    scott.sub_part_table：p2s1

    scott.sub_part_table：p2s2

在网络模式导入期间，为每个table_data对象分配其自己的工作进程，最多为PARALLEL参数指定的值。没有分配并行查询（PQ）从站，因为网络模式导入不使用并行查询（PQ）从站。可以同时卸载多个table_data对象。但是，使用单个进程卸载每个table_data对象。

在Oracle RAC环境中导入期间使用PARALLEL

在Oracle Real Application Clusters（Oracle RAC）环境中，如果导入操作的PARALLEL=1 ，则所有Data Pump进程都驻留在启动作业的实例上。因此，目录对象可以指向该实例的本地存储。

如果导入操作将PARALLEL设置为大于1的值，则数据泵进程可以驻留在作业启动的实例之外的实例上。因此，目录对象必须指向可由Oracle RAC的所有实例访问的共享存储。

限制

  • 此参数仅在Oracle Database 11g或更高版本的企业版中有效。

  • 要并行导入表或表分区（使用PQ从属），您必须具有DATAPUMP_IMP_FULL_DATABASE角色。

例

以下是使用PARALLEL参数的示例。 

> impdp hr DIRECTORY = dpump_dir1 LOGFILE = parallel_import.log JOB_NAME = imp_par3 DUMPFILE = par_exp％U.dmp PARALLEL = 3

此命令导入在运行Export PARALLEL参数的示例时创建的转储文件集。转储文件的名称为par_exp01.dmp ， par_exp02.dmp和par_exp03.dmp 。

3.4.31 PARTITION_OPTIONS

“数据泵导入”命令行模式PARTITION_OPTIONS参数指定在导入操作期间如何创建表分区。

默认值：当在TABLES参数上指定分区名称并且设置了TRANPORTABLE=ALWAYS （无论是在导入操作还是在导出期间）时，默认为departition 。否则，默认值为none 。

目的

指定在导入操作期间如何创建表分区。

语法和说明

PARTITION_OPTIONS = [NONE | DEPARTITION |合并]

值NONE会创建在执行导出操作的系统上存在的表。如果使用可传输方法使用分区或子分区筛选器执行导出，则不能使用NONE选项或MERGE选项。在这种情况下，您必须使用DEPARTITION选项。

DEPARTITION的值将每个分区或子分区提升为新的单个表。新表的默认名称是表和分区名称的串联，或者表和子分区名称的串联（如果适用）。

MERGE的值将所有分区和子分区组合到一个表中。

导入分区表时的并行处理取决于以下内容：

  • 如果将分区表导入现有分区表，则Data Pump一次只处理一个分区或子分区，而不管PARALLEL参数指定的值是什么。

  • 如果要导入的表尚不存在，并且Data Pump必须创建它，则导入将并行运行，直到导入开始时PARALLEL参数上指定的PARALLEL度。

限制

  • 您使用分区来创建和填充基于源表分区的表。为避免命名冲突，当PARTITION_OPTIONS的值设置为DEPARTITION ，不会与这些表一起创建依赖对象（如约束和索引）。

  • 如果使用可传输方法执行了创建转储文件的导出操作，并且指定了分区或子分区，则导入操作必须使用DEPARTITION选项。

  • 如果使用可传输方法执行了创建转储文件的导出操作，则导入操作不能使用PARTITION_OPTIONS=MERGE 。

  • 如果对象被分离有任何授权，则会生成错误消息，并且不会加载对象。

例

以下示例假定sh.sales表已导出到名为sales.dmp的转储文件中。它使用merge选项将sh.sales所有分区合并到scott模式中的一个非分区表中。 

> impdp系统TABLES = sh.sales PARTITION_OPTIONS = MERGE DIRECTORY = dpump_dir1 DUMPFILE = sales.dmp REMAP_SCHEMA = sh：scott

3.4.32查询

Data Pump Import命令行模式QUERY参数使您可以指定筛选导入数据的查询子句。

默认值：没有默认值

目的

使您可以指定筛选导入数据的查询子句。

语法和说明

QUERY = [[ schema_name 。] table_name ：] query_clause

query_clause通常是用于细粒度行选择的SQL WHERE子句。但是，它可以是任何SQL子句。例如，您可以使用ORDER BY子句来加速从堆组织表到索引组织表的迁移。如果未提供模式和表名，则查询将应用于源转储文件集或数据库中的所有表（并且必须对其有效）。特定于表的查询会覆盖应用于所有表的查询。

如果要将查询应用于特定表，则必须使用冒号（:)将表名与查询原因分开。您可以指定多个特定于表的查询，但每个表只能指定一个查询。

如果与QUERY参数一起指定了NETWORK_LINK参数，则必须使用NETWORK_LINK值明确限定query_clause中指定的远程（源）节点上的任何对象。否则，Data Pump假定对象位于本地（目标）节点上;如果不是，则返回错误，并从远程（源）系统导入表失败。

例如，如果指定NETWORK_LINK=dblink1 ，则QUERY参数的query_clause必须指定该链接，如以下示例所示：

QUERY =（hr.employees：“WHERE last_name IN（SELECT last_name FROM hr.employees@dblink1）”）

根据您的操作系统，为此参数指定值时使用引号可能还需要使用转义字符。Oracle建议您将此参数放在参数文件中，这样可以减少命令行上可能需要的转义字符数。请参阅“关于导入命令行模式”。

如果使用QUERY参数，则使用外部表方法（而不是直接路径方法）进行数据访问。

要在特定于表的查询中指定除您自己之外的模式，必须授予您对该特定表的访问权限。

限制

  • 您不能将QUERY参数与以下参数一起使用：

    • CONTENT = METADATA_ONLY

    • SQLFILE

    • TRANSPORT_DATAFILES

  • 为表指定QUERY参数时，Data Pump使用外部表加载目标表。外部表使用带有SELECT子句的SQL INSERT语句。QUERY参数的值包含在INSERT语句的SELECT部分的WHERE子句中。如果QUERY参数包含对另一个表的引用，其列的名称与正在加载的表匹配，并且如果在查询中使用了这些列，则必须使用表别名来区分正在加载的表中的列，以及具有相同名称的SELECT语句。

    例如，假设您正在导入的一个子集sh.sales基于信用额度在客户表sh.customers表。在以下示例中，Data Pump用于正在加载的表的表别名是KU$ 。KU$用于限定QUERY参数中的cust_id字段以加载sh.sales 。因此，Data Pump仅为信用额度超过10,000美元的客户导入行。 

    QUERY ='sales：'WHERE EXISTS（SELECT cust_id FROM customers c WHERE cust_credit_limit> 10000 AND ku $ .cust_id = c.cust_id）“'

    如果KU$未用于表别名，则会加载所有行：

    QUERY ='sales：“WHERE EXISTS（SELECT cust_id FROM customers c WHERE cust_credit_limit> 10000 AND cust_id = c.cust_id）”'

  • QUERY字符串允许的最大长度为4000字节，包括引号，这意味着允许的实际最大长度为3998字节。

例

以下是使用QUERY参数的示例。您可以通过运行为Export FULL参数提供的示例来创建此示例中使用的expfull.dmp转储文件。请参见导出FULL参数。由于QUERY值使用引号，因此Oracle建议您使用参数文件。

假设您有一个参数文件query_imp.par ，其中包含以下内容：

QUERY =部门：“WHERE department_id <120”

然后，您可以输入以下命令：

> impdp hr DIRECTORY = dpump_dir1 DUMPFILE = expfull.dmp PARFILE = query_imp.par NOLOGFILE = YES

导入expfull.dmp中的所有表，但对于departments表，仅导入符合QUERY参数中指定条件的数据。

3.4.33 REMAP_DATA

Data Pump Import命令行模式REMAP_DATA参数使您可以在将数据插入新数据库时重新映射数据。

默认值：没有默认值

目的

REMAP_DATA参数使您可以在将数据插入新数据库时重新映射数据。常见的用途是重新生成主键，以避免在将表导入目标数据库上的预先存在的表时发生冲突。

您可以指定重映射函数，该函数将转储文件或远程数据库中指定列的值作为源。然后，重映射函数返回重新映射的值，该值将替换目标数据库中的原始值。

相同的函数可以应用于被转储的多个列。当您希望保证在引用约束中重新映射子列和父列时的一致性时，此函数很有用。

语法和说明

REMAP_DATA = [ schema 。] tablename 。 column_name ：[ schema 。] pkg 。 function

以下是每个语法元素的列表，按它们在语法中出现的顺序排列：

schema ：包含要重新映射的表的模式。默认情况下，此架构是执行导入的用户的架构。

tablename ：重新映射列的表。

column_name ：要重新映射其数据的列。

schema ：包含您创建的包含重映射功能的PL / SQL包的模式。默认情况下，这是执行导入的用户的架构。

pkg ：您创建的包含重映射功能的PL / SQL包的名称。

function ：PL / SQL中的函数名称，用于重新映射指定表的每一行中的列表。

限制

  • source参数和返回值的数据类型和大小必须与表中指定列的数据类型和大小相匹配。

  • 除自治事务外，重映射函数不应执行提交或回滚。

  • 不支持使用同义词作为REMAP_DATA参数的值。例如，如果hr模式中的regions表具有regn的同义词，则如果将regn指定为REMPA_DATA规范的一部分，则会返回错误。

  • 不支持重新映射远程表的LOB列数据。

  • REMAP_DATA不支持以下类型的列：用户定义类型，用户定义类型的属性， LONG ， REF ， VARRAY ，嵌套表， BFILE和XMLtype 。

例

以下示例假定已创建名为remap的包，其中包含名为plusx的函数，该函数更改employees表中first_name的值。 

> impdp hr DIRECTORY = dpump_dir1 DUMPFILE = expschema.dmp TABLES = hr.employees REMAP_DATA = hr.employees.first_name：hr.remap.plusx

3.4.36 REMAP_SCHEMA

Data Pump Import命令行模式REMAP_SCHEMA参数将源模式中的所有对象加载到目标模式中。

默认值：没有默认值

目的

将源模式中的所有对象加载到目标模式中。

语法和说明

REMAP_SCHEMA = source_schema ： target_schema

可以指定多个REMAP_SCHEMA行，但REMAP_SCHEMA行的源架构必须不同。但是，不同的源模式可以映射到相同的目标模式。映射可能不完整;请参阅本主题中的“限制”部分。

如果在导入之前重新映射到的模式不存在，则导入操作可以创建它。要创建模式，转储文件集必须包含源模式所需的CREATE USER元数据，并且必须以足够的权限执行导入。例如，以下Export命令创建包含必要元数据的转储文件集以创建模式，因为用户SYSTEM具有必要的权限：

> expdp system SCHEMAS = hr密码： password > expdp系统FULL = YES密码： password

如果转储文件集不包含创建架构所需的元数据，或者如果您没有权限，则必须在执行导入操作之前创建目标架构。您必须在导入之前创建目标模式，因为非特权转储文件不包含导入所需的自动创建模式的信息。

对于早于Oracle数据库11克 Oracle数据库版本中，如果导入操作将创建一个架构，进口则完成后，则必须为其指定一个有效密码才能连接到它。您可以使用以下SQL语句来分配密码;请注意，您需要权限：

SQL> ALTER USER schema_name IDENTIFIED BY new_password

从Oracle Database 11g第1版（11.1.0.1）开始，不再需要重置模式密码;原始密码仍然有效。

限制

  • 只有当模式的模式是重映射的目标模式时，非特权用户才能执行模式重映射。（特权用户可以执行不受限制的架构重映射。）例如， SCOTT可以将BLAKE的对象重新映射到SCOTT ，但是SCOTT无法将SCOTT的对象重新映射到BLAKE 。

  • 映射可能不完整，因为某些模式引用导致无法查找。例如，Import没有找到嵌入在类型，视图，过程和包的定义主体中的模式引用。

  • 对于触发器， REMAP_SCHEMA仅影响触发器所有者。

  • 如果正在重新映射的模式中的任何表包含用户定义的对象类型，并且该表在导出时间和尝试导入它的时间之间发生更改，则该表的导入将失败。但是，导入操作本身仍在继续。

  • 默认情况下，如果源数据库上的架构对象具有对象标识符（OID），则会使用这些相同的OID将它们导入目标数据库。如果将对象导回到导出它的同一数据库中，但导入到另一个模式中，则新（导入）对象的OID与现有对象的OID相同，导入失败。要使导入成功，还必须在导入时指定TRANSFORM=OID:N参数。转换OID:N导致为新对象创建新的OID，这允许导入成功。

例

假设，作为用户SYSTEM ，您执行以下导出和导入命令以将hr模式重新映射到scott模式：

> expdp system SCHEMAS = hr DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp> impdp system DIRECTORY = dpump_dir1 DUMPFILE = hr.dmp REMAP_SCHEMA = hr：scott

在此示例中，如果在导入之前用户scott已存在，则Import REMAP_SCHEMA命令会将hr模式中的对象添加到现有scott模式中。您可以在导入后使用现有密码连接到scott模式（不重置密码）。

如果在执行导入操作之前用户scott不存在，则Import会自动使用不可用的密码创建它。此操作是可能的，因为转储文件hr 。 dmp由SYSTEM,创建SYSTEM,它具有创建包含创建模式所需元数据的转储文件所需的特权。但是，除非在导入完成后在目标数据库上重置scott的密码，否则在完成导入时无法连接到scott 。

3.4.37 REMAP_TABLE

Data Pump Import命令行模式REMAP_TABLE参数使您可以在导入操作期间重命名表。

默认值：没有默认值

目的

使您可以在导入操作期间重命名表。

语法和说明

您可以使用以下任一语法（请参阅使用说明）：

REMAP_TABLE = [ schema. ] old_tablename [ .partition ] :new_tablename

要么

REMAP_TABLE = [ schema. ] old_tablename [ :partition ] :new_tablename

如果表正在离开，那么您可以使用REMAP_TABLE参数重命名整个表，或重命名表分区（请参阅PARTITION_OPTIONS ）。

使用说明

  • 如果正在重新映射的表在同一模式中命名了约束，并且必须在创建表时创建约束，则REMAP_TABLE参数不起作用

3.4.46 TABLE_EXISTS_ACTION

数据泵导入命令行模式TABLE_EXISTS_ACTION参数指定导入在其尝试创建的表已存在时要执行的操作。

默认值： SKIP （请注意，如果指定了CONTENT=DATA_ONLY ，则默认为APPEND ，而不是SKIP 。）

目的

指定导入在尝试创建的表已存在时要执行的操作。

语法和说明

TABLE_EXISTS_ACTION = [跳过| APPEND | TRUNCATE |更换]

可能的值具有以下效果：

  • SKIP按原样离开表格并移动到下一个对象。当CONTENT参数设置为DATA_ONLY时，此选项无效。

  • APPEND从源加载行并保持现有行不变。

  • TRUNCATE删除现有行，然后从源加载行。

  • REPLACE删除现有表，然后从源创建并加载它。当CONTENT参数设置为DATA_ONLY时，此选项无效。

使用这些选项时，以下注意事项适用：

  • 使用TRUNCATE或REPLACE ，请确保受影响的表中的行不是任何引用约束的目标。

  • 使用SKIP ， APPEND或TRUNCATE ，源中现有的表依赖对象（如索引，授权，触发器和约束）不会被修改。对于REPLACE ，如果未明确或隐式排除（使用EXCLUDE ）并且它们存在于源转储文件或系统中，则从源中删除并重新创建从属对象。

  • 使用APPEND或TRUNCATE ，Import会在执行任何操作之前检查源中的行是否与现有表兼容。

    如果现有表具有活动约束和触发器，则使用外部表访问方法加载它。如果任何行违反了活动约束，则加载失败并且不加载任何数据。您可以通过在“导入”命令行上指定DATA_OPTIONS=SKIP_CONSTRAINT_ERRORS来覆盖此行为。

    如果您有必须加载的数据，但这可能导致约束违规，那么请考虑禁用约束，加载数据，然后在重新启用约束之前删除问题行。

  • 使用APPEND ，数据始终会加载到新空间;即使可用，现有空间也不会被重复使用。因此，您可能希望在加载后压缩数据。

  • 有关如何影响分区表的并行处理的信息，请参阅Import PARTITION_OPTIONS参数的说明，具体取决于目标表是否已存在。

注意：

当Data Pump检测到源表和目标表不匹配时（两个表没有相同的列数或目标表具有源表中不存在的列名），它会比较列之间的列名称两张桌子。如果表中至少有一个列共同，则将公共列的数据导入表中（假设数据类型兼容）。以下限制适用：

  • 网络导入不支持此行为。

  • 无法删除以下类型的列：基于主键的对象列，对象属性，嵌套表列和ref列。

限制

  • TRUNCATE不能用于群集表。

例

以下是使用TABLE_EXISTS_ACTION参数的示例。您可以通过运行为Export FULL参数提供的示例来创建此示例中使用的expfull.dmp转储文件。 

> impdp hr TABLES = employees DIRECTORY = dpump_dir1 DUMPFILE = expfull.dmp TABLE_EXISTS_ACTION = REPLACE

3.4.48表格

Data Pump Import命令行模式TABLES参数指定您要执行表模式导入。

默认值：没有默认值

目的

指定要执行表模式导入。

语法和说明

TABLES = [ schema_name 。] table_name [： partition_name ]

在表模式导入中，您可以通过指定以逗号分隔的表和分区或子分区列表来过滤从源导入的数据。

如果您不提供schema_name ，则默认为当前用户的默认值。要指定除您自己之外的模式，您必须具有DATAPUMP_IMP_FULL_DATABASE角色或将模式重新映射到当前用户。

如果要限制导入的内容，可以使用此导入模式进行过滤。

如果指定partition_name ，则它必须是关联表中的分区或子分区的名称。

您可以使用通配符%指定表名和分区名。

以下限制适用于表名：

  • 默认情况下，数据库中的表名称存储为大写字符。如果您具有大小写或小写字符的表名，并且希望保留表名的区分大小写，则必须将该名称括在引号中。名称必须与存储在数据库中的表名完全匹配。

    某些操作系统要求命令行上的引号前面有转义字符。以下是在不同导入模式下如何保持区分大小写的示例。

    • 在命令行模式下：

      TABLES = '\ “的Emp \”'

    • 在参数文件模式中：

      TABLES = ' “的Emp”'

  • 除非表名用引号括起，否则命令行中指定的表名不能包含井号（ ＃ ）。类似地，在参数文件中，如果表名包含井号（ # ），则除非将表名括在引号中，否则导入实用程序会将该行的其余部分解释为注释。

    例如，如果参数文件包含以下行，则在emp#作为注释后，Import会解释该行上的所有内容，并且不会导入表dept和mydata:

    TABLES =（emp＃，dept，mydata）

    但是，如果参数文件包含以下行，则导入实用程序会导入所有三个表，因为emp#用引号括起来：

    TABLES =（'“emp＃”'，dept，mydata）

    注意：

    有些操作系统需要单引号而不是双引号，反之亦然;请参阅您的操作系统文档。不同的操作系统对表命名也有其他限制。

    例如，UNIX C shell对美元符号（ $ ）或井号（ # ）或某些其他特殊字符附加了特殊含义。必须使用转义字符在名称中使用这些特殊字符，以便操作系统shell忽略它们，并且它们可以与Import一起使用。

限制

  • 不支持使用同义词作为TABLES参数的值。例如，如果hr模式中的regions表具有regn的同义词，那么使用TABLES=regn将无效。将返回错误。

  • 如果在导入时也指定了PARTITION_OPTIONS=DEPARTITION则只能从一个表中指定分区。

  • 如果指定TRANSPORTABLE=ALWAYS ，则TABLES参数上指定的所有分区必须位于同一个表中。

  • 为TABLES参数指定的表名列表的长度限制为最大4 MB，除非您将NETWORK_LINK参数用于Oracle数据库10.2.0.3版或更早版本或只读数据库。在这种情况下，限制为4 KB。

例

以下示例显示了使用TABLES参数从expfull.dmp文件中仅导入employees和jobs表的expfull.dmp 。您可以通过运行为Export FULL参数提供的示例来创建此示例中使用的expfull.dmp转储文件。 

> impdp hr DIRECTORY = dpump_dir1 DUMPFILE = expfull.dmp TABLES =员工，工作

> impdp hr DIRECTORY = dpump_dir1 DUMPFILE = expdat.dmp TABLES = sh.sales：sales_Q1_2012，sh.sales：sales_Q2_2012

此示例导入模式sh中表sales的分区sales_Q1_2012和sales_Q2_2012 。

3.4.51 TRANSFORM

Data Pump Import命令行模式TRANSFORM参数使您可以更改要导入的对象的对象创建DDL。

默认值：没有默认值

目的

使您可以更改要导入的对象的对象创建DDL。

语法和说明

TRANSFORM = transform_name ： value [： object_type ]

transform_name指定transform_name的名称。

指定object_type是可选的。如果提供，此参数指定应用转换的对象类型。如果未指定对象类型，则转换将应用于所有有效对象类型。

可用的变换如下，按字母顺序排列：

  • CONSTRAINT_NAME_FROM_INDEX：[Y | N]

    此转换对以下对象类型有效： TABLE和CONSTRAINT对象类型。

    此转换参数会影响引用用户创建的索引的pk或fk约束的生成。如果设置为Y ，则强制约束的名称与索引的名称匹配。

    如果设置为N （默认值），则在源数据库上创建约束为约束。

  • CONSTRAINT_USE_DEFAULT_INDEX：[Y | N]

    此转换对以下对象类型有效： TABLE和CONSTRAINT对象类型。

    该变换参数影响与pk或fk约束相关的索引的生成。如果设置为Y ，则强制自动创建索引的名称以强制约束与约束名称相同。

    如果设置为N （缺省值），则在源数据库上创建索引。

  • DISABLE_ARCHIVE_LOGGING：[Y | N]

    此转换对以下对象类型有效： INDEX和TABLE 。

    如果设置为Y ，则在导入数据之前，将禁用指定对象类型（ TABLE和/或INDEX ）的日志记录属性。如果设置为N （默认值），则导入期间不会禁用存档日志记录。加载数据后，对象的日志记录属性将恢复为其原始设置。如果未指定任何对象类型，则DISABLE_ARCHIVE_LOGGING行为将应用于TABLE和INDEX对象类型。此转换适用于文件模式导入和网络模式导入。它不适用于可传输的表空间导入。

    注意：

    如果数据库处于FORCE LOGGING模式，则在创建索引和表时， DISABLE_ARCHIVE_LOGGING选项不会禁用日志记录。

  • DWCS_CVT_IOTS：[Y | N]

    此转换对TABLE对象类型有效。

    如果设置为Y ，它会指示Oracle Data Pump通过在创建表时禁止ORGANIZATION INDEX子句将Index Organized表转换为堆组织表。

    如果设置为N （默认值），则生成的DDL将保留源对象的表特征。

  • DWCS_CVT_CONSTRAINTS：[Y | N]

    此转换对以下对象类型有效： TABLE和CONSTRAINT对象类型。

    如果设置为Y ，则会指示Oracle Data Pump将pk ， fk或uk约束创建为pk禁用。

    如果设置为N （默认值），它会指示Oracle Data Pump根据源数据库状态创建pk ， fk或uk约束。

  • INMEMORY：[Y | N]

    此转换对以下对象类型有效： TABLE和TABLESPACE

    INMEMORY转换与内存中列存储（IM列存储）相关。IM列存储是系统全局区域（SGA）的可选部分，用于存储表，表分区和其他数据库对象的副本。在IM列存储中，数据由列而不是行填充，就像在SGA的其他部分中一样，并且数据针对快速扫描进行了优化。IM列存储不会替换缓冲区缓存，而是作为补充，以便两个内存区域可以以不同的格式存储相同的数据。IM列存储包含在Oracle Database In-Memory选项中。

    如果在导入时指定了Y （默认值），则Data Pump会为具有一个的所有对象保留IM列存储子句。在导入时重新创建这些对象时，Data Pump会生成IM列存储子句，该子句与导出时这些对象的设置相匹配。

    如果在导入时指定了N ，则Data Pump会从具有一个的所有对象中删除IM列存储子句。如果存储在表空间中的对象没有IM列存储子句，则该对象将从表空间继承IM列存储子句。因此，如果要迁移数据库，并且希望新数据库使用IM列存储功能，则可以使用相应的IM列存储子句预创建表空间，然后在导入命令上使用TRANSFORM=INMEMORY:N 。然后，该对象从新预先创建的表空间继承IM列存储子句。

    如果不使用INMEMORY转换，则必须单独更改每个对象以添加适当的IM列存储子句。

    注意：

    INMEMORY转换仅在Oracle Database 12 c第1版（12.1.0.2）或更高版本中可用。

    有关使用内存中列存储（IM列存储）的信息，请参见“ Oracle数据库管理员指南” 。

  • INMEMORY_CLAUSE： "string with a valid in-memory parameter”

    此转换对以下对象类型有效： TABLE和TABLESPACE 。

    INMEMORY_CLAUSE转换与内存中列存储（IM列存储）相关。IM列存储是系统全局区域（SGA）的可选部分，用于存储表，表分区和其他数据库对象的副本。在IM列存储中，数据由列而不是行填充，就像在SGA的其他部分中一样，并且数据针对快速扫描进行了优化。IM列存储不会替换缓冲区缓存，而是作为补充，以便两个内存区域可以以不同的格式存储相同的数据。IM列存储包含在Oracle Database In-Memory选项中。

    指定此转换时，Data Pump将字符串的内容用作INMEMORY_CLAUSE的所有对象的INMEMORY_CLAUSE ，这些对象在其DDL中具有IM列存储子句。当您要覆盖转储文件中对象的IM列存储子句时，此转换很有用。

    您提供的字符串必须用双引号括起来。如果要在命令行中输入命令，请注意某些操作系统可能会在解析命令期间删除引号，从而导致错误。您可以使用反斜杠转义字符（ \ ）来避免此错误。例如：
    transform = inmemory_clause：\“INMEMORY MEMCOMPRESS for DML PRIORITY CRITICAL \”

    或者，您可以将参数放在参数文件中。处理期间会保留参数文件中的引号。

    注意：

    INMEMORY_CLAUSE转换仅在Oracle Database 12 c第1版（12.1.0.2）或更高版本中可用。

    有关使用内存中列存储（IM列存储）的信息，请参见“ Oracle数据库管理员指南” 。有关可在IM列存储子句中指定的参数的列表和说明，请参阅Oracle数据库参考

  • LOB_STORAGE：[SECUREFILE | BASICFILE |默认|没变]

    此转换对于对象类型TABLE有效。

    使用指定的存储（ SECUREFILE或BASICFILE创建LOB段。如果值为NO_CHANGE （缺省值），则使用与源数据库中相同的存储创建LOB段。如果值为DEFAULT ，则省略关键字（ SECUREFILE或BASICFILE ），并使用默认存储创建LOB段。

    指定此转换会更改作业中所有表的LOB存储，包括为实例化视图提供存储的表。

    LOB_STORAGE转换在可传输导入作业中无效。

  • OID：[Y | N]

    此转换对以下对象类型有效： INC_TYPE ， TABLE和TYPE 。

    如果在导入时指定了Y （默认值），则会将导出的OID分配给新的对象表和类型。在目标数据库上查找现有匹配类型时，Data Pump还会执行OID检查。

    如果在导入时指定了N ，则：

    • 禁止在创建新对象表和类型期间分配导出的OID。而是分配新的OID。禁止分配导出的OID对于克隆模式非常有用，但不会影响引用的对象。

    • 在为与类型关联的表加载数据之前，Data Pump在目标数据库上查找现有匹配类型时会跳过正常类型的OID检查。仍然执行使用类型，版本号和类型名称的哈希码的其他检查。

  • OMIT_ENCRYPTION_CLAUSE：[Y | N]

    此转换对TABLE对象类型有效。

    如果设置为Y ，则它会指示Oracle Data Pump禁止列加密子句。在源数据库中加密的列未在导入的表中加密。

    如果设置为N （默认值），则它会指示Oracle Data Pump创建列加密子句，如源数据库中那样。

  • PCTSPACE： some_number_greater_than_zero

    此转换对以下对象类型有效： CLUSTER ， CONSTRAINT ， INDEX ， ROLLBACK_SEGMENT ， TABLE和TABLESPACE 。

    为此转换提供的value必须是大于零的数字。它表示用于更改范围分配和数据文件大小的百分比乘数。

    您可以将PCTSPACE转换与Data Pump Export SAMPLE参数一起使用，以使存储分配的大小与采样的数据子集匹配。（请参阅SAMPLE导出参数。）

  • SEGMENT_ATTRIBUTES：[Y | N]

    此转换对以下对象类型有效： CLUSTER ， CONSTRAINT ， INDEX ， ROLLBACK_SEGMENT ， TABLE和TABLESPACE 。

    如果将值指定为Y ，则包含段属性（物理属性，存储属性，表空间和日志记录）以及适当的DDL。默认值为Y

  • SEGMENT_CREATION：[Y | N]

    此转换对于对象类型TABLE有效。

    如果设置为Y （缺省值），则此转换SEGMENT CREATION SQL SEGMENT CREATION子句添加到CREATE TABLE语句中。也就是说， CREATE TABLE语句明确表示SEGMENT CREATION DEFERRED或SEGMENT CREATION IMMEDIATE 。如果值为N ，则CREATE TABLE语句中省略了SEGMENT CREATION子句。将此参数设置为N以使用正在加载的表的默认段创建属性。（此功能适用于Oracle Database 11g第 2版（11.2.0.2）及更高版本。

  • 存储：[Y | N]

    此转换对以下对象类型有效： CLUSTER ， CONSTRAINT ， INDEX ， ROLLBACK_SEGMENT和TABLE 。

    如果该值指定为Y ，则包含存储子句，并使用适当的DDL。默认值为Y如果SEGMENT_ATTRIBUTES=N则忽略此参数。

  • TABLE_COMPRESSION_CLAUSE：[NONE | compression_clause ]

    此转换对于对象类型TABLE有效。

    如果指定了NONE ，则省略表压缩子句（并为表提供表空间的默认压缩）。否则，该值是有效的表压缩子句（例如， NOCOMPRESS ， COMPRESS BASIC等）。使用指定的压缩创建表。有关有效的表压缩语法的信息，请参见Oracle数据库SQL语言参考 。

    如果表压缩子句不止一个单词，则它必须包含在单引号或双引号中。此外，您的操作系统可能要求您将子句括在转义字符中，例如反斜杠字符。例如：

    TRANSFORM = TABLE_COMPRESSION_CLAUSE：\“COLUMN STORE COMPRESS FOR QUERY HIGH \”

    指定此转换会更改作业中所有表的压缩类型，包括为实例化视图提供存储的表。

3.4.52 TRANSPORT_DATAFILES

数据泵导入命令行模式TRANSPORT_DATAFILES参数指定通过可传输表空间模式导入导入目标数据库的数据文件列表，或者在设置期间设置TRANSPORTABLE=ALWAYS时通过表模式或全模式导入导入数据文件的列表。出口。

默认值：没有默认值

目的

指定在导出期间设置TRANSPORTABLE=ALWAYS时，通过可传输表空间模式导入或通过表模式或全模式导入导入目标数据库的数据文件列表。数据文件必须已存在于目标数据库系统上。

语法和说明

TRANSPORT_DATAFILES = datafile_name

datafile_name必须包含在目标数据库所在的系统上有效的绝对目录路径规范（不是目录对象名称）。

datafile_name还可以在绝对路径规范的文件名部分中使用通配符。星号（ * ）匹配0到N字符。问号（ ? ）恰好匹配一个字符。您不能在绝对路径规范的目录部分中使用通配符。如果使用通配符，则所有匹配的文件必须是传输集的一部分。如果找到的任何文件不属于传输集，则会显示错误，导入作业将终止。

在导入操作之前的某个时刻，您必须将数据文件从源系统复制到目标系统。您可以使用操作系统支持的任何复制方法复制数据文件。如果需要，可以在将文件复制到目标系统时重命名文件。见例2。

如果已有任何可传输模式导出生成的转储文件集，则可以通过指定转储文件（包含元数据）和TRANSPORT_DATAFILES参数来执行该转储文件的可传输模式导入。TRANSPORT_DATAFILES参数的存在告诉import它是可传输模式导入以及从何处获取实际数据。

根据您的操作系统，为此参数指定值时使用引号也可能需要使用转义字符。Oracle建议您将此参数放在参数文件中，这样可以减少在命令行上需要使用的转义字符数。

限制

  • 您不能将TRANSPORT_DATAFILES参数与QUERY参数一起使用。

  • 您无法重新启动可传输导入作业。

  • 绝对文件路径的TRANSPORT_DATAFILES目录部分不能包含通配符。但是，绝对文件路径的文件名部分可以包含通配符

例1

以下是使用TRANSPORT_DATAFILES参数的示例。假设您有一个参数文件trans_datafiles.par,其中包含以下内容：

DIRECTORY = dpump_dir1 DUMPFILE = tts.dmp TRANSPORT_DATAFILES ='/ user01 / data / tbs1.dbf'

然后，您可以发出以下命令：

> impdp hr PARFILE = trans_datafiles.par

例2

此示例说明了将数据文件重命名为可传输表空间导出和导入操作的一部分。假设您的源系统上有一个名为employees.dat的数据文件。

  1. 使用操作系统支持的方法，手动将源系统中名为employees.dat的数据文件复制到目标数据库所在的系统。作为复制操作的一部分，将其重命名为workers.dat 。

  2. 执行表空间tbs_1的可传输表空间导出。 

    > expdp hr DIRECTORY = dpump_dir1 DUMPFILE = tts.dmp TRANSPORT_TABLESPACES = tbs_1

    仅将tbs_1的元数据（无数据）导出到名为tts.dmp的转储文件。在步骤1中将实际数据复制到目标数据库。

  3. 执行可传输的表空间导入，为名为workers.dat的数据文件指定绝对目录路径：

    > impdp hr DIRECTORY = dpump_dir1 DUMPFILE = tts.dmp TRANSPORT_DATAFILES ='/ user01 / data / workers.dat'

    导入tts.dmp包含的元数据，然后Data Pump将workers.dat文件中的信息分配给数据库中的正确位置。

例3

此示例说明了在与TRANSPORT_DATAFILES参数一起使用时，在文件名中使用星号（ * ）通配符。

此参数使用导致Data Pump验证名称以emp开头的目录/db1/hrdata/payroll/类型.dbf的所有文件是传输集的一部分。

例4

这个例子说明问号的使用（ ? ）与TRANSPORT_DATAFILES参数一起使用时文件名中的通配符。 

TRANSPORT_DATAFILES ='/ DB1 / HRDATA /工资/米？emp.dbf”

3.4.53 TRANSPORT_FULL_CHECK

Data Pump Import命令行模式TRANSPORT_FULL_CHECK参数指定是否验证其他表空间中的对象是否正在引用指定的可传输表空间集。

默认值： NO

目的

指定是否验证其他表空间中的对象是否正在引用指定的可传输表空间集。

语法和说明

TRANSPORT_FULL_CHECK = [是|没有]

如果TRANSPORT_FULL_CHECK=YES ，则Import会验证可传输集内的这些对象与可传输集外的对象之间是否存在依赖关系。该检查解决了双向依赖关系。例如，如果表在可传输集内，但其索引不在，则返回失败并终止导入操作。同样，如果索引位于可传输集中但表不是，则也会返回失败。

如果TRANSPORT_FULL_CHECK=NO ，则Import仅验证可传输集中没有依赖于可传输集外部对象的对象。此检查解决了单向依赖关系。例如，一台是不依赖于指数，但是指数是依赖于表，因为没有一个表的索引是没有意义的。因此，如果可传输集包含表而不包含其索引，则此检查成功。但是，如果可传输集包含索引而不包含表，则终止导入操作。

除了此检查之外，Import还会验证在TRANSPORT_TABLESPACES指定的表空间集中定义的所有表（及其索引）的所有存储段是否实际包含在表空间集中。

限制

  • 仅当指定了NETWORK_LINK参数时，此参数才对可传输模式（或在导出时指定TRANSPORTABLE=ALWAYS为表模式或完整模式）有效。

例

在以下示例中， source_database_link将替换为有效数据库链接的名称。该示例还假定已存在名为tbs6.dbf的数据文件。

假设您有一个参数文件full_check.par ，其中包含以下内容：

DIRECTORY = dpump_dir1 TRANSPORT_TABLESPACES = tbs_6 NETWORK_LINK = source_database_link TRANSPORT_FULL_CHECK = YES TRANSPORT_DATAFILES ='/ wkdir / data / tbs6.dbf'

然后，您可以发出以下命令：

> impdp hr PARFILE = full_check.par

3.4.54 TRANSPORT_TABLESPACES

数据泵导入命令行模式TRANSPORT_TABLESPACES参数指定您希望通过数据库链接以可传输表空间模式执行导入（使用NETWORK_LINK参数指定）。

默认值：没有默认值。

目的

指定您希望通过数据库链接以可传输表空间模式执行导入（使用NETWORK_LINK参数指定）。

语法和说明

TRANSPORT_TABLESPACES = tablespace_name [，...]

使用TRANSPORT_TABLESPACES参数指定将源对象元数据从源数据库导入目标数据库的表空间名称列表。

由于此导入是可传输模式导入，因此Data Pump将自动创建导入数据的表空间。您无需预先创建它们。但是，在开始导入之前，请将数据文件复制到目标数据库。

在导入命令行上指定TRANSPORT_TABLESPACES时，还必须使用NETWORK_LINK参数指定数据库链接。数据库链接是两个物理数据库服务器之间的连接，允许客户端将它们作为一个逻辑数据库进行访问。因此，需要NETWORK_LINK参数，因为对象元数据是从源（ NETWORK_LINK指向的数据库）导出的，然后使用该数据库链接直接导入目标（发出impdp命令的数据库）。在这种情况下没有涉及转储文件。如果使用其他方法在单独的操作中将实际数据复制到目标，则指定TRANSPORT_DATAFILES参数并指示数据的位置。

注意：

如果已有可传输表空间模式导出生成的转储文件集，则可以执行该转储文件的可传输模式导入，但在这种情况下，您不指定TRANSPORT_TABLESPACES或NETWORK_LINK 。这样做会导致错误。而是指定转储文件（包含元数据）和TRANSPORT_DATAFILES参数。TRANSPORT_DATAFILES参数的存在告诉import它是可传输模式导入以及从何处获取实际数据。

执行可传输作业时，最佳做法是在源系统上保留数据文件的副本，直到在目标系统上成功完成导入作业。如果导入作业失败，那么您仍然拥有未损坏的数据文件副本。

限制

  • 您无法导出可传输的表空间，然后将它们导入到较低版本级别的数据库中。要导入的目标数据库必须与源数据库处于相同或更高版本级别。

  • TRANSPORT_TABLESPACES参数仅在指定NETWORK_LINK参数时有效。

  • 要使用TRANSPORT_TABLESPACES参数执行可传输表空间导入， COMPATIBLE初始化参数必须至少设置为11.0.0。

  • 根据您的操作系统，为此参数指定值时使用引号也可能需要使用转义字符。Oracle建议您将此参数放在参数文件中。如果使用参数文件，则可以减少必须在命令行上使用的转义字符数。

  • 可传输表空间作业不支持Data Pump Import的ACCESS_METHOD参数。

例

在以下示例中， source_database_link将替换为有效数据库链接的名称。该示例还假定已将名为tbs6.dbf的数据文件从源数据库复制到本地系统。假设您有一个参数文件tablespaces.par,其中包含以下内容：

DIRECTORY = dpump_dir1 NETWORK_LINK = source_database_link TRANSPORT_TABLESPACES = tbs_6 TRANSPORT_FULL_CHECK = NO TRANSPORT_DATAFILES ='user01 / data / tbs6.dbf'

然后，您可以发出以下命令：

> impdp hr PARFILE = tablespaces.par

3.4.55可运输

“数据泵导入”命令行模式TRANSPORTABLE参数指定是否要在表模式导入（使用TABLES参数指定）或完整模式导入期间使用可传输选项。

默认值： NEVER

目的

指定在表模式导入（使用TABLES参数指定）或完全模式导入（使用FULL参数指定）期间是否应使用可传输选项。KEEP_READ_ONLY和NO_BITMAP_REBUILD被添加为TRANSPORTABLE参数值。KEEP_READ_ONLY和NO_BITMAP_REBUILD对可传输模式导入操作有效。

语法和说明

TRANSPORTABLE = [NEVER | ALWAYS | KEEP_READ_ONLY | NO_BITMAP_REBUILD]

允许值的定义如下：

  • ALWAYS ：指示导入作业使用可移动选项。如果无法运输，则作业失败。

    在表模式导入中，使用transportable选项会导致可传输的表空间导入，其中仅导入指定表，分区或子分区的元数据。

    在完全模式导入中，使用transportable选项将导致完全可传输导入，其中导入指定数据库中所有对象的元数据。

    在这两种情况下，您必须在单独的操作中将实际数据文件复制（并可能转换）到目标数据库。

    执行可传输作业时，最佳做法是在源系统上保留数据文件的副本，直到在目标系统上成功完成导入作业。如果导入作业由于某种原因失败，您仍将拥有未损坏的数据文件副本。

  • NEVER ：指示导入作业使用直接路径或外部表方法来加载数据，而不是使用可传输选项。这是默认值。

    如果仅导入表的分区的子集并且使用了TRANSPORTABLE=ALWAYS参数，则每个分区将成为非分区表。

    如果仅导入表的分区的子集且未使用TRANSPORTABLE参数，或者参数设置为NEVER （默认值），则以下选项适用：

    • PARTITION_OPTIONS=DEPARTITION ：指定将每个分区创建为非分区表。

    • PARTITION_OPTIONS ：如果未指定值，则此设置允许创建完整表。也就是说，存在完整表的所有元数据，因此表定义在目标系统上与在源上看起来相同。但只有指定分区的数据才会插入表中。

  • KEEP_READ_ONLY ：表示您希望可传输数据文件保持read-only模式。如果使用KEEP_READ_ONLY ，则可以将数据文件插入一个或多个其他兼容数据库以进行只读访问。

  • NO_BITMAP_REBUILD ：表示您不希望重建可传输数据文件头位图。不回收未使用的数据段会减少导入操作的时间。您可以使用dbms_space_admin.tablespace_rebuild_bitmaps重建位图。

  • TRANSPORTABLE=KEEP_READ_ONLY ：如果指定，则数据文件保持read-only模式。作为此设置的结果，无法更新包含TSTZ列数据的表，并从导入中删除这些表。此外，无法重建数据文件位图。

  • TRANSPORTABLE=NO_BITMAP_REBUILD ：如果指定，那么如果在导入期间有必要更新包含TSTZ列数据的表，则数据文件将被置于read/write模式。

API或类

您可以使用现有过程DBMS_DATAPUMP.SET_PARAMETER.设置TRANSPORTABLE参数值DBMS_DATAPUMP.SET_PARAMETER.

限制

  • 仅当还指定了NETWORK_LINK参数时，Import TRANSPORTABLE参数才有效。

  • TRANSPORTABLE参数仅在表模式导入和完整模式导入中有效。

  • 执行运输导入用户既需要DATAPUMP_EXP_FULL_DATABASE源数据库上的作用，以及DATAPUMP_IMP_FULL_DATABASE在目标数据库上的作用。

  • 具有为网络导入选择的存储的所有对象必须在源系统上具有完全在管理，不可传输的表空间（ SYSTEM / SYSAUX ）内的所有存储段，或者完全在用户定义的可传输表空间内。单个对象的存储不能跨越两种表空间。

  • 要使用TRANSPORTABLE参数执行基于网络的完全可传输导入，如果源数据库是版本11.2.0.3，则必须将Data Pump VERSION参数设置为至少12.0。如果源数据库是版本12.1或更高版本，则不需要VERSION参数，但COMPATIBLE数据库初始化参数必须设置为12.0.0或更高版本。

3.4.56版本

Data Pump Import命令行模式VERSION参数指定要导入的数据库对象的版本;仅导入与指定版本兼容的数据库对象和属性。

默认值：您很少需要在导入操作上指定VERSION参数。数据泵使用以下中的任何一个：

  • 与转储文件关联的版本，或者在网络导入的情况下与源数据库关联的版本

  • 由目标数据库上的COMPATIBLE初始化参数指定的版本

目的

指定要导入的数据库对象的版本（即，仅导入与指定版本兼容的数据库对象和属性）。请注意，这并不意味着Data Pump Import可以与10.1之前的Oracle数据库版本一起使用。Data Pump Import仅适用于Oracle Database 10g第 1版（10.1）或更高版本。VERSION参数只允许您标识要导入的对象的版本。

语法和说明

版本= [兼容|最新| version_string ]

此参数可用于加载其Oracle数据库处于早于源系统兼容性版本的目标系统。设置VERSION参数后，源系统上与指定版本不兼容的数据库对象或属性不会移动到目标。例如，不会导入包含指定版本中不支持的新数据类型的表。此参数的合法值如下：

  • COMPATIBLE - 这是默认值。元数据的版本对应于数据库兼容级别。数据库兼容性必须设置为9.2.0或更高版本。

  • LATEST - 元数据的版本对应于数据库版本。当目标数据库的实际版本晚于其COMPATIBLE初始化参数中指定的版本时，在导入作业上指定VERSION=LATEST无效。

  • version_string - 特定的数据库版本（例如，11.2.0）。

限制

  • 如果将Data Pump VERSION参数指定为早于12.1的任何值，则数据泵转储文件将排除包含长度超过4000字节的VARCHAR2或NVARCHAR2列以及长度超过2000字节的任何RAW列的任何表。

  • 如果目标是Oracle Database 12c第1版（12.1.0.1）或更高版本且源是Oracle Database 11g第2版（11.2.0.3）或更高版本，则通过网络链接执行的完全导入要求您设置VERSION=12 。

  • 使用Data Pump参数VERSION=12在Oracle Database 11g发行版上创建的转储文件只能在Oracle Database 12 c第1版（12.1）及更高版本中导入。

  • VERSION参数的值会根据是否启用数据绑定排序规则（DBC）而不同地影响导入。

例

在以下示例中，假定目标是Oracle数据库12 c解除1（12.1.0.1）数据库和该源为Oracle数据库11g第2版（11.2.0.3）数据库。在这种情况下，您必须为基于网络的导入设置VERSION=12 。另请注意，即使full是默认导入模式，也必须在使用NETWORK_LINK参数时在命令行上指定它。 

> impdp hr FULL = Y DIRECTORY = dpump_dir1 NETWORK_LINK = source_database_link VERSION = 12

3.4.57 VIEWS_AS_TABLES（网络导入）

数据泵导入命令行模式VIEWS_AS_TABLES （网络导入）参数指定您希望将一个或多个视图导入为表。

默认值：没有默认值。

注意：

VIEWS_AS_TABLES此描述适用于网络导入，这意味着您为Data Pump Import NETWORK_LINK参数提供了值。如果您执行的导入不是网络导入，请参阅VIEWS_AS_TABLES （非网络导入）。

目的

指定您希望将一个或多个视图作为表导入。

语法和说明

VIEWS_AS_TABLES = [schema_name。] view_name [：table_name]，...

Data Pump导入一个表，该表具有与视图相同的列，并从视图中获取行数据。Data Pump还会导入依赖于视图的对象，例如授权和约束。不会导入不适用于表的从属对象（例如， UNDER对象权限的授予）。您可以VIEWS_AS_TABLES使用VIEWS_AS_TABLES参数，也可以与TABLES参数一起使用。如果使用其中任何一个，Data Pump将执行表模式导入。

语法元素定义如下：

schema_name ：视图所在的模式的名称。如果未提供架构名称，则默认为执行导入的用户。

view_name ：要作为表导入的视图的名称。视图必须存在，并且它必须是仅具有标量非LOB列的关系视图。如果指定了无效或不存在的视图，则会跳过该视图并返回错误消息。

table_name ：用作导入视图的元数据源的表的名称。默认情况下，Data Pump会自动创建一个临时“模板表”，其中包含与视图相同的列和数据类型，但没有行。如果数据库是只读的，则模板表的默认创建将失败。在这种情况下，您可以指定表名。该表必须与视图位于同一模式中。它必须是具有堆组织的非分区关系表。它不能是嵌套表。

如果导入作业包含具有显式指定的模板表的多个视图，则模板表必须全部不同。例如，在以下作业中（其中两个视图使用相同的模板表），将跳过其中一个视图：

impdp hr DIRECTORY = dpump_dir NETWORK_LINK = dblink1 VIEWS_AS_TABLES = v1：employees，v2：employees

返回报告省略对象的错误消息。

导入操作完成后，模板表将自动删除。虽然它们存在，但您可以执行以下查询来查看其名称（全部以KU$VAT开头）：

SQL> SELECT * FROM user_tab_comments WHERE table_name LIKE'KU $ VAT％'; TABLE_NAME TABLE_TYPE ------------------------------ -----------评论------ ----------------------------------------------- KU $ VAT_63629 TABLE数据泵元数据模板表，用于查看HR.EMPLOYEESV

限制

  • VIEWS_AS_TABLES参数不能与TRANSPORTABLE=ALWAYS参数一起使用。

  • 使用VIEWS_AS_TABLES参数创建的表不包含属于指定视图的任何隐藏列。

  • VIEWS_AS_TABLES参数不支持具有数据类型为LONG列的表。

例

以下示例执行网络导入以从只读数据库导入视图hr.v1的内容。源数据库上的hr模式必须包含与视图view1具有相同几何形状的模板表（调用此表view1_tab ）。VIEWS_AS_TABLES参数列出了以冒号分隔的视图名称和表名：

> impdp hr VIEWS_AS_TABLES = view1：view1_tab NETWORK_LINK = dblink1

视图将作为名为view1的表导入，并从视图中提取行。从模板表view1_tab复制表的元数据。

Oracle Data Pump通过允许基于URL的转储文件名称的通配符，简化了从Oracle Object Store Service将多个转储文件导入Oracle自治数据库的过程。

示例3-4基于URL的文件名中使用的通配符

此示例显示如何在文件名中使用通配符，以便从Oracle Object Store Service将多个转储文件导入Oracle自治数据库。

> impdp admin / password @ ATPC1_high directory = data_pump_dir credential = my_cred_name dumpfile = https://objectstorage.example.com/v1/atpc/atpc_user/exp%u.dmp“

注意：

您不能在URL的bucket-name组件中使用通配符。
