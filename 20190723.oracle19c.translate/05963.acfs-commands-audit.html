acfsutil audit init

目的

初始化Oracle ACFS审核。

语法和说明

acfsutil audit init -h acfsutil audit init -M audit_manager_group -A auditor_group

acfsutil audit init -h显示帮助文本和退出。

表16-48包含acfsutil audit init命令可用的选项。

表16-48 acfsutil audit init命令的选项

选项 描述

-M audit_manager_group

指定将用户分配给Oracle ACFS审计管理员角色的操作系统（OS）组。要实现职责分离，您应该专门为此目的创建一个新组，该组应该与系统管理员组，Oracle ACFS安全管理员操作系统组和Oracle ACFS审计审计员操作系统组不同。

-A auditor_group

指定将用户分配给Oracle ACFS审核审核员角色的操作系统（OS）组。要实现职责分离，您应该专门为此目的创建一个新组，该组应该与系统管理员组，Oracle ACFS安全管理员OS组和Oracle ACFS审计管理器OS组不同。

在为文件系统上的任何审计源启用审计之前，系统管理员必须运行acfsutil audit init命令。该命令设置审计所需的角色，必须在可以在文件系统上启用任何类型的审计之前运行。运行该命令后，默认情况下会为所有使用审核的功能启用审核。

由于初始化后无法为Oracle ACFS审计管理器或审计员选择其他OS组，因此在初始化Oracle ACFS审计之前，应为这些特定目的创建新的OS组。

只有系统管理员才能运行此命令。

例子

以下示例显示了acfsutil audit init命令的acfsutil 。

示例16-43使用acfsutil audit init命令

＃/ sbin / acfsutil audit init -M myaudit_mgr_grp -A myauditor_grp

表16-49包含acfsutil audit purge命令可用的选项。

表16-49 acfsutil audit purge命令的选项

选项 描述

-m mount_point

指定安装文件系统的目录。

-F

即使审计审计员未使用acfsutil audit read命令将文件标记为已读，也强制删除审计跟踪。

只有审计经理才能运行此命令。
