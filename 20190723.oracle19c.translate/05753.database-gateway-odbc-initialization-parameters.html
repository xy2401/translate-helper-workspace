D初始化参数

init.ora文件中的Oracle数据库初始化参数与网关初始化参数不同。使用特定于代理的机制在初始化参数文件中设置网关参数，或使用DBMS_HS包在Oracle数据字典中设置它们。

网关启动时，网关初始化参数文件必须可用。

以下主题包含可以为每个网关及其描述设置的网关初始化参数的列表。主题还描述了初始化参数文件语法。

初始化参数文件语法

本主题说明初始化参数文件的语法。

  • 该文件是一系列命令。

  • 每个命令应该从单独的一行开始。

  • 行尾被视为命令终止符（除非使用反斜杠进行转义）。

  • 如果初始化参数文件中存在语法错误，则所有设置都不会生效。

  • 设置参数值如下：

    [SET] [PRIVATE] 参数 = 值

    哪里：

    parameter是初始化参数名称。它是一个以字母开头的字符串，由字母，数字和下划线组成。初始化参数名称区分大小写。

    value是初始化参数值。它区分大小写。初始化参数值是：

    • 一串不包含任何反斜杠，空格或双引号（“）的字符。

    • 带引号的字符串，以双引号开头，以双引号结尾。以下内容可以在带引号的字符串中使用：

      • 反斜杠（\）是转义字符

      • \ n插入一个新行

      • \ t插入一个标签

      • \“插入双引号

      • \\插入一个反斜杠

      该行末尾的反斜杠继续下一行的字符串。如果反斜杠在任何其他字符之前，则忽略反斜杠。

    例如，要为代理启用跟踪，请按如下所示设置HS_FDS_TRACE_LEVEL初始化参数：

     HS_FDS_TRACE_LEVEL = ON

    SET和PRIVATE是可选关键字。您不能将它们用作初始化参数名称。大多数参数仅作为初始化参数需要，因此您通常不需要使用SET或PRIVATE关键字。如果未指定SET或PRIVATE ，则该参数仅用作代理的初始化参数。

    SET指定除了用作初始化参数之外，还将参数值设置为代理进程的环境变量。将SET用于驱动程序或非Oracle系统需要的参数值作为环境变量。

    PRIVATE指定初始化参数应该是代理的私有参数，不应上载到Oracle数据库。大多数初始化参数不应该是私有的。但是，如果您在初始化参数文件中存储密码等敏感信息，则可能不希望将其上载到服务器，因为上载时初始化参数和值未加密。将初始化参数设置为私有可防止上载发生，并且它们不会出现在动态性能视图中。仅当参数值包含敏感信息（如用户名或密码）时，才使用PRIVATE作为初始化参数。

    SET PRIVATE指定将参数值设置为代理进程的环境变量，并且也是私有的（不会传输到Oracle数据库，不会出现在动态性能视图或图形用户界面中）。

用于ODBC初始化参数的Oracle数据库网关

本主题列出了可以为Oracle Database Gateway for ODBC设置的初始化文件参数。

HS_DB_DOMAIN

为非Oracle系统指定唯一的网络子地址。

属性 描述

默认值

世界

价值范围

1到199字符

HS_DB_DOMAIN初始化参数类似于Oracle Database Reference描述的DB_DOMAIN初始化参数。如果使用Oracle Names服务器，则需要HS_DB_DOMAIN初始化参数。HS_DB_NAME和HS_DB_DOMAIN初始化参数定义非Oracle系统的全局名称。

注意：

HS_DB_NAME和HS_DB_DOMAIN初始化参数必须组合以在协作服务器环境中形成唯一地址。

HS_DB_INTERNAL_NAME

指定标识异构服务代理连接到的实例的唯一十六进制数。

属性 描述

默认值

01010101

价值范围

1到16十六进制字符

激活全局名称服务时，此参数的值将用作事务ID的一部分。当事务需要两阶段提交恢复操作时，指定非唯一编号可能会导致问题。

HS_DB_NAME

为给予非Oracle系统的数据存储指定唯一的字母数字名称。

属性 描述

默认值

HO

价值范围

1至8字符

这个名字标识了协作服务器环境中的非Oracle系统。HS_DB_NAME和HS_DB_DOMAIN初始化参数定义非Oracle系统的全局名称。

HS_DESCRIBE_CACHE_HWM

指定异构服务使用的描述高速缓存中的最大条目数。

属性 描述

默认值

100

价值范围

1到4000

这个限制被称为描述缓存高水位线。缓存包含异构服务重用的映射表的描述，因此它不必重新访问非Oracle数据存储。

如果要访问许多映射表，请增加高水位标记以提高性能。增加高水位标记会以内存使用为代价提高性能。

HS_LANGUAGE

为异构服务提供非Oracle数据源的字符集，语言和区域信息。

属性 描述

默认值

系统专用

价值范围

任何有效的语言名称（最多255个字符）

该值必须使用以下格式：

语言 [_ 领土 。 character_set ]

注意：

全球化支持初始化参数会影响错误消息，SQL Service的数据以及分布式外部过程中的参数。

语言

HS_LANGUAGE初始化参数的语言组件确定：

  • 日期的日期和月份名称

  • 日期和时间的AD，BC，PM和AM符号

  • 默认排序机制

请注意，Oracle不确定通用异构服务消息（ ORA-25000到ORA-28000 ）的错误消息的语言。它们由Oracle数据库中的会话设置控制。

字符集

理想情况下，Oracle数据库和非Oracle数据源的字符集是相同的。在几乎所有情况下， HS_LANGUAGE应设置为与Oracle数据库字符集完全相同，以实现最佳字符集映射和性能。如果它们不相同，则异构服务会尝试将非Oracle数据源的字符集转换为Oracle数据库字符集，然后再返回。翻译会降低性能。在某些情况下，异构服务无法将字符从一个字符集转换为另一个字符集。

注意：

指定的字符集必须是安装代理程序的平台上的操作系统字符集的超集。

由于更多Oracle数据库和非Oracle数据库使用Unicode作为数据库字符集，因此最好还以Unicode字符集运行网关。为此，您必须设置HS_LANGUAGE=AL32UTF8 。但是，当网关在Windows上运行时，Microsoft ODBC驱动程序管理器界面只能以双字节字符集UCS2交换数据。这导致所述缓冲液和柱尺寸的额外比率扩展。有关如何调整到正确尺寸的说明，请参阅HS_FDS_REMOTE_DB_CHARSET 。

领土

territory子句指定日期和星期编号的约定，默认日期格式，十进制字符和组分隔符以及ISO和本地货币符号。请注意，Oracle数据库与非Oracle数据源之间的全球化支持级别取决于网关的实现方式。

HS_LONG_PIECE_TRANSFER_SIZE

设置要传输的LONG数据的大小。

属性 描述

默认值

64 KB

价值范围

任何值最多2 GB

较小的块大小意味着更少的内存需求，但更多的往返来获取所有数据。较大的单件尺寸意味着更少的往返次数，但更多的内存需要在内部存储中间件。因此，初始化参数可用于调整系统以获得最佳性能，在往返和内存要求之间以及网络延迟或响应时间之间进行最佳权衡。

HS_OPEN_CURSORS

定义在与非Oracle系统实例的一个连接上可以打开的最大游标数。

属性 描述

默认值

50

价值范围

1到Oracle数据库的OPEN_CURSORS初始化参数的值

该值永远不会超过Oracle数据库中打开游标的数量。因此，建议在Oracle数据库中设置与OPEN_CURSORS初始化参数相同的值。

HS_RPC_FETCH_REBLOCKING

控制异构服务是否尝试优化Oracle数据库与连接到非Oracle数据存储的异构服务代理之间的数据传输性能。

属性 描述

默认值

上

价值范围

OFF或ON

可能的值如下：

  • OFF禁用对已获取数据的重新锁定，以便立即将数据从代理发送到服务器。

  • ON启用重新锁定，这意味着从非Oracle系统获取的数据将缓冲在代理中，并且在获取的数据量等于或高于HS_RPC_FETCH_SIZE初始化参数的值之前不会发送到Oracle数据库。但是，当提取指示不存在更多数据或非Oracle系统报告错误时，将立即返回任何缓冲数据。

HS_RPC_FETCH_SIZE

调整内部数据缓冲以优化服务器和代理进程之间的数据传输速率。

属性 描述

默认值

50000

价值范围

1到10000000

增加该值可以减少传输给定数量的数据所需的网络往返次数，但是也倾向于增加数据带宽并减少在发出查询和完成查询的所有提取之间所测量的等待时间。然而，增加提取大小会增加查询的初始提取结果的延迟，因为在其他数据可用之前不会传输第一次提取结果。

HS_TIME_ZONE

指定当前SQL会话的默认本地时区位移。

属性 描述

'[+ | - ] hh：mm'的默认值

派生自NLS_TERRITORY初始化参数

'[+ | - ] hh：mm'的值范围

任何有效的日期时间格式掩码

格式掩码[+ | - ] hh：mm，用于指示UTC之前或之后的小时和分钟（协调世界时 - 以前的格林威治标准时间）。例如：

HS_TIME_ZONE = [+ | - ] hh：mm

HS_TRANSACTION_MODEL

指定事务更新非Oracle数据库时使用的事务模型的类型。

属性 描述

默认值

没有

价值范围

READ_ONLY和SINGLE_SITE

可能的值如下：

  • READ_ONLY提供对非Oracle数据库的读访问权限。

  • SINGLE_SITE提供对非Oracle数据库的读写访问。但是，网关无法参与分布式更新，也无法与Oracle Streams一起使用来复制数据。

IFILE

使用IFILE初始化参数在当前初始化文件中嵌入另一个初始化文件。

属性 描述

默认值

没有

价值范围

有效参数文件名

该值应该是绝对路径，不应包含环境变量。嵌套限制的三个级别不适用。

HS_FDS_TIMESTAMP_MAPPING

将非Oracle时间戳映射到Oracle时间戳。

属性 描述

默认值

日期

价值范围

CHAR | DATE|TIMESTAMP

句法

HS_FDS_TIMESTAMP_MAPPING={ CHAR|DATE|TIMESTAMP}

如果设置为CHAR ，则非Oracle目标时间戳将映射到CHAR(26) 。如果设置为DATE （默认值），则非Oracle目标时间戳将映射到Oracle DATE 。如果设置为TIMESTAMP ，则非Oracle目标时间戳将映射到Oracle TIMESTAMP 。

HS_FDS_DATE_MAPPING

将非Oracle目标日期映射到Oracle目标日期。

属性 描述

默认值

日期

价值范围

DATE | CHAR

句法

HS_FDS_DATE_MAPPING={ DATE|CHAR}

如果设置为CHAR ，则非Oracle目标日期将映射到CHAR(10) 。如果设置为DATE ，则非Oracle目标日期将映射到Oracle Date。

HS_FDS_CONNECT_INFO

HS_FDS_CONNECT_INFO描述了与非Oracle系统的连接。

属性 描述

默认值

没有

价值范围

不适用

默认初始化参数文件已有此参数的条目。网关的HS_FDS_CONNECT_INFO语法如下：

HS_FDS_CONNECT_INFO = dsn_value

其中，Microsoft Windows上的dsn_value是Microsoft Windows ODBC数据源管理器和基于UNIX的系统中定义的系统DSN的名称，它是在odbc.ini文件中配置的数据源名称。

dsn_value的条目区分大小写。

HS_FDS_DEFAULT_OWNER

如果未在SQL语句中指定所有者，则用于非Oracle数据库表的表所有者的名称。

属性 描述

默认值

没有

价值范围

不适用

注意：

如果未指定此参数且未在SQL语句中显式指定所有者，则使用Oracle用户的用户名或创建数据库链接时指定的用户名。

HS_FDS_TRACE_LEVEL

指定是否为网关连接打开或关闭错误跟踪。

属性 描述

默认值

关闭

价值范围

OFF ， ON ， DEBUG

以下值有效：

  • OFF禁用跟踪错误消息。

  • ON可以跟踪遇到问题时出现的错误消息。结果默认写入安装网关的LOG目录中的网关日志文件。

  • DEBUG允许跟踪可用于调试的详细错误消息。

HS_FDS_SHAREABLE_NAME

指定ODBC驱动程序管理器的完整路径名。

属性 描述

默认值

没有

价值范围

不适用

这是必需参数，其格式为：

HS_FDS_SHAREABLE_NAME = odbc_installation_path /lib/libodbc.sl

哪里：

odbc_installation_path是安装ODBC驱动程序的路径。

此参数仅适用于基于UNIX的平台。

HS_FDS_FETCH_ROWS

HS_FDS_FETCH_ROWS指定获取阵列大小。这是从非Oracle数据库中提取并一次返回Oracle数据库的行数。

属性 描述

默认值

100

价值范围

1到1000之间的任何整数

句法

HS_FDS_FETCH_ROWS= num

该参数将受HS_RPC_FETCH_SIZE和HS_RPC_FETCH_REBLOCKING参数的影响。

HS_FDS_REMOTE_DB_CHARSET

仅当HS_LANGUAGE设置为AL32UTF8且网关在Windows上运行时，此参数才有效。

属性 描述

默认值

没有

价值范围

不适用

句法

HS_FDS_REMOTE_DB_CHARSET

由于更多Oracle数据库和非Oracle数据库使用Unicode作为数据库字符集，因此最好还以Unicode字符集运行网关。为此，您必须设置HS_LANGUAGE=AL32UTF8 。但是，当网关在Windows上运行时，Microsoft ODBC驱动程序管理器界面只能以双字节字符集UCS2交换数据。这导致所述缓冲液和柱尺寸的额外比率扩展。为了补偿，如果HS_FDS_REMOTE_DB_CHARSET设置为相应的非Oracle数据库字符集，则网关可以调整为正确的大小。例如， HS_FDS_REMOTE_DB_CHARSET=KO16KSC5601 。

HS_FDS_SQLLEN_INTERPRETATION

此参数仅对64位平台有效。ODBC标准指定SQLLEN （内部ODBC构造）在64位平台上为64位，但某些ODBC驱动程序管理器和驱动程序违反了此约定，并将其实现为32位。

属性 描述

默认值

64

价值范围

{64 | 32}

句法

HS_FDS_SQLLEN_INTERPRETATION = {64 | 32}

为了使Oracle Database Gateway for ODBC能够补偿其行为，如果使用这些类型的驱动程序管理器和驱动程序，则需要指定HS_FDS_SQLLEN_INTERPRETATION=32 。

HS_FDS_REPORT_REAL_AS_DOUBLE

启用Oracle Database Gateway for ODBC将SINGLE FLOAT PRECISION字段视为DOUBLE FLOAT PRECISION字段。

属性 描述

默认值

假

价值范围

TRUE ， FALSE

默认值为FALSE 。
