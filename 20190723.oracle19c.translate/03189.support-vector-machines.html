27.1关于支持向量机

支持向量机（SVM）是一种功能强大，最先进的算法，具有基于Vapnik-Chervonenkis理论的强大理论基础。SVM具有强大的正则化特性。正则化是指将模型推广到新数据。

27.1.1 SVM的优点

Oracle Data Mining SVM实现包括两种类型的解算器，一种内点法（IPM）求解器和a 子梯度下降（SGD）求解器。IPM求解器提供非常稳定和准确的解决方案，但是，它可能无法处理非常高维度的数据。对于高维数据，例如文本，评级等，SGD求解器是更好的选择。两个求解器都具有高度可扩展的并行实现，可以处理大量数据。

27.1.2 Oracle数据挖掘中SVM的优点

Oracle Data Mining有自己的支持向量机（SVM）专有实现，它利用了算法的许多优点，同时补偿了SVM框架中固有的一些限制。Oracle Data Mining SVM提供了生产质量数据挖掘系统所需的可扩展性和可用性。

27.1.2.1可用性

解释Oracle Data Mining中支持向量机（SVM）的可用性。

可用性是一项重大改进，因为SVM经常被视为专家的工具。该算法通常需要数据准备，调整和优化。Oracle Data Mining最大限度地减少了这些要求。您无需成为专家即可在Oracle Data Mining中构建高质量的SVM模型。例如：

27.1.2.2可扩展性

了解如何扩展支持向量机（SVM）的数据。

处理非常大的数据集时，通常需要采样。但是，Oracle Data Mining SVM不需要采样，因为算法本身使用分层采样来根据需要减小训练数据的大小。

Oracle Data Mining SVM经过高度优化。它通过优化小型工作集向全局解决方案逐步建立模型。训练模型直到在当前工作集上收敛，然后模型适应新数据。该过程迭代地继续，直到满足收敛条件。高斯内核使用缓存技术来管理工作集。

27.1.3基于内核的学习

了解基于内核的函数来转换支持向量机（SVM）的输入数据。

SVM是一种基于内核的算法。内核是一种将输入数据转换为解决问题的高维空间的函数。核函数可以是线性的或非线性的。

Oracle Data Mining支持线性和高斯（非线性）内核。

在Oracle数据挖掘中， 线性核函数简化为训练数据中原始属性的线性方程。当训练数据中有许多属性时，线性内核很有效。

高斯内核将训练数据中的每个情况转换为n维空间中的点，其中n是个案数。该算法试图将点分成具有同质目标值的子集。高斯核使用非线性分离器，但在核空间内构造线性方程。

注意：

主动学习与Oracle Database 12 c第2版及更高版本无关。与Active Learning类似的设置是ODMS_SAMPLING 。

27.3 SVM的数据准备

支持向量机（SVM）算法本身在数字属性上运行。SVM对数字属性使用z-score标准化。仅对二维数字列（非嵌套）进行标准化。该算法自动将分类数据“爆炸”成一组二进制属性，每个类别值一个。例如，具有married或single价值的婚姻状况的字符列将转换为两个数字属性： married和single 。新属性的值可以是1 （true）或0 （false）。

当具有简单数据类型（非嵌套）的列中缺少值时，SVM会将它们随机解释为缺失。该算法自动用模式替换缺失的分类值，并用均值替换缺失的数值。

当嵌套列中存在缺失值时，SVM会将它们解释为稀疏。该算法自动用零和矢量稀疏分类数据替换稀疏数值数据。

27.3.1标准化

支持向量机需要数字输入的标准化。规范化将数字属性的值放在相同的比例上，并防止具有较大原始比例的属性偏向解决方案。归一化还可以最大限度地降低溢出和下溢的可能性。

27.3.2 SVM和自动数据准备

了解手动或通过支持向量机（SVM）的自动数据准备（ADP）处理和转换数据。

SVM算法自动处理缺失值处理和分类数据的转换，但规范化和异常值检测必须由自动数据准备（ADP）处理或手动准备。ADP对SVM执行最小 - 最大标准化。

注意：

Oracle建议您使用SVM自动数据准备。 ADP执行的转换适用于大多数模型。

27.4 SVM分类

支持向量机（SVM） 分类基于定义决策边界的决策平面的概念。决策平面是在具有不同类别成员资格的一组对象之间进行分隔的平面。SVM找到定义分隔符的向量（“支持向量”），给出最广泛的类分离。

SVM分类支持二进制，多类和多目标分类 。多目标将多个类别标签与一行相关联。目标类型是ORA_MINING_VARCHAR2_NT类型的集合。

27.4.1班级权重

了解何时在支持向量机（SVM）中为数据实现类权重。

在SVM分类中，权重是用于指定目标值（类）的相对重要性的偏置机制。

SVM模型会自动初始化，以实现所有类别的最佳平均预测。但是，如果训练数据不代表实际分布，则可以偏置模型以补偿代表性不足的类值。如果增加一个类的权重，那么该类的正确预测百分比必须增加。

27.5一类SVM

神谕数据挖掘使用支持向量机（SVM）作为异常检测的一类分类器。当SVM用于异常检测时，它具有分类挖掘功能但没有目标。

应用单类SVM模型时，会为评分数据中的每个案例生成预测和概率。如果预测为1，则认为该情况是典型的。如果预测为0，则该情况被认为是异常的。此行为反映了使用正常数据训练模型的事实。

您可以使用SVMS_OUTLIER_RATE构建设置指定预期异常的数据百分比。如果您知道“可疑”案例的数量是您的人口的一定百分比，那么您可以将离群率设置为该百分比。该模型在应用于一般人群时大致识别出许多“罕见”案例。

27.6 SVM回归

了解如何使用epsilon-insensitivity loss函数来解决支持向量机（SVM）中的回归问题。

SVM使用epsilon不敏感的损失函数来解决回归问题。

SVM回归试图找到连续函数，使得最大数据点数位于epsilon范围的不敏感管内。落在真实目标值的epsilon距离内的预测不会被解释为错误。

epsilon因子是SVM回归的正则化设置。它平衡了误差幅度和模型稳健性，以实现对新数据的最佳推广。
