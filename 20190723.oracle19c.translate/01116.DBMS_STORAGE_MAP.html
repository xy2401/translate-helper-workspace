166 DBMS_STORAGE_MAP

使用DBMS_STORAGE_MAP包，您可以与Oracle后台进程FMON进行通信，以调用填充映射视图的映射操作。FMON与操作和存储系统供应商提供的映射库进行通信。

本章包含以下主题：

  • 概观

  • 操作说明

  • DBMS_STORAGE_MAP子程序摘要

166.1 DBMS_STORAGE_MAP概述

此术语和说明将帮助您了解DBMS_STORAGE_MAP API。

  • 映射库

    映射库可帮助您映射I / O处理堆栈元素的组件。I / O处理组件的示例包括文件，逻辑卷和存储阵列I / O目标。映射库在filemap.ora中标识。

  • 映射文件

    映射文件是描述文件的映射结构。它提供了一组属性，包括文件大小，文件组成的扩展区数和文件类型。

  • 映射元素和子元素

    映射元素是描述I / O堆栈中的存储组件的抽象映射结构。元素的示例包括镜像，条带，分区，raid5，连接元素和作为映射构建块的磁盘结构。映射子元素描述元素与I / O映射堆栈中的下一个元素之间的链接

  • 映射文件范围

    映射文件范围描述了驻留在一个元素上的连续块块。这包括设备偏移量，范围大小，文件偏移量，类型（数据或奇偶校验）以及范围所在元素的名称。对于原始设备或卷，该文件仅由一个文件扩展区组件组成。映射文件范围与Oracle范围不同。

    也可以看看：

    • 有关更多信息，请参见Oracle数据库管理员指南

    • V$MAP视图的Oracle数据库参考 ，包括V$MAP_FILE ， V$MAP_ELEMENT ， V$MAP_SUBELEMENT ， V$MAP_FILE_EXTENT

166.2 DBMS_STORAGE_MAP操作说明

如果映射信息已存在，则调用MAP_ELEMENT ， MAP_FILE和MAP_ALL函数将刷新映射（如果支持配置ID）。如果不支持配置ID，则再次调用这些函数将重建映射。

也可以看看：

“Oracle数据库管理员指南”，用于讨论配置ID，即已更改的元素或文件的属性。

166.3.1 DROP_ALL函数

此函数删除实例的共享内存中的所有映射信息。

句法

DBMS_STORAGE_MAP.DROP_ALL（dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-2 DROP_ALL函数参数

参数 描述

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

166.3.2 DROP_ELEMENT函数

此函数删除elemname定义的元素的映射信息。

句法

DBMS_STORAGE_MAP.DROP_ELEMENT（elemname IN VARCHAR2，cascade IN BOOLEAN，dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-3 DROP_ELEMENT函数参数

参数 描述

elemname

删除映射信息的元素。

级联

如果TRUE ，则DROP_ELEMENT递归调用上由所限定的DAG中的所有元素elemname,如果可能的话。

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

166.3.3 DROP_FILE函数

此函数删除由filename定义的文件映射信息。

句法

DBMS_STORAGE_MAP.DROP_FILE（文件名IN VARCHAR2，级联IN BOOLEAN，dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-4 DROP_FILE函数参数

参数 描述

文件名

删除文件映射信息的文件。

级联

如果为TRUE ，则如果可能，也会删除文件所在元素的映射DAG。

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

166.3.4 LOCK_MAP程序

此过程将映射信息锁定在实例的共享内存中。

当您需要V $ MAP表的一致快照时，这非常有用。例如，在不锁定映射信息的情况下， V$MAP_ELEMENT和V$MAP_SUBELEMENT可能不一致。

句法

DBMS_STORAGE_MAP.LOCK_MAP;

166.3.5 MAP_ALL函数

此函数为所有类型的Oracle文件（存档日志除外）构建整个映射信息，包括所有有向非循环图（DAG）元素。它获取最新的映射信息，因为它显式同步所有映射库。

句法

DBMS_STORAGE_MAP.MAP_ALL（max_num_fileext IN NUMBER DEFAULT 100，dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-5 MAP_ALL函数参数

参数 描述

max_num_fileext

定义要映射的最大文件范围数。这限制了映射文件范围时使用的内存量。默认值为100 ; max_num_fileextent是一个重载参数。

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

使用说明

您必须在冷启动方案中显式调用MAP_ALL 。

166.3.6 MAP_ELEMENT函数

此函数为elemname标识的元素构建映射信息。如果要映射的元素或其I / O堆栈中的任何一个元素（如果cascade为TRUE ）由必须显式同步的库拥有，则它可能无法获取最新的映射信息。

句法

DBMS_STORAGE_MAP.MAP_ELEMENT（elemname IN VARCHAR2，cascade IN BOOLEAN，dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-6 MAP_ELEMENT函数参数

参数 描述

elemname

构建映射信息的元素。

级联

如果为TRUE ，则映射elemname I / O堆栈DAG中的所有元素。

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

166.3.7 MAP_FILE函数

此函数为filename标识的filename构建映射信息。如果某个特定文件的映射已更改，请使用此函数。Oracle数据库服务器不必重建整个映射。

句法

DBMS_STORAGE_MAP.MAP_FILE（文件名IN VARCHAR2，文件类型IN VARCHAR2，级联IN BOOLEAN，max_num_fileextent IN NUMBER DEFAULT 100，dictionary_update IN BOOLEAN DEFAULT TRUE）;

参数

表166-7 MAP_FILE函数参数

参数 描述

文件名

构建映射信息的文件。

文件类型

定义要映射的文件的类型。它可以是“ DATAFILE", "SPFILE", "TEMPFILE", "CONTROLFILE", "LOGFILE",或"ARCHIVEFILE" 。

级联

仅在发生存储重新配置时才应为TRUE 。对于所有其他实例，例如文件大小调整（通过ALTER SYSTEM命令或扩展文件上的DML操作），可以将cascade设置为FALSE因为映射更改仅限于文件范围。

如果为TRUE ，则还为文件所在的元素构建映射DAG。

max_num_fileextent

定义要映射的最大文件范围数。这限制了映射文件范围时使用的内存量。默认值为100 ; max_num_fileextent是一个重载参数。

dictionary_update

如果为TRUE ，则更新数据字典中的映射信息以反映更改。默认值为TRUE ; dictionary_update是一个重载的参数。

使用说明

如果正在映射的文件或其I / O堆栈中的任何一个元素（如果cascade为TRUE ）拥有必须显式同步的库，则此函数可能无法获取最新的映射信息。

166.3.8 MAP_OBJECT函数

此函数构建由对象名称，所有者和类型标识的Oracle对象的映射信息。

句法

DBMS_STORAGE_MAP.MAP_OBJECT（对象名为IN VARCHAR2，所有者为IN VARCHAR2，对象为IN VARCHAR2）;

参数

表166-8 MAP_OBJECT函数参数

参数 描述

objname表

对象的名称。

所有者

对象的所有者。

OBJTYPE

对象的类型。

166.3.9恢复功能

此函数将整个映射信息从数据字典加载到实例的共享内存中。

您只能在SAVE操作后调用RESTORE 。您必须在热启动方案中显式调用RESTORE 。

句法

DBMS_STORAGE_MAP.RESTORE;

166.3.10保存功能

此函数保存将整个映射重新生成到数据字典中所需的信息。

句法

DBMS_STORAGE_MAP.SAVE;

166.3.11 UNLOCK_MAP程序

此过程解锁实例的共享内存中的映射信息。

句法

DBMS_STORAGE_MAP.UNLOCK_MAP;
