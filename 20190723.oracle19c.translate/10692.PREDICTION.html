目的

PREDICTION返回选择中每行的预测。返回预测的数据类型取决于函数是执行回归，分类还是异常检测。

  • 回归 ：返回每行的预期目标值。返回值的数据类型是目标的数据类型。

  • 分类 ：返回每行最可能的目标类（或指定成本的最低成本目标类）。返回值的数据类型是目标的数据类型。

  • 异常检测 ：每行返回1或0 。典型行分类为1。与其他数据显着不同的行归类为0。

cost_matrix_clause

成本是最小化最有害的错误分类的偏向因素。您可以为分类或异常检测指定cost_matrix_clause 。成本与回归无关。cost_matrix_clause行为与“ PREDICTION_COST ”中描述的相同。

语法选择

PREDICTION可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，或者可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态评分数据。选择语法或分析语法 ：

  • 语法 ：使用此语法使用预定义的模型对数据进行评分。提供执行分类，回归或异常检测的模型的名称。

  • 分析语法 ：使用分析语法在没有预定义模型的情况下对数据进行评分。分析语法使用mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见“ analytic_clause :: = ” 。）

    • 对于回归，请指定FOR expr ，其中expr是一个表达式，用于标识具有数字数据类型的目标列。

    • 对于Classification，请指定FOR expr ，其中expr是一个表达式，用于标识具有字符数据类型的目标列。

    • 对于异常检测，请指定关键字OF ANOMALY 。

在对分区模型进行评分时， PREDICTION函数的语法可以使用可选的GROUPING提示。请参阅GROUPING提示 。

mining_attribute_clause

mining_attribute_clause标识要用作评分预测变量的列属性。

  • 如果指定USING * ，则使用输入行中存在的所有相关属性。

  • 如果使用分析语法调用函数，则mining_attribute_clause将用于构建瞬态模型和评分。

  • 使用预定义模型调用函数时， mining_attribute_clause应包含用于创建模型的全部或部分属性。以下条件适用：

    • 如果mining_attribute_clause包含具有相同名称但与用于创建模型的数据类型不同的数据类型的属性，则数据类型将转换为模型预期的类型。

    • 如果为评分指定的属性多于用于创建模型的属性，则会以静默方式忽略额外属性。

    • 如果您为评分指定的属性少于用于创建模型的评分属性，则评分将以尽力而为的方式执行。

例

在此示例中，模型dt_sh_clas_sample预测最有可能使用关联卡的客户的性别和年龄（target = 1）。PREDICTION功能考虑了与模型相关的成本矩阵，并使用婚姻状况，教育和家庭规模作为预测因子。 

SELECT cust_gender，COUNT（*）AS cnt，ROUND（AVG（age））AS avg_age FROM mining_data_apply_v WHERE PREDICTION（dt_sh_clas_sample使用cust_marital_status，education，household_size的成本模型）= 1 GROUP BY cust_gender ORDER BY cust_gender; CUST_GENDER CNT AVG_AGE ------------ ---------- ---------- F 170 38 M 685 42

与模型dt_sh_clas_sample关联的成本矩阵存储在表dt_sh_sample_costs 。成本矩阵指定1的错误分类比0的错误分类成本高8倍。
