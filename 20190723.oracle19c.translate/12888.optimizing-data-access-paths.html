13优化数据访问路径

为了实现数据密集型查询的最佳性能，物化视图和索引对于SQL语句至关重要。但是，实现这些对象并非没有代价。创建和维护这些对象可能非常耗时。空间要求可能很重要。SQL Access Advisor使您可以通过推荐实体化视图并查看特定工作负载的日志，索引，SQL配置文件和分区来优化查询访问路径。

物化视图通过将查询结果存储在单独的模式对象中来提供对表数据的访问。与不占用存储空间或包含数据的普通视图不同，实体化视图包含来自一个或多个基表或视图的查询的行。物化视图日志是一个模式对象，它记录对主表数据的更改，以便可以逐步刷新在主表上定义的物化视图。SQL Access Advisor建议如何优化实体化视图，以便快速刷新它们并使用查询重写功能。

SQL Access Advisor还建议使用位图，基于函数和B树索引。位图索引缩短了许多类型的即席查询的响应时间，并且与其他索引相比也可以减少存储空间。基于函数的索引计算涉及一列或多列的函数或表达式的值，并将其存储在索引中。B树索引通常用于索引唯一或接近唯一的密钥。

13.1运行SQL Access Advisor

本节介绍如何运行SQL Access Advisor以为SQL工作负载提供建议。

要运行SQL Access Advisor：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 选择要用于分析的工作负载源，如“ 选择工作负载源 ”中所述 。

  3. 定义过滤器选项，如“ 应用过滤器选项 ”中所述 。

  4. 选择建议类型，如“ 指定推荐选项 ”中所述 。

  5. 安排SQL Access Advisor任务，如“ 指定任务和计划选项 ”中所述 。

13.1.1选择初始选项

运行SQL Access Advisor的第一步是在SQL Access Advisor：Initial Options页面上选择初始选项。

要选择初始选项：

  1. 访问数据库主页。

    有关详细信息，请参阅“ 访问数据库主页 ” 。

  2. 从“ 性能”菜单中，选择“ 顾问主页” 。

    如果出现“数据库登录”页面，则以具有管理员权限的用户身份登录。将显示Advisor Central页面。

  3. 单击SQL Advisors 。

    将出现“SQL Advisors”页面。

  4. 单击“ SQL Access Advisor” 。

    将出现“SQL Access Advisor：初始选项”页面。

  5. 执行以下操作之一：

    • 选择仅验证访问结构的使用（索引，物化视图，分区等）以验证现有结构。

    • 选择推荐新访问结构以使用Oracle企业管理器云控制（云控制）默认模板中定义的建议选项。

      如果选择此选项，则可以选择完成以下步骤：

      • 从先前保存的任务或模板中选择继承选项，以使用现有SQL Access Advisor任务或其他模板中定义的选项。

      • 在“任务和模板”中，选择要使用的任务或模板。

    在此示例中，选择了“ 推荐新访问结构” 。


    插图描述sql_access_initial_options.gif

  6. 单击继续 。

    将出现“SQL Access Advisor：工作负载源”页面。

  7. 继续执行下一步，如“ 选择工作负载源 ”中所述 。

13.1.2选择工作负载源

为SQL Access Advisor指定初始选项后，选择工作负载源您要用于分析，如以下部分所述：

  • 使用缓存中的SQL语句

  • 使用现有SQL调整集

  • 使用假设的工作量

13.1.2.1使用缓存中的SQL语句

您可以使用缓存中的SQL语句作为工作负载源。但是，只有当前和最近的SQL语句存储在SQL缓存中，因此此工作负载源可能无法代表数据库上的整个工作负载。

要使用缓存中的SQL语句作为工作负载源：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 在“SQL Access Advisor：工作负载源”页面上，选择“ 当前和最近的SQL活动” 。


    插图描述sql_access_source_cache.gif

  3. 按照“ 应用过滤器选项 ”中的说明继续下一步。

13.1.2.2使用现有SQL调整集

您可以使用现有SQL调优集作为工作负载源。此选项很有用，因为SQL调优集可以重复用作SQL Access Advisor和SQL Tuning Advisor的工作负载源。

要将SQL调优集用作工作负载源：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 在“SQL Access Advisor：工作负载源”页面上，选择“ 使用现有SQL调整集” 。

  3. 单击SQL Tuning Set搜索图标以使用现有SQL调优集。

    将出现“搜索和选择：SQL调整集”对话框。

  4. 在“ 架构”字段中，输入包含要使用的SQL调整集的架构的名称，然后单击“执行” 。

    将显示所选模式中包含的SQL调优集列表。

  5. 选择要用于工作负载源的SQL调整集，然后单击“ 选择” 。

    “搜索和选择：SQL调整集”对话框将关闭，选定的SQL调整集现在将显示在“ SQL调整集”字段中。

  6. 按照“ 应用过滤器选项 ”中的说明继续下一步。

13.1.2.3使用假设的工作量

维度表存储星型或雪花型架构中逻辑维度的全部或部分值。您可以从包含主键或外键约束的维表创建假设工作负载。如果要分析的工作负载不存在，则此选项很有用。在这种情况下，SQL Access Advisor会检查当前的逻辑架构设计，并根据表之间定义的关系提供建议。

要使用假设工作负载作为工作负载源：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 在SQL Access Advisor：Workload Source页面上， 从以下架构和表中选择Create a Hotheothetical Workload 。

  3. 将Schemas和Tables保留为空，然后单击Add以搜索表。

    将显示“工作负载源：搜索和选择架构和表”页面。

  4. 在“表”部分中，在“ 架构”字段中输入架构名称，然后单击“ 搜索” 。

    将显示所选模式中的表列表。

  5. 选择要在创建假设工作负载时使用的表，然后单击“ 添加表” 。

    选定的表现在显示在“ 架构和表”字段中。

  6. 单击确定 。

    此时将显示“SQL Access Advisor：工作负载源”页面，其中添加了所选表。

  7. 按照“ 应用过滤器选项 ”中的说明继续下一步。

13.1.3应用过滤器选项

选择工作负载源后，您可以选择应用筛选器以减少工作负载中找到的SQL语句的范围。过滤器有益于以下原因：

  • 使用过滤器会指示SQL Access Advisor根据工作负载的特定SQL语句子集提出建议，这可能会带来更好的建议。

  • 使用过滤器可从工作负载中删除无关的SQL语句，这可能会大大缩短处理时间。

要将过滤器应用于工作负载源：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 选择工作负载源，如“ 选择工作负载源 ”中所述 。

  3. 在“SQL Access Advisor：工作负载源”页面上，单击“ 筛选选项” 。

    “过滤器选项”部分将展开。

  4. 选择基于这些选项筛选工作负载 。

    “筛选选项”部分已启用。

  5. 定义要应用的过滤器，如以下部分所述：

    • 定义资源消耗过滤器

    • 为用户定义过滤器

    • 定义表的过滤器

    • 为SQL文本定义过滤器

    • 定义模块的过滤器

    • 定义操作的过滤器

  6. 单击下一步 。

    将出现“建议选项”页面。

  7. 按照“ 指定推荐选项 ”中的说明继续执行下一步。

13.1.3.1定义资源消耗过滤器

资源消耗过滤器将工作负载限制为仅包括您指定的高负载SQL语句的数量。

要为资源消耗定义过滤器：

  1. 在“SQL Access Advisor：工作负载源”页上的“用户资源消耗”下，在“语句数”字段中输入高负载SQL语句的数量 。

  2. 从Order by列表中，选择要对其进行排序的方法之一。

13.1.3.2为用户定义过滤器

用户过滤器限制工作负载以包括或排除由您指定的用户执行的SQL语句。

要为用户定义过滤器：

  1. 在“SQL Access Advisor：工作负载源”页上的“用户”下，选择“ 仅包含这些用户执行的SQL语句”或“ 排除 这些用户执行的 所有SQL语句” 。

  2. 要搜索可用用户，请单击“用户”搜索图标。

    将出现“搜索并选择：用户”对话框。

  3. 选择要包含或排除其SQL语句的用户，然后单击“ 选择” 。

    “搜索并选择：用户”对话框将关闭，选定的表现在将显示在“ 用户”字段中。

    在此示例中，过滤器被定义为仅包括用户SH执行的SQL语句。


    插图描述sql_access_filter_users.gif
13.1.3.3定义表的过滤器

表过滤器限制工作负载以包括或排除访问您指定的表列表的SQL语句。如果您选择了“ 从以下架构和表中创建假设工作负载”选项，则不允许使用表过滤器，如“ 使用假设工作负载 ”中所述 。

要为表定义过滤器：

  1. 要仅包括访问特定表列表的SQL语句，请在“ 仅包括访问任何这些表的SQL语句”字段中输入表名。

  2. 要排除访问特定表列表的所有SQL语句，请在“ 排除访问任何这些表的所有SQL语句”字段中输入表名。

  3. 要搜索可用表，请单击“表”搜索图标。

    将出现“搜索和选择：架构和表”对话框。

  4. 选择要包含或排除SQL语句的表，然后单击“ 选择” 。

    “搜索和选择：架构和表”对话框将关闭，选定的表现在将显示在相应的“表”字段中。

13.1.3.4为SQL文本定义过滤器

SQL文本筛选器将工作负载限制为包含或排除包含您指定的SQL文本子字符串的SQL语句。

要为SQL文本定义过滤器：

  1. 要仅包含包含特定SQL文本的SQL语句，请输入要包含在仅包含这些SQL文本子字符串的SQL语句字段中的SQL文本 。

  2. 要排除包含特定SQL文本的所有SQL语句，请在“ 排除包含这些SQL文本子字符串的所有SQL语句”字段中输入要排除的SQL文本 。

13.1.3.5定义模块的过滤器

模块筛选器限制工作负载以包括或排除与您指定的模块关联的SQL语句。

要为模块ID定义过滤器：

  1. 执行以下操作之一：

    • 要仅在工作负载中包含与特定模块ID 关联的SQL语句 ，请选择“ 仅包括与这些模块关联的SQL语句” 。

    • 要从工作负载中排除与特定模块ID关联的所有SQL语句，请选择“ 排除与这些模块关联的所有SQL语句” 。

  2. 在“模块”字段中，输入包含或排除关联SQL语句的模块的名称。

13.1.3.6定义操作的过滤器

操作筛选器限制工作负载以包括或排除与您指定的操作关联的SQL语句。

要为操作定义过滤器：

  1. 执行以下操作之一：

    • 要仅包括与工作负荷中的特定操作关联的SQL语句，请选择“ 仅包括与这些操作关联的SQL语句” 。

    • 要从工作负载中排除与特定操作关联的所有SQL语句，请选择“ 排除与这些操作关联的所有SQL语句” 。

  2. 在“操作”字段中，输入包含或排除关联SQL语句的操作。

13.1.4指定推荐选项

为了改进优化程序为工作负载选择的基础数据访问方法，SQL Access Advisor为索引，物化视图和分区提供了建议。使用这些访问结构可以通过减少从数据库读取数据所需的时间来显着提高工作负载的性能。但是，您必须平衡使用这些访问结构的好处与维护它们的成本。

要指定推荐选项：

  1. 选择初始选项，如“ 选择初始选项 ”中所述 。

  2. 选择工作负载源，如“ 选择工作负载源 ”中所述 。

  3. 定义过滤器选项，如“ 应用过滤器选项 ”中所述 。

  4. 在SQL Access Advisor：Recommendation Options页面的Access Structures to Recommend下，选择SQL Access Advisor建议的访问结构类型：

    • 索引

    • 物化观点

    • 分区

    在此示例中，选择了所有前面的访问类型。


    说明sql_access_recommend_types.gif的说明

  5. 在“范围”下，选择SQL Access Advisor运行的模式。执行以下操作之一：

    • 选择有限 。

      在受限模式下，SQL Access Advisor专注于工作负载中成本最高的SQL语句。分析速度更快，但建议可能有限。

    • 选择综合 。

      在综合模式下，SQL Access Advisor会分析工作负载中的所有SQL语句。分析可能需要更长时间，但建议是详尽无遗的。

    在此示例中，选择了限制模式 。


    说明sql_access_advisor_mode.gif的说明

  6. （可选）单击“ 高级选项”

    “高级选项”部分将进行扩展。本节包含以下小节：

    • 工作负载分类

      在本节中，您可以指定要为其推荐的工作负载类型。可以使用以下类别：

      • 工作量波动

        如果工作负载主要包含只读操作， 请选择仅考虑查询 ，如数据仓库中所示。波动率数据对于在线事务处理（OLTP）系统非常有用，其中INSERT ， UPDATE和DELETE操作的性能至关重要。

      • 工作量范围

        如果工作负载代表所有访问结构用例，请选择推荐删除未使用的访问结构

    • 空间限制

      索引和物化视图以空间为代价提高性能。执行以下操作之一：

      • 选择否，显示所有建议（无限空间）以指定无空间限制。在没有空间限制的情况下调用SQL Access Advisor时，它会提供最佳性能建议。

      • 选择是，将额外空间限制为 ，然后输入以兆字节，千兆字节或太字节为单位的空间限制。使用空间限制调用SQL Access Advisor时，它仅生成空间要求不超过指定限制的建议。

    • 调整优先级

      本部分允许您指定如何调整SQL语句。完成以下步骤：

      • 从按列表优先调整SQL语句开始，选择要调整SQL语句的方法，然后单击“ 添加” 。

      • （可选）选择考虑访问结构创建成本建议，以权衡创建访问结构的成本与SQL语句执行时间的频率和潜在改进。否则，将忽略创建成本。如果希望为经常执行的SQL语句生成特定建议，则应选择此选项。

    • 默认存储位置

      使用此部分覆盖为架构和表空间位置定义的默认值。默认情况下，索引位于它们引用的表的架构和表空间中。物化视图位于查询中引用的第一个表的模式和表空间中。物化视图日志位于它们引用的表的模式的默认表空间中。

  7. 单击下一步 。

    将出现“SQL Access Advisor：计划”页面。

  8. 按照“ 指定任务和计划选项 ”中的说明继续执行下一步。

        将出现“搜索和选择：计划”对话框。
