“Oracle OLAP用户指南”的此发行版中的更改

此版本中未对Oracle OLAP DML或Analytic Workspace Manager进行任何更改。有关早期版本的更改，请参阅以下主题。

Oracle Database 12c第2版（12.2）中的更改

自上一版本以来，此版本在Analytic Workspace Manager中不包含任何更改。

Oracle Database 12 c第1版（12.1）中的更改

以下是Oracle Database 12 c第1版（12.1）和该版本中Analytic Workspace Manager的更改。Analytic Workspace Manager是在Oracle数据库中创建，开发和管理维对象的主要工具。

新功能

以下Analytic Workspace Manager功能是此版本中的新功能：

  • 使用服务名称连接到数据库。

    请参阅“ 定义数据库连接 ” 。

  • 将分析工作区连接到Oracle数据库时指定附件模式。

    请参阅“ 显示分析工作区附件模式 ” 。

  • 创建基于OBIEE兼容的SQL函数表达式的计算度量。

    如果启用此功能，则Analytic Workspace Manager会根据需要自动创建其他计算度量，以处理您创建的计算度量。包含计算度量的多维数据集适合导出到Oracle Business Intelligence Enterprise Edition（OBIEE）。您可以使用“导出到OBIEE管理员”插件将多维数据集导出到OBIEE。

    请参见“ 创建多维数据集 ”和“ 创建计算的度量 ” 。

  • 使用度量维度表创建事实视图。

    事实视图将事实表转向，以便度量标识行而不是列。

    请参见“ 创建测量尺寸 ” 。

  • 创建度量维度。

    您现在可以创建度量维度，其度量为维度成员。尺寸现在具有Class类型，可以是None或Measure。

    请参见“ 创建测量尺寸 ” 。

  • 将映射的SQL保存到剪贴板或文件。

    请参见“ 映射多维数据集 ”

  • 将多维数据集映射到数据源时，指定聚合函数SUM ， MAX ， MIN ， AVG和COUNT 。

    请参阅“ 聚合函数 ” 。

  • 在维度层次结构的多个级别上对多维数据集进行分区。

    请参阅“ 对多维数据集进行分区 ” 。

  • 将对象定义保存到表中或从表创建对象。

    以前，您可以将对象定义保存为文件中的XML模板。您现在可以将XML模板保存在数据库表中。您可以使用XML模板重新创建维对象。

    请参见“ 使用对象定义保存和重新创建维对象 ” 。

  • 复制和粘贴对象。

    您现在可以复制分析工作区，多维数据集，维度或度量等对象，并将该对象粘贴到适当的位置。

    请参见“ 复制和粘贴维对象 ” 。

  • 将多维数据集连接到表和视图。

    Oracle Database 12 c引入了CUBE JOIN操作。此操作可提高将多维数据集连接到表或视图的性能。

    请参见“将多维数据集连接到表和视图 ” 。

不受支持的功能

Oracle不再支持以下功能。有关此版本中不支持的功能的完整列表，请参阅“ Oracle数据库升级指南” 。

  • 支持Oracle OLAP的通用仓库元模型（CWM）

其他变化

以下是此版本中的其他更改：

  • 数据安全策略使用Oracle Real Application Security。

    为多维数据集或维度创建数据安全策略时，Oracle OLAP使用Oracle Real Application Security（ORAS）而不是可扩展数据安全性（XDS），它在Oracle Database 11g发行版中使用。将Oracle数据库从11 g版本升级到12 c时 ，您的XDS数据安全策略将自动转换为ORAS。

    注意：

    11g Oracle数据库实例中定义的数据安全角色不会自动转换为ORAS。在将11 g数据库升级到12 c之前 ，必须删除11 g数据库中定义的所有数据安全角色。升级后，您可以使用分析工作区管理12c的再定义数据的安全角色。

    如果将11 g数据库升级到12 c而不删除11 g数据安全角色，则包含数据安全角色的任何数据安全策略在12 c数据库中都无效。

    12 c中数据安全实现的一些差异如下：

    • 只有对象的所有者才能创建数据安全策略或OLAP数据安全角色。在11 g中 ，所有者和对该对象具有写权限的任何人都可以创建数据安全策略或OLAP数据安全角色。

    • 您可以为维或多维数据集禁用数据安全策略，然后再次启用它。

    • 您可以创建多个OLAP数据安全角色;但是，您只能向维或多维数据集添加一个数据安全角色。在11 克 ，您可以添加多个。

    也可以看看：

    • “ 在维度和多维数据集上创建数据安全策略”，以获取有关在Analytic Workspace Manager中创建数据安全策略的信息

    • “ 禁用和启用数据安全 ”

    • 有关在Oracle数据库中配置数据安全性的信息，请参见“Oracle数据库Real Application Security管理员和开发人员指南”

  • 查看有关多维数据集分区的信息。

    多维数据集的“分区”属性的“分区成员分析”子选项卡显示有关多维数据集分区的信息。此信息取代了以前版本的多维数据集分区顾问。

    也可以看看：

    • “ 分析分区成员 ”

  • Oracle企业管理器云控制（云控制）

    在以前版本的Oracle数据库中，您使用Oracle Enterprise Manager数据库控制（数据库控制）从图形用户界面管理数据库性能调整。在此版本中，您可以使用Cloud Control图形用户界面。

    您必须与Oracle数据库分开安装Cloud Control。

    也可以看看：

    • 有关安装Cloud Control的信息，请参阅“Oracle Enterprise Manager云控制基本安装指南”

    • “ 监控分析工作区 ”
