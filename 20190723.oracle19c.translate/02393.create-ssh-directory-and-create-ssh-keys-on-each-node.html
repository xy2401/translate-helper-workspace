在每个节点上创建SSH目录并创建SSH密钥

要配置SSH，必须首先在每个群集节点上创建RSA或DSA密钥。

在每个节点上完成以下步骤：
  1. 以软件所有者身份登录（在此示例中为grid用户）。
  2. 要确保以grid方式登录，并验证用户标识是否与分配给grid用户的预期用户标识匹配，请输入以下命令： 
    $ id $ id grid

    确保您正在使用的Oracle用户组和用户以及用户终端窗口进程具有相同的组和用户ID。

    例如：

    uid = 54322（网格）gid = 54321（oinstall）groups = 54321（oinstall），54322（grid，asmadmin，asmdba）$ id grid uid = 54322（grid）gid = 54321（oinstall）groups = 54321（oinstall），54322 （网格，asmadmin，asmdba）
  3. 如有必要，在grid用户的主目录中创建.ssh目录，并对其设置权限以确保只有oracle用户具有读写权限：
    $ mkdir~ / .ssh $ chmod 700~ / .ssh
    请注意，如果权限未设置为700，则SSH配置将失败。
  4. 输入以下命令：
    $ / usr / bin / ssh-keygen -t dsa

    在提示符下，接受密钥文件的默认位置（按Enter键 ）。

    切勿将私钥分发给未授权执行Oracle软件安装的任何人。

    此命令将DSA公钥写入~/.ssh/id_dsa.pub文件，将私钥写入~/.ssh/id_dsa文件。
  5. 使用DSA密钥在要创建群集成员的每个节点上重复步骤1到4。
