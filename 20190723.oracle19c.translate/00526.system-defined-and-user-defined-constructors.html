8.5.1属性值构造函数

系统定义的构造函数（也称为属性值构造函数）要求您为构造函数传递该类型的每个属性的值。然后，构造函数将新对象实例的属性设置为这些值，如例8-6所示。

调用构造函数之前的关键字NEW是可选的，但建议使用。

8.5.2构造函数和类型演化

属性值构造函数可以省去为类型定义自己的构造函数的麻烦。但是，您必须为类型中声明的每个属性提供值，否则构造函数调用将无法编译。

如果稍后进化类型，此要求可能会产生问题，特别是因为属性值构造函数是隐式的，并且在代码中不可见，这与用户定义的构造函数不同。更改类型的属性时，类型的属性值构造函数也会更改。如果添加属性，则更新的属性值构造函数需要新属性的值;否则，现有代码中的任何属性值构造函数调用都会失败。

8.5.3用户定义构造函数的优点

与属性值构造函数不同，用户定义的构造函数不需要为类型的每个属性显式设置值。

用户定义的构造函数可以包含任意类型的任意数量的参数，并且这些参数不需要直接映射到类型属性。定义构造函数时，可以将属性初始化为任何适当的值。对于您未提供值的任何属性，系统初始化为NULL 。

如果您进化类型 - 例如，通过向该用户定义的构造函数添加属性调用，则不需要更改。当类型发展时，用户定义的构造函数不会自动修改，因此它们的签名保持不变。但是，如果不希望将new属性初始化为NULL ，则可能需要更改构造函数的定义。

8.5.4定义和实现用户定义的构造函数

您可以在类型主体中定义用户定义的构造函数，就像普通方法一样。您使用短语CONSTRUCTOR FUNCTION引入声明和定义，并以RETURN SELF AS RESULT子句结束。

类型的构造函数必须与类型具有相同的名称。例8-7为shape类型定义了两个构造函数。如示例所示，您可以通过定义具有不同签名的多个版本来重载用户定义的构造函数。

示例8-7定义和实现用户定义的构造函数

CREATE TYPE shape AS OBJECT（名称VARCHAR2（30），区域NUMBER， CONSTRUCTOR FUNCTION形状（SELF IN OUT NOCOPY形状，名称VARCHAR2） RETURN SELF AS RESULT ，CONSTRUCTOR FUNCTION shape（SELF IN OUT NOCOPY shape，name VARCHAR2，area NUMBER）RETURN自我结果）不是最终的; / CREATE TYPE BODY形状AS CONSTRUCTOR FUNCTION形状（SELF IN OUT NOCOPY形状，名称VARCHAR2）RETURN SELF AS RESULT是BEGIN SELF.name：= name; SELF.area：= 0; 返回;结束;构造函数形状（ SELF IN OUT NOCOPY形状，名称VARCHAR2，区域NUMBER）RETURN SELF AS RESULT是BEGIN SELF.name：= name; SELF.area：= area;返回;结束;结束; /

用户定义的构造函数具有隐式的第一个参数SELF 。在用户定义的构造函数的声明中指定此参数是可选的。如果确实指定了它，则必须将其模式声明为IN OUT 。

必需子句RETURN SELF AS RESULT确保返回的实例的最具体类型与SELF参数的最特定类型相同。对于构造函数，这是为其定义构造函数的类型。例如，如果对shape构造函数的调用中最具体的SELF参数类型是shape ，则此子句确保shape构造函数返回shape实例（而不是shape子类型的实例）。

调用构造函数时，系统会将SELF参数的属性初始化为NULL 。随后在函数体中初始化的属性名称可以使用SELF进行限定，例如例8-7中的 SELF.name ，以区别于构造函数的参数名称（如果它们相同）。如果参数名称不同，则不需要此限定。

函数体必须包含显式return;如图所示。return关键字后面不能跟return表达式。系统自动返回新构造的SELF实例。

用户定义的构造函数可以用PL / SQL，C或Java实现。

8.5.5重载和隐藏的构造函数

您可以像其他类型方法一样重载用户定义的构造函数。

用户定义的构造函数不是继承的，因此在超类型中定义的用户定义的构造函数不能隐藏在子类型中。但是，如果用户定义的构造函数的签名与属性值构造函数的签名完全匹配，则用户定义的构造函数会隐藏并因此取代其类型的属性值构造函数。

对于要匹配的签名，用户定义的构造函数的参数的名称和类型（在隐式SELF参数之后）必须与该类型的属性的名称和类型相同。用户定义的构造函数的参数模式（在隐式SELF参数之后）必须为IN 。

如果属性值构造函数未被具有相同名称和签名的用户定义构造函数隐藏，则仍可以调用属性值构造函数。

请注意，如果您进化类型 - 例如，通过添加属性 - 类型的属性值构造函数的签名会相应地更改。这可能导致以前隐藏的属性值构造函数再次变得可用。

8.5.6调用用户定义的构造函数

您可以像任何其他函数一样调用用户定义的构造函数，并且可以在任何可以使用普通函数的地方使用它。

SELF参数是隐式传递的，可能不会显式传递。换句话说，不允许使用以下用法：

新构造函数（instance，argument_list）

用户定义的构造函数不能出现在CREATE或ALTER TABLE语句的DEFAULT子句中，但属性值构造函数可以。属性值构造函数的参数不得包含对PL / SQL函数或其他列的引用，包括伪列LEVEL ， PRIOR和ROWNUM ，或者未完全指定的日期常量。检查约束表达式也是如此：在创建或更改表时，属性值构造函数可用作检查约束表达式的一部分，但用户定义的构造函数不能。

即使对于没有参数的构造函数调用，SQL也需要括号。在PL / SQL中，括号在调用零参数构造函数时是可选的。但是，它们确实使构造函数调用更加明显是一个函数调用。以下PL / SQL示例省略了构造函数调用中的括号以创建新形状：

shape s：= new my_schema.shape;

NEW关键字和模式名称是可选的。

8.5.7 SQLJ对象类型的构造函数

SQLJ对象类型是映射到Java类的SQL对象类型。SQLJ对象类型具有属性值构造函数。它还可以具有用户定义的构造函数，这些构造函数映射到引用的Java类中的构造函数。

示例8-9创建SQLJ对象

CREATE TYPE地址AS OBJECT EXTERNAL NAME'university.address'LANGUAGE JAVA使用SQLData（街道VARCHAR2（100）外部名称'街道'，城市VARCHAR2（50）外部名称'city'，状态VARCHAR2（50）EXTERNAL NAME'state'， zipcode NUMBER EXTERNAL NAME'zipcode'， CONSTRUCTOR FUNCTION地址（SELF IN OUT NOCOPY地址，街道VARCHAR2，城市VARCHAR2，州VARCHAR2，邮政编码NUMBER） 返回自认为结果为语言JAVA NAME'university.address（java.lang。String，java.lang。String，java.lang。String，int）返回地址'）; /

SQLJ类型的序列化表示只能有一个用户定义的构造函数。SQLJ类型的对象的内部表示对SQL是不透明的，因此SQLJ类型不可能使用属性值构造函数。
