创建ROLLBACK SEGMENT

注意：

Oracle强烈建议您以自动撤消管理模式运行数据库，而不是使用回滚段。除非必须这样做才能与早期版本的Oracle数据库兼容，否则请勿使用回滚段。有关自动撤消管理的信息，请参阅Oracle数据库管理员指南 。

目的

使用CREATE ROLLBACK SEGMENT语句创建回滚段 ，该回滚段是Oracle数据库用于存储反转或撤消事务所做更改所必需的数据的对象。

本节中的信息假定您的数据库未在自动撤消模式下运行（ UNDO_MANAGEMENT初始化参数设置为MANUAL或根本未设置）。如果数据库以自动撤消模式运行（ UNDO_MANAGEMENT初始化参数设置为AUTO ，这是默认值），则不允许回滚段。但是，抑制了回滚段操作中生成的错误。

此外，如果您的数据库具有本地管理的SYSTEM表空间，则无法在任何字典管理的表空间中创建回滚段。相反，您必须使用自动撤消管理功能或创建本地管理的表空间来保存回滚段。

注意：

表空间可以有多个回滚段。通常，多个回滚段可提高性能。

  • 表空间必须在线才能为其添加回滚段。

  • 创建回滚段时，它最初处于脱机状态。要使其可用于Oracle数据库实例的事务，请使用ALTER ROLLBACK SEGMENT语句将其联机。要在启动数据库时自动使其联机，请将段名称添加到ROLLBACK_SEGMENT初始化参数的值。

要在SYSTEM表空间以外的表空间中使用对象：

  • 如果使用回滚段进行撤消，则至少有一个回滚段（ SYSTEM回滚段除外）必须在线。

  • 如果以自动撤消模式运行数据库，则至少有一个UNDO表空间必须联机。

先决条件

要创建回滚段，您必须具有CREATE ROLLBACK SEGMENT系统特权。

语义

上市

指定PUBLIC以指示回滚段是公共的，并且可供任何实例使用。如果省略此子句，则回滚段是私有的，仅对在其初始化参数ROLLBACK_SEGMENTS命名的实例可用。

ROLLBACK_SEGMENT

指定要创建的回滚段的名称。该名称必须满足“ 数据库对象命名规则 ”中列出的要求。

TABLESPACE

使用TABLESPACE子句标识创建回滚段的表空间。如果省略此子句，则数据库将在SYSTEM表空间中创建回滚段。

注意：

Oracle数据库必须经常访问回滚段。因此，Oracle强烈建议您不要通过省略此子句显式或隐式地在SYSTEM表空间中创建回滚段。此外，为了避免包含回滚段的表空间的高争用，它不应包含其他对象（如表和索引），并且它应该需要最小的扩展区分配和释放。

要实现这些目标，请在本地管理的表空间中创建回滚段，并禁用自动分配 - 使用带有UNIFORM设置的EXTENT MANAGEMENT LOCAL子句创建的表空间。不支持AUTOALLOCATE设置。

storage_clause

storage_clause允许您指定回滚段的存储特征。

  • storage_clause的OPTIMAL参数特别重要，因为它仅适用于回滚段。

  • 您不能使用CREATE ROLLBACK SEGMENT指定storage_clause的PCTINCREASE参数。

例子

创建回滚段：示例

以下语句在适当配置的表空间中创建具有缺省存储值的回滚段：

CREATE TABLESPACE rbs_ts DATAFILE'rbs01.dbf'尺寸10M扩展管理局部均匀尺寸100K; / *如果您的数据库处于自动撤消模式，则此示例和下一个示例将失败。* / CREATE ROLLBACK SEGMENT rbs_one TABLESPACE rbs_ts;

上述声明等同于以下内容：

创建ROLLBACK SEGMENT rbs_one TABLESPACE rbs_ts存储（初始10K）;
