将Oracle Grid Infrastructure修补为软件补丁级别

从Oracle Grid Infrastructure 12c第1版（12.1）开始，可以使用名为Rolling Patch的新集群状态。就这种静默状态允许的Oracle ASM操作而言，此模式类似于现有的“滚动升级”模式。

  1. 从My Oracle Support下载要应用的修补程序：

    https://support.oracle.com

    选择“修补程序和更新”选项卡以找到修补程序。

    Oracle建议您选择Recommended Patch Advisor ，然后输入软件的产品组，发行版和平台。My Oracle Support为您提供最新的修补程序集更新（PSU）和关键修补程序更新（CPU）的列表。

    将修补程序放在可访问的目录中，例如/tmp 。

  2. 将目录更改为Grid home中的/OPatch目录。例如：

    $ cd /u01/app/19.0.0/grid/OPatch

  3. 查看要应用的修补程序的修补程序文档，并在开始修补程序升级之前完成所有必需的步骤。

  4. 按照修补程序文档中的说明应用修补程序。例如：

    ＃。/opatch应用patch directory_location / patch_ID
