说明dbfsg_vm_005.png的说明

图像fmw_ras_session_service.png类似于图像fmw_app_stack.png并显示Real Application Security，因为它在FMW堆栈中实现为Real Application Security servlet过滤器。RAS筛选器透明地设置会话，使会话与OPSS主题同步，并包含一组在会话中作为会话API运行的Real Application Security API，权限提升API，命名空间API和检查权限API。在图示之后的文本中进一步描述了该实现。

顶部有一排三个标有“Client”的框，“Authentication Server”包含一个标有“Single Sign-On”的子框和“Identity Store”。下面是一个标有“Web Logic Server”的大框。其中左侧是三个垂直排列的子框。顶部子框标记为“Authenticator”，中间子框标记为“OPSS Filter”，底部子框标记为“Real Application Security Filter”。右边是两个垂直排列的子框。顶部子框标记为“应用程序会话”，其中包含名为“人力资源”的应用程序会话。该子框包含三个垂直排列的较小子框。顶部子子框标记为“主题”。中间的一个标记为“代码”，并指向WebLogic Server框外部的一段代码。底部标有'Real Application Security API'。底部子框标记为“连接池”，包含三个标记为“Conn1，Conn2和Conn3”的连接。在大框的右侧是一个标有“数据库”的圆柱体，在两个方向上都有一个标有“JDBC”的连接箭头，指向标有“WebLogic Server”的大框。在数据库柱面内是标记为“应用程序会话”的结构。大盒子下方和左侧是一个标有“安全商店”的盒子。从许多对象连接箭头显示信息流。例如，箭头显示从客户端通过身份验证器并在身份验证服务器中使用单点登录的流程，其中在Identy Store中检查凭据。经过身份验证后，流程将通过实际应用程序安全筛选器继续通过OPSS筛选器，其中每个（OPSS筛选器和实际应用程序安全筛选器）都在安全性存储中进行检查，然后信息通过主题，代码和Real Application流入应用程序会话安全API，用于连接池中的Conn2连接，以及使用JDBC的数据库中的应用程序会话。另一个箭头显示了从Real Application Security Filter到连接池中Conn1连接的流程。
