在远程节点上继续不完整安装

查看此信息以继续在远程节点上进行不完整安装。

  1. 如果根脚本失败表明需要重新启动，请通过消息CLSRSC-400重新启动受影响的节点。否则，请按照错误输出中的报告手动修复或清除错误情况。

  2. 如果正在使用root自动化，请在第一个节点上的OUI实例上单击“ 重试 ”。

  3. 如果未使用root自动化，请按照下列步骤操作：

    1. 以root身份登录受影响的节点，然后在该节点上运行orainstRoot.sh脚本。例如：

      $ sudo -s [root @ node6] #cd / u01 / app / oraInventory [root @ node6]＃。/ orainstRoot.sh

    2. 将目录更改为Grid主目录，然后在受影响的节点上运行root.sh脚本。例如：

      [root @ node6] #cd /u01/app/19.0.0/grid [root @ node6]＃。/ root.sh

  4. 从第一个节点上的OUI实例继续安装。
