9.9物化视图对对象的支持

物化视图支持可用于包含对象，集合或REF类型列的关系表。这种物化视图称为对象关系物化视图。

对象关系实体化视图所需的所有用户定义类型必须存在于物化视图站点以及主站点。它们必须在两个站点上具有相同的对象类型ID和版本。

话题：

  • 对象，集合或REF类型列

  • 对象表

9.9.1对象，集合或REF类型列

要进行更新，基于包含对象列的表的实例化视图必须选择列作为定义视图的查询中的对象：如果查询仅选择列对象类型的某些属性，则读取实例化视图-只要。

视图定义查询还可以选择集合或REF类型的列。REF可以是基于主键的，也可以是系统生成的键，它们可以是作用域的，也可以是未作用域的。可以将实体化REF列重新调整到物化视图的站点上的不同表 - 例如，到主表的本地物化视图而不是原始远程表。

9.9.2对象表

基于对象表的物化视图称为对象物化观点。这种物化视图本身就是一个对象表。通过将OF type关键字添加到CREATE MATERIALIZED VIEW语句来创建对象实例化视图。例如：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT * FROM HR.Customer_objtab@dbs1;

与普通对象表一样，对象物化视图的每一行都是一个对象实例，因此创建物化视图的视图定义查询必须从主表中选择整个对象：查询不能仅选择对象类型的子集属性。例如，不允许使用以下物化视图：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT CustNo FROM HR.Customer_objtab@dbs1;

您可以通过省略OF type关键字从对象表创建对象关系的物化视图，但这样的视图是只读的：您无法从对象表创建可更新的对象关系物化视图。

例如，以下CREATE MATERIALIZED VIEW语句创建对象表的只读对象 - 关系实例化视图。即使视图定义查询选择了对象类型的所有列和属性，它也不会将它们选为对象的属性，因此创建的视图是对象关系和只读的：

CREATE MATERIALIZED VIEW客户AS SELECT * FROM HR.Customer_objtab@dbs1;

对于基于对象表的对象关系视图和对象实例化视图，如果主对象表的类型不是FINAL ，则实例化视图定义查询中的FROM子句必须包含ONLY关键字。例如：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT CustNo FROM ONLY HR.Customer_objtab@dbs1;

否则， FROM子句必须省略ONLY关键字。
