模型表达式

模型表达式仅用于SELECT语句的model_clause ，然后仅用于模型规则的右侧。它为先前在model_clause定义的度量列中的单元格生成一个值。有关其他信息，请参阅model_clause 。

model_expression :: =


说明model_expression.eps的描述

在模型表达式中指定度量列时，指定的任何条件和表达式都必须解析为单个值。

在模型表达式中指定聚合函数时，函数的参数是先前在model_clause定义的度量列。聚合函数只能在模型规则的右侧使用。

在模型规则的右侧指定分析函数可以直接在model_clause表达复杂的计算。在模型表达式中使用分析函数时，以下限制适用：

  • 分析函数只能在UPDATE规则中使用。

  • 如果规则的左侧包含FOR循环或ORDER BY子句，则无法在模型规则的右侧指定分析函数。

  • 分析函数的OVER子句中的参数不能包含聚合。

  • 分析函数的OVER子句之前的参数不能包含单元格引用。

当expr本身是一个模型表达式时，它被称为嵌套 单元格 引用 。以下限制适用于嵌套单元格引用：

  • 只允许一级嵌套。

  • 嵌套单元格引用必须是单单元格引用。

  • 当在model_rules_clause指定AUTOMATIC ORDER只有在嵌套单元格引用中使用的度量保持静态时，才能在模型规则的左侧使用嵌套单元格引用。

下面显示的模型表达式基于以下SELECT语句的model_clause ：

SELECT country，prod，year，s FROM sales_view_ref MODEL PARTITION BY（country）DIMENSION BY（prod，year）MEASURES（sale s）IGNORE NAV UNIQUE DIMULION RULES UPSERT序列订单（s [prod ='Mouse Pad'，year = 2000] = s ['Mouse Pad'，1998] + s ['Mouse Pad'，1999]，s ['Standard Mouse'，2001] = s ['Standard Mouse'，2000]）ORDER BY country，prod，year;

以下模型表达式表示使用符号表示法的单个单元格引用。它代表了2000年鼠标垫的销售情况。

s [prod ='鼠标垫'，年= 2000]

以下模型表达式使用CV函数表示使用位置表示法的多单元格引用。它表示2001年维度列prod的当前值的销售额。 

s [CV（prod），2001]

以下模型表达式表示聚合函数。它代表销售鼠标垫的尺寸标注列的现值之间的年的总和year少两维列的当前值year减一。 

SUM（s）['Mouse Pad'，年BETWEEN CV（） -  2和CV（） -  1]
