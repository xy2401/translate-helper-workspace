使用OLSNODES

本节包含与使用OLSNODES命令相关的主题。

  • 概观

  • 操作说明

概观

olsnodes命令提供参与集群的所有节点的节点列表和其他信息。

您可以使用此命令快速检查群集是否正常运行，并且所有节点都已注册为群集成员。此命令还提供了一种获取节点编号的简便方法。

操作说明

使用信息

当使用-n all选项时，Cluster Verification Utility（CLUVFY）使用此命令获取节点名称列表。

此命令实用程序位于$ORA_CRS_HOME/bin目录中。如果启动了CRS守护程序，则只能使用此命令。

特权和安全

您可以以root用户，安装Oracle Clusterware的用户或安装Oracle数据库的用户身份运行此命令。

OLSNODES命令摘要

olsnodes命令不使用关键字，但接受一个或多个选项。表C-1中描述了可用选项。

句法

olsnodes [[-n] [-i] [-s] [-t] [node_name | -l [-p]] | [-c]] [-a] [-g] [-v]

如果发出不带任何命令参数的olsnodes命令，该命令将返回集群中节点的列表：

[root @ node1] #olsnodes node1 node2 node3 node4

表C-1 OLSNODES命令选项

选项 描述
-n

列出参与群集的所有节点，并包括分配的节点号。
-一世

列出参与群集的所有节点，并包括分配给每个节点的虚拟Internet协议（VIP）地址（或具有节点名称的VIP地址）。
-s

显示节点的状态： active或非inactive 。

-t

显示节点类型：固定或取消固定。
NODE_NAME

显示特定节点的信息。
-l [-p]

列出本地节点并包括本地节点的专用互连。-p选项仅在与-l选项一起指定时有效。

-C

显示群集的名称。
-一个

仅显示群集中没有重复项的活动节点。
-G

记录集群验证信息以及更多详细信息。
-v

以详细模式记录集群验证信息。在调试模式下使用，仅在My Oracle Support的指导下使用。

例子

示例1：列出群集中当前所有节点的VIP地址

要列出当前为群集成员的每个节点的VIP地址，请使用以下命令：

[root @ node1] #olsnodes -i node1 168.92.1.1 node2 168.192.2.1 node3 168.192.3.1 node4 168.192.4.1

示例2：列出集群成员的节点名称和节点号

要列出群集中每个节点的节点名称和节点号，请使用以下命令：

[root @ node1] #olsnodes -n node1 1 node2 2 node3 3 node4 4

示例3：显示集群成员的节点角色

要列出群集中每个节点的节点角色，请使用以下命令：

[root @ node1] #olsnodes -a node1 Hub node2 Hub node3 node4
