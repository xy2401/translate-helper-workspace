3用于DRDA的Oracle数据库提供程序的安装和配置

考虑为DRDA安装，配置和管理Oracle数据库提供程序。

关于为DRDA安装Oracle数据库提供程序

安装包括启动Oracle Universal Installer，进入Oracle主路径，并选择要安装的“Oracle Database Provider for DRDA”产品。安装将在产品的“面试”面板中询问几个初始配置项。

请注意，以下过程在$ ORACLE_HOME /drdaas/admin/目录中创建drdaas.ora配置文件。

为DRDA安装Oracle数据库提供程序

  1. 启动Oracle Universal Installer。
  2. 输入现有ORACLE_HOME的路径，或单独安装的新路径。
  3. [可选]输入ORACLE_HOME名称。
  4. 选择用于DRDA的Oracle数据库提供程序进行安装。
  5. 在“ 面试”面板中，输入以下信息：

    • 用于DRDA侦听器的Oracle数据库提供程序主机名和/或IP地址

      指定Oracle Database Provider for DRDA作为侦听器的网络接口的主机名或IP地址。

      默认值为空字符串。

    • 用于DRDA侦听器端口号的Oracle数据库提供程序

      指定DRDA侦听器的Oracle数据库提供程序的端口号。

      默认值是1446 。

    • 用于DRDA RDB映射名的Oracle数据库提供程序

      指定客户端应用程序用作位置限定符的外部关系数据库名称。

      默认值为DRDAAS 。

    • Oracle数据库连接描述符

      指定用于连接Oracle数据库的连接描述符。有效值为：

      • TNS( tns_name_sentry ) ，如TNS(orcl)

      • oracle_sid或ORACLE_SID

        这将在Oracle Database Provider for DRDA启动之前访问本地Oracle数据库，具体取决于环境变量$ORACLE_SID的设置。

      • TNS-描述

        例如：

        DESCRIPTION =（ADDRESS =（PROTOCOL = ipc）（KEY = drdaas））（CONNECT_DATA =（SERVICE_NAME = drdaas.us.oracle.com）））;

        默认为oracle_sid 。

  6. 选择完成 。
  7. 在命令提示符处，创建诊断目录：
    ％mkdir -p $ ORACLE_HOME / log / diag / dps

为DRDA配置Oracle数据库提供程序

要为DRDA配置Oracle数据库提供程序，必须使用任何必要信息更新drdaas.ora配置文件。

此外，必须在使用Oracle Database Provider for DRDA的所有Oracle数据库实例中安装依赖于Oracle Database Provider for DRDA产品的对象。

更新drdaas.ora配置文件

通常，除安装期间指定的参数外，不需要任何其他参数。

在更高级的方案中，可能需要指定多个DATA_PORT参数或配置多个RDB_MAP条目。更复杂的安装可能需要多个Oracle数据库提供程序用于DRDA实例。

有关参数及其选项的列表，请参阅“配置参数”一节。

DATA_PORT注意事项

可以使用不同的主机名或IP地址以及未分配的网络端口号指定其他DATA_PORT条目。这称为多路复用实例配置 。

RDB_MAP注意事项

可以指定其他RDB_MAP条目以添加其他映射条目，以便在第三方关系数据库和Oracle之间进行转换。

注意：

对于某些DRDA客户端，例如IBM DB2 Database for z / OS，用户必须为每个Oracle数据库提供程序仅为DRDA实例配置一个RDB_MAP条目。这称为专用实例配置 。
用于DRDA实例注意事项的Oracle数据库提供程序

可能需要为DRDA实例定义多个Oracle数据库提供程序，以适应包含两种类型DRDA客户端的环境。因此，Oracle Database Provider for DRDA产品支持在同一drdaas.ora配置文件中定义多个实例配置。

SQL翻译配置文件

为了便于从其本机DB2方言到Oracle正确解释基于DRDA的应用程序SQL，用户必须创建SQL转换配置文件。

SQL翻译配置文件基于每个用户进行管理。相比之下，DRDA软件包是在应用程序的基础上管理的。因此，只有一个SQL转换配置文件名称可能与特定DRDA包相关联。相同的SQL翻译配置文件也可能与许多包相关联;为保持一致性，应将相同的SQL Translation Profile与每个已定义的包关联。

要创建其他翻译配置文件，DRDA用户必须从DRDA管理员请求配置文件名称，然后在$ ORACLE_HOME /drdaas/admin/目录中调用drdasqtt_translator_setup.sql脚本。“创建SQL转换接口包和转换配置文件”部分演示了如何为配置文件名称DB2ZOS创建SQL转换配置文件;此代码创建翻译配置文件的模板。

注意：

SQL Translation Profile是一项可选功能，只有在配置SQL Translation Feature时才能使用。

管理员角色

必须执行管理功能的用户必须具有DRDAAS_ADMIN_ROLE角色。这将在特定DRDA包上启用权限授予，并分配包属性（SQL转换概要文件名称）。

对于没有默认角色ALL安装，为用户提供多个默认角色，例如CONNECT或RESOURCE ，并将添加DRDAAS_ADMIN_ROLE角色添加到默认列表。

DRDAAS_ADMIN_ROLE角色不适用于必须使用DRDA软件包的用户。DRDAAS_USER_ROLE ，它们应该具有DRDAAS_USER_ROLE赋值。

管理主要涉及向用户授予和撤销权限，在包上设置属性以及删除包。

创建包或被指定为包的所有者的用户具有对该包的隐式权限，并且可以授予对其他包的访问权。例如，包所有者可以向任意数量的用户授予RUN权限。所有者还可以设置包属性并删除包。

但是，为了最初绑定包，用户必须具有BIND权限，可以是集合中的任何包，也可以是该包的专有权。只有具有DRDAAS_ADMIN_ROLE角色的用户DRDAAS_ADMIN_ROLE可以向用户授予尚未绑定或未由该用户创建或拥有的包的授权。有关如何授予对包的访问权以及如何设置包属性的信息，请参阅DRDA包授权。

用户角色

通过DRDA访问数据库的每个用户都必须具有DRDAAS_USER_ROLE用户角色作为默认值。

用户通常具有默认角色ALL ，可立即启用所有授予的角色。如果用户具有显式默认角色，则它们还必须具有DRDAAS_USER_ROLE角色，如将DRDAAS_USER_ROLE添加到默认值一节中所述。

请注意，如果未指定默认角色的完整列表，则可能会阻止用户连接到数据库，或者无法隐式地处理某些资源。

请参阅Oracle®数据库SQL语言参考和Oracle数据库安全指南 。

卸载适用于DRDA的Oracle数据库提供程序

完全卸载用于DRDA的Oracle数据库提供程序涉及删除数据库对象以及卸载用于DRDA软件的Oracle数据库提供程序。取消了与产品相关的对象，同时客户数据保持不变。

删除数据库对象

以下步骤从数据库中删除所有用于DRDA对象的Oracle数据库提供程序，并删除用户创建的表空间。

要删除数据库对象：

  1. 将目录更改为$ ORACLE_HOME /rdbms/admin 。
    > cd $ ORACLE_HOME / rdbms / admin
  2. 使用SYSDBA选项连接到数据库。
    connect / as sysdba
  3. 运行删除脚本。
    catnodrdaas.sql
  4. 删除用户创建的表空间sysibm及其内容。
    > drop tablespace sysibm;

有关DROP TABLESPACE选项，请参见Oracle®数据库SQL语言参考 。

配置参数

适用于DRDA的Oracle数据库提供程序使用多个参数来配置其环境;它们在配置文件中指定。

DATA_PORT

它指定此实例使用的DRDA数据端口，并由Internet地址和端口号表示。

默认值

没有默认端口号;必须指定显式端口号。Oracle建议使用1446 。

允许值

有效的未分配TCP / IP网络端口号，可选择以与本地计算机上已定义网络接口关联的特定主机名或IP地址为前缀。

句法

DATA_PORT = { host_name | ip_address :} number

用法示例

DATA_PORT = 10.0.0.1:1446

RDB_MAP

此string参数将关系数据库名称映射到ACCRDB命令对象，Oracle TNS条目或本地可寻址的Oracle实例。此参数可能包含多次出现的映射条目。

这没有默认值。

可选值列表包括以下内容：

  • tns_name_entry对应于本地tnsnames.ora配置文件中的TNS条目。

  • tns_entry是一个完整形成的TNS描述符字符串。它可以用来代替TNS名称条目。

  • oracle_sid使用在为DRDA实例启动Oracle数据库提供程序之前设置的$ORACLE_SID环境变量值。

请注意，使用一次或多次RDB_MAP确定了与旧应用程序请求者的兼容模式。默认值是单个专用定义，它基于ORACLE_SID环境变量连接到单个Oracle数据库实例。

允许值

有效的未分配TCP / IP网络端口号，可选择以与本地计算机上已定义网络接口关联的特定主机名或IP地址为前缀。

句法

RDB_MAP = RDB （ rdb_name ） ->TNS( tns_name_entry ) RDB_MAP = RDB （ rdb_name ） ->ORACLE_SID RDB_MAP = RDB （ rdb_name ） -> " tns_entry "

用法示例

RDB_MAP = RDB(DB2DSN1)->ORACLE_SID RDB_MAP = RDB(DB2DSN2)->TNS(ora101) RDB_MAP = RDB(DB2DSN3)->"(ADDRESS=(PROTOCOL=TCP)(HOST=10.0.0.1)(PORT=1446))"

PROTOPROC_TRACE

此参数指定用于跟踪DRDA协议处理器的跟踪工具和详细级别。所有用于DRDA会话的初始Oracle数据库提供程序都使用此设置运行。该参数由表示设施和级别值的值对组成。如果值元组用逗号分隔，则可以同时指定多个值。

它还指定所有AS会话线程执行的初始跟踪级别。PROTOPROC_TRACE是十进制数字或文本指定的等效数字。

与PROTOPROC_TRACE参数一起使用的设施名称如下：

  • TASK - 特定于任务的操作

  • NET - 特定于网络的操作

  • SQL - 特定于SQL的操作

  • OCI - OCI资源运营

  • MEM - 内存资源操作

  • ALL - 已提到的所有设施

PROTOPROC_TRACE级别的值是累加的。例如，设置ERROR(4)包括WARN(2)和INFO(1)消息。预期以下值：

  • 0或NONE - 不生成跟踪;这是默认值。

  • 1或INFO - 生成最小跟踪。

  • 2或WARN - 生成警告信息。

  • 4或ERROR - 生成错误信息。

  • 8或ADMIN - 生成管理信息。

  • 255或ALL - 生成所有详细信息。

默认值

0或没有

允许值

设施名称后跟级别。

用法示例

PROTOPROC_TRACE =“ALL ADMIN”PROTOPROC_TRACE =“TASK WARN，NET ADMIN，MEM INFO”

PROTOPROC_OPTIONS

PROTOPROC_OPTIONS参数指定原型处理选项，影响DRDA协议操作。

默认值

此参数没有默认值。

允许值

可以指定以下值：

  • QRYDTA / ELMODE

    为所有返回的Query Data对象启用扩展长度模式。通常，这用于不涉及LOB的查询。

  • EXTDTA / ELMODE

    为所有返回的扩展数据对象启用扩展长度模式。通常用于涉及LOB的查询。

  • ALLDSS / ELMODE

    为所有协议对象启用扩展长度模式。这是上述值的替代值，除非其他值没有正面影响，否则不应指定。

  • LOGNAME / <NAME>

    指定静态DRDA日志名称。日志名称应为字母数字，由18个字符组成。将DB2 for z / OS用作客户端时可能需要此选项。

  • LOGTSTMP / <YYYYMMDDHHMMSSTTTT>

    指定静态DRDA日志时间戳。时间戳必须具有以下格式：

    YYYYMMDDHHMMSSTTTT

    将DB2 for z / OS用作客户端时可能需要此值。

句法

PROTOPROC_OPTIONS = value {，value ...}

用法示例

PROTOPROC_OPTIONS = QRYDTA / ELMODE，EXTDTA / ELMODE，LOGNAME / ORACLEDB，LOGTSTMP / 201609191201020001
