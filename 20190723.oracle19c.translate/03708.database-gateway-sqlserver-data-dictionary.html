C数据字典

Oracle Database Gateway for SQL Server将引用Oracle数据库数据字典表的查询转换为从SQL Server系统表中检索数据的查询。您可以通过数据库链接对数据字典表执行查询，方法与查询Oracle数据库中的数据字典表的方式相同。网关数据字典在外观和使用上类似于Oracle数据库数据字典。

话题：

  • 数据字典支持

  • 数据字典映射

  • 网关数据字典描述

数据字典支持

以下段落描述了Oracle Database Gateway for SQL Server数据字典支持。

SQL Server系统表

SQL Server数据字典信息作为SQL Server系统表存储在SQL Server数据库中。所有SQL Server系统表的名称都以“ sys ”为前缀。SQL Server系统表定义数据库的结构。更改数据定义时，SQL Server将读取并修改SQL Server系统表以添加有关用户表的信息。

访问网关数据字典

访问网关数据字典表或视图与访问Oracle数据库中的数据字典完全相同。您发出SQL 指定数据库链接的SELECT语句。Oracle数据库数据字典视图和列名称用于访问Oracle数据库中的网关数据字典。支持的视图的同义词也是可以接受的。例如，以下语句查询数据字典表ALL_CATALOG以检索SQL Server数据库中的所有表名：

SQL> SELECT * FROM“ALL_CATALOG”@MSQL;

发出数据字典访问查询时，网关：

  1. 将请求的表，视图或同义词映射到一个或多个SQL Server系统表名称。网关将所有数据字典列名称转换为查询中对应的SQL Server列名称。如果映射涉及一个SQL Server系统表，则网关将请求的表名转换为查询中对应的SQL Server系统表名。如果映射涉及多个SQL Server系统表，则网关使用已转换的SQL Server系统表名在查询中构造连接。
  2. 将已翻译的查询发送到SQL Server。
  3. 可能会转换检索到的SQL Server数据，使其具有Oracle数据库数据字典表的外观。
  4. 将数据字典信息从已翻译的SQL Server系统表传递到Oracle数据库。

    注意：

    查询网关数据字典时返回的值可能与Oracle SQL * Plus DESCRIBE命令返回的值不同。

直接查询到SQL Server表

允许直接发布到各个SQL Server系统表的查询，但它们返回不同的结果，因为SQL Server系统表列名称与数据字典视图的名称不同。此外，SQL Server系统表中的某些列不能用于数据字典处理。

默认列值

网关数据字典与典型的Oracle数据库数据字典之间存在细微差别。SQL Server系统表中缺少的Oracle数据库列使用零，空格，空值，不适用的值（NA）或默认值填充，具体取决于列类型。

网关数据字典描述

网关数据字典表和视图提供以下信息：

  • 每列的名称，数据类型和宽度

  • 具有固定值的列的内容

注意：

当网关连接到SQL Server V7.0数据库时，已翻译的数据字典表中某些列的列宽将不同。
