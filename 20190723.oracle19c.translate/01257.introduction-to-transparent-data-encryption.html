父主题： 使用透明数据加密

什么是透明数据加密？

透明数据加密（TDE）使您可以加密存储在表和表空间中的敏感数据。

数据加密后，这些数据在授权用户或应用程序访问此数据时会被透明地解密。在存储介质或数据文件被盗的情况下，TDE有助于保护存储在介质上的数据（也称为静态数据）。

Oracle数据库使用身份验证，授权和审核机制来保护数据库中的数据，但不保存存储数据的操作系统数据文件。为了保护这些数据文件，Oracle数据库提供了透明数据加密（TDE）。TDE加密存储在数据文件中的敏感数据。为了防止未经授权的解密，TDE将加密密钥存储在数据库外部的安全模块中，称为密钥库。

您可以将Oracle Key Vault配置为TDE实施的一部分。这使您可以集中管理企业中的TDE密钥库（在Oracle Key Vault中称为TDE钱包）。例如，您可以将软件密钥库上载到Oracle Key Vault，然后将此密钥库的内容提供给其他启用TDE的数据库。

使用透明数据加密的好处

透明数据加密（TDE）可确保敏感数据经过加密，符合合规性，并提供简化加密操作的功能。

好处如下：

  • 作为安全管理员，您可以确保敏感数据已加密，因此在存储介质或数据文件被盗的情况下是安全的。

  • 使用TDE可帮助您解决与安全相关的法规遵从性问题。

  • 您无需为授权用户或应用程序创建辅助表，触发器或视图来解密数据。来自表的数据被透明地解密为数据库用户和应用程序。处理敏感数据的应用程序可以使用TDE提供强大的数据加密，只需对应用程序进行很少或不进行任何更改。

  • 对于访问此数据的数据库用户和应用程序，数据会被透明地解密。数据库用户和应用程序不需要知道他们访问的数据是以加密形式存储的。

  • 您可以使用联机表重新定义在生产系统上以零停机时间加密数据，也可以在维护期间离线加密数据。（有关联机表重新定义的详细信息，请参阅“ Oracle数据库管理员指南” 。）

  • 您无需修改应用程序即可处理加密数据。数据库管理数据加密和解密。

  • Oracle数据库自动执行TDE主加密密钥和密钥库管理操作。用户或应用程序不需要管理TDE主加密密钥。

谁可以配置透明数据加密？

您必须被授予ADMINISTER KEY MANAGEMENT系统特权才能配置透明数据加密（TDE）。

如果必须在装载阶段打开密钥库，则必须授予SYSKM管理权限，该权限包括ADMINISTER KEY MANAGEMENT系统特权和其他必要特权。

将SYSKM管理权限授予用户时，请确保为其创建密码文件，以便用户可以使用密码以SYSKM连接到数据库。这使用户能够执行查询V$DATABASE视图等操作。

要配置TDE列或表空间加密，您不需要SYSKM或ADMINISTER KEY MANAGEMENT权限。您必须具有以下附加权限才能在表和表空间上创建TDE策略：

  • 创建表

  • 更改表

  • 创建TABLESPACE

透明数据加密列加密的工作原理

透明数据加密（TDE）列加密可保护存储在表列中的机密数据，如信用卡和社会保险号。

TDE列加密使用双层基于密钥的体系结构来透明地加密和解密敏感表列。TDE主加密密钥存储在外部安全模块中，该模块可以是Oracle软件密钥库或硬件密钥库。该TDE主加密密钥对TDE表密钥进行加密和解密，TDE表密钥又加密和解密表列中的数据。

透明数据加密表空间加密的工作原理

透明数据加密（TDE）表空间加密使您可以加密整个表空间。

在加密表空间中创建的所有对象都将自动加密。如果表包含多列中的敏感数据，或者如果要保护整个表而不仅仅是单个列，则TDE表空间加密非常有用。您无需对每个表列执行粒度分析以确定需要加密的列。

此外，TDE表空间加密利用批量加密和缓存来提供增强的性能。对应用程序的实际性能影响可能有所不同

TDE表空间加密对加密表空间中存储的所有数据（包括其重做数据）进行加密。TDE表空间加密不会加密存储在表空间外部的数据。例如， BFILE数据未加密，因为它存储在数据库之外。如果在加密表空间中创建具有BFILE列的表，则不会加密此特定列。

加密表空间中的所有数据都以加密格式存储在磁盘上。对于具有查看或修改数据所需特权的授权用户，数据被透明地解密。数据库用户或应用程序不需要知道特定表中的数据是否在磁盘上加密。如果磁盘或备份介质上的数据文件被盗，则数据不会受到损害。

TDE表空间加密使用双层，基于密钥的体系结构来透明地加密（和解密）表空间。TDE主加密密钥存储在外部安全模块（软件或硬件密钥库）中。此TDE主加密密钥用于加密TDE 表空间加密密钥 ，后者又用于加密和解密表空间中的数据。

关于TDE主加密密钥的密钥库存储

Oracle数据库为透明数据加密提供了一个密钥管理框架，用于存储和管理密钥和凭据。

密钥管理框架包括用于安全存储TDE主加密密钥的密钥库和管理框架，以安全有效地管理各种数据库组件的密钥库和密钥操作。

Oracle密钥库存储已停用的TDE主加密密钥的历史记录，使您可以更改它们，并且仍然能够解密在早期TDE主加密密钥下加密的数据。

密钥库存储框架的好处

密钥管理框架为透明数据加密提供了多种好处。

  • 允许数据库管理员和管理密钥的安全管理员之间的职责分离。您可以将ADMINISTER KEY MANAGEMENT或SYSKM权限授予负责管理密钥库和密钥操作的用户。

  • 促进合规性，因为它可以帮助您跟踪加密密钥并实现密钥库密码轮换和TDE主加密密钥重置或重定密钥操作等要求。

  • 促进并帮助实施密钥库备份要求。备份是为所有关键密钥库操作创建的受密码保护的软件密钥库的副本。

    您必须为所有关键密钥库操作备份密钥库。在重置或重新生成此TDE主加密密钥之前，还必须备份TDE主加密密钥。

  • 使密钥库能够存储在ASM文件系统中。这对于Oracle Real Application Clusters（Oracle RAC）环境特别有用，在这种环境中，数据库实例共享统一的文件系统视图。

  • 启用从硬件密钥库到基于文件系统的软件密钥库的反向迁移。如果必须迁移回软件密钥库，则此选项很有用。

这些密钥库如下：

  • 自动登录软件密钥库：自动登录软件密钥库受系统生成的密码保护，无需安全管理员明确打开。访问时自动打开自动登录软件密钥库。自动登录软件密钥库可以在不同系统中使用。如果您的环境不需要必须明确打开以供使用的密钥库提供的额外安全性，那么您可以使用自动登录软件密钥库。自动登录软件密钥库是无人值守方案的理想选择。

  • 本地自动登录软件密钥库：本地自动登录软件密钥库是自动登录软件密钥库，它们是创建它们的计算机的本地密钥库。无法在创建它们的计算机之外的任何计算机上打开本地自动登录密钥库。此类型的密钥库通常用于需要额外安全性的情况（即，限制对该计算机使用自动登录），同时支持无人参与操作。

  • 受密码保护的软件密钥库：使用您创建的密码保护受密码保护的软件密钥库。您必须先打开此类型的密钥库，然后才能检索或使用密钥。

软件密钥库可以存储在ASM磁盘组或常规文件系统中。

硬件密钥库包含以下类别：

  • 硬件安全模块（HSM）：硬件安全模块是第三方物理设备，为硬件密钥库中的加密密钥提供安全存储。HSM还提供安全的计算空间（存储器）以执行加密和解密操作。使用HSM时，使用TDE主加密密钥的所有加密和解密操作都在HSM内部执行。这意味着TDE主加密密钥永远不会暴露在不安全的内存中。

  • Oracle Key Vault（OKV）密钥库： Oracle Key Vault密钥库（钱包）是Oracle提供的硬件密钥库。Oracle Key Vault使您可以集中管理加密密钥，Oracle密钥库，Java kesytores和凭证文件。

支持的加密和完整性算法

默认情况下，透明数据加密（TDE）列加密使用高级加密标准（AES）。

支持的高级加密标准密码密钥是192位长度的密码密钥（ AES192 ）。表空间和数据库加密使用128位长度密码密钥（ AES128 ）

此外，除非另有说明，否则在加密前默认将salt添加到纯文本中。您不能将salt添加到要加密的索引列。对于索引列，请为SQL ENCRYPT子句选择NO SALT参数。

对于TDE表空间加密和数据库加密，默认情况下使用高级加密标准和128位长度密码密钥（AES128）。此外，在加密之前，总是将盐添加到明文中。

您可以通过使用SQL ENCRYPT子句设置不同的算法来更改现有加密列上的加密算法和加密密钥。

多租户选项如何影响透明数据加密

在多租户环境中，您可以为整个容器数据库（CDB）或单个可插拔数据库（PDB）配置密钥库。

Oracle数据库支持以下多租户模式来管理密钥库：

  • 通过联合模式，您可以为CDB根和任何关联的联合模式PDB配置一个密钥库和TDE主加密密钥。联合模式与先前版本中多租户环境中的TDE管理方式大致相同。

  • 通过隔离模式，您可以在单个PDB中创建和管理密钥库和TDE主加密密钥。不同的隔离模式PDB可以具有不同的密钥库类型。

您可以使用这些模式配置软件密钥库，硬件密钥库和Oracle Key Vault密钥库。

根据您的站点需求，您可以使用联合模式和隔离模式的混合。例如，如果您希望大多数PDB使用一种类型的密钥库，则可以在CDB根目录中配置密钥库类型（联合模式）。对于此CDB中必须使用不同类型的密钥库的PDB，您可以配置PDB本身以使用它所需的密钥库（隔离模式）。PDB的隔离模式设置将覆盖CDB的联合模式设置。

在配置密钥库以在联合或隔离模式下使用之前，必须使用初始化参数执行一次性配置。要为联合模式和隔离模式配置密钥库，请使用ADMINISTER KEY MANAGEMENT语句。重新启动数据库后，您可以在其中使用ADMINISTER KEY MANAGEMENT语句命令将更改。例如，在配置之前，您无法在CDB根目录的ADMINISTER KEY MANAGEMENT语句中使用EXTERNAL STORE子句，但在配置之后，您可以使用。
