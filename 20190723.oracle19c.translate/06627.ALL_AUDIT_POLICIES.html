2.61 ALL_AUDIT_POLICIES

ALL_AUDIT_POLICIES描述了当前用户可访问的表和视图的细粒度审计策略。

注意：

此视图仅在未启用统一审核的Oracle数据库中填充。在Oracle数据库中启用统一审计时，审计记录将填充在新审计跟踪中，并可从UNIFIED_AUDIT_TRAIL查看。

  • 有关统一审计的详细信息，请参见“ Oracle数据库安全指南” 。

  • 有关迁移到统一审计的详细信息，请参阅“ Oracle数据库升级指南” 。

相关观点

  • DBA_AUDIT_POLICIES描述了数据库中所有细粒度的审计策略。

  • USER_AUDIT_POLICIES描述了当前用户拥有的表和视图的细粒度审计策略。此视图不显示OBJECT_SCHEMA列。

柱 数据类型 空值 描述

OBJECT_SCHEMA

VARCHAR2（128）

包含表或视图的架构的名称

OBJECT_NAME

VARCHAR2（128）

表或视图的名称

policy_owner时

VARCHAR2（128）

政策的所有者

POLICY_NAME

VARCHAR2（128）

政策名称

POLICY_TEXT

VARCHAR2（4000）

审计条件

POLICY_COLUMN

VARCHAR2（128）

相关专栏

PF_SCHEMA

VARCHAR2（128）

审计处理函数的所有者

PF_PACKAGE

VARCHAR2（128）

包含审计处理函数的包的名称

PF_FUNCTION

VARCHAR2（128）

审计处理函数的名称

启用

VARCHAR2（3）

指示策略是启用（ YES ）还是禁用（ NO ）

SEL

VARCHAR2（3）

指示策略是否应用于对象上的查询（ YES ）或否（ NO ）

INS

VARCHAR2（3）

指示策略是否应用于对象上的INSERT语句（ YES ）或否（ NO ）

UPD

VARCHAR2（3）

指示策略是否应用于对象上的UPDATE语句（ YES ）或否（ NO ）

DEL

VARCHAR2（3）

指示策略是否应用于对象上的DELETE语句（ YES ）或否（ NO ）

AUDIT_TRAIL

VARCHAR2（12）

指示将写入此审核策略生成的审核记录的审核跟踪：

  • DB - 审计记录写入DBA_FGA_AUDIT_TRAIL （细粒度审计跟踪）

  • DB+EXTENDED - 将审计记录写入DBA_FGA_AUDIT_TRAIL （细粒度审计跟踪），并为此策略填充SQL_TEXT和SQL_BIND列

  • XML - 审计记录写入V$XML_AUDIT_TRAIL （XML审计文件）

  • XML+EXTENDED - 将审计记录写入V$XML_AUDIT_TRAIL （XML审计文件），并为此策略填充SQL_TEXT和SQL_BIND列

POLICY_COLUMN_OPTIONS

VARCHAR2（11）

表示在所有列是否AUDIT_COLUMN参数（ ALL_COLUMNS ）或任何在列AUDIT_COLUMN参数（ ANY_COLUMNS ）被认为是触发细粒度审计

共同

VARCHAR2（3）

指示策略是否适用于多个容器（ YES ）或否（ NO ）

遗传

VARCHAR2（3）

指示策略是否从其他容器继承（ YES ）（ NO ）
