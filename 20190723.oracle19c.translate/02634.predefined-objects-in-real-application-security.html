在实际应用中安全预定义的对象

本附录描述了Real Application Security中的以下预定义对象：

  • 用户

  • 角色

  • 命名空间

  • 安全类

  • 访问控制列表

用户

XSGUEST - 系统定义的Real Application Security用户通常保留用于匿名访问。

角色

Real Application Security为常规应用程序角色，动态应用程序角色和数据库角色提供预定义的应用程序角色。

本节包括以下主题：

  • 常规应用角色

  • 动态应用程序角色

  • 数据库角色

常规应用角色

Real Application Security提供以下预定义的常规应用程序角色：

  • XSPUBLIC - 此应用程序角色类似到数据库中的PUBLIC角色。它被授予所有Real Application Security应用程序用户。

  • XSBYPASS - 用于绕过的角色限制ACL的系统所施加的限制。

  • XSPROVISIONER - XSPROVISIONER的角色授予PROVISION和CALLBACK权限。

  • XSSESSIONADMIN - 使用的角色用于会话管理。

  • XSNAMESPACEADMIN - 使用的角色用于名称空间属性管理

  • XSCACHEADMIN - 使用的角色用于中间层缓存管理。

  • XSDISPATCHER - 使用的角色用于调度程序的会话管理，命名空间管理和中间层缓存管理。

  • XSCONNECT - 使用的角色控制具有密码的Real Application Security应用程序用户是否可以连接到数据库。

动态应用程序角色

Real Application Security提供以下预定义的动态应用程序角色：

  • DBMS_AUTH

    这个应用角色取决于在应用程序用户的身份验证状态。只要应用程序用户在Real Application Security系统中使用任何数据库身份验证方法作为直接登录应用程序用户进行身份验证，就会启用它。

  • EXTERNAL_DBMS_AUTH

    这个应用角色取决于关于外部应用程序用户的身份验证状态。只要外部应用程序用户在Real Application Security系统中使用任何数据库身份验证方法作为外部直接登录应用程序用户进行身份验证，就会启用它。

  • DBMS_PASSWD

    这个应用角色取决于应用程序用户的身份验证状态。只要应用程序用户在Real Application Security系统中使用密码身份验证方法作为直接登录应用程序用户进行身份验证，就会启用它。

  • MIDTIER_AUTH

    这个应用角色取决于在应用程序用户的身份验证状态。只要应用程序用户通过中间层在Real Application Security系统中进行身份验证，就会启用它。中间层显式将此应用程序角色传递给数据库，指示应用程序用户已通过中间层进行身份验证。

  • XSAUTHENTICATED

    这个应用角色取决于在应用程序用户的身份验证状态。只要应用程序用户在Real Application Security系统中进行身份验证（直接或通过中间层），就会启用它。

  • XSSWITCH

    这个应用角色取决于在应用程序用户的会话状态上。每当应用程序用户的Real Application Security会话由于switch_user操作而创建时，即原始Real Application Security会话中的代理用户切换到应用程序用户时，就会启用它。

数据库角色

Real Application Security提供以下数据库角色。

  • PROVISIONER - 数据库角色具有PROVISION和CALLBACK权限。

  • XS_SESSION_ADMIN - 数据库角色具有ADMINISTER_SESSION权限。

  • XS_NAMESPACE_ADMIN - 数据库角色具有ADMIN_ANY_NAMESPACE权限。

  • XS_CACHE_ADMIN - 数据库角色可用于中间层缓存管理。

命名空间

Real Application Security提供以下预定义的名称空间：

  • XS$GLOBAL_VAR - 包含以下NLS属性： NLS_LANGUAGE ， NLS_TERRITORY ， NLS_SORT ， NLS_DATE_LANGUAGE ， NLS_DATE_FORMAT ， NLS_CURRENCY ， NLS_NUMERIC_CHARACTERS ， NLS_ISO_CURRENCY ， NLS_CALENDAR ， NLS_TIME_FORMAT ， NLS_TIMESTAMP_FORMAT ， NLS_TIME_TZ_FORMAT ， NLS_TIMESTAMP_TZ_FORMAT ， NLS_DUAL_CURRENCY ， NLS_COMP ， NLS_LENGTH_SEMANTICS ，和NLS_NCHAR_CONV_EXCP 。

    可以将XS$GLOBAL_VAR命名空间加载到Real Application Security会话中，而无需任何权限。

  • XS$SESSION - 包含以下属性： CREATED_BY ， CREATE_TIME ， COOKIE ， CURRENT_XS_USER ， CURRENT_XS_USER_GUID ， INACTIVITY_TIMEOUT ， LAST_ACCESS_TIME ， LAST_AUTHENTICATION_TIME ， LAST_UPDATED_BY ， PROXY_GUID ， SESSION_ID ， SESSION_SIZE ， SESSION_XS_USER ， SESSION_XS_USER_GUID ， USERNAME和USER_ID 。

安全类

Real Application Security提供以下预定义的安全类和应用程序权限：

  • DML - DML权限安全性类。如果ACL未指定其安全性类，则DML是ACL的默认安全性类。有关更多信息，请参阅“ DML安全类 ” 。包含以下对象操作的常用应用程序权限。

    • SELECT - 读取对象的权限。

    • INSERT - 插入对象的权限。

    • UPDATE - 更新对象的权限。

    • DELETE - 删除对象的权限。

  • SYSTEM - 系统安全类。包含以下应用程序权限：

    • PROVISION - 从FIDM更新主要文件的特权。从版本12.2开始，还扩展了PROVISION权限，用于创建，删除和修改Real Application Security主体（用户或角色）。此Real Application Security系统特权旨在取代传统使用数据库创建用户，更改用户权限等，以创建和更改Real Application Security应用程序用户和角色。

    • CALLBACK - 注册和更新全局回调的权限。

    • ADMIN_ANY_SEC_POLICY - 任何管理操作的权限。

    • ADMIN_SEC_POLICY - 在自己的架构中管理对象的权限。

    • ADMIN_NAMESPACE - 管理任何命名空间的权限。

  • SESSION_SC - 会话安全类。包含以下应用程序权限：

    • CREATE_SESSION - 创建Real Application Security用户会话的权限。

    • TERMINATE_SESSION - 终止Real Application Security用户会话的权限。

    • ATTACH_SESSION - 附加到Real Application Security用户会话的权限。

    • MODIFY_SESSION - 修改Real Application Security用户会话内容的权限。

    • ASSIGN_USER - 将用户分配给匿名Real Application Security用户会话的权限。

    • ADMINISTER_SESSION - Real Application Security用户会话管理的权限， CREATE_SESSION ， TERMINATE_SESSION ， ATTACH_SESSION ， MODIFY_SESSION和SET_DYNAMIC_ROLES 。

    • SET_DYNAMIC_ROLES - 保护Real Application Security的权限，作为附加会话的一部分并分配用户操作，启用和SET_DYNAMIC_ROLES动态角色。

  • NSTEMPLATE_SC - 命名空间模板安全性类。包含以下应用程序权限：

    • MODIFY_NAMESPACE - 修改会话命名空间的权限。

    • MODIFY_ATTRIBUTE - 修改会话命名空间属性的权限。

    • ADMIN_NAMESPACE - 命名空间管理权限， MODIFY_NAMESPACE和MODIFY_ATTRIBUTE聚合。

访问控制列表

Real Application Security提供以下预定义ACL：

  • SYSTEMACL - ACL的授予SYSTEM安全类权限。

    向PROVISIONER数据库角色和XSPROVISIONER Real Application Security角色授予PROVISION和CALLBACK权限。

    将ADMIN_ANY_SEC_POLICY权限授予DBA数据库角色。

    资助ADMIN_SEC_POLICY权限RESOURCE和XS_RESOURCE数据库角色。

    将ADMIN_ANY_NAMESPACE权限授予DBA和XS_NAMESPACE_ADMIN数据库角色以及XSNAMESPACEADMIN和MIDTIER_AUTH Real Application Security角色。

  • SESSIONACL - ACL for 授予SESSION_SC安全类权限。

    将ADMINISTER_SESSION权限授予XS_SESSION_ADMIN数据库角色和XSSESSIONADMIN Real Application Security角色。

  • NS_UNRESTRICTED_ACL - ACL到将ADMIN_NAMESPACE权限授予PUBLIC数据库角色和XSPUBLIC Real Application Security角色。
