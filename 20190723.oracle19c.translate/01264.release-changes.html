Oracle Database Advanced Security 19c中的更改

以下是Oracle数据库高级安全指南（适用于Oracle Database 19c）中的更改。

  • 改进了对加密Oracle托管表空间的密钥管理支持
    在此版本中，即使Oracle管理的表空间（ SYSTEM ， SYSAUX ， TEMP和UNDO表空间）已加密，现在也允许关闭TDE密钥库。
  • 在非Oracle管理文件模式下自动重命名的透明在线转换支持
    从此版本开始，在非Oracle管理文件模式下的透明数据加密联机转换中，您不再需要在ADMINISTER KEY MANAGEMENT SQL语句中包含FILE_NAME_CONVERT子句。文件名将保留其原始名称。
  • 支持更多离线表空间加密算法
    在以前的版本中，脱机表空间加密仅支持AES128加密算法。除了AES128 ，此版本还引入了对AES192和AES256加密算法的支持，以及用于离线表空间加密的ARIA ， GOST和3DES加密算法。

改进了对加密Oracle托管表空间的密钥管理支持

在此版本中，即使Oracle管理的表空间（ SYSTEM ， SYSAUX ， TEMP和UNDO表空间）已加密，现在也允许关闭TDE密钥库。

即使TDE密钥库处于CLOSED状态，这些表空间在加密时的内部操作仍然不受影响。

关闭TDE密钥库对加密的SYSTEM ， SYSAUX ， TEMP或UNDO表空间的查询没有影响，与用户创建的表空间的查询不同，当TDE密钥库关闭时，继续返回ORA-28365 wallet is not open错误。

用户启动的操作（例如在任何加密的Oracle管理的表空间上解密）仍然需要TDE密钥库处于OPEN状态。

此增强功能有助于防止您以后必须将文件重命名为原始名称，有时会丢失文件。

能够为每个可插拔数据库创建密钥库

每个可插拔数据库（PDB）现在都可以拥有自己的密钥库，而不是只有一个密钥库用于整个容器数据库（CDB）。

在以前的版本中，PDB与整个容器数据库（CDB）共享密钥库，其中包括CDB根和所有PDB。此密钥库存储了CDB的主加密密钥以及所有PDB。在此版本中，PDB可以像以前一样继续与整个CDB共享同一个密钥库，或者拥有单独的密钥库。这种设计在PDB之间提供了更大的隔离，因为每个独立的密钥库都可以独立管理。例如，每个密钥库都可以使用不同的密码进行保护。

此功能的另一个优点是，它可以在多租户环境中由每个租户（PDB）执行独立的密钥管理操作，而不必在CDB根级别共享密钥库。此功能对多租户和非多租户环境都有好处，因为它提供了便于配置密钥库位置和密钥库类型的参数，从而无需编辑sqlnet.ora文件。

此功能提供以下新功能：

  • 对于多租户环境，有以下两种模式：

    • 联合模式，其中密钥库和主加密密钥主要从CDB根管理，并且可以从联合模式PDB访问。在PDB中，可以仅为该PDB打开和关闭密钥库。您还可以为此密钥库创建特定于PDB的主加密密钥。

    • 隔离模式，其中密钥库和加密密钥在单个PDB中管理。这样，每个PDB可以独立配置自己的密钥库类型，并在配置后创建和管理该密钥库。

    您可以混合使用这两种模式。例如，假设您有1个CDB和10个PDB。您可以在联合模式下运行其中3个PDB，其余7个在隔离模式下运行。根据您的环境和场地要求，此设计提供最高的灵活性。

    为了适应这些模式， ADMINISTER KEY MANAGEMENT SQL语句已得到增强，在两种模式下表现不同。

  • 对于非多租户和多租户环境，以下新功能：

    • 添加WALLET_ROOT静态实例初始化参数，以指定密钥库路径。在本指南中， WALLET_ROOT指的是软件密钥库，硬件密钥库和Oracle Key Vault密钥库的配置，但此参数也可用于指定其他产品的钱包位置：企业用户安全性，安全套接字层，Oracle XML DB和安全外部密码存储。

    • 添加TDE_CONFIGURATION动态实例初始化参数，以指定要使用的密钥库的类型。您可以为TDE软件密钥库，硬件安全模块密钥库（HSM）和Oracle Key Vault设置此参数。

    • 修改SQLNET.ENCRYPTION_WALLET_LOCATION参数的行为。设置WALLET_ROOT参数后， WALLET_ROOT覆盖SQLNET.ENCRYPTION_WALLET_LOCATION 。如果WALLET_ROOT尚未设置，然后SQLNET.ENCRYPTION_WALLET_LOCATION是默认的。

这种设计在PDB之间提供了更大的隔离。此功能的另一个优点是，它可以在多租户环境中由每个租户（PDB）执行独立的密钥管理操作，而不必在CDB根级别共享密钥库。此功能对多租户和非多租户环境都有好处，因为它提供了便于配置密钥库位置和密钥库类型的参数，从而无需编辑sqlnet.ora文件。

能够创建用户定义的主加密密钥

此版本引入了创建用户定义的主加密密钥的功能，也称为“自带密钥”。

Oracle数据库现在不支持在数据库中始终生成TDE主加密密钥，而是支持使用在数据库外部生成的主加密密钥。

要创建用户定义的密钥，请在使用ADMINISTER KEY MANAGEMENT SET [ENCRYPTION] KEY或ADMINISTER KEY MANAGEMENT CREATE [ENCRYPTION] KEY语句创建主加密密钥时提供自己的主密钥标识值。此增强功能仅适用于仅在软件密钥库中使用的主加密密钥，而不适用于硬件密钥库。它可用于非多租户，独立环境和多租户环境。

此功能特别有利于Oracle Fusion Applications软件即服务部署。您可以在自己的环境中创建这些密钥。如果要插入您信任的加密密钥，但稍后决定必须替换密钥，则可以在没有云提供程序干预的情况下插入另一个密钥。

要补充此功能，您可以配置从Oracle Data Guard备用数据库自动删除非活动主加密密钥。

能够使用加密密码与Oracle Data Pump进行数据库链接

处理数据库链接密码的行为在此版本中已更改。

数据库链接中的密码现在已加密。Oracle Data Pump处理这些密码的导出和导入。仍然可以使用从旧版本导入操作和导出操作到旧版本。

此功能的好处是可以防止入侵者解密加密的数据库链接密码。
