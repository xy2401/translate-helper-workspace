适用于AIX环境的Oracle ACFS命令行工具

本主题提供AIX操作系统的Oracle ACFS命令的摘要。

在AIX上使用Oracle Grid Infrastructure安装Oracle ACFS和Oracle ADVM期间，将为Oracle ACFS和Oracle ADVM操作创建多个系统安全类（授权）。这些授权可以执行以下操作：

  • 配置Oracle ACFS和Oracle ADVM设备

  • 删除Oracle ACFS和Oracle ADVM设备

  • Oracle ACFS和Oracle ADVM设备的定义

这些授权收集在一个角色（ oracle_devmgmt ）中，该角色默认分配给Oracle Grid Infrastructure用户和Oracle ASM管理员。您可以在安装后运行lsrole或rolelist命令以查看此新角色。可以随时运行这些命令，以确保用户具有管理Oracle ACFS和Oracle ADVM的适当权限。

例如，以Oracle ASM管理员身份运行以下命令：

$ rolelist -e oracle_devmgmt

在AIX操作系统上进行初始安装后，运行根脚本的shell不会继承新角色。如果用户不存在该角色，则必须在执行Oracle ACFS或Oracle ADVM操作之前运行swrole命令。

例如，以Oracle ASM管理员身份运行以下命令：

$ swrole oracle_devmgmt

或者，您可以打开新shell以执行Oracle ACFS或Oracle ADVM操作。

在删除Oracle Grid Infrastructure期间，将从系统中删除oracle_devmgmt角色及其关联的授权。

也可以看看：

AIX上的Oracle Grid Infrastructure，有关安装的详细信息，请参阅POWER Systems（64位）上的适用于IBM AIX的Oracle Grid Infrastructure安装指南

注意：

如果在AIX上使用force选项卸载磁盘组，则该磁盘组上的所有Oracle ADVM卷设备文件仍保留在系统上。重新装入磁盘组时将删除这些文件。

fsck的

目的

检查并修复AIX操作系统上的Oracle ACFS文件系统。

语法和说明

fsck -V acfs [-n | -y] [-o options ] volume_device

表16-13包含AIX fsck命令可用的选项。

表16-13 AIX fsck命令的选项

选项 描述

-V acfs

指定AIX上的Oracle ADVM卷。 acfs指定Oracle ACFS类型。

-n

对任何提示回答否。

-y

对任何提示回答是。

-o options

指定选项（a，f，v）。选项前面带有-o标志，并以逗号分隔的字符串形式输入。例如： -o a,v

  • 一个

    指定自动修复文件系统。

  • F

    强制文件系统进入可装入状态而不完成文件系统检查或修复。

  • v

    指定详细模式。操作发生时显示进度。以详细模式运行会影响性能。

volume_device

指定主Oracle ADVM卷设备。

fsck检查并修复现有的Oracle ACFS。此命令只能在卸除的文件系统上运行。运行fsck需要root权限。必须加载Oracle ACFS驱动程序才能使fsck正常工作。

默认情况下， fsck仅检查并报告任何错误。必须指定-a标志以指示fsck修复文件系统中的错误。在修复操作期间不要中断fsck 。

在少数情况下， fsck会在继续检查文件系统之前提示问题。这些案件包括：

  • 如果fsck检测到文件系统上正在进行另一个fsck

  • 如果fsck检测到未加载Oracle ACFS驱动程序

  • 如果文件系统似乎不是Oracle ACFS

在检查模式下， fsck还会提示是否存在由于未完全关闭而未完全处理的事务日志。要以非交互模式运行，请包含-y或-n选项以对任何问题回答是或否。

fsck在检查文件系统之前创建工作文件。如果空间可用，则在/usr/tmp中创建这些工作文件。 /tmp用于如果/usr/tmp不存在。如果tmp目录中的空间不足，则fsck尝试写入当前工作目录。fsck创建的文件大致是被检查文件系统的大小除以32K。最多分配三个这样的文件。例如，正在检查的2 GB文件系统会导致fsck在/usr/tmp目录中生成一到三个64K工作文件。fsck完成后会删除这些文件。

如果fsck在文件系统中找到文件或目录，但无法确定其名称或目标位置（可能是由于其父目录中的损坏），则当fsck为fsck时，它会将此对象放在/lost+found目录中以修复模式运行。出于安全原因，只有Linux上的root用户可以读取/lost+found文件。如果管理员稍后可以根据其内容确定文件的原始名称和位置，则可以将文件移动或复制到其预期位置。

/lost+found目录中的文件名采用以下格式：

家长。 id .file。 id 。 自1970年以来的父母时间。 id .dir。 id 。 时间在仲因为-1970

id字段是文件系统中每个文件和目录的内部Oracle ACFS数字标识符。

您可以使用acfsutil info id id mount_point尝试确定与parent.关联的目录parent. id 。假定此目录是已删除对象的来源。有关acfsutil info ，请参阅“ acfsutil信息文件 ” 。

如果父目录未知，则父id字段设置为UNKNOWN 。

注意：

无法从快照中查看/lost+found目录的内容。

的mkfs

目的

在AIX操作系统上创建Oracle ACFS文件系统。

语法和说明

mkfs -V acfs -oh / dev / null mkfs -V acfs [-v name ] [-s size ] [ -  o options ] volume_device [-c release_version ]

mkfs -V acfs -o h /dev/null显示用法文本并退出。

表16-14包含AIX mkfs命令可用的选项。

表16-14 AIX mkfs命令的选项

选项 描述

-V acfs

指定AIX上的文件系统类型。 acfs指定Oracle ACFS类型。

-v name

指定文件系统的名称。名称最多可包含64个字符。 acfsutil info fs返回名称（如果已指定）。

-s size

以512字节为单位或以K |为单位指定文件系统的大小M | G | T | P指定的单位是K （千字节）， M （兆字节）， G （千兆字节）， T （兆兆字节）或P （千兆字节）。

-o options

指定选项遵循（f，h，v）。选项前面带有-o标志，并以逗号分隔的字符串形式输入。例如： -o f,v

  • F

    指定强制选项。即使卷设备上存在现有Oracle ACFS，此操作也会创建文件系统，但前提是文件系统已卸除。此选项会覆盖原始文件系统上的结构。请谨慎使用此选项。

  • H

    显示用法文本并退出。

  • i = 512 | I = 4096

    指定元数据块大小为512或4096字节的文件系统。

  • v

    指定详细模式。操作发生时显示进度。

volume_device

指定要格式化的Oracle ADVM设备文件。

-c release_version

创建Oracle ACFS文件系统，并将Oracle ACFS版本兼容性设置为release_version的值。

您可以使用mkfs创建要装入Oracle ACFS文件系统所需的磁盘结构。mkfs命令是用于构建文件系统的传统Linux命令。mkfs成功运行后， V$ASM_VOLUME视图中的USAGE列显示ACFS 。 root权限不是必需的。卷设备文件的所有权决定了谁可以运行此命令。

使用-c release_version选项指定的值必须大于或等于磁盘组的COMPATIBLE.ADVM值，并且必须小于或等于正在运行的Oracle Grid Infrastructure版本。如果未指定—c release_version ，则使用COMPATIBLE.ADVM的值。设置兼容性后，无法降级。—c release_version选项可用于无法或不希望更新磁盘组的COMPATIBLE.ADVM和COMPATIBLE.ASM的情况，但您希望使用需要增加兼容性的Oracle ACFS功能。更新Oracle ACFS兼容性后，将无法再使用较早的Oracle Grid基础结构版本安装文件系统。要更改现有文件系统的兼容性，请参阅acfsutil compat set 。有关Oracle ASM磁盘组兼容性属性的信息，请参阅磁盘组兼容性 。

对于512字节元数据块大小格式，最小文件系统大小为200 MB，对于4 KB元数据块大小格式，最小文件系统大小为512 MB。

必须加载Oracle ACFS驱动程序才能使mkfs正常工作。

4 KB扇区/元数据

以下列表总结了基于COMPATIBLE.ADVM磁盘组属性值的-oi选项和元数据块大小的使用。

  • 如果COMPATIBLE.ADVM设置为12.2或更高，则默认情况4096数据块大小为4096字节。

  • 如果COMPATIBLE.ADVM设置为小于12.2 ，则块大小设置为512字节。

  • 如果COMPATIBLE.ADVM设置为小于12.2且逻辑扇区大小不是512字节，则命令失败。

对于普通用户IO请求，用户数据IO继续支持小至512字节的传输。当文件系统的Oracle ADVM卷的逻辑磁盘扇区大小为4 K时，用户Direct IO请求应在4 K文件偏移上对齐，并且是4 KB长度的倍数，以获得最佳性能。请注意，只有4 K元数据块大小格式支持具有4 K逻辑磁盘扇区的ADVM卷。

必须将COMPATIBLE.ADVM值设置为12.2或更高，以格式化逻辑磁盘大小为4096的Oracle ADVM卷，或使用-oi=4096选项格式化逻辑磁盘扇区大小为512字节的Oracle ADVM卷。

安装

目的

在AIX操作系统上安装Oracle ACFS文件系统。

语法和说明

mount -V acfs -oh / dev / null / xxx mount -v acfs [-o options ] volume_device mount_point

mount -V acfs -o h /dev/null /xxx显示用法文本和退出。

表16-15 AIX mount命令的选项

选项 描述

-v acfs

指定AIX上的文件系统类型。 acfs指定Oracle ACFS类型。

-o options

指定选项如下。选项前面带有-o标志，后跟逗号分隔的选项字符串。例如： -o all,ro,v

可以使用以下选项：

  • 所有

    读取Oracle Clusterware命名空间中的Oracle ACFS受管文件系统，使用AUTO_START=always挂载所有文件系统。

    -o all选项需要两个占位符来满足命令参数：卷设备和目录的伪参数。

    指定不允许通过此挂载执行setuid和setgid程序。

  • nodev

    指定您无法从此安装中打开设备。

  • norootsuid

    由权限允许设置用户ID执行且由root拥有的非root用户执行二进制文件失败。尝试以非root用户身份运行这些可执行文件失败，并显示权限被拒绝错误。

  • rootsuid

    允许root用户拥有的set user Id文件的非root用户执行二进制文件。这是默认操作。

  • RO

    以只读模式安装文件系统。

  • RW

    以读写模式安装文件系统。

  • v

    指定详细模式。操作发生时显示进度。

volume_device

指定已由mkfs格式化的Oracle ADVM卷设备文件。 device是必需的，但可以是虚拟值。

装入点

指定安装文件系统的目录。在运行mount命令之前，此目录必须存在。

mount将文件系统附加到作为目录名称的安装点的Oracle ACFS层次结构。挂载发生在发出mount命令的节点上。如果文件系统未在此节点上处于卸载状态，则mount命令将返回错误。

并不总是可以将mount错误的原因返回到mount命令。发生这种情况时，Oracle ACFS会将失败原因写入系统控制台和关联的系统日志文件。

mount成功运行后， V$ASM_VOLUME视图中的MOUNTPATH字段将显示现在挂载文件系统的目录名称。

Oracle ACFS文件系统只应安装在一个安装点上。应在所有集群成员上使用相同的安装点名称。

root权限是运行mount所必需的。

使用umount / umountall取消

目的

在AIX操作系统上卸载Oracle ACFS文件系统。

语法和说明

umount [ mount_point | volume_device ] umountall -F acfs

unmountall -F acfs卸载所有Oracle ACFS文件系统。

表16-16包含umount命令可用的选项。

表16-16 AIX umount命令的选项

选项 描述

装入点

指定安装文件系统的目录。仅对umount有效。

volume_device

指定与文件系统关联的Oracle ADVM卷设备名称。仅对umount有效。

-F acfs

指定AIX上的文件系统类型。 acfs指定Oracle ACFS类型。仅对umountall有效。

umount和umountall从当前节点上的文件系统层次结构中分离Oracle ACFS。

在完全卸载文件系统之前， umount和umountall命令不会将控制权返回到操作系统提示符。

如果文件系统繁忙，则umount和umountall会失败。

运行umount和umountall命令需要root权限。
