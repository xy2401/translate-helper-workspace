4使用JSON列创建表

您可以创建具有JSON列的表。您使用SQL条件is json作为检查约束，以确保插入到列中的数据是（格式正确的）JSON数据。Oracle建议您在创建用于JSON数据的列时始终使用is_json检查约束。

例4-1和例4-2说明了这一点。他们创建并填充一个表，其中包含本文档中其他地方的示例中使用的数据。

为简洁起见，在例4-2中只插入了两行数据（一个JSON文档）。

注意：

SQL / JSON条件IS JSON和IS NOT JSON对于任何非NULL SQL值都返回true或false。但它们都为SQL NULL返回未知（既不是真也不是假）。在检查约束中使用时，它们不会阻止将SQL NULL值插入列中。（但是当在SQL WHERE子句中使用时，永远不会返回SQL NULL 。）

确实，检查约束会降低数据插入的性能。如果您确定应用程序仅将格式良好的JSON数据插入特定列，请考虑禁用检查约束，但不要删除约束。

示例4-1在检查约束中使用IS JSON以确保JSON数据格式正确

CREATE TABLE j_purchaseorder（id VARCHAR2（32）NOT NULL PRIMARY KEY，date_loaded TIMESTAMP（6）WITH TIME ZONE，po_document VARCHAR2（23767） CONSTRAINT ensure_json CHECK （po_document IS JSON ））;

4.1确定列是否必须包含JSON数据

如何判断表或视图的给定列是否是格式正确的JSON数据？只要是这种情况，该列就会列在以下静态数据字典视图中： DBA_JSON_COLUMNS ， USER_JSON_COLUMNS和ALL_JSON_COLUMNS 。

这些视图中的每一个都列出了列名，数据类型和格式（ TEXT或BINARY ）;表或视图名称（列TABLE_NAME ）;以及对象是表还是视图（列OBJECT_TYPE ）。

对于要被视为JSON数据的表列，它必须具有is json检查约束。但是在视图的情况下，以下任何一个条件都足以将列视为JSON数据：

  • 底层数据有一个is json检查约束。

  • 该列来自使用SQL / JSON函数json_query 。

  • 该列来自使用JSON生成函数，例如json_object 。

  • 该列使用SQL函数treat关键字AS JSON 。

如果is json检查约束确定表列是JSON数据，并且如果稍后停用该约束，则该列仍然在视图中列出。如果删除了检查约束，则会从视图中删除该列。

注意：

如果检查约束将条件is json与使用逻辑条件OR另一个条件相结合 ，则该列不会在视图中列出。在这种情况下，不确定列中的数据是否为JSON数据。例如，约束jcol is json OR length(jcol) < 1000 不能确保列jcol中的数据是JSON数据。
