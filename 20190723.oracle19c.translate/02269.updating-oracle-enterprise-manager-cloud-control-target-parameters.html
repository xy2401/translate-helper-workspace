更新Oracle企业管理器云控制目标参数

升级Oracle Grid Infrastructure后，升级企业管理器云控制目标。

由于Oracle Grid Infrastructure 19c是Oracle Clusterware主目录在新位置（集群主目录的Oracle Grid Infrastructure或Grid主目录）的CRS_HOME ，因此必须更改某些参数文件中CRS_HOME参数的路径。如果不更改参数，则会在Oracle Enterprise Manager Cloud Control上遇到诸如“cluster target broken”之类的错误。

要解决此问题，请更新企业管理器云控制目标，然后在运行代理的每个集群成员节点上更新企业管理器代理基本目录。

  • 升级后更新企业管理器云控制目标
    升级Oracle Grid Infrastructure后，使用新的Grid主路径更新企业管理器目标。
  • 升级后更新企业管理器代理基本目录
    升级Oracle Grid Infrastructure后，在运行代理的每个集群成员节点上更新企业管理器代理基本目录。
  • 升级后向Oracle Enterprise Manager注册资源
    升级Oracle Grid Infrastructure后，将新资源目标添加到Oracle Enterprise Manager Cloud Control。
