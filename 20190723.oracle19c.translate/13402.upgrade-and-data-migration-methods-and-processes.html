升级和数据迁移方法和过程

Oracle提供的功能和产品可自动完成升级过程，并帮助您有效地完成升级。

Oracle数据库支持以下用于将数据库升级或迁移到新版本的方法：

  • 数据库升级助手（DBUA）

    提供图形用户界面，指导您完成数据库的升级。DBUA可以在使用Oracle Universal Installer进行安装期间启动，也可以在将来随时启动DBUA作为独立工具。

  • 使用Parallel Upgrade Utility和其他命令行实用程序手动升级

    允许使用shell脚本执行升级。

  • 使用Oracle Data Pump迁移数据

    提供导出和导入实用程序。Oracle Data Pump可以从您的数据库执行完全或部分导出，然后完全或部分导入到Oracle数据库的新版本中。Oracle Data Pump中的导出/导入可以复制数据的子集，保持数据库不变。

  • CREATE TABLE AS SQL语句

    将数据从数据库迁移到新的Oracle数据库版本。通过使用此方法，您可以复制数据的子集，保持数据库不变。

  • 使用优先级列表升级CDB和PDB，以根据优先级对PDB进行分组和升级。

    使用-L选项运行并行升级实用程序（ dbupgrade或catctl.pl ）以使用优先级列表运行升级，并在升级运行时调用该列表。

  • 同步备用数据库，升级和使用Oracle GoldenGate同步升级后的数据库（零停机时间选项）

  • 使用Fleet Patching and Provisioning（FPP）升级数据库。

    在Fleet Patching and Provisioning（FPP）升级（以前称为Rapid Home Provisioning）中，您将完成新的Oracle数据库安装。测试数据库并根据要用于数据库的标准操作环境（SOE）对其进行修改后，可以创建FPP黄金映像。DBA将该黄金映像的实例部署到具有您要升级的早期版本数据库的服务器。部署这些黄金映像后，DBA可以运行单个rhpctl命令来移动文件，执行配置更改以及执行使用新二进制文件所需的其他步骤。有关Rapid Home Provisioning的详细信息，请参阅“ Oracle Clusterware管理和部署指南” 。
