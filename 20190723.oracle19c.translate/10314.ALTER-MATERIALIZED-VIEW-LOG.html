ALTER MATERIALIZED VIEW LOG

目的

物化视图日志是与物化视图的主表关联的表。使用ALTER MATERIALIZED VIEW LOG语句可以更改现有物化视图日志的存储特征或类型。

注意：

支持使用关键字SNAPSHOT代替MATERIALIZED VIEW以实现向后兼容。

也可以看看：

  • 有关创建物化视图日志的信息，请创建MATERIALIZED VIEW LOG

  • ALTER MATERIALIZED VIEW了解有关物化视图的更多信息，包括刷新它们

  • CREATE MATERIALIZED VIEW用于描述各种类型的物化视图

先决条件

您必须是主表的所有者，或者您必须具有主表的READ或SELECT权限以及物化视图日志的ALTER权限。

语义

力

如果指定FORCE并且已为物化视图日志指定了使用ADD子句指定的任何项，则Oracle数据库不会返回错误，而是静默忽略现有元素，并将任何尚未存在的项添加到物化视图日志中存在于日志中。同样，如果指定INCLUDING NEW VALUES并且已为物化视图日志指定了该属性，则Oracle数据库将忽略冗余并且不会返回错误。

模式

指定包含主表的模式。如果省略schema ，则Oracle数据库假定物化视图日志位于您自己的模式中。

表

指定与要更改的实例化视图日志关联的主表的名称。

physical_attributes_clause

physical_attributes_clause允许您更改PCTFREE ， PCTUSED和INITRANS参数的值以及物化视图日志，分区，溢出数据段或分区物化视图日志的默认特征的存储特征。

物化视图日志物理属性的限制

如果物化视图日志驻留在本地管理的表空间中，则无法使用storage_clause修改范围参数。有关这些参数的说明，请参阅CREATE TABLE 。

add_mv_log_column_clause

将列添加到实例化视图日志的主表时，数据库不会自动将列添加到实例化视图日志中。因此，请使用此子句将列添加到实例化视图日志中。如果主表的相应列已加密，Oracle数据库将对新添加的列进行加密。

alter_table_partitioning

分区子句的语法和一般功能与ALTER TABLE语句中描述的相同。请参阅ALTER TABLE文档中的alter_table_partitioning 。

改变物化视图日志分区的限制

更改实例化视图日志分区受以下限制：

  • 修改modify_LOB_storage_clause化视图日志的分区时，不能使用LOB_storage_clause或modify_LOB_storage_clause 。

  • 如果您尝试删除，截断或交换实例化视图日志分区，则Oracle数据库会引发错误。

parallel_clause

parallel_clause允许您指定物化视图日志是否支持并行操作。

有关此子句的完整信息，请参阅CREATE TABLE文档中的parallel_clause 。

logging_clause

指定实例化视图日志的日志记录属性。有关此子句的完整说明，请参阅logging_clause 。

allocate_extent_clause

使用allocate_extent_clause为实例化视图日志显式分配新范围。有关此子句的完整说明，请参阅allocate_extent_clause 。

shrink_clause

使用此子句可压缩实例化视图日志段。有关此子句的完整信息，请参阅CREATE TABLE文档中的shrink_clause 。

move_mv_log_clause

使用MOVE子句将实例化视图日志表移动到其他表空间，更改实例化视图日志的其他段或存储属性，或更改实例化视图日志的并行度。

移动物化视图日志的限制

segment_attributes的storage_clause的ENCRYPT子句对segment_attributes化视图日志无效。

CACHE | NOCACHE条款

对于将频繁访问的数据， CACHE指定在执行全表扫描时，为此日志检索的块将放置在缓冲区高速缓存中最近使用的LRU列表末尾。此属性对小型查找表很有用。NOCACHE指定将块放置在LRU列表的最近最少使用的末尾。有关此子句的更多信息，请参阅有关CREATE TABLE的文档中的“ CACHE | NOCACHE | CACHE READS ” 。

mv_log_augmentation

使用ADD子句扩充实例化视图日志，以便在更改实例化视图主表中的行时记录主键值，rowid值，对象ID值或序列。此子句还可用于记录其他列。

要停止记录任何此类信息，必须先删除物化视图日志，然后重新创建它。删除物化视图日志然后重新创建它会强制完全刷新每个现有物化视图，这些视图在下次刷新时依赖于主表。

增加物化视图日志的限制

对于每个物化视图日志，只能为列表列表指定一个PRIMARY KEY ，一个ROWID ，一个OBJECT ID ，一个SEQUENCE和每列。您只能在此ALTER语句中指定一次出现的PRIMARY KEY ， ROWID ， OBJECT ID ， SEQUENCE和列列表。此外，如果在创建时指定了任何这些值（隐式或显式），则除非使用FORCE选项，否则不能在此ALTER语句中指定该值。

对象ID

如果希望将更改的所有行的相应对象标识符记录在实例化视图日志中，请指定OBJECT ID 。

对OBJECT ID子句的限制

您只能为对象表上的日志指定OBJECT ID ，并且不能为存储表指定它。

首要的关键

如果要将更改的所有行的主键值记录在物化视图日志中，请指定PRIMARY KEY 。

ROWID

如果要将更改的所有行的rowid值记录在实例化视图日志中，请指定ROWID 。

序列

指定SEQUENCE以指示应在物化视图日志中记录提供附加排序信息的序列值。

柱

为要更改的所有行指定要在实体化视图日志中记录其值的其他列。通常，这些列是筛选器列（子查询实例化视图引用的非主键列）和连接列（在子查询的WHERE子句中定义连接的非主键列）。

新价值条款

NEW VALUES子句允许您指定Oracle数据库是否在物化视图日志中保存更新DML操作的旧值和新值。您在此子句中设置的值适用于日志中的所有列，而不仅适用于您在此ALTER MATERIALIZED VIEW LOG语句中添加的列。

包含

指定INCLUDING以在日志中保存新旧值。如果此日志用于具有单表实现聚合视图的表，并且您希望实例化视图有资格进行快速刷新，则必须指定INCLUDING 。

不包括

指定EXCLUDING以禁用在日志中记录新值。您可以使用此子句来避免记录新值的开销。

如果您在此表上定义了快速刷新的单表物化聚合视图，则除非您首先将物化视图的刷新模式更改为FAST以外的其他VALUES否则不要指定EXCLUDING NEW VALUES 。

mv_log_purge_clause

使用此子句以下列方式更改实例化视图日志的清除属性：

  • 将清除从IMMEDIATE SYNCHRONOUS更改为IMMEDIATE ASYNCHRONOUS或从IMMEDIATE ASYNCHRONOUS更改为IMMEDIATE SYNCHRONOUS

  • 将清除从IMMEDIATE更改为已安排或从已安排更改为IMMEDIATE

  • 指定新的开始时间和新的next时间和interval

如果要将清除从计划更改为IMMEDIATE ，则会删除与该物化视图日志关联的计划清除作业。如果要将清除从IMMEDIATE更改为已调度，则会使用提供的属性创建清除作业。如果要更改计划的清除属性，则只会在调度程序清除作业中更改指定的那些属性。

如果要将日志清除更改为其当前状态（即，您没有进行任何更改），则必须指定FORCE ，除非您要更改计划的清除属性。

要了解是否已为此物化视图日志设置了清除时间或间隔，请查询*_MVIEW_LOGS数据字典视图。有关此子句的完整语义，请参见CREATE MATERIALIZED VIEW LOG子句mv_log_purge_clause 。

for_refresh_clause

使用此子句可以更改将使用实例化视图日志的刷新方法。

用于同步刷新

指定此子句以从快速刷新更改为同步刷新，或完成刷新到同步刷新。将创建一个临时日志。

如果要从快速刷新更改，请在使用此子句之前确保满足以下条件：

  • 已消耗物化视图日志中的所有更改。

  • 已刷新与主表关联的任何按需刷新实例化视图。

  • 与主表关联的任何提交时刷新实体化视图已转换为按需刷新实例化视图。

使用此子句后，无法直接在主表上执行DML操作。必须使用DBMS_SYNC_REFRESH程序包中的过程来准备和执行更改数据操作。

快速刷新

指定此子句以从同步刷新更改为快速刷新，或完成刷新以快速刷新。将创建物化视图日志。

如果要从同步刷新更改为快速刷新，请确保在使用此子句之前已暂停登台日志中的所有更改。

使用此子句后，可以直接在主表上执行DML操作。

有关此子句的完整语义，请参阅CREATE MATERIALIZED VIEW LOG子句for_refresh_clause 。

例子

Rowid物化视图日志：示例

以下语句更改现有主键物化视图日志以记录rowid信息：

ALTER MATERIALIZED VIEW LOG ON order_items ADD ROWID;

物化视图日志排除新值：示例

以下语句通过添加过滤器列并排除新值来更改hr.employees上的物化视图日志。使用此日志的任何物化聚合视图将不再可快速刷新。但是，如果不再需要快速刷新，则此操作可避免记录新值的开销：

ALTER MATERIALIZED VIEW LOG ON员工ADD（commission_pct）排除新价值;
