关于监控Oracle Database Vault

您可以使用Oracle Enterprise Manager Cloud Control中的Database Vault主页来监视启用Database Vault的数据库。

此功能显示前五次尝试违规以及前五名违规者是谁。违规行为包括违反领域和命令规则的行为。尝试的违规者分为用户和客户端主机。通过单击Top 5 Attempted Violations下的Oracle Database Vault链接，您可以找到详细信息，例如违规类型，发生时间，用户是谁等等。同样，如果单击Top 5 Attempted Violators下的用户链接（例如， SYS ），您可以找到有关每个违规者的详细信息，例如他们执行的操作，发起操作的客户端主机名以及违规发生的时间。您可以手动刷新数据，并限制数据视图，例如在过去24小时内。此页面还显示了一个列出已生成的所有警报的表。

在您可以查看这些事件之前，如果尚未将数据库迁移到统一审计，则必须确保将AUDIT_TRAIL初始化参数设置为DB或DB, EXTENDED 。如果已迁移数据库以使用统一审核，则无需配置任何其他设置。您已准备好检查是否存在安全违规。

监控安全违规和配置更改

已被授予相应角色的用户可以使用Oracle Database Vault Administrator来监视安全违规和配置更改。

  1. 以已授予DV_OWNER ， DV_ADMIN或DV_SECANALYST角色和SELECT ANY DICTIONARY权限的用户身份从Cloud Control登录Oracle Database Vault Administrator。从Oracle Enterprise Cloud Control登录Oracle Database Vault说明了如何登录。
  2. 选择“ 主页”选项卡。

    出现类似于以下内容的页面：



    em_dv_home_page.png插图的描述
  3. 要查找特定时间（例如过去7天）的尝试违规，请从右上角的“ 时间序列”按钮下的菜单中进行选择。

    您还可以通过单击“ 时间序列”按钮将饼图更改为图形。

  4. 要查找配置问题报告 ， 执行审核报告 ， 配置更改审核报告和模拟模式报告 ，请选择Database Vault报告下的相应链接。

    有关Database Vault报告的详细信息，请参阅Oracle Database Vault Reports 。
