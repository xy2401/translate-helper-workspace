一个禁用和启用Oracle标签安全

您可以根据需要禁用和启用Oracle Label Security。

注意：

Oracle不支持卸载Oracle Label Security。

  • 何时必须禁用Oracle Label Security
    您可能需要禁用Oracle Label Security以执行升级任务或更正错误配置。
  • 禁用Oracle Label Security
    如果已启用Oracle Database Vault，则不要禁用Oracle Label Security。
  • 启用Oracle Label Security
    您可以在SQL * Plus中启用Oracle Label Security。

何时必须禁用Oracle Label Security

您可能需要禁用Oracle Label Security以执行升级任务或更正错误配置。

禁用Oracle Label Security的另一个原因是，如果要在不强制执行Oracle Label Security的情况下测试应用程序。完成任务后，您可以重新启用Oracle Label Security。

父主题： 禁用和启用Oracle Label Security

禁用Oracle Label Security

如果已启用Oracle Database Vault，则不要禁用Oracle Label Security。

请参阅“ Oracle Database Vault管理员指南”以查找是否已启用Database Vault。

要禁用Oracle Label Security：

  1. 以SYS用户或已被授予LBAC_DBA角色的用户身份登录数据库实例。

    例如：

    sqlplus psmith_ols - 或者，psrith_ols @hrpdb为hrpdb可插拔数据库（PDB）输入密码： 密码
  2. 运行以下过程：
    EXEC LBACSYS.OLS_ENFORCEMENT.DISABLE_OLS;
  3. 重启数据库。

    例如：

    CONN SYS as SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP
  4. 对于Oracle Real Application Cluster（Oracle RAC）环境或多租户环境，请对启用了Oracle Label Security的每个Oracle RAC节点或PDB重复这些步骤。

父主题： 禁用和启用Oracle Label Security

启用Oracle Label Security

您可以在SQL * Plus中启用Oracle Label Security。

  1. 以SYS用户或已被授予LBAC_DBA角色的用户身份登录数据库实例。

    例如：

    sqlplus psmith_ols - 或者，psmith_ols @ hrpdb为hrpdb PDB Enterp密码： 密码
  2. 运行以下过程：
    EXEC LBACSYS.OLS_ENFORCEMENT.ENABLE_OLS;
  3. 重启数据库。

    例如：

    CONN SYS as SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP
  4. 对于Oracle Real Application Cluster（Oracle RAC）环境或多租户环境，请对禁用Oracle Label Security的每个Oracle RAC节点或PDB重复这些步骤。
