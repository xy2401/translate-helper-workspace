4为Oracle Grid Infrastructure配置Oracle Solaris操作系统

在开始安装之前，请完成操作系统配置要求并检查Oracle Solaris操作系统。

  • Oracle Solaris操作系统安装指南
    确定如何安装Oracle Solaris。
  • 查看操作系统和软件升级最佳实践
    这些主题提供有关升级和迁移的一般规划指南和特定于平台的信息。
  • 查看操作系统安全通用实践
    安全操作系统是一般系统安全性的重要基础。
  • 关于安装修复脚本
    Oracle Universal Installer检测何时不满足安装的最低要求，并创建称为修复脚本的shell脚本，以完成不完整的系统配置步骤。
  • 关于操作系统要求
    根据您要安装的产品，验证您是否安装了所需的操作系统内核和软件包。
  • SPARC上的Oracle Solaris的操作系统要求（64位）
    本版本中列出的内核和软件包适用于Oracle数据库和Oracle Grid Infrastructure的SPARC 64位系统上的此版本。
  • 适用于Oracle Solaris的其他驱动程序和软件包
    有关可选驱动程序和软件包的信息。
  • 检查Oracle Solaris的软件要求
    检查Oracle Solaris操作系统的软件要求，以确定它们是否满足最低安装要求。
  • 关于SPARC上的Oracle Solaris Cluster配置
    如果要在SPARC处理器服务器上安装Oracle Grid Infrastructure，请查看以下信息。
  • 使用Oracle Solaris Cluster在x86上运行rootpre.sh脚本
    在运行Oracle Solaris的x86（64位）平台上，如果除了Oracle Clusterware之外还安装了Oracle Solaris Cluster，则完成以下任务。
  • 启用名称服务缓存守护程序
    要允许Oracle Clusterware更好地容忍使用NAS设备或NFS挂载的网络故障，请启用名称服务缓存守护程序（ nscd ）。
  • 为集群时间同步设置网络时间协议
    Oracle Clusterware要求在所有群集节点上设置相同的时区环境变量。
  • 在安装期间使用自动SSH配置
    要安装Oracle软件，请在所有集群成员节点之间配置安全shell（SSH）连接。
