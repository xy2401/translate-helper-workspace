Oracle Database Release 19c，版本19.1中的更改

Oracle数据库内存指南（适用于Oracle数据库）19c版，19.1版具有以下更改。

新功能

以下主要功能是此版本中的新功能。

  • 数据库内存等待填充

    DBMS_INMEMORY_ADMIN.POPULATE_WAIT函数启动优先级大于或等于指定优先级的所有INMEMORY对象的填充，并返回填充的状态值。用户指定的时间间隔指定函数在将值返回给调用者之前等待的最长时间。

    请参阅“ 强制内存中对象的初始填充 ” 。

  • 内存外部表的大数据和性能增强

    此版本为内存外部表功能引入了几个可管理性和性能改进：

    • 支持ORACLE_HIVE和ORACLE_BIGDATA驱动程序。

    • 支持并行查询。

    • 全表扫描会填充In-Memory外部表。在以前的版本中，使用POPULATE或REPOPULATE过程需要DBMS_INMEMORY 。

    • 内存中的后台进程，而不是前台进程，现在丢弃IM段。

    请参阅“ 内存外部表 ” 。

  • 混合分区表

    分区可以驻留在Oracle数据库段以及外部文件和源中。此功能显着增强了大数据SQL的分区，其中表的大部分可以驻留在外部分区中。只有混合分区表的内部分区INMEMORY继承INMEMORY属性。

    请参阅“ 内存表 ” 。

  • Oracle Database Resource Manager自动为Database In-Memory启用

    当INMEMORY_SIZE大于0 ，将自动启用资源管理器。

    请参阅“ IM动态扫描的界面 ” 。

  • Oracle Data Guard多实例重做应用支持IM列存储

    将初始化参数ENABLE_IMC_WITH_MIRA设置为TRUE可在Active Data Guard备用数据库上同时启用IM列存储和Data Guard多实例重做应用。默认情况下， ENABLE_IMC_WITH_MIRA为FALSE 。

    请参阅Oracle数据库参考以了解有关ENABLE_IMC_WITH_MIRA更多信息。

Oracle Database Release 18c，版本18.1中的更改

Oracle数据库内存指南（适用于Oracle数据库）18c版，18.1版具有以下更改。

  • 新功能
    以下主要功能是此版本中的新功能。

上级主题： Oracle Database In-Memory指南中此发行版的更改

新功能

以下主要功能是此版本中的新功能。

  • 自动内存

    此功能使用段和列使用情况统计信息自动管理IM列存储的内容。如果填充作业因IM列存储已满而失败，则自动内存中驱逐非活动段以为活动段腾出空间。

    请参阅“ 配置自动内存中 ” 。

  • 内存中动态扫描

    IM动态扫描通过使用轻量级进程线程自动并透明地并行化表扫描。当Oracle资源管理器感知到CPU资源空闲并且可以利用它们来加速查询时，它们会分配这些线程。

    请参阅“ 内存中动态扫描 ” 。

  • IM表达式窗口捕获

    您可以定义任意长度的表达式捕获窗口 ，以确保仅考虑在此窗口中出现的表达式进行内存中实现。当您知道代表整个工作负载的小间隔时，此机制特别有用。例如，在交易窗口期间，经纪公司可以收集表达式集，并在IM列存储中实现它们，以加速整个工作负载的未来查询处理。

    请参阅“ 表达式捕获间隔 ” 。

  • 外部表的内存支持

    您可以将外部表填充到IM列存储中。此功能对于组合内部和外部数据的分析查询非常有用。

    请参阅“ 内存外部表 ”和“ 使用DBMS_INMEMORY.POPULATE填充内存外部表：示例 ” 。

  • 内存中优化算法

    对于使用QUERY LOW压缩的表格， NUMBER列使用优化格式进行编码，以便在硬件中进行本机计算。 使用此格式的聚合和算术运算的SIMD矢量处理可以实现显着的性能提升。当INMEMORY_OPTIMIZED_ARITHMETIC设置为ENABLE时，该功能ENABLE 。

    请参阅“ 优化内存中算术 ” 。

  • 大型对象（LOB）的增强性能

    在以前的版本中，尽管在IM列存储中填充了LOB和LOB指针，但数据库通过使用缓冲区缓存来满足查询。在此版本中，将范围谓词应用于标量列或SQL运算符应用于LOB列的内存中查询受益于SIMD向量处理。

    IM列存储为IMCU内的LOB（小于4 KB的LOB）提供连续存储。对于外部LOB，IM列存储仅存储LOB定位符，即40个字节。前面的规则有一个例外。IMEU可以为定义为LOB数据类型的JSON列分配最多32 KB的连续存储。IMEU以OSON（二进制JSON）格式存储这些列。

    请参阅“ CPU架构：SIMD矢量处理 ” 。

  • 一列上的内存中连接组

    您可以使用以下语法为单个列上的自CREATE INMEMORY JOIN GROUP jg_name ( table_name ( column_name ))创建连接组： CREATE INMEMORY JOIN GROUP jg_name ( table_name ( column_name )) 。

    请参阅“ 使用加入组优化联接 ” 。

Oracle Database 12c第2版（12.2.0.1）中的更改

Oracle Database 12c第2版（12.2.0.1）的Oracle数据库 内存指南进行了以下更改。

  • 新功能
    以下主要功能是此版本中的新功能：

上级主题： Oracle Database In-Memory指南中此发行版的更改

新功能

以下主要功能是此版本中的新功能：

  • 内存中列存储（IM列存储）动态调整大小

    您现在可以动态增加内存区域的大小，而无需重新打开数据库。

    请参阅“动态增加IM列存储的大小 ” 。

  • 内存中表达式（IM表达式）

    Oracle数据库自动识别常用（“热”）表达式，这些表达式是IM列存储中填充的候选对象。候选表达式可能是(monthly_sales*12)/52 。IM表达式可以极大地提高使用计算密集型表达式和访问大型数据集的分析查询的性能。

    请参阅“ 使用内存中表达式优化查询 ” 。

  • 内存中虚拟列（IM虚拟列）

    IM虚拟列使IM列存储能够实现表中的部分或全部虚拟列。

    请参阅“ 为内存表启用和禁用列 ” 。

  • IM FastStart

    IM FastStart通过将IMCU直接存储在磁盘上来优化IM列存储中的数据库对象的数量。

    请参阅“ 为IM列存储管理IM FastStart ” 。

  • 对象级服务支持

    对于单个对象， INMEMORY ...DISTRIBUTE子句有一个FOR SERVICE子句，它将群体限制为可以运行此服务的数据库实例。例如，您可以将INMEMORY对象配置为仅在实例1上的IM列存储中填充，或仅在实例2上或在两个实例中填充。

    请参阅“ 对象级服务控件 ” 。

  • IM列存储在备用数据库上

    您可以在Oracle Active Data Guard备用数据库中启用IM列存储。您可以在主数据库和备用数据库的内存中列存储中填充完全不同的数据集，从而有效地使应用程序可用的内存中列存储的大小加倍。

    请参阅“ 使用Oracle Active Data Guard部署IM列存储 ” 。

  • ADO支持IM列存储

    您可以使用自动数据优化（ADO）策略根据热图统计信息从IM列存储中清除对象（如表，分区或子分区）。成功完成策略会导致为指定对象设置NO INMEMORY 。

    请参阅“ 为IM列存储启用ADO ” 。

  • 加入团体

    连接组是用户创建的对象，它列出了可以有意义地连接的两列。在某些查询中，连接组使数据库能够消除解压缩和散列列值的性能开销。加入组需要IM列存储。

    请参阅“ 使用加入组优化联接 ” 。
