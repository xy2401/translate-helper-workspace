5了解Oracle Net Architecture

Oracle Net侦听器是位于Oracle Net基础层之上的应用程序。数据库通过侦听器从客户端应用程序接收初始连接。

侦听器代理客户端请求，将请求切换到Oracle数据库服务器。每次客户端请求与数据库的网络会话时，侦听器都会收到初始请求。

5.1关于服务注册

侦听器通过服务注册确定数据库服务及其服务处理程序是否可用。在注册期间， 监听器注册（LREG）过程为监听器提供有关以下内容的信息：

  • 数据库提供的数据库服务的名称

  • 与服务关联的数据库实例的名称及其当前和最大负载

  • 可用于实例的服务处理程序（ 调度程序和专用服务器 ），包括其类型，协议地址以及当前和最大负载

前面的信息使监听器能够适当地指导客户端请求。

图5-2显示了两个数据库实例向两个侦听器注册信息。该图不代表可以注册的所有信息。例如，可以向监听器动态注册监听端点，例如端口号。

5.2关于监听器和连接请求

每个侦听器都配置有一个或多个指定其侦听端点的协议地址 。协议地址定义侦听器侦听的协议以及任何其他协议特定的信息。例如，侦听器可以配置为侦听以下协议地址：

（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST =销售服务器）（PORT = 1521）））

上面的示例显示了一个TCP / IP地址，它指定了侦听器（ sales-server ）的主机和端口号（1521）。

使用协议地址配置的客户端可以向侦听器发送连接请求。当客户端请求到达侦听器时，它会选择适当的服务处理程序来为请求提供服务，并将请求转发给处理程序。服务处理程序是调度程序或专用服务器进程，充当数据库的连接点。

图5-3说明了在建立连接期间侦听器的作用。该图显示了建立HTTP连接的浏览器和建立数据库连接的客户端。

  1. 浏览器或客户端向侦听器发送连接请求。

  2. 侦听器解析请求并将其转发给请求的数据库服务的服务处理程序。

  3. 浏览器或客户端连接到数据库。

5.3关于Oracle Restart

Oracle Restart增强了单实例环境中Oracle数据库的可用性。使用服务器控件（SRVCTL）实用程序，可以将组件（如侦听器）添加到Oracle Restart配置中。配置使侦听器能够在侦听器发生故障或未运行时自动启动。

使用Oracle Restart时，请注意以下事项：

  • 使用SRVCTL实用程序启动和停止侦听器。不要使用侦听器控件实用程序LSNRCTL。

  • 每个监听器必须具有唯一的名称。

5.4关于阻止的连接请求

如果在注册相应实例之前发生传入请求，或者数据库处于受限模式（例如数据库正在关闭时），则会发生阻止的连接请求。如果数据库实例处于受限模式，则LREG指示侦听器阻止与该实例的所有连接。尝试连接的客户端会收到以下错误之一：

  • ORA-12526：TNS：listener：所有适当的实例都处于受限模式

  • ORA-12527：TNS：listener：所有适当的实例都处于受限模式或阻止新连接

  • ORA-12528：TNS：listener：所有适当的实例都阻止了新的连接

当数据库实例尚未向侦听器注册时，会发生ORA-12528错误。

5.5了解数据库服务器进程架构

基于向侦听器注册的服务处理程序类型，侦听器将请求转发到共享服务器或专用服务器进程。共享服务器体系结构使数据库服务器允许许多客户端进程共享服务器进程。在一个专用在服务器配置中，侦听器为专用于为客户端提供服务的每个传入客户端连接请求启动单独的专用服务器进程。

5.5.1关于共享服务器进程

共享服务器进程在共享服务器体系结构中使用， 如图5-4所示 。使用共享服务器体系结构，客户端进程最终连接到调度程序。LREG进程使用侦听器注册调度程序的位置和负载，使侦听器能够将请求转发给负载最少的调度程序。该注册过程未在图中示出。

一个调度程序可以同时支持多个客户端连接。每个客户端连接都绑定到一个 虚电路 。虚拟电路是调度程序用于客户端数据库连接请求和回复的共享内存。当请求到达时，调度程序将虚拟电路放置在公共请求队列上。空闲的共享服务器从中获取虚拟电路请求队列，为请求提供服务，并在尝试从请求队列中检索另一个虚拟电路之前放弃虚拟电路。共享服务器将所有已完成的请求放入调度程序的响应队列中。每个调度员都有自己的调度员 SGA（系统全局区域）中的响应队列。这种方法使一小部分服务器进程能够为大量客户端提供服务。

5.5.2关于专用服务器进程

在专用服务器体系结构中，每个客户端进程都连接到专用服务器进程。任何其他客户端都不共享服务器进程。图5-5说明了专用服务器体系结构。

LREG使用侦听器注册有关专用服务器进程的信息。这使得侦听器能够在客户端请求到达时启动专用服务器进程并将请求转发给它。

注意：

专用服务器体系结构不支持HTTP，FTP或WebDAV客户端。仅支持数据库客户端。

5.6了解Oracle Connection Manager体系结构

Oracle Connection Manager是一个网关，客户端连接请求通过该网关发送到下一个跃点或直接发送到数据库服务器。通过Oracle Connection Manager中继连接请求的客户端可以利用Oracle Connection Manager上配置的会话多路复用和访问控制功能。在LREG进程注册其服务之前，它不携带任何服务信息。

Oracle Connection Manager由三个组件组成：

  • 倾听者

  • CMGW（Oracle连接管理器网关）

  • CMADMIN（Oracle连接管理器管理）

侦听器接收客户端连接并根据一组规则评估是否拒绝或允许访问。如果它允许访问，则侦听器将请求转发到网关进程，选择具有最少连接的进程。反过来，CMGW进程将请求转发到另一个Oracle Connection Manager或直接转发到数据库服务器，中继数据直到连接终止。如果已存在与服务器的连接，则网关通过现有连接多路复用或汇集其连接。CMADMIN监视网关进程和侦听器的状态，根据需要关闭或启动进程。此外，它还会向侦听器注册网关进程的位置和负载，并响应来自Oracle Connection Manager Control实用程序的请求。

在图5-6中 ，监听器屏幕连接请求。网关进程向CMADMIN进程注册 。，并且CMADMIN进程向侦听器注册。最后，侦听器将连接请求转发到网关进程。收到三个有效的客户端连接后，网关进程多路复用它们通过单一网络协议连接到数据库。根据规则集评估第四个连接时将被拒绝。
