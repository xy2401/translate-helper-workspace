14.1关于软件管理和补丁版本

软件管理涉及使用最新的产品修复程序使Oracle数据库软件保持最新。 发现产品缺陷或错误时 ，会创建一个修补程序来解决问题。补丁修正了已安装软件中的单个缺陷。单个补丁（也称为临时补丁）可供客户使用，这些客户出于业务原因不能等到下一个补丁集接收产品修复程序。

Oracle以补丁集的形式定期发布其软件的维护版本。补丁集是在维护版本发布之前一直发布的产品修订集合。补丁集是经过全面测试和集成的产品修复。补丁集中的所有产品修复程序均已经过测试，并且经过认证可以相互协作。

每个补丁或补丁集都与错误号相关联，以便识别。补丁集也与版本号相关联。例如，如果使用Oracle Database 12 c第1版（12.1.0.1），则可用的修补程序集为12.1.0.2。如果应用临时补丁，则修补软件的版本号不会更改。

每个补丁都有一个相关的README文件，描述了它如何修复软件。README文件还包含应用补丁的说明。

每个补丁集都附带一个Patch Set Notes文件，其中包含安装说明以及有关补丁集中包含的产品修订的信息。将修补程序集应用于Oracle软件时，可以更改已安装软件的维护版本号。应用修补程序集仅影响驻留在Oracle主目录中的软件，而不会更改数据库中的数据。

您可以使用Oracle OPatch和Oracle Universal Installer（OUI）分别应用修补程序和修补程序集。或者，您可以使用Oracle Enterprise Manager Cloud Control自动执行这些修补程序的应用程序。使用这些产品超出了本文档的范围。

14.3删除Oracle数据库软件

如果要删除Oracle软件安装，可以使用卸载工具从计算机中完全卸载该软件。

与Oracle数据库12c的开始，卸载工具集成了数据库的安装介质。您可以使用运行卸载工具runInstaller与命令-deinstall和-home从Oracle数据库，Oracle数据库客户端或Oracle网格基础架构的安装介质的基本目录的选项。

安装后，Oracle主目录中的卸载工具（ deinstall ）也可用作$ORACLE_HOME/deinstall/deinstall 。

运行卸载工具的默认方法是作为安装所有者从Oracle主目录中的deinstall目录：

要从计算机中删除Oracle主目录：

  1. 使用以下命令执行卸载工具：
    $ cd $ ORACLE_HOME $ $ ORACLE_HOME / deinstall / deinstall

    注意：

    在运行卸载工具之前，请不要关闭Oracle数据库或停止任何数据库进程。

  2. 根据提示输入指定的信息，或者只需按Enter键接受默认值。
  3. 出现提示时，请按照说明运行根脚本。您必须是root用户才能这样做。
