支持的SQL语句

除了少数例外，网关提供对Oracle DELETE ， INSERT ， SELECT和UPDATE语句的完全支持。

网关不支持Oracle 数据定义语言（DDL）语句。没有形式的Oracle ALTER ， CREATE ， DROP ， GRANT ，或可以使用TRUNCATE语句。相反，如果需要对Teradata数据库使用DDL语句，请使用网关的传递功能。

插入

选择

完全支持SELECT语句，但有以下例外：

  • CONNECT BY condition

  • NOWAIT

  • START WITH condition

  • WHERE CURRENT OF

UPDATE

完全支持UPDATE语句。但是，只能使用Teradata支持的Oracle功能。此外，您不能在子查询中拥有引用外部查询中相同表名的SQL语句。SET子句不支持子查询。

Oracle函数

在网关将Teradata数据库转换为Teradata SQL后，Teradata数据库将对所有函数进行评估。

Teradata不支持的功能

不支持在Teradata中没有等效功能的Oracle SQL函数 DELETE ， INSERT ，或 UPDATE语句，但如果语句是SELECT语句，则由Oracle数据库评估。也就是说，Oracle数据库执行发送到网关的SELECT语句的后处理。

如果在DELETE ， INSERT或UPDATE语句中使用了不受支持的函数，则会发生以下Oracle错误：

ORA-02070：数据库db_link_name在此上下文中不支持函数
