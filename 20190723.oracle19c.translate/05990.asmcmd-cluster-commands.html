ASMCMD成员群集管理命令

本主题提供与Oracle ASM远程实例一起使用的ASMCMD成员集群管理命令的摘要。

除非另有说明，否则ASMCMD集群管理命令应在Oracle ASM-CSS集群中执行。

有关Oracle Flex ASM的信息，请参阅“ 管理Oracle Flex ASM ” 。

表10-24列出了ASMCMD成员集群管理命令及其简要说明。

表10-24 ASMCMD成员集群管理命令摘要

命令 描述

CHCC

修改成员集群的配置。

mkcc

创建成员集群的配置。

喉癌

列出已配置的成员集群。

RMCC

删除成员集群的配置。

  • 有关安装群集和群集名称要求的信息，请参阅“Oracle Grid Infrastructure安装和升级指南”

  • 有关成员集群的信息，请参见“Oracle Clusterware管理和部署指南”

CHCC

目的

修改成员集群的配置。

语法和说明

chcc cluster_name [--direct | --indirect] [--version member_cluster_version ]

下表列出了chcc命令的选项。

表10-25 chcc命令的选项

选项 描述

CLUSTER_NAME

指定成员集群的名称。

- 直接

指定直接存储访问。

- 间接

指定间接存储访问。

--version member_cluster_version

如果它与存储服务器版本不同，则指定五位成员集群版本。如果未指定，则使用存储服务器版本。

要修改成员集群的存储访问方法，请执行以下步骤。

  1. 作为域服务群集（DSC）上的Oracle Grid Infrastructure用户，使用chcc命令更改成员群集存储访问配置。

  2. 作为域服务群集（DSC）上的Oracle Grid Infrastructure用户，使用mkcc命令将成员群集配置导出到新的群集清单文件中。

    您可以忽略发出mkcc命令时报告的警告消息。

  3. 导出新的集群清单文件后，作为成员集群节点上的root用户，将新集群清单文件复制到具有rootcrs.sh脚本的节点。

例子

以下示例显示如何从成员集群—–asm和––gimr组件的直接Oracle ASM存储修改存储访问方法。

示例10-29使用ASMCMD chcc命令

作为域服务群集（DSC）上的Oracle Grid Infrastructure用户，运行以下命令：

$ asmcmd chcc memberCluster1 --indirect $ asmcmd mkcc memberCluster1 /export/cluster_manifest_new.xml --asm --gimr

作为成员集群的root用户，使用以下命令将新集群清单文件复制到成员集群。

  1. 在成员集群中的每个节点上运行rootcrs.sh脚本。 

    ＃rootcrs.sh --convertcluster STORAGE_ACCESS -manifestfile cluster_manifest_new.xml

  2. 在成员集群中的最后一个节点上使用-lastnode选项运行rootcrs.sh脚本。 

    ＃rootcrs.sh --convertcluster STORAGE_ACCESS -manifestfile cluster_manifest_new.xml -lastnode

mkcc

目的

创建成员集群的配置。

语法和说明

mkcc cluster_name manifest_file [--direct] [--version member_cluster_version ] [--guid member_cluster_guid ] [--gimr gimr_pdb / export_dir ] [--asm] [--gimr] [--rhp] [--acfs] [ - ·TFA]

下表列出了mkcc命令的选项。

表10-27 mkcc命令的选项

选项 描述

CLUSTER_NAME

指定成员集群的名称。

manifest_file

指定要导出凭据的XML文件的完整路径。

- 直接

指定直接存储访问。如果未指定，则存储访问是间接的。

--version member_cluster_version

如果它与存储服务器版本不同，则指定五位成员集群版本。如果未指定，则使用存储服务器版本。

--guid member_cluster_guid

指定成员集群的32个字符，全局唯一标识符。

--gimr gimr_pdb/export_dir

指定将独立群集转换为成员群集时从中导入GIMR PDB的目录。

--asm

生成Oracle ASM成员集群的凭据和配置。

--gimr

生成Oracle GIMR成员集群的凭据和配置。

--rhp

生成Oracle RHP成员集群的凭据和配置。

--acfs

生成Oracle ACFS成员集群的凭据和配置。

--tfa

生成Oracle TFA成员集群的凭据和配置。

成员群集名称必须符合与群集名称相同的规则。该名称在整个企业中必须是唯一的，长度必须至少为一个字符且长度不超过15个字符，必须为字母数字，不能以数字开头，并且可能包含连字符（ - ）。不允许使用下划线字符（_）。

如果未指定任何成员集群组件选项，则缺省情况是配置所有组件并将其凭据存储在同一清单文件中。

asmcmd mkcc命令不生成GNS凭据。要将成员集群配置为GNS客户端，请以特权用户身份运行以下命令，以将GNS实例客户端数据配置导出到集群清单文件：

srvctl export gns -clientdata manifest_file -role CLIENT

例子

以下示例为成员集群—–asm和––gimr组件创建凭据，并将凭据导出到集群清单文件。

示例10-31使用ASMCMD mkcc命令

ASMCMD [+]> mkcc memberCluster1 /tmp/member_cluster1.xml --version 12.2.0.0.0 --asm --gimr
