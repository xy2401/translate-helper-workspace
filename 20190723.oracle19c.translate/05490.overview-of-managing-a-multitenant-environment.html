3.1关于配置和管理多租户环境

您可以使用Oracle Multitenant选项配置和管理多租户环境。

在配置和管理多租户环境之前，您必须满足某些先决条件。为此，您需要完成一些常见任务并使用一组工具来完成这些任务。

多租户架构使Oracle数据库可以作为多租户容器数据库（CDB）运行，该数据库包括零个，一个或多个客户创建的可插拔数据库（PDB）。PDB是模式，模式对象和非模式对象的可移植集合，在Oracle Net客户端中显示为非CDB。 Oracle Database 12c之前的所有Oracle数据库都是非CDB。

3.1.1普通用户和本地用户

普通用户是在根中以及在每个现有和将来的PDB中具有相同标识的用户。

普通用户可以登录到root用户以及已被授予CREATE SESSION权限的任何容器。普通用户可以执行的操作取决于授予普通用户的权限。某些管理任务（如创建PDB或拔出PDB）必须由普通用户执行。

CDB还支持本地用户。本地用户是仅存在于一个PDB中的用户。

3.1.2 CDB和PDB管理中的职责分离

一些数据库管理员管理整个CDB，而其他管理员管理单个PDB。

管理整个CDB的DBA作为普通用户连接到CDB，并管理整个CDB和根的属性，以及PDB的一些属性。例如，这些DBA可以创建，拔出，插入和丢弃PDB。他们还可以为root指定临时表空间和默认表空间，并且可以更改PDB的打开模式。

DBA还可以作为本地PDB管理员连接到特定PDB，然后在DBB上对非CDB执行的PDB上执行管理任务的子集。该子集包括PDB支持应用程序所需的任务。例如，任务可以包括PDB中的表空间和模式的管理，该PDB的存储参数的规范，改变当前PDB的开放模式，以及设置PDB级初始化参数。

3.2多租户环境的先决条件

多租户环境必须满足先决条件。

在创建和使用多租户环境之前，必须满足以下先决条件：

  • 安装Oracle Database 12c 。

    安装包括设置操作系统特有的各种环境变量，以及建立软件和数据库文件的目录结构。

  • 将数据库兼容级别设置为至少12.0.0 。

  • 有关数据库兼容级别的信息，请参阅Oracle数据库升级指南

3.3.1多租户环境的任务

多租户环境使您可以实现多个目标。您可以完成常规任务以配置和使用多租户环境。

这些目标在“ 多租户架构的好处 ”中有所描述。为此，您必须完成以下常规任务：

任务1规划多租户环境

创建和配置任何数据库都需要仔细规划。CDB需要特别考虑。例如，在计划CDB时，请考虑以下因素：

  • 将插入每个CDB的PDB数量

  • 支持计划的CDB所需的资源

  • 容器管理策略作为整个CDB上的聚合执行或在各个PDB上本地执行

  • 容器数据库拓扑，可以包含具有应用程序PDB的应用程序容器或具有PDB的CDB，或两者的组合

有关规划CDB的详细信息，请参阅“ 规划CDB创建 ” 。

任务2创建一个或多个CDB

完成必要的计划后，可以使用Database Configuration Assistant（DBCA）或CREATE DATABASE语句创建一个或多个CDB。在任何一种情况下，您都必须指定每个CDB的配置详细信息。

有关创建CDB的详细信息，请参阅“ 关于使用DBCA 创建CDB ”和“ 创建CDB ” 。

创建CDB后，它由root和PDB$SEED ， 如图3-1所示。CDB根仅包含Oracle维护的对象和数据结构， PDB$SEED是用于克隆目的的通用种子数据库。

任务4创建，插入和拔出PDB

PDB包含用户数据。创建CDB后，您可以创建PDB，将未插入的PDB插入其中，并在必要时从中拔出PDB。您可以从CDB拔出PDB并将此PDB插入另一个CDB。例如，如果要将PDB的工作负载从一个服务器移动到另一个服务器，则可以将PDB从一个CDB移动到另一个CDB。

有关创建PDB，插入PDB和拔出PDB的信息，请参阅“ 创建和删除PDB和应用程序容器 ” 。

任务5管理和监控CDB和应用程序容器

管理和监控CDB涉及管理整个CDB，根和PDB的一些属性。CDB和非CDB的一些管理任务是相同的，有些是不同的。

管理和监视应用程序容器类似于管理和监视CDB，但是您的操作仅影响应用程序根目录和作为应用程序容器一部分的应用程序PDB。

有关相似任务和不同任务的说明，请参阅“ 创建CDB后 ” 。另请参阅“ 管理CDB ”和“ 监控CDB和PDB ” 。

您可以使用Oracle Resource Manager在CDB中托管的PDB之间分配和管理资源，您可以使用它来分配和管理PDB中用户进程之间的资源使用。请参阅“ 使用Oracle Resource Manager for PDB ” 。

您还可以使用Oracle Scheduler在CDB和单个PDB中计划作业。请参阅“ 将Oracle Scheduler与CDB一起使用 ” 。

任务6管理和监控PDB和应用程序PDB

管理和监控PDB或应用程序PDB类似于管理和监控非CDB，但存在一些差异。请参阅“ 管理PDB ”和“ 监控CDB和PDB ” 。

在SQL * Plus

SQL * Plus是一个命令行工具，使您可以创建，管理和监视CDB和PDB。您可以使用SQL语句和Oracle提供的PL / SQL包在SQL * Plus中完成这些任务。

Oracle数据库配置助手（DBCA）

DBCA是一个具有图形用户界面的实用程序，可用于创建和复制CDB。它还使您能够创建，重定位，克隆，插入和拔出PDB。

Oracle企业管理器云控制

Cloud Control是一个系统管理工具，具有图形用户界面，使您可以管理和监控CDB及其PDB。

Cloud Control在线帮助

Oracle SQL Developer

Oracle SQL Developer是一个带有图形用户界面的客户端应用程序，使您可以配置CDB，创建PDB，插入和拔出PDB，修改PDB的状态，将PDB克隆到Oracle Cloud，热克隆/刷新PDB，在应用程序根目录之间重新定位PDB等等。

此外，Oracle SQL Developer还具有图形界面，用于资源管理，存储，安全性，配置以及CDB中容器和可插拔数据库的性能指标报告。
