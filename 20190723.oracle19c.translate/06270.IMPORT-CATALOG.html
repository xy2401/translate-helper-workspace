2.26进口目录

目的

使用IMPORT CATALOG命令将元数据从一个恢复目录模式导入到不同的目录模式中。如果您创建了不同版本的目录模式以存储多个目标数据库的元数据，则此命令使您可以为所有数据库维护单个目录模式。

创建目录

先决条件

必须将RMAN连接到目标恢复目录，该目录是要将目录数据导入其中的目录。此恢复目录不能是虚拟专用目录。

合并目录模式不需要目标数据库连接。在RMAN提示符下执行此命令。

源恢复目录模式的版本必须等于目标恢复目录模式的当前版本。如果它们不相等，则将模式升级到相同版本。

确保未在源恢复目录架构和目标目录架构中注册相同的数据库。如果数据库在两种模式中注册，然后UNREGISTER从源恢复目录这个数据库，并执行IMPORT再次命令。

使用说明

如果操作在导入过程中失败，则回滚导入。因此，不允许部分导入。取消注册操作与导入是分开的。默认情况下，导入成功后，将从源恢复目录模式中注销导入的数据库ID。

存储的脚本是全局脚本或本地脚本。由于目标模式包含具有相同名称的对象，因此导入期间全局脚本（而非本地脚本）可能会发生名称冲突。在这种情况下，RMAN将全局脚本名称重命名为COPY OF script_name 。例如，RMAN将bp_cmd重命名为COPY OF bp_cmd 。

如果重命名的全局脚本仍然不唯一，则RMAN将其重命名为COPY(2) OF script_name 。如果此脚本名称也存在，则RMAN将脚本重命名为COPY(3) OF script_name 。RMAN继续COPY( n ) OF模式，直到脚本具有唯一名称。

句法

import :: =


import.eps插图的说明

（ connectStringSpec :: = ）

语义

语法元素 描述

connectStringSpec

指定源恢复目录的连接字符串，该目录是导入其元数据的目录。

DBID integer

指定从源目录模式导入元数据的数据库的DBID列表（请参见示例2-107 ）。

如果未指定，RMAN会将源目录架构中的所有数据库ID的元数据合并到目标目录架构中。如果合并元数据的数据库已在恢复目录架构中注册，则RMAN会发出错误。

DB_NAME database_name

指定从源目录模式导入元数据的数据库列表（请参见示例2-107 ）。

如果未指定，RMAN会将源目录模式中的所有数据库的元数据合并到目标目录模式中。如果在两个恢复目录中都注册了相同的DBID，则RMAN会发出错误。

没有注册

强制RMAN将导入的数据库ID保留在源目录模式中。默认情况下，从源恢复目录架构取消注册导入的数据库ID。

例子

示例2-106导入所有已注册数据库的元数据

在此示例中，database prod包含用户rcat拥有的10.2目录模式，而database catdb包含用户rco拥有的11.1目录模式。RMAN将在rcat注册的所有数据库ID的元数据rcat到rco拥有的恢复目录中。在rcat中注册的所有目标数据库都是未注册的。 

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> IMPORT CATALOG rcat @ prod;在15-FEB-13源恢复目录数据库中启动导入目录密码：连接到源恢复目录的密码数据库导入验证从源恢复目录中取消注册的完整数据库完成15-FEB-13的导入目录

示例2-107导入已注册数据库子集的元数据

此示例是示例2-106的变体。它不会导入整个恢复目录，而只导入DBID为1618984270的数据库的元数据。 

RMAN> CONNECT CATALOG rco @ catdb恢复目录数据库密码：连接到恢复目录数据库的密码 RMAN> IMPORT CATALOG rcat @ inst1 DBID = 1618984270;在15-FEB-13源恢复目录数据库中启动导入目录密码：连接到源恢复目录的密码数据库导入验证从源恢复目录中取消注册的完整数据库完成15-FEB-13的导入目录
