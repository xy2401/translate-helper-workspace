3使用DBCA创建Oracle RAC或Oracle RAC One节点数据库

在独立模式下使用Oracle数据库配置助手（DBCA）来创建和删除Oracle Real Application Clusters（Oracle RAC）数据库。

  • 关于Database Configuration Assistant
    了解Oracle数据库配置助手（DBCA）为创建Oracle RAC数据库提供的功能。
  • 选择Oracle RAC的安装选项
    查看主题以选择安装Oracle RAC的选项。
  • 早期版本的自动侦听器迁移
    查看此信息，以便从早期数据库版本迁移侦听器。
  • 验证DBCA的要求
    在使用DBCA创建Oracle RAC数据库之前，请验证您的系统是否已准备好进行配置更改。
  • 在使用DBCA创建任何Oracle RAC数据库之前要完成的任务
    在使用DBCA创建Oracle RAC数据库之前，如果在Oracle Grid Infrastructure安装期间未完成系统配置，则必须配置系统以满足软件要求。
  • 选择DBCA选项以创建Oracle RAC或Oracle RAC One节点数据库
    查看此信息以使用Database Configuration Assistant（DBCA）创建Oracle RAC或Oracle RAC One节点数据库。
  • 使用DBCA创建Oracle RAC One节点数据库
    如果已选择仅在群集节点上安装Oracle RAC软件，则可以使用Oracle数据库配置助手（DBCA）配置Oracle RAC One节点。
  • 安装Oracle Database Vault选件
    安装和配置Oracle Database Vault需要在安装期间和安装后执行操作。
  • 使用DBCA删除Oracle RAC数据库
    使用DBCA删除Oracle RAC数据库涉及首先删除数据库，然后删除数据库的初始化参数文件，实例，最佳灵活体系结构（OFA）结构以及数据库的Oracle网络配置。
