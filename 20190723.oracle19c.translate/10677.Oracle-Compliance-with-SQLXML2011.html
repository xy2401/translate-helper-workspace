Oracle遵从SQL / XML

标准中的XML数据类型是XML 。 Oracle等效数据类型是XMLType 。如果Oracle和标准之间的唯一区别是数据类型名称的拼写，则认为标准的功能完全受支持。

表C-3描述了Oracle对SQL / XML功能的支持。

表C-3 Oracle对SQL / XML功能的支持

X010，XML类型

Oracle完全支持此功能。

X011，XML类型的数组

Oracle使用命名数组类型支持此功能

X012，XML类型的多字节

Oracle类似的多类XML类型是一个嵌套的表，其中包含一个XML类型的列。

X013，不同类型的XML

X014，XML类型的属性

X015，XML类型的字段

  • Oracle不支持<XML传递机制>;行为与标准中的BY VALUE相同。

X080，XML发布中的命名空间

在XMLElement的Oracle实现中， XMLAttributes用于定义名称空间（未实现XMLNamespaces ）。然而， XMLAttributes不支持XMLForest 。

X086， XMLTable XML名称空间声明

Oracle完全支持此功能。

X090，XML文档谓词

在Oracle中，您可以使用ISFRAGMENT方法测试XML值是否为文档。

X096， XMLExists

Oracle完全支持此功能，但有一个例外：Oracle仅支持按值传递，因此关键字BY VALUE在PASSING子句的开头是可选的，并且不支持单个参数。

X120，SQL例程中的XML参数

Oracle完全支持此功能。

X121，外部例程中的XML参数

Oracle支持使用非标准接口传递给外部例程的XML值。

X141， IS VALID谓词：数据驱动器案例

XMLISVALID方法等同于IS VALID谓词，并支持数据驱动的情况。

X142， IS VALID谓词： ACCORDING TO子句

XMLISVALID方法等同于IS VALID谓词，并包含ACCORDING TO子句的等效项。

X143， IS VALID谓词： ELEMENT子句

XMLISVALID方法等同于IS VALID谓词，并包含ELEMENT子句的等效项。

X144， IS VALID谓词：架构位置

XMLISVALID方法等同于IS VALID谓词，并支持已注册XML模式的模式位置规范。

X145， IS VALID谓词超出检查约束

XMLISVALID方法等同于IS VALID谓词，可以在检查约束之外使用。

X151， IS VALID谓词，带有DOCUMENT选项

XMLISVALID方法等效于IS VALID谓词，并执行与DOCUMENT子句等效的验证。（ XMLISVALID不支持“内容”验证。）

X156， IS VALID谓词：带有ELEMENT子句的可选NAMESPACE

XMLISVALID方法等同于IS VALID谓词，可用于验证任何命名空间中的元素。

X157， IS VALID谓词： NO带有ELEMENT子句的NAMESPACE

XMLISVALID方法等同于IS VALID谓词，可用于验证“无名称”命名空间中的元素。

X160，已注册XML模式的基本信息模式

Oracle静态数据字典视图ALL_XML_SCHEMAS提供了当前用户可访问的已注册XML模式的列表。ALL_XML_SCHEMAS 。 SCHEMA_URL列对应于标准XML_SCHEMAS 。 XML_SCHEMA_LOCATION列。可以通过检查ALL_XML_SCHEMAS来学习已注册的XML模式的目标名称空间。 SCHEMA 。 Oracle没有标准XML_SCHEMAS的其他列的等价物。

X161，已注册XML模式的高级信息模式

Oracle没有与标准中的XML_SCHEMA_NAMESPACES和XML_SCHEMA_ELEMENTS相对应的静态数据字典视图。但是，可以通过检查ALL_XML_SCHEMAS的实际XML模式来学习有关已注册XML模式的所有信息。 SCHEMA专栏。还可以检查这一点，以了解已注册的XML模式是否是不确定的，以及它的哪些名称空间和元素是不确定的。

X191， XML （ DOCUMENT （ XMLSCHEMA ））类型

Oracle不支持此语法。但是，表的列可以由注册的XML模式约束，在这种情况下，列的所有值都将是XML （ DOCUMENT （ XMLSCHEMA ））类型。

Oracle仅支持按值传递，因此关键字BY VALUE在PASSING子句的开头是可选的，并且不支持单个参数。

X221，XML传递机制BY VALUE

Oracle支持XMLQuery ， XMLTable和XMLExists的BY VALUE子句。在这些中， BY VALUE在参数列表的开头支持作为可选语法，但不作为单个参数或列的修饰符支持。

X232， XML （ CONTENT （ ANY ））类型

Oracle不支持将此语法用作类型修饰符，但Oracle XMLType支持此数据类型以用于瞬态值。持久值的类型为XML （ DOCUMENT （ ANY ）），它是XML一个子集（ CONTENT （ ANY ））。

X241， RETURNING CONTENT的XML发布

Oracle不支持此语法。在Oracle中，的发布功能（行为XMLAgg ， XMLComment ， XMLConcat ， XMLElement ， XMLForest和XMLPi ）总是RETURNING CONTENT 。

X251，持久性XML值XML （ DOCUMENT （ UNTYPED ））型

X252， XML类型的持久值（ DOCUMENT （ ANY ））

Oracle完全支持此功能。

X256， XML持久值（ DOCUMENT （ XMLSCHEMA ））类型

Oracle完全支持此功能。

X260，XML类型， ELEMENT子句

Oracle不支持此语法。但是，表的列可能受已注册的XML架构中的顶级元素约束。

X263，XML类型： NO带有ELEMENT子句的NAMESPACE

Oracle不支持此语法。但是，表的列可能受已注册的XML模式的“无名称”命名空间中的顶级元素约束。

X264，XML类型：架构位置

Oracle不支持此语法。但是，表的列可能受到由架构位置标识的已注册XML架构的约束。

X271，XMLValidate：数据驱动案例

SCHEMAVALIDATE方法等同于XMLValidate，并支持数据驱动的情况。

X272，XMLVALIDATE： ACCORDING TO条款

SCHEMAVALIDATE方法等同于XMLValidate，可用于指定特定的已注册XML模式。

X273，XMLValidate： ELEMENT子句

SCHEMAVALIDATE方法等同于XMLValidate，可用于指定特定已注册XML模式的特定元素。

X274，XMLValidate：架构位置

SCHEMAVALIDATE方法等效于XMLValidate，可用于通过其架构位置URL指定特定的已注册XML架构。

X281，带有DOCUMENT选项的XMLValidate

SCHEMAVALIDATE方法等同于XMLValidate。SCHEMAVALIDATE仅对XML文档（而非内容）执行验证。

X286，XMLValidate： NO带有ELEMENT子句的NAMESPACE

SCHEMAVALIDATE方法等效于XMLValidate，可用于指定特定已注册XML模式的“无名称”命名空间中的特定元素。
