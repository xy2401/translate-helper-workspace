2.1规划安装

在安装任何软件之前，请查看Linux的这些硬件，网络，操作系统和其他软件要求。

  • 每个GDS池数据库必须能够（在两个方向上）到达每个全局服务管理器的Listener和ONS端口。还必须向应用程序/客户端层，所有GDS池数据库，GDS目录和所有其他全局服务管理器打开全局服务管理器侦听器端口和ONS端口。

  • 必须向全局服务管理器和GDS目录打开（在两个方向上）每个GDS池数据库的TNS侦听器端口（默认值：1521）。

  • 如果从单独的计算机运行GDSCTL，则还必须从该计算机直接打开到所添加的每个GDS池数据库的端口（在两个方向上）。

全球数据服务组件需要：

  • 至少256MB的总物理内存

  • 至少20MB的可用物理内存

  • 至少6 GB的总交换空间

  • 至少1.5 GB的可用磁盘空间

  • 认证架构，例如x86_64

  • Linux系统内核版本至少是“2.6.18”

  • 程序包make-3.81可在系统上使用

  • 程序包binutils-2.17.50.0.6可在系统上使用

  • 软件包gcc-4.1.2 (x86_64)可在系统上使用

  • 程序包libaio-0.3.106 (x86_64)可在系统上使用

  • 程序包libaio-devel-0.3.106 (x86_64)可在系统上使用

  • 程序包libstdc++-4.1.2 (x86_64)在系统上可用

  • 软件包sysstat-7.0.2在系统上可用

  • 包compat-libstdc++-33-3.2.3 (x86_64)在系统上可用

  • 程序包libgcc-4.1.2 (x86_64)在系统上可用

2.2关于安装全局服务管理器需要了解的内容

全局服务管理器是全局数据服务框架的核心组件，您必须使用单独的介质安装全局服务管理器。安装和运行全局服务管理器不需要其他Oracle软件。

您可以在安装了其他Oracle产品的系统上安装全局服务管理器，但必须在单独的Oracle主目录中安装全局服务管理器。您可以在单个系统上安装多个全局服务管理器，但每个全局服务管理器必须具有单独的Oracle主目录。出于性能原因，您可能希望在专用主机上部署全局服务管理器，具体取决于全局数据服务配置中的数据库数量。

您必须为每个全局数据服务区域安装至少一个全局服务管理器。全局服务经理可以托管在物理或虚拟环境中。为了实现高可用性，Oracle建议在不同主机上运行的每个区域中安装多个（通常为3个）全局服务管理器。

注意：

Oracle Universal Installer当前不支持在多个主机上安装软件。您必须在其各自的主机上安装每个全局服务管理器。

Global Data Services管理员安装全局服务管理器。全球数据服务管理员的职责包括：

  • 管理全球服务经理

  • 管理全局数据服务目录

  • 管理区域和数据库池

注意：

全局数据服务管理员必须在部署了全局服务管理器的所有主机上拥有操作系统用户帐户，并且必须在该用户帐户下运行安装。root用户不能运行安装。

注意：

在Windows平台上安装全局服务管理器时，Oracle Universal Installer为您提供了使用Windows内置帐户或将标准Windows用户帐户指定为Oracle主目录的Oracle主用户的选项。此帐户用于运行Oracle主目录的Windows服务，它不能是管理员帐户。

请注意，如果选择“Windows内置帐户”选项，则使用LocalService帐户运行服务。对于管理任务（包括全局服务管理器安装，升级和修补），您应该使用具有管理权限的Windows帐户登录。

有关使用Oracle家庭用户的详细信息和建议，请参阅“ Oracle数据库管理员参考”中的第3章“Microsoft Windows ”。

2.3安装全局服务管理器

要安装全局服务管理器：

  1. 从软件介质的根目录启动Oracle Universal Installer，然后按照提示操作。

    安装完成后，全局服务管理器主目录包含运行全局服务管理器和全局服务管理器控制实用程序（GDSCTL）所需的二进制文件。

  2. 将ORACLE_HOME环境变量设置为您在安装期间指定的目录。
  3. 将为全局服务管理器创建的$ORACLE_HOME/bin目录添加到PATH环境变量中。
  4. 将TNS_ADMIN环境变量设置为$ORACLE_HOME/network/admin 。
  • 执行全局服务管理器的无提示安装
    您可以在命令行中运行全局服务管理器安装。

2.3.1执行全局服务管理器的无提示安装

您可以在命令行中运行全局服务管理器安装。

  1. 从OTN或edelivery.oracle.com下载全球服务管理器软件并解压缩。
    解压缩linuxx6412201_gsm.zip之后，应该有一个gsm / response目录，其中有一个名为gsm_install.rsp的响应文件。

2.4升级全球数据服务需要了解的内容

有四个组件构成分布式全局数据服务基础结构，每个组件可以驻留在单独的安装中，并且可以使用通常的升级过程独立升级;但是，应该遵循一些关于组件版本控制的规则。组件和规则如下：

  • 目录数据库：目录数据库是GDS元数据的中央存储库;它是标准的Oracle数据库安装。目录数据库的版本必须始终大于或等于连接到它的任何GDSCTL会话的版本，并且与连接到它的任何全局服务管理器服务器的版本完全相同，但有一个例外：为了简化迁移，目录可能暂时具有比连接到它的某些全局服务管理器服务器更大的版本，直到对目录进行任何更改，此时任何不同版本的已连接全局服务管理器将断开连接，并且停止不同版本的全局服务管理器将不允许连接。

    注意：

    运行版本12.1.0.1的GDSCTL会话无法更改以后的版本化目录;当运行将更新目录的命令时，GDSCTL客户端的最低版本应为12.1.0.2。

  • GDSCTL客户端：此组件以任意方式从包含全局服务管理器安装的任何系统上的终端会话运行。GDSCTL客户端的版本是它运行的全局服务管理器安装的版本。

    注意：

    将12.1.0.1 GDSCTL客户端连接到更高版本的目录时，只允许一组有限的命令，并且任何可能导致目录更改的命令都将导致兼容性错误。在版本12.1.0.2或更高版本的目录中更新目录元数据的命令应该从至少运行版本12.1.0.2的GDSCTL客户端执行。

  • 全局服务管理器服务器：全局服务管理器服务器的版本是服务器运行的全局服务管理器安装的版本。全局服务管理器服务器无法连接到版本低于其自身的任何目录数据库。全局服务管理器服务器无法连接到版本高于其自身的任何目录数据库，其中已对该较高版本的目录进行了更改。全局服务管理器可以连接到运行任何版本的Oracle Database 12c或更高版本的池数据库。

  • 池数据库：池数据库是添加到运行全局服务的GDS池的任何数据库。池数据库可以是Oracle 12 c或更高版本的任何版本，包括目录版本之后的版本。您可以随时升级或降级池数据库。

根据这些规则，可以在零服务停机时间内执行分布式GDS基础架构的滚动升级。

2.5升级全球数据服务

建议的升级顺序是：

  1. 升级目录数据库。为获得最佳结果，应使用滚动数据库升级来完成;但是，如果目录不可用，则在升级期间全局服务将保持可用，但不会发生服务故障转移。

    注意：

    使用12.1.0.2修补程序版本升级Oracle Database 12 c第1版时，必须在升级目录数据库后执行以下命令：

    修改目录-newkeys

    此命令生成加密密钥并加密存储在GDS目录中的现有GSMUSR密码。

  2. 升级用于运行GDSCTL客户端的全局服务管理器安装，这些客户端也不运行全局服务管理器服务器（如果有）。

    注意：

    全球服务经理升级应该就地完成;但是，就地升级将导致错误的错误消息，除非以下平台的以下文件的权限更新为755：

    在Linux / Solaris64 / Solaris Sparc64上：

    $ ORACLE_HOME / QOpatch / qopiprep.bat

    $ ORACLE_HOME / JDK /斌/ jcontrol

    $ ORACLE_HOME / JDK / JRE /斌/ jcontrol

    在AIX上：

    $ ORACLE_HOME / QOpatch / qopiprep.bat

    $ ORACLE_HOME / JDK / JRE /斌/经典/ libjvm.a

    $ ORACLE_HOME / JDK /斌/用policytool

    关于HPI：

    $ ORACLE_HOME / JDK / JRE / lib目录/ IA64N /服务器/ Xusage.txt

    $ ORACLE_HOME / JDK / JRE /斌/ jcontrol

    $ ORACLE_HOME / QOpatch / qopiprep.bat

    在Windows上，不需要任何错误消息。

  3. 一次一个地停止，升级和重新启动所有全局服务管理器服务器。为了确保零停机时间，应始终运行至少一个全局服务管理器服务器。比目录更早版本的全局服务管理器服务器将继续完全运行，直到进行目录更改。
  4. 在池数据库管理员的决定下，可以在全局服务管理器和目录库升级之前或之后以任何顺序升级池数据库。

2.6.1操作说明

注意：

除非指定，否则GDSCTL使用当前名称解析方法（例如TNSNAMES）解析连接字符串。全局服务管理器名称例外。GDSCTL查询gsm.ora文件以解析全局服务管理器名称。

要启动GDSCTL，请在操作系统提示符处输入以下命令：

$ gdsctl

上述命令启动GDSCTL并显示GDSCTL命令提示符。您可以在操作系统提示符或GDSCTL命令提示符下输入GDSCTL命令，如以下示例所示：

$ gdsctl add gsm -gsm gsm1 -catalog 127.0.0.1:1521:db1 GDSCTL> add gsm -gsm gsm1 -catalog 127.0.0.1:1521:db1

前面的两个命令都实现了相同的结果。第一个命令在操作系统命令提示符下运行，而第二个命令在GDSCTL命令提示符下运行。本文档中的命令语法示例使用GDSCTL命令提示符。

注意：

  • 许多GDSCTL命令要求您在运行命令之前首先连接到Global Data Services目录。

    如果从GDSCTL提示符运行命令，则必须在需要连接到全局数据服务目录的第一个GDSCTL命令之前执行connect命令。connect命令只需要在GDSCTL会话中运行一次。

  • 将数据库或代理配置添加到GDS配置时，可以指定网络服务名称而不是连接描述符。如果指定了网络服务名称，则必须在GDS配置中的每个全局服务管理器处将其解析为允许连接到正在添加的实体的连接描述符。

或者，您可以将所有GDSCTL命令收集到一个文件中，并使用GDSCTL作为批处理运行它们，如下所示：

$ gdsctl @ script_file_name

上述命令启动GDSCTL并运行指定脚本文件中包含的命令。

使用GDSCTL帮助

您可以显示GDSCTL的帮助，如下所示：

2.6.2 GDSCTL命令语法和对象

GDSCTL命令语法和选项

GDSCTL命令，对象和选项;数据库名称，实例名称，全局数据服务区域名称，全局数据服务池名称和服务名称都不区分大小写。密码和服务器池名称也区分大小写。GDSCTL使用以下命令语法：

$ gdsctl 命令 [ object ] [ options ] [ argument ]或GDSCTL> command [ object ] [ options ] [ argument ]

在GDSCTL语法中：

  • command ： add ， start ， stop或remove等动词

  • object （也称为名词）：GDSCTL执行命令的目标或对象，例如service或database 。您可以在表2-1中找到对象列表。

  • options ：可选标志，用于扩展前面命令组合的使用，以包含该命令的其他参数。例如， -gdspool选项表示-gdspool是特定全局数据服务池的名称。如果以逗号分隔的列表跟随选项，则不要在列表中的项之间使用空格。

  • argument ：GDSCTL命令的附加变量，用于指定对象的操作，或指定没有对象的GDSCTL的操作。

2.6.3 GDSCTL连接

对于某些操作，GDSCTL必须连接到全局服务管理器。要连接到全局服务管理器，GDSCTL必须与全局服务管理器在同一主机上运行。连接到全局服务管理器时，GDSCTL会查找与本地全局服务管理器关联的gsm.ora文件。

以下是需要连接到全局服务管理器的GDSCTL操作。

  • add gsm添加了一个全局服务管理器。

  • 启动gsm启动全局服务管理器。

  • stop gsm停止全局服务管理器。

  • modify gsm修改全局服务管理器的配置参数。

  • status gsm返回全局服务管理器的状态。

  • set inbound_connect_level设置INBOUND_CONNECT_LEVEL侦听器参数。

  • set trace_level设置与指定的全局服务管理器关联的侦听器的跟踪级别。

  • set outbound_connect_level设置与特定全局服务管理器关联的侦听器的出站连接的超时值。

  • set log_level设置与特定全局服务管理器关联的侦听器的日志级别。

对于所有其他操作，GDSCTL使用Oracle Net Services连接到全局数据服务目录数据库或全局数据服务配置中的另一个数据库。对于这些连接，您可以从具有必要网络配置的任何客户端或主机运行GDSCTL。

2.7关于创建全球数据服务目录需要了解的内容

每个全局数据服务配置都必须具有全局数据服务目录。全局数据服务目录可以与GDS配置数据库驻留在同一主机上，但Oracle不建议将此方案用于大型配置。Oracle建议您使用Oracle高可用性功能（如Oracle Real Application Clusters（Oracle RAC）和Oracle Data Guard）来保护全局数据服务目录免受中断。

全球数据服务目录要求

  • 全局数据服务目录必须驻留在使用服务器参数文件（SPFILE）的Oracle Database 12 c （或更高版本）数据库中。

2.16向全局数据服务池添加服务

gdsctl add service命令用于在全局数据服务池数据库上创建服务。该命令的一个简单示例如下：

GDSCTL>添加服务-gdspool hr -service emp_report1 -preferred_all

在此示例中， emp_report1是服务名称， -preferred_all选项表示服务通常应该在池中的所有数据库上运行。

“添加服务”命令中指定的服务名称可以是域限定的（例如，sales.example.com），也可以不是（例如，sales）。如果指定的名称不是域限定的，则使用默认域名“<GDS_pool_name>。<GDS_configuration_name>”创建服务，但较短的非域限定名称可与后续GDSCTL命令一起用于管理服务。如果指定的名称是域限定的，则必须在用于管理服务的所有后续GDSCTL命令中使用完全域限定的服务名称。

对于启用了Oracle RAC的池数据库，在添加服务后，运行GDSCTL modify service以指定应在哪个Oracle RAC实例上运行给定的全局服务，如以下示例所示。 

GDSCTL>修改服务-service emp_report1 -gdspool hr  - 数据库db14 -modify_instances -preferred db14_n1，db14_n2

全局服务名称在GDS池中必须是唯一的，并且在通过域限定时，在GDS配置中也必须是唯一的。如果该数据库中已存在具有相同名称的本地或全局服务，则无法在数据库中创建全局服务。

全局服务名称可以包含字母数字字符“_”和“。”。第一个字符必须是字母数字。全局服务名称的最大长度为64个字符。域限定全局服务名称的最大长度为250个字符。

用于连接到全局服务的Oracle Net连接描述符必须包含域限定的服务名称

2.18数据库客户端配置

数据库客户端使用Oracle Net连接字符串连接到数据库服务。用于全局服务的连接字符串与用于本地服务的连接字符串的区别在于以下方式：

  • 连接数据部分中的服务名称参数指定全局服务

  • 指定了多个连接端点，这些端点是全局服务管理器，而不是本地，远程或单个客户端访问名称（SCAN）侦听器

  • 可以在连接数据部分中指定数据库客户端的区域

考虑以下连接字符串：

（DESCRIPTION =（FAILOVER = on）（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（host = sales-east1）（port = 1522））（ADDRESS =（host = sales-east2）（port = 1522））（ADDRESS =（host = sales-east3）（port = 1522）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（host = sales-west1）（port = 1522））（ADDRESS =（host = sales-west2）（ port = 1522））（ADDRESS =（host = sales-west3）（port = 1522）））（CONNECT_DATA =（SERVICE_NAME = sales）（REGION = east）））

此连接字符串包含客户端本地区域（东部）中的三个全局服务管理器（sales-east1，sales-east2和sales-east3），以及三个全局服务管理器（sales-west1，sales-west2和sales-west3）在客户的好友区域（西）。

通过在每个区域的地址列表LOAD_BALANCE参数设置为ON，可以在每个区域内的全局服务管理器上启用客户端负载平衡。通过将FAILOVER参数设置为ON，可以启用区域之间的连接时故障转移。

最佳做法是在每个区域中拥有三个全局服务管理器，每个区域都有一个伙伴区域，并且要配置客户端负载平衡和连接时故障转移，如示例连接字符串所示。

如果在连接字符串中仅指定了本地区域的全局服务管理器，则REGION参数是可选的。如果GDS配置中只有一个区域，或者存在多个区域时可能就是这种情况，但是更改设计为使用单个数据库的现有客户端的连接字符串是不可行的。如果未指定REGION参数，则假定客户端的区域是用于连接到全局服务的全局服务管理器的区域。

注意：

除非在连接字符串中指定了region参数，否则12c之前的Thin JDBC客户端只能与具有单个区域的GDS配置一起使用。

Oracle Database 12c及更高版本的集成客户端使用Oracle Notification Services（ONS）接收支持负载平衡和快速连接故障转移（FCF）的快速应用程序通知（FAN）事件。集成客户端无需用户配置即可自动订阅最多3个ONS服务器，这些服务器与客户端本地和伙伴区域中的每个全局服务管理器位于同一位置。

注意：

如果必须使用SSL保护与ONS服务器的连接，则不支持自动ONS配置。您必须手动配置ONS才能启用SSL。有关如何手动配置ONS的信息，请参阅特定于客户端的指南。

12c之前的客户注意事项：

12c之前的OCI和ODP.NET客户端不支持全局服务。

12c之前的JDBC客户端支持全局服务，但您必须手动配置它以订阅与客户端本地和伙伴区域中的全局服务管理器位于同一位置的ONS服务器。最佳做法是在每个客户端的本地和好友区域中订阅三个ONS服务器。

有关如何配置ONS订阅的信息，请参阅Oracle数据库JDBC开发人员指南 。

2.19关于导出逻辑备份的GDS目录数据需要了解的内容

由于GDS目录存储整个GDS配置的元数据，因此目录数据的丢失或损坏可能需要大量工作才能手动重新创建它。虽然GDS目录的不可用性不会影响核心GDS功能，例如负载平衡，服务故障转移和应用程序通知，但在恢复目录之前，不能对GDS配置进行任何更改。因此，制定保护GDS目录的策略非常重要。

即使GDS目录受到Oracle RAC和Data Guard等HA技术的保护，强烈建议您定期备份GDS目录。您可以通过将目录数据导出到文件来创建目录的逻辑备份。GDS目录的此备份可以帮助您进入灾难恢复方案，并且需要撤消自上次备份以来对目录所做的更改。将GDS目录移动到新数据库时，也可以使用备份。

目录配置将保存到运行GDSCTL的系统上的指定文件中。对文件的访问应限于全局数据服务管理员，因为它可能包含敏感信息，例如池数据库的连接字符串。

注意：

  • 强烈建议在导出目录之前验证目录，以确保目录数据中不存在任何不一致。在导出目录数据之前，应更正validate catalog命令报告的任何错误。

  • 您不得使用导出的目录数据对文件进行任何更改。对文件的任何更改都可能会阻止将此文件用于目录库还原，或者可能导致还原后目录损坏。建议将文件校验和与备份文件一起存储。如果文件已被修改，请勿尝试还原目录配置。

2.19.3将GDS目录的逻辑备份恢复到新的目录数据库中

将目录移动到新数据库时，必须首先在数据库上创建一个空目录（请参阅创建全局数据服务目录时需要了解的内容 。）之后，可以按照上一节中的描述执行import catalog命令。

如果新目录数据库具有不同的连接字符串，则管理员有责任更改全局服务管理器系统上的连接字符串。在这种情况下，还需要重新启动所有全局服务管理器。同步过程将不会完成，因此在至少一个全局服务管理器向目录注册之前，恢复过程将不会完成。

2.20更改GSMCATUSER密码

要更改GSMCATUSER密码：

  1. 连接到GDS目录时，在SQL * Plus中运行以下命令：
    ALTER USER gsmcatuser IDENTIFIED BY ****
  2. 然后在GDSCTL中运行以下命令：
    GDSCTL>修改目录-oldpwd oldpassword -newpwd newpassword
