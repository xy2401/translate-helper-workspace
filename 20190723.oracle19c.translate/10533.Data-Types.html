数据类型

Oracle数据库操作的每个值都有一个数据类型 。值的数据类型将固定的一组属性与值相关联。这些属性使Oracle将一种数据类型的值与另一种数据类型的值区别对待。例如，您可以添加NUMBER数据类型的值，但不能添加RAW数据类型的值。

创建表或集群时，必须为其每个列指定数据类型。创建过程或存储函数时，必须为其每个参数指定数据类型。这些数据类型定义每个列可以包含或每个参数可以包含的值的域。例如， DATE列不能接受2月29日的值（闰年除外）或值2或'SHOE'。随后放在列中的每个值都假定列的数据类型。例如，如果将“ '01-JAN-98'插入DATE列，则Oracle会在验证它转换为有效日期后将'01-JAN-98'字符串视为DATE值。

字符数据类型

字符数据类型在数据库字符集或国家字符集中存储字符（字母数字）数据，它们是单词和自由格式文本。它们比其他数据类型的限制性更小，因此具有更少的属性。例如，字符列可以存储所有字母数字值，但NUMBER列只能存储数字值。

字符数据存储在字符串中，字符值对应于创建数据库时指定的字符集之一，例如7位ASCII或EBCDIC。Oracle数据库支持单字节和多字节字符集。

这些数据类型用于字符数据：

  • CHAR数据类型

  • NCHAR数据类型

  • VARCHAR2数据类型

  • NVARCHAR2数据类型

有关将字符数据类型指定为文字的信息，请参阅文本文字 。

CHAR数据类型

CHAR数据类型指定数据库字符集中的固定长度字符串。您在创建数据库时指定数据库字符集。

使用CHAR列创建表时，可以将列长度指定为size可选地后跟长度限定符。限定符BYTE表示字节长度语义，而限定符CHAR表示字符长度语义。在字节长度语义中， size是要存储在列中的字节数。在字符长度语义中， size是要存储在列中的数据库字符集中的代码点数。代码点可以具有1到4个字节，具体取决于数据库字符集和由代码点编码的特定字符。Oracle建议您指定一个长度限定符以显式记录列的所需长度语义。如果未指定限定符，则创建列的会话的NLS_LENGTH_SEMANTICS参数的值将定义长度语义，除非该表属于模式SYS ，在这种情况下，默认语义为BYTE 。

Oracle确保存储在CHAR列中的所有值都具有以所选长度语义中的size指定的长度。如果插入的值小于列长度，则Oracle会将值空白填充到列长度。如果您尝试为列插入一个太长的值，则Oracle会返回错误。请注意，如果列长度以字符（代码点）表示，则空白填充不保证所有列值具有相同的字节长度。

您可以从列定义中省略size 。默认值为1。

size的最大值是2000，这意味着2000个字节或字符（代码点），具体取决于所选的长度语义。但是，独立地，可以存储到CHAR列的任何字符值的绝对最大长度是2000个字节。例如，即使您将列长度定义为2000个字符，如果您尝试插入一个2000个字符的值（其中一个或多个代码点大于1个字节），Oracle也会返回错误。字符size的值是长度约束，而不是保证容量。如果希望CHAR列始终能够在任何数据库字符集中存储size字符，请使用size小于或等于500的值。

要确保数据库和具有不同字符集的客户端之间正确的数据转换，必须确保CHAR数据由格式良好的字符串组成。

NCHAR数据类型

NCHAR数据类型指定国家字符集中的固定长度字符串。在创建数据库时，将国家字符集指定为AL16UTF16或UTF8。AL16UTF16和UTF8是Unicode字符集的两种编码形式（UTF-16和CESU-8，相应地），因此NCHAR是仅Unicode的数据类型。

使用NCHAR列创建表时，可以将列长度指定为size字符，或者更准确地说，指定国家字符集中的代码点。一个代码点在AL16UTF16中始终为2个字节，在UTF8中始终为1到3个字节，具体取决于代码点编码的特定字符。

Oracle确保存储在NCHAR列中的所有值都具有size字符的长度。如果插入的值小于列长度，则Oracle会将值空白填充到列长度。如果您尝试为列插入一个太长的值，则Oracle会返回错误。请注意，如果国家字符集是UTF8，则空白填充不保证所有列值具有相同的字节长度。

您可以从列定义中省略size 。默认值为1。

当国家字符集为AL16UTF16时， size的最大值为1000个字符;当国家字符集为UTF8时， size的最大值为2000个字符。但是，独立地，可以存储到NCHAR列中的任何字符值的绝对最大长度是2000个字节。例如，即使您将列长度定义为1000个字符，如果您尝试插入1000个字符的值但是国家字符集是UTF8且所有代码点都是3个字节宽，Oracle也会返回错误。size的值是长度约束，而不是保证容量。如果希望NCHAR列始终能够在两个国家字符集中存储size字符，请使用size小于或等于666的值。

要确保数据库和具有不同字符集的客户端之间的正确数据转换，必须确保NCHAR数据由格式良好的字符串组成。

如果将CHAR值分配给NCHAR列，则会将值从数据库字符集隐式转换为国家字符集。如果将NCHAR值分配给CHAR列，则该值将从国家字符集隐式转换为数据库字符集。如果NCHAR值中的某些字符无法在数据库字符集中表示，那么如果会话参数NLS_NCHAR_CONV_EXCP值为TRUE ，则Oracle会报告错误。如果参数的值为FALSE ，则不可表示的字符将替换为数据库字符集的默认替换字符，通常是问号'？'或倒置的问号'¿'。

FLOAT数据类型

FLOAT数据类型是NUMBER的子类型。它可以使用或不使用精度进行指定，其定义与NUMBER相同，范围为1到126。无法指定比例，但是从数据中解释。每个FLOAT值需要1到22个字节。

要将二进制转换为十进制精度，请将n乘以0.30103。要从十进制精度转换为二进制精度，请将小数精度乘以3.32193。二进制精度的最大126位大致相当于小数精度的38位。

NUMBER和FLOAT之间的差异最好通过示例说明。在以下示例中，相同的值将插入NUMBER和FLOAT列中：

CREATE TABLE test（col1 NUMBER（5,2），col2 FLOAT（5））;插入测试值（1.23,1.23）;插入测试值（7.89,7.89）;插入测试值（12.79,12.79）;插入测试值（123.45,123.45）; SELECT * FROM test; COL1 COL2 ---------- ---------- 1.23 1.2 7.89 7.9 12.79 13 123.45 120

在此示例中，返回的FLOAT值不能超过5个二进制数字。可以用5个二进制数字表示的最大十进制数是31。最后一行包含超过31的十进制值。因此，必须截断FLOAT值，使其有效数字不需要超过5个二进制数字。因此，123.45被舍入为120，其仅具有两个有效十进制数字，仅需要4个二进制数字。

转换ANSI FLOAT数据时，Oracle数据库在内部使用Oracle FLOAT数据类型。Oracle FLOAT可供您使用，但Oracle建议您使用BINARY_FLOAT和BINARY_DOUBLE数据类型，因为它们更健壮。有关更多信息，请参阅浮点数 。

浮点数字

浮点数可以在从第一个数字到最后一个数字的任何位置具有小数点，或者根本没有小数点。可以在数字之后可选地使用指数来增加范围，例如1.777 e -20 。比例值不适用于浮点数，因为小数点后可以出现的位数不受限制。

二进制浮点数字从不同NUMBER中的值是由Oracle数据库内部存储的方式。使用NUMBER小数精度存储值。是所支持的范围和精度内的所有文字NUMBER存储完全一样NUMBER 。文字的存储完全是因为文字用十进制精度表示（数字0到9）。二进制浮点数使用二进制精度（数字0和1）存储。这种存储方案不能精确地使用小数精度表示所有值。通常，当将值从二进制转换回十进制精度时，将值从十进制转换为二进制精度时发生的错误将被撤消。文字0.1就是这样一个例子。

BINARY_FLOAT

BINARY_DOUBLE

在NUMBER列中，浮点数具有小数精度。在BINARY_FLOAT或BINARY_DOUBLE列中，浮点数具有二进制精度。二进制浮点数支持特殊值无穷大和NaN （不是数字）。

IEEE754一致性

浮点数据类型的Oracle实现基本符合电气和电子工程师协会（IEEE）二进制浮点运算标准IEEE标准754-1985（IEEE754）。浮点数据类型在以下方面符合IEEE754：

  • SQL函数SQRT实现了平方根。见SQRT 。

  • SQL函数REMAINDER实现了余数。请参阅REMAINDER 。

  • 算术运算符符合。请参见算术运算符 。

  • 除了与NaN比较之外，比较运算符符合。 Oracle针对所有其他值命令NaN最大，并且评估NaN等于NaN 。请参阅浮点条件 。

  • 转换运算符符合。请参见转换函数 。

  • 支持默认的舍入模式。

  • 支持默认的异常处理模式。

  • 支持特殊值INF ， - INF和NaN 。请参阅浮点条件 。

  • SQL函数ROUND ， TRUNC ， CEIL和FLOOR提供了将BINARY_FLOAT和BINARY_DOUBLE值舍入为整数值BINARY_FLOAT和BINARY_DOUBLE值。

  • 舍入BINARY_FLOAT / BINARY_DOUBLE为十进制，十进制为BINARY_FLOAT / BINARY_DOUBLE由SQL函数TO_CHAR ， TO_NUMBER ， TO_NCHAR ， TO_BINARY_FLOAT ， TO_BINARY_DOUBLE和CAST 。

浮点数据类型在以下方面不符合IEEE754：

  • -0被强制为+0。

  • 不支持与NaN比较。

  • 所有NaN值都强制为BINARY_FLOAT_NAN或BINARY_DOUBLE_NAN 。

  • 不支持非默认舍入模式。

  • 不支持非默认异常处理模式。

数字优先

对于支持数字数据类型的操作， 数值优先级确定Oracle在操作的参数具有不同数据类型时使用的数据类型。BINARY_DOUBLE具有最高的数字优先级，后跟BINARY_FLOAT ，最后是NUMBER 。因此，在对多个数值的任何操作中：

  • 如果任何操作数是BINARY_DOUBLE ，则Oracle会在执行操作之前尝试将所有操作数隐式转换为BINARY_DOUBLE 。

  • 如果没有操作数是BINARY_DOUBLE但是任何操作数都是BINARY_FLOAT ，那么Oracle会在执行操作之前尝试将所有操作数隐式转换为BINARY_FLOAT 。

  • 否则，Oracle会在执行操作之前尝试将所有操作数转换为NUMBER 。

如果需要任何隐式转换并失败，则操作失败。有关隐式转换的更多信息，请参阅表2-8 。

在其他数据类型的上下文中，数字数据类型的优先级低于datetime / interval数据类型，优先级高于字符和所有其他数据类型。

日期时间/间隔算术

您可以在日期（ DATE ），时间戳（ TIMESTAMP ， TIMESTAMP WITH TIME ZONE和TIMESTAMP WITH LOCAL TIME ZONE ）和间隔（ INTERVAL DAY TO SECOND和INTERVAL YEAR TO MONTH ）数据上执行许多算术运算。Oracle根据以下规则计算结果：

  • 您可以在日期和时间戳值的算术运算中使用NUMBER常量，但不能使用区间值。Oracle在内部将时间戳值转换为日期值，并将算术日期时间和间隔表达式中的NUMBER常量解释为天数。例如，明天是SYSDATE + 1。SYSDATE - 7是一周前。SYSDATE +（10/1440）距现在十分钟。从SYSDATE减去样本表employees的hire_date列返回自雇用每个雇员以来的天数。您不能乘以或除以日期或时间戳值。

  • Oracle隐式地将BINARY_FLOAT和BINARY_DOUBLE操作数转换为NUMBER 。

  • 每个DATE值包含一个时间组件，许多日期操作的结果包括一个分数。该分数表示一天的一部分。例如，1.5天是36小时。这些分数也由Oracle内置函数返回，用于DATE数据的常见操作。例如， MONTHS_BETWEEN函数返回两个日期之间的月数。结果的小数部分代表31天月份的一部分。

  • 如果一个操作数是DATE值或数值，它们都不包含时区或小数秒组件，则：

    • Oracle隐式地将另一个操作数转换为DATE数据。例外是将数值乘以间隔，该间隔返回间隔。

    • 如果另一个操作数具有时区值，则Oracle将使用返回值中的会话时区。

    • 如果另一个操作数具有小数秒值，则小数秒值将丢失。

  • 将时间戳，间隔或数值传递给仅为DATE数据类型设计的内置函数时，Oracle会将非DATE值隐式转换为DATE值。有关哪些函数导致隐式转换为DATE信息，请参阅日期时间函数 。

  • 当间隔计算返回datetime值时，结果必须是实际的datetime值，否则数据库将返回错误。例如，接下来的两个语句返回错误：

    SELECT TO_DATE（'31 -AUG-2004'，'DD-MON-YYYY'）+ TO_YMINTERVAL（'0-1'）FROM DUAL; SELECT TO_DATE（'29-FEB-2004'，'DD-MON-YYYY'）+ TO_YMINTERVAL（'1-0'）FROM DUAL;

RAW和LONG RAW数据类型

RAW和LONG RAW数据类型存储在不同系统之间移动数据时Oracle数据库无法显式转换的数据。这些数据类型用于二进制数据或字节字符串。例如，您可以使用LONG RAW存储图形，声音，文档或二进制数据数组，其解释取决于使用情况。

Oracle强烈建议您将LONG RAW列转换为二进制LOB（ BLOB ）列。与LONG列相比，LOB列的限制要少得多。有关更多信息，请参阅TO_LOB 。

RAW是一种可变长度数据类型，如VARCHAR2 ，但Oracle Net（将客户端软件连接到数据库或将一个数据库连接到另一个数据库）和Oracle导入和导出实用程序在传输RAW或LONG RAW数据时不执行字符转换。相反，Oracle Net和Oracle导入和导出实用程序会自动在不同数据库字符集之间转换CHAR ， VARCHAR2和LONG数据，如果数据在数据库之间传输，或者在数据库字符集和客户端字符集之间传输，如果数据是传输的在数据库和客户端之间。客户端字符集由客户端接口的类型（例如OCI或JDBC）和客户端配置（例如， NLS_LANG环境变量）确定。

当Oracle隐式地将RAW或LONG RAW数据转换为字符数据时，结果字符值包含二进制输入的十六进制表示，其中每个字符是十六进制数字（ 0 - 9 ， A - F ），表示RAW数据的四个连续位。例如，具有位11001011的RAW数据的一个字节变为值CB 。

当Oracle隐式地将字符数据转换为RAW或LONG RAW ，它将每个连续输入字符解释为四个连续二进制数据位的十六进制表示，并通过连接这些位来构建生成的RAW或LONG RAW值。如果任何输入字符不是十六进制数字（ 0 - 9 ， A - F ， a - f ），则报告错误。如果字符数是奇数，则结果是未定义的。

SQL函数RAWTOHEX和HEXTORAW执行与上述隐式转换等效的显式转换。使用Oracle提供的PL / SQL包UTL_RAW和UTL_I18N函数可以实现RAW和字符数据之间的其他类型的转换。

大对象（LOB）数据类型

内置LOB数据类型BLOB ， CLOB和NCLOB （内部存储）和BFILE （外部存储）可以存储大型和非结构化数据，如文本，图像，视频和空间数据。BLOB ， CLOB和NCLOB数据的大小可以达到（2 32 -1字节）*（LOB存储的CHUNK参数的值）。如果数据库中的表空间是标准块大小，并且在创建LOB列时使用了LOB存储的CHUNK参数的缺省值，那么这相当于（2 32 -1字节）*（数据库块大小） ）。尽管您的操作系统可能会对此最大值施加限制，但BFILE数据最多可达2 64 -1个字节。

创建表时，您可以选择为LOB列或LOB对象属性指定不同的表空间和存储特性，以及为表指定的属性。

如果在创建LOB列时在行中启用存储，则最多大约4000字节的CLOB ， NCLOB和BLOB值将NCLOB联方式存储。大于4000字节的LOB始终存储在外部。有关更多信息，请参阅ENABLE STORAGE IN ROW 。

LOB列包含LOB定位符，可以引用内部（在数据库中）或外部（在数据库外）LOB值。从表中选择LOB实际上返回LOB定位符而不是整个LOB值。LOB上的DBMS_LOB包和Oracle调用接口（OCI）操作是通过这些定位器执行的。

LOB类似于LONG和LONG RAW类型，但在以下方面有所不同：

  • LOB可以是对象类型（用户定义的数据类型）的属性。

  • LOB定位器存储在表列中，可以带有或不带有实际的LOB值。BLOB ， NCLOB和CLOB值可以存储在单独的表空间中。BFILE数据存储在服务器上的外部文件中。

  • 访问LOB列时，将返回定位器。

  • LOB的大小最多为（2 32 -1字节）*（数据库块大小）。尽管您的操作系统可能会对此最大值施加限制，但BFILE数据最多可达2 64 -1个字节。

  • LOB允许高效，随机，分段访问和操纵数据。

  • 您可以在表中定义多个LOB列。

  • 除NCLOB ，您可以在对象中定义一个或多个LOB属性。

  • 您可以声明LOB绑定变量。

  • 您可以选择LOB列和LOB属性。

  • 您可以插入新行或更新包含一个或多个LOB列的现有行或具有一个或多个LOB属性的对象。在更新操作中，您可以将内部LOB值设置为NULL ，为空，或用数据替换整个LOB。您可以将BFILE设置为NULL或使其指向其他文件。

  • 您可以使用另一个LOB行 - 列交集或LOB属性更新LOB行列交集或LOB属性。

  • 您可以删除包含LOB列或LOB属性的行，从而也删除LOB值。对于BFILE，不删除实际的操作系统文件。

只需发出INSERT或UPDATE语句，即可访问和填充内联LOB列（存储在数据库中的LOB列）或LOB属性（存储在数据库中的对象类型列的属性）的行。

BFILE数据类型

BFILE数据类型允许访问存储在Oracle数据库外部的文件系统中的二进制文件LOB。BFILE列或属性存储BFILE定位符，该定位符用作指向服务器文件系统上的二进制文件的指针。定位器维护目录名称和文件名。

二进制文件LOB不参与事务并且不可恢复。相反，底层操作系统提供文件完整性和持久性。尽管您的操作系统可能会对此最大值施加限制，但BFILE数据最多可达2 64 -1个字节。

数据库管理员必须确保外部文件存在，并且Oracle进程对文件具有操作系统读取权限。

BFILE数据类型支持对大型二进制文件的只读支持。您无法修改或复制此类文件。Oracle提供API来访问文件数据。用于访问文件数据的主要接口是DBMS_LOB包和Oracle调用接口（OCI）。

ROWID数据类型

Oracle数据库本机的堆组织表中的行具有称为rowid的行地址。您可以通过查询伪列ROWID来检查rowid行地址。该伪列的值是表示每行地址的字符串。这些字符串的数据类型为ROWID 。您还可以创建包含具有ROWID数据类型的实际列的表和集群。Oracle数据库不保证此类列的值是有效的rowid。有关ROWID伪列的更多信息，请参阅Pseudocolumns 。

Rowids包含以下信息：

  • 包含行的数据文件的数据块 。此字符串的长度取决于您的操作系统。

  • 数据块中的行 。

  • 包含行的数据库文件 。第一个数据文件的编号为1。此字符串的长度取决于您的操作系统。

  • 数据对象编号 ，是分配给每个数据库段的标识号。您可以从数据字典视图的数据对象编号USER_OBJECTS ， DBA_OBJECTS和ALL_OBJECTS 。共享同一段的对象（例如，同一群集中的群集表）具有相同的对象编号。

Rowid存储为基本64值，可包含字符AZ，az，0-9，加号（+）和正斜杠（/）。Rowid不能直接使用。您可以使用提供的包DBMS_ROWID来解释rowid内容。包函数提取并提供有关上面列出的四个rowid元素的信息。

UROWID数据类型

某些表的行具有非物理或永久性的地址，或者不是Oracle数据库生成的地址。例如，索引组织表的行地址存储在索引叶中，索引叶可以移动。外部表的行（例如通过网关访问的DB2表）不是标准的Oracle rowid。

Oracle使用通用rowid（ urowids ）来存储索引组织和外部表的地址。索引组织表具有逻辑urowids，外表具有外部urowids。两种类型的urowid都存储在ROWID伪列中（与堆组织表的物理rowid一样）。

Oracle根据表的主键创建逻辑rowid。只要主键不更改，逻辑rowid就不会更改。索引组织表的ROWID伪列具有UROWID的数据类型。您可以像堆组织表的ROWID伪列一样访问此伪列（使用SELECT ...ROWID声明）。如果你想to存储索引组织表的rowid，那么您可以定义类型的列UROWID为表和检索的价值ROWID虚列到该列。

对象类型

对象类型是现实世界实体的抽象，例如应用程序处理的采购订单。对象类型是具有三种组件的模式对象：

  • 一个名称 ，用于在该模式中唯一标识对象类型。

  • 属性 ，是内置类型或其他用户定义的类型。属性模拟现实世界实体的结构。

  • 方法 ，是用PL / SQL编写并存储在数据库中的函数或过程，或者用C或Java等语言编写并存储在外部。方法实现应用程序可以在现实世界中执行的操作。

REF数据类型

对象标识符 （由关键字OID表示）唯一标识对象，使您能够从其他对象或关系表引用该对象。名为REF数据类型类别表示此类引用。REF数据类型是对象标识符的容器。REF值是指向对象的指针。

当REF值指向不存在的对象时， REF被称为“悬空”。悬空REF与空REF不同。要确定REF是否悬空，请使用条件IS [ NOT ] DANGLING 。例如，给定示例模式oe中的对象视图oc_orders ，列customer_ref的类型为REF ，类型为customer_typ ，其属性为cust_email ：

SELECT o.customer_ref.cust_email FROM oc_orders o WHERE o.customer_ref不是DANGLING;

可变数组

数组是一组有序的数据元素。给定数组的所有元素都具有相同的数据类型。每个元素都有一个索引 ，它是一个与数组中元素位置相对应的数字。

数组中元素的数量是数组的大小。Oracle数组的大小可变，这就是它们被称为varrays的原因 。声明varray时，必须指定最大大小。

声明varray时，它不会分配空间。它定义了一个类型，您可以将其用作：

  • 关系表的列的数据类型

  • 对象类型属性

  • PL / SQL变量，参数或函数返回类型

Oracle通常将数组对象存储在行中（作为行数据的一部分）或脱离行（在LOB中），具体取决于其大小。但是，如果为varray指定单独的存储特性，则Oracle会将其存储在线外，无论其大小如何。请参阅varray_col_properties的CREATE TABLE约VARRAY存储更多的信息。

嵌套表

嵌套表类型为无序元素集建模。元素可以是内置类型或用户定义类型。您可以将嵌套表视为单列表，或者如果嵌套表是对象类型，则可以将其视为多列表，并为对象类型的每个属性指定一列。

嵌套表定义不分配空间。它定义了一个类型，您可以使用它来声明：

  • 关系表的列的数据类型

  • 对象类型属性

  • PL / SQL变量，参数或函数返回类型

当嵌套表作为关系表中列的类型或作为对象表的基础对象类型的属性出现时，Oracle会将所有嵌套表数据存储在单个表中，该表与封闭关系或对象表。

Oracle提供的类型

当内置或ANSI支持的类型不足时，Oracle提供基于SQL的接口来定义新类型。这些类型的行为可以在C / C ++，Java或PL / SQL中实现。 Oracle数据库自动提供输入输出所需的低级基础结构服务，新数据类型的异构客户端访问以及应用程序和数据库之间数据传输的优化。

这些接口可用于构建用户定义（或对象）类型，Oracle也可使用这些接口创建一些常用的数据类型。服务器提供了几种这样的数据类型，它们既可以用于广泛的水平应用区域（例如， Any类型），也可以用于特定的垂直应用区域（例如，空间类型）。

可扩展标记语言（XML）是由万维网联盟（W3C）开发的用于在万维网上表示结构化和非结构化数据的标准格式。通用资源标识符（URI）标识Web上任何位置的网页等资源。Oracle提供了处理XML和URI数据的类型，以及一类称为DBURIRef类型的URI，用于访问存储在数据库本身中的数据。它还提供了一组类型，用于存储和访问数据库中的外部和内部URI。

的XMLType

URI数据类型

Oracle提供了一系列URI类型 - URIType ， DBURIType ， XDBURIType和HTTPURIType - 它们与继承层次结构相关。URIType是一种对象类型，其他是URIType子类型。由于URIType是超类型，因此您可以创建此类型的列，并在此列中存储DBURIType或HTTPURIType类型实例。

HTTPURIType

您可以使用HTTPURIType将URL存储到外部Web页面或文件。Oracle使用HTTP（超文本传输协议）访问这些文件。

XDBURIType

您可以使用XDBURIType将XML数据库层次结构中的文档公开为可嵌入表中任何URIType列的URI。XDBURIType由一个URL组成，该URL包含它所引用的XML文档的层次名称以及表示XPath语法的可选片段。片段通过井号（＃）与URL部分分开。以下行是XDBURIType示例：

/home/oe/doc1.xml /home/oe/doc1.xml#/orders/order_item

DBURIType

DBURIType可用于存储DBURIRef值，该值引用数据库内的数据。存储DBURIRef值使您可以引用存储在数据库内部或外部的数据并一致地访问数据。

DBURIRef值使用类似XPath的表示来引用数据库中的数据。如果您将数据库想象为XML树，那么您将在XML文档中看到表，行和列作为元素。例如，示例人力资源用户hr将看到以下XML树：

<HR> <EMPLOYEES> <ROW> <EMPLOYEE_ID> 205 </ EMPLOYEE_ID> <LAST_NAME> Higgins </ LAST_NAME> <SALARY> 12008 </ SALARY> .. <！ - 其他栏目 - > </ ROW> .. 。<！ - 其他行 - > </ EMPLOYEES> <！ - 其他表..--> </ HR> <！ - 您有特权的其他用户模式..-->

DBURIRef是这个虚拟XML文档的XPath表达式。因此，要在员工编号为205的员工的EMPLOYEES表中引用SALARY值，您可以将DBURIRef写为，

/ HR / EMPLOYEES / ROW [EMPLOYEE_ID = 205] / SALARY

使用此模型，您可以引用存储在CLOB列或其他列中的数据，并将它们作为URL公开给外部世界。

URIFactory包

Oracle还提供了URIFactory包，它可以创建和返回URITypes的各种子类型的URITypes 。该包分析URL字符串，标识URL的类型（HTTP， DBURI等），并创建子类型的实例。要创建DBURI实例，URL必须以前缀/oradb开头。例如， URIFactory.getURI('/oradb/HR/EMPLOYEES')将创建一个DBURIType实例，而URIFactory.getUri('/sys/schema')将创建一个XDBURIType实例。

Oracle Spatial and Graph旨在使位置启用的应用程序，地理信息系统（GIS）应用程序和地理成像应用程序的用户更轻松，更自然地进行空间数据管理。将空间数据存储在Oracle数据库中之后，您可以轻松地操作，检索并将其与存储在数据库中的所有其他数据相关联。只有安装了Oracle Spatial and Graph，才能使用以下数据类型。

SDO_GEOMETRY

空间对象的几何描述存储在用户定义的表中的对象类型SDO_GEOMETRY的单个列中的单个行中。任何具有SDO_GEOMETRY类型列的表必须具有另一列或一组列，用于为该表定义唯一主键。这种表有时称为几何表。

SDO_GEOMETRY对象类型具有以下定义：

CREATE TYPE SDO_GEOMETRY AS OBJECT（sgo_gtype NUMBER，sdo_srid NUMBER，sdo_point SDO_POINT_TYPE，sdo_elem_info SDO_ELEM_INFO_ARRAY，sdo_ordinates SDO_ORDINATE_ARRAY）; /

SDO_TOPO_GEOMETRY

此类型描述了一个拓扑几何，它存储在一行中，位于用户定义表中的对象类型SDO_TOPO_GEOMETRY的单个列中。

SDO_TOPO_GEOMETRY对象类型具有以下定义：

CREATE TYPE SDO_TOPO_GEOMETRY AS OBJECT（tg_type NUMBER，tg_id NUMBER，tg_layer_id NUMBER，topology_id NUMBER）; /
