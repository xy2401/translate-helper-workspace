12.3 OJDS命令行工具

增强的ojvmjava命令使您可以操作和浏览OJDS。本节介绍以下命令：

  • ls命令

  • cd命令

  • pwd命令

  • chown命令

  • mkdir命令

  • rm命令

  • 命令

  • mv命令

  • chmod命令

  • 绑定命令

  • bindds命令

  • bindurl命令

12.3.1 ls命令

ls命令显示上下文的内容。

句法

ls [options] [context1] [context2] [obj | context] ...

选项

下表描述了ls命令选项：

选项 描述

上下文obj

指定要列出的上下文或对象的名称

-l

以长格式显示内容，包括名称，创建时间，所有者，权限等。显示对象的类。

DIR

仅显示上下文，类似于Unix ls –d命令

LDIR

像–l命令一样以长格式显示内容，但忽略绑定对象，类似于Unix -ld

[R

通过子上下文递归列出

例

以下命令以短格式显示根上下文的内容：

$ ls / etc / public /

以下命令以长格式显示根上下文的内容：

$ ls -l命令读取 写入 Exec的 所有者 日期 时间 类型 名称 PUBLIC SYS PUBLIC SYS 12月14日 14:59 背景 等 公共 公共 公共 SYS 12月14日 14:59 背景下 的公共

12.3.2 cd命令

cd命令更改工作上下文。此命令类似于Unix cd命令以更改目录。

例

以下命令将上下文更改为根上下文

$ cd /

12.3.3 pwd命令

pwd命令列出当前的工作上下文。

例

如果当前上下文是/test/alpha ，那么pw d命令的输出如下：

$ pwd / test / alpha /

12.3.4 chown命令

chown命令更改上下文或对象的所有权。

注意：

仅当您是SYS用户时，才能更改上下文或对象的所有权。

句法

chown [options] {user |角色} <对象名称>

选项

下表描述了chown命令选项：

选项 描述

用户角色

指定要成为所有者的用户或角色的名称

<对象名称>

指定要更改的上下文或对象的名称

-R

递归更改以下所有权：

  • 上下文

  • 上下文中的所有子上下文

  • 上下文和子上下文中包含的所有对象

例

以下命令使HR成为/alpha/beta/gamma命令的所有者：

$ chown HR / alpha / beta / gamma

12.3.5 mkdir命令

mkdir命令创建具有给定名称的上下文。

注意：

您必须具有目标上下文的写入权限才能创建新上下文。

选项

下表描述了mkdir命令选项：

选项 描述

<名称>

指定要创建的上下文的名称

-path | -p

如果它们不存在，则创建中间上下文

例

以下命令创建一个名为/test/alpha的Context，其中/test上下文已存在：

mkdir / test / alpha

以下命令创建一个名为/test/alpha/beta/gamma的Context，其中/test/alpha/beta上下文不存在：

$ mkdir -path / test / alpha / beta / gamma

12.3.6 rm命令

rm命令类似于rm UNIX shell命令。此命令删除对象或上下文，包括其内容。

注意：

要删除对象，必须对包含该对象的上下文具有“写入”权限。

选项

下表介绍了rm命令选项：

选项 描述

<对象>

指定上下文的名称或要删除的对象

-recurse | -r

假设上下文并递归地删除上下文及其内容

例子

以下命令删除对象/test/bank ：

rm / test / bank

以下命令将删除context /test/release3及其内容：

rm -r / test / release3

12.3.7 ln命令

ln命令类似于UNIX ln命令。链接是上下文或对象的同义词。移动上下文或对象时，对上下文或对象的引用可能会变为无效。ln命令使用旧名称创建链接，并使旧名称和新名称都可以访问该对象。

句法

ln [-symbolic | -s] <object> <link>

选项

下表描述了ln命令选项：

选项 描述

-s

创建<object>到<link>的软链接

<对象>

指定上下文或对象的名称

<链路>

指定链接上下文或对象的同义词的名称

例

即使将对象的名称更改为new ，以下命令也会保留对old对象的访问权限：

$ mv old new $ ln new old

12.3.8 mv命令

mv命令将上下文或对象的名称（或重新绑定旧名称）更改为新名称。

句法

mv <old> <new>

例

以下命令将context /test/foo的名称更改为/test/bar ：

$ mv / test / foo / test / bar

12.3.9 chmod命令

chmod命令类似于chmod UNIX shell命令。此命令更改用户或上下文或对象上的角色的权限。

注意：

仅当您是SYS用户或对象的所有者时，才能更改对象的权限。

句法

chmod [options] {+ | - } {r | w | x} {<user> | <role>，...} <objectname>

选项

下表描述了chmod命令选项：

选项 描述

+/- RWX

添加（+）或删除（ - ）读，写或执行

<user> | <作用>

添加或删除其权限的用户或角色

<对象>

权限更改的上下文或对象

-R

递归更改权限

例

以下示例将/alpha/beta/gamma上下文的权限更改为HR和NANCY ：

$ chmod + x HR，NANCY / alpha / beta / gamma

注意：

模式仅以逗号分隔。

以下示例删除/alpha/beta/gamma上下文的HR写权限：

$ chmod -w HR / alpha / beta / gamma

12.3.10 bind命令

bind命令将对象引用或上下文绑定到JNDI名称空间。

句法

bind <objectname> [options] [-context] [-rebind] {-class | -c <classname>} [-factory | -f <factory>] [-location | -l <URL>] [ - string <type_name>

<string_value> [-string <type_name> <string_value> ...]] [-binary <type_name> <string_value> [-binary <type_name> <string_value> ...]]

选项

下表介绍了bind命令选项：

选项 描述

<对象>

要绑定的是Name对象

-context

要绑定的对象是Context或InitialContext

-rebind

如果JNDI名称已存在，则将使用此对象绑定的对象替换为该对象

-class <classname>

指定绑定对象的类名

- 工厂>

指定用于创建对象的工厂类名称。JNDI使用它来创建对象。

-location <URL>

如果未使用默认位置，则指定工厂位置。这需要一个JNDI URL。

-string <type_name> <string_value>

按类型名称和值指定对象的String引用属性。

-binary <type_name> <string_value>

按类型和二进制值指定对象的二进制引用属性。给定的十六进制字符串值将转换为二进制。

例

以下将对象引用绑定到名称空间。字符串和二进制属性提供给引用。

bind / tmp / myprinter -class gen。Inkjet -factory gen。InkjetFactory -string PRINTERNAME co2 -binary DPI 0X12C

12.3.11 bindds命令

此命令绑定JNDI名称空间中的DataSource对象。此命令根据指定的选项绑定常规，XA或池数据源。

注意：

Oracle JVM仅支持kprb驱动程序和瘦驱动程序。

句法

bindds <object_name> [options] [-help | -h] [ - 描述| -d] [-version | -v] [-dstype <datasource>]

[-host <hostname> -port <portnum> -sid <SID> -driver <driver_type>] [-url <db_url>]

[-g | -grant {<user> | <role>} [，{<user> | <role>}] ...] [-recursiveGrant | -rg {<user> | <role>}

[，{<user> | <role>}] ...] [-rebind] [-user | -u <user>]

选项

下表描述了bindds命令选项：

选项 描述

<对象>

指定要绑定对象的名称

-救命

指定帮助消息

-描述

总结工具操作

-版

指定版本号

-dstype <type>

从以下类型之一指定数据源类型：

  • 没有OracleDataDource

  • xa for OracleXADatasource

  • OracleConnectionPoolDataSource pool

-host <hostname> -port <portnum> -sid <SID> -driver <drv_type>

指定数据库的位置和与数据库的连接的驱动程序类型。您也可以在-url选项中以URL格式指定此信息。-sid选项的默认值为ORCL 。 -driver选项可以具有以下值： thin ， oci或kprb 。

-url <db_url>

此JDBC URL指定数据库的位置。

-grant <user | role>，<user | role> ...

授予对<user>和<role>名称序列的读取和执行权限。重新绑定时，使用<user>和<role>名称替换具有读取或执行权限的现有用户或角色。

-recursiveGrant <user | role>，<user | role> ....

递归授予对指定对象和对象所在的所有上下文的“读取”和“执行”权限。如果上下文具有SYS的权限级别，则忽略该上下文的授权。

-rebind

如果DataSource对象已存在，则必须指定此选项以使用此新对象覆盖现有数据源。否则，此选项不会发生绑定。

-user <user>

指定用于连接数据库的用户名。在DataSource对象中存储用户名。如果在创建数据库连接时未在JNDI上下文中提供用户名，则使用此用户名。

例

以下示例将ds1数据源绑定到命名空间：

bindds / test / ds1 -host localhost -port 1522 -sid orcl -driver thin bindds / test / ds1 -url jdbc：oracle：thin：@localhost：1522：orcl

该示例将JDBC瘦驱动程序与通用数据源（即OracleDataSource 。

12.3.12 bindurl命令

bindurl命令绑定命名空间中的URL对象。

选项

下表描述了bindurl命令选项：

选项 描述

<对象>

指定要绑定的对象的名称

-救命

指定帮助消息

-描述

总结工具操作

-版

打印bindurl命令的版本

-rebind

如果JNDI名称已存在，则必须指定此选项以使用此新对象覆盖现有JNDI名称。否则，此选项不会发生绑定。

-grant <user | role>，<user | role> ...

授予对<user>和<role>名称序列的读取和执行权限。重新绑定时，可以使用<user>和<role>名称替换具有读取或执行权限的现有用户或角色。

-recursiveGrant <user | role>，<user | role> ...

递归授予对指定对象和对象所在的所有上下文的“读取”和“执行”权限。如果上下文具有SYS的权限级别，则忽略该上下文的授权。

例

以下示例将URL字符串http://www.oracle.com绑定到命名空间中的URL引用/test/myURL ：

bindurl / test / myURL http://www.oracle.com -rebind

-rebind选项用于确保如果先前存在/test/myURL引用，则使用字符串http://www.oracle.com反弹。
