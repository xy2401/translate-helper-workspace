7.1对象类型及其方法的权限

对象类型的权限存在于系统级别和架构对象级别。

话题：

  • 对象类型的系统特权

  • 架构对象权限

  • 新类型或表中使用的类型

  • 示例：对象类型的权限

  • 访问对象，类型和表的权限

7.1.1对象类型的系统权限

Oracle数据库为对象类型定义以下系统特权：

  • CREATE TYPE使您可以在自己的架构中创建对象类型

  • CREATE ANY TYPE使您可以在任何模式中创建对象类型

  • ALTER ANY TYPE允许您更改任何模式中的对象类型

  • DROP ANY TYPE允许您在任何模式中删除命名类型

  • EXECUTE ANY TYPE允许您在任何模式中使用和引用命名类型

  • UNDER ANY TYPE ，您可以创建在任何非最终对象类型亚型

  • UNDER ANY VIEW使您的任何对象视图下创建子视图

以下角色很有帮助：

  • RESOURCE角色包括CREATE TYPE系统特权。

  • DBA角色包括所有这些权限。

7.1.2架构对象权限

两个模式对象特权适用于对象类型：

  • EXECUTE使您可以使用以下类型：

    • 定义一个表。

    • 在关系表中定义列。

    • 声明命名类型的变量或参数。

    EXECUTE允许您调用类型的方法，包括构造函数。

    方法执行和关联的权限与存储的PL / SQL过程相同。

  • UNDER使您可以在授予权限的类型或视图下创建子类型或子视图。

    只有在直接超类型或超级视图上具有UNDER特权WITH GRANT OPTION的授予者才能授予子类型或子视图的UNDER特权。

短语WITH HIERARCHY OPTION为对象的所有子类型授予指定的对象特权。只有在对象视图层次结构中的对象视图上授予SELECT对象权限时，此选项才有意义。在这种情况下，该权限适用于授予该权限的视图的所有子视图。

7.1.3新类型或表中使用的类型

除了前面部分中详述的权限之外，您还需要具体的权限：

  • 创建使用其他用户创建的类型的类型或表。

  • 允许将新类型或表格用于其他用户。

对于用于定义新类型或表的任何类型，您必须具有EXECUTE ANY TYPE系统特权或EXECUTE对象特权。您必须已明确授予这些权限，而不是通过角色。

要向其他用户授予对新类型或表的访问权限，您必须具有GRANT选项所需的EXECUTE对象权限或具有WITH ADMIN OPTION选项的EXECUTE ANY TYPE系统权限。您必须已明确授予这些权限，而不是通过角色。

7.1.5对象，类型和表的访问权限

对象类型仅使用EXECUTE特权。

但是，对象表使用与关系表相同的所有权限：

  • READ或SELECT允许您从表中访问对象及其属性。

  • UPDATE允许您修改表中对象的属性。

  • INSERT允许您向表中添加新对象。

  • DELETE允许您从表中删除对象。

类似的表和列权限规定了对象类型的表列的使用。

选择对象表的列不需要对象表类型的特权。但是，选择整个行对象可以。

考虑下面在例7-6中创建的模式和查询：

示例7-6类型访问的SELECT权限

- 需要Ex。 7-1,7-2,7-3,7-4和7-5 CREATE TYPE emp_type AS OBJECT（eno NUMBER，ename VARCHAR2（36））; / CREATE TABLE emp OF emp_type; // emp TO user1上的对象表GRANT SELECT;选择价值（e）FROM emp e; SELECT eno，ename FROM emp;

对于这两个查询，Oracle数据库都会检查用户对对象表emp的SELECT权限。对于第一个查询，用户需要获取emp_type类型信息来解释数据。当查询访问emp_type类型时，数据库将检查用户的EXECUTE特权。

但是，第二个查询不涉及命名类型，因此数据库不检查类型特权。

此外， USER3可以执行以下查询：

从user2.tab2 t中选择t.col1.attr2; SELECT t.attr4.attr3.attr2 FROM tab3 t;

请注意，在两个查询中， USER3都没有对基础类型的显式权限。但是，该语句成功，因为类型和表所有者具有GRANT选项的必要权限。

Oracle数据库检查以下请求的权限，如果请求者没有该操作的权限，则返回错误：

  • 使用其REF值固定对象高速缓存中的对象会导致数据库检查包含对象的对象表的READ或SELECT特权以及对象类型的EXECUTE特权。

  • 修改现有对象或从对象缓存刷新对象会导致数据库检查目标对象表上的UPDATE特权。刷新新对象会导致数据库检查目标对象表上的INSERT权限。

  • 删除对象会导致数据库检查目标表上的DELETE权限。

  • 调用方法会导致数据库检查相应对象类型的EXECUTE特权。

Oracle数据库不为对象表提供列级权限。
