3.1运营商优先权

运算符优先级确定计算查询表达式的组件的顺序。文本查询运算符可以分为两组具有自己的评估顺序的运算符。这两组将在后面描述为第1组和第2组。

在所有情况下，查询表达式根据其运算符的优先级从左到右依次计算。首先应用具有更高优先级的运算符。具有相同优先级的运算符按照它们在表达式中从左到右的外观顺序应用。

  • 第2组操作员和角色

3.1.1第1组运营商

在查询表达式中，组1运算符具有以下从最高优先级到最低优先级的评估顺序：

  1. 等价（=）

  2. 附近（;）

  3. 重量（*） ， 阈值（>）

  4. 减号（ - ）

  9. 或（|）

  10. 累积（_）

  2. 干（$）

  3. 模糊

  4. soundex（！）

3.2关于

一般行为

使用ABOUT运算符可以返回与查询词或短语相关的文档。在英语和法语中， ABOUT使您可以查询概念，即使概念实际上不是查询的一部分。例如， ABOUT 热量的ABOUT查询可能返回与温度相关的文档，即使术语温度不是查询的一部分。

在其他语言中，使用ABOUT通常会增加返回文档的数量，并可能改善结果的排序顺序。对于所有语言，Oracle Text为ABOUT查询得分，最相关的文档获得最高分。

英国和法国的行为

在英语和法语中，使用ABOUT运算符查询概念。系统在索引的主题组件中查找概念信息。通过将INDEX_THEMES BASIC_LEXER属性设置为YES为索引创建主题组件。

注意：

您无需在索引中使用主题组件即可以英语和法语输入ABOUT查询。但是，在索引中使用主题组件会为ABOUT查询生成最佳结果。

Oracle Text检索包含与查询词或短语相关的概念的文档。例如，如果您在加利福尼亚州输入ABOUT查询，系统可能会返回包含洛杉矶和旧金山这两个加利福尼亚州城市的文档。该文档不需要包含在此ABOUT查询中返回的术语California 。

您的ABOUT查询中指定的单词或短语不需要与索引中存储的主题完全匹配。Oracle Text在索引中执行查找之前规范化单词或短语。

您可以将ABOUT运算符与CONTAINS和CATSEARCH SQL运算符一起使用。对于CATSEARCH ，您必须使用带有CONTEXT语法的查询模板来查询索引主题。请参阅示例部分中的关于使用CATSEARCH的查询 。

句法 描述

在英语和法语中，如果在索引时将BASIC_LEXER INDEX_THEMES属性设置为YES，则返回包含与短语相关的概念的文档。

句法 描述

3.6 CTXFILTERCACHE

Oracle Text提供了一个名为查询过滤器缓存的缓存层，可用于缓存查询结果。查询过滤器缓存可跨查询共享。因此，缓存的查询结果可以被多个查询重用，从而改善了查询响应时间。CTXFILTERCACHE运算符用于指定要在查询过滤器缓存中缓存的查询结果或部分查询结果。

CTXFILTERCACHE仅支持CONTEXT语法查询。不支持模板查询等CONTAINER查询。如果使用模板查询执行它，则会引发错误。

警告：

在使用CTXFILTERCACHE之前，必须按如下方式运行PURGE recyclebin ：

SQL> PURGE recylebin;

有关清除回收站中对象的完整信息，请参见“ Oracle数据库管理员指南” 。

句法

ctxfiltercache（（ query_text ）[， save_score ] [， topN ]）
query_text

指定其结果需要存储在缓存中的查询。

save_score

如果要将所有查询结果及其分数缓存在缓存中，请指定TRUE 。

默认值为FALSE 。在这种情况下，每个查询结果返回100分，并且这些分数不存储在缓存中。只有查询结果存储在缓存中。

如果要重用查询结果而不是其他查询中的分数，请指定FALSE 。当您使用查询文本作为过滤器（例如安全过滤器）时，这尤其有用，其中查询的缓存部分的相关性不会影响整个查询的相关性。因此，当与AND运算符（返回其操作数的较低分数）一起使用时，分数100不会影响整个查询的分数。

topN

如果只希望将得分最高的查询结果存储在缓存中，请指定TRUE 。Oracle Text在内部确定要在缓存中存储的最高得分查询结果的数量。这有助于减少缓存的内存消耗。

注意：

如果为topN指定TRUE ，则save_score也应为TRUE 。

CTXFILTERCACHE的示例

存储的查询结果和TopN示例

以下示例将common_predicate查询的查询结果存储在缓存中：

select * from docs where contains（txt，'ctxfiltercache（（common_predicate），FALSE）'）> 0;

这里， save_score为FALSE ，因此每个查询结果返回100分，并且分数不存储在缓存中。

在下面的例子中，的缓存结果common_predicate查询被重用new_query查询。 

select * from docs where contains（txt，'new_query＆ctxfiltercache（（common_predicate），FALSE）'）> 0;

将save_score设置为TRUE ，如以下示例所示，以将common_predicate查询的所有查询结果以及实际分数存储在缓存中。 

select * from docs where contains（txt，'ctxfiltercache（（common_predicate），TRUE）'）> 0;

  • 将返回仅包含term1的文档，分数不变。

  • 只有字词1 字词2的文件将不予退还。

  • 与字词1 字词1 字词2的文件将被退回，但比分将仅使用第一个字词1命中计算。

第三个项目符号中描述的行为与NOT的行为不同，后者不返回此类文档。

MNOT运算符比MINUS运算符更具体，因为左子项必须由正确的子项包含。如果不是，Mild Not操作符将忽略正确的子项。此外，对于Mild Not，正确的孩子是一个真正的过滤器，也就是说，它不会简单地减去左孩子和右孩子的分数。

3.18 NEAR（;）

使用NEAR运算符根据两个或多个查询字词的接近度返回分数。对于更接近的术语，Oracle Text会返回更高的分数，而对于文档中较远的术语，Oracle Text会返回较低的分数。如果单词或术语在NEAR查询中出现多次，则该单词必须在文档中出现多次才能匹配。

注意：

NEAR运算符仅适用于单词查询。您不能在ABOUT查询中使用NEAR 。

句法

NEAR（（ word1 ， word2 ，...， wordn ）[，max_span [，order [，maxreqd]]]）

向后兼容性语法：

word1 ; word2
字1-n

在逗号分隔的查询中指定术语。查询术语可以是单个单词或短语，也可以使用其他查询运算符（请参阅“ 与其他运算符一起使用 ” ）。

max_span

（可选）指定分隔丛的开始和结束字的单词数。默认值为100。如果指定的数字大于100，Oracle Text将返回错误。

丛（clump）是出现所有查询词的最小词组。所有团块都以查询字词开头和结尾。

对于具有两个术语的近似查询， max_span是两个术语之间允许的最大距离。例如，如果文档中包含“猫坐在狗身上”，那么您可以使用以下查询在3个字的狗中找到猫 ：

'附近（（狗，猫，3）'

如果文件中包含“猫和兔子坐在狗身上”，那么您可以使用以下查询在6个单词中找到猫 ， 狗和兔子 ：

'附近（（猫，狗，兔），6）'

注意：

搜索词rabbit仍包含在max_span计算中。如果你指定max_span为5，那么你找不到兔子 。替代词也包含在范围计算中。

订单

为Oracle Text指定TRUE以按指定的顺序搜索术语。默认值为FALSE 。

例如，要以最大丛大小为20的顺序搜索单词星期一 ， 星期二和星期三 ，请输入以下查询：

'附近（（星期一，星期二，星期三），20，TRUE）'

注意：

要指定order ，您必须始终为max_span指定一个数字。

当您使用具有不同设置order标志的相同查询表达式时，Oracle Text可能会为同一文档返回不同的分数。例如，当您输入以下查询时，Oracle Text可能会为同一文档返回不同的分数：

'near（（dog，cat），50，FALSE）''near（（dog，cat），50，TRUE）'
maxreqd

从Oracle Database 12 c第2版（12.2）开始，可以使用此新参数。指定必须彼此靠近的术语数导致匹配。如果未指定匹配项必须彼此接近的术语数，则所有术语必须匹配。例如，以下查询匹配包含与fish相关的单词集的文档：

'附近（（鱼，鲨鱼，海洋，鳞片，钓鱼），10，FALSE，3）'

这里，对于匹配，只有三个查询项必须在彼此的距离10之内。

近得分

NEAR运算符的得分将术语的频率与术语的接近度相结合。对于满足查询的每个文档，Oracle Text返回1到100之间的分数，该分数与文档中的块数成比例，与块的平均大小成反比。这意味着文档中的许多小块会导致更高的分数，因为小块意味着术语的接近程度。

查询中的术语数也会影响分数。具有许多术语（例如七个）的查询通常在文档中需要较少的块来获得100而不是具有少量术语的查询，例如两个。

一个 clump是所有查询术语出现的最小单词组。所有团块都以查询字词开头和结尾。使用max_span定义max_span大小 参数，如本节所述。

丛的大小不包括查询术语本身。因此对于查询NEAR((DOG, CAT), 1) ， 狗猫将是匹配，而狗吃了猫将是匹配，但坐在猫上的狗将不会匹配。

与其他运营商相邻

您可以将NEAR运算符与其他运算符（如AND和OR 。分数以常规方式计算。

例如，要查找包含tiger ， lion和cheetah术语的所有文档，其中术语lion和tiger彼此相差10个字，请输入以下查询：

'附近（（狮子，老虎），10）和猎豹'

每个文档返回的分数是近运算符和术语猎豹的较低分数。

您还可以使用等价运算符替换near查询中的单个术语：

'附近（（股票崩盘，日本=韩国），20）'

此查询要求包含日本或韩国二十个字内的短语库存崩溃的所有文档。

  • CTXRULE查询文档不支持SDATA运算符。

  • SDATA运算符对突出显示没有影响。

笔记

Oracle建议仅将SDATA运算符用作也包含非SDATA子SDATA的AND运算符的后代。实质上，使用SDATA运算符作为次要（即检查或非驱动）标准。例如，“查找具有价格> 5的DOG的文档”，而不是“查找评级> 4的文档”。其他用法可能正常运行，但可能没有最佳性能。

SDATA是AND运算符的子代，也有非SDATA子代。

虽然这里的SDATA运算符是OR子代，但它们仍然是具有非SDATA子代的AND运算符的后代。

以下示例显示了不推荐的用法：

在这里， SDATA是唯一的标准，因此也是驾驶标准。

此示例中的SDATA是OR运算符的子SDATA而不是AND 。

要在同一部分标题中查找包含术语dog和cat的所有文档，请按如下方式编写查询：

'（狗和猫）WITHIN标题'

此查询在逻辑上与以下内容不同：

'标题内的狗和标题内的猫'

此查询查找包含dog和cat的所有文档，其中术语dog和cat位于标题部分，无论它们是出现在相同的标题部分还是不同的部分。

靠近WITHIN

要在标题部分中查找狗靠近猫的所有文档，请按如下方式编写查询：

'猫在附近的标题'

注意：

near运算符的优先级高于WITHIN运算符，因此在此示例中不需要括号。此查询等同于（猫附近的猫）WITHIN标题 。

嵌套的WITHIN查询

您可以嵌套内部运算符以搜索区域部分中的区域部分。

例如，假设文档集的区域部分AUTHOR嵌套在区域BOOK部分中。编写嵌套的WITHIN查询以查找BOOK部分的AUTHOR部分中出现的所有scott ，如下所示：

'（在作者内的斯科特）在书内'

在现场部分内查询

在字段部分中查询的语法与在区域部分中查询的语法相同。上一节“ 在区域内查询 ”中给出的大多数示例的语法适用于字段部分。

但是，字段部分的行为与区域部分的行为不同

  • 可见性：使字段部分中的文本不可见。

  • 可重复性： WITHIN查询无法区分重复的字段。

  • 可嵌套性：您不能使用字段部分输入嵌套的WITHIN查询。

以下部分描述了这些差异。

野外部分的可见旗帜

在CTX_DDL.ADD_FIELD_SECTION创建可见标志为FALSE的字段部分时，只能使用WITHIN运算符查询字段部分中的文本。

例如，假设TITLE是将可见标志设置为FALSE定义的字段部分。然后查询狗没有 在WITHIN运营商将无法找到包含文件：

<TITLE>狗</ TITLE>我喜欢我的宠物。

要查找此类文档，请使用WITHIN运算符，如下所示：

'狗在'TITLE'

或者，将TITLE定义为具有CTX_DDL.ADD_FIELD_SECTION的字段部分时，将visible标志设置为TRUE 。

重复的场部分

WITHIN查询无法区分文档中的重复字段。例如，考虑带有重复部分<author>的文档：

<作者>查尔斯狄更斯</ author> <作者>马丁路德金</ author>

假设<author>被定义为字段部分，作者内的查询（如charles和martin）将返回文档，即使这些单词出现在单独的标记中。

要让WITHIN查询区分重复的部分，请将这些部分定义为区域部分。

嵌套字段

您不能使用字段部分输入嵌套的WITHIN查询。这样做会引发错误。

在句子或段落中查询

在句子或段落边界内查询对于查找出现在同一句子或段落中的单词组合很有用。要查询句子或段落，必须先将特殊部分添加到部分组，然后再编制索引。使用CTX_DDL.ADD_SPECIAL_SECTION执行此CTX_DDL.ADD_SPECIAL_SECTION 。

要查找包含单词dog但不包含cat的句子的文档：

'（狗不是猫）在室内'

查询属性部分

使用XML_SECTION_GROUP或AUTO_SECTION_GROUP作为节组类型进行索引时，在属性节中进行查询。

假设您有一个XML文档，如下所示：

<书名=“双城记”>这是最好的时期。</ book>

将节title@book定义为属性节title 。使用CTX_DLL.ADD_ATTR_SECTION过程或使用ALTER INDEX后动态执行此操作。

注意：

当您使用AUTO_SECTION_GROUP索引XML文档时，系统会自动创建属性部分，并在表单attribute@tag中为它们命名。

如果使用XML_SECTION_GROUP ，则可以使用XML_SECTION_GROUP将属性部分命名为任何CTX_DDL.ADD_ATTR_SECTION 。

要在属性部分title内搜索Tale ，请输入以下查询：

'故事中的故事'

查询属性部分的约束

以下约束适用于在属性部分中查询：

  • 除非在within子句中限定，否则对属性文本的常规查询不会命中文档。假设您有一个XML文档，如下所示：

<书名=“双城记”>这是最好的时期。</ book>

除非使用WITHIN title@book限定，否则对Tale的查询本身不会对文档产生任何影响。（当您将visible标志设置为false时，此行为类似于字段部分。）

  • 您不能在嵌套的WITHIN查询中使用属性部分。

  • 短语忽略属性文本。例如，如果原始文档看起来像：

现在是时候让所有好的<word type =“noun”>男人</ word>来帮助。

然后这个文档会打到常规查询好人 ，忽略介入的属性文本。

  • WITHIN查询可以区分重复的属性部分。此行为类似于区域部分，但与字段部分不同。例如，您有一个文档如下：

<书名=“双城记”>这是最好的时期。</ book> <book title =“人类束缚”>天空变得暗淡无光。</ book>

假设book是区域部分， book@author是属性部分。考虑查询：

'（Tale and Bondage）WITHIN book @ author'

此查询未命中文档，因为故事和束缚出现在属性部分book@author 。

笔记

部分名称

WITHIN运算符要求您知道所搜索部分的名称。可以使用CTX_SECTIONS或CTX_USER_SECTIONS视图获取已定义部分的列表。

部分边界

对于特殊区域和区域部分，查询的术语必须完全包含在文档的特定实例中，以满足查询的要求。这不是现场部分的要求。

例如，考虑粗体是区域部分的查询：

'（狗和猫）inin bold'

此查询找到：

<B>狗猫</ B>

但它找不到：

<B>狗</ B> <B>猫</ B>

这是因为狗和猫必须在同一个粗体部分。

此行为对于特殊部分尤其有用

'（狗和猫）在'句子'

意思是在同一句话中找到狗和猫 。

另一方面，字段部分用于非重复的嵌入式元数据，例如标题部分。字段部分内的查询无法区分事件。所有出现的字段都被认为是单个部分的一部分。例如，查询：

（狗和猫）在标题内

可以找到这样的文件：

<TITLE>狗</ TITLE> <TITLE>猫</ TITLE>

作为此字段部分限制以及重叠和嵌套限制的回报，字段部分查询通常比区域部分查询更快，特别是如果该部分出现在每个文档中，或者搜索术语是常见的。
