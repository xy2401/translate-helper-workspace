使用DBCA为Oracle RAC生成安装脚本

查看此主题以生成用于创建Oracle RAC数据库的脚本，使用生成的脚本创建数据库，以及准备数据库以供使用。

完成以下步骤：

  1. 启动DBCA并选择构建Oracle RAC数据库的首选选项。

    在DBCA会话的“创建选项”页面上，取消选择“创建数据库”，然后在“ 完成”之前选择“ 生成数据库创建脚本” 。您可以接受脚本的默认目标目录，也可以浏览其他位置。在任何一种情况下，请记下下一步中使用的路径名称。

  2. 转到DBCA创建脚本的目录，并查看SQL脚本以确保它们包含用于构建具有所需特征的数据库的语句。如果他们不这样做，那么Oracle建议您重新运行DBCA以创建具有所需配置的脚本，而不是自己编辑脚本。
  3. 在DBCA会话期间标识的每个群集节点上，运行脚本sid .sh ，其中sid是您在DBCA Database Name页面上输入的SID前缀。
  4. 通过在SQL * Plus中运行ALTER SYSTEM语句，或者在每个实例的SPFILE中取消注释参数，将初始化参数cluster_database设置为SPFILE中的值TRUE 。
  5. 配置Oracle Net Services以支持新的数据库和实例。
  6. 通过在SQL * Plus中输入ALTER SYSTEM语句，或者在每个实例的PFILE中取消注释参数，将REMOTE_LISTENER参数设置为REMOTE_LISTENER中的SCAN（使用Easy Connect命名语法scanname:scanport ）。
  7. 运行服务器控制实用程序（SRVCTL）以配置和启动数据库和实例应用程序，如以下文档中所述：

    “Oracle Real Application Clusters管理和部署指南”
