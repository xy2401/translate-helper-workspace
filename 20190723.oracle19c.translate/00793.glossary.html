词汇表

属性

元素的属性，由名称和由等号分隔的值组成，并包含在元素名称后面的start-tags中。

在此示例中， <Price units='USD'>5</Price> ，单位是属性，USD是其值，必须使用单引号或双引号。属性可以驻留在文档或文档类型定义（DTD）中。元素可能具有许多属性，但未定义其检索顺序。

二进制XML

一种可扩展标记语言（XML）表示，它使用紧凑的XML模式识别格式。

打回来

一种编程技术，其中一个进程启动另一个进程然后继续。然后，第二个进程会根据操作，值或其他事件调用第一个进程。在具有用户界面以允许连续交互的大多数程序中使用该技术。

盒式磁带

用Java或过程语言/结构化查询语言（PL / SQL）存储的程序，它为数据库添加必要的功能以理解和操作新的数据类型。

Cartridges通过Oracle XML Developer Kit（XDK）实现Java绑定（JAXB）规范版本8或更高版本中的扩展性框架进行接口。Oracle Text就是这样一个盒式磁带，增加了对读取，写入和搜索存储在数据库中的文本文档的支持。

另请参见Oracle Text 。

层叠样式表（CSS）

见CSS 。

CDATA

人物数据。必须不解析的文档中的文本包含在CDATA部分中。这允许包含否则将具有特殊功能的字符，例如& ， <和> 。CDATA部分可用于元素或属性的内容。

字符数据（CDATA）

见CDATA 。

儿童元素

一个完全包含在另一个元素中的元素，称为其父元素。例如， <Parent><Child></Child></Parent>显示嵌套在其父元素中的子元素。

另请参见父元素 。

类生成器

类生成器接受输入文件并创建一组具有相应功能的输出类。对于XML类生成器，输入文件是DTD或XML模式，输出是一系列可用于创建符合XML文档的类。

CLASSPATH

Java虚拟机（JVM）用于查找运行应用程序所需的类的操作系统环境变量。

CSS

层叠样式表。用于向Web文档添加样式（字体，颜色，间距等）的简单机制。

数据报

DDL

数据定义语言。定义或更改数据结构的语句。

DOCTYPE

该术语用作标记名称，用于指定XML文档中的DTD或其引用。例如， <!DOCTYPE person SYSTEM "person.dtd">在文件系统中将根元素名称声明为person，将外部DTD声明为person.dtd。内部DTD在DOCTYPE声明中声明。

文档对象模型（DOM）

见DOM 。

文档类型定义（DTD）

见DTD 。

DOM

文档对象模型。XML文档的内存中基于树的对象表示，支持以编程方式访问其元素和属性。

文档对象模型（DOM）对象及其接口是万维网联盟（W3C）建议，它指定XML文档的DOM，包括用于编程访问的应用程序编程接口（API）。DOM将解析的文档视为对象树。

DTD

文档类型定义。一组规则，用于定义XML文档的有效结构。DTD是从SGML派生其格式的文本文件。可以使用DOCTYPE元素或通过DOCTYPE引用使用外部文件将DTD包含在XML文档中。

元件

XML文档的基本逻辑单元，可用作其他元素的容器，例如子元素，数据，属性及其值。元素由开始标记（例如<name> ）和结束标记（例如</name> ）或空元素<name/>标识。

空元素

没有文本内容或子元素的元素。它只能包含属性及其值。空元素的格式为<name/>或<name></name> ，其中标记之间没有空格。

实体

一串字符，可以表示另一个字符串或不属于文档字符集的特殊字符。实体和解析器替换它们的文本在DTD中声明。

结语

XML文档的结束部分。epilog是可选的。

可扩展样式表语言。样式表中用于转换或呈现XML文档的语言。两个W3C建议涵盖XSL样式表：XSL转换（XSLT）和XSL格式化对象（XSL-FO）。

  • XSLT是一种将一个XML文档转换为另一个XML文档的语言。

  • XSL-FO是用于指定XML文档表示的XML词汇表。

XSL样式表通过描述如何将类的实例转换为使用格式化词汇表的XML文档来指定一类XML文档的表示。

XSQL servlet

基于Java的servlet，可以从一个或多个SQL查询动态生成XML文档，并可选择使用XSLT样式表转换服务器中的文档。

XSU

XML SQL实用程序。给定SQL查询或JDBC ResultSet对象时可以生成XML文档（字符串或DOM）的Oracle实用程序。XSU还可以从XML文档中提取数据，然后在数据库表中插入，更新或删除行。

XVM

XSLT虚拟机。也称为XSLTVM和XSLT VM。 Oracle XSLT虚拟机是CPU的软件实现，旨在运行已编译的XSLT代码。虚拟机概念假定编译器将XSLT样式表编译为字节码程序或XSLT CPU的机器指令。
