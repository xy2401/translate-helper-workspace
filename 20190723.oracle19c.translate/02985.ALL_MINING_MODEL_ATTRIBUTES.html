37.2 ALL_MINING_MODEL_ATTRIBUTES

ALL_MINING_MODEL_ATTRIBUTES描述了当前用户可访问的挖掘模型的属性。 此视图中仅包含模型签名中的属性。模型签名中的属性对应于用于构建模型的训练数据中的列。

挖掘模型是Oracle Data Mining创建的模式对象。

相关观点

  • DBA_MINING_MODEL_ATTRIBUTES描述数据库中所有挖掘模型的属性。

  • USER_MINING_MODEL_ATTRIBUTES描述当前用户拥有的挖掘模型的属性。此视图不显示OWNER列。

ATTRIBUTE_TYPE

VARCHAR2（11）

-

属性的逻辑类型。在模型构建或应用过程中标识类型：

  • NUMERICAL ：数字数据

  • CATEGORICAL ：字符数据

  • TEXT ：非结构化文本数据

  • PARTITION ：输入签名列用于分区键

  • MIXED ：输入签名列采用多种属性类型。

    这是由于用户定义的嵌入式转换允许将输入列转换为多个独立的挖掘属性，包括不同类型的挖掘属性。

USAGE_TYPE

VARCHAR2（8）

-

指示该属性是否用于构建模型（ ACTIVE ）或不是（ INACTIVE ）。可以通过变换或算法处理来消除一些属性。*_MINING_MODEL_ATTRIBUTES视图仅列出模型使用的属性，因此此列的值始终为ACTIVE 。

目标

VARCHAR2（3）

-

指示属性是否是预测模型的目标（ YES ）或否（ NO ）。目标描述应用模型时生成的结果。

ATTRIBUTE_SPEC

VARCHAR2（4000）

-

一个或多个关键字，用于在模型构建期间标识属性的特殊处理。价值观是：

  • FORCE_IN :(仅限GLM）启用功能选择后，强制在模型构建中包含该属性。默认情况下禁用功能选择。如果模型未使用GLM并启用了功能选择，则忽略此值。

  • NOPREP ：当ADP打开时，阻止属性的自动转换。如果ADP为OFF ，则忽略该值。

  • TEXT ：使该属性被视为非结构化文本数据。TEXT值支持三个子集： POLICY_NAME ， MAX_FEATURES ， TOKEN_TYPE和MIN_DOCUMENTS 。子集在括号内指定为名称：值对。例如： (POLICY_NAME:mypolicy) (MAX_FEATURES:2000)(TOKEN_TYPE:THEME) 。有关详细信息，请参见Oracle Data Mining用户指南 。

  • NULL ：此属性的ATTRIBUTE_SPEC为NULL 。

    ATTRIBUTE_SPEC是PL / SQL过程DBMS_DATA_MINING_TRANSFORM.SET_TRANSFORM的参数。有关详细信息，请参见Oracle Database PL / SQL包和类型参考 。
