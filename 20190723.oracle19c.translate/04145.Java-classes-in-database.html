2.4关于数据库中加载的Java类

如果未使用命令行界面，则必须将Java文件作为模式对象加载到数据库中，以使其可供Oracle JVM使用。 loadjava工具可以调用Oracle JVM的Java编译器，它将源文件编译为标准类文件。

下图显示loadjava工具可以设置存储在系统数据库表中的选项的值。除其他外，这些选项会影响Java源文件的处理。

每个Java类都存储为架构对象。对象的名称派生自类的完全限定名称，其中包含包含包的名称。例如， Handle类的全名是：

oracle.aurora.rdbms。处理

在Java模式对象名称中，斜杠替换句点，因此该类的全名变为：

ORACLE /极光/ RDBMS /把手

Oracle数据库接受最多4000个字符的Java名称。但是，Java架构对象的名称不能超过30个字符。因此，如果架构对象名称超过30个字符，则系统会为架构对象生成短名称或别名。否则，将使用完全限定名称（也称为全名）。您可以在需要它的任何上下文中指定全名。需要时，名称映射由Oracle数据库处理。
