升级后向Oracle Enterprise Manager注册资源

升级Oracle Grid Infrastructure后，将新资源目标添加到Oracle Enterprise Manager Cloud Control。

在Oracle Grid Infrastructure升级后，在Oracle Enterprise Manager中发现并添加新的资源目标。以下过程提供了升级Oracle Grid Infrastructure后发现Oracle ASM侦听器目标的示例：
  1. 登录Oracle Enterprise Manager Cloud Control。
  2. 从“ 设置”菜单中，选择“ 添加目标” ，然后选择“ 手动添加目标” 。
    将显示“ 手动添加目标”页面。
  3. 在Add Targets页面中，选择Add Using Guided Process选项和Target Type as Oracle Database, Listener and Automatic Storage Management 。
    对于要添加的任何其他资源，请在Oracle Enterprise Manager发现向导中选择适当的目标类型 。
  4. 单击“ 使用引导过程添加” 。
    将显示“目标发现”向导。
  5. 对于“ 指定主机”或“群集”字段，单击“ 搜索”图标并搜索“ 目标 Hosts 类型 ”，然后选择相应的主机。
  6. 单击下一步 。
  7. 在Target Discovery：Results页面中，选择已发现的Oracle ASM Listener目标，然后单击Configure 。
  8. 在“配置侦听器”对话框中，指定侦听器属性，然后单击“ 确定” 。
  9. 单击“ 下一步”并完成发现过程。
    在Oracle Enterprise Manager中发现侦听器目标，状态为Down。
  10. 从“ 目标”菜单中，选择目标类型。
  11. 单击目标名称以导航到目标主页。
  12. 从目标主页上显示的主机，数据库，中间件目标或应用程序菜单中，选择“ 目标设置” ，然后选择“ 监视配置” 。
  13. 在侦听器的“监视配置”页面中，在“ 计算机名”字段中指定主机名，在“ 密码”字段中指定ASMSNMP用户的密码 。
  14. 单击确定 。
将显示具有正确状态的Oracle ASM侦听器目标。
同样，您可以在Oracle Grid Infrastructure升级后将其他集群件资源添加到Oracle Enterprise Manager。
