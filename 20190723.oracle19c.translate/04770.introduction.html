1.1 OCI概述

Oracle调用接口（OCI）是一个应用程序编程接口（API），它允许您创建使用函数调用来访问Oracle数据库并控制SQL语句执行和数据访问的所有阶段的应用程序。

OCI支持C和C ++的数据类型，调用约定，语法和语义。

OCI提供：

  • 通过有效使用系统内存和网络连接实现高性能和可扩展性

  • 用于双层客户端/服务器或多层环境中的动态会话和事务管理的一致接口

  • N层身份验证

  • 使用Oracle数据库对象全面支持应用程序开发

  • 访问外部数据库，例如Oracle TimesTen内存数据库和Oracle内存数据库缓存。请参阅Oracle TimesTen内存数据库C开发人员指南 。

  • 无需额外硬件投资即可支持越来越多用户和请求的应用程序

  • 使用C编程语言操作Oracle数据库中的数据和模式的方法，以及动态运行时库（OCI库）形式的标准数据库访问和检索函数库，可以在运行时在应用程序中进行链接。

  • 封装或不透明接口，其实现细节未知

  • 简化的用户身份验证和密码管理

  • 扩展以提高应用程序性能和可伸缩性

  • 一致的事务管理界面

  • OCI扩展，支持客户端访问Oracle对象

  • 与访问Oracle数据库的其他方法相比具有显着优势：

    • 对应用程序设计的所有方面进行更细粒度的控制

    • 对程序执行的高度控制

    • 使用熟悉的第三代语言编程技术和应用程序开发工具，例如浏览器和调试器

    • 连接池，会话池和语句缓存，可以构建可伸缩的应用程序

    • 支持动态SQL

    • 可在所有Oracle编程接口的最广泛的操作系统上使用

    • 动态绑定和使用回调定义

    • 用于公开服务器元数据层的描述功能

    • 已注册客户端应用程序的异步事件通知

    • 用于阵列插入，更新和删除的增强阵列数据操作语言（DML）功能

    • 能够将提交请求与执行相关联以减少往返次数

    • 使用透明预取缓冲区优化查询以减少往返次数

    • 线程安全，无需OCI句柄上的互斥锁（互斥锁）

    • 用于设计可扩展的多线程应用程序的API，可以安全地支持大量用户

    • SQL访问函数，用于管理数据库访问，处理SQL语句以及操作从Oracle数据库检索的对象

    • 数据类型映射和操作函数，用于操作Oracle类型的数据属性

    • 数据加载函数，用于直接将数据加载到数据库中而不使用SQL语句

    • 外部过程函数，用于从PL / SQL编写C回调

1.2建立OCI申请

您编译和链接OCI程序的方式与编译和链接非数据库应用程序的方式相同。不需要单独的预处理或预编译步骤。

OCI支持大多数流行的编译器。链接OCI程序的细节因系统而异。在某些操作系统上，除了OCI库之外，可能还需要包含其他库来正确链接您的OCI程序。有关为操作系统编译和链接OCI应用程序的详细信息，请参阅Oracle数据库系统特定的文档和安装指南。

1.3 OCI的替代方案

使用Oracle调用接口（OCI）的一些替代方法包括：

  • 用于C的Oracle数据库编程接口（ODPI-C）

  • Oracle C ++调用接口（OCCI）

用于C的Oracle数据库编程接口（ODPI-C）

ODPI-C是一个C代码的开源库，它简化了Oracle数据库驱动程序和用户应用程序的常用Oracle调用接口（OCI）功能的使用。ODPI-C位于OCI之上，需要Oracle客户端库。ODPI-C：

  • 对于不需要利基特殊情况OCI功能的驱动程序编写者而言，更容易使用。

  • 使用相当少的代码提供更快的驱动程序实现。Oracle功能可以以一致的方式快速向用户公开。

  • 为绑定变量和提取提供更简单的内存管理。

  • 自动将绑定和“定义”（对于提取）转换为“本机”C类型，以便不需要进行其他调用。这对数字和日期特别有用。将LONG和LOB列作为字符串和缓冲区检索的能力也是一个优势。

  • 提供“更安全”的API，验证资源句柄。不需要演员表。引用计数机制通过阻止应用程序销毁正在使用的OCI资源来增加弹性。

  • 提供简化连接和资源管理的API。例如，它会自动执行会话池ping操作以提供更好的高可用性。

  • 提供OCI的替代编程体验，使用多个getter和setter模型来处理属性。

  • 提供将ODPI-C构建为共享库的示例Makefile。或者，ODPI-C源代码可以包含在您的项目中，并构建为构建OCI应用程序。

OCI应用程序的主要任务之一是处理SQL语句。

不同类型的SQL语句需要程序中的不同处理步骤。在编写OCI应用程序时，必须考虑到这一点。Oracle数据库可识别几种类型的SQL语句：

  • 数据定义语言 （DDL）

  • 控制声明

    • 会话控制

    • 系统控制

  • 查询

  • 数据处理语言 （DML）

  • PL / SQL

  • 嵌入式SQL

1.4.1数据定义语言

数据定义语言（DDL）语句管理数据库中的模式对象。

DDL语句创建新表，删除旧表并建立其他模式对象。它们还控制对架构对象的访问。

以下是创建和指定对表的访问的示例：

CREATE TABLE employees（名称VARCHAR2（20），ssn VARCHAR2（12），empno NUMBER（6），mgr NUMBER（6），薪水NUMBER（6））;授予更新，插入，删除员工权限;对员工进行最新更新来自jamie;

DDL语句还允许您使用Oracle数据库中的对象，如以下一系列创建对象表的语句：

CREATE TYPE person_t AS OBJECT（名称VARCHAR2（30），ssn VARCHAR2（12），地址VARCHAR2（50））; CREATE TABLE person_tab OF person_t;

1.4.4查询

查询是从数据库中检索数据的语句。

查询可以返回零行，一行或多行数据。所有查询都以SQL关键字SELECT开头，如下例所示：

SELECT dname FROM dept WHERE deptno = 42;

查询表中的访问数据，它们通常使用DML语句进行分类。但是，OCI应用程序以不同方式处理查询，因此在本指南中将单独考虑它们。

查询可能要求程序使用输入（绑定）变量向数据库提供数据，如下例所示：

SELECT name FROM employees WHERE empno =：empnumber;

在前面的SQL语句中， :empnumber是应用程序提供的值的占位符。

处理查询时，OCI应用程序还必须定义输出变量以接收返回的结果。在前面的语句中，您必须定义一个输出变量以接收从查询返回的任何name值。

1.4.5 PL / SQL

PL / SQL是Oracle对SQL语言的过程扩展。

PL / SQL处理比简单查询和SQL数据操作语言语句更复杂的任务。PL / SQL允许将一些构造分组到一个块中并作为一个单元执行。其中包括：

  • 一个或多个SQL语句

  • 变量声明

  • 作业陈述

  • 程序控制声明（IF ...然后...ELSE语句和循环）

  • 异常处理

您可以在OCI程序中使用PL / SQL块来：

  • 调用Oracle数据库存储过程和存储的函数

  • 将过程控制语句与多个SQL语句组合在一起，以便它们作为一个单元执行

  • 访问特殊的PL / SQL功能，例如记录，表，游标FOR循环和异常处理

  • 使用游标变量

  • 利用隐式结果集功能，允许重用返回隐式结果集的现有存储过程设计

  • 访问和操作Oracle数据库中的对象

在给定特定员工编号的情况下，以下PL / SQL示例发出SQL语句以从员工表中检索值。此示例还演示了在PL / SQL语句中使用占位符。

BEGIN SELECT ename，sal，comm INTO：emp_name，：salary，：commission FROM emp WHERE empno =：emp_number;结束;

请注意，此语句中的占位符不是PL / SQL变量。它们表示处理语句时传递给数据库的输入值。这些占位符必须绑定到程序中的C语言变量。

1.4.7 SQL的特殊OCI条款

本指南使用特殊术语来引用SQL语句的不同部分。

例如，请考虑以下SQL语句：

SELECT客户，地址FROM customers WHERE bus_type ='SOFTWARE'和sales_volume =：sales;

它包含以下部分：

  • 一个SQL 命令 - SELECT

  • 两个选择列表项 - customer和address

  • FROM子句中的表名 - customers

  • WHERE子句中的两个列名称 - bus_type和sales_volume

  • WHERE子句中的文字输入值 - ' SOFTWARE '

  • WHERE子句中输入变量的占位符 - :sales

在开发OCI应用程序时，可以调用为Oracle数据库指定程序输入和输出变量的地址（位置）的例程。在本指南中，为数据输入指定占位符变量的地址称为绑定操作 。指定接收选择列表项的变量的地址称为定义操作 。

对于PL / SQL，输入和输出规范都称为绑定操作。

1.5程序和非程序要素

OCI使您能够在多层体系结构中开发可伸缩的多线程应用程序，该体系结构将结构化查询语言（SQL）的非过程数据访问功能与C和C ++的过程功能相结合。

  • 在非过程语言程序中，指定要操作的数据集，但是未指定要执行的操作或如何执行操作。SQL的非过程性质使其成为一种易于学习和用于执行数据库事务的语言。它也是用于访问和操纵现代关系数据库和对象关系数据库系统中的数据的标准语言。

  • 在过程语言程序中，大多数语句的执行取决于先前或后续语句以及SQL中不可用的控制结构（例如循环或条件分支）。这些语言的程序性使它们比SQL更复杂，但它也使它们更加灵活和强大。

OCI程序中的非过程语言元素和过程语言元素的组合使得可以在结构化编程环境中轻松访问Oracle数据库。

OCI支持通过Oracle数据库提供的所有SQL数据定义，数据操作，查询和事务控制工具。例如，OCI程序可以对Oracle数据库运行查询。查询可能要求程序使用输入（绑定）变量向数据库提供数据，如下所示：

SELECT name FROM employees WHERE empno =：empnumber;

在前面的SQL语句中， :empnumber是应用程序提供的值的占位符。

您还可以利用PL / SQL，Oracle对SQL的过程扩展。您开发的应用程序比仅使用SQL编写的应用程序更强大，更灵活。OCI还提供访问和操作数据库中对象的工具。

1.6对象支持

OCI具有处理对象类型和对象的工具 。

对象类型是表示现实世界实体的抽象的用户定义的数据结构。例如，数据库可能包含person对象的定义。该对象可能具有属性 - first_name ， last_name和age - 表示人员的标识特征。

对象类型定义用作通过使用对象类型作为结构定义来创建表示对象类型实例的对象的基础，您可以创建具有属性值'John'，'Bonivento'和'30'的person对象。对象类型还可以包含表示该对象类型行为的方法 - 程序函数。

OCI为寻求使用Oracle数据库对象功能的程序员提供了全面的应用程序编程接口。

这些功能包括：

  • 执行操作对象数据和架构信息的SQL语句

  • 在SQL语句中将对象引用和实例作为输入变量传递

  • 将对象引用和实例声明为变量以接收SQL语句的输出

  • 从数据库中获取对象引用和实例

  • 描述返回对象实例和引用的SQL语句的属性

  • 用对象参数或结果描述PL / SQL过程或函数

  • 扩展提交和回滚调用以同步对象和关系功能

OCI对象功能可分为以下几大类：

  • 客户端对象缓存

  • 用于访问和操作对象的关联和导航接口

  • 对象的OCI运行时环境

  • 类型管理：映射和操作函数，用于访问有关Oracle类型的对象类型和控制数据属性的信息

  • 对象类型转换器 （OTT）实用程序，用于将内部Oracle数据库架构信息映射到客户端语言绑定变量

1.6.1客户端对象缓存

对象缓存是客户端内存缓冲区，为对象提供查找和内存管理支持。

对象缓存存储和跟踪OCI应用程序从服务器提取到客户端的对象实例。初始化OCI环境时会创建对象缓存。当多个应用程序对同一服务器运行时，每个应用程序都有自己的对象缓存缓存跟踪当前在内存中的对象，维护对象的引用，管理自动对象交换，以及跟踪元属性或键入有关对象的信息。对象缓存为OCI应用程序提供以下功能：

  • 通过减少获取和操作对象所需的客户端/服务器往返次数来提高应用程序性能

  • 通过支持从客户端缓存进行对象交换来增强可伸缩性

  • 通过支持对象级锁定来改进并发性

1.6.2关联和导航接口

OCI应用程序可以使用哪些不同类型的接口来访问对象？

使用OCI的应用程序可以通过几种类型的接口访问Oracle数据库中的对象：

  • 使用SQL SELECT ， INSERT和UPDATE语句

  • 使用C风格的指针追逐方案通过遍历相应的智能指针或REF来访问客户端缓存中的对象

OCI提供了一组带有扩展的函数，以支持使用SQL SELECT ， INSERT和UPDATE语句进行对象操作。要访问Oracle数据库对象，这些SQL语句使用一组一致的步骤，就像访问关系表一样。OCI提供访问对象所需的以下功能集：

  • 将对象类型实例和引用绑定和定义为SQL语句的输入和输出变量

  • 执行包含对象类型实例和引用的SQL语句

  • 获取对象类型实例和引用

  • 描述Oracle对象类型的选择列表项

OCI还提供了一组函数，使用C风格的指针追逐方案，在通过遍历相应的智能指针或REF将对象提取到客户端缓存后访问对象。该导航界面提供以下功能：

  • 通过固定其智能指针或REF实例化可引用的持久对象（即客户端缓存中具有对象ID的持久对象）的副本

  • 遍历通过遍历从一个到另一个指向的REF彼此连接的一系列对象

  • 动态获取和设置对象属性的值

1.6.3对象的OCI运行时环境

OCI为对象提供管理Oracle数据库对象在客户端的使用方式的功能。

这些功能提供：

  • 连接到Oracle数据库服务器以访问其对象功能，包括初始化会话，登录数据库服务器以及注册连接

  • 设置客户端对象缓存并调整其参数

  • 获取错误和警告消息

  • 控制访问数据库中对象的事务

  • 通过SQL关联访问对象

  • 描述其参数或结果为Oracle类型的PL / SQL过程或函数

1.6.4类型管理：映射和操作功能

OCI提供了两组函数来处理Oracle数据库对象。

  • 类型映射函数允许应用程序将服务器中表示的Oracle模式的属性作为内部Oracle数据类型映射到其对应的主机语言类型。

  • 类型操作函数允许宿主语言应用程序操纵Oracle模式的各个属性，例如设置和获取其值以及将其值刷新到服务器。

此外， OCIDescribeAny()函数提供有关存储在数据库中的对象的信息。

1.6.5对象类型转换器

对象类型转换器（OTT）实用程序将有关Oracle对象类型的架构信息转换为主机语言变量（如结构）的客户端语言绑定。

OTT将包含有关Oracle模式对象的元数据信息的intype文件作为输入。它生成一个outtype文件以及必须包含在针对对象模式运行的C应用程序中的头文件和实现文件。OCI应用程序和Pro * C / C ++预编译器应用程序都可以包括OTT生成的代码。 OTT是有益的，因为它：

  • 提高应用程序开发人员的工作效率：OTT使您无需编写与架构对象相对应的主机语言变量。

  • 保持SQL作为首选的数据定义语言：通过提供自动映射使用SQL创建的Oracle模式对象来托管语言变量的功能，OTT便于使用SQL作为首选的数据定义语言。这反过来又允许Oracle数据库支持一致的数据模型。

  • 促进对象类型的模式演变：OTT在模式更改时重新生成包含的头文件，允许Oracle应用程序支持模式演变。

通常通过指定intype文件， outtype文件和特定数据库连接从命令行调用OTT。使用Oracle数据库，OTT只能生成可以与OCI程序一起使用或与Pro * C / C ++预编译器程序一起使用的C结构。

1.7简单的Oracle文档访问（SODA）

SODA for C是一个C API，是Oracle Call Interface（OCI）的一部分。

SODA for C实现了简单的Oracle文档访问（SODA）。您可以使用它对任何类型的文档执行创建，读取（检索），更新和删除（CRUD）操作，并且可以使用它来查询JSON文档。您编译使用SODA for C的程序与编译其他OCI程序的方式相同。SODA是一组NoSQL风格的API，允许您在Oracle数据库中创建和存储文档集合，检索它们并查询它们，而无需了解结构化查询语言（SQL）或文档中的数据如何存储在数据库。Oracle数据库支持存储和查询JSON数据。要访问此功能，您需要使用带有特殊JSON SQL运算符的结构化查询语言（SQL）。SODA for C隐藏了SQL / JSON编程的复杂性。

1.8封装接口

OCI调用使用的所有数据结构都以不透明接口的形式封装，这些接口称为句柄。

句柄是指向由OCI库分配的存储区域的不透明指针，该存储区域存储有关SQL或PL / SQL语句的上下文信息，连接信息，错误信息或绑定信息。客户端分配某些类型的句柄，通过定义良好的接口填充这些句柄中的一个或多个，并使用这些句柄向服务器发送请求。反过来，应用程序可以使用访问器函数访问句柄中包含的特定信息。

OCI库管理句柄层次结构。使用这些句柄封装OCI接口对应用程序开发人员有几个好处，包括：

  • 减少必须保留的服务器端状态信息，从而减少服务器端内存使用量

  • 通过消除对全局变量的需求，简化错误报告以及提供OCI变量访问和使用方式的一致性来提高生产率

  • 允许在不影响应用程序的情况下对底层结构进行更改

1.9用户认证和密码管理

OCI为应用程序开发人员提供用户身份验证和密码管

这有以下几种方式支持：

  • OCI允许单个OCI应用程序对多个用户进行身份验证和维护。

  • OCI使应用程序能够更新用户的密码，如果验证尝试返回过期的密码消息，这将特别有用。

OCI支持两种类型的登录会话：

  • 用于会话的登录功能，单个用户使用登录名和密码连接到数据库

  • 单个OCI应用程序通过将登录会话（用户登录到Oracle数据库时创建的会话）与用户会话（用户创建的所有其他会话）分开来对多个会话进行身份验证和维护的机制

还支持特权连接，例如SYSDBA ， SYSOPER ，代理身份验证，外部身份验证等。

1.10提高应用程序性能和可伸缩性的功能

OCI提供了多种功能扩展，以提高应用程序性能和可伸缩性。

通过减少所需的客户端到服务器往返次数来提高应用程序性能，并且通过减少必须在服务器端保留的状态信息量来提高可扩展性。其中一些功能包括：

  • 语句缓存通过缓存重复使用的可执行语句来提高性能

  • 客户端结果缓存以限制到数据库服务器的往返次数

  • 隐式预取SELECT语句结果集以消除描述往返，减少往返次数并减少内存使用

  • 消除打开和关闭光标往返

  • 支持多线程环境

  • 通过连接进行会话复用

  • 对各种配置的一致支持，包括标准的双层客户端/服务器配置，服务器到服务器事务协调和三层事务处理（TP） - 监视器配置

  • 对本地和全局事务的一致支持，包括对XA接口的TM_JOIN操作的支持

  • 通过提供在连接上跨用户集中连接，进程和会话的能力以及消除为全局事务的每个分支创建单独会话的需要，提高了可伸缩性

  • 允许应用程序对多个用户进行身份验证，并允许代表他们启动事务
