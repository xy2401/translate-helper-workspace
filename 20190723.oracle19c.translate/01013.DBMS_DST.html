62 DBMS_DST

DBMS_DST包提供了一个接口，用于将夏令时（DST）修补程序应用于具有时区数据类型的时间戳。

62.1 DBMS_DST概述

在处理带有时区数据的时间戳时，夏令时生效或停止生效的过渡期可能会出现问题，例如数据丢失。DBMS_DST包允许在一组规则的上下文中处理这些转换。

62.2 DBMS_DST安全模型

DBMS_DST包是一个调用者权限包。

包的执行权限被授予EXECUTE_CATALOG_ROLE角色。通常授予选定用户此角色，以允许数据字典中的包和过程具有EXECUTE权限。

调用程序包的用户必须具有以下权限：

  • CREATE ANY TABLE

  • ALTER ANY TABLE

  • DROP ANY TABLE

  • SELECT ANY TABLE

  • LOCK ANY TABLE

  • ALTER ANY INDEX

  • ALTER ANY TRIGGER

  • UPDATE ANY TABLE

  • EXECUTE ANY TYPE

62.4.1 BEGIN_PREPARE程序

此过程启动准备窗口。成功启动准备窗口后，数据库属性“ DST_UPGRADE_STATE ”将设置为“ PREPARE ”，数据库属性“ SECONDARY_TT_VERSION ”将设置为新的时区版本。

准备窗口允许DBA调查受升级影响的数据，从而判断何时执行升级是最佳的。准备窗口可以与正常的数据库操作重叠。

句法

DBMS_DST.BEGIN_PREPARE（new_version IN BINARY_INTEGER）;

参数

表62-3 BEGIN_PREPARE过程参数

参数 描述

新版本

要为其升级数据库的新时区版本

62.4.2 BEGIN_UPGRADE程序

此过程将启动升级窗口。

升级窗口成功启动后，字典表中的TSTZ数据将升级以反映新的时区版本，数据库属性'DST_UPGRADE_STATE'将设置为'UPGRADE' 。成功执行BEGIN_UPGRADE ，用户必须重新启动数据库。成功重新启动后，数据库属性'PRIMARY_TT_VERSION'是新的时区版本， 'SECONDARY_TT_VERSION'是旧的时区版本。

该过程以原子方式运行，并升级所有字典表和数据库属性。必须在OPEN MIGRATE模式下在数据库中调用它。

句法

DBMS_DST.BEGIN_UPGRADE（new_version IN BINARY_INTEGER，error_on_overlap_time IN BOOLEAN：= FALSE，error_on_nonexisting_time IN BOOLEAN：= FALSE）;

参数

表62-4 BEGIN_UPGRADE过程参数

参数 描述

新版本

要升级数据库的新时区版本

error_on_overlap_time

布尔标志，指示是否报告“重叠”时间语义转换错误的错误。默认值为FALSE 。有关边界情况的更多信息，请参见Oracle数据库SQL语言参考 。

error_on_nonexisting_time

布尔标志，指示是否报告“不存在的”时间语义转换错误的错误。

默认值为FALSE 。

62.4.8 FIND_AFFECTED_TABLES过程

此过程查找由于新时区版本而影响TSTZ数据的所有表。

只能在准备窗口期间调用此过程。影响TSTZ数据的表被记录到参数affected_tables指示的表中。如果必须记录语义错误，则将它们记录到由参数log_errors_table指示的表中。

句法

DBMS_DST.FIND_AFFECTED_TABLES（affected_tables IN VARCHAR2 =：'sys.dst $ affected_tables'，log_errors IN BOOLEAN：= FALSE，log_errors_table IN VARCHAR2 =：'sys.dst $ error_table'，parallel IN BOOLEAN：= FALSE）;

参数

表62-9 FIND_AFFECTED_TABLES过程参数

参数 描述

affected_tables

具有以下模式的表的名称：

CREATE TABLE dst $ affected_tables（table_owner VARCHAR2（30），table_name VARCHAR2（30），column_name VARCHAR2（4000），row_count NUMBER，error_count NUMBER）

可以使用CREATE_AFFECTED_TABLE过程创建表。

log_errors

布尔标志，指示在升级期间是否记录错误。如果为FALSE ，则在中止当前表的转换后，不会在log_errors_table记录错误。如果为TRUE ，则将错误记录到log_errors_table 。

默认值为FALSE 。

log_errors_table

表名具有以下架构：

CREATE TABLE dst $ error_table（table_owner VARCHAR2（30），table_name VARCHAR2（30），column_name VARCHAR2（4000），rid ROWID，error_number NUMBER）

可以使用CREATE_ERROR_TABLE过程创建表。rid列记录违规行的rowid， error_number列记录相应的错误号。

平行

布尔标志，指示是否使用并行查询或串行查询查找受影响的表。默认值为FALSE 。

62.4.9 UPGRADE_DATABASE程序

此过程升级数据库中的所有表，这些表具有在TSTZ类型上定义的一个或多个列或包含TSTZ类型的ADT。

只有在启动升级窗口后才能调用此过程。每个表都在原子事务中升级。请注意，基本表及其物化视图日志表在原子事务中升级。

句法

DBMS_DST.UPGRADE_DATABASE（num_of_failures OUT BINARY_INTEGER，upgrade_data IN BOOLEAN：= TRUE，parallel IN BOOLEAN：= FALSE，continue_after_errors IN BOOLEAN：= TRUE，log_errors IN BOOLEAN：= FALSE，log_errors_table IN VARCHAR2 =：'sys.dst $ error_table'，error_on_overlap_time IN BOOLEAN：= FALSE，error_on_nonexisting_time IN BOOLEAN：= FALSE，log_triggers_table IN VARCHAR2：='sys.dst $ trigger_table'）;

参数

使用说明

只有在启动升级窗口后才能调用此过程。表格列表必须满足以下部分排序：

  1. 如果基表具有物化视图日志表，则日志表必须是列表中的下一个项。

  2. 如果物化视图的容器表出现在列表中，则物化视图的“未升级”基表和日志表必须出现在表列表中和容器表之前。

需要通过将atomic_upgrade指定为TRUE来在原子事务中升级基表及其物化视图日志表。
