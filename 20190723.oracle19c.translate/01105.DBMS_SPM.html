155.1 DBMS_SPM概述

DBMS_SPM包允许用户使用SQL计划管理来管理SQL执行计划。

SQL计划管理通过记录和评估SQL语句的执行计划来防止SQL语句执行计划的突然更改导致的性能回归，并构建由一组已知高效的现有计划组成的SQL计划基准。然后使用SQL计划基准来保留相应SQL语句的性能，而不管系统中发生的更改。SQL计划管理可以改进或保留SQL性能的常见使用场景包括：

  • 安装新优化程序版本的数据库升级通常会导致一小部分SQL语句的计划更改，大多数计划更改导致性能未发生变化或改进。但是，某些计划更改可能会导致性能下降。SQL计划基准的使用极大地减少了数据库升级导致的潜在性能回归。

  • 正在进行的系统和数据更改可能会影响某些SQL语句的计划，从而可能导致性能下降。SQL计划基准的使用有助于最小化性能回归并稳定SQL性能。

  • 部署新的应用程序模块意味着将新的SQL语句引入系统。应用程序软件可以使用在标准测试配置下为新SQL语句开发的适当SQL执行计划。如果系统生产配置与测试配置有很大不同，则SQL计划基准可以随着时间的推移而发展，以产生更好的性能。

155.2 DBMS_SPM安全模型

该软件包归SYS 。执行其过程需要EXECUTE包权限。授予ADMINISTER SQL MANAGEMENT OBJECT权限的任何用户都可以执行DBMS_SPM程序包。

例子

以下示例为SQL文本创建一个过滤器，类似于SELECT a% ：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_SQL_TEXT'，'选择％'，'TRUE'）;

以下示例过滤掉HR解析模式：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_PARSING_SCHEMA_NAME'，'HR'，'FALSE'）;

以下示例删除SQL文本的所有现有过滤器：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_SQL_TEXT'，NULL，NULL）;

以下示例删除SQL文本的任何LIKE或NOT LIKE过滤器select a% ：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_SQL_TEXT'，'select a％'，NULL）;

以下示例使用谓词创建过滤器(action LIKE 'R%') OR (action LIKE '%E_') ：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_ACTION'，'R％'，'TRUE'）; EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_ACTION'，'％E_'，'TRUE'）;

以下示例使用谓词NOT(module LIKE 'LOGGER') AND NOT(module LIKE 'UTIL__')创建一个过滤器：

EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_MODULE'，'LOGGER'，'FALSE'）; EXEC DBMS_SPM.CONFIGURE（'AUTO_CAPTURE_MODULE'，'UTIL__'，'FALSE'）;

155.6.9 EVOLVE_SQL_PLAN_BASELINE函数

此函数演变与一个或多个SQL语句关联的SQL计划基准。当一个或多个未接受的计划更改为已接受的计划或计划时，将演变SQL计划基准。

如果用户询问（参数verify =' YES '），则将每个未接受的计划的执行性能与从关联的SQL计划基准中选择的计划的性能进行比较。如果发现未接受的计划性能优于SQL计划基准性能，则未接受的计划将更改为已接受的计划，前提是用户允许此类操作（参数commit ='YES'）。

该函数的第二种形式采用计划列表格式。

句法

DBMS_SPM.EVOLVE_SQL_PLAN_BASELINE（sql_handle IN VARCHAR2：= NULL，plan_name IN VARCHAR2：= NULL，time_limit IN INTEGER：= DBMS_SPM.AUTO_LIMIT，verify IN VARCHAR2：='YES'，commit IN VARCHAR2：='YES'）RETURN CLOB; DBMS_SPM.EVOLVE_SQL_PLAN_BASELINE（plan_list IN DBMS_SPM.NAME_LIST，time_limit IN INTEGER：= DBMS_SPM.AUTO_LIMIT，verify IN VARCHAR2：='YES'，commit IN VARCHAR2：='YES'）RETURN CLOB;

PLAN_NAME

计划名称列表。列表中的每个计划都可以属于相同或不同的SQL语句。

分钟数的时间限制。这仅适用于verify =' YES '。时间限制是全局的，使用方法如下：第一次未接受的计划验证的时间限制设置为等于输入值;第二次未接受的计划验证的时限设置为（输入值 - 第一次计划验证所花费的时间）;等等。

  • DBMS_SPM 。 AUTO_LIMIT （默认）允许系统根据需要完成的计划验证数量选择适当的时间限制。

  • DBMS_SPM 。 NO_LIMIT表示没有时间限制。

  • 正整数值表示用户指定的时间限制。

校验

指定在将未接受的计划更改为已接受的计划之前是否执行计划并比较性能。性能验证涉及执行未接受的计划和从相应的SQL计划基准中选择的计划，并比较其性能统计信息。如果未接受的计划显示绩效改进，则会将其更改为已接受的计划。

  • ' YES '（默认） - 验证未接受的计划在将其更改为已接受的计划之前提供更好的性能

  • ' NO ' - 指示不执行计划，但仅将未接受的计划更改为已接受的计划

承诺

指定是否将未接受计划的ACCEPTED状态从“ NO ”更新为“ YES ”。

  • ' YES '（默认） - 执行合格的未接受计划的更新，并生成一个报告，显示验证=' YES '时的更新和性能验证的结果。

  • ' NO ' - 生成没有任何更新的报告。请注意， commit =' NO '和verify =' NO '表示无操作。

返回值

包含格式化文本报告的CLOB ，按时间顺序显示未接受的计划，每个计划都可能更改其ACCEPTED状态，如果验证=' YES '，则表示其性能验证结果。

使用说明

调用此子程序需要ADMINISTER SQL MANAGEMENT OBJECT权限。
