1.290 RESOURCE_MANAGER_PLAN

RESOURCE_MANAGER_PLAN指定用于数据库（CDB或非CDB）的资源计划。

参数类型

句法

没有默认值。

没有

在国家开发银行

在CDB的根目录中， RESOURCE_MANAGER_PLAN指定CDB资源计划。CDB资源计划在PDB之间分配资源。

使用DBMS_RESOURCE_MANAGER.CREATE_CDB_PLAN和CREATE_CDB_PLAN_DIRECTIVE创建CDB资源计划。

也可以看看：

  • 有关使用DBMS_RESOURCE_MANAGER.CREATE_CDB_PLAN和DBMS_RESOURCE_MANAGER.CREATE_CDB_PLAN_DIRECTIVE详细信息，请参阅Oracle Multitenant管理员指南

  • Oracle Database PL / SQL包和类型参考有关DBMS_RESOURCE_MANAGER包的CREATE_CDB_PLAN过程和CREATE_CDB_PLAN_DIRECTIVE过程的更多信息

会话必须是root才能使用ALTER SYSTEM语句ALTER SYSTEM CDB的RESOURCE_MANAGER_PLAN值。例如，要启用和禁用CDB资源计划：

SQL> ALTER SYSTEM SET RESOURCE_MANAGER_PLAN = CDB_resource_plan_name ; SQL> ALTER SYSTEM SET RESOURCE_MANAGER_PLAN ='';

在PDB中， RESOURCE_MANAGER_PLAN指定用于PDB的PDB资源计划。

会话必须位于PDB中以启用或禁用该PDB的PDB资源计划。例如，要启用和禁用PDB资源计划：

SQL> ALTER SYSTEM SET RESOURCE_MANAGER_PLAN = PDB_resource_plan_name ; SQL> ALTER SYSTEM SET RESOURCE_MANAGER_PLAN ='';

在PDB中，与非CDB的资源计划相比，PDB资源计划具有一些限制。以下限制适用于PDB资源计划：

  • PDB资源计划不能有子计划。

  • PDB资源计划最多可包含八个使用者组。

  • PDB资源计划不能具有多级调度策略。

为了实施某些PDB资源计划策略，CDB资源计划中应存在有关PDB之间资源分配的策略。如果没有CDB资源计划，则不会强制执行某些PDB资源计划策略。如果PDB资源计划包含CPU或并行语句排队指令并且未指定CDB资源计划，则资源管理器将自动启用DEFAULT_CDB_PLAN计划。要防止出现此行为，请将根级别的RESOURCE_MANAGER_PLAN参数设置为ORA$INTERNAL_CDB_PLAN 。

注意：

有关CDB资源计划中的CPU，I / O带宽和并行执行服务器要求的信息，以及不满足这些要求时PDB资源计划中的结果说明，请参阅“ Oracle Multitenant管理员指南” 。

在非CDB中

RESOURCE_MANAGER_PLAN指定用于非CDB中的实例的顶级资源计划。资源管理器将加载此顶级资源计划及其所有后代（子计划，指令和使用者组）。如果未指定此参数，则默认情况下资源管理器处于关闭状态。

您可以使用ALTER SYSTEM语句更改此参数的设置以打开资源管理器（如果之前已关闭）或关闭资源管理器或更改当前资源计划（如果以前打开过）。如果指定数据字典中不存在的资源计划，Oracle将返回错误消息。
