为集群二进制文件重新使用Oracle Grid Infrastructure

为集群安装Oracle Grid Infrastructure（为集群配置Oracle Clusterware和Oracle ASM）后，如果需要修改二进制文件，请使用以下过程，其中Grid_home是集群主Grid_home的Oracle Grid Infrastructure：

警告：

在重新链接可执行文件之前，必须关闭正在重新链接的Oracle主目录中运行的所有可执行文件。此外，关闭与Oracle共享库链接的应用程序。如果存在，请卸载所有Oracle自动存储管理群集文件系统（Oracle ACFS）文件系统。

以root用户身份：

#cd Grid_home / crs / install＃rootcrs.sh -unlock

作为集群所有者的Oracle Grid Infrastructure：

$ export ORACLE_HOME = Grid_home $ Grid_home / bin / relink

再次以root用户身份：

#cd Grid_home / rdbms / install /＃./rootadd_rdbms.sh #cd Grid_home / crs / install＃rootcrs.sh -lock

每次应用操作系统修补程序或执行不替换根文件系统的操作系统升级后，都必须重新链接Oracle Clusterware和Oracle ASM二进制文件。对于导致新根文件系统的操作系统升级，必须从群集中删除该节点并将其重新添加到群集中。

对于以前版本的升级，如果要卸载先前版本的Grid home，则必须先解锁先前版本的Grid home。通过从先前版本主rootcrs.sh -unlock运行命令rootcrs.sh -unlock来解锁先前版本的Grid home。脚本完成后，您可以运行deinstall命令。
