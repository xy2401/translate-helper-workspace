2 Informix网关功能和限制

安装和配置网关后，您可以使用网关访问Informix数据，将Informix命令从应用程序传递到Informix数据库，执行分布式查询和复制数据。

远程插入Rowsource

远程插入行源功能允许远程插入，要求本地Oracle数据通过Oracle数据库和Oracle数据库网关工作。此功能要求Oracle数据库和Oracle Database Gateway为12.2或更高版本。

通过Oracle数据库设计，必须在数据库链接站点执行某些分布式语句。但在某些情况下，执行这些查询所需的数据必须从原始Oracle数据库中获取。在同构连接下，远程Oracle数据库将回调源Oracle数据库以获取此类数据。但是在异构连接中，这是不可行的，因为这意味着外部数据存储必须查询只能由发出查询的Oracle实例提供的回调函数或数据。通常，这些类型的语句不能通过Oracle数据库网关支持。

以下类别的SQL语句会导致回调：

  • 任何带有子选择的DML，它指的是Oracle数据库中的一个表。

  • 任何DELETE ， INSERT ， UPDATE或“ SELECT...FOR UPDATE..."包含需要在原始Oracle数据库中执行的SQL函数或语句的SQL语句。

    这些SQL函数包括USER ， USERENV和SYSDATE ;并涉及从原始Oracle数据库中选择数据。

  • 任何涉及Oracle数据库中的表以及远程表中的LONG或LOB列的SQL语句。

可以通过远程插入行源功能工作的远程INSERT语句的示例如下：

INSERT INTO gateway_table @gateway_link select * from local_table ;

使用传递功能

网关可以使用DBMS_HS_PASSTHROUGH包将Informix命令或语句从应用程序传递到Informix数据库。

使用PL / SQL块中的DBMS_HS_PASSTHROUGH包指定要传递给Informix数据库的语句，如下所示：

DECLARE num_rows INTEGER; BEGIN num_rows：= DBMS_HS_PASSTHROUGH.EXECUTE_IMMEDIATE@IFMX（' command '）;结束; /

其中command不能是以下之一：

  • 承诺

  • 创建数据库

  • DROP DATABASE

  • ROLLBACK

  • ROLLFORWARD数据库

  • Informix工具命令

DBMS_HS_PASSTHROUGH包支持传递绑定值和执行SELECT语句。

注意：

建议您在传递中的每个DDL语句之后执行COMMIT 。

CHAR语义学

此功能允许网关可选地以CHAR语义模式运行。

而不是总是将Informix CHAR列描述为CHAR(n BYTE) ，此功能将它们描述为CHAR(n CHAR)和VARCHAR(n CHAR) 。该概念类似于Oracle数据库CHAR语义。您需要指定HS_NLS_LENGTH_SEMANTICS=CHAR网关参数以激活此选项。

多字节字符集比率抑制

此功能可选地抑制从Informix数据库到涉及多字节字符集的Oracle数据库的比率扩展。

默认情况下，Oracle网关采用最差的比率来防止数据被截断或缓冲区大小不足。但是，如果您具有Informix数据库的特定知识并且不希望进行扩展，则可以指定HS_KEEP_REMOTE_COLUMN_SIZE参数以禁止扩展。

ANSI SQL标准

美国国家标准协会（ANSI）已经为SQL建立了一套行业标准。网关仅支持符合ANSI标准的Informix数据库。有关如何创建或启动符合ANSI标准的Informix数据库的更多信息，请参阅Informix文档。

命名规则

命名规则问题包括以下内容：

  • 命名对象的规则

  • 对象名称

  • 区分大小写

命名对象的规则

Oracle和Informix使用不同的数据库对象命名规则。例如，每个对象名称允许的最大字符数可以不同。此外，使用单引号和双引号，区分大小写以及使用字母数字字符都可以不同。

也可以看看：

Oracle数据库参考和Informix文档。

对象名称

Informix数据库对象的名称最多为18个字符。对象名称可以由以下字符组成：

  • 数字0到9

  • 小写字母a到z

  • 大写字母A到Z.

  • 下划线字符（_）

区分大小写

Informix处理字母大小写与Oracle不同。Informix使用以下规则：

  • 表所有者名称默认为大写字母，除非名称由双引号字符包围

  • 列名，表名，视图名等始终被视为小写字母

除非您使用双引号字符包围标识符，否则Oracle数据库默认为大写。例如，要引用名为emp的Informix表，请输入带双引号字符的名称，如下所示：

SQL> SELECT * FROM“emp”@IFMX;

但是，要从Oracle应用程序引用SCOTT拥有的名为emp的Informix表，请输入以下内容：

SQL> SELECT * FROM“Scott”。“emp”@IFMX;

如果名为emp的Informix表由SCOTT拥有（表所有者名称为大写字母），则可以输入不带双引号字符的所有者名称，如下所示：

SQL> SELECT * FROM SCOTT。“emp”@IFMX;

要么

SQL> SELECT * FROM scott。“emp”@IFMX;

Oracle建议您使用双引号字符包围所有Informix对象名称，并使用Informix数据字典中显示的对象名称的确切字母大小写。参照中列出的支持的Oracle数据字典的表或视图时，则不需要此约定数据字典 。

如果无法根据这些约定更改现有应用程序，请在Oracle中创建视图以将Informix名称与正确的字母大小写相关联。例如，要通过仅使用大写名称从现有Oracle应用程序引用Informix表emp ，请定义以下视图：

SQL> CREATE VIEW EMP（EMPNO，ENAME，SAL，HIREDATE）AS SELECT“empno”，“ename”，“sal”，“hiredate”FROM“emp”@IFMX;

使用此视图，应用程序可以发出如下语句：

SQL> SELECT EMPNO，ENAME FROM EMP;

使用视图是一种解决方案，可复制源自Informix数据字典的数据字典信息。每当在Informix数据库中更改相应表的数据定义时，您必须准备好更新Oracle视图定义。

数据类型

数据类型问题包括以下内容：

  • Binary_ Byte和Text Literal表示法

  • 数据类型转换

二进制，字节和文本字面符号

Oracle SQL使用由单引号括起的十六进制数字来表示要比较或插入到定义为数据类型RAW列中的文字值。

此表示法未转换为与Informix BINARY ， BYTE和TEXT数据类型兼容的语法（0x后跟十六进制数字，由单引号括起）。

例如，不支持以下语句：

SQL> INSERT INTO BYTE_TAB @ IFMX VALUES（'Oxff'）;

其中BYTE_TAB包含数据类型为BINARY ， BYTE或TEXT 。插入或更新BINARY ， BYTE或TEXT数据类型时使用绑定变量。

数据类型转换

Informix不支持隐式日期转换。此类转换必须明确。

例如，网关为以下SELECT语句发出错误：

SQL> SELECT DATE_COL FROM TEST @ IFMX WHERE DATE_COL =“1-JAN-2001”;

要避免隐式转换出现问题，请添加显式转换，如下所示：

SQL> SELECT DATE_COL FROM TEST @ IFMX WHERE DATE_COL = TO_DATE（“1-JAN-2001”）;

查询

查询问题包括以下内容：

  • 行选择

  • 空字符串

  • 空绑定变量

行选择

在返回任何行之前，Informix会评估所有选定行的查询条件。如果一个或多个行的评估过程中存在错误，则即使其余行满足条件，也不会返回任何行。

Oracle逐行评估查询条件，并在评估成功时返回一行。返回行直到一行未通过评估。

空字符串

Oracle将SQL语句中的空字符串作为空值处理。Informix将空字符串作为空字符串处理。

比较空字符串

网关将文字空字符串传递给Informix数据库而不进行任何转换。如果您希望空字符串表示空值，则Informix不会以此方式处理该语句;它使用空字符串。

您可以通过在SQL语句中使用NULL或IS NULL而不是空字符串语法来避免此问题，如下例所示：

SELECT * from“emp”@IFMX where“ename”IS NULL;

选择一个空字符串

对于VARCHAR列，网关将空字符串作为NULL值返回到Oracle数据库。

对于CHAR列，网关返回列的完整大小，每个字符为空格（''）。

空绑定变量

对于VARCHAR绑定变量，网关将空绑定变量作为NULL值传递给Informix数据库。

锁定

Informix数据库的锁定模型与Oracle模型有很大不同。网关依赖于底层的Informix行为，因此以下可能的方案可能会影响通过网关访问Informix的Oracle应用程序：

  • 读访问可能会阻止写访问

  • 写访问可能会阻止读访问

  • 语句级读取一致性无法保证

交易诚信

在以下情况下，网关无法保证事务完整性：

  • 当网关处理的语句导致目标数据库中的隐式提交时

  • 目标数据库配置为在自动提交模式下工作时

    注意：

    Oracle强烈建议如下：

    • 如果您知道执行特定语句会导致目标数据库中的隐式提交，请确保在其自己的事务中执行此语句。

    • 不要将目标数据库配置为在自动提交模式下工作。

交易能力

网关不支持保存点。如果涉及网关的分布式更新事务正在进行并且用户尝试创建保存点，则会发生以下错误：

 ORA-02070：数据库dblink在此上下文中不支持savepoint

默认情况下，网关配置为COMMIT_CONFIRM ，在此事务模式下，它始终为事务更新Informix数据库时提交点站点。

如果Informix系统不使用日志记录，则无法使用默认事务功能。强烈建议您在Informix系统中启用日志记录。如果您没有使用日志记录，并且只具有只读要求，请在网关初始化参数文件中设置HS_TRANSACTION_MODEL=READ_ONLY_AUTOCOMMIT 。如果您不使用日志记录，并且需要更新Informix数据库，请在网关初始化参数文件中设置HS_TRANSACTION_MODEL=SINGLE_SITE_AUTOCOMMIT 。

PL / SQL游标循环中的COMMIT或ROLLBACK关闭打开游标

在PL / SQL游标循环中发出的任何COMMIT或ROLLBACK关闭所有打开的游标，这可能导致以下错误：

ORA-1002：取消顺序

要防止此错误，请在游标循环外移动COMMIT或ROLLBACK语句。

传递功能

如果通过网关传递的SQL语句导致Informix数据库中的隐式提交，则Oracle事务管理器不会意识到提交，并且Oracle ROLLBACK命令不能用于回滚事务。

Informix SMALLFLOAT和REAL数据类型

Informix的 SMALLFLOAT和 REAL数据类型的精度为6 。

条款的当前条款

网关不支持带有WHERE CURRENT OF子句的UPDATE和DELETE语句，因为它们依赖于Oracle ROWID实现。要通过网关更新或删除特定行，必须使用条件样式WHERE子句。

通过条款连接

网关不支持SELECT语句中的CONNECT BY子句。

在SELECT语句中使用NULL关键字

NULL关键字不能在SELECT语句的选择列表中使用，因为该语法不是ANSI SQL 。

例如，不能使用以下语句：

SQL> SELECT NULL FROM。 。 。
INSERT语句中的子查询

INSERT语句的子查询不能对同一个表使用多个别名。例如，不支持以下语句：

SQL> INSERT INTO“emp_target”@IFMX SELECT a。“empno”FROM“emp_source”@IFMX a，“emp_source”@IFMX b WHERE b。“empno”= 9999
DELETE，INSERT和UPDATE语句中的子查询

DELETE ， INSERT和UPDATE语句的子查询中的SQL语句不能引用外部查询中指定的同一个表。这是因为Informix中的锁定机制。

ROWID

不支持Oracle ROWID实现。

解释计划声明

不支持EXPLAIN PLAN语句。

在SQL * Plus

在SQL * Plus中，网关不支持使用SELECT语句从定义为数据类型BYTE的Informix列中检索数据。

您需要使用双引号来包装小写表名，例如：

复制自tkhouser / tkhouser @ inst1使用select * from“tkhodept”@ holink2插入loc_tkhodept;

数据库链接

网关不是多线程的，不支持共享数据库链接。每个网关会话都会生成一个单独的网关进程，并且无法共享连接。

网关数据字典视图

在网关数据字典中查询ALL_VIEWS和USER_VIEWS时，仅返回视图定义的前64个字符。

存储过程

网关不支持允许在非Oracle数据库中执行存储过程的过程功能。

加密格式登录

Oracle数据库不再支持初始化参数DBLINK_ENCRYPT_LOGIN 。直到版本7.3，此参数的默认TRUE值阻止了通过网络发送登录用户ID的密码（以明文形式）。更高版本会自动加密密码。

Informix BYTE和TEXT数据类型

使用BYTE和TEXT数据类型时，以下限制适用：

  • 不支持的SQL函数不能在访问定义为Informix数据类型TEXT的列的SQL语句中使用。

  • 当数据长度超过80个字符时，不能使用SQL * Plus从定义为Informix数据类型TEXT的列中选择数据。Oracle建议使用Pro * C或Oracle Call Interface访问Informix数据库中的此类数据。

  • 对于INSERT或UPDATE ， BYTE和TEXT数据类型必须为NULLABLE才能工作。

  • 包含BYTE或TEXT列的表必须在表上定义唯一索引，否则表必须具有用作主键的单独列。

  • 视图中的BYTE和TEXT数据无法访问。

  • 无法通过传递查询读取BYTE和TEXT数据。

  • 小于32,739字节的数据无法使用绑定变量插入到BYTE和TEXT列中。

网关不支持DBMS_SQL包的PL / SQL函数COLUMN_VALUE_LONG 。
