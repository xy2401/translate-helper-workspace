4.6 Oracle C ++调用接口（OCCI）

Oracle C ++调用接口（OCCI）是一种C ++ API，使您可以使用面向对象的功能，本机类和C ++编程语言的方法来访问Oracle数据库。

4.6.1关于Oracle C ++调用接口（OCCI）

OCCI接口在JDBC接口上建模，与JDBC接口一样，易于使用。OCCI本身建立在OCI之上，使用面向对象的范例提供OCI的功能和性能。

OCI是Oracle数据库的C API。它支持整个Oracle功能集，并提供对关系数据和对象数据的高效访问，但使用起来可能具有挑战性 - 特别是如果您想使用复杂的对象数据类型。C中本身不支持对象类型，并且在C中模拟它们并不容易。OCCI通过为OCI的功能提供更简单，面向对象的接口来解决这个问题。它通过为OCI定义一组包装器来实现。通过使用这些更高级别的抽象，开发人员可以利用OCI的基本功能，通过面向对象的界面来操作服务器中的对象，这种界面非常容易编程。

Oracle C ++调用接口OCCI大致可分为三组功能，即：

  • 关联关系访问

  • 关联对象访问

  • 导航访问

4.6.2 OCCI关联关系和对象接口

关联关系API和对象类提供对数据库的SQL访问。

通过这些接口，SQL在服务器上执行，以创建，操作和获取对象或关系数据。应用程序可以访问服务器上的任何数据类型，包括：

  • 大物件

  • 对象/结构化类型

  • 数组

  • 参考

4.6.3 OCCI导航界面

导航界面是一个C ++界面，它允许您以C ++对象的形式无缝访问和修改对象关系数据，而无需使用SQL。

根据需要透明地访问C ++对象并将其存储在数据库中。

使用OCCI导航界面，您可以检索对象并浏览从该对象到其他对象的引用。服务器对象在应用程序缓存中实现为C ++类实例。

应用程序可以使用OCCI对象导航调用在服务器的对象上执行以下功能：

  • 创建，访问，锁定，删除和刷新对象

  • 获取对象的引用并浏览它们
