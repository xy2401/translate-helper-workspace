3关于DML语句和事务

数据操作语言（DML）语句添加，更改和删除Oracle数据库表数据。事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。

  • 关于数据操作语言（DML）语句
    数据操作语言（DML）语句访问和操作现有表中的数据。
  • 关于交易控制报表
    事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。您需要事务来建模需要将多个操作作为一个单元执行的业务流程。
  • 提交交易
    提交事务会使其更改成为永久更改，删除其保存点并释放其锁定。
  • 回滚交易
    回滚事务会撤消其更改。您可以回滚整个当前事务，也可以只将其回滚到指定的保存点。
  • 在事务中设置保存点
    SAVEPOINT语句标记事务中的保存点 - 您可以稍后回滚的点。保存点是可选的，事务可以有多个保存点。

3.1关于数据操作语言（DML）语句

数据操作语言（DML）语句访问和操作现有表中的数据。

在SQL * Plus环境中，您可以在SQL>提示符后输入DML语句。

在SQL Developer环境中，您可以在Worksheet中输入DML语句。或者，您可以使用SQL Developer Connections框架和工具来访问和操作数据。

在提交包含DML语句的事务之前，DML语句的效果不是永久性的。事务是Oracle数据库作为一个单元处理的一系列SQL语句（它可以是单个DML语句）。在提交事务之前，它可以回滚（撤消）。有关事务的更多信息，请参阅“ 关于事务控制语句 ” 。

3.1.1关于INSERT语句

INSERT语句将行插入现有表中。

最简单的INSERT语句推荐形式具有以下语法：

INSERT INTO table_name （ list_of_columns ）VALUES（ list_of_values ）;

在list_of_columns每一列必须在list_of_values相应位置的有效值。因此，在将行插入表之前，必须知道表具有哪些列以及它们的有效值。要使用SQL Developer获取此信息，请参阅“ 教程：使用SQL Developer 查看EMPLOYEES表属性和数据 ” 。要使用SQL * Plus获取此信息，请使用DESCRIBE语句。例如：

描述员工;

3.1.2关于UPDATE语句

UPDATE语句更新（更改值）一组现有表行。

UPDATE语句的一种简单形式具有以下语法：

UPDATE table_name SET column_name = value [， column_name = value ] ...[ 条件 ];

每个值必须对其column_name有效。如果包含WHERE子句，则语句仅在满足条件的行中更新列值。

示例3-4中的UPDATE语句更新了在已知员工薪水之前插入到示例3-2中的EMPLOYEES表中的行中的SALARY列的值。

例3-5中的UPDATE语句更新了部门80中每个员工的佣金百分比。

示例3-4使用UPDATE语句添加数据

结果：

1行更新。

示例3-5使用UPDATE语句更新多行

结果：

3.1.3关于DELETE语句

DELETE语句从表中删除行。

DELETE语句的一种简单形式具有以下语法：

DELETE FROM table_name [WHERE condition ];

如果包含WHERE子句，则该语句仅删除满足条件的行。如果省略WHERE子句，则该语句将删除表中的所有行，但空表仍然存在。要删除表，请使用DROP TABLE语句。

3.2关于交易控制声明

事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。您需要事务来建模需要将多个操作作为一个单元执行的业务流程。

例如，当经理离开公司时，必须在JOB_HISTORY表中插入一行以显示经理何时离开，并且对于向该经理报告的每个员工，必须在EMPLOYEES表中更新MANAGER_ID的值。要在应用程序中对此过程建模，必须将INSERT和UPDATE语句分组到单个事务中。

基本的事务控制语句是：

  • SAVEPOINT ，它标记事务中的保存点 - 您可以稍后回滚的点。保存点是可选的，事务可以有多个保存点。

  • 结束当前事务的COMMIT使其更改成为永久更改，擦除其保存点并释放其锁定。

  • ROLLBACK ，它回滚（撤消）整个当前事务或仅撤消指定保存点之后所做的更改。

在SQL * Plus环境中，您可以在SQL>提示符后输入事务控制语句。

在SQL Developer环境中，您可以在Worksheet中输入事务控制语句。SQL Developer还有“提交更改”和“回滚更改”图标，这些图标在“ 提交事务 ”和“ 回滚事务 ”中进行了说明 。

警告：

如果您没有显式提交事务，并且程序异常终止，则数据库会自动回滚最后一个未提交的事务。

Oracle建议您通过提交或回滚它们来明确地结束应用程序中的事务。

3.3提交交易

提交事务会使其更改成为永久更改，删除其保存点并释放其锁定。

要显式提交事务，请使用COMMIT语句或（在SQL Developer环境中） Commit Changes图标。

注意：

Oracle数据库在任何数据定义语言（DDL）语句之前和之后发出隐式COMMIT语句。有关DDL语句的信息，请参阅“ 关于数据定义语言（DDL）语句 ” 。

在提交交易之前：

  • 您的更改对您可见，但对数据库实例的其他用户不可见。

  • 您的更改不是最终的 - 您可以使用ROLLBACK语句撤消它们。

提交事务后：

  • 您的更改对其他用户以及提交事务后运行的语句可见。

  • 您的更改是最终的 - 您无法使用ROLLBACK语句撤消它们。

示例3-7向REGIONS表添加一行（一个非常简单的事务），检查结果，然后提交事务。

示例3-7提交事务

结果：

事务（向表添加行）：

已创建1行。

结果：

提交完成。

3.4回滚交易

回滚事务会撤消其更改。您可以回滚整个当前事务，也可以只将其回滚到指定的保存点。

要将当前事务回滚到指定的保存点，必须使用带有TO SAVEPOINT子句的ROLLBACK语句。

要回滚整个当前事务，请使用不带TO SAVEPOINT子句的ROLLBACK语句，或者（在SQL Developer环境中）使用Rollback Changes图标。

回滚整个当前事务：

  • 结束交易

  • 扭转所有变化

  • 删除所有保存点

  • 释放任何事务锁

仅将当前事务回滚到指定的保存点：

  • 不结束交易

  • 仅反转在指定保存点之后所做的更改

  • 仅删除指定保存点后设置的保存点（不包括指定的保存点本身）

  • 释放在指定保存点之后获取的所有表锁和行锁

    已请求访问在指定保存点之后锁定的行的其他事务必须继续等待，直到提交或回滚事务为止。未请求行的其他事务可以立即请求和访问行。

要在SQL Developer中查看回滚的效果，可能必须单击“ 刷新”图标。

作为示例3-7的结果， REGIONS表具有称为“中东和非洲”的区域和称为“非洲”的区域。示例3-8更正了此问题（一个非常简单的事务）并检查更改，但随后回滚事务并检查回滚。

示例3-8回滚整个事务

交易前：

SELECT * FROM REGIONS ORDER BY REGION_ID;

结果：

REGION_ID REGION_NAME ---------- ------------------------- 1欧洲2美洲3亚洲4中东和非洲5非洲选择了5行。

交易（更改表）：

更新区域设置REGION_NAME ='中东'WHERE REGION_NAME ='中东和非洲';

结果：

1行更新。

检查更改：

结果：

回滚完成。

检查回滚：

结果：
