9.8为Oracle数据库QoS管理创建管理用户

Oracle数据库QoS管理提供了一个名为QOSCTL的命令行实用程序，可帮助您管理用户。

此实用程序安装在群集的每个节点上，但只有在运行Oracle数据库QoS管理服务器的同一节点上以Oracle Clusterware管理员用户身份执行时才能正常运行。可以通过发出以下命令来确定运行该实用程序的正确节点：

srvctl status qosmserver

QOSCTL实用程序支持创建Oracle数据库QoS管理管理用户。帐户信息存储在具有加密凭据的本地system-jazn-data.xml文件中。帐户信息也存储在Oracle Clusterware存储库（OCR）中，以支持Oracle数据库QoS管理服务器的故障转移。

9.8.1 QOSCTL实用程序参考

QOSCTL是一个命令行实用程序，允许您为Oracle数据库QoS管理执行某些配置任务。

目的

QOSCTL实用程序支持创建Oracle数据库QoS管理管理用户。

文件路径

qosctl可执行文件位于Grid_home /bin目录中。

安全要求

要使用QOSCTL实用程序，您必须以Oracle Clusterware管理员用户身份登录。

显示QOSCTL实用程序的帮助

要显示qosctl实用程序的帮助，请使用以下命令：

qosctl -help

句法

以下代码示例显示了QOSCTL命令的一般格式：

qosctl qos_admin_user command

您可以使用Oracle Database QoS Management管理用户的用户名代替qos_admin_user参数。不要包含密码。提交执行命令后，系统将提示您输入与指定qos_admin_user关联的密码。

配置Oracle数据库QoS管理系统的初始帐户时，该命令使用操作系统身份验证。

命令

表9-1 QOSCTL实用程序的命令摘要

命令语法 描述

-activatepolicy policy_name

设置当前策略。此命令使您可以通过CRON或计划作业更改策略。

包含空格的策略名称必须用双引号括起来。

-updatepolicyset

发现并添加新数据库及其各自的服务，并将每个服务的性能类别添加到现有策略集。

-adduser username

添加指定的用户并提示输入密码。此用户自动被授予使用Oracle数据库QoS管理应用程序并执行任何这些命令所需的角色。

-li stusers

列出有权运行Oracle数据库QoS管理应用程序的用户

- remuser username

删除指定的用户帐户和所有相关权限

注意：此命令不可恢复。请谨慎使用，因为您可以删除自己的帐户。

- setpasswd username

更新指定用户的密码。QOSCTL实用程序会提示您输入用户的当前密码。您必须为旧密码提供正确的值才能更改用户的密码。

如果忘记或丢失了用户的密码，则应删除用户帐户并使用新密码为该用户创建新帐户。

- help

显示QOSCTL命令的语法。
