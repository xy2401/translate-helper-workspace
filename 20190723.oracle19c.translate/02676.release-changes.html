“ Oracle数据库企业版用户安全管理员指南”中此版本的更改

  • 版本19c版本19.1的此版本中的更改

版本19c版本19.1的此版本中的更改

以下是Oracle数据库企业用户安全管理员指南（适用于Oracle数据库版本19c，版本19.1）中的更改。

新功能

以下功能是此版本中的新增功能：

  • 企业安全管理员可以配置客户端钱包（也称为外部安全密码存储），以便在使用独立实用程序时存储用户名凭据。

    这样做简化了依赖密码凭证连接数据库的部署，因为脚本将不再需要嵌入用户名和密码。这也降低了风险，因为密码不再暴露，并且无论用户名或密码何时更改，都可以更轻松地管理密码策略，而无需更改应用程序代码或脚本。

    客户端钱包可以与以下独立实用程序一起使用：

    • 企业用户安全管理器（EUSM）

      有关在钱包中配置dbuser ， keystore和ldap_user凭据以及在命令行上使用db_alias ， keystore_alias ， ldap_alias和wallet_location参数的详细信息，请参阅关于使用安全外部密码存储 。

    • 用户迁移工具

      有关在钱包中配置DBADMIN和ENTADMIN凭据以及在命令行上使用DBALIAS ， ENTALIAS和WALLETLOCATION参数的详细信息，请参阅步骤0：关于使用安全外部密码存储 。

    • 外部用户转换脚本

      有关在钱包中配置dbuser凭据以及在命令行上使用-dbalias和-wallet_location参数的详细信息，请参阅SSL外部用户转换脚本 。

Oracle Database Release 18c版本18.1中的更改

以下是Oracle数据库企业用户安全管理员指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

新功能

以下功能是此版本中的新增功能：

  • Oracle数据库企业版用户安全版18c，版本18.1包括以下新功能：

    • 企业用户安全管理器（EUSM）

      eusm命令行工具可用于管理Oracle Internet Directory（OID）目录服务器中的企业用户安全性（EUS）配置。

      有关此功能的详细信息，请参阅Enterprise User Security Manager（EUSM）命令参考 。

    • Microsoft Active Directory的集中管理用户。

      对于不需要某些更复杂的企业用户安全功能（如可信数据库链接）的新项目，此集成是复杂性，成本，维护和开发视角的首选选项。

      有关使用集中管理的用户直接对使用Microsoft Active Directory的用户进行身份验证和授权的详细信息，请参阅“ Oracle数据库安全指南”中的“ 配置身份验证和配置授权 ” 。

      请参阅将企业用户安全性与Microsoft Active Directory集成，以获取使用Microsoft Active Directory对用户进行身份验证和授权的其他选项。

    • 通过Oracle Internet Directory生成的12C验证程序，为Enterprise User Security身份验证添加了支持

      从Oracle Internet Directory生成的12C验证程序使用新的ZT标记MR-SHA512 ，用于基于密码的基于密钥派生函数（PBKDF2）的密钥哈希消息身份验证代码（HMAC），具有SHA512加密哈希函数，以提供强大的功能密码验证者。此支持在OID捆绑补丁11.1.1.9.0中添加。

      有关详细信息，请参阅Oracle®FusionMiddleware管理员指南（适用于Oracle Internet Directory） 。

    • 对于可插拔数据库，您不再受限于默认钱包位置。WALLET_LOCATION可以指向您选择的目录位置。

      有关可插拔数据库 ，请参阅钱包位置以获取更多信息。

Oracle Database 12 c第2版（12.2.0.1）中的更改

以下是Oracle Database Enterprise用户安全管理员指南（适用于Oracle Database 12 c第2版（12.2.0.1））中的更改。

新功能

以下功能是此版本中的新增功能：

  • Enterprise User Security 12 c第 2版（12.2.0.1）包括以下新功能：

    • okcreate命令行实用程序，用于在使用Kerberos身份验证部署Enterprise User Security时自动创建服务主体密钥表。

      有关此功能的详细信息，请参阅设置Active Directory以与Oracle客户端互操作 。
