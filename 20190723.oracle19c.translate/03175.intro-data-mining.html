2.1关于Oracle Data Mining

了解Oracle Data Mining的用法并了解不同的挖掘技术。

Oracle Data Mining在Oracle数据库中提供了强大的，最先进的数据挖掘功能。您可以使用Oracle Data Mining构建和部署预测性和描述性数据挖掘应用程序，为现有应用程序添加智能功能，以及为数据探索生成预测性查询。

Oracle Data Mining提供了一套全面的数据库内算法，用于执行各种挖掘任务，例如分类，回归，异常检测，特征提取，聚类和市场购物篮分析。算法可以处理标准案例数据，事务数据，星型模式以及文本和其他形式的非结构化数据。Oracle Data Mining非常适合挖掘超大型数据集。

Oracle Data Mining是Oracle数据库企业版Oracle高级分析选件的两个组件之一。另一个组件是Oracle R Enterprise，它将开源统计环境R与Oracle数据库集成在一起。Oracle Data Mining和Oracle R Enterprise共同构成了一个全面的高级数据分析分析平台。

2.2数据库内核中的数据挖掘

了解数据挖掘的实现。

Oracle Data Mining在Oracle数据库中实现核心。数据挖掘模型是一流的数据库对象。Oracle数据挖掘过程使用Oracle数据库的内置功能来最大化可扩展性并有效利用系统资源。

Oracle数据库中的数据挖掘提供了许多优势：

  • 无数据移动：某些数据挖掘产品要求从公司数据库导出数据并将其转换为专门的挖掘格式。使用Oracle Data Mining，无需数据移动或转换。这使得整个挖掘过程不那么复杂，耗时且容易出错，并且允许挖掘非常大的数据集。

  • 安全性：您的数据受Oracle数据库广泛的安全机制保护。此外，不同的数据挖掘活动需要特定的数据库权限。只有具有相应权限的用户才能定义，操作或应用挖掘模型对象。

  • 数据准备和管理：大多数数据必须在开采之前以各种方式进行清理，过滤，标准化，采样和转换。数据挖掘项目中高达80％的工作量通常用于数据准备。Oracle Data Mining可以自动管理数据准备过程中的关键步骤。此外，Oracle数据库还提供了用于准备和管理数据的广泛管理工具。

  • 易于数据刷新：Oracle数据库中的挖掘过程可以随时访问刷新的数据。Oracle Data Mining可以根据当前数据轻松提供挖掘结果，从而最大限度地提高其及时性和相关性。

  • Oracle数据库分析：Oracle数据库为高级分析和商业智能提供了许多功能。Oracle Data Mining可以轻松地与数据库的其他分析功能集成，例如统计分析和OLAP。

  • Oracle技术栈：您可以利用Oracle技术堆栈的各个方面，将数据挖掘集成到更大的商业智能或科学查询框架中。

  • 域环境：必须在适当的应用程序域环境中构建，测试，验证，管理和部署数据挖掘模型。数据挖掘结果可能需要作为域特定计算的一部分进行后处理（例如，计算估计的风险和响应概率），然后存储到永久存储库或数据仓库中。使用Oracle Data Mining，采矿前后的活动都可以在同一环境中完成。

  • 应用程序编程接口：PL / SQL API和SQL语言运算符提供对Oracle数据库中Oracle数据挖掘功能的直接访问。

2.3 Oracle Exadata中的数据挖掘

了解Oracle Exadata中的评分。

评分是指将数据挖掘模型应用于数据以生成预测的过程。该评分过程可能需要大量的系统资源。可能涉及大量数据，并且算法处理可能非常复杂。

借助Oracle Data Mining，可以将评分卸载到处理效率极高的智能Oracle Exadata存储服务器。

Oracle Exadata存储服务器结合了Oracle的智能存储软件和Oracle的行业标准Sun硬件，可提供业界最高的数据库存储性能。有关Oracle Exadata的更多信息，请访问Oracle技术网。

2.4关于分区模型

引入分区模型来组织和表示多个模型。

Oracle Data Mining支持构建持久的Oracle Data Mining分区模型。分区模型将单个模型组织并表示为单个模型实体中的分区，使用户能够轻松构建和管理针对独立数据片段定制的模型。持久意味着分区模型具有磁盘表示。该产品管理分区模型的组织，并简化了对分区模型进行评分的过程。在评分时，您必须将分区列包含在USING子句中。

分区名称，键值和分区模型的结构在ALL_MINING_MODEL_PARTITIONS视图中可见。

2.5.4预测分析

预测分析是一种在简单例程中捕获数据挖掘过程的技术。

有时被称为“一键式数据挖掘” 预测分析简化并自动化数据挖掘过程。

预测分析使用数据挖掘技术，但使用预测分析不需要数据挖掘知识。您只需指定要对数据执行的操作即可使用预测分析。您无需创建或使用挖掘模型或了解“Oracle数据挖掘基础知识”中汇总的挖掘函数和算法。

2.6数据库分析概述

Oracle数据库支持一系列独立于Oracle Advanced Analytics Option的本机分析功能。由于所有这些功能都是公共服务器的一部分，因此可以有效地组合它们。分析处理的结果可以与之结合 Oracle商业智能套件企业版和其他BI工具和应用程序。
