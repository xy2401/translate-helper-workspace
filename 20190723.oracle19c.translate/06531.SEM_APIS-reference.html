10.1 SEM_APIS.ADD_DATATYPE_INDEX

格式

SEM_APIS.ADD_DATATYPE_INDEX（数据类型IN VARCHAR2，tablespace_name IN VARCHAR2 DEFAULT NULL，并行IN PLS_INTEGER DEFAULT NULL，联机IN BOOLEAN DEFAULT FALSE，选项IN VARCHAR2 DEFAULT NULL，network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

将指定数据类型的数据类型索引添加到语义网络。

参数

数据类型

要索引的数据类型的URI。

tablespace_name

索引的目标表空间。

并行

构建索引时使用的并行度。

在线

TRUE允许在创建索引期间影响索引的DML操作; FALSE （默认值）不允许在创建索引期间影响索引的DML操作。

选项

使用OPTION_NAME = option_value形式指定创建索引的选项的字符串。与空间索引创建关联的受支持选项是SRID ， TOLERANCE和DIMENSIONS 。对于物化空间索引创建，请使用MATERIALIZE=T与文本索引创建关联的支持选项是PREFIX_INDEX ， PREFIX_MIN_LENGTH ， PREFIX_MAX_LENGTH和SUBSTRING_INDEX 。对于基于函数的numeric或dateTime索引创建，请使用FUNCTION = T.选项名称关键字区分大小写，必须以大写形式指定。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

您必须具有DBA权限才能调用此过程。

有关数据类型索引的详细信息，请参阅使用数据类型索引 。

有关创建一个类似的索引信息，请参阅轻量级文本搜索材料的全文本搜索 。

有关在RDF空间数据上创建数据类型索引的信息，请参阅索引空间数据 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例在MY_TBS表空间中的xsd:string typed literals和plain literals上创建索引。 

EXECUTE SEM_APIS.ADD_DATATYPE_INDEX（'http://www.w3.org/2001/XMLSchema#string',tablespace_name =>'MY_TBS'，parallel => 4）;

10.9 SEM_APIS.ANALYZE_ENTAILMENT

格式

SEM_APIS.ANALYZE_ENTAILMENT（entailment_name IN VARCHAR2，estimate_percent IN NUMBER DEFAULT to_estimate_percent_type（get_param（'ESTIMATE_PERCENT'）），method_opt IN VARCHAR2 DEFAULT get_param（'METHOD_OPT'），degree IN NUMBER DEFAULT to_degree_type（get_param（'DEGREE'）），cascade IN BOOLEAN DEFAULT to_cascade_type（get_param（'CASCADE'）），no_invalidate IN BOOLEAN DEFAULT to_no_invalidate_type（get_param（'NO_INVALIDATE'）），强制IN BOOLEAN DEFAULT FALSE），network_owner IN VARCHAR2 DEFAULT'MDSYS'，network_name IN VARCHAR2 DEFAULT NULL）;

描述

收集指定蕴涵的统计信息（规则索引）。

参数

entailment_name

蕴涵的名称。

estimate_percent

在内部表分区中估计的行的百分比，包含有关蕴涵的信息（ NULL表示计算）。有效范围是[0.000001,100]。使用常量DBMS_STATS 。 AUTO_SAMPLE_SIZE让Oracle确定适当的样本大小以获得良好的统计信息。这是通常的默认值。

method_opt

对于包含有关蕴涵信息的内部表分区，接受以下任一选项或两者的组合：

  • FOR ALL [INDEXED | HIDDEN] COLUMNS [size_clause]

  • FOR COLUMNS [size clause] column | attribute [size_clause] [，column | attribute [size_clause] ...]

size_clause定义为size_clause := SIZE {integer | REPEAT | AUTO | SKEWONLY}

column定义为column := column_name | (extension)

  • - integer ：直方图桶的数量。必须在[1,254]范围内。
  • - REPEAT ：仅在已有直方图的列上收集直方图。
  • - AUTO ：Oracle根据数据分布和列的工作负载确定要收集直方图的列。
  • - SKEWONLY ：Oracle根据列的数据分布确定要收集直方图的列。
  • - column_name ：列的名称
  • - extension:可以是（ column_name ， column_name [，...]）格式的列组或表达式。

通常的默认值是FOR ALL COLUMNS SIZE AUTO 。

度

包含有关蕴涵信息的内部表分区的并行度。degree的通常默认值为NULL ，这意味着使用CREATE TABLE或ALTER TABLE语句中DEGREE子句指定的表默认值。使用常量DBMS_STATS.DEFAULT_DEGREE根据初始化参数指定默认值。AUTO_DEGREE值自动确定并行度。根据对象的大小，这是1（串行执行）或DEFAULT_DEGREE （基于CPU数和初始化参数的系统默认值）。

级联

收集有关内部表分区的索引的统计信息，其中包含有关蕴涵的信息。使用常量DBMS_STATS.AUTO_CASCADE让Oracle确定是否要收集索引统计信息。这是通常的默认值。

no_invalidate

如果设置为TRUE则不会使从属游标无效。如果设置为FALSE则该过程会立即使相关游标无效。使用DBMS_STATS 。 AUTO_INVALIDATE 。让Oracle决定何时使相关游标无效。这是通常的默认值。

强制

即使蕴涵被锁定， TRUE也会收集统计数据;如果蕴涵被锁定，则FALSE （默认值）不会收集统计信息。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

细粒度访问控制（OLS）注意事项

使用OLS为整个网络启用细粒度访问控制时，只有对关联策略具有FULL访问权限的用户才可以创建蕴涵。启用OLS后，将使用SA_USER_ADMIN.SET_USER_PRIVS过程授予对OLS策略的完全访问权限。

通过生成的标签访问的推断三元组可能与直接从用户可访问的三元组和规则推断的概念三元组不同。使用三重组件子集生成的标签可能比预期的要弱。例如，推断三元组的前提之一可能比三元组中的任何组件具有更高的标签。当标签仅基于三重组件生成时，无法访问其中一个前提的最终用户仍可以访问推断的三元组。即使将前提用于自定义标签生成，生成的标签也可能比预期更强。推理过程并非详尽无遗，并且没有关于推断同一三元组的任何替代方法的信息。因此，使用给定的一组前因生成的标签可能过于强大，因为可以访问备用路径中所有三元组的用户可以通过较低的访问来推断三元组。

即使生成一个主导其所有组件和前因的标签，标签也可能不准确。当考虑支配关系的标签具有非重叠的组信息时就是这种情况。例如，考虑两个标记L:C:NY和L:C:NH其中L是水平，C是组分，NY和NH是两个组。支配这两个标签的简单标签是L:C:NY,NH ，两个标签的真正上限是L:C:US ，其中US是NY和NH的父组。不幸的是，这两个主导标签都不是精确的三重推断从前三个标签的三元组。如果L:C:NY,NH用于推断的三元组，则具有这些组中任一组的成员资格的用户可以访问推断的三元组，而同一用户无权访问其中一个前提条件。另一方面，如果L:C:US用于推断的三元组，则具有两个组但不在美国组中的成员资格的用户将无法访问推断的三元组，而该用户可以通过推断的三元组来推断三元组。直接访问其组件和前提。

由于推断三元组的这些独特挑战，在选择或实施标签生成器时必须格外小心。

10.27 SEM_APIS.CREATE_VIRTUAL_MODEL

格式

描述

vm_name

如果指定此参数，则不能同时指定entailments参数。

选项

创作选项：

  • PXN=T强制虚拟模型的基于UNION ALL的视图定义。这是具有16个或更少组件的虚拟模型的默认值。

  • PXN=F强制虚拟模型的基于IN LIST的视图定义。这是具有16个以上组件的虚拟模型的默认值。

  • PXN=F INMEMORY=T （组合）允许您创建内存中的虚拟模型。

    如果指定INMEMORY=T但不指定PXN=F ，则会创建内存中的虚拟列，但性能会受到影响。如果未指定INMEMORY=T ，则不会在内存中创建虚拟模型。（另请参阅在RDF中使用内存中的虚拟列 。）

  • REPLACE=T允许您替换虚拟模型而不丢弃它。（使用此选项类似于在视图中使用CREATE OR REPLACE VIEW。）

蕴涵

一个或多个蕴涵名称。其数据类型为SEM_ENTAILMENTS，其定义如下： TABLE OF VARCHAR2(25) 。如果此参数为null，则虚拟模型定义中不包含任何蕴涵。使用OWL推理中解释了实施例。

如果指定此参数，则不能同时指定rulebases参数。

network_owner

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

有关虚拟模型的说明（包括使用信息），请参阅虚拟模型 。

对于语义模型和规则库的每个指定组合，必须存在蕴涵。

要创建虚拟模型，您必须是（A）每个指定模型的所有者和任何相应的蕴涵，或者（B）具有DBA权限的用户。

要替换虚拟模型，您必须是虚拟模型的所有者或具有DBA权限的用户。

只有拥有相应的许可证时，才应使用选项INMEMORY=T

此过程使用以下格式创建名称视图：

  • SEMV_ 虚拟机名称 ，其对应于一个UNION ALL中的每个模型并蕴涵的三元组的。此视图可能包含重复项。

  • SEMU_ 虚拟机名称 ，其对应于在每个模型和蕴涵的三元组的一个UNION。该视图不包含重复项（因此，SEMU中的U表示唯一 ）。

要使用从模型m1，m2，m3创建的虚拟模型vm1的虚拟模型中的示例，以及使用OWLPrime规则库为m1，m2和m3创建的蕴涵，此过程将创建以下两个视图（假设m1 ，m2和m3，以及OWLPRIME蕴涵的内部model_id值为1,2,3,4）：

创建视图MDSYS.SEMV_VM1 AS SELECT p_value_id，start_node_id，canon_end_node_id，end_node_id，g_id，model_id FROM MDSYS.rdf_link $ partition（MODEL_1）UNION ALL SELECT p_value_id，start_node_id，canon_end_node_id，end_node_id，g_id，model_id FROM MDSYS.rdf_link $ partition（MODEL_2） UNION ALL SELECT p_value_id，start_node_id，canon_end_node_id，end_node_id，g_id，model_id FROM MDSYS.rdf_link $ partition（MODEL_3）UNION ALL SELECT p_value_id，start_node_id，canon_end_node_id，end_node_id，g_id，model_id FROM MDSYS.rdf_link $ partition（MODEL_4）;创建视图MDSYS.SEMU_VM1 AS SELECT p_value_id，start_node_id，canon_end_node_id，MIN（end_node_id）end_node_id，g_id，MIN（model_id）model_id FROM MDSYS.rdf_link $ WHERE model_id in（1,2,3,4）GROUP BY p_value_id，start_node_id，canon_end_node_id ，g_id;

调用该程序的用户将虚拟模型的拥有者，并有对SEMU_ 虚拟机名称 GRANT特权和SEMV_ 虚拟机名称的意见选择。要查询相应的虚拟模型，用户必须对这些视图具有select权限。

有关语义网络类型和选项的信息，请参阅语义网络 。

以下示例创建名为VM1的虚拟模型。 

执行sem_apis.create_virtual_model（'VM1'，sem_models（'model_1'，'model_2'），sem_rulebases（'OWLPRIME'））;

SELECT value_name FROM MDSYS.RDF_VALUE $ WHERE vname_suffix = SEM_APIS.VALUE_NAME_SUFFIX（'http://www.w3.org/1999/02/22-rdf-syntax-ns#type','UR'）; VALUE_NAME ------------------------------------------------- ------------------------------- http://www.w3.org/1999/02/22-rdf-syntax -ns＃类型
