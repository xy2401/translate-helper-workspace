1了解Oracle Text Application Development

Oracle Text使您可以构建文本查询应用程序和文档分类应用程序。

本章包含以下主题：

  • Oracle Text简介

  • 文件收集应用

  • 目录信息应用

  • 文件分类应用

  • XML搜索应用程序

1.1 Oracle Text简介

Oracle Text为查询应用程序和文档分类应用程序中的文本提供索引，单词和主题搜索以及查看功能。

要设计Oracle Text应用程序，请首先确定您希望运行的查询类型。了解类型后，可以为任务选择最合适的索引。

Oracle Text用于以下类别的应用程序：

  • 文件收集应用

  • 目录信息应用

  • 文件分类应用

  • XML搜索应用程序

1.2文件收集应用

一个文本查询应用程序使用户能够搜索 文档集合 ，例如网站，数字图书馆或文档仓库。

本节包含以下主题。

  • 关于文档集应用程序

  • 文本查询应用程序流程图

1.2.1关于文档收集应用程序

该集合通常是静态的，并且在初始索引运行之后内容没有显着变化。文档可以是任何大小和格式，例如HTML，PDF或Microsoft Word。这些文档存储在文档表中。首先索引文档集合即可启用搜索。

查询通常由单词或短语组成。应用程序用户通过使用OR和AND.等运算符指定单词和短语的逻辑组合AND.用户可以应用其他查询操作来改进搜索结果，例如词干，邻近搜索和通配符。

对于此类应用程序，您应检索与查询最相关的文档。文档必须在结果列表中排名很高。

查询最好配备一个文档表上的CONTEXT索引。要查询此索引，应用程序将使用 SQL CONTAINS运算符 WHERE子句 SELECT语句。

1.3目录信息应用

目录信息包括库存类型信息，例如在线书店或拍卖站点。

1.3.1关于目录信息应用程序

存储的目录信息包括文本信息，例如书名，以及相关的结构化信息，例如价格。通常会定期更新信息，以使在线目录与库存保持同步。

查询通常是文本组件和结构化组件的组合。结果几乎总是按结构化组件排序，例如日期或价格。对于这种类型的查询应用程序，良好的响应时间始终是一个重要因素

目录应用程序最好由a CTXCAT指数。用这个查询这个索引 SELECT语句的WHERE子句中的CATSEARCH运算符。

1.4文件分类申请

在一个文档分类应用程序，将传入流或一组文档与预定义的一组规则进行比较。如果文档与一个或多个规则匹配，则应用程序执行操作。

例如，假设传入的新闻文章流。您可以定义规则以表示财务类别。该规则基本上是一个或多个查询，用于选择有关财务主题的文档。该规则可能具有“股票或债券或收益”的形式。“

当文档达到华尔街盈利预测并满足此类别的规则时，应用程序会采取措施，例如将文档标记为财务或通过电子邮件发送给一个或多个用户。

要创建文档分类应用程序，请创建规则表，然后创建一个 CTXRULE指数。要对传入的文本流进行分类，请使用 SELECT语句的WHERE子句中的MATCHES运算符。请参见图1-5 分类申请的一般流程。

1.5 XML搜索应用程序

一个 XML搜索应用程序执行XML文档搜索。常规文档搜索通常搜索一组文档以返回满足文本谓词的文档; XML搜索通常使用XML文档的结构来限制搜索。通常，仅返回满足搜索的文档部分。例如，用户可能只需要注释字段包含电子的采购订单，而不是查找包含单词electric的所有采购订单。

Oracle Text使您可以使用以下方法执行XML搜索：

  • 具有XML搜索应用程序的CONTAINS运算符

  • 将Oracle Text Features与Oracle XML DB相结合（XML搜索索引）

1.5.1带有XML搜索应用程序的CONTAINS运算符

该 CONTAINS运算符非常适合结构化搜索，使您能够使用WITHIN ， HASPATH和INPATH运算符执行限制性搜索。如果你使用 CONTEXT索引中，您还可以从Oracle Text搜索的以下特征中受益：

  • 基于令牌的空白规范化搜索

  • Hitlists按相关性排名

  • 区分大小写的搜索

  • 科搜索

  • 词干和模糊搜索等语言特征

  • 针对大型文档集的性能优化查询

1.5.2.1对XML搜索索引使用xml_enable方法

XML搜索索引是一个支持XML的Oracle Text索引（CTXSYS.CONTEXT）。该索引类型在一个统一索引中支持信息检索搜索和结构化搜索。XML搜索索引还在Oracle Text表内部存储二进制持久性文档对象模型（PDOM），以便可以通过二进制PDOM对XML操作进行功能评估。 XMLTYPE数据存储支持此XML搜索索引。在存在这样的XML搜索索引的情况下，XMLEXISTS被无缝地重写为CONTAINS查询。

创建XML搜索索引时，XML文档的二进制PDOM将在Oracle Text的内部表中实现。来自Oracle Text索引的后评估被重定向以与存储在此内部表中的PDOM相对。

也可以看看：

Oracle Text的参考的信息， xml_enable的变量SET_SEC_GRP_ATTR启用XML搜索索引XML意识

以下示例创建Oracle XML搜索索引：

exec CTX_DDL.CREATE_SECTION_GROUP（'secgroup'，'PATH_SECTION_GROUP'）; exec CTX_DDL.SET_SEC_GRP_ATTR（'secgroup'，'xml_enable'，'t'）;关于T（X）indextype的CREATE INDEX po_ctx_idx是ctxsys.context参数（'section group SECGROUP'）;
1.5.2.2使用Text-on-XML方法

使用Oracle Text，您可以在包含XML数据的列上创建CONTEXT索引。列类型可以是XMLType或任何受支持的类型，前提是您对XML数据使用正确的索引首选项。

使用Text-on-XML方法，使用标准CONTAINS查询并添加结构化约束，以将搜索范围限制为特定部分，字段，标记或属性。也就是说，在文本运算符中指定结构，例如WITHIN ， HASPATH和INPATH.

例如，设置您的CONTEXT索引以创建包含XML文档的部分。请考虑以下定义采购订单的XML文档：

<？xml版本=“1.0”？> <PURCHASEORDER pono =“1”> <PNAME> Po_1 </ PNAME> <CUSTNAME> John </ CUSTNAME> <SHIPADDR> <STREET> 1033 Main Street </ STREET> <CITY> Sunnyvalue </ CITY> <STATE> CA </ STATE> </ SHIPADDR> <ITEMS> <ITEM> <ITEM_NAME>戴尔计算机</ ITEM_NAME> <DESC>奔腾2.0 Ghz 500MB RAM </ DESC> </ ITEM> <ITEM> <ITEM_NAME> Norelco R100 </ ITEM_NAME > <DESC>电动剃须刀</ DESC> </ ITEM> </ ITEMS> </ PURCHASEORDER>

要在项目描述部分中查询包含Pentium的所有采购订单，请使用WITHIN运算符：

来自po_tab的SELECT id，其中CONTAINS（doc，'Pentium WITHIN desc'）> 0;

使用INPATH运算符为XPATH表达式指定更复杂的条件：

来自po_tab的SELECT id，其中CONTAINS（doc，'Pentium INPATH（/ purchaseOrder / items / item / desc'）> 0;
