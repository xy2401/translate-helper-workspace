3.6处理缺失值

Oracle Data Mining区分 稀疏数据和包含的数据 随机缺失值 。后者意味着某些属性值未知。疏另一方面，数据包含假设已知的值，尽管它们未在数据中表示。

稀疏数据的典型示例是市场篮子数据。在数百或数千种可用物品中，只有少数存在于个别情况（篮子或交易）中。所有项目值都是已知的，但它们并非都包含在购物篮中。当前值具有数量，而未表示的项目是稀疏的（已知数量为零）。

Oracle Data Mining按如下方式解释缺少的数据：

  • 随机丢失：假设在具有简单数据类型（未嵌套）的列中缺少值，则随机丢失。

  • 稀疏：嵌套列中缺少值表示稀疏性。

3.6.1示例：缺少值还是稀疏数据？

本节中的示例说明了Oracle Data Mining如何将数据标识为稀疏或随机丢失。

3.6.1.1销售表中的稀疏性

销售表包含一组产品的销售点数据，这些产品在一段时间内在多个商店中销售给不同的客户。特定客户只购买少数产品。客户未购买的产品不会在销售表中显示为行。

如果您要计算出客户为每种产品花费的金额，那么未购买的产品的推断量为零。该值不是随机的或未知的;它为零，即使表中没有行。

请注意，销售数据的尺寸标注（按产品，商店，客户和时间），通常表示为挖掘的嵌套数据。

由于嵌套列中的缺失值始终表示稀疏性，因此必须确保此解释适用于要挖掘的数据。例如，当尝试从大型电影数据库的用户挖掘包含电影评级的多记录案例数据集时，缺失的评级是未知的（随机丢失），但Oracle数据挖掘将数据视为稀疏并推断评级为零缺失值。

3.6.1.2客户数据表中缺少值

客户数据表包含有关客户的人口统计数据。案例ID列是客户ID。属性包括年龄，教育，职业，性别，家庭大小等。并非所有数据都适用于每个客户。任何缺失值都被认为是随机丢失的。例如，如果数据中不存在客户1的年龄和客户2的职业，那么该信息就是未知的。它并不表示稀疏性。

请注意，客户数据未标注尺寸。案例与其每个属性之间存在一对一的映射。没有任何属性是嵌套的。

该算法处理稀疏数据。

3.6.3改变缺失值处理

随机将丢失的数据转换为稀疏或丢失。

如果您希望Oracle Data Mining将丢失的数据视为稀疏而不是随机丢失或随机丢失而不是稀疏，请在构建模型之前对其进行转换。

如果希望将缺失值视为稀疏值，但Oracle Data Mining会将它们随机解释为缺失，则可以使用类似NVL的SQL函数将空值替换为“NA”等值。当存在指定值时，Oracle Data Mining不会执行缺失值处理。

如果要将缺少的嵌套属性视为随机丢失，则可以将嵌套行转换为单独列中的物理属性 - 只要案例表保持在数据库强加的1000列限制范围内。填写所有可能的属性名称，并将它们指定为null。或者，在嵌套列中为所有不存在的项插入行，并为每个项分配一个值，例如均值或模式。
