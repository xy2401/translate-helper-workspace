协会统计

目的

使用ASSOCIATE STATISTICS语句将包含与统计信息收集，选择性或成本相关的函数的统计信息类型（或缺省统计信息）与一个或多个列，独立函数，包，类型，域索引或索引类型相关联。

有关所有当前统计类型关联的列表，请查询USER_ASSOCIATIONS数据字典视图。如果分析与关联统计信息关联的对象，则还可以在USER_USTATS视图中查询关联。

先决条件

要发出此语句，您必须具有相应的权限才能更改基础对象（表，函数，包，类型，域索引或索引类型）。此外，除非仅关联默认统计信息，否则必须对统计信息类型具有执行权限。必须已经定义了统计类型。

语义

column_association

指定一个或多个表列。如果未指定schema ，则Oracle数据库假定该表位于您自己的模式中。

function_association

指定一个或多个独立函数，包，用户定义的数据类型，域索引或索引类型。如果未指定schema ，则Oracle数据库假定该对象位于您自己的模式中。

  • FUNCTIONS仅指独立函数，不涉及方法类型或内置函数。

  • TYPES仅指用户定义的类型，而不是内置的SQL数据类型。

对 function_association的 限制

您无法指定已为其定义关联的对象。您必须首先取消此统计信息与此对象的关联。

也可以看看：

DISASSOCIATE STATISTICS “ 关联统计：示例 ”

using_statistics_type

指定与列，函数，包，类型，域索引或索引类型关联的统计信息类型（或类型的同义词）。必须已创建statistics_type 。

只有在将统计信息与列或索引相关联时， NULL关键字才有效。将统计信息类型与对象类型相关联时，该对象类型的列将继承统计信息类型。同样，将统计信息类型与索引类型相关联时，indextype的索引实例将继承统计信息类型。您可以通过为列或索引关联不同的统计信息类型来覆盖此继承。或者，如果您不想为列或索引关联任何统计信息类型，则可以在using_statistics_type子句中指定NULL 。

对指定统计类型的限制

您不能为函数，包，类型或索引类型指定NULL 。

也可以看看：

Oracle数据库数据盒开发人员指南 ，了解有关创建统计信息收集功能的信息

default_cost_clause

指定独立函数，包，类型，域索引或索引类型的默认成本。如果指定此子句，则必须按顺序为CPU成本，I / O成本和网络成本分别包含一个数字。每个成本用于单个执行函数或方法或用于单个域索引访问。可接受的值是零或更大的整数。

default_selectivity_clause

使用独立函数，类型，包或用户定义的运算符指定谓词的默认选择性百分比。default_selectivity_clause必须是0到100之间的数字。超出此范围的值将被忽略。

对 default_selectivity_clause的 限制

您不能指定DEFAULT SELECTIVITY域的索引或索引类型。

也可以看看：

“ 指定默认成本：示例 ”

storage_table_clause

本节仅适用于INDEXTYPE统计数据。

  • 指定WITH SYSTEM MANAGED STORAGE TABLES以指示统计数据的存储将由系统管理。您在statistics_type指定的类型应该将统计信息相关信息存储在由系统维护的表中。此外，必须已创建或更改指定的索引类型以支持WITH SYSTEM MANAGED STORAGE TABLES子句。

  • 指定WITH USER MANAGED STORAGE TABLES以指示存储用户定义的统计信息的表将由用户管理。这是默认行为。
