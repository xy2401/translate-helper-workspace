使用DBCA为Oracle RAC指定数据库初始化参数

如果要在群集中添加更多节点，请指定初始化参数。

在“初始化参数”页上，如果要在群集中添加比当前DBCA会话期间更多的节点，请单击“ 所有初始化参数” ，并将参数CLUSTER_DATABASE_INSTANCES更改为计划添加到群集的节点总数。

如果单击“ 所有初始化参数” ，请注意，如果全局数据库名称超过8个字符，则数据库名称值（在DB_NAME参数中）将截断为前8个字符，并且DB_UNIQUE_NAME参数值将设置为全局名称。

如果你是一个Linux系统上安装，请注意， 内存大小（SGA和PGA），其设置初始化参数MEMORY_TARGET或MEMORY_MAX_TARGET ，不能超过你的操作系统上的共享内存文件系统更大。

例如，如果系统上的共享内存文件系统分配为1 GB，但是将内存大小（ MEMORY_TARGET ）设置为2 GB，则在数据库启动期间会收到以下错误：

ORA-00845：此系统不支持MEMORY_TARGET ORA-01078：处理系统参数失败

此问题与其他平台无关。
