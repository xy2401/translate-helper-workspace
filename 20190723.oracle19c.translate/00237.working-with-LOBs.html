LOB列状态

访问LOB列中的单元格时使用的技术因给定单元格的状态而异。

LOB列中的单元格可以处于以下状态之一：

  • 空值

    创建表格单元格，但单元格不包含定位符或值。

  • 空

    具有定位符的LOB实例存在于单元格中，但它没有值。LOB的长度为零。

  • 人口稠密

    单元格中存在具有定位符和值的LOB实例。

锁定包含LOB的行

您可以锁定包含LOB的行，以防止其他数据库用户在事务期间写入LOB。

  • 要锁定行，请在选择行时指定FOR UPDATE子句。当行被锁定时，其他用户无法锁定或更新LOB，直到您结束交易。

LOB开放和关闭操作

LOB API包括使您能够显式打开和关闭LOB实例的操作。

您可以打开和关闭任何类型的持久性LOB实例： BLOB ， CLOB ， NCLOB或BFILE 。您打开LOB以实现以下一个或两个结果：

  • 以只读模式打开LOB

    这可确保在您明确关闭LOB之前，不能在会话中更改LOB（LOB定位符和LOB值）。例如，您可以打开LOB以确保在关键操作中使用LOB时，程序的其他部分不会更改LOB。执行操作后，您可以关闭LOB。

  • 对于持久性BLOB ， CLOB或NCLOB实例，以读写/模式打开LOB

    在读/写模式下打开LOB会延迟LOB列上的任何索引维护，直到您关闭LOB。以读/写模式打开LOB仅在LOB列上存在可扩展索引时才有用，并且您不希望每次写入LOB时数据库都执行索引维护。如果您在LOB打开时对LOB执行多次写操作，则此技术可以提高应用程序的性能。

如果您打开LOB，则必须在会话后的某个时刻关闭LOB。这是开放式LOB的唯一要求。当LOB实例打开时，您可以在LOB上执行任意数量的操作 - 前提是在给定模式下允许操作。

使用LOB定位器访问和修改LOB值

您可以使用LOB定位器来访问和修改LOB值。

LOB定位器是对LOB值位置的引用，可以访问存储在数据库中的LOB实例的值。数据库表仅存储CLOB ， BLOB ， NCLOB和BFILE列中的定位符。

请注意以下有关LOB定位器和值的信息：

  • LOB定位器被传递到各种LOB API以访问或操作LOB值。

  • 可以将LOB定位器分配给相同类型的任何LOB实例。

  • LOB实例的特征是临时或持久，但定位器不是。

LOB定位器和BFILE定位器

LOB类型BLOB ， CLOB和NCLOB的定位符的语义与BFILE类型的定位符的语义之间存在差异：

  • 对于LOB类型BLOB ， CLOB和NCLOB ，LOB列将定位符存储到LOB值。每个LOB实例都有自己独特的LOB定位符，也是LOB值的不同副本。

  • 对于初始化的BFILE列，该行将定位器存储到保存BFILE值的外部操作系统文件。给定行中的每个BFILE实例都有自己独特的定位符;但是，两个不同的行可以包含指向同一操作系统文件的BFILE定位符。

无论LOB的值存储在何处，定位器都存储在任何已初始化的LOB列的表行中。此外，当您从表中选择LOB时，返回的LOB始终是临时LOB。

注意：

当使用术语定位符而没有标识前缀术语时，它指的是LOB定位符和BFILE定位符。

LOB列初始化

NULL LOB实例没有定位器。

在将LOB实例传递给任何LOB API例程之前，实例必须包含一个定位器。例如，您可以从行中选择NULL LOB，但不能将实例传递给PL / SQL DBMS_LOB.READ过程。您必须初始化LOB实例，该实例为其提供定位器，以使其为非NULL 。然后你可以传递LOB实例。

在使用受支持的编程环境接口（如PL / SQL，OCI，Visual Basic或Java）开始将数据写入持久性LOB之前，必须使LOB列/属性为非NULL 。

LOB列的规则

LOB列受以下规则和限制的约束：

  • 您不能将LOB指定为主键列。

  • Oracle数据库对远程LOB的支持有限，并且当以不受支持的方式使用远程LOB时，可能会发生ORA-22992错误。

  • 群集不能包含LOB，可以是key或nonkey列。

  • 仅支持以下数据结构作为临时实例。您无法将这些实例存储在数据库表中：

    • 任何LOB类型的VARRAY

    • 包含LOB类型的任何类型的VARRAY ，例如具有LOB属性的对象类型

    • 任何LOB类型的ANYDATA

    • 包含LOB的任何类型的ANYDATA

  • 您不能在查询的ORDER BY子句，查询的GROUP BY子句或聚合函数中指定LOB列。

  • 您不能在SELECT指定LOB列...DISTINCT或SELECT ...UNIQUE语句或在连接中。但是，您可以在SELECT指定对象类型列的LOB属性...DISTINCT语句，使用UNION的查询，或者如果列的对象类型在其上定义了MAP或ORDER函数，则使用MINUS集合运算符。

  • LOB段的第一个（ INITIAL ）范围必须至少包含三个数据库块。

  • 最小范围大小为14个街区。对于8K块大小（默认值），这相当于112K。

  • 创建AFTER UPDATE DML触发器时，不能在UPDATE OF子句中指定LOB列。

  • 您不能将LOB列指定为索引键的一部分。但是，您可以在域索引的索引类型规范中指定LOB列。此外，Oracle Text允许您在CLOB列上定义索引。

  • 在INSERT ......AS SELECT操作，您可以将最多4000个字节的数据绑定到LOB列和属性。INSERT时没有长度限制...使用没有绑定变量的SQL从一个表到另一个表的AS SELECT 。

  • 如果表具有LONG和LOB列，则不能将超过4000个字节的数据绑定到同一SQL语句中的LONG和LOB列。但是，您可以将超过4000个字节的数据绑定到LONG或LOB列。

注意：

对于已定义AFTER UPDATE DML触发器的表，如果使用OCI函数或DBMS_LOB包更改LOB列的值或对象类型列的LOB属性，则数据库不会触发DML触发器。

LOB操作的限制

LOB操作有一定的限制。

一般LOB限制包括以下内容：

  • 在SQL Loader中，从LOB读取的字段不能用作子句的参数。

  • CLOB列上不区分大小写的搜索通常不会成功。例如，要对CLOB列执行不区分大小写的搜索：

    ALTER SESSION SET NLS_COMP = LINGUISTIC; ALTER SESSION SET NLS_SORT = BINARY_CI; SELECT * FROM ci_test WHERE LOWER（clob_col）LIKE'aa％';

    没有LOWER功能，选择失败。您可以使用Oracle Text或DBMS_LOB.INSTR()进行不区分大小写的搜索。

  • 会议共享服务器（多线程服务器）模式下的BFILE不支持迁移。这意味着打开的BFILE的操作可以持续超出对共享服务器的调用结束。在共享服务器会话中， BFILE操作绑定到一个共享服务器，它们无法从一个服务器迁移到另一个服务器。

  • 打开BFILE时，目录路径或文件名中不允许使用符号链接。检查整个目录路径和文件名，如果找到任何符号链接，则返回以下错误：

    ORA-22288：文件或LOB操作FILEOPEN路径中的软链接失败
