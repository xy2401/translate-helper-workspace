Oracle对SQL / Foundation的可选功能的支持

表C-2中列出了Oracle对SQL / Foundation可选功能的支持：

表C-2 Oracle对SQL / Foundation可选功能的支持

功能ID，功能 支持

B012，嵌入式C

Oracle完全支持此功能。

B013，嵌入式COBOL

Oracle完全支持此功能。

B021，直接SQL

Oracle完全支持此功能，如SQL * Plus。

B031，基本动态SQL

Oracle支持两种样式的动态SQL，在嵌入式语言手册中记录为“Oracle动态SQL”和“ANSI动态SQL”。

ANSI动态SQL是标准的实现，具有以下限制：

  • Oracle支持描述符项的子集。

  • 对于<input using clause>，Oracle仅支持<using input descriptor>。

  • 对于<output using clause>，Oracle仅支持<into descriptor>。

  • 动态参数由冒号后跟标识符而不是问号表示。

Oracle动态SQL与标准动态SQL类似，具有以下修改：

  • 参数由冒号后跟标识符表示，而不是问号。

  • Oracle的DESCRIBE SELECT LIST FOR语句取代了标准的DESCRIBE OUTPUT 。

  • 如果要在准备动态SQL语句的PREPARE语句之前使用动态SQL语句声明游标，Oracle会提供DECLARE STATEMENT 。

B032，扩展动态SQL

在ANSI动态SQL中，Oracle仅实现了从此功能声明全局语句和全局游标的功能;其余功能不受支持。

在Oracle动态SQL中，Oracle的DESCRIBE BIND VARIABLES等同于标准的DESCRIBE INPUT ;此功能的其余部分不受支持。

B122，常规语言C.

Oracle支持用C编写的外部例程，尽管Oracle不支持创建此类例程的标准语法。

B128，常规语言SQL

Oracle支持用PL / SQL编写的例程，这是Oracle等同于标准过程语言SQL / PSM。

F032， CASCADE丢弃行为

在Oracle中， DROP命令使所有被删除对象的依赖对象无效。在以允许成功重新编译无效对象的方式重新定义被删除对象之前，无效对象实际上是不可用的。

F033， ALTER TABLE语句： DROP COLUMN子句

Oracle提供了DROP COLUMN子句，但没有标准中的RESTRICT或CASCADE选项。

F034，扩展REVOKE语句

Oracle支持此功能的以下部分：

  • F034-01， REVOKE语句由架构对象的所有者执行

  • F034-03， REVOKE语句撤销被授权者具有WITH GRANT OPTION的权限

Oracle为此功能的以下部分提供了等效功能：

  • CASCADE ：在Oracle中， REVOKE使所有依赖对象无效，这些对象在通过后续CREATE和GRANT命令更改元数据之前实际上无法使用，从而无法成功重新编译无效对象。

F052，间隔和日期时间算术

Oracle仅支持INTERVAL YEAR TO MONTH和INTERVAL DAY TO SECOND数据类型。

F111， SERIALIZABLE以外的隔离级别

除了SERIALIZABLE ，Oracle还支持READ COMMITTED隔离级别。

F121，基本诊断管理

该功能的大部分功能都是通过嵌入式语言的SQLCA提供的。

F191，参照删除操作

Oracle支持ON DELETE CASCADE和ON DELETE SET NULL 。

F200， TRUNCATE TABLE

Oracle完全支持此功能，并通过允许截断在引用完整性约束中引用自身的表，以及使用已启用的ON DELETE CASCADE引用约束级联到子表的功能来扩展它。

F231，权限表

Oracle在以下元数据视图中提供此信息：

  • 而不是TABLE_PRIVILEGES ，使用ALL_TAB_PRIVS 。

  • 而不是COLUMN_PRIVILEGES ，使用ALL_COL_PRIVS 。

  • Oracle不支持USAGE权限，因此不存在USAGE_PRIVILEGES等效USAGE_PRIVILEGES 。

F281， LIKE增强功能

Oracle完全支持此功能。

F291， UNIQUE谓词

IS A SET条件可用于测试多集是否是集合;也就是说，每一行都是唯一的。因此，相当于

UNIQUE <table子查询>

是

CAST（<table子查询> AS MULTISET）是一个SET

F302， INTERSECT表操作员

Oracle支持INTERSECT但不支持INTERSECT ALL 。从语法上讲，Oracle与标准的不同之处在于UNION ， INTERSECT和MINUS具有相同的优先级。

F312， MERGE声明

Oracle MERGE语句与标准几乎相同，但有以下例外：

  • Oracle在表别名之前不支持可选的AS关键字。

  • Oracle不支持USING表别名后面的带括号的列名列表重命名USING子句中指定的表的列。

  • Oracle不支持<override子句>。

F314，带DELETE分支的MERGE语句

Oracle具有类似的功能，但在Oracle中，您必须首先更新一行，之后如果修订的行符合条件，您可以删除它。

F321，用户授权

Oracle为以下子功能提供了相同的功能：

  • 使用SYS_CONTEXT ('USERENV', 'SESSION_USER')代替SESSION_USER

  • 使用SYS_CONTEXT ('USERENV', 'CURRENT_USER')代替CURRENT_USER

Oracle不支持以下子功能：

  • SYSTEM_USER

  • SET SESSION AUTHORIZATION语句

F341，使用表

Oracle在视图ALL_DEPENDENCIES ， DBA_DEPENDENCIES和USER_DEPENDENCIES提供此信息。

F381，扩展模式操作

Oracle完全支持此功能的以下元素：

  • Oracle支持使用ALTER TABLE添加表约束的标准语法。

Oracle部分支持此功能的以下元素：

  • 除了Oracle不支持RESTRICT之外，Oracle支持删除表约束的标准语法。

Oracle为此功能的以下元素提供了等效功能：

  • 要更改列的默认值，请使用ALTER TABLE的MODIFY选项。

Oracle不支持此功能的以下部分：

  • DROP SCHEMA声明

  • ALTER ROUTINE声明

F382，修改列数据类型

Oracle支持此功能，但使用非标准语法。作为标准的扩展，Oracle允许您减小列的大小或精度。

F383，Set column not null子句

Oracle为此功能的两个子功能提供了相同的功能：

  • 要向现有列添加NOT NULL约束，请使用ALTER TABLE ...修改

  • 要删除NOT NULL约束，请使用ALTER TABLE按名称删除约束

F384，Drop identity属性子句

Oracle使用ALTER TABLE提供相同的功能...MODIFY ( ...DROP IDENTITY )

F386，设置标识列生成子句

Oracle提供了相同的功能。Oracle的语法和语义与标准相同，但有以下异常：

  • Oracle不支持RESTART ;请改用START WITH 。重新启动标识列时，除非在ALTER TABLE语句中显式设置，否则标识列的其他参数的值将重置为其默认值。

Oracle的START WITH LIMIT VALUE选项是标准的扩展。

F391，长标识符

Oracle支持最长128个字符的标识符。

F393，Unicode在文字中逃脱

Oracle UNISTR函数支持所有Unicode字符的数字转义序列。

F394，可选的普通表格规范

此功能将关键字NFC ， NFD ， NFKC和NKD到NORMALIZE函数和IS NORMAL谓词。没有这些关键字， NFC是默认设置（参见功能T061，UCS支持）。Oracle支持所有四种规范化形式，使用非标准语法，如下所示：

  • 对于NFC ，请使用COMPOSE

  • 对于NFD ，请使用DECOMPOSE和CANONICAL选项

  • 对于NFKD ，请使用DECOMPOSE和COMPATIBILITY选项

  • 对于NFKC ，请使用DECOMPOSE和CANONICAL选项，然后使用COMPOSE

Oracle不支持IS NORMAL谓词。

F401，扩展连接表

Oracle支持FULL外连接， CROSS连接和NATURAL连接。

F402，LOB，数组和多集的命名列连接

Oracle支持声明类型为嵌套表的列的命名列连接。Oracle不支持LOB或数组的命名列连接。

F403，分区连接表

除了FULL外连接外，Oracle支持此功能。

F411，时区规范

Oracle完全支持TIMESTAMP WITH TIME ZONE ，但不支持TIME WITH TIME ZONE 。

F421，国民性

Oracle完全支持此功能。

F431，只读可滚动游标

Oracle完全支持此功能。

F441，扩展设置功能支持

Oracle支持此功能的以下部分：

  • WHERE子句中的能力引用在视图或内联视图中使用聚合定义的列

  • COUNT没有表达式的DISTINCT

  • 引用与聚合查询相关的外部引用列的聚合。但是，Oracle将聚合查询定义为包含聚合的最内层查询，而不是定义聚合中引用的范围变量的最内层查询。

F442，集合函数中的混合列引用

Oracle完全支持此功能。

F461，命名字符集

Oracle支持许多具有Oracle定义名称的字符集。Oracle不支持此功能的任何其他方面。

F491，约束管理

Oracle完全支持此功能。

F492，可选表约束实施

标准中的ENFORCED相当于Oracle中的ENABLE VALIDATE 。NOT ENFORCED标准相当于DISABLE NOVALIDATE在Oracle中。ENABLE |的其他组合DISABLE ， VALIDATE | NOVALIDATE ，和RELY | NORELY选项是标准的扩展。

F531，临时表

Oracle支持GLOBAL TEMPORARY表。

F555，增强秒精度

Oracle为此功能提供增强支持，在小数点后最多支持9个位置。

F561，全值表达式

Oracle完全支持此功能。

F571，真值测试

Oracle的LNNVL函数等同于标准的IS NOT TRUE谓词。

F591，派生表

Oracle支持<derived table>，但以下情况除外：

  • Oracle在表别名之前不支持可选的AS关键字。

  • Oracle不支持<派生列列表>。

F641，行和表构造函数

在Oracle中，行构造函数可用于与另一行构造函数或子查询的相等或不等式比较。Oracle不支持此功能中的任何其他内容。

F690，整理支持

Oracle的NLSSORT函数可用于更改字符表达式的排序规则。

F693，SQL会话和客户端模块排序规则

要设置会话排序规则，请使用ALTER SESSION SET NLS_COMP = 'LINGUISTIC' ，并将NLS_SORT设置为所需的排序规则。Oracle不支持客户端模块排序规则。

F695，翻译支持

Oracle CONVERT函数可以在数据库字符集和国家字符集之间进行转换。对于其他字符集，将数据存储在RAW数据类型中并使用PL / SQL包函数UTL_RAW 。 CONVERT 。 Oracle不提供添加或删除字符集转换的功能。

F721，延迟约束

Oracle完全支持此功能。

F731， INSERT列权限

Oracle完全支持此功能。

F761，会话管理

Oracle为此功能的元素提供以下等价物：

  • 相当于标准的SET SESSION CHARACTERISTICS AS TRANSACTION SERIALIZABLE是ALTER SESSION SET ISOLATION_LEVEL = SERIALIZABLE 。

  • 与标准的SET SCHEMA相当的是ALTER SESSION SET CURRENT_SCHEMA 。

  • 相当于标准的SET COLLATION是ALTER SESSION SET NLS_SORT 。

F763， CURRENT_SCHEMA

Oracle的等价物是SYS_CONTEXT ('USERENV', 'CURRENT_SCHEMA')

F771，连接管理

Oracle的CONNECT语句提供与标准的CONNECT语句相同的功能，但语法不同。Oracle不提供标准的SET CONNECTION ，而是提供AT子句来指示应该对哪个连接执行SQL语句。Oracle嵌入式语言允许您使用COMMIT或ROLLBACK的RELEASE选项断开连接。

F845， TRANSLATE_REGEX函数

S023，基本结构化类型

Oracle的对象类型等同于标准中的结构化类型。

S024，增强的结构化类型

Oracle的语法是非标准的，但提供了以下内容的等价物：

  • NOT INSTANTIABLE

  • STATIC方法

  • RELATIVE ， MAP和STATE排序。Oracle for RELATIVE排序中的关键字是ORDER 。 STATE排序没有关键字（如果没有定义其他排序，这是默认值）。不同于标准，Oracle不支持EQUALS ONLY在非STATE排序。（另请参阅功能S251，用户定义的排序。）

  • SELF AS RESULT在构造函数方法的签名中

S025，最终结构化类型

Oracle的最终对象类型等同于标准中的最终结构化类型。

S026，自引用结构化类型

在Oracle中，对象类型OT可能具有引用OT的引用。

S041，基本参考类型

Oracle的引用类型等同于标准中的引用类型。要取消引用引用，使用点表示法，而不是 - >在标准中。

S043，增强型参考类型

Oracle支持此功能的以下元素：

  • DEREF运算符返回引用引用的对象

  • SCOPE子句作为表或物化视图列的约束

  • 添加和删除列的范围

  • 系统生成或从主键派生的引用（但不是来自任何其他列列表，也不是来自类型属性列表）

S051，创建类型表

Oracle的对象表相当于标准中的结构化类型的表。

S081，子表

Oracle支持对象视图的层次结构，但不支持对象基表的层次结构。要模拟基表的层次结构，请在这些基表上创建视图层次结构。

S091，基本阵列支持

Oracle VARRAY类型等同于标准中的数组类型。但是，Oracle不支持存储LOB阵列。要使用下标访问数组的单个元素，必须使用PL / SQL。 Oracle使用非标准语法支持此功能的以下方面：

  • 要构造varray类型的实例（包括空数组），请使用varray类型构造函数。

  • 要在FROM子句中删除varray，请使用TABLE运算符。

  • 要获得varray的基数，请使用PL / SQL中的COUNT方法。

S092，用户定义类型的数组

Oracle支持VARRAY的对象类型。

S094，参考类型数组

Oracle支持VARRAY的引用。

S095，通过查询的数组构造函数

Oracle使用CAST支持此功能（ MULTISET （ SELECT ...）AS varray_type ) 。不支持使用ORDER BY对数组元素进行排序的功能。

S097，数组元素赋值

在PL / SQL中，您可以使用类似于标准（SQL / PSM）的语法分配给数组元素。

S098， ARRAY_AGG

Oracle没有导致varray的聚合。相反， COLLECT聚合可用于创建多集，可以强制转换为元素类型的数组。

S111， ONLY在查询表达式中

Oracle支持视图层次结构的ONLY子句; Oracle不支持基表的层次结构。

S151，类型谓词

Oracle完全支持此功能。

S161，亚型治疗

Oracle完全支持此功能。

S162，亚型治疗参考文献

支持，具有轻微的语法差异：标准需要围绕引用类型名称的括号; Oracle不支持此位置的括号。

S201，SQL上调用的数组例程

PL / SQL提供了将数组作为参数传递并作为函数结果返回数组的功能。用C语言编写的过程和函数可以使用Oracle类型转换器（OTT）作为函数的结果传递数组和返回数组。

S202，多重集合上的SQL调用例程

PL / SQL例程可以将嵌套表作为参数，并且可以返回嵌套表。用C语言编写的例程可以使用Oracle Type Translator传递数组并返回数组作为函数的结果。

S232，阵列定位器

Oracle Type Translator支持数组描述符，实现与定位符相同的目的。

S233，Multiset定位器

Oracle支持嵌套表的定位器。

S241，变换功能

Oracle Type Translator提供与转换相同的功能。

S251，用户定义的排序

Oracle的对象类型排序功能对应于标准的功能，如下所示：

  • Oracle的MAP排序对应于标准的ORDER FULL BY MAP排序。

  • Oracle的ORDER排序对应于标准的ORDER FULL BY RELATIVE排序。

  • 如果Oracle对象类型既没有声明MAP也没有声明ORDER ，那么这对应于标准中的EQUALS ONLY BY STATE 。

  • Oracle没有无序的对象类型;你可以改变订单，但你不能放弃它。

S261，指定类型方法

ANYDATA类型的GetTypeName方法可用于学习类型的名称。

S271，基本多集支持

标准中的多字节支持作为Oracle中的嵌套表类型。基于标量类型ST的Oracle嵌套表数据类型在标准术语中等同于具有单个ST类型字段且名为column_value 。基于对象类型的Oracle嵌套表类型等同于标准中的结构化类型的多集。

Oracle使用与标准对多集合相同的语法，在嵌套表上支持此功能的以下元素：

  • CARDINALITY功能

  • SET功能

  • MEMBER谓词

  • IS A SET谓词

  • COLLECT聚合

非标准语法支持此功能的所有其他方面，如下所示：

  • 要创建标准中表示为MULTISET[]的空多重集，请使用嵌套表类型的空构造函数。

  • 要获取具有一个元素的多集的唯一元素，在标准中表示为ELEMENT （<multiset value expression>），请使用标量子查询从嵌套表中选择单个元素。

  • 要通过枚举构造多集，请使用嵌套表类型的构造函数。

  • 要通过查询构造多集，请使用带有多集参数的CAST ，强制转换为嵌套表类型。

  • 要取消多集，请在FROM子句中使用TABLE运算符。

S272，用户定义类型的多字节集

Oracle的嵌套表类型允许多组结构化类型。Oracle没有不同的类型，因此不支持多个不同类型的集合。

S274，多参考类型

嵌套表类型可以具有一个或多个引用类型列。

S275，高级多集支持

Oracle使用与标准对多集合相同的语法，在嵌套表上支持此功能的以下元素：

  • MULTISET UNION ， MULTISET INTERSECTION和MULTISET EXCEPT运算符

  • SUBMULTISET谓词

  • =和<>谓词

Oracle不支持FUSION或INTERSECTION聚合。

S281，嵌套集合类型

Oracle允许嵌套其集合类型（varray和嵌套表）。

S401，基于数组类型的不同类型

Oracle的varray类型是强类型的。

S403， ARRAY_MAX_CARDINALITY

在PL / SQL中，varray的LIMIT方法返回其最大基数。

S404， TRIM_ARRAY

在PL / SQL中，varray的TRIM方法可用于修剪varray。

T041，基本LOB数据类型支持

Oracle支持此功能的以下方面：

T111，可更新的连接，联合和列

Oracle的可更新连接视图类似于标准的可更新连接功能。与标准不同，Oracle不需要可更新的连接视图来在SELECT列表中显示强候选键。尽管可更新的连接视图可能具有多个密钥保留表，但是与标准不同，它们中的一个可以使用UPDATE或DELETE进行修改，而标准修改了可更新连接的所有密钥保留表。

T174，标识列

  • 要重新启动标识列，请在ALTER TABLE MODIFY语句中，使用START WITH LIMIT VALUE以最高值（对于增加的标识列）或最低值（对于递减标识列）重新启动;使用START WITH号码以特定号码重新开始。

T180，系统版本表

Oracle的闪回功能与标准的系统版本表基本相同。一些关键的区别是：

  • 在Oracle中，您不需要为日记指定特定的表;所有表都记录在案。

  • 在Oracle中，由于可能存在大量数据，因此需要单独指定LOB列以进行日记。该标准没有类似的规定。

  • 在Oracle中，您需要一个权限才能读取历史数据。

  • 在标准中，日志表具有用于记录行的开始和结束时间戳的列。在Oracle中，这是通过伪列提供的。

T181，申请时间段表

Oracle支持此功能的以下元素：

  • CREATE TABLE期间的应用程序时间段定义

  • 使用带有轻微语法差异的ALTER TABLE添加和删除应用程序时间段定义：Oracle需要围绕句点规范的括号;该标准不支持此位置的括号。

Oracle扩展了此功能：

  • 能够为每个表提供多个应用程序时间段。

  • 通过使开始时间和结束时间列可选。在这种情况下，Oracle将隐式创建这些列。

  • 通过允许NULL为开始时间列指示该行在结束时间列的值之前的任何时间点被视为有效。

  • 通过允许结束时间列使用NULL来指示该行在开始时间列的值之后或之后的任何时间点被视为有效。

  • 通过使用闪回查询选项VERSIONS PERIOD FOR和AS OF PERIOD FOR查询应用程序时间段表。

T211，基本触发功能

Oracle的触发器与标准的不同之处如下：

  • 对于默认情况，Oracle不为语句触发器提供可选语法FOR EACH STATEMENT 。

  • Oracle不支持OLD TABLE和NEW TABLE ;标准中指定的转换表（受影响行的前后映像的多重集）不可用。

  • 触发器主体是用PL / SQL编写的，它在功能上等同于标准的过程语言PSM，但不一样。

  • 在触发器主体中，以冒号开头引用新旧转换变量。

  • Oracle的行触发器在处理行时执行，而不是在处理所有行之后缓冲它们并执行所有行。标准的语义是确定性的，但Oracle的正在进行的行触发器更具性能。

  • Oracle的before-row和before-statement触发器可以执行标准中禁止的DML语句。但是，Oracle的行后语句不能执行DML，而标准中允许使用DML。

  • 当应用多个触发器时，标准表示它们按照定义的顺序执行。在Oracle中，执行顺序是不确定的，除非使用FOLLOWS指定。

  • Oracle使用系统特权CREATE TRIGGER和CREATE ANY TRIGGER来规范触发器的创建，而不是标准的TRIGGER特权，这是一个表特权。
