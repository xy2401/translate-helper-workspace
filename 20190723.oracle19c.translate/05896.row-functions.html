3.16 CHR

CHR将整数转换为数据库字符集或国家字符集中具有二进制等效字符的字符。

对于单字节字符集，如果n > 256，则CHR转换mod(n, 256)的二进制等价物。

对于Unicode国家字符集和所有多字节字符集， n必须解析为一个完整的代码点。代码点未经过验证，指定无效代码点的结果是不确定的。

回报价值

VARCHAR2 | NVARCHAR2

句法

CHR （n [使用NCHAR_CS]）

参数

n是数字表达式。

USING NCHAR_CS返回国家字符集中的字符。否则，返回值在数据库字符集中。OLAP引擎使用UTF8国家字符集，因此返回值可能与SQL CHR函数不同，后者使用数据库UTF16国家字符集。

例

CHR(67) ， CHR(67 USING NCHAR_CS)和CHR(323)都在基于ASCII的系统上返回字母C ，其中包含WE8DEC数据库字符集和UTF8国家字符集。CHR(323)被评价为CHR(MOD(323, 256)) 。
