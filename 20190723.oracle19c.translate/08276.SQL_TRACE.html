1.323 SQL_TRACE

SQL_TRACE启用或禁用SQL跟踪工具。

属性 描述

参数类型

布尔

默认值

假

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

价值范围

是的|假

基本

没有

将此参数设置为true可提供有关可用于提高性能的调优信息。

注意：

使用此初始化参数为整个实例启用SQL跟踪工具可能会对性能产生严重影响。使用ALTER SESSION语句为特定会话启用工具。如果必须在整个生产环境中启用该工具，则可以通过以下方式最小化性能影响：

  • 保持至少25％的空闲CPU容量

  • 为USER_DUMP_DEST位置保留足够的磁盘空间

  • 在足够的磁盘上划分磁盘空间

注意：

不推荐使用SQL_TRACE参数。Oracle建议您改用DBMS_MONITOR和DBMS_SESSION软件包。保留SQL_TRACE仅用于向后兼容。
