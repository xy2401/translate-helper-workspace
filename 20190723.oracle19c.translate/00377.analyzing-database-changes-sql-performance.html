11.1关于SPA快速检查

SQL性能分析器快速检查（SPA快速检查）在进行更改之前验证数据库更改对SQL性能的影响。

SPA快速检查适用于某些EM Express数据库管理页面，其中数据库的更改可能会影响性能。

使用SPA快速检查来验证对SQL性能的影响：

  • 更改会话可修改的初始化参数的值

  • 实现SQL配置文件

您必须在使用之前配置SPA快速检查。

11.2.1关于SQL调整集

SQL调优集（STS）是一个数据库对象，包括一个或多个SQL语句及其执行统计信息和执行上下文。

您可以使用API创建SQL调优集，将SQL语句加载到SQL调优集中，并将SQL调优集传输到另一个系统（例如与生产系统非常相似的测试系统）。

在EM Express中使用SQL性能分析器或SQL性能分析器快速检查（SPA快速检查）之前，必须为它们指定默认设置。这些工具需要SQL调优集，其中包含要分析其性能的SQL语句。

11.2.2指定SPA快速检查的默认值

在使用之前，您必须指定SPA快速检查的默认设置。

要指定SPA快速检查的默认设置：
  1. 在EM Express的任何页面上，从“ 性能”菜单中选择“ SQL性能分析器” 。

    将出现“SQL性能分析器”页面。

  2. 在“快速检查任务”选项卡上，单击“ 快速检查默认设置

    出现SPA快速检查默认值设置对话框。

  3. 在“SPA快速检查默认值设置”对话框中，指定：

    • SQL调优集

    • SPA快速检查的默认值，用于在SQL调优集中执行语句时使用

    • SPA Quck的默认值检查在将SQL调优集中的语句的性能与数据库的当前设置进行比较并在计划更改到数据库之后使用

    在“SPA快速检查默认值设置”对话框中，指定这些字段的值：

    • SQL调优集 ：指定一个SQL调优集，其中包含您希望分析其性能的SQL语句。

    • 禁用多次执行 ：指示当SPA快速检查执行SQL调整集时，SQL调优集中的语句是否应该多次运行。选择“是”时，SQL调优集中的每个SQL仅执行一次。选择“ 否”时 ，将多次执行SQL调优集中的SQL语句，然后对运行时统计信息求平均值。

    • 要获取的行数：表示在SQL Performance Analyzer测试执行期间为每个SQL提取的结果行数。允许的值为ALL_ROWS ， AVERAGE ， AUTO或整数。

      值 描述
      ALL_ROWS 将获取所有结果行。这是默认值。
      平均 结果行数将计算为SQL调优集中每个SQL处理的总行数与总执行次数之比。
      汽车 结果行的数量将使用SQL调优集中捕获的优化器环境的optimizer_mode参数的值来确定。如果optimizer_mode值为ALL_ROWS ，则将获取所有结果行。如果其值为FIRST_ROWS_n ，则SPA快速检查将获取n个结果行。
      整数 结果行的数量将等于此指定值，如果要获取的行数较少，则结果行数量将减少。

    • Per-SQL时间限制（秒） ：表示SPA快速检查执行SQL调优集中的语句时每个语句的每语句超时（以秒为单位）。最大值为2 ^ 32-1。如果设置为空，则无限制。

    • 总时间限制（秒） ：表示SPA Quick执行SQL调优集中的所有语句时的全局超时（以秒为单位）。最大值为2 ^ 32-1。如果设置为空，则无限制。

    • 比较度量标准 ：选择希望SPA快速检查在将SQL调整集中的语句的性能与数据库的当前设置进行比较以及计划更改到数据库之后使用的度量标准。

    • 工作负载影响阈值（％） ：表示SQL语句更改对工作负载的影响的阈值 。工作量变化影响低于此阈值绝对值的陈述将被视为未更改（即，这些陈述的执行既不会改善也不会退化）。

    • SQL影响阈值（％） ：对SQL语句的更改影响的阈值。SQL更改影响低于此阈值绝对值的语句将被视为未更改（即，这些语句的性能既不会改进也不会退化）。

11.3.1验证初始化参数更改的影响

您可以使用SPA快速检查来验证更改对会话可修改的数据库初始化参数的影响。

  1. 在EM Express的任何页面上，从“ 配置”菜单中选择“ 初始化参数” 。

    将出现“初始化参数”页面。

  2. 选择要为其验证会话可修改的初始化参数，以验证设置新值的影响。“初始化参数”页面上的会话可修改参数在“ 会话”列中有一个复选标记。您可以使用“ 名称”过滤器来减少显示的初始化参数的数量。

    请注意，优化程序类别中的大多数参数都是会话可修改的。

  3. 单击“ 使用SPA验证” 。

    将出现“使用SPA验证”对话框，其中包含您在“ 名称”字段中选择的初始化参数。

  4. 在“使用SPA验证”对话框中，指定以下选项的值，然后单击“ 确定” ：
    选项 描述
    参数值 输入要在数据库工作负载上验证其影响的新初始化参数值。
    任务名称 接受默认任务名称，或输入其他名称。
    描述 此字段显示初始化参数名称，当前值以及要验证其影响的新值。接受默认说明，或输入其他说明。
    总时限 接受默认值，或指定SPA快速检查可用于验证新值的影响的不同最大时间长度（以秒为单位）。

    将显示SQL Performance Analyzer页面的Quick Check Tasks选项卡，新任务的Status列显示SPA Quick Check处理任务的进度。任务执行完成后，“ 状态”列中会出现一个复选标记。

  5. 要查看比较报告，请在“ 比较报告名称”列中单击报告的名称 。

    将显示该任务的SQL Performance Analyzer报告页面。SQL Trials部分列出了SPA快速检查为该任务执行的SQL试验，SQL试验比较部分列出了为这些试验执行的一个或多个比较。“Top SQL语句”部分显示了两个试验的顶级SQL语句的比较。

    “ 状态”列显示每个语句的性能是否得到改善，退化或未更改。

    使用“ 类别”过滤器显示所有顶级SQL语句，或仅显示其中一个可用类别中的SQL语句。

    任何推荐新执行计划的SQL语句都包含新计划列中的复选标记。

  6. 在页面底部的Top SQL Statements部分中，单击SQL语句的SQL ID以查看有关两个试验中语句执行的更多详细信息。
    “摘要”部分以及“执行详细信息”部分中的“统计信息”，“计划”和“结果”选项卡提供了有关两个试验的SQL语句执行的详细信息。

11.5关于SQL Performance Analyzer

SQL Performance Analyzer通过识别每个SQL语句的性能差异，自动化评估数据库更改对SQL工作负载的总体影响的过程。

提供了一份报告，显示由于更改而对工作负载性能的净影响。对于回归的SQL语句，SQL Performance Analyzer还提供了适当的执行计划详细信息以及调优建议。因此，您可以在最终用户受到影响之前纠正任何负面结果。此外，您可以验证（节省大量时间和成本）系统更改为生产环境将导致净改进。

在使用SQL Performance Analyzer之前，必须具有一个SQL调优集，其中包含要分析其性能的SQL语句，并且必须指定SQL Performance Analyzer的默认设置。

SQL性能分析器任务是一个容器，它封装有关完整SQL性能分析器分析的所有数据。SQL性能分析器分析包括至少两个SQL试验和比较。SQL试用版封装了特定环境条件下SQL调优集的执行性能。创建SQL性能分析器任务时，选择SQL调优集作为其输入源，并在运行SQL试验时，SQL调优集用作SQL语句的源。

创建SQL Performance Analyzer任务后，您将创建一个更改前的SQL试用版。然后，使数据库更改其要评估的SQL性能影响。更改数据库后，您将创建一个更改后的SQL试用版。最后，你比较两个试验。

SQL Performance Analyzer分析显示了两次试验之间环境差异的影响。

    将出现“SPA默认值设置”对话框。

  3. 在“SPA默认值设置”对话框中，指定SQL Performance Analyzer在执行SQL调优集中的语句时使用的默认值，以及SQL Performance Analyzer在比较SQL调优中的语句性能时使用的默认值使用数据库的当前设置进行设置，并在对数据库进行更改后进行设置。指定这些字段的值，然后单击“ 确定”以保存SQL Performance Analyzer的这些默认设置：

    • 禁用多个执行 ：指示当SQL Performance Analyzer执行SQL调优集时，SQL调优集中的语句是否应该多次运行。选择“是”时，SQL调优集中的每个SQL仅执行一次。选择“ 否”时 ，将多次执行SQL调优集中的SQL语句，然后对运行时统计信息求平均值。

    • Per-SQL时间限制（秒） ：表示SQL Performance Analyzer在SQL调优集中执行语句时每个语句的每语句超时（以秒为单位）。最大值为2 ^ 32-1。如果设置为空，则无限制。

    • 要获取的行数：表示在SQL Performance Analyzer测试执行期间为每个SQL提取的结果行数。允许的值为ALL_ROWS ， AVERAGE ， AUTO或整数。

      值 描述
      ALL_ROWS 将获取所有结果行。这是默认值。
      平均 结果行数将计算为SQL调优集中每个SQL处理的总行数与总执行次数之比。
      汽车 结果行的数量将使用SQL调优集中捕获的优化器环境的optimizer_mode参数的值来确定。如果optimizer_mode值为ALL_ROWS ，则将获取所有结果行。如果其值为FIRST_ROWS_n ，则SPA快速检查将获取n个结果行。
      整数 结果行的数量将等于此指定值，如果要获取的行数较少，则结果行数量将减少。

    • 执行完整DML ：指定是以完全执行DML语句，包括获取行锁和修改行。指定No （默认）以仅执行DML的查询部分而不修改数据。如果指定Yes ，则SQL Performance Analyzer会在DML执行后发出回滚，以防止DML进行持久更改。

    • 应用捕获的优化程序环境 ：如果希望将SQL调优集中捕获的per-SQL优化程序环境用于SQL性能分析器试用，请为此字段指定“ 是 ”。否则，请指定否 。

    • 比较度量标准 ：选择希望SQL Performance Analyzer在将SQL调优集中的语句性能与数据库的当前设置进行比较以及计划更改数据库之后使用的度量标准。

    • 工作负载影响阈值（％） ：表示SQL语句更改对工作负载的影响的阈值 。工作量变化影响低于此阈值绝对值的陈述将被视为未更改（即，这些陈述的执行既不会改善也不会退化）。

    • SQL影响阈值（％） ：对SQL语句的更改影响的阈值。SQL更改影响低于此阈值绝对值的语句将被视为未更改（即，这些语句的性能既不会改进也不会退化）。

11.6.4比较两个SQL试验

在创建了预更改SQL试用版和更改后的SQL试用版之后，您可以使用SQL Performance Analyzer来比较两个SQL试验，以评估数据库更改对SQL性能的影响。

请按照以下步骤比较两个SQL试验：
  1. 在任何EM Express页面上，从“ 性能”菜单中选择“ SQL性能分析器 ”。

    将出现“SQL性能分析器”页面。

  2. 在“手动任务”选项卡上，单击SQL Performance Analyzer任务的名称，该任务包括要分析数据库更改影响的SQL语句。

    将显示该任务的SQL Performance Analyzer任务页面。

  3. 要比较两个SQL试验，请单击任务的SQL Performance Analyzer任务页面的SQL试用比较部分中的创建 。
  4. 在“创建SQL比较报告”对话框的“比较信息”选项卡上：

    • 比较报告名称 ：接受默认比较报告名称，或输入要用于报告的名称。

    • 描述 :(可选）输入比较说明。

    • 试验1名称 ：选择要在比较中使用的第一个试验。虽然您可以选择任何试用版，但通常您会选择更改前的SQL试用版作为第一次试用版。

    • 试验2名称 ：选择要在比较中使用的第二个试验。虽然您可以选择任何试用版，但通常您会选择更改后的SQL试用版作为第二次试用版。

    • 比较度量标准 ：选择用于比较的度量标准。

    如果要安排比较运行的时间，请单击右箭头按钮转到“计划”选项卡。或者，单击“ 确定 ”立即运行比较。

    执行比较时，其状态显示在任务的SQL Performance Analyzer任务页面的SQL Trial Comparisons部分的Status列中。

  5. 要查看新的比较报告，请在“SQL试用比较”部分的“ 比较报告名称”列中单击报告的名称 。
该任务的SQL Performance Analyzer Report页面显示了两个试验的比较报告。

“Top SQL语句”部分显示了两个试验的顶级SQL语句的比较。

“ 状态”列显示每个语句的性能是否得到改善，退化或未更改。

使用“ 类别”过滤器显示所有顶级SQL语句，或仅显示其中一个可用类别中的SQL语句。
