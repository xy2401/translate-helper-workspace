JSON对象访问表达式

只有在查询JavaScript Object Notation（JSON）数据列时才使用JSON对象访问表达式。它生成一个字符串，其中包含在该数据中找到的一个或多个JSON值。此类表达式的语法称为点符号语法。

  • 对于table_alias ，请为包含JSON数据列的表指定别名。此表别名是必需的，必须分配给SQL语句中其他位置的表。

  • 对于JSON_column ，请指定JSON数据列的名称。该列必须是数据类型VARCHAR2 ， CLOB或BLOB并且必须在列上定义IS JSON检查约束。

  • 您可以选择指定一个或多个JSON对象键。对象键允许您在JSON数据中定位特定的JSON值。第一个JSON_object_key必须与JSON数据顶级对象成员的键（属性）名称区分大小写。如果该对象成员的值是另一个JSON对象，则可以指定与该对象的成员的键名匹配的第二个JSON_object_key ，依此类推。如果在任何这些迭代期间遇到JSON数组，并且您没有指定array_step ，则会隐式解包该数组，并使用JSON_object_key计算数组的元素。

  • 如果JSON值是数组，则可以选择指定一个或多个array_step子句。这允许您访问JSON数组的特定元素。

    • 使用integer指定JSON数组中索引integer的元素。使用integer TO integer指定两个索引integer数值（包括两个）之间的元素范围。如果指定的元素存在于要计算的JSON数组中，则数组步骤将导致与这些元素匹配。否则，数组步骤不会导致匹配。JSON数组中的第一个元素的索引为0。

    • 使用星号通配符（ * ）指定JSON数组中的所有元素。如果要计算的JSON数组包含至少一个元素，则数组步骤将导致与JSON数组中的所有元素匹配。否则，数组步骤不会导致匹配。

JSON对象访问表达式生成数据类型为VARCHAR2(4000)字符串，其中包含目标JSON值，如下所示：

  • 对于单个目标值，字符串包含该值，无论它是JSON标量值，对象还是数组。

  • 对于多个目标值，字符串包含一个JSON数组，其元素是这些值。

如果省略JSON_object_key ，则表达式将生成一个完整包含JSON数据的字符串。在这种情况下，字符串与要查询的JSON数据列具有相同的数据类型。

JSON对象访问表达式不能返回大于4K字节的值。如果该值超过此限制，则表达式返回null。为了获得实际的价值，而不是使用JSON_QUERY功能或JSON_VALUE功能，并指定与适当的返回类型RETURNING子句。

JSON对象访问表达式的排序规则派生规则与JSON_QUERY函数的排序规则派生规则相同。
