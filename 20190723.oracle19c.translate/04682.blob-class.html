13.7 Blob类

Blob类定义BLOB类型的对象的公共属性。 BLOB是一个大型二进制对象，存储为数据库表的一行中的列值。Blob对象包含指向BLOB的逻辑指针，而不是BLOB本身。

Blob类的方法使您可以执行与Blob对象相关的特定任务。

ResultSet和Statement类的方法（如getBlob()和setBlob()使您可以访问SQL BLOB值。

NULL Blob对象上唯一有效的方法是setName（） ， isNull（）和operator =（） 。

未初始化的Blob对象可以通过以下方式初始化：

  • setEmpty（）方法。该BLOB然后可以通过将本修改BLOB到表中，然后使用它检索SELECT...FOR UPDATE 。 write（）方法修改BLOB ;但是，仅在提交事务时才将刷新的数据刷新到表中。请注意，不需要更新。

  • 为其分配初始化的Blob对象。

也可以看看：

  • Oracle Database SecureFiles和Large Objects Developer's Guide中对LOB的深入讨论

13.7.9 getStream（）

从BLOB返回Stream对象。如果流是打开的，则不允许在Blob object上打开另一个流，因此用户必须始终在执行任何Blob对象操作之前关闭该流。

句法

参数 描述 
抵消

从BLOB开始读取数据的起始位置。如果未指定offset ，则从BLOB的开头写入数据。有效值是大于或等于1 。 

量

从BLOB读取的总字节数;如果amount为0 ，则以流传输模式从输入offset读取数据，直到BLOB结束。

13.7.16 operator ==（）

比较两个Blob objects是否相等。如果两个Blob objects都引用相同的BLOB则它们是相等的。两个NULL Blob objects不被视为相等。如果Blob对象相等，则返回TRUE ;否则，返回FALSE 。

句法

bool operator ==（const Blob＆srcBlob）const;
参数 描述 
srcBlob

源BLOB要与当前进行比较BLOB 。

13.7.17运营商！=（）

比较两个Blob objects的不等式。如果两个Blob objects都引用相同的BLOB则它们是相等的。两个NULL Blob对象不被视为相等。如果Blob对象不相等，则返回TRUE ;否则，返回FALSE 。

句法

布尔运营商！=（const Blob＆srcBlob）const;
参数 描述 
srcBlob

源BLOB要与当前进行比较BLOB 。

unsigned int write（unsigned int amt，unsigned char * buffer，unsigned int bufsize，unsigned int offset = 1）;
参数 描述 
AMT

要写入BLOB的字节数。

 
缓冲

包含要写入BLOB的数据的缓冲区。

 
BUFFSIZE

buffer containing the data to be written to the BLOB buffer containing the data to be written to the的buffer containing the data to be written to the的大小.有效值是大于或等于amt 。 

抵消

开始将数据写入BLOB的起始位置。如果未指定offset ，则从BLOB的开头写入数据。有效值是大于或等于1的数字。
