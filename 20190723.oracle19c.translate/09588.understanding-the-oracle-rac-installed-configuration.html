6了解Oracle RAC安装配置

了解Oracle Real Application Clusters（Oracle RAC）安装配置的组和环境设置。

  • 了解Oracle RAC中的已配置环境
    Oracle Net Configuration Assistant（NETCA）和数据库配置助手（DBCA）配置您的环境以满足数据库创建和Oracle RAC数据库的Oracle Enterprise Manager发现的要求。
  • 了解操作系统权限组
    查看此信息以获取Oracle数据库或Oracle自动存储管理（Oracle ASM）管理所需的系统特权。
  • 了解群集节点上的时区设置
    Oracle RAC要求所有群集节点都具有相同的时区设置。
  • 了解Oracle RAC的服务器参数文件
    创建数据库时，Oracle数据库会在您指定的文件位置中创建SPFILE。此位置可以是Oracle ASM磁盘组或群集文件系统。
  • 关于Oracle RAC的ORATAB配置
    Oracle在oratab配置文件中为每个Oracle RAC数据库创建一个条目。
  • 使用Database Configuration Assistant创建的数据库组件
    了解DBCA创建的数据库组件。
  • 管理Oracle RAC中的撤消表空间
    Oracle数据库在撤消表空间中存储回滚或撤消信息。要管理撤消表空间，Oracle建议您使用自动撤消管理。
  • 关于初始化参数文件
    Oracle建议使用服务器参数文件（SPFILE）来存储Oracle数据库初始化参数。
  • Oracle RAC数据库的Oracle Net Services配置
    用户可以使用客户端和服务器配置访问Oracle RAC数据库，也可以通过一个或多个中间层访问Oracle RAC数据库，无论是否有连接池。
  • Oracle Net Services和Oracle RAC的性能特征
    Oracle RAC数据库提供了连接负载平衡和故障转移的重要优势。
  • Oracle Net Services配置文件和参数
    如果使用Easy Connect之外的命名方法，则可能需要额外配置Oracle Net Services。
