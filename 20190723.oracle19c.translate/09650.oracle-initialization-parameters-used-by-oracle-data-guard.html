16初始化参数

这是您为Data Guard环境设置的初始化参数的字母顺序列表。它指示参数是为主角色还是备用角色设置的，还提供了设置每个参数的提示。

Oracle Database Reference提供了完整的初始化参数信息，包括如何通过发出ALTER SYSTEM SET语句（例如， ALTER SYSTEM SET LOG_ARCHIVE_TRACE ）或编辑初始化参数文件来更新初始化参数。有关设置初始化参数的详细信息，请参阅Oracle操作系统特定的文档。

表16-1 Oracle Data Guard配置中实例的初始化参数

COMPATIBLE = release_number

如果您希望进行切换，请在主数据库和备用数据库上指定相同的值。如果值不同，则重做传输服务可能无法将重做数据从主数据库传输到备用数据库。有关示例，请参阅为备用数据库创建参数文件 。

如果不期望切换，则逻辑备用数据库可以具有比主数据库更高的COMPATIBLE设置。

对于使用SQL Apply进行滚动升级，请根据通过创建新的逻辑备用数据库执行滚动升级中所述的准则来设置此参数。
