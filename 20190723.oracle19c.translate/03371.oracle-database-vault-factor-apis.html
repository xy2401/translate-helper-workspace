17 Oracle Database Vault Factor API

DBMS_MACADM PL / SQL包具有与因子相关的Oracle Database Vault规则过程和函数， DVF具有管理因子的功能。

  • DBMS_MACADM因子过程和函数
    DBMS_MACADM PL / SQL包提供了配置因子的过程和函数。
  • Oracle Database Vault运行时PL / SQL过程和函数
    Oracle Database Vault提供了用于管理Database Vault安全选项和管理Database Vault安全强制实施的过程接口。
  • Oracle Database Vault DVF PL / SQL因子函数
    使用DBMS_MACADM PL / SQL程序包管理各种因素时，Oracle Database Vault维护DVF架构功能。

DBMS_MACADM因子过程和函数

DBMS_MACADM PL / SQL包提供了配置因子的过程和函数。

只有已被授予DV_OWNER或DV_ADMIN角色的用户才能使用这些过程和功能。

  • ADD_FACTOR_LINK程序
    ADD_FACTOR_LINK过程指定两个因素的父子关系。
  • ADD_POLICY_FACTOR过程
    ADD_POLICY_FACTOR过程指定因子的标签有助于策略的Oracle Label Security标签。
  • CHANGE_IDENTITY_FACTOR过程
    CHANGE_IDENTITY_FACTOR过程将身份与不同因子相关联。
  • CHANGE_IDENTITY_VALUE过程
    CHANGE_IDENTITY_FACTOR过程更新标识的值。
  • CREATE_DOMAIN_IDENTITY过程
    CREATE_DOMAIN_IDENTITY过程用于Oracle Real Application Clusters（Oracle RAC）和Oracle Label Security。
  • CREATE_FACTOR程序
    CREATE_FACTOR过程创建一个因子。
  • CREATE_FACTOR_TYPE过程
    CREATE_FACTOR_TYPE过程创建用户定义的因子类型。
  • CREATE_IDENTITY_MAP过程
    CREATE_IDENTITY_MAP过程定义了可以从链接子因子（子因子）的值派生因子身份的测试。
  • CREATE_IDENTITY过程
    CREATE_IDENTITY过程为给定因子分配标识和关联的信任级别。
  • DELETE_FACTOR程序
    DELETE_FACTOR过程删除一个因子。
  • DELETE_FACTOR_LINK程序
    DELETE_FACTOR_LINK过程会删除两个因素的父子关系。
  • DELETE_IDENTITY过程
    DELETE_IDENTITY过程从现有因子中删除标识。
  • DELETE_FACTOR_TYPE过程
    DELETE_FACTOR_TYPE过程删除因子类型。
  • DELETE_IDENTITY_MAP过程
    DELETE_IDENTITY_MAP过程删除因子的标识映射。
  • DROP_DOMAIN_IDENTITY过程
    DROP_DOMAIN_IDENTITY过程从域中删除Oracle Real Application Clusters数据库节点。
  • GET_SESSION_INFO函数
    GET_SESSION_INFO函数从SYS.V_$SESSION系统表返回当前会话的信息。
  • GET_INSTANCE_INFO函数
    GET_INSTANCE_INFO函数从SYS.V_$INSTANCE系统表返回有关当前数据库实例的信息。
  • RENAME_FACTOR程序
    RENAME_FACTOR过程重命名一个因子;名称更改将在使用因子的任何位置生效。
  • RENAME_FACTOR_TYPE程序
    RENAME_FACTOR过程重命名因子类型;名称更改将在使用因子类型的任何位置生效。
  • UPDATE_FACTOR程序
    UPDATE_FACTOR过程更新因子类型的描述。
  • UPDATE_FACTOR_TYPE过程
    UPDATE_FACTOR_TYPE过程更新因子类型。
  • UPDATE_IDENTITY过程
    UPDATE_IDENTITY过程更新因子标识的信任级别。

ADD_FACTOR_LINK程序

ADD_FACTOR_LINK过程指定两个因素的父子关系。

句法

DBMS_MACADM.ADD_FACTOR_LINK（parent_factor_name IN VARCHAR2，child_factor_name IN VARCHAR2，label_indicator IN VARCHAR2）;

参数

表17-1 ADD_FACTOR_LINK参数

参数 描述

parent_factor_name

父因子名称。

要查找当前数据库实例中的现有父子因素，请查询DBA_DV_FACTOR_LINK视图中描述的DBA_DV_FACTOR_LINK视图 。

child_factor_name

子因素名称。

label_indicator

表示链接到父因子的子因子对Oracle Label Security集成中的父因子的标签有贡献。指定DBMS_MACUTL.G_YES （表示是）或DBMS_MACUTL.G_NO （表示否）。

要查找与因子关联的Oracle Label Security策略和标签，请查询以下视图，如Oracle Database Vault数据字典视图中所述 ：

  • DBA_DV_MAC_POLICY ：列出当前数据库实例中定义的Oracle Label Security策略。

  • DBA_DV_MAC_POLICY_FACTOR ：列出与当前数据库实例的Oracle Label Security策略关联的因子。

  • DBA_DV_POLICY_LABEL ：列出每个策略的DBA_DV_IDENTITY视图中每个因子标识符的Oracle Label Security标签。

例

BEGIN DBMS_MACADM.ADD_FACTOR_LINK（parent_factor_name =>'HQ_ClientID'，child_factor_name =>'Div1_ClientID'，label_indicator => DBMS_MACUTL.G_YES）;结束; /

CREATE_DOMAIN_IDENTITY过程

CREATE_DOMAIN_IDENTITY过程用于Oracle Real Application Clusters（Oracle RAC）和Oracle Label Security。

它将Oracle RAC数据库节点添加到域因子标识中，并根据Oracle Label Security策略对其进行标记

句法

DBMS_MACADM.CREATE_DOMAIN_IDENTITY（domain_name IN VARCHAR2，domain_host IN VARCHAR2，policy_name IN VARCHAR2 DEFAULT NULL，domain_label IN VARCHAR2 DEFAULT NULL）;

参数

表17-5 CREATE_DOMAIN_IDENTITY参数

参数 描述

DOMAIN_NAME

要添加主机的域的名称。

要在分布式数据库系统中的网络结构中查找数据库的逻辑位置，请运行Oracle Database Vault DVF PL / SQL因子函数中描述的DVF.F$DATABASE_DOMAIN函数。

domain_host

Oracle Real Application Clusters主机名已添加到域中。

要查找数据库的主机名，请运行Oracle Database Vault DVF PL / SQL因子函数中描述的DVF.F$DATABASE_HOSTNAME函数。

POLICY_NAME

Oracle Label Security策略名称。如果省略策略名称，则域不与任何策略关联。

要查找可用的策略，请查询DBA_DV_MAC_POLICY视图中描述的DBA_DV_MAC_POLICY视图 。

DOMAIN_LABEL

要添加Oracle Label Security策略的域的名称。

例子

BEGIN DBMS_MACADM.CREATE_DOMAIN_IDENTITY（domain_name =>'example'，domain_host =>'mydom_host'，policy_name =>'AccessData'，domain_label =>'sensitive'）;结束; /

CREATE_FACTOR程序

CREATE_FACTOR过程创建一个因子。

创建因子后，可以使用CREATE_IDENTITY过程中描述的CREATE_IDENTITY过程为其赋予身份。

句法

DBMS_MACADM.CREATE_FACTOR（factor_name IN VARCHAR2，factor_type_name IN VARCHAR2，rule_set_name IN VARCHAR2，get_expr IN VARCHAR2，validate_expr IN VARCHAR2，identify_by IN NUMBER，labeled_by IN NUMBER，eval_options IN NUMBER，audit_options IN NUMBER，fail_options IN NUMBER）;

参数

表17-6 CREATE_FACTOR参数

参数 描述

factor_name

因子名称，混合大小写最多128个字符，不含空格。

要查找当前数据库实例中的现有因子，请查询DBA_DV_FACTOR视图中描述的DBA_DV_FACTOR视图 。

factor_type_name

因子类型，混合大小写最多128个字符，不含空格。

要查找现有因子类型，请查询DBA_DV_FACTOR_TYPE视图中描述的DBA_DV_FACTOR_TYPE视图 。

描述

该因子的用途说明，混合大小最多1024个字符。

rule_set_name

规则集名称，如果要使用规则集来控制设置因子标识的时间和方式。

要查找现有规则集，请查询Oracle Database Vault数据字典视图中描述的DBA_DV_RULE_SET视图。有关将规则集分配给因子的详细信息，另请参阅将规则集分配给因子。

get_expr

有效的PL / SQL表达式，用于检索因子的标识。在混合大小写中最多可以使用255个字符。有关更多信息，请参阅设置因子的检索方法 。另请参见audit_options参数。

validate_expr

验证因子的过程的名称。这是一个有效的PL / SQL表达式，它返回一个布尔值（ TRUE或FALSE ）以验证因子的标识。有关更多信息，请参阅设置因子的验证方法 。

identify_by

根据为get_expr参数设置的表达式确定因子身份的选项：

  • DBMS_MACUTL.G_IDENTIFY_BY_CONSTANT ：按常量

  • DBMS_MACUTL.G_IDENTIFY_BY_METHOD ：按方法

  • DBMS_MACUTL.G_IDENTIFY_BY_FACTOR ：按因子

  • DBMS_MACUTL.G_IDENTIFY_BY_CONTEXT ：按上下文

有关更多信息，请参阅设置因子标识信息。

labeled_by

标记因子的选项：

  • DBMS_MACUTL.G_LABELED_BY_SELF ：直接从与Oracle Label Security策略关联的标签中标记因子的标识（默认）

  • DBMS_MACUTL.G_LABELED_BY_FACTORS ：从其子因子标识的标签中DBMS_MACUTL.G_LABELED_BY_FACTORS因子标识标签。

有关更多信息，请参阅为因子设置Oracle Label Security标签信息。

eval_options

用户登录时评估因子的选项：

  • DBMS_MACUTL.G_EVAL_ON_SESSION ：创建数据库会话时（默认）

  • DBMS_MACUTL.G_EVAL_ON_ACCESS ：每次访问因子

  • DBMS_MACUTL.G_EVAL_ON_STARTUP ：启动时

有关详细信息，请参阅设置因子的评估信息。

audit_options

如果要生成自定义Oracle Database Vault审核记录，请选择审核因子。

  • DBMS_MACUTL.G_AUDIT_OFF ：禁用审核。

  • DBMS_MACUTL.G_AUDIT_ALWAYS ：始终审核。

  • DBMS_MACUTL.G_AUDIT_ON_GET_ERROR ：审核get_expr返回错误。

  • DBMS_MACUTL.G_AUDIT_ON_GET_NULL ：审核get_expr是否为空。

  • DBMS_MACUTL.G_AUDIT_ON_VALIDATE_ERROR ：审核验证过程是否返回错误。

  • DBMS_MACUTL.G_AUDIT_ON_VALIDATE_FALSE ：审核验证过程是否为false。

  • DBMS_MACUTL.G_AUDIT_ON_TRUST_LEVEL_NULL ：审核是否没有设置信任级别。

  • DBMS_MACUTL.G_AUDIT_ON_TRUST_LEVEL_NEG ：审核信任级别是否为负数。

有关更多信息，请参阅设置因子的审计选项 。

fail_options

报告因子错误的选项：

  • DBMS_MACUTL.G_FAIL_WITH_MESSAGE ：显示错误消息（默认）

  • DBMS_MACUTL.G_FAIL_SILENTLY ：不显示错误消息

有关更多信息，请参阅为因子设置错误选项 。

例

BEGIN DBMS_MACADM.CREATE_FACTOR（factor_name =>'Sector2_DB'，factor_type_name =>'Instance'，description =>''，rule_set_name =>'Limit_DBA_Access'，get_expr =>'UPPER（SYS_CONTEXT（''USERENV''，''DB_NAME' '））'，validate_expr =>'dbavowner.check_db_access'，identify_by => DBMS_MACUTL.G_IDENTIFY_BY_METHOD，labeled_by => DBMS_MACUTL.G_LABELED_BY_SELF ，eval_options => DBMS_MACUTL.G_EVAL_ON_SESSION，audit_options => DBMS_MACUTL.G_AUDIT_OFF，fail_options => DBMS_MACUTL.G_FAIL_SILENTLY）;结束; /

CREATE_IDENTITY过程

CREATE_IDENTITY过程为给定因子分配标识和关联的信任级别。

创建因子后，必须为其分配标识。

句法

DBMS_MACADM.CREATE_IDENTITY（factor_name IN VARCHAR2，值IN VARCHAR2，trust_level IN NUMBER）;

参数

表17-9 CREATE_IDENTITY参数

参数 描述

factor_name

因子名称。

要查找现有因子，请查询DBA_DV_FACTOR视图中描述的DBA_DV_FACTOR视图 。

值

该因子的实际值，在大小写混合中最多1024个字符。例如，IP_Address因子的标识可以是IP地址192.0.2.12。

trust_level

表示相同因子相对于其他身份的信任程度的数字。通常，设置的信任级别数越高，信任度越高。信任级别为10表示“非常信任”。负信任级别不受信任。

有关信任级别和标签安全性的更多信息，请参阅创建和配置因子标识 。

例

BEGIN DBMS_MACADM.CREATE_IDENTITY（factor_name =>'Sector2_ClientID'，value =>'intranet'，trust_level => 5）;结束; /
