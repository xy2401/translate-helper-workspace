Test Kinds

This extension point is internal

Identifier:
org.eclipse.jdt.junit.internal_testKinds

Since:
3.2

Description:
Internal extension point to register JUnit test kinds. This extension point is not intended to be used by clients. It may be changed or removed at any time.

Configuration Markup:

<!ELEMENT extension (kind)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT kind (runtimeClasspathEntry+)>

<!ATTLIST kind

id               CDATA #REQUIRED

displayName      CDATA #REQUIRED

finderClass      CDATA #REQUIRED

loaderPluginId   CDATA #REQUIRED

loaderClass      CDATA #REQUIRED

precedesTestKind CDATA #IMPLIED>

  • id - a unique test kind ID
  • displayName - name of this test kind
  • finderClass - test finder class implementing org.eclipse.jdt.internal.junit.launcher.ITestFinder
  • loaderPluginId - test loader plug-in ID
  • loaderClass - test loader class implementing org.eclipse.jdt.internal.junit.runner.ITestLoader (declared in plug-in "loaderPluginId")
  • precedesTestKind - comma-separated list of kind ids. This test kind should be shown after all the other kinds it preceds.

<!ELEMENT runtimeClasspathEntry EMPTY>

<!ATTLIST runtimeClasspathEntry

pluginId  CDATA #REQUIRED

pathToJar CDATA #IMPLIED>

  • pluginId -
  • pathToJar -
