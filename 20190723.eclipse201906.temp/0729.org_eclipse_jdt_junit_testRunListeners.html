Test Run Listeners

Identifier:
org.eclipse.jdt.junit.testRunListeners

Since:
2.1

Description:
Extension point to register additional test run listeners. A test run listener is notified about the execution of a test run.

Configuration Markup:

<!ELEMENT extension (testRunListener)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

a fully qualified identifier of the target extension point


  • id - a fully qualified identifier of the target extension point

<!ELEMENT testRunListener EMPTY>

<!ATTLIST testRunListener

class CDATA #REQUIRED>

  • class - A test run listener that extends org.eclipse.jdt.junit.TestRunListener (since 3.6). Note: Also accepts org.eclipse.jdt.junit.ITestRunListener, but that interface is deprecated.

Examples:
The following is an example of a test run listener contribution: 


 <extension point="org.eclipse.jdt.junit.testRunListeners">
  <testRunListener class="com.example.SampleTestRunListener"/>
 </extension>

API Information:
Test run listeners should extend org.eclipse.jdt.junit.TestRunListener (since 3.6). In previous versions, they could also implement the org.eclipse.jdt.junit.ITestRunListener interface, but that one is deprecated now.
