IncludePath Attribute Configuration

Identifier:
org.eclipse.wst.jsdt.ui.classpathAttributeConfiguration

Since:
3.3

Description:
This extension point allows to specify how contributed includepath attributes are presented and configured.

Configuration Markup:

<!ELEMENT extension (classpathAttributeConfiguration+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

<!ELEMENT classpathAttributeConfiguration EMPTY>

<!ATTLIST classpathAttributeConfiguration

attributeName CDATA #REQUIRED

class         CDATA #REQUIRED>

  • attributeName - identifies the name of the includepath attribute as specified in IClasspathAttribute.getName()
  • class - the name of the class that implements this attribute configuration. The class must be public and extend org.eclipse.wst.jsdt.ui.wizards.ClasspathAttributeConfiguration with a public 0-argument constructor.

Examples:
The following is an example of a includepath entry container page: 


 <extension point="org.eclipse.wst.jsdt.ui.classpathAttributeConfiguration">
  <classpathAttributeConfiguration
   key="com.example.myplugin.myClasspathAttributeKey"
   name="Test property"
   class="com.example.MyAttributeConfiguration">
  </classpathAttributeConfiguration>
 </extension>
