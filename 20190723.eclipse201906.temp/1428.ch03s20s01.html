3.20.1. Object mapping

Object mapping is the joining of the component names you have specified in your tests to the real components in the AUT. Each component name you have specified in your Test Cases and used in a Test Suite will be shown in the Object Mapping Editor to be mapped.

New component names you have created in the Object Mapping Editor but have not yet used in Test Cases will also be shown in the Object Mapping Editor.

To be able to carry out object mapping, you must have:

  • started and connected to the AUT Agent (Section 3.2, “Starting and connecting to the AUT Agent”)

  • created a Test Case containing actions with component names (Section 3.12.2, “Creating Test Cases”)

  • added Test Cases to a Test Suite (Section 3.16, “Working with Test Suites”)

  • specified an AUT (Section 3.7, “Defining applications under test (AUTs)”)

  • configured an AUT if you want to start the AUT from the ITE (Section 3.8.1, “Configuring AUTs to be started from the ITE”)

  • chosen an AUT for the Test Suite (Section 3.16.2, “Configuring Test Suites in the Properties View”)

  • connected to the AUT Agent (Section 3.2.3, “Connecting to the AUT Agent”)

  • started the AUT to be mapped (Section 3.21.2, “Starting the AUT”)

  • opened the Object Mapping Editor (Section 3.20.2.1, “Opening the Object Mapping Editor”)
