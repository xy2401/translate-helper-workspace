3.6.10. Exporting Projects

You can choose to export the currently opened Project (Section 3.6.10.1, “Exporting the currently opened Project”) or all of the Projects in the database (Section 3.6.10.2, “Exporting all of the Projects from the database”). Projects are exported in .jub format so that they can be added to e.g. version control systems. However note that the interchange format is binary.

Projects are exported along with all their test result summaries.

3.6.10.1. Exporting the currently opened Project

  1. To export a single Project from the database, the Project you want to export must be open.

  2. Select:

    Test-->Export.

  3. Choose the location to save your Project, and the name you want to save it under.

  4. Click ”OK”.

  5. The Project (including the test result summaries (Section 3.22.3, “Working with the Test Result Summary View”)) will first be refreshed and then exported. It is saved as an .jub file.

    When you export a single Project, only the currently opened version is exported.

3.6.10.2. Exporting all of the Projects from the database

  1. Select:

    Test-->Export all.

  2. Choose or create a folder where you want the Projects to be stored.

  3. Click ”OK”.

  4. The Projects (including the test result summaries (Section 3.22.3, “Working with the Test Result Summary View”)) will be saved as .jub files.

    When you export all Projects, all versions of each Project are exported.
