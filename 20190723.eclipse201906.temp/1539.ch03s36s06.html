3.36.6. I can’t execute my Test Suite

If you cannot start a Test Suite, check that:

  • The Test Suite has complete data.

    No parameter values can be empty in a Test Suite. If you no longer need the parameter, delete it.

  • The object mapping is complete (Section 3.20.1, “Object mapping”).

  • The Object Mapping Mode and the observation mode have been switched off.

  • You have connected to the AUT Agent (Section 3.2.3, “Connecting to the AUT Agent”).

  • You have started the AUT (Section 3.21.2, “Starting the AUT”).

  • It may happen that your firewall prevents the AUT from connecting to the Jubula client or the testexec command-line tool. If this happens, you can setup a specified IP address and port for the client to listen on. You can do this by setting the JVM properties jubClientAddr and jubClientPort, through the jubula.ini and testexec.vmoptions files.

You can use the Test Suite Browser to click through the Test Suite to see the exact places where data and object maps are missing.
