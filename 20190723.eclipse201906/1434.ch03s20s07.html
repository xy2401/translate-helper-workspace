3.20.7. Component specific Profile

You can set specific profiles for the object recognition for individual components. This might be useful for components which can't be found with the standard profile.

To give you an example: A button might be used in different dialogs and even though you've given the button a name it can not be found reliably. The reason for this is that the other parameters (hierarchy, context), which are used in the heuristic, are different for each dialog. Therefore it would be a good idea to only take the name into account for the lookup. Thus you should use the profile Given Names. You can change this for all components as described here Section 3.20.2.4, “The configuration view in the Object Mapping Editor” . But this might not be a good idea when not every component in your AUT has a unique name. Therefore you should change the profile only for this component.

For more information about the object recognition read: Section 3.20.2.4, “The configuration view in the Object Mapping Editor”

To change the profile for a component open the context menu on a technical name and select a profile under Set Profile...
