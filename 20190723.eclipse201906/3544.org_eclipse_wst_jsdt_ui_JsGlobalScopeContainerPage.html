Includepath Container Entry Page

Identifier:
org.eclipse.wst.jsdt.ui.JsGlobalScopeContainerPage

Description:
This extension point allows to add a wizard page to create or edit a includepath container entry.

Configuration Markup:

<!ELEMENT extension (JsGlobalScopeContainerPage+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>

  • point - a fully qualified identifier of the target extension point
  • id - an optional identifier of the extension instance
  • name - an optional name of the extension instance

<!ELEMENT JsGlobalScopeContainerPage EMPTY>

<!ATTLIST JsGlobalScopeContainerPage

id    CDATA #REQUIRED

name  CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - identifies the includepath containers for which this page will be used. The id corresponds to the first segment of the path of a container includepath entry and matches the id referenced in org.eclipse.wst.jsdt.core.JsGlobalScopeContainerInitializer
  • name - the name of the includepath container used when selecting a new container. This attribute should be a translated string.
  • class - the name of the class that implements this container page. The class must be public and implement org.eclipse.wst.jsdt.ui.wizards.IJsGlobalScopeContainerPage with a public 0-argument constructor.

Examples:
The following is an example of a classpath entry container page: 


 <extension point="org.eclipse.wst.jsdt.ui.JsGlobalScopeContainerPage">
  <JsGlobalScopeContainerPage
   id="com.example.myplugin.myContainerId"
   name="JRE System Libraries"
   class="com.example.NewJDKEntryPage">
  </JsGlobalScopeContainerPage>
 </extension>
