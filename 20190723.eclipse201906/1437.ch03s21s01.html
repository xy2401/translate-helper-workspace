3.21.1. Prerequisites for test execution

To be able to start a test you must:

  1. Create a Test Suite (Section 3.16.1, “Creating a Test Suite”).

  2. Add Test Cases to be executed to the Test Suite (Section 3.10.4, “Adding items to editors”)

  3. Define an AUT (Section 3.7, “Defining applications under test (AUTs)”).

  4. Configure an AUT (Section 3.8.1, “Configuring AUTs to be started from the ITE”) if you want to start the AUT from the ITE.

  5. Choose an AUT for the Test Suite (Section 3.16.2, “Configuring Test Suites in the Properties View”).

  6. Connect to the AUT Agent (Section 3.2.3, “Connecting to the AUT Agent”).

  7. Start the AUT (Section 3.21.2, “Starting the AUT”) (either from the ITE or using the autrun command).

  8. Complete object mapping (Section 3.20.1, “Object mapping”).

  9. Make sure that your test data is complete in your Test Cases.

It is a good idea to set your preferences for test results in the preferences dialog (Figure 3.32, “Preference Dialog”) before test execution.
