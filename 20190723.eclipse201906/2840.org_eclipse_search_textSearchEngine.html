Text Search Engine

Identifier:
org.eclipse.search.textSearchEngine

Since:
3.2

Description:
The extension point allows to register a text search engine that is used to search for text in files of the workspace.

Configuration Markup:

<!ELEMENT extension (textSearchEngine+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT textSearchEngine EMPTY>

<!ATTLIST textSearchEngine

id    CDATA #REQUIRED

label CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - Used to distinguish different search engines.
  • label - Used when search engine is presented in UI.
  • class - The class implementing org.eclipse.search.core.text.TextSearchEngine.

Examples:
As an example, here is the markup for suppling a search engine 


<extension id="MyEngine"
      point="org.eclipse.search.textSearchEngine">
   <textSearchEngine
         id="com.windriver.textSearchEngine"
         label="Wind River Search"
         class="com.windriver.text.SearchEngine" />
</extension>

API Information:
The contributed class must implement org.eclipse.search.core.text.TextSearchEngine
