生成代码

IntelliJ IDEA提供了多种方法来生成通用代码构造和重复元素，从而帮助您提高生产力。这些可以是预定义的模板，也可以是自定义模板，它们根据上下文，各种包装器和字符的自动配对而不同地应用。此外，IntelliJ IDEA还提供代码完成和Emmet支持。

在主菜单上，选择Code |生成 （ Alt + Insert ）以打开具有可生成的可用构造的弹出菜单。

生成构造函数

IntelliJ IDEA可以生成一个构造函数，该构造函数使用相应参数的值初始化特定的类字段。

要为类生成构造函数：

  1. 在代码菜单上，单击生成 （ Alt +插入 ）。

  2. 在“ 生成”弹出窗口中，单击“ 构造函数” 。

  3. 如果类包含字段，请选择要由构造函数初始化的字段，然后单击“ 确定” 。

以下代码片段显示了为类生成构造函数的结果：

public class MyClass { int aInteger ; 双 bDouble ; public MyClass （ int myAIntegerParam ， double myBDoubleParam ） { aInteger = myAIntegerParam ; bDouble = myBDoubleParam ; } }

生成委派方法

IntelliJ IDEA可以生成将行为委托给类的字段或方法的方法。该方法使得可以在不直接暴露该字段或方法的情况下访问字段或方法的数据。

要为类生成委派方法：

  1. 在代码菜单上，单击生成 （ Alt +插入 ）。

  2. 在“ 生成”弹出窗口中，单击“ 委派方法” 。

  3. 选择目标字段或方法，然后单击“ 确定” 。

  4. 选择要委派的所需方法，然后单击“ 确定” 。

生成equals（）和hashCode（）方法

IntelliJ IDEA可以为Java.lang.定义的标准equals()和hashCode()方法生成覆盖Java.lang.Object 。

public boolean equals （ Object obj ）

如果作为参数传递给它的对象等于调用此方法的对象，则前一个方法返回true 。

public int hashCode （）

上一个方法返回调用此方法的对象的整数哈希码值。

如果覆盖equals()通常需要覆盖hashCode()方法，因为hashCode()的契约是它必须为相等的对象产生相同的结果。有关更多信息，请参阅Java SE的API规范 。

要为类生成equals（）和hashCode（）方法：

  1. 在代码菜单上，单击生成 （ Alt +插入 ）。

  2. 在Generate弹出窗口中，单击equals（）和hashCode（） 。

  3. 如果要接受子类并在代码生成期间使用getter，请选中复选框。您还可以从“ 模板”列表中选择速度模板以生成代码或创建自定义模板 。单击下一步 。

  4. 选择应用于确定相等性的字段，然后单击“ 下一步” 。

  5. 选择用于计算哈希码值的字段。您只能从上一步中选择的字段中选择（用于确定相等性）。单击下一步 。

  6. 选择包含非空值的字段。此可选步骤有助于生成的代码避免检查null，从而提高性能。单击完成 。

如果类中已经存在equals()和hashCode()方法的覆盖，则会在生成新的方法之前提示您是否要删除它们。

生成getter和setter

IntelliJ IDEA可以为类中的字段生成访问器和增变器方法（ getter和setter ）。根据JavaBeans API的要求，生成的方法只有一个参数。

getter和setter方法名称由IntelliJ IDEA根据您的代码生成命名首选项生成 。

为类生成getter和setter：

  1. 在代码菜单上，单击生成 （ Alt +插入 ）。

  2. 在“ 生成”弹出窗口中，单击以下选项之一：

    • Getter生成访问器方法以获取类字段的当前值。

    • Setter用于生成用于设置类字段值的mutator方法。

    • Getter和Setter生成访问器和mutator方法。

  3. 选择要为其生成getter或setter的字段，然后单击“ 确定” 。

    您可以通过单击添加自定义getter或setter方法 并访问Getter / Setter Templates对话框。如果字段不在列表中，则已为其定义相应的getter和setter方法。

以下代码片段显示了为具有一个字段var的类生成getter和setter方法的结果：

public class MyClass { int field ; public int getField （） { return field ; } 公共 无效 setField（INT 字段 ）{ 此 。 field = field ; } }

生成toString（）

IntelliJ IDEA可以为Java.lang.定义的标准toString()方法生成覆盖Java.lang.Object 。生成的方法返回类的名称和具有当前字段值的类字段名称列表。

覆盖类的toString（）方法：

  1. 在代码菜单上，单击生成 （ Alt +插入 ）。

  2. 在Generate弹出窗口中，单击toString（） 。

  3. 配置以下内容：

    • 选择用于生成toString()方法的字段。默认情况下，会选择所有可用字段。单击“ 选择无”以添加仅返回类名的toString()方法。

    • 从“ 模板”列表中选择生成toString()方法所需的方法。

    • 如有必要，请选中“ 插入@Override”复选框。有关更多信息，请参阅生成toString（）对话框 。

    • 单击“ 设置”按钮以打开toString（）“生成设置”对话框，您可以在其中调整行为并添加自定义模板 。有关更多信息，请参阅生成toString（）设置对话框 。

  4. 单击确定 。

如果已在类中定义了toString()方法，则默认情况下会提示您是否要在继续之前删除此方法。您可以在Generate toString（）设置对话框中使用When方法已存在的选项组来将行为更改为自动替换现有方法或生成重复方法。

以下代码检查与toString()方法有关：

  • 类不覆盖'toString（）'方法可用于标识未定义toString()方法的类。此检查使用排除设置忽略具有不应转储的字段的类。另一个设置是使用与其类名匹配的正则表达式排除某些类。默认情况下，这用于排除任何Exception类。

  • 'toString（）'方法中未使用的字段可用于标识未在toString()方法中转储的toString() 。例如，如果向类中添加了新字段，但忘记将它们添加到toString()方法中。更改此检查的严重性以将错误显示为警告。这将突出显示编辑器中任何未使用的字段，并在滚动条上将其位置指示为黄色标记。

空白

使用此列表可以定义差异查看器应如何处理空格。

  • 不要忽略 ：空格很重要，所有差异都会突出显示。默认情况下选择此选项。

  • 修剪空格 :( ("\t", " ") ，如果它们出现在行的末尾和行的开头。

    • 如果两行仅在尾随空格中不同，则这些行被认为是相等的。

    • 如果两行不同，则在“ 按字”模式下不会突出显示此类尾随空格。

  • 忽略空格 ：白色空格并不重要，无论它们在源代码中的位置如何。

  • 忽略空格和空行 ：忽略以下实体：

    • 所有空格（如'忽略空白'选项）

    • 所有添加或删除的行仅包含空格

    • 所有更改包括拆分或连接线而不更改非空白部分。
      例如，在此模式下不会突出显示将abc更改为a \nbc 。

运行/调试配置：HTTP请求

此功能仅在Ultimate版本中受支持。

使用此对话框配置运行单独的HTTP请求 。有关使用HTTP请求的详细信息，请参阅IntelliJ IDEA代码编辑器中的HTTP客户端 。

该对话框包含以下区域：

组态

项目

描述

环境

从这个列表中选择一个环境 ，将定义一组请求中使用环境变量。

文件

在此字段中，提供HTTP请求文件的路径。您可以手动键入路径并在键入时使用路径完成（ Ctrl + Space ），或单击并在打开的对话框中选择所需的文件夹。

请求

从此列表中，选择要执行的请求的索引。如果所选的HTTP请求文件包含单个请求，则会禁用该字段并自动选择该请求。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

工具栏

运行/调试配置的树视图有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

删除选定的运行/调试配置。请注意，您无法删除默认配置。

按Ctrl + d

创建所选运行/调试配置的副本。请注意，您可以创建默认配置的副本。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项以运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项可自动运行bundler，作为运行或调试会话的一部分。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过start npm任务运行开发服务器。为此，请单击 ，然后在Configure React Native对话框中，选择npm脚本并从列表中选择start 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上载到远程主机 ：选择此选项可根据默认服务器访问配置将应用程序文件自动上载到 服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：在运行或调试之前添加要执行的Rake任务 。要选择Rake任务，请单击“浏览”按钮 ，然后从可用任务列表中选择所需任务。

    请注意，此处可以使用代码完成 。

  • 运行JRuby编译器 ：选择此选项以使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。

在你开始之前

  1. 下载并安装Node.js.

  2. 如管理插件中所述，在插件页面上安装Cucumber.js和Gherkin存储库插件 。

项目

描述

功能文件或目录

在此字段中，指定要运行的测试。如果要运行一组功能，请键入特定.feature文件或文件夹的路径。

Cucumber.js运行称为功能的测试，并使用Gherkin语言编写。每个功能都在具有扩展feature的单独文件中进行描述。

Cucumber.js参数

在此字段中，指定要传递给可执行文件的命令行参数，例如-r （-- --require LIBRARY|DIR ）， - -t （ - --tags TAG_EXPRESSION ）或--coffee 。有关详细信息，请参阅cucumber-js --help命令提供的本机内置帮助。

名称过滤器

在此字段中，可以选择键入要运行的特定方案的名称，而不是从要素文件或目录中键入所有方案。

如果通过从要运行的方案的上下文菜单中选择“ 创建方案：<Scenario_name>”从编辑器创建运行/调试配置，IntelliJ IDEA会自动在“ 名称过滤器”字段中填写方案的名称 。

节点解释器

在此字段中，指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

黄瓜包

在此字段中，根据您的操作系统，指定存储cucumber包的文件夹的路径或cucumber-js.cmd ， cucumber-js.bat或其他可执行文件的位置。

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

使用

设置屏幕阅读器

目前，我们完全支持Windows上的PyCharm屏幕阅读器。您可以在某种程度上在macOS上使用PyCharm的屏幕阅读器。然而，为了充分利用屏幕阅读器的支持，我们建议使用Windows。

为Windows启用屏幕阅读器

  1. 下载并启用您首选的屏幕阅读器。检查以下推荐的屏幕阅读器：

    • NVDA - 使用NVDA 2015或更高版本。如果使用32位版本的NVDA，则必须在计算机上安装32位JRE，因为此版本的NVDA需要C：\ Windows \ SysWOW64 \ WindowsAccessBridge-32。DLL与PyCharm一起使用。如果NVDA找不到此文件，NVDA事件日志窗口将显示相应的消息。

    • JAWS - 下载所需的版本并重新启动计算机以启用JAWS屏幕阅读器。

  2. 确保已为屏幕阅读器安装了Java Access Bridge和正确的Java版本，如下所示：

    • 要启用Java Access Bridge ，请打开命令提示符并键入[JRE_HOME]\bin\jabswitch -enable ，其中[JRE_HOME]是计算机上JRE的目录。对于Java版本1.8，Java Access Bridge是JDK的一部分，您无需单独下载或启用它。

    • 如果您的屏幕阅读器是32位，请安装32位JRE 1.7或更高版本。如果您的屏幕阅读器是64位，请安装64位JRE版本1.7或更高版本。

    您的计算机可能具有Java Access Bridge的某些重要组件的多个版本，并且它们可能在不同版本之间不兼容。您需要验证Java Access Bridge配置是否正确。

    如果您的屏幕阅读器是32位，请确保C：\ Windows \ SysWOW64 \ WindowsAccessBridge-32。DLL存在且版本号为7.xxx或更高。该文件的描述应为“Java（TM）Platform SE 7”。

使用ping命令，可以确保可以从源计算机访问目标计算机。打开命令行并键入以下命令： ping -a <host_IP> ，其中-a是将地址解析为主机名的命令选项（如果可能）。如果将主机名与ping命令一起使用，则会将主机名解析为IP地址。例如， ping -a example.com解析为PING example.com (93.184.216.34) 。

ping - 一个 < host_IP >

使用telnet命令，您可以测试与远程计算机的连接并发出命令。如果将端口指定为telnet命令的参数，则可以测试与给定端口上的远程主机的连接。如果连接成功，您将看到以下消息： Connected to <host_IP> 。

telnet < host_IP > < port_number >

出于安全原因，DBMS通常会丢弃所有telnet连接。telnet命令允许您检查端口是否已打开以进行通信。

第2步。检查您的连接属性

每个数据库（MySQL，PostgreSQL，Oracle或任何其他供应商）都有自己的连接设置。大多数数据库包括连接设置：

  • 主机 ：存储数据库的计算机或其他设备的主机名。它可以是IP地址（ 127.0.0.1 ）或域名（ localhost ）。

  • 数据库 ：要连接的数据库的名称。您可以在数据库服务器的设置中找到数据库名称，也可以询问数据库管理员。在某些情况下，可以在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW DATABASES; 。

  • 用户 ：具有足够权限以对数据库执行操作的用户的名称。在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW GRANTS; 。

  • 密码 ：用户的密码。

  • 端口 ：标识主机之间连接点的数字。主机使用端口号来确定必须建立连接的应用程序，服务或进程。不同的数据库供应商为其数据库使用不同的端以下列表是默认端口号列表。

第4步。检查是否需要使用SSH或SSL连接

要使数据库连接更安全，某些服务需要使用SSH或SSL。对于某些服务，您需要应用某些JDBC选项以使用SSL或SSH隧道（例如，当您尝试连接到Heroku Postgres时 ）。

使用SSL连接到数据库

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ），然后单击数据源属性图标（ ）。

  2. 选择要更改连接设置的数据源配置文件。

  3. 单击SSH / SSL选项卡，然后选择使用SSL复选框。

  4. 在CA文件字段中，导航到CA证书文件（例如， ca.pem ）。

建议使用PEM证书。

对于自签名证书，在某些情况下使用受信任的根实体颁发的证书，在使用最新的JDBC驱动程序版本时可能会遇到错误。如果Java密钥库不接受证书链，则SSL连接可能会失败。作为临时解决方案，尝试降级JDBC驱动程序（例如，对于MySQL连接器，您需要切换到5.1.40版本。）

设置屏幕阅读器

目前，我们完全支持Windows上的IntelliJ IDEA屏幕阅读器。您可以在某种程度上在macOS上使用屏幕阅读器进行IntelliJ IDEA。然而，为了充分利用屏幕阅读器的支持，我们建议使用Windows。

对于macOS ，请打开VoiceOver并安装并设置IntelliJ IDEA 。

为Windows启用屏幕阅读器

  1. 下载并启用您首选的屏幕阅读器。检查以下推荐的屏幕阅读器：

    • NVDA - 使用NVDA 2015或更高版本。如果使用32位版本的NVDA，则必须在计算机上安装32位JRE，因为此版本的NVDA需要C：\ Windows \ SysWOW64 \ WindowsAccessBridge-32。DLL与IntelliJ IDEA一起使用。如果NVDA找不到此文件，NVDA事件日志窗口将显示相应的消息。

    • JAWS - 下载所需的版本并重新启动计算机以启用JAWS屏幕阅读器。

  2. 确保已为屏幕阅读器安装了Java Access Bridge和正确的Java版本，如下所示：

    • 要启用Java Access Bridge ，请打开命令提示符并键入[JRE_HOME]\bin\jabswitch -enable ，其中[JRE_HOME]是计算机上JRE的目录。对于Java版本1.8，Java Access Bridge是JDK的一部分，您无需单独下载或启用它。

    • 如果您的屏幕阅读器是32位，请安装32位JRE 1.7或更高版本。如果您的屏幕阅读器是64位，请安装64位JRE版本1.7或更高版本。

    您的计算机可能具有Java Access Bridge的某些重要组件的多个版本，并且它们可能在不同版本之间不兼容。您需要验证Java Access Bridge配置是否正确。

    如果您的屏幕阅读器是32位，请确保C：\ Windows \ SysWOW64 \ WindowsAccessBridge-32。DLL存在且版本号为7.xxx或更高。该文件的描述应为“Java（TM）Platform SE 7”。

故障排除

如果代码完成不起作用，可能是由于以下原因之一：

  • 省电模式开启（ 文件|省电模式 ）。通过消除后台操作（包括错误突出显示，动态检查和代码完成），打开它可以最大限度地降低笔记本电脑的功耗。

  • 您的文件不驻留在内容根目录中，因此它不会获得代码完成所需的必需类定义和资源。

  • 包含要在完成建议列表中显示的类和函数的文件将标记为纯文本文件。

  • 包含要在完成建议列表中显示的函数的外部库不会添加为依赖项或全局库 。
