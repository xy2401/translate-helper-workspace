运行/调试配置：规范2

IntelliJ IDEA允许您为Specs2指定运行/调试配置。

该对话框包含以下选项卡：

  • 配置选项卡

  • 代码覆盖率选项卡

  • 日志选项卡

配置选项卡

项目

描述

测试种类

从此下拉列表中，选择测试范围并根据您的选择填写字段。
您可以选择以下选项：

  • 测试名称 - 选择此选项以运行指定的Scala测试。测试名称显示在“ 测试名称”字段中。

  • 全包 - 选择此选项可在指定的包中运行所有Scala测试。
    填写以下字段：

    • 测试包 - 指定要测试的包的名称。

    • 搜索测试 - 使用此下拉列表选择搜索范围。

  • 类 - 选择此选项可在类中运行所有测试。
    在“ 测试类”字段中指定要启动的类的完全限定名称。键入类名称或单击并在打开的对话框中选择所需的类。

VM参数

如有必要，请指定要传递给VM的字符串。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

    - Dfoo = $ { MY_ENV_VAR }

如果空间不足，可以单击并在打开的对话框中输入字符串。

环境变量

点击打开“ 环境变量”对话框，您可以在其中创建变量并指定其值。

测试选项

使用此字段指定其他测试选项。

如果空间不足，可以单击并在打开的对话框中输入字符串。

工作目录

指定运行测试时将充当当前目录的目录。它将充当所有相对输入和输出路径的根目录。默认情况下，项目文件所在的目录用作工作目录。键入目录名称，或单击并在打开的对话框中选择所需的目录。

使用模块的classpath和SDK

从此下拉列表中，选择将使用其类路径运行应用程序的模块。

将信息消息打印到控制台

如果要将信息消息打印到Scala控制台，请选中此复选框。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

IntelliJ IDEA 2019.1帮助

npm和Yarn

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA与npm和Yarn包管理器集成，因此您可以从IDE内部安装，定位，升级和删除可重用代码包。Node.js和NPM页面提供了用于管理包的专用UI。当然，您也可以从内置终端的命令行执行此操作。

IntelliJ IDEA还允许您运行和调试npm脚本 。IntelliJ IDEA解析package.json文件，识别脚本的定义，在树视图中显示脚本，并允许您在树中的脚本及其在package.json文件中的定义之间进行导航。

IntelliJ IDEA使用Yarn工作区检测项目，并索引工作区的package.json文件中列出的所有依赖项，但这些依赖项位于根node_modules文件夹中。

在你开始之前

  1. 下载并安装Node.js.请注意，npm也已安装，因此如果您要使用它，则需要执行初步步骤。

  2. 要使用Yarn，请按照纱线官方网站上的说明进行安装。

选择项目包管理器

IntelliJ IDEA使用分配给这些管理器的当前系统路径的npm和yarn别名。如果您选择另一个Node.js版本，IntelliJ IDEA还会将npm别名路径更改为指向与所选Node.js版本捆绑在一起的npm版本。

使用IntelliJ IDEA，您可以选择是否在项目中使用npm或Yarn。默认情况下，IntelliJ IDEA建议使用npm。但是，如果您使用yarn.lock文件打开项目并且计算机上安装了Yarn，则IntelliJ IDEA会自动将此项目的包管理器更改为Yarn。您还可以将Yarn设置为默认的IntelliJ IDEA包管理器 。

为当前项目选择包管理器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| Node.js和NPM 。Node.js和NPM页面打开。

  2. 从包管理器列表中，选择npm或yarn以使用系统默认安装。要使用包管理器的自定义安装，请单击“ 选择” ，然后在打开的对话框中选择包管理器的相关路径。

    当您从“ 运行/调试配置：NPM”对话框的“ 包管理器”列表中选择“ Project别名”时，IntelliJ IDEA会自动使用所选的包管理器。每次调用Run'npm install' / Run'yarn install'命令或运行npm / Yarn脚本时，IntelliJ IDEA也会使用所选包管理器的路径，有关详细信息，请参阅运行和调试脚本 。

将Yarn设置为默认的IntelliJ IDEA包管理器

  1. 在“ 设置新项目”对话框（“ 文件”|“新项目的设置...” ）中，单击“ 语言和框架”下的“ Node.js”和“NPM ”。

  2. 在打开的Node.js和NPM页面上，从包管理器列表中选择yarn 。之后，IntelliJ IDEA会在每次创建新项目时将Yarn建议为默认值。

使用内置终端安装软件包

您可能知道，npm可以在全局或作为项目依赖项或开发依赖项安装软件包，请从npm官方网站了解更多信息。使用Yarn，您还可以在全局或作为项目依赖项安装软件包，有关详细信息，请参阅Yarn官方网站。

安装包

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install --global <package_name>或yarn global add <package_name>进行全局安装。

    • npm install --save <package_name> ， npm install --save-dev <package_name>或yarn add <package_name> --dev以将程序包安装为项目依赖项或开发依赖项。

从package.json文件安装所有依赖项

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm安装

    • 纱线安装

    因此，您可以从当前文件夹中获取package.json列出的所有依赖项。从编辑package.json了解更多信息。

在Node.js和NPM页面上安装和升级软件包

IntelliJ IDEA显示Node.js和NPM页面上所有当前安装的软件包。要打开页面，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ Node.js”和“NPM ”。全局安装的软件包列在顶部。对于每个包，IntelliJ IDEA显示其当前安装的版本和最新的可用版本。

安装包

  1. 点击在打开的“ 可用包”对话框中，选择所需的包。

  2. 默认情况下，安装最新的软件包版本。要安装另一个，请选中“ 指定版本”复选框，然后从列表中选择所需的版本。

  3. 默认安装是本地安装。要使用其他安装类型，请选中“ 选项”复选框并键入-g以进行全局安装，或键入--save / --save-dev以将程序包安装为依赖项/开发依赖项。从npm和Yarn官方网站了解更多信息。

查看包的位置

  • 将鼠标指针悬停在包名称上。IntelliJ IDEA在工具提示中显示包的路径。

将软件包升级到最新版本

  • 在列表中选择包，然后单击 。

删除包

当您从require或import语句调用文档查找时，IntelliJ IDEA还会显示npm包的readme文件。

IntelliJ IDEA可帮助您处理package.json文件中的项目依赖项，从而提供广泛的编码帮助：

  • 包名称的代码完成。

  • 有关最新可用软件包版本的信息。

  • 有关可通过npm install或yarn install的版本范围的信息。按Ctrl或Cmd并将鼠标悬停在版本上以查看工具提示中的信息。有关语义版本控制的详细信息，请参阅npm官方文档 。

  • 以前的软件包版本的代码完成。当您按Ctrl + Space或开始键入与最新版本不同的版本时，IntelliJ IDEA会显示包含所有先前版本的软件包的建议列表。

  • 快速查找包的文档。

运行/调试配置：Android应用程序

使用此对话框可以创建用于在实际或虚拟设备上运行或调试Android应用程序和活动的配置。

此对话框包含三个选项卡：

  • 一般

  • 杂

  • 剖析

常规选项卡

项目

描述

模

从此下拉列表中，选择将应用此配置的模块。

安装选项

使用此区域配置以下设置：
  • 部署 - 使用此下拉列表指定将部署到目标虚拟或物理设备的.apk文件。您可以选择以下选项之一：

    • 默认APK - 选择此选项可让IntelliJ IDEA上传从上面的“ 模块”下拉列表中指定的模块构建的.apk 。.apk是自动构建的，不需要从您身边进行初步的工件配置。

    • 自定义工件 - 选择此选项可让IntelliJ IDEA上载根据手动配置的工件中指定的代码和资源构建的.apk 。从下拉列表中选择一个工件，该列表显示所选模块的所有手动创建的工件定义。有关详细信息，请参阅使用工件 。

    • 无 - 选择此选项可禁止将数据上传到设备。如果您已在IntelliJ IDEA之外手动部署应用程序，则此方法可能会有所帮助。

  • 安装标志 - 使用此字段指定pm install命令的选项。
    有关更多信息，请参阅包管理器页面。

启动选项

在此区域中，指定将在应用程序启动时启动的活动。
  • 启动 - 使用此下拉列表指定以下启动设置：

    • 默认活动 - 选择此选项可让IntelliJ IDEA自动启动标记为启动的活动 。

    • 没有 - 如果您在“ 安装选项”区域中选择了“ 无” ，请选择此选项以连接到已在运行的应用程序。使用这些设置执行运行配置与单击“ 将调试器附加到Android进程”按钮相同在工具栏上。

    • 指定的活动 - 选择此选项可让IntelliJ IDEA在应用程序启动时启动您选择的活动。如果要运行或调试应用程序的一部分并且已在“ 安装选项”区域中选择了“ 自定义工件”选项，则此选项可能会有所帮助。手动键入活动名称或单击“ 浏览” 并在打开的“ 选择活动类”对话框中选择它。

      可用活动列表由模块的选择决定。

    • URL - 选择此选项可在运行应用程序时启动浏览器。您可以在URL字段中指定URL地址。

  • 启动标志 - 使用此字段为am start命令指定其他选项。
    有关更多信息，请参阅活动管理器页面。

部署目标选项

在此区域中，选择要在其上运行/调试应用程序的设备类型。可用选项包括：
  • 显示设备选择器对话框：选择此选项可使IntelliJ IDEA在应用程序启动时显示“ 选择设备”对话框。

    如果您希望将来自动使用通过“ 选择设备”对话框选择的设备 ，请选择“ 将相同设备用于将来启动”选项。

  • USB设备：选择此选项可让IntelliJ IDEA在应用程序启动时检测到插入的USB设备。

如果要在稍后插入的物理设备上运行应用程序，则选择“ 显示设备选择器”对话框或“ USB设备”选项可能会有所帮助，因此无法预见可用设备集。

  • 仿真器：选择此选项以使用其中一个已配置的设备仿真器。从首选Android虚拟设备下拉列表中，选择将用于运行/调试指定活动的虚拟设备。如果要添加/编辑虚拟设备配置，请单击“ 浏览”按钮启动Android虚拟设备（AVD）管理器 （有关详细信息，请参阅管理虚拟设备 ）。

杂项标签

在此选项卡中，配置运行/调试会话期间显示的日志数据范围。

项目

描述

logcat的

使用此区域为Android运行/调试配置配置logcat设置：

  • 自动显示logcat ：选中此复选框可在每次部署和成功启动应用程序时自动显示logcat。

  • 启动前清除日志 ：如果希望在启动应用程序之前从日志文件中删除以前会话中的数据，请选中此复选框。

安装选项

使用此区域配置运行或调试Android应用程序的安装选项：

  • 如果APK未更改，请跳过安装：如果APK未更改，请选中此复选框以在运行应用程序时跳过APK安装。

  • 在启动活动之前强制停止运行应用程序 ：选择此复选框以在开始从IntelliJ IDEA运行活动之前停止运行已启动的应用程序。

分析标签

在此选项卡中，指定“图形跟踪”选项。

项目

描述

禁用预编译的着色器和程序

如果要禁用预编译着色器和程序并在运行时编译它们，请选中此复选框。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

工具栏

运行/调试配置的树视图有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

然后，要将配置移动到文件夹中，文件夹之间或文件夹之外，请使用拖放或和纽扣。

要删除分组，请选择一个文件夹并单击 。

排序配置

单击此按钮可按字母顺序对配置进行排序。

在启动选项之前

在此区域中，您可以指定在启动所选运行/调试配置之前必须执行的任务。任务按照它们在列表中出现的顺序执行。

项目

捷径

描述

Alt +插入

单击此图标可添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择要运行的一个或多个应用程序。如果尚未在IntelliJ IDEA中定义，请添加其定义。有关更多信息，请参阅配置第三方工具和外部工具 。

  • 运行另一个配置 ：选择执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • Build ：选择编译指定的模块。将执行Build Module命令 。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建项目 ：选择编译整个项目。Build Project命令将被执行。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建，无错误检查 ：与Build选项相同，但IntelliJ IDEA将尝试启动运行/调试配置，而不管编译结果如何。

  • 构建工件 ：选择此选项可构建工件或工件。在打开的对话框中，选择应构建的工件或工件。

  • 运行Ant目标 ：选择此选项以运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项以运行Grunt任务。在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项以运行Gulp任务。在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项以运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项可自动运行bundler，作为运行或调试会话的一部分。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过start npm任务运行开发服务器。为此，请单击 ，然后在Configure React Native对话框中，选择npm脚本并从列表中选择start 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上载到远程主机 ：选择此选项可根据默认服务器访问配置将应用程序文件自动上载到 服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：在运行或调试之前添加要执行的Rake任务 。要选择Rake任务，请单击“浏览”按钮 ，然后从可用任务列表中选择所需任务。

    请注意，此处可以使用代码完成 。

  • 运行JRuby编译器 ：选择此选项以使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。
