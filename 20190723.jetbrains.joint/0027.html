  • 概观
  • 确保启用了Hibernate支持插件
  • 在创建项目或模块时启用Hibernate支持
  • 为现有模块启用Hibernate支持

概观

为了能够使用Hibernate支持，您应该：

  • 确保已启用Hibernate支持插件 。（此插件与IDE捆绑在一起并默认启用。）

  • 在模块级别启用Hibernate支持。您可以在创建新项目或模块时执行此操作。您还可以为现有模块启用Hibernate支持。在所有这些情况下，IntelliJ IDEA将（以下某些选项将作为选项提供）：

    • 创建一个Hibernate配置文件hibernate.cfg.xml和一个带有main()方法的类，该方法输出托管实体的记录。

    • 下载实现框架的库文件，并将它们添加到相应模块的依赖项中。

    • 为数据库表生成实体类和对象/关系映射（如果有适当的数据源可用）。

    • 创建一个Hibernate方面。您将能够使用该方面来管理配置hibernate.cfg.xml文件。

    • 使“ 持久性”工具窗口可用。

确保启用了Hibernate支持插件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

  2. 在对话框的右侧，在“ 插件”页面上 ，在搜索框中键入hib 。因此，仅显示名称和描述包含hib的插件。

  3. 如果未选中Hibernate Support右侧的复选框，请选择它。

  4. 单击确定 。

  5. 如果建议，请重新启动IntelliJ IDEA。

在创建项目或模块时启用Hibernate支持

  1. 执行以下操作之一：

    • 如果要创建新项目：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件”|新的|项目 。

      结果，“ 新建项目”向导将打开。

    • 如果要将模块添加到现有项目：打开要添加模块的项目，然后选择“ 文件”|新的|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页上，在左侧窗格中，选择Java 。在页面的右侧，指定要使用的JDK 。

  3. 在Additional Libraries and Frameworks下 ，选择Hibernate复选框。

  4. 如有必要，选择创建配置文件（ hibernate.cfg.xml ）和主类（具有main()方法的类，该方法输出托管实体的记录）。

    如果在IntelliJ IDEA中将数据库定义为数据源，则可以选择导入数据库模式。这将导致为它们创建必要的实体类和对象/关系映射。

  5. 选择所需的库选项，并在必要时指定关联的设置。您可以选择：
    • 下载Hibernate实现文件并将这些文件排列在库中 。

      为此，在“ 库”下，选择“ 下载” 。（可选）单击“ 配置”以编辑库设置。（“ 下载选项”对话框将打开。）

    • 使用已定义IntelliJ IDEA的库。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。（可选）单击“ 配置”以编辑库设置。（“ 编辑库”对话框将打开。）

    • 使用计算机上可用的相应JAR文件创建新库。

      为此，请单击“ 使用库” ，然后单击“ 创建” 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请按住Ctrl键。）（可选）单击“ 配置”以编辑库设置。（“ 创建库”对话框将打开。）

    • 推迟设置库直到稍后的时间。在这种情况下，请选择稍后设置库 。

    单击下一步 。

  6. 指定名称和位置设置。

    单击完成 。

如果选择导入数据库模式， 则会打开“ 导入数据库模式”对话框 ，您可以指定要生成的实体类和相关设置。

为现有模块启用Hibernate支持

  1. 在“ 项目工具”窗口中 ，右键单击所需的模块，然后选择“ 添加框架支持” 。

  2. 在打开的“ 添加框架支持”对话框的左侧窗格中，选择“ 休眠”复选框。

  3. 如有必要，选择创建配置文件（ hibernate.cfg.xml ）和主类（具有main()方法的类，该方法输出托管实体的记录）。

    如果在IntelliJ IDEA中将数据库定义为数据源，则可以选择导入数据库模式。这将导致为它们创建必要的实体类和对象/关系映射。

  4. 选择所需的库选项，并在必要时指定关联的设置。您可以选择：
    • 下载Hibernate实现文件并将这些文件排列在库中 。

      为此，在“ 库”下，选择“ 下载” 。（可选）单击“ 配置”以编辑库设置。（“ 下载选项”对话框将打开。）

    • 使用已定义IntelliJ IDEA的库。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。（可选）单击“ 配置”以编辑库设置。（“ 编辑库”对话框将打开。）

    • 使用计算机上可用的相应JAR文件创建新库。

      为此，请单击“ 使用库” ，然后单击“ 创建” 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请按住Ctrl键。）（可选）单击“ 配置”以编辑库设置。（“ 创建库”对话框将打开。）

    • 推迟设置库直到稍后的时间。在这种情况下，请选择稍后设置库 。

  5. 单击确定 。

如果选择导入数据库模式， 则会打开“ 导入数据库模式”对话框 ，您可以指定要生成的实体类和相关设置。

IntelliJ IDEA允许您在逐个类的基础上指定调试器中不同对象的显示方式。您可以指定要显示的表达式，而不是依赖于对象的String表示。

例如，如果一个对象代表一个用户，您可能希望它由登录名表示;或者，对于缓存条目对象，其内容可能是适当的。IntelliJ IDEA将这些称为类型渲染器 。

支持所有对象类型（包括基元类型和数组）。

如果未定义渲染方案，则此对话框不显示任何控件。要开始使用渲染器，请单击 。

描述

单击此图标可从列表中删除所选方案。

单击此图标可创建所选方案的副本。

单击这些图标可在列表中向上或向下移动选定项目一行。

请注意，顺序确定在类继承产生歧义的情况下使用哪个渲染器。

渲染器名称

指定新渲染器的名称，或编辑现有渲染器名称。

将渲染器应用于类型的对象（完全限定名称）

指定将由此渲染器表示的对象类型。输入完全限定的对象名称，或单击“ 浏览” 并从“ 渲染器参考类型”对话框的列表中选择所需的类型。

渲染节点时

此选项确定在折叠节点时对象在调试器中的显示方式：

  • 显示类型和对象ID ：如果清除，则显示的类型没有类信息或id。

  • 使用默认渲染器 ：选择此选项以默认方式显示节点。

  • 使用以下表达式 ：输入要用于标识对象的Java表达式。您可以使用对象属性，常量甚至字符串数学作为渲染器的一部分。

    请注意，您可以在定义表达式时使用代码完成（ Ctrl + Space ）。

    所有方法调用和成员变量访问都与您正在呈现的对象相关。使用this可引用渲染器应用的实例。

    • 在渲染器中使用沉重的表达式可能会减慢视图中的数据渲染速度。

    • 由于可能存在副作用，应谨慎使用方法调用。

扩展节点时

此选项确定在展开节点时对象在调试器中的显示方式。

通常，在调试器中展开节点会列出对象的成员变量（使用适合相应对象类型的渲染器）。此选项允许您覆盖此行为并选择要显示的单个表达式或一系列表达式。您可以使用它来限制显示的信息量，或者更准确地显示信息的显示方式。

  • 使用默认渲染器 ：选择此选项可以默认方式显示节点子节点。

  • 使用以下表达式 ：输入要用于标识对象的Java表达式。

    测试节点是否可以展开（可选） ：输入布尔表达式。如果为true ，则渲染器将显示已定义对象的可扩展节点。否则，不显示任何节点。

  • 使用表达式列表 ：创建要计算的单独表达式列表，并将其显示为节点子项。使用：

    （ Alt + Insert ）创建一个新表达式。

    （ Alt + Delete ）从列表中删除选定的表达式。

    （ Alt + Up ）将所选表达式在列表中向上移动一行。

    （ Alt + Down ）将所选表达式在列表中向下移动一行。

如果选中渲染器旁边的“按需”列中的复选框，则将根据需要对此表达式进行评估。当您需要在变量 ， 手表或其他视图中对其进行评估时，只需单击此表达式，而不是自动评估它。

您可以在定义表达式时使用代码完成（ Ctrl + Space ）。

附加默认子项

选中此复选框可将默认子项添加到表达式列表中。仅当选中“ 使用表达式列表 ”复选框时，此复选框才可用。

      调试异步代码

      调试异步代码是一个挑战，因为执行在帧之间跳转并使得遵循代码变得更加困难。为了帮助您从代码中的特定点回顾，IntelliJ IDEA提供了捕获点 。

      捕获点是代码中的一个位置，调试器捕获堆栈跟踪，以便在到达代码中的特定点（插入点）时使用，并希望了解如何到达那里。捕获点由方法名称（和包含的类）和键表达式指定。当调试器停止时，它开始匹配堆栈帧与插入点 ，这是另一种方法和表达式。如果找到匹配项，则调试器将计算表达式，如果该值具有某些相关的堆栈信息，则它将使用捕获的堆栈替换调用堆栈的其余部分。这有助于您了解相关捕获点发生的情况，以及您如何达到目前的要点。
