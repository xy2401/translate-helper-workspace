配置Zend调试器

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

在这个部分：

  • 下载并安装Zend Debugger
  • 启用Zend Debugger与PHP解释器的集成
  • 将Zend Debugger与IntelliJ IDEA集成
  • 配置Zend Debugger以在按需模式下使用

下载并安装Zend Debugger

  1. 下载与您的操作系统对应的Zend Debugger软件包 。

  2. 在与您的PHP版本对应的目录中找到ZendDebugger.so （Unix）或ZendDebugger.dll （Windows）文件（例如4.3.x，4.4.x，5.0.x，5.1.x，5.2.x，5.3。 x，5.4.x）。

  3. 将文件复制到Web服务器可访问的位置中的Web服务器。

启用Zend Debugger与PHP解释器的集成

  1. 在编辑器中打开活动的php.ini文件：

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 语言和框架”下的“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的CLI解释器对话框中， 配置文件只读字段显示活动php.ini文件的路径。单击在编辑器中打开 。

  2. 找到或创建[Zend]部分。

  3. 要加载Zend Debugger扩展，请根据您的操作系统在[Zend]部分中添加以下行之一：

    • Linux和macOS ：

      zend_extension = < full_path_to_ZendDebugger 。 所以 >

    • Windows ：

      zend_extension_ts = < full_path_to_ZendDebugger 。 dll >

    • Windows非线程安全 ：

      zend_extension = < full_path_to_ZendDebugger 。 dll >

      Windows非线程安全二进制文件仅用于Zend Core 2.0。

  4. 要从IntelliJ IDEA启用对Zend Debugger的访问，请添加以下行：
    zend_extension = <full_path_to_zend_debugger_extension> zend_debugger.allow_hosts = 127.0.0.1 zend_debugger.expose_remotely = allowed_host zend_debugger.tunnel_min_port = <1024以上的任何整数值> zend_debugger.tunnel_max_port = <65535以下的任何整数值>

    zend_debugger.allow_hosts参数的值是连接到服务器调试器的计算机的IP。它可以是以XXXX格式（例如，192.168.0.6）的以逗号分隔的IP列表。

    对于线程安全的Windows二进制文件，请使用zend_extension_ts parameter而不是zend_extension 。

  5. 重新启动Web服务器。

  6. 要检查Zend Debugger是否已正确安装和配置，请创建一个包含以下内容的文件：

    <？php phpinfo（）; ？>
    在浏览器中打开与该文件对应的页面。输出应包含Zend Debugger部分。

将Zend Debugger与IntelliJ IDEA集成

  1. 按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，然后单击Languages＆Frameworks下的PHP 。

  2. 检查与所选PHP解释器关联的Zend Debugger安装：

    1. 在PHP页面上，从CLI解释器列表中选择相关的PHP安装，然后单击“ 浏览” 在田野旁边。该列表显示了IntelliJ IDEA中可用的所有PHP安装，请参阅配置本地PHP解释器和配置远程PHP解释器 。

    2. 打开的CLI解释器对话框显示以下内容：

      • 所选PHP安装的版本。

      • 与所选PHP安装（Xdebug或Zend Debugger）关联的调试引擎的名称和版本。如果未配置调试器，IntelliJ IDEA将显示Debugger: Not installed 。

    了解有关在验证调试引擎配置中检查Zend Debugger安装的详细信息。

  3. 定义Zend Debugger行为。单击PHP节点下的Debug 。在打开的Debug页面上，在Zend Debugger区域中指定以下设置：

    • 在“ 调试端口”文本框中，指定IntelliJ IDEA的端口以通过该工具进行通信。通过zend_debugger.tunnel_min_port和zend_debugger.tunnel_max_port在php.ini文件中指定的隧道中键入端口号。有关详细信息，请参阅http://files.zend.com/help/previous-version/Zend-Server-4-Community-Edition/zenddebugger.html

    • 要让IntelliJ IDEA通过调试端口文本框中指定的端口接受来自Zend Debugger引擎的任何传入连接，请选中“ 可以接受外部连接”复选框。

    • 要在浏览器中使用调试器工具栏，请在“ 设置广播端口”文本框中指定将调试器设置传递到浏览器的端口 。

    • 使用“ 自动检测IDE IP”复选框可以启用和禁用热IP地址的自动检测。选中此复选框后，IntelliJ IDEA将通过debug_host参数检测要发送到Zend Debugger的所有主机IP地址。所有检测到的IP地址都列在右侧的字段中。使用Vagrant ， VirtualBox或其他虚拟化工具时，自动检测IP地址非常有用。
      清除该复选框以阻止自动检测主机IP地址，并在字段中明确指定所需的地址。

  4. 在“ 外部连接”区域中，指定IntelliJ IDEA如何处理从主机接收的连接以及未注册为部署服务器配置的端口。

    • 通过未注册的服务器配置忽略外部连接：选中此复选框可使IntelliJ IDEA忽略从主机和未注册为部署服务器配置的端口收到的连接。选中此复选框后，IntelliJ IDEA不会尝试自动创建部署服务器配置。

    • 在PHP脚本的第一行中断 ：选中此复选框可在调试器与IntelliJ IDEA建立连接后立即停止（而不是在达到第一个断点之前自动运行）。或者打开Run |在主菜单上的PHP脚本选项的第一行中断 。

    • 最大。同时连接：使用此旋转框限制可以同时处理的外部连接数。

  5. 为了阻止来自请求Z-Ray系统 ，如果他们通过调用IntelliJ IDEA的调试器过于频繁地骚扰你，选择忽略Z-Ray系统请求复选框。

配置Zend Debugger以在按需模式下使用

IntelliJ IDEA 2016.2及更高版本支持按需模式，您可以在其中禁用Zend Debugger以进行全局PHP安装，并且只有在调试命令行脚本或需要代码覆盖率报告时才能按需自动启用它。这使您的命令行脚本（包括Composer和单元测试）运行得更快。

  1. 禁用Zend Debugger用于命令行脚本：

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 语言和框架”下的“ PHP ” 。

    2. 从PHP可执行文件列表中，选择相关的PHP解释器并单击在它的旁边。在打开的CLI Interpreters对话框中，单击Configuration file：<path to php.ini>文件旁边的Open in Editor链接。关闭所有对话框并切换到打开php.ini文件的选项卡。

    3. 在php.ini文件中，找到[Zend]部分并通过添加注释其中的以下行;介词： Linux和macOS ：

      zend_extension = < full_path_to_ZendDebugger 。 所以 >
      Windows ：
      zend_extension_ts = < full_path_to_ZendDebugger 。 dll >

      Windows非线程安全 ：

      zend_extension = < full_path_to_ZendDebugger 。 dll >

      Windows非线程安全二进制文件仅用于Zend Core 2.0。

    4. 打开CLI Interpreters对话框，然后单击在PHP可执行文件字段旁边。IntelliJ IDEA通知您未安装调试器：

  2. 要使IntelliJ IDEA在必要时激活Zend Debugger，请在“ 调试器”扩展字段的“ 附加”区域中指定它的路径。手动键入路径或单击并在打开的对话框中选择位置。

配置颜色和字体

作为开发人员，您需要处理大量文本资源：编辑器中的源代码，搜索结果，调试器信息，控制台输入和输出等。颜色和字体样式用于格式化此文本，并帮助您一目了然地更好地理解它。

IntelliJ IDEA使用定义首选颜色和字体的颜色方案 。

颜色方案与界面主题不同 ， 界面主题定义了窗口，对话框和控件的外观。

您可以使用预定义的颜色方案或根据自己的喜好自定义颜色方案。也可以共享方案。

选择颜色方案

  1. 按Ctrl + Alt + S打开“ 设置/首选项”对话框，然后转到“ 编辑器”|配色方案 。

  2. 使用“ 方案”列表选择颜色方案。

默认情况下，有以下预定义颜色方案：

  • 默认值 ：为Light主题设计的配色方案。

  • Darcula ：为Darcula主题设计的配色方案。

  • 高对比度 ：为视力不足的用户设计的配色方案（ 高对比度主题）。

如果您安装带有配色方案的插件，该方案将添加到预定义方案列表中。有关更多信息，请参阅共享颜色方案 。

自定义颜色方案

您可以自定义预定义的颜色方案，但建议为自定义颜色和字体设置创建副本：

复制配色方案

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案 。

  2. 选择颜色方案，单击 ，然后单击“ 复制” 。

  3. （可选）要重命名自定义方案，请单击并选择重命名 。

预定义的颜色方案以粗体字体列出。如果您自定义预定义的配色方案，它将以蓝色显示。要将预定义颜色方案恢复为默认设置，请单击并选择恢复默认值 。您无法删除预定义的配色方案。

要定义颜色和字体设置，请展开编辑器| “ 设置/首选项”对话框中的“ 颜色方案”部分（ Ctrl + Alt + S ）。设置分为几个部分。例如，“ 常规”部分定义基本编辑器颜色，例如装订线，行号，错误，警告，弹出窗口，提示等。“ 语言默认值”部分包含常用的语法突出显示设置，默认情况下应用于所有支持的编程语言。在大多数情况下，配置语言默认值并在必要时对特定语言进行调整就足够了。要更改元素的继承颜色设置，请清除“ 从中继承值”复选框。

语义突出显示

默认情况下，颜色方案为源代码中的保留字和其他符号定义语法突出显示：运算符，关键字，建议，字符串文字等。如果您的函数或方法具有许多参数和局部变量，则可能很难将它们彼此区分开来。您可以使用语义突出显示为每个参数和局部变量指定不同的颜色。

分享配色方案

如果您习惯于特定的配色方案，则可以从一个安装中导出它并将其导入另一个安装。您还可以与其他开发人员共享配色方案。如有必要，您可以从Eclipse导入您喜欢的颜色设置。

将颜色方案导出为XML

IntelliJ IDEA可以将颜色方案设置保存为扩展名为.icls的XML文件。然后，您可以将文件导入另一个安装。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案 。

  2. 从Scheme列表中，选择一种配色方案，然后单击 ，然后单击“ 导出”并选择“ IntelliJ IDEA配色方案（.icls）” 。

  3. 指定文件的名称和位置并保存。

将颜色方案导出为插件

该插件可以上传到插件存储库以供其他人安装。这种格式比XML文件有几个好处，包括元数据，反馈，下载统计和版本控制（当您上传新版本的插件时，将通知用户）。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案 。

  2. 从Scheme列表中，选择一种配色方案，然后单击 ，然后单击“ 导出”并选择“ 颜色方案插件（.jar）” 。

  3. 在“ 创建颜色方案插件”对话框中，指定版本详细信息和供应商信息。然后单击确定 。

当您使用颜色方案安装插件时，该方案将添加到预定义方案列表中。

导入配色方案

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案 。

  2. 从Scheme列表中，选择一种配色方案，然后单击 ，然后单击“ 导入方案”并选择您的首选选项：

    • IntelliJ IDEA颜色方案（.icls）或设置（.jar）用于导入标准XML文件或导出的设置（使用“ 文件”|“导出设置”菜单创建）。

    • Eclipse Color Theme（XML）用于从Eclipse导入颜色设置。

字体

要自定义默认字体，请转到编辑器| “ 设置/首选项”对话框中的字体 （ Ctrl + Alt + S ）。默认情况下，此字体在所有颜色方案中使用和继承。

自定义颜色方案字体

您可以为当前方案设置不同的字体。

如果您计划共享您的方案或在其他可能不支持所选字体的平台上使用它，则不建议这样做。在这种情况下，请使用默认的全局字体设置。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案|配色方案字体 。

  2. 选择使用颜色方案字体而不是默认复选框。

自定义控制台字体

默认情况下，控制台中的文本使用与颜色方案相同的字体。要在控制台中使用其他字体：

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案|控制台字体 。

  2. 选择使用控制台字体而不是默认复选框。

生产力提示

您可以使用“ 跳转到颜色和字体”操作导航到插入符号下符号的相关颜色方案设置。按Ctrl + Shift + A并开始键入操作名称以查找并执行它。

“ 跳转到颜色和字体”操作没有默认快捷方式。要为其指定快捷方式，请选择该快捷方式，然后按Alt + Enter 。

管理编辑器选项卡

每次打开文件进行编辑时，都会在编辑器窗口中的活动编辑器选项卡旁边添加一个专用选项卡。

PyCharm可以同时限制编辑器中打开的选项卡数量。当选项卡的数量达到其限制时，编辑器将根据编辑器中定义的选项卡关闭策略关闭选项卡一般| PyCharm设置的编辑器选项卡页面（ Ctrl + Alt + S ）默认情况下，选项卡限制为10，但您可以根据需要进行更改。

与管理编辑器选项卡相关的所有命令均可从以下位置获得：

  • 窗口|编辑器选项卡菜单。

  • 选项卡的上下文菜单。

配置编辑器选项卡的行为

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 编辑器”下的“ 常规 ” ，然后单击“ 编辑器选项卡” 。将打开“ 编辑器选项卡”页面 。

  2. 更改允许的最大选项卡数：在选项卡限制字段中键入要一次打开的编辑器选项卡所需的最大允许数。
    如果选项卡限制等于1，则将禁用选项卡。如果希望编辑器永远不关闭选项卡，请键入一些无法访问的编号。
    使用禁用的选项卡，使用View |最近的文件 （ Ctrl + E ）命令可以在文件之间快速切换。

  3. 默认情况下，选项卡标题显示在编辑器的顶部，但您可以根据需要更改其位置，并在编辑器的底部，左侧或右侧显示标题。请注意，选项卡标题放置是适用于所有项目的全局设置。
    使用“ 放置”选择器选择标签标题的所需位置。
    要完全禁用编辑器选项卡，请从“ 放置”选择器中选择“ 无 ”。

在编辑器选项卡之间导航

从当前选项卡导航到下一个或上一个选项卡

此方法允许在编辑会话需要时从一个选项卡跳转到另一个选项卡。当您在编辑器选项卡之间移动时，PyCharm会记住每个打开的文件中的插入位置。

  • 右键单击当前编辑器选项卡， 然后从上下文菜单中选择“ 选择下一个/上一个选项卡 ”

  • 按Alt + Right或Alt + Left 。这样，焦点移动到位于活动编辑器选项卡右侧或左侧的编辑器选项卡。

  • 按Ctrl + Tab键以使用切换器 。

来回浏览所访问标签的历史记录

这种方法使您可以在导航历史中来回移动，就像在Web浏览器中一样。当您在编辑会话期间从一个文件移动到另一个文件时，PyCharm会跟踪访问过的位置并使您能够返回，使用Navigate |后退/前进命令。

在macOS计算机上，您还可以使用三指从右到左和从左到右的滑动手势。

  • 在主工具栏上，单击要么 。

  • 选择导航|从主菜单返回/前进 。

  • 按Ctrl + Alt + Left或Ctrl + Alt + Right 。

要查看所有编辑器选项卡并选择活动编辑器

如果所有打开的选项卡都显示在一行中，则某些选项卡可能会变为不可见。PyCharm帮助显示打开的编辑器选项卡列表，这些选项卡不适合编辑器窗口并且变得不可见，因此您可以选择要激活的选项卡。

  1. 执行以下操作之一：

    • 选择窗口|编辑器标签|从主菜单中显示隐藏选项卡 。

    • 点击

    将显示已打开但不可见的所有选项卡的列表。因此，在明亮的背景上显示当前可见的选项卡的名称;主窗口外部的选项卡名称显示在较暗的背景上。

  2. 单击所需编辑器选项卡的名称：

    选定的编辑器选项卡变为活动状态并获得焦点。

命令显示所有选项卡和图标仅在以下情况下可用：

  • 在编辑器中选中“在单行中显示选项卡”复选框一般|编辑器选项卡 PyCharm设置页面（ Ctrl + Alt + S ）。

  • 某些打开的选项卡不可见，因为它们不适合编辑器窗口。

固定和取消固定标签

达到指定的选项卡限制时，PyCharm将在您打开新文档时关闭旧选项卡。要防止选项卡自动关闭，您可以固定此选项卡。此外，当您关闭编辑器选项卡时，您可以选择打开固定选项卡并仅关闭未固定选项卡。

在本主题中：

  • 提取样式
  • 内联风格
  • 提取布局

提取样式

Android样式有助于将应用程序设计与应用程序功能分开，就像层叠样式表（CSS）一样 。

通过提取样式重构，您可以从布局XML标记创建样式。IntelliJ IDEA在/res/values/styles.xml文件中创建样式声明，并将标记属性转换为在item元素中声明的样式属性。好处是现在您不再需要解析整个资源定义文件来更新标记属性以编辑布局。您需要做的就是更新样式定义，然后在引用样式的任何位置自动应用更改。

要应用提取样式重构

  1. 执行以下操作之一：

    • 在可视模式下 ，在“ 设计”窗格或“ 组件树”中的画布上的定义中选择使用源XML标记的组件 。

    • 在手动模式下 ，将光标定位在源XML标记内的任何位置。

  2. 从主菜单中，选择Refactor |重构此按钮或按Ctrl + Shift + Alt + T.在出现的弹出窗口中，从“ 提取”组中选择“ 样式 ”。或者，选择Refactor |提取|在选择的上下文菜单中提取样式 。

  3. 在打开的“ 提取Android样式”对话框中，指定要创建的样式的名称，并选择要包括为样式属性的标记属性。

  4. 要让IntelliJ IDEA通过使用对新创建的样式的引用替换标记属性来自动更新布局定义，请选中“在提取样式后启动可能的使用样式”复选框。

内联风格

此重构与Extract Style重构相反，并导致将样式中定义的所有属性添加到应用此样式的一个或所有组件。当您需要将父样式与其继承者合并时，也会使用此重构。

您可以从样式定义或应用样式的组件调用内联样式重构。在第一种情况下，IntelliJ IDEA删除样式定义并将相应的属性添加到使用它的所有组件。从组件调用重构时，可以将其应用于具有此样式的所有组件，或仅应用于当前组件。

要应用内联样式重构

    2. 打开的“ 内联Android样式”对话框仅提供默认选中的一个选项（ 内联所有引用并删除样式 ）。因此，IntelliJ IDEA将样式的属性转换为XML标记，将它们添加到使用样式的所有组件的定义中，并删除样式定义。

      • 要立即应用更改，请单击“ 重构” 。

      • 要在应用之前打开“ 查找”工具窗口并预览更改 ，请单击“ 预览” 。

    1. 选择具有相关样式的组件：

      • 在手动模式下 ，在Structure视图中选择具有此样式的组件，然后选择Refactor |主菜单或选择的上下文菜单中的内联样式 。

创建远程服务器配置

远程服务器是在另一台计算机（远程主机）上运行的服务器。可以通过FTP / SFTP / FTPS协议访问服务器上的文件。

要在此设置中配置对服务器的访问，您需要指定以下内容：

  1. 连接设置：服务器主机，端口和用户凭据。

  2. 服务器配置根文件夹和访问它的URL地址。

  3. 项目根文件夹 ，服务器上用于将数据从项目根文件夹复制到的文件夹 ，以及用于访问服务器上复制数据的URL地址之间的对应关系。这种对应关系称为映射 。

指定服务器配置的名称，类型和可见性

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在设置/首选项（Ctrl + Alt + S），选择下生成，执行，部署 部署 。

    • 选择工具|部署|在主菜单上配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加” 并根据您将用于与服务器交换数据的协议选择服务器配置类型。

    • FTP：选择此选项可让WebStorm通过FTP 文件传输协议访问服务器。

    • SFTP：选择此选项可让WebStorm通过SFTP文件传输协议访问服务器。

    • FTPS：选择此选项可让WebStorm通过SSL（ FTPS扩展）上的FTP文件传输协议访问服务器。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。将关闭“ 创建新服务器”对话框，并返回“ 部署”节点的“ 连接”选项卡。

创建远程服务器配置

远程服务器是在另一台计算机（远程主机）上运行的服务器。可以通过FTP / SFTP / FTPS协议访问服务器上的文件。

要在此设置中配置对服务器的访问，您需要指定以下内容：

  1. 连接设置：服务器主机，端口和用户凭据。

  2. 服务器配置根文件夹和访问它的URL地址。

  3. 项目根文件夹 ，服务器上用于将数据从项目根文件夹复制到的文件夹 ，以及用于访问服务器上复制数据的URL地址之间的对应关系。这种对应关系称为映射 。

指定服务器配置的名称，类型和可见性

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在设置/首选项（Ctrl + Alt + S），选择下生成，执行，部署 部署 。

    • 选择工具|部署|在主菜单上配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加” 并根据您将用于与服务器交换数据的协议选择服务器配置类型。

    • FTP：选择此选项可让PyCharm通过FTP 文件传输协议访问服务器。

    • SFTP：选择此选项可让PyCharm通过SFTP文件传输协议访问服务器。

    • FTPS：选择此选项可让PyCharm通过SSL（ FTPS扩展）上的FTP文件传输协议访问服务器。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。将关闭“ 创建新服务器”对话框，并返回“ 部署”节点的“ 连接”选项卡。

    单击“ 用作默认值”工具栏按钮让PyCharm在以下情况下静默应用当前配置：

    • 自动上传已更改的文件 。

    • 无需选择目标主机即可手动上载和下载文件。

    • 将本地文件和文件夹与其远程版本进行比较。

    • 编辑远程主机上的单个文件 。

  4. 仅对此项目使用“ 可见”复选框以配置服务器访问配置的可见性。

    • 选中该复选框以限制将配置用于当前项目。此类配置无法在当前项目之外重复使用。它不会出现在其他项目的可用配置列表中。例如，如果在SFTP配置中选中此复选框，则无法使用其SSH凭据来配置远程解释程序。

    • 清除该复选框后，所有PyCharm项目中都会显示该配置。其设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

指定在主机上注册期间定义的用户凭据

  1. 指定注册模式：

    • 要以常规模式登录，请在“ 用户名”字段中指定登录名 。

    • 要使用您的电子邮件地址作为密码启用对服务器的匿名访问 ，请选中“ 以匿名登录”复选框。

  2. 指定对服务器进行身份验证的方式：

    • 对于FTP服务器 ，键入密码并选择保存密码复选框以让PyCharm记住它。

    • 对于SFTP服务器 ，请选择对服务器进行身份验证的方式。执行以下操作之一：

      • 密码 ：使用密码访问主机。指定密码，然后选择保存密码复选框以将密码保存在PyCharm中。

      • 密钥对（OpenSSH或PuTTY） ：使用密钥对的SSH身份验证 。要应用此身份验证方法，您必须在客户端计算机上拥有私钥，并在远程服务器上拥有公钥。PyCharm支持使用OpenSSH实用程序生成的私钥。

        指定存储私钥的文件的路径，并在相应字段中键入密码（如果有）。要让PyCharm记住密码，请选中Save passphrase复选框。

      • OpenSSH配置和身份验证代理 ：使用由凭据帮助应用程序管理的SSH密钥（例如，Windows上的Pageant或Mac和Linux上的ssh-agent ）。

      有关使用SSH密钥的详细信息，请参阅生成新的SSH密钥并将其添加到ssh-agent教程。

    • 对于FTPS服务器 ，请指定您的用户名和密码。在“ 高级设置”区域中，选择要应用的安全机制。

      • 选择“ 显式”以应用显式（活动）安全性 。建立连接后，计算机上的FTP客户端立即向服务器发送命令，通过默认FTP端口建立安全控制连接。

        默认选择此方法。

      • 选择隐式以应用隐式（被动）安全性 。在这种情况下，在建立与服务器的连接时自动提供安全性，该服务器指定用于安全连接的单独端口。

        请注意，此方法被认为已弃用，因此不鼓励使用它。

启用与服务器的连接并指定服务器配置根目录

  1. 指定要与之交换数据的FTP / SFTP / FTPS服务器的主机名以及此服务器侦听的端口。

    默认值为：

    • 21用于FTP和FTPS

    • SFTP 22

  2. 在“ 根路径”字段中，指定在注册帐户时定义的相对于用户主 目录的服务器配置根目录 。此文件夹将是通过当前服务器配置可访问的文件夹结构中的最高文件夹。

    执行以下操作之一：

    • 接受默认/路径，该路径指向服务器上的用户主文件夹。

    • 手动键入路径。

IntelliJ IDEA项目中的文件和文件夹之间的对应关系及其在服务器上的版本是通过部署服务器映射设置的。为简单起见，IntelliJ IDEA项目中的任何文件或文件夹都称为本地 ，服务器上的任何文件或文件夹都称为远程 ，即使服务器实际安装在您的计算机上也是如此。有关详细信息，请参阅配置与Web服务器的同步 。

访问服务器

  1. 选择Tools |打开Remote Host工具窗口 部署|浏览远程主机或查看|工具窗口|主菜单中的远程主机 。

  2. 从列表中选择所需的部署服务器。工具窗口显示服务器根目录下的文件和文件夹的树视图。如果列表中没有可用的相关服务器，请单击 ，并在打开的“ 部署”对话框中配置对所需服务器的访问权限 。

将服务器上的文件和文件夹与其本地版本进行比较

每个远程文件或文件夹都映射到一个且只有一个本地文件或文件夹。因此，对于每个远程文件或文件夹，IntelliJ IDEA都会检测其本地版本，以便您随时进行比较。

  1. 打开“ 远程主机”工具窗口 （“ 工具”|“部署”|“浏览远程主机”或“ 查看”|“工具Windows”|“远程主机” ），然后从列表中选择所需的部署服

  3. 在打开的“ 差异查看器文件”对话框中，探索差异并在必要时使用它们应用它们按钮。有关详细信息，请参阅查看文件之间的差异 。
