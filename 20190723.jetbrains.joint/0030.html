初步步骤

准备ActionScript或Flex应用程序开发：

  1. 确保已启用Flash / Flex支持插件。该插件默认激活。如果插件已禁用，请在“ 插件”页面上启用它，如管理插件中所述 。

  2. 在您的计算机上下载并安装Flex或AIR SDK。IntelliJ IDEA提供与Apache Flex ， Adobe Flex ， Adobe AIR和Adobe Gaming SDK的集成。

  3. 在IntelliJ IDEA中注册Flex或AIR SDK 。（您可以将此步骤推迟到以后的某个时间。您可以在创建Flash模块时指定SDK。）

  4. 检查常规Flex编译器设置 ，如有必要，根据需要进行调整。

  5. 使用Flash模块创建新项目，或将Flash模块添加到现有项目。请参阅创建Flash模块 。

    请注意，您可以通过导入Adobe Flash Builder项目来创建项目或模块。

  6. 检查初始模块配置并进行必要的调整。这些可能包括配置模块内容 ， 添加库 ，修改现有构建配置以及创建其他构建配置等。

    但是，所有这些任务可以在需要时在稍后的时间执行，同时开发源代码 。

在IntelliJ IDEA中注册Flex或AIR SDK

在IntelliJ IDEA中注册Flex或AIR SDK

  1. 打开“ 项目结构”对话框（例如Ctrl + Shift + Alt + S ）。

  2. 在“ 平台设置”下 ，单击“ SDK” 。

  3. 单击“ 添加新SDK” 在工具栏上，然后单击 “ 添加新SDK”列表中的Flex / AIR SDK 。

  4. 在打开的对话框中 ，选择安装Flex或AIR SDK的目录，然后单击“ 确定” 。

    因此，SDK配置显示在“ 项目结构”对话框右侧的SDK页面上。

  5. 通常，您不需要对Classpath和Sourcepath选项卡进行任何更改，因为所有必需的库和源都已存在。但是，您可能需要添加外部在线文档，以便在编写代码时能够访问其他参考信息。要做到这一点：

    在“ 文档路径”选项卡上，单击“是” 。在打开的对话框中，单击“ 确定” 。 （IntelliJ IDEA建议的URL是您想要的URL。）

  6. 在Project Structure对话框中单击OK 。

配置常规Flex编译器设置

  1. 打开“ 设置/首选项”对话框（例如， Ctrl + Alt + S或在工具栏上）。

  2. 在对话框的左侧，在“ 构建，执行，部署”节点下，打开“ 编译器”节点，然后单击“ ActionScript和Flex编译器” 。

  3. 在“ 编译器”>“ActionScript和Flex编译器”页面上，根据需要配置设置。

移动并复制重构

Move重构允许您在项目中移动类，函数，模块，文件和目录。在这样做时，PyCharm跟踪这些移动并自动更正源代码中移动符号的所有引用。

可以使用以下Move重构：

  • 移动文件重构将文件移动到另一个目录。

  • Move Directory重构将目录移动到另一个目录。

  • Move Module Members重构移动Python模块的顶级符号。

  • Make本地函数/方法顶级重构将方法或本地函数转换为顶级函数并将其移动到指定文件。

将文件或目录移动到另一个目录

  4. 要根据重构结果更新对所选文件或文件夹的引用，请选中“ 搜索引用”复选框。

在新标签中显示查询结果

您可以选择在单个选项卡上查看查询结果，也可以在同一个选项卡上查看查询结果。

如果选中该复选框，则每次运行查询（ SELECT ）时都会打开带有查询结果的新选项卡。通过这种方式，您可以保留已运行的所有查询的结果。

在语句内部执行时

如果光标位于语句内，则可以使用以下选项：

  • 询问执行什么。 显示可以运行的语句列表，您可以选择一个或多个语句。

  • 最小的陈述。 执行最小的可能语句。例如，当光标位于子查询中时，将执行子查询。

  • 最大的声明。 执行最大可能的语句。例如，当光标位于子查询中时，将执行外部语句。

  • 最大的声明或批次。 对于Transact-SQL（SQL Server和Sybase），将执行当前批处理语句。对于所有其他方言 - 与上一个选项相同。

  • 整个脚本。 所有的陈述都被执行了。

否则执行

如果光标位于语句之外，例如在空行或注释中，则可以使用以下选项：

  • 没有。 没有任何陈述被执行。

  • 整个脚本。 所有的陈述都被执行了。

  • 插入物以下的一切。 执行光标位置后的所有语句。

选择执行

  • 智能扩展到脚本。 如果选择中至少有一个语句边框，则扩展选择以形成一系列有效语句。然后执行该序列。

描述

\将下一个字符标记为特殊字符或文字。例如：

+

匹配前面的字符一次或多次。例如，“zo +”匹配zoo但不匹配z 。

匹配前面的字符零或一次。例如， a?已经？ never匹配ve 。

匹配除换行符之外的任何单个字符。

(子表达式)

匹配子表达式并记住匹配。如果正则表达式的一部分括在括号中，则正则表达式的该部分将组合在一起。因此，可以将正则表达式运算符应用于整个组。

  • 如果需要在同一正则表达式中使用匹配的子字符串，则可以使用反向引用（ \num ，其中num = 1..n ）检索它。

  • 如果需要在当前正则表达式之外的某处引用匹配的子字符串（例如，在另一个正则表达式中作为替换字符串），则可以使用美元符号（ $num ，其中num = 1..n ）来检索它。

  • 如果需要将括号字符包含在子表达式中 ，请使用\(或\) 。

x | ÿ

匹配x或y 。例如， z|wood匹配z或wood 。 (z|w)oo匹配动物园或木材 。

{ n }

n是非负整数。正好匹配n次。例如， o{2}与Bob中的o不匹配，但匹配foooood中的前两个o。

{ n ，}

n是非负整数。匹配至少 n次。

例如， o{2,}与Bob中的o不匹配，并匹配“foooood”中的所有o。

o{1,}相当于o+ 。 o{0,}相当于o* 。

{ n , m }

m和n是非负整数。匹配至少 n次，最多 m次。例如， o{1,3}匹配“fooooood”中的前三个o。 o{0,1}相当于o? 。

[ xyz ]

一个字符集。匹配任何一个包含的字符。例如， [abc]匹配平原中的a 。

[^ xyz ]

负字符集。匹配任何未包含的字符。例如， [^abc]匹配普通的p 。

[ az ]

一系列人物。匹配指定范围内的任何字符。例如，“[az]”匹配范围a到z中的任何小写字母字符。

[^ mz ]

负范围字符。匹配不在指定范围内的任何字符。例如， [^mz]匹配不在m到z范围内的任何字符。

\ b

匹配单词边界，即单词和空格之间的位置。例如， er\b 从不但不是在动词 急诊室 急诊室相匹配。

匹配非数字字符。相当于[^0-9] 。

匹配换页符。

匹配换行符。

匹配回车符。

匹配任何空格，包括空格，制表符，换页等。等效于[ \f\n\r\t\v] 。

匹配制表符。

\ W

匹配任何非单词字符。相当于[^A-Za-z0-9_] 。

\ num

匹配num ，其中num是正整数，表示返回到记忆匹配的引用。

例如， (.)\1匹配两个连续的相同字符。

例如， \11和\011都匹配制表符。

\0011相当于\001 ＆ 1 。

八进制转义值不应超过256。如果他们这样做，只有前两个数字组成表达式。允许在正则表达式中使用ASCII代码。

例如， \x41匹配A \x041相当于\x04 ＆ 1 。

允许在正则表达式中使用ASCII代码。

调试JavaScript的可能性对Web开发人员至关重要。使用基于IntelliJ IDEA的产品，这种调试变得非常容易。为了说明PyCharm的JavaScript调试功能，我们将创建一个非常基本的脚本，只显示浏览器页面中的一些数字，然后在服务器上调试它。

如果在外部Web服务器上进行调试，则会在其上部署应用程序文件，并在计算机上保留其副本。无论Web服务器本身是在物理远程主机上还是在您的计算机上运行，部署在其上的应用程序文件都被视为远程服务器。

当打开加载某些JavaScript的远程HTML文件时，调试器会告诉PyCharm当前处理的文件的名称以及要处理的行的编号。PyCharm打开此文件的本地副本，并指示包含所提供号码的行。通过指定服务器上的文件和文件夹与其本地副本之间的对应关系来启用此行为。这种对应关系称为映射，它在调试配置中设置。
