编译类型

IntelliJ IDEA提出了几种编译和构建应用程序的方法。“ 构建”菜单中提供了相应的命令。

  • 编译<compilation_scope>。 编译指定范围内的所有源文件。这种情况下的范围可以是文件，包等。有关详细信息，请参阅编译目标一节。

  • 建设项目。 自上次编译以来修改的整个项目中的所有源文件都已编译。如果合适，还会编译相关的源文件。此外，还执行与修改源上的编译或构建过程相关的任务。例如，如果在“ 验证”页面上启用了相应的选项，则会执行EJB验证。有关详细信息，请参阅Building Project一节。

  • 构建模块。 递归编译是自所选模块中的上一次编译以及它依赖的所有模块中已经修改的所有源文件。有关详细信息，请参阅构建模块部分。

  • 重建项目。 重新编译项目中的所有源文件。当类路径条目发生更改时，这可能是必需的，例如，添加，删除或更改使用的SDK或库。有关详细信息，请参阅重建项目部分。

Grails Application Forge

IntelliJ IDEA允许您使用Grails Application Forge服务创建项目。Application Forge生成模板项目，并根据所选功能和配置文件自动设置所有必需的依赖项，而不是手动配置它们。它使用Gradle生成完整项目或构建文件。

使用Grails Applicaton Forge创建项目

  1. 如果当前没有项目打开，请单击欢迎屏幕上的“ 创建新项目 ”;否则选择文件|新的|项目 。

  2. 在打开的页面上 ，在左窗格中，选择Application Forge 。

  3. 在右窗格中，指定项目SDK（JDK）。如果已在IntelliJ IDEA中定义了必需的JDK，请从列表中选择它。否则，单击“ 新建”并从列表中选择“JDK”。然后，在打开的对话框中 ，选择所需JDK的安装文件夹（此时，必须已在您的计算机上安装相应的JDK）。

  4. 指定Grails SDK，其他Application Forge设置 ，然后单击“ 下一步” 。

  5. 在下一页上，指定项目信息，然后单击“ 完成” 。

IntelliJ IDEA将使用预定义结构，Gradle构建文件和一组库创建一个功能齐全的Grails项目。

用标签替换属性

使用标记重构替换属性允许您使用等效的子标记替换一组XML标记上的属性定义。如果调用此重构，则可以删除与具有所选标记名称的标记上的所选属性名称匹配的所有属性，并创建等效的子标记。随着XML模式的发展，将属性批量转换为标记非常有用。

将属性转换为标记

按预览打开重构预览窗口，显示要替换的属性的所有已找到的用法。在此窗口中，您可以排除/包含要重构的用法。

按Show XSLT ...打开XSLT预览窗口，显示相当于所请求的重构的XSLT的小片段。XSLT处理器可以使用此片段对WebStorm项目外部的文件执行请求的重构。

如果要受影响的文件是只读的，则无论如何都可能出现重构预览窗口。

  1. 在编辑器中，将光标放在要转换的属性中。

  2. 选择Refactor | XML重构|从主菜单或上下文菜单中将标记替换为标记 。

  3. 确定要替换所选属性的标记的名称。

  4. 确定更换的范围。如果选定标签上所选属性的名称与所选属性相匹配，则所有属性都将被替换。可用范围包括当前文件，整个项目或指定的目录或模块。根据是否选中“ 递归”复选框，目录范围可以包括子目录，也可以不包括子目录。如果选中“ 使用DTD限制文件：...”复选框，则范围将限制为与当前文件具有相同DOCTYPE的文件。

  5. 按“ 预览”按钮使WebStorm搜索所选属性“ 查找”窗口的用法。

  6. 单击“ 确定”继续。如果未选择“ 预览”选项，则会立即更改所有用法。

添加属性

添加属性重构允许您将属性添加到一组XML标记。如果调用此重构，则匹配所请求的属性名称和值的属性将添加到具有所选标记名称的标记中。可选地，可以仅将请求的属性添加到尚未包含所选属性的标签。随着XML模式的发展，这种大量的属性添加非常有用。

添加属性

单击“ 预览”将打开“ 重构预览”窗口，其中包含要替换的属性的所有已找到用法。在此窗口中，您可以排除/包含要重构的用法。

按Show XSLT ...打开XSLT预览窗口，显示相当于所请求的重构的XSLT的小片段。XSLT处理器可以使用此片段对WebStorm项目外部的文件执行请求的重构。

如果要受影响的文件是只读的，则无论如何都可能出现重构预览窗口。

  1. 在编辑器中，将光标放在相应的标记内。

  2. 选择Refactor | XML重构|从主菜单或上下文菜单中添加属性 。

  3. 确定要添加到所选标记的属性的名称。

  4. 确定要添加到所选标记的属性的值。

  5. 确定是否将忽略已包含所请求属性的标记，或者是否更改其值。

  6. 确定添加的范围。如果所选标签位于所选范围内，则将替换与所选标签匹配的所有标签。可用范围包括当前文件，整个项目或指定的目录或模块。根据是否选中“ 递归”复选框，目录范围可以包括子目录，也可以不包括子目录。如果选中“ 使用DTD限制文件：...”复选框，则范围将限制为与当前文件具有相同DOCTYPE的文件。

  7. 单击“ 预览”以使WebStorm搜索所选属性的用法。

  8. 单击“ 确定”继续。如果未选择“ 预览”选项，则会立即更改所有用法。

更改属性值

更改属性值重构允许您更改在一组XML标记上定义的属性的值。如果调用此重构，则与所选标记名称的标记上的所选属性名称和所选属性值匹配的所有属性的值都可能会更改。随着XML模式的发展，对属性值的批量修改可能很有用。

更改属性的值

按预览打开重构预览窗口，显示要更改的属性的所有已找到用法。在此窗口中，您可以排除/包含要重构的用法。

按Show XSLT ...打开XSLT预览窗口，显示相当于所请求的重构的XSLT的小片段。XSLT处理器可以使用此片段对WebStorm项目外部的文件执行请求的重构。

如果要受影响的文件是只读的，则无论如何都可能出现重构预览窗口。

  1. 在编辑器中，将光标放在要更改其值的属性中。

  2. 选择Refactor | XML重构|从主菜单或上下文菜单中更改属性值 。

  3. 确定属性的新值。

  4. 确定更改的范围。如果所选属性在所选范围内，则将修改与所选标记上所选属性的名称和值匹配的所有属性。可用范围包括当前文件，整个项目或指定的目录或模块。根据是否选中“ 递归”复选框，目录范围可以包括子目录，也可以不包括子目录。如果选中“ 使用DTD限制文件：...”复选框，则范围将限制为与当前文件具有相同DOCTYPE的文件。

  5. 按“ 预览”按钮使WebStorm搜索所选属性“ 查找”窗口的用法。

  6. Cstepck 可以继续。如果未选择“ 预览”选项，则会立即更改所有用法。

抑制资源压缩

默认情况下， Android资产包装工具（aapt）会在打包期间压缩资源。但是，您可以通过更改Android facet的设置来解压缩.apk文件中包含的某种类型的资源。

抑制压缩某种类型的资源

  1. 选择文件|主菜单中的项目结构 。

  2. 在左侧窗格中，选择Modules ，然后在中央窗格中单击模块下的Android facet，其中包含要解压缩的资源。

  3. 在右侧窗格中，切换到“ 编译器”选项卡。

  4. 在“ 附加命令行参数”文本框中，键入-0 <file extension for this type of resources> 。因此，具有指定扩展名的所有文件都将从压缩中排除。如果附加参数集不适合文本框，请单击并在打开的对话框中指定参数。

    例如，如果要以未压缩格式包含某种类型的资源，请键入-0 <file extension for this type of resources> 。

WebStorm 2019.1帮助

将内容转换为属性

将标记内容转换为属性重构允许您使用等效属性替换一组XML标记的内容。如果调用此重构，则与所选标记名称匹配的所有标记将删除其文本内容，并创建等效属性。随着XML模式的发展，将标记内容批量转换为属性非常有用。

将标记内容转换为属性

按预览将打开重构预览窗口，其中显示要转换的标记的所有已找到用法。在此窗口中，您可以排除/包含要重构的用法。

按Show XSLT ...打开XSLT预览窗口，显示相当于所请求的重构的XSLT的小片段。XSLT处理器可以使用此片段对WebStorm项目外部的文件执行请求的重构。

如果要受影响的文件是只读的，则无论如何都可能出现重构预览窗口。

  1. 在编辑器中，将光标放在要转换其内容的标记内。

  2. 选择Refactor | XML重构|从主菜单或上下文菜单中将标记内容转换为属性 。

  3. 根据所选标记内容确定要创建的属性的名称。

  4. 确定更换的范围。所有与所选名称相匹配的标签都会将其内容转换为属性（如果它们位于所选范围内）。可用范围包括当前文件，整个项目或指定的目录或模块。根据是否选中“ 递归”复选框，目录范围可以包括子目录，也可以不包括子目录。如果选中“ 使用DTD限制文件：...”复选框，则范围将限制为与当前文件具有相同DOCTYPE的文件。

  5. 按“ 预览”按钮使WebStorm搜索所选标签“ 查找”窗口的用法。

  6. 单击“ 确定”继续。如果未选择“ 预览”选项，则会立即更改所有用法。
