空白

使用此列表可以定义差异查看器应如何处理空格。

  • 不要忽略 ：空格很重要，所有差异都会突出显示。默认情况下选择此选项。

  • 修剪空格 :( ("\t", " ") ，如果它们出现在行的末尾和行的开头。

    • 如果两行仅在尾随空格中不同，则这些行被认为是相等的。

    • 如果两行不同，则在“ 按字”模式下不会突出显示此类尾随空格。

  • 忽略空格 ：白色空格并不重要，无论它们在源代码中的位置如何。

  • 忽略空格和空行 ：忽略以下实体：

    • 所有空格（如'忽略空白'选项）

    • 所有添加或删除的行仅包含空格

    • 所有更改包括拆分或连接线而不更改非空白部分。
      例如，在此模式下不会突出显示将abc更改为a \nbc 。

突出显示模式

选择突出显示差异粒度的方式。

可用选项包括：

  • 突出显示单词 ： 突出显示已修改的单词

  • 突出显示线条 ：修改后的线条突出显示

  • 突出显示拆分更改 ：如果选择此选项，则会将较大的更改拆分为较小的“原子”更改。
    例如， A \n B与AX \n BX将被视为两个更改而不是一个。

  • 突出显示符号 ：将突出显示已修改的符号

  • 不要突出显示 ：如果选择此选项，则不会突出显示差异。此选项适用于显着修改的文件，其中突出显示仅引入其他困难。

描述

节点解释器

在此字段中，指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

使用Spy-js，建议使用Node.js版本0.10.24或更高版本。

跟踪服务器端口

在此旋转框中，指定Spy-js将充当代理服务器的端口。此端口号必须与系统代理端口相同。如果选中“ 自动配置系统代理”复选框，则会自动为系统代理服务器设置指定的端口号。否则，您必须手动指定系统代理设置中的字段值。

跟踪服务器端口自动填充。为避免端口冲突，建议您接受建议的值并选中“ 自动配置系统代理”复选框。

使用

在此下拉列表中，指定配置跟踪会话的方式。

  • 要让Spy-js应用其内部预定义配置，请选择“ 默认配置” 。

  • 要应用自定义手动创建的配置，请选择“ 配置文件”选项，然后在下面的“ 配置”字段中指定自定义配置文件的位置。

    配置文件是扩展名为.js或.conf.js的JavaScript文件，其中包含符合Spy-js配置要求的有效JavaScript代码。如果WebStorm在项目中检测到扩展名为.conf.js的文件，则这些文件将显示在下拉列表中。

    手动键入配置文件的路径，或单击“ 浏览” 并在打开的对话框中选择位置。指定后，配置文件将添加到下拉列表中，以便下次从列表中获取而不是指定路径。

要跟踪的URL

在此字段中，指定要捕获事件的网页的URL地址。默认情况下，该字段为空。这意味着Spy-js捕获所有当前打开的Web页面上的事件。如果要将跟踪限制为某个页面，请指定其URL地址。手动键入地址或从下拉列表中选择地址（如果已在配置中指定了地址）。

自动配置系统代理

  • 选中此复选框后，将使用“ 跟踪服务器端口”字段中指定的端口自动激活系统代理服务器。

  • 清除此复选框以手动指定代理设置。了解如何在Windows ， Mac ， Ubuntu ， iOS ， Android ， Windows Phone上手动配置代理设置。请注意，某些桌面浏览器有自己的代理设置配置屏幕。

默认情况下选中该复选框，强烈建议您接受此设置并自动配置代理。

  3. 输入连接细节。请注意，设置因问题跟踪器而异。

    通常，您必须指定服务器URL和连接凭据。

    选择“ 共享URL”选项以允许您的团队中的其他成员访问服务器。

    如果要通过代理服务器访问服务器，请单击“ 代理设置” 。您可以在HTTP代理部分中找到有关代理设置的更多信息。

    （ 可选 ）启用为更改列表添加提交消息并配置消息模板。

    对于某些跟踪器（例如，对于不支持开箱即用的跟踪器），您还必须配置服务器参数。指定用于访问任务的URL和请求类型，选择跟踪服务器响应的格式 - 用于XPath的XML，用于JSONPath的 JSON或用于正则表达式的文本。您还可以使用选择器表来指定要从服务器获取的任务的详细信息。例如，这可以是创建任务的日期或其URL。请注意，您可以在此窗口中使用代码完成。

如果您使用“慢”问题跟踪系统，则特别推荐同步。

    • 功能分支名称格式 - 当您创建或打开新任务时，PyCharm将提示您创建新功能分支。在此字段中，您可以配置模板以生成新功能分支的名称。使用{id} ， {summary} ， {number} ， {project}占位符。

我们都知道，互联网服务提供商有一种超越他们联系的习惯，这有时会导致我们的联系不如我们所希望的那么好。此外，我们中的许多人都在使用Raspberry Pi来等待很酷的项目。所以，让我们使用Pi来检查我们的互联网连接！

互联网连接的关键指标之一是它与互联网上其他主机的ping时间。因此，让我们编写一个程序，定期ping其他主机并记录结果。

对于那些想要在其他远程计算机上远程执行代码的人，例如AWS实例或DigitalOcean Droplet，过程完全相同。

本教程是在Windows上创建的，代码是在RPi上执行的，RPi是一台Linux计算机。在其他受支持的操作系统上，某些路径和工作流程可能略有不同。

File Watcher是一个内置的PyCharm工具，允许您在IDE中更改或保存文件时自动运行命令行工具，如编译器，格式化程序或短接程序。

您可以从头开始配置文件监视器或使用其中一个可用模板。Configured File Watcher可以保存在项目设置或IDE设置中，并在不同的项目中使用。

配置输入文件的预期类型和位置

空白

使用此列表可以定义差异查看器应如何处理空格。

  • 不要忽略 ：空格很重要，所有差异都会突出显示。默认情况下选择此选项。

  • 修剪空格 :( ("\t", " ") ，如果它们出现在行的末尾和行的开头。

    • 如果两行仅在尾随空格中不同，则这些行被认为是相等的。

    • 如果两行不同，则在“ 按字”模式下不会突出显示此类尾随空格。

  • 忽略空格 ：白色空格并不重要，无论它们在源代码中的位置如何。

  • 忽略空格和空行 ：忽略以下实体：

    • 所有空格（如'忽略空白'选项）

    • 所有添加或删除的行仅包含空格

    • 所有更改包括拆分或连接线而不更改非空白部分。
      例如，在此模式下不会突出显示将abc更改为a \nbc 。

  • 忽略导入和格式化 ： 忽略 import语句和空格内的更改（尽管可以遵守String文字中的空格）。

突出显示模式

选择突出显示差异粒度的方式。

可用选项包括：

  • 突出显示单词 ： 突出显示已修改的单词

  • 突出显示线条 ：修改后的线条突出显示

  • 突出显示拆分更改 ：如果选择此选项，则会将较大的更改拆分为较小的“原子”更改。
    例如， A \n B与AX \n BX将被视为两个更改而不是一个。

  • 突出显示符号 ：将突出显示已修改的符号

  • 不要突出显示 ：如果选择此选项，则不会突出显示差异。此选项适用于显着修改的文件，其中突出显示仅引入其他困难。

流星

此功能仅在Ultimate版本中受支持。

Meteor是一个全栈框架，允许您在客户端和服务器端使用JavaScript。IntelliJ IDEA与Meteor集成，因此您可以在IDE内部使用它。 IntelliJ IDEA中的流星支持包括：

  • 通过检测.meteor文件夹并从项目中排除.meteor/local文件夹来自动识别Meteor项目。有关详细信息，请参阅隐藏排除文件

  • 将预定义的Meteor库自动附加到项目。这样可以启用语法突出显示，解析引用和代码完成。

  • 通过把手支持Spacebars，完成if和each指令。IntelliJ IDEA识别Spacebars模板，但作为副作用标记Meteor项目中的HTML文件 。IntelliJ IDEA提供JavaScript源代码和模板之间的导航， 转到声明 （ Ctrl + B ）。

  • 用于在一个调试会话中调试客户端和服务器端代码的专用复杂Meteor运行/调试配置，请参阅调试Meteor应用程序 。

在你开始之前

按照管理插件中的说明在插件页面上安装Meteor和Handlebars / Moustache存储库插件 。

需要macOS 10.6或更高版本。

安装过程取决于您使用的操作系统。从Meteor官方网站了解更多信息。

  • 在Meteor官方网站下载LaunchMeteor.exe安装程序。

在macOS或Linux上安装Meteor

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    $ curl https://install.meteor.com | / bin / sh的

创建一个新的Meteor应用程序

如果您还没有应用程序，可以从Meteor样板模板生成具有Meteor特定结构的IntelliJ IDEA项目。或者，创建一个空的IntelliJ IDEA项目并在其中配置Meteor支持，如下面的开始使用现有的Meteor应用程序中所述。

从样板模板创建Meteor项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，选择Meteor App ，然后单击Next 。

  4. 在向导的第二页上：

    1. 指定项目名称和要在其中创建的文件夹。

    2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

    3. 从“ 模板”列表中，选择要生成的样本。要生成基本项目结构，请选择“ 默认”选项。

创建一个空的IntelliJ IDEA项目

  1. 选择文件|新的|从主菜单中选择项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 项目类别和选项”对话框（“ 新建项目”向导的第一页）中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，再次选择Static Web ，然后单击Next 。

  4. 在向导的第二页上，指定项目文件夹和名称，然后单击“ 完成” 。

从现有的Meteor应用程序开始

如果您要继续开发现有的Meteor应用程序，请在IntelliJ IDEA中打开它，在其中配置Meteor，并按照下面的下载Meteor依赖项中所述下载所需的依赖项。

从版本控制中检出应用程序源

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以检出应用程序源。

在现有项目中配置Meteor支持

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |流星 。Meteor页面打开。

  2. 指定Meteor可执行文件的路径。如果您遵循标准安装过程，IntelliJ IDEA会自动检测文件。

  3. 要将.meteor/local文件夹及其内容包含在索引中，请清除“打开项目时自动排除”.meteor / local“目录复选框。有关详细信息，请参阅下面隐藏排除的文件 。

  4. 确保选中“ 自动导入Meteor软件包作为外部库”复选框。

    • 选中该复选框后，IntelliJ IDEA会自动从meteor/packages文件中导入外部包。因此，IntelliJ IDEA提供全范围编码帮助：解析对Meteor内置函数的引用，例如check(true) ，以及来自第三方软件包的函数，提供正确的语法和错误突出显示，支持使用源映射进行调试， 等等。

    • 清除此复选框后，IntelliJ IDEA不会自动从meteor/packages文件导入外部包。结果，没有提供编码辅助。要改善这种情况，请在编辑器中打开meteor/packages文件，然后单击Import packages as library链接或运行meteor --update命令。

  5. 确保IntelliJ IDEA已将Meteor库附加到项目中。

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript |图书馆 。

    2. 关于JavaScript。打开的库页面，确保选中“ 库”列表中Meteor项目库旁边的复选框。

运行Meteor应用程序

从技术上讲，IntelliJ IDEA为服务器端和客户端代码创建单独的运行配置，但您可以在一个专用的Meteor运行配置中指定所有设置。

IntelliJ IDEA根据Meteor类型的运行配置运行Meteor应用程序。如果您从样板模板创建了应用程序，IntelliJ IDEA会为您生成Meteor运行配置。

创建Meteor运行配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择Meteor 。运行/调试配置：Meteor打开。

  2. 在“ 配置”选项卡中，根据安装指定Meteor可执行文件的路径（请参阅安装Meteor ）。

  3. 指定存储要运行的应用程序文件的文件夹。此文件夹必须在根目录中有一个.meteor子文件夹，以便IntelliJ IDEA将您的应用程序识别为Meteor项目 。

    默认情况下，工作目录是项目根文件夹。

可选

  1. 在“ 程序参数”字段中，指定命令行在启动时传递给可执行文件的其他参数（如果适用）。这些可以是，例如， --dev ， --test ，或--prod以指示应用程序正在运行（ 开发 ， 测试或生产环境中），以便不同的资源被加载在启动时的环境。

  2. 默认情况下，IntelliJ IDEA在“ 运行”工具窗口中显示应用程序输出。要查看客户端代码执行的结果，请在“ 浏览器/实时编辑”选项卡中选择“ 启动后”复选框，然后从列表中选择要打开的浏览器。在下面的字段中，指定用于打开应用程序的URL地址。默认值为http://localhost:3000 。

运行Meteor应用程序

调试Meteor应用程序

调试器还会在/packages文件夹中存储的源中设置的断点处暂停。客户端和服务器端代码都支持此功能。

使用IntelliJ IDEA，您可以在一个调试会话中调试Meteor JavaScript代码的客户端和服务器端 。仅通过专用的Meteor运行配置启动调试会话。

从技术上讲，实现不同应用程序的几个Meteor项目可以组合在一个IntelliJ IDEA项目中。要独立运行和调试这些应用程序，请使用相关工作目录为每个应用程序创建单独的运行配置。为避免端口冲突，这些运行配置应使用不同的端口。在“ 程序参数”字段中，为每个运行配置指定一个单独的端口，格式为--port=<port_number> 。

调试Meteor应用程序

  1. 必要时在代码中设置断点 。

  2. 如上所述创建Meteor运行/调试配置。在“ 浏览/实时编辑”选项卡中，选择“ 启动后”复选框，从列表中选择“ Chrome ”，然后选中“ 使用JavaScript调试器”复选框。

  3. 要启动调试会话，请从主工具栏上的列表中选择所需的调试配置，然后单击在列表旁边或选择Run |从主菜单中调试<配置名称> 。

    调试工具窗口打开，显示两个选项卡：一个用于调试标记为的服务器端代码另一个用于调试标有的客户端代码 。

  4. 浏览暂停的程序并逐步完成该程序 。

默认情况下，WebStorm根据ESLint配置中的严重性级别标记检测到的问题。请参阅配置ESLint突出显示以了解如何覆盖这些设置。

在开始之前，请确保您的计算机上有Node.js.

WebStorm与集成ESLint带来广泛的掉毛其也可以使用插件来扩展规则。当您键入时，WebStorm会在编辑器中显示ESLint报告的警告和错误。使用ESLint，您还可以使用JavaScript标准样式 。

要查看问题的描述，请将鼠标悬停在突出显示的代码上。

安装ESLint

  1. 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install eslint -g用于全局安装。

    • npm install eslint --save-dev将ESLint安装为开发依赖项。

  2. （可选）安装其他插件，例如eslint-plugin-react to lint React应用程序。

在WebStorm中激活和配置ESLint

了解有关从ESLint官方网站配置ESLint的更多信息。

从ESLint官方网站了解有关ESLint CLI选项的更多信息。

默认情况下，WebStorm使用项目node_modules文件夹中的ESLint包和存储当前文件的文件夹中的.eslint。*配置文件。如果在当前文件文件夹中找不到.eslint。* ，WebStorm将在其父文件夹中查找一个直到项目根目录。

如果您有几个将ESLint列为依赖项的package.json文件，则WebStorm会为每个package.json启动一个单独的进程并处理其下的所有内容。这确保了使用monorepos和具有多个ESLint配置的项目的正确工作。

在所有新的WebStorm项目中，此行为是默认的。要在先前创建的项目中启用它，请转到语言和框架| JavaScript | 设置/首选项对话框中的ESLint （ Ctrl + Alt + S ），然后选择自动ESLint配置选项。

您也可以手动配置ESLint以使用自定义ESLint包和配置。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript | ESLint 。

  2. 选择“ 手动配置”选项以使用自定义ESLint包和配置。

  3. 在Node Interpreter字段中，指定Node.js的路径。如果您遵循标准安装过程，WebStorm会检测路径并填写字段本身。

  4. 在ESLint Package字段中，指定eslint或standard包的位置。

  5. 选择要使用的配置。

    • 使用自动搜索 ，WebStorm会查找.eslintrc文件或尝试检测package.json中 eslintConfig下定义的配置。WebStorm首先在文件夹中查找带有要打印文件的.eslintrc或package.json ，然后在其父文件夹中查找，依此类推到项目根目录。

    • 选择“ 配置文件”以使用自定义文件，并在“ 路径”字段中指定文件位置。

  6. 可选：

    • 在Extra ESLint Options字段中，指定运行ESLint的其他命令行选项，使用空格作为分隔符。

    • 在“ 附加规则目录”字段中，使用其他代码验证规则指定文件的位置。这些规则将在.eslintrc或上面指定的自定义配置文件的规则之后应用，因此将覆盖它们。

  • 在.eslintrc或下eslintConfig中的package.json，找到要修改其ID设置为规则1 （ warn ）或2 （ error ）。从ESLint官方网站了解更多信息。

您可以从ESLint配置中覆盖严重性，以便WebStorm忽略它们并将linter报告的所有内容显示为错误，警告或自定义颜色。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 编辑器”|检查 。将打开“ 检查”页面 。

  3. 在右侧窗格中，清除“ 从配置文件中使用规则严重性”复选框，然后选择要使用的严重性级别而不是默认级别。

从ESLint导入代码样式

您可以将某些ESLint代码样式规则导入WebStorm JavaScript代码样式设置 。这使WebStorm能够在自动完成，生成或重构代码或添加import语句时为项目使用更准确的代码样式选项。当您使用重新格式化操作时，WebStorm将不再从ESLint透视图中破坏格式正确的代码。

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。
