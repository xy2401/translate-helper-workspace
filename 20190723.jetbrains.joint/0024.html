附加和分离Perforce作业到变更列表

Perforce与IntelliJ IDEA的集成为您提供了一个用户界面，用于将Perforce 作业附加到更改列表并将其分离。

请注意，集成不支持创建Perforce作业。

要访问使用Perforce作业，请在“设置”对话框的“ Perforce”页面上选中“ 启用Perforce作业支持”复选框。

从这个主题，您将学习如何：

  • 将作业附加到更改列表：

    • 在 “ 本地”选项卡的任何工作阶段 。

    • 在 “ 提交更改”对话框的提交 期间 。

  • 使用以下命令找到要附加到更改列表的所需作业：

    • 标准搜索功能，使您可以指定多个搜索条件，从而灵活配置搜索过程。

    • 快速搜索功能，使您可以自动将找到的作业链接到更改列表。

  • 从变更列表中分离作业。

当您只需要将一个作业附加到更改列表并且您知道所需作业的确切名称或者至少可以指定名称的搜索模式时， 快速搜索很有用。

在工作的任何阶段查找和链接工作

  1. 打开“ 版本控制”工具窗口的“ 本地”选项卡。

  2. 选择要将作业链接到的更改列表。

  3. 从更改列表的上下文菜单中，选择“ 编辑关联作业” 。

  4. 找到想要的工作。执行以下操作之一：

    • 要使用标准搜索功能，请单击按钮。

    • 要使用快速搜索功能，请单击按钮。

  5. 查看找到的作业的详细信息，然后单击“ 确定” 。

在提交期间查找和链接作业

  1. 在“ 版本控制”工具窗口的“ 本地更改”选项卡中，选择要将作业链接到的更改列表，然后打开“ 提交更改”对话框。

  2. 使用“ 作业”区域中的控件查找所需的作业。执行以下操作之一：

    • 要使用标准搜索功能，请单击按钮。

    • 要使用快速搜索功能，请单击按钮。

  3. 查看找到的作业的详细信息并继续提交过程。

使用标准搜索功能查找作业

  1. 在打开的对话框中或“ 提交更改”对话框的“ 作业”区域中，单击按钮。

    您当前所在的对话框取决于您是在提交期间还是在任何其他工作阶段链接作业。

  2. 在打开的对话框中，指定所需的搜索参数。

    至少应填写其中一个字段。

  3. 单击“ 搜索”按钮。符合指定条件的作业将显示在“ 搜索结果”列表中。要查看作业的详细信息，请在列表中选择它。

  4. 选择所需的作业，然后单击“ 确定”对话框关闭，您将返回到开始搜索的对话框：

    • 如果在提交期间链接作业，则“ 提交更改”对话框。

    • 如果要在任何其他阶段工作中链接作业，则“ 编辑作业链接到更改列表”对话框。

快速查找和链接一项工作

  1. 打开“ 编辑作业链接到 更改列表”对话框或切换到“ 提交更改”对话框的“ 作业”区域。

    要使用的对话框取决于您是在提交期间还是在任何其他工作阶段链接作业。

  2. 在文本框中，键入所需的作业名称搜索模式，然后单击按钮。找到该作业并自动链接到当前的更改列表。

    如果未找到与指定模式匹配的作业，则显示相应的信息消息。

    通过快速搜索功能找到并链接的作业的详细信息仅在“ 编辑作业链接到更改列表”对话框中可用。

从更改列表中分离作业

  • 在“ 编辑作业链接到 更改列表”对话框或“ 提交更改”对话框的“ 作业”区域中，选择所需的作业，然后单击按钮。

    要使用的对话框取决于您是在提交期间还是在任何其他工作阶段分离作业。

当您在团队中工作时，当有人将更改推送到您当前正在处理的文件时，您可能会遇到这种情况。如果这些更改不重叠（即对不同的代码行进行了更改），则会自动合并冲突的文件。但是，如果相同的行受到影响，Git不能随意选择一侧而是另一侧，并要求您解决冲突。

在Git中，当您尝试执行以下操作之一时可能会出现冲突： pull ， merge ， rebase ， cherry-pick ， unstash changes或apply patch 。如果存在冲突，则这些操作将失败，系统将提示您接受上游版本，更喜欢您的版本或合并更改：

使用库项目共享Android源代码和资源

IntelliJ IDEA支持包含共享Android源代码和资源的Android 库项目 。其他Android应用程序项目可以引用库项目，并在构建时将其编译的源包含在其.apk文件中。

在IntelliJ IDEA中，库项目通过单独的库模块得到支持。

要启用共享Android源代码和资源，请执行以下操作之一：

  • 创建一个新的库模块。

  • 将包含要共享的源的应用程序模块转换为库模块。

在本主题中：

  • 创建库模块
  • 将应用程序模块转换为库模块
  • 在另一个项目中使用库模块
  • 将AndroidManifest.xml中的库模块数据添加到整个应用程序的AndroidManifest.xml中
  • 将库模块的.dex文件包含到整个应用程序的.apk中而不重建（pre-dexing）

创建库模块

  1. 执行以下操作之一：

    • 从头开始创建项目

    • 将模块添加到现有项目

  2. 在向导的第一页上，在左窗格中选择Android ，在右窗格中选择Library Module 。

  3. 完成向导。

将应用程序模块转换为库模块

您可以通过更新其Android facet将Application模块转换为Library模块。

  1. 选择File |打开Project Structure对话框主菜单中的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开要转换为“库”模块的模块的节点，然后单击“ Android” 。

  3. 在右侧窗格中，选择Android facet页面顶部的Library module复选框。

在另一个项目中使用库模块

要在另一个项目中使用库模块，您需要将此模块导入其中。

  1. 从主菜单中，选择File |新的|来自现有来源的模块 。

  2. 在打开的对话框中 ，浏览到要导入的.iml模块文件，然后单击“ 确定” 。模块节点将添加到树视图中。

  3. 将导入的库模块的依赖项添加到将要使用其数据的模块：

    1. 打开非库模块的设置：导航到File |项目结构 ，在左窗格中选择模块 ，并选择在中央窗格中的非库模块。

    2. 在右侧窗格中，切换到“ 依赖关系”选项卡。单击“ 添加”按钮在右侧工具栏中，从上下文菜单中选择Module Dependency 。

    3. 在打开的“选择模块”对话框中，从列表中选择导入的库模块，然后单击“ 确定” 。

将AndroidManifest.xml中的库模块数据添加到整个应用程序的AndroidManifest.xml中

要成功将库模块集成到另一个应用程序中，必须在应用程序AndroidManifest.xml文件中声明其组件。清单文件包含运行应用程序所需的信息（有关更多信息，请参阅App Manifest ）。您可以手动在库模块上添加此信息，也可以从库模块的AndroidManifest.xml中提取该信息，并自动将其添加到应用程序的AndroidManifest.xml中。

第二种方法称为合并清单 。

要使库模块的清单自动与应用程序清单合并：

  1. 选择File |打开Project Structure对话框主菜单中的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开非库模块，然后单击其节点下的Android facet。

  3. 在右侧窗格的Android facet页面上 ，切换到Packaging选项卡，然后选择Enable manigest mergeging选项。

将库模块的.dex文件包含到整个应用程序的.apk中而不重建（pre-dexing）

在应用程序打包期间，库模块的.class文件将转换为.dex文件。此操作称为dexing 。最后，库模块输出的.dex文件包含在最终应用程序.apk （了解有关构建和运行的构建过程的更多信息）。

通常，库模块的内容保持不变。在这种情况下，您只能将它们设置为dexed ，然后输出.dex文件包含在.apk 。这种方法被称为预解除 。

默认情况下，IntelliJ IDEA预先设定了库模式依赖项以及自上次构建以来尚未更新的外部jars 。您可以更改这些设置，以便始终将所有.class文件设置为dexed。

  1. 选择File |打开Project Structure对话框主菜单中的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开非库模块，然后单击其节点下的Android facet。

  3. 在右侧窗格的Android facet页面上 ，切换到Packaging选项卡，然后选择Pre-dex外部jar和Android库依赖项选项。

与远程存储库同步

在通过将更改推送到上游来共享工作结果之前，需要与远程存储库同步以确保项目的本地副本是最新的。您可以通过以下方式之一执行此操作： 获取更改 ， 拉 取 更改 ， 更新项目 。

获取更改

从上游获取更改时，自上次与远程存储库同步后提交的所有新数据都将下载到本地副本中。此新数据未集成到本地文件中，更改不会应用于您的代码。

提取的更改将存储为远程分支，这使您有机会在将它们与文件合并之前对其进行检查。由于fetch不会影响您的本地开发环境，因此这是一种更新远程存储库所有更改的安全方法。

要获取更改，请从主菜单中选择VCS | Git |抓取 。

拉变化

从远程仓库拉变化为方便快捷方式获取并随后合并的变化。当您拉动时 ，您不仅可以下载新数据，还可以将其集成到项目的本地工作副本中。

要进行更改，请执行以下操作：

  1. 从主菜单中，选择VCS | Git |拉 。“ 拉动更改”对话框打开。

  2. 如果您的项目有多个Git存储库，请从Git Root下拉列表中选择要更新的本地存储库的路径。

  3. 如果为存储库配置了多个远程数据库，请在“ 远程”下拉列表中选择要从中提取数据的远程数据库的URL。

  4. 选择要从中获取更改的分支，然后合并到当前签出的分支中。

  5. 从“ 策略”下拉列表中，选择将用于解决合并期间发生的冲突的合并策略 。

  6. 如有必要，请选择以下内容

    • 无提交 ：如果您不希望PyCharm自动提交合并结果，请选择此选项。在这种情况下，您可以检查它们并在必要时进行调整。

    • 无快进 ：选择此选项可生成合并提交，即使合并已解析为快进（即仅更新了分支指针）。

    • Squash commit ：选择此选项可在当前分支之上创建单个提交，而不是合并一个或多个分支。它产生工作树和索引状态，就好像真正的合并发生一样，但它实际上并没有提交或移动HEAD。

    • 添加日志信息 ：如果希望PyCharm使用除分支名称之外的正在合并的实际提交中的单行描述来填充日志消息，请选择此选项。

  7. 单击“ 拉”以从所选远程存储库中获取并应用更改。

已预先选择已跟踪的远程分支。

更新您的项目

如果您有多个项目根，或者每次与远程存储库同步时想要从所有分支获取更改，您可能会发现更新项目是一个更方便的选项。

执行更新操作时，PyCharm从所有项目根和分支中获取更改，并将跟踪的远程分支合并到本地工作副本（相当于pull ）。

如果您的项目包含子模块 ，并且它们位于分支上，则它们也将自动更新。

如果子模块处于分离的HEAD状态，PyCharm将调用git submodule update ，这将检查根存储库中引用的提交。这意味着仅当子模块中的子模块引用更改或添加了新的子模块时才会执行更新。

要更新项目，请执行以下操作：

  1. 从主菜单中，选择VCS |更新项目或按Ctrl + T. “ 更新项目”对话框打开。

  2. 选择更新类型（此策略将应用于Git版本控制下的所有根目录）：

    • 合并 ：选择此选项可在更新期间执行合并 。这相当于运行git fetch然后运行git fetch git merge或git pull --no-rebase 。

    • Rebase ：选择此选项可在更新期间执行rebase 。这相当于运行git fetch然后运行git fetch git rebase ，或者git pull --rebase （所有本地提交都将放在更新的上游头之上）。

    • 分支默认值 ：如果要为不同分支应用不同的更新策略，请选择此选项。您可以为.git/config配置文件的branch.<name>部分中的每个分支指定默认更新类型。

  3. 指定在更新之前清理工作副本时用于保存更改的方法，以便在更新完成后还原未提交的更改：

    • 使用Stash ：选择此选项可在git存储中保存本地更改。如果你需要在PyCharm之外应用带有隐藏更改的补丁，这很有用，因为它们是由Git本身生成的。

    • 使用Shelve ：选择此选项可将本地更改放入工具架 。搁架由PyCharm完成，搁置变化产生的补丁通常在PyCharm内部应用。

如果您以后选择不显示“ 更新项目”对话框，然后想要稍后修改默认更新策略，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中选择“ 版本控制”|“控制”。确认 ，在调用这些命令时选择“ 显示选项”对话框下的“更新”，并在下次执行更新时修改更新策略。

配置模块化应用程序的依赖项

此功能仅在Ultimate版本中受支持。

有两种方法可以为模块化应用程序配置依赖项。（模块化应用程序包含可动态加载的模块，请参阅Flex文档中的模块化应用程序概述 。）

一种方法是在主应用程序（也就是shell）的构建配置中列出运行时加载模块（RLM）的主类。这种方式不需要为模块创建构建配置。

另一种方法是为每个RLM创建构建配置，然后在主应用程序的构建配置中，指定对这些构建配置的依赖关系。

下面讨论这两种方法及其优点和缺点。

  • 通过列出主RLM类来指定依赖项
  • 通过列出RLM构建配置来指定依赖项

通过列出主RLM类来指定依赖项

如果主应用程序的源代码及其运行时加载的模块位于同一IntelliJ IDEA模块中，则只需在主应用程序的构建配置中列出主RLM类。此外，如果要针对主应用程序优化模块（这大大减少了已编译模块文件的大小），可以通过在UI中打开相应选项来实现。

请注意，这种指定依赖项的方法适用于Web和桌面应用程序，但不适用于移动应用程序。

好处：

  • 您无需为模块（RLM）创建构建配置。

  • 要使用load-externs和link-report编译器选项，您无需手动指定它们，UI中提供了一个复选框，用于打开或关闭模块优化。

缺点：

  • 同一组编译器选项用于应用程序和模块。

  • 主应用程序和模块的源代码必须位于同一IntelliJ IDEA模块中。

以下是要使用的过程的主要步骤：

  1. 打开主应用程序的构建配置设置。有关说明，请参阅管理构建配置及其设置，请按照下列步骤操作，并使用快捷方式打开构建配置设置，请按照下列步骤操作 。

  2. 在“ 常规”选项卡上，单击“ 运行时加载的模块”字段右侧 。（或者，单击该字段并按Shift + Enter 。）

  3. 在打开的“ 运行时加载的模块”对话框中，单击 （ Alt + Insert ）。

  4. 在打开的“选择运行时加载的主类模块”对话框中，选择相应RLM的主类，然后单击“ 确定” 。

  5. 如果要优化模块SWF文件大小，请选中“ 优化”复选框。

  6. 以类似的方式，在其他RLM上添加依赖项。

  7. 在Runtime-Loaded Modules对话框中单击OK 。

  8. 在Project Structure对话框中单击OK 。

通过列出RLM构建配置来指定依赖项

使用这种指定依赖关系的方式时，您应该为每个RLM建立配置。此外，要优化RLM的文件大小，您应手动指定load-externs和link-report编译器选项（在相应的构建配置中）。

好处：

  • 可以使用不同的编译选项集编译主应用程序和模块。（每个构建配置都有自己的一组编译器选项。）

  • 主应用程序和RLM的源代码可以在同一IntelliJ IDEA模块中，也可以在不同模块中（在同一项目中）。

WebStorm包含预定义的键盘映射，可让您自定义常用的快捷键。

WebStorm会根据您的环境自动选择预定义的键映射。确保它与您正在使用的操作系统匹配，或者选择与您习惯使用的另一个IDE（例如，Eclipse或NetBeans）匹配的快捷方式。

您无法自定义预定义的键盘图。修改任何快捷方式时，WebStorm会创建当前所选键映射的副本，您可以对其进行配置。点击复制选定的键映射，重命名，删除或将其还原为默认值。

自定义键映射不是其父键映射的完整副本。它从父键映射继承未修改的快捷方式，并仅定义已更改的快捷方式。有关键映射文件的信息，请参阅用户定义的键映射的位置 。

使用正则表达式捕获组和反向引用

您可以将正则表达式放在括号中以便对它们进行分组。每个组都有一个以1开头的数字，因此您可以在替换模式中引用（反向引用）它们。请注意，组0指的是整个正则表达式。

我们考虑以下代码：

<form action = “” class = “form-inline” method = “post” > <input type = “text” class = “form-control” name = “username” value = “Username” > <input type = “password “ class = ”form-control“ name = ”password“ value = ”Password“ > <input class = ”btn btn-default“ type = ”submit“ value = ”Login“ > </ form>

  1. 打开搜索和替换窗格（ Ctrl + R ）。

  2. 在搜索字段中，输入表示捕获组的括号() ，例如： \svalue="(.*）？“\ S *（> *）

  3. 在替换字段中，按照以1开头的数字对这些组进行反向引用 ，例如： placeholder="$1"

切换字符大小写

您可以使用正则表达式来更改符合某些条件的字符大小写。

  2. 在搜索字段中输入搜索模式。

  3. 在替换字段中，根据您要实现的目标，输入以下语法之一：

    • \l将字符更改为小写，直到字符串中的下一个字符。
      例如， Bar变成了bar 。

    • \u将字符更改为大写，直到字符串中的下一个字符。
      例如， bar成为Bar 。

    • \L将字符更改为小写，直到文字字符串（ \E ）结束。
      例如， BAR成为bar 。

    • \U将字符更改为大写，直到文字字符串（ \E ）结束。
      例如， bar变为BAR 。

解决冲突

根据您的版本控制系统，可能会在不同情况下发生冲突。

当您在团队中工作时，您可能会遇到某人提交您当前正在处理的文件的更改的情况。如果这些更改不重叠（即对不同的代码行进行了更改），则会自动合并冲突的文件。但是，如果相同的行受到影响，则您的版本控制系统不能随意选择一侧而是另一侧，并要求您解决冲突。

合并，变基或挑选分支机构时也可能出现冲突。

非分布式版本控制系统

Sass，Less和SCSS

WebStorm与将Sass ， Less和SCSS代码转换为CSS的编译器集成在一起。要在WebStorm中使用编译器，您需要将其配置为文件监视器 。对于每个受支持的编译器，WebStorm都提供预定义的文件监视器模板。要在项目中运行编译器，请根据相关模板创建特定于项目的文件监视器。

在开始之前，请确保您的计算机上有Node.js.

安装Sass / SCSS

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install -g sass

    从Sass官方网站了解更多信息。

安装更少

您也可以在Node.js和NPM页面上安装less包，如npm和Yarn中所述 。

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install --global less

    从Less官方网站了解更多信息。

将代码编译为CSS

您也可以从命令行使用编译器或将其配置为第三方工具，有关详细信息，请参阅配置第三方工具 。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 文件监视器 ”。打开的“ 文件监视器”页面显示已配置的文件监视器列表。

  2. 点击或按Alt + Insert 。根据您要使用的工具，从列表中选择Less ， Sass或SCSS预定义模板。

  3. 在“ 程序”字段中，根据所选的预定义模板指定编译器可执行文件或存档的路径。

    • lessc.cmd for Less。

    • Sass / SCSS的sass.cmd 。

    如果您使用npm遵循标准安装过程，WebStorm将自行查找所需文件并自动填写该字段。否则，请手动键入路径或单击并在打开的对话框中选择文件位置。

  4. 按照使用文件监视器中的说明继续操作。

编译代码

当您打开文件时，WebStorm会检查当前项目中是否有适用的文件观察程序。如果配置了此文件观察程序但已禁用，则WebStorm会显示一个弹出窗口，通知您已配置的文件观察程序并建议启用它。

  • 如果选中“ 自动保存已编辑的文件以触发观察者”复选框，则只要对源代码进行任何更改，就会调用“文件观察器”。

  • 如果清除了自动保存已触发观察者的已编辑文件复选框，则会在保存（ 文件|全部保存 ， Ctrl + S ）或从WebStorm移动焦点时（启用帧停用时）启动文件观察器。

  2. 选择颜色方案，接受从默认值继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。

解决冲突

根据您的版本控制系统，可能会在不同情况下发生冲突。

当您在团队中工作时，您可能会遇到某人提交您当前正在处理的文件的更改的情况。如果这些更改不重叠（即对不同的代码行进行了更改），则会自动合并冲突的文件。但是，如果相同的行受到影响，则您的版本控制系统不能随意选择一侧而是另一侧，并要求您解决冲突。

合并，变基或挑选分支机构时也可能出现冲突。

非分布式版本控制系统

当您尝试编辑服务器上具有较新版本的文件时，IntelliJ IDEA会通知您，在编辑器中显示一条消息弹出窗口：

在这种情况下，您应该在更改文件之前更新本地版本，或者稍后合并更改。

如果您尝试提交具有较新存储库版本的文件，则提交失败，并在右下角显示错误，告知您尝试提交的文件已过期。

失败的提交行为由“ 版本控制”中的“ 在失败的提交时创建更改列表”下拉列表进行调整“ 设置/首选项”对话框的确认页面。

如果将已具有本地更改的文件与其他人提交的较新存储库版本同步，则会发生冲突。冲突文件获得Merged with conflict状态。该文件保留在“ 版本控制”工具窗口的“ 本地更改”选项卡中的相同更改列表中，但其名称以红色突出显示。如果文件当前在编辑器中打开，则选项卡标题上的文件名也会以红色突出显示。

分布式版本控制系统

在分布式版本控制系统（如Git和Mercurial）下，当您在本地提交的文件更改为与最新上游版本相同的代码行并且您尝试执行以下操作之一时，会出现冲突： pull ， merge ， rebase ， cherry-pick ， unstash或apply patch 。

如果存在冲突，这些操作将失败，系统将提示您接受上游版本，更喜欢您的版本或手动合并更改：

IntelliJ IDEA提供了一种在本地解决冲突的工具。该工具由三个窗格组成。左窗格显示只读本地副本;右侧窗格显示签入存储库的只读版本。中央窗格显示了一个功能齐全的编辑器，其中显示了合并和冲突解决的结果。最初，此窗格的内容与文件的基本修订版本相同，即从中派生两个冲突版本的修订版本。

什么是远程Web服务器？

在IntelliJ IDEA中，任何在当前项目之外具有文档根目录的服务器都称为远程服务器 。此服务器实际上可能在物理远程主机或您的计算机上运行。

例如，如果您的项目位于C：/ IntelliJ IDEAProjects / MyProject中 ，并且Web服务器文档根目录是C：/ XAMPP / htdocs ，则对于IntelliJ IDEA，此Web服务器是远程的 。

如何在IntelliJ IDEA项目中将服务器上的应用程序源与本地副本同步？

要在远程 Web服务器上调试应用程序，您需要在IntelliJ IDEA项目中获取其源代码的副本。要同步本地和远程源，请按照创建远程服务器配置和配置与Web服务器同步中所述创建 部署配置 。
