降价

Markdown是一种轻量级标记语言，用于向纯文本添加格式元素。WebStorm识别Markdown文件，为它们提供专用编辑器，并在实时预览中显示呈现的HTML。

Markdown编辑器可帮助您执行基本编辑功能：管理标题，将格式应用于文本以使其变为粗体或斜体等。您还可以使用完成功能添加指向其他项目文档或图像的链接，插入各种编程语言的代码块，以及可视化DOT或PlantUML图表。

预习

默认情况下，Markdown编辑器分为代码编辑器和预览窗格。如有必要，您只能使用显示代码编辑器或预览窗格要么纽扣。

您还可以管理编辑器和预览窗格是垂直拆分还是水平拆分。要执行此操作，请打开“ 设置/首选项”对话框（ Ctrl + Alt + S ），转到“ 语言和框架”| Markdown页面并使用“ 编辑器”和“预览面板布局”选项。

WebStorm自动将预览与代码编辑器中的当前插入位置同步。您可以使用“ 自动滚动预览”按钮管理此功能（ ）。

文档导航

在降价文档中导航的有用方法是“ 结构”视图 。它显示文档标题，使您可以快速跳转到所需的段落。

您可以在“结构”工具窗口（ Alt + 7 ）和“文件结构”弹出窗口（ Ctrl + F12 ）之间进行选择。

格式化

在Markdown编辑器中，您可以将基本格式应用于文本。例如，要使所选文本变为粗体，斜体或等宽，请使用 ， ，和纽扣。

项目文件的链接

Markdown编辑器提供了对当前项目中包含的文件的链接的完成 。例如，这可以是其他降价文档或图像。

如果在项目中找不到链接文件，Markdown编辑器将发出警告。

代码块

编辑器允许您通过在代码块之前和之后放置三重反引号（```）来为不同语言插入受限代码块。

所选语言支持代码完成 ， 检查和意图操作 。

如果不需要这些功能，则可以禁用markdown文件的代码注入 。要执行此操作，请打开“设置/首选项” （ Ctrl + Alt + S ），转到“ 语言和框架”| Markdown页面并使用代码栏中的禁用自动语言注入选项。

图

WebStorm支持使用DOT和PlantUML图表语言在Markdown预览中可视化图表。

要渲染图表，请安装PlantUML框架。打开设置/首选项 （ Ctrl + Alt + S ），转到语言和框架| Markdown页面，然后单击PlantUML框架旁边的Install按钮未安装警告。该框架将安装到WebStorm 缓存中 。

要显示图表，请使用puml或plantuml语言插入相应的代码块 。您可以复制下面的代码段以查看此功能的实际效果。

```puml A - > 乙 ```
``` plantuml digraph Test { A - > B } ```

用于预览的自定义CSS

Markdown预览允许您使用自定义样式表来呈现文档。您可以指定自定义样式表以进行基本演示文稿更改（例如，在预览中增加字体大小），也可以提供全新的CSS（例如，如果要复制GitHub Markdown样式）。

要提供自定义样式表，请打开“ 设置 / 首选项”对话框（ Ctrl + Alt + S ），转到“ 语言和框架”| Markdown页面，并使用“ 从URI加载”选项设置所需CSS文件的路径。此路径可以是URL或绝对/相对路径。

或者，在Add CSS rules编辑器中直接指定CSS。

配置语法突出显示

您可以根据自己的喜好和习惯配置Markdown-aware语法突出显示。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|配色方案| Markdown 。

  2. 选择颜色方案，接受从默认值继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。

在工作目录之间切换

Mercurial与IntelliJ IDEA的集成提供了将存储库的工作目录更新到指定的变更集或特定开发线的可能性。可以通过散列或先前分配的标记标识符来标识变更集。

在本页面：

  • 打开分支列表
  • 切换到另一个命名分支或书签
  • 切换到另一个变更集

您可以将命名分支或书签更新为另一个命名分支，另一个书签或由标记或修订号标识的特定变更集。

  • 将命名分支或书签更新到另一个命名分支或书签意味着更新到它的头部 。

    可以通过菜单项VCS |调用更新到命名分支和书签Mercurial |更新到 ，从分支列表中打开Mercurial特定的“ 切换工作目录”对话框。

  • 将命名分支或书签更新为变更集意味着将分支头更新为指定的变更集。可以通过修订号或标记来识别变更集，请参阅https://www.mercurial-scm.org/wiki/Tag 。

    只能通过VCS |调用将命名分支或书签更新为特定变更集Mercurial |更新到 。

默认情况下， Mercurial要求在更新之前当前工作目录应该是干净的 ，也就是说，它不应包含任何未提交的更改。否则更新操作失败，IntelliJ IDEA显示相应的错误消息。该消息还建议您通过运行hg update <target branch, bookmark, or changeset> -C来清除当前工作目录，以丢弃未提交的更改。

如果您当前的工作副本不干净，您可以提交更改或搁置它们，如搁架和取消搁置更改中所述 。IntelliJ IDEA提供了在已调用更新操作时丢弃任何未提交的更改的可能性。此选项仅在Mercurial特定的“ 切换工作目录”对话框中可用。

打开分支列表

要打开“ 分支”列表，请执行以下操作之一：

  • 在状态栏上，单击专用hg区域中当前命名分支的名称。

  • 在主菜单上，选择VCS | Mercurial |分支机构 。

  • 在“ 编辑器”或“ 版本控制”工具窗口的上下文菜单上，选择“ Mercurial”分支机构 。

切换到另一个命名分支或书签

将命名分支或书签更新到另一个命名分支或书签意味着更新到它的头部 。

  1. 确保您当前的工作目录是干净的 ，也就是说，它不包含任何未提交的更改。提交或搁置更改（如果有）。

    如果通过“ 切换工作目录”对话框调用更新，则还可以通过丢弃任何未提交的更改来防止冲突。

  2. 通过执行以下操作之一来调用更新：

    • 在“ 分支”列表中，单击要更新到的分支或书签的名称，然后从列表中选择“ 更新到 ”：

    • 选择VCS | Mercurial |更新到主菜单或Mercurial |在编辑器的上下文菜单上更新到 。

      在打开的“ 切换工作目录”对话框中：

      1. 从Repository列表中选择目标存储库，该列表显示当前项目根目录下可用的所有Mercurial存储库。

      2. 选择“ 分支”或“ 书签”选项，然后选择指定的分支或书签以将当前工作目录更新为。

      3. 要在当前工作目录不干净的情况下防止更新期间出现故障，请选中“ 覆盖本地修改的文件（无备份）”复选框。未提交的更改将被丢弃。

切换到另一个变更集

  1. 确保您当前的工作目录是干净的 ，也就是说，它不包含任何未提交的更改。提交或搁置更改（如果有）。

    如果通过“ 切换工作目录”对话框调用更新，则还可以通过丢弃任何未提交的更改来防止冲突。

  2. 通过执行以下操作之一来调用更新：

    • 在“ 分支”列表中，单击要更新到的分支或书签的名称，然后从列表中选择“ 更新到 ”：

    • 选择VCS | Mercurial |更新到主菜单或Mercurial |在编辑器的上下文菜单上更新到 。

      在打开的“ 切换工作目录”对话框中：

      1. 从Repository列表中选择目标存储库，该列表显示当前项目根目录下可用的所有Mercurial存储库。

      2. 选择“ 分支”或“ 书签”选项，然后选择指定的分支或书签以将当前工作目录更新为。

      3. 要在当前工作目录不干净的情况下防止更新期间出现故障，请选中“ 覆盖本地修改的文件（无备份）”复选框。未提交的更改将被丢弃。

  3. 解决冲突。一旦发生冲突，“ 冲突”对话框将打开，并显示冲突文件列表。使用对话框的控件来解决问题：

    • 要保留当前工作目录的版本，请单击“ 接受您的” 。

    • 要将要合并的分支版本保留，请单击“ 接受” 。

    • 要手动解决冲突，请单击“ 合并”并使用“ 冲突解决工具” ，如“ 解决冲突”中所述 。

    如果在更新期间没有发生冲突，则操作将以静默方式传递，并且更新日志将显示在“ 版本控制”工具窗口中。

从上传/下载中排除文件和文件夹

禁止上传，下载和同步带有源的文件或文件夹，可确保源免受意外更新。应用于非源时，它可以节省系统资源，因为不再不必要地复制媒体，缓存或临时文件。

在以下情况下，您可能需要禁止上载/下载：

  1. 您将使用外部创建和上载的源代码。要在WebStorm中处理这些远程源，您必须下载它们并将它们安排在项目中。但是，有一些消息来源根本不应该更新。另一方面，远程主机上的文件夹也可能包含您工作中实际不需要的大量媒体，缓存，临时文件。

  2. 您已经从服务器下载了数据并将它们安排在WebStorm项目中。但是，出于这个或那个原因，您需要在服务器上保留一些文件或文件夹以防止上载/下载，例如，以防止意外覆盖。

  3. 应用程序的本地副本包含源代码和您不需要上载的其他数据。此外，您希望保护某些来源不被错误覆盖。在这种情况下，您可以禁止上载/下载不应上载的所有文件和文件夹。

  • 显式地，通过在“ 远程主机”工具窗口或“ 部署”对话框的“ 排除路径”选项卡中将相应路径标记为已排除。

只有通过名称排除 文件，才能保护单独的文件不被上传/下载。

在任何情况下，递归地应用排除。如果文件夹的路径显式标记为已排除或文件夹名称与模式匹配，则其所有子文件夹的内容（如果有）也会受到上载/下载保护。

  • 分支默认值：选择此选项可以应用分支的默认命令。默认命令在.git/config配置文件的branch.<name>部分中指定。

如果拒绝当前分支的推送，则自动更新

如果希望在拒绝当前分支到其跟踪分支的push操作时自动更新当前分支，请选中此复选框。

  • 如果您之前从未见过“ 推送已拒绝”对话框，并且您最初启用了复选框，则WebStorm将通过merge操作以静默方式更新冲突的本地分支。

  • 如果您已经遇到“ 推送已拒绝”对话框并选择了“ 记住更新方法选择...”选项，则WebStorm将保存您的上一个选择（ rebase或merge ），并将其应用于以静默方式更新冲突的本地分支。

    因此，要更改“记住”设置，请清除该复选框，访问“ 推送已拒绝”对话框，选择“ 自动更新，如果推送...已拒绝”选项，并调用另一个更新策略。

在抑制检查时，PyCharm会插入一条特殊注释，告诉代码分析引擎忽略特定代码段中此检查发现的问题。

有时您可能需要创建临时注释或在项目上下文之外起草一些代码。您可以使用临时文件和暂存缓冲区 ，而不是为此目的切换到其他应用程序。

  • Scratch文件是功能齐全，可运行和可调试的文件，支持语法突出显示，代码完成以及相应文件类型的所有其他功能。例如，在处理一个项目时，您可能会想出一个方法，以后可以在另一个项目中使用。您可以使用方法草稿创建临时文件，该草稿不存储在项目目录中，但可以从其他项目访问和打开。您可以使用临时文件来草拟代码构造， HTTP请求 ，JSON文档等。

  • Scratch缓冲区是简单的文本文件，没有任何编码辅助功能。Scratch缓冲区可用于简单的任务列表和自己的注释。它们也不存储在项目目录中，但可以从另一个项目访问和打开。您最多可以创建五个具有默认名称的暂存缓冲区，这些缓冲区通过清除内容进行轮换和重用。

创建一个临时文件

  1. 执行以下操作之一：

    • 按Ctrl + Shift + Alt + Insert 。

    • 按Ctrl + Shift + A ，开始键入scratch file ，然后选择相应的操作。

  2. 选择临时文件的语言。相同类型的临时文件会自动编号并添加到“ 项目”工具窗口的 “ 划痕和控制台”目录中

您可以按照配置键盘快捷键中的说明为New Scratch Buffer操作创建快捷方式 。

创建一个临时缓冲区

创建新临时缓冲区的操作没有专用菜单项，但您可以使用“ 查找操作”弹出窗口（ Ctrl + Shift + A ）并运行“ 新建临时缓冲区”操作。

PyCharm创建一个名为buffer1.txt的文本文件。您创建的下一个临时缓冲区名为buffer2.txt ，依此类推，最多为buffer5.txt 。达到该限制时，它将重新创建buffer1.txt并建议清除其内容。如果要确保在有五个之后没有清除暂存缓冲区，可以重命名它。

要查看创建的临时文件和缓冲区列表，请打开“ 项目”工具窗口 ，展开“ 划痕和控制台” ，然后展开“ 划痕” 。

控制台用于组合和执行PyCharm中定义的数据库的SQL语句作为数据源。仅当启用了“ 数据库工具”和“SQL插件”时，此功能才可用。有关更多信息，请参阅使用数据库控制台 。

查看划痕文档

  • 选择任何临时文件或缓冲区，然后按Ctrl + Q查看包含文件位置，类型，大小，创建和修改日期的快速文档 。

移动划痕

  1. 在“ 项目”工具窗口中的“ 划痕和控制台”中Scratches目录，选择要移动的临时文件。

  2. 按F6并在打开的对话框中选择目标文件夹。

临时文件和缓冲区存储在IDE配置目录下的划痕下。它们可以从任何使用此配置目录的IDE和项目中获得。

更改临时文件或缓冲区的语言

  1. 在“ 项目”工具窗口中 ，展开“ Scratches and Consoles” ，然后展开“ Scratches” 。

  2. 右键单击临时文件或缓冲区，然后单击“ 更改语言” 。当前语言在括号中指定。

  3. 选择所需的语言。您可以开始键入语言名称以缩小列表范围。

与任何其他文件一样，您可以使用暂存文件和缓冲区执行以下操作：

  • 使用装订线图标运行和调试代码

  • 使用本地历史记录 ，撤消和重做更改

  • 重命名 ， 删除 ， 移动和复制

  • 执行剪贴板操作

配置Pipenv环境

以下内容仅在安装并启用Python插件时有效！

Pipenv是一个工具，它为Python项目创建虚拟环境提供了所有必要的手段。在安装或卸载软件包时，它会通过Pipfile文件自动管理项目包。

Pipenv还生成Pipfile.lock文件，该文件用于生成确定性构建并创建工作环境的快照。当项目要求和软件包版本至关重要时，这可能对安全敏感部署特别有用。有关pipenv更多详细信息，请参阅项目文档docs.pipenv.org 。

要将Pipenv与IntelliJ IDEA一起使用，您需要实施几个准备步骤。

安装Pipenv

  1. 运行以下命令以确保在系统中安装了pip：

    $ pip --version

    您应该期望收到指示pip版本的系统响应。如果未发现pip，请按照安装说明中的说明进行安装 。或者，您可以从http://python.org下载并安装Python。

  2. 通过运行以下命令安装pipenv ：

    $ pip install --user pipenv

    安装完成后，您将看到以下消息：

  3. 为方便起见，您可以将用户库的二进制目录添加到PATH环境变量中。如果跳过此过程，IntelliJ IDEA将在添加pipenv环境时提示您指定pipenv可执行文件的路径。

    1. 运行以下命令：

      $ py -m site --user-site

      示例输出可以是：

      C：\用户\ JetBrains公司\应用程序数据\漫游\ Python的\ Python37 \站点包

    2. 将site-packages替换为此路径中的Scripts以接收用于添加到PATH变量的字符串，例如：

      $ setx PATH“％PATH％; C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts”

    1. 运行以下命令以查找用户库的二进制目录：

      $ python -m site --user-base

      样本输出可以是

      /Users/jetbrains/.local

    2. 将bin添加到此路径以接收用于添加到~/.bashrc文件的字符串，例如：

      $ export PATH =“$ PATH：Users / jetbrains / .local / bin”

    3. 运行以下命令以使更改生效：

      $ source~ / .bashrc

    4. 确保您已在bash_profile启用bashrc 。

您可以随时在项目设置中更改pipenv可执行文件的指定路径。在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，导航到“ 工具”| Python集成工具 ，并在Path ot Pipenv可执行文件字段中键入目标路径。

完成准备步骤后，您可以使用pipenv为新项目或现有项目创建虚拟环境。

为新的Python项目设置pipenv

  1. 按创建空Python项目中的描述启动创建新的Python项目 。

  2. 在右侧窗格中，单击“ 新建”以创建新项目SDK。

  3. 在“ 添加Python解释器”对话框的左侧窗格中，选择“ Pipenv” 。

  4. 如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径：

  5. 单击“ 确定”完成任务。

  6. 如果您尚未修改PATH变量，IntelliJ IDEA将显示错误消息： 找不到Pipenv可执行文件 。发现pipenv安装过程中描述的正确可执行路径，并在Pipenv可执行文件字段中输入目标字符串，例如： C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts \ pipenv.exe （Windows）或/Users/jetbrains/.local/bin/pipenv（macOS ）。

    单击“ 确定”保存更改并完成任务。

您还可以为任何早期创建的项目设置pipenv。

为现有的Python项目配置pipenv

  1. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目结构”对话框中，选择“ 平台设置”部分下的SDK ，然后单击 ，从弹出菜单中选择Python SDK 。

  3. 在此对话框的左侧窗格中，单击“ Pipenv环境” 。

  4. 如果已将用户库的二进制目录添加到PATH环境变量，则无需设置任何其他选项：将自动检测pipenv可执行文件的路径。

    如果您看到找不到Pipenv可执行文件错误消息，请按照pipenv安装过程发现可执行文件路径并将其添加到Pipenv可执行文件字段。

  5. 单击“ 确定”完成任务。

完成所有步骤后，将为您的项目设置新的pipenv环境，并安装Pipfile中列出的软件包。

如果选择此选项，IntelliJ IDEA会自动为您设置pipenv。或者，您可以单击“ 配置Python解释器”以遵循标准工作流程。

同样，当您第一次使用IntelliJ IDEA中的Pipfile文件打开项目时，例如，通过从版本控制中检出它，将自动配置Pipenv虚拟环境。

将Pipenv虚拟环境设置为项目解释程序后，将从Pipfile Pipfile中定义的源添加所有可用程序包。通过pipenv而不是通过pip在软件包列表中安装，删除和更新软件包。

  • 安装和卸载解释器包

在适当的位置提取Sass变量

  1. 在编辑器中，选择要用变量替换的表达式。您可以自己动手或使用智能表达式选择功能让WebStorm为您提供帮助。因此，请执行以下操作之一：

    • 从列表中选择一个建议的名称。为此，请双击合适的名称。或者，使用向上和向下箭头键导航到感兴趣的名称，然后按Enter键选择它。完成后，按Escape键。

    • 通过键入来编辑名称。名称显示在带红色边框的框中，并在您键入时更改。完成后，按Escape键。

表达式列表包含适用于编辑器中当前光标位置的所有表达式。

在弹出窗口中浏览建议的表达式时，编辑器中突出显示的代码会相应更改。

管理数据源

此功能仅在Ultimate版本中受支持。

数据源是应用程序的数据位置。

在IntelliJ IDEA中，您可以定义以下数据源：

  • 数据库数据源：包含数据的操作数据库（请参阅连接到数据库 ）。

  • 数据定义语言（DDL）文件：包含数据定义语言语句的SQL文件（请参阅添加DDL数据源 ）。

创建数据源

从数据库文件创建数据库数据源

  1. 如果数据库文件位于项目文件夹中，请打开“ 项目”工具窗口。否则，打开文件管理器。

  2. 打开数据库工具窗口（ 查看|工具Windows |数据库 ）。

  3. 选择要用作数据源的文件，然后将其拖到“ 数据库”工具窗口中。为每个文件创建单独的数据源。

  4. 如果您没有必要的数据库驱动程序文件，请下载它们。点击在数据库工具窗口的工具栏上。或者，从上下文菜单中选择“ 属性 ”。

  5. 在“ 数据源和驱动程序”窗口中，单击“ 下载”链接。

  6. 要确保与数据源的连接成功，请单击“ 测试连接” ，然后单击“ 确定” 。

从项目文件配置数据源

您可以使用包含数据库连接设置的XML文件（例如，Spring，Hibernate，JPA和Tomcat的context.xml ）来创建数据库数据源。

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ）。

  2. 点击在工具栏上，选择从源导入 。或者，右键单击工具栏下或现有数据源下的区域，选择“ 新建” ，然后单击“ 从源导入” 。

    在“ 数据源和驱动程序”窗口中，您可以看到绿色的候选数据源的名称。

  3. 如果缺少驱动程序文件，请下载它们。

    要指定驱动程序，请执行以下步骤之一：

    • 要下载必要的驱动程序，请单击“ 下载”链接。

    • 要指定计算机上已有的驱动程序文件，请单击Driver右侧的<DriverName>链接。

      在显示驱动程序设置的页面上，在JDBC drivers / Additional下 ，单击并选择必要的文件。

      返回包含数据源设置的页面。

  4. 要确保与数据源的连接成功，请单击“ 测试连接” 。

  5. 单击确定 。

创建DDL数据源

  1. 打开数据库工具窗口（ 查看|工具Windows |数据库 ）。

  2. 单击“ 数据源属性”图标（ ）在工具栏上，选择DDL数据源 。

  3. 指定数据源名称。

  4. 在“ DDL文件”窗体中，单击“ 添加”图标（ ）并选择必要的SQL文件。

此外，您可以从“ 文件”工具窗口或文件管理器将DDL文件拖到“ 数据库”工具窗口。

自定义数据源

配置数据源设置

  1. 在“ 数据库”工具窗口（“ 查看”|“工具Windows”|“数据库” ）中，选择要更改的数据源。

  2. 单击“ 数据源属性”图标（ ）在工具栏上（或按Shift + Enter ）。

  3. 在“ 数据源和驱动程序”窗口中，编辑数据源属性。有关数据源属性的详细信息，请参阅数据源设置和DDL数据源设置 。

组数据源

如果您有许多不同的数据库，则可以在“ 数据库”工具窗口（“ 查看”|“工具Windows”|“数据库” ）中对其数据源进行分组。要对数据源进行分组，请选择所需的数据源，按F6 ，或从上下文菜单中选择“ 移至组 ”。

“ 移至组”菜单中的数据源组可使用以下选项：

  • 为数据源创建一个新组。

  • 将数据源移动到根列表。此选项从组中删除数据源。

  • 将数据源移动到现有组。

删除数据源

要删除数据源，请在“ 数据库”工具窗口（“ 查看”|“工具Windows”|“数据库” ）中，按“ 删除” 。或者，您可以从上下文菜单中选择“ 删除 ”，或单击“ 编辑”|删除 。

此外，要删除数据源，可以使用“ 数据源和驱动程序”窗口（“ 文件”|“数据源” ）。在“ 数据源和驱动程序”窗口中，选择数据源，然后按Delete 。或者，您可以单击在工具栏上，或从上下文菜单中选择“ 删除 ”。

管理项目收藏夹

您可以在收藏项目（收藏夹）列表中排列最常用的文件，文件夹， 书签和断点 。WebStorm有一个专用的收藏夹工具窗口，用于管理您的收藏夹。

最初，有一个（空）收藏夹列表，其名称与项目相同。

您可以根据需要创建更多收藏夹列表并管理其内容。

使用“项目”工具窗口将项目添加到收藏夹

  1. 在项目中 ，选择要添加到收藏夹的项目。

  2. 执行以下操作之一：

    • 按Shift + Alt + F.

    • 选择文件|从主菜单添加到收藏夹 。

    • 从选择的上下文菜单中，选择“ 添加到收藏夹” 。

  3. 要将所选项目添加到现有收藏夹列表，请选择列表名称。

    要创建新的收藏夹列表并将所选项目添加到其中，请选择“ 添加到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

使用拖放操作将项目添加到收藏夹：从“ 项目”工具窗口拖动感兴趣的项目 ，或从“ 资源管理器”或“ Finder”拖动外部文件，然后将其拖放到“收藏夹”工具窗口中的所需收藏夹列表中。

在编辑器中，您可以将一个文件（ Shift + Alt + F ）或所有当前打开的文件添加到收藏夹。

使用编辑器将文件添加到收藏夹

评估表达式

PyCharm使您能够在“ 调试”工具窗口的“ 帧”窗格中当前选定的堆栈帧的上下文中计算表达式和代码片段。

除了正则表达式，您还可以计算运算符表达式，lambda表达式和匿名类。

可以使用以下评估模式：

  • 用于评估单行表达式的表达模式 。

  • 用于评估短代码部分的代码片段模式 。您可以评估声明，赋值，循环和if/else 。

此外，PyCharm提供了一种快速评估插入符号或编辑器中选择的表达式的方法。

使用表达式评估功能时，请注意以下事项：

  • 仅当调试器在断点处停止但尚未暂停时，才能在“ 评估”对话框中调用方法。

  • 表达评估只能是“单级”。换句话说，如果PyCharm停止在从表达式求值调用的方法中的断点处，则无法再次使用“表达式求值”功能。

请注意，在某些操作系统中，键和鼠标组合可能无法正常工作，如此处所述。在这种情况下，有必要调整操作系统的键映射。例如，如果您使用的是Ubuntu，请注意Windows管理器，其快捷方式与PyCharm的快捷方式相冲突 。

评估表达式或代码片段

  1. 在“ 框架”窗格中，选择要评估表达式的线程。要评估特定变量，请在“ 变量”窗格中选择它。

  2. 使用以下方法之一调用Evaluate Expression命令：

    • 在主菜单上，选择“运行”|评估表达

    • 在编辑器的上下文菜单中，选择“ 评估表达式”

    • 按Alt + F8

    • 点击在“ 调试”工具窗口的“ 步进”工具栏上。

  3. 单击“ 评估” 。

    如果尚未选择任何特定变量，只需在文本字段中键入任何表达式，然后单击“ 评估” 。

    如果无法计算指定的表达式，则会在对话框的“ 结果”窗格中简要描述可能的原因。

    请注意，您可以使用“ 表达式”字段中的列表在评估的表达式之间导航。

  4. 要评估代码片段，请单击“ 表达式”字段中的展开箭头（ Shift + Enter ）。

    Expression字段扩展并更改为Code片段 。将目标代码片段粘贴到文本字段中，然后单击“ 评估” 。

    要在评估的片段之间导航，请按Alt +向下和Alt +向上 。

在编辑器中评估表达式

在调试器会话期间，每次将鼠标指针悬停在工具提示上时，任何表达式的值都会显示在工具提示中。如果表达式包含子项，请单击展开节点并显示所有子节点。

在代码选择上显示值工具提示

  2. 使用鼠标选择代码片段，或按Ctrl + W.带有表达式值的工具提示会自动显示在选择项下，并在每次更改选择时更改。

使用表达式值手动调用工具提示

  1. 将插入符号放在所需位置，或选择要评估的表达式。

  2. 选择运行|在主菜单上快速评估表达式 ，或按Ctrl + Alt + F8 。带有表达式值的工具提示将显示在所选表达式下。

缩进由两部分组成。一部分来自嵌套代码块，另一部分是通过对齐来确定的。

空间

在编辑器语法中探索HTTP请求

要在IntelliJ IDEA代码编辑器中撰写HTTP请求，请使用以下常规语法：

方法Request-URI HTTP-Version标头字段：标头值Request-Body

编辑器格式的HTTP请求引入了其他功能，如以下示例所示。有关使用HTTP请求的详细信息，请参阅IntelliJ IDEA代码编辑器中的HTTP客户端 。

在HTTP请求中使用注释

  • 在请求中，使用//或#开始任何行以使其成为注释行。

    //基本请求GET http://example.com/a/

使用简短形式进行GET请求

  • 对于GET请求，您可以省略请求方法并仅指定URI。

    //基本请求http://example.com/a/

    在Java上下文中，您可以使用代码完成 （ Ctrl + Space ）根据定义的@Path注释指定URI。对Java代码中的@Path注释的任何更改都将反映在建议列表的内容中。

在单个文件中撰写多个请求

  1. 通过在其下方键入###分隔符来标记请求的结尾。

    //一个基本请求http://example.com/a/ ###

  2. 在分隔符下面撰写另一个请求。

    //一个基本请求http://example.com/a/ ### //使用GET方法的第二请求http://example.com:8080/api/html/get?id = 123＆ 值 =内容

将长请求分成几行

  • 缩进所有查询字符串行，但第一个。

    //使用缩进的换行符GET http://example.com:8080 / api / html / get？id = 123 ＆ value = content

使用身份验证访问Web服务

  • 根据您访问的Web服务，提供基本或摘要 Authorization标头。

    //基本身份验证GET http://example.com授权：基本用户名密码### //摘要身份验证GET http://example.com授权：摘要用户名密码

与其他HTTP请求的元素类似，可以通过环境变量参数化提供的username和password 。

提供请求消息正文

  • 在请求中，在请求正文前面加一个空行并执行以下操作之一：

    • 输入请求正文：

      //提供请求正文POST http://example.com:8080/api/html/post HTTP / 1.1 Content-Type：application / json Cookie： key = first-value { “key” ： “value” ， “列表” ： [ 1,2,3 ] }

      如果将Content-Type标头字段值设置为IntelliJ IDEA支持的语言之一，则相应的语言片段将自动注入HTTP请求消息正文中。如果未指定Content-Type则可以手动注入语言片段。有关更多信息，请参阅语言注入 。

    • 在Java上下文中，您可以使用代码完成 （ Ctrl + Space ）根据定义的@Produces注释指定Accept头字段值。对Java代码中@Produces注释的任何更改都将反映在建议列表的内容中。

    • 要从文件中读取请求正文，请键入<符号，然后键入文件的路径。

      //从文件POST中读取请求正文http://example.com:8080/api/html/post Content-Type：application / json <./input.json

使用multipart / form-data内容类型

  • 将请求的Content-Type为multipart/form-data 。要将文件作为multipart/form-data消息的一部分发送，请在Content-Disposition标头中包含filename参数。

    POST http://example.com/api/upload HTTP / 1.1 Content-Type：multipart / form-data ; boundary = boundary --boundary Content-Disposition：form-data ; name = “first” ; filename = “input.txt” // 'input.txt'文件将被上传<./input.txt --boundary Content-Disposition：form-data ; name = “second” ; filename = “input-second.txt” //将创建一个带有“Text”内容的临时“input-second.txt”文件并上传Text --boundary Content-Disposition：form-data ; name = “third” ; //'input.txt'文件内容将以纯文本形式发送。<./input.txt --boundary--

要加快创建multipart / form-data请求，请使用mptr live模板 。

启用或禁用以下重定向

根据您访问的Web服务，您可能希望HTTP请求遵循重定向或不遵循重定向。当遵循重定向时，返回重定向的页面响应;否则，返回实际的重定向响应头（例如301或302 ）。

  • 在请求之前，添加带有@no-redirect标记的注释行。

    // @ no-redirect example.com/status/301

启用或禁用将请求保存到请求历史记录

如有必要，可以阻止将请求保存到请求历史记录中 。如果请求包含一些敏感数据，并且您不想记录它，这可能会有所帮助。

  • 在请求之前，添加带有@no-log标记的注释行。

    // @ no-log GET example.com/api
