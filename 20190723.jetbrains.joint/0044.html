创建一个新的Dart应用程序

如果您还没有应用程序，可以从Stagehand模板生成具有Dart特定结构的PyCharm项目。或者，创建一个空的PyCharm项目并在其中配置Dart支持，如下面的开始使用现有的Dart应用程序中所述。

运行和调试Dart命令行应用程序

使用PyCharm，您可以运行和调试Dart命令行应用程序。PyCharm支持两种调试模式：

  • 本地调试 ：在此模式下，您的应用程序从PyCharm启动，并在您的计算机上本地运行。要运行或调试它，请使用Dart命令行应用程序配置。

  • 调试远程应用程序 ：在此模式下，应用程序在远程环境中运行，例如，在Docker容器中运行。要调试它，请使用Dart Remote Debug配置。

运行Dart命令行应用程序

  1. 打开Dart文件以启动应用程序或在项目工具窗口中选择它。该文件必须包含main()方法。

  2. 从选择的上下文菜单中，选择Run'<dart_file_name>' 。PyCharm生成Dart Command Line App类型的运行/调试配置，并使用它启动您的应用程序。

在本地调试Dart命令行应用程序

  1. 在Dart代码中配置和设置断点 。

  2. 打开Dart文件以启动应用程序或在项目工具窗口中选择它。该文件必须包含main()方法。

  3. 从选择的上下文菜单中，选择Debug'<dart_file_name>' 。PyCharm生成Dart命令行应用程序的运行/调试配置，并启动它的调试会话。

  4. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

创建Dart远程调试运行/调试配置

  1. 在主菜单上，选择“运行”|编辑配置 ，单击并从列表中选择Dart Remote Debug 。

  2. 在“ 主机”字段中，指定运行Dart虚拟机的计算机的地址，默认值为localhost 。

  3. 指定调试器将通过其连接到远程应用程序的端口，默认值为5858 。启动远程VM只读字段时 ， 使用命令行参数中显示指定的端口。请注意，必须使用这些参数准确启动远程应用程序。

  4. 如果您的PyCharm项目包含多个配置为内容根的Dart项目，请从“ 搜索源”列表中选择要调试的Dart项目 。

启动远程调试会话

  1. 使用Dart Remote Debug运行配置中远程Dart VM字段的命令行参数中的VM选项启动远程Dart应用程序，例如， --enable-vm-service:5858 --pause_isolates_on_start 。应用程序启动，由于--pause_isolates_on_start参数立即暂停，并等待调试器连接。

  2. 在Select run / debug configuration列表中选择新创建的Dart Remote Debug配置，然后单击 。

  3. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

运行和调试Dart Web应用程序

或者，在编辑器中打开HTML文件，按Alt + F2 ，然后从列表中选择一个浏览器。

默认Webdev服务器端口为53322 。如果此端口已经忙，PyCharm将使用最接近的端口。

确保此URL地址中的端口与“ 调试器”页面上的内置服务器端口和Chrome扩展设置中的端口相同 。

您可以在任何浏览器中运行Dart Web应用程序，而仅在Dartium和Chrome中支持调试。要运行Dart Web应用程序，请在浏览器中打开应用程序的主HTML文件。调试Dart Web应用程序是通过JavaScript Debug类型的运行配置启动的。

PyCharm与webdev服务工具集成，可在必要时将Dart代码编译为JavaScript。对于早于Dart SDK 2的版本，将使用pub serve工具。

当您在浏览器中打开Dart Web应用程序时，它通常以内置服务器URL开头，例如http://localhost:63342/project-name/web/index.html 。但是，内置服务器不用于为应用程序提供服务。相反，PyCharm会自动在Dart对话框中指定的端口上启动webdev服务器，并且浏览器页面将重定向到webdev服务器URL（如http://localhost:53322/index.html ）。

webdev服务器的工作记录在专用的Dart Webdev工具窗口中。在当前PyCharm会话期间首次开始运行或调试Dart Web应用程序时，将打开工具窗口。您可以通过单击停止该工具在工具栏上。当您再次开始运行或调试时，webdev服务器会自动重新启动。

运行Dart Web应用程序

  1. 使用Dart引用打开HTML文件，或在“ 项目工具”窗口中选择它。

  2. 从编辑器或选择的上下文菜单中，选择“ 在浏览器中打开”，然后在列表中选择所需的浏览器。

    • 使用Dart SDK 2，您只能使用Chrome，并且Dart代码可以通过dart2js或dartdevc工具编译为JavaScript。

    • 使用Dart SDK 1.x，您可以使用Dartium ，它具有内置的Dart虚拟机，因此Dart代码是本机执行的。

      要使用Dartium，请按照配置浏览器中的说明下载并配置它。确保浏览器名称为Dartium或至少包含Dartium 。

  4. URL字段已经以http://localhost:<built-in server port>/<project-name>/<relative path to the HTML file>格式显示应用程序的URL地址。在调试会话期间，浏览器将从此URL重定向到webdev服务器URL。

  2. 启动调试会话：从工具栏上的“ 编辑配置”列表中选择创建的运行配置，然后单击 。PyCharm在所选浏览器中打开指定的URL。

  3. 在打开的调试工具窗口中， 逐步执行程序 ， 停止和恢复程序执行， 暂停时检查它 ，依此类推。

如果选择了某个文件夹，则测试运行器仅在名称格式为*_test.dart的文件中查找测试。

PyCharm支持运行和调试使用dart测试包编写的Dart测试。您可以在任何目标平台上运行测试，仅支持VM测试进行调试。

从文件夹运行或调试Dart测试

  • 在“ 项目工具”窗口中 ，选择要运行测试的文件夹，然后从上下文菜单中选择“运行'<folder_name>”或“ 调试”<folder_name> 。

在你开始之前

如管理插件中所述，在插件页面上安装并启用PHP和Drupal支持存储库插件 。

下载并安装Drupal

从Drupal官方网站下载Drupal并按照Drupal安装说明中的说明进行安装 。

在IntelliJ IDEA中设置Drupal项目

选项1：使用Drupal模块创建IntelliJ IDEA项目

您可以通过Drupal Module模板创建IntelliJ IDEA项目，该项目将根据Drupal要求进行设置和配置。

  1. 打开“新建项目”对话框

    选择文件|新的|单击项目或单击欢迎屏幕上的创建新项目 。

  2. 选择项目类型和位置

    在左侧窗格中，单击列表中的PHP ，然后在右侧窗格中选择Drupal Module ，然后单击Next 。

    在向导的第二页上，指定项目名称以及将在其中创建项目的文件夹。

  3. 选择要使用的Drupal安装

    指定Drupal安装的根文件夹并选择要使用的Drupal版本，支持的版本是6,7和8。

    稍后您可以更改Drupal安装并在Frameworks页面上重新配置包含路径，如更改Drupal设置中所述

  4. 配置包含路径

    选择设置PHP |包含路径复选框以使Drupal包含为项目自动配置的路径。离开对话框后，以下路径将添加到PHP页面上的Include Paths列表中： <drupal installation root>/includes ， <drupal installation root>/modules和<drupal installation root>/sites/all/modules

选项2：导入现有的Drupal模块

当您打开现有的Drupal模块时，IntelliJ IDEA会识别Drupal特定的结构，并建议激活Drupal支持。

  1. 打开你的项目

    单击欢迎屏幕上的打开或选择文件|在主菜单上打开 ，然后选择存储Drupal模块的文件夹。

    IntelliJ IDEA检测特定于Drupal的结构并显示通知：

  2. 启用Drupal支持

    单击通知中的“ 启用 ”。在打开的Drupal Module对话框中，选择Enable Drupal integration复选框，并在使用Drupal模块创建项目时继续：指定Drupal安装的根文件夹，选择要使用的版本，并配置包含路径。

在事件日志工具窗口中报告任何检测到的不一致，并作为弹出窗口针对每个差异，IntelliJ IDEA建议修复。要应用建议，请单击报告的事件旁边的链接。

V8 CPU和内存分析

此功能仅在Ultimate版本中受支持。

使用IntelliJ IDEA，您可以使用V8基于样本的分析器捕获和分析Node.js应用程序的CPU配置文件和堆快照。

您还可以打开并浏览Google Chrome DevTools中捕获的配置文件和快照，以获取客户端代码。

在你开始之前

确保您的计算机上有Node.js.

CPU分析

CPU分析可帮助您更好地了解代码的哪些部分占用大部分CPU时间，以及V8 JavaScript引擎如何执行和优化代码。

IntelliJ IDEA中的Node.js CPU分析基于V8内置CPU分析器，它提供有关代码执行和JavaScript引擎本身行为的信息，包括垃圾收集周期，编译和重新编译以及代码优化。

探查器以特定间隔拍摄快照，这些间隔称为刻度线。测量不仅针对代码的工作，还针对引擎本身执行的活动，例如编译，系统库调用，优化和垃圾收集。

启用CPU分析

要在应用程序启动时调用V8 CPU分析，您需要在Node.js运行配置中指定其他设置。

  1. 在主菜单上，选择“运行”|编辑配置 。点击在工具栏上，从列表中选择Node.js.

  2. 从列表中，选择Node.js运行配置以激活CPU分析或创建新配置，如运行和调试Node.js中所述 。

  3. 切换到V8 Profiling选项卡，然后选中Record CPU profiling info复选框。在“ 日志文件夹”字段中，指定将存储记录日志的文件夹的路径，日志文件名为isolate-<session number> 。

收集CPU分析信息

  1. 从主工具栏上的列表中选择运行配置，然后单击或选择运行|从主菜单中运行<配置名称> 。

  2. 执行需要配置的方案时，单击以停止该过程在工具栏上。

分析CPU分析日志

当您停止应用程序时，IntelliJ IDEA会自动打开V8性能分析工具窗口，并在其中显示收集的性能分析数据。如果窗口已打开并显示另一个会话的收集数据，IntelliJ IDEA将打开一个新选项卡。自动打开的选项卡以控制应用程序执行和收集分析数据的运行配置命名。

要打开并分析以前保存的某些性能分析数据，请转到主菜单上的V8性能分析 - 分析V8性能分析日志 ，然后选择相关的isolate-<session number> 。IntelliJ IDEA创建一个单独的选项卡，其中包含所选日志文件的名称。

根据收集的分析数据，IntelliJ IDEA构建三个调用树并在单独的窗格中显示每个调用树。使用这些调用树，您可以从两个不同的角度分析应用程序执行：一方面，哪些调用是耗时的（“重”），另一方面，“谁叫谁”。

了解调用树中的度量标准

调用树使用Total和Self度量标准，这些度量标准显示函数中的滴答数或其与总执行时间的比率：

  • Total显示在函数内部花费了多少时间以及它调用的函数。

  • Self表示仅在函数内部花费了多少时间而没有考虑其子节点。

Of Parent度量标准显示函数的纯执行时间与调用它的函数（ Parent ）的执行时间之比。

V8优化器

在某些情况下，V8可以优化您的代码，有关详细信息，请参阅优化V8 ：

  • 函数名称前面的星号（ * ）表示该函数已经过优化。

  • 波浪号（ ~ ）表示该功能可能需要优化但由于某些原因尚未优化。如果代码短时间运行，引擎可以推迟优化或跳过它，但是代码可以重写的位置处的代数可以获得更好的性能。

“ 概述”窗格中的图表显示Self％指标高于1％的通话的自我时间分布。

如果您遇到繁重的功能并且想要找出它的调用位置，则自下而上的树很有用。

“ 自下而上”窗格还列出了自我指标按降序排序的已执行活动。与“ 热门呼叫”窗格不同，“ 自下而上”窗格仅显示总百分比度量值大于2的活动以及调用它们的函数。

  • 对于每个活动，将显示其在刻度线中的执行时间和Of Parent指标。

内存分析使您可以检测内存泄漏和动态内存问题，并找到导致它们的代码片段。

要在应用程序启动时调用内存快照，您需要在Node.js运行配置中指定其他设置。

  1. 在主菜单上，选择“运行”|编辑配置 。点击在工具栏上，从列表中选择Node.js.

拍摄快照并选择对其进行分析时，IntelliJ IDEA会打开包含所收集数据的V8堆工具窗口。如果窗口已打开并显示另一个会话的收集数据，IntelliJ IDEA将打开一个新选项卡。

工具窗口有三个选项卡，用于显示来自不同视点的收集信息。

  • 在遏制选项卡显示在几个顶级项分组应用程序中的对象：DOMWindow对象 ， 本地浏览器的对象 ，GC和根 ，这是根的垃圾收集器实际使用。有关详细信息，请参阅遏制视图 。

    对于每个对象，选项卡显示其与GC根的距离 ，即对象和GC根之间节点的最短简单路径，对象的浅大小以及对象的保留大小 。除了对象大小的绝对值外，IntelliJ IDEA还显示了对象占用的内存百分比。

  • “ 最大对象”选项卡显示按保留大小排序的内存最多的对象。在此选项卡中，您可以发现通过在某个全局对象中累积数据而引发的内存泄漏。

  • “ 摘要”选项卡显示应用程序中按类型分组的对象。该选项卡显示每种类型的对象数，它们的大小以及它们占用的内存百分比。该信息可能是存储器状态的线索。

每个选项卡都有一个“ 详细信息”窗格，该窗格显示GC根目录中当前所选对象的路径以及对象的保留器列表，即保留指向所选对象的链接的对象。每个堆快照都有许多“后退”引用和循环，因此每个对象总是有许多保留器。

    为此，选择对象并单击在工具栏上，或从上下文菜单中选择“ 在主树中导航” 。

  1. 在遏制选项卡，单击在工具栏上。

  2. 在打开的V8堆搜索对话框中，指定要搜索的搜索模式和范围。可用范围是：

    • 无处不在：选中此复选框可在所有范围内搜索。选中此复选框后，将禁用所有其他搜索类型。

    • 链接名称：选中此复选框以在调用C ++运行 时时搜索V8创建的对象名称。

      在V8堆工具窗口中，链接名称标有%字符（ %<link name> ）。

    • 类名：选中此复选框可在函数构造函数中进行搜索。

    • 文本字符串：选中此复选框可在对象内容中执行文本搜索。

    • 快照对象ID：选中此复选框可在对象的唯一标识符中进行搜索。在创建对象时， V8会以格式为每个对象分配这样一个唯一标识符，并保留它直到对象被销毁。这意味着您可以在同一会话中拍摄的多个快照中查找和比较相同的对象。

      在V8堆工具窗口中，对象ID用@字符标记（ @<object id> ）。

    • 标记：选中此复选框可通过单击手动搜索您手动设置的标签在遏制选项卡的工具栏上。

搜索结果显示在“ 详细信息”窗格中，在“<搜索模式>”视图的单独出现中 。要使搜索结果按照您指定的搜索范围分组，请按工具栏上的“ 按类别分组”切换按钮。

在你开始之前

确保您的计算机上有Node.js.

CPU分析

CPU分析可帮助您更好地了解代码的哪些部分占用大部分CPU时间，以及V8 JavaScript引擎如何执行和优化代码。

Node.js PyCharm中的CPU分析基于V8内置CPU分析器，它提供有关代码执行和JavaScript引擎本身行为的信息，包括垃圾收集周期，编译和重新编译以及代码优化。

探查器以特定间隔拍摄快照，这些间隔称为刻度线。测量不仅针对代码的工作，还针对引擎本身执行的活动，例如编译，系统库调用，优化和垃圾收集。

启用CPU分析

要在应用程序启动时调用V8 CPU分析，您需要在Node.js运行配置中指定其他设置。

  1. 在主菜单上，选择“运行”|编辑配置 。点击在工具栏上，从列表中选择Node.js.

  2. 从列表中，选择Node.js运行配置以激活CPU分析或创建新配置，如运行和调试Node.js中所述 。

  3. 切换到V8 Profiling选项卡，然后选中Record CPU profiling info复选框。在“ 日志文件夹”字段中，指定将存储记录日志的文件夹的路径，日志文件名为isolate-<session number> 。

收集CPU分析信息

  1. 从主工具栏上的列表中选择运行配置，然后单击或选择运行|从主菜单中运行<配置名称> 。

  2. 执行需要配置的方案时，单击以停止该过程在工具栏上。

分析CPU分析日志

当您停止应用程序时，PyCharm会自动打开V8 Profiling工具窗口并在其中显示收集的分析数据。如果窗口已打开并显示另一个会话的收集数据，PyCharm将打开一个新选项卡。自动打开的选项卡以控制应用程序执行和收集分析数据的运行配置命名。

要打开并分析以前保存的某些性能分析数据，请转到主菜单上的V8性能分析 - 分析V8性能分析日志 ，然后选择相关的isolate-<session number> 。PyCharm创建一个单独的选项卡，其中包含所选日志文件的名称。

根据收集的分析数据，PyCharm构建三个调用树并在单独的窗格中显示每个调用树。使用这些调用树，您可以从两个不同的角度分析应用程序执行：一方面，哪些调用是耗时的（“重”），另一方面，“谁叫谁”。

了解调用树中的度量标准

调用树使用Total和Self度量标准，这些度量标准显示函数中的滴答数或其与总执行时间的比率：

  • Total显示在函数内部花费了多少时间以及它调用的函数。

  • Self表示仅在函数内部花费了多少时间而没有考虑其子节点。

Of Parent度量标准显示函数的纯执行时间与调用它的函数（ Parent ）的执行时间之比。

V8优化器

在某些情况下，V8可以优化您的代码，有关详细信息，请参阅优化V8 ：

  • 函数名称前面的星号（ * ）表示该函数已经过优化。

  • 波浪号（ ~ ）表示该功能可能需要优化但由于某些原因尚未优化。如果代码短时间运行，引擎可以推迟优化或跳过它，但是代码可以重写的位置处的代数可以获得更好的性能。

分析内存快照

拍摄快照并选择分析时，PyCharm会打开V8堆工具窗口，其中包含收集的数据。如果窗口已打开并显示另一个会话的收集数据，PyCharm将打开一个新选项卡。

工具窗口有三个选项卡，用于显示来自不同视点的收集信息。

  • 在遏制选项卡显示在几个顶级项分组应用程序中的对象：DOMWindow对象 ， 本地浏览器的对象 ，GC和根 ，这是根的垃圾收集器实际使用。有关详细信息，请参阅遏制视图 。

    对于每个对象，选项卡显示其与GC根的距离 ，即对象和GC根之间节点的最短简单路径，对象的浅大小以及对象的保留大小 。除了对象大小的绝对值外，PyCharm还显示了对象占用的内存百分比。

  • “ 最大对象”选项卡显示按保留大小排序的内存最多的对象。在此选项卡中，您可以发现通过在某个全局对象中累积数据而引发的内存泄漏。

  • “ 摘要”选项卡显示应用程序中按类型分组的对象。该选项卡显示每种类型的对象数，它们的大小以及它们占用的内存百分比。该信息可能是存储器状态的线索。

每个选项卡都有一个“ 详细信息”窗格，该窗格显示GC根目录中当前所选对象的路径以及对象的保留器列表，即保留指向所选对象的链接的对象。每个堆快照都有许多“后退”引用和循环，因此每个对象总是有许多保留器。

同时处理多个功能

有时你需要在未完成的事情之间切换不同的任务，然后返回给他们。PyCharm为您提供了一些方便的方法来处理几个不同的功能而不会丢失您的工作：

  • 您可以存储或搁置挂起的更改。

    存储变化与搁置非常相似。唯一的区别在于生成和应用补丁的方式。Stas由Git生成，可以在PyCharm内部或外部应用。具有搁置更改的修补程序由PyCharm生成，并且也通过IDE应用。此外，存储涉及所有未提交的更改，而当您将更改添加到工具架时，您可以选择一些本地更改，而不是将它们全部搁置。

  • 您可以在不同的更改列表中保留与不同任务或功能相关的更改 。

  • 您可以创建分支以处理不同的不相关功能 。

搁置变化

搁置暂时存储您尚未提交的挂起更改。这很有用，例如，如果您需要切换到另一个任务，并且您希望将更改设置为稍后处理它们。

使用PyCharm，您可以搁置单独的文件和整个更改列表 。

搁置后，可根据需要多次更改。

搁置变化

  1. 打开“ 版本控制”工具窗口（ Alt + 9 ）并切换到“ 本地更改”选项卡。

  2. 右键单击要放入工具架的文件或更改列表，然后从上下文菜单中选择“ 搁置更改 ”。

  3. 在“ 搁置更改”对话框中，查看已修改文件的列表。

  4. 在“ 提交消息”字段中，输入要创建的货架的名称，然后单击“货架更改”按钮。

您还可以静默搁置更改，而不显示“ 搁置更改”对话框。要执行此操作，请选择要搁置的文件或更改列表，然后单击Shelve Silently图标在工具栏上，或按Ctrl + Alt + H.包含要搁置的更改的更改列表的名称将用作货架名称。

为了避免使用具有相同名称的多个货架（例如， 默认值 ），您可以将文件或更改列表从“ 本地更改”选项卡拖到“ 货架”选项卡，等待一秒钟直到它被激活，然后编辑新货架释放鼠标按钮时即时命名。

取消改变

令人不安的是将推迟的更改从一个架子转移到一个待定的更改列表。可以从视图中过滤掉未删除的更改或从工具架中删除。

  1. 版本控制工具窗口的保存选项卡，选择您要取消搁置一个变更或文件。

  2. 按Ctrl + Shift + U或从选择的上下文菜单中选择Unshelve 。

  3. 在打开的“ 取消保护更改”对话框中，在“ 名称”字段中指定要将未经保留的更改还原到的更改列表。您可以从列表中选择现有的更改列表，也可以键入要创建的包含未更改的更改的新更改列表的名称。您可以在“ 注释”字段中输入新更改列表的说明（可选）。

    如果要激活新的更改列表，请选择“ 设置活动”选项 。否则，当前活动的更改列表仍保持活动状态。

  4. 如果希望PyCharm在其停用时保留与新更改列表关联的任务的上下文 ，并在更改列表变为活动状态时还原上下文，请选择“ 跟踪上下文”选项（有关详细信息，请参阅任务和上下文 ）。

  5. 如果要删除要撤消的更改，请选择“ 从架子中删除已成功应用的文件”选项。未加载的文件将从此工具架中删除，并添加到另一个更改列表并标记为已应用。在通过单击明确删除之前，不会完全删除它们在工具栏上，或从上下文菜单中选择Clean Already Unshelved 。

您可以静默取消更改，而不显示“ 取消保护更改”对话框。要做到这一点，选择文件或要取消搁置一个变更，并单击取消搁置默默图标在工具栏上，或按Ctrl + Alt + U.未被控制的文件将被移动到活动的待定更改列表。

将组更改为不同的更改列表

优点 ：

  • 您可以轻松地在不同的逻辑变更集之间切换，并将它们彼此分开提交。

  • 与为相同目的使用分支不同，您可以手头进行所有更改，而无需在分支之间切换，如果您的项目非常大，可能需要一段时间。

  • 测试不同功能如何协同工作很方便。

  • 您可以在构建服务器上远程运行更改列表。

缺点 ：

  • 虽然使用更改列表可能看起来比分支更轻量级，但它并不安全，因为在您提交并推送它们之前，您的更改没有备份。如果您的本地工作副本出现问题，您的所有更改都将丢失，因为它们不属于Git项目历史记录。

  • 不可能对特征进行原子测试。

  • 不可能在同一功能上进行协作。此外，除非您通过电子邮件发送包含更改的修补程序，否则您无法从不同的计算机中做出贡献，这可能不太方便。

使用功能分支

Git中的一个分支代表了一个独立的开发线，所以如果您正在开发一个单独的功能，您希望在准备好共享工作结果并将它们集成到master之前完成并进行测试，那么在功能分支中执行它是最好的解决方案。这样，您可以确保不会将不稳定的代码提交到项目的主代码库，并且可以在必要时轻松切换到其他任务。

优点 ：

  • 与使用更改列表进行组更改相反，使用功能分支是安全的。在您向Git提交更改后，它们将成为Git项目历史记录的一部分，因此即使您损坏了工作树，也可以通过Git reflog恢复提交。推送更改后，它们将被备份。

  • 您可以开发并行的非相关功能并以原子方式对其进行测试。

  • 当您在分支中完成开发后，您可以重新排序或压缩提交 ，以便您的历史记录线性且干净。

  • 您可以轻松地在功能上进行协作，或者从不同的计算机上进行开发。

缺点 ：

  • 在真正大型项目上切换分支可能需要一些时间。

  • 一起测试相关功能并不是很方便。

  • 您必须学习使用功能分支并将更改集成到主代码库中的工作流程 。

使用功能分支并将更改集成到主代码库中有两种主要方法：

  • 合并选项

  • rebase选项

使用merge来集成功能分支中的更改

合并选项的主要好处是完全可跟踪性，因为提交合并到主代码库中保留其原始哈希和作者，并且属于一个功能的所有提交可以组合在一起。

此工作流适用于对主代码库提交更改涉及拉取请求或分层审批过程的项目，因为现有分支不会以任何方式更改。

这种方法的主要缺点是每次需要合并更改时都会创建无关的合并提交，这会严重污染项目历史记录并使其难以阅读。

  1. 为您的独立开发线创建一个分支 。

  2. 在开发过程中提交您的更改。

  3. 将分支推送到远程存储库。这应该用于备份，以便您可以在不同的计算机上进行协作或工作。

  4. 当您需要执行与您的功能无关的工作时，请切换到其他分支。

  5. 对您的功能进行审核和测试，并进行必要的修复。

  6. 当您准备将工作结果集成到主分支（例如master ）时，请执行以下操作：

    • 将您的功能分支合并到主代码库中。

    • 删除功能分支 。

    • 推。

使用rebase集成功能分支中的更改

此选项的主要好处是，您可以获得清晰的项目历史记录，以便其他人阅读和理解。您的日志不包含merge操作产生的不必要的合并提交，并且您可以获得易于导航和搜索的线性历史记录。

在决定采用此工作流程时，您应该记住， rebase重写项目历史记录，因为它会为原始要素分支中的每个提交创建新提交，因此它们将具有不同的哈希值，这会阻碍可跟踪性。

  1. 为您的独立开发线创建一个分支 。

  2. 在开发过程中经常提交更改。

  3. 将分支推送到远程存储库。这应该用于备份，以便您可以在不同的计算机上进行协作或工作。

  4. 衍合你的特性分支到master不时。如果您的功能分支很长，那么这样做才有意义。这对以下内容很有用：

    • 确保你的功能分支和master不会分开太远。

    • 当您最终将更改集成到主代码库中时，避免解决大量冲突。当你经常变换时，你可以迭代地解决冲突，并且不会因为长时间的差异而挣扎。

    • 加快检查分支，因为分支之间的切换一旦充分发散就会变慢。

    重新定位涉及以下步骤：

    • 从远程获取更改，或将更改提取到master分支。

    • 衍合你的分支到master 。

    • 强制将 rebase操作的结果推送到功能分支。

  5. 当您需要执行与您的功能无关的工作时，切换到master 。当您返回功能分支时，执行Checkout和Rebase to Current 。

  6. 对您的功能进行审核和测试，并进行必要的修复。

  7. 功能完成后执行Interactive Rebase 。这允许您重新排序和压缩提交，以使您的功能分支历史看起来漂亮和干净。

  8. 当您准备将工作结果集成到主分支（例如master ）时，请执行以下操作：

    • 签出master分支。

    • 将您的分支与master 合并 。由于master没有发散，Git只会将指针移动到功能分支的最新提交，而不是创建新的合并提交（这称为快进合并）。

    • 删除功能分支 。

    • 推 。

使用首字母或昵称（如果它很短）作为功能分支名称的前缀是有意义的。这样，您可以使用“ 分支”菜单中的速度搜索轻松找到所有分支。

将组更改为不同的更改列表

当您处理多个相关功能时，您可能会发现将更改分组到不同的更改列表中非常方便。这种方法有其优点和缺点，而不是使用功能分支来处理多个任务。

优点 ：

  • 您可以轻松地在不同的逻辑变更集之间切换，并将它们彼此分开提交。

  • 与为相同目的使用分支不同，您可以手头进行所有更改，而无需在分支之间切换，如果您的项目非常大，可能需要一段时间。

  • 测试不同功能如何协同工作很方便。

  • 您可以在构建服务器上远程运行更改列表。

缺点 ：

  • 虽然使用更改列表可能看起来比分支更轻量级，但它并不安全，因为在您提交并推送它们之前，您的更改没有备份。如果您的本地工作副本出现问题，您的所有更改都将丢失，因为它们不属于Git项目历史记录。

  • 不可能对特征进行原子测试。

  • 不可能在同一功能上进行协作。此外，除非您通过电子邮件发送包含更改的修补程序，否则您无法从不同的计算机中做出贡献，这可能不太方便。

所有更改列表都显示在“ 版本控制”工具窗口的“ 本地更改”选项卡中。所有已修改的文件都会自动放入活动更改列表中，这是默认更改列表，除非您创建了另一个并使其处于活动状态。

要创建新的更改列表，请单击在工具栏上。

要激活非默认更改列表，请右键单击它并从上下文菜单中选择“ 设置活动更改列表”。

使用功能分支

Git中的一个分支代表了一个独立的开发线，所以如果您正在开发一个单独的功能，您希望在准备好共享工作结果并将它们集成到master之前完成并进行测试，那么在功能分支中执行它是最好的解决方案。这样，您可以确保不会将不稳定的代码提交到项目的主代码库，并且可以在必要时轻松切换到其他任务。

优点 ：

  • 与使用更改列表进行组更改相反，使用功能分支是安全的。在您向Git提交更改后，它们将成为Git项目历史记录的一部分，因此即使您损坏了工作树，也可以通过Git reflog恢复提交。推送更改后，它们将被备份。

  • 您可以开发并行的非相关功能并以原子方式对其进行测试。

  • 当您在分支中完成开发后，您可以重新排序或压缩提交 ，以便您的历史记录线性且干净。

  • 您可以轻松地在功能上进行协作，或者从不同的计算机上进行开发。

缺点 ：

  • 在真正大型项目上切换分支可能需要一些时间。

  • 一起测试相关功能并不是很方便。

  • 您必须学习使用功能分支并将更改集成到主代码库中的工作流程 。

使用功能分支并将更改集成到主代码库中有两种主要方法：

  • 合并选项

  • rebase选项

使用merge来集成功能分支中的更改

合并选项的主要好处是完全可跟踪性，因为提交合并到主代码库中保留其原始哈希和作者，并且属于一个功能的所有提交可以组合在一起。

此工作流适用于对主代码库提交更改涉及拉取请求或分层审批过程的项目，因为现有分支不会以任何方式更改。

这种方法的主要缺点是每次需要合并更改时都会创建无关的合并提交，这会严重污染项目历史记录并使其难以阅读。

  1. 为您的独立开发线创建一个分支 。

  2. 在开发过程中提交您的更改。

  3. 将分支推送到远程存储库。这应该用于备份，以便您可以在不同的计算机上进行协作或工作。

  4. 当您需要执行与您的功能无关的工作时，请切换到其他分支。

  5. 对您的功能进行审核和测试，并进行必要的修复。

  6. 当您准备将工作结果集成到主分支（例如master ）时，请执行以下操作：

    • 将您的功能分支合并到主代码库中。

    • 删除功能分支 。

    • 推。

使用rebase集成功能分支中的更改

此选项的主要好处是，您可以获得清晰的项目历史记录，以便其他人阅读和理解。您的日志不包含merge操作产生的不必要的合并提交，并且您可以获得易于导航和搜索的线性历史记录。

在决定采用此工作流程时，您应该记住， rebase重写项目历史记录，因为它会为原始要素分支中的每个提交创建新提交，因此它们将具有不同的哈希值，这会阻碍可跟踪性。

  1. 为您的独立开发线创建一个分支 。

  2. 在开发过程中经常提交更改。

  3. 将分支推送到远程存储库。这应该用于备份，以便您可以在不同的计算机上进行协作或工作。

  4. 衍合你的特性分支到master不时。如果您的功能分支很长，那么这样做才有意义。这对以下内容很有用：

    • 确保你的功能分支和master不会分开太远。

    • 当您最终将更改集成到主代码库中时，避免解决大量冲突。当你经常变换时，你可以迭代地解决冲突，并且不会因为长时间的差异而挣扎。

    • 加快检查分支，因为分支之间的切换一旦充分发散就会变慢。

    重新定位涉及以下步骤：

    • 从远程获取更改，或将更改提取到master分支。

    • 衍合你的分支到master 。

    • 强制将 rebase操作的结果推送到功能分支。

  5. 当您需要执行与您的功能无关的工作时，切换到master 。当您返回功能分支时，执行Checkout和Rebase to Current 。

  6. 对您的功能进行审核和测试，并进行必要的修复。

  7. 功能完成后执行Interactive Rebase 。这允许您重新排序和压缩提交，以使您的功能分支历史看起来漂亮和干净。

  8. 当您准备将工作结果集成到主分支（例如master ）时，请执行以下操作：

    • 签出master分支。

    • 将您的分支与master 合并 。由于master没有发散，Git只会将指针移动到功能分支的最新提交，而不是创建新的合并提交（这称为快进合并）。

    • 删除功能分支 。

    • 推 。

使用首字母或昵称（如果它很短）作为功能分支名称的前缀是有意义的。这样，您可以使用“ 分支”菜单中的速度搜索轻松找到所有分支。

节点解释器

在此字段中，指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

使用Spy-js，建议使用Node.js版本0.10.24或更高版本。

若要启用跟踪的ECMAScript 6个脚本，使用Node.js的版本0.11.13或更高，并指定--harmony作为节点的参数。

节点参数

在此字段中，键入要传递给NodeJS可执行文件的Node.js特定命令行选项。例如，要启用跟踪ECMAScript 6脚本，请将--harmony指定为Node参数。请注意，Node.js必须为0.11.13或更高版本。

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

JavaScript文件

在此字段中，指定启动它的应用程序的主文件的路径（例如， Node.js Express应用程序的 bin / www ）。

如果要跟踪CoffeeScript，请指定生成的JavaScript文件的路径。该文件可以在外部生成，也可以使用文件监视器进行编译。有关更多详细信息，请参阅调试CoffeeScript 。

应用参数

在此文本框中，键入要通过process.argv数组启动时传递给应用程序的Node.js特定参数。

环境变量

在此字段中，指定Node.js可执行文件的环境变量 （如果适用）。单击“ 浏览” 在该字段的右侧并在“ 环境变量”对话框中配置变量列表，该对话框将打开：

  • 要定义新变量，请单击“ 添加” 并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击“ 删除” 。

  • 准备好后，单击“ 确定”

变量的定义显示在“ 环境变量只读”字段中，并以分号作为分隔符。可接受的变量是：

  • NODE_PATH ：A :以模块搜索路径为前缀的NODE_PATH目录列表。

  • NODE_MODULE_CONTEXTS ：设置为1以在其自己的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

配置文件

从此下拉列表中，选择具有配置设置的文件以应用于跟踪会话。

配置文件是扩展名为.js或.conf.js的JavaScript文件，其中包含符合Spy-js配置要求的有效JavaScript代码。如果IntelliJ IDEA在项目中检测到扩展名为.conf.js的文件，则这些文件将显示在下拉列表中。

手动键入配置文件的路径，或单击“ 浏览” 并在打开的对话框中选择位置。指定后，配置文件将添加到下拉列表中，以便下次从列表中获取而不是指定路径。

使用镜头模式

镜头模式可让您预览代码而无需实际滚动。将鼠标悬停在滚动条上时，默认情况下该模式在编辑器中可用。

  • 或者，在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|一般|外观并清除滚动条悬停复选框上的显示代码镜头 。

  • 要编辑面包屑设置，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中转到“ 编辑器”|一般|面包屑 。

查找最近的文件

您可以使用“ 最近的文件”弹出窗口搜索最近和最近编辑的文件。

  • 要使用最近文件列表打开“ 最近使用的文件”弹出窗口，请按Ctrl + E.

  • 要仅查看最近编辑的文件，请再次按Ctrl + E或选择“ 仅显示更改”复选框。

  • 要在弹出窗口中搜索项目，请开始键入搜索查询。WebStorm根据您的搜索显示结果。

查找最近的位置

您可以使用“ 最近的位置”弹出窗口检查最近访问过的或更改过的代码位置。

  • 要打开“ 最近的位置”弹出窗口，请按Ctrl + Shift + E.该列表从顶部的最新访问位置开始，包含代码段。

  • 在弹出窗口中，使用相同的快捷方式或选择“ 仅显示已更改”复选框以仅查看已更改代码的位置。

  • 要搜索代码段，请在“ 最近的位置”弹出窗口中，开始键入搜索查询。您可以按代码文本，文件名或面包屑进行搜索。

  • 要从搜索结果中删除位置条目，请按Delete或Backspace 。

    请记住，删除的位置也会从您使用Ctrl + Alt + Left快捷键访问的条目列表中删除。
