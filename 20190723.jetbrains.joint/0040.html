运行/调试配置：插件

使用此对话框创建运行/调试配置，以用于运行和调试插件项目。

项目

描述

VM选项

指定传递给VM的字符串以启动插件。通常此字符串包含-mx ， -verbose等选项。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如-client -ea -Xmx1024m 。

  • 如果选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg或"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠转义双引号，例如-Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果定义变量MY_ENV_VAR ，则可以将其传递给foo属性，如下所示：

    - Dfoo = $ { MY_ENV_VAR }

程序参数

键入要传递给程序的参数列表，就像在命令行中输入这些参数一样。

使用与指定VM选项相同的规则。

使用模块的classpath

选择应使用其类路径运行应用程序的模块。

使用替代JRE

选中此复选框以启用定义另一个JRE，而不是当前项目/模块使用的JRE。

显示idea.log

如果选中，则在运行插件时，IntelliJ IDEA会在控制台中显示idea.log文件的内容。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。

工具栏

运行/调试配置的树视图有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

进入新文件夹/创建新文件夹

然后，要将配置移动到文件夹中，文件夹之间或文件夹之外，请使用拖放或和纽扣。

在启动选项之前

在此区域中，您可以指定在启动所选运行/调试配置之前必须执行的任务。任务按照它们在列表中出现的顺序执行。

单击此图标可添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择要运行的一个或多个应用程序。如果尚未在IntelliJ IDEA中定义，请添加其定义。有关更多信息，请参阅配置第三方工具和外部工具 。

  • 运行另一个配置 ：选择执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • Build ：选择编译指定的模块。将执行Build Module命令 。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建项目 ：选择编译整个项目。Build Project命令将被执行。

    如果在编译期间发生错误，IntelliJ IDEA将不会尝试启动运行/调试配置。

  • 构建，无错误检查 ：与Build选项相同，但IntelliJ IDEA将尝试启动运行/调试配置，而不管编译结果如何。

  • 构建工件 ：选择此选项可构建工件或工件。在打开的对话框中，选择应构建的工件或工件。

  • 运行Ant目标 ：选择此选项以运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项以运行Grunt任务。在打开的Grunt任务对话框中，指定定义所需任务的Gruntfile.js ，选择要执行的任务，并指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项以运行Gulp任务。在打开的Gulp任务对话框中，指定定义所需任务的Gulpfile.js ，选择要执行的任务，并指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项以运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。在打开的“ NPM脚本”对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择以运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都反映在生成的JavaScript文件中。在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防检测到任何错误：

    • 如果选中“检查错误”复选框，编译器将显示所有错误，并且不会启动运行配置。

    • 如果清除了“ 检查错误”复选框，编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源地图 ：选择此选项可为CoffeeScript源生成源地图。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：在运行或调试之前添加要执行的Rake任务 。要选择Rake任务，请单击“浏览”按钮 ，然后从可用任务列表中选择所需任务。

    请注意，此处可以使用代码完成 。

  • 运行JRuby编译器 ：选择此选项以使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。

Alt + Delete键

否则，如果清除该复选框，则不会显示工具窗口。但是，当配置运行时，您可以通过按Alt + 4或Alt + 5 自行打开相应的工具窗口。

运行，调试和测试Scala

IntelliJ IDEA允许您像通常使用IntelliJ IDEA中的任何其他应用程序一样运行，调试和测试Scala应用程序。IntelliJ IDEA还允许您使用coverage运行Scala代码并配置代码覆盖率设置。

运行Scala应用程序

您可以通过IntelliJ IDEA运行Scala代码，使用sbt shell，或使用Scala工作表进行快速代码评估。

通过Intellij IDEA运行Scala应用程序

  1. 创建或导入Scala项目，就像在IntelliJ IDEA中创建或导入任何其他项目一样。

  2. 在编辑器中打开您的应用程序。

  3. 按Shift + F10执行应用程序。或者，在编辑器的左侧排水沟中，单击图标，然后选择运行'名称' 。

你也可以点击主工具栏上的图标，用于运行您的应用程序。

使用sbt shell运行Scala应用程序

您可以使用sbt shell运行您的应用程序，该shell是任何sbt项目的一部分。

  1. 打开你的sbt项目。

  2. 如果要将构建和导入委托给sbt，请在sbt projects工具窗口中单击图标打开sbt设置。
    在sbt设置页面上，选择“ 使用sbt shell进行构建和导入（必需sbt 0.13.5+）”选项，然后单击“ 确定” 。

    （当您创建或导入sbt项目时，此选项也可用。）
    如果您不想将构建和导入委托给sbt，您仍然可以在sbt shell中工作（ 并直接从它运行sbt命令 ，包括运行您的应用程序。

  3. 在sbt projects工具窗口中，单击sbt Tasks节点。

  4. 在打开的列表中，选择将运行main方法的运行任务。

    执行结果显示在sbt shell工具窗口中。

使用Scala工作表运行Scala代码

IntelliJ IDEA允许您创建Scala工作表以动态评估Scala代码结果。

  1. 右键单击您的项目，然后选择New | Scala工作表 。

    我们建议您在src目录中创建工作表，以简化项目类在类路径中的包含。这可能有助于测试目的。

  2. 在“ 新建Scala工作表”窗口中，键入Scala工作表的名称，然后单击“ 确定” 。

    因此，将打开一个扩展名为.sc的文件。

  3. 输入您的代码，然后按看到结果。

  4. 使用.sc文件时，您可以执行以下操作：

    • 您可以评估Scala对象。

    • 您可以折叠输出而不影响左侧的代码，只展开与特定语句匹配的输出块。

    • 您可以单击设置工作表编辑器工具栏上的图标，并配置将影响其他工作表的整个项目的当前工作表设置或默认工作表设置。

      • 如果要自动查看结果，请选中“ 交互模式”复选框。

      • 清除“生成项目”复选框以提高评估性能。在这种情况下，禁用项目更改的自动检查。

      • 您可以选择“ 运行类型”选项以查看不同的评估模式。例如，选择REPL类型以快速评估Scala表达式。添加新表达式时，它们会以递增方式执行。

      • 请注意， 模块的“ 使用类路径”选项已禁用，因为工作表是一个可以在其他项目之间共享的文件，应该保留在创建它的位置。您可以在默认设置中更改其位置，但它将应用于整个项目，并将影响其他工作表。如果您需要更改类路径，我们建议您创建并使用临时文件 。

      • 您还可以为工作表选择编译器配置文件。点击访问Scala编译器设置以获取更多选项。请记住，如果您通过编译器设置页面添加/更改配置文件并且您有一个sbt项目，那么您的更改将在下一个项目导入时被丢弃。

      • 有关工作表设置的更多选项，请选择“ 文件” | 设置 | 语言和框架 |单击Scala并单击“ 工作表”选项卡。例如，在“ 工作表”选项卡上，您可以设置输出截止限制，指定要在哪种模式下运行工作表，或者配置IntelliJ IDEA如何处理.sc和临时文件。

        如果选中“ 使用eclipse兼容性”模式复选框，则对象内的所有语句都将包含在工作表的输出中。

要删除工作表的结果，请按垃圾箱图标。

要将代码和评估结果复制到一个文件中，请单击图标。

调试Scala代码

IntelliJ IDEA允许您使用IntelliJ IDEA调试器或sbt shell调试代码。

使用Intellij IDEA调试Scala代码

  1. 在编辑器中打开Scala应用程序。

  2. 在左侧装订线中，为要调试的代码行设置断点。有关断点的更多信息，请参阅断点主题。

  3. 如果需要，可以访问运行/调试配置 （运行|编辑配置）并调整设置，但通常默认设置足以成功启动和完成调试会话。

  4. 按Shift + F9 。或者，在主工具栏上单击图标以启动调试过程。

  5. 在“ 调试”工具窗口中评估结果。

    有关如何在“ 调试”工具窗口中使用选项的信息，请参阅调试工具窗口参考 。

如果为run或test任务启用了分叉 ，则通过sbt shell调试任务将不起作用。

使用sbt shell调试Scala代码

  1. 打开你的sbt项目。

  2. 在编辑器中打开您的应用程序。

  3. 在编辑器中，在左侧装订线中，为要调试的代码行设置断点。

    您无法调试实际.sbt文件中定义的代码，但您可以调试可以从build.sbt调用的Scala文件中的代码。

  4. 在sbt projects工具窗口中，单击图标，启动sbt shell。

  5. 在sbt shell工具窗口中，单击用于连接调试器服务器并启动调试会话的图标。
    IntelliJ IDEA还为调试会话创建运行/调试配置 。

    如果需要，可以编辑运行/调试配置设置，但默认设置应足以成功调试代码。

  6. 运行你的程序。

    在“ 调试”工具窗口中评估结果。

空白

使用此列表可以定义差异查看器应如何处理空格。

  • 不要忽略 ：空格很重要，所有差异都会突出显示。默认情况下选择此选项。

  • 修剪空格 :( ("\t", " ") ，如果它们出现在行的末尾和行的开头。

    • 如果两行仅在尾随空格中不同，则这些行被认为是相等的。

    • 如果两行不同，则在“ 按字”模式下不会突出显示此类尾随空格。

  • 忽略空格 ：白色空格并不重要，无论它们在源代码中的位置如何。

  • 忽略空格和空行 ：忽略以下实体：

    • 所有空格（如'忽略空白'选项）

    • 所有添加或删除的行仅包含空格

    • 所有更改包括拆分或连接线而不更改非空白部分。
      例如，在此模式下不会突出显示将abc更改为a \nbc 。

  • 忽略导入和格式化 ： 忽略 import语句和空格内的更改（尽管可以遵守String文字中的空格）。

存储库和传入选项卡仅适用于非分布式版本控制系统（即WebStorm支持的除Git和Mercurial之外的所有VCS）。

“ 存储库”选项卡显示在当前项目中VCS根目录下提交到存储库的更改。“ 传入”选项卡显示其他团队成员提交到存储库的更改，但尚未在本地签出。两个选项卡都显示存储在历史记录缓存中的信息。显示的更改列表的数量取决于缓存范围 。

  • “ 更改列表”窗格显示更改列表。

  • “ 更改的文件”窗格显示在所选更改列表中修改和提交的文件列表。

请注意，如果在服务器和本地工作副本中都使用SVN 1.5或更高版本，则“ 存储库”选项卡还具有“ 合并信息”窗格，该窗格用于配置其他两个窗格中的视图，并提供对分支之间集成的控制。

承诺的变更列表通常对应于跟踪系统中的问题。您可以直接在“ 更改列表”窗格中在浏览器中打开此类问题。此功能具有以下先决条件：

  • 错误跟踪系统的模式在“ 问题导航 设置”对话框中指定。

  • 提交消息中提到了相应的问题编号。

配置问题导航后，提交消息中的问题编号将呈现为链接。单击此链接可转到问题跟踪器的相应页面。

使用PHPUnit进行测试

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA通过与PHPUnit测试框架集成，支持PHP应用程序的单元测试。

在你开始之前

  1. 确保已安装并启用了PHP插件。该插件未与IntelliJ IDEA捆绑在一起，但可以从JetBrains插件存储库中安装，如管理插件中所述 。

  2. 确保PHP解释器在PHP页面上的IntelliJ IDEA中配置 ，如配置本地PHP解释器和配置远程PHP解释器中所述 。

下载并安装PHPUnit

在开始之前，请确保您的计算机上已安装Composer，并在Composer Dependency Manager中所述的当前项目中进行初始化。

手动下载并安装phpunit.phar

  • 从PHPUnit官方网站下载phpunit.phar并将其保存在您的计算机上：

    • 如果除了能够运行PHPUnit测试之外还需要完整的编码帮助，请将phpunit.phar存储在稍后将使用PHPUnit的项目的根目录下。

    • 如果您只需要运行PHPUnit测试并且不需要任何编码帮助，则可以将phpunit.phar保存在项目之外。

使用Composer下载并安装phpunit.phar

  1. 从composer.json的上下文菜单中，选择Composer |管理依赖关系 。或者选择工具|作曲家|从主菜单管理依赖项 。

  2. 在打开的Manage Composer Dependencies对话框中，从Available Packages列表中选择phpunit/phpunit包，可能使用搜索字段。

    从要安装的版本列表中选择相关版本。

  3. 如有必要，请展开“ 设置隐藏”区域并指定高级安装选项。在“ 命令行参数”字段中，键入其他命令行参数。建议提供--dev ，option：在这种情况下，包将被添加到composer.json文件的require-dev部分，而不是默认的require部分。

  4. 单击安装 。

将PHPUnit与IntelliJ IDEA项目集成

如果您使用本地PHP解释器 ，IntelliJ IDEA会自动执行初始PHPUnit配置。

自动配置PHPUnit

  1. 将phpunit.xml或phpunit.xml.dist配置文件存储在项目根目录下。

  2. 使用Composer安装PHPUnit 。

IntelliJ IDEA将在Test Frameworks页面和PHPUnit运行/调试配置上创建本地框架配置。

您可以手动配置PHPUnit。如果使用远程PHP解释器 ，则需要手动配置。

手动配置PHPUnit

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，展开“ 语言和框架”节点，然后选择“ PHP下的测试框架 ”。

    在打开的“ 测试框架”页面上，单击“确定” 在中央窗格中，从列表中选择配置类型：

    • 在本地配置中，使用默认项目PHP解释器 ，有关详细信息，请参阅默认项目CLI解释程序。

    • 要将PHPUnit与远程PHP解释器一起使用，请在打开的对话框中选择一个配置：

  2. 在右侧窗格中，选择PHPUnit库安装类型：

    • 要使用Composer自动加载器，请在vendor文件夹中指定autoload.php文件的路径。有关详细信息，请参阅Composer

    • 要从phpunit.phar运行PHPUnit，请下载phpunit.phar ，将存档保存在项目根文件夹中，并指定它的路径。

      当你点击 ，IntelliJ IDEA检测并显示PHPUnit版本。

    • 要从PEAR运行PHPUnit， 请将PEAR配置为包含路径。

步骤1。检查您的网络设置

数据库可以在本地，服务器或云中运行。对于服务器和云数据库，您需要网络连接。要验证连接是否可用，请使用ping和telnet命令。

使用ping命令，可以确保可以从源计算机访问目标计算机。打开命令行并键入以下命令： ping -a <host_IP> ，其中-a是将地址解析为主机名的命令选项（如果可能）。如果将主机名与ping命令一起使用，则会将主机名解析为IP地址。例如， ping -a example.com解析为PING example.com (93.184.216.34) 。

ping - 一个 < host_IP >

使用telnet命令，您可以测试与远程计算机的连接并发出命令。如果将端口指定为telnet命令的参数，则可以测试与给定端口上的远程主机的连接。如果连接成功，您将看到以下消息： Connected to <host_IP> 。

telnet < host_IP > < port_number >

出于安全原因，DBMS通常会丢弃所有telnet连接。telnet命令允许您检查端口是否已打开以进行通信。

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。
