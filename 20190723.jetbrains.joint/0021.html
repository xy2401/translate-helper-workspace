使用pipfile管理依赖项

Pipfile是Pipenv虚拟环境用于管理项目依赖关系的专用文件。此文件对于使用Pipenv至关重要。为新项目或现有项目创建Pipenv环境时，将自动生成Pip文件 。同样，当您第一次使用IntelliJ IDEA中的Pipfile文件打开项目时，会自动配置Pipenv虚拟环境。

考虑从头开始创建依赖项列表的任务。

记录Pipfile中的依赖项以管理项目包

  1. 当IntelliJ IDEA为新的pipenv虚拟环境创建Pip文件时，该文件如下所示：

    python_version参数是您在创建新的pipenv环境时指定的基本解释器的版本。packages部分是您可以列出项目所需packages的位置。

  2. 请注意，IntelliJ IDEA建议您安装Pipfile特定插件。单击相应的链接以安装Tom's Obvious，Minimal Language（ TOML ）的插件。您必须重新启动IntelliJ IDEA才能启用该插件。重新启动后，您将看到确认已识别Pipfile格式的消息：

  3. 通过修改packages部分添加新的包依赖项。

    [ 包 ] django = “*”

  4. 每当您修改Pipfile文件时，IntelliJ IDEA都会建议您执行以下操作之一：

    • pipenv lock - 将新要求记录到Pipfile.lock文件中。

    • pipenv update - 将新需求记录到Pipfile.lock文件，并在项目解释器上安装缺少的依赖项。

    此步骤非常重要，因为IntelliJ IDEA根据Pipfile.lock文件中记录的信息管理项目依赖性。因此，任何添加到Pipfile但未锁定的要求都将被忽略。

    让我们选择pipenv update命令来安装Django包。

  5. 导航到文件|项目结构 （ Ctrl + Shift + Alt + S ），在平台设置中选择SDK ，然后选择Pipenv SDK。确保Django位于已安装软件包的列表中。

  6. 现在探索相反的工作流程。在“ 包”选项卡中，单击添加Flask包。请注意，pipenv使用标准过程限制配置索引。所有可用的包都是从Pipfile中指定的源添加的。通过pipenv而不是通过pip安装，删除和更新软件包。

  7. 一旦添加了Flask包，关闭Available packages对话框并查看Pipfile 。[packages]部分现在看起来如下：

    [ 包 ] django = “*” flask = “*”

[packages]和[dev-packages]部分之间的区别在于[packages]定义了生产和开发环境的要求，而[dev-packages]仅列出了用于开发目的的要求。

如果不满足Pipfile中列出的任何要求，IntelliJ IDEA会跟踪并建议您应用受影响的依赖项。

创建资源

除了创建文件和文件夹的标准方法之外，您还可以直接从res节点创建资源定义，而无需将它们放在相应的文件夹中。IntelliJ IDEA检测新资源的类型和限定符，并指定将保存资源定义文件的文件夹。您可以创建单独的资源定义文件和整个文件夹。

在本主题中：

  • 创建资源文件夹
  • 在相关资源文件夹中创建资源定义文件

创建资源文件夹

IntelliJ IDEA从资源类型和您选择的限定符组成新资源文件夹的名称，并在res文件夹下创建具有此名称的文件夹。该名称的构建符合Android命名约定。

  1. 在“ 项目”工具窗口中，右键单击存储所有Android资源定义的res节点。选择新建|上下文菜单中的Android资源目录 。将打开“ 新建资源目录”对话框。

  2. 从“ 资源类型”下拉列表中，选择要存储在新文件夹中的资源类型 。所选类型显示在“ 目录名称”字段中。

  3. 执行以下操作之一：

    • 如果您的应用程序不需要与各种Android设备兼容，因此不需要支持多屏幕 ，只需单击“ 确定”即可 。

    • 要提供备用资源 ，请指定定义特定设备配置的资源限定符 。

      • 从Available Qualifiers列表中，选择所需的限定符，然后单击按钮。所选项目将添加到“ 选择的限定符”列表中。根据特定限定符，可以手动指定其值，也可以从“ 选择限定符”旁边的下拉列表中选择所需的值。

      • 要删除限定符，请在“ 选择的限定符”列表中选择它，然后单击按钮。

      IntelliJ IDEA将所有选定的限定符附加到资源类型，并使用短划线字符作为分隔符。结果名称显示在“ 目录名称”字段中。

IntelliJ IDEA在res节点下生成具有复合名称的新文件夹。

在相关资源文件夹中创建资源定义文件

  1. 在“ 项目”工具窗口中，右键单击存储所有Android资源定义的res节点。选择新建|上下文菜单中的Android资源文件 。将打开“ 新建资源文件”对话框。

  2. 在“ 文件名”文本框中指定新资源定义文件的名称 。遵循Android命名准则

  3. 从“ 资源类型”下拉列表中，选择资源类型 。您可以使用向上和向下键盘键从File name文本框直接滚动资源类型列表。

  4. 要启用编码帮助，请在Root元素字段中将http://schemas.android.com/apk/res/android架构指定为xmlns:android的值。

  5. 执行以下操作之一：

    • 如果您的应用程序不需要与各种Android设备兼容，因此不需要支持多屏幕 ，只需单击“ 确定”即可 。

    • 要提供备用资源 ，请指定定义特定设备配置的资源限定符 。

      • 从Available Qualifiers列表中，选择所需的限定符，然后单击按钮。所选项目将添加到“ 选择的限定符”列表中。根据特定限定符，可以手动指定其值，也可以从“ 选择限定符”旁边的下拉列表中选择所需的值。

      • 要删除限定符，请在“ 选择的限定符”列表中选择它，然后单击按钮。

      IntelliJ IDEA将所有选定的限定符附加到资源类型，并使用短划线字符作为分隔符。结果名称显示在“ 目录名称”字段中。

  6. 准备好后 ，单击“ 确定” 。IntelliJ IDEA组成将保存新文件的资源文件夹的名称。该名称是根据指定的资源类型和符合Android命名约定的限定符构建的。如果此类文件夹已存在，IntelliJ IDEA会将新文件保存在其中。如果找不到这样的文件夹，IntelliJ IDEA会在res节点下创建它，并在那里保存新的资源定义文件。

PyCharm 2019.1帮助

缩小JavaScript

此功能仅在专业版中受支持。此版本是商业版，它提供了一系列出色的功能：有关详细信息，请参阅比较矩阵 。

PyCharm与Closure Compiler ， YUI Compressor和UglifyJS集成，可以动态压缩JavaScript应用程序源。

缩小或压缩意味着删除所有不必要的字符，例如空格，换行，注释而不更改源代码的功能。在开发和调试期间，这些字符使代码更易于阅读。在生产阶段，他们只增加要传输的代码的大小。

在你开始之前

  1. 下载并安装Node.js.

  2. 如果您打算使用YUI Compressor或Closure Compiler，请下载并安装Java Runtime Environment（JRE） ：

    • YUI Compressor的JRE 1.4或更高版本。

    • 闭包编译器的JRE版本7。

安装缩小工具

  • 要安装UglifyJS，请打开内置PyCharm 终端 （ Alt + F12 ）并键入npm install uglify-js或npm install uglify-js@<version> 。从UglifyJS官方网站了解更多信息。

  • 要安装YUI Compressor，请按照YUI Compressor官方网站上的说明进行操作。

  • 要安装Closure Compiler，请按照Closure Compiler官方网站上的说明进行操作。

压缩代码

要动态压缩代码，您需要将缩小工具配置为PyCharm 文件观察器 。

创建文件监视器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 文件监视器 ”。“ 文件监视器”页面将打开。

  2. 点击并从列表中选择相关的文件观察器模板。

  3. 在打开的New Watcher对话框中，在Program字段中指定相关可执行文件或.jar存档的路径：

    • compiler.jar为使用Closure Compiler。

    • YUI Compressor JS的yuicompressor-<version>.jar 。

    • uglifyjs.cmd for UglifyJS。

  4. （可选） 自定义文件观察程序的行为 。

  5. 确保选中File Watcher旁边的复选框，表示已启用File Watcher。

运行压缩机

启用缩小文件观察程序后，只要更改或保存文件观察程序范围中的JavaScript文件，就会自动开始缩小。

PyCharm使用生成的输出创建一个单独的文件。该文件具有源JavaScript文件的名称和扩展名min.js生成的文件的位置在“ 新建观察器”对话框的“ 输出路径刷新”字段中定义。但是，在项目树中 ，默认情况下它显示在源JavaScript文件下，该文件现在显示为节点。要更改默认表示，请在“ 项目”工具窗口中配置文件嵌套 。

配置配置文件

检查配置文件保留有关已启用检查的信息，它们分析的文件范围及其严重性设置。在IntelliJ IDEA中，有两种类型的配置文件。

  • 存储在IDE中的配置文件保存在IntelliJ IDEA 配置目录中 ，可用于任何项目。

  • 项目中存储的配置文件保存在.idea目录中，并在一个项目中可用。

IntelliJ IDEA附带两种默认的每种类型的配置文件（ 默认和默认项目 ），您可以自定义或创建新的。您还可以共享，导入和导出检验配置文件。

创建新的个人资料

要创建新配置文件，请复制其中一个默认配置文件并更改其设置。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|检查 。

  2. 选择要复制的配置文件，单击 ，然后选择复制 。

  3. 更改配置文件名称，然后按Enter键 。

恢复默认配置文件配置

您可以恢复默认检查设置：默认情况下启用检查及其严重性级别。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|检查 。

  2. 从列表中选择所需的配置文件，单击并选择恢复默认值 。

在计算机之间同步配置文件

存储在项目中的配置文件与其他项目配置文件一起保存，并通过VCS共享。这就是为什么在配置新的项目配置文件并与团队的其他成员共享后，此配置文件将在从VCS签出项目后自动传播到他们的项目。

如果要共享存储在IDE中的配置文件 ，可以使用与IntelliJ IDEA捆绑在一起的Settings Repository插件。

在这种情况下，您必须将此配置文件配置为所有新创建的项目的默认检查配置文件（使其成为全局）。该插件将此全局配置提升到连接到设置存储库的所有计算机。

共享存储在IDE中的配置文件

  1. 确保已配置设置存储库 。

  2. 从主菜单中，选择File |其他设置|新项目的设置 ，然后单击编辑器|检查 。

  3. 在“ 配置文件”列表中，选择要共享的配置文件，应用更改并关闭对话框。

  4. 从主菜单中，选择File |设置存储库 ，然后单击“ 覆盖远程”以将新配置发送到存储库。

  5. 在连接到存储库的另一台计算机上，转到文件|设置存储库并选择“ 覆盖本地”以下载并应用配置。

导出和导入配置文件

要共享配置文件，您还可以将其导出，然后将其导入另一台计算机。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|检查 。

  2. 点击然后选择导出 。

  3. 要导入配置文件，请单击然后选择导入配置文件

  4. 指定配置文件的路径，然后单击“ 打开” 。

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install --global flow-bin全局安装Flow。

    • npm install --save-dev flow-bin将Flow作为开发依赖项安装。

在IntelliJ IDEA中配置流程

要让IntelliJ IDEA识别Flow结构，提供正确的语法突出显示，正确报告错误并避免误报错误突出显示，将JavaScript语言级别更改为Flow ，将.flowconfig配置文件添加到项目中，并提供要检查的每个文件在// @flow注释顶部。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 语言和框架”| JavaScript 。JavaScript页面打开。

    • 类型检查：选中此复选框后，将根据从Flow服务器接收的数据提供语法和错误突出显示。清除该复选框后，只有基本的内部IntelliJ IDEA突出显示可用。

    • 导航，代码完成和类型提示：选中此复选框时，参考分辨率和代码完成的建议列表包含从与Flow集成检索的建议和IntelliJ IDEA计算的建议。清除该复选框后，仅通过IntelliJ IDEA计算解析引用。

    仅当指定了Flow可执行文件的路径时，复选框才可用。

  5. 保持选中“ 自动保存所有已修改文件”复选框以确保连续应用Flow，因为Flow仅在保存所有其他修改文件后才检查当前文件。

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    flow init

在文件中启用Flow

  • 在其顶部添加// @flow注释：只需键入flow ，按Tab键 ，IntelliJ IDEA将其扩展为// @flow 。

  • 团队成员可以通过VCS访问共享范围 ，并将其存储在项目级别。在.idea下的scopes目录中，作为具有xml扩展名的文件，即.idea / scopes / <scope_name> .xml 。

当地历史

WebStorm使用本地历史记录不断跟踪对项目所做的所有更改。与显示特定提交之间差异的版本控制系统不同， 本地历史记录会自动维护所有有意义事件的修订。这会生成项目结构和源代码更改的详细时间表，并能够在必要时回滚到任何点。

本地历史不能取代适当的源代码控制。

安装新版本的WebStorm或使缓存无效时，它将被清除。它具有保留期和最大大小，因此不能保证修订永久存在。

要查看本地历史记录：

  • 在VCS菜单上，指向“ 本地历史记录” ，然后单击“ 显示历史记录” 。

签入策略是在每次签入之前执行的规则，以确保所选的变更集可以提交。标准策略存储在服务器上，并在客户端计算机上执行。

自定义策略实现为IntelliJ IDEA的自定义插件。这些插件的ID存储在服务器上，而策略本身则在本地应用。因此，要在团队中使用策略，所有团队成员都应安装相应的插件。

在本页面：

  • 定义默认策略设置
  • 禁止在当前项目中使用默认的IntelliJ IDEA范围的签入策略设置
  • 管理可用策略列表
  • 引入自定义签到策略

    • 警告未安装的策略：选中此复选框可在未安装指定的策略定义的情况下显示警告。
