附加到流程

PyCharm可以连接到Python本地进程，同时运行从您的操作系统或使用PyCharm终端启动的Python脚本，但不是在调试模式下。请注意，您无法附加到远程进程。

Ubuntu用户的先决条件

如果您在Ubuntu上使用PyCharm（或者可能在某些其他Linux发行版上），则在您第一次尝试附加到进程时可能会出现错误消息ptrace: Operation not permitted 。在这种情况下，请执行以下操作之一：

  • 要临时禁用此限制，请输入命令：

    echo 0 | sudo tee / proc / sys / kernel / yama / ptrace_scope

  • 要永久禁用此限制，请打开/etc/sysctl.d/10-ptrace.conf文件进行编辑，并将kernel.yama.ptrace_scope = 1行更改为kernel.yama.ptrace_scope = 0 。
    要立即应用更改，请输入以上命令echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope 。或者，运行sudo service procps restart或只是重新启动系统。

你可以在这里找到更多细节。

  1. 从主菜单中，选择“运行”|附加到Process或按Ctrl + Alt + F5 。

  4. 完成后，分离进程：选择Run |停止或单击“ 停止进程”按钮 调试工具窗口 。

XPath检查

XPath语言实现提供了一些内置检查，可以在交互模式和编写XSLT脚本时编写XPath表达式时检查常见的编码错误。这些检查还提供了许多配置选项，可以在“设置”对话框的“ 检查”页面上进行配置。

由于这些检查的实施和集成方式，它们只适用于即时编辑器突出显示，而不是通过代码运行检查。检查代码 。

XPath类型检查

有两个检查处理XPath表达式中的类型转换 ： 隐式类型转换和冗余类型转换 。

隐式类型转换

此检查检查预定义的XPath类型STRING，NUMBER，BOOLEAN和NODESET之间的任何隐式转换。虽然这通常不是问题，因为标准很好地定义了转换，但这种检查可以帮助编写更具表现力的XSLT脚本，甚至可以帮助避免细微的错误：

<xsl:if test="foo" />与<xsl:if test="string(foo)" />

第一个测试检查元素foo是否存在（ count(foo) > 0) ，但后者只有在元素实际包含任何文本（ string-length(foo) > 0 ）时才为真。然后，该插件将提供更明确的类型转换。

通过提供单独启用某些类型之间的隐式转换的可能性，有几种选项可以根据个人喜好调整检查。

也可以告诉插件始终标记不会产生实际预期类型的显式转换，例如<xsl:if test="number(foo)" />并提供一个特殊选项来忽略从NODESET到BOOLEAN的转换通过使用string()函数作为写入string-length() > 0的快捷方式。

冗余类型转换

此检查检查具有函数string() ， number()或boolean()任何类型转换是否冗余，即参数类型是否与函数返回类型相同，或者表达式的预期类型是否为类型任何 。虽然这种显式转换有时可能是为了强调类型，但通常可以安全地删除。

表达有效性检查

这些检查检查表达式是否包含任何潜在的语义错误，例如引用实例文档中未出现的元素/属性名称或使用不可能匹配任何内容的谓词。

检查节点测试

此检查检查XPath表达式中使用的任何元素/属性名称是否实际上是关联XML文件的一部分，或者是否在引用的模式中定义。这有助于避免在运行脚本时可能发生的XPath表达式中的拼写错误导致的问题，甚至可能无法立即识别。

例：

<xsl：template match =“<keyword> h：txtarea </ keyword>”/>

如果前缀h绑定到XHTML名称空间，则检查会将match-expression的这一部分标记为未知元素名称，因为元素的正确名称是textarea 。

索引零使用

此检查检查谓词索引中的任何意外使用零或与函数position()的比较。这几乎总是一个bug，因为在XPath中，索引从1开始， 而不是从0开始。

例：

//someelement[position() = 0]或//something[0]

开发自定义XPath检查

XPath检查使用WebStorm的常规检查API。但是，由于XPath语言支持的集成方式，这有点复杂，而且目前还不可能开发全面的第三方XPath检查。虽然理论上可以开发使用XPath-PSI API并从org.intellij.lang.xpath.validation.inspections.派生的自定义检查org.intellij.lang.xpath.validation.inspections.XPathInspection ，不建议这样做，也不支持。

部署您的应用程序

在配置开发和生产环境的众多方法中，最常见的方法如下：

  • Web服务器安装在您的计算机上。源位于服务器文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • Web服务器安装在您的计算机上，但源存储在另一个文件夹中。您进行开发，然后将源复制到服务器。

  • Web服务器位于另一台计算机（远程主机）上。服务器上的文件可通过FTP / SFTP / FTPS协议，网络共享或已安装的驱动器获得。

WebStorm假定所有开发，调试和测试都在您的计算机上完成，然后将代码部署到生产环境中。

坚持这种“本地开发 - 部署”模式的原因在于WebStorm提供编码协助的方式，包括代码完成，代码检查和验证，代码导航等。所有这些功能都基于项目文件索引 ，WebStorm在加载项目时构建索引 ，并在编辑代码时动态更新。

为了提供有效的编码帮助，WebStorm需要快速重新编码代码，这需要快速访问项目文件。后者只能用于本地文件，即存储在硬盘上并可通过文件系统访问的文件。

WebStorm与服务器之间的交互

WebStorm与服务器之间的交互是通过服务器访问配置来控制的。无论您何时使用服务器，都需要定义服务器访问配置 ，无论您的服务器是在远程主机上还是在您的计算机上。

考虑到以上所有因素，让我们定义以下与WebStorm和服务器之间的同步相关的基本概念。

  • 就地服务器是一个服务器，其文档根目录是项目根目录的父服务器，无论是否立即生效。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • 本地服务器是在本地或装入的文件夹中运行的服务器，其文档根目录 不是项目根目录的父级。您进行开发，然后将源复制到服务器。

  • 远程服务器是在另一台计算机（远程主机）上运行的服务器。可以通过FTP / SFTP / FTPS协议访问服务器上的文件。

  • 服务器配置根是可通过服务器配置访问的本地或远程服务器上文件树中的最高文件夹。对于就地服务器，它是项目根目录。

  • 本地文件/文件夹是项目根目录下的任何文件或文件夹。

  • 远程文件/文件夹是服务器上的任何文件或文件夹。

  • 上传是将数据从项目复制到服务器（本地或远程）。

  • 下载是将数据从服务器复制到项目。

配置与服务器的同步后，您可以直接从WebStorm上载，下载和管理文件。此外，您可以禁止上载或下载特定文件或整个文件夹。最后，您可以通过配置内容根来优化工作流，这样特定文件夹就不会参与索引，从而显着节省了项目索引时间。

通过Remote Hosts Access捆绑插件提供与服务器同步，上载，下载和管理文件，默认情况下启用该插件。如果禁用该插件 ，请在“设置”对话框的“ 插件”页面中将其激活。有关详细信息，请参阅管理插件 。

WebStorm 2019.1帮助

设置本地Mercurial存储库

尽管Mercurial在安排数据和使用存储库工作方面提供了高度灵活性，但以下方案最常用于设置本地Mercurial存储库：

  • 克隆现有远程存储库并使用下载的数据创建新项目。

  • 创建一个本地存储库 ，如有必要，可以稍后将其推送到远程位置。

克隆远程Mercurial存储库

  1. 在主菜单上，选择VCS |从版本控制检出| Mercurial 。将打开“ 克隆Mercurial存储库”对话框。

  2. 在Mercurial Repository URL文本框中，键入要克隆的远程存储库的URL。

  3. 单击Mercurial Repository URL文本框旁边的Test Repository按钮，检查是否可以成功建立与远程存储库的连接。

  4. 在“ 父目录”文本框中，指定希望WebStorm为您的本地Mercurial存储库创建文件夹的目录。使用浏览 按钮，如有必要。

  5. 在“ 目录名称”文本框中，指定要将克隆存储库的新文件夹的名称。单击克隆 。

  6. 通过接受WebStorm显示的相应建议，基于克隆数据创建新项目。

创建本地Mercurial存储库

  1. 打开要存储在存储库中的项目。

  2. 在主菜单上，选择VCS |导入版本控制|创建Mercurial存储库 。将打开“ 创建Mercurial存储库”对话框。

  3. 指定新存储库的位置。
    • 要在项目根目录中创建存储库 ，请选择“ 为整个项目创建存储库”选项。WebStorm将在项目根文件夹中创建.hg目录。

      默认情况下选择此选项。

    • 要在另一个位置创建新存储库，请选择“ 选择创建存储库的位置”选项，并在下面的文本框中指定存储库位置的路径。手动键入路径或单击“ 浏览” 并在打开的“ 选择hg init目录”对话框中选择相关文件夹。

      Mercurial不支持外部路径。因此，如果您选择另一个目录，请注意它必须包含项目根所在的树。

      如果选择已在Mercurial控件下的目录，WebStorm将打开Directory Is Under hg对话框，您可以在其中选择在指定位置创建存储库或保留在父存储库中。

  4. 将所需文件放在Mercurial版本控制下。这些文件显示在“ 默认”节点下的“ 版本控制”工具窗口中。

    请注意，如果在“ 版本控制”对话框中将Mercurial指定为目录的版本控制系统，则WebStorm将建议将每个新文件放在Mercurial控件下的此目录中。

提取属性

假设您有一堆具有相同version的依赖项。

概观

此功能在PyCharm的教育版中不可用。

IntelliLang是三种基本功能的组合，旨在支持开发人员处理与PyCharm中的（自定义）语言相关的某些任务：

  • 语言注入 ：编辑嵌入到字符串和XML片段本身的代码。

    您可以在编辑器中打开和修改注入的语言代码片段。通过这种方式，您可以获得所有必要的编码帮助，就像您使用相应语言的源代码一样。

    要在编辑器中打开注入的语言代码片段，请使用编辑<语言>片段 意图操作 。

  • 模式验证 ：提供帮助以确保传入和传出方法的字符串与特定正则表达式匹配

  • 正则表达式支持 ：正则表达式的自定义语言实现

语言注入

这利用了PyCharm的新可能性将String文字，XML文本和属性视为任意语言的片段（称为语言注入 ）。该插件使这个新引入的API通过两种非常简单的方式随时可供每个人使用：通过使用一些Java注释来标记字符串字段，局部变量，方法参数和方法将字符串返回为包含某种语言，或者只是使用简单的UI配置。插件提供了一组注释，但实际注释可以自由配置以避免任何不需要的依赖。

这使得开发人员可以获得各种编辑时功能的好处，例如语法错误突出显示，完成，检查等等，同时在常规Java代码或PyCharm通常的自定义模式的XML文件中编辑JavaScript片段不知道。

模式验证

此外，该插件允许您注释String类型的Java元素，以检查它们是否符合某些正则表达式。这对于非常简单的语言非常有用，其中开发人员需要确保表达式符合某种语法，例如，String是合法的Java标识符或java.util.使用的有效类似printf的模式java.util.Formatter 。

这可以在编辑代码时以及在运行时（在方法参数和仅返回值，如PyCharm核心的@NotNull检测）中即时验证，方法是使用与提供的值匹配的断言来编译已编译的类。图案。

正则表达式支持

插件的这一部分实现了对java.util.regex.语言支持java.util.regex.Pattern和主要是通过添加对微语言的支持来创建支持IntelliLang插件，微语言可能是Strings中最常用的一种。它完全支持SDK正则表达式实现的语法，并添加了一些其他功能，例如

  • 字符属性名称的完成和验证（例如， \p{javaJavaIdentifierStart} ）无论如何通常都无法记住

  • 验证和导航以使用反向引用（例如， \1 ）。使用Ctrl + B导航到backref引用的捕获组。

  • 意图用于简化重复字符出现的用法的操作，例如， a{0,1}被提供转换为a?

  • “Surround With”捕获/非捕获组

缩小CSS

在开始之前，请确保您的计算机上有Node.js.

YUI Compressor是一种用于缩小CSS代码的工具。缩小或压缩意味着删除所有不必要的字符，例如空格，换行，注释而不更改源代码的功能。在开发和调试期间，这些字符使代码更易于阅读。在生产阶段，他们只增加要传输的代码的大小。

安装和配置YUI Compressor

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm安装yuicompressor

压缩CSS代码

要动态压缩代码，您需要将压缩器配置为WebStorm 文件观察器 。

创建文件监视器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 文件监视器 ”。打开的“ 文件监视器”页面显示已配置的文件监视器列表。

  2. 点击或按Alt + Insert并从列表中选择YUI Compressor CSS预定义模板。

  3. 在“ 程序”文本框中，指定yuicompressor-<version>.jar文件的路径。如果您通过节点包管理器安装该工具，WebStorm将自行查找所需的文件并自动填写该字段。否则，请手动键入路径或单击并在打开的对话框中选择文件位置。

  4. 按照使用文件监视器中的说明继续操作。

启用缩小文件观察程序时，只要更改或保存文件观察程序范围中的CSS文件，就会自动开始缩小。

WebStorm使用生成的输出创建单独的文件。该文件具有源CSS文件的名称和扩展名min.css 。生成的文件的位置在“ 新建观察器”对话框的“ 输出路径刷新”字段中定义。但是，在项目树中 ，默认情况下它显示在源CSS文件下，该文件现在显示为节点。要更改默认表示，请在“ 项目”工具窗口中配置文件嵌套 。
