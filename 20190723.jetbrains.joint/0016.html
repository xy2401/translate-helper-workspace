突出用途

删除突出显示

要删除高亮显示的用法，请按Escape键。

IntelliJ IDEA提供以下视图模式：

  • 全屏模式允许您使用整个屏幕进行编码。此模式从视图中删除所有菜单以及操作系统控件。但是，您可以使用上下文菜单和键盘快捷键。将鼠标指针悬停在屏幕顶部时，也可以使用主菜单。

配置与Web服务器的同步

在配置开发和生产环境的众多方法中，最常见的方法如下：

  • Web服务器安装在您的计算机上。源位于服务器文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • Web服务器安装在您的计算机上，但源存储在另一个文件夹中。您进行开发，然后将源复制到服务器。

  • Web服务器位于另一台计算机（远程主机）上。服务器上的文件可通过FTP / SFTP / FTPS协议，网络共享或已安装的驱动器获得。

WebStorm区分了就地服务器， 本地服务器和远程服务器，但WebStorm上下文中这些术语的含义与它们的常见含义略有不同：

  • 就地服务器是一个服务器，其文档根目录是项目根目录的父服务器，无论是否立即生效。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • 本地服务器是在本地或装入的文件夹中运行的服务器，其文档根目录 不是项目根目录的父级。您进行开发，然后将源复制到服务器。

  • 远程服务器是在另一台计算机（远程主机）上运行的服务器。可以通过FTP / SFTP / FTPS协议访问服务器上的文件。

通过Remote Hosts Access捆绑插件提供与服务器同步，上载，下载和管理文件，默认情况下启用该插件。如果禁用该插件 ，请在“设置”对话框的“ 插件”页面中将其激活。有关详细信息，请参阅管理插件 。

服务器访问配置

WebStorm通过服务器访问配置控制与Web服务器的交互。无论您何时使用服务器，都需要定义服务器访问配置，无论您的服务器是在远程主机上还是在您的计算机上。

服务器访问配置定义以下内容：

  • 服务器类型（ 就地 ， 本地或远程 ）。

  • 运行服务器的计算机（主机）。对于就地和本地服务器，WebStorm前提是，它是你的项目是当前计算机。

  • 服务器访问配置根 ：服务器层次结构中可通过服务器配置访问的最高文件夹。

  • 用于访问服务器配置根目录的URL地址。

  • 传输数据的协议和连接参数。

  • 本地文件夹（即项目根目录下的文件夹），服务器上的目标文件夹（本地或远程）和用于访问服务器上数据的URL地址之间的对应关系。这种对应关系称为映射 。

    例如，您的项目存储在C：/ Projects / My_Project /下，并包含文件夹C：/ Projects / My_Project / My_Folder ，而本地服务器的文档根目录在C：/ xampp / htdocs下 。您将整个项目树上传到C：/ xampp / htdocs / My_Project 。在WebStorm术语中， C：/ Projects / My_Project / My_Folder文件夹称为本地 ， C：/ xampp / htdocs / My_Project / My_Folder文件夹称为远程 。

您可以根据需要定义任意数量的配置，从而实现上载/下载设置之间的灵活切换。

如果需要将代码部署到多个服务器，则可以创建服务器组并避免单独部署到每个服务器。

您可以将服务器配置设置为默认值，以便WebStorm在以下情况下以静默方式选择它：

  • 自动上传已更改的文件 。

  • 无需选择目标主机即可手动上载和下载文件。

  • 将本地文件和文件夹与其远程版本进行比较。

  • 编辑远程主机上的单个文件 。

卸载模块 - 帮助| IntelliJ IDEA

IntelliJ IDEA 2019.1帮助

卸载模块

为了使IntelliJ IDEA更快地工作，您可以临时预留（卸载）目前不需要的模块。当您搜索或重构代码或编译项目时，IDE会忽略卸载的模块。

卸载模块时，可以在本地执行 - 有关卸载模块的信息不会通过版本控制共享。

手动卸载模块

  1. 在“ 项目”工具窗口中，右键单击模块，然后选择“ 加载/卸载模块” 。

  2. 您可以双击对话框中的模块来加载或卸载它，或使用对话框中间的按钮。

自动加载和卸载新模块

如果您的队友将新模块添加到项目中，您将在项目更新时将它们下载到您的计算机。之后，IDE将分析更新项目中所有模块之间的依赖关系。

如果卸载了模块，IntelliJ IDEA将根据依赖性分析的结果加载或卸载新模块。

如果新模块依赖于现有的卸载模块，则新模块将标记为已卸载。IntelliJ IDEA将忽略它们，否则您在尝试编译它们时可能会遇到错误。

如果现有加载的模块直接依赖于新模块，则新模块将标记为已加载。

如果现有加载的模块对新添加的模块没有依赖关系，则新模块将标记为已卸载。您可以在需要时手动将它们标记为已加载。

使用卸载的模块提交更改

如果卸载了模块，并且对卸载的模块所依赖的文件进行了更改，则在加载这些模块后，这些模块的编译可能会失败。

要避免卸载模块的编译失败，请确保在“ 提交更改”对话框中选中“ 编译受影响的已卸载模块”选项。

在提交更改的文件之前，IntelliJ IDEA将编译已卸载的模块，以确保更改不会影响这些模块。IDE将通知您检测到的错误，并建议在提交之前解决它们。

故障排除

如果项目中的模块相互依赖，则卸载其中一个或多个模块时可能会遇到错误。

例如，如果模块1依赖于模块2，并且卸载模块2，则IntelliJ IDEA将无法解析对模块2中的类的引用。此外，模块1的编译可能会失败。

为避免此类错误，IDE在加载或卸载模块时会分析依赖关系。加载模块时，IntelliJ IDEA会建议加载所有依赖项。卸载模块时，IDE将找到所有相关模块，并将卸载它们。

如果卸载模块1，则可能在模块2中的代码中看不到任何错误，您还可以编译模块2。但是，您可能会在第2单元中通过更改代码来意外破坏模块1中相关代码的编译。由于卸载了模块1，因此在将其加载并编译之前，您将无法看到任何错误。

如果您调用模块2中包含的查找用法（ Alt + F7 ）或类，字段或方法（ Ctrl + Shift + Alt + T ）上的重构，结果可能不完整，因为未采用模块1的内容考虑到了。IntelliJ IDEA将通知您。

此外，IDE将在每次提交更改时编译已卸载的模块，并将检查更改是否不会影响卸载的模块。请参阅使用卸载的模块提交更改中的更多信息。

为项目或目录启用Perforce集成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制” 。

  2. 在打开的“ 版本控制”页面中，指向所需的根。

    这可以是<Project Root>或内容根 。

  3. 从VCS下拉列表中，选择Perforce 。

配置Perforce集成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ Perforce ”。Perforce页面打开。

  2. 要建立与Perforce服务器的实时连接，请选中Perforce is online复选框。

  3. 指定要用于连接Perforce服务器的凭据。
    • 要使用P4CONFIG文件中的连接设置，请选择“ 使用P4CONFIG”或“默认连接”选项。

      如果您使用P4CONFIG文件进行配置，WebStorm会显示它找到的配置文件以及使用的其他默认设置。这样，您可以确保检测到并考虑了您的P4CONFIG文件。

    • 要手动配置连接，请选择“ 使用连接参数”选项，并在相应的文本框中指定以下设置：

      1. Perforce客户端将侦听的端口 。

      2. 客户名称 。

      3. 用于向服务器进行身份验证的用户名和密码 。

  4. 要使用基于票证的身份验证，请选中登录身份验证复选框。否则，将使用基于密码的身份验证。在任何一种情况下，WebStorm都使用对话框中指定的或存储在P4CONFIG文件中的登录名和密码。

  5. 要尝试在未经身份验证的情况下登录Perforce服务器，请选中“ 尝试以静默方式登录”复选框。

  6. 要让WebStorm创建P4.output文件并将Perforce命令的输出存储在其中，请选中Dump Perforce Commands to：复选框。

  7. 指定Perforce可执行文件的路径。单击“ 测试连接”以确保您的设置确保连接成功。

  8. 在Timeout字段中，指定WebStorm等待服务器响应的时间间隔（以秒为单位）。如果服务器未及时响应，则会提示用户禁用集成。

  9. 要启用显示指定文件的分支历史记录（包括所有文件分支点，编辑和合并），请选中“ 显示分支历史记录”复选框。

  10. 要使WebStorm指向也已集成到其他更改列表的已提交更改，并提供有关收到相关内容的目标更改列表的信息，请选中“ 在已提交更改中显示集成更改列表”复选框。

  11. 要获取用于将Perforce作业附加到更改列表并将其分离到更改列表的用户界面，请选中“ 启用Perforce作业支持”复选框。

TODO评论

在处理应用程序时，您可能希望标记代码的某些部分以供将来参考：要优化和改进的区域，可能的更改，要讨论的问题等等。在WebStorm中，您可以添加特殊类型的注释，这些注释在编辑器中突出显示并在TODO工具窗口中列出。

使用默认模式

默认情况下，WebStorm识别两种模式 - TODO和FIXME都是低位和大写。这些模式可以在任何支持的文件类型的注释中使用。

创建TODO评论

  1. 将光标放在要创建TODO项目的位置并添加注释，例如，按Ctrl + / ，然后键入TODO或FIXME ，然后键入注释。

  2. 在TODO工具窗口（ Alt + 6 ）中查看TODO项目列表。工具窗口由四个选项卡组成，其中WebStorm显示整个项目的TODO项目，活动编辑器选项卡中的文件，所选范围或活动更改列表中的文件。

    • 默认情况下，当您选择项目时，您会自动跳转到源代码中的相应注释。要禁用此自动导航，请右键单击工具窗口标题栏，单击“ 查看选项” ，然后删除“ 自动滚动到源”旁边的勾选。

    • 禁用“ 自动滚动到源” ，双击所需的TODO项目以跳转到源代码中的相应注释。

在JavaScript和TypeScript文件中，WebStorm还支持多行TODO注释。使用通常的TODO启动TODO注释，并在下一行添加额外的缩进以继续TODO注释。如果删除缩进，WebStorm会将其视为正常注释。

WebStorm默认支持多行注释。

要禁止多行TODO注释

  1. 在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）

    ，单击编辑器下的TODO 。

  2. 在打开的TODO页面上，清除以下行中的缩进文本作为同一TODO复选框的一部分。

使用自定义模式

或者，单击在TODO工具窗口中，然后单击“ 编辑过滤器” 。

您可以修改默认模式或添加自己的模式并过滤列表以仅显示与特定模式匹配的TODO项目。例如，您可能希望标记需要使用OPTIMIZE模式进行优化的代码片段，并忽略所有其他类型的TODO项。

  1. 在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）

    ，单击编辑器下的TODO 。TODO页面打开。

  2. 要为OPTIMIZE项添加模式，请单击在“ 模式”区域中，键入以下模式： \boptimize\b.*

  3. 要使用新模式添加过滤器，请单击在“ 过滤器”区域中，指定新过滤器的名称，然后选择要包含在此过滤器中的新模式。

要应用自定义过滤器

  • 在TODO工具窗口中，单击并选择必要的过滤器。

布局定义活动或应用程序窗口小部件 （片段）的用户界面。布局在XML 资源定义文件中声明。有关如何创建资源文件夹和资源定义文件的说明，请参阅创建资源 。

布局编辑模式

IntelliJ IDEA提出了两种设计Android应用程序用户界面的主要方法：

  • 手动编辑布局定义文件，可能使用IntelliJ IDEA提供的特定于Android的重构，并预览立即反映在专用预览工具窗口中的更改 ，您可以在其中调整布局到各种平台和设备。

  • 在专用的Designer工具窗口中编写布局 ：从“ 选用板”窗格中拖放布局元素，并在“ 属性”窗格中指定其属性。在此模式下，所有更改也会立即反映在编辑器中显示的预览中。

    在IntelliJ IDEA级别，此功能通过捆绑的Android Support插件提供，该插件默认启用。如果没有，请在“ 设置/首选项”对话框的“ 插件”页面中启用它。

在设计和文本模式之间切换

您可以通过在打开布局定义文件的编辑器中的“ 设计”和“ 文本”选项卡之间切换来在这些模式之间切换。

请注意，可用窗格和工具窗口集取决于当前布局编辑模式： 设计器工具窗口及其窗格在“ 设计”模式下可用，而“ 预览”工具窗口在“ 文本”模式下可用。

在设计模式下，您可以通过在“ 设计”窗格的上下文菜单中选择“ 转到声明”或单击“设置”来切换到手动模式。 预览窗口工具栏中的图标 。

在活动或片段及其相关布局定义文件之间导航
