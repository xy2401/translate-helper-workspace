云支持概述

IntelliJ IDEA让您：

  • 将应用程序工件部署到云端。

  • 将代码部署到Heroku。

  • 在Heroku上调试应用程序。

您还可以监视和控制部署过程，并在“ 应用程序服务器”工具窗口中查看应用程序日志。

使用云平台：流程概述

  1. 在云服务提供商网站上注册用户帐户。

  2. 确保已启用相应的云集成插件 。

  3. 在IntelliJ IDEA中注册您的云用户帐户。

    • 另外，在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）： 构建，执行，部署|云|

    • 创建用于与云提供商合作的项目或模块时： 文件|新的|项目|云

    • 创建云运行配置时： 运行|编辑配置| | <云名称>部署

    有关云用户帐户设置的信息，请参阅云 。

  4. 为您的应用程序创建工件配置 。对于Heroku，这可能是不必要的。您可以将源代码部署到云中，云平台将为您完成剩余的工作。对于CloudBees和Cloud Foundry，IntelliJ IDEA可以自动创建必要的配置。

  5. 构建工件。您可以单独执行（ Build | Build Artifacts ），也可以在执行相应的运行配置时执行此操作。在将代码部署到Heroku的情况下，不需要工件。

  6. 创建云运行配置，以将工件或代码部署到云。IntelliJ IDEA可以自动创建这样的运行配置。

  7. 执行运行配置以将工件或代码部署到云。

云集成插件

插件名称通常包括云名称和单词集成 ，例如， Heroku集成 。

有关更多信息，请参阅管理插件 。

  • 运行/调试配置：CloudBees部署

除了这两个跟踪分析器之外，PyCharm还支持采样（统计）分析器vmprof ，它应该安装在选定的Python解释器上。

如果您是Windows 64位用户，则必须安装Python 32位才能使vmprof正常工作。

按照Python Releases for Windows页面中的说明安装32位Python。

创建拉取请求

Pull请求用于开源项目或某些公司工作流程中，以管理来自贡献者的更改，并在合并此类更改之前启动代码审查。

重新格式化源代码

PyCharm允许您重新格式化源代码以满足代码样式的要求。PyCharm将列出间距，缩进，关键字等。重新格式化可以应用于选定的文本，整个文件或整个项目。

也可以使用格式标记将重新格式化仅应用于源代码的各个部分。

重新格式化目录的代码

要重新格式化目录的代码，请按照下列步骤操作：

  1. 在“项目”工具窗口中，选择要应用重新格式化的目录。

  2. 选择代码|在主菜单上重新格式化代码或按Ctrl + Alt + L.

    或者，在“ 项目工具”窗口中 ，右键单击该目录，然后从上下文菜单中选择“ 重新格式化代码” 。

  3. 在“重新格式代码”对话框中，为重新格式化指定必要的选项和过滤器，然后单击“运行” 。

重新格式化当前文件的代码

要重新格式化当前文件的代码，请按照下列步骤操作：

  1. 在当前打开的文件的编辑器中，按Ctrl + Shift + Alt + L.

    请注意，如果选择Code |从主菜单重新格式化代码或按Ctrl + Alt + L ，PyCharm将尝试自动重新格式化源代码，而无需打开“ 重新格式化文件”对话框。

  2. 在“ 重新格式化文件”对话框中，指定重新格式化的选项，然后单击“运行” 。

重新格式化源代码时跳过区域

要启用格式化程序标记，请确保在“设置/首选项”对话框的“ 标记”页面中的注释中选中“ 启用格式化程序标记 ”复选框，然后在“ 格式化程序关闭/开启”字段中键入标记。

要在重新格式化时跳过某个区域，请按照下列步骤操作：

  1. 在区域的开头，创建一个行注释（ Ctrl + / ），然后手动键入在Code Style页面的Formatter off字段中指定的标记。

  2. 在区域的末尾，创建一个行注释（ Ctrl + / ），然后手动键入在Code Style页面的Formatter on字段中指定的标记。

  3. 执行代码重新格式化，如上所述。

WebStorm执行两种不同的有效性检查：

本地和远程运行配置

服务器运行/调试配置可以是本地的或远程的。

本地配置适用于计算机上安装的服务器。此类配置包括定义如何启动相应服务器的设置。因此，当您执行本地配置时，IntelliJ IDEA会启动服务器。

远程配置无法启动服务器。通常，它们用于在不同（远程）计算机上运行的服务器。如果您不希望运行配置启动（或停止）服务器，它们也可以用于本地安装的服务器（在您的计算机上）。

执行远程配置时，IntelliJ IDEA连接到服务器以便能够部署应用程序工件或执行其调试。那时，相应的服务器必须已经运行。

总而言之，主要区分本地和远程配置的不是服务器的物理安装位置（尽管这也很重要），而是服务器是否通过相应的运行配置启动（或停止）。

启动服务器运行配置时会发生什么

当您启动服务器运行/调试配置时，IntelliJ IDEA通常会执行以下操作：

  1. 执行Before启动任务。默认情况下，这些是Make和Build工件 ：IntelliJ IDEA编译项目并构建应用程序工件。Build Artifacts任务最初不存在于运行配置中，而是在您指定要部署到服务器上的工件时自动添加。

  2. 启动服务器（用于本地配置）或连接到服务器（用于远程配置）。（此时，将打开“ 运行”或“ 调试”工具窗口，以便您可以监视和控制该过程。）

  4. 如果已指定，则启动Web浏览器并打开指定的URL（通常对应于应用程序的起始页）。

配置项目结构

在WebStorm中， 内容是您当前使用的文件集合，可能以子文件夹的层次结构进行组织。项目的顶级文件夹是其内容根 。

您可以添加任何文件夹作为项目的一部分。为此，只需在“ 目录”页面上将此文件夹添加为新的内容根目录 ，如使用多个内容根 目录中所述。

配置内容根目录中的文件夹

您也可以排除特定文件 。

或者，单击“ 设置/首选项”对话框中的“ 目录 ” （ Ctrl + Alt + S ），然后在打开的“ 目录”页面上配置文件夹类别。

在内容根目录中，WebStorm可以区分包含您编辑的源代码的文件夹和包含其他类型内容的文件夹。默认情况下，任何文件夹都被视为源，并用除非您将其标记为其他类型：

  • 测试 （ ）文件夹允许您将与测试相关的代码与生产代码分开。

  • 资源根 （ ）文件夹：可以相对于这些文件夹引用资源根文件夹中的文件，而不是指定完整路径。

  • 排除 （ ）代码完成，导航和检查会忽略文件夹。排除目前不需要的文件夹以提高WebStorm性能。

配置文件夹的类别

  • 在“ 项目”工具窗口中右键单击该文件夹，从上下文菜单中选择“ 标记目录” ，然后选择所需的类别。

要还原文件夹的上一个类别

  • 在“ 项目”工具窗口中右键单击该文件夹，从上下文菜单中选择“ 标记目录” ，然后选择“ 取消标记为<当前类别>” 。对于Excluded folders，选择Cancel Exclusion 。

排除文件

如果您现在不需要文件但又不想完全删除它，则可以暂时从项目中排除该文件。代码完成，导航和检查将忽略排除的文件。

  • 要排除文件，请在“ 项目”工具窗口中右键单击该文件，然后从菜单中选择“ 标记为纯文本 ”。该文件标有目录树中的图标。

  • 要还原更改，请右键单击该文件，然后从菜单中选择标记为<文件类型> 。

按名称模式排除文件和文件夹

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 目录”。 将打开“ 目录”页面。

  2. 在“ 排除文件”字段中，键入定义要排除的文件和文件夹名称的掩码，例如*.min.js 。使用分号作为分隔符。
    如果文件名与任何这些模式匹配，则将该文件视为已排除。如果文件夹的名称与模式匹配，则此文件夹及其子文件夹中的所有文件都标记为已排除。

使用多个内容根

您可以在项目中拥有多个内容根。这使您可以处理来自几个没有公共直接父级的目录中的文件。当您使用静态内容（例如图标）时，这很有用。您可以将它们全部保存在一个文件夹中，然后将此文件夹作为内容根添加到多个项目中。

  • 要删除内容根，请在右侧窗格中选择它，然后单击在它的旁边。确认删除。请注意，不会物理删除相应的文件夹。

缩小JavaScript

此功能仅在Ultimate版本中受支持。

IntelliJ IDEA与Closure Compiler ， YUI Compressor和UglifyJS集成，可让您即时压缩JavaScript应用程序源。

缩小或压缩意味着删除所有不必要的字符，例如空格，换行，注释而不更改源代码的功能。在开发和调试期间，这些字符使代码更易于阅读。在生产阶段，他们只增加要传输的代码的大小。

在你开始之前

  1. 下载并安装Node.js.

  2. 如果您打算使用YUI Compressor或Closure Compiler，请下载并安装Java Runtime Environment（JRE） ：

    • YUI Compressor的JRE 1.4或更高版本。

    • 闭包编译器的JRE版本7。

安装缩小工具

  • 要安装UglifyJS，请打开内置IntelliJ IDEA 终端 （ Alt + F12 ）并键入npm install uglify-js或npm install uglify-js@<version> 。从UglifyJS官方网站了解更多信息。

  • 要安装YUI Compressor，请按照YUI Compressor官方网站上的说明进行操作。

  • 要安装Closure Compiler，请按照Closure Compiler官方网站上的说明进行操作。

压缩代码

要动态压缩代码，您需要将缩小工具配置为IntelliJ IDEA 文件监视器 。

创建文件监视器

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 文件监视器 ”。“ 文件监视器”页面将打开。

  2. 点击并从列表中选择相关的文件观察器模板。

创建和管理TFS工作区

TFS服务器和本地项目之间的交互是通过工作空间配置的。工作区主要将存储库中的文件夹及其副本映射到计算机上。

在IntelliJ IDEA中，您可以配置对多个TFS服务器的访问权限，并根据需要在其下具有任意数量的工作空间。可用的TFS服务器访问配置和工作空间列表通过“ 管理TFS服务器和工作空间”对话框进行处理。

在本主题中：

  • 打开“管理TFS服务器和工作空间”对话框
  • 配置对TFS服务器的访问
  • 创建工作区

打开“管理TFS服务器和工作区”对话框

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。“ 管理TFS服务器和工作空间”对话框显示其中所有可用服务器和工作空间的列表。

配置对TFS服务器的访问

要放弃服务器访问配置，请在列表中选择服务器，然后单击“ Team Servers”区域中的“ Remove”按钮。

  1. 打开“管理TFS服务器和工作区”对话框，其中包含所有可用服务器和工作区的列表。

  2. 单击“ Team Servers”区域中的“ Add”按钮。

  3. 在“打开的添加Team Foundation Server”对话框中，在“ 地址”文本框中指定目标服务器的URL地址。

    在“身份验证”字段中，指定用于访问服务器的身份验证协议。TFS使用NTLM身份验证 ，因此本机Windows应用程序（即Microsoft Team Explorer ）使用系统凭据进行静默身份验证。由于Java Runtime构成的限制，IntelliJ IDEA用户必须始终指定其用户名和密码。

    要通过OAuth进行身份验证（Windows Live ID ），请从“ 身份 验证”下拉列表中选择“ 备用 ”。

  4. 对于NTLM和备用身份验证类型，请指定您的凭据：

    • 对于NTLM ：TFS服务器所在的网络域，TFS用户名和TFS密码。

    • 对于备用 ：您的TFS用户名和密码。

  5. 单击确定 。 IntelliJ IDEA返回“ 管理TFS服务器和工作空间”对话框，其中新服务器已添加到列表中。

创建服务器工作区

工作空间由其名称及其所有者的名称标识，包含计算机的名称，工作空间所属的服务器的URL地址，以及可通过工作空间访问的远程和本地工作文件夹之间的一组映射。

  1. 打开“管理TFS服务器和工作区”对话框，然后选择有问题的服务器。要刷新可用服务器工作空间的列表，请单击“ 重新加载”工作空间按钮。

  2. 单击“ 工作空间”区域中的“ 创建”按钮。

  3. 在打开的“ 创建工作空间”对话框中，指定工作空间名称。（可选）在“ 注释”文本框中提供工作区的简要说明。

  4. 在“ 工作文件夹”区域中，定义映射。

    1. 单击“ 添加”按钮 。新的行添加到映射列表中。

    2. 在“ 服务器路径”文本框中，指定需要使用的服务器上的文件夹。

    3. 在“ 本地路径”文本框中，指定用于存储下载数据的本地文件夹。

    4. 在“ 状态”下拉列表中指定映射的状态 。

      • 要根据映射启用从服务器检索数据，请选择“ 活动” 。

      • 要防止根据映射从服务器下载数据，请选择Cloaked 。

    5. 要放弃映射，请在列表中选择它，然后单击“ 删除”按钮 。

软件开发工具包

软件开发工具包或SDK是为特定软件框架开发应用程序所需的工具集合。例如，要使用Java开发应用程序，您需要一个Java SDK（JDK）。SDK包含二进制文件，二进制文件的源代码和源代码的文档。对于Java，SDK还包含注释。

通常，SDK是全局的。这意味着可以在多个项目和模块中使用一个SDK。创建新项目并为其定义SDK后，可以在此项目中配置模块以继承其SDK。您还可以单独为每个模块指定SDK。有关更多信息，请参阅更改模块SDK 。

支持的SDK

  • Java SE开发套件（JDK）

  • Java Micro Edition（ME）SDK
  • Android SDK

  • Flex和AIR SDK

  • Google App Engine SDK

  • IntelliJ平台插件SDK（用于开发IntelliJ IDEA插件）

定义SDK

定义SDK意味着让IntelliJ IDEA知道您计算机上的哪个文件夹中安装了必要的SDK版本。该文件夹称为SDK主目录。

管理全局SDK

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 平台设置”部分下，选择“ SDK” 。

  3. 要添加新SDK或新SDK版本，请单击按钮（ Shift + Enter ），选择SDK类型并指定其安装目录。

  4. 要删除SDK，请在列表中选择它，然后单击按钮（ Alt + Delete ）。

更改项目SDK

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目设置”部分下，选择“ 项目” 。

  3. 从Project SDK列表中，选择另一个SDK或SDK版本。

    如果尚未在IntelliJ IDEA中定义必要的SDK，请单击“ 新建”并指定其主目录。

更改模块SDK

  1. 从主菜单中，选择File |项目结构 （ Ctrl + Shift + Alt + S ）。

  2. 在“ 项目设置”部分下，选择“ 模块” 。

  3. 选择所需的模块，然后单击“ 依赖关系” 。

  4. 从“ 模块SDK”列表中，选择要使用的其他SDK或SDK版本。

    如果尚未在IntelliJ IDEA中定义必要的SDK，请单击“ 新建”并指定其主目录。

IntelliJ IDEA了解特定的Ant语法。但是，您必须让IntelliJ IDEA知道某个XML文件实际上是一个Ant构建文件。要被识别为构建文件并启用所有高级编辑功能，Ant构建文件应满足以下至少一个要求：

  • 该文件应该正确添加到项目中 。

  • <project>根元素应该具有default属性。

否则，此类文件将被视为具有基本编辑支持的常规XML文件。将构建文件添加到项目后，可以使用它来运行构建并修改其属性。

Ant构建目标

使用唯一名称标识构建目标 ，并定义应执行以完成特定任务的过程，例如，创建JAR文件或生成API文档。在<project>元素的default属性中指定的目标被视为默认目标，该目标在未指定其他目标时执行。此目标称为主目标 ，并在Ant构建工具窗口中以粗体字体标记。

编码协助

在IntelliJ IDEA中编辑Ant构建文件时，您可以享受以下高级编辑功能：

  • 语法突出显示。

  • 代码完成。特别是，为File类型的属性提供了代码完成。

  • 导航到声明Ctrl + B.

  • 使用结构视图。

  • 重命名重构。

  • 代码折叠。

  • 重新格式化。

  • 验证。

  • 查看参数信息Ctrl + P.

  • 查看快速信息Ctrl + Q.特别是，如果将classpath定义为类似路径的结构，则fileset或dirset指令的View Quick Info命令将显示磁盘上的实际文件和目录，这些指令将被解析。

路径式结构

IntelliJ IDEA允许在任务定义中使用类似路径的结构。如果将类路径定义为类似路径的结构，则fileset和dirset指令中的路径将解析为磁盘上的实际文件和目录。执行任务所需的所有JAR都应放在包含带任务定义的JAR的同一位置。
