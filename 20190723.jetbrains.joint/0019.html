默认情况下禁用Perforce集成。如果要直接从PyCharm执行与Perforce相关的操作，请在IDE级别启用集成，并将项目根目录或特定目录与Perforce关联。“ 启用版本控制 ”一节中介绍了常规过程。

在这个部分：

  • 为项目或目录启用Perforce集成
  • 配置Peforce集成设置

为项目或目录启用Perforce集成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制” 。

  2. 在打开的“ 版本控制”页面中，指向所需的根。

    这可以是<Project Root>或内容根 。

  3. 从VCS下拉列表中，选择Perforce 。

如果指定了错误的客户端工作空间，并且项目根目录与工作区根目录不匹配，则PyCharm会显示一个带有警告的弹出窗口。

单击“ 检查”以查看并修复差异。

配置Perforce集成

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ Perforce ”。Perforce页面打开。

  2. 要建立与Perforce服务器的实时连接，请选中Perforce is online复选框。

  3. 指定要用于连接Perforce服务器的凭据。
    • 要使用P4CONFIG文件中的连接设置，请选择“ 使用P4CONFIG”或“默认连接”选项。

      如果您使用P4CONFIG文件进行配置，PyCharm会显示它找到的配置文件以及使用的其他默认设置。这样，您可以确保检测到并考虑了您的P4CONFIG文件。

    • 要手动配置连接，请选择“ 使用连接参数”选项，并在相应的文本框中指定以下设置：

      1. Perforce客户端将侦听的端口 。

      2. 客户名称 。

      3. 用于向服务器进行身份验证的用户名和密码 。

  4. 要使用基于票证的身份验证，请选中登录身份验证复选框。否则，将使用基于密码的身份验证。在任何一种情况下，PyCharm都使用对话框中指定的或存储在P4CONFIG文件中的登录名和密码。

  5. 要尝试在未经身份验证的情况下登录Perforce服务器，请选中“ 尝试以静默方式登录”复选框。

  6. 要让PyCharm创建P4.output文件并将Perforce命令的输出存储在其中，请选中Dump Perforce Commands to：复选框。

  7. 指定Perforce可执行文件的路径。单击“ 测试连接”以确保您的设置确保连接成功。

  8. 在Timeout字段中，指定PyCharm等待服务器响应的时间间隔（以秒为单位）。如果服务器未及时响应，则会提示用户禁用集成。

  9. 要启用显示指定文件的分支历史记录（包括所有文件分支点，编辑和合并），请选中“ 显示分支历史记录”复选框。

  10. 要让PyCharm指向也已集成到其他更改列表的已提交更改，并提供有关收到相关内容的目标更改列表的信息，请选中“ 在已提交更改中显示集成更改列表”复选框。

  11. 要获取用于将Perforce作业附加到更改列表并将其分离到更改列表的用户界面，请选中“ 启用Perforce作业支持”复选框。

内置Web服务器

此功能仅在Ultimate版本中受支持。

以下内容仅在安装并启用PHP插件时有效！

IntelliJ IDEA有一个内置的Web服务器，可用于预览和调试您的应用程序。此服务器始终在运行，不需要任何手动配置。根据项目结构，所有项目文件都在内置服务器上提供，其根URL为http://localhost:<built-in server port>/<project root> 。

内置服务器只能提供HTML，JavaScript和CSS等静态内容。要将它与PHP文件一起使用，您需要为项目指定本地PHP解释器 。配置解释器后，IntelliJ IDEA将自动启动PHP内置Web服务器，并在运行PHP应用程序后立即将所有PHP请求重定向到它。要运行PHP应用程序，请在浏览器中打开文件或创建专用的运行/调试配置并启动它 。

IntelliJ IDEA 11.1及更高版本支持与内置Web服务器的集成。

配置内置Web服务器

如有必要，您可以自定义内置Web服务器的参数。按Ctrl + Alt + S或选择文件|打开设置/首选项对话框 设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，然后单击Build，Execution，Deployment下的Debugger 。

描述

港口

使用此旋转框指定运行内置Web服务器的端口。默认情况下，此端口设置为端口63342 ，IntelliJ IDEA通过端口接受来自服务的连接。您可以将端口号设置为以1024及更高版本开头的任何其他值。

可以接受外部连接

如果选中此复选框，则可以从另一台计算机访问在指定端口上运行的内置服务器上的文件。
如果清除此复选框（默认情况下），则调试器仅侦听本地连接。

IntelliJ IDEA中的上下文和依赖关系注入 （CDI）支持基于Java EE：Context和Dependency Injection插件。此插件与IDE捆绑在一起并默认启用。

您可以在创建项目或模块时启用CDI支持。您还可以为现有模块添加CDI支持。

模块中的CDI由模块依赖项中的相应CDI库表示。您可以直接从IDE中下载所有必需的库文件。您还可以在需要时更改正在使用的CDI版本。

确保已启用CDI插件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件”已安装 。

  2. 在列表中找到Java EE：Contexts和Dependency Injection插件，并确保选中它旁边的复选框。

  3. 在向导的右侧，指定要使用的JDK，选择要支持的Java EE版本，并指定要使用的应用程序服务器。

  4. 从Additional Libraries and Frameworks列表中，选择CDI：Context and Dependency Injection复选框。

  5. 配置库。您可以使用现有库，创建和使用新库，下载库文件，或推迟设置库，直到以后。

    单击下一步 。

  6. 在向导的下一页上，指定新项目或模块的名称和位置。单击完成 。

  1. 在“项目”工具窗口中，右键单击所需的模块，然后选择“ 添加框架支持” 。

      xxx（。+）yyy（。+）zzz

      xxx select * yyy from family zzz ，

      如果选项关闭， select *和from family将被视为两个独立的片段（或“文件”）。如果选项处于启用状态，则select * from family将被视为单个单元或“文件”。

  • XPath条件 - 在此文本框中，指定XPath表达式以更精确地处理注入目标。评估表达式的上下文是周围的XML标记。

    可以使用Jaxen XPath引擎提供的XPath 扩展函数 ，例如lower-case() 。此外，还有三个附加功能可用于确定当前文件的名称，扩展名和文件类型： file-name() ， file-ext()和file-type() 。或者，可以通过标准代码完成来检索可用功能的列表。

使用补丁

您可以将它们放在.patch文件中，而不是提交更改，以后可以将其应用于您的源，或者通过电子邮件发送，等等。

创建补丁

要基于未提交的更改创建修补程序，请执行以下操作：

  1. 打开“ 版本控制”工具窗口（ Alt + 9 ）并切换到“ 本地更改”选项卡。

  2. 选择要根据其创建修补程序的文件或更改列表。

  3. 从上下文菜单或主VCS菜单中选择“ 创建补丁” 。

  4. 在打开的对话框中，确保选中要包含在修补程序中的所有更改，输入提交注释（可选）并单击“ 创建修补程序” 。

  5. 在“ 修补程序文件设置”对话框中，根据需要修改默认修补程序文件位置，然后单击“ 确定” 。

如果您不需要将修补程序保存到文件中，并希望（例如）通过电子邮件或通过信使发送它，则可以从所选文件或更改列表的上下文菜单中选择“ 复制为修补程序到剪贴板” ，然后将其粘贴到邮件正文中。

您还可以通过调用“提交更改”对话框 ，选择要包含在修补程序中的更改，将鼠标光标悬停在“ 提交”按钮上并选择“ 创建修补程序”来创建修补程序 。

您还可以根据已提交的更改创建修补程序。要从整个提交创建补丁，请在“ 日志”视图中找到它，然后从上下文菜单中选择“ 创建补丁 ”。如果需要在单个文件上创建补丁，并且相应的提交包含多个文件，请执行以下操作：

  1. 在任何视图中选择所需文件（在“ 项目”工具窗口中，在编辑器中，在“ 版本控制”工具窗口的“ 本地更改”选项卡中，选择一个子项）。

  2. 选择Git |从主VCS菜单或选择的上下文菜单中显示历史记录 。“ 历史记录”选项卡将添加到“ 版本控制”工具窗口。

  3. 右键单击修订并从上下文菜单中选择“ 创建修补程序 ”，或单击“ 创建修补程序”图标在工具栏上。

应用补丁

要将存储在文件中的更改应用于源，请执行以下操作：

  1. 选择VCS |从主菜单应用补丁 。

  2. 在打开的“ 应用修补程序”对话框中，指定要应用的.patch文件的路径。

  3. 如有必要，请单击“ 映射基目录”图标指定相对于将解释补丁文件中的文件名的目录。您可以将基目录映射到单个文件，目录或选择。

  4. 如果在创建补丁后编辑了源代码，则可能会发生冲突。要检查是否可以在不冲突的情况下应用修补程序，请单击“ 显示差异”图标或按Ctrl + D.如果存在冲突，相应的行将以红色突出显示。

  5. 如果要对修补程序中指定的文件存储在不同位置的文件应用更改，可以使用“ 条带目录”按钮去除前导目录 。

  6. 选择要从列表中添加修补程序的现有更改列表，或在“ 名称”字段中指定新更改列表的名称 ，并可选择为此更改列表输入注释。

  7. 如果要激活此更改列表，请选择“ 设置活动”选项。

  8. 如果希望IntelliJ IDEA在其停用时保留与新更改列表关联的任务的上下文，并在更改列表变为活动状态时还原上下文，请选择“ 跟踪上下文”选项（有关详细信息，请参阅管理任务和上下文 ）。

您可以将文件或电子邮件附件拖动到编辑器中的任何位置。

您还可以复制补丁文件内容并通过选择VCS |来应用它从主菜单中应用剪贴板中的Patch 。例如，当您通过电子邮件收到修补程序并且不想保存它时，这很方便。对于Git格式补丁，IntelliJ IDEA提取提交消息和作者，并自动填写“ 提交更改”对话框中的相应字段。

注册现有帐户

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 单击添加帐户 。

  3. 在打开的对话框中，指定您的GitHub服务器URL（github.com或企业实例）。

  4. 执行以下操作之一：

    • 如果您已有令牌，请单击“ 输入令牌”链接并将其粘贴到那里。

    令牌必须在您的帐户权限中启用repo和gist作用域（请参阅了解作用域 ）。

创建一个新的GitHub帐户

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 单击“ 添加帐户” ，然后在打开的对话框中单击“ 注册Github” 。

  3. 在打开的注册GitHub页面上注册您的帐户。

  4. 返回IntelliJ IDEA设置并指定您的凭据。

管理多个帐户

您可以在IntelliJ IDEA中使用多个GitHub帐户：例如，用于处理开源项目的个人帐户以及用于主要工作的公司帐户。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 使用按钮可根据需要添加任意数量的帐户。

  3. （可选）要将帐户设置为当前项目的默认帐户，请选择该帐户并单击 。如果设置了默认帐户，IntelliJ IDEA将不会要求您在GitHub上共享项目时选择要使用的帐户，重新分叉 ， 创建拉取请求或创建要点 。

TFS登机政策

签入策略是在每次签入之前执行的规则，以确保所选的变更集可以提交。标准策略存储在服务器上，并在客户端计算机上执行。

自定义策略实现为WebStorm的自定义插件。这些插件的ID存储在服务器上，而策略本身则在本地应用。因此，要在团队中使用策略，所有团队成员都应安装相应的插件。

在本页面：

  • 定义默认策略设置
  • 禁止在当前项目中使用默认的WebStorm范围的签入策略设置
  • 管理可用策略列表
  • 引入自定义签到策略

定义要在WebStorm级别应用的默认策略设置

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，选中Checkin策略兼容性区域中的适用复选框。

    • 评估团队资源管理器策略：选中此复选框以在客户端计算机上安装并执行Microsoft团队资源管理器策略定义。

    • 评估Teamprise策略：选中此复选框以在客户端计算机上安装并执行Teamprise策略定义。

    • 警告未安装的策略：选中此复选框可在未安装指定的策略定义的情况下显示警告。

禁止将默认签入策略设置应用于项目

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。

  3. 在“ 管理TFS服务器和工作空间”对话框中，打开该对话框，从“ 团队项目”下拉列表中选择有问题的项目 。

  4. 在“ 兼容性”区域中，选中“ 替代团队项目<项目名称>的默认设置”复选框。

  5. 通过选择或清除下面相应的复选框重新定义默认设置。

    • 评估团队资源管理器策略：选中此复选框以在客户端计算机上安装并执行Microsoft团队资源管理器策略定义。

    • 评估Teamprise策略：选中此复选框以在客户端计算机上安装并执行Teamprise策略定义。

    • 警告未安装的策略：选中此复选框可在未安装指定的策略定义的情况下显示警告。

管理可用策略列表

可用策略列表包括标准第三方策略和自定义用户定义策略。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 版本控制”下的“ TFS ”。

  2. 在打开的TFS页面上，单击“ 服务器和工作区”区域中的“ 管理”按钮。

  3. 在打开的“ 管理TFS服务器和工作空间”对话框中，选择所需的工作空间，然后单击“ 签入策略”按钮。

  4. 在打开的“ 编辑签入策略”对话框中，配置策略列表：

    • 要激活策略，请选中它旁边的已启用复选框。

    • 要取消策略，请清除其旁边的“已启用”复选框。

    • 要永久丢弃策略，请在列表中选择该策略，然后单击“ 删除”按钮。

引入自定义登记政策

  1. 将所需策略实现为自定义插件。

  2. 按照管理插件中的说明下载，安装和启用插件 。

安装，卸载和升级包

PyCharm提供了为特定Python解释器安装，卸载和升级Python包的方法。

要管理项目解释器的Python包，请在项目“设置/首选项 ”中选择“ 项目解释器”页面，或在“ Python解释器”窗口小部件中选择“ 解释器设置” 。

如果选择具有已配置的Conda环境的Python SDK，则“包”区域工具栏中将显示“ 使用Conda程序包管理器”切换。

使用此切换来管理Conda环境存储库中的包。默认情况下，此切换关闭，您只能管理所选Python解释器可用的包。

安装包

  1. 单击安装 （ ）。

  2. 在打开的“ 可用包”对话框中，预览可用包的列表。

    要指定自定义存储库（包括devpi或PyPi） ，请单击“ 管理存储库” 。
    在打开的“ 管理存储库”对话框中，单击“确定” 添加本地存储库的URL，例如， http：// localhost：3141 / root / pypi / + simple / ，然后单击“ 确定” 。在“ 可用包”对话框中，单击重新加载包列表。

  3. 在“ 搜索”字段中键入要安装的包的名称。该列表缩小以仅显示匹配的包。

  4. 如果需要，请选中以下复选框：

    • 指定版本 ：如果选中此复选框，则可以从可用版本列表中选择所需的版本。默认情况下，采用最新版本。

    • 选项 ：如果选中此复选框，则可以在文本字段中键入选项。

    • 安装到用户的站点包目录<path> ：如果清除此复选框（默认情况下），则包将安装到当前解释器包目录中。如果选中该复选框，则软件包将安装到指定的目录中。此选项不适用于Conda环境。

  5. 选择目标包，然后单击“ 安装包” 。

卸载包

  1. 在包列表中，选择要删除的包。

  2. 单击卸载 （ ）。选定的软件包将从磁盘中删除。

PyCharm通过显示当前安装的软件包版本（列版本 ）和最新可用版本（列最新版本 ），巧妙地跟踪软件包的状态并识别过期版本。当检测到更新版本的软件包时，PyCharm会用箭头符号标记它并建议升级它。

默认情况下，列最新版本仅显示软件包的稳定版本。如果要将最新可用版本的范围扩展到任何预发行版本（例如测试版或候选发布版 ），请单击“ 显示早期版本” 。

升级包

  1. 在包列表中，选择要升级的包。

  2. 点击升级 （ ）。请注意，单击“ 应用 ”不会导致包升级。此按钮仅使用所有已执行的包更改。

    选定的包将升级到最新的可用版本。

您可以一次升级多个包。按住Cmd （macOS）或Ctrl （Unix或Windows），左键单击以选择包列表中的多个项目，然后单击“ 升级” 。

深度

创建就地服务器配置

就地服务器是一个服务器，其文档根目录是项目根目录的父服务器，无论是否立即生效。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

要在此设置中配置对服务器的访问，您只需指定服务器文档根目录的URL地址，指定项目根文件夹，并指定访问它的URL地址。

指定服务器配置的名称，类型和可见性

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在设置/首选项（Ctrl + Alt + S），选择下生成，执行，部署 部署 。

    • 选择工具|部署|在主菜单上配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加” 并从列表中选择就地 。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。将关闭“ 创建新服务器”对话框，并返回“ 部署”节点的“ 连接”选项卡。

    单击“ 用作默认值”工具栏按钮让WebStorm在以下情况下以静默方式应用当前配置：

    • 自动上传已更改的文件 。

    • 无需选择目标主机即可手动上载和下载文件。

    • 将本地文件和文件夹与其远程版本进行比较。

    • 编辑远程主机上的单个文件 。

  4. 仅对此项目使用“ 可见”复选框以配置服务器访问配置的可见性。

    • 选中该复选框以限制将配置用于当前项目。此类配置无法在当前项目之外重复使用。它不会出现在其他项目的可用配置列表中。

      服务器记录与项目一起存储在.idea目录中，允许通过VCS在团队成员之间共享它们。

      在左侧窗格的服务器访问配置列表中，仅在当前项目中可见的配置标有图标。

    • 清除该复选框后，配置在所有WebStorm项目中都可见。其设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

指定服务器文档根目录的URL地址

  • 在“ Web服务器URL”字段中，键入与Web服务器配置文件中定义的Web服务器的文档根关联的URL地址。此URL地址将是构建应用程序URL地址的起点。

    支持HTTP和HTTPS协议。

例如，默认文档根目录设置为htdocs文件夹，访问其中数据的默认URL地址为http://localhost 。如果更改了默认端口80 ，则必须明确指定端口： http://localhost:<port> 。

指定项目根文件夹和URL地址以访问它

  1. 切换到Mappings选项卡。

  2. 在“ 本地路径”字段中，指定项目根文件夹的完整路径。手动键入路径，或单击并在对话框中选择打开的文件夹。

  3. 在Web路径字段中，键入项目根文件夹相对于服务器配置文件中指定的服务器文档根目录的路径。

您可以使用TypeScript编写应用程序，然后以与使用JavaScript编写的客户端应用程序相同的方式运行和调试它。

在运行或调试之前，您需要将TypeScript代码编译为JavaScript。PyCharm允许您通过与TypeScript编译器集成来动态执行此操作，如验证TypeScript并将其编译为JavaScript中所述 。您还可以使用其他工具，例如webpack 。从TypeScript官方网站了解更多信息。

运行客户端TypeScript应用程序

您不需要任何配置来从PyCharm运行TypeScript。

  1. 将TypeScript代码编译为JavaScript 。

  2. 在编辑器中，打开HTML文件，并引用生成的JavaScript文件。此HTML文件不一定必须是实现应用程序起始页面的HTML文件。

  3. 执行以下操作之一：

    • 选择查看|在主菜单上的浏览器中打开或按Alt + F2 。然后从列表中选择所需的浏览器。

    • 将鼠标指针悬停在代码上以显示浏览器图标栏： 单击指示所需浏览器的图标。

调试TypeScript

在开始之前，按照配置JavaScript调试器中的描述配置内置调试器 。

通常，您可能希望调试在外部开发Web服务器上运行的客户端应用程序，例如由Node.js提供支持。如果您的应用程序在内置PyCharm服务器上运行 ，您也可以像在内置服务器上运行的JavaScript一样调试它。

使用PyCharm，您可以在原始TypeScript代码中设置断点，而不是实际执行的已编译JavaScript。PyCharm使用源映射识别和处理TypeScript中的断点。

在编译期间生成源映射

  • 打开tsconfig.json文件并确保sourceMap属性设置为true 。

调试在外部Web服务器上运行的TypeScript应用程序

  1. 将TypeScript代码编译为JavaScript，请参阅编译TypeScript 。确保生成源映射。

  2. 在TypeScript代码中配置和设置断点 。

  3. 在开发模式下运行该应用程序。通常你需要为此运行npm start 。开发服务器准备就绪后，在浏览器中复制运行应用程序的URL地址 - 您需要在运行/调试配置中指定此URL地址。

  4. 选择运行|从主菜单中编辑配置 ，单击在工具栏上，从列表中选择JavaScript Debug 。在“ 运行/调试配置：打开的JavaScript调试”对话框中，指定运行应用程序的URL地址。可以从浏览器的地址栏复制此URL，如上面的步骤3中所述。

  5. 在工具栏上的“ 选择运行/调试配置”下拉列表中选择新创建的配置，然后单击“确定” 。运行配置中指定的URL地址将在所选浏览器中打开，并显示“ 调试工具窗口 ”。

  6. 在调试工具窗口中，像往常一样继续： 逐步执行程序 ， 停止并恢复程序执行， 暂停时检查它 ， 查看实际的HTML DOM ，等等
