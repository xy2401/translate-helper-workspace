概观

答案占位符是一个框架，用于替换原始练习代码的一部分，学生必须在其中编写解决方案。

当占位符已存在时，上下文菜单命令“ 添加应答占位符”将替换为一组命令，这些命令可用于编辑占位符，删除占位符或删除所有应答占位符。

如果学生认为他/她无法解决问题，他/她可能会寻求帮助。

本页内容适用于教育工作者和学生 。

以下功能适用于教育工作者！

创建答案占位符

创建答案占位符

  1. 选择要替换为某些文本的练习代码片段。

  2. 右键单击选择，然后在上下文菜单中选择“ 添加应答占位符” 。

  3. 在“ 添加应答占位符”对话框中，指定将替换学生项目中所选片段的文本。

    如有必要，添加提示 。

编辑答案占位符

编辑答案占位符

  1. 将插入符号放在要更改的答案占位符上。

  2. 在上下文菜单中，选择“ 编辑应答占位符” 。

  3. 如有必要，更改任务窗口文本和提示的文本。

  4. 准备好后单击确定 。

删除答案占位符

删除答案占位符

  1. 将插入符号放在要删除的答案占位符上。

  2. 在上下文菜单中，选择“ 删除答案占位符” 。

    如果不再需要所有答案占位符 ，请选择“ 删除所有答案占位符” 。

占位符将以静默方式删除。

此操作仅适用于学生！

放弃

获得答案

  • 在主菜单上，选择“ 帮助”|填写答案占位符 。

当前页面上的所有答案占位符都填入了正确的解决方案。

文件集

  • 单个文件由文件名定义，即MyDir/MyFile.txt

  • 目录中所有文件的集合，不会递归到子目录中，在斜杠后用星号定义，例如： file:src/main/myDir/*

  • 包含子目录内容的目录中的所有文件集由双斜杠后的星号定义，例如file:src/main/myDir//*

逻辑运算符

范围语言允许您使用常见的逻辑运算符：

&& for AND || 对于 OR ！ 不

除此之外，括号可用于将逻辑运算符连接到组中。例如，以下范围

（<A> ||的<b>）&& <C>

暗示<a>和<c>，或<b>和<c>。

从现有范围创建新范围

GWT包结构

标准GWT包布局有助于区分客户端代码和服务器端代码。下图说明了标准GWT包的结构。

  • 客户端 - 此目录包含客户端源文件和子包。

  • Public - 此目录包含可以公开提供的各种静态资源。默认情况下，不在项目中创建此目录。

  • 服务器 - 此目录包含服务器端代码和子包。

  • GWT模块XML描述符。

GWT模块

GWT配置的各个单元是称为模块的XML文件。模块捆绑了GWT项目所需的所有配置设置，即：

  • 继承的模块。

  • 入口点应用程序类名称;这些是可选的，尽管HTML中引用的任何模块必须至少指定一个入口点类。

  • 源路径条目。

  • 公共路径条目。

  • 延迟绑定规则，包括属性提供程序和类生成器。

GWT模块XML描述符（5）应驻留在标准项目布局的根包中。IntelliJ IDEA可以为您生成具有相应项目结构的GWT模块 。

入口点

模块入口点是可分配给EntryPoint任何类，可以在没有参数的情况下构造。加载模块时，将实例化每个入口点类并调用其EntryPoint.onModuleLoad()方法。

属于加载路径的目录标记为源根目录在项目视图中。

通过“项目”工具窗口的上下文菜单定义加载路径

  1. 在“ 项目工具”窗口中右键单击所需目录。

未知语言ID

此检查为使用不存在的语言ID提供了验证。它标记@Language值属性的错误值的用法，例如@Language("NonExistingID") 。

语言不匹配

验证是否使用对注释为包含不同语言或根本没有注释的元素的引用。检查提供了一个Quick-Fix，用于为预期语言添加正确的注释来注释这些元素。

注射不适用

此检查检查@Language或任何派生的注释是否用于除String或String[]类型的元素之外的任何内容。

模式验证

此类别包含有关验证@Pattern或其派生注释的使用的检查。

  • 验证带注释的模式
  • 模式注释不适用
  • 非注释方法会覆盖@Pattern方法

验证带注释的模式

此检查验证表达式（字符串文字，以及其他编译时常量或替换表达式）是否与@Pattern注释所需的模式匹配。检查可以选择忽略包含非替换引用的非常量表达式，并提供Quick-Fix以在适用的情况下添加替换。

模式注释不适用

检查模式验证注释（ @Pattern或派生的注释）是否有效应用于带注释的元素。只能对String类型的元素进行注释。

非注释方法会覆盖@Pattern方法

此检查检查没有任何@Pattern或派生注释的方法是否从其基类覆盖带注释的方法。这对于编辑器中的错误突出显示不是必需的，但是运行时检查检测不会从基类方法中获取注释。

提供了一个Quick-Fix来添加一个与基类方法相匹配的注释。这可确保运行时检查工具正常工作。

使用多个构建JDK

为了能够编译应用程序，您需要指定构建JDK。可以在项目级别配置构建JDK。如果您有一个复杂的项目，您可能希望为不同的模块配置不同的JDK。

  • IntelliJ IDEA如何知道使用哪个JDK？

IntelliJ IDEA如何知道使用哪个JDK？

本节介绍了如果在每个模块的基础上配置多个JDK，则IntelliJ IDEA选择构建JDK的算法。IntelliJ IDEA执行以下操作以确定要用于编译的JDK：

  • IntelliJ IDEA检查项目中使用的所有JDK（即在项目和模块级别定义的JDK）。

  • IntelliJ IDEA计算最新的JDK。这是确保可以编译所有模块的必要条件。

  • 如果配置的最新JDK版本低于1.6，将使用用于运行IntelliJ IDEA的JDK版本。此限制与以下事实有关：从JDK 1.6开始支持IntelliJ IDEA用于构建项目的编译器API。

  • 虽然将使用特定版本的编译器（根据所选的JDK版本），但是每个单独的模块将使用javac的交叉编译功能针对在项目设置中为此特定模块定义的JDK库进行编译。

    这可以保护您免受针对较新库编译模块的情况的影响，而不是针对设置了依赖项的库。

配置构建JDK

  1. 打开“ 项目结构”对话框 。

  2. 执行以下操作之一：

    • 要为整个项目设置构建JDK，请在左窗格中选择“ 项目 ”。在右侧的Project SDK下 ，从下拉列表中选择所需的SDK。

    • 要为特定模块设置构建JDK，请在左窗格中选择Modules ，然后在中央窗格中选择所需模块，并在Module SDK下拉列表中指定JDK。

      如果之前未创建任何项目并指定了JDK的路径，请单击“ 新建” ，选择“ JDK”并在打开的对话框中浏览JDK安装文件夹。

请注意，您还可以通过编辑相应的运行/调试配置来覆盖特定调试或测试会话的项目级或模块级JDK设置。

@ParametersAreNonnullByDefault

@ParametersAreNonnullByDefault注释可帮助您定义类或包中的所有方法参数都具有@NotNull语义，除非使用@Nullable注释明确注释它们。

@ParametersAreNonnullByDefault批注可以与包，类或方法一起使用。

要使用注释，请将jsr305库添加到模块依赖项：

  1. 打开“ 项目结构”对话框（ Ctrl + Shift + Alt + S ），然后转到“ 模块”|依赖性 。

  2. 单击添加 并选择Library | Java 。

  3. 在IntelliJ IDEA主目录中，选择lib \ jsr305.jar。

  4. （ 可选 ）在下一个对话框中，您可以修改库名称和级别。

  5. 应用更改并关闭对话框。

将JAR添加到项目后，可以开始使用@ParametersAreNonnullByDefault批注。例如，请考虑以下代码：

public static < T extends Comparable < T >> List < T > sort （ List < T > list ） { if （ list ！）= null ）{ List < T > copy = new ArrayList < T >（ list ）; 排序 （ 复制 ）; 返回 副本 ; } else { return null ; } }

如果使用@ParametersAreNonnullByDefault注释sort()方法，IntelliJ IDEA会立即识别if语句是无关的，并报告始终为true的条件。

但是，如果将方法sort()的参数注释为可为空，则不会看到任何检查消息。
