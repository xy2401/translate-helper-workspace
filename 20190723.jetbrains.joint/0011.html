引用XML模式和DTD

您的XML文件可能引用外部XML架构（XSD）或DTD文件，例如

<root xmlns = “http://www.example.org” xmlns：xsi = “http://www.w3.org/2001/XMLSchema-instance” xsi：schemaLocation = “http://www.example.org http://www.example.org/xsds/example.xsd“ >
要么
<！DOCTYPE root SYSTEM “http://www.example.org/dtds/example.dtd” >
如果引用的URL或名称空间URI是“不熟悉的”，则将其标记为错误。要解决此问题，请使用以下意图操作之一 ：

  • 获取外部资源。 PyCharm下载引用的文件并将其与URL（或名称空间URI）相关联。错误突出显示消失。根据下载的模式或DTD验证XML文件。（URL和命名空间URI与模式和DTD文件的关联显示在“设置”对话框的“ 模式和DTD”页面上。）

  • 手动设置外部资源。 如果已在本地提供适当的架构或DTD文件，请使用此选项。将打开“ 映射外部资源”对话框 ，您将能够为指定的URL或名称空间URI选择文件。操作的结果与获取资源的情况相同。

  • 忽略外部资源。 URL或名称空间URI将添加到“忽略的架构和DTD”列表中。（此列表显示在“设置”对话框的“ 架构和DTD”页面上。）错误突出显示消失。PyCharm不会验证XML文件，但它会检查XML文件是否格式正确。

您可能会发现另外一个有用的意图操作： 为外部资源添加Xsi架构位置 。此意图操作允许您完成根XML元素。如果已经指定了名称空间，PyCharm可以添加一些缺少的属性。

并且您对xmlns属性的值调用意图操作，结果将是：
< 根 的xmlns = “http://www.example.org” 的xmlns： 的xsi = “http://www.w3.org/2001/XMLSchema-instance” 的xsi： 的schemaLocation =“HTTP：//www.example.org >
在此步骤中，您可以添加架构URL，然后将URL（或命名空间URI）映射到相应的架构文件，或将URL（或URI）添加到“忽略的架构和DTD”列表中。

引用XML模式和DTD

您的XML文件可能引用外部XML架构（XSD）或DTD文件，例如

<root xmlns = “http://www.example.org” xmlns：xsi = “http://www.w3.org/2001/XMLSchema-instance” xsi：schemaLocation = “http://www.example.org http://www.example.org/xsds/example.xsd“ >
要么
<！DOCTYPE root SYSTEM “http://www.example.org/dtds/example.dtd” >
如果引用的URL或名称空间URI是“不熟悉的”，则将其标记为错误。要解决此问题，请使用以下意图操作之一 ：

  • 获取外部资源。 IntelliJ IDEA下载引用的文件并将其与URL（或名称空间URI）相关联。错误突出显示消失。根据下载的模式或DTD验证XML文件。（URL和命名空间URI与模式和DTD文件的关联显示在“设置”对话框的“ 模式和DTD”页面上。）

  • 手动设置外部资源。 如果已在本地提供适当的架构或DTD文件，请使用此选项。将打开“ 映射外部资源”对话框 ，您将能够为指定的URL或名称空间URI选择文件。操作的结果与获取资源的情况相同。

  • 忽略外部资源。 URL或名称空间URI将添加到“忽略的架构和DTD”列表中。（此列表显示在“设置”对话框的“ 架构和DTD”页面上。）错误突出显示消失。IntelliJ IDEA不会验证XML文件，但是，它会检查XML文件是否格式正确。

您可能会发现另外一个有用的意图操作： 为外部资源添加Xsi架构位置 。此意图操作允许您完成根XML元素。如果已指定名称空间，则IntelliJ IDEA可以添加一些缺少的属性。

例如，如果您有这样的片段：

< root xmlns = “http://www.example.org” >
并且您对xmlns属性的值调用意图操作，结果将是：
< 根 的xmlns = “http://www.example.org” 的xmlns： 的xsi = “http://www.w3.org/2001/XMLSchema-instance” 的xsi： 的schemaLocation =“HTTP：//www.example.org >
在此步骤中，您可以添加架构URL，然后将URL（或命名空间URI）映射到相应的架构文件，或将URL（或URI）添加到“忽略的架构和DTD”列表中。

基本

PyCharm中的代码覆盖允许您以低运行时开销为代码执行即时线覆盖测量。一般来说，线路覆盖率回答了这样一个问题：“在单元测试模拟过程中是否执行了这一行代码？“

代码覆盖率结果反映在专用Coverage工具窗口 ， Project Tool窗口的Project视图和编辑器中。工具窗口显示以下信息：

  • 对于目录：覆盖的类和行的百分比。

  • 对于文件：包含行的百分比。

在编辑器中打开文件时，每行都会突出显示其代码覆盖状态：

  • 模拟期间执行的行标记为绿色。

  • 模拟期间未执行的行标记为红色。

覆盖测量结果包括覆盖套件 。您可以将新模拟的结果与任何现有套件合并。在这种情况下，如果一条线被至少一个模拟覆盖，则该线将被视为已覆盖。

每次执行具有代码覆盖率测量的测试或应用程序时，都会生成覆盖套件 。可以拥有无限量的覆盖套件。

运行代码覆盖率

要在项目中使用代码覆盖率，请遵循以下常规步骤

  1. 指定您希望如何处理覆盖率结果 。

  2. 如果要测量代码覆盖率以进行测试 ，请为目标代码创建测试。

  3. 在所需的运行/调试配置中配置代码覆盖率测量 。

  4. 使用主菜单Run |上的专用命令运行coverage 使用Coverage运行 ，或 。

  5. 执行覆盖率运行后，您可以执行以下操作：

    • 使用各种覆盖套件 。

    • 查看代码覆盖率数据 。

    • 生成代码覆盖率报告 。

@Nullable和@NotNull

@Nullable和@NotNull注释允许您检查变量，参数或返回值的可为空性。它们可以帮助您控制整个方法层次结构中的合同，如果IntelliJ IDEA发现合同被违反，它将报告检测到的问题，并指向可能发生NullPointerException的代码。

例如，如果您创建一个参数具有@NotNull注释的方法，然后使用可能为null的参数调用此方法，IntelliJ IDEA将动态突出显示该问题。

检查由Constant条件和例外以及@NotNull / @Nullable问题检查完成。您可以在“ 设置/首选项” （ Ctrl + Alt + S ）对话框中配置这些检查的工作方式。转到编辑|检查| Java |可能的错误 。

编译项目时，IDE会将断言添加到使用@NotNull注释注释的所有方法和参数。如果在需要@NotNull代码中传递null，则断言将失败。您可以在“ 设置/首选项”对话框Ctrl + Alt + S中禁用此选项并配置注释列表。转到构建，执行，部署|编译器 。

@Nullable

@Nullable注释可帮助您检测：

  • 可以返回null的方法调用

  • 变量（字段，局部变量和参数），可以为null

在父方法中使用@Nullable注释的方法可以在子类方法中使用@Nullable或@NotNull注释。

该@Nullable父方法参数的注释需要@Nullable在子类中的方法参数注解。

@NotNull

实际上，@ @NotNull注释是一个明确的合同，声明：

  • 方法不应返回null

  • 变量（字段，局部变量和参数）不能包含空值

如果违反这些合同，IntelliJ IDEA会发出警告。

该@NotNull父方法的注释需要@NotNull的子类方法的注释。

在父方法中使用@NotNull注释的方法可以在子类方法参数中具有@Nullable或@NotNull注释（或者不包含任何注释）。

内容根

在PyCharm中， 内容是您当前正在使用的文件集合，可能以子文件夹的层次结构进行组织。此层次结构中最高的文件夹称为内容根文件夹或内容根 （显示为 ）简而言之。项目至少有一个内容根文件夹，默认情况下是项目文件夹本身。

拥有多个内容根使您可以处理来自几个没有共同直接父级的目录中的文件。当您使用静态内容（例如图标）时，这很有用。您可以将它们全部保存在一个文件夹中，然后将此文件夹指定为多个项目中的额外内容根。

内容根类型

默认情况下，内容根文件夹中的所有文件都涉及索引，搜索，解析，代码完成等。要更改此状态，可以将内容根目录中的文件夹分配给以下类别：

  • 常规内容根 ，按照配置内容根部分中的描述创建。这些根被标记 。

    内容根是包含组成项目的文件的文件夹。

  • 源根 （或源文件夹;显示为 ）。

    这些根包含实际的源文件和资源。PyCharm使用源根作为解析导入的起点。

    源根目录下的文件根据其类型进行解释。PyCharm可以解析，检查，索引和编译这些根的内容。

  • 资源根 （或资源文件夹;显示为 ）。

    这些根用于应用程序中的资源文件（图像，样式表等）通过将文件夹分配给此类别，您可以告诉PyCharm可以相对于此文件夹引用其中和其子文件夹中的文件，而不是指定它们的完整路径。

  • 被排除的根 （显示为 ）是PyCharm“几乎无视”的那些。

    这些根包含PyCharm在索引，搜索，解析，观看等时忽略的文件和文件夹。

    PyCharm看不到被排除的根。通常，人们希望排除临时构建文件夹，生成的输出，日志和其他项目输出。排除不必要的路径是显着提高性能的好方法。

  • 模板根标记为包含各种Web项目的模板。

没有内容根的模块：依赖项的集合

模块可以单独用作其他模块的依赖项集合。在这种情况下，您可以在相应模块上添加依赖项，而不是单独指定必要的依赖项。

显然，用于此目的的模块不需要内容根。

功能分支用于处理特定功能。它通常由从主干下载的数据构成，并在完成对功能的处理时集成回主干。您可以应用所有更改或选择更改的子集。PyCharm使用合并的更改创建一个更改列表，并提供它以进行提交。

    • tags ：选择此选项可将更改从特定分支应用于当前分支。从“ 标签”弹出窗口中选择源分支。

      要应用所有更改而不管选择，请单击“ 全部合并”按钮。

    • 接受你的 - 单击此按钮让PyCharm强制你的更改。

使用Logcat进行调试

在IntelliJ IDEA中，通过支持存储系统调试输出日志的logcat功能来提供Android应用程序的调试。当模拟器抛出错误时，日志消息包括堆栈跟踪，因此您可以导航到源代码中的异常位置 。

logcat功能由Android Debug Bridge （adb）处理。此服务支持您的开发环境，Android设备，模拟器和其他工具（例如DDMS）之间的交互。

如果同时启动使用ADB的各种工具，它们可能会相互冲突，因此建议在从IntelliJ IDEA切换到ADB管理的工具之前禁用logcat功能 。

在IntelliJ IDEA中，可以通过Android Monitor工具窗口的Logcat选项卡获得logcat功能。默认情况下，每次部署和成功启动应用程序时，都会自动激活该选项卡。

当logcat连接到设备时，它会自动滚动到日志的底部。如果要停止自动滚动，请使用鼠标开始滚动日志。

在本页面：

  • 打开和关闭Logcat功能
  • 显示和隐藏Logcat选项卡
  • 定义要显示的日志数据的范围

打开和关闭Logcat功能

要启用/禁用logcat，请在主菜单中导航到Tools | Android并切换“ 启用ADB集成”选项。

显示和隐藏Logcat选项卡

默认情况下，每次部署和成功启动应用程序时，都会自动激活Logcat选项卡。您可以通过执行以下步骤来禁用Logcat窗格的自动显示：

  1. 开始创建Android运行/调试配置或打开现有配置进行编辑（在主菜单中，导航到“运行”|“编辑配置”并选择所需的配置）。

  2. 切换到其他选项卡并清除自动显示logcat复选框。

定义要显示的日志数据的范围

在调试会话期间，您可以切换到Android监视器工具窗口并配置要显示的日志数据的范围。您可以：

  • 选择仅查看与特定进程相关的消息。

  • 定义要显示的消息的日志级别。

  • 创建日志数据过滤器配置。

有关如何配置这些选项的详细说明，请参阅Android Monitor工具窗口 。
