在适当的位置提取Sass变量

  1. 在编辑器中，选择要用变量替换的表达式。您可以自己动手或使用智能表达式选择功能让PyCharm为您提供帮助。因此，请执行以下操作之一：
    • 突出显示表达式。然后选择Refactor |提取|主菜单或上下文菜单上的变量 。

      或者，按Ctrl + Alt + V.

    • 将光标放在表达式之前或之内。选择Refactor |在主菜单或上下文菜单中提取变量 。或者按Ctrl + Alt + V.从“ 表达式”列表中，选择表达式。为此，请单击所需的表达式。或者，使用向上和向下箭头键导航到感兴趣的表达式，然后按Enter键将其选中。

  2. 如果找到多个选定表达式，请选择“ 仅替换此事件”或“ 找到多个匹配项”弹出窗口中的“ 替换所有匹配项 ”。

    要选择所需选项，只需单击它即可。或者，使用向上和向下箭头键导航到感兴趣的选项，然后按Enter键将其选中。

  3. 在源代码中选择将声明新变量的位置。声明可以是全局的 （变量在整个文件中可用）或本地声明（变量在使用前立即声明，并且仅在当前块中可用）。

  4. 指定变量的名称。执行以下操作之一：

    • 从列表中选择一个建议的名称。为此，请双击合适的名称。或者，使用向上和向下箭头键导航到感兴趣的名称，然后按Enter键选择它。完成后，按Escape键。

    • 通过键入来编辑名称。名称显示在带红色边框的框中，并在您键入时更改。完成后，按Escape键。

表达式列表包含适用于编辑器中当前光标位置的所有表达式。

在弹出窗口中浏览建议的表达式时，编辑器中突出显示的代码会相应更改。

使用对话框提取Sass变量

如果在“编辑器”设置中清除了“ 就地启用重构”复选框，则“ 提取变量”重构将通过“ 提取变量”对话框执行 。

  1. 选择所需的表达式，并如上所述调用Extract Variable refactoring。

  2. 如果检测到当前插入符号位置有多个表达式，则会出现“ 表达式”列表。如果是这种情况，请单击表达式以选择它。或者，按向上或向下导航到感兴趣的表达式，然后按Enter键将其选中。

  3. 在Sass的Extract Variable对话框中 ：

    • 指定变量名称。您可以从列表中选择一个建议的名称，或在“ 名称”字段中键入名称 。

    • 指定声明的地方。从下拉列表中选择所需的位置（全局或本地）。

    • 如果找到多个选定表达式，则可以通过选中相应的复选框来选择替换所有找到的匹配项。如果要仅替换当前事件，请清除“ 替换所有实例”复选框。

    • 单击确定 。

共享IDE设置

PyCharm允许您在IDE的不同实例之间共享设置。如果您在不同的计算机上工作，这可以帮助您重新创建一个舒适的工作环境，并避免看起来或表现与您习惯的不同的烦恼。

您可以通过以下方式之一共享设置：

  • 通过使用IDE设置同步功能。它使用JetBrains服务器，因此无需其他配置。已同步的设置已链接到您的JetBrains帐户 ，因此其他用户无法使用这些设置。

    您可以同步的设置包括：IDE主题，键盘映射，配色方案，系统设置，UI设置，菜单和工具栏设置，项目视图设置，编辑器设置，代码完成设置，参数名称提示，实时模板，代码样式和列表启用和禁用插件。

  • 通过配置设置存储库 。这允许您同步任何可配置组件（启用和禁用插件列表除外），但需要使用您要共享的设置设置Git存储库。

    如果要在团队成员中实施相同的设置，此选项很有用。

通过设置同步共享设置

确保在“插件”下的“设置/首选项”对话框（ Ctrl + Alt + S ）中启用了“ IDE设置同步 ” 插件 。

在IDE实例之间同步设置

  1. 登录以下任一项：

    • 您的IDE：从主菜单中选择Help |注册 ，选择使用JetBrains帐户激活您的许可证并输入您的凭据。

    • 工具箱应用程序 ：单击齿轮图标在应用程序的右上角，选择“设置” ，然后单击“ 登录”按钮。请注意，通过登录Toolbox App，您将自动登录到您运行的所有JetBrains产品。

  2. 在PyCharm窗口的右下角，单击齿轮图标并选择启用设置同步 。您的本地设置将导出到链接到您帐户的JetBrains存储库。

  3. 如果要自动同步所有已启用和已禁用插件的列表，请选择“ 静默同步插件”选项。有关如何禁用手动同步插件的说明，请参阅同步插件 。

  4. 在要应用这些设置的其他计算机上，单击齿轮按钮，然后选择“ 启用同步” 。在打开的对话框中，单击“从帐户获取设置”以从存储库导入设置。

    如果要使用本地设置覆盖存储库，请单击“ 保留并同步本地设置” 。

每次运行不同的IDE实例时，您的本地设置将自动与存储在存储库中的设置同步（或者在超过一小时不活动后激活它），或者当任何这些设置被修改并且已应用此更改时。

同步插件

安装或卸载插件或更改其状态（启用/禁用）时，可以将这些更改应用于所有IDE安装。

如果要在IDE实例之间自动同步插件，请在启用设置同步时选择“ 静默同步插件”选项。

手动同步插件

  1. 在PyCharm窗口的右下角，单击齿轮图标并选择同步插件 。

  2. 将打开一个对话框，显示自上次同步以来已修改的所有插件的列表。单击每个插件旁边的箭头按钮，然后选择修改插件的状态，将存储库状态应用于所有安装，在本地跳过此更改，或在所有IDE实例中跳过它。

  3. 在为每个插件选择要执行的操作后，单击“ 应用更改” 。

通过设置存储库共享设置

在开始配置设置存储库之前，请确保在“插件”下的“设置/首选项”对话框（ Ctrl + Alt + S ）中启用了“设置存储库 ” 插件 。

配置设置存储库

  1. 在任何托管服务上创建Git存储库，例如Bitbucket或GitHub 。

  2. 在安装了要共享其设置的PyCharm实例的计算机上，选择“ 文件”|从主菜单中设置存储库 。指定您创建的存储库的URL，然后单击“ 覆盖远程” 。

  3. 在要应用设置的每台计算机上，在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，展开“ 工具”节点，然后选择“设置存储库” 。指定您创建的存储库的URL，然后单击“ 覆盖本地” 。

    如果希望存储库保留远程设置和本地设置的组合，则可以单击“ 合并” 。如果检测到任何冲突，将显示一个对话框，您可以在其中解决这些冲突。

    如果要使用本地设置覆盖远程设置，请单击“ 覆盖远程” 。

如果您选择使用Bitbucket来托管您的存储库，建议使用App密码进行身份验证。您需要为存储库设置读/写权限。

每次执行更新项目或推送操作，或关闭项目或退出PyCharm时，本地设置将自动与存储库中存储的设置同步。

在第一次同步时，系统将提示您指定用户名和密码。建议使用访问令牌进行GitHub身份验证。如果由于某种原因，您想使用用户名和密码而不是访问令牌，或者您的Git托管服务提供商不支持它，则建议配置Git凭据帮助程序 。

支持macOS Keychain ，这意味着您可以在所有基于IntelliJ Platform的产品之间共享您的凭据（如果原始IDE与请求者IDE不同，系统将提示您授予访问权限）。

如果要禁用自动设置同步，请从主菜单中选择文件|设置|工具|设置存储库并禁用“ 自动同步”选项。您可以通过选择VCS |手动更新设置从主菜单中同步设置 。

通过其他只读存储库共享更多设置

除了Settings Repository之外 ，您还可以配置任意数量的其他存储库，其中包含您要共享的任何类型的设置，包括实时模板，文件模板，方案，部署选项等。

这些存储库称为只读源 ，因为它们不能被覆盖或合并，只是用作设置源。

要配置此类存储库：

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具 ”下的“设置存储库 ”。

  2. 点击并添加包含您要共享的设置的GitHub存储库的URL。

与只读源的设置同步的方式与设置存储库的方式相同。

摩卡

Mocha是一个JavaScript测试框架，对执行异步测试场景特别有用。WebStorm与Mocha集成，因此您可以在IDE外部使用它。您可以在树视图中查看测试结果，并从那里轻松导航到测试源。测试状态显示在编辑器的测试旁边，可以选择快速运行或调试它。

在开始之前，请确保您的计算机上有Node.js.

安装Mocha

您还可以在Node.js和NPM页面上安装mocha包，如npm和Yarn中所述 。

  • 在嵌入式终端 （ Alt + F12 ）中，键入以下命令之一：

    • npm install mocha在项目中npm install mocha进行本地安装。

    • npm install -g mocha用于全局安装。

    • npm install --save-dev mocha将Mocha安装为开发依赖项 。

    另请参阅Mocha官方网站上的入门 。

运行测试

使用WebStorm，您可以直接从编辑器快速运行单个Mocha测试，或创建运行/调试配置以执行部分或全部测试。

从编辑器运行单个测试

  • 点击要么在左侧排水沟中，从列表中选择“运行<test_name> ”。

    由于测试状态图标，您还可以在编辑器中查看测试是否已通过或失败在左边的排水沟里。

创建Mocha运行配置

  1. 在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。运行/调试配置：Mocha对话框打开。

  2. 指定要使用的节点解释器以及mocha包的位置。

  3. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  4. （可选）配置在相关源文件中的更改时自动重新运行测试。为此，请在Extra Mocha选项字段中添加--watch标志。

  5. 指定要运行的测试。这可以是特定的测试或套件，整个测试文件或包含测试文件的文件夹。您还可以定义模式以仅运行匹配文件中的测试。

  6. 选择要运行的测试中使用的接口 。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右侧。

  2. 测试服务器自动启动，无需您执行任何步骤。在“ 运行”工具窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在Test Runner选项卡中 ，单击在工具栏上。WebStorm将执行上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请在其上下文菜单中选择“运行<测试名称> ”。

有关详细信息，请参阅重新运行测试 。

导航

使用WebStorm，您可以在文件和相关测试文件之间跳转，或者从Test Runner选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择Go To |测试或转到|从上下文菜单中测试主题 ，或者只需按Ctrl + Shift + T.

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源 ”。测试文件在编辑器中打开，光标位于测试定义中。

  • 对于失败的测试，WebStorm会从堆栈跟踪中引导您进入测试中的故障线。如果确切的行不在堆栈跟踪中，您将进入测试定义。

测试文件应遵循流行的命名约定，例如，具有.test. ， .spec.或_spec.后缀，应位于源文件旁边或test文件夹中。

调试测试

使用WebStorm，您可以直接从编辑器快速开始调试单个Mocha测试，或创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 点击要么在左侧排水沟中，从列表中选择Debug <test_name> 。

通过运行/调试配置启动测试调试

  1. 如上所述创建Mocha运行/调试配置。

  2. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右侧。

  3. 在打开的调试工具窗口中，像往常一样继续： 逐步执行测试 ， 停止并恢复测试执行， 暂停时检查测试 ，等等。

监控代码覆盖率

使用WebStorm，您还可以监控Mocha测试覆盖的代码量。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记已覆盖和未覆盖的行。要监控覆盖范围，您需要安装nyc ，即伊斯坦布尔的命令行界面。

要安装nyc

  • 在嵌入式终端 （ Alt + F12 ）中，键入：

    npm install --save-dev nyc

您也可以安装伊斯坦布尔本身，版本1.1.0或更高版本。这些版本支持ES6和TypeScript。

使用coverage运行测试

  1. 如上所述创建Mocha运行/调试配置。

  2. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右侧。
    或者，从编辑器快速运行特定套件或测试覆盖：单击要么在左侧排水沟中，从列表中选择带有Coverage的Run <test_name> 。

  3. 在Coverage工具窗口中监视代码覆盖率。该报告显示测试覆盖了多少文件以及其中包含的行的百分比。从报告中，您可以跳转到该文件并查看覆盖的线条 - 标记为绿色 - 以及未覆盖的线条 - 标记为红色：

管理Mercurial分支和书签

使用WebStorm，您可以使用命名分支和轻量级分支（书签） 。WebStorm提供了在分支和书签之间创建，合并和切换的界面，请参阅在工作目录之间切换 。您还可以在嵌入式终端中运行命令，请参阅使用嵌入式本地终端 。

在本页面：

  • 打开分支列表
  • 创建命名分支
  • 创建书签
  • 关闭一个分支
  • 合并命名分支和书签
  • 将命名分支或书签与另一个命名分支或书签合并
  • 将命名分支或书签与变更集合并

有关在分支和书签之间切换的信息，请参阅在工作目录之间切换 。

打开分支列表

大多数具有分支和书签的操作都是从分支列表中调用的。

该列表显示项目根目录下的所有Mercurial存储库，所有已命名的分支以及其中的所有书签。当前存储库和当前书签都标有勾号。当前命名分支的名称显示在状态栏上的专用hg区域中：

要打开“ 分支”列表，请执行以下操作之一：

  • 在状态栏上，单击专用hg区域中当前命名分支的名称。

  • 在主菜单上，选择VCS | Mercurial |分支机构 。

  • 在“ 编辑器”或“ 版本控制”工具窗口的上下文菜单上，选择“ Mercurial”分支机构 。

创建命名分支

新分支立即变为活动状态，其名称显示在hg区域的状态栏中。

  1. 在“ 分支”列表中，单击“ 新建分支” 。

  2. 在打开的“ 创建新分支”对话框中，指定新分支的名称。

创建书签

  1. 在“ 分支”列表中，单击“ 新建书签” 。

  2. 在打开的“ 新建书签”对话框中 ，指定要创建的书签的名称。

  3. 指定是否要立即切换到新书签。

    • 要激活新书签，从而启用跟踪和更新书签标识的轻量级分支，请清除“ 非活动”复选框。新书签立即变为活动状态，其名称在分支列表中标有勾号。

    • 要创建非活动书签，即保留在当前轻量级分支（书签）或命名分支中并稍后切换到新书签，请选中“ 非活动”复选框。

关闭一个分支

根据Mercurial工作流程 ，当您完成功能开发并且不期望任何进一步的更改时，您将关闭相应的分支。在活动分支之间， 日志视图等中不显示已关闭的分支。要关闭分支，请执行以下操作：

  1. 在“ 分支”弹出窗口中，单击“ 关闭分支” 。将显示“ 提交更改”对话框 。

  2. 单击提交并关闭 。将提交所有更改，并关闭当前分支。

请注意，如果“ 存储库”部分中列出了多个存储库 ，则相应的菜单选项将切换为“ 关闭”分支 ，并且close操作将应用于所有存储库 。

合并命名分支和书签

您可以将命名分支或书签与另一个命名分支，另一个书签或由标记或修订号标识的特定变更集合并。

使用数据库控制台

此功能仅在Ultimate版本中受支持。

数据库控制台是SQL文件，您可以在其中编写和执行SQL语句。与临时文件不同，控制台附加到数据源。创建数据源时，会自动创建数据库控制台。如有必要，您可以创建其他控制台。

创建数据库控制台

要创建控制台，请在“ 数据库”工具窗口中使用以下操作之一（“ 查看”|“工具Windows”|“数据库” ）。

  • 单击数据源，然后选择“ 文件”|新的|控制台 。

  • 右键单击数据源，然后选择“ 新建”|控制台 。

  • 单击数据源，按Alt + Insert ，然后选择控制台 。

  • 单击数据源，按Ctrl + Shift + F10 ，然后选择新建控制台 。

新控制台表示与数据源的新连接。您只能使用一个数据源连接，并且所有控制台都使用相同的一个连接。在启用单连接模式中，阅读有关单连接模式的更多信息。

打开一个控制台

  • 在“ 文件”工具窗口（“ 查看”|“工具Windows”|“文件” ）中，导航到“ 划痕和控制台”数据库控制台| <data_source_name> ，双击控制台文件。

  • 在“ 数据库”工具窗口中，单击“ 跳转到控制台”图标（ ）并选择一个控制台。

  • 单击数据源，按F4 ，然后选择所需的控制台。

修改控制台设置

  • 打开控制台，然后单击“ 设置”图标（ ）。

    在控制台中更改SQL方言

    右键单击输入窗格的编辑区域，选择“ 更改方言”（<current_dialect>） ，然后选择所需的方言。

    默认情况下，SQL控制台方言由附加到控制台的数据源定义。

    关闭控制台

    1. 点击 （ Ctrl + Shift + F4 ）关闭数据库控制台工具窗口。

    2. 点击在编辑器选项卡（ Ctrl + F4 ）上关闭输入窗格。

    您可以：

    • 重命名您的游戏机。创建新控制台时，它具有数据源的名称，最后带有数字，例如MySQL_1 。如果要为控制台提供更具描述性的名称，请选择控制台并执行以下操作之一：

    JSTestDriver

    WebStorm支持与JSTestDriver测试框架的集成。

    1. 将测试框架配置为WebStorm JavaScript库 。

    2. 如“ 管理插件”中所述，在“ 插件”页面上安装JSTestDriver插件。

    在项目中配置测试框架

    1. 下载您选择的框架并将其配置为WebStorm JavaScript库 。

    2. 执行以下操作之一：

      • 将项目文件夹添加到库范围 。

      • 使用Add <test framework>支持意图操作在测试创建期间即时启用框架支持。

      • 要使用Jasmine，请将jasmine-jstd-adapter添加到配置文件中。

        打开jsTestDriver.conf并在其中键入以下代码：

        load：lib / jasmine / jasmine.js lib / jasmine-jstd-adapter / JasmineAdapter.js

    手动创建测试运行器配置文件

    测试运行器配置文件定义用于加载加载顺序的测试和生产文件。WebStorm将任何扩展名为*.jstd或*.conf文件视为测试运行器配置文件。

    创建配置文件

    1. 在项目树中，选择生产和测试文件夹的父文件夹，然后选择新建|上下文菜单中的文件 。

    2. 在打开的“ 新建文件”对话框中，键入扩展名为jstd或conf的配置文件的名称。

    3. 在编辑器中打开新文件，并指定当前文件夹的完整路径以及要相对于其加载的文件的路径。在文件名模式中使用通配符。

    4. 使用YAML完成配置文件，请参阅测试运行器配置文件的说明 。

    运行测试

    使用WebStorm，您可以直接从编辑器快速运行单个JSTestDriver测试，或创建运行/调试配置以执行部分或全部测试。

    • 点击要么在左侧排水沟中，从列表中选择“运行<test_name> ”。由于测试状态图标，您还可以在编辑器中查看测试是否已通过或失败在左边的排水沟里。

    仅当测试中使用的测试框架与项目相关联时，箭头才会出现，因此WebStorm可以识别测试。

    创建JSTestDriver运行配置

    1. 在“ 项目”工具窗口中选择一个测试文件，然后从上下文菜单中选择“ 创建<文件名> ”。将打开“ 运行/调试配置：JSTestDriver”对话框。

    2. 指定要运行的测试，配置文件的路径以及在测试执行之前要执行的活动。

    启动WebStorm默认的JSTestDriver测试服务器

    1. 确保项目中至少有一个配置文件。

    2. 打开JSTestDriver Server工具窗口（ View | Tool Windows | JSTestDriver Server ），然后单击在工具栏上。

    在完成单元测试时停止服务器

    • 单击“ 停止本地服务器”工具栏按钮 。

    要使用在另一台计算机上运行的测试服务器或侦听自定义端口，请根据特定于服务器的说明启动它，并在“ 运行/调试配置：JSTestDriver”对话框的“ 服务器”中指定其URL地址。

    捕获浏览器

    1. 如果JSTestDriver服务器尚未运行，请启动它，然后切换到JSTestDriver Server工具窗口。

    2. 要使用JSTestDriver的远程控制台启动本地浏览器，请执行以下操作之一：

      • 单击指示所选浏览器的图标。

      • 如果浏览器已打开，请使用URL只读字段复制Capture a browser的内容， 并将URL粘贴到地址栏中。

      在任何一种情况下，指示所选浏览器的图标都将变为活动状态。

    3. 切换到JSTestDriver Server工具窗口，然后单击指示刚刚打开的浏览器的图标。WebStorm显示一条消息，通知您已准备好执行测试。

    通过运行配置运行测试

    1. 从主工具栏上的列表中选择JSTestDriver运行/调试配置，然后单击在列表的右侧。

    2. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 插件” 。

    2. 在“ 插件”对话框中，单击然后单击“ 管理插件存储库” 。

    3. 在“ 自定义插件存储库”对话框中，单击“确定” 并指定您的存储库URL。它必须指向updatePlugins.xml文件的位置。该文件可以与自定义插件位于同一服务器上，也可以位于专用插件上。

    4. 在“ 自定义插件存储库”对话框中单击“ 确定 ”以保存插件存储库列表。

    5. 单击“ 插件”对话框中的“ 确定 ”以应用更改。

    要浏览自定义插件存储库，请在“ 插件”对话框的“ 市场”选项卡中键入repository:然后键入repository:的URL。例如：

    存储库：“http://plugins.example.com:8080/updatePlugins.xml”myPlugin

    或者，您可以使用自定义存储库URL替换默认的JetBrains插件存储库。如果您只想从PyCharm获得自定义存储库插件，这将非常有用。为此，请编辑平台属性或VM选项文件，如下所述。有关更多信息，请参阅调整PyCharm 。

    用自定义插件替换默认插件存储库

    1. 在主菜单上，选择“ 帮助”|编辑自定义属性或帮助|编辑自定义VM选项 。

    2. 将idea.plugins.host属性添加到VM选项文件的平台属性文件或-Didea.plugins.host选项。例如：

      idea.plugins.host = “http://plugins.example.com:8080/”
      -Didea.plugins.host = “http://plugins.example.com:8080/”

      要添加多个URL，请使用分号（ ; ）分隔它们。

      确保没有plugins.jetbrains.com网址。

    3. 重启PyCharm。

    如果使用自定义插件库替换默认插件存储库，则“ 插件”对话框的“ 市场”选项卡上的搜索字段将仅浏览自定义存储库中的插件。

    必需的插件

    项目可能需要添加对某些技术或框架的支持的插件。您可以将此类插件添加到当前项目所需的插件列表中，以便PyCharm验证插件是否已安装并启用。如果您忘记了某个插件，或者您的团队中的某个人在项目上工作时不了解依赖关系，它会通知您。

    为您当前的项目添加所需的插件

    1. 确保安装了所需的插件。

    2. 在Settings / Preferences对话框（ Ctrl + Alt + S ）中，选择Build，Execution，Deployment |必需的插件 。

    3. 在Required Plugins对话框中，单击并选择插件。（可选）指定插件的最小和最大版本。

    要指定所需的PyCharm版本，请将IDE Core添加到所需插件列表中。

    添加所需的插件后，当您在PyCharm中打开项目时，它将通知您插件是否已禁用，未安装或需要更新。

    单击通知消息中的链接以快速启用，安装或更新所需的插件。

    开发自己的插件

    您可以使用任何版本的PyCharm来开发插件。它提供了一个开放的API，一个专用的SDK，模块和运行/调试配置来帮助您。

    建议的工作流程是使用Gradle。还支持使用内部PyCharm构建系统的旧工作流程。有关更多信息，请参阅“ IntelliJ平台插件快速入门指南” 。
