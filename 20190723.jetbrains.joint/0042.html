构建并启动应用程序

  • 如果您是第一次启动应用程序，或者自上次运行以来您已更新其本机代码，请选中“ 生成并启动”复选框。

  • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，PyCharm会等待您在模拟器中打开您的应用程序，并启用远程调试，如React Native官方网站上所述。

  • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序会自行处理该过程。

平台

在此区域中，选择应用程序所针对的平台。根据您的选择，PyCharm将使用react-native run-ios或react-native run-android运行bundler。

  • 要模拟Android平台，请使用Android虚拟设备 。

  • 要模拟iOS平台，您需要全局安装ios-sim命令行工具 。您可以通过Node Package Manager（npm） ，查看npm和Yarn ，或运行sudo npm install ios-sim -g命令，具体取决于您的操作系统。

参数

在此字段中，键入要传递给React Native的参数，例如，通过‑‑simulator标志指定模拟器类型： - ‑‑simulator="iPhone 4s" 。

浏览器进行调试

在调试React Native应用程序时，PyCharm依赖于Chrome运行时，它由React Native本身使用。您也可以将DevTools与PyCharm一起使用。当您启动调试会话时，PyCharm会启动一个新的Chrome实例并附加到它。

请注意，在PyCharm中停止内置调试程序后，Chrome Headless将继续运行。

Bundler主持人

在此字段中，指定运行捆绑程序的主机，默认值为localhost 。

如果您使用的是Expo，则可能需要更改默认的bundler主机以确保与调试器成功连接。此连接可能会失败，因为默认情况下，PyCharm使用localhost开始调试，而Expo需要127.0.0.1或外部IP地址，具体取决于Metro捆绑器配置的“连接”字段中选择的内容。

Bundler端口

在此字段中，指定捆绑程序运行的端口。

  • PyCharm会自动选择8081 ，这是React Native捆绑器的默认设置。从React Native官方网站了解更多信息。

  • 如果您的应用程序使用Expo，请将端口更改为19001 。

节点解释器

在此字段中，指定要使用的Node.js解释器。这可以是Windows子系统Linux上的本地Node.js解释器或Node.js。

React Native包

在此字段中，指定react-native-cli的路径。

工作目录

指定应用程序的工作目录 。默认情况下，该字段显示项目根文件夹 。

环境变量

在此字段中，可以选择键入环境变量以执行react-native run-android或react-native run-ios命令。

Alt + F9

单击此按钮可恢复程序执行并暂停，直到执行点到达编辑器中当前光标位置的行。不需要断点。实际上，在插入符号处为当前行设置了临时断点，一旦程序执行暂停，该断点就会被删除。因此，如果插入符号位于已经执行的行，则程序将刚刚恢复以进一步执行，因为无法回滚到先前的断点。当您深入了解方法序列并需要一次性退出多个方法时，此操作尤其有用。

如果在将您带到指定行之前为应执行的行设置了断点，则调试器将在第一个遇到的断点处暂停。

当在特定行需要一种临时断点时，请使用此操作，不应中断程序执行。

断点

断点是源代码标记，它允许您在特定点暂停程序执行并检查其行为。

设置后，断点将保留在项目中，直到您明确删除它（ 临时行断点除外）。如果在外部修改了带有断点的文件，例如，通过VCS更新或在外部编辑器中更改，并且行号已更改，则将相应地移动断点。请注意，进行此类更改时必须运行IntelliJ IDEA，否则它们将被忽略。

线断点

可以在可执行的代码行上设置换行符。线程执行在具有此断点的行之前暂停，IntelliJ IDEA在该线程的堆栈上显示堆栈帧。

设置换行符

  1. 将插入符号放在要设置断点的可执行代码行上。

    如果要在默认类构造函数中设置断点，请将其设置在此类的第一行，因为默认构造函数已映射到它。

  2. 单击该行旁边的左侧装订线，或按Ctrl + F8 。

    要设置临时行断点，请按Ctrl + Shift + Alt + F8 。它会在被击中后立即从您的项目中删除。

对于lambda表达式，您可以在一行中设置多个断点。单击左侧装订线，然后从列表中选择要设置断点的方法：

在折叠方法上设置断点时，将在此方法声明后的第一个可执行行上设置断点。

删除换行符

  • 单击左侧装订线中的断点图标。

为避免意外删除断点并丢失其参数，您可以选择通过将断点拖动到编辑器或单击鼠标中键来删除断点：在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）转到“ 构建”，“执行” ，部署|调试器并选择拖动到编辑器或单击鼠标中键 。只需单击断点即可切换其状态（启用/禁用）。

方法断点

方法断点用于响应程序进入或退出特定方法。它们允许您在方法级别跟踪程序流程，并检查进入和退出条件。

设置方法断点

  1. 将插入符号放在方法声明中。

  2. 单击该行旁边的左侧装订线。

默认情况下，IntelliJ IDEA设置模拟方法断点，它是在所有实现或重写方法的第一个语句中设置的行断点的组合。此类模拟断点不会影响调试的应用程序性能，而“真正的”方法断点会因JVM设计而显着降低调试器的速度。

模拟方法断点不适用于没有行号信息的本机方法和类。如果是这种情况，请右键单击方法断点并取消选择“ 模拟”选项：

服务器命令

在服务器端使用以下命令启用远程调试：

rdebug-ide --host 0.0.0.0 --port <端口号> --dispatcher-port <端口号> - $ COMMAND $

此只读字段显示应在服务器端输入以启动调试服务器的命令。此处的端口和调度程序端口号取自端口和本地端口字段。$COMMAND$对应于要调试的脚本的名称。

远程主机

指定将进行远程调试的主机的URL。

远程端口

指定服务器端的端口号。请注意，在此字段中输入的数字将自动用于rdebug命令。键入端口号时，建议的命令字符串会相应更改。

远程根文件夹

在服务器端指定要调试的脚本所在的根目录。此字段定义到本地根文件夹的映射。

本地港口

指定本地端口号。请注意，在此字段中输入的数字将自动用于rdebug命令。键入本地端口号时，建议的命令字符串会相应更改。

本地根文件夹

指定包含相关脚本的本地根目录。输入路径，或单击并在“选择路径”对话框中找到所需的根。

描述

主办

在此字段中，键入运行PHP内置Web服务器的主机的名称。默认情况下，host设置为localhost ，因为内置服务器位于您的计算机上。

港口

使用此旋转框指定运行PHP内置Web服务器的端口。默认情况下，此端口设置为端口80 。您可以将端口号设置为以1024及更高版本开头的任何其他值。

文件根

在此字段中，键入将被视为服务器文档根目录的文件夹的完整路径。这可以是项目根目录或其下的任何其他文件夹。文档根目录下的所有文件夹将以递归方式提交给PHP解释器。

将在上面指定的host：port上通过HTTP访问服务器文档根文件夹。应用程序其他页面的URL地址将根据此映射进行组合。

手动键入路径或单击“ 浏览” 并在打开的对话框中选择相关文件夹。

使用路由器脚本

选中此复选框可在每次启动HTTP请求时执行PHP路由器脚本 。该脚本针对每个HTTP请求运行。如果此脚本返回FALSE ，则按原样返回请求的资源。否则，脚本输出将返回到浏览器。

在该字段中，指定要运行的脚本的位置。手动键入路径或单击“ 浏览” 并在打开的对话框中选择文件。

在此字段中，指定要传递给PHP可执行文件的选项 。它们覆盖PHP解释器的默认行为和/或确保执行其他活动。

如有必要，请单击并在“ 命令行选项”对话框中键入所需的选项。在新行上键入每个选项。关闭对话框时，它们都显示在命令行选项字段中，空格作为分隔符。

在此字段中，使用源指定文件夹外部的文件的位置，并通过相对路径引用。手动键入路径或单击“ 浏览” 并在打开的对话框中选择所需的文件夹。

环境变量

在此字段中，指定要传递给内置服务器的环境变量 。有关详细信息，请参阅Apache中的环境变量 。

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

如果使用基于目录的项目格式 ，则运行/调试配置的设置将存储在.idea\runConfigurations文件夹中的单独.xml文件中（如果共享运行/调试配置），或存储在.idea\workspace.xml否则为.idea\workspace.xml文件。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以用于共享配置，否则存储在.iws文件中。

允许并行运行

禁用时，每次启动新的运行/调试配置时，IntelliJ IDEA都会检查是否存在相同运行/调试配置的其他实例，并显示确认对话框。如果在确认对话框中单击“ 确定 ”，则将停止第一个运行器实例，下一个实例将取代它。

当某些资源的使用可能导致冲突时，或者当启动两个相同类型的运行/调试配置时，会消耗太多的CPU和内存资源，这是有意义的。

如果选中此复选框，则可以根据需要启动任意数量的运行/调试配置实例。这样，每个运行器将在Run Tool Window或Debug Tool Window的自己的选项卡中启动 。
