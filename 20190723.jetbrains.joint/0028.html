Grunt工具窗口

Gruntfile.js的上下文菜单 - 显示Grunt任务
查看|工具窗口|咕噜

使用此工具窗口运行Grunt任务 。执行任务的结果显示在“ 运行”工具窗口中 。工具窗口显示Grunt输出，报告发生的错误，列出尚未找到的包或插件等。最后执行的任务的名称显示在工具窗口的标题栏上。

在你开始之前

  1. 下载并安装Node.js.

  2. 全局安装grunt-cli软件包（Grunt命令行界面）并安装grunt软件包作为开发依赖项。有关详细信息，请参阅安装Grunt 。

运行任务和目标

在PyCharm会话期间第一次构建任务树时， Grunt工具窗口尚未打开。

打开Grunt工具窗口

  • 在“ 项目”工具窗口中选择所需的Gruntfile.js文件，或在编辑器中将其打开，然后从上下文菜单中选择“ 显示Grunt任务 ”。

构建任务树

  • 在Grunt工具窗口中，单击在工具栏上，从列表中选择所需的Gruntfile.js文件。默认情况下，PyCharm在项目的根目录中显示Gruntfile.js文件。

  • 如果您有另一个Gruntfile.js文件，请单击Choose Gruntfile.js并在打开的对话框中选择所需的Gruntfile.js文件。PyCharm在其标题上添加一个新节点，其中包含所选Gruntfile.js文件的路径，并在新节点下构建任务树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的任务进行排序

  • 点击在工具栏上，从菜单中选择排序依据 ，然后选择名称 。
    默认情况下，树以Gruntfile.js （选项定义顺序 ）中的顺序显示任务。

运行任务或目标

  • 双击任务或目标。

  • 在树中选择任务或目标，然后按Enter或在上下文菜单中选择“运行<任务名称> ”。

运行默认任务

  • 在树中选择根节点，然后在选择的上下文菜单中选择“运行默认值 ”。

运行多个任务或目标

  • 使用多选模式：按住Shift （对于相邻项）或Ctrl （对于非相邻项）键并选择所需的任务或目标，然后在选择的上下文菜单中选择“运行”或“ 调试 ”。

跳转到任务或目标的定义

  • 在树中选择所需的任务或目标，然后在选择的上下文菜单中选择“ 跳转到源 ”。

工具栏

添加Gruntfile

单击此按钮可为另一个Gruntfile.js文件创建任务树。从列表中选择所需的Gruntfile.js文件。PyCharm添加了一个新节点并在其下构建了一个任务树。

删除Gruntfile

单击此按钮可删除所选节点下的任务树。

重新加载任务

单击此按钮可重新构建所选节点下的任务树。更新相应的Gruntfile.js文件后，您可能需要重新构建树，因为Grunt不会动态更改树。

全部收缩

单击此按钮可隐藏所有任务树，并仅显示Gruntfile.js节点。

单击此按钮可配置当前视图并更改工具窗口的查看模式 ，有关详细信息，请参阅工具Windows查看模式 。请注意，大多数菜单项都是可以打开或关闭的选项。打开的选项在其名称的左侧有一个复选标记。Grunt特定选项是：

  • Grunt设置：选择此菜单项以打开Grunt设置对话框并重新配置Grunt和节点解释器的当前设置，请参阅任务运行器 。

  • 排序方式：选择此菜单项以配置任务在树中显示的顺序。点击在工具栏上，从菜单中选择排序依据 ，然后选择名称 。
    默认情况下，树以Gruntfile.js （选项定义顺序 ）中的顺序显示任务。

隐藏

单击此按钮可隐藏工具窗口。要再次显示它，请选择“ 查看”|工具窗口|主菜单上的咕噜声 。工具窗口再次显示所有先前构建的任务树。

树的上下文菜单

咕噜声设置

选择此菜单项以打开Grunt设置对话框并查看或编辑Node.js配置

跳转到源

选择此菜单项以打开Gruntfile.js构建当前树的Gruntfile.js文件。

重新加载任务

选择此菜单项可重新构建所选节点下的任务树。

复制路径

选择此菜单项以保存Gruntfile.js文件的路径，根据该文件将当前树构建到剪贴板。

删除Gruntfile.js

选择此菜单项可删除所选节点下的任务树。

任务或目标的上下文菜单

运行<任务/目标名称>

选择此菜单项以运行选定的任务或目标。

调试<任务/目标名称>

选择此菜单项以调试所选任务或目标。

编辑<任务/目标名称>设置

选择此菜单项以打开“ 运行/调试配置”对话框，并编辑所选任务或目标的预定义设置。

跳转到源

选择此菜单项以打开Gruntfile.js当前树的Gruntfile.js文件，并导航到所选任务或目标的定义。

语言注射

语言注入允许您处理字符串文字内的代码片段以及类似XML的语言中的标签或属性。

如果在字符串文字，标记或属性中有一段正式语言（如HTML，CSS，XML，RegExp等），则可以获得全面的语言帮助来编辑该片段。

在开始使用语言注入之前，请确保已启用 IntelliLang插件。

暂时注入一种语言

  1. 将光标放在要注入语言的字符串文字，标记或属性中，然后按Alt + Enter （或使用意图操作图标） ）。

  2. 选择“ 注入语言”或“引用”，然后选择要注入的语言。

在专用编辑器部分打开代码片段

  1. 将插入符号放在注入的代码段中，然后按Alt + Enter （或使用意图操作图标 ）。

  2. 选择编辑<语言ID>片段 。

    IntelliJ IDEA将打开一个专用的编辑器部分，用于使用注入的语言编辑代码。此编辑器提供完整的代码帮助，包括代码完成，检查，意图和代码样式操作。

使用语言注入注释

  • 在目标字符串文字前添加一个空行，然后键入以下注释：

    - language = < language_ID >

    对于注释，请使用要注入的语言的语法。语言ID通常是直观的，例如SQL，RegExp，XML，HTML。

语言片段可以与作为包装器一起起作用的前缀和后缀组合，将片段转换为语法上完整的语言单元。编辑代码时，只能在片段编辑器中看到前缀和后缀。它们不会显示在主编辑器中。

前缀和后缀可以包含在注释注释中，如下所示：

// language = <language_ID> prefix = <prefix> suffix = <suffix>

使用@Language注释

在某些语言中（例如，在Java，Groovy或Kotlin中），您还可以使用@Language("language_ID")注释来注入语言。

  1. 在目标字符串文字之前的空白行上，键入@Language("language_ID") 。

  2. 将光标放在注释上，按Alt + Enter并选择“ 添加'注释'到类路径 。

    IDE将提示您从Maven下载带有注释的库。

  3. 在搜索字段中，如果使用JDK 1.8或更高版本，请键入org.jetbrains:annotations:16.0.2 。

    对于JDK org.jetbrains:annotations-java5:16.0.2或1.7，请键入org.jetbrains:annotations-java5:16.0.2 。

  4. 单击确定 。

  5. 按Alt + Enter并选择导入类 。

配置注入规则

您可以在编辑器上配置语言注入规则IntelliJ IDEA设置的语言注入页面（ Ctrl + Alt + S ）。

所有预定义的注入规则都是为内置范围配置的。换句话说，它们是全局的（因此可用于所有IntelliJ IDEA项目）。可以为IDE或仅为一个项目配置自定义规则。要更改自定义进样的范围，请使用图标。

要配置自定义注入规则，请单击添加新规则，或复制预定义规则并更改其设置。

示例：在Java中注入SQL

您可以配置设置，以便每次使用某种方法时，IDE将注入所选语言，例如SQL。

  1. 在设置/首选项对话框（ Ctrl + Alt + S ）中，转到编辑器|语言注射 。

  2. 点击并选择Java参数 。

  3. 从ID列表中，选择SQL 。

  4. 在“ 类方法”字段中，输入要获取SQL参数的方法。您可以手动键入方法，也可以单击并在“ 选择类”对话框中选择方法。

    如果该方法采用多个参数，它们将显示在对话框中。选择要使用的那些。

  5. 应用更改并关闭对话框。

取消注射

  1. 将光标放在代码片段上，然后按Alt + Enter （或使用意图操作图标 ）。

  2. 选择取消注入语言/参考 。

要取消语言注入，您还可以删除注入注释或注释。

创建单元测试

在Android中，单元测试基于JUnit ， JUnit的平面使用足以测试专门基于Java代码的功能。

本地服务器是在本地或装入的文件夹中运行的服务器，其文档根目录 不是项目根目录的父级。您进行开发，然后将源复制到服务器。

  1. 服务器配置根文件夹和访问它的URL地址。

  2. 项目根文件夹 ，服务器上用于将数据从项目根文件夹复制到的文件夹 ，以及用于访问服务器上复制数据的URL地址之间的对应关系。这种对应关系称为映射 。

指定服务器配置的名称，类型和可见性

    • 在设置/首选项（Ctrl + Alt + S），选择下生成，执行，部署 部署 。

    • 选择工具|部署|在主菜单上配置 。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。将关闭“ 创建新服务器”对话框，并返回“ 部署”节点的“ 连接”选项卡。

    单击“ 用作默认值”工具栏按钮让PyCharm在以下情况下静默应用当前配置：

    • 自动上传已更改的文件 。

    • 无需选择目标主机即可手动上载和下载文件。

    • 将本地文件和文件夹与其远程版本进行比较。

    • 编辑远程主机上的单个文件 。

  4. 仅对此项目使用“ 可见”复选框以配置服务器访问配置的可见性。

    • 选中该复选框以限制将配置用于当前项目。此类配置无法在当前项目之外重复使用。它不会出现在其他项目的可用配置列表中。

    编辑项目历史记录

    除非绝对必要，否则请避免修改具有多个贡献者的远程分支的历史记录，例如，如果您不小心推送了一些敏感数据。

    Git允许您编辑项目历史记录。当您正在处理功能分支并想要将其清理并在与其他人共享之前使其看起来像您想要的方式时，这非常有用。例如，您可以编辑提交消息，将与相同功能相关的较小提交压缩在一起，或者将包含无关更改的提交拆分为单独的提交，将更改添加到先前的提交，依此类推。

    如果您修改了其历史记录的分支位于远程存储库中，则会拒绝推送此类修改以防止数据丢失，因此您必须强制推送更改。

    您无法修改不允许使用push --force 受保护分支的历史记录（在“ 版本控制”|“Git”下的“设置/首选项”对话框（ Ctrl + Alt + S ）中配置受保护的分支）。

    编辑提交消息

    如果您需要更改的唯一内容是提交消息，则可以在推送此提交之前对其进行编辑。

    1. 在“ 日志”视图中右键单击要编辑其消息的提交， 然后从上下文菜单中选择“重定位 ”，或按F2 。

    2. 在打开的对话框中，输入新的提交消息，然后单击“ 确定” 。

    修改先前的提交

    有时您可能过早地提交并忘记添加一些文件，或者在上次提交时注意到您要修复的错误，而不创建单独的提交。

    您可以使用Amend commit选项将上次更改附加到上一次提交。因此，您最终只能使用一个提交而不是两个不同的提交。

    1. 在“ 本地更改”视图中选择要添加到上一个提交的更改 ，然后单击在工具栏上或按Ctrl + K.

    2. 在“ 提交更改”对话框中，在提交更改之前，在右侧选择“ 修改提交”选项。

    修改任何先前的提交

    如果您需要将更改添加到任何先前的提交而不是单独提交它们，则可以使用fixup或squash操作来执行此操作。这两个命令都会对所选提交附加阶段性更改 ，但处理提交消息的方式不同：

    • squash将新的提交消息添加到原始提交

    • fixup丢弃新的提交消息，只保留原始提交的消息

    这两个命令都需要rebase，因为它们会更改提交哈希值。

    1. 进行要附加到任何早期提交的更改（它们将显示在“ 本地更改”视图中）。

    2. 在“ 日志”视图中，右键单击要使用本地更改修改的提交，然后从上下文菜单中选择“ Fixup”或“ Squash Into ”。

    3. 如果您选择压缩更改，则在必要时修改提交消息。

    4. 单击Commit按钮上的箭头，然后选择Commit and Rebase 。

    通过执行交互式rebase编辑项目历史记录

    Git允许您编辑项目历史记录，以便通过执行交互式rebase使其变得线性和有意义。这允许您在将更改从功能分支 集成到另一个分支之前，通过更改单个提交，更改其顺序，将提交压缩为一个，跳过包含无关更改的提交等来清理提交历史记录。

    编辑当前分支的历史记录

    IntelliJ IDEA允许您在将更改应用于其他分支之前编辑当前分支中的一系列最近提交。

    1. 在“ 版本控制”工具窗口的“ 日志”选项卡中，选择要编辑的一系列提交中最早的提交，右键单击它并从此处选择“ 交互式重新生成” 。

      将显示Interactive Rebase对话框，其中包含在所选提交之后在当前分支中进行的所有提交的列表。

    2. 如果您需要更改应用提交的顺序，请使用和纽扣。

    3. 使用“ 操作”列表将以下操作应用于提交（可以使用Alt +向上/向下选择操作）：

      • 选择 ：按原样应用所选提交。

      • 编辑 ：选择在应用之前编辑受此提交影响的文件。

      • 跳过 ：忽略所选的提交，因此不会应用这些更改。

      • 壁球 ：选择将所选提交与前一个提交相结合。两个提交的提交消息将合并，因此此操作将反映在分支历史记录中。如有必要，更改提交顺序，以便要添加更改的提交是列表中的前一个提交。

      • Reword ：选择此选项可在应用之前编辑提交消息。

      • 修复 ：选择此选项可将选定的提交与前一个提交相结合。修复提交的提交消息将被丢弃，因此此更改将在分支历史记录中不可见。如有必要，更改提交顺序，以便要添加更改的提交是列表中的上一个。

    编辑分支历史记录并将其集成到另一个分支中

    IntelliJ IDEA允许您在应用更改之前，在另一个分支的顶部重新绑定分支并编辑源分支历史记录。

    1. 从主菜单中选择VCS | Git | Rebase 。

    2. 在“ 分支”字段中选择要重新绑定的分支 ，在“ Onto”字段中选择目标分支。

    3. 在“ 发件人”字段中，选择要从中将所选分支应用于新基础的提交。

    4. 确保互动选项被选中，并单击再次基于 。

      将显示Interactive Rebase对话框，其中包含在所选提交之后在当前分支中进行的所有提交的列表。

    5. 如果您需要更改应用提交的顺序，请使用和纽扣。

    6. 使用“ 操作”列表将以下操作应用于提交（可以使用Alt +向上/向下选择操作）：

      • 选择 ：按原样应用所选提交。

      • 编辑 ：选择在应用之前编辑受此提交影响的文件。

      • 跳过 ：忽略所选的提交，因此不会应用这些更改。

    管理项目收藏夹

    您可以在喜爱的项目列表（收藏夹）中排列最常用的项目项（文件，文件夹，包，实例和类成员等）， 书签和断点 。IntelliJ IDEA有一个专用的收藏夹工具窗口，用于管理您的收藏夹。

    最初，有一个（空）收藏夹列表，其名称与项目相同。

    您可以根据需要创建更多收藏夹列表并管理其内容。

    使用“项目”工具窗口将项目添加到收藏夹

    1. 在项目中 ，选择要添加到收藏夹的项目。

    2. 执行以下操作之一：

      • 按Shift + Alt + F.

      • 选择文件|从主菜单添加到收藏夹 。

      • 从选择的上下文菜单中，选择“ 添加到收藏夹” 。

    3. 要将所选项目添加到现有收藏夹列表，请选择列表名称。

      要创建新的收藏夹列表并将所选项目添加到其中，请选择“ 添加到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

    使用拖放操作将项目添加到收藏夹：从“ 项目”工具窗口拖动感兴趣的项目 ，或从“ 资源管理器”或“ Finder”拖动外部文件，然后将其拖放到“收藏夹”工具窗口中的所需收藏夹列表中。

    1. 右键单击感兴趣的编辑器选项卡，然后选择以下选项之一：

      • 如果要将当前文件添加到收藏夹，请选择“ 添加到收藏夹” 。（对于当前文件，还可以使用“ 文件”|“添加到收藏夹” 。）

      • 如果要将编辑器中打开的所有文件添加到收藏夹，请选择“ 全部添加到收藏夹” 。

    2. 要将项目添加到现有收藏夹列表，请选择列表的名称。

      要创建新的收藏夹列表并将项目添加到其中，请选择“ 添加到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

    最初，有一个（空）收藏夹列表，其名称与项目相同。

    您可以根据需要创建更多收藏夹列表并管理其内容。

    使用“项目”工具窗口将项目添加到收藏夹

    1. 在项目中 ，选择要添加到收藏夹的项目。

    2. 执行以下操作之一：

      • 按Shift + Alt + F.

      • 选择文件|从主菜单添加到收藏夹 。

    3. 要将所选项目添加到现有收藏夹列表，请选择列表名称。

      要创建新的收藏夹列表并将所选项目添加到其中，请选择“ 添加到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

    使用拖放操作将项目添加到收藏夹：从“ 项目”工具窗口拖动感兴趣的项目 ，或从“ 资源管理器”或“ Finder”拖动外部文件，然后将其拖放到“收藏夹”工具窗口中的所需收藏夹列表中。

    在编辑器中，您可以将一个文件（ Shift + Alt + F ）或所有当前打开的文件添加到收藏夹。

    使用编辑器将文件添加到收藏夹

    1. 右键单击感兴趣的编辑器选项卡，然后选择以下选项之一：

      • 如果要将当前文件添加到收藏夹，请选择“ 添加到收藏夹” 。（对于当前文件，还可以使用“ 文件”|“添加到收藏夹” 。）

    2. 要将项目添加到现有收藏夹列表，请选择列表的名称。

      要创建新的收藏夹列表并将项目添加到其中，请选择“ 添加到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

      • 点击在工具栏上。

      要创建新的收藏夹列表并在其中移动项目，请选择“ 发送到新收藏夹列表” 。在“ 添加新收藏夹列表”对话框中，指定新列表的名称，然后单击“ 确定” 。

    共享IDE设置

    IntelliJ IDEA允许您在IDE的不同实例之间共享设置。如果您在不同的计算机上工作，这可以帮助您重新创建一个舒适的工作环境，并避免看起来或表现与您习惯的不同的烦恼。

    • 通过使用IDE设置同步功能。它使用JetBrains服务器，因此无需其他配置。已同步的设置已链接到您的JetBrains帐户 ，因此其他用户无法使用这些设置。

      您可以同步的设置包括：IDE主题，键盘映射，配色方案，系统设置，UI设置，菜单和工具栏设置，项目视图设置，编辑器设置，代码完成设置，参数名称提示，实时模板，代码样式和列表启用和禁用插件。

    • 通过配置设置存储库 。这允许您同步任何可配置组件（启用和禁用插件列表除外），但需要使用您要共享的设置设置Git存储库。

    确保在“插件”下的“设置/首选项”对话框（ Ctrl + Alt + S ）中启用了“ IDE设置同步 ” 插件 。

      • 您的IDE：从主菜单中选择Help |注册 ，选择使用JetBrains帐户激活您的许可证并输入您的凭据。

      • 工具箱应用程序 ：单击齿轮图标在应用程序的右上角，选择“设置” ，然后单击“ 登录”按钮。请注意，通过登录Toolbox App，您将自动登录到您运行的所有JetBrains产品。

    3. 如果要自动同步所有已启用和已禁用插件的列表，请选择“ 静默同步插件”选项。有关如何禁用手动同步插件的说明，请参阅同步插件 。

    将更改放到架子上

    1. 打开“ 版本控制”工具窗口（ Alt + 9 ）并切换到“ 本地更改”选项卡。

    2. 右键单击要放入工具架的文件或更改列表，然后从上下文菜单中选择“ 搁置更改 ”。

    3. 在“ 搁置更改”对话框中，查看已修改文件的列表。

    4. 在“ 提交消息”字段中，输入要创建的货架的名称，然后单击“货架更改”按钮。

    1. 版本控制工具窗口的保存选项卡，选择您要取消搁置一个变更或文件。

    • 您的源和测试根已正确配置 - 源根将显示为蓝色文件夹，测试文件夹将具有绿色背景。

    • 您已在项目中创建了所需的包。

    创建您的第一个测试

    鉴于我们首先编写测试而不必使用我们正在测试的类，我们将通过项目面板创建第一个测试。

    1. 右键单击要在其中创建测试的包，然后选择New | Java类 。

    2. 输入测试名称 - 假设我们正在使用JUnit进行测试，这可能是[Something] Test，例如MoodAnalyserTest 。

    3. 将光标放在类的花括号内的某处，按Alt + Insert 。

    4. 选择测试方法|从菜单中选择JUnit 4 。这将使用默认模板创建一个测试方法。填写测试名称，按enter键，光标将以方法体结束。

      您可以更改默认的测试方法模板 - 例如，如果您希望将方法名称的开头从test更改为should 。

    编写测试体

    为不存在的类和方法编写测试代码似乎是违反直觉的，但IntelliJ IDEA在保持编译器满意的同时使这一点变得简单。IntelliJ IDEA可以为您创建类和方法（如果它们尚不存在）。

    1. 编写描述您想要实现的内容的测试，在任何不存在的类上按Alt + Enter并选择“Create class' [ClassName]'”。这将为您提供最低限度的实现，使编译器满意。

      与往常一样，您可以使用IntelliJ IDEA的重构工具来创建存储结果的变量，如果类路径上有正确的库，IntelliJ IDEA将为您导入最合适的类。

    2. 继续编写测试体，包括您不需要的方法名称，再次使用Alt + Enter并选择“Create method'[methodName]'”让IntelliJ IDEA创建一个裸骨架方法。

    运行测试

    遵循TDD方法时，通常会经历Red-Green-Refactor循环。您将运行测试，看到它失败（变为红色），实现最简单的代码以使测试通过（变为绿色），然后重构代码以使您的测试保持绿色并且您的代码足够干净。

    这个循环的第一步是运行测试并看到它失败。

    鉴于我们已经使用IntelliJ IDEA的功能来创建我们正在测试的方法的最简单的空实现，我们不希望我们的测试通过。

    • 在测试内部，按Ctrl + Shift + F10运行此单独测试。

      结果将显示在运行对话框中 。测试名称旁边会有一个图标 - 红色表示异常，黄色表示失败的断言。对于任何一种类型的故障，还会显示一条消息，说明出现了什么问题。

    实施守则

    下一步是让测试通过，这意味着实现最简单的工作。

    1. 您可以使用常用方法导航到正在测试的代码 - 单击方法名称，在光标位于方法名称时按Ctrl + Alt + B ，或按Ctrl + Shift + T在测试和生产之间切换码。

    2. 对方法进行更改以使测试通过。通常使用TDD，最简单的方法可能是对您的预期值进行硬编码。稍后我们将看到迭代这个过程将如何产生更真实的生产代码。

    3. 重新运行测试，使用Shift + F10重新运行最后一次测试。

    4. 查看测试通道 - 测试方法旁边的图标应为绿色。如果没有，请进行必要的更改，直到测试通过。

    重复

    开发代码是一个迭代过程。遵循TDD风格的方法时，情况更是如此。为了消除更复杂的行为，我们为其他案例添加了测试。

    1. 在测试类中，再次使用Alt + Insert创建新的测试方法。

    2. 选择第二个测试用例，指定您正在测试的方法的更多要求。请记住，您可以使用IntelliJ IDEA的功能来创建类和方法，以使编译器满意。

    3. 运行第二个测试用例，显示它因正确原因而失败。

    4. 更改正在测试的方法中的代码以使此测试通过。

    5. 通过在测试类中按Ctrl + Shift + F10重新运行这两个测试，而不是在单个方法内，并看到两个测试现在都通过了。如果任一测试失败，请对代码进行必要的更改以确保测试通过。

    摘要

    以测试优先的方式编写第一个测试需要进行少量设置 - 创建测试类，创建测试方法，然后创建最终将成为生产代码的代码的空实现。IntelliJ IDEA可以自动完成大量初始设置。

    当您遍历整个过程，创建测试然后进行必要的更改以使这些测试通过时，您可以为所需的功能构建一套全面的测试，以及满足这些要求的最简单的解决方案。

      • 由实体豆

      • 通过Hibernate Mappings （此选项不适用于启用Hibernate的模块和会话工厂）

    1. 右键单击持久性单元或会话工厂，指向“ 新建” ，然后选择“ 实体” 。

      • 目标包：应在其中创建类的包。

      • 目标目标目录：目标源根文件夹（如果有多个）。

      • 对象/关系映射XML文件：信息将作为<entity-mappings>的<entity>元素添加。

      • 类文件本身（.java）：将添加@Entity注释。

      IntelliJ IDEA还将信息添加到persistence.xml ，作为<class>或<mapping-file>元素。

      对于Hibernate，不显示“ 选择元数据目标”对话框。@Entity注释被添加到类中， <mapping class="">元素被添加到hibernate.cfg.xml 。

    2. 在打开的对话框中，单击必要的“ 数据源”单元格，然后从列表中选择数据源。（要删除与数据源的关联，请选择<无> 。）

    2. 在打开的对话框中，单击必要的“ 命名策略”单元格，然后从列表中选择实现类。
