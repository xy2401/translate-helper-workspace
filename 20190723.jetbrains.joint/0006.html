IntelliJ IDEA与Warbler集成，可以生成应用程序的.war存档。与Warbler的集成需要JRuby。

生成Web存档

  1. 在主菜单上，选择“ 工具”|建立。WAR文件 。

  2. 在“ 输出文件名 ”字段中指定目标存档的名称 。

  3. 在Build中。在WAR文件对话框中，选择要包含在目标存档中的目录和文件旁边的复选框。

  4. 根据需要启用或禁用以下行为选项： 可执行 - 如果选中此复选框，目标存档将是可执行的。
    预编译 - 如果选中此复选框，Ruby文件将编译为.class文件。
    包含gem存储库 - 如果选中此复选框，则gem存储库将包含在存档中。
    打开调用/执行跟踪，启用完全回溯 - 如果选中此复选框，将启用跟踪模式。

  5. 选择要包含在Web存档的WEB-INF目录中的一个或多个文件，然后按WEB-INF配置按钮。

  6. 单击“ 确定”以生成存档。

路径变量

为了避免在将项目从一台计算机移动到另一台计算机时出现复杂情况，PyCharm提供了路径变量（路径变量是一个变量，其值是目录或文件的绝对路径）。

使用存储在项目目录外部的第三方库时，路径变量特别有用。

简而言之，在以下条件下使用路径变量是合理的：

  • 项目级别定义了项目中的第三方库。

  • 这些库存储在项目内容根目录之外。

  • 需要在多台计算机上使用该项目（例如，通过版本控制在开发团队成员之间共享项目）。

用例

为了说明，我们假设在一组开发人员之间共享一个项目（例如，通过版本控制），并且在项目级别定义了该项目中的库。

如果库位于项目内容根目录下，则一切正常。然而，这种情况很少发生。

更典型的情况是库位置在项目外部。这样的库由其绝对路径引用，并且不能保证该路径在团队使用的每台计算机上都是相同的。

显而易见的解决方案是为库位置定义路径变量。在这种情况下，可以在每台计算机上单独设置库路径。

忽略变量

如果在打开项目时，PyCharm检测到未解析的路径变量，它会要求您为它们定义正确的值。如果由于某种原因您不想这样做（例如，您不打算使用相应的库或库），则可以选择将相应的变量添加到忽略的变量列表中。

当忽略变量列表有用时，可能还有其他情况。

在内部级别，路径变量由字符串表示，其中变量的名称包含在一对美元符号字符之间，例如$MY_PATH_VARIABLE$ 。

原则上，这样的模式可能出现在您的项目中，而没有路径变量的含义。

要告诉PyCharm，以美元符号字符开头和结尾的字符串（例如， $SOME_STRING$ ）实际上不是路径变量，您应该将这样的字符串（例如， SOME_STRING ）添加到忽略变量列表中。

路径变量是占位符，代表链接到项目但存储在其外部的资源的绝对路径。如果您在团队中工作，您队友的计算机上的这些绝对路径可能会有所不同。使用路径变量，您可以灵活地共享代码，以便正确解析所有对链接资源的引用，因为路径变量根据每台特定计算机上的配置接受值。

在WebStorm中，有一些预定义的变量：

  • $USER_HOME$ - 代表您的主目录。

  • $PROJECT_DIR$ - 代表项目存储的目录。

创建新的路径变量

例如，您有一个未存储在项目目录中的第三方库 。如果要在从VCS更新项目后确保队友计算机上的路径正确，则可以创建新变量。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 外观和行为”|路径变量 ，然后单击按钮。

  2. 输入新变量的名称（例如， PATH_TO_LIB ）及其指向磁盘上库位置的值。

  3. 通过版本控制系统共享IML文件。

  4. 在队友从VCS更新项目后，他们将更改PATH_TO_LIB变量值，使其指向计算机上的库位置。

忽略的路径变量

打开项目时，WebStorm会检查是否存在任何未解析的路径变量。如果IDE检测到任何，它将要求您为它们定义值。如果由于某种原因您不想这样做（例如，如果您不打算使用具有未解析的路径变量的文件或目录），则可以将它们添加到忽略的变量列表中。

例如，如果在运行/调试配置中传递给JVM的程序参数与内部($SOME_STRING$)路径变量具有相同的格式，则还可以使用忽略变量列表。在这种情况下，您可以将此参数添加到忽略变量列表中，以避免混淆。在“ 路径变量”对话框的“ 忽略的变量”字段中输入SOME_STRING 。

用标签替换标签

使用属性重构替换标记允许您使用等效属性替换一组XML标记上的子标记定义。如果调用此重构，则可以删除与具有所选父标记名称的标记上的所选标记名称匹配的所有标记，并创建等效属性。随着XML模式的发展，将子标记批量转换为属性非常有用。

将标记转换为属性

按预览打开重构预览窗口，显示要替换的标记的所有已找到的用法。在此窗口中，您可以排除/包含要重构的用法。

按Show XSLT ...打开XSLT预览窗口，显示相当于所请求的重构的XSLT的小片段。XSLT处理器可以使用此片段对IntelliJ IDEA项目外部的文件执行请求的重构。

如果要受影响的文件是只读的，则无论如何都可能出现重构预览窗口。

  1. 在编辑器中，将光标放在要转换的标记内。

  2. 选择Refactor | XML重构|从主菜单或上下文菜单中替换Tag with Attribute 。

  3. 确定要替换所选标记的属性的名称。

  4. 确定更换的范围。如果选定的父标签上选择的标签名称与所选标签的名称相同，则所有标签都将被替换。可用范围包括当前文件，整个项目或指定的目录或模块。根据是否选中“ 递归”复选框，目录范围可以包括子目录，也可以不包括子目录。如果选中“ 使用DTD限制文件：...”复选框，则范围将限制为与当前文件具有相同DOCTYPE的文件。

  5. 按“ 预览”按钮使IntelliJ IDEA搜索所选标签“ 查找”窗口的用法。

  6. 单击“ 确定”继续。如果未选择“ 预览”选项，则会立即更改所有用法。

      • immediates ：选择此选项以涉及当前文件的直接子项。

      • 无穷大 ：选择此选项可启用完全递归。

    • 包括被忽略的资源 ：选择此选项可以包含已添加到Subversion忽略列表且不受版本控制到导入的文件。

  5. （可选）输入提交消息，或从“ 最近消息”下拉列表中选择一个消息 ，然后单击“ 确定” 。

与gists共享代码

使用Gists可以共享代码片段，整个文件甚至应用程序。在运行，调试或测试代码时，您还可以使用gists来保存和共享控制台输出。

每个要点都是一个可以被其他人克隆或分叉的存储库。

创造一个要点

  1. 在编辑器中选择代码片段，或在“ 项目”工具窗口中选择文件和文件夹。要将控制台输出保存到要点，请右键单击包含该输出的工具窗口或选项卡中的任意位置。

  2. 在选择的上下文菜单中，选择Create Gist 。

  3. 在Create Gist对话框中，在Filename下指定gist的名称 ，然后输入要发布的代码的描述。

  4. 选择“ 秘密”选项以创建秘密要点。秘密要点无法搜索，也不会出现在Discover中 。它们只能用于您自己的目的，因为您无法共享它们。

    如果要创建公共要点，请确保取消选择此选项。公共要塞是可搜索的，它们出现在Discover中 ，人们可以浏览新出现的要点。如果您希望其他人能够查找和查看您的代码，请使用公共要点。

    创建要点后，您无法将其从秘密转换为公共，反之亦然。

  5. 如果要在默认浏览器中打开新创建的要点，请选择“ 在浏览器中打开”选项，然后单击“ 确定” 。
