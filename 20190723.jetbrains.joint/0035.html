测试RESTful Web服务

此功能仅在Ultimate版本中受支持。

介绍

通过HTTP Client捆绑插件支持测试RESTful Web服务，默认情况下启用该插件。如果没有，请在“ 设置”对话框的“ 插件”页面中将其激活。

当您需要撰写并运行对RESTful Web服务的请求时，有两个主要用例：

  • 当您开发和部署RESTful Web服务并希望确保它按预期工作时：它可以按照规范访问，并且可以正确响应。

  • 在开发解决RESTful Web服务的应用程序时。在这种情况下，在开始开发之前调查对服务的访问和所需的输入数据是有帮助的。在开发过程中，您还可以从应用程序外部调用Web服务。这可能有助于在应用程序导致意外输出时找到错误，同时在代码中未检测到逻辑错误，并且您怀疑瓶颈是与Web服务的交互。

测试RESTful Web服务包括以下检查：

  • 根据服务部署端点和方法注释正确构成URL地址。

  • 生成的服务器请求调用相应的方法。

  • 该方法返回可接受的数据。

IntelliJ IDEA使您能够通过组合并向服务器提交请求 ，查看和分析服务器响应 ，从REST客户端工具窗口运行这些检查。

如有必要，请在“设置”对话框的“ HTTP代理”页面上配置“代理”设置。

编写测试请求并将其提交给Web服务方法

  1. 如果要测试自己的Web服务，请确保已部署并运行。

  2. 选择工具| HTTP客户端|测试RESTful Web服务 。将打开REST客户端专用工具窗口 。

  3. 要让IntelliJ IDEA生成将在基本身份验证中使用的身份验证标头 ，请单击Generate Authorization Header按钮，然后在打开的对话框中，指定用于访问目标RESTful Web服务的用户名和密码。基于这些凭证，IntelliJ IDEA将生成将在基本身份验证中使用的身份验证标头 。欲了解更多信息， 请访问http://www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm 。

  4. 从HTTP方法下拉列表中选择测试请求方法 。可用选项包括：

    • 得到

    • POST

    • 放

    • 补丁

    • 删除

    • 头

    • OPTIONS

  5. 提供数据以计算目标方法的URL地址：

    1. 在“ 主机/端口”文本框中，键入部署Web服务的主机的URL地址。

    2. 从“ 路径”下拉列表中，选择与您需要测试的调用方法对应的@path注释。

      IntelliJ IDEA支持此列表的内容与Web服务Java源代码之间的集成。对@Path注释的任何更改都会反映在“ 资源路径”列表的内容中。

      要将“ 路径”下拉列表的内容与@Path注释同步，请单击“ 从代码更新资源路径”按钮 。

    您可以在主机/端口字段中输入要测试的方法的完整URL地址。无论选择哪种HTTP方法 ，按下时都是如此 IntelliJ IDEA会将URL地址拆分为主机/端口和方法的路径。提取的相对路径将显示在“ 路径”字段中，提取的参数将添加到“ 请求”选项卡的“ 请求参数”窗格中的列表中。

  6. 在“ 标头数据”窗格中，指定请求标头中包含的技术数据。这些数据通过头字段传递，并定义输入参数（ 接受字段）的格式，响应格式（ 内容类型字段），缓存机制（ 缓存控制字段）等。

    要向列表中添加字段，请单击“ 添加” ，然后在“ 名称”文本框中指定字段名称，并在“ 值”下拉列表中指定字段值。

    字段集及其值应符合Web服务API。换句话说，指定的输入格式应该是Web服务所期望的格式，并且预期的响应格式应该是服务返回的格式。

    对于accept ， content-type和其他一些字段，IntelliJ IDEA提供了建议值列表。从“ 值”下拉列表中选择相关的格式类型。

  7. 创建一组要传递给目标方法的参数并指定它们的值。根据所选的请求方法，您可以通过两种方式创建参数列表：
    • 对于GET请求，请指定要作为URL内的查询字符串传递的参数。使用“ 请求参数”窗格。默认情况下，窗格显示带有一行的空列表。

      • 要添加参数，请单击“ 添加” ，然后在“ 名称”文本框中指定参数的名称，并在“ 值”下拉列表中指定参数的值 。

      • 要从列表中删除参数，请选择该参数，然后单击“ 删除” 。

      • 要禁止发送指定的查询字符串参数并禁用“ 请求参数”窗格中的控件， 请按“不发送任何内容”切换按钮 。

    • 要将参数传递给请求消息正文中的目标方法，请使用“ 请求 正文”窗格或将它们插入到本地文件的请求中。选择GET ， DELETE ， HEAD或OPTIONS请求方法时，将禁用“ 请求正文”窗格。

      • 要显式指定参数，请选择“ 文本”选项，然后在文本框中键入参数和值。

      • 要从文本文件中插入参数，请选择“ 文件内容”选项，然后在“ 要发送的文件”字段中指定文件位置。

      • 要在请求中转换和发送二进制文件，请选择“ 文件上载（multipart / form-data）”选项，并在“ 要发送的文件”字段中指定文件位置。

    在任何一种情况下，参数集及其类型都应符合Web服务API，特别是它们应与目标方法的输入参数完全相同。
  8. 要向服务器提交请求 ，请单击“ 提交请求”按钮 。

    请注意，服务器可能缺少证书，或者不受信任。

    如果服务器不受信任，IntelliJ IDEA会显示一个对话框，建议接受服务器或拒绝服务器。如果您将服务器接受为受信任，则IntelliJ IDEA会将其证书写入信任存储区。在下一次连接到服务器时，将不会显示此对话框。

查看和分析Web服务的响应

  1. 要查看对服务器请求的响应 ，请切换到“ 响应”选项卡。收到回复后，该选项卡会自动打开。默认情况下，服务器响应以格式显示，在请求标头中通过content-type字段指定。

  2. 要将响应转换为另一种格式并在编辑器的单独选项卡中打开，请使用View as HTML ， 以XML格式查看 ， 以JSON格式查看 纽扣。

  3. 要查看Web服务响应标头中提供的技术数据，请切换到“ 响应标头”选项卡。

使用cookie

使用专用的Cookie选项卡，您可以创建，保存，编辑和删除Cookie，这些Cookie都是通过响应接收并手动创建的。如果尚未达到到期日期 ，则cookie的名称和值将自动包含在与为cookie指定的域和路径匹配的URL地址的每个请求中。

该选项卡显示您通过响应收到或手动创建的所有当前可用cookie的列表。Cookie按照添加到列表中的顺序显示。单击cookie时，其详细信息将变为可编辑状态并显示在文本框中。有关详细信息，请参阅REST Client工具窗口

  • 保存通过响应收到的cookie不需要特定步骤，从服务器收到的所有cookie都会自动保存。要编辑收到的cookie，请单击列表中包含cookie的行，并更新现在显示在可编辑文本框中的详细信息。

  • 要手动创建cookie，请单击并指定以下内容：

    • 要包含在请求中的cookie的名称和值 。

    • 必须提供的请求的域和路径 ，其中包含cookie的名称 ， 值和到期日期 。

  • 要从列表中删除cookie，请选择包含cookie的行并单击 。

配置代理设置

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为 ”下的“ 系统设置” ，然后选择“ HTTP代理” 。

  2. 在打开的“ 代理”对话框中，选择“ 手动代理配置”并指定以下内容：

    • 在主机名和端口号字段中输入代理主机名和端口号。

    • 要启用授权，请选中“ 代理身份验证”复选框，然后在相应字段中键入用户名和密码。

重用请求

在IntelliJ IDEA会话期间，IntelliJ IDEA会跟踪或请求，您可以运行任何先前执行的请求。您还可以将请求的设置保存在XML文件中，以便在另一个IntelliJ IDEA会话中可用。必要时，您可以检索已保存的设置并再次运行请求。

在IntelliJ IDEA会话中重新运行请求

  1. 单击“ 重播最近请求”按钮 。

  2. 从“ 最近请求”列表中，选择相关请求。字段将填入所选请求的设置。

  3. 单击“ 提交请求”按钮 。

保存请求的设置，以便可以在另一个IntelliJ IDEA会话中检索它们

  • 单击“ 导出请求”按钮 。在打开的对话框中，指定要保存设置的文件的名称及其父文件夹。

在先前的IntelliJ IDEA会话期间运行保存的请求

  1. 单击“ 导入请求”按钮 。

  2. 在打开的对话框中，选择相关的XML文件。字段将填入所选请求的设置。

  3. 单击“ 提交请求”按钮 。

PyCharm 2019.1帮助

断点

断点是源代码标记，它允许您在特定点暂停程序执行并检查其行为。

设置后，断点将保留在项目中，直到您明确删除它（ 临时行断点除外）。如果在外部修改了带有断点的文件，例如，通过VCS更新或在外部编辑器中更改，并且行号已更改，则将相应地移动断点。请注意，PyCharm必须在进行此类更改时运行，否则它们将被忽略。

线断点

可以在可执行的代码行上设置换行符。线程执行在具有此断点的行之前暂停，PyCharm在该线程的堆栈上显示堆栈帧。

Django行断点被分配给Django模板的行，并且只能在包含Django标记的行上设置。当Django模板引擎呈现相应的标记时会触发它们（请参阅调试Django模板 ）。

设置换行符

  1. 将插入符号放在要设置断点的可执行代码行上。

  2. 单击该行旁边的左侧装订线，或按Ctrl + F8 。

    要设置临时行断点，请按Ctrl + Shift + Alt + F8 。它会在被击中后立即从您的项目中删除。

删除换行符

  • 单击左侧装订线中的断点图标。

为避免意外删除断点并丢失其参数，您可以选择通过将断点拖动到编辑器或单击鼠标中键来删除断点：在“ 设置/首选项”对话框中（ Ctrl + Alt + S ）转到“ 构建”，“执行” ，部署|调试器并选择拖动到编辑器或单击鼠标中键 。只需单击断点即可切换其状态（启用/禁用）。

异常断点

抛出指定的异常时会触发异常断点。它们全局应用于异常条件，不需要特定的源代码引用。

PyCharm为Python，Django，Jinja2和JavaScript提供异常断点。Django异常断点允许您在发生Django模板错误时暂停程序执行（ TemplateDoesNotExist和VariableDoesNotExist ）。

设置异常断点

  1. 按Ctrl + Shift + F8或选择运行|从主菜单中查看断点 。

  2. 在“ 断点”对话框中，按Alt + Insert或单击 ，并选择Python异常断点 ， Django异常断点 ， Jinja2异常断点或JavaScript异常断点 。

  3. 在“ 输入异常类”对话框中，从库或项目中指定异常类。

删除异常断点

  1. 按Ctrl + Shift + F8或选择运行|从主菜单中查看断点 。

  2. 在“ 断点”对话框中，选择要删除的断点并单击或按N / A.

断点属性

根据断点类型，您可以配置其他属性，例如：

  • 命中断点时要执行的操作

  • 一个挂起策略，用于定义在遇到断点时是否必须挂起应用程序

  • 对其他断点的依赖

  • 定义何时必须命中断点的条件

要编辑断点属性，请按Ctrl + Shift + F8 ，或右键单击编辑器装订线中的断点。

运行|编辑配置| | Firefox远程

在此对话框中，创建用于在Firefox，版本36及更高版本中本地调试应用程序的配置。请注意以下限制：

  • 目前不支持Firefox中的远程调试。

  • WebStorm在FireFox中的调试会话期间不支持源映射 ，这会阻止压缩或机器生成的JavaScript的调试。

在此文本框中，指定运行应用程序的主机。目前它只是localhost 。

港口

在此旋转框中，指定调试器将侦听的端口。它必须是您在Firefox中启用调试时指定的端口，请参阅打开工具窗口（）并在命令提示符下键入。稍后您将在运行配置中指定此端口号。您可以设置任何端口号，但建议您使用6000或更高版本。 。默认值为6000。

常见选项

编辑运行配置（但不是运行配置模板）时，可以为其指定以下选项：

项目

描述

名称

在此文本框中，指定运行/调试配置的名称。当您选择稍后编辑或调用时，该名称将帮助您识别创建的配置，例如，从“ 运行”弹出窗口（ Shift + Alt + F10 ）。

分享

选中此复选框可使运行/调试配置可供其他团队成员使用。

共享运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

    本指南适用于使用文本编辑器（如Vim，Emacs或Sublime Text）的Python开发人员，需要有关切换到PyCharm的信息。它包括对IDE概念的简要讨论，然后探讨具体的差异。

    什么是IDE？

    Python开发人员长期以来使用各种工具来编写代码。尽管线条模糊，但这些工具分为两大类：文本编辑器和集成开发环境（IDE）。文本编辑器过于简单化，专注于编辑单个文件，提供非常精简和平均的体验。

    顾名思义，IDE具有更广泛的视野。他们希望查看整个项目以及所有与编码相关的活动，并将这些活动统一到一致，强大的UI中。虽然这两个部分重叠，但整体范围是主要差异。

    因为像PyCharm这样的IDE会查看所有代码 - 在项目中，在依赖项中以及在平台本身中 - 它可以在编码活动中提供很多帮助。这种分析和帮助显示在许多强大的设施中：

    • 通过完成输入， 自动完成可加快准确编码速度

    • 快速修复发现常见错误并提供特定于语言的更正

    • 代码意图建议对常见的Python模式进行优化和改进

    • 代码重构将经常使用的Python重构中的苦差事带走了

    • 模板自动执行重复任务

    • 代码导航分析所有代码的结构和语义，以及它使用的代码，以提供在软件中移动的丰富方法

    除了这些编码辅助工具之外，PyCharm还将相关操作与相同的环境联系起来：版本控制，数据库管理，性能分析，测试运行，远程环境等。这些设施中的每一个都以集成方式提供：应用更改，同时查看差异可以自动触发重新运行测试。

    最后，这种力量伴随着以产品为中心的思维方式：一切正常。虽然其他工具有很多可能的功能，但您可以自行组装和维护聚合。这对那些喜欢修补的人来说很好。使用PyCharm作为您的IDE，这些功能可以开箱即用，并且年复一年地得到专业支持。

    标签和拆分窗口

    成熟的文本编辑器（如Emacs和vim）以其丰富的方式而闻名，他们可以将屏幕分成多个区域，允许一次显示多个文件。这些工具不仅仅是基础知识：您可以以丰富的方式在这些区域之间移动，垂直或水平分割，重新分割，并通过键盘完成所有这些工作。

    PyCharm还具有一组成熟的功能来显示多个文件。最基本的是，您可以将多个文件打开到选项卡中，可以手动重新组织或按字母顺序排序。标签可以固定或分离到单独的窗口中。

    除了选项卡之外，您还可以将编辑器窗口拆分为独立窗格，以便一次显示多个文件/选项卡。分割可以垂直或水平完成。您可以通过将选项卡移动到相反的组来重新排列当前打开的文件。

    此外，PyCharm在专用工具窗口中运行您的代码，该窗口提供了许多用于处理运行代码的功能：停止，重新运行等。输出显示在迷你控制台中，错误显示带有可以单击以跳转到该文件的行号的行的回溯。此工具窗口以及运行和重新运行代码都可以通过键盘驱动。

      除了克隆远程存储库之外 ，您还可以基于现有项目的源创建本地存储库。

      将整个项目导入单个Git存储库

      2. 从主菜单中，选择VCS |启用版本控制集成 。

      3. 在打开的对话框中，从下拉列表中选择Git ，然后单击“ 确定” 。

      1. 当您准备通过选择VCS |来推送提交时，调用Push对话框Git |推从主菜单，或按下Ctrl + Shift + K。

      2. 如果到目前为止尚未添加任何遥控器，将显示“ 定义远程”链接而不是远程名称。单击它以添加遥控器。

      3. 在打开的对话框中，指定远程名称及其托管位置，然后单击“ 确定” 。

      添加第二个遥控器

      1. 从主菜单中，选择VCS | Git |遥控器 。Git Remotes对话框将打开。

      2. 单击“ 添加” 按钮在工具栏上或按Alt + Insert 。

      3. 在打开的对话框中，指定远程名称和URL，然后单击“ 确定” 。


      要编辑远程（例如，更改已克隆的原始项目的名称），请在Git Remotes对话框中选择它，然后单击编辑 按工具栏上的按钮，或按Enter键 。

      要删除不再有效的存储库，请在Git Remotes对话框中选择它，然后单击Remove 按钮在工具栏上，或按Alt + Delete 。

      您还可以通过单击其名称来编辑推送对话框中的远程。

      我什么时候需要用力推？

      当您运行push ，如果远程存储库包含您缺少的更改并且您要使用存储库的本地副本覆盖，Git将拒绝完成操作。通常，在使用更改进行更新之前，需要执行pull以与远程同步。

      --force push命令禁用此检查，并允许您覆盖远程存储库，从而删除其历史记录并导致数据丢失。

      当您可能仍需要执行--force push时，可能会出现这样的情况：当您重新--force push推送的分支然后想要将其推送到远程服务器时。在这种情况下，当您尝试推送时，Git将拒绝您的更改，因为远程引用不是本地引用的祖先。如果你在这种情况下执行pull ，你将得到两个你需要合并的分支副本。

      如果您决定强制推送重新分支的分支并且您正在团队中工作，请确保：

      • 没有人拉你的分支并对它进行了一些局部更改

      • 所有挂起的更改都已提交并推送

      • 您拥有该分支的最新更改

      实时模板变量

      扩展实时模板缩写时，变量将替换为预定义值（默认值或使用函数计算），并显示为用户可以修改的输入字段。

      模板中的变量以以下格式声明：

      $ <变量> $

      要向模板文本添加美元字符（ $ ）以使其不被视为变量声明，请使用特殊变量$$ 。

      每个变量都由表达式定义，并且可以在表达式无法计算时接受某些默认值。

      此表达式可能包含以下基本类型的构造：

      • 双引号中的字符串常量。

      • 实时模板中定义的另一个变量的名称。

      • 具有可能参数的预定义函数。

      可以在“ 编辑模板变量”对话框中配置模板变量 。

      配置模板变量

      1. 按Ctrl + Alt + S打开“ 设置/首选项”对话框，然后转到“ 编辑器”|实时模板 。

      2. 选择要配置变量的模板。

      3. 在模板文本中指定变量，然后单击编辑变量

      4. 在“ 编辑模板变量”对话框中，您可以对每个变量执行以下操作：

        • 更改变量名称。

        • 使用预定义函数定义表达式。

        • 指定表达式无法评估的情况的默认值。

        • 如果使用其表达式定义变量，请指定在提示用户输入时是否要跳过变量。

      预定义的模板变量

      IntelliJ IDEA支持以下预定义的无法修改的实时模板变量：

      • $END$表示代码片段完成$END$标的位置，您无法再按Tab键跳转到下一个变量。

      • $SELECTION$用于环绕模板，表示要包装的代码片段。展开模板后，所选文本将按照模板中的指定进行包装。例如，如果在代码中选择EXAMPLE并通过指定的缩写调用"$SELECTION$"模板，或者按Ctrl + Alt + T并从列表中选择所需的模板，IntelliJ IDEA将用双引号将选择包装为如下： "EXAMPLE" 。

      使用WebStorm，您可以使用编译成JavaScript的CoffeeScript 。WebStorm识别*.coffee文件并用它们标记 。您可以直接在CoffeeScript代码中设置断点，WebStorm使用编译期间生成的源映射来识别它们。

      WebStorm通过建议完成关键字，标签，变量，参数和函数来帮助您编写CoffeeScript代码。您可以从符号跳转到其声明（ Ctrl + B ）或实现（ Ctrl + Alt + B ），也可以按名称查找任何符号（ Ctrl + Shift + Alt + N ）。

      • 下载并安装Node.js.

      将CoffeeScript编译为JavaScript

      您也可以从命令行使用编译器或将其配置为第三方工具，有关详细信息，请参阅配置第三方工具 。

      WebStorm提供了一个通用的过程和用户界面，用于创建所有类型的文件监视器 。唯一的区别在于您在每种情况下选择的预定义模板。

      编译代码

      当您打开文件时，WebStorm会检查当前项目中是否有适用的文件观察程序。如果配置了此文件观察程序但已禁用，则WebStorm会显示一个弹出窗口，通知您已配置的文件观察程序并建议启用它。

      如果在当前项目中配置并启用了适用的文件观察程序，则WebStorm会根据“ 新建观察器”对话框中指定的事件自动启动它。

      • 如果选中“ 自动保存已编辑的文件以触发观察者”复选框，则只要对源代码进行任何更改，就会调用“文件观察器”。

      2. 输入提交消息，并在将所选文件提交到本地存储库之前选择希望PyCharm执行的Before Commit操作。

        你可以点击从最近的提交消息列表中进行选择。

      3. 如有必要，请在Git部分中选择以下选项：

        • 作者 ：如果您要提交其他人所做的更改，您可以指定这些更改的作者。

        • 修改提交 ：如果要将本地更改添加到最新提交，请选择此选项（有关详细信息，请参阅使用上一次提交（修改提交）组合暂存更改 ）。

        • 签名提交 ：如果您要签署提交以证明您将要签入的更改是由您做出的，或者您对您提交的代码负责，请选择此选项。

          启用此选项后，将在提交消息的末尾自动添加以下行： Signed off：<username>

      4. （可选）在“ 提交后”区域的“上载文件”列表中，选择服务器访问配置或用于将提交的文件上载到本地或远程主机，装入的磁盘或目录的服务器组 。有关详细信息，请参阅部署应用程序

      5. 单击“ 提交”按钮或将鼠标悬停在此按钮上以显示以下可用提交选项之一：

        • 提交和推送 ：在提交后立即将更改推送到远程存储库。

      我什么时候需要用力推？

      当您运行push ，如果远程存储库包含您缺少的更改并且您要使用存储库的本地副本覆盖，Git将拒绝完成操作。通常，在使用更改进行更新之前，需要执行pull以与远程同步。

      --force push命令禁用此检查，并允许您覆盖远程存储库，从而删除其历史记录并导致数据丢失。

      当您可能仍需要执行--force push时，可能会出现这样的情况：当您重新--force push推送的分支然后想要将其推送到远程服务器时。在这种情况下，当您尝试推送时，Git将拒绝您的更改，因为远程引用不是本地引用的祖先。如果你在这种情况下执行pull ，你将得到两个你需要合并的分支副本。

      如果您决定强制推送重新分支的分支并且您正在团队中工作，请确保：

      • 没有人拉你的分支并对它进行了一些局部更改

      • 所有挂起的更改都已提交并推送

      • 您拥有该分支的最新更改

      探索并完成运行配置
