错误突出显示

此功能仅在Ultimate版本中受支持。

XSLT支持可以检测XSLT结构中的一系列错误，例如拼写错误的模板名称，缺少模板参数，错误匹配模式，对未声明变量的引用，错误或无用的嵌入式XPath表达式等，还提供快速修复以自动修复其中一些错误。

语法突出显示

在允许的情况下，根据当前活动的颜色方案突出显示XPath函数调用，轴名称，数字，字符串等。默认情况下，插件使用为相应Java类型定义的颜色，例如数字和字符串文字等。如果需要不同的颜色，可以在“设置/首选项”对话框的“ 颜色方案”页面上的选项卡中配置这些颜色。

XPath语法检查

就像交互式XPath表达式评估一样，XSLT支持捕获样式表中使用的XPath表达式中的任何语法错误。

XPath类型检查

在XPath中，几乎所有类型都可以通过某些明确定义的转换语义相互分配。但是，没有为任何类型定义到NODESET的转换，也没有（便携式）转换功能可用。此类型转换会突出显示为错误。

模式验证

XPath表达式的一种特殊形式是XSLT中的模式。它们例如用作xsl:template元素中match -attribute的值。此处仅允许XPath表达式的某个子集，XSLT支持检查该子集。

未解决的参考文献

检测到尚未声明或无法从当前作用域访问的变量的引用，并将其突出显示为错误。有一些快速修复可用于为此类未解析的变量引用创建变量或参数声明。

快速修复：

重复声明

在XSLT中，在同一作用域级别上声明的变量或参数不得超过一个。也不允许有多个具有相同名称的模板。该插件将识别此类重复声明并在编辑器中突出显示它们。

其他检查

阴影变量

尽管可能在不同的嵌套级别中具有相同名称的变量或参数，但这可能会造成混淆并且可能导致编程错误。该插件可以识别阴影声明，并提供Quick-Fixes来重命名本地或外部变量。

缺少模板参数

XSLT支持执行的另一项检查是，是否使用xsl:call-template指定了所有必需参数。如果没有默认值，则认为需要参数，即如果没有选择属性且参数的声明元素具有空体。

快速修复：

多余的模板参数

还有一个补充检查，用于标记未声明为模板参数的参数。有一些快速修复可用于从模板调用中删除参数或向被调用模板添加相应的参数。

快速修复：

函数调用参数

与交互式XPath表达式评估一样，XSLT支持检查函数参数的数量和类型是否与其对XPath和XSLT的内置函数的声明相匹配。

XPath检查

编辑XSLT文档时支持所有XPath检查 。这些检查也可以通过使用noinspection XML注释以类似于IntelliJ IDEA用于Java代码的标准抑制机制的方式进行抑制。可以在指令级别，模板级别（如果适用）或样式表级别上的不同级别进行抑制。

如果您不希望在每次与远程同步时指定yor凭据或推送提交，则可以配置WebStorm以保存帐户信息（请参阅配置密码策略 ）。

注册现有帐户

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 单击添加帐户 。

  3. 在打开的对话框中，指定您的GitHub服务器URL（github.com或企业实例）。

  4. 执行以下操作之一：

    • 如果您已有令牌，请单击“ 输入令牌”链接并将其粘贴到那里。

    • 如果要获取新令牌，请输入您的登录名和密码。如果启用了双因素身份验证 ，系统将要求您输入将通过SMS或移动应用程序发送给您的代码。

    令牌必须在您的帐户权限中启用repo和gist作用域（请参阅了解作用域 ）。

创建一个新的GitHub帐户

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 单击“ 添加帐户” ，然后在打开的对话框中单击“ 注册Github” 。

  3. 在打开的注册GitHub页面上注册您的帐户。

  4. 返回WebStorm设置并指定凭据。

管理多个帐户

您可以在WebStorm中使用多个GitHub帐户：例如，用于处理开源项目的个人帐户以及用于主要工作的公司帐户。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|左侧窗格中的GitHub 。

  2. 使用按钮可根据需要添加任意数量的帐户。

  3. （可选）要将帐户设置为当前项目的默认帐户，请选择该帐户并单击 。如果设置了默认帐户，WebStorm将不会要求您在GitHub上共享项目时选择要使用的帐户，重新分叉 ， 创建拉取请求或创建要点 。

  5. 再次检查灯泡菜单（ Alt + Enter ）。编辑XML片段命令已变为可用，这意味着已为MYDATA类型的值注入了XML。

范围语言语法参考

范围语言用于指定各种分析中涉及的项目范围 。

类集

  • 单个类由类名定义，即com.intellij.openapi.我的课

  • 包中的所有类的集合，而不是递归到子包中，由点后面的星号定义，例如： com.intellij.openapi.*

  • 包中包含内容的所有类的集合由双点后的星号定义，例如com.intellij.openapi..*

文件集

  • 单个文件由文件名定义，即MyDir/MyFile.txt

  • 目录中所有文件的集合，不会递归到子目录中，在斜杠后用星号定义，例如： file:src/main/myDir/*

  • 包含子目录内容的目录中的所有文件集由双斜杠后的星号定义，例如file:src/main/myDir//*

修饰符

位置修饰符

位置修饰符可帮助您指定所需文件的位置：

  • src: - 用于源文件

  • lib: - 用于库类

  • test: - 用于测试代码

例如， src:com.intellij.openapi.* patten将com.intellij.openapi包中源根目录下的所有类放在范围内，不包括子包。默认位置是模块根目录。

模块修饰符

模块修饰符通过指定相关模块的名称来帮助您缩小范围：

  • src [模块名称]：<E>
  • lib [模块名称]：<E>
  • test [模块名称]：<E>

例如， src[MyModule]:com.intellij.openapi.*模式在范围内放置与com.intellij.openapi包中的MyModule模块相关的源文件夹下的所有类，不包括子包。

逻辑运算符

范围语言允许您使用常见的逻辑运算符：

&& for AND || 对于 OR ！ 不

除此之外，括号可用于将逻辑运算符连接到组中。例如，以下范围

（<A> ||的<b>）&& <C>

暗示<a>和<c>，或<b>和<c>。

另一个例子

文件[*网络*]：的src / main / java的// *
表示名称包含web的所有模块的范围，以及目录src/main/java递归的所有文件。

从现有范围创建新范围

您可以从几个现有范围编译新范围。在这种情况下，您可以使用$ （ $MyScope ）引用现有范围。

例如， $Scope1||$Scope2模式在$Scope1||$Scope2放置了Scope1和Scope2中的所有文件。

例子

  • file[MyMod]:src/main/java/com/example/my_package//* - 在项目中包含模块“MyMod”中的所有文件，位于指定目录和所有子目录中。

  • src[MyMod]:com.example.my_package..* - 递归地包含模块源目录中包中的所有类。

  • lib:com.company..*||com.company..* - 以递归方式包含项目和库中包中的所有类。

  • test:com.company.* - 包括包中的所有测试类，但不包括在子包中。

  • [MyMod]:com.company.util.* - 包含指定模块包中的所有类和测试类。

  • file:*.js||file:*.coffee - 包含所有JavaScript和CoffeeScript文件。

  • file:*js&&!file:*.min.* - 包括所有JavaScript文件，除了通过缩小生成的文件，由min扩展名表示。

混帐

为了能够使用的Git版本控制，确保Git的整合插件在插件下的设置/首选项（ 按Ctrl + Alt + S）启用。

在为现有本地项目启用Git版本控制或从远程存储库克隆Git项目之前 ，请执行以下操作：

  1. 下载并安装Git。支持最低版本的Windows为2.4，Linux和MacOS为1.8.0.0。

  2. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 版本控制”|在左窗格中Git并指定Git可执行文件的路径。

  3. 设置远程Git存储库的密码

设置Git遥控器的密码

每次与远程Git存储库交互时（例如，在拉取 ， 更新或推送操作期间），都需要授权。您可以将PyCharm配置为记住您的密码 ，这样您就不必在每次需要授权时都指定凭据。

配置密码策略

  1. 在“ 设置”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为”系统设置|密码在左边。

  2. 选择您希望PyCharm如何处理Git远程存储库的密码：

    • 在本机Keychain中 ：选择此选项以使用本机Keychain存储您的密码。此设置仅适用于MacOS和Linux。

    • 在KeePass中 ：选择此选项以使用KeePass密码管理器来存储您的密码。使用KeePass密码管理器时 ，将使用主密码来访问存储个人密码的文件。一旦PyCharm记住了你的密码，除非你需要访问密码数据库，否则它不会要求它们。输入将用于访问MasterPassword字段中的c.kdbx文件的密码。

      您可以在“ 数据库”字段中更改c.kdbx文件的默认位置。

      要导入c.kdbx文件，请单击并从下拉菜单中选择“ 导入 ”，或单击“确定” 并指定包含密码的本地文件的路径。

      如果要从数据库中删除现有密码，请选择“ 清除” 。

    • 重启后不要保存，忘记密码 ：如果要在关闭PyCharm后重置密码，请选择此选项。

启用版本控制

PyCharm支持两个级别的版本控制集成：

  • 在IDE级别，通过默认启用的一组捆绑插件提供VCS集成。

  • 在项目级别，通过将项目文件夹与一个或多个版本控制系统相关联来启用VCS集成。

将项目根与版本控制系统相关联

PyCharm允许您快速启用项目与版本控制系统的集成，并将其与项目根关联。有关如何将单独的项目目录与不同版本控制系统关联的说明，请参阅将目录与版本控制系统关联 。

  1. 选择VCS |在主菜单上启用版本控制集成 ，或按Alt +` ，然后选择启用版本控制集成....

  2. 在打开的“启用版本控制集成”对话框中，从下拉列表中选择要与项目根关联的版本控制系统。

将目录与版本控制系统相关联

PyCharm支持基于目录的版本控制模型，这意味着每个项目目录可以与不同的版本控制系统相关联。

  1. 打开版本控制设置 （ 文件|设置|版本控制 ）。此页面显示与其关联的项目目录和版本控制系统的列表（如果未添加任何目录，则列表仅包含项目根目录）。

  2. 单击“ 添加”按钮在右边。

  3. 在打开的“ 添加VCS目录映射”对话框中，选择“ 目录”选项。键入要与版本控制系统关联的目录的路径，或单击“ 浏览”按钮并在打开的对话框中选择目录。

  4. 从VCS下拉列表中，选择将用于控制此目录中文件的版本控制系统。请注意，此列表仅包含当前启用相应插件的版本控制系统。

  5. （可选）单击“ 配置VCS”按钮 ，该按钮允许您指定所选版本控制系统的设置。版本控制设置节点下也提供相同的设置。

  6. 单击“ 确定”保存映射并返回“ 版本控制”页面。

管理未注册的目录

对于启用了Git或Mercurial集成的项目，PyCharm扫描项目目录以检查是否存在不受IDE控制的Git / Mercurial存储库。如果检测到此类存储库，PyCharm将显示通知。

要添加未注册的根，请单击通知中的“ 添加根”链接。或者，打开“ 版本控制”设置页面 ，选择要添加的未注册根（它们标记为灰色），然后按照将目录与版本控制系统关联的过程进行操作 。

如果您不希望再次收到有关这些根的通知，请单击通知中的“ 忽略”链接。请注意，如果将新的未注册存储库添加到项目中，PyCharm将通知您有关它们的信息。

更改VCS关联

  1. 打开“ 版本控制”设置页面。此设置页面显示具有关联版本控制系统的目录表。

  2. 在表中，找到与要放在另一个版本控制系统下的目录对应的行。

  3. 单击VCS列。从显示的下拉列表中，选择一个新的版本控制系统。

  4. 单击“ 确定”以保存新映射并关闭“ 版本控制”对话框。

（可选）单击“ 配置VCS”按钮。将打开“ 版本控制配置”对话框，您可以在其中配置所选版本控制系统的设置（有关详细信息，请参阅“ 版本控制设置”页面参考中的相应配置设置）。

配置与Web服务器的同步

在配置开发和生产环境的众多方法中，最常见的方法如下：

  • Web服务器安装在您的计算机上。源位于服务器文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • Web服务器安装在您的计算机上，但源存储在另一个文件夹中。您进行开发，然后将源复制到服务器。

  • Web服务器位于另一台计算机（远程主机）上。服务器上的文件可通过FTP / SFTP / FTPS协议，网络共享或已安装的驱动器获得。

PyCharm区分了就地服务器， 本地服务器和远程服务器，但是这些术语在PyCharm环境中的含义与它们的常见含义略有不同：

  • 就地服务器是一个服务器，其文档根目录是项目根目录的父服务器，无论是否立即生效。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），您可以直接在服务器上进行开发。

  • 本地服务器是在本地或装入的文件夹中运行的服务器，其文档根目录 不是项目根目录的父级。您进行开发，然后将源复制到服务器。

  • 远程服务器是在另一台计算机（远程主机）上运行的服务器。可以通过FTP / SFTP / FTPS协议访问服务器上的文件。

通过Remote Hosts Access捆绑插件提供与服务器同步，上载，下载和管理文件，默认情况下启用该插件。如果禁用该插件 ，请在“设置”对话框的“ 插件”页面中将其激活。有关详细信息，请参阅管理插件 。

服务器访问配置

PyCharm通过服务器访问配置控制与Web服务器的交互。无论您何时使用服务器，都需要定义服务器访问配置，无论您的服务器是在远程主机上还是在您的计算机上。

服务器访问配置定义以下内容：

  • 服务器类型（ 就地 ， 本地或远程 ）。

  • 运行服务器的计算机（主机）。对于就地和本地服务器，PyCharm的先决条件，这是你的项目是当前计算机。

  • 服务器访问配置根 ：服务器层次结构中可通过服务器配置访问的最高文件夹。

  • 用于访问服务器配置根目录的URL地址。

  • 传输数据的协议和连接参数。

  • 本地文件夹（即项目根目录下的文件夹），服务器上的目标文件夹（本地或远程）和用于访问服务器上数据的URL地址之间的对应关系。这种对应关系称为映射 。

    例如，您的项目存储在C：/ Projects / My_Project /下，并包含文件夹C：/ Projects / My_Project / My_Folder ，而本地服务器的文档根目录在C：/ xampp / htdocs下 。您将整个项目树上传到C：/ xampp / htdocs / My_Project 。在PyCharm术语中， C：/ Projects / My_Project / My_Folder文件夹称为本地 ， C：/ xampp / htdocs / My_Project / My_Folder文件夹称为远程 。

您可以根据需要定义任意数量的配置，从而实现上载/下载设置之间的灵活切换。

如果需要将代码部署到多个服务器，则可以创建服务器组并避免单独部署到每个服务器。

您可以将服务器配置设置为默认值，以便PyCharm在以下情况下以静默方式选择它：

  • 自动上传已更改的文件 。

  • 无需选择目标主机即可手动上载和下载文件。

  • 将本地文件和文件夹与其远程版本进行比较。

  • 编辑远程主机上的单个文件 。

如果在多次提交中修改了相同的文件，则只有在选择这些提交或整个存储库时才会列出一次，如果为此文件调用差异查看器 ，则所有更改将一起压缩。

使用强制推动

当您运行push ，如果远程存储库包含您缺少的更改并且您要使用存储库的本地副本覆盖，Mercurial将拒绝完成操作。通常，在使用更改进行更新之前，需要执行pull以与远程同步。

--force push命令禁用此检查，并允许您覆盖远程存储库，从而删除其历史记录并导致数据丢失。

当您可能仍需要执行--force push时，可能会出现这样的情况：当您重新--force push推送的分支然后想要将其推送到远程服务器时。在这种情况下，当您尝试推送时，Mercurial将拒绝您的更改，因为远程引用不是本地引用的祖先。如果你在这种情况下执行pull ，你将得到两个你需要合并的分支副本。

除非绝对必要（例如，如果您不小心推送了一些敏感数据），应该避免重新推出推送分支并修改其历史记录。

使用--force会导致所有新分支被推到所有分支上，这可能会给您的团队造成混乱。

如果您决定强制推送重新分支的分支并且您正在团队中工作，请确保：

  • 没有人拉你的分支并对它进行了一些局部更改

  • 所有挂起的更改都已提交并推送

  • 您拥有该分支的最新更改

    4. 在打开的对话框中，确保选中要包含在修补程序中的所有更改，输入提交注释（可选）并单击“ 创建修补程序” 。

    1. 在任何视图中选择所需文件（在“ 项目”工具窗口中，在编辑器中，在“ 版本控制”工具窗口的“ 本地更改”选项卡中，选择一个子项）。

    IntelliJ IDEA 2019.1帮助

    模块

    在IntelliJ IDEA中，模块是任何项目的重要组成部分 - 它与项目一起自动创建。项目可以包含多个模块 - 您可以添加新模块 ，对它们进行分组 ，以及卸载目前不需要的模块 。

    模块由内容根和模块文件组成。内容根是存储代码的文件夹。通常，它包含源代码，单元测试，资源文件等的子文件夹。模块文件（ .iml文件）用于保持模块配置，包括内容或源根，依赖关系，构面中的框架特定设置等。

    具有多个模块的项目

    IntelliJ IDEA允许您在一个项目中拥有许多模块，它们不应该只是Java。您可以为Java应用程序创建一个模块，为Ruby on Rails应用程序或任何其他受支持的技术创建另一个模块。

    由客户端和服务器端组成的应用程序是双模块项目的一个很好的例子。

    客户端（ client-common模块）可能包括desktop-gui ， mobile-gui和command-line模块。

    服务器端（ web-service模块）可能包含其他几个模块，例如： web service ， servlets和rest-api 。

    将新模块添加到项目中

    1. 右键单击“项目”工具窗口中的顶级目录，然后选择“ 新建”|模块 。“ 新建模块”向导将打开。

    2. 从左侧列表中选择模块类型。

    3. 在对话框的右侧，从Module SDK列表中选择要使用的SDK 。您可以使用项目SDK或指定一个新项目。

    4. 在“ 附加库和框架”部分中，选择要在此模块中使用的其他资产。

    5. 在下一步中，为模块命名并指定内容根目录和.iml文件。您可以将它们放在项目内部或外部。

    6. 单击完成 。

    将现有模块导入项目

    您可以导入模块并将其添加到项目中，也可以将项目用作多个单独模块的共享视图。如果您有多个项目并且您的工作流程要求您在它们之间切换，这将非常有用。

    要将现有模块添加到项目中，请导入.iml文件：

    1. 从主菜单中，选择File |新的|来自现有来源的模块 。

    2. 在打开的对话框中，指定要导入的模块的.iml文件的路径，然后单击“ 打开” 。

      在此阶段，模块不是项目的一部分。

    3. 在“项目”工具窗口中，将导入的模块拖到顶级目录以将其添加到项目中。

    组模块

    在IntelliJ IDEA中，您可以对模块进行逻辑分组。如果您有一个包含多个模块的大型项目，则分组将使您可以更轻松地浏览项目。要整理模块，请为其提供完全限定的名称。例如，如果要对所有CDI模块进行分组，请将cdi前缀添加到其名称中。

    1. 打开“ 项目结构”对话框（ Ctrl + Shift + Alt + S ），然后单击“ 模块” 。

    2. 选择要分组的模块，打开上下文菜单，然后单击“ 更改模块名称” 。

    3. 指定前缀并应用更改。

      要在“ 项目结构”对话框中查看同一级别上的所有模块，请使用“展平模块”上下文菜单选项。

    将模块组转换为限定名称

    在早期版本中，IntelliJ IDEA使用组来将模块连接在一起。如果已配置手动模块组，则可以继续使用它们，但不能使用限定名称。要启用限定名称：

    1. 从主菜单中，选择File |将模块组转换为合格的名称 。

    2. 在下一个对话框中，查看新模块名称并在必要时进行调整。

    3. 应用更改并关闭对话框。

    什么是神器？

    工件是您组合在一起以测试，部署或分发软件解决方案或其部分的项目资产的程序集。示例包括已编译的Java类或打包在Java归档中的Java应用程序，Web应用程序作为目录结构或Web应用程序归档等。

    工件可以是存档文件或包含以下结构元素的目录结构：

    • 一个或多个模块的编译输出

    • 库包含在模块依赖项中

    • 资源集合（网页，图像，描述符文件等）

    • 其他文物

    • 单个文件，目录和档案。

    使用工件配置

    根据工件配置生成工件。工件配置在“ 项目结构”对话框（“ 文件”|“项目结构”|“工件” ）中进行管理。

    配置工件的关键部分是在“ 输出布局”选项卡上指定工件结构和内容。

    建筑物

    您可以自己开始构建工件： 构建|构建工件 。

    您也可以通过执行运行/调试配置来构建工件：

    在相应的运行/调试配置中，将Build <ArtifactName>工件任务添加到Before启动任务列表中。执行运行/调试配置（ 运行|运行或运行|调试 ）时，将自动构建工件。

    当运行/调试配置以某种方式使用相应的工件时，在运行/调试配置中包括构建工件任务是有意义的，例如，启动打包在JAR工件中的应用程序或将WAR或EAR工件部署到应用程序服务器。

    默认情况下，构建时的工件将放置在out/artifacts/<artifact_dir>文件夹中。

    构建选项（构建，重建等）

    构建工件（ Build | Build Artifacts ）时，您有以下选项：

    • 建立。 第一次使用时，会构建整个工件。每次下次使用此选项时，只会构建受上次构建后所做更改影响的工件部分，并将其添加到输出文件夹中。

    • 重建。 像第一次一样构建整个工件。从技术上讲，这是Clean，然后是Build 。

    • 清洁。 删除工件输出目录的所有内容。

    • 编辑。 编辑工件配置 。

    运行JAR工件

    要运行打包在Java归档（JAR）中的Java应用程序，IntelliJ IDEA提供JAR应用程序运行配置。要创建这样的运行配置：

    1. 打开“ 运行/调试配置”对话框（例如“运行”|“编辑配置” ）。

    2. 点击并选择JAR应用程序 。

    将工件部署到应用程序服务器和云平台

    许多工件格式（例如WAR，Exploded WAR，EAR，Exploded EAR）适合部署到应用程序服务器和云平台。以下是部署此类工件的方法：

    1. 在服务器或云运行/调试配置中，指定要部署的工件。（使用“ 部署”选项卡或字段。）

    2. 执行run / debug配置或使用Deploy命令（ ）在Application Servers ， Run或Debug工具窗口中。
