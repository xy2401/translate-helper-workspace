配置项目

IntelliJ IDEA中的项目允许您方便地管理模块并在它们之间共享设置。这些设置包括SDK，语言级别和编译器输出。

模块是独立的实体。每个模块都可以有自己的设置，可以配置为开发特定于框架的应用程序。模块可以引用其他模块，即相互依赖。

在IntelliJ IDEA中，您可以将项目与构建工具集成，例如Maven或Gradle 。在这种情况下，IDE将为您提供编码帮助，以便编辑构建文件，代码完成，检查和快速修复，重构等。

使缓存无效

IntelliJ IDEA缓存大量文件，因此系统缓存可能会过载。有时，如果您经常使用频繁的短期项目，则永远不会再需要缓存。

当您使缓存无效时，IntelliJ IDEA将重建在当前版本的IDE中运行的所有项目。

  • 在重新启动IntelliJ IDEA之前，不会删除文件。

  • 打开和关闭项目不会导致删除任何文件。

清除系统缓存

  1. 从主菜单中，选择File |使高速缓存/重新启动无效 。

  2. 在“ 无效缓存”对话框中，选择一个操作。您可以使高速缓存无效并重新启动IDE，在不重新启动IDE的情况下使高速缓存无效，或者只是重新启动IDE。

路径变量

路径变量是占位符，代表链接到项目的资源的路径。它们提供共享灵活性，因为您不必引用计算机上的固定位置。

例如，如果您在项目或模块级别定义了第三方库，并且它存储在项目目录或模块内容根目录之外（对于通过VCS共享的项目），则路径变量很有用。这样的库以其绝对路径引用，并且无法保证此路径在您的队友计算机上是相同的。如果使用路径变量，则会在每台计算机上单独配置每个变量时使路径相对。

在IntelliJ IDEA中，有一些预定义的变量：

  • $USER_HOME$ ：代表您的主目录。

  • $PROJECT_DIR$ ：代表存储项目的目录。

  • $MODULE_DIR$ ：代表保存模块配置文件（ IML ）的目录。

要配置路径变量，请在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中选择“ 外观和行为”路径变量 。

示例：创建新的路径变量

例如，您有一个未存储在项目目录中的第三方库。如果要在从VCS更新项目后确保队友计算机上的路径正确，则可以创建新变量。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，选择“ 外观和行为”|路径变量 ，然后单击按钮。

  2. 输入新变量的名称（例如， PATH_TO_LIB ）及其指向磁盘上库位置的值。

  3. 通过版本控制系统共享IML文件。

  4. 在队友从VCS更新项目后，他们将更改PATH_TO_LIB变量值，使其指向计算机上的库位置。

忽略路径变量

当您打开项目时，IntelliJ IDEA会检查是否存在任何未解析的路径变量。如果IDE检测到任何，它将要求您为它们定义值。如果由于某种原因您不想这样做（例如，如果您不打算使用具有未解析的路径变量的文件或目录），则可以将它们添加到忽略的变量列表中。

例如，如果在运行/调试配置中传递给JVM的程序参数与内部($SOME_STRING$)路径变量具有相同的格式，则还可以使用忽略变量列表。在这种情况下，您可以将此参数添加到忽略变量列表中，以避免混淆。在“ 路径变量”对话框的“ 忽略的变量”字段中输入SOME_STRING 。

资源文件

资源包括属性文件 ，图像，DTD和XML文件。这些文件位于应用程序的类路径中，通常使用以下方法从类路径加载：

  • ResourceBundle.getBundle()用于属性文件和资源包

  • loadResourceAsStream()用于图标和其他文件

构建应用程序时，IntelliJ IDEA会将所有资源复制到输出目录中，从而保留相对于源路径的资源的目录结构。默认情况下，以下文件类型被识别为资源：

Play 2.x入门

此功能仅在Ultimate版本中受支持。

在开始生成Play项目之前，请确保在IntelliJ IDEA中下载并启用了Scala插件。

IntelliJ IDEA支持Play版本2.4及更高版本。

让我们看一下在IntelliJ IDEA中生成Play 2项目的以下常见场景：

  • 如果您已有一个Play 2项目，则可以将其导入或从版本控制中签出。在这种情况下，您可以按照常规过程导入或克隆项目。

  • 如果您不熟悉Play 2框架并想学习如何使用它，请通过IntelliJ IDEA创建一个Play项目，该项目将生成可执行应用程序。

  • 如果要查看可以生成Play 2项目的类型，请使用Lightbend Project Starter并使用其中一个提供的模板创建项目。

  • 如果要手动添加对Play 2框架的支持，请创建一个sbt项目。

创建一个新的Play 2项目

  1. 打开新建项目向导。从右侧的选项中，选择Play 2.x.

  2. 按照向导中建议的步骤操作，然后单击“ 完成” 。

IntelliJ IDEA使用适当的结构，依赖项和可执行应用程序生成Play 2项目。

使用Lightbend项目启动器创建Play 2模板

  1. 打开一个新项目向导，然后从右侧的选项中选择Lightbend Project Starter 。

  2. 在向导的下一页上，选择要用于项目的模板。例如，选择Play Scala Seed模板，该模板将创建用于开发Play应用程序的可执行Play模板项目。

    单击完成 。

使用Play 2框架创建一个sbt项目

  1. 创建一个sbt项目 。

    按照向导中建议的步骤操作，然后单击“ 完成” 。
    IntelliJ IDEA创建一个sbt项目。

  2. 在编辑器中打开build.sbt文件，并将Play添加为项目依赖项。

  3. 在“ 项目”工具窗口的项目源根目录中，选择“ 新建”|文件创建plugins.sbt文件。

  4. 在编辑器中打开plugins.sbt文件并添加Play 2插件。

IntelliJ IDEA下载相应的依赖项，并将它们添加到sbt projects工具窗口中的Dependencies节点。

请注意，未生成Play 2框架项目结构，您需要手动添加所有内容。检查Play 2项目结构。

此时，您可以开始创建Play 2模板，因为一旦添加了对Play 2框架的支持，IntelliJ IDEA就会支持它们，并进一步开发您的项目。您可以在Play 2文件中使用代码完成，导航和动态代码分析功能。IntelliJ IDEA还支持路由文件和代码检查的代码帮助。

运行和调试Play 2应用程序

  • 如果要运行该应用程序：

    1. 在项目工具窗口中，右键单击该应用程序。

    2. 在上下文菜单中，选择Run Play 2 App 。

  • 如果需要访问“ 运行/调试配置”对话框：

    1. 在主菜单上，选择“运行”|编辑配置 。

    2. 点击并选择Play 2 App以创建新配置。

    3. 指定运行/调试配置设置。默认设置足以进行运行和调试。

    4. 在主工具栏上，单击要运行的图标或用于调试应用程序的图标。

您可以在默认浏览器http://localhost:9000检查应用程序的输出。如果要停止服务器会话，请按Alt + D （对于macOS，为Cmd + P ）。

使用Play2设置

您可以使用Play 2设置来配置编译器选项并配置路由文件的格式。您还可以查看和编辑默认模板导入列表。

  1. 按Ctrl + Alt + S，然后从左侧的选项中选择Languages Frameworks | Play2 。

  2. 根据您要配置的设置，在Play2设置页面上，选择相应的选项卡并调整设置。使用“ 其他”选项卡可以检查和编辑刷新项目时从描述符中检索的模板导入列表。

    要手动添加模板，删除或编辑现有模板 ，请选择“ 手动选择模板导入”选项，以使“ 导入”部分变为活动状态并相应地调整导入列表。

使用嵌入式本地终端

IntelliJ IDEA有一个嵌入式本地终端，您可以在不离开IDE的情况下使用命令行模式。根据您的平台，您可以使用命令提示符，Far， powershell ， bash等。

配置终端

  • 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，单击“ 工具”下的“ 终端 ”。

在“ 终端”页面上，您可以配置所需的shell，新会话选项卡的默认名称以及其他选项。

嵌入式本地终端从IDE设置继承以下设置：

  • 在Keymap页面上，您可以配置复制（ Ctrl + C ）和粘贴（ Ctrl + V ）快捷方式。

  • 在编辑器的“ 外观”页面上，您可以配置消除锯齿和插入符号闪烁。设置使用块插入符不会在终端中继承：它的插入符始终是块。

  • 在编辑器的配色方案设置下，您可以更改以下选项：

    • 在“ 控制台字体”页面上：行间距和控制台字体。

    • 在“ 控制台颜色”页面上：控制台颜色。

    • 在“ 常规”页面上：选择前景色和背景色。

运行终端

执行以下操作之一：

  • 按Alt + F12 。

  • 选择查看|工具窗口|主菜单中的终端 。

  • 单击终端工具窗口按钮 。

  • 将鼠标指针悬停在上方在IDE的左下角，然后从菜单中选择终端 。

开始一个新的会话

  • 点击在工具栏上或右键单击会话选项卡，然后从上下文菜单中选择“ 创建新会话 ”。在单独的选项卡中打开一个新会话：

关闭项目或IntelliJ IDEA：选项卡名称，当前工作目录，甚至shell历史记录时，会保存会话。

要关闭活动会话，请单击在终端工具栏上或右键单击当前会话选项卡，然后从上下文菜单中选择关闭会话 。

在标签之间切换

  • 按Alt + Right和Alt + Left切换活动选项卡。

或者，您可以按Alt +向下键查看所有终端选项卡的列表。

重命名选项卡

  • 右键单击选项卡，然后从上下文菜单中选择“ 重命名”选项卡 。

  • 使用向上和向下键。

版权

您的项目可能包含您希望通过版权保护的文件。您可以使用版权配置文件，而不是单独在每个文件中插入版权声明。

配置文件定义版权声明文本和一组文件（ 范围 ），其中将包含此文本。

对于同一项目中的文件，您可能需要多个不同的版权声明。在这种情况下，您可以配置多个配置文件并将它们与不同的范围相关联。

版权设置通过VCS共享，这就是您必须使用共享范围的原因 。本地范围不适用于版权配置文件配置。

配置新的版权配置文件

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|版权|版权简介 。

  2. 点击并命名新的个人资料。

  3. 输入版权声明文字。

    您可以输入纯文本或配置Velocity模板。对于模板，请使用变量 ，然后单击“ 验证”以确保正确配置。

将配置文件分配给文件范围

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|版权 。

  2. 点击 （ Alt + Insert ），然后从列表中选择现有的共享范围。

    如有必要，您可以定义新范围。单击选择范围以在页面下部添加范围或修改现有链接。

  3. 从“ 版权”列表中，选择要与范围链接的配置文件。

  4. 应用更改并关闭对话框。

将版权文本插入文件

  • 要将文本插入单个文件，请在编辑器中将其打开，按Alt + Insert ，然后从弹出窗口中选择“ 版权 ”。

  • 要将文本插入到一组文件中，请右键单击“项目”工具窗口中的节点，然后选择“ 更新版权” 。系统将提示您选择要更新通知的范围。

    节点可以包括属于不同范围的文件。在这种情况下，将根据分配的配置文件生成版权声明。

    如果节点包含不属于任何范围的文件，IDE将为其分配默认配置文件 。

更新版权文字

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|版权|版权简介 。

  2. 选择必要的配置文件并编辑版权文本。

  3. 在Regex中检测注释中的版权字段，请指定一个正则表达式，用于查找文件中的现有版权声明。

  4. 在“ 如果旧版权匹配字段允许替换版权”字段中，请指定IDE将用于搜索应更新版权文本的文件的正则表达式。

    例如，如果要更新其版权中包含2000-2006和2010- 20[0-1][0-6]年份的所有文件中的版权文本，请指定20[0-1][0-6] 。

  5. 应用更改并关闭对话框。

  6. 在“项目”工具窗口中，右键单击要为其更新文本的节点，然后选择“ 更新版权” 。

设置默认版权配置文件

默认配置文件的设置将应用于未明确包含在配置文件所涵盖的任何范围内的文件。

  1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|版权 。

  2. 从默认项目版权列表中，选择要用作默认配置文件的配置文件。

创建Rails应用程序元素

IntelliJ IDEA可帮助您使用所有必需元素的存根来填充Rails应用程序：控制器，视图，模型等。您可以通过以下几种方式启动Rails生成器：

  • 按Ctrl键两次，然后开始键入所需的命令或其部分（例如， rails g controller ）。然后，从建议列表中选择此命令，然后按Enter键 。

在本主题中，我们将向您展示如何创建Rails应用程序元素的几个示例。

创建控制器和操作

要生成控制器及其操作（可选），请执行以下操作：

  1. 按Ctrl键两次，然后开始键入rails g controller 。选择rails g controller并按Enter键 。

  2. 在调用的“ 添加新控制器”对话框中，指定控制器名称。（可选）添加由空格分隔的操作名称（例如， new create ）。单击确定 。

    其他选项允许您提供其他参数。例如，您可以添加--no-test-framework参数。

  3. IntelliJ IDEA生成所需文件（控制器，视图和其他）并在“运行”工具窗口中显示输出。

    您可以单击任何文件以在编辑器中快速打开它。

从操作创建视图

您可以在编辑器中为指定的控制器操作创建视图。去做这个：

一般用法

此功能在PyCharm的教育版中不可用。

该插件的用法简单明了，但非常灵活。将提供的注释集添加到项目中并开始使用它们，配置插件以使用一组自定义注释，或者只是使用UI配置使PyCharm了解到例如Pattern.compile()的String参数应该被视为一个正则表达式。

使用注释

IntelliLang使用三个基本注释： @Language ， @Pattern和@Subst 。

  • @Language负责语言注入功能。

  • @Pattern用于根据某种正则表达式模式验证字符串。

  • @Subst用于替换具有固定值的非编译时常量表达式。这允许您验证模式并构建注入语言的前缀/后缀（参见下文），即使对于已知在运行时包含某些值的非常量表达式也是如此。

IntelliLang提供的注释位于文件annotations.jar ，可以在/plugins/IntelliLang/lib 。

@语言

@Language注释可用于注释String字段，局部变量，方法参数和返回字符串的方法。这将导致分配给字段/变量的字符串文字，作为参数传递或用作方法的返回值以解释为指定的语言。

此外，还有语言不匹配检查，它在分配字段/变量或从方法返回的值时检查预期语言与实际语言之间的冲突。

该插件支持直接和间接注释，即您可以直接使用这样的注释：

@ Language （ “JavaScript” ） 字符串 代码 = “var x = 1 + 2” ;

或注释另一个注释类，如下所示：

@Language （ “XPath” ） public @interface XPath { }

然后可以简单地用于将元素注释为@XPath 。

获取必须作为注释的value属性提供的language-id非常容易：IntelliLang通过常规代码完成操作提供可用语言的列表。只需从ctrl-space弹出窗口中选择适当的语言：

@图案

@Pattern注释负责标记必须符合某个正则表达式的字符串，并且可以像@Language注释一样使用。这意味着，可以创建派生注释，例如需要String由一个或多个数字组成的@Number注释：

@Pattern （ “\\ d +” ） public @interface Number { }

实际上，预定义注释已经包含两个这样的派生注释：第一个注释@PrintFormat与java.util.使用的类似printf的模式匹配java.util.Formatter和另一个@Identifier描述了一个有效的Java标识符。这些都可以使用而无需指定任何其他模式。

@Subst

@Subst注释用于替换非编译时常量的引用，这使得插件可以基于替换值与运行时期望的值兼容的假设进行模式验证 。如果值与预期模式不匹配，插件将会抱怨。

它还有助于为语言注入功能构建有效的前缀/后缀上下文（请参阅下一节）。考虑这个例子：

@Subst （ “Tahoma” ） final String font = new JLabel （）。 getFont （）。 getName （）; @Language （ “HTML” ） String message = “<html> <span style ='font：” + font + “; font-size：smaller'>” + ... + “</ span> </ html>” ;

如果没有用值Tahoma替换变量font的值（实际上它可能只是这里的单个字符），注入的片段在语法上会不正确，导致错误一个术语，预期在“ font: ”指令之后显示。

提供上下文：前缀和后缀

在对元素进行注释时，可以提供前缀和后缀，在解析语言片段时应该添加前缀/后缀。这可用于提供上下文信息，即如果JavaScript注入的前缀是“var someContextVariable;”，PyCharm将知道变量someContextVariable被声明，并且在使用它时不会警告它是未声明的。

除了提供前缀和后缀的手动功能外，IntelliLang还可以从上下文中动态确定这些值，并使用字符串文字：

@ Language （ value = JavaScript ， prefix = “function doSomethingElse（a）{}” ） String code = “function doSomething（）{\ n” + “var x = 1; \ n” + “doSomethingElse（x）; \ n “ + ”}“ ;

在此示例中，JavaScript语言将注入三个String文字中的每一个，并且将根据前面和后面的表达式计算每个前缀和后缀，以便正在解析的结果文本是有效的JavaScript语法：

  • 不会显示“missing'}'”错误：右括号是第一个文字后缀的一部分。

  • doSomethingElse(a);使用的变量x doSomethingElse(a);将被声明：它的声明是第二个字面的前缀的一部分。

  • 函数doSomethingElse()也是已知的：它是在静态提供的前缀中定义的。

重要笔记

  • PyCharm语言注入API存在一些问题，这些问题对注入语言片段的前缀/后缀施加了某些限制。例如，不允许语言的标记跨越元素的前缀/后缀。如果前缀以空白字符结尾并且片段以空格开头，则可能发生这种情况。

    该插件以修剪前缀/后缀的方式处理这种特殊情况，并将一个空格字符作为分隔符插入。但是，如果空格字符不是标记分隔符（例如，适用于JavaScript字符串文字），则此方法不起作用。

    这种情况不能自动处理，PyCharm核心会产生断言。

  • 即使动态前缀/后缀计算为语言片段提供了适当的上下文，但有些事情可能无法按预期工作。最值得注意的是这是重构（重命名）和导航功能。

外部注释

在源代码中存储注释并不总是方便的。例如，如果您在团队中处理项目，其中每个人都使用不同的IDE，如果您使用库类，或者您只是想让代码免于注释。这是您可以使用外部注释的时候。

外部注释是常规注释，存储在源代码之外的名为annotations.xml的XML文件中。

在代码中，外部注释标有天沟图标，并以灰色突出显示。默认情况下，IntelliJ IDEA在编辑器中显示外部注释。

启用外部注释

要使用外部注释，您需要在设置中启用此选项。如果跳过此步骤，则可以在外部注释库代码，但是您将无法在源代码中使用外部注释。

  1. 按Ctrl + Alt + S打开“ 设置/首选项”对话框。

  2. 转到编辑|代码风格| Java |代码生成并选择使用外部注释复选框。

  3. 应用更改并关闭对话框。

在外部制作注释

  • 要将常规注释快速转换为外部注释，请将光标放在其上，然后按Alt + Enter （或单击意图操作图标），然后选择外部注释 。

添加新注释

  1. 在要注释的元素上按Alt + Enter （或单击意图操作图标）。

  2. 选择Annotate <element> <element name> （例如， Annotate field 'out' ），然后选择一个注释。

    如果只能使用一个注释，请选择Annotate <element> <element name>作为@ <Annotation>选项（例如，将Annotate class 'String' as @Deprecated ）。

  3. 在下一个对话框中，单击“ 外部添加” 。

  4. 指定要使用外部注释存储文件的目录（外部注释根目录）。

    此模块中的每个下一个外部注释都将存储在同一文件夹中。如果您有多个注释根，IntelliJ IDEA将询问您使用哪一个。

配置注释根目录

注释根目录是一个文件夹，用于存储项目当前模块中使用的外部注释。您可以更改，删除和添加新根：

  1. 按Ctrl + Shift + Alt + S打开“ 项目结构”对话框。

  2. 转到模块|路径 。

  3. 找到“ 外部注释”区域，您可以在其中管理附加到模块的外部注释。

也可以在项目级别配置注释。在“ 项目结构”对话框中，单击“ SDK”|注释 。

隐藏外部注释

默认情况下，外部注释显示在编辑器中。您可以在代码中隐藏它们：

  1. 按Ctrl + Alt + S并选择编辑器|一般|外观 。

  2. 清除“ 显示内联外部注释”复选框。

  3. 应用更改并关闭对话框。

当外部注释被隐藏时，只有天沟中的图标标志着他们在代码中的存在。

您还可以右键单击外部注释，并禁用“ 显示外部注释”选项。

从Maven下载外部注释

  • 您可以将外部注释存储为Maven工件，并将它们与库一起下载。为此，请从Maven Repository对话框的“ 下载库”中选择“ 注释”复选框。

IntelliJ IDEA将使用annotations分类器搜索所有工件，并在存储库中可用时下载它们。

有关从Maven下载库的更多信息，请参阅从Maven 下载库

范围

范围 （ ）是项目中文件，包和/或目录的子集，您可以限制特定操作的应用，例如搜索 ， 代码检查 ，插入版权声明等。此外，您可以为每个范围配置着色 ，以便立即查看您正在处理的文件类型。

随着项目规模的扩大，范围变得更有帮助。有许多预定义范围涵盖基本案例。此外，还可以向项目添加自定义范围 。例如，您可以为测试或团队中负责的文件创建自定义范围。

范围的类型

范围可以是共享的，也可以是本地的：

  • 团队成员可以通过VCS访问共享范围 ，并将其存储在项目级别。如果项目具有基于文件的格式，则共享范围存储在*.ipr文件中;如果项目具有基于目录的格式，则共享范围存储在.idea下的scopes目录中，作为具有xml扩展名的文件，即.idea / scopes / <scope_name> .xml 。

  • 本地作用域仅供个人使用，并存储在工作区中（ *.iws文件采用基于文件的项目格式，或存储在.idea下的workspace.xml文件中，采用基于目录的格式）。

如有必要，您可以共享本地范围，在本地创建共享范围，或创建范围的副本。有关更多信息，请参阅配置范围和文件颜色 。

文件和代码模板

文件模板是创建新文件时要生成的默认内容的规范。根据您创建的文件类型，模板提供了预期在该类型的所有文件中的初始代码和格式（根据行业标准，您的公司政策或其他原因）。

PyCharm为所有支持的文件类型提供预定义模板，这些模板是在创建新文件时建议的。建议的文件类型集取决于模块和配置，以及“ 项目”工具窗口中当前位置的属性。例如，PyCharm不会建议在Python源代码或测试目录之外创建Python文件。

在创建项目时配置JDK

要在IntelliJ IDEA中开发应用程序，您需要Java SDK（JDK）。JDK是一个软件包，包含库，用于开发和测试Java应用程序的工具（开发工具），以及用于在Java平台上运行应用程序的工具（Java Runtime Environment - JRE）。

JRE可以与JDK分开获得，但它不适合应用程序开发，因为它没有必要的组件，如编译器和调试器。

请注意，捆绑的JRE用于运行IDE本身，并不足以开发Java应用程序。在开始使用Java开发之前，必须获取并安装独立的JDK 。

除了JDK之外，一些框架还需要自己的SDK。例如，Android或Grails。

IntelliJ IDEA没有随JDK一起提供，因此如果您没有必要的JDK版本，则必须下载并安装它。

  1. 在New Project向导中，单击Project SDK列表下的Download JDK 。您将被重定向到Oracle官方网站 。

  2. 选择并下载必要的JDK版本。

  3. 运行Java安装向导并按照其步骤操作，直到安装Java。

  4. 单击Project SDK列表旁边的New ，然后选择JDK安装文件夹（主目录）。

    在Windows上，转到C：\ Program Files \ Java ，在macOS上转至/ System / Library / Java 。然后选择Java目录，例如， jdk1.8.0_144 （数字代表您已安装的JDK版本）。

    要确保您指定JDK目录的路径，请检查是否存在bin/javac可执行文件，因为JRE目录没有。

    您可以在处理项目时随时指定其他JDK版本，配置其他SDK以及更改SDK级别。

    配置检查严重性

    检查严重性表示检测到的代码问题对项目的影响程度，并确定如何在编辑器中突出显示检测到的问题。默认情况下，每次检查都具有以下严重性级别之一：

    • 服务器问题

    • 错字

    • 没有突出显示，只有修复

    • 弱警告

    • 警告

    • 错误

    您可以增加或减少每次检查的严重性级别。例如，您可以强制PyCharm显示一些警告，如错误或弱警告。以类似的方式，最初被认为是弱警告的内容可以显示为警告，错误或仅作为参考信息。

    您还可以配置用于突出显示每个严重性级别的颜色和字体样式。此外，您可以创建自定义严重性级别并将其设置为特定检查。

    如有必要，您可以为不同范围内的相同检查设置不同的严重性级别。

    附加和分离Perforce作业到变更列表

    Perforce与PyCharm的集成为您提供了一个用户界面，用于将Perforce 作业附加到更改列表并将其分离。

    请注意，集成不支持创建Perforce作业。

    要访问使用Perforce作业，请在“设置”对话框的“ Perforce”页面上选中“ 启用Perforce作业支持”复选框。

    从这个主题，您将学习如何：

    • 将作业附加到更改列表：

      • 在 “ 本地”选项卡的任何工作阶段 。

      • 在 “ 提交更改”对话框的提交 期间 。

    • 使用以下命令找到要附加到更改列表的所需作业：

      • 标准搜索功能，使您可以指定多个搜索条件，从而灵活配置搜索过程。

      • 快速搜索功能，使您可以自动将找到的作业链接到更改列表。

    • 从变更列表中分离作业。

    当您只需要将一个作业附加到更改列表并且您知道所需作业的确切名称或者至少可以指定名称的搜索模式时， 快速搜索很有用。

    在工作的任何阶段查找和链接工作

    1. 打开“ 版本控制”工具窗口的“ 本地”选项卡。

    2. 选择要将作业链接到的更改列表。

    3. 从更改列表的上下文菜单中，选择“ 编辑关联作业” 。

    4. 找到想要的工作。执行以下操作之一：

      • 要使用标准搜索功能，请单击按钮。

      • 要使用快速搜索功能，请单击按钮。

    5. 查看找到的作业的详细信息，然后单击“ 确定” 。

    在提交期间查找和链接作业

    1. 在“ 版本控制”工具窗口的“ 本地更改”选项卡中，选择要将作业链接到的更改列表，然后打开“ 提交更改”对话框。

    2. 使用“ 作业”区域中的控件查找所需的作业。执行以下操作之一：

      • 要使用标准搜索功能，请单击按钮。

      • 要使用快速搜索功能，请单击按钮。

    3. 查看找到的作业的详细信息并继续提交过程。

    使用标准搜索功能查找作业

    1. 在打开的对话框中或“ 提交更改”对话框的“ 作业”区域中，单击按钮。

      您当前所在的对话框取决于您是在提交期间还是在任何其他工作阶段链接作业。

    2. 在打开的对话框中，指定所需的搜索参数。

      至少应填写其中一个字段。

    3. 单击“ 搜索”按钮。符合指定条件的作业将显示在“ 搜索结果”列表中。要查看作业的详细信息，请在列表中选择它。

    4. 选择所需的作业，然后单击“ 确定”对话框关闭，您将返回到开始搜索的对话框：

      • 如果在提交期间链接作业，则“ 提交更改”对话框。

      • 如果要在任何其他阶段工作中链接作业，则“ 编辑作业链接到更改列表”对话框。

    快速查找和链接一项工作

    1. 打开“ 编辑作业链接到 更改列表”对话框或切换到“ 提交更改”对话框的“ 作业”区域。

      要使用的对话框取决于您是在提交期间还是在任何其他工作阶段链接作业。

    2. 在文本框中，键入所需的作业名称搜索模式，然后单击按钮。找到该作业并自动链接到当前的更改列表。

      如果未找到与指定模式匹配的作业，则显示相应的信息消息。

      通过快速搜索功能找到并链接的作业的详细信息仅在“ 编辑作业链接到更改列表”对话框中可用。

    从更改列表中分离作业

    • 在“ 编辑作业链接到 更改列表”对话框或“ 提交更改”对话框的“ 作业”区域中，选择所需的作业，然后单击按钮。

      要使用的对话框取决于您是在提交期间还是在任何其他工作阶段分离作业。
