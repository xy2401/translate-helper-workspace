处理修改没有Checkout文件

如果要在Perforce版本控制下修改或删除文件，则应删除此类文件的只读状态。WebStorm负责自动使文件可写。但是，您可以手动更改只读状态，这可能以多种方式发生;例如：

  • 启用“ 清除只读状态”选项后，可以使用文件系统使文件可写。

  • 在编辑器中打开只读文件时，双击锁定图标在状态栏中。

  • 使用文件属性从外部删除只读属性。

在这些情况下，文件在没有签出的情况下获得状态修改，并显示在“版本控制”工具窗口的“ 本地更改”选项卡中 。

解决'修改后无需结帐'文件

  1. 在“ 版本控制”工具窗口的“ 本地更改”选项卡中 ，展开“ 不带校验的已修改”节点，然后选择所需的文件。

  2. 在文件的上下文菜单中，选择“ 签出” 。该文件变为可写，并移至活动的更改列表。

对Subversion进行身份验证

为Ant构建目标配置触发器

您可以配置在特定事件之前或之后运行Ant目标的触发器，例如，编译，运行或测试。这些触发器分别称为编译触发器和执行触发器 。

设置编译触发器

  1. 在Ant构建工具窗口中 ，右键单击所需的构建目标。

  2. 在目标的上下文菜单上，选择“ 执行” 。

  3. 在子菜单上，根据需要选择编译前或编译后 。请注意，您可以设置这两个选项。

设置执行触发器

  1. 在Ant Build工具窗口中，右键单击所需的构建目标。

  2. 在目标的上下文菜单上，选择“ 执行” 。

  3. 在子菜单上，选择Before Run / Debug 。出现“执行目标运行/调试”对话框，显示可能的执行类型列表。

  4. 在对话框中，选择应触发所选Ant目标的执行类型旁边的复选框，然后单击“ 确定” 。
