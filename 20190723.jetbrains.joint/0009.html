使用文本编辑器编辑UI布局

现在，当您熟悉使用内置设计器的基本UI布局编辑选项时，让我们通过手动编辑其布局，将更多元素添加到应用程序的主视图中。

1。切换到文本视图

切换到编辑器底部的“ 文本”选项卡。IntelliJ IDEA将显示当前所选布局文件的XML源代码：

2。添加水平标尺

让我们添加一些插入分隔符的标记。添加水平分隔线的最简单方法是在源代码中添加以下内容：

<查看 android：layout_width = “fill_parent” android：layout_height = “5dp” android：layout_marginTop = “60dp” android：background = “＃00ff00” />

分隔符为5个单位厚，涂有绿色背景并放置在最近的元素下方。

3。添加TextView元素

要添加另一个TextView元素，请将以下内容添加到源代码中：

<TextView android：layout_width = “wrap_content” android：layout_height = “wrap_content” android：layout_marginTop = “60dp” android：text = “警告！不要触摸机器人......“ android：id = ”@ + id / message“ android：layout_gravity = ”center_horizontal“ />

新元件将放置在分离器下方60个单元;它水平居中并使用默认字体颜色。与元素关联的文本字符串是显式声明的。

但是，与分隔符不同， TextView元素也具有id属性。其语法指示“/”符号后面的字符串是必须被视为ID资源的字符串，并将用于引用视图元素。Android运行时会适当地处理此信息，使您可以编写与TextView组件交互的Java代码。

共享实时模板

PyCharm存储自动生成的XML配置文件中添加到预定义模板组的自定义实时模板组和模板的定义。

  • 对于自定义组，该文件包含该组包含的所有模板的定义。

  • 对于修改的预定义组，该文件仅包含添加（或修改）的实时模板的定义。

实时模板组配置文件存储在IDE配置目录的templates目录中。通过复制模板目录中的相关文件，您可以在团队成员和多个PyCharm安装之间共享实时模板。此外，您可以基于IntelliJ平台在所有IDE之间共享实时模板。

如果通过IDE Settings Sync插件同步设置 ，则模板将存储在IDE配置目录中的jba_config / templates下。

不要将实时模板配置文件复制到项目目录中。

PyCharm提供了用于共享实时模板的导出和导入功能，这可能比手动复制配置文件更方便。

导出实时模板配置

  1. 选择文件|从菜单中导出设置 。

  2. 在“ 导出设置”对话框中，确保选中“ 实时模板”复选框，并指定存档的路径和名称，以便保存导出的设置。

  3. 单击“ 确定” ，根据实时模板配置文件生成文件。您可以与团队成员共享此文件，也可以在其他PyCharm安装中导入该文件。

导入实时模板配置

  1. 选择文件|从菜单中导入设置 。

  2. 使用导出的实时模板配置指定存档的路径。

  3. 在Import Settings对话框中，选择Live templates复选框，然后单击OK 。

  4. 重新启动PyCharm后，您将在Editor |上看到导入的实时模板PyCharm设置的实时模板页面（ Ctrl + Alt + S ）。

概观

Django为国际化和本地化提供支持：

  • 国际化（i18n）涉及从源代码中提取字符串并将它们呈现为源代码中进一步引用的属性。

  • 本地化可以将这些属性转换为目标语言。

PyCharm提供了有用的功能，可以简化软件国际化和本地化问题。这些功能是：

  • 在Django模板中围绕文本块的意图动作。

  • 在文本块和区域设置之间导航。

先决条件

Django应用程序可以使用国际化和本地化支持。

  1. 从http://ftp.gnu.org/pub/gnu/gettext/下载gettext归档文件下载并安装在您的计算机上。

  2. 解压缩下载的存档。

  3. 将gettext-runtime和gettext-tools的bin目录的路径添加到$PATH变量。

  4. 在项目根目录中创建locale目录。

  5. 将Django设置为项目模板语言 。

每次编译模块时，Osmorc都会为该模块创建一个包JAR 。运行Osmorc运行配置时使用此JAR 。您可以更改JAR的命名方式和放置位置，但在大多数情况下，创建的默认值应该是正常的。
