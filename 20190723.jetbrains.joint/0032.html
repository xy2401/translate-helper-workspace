介绍

IntelliJ IDEA中的每个Flash模块都包含一个或多个构建配置。构建配置定义如何将模块源文件转换为目标输出类型（SWF或SWC），然后打包。

创建模块时会创建一个构建配置。您可以根据需要添加更多构建配置。

构建配置类型

构建配置类型由以下定义：

  • 目标平台（即将使用开发内容的环境）：Web（用于Flash播放器/ Web浏览器目标内容），桌面（用于Adobe AIR目标内容）或Mobile（AIR Mobile，用于内容）适用于Android和iOS移动设备）。

  • 构建配置是使用Flex框架还是纯ActionScript。

  • 输出类型：应用程序（可运行的应用程序，SWF文件），库（SWC文件）或运行时加载的模块（可动态加载的模块 ，SWF文件）。

构建配置的主要选项

定义构建配置类型后，可以指定以下主要选项：

  • 对于Applications和运行时加载的模块： 主类 。

  • 输出文件名。

  • 输出文件夹。

  • 对于Web应用程序：包含HTML包装器模板的文件夹。

  • 对于Web和桌面应用程序： 模块和运行时样式表 。

构建配置依赖项（构建路径）

通常，每个构建配置的构建路径可以通过以下方式定义：

  • Flex SDK。根据构建配置类型自动选择必要的SDK SWC。此外，您可以通过选择以下内容来管理SWC集：

    • 对于Web目标平台：Flash播放器版本。如果SDK包含多个播放器版本，您可以选择应使用哪个相应的SWC。

    • 对于基于Flex框架的构建配置：Flex 4组件集。您可以指定只应使用Spark或MX或两个组件集。

      对于Flex组件，您可以指定其框架链接类型 （Merged Into Code，RLS或External）。

  • 生成库和运行时加载模块的其他构建配置。

  • 第三方库，包括SWC和原始ActionScript。

编译器选项

每个构建配置都与一组编译器选项相关联。

IntelliJ IDEA在IDE，项目和模块级别提供默认的编译器选项集。

较低级别的默认值可以从较高级别继承或重新定义。在单个构建配置级别，类似地，编译器选项可以从模块级别默认值继承。或者，可以指定构建配置特定值。

IntelliJ IDEA提供了一个方便的界面，用于编辑各种级别的默认值。还可以恢复已更改的值的默认值。

主动构建配置

其中一个构建配置对相应的模块设置为活动状态。

活动构建配置为编辑器中的源代码突出显示提供了基础。因此，当您更改活动配置时，模块中突出显示的代码也会更改。

更改构建配置类型时可能的更改

更改构建配置类型时 （通过使用“更改类型”命令或创建构建配置的副本），如果构建配置输出类型已更改（例如，从“应用程序”到“库”），则可能会发生以下更改：

  • 如果输出类型已更改为库，则会删除对运行时加载的模块（如果存在）的依赖性。

  • 如果对具有已更改类型的构建配置的依赖性变得不合适，则会删除此类依赖项。

创建构建配置的副本时，还会发生以下更改：

  • 输出文件名更改。

  • 如果适用，包文件名或名称会更改。

初步步骤

安装Erlang OTP

设置Erlang环境的第一件事是安装Erlang OTP，这是一组对开发至关重要的Erlang库。

视窗

如果您是Windows用户，请下载Erlang OTP包并运行安装向导。安装完成后，将安装路径+ \bin到PATH环境变量中。

苹果系统

如果您是macOS用户，要安装Erlang OTP，请在终端提示符下键入以下内容（确保在您的计算机上安装了Homebrew ）：

brew安装erlang

如果您更喜欢MacPorts到Homebrew，您的命令行应该是不同的：

port install erlang + ssl

Linux的

Linux的安装过程与macOS类似，不同之处在于您必须使用apt-get （Linux软件包管理实用程序）而不是brew或port ：

apt-get install erlang

您可以随时为任何操作系统下载最新版本的Erlang OTP软件包。

  • 按向左和向右键跳转到父/子提交。如果您在“ 日志”视图中提交了不同的存储库并且多个分支都已混合，则此功能尤其有用。

  3. 点击工具栏上的按钮。

      从其中一个节点中选择所需文件，然后单击“ 显示差异” 工具栏上的图标或按Ctrl + D.差异查看器将显示双面板差异，允许您将当前版本与所选父项进行比较。

配置注释选项

  • 右键单击注释装订线，然后从上下文菜单中选择“ 选项 ”：

    • 忽略空格 ：空格将被忽略（git blame -w ）。这意味着注释将指向先前有意义的提交。

    • 检测文件中的移动：当提交移动或复制同一文件中的行时，将忽略此类更改（git blame -M ）。这意味着注释将指向先前有意义的提交。

    • 检测跨文件的移动：当提交移动或复制在同一提交中修改的其他文件中的行时，将忽略此类更改（git blame -C ）。这意味着注释将指向先前有意义的提交。

  • 注释先前版本 ：如果您发现自己处于特定行中的最后一次更改毫无意义的情况下，此选项很有用，例如，如果所有更改都是代码格式。在这种情况下，您可以检查文件的先前版本是什么样的。

WebStorm与Vagrant集成，可帮助您创建由VagrantFile配置文件定义的可重现的开发环境。您可以创建和删除虚拟框 ， 初始化Vagrant框 ，通过从Vagrant配置文件中读取设置来配置远程节点解释器，并在不离开IDE的情况下执行其他Vagrant命令。执行vagrant up命令（Multiple Vagrant配置）时，您还可以在多个虚拟框中进行选择。

在WebStorm的上下文中，以下术语用于表示Vagrant特定的概念：

  • 在当前文档中的WebStorm用户界面或流浪碱箱 A 流浪盒是一个盒子中的本机流浪术语。它表示纯图像，骨架，在其基础上，在您的计算机上自定义，配置和部署特定环境。

  • 实例是虚拟机 。换句话说，它是个性化，配置和颠沛流离的底盒的基础上，并按照部署你的机器上一个特定的环境Vagrantfile 。在其他类似的产品和文档中， 实例可以称为虚拟机 。

实时模板变量

展开实时模板缩写时，其变量或者显示为输入字段，您可以在其中键入值或替换为值。这些可能是您可以修改的默认值或使用函数计算的值。

模板中的变量以以下格式声明：

$ <变量> $

要向模板文本添加美元字符（ $ ）以使其不被视为变量声明，请使用特殊变量$$ 。

每个变量都由表达式定义，并且可以在表达式无法计算时接受某些默认值。

此表达式可能包含以下基本类型的构造：

  • 双引号中的字符串常量。

  • 实时模板中定义的另一个变量的名称。

  • 具有可能参数的预定义函数。

可以在“ 编辑模板变量”对话框中配置模板变量 。

配置模板变量

  1. 按Ctrl + Alt + S打开“ 设置/首选项”对话框，然后转到“ 编辑器”|实时模板 。

  2. 选择要配置变量的模板。

  3. 在模板文本中指定变量，然后单击编辑变量

  4. 在“ 编辑模板变量”对话框中，您可以对每个变量执行以下操作：

    • 更改变量名称。

    • 使用预定义函数定义表达式。

    • 指定表达式无法评估的情况的默认值。

    • 如果使用其表达式定义变量，请指定在提示用户输入时是否要跳过变量。

预定义的模板变量

WebStorm支持以下预定义的无法修改的实时模板变量：

  • $END$表示代码片段完成$END$标的位置，您无法再按Tab键跳转到下一个变量。

  • $SELECTION$用于环绕模板，表示要包装的代码片段。展开模板后，所选文本将按照模板中的指定进行包装。例如，如果在代码中选择EXAMPLE并通过指定的缩写调用"$SELECTION$"模板，或者按Ctrl + Alt + T并从列表中选择所需的模板，WebStorm将使用双引号将选择内容包装如下： "EXAMPLE" 。

    JSON

    JSON格式通常用于存储数据和配置文件。WebStorm可以帮助您处理JSON文件 - 它会检查它们的语法和格式。在流行的配置文件类型中，WebStorm提供了代码完成功能，这要归功于JSON Schema ，它是一种用于描述此类文件的结构和内容的特殊格式。您还可以使用自定义JSON模式在JSON文件中启用代码完成并验证它们。

    启用JSON5

    WebStorm认可了许多最受欢迎的JSON标准，包括JSON5 。WebStorm默认将json5扩展名的文件视为JSON5文件，并在其中支持这种新语法。

    将JSON5语法扩展为所有JSON文件

    1. 在“ 设置/首选项”对话框（ Ctrl + Alt + S ）中，转到“ 编辑器”|文件类型 。将打开“ 文件类型”页面。

    JSON Schema Store中的模式也可以应用于YAML文件。

    WebStorm可以自动从JSON架构存储中下载和使用架构，该架构托管许多常用配置文件的架构文件。只要打开名称与其中一个可用模式关联的文件（例如tslint.json ），WebStorm就会下载并使用此模式。应用架构的名称显示在状态栏上。

    除了使用IntelliJ IDEA从任务tress运行任务或目标时自动创建的临时运行配置，您还可以创建并启动自己的Grunt.js运行配置。
