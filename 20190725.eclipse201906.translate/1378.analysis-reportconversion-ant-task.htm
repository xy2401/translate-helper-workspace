API工具分析报告转换Ant任务

目的

此任务将apitooling.analysis Ant任务创建的XML报告转换为HTML文件。

用法

描述

Ant任务的名称是： apitooling.analysis_reportconversion 。要使用，jar文件apitooling-ant.jar必须位于Ant类路径上。 

<apitooling.analysis_reportconversion xmlfiles = “...” htmlfiles = “...” debug = “...” />

参数

属性 描述 需要
XMLFILES 设置检索xml报告的位置。

使用绝对路径设置位置。 		是
htmlfiles 设置生成html报告的位置。

如果未设置，则会在与xml文件相同的文件夹中创建html文件。

使用绝对路径设置位置。 		没有
调试 设置调试值。

可能的值为： true ， false
默认值为false 。 		没有

例子

 <apitooling.analysis_reportconversion xmlfiles = “D：\ reports \ xml” htmlfiles = “D：\ reports \ html” debug = “true” />

这将在文件夹D:\reports\html创建一个index.html 。将在D:\reports\html的子文件夹中创建每个包的D:\reports\html 。每个子文件夹都包含一个report.html 。

由于启用了调试，因此一些调试跟踪将显示在Ant控制台中。
