BPMN2新文件向导

Eclipse New File向导用于创建新的BPMN2文件。在主菜单中，单击文件 - >新建 - >其他 ，这将打开“新建文件向导”对话框。导航到BPMN2类别，您将看到如下所示的几个条目：

图 4：Eclipse“新文件”向导

BPMN模型向导

第一个条目BPMN2 Model实际上是由Eclipse BPMN2 Metamodel插件提供的。这将创建一个仅包含根元素的空BPMN2文件。当使用BPMN2模型编辑器 （一个由BPMN2 Metamodel插件提供的简单的面向树的编辑器 ）打开它时，它看起来像这样：

图 5：BPMN2元模型编辑器

由于这些组件是Eclipse BPMN2 Metamodel项目的一部分，因此本文档不再对它们进行进一步讨论。但是，只要说这个编辑器可以用于查看BPMN2文件的物理结构就足够了，并且比可视化XML编辑器略胜一筹。

 

 

通用BPMN 2.0图表向导

BPMN2类别中的第二个条目Generic BPMN 2.0 Diagram由BPMN2 Modeler插件提供，可用于创建新的，正确初始化的Diagram文件。此向导将创建一个不用于部署到任何特定BPM流程引擎的文件（有关详细讨论，请参阅Target Runtime Extensions ）。选择此条目将显示向导的第一页，如下所示：

图 6：BPMN2建模器向导第1页

这允许您选择要创建的图表类型。此页面包含每种图表类型的简要说明;单击描述旁边的图像选择一个，然后单击下一步>按钮。有关图表类型的更详细讨论，请参阅附录 。

向导的下一页要求提供位置，文件名和目标命名空间。这些字段已填入合理的默认值，但您可能需要根据需要进行更改。

图 7：BPMN2建模器向导第2页

单击“ 完成”按钮为选定的图表类型创建模板，然后打开BPMN2 Modeler。

jBPM流程图向导

BPMN2 Modeler扩展机制的一部分是插件能够提供新文件向导条目的能力。这些将显示为BPMN2类别下的附加条目。这里， jBPM流程图向导专门用于创建适合部署到jBPM引擎的流程。

图 8：jBPM新文件向导

此向导的第一页也是唯一一页要求提供一个进程名称，该进程名称在编辑时会自动为其余字段填写适当的值（ 进程ID和文件名 ）。这些可以根据需要进行编辑。

还需要一个包名称 ;这是jBPM引擎所需的BPMN2语言的扩展 。

标记为“ 设置jBPM运行时 ”的可选复选框作为此项目的默认设置，将创建适用于jBPM目标运行时的项目设置。
