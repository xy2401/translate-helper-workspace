C / C ++项目属性：预处理器包括路径，宏等 - 提供程序选项卡

C / C ++索引器和其他语言设置的包含路径和预处理器宏由语言设置提供程序提供，作为Scanner Discovery的一部分。可以在“提供程序”选项卡中为所选配置设置和配置语言设置提供程序。

“提供者”选项卡仅显示在项目的属性中，而不显示在文件或文件夹的属性中。

提供程序选项卡控件
组 描述
组态 项目的一组属性。不需要调试。
管理配置...... 用于创建或管理配置的对话框。不需要调试。
语言设置提供商列表 列出可用于当前配置的所有提供程序。选择要分配给配置的提供程序。
清除条目 清除所选提供商的所有条目。按“应用”或“确定”后，此操作生效。通常，如果提供程序提供内置编译器设置，则会触发重新运行。构建输出解析器将在构建解析新构建输出后重新获得语言设置。
如果提供程序没有任何语言设置，则此操作可能不可用。某些提供商可能无法清除。
在此属性页上，此操作仅适用于非共享提供程序。
重启 将所选提供程序的选项重置为默认值。此操作还会清除提供程序条目。如果更改了“语言设置提供程序选项”面板中的某些选项，则可以启用该按钮。
在此对话框中，此操作仅适用于非共享提供程序。
提升 将所选提供程序在列表中向上移动。
下移 将选定的提供者移动到列表中较低的位置。

用户可以在此属性页上为配置提供程序（或“非共享”）定义一些选项。请注意，提供程序可以是每个配置共享或定义的。可以在C / C ++首选项：扫描程序发现上配置全局（或“共享”）提供程序的选项 。

为每个配置定义的非共享提供程序添加了2个特定选项。它们在下表中描述。

发现选项卡
组 描述
使用项目之间共享的全局提供程 提供者可以是全局的或属于配置。全局提供者在引用它们的所有配置之间共享它们的条目。
将条目存储在项目设置文件夹中 提供者可以在工作区或项目文件夹下保留其条目。在适当的时候在项目下存储条目可以更容易地与其他团队成员共享。
