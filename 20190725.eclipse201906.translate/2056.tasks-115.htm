与存储库同步

使用共享资源需要在存储库内容的上下文中跟踪其状态。这可以通过创建一个同步来填充Synchronize视图并获得以下优势来完成：
  • 正在同步的资源的本地修改状态保持最新。这意味着，如果您在本地修改同步范围内显示的同步范围内的文件，则资源将自动显示在视图中（如果尚未存在）。
  • 可以使用Synchronize刷新正在同步的所有资源的远程状态 “同步”视图工具栏中的按钮。
  • 可以使用“同步”视图中的上下文菜单中的“同步”命令刷新所选资源的远程状态。

以下方法可用于创建同步。

方法1：使用上下文菜单

要将Workbench中的资源与存储库中的资源同步：

  1. 在其中一个导航视图中，选择要同步的资源。
  2. 右键单击并选择Team> Synchronize with Repository 。“同步”视图将打开。

方法2：使用同步操作

  1. 从Team Synchronizing透视图中选择Synchronize ... 来自Synchronize的动作按钮下拉。
  2. 从可能的同步类型列表中选择CVS ，然后单击“ 下一步” 。
    注意：如果没有向Workbench注册其他存储库提供程序，则您将完全绕过此屏幕。
  3. 通过选择Workspace，Selected Resources或Working Set选择同步的资源范围。然后选择Finish。
  4. “同步”视图将打开。

注意 ：默认情况下，在其他透视图中未启用同步操作。您可以通过选择使操作显示在当前透视图中 窗口>透视>自定义透视 。然后单击“ 命令”选项卡并选中“ 团队” 。

提示 ：您可以使用下面的方法3来避免丢失创建的同步。

方法3：在Synchronize视图中使用固定的CVS工作区同步

在“同步”视图中进行CVS工作区同步后，您可以将其固定。这将防止它被使用前两种方法之一启动的下一个CVS工作区同步替换。以下是使用固定同步的一些优点。

  • 您可以删除任何未固定的同步，以使固定的同步可见。为此，请从“同步”视图工具栏的菜单中选择“ 删除当前同步 ”。
  • 您可以安排刷新远程状态以特定间隔（例如每小时）发生

这样做的含义是，您可以在不从服务器重新获取远程状态的情况下看到外发资源（可能是长时间运行的操作）。此外，远程状态的获取在后台运行，因此您可以在获取远程状态时执行其他操作（例如，检查更改）。

在同步视图中

使用工具栏按钮切换此视图的模式。有四种模式：

  • 传入模式 - 仅显示传入更改（存储库中的资源与Workbench中的资源不同）。
  • 传出模式 - 仅显示传出更改（在Workbench中修改的资源）。
  • 传入/传出模式 - 显示传入和传出更改。
  • 冲突模式 - 仅显示冲突的资源。

注意：自您开始处理文件以来，有人可能已提交了您的文件的新修订版。这将导致冲突 ，必须小心解决这个问题。因此，在“同步”视图的所有模式中都会显示冲突。

要点：最好先更新Workbench中的资源，通过合并解决存在的任何冲突，然后将Workbench资源提交到存储库。
