跟踪选项卡

“ 跟踪”选项卡允许您将平台置于调试模式并启用跟踪，从而将调试语句打印到控制台。

追踪

该平台提供了一种在运行时跟踪插件活动的机制。它允许您使用跟踪标志，这将导致跟踪信息打印到控制台。

这些标志在.options文件中定义。这些属性文件必须位于插件的根目录，并且必须具有以下语法：

  • <plug-in ID> / debug = true / false（主交换机）
  • <插件ID> / <跟踪标志> = <值>

例如，上面选择的org.eclipse.ui.ide插件有一个包含三个键值对的.options文件：

  • org.eclipse.ui.ide / debug = true
  • org.eclipse.ui.ide / debug / gc = true
  • org.eclipse.ui.ide / debug / internalerror / openDialog = true

在您的代码中，要检查您的插件是否处于调试模式（即主交换机的状态），您可以检查<Activator> .isDebugging（）的返回值，其中<Activator>是Bundle中指定的类-插件的MANIFEST.MF文件的-Activator标题。

可以通过调用Platform.getDebugOption（optionName）获取各个标志名称的值。

启用跟踪

选中 “ 启用所选插件的跟踪”选项后，PDE将使用-debug <filename>程序参数启动。程序参数的<filename>部分是PDE生成的属性文件，其中包含要跟踪的插件的键值对和每个跟踪标志的状态。

请注意，在“ 跟踪”选项卡上切换跟踪标志不会导致修改.options文件。跟踪标志值在启动配置本身中保存为元数据，并复制到通过-debug参数传递给运行时的主属性文件中。
