透视

透视图是Workbench窗口中的视图布局（开发工具）。每种类型的透视图都是视图，菜单和工具栏的组合，使您可以执行特定任务。在独立调试器中，您只能选择Debug透视图，该透视图具有允许您调试程序的视图。当您使用独立调试器时，您将进入Debug透视图。

选择/打开视图：
  • 您可以向透视图添加视图。从菜单栏中选择“ 窗口”>“显示视图”并打开一个类别，然后从“ 显示视图”对话框中选择一个新视图。
  • 要将Debug透视图重置为其原始布局，请从菜单栏中选择Window> Reset Perspective 。

调试透视图

调整Debug透视图以调试C / C ++程序。默认情况下，它包含编辑器区域和以下视图：

  • 调试
  • 变量
  • 断点
  • 大纲
  • 安慰
  • 任务
  • 问题
  • 可执行文件

在调试会话期间可能有一些其他视图可能很有用。您可以通过单击Window> Show View> Debug来添加它们，以在Debug透视图中显示它们。

其他C / C ++视图

提供了一些可以使用Window> Show View> C / C ++访问的其他C / C ++视图。这些通常不需要调试，但如果存在问题并且用户对完整的CDT IDE有所了解，则可能会有所帮助。其他观点是：

  • C / C ++项目
  • C / C ++索引视图
  • 调用层次结构
  • 包括浏览器
  • 键入层次结构
