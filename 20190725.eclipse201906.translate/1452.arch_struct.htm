运行时核心

平台运行时核心实现运行时引擎，该引擎启动平台基础并动态发现和运行插件。插件是一个结构化组件，它使用OSGi清单（ MANIFEST.MF ）文件和插件清单（ plugin.xml ）文件向系统描述自身。该平台维护已安装插件的注册表及其提供的功能。

运行时的一般目标是最终用户不应为已安装但未使用的插件支付内存或性能损失。可以安装插件并将其添加到注册表中，但除非根据用户的活动请求插件提供的功能，否则不会激活插件。

平台运行时使用OSGi服务模型实现。虽然运行时的实现细节对许多应用程序开发人员来说可能并不重要，但那些已经熟悉OSGi的人会认识到Eclipse插件实际上是一个OSGi包。

了解运行时系统的最佳方法是构建插件。请参阅插入：Hello World与工作台相遇以开始构建插件。要了解运行时系统的细节，请参阅运行时概述 。

资源管理

资源管理插件定义了用于管理工具插件工件的公共资源模型。插件可以创建和修改项目 ， 文件夹和文件，以便在磁盘上组织和存储开发工件。

资源概述提供了资源管理系统的概述。

Workbench UI

Workbench UI插件实现了工作台UI，并定义了许多扩展点，允许其他插件提供菜单和工具栏操作，拖放操作，对话框，向导以及自定义视图和编辑器。

插入工作台介绍了工作台UI扩展点和API。

其他UI插件定义了通常对用户界面开发有用的框架。这些框架用于开发工作台本身。使用框架不仅可以简化插件用户界面的开发，还可以确保插件具有通用的外观和一致的工作台集成。

Standard Widget Toolkit（SWT）是一个独立于操作系统的低级工具包，支持平台集成和可移植API。它在Standard Widget Toolkit中描述。

JFace UI框架提供更高级别的应用程序构造，用于支持对话框，向导，操作，用户首选项和窗口小部件管理。JFace中的功能在Dialogs和向导以及JFace：用于插件的UI框架中进行了描述。

团队支持

Team插件允许其他插件定义和注册团队编程，存储库访问和版本控制的实现。Eclipse SDK包含一个CVS插件，该插件使用团队支持在SDK中提供CVS客户端支持。

团队支持在团队支持中描述。

调试支持

Debug插件允许其他插件实现特定于语言的程序启动器和调试器。

程序调试和启动支持中描述了调试支持 。

帮助系统

帮助插件实现了平台优化的帮助Web服务器和文档集成工具。它定义了插件可用于提供帮助或其他插件文档作为可浏览书籍的扩展点。文档Web服务器包括特殊功能，允许插件使用基于逻辑插件的URL而不是文件系统URL来引用文件。

提供了其他功能，用于在产品级文档配置中集成帮助主题。

插件帮助中描述了帮助工具。

Java开发工具（JDT）

Java开发工具（JDT）插件通过提供用于编辑，查看，编译，调试和运行Java代码的专用功能来扩展平台工作台。

JDT安装为SDK中包含的一组插件。 Java开发用户指南介绍了如何使用Java工具。JDT Plug-in Developer Guide描述了JDT的结构和API。

插件开发环境（PDE）

插件开发环境（PDE）提供了自动创建，操作，调试和部署插件的工具。

PDE安装为SDK中包含的一组插件。PDE指南介绍了如何使用环境。
