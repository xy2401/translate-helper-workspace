调试模型演示

由于平台中的调试元素有一个通用的统一模型，因此可以为实现调试器UI提供一个起点。调试器UI支持的核心是调试模型表示 （ IDebugModelPresentation ）。调试模型演示文稿负责提供与特定调试元素关联的标签，图像和编辑器。

定义自己的调试模型的插件通常提供用于在模型中显示调试元素的调试模型表示。这是使用org.eclipse.debug.ui.debugModelPresentations扩展点完成的。此扩展点允许IDebugModelPresentation的实现与特定调试模型的标识符相关联。

回想一下， 调试模型元素知道它们的原始调试模型的id。这意味着给定任何调试元素，调试平台可以获取调试模型的id，然后查询扩展注册表以获取任何相应的调试模型演示。

添加调试模型演示文稿的标记如下所示：

<extension point =“org.eclipse.debug.ui.debugModelPresentations”> <debugModelPresentation class =“org.eclipse.jdt.internal.debug.ui。JDIModelPresentation“id =”org.eclipse.jdt.debug“detailsViewerConfiguration =”org.eclipse.jdt.internal.debug.ui.display。DetailsViewerConfiguration“> </ debugModelPresentation> </ extension>

除了调试模型演示之外，还可以指定可选的detailsViewerConfiguration 。详细信息查看器必须扩展JFace SourceViewerConfiguration类。“详细信息”的含义由调试模型解释。详细信息由调试模型演示计算并传递给详细信息查看器。例如，Java调试器使用详细信息查看器在计算表达式时在变量视图中显示代码帮助。

当需要对显示堆栈帧的源代码的编辑器进行更多控制时， IDebugModelPresentation的实现者也可以实现IDebugEditorPresentation 。实现者可以控制编辑器的位置或源中的注释。
