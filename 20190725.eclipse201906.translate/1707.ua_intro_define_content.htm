定义介绍内容

现在我们可以看一下内容本身。内容以页面形式描述。所有页面都有一个id属性。这是在定义主页和备用页以及存在对页面的引用的其他位置时使用的标识。否则，相关属性取决于定义的页面类型。有两种基本类型的页面：

  • 静态页面是纯HTML文件。这些页面使用常规HTML机制链接到其他页面。除主页外，无需在配置内容文件中定义静态页面。由于主页由表示元素中的id（ home-page-id ）指定，因此必须在内容文件中使用该id定义页面。此页面只需要定义一个URL 。由于HTML页面本身将描述页面内容，因此将忽略所有其他子元素。插件提供的所有其他HTML介绍页面必须包含在插件中，但不需要在内容文件中指定。也可以引用位于其他插件或Web上的HTML文件。
  • 使用描述页面内容的子元素在XML内容文件中描述动态页面 。子元素是经常在类似HTML的页面中找到的UI项目。根据实现，这些页面将动态转换为HTML（当实现类型为html时）或动态创建为基于SWT的UI表单（当实现类型为swt时）。可以在页面中定义以下子元素：
    • 组用于对其他子元素进行分组，并在整个组中定义一致的样式。
    • 链接定义了可以使用图像和文本显示的链接。该链接可以导航到另一个页面，并可选择运行介绍操作。操作在URL中指定为命令。
    • text和img元素显示文本和图像内容。
    • include元素包括先前定义的子元素。该元素由其id引用。
    • head元素定义了在使用html实现时要包含在页面头部的其他HTML。
    • 当使用html实现时， html元素定义要包含在页面主体中的其他HTML。
    还可以定义页面的标题 。页面还可以指定其内容在单独的内容文件中定义。当关注性能时，将页面分成单独的文件可能很有用，因为在需要之前不会初始化介绍页面的内容。

了解内容定义格式的最佳方法是浏览Eclipse SDK中的实现。下面的代码段显示了一个示例：

   <introContent>
      <page alt-style="css/root_swt.properties" style="css/root.css" id="root" style-id="page">
         <title style-id="intro-header">Welcome to Eclipse Platform</title>
         <group id="links-background">
            <group id="page-links">
               <link label="Overview" url="http://org.eclipse.ui.intro/showPage?id=overview" id="overview" style-id="left">
                  <text>Find out what Eclipse is all about</text>
               </link>
               <link label="Tutorials" url="http://org.eclipse.ui.intro/showPage?id=tutorials" id="tutorials" style-id="left">
                  <text>Let us guide you through Eclipse end-to-end tutorials</text>
               </link>
               <link label="Samples" url="http://org.eclipse.ui.intro/showPage?id=samples" id="samples" style-id="right">
                  <text>Explore Eclipse development through code samples</text>
               </link>
               <link label="Whats New" url="http://org.eclipse.ui.intro/showPage?id=news" id="news" style-id="right">
                  <text>Find out what is new in this release</text>
               </link>
            </group>
         </group>
      ...

页面上的元素也可以从特定实现中过滤 。这允许页面设计者在设计时考虑特定平台。在描述页面及其内容时，可以使用许多更强大的属性。有关有效元素，子元素及其属性的完整参考，请参阅org.eclipse.ui.intro.config的扩展点文档及其关联的介绍内容文件格式规范 。
