外部工具

外部工具允许您使用Workbench配置和运行程序，批处理文件，Ant构建文件和其他工具。您可以保存这些外部工具配置并在以后运行它们。

外部工具的输出显示在“ 控制台”视图中 。

您可以添加外部工具作为项目构建过程的一部分。每次构建项目时，这些外部工具都将以指定的顺序运行。

配置外部工具时，可以使用以下变量。每次运行外部工具时，这些变量都会自动扩展。

变量名

描述
$ {} workspace_loc 系统硬盘到Eclipse工作区目录的绝对路径
$ {workspace_loc： <资源路径> } 系统硬盘到指定资源的绝对路径。<资源路径>是资源相对于工作空间根目录的完整路径。例如$ {workspace_loc：/MyProject/MyFile.txt}。请注意，如果MyProject的项目内容目录位于工作空间目录之外，则此变量的扩展结果与$ {workspace_loc} /MyProject/MyFile.txt不同。
$ {} project_loc 如果外部工具作为构建的一部分运行，则系统硬盘驱动器上当前所选资源的项目或正在构建的项目的绝对路径。
$ {project_loc： <资源路径> } 系统硬盘驱动器到指定资源项目的绝对路径。<资源路径>是资源相对于工作空间根目录的完整路径。例如$ {workspace_loc：/MyProject/MyFile.txt}。请注意，如果MyProject的项目内容目录位于工作空间目录之外，则此变量的扩展结果与$ {workspace_loc} / MyProject不同。
$ {} container_loc 系统硬盘驱动器上当前所选资源的父级（文件夹或项目）的绝对路径。
$ {container_loc： <资源路径> } 系统硬盘驱动器上指向资源的父级（文件夹或项目）的绝对路径。<资源路径>是资源相对于工作空间根目录的完整路径。例如：$ {workspace_loc：/MyProject/MyFolder/MyFile.txt}。请注意，如果MyProject的项目内容目录位于工作空间目录之外，则此变量的扩展结果与$ {workspace_loc} / MyProject / MyFolder不同。
$ {} resource_loc 系统硬盘驱动器上当前所选资源的绝对路径。
$ {resource_loc： <资源路径> } 系统硬盘到指定资源的绝对路径。<资源路径>是资源相对于工作空间根目录的完整路径。例如$ {workspace_loc：/MyProject/MyFile.txt}。请注意，如果MyProject的项目内容目录位于工作空间目录之外，则此变量的扩展结果与$ {workspace_loc} /MyProject/MyFile.txt不同。
$ {} project_path 如果外部工具作为构建的一部分运行，则相对于工作空间根目录的当前所选资源项目或正在构建的项目的完整路径。
$ {} container_path 当前所选资源的父级（文件夹或项目）的相对于工作空间根的完整路径。
$ {} resource_path 当前所选资源的相对于工作空间根的完整路径。
$ {} PROJECT_NAME 如果外部工具作为构建的一部分运行，则当前所选资源的项目或正在构建的项目的名称。
$ {} CONTAINER_NAME 当前所选资源的父级（文件夹或项目）的名称。
$ {} RESOURCE_NAME 当前所选资源的名称。
$ {} build_type 外部工具作为构建的一部分运行时的构建类型。该值可以是“完整”，“增量”或“自动”之一。如果外部工具在构建之外运行，则值为“none”。

假设你的Eclipse工作区目录是c:\eclipse\workspace ，你有两个项目， MyProject1和MyProject2 。第一个项目MyProject1位于工作区目录内，第二个项目MyProject2位于工作区目录外的c:\projects\MyProject2 。让我们看一下如果选择资源/MyProject2/MyFolder/MyFile.txt ，在运行外部工具时如何扩展下面的变量示例。

变量示例

扩大的结果
$ {} workspace_loc C：\蚀\工作区
$ {workspace_loc：/MyProject1/MyFile.txt} C：\蚀\工作空间\ MyProject的\ MyFile.txt的
$ {workspace_loc：/MyProject2/MyFile.txt} C：\项目\ MyProject2 \ MyFile.txt的
$ {} project_loc C：\项目\ MyProject2
$ {project_loc：/MyProject1/MyFile.txt} C：\蚀\工作空间\ MyProject的
$ {} container_loc C：\项目\ MyProject2 \ MyFolder文件
$ {} resource_loc C：\项目\ MyProject2 \ MyFile.txt的
$ {} project_path / MyProject2
$ {} container_path / MyProject2 / MyFolder文件
$ {} resource_path /MyProject2/MyFolder/MyFile.txt
$ {} PROJECT_NAME MyProject2
$ {} CONTAINER_NAME MyFolder文件
$ {} RESOURCE_NAME MyFile.txt的
$ {} build_type 没有
