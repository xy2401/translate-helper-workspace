定义介绍部分

IIntroPart接口和org.eclipse.ui.intro扩展点构成了可用于为给定产品创建自己的介绍支持的通用机制。此扩展的主要目的是定义实现IIntroPart的类，并指定产品ID和介绍部分之间的绑定。例如，以下贡献定义了工作台在启动时显示的假设介绍部分：

<extension point =“org.eclipse.ui.intro”> <intro class =“com.example。SampleIntroPart“id =”someId“icon =”someIcon.png“> </ intro> <introProductBinding introId =”someId“productId =”com.example.someProductId“> </ introProductBinding> </ extension>
此贡献首先定义介绍部分并为其分配id“someId”。然后它将此介绍部分绑定到id为“com.example.someProductId”的产品。在平台启动时， class属性中指定的类将由工作台实例化，并作为产品介绍呈现给用户。这是与IIntroPart接口的最低级别集成。

该平台提供了自己的名为CustomizableIntroPart的IIntroPart实现，允许自定义内容和介绍。下面是定义工作台的介绍部分的片段。我们不会讨论实现介绍部分的机制，因为我们希望专注于定义介绍内容。（如果需要，请参阅上面引用的扩展点文档和javadoc以获取更多详细信息。） 

<extension point =“org.eclipse.ui.intro”> <intro class =“org.eclipse.ui.intro.config。CustomizableIntroPart“id =”org.eclipse.platform.intro“> </ intro> <introProductBinding introId =”org.eclipse.platform.intro“productId =”org.eclipse.platform“> </ introProductBinding> </ extension>
上面的贡献将CustomizableIntroPart定义为用于Eclipse SDK平台的介绍部分。本讨论的其余部分将向您展示如何使用和扩展此部分。
