来源附件属性页面

要浏览库中包含的类型的源，可以将源存档或源文件夹附加到此库。然后编辑器将显示源代码而不是反编译代码。设置源附件还允许使用调试器进行源级别步进。

可以通过多种方式访问“源附件”对话框：

  • 在Package Explorer中选择一个JAR，然后从上下文菜单或Project菜单中选择Properties> Java Source Attachment
  • 打开项目的Java Build Path页面（ Projects> Properties> Java Build Path ）。在“ 库”页面上，展开库的节点，然后选择“ 源附件”属性并按“ 编辑”
  • 在类文件上打开编辑器。如果尚未为此JAR配置源附件，则编辑器包含按钮Attach Source

根据JAR对类路径的贡献，您可以看到不同类型的源附件对话框：

罐

在“ 路径”字段中，输入存档的路径或包含源的文件夹。使用“ 工作空间” ，“ 外部文件”或“ 外部文件夹”按钮浏览位置。

对于外部位置中的源附件，您还可以指定用于读取源附件的可选编码 。默认情况下，使用工作空间编码。

变量

在“ 位置变量路径”字段中，输入指向源附件位置的变量路径 。变量路径具有作为第一段的变量（将解析为文件夹或文件），其余是可选的路径扩展（例如MYVARIABLE / src.jar ）。使用“ 变量”按钮选择现有变量，使用“ 扩展”按钮选择扩展路径。扩展按钮仅在可以扩展变量时解析（解析为文件夹）

JRE_SRC是一个保留变量，指向在Installed JREs首选项页面中选择的JRE （ Java>已安装的JRE ）。转到此首选项页面以配置JRE库的源附件。
