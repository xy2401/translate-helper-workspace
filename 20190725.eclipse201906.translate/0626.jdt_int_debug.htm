JDT调试

JDT Debug由几个支持Java代码运行和调试的插件组成。

  • org.eclipse.jdt.launching是定义Java启动和运行时支持的插件。如果需要以编程方式启动Java虚拟机，则需要此插件。JDT的启动与平台启动设施密切相关，平台启动设施在启动程序中有所描述。

    org.eclipse.jdt.launching包提供了从平台启动Java运行时的类。JavaRuntime实现静态方法以访问已注册的VM并计算运行时类路径和源查找路径。VM系列（例如JDK）由IVMInstallType类表示。 IVMInstall表示一个系列中的特定安装。IVMRunner用于启动特定的Java VM并使用调试插件注册其进程。

    org.eclipse.jdt.launching.sourcelookup.containers包定义了用于在文件系统中定位源代码元素的类。

    包org.eclipse.jdt.launching.sourcelookup.advanced定义了用于在文件系统中定位源代码元素或使用高级源查找按需下载的类。

  • org.eclipse.jdt.debug是定义Java调试模型的插件。如果需要以编程方式访问正在调试的程序中的对象，则需要此插件。JDT调试模型与平台调试模型密切相关，平台调试模型在平台调试模型中进行了描述。

    org.eclipse.jdt.debug.core包支持基于JDI / JDWP的Java调试模型，该模型可用于控制调试下的Java程序。

    org.eclipse.jdt.debug.eval包提供了用于评估Java表达式和报告结果的基础结构。

  • org.eclipse.jdt.debug.ui是定义Java调试UI扩展的插件。大多数调试器API由调试模型表示和调试UI实用程序类中描述的平台调试器基础结构提供。Java调试UI API侧重于访问提示源定位器和Java启动配置选项卡。

    org.eclipse.jdt.debug.ui.launchConfigurations包定义了本地和远程Java应用程序的启动配置选项卡。
