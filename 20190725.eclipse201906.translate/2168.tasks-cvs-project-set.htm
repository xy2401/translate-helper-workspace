使用项目集共享工作区设置

您的工作区设置可能包含来自一个或多个存储库的多个项目。设置工作区后，可以通过导出团队项目集与他人共享。项目集是一个文本文件，其中包含指向项目集中包含的每个项目的指针。导入项目集时，这些指针用于从存储库中获取项目。项目集可以包括映射到为其提供支持的存储库工具的任何项目，例如CVS。要导出项目集：

  1. 通过从CVS中检出它们或以适合您正在使用的存储库工具的任何方式获取它们来设置您想要处理的所有项目的工作空间。
  2. 从“ 文件”菜单中，选择“ 导出” 。该导出对话框将打开。
  3. 在Export对话框中，选择Team> Team Project Set ，然后单击Next 。该对话框现在将显示所有符合导出条件的项目。
  4. 检查要包含在项目集中的项目。浏览或键入要保存项目集的文件的名称，然后单击“ 完成” 。

现在您已经创建了一个项目集文件。您现在可以与他人共享此文件，也可以自己使用它来重新创建工作区。要导入项目集：

  1. 从“ 文件”菜单中，选择“ 导入” 。这将打开导入对话框 。
  2. 在Import对话框中，选择Team> Team Project Set ，然后单击Next 。
  3. 浏览或键入包含项目集的文件的名称，然后单击“完成”。
  4. 项目集中包含的项目将从存储库中获取。

对于CVS，系统将提示您提供存储库的身份验证信息（用户名和密码），因为它未包含在项目集中。

指定存储库信息对话框

当项目集仅包含部分存储库信息（并非所有位置都已知）时，将显示“指定存储库信息”窗口，其中提供了三个存储库位置选项：

  1. 使用存储库位置列下拉列表第一行中显示的原始位置。
  2. 单击“ 确定”从同一下拉列表中选择已知的存储库位置（默认）。
  3. 单击“ 创建位置”按钮创建新的存储库位置。

用户可以完全配置存储库位置，指定用户名，并在需要时更改连接方法。例如，非提交者可以使用包含extssh连接的项目集文件，因为他们可以在导入到pserver时重新配置存储库位置连接方法。
