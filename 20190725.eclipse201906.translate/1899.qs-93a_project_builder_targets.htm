项目构建器Ant目标

使用Ant构建文件创建新项目构建器时，要设置和理解的关键信息之一是目标以及何时执行它们。

在创建新的Ant构建文件项目构建器时，默认目标设置为在“清理”和“ 手动构建 ”之后运行，其余两种; 自动生成和“清洁”期间设置为不执行。如果要将构建器配置为从构建文件中为特定构建类型执行特定目标，则需要更多地了解构建类型，更重要的是，何时将执行它们。

以下部分描述了每种构建类型以及何时执行。

经过“清洁”之后

在通过Project> Clean ...命令清理项目后执行此构建类型。

无论您的工作区自动构建设置如何，在清理后立即执行下一次构建期间执行“清理” 。如果你有工作区自动建立在这个会自动执行后的清洁，否则你将不得不调用手工打造的一个命令; 项目>构建项目 ， 项目>全部构建或启用项目>自动构建 。

手动构建

当您关闭工作区自动构建（ Project> Build Automatically ）并通过Project> Build Project或Project> Build All （ Ctrl + B ）命令调用构建时，将执行此构建类型。

手动构建目标执行时有三个例外：

  1. 如果构建是干净后的第一个构建（ Project> Clean ... ），那么将执行After a“Clean”构建
  2. 如果没有设置要运行的目标在“清理”之后 ，则在清理后执行手动构建目标（ 项目>清理... ）
  3. 如果没有要构建，即没有对项目进行任何更改，则不会执行手动构建目标

自动构建

当你有工作区自动生成（ 项目>自动构建 ），将执行此样订做开启和您在您的项目更改并保存它们。

在“清洁”期间

通过（ Project> Clean ... ）命令调用干净构建时，将执行此构建类型。

在“清理”期间 ，不依赖于工作区自动构建设置（ 项目>自动构建 ），因为它在干净构建期间执行，而不是在“清理”或“ 手动构建 ”之后执行 。
