AntRunner应用程序入口点

Ant核心插件声明了一个主入口点，允许在无头环境中运行Eclipse的集成Ant支持。然后可以直接从主平台启动器运行Ant构建。

这是一个示例.bat文件，显示如何在Windows上使用antRunner应用程序入口点。

回声setlocal REM ********************************************** ****** REM要使用的JRE java.exe设置JAVAEXE =“C：\ jdk1.4.2 \ jre \ bin \ java.exe”REM Eclipse启动程序jar设置LAUNCHERJAR =“C：\ eclipse \ plugins \ org.eclipse.equinox.launcher_1.0.0.v20070319.jar“REM工作空间集的位置WORKSPACE = C：\ runtime-workspace REM用于构建集的构建文件BUILDFILE = build.xml REM ****** **********************************************如果不存在％ JAVAEXE％echo ERROR：不正确的java.exe =％JAVAEXE％，编辑此文件并更正JAVAEXE envar如果不存在％JAVAEXE％goto如果不存在则完成％LAUNCHERJAR％echo ERROR：不正确的equinox.launcher jar =％LAUNCHERJAR％，编辑此文件并更正LAUNCHERJAR envar如果不存在％LAUNCHERJAR％goto如果不存在则完成％WORKSPACE％echo ERROR：不正确的工作区=％WORKSPACE％，编辑此文件并更正WORKSPACE envar如果不存在％WORKSPACE％goto完成如果不存在％BUILDFILE％echo ERROR：不正确的buildfile =％BUILDFILE％，编辑它文件并更正BUILDFILE envar如果不存在％BUILDFILE％goto done：运行@echo on％JAVAEXE％-cp％LAUNCHERJAR％org.eclipse.equinox.launcher。Main -noupdate -application org.eclipse.ant.core.antRunner -data％WORKSPACE％-buildfile％BUILDFILE％：完成暂停
