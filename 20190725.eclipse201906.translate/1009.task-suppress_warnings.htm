使用@SuppressWarnings排除警告

从Java 5.0开始，您可以使用java.lang.禁用相对于编译单元子集的编译警告java.lang.SuppressWarning注释。

@SuppressWarning（“unused”）public void foo（）{String s; }

如果没有注释，编译器会抱怨从不使用局部变量s 。使用注释，编译器在本地静默忽略此警告到foo方法。这样可以将警告保留在同一编译单元或同一项目的其他位置。
