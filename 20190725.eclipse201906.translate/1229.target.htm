目标平台

目标平台是指将构建和运行工作空间的插件。它描述了您正在开发的平台。使用PDE进行开发时，目标平台用于：

  1. 编译 - 工作区中的插件是针对目标平台构建的，因此您不必拥有工作区中的所有内容
  2. 启动 - 使用PDE的启动器时，您可以选择要启动的插件集。默认情况下，Eclipse Application Launcher将从目标中的所有插件开始，但在可用时将使用工作区插件
  3. 计算依赖关系 - PDE通过查看目标平台中可用的内容，帮助您找到插件包含/需要正确编译的插件
  4. 状态 - 为目标平台创建OSGi状态，以便其他工具可以使用它。还可以使用目标平台状态视图详细查看状态
  5. 其他工具 - 其他PDE工具使用目标平台确定其选项，例如导入向导

目标平台引用您当前活动的捆绑包，而目标定义是一种确定要添加到状态的插件的方法。您可以拥有多个目标定义，但只能选择一个定义作为目标平台。

目标平台和目标定义在目标平台首选项页面上进行管理。此页面列出了PDE可以访问的所有目标定义，并显示正在将哪个定义用作当前目标平台。可以使用“目标定义内容向导”在此处创建和编辑目标定义。为了便于在开发团队之间共享，还可以在工作区中创建和编辑目标，将其作为扩展名为“.target”的xml文件。可以使用目标定义编辑器编辑这些文件，并可以使用“新建目标定义向导”创建这些文件。

默认情况下，PDE提供了一个默认目标平台，它只指向当前运行的Eclipse实例（通常称为主机）中的插件。对于Eclipse插件开发人员而言，这可能就是您所需要的。RCP开发人员必须更加谨慎地管理他们的目标，因为添加依赖项会增加其应用程序的大小和复杂性。为RCP创建新的目标定义时，请考虑使用RCP模板（可带或不带源）。

有关最新目标故障排除帮助，请参阅目标定义Wiki页面 。
