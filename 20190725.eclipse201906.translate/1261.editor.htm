扩展点架构编辑器

关于Extension Point架构

扩展点模式是XML模式文件，它定义扩展必须符合的扩展点的合同，以便插入和扩展平台。更具体地说，扩展点模式为扩展点的元素和属性指定了形式语法。该语法规定了任何贡献扩展的XML标记的结构，内容和语义。

在插件开发期间使用扩展点模式来执行以下操作：

  • 验证工作区插件中定义的扩展。
  • 生成参考扩展点文档。
  • 协助用户进行扩展创建。

关于此编辑器

Extension Point架构编辑器允许您执行以下操作：

  • 创建和编辑扩展点模式的语法。
  • 包括并使用其他共享扩展点模式定义的语法。
  • 记录扩展点模式的语法。
  • 指定PDE使用的模式元数据以自动执行常见任务。
  • 预览扩展点架构的参考文档。

此编辑器将扩展点模式的文件标记抽象为一个您不必学习使用的实现细节。

编辑页面

该编辑器由以下页面组成：

  • 架构概述页面

全局编辑器操作

单击“ 预览参考文档”超链接以预览与已编辑的扩展点模式的已保存或未保存内容相关的参考文档。此操作生成架构的HTML表示形式，并在浏览器中打开它。

PDE XML架构支持

扩展点模式是XML模式。因此，它们是用XML编写的，并且能够利用模式注释来提高处理和可读性。但是，扩展点模式仅使用W3C XML Schema规范定义的一小部分XML模式功能。

使用小型XML模式特征子集可以显着简化扩展点模式的结构和语法，同时仍能满足扩展点问题域的所需需求。它还提供了模式语法与等效DTD语法的美学一对一映射。

主要扩展点架构限制如下：

  • 仅允许全局元素声明。
  • 仅允许本地属性声明。
  • 仅支持序列和选择合成器。
  • 只允许使用字符串和布尔属性类型。
  • 对于string类型的属性，仅支持枚举限制。
  • 仅允许本地元素类型声明。
  • 不支持XML命名空间。
  • 可以声明一个元素来指定元素内容或字符内容;但是，不是混合内容（元素和字符内容）。
