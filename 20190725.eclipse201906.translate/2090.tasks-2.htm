升级Eclipse

如果要从较旧版本升级到较新版本的Eclipse，则可以通过简单的步骤将工作区迁移到新版本。您的工作区是磁盘上包含所有项目文件的目录，以及元数据，例如您可能已自定义的首选项。升级所遵循的步骤取决于在启动Eclipse时是否使用了“-data”命令行参数。建议使用“-data”参数，因为它清楚地指定了工作区的位置。

提示：在升级之前备份工作区并没有什么坏处。升级工作区后，您将无法再使用旧版本的Eclipse。如果您想要“及时”回到早期版本，那么您需要备份！

工作区选择器对话框允许您选择工作区的位置。在没有-data参数的情况下，首次启动时会出现此对话框。此对话框提供的默认位置将是主目录的“工作区”子项（例如，C：\ Documents and Settings \ UserName \ workspace。）

除非您具有以前Eclipse版本的现有工作空间，否则您可以保留此默认值或选择其他位置。您不应将工作区存储在Eclipse安装目录中，因为这会使升级到较新版本的Eclipse变得更加困难。您不应复制或移动工作空间目录，因为它可能包含具有绝对文件系统路径的元数据，如果将工作空间复制到其他位置，这些路径将无效。

在Eclipse 3.0及更早版本中，Eclipse的工作空间的默认位置位于Eclipse目录下。例如，如果eclipse.exe位于D：\ eclipse-SDK-3.0.1 \ eclipse中，则默认工作空间位置为D：\ eclipse-SDK-3.0.1 \ eclipse \ workspace。

要加载使用3.0或更早版本创建的工作空间，只需使用工作区选择器指向旧工作区目录，无论它位于旧安装目录下还是其他位置。

 

使用“-data”参数

如果您以前使用“-data”参数启动Eclipse，则升级路径会更容易：

  1. 将Eclipse的新版本安装在一个新的位置，与任何旧版本的Eclipse分开。
  2. 启动此新版本的Eclipse，使用“-data”命令行参数指向旧的工作空间位置。

有关说明，请参阅上一节中的示例。

添加第三方插件

如果您在Eclipse环境中安装了额外的插件，则需要将这些新插件添加到您安装的每个新版本的Eclipse中。在执行此操作之前，请参阅这些插件的文档，以确保它们与您要迁移到的Eclipse版本兼容。有几种方法可以将这些额外的插件添加到新的Eclipse安装中：

  1. 将每个插件的目录复制到新Eclipse版本的“dropins”目录中。
  2. 使用Eclipse更新站点在新版本的Eclipse中重新安装这些额外的插件或功能。
  3. 如果您使用的是产品扩展，只需将“links”目录复制到新的Eclipse版本安装目录中即可。有关产品扩展的更多详细信息，请参阅“ 程序员指南”>“打包和交付基于Eclipse的产品”>“产品扩展 ”下的“平台插件开发人员指南”中的文档。
