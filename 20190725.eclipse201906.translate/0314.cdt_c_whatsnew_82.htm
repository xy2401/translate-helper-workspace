编辑

组织包括命令

Organize Includes命令类似于Java中的Organize Imports 。它添加了缺少的include语句，删除了不必要的语句，并根据用户首选项重新排序。

在确定必须包含哪些头文件时，“ 组织包含”命令将遵循“包括您使用的内容”规则。了解为什么包括您使用的内容？ 什么是用途？ 。

调试

增强表达式

表达式视图已扩展为允许用户手动创建增强表达式 。增强表达式定义了一组表达式，可以使用glob-pattern匹配轻松描述。用户通过在其前面加上'='来指定增强表达式 。例如：

  • 模式匹配的局部变量的排序组，其中符号* []？可以用，例如，
=v?r     -- Will show local variables starting with a 'v' and ending with 'r' with a single character in between
=*       -- Will show all local variables of the selected stack frame in sorted order (the '=' is optional for this expression, i.e., '*')
=*x      -- Will show local variables ending with 'x'
  • 数组范围包括glob-expressions
=array[30-40]        -- Will show array elements from 30 to 40
=array[1-5,20,30-31] -- Will show array elements from 1 to 5, 20 and 30 to 31
=array?[1-5]         -- Will show array elements from 1 to 5 for any array starting with 'array' followed by a single character
  • 模式匹配的排序寄存器组，其中符号* []？可以用，例如，
=$e?x     -- Will show all registers starting with 'e' and ending with 'x' with a single character in between
=$*       -- Will show all registers (the '=' is optional for this expression, i.e., '$*')
=$*x      -- Will show registers ending with 'x'
=$st[3-5] -- Will show registers $st3, $st4, $st5
  • 以分号分隔的，单独排序的表达式组，例如，
var1; var2 -- Will create a group containing both var1 and var2 
$eax; var1 -- Will show a group containing register $eax and variable var1
var1; =$e*  -- Will show a group containing variable var1 as well as all registers starting with 'e'

此功能允许快速定义用户感兴趣的多个表达式。由于组是从这些特殊表达式创建的，因此可以在不感兴趣时折叠它们，并在以后重新展开，而不必由用户重新输入。

此功能已于2012年7月10日完成，并于2012年12月19日更新为本地变量。有关详细信息，请参阅错误381754和错误 394408 。

请注意，逗号（，）不允许作为组分隔符，因为它在使用模板的有效表达式中使用（例如， （（（（（class std :: _ Vector_base <int，std :: allocator <int >>） v））。_M_impl））。_M_start） ）。

OS资源视图

CDT有一个名为“OS资源”的新视图。此视图将显示有关操作系统资源的不同信息。例如，它可以提供目标上运行的所有进程的列表。该视图将显示GDB提供的信息。

撰写时，GDB支持以下信息：

Processes             - Listing of all processes
Process groups        - Listing of all process groups
Threads               - Listing of all threads
File descriptors      - Listing of all file descriptors
Sockets               - Listing of all internet-domain sockets
Shared-memory regions - Listing of all shared-memory regions
Semaphores            - Listing of all semaphores
Message queues        - Listing of all message queues
Kernel modules        - Listing of all loaded kernel modules


笔记：

  • 出于性能原因，视图不会自动刷新。按视图工具栏上的“ 刷新”按钮以获取最新信息。将鼠标悬停在此“ 刷新”按钮上将显示上次获取信息的时间。
  • 列可以重新调整大小。
  • 可以使用视图菜单删除或添加列。
  • 按列列标题可以按列对列表进行排序。
  • 在调试远程目标时，视图中的信息与远程目标有关。

此功能需要GDB 7.5及更高版本。此外，从GDB 7.5开始，此功能仅适用于Linux。

此功能于2012年9月20日完成。有关详细信息，请参阅Bug 360314 。

断点过滤

CDT增强了“断点”视图中“显示所选目标支持的断点”选项的标准行为。在CDT中使用此选项现在只显示实际适用于当前调试会话的断点。因此，在调试C / C ++应用程序时，用户不会被另一个C / C ++应用程序的代码中设置的断点所困扰。

为了向后兼容，提供了一个首选项以将此新行为还原为原始行为。此选项的原始行为是让Breakpoints视图显示与当前调试会话类型相同的所有断点。例如，如果调试Java，则只显示Java断点，如果显示调试C / C ++，则只显示C / C ++断点。可以在“C / C ++ - >调试 - > GDB - >使用积极断点过滤”下找到此首选项。

此功能于2012年10月26日完成。有关详细信息，请参阅Bug 360735 。

增强的GDB控制台支持

正在改进CDT以通过用户从GDB控制台对GDB所做的任何更改来更新其视图。正在逐步添加更新，最终目标是允许用户从GDB控制台执行任何命令，并使CDT与更改保持同步。

断点，观察点和跟踪点

现在，在断点视图中显示从GDB控制台设置的断点，观察点或跟踪点。支持所有与断点相关的GDB命令并与UI同步。尚未支持捕获点。此功能需要GDB 7.4或更高版本。

此功能于2012年11月29日完成。有关详细信息，请参阅错误392512 。

记忆和变量

现在，可以在内存，内存浏览器，变量和表达式视图中更新从GDB控制台修改的内存和变量。此功能需要GDB 7.6或更高版本。

此功能已于2013年1月26日完成。有关详细信息，请参阅错误397715 。

反向调试状态

如果通过GDB控制台修改状态，CDT现在将正确更新反向调试的状态。此功能需要GDB 7.6或更高版本。

此功能已于2013年1月31日完成。有关详细信息，请参阅错误399163

用于控制反向调试的断点操作

现在可以通过断点操作来控制反向调试模式的启用，禁用和切换。反向调试模式可用于调试，但在启用时具有显着的性能成本。使用新的断点操作启用反向调试模式，可以编写断点以在可疑源代码附近启用该模式。这样，在达到这一点之前，不会感觉到性能影响。

要使用此功能，请右键单击断点并选择“断点属性”。然后在左侧页面中选择“操作”。然后单击“新建”。在新对话框中，选择“操作类型”“反向调试操作”。然后选择是否应该启用，禁用或切换反向调试模式。最后为动作选择了一个名称。完成后，单击“确定”

新创建的操作将显示在可用操作列表中，可以附加到任何断点要将其附加到当前断点，请单击“附加”。

注意：要使反向调试断点操作起作用，必须在当前调试会话中提供反向调试。例如，如果“不间断”模式处于活动状态，它将无法工作。

此功能已于2012年12月13日完成。有关详细信息，请参阅错误365776 。

浮点渲染器已添加到内存包中

浮点渲染已添加到内存包中。所以现在有传统渲染器和浮点渲染器可用。此渲染允许显示和编辑浮点值。由于它源自传统渲染，因此它的工作流程和样式是相似的。没有ASCII数据窗格显示，因为这对浮点数据似乎没有多大意义。

该功能于2012年11月16日完成，作为Bug 394509的一部分。

在一个调试会话中调试多个进程

对于从7.2开始的GDB版本，在同一调试会话中调试新的可执行文件已得到改进。 “调试新的可执行文件”操作已添加到“调试”视图的上下文菜单中。


用于指定要为本地会话调试的可执行文件的对话框已更改。


添加了使用gdbserver调试远程可执行文件的功能。

此功能于2013年1月25日完成。有关详细信息，请参阅错误344890 。


在Multicore Visualizer中加载信息 - CPU /核心负载计量表

Multicore Visualizer视图已得到增强，可以以图形负载表的形式支持系统负载监控，显示正在调试的系统（本地或远程）的负载。这仅适用于Linux目标。默认情况下，此功能被禁用，因为远程调试时可能会导致一些负载增加。启用时，它的外观如下：

启用并允许空间时，将为每个核心显示负载计量表，并为每个CPU显示一个负载计量表，其中显示所有包含核心的平均负载。

要启用负载计量表，必须使用Multicore Visualizer的上下文菜单;右键单击可视化器，进入“Load Meters”子菜单并选择“Enable Load Meters”：

也可以选择负载表刷新间隔。启用负载表后，“加速计”子菜单中将添加“刷新速度”条目。在那里你可以选择三种刷新速度。选择一个使它生效。


此功能已于2013年2月28日完成。有关详细信息，请参阅错误396268 。

在Create上编辑Tracepoint

不再需要首先创建CDT跟踪点，然后编辑其属性。创建跟踪点时，可以设置启用，传递计数，条件等属性。

在创建Tracepoint之前，有不同的方法可以打开属性对话框。首先，通过右键单击编辑器装订线使用编辑器弹出菜单，必须将断点类型设置为“C / C ++ Tracepoints”。

  • 添加断点操作。
    1. 选择Add Breakpoint ...菜单项。
    2. 这将为预先填充跟踪点位置的线跟踪点打开跟踪点属性对话框。
  • 控制 - 双击
    1. 双击编辑装订线时按住Control键。
    2. 这将打开预填充断点位置的行断点的断点属性对话框。

步入选择

使用DSF调试器，现在可以选择要进入的功能。
所选方法可以与正在执行的方法不同
这样做的两个主要方法是

  • 使用上下文菜单

选择一种方法，右键单击并选择“Step Into Selection”
或使用快捷键Ctrl + F5

  • 使用超链接导航

按Ctrl-Alt，将鼠标悬停在某个功能上并单击鼠标

注意：第一个实现运行到选定的行，步进其方法并在输入的方法具有相同名称和所选方法的参数数量时停止，即此时未验证参数类型。

此功能已于2013年5月1日完成。有关详细信息，请参阅错误244865 。

Multicore Visualizer增强了选择和过滤功能

多核可视化器已得到增强，因此除了线程外，现在还可以选择CPU和内核。以前的选择方法仍然有效（按住Ctrl键单击“添加 - 选择”，左键单击并拖动等），但现在也适用于CPU和核心图形对象。

此功能于2013年4月26日完成。有关详细信息，请参阅错误404894 。


在增强选择的基础上，现在还可以过滤多核可视化器中显示的内容。这在有许多CPU /核心/线程，并且想要专注于子集的情况下尤其有用。要过滤视图，首先选择要保留的图形元素，然后从上下文菜单中选择“过滤到选择”。将显示所选的图形元素以及任何父对象;例如，一个线程将在其核心和CPU中显示。使用上下文菜单“清除过滤器”，过滤器保持原位直至清除。

此功能已于2013年5月6日完成。有关详细信息，请参阅Bug 405390 。

处理控制台中显示的退出代码

当进程正常完成执行时，其退出代码将显示在该进程的控制台的标题中。如果用户终止或断开进程，则不会显示退出代码。

请注意，如果没有进程控制台，则不能显示退出代码。这会影响远程调试和附加调试。我们希望在下一版CDT中解决这个问题。

此功能已于2013年5月6日完成。有关详细信息，请参阅错误402054 。
