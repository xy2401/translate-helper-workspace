使用“ 发现”选项卡 构建首选项面板以定义全局扫描程序发现选项

在“发现”选项卡中，您可以看到“ 语言设置提供程序”列表。语言设置提供程序为索引器和CDT的其他组件提供语言设置，例如包含路径和预处理器宏。

用户可以在此首选项上为全局（或“共享”）提供程序定义一些属性。请注意，提供程序可以是每个配置共享或定义的。可以在项目属性中定义非共享提供程序的属性：预处理程序包括路径，宏等。

内置编译器设置提供程序选项

内置编译器设置提供程序选项
提供商 选项
用于获取编译器规范的命令 使编译器打印内置设置的命令。以下变量被识别：
  • $ {COMMAND} - 来自工具链的编译器命令，例如“gcc”。
  • $ {FLAGS} - 工具链中适用的编译器标志，例如语言方言标志“-std = c ++ 0x”。
  • $ {INPUTS} - 通常表示编译器需要的空specs文件。如果不存在此类文件，则会在Eclipse工作区插件区域中临时创建该文件。
  • $ {EXT} - 语言的扩展，用于发现内置设置。提供程序可以与多种语言关联，它将为每种语言运行命令。
浏览 使用此按钮可在文件系统上找到编译器。结果放入“Command to get compiler specs”输入字段。
在控制台视图中分配控制台 启用以解决运行该命令的问题。启用后，提供程序的每次运行的输出将定向到控制台视图中的特殊控制台。另请参见扫描程序发现 - 控制台视图。

要查看提供程序发现的条目，请转到项目属性（或感兴趣的资源的属性），请参阅项目属性：预处理程序包含路径，宏等。

提示： 编译器升级后 ，使用相应提供程序的“清除条目”重新发现已更改的设置。
