软件站点位置向导

软件站点位置向导用于在目标定义中创建和编辑软件站点位置。要创建新的软件站点位置，请使用“ 位置”选项卡或“ 定义”页面中的“ 添加位置向导” 。

软件站点位置可以指向存储库或更新站点中的软件。该网站可以是一个远程网址，方便共享。要开始搜索软件，您必须选择一个站点。使用下拉列表选择特定站点或尝试使用“ 所有可用站点” 。您也可以按添加或单击链接以修改可用站点。

如果选择单个站点，则只能从该站点下载软件。此外，站点URL将存储在目标定义中，以便打开它的其他人可以使用它。

选择站点后，该站点提供的软件将显示在列表中。检查要包含在目标中的一个或多个项目。为了帮助您快速找到所需的软件，可以通过键入过滤器文本框来过滤列表，或将选项更改为按类别分组和仅显示最新版本 。如果在列表中选择条目，则详细信息部分将显示有关该软件的更多信息。按属性将显示有关选择的所有可用信息。

向导底部的Included Software选项会影响如何确定要添加到目标的插件集。默认情况下，包含所有必需的软件。这意味着我们可以找到所选软件所需的所有插件。所有这些插件必须在您选择的站点上可用。如果缺少任何一块，目标将无法解决并显示错误，说明缺少哪些要求。这是大多数用户的推荐设置，因为它可以确保您在目标中获得完整的插件集。在确定所需的插件时，将使用目标定义的环境设置，以便包含特定于平台的片段。

“ 包含所需的软件”设置将在站点中搜索所选项目所需的任何依赖项。建议您保持此选项。如果要构建不包含所有依赖项的目标，或者如果站点缺少某些必需的插件，则可以关闭此选项。关闭时，将添加站点中找到的任何必需插件，但不会报告丢失项目的错误。

如果关闭包含所需软件的选项，则“ 包括所有环境”选项将可用。打开它将导致将所有可用的特定于环境的插件添加到目标，而不仅仅是应用于目标环境设置的插件。

“ 包含源（如果可用）”选项将在软件站点中搜索您选择的内容的来源。

Include configure phase是一个高级选项，用于控制是否运行下载操作的配置阶段。配置阶段允许下载的软件运行其他操作。如果这会导致软件站点出现问题，则可以关闭此选项。
