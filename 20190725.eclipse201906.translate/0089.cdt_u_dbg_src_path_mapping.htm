编辑源查找路径映射

使用“源查找路径映射”编辑“路径映射”源容器类型。

源查找路径映射选项
选项 描述
名称 路径映射的名称。CDT在自动创建路径映射时仅使用名称“Found Mappings”，例如来自C Source Not Found Editor。
编译路径： 调制信息中的文件夹路径。这是编译源时的文件夹路径。

此路径必须与调试信息中的路径完全匹配。例如，如果使用正向（/）斜杠编译文件，则应输入正斜杠作为路径分隔符。类似地，如果编译路径中存在相对路径（例如“..”），则也应包含该路径。

使用“查找文件...”时，C源代码未找到编辑器会自动创建正确的表单。

本地文件系统路径： 该文件夹的本地路径，Eclipse可以在其中找到该文件。
使用调试器后端解析路径映射。 允许使用支持的调试器后端执行映射。这允许与后端的任何直接交互（例如通过控制台）具有一致的文件名视图。

此选项是默认选项，提供更完整的路径映射解决方案。

加 向表中添加新行。需要指定编译路径和本地文件系统路径。
去掉 删除当前选定的行。
向上 将当前选定的行在列表中向上移动。
下 将当前选定的行移动到列表中较低的位置。
