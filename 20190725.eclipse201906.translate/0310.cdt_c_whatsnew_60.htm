CDT 6.0 - 新的和值得注意的

注意： 以前版本的 “New and Noteworthy”位于此文件的底部。

请参阅CDT wiki上CDT 6.0中的新功能 ;可能包含其他信息。

编辑

在文件中重命名

 要在编辑器中以交互方式进行重命名重构，请将光标放在编辑器中的标识符上，然后按Ctrl-1（修补程序键）。选择“在文件中重命名”。同时更改标识符会更改对该标识符的所有其他引用。

重构

 通过编辑器上下文菜单添加“提取局部变量”。在编辑器中选择一个表达式，然后从上下文菜单中选择“重构”>“提取局部变量”。

为新的局部变量命名

从表达式创建局部变量，并将其用法插入到原始表达式中。

添加包含

 添加包含显着改善。
  • CDT可以从其使用中推断出函数或类的头文件，并自动插入适当的#include指令。例如，使用main方法和对printf的调用创建文件。右键单击对printf的调用，然后选择Source> Add Include。CDT将推断printf是在cstdio头中定义的，并自动为该头插入#include指令。
    • 此功能在CDT 6.0中运行得更好。

大纲视图
  • 非活动代码显示在“大纲视图”中。CDT解析器现在可以检测非活动代码块中的顶级声明，并在大纲视图中显示它们。非活动声明显示为略微灰色，图标上有斜线。

     

  • 在“大纲”视图中对方法定义进行分组。

    大纲视图现在能够以与显示类中定义的成员相同的方式显示在类外定义的类成员。

    可以在首选项中打开/关闭此功能。

     

     

宏观探索

 改进的宏观探索控制。
  • 宏扩展悬停现在在控件底部有一个工具栏，带有后退，前进和打开声明按钮。
  • Macro Expansion悬停的标题栏现在可以拖动。

块选择模式

 块选择模式现在可以在CDT编辑器中使用。

C风格块评论

 C样式块注释（Ctrl-Shift- /）改进了
索引

隐式引用和重载运算符

隐式引用和重载运算符的索引支持：
  • 索引器现在为重载运算符的使用选取引用。
  • 编辑器现在完全支持重载操作符，包括语义突出显示，标记出现和开放声明。

     

  • 已为重载运算符的语法突出显示选项添加了一个类别。

     

  • 现在可以搜索对重载运算符的引用。

     

  • 现在，重载运算符的使用将显示在调用层次结构视图中。

     

  • 索引器正在获取更多信息以支持此功能，因此索引时间可能会更长，索引文件的大小也会更大。因此，索引器选项页面提供了关闭隐式引用集合的选项。

     

系统包括
  • 拾取其他未解决的启发式包括来自项目的文件

    在旧版本的CDT中，用户被迫手动设置构建系统未发现的所有包含路径。现在，CDT能够自动查找项目中的任何头文件，无需手动设置。

    （见错误213562 ）

     

分度器准确度
  • 模板实例化的返工

    CDT编辑器对支持C ++模板进行了重大改进。内容辅助现在可以在大量使用模板的代码中更好地工作。

     

  • CDT解析器和索引器的性能和准确性已经有了很多改进。
导航和搜索

公开宣言

 开放式声明（F3）得到改进，特别是对于未解决的符号。

Open Declaration能够检测潜在的匹配。

现在可以从delete语句导航到隐式调用的析构函数方法。
项目和建设

转换器
  • 添加了转换器以将常规项目转换为托管生产项目
  • 修复了Makefile转换器

工作集
  • 完成对工作集配置的用户界面的大修：
    • 更新了“管理工作集配置”对话框，以简化工作流程
    • 通过工作集的属性页快速访问配置

       

    • Project Explorer中工作集上的上下文菜单操作，用于激活和构建配置

    • 有关更多详细信息，请参阅工作集设计文档

制作目标视图
  • 图标，菜单和对话框已得到增强。构建操作使用锤子图标与编辑器工具栏中的项目构建图标保持一致。

     

  • 在制作目标视图中拖放制作目标。可以复制/粘贴，将文件从Project Explorer拖动到视图，或将目标设置为外部编辑器，提供各种选项。

     

错误和问题管理

 ErrorParserManager已得到显着简化和改进。请参阅错误264715及其子任务。
  • 使用更精确的算法将构建输出中的错误/警告与工作空间中的文件相关联。
  • 在查找文件时，会考虑工作空间中的其他C / C ++项目。
  • 支持RSE EFS项目和EFS资源。
  • 改善了性能。

在“ 问题视图”中 ，外部文件位置（工作区外部）已移至“位置”列，并与“打开外部位置”菜单关联。

  • 远程项目支持
    • 新的ICommandLauncher接口支持覆盖构建命令的启动方式。
    • Managed Build现在支持EFS资源 。
  • XL C / C ++编译器支持
    • XL C / C ++错误分析器已经过现代化改造。它可以更好地处理编译器错误/警告，现在还可以处理xlC链接器警告。
    • 托管构建中的XL C / C ++ v10.1选项支持
    • 添加了XL UPC编译器支持。
  • 无头构建您现在可以在不启动IDE的情况下导入和构建CDT项目。
调试和启动

DSF集成

 DSF（Eclipse Debugger Services Framework）集成。有关更多信息，请参阅DSF功能 。

发射组

新的“Launch Group”发布配置。允许一次启动多个进程。
错误修复

错误修复

CDT 5.0 - 新的和值得注意的

CDT 5.0包括编辑器，调用层次结构，重构，索引器，项目和构建以及调试中的新功能。

有关包括屏幕截图在内的更多信息，请参阅CDT Wiki上CDT 5.0中的新功能 。

亮点包括：

编辑

  • 代码和文件模板 - 用户可定义的模板可用于New Class和New Source / Header File向导。
  • 代码模板的新首选项页面
  • 程序集文件的大纲视图
  • 代码格式化程序改进，包括新的空格和换行选项，改进的GNU编码样式合规性
  • 内容辅助改进
  • Doxygen编辑器支持 - 现在可以自动生成标签和其他文档工具的可插拔框架。
  • 标记出现次数 - 突出显示所选标识符在编辑器中的其他位置出现的位置
  • 折叠复合语句
  • 宏扩展悬停和探索工具
  • 拼写检查可用并默认启用
  • 用于处理非常大的文件的可伸缩性模式。
  • Visual Studio键绑定

导航和搜索

  • 打开操作员声明，空宏，大纲视图中的元素等
  • Open Element支持静态函数/变量，其他改进
  • 搜索 - 静态函数/变量，宏，局部变量的引用，搜索视图可用性改进

调用层次结构

  • 读/写装饰器指示变量的读/写状态
  • 多态方法调用（虚方法）支持

重构

  • 重构基础设施大大改进，AST上的模型转换
  • 生成Getters和Setter
  • 隐藏方法
  • 实施方法
  • 提取常数
  • 提取功能

索引

  • 提高了准确性和性能
  • 全新的预处理器
  • 改善基础设施
  • 索引器状态的可视指示

项目和建设

  • 项目属性提高了可用性
  • 创建目标位置 - 在项目构建目录中构建项目级别的目标

调试

  • 可执行文件视图
  • 事件断点 - 添加了gdb catchpoints支持
