处理相互冲突的更改

在某些情况下，两个用户正在编辑相同的文件，当第二个用户提交到存储库尝试提交更改时，存储库将不允许提交因冲突而成功。让我们通过让Fred和Jane更改相同的文件来模拟这一点。

  1. 在Fred的工作区中打开其中一个导航视图并编辑file1.txt 。使文本如下：
    Fred line 1
This is the contents
Fred-update
of file 1.
  2. Fred还将更改file2.txt并进行以下更改：
    File2 is a (Fred was here again)
with simple text.
  3. Fred将更改提交到存储库。
  4. 接下来，Jane正在对file1.txt进行更改。她在文件末尾添加了以下行：
    This is the contents
Fred-update
of file 1.
Jane was at the end
  5. 最后，Jane将file2.txt更改为以下内容：
    File2 is a (Jane was here) small file
with simple text.
  6. 当Jane完成更改后，她同步了项目并发现以下内容出现在Synchronize View中：
  7. file1.txt和file2.txt都显示一个红色图标，表示它们有相互冲突的更改。在解决冲突之前，您无法提交文件。单击file1.txt并注意Fred和Jane对文件的两个不同部分进行了更改。在这种情况下，Jane可以简单地更新文件，Fred添加的行将合并到Jane的本地文件中。选择file1.txt，然后从上下文菜单中选择Update 。
  8. 接下来，在file2.txt上加倍以查看冲突。在这种情况下，您可以看到Jane和Fred都改变了同一行。对于此类冲突更改，常规更新无法解决冲突。这里有三个选项（使用括号中的命令）：接受来自Fred（更改和更新 ）的更改，忽略Fred的更改（ 标记为合并 ），或手动合并比较编辑器中的文件。
  9. 对于此示例，假设Jane更新了file1.txt并为file2.txt选择了覆盖和更新。运行操作后，冲突将变为传出更改。Jane可以查看更改并提交它们。
