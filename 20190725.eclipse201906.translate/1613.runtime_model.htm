运行时插件模型

当用户启动使用Eclipse开发的应用程序时，将启动平台运行时引擎。运行时实现平台使用的基本插件模型和基础结构。它跟踪所有已安装的插件及其提供的功能。

插件是一种结构化组件，它为系统提供代码（或文档或两者），并以结构化方式对其进行描述。插件可以定义扩展点 ，定义明确的功能点，可以通过其他插件进行扩展。当插件为扩展点提供实现时，我们说它为平台添加了扩展 。这些扩展和扩展点在插件的manifest（ plugin.xml ）文件中声明。

使用通用扩展模型为插件提供了一种结构化的方式来描述它们的扩展方式，并为客户端插件描述它们提供的扩展。定义扩展点非常类似于定义任何其他API。唯一的区别是扩展点是使用XML而不是代码签名声明的。同样，客户端插件使用XML来描述其对系统的特定扩展。

运行时的一般目标是最终用户不应为已安装但未使用的插件支付内存或性能损失。平台扩展模型的声明性质允许运行时引擎确定插件提供的扩展点和扩展，而无需运行它。因此，可以安装许多插件，但是在根据用户的活动请求插件提供的功能之前，不会激活任何插件。这是提供可扩展，强大的平台的重要特征。
