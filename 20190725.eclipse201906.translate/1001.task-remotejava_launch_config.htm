使用远程Java应用程序启动配置

在调试在远程VM上运行的应用程序时，应使用远程Java应用程序启动配置。由于应用程序是在远程系统上启动的，因此启动配置不会指定有关JRE，程序参数或VM参数的常用信息。而是提供有关连接到应用程序的信息。

要创建远程Java应用程序启动配置，请执行以下操作：

  1. 选择从工作台菜单栏运行>调试配置... （或从调试工具栏按钮的下拉菜单中调试配置... ）以显示启动配置对话框。
  2. 在左侧的配置类型列表中选择Remote Java Application 。
  3. 单击“ 新建”工具栏按钮。将创建一个新的远程启动配置，并显示三个选项卡： Connect ， Source和Common 。
  4. 在“ 连接”选项卡的“ 项目”字段中，键入或浏览以选择要用作启动参考的项目（用于源查找）。不需要指定项目。
  5. 在连接选项卡的连接类型字段允许您选择您将如何连接到虚拟机。在大多数情况下，您将在特定位置附加到VM，在这种情况下，请选择标准（套接字连接） 。
    • 在“ 连接”选项卡的“ 主机”字段中，键入运行Java程序的主机的IP地址或域名。
      如果程序与工作台在同一台计算机上运行，请键入localhost 。
    • 在“ 连接”选项卡的“ 端口”字段中，键入远程VM接受连接的端口。通常，在启动远程VM时指定此端口。
    标准（套接字侦听）连接类型侦听来自远程VM的传入连接。它通常用于调试跨多个VM的应用程序，例如分布式系统。
    • 在“ 连接”选项卡的“ 端口”字段中，键入远程VM应连接到的本地计算机上的端口。通常，在启动远程VM时指定主机和此端口。
    • 在连接片的所述连接限制字段，输入预期的连接数，或0至支持的连接的数量不受限制。
  6. 允许终止远程VM标志是一个切换，用于确定调试器中是否启用了Terminate命令。如果希望能够终止要连接的VM，请选择此选项。
  7. 单击Debug 。启动尝试连接到指定地址和端口的VM，结果显示在Debug视图中。如果启动程序无法连接到指定地址的VM，则会显示错误消息。

应从VM提供程序获取有关设置远程VM的具体说明。
