JDT结构和API

鉴于JDT提供了一个功能齐全的Java IDE，为什么还需要使用JDT API？

如果要构建与Java程序或资源交互的插件作为其功能的一部分，则可能需要执行以下一项或多项操作：

  • 以编程方式操作Java资源，例如创建项目，生成Java源代码，执行构建或检测代码中的问题。
  • 以编程方式从平台启动Java程序
  • 提供新类型的VM启动程序以支持新的Java运行时系列
  • 向Java IDE本身添加新功能和扩展

如果要在插件中使用JDT API，则必须将JDT插件作为先决条件包含在plug-in.xml中。

  • org.eclipse.jdt.core是定义核心Java元素和API的插件。您应该始终列出此插件作为先决条件。
  • org.eclipse.jdt.ui是定义Java UI API的插件。如果您使用的是JDT UI类，则应将此插件列为先决条件。
  • org.eclipse.jdt.junit是定义JUnit集成API的插件。如果您使用的是JUnit集成类，则应将此插件列为先决条件。
  • org.eclipse.jdt.launching是定义Java启动和运行时支持的插件。如果需要以编程方式启动Java虚拟机，则应将此插件列为先决条件。
  • org.eclipse.jdt.debug是定义Java调试模型的插件。如果需要以编程方式访问正在调试的程序中的对象，则应将此插件列为先决条件。
  • org.eclipse.jdt.debug.ui是定义Java调试UI扩展的插件。如果需要以编程方式访问提示源定位器或Java启动配置选项卡，则应将此插件列为先决条件。
