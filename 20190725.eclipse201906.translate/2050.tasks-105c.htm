移动版本标签

警告：虽然许多人希望CVS版本能够及时冻结而且不可修改，但CVS中的版本和分支标签是可变的。因此，许多人确信修改版本是不好的做法，但有几种情况它实际上派上用场。话虽如此，在移动标签时请务必小心。

在单个文件上移动标记

假设您刚刚通过将项目版本化为R1来提交构建版本。但是你很快发现文件中有一个小的更改应该被创建并包含在构建中。您无需为项目重新版本，也可以移动修改后的文件的R1版本标记。

  1. 修改文件选择创建R1后修改的文件，然后从上下文菜单中选择“ 团队”>“在资源历史记录中显示” 。
  2. 在“资源历史记录”视图中，选择应使用R1版本标记的修订。
  3. 从上下文菜单中选择Tag with Existing ...。
  4. 从对话框中选择R1版本，然后按确定。
  5. 将更新资源历史记录视图以确认已移动版本。

从存储库视图中移动标记

许多项目在HEAD中使用定义良好的版本名称作为其当前稳定的阵容。例如，通过使用STABLE标记对HEAD进行版本控制，构建脚本可以简单地检查构建的STABLE版本。随着代码的发展，STABLE标签会定期移动以标记最新的稳定阵容。存储库视图提供移动现有标记的操作。

  1. 打开“存储库”视图并选择资源。
  2. 从上下文菜单中选择Tag with Existing
  3. 将出现一个标签选择对话框，您可以从中选择要移动的标签。如果要应用的标记未显示在列表中，则可以单击“ 从存储库刷新” 。如果找不到该标记，则可以单击“ 配置标记” ，打开一个对话框，允许您在存储库中的特定文件上搜索标记。
  4. 然后按确定 ，您的标签将被移动。该操作将现有标记移动到所选资源，并可用于移动版本和分支标记。
