字符串外部化

Java工具可帮助您开发可在国际平台上运行的应用程序。设计在不同国家使用的程序的一个重要方面是程序显示的文本的本地化或外部化。通过外部化字符串，可以在不重建Java程序的情况下翻译不同国家和语言的文本。

JDT为国际化和字符串外化提供以下支持：

  • 编译器选项允许您将非外部化字符串标记为编译时警告或错误。
    • 请参阅的代码样式部分中的非外部化字符串选项 Java>编译器>错误/警告首选项页面。
  • 允许您查找尚未外部化的字符串的工具。
  • 一个向导，将指导您外部化字符串。
  • 帮助您查找属性文件中字符串的未使用和错误使用的键的工具。

注释可用于表示不应外部化的字符串，不应导致编译时警告或错误。这些注释的形式为//$NON-NLS-n$ ，其中n是一行代码中字符串的从1开始的索引。

有关国际化应用程序的更多信息，请参阅以下文档：

  • http://eclipse.org/articles/Article-Internationalization/how2I18n.html
  • http://download.oracle.com/javase/tutorial/i18n/
