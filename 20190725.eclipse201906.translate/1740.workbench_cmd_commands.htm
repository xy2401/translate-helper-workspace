命令

命令是id的行为声明。命令用于声明语义行为，以便处理程序在其他地方定义动作实现。命令与行为实现的分离允许多个插件定义实现相同语义命令的实现。该命令与特定键绑定相关联。

工作台在其plugin.xml文件中定义了许多常用命令，并鼓励插件将它们自己的实现与这些命令相关联。以这种方式，在不同插件中实现的语义上类似的行为可以共享相同的密钥绑定。

定义命令

使用org.eclipse.ui.commands扩展点定义命令。以下来自Info示例标记：

<extension
	point="org.eclipse.ui.commands">
	...
      <command
            categoryId="org.eclipse.ui.examples.contributions.commands.category"
            id="org.eclipse.ui.examples.contributions.view.count"
            description="%contributions.view.count.desc"
            name="%contributions.view.count.name">
      </command>
	...

命令定义指定行为的名称 ， 描述和ID 。它还指定命令的类别的id，用于在首选项对话框中对命令进行分组。这些类别也在org.eclipse.ui.commands扩展点中定义：

      ...
      <category
            name="%contributions.commands.category.name"
            description="%contributions.commands.category.desc"
            id="org.eclipse.ui.examples.contributions.commands.category">
      </category>
      ...

请注意，没有为命令指定实现。只有当插件将其处理程序或操作与命令ID相关联时，命令才会变得具体。我们将讨论将处理程序实现与org.eclipse.ui.handlers部分中的命令相关联的不同方法。我们将讨论将键序列绑定到org.eclipse.ui.bindings部分中的命令。
