代码模板

该 “代码模板”首选项面板允许您配置新生成的代码存根，注释和文件的格式。

代码模板页面包含生成代码和/或源文件的操作使用的模板。例如，New Class向导使用源文件和头文件模板来创建新文件和代码模板以插入生成的方法存根。模板包含应用模板时替换的变量。某些变量在所有模板中都可用，有些变量特定于模板。

选项 描述
新...

打开“代码模板”对话框以创建新的文件模板。
编辑...

打开“代码模板”对话框以编辑当前选定的代码模板。
去掉

删除用户添加的文件模板。无法删除预定义的模板。
进口...

从文件系统导入模板。
出口...

将所有选定的模板导出到文件系统。
全部导出...

将所有模板导出到文件系统。

文件模板

不仅可以为C / C ++文件定义和修改文件模板，还可以为其他内容类型（如程序集文件或纯文本文件）定义和修改文件模板。“模板中的新文件”向导可用于根据此页面中定义的某个模板创建新文件。
模板名称 描述
默认C ++源模板

创建新的C ++源文件时，由New Class向导和New Source File向导使用。

模板可以指定添加注释的位置。请注意，模板可以包含变量$ {typecomment}和$ {filecomment} ，它们将由Types Types Files注释模板的评估替代。
默认C ++标头模板

创建新的C ++头文件时，由New Class向导和New Header File向导使用。

模板可以指定添加注释的位置。请注意，模板可以包含变量$ {typecomment}和$ {filecomment} ，它们将由Types Types Files注释模板的评估替代。
默认C源模板

创建新的C源文件时，由“新建源文件”向导使用。

模板可以指定添加注释的位置。请注意，模板可以包含变量$ {typecomment}和$ {filecomment} ，它们将由Types Types Files注释模板的评估替代。
默认C标头模板

创建新的C源文件时，由“新建头文件”向导使用。

模板可以指定添加注释的位置。请注意，模板可以包含变量$ {typecomment}和$ {filecomment} ，它们将由Types Types Files注释模板的评估替代。
其他文件模板

可以为其他内容类型定义文件模板，例如汇编或纯文本。
