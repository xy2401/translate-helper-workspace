跳到错误

CDT将解析make和编译器/链接器的输出。如果CDT可以确定错误或警告，则该行在控制台视图中突出显示并作为问题标记添加到问题标记中 或警告图标。信息性消息标有Info 图标。

要跳转到错误源，请执行以下操作之一：

  • 在“控制台”视图中，双击突出显示的行。

  • 在“问题”视图中，双击表中特定错误的行。

  • 如果在“问题”视图中将错误标识为“位置”列中的“ 外部位置” ，请使用鼠标右键单击它，然后单击上下文菜单中的“ 打开外部位置项”。

如果CDT可以确定文件和错误行并且该文件存在 - 该文件将在C / C ++编辑器中打开，并且光标将移至带有错误的行。

要按顺序跳转到错误：

  • 在编辑器中使用Next Annotation 或以前的注释工具栏上的图标。
  • 在Console视图中使用Next Error 或以前的错误纽扣。
