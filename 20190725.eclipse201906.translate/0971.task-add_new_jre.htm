添加新的JRE定义

在您的工作区中，您可以拥有任意数量的JRE定义。您还可以根据需要添加任意数量的JRE定义。

要添加新的JRE定义：

  1. 打开 Java>已安装的JRE首选项页面。
  2. 选择添加...按钮。将打开“添加JRE”向导。
  3. 您必须选择要创建的JRE类型：
    • 标准VM - 您将选择安装JRE的文件夹。JRE的详细信息将通过扫描位置来确定。您可以进一步自定义JRE安装，指定vm参数，源附件等。
    • 标准VM 1.xx - 与标准VM安装相同的选项，但安装是自定义的，以处理1.xx安装
    • 执行环境描述 - 您将选择一个执行环境（EE）文件，该文件描述了设置JRE所需的所有详细信息。您将能够自定义安装的某些方面。

标准VM：

  1. 在JRE主页字段中，单击“ 浏览”... 选择JRE安装的根目录的路径（通常是包含JRE的bin和lib目录的目录）。您也可以在此字段中键入文件夹名称。
  2. 在JRE名称字段中，键入新JRE定义的名称。所有相同类型的JRE必须具有唯一的名称。
  3. 在“ 默认VM参数”字段中，您可以添加/编辑将在启动时传递给VM的默认参数。
  4. JRE系统库列表中的JRE将显示默认库。您可以根据需要修改库。
    • 可以为引用的JAR指定Source和javadoc位置。Javadoc导出向导将Javadoc位置用作默认值，并通过“Open External Javadoc”操作使用。
    • 罐子可以添加，删除和重新排序。
  5. 完成后单击“ 完成” 。

标准1.1.x VM：

按照与标准VM JRE相同的步骤进行操作。

执行环境文件：

有关EE文件结构的更多信息，请参阅Wiki页面 。

  1. 在“ 定义主目录”字段中，单击“ 文件...” 选择执行环境描述文件以从中加载JRE安装。您也可以在此字段中键入文件路径。
  2. 在JRE名称字段中，键入新JRE定义的名称。所有相同类型的JRE必须具有唯一的名称。
  3. 在“ 默认VM参数”字段中，您可以添加/编辑将在启动时传递给VM的默认参数。
  4. JRE系统库列表中的JRE将显示默认库。您可以根据需要修改库。
    • 可以为引用的JAR指定Source和javadoc位置。Javadoc导出向导将Javadoc位置用作默认值，并通过“Open External Javadoc”操作使用。
    • 罐子可以添加，删除和重新排序。
  5. 完成后单击“ 完成” 。
