API使用报告转换Ant任务

目的

此任务将apitooling.apiuse Ant任务创建的XML报告转换为HTML文件。HTML报告将按照生成它们的包来组织引用（即包含被引用的api /类型的包）。

用法

描述

Ant任务的名称是： apitooling.apiuse_reportconversion 。要使用，jar文件apitooling-ant.jar必须位于Ant类路径上。 

<apitooling.apiuse_reportconversion xmlfiles = “...” htmlfiles = “...” xsltfile = “...” filterpatterns = “...” tofilterpatterns = “...” debug = “...” />

参数

属性 描述 需要
XMLFILES 此属性指定要转换的xml文件的位置。

使用绝对路径指定位置。 		是
htmlfiles 此属性指定要生成的html文件的位置。

使用绝对路径指定位置。 		是
xsltfile 此属性指定用于以所需顺序生成引用详细信息页面的XSLT文件的位置。

apitooling-ant.jar JAR文件中提供了一个默认的XSLT文件。

使用绝对路径指定位置。 		没有
filterpatterns 该属性指定用来从最终报告匹配名称模式修剪引用正则表达式模式以逗号分隔的列表。

该模式必须是如上所定义的格式良好的正则表达式这里 。 		没有
tofilterpatterns 该属性指定用来修剪引用从最终报告匹配名称模式正则表达式模式以逗号分隔的列表。

该模式必须是如上所定义的格式良好的正则表达式这里 。 		没有
调试 设置调试值。

可能的值为： true ， false
默认值为false 。 		没有

例子

 <apitooling.apiuse_reportconversion xmlfiles = “/ eclipse / apiuse-both / XML” htmlfiles = “/ eclipse / apiuse-both / HTML” xsltfile = “ /eclipse/references.xsl ” debug = “true” />

这将在文件夹/eclipse/apiuse-both/HTML运行创建*.html文件的任务，对应于XML文件位置的文件夹结构

如果启用了调试，则会在Ant控制台中显示某些调试跟踪。
