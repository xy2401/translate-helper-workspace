文件类型

存储库管理系统的存在可以规定文件的特殊处理需求。例如，某些文件应该从版本控制中省略。一些提供商对文本与二进制文件进行特殊处理。团队插件定义了扩展点，允许其他插件提供有关其文件类型的信息。在所有情况下，特殊处理最终都通过团队首选项页面留给用户。这些扩展允许插件使用适合插件的值来播种首选项。

忽略的文件

在某些情况下，可能没有必要将某些文件保留在存储库控制之下。例如，通常可以从存储库中省略从现有资源派生的资源。例如，编译的源文件（例如Java“.class”文件）可以省略，因为它们对应的源（“.java”）文件在存储库中。对于由存储库提供程序生成的版本控制元数据文件，它也可能是不合适的。org.eclipse.team.core.ignore扩展点允许提供程序声明应为存储库提供程序操作忽略的文件类型。例如，CVS客户端声明以下内容：

<extension point =“org.eclipse.team.core.ignore”> <ignore pattern =“。＃*”selected =“true”/> </ extension>

标记只是声明一个应该被忽略的文件名模式 ，以及一个在Preferences对话框中声明文件类型的默认选择值的selected属性。最终由用户决定应该忽略哪些文件。用户可以从默认的被忽略文件列表中选择，取消选择，添加或删除文件类型。

文本与二进制文件

一些存储库对文本与二进制文件实现不同的处理。org.eclipse.team.core.fileTypes扩展允许插件将文件类型声明为文本或二进制文件。例如，Java工具声明如下：

<extension point =“org.eclipse.team.core.fileTypes”> <fileTypes extension =“java”type =“text”/> <fileTypes extension =“classpath”type =“text”/> <fileTypes extension =“properties “type =”text“/> <fileTypes extension =”class“type =”binary“/> <fileTypes extension =”jar“type =”binary“/> <fileTypes extension =”zip“type =”binary“/> </分机>

该标记可以让插件通过扩展定义文件类型和分配类型的文本或二进制的。与忽略的文件一样，最终由用户来管理文本列表和二进制文件类型。
