扫描仪发现

一般

Scanner Discovery （有时称为Autodiscovery ）是一种查找包含路径和预定义预处理器宏的方法，编译器在编译代码时使用这些宏。

即使是只打印“Hello World”的最简单的C / C ++程序也需要包含系统头文件。考虑：

#include <stdio.h>
#include <stdlib.h>

int main(void) {
  puts("!!!Hello World!!!");
  return EXIT_SUCCESS;
}

索引器需要能够找到标头stdio.h和stdlib.h并解析它们以提供准确的索引。在此示例中，标头定义函数puts和宏EXIT_SUCCESS的原型。

独立调试器将尝试自动发现包含路径和预处理器符号。有两种主要方法可以发现这些：

  1. 内置设置。调试器将尝试检测内置编译器符号，并包含使用特殊选项运行编译器的路径，并解析此特殊运行的输出。大多数编译器都提供了打印内置包含路径和符号的选项。内置设置是隐含的，在常规编译期间不会传递给编译器。
  2. 构建输出分析器（BOP） 。调试器使用的另一种方法是分析您为其Build Output Parser提供的构建输出。通常，使用-I选项将包含路径提供给编译器，使用-D选项和BOP的宏将尝试在输出中查找这些路径。该方法依赖于构建的详细构建输出，其中所有这些选项实际上都是由make打印的。要指定构建输出，请使用-b命令行选项，或者在使用“ 文件”>“新建可执行文件...”对话框时指定构建日志的位置。

调试器使用语言设置提供程序来发现设置（从版本CDT 8.1开始）。通常，语言设置提供程序是专门的。一个将提供内置编译器设置和另一个由Build output Parser检测到的设置。

可以在“预处理器包含路径，宏等”的项目属性中检查Scanner Discovery找到的设置条目。页。可以展开每个语言设置提供程序以显示它发现的条目。

如果通过自动发现检索的信息不足，则用户可以在“用户设置条目”下手动输入其他包含路径和宏。

控制台视图

可以检查内置编译器规格检测器的输出以解决运行该命令的问题。为此，请启用“在控制台视图中分配控制台”复选框。

当启用“在控制台视图中分配控制台”复选框时，提供程序的每次运行的输出将被定向到控制台视图中的特殊控制台。
