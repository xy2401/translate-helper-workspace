开放资源

此对话框允许您浏览工作台以查找要在编辑器中打开的文件

  • 选择要打开的项目 ：在此字段中，键入要选择的文件名称的前几个字符。

    支持以下模式类型：


    通配符 ：
    • “*”表示任何字符串和“？“对于任何角色
    • 终止“<”或“”（空格）以防止自动前缀匹配，例如“M * file <”匹配Makefile和MockFile但不匹配MakeTheFiles
    骆驼案 ：
    • 包含“C”和“S”的文件名的“CS”为驼峰式表示法中的大写字母，例如CreateStuff.java
    • 包含“Cre”，“S”和“No”的文件名的“CreSNo”作为camel-case表示法中的部分，例如CreateStuffNow.jsp
    • 终止“<”或“”（空格）以修复驼峰部分的数量，例如“CreS <”匹配“CreateStuff.java”，但不匹配“CreateStuffNow.jsp”或“Createstuff.txt”。
    文件夹前缀 ：

    两种模式类型也支持文件夹前缀 。如果模式包含/，则最后一个/之前的部分用于匹配工作空间中的路径，例如“org.eclipse.ui / plugin.xml”。
    或者更复杂：“* / ui </pack*.html”匹配名为“ui”的文件夹中名为“package.html”的文件（但不在“uitools”或“* / ui / internal”等文件夹中）。


    相对路径 ：

    例如， ”。/ Mak“匹配活动编辑器或选择文件夹中以”Mak“开头的所有文件，而”..“/ Mak“匹配父文件夹中的此类文件。


  • 匹配项目 ：此列表显示您在“ 选择要打开的项目”字段中键入的模式的匹配项 。
    • 最近打开的文件显示在列表顶部的历史记录部分中。
    • 如果模式匹配许多具有相同名称的文件，则更靠近当前编辑或选定资源的文件将显示在匹配项列表的顶部。

“ 打开”按钮使用所选文件的默认编辑器打开文件。可以在文件的上下文菜单中或“ 打开方式”下拉按钮中选择不同的编辑器。

Show In下拉按钮可用于在视图中显示所选文件。
