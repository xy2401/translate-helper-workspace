ColumnLayout

UI Forms中的另一个自定义布局是RowLayout的变体。如果我们将RowLayout配置为垂直放置子项（在列中），并使列中的所有控件都相同，我们将获得多个列（取决于控件的宽度），但最后一列通常不会被完全填充（取决于控件的数量）。同样，如果放在一个表单中，我们会将所有控件放在一列中，因为 RowLayout不能进行“垂直”换行。如果我们使用GridLayout ，我们必须预先选择列数并与选择一起生活。

在更复杂形式的情况下，我们希望列数是自适应的。换句话说，我们希望数字根据表单的宽度而改变 - 尽可能使用更多，随着宽度减小而减少数字。我们还希望更加平等地填充表单区域（所有列的高度大致相同）。所有这些都可以通过ColumnLayout实现。

与TableWrapLayout相比， ColumnLayout更简单。几乎不需要任何配置。您需要做的唯一选择是您想要的列范围（默认值为1到3）。
