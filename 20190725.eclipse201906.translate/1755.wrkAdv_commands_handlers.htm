命令

命令是id的用户操作声明。命令用于声明语义动作，以便动作集和编辑器中定义的动作实现可以将自己与特定的语义命令相关联。命令与动作实现的分离允许多个插件定义实现相同语义命令的动作。该命令与特定键绑定相关联。

工作台在其plugin.xml文件中定义了许多常用命令，并鼓励插件将自己的操作与这些命令相关联。以这种方式，在不同插件中实现的语义上类似的动作可以共享相同的键绑定。

定义命令

使用org.eclipse.ui.commands扩展点定义命令。以下来自工作台标记：

<extension point =“org.eclipse.ui.commands”> ...<command name =“％command.save.name”description =“％command.save.description”categoryId =“org.eclipse.ui.category.file”id =“org.eclipse.ui.file.save”> < / command> ...

命令定义指定操作的名称 ， 描述和ID 。它还指定命令的类别的id，用于在首选项对话框中对命令进行分组。这些类别也在org.eclipse.ui.commands扩展点中定义：

...<category name =“％category.file.name”description =“％category.file.description”id =“org.eclipse.ui.category.file”> </ category> ...

请注意，没有为命令指定实现。只有当插件将其操作与命令ID相关联时，命令才会变得具体。
