创建Java应用程序启动配置

当您选择“运行”>“运行方式”>“Java应用程序”来启动您的类时，您将使用通用Java应用程序启动配置来运行您的类，该配置从Java项目和工作台首选项派生大多数启动参数。在某些情况下，您需要覆盖派生参数或指定其他参数。

您可以通过创建自己的Java Application启动配置来完成此操作。

  1. 选择 运行>运行配置...或从工作台菜单栏运行>调试配置....这将打开一个对话框，允许您创建，修改和删除不同类型的启动配置。
  2. 在启动配置类型的左侧列表中选择Java Application ，然后按工具栏中的“ 新建”按钮。这将为Java应用程序创建新的启动配置。右侧的选项卡允许您控制启动的特定方面。
  • Main选项卡定义要启动的类。在项目字段中输入包含要启动的类的项目的名称，并在Main类字段中输入主类的完全限定名称。如果希望程序在调试模式下启动时在main方法中停止，请选中Stop in main复选框。
    注意：您不必指定项目，但这样做允许选择默认的类路径，源查找路径和JRE。
  • Arguments选项卡定义要传递给应用程序和虚拟机（如果有）的参数。您还可以指定要启动的应用程序使用的工作目录。
  • JRE选项卡定义用于运行或调试应用程序的JRE。您可以从已定义的JRE中选择JRE，也可以定义新的JRE。
  • Classpath选项卡定义运行或调试应用程序时使用的类文件的位置。默认情况下，用户和引导类位置是从关联项目的构建路径派生的。您可以在此处覆盖这些设置。
  • “ 源”选项卡定义在调试Java应用程序时用于显示源的源文件的位置。默认情况下，这些设置是从关联项目的构建路径派生的。您可以在此处覆盖这些设置。
  • “ 环境”选项卡定义在运行或调试Java应用程序时要使用的环境变量值。默认情况下，环境是从Eclipse运行时继承的。您可以覆盖或附加到继承的环境。选项卡中指定的变量始终替换基础本机环境中的值。但是，当选择“将环境添加到本机环境”时，启动的环境将使用本机环境播种，之后选项卡中的变量将替换（现有变量）或扩充环境变量集。选择“使用指定环境替换本机环境”时，启动的环境仅包含选项卡中指定的变量。
  • Common选项卡定义有关启动配置的常规信息。您可以选择将启动配置存储在特定文件中，并指定在启动启动配置时哪些透视图变为活动状态。
