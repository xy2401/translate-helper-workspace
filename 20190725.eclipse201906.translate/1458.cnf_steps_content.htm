内容和动作绑定

以下部分定义了基于CNF的查看器的资源内容使用和自己内容定义的步骤集。最后，描述了Action Binding所需的步骤。如所描述的先前 ，为了呈现内容观众选择基于所选择的对象的表达式求值对应的化学实体。

资源内容的使用

使用Common Navigator的一个用例是操作工作区资源。与资源相关的内容由org.eclipse.ui.navigator.resources插件定义。可以在org.eclipse.ui.examples.navigator.resources找到CNF的资源使用示例。

  1. 执行创建Common Navigator Viewer中描述的一般步骤
  2. 将以下内容添加为从属插件：
    • org.eclipse.ui.navigator.resources
  3. 添加具有以下内容的org.eclipse.ui.navigator.viewer扩展：
    • viewerContentBinding ，将其指向视图的Id（example.view）
    • 包括：
      • org.eclipse.ui.navigator.resourceContent
      • org.eclipse.ui.navigator.resources。*
    <extension point =“org.eclipse.ui.navigator.viewer”> <viewerContentBinding viewerId =“example.view”> <includes> <contentExtension pattern =“org.eclipse.ui.navigator.resourceContent”/> <contentExtension pattern = “org.eclipse.ui.navigator.resources.filters。*”/> </ includes> </ viewerContentBinding> </ extension>

请注意，在您自己的RCP中使用CNF需要三个额外的步骤。

  1. 将org.eclipse.ui.ide添加为依赖插件
  2. 要将资源工作空间根目录（ IWorkspaceRoot ）作为默认页面输入，请覆盖WorkbenchAdvisor.getDefaultPageInput()方法：
    public IAdaptable getDefaultPageInput（）{return ResourcesPlugin.getWorkspace（）。getRoot（）; }
  3. 在使用工作台适配器之前正确挂钩它们，因此将此代码添加到WorkbenchAdvisor.initialize(IWorkbenchConfigurer)方法：
    public void initialize（IWorkbenchConfigurer configurer）{IDE.registerAdapters（）; }

自己的内容定义

除了org.eclipse.ui.navigator.resources插件提供的资源内容外，还可以在基于CNF的查看器中定义和呈现特定于应用程序的内容。为了定义和使用特定于应用程序，您需要：

  1. 使用扩展名org.eclipse.ui.navigator.navigatorContent并声明NCE，以及内容的提供者和触发条件：
    <extension point =“org.eclipse.ui.navigator.navigatorContent”> <navigatorContent id =“org.eclipse.ui.examples.navigator.propertiesContent”name =“Properties File Contents”contentProvider =“org.eclipse.ui.examples 。航海家。PropertiesContentProvider“labelProvider =”org.eclipse.ui.examples.navigator。PropertiesLabelProvider“activeByDefault =”true“icon =”icons / prop_ps.gif“priority =”normal“> <triggerPoints> <或> <和> <instanceof value =”org.eclipse.core.resources。IResource“/> <test forcePluginActivation =”true“property =”org.eclipse.core.resources.extension“value =”properties“/> </ and> <instanceof value =”org.eclipse.ui.examples.navigator。PropertiesTreeData“/> </或> </ triggerPoints> <possibleChildren> <或> <instanceof value =”org.eclipse.ui.examples.navigator。PropertiesTreeData“/> </或> </ possibleChildren> </ navigatorContent> </ extension>
    有关更多详细信息，请参阅CNF配置和操作主题 。
  2. 使用org.eclipse.ui.navigator.viewer扩展org.eclipse.ui.navigator.viewer内容绑定到查看器。
    <extension point =“org.eclipse.ui.navigator.viewer”> <viewerContentBinding viewerId =“example.view”> <includes> <contentExtension pattern =“org.eclipse.ui.examples.navigator.propertiesContent”/> </包括> </ viewerContentBinding> </ extension>
    请注意，不是使用内容的精确Id，而是在pattern属性中允许使用正则表达式，因此org.eclipse.ui.examples.navigator.*也可以使用。

动作绑定

查看器中操作的使用遵循与内容绑定相同的模式。对于org.eclipse.ui.navigator.resources插件中定义的资源的现有操作的使用：

  1. 将以下内容添加为从属插件：
    • org.eclipse.ui.navigator.resources
  2. 添加具有以下内容的org.eclipse.ui.navigator.viewer扩展：
    • actionContentBinding ，将其指向视图的Id（example.view）
    • 包括：
      • org.eclipse.ui.navigator.resources。*
    <extension point =“org.eclipse.ui.navigator.viewer”> <viewerActionBinding viewerId =“example.view”> <includes> <actionExtension pattern =“org.eclipse.ui.navigator.resources。*”/> </包括> </ viewerActionBinding> </ extension>
