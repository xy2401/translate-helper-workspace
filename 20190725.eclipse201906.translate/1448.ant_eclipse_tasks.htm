平台提供的Ant任务

该平台提供了一些与工作区交互的有用Ant任务和属性。它们可以与构建文件一起使用，这些构建文件设置为在与工作区相同的JRE中构建。

eclipse.refreshLocal

此任务是IResource.refreshLocal（）方法的包装器。例：

<eclipse.refreshLocal resource =“MyProject / MyFolder”depth =“infinite”/>
  • resource是相对于工作空间的资源路径
  • 深度可以是以下之一： 零 ， 一或无限

eclipse.incrementalBuild

提供project属性时，此任务是IProject.build（）的包装器。否则，此任务是方法的包装： IWorkspace.build（） 。

例子：

<eclipse.incrementalBuild />
<eclipse.incrementalBuild project =“MyProject”kind =“incremental”/>
  • project项目的名称
  • 善待执行的构建。可能的值包括：
    • “增量”执行增量构建
    • “full”放弃现有构建器状态并从头开始重建所有内容
    • “clean”以丢弃现有构建器状态但不重建
  • builder要运行的构建器的名称;如果未指定此值，则运行所有构建器

eclipse.convertPath

将文件系统路径转换为资源路径，反之亦然。结果值将分配给给定的属性。必须指定property属性，以及fileSystemPath或resourcePath属性。提供文件系统路径时，此任务是IWorkspaceRoot.getContainerForLocation（IPath）的包装器。提供资源路径时，此任务是IResource.getLocation（）的包装器。

例子：

<eclipse.convertPath fileSystemPath =“$ {basedir}”property =“myPath”/>
<eclipse.convertPath resourcePath =“MyProject / MyFile”property =“myPath”/>
