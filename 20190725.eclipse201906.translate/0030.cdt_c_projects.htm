C / C ++项目

通常，在使用CDT提供的插件之前，必须创建一个Eclipse项目来存储源代码，makefile，二进制文件和相关文件。但是，当您使用独立调试器时，会自动为您的可执行文件创建一个项目（如果只处理一个可执行文件，通常是Executables项目）。代表您创建的项目只包含可执行文件和指向调试信息中指定的源文件的链接。在大多数情况下，您不需要了解要调试的项目，但如果您希望查看所使用的各种文件并直接使用编辑器浏览它们，则可以在C / C ++ Projects视图中找到该项目。虽然允许您将文件导入C / C ++项目，但此操作通常用于将文件添加到可以构建的项目中。由于独立调试器不包含CDT（C / C ++ Developer Tools）构建功能，因此添加文件无助于调试，如果源中存在错误，则可能会使索引器混淆。
