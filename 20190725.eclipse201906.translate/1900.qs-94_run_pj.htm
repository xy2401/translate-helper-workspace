执行项目构建器

项目构建者的全部意义在于您无需明确告诉他们运行。相反，只要为拥有构建文件的项目进行合格构建，它们就会自动运行。请记住，构建器是按照“ 外部工具”对话框中“ 构建选项”选项卡上的指示触发的，可以是清理后，手动构建期间或自动构建期间的任意组合。让我们看看它是如何工作的。

  1. 在其中一个导航视图中选择HW项目。在工作台菜单栏中，选择“ 项目”>“清理”... 选择清除所选项目 ，然后单击确定
  2. 重建项目并运行projectBuilder.xml构建文件。请注意Console视图中此构建文件的输出。
  3. 确保打开Autobuild首选项，然后对HelloWorld.java进行一些微不足道的更改并保存更改。保存会触发自动构建，但自动构建不会触发项目构建器。
  4. 假设我们不希望每次运行时都看到构建文件输出。返回HW上项目属性对话框的External Tools Builders页面。选择Makejar条目，然后单击编辑....在Main选项卡上，取消选中Capture Output选项，应用更改并退回到工作台。

最后，我们将Ant构建文件视为Eclipse中的项目构建器。值得重复的是，尽管此示例使用的是Java项目，但项目构建器与Java无关，可用于任何类型的项目 。
