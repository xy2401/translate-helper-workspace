调试现有的可执行文件

使用导入功能导入和调试已构建的可执行文件。此功能将自动为您创建项目和调试配置。如果您使用调试符号构建可执行文件但可能没有用于构建可执行文件的项目，则此选项很有用。

导入可执行文件以进行调试

  1. 选择“ 文件”>“导入...”菜单项。
  2. 在Import wizard对话框中，单击C / C ++并选择C / C ++ Executable 。

  3. 单击下一步 。
  4. 在导入对话框中可执行文件的导入C / C ++可执行文件页面：
    • 从Select binary parser列表中选择一个解析器。
    • 在Select executable框中键入可执行文件路径，或单击Browse以找到可执行文件。
    • 键入或单击“ 浏览”以在“ 搜索目录”框中输入目录路径，然后从“ C / C ++可执行文件”列表中选择一个或多个可执行文件 。

    注意您可以导入任何可执行文件，但只有那些使用调试符号构建的可执行文件才能让您单步执行源代码并查看变量值。

  5. 单击下一步 。
  6. 在“ 导入可执行文件”对话框的“选择项目”页面中：
    • 要作为新项目导入，请激活“ 新建项目名称”并键入项目名称。
    • 要导入现有项目，请激活“ 现有项目” ，然后单击“ 搜索” 。
    • 要添加启动配置，请激活“ 创建启动配置”并从列表中选择配置，然后在“ 名称”框中键入名称 。

  7. 单击完成 。

    可执行文件将添加到“ C / C ++项目”视图中。

  8. 如果激活了“ 创建启动配置” ，则会显示“ 调试”对话框。

  9. 单击Debug 。

    调试器启动，您可以开始调试可执行文件。

设置断点

展开C ++ Projects视图中的可执行文件以查看源文件列表。只有使用调试符号构建的可执行文件才会显示源文件列表。可以在编辑器中打开源文件，以便设置断点。仅将这些文件用于调试，您无法更改它们并使用此项目重建可执行文件。
