OSGi框架启动器

PDE提供了一个OSGi Framework启动程序，它允许您启动OSGi框架并测试或调试您的OSGi包。与Eclipse中的所有其他启动程序 （例如Java Application和Java Applet启动程序 ......）一样，可以通过快捷方式调用OSGi Framework启动程序，并在启动配置对话框中集中管理其启动配置 。

请注意，在Eclipse中，单词plug-in和bundle可以互换使用。插件是一个捆绑包，捆绑包是一个插件 。

通过快捷方式启动

测试捆绑包的一种快速方法是从插件项目的上下文菜单中通过Run As> OSGi Framework启动OSGi Framework 。要以调试模式启动，请从插件项目的上下文菜单中选择Debug As> OSGi Framework 。

使用快捷方式时，PDE将搜索现有的OSGI Framework启动配置，并仅在未找到任何配置时创建新的启动配置。如果找到单个启动配置，则使用它。如果找到多个配置，PDE将显示一个包含所有现有OSGi Framework启动配置的选择对话框，并提示用户选择一个。如果创建了新的OSGi Framework启动配置，PDE将使用OSGi Frameworks首选项页面上指定的默认OSGi Framework 。

自定义启动配置

为了完全控制启动OSGi Framework ，您可以在“ 启动配置”对话框中创建和自定义启动配置 。

可以通过顶级菜单中的Run> Run ...或Debug> Debug ...调用Launch Configuration对话框 。通过双击左侧树查看器中的OSGi Framework节点，可以创建新的OSGi Framework启动配置。

OSGi Framework启动配置包含六个选项卡，允许您自定义测试启动的所有方面。有关详细信息，请参阅以下文档：

  • 捆绑标签
  • 参数选项卡
  • 设置标签
  • 跟踪选项卡
  • 环境标签
  • 常用标签
  • 原型标签
