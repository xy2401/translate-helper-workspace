搜索工作台

在本节中，您将在工作台中搜索Java元素。

在“搜索”对话框中，您可以执行文件，文本或Java搜索。Java搜索操作代码的结构。文件搜索按名称和/或文本内容对文件进行操作。Java搜索更快，因为代码结构有一个底层索引结构。文本搜索允许您在注释和字符串中查找匹配项。

从工作台执行Java搜索

  1. 在Java透视图中，单击“ 搜索” （ ）在工作台工具栏中的按钮或使用菜单栏中的搜索> Java ....
  2. 如果尚未选中，请选择“ Java搜索”选项卡。
  3. 在“ 搜索字符串”字段中，键入runTest 。在“ 搜索”区域中，选择“ 方法” ，然后在“ 限制到”区域中，选择“ 引用” 。
    验证Scope是否设置为Workspace 。

    然后单击搜索 。搜索时，您可以随时单击取消以停止搜索。将显示部分结果。
  4. 在Java透视图中，“搜索”视图显示搜索结果。

    使用显示下一个匹配项 （ ）并显示上一场比赛 （ ）按钮导航到每个匹配。如果找到匹配项的文件当前未打开，则会在编辑器中打开。
  5. 使用“搜索”视图按钮导航到搜索匹配时，文件将在编辑器中的匹配位置打开。搜索匹配在标尺中标记为搜索标记。

从Java视图中搜索

还可以从特定视图执行Java搜索，包括“大纲”，“类型层次结构”视图和“包资源管理器”视图。

  1. 在Package Explorer视图中，双击junit.framework。Assert.java在编辑器中打开它。
  2. 在Outline视图中，选择fail（String）方法，然后从其上下文菜单中选择References> Workspace 。

从编辑器中搜索

从Package Explorer视图中，打开junit.framework。TestCase.java 。在编辑器中，选择类名TestCase，然后从上下文菜单中选择References> Workspace 。

从搜索视图继续搜索

“搜索结果”视图显示TestCase搜索的结果。选择搜索结果并打开上下文菜单。您可以继续搜索所选元素的引用和声明。

如果要跟随多级方法调用，还可以使用“ 导航”>“打开调用层次结构” 。

执行文件搜索

  1. 在Java透视图中，单击工作台工具栏中的“ 搜索”按钮，或从菜单栏中选择“ 搜索”>“文件 ”。
  2. 如果尚未选中，请选择“ 文件搜索”选项卡。
  3. 在“ 包含”文本字段中，键入TestCase 。确保“ 文件名模式”字段设置为* .java 。范围应设置为Workspace 。然后单击搜索 。

  4. 要查找给定文件名模式的所有文件，请将“包含文本”字段留空。

查看以前的搜索结果

在“搜索结果”视图中，单击“ 上一个搜索结果”工具栏按钮旁边的箭头，以查看包含最新搜索列表的菜单。您可以从此菜单中选择项目以查看以前的搜索。可以通过选择“ 清除历史记录”清除列表。

“ 上一个搜索结果”按钮将显示一个对话框，其中包含当前会话中所有先前搜索的列表。

从此对话框中选择上一个搜索将允许您查看该搜索。
