定义项目构建设置

“ 构建器设置”页面允许您：

  • 选择是应该使用CDT生成的makefile还是自定义makefile。
  • 选择内部或外部构建器（仅适用于生成的makefile）。
  • 更改默认构建命令（对于外部构建器或自定义makefile）。
  • 配置CDT处理错误的方式。
  • 更改并行构建设置。
  • 选择构建或重建时，映射传递给make的目标。
  • 重新定义构建目录（仅适用于自定义makefile）

要定义构建设置：
在“C / C ++项目”视图中，右键单击项目，然后选择“ 属性” 。从列表中选择C / C ++ Build 。

  • 要在遇到错误时停止构建，请选择第一个构建错误时停止 。
  • 选择以下构建命令设置之一：
    • 要使用默认的make命令，请选中“ 使用默认值”复选框。
    • 要使用默认make命令“ 生成命令”框之外的生成实用程序，请清除“ 使用默认值”复选框。
  • 在Workbench Build Behavior框中，执行以下操作：
    • 要在保存资源时构建项目并更改默认的make build目标，请选中Build on resource save（Auto Build）复选框。在Make build target框中输入新的构建目标。
    • 要更改构建默认的make build target，请选中Build（Incremental Build）复选框。在Make build target框中输入新的构建目标。
    • 要更改重建默认make build target，请选中Rebuild（Full Build）复选框。在Make build target框中输入新的构建目标。
  • 单击确定 。
