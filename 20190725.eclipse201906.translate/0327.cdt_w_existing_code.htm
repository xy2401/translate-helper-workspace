将C / C ++源文件导入Eclipse

使用CVS Repository Exploring透视图，您可以将模块或目录签出到现有项目中，或创建新项目。开始使用CDT时可能遇到的一个常见情况是确定如何将现有的C / C ++源文件导入Eclipse。有很多方法可以做到这一点。下面描述的方案是推荐的方法。

从CVS中的源文件创建项目

如果您的现有源代码树是在CVS中管理的，则可以使用CVS Repository透视图来“Checkout As ...”存储库中的任何文件夹。第一次“Checkout As ...”时，将启动New Project向导，您需要为该文件夹创建C或C ++项目。有关更多信息，请参阅创建项目 ， 通过项目模板创建项目和使用C / C ++项目文件 。

执行结帐操作后，项目的CVS结帐进入项目的位置。建议您最终将CDT项目文件添加并提交回CVS。这些文件包括.project，.cdtproject和.cdtbuild（适用于托管生成项目），位于每个CDT项目的根文件夹中。

将代码从CVS引入IDE会略有不同，具体取决于您要导入的内容：

  • 现有的C / C ++项目
  • 现有的C / C ++代码，不属于项目的一部分
  • 需要添加到现有项目的现有C / C ++代码

从CVS导入C / C ++项目

要检查现有的C / C ++从CVS仓库项目到工作区中，右键单击CVS库视图中的项目，并从菜单中选择退房 。与CVS模块同名的项目将签出到您的工作区。

从CVS导入C / C ++代码

要查看不属于项目的现有C / C ++代码：

  1. 右键单击CVS Repositories视图中的模块或目录，然后选择Check Out As ...从菜单中。

    将显示“签出为”向导。

  2. 选择如何查看此项目：
    • 作为使用“新建项目”向导配置的项目

      要么：

    • 作为工作区中的新项目

      要么：

    • 作为标准制作C / C ++项目 - 如果您需要创建自己的Makefile以与现有构建过程集成

    选择此项目的工作空间位置，然后选择要签出的CVS标记。

  3. 单击“ 完成”退出“ 签出为”对话框。
  4. 单击“ 下一步”继续。

将C / C ++代码导入现有项目

要将充满C / C ++代码的目录导入现有项目：

  1. 右键单击“CVS存储库”视图中的模块或目录，然后从菜单中选择“ 签出为 ”。

    IDE将显示“签出为”对话框。

  2. 选择“ 签出到现有项目” ，然后单击“ 下一步” 。IDE将显示Check Out Into对话框：
  3. 从列表中选择现有项目，然后单击“ 完成”将CVS中的代码添加到所选项目中。

从现有源根创建新项目

如果您的资源代码不是在CVS中管理但可以从文件系统获得，那么您需要执行两个步骤：
  1. 确定源代码树的“根文件夹”。
  2. 使用“新建项目向导”创建新的C / C ++项目，并将“根文件夹”指定为新项目的非默认位置。

通常，现有项目将具有自己的makefile，因此您应该创建一个新的Make C / C ++项目。有关更多信息，请参阅创建项目和使用C / C ++项目文件 。

为帮助您识别项目的根文件夹，请考虑以下准则：

  • 项目的所有源代码都可以在根文件夹上或下面找到
  • 构建结果也在根文件夹中或下面生成
  • 根文件夹中经常有一个makefile。在复杂项目中，根文件夹中的makefile调用其他目录中的其他makefile以生成构建结果。
  • 外部头文件和库文件不需要位于根文件夹中或下面。

项目的资源维护在指定的远程位置，而不是Eclipse的工作区文件夹中。但是，现有的文件夹结构显示在“C / C ++项目”视图中。项目的元数据（例如项目索引和现有源的链接）存储在工作空间文件夹的元数据目录中。有关workspace文件夹的更多信息，请参阅Workbench用户指南>任务>升级Eclipse 。

创建CDT项目后，您无法轻松移动它或重新定义其根文件夹。如果需要，可以删除CDT项目（不删除其内容），然后重新创建它，指定不同的非默认位置。

导入C / C ++源文件系统

另一种方法是创建一个C / C ++项目，然后导入现有的文件系统。有关导入的详细信息，请参阅“ Workbench用户指南”>“任务”>“导入”>“从文件系统导入资源” 。

此方法将文件从文件系统复制到Eclipse Workbench项目或文件夹。原始源文件保持不变，它是将使用CDT编辑，构建和调试的文件的副本。成功导入现有文件系统后，文件夹结构将显示在“C / C ++项目”视图中。同样，您应该确定要从中导入的相应“根文件夹”。

小费：

  • 导入现有文件系统会占用大量磁盘空间，具体取决于文件的大小。
  • 您的文件可能会与先前引用原始文件位置的现有源控制系统（例如ClearCase视图）分离。
