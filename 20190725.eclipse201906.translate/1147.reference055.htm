定制

下表列出了persistence.xml编辑器的“定制”页面中可用的属性。

属性 描述 默认

织造

指定是否执行实体类的编织。EclipseLink JPA持久性提供程序使用编织来增强JPA实体，以实现延迟加载，更改跟踪，获取组和内部优化等属性。从以下选项中选择：

  • 没有编织

  • 动态编织

  • 静态编织 - 如果计划在不允许在JVM命令行上使用-javaagent:eclipselink.jar的环境中在Java EE 5容器外执行应用程序，请使用此选项。这假定类已经静态编织。在部署之前在类上运行静态weaver。

动态编织

懒

选择此选项可启用延迟编织。

真正

获取组

选择此选项可通过编织启用提取组。在以下情况下将此选项设置为false

  • 没有编织。

  • 在编织期间不应更改类（例如，在调试时）。

对于不受支持的平台，请将此属性设置为false。

真正

内部

指定Dali是否通过编织使用内部优化。

如果启用，则通过编织启用延迟的一对一和多对一映射。

真正

急于

指定Dali是否在渴望关系上使用间接。

假

变更跟踪

选择此选项可使用编织来检测对象的哪些字段或属性发生更改。

真正

仅验证

指定Dali是否应通过初始化描述符但不连接到数据源来验证部署。

真正

映射文件架构验证

假

抛出异常

选择此选项可将EclipseLink设置为在遇到persistence.xml文件<mapping-file>元素中列出的任何文件的问题时抛出异常或记录警告。

真正

异常处理程序

选择（或创建）Java类（实现org.eclipse.persistence.exceptions.ExceptionHandler接口）来处理异常。

会话定制器

选择会话定制程序类：实现eclipselink.tools.sessionconfiguration.的Java类eclipselink.tools.sessionconfiguration.SessionCustomizer接口并提供默认（零参数）构造函数。使用此类的customize方法，该方法需要一个eclipselink.sessions.Session ，以编程方式访问高级EclipseLink会话API。

探查

指定要使用的性能分析器以捕获运行时统计信息。

  • No Profiler - 不要使用性能分析器。

  • Performance Profiler - 使用EclipseLink性能分析器（ org.eclipse.persistence.tools.profiler.PerformanceProfiler class ）。

  • 查询监视器 - 监视查询执行和缓存命中（ org.eclipse.persistence.tools.profiler.QueryMonitor class ）。

NoProfiler
