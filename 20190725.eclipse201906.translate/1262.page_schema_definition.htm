架构定义页面

关于本页

此表单页面允许您编辑构成已编辑模式语法的模式对象的结构和属性。

架构对象的结构在“ 扩展点元素”部分中进行编辑。此部分占据编辑区域的左上角部分。

在<schema-object> Details部分中编辑模式对象的属性。这些部分在编辑器区域的右上角部分激活，具体取决于在“ 扩展点元素”部分中选择的模式元素的类型。

页面部分

此页面包含以下部分：

  • 扩展点元素部分
  • 说明部分
  • 元素细节部分
  • 合成器详细信息部分
  • 元素参考详细信息部分
  • 属性细节部分

扩展点元素部分

本部分允许您执行以下操作：

  • 可视化和编辑构成已编辑模式语法的模式对象的结构。
  • 激活用于编辑所选模式对象属性的相应<schema-object> Details部分。

可以通过“ 架构定义”表单页面直接访问此部分。

扩展点元素部分字段

使用扩展点元素查看器字段执行以下操作：

  • 可视化和编辑构成编辑模式语法的元素，属性和合成器的结构。
  • 激活用于编辑所选元素，属性或合成器属性的相应<schema-object> Details部分。

此查看器包含以下节点：

内容：

  • - 选择“ 元素”节点，以激活“ 元素详细信息”部分。
  • - 选择“ 元素引用”节点，以激活“ 元素引用详细信息”部分。

排字：

  • - 选择Sequence Compositor节点，以激活“ 合成器详细信息”部分。
  • - 选择Choice Compositor节点，以激活“ 合成器详细信息”部分。

属性：

  • - 选择“ 必需属性”节点，以激活“ 属性详细信息”部分。
  • - 选择“ 可选属性”节点，以激活“ 属性详细信息”部分。
  • - 选择“ 类属性”节点，以激活“ 属性详细信息”部分。
  • - 选择“ 资源属性”节点，以激活“ 属性详细信息”部分。

示例： extension 元素指定允许一个或多个快捷方式 元素的序列作为其合法子元素 。元素引用用于表示快捷方式 元素 。序列 合成器用于表示元素序列。

示例： 快捷方式 元素指定允许描述或类别元素作为其合法子元素。 元素引用用于表示描述和类别 元素 。Choice 合成器用于表示元素的选择。

扩展点元素部分操作

单击“ 添加元素”按钮，将新的全局元素声明添加到扩展点模式语法中。选择任何节点时，此操作可用。注意：元素声明顺序并不重要

单击“ 添加属性”按钮，将新的本地属性声明添加到扩展点模式语法中的选定元素。此操作可用于除所述延伸 元件节点的所有选定的元素节点。扩展 元素节点始终具有相同的属性： id ， name ， point 。注意：属性声明顺序并不重要。

要从扩展点模式语法中删除模式对象，请右键单击所选节点，然后选择“ 删除” 。除扩展 元素节点外，此操作可用于所有选定节点。对于非共享模式， 扩展 元素节点是用于定义编辑模式语法的入口点的特殊根元素。插件清单编辑器中定义的任何扩展都必须使用扩展父元素来指定其XML相关标记。

要指定元素的子元素由扩展点模式语法中的序列或其他元素的选择组成，请执行以下操作：

  1. 选择并右键单击要指定为父元素的元素节点。
  2. 选择New> Compositor> Sequence以定义元素序列。选择New> Compositor> Choice以定义元素选择。
  3. 选择并右键单击新的合成器节点。
  4. 从“ 新建”>“引用”子菜单中选择要指定为父元素的子元素的元素。Reference子菜单填充了扩展点模式语法中全局声明的所有元素。
  5. 重复步骤（3）和（4）以指定其他子元素。如果指定了序列合成器，则相当于指定序列中的其他元素。如果指定了Choice合成器，则相当于指定其他元素选择。

注意：要指定元素是另一个元素的子元素，父元素必须始终使用中间序列或选择合成器。合成器节点是连接器节点;而元素参考节点是叶节点。因此， 合成器节点应始终指定至少一个子元素引用节点。

注意： Sequence和Choice合成器可以嵌套任意次，以创建更复杂的语法。

注意：语法不得包含自引用循环。当元素A要求元素B作为子元素而元素B要求元素A作为子元素时，发生禁止周期的最基本形式。

要指定元素没有子元素并允许在其开始和结束XML标记（字符内容）中指定文本，只需将元素的结构保留为原样，方法是不向其添加属性，合成器和元素引用（元素内容）。注意：元素不能指定混合内容（字符内容和元素内容）。

说明部分

本部分允许您为编辑的模式语法中的任何元素，属性或合成器提供简要说明。

可以通过“ 架构定义”表单页面直接访问此部分。

描述部分字段

编辑描述文本字段以提供在“ 扩展点元素”部分中选择的当前元素 ， 属性或合成器节点的简要描述。根据需要，可以在此字段中使用有效的HTML标记。

注意：更新元素引用描述也会更新它引用的全局元素声明的描述，反之亦然。

元素细节部分

本部分允许您编辑元素的属性。有两种元素：扩展元素，非扩展元素。扩展元素是特殊的根元素声明，用于定义编辑的模式语法的入口点。非扩展元素是一个全局元素声明，用作编辑的模式语法中的基本结构构建块。

通过在“ 扩展点元素”部分中选择扩展 元素节点或非扩展元素节点来激活此部分。

“ 扩展元素的元素详细信息”部分。

非扩展元素的“ 元素详细信息”部分。

元素详细信息部分字段

编辑“ 名称”字段以指定元素名称。

单击“ 内部”字段中的“ 真”按钮或“ 假”按钮以指示元素是否为内部。Plug-in Manifest Editor将标记内部带有警告的任何扩展XML标记（默认情况下）。此字段类似于导出包上的x-internal属性。

单击Deprecated字段中的true按钮或false按钮以指示该元素是否已弃用。Plug-in Manifest Editor将标记任何不推荐使用的扩展XML标记（默认情况下）。该字段类似于Java的Javadoc @deprecated标记。

如果将扩展元素标记为已弃用，请编辑“ 替换”字段以指定用户应在其位置使用的不推荐使用的扩展点，以实现相同的功能。

从Label Property字段中选择元素定义的属性之一，以指定要在Plug-in Manifest Editor中用作元素标签的属性值。例如，如果选择了某个值，则Plug-in Manifest Editor中 Extensions页面的All Extensions部分中的元素节点标签将如下所示： selected-attribute-value （ element-name ）。否则，如果未选择任何值，则元素的节点标签将如下所示：（ element-name ）。

从Icon字段中选择元素定义的资源类型属性之一，以指定用于在Plug-in Manifest Editor中表示元素节点的图标资源。为了使元素的节点图标能够在Plug-in Manifest Editor的Extensions页面的All Extensions部分中正确显示 ，必须满足以下条件：

  • 元素必须定义资源类型属性。
  • 必须选择该属性作为此字段的值。
  • 提供属性值的贡献XML标记必须指向有效的图标资源文件。

单击“可翻译”字段中的“ 真”按钮或“ 假”按钮，以指示元素的字符内容是否可翻译。此字段仅适用于没有指定子元素（元素内容）或属性的元素，并允许在其开始和结束XML标记（字符内容）中指定文本。如果元素的字符内容被标记为可翻译，则PDE工具可用于提供的扩展XML标记，以将内容外部化为属性文件以支持多语言。

使用只读DTD近似字段来查看元素模式语法与等效元素DTD语法的近似一对一映射。DTD（文档类型定义）是XML Schema的一种易于阅读，功能较弱的替代方法，用于指定XML文档的正式语法。以下表示法用于指定DTD近似值：

  • element-name - 表示已定义语法中的元素。
  • EMPTY - 表示不允许任何元素内容或字符内容（仅属性声明）。
  • #PCDATA - 表示只允许使用字符内容（Parsed Character Data）。
  • （） - 用于对元素的已定义序列和选择合成器进行分组。
  • | - 用于分隔元素选择。代表逻辑或分离。
  • ， - 用于分隔元素序列。表示逻辑和连接。
  • + - 允许用于指示一个或多个实例的限定符。
  • * - 允许用于指示零个或多个实例的限定符。

合成器详细信息部分

本部分允许您编辑合成器的属性。有两种类型的合成器：序列合成器，选择合成器。序列合成器允许您指定元素的子元素由编辑的模式语法中的一系列其他元素组成。选择合成器允许您指定元素的子元素由编辑的模式语法中的其他元素组成。

通过在“ 扩展点元素”部分中选择“ 序列”或“ 选择”节点来激活此部分。

合成器详细信息部分字段

编辑Min Occurrences字段以指定允许其他元素和/或合成器的合成器出现在提供的扩展XML标记中的最小出现次数 。PDE使用此字段的值来验证Plug-in Manifest Editor中的多重性约束。如果元素A只需要一个元素B或元素C作为其子元素 ，并且它使用选择合成器指定2个最小出现次数，那么元素B与元素C的任何2个实例必须在贡献的扩展XML标记中按顺序定义有效。

编辑Max Occurrences字段以指定允许其他元素和/或合成器的合成器出现在提供的扩展XML标记中的最大出现次数。PDE使用此字段的值来验证Plug-in Manifest Editor中的多重性约束。如果元素A仅需要一个元素B或元素C作为其子元素 ，并且它使用选择合成器指定2个最大出现次数，那么可以在贡献的扩展XML标记中定义元素B与元素C的组合不超过2个实例保持有效。

检查Unbounded字段以指定此合成器可能在提供的扩展XML标记中出现的最大允许出现次数没有约束。

从“ 类型”字段中选择一个值，以指定此合成器是序列还是选择 。

使用只读DTD近似字段查看合成器模式语法与等效合成器DTD语法的近似一对一映射。用于指定DTD近似的表示法在本文档的“ 元素详细信息部分字段”标题下进行了描述。

元素参考详细信息部分

本部分允许您编辑元素引用的属性。元素引用是对已编辑的模式语法中的全局元素声明的引用。元素引用由合成器用作原子构建块，以定义全局元素声明的语法。

通过在“ 扩展点元素”部分中选择元素引用节点来激活此部分。

元素参考详细信息部分字段

编辑“ 最小次数”字段以指定允许此元素出现在提供的扩展XML标记中的最小出现次数 。PDE使用此字段的值来验证Plug-in Manifest Editor中的多重性约束。如果元素A指定了2个最小出现次数，则必须在提供的扩展XML标记中定义2个元素A实例才能生效。

编辑Max Occurrences字段以指定允许此元素出现在提供的扩展XML标记中的最大出现次数 。PDE使用此字段的值来验证Plug-in Manifest Editor中的多重性约束。如果元素A指定了2个最大出现次数，则可以在提供的扩展XML标记中定义不超过2个元素A的实例以保持有效。

检查Unbounded字段以指定此元素在提供的扩展XML标记中可能出现的最大允许出现次数没有约束。

使用只读DTD近似字段来查看元素模式语法与等效元素DTD语法的近似一对一映射。用于指定DTD近似的表示法在本文档的“ 元素详细信息部分字段”标题下进行了描述。

元素参考详细信息部分操作

单击“ 引用”字段中的元素名称超链接以编辑此本地元素引用引用的全局元素声明的属性。完成此操作后，将在“ 扩展点元素”部分中选择引用的元素节点，并激活相应的“ 元素详细信息”部分。

属性细节部分

本部分允许您编辑属性的属性。属性用于提供有关元素的其他信息。

通过在“ 扩展点元素”部分中选择必需属性 ， 可选属性 ， 类属性或资源属性节点来激活此部分。

“ 属性详细信息”部分，用于指定类型字符串的必需属性，用于指定枚举限制。

“ 属性详细信息”部分，用于指定类型为boolean的可选属性，用于指定默认值。

类属性的“ 属性详细信息”部分，指示由属性值表示的类必须扩展指定的类和/或实现指定的接口。

资源属性的“ 属性详细信息”部分。

使用架构标识符属性

扩展点作者可以定义引用其他属性的属性。这可以通过将扩展点属性设置为类型标识符并选择它引用的属性来实现。这方面的一个例子是org.eclipse.ui.handlers扩展点。commandId属性引用org.eclipse.ui.commands扩展点中定义的命令。

然后，贡献处理程序的扩展开发人员可以从commandId属性的所有有效标识符中进行选择。将为不存在的标识符发出警告，以防止拼写错误的标识符的常见错误。

属性详细信息部分字段

编辑“ 名称”字段以指定属性名称。

单击“ 不推荐使用”字段中的“ 真”按钮或“ 假”按钮，以指示该属性是否已弃用。Plug-in Manifest Editor将标记任何不推荐使用的扩展XML标记（默认情况下）。该字段类似于Java的Javadoc @deprecated标记。

从“ 使用”字段中选择一个值，以指示是否必须在其提供的扩展XML标记内的定义元素上指定该属性。可以使用以下选项：

  • optional - 可以在其定义元素上选择性地指定属性。
  • required - 必须在其定义元素上指定该属性。
  • default - 可以在其定义元素上可选地指定属性。如果未指定该属性，则其值将采用定义的默认值。

编辑“ 默认值”字段以指定在属性未在其提供的扩展XML标记中的定义元素上指定时用于属性的默认值。仅当选择“ 使用”字段中的默认值时，此字段才适用。

从“ 类型”字段中选择一个值以指定属性值类型。可以使用以下选项：

  • string - 属性值可以是任何有效的字符串。
  • boolean - 属性值可以是true或false （不区分大小写）。
  • java - 属性值可以是目标平台类路径上的任何完全限定类名（例如java.lang。串）。
  • resource - 属性值可以是现有文件或文件夹资源的任何项目相对路径。

单击“可翻译”字段中的“ 真”按钮或“ 假”按钮，以指示属性的值是否可翻译。仅当选择“ 类型”字段中的字符串值时，此字段才适用。如果属性的值被标记为可翻译，则可以在提供的扩展XML标记上使用PDE工具，以将这些值外部化为属性文件以支持多语言。

使用“ 限制”字段可以查看属性限制为使用的字符串值列表作为自己的值。如果该字段不包含任何值，则对属性值没有限制。仅当选择“ 类型”字段中的字符串值时，此字段才适用。注意：在编辑的模式的语法中使用字符串基类型的枚举限制来完成此功能。

编辑Extends字段以在目标平台的类路径上指定完全限定的类名（例如org.eclipse.ui.part。EditorPart）必须由指定为属性值的类进行子类化。PDE将验证提供的扩展XML标记中的类属性值，以确保它扩展超类。仅当选择“ 类型”字段中的java值时，此字段才适用。

编辑Implements字段以在目标平台的类路径上指定完全限定的接口名称（例如org.eclipse.ui。IEditorPart）必须由指定为属性值的类实现。PDE将验证提供的扩展XML标记中的类属性值，以确保它实现接口。仅当选择“ 类型”字段中的java值时，此字段才适用。

属性详细信息部分操作

单击“ 限制”字段中的“ 添加...”按钮以打开“ 新建限制”对话框，以便为属性值添加新限制。对话框完成后，新限制将添加到字符串值列表中，该属性限制为使用其作为自己的值。仅当选择“ 类型”字段中的字符串值时，此操作才可用。

在“ 限制”字段中选择一个字符串值，然后单击“ 删除”按钮以从属性限制为使用的字符串值列表中删除限制作为其自己的值。仅当选择“ 类型”字段中的字符串值时，此操作才可用。

如果Extends字段中指定的值引用目标平台类路径上的完全限定类名，则单击Extends超链接将在新的Java源编辑器中打开该类。否则，单击Extends超链接将打开New Java Class向导，允许您创建新类。仅当选择“ 类型”字段中的java值时，此操作才可用。

如果Implements字段中指定的值引用目标平台的类路径上的完全限定的接口名称，则单击Implements超链接将在新的Java源编辑器中打开该接口。否则，单击Implements超链接将打开New Java Class向导，允许您创建新接口。仅当选择“ 类型”字段中的java值时，此操作才可用。

单击“ 浏览...”按钮以打开“ 选择类型”对话框，以便在目标平台的类路径中选择现有类，以用作“ 扩展”字段的值。仅当选择“ 类型”字段中的java值时，此操作才可用。

单击“ 浏览...”按钮以打开“ 选择类型”对话框，以选择目标平台的类路径上的现有接口，以用作“ 实现”字段的值。仅当选择“ 类型”字段中的java值时，此操作才可用。
