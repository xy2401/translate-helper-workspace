CVS工作区同步

使用Team> Synchronize菜单命令或Team Perspective中的Synchronize工具栏命令启动的CVS工作区同步将显示在Synchronize视图中。此视图允许您检查本地Workbench资源与其远程对应项之间的差异，以及更新Workbench中的资源并将资源从Workbench提交到存储库。

以下是Synchronize视图中的CVS工作区同步：

特征

以下是同步视图的一些功能的简要概述。

同步状态

同步视图显示工作区中资源的同步状态与存储库中的资源的同步状态。此状态通过使用图标显示，也可以配置为将状态显示为附加到资源名称的文本。图标说明如下表所示：

传入添加意味着资源已添加到存储库。更新会将资源传输到您的工作区。
传入更改意味着文件已在存储库中更改。 更新会将新文件修订版传输到您的工作区。
传入删除意味着资源已从服务器中删除。 更新将删除您的本地资源。
传出添加意味着该文件已添加到您的工作区，但尚未存储在存储库中。添加然后提交将新文件传输到存储库。
传出更改意味着文件在本地更改。提交文件会将更改传输到存储库并创建文件的新修订。
传出删除是本地删除的资源。提交这些资源将导致删除远程资源。 注意 ：在CVS目录中永远不会从存储库中真正删除。而是删除文件，并从工作区中删除空目录。
冲突的添加意味着资源已在本地和远程添加。
冲突的更改意味着文件已在本地和远程更改。需要手动或自动合并。此外，视图中包含冲突子项的任何条目也将使用冲突图标进行修饰。这样做是为了使冲突易于查找。
冲突删除意味着资源已在本地和远程删除。

模式

可以使用工具栏操作或视图下拉菜单中的菜单项使用模式过滤“同步”视图。模式可用于仅显示传入，传出或冲突的更改。模式的优点是：

  • 他们支持并鼓励理想的工作流程。
  • 它们可以减少您在任何时候遇到的信息量。
  • 它们提供一定程度的安全性，限制您适合该模式的操作。例如，在传入模式下，您不会意外地提交冲突。

要点 ：最好先更新Workbench中的资源，通过合并解决存在的任何冲突，然后将Workbench资源提交到存储库。

楷模

“同步”视图能够显示资源的不同模型表示。每种产品都可能包含自己的产品特定表示。Eclipse SDK附带三个模型：

  • 工作区模型 ：显示基于资源的模型。可以从下拉菜单中的“首选项”对话框控制此模型的布局选项。Workspace模型的布局选项包括：
    • 平面布局将所有不同步资源显示为其项目的直接子项。
    • 树布局显示项目资源管理器中显示的资源层次结构。
    • “压缩文件夹”显示按项目分组的更改，然后按文件夹分组。这导致层次结构最多为三个级别，文件夹路径被压缩为单个级别（类似于Java包）。
  • Java Model ：显示基于Java的模型（类似于Package Explorer中显示的模型）。
  • 变更集 ：显示基于变更集的模型。此模型显示按提交注释分组的传入更改，这对于查看谁发布了什么以及原因很方便。对于传出更改，您可以创建更改集以将相关更改分组在一起。使用上下文菜单中的各种命令执行分组。

除了模型之外，还有一个Flat Presentation ，它将所有不同步元素显示为顶级元素。

导航

“同步”视图提供工具栏操作，用于浏览视图中的更改。这些操作不仅可以在文件之间导航，还可以在文件中从更改变为更改。

更新和提交操作

“同步”视图中提供了多种更新和提交操作。您可以对所有可见的适用更改或选定的子集执行标准更新和提交操作。您还可以选择覆盖和更新，从而忽略任何本地更改，或覆盖和提交，从而使远程资源与本地资源的内容匹配。您还可以选择清除本地修改过的文件的时间戳（可能是外部构建工具），但其内容与服务器的内容相匹配。

冲突处理

处理冲突时，您可以先执行更新和任何冲突的更改更新操作将正确更新可自动合并的冲突（即文件内容更改不重叠），但会跳过包含重叠更改的文件。或者，可以使用比较编辑器处理冲突。可以通过在冲突上双击（或单击，如果您在首选项中更改了打开策略）单击或通过从上下文菜单中选择“ 在比较编辑器中打开”来打开比较编辑器 。Compare编辑器允许您手动解决文件中的冲突。完成后，对冲突执行标记为合并以表明您已完成。这将把冲突变成一个外向的变化。

问题标记

“同步”视图将在视图中显示的任何更改上显示错误或警告问题标记，或者在包含具有此类问题标记的资源的视图中显示的任何文件夹或项目中。这样做是为了防止有问题的资源被提交到存储库。

工具栏

同步

此命令允许您重复当前同步，该同步将重新获取资源的远程状态。工具栏项的下拉菜单允许您使用Synchronize ...菜单命令选择其他现有同步或创建新同步 。

引脚电流同步

此命令允许您固定或取消固定当前同步。固定同步不会被同一类型的下一次同步替换，而取消固定同步将被替换。这允许您保持同步以便于访问。当对Workbench资源进行更改时，此类同步将自动更新，并且可以配置为执行计划刷新，以便与资源的远程状态保持同步。

型号下降

此下拉列表显示可以在“同步视图”中显示的模型。

下一个差异

此命令将转到视图显示的下一个差异。在打开视图中的下一个文件之前，将访问单个文件中的所有差异。

以前的差异

此命令将转到视图显示的上一个差异。在打开视图中的上一个文件之前，将访问单个文件中的所有差异。

全部收缩

折叠视图中的所有展开条目。

传入模式

在此模式下，仅显示自上次加载或与Workbench中的那些资源（传入更改）同步后在存储库中更改的资源。

传出模式

在此模式下，只有在Workbench中已修改的资源（传出更改）才可见。

传入/传出模式

在此模式下，将显示传入和传出更改，您可以更新和提交。使用此双模式的优点是您可以根据需要执行任一任务。缺点是执行同时双向合并通常很复杂。

冲突模式

在此模式下，仅显示冲突（在Workbench和存储库中都已修改的资源）。

更新所有传入的更改

此命令更新视图中可见的所有传入更改。该命令还将更新可自动合并的冲突，但会跳过内容包含冲突的文件。

承诺所有传出的变化

此命令提交视图中可见的所有传出更改。提交中不包含冲突。

下拉式菜单

下拉菜单允许您固定或删除当前或删除所有同步，更改模式或布局，从视图项中恢复，配置当前同步以执行计划刷新以及设置其他视图首选项。

上下文菜单

从Synchronize视图的上下文菜单中，您可以执行许多有趣的操作。

打开

此命令在文件类型的默认Workbench编辑器中打开所选更改的本地资源。

打开用

此命令允许您在Workbench编辑器中打开所选更改的本地资源。

显示

此子菜单用于在另一个视图中显示当前选定的资源。

在比较编辑器中打开

此命令在比较编辑器中打开所选更改，允许您检查文件中的更改。

编辑

此菜单允许重命名，删除，移动或复制资源。

同步

此命令刷新所选资源的远程状态。此外，此命令将调整其内容已与服务器上相应资源的内容匹配的传出更改的时间戳，以使该文件不再是传出更改。这在构建工具重新生成内容未更改的文件的情况下非常有用。

展开全部

此命令扩展所有选定资源的子项。

从视图中删除

此命令从视图中删除所选资源。只有在已删除资源的状态发生更改或重新启动Workbench或从“下拉”菜单调用“ 还原已删除项目”操作时，才会重新显示资源。

更新

此命令更新所选资源。冲突可以包含在更新中，但只会更新自动合并冲突。将跳过不可合并的冲突，并且应使用比较编辑器手动合并。

承诺

此命令提交所选资源。只能提交传出更改。

覆盖和更新

此命令对冲突和传出更改进行操作，并使用服务器中的内容替换这些资源的本地内容。

标记为合并

此命令调整冲突更改的CVS时间戳，以便它们成为传出更改。在使用比较编辑器手动合并冲突后，应执行此命令。

添加到.cvsignore

此命令将所选资源添加到.cvsignore列表中，这将导致CVS在所有CVS操作期间跳过资源。

创建补丁

此命令将打开Create Patch向导并使用所选资源填充它。

科...

此命令将创建一个新分支和一个新分支标记，可用于将分支合并回源分支。

显示注释

根据您的注释首选项，此命令将启动Quick Diff annotate或打开所选资源的Annotate视图。

更改ASCII /二进制属性...

此命令将打开一个包含所有选定资源的对话框。然后，您可以更改每个资源的ASCII /二进制属性。

将更改重新分配给

此命令将更改移动到从子菜单中选择的更改集。

编辑更改集...

此命令将打开一个对话框，允许创建或编辑所选更改集的名称和注释。

更改未分配

此命令从选定的更改集中删除更改。如果删除所有更改，则删除更改集。

标记更改设置为默认值

此命令将选定的更改集标记/取消标记为默认值。

不要使用默认更改集

此命令将从当前标记为默认值的更改集中删除默认标志。
