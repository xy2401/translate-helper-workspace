帮助系统定制

可以通过为帮助首选项的数量指定自定义默认值来配置和标记Eclipse帮助系统以适合您的产品。

帮助系统本身分为许多单独的插件。这些表显示了可用的首选项，以及哪个插件定义了它们。

org.eclipse.help插件：

首选键 用法
 默认
HELP_DATA 指定包含帮助数据的XML文件的路径，例如应显示目录的顺序，以及应从用户隐藏哪些书籍和索引贡献。路径可以是PLUGINS_ROOT / pluginId / path或“path”的形式。如果该值不以PLUGINS_ROOT开头，则相对于产品插件解析路径。例如，“ helpData.xml ”或“PLUGINS_ROOT / org.eclipse.platform / helpData.xml”。有关详细信息，请参阅架构文档
filterInfocenter 指定启用过滤器是否应该应用于信息中心。始终应用启用过滤器以帮助工作台和独立模式。 假
baseTOCS （不建议使用;请改用HELP_DATA ） Toc订购。有序的帮助TOC（书籍）或TOC类别列表，它们将出现在书架上。所有其他TOCS和类别将遵循这些。此列表中的非现有TOC或类别将被忽略。对于TOC，请将TOC的位置用作/pluginId/path/to/toc.xml。对于类别，请使用类别ID。
ignoredTOCS （不建议使用;请改用HELP_DATA ） 禁用TOC。帮助系统将忽略的帮助TOC（书籍）或TOC类别列表。禁用的TOC /类别不会出现在书籍列表中，无法链接。由禁用的TOC定义的主题将无法从搜索中获得。此列表中的非现有TOC或类别将被忽略。对于TOC，请将TOC的位置用作/pluginId/path/to/toc.xml。对于类别，请使用类别ID。
ignoredIndexes （不建议使用;请改用HELP_DATA ） 禁用关键字索引。帮助系统将忽略的帮助关键字索引贡献列表。禁用索引中的关键字不会出现在索引视图中。对于索引XML文件，请使用/pluginId/path/to/index.xml形式的文件路径。

五个css首选项允许通过将css文件插入到帮助服务器所服务的每个页面来控制页面外观。每个都可以包含零个或多个逗号分隔的形式/插件/路径的路径。这些路径应仅包含ASCII字符。
如果路径包含$ {os}，那么将替换为OS的名称，$ {parameter}形式的任何其他参数将保留供将来使用。

remote_css 要包含在由帮助系统提供的每个页面中的css文件列表，该文件源自远程信息中心。  
disabled_css 要由帮助系统从未启用的功能提供的每个页面中包含的css文件列表。 /PRODUCT_PLUGIN/disabled_book.css

org.eclipse.help.appserver插件：

不推荐使用：帮助不再使用此插件。有关替代用法，请参见表格
首选键 用法 默认
port （已弃用）

服务器侦听http请求的端口号。如果未给出端口，则系统将选择任意端口。

用“-vmargs -Dserver_port = <port>”启动Eclipse

  
host （不建议使用）

用于连接服务器的主机地址或名称。默认值是什么，eclipse将获取可用的本地地址。

在本地模式（工作台或独立，而非信息中心）中使用帮助的产品可以将此首选项设置为“ 127.0.0.1 ”，以确保帮助服务器不会暴露给网络上的其他用户。

使用“-vmargs -Dserver_host = <host>”启动Eclipse

 此Connector要创建的最大请求处理线程数，因此确定可以处理的最大并发请求数。 75
