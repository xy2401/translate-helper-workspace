发现选项属性

请注意，此功能已被弃用，不再完全受支持

使用“ 发现选项”属性面板可以控制如何发现构建项目所需的信息，增强搜索和内容辅助功能，以及自动增强makefile，例如包含路径和符号定义。

发现选项首选项
组 描述
组态 点击此处查看说明。
管理配置...... 点击此处查看说明。
发现配置文件范围 指定要为发现设置的配置文件的类型：
  • 每种语言 - 允许不同配置文件与不同资源类型（不同工具和输入类型）的关联，以便发现不同的设置，例如，对于C和C ++源文件以及项目使用的不同工具。此外，选择此选项可让您为不同的文件夹设置不同的配置文件;但是，只允许项目配置文件类型。
  • 配置范围 - Eclipse CDT仅使用一个配置文件来发现整个项目（配置）的扫描程序信息。这意味着允许项目和每文件发现配置文件。
工具 （语言特定列表） 显示特定于语言的编译器列表。从列表中选择一种语言。
自动发现路径和符号
自动发现路径和符号 扫描构建输出以填充路径和符号表，例如符号定义，系统包含目录，本地包含目录，宏和包含文件。
报告路径检测问题 设置Eclipse CDT在解析构建输出时无法解析的包含路径的诊断错误通知。
发现档案 指示用于路径和符号检测的发现配置文件。您指定的配置和发现配置文件范围的类型决定了此选项卡上显示的发现配置文件选项的内容。
发现配置文件选项
启用构建输出扫描程序信息发现 配置扫描程序以使用指定预处理程序符号定义的选项解析编译器命令的构建输出，并包括搜索路径（分别用于GCC编译器，-D和-I）。仅当“ 配置”设置为“发布”且“ 发现配置文件范围”设置为“ 配置 范围”时，此按钮才可见。
从文件加载构建输出 指定从中加载构建输出所选文件的名称。
加载 允许您加载文件以基于先前构建的输出发现路径和符号。要激活发现，请选择构建日志文件，然后选择“ 加载”按钮。仅当“ 配置”设置为“发布”且“ 发现配置文件范围”设置为“ 配置 范围”时，此按钮才可见。

注意：单击“ 变量”以打开“选择变量”窗口以定义构建输出文件。
浏览... 单击以查找以前构建的输出文件以从中加载构建输出。仅当“ 配置”设置为“发布”且“ 发现配置文件范围”设置为“ 配置 范围”时，此按钮才可见。
变量... 单击以指定参数，或创建和配置简单启动变量，您可以在支持变量的启动配置中引用这些变量。仅当“ 配置”设置为“发布”且“ 发现配置文件范围”设置为“ 配置 范围”时，此按钮才可见。
启用生成扫描仪信息命令 允许从扫描仪检索信息。如果未选中，则将包含默认gcc系统包含的包含; Eclipse根据指定的工具链收集编译器设置。这意味着Eclipse CDT可以获得默认的gcc系统包含与项目相关联。

选中后，您可以在编译器调用命令字段中指定任何所需的编译器特定命令。
编译器调用命令 指示用于调用编译器的编译器特定命令（例如gcc ，或用于执行将C / C ++源文件编译为.o文件以及将目标文件和库链接到可执行程序的所有必要步骤的更复杂命令。例如，命令gcc -E -P -v hello.c | hello.cpp读取编译器的配置文件并打印出包含编译器内部定义的预处理器符号的信息，并包含搜索路径。该信息与解析输出时发现的扫描仪配置（如果已启用“ 启用构建输出扫描程序信息发现”选项）相互补充，并添加到项目的扫描程序配置中。如果需要，可以单击“ 浏览”找到此命令。

注意：扫描程序信息的构建输出的解析是特定于编译器的。例如，GNU工具链编译器（gcc和g ++）使用-I表示包含路径，使用-D表示符号定义。有关扫描仪信息命令的更多信息，请参阅编译器特定文档，例如以下gcc命令：

  • -D 名字
  • -一世
  • -U 名字
  • -一世-
  • -nostdinc
  • -nostdinc ++
  • 包含文件
  • -imacros 文件
  • -idirafter dir
  • -isystem 目录
  • -iprefix 前缀
  • -iwithprefix 目录
  • -iwithprefixbefore DIR
浏览... 单击以浏览要包含在编译器调用命令中的文件。仅当“ 配置”设置为“发布”且“ 发现配置文件范围”设置为“ 配置 范围”时，此按钮才可见。
编译器调用参数 输入其他参数以发送到所选的编译器。
