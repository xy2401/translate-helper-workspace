提交

您可以将已修改的Workbench资源提交到存储库，以便其他团队成员可以查看您的工作。只有那些在该分支上提交的更改才能在该分支上工作。commit命令可以从两个位置发出： Team> Commit菜单或Synchronize视图。

使用Team> Commit提交更改

  1. 在其中一个导航视图中，选择要提交的资源。
  2. 右键单击资源，然后从弹出菜单中选择Team> Commit 。
  3. 如果有新文件的文件类型无法自动确定，则“提交”对话框的第一页将显示未知类型，并允许您将它们适当地设置为ASCII或二进制。单击“ 下一步”继续。
  4. 在“注释”页面上，为您的更改提供注释（例如， 修复拼写错误 ）。

您可以从“评论”页面预览即将提交的文件。如果已知任何文件是冲突的更改，则不允许提交。如果没有已知的冲突更改，则将允许提交，但服务器上仍可能存在冲突的更改（即，在同步操作期间，客户端会知道服务器上的冲突更改）。如果提交的任何文件存在冲突更改，则操作将失败。如果发生这种情况，您必须执行更新或使用“同步”视图来解决冲突。它被认为是一个更理想的工作流程，在提交之前始终更新，以确保在提交更多更改之前您拥有存储库的最新状态。

如果提交的一个或多个资源是新的且尚未添加到CVS控件，则将自动添加它们，除非通过从上下文菜单中选择“ 从视图中删除”来明确删除它们。

在“同步”视图中提交更改

  1. 在其中一个导航视图中，选择要提交的资源。
  2. 右键单击以打开弹出菜单，然后选择Team> Synchronize with Repository 。“同步”视图将打开。
  3. 在“同步”视图的工具栏上，选择“ 传出模式”按钮以显示您可能具有的任何已修改的Workbench资源（传出更改）。
  4. 如果存在冲突（红色箭头），请解决它们。为此，请在比较编辑器中打开它们，并使用文本比较区域将资源与冲突合并。您可以将更改从文件的存储库修订复制到文件的Workbench修订，并保存合并的Workbench资源。一旦解析了Structure Compare区域中的所有冲突，就在Synchronize视图中的资源上执行Mark as Merged ，以使更改成为传出更改，并且您已准备好提交。
  5. 在Structure Compare窗格中，右键单击要提交的层次结构的顶部，然后从弹出菜单中选择Commit 。
  6. 在“提交注释”对话框中，为更改提供注释（例如， 修复拼写错误 ）。同样，如果存在未知类型的新文件，则会要求您指定它们应该是什么类型。

提示：您可以通过执行覆盖和提交来提交冲突的文件。这会将资源的Workbench副本提交到存储库中，从而删除任何传入的更改。

警告：上述“ 覆盖和提交”命令的行为仅适用于“同步”视图的传出模式。在视图的传入/传出模式下 ，传出更改和冲突的行为是相同的，但命令将传入的更改还原为本地Workbench内容的任何内容。如果在传入/传出模式下使用此命令，请务必小心。
