CDT项目

在开始使用CDT之前，必须创建一个项目来存储源代码，makefile，二进制文件和相关文件。C / C ++项目显示在C / C ++ Projects视图中。

提示：不支持嵌套项目。每个项目必须组织为一个独立的实体。通过允许项目引用驻留在工作空间中的其他项目来支持项目依赖性。有关更多信息，请参阅选择引用的项目 。

有关项目及其存储位置的更多信息，请参阅：

  • Workbench用户指南>任务>资源
  • Workbench用户指南>任务>运行Eclipse

项目类型

您可以创建C或C ++项目。

要构建项目，您可以创建自己的makefile，也可以让CDT自动为您生成makefile。

您可以随时为现有项目切换这些模式（生成或不生成makefile）。使用属性。

创建新项目时，需要指定项目类型。此项目类型将确定工具链和数据以及CDT使用的选项卡。在“新建CDT项目向导”中，您可以从以下项目类型中进行选择：

  • 可执行文件 - 提供可执行应用程序。此项目类型文件夹包含三个模板。
    • Hello World C ++示例提供了一个带main（）的简单C ++ Hello World应用程序。
    • Hello World ANSI C Example提供了一个带main（）的简单C Hello World应用程序。
    • Empty Project提供一个不包含文件的单个源项目文件夹。

    选择模板后，结果是一个只包含项目类型所需的元数据文件的项目。您需要根据需要修改这些源文件，并为项目的目标提供源文件。

    可执行项目类型的makefile由CDT自动创建。

  • 共享库 - 单独编译和链接的可执行模块。创建使用共享库（libxx.so）的项目时，将共享库的项目定义为应用程序的项目引用。对于此项目类型，CDT将对象文件组合在一起并加入它们，以便它们可重定位并可由许多进程共享。共享库使用libxx.so格式命名。 version ，其中version是一个默认值为1的数字。libxx.so文件通常是指向最新版本的符号链接。

    此项目类型的makefile由CDT自动创建。

  • 静态库 - 可以链接到另一个应用程序（libxx.a）的目标文件的集合。CDT将目标文件（即* .o）组合成一个直接链接到可执行文件的存档（* .a）。

    此项目类型的makefile由CDT自动创建。

  • Makefile项目 - 创建没有元数据文件的空项目。此选择对于导入和修改现有的基于makefile的项目非常有用;不为此项目类型创建新的makefile。

项目转换

您可以将项目从C转换为C ++（或从C ++转换为C）。例如，如果您的需求发生变化，并且您必须将现有C项目转换为C ++，则可以在不重新创建项目的情况下执行此操作。CDT转换您的项目文件并解决任何源控制问题。

关于项目的一些注意事项

  • 在项目中创建文件时，会创建每个更改的记录（本地历史记录）。有关本地历史记录的详细信息，请参阅Workbench用户指南>参考>用户界面信息>开发环境>本地历史记录 。
  • 项目和文件名中的空格可能会导致某些工具出现问题，例如make实用程序或编译器。
  • 仅使用大小写来区分文件和项目时要小心。基于UNIX的操作系统文件名区分大小写，但Windows文件名不区分大小写。因此，Hello.c和hello.c在UNIX中是单独的文件，但在Windows中相互覆盖。

有关项目的更多信息，请参阅“ Workbench用户指南”>“概念”>“工作台”>“资源” 。
