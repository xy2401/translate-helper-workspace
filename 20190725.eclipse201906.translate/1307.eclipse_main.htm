主标签

Main选项卡允许您自定义启动配置的重要设置，包括运行时工作空间的位置，要运行的程序以及运行程序的Java运行时环境。

工作区数据

Workspace Data位置是磁盘上的目录，它将包含运行时Eclipse应用程序的所有工作和元数据（例如首选项等）。当然，此位置必须与主机工作空间的位置不同，因为您应该在沙箱中测试插件。

默认情况下，PDE将运行时工作空间的位置设置为具有基于启动配置名称的唯一名称，并且是主机工作空间目录的兄弟。只要位置可写，您就可以将运行时工作空间的位置设置为您喜欢的任何位置。变量名称可以在运行时工作空间位置的路径中使用，从而允许在团队成员之间共享启动配置。

如果要在新工作空间或干净的日志文件中测试插件，则“ 清除”选项很有用。默认情况下，此选项处于关闭状态，因为Eclipse应用程序工作空间通常包含无需从一次启动到下一次启动清除的有用数据。

清除前询问确认选项会在删除运行时工作空间数据之前提示用户。默认情况下，此选项处于打开状态，以防止意外丢失测试工作区中的数据。

程序运行

您可以在“运行程序”部分指定要测试的产品或应用程序。

产品组合框中填充了工作空间和目标插件中定义的所有产品（ org.eclipse.core.runtime.products扩展）的ID。最初选择的产品ID是config.ini文件中位于$ {target_home} / configuration目录中的eclipse.product键指定的缺省产品。$ {target_home}变量是指Windows> Preferences ...> Plug-in Development> Target Platform首选项页面上指定的目标平台的位置。在Eclipse SDK中，默认产品ID为org.eclipse.sdk.ide 。

应用程序组合框中填充了工作空间和目标插件中定义的所有应用程序（ org.eclipse.core.runtime.applications扩展）的ID。最初选择的应用程序标识是config.ini文件中位于$ {target_home} / configuration目录中的eclipse.application项指定的缺省应用程序。在Eclipse SDK中，默认应用程序ID是org.eclipse.ui.ide.workbench 。

默认情况下选择“ 运行产品”选项，因为测试产品（品牌应用程序）比测试普通应用程序带来更丰富的体验。

Java可执行文件

Java Executable选项允许您指定运行Eclipse Application的可执行文件。默认可执行文件可能有所不同，但它显示在UI中。

Java运行时环境

您可以根据您选择的Java Runtime Environment （JRE）测试您的插件;此外，此运行时JRE不一定必须与用于编译工作区插件的JRE（具有关联的SDK）相同。

运行时JRE组合框中填充了Windows>首选项....> Java>已安装JRE首选项页面上列出的所有JRE。您可以通过按Installed JREs ...按钮动态地将更多JRE添加到已安装的JRE列表中，这将打开Installed JREs首选项页面。

Bootstrap条目

启动宿主Eclipse实例通常涉及本机启动程序可执行文件，后者又调用org.eclipse.equinox.launcher。主类启动引导过程。Main类位于名为org.eclipse.equinox.launcher的包中。

启动运行时Eclipse应用程序会绕过本机可执行文件并直接调用Main类，就像运行任何常规Java应用程序一样。因此，PDE计算Main类的位置，并将其隐式添加到启动配置的类路径中。

某些分析器也需要一些本机库位于启动配置的类路径中。Bootstrap Entries字段允许您指定这些附加位置并将其附加到类路径。
