请参阅org.eclipse.ui.commands部分以创建命令定义，使用键绑定将它们连接起来，然后将它们放在菜单中。

将遗留操作与命令相关联

当仍然使用旧的旧操作扩展点时，操作可以与代码中的命令或操作集的plugin.xml中的命令相关联。您的选择取决于操作的定义位置。

您应该更喜欢激活AbstractHandler，而不是使用IAction 。 

// IWorkbench，IWorkbenchWindow和IWorkbenchPartSite都是IServiceLocators IHandlerService handlerService =（IHandlerService）locator.getService（IHandlerService.class）; handlerService.activateHandler支持（org.eclipse.ui。IWorkbenchCommandConstants。FILE_SAVE，新的MySaveHandler（））;

但是在处理动作时，在代码中实例化的动作也可以使用IAction协议与动作定义相关联。这通常在创建操作时完成。SaveAction在初始化时使用此技术。

public SaveAction（IWorkbenchWindow window）{...的setText ...的setToolTipText ...setImageDescriptor ...
	setActionDefinitionId（“org.eclipse.ui.file.save”） ; }

（注意：方法名称setActionDefinitionID可以更恰当地命名为setCommandID 。方法名称反映了键绑定的原始实现，并使用了过时的术语。）

通过调用setActionDefinitionID ，实现操作（ SaveAction ）与命令定义标记中使用的命令ID相关联。最好为操作定义定义常量，以便在代码中轻松引用它们。

如果在操作集中定义操作，则通常不需要自己实例化操作。当用户从菜单或键盘调用您的操作时，工作台将为您执行此操作。在这种情况下，您可以将操作与XML标记中的命令ID相关联。以下显示了操作集的假设标记：

<extension point =“org.eclipse.ui.actionSets”> <actionSet id =“com.example.actions.actionSet”label =“Example Actions”visible =“true”> <action id =“com.example.actions。 action1“menubarPath =”additions“label =”示例保存操作“class =”org.example.actions。ExampleActionDelegate“ definitionId =”org.eclipse.ui.file.save“ > </ action> ...</ actionSet> </ extension>

definitionId属性用于声明操作的命令ID。

使用这两种技术，将操作与命令ID相关联会导致为命令org.eclipse.ui.file.save定义的任何键绑定在适当时调用您的操作。
