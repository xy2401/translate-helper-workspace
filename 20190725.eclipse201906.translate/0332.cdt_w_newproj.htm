创建一个Makefile项目

本教程描述了创建包含makefile和调试程序的新C ++项目的过程。

您需要创建一个项目来包含您的源代码和相关文件。项目有一个关联的构建器，可以在更改源文件时逐步编译它们。

要创建项目：

  1. 选择文件>新建>项目 。


    创建新项目时，需要指定项目类型。此项目类型将确定CDT使用/显示的工具链，数据和选项卡。


  2. 选择要创建的项目类型。在本教程中，展开C / C ++文件夹并选择C ++ Project 。

    “ C ++项目”向导将打开。点击这里查看插图。


    默认情况下，CDT过滤工具链和当前显示在这些列表中的项目类型基于您为本教程选择的C ++项目向导的语言支持。


  3. 在“ 项目名称”字段中，键入HelloWorld。
  4. 保持选中“ 使用默认位置”选项。


    接下来，您要选择要创建的项目类型。在“新建CDT项目向导”中，您可以从以下项目类型中进行选择：

    • 可执行文件 - 提供可执行应用程序。此项目类型文件夹包含三个模板。
      • Hello World C ++示例提供了一个带main（）的简单C ++ Hello World应用程序。
      • Hello World ANSI C Example提供了一个带main（）的简单C Hello World应用程序。
      • Empty Project提供一个不包含文件的单个源项目文件夹。

        选择此模板后，结果是一个项目，其中只包含项目类型所需的元数据文件。您应该为项目的目标提供源文件。

      可执行项目类型的makefile由CDT自动创建。

       

    • 共享库 - 单独编译和链接的可执行模块。创建使用共享库（libxx.so）的项目时，将共享库的项目定义为应用程序的项目引用。对于此项目类型，CDT将对象文件组合在一起并加入它们，以便它们可重定位并可由许多进程共享。共享库使用libxx.so格式命名。 version ，其中version是一个默认值为1的数字。libxx.so文件通常是指向最新版本的符号链接。此项目类型的makefile由CDT自动创建。
    • 静态库 - 可以链接到另一个应用程序（libxx.a）的目标文件的集合。CDT将目标文件（即* .o）组合成一个直接链接到可执行文件的存档（* .a）。此项目类型的makefile由CDT自动创建。
    • Makefile项目 - 创建没有元数据文件的空项目。此选择对于导入和修改现有的基于makefile的项目非常有用;不为此项目类型创建新的makefile。

    默认情况下，CDT会根据您为本教程选择的C ++项目向导的语言支持，过滤当前显示在这些列表中的工具链和项目类型 。

     

  5. 从Project types列表中，展开Makefile项目并选择Empty Project 。该项目允许您自己输入源文件和makefile。
  6. 从工具链列表中选择所需的工具链 。

    工具链是一组用于构建项目的工具（如编译器，链接器和汇编程序）。其他工具（如调试器）可以与工具链相关联。
    可以使用多个工具链，具体取决于系统上安装的编译器。

  7. 单击下一步 。
  8. （可选）在“ 选择配置”页面上，单击“ 高级设置” 。
    • 展开C / C ++ Build> Settings 。单击二进制解析器选项卡。


      点击这里查看插图。


    • 在Binary Parser列表中，选择适合您平台的错误解析器。其他项目设置也可以在这里修改。默认选择可能没问题。

      要确保C / C ++项目视图的准确性以及成功运行和调试程序的能力，选择正确的解析器非常重要。为开发环境选择正确的解析器并构建项目后，可以在C / C ++ Projects视图中查看.o文件的组件。您还可以在C / C ++编辑器中查看.o文件的内容。


      单击“ 确定”关闭“项目属性”窗口。
  9. 单击完成 。
  10. 如果消息框提示您更改透视图，请单击“ 是” 。

您的新项目显示在Project Explorer视图中。您的项目为空，因为您尚未为项目创建文件。您可能会看到一个错误，因为您的项目尚未构建任何内容。您现在可以开始编写HelloWorld程序的代码。

提示：您可以随时通过在Project Explorer视图中右键单击项目并单击Properties来查看和修改HelloWorld项目的属性。
