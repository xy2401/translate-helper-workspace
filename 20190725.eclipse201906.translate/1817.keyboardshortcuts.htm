按键

可以使用Eclipse在Eclipse中广泛定制键盘的功能 常规>键首选项页面。在Eclipse中，分配键击和键序列以调用特定命令。

键笔划，键序列和键绑定

“按键行程”是按下键盘上的一个键，同时可以按住这些修改键中的一个或多个： Ctrl ， Alt （在macOS上为⌥ ）， Shift或⌘ （仅在macOS上）。例如，按住Ctrl然后按A会产生按键Ctrl+A按下修改键本身并不构成击键。

“键序列”是一个或多个键击。传统上，Emacs为特定命令分配了两个或三个关键笔划键序列。例如，在emacs中分配给Close All的正常键序列是Ctrl+X Ctrl+C要输入此键序列，请按下键Ctrl+X键，然后按键Ctrl+C虽然Eclipse支持任意长度的键序列，但建议键盘快捷键的长度为（或更短）四个键击。

“键绑定”是将键序列分配给命令。

计划

'scheme'是一组绑定。Eclipse包括两个方案：

  • 默认
  • Emacs（扩展默认）

默认方案包含一组通用绑定，在许多情况下可识别为众所周知的命令的传统键序列。例如， Ctrl+A分配给Select All ， Ctrl+S分配给Save 。

Emacs方案包含一组Emacs用户熟悉的密钥绑定。例如， Ctrl+X H分配给Select All ， Ctrl+X S分配给Save 。

重要的是要理解为什么Emacs方案说它“延伸默认”。Emacs方案不是一套完整的绑定，如Default方案。相反，它尽可能借用Default方案，仅定义显式Emacs样式绑定，它们与Default方案不同。通常，只有众所周知的命令如Select All ， Save等具有与之关联的特定Emacs键序列。

通过更改按键首选项页面上的“方案”设置，选择您最熟悉的方案。如果选择“ 默认”方案，则会忽略所有Emacs绑定。如果选择Emacs方案，则显式Emacs样式的键序列分配优先于默认方案中的任何冲突分配。

上下文

键绑定可以根据Eclipse的当前上下文而变化。

例如，有时活动部分可能是Java文件编辑器，其中一组不同的键序列分配可能比活动部分是html文件编辑器更合适。作为一个具体示例，通常在诸如Java文件编辑的上下文中将Ctrl+B分配给Build ，而在诸如HTML文件编辑的上下文中将Ctrl+B分配给Make Text Bold 。此上下文通常由活动部件确定，但也可能受活动窗口或对话框的影响。如果活动部件未选择特定上下文，则工作台将活动上下文设置为In Windows 。

Eclipse包含许多不同的上下文。一些例子是：

  • 在对话框和Windows中
  • 在Windows中（在对话框和Windows中扩展）
  • 在对话框中（在对话框和Windows中扩展）
  • 编辑文本（在Windows中扩展）
  • 编辑Java源代码（扩展编辑文本）
  • 调试（在Windows中扩展）
  • 调试Java（扩展调试）
  • 在控制台中
  • 编辑Ant构建文件

与配置非常相似，上下文可以扩展其他上下文。例如，“ 编辑Java源”上下文借用了“ 编辑文本”上下文中的键绑定，而后者又依赖于“ 在Windows”上下文中的键绑定。

注意：不建议将密钥绑定提升到它扩展的上下文。例如，建议不要将编辑文本键绑定移动到“ 对话框”和“Windows”上下文中。这可能会产生意外结果。

某些键绑定可以在对话框中工作。这些键绑定分配给In Dialogs和Windows上下文。这种键绑定的一个例子是“剪切”的键绑定。可以更改这些键绑定。例如，可以在对话框中将Ctrl + X作为剪切，但在窗口中按Ctrl + W剪切。

平台和区域设置

键绑定也因平台和区域设置而异。在macOS平台上， ⌘+S被指定为Save ，而不是通常的Ctrl+S在中文语言环境（zh）上， Alt+/被分配给Content Assist ，而不是通常的Ctrl+Space 。

当前平台和语言环境是在Eclipse启动时确定的，并且在Eclipse实例的过程中不会发生变化。

自定义键绑定

对于多笔划键序列，方案和上下文，在自定义键绑定时需要记住很多事项。为了简化操作，所有关键的自定义都是在 常规>键首选项页面。

在此示例中，我们要将CTRL + 5绑定到About命令。默认情况下，键首选项页面将显示所有可能的键绑定。您可以在“帮助”类别中看到“关于”命令。您可以通过将焦点放在“绑定”文本框中并按下CTRL和5来绑定命令，就像执行命令一样。

当您键入CTRL + 5时，您已为About创建了一个绑定。最右边的列将通过显示U来指示这是用户绑定。如果与另一个键发生冲突，此列也会显示一个C.绑定将在默认上下文“在Windows中”。您现在可以使用When组合框更改键绑定上下文（例如，将此绑定移动到“编辑文本”）。

如果要向About添加第二个键绑定，可以使用“ 复制命令”按钮创建第二个命令条目，以便将另一个键绑定到。如果要删除绑定，可以使用“ 删除绑定”按钮或只是将焦点置于“绑定”文本框并单击“ 退格” 。

键绑定的动态性

密钥绑定由插件提供，在Eclipse中，可以添加或删除插件。这可能导致添加或删除这些插件声明的键绑定。Eclipse以某种方式存储自定义键绑定以补偿这一点。考虑上面的示例，其中在默认方案中将CTRL+6分配给About 。假设您安装了一个新的插件，将CTRL+6分配给特定命令。Eclipse将保留您对About的分配。

解决冲突

只有有限数量的简单公共键击可用于分配多个命令。我们已经看到，方案，上下文，平台和区域设置都将密钥序列分配分配到不会相互冲突的域中。如果上下文不存在，请考虑上面的Ctrl+B情况。一个插件将Ctrl+B分配给Build ，另一个插件将Ctrl+B分配给Make Bold Text 。Eclipse如何正确解决这一冲突？

尽管通过采用上述机制大大减少了冲突，但它们仍然可以发生。两个插件彼此独立，可以使用相同的上下文，方案，平台和语言环境为不同的命令分配相同的键序列。考虑插件是否在In Windows上下文中为Ctrl+F4分配，并将Default方案分配给其中一个命令。这与Eclipse在同一上下文和方案中将Ctrl+F4分配给close命令直接冲突。

这是一场冲突。调用这两个命令是不合适的，也不适合简单地选择两个命令之一来接收键击。我们使用冲突的命令弹出Key Assist Dialog，并允许用户选择一个。“键辅助”对话框是显示多个键击键绑定的命令选项的同一对话框。例如，如果将2个命令绑定到F12，您可能会看到：

如果用户设置了键绑定并产生冲突，则冲突绑定将显示在冲突列表中。这可用于在冲突的键绑定之间导航，以便可以更改它们。

可以通过将键序列显式分配给其中一个命令或从另一个命令中删除它来解决这些类型的冲突。

另一种类型的冲突可以由多键笔划键序列引起。例如，在Emacs方案中，有许多以键击Ctrl+X开始的多键笔划键序列。 Ctrl+X K被指定为Close 。Ctrl+X H分配给Select All 。

如前所述， Emacs方案借用了Default方案的键绑定。在默认方案中， Ctrl+X被分配给Cut 。虽然Emacs方案没有明确重新定义Ctrl+X ，但是按Ctrl+X是许多键绑定的一部分。在Emacs方案中，当按下Ctrl+X ，一个是输入许多可能分配的键序列之一的一半。人们不希望此时调用Cut动作。

对于此类冲突，规则是将忽略分配给Ctrl+X的Cut键序列。否则，将无法在Emacs配置中完成许多键绑定。

导出密钥绑定

绑定可以导出为CSV文件。为此，请按“ 导出CSV ... ”按钮。这将启动一个文件对话框，您可以在其中指定导出文件的位置。注意：导出文件仅用于报告目的， 不能用于将绑定文件导入Eclipse。
