插入搜索引擎

帮助系统中的新联合信息搜索使用搜索引擎类型和搜索引擎的概念。引擎类型是元引擎，通过参数化可以从中创建许多具体的搜索引擎。

通过org.eclipse.help.ui.searchEngine提供新的引擎类型：

<extension point =“org.eclipse.help.ui.searchEngine”> <engineType scopeFactory =“com.example.xyz。XYZScopeFactory“label =”XYZ搜索“class =”com.example.xyz.search。XYZSearch“icon =”icons / etool16 / xyzsearch.gif“pageClass =”com.example.xyz.search。XYZSearchPage“id =”com.example.xyz。XYZSearch“> <description> XYZ搜索实例搜索XYZ站点。</ description> </ engineType>
此扩展点用于在信息搜索中插入搜索参与者。每个搜索引擎都可以单独配置。启动搜索时，每个搜索引擎都作为后台作业执行，结果将立即整理到查询下的帮助视图中。

除非未定义productId属性，否则此处定义的搜索引擎将不会自动显示为联合搜索参与者，直到建立引擎产品绑定为止。对于定义它的引擎，只有那些绑定到特定产品的引擎才会在该产品运行时显示。

搜索引擎可以简单地组成一个URL，并且只提供一个包含该URL作为href 。可以像这样插入API支持需要许可的热门搜索引擎。另一方面，搜索引擎可以与服务器通信，并通过标签，href，简短描述，得分等信息接收个人命中。本地帮助引擎可以通过这种方式产生命中。

无论搜索机制如何，引擎都可以使用JFace首选项页面提供各种搜索范围设置。从“帮助”视图中按照“高级设置”链接时会显示这些页面。除了使用引擎定义的根首选项页面之外，还可以插入其他首选项子页面以进行更高级的设置。

使用IPreferenceStore对象加载和存储范围设置。所有引擎的范围设置在命名范围集下组合在一起。首次打开时，会创建默认范围设置（“默认”），但用户可以定义更多范围集并在它们之间进行切换。

由于联合搜索支持是org.eclipse.help.base插件的一部分，因此需要工厂从首选项存储中的数据创建搜索范围对象。插入范围首选项页面的客户端也需要插入范围工厂。

默认情况下，此扩展点中定义的引擎不会显示在UI中。这里显示的是一个可以单独修改的搜索引擎的具体实例 。产品可以使用注册引擎类型的多个实例预先配置帮助系统，可能参数化以以期望的方式执行。此外，用户可以添加自己的已注册引擎实例并根据自己的喜好进行配置：

<engine enabled =“true”engineTypeId =“com.example.xyz.search。XYZSearch“id =”com.example.xyz。XYZSearch“label =”XYZ搜索“> </ engine> <engine enabled =”true“engineTypeId =”org.eclipse.help.ui.web“id =”org.eclipse.sdk。Eclipse“label =”％搜索。Eclipse.label“> <description>％search。Eclipse.desc </ description> <param name =“url”value =“http://eclipse.org/search/search.cgi?q={expression}&amp;ul=&amp;ps=20&amp;m=all” > </ param> </ engine>
