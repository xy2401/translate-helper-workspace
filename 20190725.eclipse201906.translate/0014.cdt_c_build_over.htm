构建C / C ++项目

CDT依赖于外部make实用程序（例如GNU make）来构建项目。当您创建Managed Make C项目或Managed Make C ++项目时，CDT可以自动生成makefile。您可以选择创建Standard Make C项目或Standard Make C ++项目并自己提供makefile。

所需工具

您必须安装并配置以下实用程序：

  • 建立（例如制作）。
  • 编译（例如gcc）。
  • 调试（例如gdb）。
注意：虽然make，gcc和gdb是文档中使用的示例，但几乎可以使用任何类似的工具或实用程序集。

提示： Cygwin包含Windows环境的这些实用程序（make，gcc和gdb）。在运行cygwin安装时，请确保选择gcc和make ，因为默认情况下不安装它们。有关更多信息，请访问http://www.cygwin.com 。如果您是Red Hat用户，则构建项目所需的全部操作都包含在Red Hat Linux安装中。对于其他操作系统，请参阅安装文档。

建立术语

CDT使用许多术语来描述构建的范围。

建设项目

这是一个增量构建（make all，假设所有内容都在makefile中定义）。仅构建受特定项目中的已修改文件影响的组件。

重建项目

构建项目中的每个文件，无论自上次构建以来文件是否已被修改。重建是干净的，然后是构建。

有关构建的更多信息，请参阅：

  • Workbench用户指南>概念>工作台>构建
  • Workbench用户指南>任务>构建资源

与构建相关的信息显示如下：

  • Console视图显示构建工具的输出。
  • “问题”视图显示与项目相关的编译器错误和警告的列表。
  • 对于Standard Make项目，Makefile目标显示在Make Targets视图中。

有关“问题”视图的详细信息，请参阅“ Workbench用户指南”>“参考”>“用户界面信息”>“视图和编辑器”>“问题”视图 。

获取makefile

您可以创建一个为其提供makefile的C / C ++项目，也可以创建一个CDT自动生成makefile的C / C ++项目。

要创建新项目，请从菜单栏中选择“ 文件”>“新建”>“项目” 。在出现的对话框中，展开C / C ++组并选择例如C Project

  • 在生成的向导页面中，要创建为其提供makefile的 项目 ，请选择Makefile project并选择其中一个替代项。可以创建一个空项目或一个简单的“Hello World”。您自己编辑和管理makefile。

     

  • 要创建CDT提供基本makefile的项目，请选择另一个项目类型，例如Executable，然后选择其中一个示例，或选择Empty Project 。

设置构建首选项

您可以在Eclipse中设置构建首选项：

建立订单
如果必须先在其他项目之前构建某些项目，则可以设置构建顺序 。如果您的项目引用另一个项目，CDT必须首先构建另一个项目。要设置构建顺序，请从菜单栏中选择Window> Preferences，然后选择General> Preferences> Build Order 。

设置构建顺序时，CDT不会重建依赖于项目的项目;您必须重建所有项目以确保传播所有更改。

自动保存
您可以将CDT设置为在执行手动构建时自动保存所有已修改的资源。在首选项对话框中，选择常规>工作区，然后选中构建前自动保存 。默认情况下， 不启用此功能。

控制项目的构建

对于Makefile项目，项目使用的C / C ++编译器由项目的Properties设置控制。要查看项目的属性，请右键单击该项目，然后选择“ 属性” 。在出现的对话框中， C / C ++ Build页面使您可以控制各种设置，包括：

构建命令
在“ 构建器设置”选项卡上，它控制使用哪个make 。要更改它，请取消选中Use default build命令并对其进行更改或向make命令添加参数。
构建设置
在“ 行为”选项卡上，它控制编译器是否在第一次生成错误时停止 （继续）。取消选中Stop on first build error将强制编译器尝试构建所有引用的项目，即使当前项目有错误也是如此。
Workbench构建行为
在Behavior选项卡上，它控制将根据构建的范围构建makefile目标，例如all或clean 。

对于标准（非Makefile）项目（通常称为早期CDT版本的“Managed Build”或“Managed Make”项目），项目属性对话框使您可以管理项目的构建配置。有关其他信息，请参阅

  • 参考> C / C ++属性> C / C ++项目属性>托管生成项目
  • 参考> C / C ++属性> C / C ++项目属性>托管生成文件属性

查看构建信息

与构建相关的信息显示如下：

  • Console视图显示make实用程序的输出。
  • “ 问题”视图显示与项目相关的编译器错误和警告的列表。
  • 对于标准生成项目，生成操作显示在“生成目标”视图中。
