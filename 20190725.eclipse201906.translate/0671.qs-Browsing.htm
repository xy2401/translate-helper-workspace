使用Java浏览透视图

在本节中，您将使用Java浏览透视图来浏览和操作代码。使用Package Explorer浏览Java元素概述了使用Package Explorer浏览元素。与Package Explorer相比，Package Explorer组织了树中的所有Java元素，包括项目，包，编译单元，类型等，浏览透视图使用不同的视图来呈现相同的信息。在一个视图中选择元素，将在另一个视图中显示其内容。

要打开浏览透视图激活从Java透视图中的Window> Open Perspective> Java Browsing或使用Open a Perspective工具栏按钮的上下文菜单。

透视图以下列方式相互连接：

  • 在项目视图中选择的元素显示了其在Packages视图包。
  • “ 类型”视图显示在“ 包”视图中选择的包中包含的类型。
  • “ 成员”视图显示所选类型的成员。从功能上讲， Members视图与普通Java透视图中使用的Outline视图相当。
  • 在“ 成员”视图中选择元素会在编辑器中显示该元素。如果没有为元素打开编辑器，双击该元素将打开相应的编辑器。

默认情况下，所有四个视图都链接到活动编辑器。这意味着视图将根据活动编辑器中显示的文件调整其内容和选择。以下步骤说明了此行为：

  1. 在Packages视图中选择junit.extensions 。
  2. 在类型视图中双击它，在编辑器中打开TestSetup 类型 。
  3. 现在，通过单击编辑器选项卡，将焦点重点放在文件TestCase.java上打开的编辑器中。“ 包” ，“ 类型”和“ 成员”视图调整其内容和选择以反映活动编辑器。Packages视图的选择设置为junit.framework ，而Types视图显示junit.framework包的内容。此外，还选择了TestCase类型。

从功能上讲，Java浏览视角与Java视角完全可比。项目，包，类型等的上下文菜单以及全局菜单和工具栏是相同的。因此，激活这些函数类似于在Java透视图中激活它们。
