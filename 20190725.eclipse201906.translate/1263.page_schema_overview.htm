架构概述页面

关于本页

此表单页面允许您标识和记录扩展点模式。它还允许您包含和使用由其他共享扩展点模式定义的语法。

页面部分

此页面包含以下部分：

  • 一般资料科
  • 架构包含部分
  • 文件科

一般资料科

此部分唯一标识已编辑的扩展点架构。扩展点模式ID的格式与其对应扩展点的完全限定ID相同： <plugin-id>。<point-id>

一般信息部分字段

编辑插件ID字段以指定唯一扩展点模式ID的限定符部分。此字段的值应与定义架构的相应扩展点的插件的ID匹配。

编辑“ 点ID”字段以指定唯一扩展点架构ID的ID部分。此字段的值应与用于定义架构的相应扩展点的ID匹配。

编辑“ 点名称”字段以指定扩展点架构的可读名称。此字段的值应与用于定义架构的相应扩展点的名称匹配。

架构包含部分

本部分允许您包含和使用来自其他共享扩展点模式的语法。

模式包含部分字段

viewer字段列出了共享的扩展点模式，其编解器包含并由编辑的扩展点模式使用。

可以包括位于与编辑的扩展点模式相同的插件中的共享扩展点模式。这些模式在查看器中通过其文件名的相对路径表示： <relative-path> / <schema-file-name>

可以包括位于另一工作空间插件中的共享扩展点模式。这些模式在查看器中通过其文件名的特殊绝对路径表示： schema：// <defined-plugin-id> / <path> / <schema-file-name>

模式包含部分操作

单击“ 添加...”按钮，打开“ 文件选择”对话框，以便在编辑的模式中包含和使用现有共享扩展点模式中的语法。该对话框显示所有工作区扩展点模式的过滤分层视图。如果选择了包含在与编辑的模式不同的插件中的扩展点模式，则将构造特殊的绝对路径并将其输入到查看器字段中。如果选择了包含在与编辑的模式相同的插件中的扩展点模式，则计算从编辑的模式到所选模式的相对路径并将其输入到查看器字段中。

单击“ 删除”按钮，从包含中删除选定的共享扩展点模式的语法，并使用编辑的扩展点模式。仅当在查看器字段中选择了架构时，此操作才可用。

双击查看器字段中的扩展点模式，以在新的扩展点模式编辑器中打开该模式。

文件科

本部分允许您记录编辑的扩展点模式。它的内容将用于组成编辑的扩展点模式的参考文档。因此，可以在任何部分的字段中使用有效的HTML标记。

此部分包含以下选项卡：

  • 描述
  • 以来
  • 例子
  • API信息
  • 提供的实施
  • 版权

文档部分字段

编辑“ 描述”选项卡的字段以指定此编辑的模式的定义扩展点的详细描述。通常，此字段用于描述扩展点的用途，实用程序和相关概念。

编辑Since选项卡的字段以指定首次添加此编辑的模式的Eclipse的版本。例如，如果首次将此编辑的模式添加到Eclipse 3.0，则该字段的值应为3.0 。假设编辑后的模式的语法是API，一旦引入就会持续释放发布，并且为了向后兼容而维护。该字段类似于Java的Javadoc @since标记。

编辑“ 示例”选项卡的字段以指定符合此编辑的架构语法的扩展的有效XML片段。尝试使用一个示例来说明扩展的典型用例。

编辑“ API信息”选项卡的字段，以描述扩展点的元素和属性的结构，内容和语义。

编辑“ 提供的实现”选项卡的字段，以指定为此编辑的模式定义扩展点的插件是否也为其提供扩展。该字段将用户指向实际的扩展实现，他们可以将其用作模板或引用来定义自己的扩展。

编辑“ 版权”选项卡的字段，以概述复制，发布，销售或分发已编辑架构的专有合法权利。
