C / C ++项目属性，路径和符号，项目参考

项目引用是表达不同项目中的Build配置之间的依赖关系的强大方式。具体来说，他们扮演两个角

  • 将导出的设置从一个项目自动传播到另一个项目，这取决于它
  • 如有必要，在依赖项目之前构建引用的项目

与Eclipse Common References页面不同 ，此页面允许设置对特定配置的引用。

自CDT 7以来，此功能得到了改进：

  • 订购参考项目

    并且可以使用提供的按钮更改此订单。引用的项目按对话框中列出的顺序构建，订单在构建中用于设置“包含路径”和“库”链接线。

  • 可以引用同一项目中的其他配置

    允许引用同一项目中的另一个配置。

  • 工作区中不存在的引用项目以灰色显示。

项目清单（主窗格）
