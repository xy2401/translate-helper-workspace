设置包含C / C ++索引器的路径和宏

独立调试器支持基于应用程序代码的许多操作，例如代码突出显示，导航，内容辅助和静态分析。通常，C / C ++项目必须配置为享受这些功能，因为大多数现实生活中的C / C ++项目在很大程度上依赖于项目本身之外的代码，例如系统头或第三方库。C / C ++索引器需要处理这些索引以提供准确的索引。

独立调试器将尝试自动发现包含路径和预处理器符号。此过程称为扫描程序发现或自动发现 。这种符号的发现是双重的。第一种方法是独立调试器将尝试检测内置编译器符号，并包含使用特殊选项运行编译器的路径并解析输出。独立调试器使用的另一种方法是分析从Makefile构建生成的输出。由于独立调试器不包含构建支持，因此必须在外部保存此输出并使用-b选项在命令行上传递给调试器，或者使用File> New Executable指定构建日志...对话。通常，include路径使用-I选项提供给编译器，使用-D选项提供宏。这依赖于构建的详细构建输出，其中所有这些选项实际上都是由make打印的。

Scanner Discovery使用语言设置提供程序查找包含路径和预处理程序符号。语言设置提供程序可以在项目属性页面“预处理程序包含路径，宏等”，配置的提供程序选项卡和首选项页面上配置C / C ++首选项：共享提供程序的扫描程序发现 。

如果通过自动发现检索的信息不足，则用户可以检查发现的条目，并在属性页“预处理器包含路径，宏等”，“ 条目”选项卡上手动输入其他包含路径和宏。
