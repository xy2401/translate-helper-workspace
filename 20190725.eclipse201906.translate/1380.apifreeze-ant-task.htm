API冻结Ant任务

目的

此任务标识已相对于API基准添加，修改或删除的API。此任务旨在验证API冻结后不会发生API更改。

用法

描述

Ant任务的名称是： apitooling.apifreeze 。要使用，jar文件apitooling-ant.jar必须位于Ant类路径上。

<apitooling.apifreeze
	baseline="..."
	profile="..."
	report="..."
	excludelist="..."
	includelist="..."
	debug="..."
	eefile="..."
	processunresolvedbundles="..."
/>

参数

底线 此属性指定参考基线的位置。

它可以是.zip，.jar，.tgz，.tar.gz文件或与Eclipse安装文件夹对应的目录。这是您可以找到Eclipse可执行文件的目录。

使用绝对路径指定位置。 		是

excludelist 设置排除列表位置。

排除列表位置指定列出要从分析中排除的包的文本文件。默认情况下，将分析配置文件中的所有捆绑包。包含和排除列表属性可用于有选择地包括和排除包。此外，此任务还支持在包含和排除列表中列出成员。包含列表对要分析的包和成员集合进行种子处理（省略时，包括所有包和成员），然后应用排除列表。该文件的每一行指定以下之一：
  • 特定的包名称
  • 特定的会员名称
  • 用于匹配包或成员名称的正则表达式（带有“R：”的行）
  • 评论（带有'＃'的行）

调试 设置调试值。

可能的值为： true ， false
默认值为false 。 		没有
eefile 设置要使用的执行环境文件。

缺省情况下，使用与JavaSE-1.6执行环境对应的执行环境文件。

此Wiki页面中描述了该文件的格式。

使用绝对路径指定文件。 		没有
processunresolvedbundles 设置是否应分析具有解析器错误的捆绑包。

如果设置，则分析中将包含具有解析程序错误（例如缺少依赖项）的包。错误可能会影响结果，因此xml输出中会包含解析程序错误列表，并且在报表转换期间会将警告添加到html中。

可能的值为： true ， false
默认为true 。 		没有

例子

	<apitooling.apifreeze
		baseline="D:\eclipse\3.4.1\eclipse"
		profile="D:\eclipse-SDK-I20081118-0800-linux-gtk.tar.gz"
		report="D:\report\report.xml"
		excludelist="D:\exclude_list.txt"
		debug="true"
	/>

这将在文件夹report.xml运行创建D:\report的任务。它将使用排除列表D:\exclude_list_external.txt来减少要报告的问题数。

由于启用了调试，因此一些调试跟踪将显示在Ant控制台中。
