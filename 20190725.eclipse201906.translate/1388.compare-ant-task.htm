比较Ant任务

目的

此任务运行API配置文件相对于基线的完整比较。

用法

描述

Ant任务的名称是： apitooling.compare 。要使用，jar文件apitooling-ant.jar必须位于Ant类路径上。

<apitooling.compare
	baseline="..."
	profile="..."
	report="..."
	excludelist="..."
	includelist="..."
	components="..."
	visibility="..."
	debug="..."
/>

参数

属性 描述 需要
底线 此属性指定参考基线的位置。

它可以是.zip，.jar，.tgz，.tar.gz文件或与Eclipse安装文件夹对应的目录。这是您可以找到Eclipse可执行文件的目录。

使用绝对路径指定位置。 		是
轮廓 此属性指定要与参考基线进行比较的当前产品或配置文件的位置。

它可以是.zip，.jar，.tgz，.tar.gz文件或与Eclipse安装文件夹对应的目录。这是您可以找到Eclipse可执行文件的目录。

使用绝对路径指定位置。 		是
报告 设置生成报告的输出位置。

任务完成后，此目录中将显示名为“compare.xml”的报告。

使用绝对路径指定位置。 		是
excludelist 设置排除列表位置。

排除列表位置指定列出要从分析中排除的包的文本文件。默认情况下，将分析配置文件中的所有捆绑包。包含和排除列表属性可用于有选择地包括和排除包。包含列表对要分析的包集合进行种子处理（省略时，包括所有包），然后应用排除列表。该文件的每一行指定以下之一：
  • 特定的包名称
  • 一个正则表达式，以匹配包名称（行与“R：”）
  • 评论（带有'＃'的行）

使用绝对路径指定位置。
以下是排除列表的示例：
 # DOC BUNDLES
 org.eclipse.jdt.doc.isv
 org.eclipse.jdt.doc.user
 org.eclipse.pde.doc.user
 org.eclipse.platform.doc.isv
 org.eclipse.platform.doc.user
 # NON-ECLIPSE BUNDLES
 com.ibm.icu
 com.jcraft.jsch
 javax.servlet
 javax.servlet.jsp
 # Exclude all pde bundles
 R:org\.eclipse\.pde\..*
 ...
如果行以“R：”开头，则排除列表可以包含正则表达式。 		没有
includelist 设置包含列表位置。

包含列表位置指定列出要包括在分析中的包的文本文件。默认情况下，将分析配置文件中的所有捆绑包。包含和排除列表属性可用于有选择地包括和排除包。包含列表对要分析的包集合进行种子处理（省略时，包括所有包），然后应用排除列表。该文件的每一行指定以下之一：

调试 设置调试值。

可能的值为： true ， false
默认值为false 。 		没有
组件 设置组件的列表位置。

组件列表包含必须比较的所有组件名称。如果省略，则比较给定基线中的所有组件。
使用绝对路径指定位置。
组件列表文件的格式如下所示：

能见度 设置可见性值。

可能的值为： API ， ALL
默认是API 。		 没有

例子

	<apitooling.compare
		baseline="D:\eclipse\3.4.1\eclipse"
		profile="D:\eclipse-SDK-I20081118-0800-linux-gtk.tar.gz"
		report="D:\reports\xml"
		excludelist="D:\exclude_list_external.txt"
		debug="true"
	/>

这将在文件夹compare.xml运行创建D:\reports\xml文件的任务。它将使用排除列表来减少要报告的问题数量。

由于启用了调试，因此一些调试跟踪将显示在Ant控制台中。
