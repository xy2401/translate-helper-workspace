CDT概述

C / C ++开发工具包（CDT）是一组Eclipse插件，为Eclipse工作台提供C和C ++扩展。有关Eclipse的更多信息，请参阅“ Workbench用户指南”>“概念”>“Workbench” 。

完整的CDT提供了一个C / C ++ IDE，它简化了许多可以从命令行使用的相同工具。CDT还可以与许多外部实用程序通信并解释其响应，例如：

  • 构建（例如make）。
  • 编译（如gcc）。有关更多信息，请访问http://gcc.gnu.org 。
  • 调试（如gdb）。有关更多信息，请参阅http://sources.redhat.com/gdb/ 。
注意：虽然make，gcc和gdb是文档中使用的示例，但几乎可以使用任何类似的工具或实用程序集。

独立调试器应用程序是由CDT的各种Eclipse插件构建的，这些插件与Debug和Edit有关。不包括与Build和Compile相关的插件。虽然可以在独立调试器中编辑文件，但任何更改都需要重建，这必须在调试器外部执行。如果你想要一个完整的C / C ++ IDE，那么建议使用完整的CDT，而不是在其中包含C / C ++调试器。
