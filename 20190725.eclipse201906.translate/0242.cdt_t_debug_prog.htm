调试程序

要调试程序，请使用调试启动配置。调试配置包含诸如正在调试的可执行文件，如何运行可执行文件以及调试器本身的参数（在本例中为：gdb）之类的信息。一旦创建了启动配置，就可以再次使用它（例如，在调试会话终止后重新启动可执行文件）。默认情况下，当您在命令行上指定可执行文件或使用New Executable ...对话框时，独立调试器会为您创建调试启动配置。

有多种调试启动配置类型。最常见的是用于运行本地应用程序的本地C / C ++应用程序配置，这是当您以上述任一方式指定可执行文件时，独立调试器将为您设置的配置。本地C / C ++启动配置通常与包含您要调试的可执行文件的Eclipse项目相关联。您不必创建Eclipse项目，因为独立调试器还会代表您创建Eclipse项目，作为导入可执行文件的一部分。默认情况下，这通常是为此目的而预留的特殊Executables项目，但如果您同时导入了多个可执行文件，则将创建其他ExecutablesNNNN项目以将每个可执行文件保存在单独的项目中。这样做是为了帮助索引器跟踪可执行文件及其所有相关的源代码。

您可能希望使用其他C / C ++调试配置类型，例如：

  • C / C ++附加到应用程序
  • C / C ++ Postmortem Debugger

这些额外的启动配置要求您手动创建它们。去做这个：

  1. 在“C / C ++项目”视图中，选择一个项目。启动独立调试器时，默认情况下不显示C / C ++ Projects视图。您可以使用“ 窗口”>“显示视图”>“C / C ++”>“C / C ++项目”来访问它。如上所述，在大多数情况下，您只能看到Executables项目。
  2. 单击“运行”>“调试配置...” 。
  3. 在“调试配置”对话框中，从“ 配置”列表中选择所需的调试配置类型。

  4. 单击左上角的“ 新建”按钮。

  5. 在“ 名称”框中，键入此调试配置的描述性名称。
  6. 在“ 项目”框中，键入包含要调试的应用程序的项目的名称。如果您选择了一个项目，那么这个项目将为您填写。几乎在所有情况下，这都是可执行文件项目。
  7. 在“ C / C ++应用程序”框中，键入要运行的可执行文件的名称。这必须是已导入项目的应用程序，或者在“附加到应用程序”配置的情况下，此字段可以为空。
  8. 单击Debug 。

    将打开调试透视图，并在顶部打开应用程序窗口。在附加配置的情况下，将弹出一个对话框，允许您选择正在运行的应用程序。

欲获得更多信息：

  • 要指定应用程序使用的执行参数以及运行配置的工作目录，请参阅指定执行参数 。
  • 要设置应用程序运行时要使用的环境变量和值，请参阅设置环境变量
  • 要选择调试应用程序时要使用的调试器 ，请参阅选择调试器
  • 要指定调试C或C ++应用程序时使用的源文件的位置 ，请参阅指定源文件的位置
  • 要指定运行配置的存储位置，访问方式以及运行应用程序时要打开的透视图，请参阅指定运行配置的位置

要使用调试配置：

您可以重用以前创建的调试启动配置来调试程序。

  1. 单击运行>调试 。
  2. 在“调试”对话框中，从“配置”列表中选择调试配置。
  3. 单击Debug 。
