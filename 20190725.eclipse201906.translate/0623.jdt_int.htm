JDT程序员指南

Eclipse平台提供了功能齐全的Java集成开发环境（IDE）。Java开发工具（JDT）允许用户编写，编译，测试，调试和编辑用Java编程语言编写的程序。

JDT使用了Platform Plug-in Developer Guide中描述的许多平台扩展点和框架。最简单的方法是将JDT视为一组插件，这些插件将Java特定行为添加到通用平台资源模型，并将Java特定视图，编辑器和操作贡献给工作台。

本指南讨论了JDT提供的扩展点和API。我们假设您已经了解了插件，扩展点，工作区资源和工作台UI的概念。

鉴于JDT提供了一个功能齐全的Java IDE，为什么还需要使用JDT API？如果要构建与Java程序或资源交互的插件作为其功能的一部分，则可能需要执行以下一项或多项操作：

  • 以编程方式操作Java资源，例如创建项目，生成Java源代码，执行构建或检测代码中的问题。
  • 以编程方式从平台启动Java程序
  • 提供新类型的VM启动程序以支持新的Java运行时系列
  • 向Java IDE本身添加新功能和扩展

JDT分为三个主要组成部分：

  • JDT Core - 用于编译和操作Java代码的无头基础结构。
  • JDT UI - 提供IDE的用户界面扩展。
  • JDT Debug - 特定于Java编程语言的程序启动和调试支持。
