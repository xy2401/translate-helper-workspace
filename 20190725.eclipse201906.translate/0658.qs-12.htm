运行你的程序

在本节中，您将了解有关在工作台中运行Java程序的更多信息。

  1. 在Package Explorer视图中，找到junit.textui / TestRunner.java并双击它以在编辑器中打开它。
  2. 在Outline视图中，请注意TestRunner类有一个图标，表示该类定义了一个main方法。

  3. 右键单击Package Explorer中的TestRunner.java，然后选择Run As > Java Application 。这将启动所选类作为本地Java应用程序。“ 运行方式”上下文菜单项也可用于其他位置，例如“大纲”视图。

  4. 请注意，程序已完成运行，并且Console视图中显示以下消息，告知您程序需要执行参数。从Package Explorer作为Java应用程序运行类使用默认设置来启动所选类，并且不允许您指定任何参数。

  5. 要指定参数，请使用工具栏中的下拉运行菜单，然后选择运行配置....

    您也可以在下拉菜单中按Ctrl +单击配置以开始编辑该配置。
  6. 这次，将打开“启动配置”对话框，并选择TestRunner启动配置。启动配置允许您配置程序的启动方式，包括其参数，类路径和其他选项。（选择“运行”>“Java应用程序”时，为您创建了默认启动配置）。

  7. 选择Arguments选项卡，然后键入junit.samples。程序参数区域中的VectorTest 。

  8. 单击运行 。这次程序正确运行，表明运行的测试数量。

  9. 切换到Debug透视图。在“调试”视图中，请注意在程序运行时注册了上次程序启动的进程。

    默认情况下，Debug视图会在创建新启动时自动删除任何已终止的启动。可以在上配置此首选项 运行/调试>启动首选项页面

    注意：您可以通过从其上下文菜单中选择“ 重新启动”来重新启动已终止的进程。
  10. 从工作台工具栏的“运行”按钮中选择下拉菜单。此列表包含以前启动的程序。可以通过在历史列表中选择这些程序来重新启动这些程序。

  11. 默认情况下，当命中运行按钮时，将启动当前选定的资源或活动编辑器。如果这些都不可启动，则将启动当前项目。您可以在下配置此行为 运行/调试>启动 。

  12. 从“调试”视图（或等效工具栏按钮）的上下文菜单中，选择“ 全部终止”以清除已终止的启动过程的视图。
