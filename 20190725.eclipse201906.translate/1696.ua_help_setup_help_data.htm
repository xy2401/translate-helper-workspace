帮助数据

标识符：
org.eclipse.help。HELP_DATA
以来：
3.3
描述：

产品使用帮助数据XML文件来控制帮助目录中的书籍顺序，以及是否应该显示书籍或关键字索引集。必须使用org.eclipse.help/HELP_DATA属性在产品的plugin_customization.ini文件中引用该文件。

配置标记：

<！ELEMENT 扩展 （ tocOrder ?, 隐藏 ？）>

帮助的扩展数据。



<！ELEMENT tocOrder （ toc | category ）*>

指定顶级目录条目（也称为“书籍”）或书籍类别应出现在“帮助”中的顺序。如果列出的其中一个项目不可用，则会被忽略。如果有可用的项目未列出且未隐藏，则会在此处列出的项目之后显示。



<！ELEMENT toc EMPTY>

<！ATTLIST toc

id CDATA #REQUIRED>

对顶级目录（TOC）条目的引用，也称为“书”。


  • id - 本书的唯一标识符。对于XML文件TOC贡献，这是文件的路径，格式为“ /<plugin_id>/<path>/<file> ”（例如，“ /<plugin_id>/<path>/<file> /org.eclipse.platform.doc.user/toc.xml ”） 。通常，这是由其原始AbstractTocProvider提供的TocContribution的ID。

<！ELEMENT 类别 EMPTY>

<！ATTLIST类别

id CDATA #REQUIRED>

对顶级目录（TOC）条目（书籍）类别的引用。当目录贡献声明自己属于该类别时，隐式创建类别，例如，通过在org.eclipse.help.toc扩展点中指定toc元素的category属性。


  • id - 类别的唯一ID。

<！ELEMENT 隐藏 （ toc | category | index ）*>

包含应该向用户隐藏的一组帮助项。



<！ELEMENT 索引 EMPTY>

<！ATTLIST指数

id CDATA #REQUIRED>

对帮助索引关键字的贡献的引用。


  • id - 此关键字贡献的唯一标识符。对于XML文件索引贡献，这是文件的路径，格式为“ /<plugin_id>/<path>/<file> ”（例如，“ /<plugin_id>/<path>/<file> /org.eclipse.platform.doc.user/index.xml ”） 。通常，这是由其原始AbstractIndexProvider提供的IndexContribution的ID。

例子：

以下示例显示如何按所示顺序排列以下书籍：

  • 书＃1：导论“XYZ”（类别： “ user.intro ”）在/com.xyz.doc.user/introToc.xml
  • 第2册： /com.xyz.doc.user/usingToc.xml “使用XYZ”（类别：“ user.content ”）
  • 书＃3： “故障排除”（类别： “ user.reference ”）在/com.xyz.doc.user/refToc.xml

除了隐藏以下书籍/类别和相关的关键字索引：

  • 第4册： /org.abc.doc.isv/toc.xml “平台ABC”（类别： none ）
  • 书＃5： “DEF工具包”（类别： “ isv.reference ”）在/com.def.doc.isv/toc.xml
  • 图书＃6： “GHI支持”（类别： “ isv.reference ”）在/com.ghi.doc.isv/toc.xml

标记如下：

<extensions>
   <tocOrder>
      <toc id="/com.xyz.doc.user/introToc.xml"/>

      <category id="user.content"/>
      <toc id="/com.xyz.doc.user/refToc.xml"/>
   </tocOrder>
   <hidden>
      <toc id="/org.abc.doc.isv/toc.xml"/>

      <category id="isv.reference"/>
      <index id="/org.abc.doc.isv/index.xml"/>
      <index id="/com.def.doc.isv/index.xml"/>
   </hidden>

</extensions>
提供的实施：
任何基于org.eclipse.help帮助实现都支持此API，包括Eclipse提供的默认帮助实现。
