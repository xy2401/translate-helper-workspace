Java Build Path页面

您可以在第一页上按“ 完成”跳过此页面。否则，请按Next以配置Java构建路径。

Java构建路径由源，库和项目条目组成。除源选项卡外，用户界面与Build Path属性页的用户界面相同：

来源标签

源文件夹是项目层次结构中的顶级文件夹。它们是包含.java文件的包的根。编译器会将包含的文件转换为将写入输出文件夹的.class文件。

源文件夹允许构建项目，例如将测试与两个源文件夹中的应用程序分开。在源文件夹中，可以通过使用包来实现更详细的结构。

每个源文件夹都可以定义排除过滤器，以指定文件夹中的哪些资源不应对编译器可见。

源文件夹中存在的资源将复制到输出文件夹，除非Java>编译器>构建首选项页面中的设置指定资源已过滤。输出文件夹是按项目定义的，除非源文件夹指定了自己的输出文件夹。

该树显示了切换到包资源管理器时的项目。可以在此树上执行多个操作以更改项目的结构。

图标 选项 描述

包括 包括先前排除的资源。
配置源文件夹属性

编辑源文件夹属性菜单有两个操作

  1. 配置包含/排除过滤器 ：通过定义字符串模式自定义包含和排除过滤器。可以在模式中使用通配符（即排除以“Test”开头的所有java文件写入“Test * .java”）。
  2. 配置输出文件夹 ：更改源文件夹的输出文件夹。仅当启用了允许源文件夹的输出文件夹时，才会启用此操作。
撤消所有更改 将撤消已在此向导中应用于项目的所有更改，并重建项目的原始状态。
  允许源文件夹的输出文件夹 如果启用，则每个源文件夹都可以拥有自己的输出文件夹。否则，所有源文件夹都将使用默认输出文件夹。

在项目树下面的“ 详细信息”窗格中可以看到所有操作的简短描述。
