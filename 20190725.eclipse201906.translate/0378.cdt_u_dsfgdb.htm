GDB调试首选项

使用此首选项面板控制CDT调试器在使用GDB进行调试时的行为，特别是在使用GDB（DSF）启动程序时。

GDB首选项面板选项
类别 选项 描述
调试配置默认值
GDB调试器 指定默认情况下将为每个新创建的调试启动使用的GDB调试器二进制文件。使用绝对路径将直接指向二进制文件（例如，/ home / user / myGdb），而使用二进制名称将使CDT查看该二进制文件的PATH变量（例如，myGdb）。这可能对您的GDB二进制文件没有名为'gdb'有用。每个启动配置都允许在“调试器”选项卡中覆盖此设置。
GDB命令文件 指定默认情况下将为每个新创建的调试启动使用的GDB调试器命令文件。如果您经常/总是希望使用GDB命令文件进行启动，这将非常有用。每个启动配置都允许在“调试器”选项卡中覆盖此设置。
启动时停止 选中后，CDT调试器将在启动或重新启动进程时停止在指定符号处执行。默认情况下，此设置使CDT调试器在输入“main”时停止。
命令超时 选中时，如果发送到GDB的任何命令在指定的超时之前没有得到答案，则CDT调试器将中止调试会话。使用它可以防止调试会话在调试不可靠的目标时永远挂起。

使用“ 高级...”按钮可以为各个命令设置不同的超时。值零可用于为特定命令指定“无超时”。MI命令必须以连字符（' - '）开头。例如，'-target-select'。
不间断模式 选中后，CDT调试器将在每次新创建的调试启动时以非停止模式运行。不间断模式允许独立控制调试会话的每个线程。这允许在保持其他线程运行的同时停止一个或多个线程。相反， 全停模式会同时中断并恢复所有线程。请注意， 不间断模式仅在GDB 7.0开始时可用。
一般行为
上次进程退出时终止GDB 选中后，CDT调试器将在调试的所有目标进程终止后终止GDB进程（在主机上）。否则会让它继续运行。如果您希望在先前调试的进程终止后希望附加或创建新进程以进行调试，那么保持GDB运行会很有用。
使用增强的调试悬停 选中时，在调试会话期间将鼠标悬停在编辑器中的表达式上将显示增强的表达式评估控件。否则，使用更基本的控制。当然，这假设您已在C / C ++>编辑器>悬停中启用了调试器的悬停功能。如果没有，则此复选框无效。
在Debug视图中仅显示挂起的线程 选中时，Debug视图将仅显示已挂起的线程。处理大量线程时，这有助于关注正在检查的线程。显然，此选项仅在非停止模式下有价值，其中某些线程可以在其他线程停止时运行。
使用积极的断点过滤 选中时，“断点”视图的“显示所选目标支持的断点”按钮将仅显示影响当前所选C / C ++调试上下文的断点（例如，线程，进程）。否则，“显示所选目标支持的断点”将显示适用于任何C / C ++目标的所有断点。
启用GDB跟踪 选中后，调试会话将生成一个包含所有gdb / mi活动的附加控制台。基本上，这详细说明了CDT调试器和GDB之间的交互。当调试器没有按预期运行时，此信息通常很关键。在针对CDT调试器提交bugzilla报告时包括此控制台输出。建议始终启用此设置，因为它不会产生任何负面影响。

一旦达到限制字段指定的字符数，将逐渐丢弃较旧的跟踪线。这是为了避免这些跟踪消耗太多内存，用于非常长的调试会话。
显示运行时类型的变量 选中后，CDT调试器将在变量和表达式视图中显示实际变量类型（不是声明的变量类型）。这不仅允许查看变量的当前运行时类型，还允许访问可能不在基类中的字段。请注意，此功能仅适用于从GDB 7.5开始。
漂亮的印刷
在变量/表达式树中启用漂亮的打印机 选中后，CDT调试器将以用户友好的方式显示STL类型。这可以在变量和表达式视图中以及高级编辑器悬停中看到。例如，地图，列表和向量将以数组样式格式显示，而不是显示数据结构的实际实现的详细信息。请注意， 漂亮的打印需要一个启用了python支持的GDB和STL漂亮打印机的用户。
对于集合，最初将子计数限制为 使用漂亮的打印时，集合（例如，地图，列表等）可以包含大量元素。试图同时显示所有这些孩子会导致响应能力非常差。此值将限制最初显示的子项数，同时允许用户直接从视图中手动请求更多子项。
