项目属性页面 - JPA

使用“ 属性”页面上的JPA选项选择要与项目一起使用的数据库连接。


注意：

必须激活连接才能获得特定于数据源的帮助和验证。

此表列出了JPA选项首选项页面中可用的属性。

属性 描述

平台

选择特定于供应商的平台。

JPA实施

类型

选择用户库以从可用的用户定义或下载的库中进行选择。

如果选择“禁用”，则必须在项目类路径上手动包含JPA实现库。

图书馆

选择特定的JPA库配置。

单击“ 管理库”以创建或更新用户库。

单击“ 下载库”以下载特定的库配置。

包含此应用程序的库

指定在部署应用程序时是否包含所选库。

连接

用于映射持久性实体的数据库连接。

  • 要创建新连接，请单击“ 添加连接” 。

  • 要重新连接到现有连接，请单击“ 重新连接” 。

从连接覆盖默认目录

选择从连接信息派生的默认目录以外的目录。如果默认目录不正确或无法使用，请使用此选项。

从连接覆盖默认架构

选择从连接信息派生的默认模式以外的模式。如果默认架构不正确或无法使用，请使用此选项。例如，当部署登录与设计时登录不同时，请使用此选项。

持久的班级管理

指定Dali是否将自动发现带注释的类 ，或者是否必须在persistence.xml文件中列出带注释的类 。

注意 ：为确保应用程序的可移植性，您应该明确列出持久性单元中包含的托管持久性类。

规范化模型

选择元模型源的位置。
