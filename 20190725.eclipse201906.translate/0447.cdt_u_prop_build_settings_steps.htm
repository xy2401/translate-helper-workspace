C / C ++属性，构建，设置，构建步骤选项卡

使用“ 构建步骤”属性选项卡自定义所选的构建配置，以允许指定用户定义的构建命令步骤，以及在构建输出中，在正常构建处理执行之前和之后立即显示描述性消息。

注意：为确保合理的自定义构建步骤行为，在指定自定义构建步骤信息时必须提供合理的输入。自定义构建步骤在任何方面都不会验证其正确性，并且完全按照输入到构建流中的方式传递。

在下面的描述性文本中，术语“主构建”被定义为在调用构建时要执行的命令序列，不包括预构建或构建后步骤。

构建步骤选项
组 描述
组态 点击此处查看说明。
管理配置...... 点击此处查看说明。
预建步骤 标识在构建之前要发生的任何步骤。

注意：如果确定主构建的状态是最新的，则不执行预构建步骤;否则它被执行。无论执行预构建步骤的成功与否，都将尝试执行主构建。
命令 指定在执行构建之前立即执行的一个或多个命令。使用分号分隔多个命令。
描述 指定与预构建步骤关联的可选描述性文本，该步骤在执行预构建步骤命令（或多个命令）之前立即显示在构建输出中。
构建后的步骤 标识构建发生后要执行的所有步骤。

注意：如果确定主构建的状态是最新的，则不执行构建后步骤。只有在main-build成功执行时才会执行它。
命令 指定在执行构建后立即执行的一个或多个命令。使用分号分隔多个命令。
描述 指定与生成后步骤关联的可选描述性文本，该步骤在执行生成后步骤命令（或命令）后立即显示在生成输出中。

适用于： CDT项目。
