定义API限制

API Tools提供Javadoc标记以明确记录和限制API的使用。下表总结了每个成员支持的Javadoc标记以及每个标记的语义。

客户端引用需要绑定的插件或包，其中定义了关联的API。限制不会应用于定义API的同一个包中。例如，还允许将接口定义为@noimplement的bundle提供该接口的实现。

限制语义

@noimplement

表示客户端不得实现此接口。任何使用implements关键字的类关联接口或没有实现超类的关联接口的父级都将被标记为有问题。
@noextend
表示客户端不得扩展此类或接口。使用相关类型的extends关键字的任何类或接口都将标记有问题。
@noinstantiate
表示客户端不得实例化此类。任何使用任何构造函数实例化关联类的代码都将被标记为有问题。
@nooverride
表示客户端不得重新声明此方法。定义覆盖关联方法的方法的任何子类都将标记有问题。
@noreference

指示客户端不得引用此类型（类，接口，枚举或注释），方法，构造函数或非最终字段。任何直接调用相关方法或构造函数或引用相关非最终字段的代码都将标记有问题。

当标签用于某个类型时，它的行为就像标签被添加到任何类型的成员一样。例如，将标记添加到类将标记对该类的任何方法或非最终字段的任何引用。
