JDT核心选项

JDT Core选项控制核心功能的行为，例如Java编译器，代码格式化程序，代码辅助和其他核心行为。用于访问选项的API在JavaCore中定义。选项可以按组访问，如下所示：

  • JavaCore.getDefaultOptions（） - 回答选项的默认值。
  • JavaCore.getOptions（） - 回答选项的当前值。
  • JavaCore.setOptions（Hashtable newOptions） - 用新值替换选项值。

也可以通过字符串名称单独访问选项。

  • JavaCore.getOption（String optionName） - 回答特定选项的值。

还可以找到可以配置问题严重性的选项。

  • JavaCore.getOptionForConfigurableSeverity（int problemID） - 回答用于配置由problemID标识的问题严重性的problemID 。

选项存储为所有已知可配置选项及其值的哈希表。已在JavaCore上为每个选项ID及其可能的常量值定义了助手常量。

非空类型的注释类型的名称 （ COMPILER_NONNULL_ANNOTATION_NAME ）
此选项定义完全限定的Java类型名称，编译器可以使用该名称执行特殊的空值分析。

如果此选项指定的注释应用于方法签名或变量声明中的类型，则这将被解释为null在该位置不是合法值的规范。当前支持的位置是：方法参数，方法返回类型和局部变量。

对于使用此批注声明的值，编译器将永远不会触发空引用诊断（由COMPILER_PB_POTENTIAL_NULL_REFERENCE和COMPILER_PB_NULL_REFERENCE控制），因为假设在运行时这些位置永远不会出现null。

编译器还可以检查对空指定的遵守情况，进一步由COMPILER_PB_NULL_SPECIFICATION_VIOLATION ， COMPILER_PB_NULL_ANNOTATION_INFERENCE_CONFLICT和COMPILER_PB_NULL_UNCHECKED_CONVERSION控制 。

如果启用了COMPILER_ANNOTATION_NULL_ANALYSIS选项，则此选项仅起作用。
