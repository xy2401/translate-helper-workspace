Http服务和JSP支持

Equinox支持在RCP和应用程序服务器环境中构建和运行基于OSGi的Web应用程序。在OSGi Web应用程序中，使用OSGi Http Service取代了web.xml部署文件的使用。Http Service提供了一种以与Eclipse中的动态环境更加一致的方式注册servlet和资源的方法，其中bundle可以来来去去。

平台SDK与org.eclipse.equinox.http.jetty一起打包， org.eclipse.equinox.http.jetty是OSGi Http服务的一个实现，它使用Jetty作为其底层servlet容器。此实现允许自定义和配置对集成很重要的许多服务器设置。

Java Server Pages在许多Java服务器环境中发挥着核心作用，允许呈现动态内容。JSP的动态编译由org.eclipse.equinox.jsp.jasper的JSPServlet提供 。

Http服务扩展点

在许多情况下，使用扩展注册表提供的更具声明性的交互方式更容易。为了支持这种使用方式， org.eclipse.equinox.http.registry为httpcontexts ， servlet和资源提供了扩展点，允许在Http服务中等效注册Web资源。

org.eclipse.equinox.jsp.jasper.registry中的 JSP扩展工厂类提供了JSP支持，以便与servlets扩展点一起使用。
