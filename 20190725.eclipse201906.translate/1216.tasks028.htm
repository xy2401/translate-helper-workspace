错误消息

本节包含使用Dali时可能遇到的错误消息（包括如何解决问题）的信息。

应指定异常处理程序类。

使用自定义异常处理程序时，必须选择（或创建）Java类来处理异常。请参阅“自定义” 。

属性“ <ATTRIBUTE__NAME> ”在此上下文中具有无效的映射类型

映射的属性无效。更改映射类型或更改实体类型。

有关更多信息，请参阅“映射实体” 。

无法解析属性“ <ATTRIBUTE_NAME> ”。

Dali无法将属性映射到数据库表和列。验证数据库连接信息是否正确。

有关更多信息，请参阅“创建新的JPA项目” 。

类“ <CLASS_NAME> ”未注释为持久类。

该类尚未被识别为持久类。将类配置为Entity，Mapped Superclass或Embeddable persistent entity。

请参阅“为类添加持久性” 。

无法解析列“ <COLUMN_NAME> ”。

您将实体的字段映射到数据库表中的不正确或无效的列。默认情况下，Dali将尝试使用数据库表中具有相同名称的行映射实体中的每个字段。如果字段的名称与行的名称不同，则必须显式创建映射。

将字段映射到数据库表中的有效行，如“映射实体”中所示 。

转换器未命名。所有转换器都需要名称。

创建转换器时，必须指定其名称。请参阅“转换器” 。

转换器名称不能是保留的转换器名称。

创建转换器时，您必须使用以下保留名称：

  • 连载

  • 类实例

  • 没有

复制类“ <CLASS_NAME> ”。

您创建了具有相同名称的持久性类。每个Java类必须具有唯一的名称。有关详细信息，请参阅“为类添加持久性” 。

实体没有ID或嵌入式ID。

您创建了一个持久性实体，但未标识其主键。持久化实体必须具有使用@Id或@EmbeddedId注释指定的主键字段。

向实体添加ID映射，如“ID映射”或“嵌入式ID映射”中所示 。

名为“<GENERATOR_NAME>”的多个生成器在此持久性单元中定义。

创建生成器时，转换器名称在持久性单元中必须是唯一的。请参阅“主键生成” 。

项目中有多个persistence.xml文件。

您创建了一个包含多个persistence.xml文件的JPA项目。每个JPA项目必须包含一个单一 persistence.xml文件。

有关更多信息，请参阅“管理persistence.xml文件” 。

名为“ <CONVERTER_NAME> ”的多个转换器在此持久性单元中定义

创建转换器时，转换器名称在持久性单元中必须是唯一的。请参见“添加转换器对话框” 。

没有定义持久性单元。

persistence.xml文件中没有定义持久性单元。使用<persistence-unit name =“ <PERSISTENCE_UNIT_NAME> ”标记来定义持久性单元。

有关更多信息，请参阅“管理orm.xml文件” 。

项目中没有persistence.xml文件。

您创建了一个没有persistence.xml文件的JPA项目。每个JPA项目必须包含一个单一 persistence.xml文件。

有关更多信息，请参阅“管理persistence.xml文件” 。

由于shared-cache-mode设置为NONE，属性“<PROPERTY_NAME>”将被忽略。

由于共享缓存模式选项设置为NONE ，因此Dali将忽略该属性。请参阅“缓存” 。

无法解析连接列“ <COLUMN_NAME> ”中的引用列“ <COLUMN_NAME> ”。

您选择加入关系映射的列在数据库表中不存在。在“ 连接策略”上选择不同的列，或在数据库表上创建必要的列。

有关详细信息，请参阅“JPA详细信息视图（属性）” 。

无法为表/连接表“< TABLE_NAME >”解析模式“< SCHEMA_NAME >”。

在持久性单元中定义默认数据库架构信息。

有关更多信息，请参阅“管理orm.xml文件” 。

表“ <TABLE_NAME> ”无法解析。

您将持久性实体与不正确或无效的数据库表相关联。默认情况下，Dali将尝试将每个持久性实体与具有相同名称的数据库表相关联。如果实体的名称与表的名称不同，则必须显式创建关联。

将实体与有效的数据库表相关联，如“向类中添加持久性”中所示 。

实体<ENTITY_NAME>上的@Cache注释同时指定了expiry（）和expiryTimeOfDay（）。

您试图在@Cache批注中包含expiry和expiryTimeOfDay 。你可能只使用一个。请参阅“缓存” 。

项目类路径中不存在转换器类“<CLASS_NAME>” 。

您定义了转换类，但未在项目中包含该类。请参阅“转换器” 。

转换器类“ <CLASS_NAME> ”未实现org.eclipse.persistence.mappings.converters。转换器接口

创建转换器时，其类必须实现org.eclipse.persistence.mappings.converters.Converter接口。请参阅“转换器” 。

必须定义转换器类。

您试图在不定义类的情况下使用转换器。请参阅“转换器” 。

实体定制程序类“<CLASS_NAME>”未实现org.eclipse.persistence.config。DescriptorCustomizer接口。

将customer类用于实体时，该类必须实现org.eclipse.persistence.config.DescriptorCustomizer接口。见“高级”

异常处理程序类“ <CLASS_NAME> ”未实现org.eclipse.persistence.exceptions。ExceptionHandler接口。

使用自定义异常处理程序时，必须选择（或创建）实现org.eclipse.persistence.exceptions.的Java类org.eclipse.persistence.exceptions.ExceptionHandler类。请参阅“自定义” 。

版本映射的持久字段或属性必须是int，Integer，short，Short，long，Long或Timestamp类型。

版本映射只能与以下属性类型一起使用：

  • INT

  • 整数

  • 简短，短

  • 很久很久

  • 时间戳

请参阅“版本映射” 。

结构转换器类“ <CLASS_NAME> ”未实现org.eclipse.persistence.platform.database.converters。StructConverter接口。

创建Struct转换器时（启用java.sql.自定义处理） java.sql.Struct类型），其类必须实现org.eclipse.persistence.mappings.converters.StructConverter接口。请参阅“转换器” 。

未解析的生成器“ <GENERATOR_NAME> ”在持久性单元中定义。

您创建了一个使用排序或表生成器的持久性实体，但没有在持久性单元中定义生成器。通过使用注释定义生成器或将其包含在XML映射文件中。
