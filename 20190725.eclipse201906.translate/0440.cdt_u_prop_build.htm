C / C ++项目属性，C / C ++构建

此页面用作包含所有特定于构建器的属性页面的主窗口。此外，您可以直接从此窗口定义Builder设置和行为属性的首选项。

注意：修改某些设置（例如“ 自动生成makefile”选项）可能会影响其他参数（在某些情况下将它们从启用设置为禁用），此外，还会更改其他属性页的可见性。


Builder设置首选项

选项 描述
组态 点击此处查看其说明。
管理配置 点击此处查看其说明。
生成器类型 指定要使用的构建器的类型： 内部构建器 （使用实现C / C ++语言规范的编译器构建C / C ++程序）和外部构建器 （外部工具允许您使用Workbench配置和运行程序和Ant构建文件，可以保存并在以后运行以执行构建）。
使用default build命令 启用时，此选项表示您要使用默认的make命令。禁用时，它表示使用新的make命令。此选项仅在“ 构建类型”选项设置为“ 外部”时可用。
构建命令 指定用于启动特定工具链的构建实用程序的默认命令。如果要使用除默认make命令之外的构建实用程序，则使用此字段（当未选择使用默认构建命令时，该字段处于活动状态，并且当您使用外部构建器或自定义makefile时，您可以提供特定命令）。
变量 打开“选择构建变量”对话框，您可以在其中将环境变量和自定义变量添加到构建命令。


自动生成Makefile 选择后，Eclipse会在两种不同的CDT模式之间切换：它使用客户的makefile进行构建（如果存在，则称为CDT 3.x中的“标准制作项目”），或者为用户生成makefile（称为' CDT 3.x中的托管构建项目。默认情况下，会自动设置此选项。
展开环境Makefile中的变量参考 定义是否应在makefile中扩展环境变量（$ {xxx}）。默认情况下，会自动设置此选项。
构建目录 定义构建操作发生的位置。此位置将包含构建过程中生成的工件。启用“ 自动生成Makefile”选项时，将禁用此选项。
工作区 打开“文件夹选择”对话框，您可以在其中选择项目的工作空间位置。这是包含要构建的插件和功能的目录，包括任何生成的工件。仅当未选择自动生成makefile时，此按钮才可见。
文件系统 打开文件系统导航器，您可以在其中指定要使用的其他文件系统。仅当未选择自动生成makefile时，此按钮才可见。
变量 打开“选择构建变量”对话框，您可以在其中选择要指定为构建目录的参数的变量，或者创建和配置可在支持变量的构建配置中引用的简单构建变量。仅当未选择自动生成makefile时，此按钮才可见。


行为偏好

选项 描述
停止第一次构建错误 Eclipse遇到错误时停止构建。

注意：如果未选择此选项，则有助于构建大型项目，因为它告诉make即使在一个规则失败时也继续制定其他独立规则。
使用并行构建 启用后，此选项表示激活并行构建的生成。

如果启用此选项，则需要确定要执行的并行作业数：

  • 使用最佳作业编号 - 允许系统确定要执行的最佳并行作业数。
  • 使用并行作业 - 允许您指定要执行的最大并行作业数。
Workbench构建行为 默认情况下，以下设置是构建器在指示构建，重建，清理等时使用的设置。您可以更改这些设置，以便新项目可以使用不同的目标（如果这些默认值不合适）。
以资源保存为基础（自动构建） 选中后，只要保存资源，就构建项目。默认情况下，选择此选项并在每次修改资源时自动生成构建。如果未选择此选项，则仅使用菜单项手动进行构建。如果您需要更多地控制何时进行构建（例如，构建应该等到您完成大量更改），请禁用此选项并自行手动调用构建。
make build target （用于构建资源保存（自动构建）） 要在保存资源时构建项目并更改默认的make build目标，请启用Build on resource save（自动构建）选项，并在Make build target字段中指定新的构建目标。
变量 打开“选择构建变量”对话框，您可以在其中向构建命令添加变量。

构建（增量构建） 定义执行增量构建时构建器调用的内容。启用此选项后，将发生增量构建，这意味着仅考虑自上次构建以来已更改的资源。如果禁用此选项，则会发生完整构建，这意味着将考虑构建范围内的所有资源。
make build target （用于Build（Incremental build）） 要更改构建默认的make build target，请启用Build（Incremental build）选项，并在Make build target字段中指定新的构建目标。
变量 打开“选择构建变量”对话框，您可以在其中将变量添加到make build target命令。
清洁 定义执行清理时构建器调用的内容。make clean在makefile中定义。
制作目标 （用于清洁） 要更改rebuild default make build target，请启用Clean选项，并在Make build target字段中指定新的构建目标。
变量 打开“选择构建变量”对话框，您可以在其中将变量添加到make build target命令。
