在你开始之前

根据您获得CDT的方式，您可能还收到了一个内置CDT集成的工具链。但是，如果您从Eclipse网站下载了CDT，那么在构建和调试任何项目之前，您将需要一个工具链。

标准CDT支持与GNU工具链的集成。这包括GNU的make，gcc编译器和gdb调试器实用程序。如果您需要工具链来为开发主机构建软件，这是开始使用的最佳选择。

运行CDT的每个平台都需要不同的步骤来获取此工具链。

视窗

对于Windows，MinGW和Cygwin是获取GNU工具链的两个主要平台选择。了解它们之间的区别非常重要。Cygwin生成使用Cygwin POSIX运行时的可执行文件。请注意，此运行时是GPL许可的。MinGW生成本机Windows可执行文件，不需要单独的运行时。

  • 对于MinGW ，建议使用MinGW安装程序mingw-get来管理MinGW安装。从MinGW下载页面下载并运行最新的mingw-get-inst软件包。如果CDT MinGW工具链位于默认的C：\ MinGW目录中，MinGW bin目录放在路径中，或者MinGW位置存储在MINGW_HOME环境变量中，则它将找到此安装。

    请注意，Wascana IDE已停产。

  • Cygwin的可以从Cygwin的现场安装http://www.cygwin.com 。您需要手动选择gcc，gdb和make包来安装工具链。

  • Windows SDK提供了创建Windows应用程序所需的Visual C ++编译器和头文件和库。CDT Visual C ++构建集成将根据您安装SDK的位置找到这些文件。无需其他设置。

    注意：对于此版本，集成应视为beta质量。不建议用于生产。

Linux的

所有Linux发行版都包含GNU工具链。但是，默认情况下可能不会安装它们。有关安装适用于Linux的GNU工具链的说明，请参阅特定发行版的说明。

Mac OS X.

CDT支持Xcode IDE中包含的Apple GNU工具链。它可以从Apple的开发者网站http://developer.apple.com下载

其他平台

CDT支持的所有平台都支持GNU工具链。有关在平台上安装GNU工具链的说明，请咨询您的平台供应商。
