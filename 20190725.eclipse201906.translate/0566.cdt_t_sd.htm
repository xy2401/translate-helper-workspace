设置包含C / C ++索引器的路径和宏

CDT通过代码突出显示，导航，内容辅助和静态分析提供舒适的环境。但是，大多数C / C ++项目必须配置为享受这些功能。大多数现实生活中的C / C ++项目在很大程度上依赖于项目本身之外的代码，例如系统头或第三方库。CDT索引器需要处理那些以提供准确的索引。如果未能正确设置这些重要设置，将导致辅助工具行为不当。

CDT将尝试为支持的工具链自动发现包含路径和预处理器符号。此过程称为扫描程序发现或自动发现 。这种符号的发现是双重的。一种方法是，CDT将尝试检测内置编译器符号，并包含使用特殊选项运行编译器的路径并解析输出。CDT采用的另一种方法是使用Build Output Parser分析常规构建的构建输出。通常，include路径使用-I选项提供给编译器，使用-D选项提供宏。这依赖于构建的详细构建输出，其中所有这些选项实际上都是由make打印的。

Scanner Discovery使用语言设置提供程序查找包含路径和预处理程序符号。语言设置提供程序可以在项目属性页面“预处理程序包含路径，宏等”，配置的提供程序选项卡和首选项页面上配置C / C ++首选项：共享提供程序的扫描程序发现 。

如果通过自动发现检索的信息不足或项目偏离CDT支持的标准信息，则用户可以检查发现的条目并在属性页“预处理器包含路径，宏等”上手动输入其他包含路径和宏， 条目标签 。

还有其他方法可以使C / C ++索引器识别包含路径或宏。一种方法是通过“路径和符号”项目属性在MBS中设置它们。请参阅在托管构建系统中包括路径和符号 。这些条目通过MBS语言设置提供程序提供给索引器。
