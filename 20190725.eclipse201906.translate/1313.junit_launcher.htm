JUnit插件测试启动器

PDE通过向SDK提供JUnit插件测试启动程序，为测试插件提供JUnit支持。与Eclipse中的所有其他启动程序 （例如Java应用程序和Java Applet启动程序 ......）一样，可以通过快捷方式调用JUnit插件测试启动程序，并在启动配置对话框中集中管理其启动配置 。

UI测试与无头测试

启动JUnit插件测试启动配置时，轻量级PDE应用程序是运行时Eclipse实例的主要入口点。如果用户正在测试的应用程序需要用户界面，则PDE应用程序会生成该应用程序的工作台。工作台启动后，测试工具将在工作台内部运行。运行所有测试后，PDE应用程序关闭工作台并退出;因此，终止测试运行。

如果正在运行的测试不需要用户界面（即无头模式），那么轻量级PDE应用程序将在不启动工作台的情况下运行测试。运行所有测试后，PDE应用程序退出。

有关如何运行不同类型的测试的详细信息，请参阅JUnit插件测试启动配置的主选项卡 。

通过快捷方式启动

运行JUnit插件测试的快捷方法是通过插件项目的上下文菜单中的Run As> JUnit Plug-in Test 。要以调试模式启动，请从插件项目的上下文菜单中选择Debug As> JUnit Plug-in Test 。这些操作也可以在插件项目内的源文件夹，包，源文件和方法的上下文菜单中使用。

通过快捷方式创建新的启动配置时，PDE会检查封闭插件的依赖关系。如果插件直接或间接需要org.eclipse.swt插件，则会自定义创建的启动配置以运行UI测试;否则，定制启动配置以运行无头测试。

自定义启动配置

为了完全控制启动JUnit插件测试 ，您可以在“ 启动配置”对话框中创建和自定义启动配置 。

可以通过顶级菜单中的Run> Run ...或Debug> Debug ...调用Launch Configuration对话框 。通过双击左侧树查看器中的JUnit Plug-in Test节点，可以创建新的JUnit Plug-in Test启动配置。

JUnit插件测试启动配置包含八个选项卡，允许您自定义测试启动的所有方面。有关更多详细信息，请参阅以下文档：
