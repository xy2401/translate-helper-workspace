使用共享许可证

许多Eclipse功能使用相同的许可证。例如，Eclipse SDK中的所有功能都使用Eclipse Foundation Software User Agreement。如果为使用PDE构建的每个功能添加了单独的许可证，更新它们将非常繁琐。使用共享许可证支持允许多个功能从单个位置引用许可证文本和文件。这使得跨多个功能更新许可证更容易，更不容易出错。

要创建许可证功能，请创建不包含任何捆绑包或功能的功能。添加必要的许可证文件，并确保它们包含在build.properties中。feature.properties应该有两个属性， licenseURL和license 。

要使用共享许可证功能，只需在功能编辑器中打开feature.xml文件即可。选择“ 信息”选项卡，然后选择“ 许可协议”部分选择共享许可证将允许您指定包含许可证文本和可选许可证文件的功能和版本。license-feature和license-feature-version属性将添加到feature.xml的feature元素中。如果要将单个许可证转换为共享许可证，则应从各个功能中删除许可证文件。应从feature.properties和build.properties中删除许可证信息。

新的共享许可证功能必须包含在构建中，因此请相应地更新地图文件。您不需要将该功能嵌套在另一个功能中以获取它，它会自动发生。不会在p2存储库中为许可证功能创建IU，但许可证文本将包含在p2元数据中。
