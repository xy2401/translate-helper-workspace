技巧和窍门

在PDE启动配置中显示命令行按钮 通过单击PDE启动配置对话框上的“ 显示命令行”按钮，可以查看用于启动应用程序的命令行 。

禁用API分析构建器 您可以在Plug-in Development首选项页面上禁用API分析构建器。

布局间谍工具 要调试SWT布局，可以使用任何窗口中的快捷键Ctrl + Alt + Shift + F9激活布局间谍工具。

用法：

  • 单击“ 选择控件”，然后单击要检查的控件。
  • 启用“ 显示叠加”并导航窗口小部件层次结构，直到找到边界正确的控件（红色矩形）且其子边界不正确（黄色矩形）。
  • 查看computeSize的结果。如果computeSize的结果正确，则问题出在父布局或其属性中。否则，问题出在子窗口小部件中。
更改问题严重性 您可以通过调用快速修复 （ Ctrl + 1 ）来配置API工具问题或PDE编译器问题的严重性，该快速修复将打开相应的首选项页面并突出显示可配置的问题。

Workspace插件覆盖目标插件 在Plug-in Development首选项页面上，有一个选项可指定工作区插件是否应覆盖具有相同ID的目标平台插件。默认情况下，启用此选项。禁用时，将使用工作区和目标平台中的所有插件版本，对于插件ID，将选择最佳可用插件。

支持OSGi声明性服务注释 OSGi Declarative Services为开发复杂的面向服务的应用程序提供了强大的机制。通过适当的工具，添加到Declarative Services的注释可以轻松生成和维护所需的组件描述符文件。

启用后，PDE会从适当注释的Java源自动生成和更新Declarative Services组件描述符文件。无需在项目中配置任何其他构建器即可完成此操作。

当您使用@Component注释组件实现类，以及使用@Reference注释任何引用绑定方法时，PDE将验证您的注释并生成相应的组件描述符文件。验证期间发现的任何错误都会突出显示并报告

此外，PDE还维护项目的MANIFEST.MF和build.properties文件 - 生成的组件描述符文件自动添加到清单中并包含在构建中。

要启用此功能，请转到“首选项”或“ 项目属性”>“插件开发”>“DS注释”，然后选中“ 从带注释的源生成描述符” 。

将API工具Javadoc标记转换为注释 使用API Tools的插件可以使用Javadoc标记或注释添加对API Java类型（例如No Reference，No Extend）的限制。建议项目使用注释，因此PDE提供了一个转换向导，该向导将使用等效的PDE注释替换标记。

右键单击启用API Tools的插件项目并选择Plug-in Tools> Convert API Tools Javadoc Tags，即可使用该向导。

API Tools wiki页面上提供了可用限制的列表。

在状态栏中显示当前目标平台 如果您经常在目标平台之间切换，PDE会提供在状态栏中显示当前目标平台的选项。在Plug-in Development首选项页面上，选择在状态栏中显示当前目标平台 。状态栏条目将显示活动目标定义的名称，如果目标存在问题，将显示错误图标。
通过在功能选择对话框中输入插件名称来查找功能 向产品，功能或启动配置添加功能时，可以在功能选择对话框的过滤器文本框中输入插件的名称。包含该插件的功能将与过滤器匹配并显示。
目标编辑器中的重新加载按钮 在由于网络连接不稳定等问题导致目标编辑器处于不一致状态且无法手动编辑目标编辑器文件而无法解决条件的情况下，可以使用目标编辑器的“ 位置”部分中的“ 重新加载”按钮。它清除缓存的p2配置文件，然后解析目标。
设置产品的默认首选项值 要为产品设置默认首选项值，您可以使用产品编辑器的“ 自定义”部分中的“ 转换首选项”向导，而不是手动将首选项添加到产品的自定义文件中。
从目标定义创建API基线 要利用任何目标定义的 API工具功能，您可以在Plug-in Development> API Baselines首选项页面中创建API Baseline。
快速搜索任何插件工件 使用“ 打开插件工件”对话框可以快速查找和打开插件工件。按插件，功能或产品的名称搜索，或搜索特定的包，扩展点或扩展名。这些图标可用于查看工件是否在工作区中可用，或者它是否来自目标平台。您可以按Ctrl-Shift-A打开对话框。
清洁或不清洁 创建新的运行时工作台启动配置时，PDE会在启动配置上预设程序参数以包含-clean参数。

此-clean参数将运行时工作台中的所有运行时缓存数据从一次调用清除到下一次调用，以确保在主机工作台中进行的所有更改（例如，新Java包已添加到插件项目等）中被选中启动运行时工作台时启动。

如果目标平台包含大量插件，则清除缓存可能会影响性能。

因此，如果您的目标平台有大量插件并且您处于不主动添加/删除插件项目中的软件包的阶段，则可以删除-clean启动配置中的参数可以改善启动时间。
使用链接导入 导入外部插件和片段可能非常耗时，并且可能会导致较大的工作空间，具体取决于要导入的插件的内容。因此，“导入外部插件和片段”向导为您提供了使用链接导入的选项。这意味着导入操作不会复制导入工作区的资源。它只会创建指向要导入的文件的链接。您将能够浏览这些链接的资源，就好像它们已被复制到您的工作区中一样。但是，它们在您的文件系统上实际上不存在，因此您将无法修改它们。请注意依赖于工作区中物理文件的操作，因为它们不适用于链接的资源。
模板 为了快速入门，PDE提供了几个模板插件，可以生成一个带有一个或多个完全工作扩展的插件。此外，如果您想在任何时候从模板列表中添加新扩展（无需生成插件），您可以直接从清单编辑器访问这些扩展模板。在编辑器的“扩展”页面中，点击“添加...””。在出现的向导中，选择左窗格中的“扩展模板”，然后在右窗格中选择所需的模板。
插件依赖程度 如果您曾查看过插件所依赖的插件列表，并想知道插件需要特定插件X的原因，那么现在您可以轻松找到原因。

在几个上下文（包括清单文件依赖项页面和插件依赖项视图）的上下文菜单中找到的Compute Dependency Extent操作执行组合Java和插件搜索，以查找插件X提供的所有Java类型和扩展点。由您的插件引用。结果将显示在“搜索”视图中。在“搜索结果”视图中选择类型时，上下文菜单中的“ MyPlugIn”操作中的“ 引用”将搜索插入所选类型的插件中的位置。

如果搜索返回0结果，您肯定应该从依赖项列表中删除插件X，因为它根本没有被使用，并且它只会减慢类加载速度。

Compute Dependency Extent对于检查是否使用插件X中的内部（非API）类也很有用，这可能是不可取的。
查找未使用的依赖项 最大限度地减少插件的依赖项数量肯定会提高性能。随着插件的发展，它的依赖列表可能会变得陈旧，因为它可能仍然包含对它不再需要的插件的引用。检查插件实际使用的所有依赖项的快速方法是运行“查找未使用的依赖项”实用程序，该实用程序可通过PDE清单编辑器的“依赖项”页面的上下文菜单获得。
扩展Java搜索范围 Java搜索仅限于工作区中的项目和这些项目引用的外部jar。如果要将更多库从外部插件添加到搜索中：打开插件视图 ，选择一个插件，然后从上下文菜单中选择“ 添加到Java工作区范围 ”。这对于了解依赖于您正在处理的插件的其他插件非常方便。

在Plug-in Development首选项页面上，您还可以打开在Java工作区范围内包含目标中的所有插件 ，这将把目标平台中的每个插件添加到搜索范围。
