添加新方法

  1. 通过在Java编辑器中的VectorTest.java文件的末尾（但在类型的右括号之前）键入以下内容来开始添加方法：
    public void testSizeIsThree（）
    只要在编辑器区域中键入方法名称，新方法就会显示在“大纲”视图的底部。

    此外，
    • 错误注释（红色框）出现在位于编辑器右侧的概览标尺中，
    • 错误图标出现在编辑器左侧的垂直标尺中，
    • 错误指示符出现在编辑器的右上角，
    • 错误标记在文本中。
    这些错误注释表明编译单元当前不正确。如果将鼠标悬停在文本中的错误上，则会出现一个工具提示： 令牌上的语法错误“）”，{此标记后的预期 。这是正确的，因为该方法还没有主体。请注意，编辑器中的错误注释会在您键入时更新。当您键入选项时，可以通过报告问题来控制此行为 Java>编辑器首选项页面。

  2. 单击“ 保存”按钮。编译单元自动编译，错误出现在Package Explorer视图，Problems视图和垂直标尺（编辑器的左侧）中。在Package Explorer视图中，错误会传播到包含错误的编译单元的项目。

  3. 键入以下内容完成新方法：

    {
    // TODO：检查尺寸

    请注意，已自动插入结束大括号。
  4. 保存文件。请注意，错误指示符消失，因为已添加缺少的括号。
