运行时应用程序模型

平台运行时提供用于控制和执行应用程序的应用程序容器。运行时应用程序容器实现OSGi R4规范中包含的Application Admin服务规范。应用程序容器负责发现所有可用的应用程序，并为每个可用的应用程序注册ApplicationDescriptor OSGi服务。ApplicationDescriptor服务可用于启动应用程序。启动应用程序时，将注册ApplicationHandle OSGi服务以表示正在运行的应用程序的实例。ApplicationHandle服务可用于关闭应用程序。有关更多信息，请参阅org.osgi.service.application包和OSGi R4.2规范 。

默认应用程序

给定的Eclipse配置可能包含许多产品和应用程序。Eclipse配置指定在平台运行时启动并运行后由应用程序容器启动的默认应用程序。可以通过以下配置选项之一指定默认应用程序

  • eclipse.product - 标识用于启动平台运行时的产品。产品提供品牌信息（窗口图标，标题栏文本等）以及定义要运行的默认应用程序。
  • eclipse.application - 标识用于启动平台运行时的应用程序。此选项会覆盖产品定义的默认应用程序。

定义应用程序

插件可以使用org.eclipse.core.runtime.applications扩展来定义自己的应用程序。在此扩展中，插件定义应用程序的名称和ID，以及实现应用程序的类。实现应用程序的类用于启动和关闭应用程序实例。
