记录

此表列出了persistence.xml编辑器的“日志记录”页面的属性。


注意：

此页面不适用于使用Generic平台的项目。

属性 描述 默认

记录级别

通过选择日志级别（按信息的升序）指定日志输出的数量和详细信息：

以下是java.util.logging.Level ：

  • OFF -disables日志记录

  • SEVERE -logs异常表示TopLink无法继续，以及登录期间生成的任何异常。这包括堆栈跟踪。

  • 警告 -logs不强制TopLink停止的异常，包括未记录严重级别的所有异常。这不包括堆栈跟踪。

  • INFO -logs每个服务器会话的登录/注销，包括用户名。获取会话后，将记录详细信息。

  • CONFIG -logs仅登录，JDBC连接和数据库信息。

  • FINE -logs SQL。

  • FINER -similar到警告。包括堆栈跟踪。

  • 最佳 -包括其他低级别信息。

示例 ： persistence.xml文件

<property name="eclipselink.logging.level" value="INFO"/>

信息

时间戳

控制是否在每个日志条目中记录时间戳。

以下是有效值：

  • 真 -log一个时间戳。

  • 错误 -不记录时间戳。

示例 ： persistence.xml文件

<property name="eclipselink.logging.timestamp" value="false"/>

真正

线

控制是否在每个日志条目中记录线程标识符。

以下是有效值：

  • true -log一个线程标识符。

  • false -do不记录线程标识符。

真正

会议

控制是否在每个日志条目中记录EclipseLink会话标识符。

以下是有效值：

  • true -log EclipseLink会话标识符。

  • false -do不记录EclipseLink会话标识符。

示例 ： persistence.xml文件

<property name="eclipselink.logging.session" value="false"/>

真正

例外

控制在将异常返回给调用应用程序之前是否记录从EclipseLink代码中抛出的异常。确保记录所有异常，并且不会被应用程序代码屏蔽。

以下是有效值：

  • true -log所有异常。

  • false -do不记录异常。

示例 ： persistence.xml文件

<property name="eclipselink.logging.exceptions" value="true"/>

假

日志文件

指定日志输出的文件位置（而不是标准输出）。

示例 ： persistence.xml文件

<property name="eclipselink.logging.file" value="C:\myout\" />

标准输出

记录仪

选择要使用的记录器类型：

以下是有效值：

  • DefaultLogger - EclipseLink本机记录器eclipselink.logging.DefaultSessionLog 。

  • JavaLogger- java.util.logging logger eclipselink.logging eclipselink.logging.JavaLog 。

  • ServerLogger- java.util.logging logger eclipselink.platform.server.ServerLog 。与eclipselink.platform.server.ServerPlatform.

  • 自定义记录器的完全限定类名。自定义记录器必须实现eclipselink.logging.SessionLog接口。

示例 ： persistence.xml文件

<property name="eclipselink.logging.logger" value="acme.loggers.MyCustomLogger" />

DefaultLogger
