在帮助中嵌入命令

可以在帮助内容中创建指向工作台命令的链接。当用户单击该链接时，将执行该命令。此功能类似于Active Help ，因为它使用JavaScript在文档HTML和Eclipse Java运行时之间进行桥接。与Active Help一样，命令框架也是可扩展的;所以你可以用有趣的行为贡献新的命令。但是，与Active Help不同，工作台中已经定义了大量有用的命令，并且一直在添加更多有用的命令。这应该对文档作者有吸引力，因为链接到现有命令不需要编写任何Java代码。

命令链接可用于：

  • 打开特定的首选项页面，例如 一般>外观 。
  • 选择一个新的透视图或切换到预先确定的透视图，如 Java浏览 。透视
  • 选择一个视图打开或打开预定视图，如 书签 。视图
  • 打开像这样的向导 新的插件项目向导和 导出首选项 。巫师
  • 打开备忘单 。
  • 打开一个消息对话框 。
  • 打开 关于Eclipse SDK对话框。
  • 以及更多！

要了解可用的命令范围，请查看 常规>键首选项页面。此页面用于将键序列绑定到命令，以便显示可用命令列表。请注意，并非每个命令都会从帮助内容中执行有用的操作。许多命令被设计为在稍微狭窄的上下文中工作（如特定视图，或期望某种类型的选择）。

如果要定义自己的命令，请参阅org.eclipse.core.commands API包和org.eclipse.ui.commands和org.eclipse.ui.handlers扩展点的文档。
