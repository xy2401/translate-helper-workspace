调试

马不停蹄

 支持不间断调试 （在其他线程继续执行时停止一个或多个线程）。需要GDB 7.0或更高版本。

特征
  • DSF-GDB 。 CDT的默认调试器现在使用不同的调试框架，称为DSF-GDB。它为较新的GDB版本提供了更好的支持，例如6.7,6.8和7.x，对于旧版本，您必须编辑首选项以默认使用旧框架（CDI）。请在此处查看DSF-GDB功能的完整列表。
  • 现在支持GDB跟踪点。查看维基页面 。仅适用于DSF-GDB。
  • 反向调试 ，允许撤消执行。需要GDB 7.0或更高版本，仅在Linux上可用。仅适用于DSF-GDB。关于该主题的网络研讨会，请点击这里 。
  • Run To Line功能现在允许运行到程序中的任何行，而不是仅运行在同一方法中的行。
  • 支持GDB的Pretty Printing功能。需要GDB 7.0或更高版本以及python和GDB的python漂亮打印机。这里有一些信息。
  • 调试应用程序时Windows平台上的中断得到了改进支持。

 
错误修复

固定

 此版本中关闭了所有Bugzilla任务： CDT 7.0错误修复

 
已知限制

限制

 无法在工作区/项目名称中包含GB18030字符的项目中运行或调试类。大多数类库都不能正确支持系统进程的创建（通过java.lang。的Runtime.exec（...））当指定的命令行包含GB18030个字符时。此限制意味着扫描程序发现机制和调试程序在其生成的命令行包含GB18030字符时无法启动应用程序。Bug 308803中的详细信息
