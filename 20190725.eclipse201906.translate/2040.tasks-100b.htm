从分支合并

在CVS分支中创建和工作一段时间后，您可能希望将更改从分支合并到另一个分支或HEAD中。要做到这一点，您需要知道：

  1. 包含更改的分支或版本的名称。
  2. 创建分支的版本。这是分支时提供的版本名称。

要合并您的更改：

  1. 确保将目标加载到工作区中。例如，如果要将更改合并到HEAD中，请确保在工作区中与HEAD共享项目。为此，请选择项目，然后从上下文菜单中选择“ 替换为”>“另一个分支或版本 ”。然后选择要替换的分支。
  2. 选择项目，然后选择Team> Merge 。
  3. 输入包含要合并的更改的分支或版本。
  4. 合并对话框将尝试猜测一个合适的起点。如果失败，请输入合并的起始点。这是创建分支的版本。单击完成 。
  5. CVS Merge同步将添加到Synchronize视图，显示工作区与具有更改的分支之间的所有差异。
  6. 将所有所需更改加载到工作区中。这可以通过手动合并更改，或通过从树的上下文菜单中选择“ 更新” ，“ 覆盖和更新 ”或“ 标记为合并”来完成。
  7. 在工作区中进行了所有需要的更改后，选择Team> Synchronize with Repository 。然后，您可以将所有更改提交到存储库。

提示 ：如果您不知道合并的起点，则可以选择直接将合并执行到工作区中。这依赖于CVS服务器来选择合适的起点。由于合并是直接在工作区中执行的，因此CVS可能会将基于文本的冲突标记引入文件中。此标记通常很难处理，因此只有在必要时才能直接合并到工作空间中。

合并行动

合并编辑器中的操作补充了合并编辑器下半部分中可用的手动合并工具栏操作。

合并 运行此操作会将更改带入工作区中的文件。将跳过任何不可自动合并的冲突。
覆盖和更新 对具有冲突更改的文件启用此操作。运行此操作将丢弃您拥有的任何本地更改，并将文件替换为远程内容。
标记为合并 此操作将从视图中删除所选更改。只有在资源的远程状态发生更改并刷新CVS合并同步时，才会再次出现更改。

提示 ：您可以通过在“同步”视图中固定CVS合并同步来执行持续合并。这将允许您使用发布到不同分支的更改使工作区保持最新。
