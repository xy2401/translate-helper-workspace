目标定义编辑器

目标定义编辑器用于编辑工作空间中的目标定义文件，该文件应以扩展名“.target”结尾。可以使用“ 新建目标定义向导”或将目标定义文件移动到“ 目标平台首选项”页面上的工作区来创建目标定义文件。

编辑器由四页组成：

  1. 定义页面 - 为您的定义提供描述性名称并提供插件的位置
  2. 内容页面 - 查看目标中可用的插件，并选择要包含在目标平台中的子集
  3. 环境页面 - 修改目标定义的其他设置，包括目标环境，JRE，参数和隐式依赖项
  4. 源页面 - 在完成辅助，错误诊断和更新命令的帮助下查看和编辑目标定义的源代码

在编辑器的三个第一页的每一页的顶部都有一个链接设置为目标平台 。激活此链接会将此目标定义设置为活动目标平台。要查看您的活动目标平台是什么，请使用“ 目标平台首选项”页面 。

解决目标

在可以编辑目标的内容之前，可以在将目标定义设置为活动目标平台之前，必须解析它。解析目标定义后，将搜索定义的位置以查找插件。根据目标中的位置类型，这可能涉及搜索目录，读取安装的元数据，读取功能或从远程站点下载。在解析期间，将运行后台作业。完成后，编辑器将更新发生的任何问题。

有关最新目标故障排除帮助，请参阅目标定义Wiki页面 。
