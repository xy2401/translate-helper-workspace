调试信息

使用独立调试器时，您将使用Debug透视图。Debug透视图允许您在Eclipse Workbench中管理程序的调试或运行。您可以通过设置断点，暂停已启动的程序，单步执行代码以及检查变量的内容来控制程序的执行。

Debug透视图显示以下信息：

  • 正在调试的每个目标的挂起线程的堆栈帧
  • 程序中的每个线程都表示为树中的节点
  • 您正在运行的每个程序的过程

Debug透视图还驱动C / C ++编辑器 。当您单步执行程序时， C / C ++编辑器会突出显示执行指针的位置。

变量

您可以在“变量”视图中查看有关所选堆栈框架中变量的信息。执行停止时，更改的值默认以红色突出显示。与其他与调试相关的视图一样，在运行可执行文件时，“变量”视图不会刷新。执行停止时发生刷新。

表达式

表达式是一段代码，可以对其进行求值以生成结果。表达式的上下文取决于特定的调试模型。某些表达式可能需要在程序中的特定位置进行评估，以便可以引用变量。您可以在“表达式”视图中查看有关表达式的信息。

寄存器

您可以在选定的堆栈框架中查看有关寄存器的信息。当程序停止时，已在“寄存器”视图中突出显示已更改的值。

记忆

您可以检查和更改进程内存。

拆卸

您可以查看与源信息混合的反汇编代码。

模块

您可以查看有关当前调试会话中加载的模块（可执行文件和共享库）的信息。

信号

您可以查看在所选调试目标上定义的信号以及调试器如何处理每个信号。
