C / C ++项目属性，构建，设置，容器设置

使用“ 容器设置”属性选项卡指定在Docker容器中构建项目的选项。此属性选项卡随可选的Docker C / C ++启动功能一起提供。默认情况下，项目配置将在本地构建，但是，如果要在Container中构建项目的特定配置，请使用启用复选框。在Container中启用构建后，您需要在该Connection中选择一个活动的Docker Connection和一个下载的Image来运行内置。Image必须具有执行已安装的构建所需的任何工具。Docker Images将基于Linux，因此构建将使用适当的Linux工具集并创建与Linux兼容的可执行文件。

默认情况下，C / C ++项目和所有相关项目内容将安装到正在进行构建的Container中。这允许构建查找项目源和头文件。如果构建项目需要其他主机目录（例如，本地头文件目录不是Docker Image中已安装软件包的一部分），则可以将这些目录添加到Data Volumes表中，然后将它们安装在Container中构建发生。如果Docker守护程序在本地运行，那么这将是一个简单的安装。如果Docker守护程序是远程的，则安装将涉及自动将目录内容复制到Container，因为系统安装不能跨机器进行。

为了适应索引，将在Docker容器中执行构建规范发现。通过规范检测发现的目录将被本地复制以供索引器使用。对于构建时发现的任何头文件目录，也会发生此复制过程。头文件存储在工作区中，每个Docker镜像只上传一次。例如，如果项目不再需要为该特定Docker镜像构建/索引，则可以从“首选项”页面中删除这些头文件。

容器设置选项
组 描述
组态 点击此处查看说明。
管理配置...... 点击此处查看说明。
在Docker镜像内部构建 （复选框） 允许构建在Container内部发生。
连接 下拉包含已知Docker守护程序连接的列表，以便在选中启用按钮时运行内置。
图片 包含Docker Image名称的下拉列表，用于创建构建将运行的Container。此列表由Docker Connection选项和启用复选框启用。
数据卷 在构建之前要装入Container的可选目录列表。默认情况下，C / C ++项目和任何引用的项目都会自动挂载到Container中。可以使用列表右侧的按钮添加，编辑或删除目录。
加... 用于调出“ 数据卷”对话框的按钮，用于指定要在Docker容器中装入的数据卷。这通常涉及从本地系统中选择目录。完成后，装载将显示在“数据卷”列表中。
编辑... 用于编辑“数据卷”列表中当前所选数据卷的按钮。
去掉 用于删除“数据卷”列表中当前所选数据卷的按钮。

适用于： CDT项目。
