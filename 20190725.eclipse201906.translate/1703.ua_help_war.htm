将信息中心部署为Web存档

使用Eclipse 3.4或更高版本，可以将帮助插件配置为部署为Web存档（war文件），这将充当完全充当
运作的信息中心。下面的说明假定已安装Tomcat服务器，但稍作修改
这些步骤适用于任何功能齐全的服务器。

在Eclipse安装中找到插件org.eclipse.help.webapp。<version> .jar并将其复制到临时目录，
将副本解压缩到该文件夹中。

  1. 在webapp插件中找到web-archive目录，在其下面将有两个名为“help”和的目录
    “org.eclipse.help.infocenter特征”。
  2. 使用File-> Import-> Existing Project导入org.eclipse.help.infocenter-feature。
  3. 将org.eclipse.help.infocenter-feature导出为可部署功能，并在org.eclipse.help.webapp。<version> .jar解压缩的区域中将目标设置为web-archive / help / WEB-INF。
  4. 将一些文档插件添加到webapps / help / WEB-INF / plugins目录中。
  5. 从equinox下载站点下载org.eclipse.equinox.http.servletbridge_ <version> .jar和org.eclipse.equinox.servletbridge_ <version> .jar。选择与您运行的Eclipse版本匹配的Equinox版本，以转到下载页面。
  6. 添加文件org.eclipse.equinox.servletbridge_ <version> .jar。到help / WEB-INF / lib目录。您可能需要创建此目录。
  7. 将文件org.eclipse.equinox.http.servletbridge_ <version> .jar添加到help / WEB-INF / plugins目录
  8. 在此阶段，您可以从帮助目录创建war文件，也可以将目录及其内容复制到Tomcat安装的webapps文件夹中。
  9. 仅适用于Tomcat。例如，在conf / server.xml中，将URIEncoding =“UTF-8”添加到连接器元素
    <Connector port="8080" URIEncoding="UTF-8" etc.>
    如果未执行此步骤，则如果搜索项包含非ASCII字符，则搜索将失败。
  10. 启动Tomcat并查看帮助系统启动。
注意：如果你查看目录help / WEB_INF / configuration下的help.war文件中的config.ini，你会注意到
line eclipse.product = org.eclipse.productname。如果您的产品在产品插件中有帮助系统自定义，您可以
通过更改此行以指向您的产品插件来激活这些。
