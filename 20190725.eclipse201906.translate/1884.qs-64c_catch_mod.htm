做出我们自己的改变

Fred对file1.txt和file2.txt进行了一些更改，并将它们提交到了存储库。在本节中，将进行其他更改，然后与存储库同步。在同步时，期望看到本节中所做的更改以及Fred所做的更改。

  1. 添加file3.txt如下：
    新内容:(将以下行添加到文件中）
    This is the brief contents
of file 3
  2. 观察Project Explorer显示资源的CVS状态。请注意，Jane添加的新文件前面带有“>”。
  3. 选择项目JanesTeamProject，然后从项目的上下文菜单中选择Team> Synchronize with Repository 。当要求切换透视图时选择是 。将打开“团队同步”透视图，您将看到已更改的文件显示在“同步视图”中。双击file1.txt ，你会看到一个比较编辑器打开：
  4. 还有一些值得观察的事情。首先， file1.txt旁边的图标（在结构化比较区域中）表示file1.txt有一个传入的更改。这意味着您需要将更改发布到服务器。查看file1.txt，我们可以看到Fred已添加到文件中的文本。另外，请注意窗口底部，在状态行中，旁边有数字的箭头。它们显示传入，传出和冲突的文件数。文件旁边的第一个数字是工作区中的修订版，另一个是上次同步时服务器上的修订版。
  5. 通常，您应该更新其他人所做的更改，然后使用更改测试工作区，然后在确定其他人的新更改未破坏工作区中的任何内容后提交更改。
  6. 在决定接受弗雷德的变化之前，你可能想知道他为什么要做出改变。选择file1.txt，然后从上下文菜单中选择Show in History 。
  7. 以*开头的行表示这是当前加载的修订版。在这种情况下，您可以看到Fred在发布1.2版时所做的评论。

    提示 ：您可以在“历史记录”视图中选择“ 使用编辑器和选择链接”工具栏按钮，以便在打开新编辑器或选择更改时自动更新历史记录。这允许快速浏览评论。

  8. 要更新，只需在Synchronize View中选择JanesTeamProject ，然后从上下文菜单中选择Update 。
  9. Synchronize View将更新以反映file1.txt和file2.txt不再与存储库不同步的事实。您现在应该只能看到file3.txt了。
  10. 接下来，您可以提交file3.txt 。
