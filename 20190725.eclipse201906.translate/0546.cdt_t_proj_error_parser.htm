调整错误分析器

错误解析器逐行扫描构建输出，查找错误和警告（也适用于某些信息性消息）。它们生成问题标记 ，用户可视地指示这些问题。这些问题在控制台上突出显示，填充“问题”视图并显示在编辑器中。问题标记被组织成错误 ， 警告和信息类别。

有几个预定义的错误解析器，每个解析器都可以处理特定工具的输出。通常，工具链定义一组错误分析器来处理每个工具的输出。可以在项目属性中基于每个项目重新定义该集合：

  1. 在“C / C ++项目”视图中，右键单击项目，然后选择“ 属性” 。
  2. 展开C / C ++ Build 。
  3. 选择设置 。
  4. 单击“ 错误分析程序”选项卡。
  5. 在“ 错误解析器”列表中，选择错误解析器。
  6. 单击确定 。

请注意，错误解析器的顺序很重要。通常情况下，错误解析器会消耗它发现错误/警告的行，而其他行下的错误解析器将无法解析它。

CDT提供了一种特殊的错误解析器， CDT pushd / popd CWD Locator 。此解析器不会创建问题标记，但它正在查找pushd和popd命令的输出，这些命令通常用于makefile以打印当前工作目录（CWD）。CDT可以使用此信息将输出中出现的文件名与CWD相关联。请注意，这种解析器需要是第一个能够将CWD提供给其他解析器的解析器。

大多数错误解析器都可以通过正则表达式模式进行配置，用户可以：

  • 调整图案以更准确地检测问题，
  • 禁用无用或增殖问题（请注意，它可以对控制台的性能产生积极影响），
  • 为自定义工具创建自己的错误解析器。提示：要运行自定义工具，您可以使用“生成目标视图” 。

要自定义错误解析器，请执

  1. 选择Window> Preferences 。
  2. 展开C / C ++ 。
  3. 选择“ 错误分析器”选项卡。
  4. 选择错误解析器或添加新错误解析器。
  5. 按照错误分析器选项定义自定义模式
