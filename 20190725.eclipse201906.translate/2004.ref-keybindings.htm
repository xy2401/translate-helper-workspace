键绑定列表

Eclipse中可用键绑定的列表取决于许多因素，包括选择了哪个视图或编辑器，对话框是否打开，安装了哪些插件以及正在使用的操作和窗口系统。您可以随时使用Key Assist获取可用键绑定列表（ 帮助>关键辅助......或Ctrl + Shift + L）。下表列出了Eclipse SDK中可用的一些常用键绑定。

 

文件操作

新 创建Java元素或新资源。在“窗口”>“透视”>“自定义透视”中配置子菜单中显示的元素。在Java透视图中，默认情况下，可以使用创建项目，包，类，接口，源文件夹，剪贴簿，文件和文件夹的操作。 Ctrl + N.
关 关闭当前编辑器。如果编辑器包含未保存的数据，则会显示保存请求对话框。 Ctrl + F4
关闭所有 关闭所有编辑器。如果编辑器包含未保存的数据，则会显示保存请求对话框。 Ctrl + Shift + F4
保存 保存当前编辑器的内容。如果编辑器不包含未保存的更改，则禁用。 Ctrl + S.
另存为 以新名称保存当前编辑器的内容。
保存全部 使用未保存的更改保存所有编辑器的内容。如果没有编辑器包含未保存的更改则禁用 Ctrl + Shift + S.
打印 打印当前编辑器的内容。编辑器具有焦点时启用。 Ctrl + P.
属性 打开选择元素的属性页面。在Java项目上打开了Java Build Path页面和Javadoc Location页面。对于JAR存档，请在此处配置JAR的源附件和Javadoc位置。 Alt + Enter

编辑动作

解开 还原编辑器中的最后一个更改 Ctrl + Z.
重做 恢复未完成的更改 Ctrl + Y.
切 将当前选定的文本或元素复制到剪贴板并删除该元素。在元素上，在粘贴剪贴板之前不执行删除。 Ctrl + X.
复制 将当前选定的文本或元素复制到剪贴板 Ctrl + C.
糊 将当前内容作为文本粘贴到编辑器，或作为当前所选元素的兄弟或子元素。 Ctrl + V.
删除 删除当前文本或元素选择。 删除
全选 选择所有编辑器内容.. Ctrl + A.
查找/替换 打开“查找/替换”对话框。仅限编辑。 Ctrl + F.
找下一个 查找当前所选文本的下一个匹配项。仅限编辑。 Ctrl + K.
找到上一个 查找当前所选文本的上一个匹配项。仅限编辑。 Ctrl + Shift + K.
增量查找下一个 启动增量查找模式。调用后，按照状态栏中的说明输入搜索文本。仅限编辑。 Ctrl + J
增量查找上一个 启动增量查找模式。调用后，按照状态栏中的说明输入搜索文本。仅限编辑。 Ctrl + Shift + J.
添加任务 将用户定义的任务添加到当前文本选择或所选元素。 Alt + Enter
将选择展开到 Enclosing Element：在代码中选择封闭的表达式，块，方法。此操作了解Java语法。当代码有语法错误时，它可能无法正常工作。（向上箭头）。 Alt + Shift +箭头键
下一个元素：选择当前和下一个元素。（向右箭头）
上一个元素：选择当前元素和上一个元素（向左箭头）
恢复上次选择：调用展开选择后，恢复上一个选择。（箭头向下）
显示工具提示说明 显示将出现在当前光标位置的悬停值。显示的对话框是可滚动的，不会缩短说明。 F2
内容辅助 在当前光标位置打开上下文辅助对话框以显示Java代码辅助提议和模板。有关可用模板的信息，请参阅模板首选项页面（Java>编辑器>模板），然后转到编辑器首选项页面（Java>编辑器>代码辅助）以配置代码帮助的行为。 Ctrl +空格
快速解决 如果光标位于有问题指示的位置，则会在当前光标处打开上下文辅助对话框以显示可能的更正。 Ctrl + 1
参数提示 如果光标位于方法参考的参数规范中，则此操作将显示带有参数类型信息的悬停。当前光标位置的参数以粗体显示。 Ctrl + Shift + Space

导航操作

打开 尝试解析当前代码选择中引用的元素，并打开声明引用的文件。 F3
打开“类型层次结构” 尝试解析当前代码选择中引用的元素，并在“类型层次结构”视图中打开该元素。在元素上调用，打开元素的类型层次结构。在Java编辑器和支持Java元素的视图中受支持。 F4
打开外部Javadoc 打开当前所选元素或文本选择的Javadoc文档。JAR或项目的Javadoc的位置在项目或JAR的Javadoc Location属性页中指定。请注意，此外部Javadoc文档可能与当前代码中指定的Javadoc不同。您可以使用Javadoc导出向导为Java项目中的源文件创建Javadoc文档。 Shift + F2
打开类型 打开“打开类型”选择对话框以在编辑器中打开类型。“打开类型”选择对话框显示工作空间中存在的所有类型。 Ctrl + Shift + T.
打开类型层次结构 打开“打开类型”选择对话框以在编辑器和“类型层次结构”视图中打开类型。“打开类型”选择对话框显示工作空间中存在的所有类型。 Ctrl + Shift + H.
显示大纲 打开当前所选类型的轻量级大纲。 Ctrl + O.
转到下一个问题 选择下一个问题。在Java编辑器中受支持。 Ctrl +。（期）
转到上一个问题 选择上一个问题。在Java编辑器中受支持。 Ctrl +，（逗号）
转到上次编辑位置 显示上次编辑发生的位置。 Ctrl + Q.
去线 打开一个对话框，允许输入编辑器应跳转到的行号。仅限编辑。 Ctrl + L.

搜索操作

搜索... 打开搜索对话框 Ctrl + H.
文件中的出现次数 在其文件中查找所选出现的所有Java元素 Ctrl + Shift + U.

项目行动

全部构建 构建工作区中的所有项目。这是一个增量构建，意味着构建器会分析自上次构建以来的更改，并最大限度地减少已更改文件的数量。 Ctrl + B.

来源行动

评论 注释掉包含当前选择的所有行。 Ctrl + /
取消注释 取消注释包含当前选择的所有行。 Ctrl + \
缩进 缩进当前行。 Ctrl + I
向右移动 增加当前选择行的缩进级别。仅在选择涵盖多行或单个整行时激活。 标签
向左移动 减少当前选定行的缩进级别。仅在选择涵盖多行或单个整行时激活。 Shift + Tab
格式 使用代码格式化程序格式化当前文本选择。格式化选项在Code Formatter首选项页面上配置（Java> Code Formatter） Ctrl + Shift + F.
组织进口 在当前打开或选择的编译单元中组织导入声明。将删除不必要的导入声明，并按照“组织导入”首选项页面（Java>组织导入）中的指定对所需的导入声明进行排序。组织导入可以在不完整的源上执行，并且当引用的类型名称无法唯一地映射到当前项目中的类型时将提示您。您还可以通过调用包上的操作或选择一组编译单元来组织多个编译单元。 Ctrl + Shift + O.
添加导入 为当前选定的类型引用创建导入声明。如果类型引用符合条件，则将尽可能删除资格。如果引用的类型名称无法唯一映射到当前项目的类型，系统将提示您指定正确的类型。添加导入尝试按照“组织导入”首选项页面中指定的导入顺序进行操作。 Ctrl + Shift + M.

重构行动

解开 是否撤消了最后一次重构。重构撤消缓冲区仅在没有其他源更改而不是重构的情况下才有效。 Alt + Shift + Z.
重做 重做最后一次重做的重做。重构撤消/重做缓冲区只有在没有其他源更改而不是重构的情况下才有效。 Alt + Shift + Y.
改名 启动“重命名重构”对话框：重命名所选元素，并（如果启用）更正对元素的所有引用（也在其他文件中）。可用于方法，字段，局部变量，方法参数，类型，编译单元，包，源文件夹，项目以及解析为这些元素类型之一的文本选择。 Alt + Shift + R.
移动 启动“移动重构”对话框：移动选定的元素，并（如果启用）更正对元素的所有引用（也在其他文件中）。可以应用于一个实例方法（可以移动到组件），一个或多个静态方法，静态字段，类型，编译单元，包，源文件夹和项目以及解析为这些元素类型之一的文本选择。 Alt + Shift + V.
排队 启动Inline重构对话框。内联局部变量，方法或常量。这种重构可用于解析方法，静态最终字段或局部变量的方法，静态最终字段和文本选择。 Alt + Shift + I
提取方法 启动“提取方法”重构对话框。创建一个包含当前所选语句或表达式的新方法，并使用对新方法的引用替换选择。您可以使用“编辑”菜单中的“扩展选择”来获取有效的选择范围。此功能对于清除冗长，杂乱或过于复杂的方法非常有用。 Alt + Shift + M.
提取局部变量 启动“提取变量”重构对话框。创建分配给当前所选表达式的新变量，并使用对新变量的引用替换选择。此重构可用于解析为局部变量的文本选择。您可以使用“编辑”菜单中的“扩展选择”来获取有效的选择范围。 Alt + Shift + L.
