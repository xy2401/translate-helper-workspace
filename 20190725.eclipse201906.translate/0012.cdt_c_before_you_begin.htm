在你开始之前

根据您获得独立调试器的方式，您可能还收到了工具链。但是，如果从Eclipse Web站点下载了独立调试器，那么在调试任何项目之前，您将需要一个工具链。

独立调试器使用CDT（C / C ++开发工具）项目中与调试应用程序有关的插件。这些CDT调试组件需要一些GNU工具链元素，例如gdb调试器和gcc编译器。独立调试器允许编辑文件，但不支持重建项目;用户可以从命令行或使用支持编辑/编译/构建/调试的完整CDT IDE在独立调试器外部执行此操作。

运行独立调试器的每个平台都需要不同的步骤来获取GNU工具链。

视窗

对于Windows，MinGW和Cygwin是获取GNU工具链的两个主要平台选择。了解它们之间的区别非常重要。Cygwin生成使用Cygwin POSIX运行时的可执行文件。请注意，此运行时是GPL许可的。MinGW生成本机Windows可执行文件，不需要单独的运行时。

  • 对于MinGW ，建议使用MinGW安装程序mingw-get来管理MinGW安装。从MinGW下载页面下载并运行最新的mingw-get-inst软件包。

    请注意，Wascana IDE已停产。

  • Cygwin的可以从Cygwin的现场安装http://www.cygwin.com 。您需要手动选择gdb和gcc包。gcc包用于确定默认包含路径和内置宏，以便为导航目的索引代码。

  • Windows SDK提供了创建Windows应用程序所需的Visual C ++编译器和头文件和库。

    注意：对于此版本，集成应视为beta质量。不建议用于生产。

Linux的

所有Linux发行版都包含GNU工具链。但是，默认情况下可能不会安装它们。有关安装适用于Linux的GNU工具链的说明，请参阅特定发行版的说明。

Mac OS X.

独立调试器支持Xcode IDE中包含的Apple GNU工具链。它可以从Apple的开发者网站http://developer.apple.com下载

其他平台

独立调试器支持的所有平台都支持GNU工具链。有关在平台上安装GNU工具链的说明，请咨询您的平台供应商。
