内存视图首选项

您可以为内存渲染设置表格渲染，代码页和填充字符串首选项。此外，您可以修改重置内存渲染的首选行为。

从内存视图向下箭头图标菜单打开内存视图首选项对话框。要打开“内存视图首选项”对话框，请单击“内存”视图下箭头图标，然后从菜单中选择“首选项 ”。要打开“内存”视图表渲染“首选项”对话框，请单击“内存”视图下箭头图标，然后从菜单中选择“ 表格渲染首选项 ”。

要将您在首选项中所做的任何更改还原为其默认设置，请单击“还原默认值” 。

首选项：重置内存监视器

如果您已经远离它，可以将渲染重置为基址。将渲染重置为基址时，可以将其设置为仅重置可见渲染 - 或者可以将其设置为重置所有渲染。如果选择重置所有渲染，则可能会对重置操作的性能产生负面影响。要设置此首选项，请打开“首选项”对话框，然后选择“ 重置内存监视器”节点。在Reset Memory Monitor页面中，选择相应的单选按钮。

首选项：填充字符串

填充字符串是无法检索内存时将出现在内存内容中的字符串。要设置填充字符串，请打开“首选项”对话框，然后选择“ 填充字符串”节点。在“填充字符串”页面中，指定无法确定内存内容时要显示的字符串。

首选项：选择代码页

在“ Renderings”窗格中监视ASCII和EBCDIC基于文本的渲染（以及映射的内存，如果它在您安装此调试器的产品中可用）时，可以设置要在其中显示渲染的代码页。

要将用于渲染内存的代码页设置为ASCII / EBCDIC，请打开“首选项”对话框，然后选择“ 选择代码页”节点。在“选择代码页”页面中，指定要更改的字符集的代码页（对于ASCII渲染，EBCDIC渲染或两者）。

表格渲染首选项

要设置表中显示的内存渲染的首选项，请单击“内存”视图下箭头图标，然后选择“ 表格渲染首选项” 。在生成的首选项对话框中，有两个选项：

  • 自动单选按钮：如果希望调试器在Memory视图中的可见区域之前和之后自动预加载内存缓冲区，请选择此选项。选择此设置后，您可以使用滚动和向上/向下页面操作轻松地从可见的内存区域滚动。

    如果缓冲区大小很大，刷新内存视图的性能可能会受到负面影响（当内存更改可能时，内存视图会刷新 - 例如，当您执行步骤时，内存视图会刷新，运行后会暂停，更改变量或寄存器，或修改内存）。这是因为每次刷新都必须从调试引擎请求更多内存。另一方面，如果缓冲区大小太小，则滚动的性能会受到负面影响。这是因为当缓冲区耗尽时，必须从调试引擎请求更多内存。鉴于此性能权衡，您需要设置此选项以满足您的需求。

  • 手动单选按钮：如果选择此设置，则指定的每页行数将加载到“ 渲染”窗格中。使用此设置时，您可以更准确地选择要监视的内容，但是，您将无法在此页面大小设置定义的缓冲区外滚动。相反，要从下一页或上一页查看内存，必须右键单击以使用弹出菜单中的“ 上一页”和“ 下一页”操作。

    如果缓冲区大小很大，刷新内存视图的性能可能会受到负面影响（当内存更改可能时，内存视图会刷新 - 例如，当您执行步骤时，内存视图会刷新，运行后会暂停，更改变量或寄存器，或修改内存）。这是因为每次刷新都必须从调试引擎请求更多内存。
