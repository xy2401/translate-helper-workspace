调试项目

调试器允许您通过设置断点，暂停执行的程序，单步执行代码以及检查变量的内容来控制程序的执行。

调试项目：

  1. 单击“ 运行”>“调试配置...”菜单选项。

    “ 调试 配置”对话框将打开。



  2. 双击“ C / C ++应用程序”以创建新的启动配置。



    如果在Project Explorer视图中选择了项目，则会自动输入以下数据，请花点时间验证其准确性或根据需要进行更改。

    1. 在“ 名称”编辑框中，键入项目名称，例如Hello World 。

      现在，您可以在下次调试此项目时按名称选择此调试启动配置。

    2. 在“ C / C ++应用程序”编辑框中，键入可执行文件的名称（如果尚未输入）。

      如果没有，请单击“ 搜索项目”按钮以在项目中找到可执行文件。

    3. 在项目编辑框中，键入或选择您的项目，例如HelloWorld 。

       

      注意如果看到错误“ [Debugger]：No debugger available ”，请选择Debugger选项卡并为您的平台选择有效的调试器（例如gdb / mi ）。

  3. 单击Debug 。

    系统可能会提示您切换到Debug透视图。单击是


    出现Debug透视图，打开helloworld.exe应用程序窗口。C / C ++编辑器在透视图中重新定位。


             注意：如果在编辑器视图中看到错误“ 找不到源文件 ”，请使用“ 定位文件”按钮找到源文件。


  4. 在main.cpp窗口的左边距中，双击以设置断点：

     cout << 你刚进入

    你会看到一个蓝色圆圈（ ）表示断点已设置。

  5. 单击运行>恢复 。

    或者，您可以使用“ 继续”按钮（ ）在Debug视图的工具栏上。

  6. 出现提示时，在“ 控制台”视图中，键入“m”以外的值，然后按Enter键。

    断点将被击中。

  7. 在“ 变量”视图中，验证变量不是“m”。

  8. 单击运行>恢复 。

  9. 出现提示时，再次在“ 控制台”视图中键入“m”以外的值，然后按Enter键。

    断点将被击中。

  10. 在“ 变量”视图中，验证变量不是“m”。

  11. 在“ 变量”视图中，右键单击输入变量，然后选择“ 更改值...” 并在单引号之间键入“m”并单击“ 确定” 。

  12. 单击运行>恢复 。

    helloworld.exe应用程序窗口中的输出是：“ 谢谢。退出。 “

  13. 应用程序终止，调试会话结束。Debug透视图保持打开状态。

要了解有关调试的更多信息，请参阅相关的调试概念主题。
