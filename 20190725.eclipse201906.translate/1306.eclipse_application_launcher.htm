Eclipse Application Launcher

PDE提供了一个Eclipse Application启动程序 ，它允许您通过启动单独的Eclipse应用程序来运行和调试您的插件。与Eclipse中的所有其他启动程序一样 （例如Java应用程序和Java Applet启动程序 ，...），可以通过快捷方式调用Eclipse Application启动程序，并在启动配置对话框中集中管理其启动配置 。

通过快捷方式启动

测试插件的一种快速方法是通过Eclipse Application启动快捷方式启动单独的应用程序 ，该快捷方式可用：

  • 插件清单编辑器的“ 概述”页面的“ 测试”部分中的热链接
  • 插件项目的Run As>和Debug As>上下文菜单中的操作

启动快捷方式是上下文相关的。如果所选资源是插件项目或插件清单编辑器，它声明了应用程序（ org.eclipse.core.runtime.applications扩展名）或产品（ org.eclipse.core.runtime.products扩展名），然后PDE启动该应用程序或产品。PDE启动的插件集是一个由所选插件及其所有先决条件组成的最小集合。如果所选插件包含产品扩展和应用程序扩展，则产品扩展优先。

如果所选插件不包含产品扩展或应用程序扩展，则PDE将启动config.ini文件的eclipse.product键中定义的缺省产品，该文件位于$ {target_home} / configuration目录中。 $ {target_home}是指Windows> Preferences ...> Plug-in Development> Target Platform首选项页面上指定的目标平台的位置。如果启动了默认产品，则启动中使用的插件集是所有工作区插件以及在Target Platform首选项页面上检查的所有插件。其ID与工作区插件的ID冲突的目标插件不包含在启动配置中。

PDE创建一个新的Eclipse Application启动配置，并使用合理的默认值对其进行预配置。这只在需要时完成 - 不是每次调用Eclipse Application启动快捷方式时都这样做。PDE首先搜索现有Eclipse Application启动配置，并重用已与正在启动的产品或应用程序关联的配置。如果有多个与正在启动的产品或应用程序关联的现有启动配置，PDE将显示一个对话框，其中包含所有匹配的启动配置，并允许用户决定。

自定义启动配置

如果您需要完全控制如何启动Eclipse应用程序，则可以在“ 启动配置”对话框中创建和自定义启动配置 。

可以通过顶级菜单中的Run> Run ...或Debug> Debug ...调用Launch Configuration对话框 。通过双击左侧树查看器中的Eclipse Application节点，可以创建新的Eclipse Application启动配置。

Eclipse Application启动配置包含七个选项卡，允许您自定义测试启动的所有方面。有关更多详细信息，请参阅以下文档：

  • 主标签
  • 参数选项卡
  • 插件选项卡
  • 配置选项卡
  • 跟踪选项卡
  • 环境标签
  • 常用标签
  • 原型标签
