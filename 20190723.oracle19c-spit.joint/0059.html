为独立服务器安装Oracle Grid Infrastructure，该服务器安装Oracle Restart和Oracle ASM，并创建一个磁盘组。

在开始安装之前，您应该可以使用网络信息，存储信息以及操作系统用户和组。您还应该准备好运行根脚本或提供自动化根脚本的信息。

  1. 以Oracle Restart软件所有者用户（ oracle ）身份登录。
  2. 下载Oracle Grid Infrastructure以获取独立服务器安装映像文件，创建网格主目录，并在此网格主目录中提取映像文件。

    $ mkdir -p /u01/app/oracle/product/19.0.0/grid $ chgrp oinstall /u01/app/oracle/product/19.0.0/grid $ cd /u01/app/oracle/product/19.0.0/ grid $ unzip -q /tmp/grid_home.zip

      #cd /u01/app/oracle/product/19.0.0/grid/bin#./asmcmd afd_label DATA1 / dev / sdb --init＃。/ asmcmd afd_label DATA2 / dev / sdc --init＃。/ asmcmd afd_label DATA3 / dev / sdd --init
    3. 使用ASMCMD afd_lslbl命令验证设备是否已标记为与Oracle ASMFD一起使用。例如：
      ＃。/ asmcmd afd_lslbl / dev / sdb＃。/ asmcmd afd_lslbl / dev / sdc＃。/ asmcmd afd_lslbl / dev / sdd

      #unset ORACLE_BASE

    $ Grid_home /gridSetup.sh

    • 默认磁盘组名称为DATA。您可以为磁盘组输入新名称，或使用默认名称。

    • 您创建的任何其他磁盘设备必须由执行网格安装的用户拥有。

    如果为运行根脚本配置自动化，并且根脚本失败，则可以手动修复问题，然后单击“ 重试”以再次运行根脚本。

  11. 启动Oracle数据库安装，并为Oracle数据库文件存储选择Oracle ASM磁盘组。有关安装期间的帮助 ，请单击Oracle Universal Installer页面上的“ 帮助” ，以获取更多信息。

1 Oracle数据库测试简介

Oracle数据库的Oracle Real Application Testing选项可帮助您安全地确保数据库更改的完整性并管理测试数据。

神谕 Real Application Testing选项使您可以执行Oracle数据库的实际测试。通过在生产部署之前捕获生产工作负载并评估系统更改对这些工作负载的影响，Oracle Real Application Testing可最大程度地降低与系统更改相关的不稳定性风险。SQL性能分析器和数据库重放是关键 Oracle Real Application Testing的组件。根据所测试的系统更改的性质和影响，以及将要执行测试的系统类型，您可以使用其中一个或两个组件来执行测试。

1.1 SQL性能分析器

系统更改（例如升级数据库或添加索引）可能会导致更改SQL语句的执行计划，从而对SQL性能产生重大影响。 在某些情况下，系统更改可能会导致SQL语句退化，从而导致性能下降。在其他情况下，系统更改可能会提高SQL性能。能够准确预测系统更改对SQL性能的潜在影响使您能够在SQL语句退化的情况下预先调整系统，或者在SQL语句性能提高的情况下验证和测量性能增益。

SQL Performance Analyzer通过识别每个SQL语句的性能差异，自动完成评估更改对整个SQL工作负载的总体影响的过程。提供了一份报告，显示由于更改而对工作负载性能的净影响。退步 SQL语句，SQL性能分析器还提供适当的执行计划详细信息以及调优建议。因此，您可以在最终用户受到影响之前纠正任何负面结果。此外，您可以通过显着的时间和成本节省进行验证 - 系统更改为生产环境将导致净改进。

您可以使用SQL Performance Analyzer 分析任何类型的系统更改对SQL性能的影响，包括：

  • 数据库升级

  • 可插拔数据库（PDB）和手动架构合并的数据库整合测试

  • 配置更改操作系统或硬件

  • 架构更改

  • 更改数据库初始化参数

  • 刷新优化程序统计信息

  • 验证SQL调优操作

1.2数据库重放

在进行系统更改（例如硬件和软件升级）之前，通常在测试环境中执行大量测试以验证更改。 但是，尽管进行了测试，新系统在进入生产时经常会出现意外行为，因为测试不是使用实际工作负载来执行的。在测试期间无法模拟实际工作负载是验证系统更改时的最大挑战之一。

数据库重放通过在测试系统上重新创建生产工作负载环境，实现对系统更改的真实测试。使用数据库重放，您可以捕获生产系统上的工作负载，并使用原始工作负载的准确时间，并发性和事务特征在测试系统上重放它。这使您可以全面评估更改的影响，包括不良结果，新争用点或计划回归。提供广泛的分析和报告，以帮助识别任何潜在的问题，例如遇到的新错误和性能差异。

数据库重放在数据库级别捕获外部数据库客户端的工作负载，并且具有可忽略的性能开销。捕获生产工作负载消除了开发模拟工作负载或脚本的需要，从而显着降低成本并节省时间。通过使用数据库重放，现在可以在几天内完成以前使用负载模拟工具花费数月时间的复杂应用程序的实际测试。这使您能够以更高的置信度和更低的风险快速测试变更并采用新技术。

您可以使用数据库重放以测试任何重大系统更改，包括：

  • 数据库和操作系统升级

  • PDB的数据库整合测试和手动架构整合

  • 使用工作负载扩展来编写和试验各种方案

  • 配置更改，例如将数据库从单个实例转换为Oracle Real Application Clusters（Oracle RAC）环境

  • 存储，网络和互连更改

  • 操作系统和硬件迁移

为直接NFS客户端创建oranfstab文件

Direct NFS使用配置文件oranfstab来确定可用的挂载点。

使用Direct NFS Client为要访问的每个NFS服务器创建一个具有以下属性的oranfstab文件：

  • 服务器

    NFS服务器名称。

  • 本地

    数据库主机上最多有四个路径，由IP地址或名称指定，使用在数据库主机上运行的ifconfig命令显示。

  • 路径

    最多四个到NFS服务器的网络路径，由IP地址或名称指定，如NFS服务器上的ifconfig命令所示。

  • 出口

    NFS服务器的导出路径。

  • 安装

    导出卷的相应本地安装点。

  • mnt_timeout

    指定（以秒为单位）Direct NFS Client在超时之前应等待成功安装的时间。此参数是可选的。默认超时为10分钟（600）。

  • nfs_version

    指定Direct NFS Client使用的NFS协议版本。可能的值包括NFSv3，NFSv4，NFSv4.1和pNFS。默认版本为NFSv3。如果选择NFSv4.x，则必须在oranfstab为nfs_version配置值。如果要将Direct NFS与并行NFS一起使用，请将nfs_version指定为pNFS 。

  • security_default

    指定适用于服务器条目的所有导出的NFS服务器路径的默认安全模式。此参数是可选的。 sys是默认值。请参阅security参数支持的安全级别的security_default参数说明。

  • 安全

    指定安全级别，以使用带有Direct NFS Client的Kerberos身份验证协议启用安全性。可以根据导出安装对指定此可选参数。security_default和security参数支持的安全级别为：

    • sys ：UNIX级别安全性AUTH_UNIX基于用户标识符（UID）和组标识符（GID）值的身份验证。这是安全性参数的默认值。
    • krb5 ：Direct NFS使用普通Kerberos身份验证运行。服务器被认证为它声称的真实服务器。
    • krb5i ：Direct NFS运行Kerberos身份验证和NFS完整性。对服务器进行身份验证，并检查每个邮件传输的完整性。
    • krb5p ：Direct NFS运行Kerberos身份验证和NFS隐私。服务器已通过身份验证，并且所有数据都已完全加密。

    security参数（如果已指定）优先于security_default参数。如果未指定这些参数，则sys是默认身份验证。

    对于NFS服务器Kerberos安全设置，请查看相关的NFS服务器文档。对于Kerberos客户端设置，请查看相关的操作系统文档。

  • dontroute

    指定传出消息不应由操作系统路由，而是使用它们绑定的IP地址发送。

    注意：

    dontroute选项是POSIX选项，有时在同一子网中具有多个路径的Linux系统上不起作用。

  • 管理

    使Direct NFS Client能够使用管理接口进行SNMP查询。如果SNMP在NFS服务器上的单独管理接口上运行，则可以使用此参数。默认值是服务器参数值。

  • 社区

    指定用于SNMP查询的社区字符串。默认值为public 。

协议注意事项

DRDA是一种数据协议，与Oracle的SQL * Net数据协议有一些相似之处。虽然DRDA旨在在客户端和服务器之间移动关系数据，但它缺乏SQL * Net更强大的管理和路由控制。 DRDA和SQL * Net之间的主要区别是协议本身的语言。DRDA和SQL * Net不兼容，因此无法使用SQL * Net客户端连接到DRDA服务器，反之亦然。

DRDA使用的术语也类似于SQL * Net，一般概念转换为传统的Oracle定义，如以下示例所示：

  • 应用程序请求程序 （AR）是客户端程序用于创建基于SQL的请求并将其发送到应用程序服务器的接口。

  • 应用程序服务器 （AS）是服务器和数据库端程序，它代表客户端接受此类请求，执行数据库操作，并将结果数据返回给客户端。

两阶段提交和事务恢复

DRDA和DB2实现了两个支持事务提交和回滚的命令集。它们确保在事务更新期间维护数据完整性，并且如果连接或应用程序在提交事务时失败，则可以恢复这些更新。DRDA支持实现SingleSite和TwoPhase提交协议的命令。AS必须至少支持SingleSite承诺。

  • 单站点提交协议由一个简单的操作组成，该操作无法在可能涉及分布式事务的节点之间进行协调。它是提交数据的基本机制，并且被大多数常见应用程序使用。

  • 两阶段提交协议支持在同一节点或单独的网络节点上协调多个事务。

服务自治

Application Server位于Oracle数据库服务器的外部。因此，该应用程序具有广泛的位置选项。

在典型配置中，AS在与Oracle数据库相同的计算机上运行，如DRDA连接模型中的远程DB2应用程序部分所述。由于AS未紧密集成到Oracle数据库中，因此可以将其安装到自己的Oracle Home中，也可以安装在与客户端或数据库分开的计算机上， 如图2-1所示 。此中间层配置可实现服务资源的分离。它还允许更好地扩展服务，因为可以将单独的机器资源专用于AS和数据库。

包

与DRDA应用程序关联的资源称为包 。更具体地说，应用程序请求者利用包作为对应用程序执行操作的引用：包是存储语句的位置。该申请通过章节编号引用该陈述。应用程序语句有两大类：静态和动态。

  • 静态语句包含硬编码的SQL，其中SQL文本在应用程序运行期间不会更改。它们执行起来非常快，并且通常在运行之前进行优化以实现高性能。因为它是预定义的，所以静态SQL在第一次调用时的执行时间更短。

  • 动态语句主要是空占位符，有时也称为通用游标 。它们在运行时之前没有SQL文本，应用程序在运行期间构造它需要的实际SQL语句，并在运行时对它们进行优化。在第一次调用之后，通常会对已处理的动态SQL语句进行高速缓存，因此同一语句的后续执行时间与具有类似复杂性的静态SQL语句相当。

这些包是通过专有工具构建的。例如，在DB2应用程序环境中，开发人员经常编写包含嵌入式SQL语句的应用程序。应用程序源由SQL PreCompiler处理，类似于Oracle的Pro * C预编译器。输出通常后处理为源模块，以及源程序中使用的语句的磁盘上资源形式。在DB2术语中，这将创建数据库请求模块或DBRM。大多数创建此文件的实现都以专有格式将其存储在外部。

必须将DBRM的内容加载到远程数据库中，或者在执行时将其提供给AS。但是，加载专有格式的数据存在许多挑战。DRDA通过提供一组命令请求来远程将资源定义上载到目标AS中来解决此问题。大多数AR实现提供了在应用程序的SQL会话之前或期间上载资源的选项或工具。此过程称为绑定包 。

在将资源定义（DBRM）作为包绑定到远程数据库之后，AS可以提前加载它以获得更好的性能。

SQL方言

虽然Oracle部分符合ANSI SQL，但与当今大多数基于SQL的数据库系统一样，也有一些例外。数据库供应商以不同方式实现ANSI SQL标准;这导致SQL'方言'在语句执行期间提出了一些挑战。由于与DRDA一起使用的原始目标数据库是DB2，因此此处讨论的应用程序使用特定于DB2的SQL方言。

大多数数据操作语言（DML）和某些数据定义语言（DDL）已在ANSI SQL中针对常用对象（如表，视图，索引，简单过程或函数定义）进行了标准化。但是，每个数据库供应商仍将拥有自己的一组产品特定的DDL和DML扩展。 DRDA协议将SQL语句视为数据库必须处理的特定于数据库的实体;它向Oracle数据库表明它的SQL是DB2方言，并且它必须有一个翻译服务来处理它。此转换由SQL Translation Framework功能提供，在Oracle Database SQL Translation and Migration Guide中有详细描述。

目的

使用CREATE TABLESPACE SET语句创建表空间集。表空间集可以在分片数据库中用作一个或多个分片表和索引的逻辑存储单元。

表空间集包含分布在分片空间中的分片中的多个表空间。数据库自动在表空间集中创建表空间。表空间的数量是自动确定的，并且等于相应分片空间中的块数。

表空间集中的所有表空间都是永久的bigfile表空间;表空间集不包含SYSTEM ，undo或临时表空间。数据库自动为每个表空间创建一个数据文件。表空间集中的所有表空间共享相同的属性。您可以使用ALTER TABLESPACE SET语句修改表空间中所有表空间的属性。

PDB-list接受由glob模式表示的PDB名称列表，例如“PDB？”或“PDB * a”，“PDB2”。glob模式规则与UNIX shell的规则类似。星号（*）和问号（？）支持通配符。的？通配符恰好匹配一个未知字符，*通配符匹配任意数量的未知字符。此外，减号（ - ）字符可用作PDB名称中的第一个字符，以指示应在备用数据库上排除PDB。

以下字符在PDB名称中有效：字母数字字符，下划线（_），数字符号（＃）和美元符号（$）。没有其他字符有效。Oracle在处理PDB名称之前删除双引号。因此，您不能使用双引号在名称中嵌入其他字符。PDB名称不区分大小写。这些是与数据库名称相同的命名约定（如DB_NAME初始化参数说明中所述）。

这些规则适用于ENABLED_PDBS_ON_STANDBY参数：

  • 任何匹配以减号（ - ）开头的glob模式的PDB名称都不是备用数据库的一部分。

  • 如果PDB名称同时匹配PDB列表中的多个模式，则最右边匹配的模式具有最高优先级。

  • 如果未在init.ora文件中指定此参数，则假定默认情况下指定“*”。

  • 如果在此参数中指定了一个或多个模式，则假定在最左侧位置隐式指定“ - *”。例如， ENABLED_PDBS_ON_STANDBY=“PDB1”, “PDB2”等同于ENABLED_PDBS_ON_STANDBY=“-*”, “PDB1”, “PDB2” 。这意味着不允许与任何模式匹配的PDB名称成为备用数据库的一部分。

语法元素 描述

中止

执行目标实例的不一致关闭，结果如下：

  • 所有当前客户端SQL语句都会立即终止。

  • 未提交的事务在下次启动之前不会回滚。

  • 所有连接的用户都已断开

  • 在下次启动时对数据库执行实例恢复。

即时

对目标数据库执行立即，一致的关闭，并产生以下后果：

  • 允许数据库处理的当前客户端SQL语句完成。

  • 回滚未提交的事务。

  • 所有连接的用户都已断开

正常

使用普通优先级（默认选项）执行目标数据库的一致关闭，这意味着：

  • 发布声明后不允许新连接。

  • 在关闭之前，数据库会等待当前连接的用户断开连接

  • 下次启动数据库不需要实例恢复。

TRANSACTIONAL

执行目标数据库的一致关闭，同时最大限度地减少对客户端的干扰，并产生以下后果：

  • 允许当前正在进行事务的客户端完成，即在关闭之前提交或终止。

  • 没有客户端可以在此实例上启动新事务;尝试启动新事务的任何客户端都将断开连接。

  • 在所有事务已提交或终止后，任何仍连接的客户端都将断开连接。

执行以下步骤以克隆Oracle主目录。

注意：

    $ cd $ ORACLE_HOME / OPatch

    $ opatch lsinventory

  3. 使用Oracle主目录（但不是Oracle基础）目录创建ZIP或TAR文件。

    ＃zip -r dbhome_1.zip /u01/app/oracle/product/19.0.0/dbhome_1

    #tar -cvf dbhome_1.tar /u01/app/oracle/product/19.0.0/dbhome_1

    #unzip -d / dbhome_1.zip #tar-xvf dbhome_1.tar

    #chown -R oracle：oinstall /u01/app/oracle/product/19.0.0/dbhome_1

目的

方差函数的单向分析（ STATS_ONE_WAY_ANOVA ）通过比较两个不同的方差估计来测试均值（对于组或变量）的统计显着性差异。一个估计是基于每个组或类别内的差异。这被称为内的均方或均方误差 。另一项估计是基于各组平均值之间的差异。这被称为平均值之间的平方 。如果组的平均值明显不同，则两者之间的均方值将大于预期，并且与其中的均值平方不匹配。如果组的均方是一致的，那么两个方差估计将大致相同。

STATS_ONE_WAY_ANOVA接受两个必需参数： expr1是一个独立或分组变量，它将数据分成一组， expr2是一个因变量（一个数值表达式），包含对应于组中每个成员的值。可选的第三个参数允许您指定此函数返回的NUMBER值的含义，如表7-8所示。对于此参数，您可以指定文本文字，或指定为常量字符值的绑定变量或表达式。如果省略第三个参数，则默认为'SIG' 。

D.6 chactl查询诊断

使用chactl query diagnosis命令可以返回问题和诊断，并建议与特定群集节点或Oracle Real Application Clusters（Oracle RAC）数据库的问题相关的纠正措施。

句法

chactl查询诊断[-cluster | -db db_unique_name ] [-start time -end time ] [-htmlfile file_name ] [ -  help]

以YYYY-MM-DD HH24:MI:SS格式指定日期和时间。

在前面的语法中，您必须考虑以下几点：

  • 如果未提供任何选项，则chactl query diagnosis命令将返回所有受监视节点和数据库的当前状态。chactl query diagnosis命令通过显示其诊断标识符（例如， Storage Bandwidth Saturation报告目标的一般状态，例如ABNORMAL 。这是检查数据库或集群中任何ABNORMAL状态的快速方法。

  • 如果在目标名称后面提供时间选项，则chactl query diagnosis命令将指定目标的状态限制为指定时间间隔内的条件。压缩时间序列列出了在时间间隔，开始和结束时间中发生的不同事件的原因的标识符。

  • 如果事件和原因在特定时间间隔内重复出现，则问题仅报告一次。开始时间是第一次发生事件的开始时间，结束时间是特定时间间隔内最后一次事件发生的结束时间。

  • 如果指定不带数据库名称的–db选项，则chactl query diagnosis命令将显示所有数据库的诊断信息。但是，如果指定了数据库名称，则chactl query diagnosis命令将显示正在监视的所有数据库实例的诊断信息。

  • 如果指定不带主机名的–cluster选项，则chactl query diagnosis命令将显示该群集中所有主机的诊断信息。

  • 如果未指定时间间隔，则chactl query diagnosis命令仅显示所有或指定目标的当前问题。chactl query diagnosis命令不会显式显示频率统计信息。但是，您可以计算过去24小时内目标中发生的正常和异常事件的数量。

  • 如果在指定的时间间隔内未发生任何事件，则chactl query diagnosis命令将返回文本消息，例如， 数据库/主机正在运行NORMALLY ，或者未发现任何事件 。

  • 如果目标的状态为NORMAL ，则命令不报告它。chactl query diagnosis命令仅报告指定时间间隔内具有ABNORMAL状态的目标。

输出参数：

  • 事件开始时间

  • 事件结束时间（仅适用于默认数据库和/或主机，非详细输出）

  • 目标（例如，数据库，主机）

  • 问题

    说明：问题的详细说明

    原因：问题的根本原因和影响因素

  • 操作：纠正诊断中涵盖的异常状态的操作

报告格式：诊断信息以时间压缩或时间序列顺序显示，按组件分组。

安全概述

当您连接多个不同的系统时，通常具有最严格安全要求的系统会指示和规则系统。

网关安全涉及两组：

  • 允许访问给定网关实例和DRDA数据库服务器的用户和应用程序

  • 用户和应用程序能够查询和更新的服务器数据库对象

您可以在多个点控制网关体系结构中的访问。每个DRDA数据库服务器都使用GRANT和基于用户ID的相关本机授权机制来控制数据库对象访问。

当网关涉及SQL请求时，安全机制对网关遇到的每个DRDA系统组件都有效。遇到的第一个系统组件是应用程序工具或3GL程序。遇到的最后一个系统组件是DRDA数据库。

验证应用程序登录

在使用之前，应用程序必须连接到Oracle数据库网关。您使用的登录身份验证类型决定了生成的Oracle用户ID，并且可能会影响网关操作。有两种基本类型的身份验证：

  • Oracle身份验证：使用Oracle身份验证时，每个Oracle用户ID都具有Oracle数据库已知的密码。当应用程序连接到服务器时，它会提供用户标识和密码。Oracle数据库确认存在用户标识，并且密码与数据库中保存的密码相匹配。

  • 操作系统身份验证：使用操作系统身份验证，操作系统下的服务器负责身份验证。使用操作系统身份验证访问使用IDENTIFIED EXTERNALLY属性创建的Oracle用户标识，而不是密码。要登录此类用户ID，应用程序会为用户ID提供正斜杠（/），并且不提供密码。

    要执行操作系统身份验证，服务器会确定请求者的操作系统用户ID，可选地为其添加固定前缀，并将结果用作Oracle用户ID。服务器确认用户ID存在且IDENTIFIED EXTERNALLY已IDENTIFIED EXTERNALLY ，但未进行密码检查。基本假设是用户在登录操作系统时进行了身份验证。

    操作系统身份验证并非在所有平台上都可用，并且在某些平台上不可用 Oracle Net （客户端 - 服务器）和多线程服务器配置。请参阅数据库安装指南和Oracle Net文档以确定此功能的可用性。

有关验证应用程序登录的详细信息，请参阅Oracle数据库开发指南 。

定义和控制数据库链接

此处的信息特定于网关。有关数据库链接的其他信息，请参阅“ Oracle数据库管理员指南” 。

链接可访问性

给定用户应该可以访问数据库链接。任何用户ID都可以使用公共数据库链接。私有数据库链接只能由创建它的用户使用。服务器不区分使用类型（例如只读与更新或写入）或远程对象的可访问性。访问的DRDA数据库负责这些区别。

链接和CONNECT子句

CONNECT子句是数据库链接的另一个与安全相关的属性。您可以使用CONNECT子句指定显式用户标识和密码，该标识可以与用户的Oracle数据库用户标识和密码不同。首次打开数据库链接连接时，此CONNECT用户标识和密码组合将发送到网关。根据网关选项，网关可能会将该用户标识和密码发送到DRDA服务器以进行验证。

如果创建的数据库链接没有CONNECT子句，则在打开连接时CONNECT用户的Oracle数据库用户ID和密码发送到网关。如果用户使用操作系统身份验证登录Oracle数据库，则网关不会从Oracle数据库接收任何用户ID或密码。在这种情况下，如果同一主机上的所有用户都可以使用相同的DRDA数据库用户ID，则DRDA服务器上的用户ID映射工具可用于实现此类连接。

网关初始化文件中的密码

目的

PERCENTILE_CONT是一个逆分布函数，它假设一个连续的分布模型。它采用百分位值和排序规范，并返回一个内插值，该值将根据排序规范落入该百分位数值。在计算中忽略空值。

此函数将任何数值数据类型或任何可以隐式转换为数字数据类型的非数字数据类型作为参数。该函数返回与参数的数值数据类型相同的数据类型。

第一个expr必须求值为0到1之间的数值，因为它是百分位值。此expr在每个聚合组中必须是常量。ORDER BY子句采用必须是数字或日期时间值的单个表达式，因为这些是Oracle可以执行插值的类型。

PERCENTILE_CONT的结果是在排序后通过值之间的线性插值计算的。使用聚合组中的百分位值（P）和行数（N），可以在根据排序规范对行进行排序后计算您感兴趣的行号。根据公式RN = (1+(P*(N-1))计算该行号（RN (1+(P*(N-1)) 。通过行号CRN = CEILING(RN)和FRN = FLOOR(RN)行之间的线性插值来计算聚合函数的最终结果。

最终结果将是：

如果（CRN = FRN = RN）则结果是（来自RN的行的表达式的值）否则结果是（CRN-RN）*（FRN处的行的表达式的值）+（RN-FRN）*（的值） CRN行的表达式）

您可以使用PERCENTILE_CONT函数作为分析函数。您只能在其OVER子句中指定query_partitioning_clause 。它为每一行返回在每个分区中的一组值中落入指定百分位数的值。

MEDIAN函数是PERCENTILE_CONT的特定情况，其中百分位数值默认为0.5。有关更多信息，请参阅MEDIAN 。

注意：

在使用PERCENTILE_CONT函数处理大量数据之前，请考虑使用以下方法之一来获得比精确结果更快的近似结果：

  • 在使用PERCENTILE_CONT函数之前，将APPROX_FOR_PERCENTILE初始化参数设置为PERCENTILE_CONT或ALL 。有关此参数的详细信息，请参阅Oracle数据库参考 。
