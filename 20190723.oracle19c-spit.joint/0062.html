确定性

DETERMINISTIC子句在函数声明中最多只出现一次，在函数定义中最多出现一次。

DETERMINISTIC函数可能不会引发未处理的异常。

使用说明

DETERMINISTIC子句是一个断言，该函数服从语义规则。如果函数没有，编译器，SQL执行或PL / SQL执行都不能诊断问题，并且可能会无声地生成错误的结果。

不要为结果取决于会话变量或模式对象状态的函数指定DETERMINISTIC ，因为结果可能因调用而异。

不要指定此子句来定义使用包变量的函数，或以任何可能影响函数返回结果的方式访问数据库的函数。

不允许为多态表函数指定此子句。

当出现DETERMINISTIC选项时，编译器可以使用该标记来提高函数执行的性能。

制作属于这些类别的函数是一种很好的编程习惯DETERMINISTIC ：

  • WHERE ， ORDER BY或GROUP BY子句中使用的函数

目的

LAST_VALUE是一种分析函数，可用于数据密集化。它返回有序值集中的最后一个值。

注意：

此语法的两种形式具有相同的行为。顶部分支是ANSI格式，Oracle建议ANSI兼容。

{ RESPECT | IGNORE } NULLS确定expr空值是否包含在计算中或从计算中消除。默认值为RESPECT NULLS 。如果设定的最后一个值是零，则函数返回NULL ，除非您指定IGNORE NULLS 。如果指定IGNORE NULLS LAST_VALUE ，则LAST_VALUE返回LAST_VALUE的最后一个非空值，如果所有值都为null，则返回NULL 。有关数据密集化的示例，请参阅“ 使用分区外连接：示例 ” 。

您不能通过使用LAST_VALUE或expr任何其他分析函数来嵌套分析函数。但是，您可以为expr使用其他内置函数表达式。有关有效形式的expr信息，请参阅“ 关于SQL表达式 ” 。

如果省略analytic_clause的windowing_clause ，则默认为RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW 。此默认值有时会返回意外值，因为窗口中的最后一个值位于窗口的底部，而不是固定的。它会随着当前行的变化而不断变化。对于预期的结果，指定windowing_clause为RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING 。或者，您可以将windowing_clause指定为RANGE BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING 。

分区和表压缩

可以在多个分区或完整的分区堆组织表上执行压缩。

您可以通过将完整的分区表定义为压缩或通过在每个分区级别定义来执行此压缩。没有特定声明的分区从表定义继承该属性，或者，如果在表级别没有指定任何内容，则从表空间定义继承该属性。

分区是应该压缩还是未压缩的决定遵循与非分区表相同的规则。但是，由于分区将数据逻辑分离到不同的分区，这样的分区表是压缩部分数据（分区）的理想选择。例如，在老化数据老化之前，在所有滚动窗口操作中作为中间阶段是有益的。通过数据段压缩，您可以在线保留更多旧数据，从而最大限度地减少额外存储消耗的负担。

您也可以稍后更改任何现有的未压缩表分区，添加新的压缩和未压缩分区，或者将压缩属性更改为需要数据移动的任何分区维护操作的一部分，例如MERGE PARTITION ， SPLIT PARTITION或MOVE PARTITION 。分区可以包含数据，也可以为空。

部分或完全压缩的分区表的访问和维护与完全未压缩的分区表的访问和维护相同。适用于完全未压缩的分区表的所有内容对部分或完全压缩的分区表也有效。

表压缩和位图索引

在使用位图索引的分区表上使用压缩之前，有几个必要的步骤。

要对具有位图索引的分区表使用表压缩，必须在首次引入压缩属性之前执行以下操作：

  1. 标记位图索引不可用。

  2. 设置压缩属性。

  3. 重建索引。

第一次将压缩分区作为现有的完全未压缩分区表的一部分时，必须先删除所有现有位图索引，或者在添加压缩分区之前将其标记为UNUSABLE 。无论任何分区是否包含任何数据，都必须这样做。它还与导致一个或多个压缩分区成为表的一部分的操作无关。这不适用于仅具有B树索引的分区表。

这种位图索引结构的重建对于在启用表压缩的情况下容纳为每个数据块存储的潜在更多行数是必要的。必须首次启用表压缩。所有后续操作（无论它们是否影响压缩或未压缩分区）或更改压缩属性，对于未压缩，部分压缩或完全压缩的分区表的行为都相同。

为避免重新创建任何位图索引结构，Oracle建议您在计划将来部分或完全压缩分区表时，使用至少一个压缩分区创建每个分区表。此压缩分区可以保持为空，甚至可以在创建分区表后删除。

具有压缩分区的分区表可以导致未压缩分区的稍大位图索引结构。但是，压缩分区的位图索引结构通常小于表压缩之前的适当位图索引结构。这在很大程度上取决于实现的压缩率。

注意：

如果首次将压缩引入对象并且存在可用的位图索引段，则Oracle数据库会引发错误。

异构复制

可以使用物化视图在非Oracle系统和Oracle数据库之间复制数据。

注意：

还有另一种在Oracle和非Oracle数据库之间复制信息的方法 Oracle GoldenGate。

有关使用Oracle GoldenGate的信息，请参阅Oracle GoldenGate文档。

物化视图在特定时间实例化从非Oracle主站点的表中捕获的数据。此瞬间由刷新操作定义，该操作将此数据复制到Oracle数据库，并将Oracle系统上的副本与非Oracle系统上的主副本同步。然后，物化数据可作为Oracle数据库的视图使用。

复制工具提供了计划刷新和将实体化视图收集到复制组以便于管理的机制。刷新组允许刷新多个物化视图，就像它们是单个对象一样。

异构复制支持必须限于完整的Oracle到Oracle复制功能的子集：

  • 只有非Oracle系统才能成为主站点。这是因为只能在Oracle数据库上创建物化视图。

  • 物化视图必须使用完整的刷新。这是因为快速刷新需要非Oracle系统中特定于Oracle的功能。

  • 并非所有类型的物化视图都可以创建用于引用非Oracle系统上的表。支持主键和子查询物化视图，但不支持ROWID和OBJECT ID物化视图。这是因为ROWID的格式和内容没有SQL标准，非Oracle系统不实现Oracle对象。

其他限制适用于通过Oracle的异构服务工具访问非Oracle数据。其中最重要的是：

  • 映射到固定视图的表列中的非Oracle数据类型必须与Oracle数据类型兼容（即具有到或来自的映射）。对于ANSI SQL标准定义的数据类型，通常都是如此。

  • 子查询物化视图可能无法使用受单个非Oracle系统限制的语言功能。在许多情况下，异构服务通过在Oracle数据库中处理查询来支持此类语言功能。有时，非Oracle系统会施加在异构服务尝试执行查询之前无法检测到的限制。

TO_UTC_TIMESTAMP_TZ

句法


插图描述to_utc_timestamp_tz.eps

目的

SQL函数TO_UTC_TIMESTAMP_TZ将ISO 8601日期格式字符串作为varchar输入，并返回SQL数据类型TIMESTAMP WITH TIMEZONE的实例。它将输入标准化为UTC时间（协调世界时，以前格林威治标准时间）。与SQL函数TO_TIMESTAMP_TZ不同，新函数假定输入字符串使用ISO 8601日期格式，将时区默认为UTC 0。

如果输入varchar不符合ISO 8601格式，您将看到ORA-40688错误。

此函数的典型用法是将其输出提供给SQL函数SYS_EXTRACT_UTC ，获取UTC时间，然后将其作为SQL绑定变量传递给SQL / JSON条件JSON_EXISTS ，以执行时间戳范围比较。

这是日期和时间的允许语法：

  • 日期（仅）： YYYY - MM - DD

  • 日期与时间： YYYY - MM - DD T hh : mm : ss [ . s [ s [ s [ s [ s [ s ]]]]][ Z |( + | - ) hh : mm ]

哪里：

  • YYYY指定年份 ，为四位小数。

  • MM指定月份 ，为两位小数， 00到12 。

  • DD将日期指定为两位十进制数字00到31 。

  • hh指定小时 ，为两位小数， 00到23 。

  • mm指定分钟 ，为两位小数， 00到59 。

  • ss [ . s [ s [ s [ s [ s ]]]]]指定秒 ，作为两个十进制数字， 00到59 ，可选地后跟一个小数点和1到6个十进制数字（表示秒的小数部分）。

自动大表缓存使并行查询和串行查询能够使用缓冲区缓存，从而增强了Oracle数据库的内存查询功能。自动大表缓存主要用于增强数据仓库工作负载的性能，但它也可以提高混合工作负载的性能。

从Oracle Database 12 c第1版（12.1.0.2）开始，表扫描可以在以下方案中使用不同的算法：

  • 并行查询：

    在单实例和Oracle Real Application Clusters（Oracle RAC）数据库中，当DB_BIG_TABLE_CACHE_PERCENT_TARGET初始化参数设置为非零值且并且PARALLEL_DEGREE_POLICY初始化参数设置为AUTO或ADAPTIVE时，并行查询可以使用自动大表缓存。

  • 串口查询：

    仅在单实例配置中，当DB_BIG_TABLE_CACHE_PERCENT_TARGET初始化参数设置为非零值时，串行查询可以使用自动大表缓存。

为DB_BIG_TABLE_CACHE_PERCENT_TARGET参数指定非零值时，该值指示要为大表缓存保留的缓冲区高速缓存的百分比。可以使用DB_BIG_TABLE_CACHE_PERCENT_TARGET参数指定的DB_BIG_TABLE_CACHE_PERCENT_TARGET为90，除了表扫描之外， DB_BIG_TABLE_CACHE_PERCENT_TARGET保留10％的缓冲区高速缓存。

此参数的默认值为0 。因此，默认情况下不启用自动大表缓存。如果未启用自动大表缓存，则查询可能使用最近最少使用（LRU）机制进行缓存读取，或者可能决定对表扫描使用直接读取。

如果一个大表大小与所有实例的大表缓存大小相当，那么该表将被分区并缓存或大部分缓存在所有实例上。使用内存中并行查询，这可以消除表上查询的大多数磁盘读取，或者数据库只能从磁盘中智能地读取不适合大表缓存的表部分。如果大表缓存无法缓存要扫描的所有表，则只缓存最常访问的表，其余表将通过直接读取自动读取。

设置参数时请使用以下准则：

  • 如果未在Oracle RAC环境中启用自动并行度（DOP），请不要设置此参数，因为在该情况下不使用大表缓存。

  • 设置此参数时，请考虑工作负载组合：OLTP的工作负载量是多少;插入，更新和随机访问;以及有多少工作负载涉及表扫描。由于数据仓库工作负载通常执行大型表扫描，因此您可以考虑为大型表缓存部分提供更高百分比的缓冲区缓存空间用于数据仓库。

  • 如果工作负载发生变化，则可以动态更改此参数。更改可能需要一些时间才能达到目标（取决于当前工作负载），因为此时可能会主动使用缓冲区高速缓存。

注意：

自动大表缓存使用温度和基于对象的算法来跟踪中型和大型表。Oracle将缓存非常小的表，但它们不会被自动大表缓存跟踪。

脚注1

通过将THREADED_EXECUTION初始化参数设置为true来启用多线程Oracle。

当此参数设置为false ，侦听器直接生成专用服务器。

当此参数设置为true ，侦听器将连接交给专用连接代理，该代理生成专用服务器。与侦听器不同，专用连接代理是数据库进程，因此它可以在生成之前实现利用数据库信息的策略。

要使用USE_DEDICATED_BROKER初始化参数启用专用连接代理，您必须：

  1. 将USE_DEDICATED_BROKER初始化参数设置为true 。

  2. 在listener.ora文件中将DEDICATED_THROUGH_BROKER_ listener-name参数设置为on 。

执行这两个步骤可配置连接代理进程（N nnn ）。

注意：

启用多线程Oracle时，还会启用并使用专用连接代理进程。

经纪人的配置由CONNECTION_BROKERS初始化参数控制。
