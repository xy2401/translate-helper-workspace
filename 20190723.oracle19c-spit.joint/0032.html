升级后收集字典统计信息

为了确保良好的性能，请在完成升级后使用此过程收集字典统计信息。

Oracle建议您在升级数据库之前和之后收集字典统计信息，因为在升级期间会修改和创建数据字典表。对于Oracle Database 12 c第 2版（12.2）及更高版本，当您以正常模式启动数据库时，可以在升级后将统计信息收集为手动过程。

  • 非CDB Oracle数据库：Oracle建议您使用DBMS_STATS.GATHER_DICTIONARY_STATS过程来收集这些统计信息。例如，输入以下SQL语句：

    SQL> EXEC DBMS_STATS.GATHER_DICTIONARY_STATS;

  • CDB：Oracle建议您使用catcon在整个多租户架构中收集数据字典统计信息

    要收集容器数据库中所有PDB的字典统计信息，请使用以下语法

    $ ORACLE_HOME / perl / bin / perl $ ORACLE_HOME / rdbms / admin / catcon.pl -l / tmp -b gatherstats - --x“exec dbms_stats.gather_dictionary_stats”

    要收集特定PDB上的字典统计信息，请使用类似于以下内容的语法：

    $ ORACLE_HOME / perl / bin / perl $ ORACLE_HOME / rdbms / admin / catcon.pl -l / tmp -c'SALES1'-b gatherstats  -  --x“exec dbms_stats.gather_dictionary_stats”

    在前面的示例中， -c SALES1选项为您运行的命令指定PDB包含列表，指定名为SALES1的数据库。选项-b gatherstats指定日志的基本名称。选项--x指定要执行的SQL命令。SQL命令本身在引号内。

SCN_TO_TIMESTAMP

句法


scn_to_timestamp.eps插图的描述

目的

SCN_TO_TIMESTAMP将一个数字作为参数，该数字的计算结果为系统更改号（SCN），并返回与该SCN关联的大致时间戳。返回值为TIMESTAMP数据类型。只要您想知道与SCN关联的时间戳，此功能就很有用。例如，它可以与ORA_ROWSCN伪列结合使用，以将时间戳与行的最新更改相关联。

笔记：

  • 结果值的通常精度为3秒。

  • 数据库在一段有限的时间内记住SCN与生成SCN时的时间戳之间的关联。如果数据库以自动撤消管理模式运行，并且数据库中所有闪回存档的保留时间不少于120小时，则此期间是自动调整的撤消保留期的最大值。只有在数据库打开时，关联才会过时。如果为SCN_TO_TIMESTAMP的参数指定的SCN太旧， SCN_TO_TIMESTAMP返回错误。

关于多个Oracle Homes支持

Oracle数据库支持多个Oracle主目录。您可以在同一系统上，在不同的Oracle主目录中多次安装此版本或更早版本的软件。

仔细选择安装点名称可以使Oracle软件更易于管理。按照Optimal Flexible Architecture（OFA）规则配置多个Oracle主目录具有以下优势：

  • 您可以在同一系统上，在不同的Oracle主目录中多次安装此发行版或该软件的早期版本。但是，您无法将一个Oracle数据库版本的产品安装到另一个版本的Oracle主目录中。例如，您无法将Oracle Database 19c软件安装到现有的Oracle 18c Oracle主目录中。

  • 不同用户拥有的不同版本的多个数据库可以同时共存。

  • 您必须在与早期版本的Oracle数据库分开的新Oracle主目录中安装新的Oracle数据库发行版。

    您无法在一个Oracle主目录中安装多个版本。Oracle建议您根据Optimal Flexible Architecture（OFA）指南为每个版本创建单独的Oracle Database Oracle主目录。

  • 在生产中，Oracle数据库服务器软件版本必须与前几个数字（主要版本，维护版本和修补程序版本号）的Oracle数据库字典版本相同。

  • 后来的Oracle Database版本可以访问早期的Oracle Database版本。但是，此访问权限仅用于升级。例如，如果18c数据库以升级模式启动，Oracle Database 19c可以访问Oracle数据库18c。

  • 如果两个产品处于同一版本级别，则Oracle Database Client可以安装在同一Oracle数据库主目录中。例如，您可以将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.2.0.1主目录中，但不能将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.1.0.2主目录中。如果在安装客户端之前应用修补程序集，则必须再次应用修补程序集。

  • 目录和文件的结构化组织以及数据库文件的一致命名简化了数据库管理。

  • 当数据库管理员添加，移动或删除Oracle主目录时，登录主目录不会有风险。

  • 您可以在与生产数据库所在的Oracle主目录不同的目录中测试Oracle主目录中的软件升级。

POWERMULTISET

句法


插图描述powermultiset.eps

目的

POWERMULTISET将嵌套表作为输入，并返回嵌套表的嵌套表，该表包含输入嵌套表的所有非空子集（称为子矩阵）。

  • expr可以是任何求值为嵌套表的表达式。

  • 如果expr解析为null，则Oracle数据库返回NULL 。

  • 如果expr解析为空的嵌套表，则Oracle返回错误。

  • 嵌套表的元素类型必须具有可比性。有关非标量类型可比性的信息，请参阅“ 比较条件 ” 。

    注意：

    PL / SQL不支持此功能。

12.2 Hang Manager的可选配置

您可以调整灵敏度，并控制Hang Manager使用的日志文件的大小和数量。

灵敏度

如果Hang Manager检测到挂起，则Hang Manager会等待某个阈值时间段以确保会话挂起。通过使用DBMS_HANG_MANAGER将sensitivity参数设置为“ Normal或“ High来更改阈值时间段。如果sensitivity参数设置为“ Normal ，则Hang Manager将等待默认时间段。但是，如果灵敏度设置为High ，则时间段减少50％。

默认情况下， sensitivity参数设置为Normal 。要设置Hang Manager灵敏度，请在SQL * Plus中以SYS用户身份运行以下命令：

  • 要将sensitivity参数设置为Normal ：
    exec dbms_hang_manager.set（dbms_hang_manager.sensitivity，dbms_hang_manager.sensitivity_normal）;
  • 要将sensitivity参数设置为High ：
    exec dbms_hang_manager.set（dbms_hang_manager.sensitivity，dbms_hang_manager.sensitivity_high）;

跟踪日志文件的大小

Hang Manager使用文件名中的_base_记录跟踪文件中挂起的详细诊断。使用base_file_size_limit参数更改跟踪文件的大小（以字节为base_file_size_limit 。例如，在SQL * Plus中运行以下命令，将跟踪文件大小限制设置为100 MB：
exec dbms_hang_manager.set（dbms_hang_manager.base_file_size_limit，104857600）;

跟踪日志文件的数量

基本的Hang Manager跟踪文件是跟踪文件集的一部分。使用base_file_set_count参数更改跟踪文件集中的跟踪文件数。例如，在SQL * Plus中运行以下命令，将跟踪文件中的跟踪文件数设置为6：
exec dbms_hang_manager.set（dbms_hang_manager.base_file_set_count，6）;

默认情况下， base_file_set_count参数设置为5。

SOUNDEX

句法

目的

SOUNDEX返回包含char的语音表示的字符串。此功能可让您比较拼写不同的单词，但英语听起来相似。

语音表示在Donald E. Knuth 的“计算机编程艺术 ，第3卷：排序和搜索”中定义如下：

  1. 保留字符串的第一个字母并删除以下字母的所有其他出现：a，e，h，i，o，u，w，y。

  2. 将数字分配给剩余的字母（在第一个之后），如下所示：

    b，f，p，v = 1 c，g，j，k，q，s，x，z = 2 d，t = 3 l = 4 m，n = 5 r = 6

  3. 如果两个或多个具有相同编号的字母在原始名称中相邻（在步骤1之前），或者除了任何插入的h和w之外相邻，则保留第一个字母并省略具有相同编号的所有相邻字母的其余部分。

  4. 返回用0填充的前四个字节。

char可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 。返回值与char数据类型相同。

此函数不直接支持CLOB数据。但是， CLOB可以通过隐式数据转换作为参数传递。

根据Oracle维护类型升级表

从Oracle Database 12 c第2版（12.2）及更高版本开始，您必须手动升级依赖于Oracle维护类型的用户表。

如果数据库具有依赖于Oracle维护类型的用户表（例如， AQ队列表），则在升级后运行utluptabdata.sql命令，以对受Oracle维护更改影响的任何用户表执行ALTER TABLE UPGRADE类型。此行为更改使用户表在升级期间保持READ ONLY状态。禁止用户使用SYSDBA权限（ AS SYSDBA ）登录应用程序，并更改依赖于Oracle维护类型的应用程序表。

要确定数据库升级完成后需要升级的表，请连接到数据库AS SYSDBA ，然后运行以下查询：

COLUMN所有者FORMAT A30 COLUMN table_name FORMAT A30 SELECT DISTINCT所有者，table_name FROM dba_tab_cols WHERE data_upgraded ='NO'ORDER BY 1,2;

此查询列出未列为UPGRADED所有表。但是， utluptabdata.sql脚本仅升级依赖于Oracle维护类型的表。如果查询列出了任何表，则运行utluptabdata.sql脚本以在依赖用户表上执行ALTER TABLE UPGRADE命令，以便将这些Oracle维护类型升级到该类型的最新版本。

当参数SERVEROUTPUT设置为ON ， utluptabdata.sql脚本将显示所有已升级表的名称，并列出表升级期间遇到的任何错误。要将服务器输出设置为ON ，请运行以下命令：

SET SERVEROUTPUT ON @ utluptabdata.sql

  5. 升级后，授予特定的所需权限。访问权限基于原始数据库中的使用情况。

关于按顺序升级可插拔数据库（PDB）

您可以通过从早期版本的CDB中拔出PDB，将其插入更高版本的CDB，然后将该PDB升级到更高版本来升级PDB。

CDB可以包含零个，一个或多个可插拔数据库（PDB）。安装新的Oracle数据库版本后，或升级CDB（ CDB$ROOT ）后，可以升级一个或多个PDB，而无需升级CDB上的所有PDB。

您可以选择满足服务交付需求的升级计划。例如，您可以使用Oracle数据库升级助手（DBUA）升级一组PDB，也可以使用手动升级单独升级PDB，或使用包含或排除列表。您可以升级CDB和所有PDB（并行手动升级），也可以升级CDB，然后依次升级PDB，可以使用包含或排除列表单独或成组升级。

以下是顺序PDB升级所需步骤的高级列表：

  1. 从早期版本的CDB中拔出早期版本的PDB。

  2. 从CDB中删除PDB。

  3. 将早期版本的PDB插入更高版本的CDB。

  4. 将早期版本的PDB升级到更高版本。

使用Oracle Database 12 c第2版（12.2）及更高版本，您可以向Parallel Upgrade Utility提供列表以升级PDB：

  • 优先级列表，用于设置PDB升级的顺序

  • 包含列表，使您可以在升级优先级列表中列出的PDB后指定要升级的一组PDB

  • 排除列表，使您可以指定一组未升级的PDB

注意：

除非备份PDB，否则无法恢复PDB。使用创建CDB并插入PDB的方法进行升级后，请务必备份PDB。

关于多个Oracle Homes支持

Oracle数据库支持多个Oracle主目录。您可以在同一系统上，在不同的Oracle主目录中多次安装此版本或更早版本的软件。

仔细选择安装点名称可以使Oracle软件更易于管理。按照Optimal Flexible Architecture（OFA）规则配置多个Oracle主目录具有以下优势：

  • 您可以在同一系统上，在不同的Oracle主目录中多次安装此发行版或该软件的早期版本。但是，您无法将一个Oracle数据库版本的产品安装到另一个版本的Oracle主目录中。例如，您无法将Oracle Database 19c软件安装到现有的Oracle 18c Oracle主目录中。

  • 不同用户拥有的不同版本的多个数据库可以同时共存。

  • 您必须在与早期版本的Oracle数据库分开的新Oracle主目录中安装新的Oracle数据库发行版。

    您无法在一个Oracle主目录中安装多个版本。Oracle建议您根据Optimal Flexible Architecture（OFA）指南为每个版本创建单独的Oracle Database Oracle主目录。

  • 在生产中，Oracle数据库服务器软件版本必须与前几个数字（主要版本，维护版本和修补程序版本号）的Oracle数据库字典版本相同。

  • 后来的Oracle Database版本可以访问早期的Oracle Database版本。但是，此访问权限仅用于升级。例如，如果18c数据库以升级模式启动，Oracle Database 19c可以访问Oracle数据库18c。

  • 如果两个产品处于同一版本级别，则Oracle Database Client可以安装在同一Oracle数据库主目录中。例如，您可以将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.2.0.1主目录中，但不能将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.1.0.2主目录中。如果在安装客户端之前应用修补程序集，则必须再次应用修补程序集。

  • 目录和文件的结构化组织以及数据库文件的一致命名简化了数据库管理。

  • 当数据库管理员添加，移动或删除Oracle主目录时，登录主目录不会有风险。

  • 您可以在与生产数据库所在的Oracle主目录不同的目录中测试Oracle主目录中的软件升级。

  • 水平实现 （在一个层内）描述了一种角色分离方法，该方法使用分配给服务器池和策略管理数据库或应用程序的访问控制列表授予的资源的访问权限来限制一个层内的资源访问。

关于多个Oracle Homes支持

Oracle数据库支持多个Oracle主目录。您可以在同一系统上，在不同的Oracle主目录中多次安装此版本或更早版本的软件。

仔细选择安装点名称可以使Oracle软件更易于管理。按照Optimal Flexible Architecture（OFA）规则配置多个Oracle主目录具有以下优势：

  • 您可以在同一系统上，在不同的Oracle主目录中多次安装此发行版或该软件的早期版本。但是，您无法将一个Oracle数据库版本的产品安装到另一个版本的Oracle主目录中。例如，您无法将Oracle Database 19c软件安装到现有的Oracle 18c Oracle主目录中。

  • 不同用户拥有的不同版本的多个数据库可以同时共存。

  • 您必须在与早期版本的Oracle数据库分开的新Oracle主目录中安装新的Oracle数据库发行版。

    您无法在一个Oracle主目录中安装多个版本。Oracle建议您根据Optimal Flexible Architecture（OFA）指南为每个版本创建单独的Oracle Database Oracle主目录。

  • 在生产中，Oracle数据库服务器软件版本必须与前几个数字（主要版本，维护版本和修补程序版本号）的Oracle数据库字典版本相同。

  • 后来的Oracle Database版本可以访问早期的Oracle Database版本。但是，此访问权限仅用于升级。例如，如果18c数据库以升级模式启动，Oracle Database 19c可以访问Oracle数据库18c。

  • 如果两个产品处于同一版本级别，则Oracle Database Client可以安装在同一Oracle数据库主目录中。例如，您可以将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.2.0.1主目录中，但不能将Oracle Database Client 12.2.0.1安装到现有的Oracle Database 12.1.0.2主目录中。如果在安装客户端之前应用修补程序集，则必须再次应用修补程序集。

  • 目录和文件的结构化组织以及数据库文件的一致命名简化了数据库管理。

  • 当数据库管理员添加，移动或删除Oracle主目录时，登录主目录不会有风险。

  • 您可以在与生产数据库所在的Oracle主目录不同的目录中测试Oracle主目录中的软件升级。

Oracle Flex Cluster手动分配的地址

查看此信息以手动为群集节点分配群集节点VIP名称。

手动分配地址的要求

如果选择配置手动分配的VIP，则在安装期间，必须使用以下选项之一为所有群集节点配置群集节点VIP名称：

  • 手动名称：手动输入每个节点的主机名和虚拟IP名称。您提供的名称必须解析为DNS上配置的地址。名称必须符合RFC 952标准，该标准允许使用字母数字字符和连字符（“ - ”），但不允许使用下划线（“_”）。

  • 自动分配的名称：输入与您在DNS上配置的主机名对应的值的字符串变量。字符串变量允许您在安装期间快速分配大量名称。使用以下特征在DNS上配置地址：

    • 主机名前缀：在DNS上配置的每个地址中使用的前缀字符串，供集群成员节点使用。例如： mycloud 。

    • 范围：要分配给集群成员节点的一系列数字，由起始节点编号和结束节点编号组成，指定范围的结尾。例如： 001和999 。

    • 节点名称后缀：在范围编号结束后添加到公共节点名称的后缀。例如： nd 。

    • VIP名称后缀：在虚拟IP节点名称结束后添加的后缀。例如： -vip 。

句法

您可以使用字母数字字符串创建手动地址。

使用只读表空间运行升级

使用带有-T选项的Parallel Upgrade Utility可在升级期间使基于模式的表空间脱机。

Oracle数据库可以读取在早期版本中创建的文件头，因此在升级期间不需要对它们执行任何操作。READ ONLY表空间的文件头在更改为READ WRITE时会更新。

如果升级遇到灾难性错误，则升级无法使表空间重新联机，请查看升级日志文件。日志文件包含使表空间可用所需的实际SQL语句。要使表空间重新联机，必须在数据库的日志文件中运行SQL语句，或运行每个PDB的日志文件。

在升级日志文件中查看表空间命令

如果发生灾难性升级失败，则可以导航到日志目录（ Oracle_base /cfgtoologs/dbua ），并手动运行日志文件中的命令以调出表空间。您可以在以下日志文件中查看表空间命令：

  • 非CDB升级： catupgrd0.log

  • PDB数据库： catupgrd pdbname 0.log ，其中pdbname是要升级的PDB的名称。

在每个日志文件的开头，您可以找到以下SQL语句，这些语句将表设置为READ ONLY ：

SQL> ALTER TABLESPACE ARGROTBLSPA6只读;表空间已更改。SQL> ALTER TABLESPACE ARGROTBLSPB6只读;表空间已更改。

在每个日志文件的末尾，您会找到将表重置为READ WRITE SQL语句：

SQL> ALTER TABLESPACE ARGROTBLSPA6 READ WRITE;表空间已更改。SQL> ALTER TABLESPACE ARGROTBLSPB6 READ WRITE;表空间已更改。

Oracle数据库的文件系统选项

如果在文件系统上安装Oracle数据库文件，则Oracle Database Configuration Assistant会在计算机上安装的文件系统上的目录中创建数据库文件。

Oracle建议将文件系统与操作系统或Oracle数据库软件使用的文件系统分开。

文件系统可以是以下任何一种：

标准Oracle数据库创建选项

  • 磁盘上物理连接到系统的文件系统。

    如果要在非逻辑卷或RAID设备的基本磁盘上创建数据库，则Oracle建议您遵循Optimal Flexible Architecture（OFA）建议并将数据库文件分发到多个磁盘上。

  • 逻辑卷管理器（LVM）卷或RAID设备上的文件系统。

创建操作系统权限组

以下部分介绍如何为Oracle Grid Infrastructure和Oracle数据库创建操作系统组：

  • 为ASM Group创建OSDBA
    您必须在安装期间将组指定为ASM（ asmdba ）组的OSDBA。该组的成员被授予对Oracle自动存储管理的访问权限。
  • 为ASM Group创建OSOPER
    您可以选择在安装期间将组指定为ASM组的OSOPER（ asmoper ）。该组的成员被授予Oracle自动存储管理的启动和关闭权限。
  • 为数据库安装创建OSDBA组
    每个Oracle数据库都需要将操作系统组指定为OSDBA组。该组的成员被授予SYSDBA系统特权以管理数据库。
  • 为数据库安装创建OSOPER组
    仅当您要使用一组有限的数据库管理权限（SYSOPER操作员权限）标识一组操作系统用户时，才创建OSOPER组。
  • 为数据库安装创建OSBACKUPDBA组
    您必须在安装期间将组指定为OSBACKUPDBA组。该组的成员被授予SYSBACKUP权限，以使用RMAN或SQL * Plus执行备份和恢复操作。
  • 为数据库安装创建OSDGDBA组
    您必须在安装期间将组指定为OSDGDBA组。该组的成员被授予SYSDG权限以执行Data Guard操作。
  • 为数据库安装创建OSKMDBA组
    您必须在安装期间将组指定为OSKMDBA组。该组的成员被授予SYSKM权限以执行透明数据加密密钥库操作。
  • 为数据库安装创建OSRACDBA组
    您必须在数据库安装期间将组指定为OSRACDBA组。该组的成员被授予SYSRAC权限，以在Oracle RAC集群上执行Oracle数据库的日常管理。

在非Oracle数据库上执行用户定义的函数

您可以在远程非Oracle数据库中执行用户定义的函数。

例如：

SELECT getdeptforemp @ Remote_DB（7782）FROM dual;

在此示例中，发出了一个SELECT语句，该语句在远程数据库中执行用户定义的函数，该函数返回员工7782的部门信息。

当远程函数驻留在Oracle数据库中时，Oracle数据库会自动确保远程函数不更新任何数据库状态（例如更新数据库中的行或更新PL / SQL包状态）。当远程功能驻留在非Oracle数据库中时，网关无法验证这一点。因此，您有责任确保用户定义的函数不会更新任何数据库中的状态。确保不需要对数据库进行更新以保证读取的一致性。

作为安全措施，您必须在特定于网关的初始化参数文件中的HS_CALL_NAME参数中指定要远程执行的功能及其所有者。例如：

HS_CALL_NAME =“owner1。A1，所有者2。A2 ”
owner1和owner2是远程函数所有者名称。A1和A2是远程功能名称。您不需要在SQL语句中指定远程函数所有者。默认情况下，远程功能需要驻留在Database Gateway连接到的架构中。如果不是这种情况，则必须在SQL语句中指定远程函数的所有者。

CONCAT

         

         

            

               

                  
句法

               

               
               

                  

插图描述concat.eps

                  
               

               
               

                  
目的

               

               
               

                  
CONCAT返回与char2连接的char1 。char1和char2都可以是CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB任何数据类型。返回的字符串与char1字符集相同。其数据类型取决于参数的数据类型。
                  

                  
在两种不同数据类型的连接中，Oracle数据库返回导致无损转换的数据类型。因此，如果其中一个参数是LOB，则返回的值是LOB。如果其中一个参数是国家数据类型，则返回的值是国家数据类型。例如：

                  
    
                     
  • 
                        
    CONCAT （ CLOB ， NCLOB ）返回NCLOB
    
                     
    • 
                     
  • 
                        
    CONCAT （ NCLOB ， NCHAR ）返回NCLOB
    
                     
    • 
                     
  • 
                        
    CONCAT （ NCLOB ， CHAR ）返回NCLOB
    
                     
    • 
                     
  • 
                        
    CONCAT （ NCHAR ， CLOB ）返回NCLOB
    
                     
    • 
                  

                  
此函数等效于连接运算符（||）。

3.6管理Oracle Cluster Health Advisor存储库

         

         

            

               
Oracle Cluster Health Advisor存储库存储群集主机问题，数据库问题和关联度量标准的历史记录以及模型。

               
Oracle Cluster Health Advisor存储库用于诊断和分类周期性问题。默认情况下，存储库的大小可以保留16个目标（节点和数据库实例）的数据72小时。如果目标数量增加，则保留时间会自动减少。当保留时间低于72小时时，Oracle Cluster Health Advisor会生成警告消息，并在保留时间低于24小时时停止监视并生成严重警报。

               
使用CHACTL命令管理存储库并设置最长保留时间。

               
    
                  
  1. 
                     
    要检索存储库详细信息，请使用以下命令：
    
                     
    $ chactl查询存储库
    
                     
    例如，运行前面提到的命令会显示以下输出：
    指定的最长保留时间（小时）：72可用保留时间（小时）：212可用实体数：2个已分配的实体数：0总存储库大小（gb）：2.00已分配的存储库大小（gb）：0.07
    
                  
    2. 
                  
  2. 
                     
    要以小时为单位设置最长保留时间，请根据当前监视的目标数量，使用以下命令：
    
                     
    $ chactl set maxretention -time number_of_hours
    
                     
    例如：
    $ chactl set maxretention -time 80 max retention成功设置为80小时
    
                     
    
                        
    
                           
    注意：
    
                           
    maxretention设置限制存储库中保留的最旧数据，但如果监视目标的数量增加，则无法保证维护。在这种情况下，如果受监控目标和小时数的组合不足，则增加Oracle Cluster Health Advisor存储库的大小。
                           
    
                        
    
                     
    
                  
    3. 
                  
  3. 
                     
    要增加Oracle Cluster Health Advisor存储库的chactl resize repository ，请使用chactl resize repository命令。
                     
    
                     
    例如，要使用当前设置的最长保留时间调整存储库大小以支持32个目标，可以使用以下命令：
    
                     
    $ chactl resize repository -entities 32个存储库成功调整了32个目标的大小

改变属性维度

         

         

            

               

                  
目的

                  
使用ALTER ATTRIBUTE DIMENSION语句重命名或编译属性维度。对于其他更改，请使用CREATE OR REPLACE ATTRIBUTE DIMENSION 。

               

               
               

                  
先决条件

                  
要更改自己架构中的属性维度，您必须具有ALTER ATTRIBUTE DIMENSION系统特权。要更改另一个用户架构中的属性维度，您必须具有ALTER ANY ATTRIBUTE DIMENSION系统特权，或者已直接在属性维度上授予ALTER 。
                  

               

               
               

                  
句法

               

               
               

                  
alter_attribute_dimension :: =

                  

插图描述alter_attribute_dimension.eps

                  
               

               
               

                  
语义

               

               
               

                  
模式

                  
指定属性维存在的模式。如果未指定架构，则Oracle数据库会在您自己的架构中查找属性维度。

               

               
               

                  
attr_dim_name

                  
指定属性维的名称。

               

               
               

                  
重命名

                  
指定RENAME TO以更改属性维的名称。对于new_attr_dim_name ，请为属性维指定新名称。
                  

               

               
               

                  
COMPILE

                  
指定COMPILE以编译属性维度。
                  

               

               
               

                  
例

                  
以下语句更改属性维的名称：
ALTER ATTRIBUTE DIMENSION product_attr_dim重命名为my_product_attr_dim;

关于安装修复脚本

         

         

            

               
Oracle Universal Installer检测何时不满足安装的最低要求，并创建称为修复脚本的shell脚本，以完成不完整的系统配置步骤。

               
如果Oracle Universal Installer检测到不完整的任务，则会生成runfixup.sh脚本（ runfixup.sh ）。您可以运行修复脚本，然后单击“ 修复并再次检查” 。fixup脚本修改了内存中的持久性参数设置和参数，因此您不必重新启动系统。
               

               
Fixup脚本执行以下任务：

               
    
                  
  • 
                     
    如有必要，将内核参数设置为成功安装所需的值，包括：
    
                     
      
                        
    • 
                           
      共享内存参数。
      
                        
      • 
                        
    • 
                           
      打开文件描述符和UDP发送/接收参数。
      
                        
      • 
                     
    
                  
    • 
                  
  • 
                     
    在Oracle Inventory（中央库存）目录上创建和设置权限。
    
                  
    • 
                  
  • 
                     
    如有必要，为Oracle Inventory目录和操作系统特权组创建或重新配置安装所有者的主要和辅助组成员身份。
    
                  
    • 
               

               
    
                  
  • 
                     
    如有必要，将shell限制设置为所需的值。
    
                  
    •

Oracle数据库的文件系统选项

         

         

            

               
如果在文件系统上安装Oracle数据库文件，则Oracle Database Configuration Assistant会在计算机上安装的文件系统上的目录中创建数据库文件。

               
Oracle建议将文件系统与操作系统或Oracle数据库软件使用的文件系统分开。

               
文件系统可以是以下任何一种：

               

                  
标准Oracle数据库创建选项

                  
    
                     
  • 
                        
    磁盘上物理连接到系统的文件系统。
    
                        
    如果要在非逻辑卷或RAID设备的基本磁盘上创建数据库，则Oracle建议您遵循Optimal Flexible Architecture（OFA）建议并将数据库文件分发到多个磁盘上。
    
                     
    • 
                     
  • 
                        
    逻辑卷管理器（LVM）卷或RAID设备上的文件系统。
    
                        
    如果在LVM或RAID配置中使用多个磁盘，则Oracle建议您使用Stripe and Mirror Everything（SAME）方法来提高性能和可靠性。使用此方法，您不必为数据库存储指定多个文件系统装入点。
    
                     
    • 
                     
  • 
                        
    从经认证的网络连接存储（NAS）设备安装的网络文件系统（NFS）。您还可以选择使用Direct NFS Client，它简化了NFS配置的管理并提高了性能。
    
                        
    如果NAS设备已通过Oracle认证，则可以将数据库文件存储在其中。
    
                     
    • 
                  

               

               
               

                  
高级Oracle数据库创建选项

                  
    
                     
  • 
                        
    标准Oracle数据库安装可用的三个文件系统选项。
    
                     
    • 
                     
  • 
                        
    使用Oracle Managed Files，您可以指定数据库在数据库对象级别自动创建，命名和管理文件的文件系统目录。
    
                        
    如果使用Oracle托管文件功能，则在创建或删除数据库文件时，必须仅指定数据库对象名而不是文件名。

创建操作系统权限组

         

         

            

               
以下部分介绍如何为Oracle Grid Infrastructure和Oracle数据库创建操作系统组：

            

            

               
    
                  
  • 为ASM Group创建OSDBA
    您必须在安装期间将组指定为ASM（ asmdba ）组的OSDBA。该组的成员被授予对Oracle自动存储管理的访问权限。
                  
    • 
                  
  • 为ASM Group创建OSOPER
    您可以选择在安装期间将组指定为ASM组的OSOPER（ asmoper ）。该组的成员被授予Oracle自动存储管理的启动和关闭权限。
                  
    • 
                  
  • 为数据库安装创建OSDBA组

    • 
                  
  • 为数据库安装创建OSOPER组

确定您是否具有现有Oracle中央库存，并确保为所有Oracle软件安装使用相同的Oracle Inventory。此外，请确保您打算用于安装的所有Oracle软件用户都有权写入此目录。

#more /var/opt/oracle/oraInst.loc
inventory_loc = / u01 / app / oraInventory inst_group = oinstall

9为独立服务器配置Oracle Grid Infrastructure的存储

         

         

            

               
完成这些过程以将Oracle Grid Infrastructure用于独立服务器，其中包括Oracle自动存储管理（Oracle ASM）。

               
用于独立服务器的Oracle Grid Infrastructure（也称为Oracle Restart）为单实例Oracle数据库提供系统支持。Oracle ASM是Oracle数据库文件的卷管理器和文件系统，支持单实例Oracle数据库和Oracle Real Application Clusters（Oracle RAC）配置。Oracle自动存储管理还支持满足您的应用程序需求的通用文件系统，包括Oracle数据库二进制文件。Oracle自动存储管理是Oracle推荐的存储管理解决方案。它提供了传统卷管理器和文件系统的替代方案。

               

                  

                     
注意：

                     
    
                        
  • 
                           
    如果要使用Oracle ASM或Oracle Restart，则必须在安装和创建数据库之前为独立服务器安装Oracle Grid Infrastructure。否则，您必须使用Oracle Restart手动注册数据库。
    
                        
    • 
                        
  • 
                           
    您既不能在Oracle Grid Infrastructure集群成员节点上安装Oracle Restart，也不能将Oracle Restart服务器添加到Oracle Grid Infrastructure集群成员节点。Oracle Restart支持在一台服务器上支持单实例数据库，而Oracle Grid Infrastructure for Cluster支持群集上的单实例或Oracle RAC数据库。
    
                        
    • 
                     

                  

               

            

            

               
    
                  
  • 配置Oracle自动存储管理存储
    确定存储要求和Oracle ASM磁盘组选项。
                  
    • 
                  
  • 使用Oracle ASMFD配置存储设备路径持久性
    Oracle ASM筛选器驱动程序（Oracle ASMFD）维护存储文件路径持久性，并有助于保护文件免遭意外覆盖。
                  
    • 
                  
  • 在Oracle Solaris上为Oracle ASM配置磁盘设备
    完成这些任务以配置磁盘设备以与Oracle自动存储管理（Oracle ASM）配合使用。
                  
    • 
                  
  • 为Oracle自动存储管理创建DAS或SAN磁盘分区
    您可以将直接连接存储（DAS）和存储区域网络（SAN）磁盘与Oracle ASM一起使用。
    • 
                  
  • 创建Oracle数据库文件的目录
    您可以将Oracle数据库和恢复文件存储在与配置文件不同的文件系统中。
                  
    • 
                  
  • 在NAS设备上创建文件以与Oracle自动存储管理一起使用
    如果您拥有经过认证的NAS存储设备，则可以在NFS安装目录中创建零填充文件，并将这些文件用作Oracle ASM磁盘组中的磁盘设备。

关于快速恢复区和快速恢复区磁盘组

         

         

            

               
快速恢复区域是与恢复相关的所有Oracle数据库文件的统一存储位置。为最近的数据启用快速备份可以减少向系统管理员请求检索备份磁带以进行恢复操作的请求。

               
数据库管理员可以将DB_RECOVERY_FILE_DEST参数定义为快速恢复区域的路径，以启用磁盘备份和快速恢复数据。在init.ora文件中启用快速恢复时，Oracle数据库会将所有RMAN备份，存档日志，控制文件自动备份和数据库副本写入快速恢复区域。RMAN通过删除不再需要的过时备份和归档文件来自动管理快速恢复区中的文件。
               

               
Oracle建议您创建快速恢复区磁盘组。Oracle Clusterware文件和Oracle数据库文件可以放在同一磁盘组中，您也可以将快速恢复文件放在同一磁盘组中。但是，Oracle建议您创建单独的快速恢复磁盘组以减少存储设备争用。

               
通过设置DB_RECOVERY_FILE_DEST参数来启用快速恢复区域。快速恢复区域的大小使用DB_RECOVERY_FILE_DEST_SIZE设置。作为一般规则，快速恢复区域越大，它变得越有用。为便于使用，Oracle建议您在存储设备上创建一个可包含至少三天恢复信息的快速恢复区磁盘组。理想情况下，快速恢复区域足以容纳所有数据文件和控制文件的副本，联机重做日志以及使用保留策略下保留的数据文件备份恢复数据库所需的归档重做日志文件。
               

               
多个数据库可以使用相同的快速恢复区域。例如，假设您已在具有150 GB存储空间的磁盘上创建了快速恢复区磁盘组，由3个不同的数据库共享。您可以根据每个数据库的重要性为每个数据库设置快速恢复的大小。例如，如果database1是您最不重要的数据库，则database2更重要，而database3最重要，那么您可以为每个数据库设置不同的DB_RECOVERY_FILE_DEST_SIZE设置以满足每个数据库的保留目标：30 GB用于database1,50 GB对于database2，对于database3为70 GB。

长度

         

         

            

               

                  
句法

               

               
               

                  
length :: =

               

               
               

                  

说明length.eps的说明

                  
               

               
               

                  
目的

               

               
               

                  
LENGTH函数返回char的长度。LENGTH使用输入字符集定义的字符计算长度。LENGTHB使用字节而不是字符。LENGTHC使用Unicode完整字符。LENGTH2使用UCS2代码点。LENGTH4使用UCS4代码点。
                  

                  
char可以是CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB任何数据类型。例外是LENGTHC ， LENGTH2和LENGTH4 ，它们不允许char为CLOB或NCLOB 。返回值的数据类型为NUMBER 。如果char数据类型为CHAR ，则长度包括所有尾随空白。如果char为null，则此函数返回null。

对LENGTHB的限制

               

               
               

                  
LENGTHB函数仅支持单字节LOB。它不能与多字节字符集中的CLOB和NCLOB数据一起使用。
