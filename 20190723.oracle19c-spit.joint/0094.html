11.1.1关于拔出PDB

拔掉PDB会使PDB与CDB解除关联。 PDB仅在插入CDB时才可用。

如果要执行以下任何操作，请拔下PDB：

  • 将PDB移动到其他CDB

  • 存档PDB供以后使用

  • 使PDB无法使用

要拔出PDB，请连接到其CDB根目录或应用程序根目录，并使用ALTER PLUGGABLE DATABASE语句指定以下任一项：

  • XML文件

    XML文件（.xml扩展名）包含有关PDB拔出后的元数据。此元数据包含在目标CDB上启用CREATE PLUGGABLE DATABASE语句以插入PDB所需的信息。

  • .pdb文件

    .pdb文件包含XML文件的压缩存档，该存档描述PDB和PDB使用的文件（例如数据文件和钱包文件）。使用.pdb文件可以将单个压缩文件（而不是多个文件）复制到新位置，以将PDB插入CDB。

11.1.2拔掉PDB

使用ALTER PLUGGABLE DATABASE ...拔掉PDB ALTER PLUGGABLE DATABASE ...UNPLUG INTO声明。

先决条件

必须满足以下先决条件：

  • 当前用户必须具有SYSDBA或SYSOPER管理权限，并且必须在PDB中共同授予或本地授予该权限。用户必须在连接时使用AS SYSDBA或AS SYSOPER行使权限。

  • PDB必须至少打开一次。

注意：

如果要拔出包含使用透明数据加密加密的数据的PDB，请按照“ Oracle数据库高级安全性指南”中的说明进行操作。

拔掉PDB：

  1. 在SQL * Plus中，确保当前容器是PDB的根。

    如果PDB插入CDB根目录，则当前容器必须是CDB根目录。如果将PDB插入应用程序根目录，则当前容器必须是应用程序根目录。

    如果要拔出应用程序容器，则当前容器必须是CDB根目录，并且应用程序容器不能将任何应用程序PDB插入其中。

  2. 关闭PDB。

    在Oracle Real Application Clusters（Oracle RAC）环境中，必须在所有实例上关闭PDB。

  3. 使用UNPLUG INTO子句运行ALTER PLUGGABLE DATABASE语句，并指定要拔出的PDB以及PDB的XML元数据文件或.pdb文件的名称和位置。

示例11-1将PDB salespdb解压缩到XML元数据文件中

此ALTER PLUGGABLE DATABASE语句ALTER PLUGGABLE DATABASE PDB salespdb并在/oracle/data/目录中创建salespdb.xml元数据文件：

ALTER PLUGGABLE DATABASE salespdb UNPLUG INTO'/oracle/data/salespdb.xml';

示例11-2将PDB salespdb插入存档文件

此ALTER PLUGGABLE DATABASE语句将salespdb PDB salespdb并在/oracle/data/目录中创建sales.pdb存档文件。sales.pdb存档文件是一个压缩文件，包含XML元数据文件和PDB的文件（例如数据文件和钱包文件）。 

ALTER PLUGGABLE DATABASE salespdb UNPLUG INTO'/oracle/data/sales.pdb';

11.2删除PDB

如果要将PDB移动到新CDB或不再需要它时，请删除PDB。

删除PDB时，将修改CDB的控制文件以消除对已删除PDB的所有引用。存档的重做日志文件和与PDB关联的备份不会被删除，但您可以使用Oracle Recovery Manager（RMAN）将其删除。

删除PDB时，可以使用DROP PLUGGABLE DATABASE语句的以下子句之一来保留或删除PDB的数据文件：

  • 保留KEEP DATAFILES （默认值）保留数据文件。

    即使指定了KEEP DATAFILES也会删除PDB临时文件，因为不再需要临时文件。

    指定KEEP DATAFILES ，必须拔掉PDB。

  • INCLUDING DATAFILES会从磁盘中删除数据文件。

    如果使用SNAPSHOT COPY子句创建PDB，则必须在删除PDB时指定INCLUDING DATAFILES 。

先决条件

必须满足以下先决条件：

  • PDB必须处于安装模式，或者必须拔下插头。

    请参阅“ 修改一个或多个PDB的开放模式 ” 。

    请参阅“ 从CDB中拔出PDB ” 。

  • 当前用户必须具有SYSDBA或SYSOPER管理权限，并且必须在PDB中共同授予或本地授予该权限。用户必须在连接时使用AS SYSDBA或AS SYSOPER行使权限。

注意：

这种操作具有破坏性。

删除PDB：

  1. 在SQL * Plus中，确保当前容器是CDB根目录，或者对于应用程序PDB，确保包含应用程序PDB的应用程序根目录。

    如果PDB插入CDB根目录，则当前容器必须是CDB根目录。如果将PDB插入应用程序根目录，则当前容器必须是该应用程序根目录或CDB根目录。

    如果要删除应用程序容器，则当前容器必须是CDB根目录，并且应用程序容器不能将任何应用程序PDB插入其中。

  2. 运行DROP PLUGGABLE DATABASE语句并指定要删除的PDB。

示例11-3在保留其数据文件的同时删除PDB salespdb

DROP PLUGGABLE DATABASE salespdb KEEP DATAFILES;

示例11-4删除PDB salespdb及其数据文件

DROP PLUGGABLE DATABASE salespdb包括DATAFILES;

23.4 OCI客户端访问二进制XML

中间层和客户端层可以以二进制XML格式生成，使用和处理XML。

C应用程序从XML DB存储库获取数据，使用DOM执行XML更新，并将其存储回数据库。或者在客户端上创建或输入XML文档，并在其上使用XSLT，XQuery和其他实用程序。然后输出XML保存在XML DB中。

客户端应用程序需要与元数据存储库（通常是后端数据库）的连接（称为元数据连接），以在编码或解码二进制XML文档时获取标记定义，XML模式和DTD。

使用专用连接或连接池初始化存储库上下文。从存储库上下文获取的连接用于获取元数据，例如令牌定义和XML模式。相反，应用程序还具有数据连接，用于定期向数据库传输数据（包括XML数据）。存储库上下文与（一个或多个）数据连接显式关联。使用数据连接从数据库读取或写入XML数据时，将在基础编码或解码操作期间访问相应的存储库上下文。根据需要，元数据连接用于从存储库中获取元数据。

23.4.1从OCI应用程序访问XML数据

您的C应用程序可以使用OCI访问数据库中的持久性XML，使用Unified XML C API来处理获取的XML数据。

客户端应用程序采取以下步骤：

  1. 创建常用的OCI句柄，如OCIEnv ， OCISvcCtx和OCIError 。
  2. 创建一个或多个存储库上下文以获取二进制XML元数据。
  3. 将存储库上下文与数据连接相关联。
  4. 将（ xmldocnode ）变量绑定或定义到select，insert和update语句中。
  5. 执行select，insert或update语句以获取或存储XML文档。此时，客户端OCI库与数据库后端交互以获取所需的XML模式，DTD，令牌定义等。
  6. 使用Unified C API操作XML数据（DOM）。

23.4.2存储库上下文

OCIBinXmlReposCtx是存储库上下文数据结构。

客户端应用程序通过向元数据存储库提供连接信息来创建此上下文。应用程序可以创建多个存储库上下文以连接到多个令牌存储库。存储库上下文与数据连接（ OCISvcCtx ）显式关联。当系统必须获取元数据以将数据编码到数据连接或从数据连接解码数据时，它将访问适当的元数据。

建议应用程序为每个OCIEnv创建一个存储库上下文。这为多线程应用程序提供了更好的并发性。

可以从专用OCI连接或OCI连接池创建存储库上下文。

23.4.3从专用OCI连接创建存储库上下文

OCIBinXmlCreateReposCtxFromConn()使用指定的专用OCI连接创建存储库上下文。

OCI连接仅用于元数据访问，不应由应用程序在任何其他方案中使用。另请注意，对此连接的访问是序列化的;也就是说，如果多个线程尝试使用相同的连接，则访问一次仅限于一个线程。出于可伸缩性的原因，建议应用程序使用连接池创建存储库上下文，如下一节中所述。

注意：您还可以传入与用于数据的连接相同的连接。但是，在某些情况下，如果客户端系统尝试在另一个操作（例如select或insert）的一部分中尝试联系元数据存储库，则可能会导致错误。

以下主题介绍如何使用响应文件安装和配置Oracle产品。

介绍

通过在启动Oracle Universal Installer时指定响应文件，可以完全或部分自动完成Oracle软件的安装和配置。Oracle Universal Installer使用响应文件中包含的值来提供部分或全部Oracle Universal Installer提示的答案：

  • 如果包含响应文件中所有提示的响应，并在启动Oracle Universal Installer时指定-silent选项，则Oracle Universal Installer将以静默方式运行。在静默模式安装期间，Oracle Universal Installer不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。

  • 如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则Oracle Universal Installer将以抑制模式运行。在抑制模式安装期间，Oracle Universal Installer仅显示未指定所有必需信息的屏幕。您还可以在响应文件或命令行选项中使用变量来禁止不提示输入信息的其他安装程序屏幕，例如“欢迎”屏幕或“摘要”屏幕。

下表描述了您可能希望以静默模式或抑制模式运行Oracle Universal Installer的几个原因：

模式 用途

无声

如果您想要使用静音模式：

  • 完成无人参与安装，您可以使用操作系统实用程序（如at

  • 在多个系统上完成几个类似的安装，无需用户交互

  • 在没有安装X Window System软件的系统上安装软件

Oracle Universal Installer在您用于启动它的终端中显示进度信息，但它不显示任何Oracle Universal Installer屏幕。

抑制

如果要在多个系统上完成类似的Oracle软件安装，请使用抑制模式，为某些（但不是全部）Oracle Universal Installer提示提供默认答案。

如果未在响应文件中指定特定安装程序屏幕所需的信息，Oracle Universal Installer将显示该屏幕。它会抑制您提供了所有必需信息的屏幕。

磁盘组的存储资源

               

                  
可以使用各种存储资源来创建Oracle ASM磁盘组。

                  
您可以使用以下存储资源之一创建Oracle ASM磁盘组：

                  
    
                     
  • 
                        
    磁盘分区
    
                        
    磁盘分区可以是整个磁盘驱动器，也可以是磁盘驱动器的一部分。但是，Oracle ASM磁盘不能位于包含分区表的分区中，因为分区表将被覆盖。
                        
    
                     
    • 
                     
  • 
                        
    逻辑单元号（LUN）
    
                        
    LUN是由存储阵列呈现给计算机系统的磁盘。Oracle建议您使用硬件RAID功能来创建LUN。存储硬件RAID 0 + 1或RAID5以及其他RAID配置可作为Oracle ASM磁盘提供给Oracle ASM。
    
                     
    • 
                     
  • 
                        
    逻辑卷
    
                        
    逻辑卷 ，其中一个逻辑卷被映射到LUN在较不复杂的结构支撑，或逻辑卷使用磁盘或原始分区。Oracle不建议使用逻辑卷配置，因为它们会创建重复的功能。Oracle也不建议使用逻辑卷管理器进行镜像，因为Oracle ASM提供镜像。
                        
    
                     
    • 
                     
  • 
                        
    网络文件系统（NFS）
    
                        
    可以从NFS文件创建Oracle ASM磁盘组，包括Oracle Direct NFS（dNFS）。配置到磁盘组的NFS文件可能来自多个NFS服务器，以提供更好的负载平衡和灵活的容量规划。
    
                        
    您可以使用带有或不带有Direct NFS的NFS来存储数据文件。但是，Oracle Clusterware文件不支持NFS。要使用NFS在Windows上安装Oracle Real Application Clusters（Oracle RAC），您还必须能够访问除NFS for Oracle Clusterware文件之外的共享存储方法。
    
                        
    基于NFS的仲裁磁盘（仲裁失败组）不应使用Direct NFS（dNFS），因为dNFS不支持软安装。而是使用NFS挂载点的软安装来进行仲裁磁盘。使用软安装时，Oracle ASM会正常处理I / O故障，并仅将关联的仲裁磁盘设置为脱机。
    
                        
    使用硬安装时，如果NFS服务器不可用，Oracle ASM或数据库实例可能会挂起。请注意，无论是否使用Direct NFS以及Oracle ASM是否用于镜像，都可能发生这些挂起情况。
    
                        
    
                           
    也可以看看：
    
                           
    适用于您的操作系统的Oracle Grid Infrastructure安装和升级指南 ，以获取有关Oracle ASM的Oracle Direct NFS和存储要求的信息
    
                        
    
                     
    • 
                  

                  

                     
注意：

                     
    
                        
  • 
                           
    目前，从NFS或通用Internet文件系统（CIFS）文件创建的磁盘组不支持Oracle ASM动态卷管理器（Oracle ADVM）卷和Oracle自动存储管理群集文件系统（Oracle ACFS）文件系统。但是，在某些情况下，Oracle ACFS文件系统可能会作为NFS或CIFS文件系统导出到网络客户端。Windows上的Samba / CIFS客户端在与Oracle ACFS Linux，Solaris或AIX服务器连接时无法使用ACL。
    
                        
    • 
                        
  • 
                           
    不支持在Oracle ACFS文件上挂载回送文件系统。
    
                        
    • 
                        
  • 
                           
    Oracle Universal Installer（OUI）或Database Configuration Assistant（DBCA）不支持块或原始设备。
    
                        
    • 
                     

                  

                  
为Oracle ASM准备存储资源的过程如下：

    • 
                     
  • 
                        
    更改存储设备资源的所有权和权限。
    
                        
    例如，Linux系统上需要执行以下步骤：

      更改设备的用户和组所有权，例如grid:asmadmin
      
                              
      有关Oracle ASM权限的信息，请参阅关于 Oracle ASM的权限。
      
                           
      •

Oracle ASM和多路径

               

                  
多路径解决方案通过使用冗余物理路径组件提供故障转移。

                  
这些冗余物理路径组件包括驻留在服务器和存储子系统之间的适配器，电缆和交换机。如果这些组件中的一个或多个发生故障，则应用程序仍可访问其数据，从而消除了多端口存储阵列上的存储区域网络（SAN） ，主机总线适配器，接口电缆或主机端口的单点故障。
                  

                  
多路径是在操作系统设备驱动程序级别实现的软件技术。多路径创建一个伪设备，以便在所有可用I / O路径上共享和平衡I / O操作。多路径还可以通过在所有可用路径上分配I / O负载来提高系统性能，通过自动故障转移和故障恢复提供更高级别的数据可用性。

                  
虽然Oracle ASM没有设计多路径功能，但Oracle ASM确实使用多路径技术。多路径技术可从许多来源获得。存储供应商提供多路径产品以支持其特定存储产品，而软件供应商通常开发多路径产品以支持多个服务器平台和存储产品。

                  

                     
也可以看看：

                     
有关特定平台和存储产品的多路径选项的详细信息，请参见存储或软件供应商多路径文档

                  

                  
使用Oracle ASM，您可以通过将ASM_DISKSTRING初始化参数的值设置为与表示多路径磁盘的伪设备匹配的模式来确保发现多路径磁盘。当I / O被发送到伪设备时，多路径驱动程序拦截它并为底层子路径提供负载平衡。
                  

                  
如果Oracle ASM发现到同一磁盘设备的多个路径，则Oracle ASM会引发错误。由于单个磁盘在多路径配置中可能出现多次，因此必须将Oracle ASM配置为仅发现多路径磁盘。

存储准备建议

               

                  
本主题讨论了使用Oracle ASM进行存储准备的建议。

                  
以下是准备与Oracle ASM一起使用的存储的准则：

                  
    
                     
  • 
                        
    为以下内容配置单独的磁盘组：
    
                        
      
                           
    • 
                              
      Oracle Cluster Registry（OCR）和投票文件
      
                           
      • 
                           
    • 
                              
      网格基础结构管理存储库（GIMR）文件
      
                           
      • 
                           
    • 
                              
      数据库数据文件
      
                           
      • 
                           
    • 
                              
      快速恢复区
      
                           
      • 
                        
    
                     
    • 
                     
  • 
                        
    每个磁盘组的LUN数（Oracle ASM磁盘）应至少等于活动I / O路径数的四倍。例如，如果磁盘组具有两个活动I / O路径，则应使用最少八个LUN。每个磁盘组的LUN大小和性能应相同。
    
                        
    I / O路径是呈现LUN和服务器的存储之间的独特通道或连接。活动 I / O路径是I / O路径，其中LUN上的I / O负载通过多路径软件进行多路复用。
                        
    
                     
    • 
                     
  • 
                        
    确保磁盘组中的所有Oracle ASM磁盘具有类似的存储性能和可用性特征。在具有混合速度驱动器的存储配置中，例如闪存和硬盘驱动器（HDD），I / O性能受到最慢速驱动器的限制。
    
                     
    • 
                     
  • 
                        
    Oracle ASM数据分发策略是基于容量的。确保磁盘组中的Oracle ASM磁盘具有相同的容量以保持平衡。
    
                     
    • 
                     
  • 
                        
    为正常冗余磁盘组配置至少三个故障组，为高冗余磁盘组配置五个故障组，以维护伙伴状态表（PST）所需的副本数，以确保存储硬件故障的稳健性。
    
                     
    • 
                     
  •

    要创建，修改或删除转换，必须在DBMS_TRANSFORM上授予用户执行权限。用户还必须对用户定义的类型具有执行权限，这些类型是转换的源和目标类型。此外，用户还必须对转换函数中使用的任何PLSQL函数具有执行权限。
                              
    
                           
    • 
                           
  • 
                              
    转换无法写入数据库状态（执行DML）或提交或回滚当前事务。
    
                           
    • 
                           
  • 
                              
    转换必须是源类型为输入类型的SQL函数，返回目标类型的对象。它也可以是目标类型的SQL表达式，指的是源类型。对源类型的所有引用必须是source.user_data形式。
                              
    
                           
    • 
                           
  • 
                              
    源类型和目标类型都必须是非标量数据库类型。空转换表达式映射到空目标对象。
    
                           
    • 
                        

                        
要在enqueue和dequeue时使用转换，调用该操作的登录用户必须对转换使用的PLSQL函数具有执行权限。对于传播，队列的拥有模式必须具有这些特权。
