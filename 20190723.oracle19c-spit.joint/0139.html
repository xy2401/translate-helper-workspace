52.1 DBMS_DBFS_SFS概述

DBMS_DBFS_SFS包是实现和扩展DBMS_DBFS_CONTENT_SPI接口的包的示例实现。它提供了存储在RDBMS中的符合POSIX的文件系统。

  • 如果指定的存储表由当前用户注册，则将从DBMS_DBFS_CONTENT包和POSIX元数据表中描述的内容接口取消注册。

  • 在取消注册之后，将尝试存储表格。如果其他用户当前正在使用此商店表，则此操作可能会失败。

2.16.1队列监视进程

Oracle建议AQ_TM_PROCESSES指定AQ_TM_PROCESSES参数，让系统自动调整。

许多Oracle数据库高级队列任务在后台执行。这些包括将DELAY指定的消息转换为READY状态，消息到期，将消息移动到异常队列，溢出和恢复缓冲的消息以及类似的操作。

使用Oracle Database AQ时，不再需要设置AQ_TM_PROCESSES 。如果指定了值，则在启动Qxx进程时会考虑该值。但是，Qxx进程的数量可能与AQ_TM_PROCESSES指定的AQ_TM_PROCESSES 。

无需单独的API即可禁用或启用后台进程。这可以通过将AQ_TM_PROCESSES设置为零或非零来控制。但是，Oracle建议您不指定AQ_TM_PROCESSES参数，并让系统自动调整。

注意：

如果你想禁用队列监视器协调器，然后必须在pfile或spfile设置AQ_TM_PROCESSES = 0 。Oracle强烈建议您不要设置AQ_TM_PROCESSES = 0 。

2.16.2作业队列过程

传播和PL / SQL通知由作业队列（Jnnn）进程处理。

不再需要指定参数JOB_QUEUE_PROCESSES 。数据库调度程序自动启动传播和通知作业所需的作业队列进程。

2.16.3 AQ背景架构

Oracle数据库高级队列12c第1版（12.1）引入了一种新的AQ背景架构，采用3层设计。

  • 第1层（AQPC）：在实例启动时创建称为高级排队过程协调器的单个后台进程。它将负责创建和管理各种主进程。可以使用GV$AQ_BACKGROUND_COORDINATOR查看协调员统计信息。

  • 第2层（QM **）：会有许多名为Queue Monitors的主进程。每个人都将负责处理不同类型的工作。作业可以是类型通知（Emon池），队列监视器（处理分片队列时间管理器等），跨进程等。

    注意：

Oracle Database Vault中的安全应用程序角色是什么？

在Oracle Database Vault中，您可以创建使用Oracle Database Vault规则集启用的安全应用程序角色。

通过自定义PL / SQL过程启用常规Oracle数据库安全应用程序角色。您使用安全应用程序角色来阻止用户从应用程序外部访问数据。这会强制用户在已授予角色的应用程序权限的框架内工作。

在多租户环境中，您只能在PDB中创建安全应用程序角色，而不能在CDB根目录或应用程序根目录中创建。

为规则集上的角色建立数据库访问权限的优点是，您可以将数据库安全策略存储在一个中心位置，而不是将其存储在所有应用程序中。基于规则集的角色提供了一致且灵活的方法来实施角色提供的安全策略。这样，如果必须更新应用程序角色的安全策略，则可以在一个位置（规则集）中执行此操作。此外，无论用户如何连接到数据库，结果都是相同的，因为规则集绑定到角色。您需要做的就是创建角色，然后将其与规则集相关联。关联的规则集验证尝试启用该角色的用户。

    有关更多信息，请参阅DBMS_MACADM安全应用程序角色过程 。

  2. 使用DBMS_MACSEC_ROLES.SET_ROLE函数修改应用程序以调用该角色。

    有关更多信息，请参见SET_ROLE过程 。

  3. 然后，Oracle Database Vault会评估与安全应用程序角色关联的规则集。

    如果规则集的计算结果为true，则Oracle Database Vault将为当前会话启用该角色。如果规则集的计算结果为false，则不会启用该角色。在任何一种情况下，Oracle Database Vault都会处理与安全应用程序角色关联的规则集的关联审计和自定义事件处理程序。

添加群集节点的先决条件步骤

注意：

在将节点添加到群集之前，请确保执行Oracle Grid Infrastructure安装和升级指南（适用于Linux）中列出的预安装任务。

不要安装Oracle Clusterware。将节点添加到群集时，将从现有节点复制该软件。

完成以下步骤以准备要添加到群集的节点：

  1. 建立物理连接。

    将节点的硬件连接到群集的网络基础结构。这包括建立电气连接，配置网络互连，配置共享磁盘子系统连接等。有关此步骤的详细信息，请参阅硬件供应商文档。

  2. 安装操作系统。

    在群集中的其他节点上安装与操作系统匹配的操作系统的克隆映像。这包括安装所需的服务补丁，更新和驱动程序。有关此过程的详细信息，请参阅操作系统供应商文档。

    注意：

    Oracle建议您使用克隆的映像。但是，如果安装符合安装要求，则根据供应商文档安装操作系统。

  3. 创建Oracle用户。

    您必须在现有节点上存在的新节点上创建所有Oracle用户。例如，如果要将节点添加到具有两个节点的集群，并且这两个节点具有Oracle Grid Infrastructure主目录和Oracle主目录的不同所有者，则必须在新节点上创建这些所有者，即使您这样做不打算在新节点上安装Oracle主目录。

    注意：

    仅对Linux和UNIX系统执行此步骤。

    以root用户身份，使用与现有节点上相同的用户ID和组ID创建Oracle用户和组。

  4. 确保在节点上配置了SSH。

    注意：

    安装Oracle Clusterware 12 c时配置SSH。如果未配置SSH，请参阅“ Oracle Grid Infrastructure安装和升级指南”以获取有关配置SSH的信息。

  5. 使用Cluster Verification Utility（CVU）验证硬件和操作系统安装。

    在要添加的节点上配置硬件和操作系统后，可以运行以下命令以验证要添加的节点是否可由群集中的其他节点访问。您还可以使用此命令验证用户与本地节点中所有给定节点的等效性，所有给定节点之间的节点连接性，所有给定节点对共享存储的可访问性等等。

    1. 从现有节点上的Grid_home /bin目录中，运行CVU命令以获取参考节点的属性与属于当前集群环境的所有其他节点的详细比较。将ref_node替换为现有集群中您希望CVU与要添加的节点进行比较的节点的名称。在-n选项后指定以逗号分隔的节点列表。在以下示例中， orainventory_group是Oracle Inventory组的名称， osdba_group是OSDBA组的名称：

      $ cluvfy comp peer [-refnode ref_node ] -n node_list [-orainv orainventory_group ] [-osdba osdba_group ] [-verbose]

    2. 确保Grid Infrastructure Management Repository至少为每个节点添加了超过500 MB的空间，如下所示：

      $ oclumon manage -get repsize

      如果需要，添加额外空间，如下所示：

      $ oclumon manage -repos total_in_MB

      也可以看看：

      有关使用OCLUMON的详细信息，请参见Oracle Autonomous Health Framework用户指南

    注意：

    对于参考节点，选择要与CVU进行比较的群集节点，例如，使用-n选项指定要添加的节点。

完成本节中的过程后，即可将节点添加到群集中。

注意：

完成Oracle Clusterware安装后，请避免更改主机名，包括添加或删除域资格。必须从群集中删除具有已更改主机名的节点，并使用新名称添加回来。

使用Fleet Patching和Provisioning添加节点

如果您有Fleet Patching和Provisioning Server，则可以使用Fleet Patching和Provisioning使用一个命令将节点添加到集群，如以下示例所示：

$ rhpctl addnode gihome -client rhpclient -newnodes clientnode2：clientnode2-vip -root

前述示例将一个命名节点clientnode2与VIP clientnode2-vip对舰队修补和供应客户端命名rhpclient ，使用根证书（登录为要添加的节点）。

如果您不想使用Fleet Patching和Provisioning将节点添加到群集，则可以使用Oracle Grid Infrastructure安装程序完成该任务。

注意：

您可以使用$Oracle_home /install/response/gridSetup.rsp模板创建响应文件，以使用Oracle Grid Infrastructure安装程序为非交互（静默模式）操作添加节点。

使用Oracle Grid Infrastructure安装程序将节点添加到集群

  1. 运行./gridSetup.sh以启动安装程序。

  2. 在“ 选择配置选项”页面上，选择“ 向群集添加更多节点” 。

  3. 在“ 群集节点信息”页上，单击“ 添加...”以提供要添加的节点的信息。

  4. 验证过程在“ 执行先决条件检查”页面上完成后，请检查摘要，然后单击“ 安装” 。

  5. 如果出现提示，则以root身份运行orainstRoot.sh脚本，以使用中央库存的位置填充/etc/oraInst.loc文件。例如：

    #/opt/oracle/oraInventory/orainstRoot.sh

  6. 如果在群集上配置了Oracle RAC或Oracle RAC One Node数据库并且您具有本地Oracle主目录，请执行以下操作将Oracle数据库主目录扩展到node3 ：

    1. 导航到node1上的Oracle_home /addnode目录，并使用以下语法以安装Oracle RAC的用户身份运行addnode.sh脚本：

      $ ./addnode.sh“CLUSTER_NEW_NODES = {node3}”

    2. 以root /root.sh在node3上运行Oracle_home /root.sh脚本，其中Oracle_home是Oracle RAC主目录。

    如果您具有使用Oracle自动存储管理群集文件系统（Oracle ACFS）共享的共享Oracle主目录，请执行以下操作以将Oracle数据库主目录扩展到node3 ：

    1. 以root /root.sh在node3上运行Grid_home /root.sh脚本，其中Grid_home是Oracle Grid Infrastructure主目录。

    2. 从要添加的节点上的Oracle_home /oui/bin目录中安装Oracle RAC的用户运行以下命令，以添加Oracle RAC数据库主目录：

      $ ./runInstaller -attachHome ORACLE_HOME =“ORACLE_HOME”“CLUSTER_NODES = {node3}”LOCAL_NODE =“node3”ORACLE_HOME_NAME =“ home_name ”-cfs

    3. 导航到node1上的Oracle_home /addnode目录，并使用以下语法以安装Oracle RAC的用户身份运行addnode.sh脚本：

      $ ./addnode.sh -noCopy“CLUSTER_NEW_NODES = {node3}”

      注意：

      使用-noCopy选项，因为目标节点上的Oracle主目录已经完全填充了软件。

    如果在不是 Oracle ACFS的共享文件系统上具有共享Oracle主目录，则必须首先在目标节点上为Oracle RAC数据库主目录创建装入点，装载并附加Oracle RAC数据库主目录，然后更新Oracle库存，如下：

    1. 在群集中的现有节点上运行srvctl config database -db db_name命令以获取装入点信息。

    2. 在node3以root身份运行以下命令以创建挂载点：

      #mkdir -p mount_point_path

    3. 装载承载Oracle RAC数据库主目录的文件系统。

    4. 从要添加的节点上的Oracle_home /oui/bin目录中安装Oracle RAC的用户运行以下命令，以添加Oracle RAC数据库主目录：

      $ ./runInstaller -attachHome ORACLE_HOME =“ORACLE_HOME”“CLUSTER _NODES = { local_node_name }”LOCAL_NODE =“ node_name ”ORACLE_HOME_NAME =“ home_name ”-cfs

      导航到node1上的Oracle_home /addnode目录，并使用以下语法以安装Oracle RAC的用户身份运行addnode.sh脚本：

      $ ./addnode.sh -noCopy“CLUSTER_NEW_NODES = {node3}”

      注意：

      使用-noCopy选项，因为目标节点上的Oracle主目录已经完全填充了软件。

    注意：

    运行addnode.sh ，确保Grid_home /network/admin/samples目录的权限设置为750。

  7. 运行Grid_home /root.sh上的脚本node3为root和运行脚本以后，按指示。

    注意：

    • 如果在步骤6中运行了root.sh脚本，则无需再次运行它。

    • 如果您有策略管理的数据库，则必须确保在运行root.sh脚本之前将Oracle主目录克隆到新节点。

    • 如果在要添加到集群的节点上配置了任何管理员管理的数据库实例，则必须在运行root.sh脚本之前将Oracle主目录扩展到新节点。

      或者，使用srvctl remove instance命令删除管理员管理的数据库实例。

  8. 通过从Grid_home /bin目录以root身份运行以下命令，在新节点上启动Oracle ACFS资源：

    #srvctl start filesystem -device volume_device_name -node node3

    注意：

    确保Oracle ACFS资源（包括Oracle主服务器所在的Oracle ACFS注册表资源和Oracle ACFS文件系统资源）在新添加的节点上处于联机状态。

  9. 以安装Oracle Clusterware的用户身份运行以下CVU命令以检查群集完整性。此命令验证是否已成功将任意数量的指定节点添加到网络，共享存储和集群件级别的集群中：

    $ cluvfy stage -post nodeadd -n node3 [-verbose]

在Linux和UNIX系统上删除群集节点

从Linux和UNIX系统上的群集中删除节点。

注意：

  • 在从群集中删除节点之前，可以从节点中删除Oracle RAC数据库实例，但不需要执行此步骤。如果不删除实例，则仍配置实例但从不运行。从群集中删除节点不会从群集中删除节点的配置信息。剩余配置信息不会干扰群集的操作。

    另请参阅：“ Oracle Real Application Clusters管理和部署指南” ，以获取有关删除Oracle RAC数据库实例的更多信息

  • 如果删除GNS服务的群集的最后一个节点，则必须从GNS中删除该群集的条目。

  • 如果群集中的节点已取消固定，则Oracle Clusterware会在一段时间后忽略这些节点，并且您无需删除它们。

  • 如果为节点创建特定于节点的配置（例如，禁用特定节点上的服务，或将节点添加到服务器池的候选列表），则从群集中删除节点时，不会删除特定于节点的配置。必须手动删除此类节点特定的配置。

  • 在对群集进行任何更改后，将在OCR中自动备份表决文件。

  • 如果要从Oracle Flex群集中删除非Hub节点，则只需完成此过程的步骤1到步骤4。

要从群集中删除节点：

  1. 确保Grid_home在每个节点上正确指定Oracle Clusterware主目录的完整目录路径，其中Grid_home是已安装的Oracle Clusterware软件的位置。
  2. 以root用户身份或以安装Oracle Clusterware的用户身份运行以下命令，以确定要删除的节点是否处于活动状态以及是否已固定：
    $ olsnodes -s -t

    如果节点已固定，则运行crsctl unpin css命令。否则，请继续执行下一步。

  3. 在要删除的节点上，根据您是否具有共享或本地Oracle主目录，请以安装Oracle Clusterware的用户身份完成以下过程之一：

    • 对于本地主目录，请通过运行以下命令从要删除的节点卸载Oracle Clusterware主目录，其中Grid_home是为Oracle Clusterware主目录定义的路径：

      $ Grid_home / deinstall / deinstall -local

      警告：

      • 如果未指定-local标志，那么该命令将从群集中的每个节点中删除Oracle Grid Infrastructure主目录。

      • 如果剪切并粘贴上述命令，则将其粘贴到文本编辑器中，然后将其粘贴到命令行以删除此文档可能包含的任何格式。

      注意：

      或者，在配置Oracle Grid Infrastructure之后，如果要从网格基础结构中删除任何节点，则可以通过运行Grid_home /gridSetup.sh ， 从群集中选择“ 删除节点”并按照提示进行删除 。

    • 如果您有共享主目录，请在要删除的节点上按以下顺序运行以下命令。

      运行以下命令以取消配置Oracle Clusterware：

      $ Grid_home /crs/install/rootcrs.sh -deconfig -force

      从Grid_home /oui/bin目录运行以下命令以分离Grid home：

      $ ./runInstaller -detachHome ORACLE_HOME = Grid_home -silent -local

      按照安装实用程序的提示手动删除任何配置文件。

  4. 从您未删除的任何节点，以root身份从Grid_home /bin目录运行以下命令，以从群集中删除该节点：
    node_to_be_deleted delete node -n node_to_be_deleted
  5. 运行以下CVU命令以验证是否已从群集中成功删除指定的节点：
    $ cluvfy stage -post nodedel -n node_list [-verbose]
  6. 如果删除Oracle Clusterware关闭的群集节点，请确定已删除节点的VIP是否仍然存在，如下所示：
    $ srvctl config vip -node deleted_node_name

    如果VIP仍然存在，则删除它，如下所示：

    $ srvctl stop vip -node deleted_node_name $ srvctl remove vip -vip deleted_vip_name

使用Fleet Patching和Provisioning删除节点

前面的例子中删除一个指定节点clientnode2从舰队修补和供应客户端命名rhpclient ，使用根证书（登录为要删除的节点）。

部署Oracle RAC以实现高可用性

关于设计高可用性系统

对于关键任务系统，您必须能够执行故障转移和恢复，并且您的环境必须能够适应所有类型的故障。

对于关键任务系统，您必须能够执行故障转移和恢复，并且您的环境必须能够适应所有类型的故障。要实现这些目标，请首先定义业务的服务级别要求。这些要求应包括对数据中心内故障（如节点故障）或灾难恢复（如果整个数据中心发生故障）的最大事务响应时间和恢复期望的定义。通常，服务级别目标是工作的目标响应时间，无论故障如何。确定每个冗余组件的恢复时间。即使您可能具有以主动/主动模式运行的硬件组件，也不要假设如果一个组件发生故障，则其他硬件组件可以在修复故障组件时保持运行。此外，当组件以主动/被动模式运行时，请执行常规测试以验证故障转移时间。例如，存储通道的恢复时间可能需要几分钟。确保中断时间在您的业务服务级别协议中，如果不在，请与硬件供应商一起调整配置和设置。

在部署关键任务系统时，测试应包括功能测试，破坏性测试和性能测试。破坏性测试包括在系统中注入各种故障以测试恢复并确保其满足服务级别要求。破坏性测试还允许为生产系统创建操作程序。

为帮助您设计和实施关键任务或高可用性系统，Oracle为每个组织提供了一系列解决方案，无论规模大小。小型工作组和全球企业都能够扩展其关键业务应用程序的范围。借助Oracle和Internet，应用程序及其数据现在可以随时随地可访问。Oracle最高可用性体系结构（MAA）是Oracle最佳实践蓝图，它基于成熟的Oracle高可用性技术和建议。MAA的目标是消除设计最佳高可用性架构的复杂性。

应用程序可以利用许多Oracle数据库，Oracle Clusterware和Oracle RAC特性和功能来最小化或屏蔽Oracle RAC环境中的任何故障。例如，您可以：

  • 使用VIP地址连接到数据库，删除TCP / IP超时等待。

  • 创建详细的操作过程并确保您具有适当的支持合同，以匹配基础架构中所有组件的已定义服务级别。

  • 利用Oracle RAC自动工作负载管理功能，例如连接时故障转移，快速连接故障转移，快速应用程序通知和负载平衡咨询。

  • 将表决磁盘放在不同的卷组上，以减少因I / O吞吐量较慢而导致的中断。为了幸免于x投票设备的故障，请配置2 x + 1个镜像。

  • 使用Oracle数据库服务质量管理（Oracle数据库QoS管理）来监视系统并检测性能瓶颈。

  • 将OCR放置在I / O服务时间的2毫秒（ms）或更短的时间内。

  • 使用FAST_START_MTTR_TARGET初始化参数调整数据库恢复。

  • 使用Oracle自动存储管理（Oracle ASM）管理数据库存储。

  • 确保实施强有力的变更控制程序。

  • 检查周围的基础架构以获得高可用性和弹性，例如LDAP，NIS和DNS。这些实体会影响Oracle RAC数据库的可用性。如果可能，请定期执行本地备份过程。

  • 使用Oracle Enterprise Manager管理整个Oracle RAC环境，而不仅仅是Oracle RAC数据库。使用Oracle Enterprise Manager创建和修改服务，以及启动和停止集群数据库实例和集群数据库。

  • 使用Recovery Manager（RMAN）备份，还原和恢复数据文件，控制文件，服务器参数文件（SPFILE）和归档重做日志文件。您可以将RMAN与媒体管理器一起使用，以将文件备份到外部存储。您还可以在备份或恢复Oracle RAC数据库时配置并行性。在Oracle RAC中，可以跨所有Oracle RAC实例动态分配RMAN通道。通道故障转移使一个节点上的失败操作能够在另一个节点上继续。您可以从Oracle Enterprise Manager备份管理器或命令行启动RMAN。

  • 如果使用序列号，则始终使用带有NOORDER选项的CACHE ，以便在序列号生成中获得最佳性能。但是，使用CACHE选项，序列号中可能存在间隙。如果您的环境无法容忍序列号间隙，请使用NOCACHE选项或考虑预生成序列号。如果您的应用程序需要序列号排序但可以容忍间隙，那么使用CACHE和ORDER在Oracle RAC中缓存和排序序列号。如果您的应用程序需要有序序列号而没有间隙，则使用NOCACHE和ORDER 。与其他缓存和排序组合相比， NOCACHE和ORDER组合对性能的负面影响最大。

    注意：

    如果您的环境无法容忍序列号间隙，请考虑预先生成序列号或使用ORDER和CACHE选项。

    从Oracle Database 18c开始，您可以使用可伸缩序列来提供更好的数据加载可伸缩性，而不是配置非常大的序列缓存。可伸缩序列可提高并发数据加载操作的性能，尤其是在序列值用于填充表的主键列时。

  • 如果使用索引，则考虑替代方法，例如反向键索引，以优化索引性能。如果频繁插入索引的一端（例如基于插入日期的索引），则反向键索引特别有用。

在合并期间管理全局缓存服务进程

Oracle建议实时的数量服务器上的全局缓存服务进程（LMS n ）小于或等于处理器数。（请注意，这是包含核心的已识别CPU的数量。例如，双核CPU被认为是两个CPU。）在节点上添加实例时加载测试系统非常重要，以确保您有足够的容量来支持工作负载。

如果要将许多小型数据库合并到群集中，则可能需要减少Oracle RAC实例创建的LMS n的数量。默认情况下，Oracle数据库根据在服务器上找到的CPU数量计算进程数。此计算可能导致更多LMS n进程，而不是Oracle RAC实例所需的进程。一个LMS进程可能足以支持最多4个CPU。至减少LMS n进程的数量，将GC_SERVER_PROCESSES初始化参数设置为GC_SERVER_PROCESSES 1。为应用程序所需的每四个CPU添加一个进程。通常，最好只有很少繁忙的LMS n进程。Oracle数据库计算实例启动时的进程数，您必须重新启动实例才能更改该值。

使用数据库云进行整合

数据库云是由全球数据服务框架集成到单个虚拟服务器中的一组数据库，提供一个或多个全局服务，同时确保高性能，可用性和资源的最佳利用。

全局数据服务以最小的管理开销管理这些虚拟化资源，并允许数据库云快速扩展以处理其他客户端请求。构成云的数据库可以是全局分布的，客户端可以通过简单地指定服务名称来连接到数据库云，而无需了解有关云的组件和拓扑的任何信息。

数据库云可以由多个数据库池组成。数据库池是数据库云中的一组数据库，它们提供一组唯一的全局服务并属于某个管理域。通过允许每个池由不同的管理员进行管理，将云数据库划分为多个池简化了服务管理并提供了更高的安全性。数据库云可以跨越多个地理区域。区域是一个逻辑边界，包含被认为彼此接近的数据库客户端和服务器。通常，区域对应于数据中心，但是如果它们之间的网络延迟满足访问这些数据中心的应用程序的服务级别协议，则多个数据中心可以位于同一区域中。

全局服务使您能够将本地和全球分布式，松散耦合的异构数据库集成到可扩展且高度可用的私有数据库云中。该数据库云可以由全球客户共享。使用私有数据库云可以最佳地利用可用资源并简化数据库服务的配置。

Oracle RAC的可伸缩性

Oracle RAC提供对来自多个系统的单个数据副本的并发，事务一致性访问。它提供超出单个服务器容量的可扩展性。如果您的应用程序在对称多处理（SMP）服务器上透明地扩展，那么期望应用程序在Oracle RAC上很好地扩展是可行的，而无需更改应用程序代码。

传统上，当数据库服务器的容量不足时，它将被替换为新的更大的服务器。随着服务器容量的增长，它们变得更加昂贵。但是，对于Oracle RAC数据库，您可以选择增加容量：

  • 您可以迁移传统上在大型SMP服务器上运行的应用程序，以在小型服务器群集上运行。

  • 您可以维护当前硬件的投资，并向群集添加新服务器（或创建或添加新群集）以增加容量。

使用Oracle Clusterware和Oracle RAC将服务器添加到群集不需要中断。一旦启动新实例，应用程序就可以利用额外的容量。

群集中的所有服务器必须运行相同的操作系统和相同版本的Oracle数据库，但服务器不必具有相同的容量。使用Oracle RAC，您可以构建满足您需求的集群，无论集群是由每个服务器是双CPU商用服务器的服务器组成，还是服务器在每个服务器中具有32或64个CPU的集群。Oracle并行执行功能允许将单个SQL语句划分为多个进程，其中每个进程完成一部分工作。在Oracle RAC环境中，您可以将并行进程定义为仅在用户连接的实例上运行，或者在群集中的多个实例上运行。

透明数据加密和密钥库

Oracle数据库使Oracle RAC节点能够共享密钥库 （钱包）。这消除了在所有节点上手动复制和同步密钥库的需要。Oracle建议您在共享文件系统上创建密钥库。这允许所有实例访问同一共享密钥库。

Oracle RAC通过以下方式使用密钥库：

  1. 在任何一个Oracle RAC实例上执行的任何密钥库操作（如打开或关闭密钥库）都适用于所有其他Oracle RAC实例。这意味着当您打开和关闭一个实例的密钥库时，它会打开并关闭所有Oracle RAC实例的密钥库。

  2. 使用共享文件系统时，请确保所有Oracle RAC实例的ENCRYPTION_WALLET_LOCATION参数指向同一共享密钥库位置。安全性管理员还必须通过分配适当的目录权限来确保共享密钥库的安全性。

    注意：

    如果Oracle自动存储管理群集文件系统（Oracle ACFS）可用于您的操作系统，则Oracle建议您将密钥库存储在Oracle ACFS中。如果Oracle ASM中没有Oracle ACFS，则使用Oracle ASM Configuration Assistant（ASMCA）创建它。您必须将挂载点添加到每个实例中的sqlnet.ora文件中，如下所示：

    ENCRYPTION_WALLET_LOCATION =（SOURCE =（METHOD = FILE）（METHOD_DATA =（DIRECTORY = / opt / oracle / acfsmounts / data_keystore）））

    实例启动时会自动挂载此文件系统。打开和关闭密钥库，以及设置或重新密钥和旋转TDE主加密密钥的命令在所有节点之间同步。

  3. 在一个实例上执行的主密钥重新生成适用于所有实例。当出现新的Oracle RAC节点时，它会知道当前的密钥库打开或关闭状态。

  4. 设置或更改主密钥时，请勿发出任何密钥库ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN或CLOSE SQL语句。

密钥库不存在共享存储的部署要求每个Oracle RAC节点都维护本地密钥库。在单个节点上创建和置备密钥库后，必须复制密钥库并使其可用于所有其他节点，如下所示：

  • 对于使用透明数据加密和加密密钥库的系统，您可以使用任何标准文件传输协议，但Oracle建议使用安全文件传输。

  • 对于使用带有自动登录密钥库的透明数据加密的系统，建议通过安全通道进行文件传输。

要指定密钥库必须驻留的目录，请在sqlnet.ora文件中设置或ENCRYPTION_WALLET_LOCATION参数。在透明数据加密使用期间，密钥库的本地副本无需同步，直到通过ADMINISTER KEY MANAGEMENT SET KEY SQL语句重新键入服务器密钥。每次在数据库实例上发出ADMINISTER KEY MANAGEMENT SET KEY语句时，都必须再次复制驻留在该节点上的密钥库，并使其可供所有其他节点使用。然后，您必须关闭并重新打开每个节点上的密钥库。为避免不必要的管理开销，请在您认为服务器主密钥可能已被泄露且未重新键入的特殊情况下保留重新键入可能会导致严重的安全问题。

Windows防火墙注意事项

默认情况下，Windows Server 2003 Service Pack 1及更高版本的所有安装都使Windows防火墙几乎可以阻止所有TCP网络端口到传入连接。因此，任何侦听TCP端口上的传入连接的Oracle产品都不会收到任何这些连接请求，并且建立这些连接的客户端将报告错误。

使用钱包安全地运行ONS客户端

您可以配置和使用SSL证书，以在数据库层中的ONS服务器和中间层中的通知客户端之间设置身份验证。

JDBC或Oracle通用连接池以及其他Oracle RAC功能（如快速连接故障转移）可订阅来自Oracle RAC节点上运行的Oracle Notification Service（ONS）的通知。这些连接通常不经过身份验证。
  1. 使用orapki界面创建Oracle Wallet以存储SSL证书：
    1. 创建一个存储钱包的目录（如果它不存在）。
      $ cd Grid_home / opmn / conf $ mkdir sslwallet
    2. 使用auto_login选项创建钱包。
      $ orapki wallet create -wallet sslwallet -auto_login
    3. 将自签名证书请求添加到钱包。
      $ orapki wallet add -wallet sslwallet -dn“CN = ons_test，C = US”-keysize 1024 -self_signed -validity 9999
    4. 通过将证书导出到文件来签署证书。
      $ orapki wallet export -wallet sslwallet -dn“CN = ons_test，C = US”-cert sslwallet / cert.txt
    5. 将您在步骤c中创建的钱包复制到同一位置的所有其他群集节点。
  2. 配置ONS以使用在步骤1中创建的钱包，如下所示：
    $ srvctl modify nodeapps -clientdata sslwallet / cwallet.sso
  3. 如果您在中间层上运行客户端ONS守护程序，则有两种可能的配置：
    • ONS从OPMN（如OracleAS 10.1.3.x）开始，它使用opmn.xml进行配置。
    • ONS独立启动（与使用ONSCTL时一样），它使用ons.config进行配置。

    有关第一个配置，请参阅Oracle Application Server版本的OPMN管理员指南。这涉及修改opmn.xml文件以指定钱包位置。

    对于第二种配置，客户端ONS守护程序可以在不同的服务器上运行。将步骤1中创建的钱包复制到这些客户端服务器，并在ons.config文件或opmn.xml文件中指定该客户端服务器上的路径。

  4. 如果您在没有客户端ONS守护程序的情况下运行远程ONS配置，则将步骤1中创建的钱包复制到这些客户端服务器，并在ons.config文件或ons.config文件中指定该客户端服务器上的路径opmn.xml文件。
    或者，您可以将以下字符串指定为setONSConfiguration参数：
    propertiesfile = location_of_a_Java_properties_file
    Java属性文件必须包含下面列出的一个或多个ONS Java属性，但必须至少包含oracle.ons.nodes属性：
    • oracle.ons.nodes
    • oracle.ons.walletfile
    • oracle.ons.walletpassword

Oracle Text索引的基本类型;文本列的索引。当源文本包含许多大型连贯文档时， CONTEXT索引非常有用。使用CONTEXT索引的应用程序使用CONTAINS查询运算符来检索文本。

文本列和一个或多个其他列的组合索引。通常用于索引小文档或文本片段，例如通常在目录中找到的项目名称，价格和描述。CTXCAT索引通常具有比CONTEXT索引更好的混合查询性能。

存储文本的方法。通过指定特定类型的存储偏好来确定该方法。例如， DIRECT_DATASTORE类型将数据直接存储到文本列中，而URL_DATASTORE指定数据存储在URL指定的位置外部。

在文档级别工作的服务，例如在查询操作期间突出显示文档中的查询术语，标记文档或生成文档片段。CTX_DOC PL / SQL包提供了请求文档服务的过程和函数。另见： 知识库 。

现场部分

与区域部分类似，主要区别在于您可以将字段部分的开始和结束标记之间的内容与文档的其余部分分开索引。这种单独的索引使字段部分内容能够从普通查询中“隐藏”。（可以使用INPATH和WITHIN运算符在这样的部分中查找该术语。）当节在文档中出现一次（例如新闻标题中的字段）时，字段部分很有用。使用CTX_DDL.ADD_FIELD_SECTION过程或ALTER INDEX语句将节字段添加到节组。另请参见： INPATH运算符 ， 节 ， WITHIN运算符 ， 区域节 。

上下文中的关键词（KWIC）

元数据

有关不属于文档常规内容的文档的信息。例如，如果HTML文档包含<author>Smith</author> ，则将author视为元数据类型，并将Smith视为author的值。

使用CTX_DDL.ADD_MDATA_SECTION过程将包含元数据的节（称为MDATA节）添加到文档中。元数据可以加速混合查询。可以使用MDATA运算符进行此类查询。另请参阅： 混合查询 ， 部分 。

混合查询

搜索两种不同类型信息的查询;例如，文本内容和文档类型。例如，在<title>元数据中搜索Jones是一个混合查询。

名称搜索

一种解决方案，用于匹配由于正交变化而在拼写方面可能不同的专有名称。它还使您能够搜索有些不准确的数据，例如当记录的名字和姓氏未正确分段时可能会出现这种情况。也称为名称匹配。

NEWS_SECTION_GROUP

用于定义RFC 1036定义的新闻组格式文档中的节的节组类型。另请参见： section ， section group 。

标准化的词

根据有效的转换规则，将其转换为索引之后的单词形式。根据有效的规则，单词的规范化形式可能与源文档中的表单相同。单词的规范化形式还可以包括原始版本和转换版本。例如，如果指定“新德语拼写”，则“ 潜在 ”一词将标准化为“ Potenzial”和“ Potential”。

NULL_SECTION_GROUP

未定义任何部分或仅定义SENTENCE或PARAGRAPH部分时的默认部分组类型。另请参阅： 部分 ， 部分组 ， 特殊部分 。

PATH_SECTION_GROUP

用于索引XML文档的节组类型。它类似于AUTO_SECTION_GROUP类型，只是它允许使用HASPATH和INPATH运算符。另请参见： AUTO_SECTION_GROUP ， HASPATH运算符 ， INPATH运算符 ， 节 ， 节组 。

偏爱

一个可选参数，它影响Oracle Text创建索引的方式。例如，词法分析器首选项指定处理文档时使用的词法分析器，例如JAPANESE_VGRAM_LEXER.存储，过滤，词法分析器，分类器，单词列表，部分类型等都有首选项。偏好可以与属性相关，也可以不与属性相关联。 使用CTX_DDL.CREATE_PREFERENCE过程设置首选项。另见： 属性 。

printjoin

一个或多个非字母数字字符，无论它们出现在单词的开头，中间还是结尾，都是按字母数字处理的，并且包含在Oracle Text索引中的标记中。还包括连续的printjoins。

例如，如果你定义的连字符（ - ）和下划线（_）字符作为printjoins，诸如伪知识分子和_file_存储在Oracle Text的指数作为pseudo-intellectual和_file_ 。

Printjoins与endjoins和startjoins的区别与那个位置无关。例如，如果将$定义为startjoin或printjoin，则将$ 35索引为一个标记，如果将其定义为endjoin，则将其作为两个标记。另请参见： endjoin ， printjoin ， startjoin 。

结果集

通过共享开销来提高性能的接口。它使您能够一次性生成一页搜索结果所需的不同元素（例如前几个文档的元数据，总命中数和每个单词命中数）。您还可以返回难以在SQL中表达的数据视图。

在早期版本的Oracle Text中生成这些结果需要多次查询和调用。每次额外调用都需要时间来重新分析查询并查找索引元数据。此外，一些搜索操作，例如迭代查询细化对于SQL来说很难。

基于规则的分类

见： 分类 。

结构化/排序数据（SDATA）部分

支持相等和范围搜索的节类型。默认情况下，所有FILTER BY和ORDER BY列都映射为SDATA节。SDATA部分包含用户定义的索引元数据。使用这种类型的部分可以加速混合的CONTAINS查询。另请参阅： 混合查询 ， 部分 。

部分

文件的细分;例如，HTML页面的<a>...</a>部分中的所有内容。各种部分类型包括属性，字段，HTML，MDATA，特殊，停止，XML和区域部分。

通过将文档分成几个部分然后在部分内搜索，您可以将文本查询缩小到文档中的文本块。当文档具有内部结构（例如HTML和XML文档）时，节搜索很有用。您还可以在句子和段落级别搜索文本。

使用HASPATH, ISPATH,或WITHIN运算符执行节搜索。建立索引时，使用节组启用节搜索。另请参阅： 部分组 。

部分组

标识文档集类型的组，隐式指示用于索引的标记结构。例如，要索引HTML标记的文档，请使用HTML_SECTION_GROUP部分组类型。同样，要索引XML标记的文档， XML_SECTION_GROUP部分组类型。使用CTX_DDL.CREATE_SECTION_GROUP过程或ALTER INDEX语句声明节组。另见： 部分 。

skipjoin

非字母数字字符，当它出现在单词中时，将该单词标识为单个标记;但是，该字符不会与Oracle Text索引中的令牌一起存储。例如，如果你定义了连字符（ - ）作为skipjoin，那么这个词pseudointellectual存储在Oracle Text的索引pseudointellectual。另请参见： endjoin ， printjoin ， startjoin 。

startjoin

一个或多个非字母数字字符，当作为令牌中的第一个字符遇到时，明确标识令牌的开头。字符以及紧随其后的任何其他startjoins字符都包含在令牌的Oracle Text索引条目中。例如，如果将' $ '定义为startjoin，则将$ 35索引为单个标记。此外， startjoins字符串中的第一个startjoins字符隐式结束前一个标记。另请参见： endjoin ， printjoin ， skipjoin 。

词干

查询术语的扩展包括具有相同根词的所有术语。例如，词干动词说话产量交谈 ， 会谈 ， 谈话 ，以及通话 （但不对讲机 ）。词干与通配符扩展不同，其中结果仅通过拼写而非通过形态学相关。另请参见： 通配符扩展 。

特节

不受标签限制的文档部分。相反，部分由明文文档结构（例如句子和段落）形成。使用CTX_DDL.ADD_SPECIAL_SECTION过程将特殊部分添加到节组中。另请参见： section ， section group 。

停止部分

添加到AUTO_SECTION_GROUP,在索引期间导致该类型的文档部分的信息被忽略的部分;但是，仍然可以搜索部分内容。使用CTX_DDL.ADD_STOP_SECTION过程将停止节添加到节组。另请参见： AUTO_SECTION_GROUP ， section ， section group 。

stopclass

一类令牌，例如NUMBER ，在索引期间要跳过。要指定停止类，请使用CTX_DDL.ADD_STOPCLASS.将它们添加到停止列表 CTX_DDL.ADD_STOPCLASS.另见： stoplist 。

终止列表

要索引的话，被称为停止词 ，主题（stopthemes），和不属于数据类（stopclasses）的列表。默认情况下，系统使用系统提供的与给定数据库语言对应的停止列表来索引文本。

Oracle Text为大多数常见语言提供默认的停止列表，包括英语，法语，德语，西班牙语，中文，荷兰语和丹麦语。这些默认的停止列表仅包含停用词。使用CTX_DDL.CREATE_STOPLIST过程或ALTER INDEX语句创建停止列表。另请参见： stopclass ， stoptheme ， 停用词 。

stoptheme

索引期间要跳过的主题。通过使用CTX_DDL.ADD_STOPTHEMES过程将停止主题添加到停止列表来指定停止 CTX_DDL.ADD_STOPTHEMES 。另见： stoplist 。

停用词

索引期间要跳过的单词。通过使用CTX_DDL.ADD_STOPWORD过程将停用词添加到停止列表来指定停用CTX_DDL.ADD_STOPWORD 。您还可以使用ALTER INDEX语句将它们动态添加到ALTER INDEX中。另见： stoplist 。

分词法

见： lexer 。

监督分类

见： 分类 。

主题

与给定文档关联的主题。文档可能有很多主题。主题不必出现在文档中;例如，包含旧金山字样的文档可能将加利福尼亚作为其主题之一。

使用BASIC_LEXER首选项的INDEX_THEMES属性向索引添加主题组件;使用CTX_DOC.THEMES过程从文档中提取它们，并使用ABOUT运算符查询它们。

无监督分类

也称为聚类 。见： 分类 。

通配符扩展

扩展查询术语以返回适合给定模式的单词。例如，查询项％rot％的扩展会返回trot和rotten 。通配符扩展与词干不同。另见： 词干 。

空白

被视为标记之间空格的字符。空白的预定义默认值是“space”和“tab”。BASIC_LEXER使用空格字符（与标点符号和换行符一起使用）来标识用作句子和段落搜索的句子分隔符的字符串。

WITHIN运算符

CONTAINS查询运算符，用于在给定的XML文档部分中搜索查询术语。它类似于INPATH运算符，但不太通用。另请参见： INPATH运算符 。

词汇表

Oracle Text首选项，支持模糊，词干和前缀索引等功能，以实现更好的通配符搜索，以及子字符串和前缀索引。wordlist首选项通过CONTAINS和CATSEARCH.提高了通配符查询的性能CATSEARCH.使用CTX_DDL.ADD_WORDLIST过程或ALTER INDEX语句创建单词表。另见： 偏好 。

XML部分

由XML标记定义的部分，支持XML部分搜索。使用XML部分建立索引可以自动切片并创建文档类型敏感部分。XML部分搜索包括使用INPATH, HASPATH,和WITHIN运算符进行属性搜索和路径部分搜索。另见： 部分 。

XML_SECTION_GROUP

用于标识XML文档以进行索引的节组。另请参见： section ， section group 。

区域部分

文件部分的基本类型;由文档中的开始和结束标记分隔的文本正文。区域部分非常适合定义HTML和XML文档中的部分。使用CTX_DDL.ADD_ZONE_SECTION过程或ALTER INDEX语句将区段添加到节组。另请参见： 字段部分 ， 部分 ， 部分组 。
