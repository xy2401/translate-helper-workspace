6个 JSON数据的字符集和字符编码

JSON数据始终使用Unicode字符集。在这方面，JSON数据比XML数据更易于使用。这是JSON数据交换格式（RFC 4627）的重要组成部分。对于Oracle数据库处理的JSON数据，将自动执行任何所需的字符集转换。

Oracle数据库在处理JSON数据（解析，查询）时在内部使用UTF-8。如果输入到此类处理的数据或从其输出的数据必须与UTF-8的字符集不同，则相应地执行字符集转换。

字符集转换可能会影响性能。在某些情况下，它可能是有损的。将输入数据转换为UTF-8是无损操作，但是对于无法在输出字符集中表示的字符，转换为输出可能导致信息丢失 。

如果您的JSON数据以Unicode格式存储在数据库中，则无需进行字符集转换即可进行存储或检索。如果数据库字符集是AL32UTF8（Unicode UTF-8）或者您的JSON数据存储在具有字符集AL16UTF16的CLOB实例中，则会出现这种情况。Oracle建议您尽可能使用AL32UTF8作为数据库字符集 。

无论数据库字符集如何，使用数据类型BLOB存储的JSON数据都不会进行字符集转换以进行存储或检索。可以使用数据类型BLOB将JSON数据存储为AL32UTF8，AL16UTF16或AL16UTF16LE。

如果使用SQL / JSON函数或PL / SQL方法转换 JSON数据，并使用数据类型BLOB返回转换结果，则结果将编码为AL32UTF8。即使输入BLOB数据使用另一种Unicode编码，也是如此。

例如，如果使用函数json_query从BLOB输入中提取某些JSON数据并将结果作为BLOB返回，则使用AL32UTF8返回它。

如果应用SQL / JSON函数或PL / SQL方法指定VARCHAR2或CLOB的返回数据类型且数据库字符集不是Unicode，则可能发生有损字符集转换。如果输入JSON数据存储在BLOB实例中，那么即使它最终再次写为BLOB ，如果其中一些临时更改为VARCHAR2或CLOB则生成的BLOB数据可能会遭受有损转换。

跨平台传输数据

您可以跨平台传输数据。

跨平台传输数据的功能可用于：

  • 使数据库可以从一个平台迁移到另一个平台。

  • 为内容提供商提供更简单，更有效的方法来发布结构化数据并将其分发给在不同平台上运行Oracle数据库的客户。

  • 简化从数据仓库环境到数据集市的数据分发，数据集市通常在较小的平台上运行。

  • 假设您可以从这些平台访问存储系统并且平台都具有相同的字节顺序，从而在不同操作系统或平台上的Oracle数据库安装之间共享只读表空间，如以下各节所述。

跨平台数据传输支持许多（但不是全部）平台。您可以查询V$TRANSPORTABLE_PLATFORM视图以查看支持的平台，并确定每个平台的endian格式（字节顺序）。以下查询显示支持跨平台数据传输的平台：

如果源平台和目标平台具有相同的字节顺序，则数据将从源平台传输到目标平台，而无需任何数据转换。

  • DBMS_FILE_TRANSFER包中的GET_FILE或PUT_FILE过程

    当您使用这些过程之一在源平台和目标平台之间移动数据文件时，每个数据文件中的每个块都将转换为目标平台的字节顺序。转换发生在目标平台上。

  • RMAN CONVERT命令

    在源或目标平台上运行RMAN CONVERT命令。此命令将传输的数据转换为目标平台格式。

目的

SUBSTR函数返回一部分char ，从字符position开始， substring_length字符长。SUBSTR使用输入字符集定义的字符计算长度。SUBSTRB使用字节而不是字符。SUBSTRC使用Unicode完整字符。SUBSTR2使用UCS2代码点。SUBSTR4使用UCS4代码点。

  • 如果position为0，则将其视为1。

  • 如果position为positive，则Oracle Database从char的开头计算以查找第一个字符。

  • 如果position为负数，则Oracle从char的末尾开始向后计数。

    $ $ ORACLE_HOME / runInstaller -help

  • 对于Oracle Grid Infrastructure：

    $ /u01/app/19.0.0/grid /gridSetup.sh -help

一段时间后，帮助信息会显示在窗口中。

  1. 完成正常安装的预安装任务。

3.24升级目录

目的

使用UPGRADE CATALOG命令将恢复目录架构从旧版本升级到RMAN客户端所需的版本。

先决条件

必须将RMAN作为恢复目录的所有者连接到必须打开的恢复目录数据库。连接到虚拟专用目录时，不能使用UPGRADE CATALOG命令（请参阅CREATE CATALOG命令）。只能升级基本目录。

恢复目录的版本不得超过RMAN可执行文件所需的版本;否则，RMAN发出错误。RMAN在消息日志中显示升级期间生成的所有错误消息。

Oracle数据库10g R1版本的恢复目录模式需要 CREATE TYPE权限。如果您在10 g R1之前的版本中创建了恢复目录所有者，并且您已授予了当角色未包含CREATE TYPE权限时，此用户的RECOVERY_CATALOG_OWNER角色，则必须在执行升级之前明确地向此用户授予CREATE TYPE 。

如果要将恢复目录升级到Oracle Database 12 c第2版（12.2）或更高版本，则必须运行管理恢复目录特权的dbmsrmansys.sql脚本。此外，如果使用虚拟专用目录，则必须运行升级虚拟专用目录的dbmsrmanvpc.sql脚本。从Oracle Database 12c第1版（12.1.0.2）开始，恢复目录数据库必须使用Oracle数据库企业版。

使用说明

RMAN会提示您连续两次输入UPGRADE CATALOG命令以确认升级。要绕过其他确认步骤，请在第一次运行时输入带有NOPROMPT选项的UPGRADE CATALOG命令。

如果恢复目录已经是最新的，则RMAN允许运行该命令，以便在必要时可以重新创建包。

如果升级到基本恢复目录需要更改现有虚拟专用目录，则RMAN会在下次RMAN连接到该虚拟专用目录时自动进行这些更改。

UPGRADE CATALOG命令不运行脚本来执行升级。相反，RMAN将各种SQL DDL语句发送到恢复目录，以使用新表，视图，列等更新恢复目录模式。

2.2 @@（符号加倍）

目的

使用@@命令执行存储在具有指定文件名的操作系统文件中的一系列RMAN命令。

如果命令文件中包含@@ ，则@@ filename指示RMAN在与调用它的命令文件相同的目录中查找指定的文件名。如果未在命令文件中使用，则@@命令与@ (at sign)命令相同。

先决条件

命令文件必须包含完整的RMAN命令。

使用说明

命令文件是RMAN客户端的本地文件。文件的名称解析取决于操作系统。例如，UNIX或Windows中的@tmp/cmd1.rman表示tmp是当前目录的子目录，文件cmd1.rman位于此子目录中。

为了说明@和@@命令之间的差异，假设您按如下方式调用RMAN：

％rman TARGET / RMAN> @ / tmp / cmd1.rman

假设命令@@cmd2.rman出现在cmd1.rman脚本中。在这种情况下， @@命令指示RMAN在目录/tmp搜索文件cmd2.rman 。

与@命令一样，您可以在命令文件中指定替换变量，然后在执行@@期间将值传递给命令文件（参见示例2-4 ）。

句法

  • ALTER MATERIALIZED VIEW ：从索引组织的物化视图的溢出段中释放未使用的空间（参见ALTER MATERIALIZED VIEW ）

  • ALTER TABLE ：从表中释放未使用的空间，表分区，表子分区，索引组织表的映射表，索引组织表的溢出段或LOB存储段（请参阅ALTER TABLE )

您不能在同一语句中同时指定deallocate_unused_clause和allocate_extent_clause 。

Oracle数据库仅释放高水位线以上的未使用空间（数据库块尚未格式化以接收数据的点）。Oracle从对象的末尾开始释放未使用的空间，并将对象的开头移动到高水位线。

如果扩展中完全包含扩展区，则可以释放整个扩展区以供重用。如果在重新分配中部分地包含范围，则使用到高水位线的部分成为范围，并且剩余的未使用空间被释放以供再次使用。

Oracle将释放空间量计入发生重新分配的表空间的用户配额。

释放的确切空间量取决于INITIAL ， MINEXTENTS和NEXT存储参数的值。有关这些参数的说明，请参阅storage_clause 。

保持 整数

指定在重新分配后数据库对象的段所具有的高水位线上方的字节数。

  • 如果省略KEEP并且高水位线高于INITIAL和MINEXTENTS的大小，则释放高水位MINEXTENTS所有未使用空间。当在高水位小于大小INITIAL或MINEXTENTS ，那么上述所有未使用的空间MINEXTENTS被释放。

  • 如果指定KEEP ，则保留指定的空间量并释放剩余空间。当剩余的区数小于MINEXTENTS ，Oracle MINEXTENTS调整为新的区数。如果初始范围变得小于INITIAL ，则Oracle会将INITIAL调整为新大小。

  • 在任何一种情况下，Oracle都会将NEXT存储参数的值设置为已解除分配的最后一个范围的大小。

关于Oracle Flex ASM集群网络

从Oracle Grid Infrastructure 12c第1版（12.1）开始，作为Oracle Flex Cluster安装的一部分，Oracle ASM在Oracle Grid Infrastructure中配置以提供存储服务。

Oracle Flex ASM使Oracle ASM实例能够在与数据库服务器不同的物理服务器上运行。可以群集许多Oracle ASM实例以支持众多数据库客户端。每个Oracle Flex ASM集群都有自己的名称，在企业内部是全球唯一的。

您可以将所有存储要求合并到一组磁盘组中。所有这些磁盘组都由在单个Oracle Flex Cluster中运行的一小组Oracle ASM实例进行管理。

每个Oracle Flex ASM集群都有一个或多个运行Oracle ASM实例的集群节点。

Oracle Flex ASM可以使用与Oracle Clusterware相同的专用网络，也可以使用自己的专用专用网络。每个网络都可以分为PUBLIC ， ASM & PRIVATE ， PRIVATE或ASM 。

可以在安装期间配置Oracle ASM网络，也可以在安装后配置或修改Oracle ASM网络。

关于群集节点上的Oracle Flex ASM群集配置

可以使用以下特征配置Oracle Flex ASM集群节点：

  • 类似于先前发布的Oracle Grid Infrastructure集群成员节点，因为配置了集群节点角色的所有服务器都是对等服务器。

  • 与Oracle ASM磁盘有直接连接。

  • 运行Direct ASM客户端进程。

  • 运行Oracle ASM过滤器驱动程序，其功能之一是为Oracle Flex ASM群集提供群集防护安全性。

  • 通过为Oracle Flex ASM集群配置的全局Oracle ASM侦听器响应委派给它们的服务请求，该集群指定三个Oracle Flex ASM集群成员节点侦听器作为Oracle Flex ASM集群的远程侦听器。

  • 可以为在Oracle ASM集群的节点上运行的数据库客户端提供对Oracle ASM的远程访问以获取元数据，并允许数据库客户端直接对Oracle ASM磁盘执行块I / O操作。运行Oracle ASM服务器和远程数据库客户端的主机必须都是群集节点。

关于使用Oracle IOServer（IOS）配置的Oracle Flex ASM群集

Oracle IOServer实例为Oracle成员群集的节点上的Oracle数据库实例提供Oracle ASM文件访问，这些节点没有与Oracle ASM托管磁盘的连接。IOS使您可以在此类节点上配置Oracle成员群集。在存储群集上，每个节点上的IOServer实例会打开客户端发送其I / O的网络端口。 IOServer实例从客户端接收数据包，并对Oracle ASM磁盘执行适当的I / O，类似于任何其他数据库客户端。在客户端，数据库可以使用直接NFS（dNFS）与IOServer实例进行通信。但是，使用IOServer不需要客户端配置，因此您无需提供服务器IP地址或任何其他配置信息。在配置为通过IOServer访问Oracle ASM文件的节点和集群上，Oracle IOS实例的发现会自动发生。

要安装Oracle成员集群，Oracle域服务集群的管理员使用crsctl命令创建Oracle成员集群，该命令用于创建成员集群清单文件。在Oracle Grid Infrastructure安装期间，如果选择安装Oracle成员集群，则安装程序会提示您输入成员集群清单文件。“成员集群清单”文件中的属性指定是否希望Oracle成员集群通过IOServer实例访问Oracle ASM文件。

对于Oracle Database Client，响应文件位于$ORACLE_HOME/response目录中。

所有响应文件模板都包含注释条目，示例格式，示例和其他有用的说明。请阅读这些说明，因为它们可帮助您指定响应文件中列出的变量的值并自定义安装。

下表列出了此软件提供的响应文件：

表A-1 Oracle Database Client的响应文件

响应文件 描述

client_install.rsp

无提示安装Oracle数据库客户端

netca.rsp

使用Oracle Net Configuration Assistant静默配置Oracle Net。

警告：

修改响应文件模板并保存文件以供使用时，响应文件可能包含纯文本密码。响应文件的所有权应仅提供给Oracle软件安装所有者，响应文件的权限应更改为600。Oracle强烈建议数据库管理员或其他管理员在不使用时删除或保护响应文件。

要复制和修改响应文件：

  1. 将响应文件从响应文件目录复制到系统上的目录：
    $ cp / directory_path / inventory / response / response_file .rsp local_directory

    在此示例中， directory_path是您复制安装二进制文件的目录的路径。

  2. 在文本编辑器中打开响应文件：
    $ vi / local_dir / response_file .rsp
  3. 按照文件中的说明进行编辑。

    注意：

    如果未正确配置响应文件，安装程序或配置助手将失败。另外，请确保您的响应文件名具有.rsp后缀。

  4. 通过将文件的权限更改为600来保护响应文件：
    $ chmod 600 / local_dir / response_file .rsp

    确保只有Oracle软件所有者用户可以查看或修改响应文件，或者在安装成功后考虑删除它们。

    注意：

    Oracle Database Client安装的完全指定的响应文件包含数据库管理帐户的密码以及作为OSDBA组成员的用户的密码（自动备份所需）。
