检查直接NFS客户端的TCP网络协议缓冲区

检查TCP网络缓冲区大小，确保它足以满足服务器的速度。

默认情况下，TCP的网络缓冲区大小设置为1 MB，UDP的网络缓冲区大小设置为2 MB。 TCP缓冲区大小可以设置文件传输限制，这可能会对Direct NFS客户端用户的性能产生负面影响。

要检查当前的TCP缓冲区大小，请输入以下命令：

#sysctl -a | grep -e net.ipv4.tcp_ [rw] mem

此命令的输出类似于以下内容：

net.ipv4.tcp_rmem = 4096 87380 1056768 net.ipv4.tcp_wmem = 4096 16384 1056768

Oracle建议您根据服务器的链接速度设置值。例如，执行以下步骤：

  1. 以root身份，使用文本编辑器打开/etc/sysctl.conf ，并添加或更改以下内容：

    net.ipv4.tcp_rmem = 4096 87380 4194304 net.ipv4.tcp_wmem = 4096 16384 4194304

  2. 运行以下命令应用您的更改：

    #sysctl -p

  3. 重启网络：

    ＃/ etc / rc.d / init.d / network restart

关于私有IP地址

Oracle Clusterware使用标记为私有的接口进行节点间通信。

每个群集节点必须具有您在安装期间识别为私有接口的接口。专用接口必须具有为接口本身配置的地址，但不需要其他配置。Oracle Clusterware使用您标识为群集互连专用的接口。如果在专用网络的信息期间识别多个接口，则Oracle Clusterware会使用冗余互连使用情况对其进行配置。您标识为私有的任何接口都必须位于连接到群集的每个节点的子网上。Oracle Clusterware使用您标识的所有接口作为专用接口。

对于专用互连，由于Cache Fusion和节点之间的其他流量，Oracle强烈建议使用物理上独立的专用网络。如果使用DNS配置地址，则应确保只能由群集节点访问专用IP地址。

您可以使用oifcfg setif命令在安装或安装后安装期间选择多个互连。

安装后，如果使用CLUSTER_INTERCONNECTS初始化参数修改Oracle Real Application Clusters（Oracle RAC）的互连，则必须将互连更改为未与公共IP地址一起使用的子网上的专用IP地址，作为oifcfg的公共子网。Oracle不支持使用您指定为公有子网的子网将互连更改为接口。

您不应在网络上使用专用网络IP地址的防火墙，因为这可能会阻止互连流量。

关于只读Oracle Homes

从Oracle Database 18c开始，您可以以只读模式配置Oracle主目录。

在只读Oracle主目录中，所有配置数据和日志文件都位于只读Oracle主目录之外。此功能允许您将只读Oracle主目录用作可以跨多个服务器分布的软件映像。

除了传统的ORACLE_BASE和ORACLE_HOME目录之外，以下目录包含以前在ORACLE_HOME中的文件：

  • ORACLE_BASE_HOME

  • ORACLE_BASE_CONFIG

只读Oracle主目录的优点

  • 无需延长停机时间即可无缝修补和更新Oracle数据库。

  • 简化修补和批量部署，因为只需更新一个映像即可将补丁分发到许多服务器。

  • 通过实施安装和配置分离来简化配置。

注意：

此功能不会影响数据库管理员监视，诊断和调整系统性能的方式。

关于自动内存管理安装选项

确定是否要在安装期间配置自动内存管理。

在典型安装期间，使用Oracle Database Configuration Assistant（DBCA）创建数据库，并启用自动内存管理。如果选择“ 高级安装”，则可以手动指定内存分配，也可以启用自动内存管理。

如果数据库实例的总物理内存大于4 GB，则在数据库安装和创建期间无法选择Oracle自动内存管理选项。而是使用自动共享内存管理。自动共享内存管理根据需要自动在各种组件之间分配可用内存，从而允许系统最大限度地使用所有可用的SGA内存。

通过自动内存管理，Oracle数据库实例可以自动管理和调整内存。通过自动内存管理，您可以选择内存目标，实例会自动在系统全局区域（SGA）和实例程序全局区域（实例PGA）之间分配内存。随着内存需求的变化，实例动态地在SGA和实例PGA之间重新分配内存。

您可以在数据库安装期间或之后启用自动内存管理。安装后启用自动内存管理涉及关闭并重新启动数据库。

注意：

默认情况下，在具有4 GB以上RAM的节点上执行典型安装时，将禁用自动内存管理。

异构服务代理

异构服务代理是Oracle数据库连接到非Oracle系统的过程。

访问非Oracle系统的代理进程称为网关。对所有网关的访问通过Oracle数据库中的异构服务组件进行，所有网关都包含相同的代理通用代码。每个网关都有一个不同的驱动程序，它将异构服务映射到非Oracle系统的客户端应用程序编程接口（API）。

代理程序包含两个组件。这些是代理通用代码和非Oracle系统特定的驱动程序。代理主要用于将Oracle数据库与第三方代码隔离。对于访问非Oracle系统的进程，必须将非Oracle系统客户端库链接到该系统。在没有代理程序进程的情况下，这些库必须直接链接到Oracle数据库软件，此代码中的问题可能导致Oracle数据库失败。代理进程将Oracle数据库与第三方代码中的任何问题隔离开来。即使第三方代码中发生致命错误，也只有代理进程会结束。

代理可以存在于以下位置：

  • 与非Oracle系统在同一台计算机上

  • 与Oracle数据库在同一台计算机上

  • 在与这两者不同的计算机上

当用户会话通过数据库链接访问非Oracle系统时，将启动代理进程。这些连接使用Oracle的远程数据访问软件Oracle Net Services进行 ，它支持客户端/服务器和服务器/服务器通信。代理进程继续运行，直到断开用户会话或显式关闭数据库链接。

多线程代理的行为不同。它们必须由数据库管理员显式启动和关闭，而不是由Oracle Net Services自动生成

响应文件的工作原理

响应文件可以帮助您在多台计算机上多次安装Oracle产品。

启动Oracle Universal Installer（OUI）时，您可以使用响应文件自动完成或部分自动安装和配置Oracle软件。OUI使用响应文件中包含的值来提供部分或全部安装提示的答案。

通常，安装程序以交互模式运行，这意味着它会提示您在图形用户界面（GUI）屏幕中提供信息。使用响应文件提供此信息时，可以使用以下任一模式从命令提示符运行安装程序：

  • 静音模式

    如果包含响应文件中所有提示的响应，并在启动安装程序时指定-silent选项，则它将以静默方式运行。在静默模式安装期间，安装程序不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。

  • 响应文件模式

    如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则安装程序将以响应文件模式运行。在响应文件模式安装期间，安装程序将显示所有屏幕，在响应文件中为其指定信息的屏幕，以及未在响应文件中指定所需信息的屏幕。

您可以通过输入响应文件中列出的变量的值来定义静默或响应文件安装的设置。例如，要指定Oracle主目录名，请为ORACLE_HOME环境变量提供Oracle主路径：

4.5了解逆转换

         

         

            

               
了解为什么需要反向转换。

               
反向转换可确保模型返回的信息以与用于训练模型的数据格式类似或相同的格式表示。内部转换在模型细节和评分结果中相反。

               
模型使用的某些属性对应于构建数据中的列。但是，由于算法特有的逻辑，嵌套数据和转换，某些属性不对应于列。

               
例如，训练数据中的嵌套列不会被模型解释为属性。在模型构建期间，Oracle Data Mining会爆炸嵌套列，并且每一行（属性名称/值对）都将成为属性。

               
某些算法（例如支持向量机（SVM）和广义线性模型（GLM））仅对数字属性进行操作。构建数据中的任何非数字列都会分解为二进制属性，一个用于列中的每个不同值（SVM）。GLM不会为原始列中的最常见值生成新属性。仅当案例的列值等于与二进制属性关联的值时，才将这些二进制属性设置为1。

               
生成系数的算法在结果的可解释性方面提出了挑战。示例是SVM和非负矩阵分解（NMF）。这些算法产生与变换后的属性结合使用的系数。系数与变换尺度上的数据相关，而不是原始数据尺度。

               
由于所有这些原因，模型详细信息中列出的属性与用于训练模型的数据列类似。但是，经过嵌入式转换的属性（无论是由自动数据准备（ADP）还是由用户指定的转换列表启动）都会在预转换状态下显示在模型详细信息中，尽可能接近原始列值。虽然属性在模型使用时会被转换，但它们在模型详细信息中以可由用户解释的形式显示。

响应文件的工作原理

         

         

            

               
响应文件可以帮助您在多台计算机上多次安装Oracle产品。

               
启动Oracle Universal Installer（OUI）时，您可以使用响应文件自动完成或部分自动安装和配置Oracle软件。OUI使用响应文件中包含的值来提供部分或全部安装提示的答案。

               
通常，安装程序以交互模式运行，这意味着它会提示您在图形用户界面（GUI）屏幕中提供信息。使用响应文件提供此信息时，可以使用以下任一模式从命令提示符运行安装程序：

               
    
                  
  • 
                     
    静音模式
    
                     
    如果包含响应文件中所有提示的响应，并在启动安装程序时指定-silent选项，则它将以静默方式运行。在静默模式安装期间，安装程序不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。
                     
    
                  
    • 
                  
  • 
                     
    响应文件模式
    
                     
    如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则安装程序将以响应文件模式运行。在响应文件模式安装期间，安装程序将显示所有屏幕，在响应文件中为其指定信息的屏幕，以及未在响应文件中指定所需信息的屏幕。
                     
    
                  
    • 
               

               
您可以通过输入响应文件中列出的变量的值来定义静默或响应文件安装的设置。例如，要指定Oracle主目录名，请为ORACLE_HOME环境变量提供Oracle主路径：
ORACLE_HOME = / U01 /应用程序/ ORACLE /产品/ 19.0.0 / dbhome_1

Oracle建议您在安装，修补或升级数据库后运行utlrp.sql脚本，以识别和重新编译无效对象。
               

               
utlrp.sql脚本重新编译所有无效对象。安装后立即运行脚本，以确保用户不会遇到无效对象。
               

               
               
    
                  
  1. 启动SQL * Plus：
    sqlplus“/ AS SYSDBA”
    
                  
    2. 
                  
  2. 运行utlrp.sql脚本，其中Oracle_home是Oracle主路径：
    SQL> @ Oracle_home /rdbms/admin/utlrp.sql

响应文件的工作原理

         

         

            

               
响应文件可以帮助您在多台计算机上多次安装Oracle产品。

               
启动Oracle Universal Installer（OUI）时，您可以使用响应文件自动完成或部分自动安装和配置Oracle软件。OUI使用响应文件中包含的值来提供部分或全部安装提示的答案。

               
通常，安装程序以交互模式运行，这意味着它会提示您在图形用户界面（GUI）屏幕中提供信息。使用响应文件提供此信息时，可以使用以下任一模式从命令提示符运行安装程序：

               
    
                  
  • 
                     
    静音模式
    
                     
    如果包含响应文件中所有提示的响应，并在启动安装程序时指定-silent选项，则它将以静默方式运行。在静默模式安装期间，安装程序不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。
                     
    
                  
    • 
                  
  • 
                     
    响应文件模式
    
                     
    如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则安装程序将以响应文件模式运行。在响应文件模式安装期间，安装程序将显示所有屏幕，在响应文件中为其指定信息的屏幕，以及未在响应文件中指定所需信息的屏幕。
                     
    
                  
    • 
               

               
您可以通过输入响应文件中列出的变量的值来定义静默或响应文件安装的设置。例如，要指定Oracle主目录名，请为ORACLE_HOME环境变量提供Oracle主路径：
ORACLE_HOME = / U01 /应用程序/ ORACLE /产品/ 19.0.0 / dbhome_1

13.2什么以及何时加载

         

         

            

               
您必须使用loadjava工具加载资源文件。如果使用传统编译器在数据库外部创建.class文件，则必须使用loadjava工具加载它们。加载类文件的替代方法是加载源文件，让Oracle数据库编译和管理生成的类模式对象。在Oracle Database 12c第1版（12.1）中，最有效的方法是在数据库外编译和调试大部分代码，然后加载.class文件。对于特定的Java类，您可以加载其.class文件或相应的.java文件，但不能同时加载它们。
               

               
loadjava工具接受包含源文件和资源文件或类和资源文件的Java Archive（JAR）文件。将JAR或ZIP文件传递给loadjava工具时，默认情况下会打开存档并单独加载其成员。
               

               

                  
注意：

                  
将JAR的内容加载到数据库中时，可以选择创建表示JAR本身的数据库对象。

               

               
自上次加载以来内容未更改的文件不会重新加载。因此，加载JAR文件几乎没有性能损失。加载JAR文件是一种使用loadjava工具的简单， loadjava方法。
               

               
同一模式中的两个模式对象定义同一个类是非法的。例如，假设a.java定义了类x并且您希望将x的定义移动到b.java 。如果a.java已经加载，那么loadjava工具将拒绝尝试加载b.java 。相反，请执行以下任一操作：

               
    
                  
  • 
                     
    删除a.java ，加载b.java ，然后加载新的a.java ，它不定义x 。
                     
    
                  
    • 
                  
  • 
                     
    加载新的a.java ，它不定义x ，然后加载b.java 。

OPatch是一个基于Java的实用程序，可以使用Oracle Universal Installer进行安装。Opatch与平台无关。它在所有支持的操作系统上运行。另一个版本的OPatch，称为独立OPatch，也可用。它在没有Oracle Universal Installer的Oracle家庭上运行。

               
修补程序是复制到现有安装的一小部分文件。它们与特定版本的Oracle产品相关联。应用于已安装产品的正确版本时，修补程序会生成产品的升级版本。

opatch lsinventory -patch

响应文件的工作原理

         

         

            

               
响应文件可以帮助您在多台计算机上多次安装Oracle产品。

               
启动Oracle Universal Installer（OUI）时，您可以使用响应文件自动完成或部分自动安装和配置Oracle软件。OUI使用响应文件中包含的值来提供部分或全部安装提示的答案。

               
通常，安装程序以交互模式运行，这意味着它会提示您在图形用户界面（GUI）屏幕中提供信息。使用响应文件提供此信息时，可以使用以下任一模式从命令提示符运行安装程序：

               
    
                  
  • 
                     
    静音模式
    
                     
    如果包含响应文件中所有提示的响应，并在启动安装程序时指定-silent选项，则它将以静默方式运行。在静默模式安装期间，安装程序不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。
                     
    
                  
    • 
                  
  • 
                     
    响应文件模式
    
                     
    如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则安装程序将以响应文件模式运行。在响应文件模式安装期间，安装程序将显示所有屏幕，在响应文件中为其指定信息的屏幕，以及未在响应文件中指定所需信息的屏幕。
                     
    
                  
    • 
               

               
您可以通过输入响应文件中列出的变量的值来定义静默或响应文件安装的设置。例如，要指定Oracle主目录名，请为ORACLE_HOME环境变量提供Oracle主路径：
ORACLE_HOME = / U01 /应用程序/ ORACLE /产品/ 19.0.0 / dbhome_1

了解使用批次滚动升级

         

         

            

               
您可以批量执行Oracle Grid Infrastructure的滚动升级。

               
您可以使用root用户自动化在升级期间自动运行rootupgrade.sh脚本。使用root用户自动化时，可以将节点分成组或批次，并开始升级这些批次。在批处理之间，您可以将服务从运行先前版本的节点移动到已升级的节点，以便服务不受升级的影响。Oracle建议您使用root自动化，并允许rootupgrade.sh脚本自动停止和启动实例。您还可以继续手动运行root脚本。
               

               
在不使用root用户自动化的情况下升级Oracle Grid Infrastructure时，将升级整个群集。您无法选择或取消选择单个节点进行升级。Oracle不支持在滚动升级期间尝试向群集添加其他节点。Oracle建议您在升级Oracle Clusterware时保持Oracle RAC实例正常运行。在每个节点上启动root脚本时，将关闭该节点上的数据库实例，然后rootupgrade.sh脚本将再次启动实例。
               

               

                  
选择节点进行批量升级的限制

  • 
                        
    您可以批量池化节点进行升级，最多可以批量处理三个批次。
    
                     
    • 
                     
  • 
                        
    必须以批处理方式升级运行Oracle Universal Installer（OUI）的本地节点。

ROWID伪列

         

         

            

               

                  
对于数据库中的每一行， ROWID伪列返回行的地址。Oracle数据库rowid值包含查找行所需的信息：

                  
    
                     
  • 
                        
    对象的数据对象编号
    
                     
    • 
                     
  • 
                        
    行所在的数据文件中的数据块
    
                     
    • 
                     
  • 
                        
    数据块中行的位置（第一行为0）
    
                     
    • 
                     
  • 
                        
    行所在的数据文件（第一个文件为1）。文件号是相对于表空间的。
    
                     
    • 
                  

                  
通常，rowid值唯一标识数据库中的行。但是，同一群集中存储在一起的不同表中的行可以具有相同的rowid。

                  
ROWID伪列的值具有数据类型ROWID或UROWID 。有关更多信息，请参阅Rowid数据类型和UROWID数据类型 。
                  

                  
 Rowid值有几个重要用途：

                  
    
                     
  • 
                        
    它们是访问单行的最快方式。
    
                     
    • 
                     
  • 
                        
    他们可以向您展示如何存储表中的行。
    
                     
    • 
                     
  • 
                        
    它们是表中行的唯一标识符。
    
                     
    • 
                  

                  
您不应该使用ROWID作为表的主键。例如，如果使用“导入”和“导出”实用程序删除并重新插入行，则其rowid可能会更改。如果删除一行，则Oracle可能会将其rowid重新分配给稍后插入的新行。
                  

                  
尽管可以在查询的SELECT和WHERE子句中使用ROWID伪列，但这些伪列值实际上并不存储在数据库中。您无法插入，更新或删除ROWID伪列的值。
                  

               

               
               

                  
例

               

               
               

                  
此语句选择包含部门20中员工数据的所有行的地址：
SELECT ROWID，last_name FROM employees WHERE department_id = 20;

手动配置存储设备路径持久性

         

         

            

               
您可以通过创建规则文件来维护存储文件路径持久性。

               
默认情况下，设备文件命名方案udev在服务器启动时动态创建设备文件名，并将其所有权分配给root。如果udev应用默认设置，则它会更改磁盘的Oracle设备文件名和所有者，从而使服务器重新启动时无法访问磁盘。例如，用户grid拥有的名为/dev/sdd的设备上的表决磁盘可能位于重新启动服务器后由root拥有的名为/dev/sdf的设备上。
               

               
如果使用Oracle ASMFD，则无需在udev确保权限和设备路径持久性。

               
如果不使用Oracle ASMFD，则必须创建自定义规则文件。Linux供应商定制他们的udev配置并使用不同的顺序来读取规则文件。例如，在启动udev某些Linux发行版中，它会依次执行规则文件中定义的规则（配置指令）。这些文件位于路径/etc/udev/rules.d/ 。规则文件以词法顺序读取。例如，文件10-wacom.rules中的规则在规则文件90-ib.rules中的规则之前被解析和执行。
               

               
在udev规则文件中指定设备信息时，请确保在显示的顺序中的所有其他特征之前指定了OWNER，GROUP和MODE。例如，要在UDEV行上包含特征ACTION，请在OWNER，GROUP和MODE之后指定ACTION。

               
规则文件描述相同设备的位置，在受支持的Linux内核版本上，最后读取的文件是应用的文件。

挖掘模型是执行数据挖掘的数据库模式对象。

               
与所有架构对象一样，访问挖掘模型由数据库权限控制。可以导出和导入模型。嘿支持注释，它们可以在数据库中跟踪听觉系统。

注意：

了解使用批次滚动升级

         

         

            

               
您可以批量执行Oracle Grid Infrastructure的滚动升级。

               
您可以使用root用户自动化在升级期间自动运行rootupgrade.sh脚本。使用root用户自动化时，可以将节点分成组或批次，并开始升级这些批次。在批处理之间，您可以将服务从运行先前版本的节点移动到已升级的节点，以便服务不受升级的影响。Oracle建议您使用root自动化，并允许rootupgrade.sh脚本自动停止和启动实例。您还可以继续手动运行root脚本。
               

               
在不使用root用户自动化的情况下升级Oracle Grid Infrastructure时，将升级整个群集。您无法选择或取消选择单个节点进行升级。Oracle不支持在滚动升级期间尝试向群集添加其他节点。Oracle建议您在升级Oracle Clusterware时保持Oracle RAC实例正常运行。在每个节点上启动root脚本时，将关闭该节点上的数据库实例，然后rootupgrade.sh脚本将再次启动实例。

Unicode是一种通用编码字符集，允许您使用单个字符集存储来自任何语言的信息。现代标准（如XML，Java，JavaScript和LDAP）需要Unicode。 Unicode符合ISO / IEC标准10646。有关ISO标准的信息，请访问国际标准化组织的网站：
http://www.iso.ch/

Oracle数据库18c中与Unicode标准的9.0版本规范。有关Unicode标准的最新信息，请访问Unicode Consortium的Web站点：
http://www.unicode.org

Oracle通过AL32UTF8字符集支持Unicode标准的UTF-8编码方案，通过AL16UTF16字符集支持UTF-16BE编码方案，通过AL16UTF16LE字符集支持UTF-16LE编码方案。AL32UTF8作为基于ASCII的平台上的客户端和数据库字符集有效。AL16UTF16作为所有平台上的国家（ NCHAR ）字符集有效。AL16UTF16LE无效作为客户端，数据库或国家字符集。

转换函数支持所有提到的Oracle字符集。

                  
Oracle建议使用AL32UTF8字符集和AL16UTF16国家（ NCHAR ）字符集创建基于ASCII的平台上的数据库。Oracle建议您避免使用NCHAR数据类型和相关的国家字符集，因为某些RDBMS组件（如Oracle Text和Oracle XDB）不支持它们。

    静音模式
    
                     
    如果包含响应文件中所有提示的响应，并在启动安装程序时指定-silent选项，则它将以静默方式运行。在静默模式安装期间，安装程序不显示任何屏幕。相反，它会在您用于启动它的终端中显示进度信息。
                     
    
                  
    • 
                  
  • 
                     
    响应文件模式
    
                     
    如果在响应文件中包含部分或全部提示的响应并省略-silent选项，则安装程序将以响应文件模式运行。在响应文件模式安装期间，安装程序将显示所有屏幕，在响应文件中为其指定信息的屏幕，以及未在响应文件中指定所需信息的屏幕。
                     
    
                  
    • 
               

               
您可以通过输入响应文件中列出的变量的值来定义静默或响应文件安装的设置。例如，要指定Oracle主目录名，请为ORACLE_HOME环境变量提供Oracle主路径：
ORACLE_HOME = / U01 /应用程序/ ORACLE /产品/ 19.0.0 / dbhome_1
