1 Oracle提供的PL / SQL包和类型简介

Oracle为Oracle服务器提供了许多PL / SQL包，以扩展数据库功能并提供对SQL功能的PL / SQL访问。您可以在创建应用程序时使用提供的包，也可以在创建自己的存储过程时使用提供的包。

本手册涵盖Oracle数据库服务器随附的软件包。不包括随其他产品提供的软件包，例如Oracle Developer或Oracle Application Server。

请注意，并非默认情况下安装本手册或Oracle数据库文档库中其他位置描述的每个包或类型。在这种情况下，文档说明了这一点并解释了如何安装对象。以适当的特权用户身份运行此查询：

1.1包概述

包是一起存储在数据库中的相关程序对象的封装集合。程序对象是过程，函数，变量，常量，游标和异常。

1.1.1包装组件

PL / SQL包有两个部分：规范和正文，尽管有时不需要正文。规范是您的应用程序的接口;它声明了可供使用的类型，变量，常量，异常，游标和子程序。主体完全定义了游标和子程序，因此实现了规范。

与子程序不同，无法调用，参数化或嵌套包。但是，包和子程序的格式类似：

CREATE PACKAGE名称AS  - 规范（可见部分） - 公共类型和项目声明 - 子程序规范END [name]; CREATE PACKAGE BODY名称AS  -  body（隐藏部分） - 私有类型和项目声明 - 子程序体[BEGIN  - 初始化语句] END [name];

该规范包含您的应用程序可见的公共声明。正文包含对应用程序隐藏的实现详细信息和私有声明。您可以在不更改规范的情况下调试，增强或替换包体。您可以在不重新编译调用程序的情况下更改包体，因为正文中的实现详细信息对应用程序是隐藏的。

1.1.2使用Oracle提供的软件包

创建数据库时，会自动安装大多数Oracle提供的软件包。某些包未自动安装。有关这些软件包的特殊安装说明，请参见各章节。

要从SQL调用PL / SQL函数，您必须拥有该函数或对该函数具有EXECUTE特权。要从使用PL / SQL函数定义的视图中进行选择，您必须对视图具有SELECT权限。从视图中选择不需要单独的EXECUTE特权。有关包装特殊要求的说明在各章节中有说明。

1.1.3创建新包

要创建包并将它们永久存储在Oracle数据库中，请使用CREATE PACKAGE和CREATE PACKAGE BODY语句。您可以从SQL * Plus或Enterprise Manager以交互方式执行这些语句。

要创建新包，请执行以下操作：

  1. 使用CREATE PACKAGE语句创建包规范。

    您可以在包规范中声明程序对象。这些对象称为公共对象。可以在包外部以及包中的其他对象引用公共对象。

    注意：

    添加它通常更方便 CREATE PACKAGE语句中的OR REPLACE子句。但请注意，如果您要覆盖具有相同名称的现有包， CREATE PACKAGE发出警告，而CREATE OR REPLACE just覆盖它而不会发出警告。

  2. 使用CREATE PACKAGE BODY语句创建包体。

    您可以在包体中声明和定义程序对象。

    • 您必须定义包规范中声明的公共对象。

    • 您可以声明和定义其他包对象，称为私有对象。私有对象在包体中而不是在包规范中声明，因此它们只能由包中的其他对象引用。它们不能在包外引用。

1.1.3.1分离规格和主体

包的规范声明了在包的直接范围之外可见的公共类型，变量，常量和子程序。包的主体定义规范中声明的对象，以及包外的应用程序不可见的私有对象。

Oracle将包的规范和主体分别存储在数据库中。调用或引用公共程序对象的其他模式对象仅取决于程序包规范，而不取决于程序包主体。使用此区别，您可以更改程序包主体中程序对象的定义，而不会导致Oracle使调用或引用程序对象的其他模式对象无效。仅当您更改程序包规范中的程序对象的声明时，Oracle才会使依赖的模式对象无效。

OLAP简介Java API描述了OLAP维度数据模型，并简要介绍了实现该模型的一些OLAP Java API类。这些类位于oracle.olapi.metadata包中。使用这些类，您可以执行以下任务。

  • 获取对可用元数据对象的访问权限

  • 创建新的元数据对象

  • 在分析工作区中或作为关系对象部署元数据对象

  • 将元数据对象映射到数据源

  • 将元数据对象导出到XML或从XML导入它们

  • 创建Source对象以查询数据

为了将元数据对象导入为XML模板， MdmMetadataProvider具有importXML方法的几个签名。

  • 一个java.io.用于输入XML的Reader或包含要导入的XML的String 。

  • 用于包含新的或已修改的元数据对象的MdmDatabaseSchema 。

  • 一个boolean modifyIfExists ，指示您是否希望导入的XML定义中的差异能够修改同名的现有对象。

  • 一个可选的Map ， bindValues ，它包含绑定变量作为键，并且作为键的对象， String值用于替换绑定变量。有关此参数中绑定值的更多信息，请参阅“ 在XML模板中描述绑定变量 ” 。

  • oracle.olapi.metadata.的可选实现oracle.olapi.metadata.XMLParserCallback接口。

如果modifyIfExists参数的值为true并且导入的XML包含已存在的对象的完整定义且对象定义与XML不同，则该方法将对象定义的新元素或已更改元素与现有元素合并对象的定义。如果modifyIfExists值为false并且XML包含已存在的对象的完整定义，则importXML方法将引发异常。

您可以将XMLParserCallback接口的实现作为parserCallback参数传递给importXML方法。使用XMLParserCallback ，您可以指定Oracle OLAP如何处理导入XML时可能发生的错误。 XML11_2_ParserCallback接口添加了重命名导入对象和抑制导入对象属性的方法。

2.4.1.2代表措施

MdmMeasure是一个抽象类，表示由一个或多个MdmPrimaryDimension对象组织的一组数据。数据的结构类似于多维数组的结构。与提供用于标识数组中特定单元格的索引的数组的MdmPrimaryDimension ，组织MdmMeasure的MdmPrimaryDimension对象提供用于标识MdmMeasure的特定值的MdmMeasure 。

例如，假设您有一个MdmMeasure ，其中包含的数据记录了在一段时间内通过销售渠道销售给客户的产品数量。该度量的数据按产品，时间，客户和渠道的维度进行组织（通道代表销售渠道，例如目录或互联网）。您可以将数据视为占用四维数组，其中包含提供组织结构的产品，时间，客户和渠道维度。这四个维度的值是用于识别阵列中的每个特定单元的索引。每个单元格包含销售单位数量的单个数据值。您必须为每个维指定一个值，以便标识数组中的值。

MdmMeasure的值通常是数字，但度量可以包含其他数据类型的值。MdmMeasure的具体子类是MdmBaseMeasure和MdmDerivedMeasure 。

分析工作区中的MdmBaseMeasure具有关联的物理存储结构。典型地， MdmCube获取的基本数据用于MdmBaseMeasure从一个事实表中的列。然后，Oracle OLAP计算度量的聚合值，并将这些值存储在多维数据集的OLAP视图中。

创建MdmBaseMeasure ，可以使用setSQLDataType方法指定度量的SQL数据类型。如果未指定，则MdmBaseMeasure具有您映射到的源数据的数据类型。

通过使用setAllowAutoDataTypeChange方法指定true ，可以允许Oracle OLAP自动设置度量的SQL数据类型。如果度量的数据类型发生更改，这可能很有用。如果允许自动更改SQL数据类型，则无论是否使用setSQLDataType方法指定了SQL数据类型，Oracle OLAP都会确定相应的SQL数据类型。

MdmDerivedMeasure没有关联的物理存储。Oracle OLAP根据需要动态计算MdmDerivedMeasure的值。

一个的值MdmMeasure由结构决定MdmPrimaryDimension所述的对象MdmMeasure 。也就是说， MdmMeasure每个值都由元组标识，元组是来自MdmPrimaryDimension对象的成员的唯一组合。

所述MdmPrimaryDimension一个的对象MdmMeasure是MdmStandardDimension或MdmTimeDimension对象。它们通常至少具有一个层次结构。那些MdmPrimaryDimension对象包括其组件MdmHierarchy对象的所有成员。由于这种结构， MdmMeasure的值是以下一个或多个：

  • 来自MdmMeasure所基于的事实表列，视图或计算的值。这些值由维度层次结构的叶级别的成员组合来标识。

  • Oracle OLAP提供的聚合值。这些度量值由层次结构的聚合级别中的至少一个成员标识。

  • 由Expression for MdmDerivedMeasure或自定义维度成员指定的值。

例如，假设一个MdmBaseMeasure ，其尺寸为MdmTimeDimension和MdmStandardDimension产品。度量和维度的元数据对象是mdmUnitCost ， mdmTimeDim和mdmProdDim 。每个mdmTimeDim和mdmProdDim对象都具有它所代表的维度的所有叶子成员和聚合成员。叶子成员是没有孩子的成员。聚合成员有一个或多个孩子。

两个成员的唯一组合（一个来自mdmTimeDim ，一个来自mdmProdDim ）标识每个mdmUnitCost值，每个可能的维度成员组合用于指定整个mdmUnitCost值集合。

某些mdmUnitCost值由叶成员的组合标识（例如，特定产品项和特定月份）。其他mdmUnitCost值由聚合成员（例如，特定产品系列和特定季度）的组合标识。还有其他mdmUnitCost值由叶子和聚合成员的混合物识别。

仅由叶成员标识的mdmUnitCost的值直接来自数据库事实表（或事实表计算）中的列。它们代表最低级别的数据。但是，由Oracle OLAP计算由至少一个聚合成员标识的值。这些更高级别的值表示聚合或汇总的数据。因此，由MdmBaseMeasure表示的数据是来自数据存储的事实表数据和Oracle OLAP可用于分析操作的聚合数据的混合。

2.4.2.3.1表示基于级别的层次结构

MdmLevelHierarchy是的子类MdmHierarchy 。MdmLevelHierarchy具有树状结构，具有顶部或最高级别，以及叶子或最低级别。每个成员可以在层次结构中具有零个或一个父级。不允许循环，例如，如果成员A是成员B的父成员，成员B是成员C的父成员，成员C是成员A的父成员。

不属于任何其他成员的孩子的成员是最高成员。会员有孩子聚集或层级的集合体部件 。没有子节点的成员是层次结构的叶子或叶子成员 。

如果层次结构的成员具有父级，则该父级必须处于更高级别。Oracle OLAP期望层次结构中的所有叶成员都处于叶级别。您可以指定Oracle OLAP允许层次结构不规则 。在不规则的层次结构中，一个或多个叶子成员不在叶级别中。您可以通过调用setIsRagged(true)方法来指定允许层次结构MdmLevelHierarchy 。

2.4.2.3.2表示基于值的层次结构

2.4.2.4表示层次结构级别

MdmHierarchyLevel是的子类MdmSubDimension 。一个MdmHierarchyLevel相关联的MdmDimensionLevel与MdmLevelHierarchy 。

131.1 DBMS_REDACT概述

数据编辑提供了一种为应用程序定义屏蔽策略的方法。Oracle Data Redaction提供了屏蔽（编辑）从应用程序中的用户SELECT查询返回的数据的功能。掩蔽是实时进行的。数据编辑策略适用于查询用户，具体取决于此用户的SYS_CONTEXT或XS_SYS_CONTEXT值。该编校过程不要求查询的数据是静态的或不变的，或者要求以离线方式一次编辑整个数据集。Oracle数据库仅编辑用户查询指定的行的数据，而不是整个列的数据。在数据返回到查询用户或应用程序之前立即进行编校。

131.2 DBMS_REDACT安全模型

如果查询用户具有EXEMPT REDACTION POLICY系统特权，则不会执行编校。如果用户没有EXEMPT REDACTION POLICY系统特权，则将在当前用户的环境中评估策略表达式。如果策略表达式的计算结果为TRUE ，则将执行编校，否则将不执行编校。

您需要DBMS_REDACT包的EXECUTE特权才能执行其子程序。接口中的过程使用当前用户的权限执行。

  9. 在“确认”窗口中，单击“ 确定” 。

关于使用Database Vault Operations Control

您可以自动限制普通用户访问自治，常规云或内部部署环境中的可插拔数据库（PDB）本地数据。

要实现此目的，您可以使用Oracle Database Vault操作控制，该控制适用于普通用户（例如基础架构数据库管理员和应用程序）。

Database Vault操作控制对于数据库管理员必须以高权限用户身份登录CDB根目录但仍无法访问PDB客户数据的情况非常有用。数据库操作控制不会阻止PDB数据库管理员。要阻止这些用户，请在PDB中启用Oracle Database Vault，然后使用域库控制等数据库保险库功能来阻止这些用户。

对于公共用户或应用程序必须执行必须访问PDB上的本地数据的任务的情况，您可以为公共用户和程序包的Database Vault操作控制创建例外列表。您要为例外列表指定的普通用户类型的示例是CTXSYS应用程序帐户，该帐户负责Oracle Text。通过在例外列表中指定包，您可以应用更细粒度的控件，而不是在例外列表中提供对用户的完全访问权限。

使用Database Vault操作控制的一般过程如下：

  1. 启用Database Vault操作控制并为生产环境启用它。
  2. 在此阶段，无论PDB是否已启用Database Vault，Database Vault操作控制都适用于环境中的所有PDB。
  3. 要使特定用户和程序包能够访问PDB的本地模式，请将它们添加到例外列表中。当用户或包不再需要访问权限时，您可以从例外列表中删除它们。例如，如果数据库使用的是Oracle Text，则可以将CTXSYS管理用户帐户和程序包添加到例外列表中。

启用Database Vault Operations Control

要启用Database Vault操作控制，请使用DBMS_MACADM.ENABLE_APP_PROTECTION PL / SQL过程。

Oracle建议，如果您选择对多租户生产服务器使用Database Vault操作控制，则应始终启用Database Vault操作控制。

在大多数情况下，您将为整个CDB启用数据库操作控制，而不仅仅是特定的PDB。如果需要为特定PDB禁用它（例如，用于故障排除），则可以在PDB上执行DBMS_MACADM.DISABLE_APP_PROTECTION过程。完成PDB故障排除后，请重新启用它以进行Database Vault操作控制，如本主题中的示例所示。
在启用Database Vault操作控制之前，必须在CDB根目录中启用并配置Database Vault。但是，不需要在PDB中启用Database Vault。
  1. 以已被授予DV_OWNER角色的普通用户身份登录CDB根目录。
    例如：
    sqlplus c ## sec_admin_owen_root输入密码： 密码
  2. 执行DBMS_MACADM.ENABLE_APP_PROTECTION过程。
    • 要为CDB环境中的所有PDB启用Database Vault操作控制：
      EXEC DBMS_MACADM.ENABLE_APP_PROTECTION;
    • 由于故障排除原因，可能已禁用特定PDB的操作控制。要为特定PBB重新启用Database Vault操作控制（例如， HRPDB ）：
      EXEC DBMS_MACADM.ENABLE_APP_PROTECTION（'HRPDB'）;
在此阶段，启用一个或所有PDB以进行Database Vault操作控制。您可以通过连接为用户确认SYS与SYSDBA管理权限，然后执行SELECT * FROM DBA_DV_STATUS;查询。如果特定的受信任公共用户或程序包必须能够访问这些PDB的本地模式以执行特殊操作，则可以使用DBMS_MACADM.ADD_APP_EXCEPTION过程将用户或程序包添加到Database Vault操作控制的例外列表中。

24.1.1 SQL Tuning Advisor的用途

SQL Tuning Advisor是一种用于解决与次优执行SQL语句相关的问题的机制。

使用SQL Tuning Advisor获取有关提高高负载SQL语句性能的建议，并通过仅执行最佳计划来防止回归。

调整建议包括：

  • 对象统计的集合

  • 创建索引

  • 重写SQL语句

  • 创建SQL配置文件

  • 创建SQL计划基准

SQL Tuning Advisor生成的建议可帮助您实现以下特定目标：

  • 避免劳动密集型手动调整

    即使对于专家来说，识别和调整高负载SQL语句也具有挑战性。SQL Tuning Advisor使用优化器为您调优SQL。

  • 自动生成建议并实施SQL配置文件

    您可以将自动SQL调整任务配置为在维护窗口中每晚运行。以这种方式调用时，顾问程序可以生成建议并自动实现SQL配置文件。

  • 分析数据库生成的统计信息以实现最佳计划

    该数据库包含大量有关其自身操作的统计信息。SQL Tuning Advisor可以执行深度挖掘和内部信息分析，以改进执行计划。

  • 使开发人员能够在测试系统而不是生产系统上调优SQL

    当在生产数据库上发生次优执行SQL语句时，开发人员可能不希望直接在生产数据库上进行调查和调优。DBA可以将有问题的SQL语句传输到测试数据库，开发人员可以在其中安全地分析和调整它们。

调整多个语句时，SQL Tuning Advisor无法识别语句之间的相互依赖性。相反，SQL Tuning Advisor提供了一种方便的方法来获取许多语句的调优建议。

24.1.2.1 SQL Tuning Advisor的输入

SQL Tuning Advisor的输入可以来自多个来源，包括ADDM，AWR，共享SQL区域和SQL调优集。

SQL Tuning Advisor使用其输入源，如下所示：

  • 自动数据库诊断监视器（ADDM）

    SQL Tuning Advisor的主要输入源是ADDM（发音为Adam ）。默认情况下，ADDM每小时主动运行一次。为了识别涉及高负载SQL语句的性能问题，ADDM分析了自动工作负载存储库（AWR）在过去一小时内收集的关键统计信息。如果标识了高负载SQL语句，则ADDM建议在SQL上运行SQL Tuning Advisor。

  • AWR

    AWR定期拍摄系统活动的快照，包括按相关统计信息排序的高负载SQL语句，如CPU消耗和等待时间。

    您可以查看AWR并手动识别高负载SQL语句。您可以在这些语句上运行SQL Tuning Advisor，尽管Oracle数据库会自动执行此工作作为自动SQL调优的一部分。默认情况下，AWR会保留过去八天的数据。您可以使用此技术查找和调整在AWR保留期内运行的任何高负载SQL。

  • 共享SQL区域

    数据库使用共享SQL区域来调整尚未在AWR中捕获的最新SQL语句。共享SQL区域和AWR提供了从AWR保留允许的当前时间识别和调整高负载SQL语句的功能，默认情况下至少为8天。

  • SQL调优集

    SQL调优集（STS）是一个数据库对象，它存储SQL语句及其执行上下文。STS可以包括尚未部署的SQL语句，目的是衡量其个人绩效，或识别性能低于预期的那些。当一组SQL语句用作输入时，数据库必须首先构造并使用STS。

例如，假设在调优之前执行时间是100秒，并且在实施推荐之后，新的执行时间预计为33秒。此绩效改进的收益计算如下：

67％=（100  -  33）/（100）

您可以选择是否接受优化SQL语句的建议。根据其配置方式，Automatic SQL Tuning Advisor可以实现SQL配置文件建议，以便在无需用户干预的情况下调整语句。当按需调用时，SQL Tuning Advisor可以建议用户实现SQL配置文件，但永远不能自动实现它。

“图24-8结构分析”的描述

某些语法变体会对性能产生负面影响在结构分析中，自动调优优化器根据一组规则评估语句，识别低效的编码技术，并在可能的情况下推荐替代语句。

如图24-8所示，Automatic Tuning Optimizer可识别以下类别的结构问题：

  • SQL构造函数的使用效率低下

    执行次优的语句可能使用NOT IN而不是NOT EXISTS ，或UNION而不是UNION ALL 。在UNION操作，而不是在UNION ALL操作，采用了独特的排序，以确保没有重复的行是在结果集中。如果您知道两个查询不返回重复项，则使用UNION ALL 。

  • 数据类型不匹配

    如果索引列和比较值的数据类型不匹配，则由于隐式数据类型转换，数据库不使用索引。此外，数据库必须花费额外的资源来转换数据类型，并且某些SQL语句可能会失败，因为数据值无法正确转换。常见错误包括包含数字数据但从不用于算术运算的列：电话号码，信用卡号和支票号。为避免基数估计不佳，次优计划和ORA-01722错误，开发人员必须确保绑定变量的类型为VARCHAR2而不是数字。

  • 设计错误

    设计错误的典型示例是缺少连接条件。如果n是查询块中的表数，则必须存在n -1个连接条件才能避免使用笛卡尔积。

在每种情况下，自动调整优化程序都会提供相关建议来重构语句。建议的替代声明与原始声明类似，但不相同。例如，建议的语句可能使用UNION ALL而不是UNION 。然后，您可以确定建议是否合理。

许多DBA没有时间进行SQL调优所需的密集分析。即使他们这样做，SQL调优也涉及几个手动步骤。由于几个不同的SQL语句在任何给定的日期可能都是高负载，因此DBA可能不得不花费相当大的精力来监视和调整它们。 。

中自动SQL调整任务不处理以下类型的SQL语句：

  • 在一周内不重复的即席SQL语句或SQL语句

  • 并行查询

  • 在SQL配置文件之后运行时间太长的查询，因此SQL Tuning Advisor测试执行是不切实际的

  • 递归SQL

您可以根据需要运行SQL Tuning Advisor来调整前面的SQL语句类型。

24.2.3.2使用命令行配置自动SQL调整任务

DBMS_AUTO_SQLTUNE包使您可以通过使用SET_AUTO_TUNING_TASK_PARAMETER过程指定任务参数来配置自动SQL调整。

由于任务由SYS拥有，因此只有SYS可以设置任务参数。

ACCEPT_SQL_PROFILE调整任务参数指定是自动实现SQL配置文件（ true ）还是需要用户干预（ false ）。默认值为AUTO ，如果至少有一个SQL语句与SQL配置文件一起存在，则表示为true如果不满足此条件，则为false 。

注意：

启用自动实现时，顾问程序仅实现创建SQL配置文件的建议。不会自动实现诸如创建新索引，收集优化程序统计信息和创建SQL计划基准等建议。

假设

本教程假设以下内容：

  • 您希望数据库自动实现SQL配置文件，但每次执行不超过50个SQL配置文件，并且数据库上的配置文件总数不超过50个。

  • 您希望任务在每次执行1200秒后超时。

设置自动SQL调整任务参数：

  1. 使用适当的权限将SQL * Plus连接到数据库，然后可选地查询当前任务设置。

    例如，使用管理员权限将SQL * Plus连接到数据库并执行以下查询：

    COL PARAMETER_NAME FORMAT a25 COL VALUE FORMAT a10 SELECT PARAMETER_NAME，PARAMETER_VALUE作为“VALUE”来自DBA_ADVISOR_PARAMETERS WHERE（（TASK_NAME ='SYS_AUTO_SQL_TUNING_TASK'）和（（PARAMETER_NAME LIKE'％PROFILE％'）或（PARAMETER_NAME ='LOCAL_TIME_LIMIT'）或（PARAMETER_NAME = 'EXECUTION_DAYS_TO_EXPIRE'）））;

24.3.2创建SQL调整任务

要创建SQL调优任务，请执行DBMS_SQLTUNE.CREATE_TUNING_TASK函数。

您可以从以下任何一项创建调整任务：

  • 单个SQL语句的文本

  • 包含多个语句的SQL调优集

  • SQL标识从共享SQL区域中选择的SQL语句

  • 从AWR通过SQL标识符选择的SQL语句

scope参数是此函数最重要的参数之一。您可以将此参数设置为以下值：

  • 有限

    SQL Tuning Advisor基于统计检查，访问路径分析和SQL结构分析生成建议。不生成SQL配置文件建议。

  • 全面

    SQL Tuning Advisor执行在有限范围内执行的所有分析以及SQL分析。

假设

本教程假设以下内容：

  • 您希望以具有ADVISOR权限的用户hr进行调整。

  • 您想调整以下查询：

    SELECT / * + ORDERED * / * FROM employees e，locations l，departments d WHERE e.department_id = d.department_id AND l.location_id = d.location_id AND e.employee_id <：bnd;

  • 您希望将绑定变量100传递给前面的查询。

  • 您希望SQL Tuning Advisor执行SQL分析。

  • 您希望任务运行时间不超过60秒。

要创建SQL调整任务：

  1. 使用适当的权限将SQL * Plus连接到数据库，然后运行DBMS_SQLTUNE.CREATE_TUNING_TASK函数。

    例如，执行以下PL / SQL程序：

    DECLARE my_task_name VARCHAR2（30）; my_sqltext CLOB; BEGIN my_sqltext：='SELECT / * + ORDERED * / *'|| '来自员工e，地点l，部门d'|| 'WHERE e.department_id = d.department_id AND'|| 'l.location_id = d.location_id AND'|| 'e.employee_id <：bnd'; my_task_name：= DBMS_SQLTUNE.CREATE_TUNING_TASK（sql_text => my_sqltext，bind_list => sql_binds（anydata。ConvertNumber（100）），user_name =>'HR'，scope =>'COMPREHENSIVE'，time_limit => 60，task_name =>'STA_SPECIFIC_EMP_TASK'，description =>'调整对指定员工的查询的任务'）;结束; /

  2. （可选）查询任务的状态。

    以下示例查询当前用户拥有的所有任务的状态，在此示例中为hr ：

    COL TASK_ID FORMAT 999999 COL TASK_NAME FORMAT a25 COL STATUS_MESSAGE FORMAT a33从USER_ADVISOR_LOG中选择TASK_ID，TASK_NAME，STATUS，STATUS_MESSAGE;

    示例输出显示如下：

    TASK_ID TASK_NAME状态STATUS_MESSAGE ------- ------------------------- ----------- --- ----------- 884 STA_SPECIFIC_EMP_TASK INITIAL

    在前面的输出中， INITIAL状态表示任务尚未开始执行。

24.3.3配置SQL调优任务

要在创建调整任务后更改调整任务的参数，请执行DBMS_SQLTUNE.SET_TUNING_TASK_PARAMETER函数。

本教程假设以下内容：

  • 您希望使用已授予ADVISOR权限的用户帐户hr进行调整。

  • 您想要调整在“ 创建SQL调整任务 ”中创建的STA_SPECIFIC_EMP_TASK 。

  • 您希望将SQL调整任务可以运行的最长时间更改为300秒。

要配置SQL调优任务：

  1. 使用适当的权限将SQL * Plus连接到数据库，然后运行DBMS_SQLTUNE.SET_TUNING_TASK_PARAMETER函数。

    例如，执行以下PL / SQL程序将调整任务的时间限制更改为300秒：

    BEGIN DBMS_SQLTUNE.SET_TUNING_TASK_PARAMETER（task_name =>'STA_SPECIFIC_EMP_TASK'，参数=>'TIME_LIMIT'，值=> 300）;结束; /

  2. （可选）验证任务参数是否已更改。

    以下示例查询任务STA_SPECIFIC_EMP_TASK中所有已使用参数的值：

    COL PARAMETER_NAME FORMAT a25 COL VALUE FORMAT a15 SELECT PARAMETER_NAME，PARAMETER_VALUE为“VALUE”来自USER_ADVISOR_PARAMETERS，其中TASK_NAME ='STA_SPECIFIC_EMP_TASK'和PARAMETER_VALUE！='未使用'订阅PARAMETER_NAME;

    示例输出显示如下：

    PARAMETER_NAME VALUE ------------------------- --------------- DAYS_TO_EXPIRE 30 DEFAULT_EXECUTION_TYPE TUNE SQL EXECUTION_DAYS_TO_EXPIRE UNLIMITED CHINESEING信息模式综合SQL_LIMIT -1 SQL_PERCENTAGE 1 TARGET_OBJECTS 1 TEST_EXECUTE AUTO TIME_LIMIT 300

从Oracle Database 12c第2版（12.2）开始，您可以通过在database_link_to参数中指定数据库链接来调整备用数据库工作负载。出于安全原因，Oracle建议使用专用数据库链接。该链接必须由SYS拥有，并由特权用户访问。Oracle数据库包括名为SYS$UMF的默认特权用户。
