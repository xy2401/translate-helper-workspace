Oracle RAC数据库的连接故障转移

Oracle RAC通过在多个节点上配置多个侦听器来管理同一数据库服务的客户端连接请求，从而为节点VIP地址提供故障转移。

当客户端使用SCAN发出连接请求时，三个SCAN地址将返回给客户端。如果第一个地址失败，则对SCAN的连接请求将故障转移到下一个地址。使用多个地址允许客户端连接到数据库的实例，即使初始实例已失败。

如果节点发生故障，则连接到VIP的服务将透明地重新定位到正常运行的节点，从而能够通过VIP快速向客户端发送故障通知。如果应用程序和客户端配置了透明应用程序故障转移选项，则客户端将重新连接到正常运行的节点。

2.2 Java应用程序的执行控制

在Java 2平台标准版（J2SE）环境中，您可以开发Java 应用程序使用main()方法，在运行类时由解释程序调用。在命令行中输入以下命令时，将调用main()方法：

java classname

此命令启动Java解释器，并将所需的类（即classname指定的classname ）传递给Java解释器。解释器加载类并通过调用main()开始运行应用程序。但是，数据库中的Java应用程序不是通过调用main()方法启动的。

在数据库中加载Java应用程序后，可以通过调用加载的类中的任何static方法来运行它。必须先发布一个或多个类，然后才能运行它们。在Oracle数据库中，Java应用程序的入口点不被假定为main() 。而是在运行Java应用程序时，在加载的类中指定方法名称作为入口点。

例如，在标准Java环境中，您可以通过运行以下命令在服务器上启动Java对象：

java myprogram

其中， myprogram是包含main()方法的类的名称。在myprogram ， main()立即调用mymethod()来处理传入的信息。

在Oracle数据库中，将myprogram.class文件加载到数据库中将mymethod()作为入口点发布。然后，客户端或触发器可以显式调用mymethod() 。

关于网格命名服务（GNS）虚拟IP地址

GNS虚拟IP地址是在域名系统（DNS）中配置的静态IP地址。

DNS将查询委托给GNS虚拟IP地址，GNS守护程序响应该地址的传入名称解析请求。在子域内，GNS使群集能够在群集中添加和删除节点时动态映射主机名和IP地址，而无需在DNS中进行其他主机配置。

要启用GNS，您必须让网络管理员为分配给群集的子域提供一组IP地址（例如， grid.example.com ），并将该子域的DNS请求委派给群集的GNS虚拟IP地址，GNS服务。DHCP为群集提供一组IP地址。DHCP必须在群集的公共网络上可用。

检查TCP网络缓冲区大小，确保它足以满足服务器的速度。

默认情况下，TCP的网络缓冲区大小设置为1 MB，UDP的网络缓冲区大小设置为2 MB。 TCP缓冲区大小可以设置文件传输限制，这可能会对Direct NFS客户端用户的性能产生负面影响。

要检查Oracle Solaris 11上的当前TCP缓冲区大小：

#ipad show-prop -p max_buf tcp

Oracle建议您根据服务器的链接速度设置值。例如：

在Oracle Solaris 11上：

#ipad set-prop -p max_buf = 1048576 tcp

此外，请检查TCP发送窗口大小和TCP接收窗口大小，以确保它们足以满足服务器的速度。

要在Oracle Solaris 11上检查当前TCP发送窗口大小和TCP接收窗口大小：

#ipad show-prop -p send_buf tcp #ipadm show-prop -p recv_buf tcp

Oracle建议您根据服务器的链接速度设置值。例如：

在Oracle Solaris 11上：

#ipad set-prop -p send_buf = 1056768 tcp #ipadm set-prop -p recv_buf = 1056768 tcp

30.5了解反向转换

了解为什么需要反向转换。

反向转换可确保模型返回的信息以与用于训练模型的数据格式类似或相同的格式表示。内部转换在模型细节和评分结果中相反。

模型使用的某些属性对应于构建数据中的列。但是，由于算法特有的逻辑，嵌套数据和转换，某些属性不对应于列。

例如，训练数据中的嵌套列不会被模型解释为属性。在模型构建期间，Oracle Data Mining会爆炸嵌套列，并且每一行（属性名称/值对）都将成为属性。

某些算法（例如支持向量机（SVM）和广义线性模型（GLM））仅对数字属性进行操作。构建数据中的任何非数字列都会分解为二进制属性，一个用于列中的每个不同值（SVM）。GLM不会为原始列中的最常见值生成新属性。仅当案例的列值等于与二进制属性关联的值时，才将这些二进制属性设置为1。

生成系数的算法在结果的可解释性方面提出了挑战。示例是SVM和非负矩阵分解（NMF）。这些算法产生与变换后的属性结合使用的系数。系数与变换尺度上的数据相关，而不是原始数据尺度。

由于所有这些原因，模型详细信息中列出的属性与用于训练模型的数据列类似。但是，经过嵌入式转换的属性（无论是由自动数据准备（ADP）还是由用户指定的转换列表启动）都会在预转换状态下显示在模型详细信息中，尽可能接近原始列值。虽然属性在模型使用时会被转换，但它们在模型详细信息中以可由用户解释的形式显示。

2.7缩短的班级名称

每个Java源，类和资源都存储在服务器中自己的架构对象中。架构对象的名称源自完全限定名称，该名称包括相关路径或包信息。点用斜线代替。

但是，架构对象名称最多只能包含30个字符，并且所有字符必须合法且可转换为数据库字符集中的字符。如果任何完全限定名称超过30个字符或包含非法或不可转换的字符，则Oracle数据库会将其转换为短名称或别名，以用作架构对象的名称。Oracle数据库会跟踪名称以及如何在它们之间进行转换。如果完全限定名称不超过30个字符且没有非法或不可转换的字符，则将其用作架构对象名称。

4.1关于转换

了解如何使用自动数据准备（ADP）和嵌入式数据转换来转换数据。

转换是一种SQL表达式，用于修改一列或多列中的数据。数据通常必须经过某些转换才能用于构建模型。许多数据挖掘算法都有特定的转换要求。在对数据进行评分之前，必须以与转换培训数据相同的方式对其进行转换。

Oracle Data Mining支持自动数据准备（ADP），它自动实现算法所需的转换。转换嵌入在模型中，并在应用模型时自动执行。

如果需要其他转换，则可以将它们指定为SQL表达式，并在创建模型时将它们作为输入提供。这些转换与ADP一样嵌入到模型中。

通过自动和嵌入式数据转换，您可以处理大部分数据准备工作。您只需几个步骤即可创建模型并对多个数据集进行评分：

1.8专用模式会话的内存模型

自Oracle Database 10g以来，Oracle JVM为通过专用服务器连接到数据库的会话提供了新的内存模型。与Oracle相关的基本内存结构包括：

  • 系统全局区域（SGA）

    SGA是一组共享内存结构，称为SGA组件，包含一个Oracle数据库实例的数据和控制信息。SGA由所有服务器和后台进程共享。存储在SGA中的数据的示例包括高速缓存的数据块和共享的SQL区域。

  • 全球计划领域（PGA）

    PGA是包含服务器进程的数据和控制信息的内存区域。它是由Oracle在启动服务器进程时创建的非共享内存。访问PGA是服务器进程独有的。每个服务器进程都有一个PGA。后台进程还分配自己的PGA。为连接到Oracle实例的所有后台和服务器进程分配的总PGA内存称为聚合PGA 。

管理内存的最简单方法是允许数据库自动管理和调整内存。要做到这一点，你只设置一个目标内存大小初始化参数（ MEMORY_TARGET ）和一个最大内存大小初始化参数（ MEMORY_MAX_TARGET ），在大多数平台上。然后数据库调整到目标内存大小，根据需要在SGA和聚合PGA之间重新分配内存。由于目标内存初始化参数是动态的，因此您可以随时更改目标内存大小，而无需重新启动数据库。最大内存大小作为上限，以便您不会意外地将目标内存大小设置得太高。由于某些SGA组件不能轻易缩小或必须保持最小大小，因此数据库还会阻止您将目标内存大小设置得太低。

关于管理端口

在安装过程中，Oracle Universal Installer会从一组默认端口号为组件分配端口号。

许多Oracle数据库组件和服务都使用端口。作为管理员，了解这些服务使用的端口号非常重要，并确保主机上的两个服务不使用相同的端口号。输入以下命令以标识计算机上当前使用的端口：

$ / bin / netstat -a

大多数端口号在安装期间分配。每个组件和服务都有一个分配的端口范围，这是Oracle数据库在分配端口时尝试使用的端口号集。Oracle数据库以范围中的最小数字开头，并执行以下检查：

  • 是否在主机上安装了另一个Oracle数据库的端口？

    安装可能在当时上升或下降。Oracle数据库仍然可以检测是否使用了该端口。

  • 当前正在运行的进程使用的端口是什么？

    这可以是主机上的任何进程，甚至是非Oracle数据库进程。

  • 端口是否在/etc/services文件中列出？

如果上述任何问题的答案为是，则Oracle数据库移动到分配的端口范围中的下一个最高端口，并继续检查，直到找到空闲端口。
