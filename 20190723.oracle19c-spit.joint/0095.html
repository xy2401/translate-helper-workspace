数据库对象名称和限定符

某些数据库对象由您可以或必须命名的部分组成，例如表或视图中的列，索引和表分区和子分区，表上的完整性约束以及存储在包中的对象，包括过程和存储的功能。本节提供：

  • 命名数据库对象和数据库对象位置限定符的规则

  • 命名数据库对象和限定符的准则

注意：

对于隐式生成的数据库对象和子对象，Oracle使用以“ SYS_ ”开头的系统生成的名称，对于某些Oracle提供的对象，使用以“ ORA_ ”开头的名称。Oracle不鼓励您在显式提供给数据库对象和子对象的名称中使用这些前缀，以避免名称解析可能发生冲突。

数据库对象命名规则

每个数据库对象都有一个名称。在SQL语句中，你代表了带引号的标识符或带引号的标识符的对象的名称。

  • 带引号的标识符以双引号（“）开头和结尾。如果使用带引号的标识符命名架构对象，则每次引用该对象时都必须使用双引号。

  • 未加引号的标识符不包含任何标点符号。

您可以使用带引号或非带引号的标识符来命名任何数据库对象。但是，数据库名称，全局数据库名称，数据库链接名称，磁盘组名称和可插入数据库（PDB）名称始终不区分大小写，并以大写形式存储。如果将此类名称指定为带引号的标识符，则会以静默方式忽略引号。

注意：

Oracle建议不要对数据库对象名称使用带引号的标识符。SQL * Plus接受这些带引号的标识符，但在使用管理数据库对象的其他工具时它们可能无效。

除非另有说明，否则以下规则列表适用于引用和非引用标识符：

  1. 标识符名称的最大长度取决于COMPATIBLE初始化参数的值。

    • 如果COMPATIBLE设置为12.2或更高的值，则名称必须为1到128个字节长，但有以下例外：

      • 数据库的名称限制为8个字节。

      • 磁盘组，可插拔数据库（PDB），回滚段，表空间和表空间集的名称限制为30个字节。

      如果标识符包含由句点分隔的多个部分，则每个属性最长可达128个字节。每个句点分隔符以及任何周围的双引号都计为一个字节。例如，假设您标识了这样的列：

      “ schema ”。“ table ”。“ column ”

      模式名称可以是128个字节，表名称可以是128个字节，列名称可以是128个字节。每个引号和句点都是单字节字符，因此本例中标识符的总长度最多为392个字节。

    • 如果COMPATIBLE设置为低于12.2的值，则名称长度必须为1到30个字节，但以下情况除外：

      • 数据库的名称限制为8个字节。

      • 数据库链接的名称可以长达128个字节。

      如果标识符包含由句点分隔的多个部分，则每个属性最长可达30个字节。每个句点分隔符以及任何周围的双引号都计为一个字节。例如，假设您标识了这样的列：

      “ schema ”。“ table ”。“ column ”

      模式名称可以是30个字节，表名称可以是30个字节，列名称可以是30个字节。每个引号和句点都是单字节字符，因此本例中标识符的总长度最多为98个字节。

  2. 不带引号的标识符不能是Oracle SQL保留字。带引号的标识符可以是保留字，但不建议这样做。

    根据您计划用于访问数据库对象的Oracle产品，名称可能会受到其他特定于产品的保留字的进一步限制。

    注意：

    保留字ROWID是此规则的一个例外。您不能使用引号或非引号的大写单词ROWID作为列名。但是，您可以将大写单词用作不是列名的带引号的标识符，并且可以将带有一个或多个小写字母的单词（例如，“ Rowid ”或“ rowid ”）用作任何带引号的标识符，包括列名。

    数据库字符集大写规则可以在查看为某种自然语言时产生不正确的结果。例如，德语中使用的小写字母s（“ß”）根据数据库字符集大写规则不具有大写形式。当标识符转换为大写时，它不会被修改，而德语中的预期大写形式是大写字母S（“SS”）的两个字符的序列。类似地，根据数据库字符集大写字母规则，小写字母i的大写形式是大写字母I.但是，土耳其语和阿塞拜疆语中的预期大写形式是带有上方点的大写字母I.

    数据库字符集大写规则确保在会话的任何语言配置中解释标识符相同。如果您希望标识符以某种自然语言正确显示，则可以引用它以保留小写形式，或者您可以在使用该标识符时使用语言正确的大写形式。

  11. 同一表或视图中的列不能具有相同的名称。但是，不同表或视图中的列可以具有相同的名称。

模式对象命名指南

以下是命名对象及其部件的一些有用指南：

  • 使用完整的，描述性的，可发音的名称（或众所周知的缩写）。

  • 使用一致的命名规则。

  • 使用相同的名称来跨表描述相同的实体或属性。

命名对象时，要平衡保持名称简短易用的目标，以使名称尽可能具有描述性。如有疑问，请选择更具描述性的名称，因为数据库中的对象可能会在一段时间内被许多人使用。十年后你的对手可能难以理解一个名为pmdd而不是payment_due_date的表格列。

使用一致的命名规则可帮助用户了解每个表在应用程序中播放的部分。一个这样的规则可能是使用fin_开始属于FINANCE应用程序的所有表的名称。

使用相同的名称来描述表中的相同内容。例如，示例employees和departments表的部门编号列都命名为department_id 。

Oracle®Spatialand Graph Map可视化JavaScript API参考（版本19.1）

这是基于Oracle Maps HTML5的JavaScript API的在线文档。
Oracle Maps JavaScript API版本2（V2）利用了现代浏览器的功能。它的一些功能包括：

  • 内置支持各种第三方地图图块服务，例如maps.oracle.com，诺基亚地图，Bing地图，OpenStreet地图和其他地图服务提供商。
  • 丰富的客户端渲染地理空间数据，通过渲染样式和效果（如渐变，动画和阴影）实时应用。
  • 自动聚集大量点和客户端热图生成。
  • 一组内置的地图控件和工具，包括可自定义的导航栏和信息窗口，可配置的图层控件以及红线和距离测量工具。

V2 API 不与现有的Oracle Maps JavaScript V1 API应用程序向后兼容。API具有以下顶级类和子包，所有这些都在命名空间OM中 ：

  • Map类是API的主要类。
  • Feature类表示单个地理要素 （或V1中已知的FOI）。
  • MapContext类是封装一些基本上下文信息的顶级类，例如当前地图中心点和缩放级别。它通常被传递给事件监听器。
  • 控件包包含所有地图控件，例如导航栏和概览图。
  • 事件包包含所有地图和图层事件类。
  • 过滤器包包含用于选择或子集化所显示的矢量图层要素的所有客户端过滤器（空间或关系）。
  • 几何包包含各种几何类。
  • 图层包包含各种图块和矢量图层类。切片图层类包括访问一些在线地图服务，如Oracle，Nokia，Bing和OpenStreetMap。矢量图层是交互式要素图层，对应于V1的MVThemeBasedFOI和MVFOI类。
  • infowindow包中包含可自定义的信息窗口及其样式。
  • 样式包包含适用于客户端矢量数据的样式。它还包括视觉效果，如动画，渐变和阴影。
  • 该工具包包含各种地图工具，例如用于距离测量，红线和几何图形。
  • Universe包中包含内置或预定义的地图Universe 。地图Universe定义地图内容的边界框和缩放级别定义集。它类似于V1 API中的切片层配置。
  • util包包含各种实用程序类。
  • visualfilter包为各种视觉效果提供了界面，例如渐变和阴影。
  • gv包中包含配置选项的全局变量，例如资源路径或请求的协议（http或https）。

22 JSON的PL / SQL对象类型概述

PL / SQL对象类型允许细粒度编程构造和内存中JSON数据的操作。您可以对其进行内省，修改并将其序列化为文本JSON数据。

主要的PL / SQL JSON对象类型是JSON_ELEMENT_T ， JSON_OBJECT_T ， JSON_ARRAY_T和JSON_SCALAR_T 。另一个较少使用的对象类型是JSON_KEY_LIST ，它是VARCHAR2(4000)的varray。对象类型也称为抽象数据类型（ADT）。

这些JSON对象类型提供存储在数据库中的JSON数据的内存，分层（树状）编程表示。脚1

您可以使用对象类型以编程方式操作内存中的JSON数据，以执行以下操作：

  • 检查现有JSON数据的结构，类型或值。例如，检查给定对象字段的值是否满足某些条件。

  • 转换现有的JSON数据。例如，转换地址或电话号码格式以遵循特定约定。

  • 使用与数据表示的任何特征匹配的编程规则创建JSON数据。例如，如果要表示为JSON对象的产品是易燃的，则包括表示安全信息的字段。

PL / SQL对象类型实例是瞬态的 。要永久存储它们包含的信息，必须将它们序列化为VARCHAR2或LOB数据，然后可以将这些数据存储在数据库表中或编组到数据库客户机（如Java Database Connectivity（JDBC））。

您可以通过解析JSON文本或零碎的方式在内存中构建一个对象类型实例，从空对象或数组实例开始，并向其添加对象成员或数组元素。

未使用的对象类型实例会自动进行垃圾回收;你不能，也不必，释放你不再需要的实例所使用的内存。

通常，在构造PL / SQL对象类型实例并且可能以各种方式以编程方式使用它之后，将序列化为数据类型为VARCHAR2 ， CLOB或BLOB的实例。也就是说，您将内存中JSON数据的瞬态表示转换为数据库中的持久表示。（或者，您可以仅将其序列化为要打印的文本。）

JSON对象类型之间的关系

类型JSON_ELEMENT_T是其他JSON对象类型的超类型：它们中的每一个都将其扩展为子类型。子类型JSON_OBJECT_T和JSON_ARRAY_T分别用于JSON对象和数组。子类型JSON_SCALAR_T用于标量JSON值：字符串，数字，布尔值true和false以及值null 。

您只能通过解析JSON文本来构造JSON_ELEMENT_T类型的实例。解析创建一个JSON_ELEMENT_T实例，它是JSON数据的内存中表示。您不能构造JSON_ELEMENT_T类型的空实例或键入JSON_SCALAR_T 。

类型JSON_OBJECT_T和JSON_ARRAY_T各自具有与该类型相同名称的构造函数，您可以使用它来构造该类型的实例：JSON对象或数组的空（In-Memory）表示。然后，您可以根据需要填充此对象或数组，添加由PL / SQL对象类型实例表示的对象成员或数组元素。

您可以使用PL / SQL函数treat JSON_ELEMENT_T的实例JSON_ELEMENT_T转换为子类型实例。例如， treat(elt AS JSON_OBJECT_T)将实例elt强制转换为JSON对象（ JSON_OBJECT_T实例）。

解析函数

静态函数parse接受VARCHAR2 ， CLOB或BLOB类型的实例作为参数，它将其解析为JSON文本以返回类型为JSON_ELEMENT_T ， JSON_OBJECT_T或JSON_ARRAY_T 。

序列化方法

解析接受输入JSON数据作为文本并返回PL / SQL JSON对象类型的实例。序列化基本上相反：您将它应用于JSON数据的PL / SQL对象表示，并返回该对象的文本表示。序列化方法的名称以前缀to_开头。例如，方法to_string()返回您应用它的JSON对象类型实例的字符串（ VARCHAR2 ）表示。

大多数序列化方法都是成员函数。但是，对于作为CLOB或BLOB实例的序列化，有两种形式的方法：成员函数和成员过程。成员函数不接受任何参数。它创建一个临时LOB作为序列化目标。所述构件过程接受一个LOB IN OUT参数（ CLOB为方法实例to_clob ， BLOB为方法to_blob ）。因此，您可以将要用于序列化表示的LOB（可能为空）传递给它。

Getter和Setter方法

类型JSON_OBJECT_T和JSON_ARRAY_T具有getter和setter方法，它们分别获取和更新给定对象字段或给定数组元素位置的值。

getter方法有两种：

  • 方法get()返回对应用它的原始对象的引用，作为JSON_ELEMENT_T类型的实例。也就是说，应用它的对象通过引用传递 ：如果然后修改返回的JSON_ELEMENT_T实例，则修改将应用于您应用get()的原始对象。

  • 名称前缀为get_ getter方法返回在应用它们的对象或数组中定位的任何数据的副本 。该数据按值传递 ，而不是参考。

    例如，如果将方法JSON_OBJECT_T get_string()应用于JSON_OBJECT_T实例，并将给定字段作为参数传递，则会返回该字段值的字符串副本。如果将JSON_ARRAY_T get_string()应用于JSON_ARRAY_T实例，将给定元素位置作为参数传递，则会返回该数组中该位置的字符串副本。

与序列化方法一样，大多数getter方法都是成员函数。但是get_clob()和get_blob()方法返回给定对象字段的值或给定数组位置的元素作为CLOB或BLOB实例，有两种形式（如序列化方法to_clob()和to_blob() ） ：成员函数和成员过程 。成员函数不接受目标对象字段或数组位置以外的任何参数。它创建并返回一个临时LOB实例。所述构件过程还接受LOB IN OUT参数（ CLOB为get_clob ， BLOB为get_blob ）。因此，您可以将（可能为空的）LOB实例传递给它。

setter方法是put() ， put_null()和（仅适用于JSON_ARRAY_T ） append() 。它们更新应用它们的对象或数组实例，设置目标对象字段或数组元素的值。注意：setter方法修改现有实例 ，而不是返回它的修改副本。

方法append()在数组实例的末尾添加一个新元素。方法put_null()将对象字段或数组元素值设置为JSON null 。

方法put()需要第二个参数（除了对象字段名称或数组元素位置），这是要设置的新值。对于数组， put()也接受可选的第三个参数OVERWRITE 。这是一个BOOLEAN值（默认为FALSE ），表示是否替换给定位置的现有值 。

  • 如果对象已经具有相同名称的字段，则put() 将该值替换为新值。

  • 如果数组在给定位置已经有一个元素，则默认情况下， put()将该元素和任何连续元素向前移动（将它们的位置递增1），以便为放置在给定位置的新元素腾出空间。但是如果存在可选参数OVERWRITE并且为TRUE ，那么给定位置的现有元素将被新元素替换 。

如果您分配多个磁盘通道并希望每个通道写入其他目录，则FORMAT参数很有用。CONFIGURE CHANNEL或ALLOCATE CHANNEL指定的FORMAT参数在语义上等同于BACKUP命令中指定的FORMAT参数（而不是forRecoveryOfSpec中的DATAFILECOPY FORMAT参数）。如果在BACKUP命令中指定FORMAT参数，则它将覆盖CONFIGURE CHANNEL或ALLOCATE CHANNEL指定的FORMAT参数。

如果未指定FORMAT ，则RMAN默认使用%U ，这保证了备份文件名称的唯一名称。如果配置了快速恢复区域，则RMAN会在默认磁盘位置创建备份文件。否则，默认磁盘位置是特定于操作系统的（例如， ?Solaris上的/dbs ）。

您最多可以指定四个FORMAT字符串。仅当BACKUP COPIES ， SET BACKUP COPIES或CONFIGURE ...时，RMAN才使用第二个，第三个和第四个值CONFIGURE ...BACKUP COPIES生效。在选择用于每个备份片段的格式时，RMAN使用副本1的第一个格式值，副本2的第二个格式值，依此类推。如果格式值的数量超过副本数，则不使用额外的格式。如果格式值的数量小于副本数，则RMAN将重用格式值，从第一个开始。

由于通道对应于目标数据库上的服务器会话，因此FORMAT字符串必须使用目标主机的约定，而不是客户端主机。例如，如果RMAN客户端在Windows主机上运行且目标数据库在Linux主机上运行，则FORMAT字符串必须符合Linux文件系统或原始设备的命名约定。

1.1什么是数据挖掘？

了解数据挖掘。

数据挖掘是一种发现以前未知的数据关系的技术。数据挖掘是自动搜索大型数据库以发现超出简单分析的模式和趋势的实践。数据挖掘使用复杂的数学算法来分割数据，并根据过去的事件预测未来事件的可能性。数据挖掘也称为数据中的知识发现（KDD）。

数据挖掘的关键属性是：

  • 自动发现模式

  • 预测可能的结果

  • 创建可操作的信息

  • 专注于大型数据集和数据库

数据挖掘可以回答无法通过简单查询和报告技术解决的问题。

1.1.1自动发现

数据挖掘由使用算法作用于一组数据的模型执行。数据挖掘模型可用于挖掘构建它们的数据，但大多数类型的模型可推广到新数据。将模型应用于新数据的过程称为评分 。

1.1.2预测

许多形式的数据挖掘都是预测性的。例如，模型可以根据教育和其他人口统计因素预测收入。预测具有相关的概率（这种预测有多大可能是真的？）。预测概率也称为置信度 （我对这种预测有多大信心？）。

某些形式的预测数据挖掘会生成规则 ，这些规则意味着给定的结果。例如，规则可以指定具有学士学位并且居住在某个社区的人可能具有大于区域平均水平的收入。规则有相关的支持 （人口中有多少百分比满足规则？）。

1.1.3分组

其他形式的数据挖掘识别数据中的自然分组。例如，模型可以识别具有指定范围内的收入，具有良好驾驶记录并且每年租赁新车的人口部分。

1.1.4可操作的信息

数据挖掘可以从大量数据中获取可操作的信息。例如，城镇规划者可能会使用一种模型来预测基于人口统计的收入，以制定低收入住房计划。汽车租赁代理商可能会使用识别客户细分的模型来设计针对高价值客户的促销。

1.1.5数据挖掘和统计

数据挖掘和数据挖掘之间存在很多重叠统计。事实上，数据挖掘中使用的大多数技术都可以放在统计框架中。但是，数据挖掘技术与传统统计技术不同。

统计模型通常对数据做出强有力的假设，并且基于这些假设，他们对结果做出强有力的陈述。但是，如果假设存在缺陷，那么模型的有效性就会变得有问题。相比之下，数据挖掘中使用的机器学习方法通常会对数据做出微弱的假设。因此，数据挖掘通常不能对结果做出如此强烈的陈述。然而，无论数据如何，数据挖掘都可以产生非常好的结果。

通常，传统的统计方法需要大量的用户交互才能验证模型的正确性。结果，统计方法可能难以自动化。统计方法依赖于检验假设或基于较大人口的较小的代表性样本找到相关性。

数据挖掘需要较少的用户交互和较少的数据知识。用户不需要按摩数据以保证方法对给定数据集有效。数据挖掘技术比传统统计技术更容易自动化。

1.1.6数据挖掘和OLAP

在线分析处理（OLAP）可以定义为快速分析多维数据。OLAP和数据挖掘是不同但互补的活动。

OLAP支持数据汇总，成本分配，时间序列分析和假设分析等活动。但是，大多数OLAP系统都没有归纳推理能力超出了对时间序列预测的支持。归纳推理是从具体例子中得出一般结论的过程，是数据挖掘的一个特征。归纳推理也称为计算学习。

OLAP系统提供数据的多维视图，包括对层次结构的完全支持。这种数据视图是分析企业和组织的自然方式。

数据挖掘和OLAP可以通过多种方式集成。OLAP可用于分析不同粒度级别的数据挖掘结果。数据挖掘可以帮助您构建更有趣和有用的多维数据集。例如，可以将预测数据挖掘的结果作为自定义度量添加到多维数据集。这些措施可以为每个客户提供诸如“可能违约”或“可能购买”的信息。然后，OLAP处理可以聚合和汇总概率。

1.1.7数据挖掘和数据仓库

数据可以是否存储在平面文件，电子表格，数据库表或其他一些存储格式中。数据的重要标准不是存储格式，而是它对要解决的问题的适用性。

适当的数据清理和准备对于数据挖掘非常重要，数据仓库可以促进这些活动。但是，如果数据仓库不包含解决问题所需的数据，则无用。

1.2数据挖掘可以做什么和不做什么？

数据挖掘是一种功能强大的工具，可以帮助您查找数据中的模式和关系。但数据挖掘本身并不起作用。它不会消除了解您的业务，了解您的数据或理解分析方法的需要。数据挖掘可以发现数据中的隐藏信息，但它无法告诉您组织信息的价值。

随着时间的推移，您可能已经意识到了重要的模式。除了寻找通过简单观察无法立即辨别的新模式之外，数据挖掘还可以确认或限定此类经验观察。

重要的是要记住，通过数据挖掘发现的预测关系不是因果关系。例如，数据挖掘可能会确定订阅某些杂志的收入在50,000美元到65,000美元之间的男性可能会购买给定的产品。您可以使用此信息来帮助您制定营销策略。但是，您不能假设通过数据挖掘识别的人口购买产品， 因为他们属于这一群体。

数据挖掘产生概率，而非精确答案。重要的是要记住罕见的事件可能发生;他们只是不经常发生。

1.2.1提出正确的问题

数据挖掘不会在没有指导的情况下自动发现信您通过数据挖掘找到的模式非常不同，具体取决于您如何制定问题。

要获得有意义的结果，您必须学习如何提出正确的问题。例如，您可能会尝试找到过去曾对您的请求做出回应的人的特征，而不是尝试学习如何“改进对直接邮件征集的响应”。

1.2.2了解您的数据

确保有意义的数据挖掘结果，您必须了解您的数据。数据挖掘算法通常对数据的特定特征敏感： 异常值（与数据库中的典型值非常不同的数据值），不相关的列，一起变化的列（例如年龄和出生日期），数据编码以及您选择包含或排除的数据。Oracle Data Mining可以自动执行算法所需的大部分数据准备。但是一些数据准备通常特定于域或数据挖掘问题。无论如何，您需要了解用于构建模型的数据，以便在应用模型时正确解释结果。

1.3.1问题定义

数据挖掘项目的初始阶段侧重于理解项目目标和要求。从业务角度指定问题后，您可以将其制定为数据挖掘问题并制定初步实施计划。

例如，您的业务问题可能是：“我如何向客户销售更多产品？“您可能会将此转化为数据挖掘问题，例如：”哪些客户最有可能购买该产品？“预测谁最有可能购买产品的模型必须建立在描述过去购买过产品的客户的数据之上。在构建模型之前，您必须汇总可能包含已购买产品的客户与未购买产品的客户之间关系的数据。顾客属性可能包括年龄，孩子数，居住年限，所有者/租房者等。

1.3.2数据收集，准备和特征工程

了解如何收集数据，准备数据和工程师功能以解决业务问题。

数据理解阶段涉及数据收集和探索。当您仔细查看数据时，您可以确定它解决业务问题的程度。您决定删除一些数据或添加其他数据。这也是识别数据质量问题和扫描数据模式的时候。

数据准备阶段涵盖了创建所涉及的所有任务用于构建模型的表或视图。数据准备任务可能会多次执行，而不是按照任何规定的顺序执行。任务可以包括列选择和视图创建，以及数据清理和转换。例如，您可以将DATE_OF_BIRTH列转换为AGE ;您可以在INCOME空的情况下插入中位数收入。

此外，您可以添加新的计算属性，以便更接近通常称为特征工程的数据表面的信息。例如，您可以创建一个新属性，而不是使用购买金额：“购买的次数金额在12个月的时间内超过500美元。”经常进行大额购买的客户也可能与回复或不回应优惠的客户有关。

周密的数据准备和创建捕获领域知识的新“工程特性”可以显着改善通过数据挖掘发现的信息。使数据分析师能够在Oracle数据库中执行这些数据汇编，数据准备，数据转换和功能工程是Oracle的一个重要区别。

注意：

Oracle Data Mining支持自动数据准备（ADP）， 这极大地简化了数据准备过程。

10.1关于代理验证

代理身份验证是使用中间层进行用户身份验证的过程。您可以使用以下三种形式的代理身份验证，以安全的方式设计中间层服务器以代理客户端：

  • 中间层服务器使用数据库服务器和客户端对自身进行身份验证。在这种情况下，应用程序用户或其他应用程序使用中间层服务器对自身进行身份验证。客户端身份可以一直维护到数据库。

  • 客户端（即数据库用户）未经中间层服务器验证。客户端的标识和数据库密码通过中间层服务器传递到数据库服务器以进行身份验证。

  • 客户端（即全局用户）由中间层服务器进行身份验证，并通过中间层传递可分辨名称（DN）或证书以检索客户端的用户名。

    注意：

    可以审核中间层服务器代表客户端执行的操作。

在所有情况下，管理员必须授权中间层服务器代理客户端，即代表客户端行事。假设，中间层服务器最初以用户HR身份连接到数据库并以用户jeff激活代理连接，然后发出以下语句以授权中间层服务器代理客户端：

ALTER USER jeff GRANT通过HR连接;

你也可以：

  • 指定在作为客户端连接时允许中间层激活的角色。例如，

    CREATE ROLE role1; GRANT SELECT ON员工TO role1; ALTER USER jeff GRANT通过HR ROLE角色1连接;

    role子句仅限制对角色列表中提到的那些数据库对象的访问。角色列表可以为空。

  • 通过查询PROXY_USERS数据字典视图，查找当前有权通过中间层连接的用户。

  • 使用ALTER USER语句的REVOKE CONNECT THROUGH子句禁止代理连接。

注意：

在代理身份验证的情况下，与数据库的JDBC连接在身份验证期间创建数据库会话，然后可以在连接的生命周期内创建其他会话。

您需要使用oracle.jdbc.存在的不同字段和方法oracle.jdbc.OracleConnection接口设置不同类型的代理连接。

10.5高速缓存代理连接

可以缓存代理连接（如标准连接）。缓存代理连接可提高性能。要缓存代理连接，您需要在启用缓存的OracleDataSource对象上使用其中一个getConnection方法创建连接。

可以使用连接高速缓存的连接属性功能将代理连接高速缓存在连接高速缓存中。连接属性是用户定义的名称/值对，并在将连接返回到连接缓存以供重用之前帮助标记连接。检索标记连接时，可以直接使用它，而无需进行往返创建或关闭代理会话。Universal Connection Pool支持缓存任何用户/密码验证的连接。因此，可以缓存和检索任何用户身份验证的代理连接。

建议不应用连接属性来关闭代理连接。如果在未应用连接属性的情况下关闭代理连接，则会将连接返回到连接缓存以供重用，但无法检索。连接缓存机制不记得或重置会话状态。

通过直接关闭连接，可以从连接缓存中删除代理连接。

在SQL中，面向对象编程基于对象类型，对象类型是用户定义的复合数据类型，它封装了数据结构以及操作数据所需的函数和过程。形成数据结构的变量称为ttributes。表征对象类型行为的函数和过程称为方法，可以用Java编写。

与包一样，对象类型有两部分：规范和身体。规范是应用程序的接口，并声明数据结构，这是一组属性，以及操作数据所需的操作或方法。正文通过定义PL / SQL子程序体或调用规范来实现规范。

如果规范仅声明属性或调用规范，则不需要正文。如果在Java中实现所有方法，则可以将其调用规范放在对象类型的规范部分中，并省略正文部分。

在SQL * Plus中，您可以使用以下语法以交互方式定义SQL对象类型：

CREATE [OR REPLACE] TYPE type_name [AUTHID {CURRENT_USER | DEFINER}] {IS | AS} OBJECT（attribute_name data_type [，attribute_name data_type] ...[{MAP | ORDER} MEMBER {function_spec | call_spec}，] [{会员| STATIC} {subprogram_spec | call_spec} [，{会员| STATIC} {subprogram_spec | call_spec}] ...]）; [CREATE [OR REPLACE] TYPE BODY type_name {IS | AS} {{MAP | ORDER} MEMBER function_body; | {会员| STATIC} {subprogram_body | call_spec};} [{会员| STATIC} {subprogram_body | call_spec}] ...结束;]

该 AUTHID子句确定该类型的所有成员方法是否以其定义者（ AUTHID DEFINER ）（默认值）或调用者（ AUTHID CURRENT_USER ）的权限运行。它还确定是否在定义者或调用者的模式中解析对模式对象的非限定引用。

  • 声明方法

7.5.1关于属性

在对象类型规范中，必须在任何方法之前声明所有属性。此外，您必须至少声明一个属性。可以声明的最大属性数为1000。方法是可选的。

与Java变量一样，您声明具有名称和数据类型的属性。该名称在对象类型中必须是唯一的，但可以在其他对象类型中重用。数据类型可以是任何SQL类型， LONG ， LONG RAW ， NCHAR ， NVARCHAR2 ， NCLOB ， ROWID和UROWID除外。

您不能使用赋值运算符或DEFAULT子句在其声明中初始化属性。此外，您不能对属性强加NOT NULL约束。但是，对象可以存储在可以施加约束的数据库表中。

7.5.2声明方法

声明属性后，您可以声明方法。MEMBER方法接受称为SELF的内置参数，该参数是对象类型的实例。无论是隐式还是显式声明，它始终是传递给MEMBER方法的第一个参数。在方法体中， SELF表示调用其方法的对象。在实例上调用MEMBER方法，如下所示：

instance_expression.method（）

无法接受或引用SELF STATIC方法在对象类型而不是其实例上调用，如下所示：

object_type_name.method（）

如果要调用非static Java方法，则必须在其调用规范中指定关键字MEMBER 。同样，如果要调用static Java方法，则必须在其调用规范中指定关键字STATIC 。

7.5.2.1地图和订购方法

SQL标量数据类型（例如CHAR ）的值具有预定义的顺序，因此可以与其他值进行比较。但是，对象类型的实例没有预定义的顺序。为了使它们按顺序排列，SQL调用用户定义的map方法。

SQL使用排序来计算布尔表达式，例如x > y ，并进行DISTINCT ， GROUP BY和ORDER BY子句隐含的比较。map方法返回对象在所有此类对象的排序中的相对位置。对象类型只能包含一个map方法，该方法必须是不带任何参数的函数，并且具有以下返回类型之一： DATE ， NUMBER或VARCHAR2 。

或者，您可以使用order方法提供SQL，该方法比较两个对象。order方法只接受两个参数：内置参数SELF和相同类型的另一个对象。如果o1和o2是对象，则比较（例如o1 > o2自动调用order方法。该方法返回负数，零或正数，表示SELF小于，等于或大于另一个参数。对象类型只能包含一个order方法，该方法必须是返回数字结果的函数。

您可以声明map方法或order方法，但不能同时声明两者。如果声明这些方法中的任何一个，那么您可以比较SQL和PL / SQL中的对象。但是，如果您不声明这两种方法，那么您只能在SQL中比较对象，并且仅用于相等或不等式。

注意：

如果相应属性的值相等，则相同类型的两个对象相等。

7.5.2.2构造方法

每个对象类型都有一个构造函数，它是一个系统定义的函数，其名称与对象类型相同。构造函数初始化并返回该对象类型的实例。

Oracle数据库为每种对象类型生成默认构造函数。构造函数的形式参数与对象类型的属性匹配。也就是说，参数和属性以相同的顺序声明，并具有相同的名称和数据类型。SQL从不隐式调用构造函数。因此，您必须明确地调用它。只要允许函数调用，就允许构造函数调用。

注意：

要从SQL调用Java构造函数，必须在static方法中对其进行包装调用，并将相应的调用规范声明为对象类型的STATIC成员。

多个ASMB

鉴于+ ASM1已安装dg1但未安装dg2，并且+ ASM2已安装dg2但未安装dg1，则多个ASMB项目允许数据库通过同时连接到两个ASM实例来同时使用dg1和dg2。我们现在可以拥有ASMBn，而不仅仅是ASMB。

此功能允许DB使用多个磁盘组，即使给定的ASM实例没有安装所有磁盘组，也可以提高Real Application Clusters（RAC）堆栈的可用性。

奇偶校验保护文件

REDUNDANCY文件类型属性指定文件组的冗余。PARITY值指定冗余的单奇偶校验。奇偶校验设置适用于一次写入文件，例如存档日志和备份集。

当两个或三个ASM镜像用于与数据库备份操作相关联的文件时，会消耗大量空间。备份文件是一次写入文件，此功能允许保护奇偶校验而不是传统镜像。结果可节省大量空间。

零停机Oracle Grid Infrastructure补丁

零停机Oracle Grid Infrastructure Patching可以在不中断数据库操作的情况下修补Oracle Grid Infrastructure。修补程序以非滚动方式应用，一次修补一个节点，同时节点上的数据库实例保持运行。零停机Oracle Grid Infrastructure Patching支持具有两个或更多节点的集群上的Oracle Real Application Clusters（Oracle RAC）数据库。

零停机时间网格基础结构修补通过允许客户执行Oracle Grid Infrastructure的滚动补丁而不中断正在修补的节点上的数据库操作而不影响这些数据库实例的容量或性能，显着提高了数据库可用性。

使用Oracle Fleet补丁和配置实现零停机Oracle Grid Infrastructure补丁

零停机时间Oracle Grid Infrastructure Patching可以在不影响Oracle RAC数据库实例的情况下应用一次性Oracle Grid Infrastructure修补程序。使用Oracle Fleet Patching and Provisioning（Oracle FPP）在整个群集中以滚动方式应用修补程序。此功能适用于具有两个或更多节点的所有Oracle Real Application Cluster，但目前仅适用于一次性补丁（不是RU和RUR）。

使用Oracle Fleet Patching和Provisioning应用一次性Oracle Grid Infrastructure补丁，数据库实例停机时间为零，可以减少对Oracle RAC数据库实例的用户和服务中断的影响，从而减少对nil的影响。使用以前的数据库版本，您必须在应用Oracle Grid Infrastructure修补程序之前关闭数据库实例，这显然会影响企业操作。

9.5.5指定分类器的评估顺序

分类器生成布尔表达式，每次工作请求进入系统时都会对其进行评估。

评估为TRUE的第一个分类器确定该工作请求的Performance Class。要确保将工作请求放入正确的Performance Classes中，您必须小心指定评估分类器的顺序。

要设置分类器的评估顺序，请执行以下步骤：

  1. 启动“策略集编辑器”向导。
  2. 继续执行向导中的第三页，标题为“编辑策略集：分类器排序”。
  3. 使用分类器右侧的箭头键在列表中向上或向下移动分类器。首先评估列表顶部的Performance Classes的分类器。如果工作请求与该Performance Class的分类器不匹配，则继续评估列表中的下一个Performance Class，直到不再进行进一步的评估。如果工作请求与Performance Class的分类器匹配，则工作请求将与该Performance Class相关联，并且评估将停止。

    为了正确分类工作请求，您应该将具有最严格分类器的Performance Classes放在列表顶部，并将具有最多松散分类器的Performance Classes放在列表底部附近。具有最常规分类器的Default_pc性能类应始终位于列表的底部。

  4. 然后，您将前进到“编辑策略集”向导的末尾，然后单击“ 提交策略集”以使更改成为永久更改。

闪回表

目的

如果出现人为或应用程序错误，请使用FLASHBACK TABLE语句恢复TABLE的早期状态。表可以闪回的过去时间取决于系统中的撤消数据量。此外，Oracle数据库无法通过任何更改表结构的DDL操作将表还原到早期状态。

注意：

Oracle强烈建议您将UNDO_MANAGEMENT初始化参数设置为AUTO （默认值），以自动撤消模式运行数据库。此外，将UNDO_RETENTION初始化参数设置为足够大的间隔，以包含您预期需要的最旧数据。有关更多信息，请参阅UNDO_MANAGEMENT和UNDO_RETENTION初始化参数的文档。

您无法回滚FLASHBACK TABLE语句。但是，您可以发出另一个FLASHBACK TABLE语句并指定当前时间之前的时间。因此，建议在发出FLASHBACK TABLE子句之前记录当前的SCN。

  • 有关使用FLASHBACK TABLE语句的其他信息，请FLASHBACK “Oracle数据库备份和恢复用户指南”

先决条件

要将表闪回到较早的SCN或时间戳，您必须具有表上的FLASHBACK对象特权或FLASHBACK ANY TABLE系统特权。此外，您必须对表具有READ或SELECT对象特权，并且您必须对表具有INSERT ， DELETE和ALTER对象特权。

行移动必须在闪回列表中的所有表，除非你是闪回表启用TO BEFORE DROP 。该操作称为闪回删除操作，它使用回收站中的已删除数据而不是撤消数据。有关启用行移动的信息，请参阅row_movement_clause 。

要将表闪回到还原点，必须具有SELECT ANY DICTIONARY或FLASHBACK ANY TABLE系统特权或SELECT_CATALOG_ROLE角色。

要在DROP TABLE操作之前将TABLE闪回，您只需要删除表所需的权限。

语义

在Oracle闪回表操作期间，Oracle数据库会在闪回列表中指定的所有表上获取独占DML锁。这些锁在它们恢复到早期状态时会阻止对表的任何操作。

无论闪回列表中指定的表数如何，闪回表操作都在单个事务中执行。要么所有表都恢复到之前的状态，要么都没有。如果闪回表操作在任何表上失败，则整个语句将失败。

在闪回表操作的完成，在数据table是一致table在较早的时间。然而， FLASHBACK TABLE TO SCN或者TIMESTAMP不保留的rowid，和FLASHBACK TABLE TO BEFORE DROP不恢复引用约束。

Oracle数据库不会将与table关联的统计信息还原为其早期格式。当前存在的table上的索引将被还原，并反映闪回点处的表的状态。如果索引现在存在但在闪回点处尚不存在，则数据库会更新索引以反映闪回点处表的状态。但是，不会恢复在闪回点和当前时间之间的间隔期间丢弃的索引。

模式

指定包含该表的架构。如果省略schema ，则数据库假定该表位于您自己的模式中。

表

指定包含要还原为早期版本的数据的一个或多个表的名称。

闪回表的限制

本声明受以下限制：

  • 闪回表操作对以下类型对象无效：属于集群的表，物化视图，高级队列（AQ）表，静态数据字典表，系统表，远程表，对象表，嵌套表或单个表分区或子分区。

  • 以下DDL操作会更改表的结构，以便您随后不能使用TO SCN或TO TIMESTAMP子句将表TIMESTAMP回到操作之前的时间：升级，移动或截断表;向表中添加约束，向集群添加表;修改或删除列;更改列加密密钥;添加，删除，合并，拆分，合并或截断分区或子分区（添加范围分区除外）。

关于闪回丢弃表的注意事项

以下注释适用于闪回的表格：

  • Oracle数据库检索从回收站检索到的表上定义的所有索引，但位图连接索引和域索引除外。（在DROP TABLE操作期间，位图连接索引和域索引不会放在回收站中，因此无法检索。）

  • 除了引用其他表的引用完整性约束之外，数据库还检索在表上定义的所有触发器和约束。

    检索到的索引，触发器和约束具有回收站名称。因此，建议在发出FLASHBACK TABLE之前查询USER_RECYCLEBIN视图...TO BEFORE DROP语句，以便您可以重命名检索的触发器和约束，以更实用的名称。

  • 删除表时，表中定义的所有物化视图日志也将被删除，但不会放在回收站中。因此，物化视图日志不能与表一起闪回。

  • 删除表时，表中的所有索引都将被删除，并与表一起放入回收站。如果出现后续空间压力，则数据库通过首先清除索引从回收站回收空间。在这种情况下，当您闪回表时，您可能无法获取表中定义的所有索引。

  • 如果表由于某些空间回收操作由用户或Oracle数据库清除，则无法闪回表。

关于透明数据加密的透明度问题

透明数据加密以各种方式处理数据的透明度。

安全审核员偶尔会询问有关透明数据加密（TDE）使用的加密的详细问题。他们请求有关TDE密钥，算法，长度和密钥库的信息，然后直接与PCI-DSS等法规要求进行比较。本主题包含有关TDE加密和密钥管理的重要详细信息。此信息是Oracle Database 12 c （12.1.0.2）中的最新信息。它旨在帮助TDE客户快速准确地回答审计员的问题。

  1. 透明数据加密是否与我的应用软件兼容？

    透明数据加密默认与应用程序兼容，因为它不会更改入站SQL语句或出站SQL查询结果。Oracle执行某些Oracle和第三方应用程序软件的内部测试和验证，以捕获有用的部署技巧或脚本，并评估性能配置文件。

    请注意透明数据加密与DBMS_CRYPTO PL / SQL包之间的区别。此软件包适用于不同的客户使用案例。它是一个API和工具包解决方案，因此它是不透明的。

  2. 透明数据加密是否与我正在使用的其他Oracle数据库工具和技术兼容？

    透明数据加密的主要优势之一是它与常用的Oracle数据库工具和技术集成，如高可用性集群，存储压缩，备份压缩，数据移动，数据库备份和恢复以及数据库复制。直接与透明数据加密集成的特定Oracle技术包括Oracle Real Application Clusters（Oracle RAC），Oracle Recovery Manager（RMAN），Oracle Data Guard，Advanced Compression，Oracle Data Pump和Oracle GoldenGate等。透明数据加密还与Oracle Exadata具有特殊的集成点，可充分利用Oracle工程系统的独特功能。

    透明数据加密还可以轻松使用Oracle数据库的安全功能。借助透明数据加密，权限授予，角色，Oracle Database Vault领域，虚拟专用数据库策略和Oracle Label Security标签仍然有效。您可以将这些和其他安全功能与透明数据加密加密结合使用。

  3. 是否存在任何已知的透明数据加密限制或不兼容性？

    • TDE列加密： TDE列加密在数据通过SQL层时透明地加密和解密数据。Oracle的某些功能将绕过SQL层，因此无法从TDE列加密中受益。以下是TDE列加密不支持的已知数据库功能及其相关软件版本号：

      • 物化视图日志（Oracle Database 11g第2版之前不支持）

      • 数据仓库（CDC）的同步和异步更改数据捕获

      • 可传输的表空间

      • 的LOB

      请注意，安全文件是在Oracle Database 11g第1版中引入的，因此在该版本之前不支持TDE列加密

    • TDE表空间加密： TDE表空间加密对存储在块级别的表空间中存储的所有内容进行加密，并且通常不会与其他数据库功能冲突。TDE表空间加密没有TDE列加密所具有的任何限制。但是，从TDE加密的源到TDE加密的目标，您可以使用完整的可传输表空间（TTS）与Oracle Data Pump压缩和加密。您必须具有可用的Oracle Database 12c或更高版本的数据库实例，以便您可以使用其密钥导出或密钥库（钱包）合并功能来获取目标数据库主机的正确主加密密钥，而无需覆盖原始Oracle钱包文件。此过程受标准TTS限制，您必须记住检查兼容的字节顺序。

  4. TDE使用哪些类型的密钥和算法？

    TDE依赖于两组不同的加密密钥。第一组加密密钥是TDE表空间加密密钥，用于透明地加密和解密存储的数据。DEK由数据库自动生成，以加密形式存储在数据库内部，并且主要在幕后进行管理。最终用户与DEK交互的一个地方是选择TDE将使用的加密算法和密钥长度，可以是3DES168，AES128，AES192或AES256。对于包含加密列和每个加密表空间的每个表，将独立进行此选择。您可能还会听到称为表键（列加密）或表空间键（表空间加密）的DEK。表密钥用于密码块链接（CBC）操作模式，表空间密钥用于密码反馈（CFB）操作模式。

    第二组加密密钥由当前和历史密钥加密密钥（KEK）组成，也称为主加密密钥。主加密密钥由数据库自动生成，根据需要自动用于加密和解密DEK，并存储在受保护的密钥库中。用户可以通过定期重新加密它，修改某些键属性等来与当前主加密密钥交互。通常，主加密密钥的密钥库是Oracle钱包（开箱即用解决方案）或Oracle Key Vault（专用密钥管理产品）。虽然数据库一次只使用一个TDE主密钥，但所有重新密钥的主加密密钥都保留在密钥库中，以便长期恢复加密数据备份。主加密密钥始终为AES256。他们使用CBC操作模式加密和解密DEK。对于DEK和主加密密钥，底层密钥材料不会直接公开。最终用户只能看到管理TDE所需的密钥属性。

  5. 包含主加密密钥的Oracle密钥库如何受到保护？

    当您使用Oracle钱包作为主加密密钥的密钥库时，需要考虑三种不同类型的密钥库：基于密码，自动登录和本地自动登录。所有这些密钥库外部化主加密密钥，因此它们与TDE加密数据分开。Oracle建议您将钱包文件放在受严格文件权限和其他安全措施保护的本地或网络目录中。

    基于密码的钱包是遵循PKCS＃12标准的加密密钥存储文件（ ewallet.p12 ）。它根据PKCS＃5标准由密码派生密钥加密。人类用户必须输入包含数据库密码的命令才能打开钱包，解密其内容并获得密钥访问权限。基于密码的钱包是TDE主密钥的默认密钥库。过去，它使用3DES168加密算法和CBC操作模式进行加密。orapki命令convert wallet使您可以将基于密码的钱包转换为AES256和CBC操作模式。“Oracle数据库安全指南”提供了有关使用orapki转换钱包的详细信息。

    自动登录钱包（ cwallet.sso ）可选地源自基于密码的标准钱包，用于需要自动启动数据库而无需人工交互输入钱包密码的特殊情况。使用自动登录钱包时，必须保留基于主密码的钱包，因为需要重新加密主加密密钥。除了将自动登录钱包存储在受严格文件权限保护的本地或网络目录中的最佳实践之外，数据库还使用专有方法对文件内容进行加密，以增加安全性。自动登录钱包上的一个名为本地自动登录钱包的轻微变化具有类似的行为。与本地自动登录钱包的一个显着区别是，其内容使用从创建文件的主机中获取的其他因素进行加扰。这会使本地自动登录钱包在其他主机上无法使用。主机因素和加扰技术的细节是专有的。

  6. 什么是Oracle Key Vault以及它如何管理TDE主密钥？

    Oracle Key Vault集中管理TDE主密钥，Oracle钱包，Java密钥库等。它可以帮助您控制增殖密钥和密钥存储文件。它包括专门针对TDE和Oracle堆栈的其他组件的优化。有关将Oracle Key Vault与TDE配合使用的更多信息，请参阅Oracle技术网和Oracle Key Vault管理员指南中的产品页面。

    • TDE列加密：

      另一个主要的性能提升来自于使用支持基于CPU的加密加速的硬件和软件，这些加速可用于Intel AES-NI和Oracle SPARC T4 / T5。要利用此功能，您必须运行最新版本的数据库，安装最新版本的操作系统，并使用在其CPU /内核中包含加密加速电路的硬件。

      数据库压缩进一步加快了透明数据加密性能，因为加密处理发生在已经压缩的数据上，导致加密和解密的总数据更少。

  3. 是否存在可能降低TDE性能的特定问题，如果是，我该如何避免它们？

    如果由于旧硬件，较旧的数据库版本或较旧的操作系统等因素导致数据库无法在主机上使用基于CPU的硬件加速，则TDE表空间性能会降低。

    请注意以下有关特定数据库工作负载的信息：

    • 一次加密整个数据集（例如，在将“批量数据加载”到Oracle数据仓库中时）：在将新数据批量加载到数据库或数据仓库期间，已观察到较低的加密性能。无法在SGA中缓存新数据，因此绕过了TDE表空间加密性能优化。因此，透明数据加密在此类操作中没有奖励性能优势。

      请遵循以下准则：

      • 确保数据库在具有基于CPU的加密加速的服务器上运行。这不仅加速了解密操作，还加速了操作（用于加载新数据）。通过预先加密数据集然后使用可传输表空间（TTS）加载到数据库中来进行带外加密处理。尝试尽可能并行化此过程。这要求数据库实例将所需的TDE密钥复制到目标数据库上的密钥库。当存在用于加密和加载的固定时间窗口时，该过程可能是不可行的，并且这些必须连续地进行。

      • 考虑使用TDE列加密。仅加密少数敏感的受控列，而不是加密整个表空间。

    • 一次解密整个数据集（例如，通过直接从磁盘读取执行全表扫描，而不从SGA读取）：

      运行全表扫描查询时会观察到较低的加密性能，其中数据直接从存储中读取。绕过了TDE表空间加密的某些性能优化（无缓存）。因此，透明数据加密在此类操作中没有奖励性能优势。

      请遵循以下准则：

      • 确保数据库在具有基于CPU的加密加速的服务器上运行。

      • 使用较大的SGA大小重新测试全表扫描查询，以测量从缓存中读取数据时的性能。尝试设置Oracle事件编号10949以禁用直接路径读取。

      • 对数据库进行分区，以便通过全表扫描操作扫描较少的数据。生产数据库通常对这种情况使用分区（即限制扫描的数据总量）。

      • 考虑使用TDE列加密。仅加密少数敏感的受控列，而不是加密整个表空间。
