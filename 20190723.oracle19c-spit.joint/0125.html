A.1.2从版本11.1,11.2或12.1升级RDF语义图支持

如果要从包含语义技术支持的Oracle Database Release 11.1或11.2升级，则在升级数据库时，语义技术支持会自动升级到版本12.1或更高版本。

但是，如果现有版本11.1或11.2 RDF网络包含三元组，其中包含类型为xsd的类型文字值：float，xsd：double，xsd：boolean或xsd：time，则可能还需要迁移RDF数据。

要检查是否需要迁移RDF数据，请以具有DBA权限的用户身份连接到数据库，并查询MDSYS.RDF_PARAMETER表，如下所示：

SELECT命名空间，属性，值FROM mdsys.rdf_parameter WHERE namespace ='MDSYS'AND属性IN（'FLOAT_DOUBLE_DECIMAL'，'XSD_TIME'，'XSD_BOOLEAN'，'DATA_CONVERSION_CHECK'）;

如果FLOAT_DOUBLE_DECIMAL，XSD_TIME或XSD_BOOLEAN属性具有字符串值INVALID或者如果DATA_CONVERSION_CHECK属性具有字符串值FAILED_UNABLE_TO_LOCK_APPLICATION_TABLES ， FAILED_INSUFFICIENT_WORKSPACE_PRIVILEGES或FAILED_OLS_POLICIES_ARE_ENABLED ，则需要迁移RDF数据。

但是，如果FLOAT_DOUBLE_DECIMAL，XSD_TIME和XSD_BOOLEAN属性不存在或有字符串值VALID ，如果DATA_CONVERSION_CHECK属性不存在，你不需要迁移RDF数据。但是，如果您的语义网络可能有任何空的RDF文字，请参阅处理空RDF文字 ;如果您选择将现有的空文字迁移到新格式，请按照本节中的步骤操作。

要迁移RDF数据，请按照下列步骤操作：

  1. 以具有SYSDBA权限的SYS用户身份（SYS AS SYSDBA，并在出现提示时输入SYS帐户密码）连接到数据库，然后输入： SET CURRENT_SCHEMA=MDSYS

  2. 确保用户MDSYS具有以下权限：

    • 对语义网络中的所有应用程序表的INSERT特权

    • ALTER ANY INDEX特权（可选：仅在使用文档的语义索引时才需要）

    • 对已修改已启用版本的应用程序表的任何工作空间的ACCESS权限（可选：仅在将Workspace Manager用于RDF数据时才需要）

  3. 确保暂时禁用任何针对RDF数据的OLS策略（可选：仅在使用OLS for RDF数据时才需要）。运行convert_old_rdf_data后，可以重新启用OLS策略。

  4. 启动SQL * Plus。如果要在不将现有空文本转换为新格式的情况下迁移RDF数据（请参阅处理空RDF文字 ），请输入以下语句：

    执行sdo_rdf_internal.convert_old_rdf_data;

    如果要迁移RDF数据并将现有的空文本转换为新格式，请调用convert_old_rdf_data ，并将flags参数设置为'CONVERT_ORARDF_NULL' 。此外，您可以使用可选的tablespace_name参数来指定在数据迁移期间创建中间表时要使用的表空间。例如，以下语句迁移旧的语义数据，将现有的"orardf:null "值转换为“”，并将MY_TBS表空间用于任何中间表：

    EXECUTE sdo_rdf_internal.convert_old_rdf_data（flags =>'CONVERT_ORARDF_NULL'，tablespace_name =>'MY_TBS'）;

    如果语义网络包含许多使用（或受其影响）xsd：float，xsd：double，xsd：time或xsd：boolean typed literals的三元组，则sdo_rdf_internal.convert_old_rdf_data过程可能需要很长时间才能运行。

  5. 输入以下语句：

    • Linux： @$ORACLE_HOME/md/admin/semrelod.sql

    • Windows： @%ORACLE_HOME%\md\admin\semrelod.sql

注意：

如果在升级到版本12.1或更高版本后未运行sdo_rdf_internal.convert_old_rdf_data过程和semrelod.sql脚本，则在执行SEM_MATCH查询时可能会遇到ORA-00904（无效标识符）错误。

A.1.2.1升级后所需的数据迁移

数据库升级完成后，如果您具有先前版本中的现有RDF数据，则必须迁移RDF数据。如果不执行数据迁移，则在运行SEM_MATCH查询时将遇到以下错误：

ORA-20000：RDF_VALUE $表需要使用SEM_APIS.MIGRATE_DATA_TO_CURRENT进行数据迁移

列已添加到版本12.2中的MDSYS.RDF_VALUE $表中（请参阅增强的RDF ORDER BY查询处理 。升级现有RDF网络后，必须填充这些列。将使用MDSYS.RDF_PARAMETER表中的以下行记录对迁移的需求：

  • NAMESPACE： MDSYS

  • ATTRIBUTE： RDF_VALUE$

  • VALUE： INVALID_ORDER_COLUMNS

  • 描述： RDF_VALUE$ Table needs data migration with SEM_APIS.MIGRATE_DATA_TO_CURRENT

如果需要迁移，RDF语义图安装最初将标记为INVALD ，在MDSYS.RDF_PARAMETER中使用以下行表示：

  • NAMESPACE： MDSYS

  • VALUE :(以12.2开头的字符串）

  2. 运行以下语句：

如果数据迁移成功，则将从MDSYS.RDF_PARAMETER中删除INVALID_ORDER_COLUMNS行，并且SEM_VERSION行的DESCRIPTION值将为VALID 。

要检查是否需要执行此额外的RDF数据迁移，请以具有DBA权限的用户身份连接到数据库，并查询MDSYS.RDF_PARAMETER表，如下所示：

但是，如果FLOAT_DOUBLE_DECIMAL，XSD_TIME和XSD_BOOLEAN属性不存在或有字符串值VALID ，如果DATA_CONVERSION_CHECK属性不存在，你不需要迁移RDF数据。但是，如果您的语义网络可能有任何空的RDF文字，请参阅处理空RDF文字 ;如果您选择将现有的空文字迁移到新格式，请按照本节中的步骤操作。

69.1 DBMS_FLASHBACK概述

DBMS_FLASHBACK为用户提供了一个界面，供用户在过去的特定时间查看数据库，并通过事务退出功能提供额外的容量，以便有选择地删除单个事务的影响。这与闪回数据库不同，闪回数据库将数据库及时移回。

启用DBMS_FLASHBACK ，用户会话使用数据库的闪回版本，应用程序可以针对数据库的闪回版本执行。

您可能希望使用DBMS_FLASHBACK ，原因如下：

  • 自助修复：如果您不小心从表中删除了行，则可以恢复已删除的行。

  • 打包的应用程序，如电子邮件和语音邮件：您可以使用闪回通过将已删除的邮件重新插入当前邮件框来还原已删除的电子邮件。

  • 决策支持系统（DSS）和在线分析处理（OLAP）应用程序：您可以执行数据分析或数据建模以跟踪季节性需求。

69.5 DBMS_FLASHBACK操作说明

会话结束时， DBMS_FLASHBACK会自动关闭，可以通过断开连接或启动另一个连接来关闭。

在闪回模式下打开的PL / SQL游标返回闪回时间或SCN的行。数据库中的不同并发会话（连接）可以执行闪回到不同的挂钟时间或SCN。会话在闪回模式下运行时，不允许DML和DDL操作以及分布式操作。您可以在禁用Flashback执行DML之前使用PL / SQL游标。

在自动还原管理（AUM）模式下，您可以使用保留控制来控制所需数据库版本的时间。如果需要在24小时内执行闪回，则DBA必须将undo_retention参数设置为24小时。这样，系统保留足够的撤消信息以重新生成旧版本的数据。

您可以为撤消表空间设置RETENTION GUARANTEE子句，以确保不会丢弃未过期的撤消。 UNDO_RETENTION本身并不是一种保证，因为如果系统处于空间压力之下，未过期的撤销可能会被新生成的撤销覆盖。在这种情况下， RETENTION GUARANTEE可以防止这种情况发生。有关更多信息，请参见“ Oracle数据库管理员指南” 。

在启用闪回的会话中， SYSDATE不受影响;它继续提供当前时间。

DBMS_FLASHBACK可以在登录触发器中使用，以在不更改应用程序代码的情况下启用闪回。

26.1关于分析视图

分析视图将数据层次/维度模型分层。

分析视图是在维度表和星形或雪花模式的事实表上定义的。您还可以在非规范化表上定义分析视图，其中维度属性和事实数据位于同一个表中。层次结构是在维度表上定义的。分析视图引用层次结构和事实表。

即使分析视图是在建模为星型模式的数据上定义的，也不需要将数据存储在星型模式中。您可以使用视图将其他形式的存储数据表示为分析视图。通常，如果表或视图在星型查询中表现良好，则它们可以很好地与分析视图配合使用。较小的数据集可能适用于视图。对于星型模式中的表，较大的数据集可能表现更好。最高性能的模式是使用Oracle Database In-Memory选项加载到内存中列存储中的星型模式。

与内存中列存储一起使用时，分析视图会优化SQL执行计划以利用内存中聚合（即矢量转换执行计划）。分析视图可以利用物化视图来进一步加速聚合级别查询（请注意，物化视图可以加载到内存中的列存储中）。

分析视图的最低要求包括以下内容：

  • 维度表（或视图）。此表应具有主键，该主键提供唯一的值列表并连接到事实表。

  • 具有至少一个事实（度量）列和连接到维度表的主键的键列的事实表。

更典型地，分析视图具有以下特征：

  • 定义使用两个或多个维度表，这使得能够切片和切块数据。

  • 一个或多个维度表包含不同聚合级别的数据（例如：天，月，季度和年）。

分析视图包含三种类型的对象：属性维，层次结构和分析视图。

属性维度是一个元数据对象，它引用表或视图，并将列组织为更高级别的对象，如属性和级别。与维和层次结构相关的大多数元数据都在属性维对象中定义。

层次结构是一种视图。层次结构引用属性维对象。层次结构使用层次结构成员之间的层次关系来组织数据。层次结构的查询返回详细信息和聚合级别键（“层次结构值”）以及这些值的属性。

分析视图是一种返回事实数据的视图。分析视图引用事实表和层次结构。从分析视图中选择层次结构和度量数据。

26.2分析观点的测量

分析视图度量指定事实数据以及要对数据执行的计算或其他操作。

在分析视图定义中，您可以指定一个或多个基本度量和计算度量。

基本措施

基本度量是对事实表中列的引用。您可以选择指定meas_aggregate_clause ，它会覆盖分析视图的默认聚合方法。每个基本度量可以指定默认聚合。聚合可以是简单的操作，如SUM或AVG ，也可以是由属性维度变化的复杂操作嵌套。

您可以使用default_aggregate_clause为没有meas_aggregate_clause基本度量指定默认聚合方法。default_aggregate_clause的默认值为SUM 。

计算的措施

计算度量是一种表达式，可以是用户定义的表达式，也可以是许多预定义的分析计算之一。计算的度量表达式可以包括其他度量，行函数和层次函数。层次结构函数允许基于识别和处理层次结构中的相关成员进行计算。表达式可以引用分析视图中的其他度量，但可能不引用事实列。由于计算可以参考其他度量，因此您可以通过嵌套轻松构建复杂的计算。

在定义计算的度量表达式时，无论您定义分析视图的度量的顺序如何，您都可以在分析视图中使用任何其他度量。唯一的限制是在计算中不会引入任何循环。

除了在分析视图的定义中使用计算度量之外，还可以在查询分析视图的SELECT语句中添加计算度量。为此，您可以在语句的WITH或FROM子句中使用ADD MEASURES关键字。无论是在分析视图的定义中还是在SELECT语句中，计算度量的语法都是相同的。

计算出的度量表达式的类别如下：

  • 分析视图度量表达式

  • 分析视图分层表达式

  • 简单的表达

  • 单行函数表达式

  • 复合表达式

  • 日期时间表达式

  • 区间表达式

  • 领先和滞后

  • 合格数据参考（QDR）

  • 秩

  • 相关成员

      ORADIM将日志文件写入ORACLE_HOME\database目录。

      注意：

      如果您使用Oracle Home User帐户拥有Oracle主目录，则ORADIM命令会提示您输入该用户名和密码。

    1. 您的ORACLE_SID设置正确

将非CDB Oracle数据库升级到CDB上的PDB

使用此过程升级早期版本的非CDB体系结构Oracle数据库，使其成为可插拔数据库（PDB）并将PDB插入容器数据库（CDB）。

您可以使用DBUA或并行升级实用程序升级早期版本的Oracle数据库，然后将升级后的数据库设置为可插拔数据库（PDB）。然后，您可以将升级的数据库插入多租户容器数据库（CDB）。

以下过程假定以下条件：

  • 您已完成操作系统的Oracle数据库文档中描述的所有升级前过程。

  • 较早的数据库和升级的数据库位于同一系统上。

  • 升级前后数据文件保留在同一位置。

    如果数据文件已复制到其他位置（例如，与Oracle ASM一起存储），则必须在步骤8中指定参数SOURCE_FILE_NAME_CONVERT 。

  1. 安装新的Oracle Database 19c软件。

  2. 按照本指南中的说明升级数据库。

  3. 如果您尚未在升级过程中执行此操作，请将COMPATIBLE参数设置为19.0.0。

  4. 使用以下SQL命令确保数据库处于只读模式：

    SQL> startup mount SQL> alter database open read only;

  5. 确保满足插入未插入PDB的先决条件。

  6. 为PDB创建XML文件。 XML文件的根名称与PDB的名称匹配。在以下语法示例中， path的值是保存XML的位置， myPDB .xml是可插入数据库文件的名称。您可以选择要放置文件的位置。 

    SQL> exec DBMS_PDB.DESCRIBE（' path / myPDB .xml'）;

    例如， path是/home/oracle ， myPDB是salespdb ：

    SQL> exec DBMS_PDB.DESCRIBE（'/ home / oracle / salespdb.xml'）;

  7. 使用以下命令关闭旧（源）Oracle主目录中的数据库：

    SQL> SHUTDOWN IMMEDIATE

  8. 将目录更改为新的Oracle主目录，然后运行DBMS_PDB.CHECK_PLUG_COMPATIBILITY函数。

    运行该功能时，请设置以下参数：

    • pdb_descr_file将此参数设置为XML文件的完整路径。

    • pdb_name指定新PDB的名称。如果省略此参数，则使用XML文件中的PDB名称。

    例如，要确定文件/disk1/usr/salespdb.xml描述的PDB是否与当前CDB兼容，请从新的Oracle主目录运行以下PL / SQL块：

    sqlplus / as sysdba SQL>在SQL上设置serveroutput> r 1 DECLARE 2兼容CONSTANT VARCHAR2（3）：= 3 CASE DBMS_PDB.CHECK_PLUG_COMPATIBILITY（4 pdb_descr_file =>'/ home / oracle / ORAOP2.xml'，5 pdb_name =>'SALESPDB '）6如果是，那么'是'7'否''8结束; 9 BEGIN 10 DBMS_OUTPUT.PUT_LINE（兼容）; 11 * END; YES PL / SQL过程成功完成。

    如果输出为YES ，则PDB兼容，您可以继续下一步。

    如果输出为NO ，则PDB不兼容，您可以检查PDB_PLUG_IN_VIOLATIONS视图以查看它不兼容的原因。

  9. 使用以下命令语法创建可插拔数据库，并将数据库插入CDB：

    SQL> CREATE PLUGGABLE DATABASE SALESPDB使用' path myPDB .xml'NOCOPY TEMPFILE REUSE;

    以下示例显示了创建可插拔数据库salespdb的命令：

    SQL> CREATE PLUGGABLE DATABASE salespdb使用'/home/oracle/salespdb.xml'NOCOPY TEMPFILE REUSE;

    您可以为PDB使用任何名称，但您使用的名称在此CDB中必须是唯一的。 TEMPFILE REUSE指定可以重用现有的TEMP表空间。

    此SQL命令完成时，应显示以下消息：

    可插拔数据库已创建。

    升级后的数据库现在是PDB，您可以将其放入CDB中。

    警告：

    在您使用NOCOPY选项之前，Oracle强烈建议您使用有效的备份。如果此命令失败，无论出于何种原因，那么您的数据库可能会损坏且无法恢复。

  10. 使用以下命令连接到PDB：

    SQL> ALTER SESSION set container = salespdb;

  11. 将字典转换为PDB类型。从admin目录中，运行noncdb_to_pdb.sql脚本。您必须先运行此脚本，然后才能首次打开PDB。

    例如：

    @ $ ORACLE_HOME / RDBMS /管理/ noncdb_to_pdb.sql

    注意：

    请注意，此脚本的运行时间可能从几分钟到一个多小时不等，具体取决于必须转换的新PDB字典中对象的数量和类型。

  12. 以读/写模式启动并打开新的PDB。您必须在Oracle数据库的读/写模式下打开新的PDB，才能完成新PDB与CDB的集成。

    例如，因为您已将PDB容器设置为salespdb ，请输入以下命令以启动PDB：

    SQL> STARTUP

  13. 使用RMAN（恢复管理器）备份PDB。

    Oracle强烈建议您使用RMAN执行PDB备份，因为在将数据库转换为PDB之前，您无法再使用从数据库中获取的ARCHIVELOG和备份。

    警告：

    您必须执行立即备份以确保可恢复性。

B Oracle Locator

Oracle Locator（也称为Locator）是Oracle数据库标准版的一项功能。Locator提供Oracle Spatial and Graph中提供的核心功能和服务。

它提供了支持基于Internet和无线服务的应用程序和基于合作伙伴的GIS解决方案所需的重要功能。定位器不是为需要复杂空间数据管理的地理信息系统（GIS）应用程序而设计的。如果需要线性参考，高级空间函数或空间和图形Web服务等功能，请使用Oracle Spatial and Graph而不是Locator。

与Spatial和Graph一样，Locator不是最终用户应用程序，而是应用程序开发人员的一组空间功能。

Locator适用于Oracle数据库的Standard Edition One，Standard Edition和Enterprise Edition，以及Oracle Database XE（Express Edition）。Spatial and Graph是一种定价选项，仅适用于Oracle数据库企业版。Spatial和Graph包括Locator的所有功能以及Locator不具备的其他功能。

虽然表B-2中不允许使用Oracle Locator的功能的最终列表，但您可以按照安装和卸载定位器或手动和空间和图形中的说明删除大多数这些功能。有关Spatial和Graph and Locator的许可信息 ，请参阅Oracle数据库许可信息 。

注意：

一些仅可用于Spatial和Graph的底层组件可以安装在Locator中以支持某些Locator功能。（ 表B-2列出了不允许与Oracle Locator一起使用的功能。）

Locator的安装取决于Oracle Multimedia的成功和正确安装。Oracle Multimedia随Oracle Database一起安装和配置。在安装Oracle Multimedia期间，安装了Locator。

通常，Locator包括Oracle Spatial和Graph的数据类型，运算符和索引功能，以及Spatial和Graph的一组有限的子程序（函数和过程）。定位器功能包括以下内容：

  • 描述和支持任何类型几何的对象类型（SDO_GEOMETRY）

3关于DML语句和事务

数据操作语言（DML）语句添加，更改和删除Oracle数据库表数据。事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。

  • 关于数据操作语言（DML）语句
    数据操作语言（DML）语句访问和操作现有表中的数据。
  • 关于交易控制报表
    事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。您需要事务来建模需要将多个操作作为一个单元执行的业务流程。
  • 提交交易
    提交事务会使其更改成为永久更改，删除其保存点并释放其锁定。
  • 回滚交易
    回滚事务会撤消其更改。您可以回滚整个当前事务，也可以只将其回滚到指定的保存点。
  • 在事务中设置保存点
    SAVEPOINT语句标记事务中的保存点 - 您可以稍后回滚的点。保存点是可选的，事务可以有多个保存点。

3.2关于交易控制声明

事务是Oracle数据库作为一个单元处理的一个或多个SQL语句的序列：要么执行所有语句，要么都不执行。您需要事务来建模需要将多个操作作为一个单元执行的业务流程。

例如，当经理离开公司时，必须在JOB_HISTORY表中插入一行以显示经理何时离开，并且对于向该经理报告的每个员工，必须在EMPLOYEES表中更新MANAGER_ID的值。要在应用程序中对此过程建模，必须将INSERT和UPDATE语句分组到单个事务中。

基本的事务控制语句是：

  • SAVEPOINT ，它标记事务中的保存点 - 您可以稍后回滚的点。保存点是可选的，事务可以有多个保存点。

  • 结束当前事务的COMMIT使其更改成为永久更改，擦除其保存点并释放其锁定。

  • ROLLBACK ，它回滚（撤消）整个当前事务或仅撤消指定保存点之后所做的更改。

在SQL * Plus环境中，您可以在SQL>提示符后输入事务控制语句。

在SQL Developer环境中，您可以在Worksheet中输入事务控制语句。SQL Developer还有“提交更改”和“回滚更改”图标，这些图标在“ 提交事务 ”和“ 回滚事务 ”中进行了说明 。

警告：

如果您没有显式提交事务，并且程序异常终止，则数据库会自动回滚最后一个未提交的事务。

Oracle建议您通过提交或回滚它们来明确地结束应用程序中的事务。

3.3提交交易

提交事务会使其更改成为永久更改，删除其保存点并释放其锁定。

要显式提交事务，请使用COMMIT语句或（在SQL Developer环境中） Commit Changes图标。

注意：

Oracle数据库在任何数据定义语言（DDL）语句之前和之后发出隐式COMMIT语句。有关DDL语句的信息，请参阅“ 关于数据定义语言（DDL）语句 ” 。

在提交交易之前：

  • 您的更改对您可见，但对数据库实例的其他用户不可见。

  • 您的更改不是最终的 - 您可以使用ROLLBACK语句撤消它们。

提交事务后：

  • 您的更改对其他用户以及提交事务后运行的语句可见。

  • 您的更改是最终的 - 您无法使用ROLLBACK语句撤消它们。

3.4回滚交易

回滚事务会撤消其更改。您可以回滚整个当前事务，也可以只将其回滚到指定的保存点。

要将当前事务回滚到指定的保存点，必须使用带有TO SAVEPOINT子句的ROLLBACK语句。

要回滚整个当前事务，请使用不带TO SAVEPOINT子句的ROLLBACK语句，或者（在SQL Developer环境中）使用Rollback Changes图标。

回滚整个当前事务：

  • 结束交易

  • 扭转所有变化

  • 删除所有保存点

  • 释放任何事务锁

仅将当前事务回滚到指定的保存点：

  • 不结束交易

  • 仅反转在指定保存点之后所做的更改

  • 仅删除指定保存点后设置的保存点（不包括指定的保存点本身）

  • 释放在指定保存点之后获取的所有表锁和行锁

    已请求访问在指定保存点之后锁定的行的其他事务必须继续等待，直到提交或回滚事务为止。未请求行的其他事务可以立即请求和访问行。

要在SQL Developer中查看回滚的效果，可能必须单击“ 刷新”图标。

14.1关于在OCI中注册用户回调

应用程序可以使用OCIUserCallbackRegister()函数注册用户回调库。

回调在环境句柄的上下文中注册。应用程序可以使用OCIUserCallbackGet()函数检索有关使用句柄注册的回调的信息。

用户定义的回调是针对OCI调用和环境句柄注册的子例程。它可以指定为条目回调，替换回调或退出回调。

  • 如果是条目回调，则在程序进入OCI功能时调用它。

  • 在条目回调之后执行替换回调。如果替换回调返回值OCI_CONTINUE ，则执行后续替换回调或正常的OCI特定代码。如果替换回调返回OCI_CONTINUE以外的任何内容，则后续替换回调和OCI代码不会执行。

  • 在替换回调返回OCI_CONTINUE其他OCI_CONTINUE或OCI函数成功执行之后，程序控制将转移到退出回调（如果已注册）。

如果替换或退出回调返回OCI_CONTINUE以外的任何内容，则从关联的OCI调用返回回调的返回码。

14.1.2用户回调函数

有关用户回调函数的详细信息。

用户回调函数必须使用以下语法：

typedef sword（* OCIUserCallback）（void * ctxp，/ *用户回调的上下文* / void * hndlp，回调的/ *句柄，现在的* / ub4类型的env句柄，/ *此版本的OCI_HTYPE_ENV） * / ub4 fcode，/ * OCI调用的函数代码* / ub1何时，/ *回调类型，入口或出口* / sword returnCode，/ * OCI返回码* / ub4 * errnop，/ * Oracle错误号* / va_list arglist）; / * oci调用的参数* /

除了OCIUserCallbackRegister()调用中描述的参数之外，还使用返回码， errnop以及原型定义声明的原始OCI的所有参数调用回调。

返回代码始终作为OCI_SUCCESS传入，并且*errnop始终作为0传递给第一个条目回调。需要注意的是*errnop指的是内容errnop因为errnop是一个IN / OUT参数。

如果回调不想更改OCI返回码，则它必须返回OCI_CONTINUE ，并忽略*errnop返回的值。但是，如果回调返回除OCI_CONTINUE之外的任何返回码，则最后返回的返回码将成为该调用的返回码。此时， *errnop返回的值在错误句柄中设置，或者在环境句柄中设置，如果由于缺少某些OCI调用（如OCIHandleAlloc()的错误句柄而在环境句柄中返回错误信息。

对于替换回调， returnCode是前一个回调或OCI调用的非OCI_CONTINUE返回码，而*errnop是错误句柄中返回的错误号的值。这允许后续回调在需要时更改返回码或错误信息。

替换回调的处理方式不同，如果它返回OCI_CONTINUE以外的任何内容，则会绕过后续替换回调和OCI代码并处理跳转到退出回调。

请注意，如果替换回调返回OCI_CONTINUE以允许处理OCI代码，则忽略条目回调中的返回代码。

OCI调用的所有原始参数都作为变量参数传递给回调，并且回调必须使用va_arg宏检索它们。回调演示程序提供了示例。

可以注册空值以取消注册回调。也就是说，如果OCIUserCallbackRegister()调用中的回调值（ OCIUserCallback() ）为NULL ，则取消注册用户回调。

使用线程安全模式时，OCI程序在调用用户回调之前获取所有互斥锁。

14.1.5条目回调中的错误

如果条目回调想要向OCI调用的调用者返回错误，则它必须注册替换或退出回调。

这是因为如果执行OCI代码，则忽略条目回调中的错误代码。因此，条目回调必须通过其自己的上下文将错误传递给替换或退出回调。

14.1.6动态回叫注册

因为期望用户回调用于监视OCI行为或访问其他数据源，所以希望回调的注册是透明且非侵入地完成的。

这是通过在OCI初始化时加载用户创建的动态链接库来实现的。这些动态链接库称为包 。用户创建的包注册所选OCI调用的用户回调。在运行时接收控件时，这些回调可以根据需要进一步注册或取消注册用户回调。

ociucb.mk演示程序提供了makefile（Solaris上的ociucb.mk ）来创建程序包。此程序包的确切命名和位置取决于操作系统。包的源代码必须提供在OCI初始化和环境创建时调用的特殊回调的代码。

设置操作系统环境变量ORA_OCI_UCBPKG控制程序包的加载。此变量以通用方式命名包。软件包必须位于$ORACLE_HOME/lib目录中。

14.1.7关于加载多个包

ORA_OCI_UCBPKG变量可以包含以分号分隔的包名列表。包按照列表中指定的顺序加载。

例如，在过去，包被指定为：

setenv ORA_OCI_UCBPKG mypkg

目前，您仍然可以像以前一样指定包，但另外可以将多个包指定为：

setenv ORA_OCI_UCBPKG“mypkg; yourpkg; oraclepkg; sunpkg; msoftpkg”

所有这些包都按顺序加载。也就是说， mypkg加载msoftpkg ，最后加载msoftpkg 。

最多可以指定五个包。

注意：

示例makefile ociucb.mk在Solaris或Windows系统上的ociucb.dll上创建ociucb.so.1.0 。要加载ociucb包，必须将环境变量ORA_OCI_UCBPKG设置为ociucb 。在Solaris上，如果包名以.so结尾，则OCIEnvCreate（）或OCIEnvNlsCreate（）将失败。包名称必须以.so.1.0 。

有关创建动态链接库的更多详细信息，请阅读操作系统演示目录中提供的Makefile。有关用户定义的回调的更多信息，请参阅有关编译和链接应用程序的特定于操作系统的文档。

14.1.8包格式

包源必须提供两个功能。

在过去，包必须指定OCIEnvCallback()函数的源代码。但是， OCIEnvCallback()函数已过时。相反，包源必须提供两个功能。第一个函数必须命名为后缀为Init的 packagename 。例如，如果包名为foo ，那么源文件（例如，但不一定是foo.c）必须包含一个fooInit()函数，其调用OCISharedLibInit()函数的方式完全如下：

sword fooInit（metaCtx，libCtx，argfmt，argc，argv）void * metaCtx; / * metacontext * / void * libCtx; / *此包的上下文。* / ub4 argfmt; / *包参数格式* / sword argc; / * package arg count * / void * argv []; / * package arguments * / {return（OCISharedLibInit（metaCtx，libCtx，argfmt，argc，argv，fooEnvCallback））; }

所述的最后一个参数OCISharedLibInit()函数， fooEnvCallback()在这种情况下，是所述第二函数的名称。它可以被命名为任何东西，但按照惯例，它被命名为packagename，后缀为EnvCallback 。

此函数是OCIEnvCallback()的替代品。目前，必须在此函数中注册所有动态用户回调。该函数必须是OCIEnvCallbackType类型，其指定为：

typedef sword（* OCIEnvCallbackType）（OCIEnv * env，ub4 mode，size_t xtramem_sz，void * usrmemp，OCIUcb * ucbDesc）;

创建环境句柄时，最后会调用此回调函数。env参数是新创建的环境句柄。

mode xtramem_sz和usrmempp是传递给usrmempp （）调用的参数。最后一个参数ucbDesc是传递给包的描述符。该包使用此描述符来注册用户回调，如稍后所述。

14.1.10关于通过OCI访问其他数据源

由于Oracle数据库是访问的主要数据库软件，因此应用程序可以利用OCI接口通过使用用户回调来访问非Oracle数据。

这允许用OCI编写的应用程序访问Oracle数据而不会造成任何性能损失。可以编写驱动程序来访问用户回调中的非Oracle数据。由于OCI提供了非常丰富的接口，因此通常可以直接将OCI调用映射到大多数数据源。此解决方案比为其他中间层（如ODBC）编写应用程序更好，这会对所有数据源造成性能损失。使用OCI不会对访问Oracle数据源造成任何惩罚，并且会对ODBC对非Oracle数据源造成相同的损失。

14.1.11回调函数的限制

详细说明了对回调函数的限制。

回调函数的使用有一些限制，包括OCIEnvCallback() ：

  • 回调不能调用除OCIUserCallbackRegister() ， OCIUserCallbackGet() ， OCIHandleAlloc()和OCIHandleFree()之外的其他OCI函数。即使对于这些函数，如果在用户回调中调用它们，则不会调用它们上的回调以避免递归。例如，如果在OCIHandleFree()的回调中调用OCILogoff() ，则在执行OCILogoff()的回调期间将禁用OCIHandleFree()的回调。

  • 回调不能修改OCI数据结构，例如环境或错误句柄。

  • 无法为OCIUserCallbackRegister()调用本身或以下任何调用注册回调：

    • OCIUserCallbackGet（）

    • OCIEnvCreate（）

    • OCIInitialize() （已弃用）

    • OCIEnvNlsCreate（）
