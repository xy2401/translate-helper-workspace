28.7.5 OCIStringResize（）

调整给定字符串的内存大小。

目的

调整给定字符串的内存大小。

句法

sword OCIStringResize（OCIEnv * env，OCIError * err，ub4 new_size，OCIString ** str）;

参数

env（IN / OUT）

OCI环境句柄在对象模式下初始化。有关详细信息，请参阅OCIEnvNlsCreate() ， OCIEnvCreate()和OCIInitialize() （不建议使用）的说明。

错误（输入/输出）

OCI错误句柄。如果有错误， OCI_ERROR err记录，此函数返回OCI_ERROR 。通过调用OCIErrorGet()获取诊断信息。

new_size（IN）

字符串的新内存大小（以字节为单位）。new_size参数必须包含NULL字符的空格作为字符串终止符。

str（IN / OUT）

为从OCI对象缓存中释放的字符串分配的内存。

评论

此函数调整对象缓存中给定可变长度字符串的内存大小。不保留字符串的内容。此函数可以在新的内存区域中分配字符串，在这种情况下，释放给定字符串占用的原始内存。如果str为NULL ，则此函数为字符串分配内存。如果new_size为0，则此函数释放str占用的内存，并返回NULL指针值。

265.1 UTL_REF概述

UTL_REF过程使您能够在不知道对象表名称的情况下编写泛型类型方法。

Oracle支持用户定义的复合类型或对象类型。对象类型的任何实例都称为对象。对象类型可以用作列的类型或表的类型。

在对象表中，表的每一行都存储一个对象。您可以使用对象标识符唯一标识对象表中的对象。

引用是指向对象的持久指针，每个引用都可以包含对象标识符。引用可以是对象类型的属性，也可以存储在表的列中。给定引用，可以检索对象。

265.2 UTL_REF安全模型

使用此包需要程序选项。必须在SYS （ CONNECT /AS SYSDBA ）下创建此程序包。此程序包提供的操作在当前调用用户下执行，而不是在程序包所有者SYS下执行。

您可以使用服务器上存储的PL / SQL过程/包中的UTL_REF包，以及客户端/侧PL / SQL代码。

当从服务器上的PL / SQL过程/包调用时， UTL_REF验证调用者是否具有访问REF指向的对象的适当权限。

注意：

这与服务器上的PL / SQL包/过程形成对比，后者使用定义者的权限操作，其中包所有者必须具有执行所需操作的适当权限。

因此，如果在用户SYS下定义了UTL_REF ，则用户A调用UTL_REF 。 SELECT从引用中选择一个对象，然后用户A（调用者）需要检查权限。

从客户端PL / SQL代码调用时， UTL_REF以客户端会话的权限运行，在该会话下执行PL / SQL。

265.3 UTL_REF类型

对象类型是由用户定义的复合数据类型，或作为库类型提供。

您可以使用以下语法创建对象类型employee_type ：

CREATE TYPE employee_type AS OBJECT（名称VARCHAR2（20），id NUMBER，成员函数GET_ID（名称VARCHAR2）返回成员）;

对象类型employee_type是一个用户定义的类型，它包含两个属性： name和id ，以及一个成员函数GET_ID （）。

您可以使用以下SQL语法创建对象表：

CREATE TABLE employee_table OF employee_type;

265.4 UTL_REF例外

出于各种原因，在执行UTL_REF函数期间可以返回异常。

例如，以下方案将导致异常：

  • 所选对象不存在。这可能是因为：

    1. 该对象已被删除，或者给定的引用是悬空（无效）。

    2. 对象表已删除或不存在。

  • 无法在可序列化的事务中修改或锁定该对象。可序列化事务启动后，该对象被另一个事务修改。

  • 您无权选择或修改对象。UTL_REF子程序的调用者必须对正在选择或修改的对象具有适当的权限。

Oracle Flex集群概述

安装在Oracle Flex Cluster配置中的Oracle Grid Infrastructure是一个可扩展，动态，强大的节点网络。

Oracle Flex Clusters为各种应用程序提供了一个平台，包括具有大量节点的Oracle Real Application Clusters（Oracle RAC）数据库。Oracle Flex Clusters还为其他服务部署提供了一个平台，需要协调和自动化以实现高可用性。

Oracle Flex Cluster中的所有节点都属于单个Oracle Grid Infrastructure集群。此体系结构集中了基于应用程序需求部署资源的策略决策，以考虑各种服务级别，负载，故障响应和恢复。

Oracle Flex Clusters包含Hub节点和任意数量的其他受支持节点。Oracle Flex Cluster中的Hub节点数可以多达64个。其他节点的数量可以更多。集线器节点可以托管不同类型的应用程序。

集线器节点类似于Oracle Clusterware标准集群配置中的Oracle Grid Infrastructure节点：它们紧密连接，可直接访问共享存储。使用集线器节点来承载读写数据库实例。

Oracle Flex Cluster中的其他受支持节点与标准Oracle Grid Infrastructure节点不同，因为它们不需要直接访问共享存储，而是通过集线器节点请求数据。使用其他节点来托管只读数据库实例。

注意：

同一主数据库的读写和只读数据库实例可以在Oracle Flex Cluster中共存。

集线器节点可以在Oracle Flex Cluster配置中运行，而无需将任何其他节点作为集群成员节点，但其他节点必须是包含至少一个集线器节点的集群的成员。

注意：

如果升级Oracle Flex Cluster，则Oracle建议您首先升级集线器节点，并且在升级过程中还要升级并运行任何已升级的集线器节点。

读者节点

您可以使用其他节点来托管以只读模式运行的Oracle RAC数据库实例，这些实例将成为读取器节点 。您可以通过为节点配置大量内存来优化这些节点以进行并行查询操作，以便将数据缓存在主机节点中。

主机节点将定期心跳消息发送到其关联的集线器节点，这与集线器节点之间发生的心跳消息不同。在计划关闭集线器节点期间，托管节点会尝试连接到另一个集线器节点，除非托管节点仅连接到一个集线器节点。如果集线器节点被驱逐，则托管节点也将从集群中逐出。

更改群集模式

您可以将现有Oracle Clusterware标准群集的模式更改为Oracle Flex群集。

注意：

  • 更改群集模式需要群集停机时间。

  • Oracle不支持将Oracle Flex Cluster更改为Oracle Clusterware标准群集。

  • Oracle Flex Cluster需要网格命名服务（GNS）。

  • 区域授权不是必需的。

将Oracle Clusterware标准群集更改为Oracle Flex群集

使用CRSCTL将现有Oracle Clusterware标准群集更改为Oracle Flex群集。

执行以下步骤：

  1. 运行以下命令以确定群集的当前模式：

    $ crsctl获取集群模式状态

  2. 运行以下命令以确保为网格命名服务（GNS）配置了固定的VIP：

    $ srvctl配置gns

    除非GNS配置了固定的VIP，否则此过程不会成功。如果没有GNS，那么，以root身份创建一个，如下所示：

    #srvctl add gns vip_name | ip_address

    以root身份运行以下命令以启动GNS：

    #srvctl启动gns

  3. 在更改群集模式之前，请使用Oracle自动存储管理配置助手（ASMCA）在群集中启用Oracle Flex ASM。

  4. 以root身份运行以下命令，将集群模式更改为Oracle Flex集群：

    #crsctl set cluster mode flex

  5. 通过在群集中的每个节点上以root身份运行以下命令来停止Oracle Clusterware：

    #crsctl停止crs

  6. 通过在群集中的每个节点上以root身份运行以下命令来启动Oracle Clusterware：

    #crsctl start crs -wait

    注意：

    使用-wait选项显示进度和状态消息。

Oracle Extended Clusters

您可以跨两个或多个地理位置不同的站点扩展Oracle RAC集群，每个站点都配有自己的存储。如果其中一个站点发生故障，另一个站点将充当活动备用站点。

通常，Oracle ASM和Oracle数据库堆栈都设计为在数据中心中使用企业级共享存储。但是，光纤通道技术使您可以跨两个或多个数据中心分配计算和存储资源，并通过以太网电缆和光纤通道连接它们，分别满足计算和存储需求。

虽然您可以在安装Oracle Grid Infrastructure时配置Oracle Extended Clusters，但也可以使用ConvertToExtended脚本在安装后执行此操作。您可以使用CRSCTL管理Oracle Extended Cluster。

配置Oracle Extended Clusters

只有已安装或升级到Oracle Grid Infrastructure 12 c第 2版（12.2）或更高版本的群集才支持此过程，这些群集通常配置有一个站点（默认站点）。

注意：

此过程要求可以访问群集中的所有节点。还会出现群集中断，在此期间数据库访问中断。
您可以使用一个或多个磁盘组以及多个故障组配置Oracle Extended Cluster。使用ConvertToExtended脚本，您可以创建多个数据站点并将节点与每个数据站点相关联。所有Oracle Flex ASM存储都与默认群集站点保持关联，因为没有将现有磁盘组转换为扩展磁盘组的机制。将群集转换为Oracle Extended Cluster后，投票文件成员资格保持不变，而不是分层。
您还必须添加扩展磁盘组，并将投票文件迁移到扩展磁盘组，以利用特定于站点的分层投票文件算法。

制定升级Oracle数据库的测试计划

查看这些主题以了解如何创建一系列精心设计的测试以验证升级过程的所有阶段。

Oracle建议您对数据库和应用程序执行严格的测试。成功运行并完成测试后，您将有助于确保了解升级生产数据库的过程，以便升级过程可预测且成功。Oracle强烈建议您在升级生产数据库之前执行尽可能多的测试。不要低估完整且可重复的测试过程的重要性。

您可以选择手动执行测试，也可以使用实用程序来协助测试，例如Oracle Real Application Testing功能，如数据库重放或SQL性能分析器。在任何一种情况下，您执行的测试类型都是相同的。

您的测试计划必须包括以下类型的测试：

  • 升级测试
    创建，测试和验证升级计划。
  • 最小的测试
    在测试新版本环境中对应用程序执行最低限度测试时，可以找到应用程序启动或调用问题。
  • 升级后的功能测试
    升级完成后，对升级后的Oracle数据库执行功能测试。
  • 高可用性测试
    计划在升级后的系统上执行高可用性测试。
  • 集成测试以确保应用程序兼容
    Oracle数据库的集成测试检查系统组件之间的交互。
  • 性能测试升级的Oracle数据库
    规划早期版本和新版Oracle数据库之间的性能测试比较。
  • Oracle数据库升级的卷和负载压力测试
    使用数据库重放在高容量和负载下执行整个升级的Oracle数据库的卷和负载压力测试。
  • Oracle数据库升级规划的测试计划指南
    对当前数据库以及升级到新Oracle Database发行版的测试数据库执行计划测试。

最小的测试

在测试新版本环境中对应用程序执行最低限度测试时，可以找到应用程序启动或调用问题。

对Oracle数据库的最小测试需要将应用程序的全部或部分从当前数据库移动到新数据库，并在不启用任何新数据库功能的情况下运行应用程序。最小测试可能无法揭示实际生产环境中出现的问题。但是，最小的测试会立即显示任何应用程序启动或调用问题。

升级后的功能测试

升级完成后，对升级后的Oracle数据库执行功能测试。

Oracle数据库的功能测试是一组测试，其中升级后测试系统的新功能和现有功能。功能测试包括所有数据库，网络和应用程序组件。功能测试的目的是验证系统的每个组件是否像升级之前那样运行，并验证新功能是否正常工作。

SQL计划管理

查看此主题可了解升级后如何执行SQL计划管理以避免性能下降。

安装新优化程序版本的数据库升级通常会导致一小部分SQL语句的计划更改。大多数计划更改都不会导致性能变化或改进。但是，某些计划更改可能会导致性能下降。SQL计划管理通过提供用于捕获，选择和发展SQL计划信息的组件来防止因SQL语句的执行计划的突然更改而导致的性能回归。SQL计划管理是一种预防机制，可以记录和评估SQL语句的执行计划，并构建由一组现有计划组成的SQL计划基准，这些计划在重复使用后被证明是有效的。SQL计划管理使用SQL计划基准来保留相应SQL语句的性能，而不管系统中发生的更改。

通过SQL计划管理，优化程序可自动管理执行计划并确保仅使用已知或已验证的计划。当SQL计划管理找到SQL语句的新计划时，在数据库验证新计划具有与当前计划相当或更好的性能之前，它不会使用此计划。如果使用当前的执行计划为SQL计划管理提供种子，那么这些计划将成为每个语句的SQL计划基准。优化程序在升级后使用这些计划。如果Oracle Database 12 c优化器确定不同的计划可以带来更好的性能，那么新计划将排队等待验证，并且在确认其具有与当前计划相当或更好的性能之前不会使用。

有几种方法可以使用执行计划为SQL管理库（SMB）提供种子或填充：

从游标缓存批量加载SQL管理库

升级从Oracle数据库11 g的数据库，Oracle数据库的最新版本时，从游标缓存执行计划或SQL计划基准批量装载是非常有用的。游标缓存是一个共享SQL区域，自动接受批量加载的SQL计划，并将其作为SQL计划基准添加到现有或新计划历史记录中。

  1. 在Oracle数据库的源代码版本中，使用DBMS_SPM.LOAD_PLAN_FROM_CURSOR_CACHE过程或Oracle Enterprise Manager将游标缓存中的所有执行计划加载到SQL Management Base中。

  2. 升级数据库。

可以使用SQL调整集批量加载执行计划或SQL计划基准。从Oracle数据库10g升级（从没有SQL管理库（SMB）到从游标缓存直接加载）或从自动工作负载存储库加载历史计划时，这非常有用。

  1. 在Oracle数据库12c的测试系统，完成所有测试和调整后，使用DBMS_SPM.LOAD_PLAN_FROM_CURSOR_CACHE过程或企业管理器加载所有的游标缓存到SQL管理库的执行计划。

使用数据库重放在高容量和负载下执行整个升级的Oracle数据库的卷和负载压力测试。

卷描述了被操纵的数据量。Load描述了系统上并发需求的级别。卷和负载测试可以模拟生产系统在各种卷和负载下的行为方式。

体积和负载压力测试至关重要。但是，它常被忽视。Oracle发现客户通常不进行任何类型的量或负载压力测试。相反，客户通常依赖于不是业务应用程序特征的基准。Oracle建议您对应用程序进行基准测试。基准测试可以帮助您发现与功能，性能和集成相关的问题。但是，它们无法取代体积和负载压力测试。

负载测试涉及针对新的Oracle数据库版本运行应用程序负载。负载测试可确保应用程序不会遇到问题，例如新错误或在您认为可能在生产中发现的负载条件下的性能问题。很多时候，问题仅在某些负载条件下出现，并且通常在功能测试中看不到。数据库重放功能非常适合此类负载测试。数据库重放使您可以从生产环境捕获系统工作负载，并在测试系统上以相同的方式重放它。

编写PGAU声明

在评估RHT之后，将TIP定义为PGAU以放置在 PG DD。

  1. 写一个数据列表中每个条目的DEFINE DATA语句。例如，如果您的RHT有三种不同的数据结构，那么您的数据定义可能是：

    DEFINE DATA dname1 LANGUAGE（IBMVSCOBOLII）INFILE（dnamel.ext）; DEFINE DATA dname2 LANGUAGE（IBMVSCOBOLII）INFILE（dname2.ext）; DEFINE DATA dname3 LANGUAGE（IBMVSCOBOLII）INFILE（dname3.ext）;

    然后，您必须将包含这些数据定义的源文件复制或传输到PGAU可以将其作为输入读取的目录。

  2. 写一个您的通话清单中每个条目的DEFINE CALL语句。例如，如果您的RHT有一个接收发送接收发送序列，您的呼叫定义将是：

    DEFINE CALL cname1 PARMS（（dnamel IN））; DEFINE CALL cname2 PARMS（（dname2 OUT））; DEFINE CALL cname3 PARMS（（dname3 IN））; DEFINE CALL cname4 PARMS（（dname2 OUT））;

    注意：

    您可以选择重写您的呼叫定义，以便将数据传输合并到更少的交换中，只要您不改变数据传输顺序即可。例如：

    DEFINE CALL cname1 PARMS（（dname1 IN），

    （dname2 OUT））;

    DEFINE CALL cname3 PARMS（（dname3 IN），

    （dname2 OUT））;

    这减少了应用程序和TIP之间的调用，从四次调用到每次调用时传递IN和OUT参数的两次调用。由于TIP总是在OUT参数之前处理IN参数，因此数据传输顺序不变。但是，这种合并并非总是可行。

    如果您的通信协议是SNA ：有关PGAU DEFINE CALL命令的更多信息，请参阅灵活调用序列 。

    如果您的通信协议是TCP / IP ：有关PGAU DEFINE CALL命令的更多信息，请参阅灵活调用序列 。

28 DBMS_AUTO_SQLTUNE

DBMS_AUTO_SQLTUNE包是用于管理自动SQL调整任务的界面。与DBMS_SQLTUNE不同， DBMS_AUTO_SQLTUNE包需要DBA角色。

本章包含以下主题：

  • DBMS_AUTO_SQLTUNE概述

  • DBMS_AUTO_SQLTUNE安全模型

  • DBMS_AUTO_SQLTUNE子程序摘要

28.1 DBMS_AUTO_SQLTUNE概述

在AutoTask框架内运行时， DBMS_AUTO_SQLTUNE包是SQL Tuning Advisor（ DBMS_SQLTUNE ）的接口。

数据库创建自动化系统任务SYS_AUTO_SQL_TUNING_TASK作为目录脚本的一部分。此任务自动从AWR中选择一组高负载SQL，并在此SQL上运行SQL Tuning Advisor。自动化任务执行与任何其他SQL调整任务相同的综合分析。

自动化任务通过执行旧的和新的查询计划来测试它找到的任何SQL配置文件。自动SQL调优与手动SQL调优的区别在于一个重要方面。如果启用了SQL配置文件的自动实现（默认为禁用），则数据库将实现任何可保证良好性能优势的SQL配置文件。实现在调优时进行，以便数据库可以立即从新计划中受益。您可以使用SET_AUTO_TUNING_TASK_PARAMETER API设置ACCEPT_SQL_PROFILES参数来启用或禁用自动实现。

在每个维护窗口中，自动调整任务将其结果存储为新执行。每个执行结果具有相同的任务名称，但具有不同的执行名称。查询DBA_ADVISOR_EXECUTIONS视图以获取有关任务执行的信息。要查看跨多个执行的报告，请使用REPORT_AUTO_TUNING_TASK函数 。

28.2 DBMS_AUTO_SQLTUNE安全模型

具有DBA角色的用户可以使用此包。对于其他用户，您必须明确授予程序包的EXECUTE特权。请注意， EXECUTE_AUTO_TUNING_TASK过程是一个例外：只有SYS可以调用它。

用户可以调用此程序包中的API来控制自动调整任务在运行时的行为方式，例如启用自动SQL配置文件创建以及配置任务运行的总时间和每个SQL时间限制。由于这些设置会影响数据库的整体性能，因此具有ADVISOR权限的所有用户可能无法访问此程序包。

10.1关于创建代理PDB

代理PDB提供对远程CDB中的PDB的访问。它类似于符号链接。

CREATE PLUGGABLE DATABASE语句通过引用不同CDB中的PDB来创建代理PDB，该CDB称为引用的PDB 。如果需要远程PDB的本地上下文，可以使用代理PDB。此外，当不同CDB中的应用程序容器具有相同的应用程序时，您可以使其应用程序根与代理PDB保持同步。

要使用此技术，请在将包含代理PDB的CDB中运行CREATE PLUGGABLE DATABASE语句。你必须包括：

  • AS PROXY子句指定您正在创建代理PDB。

  • 一个FROM子句，指定代理PDB引用的PDB。

  • 指向FROM子句中引用的PDB的当前位置的数据库链接。必须在将包含代理PDB的CDB的根目录中创建数据库链接，并且数据库链接连接到远程CDB的根目录或远程引用的PDB。

10.1.1代理PDB和SQL语句

通常，当代理PDB是当前容器时，提交在代理PDB中执行的SQL语句在引用的PDB中执行。

远程执行的结果将返回给代理PDB。例如，数据定义语言（DDL）语句，数据操作语言（DML）语句和在代理PDB中执行的查询被发送到引用的PDB以供执行，并且结果被返回到代理PDB。

该规则有一个例外。当代理PDB是当前容器时，以及执行ALTER PLUGGABLE DATABASE和ALTER DATABASE语句时，这些语句仅影响代理PDB。它们不会被发送到引用的PDB以供执行。同样，当前容器是代理PDB所属的根时， ALTER PLUGGABLE DATABASE语句仅影响代理PDB。例如，在CDB根，应用程序根或代理PDB中执行的ALTER PLUGGABLE DATABASE语句可以打开或关闭代理PDB，但此语句不会打开或关闭引用的PDB。

10.1.2代理PDB和数据库链接

创建代理PDB时需要数据库链接。

创建代理PDB后，代理PDB不再使用创建期间指定的数据库链接。相反，代理PDB直接与引用的PDB通信。

此直接通信需要包含引用的PDB的CDB的侦听器的端口号和主机名。在代理PDB创建期间，代理PDB默认使用以下值：

  • 监听端口号： 1521

    如果引用的PDB的侦听器不使用默认端口号，则必须使用PORT子句指定侦听器的端口号。您可以在创建代理PDB时指定端口号，也可以更改代理PDB以更改端口号。

  • 监听器主机名：包含引用的PDB的CDB的主机名

    如果引用的PDB的侦听器不使用默认主机名，则必须使用HOST子句指定侦听器的主机名。您可以在创建代理PDB时指定主机名，也可以更改代理PDB以更改主机名。

10.2创建代理PDB

通过引用不同CDB中的PDB来创建代理PDB。

先决条件

必须满足以下先决条件：

  • 完成“ PDB创建的一般先决条件”中描述的先决条件 。

  • 当前用户必须在正在创建代理PDB的CDB的根目录中具有CREATE PLUGGABLE DATABASE系统特权。

  • 包含引用的PDB的CDB必须处于本地撤消模式。

  • 包含引用的PDB的CDB必须处于ARCHIVELOG模式。

  • 创建代理PDB时，引用的PDB必须处于打开的读/写模式。创建代理PDB后，可以更改引用的PDB的打开模式。

  • 数据库链接必须启用从创建代理PDB的CDB的根目录到引用的PDB的位置的连接。数据库链接可以连接到远程CDB的根目录或远程PDB。

  • 如果数据库链接连接到包含引用的PDB的远程CDB中的根，则数据库链接所连接的用户必须是普通用户。

  • 如果数据库链接连接到引用的PDB，则数据库链接在引用的PDB中连接的用户必须具有CREATE PLUGGABLE DATABASE系统特权。

  • 如果要在应用程序容器中创建代理PDB，则以下先决条件适用：

    • 引用的PDB必须是应用程序根目录或应用程序容器中的应用程序PDB。

    • 代理PDB的应用程序容器的应用程序名称和版本必须与引用的PDB的应用程序名称和版本匹配。

    • 在应用程序容器中创建代理PDB时，数据库链接必须启用从创建代理PDB的应用程序容器的根目录到引用的PDB的位置的连接。数据库链接可以连接到远程应用程序容器的根目录或远程应用程序PDB。

    • 如果数据库链接连接到包含引用的PDB的远程应用程序容器中的根，则数据库链接所连接的用户必须是应用程序公共用户。

    • 如果数据库链接连接到引用的应用程序PDB，则数据库链接在引用的应用程序PDB中连接的用户必须具有CREATE PLUGGABLE DATABASE系统特权。

    注意：

    您可以在CDB根目录中创建基于应用程序容器中引用的PDB的代理PDB。

要创建代理PDB：

  1. 在SQL * Plus中，确保当前容器是正在创建代理PDB的CDB根目录或应用程序根目录。

    当前容器是CDB根时，将在CDB中创建代理PDB。当前容器是应用程序根时，将在应用程序容器中创建代理PDB。

  2. 运行CREATE PLUGGABLE DATABASE语句。指定AS PROXY子句，并在FROM子句中使用数据库链接名称指定引用的PDB。在需要时指定其他子句。

    创建代理PDB后，它处于装入模式，其状态为NEW 。您可以通过查询V$PDBS视图中的OPEN_MODE列来查看PDB的打开模式。您可以通过查询CDB_PDBS或DBA_PDBS视图的STATUS列来查看PDB的STATUS 。

    为PDB创建新的默认服务。该服务与PDB具有相同的名称，可用于访问PDB。必须正确配置Oracle Net Services才能使客户端访问此服务。

  3. 以读/写模式打开新PDB。

    您必须在Oracle数据库的读/写模式下打开新的PDB，才能完成新PDB与CDB的集成。如果尝试以只读模式打开PDB，则会返回错误。在读/写模式下打开PDB后，其状态为NORMAL 。

  4. 备份PDB。

    除非备份PDB，否则无法恢复PDB。

注意：

如果在创建代理PDB期间返回错误，则正在创建的PDB可能处于UNUSABLE状态。您可以通过查询CDB_PDBS或DBA_PDBS视图来检查PDB的状态，并通过检查警报日志了解有关PDB创建错误的更多信息。只能删除不可用的PDB，并且必须先删除它，然后才能创建与不可用的PDB同名的PDB。

何时使用基于函数的索引

基于函数的索引计算涉及一个或多个列的表达式的值，并将其存储在索引中。索引表达式可以是算术表达式或包含SQL函数，PL / SQL函数，包函数或C标注的表达式。基于函数的索引还支持基于归类键，SQL语句的高效语言归类和不区分大小写的排序的语言排序。

基于函数的索引可提高使用索引表达式的查询的性能（特别是在表达式计算密集的情况下）。然而：

  • 数据库还必须评估索引表达式以处理不使用它的语句。

  • 经常修改的列上基于函数的索引对于数据库来说维护起来很昂贵。

优化程序只能将基于函数的索引用于基于成本的优化，而它可以使用列上的索引进行基于成本的优化和基于规则的优化。

注意：

  • 基于函数的索引不能包含NULL 。因此，要么确保索引表达式中没有涉及的列可以包含NULL要么使用索引表达式中的NVL函数将另一个值替换为NULL 。

  • Oracle数据库将降序索引视为基于函数的索引。

基于函数的索引的优点

基于函数的索引具有以下优点：

  • 基于函数的索引会增加数据库可以执行索引范围扫描而不是完整索引扫描的情况数。

    当WHERE子句选择少于15％的大型表行时，索引范围扫描通常具有快速响应时间。如果表达式在基于函数的索引中具体化，则优化程序可以更准确地估计表达式选择的行数。

    Oracle数据库将索引表达式表示为虚拟列， ANALYZE语句可以在其上构建直方图。

  • 基于函数的索引预先计算并存储表达式的值。

    查询可以从索引中获取表达式的值，而不是计算它。索引表达式需要的值越多，计算得越多，索引就会提高应用程序性能。

  • 您可以在对象列或REF列上创建基于函数的索引。

    索引表达式可以是返回对象类型的方法的调用。

  • 基于函数的索引可以让您执行更强大的排序。

    对于不区分大小写的排序（如例11-3 ），索引表达式可以调用SQL函数UPPER和LOWER对于基于语言的排序，可以调用SQL函数NLSSORT 。

基于函数的索引的缺点

基于函数的索引具有以下缺点：

  • 优化程序只能将基于函数的索引用于基于成本的优化，而不能用于基于规则的优化。

    基于成本的优化程序使用存储在字典中的统计信息。要收集基于函数的索引的统计信息，请调用DBMS_STATS 。 GATHER_TABLE_STATS或DBMS_STATS 。 GATHER_SCHEMA_STATS 。

  • 在分析索引本身及其定义的表之前，数据库不使用基于函数的索引。

    要分析索引和定义索引的表，请调用DBMS_STATS 。 GATHER_TABLE_STATS或DBMS_STATS 。 GATHER_SCHEMA_STATS 。

  • 进行OR扩展时，数据库不使用基于函数的索引。

  • 您必须确保索引表达式调用的任何模式级或包级PL / SQL函数都是确定性的（即，该函数始终为同一输入返回相同的结果）。

    您必须将该函数声明为DETERMINISTIC ，但由于Oracle数据库不检查此断言，您必须确保该函数确实是确定性的。

    如果更改DETERMINISTIC函数的语义并重新编译它，则必须手动重建任何依赖于函数的索引和物化视图。否则，它们会报告该函数的先前版本的结果。

  • 如果索引表达式是函数调用，则不能约束函数返回类型。

    因为您不能使用NOT NULL约束函数返回类型，所以必须确保使用索引的查询无法获取NULL值。否则，数据库将执行全表扫描。

  • 索引表达式无法调用聚合函数。

  • 位图基于函数的索引不能是降序索引。

  • 索引表达式的数据类型不能是VARCHAR2 ， RAW ， LONGRAW或未知长度的PL / SQL数据类型。

    也就是说，您无法索引未知长度的表达式。但是，您可以索引该表达式的已知长度子字符串。例如：

什么是大物件？

大对象（LOB）是一组旨在容纳大量数据的数据类型。

单个LOB的最大大小可以从8 TB到128 TB不等，具体取决于数据库的配置方式。在LOB中存储数据使您可以在应用程序中有效地访问和操作数据。

为什么要使用大对象？

大对象允许您在多种类型的结构中存储大量数据。

使用大对象的数据类型

大对象适用于半结构化和非结构化数据。

大对象功能允许您将这些类型的数据存储在数据库和从数据库访问的操作系统文件中。

  • 半结构化数据

    半结构化数据具有通常不由数据库解释的逻辑结构，例如，应用程序或外部服务处理的XML文档。Oracle数据库提供了诸如Oracle XML DB，Oracle多媒体和Oracle Spatial and Graph等功能，可帮助您的应用程序使用半结构化数据。

  • 非结构化数据

    非结构化数据很容易分解为较小的逻辑结构，通常不会被数据库或应用程序解释，例如存储为二进制文件的摄影图像。

  • 结构简单的数据

    可以将简单的结构化数据组织到基于业务规则构建的关系表中。

  • 复杂的结构化数据

    复杂的结构化数据比简单的结构化数据更复杂，适用于Oracle数据库的对象关系功能，如集合，引用和用户定义的类型。

随着Internet和内容丰富的应用程序的增长，Oracle数据库必须提供以下LOB支持：

  • 可以高效地存储非结构化和半结构化数据

  • 针对大量数据进行了优化

  • 提供访问存储在数据库中或数据库外部的数据的统一方式

LOB用于半结构化数据

半结构化数据包括文档文件，例如XML文档或文字处理器文件，其包含由应用程序处理或解释的逻辑结构中的数据，并且当存储在数据库中时不分解为较小的逻辑单元。

使用半结构化数据的应用程序通常使用大量字符数据。字符大对象（ CLOB ）和国家字符大对象（ NCLOB ）数据类型是存储和操作此类数据的理想选择。

二进制文件对象（ BFILE数据类型）也可以存储字符数据。您可以使用BFILE将操作系统文件中的只读数据加载到CLOB或NCLOB实例中，然后在应用程序中进行操作。

用于非结构化数据的LOB

非结构化数据是无法分解为标准组件的数据。

这与结构化数据形成对比，例如关于通常包含这些组件的员工的数据：名称，存储为字符串;标识符，例如ID号;薪水;等等。

非结构化数据（例如照片）由1和0的长流组成。这些位用于打开或关闭像素，以便您可以在显示器上看到图像，但这些位不会分解为数据库存储的任何标准组件。

此外，诸如文本，图形图像，静止视频剪辑，全动态视频和声音波形之类的非结构化数据往往尺寸较大。典型的员工记录可能是几百个字节，而即使少量的多媒体数据也可能大几千倍。

适用于大量非结构化二进制数据的SQL数据类型包括BLOB数据类型（二进制大对象）和BFILE数据类型（二进制文件对象）。

为什么不使用LONGs？

Oracle数据库支持LONG和LOB数据类型。但是，LOB提供了下面描述的额外好处。

建议使用LOB数据类型来存储大量结构化和半结构化数据（来自Oracle8 i及以上）。开发用于Oracle7及更早版本的应用程序使用LONG或LONG RAW数据类型来存储大量非结构化数据。

您可以使用LONG -to-LOB迁移轻松迁移访问LONG列的现有应用程序，以使用LOB列。

LOB数据类型相对于LONG和LONG RAW类型的优点：

  • LOB容量：LOB可以存储更大量的数据。LOB可以存储4 GB或更多数据，具体取决于您的系统配置。LONG和LONG RAW类型限制为2 GB的数据。

  • 表中的LOB列数：表可以有多个LOB列。表中的LOB列可以是任何LOB类型。在Oracle7 7.3及更高版本中，表格仅限于一个LONG或LONG RAW列。

  • 随机分段访问：LOB支持随机访问数据，但LONG仅支持顺序访问。

  • LOB也可以是对象属性。

持久和临时LOB

持久和临时LOB都是内部LOB（数据库中的LOB）。

  • 持久性LOB是存在于数据库的表行中的LOB实例。

  • 仅在本地应用程序范围内实例化LOB时，将创建临时LOB实例。

将实例插入表行时，临时实例将成为持久实例。

持久性LOB使用复制语义并参与数据库事务。您可以在事务或介质发生故障时恢复持久性LOB，并且可以提交或回滚对持久性LOB值的任何更改。换句话说，适用于数据库对象的所有原子性，一致性，隔离性和持久性（ACID）属性都适用于持久性LOB。

外部LOB和BFILE数据类型

外部LOB是存储在数据库表空间外部的操作系统文件中的数据对象。

BFILE是数据库用于访问外部LOB的SQL数据类型，是唯一可用于外部LOB的SQL数据类型。

BFILE是只读数据类型。数据库允许只读字节流访问存储在BFILE的数据。您无法在应用程序中写入或更新BFILE 。

数据库使用带有BFILE列的引用语义。存储在BFILE类型的表列中的数据实际上位于操作系统文件中，而不是数据库中。

您通常使用BFILE来保存：

  • 应用程序运行时不会更改的二进制数据，例如图形

  • 加载到其他大对象类型（例如BLOB或CLOB ）中的数据，然后可以在其中操作数据

  • 适合字节流访问的数据，例如多媒体

操作系统访问的任何存储设备都可以保存BFILE数据，包括硬盘驱动器，CD-ROM，PhotoCD和DVD。如果操作系统支持对操作系统文件的流模式访问，则数据库可以访问BFILE 。

注意：

外部LOB不参与交易。必须由操作系统管理的基础文件系统提供对完整性和持久性的任何支持。

LOB定位器

LOB实例具有定位符和值。

LOB定位器是对物理存储LOB值的引用。LOB值是存储在LOB中的数据。

在诸如将LOB作为参数传递的操作中使用LOB时，实际上是在传递LOB定位器。在大多数情况下，您可以在应用程序中使用LOB实例，而无需关心LOB定位器的语义。没有要求取消引用LOB定位器，正如某些编程语言中的指针所要求的那样。
