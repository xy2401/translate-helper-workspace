关于管理端口

在安装过程中，Oracle Universal Installer会从一组默认端口号为组件分配端口号。

许多Oracle数据库组件和服务都使用端口。作为管理员，了解这些服务使用的端口号非常重要，并确保主机上的两个服务不使用相同的端口号。输入以下命令以标识计算机上当前使用的端口：

$ / bin / netstat -a

大多数端口号在安装期间分配。每个组件和服务都有一个分配的端口范围，这是Oracle数据库在分配端口时尝试使用的端口号集。Oracle数据库以范围中的最小数字开头，并执行以下检查：

  • 是否在主机上安装了另一个Oracle数据库的端口？

    安装可能在当时上升或下降。Oracle数据库仍然可以检测是否使用了该端口。

  • 当前正在运行的进程使用的端口是什么？

    这可以是主机上的任何进程，甚至是非Oracle数据库进程。

  • 端口是否在/etc/services文件中列出？

如果上述任何问题的答案为是，则Oracle数据库移动到分配的端口范围中的下一个最高端口，并继续检查，直到找到空闲端口。

$ crsctl status resource resource_name -t

列为TARGET的资源：OFFLINE和STATE：OFFLINE不需要监控。它们表示已注册但未启用的组件，因此它们不使用任何系统资源。如果系统上安装了Oracle产品或组件，并且需要特定资源联机，则软件会提示您激活所需的脱机资源。

网络附加存储和NFS协议

网络附加存储（NAS）系统使用网络文件系统（NFS）协议来访问网络上的文件，这使客户端服务器能够像直接连接到服务器的存储设备一样轻松地通过网络访问文件。您可以在受支持的NFS系统上存储数据文件。NFS是一种共享文件系统协议，因此NFS可以支持单实例和Oracle Real Application Clusters数据库。

注意：

存储在NAS设备上的Oracle软件和数据库的性能取决于服务器与网络连接存储设备之间的网络连接性能。为了获得更好的性能，Oracle建议您使用专用的专用网络连接将服务器连接到NAS设备。NFS网络连接应使用千兆以太网或更好。

请参阅供应商文档以完成NFS配置和安装。

使用NFS存储的要求

在开始安装之前，必须装入NFS文件系统并使其可供服务器使用。

术语升级和迁移的定义

升级和迁移是不同类型的数据库更改。

升级将现有Oracle数据库环境（包括已安装的组件和关联的应用程序）转换为新版本的Oracle数据库环境。数据库的数据字典已升级到新版本。升级不会直接影响用户数据;升级期间不会触摸，更改或移动任何数据。

迁移数据是指将数据从一个Oracle数据库移动到先前为迁移或移动数据而创建的另一个数据库。当您需要将数据库环境移动到新的硬件或操作系统平台或新的字符集时，可以迁移数据。迁移不包括升级到最新版本。迁移数据后，将单独处理升级过程。

除非另有说明，否则“Oracle数据库升级指南 ”中的升级步骤适用于所有操作系统。某些操作系统可能需要其他升级步骤。

9.2创建数据库表

规划数据库模式后，为模式计划创建数据库表。如下定义Customers表：

CREATE TABLE客户（CustNo NUMBER（3）NOT NULL，CustName VARCHAR2（30）NOT NULL，Street VARCHAR2（20）NOT NULL，City VARCHAR2（20）NOT NULL，State CHAR（2）NOT NULL，Zip VARCHAR2（10）NOT NULL，Phone VARCHAR2（12），PRIMARY KEY（CustNo））;

Customers表存储有关客户的信息。基本信息定义为NOT NULL 。例如，每个客户都必须有一个送货地址。但是， Customers表不管理客户与其采购订单之间的关系。因此，此关系必须由Orders表管理，您可以按如下方式定义：

CREATE TABLE订单（PONo NUMBER（5），Custno NUMBER（3）REFERENCES Customers，OrderDate DATE，ShipDate DATE，ToStreet VARCHAR2（20），ToCity VARCHAR2（20），ToState CHAR（2），ToZip VARCHAR2（10），PRIMARY KEY （PONo））;

订单项与采购订单和库存商品有关系。LineItems表使用外键管理这些关系。例如， StockNo的外键列LineItems表引用StockNo在主键列StockItems表，您可以定义如下：

CREATE TABLE StockItems（StockNo NUMBER（4）PRIMARY KEY，Description VARCHAR2（20），Price NUMBER（6,2）））;

Orders表使用CustNo外键列管理客户和采购订单之间的关系，该列引用Customers表中的CustNo主键列。但是， Orders表不管理采购订单与其订单项之间的关系。因此，此关系必须由LineItems表管理，您可以按如下方式定义该表：

CREATE TABLE LineItems（LineNo NUMBER（2），PONo NUMBER（5）REFERENCES Orders，StockNo NUMBER（4）REFERENCES StockItems，Quantity NUMBER（2），Discount NUMBER（4,2），PRIMARY KEY（LineNo，PONo））;

8.4从PL / SQL调用Java

您可以从任何PL / SQL块，子程序或包调用Java存储过程。例如，假设以下Java类的可执行文件存储在Oracle数据库中：

import java.sql。*; import oracle.jdbc。*; public class Adjuster {public static void raiseSalary（int empNo，float percent）throws SQLException {Connection conn = DriverManager.getConnection（“jdbc：default：connection：”）; String sql =“UPDATE员工SET薪水=薪水*？WHERE employee_id =？“; try {PreparedStatement pstmt = conn.prepareStatement（sql）; pstmt.setFloat（1，（1 + percent / 100））; pstmt.setInt（2，empNo）; pstmt.executeUpdate（）; pstmt.close（）; } catch（SQLException e）{System.err.println（e.getMessage（））;}}}

关于管理端口

在安装过程中，Oracle Universal Installer会从一组默认端口号为组件分配端口号。

许多Oracle数据库组件和服务都使用端口。作为管理员，了解这些服务使用的端口号非常重要，并确保主机上的两个服务不使用相同的端口号。输入以下命令以标识计算机上当前使用的端口：

$ / bin / netstat -a

大多数端口号在安装期间分配。每个组件和服务都有一个分配的端口范围，这是Oracle数据库在分配端口时尝试使用的端口号集。Oracle数据库以范围中的最小数字开头，并执行以下检查：

  • 是否在主机上安装了另一个Oracle数据库的端口？

    安装可能在当时上升或下降。Oracle数据库仍然可以检测是否使用了该端口。

  • 当前正在运行的进程使用的端口是什么？

    这可以是主机上的任何进程，甚至是非Oracle数据库进程。

  • 端口是否在/etc/services文件中列出？

如果上述任何问题的答案为是，则Oracle数据库移动到分配的端口范围中的下一个最高端口，并继续检查，直到找到空闲端口。

检查直接NFS客户端的TCP网络协议缓冲区

检查TCP网络缓冲区大小，确保它足以满足服务器的速度。

默认情况下，TCP的网络缓冲区大小设置为1 MB，UDP的网络缓冲区大小设置为2 MB。 TCP缓冲区大小可以设置文件传输限制，这可能会对Direct NFS客户端用户的性能产生负面影响。

要检查当前的TCP缓冲区大小，请输入以下命令：

#sysctl -a | grep -e net.ipv4.tcp_ [rw] mem

此命令的输出类似于以下内容：

net.ipv4.tcp_rmem = 4096 87380 1056768 net.ipv4.tcp_wmem = 4096 16384 1056768

Oracle建议您根据服务器的链接速度设置值。例如，执行以下步骤：

  1. 以root身份，使用文本编辑器打开/etc/sysctl.conf ，并添加或更改以下内容：

    net.ipv4.tcp_rmem = 4096 87380 4194304 net.ipv4.tcp_wmem = 4096 16384 4194304

  2. 运行以下命令应用您的更改：

    #sysctl -p

  3. 重启网络：

    ＃/ etc / rc.d / init.d / network restart

30.1关于转换

了解如何使用自动数据准备（ADP）和嵌入式数据转换来转换数据。

转换是一种SQL表达式，用于修改一列或多列中的数据。数据通常必须经过某些转换才能用于构建模型。许多数据挖掘算法都有特定的转换要求。在对数据进行评分之前，必须以与转换培训数据相同的方式对其进行转换。

Oracle Data Mining支持自动数据准备（ADP），它自动实现算法所需的转换。转换嵌入在模型中，并在应用模型时自动执行。

如果需要其他转换，则可以将它们指定为SQL表达式，并在创建模型时将它们作为输入提供。这些转换与ADP一样嵌入到模型中。

通过自动和嵌入式数据转换，您可以处理大部分数据准备工作。您只需几个步骤即可创建模型并对多个数据集进行评分：

  1. 确定要包含在案例表中的列。

  2. 如果要包含事务数据，请创建嵌套列。

  3. 为任何未由ADP处理的转换编写SQL表达式。

  4. 创建模型，提供SQL表达式（如果指定）并标识包含文本数据的任何列。

  5. 确保评分数据中的部分或全部列与用于训练模型的列具有相同的名称和类型。

使用密码响应文件运行安装后配置

完成此过程以使用configToolAllCommands脚本运行配置助手。

  1. 创建密码响应文件，如创建密码文件中所述 。
  2. 将目录更改为$ORACLE_HOME/cfgtoollogs 。
  3. 使用以下语法运行配置脚本：
    configToolAllCommands RESPONSE_FILE = / path / name .properties

    例如：

    $ ./configToolAllCommands RESPONSE_FILE = / home / oracle / pwdrsp.properties
