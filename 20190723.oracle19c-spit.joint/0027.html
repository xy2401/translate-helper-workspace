关于磁盘扫描订购

ORACLEASM_SCANORDER变量指定要首先扫描的磁盘。

ASMLIB配置文件位于路径/etc/sysconfig/oracleasm 。它包含您使用命令/etc/init.d/oracleasm configure指定的所有启动/etc/init.d/oracleasm configure 。该命令无法配置扫描顺序。

配置文件包含许多配置变量。ORACLEASM_SCANEXCLUDE变量指定要忽略的磁盘。

使用以空格分隔的前缀字符串配置ORACLEASM_SCANORDER值。前缀字符串是与某种类型的磁盘相关联的公共字符串。例如，如果使用前缀字符串sd ，则此字符串匹配所有SCSI设备，包括/dev/sda ， /dev/sdb ， /dev/sdc等。请注意，这些不是全球。他们不使用外卡。它们是简单的前缀。另请注意，路径不是前缀的一部分。例如， /dev/ path不是路径/dev/sd* SCSI磁盘前缀的一部分。

对于Oracle Linux和Red Hat Enterprise Linux，在扫描时，内核会将设备视为/dev/mapper/ XXX条目。默认情况下，设备文件命名方案udev为人类可读性创建/dev/mapper/ XXX名称。使用ORACLEASM_SCANORDER任何配置都应使用/dev/mapper/ XXX条目。

手动设置UDP和TCP内核参数

如果不使用Fixup脚本或CVU设置临时端口，则设置TCP / IP临时端口范围参数，以便为预期的服务器工作负载提供足够的临时端口。

确保将较低范围设置为至少9000或更高，以避免Well Known端口，并避免Oracle和其他服务器端口常用的Registered Ports范围内的端口。将端口范围设置得足够高，以避免为您打算使用的任何应用程序保留端口。如果您所拥有的范围的较低值大于9000，并且该范围足以满足您的预期工作负载，那么您可以忽略有关临时端口范围的Oracle Universal Installer警告。

在Oracle Solaris 11上，使用ipadm命令检查临时端口的当前范围：
#ipad show-prop -p smallest_anon_port，largest_anon_port tcp PROTO PROPERTY PERM CURRENT PERSISTENT DEFAULT POSSIBLE tcp smallest_anon_port rw 32768  -  32768 1024-65535 tcp largest_anon_port rw 65500  -  65535 32768-65535

在前面的示例中，临时端口设置为默认范围（32768-65535）。

如果您的预期工作负载或服务器数量有必要，请将UDP和TCP临时端口范围更新到更宽的范围。例如：

在Oracle Solaris 11上：
#ipad set-prop -p smallest_anon_port = 9000 tcp #ipadm set-prop -p largest_anon_port = 65500 tcp #ipadm set-prop -p smallest_anon_port = 9000 udp #ipadm set-prop -p largest_anon_port = 65500 udp

Oracle建议您永久保留这些设置。有关如何在系统重新启动时自动执行此临时端口范围更改的信息，请参阅系统管理文档。

目的

DATAOBJ_TO_MAT_PARTITION仅对正在对用于存储域索引数据的系统分区表执行数据维护或查询操作的Data Cartridge开发人员有用。DML或查询操作由域索引的基表上的相应操作触发。

使用DBMS_STATS重新收集固定对象统计信息

升级后或其他数据库配置更改后，Oracle强烈建议您在Oracle数据库上运行代表性工作负载后重新收集固定对象统计信息。

固定对象是X$表及其索引。V$性能视图通过X$表定义。收集固定对象统计信息对于数据库性能很有价值，因为这些统计信息可帮助优化器生成良好的执行计划，从而提高数据库性能。未能获得具有代表性的统计数据可能会导致执行计划不理想，从而导致严重的性能问题。

使用DBMS_STATS.GATHER_FIXED_OBJECTS_STATS PL / SQL过程收集固定对象统计信息。DBMS_STATS.GATHER_FIXED_OBJECTS_STATS还显示有关从INIT.ORA或SPFILE删除所有隐藏或下划线参数和事件的建议。

由于X$表的瞬态特性，当系统上存在代表性工作负载时，您必须收集固定对象统计信息。如果在高峰加载期间无法收集固定对象统计信息，则Oracle建议您在系统处于运行时状态后执行此操作，并填充最重要类型的固定对象表。

要收集固定对象的统计信息，请运行以下PL / SQL过程：

SQL> execute dbms_stats.gather_fixed_objects_stats;

手动设置UDP和TCP内核参数

如果不使用Fixup脚本或CVU设置临时端口，则设置TCP / IP临时端口范围参数，以便为预期的服务器工作负载提供足够的临时端口。

确保将较低范围设置为至少9000或更高，以避免Well Known端口，并避免Oracle和其他服务器端口常用的Registered Ports范围内的端口。将端口范围设置得足够高，以避免为您打算使用的任何应用程序保留端口。如果您所拥有的范围的较低值大于9000，并且该范围足以满足您的预期工作负载，那么您可以忽略有关临时端口范围的Oracle Universal Installer警告。

例如，使用IPv4，使用以下命令检查临时端口的当前范围：
$ cat / proc / sys / net / ipv4 / ip_local_port_range 32768 61000

在前面的示例中，最低端口（32768）和最高端口（61000）设置为默认范围。

如有必要，请将UDP和TCP临时端口范围更新到足以满足预期系统工作负载的范围，并确保短暂端口范围从9000及更高开始。例如：
#echo 9000 65500> / proc / sys / net / ipv4 / ip_local_port_range
Oracle建议您永久保留这些设置。例如，以root身份使用文本编辑器打开/etc/sysctl.conf ，并添加或更改为以下内容： net.ipv4.ip_local_port_range = 9000 65500 ，然后重新启动网络：
＃/ etc / rc.d / init.d / network restart

有关在系统重新启动时自动执行临时端口范围更改的信息，请参阅Linux分发系统管理文档。

关于Oracle数据库的默认SGA权限的更改

从Oracle Database 12c第2版（12.2.0.1）开始，默认情况下，读取和写入系统全局区域（SGA）的权限仅限于Oracle软件安装所有者。

Oracle数据库初始化参数ALLOW_GROUP_ACCESS_TO_SGA确定Oracle数据库安装所有者帐户（Oracle文档示例中的oracle ）是否是唯一可以读取和写入数据库系统全局区域（SGA）的用户，或者OSDBA组的成员是否可以读取SGA。在Oracle Database 12c第2版（12.2）和更高版本中，此参数的默认值为FALSE，因此只有Oracle数据库安装所有者具有对SGA的读写权限。默认情况下，将删除对SGA的组访问权限。此更改会影响所有Linux和UNIX平台。

如果OSDBA组的成员需要对SGA的读访问权限，则可以将初始化参数ALLOW_GROUP_ACCESS_TO_SGA设置从FALSE更改为TRUE。 Oracle强烈建议您接受将SGA访问权限限制为oracle用户帐户的默认权限。

关于Oracle数据库的默认SGA权限的更改

从Oracle Database 12c第2版（12.2.0.1）开始，默认情况下，读取和写入系统全局区域（SGA）的权限仅限于Oracle软件安装所有者。

在以前的版本中，Oracle安装所有者帐户和OSDBA组的成员都可以访问共享内存。Oracle Database 12c第2版（12.2）及更高版本中的更改将默认情况下限制为Oracle安装所有者帐户，这提供了比以前的配置更高的安全性。但是，此更改可能会阻止无权访问Oracle安装所有者帐户的DBA管理数据库。

Oracle数据库初始化参数ALLOW_GROUP_ACCESS_TO_SGA确定Oracle数据库安装所有者帐户（Oracle文档示例中的oracle ）是否是唯一可以读取和写入数据库系统全局区域（SGA）的用户，或者OSDBA组的成员是否可以读取SGA。在Oracle Database 12c第2版（12.2）和更高版本中，此参数的默认值为FALSE，因此只有Oracle数据库安装所有者具有对SGA的读写权限。默认情况下，将删除对SGA的组访问权限。此更改会影响所有Linux和UNIX平台。

如果OSDBA组的成员需要对SGA的读访问权限，则可以将初始化参数ALLOW_GROUP_ACCESS_TO_SGA设置从FALSE更改为TRUE。 Oracle强烈建议您接受将SGA访问权限限制为oracle用户帐户的默认权限。

手动设置UDP和TCP内核参数

如果不使用Fixup脚本或CVU设置临时端口，则设置TCP / IP临时端口范围参数，以便为预期的服务器工作负载提供足够的临时端口。

确保将较低范围设置为至少9000或更高，以避免Well Known端口，并避免Oracle和其他服务器端口常用的Registered Ports范围内的端口。将端口范围设置得足够高，以避免为您打算使用的任何应用程序保留端口。如果您所拥有的范围的较低值大于9000，并且该范围足以满足您的预期工作负载，那么您可以忽略有关临时端口范围的Oracle Universal Installer警告。

例如，使用IPv4，使用以下命令检查临时端口的当前范围：
$ cat / proc / sys / net / ipv4 / ip_local_port_range 32768 61000

在前面的示例中，最低端口（32768）和最高端口（61000）设置为默认范围。

#echo 9000 65500> / proc / sys / net / ipv4 / ip_local_port_range

排序查询结果

使用ORDER BY子句对查询选择的行进行排序。按位置排序在以下情况下很有用：

  • 要按冗长的选择列表表达式排序，可以在ORDER BY子句中指定其位置，而不是复制整个表达式。

  • 对于包含集合运算符UNION ， INTERSECT ， MINUS或UNION ALL复合查询， ORDER BY子句必须指定位置或别名而不是显式表达式。此外， ORDER BY子句只能出现在最后一个组件查询中。ORDER BY子句对整个复合查询返回的所有行进行排序。

Oracle数据库对ORDER BY子句的字符值进行排序的机制（也称为排序规则）由NLS_SORT会话参数指定。如果未设置此参数，则其默认值从NLS_LANGUAGE会话参数派生。您可以使用ALTER SESSION SET NLS_SORT语句动态更改排序NLS_SORT 。您还可以通过将要作为参数排序的字符表达式包含在NLSSORT函数中来应用特定的排序NLSSORT ，并在第二个参数中指定排序规则。

当字符值在语言上与ORDER BY子句进行比较时，它们首先被转换为校对键，然后像RAW值一样进行比较。排序键是按照NLSSORT指定显式生成的，或者是使用与NLSSORT使用的方法相同的隐式生成的。显式和隐式生成的排序规则键都受“ NLSSORT ”中描述的相同限制。作为这些限制的结果，如果两个值在用于产生校对键的前缀上没有差异，则它们可以在语言上相等，即使它们在值的其余部分中不同。

手动设置UDP和TCP内核参数

如果不使用Fixup脚本或CVU设置临时端口，则设置TCP / IP临时端口范围参数，以便为预期的服务器工作负载提供足够的临时端口。

确保将较低范围设置为至少9000或更高，以避免Well Known端口，并避免Oracle和其他服务器端口常用的Registered Ports范围内的端口。将端口范围设置得足够高，以避免为您打算使用的任何应用程序保留端口。如果您所拥有的范围的较低值大于9000，并且该范围足以满足您的预期工作负载，那么您可以忽略有关临时端口范围的Oracle Universal Installer警告。

在Oracle Solaris 11上，使用ipadm命令检查临时端口的当前范围：
#ipad show-prop -p smallest_anon_port，largest_anon_port tcp PROTO PROPERTY PERM CURRENT PERSISTENT DEFAULT POSSIBLE tcp smallest_anon_port rw 32768  -  32768 1024-65535 tcp largest_anon_port rw 65500  -  65535 32768-65535

在前面的示例中，临时端口设置为默认范围（32768-65535）。

如果您的预期工作负载或服务器数量有必要，请将UDP和TCP临时端口范围更新到更宽的范围。例如：

在Oracle Solaris 11上：
#ipad set-prop -p smallest_anon_port = 9000 tcp #ipadm set-prop -p largest_anon_port = 65500 tcp #ipadm set-prop -p smallest_anon_port = 9000 udp #ipadm set-prop -p largest_anon_port = 65500 udp

Oracle建议您永久保留这些设置。有关如何在系统重新启动时自动执行此临时端口范围更改的信息，请参阅系统管理文档。

关于Oracle RAC的ORATAB配置

Oracle在oratab配置文件中为每个Oracle RAC数据库创建一个条目。

oratab文件由安装期间的root.sh脚本创建，并在创建或删除数据库时由Database Configuration Assistant更新。

如果节点固定在CSS中，Oracle ASM代理会创建Oracle ASM oratab条目。如果您的系统满足以下任一条件，则首次在先前未运行的节点上启动数据库时，数据库代理也会自动创建oratab文件条目：
  • 数据库版本是Oracle Database 12c第1版（12.1）或更早版本
  • 该数据库是单个实例Oracle数据库
  • 该数据库是管理员管理的Oracle RAC数据库

Oracle Enterprise Manager在服务发现期间使用oratab文件来确定Oracle RAC数据库的名称，并确定在重新启动系统时是否必须自动启动数据库。

数据库条目具有以下语法：

$ DB_UNIQUE_NAME：$ ORACLE_HOME：N

冒号（:)用作字段终止符。新行终止条目。以井号（＃）开头的行是注释。由于Oracle RAC数据库的所有实例都具有相同的DB_UNIQUE_NAME ，但每个实例都有自己的ORACLE_SID ，因此请使用oratab文件中的$DB_UNIQUE_NAME环境变量作为数据库条目。

Oracle RAC数据库的$DB_UNIQUE_NAME标识符在整个企业中必须是唯一的。$ORACLE_HOME是数据库的目录路径， N表示不应在重启时启动数据库。以下是名为sales的数据库的示例条目：

销售：/ U01 /应用/ ORACLE /销售：N
