使用响应文件运行Oracle DBCA

您可以在响应文件模式下运行Oracle数据库配置助手（Oracle DBCA），以在系统上配置和启动Oracle数据库。

要在响应文件模式下运行Oracle DBCA，您必须复制和编辑响应文件模板。Oracle在ORACLE_HOME/assistants/dbca目录中提供名为dbca.rsp的响应文件模板。要在响应文件模式下运行Oracle DBCA，必须将-responseFile标志与-silent标志结合使用。您还必须使用图形显示并设置DISPLAY环境变量。

要以响应文件模式运行Oracle DBCA：

  1. 将dbca.rsp响应文件模板从响应文件目录复制到系统上的目录：
    $ cp / directory_path /assistants/dbca/dbca.rsp local_directory

    在此示例中， directory_path是您复制安装二进制文件的目录的路径。

    作为编辑响应文件模板的替代方法，您还可以通过在运行Oracle DBCA时将所有必需信息指定为命令行选项来创建数据库。有关支持的选项列表的信息，请输入以下命令：

    $ $ ORACLE_HOME / bin / dbca -help
  2. 在文本编辑器中打开响应文件：
    $ vi / local_dir /dbca.rsp
  3. 按照文件中的说明编辑文件。

    注意：

    如果未正确配置响应文件，Oracle DBCA将失败。

  4. 以Oracle软件所有者用户身份登录，并设置ORACLE_HOME环境变量以指定正确的Oracle主目录。
  5. 要在响应文件模式下运行Oracle DBCA，请设置DISPLAY环境变量。
  6. 使用以下命令语法使用响应文件以静默或响应文件模式运行Oracle DBCA：
    $ ORACLE_HOME / bin / dbca [-silent] -responseFile / local_dir /dbca.rsp

    在这个例子中：

    • -silent选项表示Oracle DBCA以静默方式运行。

    • local_dir是复制dbca.rsp响应文件模板的目录的完整路径。

    在配置期间，Oracle DBCA显示一个包含状态消息和进度条的窗口。

listener.ora文件是listener.ora器的配置文件。它可以包括接受连接请求的协议地址，数据库列表以及它正在侦听的其他服务，以及监听器使用的控制参数。

您可以使用服务器控制实用程序（SRVCTL）命令或使用NETCA修改Oracle Clusterware和Oracle RAC使用的侦听器的配置。不需要手动编辑listener.ora文件。

每个侦听器都配置有一个或多个指定其侦听端点的协议地址。侦听器代理使用侦听器动态更新端点。从Oracle Database 11g第2版开始， listener.ora文件现在只包含IPC密钥和以下信息：

（ADDRESS =（PROTOCOL = TCP）（HOST =）（PORT = 1521））

用于作业角色分离的扩展Oracle数据库组

Oracle Database 12c第1版（12.1）及更高版本提供了一组扩展的数据库组，用于为数据库管理授予特定于任务的系统特权。

扩展的Oracle数据库系统特权组是特定于任务的，并且权限低于OSDBA / SYSDBA系统特权。它们旨在提供执行日常数据库操作的权限。授予这些系统权限的用户也可通过操作系统组成员身份获得授权

您不必创建这些特定的组名，但在交互式和静默安装期间，必须分配其成员被授予对这些系统特权的访问权限的操作系统组。您可以指定同一组来为这些权限提供授权，但Oracle建议您提供一个唯一的组来指定每个权限。

OSDBA作业角色分离权限和组的子集包含以下内容：

  • Oracle数据库的OSBACKUPDBA组（通常是backupdba ）

    如果希望单独的一组操作系统用户具有一组有限的数据库备份和恢复相关管理权限（SYSBACKUP权限），请创建此组。

  • 用于Oracle Data Guard的OSDGDBA组（通常为dgdba ）

    如果您希望一组独立的操作系统用户拥有一组有限的权限来管理和监视Oracle Data Guard（SYSDG权限），请创建此组。要使用此权限，请将Oracle数据库安装所有者添加为该组的成员。

  • 用于加密密钥管理的OSKMDBA组（通常为kmdba ）

    如果您希望一组独立的操作系统用户具有一组有限的加密密钥管理权限（如Oracle Wallet Manager管理（SYSKM权限）），请创建此组。要使用此权限，请将Oracle数据库安装所有者添加为该组的成员。

  • 适用于Oracle Real Application Clusters Administration的OSRACDBA组（通常为racdba ）

    如果您希望一组独立的操作系统用户拥有一组有限的Oracle Real Application Clusters（RAC）管理权限（SYSRAC权限），请创建此组。要使用此权限：

    • 将Oracle数据库安装所有者添加为该组的成员。

    • 对于Oracle Restart配置，如果您有单独的Oracle Grid Infrastructure安装所有者用户（ grid ），则还必须将grid用户添加为数据库的OSRACDBA组的成员，以使Oracle Grid Infrastructure组件能够连接到数据库。

使用子查询

子查询回答了多部分问题。例如，要确定谁在Taylor的部门工作，您可以先使用子查询来确定Taylor工作的部门。然后，您可以使用父SELECT语句回答原始问题。SELECT语句的FROM子句中的子查询也称为内联视图 。您可以在内联视图中嵌套任意数量的子查询。SELECT语句的WHERE子句中的子查询也称为嵌套子查询 。您可以在嵌套子查询中嵌套最多255个子查询级别。

子查询可以包含另一个子查询。Oracle数据库对顶级查询的FROM子句中的子查询级别数量没有限制。您可以在WHERE子句中嵌套最多255个子查询级别。

如果子查询中的列与contains语句中的列具有相同的名称，则必须使用包含表名或别名的contains语句对表的列的任何引用作为前缀。要使语句更易于阅读，请始终使用表，视图或实例化视图的名称或别名限定子查询中的列。

当嵌套子查询从引用父语句的表引用子查询或嵌套子查询之上的一个或多个级别时，Oracle会执行相关子查询。父语句可以是SELECT ， UPDATE或DELETE语句，子查询嵌套在该语句中。对于父语句处理的每一行，概念上对相关子查询进行一次计算。但是，优化器可以选择将查询重写为连接，或者使用其他技术来制定语义上等效的查询。Oracle通过查看子查询中指定的表，然后查找父语句中指定的表，解析子查询中的非限定列。

相关子查询回答多部分问题，其答案取决于父语句处理的每一行中的值。例如，您可以使用相关子查询来确定哪些员工的收入高于其部门的平均工资。在这种情况下，相关子查询专门计算每个部门的平均工资。

将子查询用于以下目的：

  • 定义要插入INSERT或CREATE TABLE语句的目标表的行集

  • 在CREATE VIEW或CREATE MATERIALIZED VIEW语句中定义要包含在视图或物化视图中的行集

  • 定义要分配给UPDATE语句中现有行的一个或多个值

  • 为WHERE子句， HAVING子句或SELECT ， UPDATE和DELETE语句的START WITH子句中的条件提供值

用于作业角色分离的扩展Oracle数据库组

Oracle Database 12c第1版（12.1）及更高版本提供了一组扩展的数据库组，用于为数据库管理授予特定于任务的系统特权。

扩展的Oracle数据库系统特权组是特定于任务的，并且权限低于OSDBA / SYSDBA系统特权。它们旨在提供执行日常数据库操作的权限。授予这些系统权限的用户也可通过操作系统组成员身份获得授权

您不必创建这些特定的组名，但在交互式和静默安装期间，必须分配其成员被授予对这些系统特权的访问权限的操作系统组。您可以指定同一组来为这些权限提供授权，但Oracle建议您提供一个唯一的组来指定每个权限。

OSDBA作业角色分离权限和组的子集包含以下内容：

  • Oracle数据库的OSBACKUPDBA组（通常是backupdba ）

    如果希望单独的一组操作系统用户具有一组有限的数据库备份和恢复相关管理权限（SYSBACKUP权限），请创建此组。

  • 用于Oracle Data Guard的OSDGDBA组（通常为dgdba ）

    如果您希望一组独立的操作系统用户拥有一组有限的权限来管理和监视Oracle Data Guard（SYSDG权限），请创建此组。要使用此权限，请将Oracle数据库安装所有者添加为该组的成员。

  • 用于加密密钥管理的OSKMDBA组（通常为kmdba ）

    如果您希望一组独立的操作系统用户具有一组有限的加密密钥管理权限（如Oracle Wallet Manager管理（SYSKM权限）），请创建此组。要使用此权限，请将Oracle数据库安装所有者添加为该组的成员。

  • 适用于Oracle Real Application Clusters Administration的OSRACDBA组（通常为racdba ）

    如果您希望一组独立的操作系统用户拥有一组有限的Oracle Real Application Clusters（RAC）管理权限（SYSRAC权限），请创建此组。要使用此权限：

    • 将Oracle数据库安装所有者添加为该组的成员。

Oracle数据库网关概述

Oracle数据库网关提供了从Oracle环境透明地访问驻留在非Oracle系统中的数据的功能。

异构数据访问是一个影响很多公司的问题。许多公司运行几个不同的数据库系统。这些系统中的每一个都存储数据，并具有一组针对它运行的应用程序。在一个数据库系统中合并这些数据通常很难，很大程度上是因为许多针对一个数据库运行的应用程序可能没有与另一个数据库运行的等效项。在迁移到一个统一数据库系统变得可行之前，各种异构数据库系统必须进行互操作。

Oracle数据库网关提供了从Oracle环境透明地访问驻留在非Oracle系统中的数据的功能。这种透明性消除了应用程序开发人员定制其应用程序以访问来自不同非Oracle系统的数据的需要，从而减少了开发工作并提高了应用程序的移动性。可以使用Oracle和Informix的一致Oracle接口开发应用程序。

网关技术由两部分组成：具有连接到非Oracle系统的通用技术的组件，所有非Oracle系统都通用，称为异构服务，以及特定于非Oracle的组件网关连接的系统。异构服务与Oracle Database Gateway代理结合使用，可以从Oracle环境透明地访问非Oracle系统。

关于异构服务技术

异构服务提供了连接到非Oracle系统的通用技术。

作为数据库的集成组件，异构服务可以利用数据库的功能，例如强大的SQL解析和分布式优化功能。

异构服务扩展了Oracle SQL引擎，以识别远程非Oracle系统的SQL和过程功能以及获取必要数据字典信息所需的映射。异构服务提供两种类型的转换：将Oracle SQL转换为非Oracle系统的正确方言的能力，以及以本地格式显示非Oracle系统的元数据的数据字典转换。对于没有可用转换的情况，可以使用异构服务的传递功能向非Oracle系统发出本机SQL。

异构服务还维护Oracle与远程非Oracle系统之间的事务协调，例如提供两阶段提交协议以确保分布式事务完整性，即使对于本身不支持两阶段提交的非Oracle系统也是如此。

市场篮子数据通常是交易性的 。在交易中数据，案例是事务，事务的数据存储在多行中。Oracle Data Mining关联模型可以基于事务数据或单记录案例数据构建。ODMS_ITEM_ID_COLUMN_NAME和ODMS_ITEM_VALUE_COLUMN_NAME设置指定关联规则的数据是否为事务格式。

注意：

关联模型是可以在本机事务数据上构建的唯一模型类型。对于所有其他类型的模型，Oracle Data Mining要求数据以单记录格式呈现。

Apriori算法假设数据是事务性的，并且它有许多缺失值。Apriori将所有缺失值解释为稀疏数据，并且它有自己的处理稀疏数据的本机机制。

如果您拥有经过认证的NAS存储设备，则可以在NFS安装目录中创建零填充文件，并将这些文件用作Oracle ASM磁盘组中的磁盘设备。

确保为Oracle ASM磁盘指定ASM发现路径。
在安装Oracle Grid Infrastructure 19c期间，Oracle Universal Installer（OUI）可以在您指定的NFS安装目录中创建文件。以下过程说明如何在NFS安装目录中手动创建文件以用作Oracle ASM磁盘组中的磁盘设备：
  1. 如有必要，请在NAS设备上为磁盘组文件创建导出目录。
  2. 将用户切换到root 。
  3. 在本地系统上创建挂载点目录。

    例如：

    #mkdir -p / mnt / oracleasm
  4. 要确保在系统重新启动时装入NFS文件系统，请在装入文件/etc/fstab为文件系统添加一个条目。
  5. 输入类似于以下内容的命令，以在本地系统上挂载NFS：
    #mount / mnt / oracleasm
  6. 选择要创建的磁盘组的名称，并使用磁盘组名称作为目录名称为NFS文件系统上的文件创建目录。
    例如，如果要为销售数据库设置磁盘组：
    #mkdir / mnt / oracleasm / sales1
  7. 使用类似于以下命令在此目录中创建所需数量的零填充文件：
    #dd if = / dev / zero of = / mnt / oracleasm / sales1 / disk1 bs = 1024k count = 1000

    此示例在NFS文件系统上创建1 GB文件。您必须分别创建一个，两个或三个文件以创建外部，普通或高冗余磁盘组。

    在同一NAS设备上创建多个零填充文件无法防止NAS故障。而是为每个NAS设备创建一个文件，并使用Oracle ASM技术对其进行镜像。

    #chown -R grid：asmadmin / mnt / oracleasm＃chmod -R 660 / mnt / oracleasm
    在此示例中，安装所有者是grid ，OSASM组是asmadmin 。
  9. 在Oracle数据库安装期间，编辑Oracle ASM磁盘发现字符串以指定与您创建的文件名匹配的正则表达式。

  • 导航访问

  • Oracle数据库的OSBACKUPDBA组（通常是backupdba ）

  • 用于Oracle Data Guard的OSDGDBA组（通常为dgdba ）

关于Oracle成员集群

Oracle成员集群使用Oracle域服务集群中的集中服务，可以托管数据库或应用程序。Oracle成员集群可以有两种类型 - Oracle数据库的Oracle成员集群或应用程序的Oracle成员集群。

Oracle成员群集不需要直接连接到共享磁盘。使用共享的Oracle ASM服务，他们可以利用与IO服务或ACFS远程服务的网络连接来访问集中管理的存储池。要使用Oracle域服务群集中的共享Oracle ASM服务，成员群集需要连接到Oracle域服务群集的Oracle ASM网络。

Oracle成员集群无法为其他集群提供服务。例如，您无法将成员集群配置和使用为GNS服务器或Oracle Fleet Patching and Provisioning Server。

适用于Oracle数据库的Oracle成员集群

Oracle数据库的Oracle成员集群支持Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库实例。此群集向管理存储库服务注册并使用集中式TFA服务。它可以根据需要使用其他服务。可以使用本地Oracle ASM存储管理配置Oracle Oracle数据库成员集群，也可以使用Oracle域服务集群提供的整合Oracle ASM存储管理服务。

Oracle数据库的Oracle成员集群始终使用其Oracle域服务集群中的远程网格基础结构管理存储库（GIMR）。对于双节点或四节点群集，在远程群集上托管GIMR可减少在群集上运行额外基础架构存储库的开销。

Oracle成员集群应用程序

Oracle成员集群应用程序托管Oracle数据库以外的应用程序，作为Oracle集群域的一部分。Oracle成员集群需要连接到Oracle集群域服务，以实现集中管理和资源效率。Oracle成员集群使用远程Oracle ASM存储，通过Oracle ACFS Remote服务提供任何所需的共享存储。此群集配置可实现任何软件应用程序的高可用性。

与需要公共和专用网络互连的其他群集配置不同，Oracle成员群集应用程序可以配置为使用单个公共网络接口。

Oracle成员集群Oracle数据库不支持Oracle Database 12.1或更早版本，其中Oracle成员集群使用Oracle ASM存储直接或间接配置。

关于Oracle成员集群

Oracle成员集群使用Oracle域服务集群中的集中服务，可以托管数据库或应用程序。Oracle成员集群可以有两种类型 - Oracle数据库的Oracle成员集群或应用程序的Oracle成员集群。

Oracle成员群集不需要直接连接到共享磁盘。使用共享的Oracle ASM服务，他们可以利用与IO服务或ACFS远程服务的网络连接来访问集中管理的存储池。要使用Oracle域服务群集中的共享Oracle ASM服务，成员群集需要连接到Oracle域服务群集的Oracle ASM网络。

Oracle成员集群无法为其他集群提供服务。例如，您无法将成员集群配置和使用为GNS服务器或Oracle Fleet Patching and Provisioning Server。

适用于Oracle数据库的Oracle成员集群

Oracle数据库的Oracle成员集群支持Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库实例。此群集向管理存储库服务注册并使用集中式TFA服务。它可以根据需要使用其他服务。可以使用本地Oracle ASM存储管理配置Oracle Oracle数据库成员集群，也可以使用Oracle域服务集群提供的整合Oracle ASM存储管理服务。

Oracle数据库的Oracle成员集群始终使用其Oracle域服务集群中的远程网格基础结构管理存储库（GIMR）。对于双节点或四节点群集，在远程群集上托管GIMR可减少在群集上运行额外基础架构存储库的开销。

Oracle成员集群应用程序

Oracle成员集群应用程序托管Oracle数据库以外的应用程序，作为Oracle集群域的一部分。Oracle成员集群需要连接到Oracle集群域服务，以实现集中管理和资源效率。Oracle成员集群使用远程Oracle ASM存储，通过Oracle ACFS Remote服务提供任何所需的共享存储。此群集配置可实现任何软件应用程序的高可用性。

与需要公共和专用网络互连的其他群集配置不同，Oracle成员群集应用程序可以配置为使用单个公共网络接口。

注意：

在运行Oracle Universal Installer之前，必须通过创建成员集群清单文件来指定Oracle成员集群的Oracle域服务集群配置详细信息。

Oracle成员集群Oracle数据库不支持Oracle Database 12.1或更早版本，其中Oracle成员集群使用Oracle ASM存储直接或间接配置。

4.9.2 SQL * Loader

SQL * Loader实用程序将数据从外部文件移动到Oracle数据库中的表中。

SQL * Loader移动的文件可能包含由基本标量数据类型（如INTEGER ， CHAR或DATE组成的数据，以及复杂的用户定义数据类型（如行和列对象）（包括具有对象，集合或对象的对象） REF属性），集合和LOB。目前，SQL * Loader仅支持单级集合：您还不能使用SQL * Loader来加载多级集合，即其元素是或包含其他集合的集合。SQL * Loader使用包含SQL * Loader数据定义语言（DDL）语句的控制文件来描述数据文件的格式，内容和位置。

SQL * Loader提供了两种加载数据的方法：

  • 常规路径加载 ，它使用SQL INSERT语句和绑定数组缓冲区将数据加载到数据库表中

  • 直接路径加载 ，它使用Direct Path Load API代表SQL * Loader客户端将数据块直接写入数据库。

    直接路径加载不使用SQL接口，因此避免了处理关联SQL语句的开销。因此，直接路径加载通常提供比传统路径加载好得多的性能。

这两种方法都可用于加载受支持对象和集合数据类型的数据。

注意：
