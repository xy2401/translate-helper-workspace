XDK JavaBeans简介

XDK JavaBeans是一组可扩展标记语言（XML）组件，可以在Java应用程序和applet中使用。

使用XDK JavaBeans的先决条件

描述了使用XDK JavaBeans的先决条件。

本附录假定您熟悉这些技术：

  • JavaBeans 。JavaBeans组件或Bean是可重用的软件组件，可以在构建器工具中以可视方式进行操作。

  • Java数据库连接（JDBC） 。数据库连接包含在XDK JavaBeans中。Bean可以直接连接到启用JDBC的数据库，以检索和存储XML和可扩展样式表语言（XSL）文件。

  • 文档对象模型（DOM） 。DOM是XML文档结构的内存树表示。

  • 简单的XML API（SAX） 。SAX是基于事件的XML解析的标准。

  • XML Schema语言 。有关概述和建议阅读的链接，请参阅使用XML Schema Processor for Java 。

XDK JavaBeans的标准和规范

XDK JavaBeans需要1.2或更高版本的XDK，它们可以与任何版本的JDK 1.2或更高版本一起使用。XDK JavaBeans符合Sun JavaBeans规范，并包含扩展java.beans.所需的BeanInfo类java.beans.SimpleBeanInfo 。

JavaBeans 1.01规范描述了JDK 1.1中存在的JavaBeans。

Java 2平台添加到JavaBeans核心规范中为开发人员提供了创建更复杂的JavaBeans组件的标准方法。

XDK JavaBeans功能

XDK JavaBeans有助于向XML应用程序添加图形用户界面（GUI）。Bean封装包括可以直接从Java集成开发环境（IDE）（如Oracle JDeveloper）访问的文档和描述符。

DOMBuilder

oracle.xml.async.DOMBuilder bean从XML文档构造DOM树。DOMBuilder JavaBean使用bean接口封装DOMParser类的XML解析器，并支持异步解析。通过注册监听器，Java程序可以启动对大型或多个文档的解析，并立即将控制权返回给调用者。

这个bean的一个主要好处是在解析多个文件时提高了效率，特别是在文件很大的情况下。DOMBuilder还可以在交互式可视化应用程序中的后台线程中提供异步解析。如果没有异步解析，GUI将无法使用，直到要解析的文档。使用DOMBuilder ，应用程序调用parse方法然后恢复控制。应用程序可以显示进度条，允许用户取消解析，等等。

XSLTransformer

oracle.xml.async.XSLTransformer JavaBean支持异步转换。它接受XML文档，应用可扩展样式表语言转换（XSLT）样式表，并创建输出文件。它允许您将XML文档转换为几乎任何基于文本的格式，包括XML，HTML和数据定义语言（DDL）。

此bean还可以用作服务器端应用程序或servlet的基础，以将XML文档（例如查询结果的XML表示形式）呈现为HTML，以便在浏览器中显示。

XSLTransformer bean的主要好处是它可以并行转换多个文件。与DOMBuilder一样，您也可以在可视化应用程序中使用它，以避免GUI无响应的长时间。通过实现XSLTransformerListener接口，调用应用程序在转换完成时接收通知。

DBACCESS

oracle.xml.dbaccess.DBAccess bean维护包含多个XML和文本文档的字符大对象（ CLOB ）表。

您可以使用它来存储和检索数据库中的XML文档，但不能在数据库中处理它们。使用DBAccess bean的Java应用程序通过JDBC连接到数据库。存储在XML文档CLOB表是类型的不是XMLType没有自己的实体扩展。

DBAccess bean使您可以执行以下任务：

  • 创建和删除CLOB类型的表。

  • 查询CLOB表的内容。

  • 对存储在CLOB表中的XML文档执行INSERT ， UPDATE和DELETE操作。

XMLDBAccess

oracle.xml.xmldbaccess.XMLDBAccess bean扩展了DBAccess bean以支持存储在XMLType表中的XML文档。该类提供了列出，删除或检索XMLType实例及其表的方法。例如， getXMLXPathTextData()方法从XML文档中检索XPath表达式的值。

DBAccess JavaBean维护可以包含多个XML和文本文档的XMLType表。每个XML或文本文档都存储在表中的一行中。该表是使用此结构化查询语言（SQL）语句创建的：

CREATE TABLE（FILENAME CHAR（）UNIQUE，FILEDATA SYS.XMLType）;

FILENAME字段包含一个唯一的字符串，用作检索，更新或删除行的键。文档文本存储在FILEDATA字段中。

XMLDBAccess bean执行以下任务：

  • 创建和删除XMLType表

  • 列出XMLType列的内容

  • 对存储在XMLType表中的XML文档执行INSERT ， UPDATE和DELETE操作

的xmldiff

在比较XML文档时，逐个字符地比较它们通常是无益的。大多数XML比较都关注结构和重要文本内容的差异，而不是空白区域的差异。

oracle.xml.differ.XMLDiff bean执行以下任务：

  • 构造并比较两个输入XML文档的DOM树，并指出文档是否不同。

  • 提供两个XML文件之间差异的图形显示。具体来说，您可以参考节点插入，删除，修改或移动。

  • 生成一个XSLT样式表，可以将其中一个输入XML文档转换为另一个文档。

XMLDiff bean在管道应用程序中特别有用。例如，应用程序可以更新XML文档，将其与文档的先前版本进行比较，并存储显示它们之间差异的XSLT样式表。

XMLCompress

Oracle XML解析器包含一个压缩程序，可以将XML文档对象序列化为二进制流。

压缩和解压缩XML中对此进行了解释。虽然是一个有用的工具，但使用XML解析器进行压缩有以下缺点：

  • 反序列化XML数据时，必须重新解析。

  • 将XML数据封装在标签中会大大增加其大小。

oracle.xml.xmlcomp.XMLCompress bean是XML压缩功能的封装。它在序列化和反序列化XML时提供了这些优势：

  • 它封装了序列化DOM的方法，该方法生成一个流。

  • XML处理器可以从压缩流中重新生成DOM，而无需重新分析XML。

该bean支持DOMParser或SAXParser解析的输入XML的压缩和解压缩。DOM压缩支持XMLType ， CLOB和BLOB类型的输入。

要使用不同的解析选项，请在输入之前解析文档，然后将XMLDocument对象传递给压缩程序bean。压缩因子是基于输入XML文件和压缩文件的文件大小的粗略值。压缩因子方法的局限性在于只有在使用java.io.执行压缩时才能使用它。File对象作为参数。

XSDValidator

oracle.xml.schemavalidator.XSDValidator bean封装了XSDValidator类，并添加了验证DOM树的功能。

此bean的一个有用功能涉及验证错误。如果应用程序抛出验证错误，则方法getStackList()返回导致无效节点的DOM树路径列表。具有错误的节点在堆栈树的向量中返回，其中堆栈的顶部元素表示根节点。您可以通过从堆栈中提取子节点来获取子节点。要使用getStackList() ，必须使用实例化java.util.Vector和java.util.Stack类。

init.ora文件中的Oracle数据库初始化参数与网关初始化参数不同。使用特定于代理的机制在初始化参数文件中设置网关参数，或使用DBMS_HS包在Oracle数据字典中设置它们。网关启动时，网关初始化参数文件必须可用。

初始化参数文件语法

初始化参数文件的语法如下：

  • 该文件是一系列命令。

  • 每个命令应该从单独的一行开始。

  • 行尾被视为命令终止符（除非使用反斜杠进行转义）。

  • 如果初始化参数文件中存在语法错误，则所有设置都不会生效。

  • 设置参数值如下：

    [SET] [PRIVATE] 参数 = 值

    哪里：

    parameter是初始化参数名称。它是一个以字母开头的字符串，由字母，数字和下划线组成。初始化参数名称区分大小写。

    value是初始化参数值。它区分大小写。初始化参数值是：

    • 一串不包含任何反斜杠，空格或双引号的字符（“）

    • 带引号的字符串，以双引号开头，以双引号结尾。以下内容可以在带引号的字符串中使用：

      • 反斜杠（\）是转义字符

      • \ n插入一个新行

      • \ t插入一个标签

      • \“插入双引号

      • \\插入一个反斜杠

      该行末尾的反斜杠继续下一行的字符串。如果反斜杠在任何其他字符之前，则忽略反斜杠。

    例如，要为代理启用跟踪，请按如下所示设置HS_FDS_TRACE_LEVEL初始化参数：

     HS_FDS_TRACE_LEVEL = ON

    SET和PRIVATE是可选关键字。您不能将它们用作初始化参数名称。大多数参数仅作为初始化参数需要，因此您通常不需要使用SET或PRIVATE关键字。如果未指定SET或PRIVATE ，则该参数仅用作代理的初始化参数。

    SET指定除了用作初始化参数之外，还将参数值设置为代理进程的环境变量。将SET用于驱动程序或非Oracle系统需要的参数值作为环境变量。

    PRIVATE指定初始化参数应该是代理的私有参数，不应上载到Oracle数据库。大多数初始化参数不应该是私有的。但是，如果您在初始化参数文件中存储密码等敏感信息，则可能不希望将其上载到服务器，因为上载时初始化参数和值未加密。将初始化参数设置为私有可防止上载发生，并且它们不会出现在动态性能视图中。仅当参数值包含敏感信息（如用户名或密码）时，才使用PRIVATE作为初始化参数。

    SET PRIVATE指定将参数值设置为代理进程的环境变量，并且也是私有的（不会传输到Oracle数据库，不会出现在动态性能视图或图形用户界面中）。

HS_LANGUAGE

属性 描述

默认值

系统专用

价值范围

任何有效的语言名称（最多255字符）

为异构服务提供非Oracle数据源的字符集，语言和区域信息。该值必须使用以下格式：

语言 [_ 领土 。 character_set ]

注意：

全球化支持初始化参数会影响错误消息，SQL Service的数据以及分布式外部过程中的参数。

字符集

理想情况下，Oracle数据库和非Oracle数据源的字符集是相同的。在几乎所有情况下， HS_LANGUAGE应设置为与Oracle数据库字符集完全相同，以实现最佳字符集映射和性能。如果它们不相同，则异构服务会尝试将非Oracle数据源的字符集转换为Oracle数据库字符集，然后再返回。翻译会降低性能。在某些情况下，异构服务无法将字符从一个字符集转换为另一个字符集。

注意：

指定的字符集必须是安装代理程序的平台上的操作系统字符集的超集。

由于更多Oracle数据库和非Oracle数据库使用Unicode作为数据库字符集，因此最好还以Unicode字符集运行网关。为此，您必须设置HS_LANGUAGE=AL32UTF8 。但是，当网关在Windows上运行时，Microsoft ODBC驱动程序管理器界面只能以双字节字符集UCS2交换数据。这导致所述缓冲液和柱尺寸的额外比率扩展。有关如何调整到正确尺寸的说明，请参阅HS_FDS_REMOTE_DB_CHARSET 。

语言

HS_LANGUAGE初始化参数的语言组件确定：

  • 日期的日期和月份名称

  • 日期和时间的AD，BC，PM和AM符号

  • 默认排序机制

请注意，Oracle不确定通用异构服务消息（ ORA-25000到ORA-28000 ）的错误消息的语言。它们由Oracle数据库中的会话设置控制。

领土

territory子句指定日期和星期编号的约定，默认日期格式，十进制字符和组分隔符以及ISO和本地货币符号。请注意，Oracle数据库与非Oracle数据源之间的全球化支持级别取决于网关的实现方式。

属性 描述

价值范围

任何值最多2 GB

设置要传输的LONG数据的大小。较小的块大小意味着更少的内存需求，但更多的往返来获取所有数据。较大的单件尺寸意味着更少的往返次数，但更多的内存需要在内部存储中间件。因此，初始化参数可用于调整系统以获得最佳性能，在往返和内存要求之间以及网络延迟或响应时间之间进行最佳权衡。

HS_RPC_FETCH_REBLOCKING

属性 描述

默认值

上

价值范围

OFF或ON

控制异构服务是否尝试优化Oracle数据库与连接到非Oracle数据存储的异构服务代理之间的数据传输性能。

属性 描述

默认值

50000

价值范围

13.2 CDB和PDB概述

Oracle数据库可以包含一个可移植的模式，模式对象和非复制对象集合，这些对象在Oracle Net客户端中显示为单独的数据库。这个独立的集合称为PDB。 CDB可以包括零个或多个PDB。

CDB还可以包括零个或多个应用程序容器。每个应用程序容器都包含一个应用程序根目录和插入应用程序根目录的应用程序PDB。

EM Express使数据库管理员能够管理CDB及其PDB。

13.3关于可以在EM Express中管理的PDB

您可以使用EM Express管理不同类型的PDB。 例如：

  • PDB：PDB是CDB中的标准PDB。这是一个PDB，它不是此列表中其他类型的PDB之一。

  • 可刷新的PDB：可刷新的克隆PDB（可刷新的PDB）是源PDB的只读克隆，其可以周期性地与源PDB同步。

  • 代理PDB：引用不同PDB的PDB。本地代理PDB与其引用的PDB在同一CDB中，而远程代理PDB在不同的CDB中。

    EM Express使您可以查看代理PDB的名称和状态，但EM Express不支持代理PDB的性能管理。

  • 应用程序根目录：这是应用程序容器中的应用程序根目录。

  • Application PDB：应用程序容器中的PDB。

13.7使用EM Express为CDB配置Oracle托管文件

本节提供有关为CDB配置Oracle托管文件的信息。

使用Oracle Managed Files简化了Oracle数据库的管理。Oracle Managed Files使DBA无需直接管理组成Oracle数据库的操作系统文件。使用Oracle Managed Files，您可以指定数据库在数据库对象级别自动创建，命名和管理文件的文件系统目录。

通过初始化参数，指定要用于特定类型文件的文件系统目录。然后，数据库确保在不再需要时创建和删除唯一文件（Oracle托管文件）。

此功能不会影响管理文件的创建或命名，例如跟踪文件，审计文件，警报日志和核心文件。

您可以使用EM Express为CDB配置Oracle托管文件。

注意：

在为CDB配置Oracle托管文件之前：

  • CDB必须存在。

  • 必须配置EM Express才能管理CDB。

要为CDB配置Oracle托管文件：

  1. 在EM Express中，导航到要为其配置Oracle托管文件的CDB的“容器”页面。

    有关更多信息，请参阅“ 访问CDB的容器页面 ”。

  2. 在“容器”页面的顶部附近，单击“ 配置Oracle管理的文件”按钮。

  3. 通过“配置Oracle管理的文件”向导，您可以为DB_CREATE_FILE_DEST初始化参数设置值（目录位置）。您指定的目录将是Oracle Managed Files的目标。

    在“配置Oracle托管文件”向导中指定这些字段的值：

    • 范围 ：指定内存以进行内存更改，立即生效并持续直到数据库关闭。选择SPFile以在服务器参数文件中进行更改，以便在重新启动数据库后进行更改。选择Memory和SPFile以立即更改值，并在重新启动数据库后使其保持有效。

    • 延迟 ：如果指定，则延迟选项允许仅为将来连接到数据库的会话修改参数的值。延迟时，当前会话保留旧值。如果未指定deferred，则立即更改该值。

    • 值 ：指定用于DB_CREATE_FILE_DEST初始化参数的目录。该目录将是Oracle Managed Files的目标。

    • 注释 ：（可选）输入有关所做更改的注释。

  4. 单击确定 。

    “确认”框会通知您已将Oracle Managed Files的默认目录设置为指定位置。在Containers页面的Status部分中， Using Oracle-Managed Files字段显示值Yes 。

    也可以看看：

要从种子创建新的PDB：

    • 密码 ：输入管理用户的密码。

    • 授予DBA角色 ：启用此选项可将DBA角色授予您正在创建的管理用户。

    • 如果使用Oracle Managed Files启用了目标CDB（您正在其中创建PDB），并且您希望使用相同的CDB，则选择“ 使用Oracle托管文件（OMF）” 。

    如果未启用无限存储，则必须为这些字段指定值：

    • 最大大小 ：属于PDB的所有表空间可以使用的存储量。

    • 最大共享临时大小 ：连接到PDB的会话可以使用的所有PDB共享的默认临时表空间中的存储量。

    如果没有为这些字段指定值，则在创建PDB时，它们的值将设置为无限制。

    • 内存最小值（％） ：为此PDB分配新的最小内存百分比，或保留当前值

    • 内存限制（％） ：为此PDB分配新的最大内存百分比，或保留当前值。

13.8.3使用EM Express插入未插入的PDB

  • 描述未插入的PDB的XML文件和与未插入的PDB关联的其他文件（例如数据文件）必须存在且必须是可读的。

  • 源CDB主机的平台（以前包含未插入的PDB的CDB所在的主机）和目标CDB主机（安装目标CDB的主机）必须具有相同的字节顺序，并且必须具有相同的字节顺序安装的数据库选项集。

  • 包含未插入的PDB和目标CDB的CDB必须具有兼容的字符集和国家字符集。为了兼容，字符集和国家字符集必须满足“ Oracle数据库全球化支持指南”中指定的所有要求。

将未插入的PDB插入CDB：

  1. 在EM Express中，导航到要将未插入的PDB插入的CDB的“容器”页面。

    • 从元数据文件重用源数据文件位置 ：选择此选项，或禁用它并指定新的源数据文件位置，以便在插入CDB时用于未插入的PDB。

    单击右箭头按钮转到“存储”页面。

  4. 在“存储”页面上，选择要为PDB存储数据文件的位置类型：

    • 如果使用Oracle Managed Files启用了目标CDB（插入未插入的PDB），并且您希望使用相同的CDB，则选择Use Oracle Managed Files（OMF） 。

    • 如果目标CDB不使用OMF，则指定数据文件位置或接受数据文件位置字段中指定的默认值。

  5. 同样在“存储”页面上，选择是否为数据文件启用无限存储。

    如果未启用无限存储，则必须为这些字段指定值：

    • 最大大小 ：属于PDB的所有表空间可以使用的存储量。

    • 最大共享临时大小 ：连接到PDB的会话可以使用的所有PDB共享的默认临时表空间中的存储量。

    如果没有为这些字段指定值，则在插入PDB时，它们的值将设置为无限制。

  6. 如果CDB具有当前活动资源计划，则会显示“资源限制”页面。

    为这些字段提供值，或保留您插入的PDB的默认值：

    • 共享 ：为此PDB分配新的共享值，或保留当前值。

    • CPU利用率限制（％） ：为此PDB分配新的CPU利用率百分比值，或保留当前值。

    • 并行服务器限制（％） ：为此PDB分配新的并行服务器利用率百分比值，或保留当前值。

    • 内存最小值（％） ：为此PDB分配新的最小内存百分比，或保留当前值。

    • 内存限制（％） ：为此PDB分配新的最大内存百分比，或保留当前值。

13.8.4.1通过使用EM Express从远程CDB克隆PDB来创建PDB

您可以使用EM Express从远程CDB克隆现有PDB来创建新PDB。

注意：

在通过从远程CDB克隆现有PDB创建新PDB之前：

  • 源PDB（要克隆的PDB）必须存在，并且必须配置EM Express才能访问PDB。

  • 源PDB必须是开放的。

  • 目标CDB（要插入克隆PDB的CDB）必须存在，并且EM Express必须配置为访问CDB。

  • 目标CDB必须处于读/写模式。

  • 目标主机用户必须是CDB所属的Oracle主目录的所有者。

  • 要执行热克隆，必须满足“ 关于远程PDB克隆，重定位PDB和刷新PDB ”中的先决条件，并且要克隆的PDB必须以读/写模式打开。如果PDB未在读/写模式下打开，则执行冷克隆。

通过从远程CDB克隆现有PDB来创建新PDB：

  1. 在EM Express中，导航到要在其中创建PDB的CDB的“容器”页面。

    有关更多信息，请参阅“ 访问CDB的容器页面 ”。

  2. 在“容器”页面的“容器”部分中，从“ 操作”菜单中选择“ 远程克隆 ”。将出现“克隆远程PDB”向导。

    • PDB名称 ：输入要用于将由克隆操作创建的PDB的名称。

    • 源PDB名称 ：输入要在远程CDB中克隆的PDB的名称。

    • 源数据库链接 ：输入包含要克隆的PDB的远程CDB的数据库链接的名称。

    • 并行度 ：指定用于将新PDB的数据文件复制到新位置的并行执行服务器数量的值。您必须指定值2或更大，否则CDB不会并行创建PDB。根据当前数据库负载和可用并行执行服务器的数量，CDB可能不会授予您为PDB创建请求的并行度。

    • 刷新选项 ：启用此选项以选择刷新模式，以从源PDB创建可刷新的克隆PDB（可刷新的PDB）。启用此选项后，您可以选择手动或自动刷新模式。

    • 手动 ：必须在创建的可刷新PDB上手动执行刷新操作。

    • 自动 ：刷新操作以您在“自动刷新频率”字段中指定的间隔自动发生。

      可刷新的PDB可以是只读模式或关闭。必须关闭使用手动刷新模式或自动刷新模式创建的可刷新PDB才能执行刷新。启用自动刷新后，如果在尝试自动刷新时未关闭可刷新的PDB，则刷新将推迟到下一个计划的刷新。

    • 自动刷新频率（分钟） ：选择可刷新PDB自动刷新的间隔（以分钟为单位）。

    • 服务选项 ：数据库中的每个PDB都应具有唯一的服务名称。默认情况下，克隆操作将源PDB的服务名称用作目标（新）PDB的服务名称，这会导致新PDB的PDB_PLUG_IN_VIOLATIONS视图中违规列表中出现违规。要避免此违规，请启用“ 服务选项”选项。

    • 源PDB服务 ：指定源PDB的服务名称。

    • 目标PDB服务 ：指定将由克隆操作创建的新PDB的服务名称。

    单击右箭头按钮转到“存储”页面。

  4. 在“存储”页面上，选择要为PDB存储数据文件的位置类型：

    • 如果使用Oracle Managed Files启用了目标CDB（您正在其中创建PDB），并且您希望使用相同的CDB，则选择“ 使用Oracle托管文件（OMF）” 。

    • 如果目标CDB不使用OMF，则指定数据文件位置或接受数据文件位置字段中指定的默认值。

  5. 同样在“存储”页面上，选择是否为数据文件启用无限存储。

    如果未启用无限存储，则必须为这些字段指定值：

    • 最大大小 ：属于PDB的所有表空间可以使用的存储量。

    • 最大共享临时大小 ：连接到PDB的会话可以使用的所有PDB共享的默认临时表空间中的存储量。

    如果没有为这些字段指定值，则在创建PDB时，它们的值将设置为无限制。

  6. 如果要克隆远程PDB的目标CDB具有当前活动资源计划，则会显示“资源限制”页面。

    为这些字段提供值，或保留您正在创建的PDB的默认值：

    • 共享 ：为此PDB分配新的共享值，或保留当前值。

    • CPU利用率限制（％） ：为此PDB分配新的CPU利用率百分比值，或保留当前值。

    • 并行服务器限制（％） ：为此PDB分配新的并行服务器利用率百分比值，或保留当前值。

  7. 单击确定 。

    除非新PDB是可刷新的PDB，否则将以读/写模式创建和打开新PDB。在目标上以封闭模式创建新的可刷新PDB。EM Express将PDB添加到容器页面上“顶部容器”部分中显示的容器列表中。您可能需要刷新“容器”页面以查看容器列表中的新PDB。

13.8.4.2重新定位远程PDB

您可以使用EM Express以最短的停机时间将PDB从远程CDB重定位到另一个CDB。

在重定位期间，重定位的远程PDB在读/写模式下保持打开状态。与PDB关联的文件将作为重定位的一部分移动到新CDB。

注意：

在从远程CDB重新定位PDB之前：

  • 源PDB（要重定位的PDB）必须存在，并且EM Express必须配置为访问PDB。

  • 源PDB必须是开放的。

  • 目标CDB（要将源PDB重新定位到的CDB）必须存在，并且必须配置EM Express才能访问CDB。

  • 目标CDB必须处于读/写模式。

  • 目标主机用户必须是CDB所属的Oracle主目录的所有者。

  • 要执行PDB重定位，必须满足“ 关于远程PDB克隆，重定位PDB和刷新PDB ”中的先决条件，并且必须在读/写模式下打开要重定位的PDB。

通过重新定位远程PDB来创建新的PDB：

  1. 在EM Express中，导航到要重定位远程PDB的CDB的“容器”页面。
    有关更多信息，请参阅“ 访问CDB的容器页面 ”。
  2. 在“容器”页面的“容器”部分中，从“ 操作”菜单中选择“ 重新定位 ”。出现Relocate PDB向导。
  3. 在“重定位PDB”向导的“常规”页面上，为这些字段提供值：

    • PDB名称 ：输入要用于将重定位到此CDB的PDB的名称。

    • 源PDB名称 ：输入要重定位到此CDB的远程PDB的名称。

    • 源数据库链接 ：输入包含要重定位的PDB的远程CDB的数据库链接的名称。

    • PDB重定位模式 ：为重定位操作指定正常模式或最大可用性模式的值。

    • 正常 ：使用此模式，源PDB将在重定位后关闭并放入源CDB中。PDB只存在于新的CDB中。

    • 最高可用性 ：使用此模式，源PDB保持打开状态，可以继续提供请求，直到PDB重定位完成。在PDB重定位完成并且打开目的地CDB中的新PDB以服务请求之后，任何指向源PDB的请求都被转发到目的地CDB中的新重定位PDB。在最大可用性模式下成功重定位后，源PDB将成为源CDB中安装模式下的逻辑删除PDB。迁移所有客户端以使用新的重定位PDB后，应删除源CDB中的此逻辑删除PDB条目。

    • 并行度 ：指定用于将新PDB的数据文件复制到新位置的并行执行服务器数量的值。您必须指定值2或更大，否则CDB不会并行创建PDB。根据当前数据库负载和可用并行执行服务器的数量，CDB可能不会授予您为PDB创建请求的并行度。

  4. 在“存储”页面上，选择要为PDB存储数据文件的位置类型：

    • 如果使用Oracle Managed Files启用了目标CDB（您正在重新定位PDB）并且您想要使用它，则选择Use Oracle Managed Files（OMF） 。

    • 如果目标CDB不使用OMF，则指定数据文件位置或接受数据文件位置字段中指定的默认值。

  5. 同样在“存储”页面上，选择是否为数据文件启用无限存储。

    如果未启用无限存储，则必须为这些字段指定值：

    • 最大大小 ：属于PDB的所有表空间可以使用的存储量。

    • 最大共享临时大小 ：连接到PDB的会话可以使用的所有PDB共享的默认临时表空间中的存储量。

    如果没有为这些字段指定值，则在创建PDB时，它们的值将设置为无限制。

  6. 如果要将远程PDB重新定位到的目标CDB具有当前活动资源计划，则会显示“资源限制”页面。

    为这些字段提供值，或保留您正在创建的PDB的默认值：

    • 共享 ：为此PDB分配新的共享值，或保留当前值。

    • CPU利用率限制（％） ：为此PDB分配新的CPU利用率百分比值，或保留当前值。

    • 并行服务器限制（％） ：为此PDB分配新的并行服务器利用率百分比值，或保留当前值。

  7. 单击确定 。

    重新定位的PDB以读/写模式创建和打开。EM Express将PDB添加到容器页面上“顶部容器”部分中显示的容器列表中。您可能需要刷新“容器”页面以查看容器列表中的新PDB。
