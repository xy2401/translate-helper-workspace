删除基于RPM的Oracle数据库安装

使用此过程删除和卸载使用yum命令安装的数据库。

  1. 以root身份登录。
  2. 运行以下命令以删除基于RPM的数据库安装：
    #yum -y remove oracle-database-ee-19c
yum命令检测Oracle主目录中的默认配置组件，例如数据库（ORCLCDB）和侦听器（LISTENER），并为您删除这些组件。

除了默认数据库（ORCLCDB）和侦听器（LISTENER）之外，如果yum命令检测到Oracle主目录中的其他已配置组件（如其他数据库或侦听器），则会停止卸载过程。将显示一条消息，指示您手动删除这些已配置的组件。要删除这些已配置的组件：

  1. 以Oracle数据库安装所有者用户（ oracle ）身份登录。

  2. 要删除与安装关联的任何数据库，请使用Oracle数据库配置助手（Oracle DBCA）。

    $ cd $ ORACLE_HOME / bin $ ./dbca

  3. 要删除与安装关联的任何侦听器，请运行Oracle Net Configuration Assistant（Oracle NETCA）。

    $ cd $ ORACLE_HOME / bin $ ./netca

  4. 以root身份登录并再次运行yum命令以删除基于rpm的数据库安装。 

    #yum -y remove oracle-database-ee-19c

在文件系统或逻辑卷上放置Oracle数据库文件的准则

如果选择将Oracle数据库文件放在文件系统上，则在决定放置它们的位置时，请使用以下准则：

  • Oracle Universal Installer为数据库文件目录建议的默认路径是Oracle基本目录的子目录。

  • 您可以选择单个文件系统或多个文件系统来存储数据库文件：

    • 如果要使用单个文件系统，请在专用于数据库的物理设备上选择文件系统。

      为获得最佳性能和可靠性，请在多个物理设备上选择RAID设备或逻辑卷，并实施条带和镜像所有（SAME）存储策略。

    • 如果要使用多个文件系统，请在专用于数据库的单独物理设备上选择文件系统。

      此方法使您可以分配物理输入输出操作并在不同设备上创建单独的控制文件，以提高可靠性。它还使您能够完全实施Oracle Optimal Flexible Architecture（OFA）指南。选择“ Advanced数据库创建”选项以实现此方法。

  • 如果您打算在安装期间创建预配置的数据库，则您选择的文件系统（或文件系统）必须至少具有2 GB的可用磁盘空间。

    对于生产数据库，必须根据数据库的使用情况估计磁盘空间要求。

  • 为获得最佳性能，您选择的文件系统必须位于仅由数据库使用的物理设备上。

  • 运行Oracle数据库安装的Oracle用户必须具有写入权限才能在您指定的路径中创建文件。

网络接口配置选项

  • 私人的

  • ASM

  • ASM和私人

  • 不使用

网络接口配置选项

您必须对Oracle Clusterware和Oracle RAC使用相同的专用适配器。为网络选择的精确配置取决于要配置的群集的大小和使用以及所需的可用性级别。网络接口必须至少为1 GbE，建议使用10 GbE。或者，使用InfiniBand进行互连。

什么是Oracle数据库升级的互操作性？

在升级Oracle数据库的环境中， 互操作性是指不同版本的Oracle数据库在分布式环境中进行通信和工作的能力。

分布式数据库系统可以包含不同版本的Oracle数据库，所有受支持的Oracle数据库版本都可以参与分布式数据库系统。但是，使用分布式数据库的应用程序还必须能够与系统中每个节点上可用的功能和功能进行互操作。

不同操作系统和操作系统版本之间的互操作性可能会导致问题（特别是在滚动升级期间），因为新Oracle数据库版本的最低要求可能要求您升级某些或所有主机上的操作系统。因此，在开始Oracle数据库升级之前，必须检查以确保驱动器，网络和存储在滚动升级期间与系统的所有临时升级状态兼容。

注意：

由于Oracle数据库升级指南讨论了不同版本的Oracle数据库之间的升级和降级，因此互操作性的定义适用于Oracle数据库版本。其他Oracle文档可能使用术语互操作性的更广泛定义。例如，在某些情况下，互操作性可以描述不同硬件平台和操作系统之间的通信。

关于将Oracle ASM与多路径磁盘配合使用

Oracle ASM要求每个磁盘都是唯一标识的。如果同一磁盘出现在多个路径下，则会导致错误。

在多路径磁盘配置中，同一磁盘可以出现三次：磁盘的初始路径，磁盘的第二个路径以及多路径磁盘访问点。

例如：如果您有一个本地磁盘， /dev/sda和一个连接外部存储的磁盘，那么您的服务器将显示该外部存储的两个连接或路径。Linux SCSI驱动程序显示两个路径。它们显示为/dev/sdb和/dev/sdc 。系统可以访问/dev/sdb或/dev/sdc ，但访问权限是访问同一磁盘。

如果启用多路径，则会有一个多路径磁盘（例如， /dev/multipatha ），它可以访问/dev/sdb和/dev sdc ;任何对multipatha I / O都可以使用sdb或sdc路径。如果系统正在使用/dev/sdb路径，并且该电缆已拔下，则系统会显示错误。但是多路径磁盘将从/dev/sdb路径切换到/dev/sdc路径。

大多数系统软件都不知道多路径配置。他们可以使用任何路径（ sdb ， sdc或multipatha ）。ASMLIB也不知道多路径配置。

默认情况下，ASMLIB识别Linux向其报告的第一个磁盘路径，但由于它在该磁盘上标记了一个标识，因此它仅在一个路径下识别该磁盘。根据您的存储驱动程序，它可能识别多路径磁盘，或者它可能识别其中一个单磁盘路径。

您应该配置Oracle ASM以识别多路径磁盘，而不是依赖于默认值。

12.1 Hang Manager体系结构

Hang Manager在数据库中自动作为DIA0任务运行。

图12-1 Hang Manager体系结构


“图12-1 Hang Manager架构”的说明

Hang Manager的工作分为以下三个阶段：

  • 检测：在此阶段，Hang Manager收集所有节点上的数据，并检测等待另一个会话持有的资源的会话。

  • 分析：在此阶段，Hang Manager会分析在“ 检测”阶段中检测到的会话，以确定会话是否是潜在挂起的一部分。如果会话被怀疑为挂起，则Hang Manager会等待一定的阈值时间段以确保会话挂起。

  • 验证：在此阶段，在阈值时间段结束后，Hang Manager会验证会话是否已挂起并选择受害者会话。受害者会话是导致挂起的会话。

选择受害者会话后，Hang Manager会在受害者会话中应用挂起解决方法。如果会话链或挂起自动解决，则Hang Manager不会应用挂起解析方法。但是，如果挂起本身无法解决，则Hang Manager会通过终止受害者会话来解决挂起问题。如果终止会话失败，则Hang Manager将终止会话进程。整个过程是自治的，不会长时间阻止资源，也不会影响性能。

Hang Manager还会考虑用于维护性能目标的Oracle数据库QoS管理策略，性能类和排名。

例如，如果挂起的会话链中包含高排名会话，则Hang Manager会加快受害者会话的终止。终止受害者会话可防止高级别会话等待太长时间并有助于维持高级别会话的性能目标。

活动数据库复制概述

活动数据库复制不需要备份源数据库。它通过网络将数据库文件复制到辅助实例，将实时源数据库复制到目标主机。RMAN可以将所需文件复制为映像副本或备份集。

对于活动数据库复制，使用的复制技术确定哪个通道执行主要工作。使用备份集执行活动数据库复制时，复制的主要工作由辅助通道执行。使用图像副本时，主要工作由目标通道执行。

要执行活动数据库复制，需要连接到目标数据库。Oracle建议您一般使用活动数据库复制，除非源主机和目标主机之间的网络带宽是约束。活动数据库复制需要最少的设置，并且执行起来更简单。

注意：

对于活动数据库复制，源数据库必须使用服务器参数文件。

使用备份集的活动数据库复制可能比使用映像副本更受欢迎的一些方案是：

  • 您希望在复制数据库时使用多节备份，压缩或加密。

  • 源数据库没有足够的网络资源来将所需的数据库文件传输到重复数据库。

关于SCAN VIP地址

SCAN虚拟IP地址（VIP）的功能类似于节点VIP。但是，与节点VIP不同，SCAN VIP可以在群集中的任何节点上运行。

使用SCAN而不是节点VIP名称或地址连接的客户端（用户或应用程序）在向集群添加节点或从集群中删除节点时，不必更新其本地tnsnames.ora文件中的节点名称或地址列表，或者当数据库实例在不同的节点上运行时。

注意：

在DNS中配置三个SCAN VIP本身并不能确保连接的故障转移。相反，Oracle客户端使用返回的SCAN VIP将连接请求故障转移到其他SCAN侦听器。如果对SCAN VIP的连接尝试失败，则客户端使用下一个返回的SCAN VIP地址进行连接。因此，Oracle建议您将Oracle Client 11g第2版或更高版本的客户端用于使用SCAN的连接。

如果使用GNS进行名称解析，则仅在安装期间提供SCAN名称（例如， sales1-scan ）。GNS获取三个IP地址的DHCP地址租约，并将这些地址解析为SCAN。 GNS守护程序侦听注册。当SCAN VIP在节点上启动时，它会向GNS注册其地址。

对GNS管理的群集域的服务请求将路由到GNS VIP地址，后者将这些请求路由到群集的GNS守护程序。当GNS从DNS接收SCAN请求时，它将SCAN侦听器的注册地址返回给DNS。然后，DNS将三个SCAN VIP地址返回给客户端。

目的

SYS_TYPEID返回操作数的最特定类型的typeid。该值主要用于标识可替换列下面的类型判别列。例如，您可以使用SYS_TYPEID返回的值在类型判别列上构建索引。

您只能在对象类型操作数上使用此函数。所有最终根对象类型 - 不属于类型层次结构的最终类型 - 具有null typeid。Oracle数据库为属于类型层次结构的所有类型分配唯一的非null typeid。

也可以看看：

有关typeids的更多信息，请参见Oracle数据库对象关系开发人员指南

例子

以下示例使用在“可替换表和列示例 ”中创建的表persons和books 。第一个查询返回存储在persons表中的最具体类型的对象实例。 

SELECT name，SYS_TYPEID（VALUE（p））“Type_id”FROM persons p; NAME Type_id ------------------------- ----------------------- --------- Bob 01 Joe 02 Tim 03

下一个查询返回存储在表格books的最具体类型的作者：

SELECT b.title，b.author.name，SYS_TYPEID（作者）“Type_ID”FROM books b; TITLE AUTHOR.NAME Type_ID ------------------------- -------------------- -------------------自传Bob 01商业规则Joe 02混合学校和工作Tim 03
