151 DBMS_SHARED_POOL

DBMS_SHARED_POOL包提供对共享池的访问，共享池是存储游标和PL / SQL对象的共享内存区域。DBMS_SHARED_POOL使您可以显示共享池中对象的大小，并标记它们是为了保持还是不保留，以减少内存碎片。

本章包含以下主题：

  • 概观

  • 操作说明

  • DBMS_SHARED_POOL子程序的摘要

151.1 DBMS_SHARED_POOL概述

加载大型PL / SQL对象时，此处提供的过程可能很有用。加载大型PL / SQL对象时，用户响应时间会受到影响，因为需要从共享池中老化的大量较小对象才能腾出空间（由于内存碎片）。在某些情况下，可能没有足够的内存来加载大对象。

DBMS_SHARED_POOL对于频繁执行的触发器也很有用。您可能希望将已编译的触发器保留在共享池中的常用表上。

此外， DBMS_SHARED_POOL支持序列。当序列从共享池中老化时，序列号将丢失。DBMS_SHARED_POOL对于将序列保留在共享池中非常有用，从而防止丢失序列号。

151.2 DBMS_SHARED_POOL操作说明

要创建DBMS_SHARED_POOL，请运行DBMSPOOL.SQL脚本。运行DBMSPOOL.SQL后，将自动执行PRVTPOOL.PLB脚本。这些脚本不是作为标准数据库创建的一部分运行的。

151.3.1 ABORTED_REQUEST_THRESHOLD程序

此过程设置共享池的中止请求阈值。

句法

DBMS_SHARED_POOL.ABORTED_REQUEST_THRESHOLD（threshold_size NUMBER）;

参数

表151-2 ABORTED_REQUEST_THRESHOLD过程参数

参数 描述

threshold_size

请求的大小（以字节为单位），该请求不会尝试释放共享池中的未固定（非“非保持”）内存。threshold_size的范围是5000到2 GB（含）。

例外

如果阈值不在有效范围内，则会引发异常。

使用说明

通常，如果在空闲列表中无法满足请求，则RDBMS会尝试通过从LRU列表中释放对象并定期检查以查看是否可以满足请求来回收内存。完成此步骤后，RDBMS执行了近似等效的' ALTER SYSTEM FLUSH SHARED_POOL '。

因为这会影响系统上的所有用户，所以此过程thresh_hold对该进程的影响“本地化”，从而无法找到大小超过thresh_hold大小的共享池内存。此用户在不尝试搜索LRU列表的情况下获得“内存不足”错误。

151.3.2保持程序

此过程将对象保留在共享池中。一旦对象保留在共享池中，它就不会从池中老化。这对于经常使用的大型对象可能很有用。当大对象进入共享池时，可能需要对几个对象进行老化以创建足够大的连续区域。

句法

DBMS_SHARED_POOL.KEEP（名称VARCHAR2，标志CHAR DEFAULT'P'）;
DBMS_SHARED_POOL.KEEP（schema VARCHAR2，objname VARCHAR2，namespace NUMBER，堆NUMBER，edition_name VARCHAR2 DEFAULT NULL）;
DBMS_SHARED_POOL.KEEP（散列VARCHAR2，命名空间NUMBER，堆NUMBER）;

关于Oracle ACFS

Oracle ACFS设计为通用的独立服务器和群集范围的文件系统，可为所有客户文件提供支持。用户和应用程序可以使用本机操作系统文件系统应用程序编程接口（API）和命令行界面（CLI）工具访问和管理Oracle ACFS。用户还可以使用Oracle ASM Configuration Assistant（ASMCA）管理Oracle ACFS。

Oracle ACFS支持具有64位文件和文件系统数据结构大小的大型文件，从而在64位平台上实现具备exabyte功能的文件和文件系统容量。基于可变范围的存储分配和高性能目录有助于实现快速性能和共享磁盘配置，从而为每个集群成员提供Oracle ACFS文件数据的直接存储路径。使用Oracle ACFS元数据校验和和日记功能可实现文件系统完整性和快速恢复。Oracle ACFS设计为多节点共享文件系统模型，可为每个集群成员的Oracle ACFS文件数据提供连贯，缓存的直接存储路径。

Oracle ACFS文件系统通常配置为进行群集范围访问。文件系统，文件和目录是可见的，可从所有集群成员访问，并且可由用户和应用程序使用来自任何集群成员的相同路径名引用。此设计可简化跨群集成员的应用程序部署，并促进多实例群集应用程序和未修改的独立服务器应用程序的高可用性（HA）故障转移。

Oracle ACFS跨群集配置提供单一系统文件访问语义。所有集群成员上的应用程序和用户始终显示共享Oracle ACFS文件数据的相同视图，由Oracle ACFS集群范围用户和元数据缓存一致性机制支持。

关于Oracle ACFS Mount Model和Namespace

Oracle ACFS被设计为一个分层文件系统，其中包含组织成树状结构命名空间的文件和子目录，文件位于树叶。命名空间设计是独立服务器和群集配置的单文件系统命名模型。此设计使每个集群成员能够使用相同的路径名向集群应用程序提供共享文件，从而简化了多节点应用程序和用户访问以及整个文件系统管理。Oracle ACFS安装模型还适用于群集配置中的节点本地安装和群集节点子集安装，以满足其他客户要求。

Oracle ACFS文件系统的最佳实践是使用Oracle Clusterware资源管理的Oracle Clusterware，以确保在Oracle Grid Infrastructure启动和关闭期间正确处理它们。

您可以显式使用mount命令。但是，如果已创建资源，则可能已安装文件系统。

Oracle ACFS中数据文件的最佳实践是将集群范围的资源用于存储数据文件的Oracle ACFS文件系统。

如果在数据库安装完成后添加数据文件，则必须修改数据库依赖关系以列出新的Oracle ACFS文件系统。如果不这样做会导致应用程序可靠性出错。要在依赖关系列表中指定文件系统，请使用SRVCTL数据库对象命令修改资源使用的Oracle ACFS路径。

Oracle Grid Infrastructure 18c群集和独立（Oracle Restart）配置中的Oracle ACFS支持Oracle Database 11.2.0.4或更高版本的所有数据库文件。可以将Oracle ACFS配置为与数据库一起使用，尤其是利用Oracle ACFS快照进行数据库测试和开发。要在群集配置中支持数据库文件，必须将包含Oracle ACFS文件系统的磁盘组的COMPATIBLE.ADVM属性设置为12.1或更高。在Oracle Restart配置中， COMPATIBLE.ADVM属性必须设置为12.2.0.1或更高版本才能支持所有数据库文件。

Windows上对数据库数据文件的支持始于Oracle Grid 12 c第1版（12.1.0.2）。要在Windows上支持数据库文件，必须将COMPATIBLE.ADVM属性设置为12.1.0.2或更高版本。

Oracle Exadata（Linux）上对数据库数据文件的支持始于Oracle Grid 12 c第1版（12.1.0.2）。但是，Oracle ACFS目前无法将数据库操作直接推送到存储中。

Oracle Grid Infrastructure 12.1.0.2上的Oracle ACFS还支持Oracle Exadata（Linux）存储上的Oracle Database 10g第2版（10.2.0.4和10.2.0.5）的所有数据库文件。对于Oracle Exadata存储上Oracle Database 10g第2版（10.2.0.4和10.2.0.5）的数据库文件支持，必须满足以下条件：

  • 使用DBCA创建Oracle数据库时，必须将REMOTE_LISTENER初始化参数设置为your_scan_vip :1521否则DBCA在创建过程中失败。

  • 您必须修改数据库实例资源的所有启动和停止依赖关系，以确保在启动Oracle Clusterware时启动资源。

关于Oracle ACFS和Oracle数据库主目录

可以将Oracle ACFS文件系统配置为Oracle数据库主目录。

在群集中安装数据库时，需要为数据库主目录使用共享Oracle ACFS文件系统。您可以将Oracle ACFS文件系统用于Oracle 11g第2版（11.2）或更高版本的Oracle数据库主目录。

安装Oracle软件时，必须有与每个操作系统用户关联的单独Oracle基础（ ORACLE_BASE ）。例如，网格用户和数据库用户应该有一个单独的Oracle库。

您可以在Oracle ACFS文件系统上找到Oracle数据库库（ ORACLE_BASE for database）目录和home（ ORACLE_HOME for database）目录。Oracle数据库基础（ ORACLE_BASE for database）目录不应是Oracle Grid Infrastructure基础（ ORACLE_BASE for grid）目录，也不应位于Oracle Grid Infrastructure基本目录（ ORACLE_BASE for grid）下。

Oracle Grid Infrastructure基础（网格的ORACLE_BASE ）目录和主目录（ ORACLE_HOME for grid）目录不能位于Oracle ACFS文件系统上，因为在安装Oracle Grid Infrastructure之前无法创建Oracle ACFS文件系统。

Oracle ACFS上的一个或多个Oracle数据库主目录可以在相同的安装点下创建，每个主目录使用单独的Oracle ACFS文件系统。

安装Oracle Grid Infrastructure软件之后，在使用Oracle Universal Installer（OUI）安装Oracle数据库软件之前，您可以创建要配置为用作Oracle数据库主目录的Oracle ACFS文件系统。

您还可以使用Oracle ASM Configuration Assistant（ASMCA）或Oracle ACFS命令来创建文件系统。

注意：

当Oracle ACFS文件系统包含Oracle数据库主目录或Oracle数据库使用文件系统进行任何文件存储时，文件系统必须具有Oracle ACFS文件系统资源。如果尚未使用Oracle ASM Configuration Assistant设置安装点，则必须使用服务器控制实用程序（SRVCTL）命令来设置Oracle数据库依赖关系。

在Oracle Grid Infrastructure集群件配置中，将Oracle ACFS用于数据库主目录时，需要集群范围的Oracle ACFS资源。要使数据库所有者能够对资源执行操作，必须在创建资源时将所有者指定为允许的用户。您可以使用srvctl add filesystem或acfsutil registry命令的-u选项指定数据库所有者。在Linux或Unix环境中添加资源时需要root权限。

创建Oracle ACFS文件系统和资源后，可以通过在Oracle Universal Installer（OUI）数据库软件安装期间浏览并选择目录，将Oracle基于Oracle ACFS的数据库主安装点位置选择为Oracle数据库主位置。

您可以使用srvctl start filesystem命令手动安装Oracle ACFS文件系统。

注意：

在创建后将其他Oracle ACFS文件系统添加到Oracle数据库时，必须在Oracle数据库资源依赖关系列表中指定它们。如果不这样做会导致应用程序可靠性出错。要在依赖关系列表中指定文件系统，请使用SRVCTL数据库对象命令修改资源使用的Oracle ACFS路径。

Oracle ACFS文件系统也可以配置为用作应用程序的主目录。但是，Oracle ACFS文件系统不能用于包含Oracle Clusterware，Oracle ASM，Oracle ACFS和Oracle ADVM组件软件的Oracle基本目录或Oracle Grid Infrastructure主目录。

为了减少在Oracle ACFS上共享Oracle数据库主目录的Oracle RAC环境中对Oracle ACFS文件系统的争用，应将Oracle数据库审计操作系统文件配置为特定于节点。对于特定于节点的设置，必须确保每个数据库实例的配置文件中的AUDIT_FILE_DEST初始化参数指向唯一的位置，而不是所有数据库实例的一个位置。

例如，如果您有一个Oracle名称设置为TEST的数据库，并且您希望确保每个数据库实例（例如TEST1或TEST2的AUDIT_FILE_DEST初始化参数的位置指向该实例的节点特定位置，则可以运行以下SQL语句：

SQL> ALTER SYSTEM SET AUDIT_FILE_DEST ='$ ORACLE_BASE / admin / adump / TEST / @'SCOPE = SPFILE SID ='*';

在前面的示例中， @扩展为每个实例的ORACLE_SID 。如果在此示例/acfsmounts ORACLE_BASE设置为/acfsmounts ，则可以使用该值代替ORACLE_BASE变量。

关于Oracle ACFS Mount Registry

Oracle ACFS装入注册表支持Oracle Grid Infrastructure群集配置，但不支持Oracle Restart配置。

关于集群域中的Oracle ACFS

除了支持具有附加本地存储的Oracle数据库成员集群之外，Oracle ACFS还为Oracle数据库成员集群上的本机Oracle ACFS功能提供Oracle ACFS远程服务，而无需连接本地存储（InDirect存储成员集群）。利用Oracle域服务集群（DSC）上的Oracle ACFS部署，Oracle ACFS远程服务可用于Oracle数据库应用程序集群和Oracle数据库成员集群，从而实现灵活的基于文件系统的应用程序和数据库部署。与基于NFS的导出不同，Oracle ACFS远程服务完全支持高级Oracle ACFS功能;例如复制，快照和标记;在目标成员集群上。

Oracle数据库成员集群

可以使用或不使用本地存储来配置Oracle数据库成员集群。

  • 具有附加本地存储的Oracle数据库成员集群

    可以使用本地存储配置Oracle数据库成员集群。这些群集使用来自域服务群集的GIMR，其操作类似于独立的Oracle ASM群集。这些集群中的Oracle ACFS通过完整的Oracle ASM代理和所有传统工具正常运行。这些成员集群管理自己的存储。

  • 没有附加本地存储的Oracle数据库成员集群

    也可以配置Oracle数据库成员集群而无需连接本地存储。这些InDirect存储成员群集利用来自域服务群集（DSC）的整合存储服务。要使用Oracle ACFS文件系统和Oracle ADVM卷，Oracle数据库成员集群使用域服务集群中的Oracle ACFS远程服务。

    InDirect存储成员集群提供可以运行传统数据库或其他应用程序的小型轻量级Oracle Clusterware（CRS）堆栈。此成员集群占用的内存更少，更易于配置，并且可以轻松地托管在VM中，因为没有直接的附加存储要求。InDirect Storage成员集群非常适合资源和应用程序整合。在此方案中，Oracle ACFS Remote可在域服务群集上启用完全存储和资源整合，从而实现新一代紧凑且廉价的计算资源。

Oracle ACFS远程服务

Oracle ACFS远程服务支持在InDirect存储成员集群中使用Oracle ACFS。Oracle ACFS远程服务作为DSC的另一项服务提供。由于InDirect存储成员集群没有直接连接到Oracle数据库成员集群的存储，因此Oracle ACFS远程服务在DSC和成员集群之间提供了启用Oracle ACFS的存储快速路径，以便通过标准块传输使用Oracle ACFS。

当Oracle ACFS远程服务正在使用时，Oracle数据库成员集群可以访问所有Oracle ACFS功能，包括快照，调整大小，Oracle数据库和其他POSIX应用程序，以及ACFS NAS | MAX，标记，检查，修复，清理，安全性，加密和压缩。当应用程序在成员集群上使用Oracle ACFS远程服务导出功能时，Oracle ACFS在本地或远程使用之间不存在功能或程序上的差异。此外，所有Oracle ACFS工具和实用程序都继续正常运行。作为其默认配置的一部分，Oracle ACFS远程服务提供了高可用性的导出。如果传输或DSC节点不可用，则操作将在DSC的其他节点上继续。此延续提供了成员集群操作的完全故障转移和可用性。Oracle ACFS Remote也以类似方式处理临时传输级别中断。

存储库，导出和传输

存储库，导出和传输是Oracle ACFS远程服务的组成部分。

  • 知识库

    存储库是DSC上的Oracle ACFS文件系统中的目录，用作导出的主机。文件系统是标准的Oracle ACFS文件系统，可以将标准的Oracle ACFS功能应用于文件系统，包括调整大小，复制和快照。由于它是Oracle ACFS文件系统，因此存储库受到1 PB的Oracle ACFS文件系统限制。如果Oracle ACFS文件系统上所有存储库中预期的后备文件总数超过文件系统大小，则需要多个存储库。

    存储库不支持Oracle ACFS文件压缩。

  • 出口

    DSC提供出口。这些导出作为后备文件启动，这些文件驻留在DSC上的Oracle ACFS远程服务存储库中。然后使用传输导出该文件，以便只有预期的成员群集节点才能使用该传输访问导出。使用advmutil export系列命令管理export 。在Oracle ACFS远程服务正在使用文件之后，在Oracle ACFS远程服务将文件从中删除之前，无法删除该文件。

  • 运输

    Oracle ACFS远程服务使用传输来提供对导出的访问。对于Oracle ASM版本18c，版本18.1，仅支持iSCSI传输。此传输使用小型传输定义文件进行配置，并由Oracle ACFS远程服务自动管理。使用advmutil transport系列命令管理transport 。

关于Oracle ACFS快照

Oracle ACFS快照是Oracle ACFS文件系统的在线，只读或读写时间点副本。

快照副本节省空间并使用写时复制功能。在修改或删除Oracle ACFS文件范围之前，会将其当前值复制到快照以维护文件系统的时间点视图。

Oracle ACFS快照在创建后立即可用。快照是在.中创建的.文件系统的ACFS/snaps/目录。在安装文件系统时，它们始终在线。因此，Oracle ACFS快照可以支持无意中修改或从文件系统中删除的文件的联机恢复。Oracle ACFS快照也可以用作文件系统备份的源，因为它可以根据需要创建，以提供活动文件系统的当前一致的在线视图。

每个文件系统最多支持1023个只读，读写或只读和读写快照视图的组合，从而可以使用跨多个视图的灵活的在线文件恢复解决方案。在64位系统上，Oracle ACFS支持1023个快照。快照总数可以是只读和读写快照的任意组合。要支持创建63个以上的快照， ADVM的磁盘组兼容性属性必须设置为12.1.0.2或更高版本。此外，必须满足以下条件才能创建超过63个快照。

  • 必须删除使用COMPATIBLE.ADVM设置为小于12.1.0.2创建的文件系统的所有快照。

  • 必须删除在存在较旧快照时将COMPATIBLE.ADVM设置为12.1.0.2或更高版本后创建的文件系统的所有快照。

Oracle ACFS读写快照可以快速创建可以读取和写入的快照映像，而不会影响托管快照映像的Oracle ACFS文件系统的状态。您可以使用读写快照：

  • 在生成文件数据上测试新版本的应用程序软件，反映在读写快照映像中，而无需修改原始生产文件系统

  • 在不修改原始生产文件系统的情况下在实际数据集上运行测试方案

19.1对象关系存储的XPath重写概述

Oracle XML DB通常可以优化使用XPath表达式的查询 - 例如，涉及SQL函数的查询，例如XMLQuery ， XMLTable和XMLExists ，它们将XPath（XQuery）表达式作为参数。实际上，XPath表达式是针对XML文档进行评估的，而无需在内存中构建XML文档。

此优化称为XPath重写 。它是XML查询优化的适当子集，它还涉及XQuery表达式的优化，例如FLWOR表达式，它们不是XPath表达式。XPath重写还允许优化器在查询评估中使用列（如果存在于列上）。

可以由Oracle XML DB重写的XPath表达式是Oracle XML DB支持的那些表达式的适当子集。只要您可以在不丢失功能的情况下执行此操作，请使用可以重写的XPath表达式。

XPath重写可以在这些上下文（或其组合）中发生：

  • 当XMLType数据存储在对象关系列或表中时，或者在关系数据上构建XMLType视图时。

  • 使用XMLIndex索引时。

第一种情况是重写使用对象关系XML数据或XMLType视图的查询，这里将介绍。XMLType视图可以是基于XML模式的，也可以不是。XMLType数据的对象关系存储始终基于XML模式。本章中的示例与基于XML模式的表有关。

当对象关系XML数据可以进行XPath重写时，数据库优化器可以基于传统的关系代数推导出执行计划。这反过来意味着Oracle XML DB可以利用数据库的所有功能，并确保包含XQuery和XPath表达式的SQL语句以高性能和高效的方式执行。这种重写几乎没有开销，因此Oracle XML DB以接近关系的速度执行基于XQuery和基于XPath的查询。

在某些情况下，XPath重写是不可能的。当没有与XPath表达式等效的SQL时，通常会发生这种情况。在这种情况下，Oracle XML DB会对XPath表达式执行功能评估，这通常成本更高，尤其是在要处理的文档数量很大的情况下。

19.4使用执行计划分析和优化XPath查询的指南

同时使用这些指南，将所有适用的考虑在内。

  • 指南：在执行计划中查找基础表与XML函数
    重写的查询的执行计划是指作为查询的XMLType数据的基础的对象关系表和列的名称。如果这些名称是从XML元素或属性名称派生的，或者如果使用XML Schema批注xdb:defaultTable则这些名称对您有意义。
  • 准则：命名对象关系表，以便在执行计划中识别它们
    在设计XML模式时，请使用注释xdb:defaultTable来命名与在性能很重要的查询中选择的元素对应的基础表。这使您可以在执行计划中轻松识别它们，通过它们是否存在来指示查询是否已被重写。
  • 准则：在谓词所针对的列上创建索引
    通过创建应用于谓词所针对的列的索引，有时可以提高重写的查询的性能，使其包含SQL谓词。
  • 指南：在有序集合表上创建索引
    如果集合存储为有序集合表（OCT）或XMLType实例，则可以直接访问集合的成员。每个成员都成为一个表行，因此您可以使用SQL直接访问它。您通常可以通过索引此类集合成员来提高性能。

    如果尚未优化查询，则可以使用系统变量XMLOptimizationCheck来尝试确定原因。

19.4.4指南：在有序集合表上创建索引

如果集合存储为有序集合表（OCT）或XMLType实例，则可以直接访问集合的成员。每个成员都成为一个表行，因此您可以使用SQL直接访问它。您通常可以通过索引此类集合成员来提高性能。

您可以通过在（a）与集合XML元素或其属性对应的对象属性和（b）伪列NESTED_TABLE_ID上创建复合索引来完成此NESTED_TABLE_ID 。

例19-11显示了查询的执行计划，以查找包含部件号717951002372（具有值属性值717951002372的Id属性的Part元素）的订单的文档中的Reference元素。LineItem元素的集合作为行存储在有序集合表lineitem_table 。

注意：

示例19-11不使用示例数据库模式OE的purchaseorder表。它使用purchaseorder表，该表使用名为lineitem_table的有序集合表（OCT）作为集合元素LineItem 。

272 ANYDATASET TYPE

ANYDATASET TYPE包含给定类型的描述以及该类型的一组数据实例。如果需要， ANYDATASET可以持久存储在数据库中，或者它可以用作接口参数来传递自描述数据集，所有这些数据都属于某种类型。

272.1 ANDDATASET TYPE结构

ANYDATASET需要按值依次构建。

对于每个数据实例（ ANYDATASET的类型），必须调用ADDINSTANCE函数。这会向ANYDATASET添加一个新的数据实例。随后，可以调用SET*来完整地设置每个值。

通过调用PIECEWISE.可以将构造/访问MODE更改为属性/集合元素PIECEWISE.

  • 如果ANYDATASET的类型是TYPECODE_OBJECT ，则将使用后续SET*调用设置各个属性。同样在访问。

  • 如果当前数据值的类型是集合类型，则将使用后续SET*调用设置单个集合元素。同样在访问。这个调用非常相似ANYDATA.PIECEWISE对该类型的呼叫ANYDATA 。

请注意，不支持分段构造和访问对象类型或集合类型的嵌套（非顶级）属性。

应该调用ENDCREATE来完成构建过程（在此之前不能进行访问调用）。

选择网关的系统标识符

该网关系统标识符 （SID）是标识网关实例的字母数字字符串。对于要访问的每个Informix数据库，您需要一个网关实例，因此需要一个网关SID。SID用作初始化参数文件的文件名的一部分。默认SID是 dg4ifmx 。

您可以定义网关SID，但使用默认的dg4ifmx更容易，因为您不需要更改初始化参数文件名。但是，如果要访问两个Informix数据库，则需要两个网关SID，一个用于网关的每个实例。如果您只有一个Informix数据库并且希望有时使用一组网关参数设置来访问它，而其他时候使用不同的网关参数设置，那么您将需要多个网关SID用于单个Informix数据库。

自定义初始化参数文件

启动网关时，初始化参数文件必须可用。在安装过程中，将创建以下默认初始化参数文件：

 ORACLE_HOME \ dg4ifmx \ admin \ initdg4ifmx.ora

其中ORACLE_HOME是安装网关的目录。

此初始化文件用于默认网关SID。如果未使用dg4ifmx作为网关SID，则必须使用在上一步中选择的SID重命名初始化参数文件选择网关的系统标识符 。此默认初始化参数文件足以启动网关，验证安装是否成功以及运行演示脚本。

可以使用许多初始化参数来修改网关行为。有关可以设置的初始化参数的完整列表，请参阅初始化参数 。对初始化参数所做的更改仅在下一个网关会话中生效。最重要的参数是HS_FDS_CONNECT_INFO ，它描述了与非Oracle系统的连接。

默认初始化参数文件已有此参数的条目。HS_FDS_CONNECT_INFO的语法如下：

创建数据库链接

连接到Oracle数据库的任何Oracle客户端都可以通过网关访问Informix数据。Oracle客户端和Oracle数据库可以驻留在不同的计算机上。网关仅接受来自Oracle数据库的连接。

首次在Oracle会话中使用时，通过数据库链接建立与网关的连接。在此上下文中，连接是指Oracle数据库与网关之间的连接。在Oracle会话结束之前，连接将保持建立状态。另一个会话或用户可以访问相同的数据库链接，并获得与网关和Informix数据库的不同连接。

4.1关于使用主机变量

Oracle使用主机变量将数据和状态信息传递给您的程序;您的程序使用主机变量将数据传递给Oracle。

4.1.1输出与输入主机变量

根据它们的使用方式，主变量称为输出或输入主变量。SELECT或FETCH语句的INTO子句中的主机变量称为输出主机变量，因为它们包含Oracle输出的列值。Oracle将列值分配给INTO子句中的相应输出主机变量。

SQL语句中的所有其他主机变量都称为输入主机变量，因为您的程序将其值输入到Oracle。例如，您在INSERT语句的VALUES子句和UPDATE语句的SET子句中使用输入主机变量。它们也用在WHERE ， HAVING和FOR子句中。事实上，只要允许值或表达式，输入主变量就可以出现在SQL语句中。

在ORDER BY子句中，您可以使用主机变量，但它被视为常量或文字，因此主机变量的内容无效。例如，SQL语句

EXEC SQL SELECT ename，empno INTO：name，：number FROM emp ORDER BY：ord;

似乎包含输入主机变量ord 。但是，在这种情况下，主变量被视为常量，并且无论ord的值如何，都不进行排序。

您不能使用输入主机变量来提供SQL关键字或数据库对象的名称。因此，您不能在数据定义语句（有时称为DDL ）中使用输入主机变量，例如ALTER ， CREATE和DROP 。在以下示例中， DROP TABLE语句无效 ：

EXEC SQL BEGIN DECLARE SECTION; table_name CHARACTER（30）; EXEC SQL END DECLARE SECTION;显示'表名？“; read table_name; EXEC SQL DROP TABLE：table_name; - 不允许使用主变量

在Oracle执行包含输入主机变量的SQL语句之前，程序必须为它们分配值。请考虑以下示例：

EXEC SQL BEGIN DECLARE SECTION; emp_number INTEGER; emp_name CHARACTER（20）; EXEC SQL END DECLARE SECTION; - 获取输入主变量的值显示'员工编号？“;读取emp_number;显示'员工姓名？“;读取emp_name; EXEC SQL INSERT INTO EMP（EMPNO，ENAME）VALUES（：emp_number，：emp_name）;

请注意， INSERT语句的VALUES子句中的输入主机变量以冒号为前缀。

4.2关于使用指标变量

您可以将任何主机变量与可选的指示符变量相关联。每次在SQL语句中使用主机变量时，结果代码都存储在其关联的指示符变量中。因此，指标变量可让您监控主变量。

您可以在VALUES或SET子句中使用指示符变量将空值分配给输入主机变量，并在INTO子句中检测输出主机变量中的空值或截断值。

4.2.1输入变量

对于输入主变量，程序可以赋给指标变量的值具有以下含义：

  • -1：Oracle将为列分配null，忽略主变量的值。

  • > = 0： Oracle会将主变量的值分配给列。

4.2.2输出变量

输出主变量，Oracle可以赋给指标变量的值具有以下含义：

  • -2：Oracle分配了一个截断列值到主变量，但无法将列值的原始长度分配给指示符变量，因为数字太大。

  • -1：列值为null，因此宿主变量的值是不确定的。

  • 0：Oracle为主机变量分配了完整的列值。

  • > 0：Oracle为主变量分配了一个截断的列值，将原始列长度（以字节，而不是字节表示，用于多字节全球化支持主机变量）分配给指示符变量，并将SQLCA中的SQLCODE设置为零。

请记住，必须在声明部分中将指示符变量定义为2字节整数，并且在SQL语句中，必须以冒号为前缀并附加到其主变量（除非使用关键字INDICATOR）。

4.2.3插入空格

您可以使用指示符变量来插入空值。在插入之前，对于每个要为null的列，将相应的指示符变量设置为-1，如以下示例所示：

设置ind_comm = -1; EXEC SQL INSERT INTO EMP（EMPNO，COMM）VALUES（：emp_number，：commission：ind_comm）;

指示符变量ind_comm指定将在COMM列中存储null。

您可以硬编码null，如下所示：

EXEC SQL INSERT INTO EMP（EMPNO，COMM）VALUES（：emp_number，NULL）;

虽然这不太灵活，但可能更具可读性。

通常，您有条件地插入空值，如下一个示例所示：

显示'输入员工编号，如果不可用则输入0：';读取emp_number;如果emp_number = 0那么设置ind_empnum = -1; ELSE设置ind_empnum = 0;万一; EXEC SQL INSERT INTO EMP（EMPNO，SAL）VALUES（：emp_number：ind_empnum，：salary）;

4.2.4处理返回的空值

您还可以使用指示符变量来操作返回的空值，如以下示例所示：

EXEC SQL SELECT ENAME，SAL，COMM INTO：emp_name，：salary，：commission：ind_comm FROM EMP WHERE EMPNO =：emp_number;如果ind_comm = -1则设置pay = salary; - 佣金为空;忽略它ELSE设定薪水=工资+佣金;万一;

4.2.5获取空值

当DBMS = NATIVE，V7或V8时，如果在缺少指示符变量的主机变量中选择或获取空值，Oracle会发出以下错误消息：

ORA-01405：获取的列值为NULL

4.2.6测试空值

您可以在WHERE子句中使用指示符变量来测试空值，如以下示例所示：

EXEC SQL SELECT ENAME，SAL INTO：emp_name，：salary FROM EMP WHERE：commission：ind_comm IS NULL ...

但是，您不能使用关系运算符来比较空值或其他值。例如，如果COMM列包含一个或多个空值，则以下SELECT语句将失败：

EXEC SQL SELECT ENAME，SAL INTO：emp_name，：salary FROM EMP WHERE COMM =：commission：ind_comm;

下一个示例显示了当其中一些值可能为空时，如何比较相等的值：

EXEC SQL SELECT ENAME，SAL INTO：emp_name，：salary FROM EMP WHERE（COMM =：commission）OR（（COMM IS NULL）AND（：commission：ind_comm IS NULL））;

4.2.7获取截断值

如果在缺少指示符变量的主变量中选择或获取截断的列值，则不会生成错误。

4.3基本SQL语句

可执行的SQL语句使您可以查询，操作和控制Oracle数据创建，定义和维护Oracle对象，例如表，视图和索引。本章重点介绍数据操作语句（有时称为DML ）和游标控制语句。以下SQL语句允许您查询和操纵Oracle数据：

  • SELECT ：返回一个或多个表中的行。

  • INSERT ：向表中添加新行。

  • UPDATE ：修改表中的行。

  • DELETE ：从表中删除行。

在执行INSERT ， UPDATE或DELETE等数据操作语句时，除了设置任何输入主机变量的值之外，唯一需要考虑的是语句是成功还是失败。要找到答案，您只需查看SQLCA即可。 （执行任何SQL语句都会设置SQLCA变量。）您可以通过以下两种方式办理登机手续：

  • 使用WHENEVER语句进行隐式检查

  • 显式检查SQLCA变量

或者，当MODE={ANSI|ANSI14} ，您可以检查状态变量SQLSTATE或SQLCODE 。有关更多信息，请参阅“ 关于在MODE = {ANSI | ANSI14}时使用状态变量 ” 。

执行SELECT语句时（ 但是，您还必须处理它返回的数据行。查询可分为以下几类：

  • 不返回任何行的查询（即仅检查是否存在）

  • 只返回一行的查询

  • 返回多行的查询

返回多行的查询需要显式声明的游标或游标变量（或使用主机数组，这在使用主机阵列中讨论）。以下嵌入式SQL语句允许您定义和控制显式游标：

  • DECLARE ：命名游标并将其与查询相关联。

  • OPEN ：执行查询并识别活动集。

  • FETCH ：前进游标并逐个检索活动集中的每一行。

  • CLOSE ：禁用光标（活动集变为未定义）。

在以下部分中，首先将学习如何编写INSERT ， UPDATE ， DELETE和单行SELECT语句的代码。然后，您将进入多行SELECT语句。

4.3.1选择行

查询数据库是一种常见的SQL操作。要发出查询，请使用 SELECT语句。在下面的例如，您查询EMP表：

EXEC SQL SELECT ENAME，JOB，SAL + 2000 INTO：emp_name，：job_title，：salary FROM EMP WHERE EMPNO =：emp_number;

关键字SELECT后面的列名和表达式构成选择列表 。我们示例中的选择列表包含三个项目。在WHERE子句（以及以下子句，如果存在）中指定的条件下，Oracle将列值返回到INTO子句中的主变量。选择列表中的项目数应该等于INTO子句中的主机变量数，因此可以存储每个返回的值。

在最简单的情况下，当查询返回一行时，其形式如上一个示例所示（其中EMPNO是唯一键）。但是，如果查询可以返回多行，则必须使用游标获取行或将其选择到主机数组中。

如果您编写查询以仅返回一行但实际上可能返回多行，则结果取决于您指定SELECT_ERROR选项的方式。当SELECT_ERROR = YES （缺省值）时，如果返回多行，Oracle会发出以下错误消息：

ORA-01422：精确提取返回超过请求的行数

当SELECT_ERROR=NO ，返回一行，Oracle不会生成错误。

4.3.2可用条款

您可以在SELECT语句中使用以下所有标准SQL子句： INTO ， FROM ， WHERE ， CONNECT BY ， START WITH ， GROUP BY ， HAVING ， ORDER BY和FOR UPDATE OF 。

4.3.3插入行

您可以使用INSERT语句将行添加到表或视图中。在以下示例中，您向EMP表添加一行：

EXEC SQL INSERT INTO EMP（EMPNO，ENAME，SAL，DEPTNO）VALUES（：emp_number，：emp_name，：salary，：dept_number）;

您在列列表中指定的每个列都必须属于在该列表中指定的表 INTO 条款。VALUES子句指定要插入的值行。值可以是常量，主变量，SQL表达式或伪列，例如USER和SYSDATE 。

VALUES子句中的VALUES数必须等于列列表中的名称数。但是，如果VALUES子句包含表中每列的值，则可以省略列列表，其顺序与CREATE TABLE定义的顺序相同。

4.3.4使用子查询

一个 子查询是一个嵌套的SELECT语句。子查询允许您进行多部分搜索。他们可以习惯

  • 提供SELECT ， UPDATE和DELETE语句的WHERE ， HAVING和START WITH子句中的比较值

  • 定义要由CREATE TABLE或INSERT语句插入的行集

  • 定义UPDATE语句的SET子句的值

对于例如，要将行从一个表复制到另一个表，请将INSERT语句中的VALUES子句替换为子查询，如下所示：

EXEC SQL INSERT INTO EMP2（EMPNO，ENAME，SAL，DEPTNO）从EMP中选择EMPNO，ENAME，SAL，DEPTNO WHERE JOB =：job_title;

请注意INSERT语句如何使用子查询来获取中间结果。

4.3.5更新行

你用的是 UPDATE语句，用于更改表或视图中指定列的值。在以下示例中，您将更新EMP表中的SAL和COMM列：

EXEC SQL UPDATE EMP SET SAL =：salary，COMM =：Commission WHERE EMPNO =：emp_number;

您可以使用可选的WHERE子句指定更新行的条件。请参阅“ 使用WHERE子句 ” 。

该 SET子句列出了必须为其提供值的一个或多个列的名称。您可以使用子查询来提供值，如以下示例所示：

EXEC SQL UPDATE EMP SET SAL =（SELECT AVG（SAL）* 1.1 from EMP WHERE DEPTNO = 20）WHERE EMPNO =：emp_number;

4.3.6删除行

您使用DELETE语句从表或视图中删除行。在以下示例中，您将从EMP表中删除给定部门中的所有员工：

EXEC SQL DELETE FROM EMP WHERE DEPTNO =：dept_number;

您可以使用可选的WHERE子句指定删除行的条件。

4.3.7使用WHERE子句

您可以使用WHERE子句仅选择，更新或删除满足搜索条件的表或视图中的那些行。WHERE -clause 搜索条件是一个布尔表达式，可以包括标量主变量，主机数组（不在SELECT语句中）和子查询。

如果省略WHERE子句，则处理表或视图中的所有行。如果省略UPDATE或DELETE语句中的WHERE子句，Oracle会将SQLCA中的SQLWARN（5）设置为“W”，以警告所有行都已处理。

4.4游标

当查询返回多个时行，您可以显式定义游标：

  • 处理超出查询返回的第一行

  • 跟踪当前正在处理的行

游标标识查询返回的行集中的当前行。这允许您的程序一次处理一行。以下语句允许您定义和操作游标：

  • 宣布

  • 打开

  • 取

  • 关

第一您使用DECLARE语句来命名游标并将其与查询相关联。

OPEN语句执行查询并标识满足查询搜索条件的所有行。这些行形成一个称为光标活动集的集合。打开游标后，您可以使用它来检索其关联查询返回的行。

逐个检索活动集的行（除非您使用主机数组）。您使用FETCH语句来检索活动集中的当前行。您可以重复执行FETCH ，直到检索到所有行。

完成从活动集中获取行时，使用CLOSE语句禁用游标，并且活动集变为未定义。

4.4.1声明游标

您可以使用DECLARE语句通过为游标指定名称并将其与查询相关联来定义游标，如以下示例所示：

 EXEC SQL DECLARE emp_cursor CURSOR FOR SELECT ENAME，EMPNO，SAL from EMP WHERE DEPTNO =：dept_number;

该游标名称是预编译器使用的标识符， 而不是主机或程序变量，不应在声明部分中定义。因此，游标名称不能从一个预编译单元传递到另一个预编译单元。此外，游标名称不能连字符。它们可以是任意长度，但只有前31个字符才有意义。对于ANSI兼容性，请使用不超过18个字符的游标名称。

与游标关联的SELECT语句不能包含INTO子句。相反， INTO子句和输出主机变量列表是FETCH语句的一部分。

因为它是声明性的，所以DECLARE语句必须在物理上（不仅仅是逻辑上）在引用游标的所有其他SQL语句之前。那是， 不允许对游标进行前向引用。在以下示例中， OPEN语句放错了位置：

EXEC SQL OPEN emp_cursor; - 错误的OPEN语句EXEC SQL DECLARE emp_cursor CURSOR FOR SELECT ENAME，EMPNO，SAL FROM EMP WHERE ENAME =：emp_name;

游标控制语句必须全部出现在同一预编译单元中。例如，您不能在文件A中声明游标，然后在文件B中打开它。

您的宿主程序可以根据需要声明尽可能多的游标。但是，在给定文件中，每个DECLARE语句都必须是唯一的。也就是说，您不能在一个预编译单元中声明两个具有相同名称的游标，即使是跨块或过程也是如此，因为范围光标在文件中是全局的。 如果您将使用多个游标，则可能需要指定MAXOPENCURSORS选项。有关更多信息，请参阅“ MAXOPENCURSORS ” 。

4.4.2打开光标

使用 OPEN语句，用于执行查询并识别活动集。在下面的例如，打开名为emp_cursor的游标。 

EXEC SQL OPEN emp_cursor;

OPEN将光标定位在活动集的第一行之前。它还将SQLCA中SQLERRD（3）保留的行处理计数归零。但是，此时实际上没有检索到任何行。这将由FETCH语句完成。

打开游标后，在重新打开游标之前，不会重新检查查询的输入主机变量。因此，活动集不会改变。要更改活动集，必须重新打开游标。

通常，您应该在重新打开之前关闭游标。但是，如果指定MODE=ORACLE （默认值），则无需在重新打开之前关闭游标。这可以提高性能;有关详细信息，请参阅性能调整

OPEN完成的工作量取决于三个预编译器选项的值： HOLD_CURSOR, RELEASE_CURSOR和MAXOPENCURSORS 。有关更多信息，请参阅“ 关于使用预编译器选项 ” 。

4.4.3从光标中获取

您可以使用FETCH语句从活动集中检索行，并指定将包含结果的输出主机变量。回想一下，与游标关联的SELECT语句不能包含INTO子句。相反， INTO子句和输出主机变量列表是FETCH语句的一部分。在以下示例中，您将获取三个主机变量：

EXEC SQL FETCH emp_cursor INTO：emp_name，：emp_number，：salary;

必须先声明并打开游标。第一次执行FETCH ，光标从活动集中的第一行之前移动到第一行。该行成为当前行。FETCH每次后续执行都会将光标前进到活动集中的下一行，从而更改当前行。光标只能在活动集中向前移动。要返回已经获取的行，必须重新打开游标，然后在活动集的第一行再次开始。

如果要更改活动集，则必须将新值分配给与游标关联的查询中的输入主机变量，然后重新打开游标。当MODE={ANSI | ANSI14 | ANSI13} ，您必须在重新打开之前关闭光标。

如下一个示例所示，您可以使用不同的输出主机变量集从同一游标中获取。但是，每个FETCH语句的INTO子句中的相应主机变量必须具有相同的数据类型。 

EXEC SQL DECLARE emp_cursor CURSOR FOR SELECT ENAME，SAL FROM EMP WHERE DEPTNO = 20; ...EXEC SQL OPEN emp_cursor; EXEC SQL WHENEVER NOT FOUND DO ...LOOP EXEC SQL FETCH emp_cursor INTO：emp_name1，：salary1; EXEC SQL FETCH emp_cursor INTO：emp_name2，：salary2; EXEC SQL FETCH emp_cursor INTO：emp_name3，：salary3; ...ENDLOOP;

如果如果活动集为空或不包含更多行，则FETCH将“无数据找到”Oracle警告代码返回到SQLCA中的SQLCODE （或者当MODE = ANSI时，返回到状态变量SQLSTATE ）。输出主机变量的状态是不确定的。（在典型程序中， WHENEVER NOT FOUND语句检测到此错误。）要重新使用游标，必须重新打开它。

4.4.4关闭光标

完成从活动集中获取行后，关闭游标以释放通过打开游标获取的资源，例如存储。关闭游标时，将释放解析锁。释放的资源取决于您如何指定HOLD_CURSOR和RELEASE_CURSOR选项。在以下示例中，您将关闭名为emp_cursor的游标： 

EXEC SQL CLOSE emp_cursor;

您无法从关闭的游标中获取，因为它的活动集变为未定义。如有必要，您可以重新打开游标（例如，使用输入主机变量的新值）。

当MODE={ANSI13|ORACLE} ，发出提交或回滚将关闭CURRENT OF子句中引用的游标。其他游标不受提交或回滚的影响，如果打开，则保持打开状态。但是，当MODE={ANSI|ANSI14} ，发出提交或回滚将关闭所有显式游标。

4.4.5使用CURRENT OF子句

您可以在DELETE或UPDATE语句中使用CURRENT OF cursor_name子句来引用从命名游标中获取的最新行。光标必须打开并放在一行上。如果未执行提取或游标未打开，则CURRENT OF子句将导致错误并且不处理任何行。

声明在UPDATE或DELETE语句的CURRENT OF子句中引用的游标时， FOR UPDATE OF子句是可选的。CURRENT OF子句指示预编译器在必要时添加FOR UPDATE子句。有关更多信息，请参阅“ 关于使用FOR UP UPDATE OF子句 ” 。

在以下示例中，您使用CURRENT OF子句来引用从名为emp_cursor的游标中获取的最新行：

EXEC SQL DECLARE emp_cursor CURSOR FOR SELECT ENAME，SAL from EMP WHERE JOB ='CLERK'FOR SALDATE SAL; ...EXEC SQL OPEN emp_cursor; EXEC SQL WHENEVER NOT FOUND DO ...LOOP EXEC SQL FETCH emp_cursor INTO：emp_name，：salary; ...EXEC SQL UPDATE EMP SET SAL =：new_salary WHERE CURRENT OF emp_cursor; ENDLOOP;

4.4.6限制

显式FOR UPDATE OF或隐式FOR UPDATE获取独占行锁。所有行都在打开时锁定，而不是在提取时锁定，并在提交或回滚时释放。如果在提交后尝试从FOR UPDATE游标获取，Oracle会生成以下错误：

ORA-01002：取消顺序

您不能将主机数组与CURRENT OF子句一起使用。有关替代方案，请参阅“ 关于模仿当前条款 ” 。此外，您不能在关联的FOR UPDATE OF子句中引用多个表，这意味着您无法与CURRENT OF子句进行连接。最后，您不能在动态SQL中使用CURRENT OF子句。

以下示例显示了应用程序中光标控制语句的典型顺序：

- 定义光标。EXEC SQL DECLARE emp_cursor CURSOR FOR SELECT ENAME，JOB from EMP WHERE EMPNO =：emp_number FOR JODATE OF JOB; - 打开光标并识别活动集。EXEC SQL OPEN emp_cursor; - 如果最后一行已被提取，则退出。EXEC SQL WHENEVER NOT FOUND DO no_more; - 循环中获取和处理数据。LOOP EXEC SQL FETCH emp_cursor INTO：emp_name，：job_title; - 对获取的数据进行操作的主机语言语句EXEC SQL UPDATE EMP SET JOB =：new_job_title WHERE CURRENT OF emp_cursor; ENDLOOP; ...ROUTINE no_more BEGIN  - 禁用光标。EXEC SQL CLOSE emp_cursor; EXEC SQL COMMIT WORK RELEASE;退出计划;结束no_more;

4.5光标变量

本节简要概述了游标变量。有关更多信息，请参阅宿主语言补充和光标变量 。

将静态嵌入式SQL与Pro * COBOL和Pro * FORTRAN预编译器一起使用时，可以声明游标变量。与游标一样，游标变量指向多行查询的活动集中的当前行。游标与常量与变量的不同之处不同于游标变量。当游标是静态的时，游标变量是动态的，因为它不依赖于特定的查询。您可以为任何类型兼容的查询打开游标变量。

此外，您可以为游标变量分配新值，并将其作为参数传递给子程序，包括存储在Oracle数据库中的子程序。这为您提供了一种集中数据检索的便捷方式。

首先，声明游标变量。声明变量后，使用四个语句来控制游标变量：

  • 分配

  • 打开......对于

  • 取

  • 关

声明游标变量并为其分配内存后，必须将其作为输入主机变量（绑定变量）传递给PL / SQL，打开它以便在服务器端进行多行查询，在客户端进行FETCH查询，然后CLOSE它的任何一方。

4.5.1关于声明游标变量

声明游标变量的方式取决于您的宿主语言。有关声明游标变量的说明，请参阅主机语言补充。

4.5.2分配游标变量

您使用ALLOCATE语句为游标变量分配内存。语法如下：

EXEC SQL ALLOCATE <cursor_variable>;

4.5.3打开游标变量

你用的是 OPEN ...FOR语句将游标变量与多行查询相关联，执行查询并标识活动集。语法如下：

EXEC SQL OPEN <cursor_variable> FOR <select_statement>;

SELECT语句可以引用输入主机变量和PL / SQL变量，参数和函数，但不能是FOR UPDATE 。在以下示例中，您将打开名为emp_cv的游标变量：

EXEC SQL OPEN emp_cv FOR SELECT * FROM EMP;

您必须在服务器端打开游标变量。您可以通过将其作为输入主机变量传递给匿名PL / SQL块来实现。在运行时，该块将发送到Oracle Server以供执行。在下面的示例中，您声明并初始化游标变量，然后将其传递给PL / SQL块，这将打开游标变量：

EXEC SQL BEGIN DECLARE SECTION; ...- 声明游标变量emp_cur SQL_CURSOR; EXEC SQL END DECLARE SECTION; - 初始化游标变量EXEC SQL ALLOCATE：emp_cur; EXEC SQL EXECUTE  - 将游标变量传递给PL / SQL块BEGIN  - 打开游标变量OPEN：emp_cur FOR SELECT * FROM EMP;结束; END-EXEC;

通常，通过调用存储过程将游标变量传递给PL / SQL，该存储过程将游标变量声明为其形式参数之一。例如，以下打包过程打开名为emp_cv的游标变量：

您可以从任何应用程序调用该过程，如下所示：

EXEC SQL EXECUTE BEGIN emp_data.open_emp_cv（：emp_cur）;结束; END-EXEC;

4.5.4从游标变量中获取

打开多行查询的游标变量后，使用FETCH语句一次从活动集中检索行。语法如下：

EXEC SQL FETCH cursor_variable_name INTO {record_name | variable_name [，variable_name，...]};

游标变量返回的每个列值都分配给INTO子句中的相应字段或变量， INTO是它们的数据类型是兼容的。

必须在客户端执行FETCH语句。在下面的例如，您将行提取到名为emp_rec的主机记录中：

- 完成提取后退出循环执行SQL WHENEVER NOT FOUND DO no_more; LOOP  - 将行提取到记录EXEC SQL FETCH：emp_cur INTO：emp_rec; - 处理数据ENDLOOP;

4.5.5关闭游标变量

10.1什么是套餐？

包是一个模式对象，它对逻辑上相关的PL / SQL类型，变量，常量，子程序，游标和异常进行分组。包被编译并存储在数据库中，其中许多应用程序可以共享其内容。

包始终具有规范 ，该规范声明可以从包外引用的公共项 。

如果公共项目包括游标或子程序，则包裹也必须具有正文 。正文必须定义公共游标的查询和公共子程序的代码。正文还可以声明和定义无法从包外引用的私有项 ，但对于包的内部工作是必需的。最后，正文可以有一个初始化部分 ，其语句初始化变量并执行其他一次性设置步骤，以及异常处理部分。您可以在不更改规范或公共项目引用的情况下更改正文;因此，您可以将包装体视为黑盒子。

在包规范或包体中，可以使用调用规范将包子程序映射到外部Java或C子程序，该规范将外部子程序名称，参数类型和返回类型映射到它们的SQL副本。

包规范的AUTHID 子句确定包中的子程序和游标是否以其定义者（默认）或调用者的权限运行，以及它们是否在定义者或调用者的模式中解析了对模式对象的非限定引用。

程序包规范的ACCESSIBLE BY 子句允许您指定可以访问程序包的PL / SQL单元的白名单。您可以在以下情况中使用此子句：

  • 您将PL / SQL应用程序实现为多个包 - 一个包提供应用程序编程接口（API）和帮助程序包来完成工作。您希望客户端可以访问API，但不能访问帮助程序包。因此，您省略了API包规范中的ACCESSIBLE BY子句，并将其包含在每个帮助程序包规范中，您指定只有API包才能访问帮助程序包。

  • 您可以创建一个实用程序包，以便为同一模式中的某些（但不是所有）PL / SQL单元提供服务。要将程序包的使用限制在目标单元中，请在程序包规范中的ACCESSIBLE BY子句中列出它们。

10.2使用包裹的原因

软件包支持开发和维护可靠，可重用的代码，具有以下功能：

  • 模块化

    通过包，您可以在命名的PL / SQL模块中封装逻辑上相关的类型，变量，常量，子程序，游标和异常。您可以使每个包易于理解，并使包之间的接口简单，清晰且定义良好。这种做法有助于应用程序开发。

  • 更轻松的应用设计

    在设计应用程序时，您最初需要的只是包规范中的接口信息。您可以在没有身体的情况下编写和编译规范。接下来，您可以编译引用包的独立子程序。在准备好完成应用程序之前，无需完全定义包体。

  • 隐藏的实施细节

    使用包可以在包规范中共享您的接口信息，并在包体中隐藏实现细节。隐藏实体细节有以下优点：

    • 您可以在不影响应用程序界面的情况下更改实施细节。

    • 应用程序用户无法开发依赖于您可能想要更改的实现细节的代码。

  • 增加了功能

    包公共变量和游标可以在会话的整个生命周期中持续存在。它们可以由在环境中运行的所有子程序共享。它们允许您跨事务维护数据，而无需将其存储在数据库中。（对于包公共变量和游标在会话期间不会持续存在的情况，请参阅“ 包状态 ” 。）

  • 更好的性能

    第一次调用包子程序时，Oracle数据库会将整个包加载到内存中。在包中同时调用其他子程序不需要磁盘I / O.

    包防止级联依赖和不必要的重新编译。例如，如果更改包函数的主体，Oracle数据库不会重新编译调用该函数的其他子程序，因为这些子程序仅依赖于规范中声明的参数和返回值。

  • 更容易授予角色

    您可以在包上授予角色，而不是在包中的每个对象上授予角色。

注意：

您无法从包内引用主变量。

10.3包装规格

包规范声明公共项 。公共项的范围是包的模式。公共项目在架构中的任何位置都可见。要引用范围但不可见的公共项目，请使用包名称对其进行限定。（有关范围，可见性和限定条件的信息，请参阅“ 标识符的范围和可见性 ” 。）

每个公共项目声明都包含使用该项目所需的所有信息。例如，假设包规范以这种方式声明函数factorial ：

功能因子（n INTEGER）返回整数; - 返回n！

该声明显示factorial需要类型的一个参数， INTEGER和返回类型的值INTEGER ，这是调用者必须知道要调用factorial 。Invokers不需要知道如何实现factorial因（例如，它是迭代的还是递归的）。

注意：

要将程序包的使用限制为指定的PL / SQL单元，请在程序包规范中包含ACCESSIBLE BY子句。

10.5包实例化和初始化

当会话引用包项时，Oracle数据库会为该会话实例化包。引用包的每个会话都有自己的包的实例化。

当Oracle数据库实例化一个包时，它会初始化它。初始化包括以下适用的任何一项：

  • 将初始值分配给公共常量

  • 将初始值分配给声明指定它们的公共变量

  • 执行包体的初始化部分

10.6包装状态

包声明的变量，常量和游标的值（在其规范或正文中）构成其包状态 。

如果PL / SQL包声明至少一个变量，常量或游标，那么包是有状态的 ;否则，它是无国籍的 。

引用包项的每个会话都有自己的包的实例化。如果包是有状态的，则实例化包括其状态。

包状态在会话期间持续存在，除非在以下情况中：

  • 包是SERIALLY_REUSABLE 。

  • 包体重新编译。

    如果重新编译实例化的有状态包的主体（显式地，使用“ ALTER PACKAGE语句 ”或隐式），则下一次调用包中的子程序会导致Oracle数据库丢弃现有包状态并引发异常ORA -04068。

    在PL / SQL引发异常之后，对包的引用会导致Oracle数据库重新实例化该包，并重新初始化它。因此，先前对包状态的更改将丢失。

  • 任何会话的实例化包都将失效并重新生效。

    如果任何会话的实例化包被无效并重新生效，则会丢失所有会话的包实例化（包括包状态）。

如果程序包的状态在会话期间（或更长时间）保持不变，则Oracle数据库会将程序包视为无状态。这是一个包的项，它的所有项都是编译时常量。

编译时常量是一个常量，PL / SQL编译器可以在编译时确定该值。初始值为文字的常量始终为编译时常量。一个常量，其初始值不是文字，但优化器减少为文字，也是编译时常量。PL / SQL优化器是否可以将非文字表达式减少为文字取决于优化级别。因此，在一个优化级别编译时无状态的程序包在以不同的优化级别编译时可能是有状态的。

10.7 SERIALLY_REUSABLE包

SERIALLY_REUSABLE包使您可以设计更好地管理内存的应用程序以实现可伸缩性。

如果包不是SERIALLY_REUSABLE ，则其包状态存储在每个用户的用户全局区域（UGA）中。因此，所需的UGA存储量随着用户数量的增加而线性增加，从而限制了可扩展性。包状态可以持续一段时间，锁定UGA内存直到会话结束。在某些应用程序（如Oracle Office）中，典型会话持续数天。

如果包是SERIALLY_REUSABLE ，则其包状态存储在系统全局区域（SGA）中的小池中的工作区中。程序包状态仅在服务器调用的生命周期内持续存在。服务器调用后，工作区返回池。如果后续服务器调用引用该包，则Oracle数据库将重用池中的实例。重用实例化会重新初始化它;因此，在先前的服务器调用中对包状态所做的更改是不可见的。（有关初始化的信息，请参阅“ 包实例化和初始化 ” 。）

注意：

尝试从数据库触发器或SQL语句调用的PL / SQL子程序访问SERIALLY_REUSABLE包会引发错误。

10.8包装书写指南

  • 熟悉Oracle数据库提供的软件包，避免编写复制其功能的软件包。

    有关Oracle数据库提供的软件包的更多信息，请参阅Oracle Database PL / SQL软件包和类型参考 。

  • 保持您的包一般，以便将来的应用程序可以重用它们。

  • 在包体之前设计和定义包规范。

  • 在包规范中，仅声明调用程序必须可见的项。

    这种做法可以防止其他开发人员对您的实现细节构建不安全的依赖关系，并减少重新编译的需要。

    如果更改包规范，则必须重新编译调用包的公共子程序的任何子程序。如果仅更改包体，则无需重新编译这些子程序。

  • 在包规范中声明公共游标并在包体中定义它们，如例10-7所示 。

    此练习允许您从包用户隐藏游标的查询，并在不更改游标声明的情况下更改它们。

  • 在包体的初始化部分而不是在声明中分配初始值。

    这种做法具有以下优点：

    • 用于计算初始值的代码可以更复杂并且更好地记录。

    • 如果计算初始值会引发异常，则初始化部分可以使用自己的异常处理程序来处理它。

  • 如果将数据库应用程序实现为多个PL / SQL包 - 一个提供API和帮助程序包的包来完成工作，那么使帮助程序包仅可用于API包，如例10-8所示 。

在例10-7中 ，光标c1的声明和定义分别在包emp_stuff的规范和正文中。游标声明仅指定返回值的数据类型，而不是查询，它出现在游标定义中（有关完整语法和语义，请参阅“ 显式游标声明和定义 ” ）。

10.10 STANDARD包如何定义PL / SQL环境

名为STANDARD的包定义了PL / SQL环境。包规范声明公共类型，变量，异常，子程序，它们可自动提供给PL / SQL程序。例如，包STANDARD声明函数ABS ，它返回其参数的绝对值，如下所示：

功能ABS（n号）返回号码;

程序包STANDARD的内容对应用程序直接可见。您不需要通过为包名称添加前缀来限定对其内容的引用。例如，您可以从数据库触发器，存储的子程序，Oracle工具或3GL应用程序中调用ABS ，如下所示：

abs_diff：= ABS（x  -  y）;

如果您声明自己的ABS版本，则您的本地声明会覆盖公共声明。您仍然可以通过指定其全名来调用SQL函数：

abs_diff：= STANDARD.ABS（x  -  y）;

大多数SQL函数都是重载的。例如，包STANDARD包含以下声明：

功能TO_CHAR（右DATE）RETURN VARCHAR2;功能TO_CHAR（左数）RETURN VARCHAR2; FUNCTION TO_CHAR（左DATE，右VARCHAR2）RETURN VARCHAR2;功能TO_CHAR（左边NUMBER，右边VARCHAR2）RETURN VARCHAR2;

PL / SQL通过匹配形式和实际参数的数量和数据类型来解析TO_CHAR的调用。
