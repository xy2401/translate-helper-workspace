请注意，永远不会删除第一个段，因为它可能包含有关进程初始状态的相关信息。

了解密码区分大小写和升级

默认情况下，Oracle Database 12 c第2版（12.2）及更高版本将升级为独占模式。独占模式不支持不区分大小写的基于密码的身份验证。

当服务器以独占模式运行时，仅具有10G密码版本的帐户将无法访问。

在以前的Oracle数据库发行版中，您可以配置身份验证协议，以便通过设置SEC_CASE_SENSITIVE_LOGON=FALSE来允许不区分大小写的基于密码的身份验证。从Oracle Database 12 c第 2版（12.2）开始，默认的基于密码的身份验证协议配置不包括使用不区分大小写的10G密码版本。默认情况下， SQLNET.ORA参数SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为12 ，这是一种独占模式。在独占模式下配置数据库时，基于密码的身份验证协议要求对要进行身份验证的帐户存在一个区分大小写的密码版本（ 11G或12C ）。此模式不包括使用早期版本中使用的10G密码版本。升级到Oracle Database 12 c第 2版及更高版本后，仅具有不区分大小写的10G密码版本的帐户将无法访问。发生此更改是因为默认情况下服务器以独占模式运行。在独占模式下配置Oracle数据库时，它无法使用旧的10G密码版本来验证客户端。服务器没有用于验证客户端的密码版本。

为了提高安全性，Oracle建议您启用区分大小写的基于密码的身份验证。此设置是默认设置。但是，您可以在升级到新的Oracle数据库版本期间临时禁用区分大小写的身份验证。升级后，您可以决定是否要在实施计划中启用区分大小写的基于密码的身份验证功能来管理密码版本。

在升级之前，Oracle建议您确定对基于密码的默认身份验证协议配置的此更改是否会影响您。执行以下检查：

  • 确定您是否拥有仅使用10G不区分大小写的密码身份验证版本的帐户。

  • 确定您是否拥有Oracle Database 11g第 2版（11.2.0.3）数据库或未应用关键修补程序更新CPUOct2012或更高版本更新的早期客户端，以及任何不具有不区分大小写的10G密码版本的帐户。

  • 确保您没有将已弃用的参数SEC_CASE_SENSITIVE_LOGON设置为FALSE。将此参数设置为FALSE可防止使用区分大小写的密码版本（ 11G和12C密码版本）进行身份验证。

如果在源数据库上配置了透明加密，则必须确保源数据库中的Oracle软件密钥库可用于辅助实例。将密钥库从源数据库手动复制到目标主机。

Oracle软件密钥库包含用于以下内容的TDE主密钥：

  • 在执行基于备份的复制时解密加密的备份。

  • 在执行TDE加密的数据库或表空间的活动数据库复制时解密数据库或表空间数据。

目的

NTH_VALUE返回analytic_clause定义的窗口中第n行的measure_expr值。返回的值具有measure_expr的数据类型。

  • { RESPECT | IGNORE } NULLS确定measure_expr空值是否包含在计算中或从计算中消除。默认值为RESPECT NULLS 。

手动配置群集的IP名称和地址要求

对于Oracle Flex Clusters和Oracle Member Cluster for Applications，如果未启用GNS，请配置静态群集节点名称和地址。

静态群集的IP地址要求

公共和虚拟IP名称必须符合RFC 952标准，该标准允许使用字母数字字符和连字符（“ - ”），但不允许使用下划线（“_”）。

Oracle Clusterware在安装访问期间在您标识为私有的接口上管理私有子网中的私有IP地址。

公共IP地址要求

群集必须具有每个节点的公共IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前为每个节点配置，并在安装之前解析为该节点

  • 与集群中所有其他公共IP地址，VIP地址和SCAN地址位于同一子网中

虚拟IP地址要求

群集必须具有每个节点的虚拟IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前为每个节点配置，但当前未使用

  • 与集群中所有其他公共IP地址，VIP地址和SCAN地址位于同一子网中

单客户端访问名称要求

群集必须具有群集的单客户端访问名称（SCAN），具有以下特征：

  • 在安装之前在域名服务器（DNS）上配置三个静态IP地址，以便三个IP地址与作为SCAN提供的名称相关联，并且所有三个地址由DNS以随机顺序返回给请求者

  • 在DNS中安装之前配置以解析当前未使用的地址

  • 给定与所有其他公共IP地址，VIP地址和群集中的SCAN地址在同一子网上的地址

  • 给定名称符合RFC 952标准，该标准允许使用字母数字字符和连字符（“ - ”），但不允许使用下划线（“_”）

专用IP地址要求

群集必须具有每个节点的专用IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前配置，但在具有其自己的子网的单独专用网络上配置，除了其他集群成员节点之外，该网络不可解析

SCAN是用于为客户端提供对群集的服务访问的名称。由于SCAN作为整体与集群相关联，而不是与特定节点相关联，因此SCAN可以在不需要重新配置客户端的情况下从集群添加或删除节点。它还为数据库添加了位置独立性，因此客户端配置不必依赖于运行特定数据库的节点。客户端可以像以前的版本一样继续访问群集，但Oracle建议访问群集的客户端使用SCAN。

注意：

在安装期间，SCAN和群集名称将在单独的字段中输入，因此群集名称要求不适用于SCAN名称。

Oracle强烈建议您不要在hosts文件中配置SCAN VIP地址。对SCAN VIP使用DNS解析。如果使用hosts文件解析SCAN，则SCAN只能解析为一个IP地址。

在DNS或hosts文件中配置SCAN是唯一受支持的配置。不支持在网络信息服务（NIS）中配置SCAN。

关于直接NFS客户端安装到NFS存储设备

Direct NFS Client直接在Oracle软件中集成NFS客户端功能，以优化Oracle和NFS服务器之间的I / O路径。这种集成可以显着提高性能。

Direct NFS Client支持NFSv3，NFSv4，NFSv4.1和pNFS协议来访问NFS服务器。Direct NFS Client还简化了数据库工作负载的NFS客户端配置的性能优化，并在许多情况下实现了自动化。

从Oracle Database 12c第2版开始，启用Direct NFS时，还可以启用Direct NFS调度程序。Direct NFS调度程序将从数据库实例创建的TCP连接数合并到NFS服务器。在大型数据库部署中，使用Direct NFS调度程序可提高可伸缩性和网络性能。并行NFS部署还需要大量连接。因此，建议使用Direct NFS调度程序进行并行NFS部署。

Direct NFS Client可以从操作系统装入条目或oranfstab文件中获取NFS装入点。

直接NFS客户端要求

  • NFS服务器的写入大小值（ wtmax ）必须为32768或更高才能与Direct NFS Client一起使用。

  • 即使您将Direct NFS Client配置为提供文件服务，也必须由操作系统内核NFS客户端和Direct NFS Client挂载NFS挂载点。

    如果Oracle数据库无法使用Direct NFS Client连接到NFS服务器，则Oracle Database将使用操作系统内核NFS客户端连接到NFS服务器。当Oracle数据库无法通过Direct NFS Client连接到NAS存储时，它会在Oracle警报和跟踪文件中记录有关Direct NFS客户端连接错误的信息性消息。

  • 遵循标准指南，以维护操作系统NFS和Direct NFS客户端安装的Oracle数据库文件的完整性。

直接NFS挂载点搜索顺序

Direct NFS Client按以下顺序搜索装入条目：

  1. $ ORACLE_HOME / DBS / oranfstab

  2. 在/ etc / oranfstab

  3. 在/ etc / mtab中

Direct NFS Client使用第一个匹配条目作为安装点。

手动配置群集的IP名称和地址要求

对于Oracle Flex Clusters和Oracle Member Cluster for Applications，如果未启用GNS，请配置静态群集节点名称和地址。

静态群集的IP地址要求

公共和虚拟IP名称必须符合RFC 952标准，该标准允许使用字母数字字符和连字符（“ - ”），但不允许使用下划线（“_”）。

Oracle Clusterware在安装访问期间在您标识为私有的接口上管理私有子网中的私有IP地址。

公共IP地址要求

群集必须具有每个节点的公共IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前为每个节点配置，并在安装之前解析为该节点

  • 与集群中所有其他公共IP地址，VIP地址和SCAN地址位于同一子网中

虚拟IP地址要求

群集必须具有每个节点的虚拟IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前为每个节点配置，但当前未使用

  • 与集群中所有其他公共IP地址，VIP地址和SCAN地址位于同一子网中

单客户端访问名称要求

群集必须具有群集的单客户端访问名称（SCAN），具有以下特征：

  • 在安装之前在域名服务器（DNS）上配置三个静态IP地址，以便三个IP地址与作为SCAN提供的名称相关联，并且所有三个地址由DNS以随机顺序返回给请求者

  • 在DNS中安装之前配置以解析当前未使用的地址

  • 给定与所有其他公共IP地址，VIP地址和群集中的SCAN地址在同一子网上的地址

  • 给定名称符合RFC 952标准，该标准允许使用字母数字字符和连字符（“ - ”），但不允许使用下划线（“_”）

专用IP地址要求

群集必须具有每个节点的专用IP地址，具有以下特征：

  • 静态IP地址

  • 在安装之前配置，但在具有其自己的子网的单独专用网络上配置，除了其他集群成员节点之外，该网络不可解析

SCAN是用于为客户端提供对群集的服务访问的名称。由于SCAN作为整体与集群相关联，而不是与特定节点相关联，因此SCAN可以在不需要重新配置客户端的情况下从集群添加或删除节点。它还为数据库添加了位置独立性，因此客户端配置不必依赖于运行特定数据库的节点。客户端可以像以前的版本一样继续访问群集，但Oracle建议访问群集的客户端使用SCAN。

注意：

在安装期间，SCAN和群集名称将在单独的字段中输入，因此群集名称要求不适用于SCAN名称。

Oracle强烈建议您不要在hosts文件中配置SCAN VIP地址。对SCAN VIP使用DNS解析。如果使用hosts文件解析SCAN，则SCAN只能解析为一个IP地址。

在DNS或hosts文件中配置SCAN是唯一受支持的配置。不支持在网络信息服务（NIS）中配置SCAN。

使Oracle密钥库可用于目标主机

如果在源数据库上配置了透明加密，则必须确保源数据库中的Oracle软件密钥库可用于辅助实例。将密钥库从源数据库手动复制到目标主机。

Oracle软件密钥库包含用于以下内容的TDE主密钥：

  • 在执行基于备份的复制时解密加密的备份。

  • 在执行TDE加密的数据库或表空间的活动数据库复制时解密数据库或表空间数据。

以下是重复数据库中密钥库的要求：

  • 密钥库必须位于默认位置，或者位于sqlnet.ora文件指示的位置。

  • 必须设置Oracle密钥库文件的权限，以便数据库可以访问该文件。

  • 在复制期间，重新启动辅助实例，从而导致Oracle软件密钥库变得不可用。要确保辅助实例可以访问密钥库，请在sqlnet.ora文件中设置ENCRYPTION_WALLET_LOCATION参数，使其指向密钥库位置。

    Oracle Database 19c版中不推荐使用ENCRYPTION_WALLET_LOCATION sqlnet.ora参数。使用WALLET_ROOT初始化参数与TDE_CONFIGURATION初始化参数配置软件密钥存储的位置。

  • 使用Oracle Real Application Clusters（Oracle RAC），使用Oracle Grid Infrastructure侦听器静态注册辅助实例，并使用Oracle Grid home的sqlnet.ora文件中的ENVS参数指定设置密钥库位置和唯一名称的环境变量的数据库。

    以下示例在sqlnet.ora设置ENVS参数以指定密钥库位置和唯一数据库名称：

    （ENVS =“ORACLE_UNQNAME = cdbrptl，ENCRYPTION_WALLET_LOCATION =（SOURCE =（METHOD = FILE）（METHOD_DATA =（DIRECTORY = / etc / ORACLE / WALLETS / cdbrpt1））））”

  • 如果源数据库使用基于密码的软件密钥库（而不是自动登录软件密钥库），则必须在开始复制之前提供密钥库密码。

    使用带有DECRYPTION WALLET OPEN IDENTIFIED BY子句的SET命令指定必须用于打开密钥库的密码。

    以下命令指定用于打开密钥库的password （其中password是您输入的实际密码的占位符）：
    SET DECRYPTION WALLET OPEN由password识别;

关于直接NFS客户端安装到NFS存储设备

Direct NFS Client直接在Oracle软件中集成NFS客户端功能，以优化Oracle和NFS服务器之间的I / O路径。这种集成可以显着提高性能。

Direct NFS Client支持NFSv3，NFSv4，NFSv4.1和pNFS协议来访问NFS服务器。Direct NFS Client还简化了数据库工作负载的NFS客户端配置的性能优化，并在许多情况下实现了自动化。

从Oracle Database 12c第2版开始，启用Direct NFS时，还可以启用Direct NFS调度程序。Direct NFS调度程序将从数据库实例创建的TCP连接数合并到NFS服务器。在大型数据库部署中，使用Direct NFS调度程序可提高可伸缩性和网络性能。并行NFS部署还需要大量连接。因此，建议使用Direct NFS调度程序进行并行NFS部署。

Direct NFS Client可以从操作系统装入条目或oranfstab文件中获取NFS装入点。

直接NFS客户端要求

  • NFS服务器的写入大小值（ wtmax ）必须为32768或更高才能与Direct NFS Client一起使用。

  • 即使您将Direct NFS Client配置为提供文件服务，也必须由操作系统内核NFS客户端和Direct NFS Client挂载NFS挂载点。

    如果Oracle数据库无法使用Direct NFS Client连接到NFS服务器，则Oracle Database将使用操作系统内核NFS客户端连接到NFS服务器。当Oracle数据库无法通过Direct NFS Client连接到NAS存储时，它会在Oracle警报和跟踪文件中记录有关Direct NFS客户端连接错误的信息性消息。

  • 遵循标准指南，以维护操作系统NFS和Direct NFS客户端安装的Oracle数据库文件的完整性。

直接NFS挂载点搜索顺序

Direct NFS Client按以下顺序搜索装入条目：

  1. $ ORACLE_HOME / DBS / oranfstab

  2. 在/ var /选择/ ORACLE / oranfstab

  3. 在/ etc / mnttab中

Direct NFS Client使用第一个匹配条目作为安装点。

注意：

每个实例只能有一个活动的Direct NFS客户端实现。在实例上使用Direct NFS Client会阻止另一个Direct NFS Client实施。

5.1 JDBC瘦客户端概述

JDBC瘦客户端是纯Java，Type IV驱动程序。它重量轻，易于安装。它提供了高性能，与JDBC Oracle调用接口（OCI）驱动程序提供的性能相当。JDBC Thin驱动程序完全用Java编写，因此它与平台无关。此外，此驱动程序不需要客户端上的任何其他Oracle软件。

JDBC Thin驱动程序使用TTC与服务器进行通信，TTC是Oracle开发的用于访问Oracle数据库数据的协议。它可以用于应用程序服务器。该驱动程序允许通过提供在Java套接字之上实现Oracle Net和TTC的TCP / IP实现来直接连接到数据库。这两个协议都是服务器上对应协议的轻量级实现版本。Oracle Net协议仅在TCP / IP上运行。

JDBC Thin驱动程序可以在客户端和服务器端使用。在客户端，驱动程序可用于在客户端或三层配置的中间层运行的Java应用程序。在服务器端，此驱动程序用于访问远程Oracle数据库实例或同一数据库上的另一个会话。

5.2支持的其他功能

JDBC Thin驱动程序支持所有标准JDBC功能。JDBC Thin驱动程序还支持以下附加功能：

  • 默认支持Native XA

  • 支持Transaction Guard

  • 支持应用程序连续性

5.2.1对本机XA的默认支持

与JDBC OCI驱动程序类似，JDBC Thin驱动程序还提供对Native XA的支持。但是，JDBC Thin驱动程序默认情况下为Native XA提供支持。这与JDBC OCI驱动程序的情况不同，其中默认情况下不启用对Native XA的支持。

9.1 Oracle RAC功能概述

UCP JDBC连接池提供与各种Oracle Real Application Clusters（Oracle RAC）数据库功能的紧密集成。这些功能包括快速连接故障转移（FCF），运行时连接负载平衡和连接关联。这些功能需要使用Oracle客户端软件附带的Oracle JDBC驱动程序，Oracle RAC数据库和Oracle Notification Service库（ ons.jar ）。

应用程序使用Oracle RAC功能来最大化连接性能和可用性，并减少因连接问题导致的停机时间。应用程序具有不同的可用性和性能要求，因此应相应地实现Oracle RAC功能。

1.1初始化参数的使用

初始化参数用于执行各种功能。

例如，初始化参数用于：

  • 设置整个数据库的限制

  • 设置用户或进程限制

  • 设置数据库资源限制

  • 影响性能（这些称为变量参数 ）

    变量参数对数据库管理员特别重要，因为这些参数主要用于提高数据库性能。

数据库管理员可以使用初始化参数：

  • 通过调整内存结构（例如内存中的数据库缓冲区数）来优化性能

  • 设置数据库范围的默认值，例如在创建上下文区域时最初为上下文区域分配的空间量

  • 设置数据库限制，例如最大数据库用户数

  • 指定数据库所需的文件或目录的名称

可以对许多初始化参数进行微调以提高数据库性能。其他参数不应更改或仅在Oracle支持服务的监督下更改。

所有初始化参数都是可选的Oracle为每个参数都有一个默认值。该值可能取决于操作系统，取决于参数。

1.1.1初始化参数的类型

Oracle数据库服务器具有派生参数，操作系统相关参数和变量参数。

  • 派生参数

  • 操作系统相关参数

  • 变量参数 （这些参数可以是动态参数或任何前面的参数）

1.1.1.1派生参数

导出一些初始化参数，这意味着它们的值是根据其他参数的值计算的。通常，您不应更改派生参数的值，但如果这样做，则指定的值将覆盖计算的值。

例如， SESSIONS参数的默认值是从PROCESSES参数的值派生的。如果PROCESSES的值发生更改，则SESSIONS的默认值也会更改，除非您使用指定值覆盖它。

1.1.1.2与操作系统相关的参数

某些初始化参数的有效值或值范围取决于主机操作系统。例如，参数DB_BLOCK_BUFFERS表示主内存中的数据缓冲区数，其最大值取决于操作系统。由DB_BLOCK_SIZE设置的那些缓冲区的大小具有依赖于操作系统的默认值。

特定于操作系统的Oracle文档，以获取有关依赖于操作系统的Oracle参数和操作系统参数的更多信息

1.1.1.3变量参数

变量初始化参数提供了提高系统性能的最大潜力。某些可变参数设置容量限制但不影响性能。例如，当OPEN_CURSORS值为10时，尝试打开其第十一个游标的用户进程会收到错误。其他可变参数会影响性能，但不会施加绝对限制。例如，减少DB_BLOCK_BUFFERS的值并不会阻止工作，即使它可能会降低性能。

增加可变参数的值可以提高系统的性能，但增加大多数参数也会增加系统全局区域（SGA）的大小。较大的SGA可以提高数据库性能。在虚拟内存操作系统中，如果SGA交换进出内存，则太大的SGA会降低性能。控制虚拟内存工作区的操作系统参数应考虑SGA大小。操作系统配置还可以限制SGA的最大大小。

34.1关于XStream接口

自Oracle Database 11g第2版以来，可以使用称为XStream Out和XStream In的API。

该技术可在Oracle数据库和非Oracle数据库，非RDBMS Oracle产品，文件系统，第三方软件应用程序等之间实现高性能，近实时的信息共享基础架构。XStream构建于Oracle Streams使用的基础架构之上。

也可以看看：

OCI XStream功能在OCI XStream函数中

34.1.1 XStream Out

XStream Out允许远程客户端连接到出站服务器并以逻辑更改记录（LCR）的形式提取行更改。

要使用XStream Out，必须创建捕获进程和出站服务器。XStream支持Oracle Streams支持的所有数据类型，包括LOB， LONG和XMLType 。捕获进程和出站服务器不必位于同一数据库实例上。捕获过程和出站服务器启动后，将捕获行更改并将其发送到出站服务器。外部客户端应用程序可以使用OCI连接到此出站服务器。建立连接后，客户端应用程序可以在等待来自出站服务器的LCR时循环。客户端应用程序可以注册每次收到LCR时要调用的客户端回调。客户端应用程序可以根据需要随时从出站服务器分离。重启后，出站服务器知道重做流中的哪个位置开始将LCR流式传输到客户端应用程序。

也可以看看：

  • Oracle Database XStream指南 ，介绍XStream Out

34.1.2 XStream In

要将非Oracle数据复制到Oracle数据库，请使用XStream In。

此技术允许远程客户端应用程序连接到入站服务器，并以LCR的形式发送行和DDL更改。

外部客户端应用程序使用OCI连接到入站服务器。建立连接后，客户端应用程序通过将LCR流式传输到其中，充当入站服务器的捕获代理。客户端应用程序只能连接到每个数据库连接的一个入站服务器，每个入站服务器只允许一个客户端应用程序连接到它。

34.1.4 XStream和字符集

对于XStream Out，通常，将客户端应用程序字符集设置为出站服务器数据库字符集是最佳做法。

XStream Out隐式地将LCR中的字符数据从出站服务器数据库字符集转换为客户端应用程序字符集。XStream In将LCR中的字符数据从客户端应用程序字符集隐式转换为入站服务器数据库字符集。

要提高性能，请完成以下任务：

  • 分析从源到目标的LCR数据流。

  • 将OCI客户端应用程序的客户端字符集设置为最小化字符转换，不会导致数据丢失，并利用XStream或目标完成的隐式转换。
