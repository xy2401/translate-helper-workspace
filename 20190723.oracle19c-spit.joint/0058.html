7.1关于协会

关联是一种数据挖掘功能，可以发现集合中项目共现的概率。共现项目之间的关系表示为关联规则 。

7.1.1关联规则

关联模型的结果是识别数据内关联模式的规则。Oracle Data Mining不支持关联建模的评分操作。

关联规则可以应用如下：

  • 支持 ：这些项目在数据中一起出现的频率是多少？
  • 信心 ：在包含前提的交易中，结果发生的频率。
  • 价值 ：与商品关联相关的商业价值

7.1.2市场篮子分析

关联规则通常用于分析销售交易。例如，注意到在杂货店购买谷物的顾客经常同时购买牛奶。事实上，关联分析发现，85％的包括谷物的结账会议也包括牛奶。这种关系可以表述为以下规则：

谷物意味着牛奶的置信度为85％

这种关联建模的应用称为市场篮子分析 。它对于直接营销，促销和发现业务趋势很有价值。市场购物篮分析也可以有效地用于商店布局，目录设计和交叉销售。

7.1.3关联规则和电子商务

关联建模在其他领域也有重要的应用。例如，在电子商务应用程序中，关联规则可用于网页个性化。关联模型可能会发现访问页面A和B的用户有70％可能同时访问同一会话中的页面C.根据此规则，可以为可能对页面C感兴趣的用户创建动态链接。关联规则表示如下：

并行执行中的缓冲区缓存使用

默认情况下，并行执行不使用SGA（缓冲区缓存）来缓存已扫描的块，除非该对象非常小或被声明为CACHE。

通过将参数PARALLEL_DEGREE_POLICY设置为AUTO来启用内存中并行执行，使并行语句能够利用SGA来缓存对象块。Oracle数据库决定使用并行执行访问的对象是否会受益于在SGA中缓存。缓存对象的决定基于一组明确定义的启发式方法，包括对象的大小和访问它的频率。在Oracle Real Applications Cluster（Oracle RAC）环境中，Oracle数据库将对象的各个部分映射到活动实例上的每个缓冲区高速缓存中。通过创建此映射，Oracle数据库自动知道要访问哪个缓冲区高速缓存以查找对象的不同部分或多个部分。使用此信息，Oracle数据库可防止多个实例一遍又一遍地从磁盘读取相同的信息，从而最大化可以缓存对象的内存量。它通过在缓存块的实例上使用PX服务器来实现此目的。

如果对象的大小大于特定阈值，则基于缓冲区高速缓存的总大小（单个实例）或缓冲区高速缓存的大小乘以Oracle RAC集群中的活动实例数，则对象为使用直接路径读取读取而不在SGA中缓存。

自动大表缓存

自动大表缓存将查询与缓冲区缓存集成，以在单实例和Oracle RAC环境中增强Oracle数据库的内存查询功能。

在Oracle Real Application Clusters（Oracle RAC）环境中，只有并行查询才支持此功能。在单实例环境中，并行和串行查询都支持此功能。

为大表缓存保留的缓存部分用于缓存表扫描的数据。虽然大表缓存主要用于增强数据仓库工作负载的性能，但它还可以提高运行混合工作负载的Oracle数据库的性能。

自动大表缓存使用基于温度和对象的算法来跟踪中型和大型表。Oracle确实缓存了非常小的表，但自动大表缓存不会跟踪这些表。

要为串行查询启用自动大表缓存，必须为DB_BIG_TABLE_CACHE_PERCENT_TARGET初始化参数设置值（百分比）。此外，必须将PARALLEL_DEGREE_POLICY初始化参数设置为AUTO或ADAPTIVE以使并行查询能够使用自动大表缓存。在Oracle RAC环境中，仅在并行查询中支持自动大表缓存，因此两个设置都是必需的。

如果一个大表大约是所有实例的大表缓存的组合大小的大小，那么该表将在所有实例上进行分区和缓存，或者主要是缓存。内存中查询可以消除表上查询的大多数磁盘读取，或者数据库只能从磁盘中智能地读取不适合大表缓存的那部分表。如果大表缓存无法缓存所有要扫描的表，则只缓存最常访问的表，其余表通过直接读取自动读取。

DB_BIG_TABLE_CACHE_PERCENT_TARGET参数确定用于扫描的缓冲区高速缓存大小的百分比。如果DB_BIG_TABLE_CACHE_PERCENT_TARGET设置为80 （％），则缓冲区高速缓存的80 （％）用于扫描，剩余的20 （％）用于OLTP工作负载。

仅当PARALLEL_DEGREE_POLICY设置为AUTO或ADAPTIVE才会在Oracle RAC环境中启用DB_BIG_TABLE_CACHE_PERCENT_TARGET参数。 DB_BIG_TABLE_CACHE_PERCENT_TARGET的缺省值为0 （禁用），上限为90 （％），保留至少10％的缓冲区高速缓存，以便在表扫描之外使用。当值为0 ，内存中查询将使用现有的最近最少使用（LRU）机制运行。您可以动态调整DB_BIG_TABLE_CACHE_PERCENT_TARGET参数。

设置DB_BIG_TABLE_CACHE_PERCENT_TARGET参数时，请使用以下准则：

  • 如果未在Oracle RAC环境中启用自动并行度（DOP），则不应设置此参数，因为在该情况下不使用大表缓存部分。

  • 设置此参数时，您应该考虑工作负载组合：OLTP的工作负载量是多少;插入，更新和随机访问;以及有多少工作负载涉及表扫描。由于数据仓库工作负载通常执行大型表扫描，因此您可以考虑为大型表缓存部分提供更高百分比的缓冲区缓存空间用于数据仓库。

  • 如果工作负载发生变化，则可以动态更改此参数。根据当前达到目标的工作负载，更改可能需要一些时间，因为此时可能会主动使用缓冲区高速缓存。

当PARALLEL_DEGREE_POLICY设置为AUTO或ADAPTIVE ，将添加数据仓库加载和扫描缓冲区的其他对象级统计信息，以表示特定（帮助程序）实例上对象上的并行查询（PQ）扫描数。

安全概述

当您连接多个不同的系统时，通常具有最严格安全要求的系统会指示和规则系统。

网关安全涉及两组：

  • 允许访问给定网关实例和DRDA数据库服务器的用户和应用程序

  • 用户和应用程序能够查询和更新的服务器数据库对象

您可以在多个点控制网关体系结构中的访问。每个具有GRANT的DRDA数据库服务器和基于用户ID的相关本机授权机制提供对数据库对象访问的控制。

当网关涉及SQL请求时，安全机制对网关遇到的每个DRDA系统组件都有效。遇到的第一个系统组件是应用程序工具或3GL程序。遇到的最后一个系统组件是DRDA数据库。

验证应用程序登录

在使用之前，应用程序必须连接到Oracle数据库网关。您使用的登录身份验证类型决定了生成的Oracle用户ID，并且可能会影响网关操作。有两种基本类型的身份验证：

  • Oracle身份验证：使用Oracle身份验证时，每个Oracle用户ID都具有Oracle数据库已知的密码。当应用程序连接到服务器时，它会提供用户标识和密码。Oracle数据库确认存在用户标识，并且密码与数据库中保存的密码相匹配。

  • 操作系统身份验证：使用操作系统身份验证，服务器的底层操作系统负责身份验证。使用操作系统身份验证访问使用IDENTIFIED EXTERNALLY属性创建的Oracle用户标识，而不是密码。要登录此类用户ID，应用程序会为用户ID提供正斜杠（/），并且不提供密码。

    要执行操作系统身份验证，服务器会确定请求者的操作系统用户ID，可选地为其添加固定前缀，并将结果用作Oracle用户ID。服务器确认用户ID存在且IDENTIFIED EXTERNALLY已IDENTIFIED EXTERNALLY ，但未进行密码检查。基本假设是用户在登录操作系统时进行了身份验证。

    操作系统身份验证并非在所有平台上都可用，并且在某些平台上不可用 Oracle Net （客户端 - 服务器）和多线程服务器配置。有关UNIX系统和Oracle Net文档，请参阅Oracle数据库安装指南11g以确定此功能的可用性。

有关验证应用程序登录的详细信息，请参阅Oracle数据库参考 。

定义和控制数据库链接

此处的信息特定于网关。有关数据库链接的其他信息，请参阅Oracle数据库参考 。

链接可访问性

给定用户应该可以访问数据库链接。任何用户标识都可以使用公共数据库链接。只有创建它的用户才能使用私有数据库链接。服务器不区分使用类型（例如只读与更新或写入）或远程对象的可访问性。访问的DRDA数据库负责这些区别。

链接和CONNECT子句

CONNECT子句是数据库链接的另一个与安全相关的属性。您可以使用CONNECT子句指定显式用户标识和密码，该标识可以与用户的Oracle数据库用户标识和密码不同。首次打开数据库链接连接时，此CONNECT用户标识和密码组合将发送到网关。根据网关选项，网关可能会将该用户标识和密码发送到DRDA服务器以进行验证。

如果创建的数据库链接没有CONNECT子句，则在打开连接时CONNECT用户的Oracle数据库用户ID和密码发送到网关。如果用户使用操作系统身份验证登录Oracle数据库，则网关不会从Oracle数据库接收任何用户ID或密码。在这种情况下，如果同一主机上的所有用户都可以使用相同的DRDA数据库用户ID，则DRDA服务器上的用户ID映射工具可用于实现此类连接。

网关初始化文件中的密码

网关使用用户标识和密码来访问DRDA服务器上远程数据库中的信息。必须在网关初始化文件中定义一些用户标识和密码，以处理资源恢复等功能。在当前具有安全意识的环境中，具有可在初始化文件中访问的纯文本密码被认为是不安全的。作为异构服务的通用连接的一部分，添加了加密功能，以帮助提高安全性。此网关可以访问此功能。包含敏感值的初始化参数可能以加密形式存储。有关如何使用该功能的更多信息，请参见“ Oracle数据库异构连接用户指南 ”中的第4.2.3节“加密初始化参数” 。

您可以使用RUNFIX列条目禁用自动修复，除非禁用修订违反安全性或Oracle策略。

6.2存储过程的优点

存储过程提供了几个优点。本节介绍了以下优点：

  • 性能

  • 生产力和易用性

  • 可扩展性

  • 可维护性

  • 互通性

  • 复制

  • 安全

6.2.1性能

存储过程编译一次并以可执行的形式存储。因此，过程调用快速而有效。可执行代码自动缓存并在用户之间共享。这降低了内存需求和调用开销。

通过对SQL语句进行分组，存储过程允许使用单个调用处理语句。这样可以减少网络流量并改善往返响应时间。

此外，存储过程使您可以利用服务器的计算资源。例如，您可以将计算绑定过程从客户端移动到服务器，从而更快地运行它们。存储函数通过在服务器中运行应用程序逻辑来增强性能。

6.2.2生产力和易用性

通过围绕一组通用存储过程设计应用程序，可以避免冗余编码并提高生产率。此外，存储过程允许您扩展数据库的功能。

您可以使用所选的Java集成开发环境（IDE）来创建存储过程。它们可以通过标准Java接口（如Java数据库连接（JDBC））以及编程接口和开发工具（如SQLJ，Oracle调用接口（OCI），Pro * C / C ++和JDeveloper）调用。

通过对存储过程的广泛访问，您可以跨应用程序共享业务逻辑。例如，可以从各种客户端应用程序调用实现业务规则的存储过程，所有这些应用程序都可以共享该业务规则。此外，您可以利用服务器的Java工具，同时继续为首选编程接口编写应用程序。

6.2.3可扩展性

数据库中的Java继承了Oracle数据库的可伸缩会话模型。存储过程通过隔离服务器上的应用程序处理来提高可伸缩性。此外，存储过程的自动依赖关系跟踪有助于开发可伸缩的应用程序。

6.2.4可维护性

验证存储过程后，您可以放心地在任意数量的应用程序中使用它。如果其定义发生更改，则只会影响该过程，而不会影响调用它的应用程序。这简化了维护和增强。此外，在服务器上维护过程比在不同客户端计算机上维护副本更容易。

6.2.5互操作性

Oracle数据库中的Java完全符合Java语言规范（JLS），并提供了通用的面向对象编程语言的所有优点。此外，与PL / SQL一样，Java提供对Oracle数据的完全访问权限。因此，用PL / SQL编写的任何过程也可以用Java编写。

PL / SQL存储过程补充了Java存储过程。通常，希望程序扩展的SQL程序员喜欢PL / SQL，而希望轻松访问Oracle数据的Java程序员也喜欢Java。

Oracle数据库允许Java和PL / SQL之间的高度互操作性。 Java应用程序可以使用嵌入式JDBC驱动程序调用PL / SQL存储过程。相反，PL / SQL应用程序可以直接调用Java存储过程。

6.2.6复制

使用Oracle Advanced Replication，您可以将存储过程从一个Oracle数据库实例复制到另一个Oracle数据库实例。这使您可以使用存储过程来实现一组中央业务规则。编写过程后，您可以将它们复制并分发到整个公司的工作组和分支机构。通过这种方式，您可以在中央服务器而不是单个服务器上修改策略。

6.2.7安全性

安全是一个大型舞台，包括：

  • 连接的网络安全性

  • 访问和执行控制操作系统资源或JVM和用户定义的类

  • 从外部源导入的JAR文件的字节码验证。

在Oracle数据库中，所有类都加载到安全数据库中，因此不受信任。用户需要适当的权限才能访问类和操作系统资源。同样，所有存储过程都受到其他用户的保护。您可以将EXECUTE数据库权限授予需要访问存储过程的用户。

您可以通过允许用户仅通过以其定义者权限运行的存储过程来操作数据来限制对Oracle数据的访问。例如，您可以允许访问更新数据库表的过程，但拒绝访问表本身。

第四部分查询JSON数据

您可以使用简单的点表示法查询JSON数据，或者，对于更多功能，使用SQL / JSON函数和条件。您可以创建和查询数据指南 ，该指南汇总了一组JSON文档的结构和类型信息。

由于JSON数据使用标准数据类型（ VARCHAR2 ， BLOB和CLOB ）存储在数据库中，因此SQL查询使用与任何其他数据库数据相同的JSON数据。

要查询特定的JSON字段，或将特定的JSON字段映射到SQL列，可以使用SQL / JSON 路径语言 。最简单的形式是路径表达式由一个或多个以句点（ . ）分隔的字段名称组成。更复杂的路径表达式可以包含过滤器和数组索引。

Oracle提供了两种查询JSON内容的方法：

  • 点符号语法 ，本质上是一个表别名，后跟一个JSON列名，后跟一个或多个字段名 - 全部用句点（ . ）分隔。数组步骤可以跟随每个字段名称。此语法旨在使用简单并尽可能返回JSON值。

  • SQL / JSON函数和条件 ，完全支持路径语言，并提供比使用点符号语法更多的功能和灵活性。您可以使用它们来创建，查询和操作存储在Oracle数据库中的JSON数据。

    • 条件json_exists测试某些JSON数据中是否存在特定值。

    • 条件is json并且is not json测试一些数据是否是格式良好的JSON数据。前者特别用作检查约束。

    • 函数json_value从一些JSON数据中选择一个标量值作为SQL值。

    • 函数json_query从一些JSON数据中选择一个或多个值，作为表示JSON值的SQL字符串。它尤其用于检索JSON文档的片段，通常是JSON对象或数组。

    • 函数json_table将一些JSON数据作为虚拟表json_table ，您也可以将其视为内联视图。

由于路径语言是查询语言的一部分，因此不会对数据施加固定的模式。此设计支持无模式开发 。实际上，“模式”通过指定给定路径在查询时动态定义。这与使用SQL在存储时为数据定义模式（一组表行和列）的更常用方法形成对比。

Oracle SQL条件json_equal不接受path-expression参数。它只是比较两个JSON值，如果相等则返回true，否则返回false。对于此比较，忽略无关紧要的空白和无关紧要的对象成员顺序。例如，如果JSON对象具有相同的成员，则无论其顺序如何，它们都是相等的。但是，如果两个比较对象中的任何一个具有一个或多个重复字段，则json_equal返回的值未指定。

您可以生成和查询JSON 数据指南 ，以帮助您开发用于导航JSON内容的表达式。数据指南可以让您深入了解JSON文档的结构和类型信息。数据指南信息可以自动更新，以跟踪您添加的新文档。

为直接NFS客户端创建oranfstab文件

Direct NFS使用配置文件oranfstab来确定可用的挂载点。

使用Direct NFS Client为要访问的每个NFS服务器创建一个具有以下属性的oranfstab文件：

  • 服务器

    NFS服务器名称。

  • 本地

    数据库主机上最多有四个路径，由IP地址或名称指定，使用在数据库主机上运行的ifconfig命令显示。

  • 路径

    最多四个到NFS服务器的网络路径，由IP地址或名称指定，如NFS服务器上的ifconfig命令所示。

  • 出口

    NFS服务器的导出路径。

  • 安装

    导出卷的相应本地安装点。

  • mnt_timeout

    指定（以秒为单位）Direct NFS Client在超时之前应等待成功安装的时间。此参数是可选的。默认超时为10分钟（600）。

  • nfs_version

    指定Direct NFS Client使用的NFS协议版本。可能的值包括NFSv3，NFSv4，NFSv4.1和pNFS。默认版本为NFSv3。如果选择NFSv4.x，则必须在oranfstab为nfs_version配置值。如果要将Direct NFS与并行NFS一起使用，请将nfs_version指定为pNFS 。

  • security_default

    指定适用于服务器条目的所有导出的NFS服务器路径的默认安全模式。此参数是可选的。 sys是默认值。请参阅security参数支持的安全级别的security_default参数说明。

  • 安全

    指定安全级别，以使用带有Direct NFS Client的Kerberos身份验证协议启用安全性。可以根据导出安装对指定此可选参数。security_default和security参数支持的安全级别为：

    • sys ：UNIX级别安全性AUTH_UNIX基于用户标识符（UID）和组标识符（GID）值的身份验证。这是安全性参数的默认值。
    • krb5 ：Direct NFS使用普通Kerberos身份验证运行。服务器被认证为它声称的真实服务器。
    • krb5i ：Direct NFS运行Kerberos身份验证和NFS完整性。对服务器进行身份验证，并检查每个邮件传输的完整性。
    • krb5p ：Direct NFS运行Kerberos身份验证和NFS隐私。服务器已通过身份验证，并且所有数据都已完全加密。

    security参数（如果已指定）优先于security_default参数。如果未指定这些参数，则sys是默认身份验证。

    对于NFS服务器Kerberos安全设置，请查看相关的NFS服务器文档。对于Kerberos客户端设置，请查看相关的操作系统文档。

  • dontroute

    指定传出消息不应由操作系统路由，而是使用它们绑定的IP地址发送。

    注意：

    dontroute选项是POSIX选项，有时在同一子网中具有多个路径的Linux系统上不起作用。

  • 管理

    使Direct NFS Client能够使用管理接口进行SNMP查询。如果SNMP在NFS服务器上的单独管理接口上运行，则可以使用此参数。默认值是服务器参数值。

  • 社区

    指定用于SNMP查询的社区字符串。默认值为public 。

B.1创建数据字典

使用Database Configuration Assistant创建数据库时，Oracle会自动创建数据字典。 此后，每当数据库运行时，Oracle都会更新数据字典以响应每个DDL语句。

数据字典基表是在任何Oracle数据库中创建的第一个对象。它们是在SYSTEM表空间中创建的，必须保留在那里。数据字典基表存储有关数据库中所有用户定义对象的信息。
