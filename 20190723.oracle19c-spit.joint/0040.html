句法

ATTR [IBUTE] [ type_name.attribute_name [ option ...]]

其中option表示以下子句之一：

ALI [AS] 别名 CLE [AR] FOR [MAT] 格式 LIKE { type_name.attribute_name | 别名 } ON |关闭

指定显示“对象类型”列的给定属性的特征，例如NUMBER数据的格式。列和属性不应具有相同的名称，因为它们共享公共命名空间。

还列出了单个属性或所有属性的当前显示特性。

输入ATTRIBUTE 后跟type_name.attribute_name ，没有其他子句列出仅指定属性的当前显示特性。输入ATTRIBUTE 没有子句列出所有当前属性显示特征。

条款

type_name.attribute_name

标识对象类型type_name的给定对象的属性集内的数据项（通常是属性的名称）。

如果选择相同对象类型的对象，则该type_name.attribute_name的ATTRIBUTE命令将应用于您在该会话中引用的所有此类对象。 

ALI [AS] 别名

为type_name.attribute_name分配指定的别名，该别名可用于引用其他ATTRIBUTE命令中的type_name.attribute_name 。 

明确]

重置attribute_name的显示特征。格式规范必须是文本常量，例如A10或$ 9,999，而不是变量。 

FOR [MAT] 格式

指定列的显示格式。格式规范必须是文本常量，例如A10或$ 9,999，而不是变量。 

 LIKE { type_name.attribute_name | 别名 }

副本另一个属性的显示特征。LIKE仅复制当前ATTRIBUTE命令中未由另一个子句定义的特征。 

ON |关闭

控制列的显示特征的状态。关闭禁用属性的特征而不影响特征的定义。上恢复特征。

用法

您可以为一个或多个属性输入任意数量的ATTRIBUTE命令。为每个属性设置的所有属性特征在会话的其余部分保持有效，直到您关闭该属性，或直到您使用该属性 CLEAR COLUMN命令。因此，ATTRIBUTE 您输入的命令可以控制多个SQL SELECT命令的属性的显示特性。

当您输入多个ATTRIBUTE 对于相同属性的命令，SQL * Plus统一应用它们的子句。如果多个ATTRIBUTE命令将同一子句应用于同一属性，则输入的最后一个将控制输出。

专用IP接口配置要求

专用接口的要求取决于您使用的是单个还是多个接口。

单接口专用网络集群的网络要求

  • 每个节点的互连专用接口必须位于同一子网中。

  • 子网必须连接到群集的每个节点。

    例如，如果专用接口的子网掩码为255.255.255.0，则您的专用网络的范围为192.168.0.0--192.168.0.255，您的专用地址必须在192.168.0范围内。[0- 255]。如果专用接口的子网掩码为255.255.0.0，则您的专用地址可以在192.168范围内。[0-255][0-255]

  • 支持IPv4和IPv6地址。

冗余互连使用群集的网络要求

通过冗余互连使用，您可以识别用于群集专用网络的多个接口，而无需使用绑定或其他技术。

定义多个接口时，Oracle Clusterware会创建一到四个高可用性IP（HAIP）地址。Oracle RAC和Oracle自动存储管理（Oracle ASM）实例使用这些接口地址来确保节点之间高度可用的负载平衡接口通信。安装程序启用冗余互连使用以提供高可用性专用网络。默认情况下，Oracle Grid Infrastructure软件使用所有HAIP地址进行专用网络通信，从而为您为专用网络识别的一组接口提供负载平衡。如果专用互连接口出现故障或变为非通信接口，则Oracle Clusterware会将相应的HAIP地址透明地移动到其余功能接口之一。

  • 每个专用接口应位于不同的子网上。

  • 每个集群成员节点必须在每个专用互连子网上都有一个接口，并且这些子网必须连接到集群的每个节点。

    例如，您可以在子网192.168.0和10.0.0上拥有专用网络，但每个集群成员节点必须具有连接到192.168.0和10.0.0子网的接口。

  • 所有指定的互连接口的端点必须在网络上完全可达。应该没有未连接到每个专用网络接口的节点。

    您可以使用ping测试是否可以访问互连接口。

  • 您可以将IPv4和IPv6地址用于与Oracle Clusterware冗余互连的接口。

注意：

在安装过程中，您最多可以为专用网络定义四个接口。安装期间创建的HAIP地址数基于为网络适配器配置的物理和逻辑接口。安装后，您可以定义其他接口。如果将四个以上的接口定义为专用网络接口，请注意Oracle Clusterware一次仅激活四个接口。但是，如果四个活动接口中的一个出现故障，则Oracle Clusterware会将配置为故障接口的HAIP地址转换为定义的一组专用接口中的一个保留接口。

Oracle数据库网关概述

异构数据访问是一个影响很多公司的问题。许多公司运行几个不同的数据库系统。这些系统中的每一个都存储数据，并具有一组针对它运行的应用程序。在一个数据库系统中合并这些数据通常很难 - 很大程度上是因为许多针对一个数据库运行的应用程序可能没有与另一个数据库运行的等效项。在迁移到一个统一数据库系统变得可行之前，各种异构数据库系统必须进行互操作。

Oracle数据库网关提供了从Oracle环境透明地访问驻留在非Oracle系统中的数据的功能。这种透明性消除了应用程序开发人员定制其应用程序以访问来自不同非Oracle系统的数据的需要，从而减少了开发工作并提高了应用程序的移动性。可以使用Oracle和Sybase的一致Oracle接口开发应用程序。

网关技术由两部分组成：具有连接到非Oracle系统的通用技术的组件，所有非Oracle系统都通用，称为异构服务，以及特定于非Oracle的组件网关连接的系统。异构服务与Oracle Database Gateway代理结合使用，可以从Oracle环境透明地访问非Oracle系统。

关于异构服务技术

异构服务提供了连接到非Oracle系统的通用技术。作为数据库的集成组件，异构服务可以利用数据库的功能，例如强大的SQL解析和分布式优化功能。

如果SCAN由DNS解析，则DNS将所有三个SCAN VIP地址返回给客户端。如果SCAN由GNS解析，则DNS区域委托将查找请求发送给GNS，然后GNS将所有三个SCAN VIP地址返回给客户端。

（DESCRIPTION =（CONNECT_TIMEOUT = 90）（RETRY_COUNT = 20）（RETRY_DELAY = 3）（TRANSPORT_CONNECT_TIMEOUT = 3）（ADDRESS =（PROTOCOL = TCP）（HOST = scan ）（PORT = 1521））（CONNECT_DATA =（SERVICE_NAME = service_name ） ））

scan表示群集的SCAN。如果未指定端口号，则默认值1521用于TCP端口标识符。service_name是动态数据库服务的名称。

然后，客户端使用返回的SCAN VIP地址之一来联系SCAN监听器。当SCAN侦听器从客户端收到连接请求时，SCAN侦听器会识别群集中提供所请求服务的最少负载实例。然后，它将连接请求重定向到运行负载最少的实例的节点上的本地侦听器，并为客户端提供本地侦听器地址。然后，本地侦听器创建与数据库实例的连接。

一旦类型在服务器端发展，使用此类型的所有客户端应用程序都需要对与该类型相关联的结构进行必要的更改。您可以使用OTT / JPUB执行此操作。

您还可能需要进行与结构更改相关的编程更改。进行这些更改后，您必须重新编译应用程序并重新链接。

可以在第三方应用程序的版本之间更改类型。要通知客户端应用程序需要重新编译以与第三方应用程序的最新版本兼容，您可以让客户端调用面向发布的兼容性初始化函数。

此函数可以将一个字符串作为输入，告诉它客户端应用程序正在使用哪个版本。如果释放字符串与最新版本不匹配，则会生成错误。然后，客户端应用程序必须更改发布字符串，作为与最新版本兼容所需的更改的一部分。

例如：

功能compatibility_init（rel IN VARCHAR2，errmsg OUT VARCHAR2）返回号码;

哪里：

  • rel是产品选择的发布字符串，例如'Release 10.1'

  • errmsg是可能需要返回的任何错误消息

  • 该函数在成功时返回0 ，在错误时返回非零值

9.11.2关于更改默认构造函数

更改类型时，需要按顺序更改其默认的系统定义构造函数（例如），以在参数列表中包含新添加的属性。如果使用默认构造函数，则需要在程序中修改它们的调用以便进行编译调用。

如果定义自己的构造函数而不是使用系统定义的默认函数，则可以避免必须修改构造函数调用。请参阅“ 用户定义的构造函数的优点 ” 。

9.11.3关于改变类型的最终属性

当您将类型T1从FINAL更改为NOT FINAL ，客户端程序中任何类型T1属性都会从内联结构变为指向T1的指针。这意味着您需要更改程序以在访问此属性时使用解除引用。

相反，当您将类型从NOT FINAL更改为FINAL ，该类型的属性将从指针更改为内联结构。

例如，假设你有类型T1(a int)和T2(b T1) ，其中T1的属性是FINAL 。对应于T2的C / JAVA结构是T2(T1 b) 。但如果将T1的属性更改为NOT FINAL ，则T2的结构变为T2(T1 *b) 。

Teradata的Oracle数据库网关概述

异构数据访问是一个影响很多公司的问题。许多公司运行几个不同的数据库系统。这些系统中的每一个都存储数据，并具有一组针对它运行的应用程序。在一个数据库系统中整合这些数据通常很难 - 很大程度上是因为许多针对一个数据库运行的应用程序可能没有与另一个数据库运行的等效数据库。在迁移到一个统一数据库系统变得可行之前，各种异构数据库系统必须进行互操作。

冗余互连使用群集的网络要求

通过冗余互连使用，您可以识别用于群集专用网络的多个接口，而无需使用绑定或其他技术。

定义多个接口时，Oracle Clusterware会创建一到四个高可用性IP（HAIP）地址。Oracle RAC和Oracle自动存储管理（Oracle ASM）实例使用这些接口地址来确保节点之间高度可用的负载平衡接口通信。安装程序启用冗余互连使用以提供高可用性专用网络。默认情况下，Oracle Grid Infrastructure软件使用所有HAIP地址进行专用网络通信，从而为您为专用网络识别的一组接口提供负载平衡。如果专用互连接口出现故障或变为非通信接口，则Oracle Clusterware会将相应的HAIP地址透明地移动到其余功能接口之一。

  • 每个专用接口应位于不同的子网上。

  • 每个集群成员节点必须在每个专用互连子网上都有一个接口，并且这些子网必须连接到集群的每个节点。

    例如，您可以在子网192.168.0和10.0.0上拥有专用网络，但每个集群成员节点必须具有连接到192.168.0和10.0.0子网的接口。

  • 所有指定的互连接口的端点必须在网络上完全可达。应该没有未连接到每个专用网络接口的节点。

    您可以使用ping测试是否可以访问互连接口。

  • 您可以将IPv4和IPv6地址用于与Oracle Clusterware冗余互连的接口。

注意：

在安装过程中，您最多可以为专用网络定义四个接口。安装期间创建的HAIP地址数基于为网络适配器配置的物理和逻辑接口。安装后，您可以定义其他接口。如果将四个以上的接口定义为专用网络接口，请注意Oracle Clusterware一次仅激活四个接口。但是，如果四个活动接口中的一个出现故障，则Oracle Clusterware会将配置为故障接口的HAIP地址转换为定义的一组专用接口中的一个保留接口。

4.2关于在UCP中控制池大小

UCP JDBC连接池包含一组用于控制池大小的属性。这些属性允许池中的连接数随着需求的增加和减少而增加和减少。此动态行为有助于节省在维护不必要的连接时丢失的系统资源。

本节介绍以下主题：

  • 设置初始池大小

  • 设置最小池大小

  • 设置最大池大小

4.2.1设置初始池大小

初始池大小属性指定最初创建或重新初始化连接池时创建的可用连接数。此属性通常用于减少将池启动到其最佳大小所引起的加速时间。

值为0表示未预先创建任何连接。默认值为0 。以下示例演示如何配置初始池大小：

pds.setInitialPoolSize（5）;

如果初始池大小属性大于最大池大小属性，则仅初始化最大连接数。

如果初始池大小属性小于最小池大小属性，则只初始化和维护初始连接数，直到创建足够的连接以满足最小池大小值。

4.2.2设置最小池大小

最小池大小属性指定池维护的最小可用连接数和借用连接数。除非尚未达到最小数量，否则连接池始终会尝试返回指定的最小池大小。例如，如果最小限制设置为10并且只创建和借用了2个连接，则池维护的连接数保持为2因为此数字小于最小池大小。

此属性允许池中的连接数随着需求的减少而减少。同时，该属性确保不会浪费系统资源来维护不必要的连接。

默认值为0 。以下示例演示如何配置最小池大小：

pds.setMinPoolSize（2）;

4.2.3设置最大池大小

最大池大小属性指定池维护的最大可用和借用（使用中）连接数。如果借用了最大连接数，则在将连接返回到池之前，将不会有任何连接可用。

此属性允许池中的连接数随着需求的增加而增加。同时，该属性确保池不会增长到耗尽系统资源的程度，这最终会影响应用程序的性能和可用性。

值为0表示池没有维护连接。尝试获取连接会导致异常。默认值是允许池继续创建最多为Integer.连接Integer.MAX_VALUE （默认为2147483647）。以下示例演示如何配置最大池大小：

pds.setMaxPoolSize（100）;

为集群时间同步设置网络时间协议

Oracle Clusterware要求在所有群集节点上设置相同的时区环境变量。

在安装过程中，安装过程会在运行OUI的节点上获取网格安装所有者的时区环境变量设置，并在所有节点上使用该时区值作为Oracle Clusterware管理的所有进程的默认TZ环境变量设置。时区默认用于数据库，Oracle ASM和任何其他托管进程。群集时间同步有两个选项：

  • 操作系统配置的网络时间协议（NTP）

  • Oracle集群时间同步服务

Oracle Cluster Time Synchronization Service专为群集服务器无法访问NTP服务的组织而设计。如果使用NTP，则Oracle Cluster Time Synchronization守护程序（ctssd）将以观察者模式启动。如果您没有NTP守护程序，则ctssd将以活动模式启动，并在不接触外部时间服务器的情况下在集群成员之间同步时间。

在Oracle Solaris Cluster系统上，Oracle Solaris Cluster软件提供名为ntp.conf.cluster的模板文件（请参阅已安装的群集主机上的/etc/inet/ntp.conf.cluster ），该文件在所有群集主机之间建立对等关系。一个主机被指定为首选主机。主机由其私有主机名标识。跨群集互连发生时间同步。如果Oracle Clusterware检测到Oracle Solaris Cluster NTP或外部NTP服务器在/etc/inet/ntp.conf或/etc/inet/ntp.conf.cluster文件中的系统中设置了默认NTP服务器，则CTSS设置为观察者模式。有关为Oracle Solaris配置NTP的详细信息，请参见Oracle Solaris 11信息库 。

注意：

在开始安装Oracle Grid Infrastructure之前，Oracle建议您确保所有节点上的时钟设置为同一时间。

如果您的服务器上有NTP守护程序，但您无法将它们配置为与时间服务器同步时间，并且您希望使用群集时间同步服务在群集中提供同步服务，则停用并卸载NTP。

要禁用NTP服务，请以root用户身份运行以下命令：

＃/ usr / sbin / svcadm禁用ntp

当安装程序发现NTP协议未处于活动状态时，群集时间同步服务将以活动模式安装，并在节点之间同步时间。如果发现NTP已配置，则群集时间同步服务将以观察者模式启动，并且群集中的Oracle Clusterware不会执行活动时间同步。

要确认安装后ctssd处于活动状态，请输入以下命令作为网格安装所有者：

$ crsctl检查ctss

如果您使用的是NTP，并且您希望继续使用它而不是群集时间同步服务，则需要修改NTP配置。完成此任务后，重新启动网络时间协议守护程序。

您可以在Oracle Solaris 11及更高版本的以下示例中修改NTP配置：

以root用户身份运行以下命令：

＃/ usr / sbin / svccfg -s svc：/ network / ntp：默认setprop config / slew_always = true

然后，重新启动NTP服务：

＃/ usr / sbin / svcadm refresh svc：/ network / ntp：default

要在禁用NTP后启用NTP，请输入以下命令：

＃/ usr / sbin / svcadm enable ntp

注意：

PRE_PAGE_SGA可以增加进程启动持续时间，因为每个启动的进程都必须访问SGA中的每个页面。这个策略的成本是固定的;但是，您可能只是确定每次进程开始时必须触摸20,000页。这种方法对某些应用程序很有用，但不适用于所有应用程序。如果您的系统经常通过例如持续登录和注销来创建和销毁进程，则开销可能很大。

PRE_PAGE_SGA可以承受的优势取决于页面大小。例如，如果SGA为80 MB且页面大小为4 KB，则必须触摸20,000页以刷新SGA（80,000 / 4 = 20,000）。

如果系统允许您设置4 MB页面大小，则只需触摸20页即可刷新SGA（80,000 / 4,000 = 20）。页面大小是特定于操作系统的，通常无法更改。但是，某些操作系统具有共享内存的特殊实现，您可以通过它来更改页面大小。

Oracle建议您完成这些更新Oracle数据库的良好实践指南。建议手动和DBUA升级使用这些做法。

  • 备份数据库
    执行生产数据库的完整备份。
  • 方案非CDB运行postupgrade_fixups.sql脚本
    查看此过程以了解如何对非CDB数据库使用postupgrade_fixups.sql脚本。
  • 升级后收集字典统计信息
    为了确保良好的性能，请在完成升级后使用此过程收集字典统计信息。
  • 使用DBMS_STATS重新收集固定对象统计信息
    升级后或其他数据库配置更改后，Oracle强烈建议您在Oracle数据库上运行代表性工作负载后重新收集固定对象统计信息。
  • 重置密码以强制区分大小写
    对于升级的数据库，通过对默认用户帐户和用户帐户使用区分大小写的密码来提高安全性。
  • 添加适当的新功能
    在数据库升级计划中查看新功能。
  • 根据需要制定新的行政程序
    计划对脚本和过程的审核，并根据需要进行更改。
  • 设置表空间警报的阈值
    升级后，已升级的Oracle数据库表空间警报的阈值将设置为null，从而禁用警报。
  • 从回滚段迁移到自动撤消模式
    如果数据库版本早于Oracle Database 11g ，则必须将正在升级的数据库从使用回滚段（手动撤消管理）迁移到自动撤消管理。
  • 配置Oracle Data Guard Broker
    InitialConnectIdentifier由DGConnectIdentifier替换，后者会影响Oracle Database 10g的升级。
  • 将表从LONG数据类型迁移到LOB数据类型
    您可以使用ALTER TABLE语句将LONG列的数据类型更改为CLOB ，将LONG RAW列的数据类型更改为BLOB 。
  • 确定重建的Oracle文本索引
    您可以运行一个脚本，帮助您识别带有令牌表的Oracle Text索引索引，这些令牌表可以在升级到新的Oracle数据库发行版后重建。
  • 关于测试升级的生产Oracle数据库
    对您在测试数据库上执行的生产数据库重复测试，以确保应用程序按预期运行。

目的

ROUND（数字）

句法

round_number :: =


插图描述round_number.eps

目的

ROUND返回n四舍五入到integer位，小数点右边。如果省略integer ，则n舍入为零。如果integer为负数，则n将舍入到小数点的左侧。

n可以是任何数值数据类型或可以隐式转换为数字数据类型的任何非数字数据类型。如果省略integer ，则该函数返回与数字数据类型n相同的数据类型中的值ROUND （n，0）。如果包含integer ，则该函数返回NUMBER 。

ROUND使用以下规则实现：

  1. 如果n为0，则无论integer如何， ROUND始终返回0。

  2. 如果n为负，则ROUND （n，整数）返回 - ROUND （-n，整数）。

  3. 如果n是正数，那么

    ROUND（n，整数）= FLOOR（n * POWER（10，整数）+ 0.5）* POWER（10，-integer）

应用于NUMBER值的ROUND可能会给出与应用于浮点表示的相同值的ROUND略有不同的结果。不同的结果来自NUMBER和浮点值的内部表示的差异。如果出现差异，则在舍入的数字中差异为1。

目的

TRANSLATE返回expr其中from_string中每个字符的所有出现都被其在to_string相应字符替换。expr中不在from_string中的from_string不会被替换。from_string参数可以包含比to_string更多的字符。在这种情况下， from_string末尾的额外字符在to_string没有对应的字符。如果这些额外字符出现在expr ，则它们将从返回值中删除。

如果字符在from_string多次出现，则使用与第一次出现相对应的to_string映射。

您不能使用空字符串to_string从返回值中删除from_string中的所有字符。Oracle数据库将空字符串解释为null，如果此函数具有null参数，则返回null。要删除from_string中的所有字符， from_string另一个字符连接到from_string的开头，并将此字符指定为to_string 。例如， TRANSLATE （ expr ，' x0123456789 '，' x '）从expr删除所有数字。

TRANSLATE提供与REPLACE函数提供的功能相关的功能。REPLACE允许您将单个字符串替换为另一个字符串，以及删除字符串。TRANSLATE允许您在一次操作中进行多个单字符，一对一替换。

此函数不直接支持CLOB数据。但是， CLOB可以通过隐式数据转换作为参数传递。

  • 单击Delta（Δ）符号提高新票证。Oracle Health Check Collections Manager仅在“ 集合”和“ 浏览”选项卡中显示增量符号

  • “ 浏览”选项卡使您可以在单独检查上创建新票证

  • “ 集合”选项卡使您可以为整个集合创建单个故障单

  • Delta（Δ）符号根据故障单状态以红色，蓝色和绿色进行颜色编码

    • RED（不存在事故故障单） ：启动该过程以为集合或单独检查创建新事故故障单

    • 蓝色（存在打开的突发事件故障单） ：打开事故单以进行编辑

    • 绿色（存在已关闭的事故单） ：打开已关闭的事故单以供查看

  • 在故障单的更新区域中跟踪故障单的进度，或添加事件的附件和链接

  • 使用标记对事件进行分类，并在报告中使用生成的标记云

  • 事件访问和管理仅在您的访问控制范围内发生

注意：

事件跟踪功能是一个基本的独立系统，它不是为与其他商业级企业级故障单系统集成而设计的。
