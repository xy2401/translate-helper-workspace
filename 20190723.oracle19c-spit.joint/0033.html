使用只读表空间运行升级

使用带有-T选项的Parallel Upgrade Utility可在升级期间使基于模式的表空间脱机。

Oracle数据库可以读取在早期版本中创建的文件头，因此在升级期间不需要对它们执行任何操作。READ ONLY表空间的文件头在更改为READ WRITE时会更新。

如果升级遇到灾难性错误，则升级无法使表空间重新联机，请查看升级日志文件。日志文件包含使表空间可用所需的实际SQL语句。要使表空间重新联机，必须在数据库的日志文件中运行SQL语句，或运行每个PDB的日志文件。

在升级日志文件中查看表空间命令

如果发生灾难性升级失败，则可以导航到日志目录（ Oracle_base /cfgtoologs/dbua ），并手动运行日志文件中的命令以调出表空间。您可以在以下日志文件中查看表空间命令：

  • 非CDB升级： catupgrd0.log

  • PDB数据库： catupgrd pdbname 0.log ，其中pdbname是要升级的PDB的名称。

在每个日志文件的开头，您可以找到以下SQL语句，这些语句将表设置为READ ONLY ：

SQL> ALTER TABLESPACE ARGROTBLSPA6只读;表空间已更改。SQL> ALTER TABLESPACE ARGROTBLSPB6只读;表空间已更改。

在每个日志文件的末尾，您会找到将表重置为READ WRITE SQL语句：

SQL> ALTER TABLESPACE ARGROTBLSPA6 READ WRITE;表空间已更改。SQL> ALTER TABLESPACE ARGROTBLSPB6 READ WRITE;表空间已更改。

12.44 SPOOL

句法

SPO [OL] [ file_name [。 分机 ] [CRE [ATE] | REP [LACE] | APP [END]] |关闭| OUT]

商店询问结果在文件中，或者可选地将文件发送到打印机。

条款

file_name [。 分机 ]

代表名称您想要假脱机的文件。SPOOL后跟file_name开始将显示的输出假脱机到指定的文件。如果未指定扩展名，SPOOL将使用默认扩展名（大多数系统上的LST或LIS）。扩展名不会附加到系统文件，例如/ dev / null和/ dev / stderr。 

创建]

创建具有指定名称的新文件。 

更换]

替换现有文件的内容。如果该文件不存在，REPLACE将创建该文件。这是默认行为。 

附加]

将缓冲区的内容添加到指定文件的末尾。 

关闭

停止假脱机。

OUT

停止假脱机并将文件发送到计算机的标准（默认）打印机。某些操作系统上不提供此选项。

输入没有子句的SPOOL列出当前假脱机状态。

用法

要在脚本中将命令生成的输出假脱机而不在屏幕上显示输出，请使用SET 关闭。 SET TERMOUT OFF不会影响以交互方式运行的命令的输出。

您必须在包含空格的文件名周围使用引号。

要使用SPOOL APPEND命令创建有效的HTML文件，必须使用PROMPT或类似命令来创建HTML页眉和页脚。SPOOL APPEND命令不解析HTML标记。

将SQLPLUSCOMPAT [IBILITY]设置为9.2或更早版本以禁用CREATE，APPEND和SAVE参数。请参阅SQL * Plus Compatibility Matrix以确定SET SQLPLUSCOMPAT [IBILITY]命令控制的功能。

SPOOL命令的示例

要使用默认文件扩展名在新文件DIARY中记录输出，请输入

SPOOL DIARY CREATE

要将输出附加到现有文件DIARY，请输入

SPOOL DIARY APPEND

要将输出记录到文件DIARY，请覆盖现有内容，然后输入

关于SCAN

在安装Oracle Grid Infrastructure期间，将为SCAN创建多个Oracle Clusterware资源。

  • 为Oracle单客户端访问名称（SCAN）解析的每个IP地址创建一个SCAN虚拟IP（VIP）

  • 为每个SCAN VIP创建SCAN侦听器

  • 为SCAN侦听器配置了对SCAN VIP的依赖性

使用以下两个选项之一定义SCAN：

  • SCAN在DNS中定义

    如果手动配置SCAN并使用DNS进行名称解析，则网络管理员应为SCAN创建一个名称，该名称解析为与群集的公共网络位于同一网络上的三个IP地址。SCAN名称必须是可解析的，不带域后缀（例如，地址sales1-scan.example.com必须使用sales1-scan可解析）。不得将SCAN分配给网络接口，因为Oracle Clusterware会解析SCAN。

    默认SCAN是cluster_name -scan. domain_name 。例如，在不使用GNS的群集中，如果群集名称为sales1 ，而您的域为example.com ，则默认SCAN地址为sales1-scan.example.com:1521 。

  • SCAN在GNS中定义

    使用GNS和DHCP时，Oracle Clusterware会为群集配置期间提供的SCAN名称配置VIP地址。使用GNS时，从DHCP服务器获取节点VIP和三个SCAN VIP。如果新服务器加入群集，则Oracle Clusterware会从DHCP服务器动态获取所需的VIP地址，更新群集资源，并通过GNS访问服务器。

Oracle建议您将连接到群集的客户端配置为使用SCAN名称，而不是Oracle Grid Infrastructure 11g第2版（11.2）之前的版本中使用的节点VIP。使用SCAN连接到Oracle RAC数据库的客户端不必配置承载特定数据库或数据库实例的每个节点的地址。例如，如果为群集配置策略管理的服务器池，则使用SCAN连接到数据库将启用与该数据库中的服务器池的连接，而不管哪个节点分配给服务器池。您可以在数据库中添加或删除节点，而无需重新配置连接到数据库的客户端。

关于快速恢复区和快速恢复区磁盘组

快速恢复区域是与恢复相关的所有Oracle数据库文件的统一存储位置。为最近的数据启用快速备份可以减少向系统管理员请求检索备份磁带以进行恢复操作的请求。

数据库管理员可以将DB_RECOVERY_FILE_DEST参数定义为快速恢复区域的路径，以启用磁盘备份和快速恢复数据。在init.ora文件中启用快速恢复时，Oracle数据库会将所有RMAN备份，存档日志，控制文件自动备份和数据库副本写入快速恢复区域。RMAN通过删除不再需要的过时备份和归档文件来自动管理快速恢复区中的文件。

Oracle建议您创建快速恢复区磁盘组。Oracle Clusterware文件和Oracle数据库文件可以放在同一磁盘组中，您也可以将快速恢复文件放在同一磁盘组中。但是，Oracle建议您创建单独的快速恢复磁盘组以减少存储设备争用。

通过设置DB_RECOVERY_FILE_DEST参数来启用快速恢复区域。快速恢复区域的大小使用DB_RECOVERY_FILE_DEST_SIZE设置。作为一般规则，快速恢复区域越大，它变得越有用。为便于使用，Oracle建议您在存储设备上创建一个可包含至少三天恢复信息的快速恢复区磁盘组。理想情况下，快速恢复区域足以容纳所有数据文件和控制文件的副本，联机重做日志以及使用保留策略下保留的数据文件备份恢复数据库所需的归档重做日志文件。

多个数据库可以使用相同的快速恢复区域。例如，假设您已在具有150 GB存储空间的磁盘上创建了快速恢复区磁盘组，由3个不同的数据库共享。您可以根据每个数据库的重要性为每个数据库设置快速恢复的大小。例如，如果database1是您最不重要的数据库，则database2更重要，而database3最重要，那么您可以为每个数据库设置不同的DB_RECOVERY_FILE_DEST_SIZE设置以满足每个数据库的保留目标：30 GB用于database1,50 GB对于database2，对于database3为70 GB。

关于独立服务器的Oracle Grid Infrastructure

如果您计划使用Oracle自动存储管理（Oracle ASM），则必须在安装数据库之前安装Oracle Restart。

用于独立服务器的Oracle Grid Infrastructure是支持单实例数据库的Oracle Grid Infrastructure版本。此支持包括卷管理，文件系统和自动重启功能。用于独立服务器的Oracle Grid Infrastructure包括Oracle Restart和Oracle自动存储管理。Oracle将这两种基础架构产品组合成一组安装在Oracle Restart主目录中的二进制文件。

Oracle Restart是作为Oracle Grid Infrastructure的一部分提供的功能。Oracle Restart监视并可以重新启动Oracle数据库实例，Oracle Net Listeners和Oracle ASM实例。Oracle Restart目前仅限于管理单实例Oracle数据库和Oracle ASM实例。

Oracle自动存储管理是Oracle数据库文件的卷管理器和文件系统，支持单实例Oracle数据库和Oracle Real Application Clusters（Oracle RAC）配置。Oracle自动存储管理还支持满足您的应用程序需求的通用文件系统，包括Oracle数据库二进制文件。Oracle自动存储管理是Oracle推荐的存储管理解决方案，可替代传统的卷管理器和文件系统。

Oracle Restart通过提供以下服务来提高Oracle数据库的可用性：

  • 当出现硬件或软件故障时，Oracle Restart会自动启动所有Oracle组件，包括Oracle数据库实例，Oracle Net Listener，数据库服务和Oracle ASM。

  • 重新启动数据库主机时，Oracle Restart以正确的顺序启动组件。

  • Oracle Restart会定期检查以监视Oracle组件的状态。如果组件的检查操作失败，则组件将关闭并重新启动。

请注意以下使用Oracle Restart的限制：

  • 您既不能在Oracle Grid Infrastructure集群成员节点上安装Oracle Restart，也不能将Oracle Restart服务器添加到Oracle Grid Infrastructure集群成员节点。Oracle Restart支持在一台服务器上支持单实例数据库，而Oracle Grid Infrastructure for Cluster支持群集上的单实例或Oracle RAC数据库。

  • 如果要使用Oracle ASM或Oracle Restart，则应在安装和创建数据库之前为独立服务器安装Oracle Grid Infrastructure。否则，您必须安装Oracle Restart，然后使用Oracle Restart手动注册数据库。

  • 您只能在单实例（非集群）环境中使用Oracle Grid Infrastructure的Oracle Restart实现。将Oracle Grid Infrastructure与Oracle Clusterware一起用于群集环境。

评估网络实用程序包的依赖关系和添加ACL

您可以收到有关网络实用程序包依赖项的警告。使用此过程评估依赖关系，并通过添加适当的访问控制列表（ACL）来提供访问权限。

  1. 运行升级前信息工具。

  2. 检查升级前信息工具（ preupgrade.jar ）的输出以获取警告消息，例如以下示例：

    警告： - >数据库包含具有依赖于网络包的对象的模式。....有关配置网络ACL的说明，请参阅“数据库升级指南”。....USER WKSYS具有依赖对象。....USER SYSMAN具有依赖对象。....USER FLOWS_010600具有依赖对象。。

  3. 查询视图DBA_DEPENDENCIES以获取有关依赖项的更多信息。例如：

    SELECT * FROM DBA_DEPENDENCIES WHERE referenced_name IN（'UTL_TCP'，'UTL_SMTP'，'UTL_MAIL'，'UTL_HTTP'，'UTL_INADDR'，'DBMS_LDAP'）和所有者NOT IN（'SYS'，'PUBLIC'，'ORDPLUGINS'）;

    使用程序包DBMS_NETWORK_ACL_ADMIN更新数据库访问控制列表（ACL）。您可以使用此程序包为新的访问控制创建，分配和添加权限，以便更新的访问控制包可以像以前的版本一样工作。请参阅Oracle数据库Real Application Security管理员和开发人员指南中提供的示例脚本，了解如何使用DBMS_NETWORK_ACL_ADMIN更新访问控制列表。

  5. 升级后，授予特定的所需权限。访问权限基于原始数据库中的使用情况。

关于独立服务器的Oracle Grid Infrastructure

如果您计划使用Oracle自动存储管理（Oracle ASM），则必须在安装数据库之前安装Oracle Restart。

用于独立服务器的Oracle Grid Infrastructure是支持单实例数据库的Oracle Grid Infrastructure版本。此支持包括卷管理，文件系统和自动重启功能。用于独立服务器的Oracle Grid Infrastructure包括Oracle Restart和Oracle自动存储管理。Oracle将这两种基础架构产品组合成一组安装在Oracle Restart主目录中的二进制文件。

Oracle Restart是作为Oracle Grid Infrastructure的一部分提供的功能。Oracle Restart监视并可以重新启动Oracle数据库实例，Oracle Net Listeners和Oracle ASM实例。Oracle Restart目前仅限于管理单实例Oracle数据库和Oracle ASM实例。

Oracle自动存储管理是Oracle数据库文件的卷管理器和文件系统，支持单实例Oracle数据库和Oracle Real Application Clusters（Oracle RAC）配置。Oracle自动存储管理还支持满足您的应用程序需求的通用文件系统，包括Oracle数据库二进制文件。Oracle自动存储管理是Oracle推荐的存储管理解决方案，可替代传统的卷管理器和文件系统。

Oracle Restart通过提供以下服务来提高Oracle数据库的可用性：

  • 当出现硬件或软件故障时，Oracle Restart会自动启动所有Oracle组件，包括Oracle数据库实例，Oracle Net Listener，数据库服务和Oracle ASM。

  • 重新启动数据库主机时，Oracle Restart以正确的顺序启动组件。

  • Oracle Restart会定期检查以监视Oracle组件的状态。如果组件的检查操作失败，则组件将关闭并重新启动。

请注意以下使用Oracle Restart的限制：

  • 您既不能在Oracle Grid Infrastructure集群成员节点上安装Oracle Restart，也不能将Oracle Restart服务器添加到Oracle Grid Infrastructure集群成员节点。Oracle Restart支持在一台服务器上支持单实例数据库，而Oracle Grid Infrastructure for Cluster支持群集上的单实例或Oracle RAC数据库。

  • 如果要使用Oracle ASM或Oracle Restart，则应在安装和创建数据库之前为独立服务器安装Oracle Grid Infrastructure。否则，您必须安装Oracle Restart，然后使用Oracle Restart手动注册数据库。

  • 您只能在单实例（非集群）环境中使用Oracle Grid Infrastructure的Oracle Restart实现。将Oracle Grid Infrastructure与Oracle Clusterware一起用于群集环境。

IPv4和IPv6协议要求

Oracle Grid Infrastructure和Oracle RAC支持RFC 2732指定的标准IPv6地址表示法以及RFC 4193定义的全局和站点本地IPv6地址。

配置公共VIP

群集成员节点接口可以配置为使用IPv4，IPv6或两种类型的Internet协议地址。在安装过程中，您可以将给定公共网络的VIP配置为IPv4或IPv6类型的地址。您可以通过选择解析为群集的IPv6子网中的地址的VIP和SCAN名称，并在安装期间将该子网选择为公共来配置IPv6群集。安装后，您还可以使用IPv4和IPv6地址的混合配置集群成员节点。

如果使用IPv4群集中的静态虚拟IP（VIP）地址进行安装，则在安装期间提供的VIP名称应仅解析为IPv4地址。如果使用静态IPv6地址进行安装，则在安装期间提供的VIP名称应仅解析为IPv6地址。

在安装过程中，您无法使用可解析为IPv4和IPv6地址的VIP和SCAN名称配置群集。您无法在某些群集成员节点上配置VIP和SCANS以解析为IPv4地址，并且无法在其他群集成员节点上配置VIP和SCAN以解析为IPv6地址。Oracle不支持此配置。

配置专用IP接口（互连）

您可以将专用网络配置为IPv4网络或IPv6网络。

冗余网络接口

如果为公共或VIP节点名称配置冗余网络接口，则将冗余对的两个接口配置为相同的地址协议。还要确保专用IP接口使用相同的IP协议。Oracle不支持使用具有混合IP协议的冗余接口配置的名称。您必须使用相同的IP协议配置冗余对的两个网络接口。

GNS或多群集地址

Oracle Grid Infrastructure支持IPv4 DHCP地址和使用无状态地址自动配置协议配置的IPv6地址，如RFC 2462中所述。

不支持RFC 1884中定义的链路本地和站点本地IPv6地址。

12.54 XQUERY

句法

XQUERY xquery_statement

SQL * Plus XQUERY命令使您可以在指定的数据库上执行XQuery 1.0查询。Oracle Database 10g （第2版）及更高版本支持XQUERY。尝试在早期版本的Oracle数据库上使用XQUERY会出现错误：

SP2-614服务器版本太低

条款

xquery_statement

指定要运行的XQuery语句。使用标准XQuery语法输入语句。XQUERY语句以正斜杠“/”结束。

用法

使用SQL * Plus命令XQUERY对XQuery语句进行前缀，并使用斜杠（/）终止XQUERY命令。XQUERY是一个SQL * Plus关键字。如果在其他工具中执行XQueries，则可能不需要该关键字。

根据活动的SET命令选项，XQUERY命令的XML输出显示为本机XML。通常需要设置SET LONG。考虑以下设置可能很有用：

  • 行的大小超过默认的80个字符（SET LINESIZE）。

  • 行的LOB，LONG和XML类型大小超过默认的80个字符（SET LONG）。

  • 输出页面设置以匹配输出（SET PAGESIZE）。

  • 显示标题以抑制“结果序列”列标题（SET HEADING OFF）。

XQUERY命令需要活动数据库连接。该命令不适用于SQLPLUS / NOLOG。

XQUERY命令不支持绑定变量。

有四个特定于XQUERY命令的SET命令。SHOW XQUERY命令提供这些设置的状态。他们是：

  • SET XQUERY BASEURI {text}

  • SET XQUERY ORDERING {UNORDERED |订购|默认}

  • SET XQUERY NODE {BYVALUE |相关|默认}

  • SET XQUERY上下文{text}

/u01/app/19.0.0/grid

对于查询返回的每一行， ROWNUM伪列返回一个数字，表示Oracle从一个表或一组连接行中选择行的顺序。选择的第一行的ROWNUM为1，第二行的数量为2，依此类推。

您可以使用ROWNUM来限制查询返回的行数，如下例所示：

SELECT * FROM employees WHERE ROWNUM <11;

如果ORDER BY子句在同一查询中跟随ROWNUM ，则ORDER BY子句将对这些行进行重新排序。结果可能会有所不同，具体取决于访问行的方式。例如，如果ORDER BY子句导致Oracle使用索引来访问数据，那么Oracle可以以与没有索引的顺序不同的顺序检索行。因此，以下语句不一定返回与前面示例相同的行：

SELECT * FROM employees WHERE ROWNUM <11 ORDER BY last_name;

如果在子查询中嵌入ORDER BY子句并将ROWNUM条件放在顶级查询中，则可以在排序行之后强制应用ROWNUM条件。例如，以下查询返回具有10个最小员工编号的员工。这有时被称为前N个 报告 ：

SELECT * FROM（SELECT * FROM employees ORDER BY employee_id）WHERE ROWNUM <11;

在前面的示例中， ROWNUM值是顶级SELECT语句的值，因此它们是在子查询中employee_id已经对行进行排序之后生成的。

测试ROWNUM值大于正整数的条件始终为false。例如，此查询不返回任何行：

SELECT * FROM employees WHERE ROWNUM> 1;

获取的第一行被赋予ROWNUM为1并使条件为假。要获取的第二行现在是第一行，并且还指定了ROWNUM为1并使条件为false。所有行随后都无法满足条件，因此不返回任何行。

您还可以使用ROWNUM为表的每一行分配唯一值，如下例所示：

UPDATE my_table SET column1 = ROWNUM;

有关为行分配唯一编号的替代方法，请参阅函数ROW_NUMBER 。

注意：

在查询中使用ROWNUM会影响视图优化。

Oracle Grid Infrastructure和Oracle RAC支持RFC 2732指定的标准IPv6地址表示法以及RFC 4193定义的全局和站点本地IPv6地址。

配置公共VIP

群集成员节点接口可以配置为使用IPv4，IPv6或两种类型的Internet协议地址。在安装过程中，您可以将给定公共网络的VIP配置为IPv4或IPv6类型的地址。您可以通过选择解析为群集的IPv6子网中的地址的VIP和SCAN名称，并在安装期间将该子网选择为公共来配置IPv6群集。安装后，您还可以使用IPv4和IPv6地址的混合配置集群成员节点。

配置专用IP接口（互连）

您可以将专用网络配置为IPv4网络或IPv6网络。

冗余网络接口

如果为公共或VIP节点名称配置冗余网络接口，则将冗余对的两个接口配置为相同的地址协议。还要确保专用IP接口使用相同的IP协议。Oracle不支持使用具有混合IP协议的冗余接口配置的名称。您必须使用相同的IP协议配置冗余对的两个网络接口。

GNS或多群集地址

Oracle Grid Infrastructure支持IPv4 DHCP地址和使用无状态地址自动配置协议配置的IPv6地址，如RFC 2462中所述。

不支持RFC 1884中定义的链路本地和站点本地IPv6地址。

升级和数据迁移方法和过程

Oracle提供的功能和产品可自动完成升级过程，并帮助您有效地完成升级。

Oracle数据库支持以下用于将数据库升级或迁移到新版本的方法：

  • 数据库升级助手（DBUA）

    提供图形用户界面，指导您完成数据库的升级。DBUA可以在使用Oracle Universal Installer进行安装期间启动，也可以在将来随时启动DBUA作为独立工具。

  • 使用Parallel Upgrade Utility和其他命令行实用程序手动升级

    允许使用shell脚本执行升级。

  • 使用Oracle Data Pump迁移数据

    提供导出和导入实用程序。Oracle Data Pump可以从您的数据库执行完全或部分导出，然后完全或部分导入到Oracle数据库的新版本中。Oracle Data Pump中的导出/导入可以复制数据的子集，保持数据库不变。

  • CREATE TABLE AS SQL语句

    将数据从数据库迁移到新的Oracle数据库版本。通过使用此方法，您可以复制数据的子集，保持数据库不变。

  • 使用优先级列表升级CDB和PDB，以根据优先级对PDB进行分组和升级。

    使用-L选项运行并行升级实用程序（ dbupgrade或catctl.pl ）以使用优先级列表运行升级，并在升级运行时调用该列表。

  • 同步备用数据库，升级和使用Oracle GoldenGate同步升级后的数据库（零停机时间选项）

  • 使用Fleet Patching and Provisioning（FPP）升级数据库。

    在Fleet Patching and Provisioning（FPP）升级（以前称为Rapid Home Provisioning）中，您将完成新的Oracle数据库安装。测试数据库并根据要用于数据库的标准操作环境（SOE）对其进行修改后，可以创建FPP黄金映像。DBA将该黄金映像的实例部署到具有您要升级的早期版本数据库的服务器。部署这些黄金映像后，DBA可以运行单个rhpctl命令来移动文件，执行配置更改以及执行使用新二进制文件所需的其他步骤。有关Rapid Home Provisioning的详细信息，请参阅“ Oracle Clusterware管理和部署指南” 。

确定您是否具有现有Oracle中央库存，并确保为所有Oracle软件安装使用相同的Oracle Inventory。此外，请确保您打算用于安装的所有Oracle软件用户都有权写入此目录。

为Oracle XML DB配置FTP和HTTP端口以及HTTP身份验证

Oracle数据库配置助手（DBCA）不会对Oracle数据库12c中配置的Oracle XML DB港口和以后升级版本使用摘要身份验证。

Oracle建议您在配置端口时，还要配置HTTP身份验证以访问Oracle XML DB存储库，以利用改进的安全功能。

与Oracle数据库12c的开始，Oracle通过支持摘要式身份验证增强数据库的安全性。摘要式身份验证是一种行业标准协议，通常与HTTP协议一起使用。大多数HTTP客户端都支持它。摘要式身份验证可确保始终以安全的方式传输密码，即使未使用加密（HTTPS）连接也是如此。支持摘要式身份验证使组织能够部署使用Oracle XML DB HTTP的应用程序，而无需担心密码被泄露。Oracle XML DB中的摘要式身份验证支持还可确保Oracle XML DB HTTP服务器与Microsoft Web Folders WebDAV客户端保持兼容。

安装或升级新版本后，必须手动为Oracle XML DB配置FTP和HTTP端口，如下所示：

  1. 使用DBMS_XDB_CONFIG.setHTTPPort( HTTP_port_number )为Oracle XML DB设置HTTP端口：

    SQL> exec DBMS_XDB_CONFIG.setHTTPPort（ port_number ）;

  2. 使用DBMS_XDB_CONFIG.setFTPPort( FTP_port_number )设置Oracle XML DB的FTP端口：

    SQL> exec DBMS_XDB_CONFIG.setFTPPort（ FTP_port_number ）;

    注意：

    您可以分别使用DBMS_XDB_CONFIG.getFTPPort和DBMS_XDB_CONFIG.getHTTPPort查询过程中用于FTP和HTTP的端口号。

  3. 要查看所有使用的端口号，请查询DBMS_XDB_CONFIG.usedport 。

关于安装期间的字符集选择

在创建数据库之前，请确定要使用的字符集。

创建数据库后，更改其字符集通常在时间和资源方面非常昂贵。此类操作可能需要通过导出整个数据库并将其导回来转换所有字符数据。因此，在安装时仔细选择数据库字符集非常重要。

Oracle数据库使用以下字符集：

  • 存储在SQL字符数据类型（ CHAR ， VARCHAR2 ， CLOB和LONG ）中的数据。

  • 表名，列名和PL / SQL变量等标识符。

  • 存储的SQL和PL / SQL源代码，包括嵌入在此代码中的文本文字。

从Oracle Database 12c第2版（12.2）开始，从通用/事务处理或数据仓库模板创建的数据库的缺省数据库字符集是Unicode AL32UTF8 。

Unicode是通用字符集，支持世界上大多数当前使用的语言。它还支持许多历史脚本（字母表）。Unicode是许多技术的本机编码，包括Java，XML，XHTML，ECMAScript和LDAP。 Unicode非常适合支持互联网和全球经济的数据库。

由于AL32UTF8是一个多字节字符集，因此与单字节数据库字符集（如WE8ISO8859P1或WE8MSWIN1252相比，对字符数据的数据库操作可能会稍微慢WE8MSWIN1252 。与支持该语言的传统字符集相比， AL32UTF8中使用ASCII AL32UTF8集以外的字符的大多数语言的文本的存储空间要求更高。仅当存储在CLOB （字符大对象）列中时，英语数据可能需要更多空间。非字符数据类型（如NUMBER或DATE ）的存储不依赖于字符集。Unicode的普遍性和灵活性通常超过了这些额外的成本。

仅当数据库需要支持单组语言时才考虑使用旧版字符集，并且使用旧版字符集对于满足兼容性，存储或性能要求至关重要。在这种情况下要选择的数据库字符集是连接到此数据库的大多数客户端的字符集。

多租户容器数据库（CDB）的数据库字符集确定以后可以插入哪些数据库。确保为CDB选择的字符集与要插入此CDB的数据库的数据库字符集兼容。如果使用Unicode AL32UTF8作为CDB字符集，则可以在Oracle数据库支持的任何数据库字符集中插入可插拔数据库（PDB）（基于EBCDIC的字符集除外）。

关于删除分区表

删除分区表类似于删除非分区表。

Oracle数据库处理分区表的DROP TABLE语句的方式与处理非分区表的语句的方式相同。使用PURGE关键字时有一个例外。

为避免遇到资源限制， DROP TABLE ...分区表的PURGE语句将表丢弃在多个事务中，其中每个事务都丢弃分区或子分区的子集，然后提交。该表在最终交易结束时被删除。

此行为伴随对DROP TABLE语句的一些更改。首先，如果DROP TABLE ...PURGE语句失败，然后您可以采取纠正措施（如果有），然后重新发出该语句。声明在失败时恢复。第二，而DROP TABLE ......PURGE语句正在进行中，通过在以下数据字典视图中将STATUS列设置为值UNUSABLE ，将表标记为不可用：

  • USER_TABLES ， ALL_TABLES ， DBA_TABLES

  • USER_PART_TABLES ， ALL_PART_TABLES ， DBA_PART_TABLES

  • USER_OBJECT_TABLES ， ALL_OBJECT_TABLES ， DBA_OBJECT_TABLES

您可以通过查询这些视图的STATUS列来列出所有UNUSABLE分区表。

针对与分区有关的其他数据字典视图（例如DBA_TAB_PARTITIONS和DBA_TAB_SUBPARTITIONS ）的查询排除了属于UNUSABLE表的行。

在将表标记为UNUSABLE ，可以针对它发出的唯一语句是另一个DROP TABLE ...PURGE语句，仅当前一个DROP TABLE ...PURGE声明失败。针对UNUSABLE表发出的任何其他语句都会导致错误。在丢弃操作完成之前，该表仍处于UNUSABLE状态。

关于安装期间的字符集选择

在创建数据库之前，请确定要使用的字符集。

创建数据库后，更改其字符集通常在时间和资源方面非常昂贵。此类操作可能需要通过导出整个数据库并将其导回来转换所有字符数据。因此，在安装时仔细选择数据库字符集非常重要。

Oracle数据库使用以下字符集：

  • 存储在SQL字符数据类型（ CHAR ， VARCHAR2 ， CLOB和LONG ）中的数据。

  • 表名，列名和PL / SQL变量等标识符。

  • 存储的SQL和PL / SQL源代码，包括嵌入在此代码中的文本文字。

从Oracle Database 12c第2版（12.2）开始，从通用/事务处理或数据仓库模板创建的数据库的缺省数据库字符集是Unicode AL32UTF8 。

Unicode是通用字符集，支持世界上大多数当前使用的语言。它还支持许多历史脚本（字母表）。Unicode是许多技术的本机编码，包括Java，XML，XHTML，ECMAScript和LDAP。 Unicode非常适合支持互联网和全球经济的数据库。

由于AL32UTF8是一个多字节字符集，因此与单字节数据库字符集（如WE8ISO8859P1或WE8MSWIN1252相比，对字符数据的数据库操作可能会稍微慢WE8MSWIN1252 。与支持该语言的传统字符集相比， AL32UTF8中使用ASCII AL32UTF8集以外的字符的大多数语言的文本的存储空间要求更高。仅当存储在CLOB （字符大对象）列中时，英语数据可能需要更多空间。非字符数据类型（如NUMBER或DATE ）的存储不依赖于字符集。Unicode的普遍性和灵活性通常超过了这些额外的成本。

仅当数据库需要支持单组语言时才考虑使用旧版字符集，并且使用旧版字符集对于满足兼容性，存储或性能要求至关重要。在这种情况下要选择的数据库字符集是连接到此数据库的大多数客户端的字符集。

多租户容器数据库（CDB）的数据库字符集确定以后可以插入哪些数据库。确保为CDB选择的字符集与要插入此CDB的数据库的数据库字符集兼容。如果使用Unicode AL32UTF8作为CDB字符集，则可以在Oracle数据库支持的任何数据库字符集中插入可插拔数据库（PDB）（基于EBCDIC的字符集除外）。

数据字典翻译

数据字典是有关由系统的各个用户创建的数据库对象的信息的集合。

大多数数据库系统都有某种形式的数据字典。对于关系数据库，数据字典是一组表和视图，其中包含有关数据库中数据的信息。此信息包括有关正在使用系统的用户以及他们创建的对象（例如表，视图和触发器）的信息。几乎所有数据字典（无论数据库系统如何）都包含相同的信息，但每个数据库系统以不同方式组织信息。

例如， ALL_CATALOG Oracle数据字典视图提供数据库中的表，视图和序列的列表。它有三列：第一列叫做OWNER ，它是对象所有者的名字;第二个叫做TABLE_NAME ，它是对象的名称;第三个叫做TABLE_TYPE ，它是数据类型。该字段具有值TABLE ， VIEW ， SEQUENCE等，具体取决于对象类型。但是，在Sybase中，相同的信息存储在两个名为sysusers和sysobjects的表sysusers ，这些表的列名与Oracle ALL_CATALOG表的列名不同。此外，在Oracle数据库中，表类型是一个字符串，其值为TABLE或VIEW 。对于Sybase，它是一个字母，例如， U表示用户表; S表示系统表; V表示视图，依此类推。

如果客户端程序需要Sybase系统上的表ALL_CATALOG信息，则它会将引用ALL_CATALOG@ database_link的查询发送到网关。异构服务将此查询转换为systables上的相应查询，然后将已翻译的查询发送到Sybase系统，例如：

SELECT SU。“name”OWNER，SO。“name”TABLE_NAME，DECODE（SO。“type”，“U”，“TABLE”，“S”，“TABLE”，“V”，“VIEW”）TABLE_TYPE FROM“ dbo“。”sysusers“@remote_db SU，”dbo“。”sysobjects“@remote_db SO WHERE SU。”uid“= SO。”uid“AND（SO。”type“='V'或SO。”type“= 'S'或SO。“type”='U'）;

要将Oracle数据字典表上的查询转换为非Oracle系统数据字典表中的等效转换，异构服务需要该非Oracle系统的数据字典转换。数据字典转换是一个或多个非Oracle系统数据字典表的视图定义（本质上是SELECT语句），它们看起来像Oracle数据字典表，具有相同的列名和相同的格式。大多数数据字典翻译并不像前面的例子那么简单。通常，信息分散在许多表中，而数据字典转换是这些表的复杂连接。

在某些情况下，Oracle数据字典表没有转换，因为非Oracle系统上不存在该信息。在这种情况下，网关不得上传翻译，或者网关可能实施称为模仿的替代方法。 如果网关模仿数据字典表，则通知异构服务，异构服务将通过查询本地数据库获取数据字典表的描述。当要求检索数据时，它将报告未选择任何行。

    使用GNS和DHCP时，Oracle Clusterware会为群集配置期间提供的SCAN名称配置VIP地址。使用GNS时，从DHCP服务器获取节点VIP和三个SCAN VIP。如果新服务器加入群集，则Oracle Clusterware会从DHCP服务器动态获取所需的VIP地址，更新群集资源，并通过GNS访问服务器。

关于安装期间的字符集选择

在创建数据库之前，请确定要使用的字符集。

创建数据库后，更改其字符集通常在时间和资源方面非常昂贵。此类操作可能需要通过导出整个数据库并将其导回来转换所有字符数据。因此，在安装时仔细选择数据库字符集非常重要。

Oracle数据库使用以下字符集：

  • 存储在SQL字符数据类型（ CHAR ， VARCHAR2 ， CLOB和LONG ）中的数据。

  • 表名，列名和PL / SQL变量等标识符。

  • 存储的SQL和PL / SQL源代码，包括嵌入在此代码中的文本文字。

从Oracle Database 12c第2版（12.2）开始，从通用/事务处理或数据仓库模板创建的数据库的缺省数据库字符集是Unicode AL32UTF8 。

Unicode是通用字符集，支持世界上大多数当前使用的语言。它还支持许多历史脚本（字母表）。Unicode是许多技术的本机编码，包括Java，XML，XHTML，ECMAScript和LDAP。 Unicode非常适合支持互联网和全球经济的数据库。

由于AL32UTF8是一个多字节字符集，因此与单字节数据库字符集（如WE8ISO8859P1或WE8MSWIN1252相比，对字符数据的数据库操作可能会稍微慢WE8MSWIN1252 。与支持该语言的传统字符集相比， AL32UTF8中使用ASCII AL32UTF8集以外的字符的大多数语言的文本的存储空间要求更高。仅当存储在CLOB （字符大对象）列中时，英语数据可能需要更多空间。非字符数据类型（如NUMBER或DATE ）的存储不依赖于字符集。Unicode的普遍性和灵活性通常超过了这些额外的成本。

仅当数据库需要支持单组语言时才考虑使用旧版字符集，并且使用旧版字符集对于满足兼容性，存储或性能要求至关重要。在这种情况下要选择的数据库字符集是连接到此数据库的大多数客户端的字符集。

多租户容器数据库（CDB）的数据库字符集确定以后可以插入哪些数据库。确保为CDB选择的字符集与要插入此CDB的数据库的数据库字符集兼容。如果使用Unicode AL32UTF8作为CDB字符集，则可以在Oracle数据库支持的任何数据库字符集中插入可插拔数据库（PDB）（基于EBCDIC的字符集除外）。
