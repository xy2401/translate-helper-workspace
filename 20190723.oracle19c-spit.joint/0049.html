9.5.1全球数据服务概述

自Oracle Database 12 c第1版（12.1）以来，全球数据服务（GDS）功能可用。通过此功能，仅在Oracle RAC中可用的快速连接故障转移，运行时连接负载平衡和连接关联功能已扩展为一组提供公共服务的复制数据库。

该组数据库可能包括通过Data Guard，GoldenGate或任何其他复制技术互连的Oracle RAC和单实例Oracle数据库。可以由多个数据库提供的数据库服务称为全局服务，因此可以将其与仅可由单个数据库提供的传统服务区分开来。此组合使服务可以部署在此全局分布式配置中的任何位置，从而支持负载平衡，高可用性，数据库关联性等。

9.5.2配置使用GDS的应用程序

UCP连接到Global Data Services的方式与连接到Oracle RAC上的本地服务的方式相同。连接字符串中的服务名称应该是全局服务的名称。端点应该是GDS侦听器的端点，而不是数据库的本地，远程或SCAN侦听器的端点。

客户端必须在连接字符串的REGION参数中指定其区域。这是GDS的新要求。区域名称是必需的，因为在GDS的情况下，会针对特定区域自定义运行时负载平衡建议。以下是典型连接字符串的示例：

（DESCRIPTION =（ADDRESS =（GDS_protocol_address_information））（CONNECT_DATA =（SERVICE_NAME = global_service_name）（REGION = region_name）））

与本地服务一样，UCP可以在同一连接字符串中指定多个GDS侦听器，以用于侦听器故障转移，负载平衡或两者。

注意：

GDS侦听器不支持SCAN，因此必须指定每个侦听器的端点。

（DESCRIPTION =（ADDRESS_LIST =（LOAD_BALANCE = ON）（FAILOVER = ON）（ADDRESS =（GDS_protocol_address_information））（ADDRESS =（GDS_protocol_address_information）））（CONNECT_DATA =（SERVICE_NAME = global_service_name）（REGION = region_name）））

如果在客户端连接字符串中仅指定了本地区域的全局服务管理器，则REGION参数是可选的。当GDS配置中只有一个区域时，或者当存在多个区域时，可能就是这种情况。但是，更改设计为使用单个数据库的现有客户端的连接字符串是不可行的。如果未指定REGION参数，则假定客户端的区域是用于连接到全局服务的全局服务管理器的区域。

注意：

除非在连接字符串中指定了REGION参数，否则只能使用具有单个区域的GDS配置的12c之前的瘦JDBC客户端。

前面示例中的所有GDS侦听器都属于运行UCP的同一区域，即本地区域。要提供高可用性，当本地区域中的所有GDS都不可用时，您可以在其他ADDRESS_LIST描述符中为伙伴区域指定GDS侦听器。 

（DESCRIPTION =（FAILOVER = on）（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（global_protocol_address_information））（ADDRESS =（global_protocol_address_information）））（ADDRESS_LIST =（LOAD_BALANCE = ON）（ADDRESS =（global_protocol_address_information））（ADDRESS =（ global_protocol_address_information）））（CONNECT_DATA =（SERVICE_NAME = global_service_name）（REGION = region_name）））

  • 要为GDS启用自动ONS配置，必须在UCP上启用快速连接故障转移（FCF）。

  • 自动ONS配置仅适用于Oracle GDS和Oracle RAC。它不适用于单实例Oracle数据库。

    • 调用PoolDataSource.setONSConfiguration(String)方法

    • 在本地ONS配置文件中添加ONS钱包或密钥库参数

B DBMS_JAVASCRIPT包

DBMS_JAVASCRIPT包提供了一个PL / SQL过程，用于从其参数标识的位置运行DBMS驻留的JavaScript代码。

本章包含以下主题：

  • DBMS_JAVASCRIPT概述

  • DBMS_JAVASCRIPT安全模型

B.1 DBMS_JAVASCRIPT概述

DBMS_JAVASCRIPT包是AUTHID CURRENT_USER包，并提供单个过程RUN ，用于运行JavaScript代码。RDBMS用户将他们的JavaScript代码保存在RDBMS中，作为存储在各自模式中的Java资源对象。

您可以使用以下两种方法之一在模式中创建脚本：

  • 使用loadjava工具或调用dbms_java.loadjava过程

  • 使用CREATE [OR REPLACE] JAVA RESOURCE命令

B.1.1运行程序

此过程调用Java代码，该代码定位Java资源，其名称作为script_name参数提供。然后，它通过Nashorn JavaScript编译器和运行时引擎提交其内容，以进行语法和语义分析，编译和评估。

在执行之前，Nashorn代码被编译为Java字节码。

句法

procedure run（script_name varchar2）;

您可以通过以下方式调用此过程：

调用dbms_javascript.run（'my_script.js'）;

注意：

  • 要运行此过程，必须授予DBJAVASCRIPT角色。

  • DBMS_JAVASCRIPT.RUN过程不运行JavaScript文本，也不查找文件系统中的脚本。Java资源名称是此过程的唯一有效输入。

B.2 DBMS_JAVASCRIPT安全模型

为了能够运行JavaScript代码，必须为RDBMS用户授予角色DBJAVASCRIPT。

DBMS_JAVASCRIPT.RUN过程的参数必须是一个字符串，用于标识先前加载到用户自己的模式中的有效Java资源。该过程拒绝以下参数：

  • 开放编码的脚本

  • 文件路径

  • 网址

这些限制对于防止用户编写PL / SQL代码很重要，PL / SQL代码从可能与JavaScript注入攻击有关的片段构建脚本。DBMS_JAVASCRIPT.RUN过程调用的Nashorn引擎的脚本模式扩展名被无条件禁用，包括SYS用户在内的任何人都无法运行它们。引擎实例化和使用依赖于oracle.DbmsJavaScriptUser permission授予。此外，所有与Nashorn相关的API都位于Oracle JVM的受限制的包空间中，并且当脚本尝试加载Java类时，将无条件地应用类过滤。类过滤实现以下目的：

  • 拒绝任何未通过SecurityManager管理的Java包访问检查的类

  • 无条件地限制包java.security ， java.lang.reflect和oracle.aurora.rdbms.security以供任何用户加载

DBA使用DBJAVASCRIPT角色来管理数据库中的JavaScript使用。该角色被授予唯一的授权，这是一个oracle.DbmsJavaScriptUser权限。

DML锁是在正在进行DML操作（插入，更新，删除）的表上获得的锁。DML_LOCKS值应等于所有用户当前引用的表的锁总数。例如，如果三个用户正在修改一个表中的数据，则需要三个条目。如果三个用户正在修改两个表中的数据，那么将需要六个条目。

  • 您不能使用DROP TABLE ， CREATE INDEX语句

  • 您不能使用显式锁定语句，例如LOCK TABLE IN EXCLUSIVE MODE

  • 企业管理器无法在DML_LOCKS设置为0任何实例上运行

3.1.3 PL / SQL如何处理未初始化的对象

用户定义的类型，就像集合一样，在原子上为null，直到通过调用其对象类型的构造函数来初始化对象。也就是说，对象本身是null，而不仅仅是它的属性。

将null对象与任何其他对象进行比较总会产生NULL 。此外，如果将原子空对象分配给另一个对象，则另一个对象将自动变为空（并且必须重新初始化）。同样，如果将非值NULL指定给对象，则该对象将自动变为空。

在表达式中，未初始化对象的属性求值为NULL 。当应用于未初始化的对象或其属性时， IS NULL比较运算符产生TRUE 。

3.3带有继承的PL / SQL中的重载使用

重载允许您将子类型值替换为超类型的形式参数。这种能力被称为可替代性。

以下规则涉及重载和可替代性。

替代规则

如果重载过程的多个实例与过程调用匹配，则以下替换规则确定调用哪个过程（如果有）：

  • 如果重载过程的签名仅在于某些参数是来自相同超类型子类型层次结构的对象类型，则使用最接近的匹配。最接近的匹配是所有参数至少与任何其他重载实例一样接近的匹配，由子类型和超类型之间的继承深度确定，并且至少一个参数更接近。

  • 如果两个重载方法的实例匹配，并且某些参数类型在一个重载过程中更接近而其他参数在第二个过程中更接近，则会发生语义错误。

  • 如果某些参数在对象类型层次结构中的位置不同，并且其他参数具有不同的数据类型，因此需要进行隐式转换，则会发生语义错误。

安装前，安装介质中提供了卸载工具（ deinstall ），安装后可在Oracle主目录中使用。它位于ORACLE_HOME \deinstall 。

deinstall命令停止Oracle软件，并删除操作系统上的Oracle软件和配置文件。

该脚本使用以下语法，其中变量内容以斜体表示：

deinstall -home home的完整路径 [-silent] [-checkonly] [-local] [-paramfile 输入参数属性文件的完整路径 ] [-params name1 = value name2 = value 。 。 。] [-o 保存文件的目录的完整路径 ] [-help | -H]

选项是：

  • -无声

    使用此标志以非交互模式运行命令。此选项需要一个属性文件，其中包含正在卸载或取消配置的Oracle主目录的配置值。

  • -checkonly

    使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行该命令不会删除Oracle配置。

  • -本地

    在多节点环境中使用此标志可在群集中取消配置Oracle软件。

    使用此标志运行deconfig时，它会在本地节点（运行deconfig的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

TO_BINARY_FLOAT

句法


to_binary_float.eps插图的描述

目的

TO_BINARY_FLOAT将expr转换为单精度浮点数。

  • expr可以是任何计算结果为CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2类型的字符串的表达式， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE类型的数值，或null。如果expr是BINARY_FLOAT ，则该函数返回expr 。如果expr求值为null，则该函数返回null。否则，该函数将expr转换为BINARY_FLOAT值。

  • 可选的DEFAULT return_value ON CONVERSION ERROR子句允许您在将expr转换为BINARY_FLOAT时发生错误时指定此函数返回的值。如果在计算expr时发生错误，则此子句无效。return_value可以是表达式或绑定变量，并且必须求值为CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2类型的字符串， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE类型的数值，或null。该函数将return_value到BINARY_FLOAT在其转换以同样的方式expr到BINARY_FLOAT 。如果return_value无法转换为BINARY_FLOAT ，则该函数返回错误。

  • 可选的' fmt '和' nlsparam '参数与TO_NUMBER函数的用途相同。如果指定这些参数，那么expr和return_value （如果指定）必须都是字符串或null。如果其中一个是字符串，则该函数使用fmt和nlsparam参数将字符串转换为BINARY_FLOAT值。

如果expr或return_value计算为以下字符串，则该函数将按如下方式转换它们：

  • 不区分大小写的字符串' INF '将转换为正无穷大。

  • 不区分大小写的字符串' - INF '将转换为否定标识。

  • 不区分大小写的字符串' NaN '将转换为NaN （不是数字）。

您不能在字符串expr使用浮点数字格式元素（ F ， f ， D或d ）。

从字符串或NUMBER到BINARY_FLOAT转换可能不准确，因为NUMBER和字符类型使用小数精度来表示数值，而BINARY_FLOAT使用二进制精度。

如果BINARY_DOUBLE值使用比BINARY_FLOAT支持的精度更多的位，则从BINARY_DOUBLE到BINARY_FLOAT转换是不精确的。

目的

CUBE_TABLE从多维数据集或维度中提取数据，并以关系表的二维格式返回，可由基于SQL的应用程序使用。

该函数采用单个VARCHAR2参数。可选的层次结构子句使您可以指定维层次结构。多维数据集可以有多个层次结构子句，每个维度一个。

您可以生成这些不同类型的表：

  • 多维数据集表包含每个维度的键列以及多维数据集中每个度量和计算度量的列。要创建多维数据集表，可以指定具有或不包含多维数据集层次结构子句的多维数据集。对于具有多个层次结构的维，此子句将返回值限制为指定层次结构中的维成员和级别。如果没有层次结构子句，则包括所有维度成员和所有级别。

  • 维度表包含键列，以及每个级别和每个属性的列。它还包含MEMBER_TYPE列，该列使用以下代码之一标识每个成员：

    • L - 从表，视图或同义词加载

    • A - 已加载的成员和维度中所有层次结构的单个根，即“所有”聚合成员

    • C - 计算成员

    所有维度成员和所有级别都包含在表中。要创建维度表，请指定不带维度层次结构子句的维度。

  • 层次结构表包含维度表的所有列以及父成员的列和每个源级别的列。它还包含MEMBER_TYPE列，如维度表所述。从表中排除任何不属于命名层次结构的维度成员和级别。要创建一个层次结构表，请用维层次条款的维度。

以root身份运行运行状况检查此外，将运行状况检查作为Oracle数据库主所有者或Oracle Grid Infrastructure主所有者运行。

大多数运行状况检查不需要root访问权限。但是，您需要root权限才能运行运行状况检查的子集。

要运行root权限检查，Oracle ORAchk使用脚本root_orachk.sh ，Oracle EXAchk使用脚本root_exachk.sh 。

默认情况下， root_orachk.sh和root_exachk.sh脚本在Oracle ORAchk和Oracle EXAchk使用的$HOME目录中创建。通过设置环境变量RAT_ROOT_SH_DIR更改目录。

指定sudo远程访问的位置，如下所示：
export RAT_ROOT_SH_DIR = / mylocation
在/ etc / sudoers中添加一个条目，如下所示：
oracle ALL =（root）NOPASSWD：/mylocation/root_orachk.sh

出于安全原因，请在自定义目录中的标准临时目录之外创建root脚本。

要确定运行Oracle ORAchk和Oracle EXAchk的用户：

  1. 使用RAT_ROOT_SH_DIR环境变量指定自定义目录。
    export RAT_ROOT_SH_DIR = / orahome / oradb /
  2. 指定sudo远程访问的位置。
    export RAT_ROOT_SH_DIR = / mylocation
  3. 在/etc/sudoers文件中添加一个条目。
    oracle ALL =（root）NOPASSWD：/mylocation/root_orachk.sh

    注意：

    指定/etc/sudoers文件中条目的完整路径。不要使用环境变量。

  4. （推荐）以root身份运行Oracle ORAchk和Oracle EXAchk。

    使用root用户凭据运行Oracle ORAchk和Oracle EXAchk。

    以root身份运行的Oracle ORAchk和Oracle EXAchk进程，为拥有Oracle数据库主目录和Oracle Grid Infrastructure主root的用户执行用户查找。如果不需要root访问权限，则Oracle ORAchk和Oracle EXAchk进程使用su命令作为适用的Oracle数据库主用户或Oracle Grid Infrastructure主用户运行运行状况检查。具有较低权限的帐户无法提升对运行需要root访问权限的运行状况检查的访问权限。

    以root身份运行运行状况检查在角色分离环境或具有更严格安全性的环境中具有优势。

  5. 将Oracle ORAchk和Oracle EXAchk作为Oracle数据库主人或Oracle Grid Infrastructure主人运行：

    使用Oracle数据库主所有者或Oracle Grid Infrastructure主所有者凭据来运行Oracle ORAchk和Oracle EXAchk。

    运行Oracle ORAchk和Oracle EXAchk的用户必须以root身份提升访问root才能运行需要root访问权限的运行状况检查。

    运行运行状况检查，因为Oracle数据库主所有者或Oracle Grid Infrastructure主所有者需要在角色分离环境中进行多次运行。更严格的安全要求不允许提升访问权限。

    还有其他几种选择：

    • 跳过需要root访问权限的检查。

    • 出现提示时指定root用户标识和密码。

    • 配置sudo 。

      如果您使用的是sudo ，则为root脚本添加一个条目，该条目位于/etc/sudoers文件中的$HOME中，该文件对应于运行运行状况检查的用户。

      要确定$HOME的设置，请运行echo $HOME命令。

      例如：
      用户ALL =（root）NOPASSWD：/root/root_orachk.sh
      user ALL =（root）NOPASSWD：/root/root_exachk.sh

    • 预配置无密码SSH连接。
