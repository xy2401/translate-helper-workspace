值

DATA_GUARD_SYNC_LATENCY控制Log Writer（LGWR）进程在一系列Oracle Data Guard SYNC重做传输模式连接中等待第一个响应之后的秒数。

属性 描述

参数类型

整数

默认值

0

可修改

改变系统

可在PDB中修改

价值范围

基本

Oracle RAC

必须在所有实例上使用相同的值。

默认值为0 ，这意味着LGWR将等待每个SYNC备用目标的LOG_ARCHIVE_DEST_n参数的NET_TIMEOUT属性指定的NET_TIMEOUT 。

例如，在具有两个在SYNC模式下接收重做的备用数据库的Oracle Data Guard配置中，您将定义LOG_ARCHIVE_DEST_2和LOG_ARCHIVE_DEST_3以使用SYNC传输， DATA_GUARD_SYNC_LATENCY设置为2秒的值。当需要将重做发送到备用数据库时，它将并行发送到LOG_ARCHIVE_DEST_2和LOG_ARCHIVE_DEST_3连接，并且LGWR将等待确认。如果LOG_ARCHIVE_DEST_2以其确认首先收到重做的消息进行响应，则LGWR将不会等待LOG_ARCHIVE_DEST_3响应超过2秒（ DATA_GUARD_SYNC_LATENCY ）。如果LOG_ARCHIVE_DEST_3在这2秒内没有响应，则LGWR与LOG_ARCHIVE_DEST_3断开连接，使其进入错误状态。重做继续像往常一样发送到LOG_ARCHIVE_DEST_2 。在LOG_ARCHIVE_DEST_3的REOPEN属性的持续时间过去之后，LGWR重新连接到LOG_ARCHIVE_DEST_3并继续重做传输。通过这种方式，具有第二个SYNC备用数据库的最大影响将比仅有一个SYNC备用数据库限制为2秒，同时允许Oracle Data Guard配置保持所需的保护级别。

无论DATA_GUARD_SYNC_LATENCY的值如何，LGWR永远不会等待LOG_ARCHIVE_DEST_n参数的NET_TIMEOUT属性的值。

值

可以为USE_LARGE_PAGES指定这些值：

  • 真正

    指定如果在系统上配置了大页面，则实例可以使用大页面。

    在Oracle Database 11g第2版（11.2.0.2）中，如果系统上没有配置足够的大页面，则将使用常规大小的页面来分配SGA内存。这可能导致空闲大页面未使用，操作系统可以分配大量内存来创建页表，以将SGA映射到Oracle进程的物理页面。这可能会导致ORA-04030错误并严重影响实例性能。

  • 假

    指定实例不使用大页面。建议不要使用此设置，因为它可能会导致实例性能严重下降。

  • 只要

    指定如果大页面不能用于整个SGA内存，则实例将无法启动。

关于卸载工具

安装前，安装介质中提供了卸载工具（ deinstall ），安装后可在Oracle主目录中使用。它位于路径$ORACLE_HOME/deinstall 。

deinstall命令停止Oracle软件，并删除操作系统上的Oracle软件和配置文件。

该命令使用以下语法，其中变量内容以斜体表示：

deinstall -home home的完整路径 [-silent] [-checkonly] [-local] [-paramfile 输入参数属性文件的完整路径 ] [-params name1 = value name2 = value 。 。 。] [-o 保存文件的目录的完整路径 ] [-help | -H]

选项是：

  • -无声

    使用此标志以静默或响应文件模式运行命令。如果使用-silent标志，则必须使用-paramfile标志，并提供一个参数文件，其中包含要卸载或取消配置的Oracle主目录的配置值。

    您可以通过使用-checkonly标志运行deinstall来生成要使用或修改的参数文件。然后， deinstall命令会发现要从中卸载和取消配置的Oracle主目录中的信息。它会生成属性文件，然后您可以将其与-silent选项一起使用。

    您还可以修改位于响应文件夹中的模板文件deinstall.rsp.tmpl 。

  • -checkonly

    使用此标志可以检查Oracle软件主目录配置的状态。使用-checkonly标志运行该命令不会删除Oracle配置。-checkonly标志生成一个可以与deinstall命令一起使用的参数文件。

  • -本地

    在多节点环境中使用此标志可在群集中取消配置Oracle软件。

    使用此标志运行deconfig时，它会在本地节点（运行deconfig的节点）上取消配置并卸载Oracle软件。在远程节点上，它取消配置Oracle软件，但不卸载Oracle软件。

  • -paramfile complete path of input parameter property file

    使用此标志可以使用默认位置以外的参数文件运行deconfig。使用此标志时，请提供参数文件所在的完整路径。

    参数文件的默认位置取决于deconfig的位置：

    • 从安装介质或阶段位置： $ORACLE_HOME/inventory/response

    • 来自OTN的解压缩归档文件： /ziplocation/response

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -params [name1= value name 2= value name3= value . . .]

    将此标志与参数文件一起使用，以覆盖要在已创建的参数文件中更改的一个或多个值。

  • -o complete path of directory for saving response files

    使用此标志提供除保存属性文件（ deinstall.rsp.tmpl ）的默认位置以外的路径。

    参数文件的默认位置取决于deconfig的位置：

    • 安装前从安装介质或阶段位置： $ORACLE_HOME/

    • 来自OTN的解压缩归档文件： /ziplocation/response/

    • 从已安装的Oracle主目录安装后： $ORACLE_HOME/deinstall/response

  • -help | -H

    使用help选项（ -help或-h ）获取有关命令选项标志的其他信息。

备份要升级的数据库后，在新位置准备新的Oracle主目录。将新Oracle Database发行版的软件安装到新位置。

  1. （仅限手动升级）将要升级的数据库的Oracle主目录中的配置文件复制到新版本Oracle Database Oracle主目录。如果您使用的是DBUA，则可以忽略此步骤，因为会自动为您复制配置文件。

    使用以下过程将配置文件复制到新的Oracle主目录：

    1. 如果参数文件位于旧环境Oracle主目录中，则将其复制到新的Oracle主目录。默认情况下，Oracle在Linux或UNIX平台上的ORACLE_HOME /dbs目录中以及Windows操作系统上的ORACLE_HOME \database目录中查找参数文件。升级后，参数文件可以驻留在其他任何位置，但它不能驻留在旧环境的Oracle主目录中。

      注意：

      如有必要，从服务器参数文件（ SPFILE ）创建文本初始化参数文件（ PFILE ），以便您可以编辑初始化参数。

    2. 如果参数文件位于Oracle ASM实例中，请使用以下命令备份参数文件：

      CREATE pfile FROM spfile;

      如果必须降级数据库并且SPFILE位于Oracle ASM中，则必须在降级之前恢复参数文件。

    4. 如果您的密码文件位于旧环境Oracle主目录中，则将密码文件移动或复制到新的Oracle主目录。

      密码文件的名称和位置是特定于操作系统的。如果SID是您的Oracle实例ID，您可以在以下位置找到密码文件：

      • Linux或UNIX平台：默认密码文件是orapw SID 。它位于ORACLE_HOME /dbs 。

      • Windows操作系统：默认密码文件是pwd SID .ora 。它位于ORACLE_HOME \database目录中。

PIPE ROW语句只能出现在流水线表函数的主体中，它会向函数的调用者返回一个表行（但不是控件）。

注意：

  • 如果流水线表函数是自治事务的一部分，那么它必须在每个PIPE ROW语句之前执行COMMIT或ROLLBACK ，以避免调用子程序中的错误。

  • 为了提高性能，PL / SQL运行时系统将管道行分批传递给调用者。

Memoptimized Rowstore提高了应用程序（如物联网（IoT））的数据查询性能，这些应用程序经常根据主键值查询表。

Memoptimized Rowstore提供了快速查找主要基于主键列查询的表的数据的功能。

此参数指定一个整数值，以指示用于为memoptimize池分配以下结构的SGA数量：

  • 缓冲区缓存区域的大小：这是所有MEMOPTIMIZE FOR READ表的块总数。

  • 散列索引的大小分段数据结构，指向MEMOPTIMIZE FOR READ表的特殊块。

计算正在考虑用于MEMOPTIMIZE FOR READ的表的缓冲区高速缓存要求，并包括对散列索引分段数据结构的额外25％内存要求。

这些结构在实例启动时从SGA分配。

为此参数指定的值将计入SGA_TARGET 。例如，如果将SGA_TARGET设置为10 GB并将MEMOPTIMIZE_POOL_SIZE设置为2 GB，则会将20％的SGA_TARGET设置分配给memoptimize池。

与其他SGA组件（如缓冲区高速缓存和共享池）不同，memoptimize池大小不受自动内存管理控制。当缓冲区缓存或共享池需要更多内存时，数据库不会自动收缩memoptimize池，或者当空间不足时增加memoptimize池。您只能通过手动调整MEMOPTIMIZE_POOL_SIZE初始化参数来增加memoptimize池的大小。
