ALTER DATABASE LINK

目的

当连接或身份验证用户的密码更改时，使用ALTER DATABASE LINK语句修改固定用户数据库链接。

注意：

  • 您不能使用此语句更改与数据库链接关联的连接或身份验证用户。要更改user ，必须重新创建数据库链接。

  • 您不能使用此语句来更改连接或身份验证用户的密码。为此，必须使用ALTER USER语句，然后使用ALTER DATABASE LINK语句ALTER DATABASE LINK 。

  • 此语句仅对固定用户数据库链接有效，对连接用户或当前用户数据库链接无效。有关这两种类型的数据库链接的详细信息，请参阅CREATE DATABASE LINK 。

先决条件

要更改专用数据库链接，您必须具有ALTER DATABASE LINK系统特权。若要更改公用数据库链接，您必须具有ALTER PUBLIC DATABASE LINK系统特权。

3.2注册数据库

目的

使用REGISTER DATABASE命令在恢复目录中注册目标数据库，以便RMAN可以维护其元数据。RMAN从目标数据库本身获取注册目标数据库所需的所有信息。

也可以看看：

Oracle数据库备份和恢复用户指南和CREATE CATALOG

先决条件

仅在RMAN提示符下执行此命令。必须将RMAN连接到恢复目录和已装入或打开的目标数据库。您注册的数据库当前不能在恢复目录中注册。

您只能使用恢复目录中唯一的DBID注册目标数据库。如果DBID值不同，则允许使用具有相同名称的数据库。您注册的数据库不能是备用数据库。

使用说明

当备份数据库的主数据库在恢复目录中注册并且满足以下任一条件时，RMAN会自动在恢复目录中注册新的备用数据库：

  • RMAN连接到具有恢复目录未知的DB_UNIQUE_NAME的数据库实例。

  • 您对恢复目录不知道的数据库执行CONFIGURE DB_UNIQUE_NAME命令。

当RMAN检测到重复的DBID时， REGISTER DATABASE命令失败。通过从现有数据库复制文件而不是使用DUPLICATE命令创建数据库时，可能会出现这种情况。如果发生此故障，则可以使用DBNEWID实用程序更改复制的数据库的DBID，然后重试REGISTER DATABASE命令。

如果使用RESETLOGS选项打开数据库并稍后在恢复目录中注册此数据库，则恢复目录DB_NAME旧版本的DB_NAME记录为UNKNOWN因为之前未注册旧版本。不要试图删除这些记录。

注意：

如果将RMAN与具有相同数据库名称和DBID的不同目标数据库一起使用，则在调用RMAN时请务必始终指定正确的恢复目录模式。

关于Oracle Inventory目录和安装

您指定为Oracle Inventory目录（oraInventory）的目录存储系统上安装的所有软件的清单。

服务器上的所有Oracle软件安装所有者都被授予OINSTALL权限以读取和写入此目录。如果您在服务器上安装了以前的Oracle软件，则其他Oracle软件安装会从/etc/oraInst.loc文件中检测此目录，并继续使用该Oracle Inventory。确保指定为OINSTALL组的组可用作所有计划的Oracle软件安装所有者的主要组。

如果您是第一次安装Oracle软件，则OUI会创建Oracle基础和中央库存，并使用以下优先级的信息创建Oracle库存：

  • 在为安装所有者用户帐户设置的ORACLE_BASE环境变量中指示的路径中

  • 在最佳灵活体系结构（OFA）路径中（u [01-99] /app/ owner ，其中owner是运行安装的用户帐户的名称），并且该用户帐户具有写入该路径的权限

  • 在用户主目录中的路径/app/ owner ， owner是运行安装的用户帐户的名称

例如：

如果您正在执行Oracle数据库安装，并且在安装之前将ORACLE_BASE for user oracle设置为路径/u01/app/oracle ，并为该路径授予oracle 755权限，那么Oracle Universal Installer会创建一个以上级别的Oracle Inventory目录路径ORACLE_BASE/../oraInventory中的ORACLE_BASE/../oraInventory ，因此Oracle Inventory路径为/u01/app/oraInventory 。Oracle Universal Installer在ORACLE_BASE路径中安装该软件。如果要为群集安装执行Oracle Grid Infrastructure，则在安装后将Grid安装路径更改为root所有权，并且Grid home软件位置应与Grid用户Oracle base位于不同的路径中。

如果创建OFA路径/u01 ，并授予oracle 755写入该路径的权限，则会在路径/u01/app/oraInventory创建Oracle Inventory目录，Oracle Universal Installer会创建路径/u01/app/oracle ，并将Oracle用户的ORACLE_BASE环境变量配置到该路径。如果您正在执行Oracle数据库安装，那么Oracle主目录将安装在Oracle基础之下。但是，如果要为群集安装Oracle Grid Infrastructure，请注意在安装后Grid Home的路径的所有权将更改为root ，并且Grid base和Grid home应位于不同的位置，例如/u01/app/19.0.0/grid网格主目录路径为/u01/app/19.0.0/grid ，网格基础为/u01/app/grid 。例如：

关于Oracle Extended Clusters

Oracle扩展群集由位于多个位置（称为站点）的节点组成。

部署Oracle独立群集时，还可以选择将群集配置为Oracle Extended Cluster。您可以跨两个或多个地理位置不同的站点扩展Oracle RAC集群，每个站点都配有自己的存储。如果其中一个站点发生故障，另一个站点将充当活动备用站点。

通常，Oracle ASM和Oracle数据库堆栈都设计为在数据中心中使用企业级共享存储。但是，光纤通道技术使您可以跨两个或多个数据中心分配计算和存储资源，并通过以太网电缆和光纤通道连接它们，分别满足计算和存储需求。

您可以在安装Oracle Grid Infrastructure时配置Oracle Extended Cluster。您也可以使用ConvertToExtended脚本在安装后执行此操作。您可以使用CRSCTL管理Oracle Extended Cluster。

您可以将节点和故障组分配给站点。站点包含故障组，故障组包含磁盘。

7.4编写打包的呼叫规范

一个 PL / SQL包是一个模式对象，它对逻辑上相关的类型，项和子程序进行分组。通常，包有两个部分，一个规范和一个身体。规范是应用程序的接口，并声明可供使用的类型，常量，变量，异常，游标和子程序。正文定义了游标和子程序。

在SQL * Plus中，您可以使用以下语法以交互方式定义PL / SQL包：

CREATE [OR REPLACE] PACKAGE package_name [AUTHID {CURRENT_USER | DEFINER}] {IS | AS} [type_definition [type_definition] ...][cursor_spec [cursor_spec] ...][item_declaration [item_declaration] ...][{subprogram_spec | call_spec} [{subprogram_spec | call_spec}] ...]END [package_name]; [CREATE [OR REPLACE] PACKAGE BODY package_name {IS | AS} [type_definition [type_definition] ...][cursor_body [cursor_body] ...][item_declaration [item_declaration] ...][{subprogram_spec | call_spec} [{subprogram_spec | call_spec}] ...][BEGIN sequence_of_statements] END [package_name];]

该规范包含对您的应用程序可见的公共声明。正文包含实现细节和私有声明，这些声明对应用程序是隐藏的。包的声明部分之后是body，它是可选的初始化部分。它包含初始化包变量的语句。它只运行一次，第一次引用包时。

在包规范中声明的调用规范不能具有相同的签名，即名称和参数列表，作为包体中的子程序。如果将包规范中的所有子程序声明为调用规范，则不需要包体，除非您要定义游标或使用初始化部分。

该 AUTHID子句确定所有打包的子程序是否以其定义者（ AUTHID DEFINER ）（默认值）或调用者（ AUTHID CURRENT_USER ）的权限运行。它还确定是否在定义者或调用者的模式中解析对模式对象的非限定引用。

7.2为性能类创建分类器的策略

查看性能类和工作负载分类器的要求。

目前，Oracle Database QoS Management管理的唯一工作负载是OLTP数据库工作负载。要管理数据库的工作负载，必须将传入的工作请求分配给性能类。Workload使用分类器映射到Performance Class。

在多层环境中，来自客户端的请求由中间层或通过负载平衡路由到不同的数据库会话，这使得跨数据库会话的客户端跟踪变得困难。分类器使用会话属性来标识工作请求。使用的属性是服务名称，用户名，模块，操作和程序。

每个分类器必须指定一个或多个服务名称。如果分类器指定多个服务名称，则在将连接数据与Performance Class匹配时，将使用OR运算来评估服务名称。如果分类器中指定的任何一个服务名称与工作请求中的服务名称匹配，则比较结果为TRUE 。

要设置MODULE和ACTION属性，请使用OCIAttrSet()调用。使用默认命名空间USERENV作为应用程序上下文。

您还可以选择在分类器中包含UserName和程序名称 。用户名是会话连接的数据库用户的名称。program属性是用于登录服务器的客户端程序的名称。

如果Performance Class的分类器指定了多个属性，则使用AND操作组合会话属性。如果分类器中指定的所有属性值与工作请求中的会话属性值匹配，则比较结果为TRUE 。如果您有多个使用类似属性值的分类器，则首先放置具有最细粒度条件的分类器。

例如，请考虑以下分类器：

  • create_invoice_taxes_pc ，指定sales_cart服务， ORDER模块和CALCULATE TAX操作

  • create_invoice_pc ，指定sales_cart服务和ORDER模块

该create_invoice_taxes_pc分类应该在之前进行评估create_invoice_pc分类。如果工作请求使用sales_cart服务，并且正在ORDER模块中执行CALCULATE TAX操作，则工作请求将分配给create_invoice_taxes_pc 。如果工作请求没有所有属性的匹配值，则将工作请求与create_invoice_pc的下一个分类器进行比较。如果首先评估create_invoice_pc分类器，则无论工作请求执行什么操作，都将将使用sales_cart服务和ORDER模块的任何工作请求分配给create_invoice_pc性能类。

您最多可以为群集创建47个性能类。如果您的群集有超过47个服务，则在分类器中使用多个服务名称。找到分类器的匹配项后，Oracle数据库QoS管理会为匹配的工作请求分配标记。标记基于评估为TRUE的分类器。

关于SCAN

Oracle数据库客户端使用单客户端访问名称（SCAN）连接到数据库。

SCAN及其关联的IP地址为客户端提供了一个稳定的名称，用于连接，与组成集群的节点无关。SCAN地址，虚拟IP地址和公共IP地址必须位于同一子网中。

SCAN是虚拟IP名称，类似于用于虚拟IP地址的名称，例如node1-vip 。但是，与虚拟IP不同，SCAN与整个群集相关联，而不是与单个节点相关联，并且与多个IP地址相关联，而不仅仅与一个地址相关联。

SCAN解析为多个IP地址，反映群集中处理公共客户端连接的多个侦听器。当客户端提交请求时，侦听SCAN IP地址和SCAN端口的SCAN侦听器可供客户端使用。由于群集上的所有服务都是使用SCAN侦听器注册的，因此SCAN侦听器会在当前正在提供服务的最小负载节点上回复本地侦听器的地址。最后，客户端通过提供服务的节点上的侦听器建立与服务的连接。所有这些操作都透明地发生在客户端上，而客户端不需要任何明确的配置。

在安装期间，将创建侦听器。这些SCAN侦听器侦听SCAN IP地址。SCAN侦听器在Oracle Clusterware确定的节点上启动。Oracle Net Services将应用程序请求路由到提供服务的最少负载的实例。由于SCAN地址解析为群集，而不是群集中的节点地址，因此可以在不影响SCAN地址配置的情况下将节点添加到群集或从群集中删除节点。SCAN侦听器还支持用于与Oracle XML数据库（XDB）通信的HTTP协议。

应配置SCAN，以便通过在群集中使用网格命名服务（GNS）或使用域名服务（DNS）解析来解析SCAN。为了实现高可用性和可伸缩性，Oracle建议您配置SCAN名称，以便将其解析为三个IP地址。SCAN必须至少解析为至少一个地址。

如果指定GNS域，则SCAN名称默认为clustername -scan. cluster_name.GNS_domain 。否则，它默认为clustername -scan. current_domain 。例如，如果从服务器node1启动Oracle Grid Infrastructure安装，则群集名称为mycluster ，GNS域为grid.example.com ，则SCAN名称为mycluster-scan.mycluster.grid.example.com 。

配置为在Oracle Database 11g第2版之前使用Oracle数据库版本的IP地址的客户端可以继续使用其现有的连接地址;不需要使用SCAN。升级到Oracle Clusterware 12c第1版（12.1）或更高版本时，SCAN可用，您应该使用SCAN连接到Oracle Database 11g第2版或更高版本的数据库。升级早期版本的Oracle数据库时，它会向SCAN侦听器注册，客户端可以开始使用SCAN连接到该数据库。数据库通过init.ora文件中的远程侦听器参数向SCAN侦听器注册。REMOTE_LISTENER参数必须设置为SCAN:PORT 。不要将其设置为具有SCAN的单个地址的TNSNAMES别名，例如，使用HOST= SCAN_name 。

对于大多数部署，SCAN是可选的。但是，使用服务器池的Oracle Database 11g第2版及更高版本的策略管理数据库的客户端必须使用SCAN访问数据库。这是必需的，因为策略管理的数据库可以在不同的时间在不同的服务器上运行，因此无法使用策略管理的数据库的虚拟IP地址连接到特定节点。

为客户端访问群集提供SCAN地址。如果使用DNS，则必须将这些地址配置为域名服务（DNS）上的循环地址。Oracle建议您提供三个SCAN地址。

识别公共和私有接口。Oracle Universal Installer配置公共接口以供公共和虚拟IP地址使用，并在专用接口上配置专用IP地址。专用接口使用的私有子网必须连接您打算拥有的所有节点作为集群成员。SCAN必须与公共接口位于同一子网中。

跨平台传输数据

您可以跨平台传输数据。

跨平台传输数据的功能可用于：

  • 使数据库可以从一个平台迁移到另一个平台。

  • 为内容提供商提供更简单，更有效的方法来发布结构化数据并将其分发给在不同平台上运行Oracle数据库的客户。

  • 简化从数据仓库环境到数据集市的数据分发，数据集市通常在较小的平台上运行。

  • 假设您可以从这些平台访问存储系统并且平台都具有相同的字节顺序，从而在不同操作系统或平台上的Oracle数据库安装之间共享只读表空间，如以下各节所述。

跨平台数据传输支持许多（但不是全部）平台。您可以查询V$TRANSPORTABLE_PLATFORM视图以查看支持的平台，并确定每个平台的endian格式（字节顺序）。以下查询显示支持跨平台数据传输的平台：

如果源平台和目标平台具有相同的字节顺序，则数据将从源平台传输到目标平台，而无需任何数据转换。

  • DBMS_FILE_TRANSFER包中的GET_FILE或PUT_FILE过程

    当您使用这些过程之一在源平台和目标平台之间移动数据文件时，每个数据文件中的每个块都将转换为目标平台的字节顺序。转换发生在目标平台上。

  • RMAN CONVERT命令

    在源或目标平台上运行RMAN CONVERT命令。此命令将传输的数据转换为目标平台格式。

    具有还原段的数据文件不支持在不同的endian格式之间转换数据文件。
