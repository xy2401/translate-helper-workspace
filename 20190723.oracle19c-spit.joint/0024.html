关于Oracle Base Homes

在只读ORACLE_HOME和读/写ORACLE_HOME中，用户特定文件，特定于实例的文件和日志文件都位于称为ORACLE_BASE_HOME的位置。

在读/写ORACLE_HOME中，ORACLE_BASE_HOME路径与ORACLE_HOME目录相同。但是，在只读ORACLE_HOME中，ORACLE_BASE_HOME目录与ORACLE_HOME不在同一位置，而是位于ORACLE_BASE / homes / HOME_NAME。

其中，HOME_NAME是ORACLE_HOME的内部名称。

例如，网络目录network/admin ， network/trace和network/log位于ORACLE_BASE_HOME目录中。在读/写ORACLE_HOME中，网络目录似乎位于ORACLE_HOME中，因为ORACLE_BASE_HOME与ORACLE_HOME位于同一位置，而在只读ORACLE_HOME中，网络目录位于ORACLE_BASE / homes / HOME_NAME中。

要打印ORACLE_BASE_HOME路径，请从$ORACLE_HOME/bin目录运行orabasehome命令：
$ setenv ORACLE_HOME /u01/app/oracle/product/19.0.0/dbhome_1 $ cd $ ORACLE_HOME / bin $ ./orabasehome

例如：

$ ./orabasehome $ / u01 / app / oracle / homes / OraDB19Home1

其中， /u01/app/oracle是ORACLE_BASE， OraDB19Home1是HOME_NAME

配置活动数据库复制的通道

使用活动数据库复制，您无需更改源数据库通道配置或配置辅助通道。但是，您可能希望增加源数据库磁盘通道的并行度设置，以便RMAN并行通过网络复制文件。

使用的活动数据库复制技术的类型确定哪些通道执行复制的主要工作。使用映像副本执行活动数据库复制时，主要工作由目标通道执行。在源数据库上配置多个目标通道以提高复制性能。当使用备份集执行活动数据库复制时，复制的主要工作由辅助通道执行。因此，建议您分配其他辅助通道。辅助信道的数量必须大于或等于目标信道的数量。使用备份集进行主动复制还可以实现并行性，从而提高复制过程的速度。

9.2对象比较的性能

您可以通过调用map或order方法来比较对象。

map方法将对象转换为标量值，同时保留对象的顺序。使用map方法是首选，因为它允许系统有效地对对象进行排序。

注意：

对于任何一种对象类型，您可以实现map或order方法，但不能同时实现两者。两者都不是必需的。

在使用ORDER BY或GROUP BY进程对对象进行排序时，对象的映射方式会产生重大的性能影响。可能需要将对象多次与其他对象进行比较，如果可以首先将对象映射到标量值（map方法），则效率会高得多。如果比较语义非常复杂，或者无法将对象映射到标量值以进行比较，则可以定义一个order方法，给定两个对象，返回由对象实现者确定的排序。Order方法不如map方法有效，因此如果使用order方法，性能可能会受到影响。

考虑一个由四个字符属性组成的对象类型address ： street ， city ， state和zipcode 。这里，最有效的比较方法是map方法，因为每个对象都可以很容易地转换为标量值。例如，您可以定义一个按状态排序所有对象的map方法。

另一方面，假设您想要比较二进制对象，例如图像。在这种情况下，比较语义可能太复杂而无法使用map方法;如果是这样，您可以使用订单方法进行比较。例如，您可以创建一个订单方法，根据亮度或每个图像中的像素数比较图像。

如果对象类型没有map或order方法，则只允许在该类型的对象上进行相等比较。在这种情况下，Oracle通过按照定义的顺序对相应的对象属性进行逐个字段的比较来执行比较。如果比较在任何时候失败，则返回FALSE值。如果比较在每个点匹配，则返回TRUE值。但是，如果对象具有LOB或ANYDATA属性，则Oracle不会逐个字段地比较该对象。此类对象必须具有映射或顺序方法才能执行比较。

$ setenv ORACLE_HOME /u01/app/oracle/product/19.0.0/dbhome_1 $ cd $ ORACLE_HOME / bin $ ./orabasehome

通用服务器池存储任何非策略管理的Oracle数据库。

此外，通用服务器池包含在服务器池的SERVER_NAMES属性中指定的名称的服务器，这些服务器池将通用服务器池列为父服务器池。

通用服务器池的属性受到限制，如下所示：

  • 没有人可以修改通用服务器池的配置属性（所有属性都是只读的）

  • 当DBCA或SRVCTL在HOSTING_MEMBERS资源属性中指定服务器名称时，Oracle Clusterware仅在服务器为：

    • 在线并存在于通用服务器池中

关于私有IP地址

Oracle Clusterware使用标记为私有的接口进行节点间通信。

每个群集节点必须具有您在安装期间识别为私有接口的接口。专用接口必须具有为接口本身配置的地址，但不需要其他配置。Oracle Clusterware使用您标识为群集互连专用的接口。如果在专用网络的信息期间识别多个接口，则Oracle Clusterware会使用冗余互连使用情况对其进行配置。您标识为私有的任何接口都必须位于连接到群集的每个节点的子网上。Oracle Clusterware使用您标识的所有接口作为专用接口。

对于专用互连，由于Cache Fusion和节点之间的其他流量，Oracle强烈建议使用物理上独立的专用网络。如果使用DNS配置地址，则应确保只能由群集节点访问专用IP地址。

您可以使用oifcfg setif命令在安装或安装后安装期间选择多个互连。

安装后，如果使用CLUSTER_INTERCONNECTS初始化参数修改Oracle Real Application Clusters（Oracle RAC）的互连，则必须将互连更改为未与公共IP地址一起使用的子网上的专用IP地址，作为oifcfg的公共子网。Oracle不支持使用您指定为公有子网的子网将互连更改为接口。

您不应在网络上使用专用网络IP地址的防火墙，因为这可能会阻止互连流量。

11.3使用UCP的中间层路由

从Oracle Database Release 18c开始，Oracle通用连接池（UCP）引入了中间层路由功能。此功能可帮助使用Sharding功能的Oracle客户拥有从客户端应用程序到分片数据库的专用中间层。

通常，中间层连接池将数据库请求路由到特定分片。在此类路由期间，每个中间层连接池都会建立与每个分片的连接，从而创建与数据库的过多连接。中间层路由功能通过为每个数据中心或云提供专用中间层（Web服务器或应用程序服务器）以及将客户端请求直接路由到相关中间层（其中包含客户端数据的分片）来解决此问题（对应于客户端分片键）驻留。

UCP中的OracleShardRoutingCache类提供了中间层路由API，可用于将客户端请求路由到适当的中间层。此类的实例表示UCP的内部分片路由缓存，可以通过提供连接属性（如分片目录的用户，密码和URL）来创建。从Oracle Database Release 19c开始，您还必须指定新的连接属性serviceName 。它是全局服务名称的名称。

路由缓存连接到分片目录以检索分片映射拓扑的密钥并将其存储在其缓存中。getShardInfoForKey(shardKey, superShardKey)方法使用UCP的路由缓存来获取有关指定分片键的分片的信息。ShardInfo实例封装了一个唯一的分片名称和分片的优先级。使用中间层API的应用程序可以将返回的唯一分片名称值映射到具有与特定分片的连接的中间层。

通过订阅相应的ONS事件，在块移动或拆分时自动刷新或更新路由缓存。

关于只读Oracle Homes

从Oracle Database 18c开始，您可以以只读模式配置Oracle主目录。

在只读Oracle主目录中，所有配置数据和日志文件都位于只读Oracle主目录之外。此功能允许您将只读Oracle主目录用作可以跨多个服务器分布的软件映像。

除了传统的ORACLE_BASE和ORACLE_HOME目录之外，以下目录包含以前在ORACLE_HOME中的文件：

  • ORACLE_BASE_HOME

  • ORACLE_BASE_CONFIG

只读Oracle主目录的优点

  • 无需延长停机时间即可无缝修补和更新Oracle数据库。

  • 简化修补和批量部署，因为只需更新一个映像即可将补丁分发到许多服务器。

  • 通过实施安装和配置分离来简化配置。

注意：

此功能不会影响数据库管理员监视，诊断和调整系统性能的方式。

使用包含或排除列表进行PDB升级

如果要升级早期版本PDB的子集，请使用包含或排除列表以避免重新升级处于新版本级别的CDB或PDB。

Oracle建议您记录升级的容器，并使用包含或排除列表将这些容器从连续的批量升级中排除。从连续批量升级中排除升级容器可确保升级仅在需要升级的PDB上运行。避免重新升级可以最大限度地缩短整体升级时间，并避免不必要的不可用性。

例如：如果您使用多租户架构部署安装了Oracle数据库，那么容器CDB$ROOT ， PDB$SEED以及创建CDB时创建的任何其他PDB都是新版本多租户架构的一部分。如果升级了CDB，同时将一组PDB升级到新版本，则无需再次升级CDB容器或升级后的PDB。

在任何一种情况下，当您插入早期版本的PDB然后升级它们时，请使用排除列表或包含列表升级PDB：

  • 使用包含列表仅指定要升级的PDB集。

  • 使用排除列表排除已升级的CDB和PDB容器。

如果不使用包含列表或排除列表来限制升级范围，则并行升级实用程序（ catctl.pl ）会尝试升级整个CDB，而不仅仅是需要升级的PDB。在升级过程中，您的系统会遇到不必要的停机时间。包含列表和排除列表选项是互斥的。
