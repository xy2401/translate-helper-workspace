关于Oracle ASM文件名

在Oracle ASM中创建的每个文件都提供由Oracle Managed Files生成的文件名，称为完全限定文件名（或系统生成的文件名）。完全限定的文件名表示Oracle ASM文件系统中的完整路径名。完全限定文件名的示例是：

+数据/ ORCL /控制文件/ Current.256.541956473

您可以使用完全限定的文件名来引用（读取或检索）Oracle ASM文件。您还可以使用其他缩写文件名格式（例如别名Oracle ASM文件名）来引用Oracle ASM文件。

在任何创建文件的请求时，Oracle ASM都会生成完全限定的文件名。创建请求不会或不能指定完全限定的文件名。相反，它使用更简单的语法来指定文件，例如别名或磁盘组名称。然后，Oracle ASM根据文件类型创建文件，将其放在正确的Oracle ASM路径中，然后分配适当的完全限定文件名。如果在创建请求中指定别名，Oracle ASM还会创建别名，以便它引用完全限定的文件名。

Oracle ASM文件创建请求可以是单个文件创建请求，也可以是多个文件创建请求。

注意：

  • 您可以在显示Oracle文件名的数据库视图中找到生成的名称，例如V$DATAFILE和V$LOGFILE 。如果以后必须在SQL语句中引用Oracle ASM文件，则可以使用此名称或其缩写形式。与其他Oracle数据库文件名一样，Oracle ASM文件名保存在控制文件和RMAN目录中。

  • 如果指定REUSE关键字，则可以在单文件创建中使用完全限定和数字文件名，如在SQL语句中使用Oracle ASM文件名中所述 。

  • 正斜杠（/）和反斜杠（\）在文件名中是可互换的。文件名不区分大小写，但保留大小写。

单个文件创建表单

单个文件创建表单用于创建单个文件，例如数据文件或控制文件。此类型中的Oracle ASM文件名的格式是别名，例如+data/orcl/ctl.f ，或者是不完整的文件名，例如以加号（ +data ）开头的磁盘组名称。您可以使用在SQL语句中调用文件名的别名或磁盘组名称，例如CREATE TABLESPACE ， CREATE CONTROLFILE 。

多个文件创建表单

多次使用多文件创建表单来创建Oracle ASM文件，例如在为初始化参数DB_CREATE_FILE_DEST分配值时。您还可以多次运行CREATE TABLESPACE语句（没有文件名规范）。每次，Oracle ASM都会创建不同的唯一数据文件名。

使用多文件创建表单时，可以使用不完整的文件名表单，例如以加号（ +data ）开头的磁盘组名称。在这种情况下，您将DB_CREATE_FILE_DEST设置为不完整的文件名（例如， +data ），并且每当运行必须在DB_CREATE_FILE_DEST创建数据库文件的DB_CREATE_FILE_DEST ，将在指定的磁盘组中创建该文件并DB_CREATE_FILE_DEST分配唯一的完全限定名称。您可以在其他*_DEST初始化参数中使用不完整的文件名。

完全合格的文件名表

除了创建磁盘组之外，您可以使用完全限定的表单来引用Oracle ASM操作中的现有Oracle ASM文件。此表单是Oracle ASM文件名，Oracle ASM在创建Oracle ASM文件时自动生成该文件名。

完全限定的文件名在非多租户环境中具有以下形式：

+ diskgroup / dbname / filetype / filetypetag.file.incarnation

对于多租户数据库，完全限定的文件名具有以下形式：

+ diskgroup / cdb / pdb_guid / filetype / filetypetag.file.incarnation

完全限定文件名的字段是：

  • + diskgroup是以加号diskgroup的磁盘组名称。

    您可以将加号（ + ）视为Oracle ASM文件系统的根目录，类似于UNIX或Linux计算机上的斜杠（ / ）。

  • dbname是文件所属的数据库的DB_UNIQUE_NAME 。

  • filetype是Oracle文件类型，可以是表5-3中显示的文件类型之一。

  • filetypetag是有关文件的特定于类型的信息，可以是表5-3中显示的标记之一。

  • file.incarnation是文件/化身对，用于确保唯一性。

  • cdb是cdb的名称

  • pdb_guid是PDB GUID值。

别名Oracle ASM文件名表单

您可以使用Alias Oracle ASM文件名来引用现有Oracle ASM文件和创建新的Oracle ASM文件。别名以磁头组名称开头，后面加一个加号，然后指定名称字符串。别名文件名使用分层目录结构实现，斜杠（/）或反斜杠（\）字符分隔名称组件。您可以在Oracle Managed Files（由系统生成）生成或由用户创建的任何Oracle ASM目录中创建别名。别名必须包含磁盘组名称;您无法在根级别（ + ）创建别名。

使用别名文件名创建Oracle ASM文件时，将使用完全限定名创建文件，并另外创建别名文件名。然后，您可以使用任一名称访问该文件。

别名Oracle ASM文件名与完全限定文件名或数字文件名不同，因为它们不以点对数字结尾。尝试创建以点对数字结尾的别名是错误的，例如格式为USERS.259.685366091 。Oracle ASM别名文件名的示例如下：

+ data / orcl / control_file1 + fra / recover / second.dbf

Oracle数据库按别名文件名引用数据库文件，但前提是您使用别名创建数据库文件。如果您创建没有别名的数据库文件，然后再添加别名，则数据库将通过其完全限定的文件名引用这些文件。以下是数据库如何使用别名文件名的示例：

  • 别名文件名出现在V $视图中。例如，如果创建表空间并使用数据文件的别名文件名，则V$DATAFILE视图将显示别名文件名。

  • 当控制文件指向数据文件和联机重做日志文件时，它可以使用别名文件名。

  • CONTROL_FILES初始化参数可以使用控制文件的别名文件名。Database Configuration Assistant（DBCA）使用别名文件名创建控制文件。

注意：

使用别名文件名创建的文件不被视为Oracle托管文件，如果不再需要，将来可能需要手动删除。

有关别名文件名的详细信息，请参阅管理Oracle ASM文件名的别名 。

管理磁盘组目录

Oracle ASM磁盘组包含系统生成的分层目录结构，用于存储Oracle ASM文件。Oracle ASM分配给每个文件的系统生成的文件名表示此目录层次结构中的路径。以下是系统生成的文件名的示例：

+数据/ ORCL / CONTROLFILE / Current.256.541956473

加号表示Oracle ASM文件系统的根。data目录是data磁盘组中所有文件的父目录。orcl目录是orcl数据库中所有文件的父目录， CONTROLFILE目录包含orcl数据库的所有控制文件。

您可以在此层次结构中创建自己的目录，以存储您创建的别名。因此，除了具有用于Oracle ASM文件的用户友好别名之外，您还可以拥有指向这些名称的用户友好路径。

  • 重命名目录

  • 删除目录

使用ALTER DISKGROUP语句的ADD DIRECTORY子句为Oracle ASM文件的别名创建分层目录结构。使用斜杠（/）或反斜杠（\）字符分隔目录路径的组件。目录路径必须以磁盘组名称开头，前面带有加号（+），后跟您选择的所有子目录名称。

在尝试在该目录中创建子目录或别名之前，父目录必须存在。必须在磁盘组级别下创建目录。

。虚拟文件夹的存储库路径是/sys/asm 。该文件夹是虚拟的，因为其内容实际上并不存在于存储库中;它们作为普通的Oracle ASM文件和目录存在。例如， /sys/asm提供了一种通过编程API（如DBMS_XDB包）和XML DB协议（如FTP和HTTP / WebDAV）访问和操作Oracle ASM文件和目录的方法。您还可以使用ASMCMD cp命令复制文件。

此功能的典型用法可能是在图形用户界面（使用WebDAV协议）中将/sys/asm视为Web文件夹，然后将Data Pump转储集从Oracle ASM磁盘组复制到操作系统文件系统拖放。

您必须以SYS以外的用户身份登录，并且必须已获得DBA角色才能使用XML DB协议访问/sys/asm 。

注意：

出于安全原因，默认情况下禁用FTP。这是因为IETF FTP协议规范要求以明文形式传输密码。通过将FTP服务器端口配置为零（0）来完成禁用。要启用FTP，请将ftp-port参数设置为要使用的FTP端口，例如2100 。

以下是/sys/asm使用限制：

  • 您无法使用API（如DBMS_XDB.LINK创建指向现有Oracle ASM文件或目录的硬链接。

  • 您无法将Oracle ASM文件重命名（移动）到另一个磁盘组或Oracle ASM外部的目录。

您可以使用目录/sys/asm来存储磁盘组的名称。您无法在此目录中存储其他文件。在/sys/asm下的磁盘组目录中，例如/sys/asm/DATA ，您只能在这些子目录中存储数据库文件。Oracle ASM拒绝在这些目录中存储非数据库文件的尝试。

管理磁盘组模板

模板用于设置在Oracle ASM磁盘组中创建的文件的属性。

创建文件时，将根据显式命名的模板或作为文件类型的默认模板的系统模板为该文件设置冗余和条带化属性。

创建磁盘组时，Oracle ASM会为该磁盘组创建一组默认模板。该集由Oracle ASM支持的每种文件类型（数据文件，控制文件，重做日志文件等）的一个模板组成。例如，名为ONLINELOG的模板为写入Oracle ASM磁盘的所有重做日志文件提供默认文件冗余和条带化属性。默认模板设置取决于磁盘组类型。普通冗余磁盘组的数据文件的默认模板设置双向镜像，而高冗余磁盘组中的相应默认模板设置三向镜像。您可以修改这些默认模板。

例如，正常冗余磁盘组的联机重做日志文件（ ONLINELOG模板）的默认冗余是MIRROR 。在示例4-1中 ，此设置意味着当重做日志文件范围的一个副本写入故障组controller1的磁盘时，文件范围的镜像副本将写入故障组controller2的磁盘。要支持普通冗余磁盘组的默认镜像，必须至少定义两个故障组。

A.1使用OLAP_TABLE创建关系视图

OLAP_TABLE是一个SQL函数，它返回一个对象表，这些对象可以连接到关系表和视图，以及由OLAP_TABLE填充的其他对象表。在SQL语句中，您可以在提供表或视图名称的任何位置指定OLAP_TABLE函数调用。

OLAP_TABLE 使用限制映射将分析工作区中定义的维度和度量映射到逻辑表中的列。限制映射与SQL SELECT语句的WHERE子句组合，以生成在分析工作区中执行的一系列OLAP DML LIMIT命令。

OLAP_TABLE可以使用带有预定义逻辑表的限制映射，也可以使用限制映射中的信息在运行时动态生成逻辑表。

A.1.1必需的OLAP DML对象

必须在分析工作区中预定义多个对象，以支持在限制图中映射维层次结构：

  • 一个inhierarchy变量或valueset，它指定哪些维度成员属于层次结构的每个级别。有关详细信息，请参阅“ Inhier值集或变量 ” 。

  • 分组ID变量，用于标识每个维度成员的层次结构中的深度。有关更多信息，请参阅“ Gidrel关系 ” 。

  • 族关系 ，它提供层次结构中每个维度成员的完整父级。有关更多信息，请参阅“ Familyrel关系 ” 。

A.1.2创建供OLAP_TABLE使用的逻辑表

由OLAP_TABLE填充的逻辑表实际上是一种表类型，其行是用户定义的对象类型，也称为抽象数据类型或ADT 。

一个用户定义的对象类型由属性组成，这些属性等同于表的列。定义行的基本语法如下。 

CREATE TYPE object_name AS OBJECT（ attribute1 数据类型 ， attribute2 数据类型 ， 属性 n 数据类型 ）;

一个表类型是对象类型的集合;此集合相当于表的行。创建表类型的基本语法如下。 

CREATE TYPE table_name AS TABLE OF object_name ;

OLAP_TABLE可以使用带有预定义逻辑表的限制映射，也可以使用限制映射中的信息在运行时动态生成逻辑表。

A.2.1将OLAP DML表达式用作单行函数

SQL函数通常是单行的为查询的表或视图的每一行返回单个结果行的函数。Oracle支持多种预定义的SQL 单行函数，例如返回数字数据的COS ， LOG和ROUND ，以及返回字符数据的UPPER和LOWER 。

在SELECT FROM OLAP_TABLE语句的上下文中，有四个SQL函数可用于包装OLAP DML函数，以便OLAP DML函数充当单行函数。有四个返回不同的OLAP DML数据类型的表达式的结果： OLAP_EXPRESSION为Oracle OLAP数字表达式， OLAP_EXPRESSION_BOOL为Oracle OLAP布尔表达式， OLAP_EXPRESSION_DATE为Oracle OLAP日期时间表达式，并OLAP_EXPRESSION_TEXT为Oracle OLAP的文本表述。每个SQL函数的一个参数是OLAP DML函数。

您可以像指定其他Oracle单行函数一样指定OLAP_EXPRESSION函数及其变体，特别是在选择列表， WHERE和ORDER BY子句中。

A.2.2在SELECT FROM OLAP_TABLE语句中修改分析工作区

在执行OLAP_TABLE期间，有几种机制可以动态修改分析工作区。

您可以使用OLAP_CONDITION SQL函数在SELECT FROM OLAP_TABLE语句的上下文中修改分析工作区。您可以像其他Oracle函数一样指定OLAP_CONDITION ，通常在WHERE子句中。使用OLAP_CONDITION ，您可以设置选项，执行LIMIT命令，执行OLAP模型或预测，或运行程序。对工作空间所做的更改可能是暂时的，也可能在查询完成后持续存在于会话中。

除了OLAP_CONDITION ，您还可以使用OLAP_TABLE函数本身支持的语法：限制映射中的PREDMLCMD和POSTDMLCMD子句以及olap_command参数。OLAP_CONDITION具有可移植性的优点，因为它不嵌入在OLAP_TABLE ，并且具有多功能性，因为它可以应用于不同的入口点。

OLAP_TABLE在执行生成查询结果集的LIMIT命令之前，将维度的状态保存在限制映射中。获取数据后， OLAP_TABLE将恢复维度的状态。您可以在限制映射中指定PREDMLCMD子句，以便在保存维度状态之前执行OLAP DML命令。执行OLAP_TABLE后， PREDMLCMD子句产生的修改仍保留在工作空间中，除非使用POSTDMLCMD子句进行反转。有关更多信息，请参阅“limit_map” 。

OLAP_TABLE的olap_command参数指定在获取结果集之前立即执行的OLAP DML命令。在某些情况下， olap_command参数可能包含OLAP DML FETCH命令，该命令本身管理提取。olap_command参数设置的限制仅在执行OLAP_TABLE期间有效。有关更多信息，请参阅“olap_command” 。

      在单个dense_list中，您无法以多种方式指定维度。换句话说，您不能为同一维度指定维度和值集;您不能为同一维度指定两个值集。

      但是，您可以为外部DENSE短语中的维度设置一个值集，为内部DENSE短语中的相同维度设置另一个值集。如果在内部和外部DENSE短语中都指定了维度的维度或值集，则Oracle OLAP会在密集区域的并集上密集循环。如果维度本身出现在任何一个位置，则Oracle OLAP会在整个维度上密集循环。

    • aggmap是aggmap对象的名称。当您仅指定aggmap对象的名称时，Oracle OLAP使用aggmap的PRECOMPUTE短语中的值来标识要循环的值

    • variable是aggmap_name聚合的变量的名称。

    • dimension_list是一个或多个合成，分区模板或维度的列表。

  • PREDMLCMD条款

    PREDMLCMD条款指定在将数据从分析工作空间提取到目标表之前执行的OLAP DML命令。例如，它可用于执行OLAP模型或预测，其结果将被提取到表中。该命令的结果在执行限制映射期间生效，并在OLAP_TABLE执行完成后继续进入您的会话。请参阅“ OLAP_TABLE中的处理顺序 ” 。

  • POSTDMLCMD条款

    POSTDMLCMD子句指定在将数据从分析工作空间提取到目标表之后执行的OLAP DML命令。例如，它可用于删除PREDMLCMD子句中的命令创建的对象或数据，或者用于恢复PREDMLCMD子句中更改的维度状态。请参阅“ OLAP_TABLE中的处理顺序 ” 。

OLAP_TABLE中的处理顺序

该以下列表标识OLAP_TABLE函数处理限制映射中可以更改分析工作空间中维度状态的指令的顺序。

  1. 执行在限制映射的PREDMLCMD参数中指定的任何OLAP DML命令。

  2. 将所有维度的当前状态保存在限制图中，以便以后可以恢复（ PUSH状态）。

  3. 仅保留状态仅限于由限制映射（ LIMIT KEEP ）中的INHIERARCHY子条款指定的维度成员。

  4. 在步骤3中设置的状态内，仅保留满足包含OLAP_TABLE函数（ LIMIT KEEP ）的SQL SELECT语句的WHERE子句的维度成员。

  5. 执行OLAP_TABLE函数的olap_command参数中指定的任何OLAP DML命令。（如果olap_command包含FETCH ，则获取数据。）

  6. 获取数据（除非在olap_command参数中指定了OLAP DML FETCH命令）。

  7. 恢复限制图中所有维的状态（ POP状态）。

  8. 执行在限制映射的POSTDMLCMD参数中指定的任何OLAP DML命令。

要在当前数据库实例中查找现有规则集，请查询DBA_DV_RULE_SET视图中描述的DBA_DV_RULE_SET视图 。

启用

指定以下选项之一以设置命令规则的状态：

  • DBMS_MACUTL.G_YES或'y'启用命令规则（默认）

  • DBMS_MACUTL.G_NO或'n'禁用命令规则，包括捕获模拟日志中的违规

  • DBMS_MACUTL.G_SIMULATION或's'使SQL语句能够执行，但在模拟日志中捕获违规

EVENT_NAME

命令规则定义的事件。此设置使命令规则对应于ALTER SYSTEM SET EVENTS event_name语句。例如，要跟踪跟踪事件，可以将event_name设置为TRACE 。

COMPONENT_NAME

event_name设置的一个组件。示例设置为DV ， OLS或GCS 。

您可以通过以用户SYS发布ORADEBUG DOC COMPONENT RDBMS来查找有效的组件名称。输出显示父组件和子组件，您可以将其用于component_name设置。例如， XS （父级）和XSSESSION （ XS子级）都是有效的组件名称。如果选择父组件，则命令规则将应用于它和子组件。

ACTION_NAME

component_name设置的操作

范围

对于多租户环境，确定如何执行此过程。默认为本地。选项如下：

  • DBMS_MACUTL.G_SCOPE_LOCAL （或1 ）如果命令规则在当前PDB中是本地的

  • DBMS_MACUTL.G_SCOPE_COMMON （或2 ）如果命令规则在应用程序根目录中

如果更新应用程序根目录中的公共命令规则并希望它对关联的PDB可见，则必须同步应用程序。例如：

ALTER PLUGGABLE DATABASE APPLICATION saas_sales_app SYNC;

pl_sql_stack

启用模拟模式时，指定是否记录失败操作的PL / SQL堆栈。输入TRUE以记录PL / SQL堆栈，输入FALSE以不记录。

例

以下示例说明如何在多租户环境中更新公共系统事件命令规则。此命令规则位于应用程序根目录中，因此运行此过程的用户必须位于CDB根目录中。与此命令规则关联的任何用户名或规则集必须是通用的。

BEGIN DBMS_MACADM.UPDATE_SYSTEM_EVENT_CMD_RULE（rule_set_name =>'Disabled'，event_name =>'TRACE'，component_name =>'DV'，enabled =>'n'，scope => DBMS_MACUTL.G_SCOPE_COMMON）;结束; /
