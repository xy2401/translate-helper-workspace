14.1关于显式语义分析

在Oracle数据库12c第2版中，引入了显式语义分析（ESA）作为Oracle数据挖掘用于特征提取的无监督算法。从Oracle Database 18c开始，ESA作为分类的监督算法得到了增强。

作为特征提取算法，ESA不会发现潜在特征，而是使用现有知识库中表示的显式特征。作为特征提取算法，ESA主要用于计算文本文档的语义相似度和显式主题建模。作为分类算法，ESA主要用于对文本文档进行分类。ESA的特征提取和分类版本也可以应用于数字和分类输入数据。

ESA的输入是一组属性向量。每个属性向量都与一个概念相关联。该概念是特征提取的特征，或者是分类的目标类。对于特征提取 ，只有一个属性向量可以与任何特征相关联。对于分类 ，训练集可以包含与任何给定目标类相关联的多个属性向量。与一个目标类相关的这些行通过ESA算法聚合为一个。

ESA的输出是稀疏属性概念矩阵，其包含最重要的属性 - 概念关联。通过每个属性 - 概念对的权重值来捕获关联的强度。属性概念矩阵存储为反向索引，列出每个属性的最重要概念。

注意：

对于特征提取 ，ESA算法不会投影原始要素空间，也不会降低其维数。ESA算法使用有限或无信息的属性集过滤掉特征。

的范围 ESA处理的分类任务与分类算法（如朴素贝叶斯和支持向量机）不同 。ESA可以执行大规模分类 ，其中不同类的数量高达数十万。大规模分类需要巨大的训练数据集，其中一些类具有大量训练样本，而其他类在训练数据集中稀疏地表示。

14.1.1用ESA评分

学习使用显式语义分析（ESA）进行评分。

ESA的典型特征提取应用程序是识别给定输入的最相关特征并对其相关性进行评分。评分ESA模型会在概念特征空间中生成数据投影。如果ESA模型是从任意文档集合构建的，则每个文档都被视为一个特征。然后很容易识别集合中最相关的文档。特征提取功能包括： FEATURE_DETAILS ， FEATURE_ID ， FEATURE_SET ， FEATURE_VALUE和FEATURE_COMPARE 。

ESA的典型分类应用是预测给定文档的类别并估计预测的概率。作为分类算法，ESA实现以下评分函数： PREDICTION ， PREDICTION_PROBABILITY ， PREDICTION_SET ， PREDICTION_DETAILS ， PREDICTION_COST 。

14.3 ESA的数据准备

自动数据准备将输入向量标准化为显式语义分析（ESA）的单位长度。

当具有简单数据类型（未嵌套）的列中存在缺失值时，ESA将使用模式替换缺少的分类值，并使用均值替换缺少的数值。当嵌套列中缺少值时，ESA会将它们解释为稀疏。该算法用零和用零向量的稀疏分类数据替换稀疏数值数据。Oracle Data Mining数据准备将输入文本转换为实数向量。这些数字代表了文本中各个单词的重要性。

14.4显式语义分析中的术语

讨论与显式语义分析（ESA）相关的术语。

多目标分类

这些大规模分类中的培训项目属于几个类别。在这种情况下分类的目标是检测一个项目的可能的多个目标类别。这种分类称为多目标分类。扩展了基于ESA的分类的目标列。允许集合作为目标列值。基于ESA的分类中目标的集合类型为ORA_MINING_VARCHAR2_NT 。

大规模分类

大规模分类适用于包含大量类别的本体，通常范围为数十或数十万。这种大规模分类还需要通常不平衡的巨大训练数据集，即，一些类可能具有大量训练样本，而其他类可能在训练数据集中稀疏地表示。大规模分类通常会导致给定测试用例的多个目标类分配。

主题建模

主题建模是指文档最重要主题的派生。主题建模可以是明确的或潜在的。显式主题建模导致从给定文档的预定义集合中选择最相关的主题。明确的主题有名称，可以用语言表达。潜在主题建模识别一组文档集合的潜在主题。这些潜在主题的子集与正在检查的每个文档相关联。潜在主题没有口头描述或有意义的解释。

可以在借用连接时使用池属性验证连接，也可以使用ValidConnection接口以编程方式ValidConnection连接。本节详细介绍了这两种方法。无效连接可能会影响应用程序性能和可用性。

3.3.1借用时的验证

当从连接池借用连接时，可以通过在连接上执行SQL语句来验证连接。两个连接池属性结合使用以启用连接验证：

  • setValidateConnectionOnBorrow(boolean):指定从连接池借用连接时是否验证连接。该方法可以验证从池中借用的每个连接。值false表示不执行验证。默认值为false 。

  • setSQLForValidateConnection(String):指定从池中借用连接时在连接上执行的SQL语句。

注意：

3.3.2使用setSecondsToTrustIdleConnection（）方法最小化连接验证

3.3.3检查连接是否有效

oracle.ucp.jdbc.ValidConnection接口提供了两种方法： isValid和setInvalid 。isValid方法返回连接是否可用，并且setInvalid方法用于指示应从池实例中删除连接。

注意：

12为DRDA使用Oracle数据库提供程序的限制

在最初为IBM DB2设计的Oracle数据库应用程序中进行自定义或维护时，可能会使用一些限制和已知的解决方法。

重组经理

适用于DRDA的Oracle数据库提供程序支持分布式工作单元的Sync Point Manager服务。它还支持Resynchronization Manager服务，以便在使用没有日志的源Sync Point Manager的迁移期间进行重新同步。

此版本的Oracle数据库提供程序for DRDA不支持活动的不确定事务解析服务。已迁移且不确定的事务需要在客户端系统和Oracle数据库之间进行手动解析。有关手动解决不确定事务的信息，请参见“ Oracle®数据库管理员指南” 。

游标HOLD属性语义

标有HOLD属性的游标具有以下限制：

  1. 在远程工作单元（RUOW）下，使用FOR UPDATE子句准备的游标在COMMIT或ROLLBACK上隐式关闭。

  2. 在Distribute Unit of Work（DUOW）下，如果对包含COMMIT或ROLLBACK上的打开游标的服务器进行任何更新，则会隐式关闭所有游标。

DB2密码空白填充

当密码被加密并通过DRDA发送时，DB2 for z / OS会将空格插入到少于8个字符的密码中。这导致登录失败，错误ORA-01017 。Oracle建议用户帐户密码长度至少为8字符。

数据类型的限制

使用数据类型有几个限制。

DATE数据类型

Oracle DATE数据类型包含DRDA DATE数据类型不支持的时间组件。如果DATE数据包含时间组件，则对Oracle DATE数据进行操作可能无法产生预期结果。为了保持一致性，在使用Oracle本机DATE语法插入DATE数据时不要存储时间组件。或者，将DATE列重新映射到TIMESTAMP 。

Oracle对象关系数据类型

此版本不支持对包含通过Object-Relational数据类型定义的列的对象的查询。

此版本不支持调用通过Object-Relational数据类型定义的SQL过程作为其输入或返回参数。

TIMESTAMP数据类型

用于DRDA的Oracle数据库提供程序表示TIMESTAMP ，其固定精度为6位小数。

出于兼容性原因，在使用TIMESTAMP数据时应特别小心，并且可能必须进行编程调整，例如类型转换。有关使用TIMESTAMP数据类型进行转换的信息，请参见“ Oracle®数据库SQL语言参考 ”。

TIMESTAMP与TIMEZONE数据类型

TIMESTAMP WITH TIMEZONE表示在Oracle数据库和DB2之间存在显着差异。

根据Oracle的演示规则，用于DRDA的Oracle数据库提供程序代表TIMESTAMP WITH TIMEZONE 。为了在客户端和服务器之间实现最佳兼容性，请使用四位时区后缀表示法而不是写入时区描述表示法，例如-08:00 。

XML数据类型

所述DRDA XML数据类型（ 988 ， 989 ）不支持如在此版本中的程序或绑定变量的数据类型。

SYS.XMLType数据类型

此版本不支持Oracle XML数据类型SYS.XMLType 。

扩展长度模式

最新版本的DB2 for z / OS（v11.1）不支持游标生成的查询结果的“流层B模式”协议。这些游标使用ROWSETs参数定义，该参数生成超过32767字节数据的行集。

游标声明的示例

EXEC SQL DECLARE DT游标，为SEL提供ROWSET定位;

根据DRDA标准，目标服务器可以确定任何给定查询的返回形式的查询数据。为此，流层B模式是最有效的形式。但是，当涉及ROWSET时，某些版本的DB2不支持这种情况，并且会导致将错误返回给DB2客户端应用程序：

DSNT4081 SQLCODE = -30020，错误：由于分配协议错误而导致的执行失败：原因124C（0100）

在这些条件下，DB2将不接受Streaming Layer B模式对象。但是，它将接受扩展长度模式对象。

您可以设置PROTOPROC_OPTIONS配置参数以启用此模式，直到DB2支持Streaming Layer B模式。

时间序列分析根据过去的历史记录提供对未来值的预测。例如，根据先前的销售顺序预测销售额。预测是企业和政府决策的重要组成部分。

新算法

  • 随机森林

    Random Forest是一款功能强大的机器学习算法。它使用一种集合方法，该方法结合了使用随机特征选择构建的多个树。实际上，单个树构建在随机子空间中并使用套袋集合方法组合。

    随机森林是一种非常流行的算法，它在许多基准测试中具有出色的性能。它是Oracle R Enterprise（ORE）的一部分，但实现基于公共R包。将其实现为内核代码可带来显着的性能和可伸缩性优势。

    见随机森林 。

  • 增强的显式语义分析机器学习算法支持分类

    显式语义分析（ESA）在Oracle Database 12 c第2版中公开，仅作为FEATURE_EXTRACTION下的主题模型。它通常使用数十万个显式功能。该算法可以很容易地适用于执行分类，以解决具有数十万类具有挑战性的分类问题的用例，这些问题未被当前的Oracle Advanced Analytics（OAA）算法正确解决。

    在大数据的背景下，大文本分类的任务非常重要。将ESA扩展到分类显着增强了我们在文本分类领域的产品，并允许OAA解决目前对产品难以处理的用例。

    请参阅显式语义分析 。

  • 神经网络

    神经网络算法是一种生物学启发的方法，其中一组互连单元（ 神经元 ）学习近似函数。神经网络适用于分类和回归问题中的非线性近似。

    神经网络是可以学习任意非线性函数的强大算法。已经成功地用于许多难题，包括非线性回归或时间序列，计算机视觉和语音识别。

    参见神经网络 。

  • 基于CUR分解的属性和行重要性算法

    CUR算法允许用户找到最能解释其数据的列和特征。该算法已经普及，因为它允许用户使用易于理解的术语来深入了解他们的数据。相反，像奇异值分解（SVD）这样的分解方法导出了难以解释的隐含特征。CUR尝试使用从SVD派生的见解，但是根据原始行和列进行翻译。

    基于CUR的属性和行重要性可用于提供数据洞察以及数据过滤，然后进行其他分析处理。这将是第一个Oracle Advanced Analytics（OAA）算法，它不仅可以列出重要的列，还可以列出重要的行。

    请参见CUR矩阵分解 。

  • 指数平滑

    指数平滑 （ESM）允许用户根据时间序列数据进行预测。指数平滑方法（ESM）广泛用于从时间序列数据预测。最初，被认为比竞争对手（如ARIMA）更不灵活和准确，ESM最近被证明可以涵盖更广泛的模型，并且已经扩展到增加其描述性的真实性和准确性。Oracle ESM包括许多最近的扩展，共有14个模型，包括流行的Holt（趋势）和Holt-Winters（趋势和季节性）模型，以及处理不规则时间序列间隔的能力。

    请参见指数平滑 。

29.6处理缺失值

Oracle Data Mining区分 稀疏数据和包含的数据 随机缺失值 。后者意味着某些属性值未知。疏另一方面，数据包含假设已知的值，尽管它们未在数据中表示。

稀疏数据的典型示例是市场篮子数据。在数百或数千种可用物品中，只有少数存在于个别情况（篮子或交易）中。所有项目值都是已知的，但它们并非都包含在购物篮中。当前值具有数量，而未表示的项目是稀疏的（已知数量为零）。

Oracle Data Mining按如下方式解释缺少的数据：

  • 随机丢失：假设在具有简单数据类型（未嵌套）的列中缺少值，则随机丢失。

  • 稀疏：嵌套列中缺少值表示稀疏性。

29.6.1示例：缺少值或稀疏数据？

本节中的示例说明了Oracle Data Mining如何将数据标识为稀疏或随机丢失。

29.6.1.1销售表中的稀疏性

销售表包含一组产品的销售点数据，这些产品在一段时间内在多个商店中销售给不同的客户。特定客户只购买少数产品。客户未购买的产品不会在销售表中显示为行。

如果您要计算出客户为每种产品花费的金额，那么未购买的产品的推断量为零。该值不是随机的或未知的;它为零，即使表中没有行。

请注意，销售数据的尺寸标注（按产品，商店，客户和时间），通常表示为挖掘的嵌套数据。

由于嵌套列中的缺失值始终表示稀疏性，因此必须确保此解释适用于要挖掘的数据。例如，当尝试从大型电影数据库的用户挖掘包含电影评级的多记录案例数据集时，缺失的评级是未知的（随机丢失），但Oracle数据挖掘将数据视为稀疏并推断评级为零缺失值。

29.6.1.2客户数据表中缺少值

客户数据表包含有关客户的人口统计数据。案例ID列是客户ID。属性包括年龄，教育，职业，性别，家庭大小等。并非所有数据都适用于每个客户。任何缺失值都被认为是随机丢失的。例如，如果数据中不存在客户1的年龄和客户2的职业，那么该信息就是未知的。它并不表示稀疏性。

请注意，客户数据未标注尺寸。案例与其每个属性之间存在一对一的映射。没有任何属性是嵌套的。

使用CREATE FLASHBACK ARCHIVE语句创建闪回数据存档，该存档提供自动跟踪和存档对指定数据库对象的事务数据更改的功能。闪回数据存档由多个表空间组成，并存储来自跟踪表的所有事务的历史数据。数据存储在内部历史表中。

闪回数据存档使用RETENTION参数保留指定持续时间的历史数据。可以使用闪回查询AS OF子句查询历史数据。已自动清除已超过指定保留期的已存档历史数据。

闪回数据存档保留跨数据定义语言（DDL）更改的历史数据，以便为闪回数据存档启用表。闪回数据存档支持许多常见的DDL语句，包括一些改变表定义或导致数据移动的DDL语句。不支持的DDL语句导致错误ORA-55610。

2.12 Oracle数据库中的线程概述

Oracle JVM基于数据库会话模型，该模型是单客户端，非抢占式线程模型。虽然Oracle数据库中的Java允许运行线程程序，但它在执行级别是单线程的。在此模型中，JVM在单个操作系统线程上运行与数据库会话关联的所有Java线程。一旦调度，线程继续执行，直到它通过调用Thread.yield(),显式产生，通过调用Thread.yield(),阻塞Socket.read(),或者被执行引擎抢占。一旦线程产生，阻塞或被抢占，JVM就会调度另一个线程。

注意：

从11g版本1（11.1）开始，Oracle JVM支持线程抢占。Java规范并未规定线程抢占，但需要正确支持JDK1.5中新的java.util.concurrent API。

Oracle JVM添加了以下功能以获得更好的性能和线程管理：

  • 系统调用至少是。Oracle JVM使用非系统解决方案交换了一些标准系统调用。例如，输入监视器同步的块或方法不需要系统调用。

  • 检测到死锁。

    • Oracle JVM监视线程之间的死锁。如果一个发生死锁，然后Oracle JVM终止其中一个线程并抛出 oracle.aurora.vm.DeadlockError异常。

    • 单线程应用程序无法挂起。如果应用程序只有一个线程并且您尝试暂停它，那么 oracle.aurora.vm.抛出LimboError异常。

2.12.1线程生命周期

在单线程应用程序中，当发生以下事件之一时，调用结束：

  • 线程返回其调用者。

  • 抛出异常并且不会在Java代码中捕获。

  • System.exit(), OracleRuntime.exitSession(),或oracle.aurora.vm.OracleRuntime.exitCall()方法。

  • DBMS_JAVA.endsession()或DBMS_JAVA.endsession_and_related_state()方法。

如果初始线程创建并启动其他Java线程，则调用以下列方式之一结束：

  • 主线程返回其调用者或抛出异常并且未在此线程中捕获，并且在任何一种情况下都处理所有其他非守护程序线程。非守护程序线程通过从其初始方法返回或因为抛出异常而未在线程中捕获而完成。

  • 任何线程都调用System.exit(), OracleRuntime.exitSession(),或oracle.aurora.vm.OracleRuntime.exitCall()方法。

  • 调用DBMS_JAVA.endsession()或DBMS_JAVA.endsession_and_related_state()方法。

在11 g版本1（11.1）之前，当一个调用因为调用System.exit()或oracle.aurora.vm.OracleRuntime.exitCall() ，Oracle JVM在专用和共享服务器模式下突然终止了调用并终止了所有线程。从11g版本1（11.1）开始，通过在DBMS_JAVA包中添加以下PL / SQL函数来解决这个DBMS_JAVA ：

  • FUNCTION endsession RETURN VARCHAR2;

  • 功能endsession_and_related_state RETURN VARCHAR2;

在调用期间，Java程序可以递归地运行更多的Java代码。例如，您的程序可以使用JDBC或SQLJ发出SQL查询，而JDBC或SQLJ又调用用Java编写的触发器。前面关于调用生命周期的所有注释都适用于对Java代码的最顶层调用，而不适用于递归调用。例如，从递归调用中调用System.exit()退出对Java的最顶层调用，而不仅仅是递归调用。

2.12.2 System.exit（），OracleRuntime.exitSession（）和OracleRuntime.exitCall（）

System.exit()方法终止JVM，不保留Java状态。它不会导致数据库会话终止或客户端断开连接。但是，数据库会话可能并且经常会在之后立即终止。OracleRuntime.exitSession()也终止JVM，不保留Java状态。但是，它还会终止数据库会话并断开客户端连接。

的行为OracleRuntime.exitCall()取决于OracleRuntime.threadTerminationPolicy()此方法返回一个boolean值。如果此值为true ，则应在数据库调用结束时终止任何活动线程，而不是保持静止。

  • 在共享服务器进程中， threadTerminationPolicy()始终为true 。

  • 在阴影（专用）过程中，默认值为false 。您可以通过调用OracleRuntime.setThreadTerminationPolicy()来更改该值。

    • 如果将值设置为false（即默认值），则所有线程都将保持静止状态，但会收到ThreadDeath异常以进行正常终止。

    • 如果值为true，则突然终止所有线程。

另外，还有另一种方法， OracleRuntime.callExitPolicy() 。如果没有调用OracleRuntime.exitSession() ， OracleRuntime.exitCall()或System.exit()方法，则此方法确定何时退出调用。可以使用OracleRuntime.setCallExitPolicy()将呼叫退出策略设置为以下之一：

  • OracleRuntime。EXIT_CALL_WHEN_MAIN_THREAD_TERMINATES

    如果设置为此值，则只要主线程返回或主线程上发生未捕获的异常，所有剩余的线程（守护程序和非守护程序）都是：

    • 如果threadTerminationPolicy()为true，则threadTerminationPolicy() ，共享服务器模式。

    • 如果threadTerminationPolicy()为false，则threadTerminationPolicy()静止。

  • OracleRuntime。EXIT_CALL_WHEN_ALL_NON_DAEMON_THREADS_TERMINATE

    这是默认值。如果设置了此值，则只有守护程序线程保持运行时，调用才会结束。在此刻：

    • 如果threadTerminationPolicy()为true ，则始终处于共享服务器模式，然后将threadTerminationPolicy()守护程序线程。

    • 如果threadTerminationPolicy()为false ，则守护程序线程将保持静止，直到下一次调用。这是影子（专用）服务器模式的默认设置。

  • OracleRuntime。EXIT_CALL_WHEN_ALL_THREADS_TERMINATE

    如果设置为此值，则仅当所有线程由于未捕获的异常而返回或结束时，调用才会结束。此时，无论threadTerminationPolicy()的值如何，调用都将结束。

注意：

在Oracle数据库9.x和早期数据库版本中，JVM的行为就像callExitPolicy()是OracleRuntime.对于共享和专用服务器进程， EXIT_CALL_WHEN_ALL_NON_DAEMON_THREADS_TERMINATE和threadTerminationPolicy()都为true 。这意味着此时杀死守护程序线程。此外，如果执行了exitCall() ，则在共享和专用服务器进程中，在调用结束之前，所有线程都将被终止。

目的

CLUSTER_SET为选择中的每一行返回一组集群ID和概率对。返回值是与字段名对象的VARRAY CLUSTER_ID和PROBABILITY 。群集标识符是Oracle NUMBER ;概率是BINARY_DOUBLE 。

topN和cutoff

您可以指定topN和cutoff来限制函数返回的簇数。默认情况下， topN和cutoff都为null，并返回所有集群。

  • topN是N最可能的簇。如果多个群集共享第N个概率，则该函数选择其中一个。

  • cutoff是概率阈值。仅返回概率大于或等于cutoff聚类。要仅按cutoff过滤，请为topN指定NULL 。

要返回大于或等于cutoff的N最可能的簇，请同时指定topN和cutoff 。

语法选择

CLUSTER_SET可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，也可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态挖掘数据。选择语法或分析语法 ：

  • 语法 - 使用第一种语法使用预定义的模型对数据进行评分。提供聚类模型的名称。

  • 分析语法 - 使用分析语法在没有预定义模型的情况下对数据进行评分。包括INTO n ，其中n是要计算的集群数，以及mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见analytic_clause :: = 。）

在对分区模型进行评分时， CLUSTER_SET函数的语法可以使用可选的GROUPING提示。请参阅GROUPING提示 。
