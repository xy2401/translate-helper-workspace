Root脚本自动化支持Oracle数据库安装

支持OCR和表决磁盘的直接文件放置

从Oracle Grid Infrastructure 19c开始，Oracle Standalone Clusters将取消对共享文件系统上直接OCR和Voting磁盘文件放置的支持。对于Oracle域服务集群，要求将OCR和表决文件置于Oracle自动存储管理（Oracle ASM）中，托管在共享文件系统上并用作ASM磁盘的文件之上。

在Oracle Grid Infrastructure 12c第2版（12.2）中，Oracle宣布它将不再支持直接在共享文件系统上放置Oracle Grid Infrastructure Oracle Cluster Registry（OCR）和投票文件。这种支持现在已经废除。从Oracle Grid Infrastructure 19c（19.3）开始，使用Oracle Standalone Clusters，您可以再次将OCR和Voting磁盘文件直接放在共享文件系统上。

Grid Infrastructure Management存储库的可选安装

从Oracle Grid Infrastructure 19c开始，Grid Infrastructure Management Repository（GIMR）对于Oracle Standalone Cluster的新安装是可选的。Oracle Domain Services Clusters仍然需要安装GIMR作为服务组件。

GIMR中包含的数据是基于应用机器学习的预防性诊断的基础，可以帮助提高Oracle Real Application Clusters（Oracle RAC）数据库的可用性。对GIMR进行可选安装可以实现更灵活的存储空间管理和更快的部署，尤其是在安装测试和开发系统期间。

Oracle Database Release 18c，版本18.1中的更改

以下是Oracle数据库全球化支持指南（适用于Oracle数据库版本18c，版本18.1）中的更改。

新功能

  • 支持Unicode 9.0，Unicode标准的主要版本取代以前的所有版本。

    请参阅“ Unicode支持 ” 。

  • 您现在可以独立于分区键列的声明排序规则，按范围，列表或引用对表进行分区。在Oracle Database 18c之前，在分区键中只允许使用排序规则BINARY, USING_NLS_COMP, USING_NLS_SORT,和USING_NLS_SORT_CS 。因此，您现在可以通过为表分区键列声明不区分大小写的排序规则，以不区分大小写的方式按字符值对表进行分区。

    请参阅“ 数据绑定排序对其他数据库对象的影响 ” 。

  • 在Oracle调用接口（OCI）中，您现在可以使用OCI_ATTR_COLLATION_ID属性的值在查询或DML语句中为绑定变量传递排序OCI_ATTR_COLLATION_ID 。通过设置绑定变量的排序规则，您可以控制用于比较条件中绑定变量的值和绑定变量作为参数的其他SQL操作的排序规则。使用OCI属性控制排序规则比将COLLATE运算符插入SQL语句文本更可取。后者会增加引入SQL注入问题的风险。

    请参阅“ 整理派生 ” 。

  • 新的重载函数VALIDATE_CHARACTER_ENCODING被添加到PL / SQL包UTL_I18N以验证VARCHAR2，NVARCHAR2，CLOB和NCLOB数据的字符编码。

    请参阅“ GDK for PL / SQL提供的包 ” 。

    另请参见Oracle Database PL / SQL包和类型参考 。

  • 支持其他语言和地区。

    • 此版本支持的其他语言：

      BASQUE, TURKMEN, URDU和URDU 。

      请参阅“ 语言 ” 。

    • 此版本支持的其他地区：

      ANGOLA, ANTIGUA AND BARBUDA, ARUBA, BARBADOS, BOTSWANA, CAYMAN ISLANDS, CURACAO, DOMINICA, DOMINICAN REPUBLIC, GHANA, GRENADA, GUYANA, HAITI, JAMAICA, LIECHTENSTEIN, MACAO, MALAWI, MAURITIUS, MOLDOVA, MOZAMBIQUE, NAMIBIA, SAINT KITTS AND NEVIS, SAINT LUCIA, SIERRA LEONE, SOUTH SUDAN, SURINAME, SWAZILAND, TRINIDAD AND TOBAGO, TURKMENISTAN,和ZIMBABWE 。

Oracle Database 12 c第2版（12.2）中的更改

以下是Oracle Database 12 c第2版（12.2）的Oracle数据库全球化支持指南中的更改。

新功能

  • 支持Unicode 7.0，Unicode标准的主要版本取代所有先前版本的标准。

    请参阅“ Unicode支持 ” 。

  • 当使用Oracle Universal Installer（OUI）或Oracle Database Configuration Assistant（DBCA）创建数据库时，Unicode字符集AL32UTF8现在用作默认数据库字符集。AL32UTF8字符集是Oracle的行业标准UTF-8编码的实现，它支持世界上大多数书面语言。使AL32UTF8字符集成为新数据库部署的默认字符集使数据库能够支持开箱即用的多语言全球化应用程序。

    请参阅“ 为数据库启用多语言支持 ” 。

  • 支持多租户容器数据库（CDB）中具有不同数据库字符集的可插拔数据库（PDB）。在早期的Oracle数据库发行版中，必须为CDB中的所有PDB设置相同的数据库字符集。

    请参阅“ 为多租户容器数据库选择数据库字符集 ” 。

  • 列级排序规则功能允许您在列级别声明字符比较规则。为列声明的排序规则将自动应用于引用该列的所有排序规则敏感的SQL操作。这使应用程序能够始终如一地将特定于语言的比较规则应用于需要这些规则的数据。实现此功能简化了从支持列级排序规则声明的非Oracle数据库系统到Oracle数据库的应用程序迁移。

    请参阅“ 列级排序规则和区分大小写 ” 。

    列级排序规则功能还允许您为表或模式声明不区分大小写的排序规则，以便始终以不区分大小写的方式比较表或模式中的所有列。因此，使用此功能，您现在可以轻松地将整个数据库创建为不区分大小写的数据库。

    请参阅“ 不区分大小写的数据库 ” 。

不推荐的功能

  • 此版本中不推荐使用Unicode归类算法（UCA）6.1归类（UCA0610_ *）。它们可能在将来的版本中被不支持和不可用。Oracle建议使用UCA 7.0排序规则（UCA0700_ *）。

    有关此版本中支持的UCA归类列表，请参阅表A-17 。

2.1使用UCP的要求

UCP具有以下设计时和运行时要求：

  • JRE 1.5或更高

  • JDBC潜水员或能够返回java.sql.的连接工厂类java.sql.Connection和javax.sql.XAConnection对象

    注意：

    支持10.1或更高版本的Oracle驱动程序。高级Oracle数据库功能（如Oracle RAC和快速连接故障转移）需要Oracle客户端软件附带的Oracle Notification Service库（ ons.jar ）。

  • ucp.jar库包含在应用程序的类路径中

  • ojdbc8.jar库包含在应用程序的类路径中

    注意：

    即使您将UCP与第三方数据库和驱动程序一起使用，也必须使用Oracle ojdbc8.jar库，因为UCP依赖于此库。

  • 支持SQL的数据库。 Oracle RAC和快速连接故障转移等高级功能需要Oracle数据库。

2.2 UCP中的基本连接步骤

UCP提供了一个启用池的数据源，应用程序使用该数据源从UCP JDBC连接池借用连接。没有为最基本的用例明确定义连接池。而是在借用连接时隐式创建默认连接池。

以下步骤描述如何从启用UCP池的数据源获取连接以访问数据库。示例2-1中提供了完整的示例：

  1. 使用UCP数据源工厂（ oracle.ucp.jdbc.PoolDataSourceFactory ）使用getPoolDataSource方法获取启用池的数据源的getPoolDataSource 。数据源实例必须是PoolDataSource类型。例如：
    PoolDataSource pds = PoolDataSourceFactory.getPoolDataSource（）;
  2. 设置获取与数据库的物理连接所需的连接属性。这些属性在数据源实例上设置，包括：用于连接数据库的URL，用户名和密码以及用于获取物理连接的连接工厂。这些属性特定于JDBC驱动程序和数据库。例如：
    pds.setConnectionFactoryClassName（“oracle.jdbc.pool。OracleDataSource“）; pds.setURL（”jdbc：oracle：thin：@ // localhost：1521 / XE“）; pds.setUser（” <user> “）; pds.setPassword（” <password> “）;
  3. 设置任何池属性以覆盖连接池的默认行为。池属性在数据源实例上设置。例如：
    pds.setInitialPoolSize（5）;
  4. 使用数据源实例获取连接。返回的连接是数据源连接池中物理连接的逻辑句柄。例如：
    Connection conn = pds.getConnection（）;
  5. 使用该连接在数据库上执行一些工作：
    语句stmt = conn.createStatement（）; stmt.execute（“SELECT * FROM foo”）;
  6. 关闭连接并将其返回池中。

删除类型同义词的基础类型对依赖对象的影响与删除同义词相同。

7.3.2.5重命名类型同义词

您可以使用RENAME语句重命名类型同义词。重命名同义词等同于删除它，然后使用新名称重新创建它。如果类型同义词具有依赖表或有效对象类型，则无法重命名该类型同义词。以下示例失败，因为同义词syn1具有从属对象类型：

目的

  • 有关缓存游标的执行计划的信息，请查询V$SQL_PLAN 。

  • 对于高速缓存游标执行计划的每个步骤或操作的执行统计信息（例如，生成的行数，读取的块数），查询V$SQL_PLAN_STATISTICS 。

  • 对于前三个视图的选择性预计算连接，请查询V$SQL_PLAN_STATISTICS_ALL 。

先决条件

要发出EXPLAIN PLAN语句，您必须具有将行插入到指定用于保存执行计划的现有输出表所需的权限。

您还必须具有执行要为其确定执行计划的SQL语句所需的特权。如果SQL语句访问视图，则您必须具有访问视图所基于的任何表和视图的权限。如果视图基于另一个基于表的视图，则您必须具有访问其他视图及其基础表的权限。

EXPLAIN PLAN语句是一种数据操作语言（DML）语句，而不是数据定义语言（DDL）语句。因此，Oracle数据库不会隐式提交EXPLAIN PLAN语句所做的更改。如果要在输出表中保留EXPLAIN PLAN语句生成的行，则必须提交包含该语句的事务。

以下说明适用于EXPLAIN PLAN ：

  • 如果statement包含parallel_clause ，则生成的执行计划将指示并行执行。但是， EXPLAIN PLAN实际上将语句插入到计划表中，因此您提交的并行DML语句不再是事务中的第一个DML语句。这违反了单个事务中对一个并行DML语句的Oracle数据库限制，并且该语句将以串行方式执行。要保持语句的并行执行，必须提交或回滚EXPLAIN PLAN语句，然后提交并行DML语句。

  • 要确定临时表上操作的执行计划，必须从同一会话运行EXPLAIN PLAN ，因为临时表中的数据是特定于会话的。

此提示导致将输入数据集划分为不同的数据切片，以便在前进到下一个分区之前对每个分区进行整体评分。但是，仍然可以通过分区进行并行处理。数据切片由构建模型时使用的分区键列确定。此方法可用于针对分区模型的任何数据挖掘功能。在对与许多分区关联的大数据进行评分时，该提示可能会产生查询性能提升，但在对大型系统上具有较少分区的大数据进行评分时可能会对性能产生负面影响。通常，如果将提示用于单行查询，则不会有性能提升。

非持久性队列

Oracle数据库高级队列可以向订阅者异步传递非持久性消息。这些消息可以是事件驱动的，并且不会超出系统（或实例）的故障。消息存储在系统创建的队列表中。Oracle数据库高级队列支持使用通用API的持久和非持久消息。

非持久队列（可以是单用户或多用户）提供向所有当前连接的用户通知的机制。订户可以添加到多个非多余的非持久队列，非持久队列可以是传播的目的地。

什么是Unified C ++ API？

用于可扩展标记语言（XML）工具的统一C ++应用程序编程接口（API）代表Oracle XML工具的一组C ++接口。所有三种C ++接口：抽象类，模板和由通用模板参数表示的隐式接口，由统一框架使用。

这种统一的方法提供了一个通用的，基于接口的框架，允许改进，更新，替换或添加XML工具，而不会影响任何基于接口的用户代码，并且最小程度地影响应用程序驱动程序，甚至可能影响应用程序配置。

注意：

在xml.hpp使用统一的C ++ API，用于Oracle XML Developer's Kit（XDK）应用程序。在较旧的，nonunified C ++ API oraxml.hpp已被废弃，仅支持向后兼容。它将在以后的版本中删除。

统一的C ++ API尽可能地支持万维网联盟（W3C）规范。但是，Oracle无法保证我们的实现完全支持该规范，因为W3C规范不涉及C ++实现。

ErrorHandler接口 - 这是根错误处理程序类。它处理错误的本地处理，主要来自底层的C实现。在某些实现中，它可能会抛出XmlException 。为了满足所有实现的需要，未在其签名中指定此行为。但是，它可以创建异常对象。错误处理程序传递给TCtx构造函数时TCtx被初始化。该界面的实现由用户提供。

如果将MemAllocator接口作为参数传递给TCtx构造函数，则覆盖operator new通常是有意义的。在这种情况下，C和C ++中的所有内存分配都可以由同一个分配器完成。

Tctx接口 - 这是XML上下文实现的隐式接口。它主要用于内存分配，错误（非异常）处理和不同的编码处理。上下文接口是一个隐式接口，应该用作类型参数。名称TCtx用作相应的类型参数名称。其实际替换是通过实际上下文实现参数化（模板化）的实现的实例化。在出现错误的情况下，可能会抛出XmlException 。所有构造函数都创建并初始化上下文实现。在多线程环境中，必须为每个线程初始化单独的上下文实现。

physical_attributes_clause

目的

physical_attributes_clause允许您指定PCTFREE ， PCTUSED和INITRANS参数的值以及表，集群，索引或物化视图的存储特征。

您可以在以下语句中指定physical_attributes_clause ：

  • CREATE CLUSTER和ALTER CLUSTER ：设置或更改集群的物理属性以及集群中的所有表（请参阅CREATE CLUSTER和ALTER CLUSTER ）。

  • CREATE TABLE ：设置TABLE的物理属性，表分区，对象表的OIDINDEX或索引组织表的溢出段（请参阅CREATE TABLE ）。

  • ALTER TABLE ：更改表的物理属性，未来表分区的默认物理属性或现有表分区的物理属性（请参阅ALTER TABLE ）。以下限制适用：

    • 您无法为临时表指定物理属性。

    • 您无法为群集表指定物理属性。集群中的表继承集群的物理属性。

  • CREATE INDEX ：设置索引或索引分区的物理属性（请参阅CREATE INDEX ）。

  • ALTER INDEX ：更改索引的物理属性，未来索引分区的默认物理属性或现有索引分区的物理属性（请参阅ALTER INDEX ）。

  • CREATE MATERIALIZED VIEW ：设置实体化视图的物理属性，其中一个分区，或Oracle Database生成的索引，以维护实体化视图（请参阅CREATE MATERIALIZED VIEW ）。

  • ALTER MATERIALIZED VIEW ：更改物化视图的物理属性，未来分区的默认物理属性，现有分区的物理属性或Oracle为维护物化视图而创建的索引（请参阅ALTER MATERIALIZED VIEW ）。

  • CREATE MATERIALIZED VIEW LOG和ALTER MATERIALIZED VIEW LOG ：设置或更改的物化视图日志的物理属性（见CREATE物化视图日志和ALTER物化视图日志 ）。

本节介绍physical_attributes_clause的参数。有关其他信息，请参阅为特定数据库对象设置或重置这些参数的SQL语句。

PCTFREE 整数

指定一个整数，表示为将来更新对象行而保留的数据库对象的每个数据块中的空间百分比。PCTFREE的值必须是0到99之间的值。值为0表示可以通过插入新行来填充整个块。默认值为10。此值保留每个块的10％以更新现有行，并允许插入新行以填充每个块的最多90％。

PCTFREE在创建和更改表，分区，集群，索引，物化视图，物化视图日志和区域映射的语句中具有相同的功能。PCTFREE和PCTUSED的组合确定是将新行插入现有数据块还是插入新块。请参阅“ PCTFREE和PCTUSED如何协同工作 ” 。

对PCTFREE条款的限制

更改索引时，只能在modify_index_default_attrs子句和split_index_partition子句中指定此参数。

PCTUSED 整数

指定一个整数，表示Oracle为数据库对象的每个数据块维护的已用空间的最小百分比。PCTUSED指定为0到99之间的正整数，默认为40。

PCTUSED在创建和更改表，分区，集群，物化视图，物化视图日志和区域映射的语句中具有相同的功能。

PCTUSED不是索引组织表的有效表存储特性。

PCTFREE和PCTUSED的总和必须等于或小于100。您可以一起使用PCTFREE和PCTUSED来更有效地利用数据库对象中的空间。请参阅“ PCTFREE和PCTUSED如何协同工作 ” 。

对PCTUSED条款的限制

PCTUSED参数受以下限制：

  • 您不能为索引或索引组织表的索引段指定此参数。

  • 此参数无效，对具有自动段空间管理的对象将被忽略。

PCTFREE和PCTUSED如何协同工作

在新分配的数据块中，可用于插入的空间是块大小减去块开销和可用空间（ PCTFREE ）的总和。对现有数据的更新可以使用块中的任何可用空间。因此，更新可以将块的可用空间减少到小于PCTFREE 。

在数据块填充到PCTFREE确定的限制之后，Oracle数据库会认为该块不可用于插入新行，直到该块的百分比低于参数PCTUSED 。在达到此值之前，Oracle数据库仅使用数据块的可用空间来更新已包含在数据块中的行。当块的使用空间低于PCTUSED时，块成为行插入的候选。

INITRANS 整数

指定分配给数据库对象的每个数据块中分配的初始并发事务条目数。此值的范围为1到255，默认为1，但以下情况除外：

  • 默认INITRANS为群集值是2或默认INITRANS其中集群所在的表空间中的较大者的值。

  • 索引的默认值为2。

通常，您不应更改INITRANS值的默认值。

更新块的每个事务都需要块中的事务条目。此参数确保最小数量的并发事务可以更新块，并有助于避免动态分配事务条目的开销。

该INITRANS参数用来在创建和更改表，分区，集群，索引的语句同样的目的，物化视图和物化视图日志。

MAXTRANS参数

在早期版本中， MAXTRANS参数确定了段中每个数据块允许的最大并发更新事务数。此参数已被弃用。Oracle现在自动允许任何数据块最多255个并发更新事务，具体取决于块中的可用空间。

已经设置了MAXTRANS值的现有对象保留该设置。但是，如果您尝试更改MAXTRANS的值，Oracle将忽略新规范并替换值255而不返回错误。

storage_clause

storage_clause允许您为表，对象表OIDINDEX ，分区，LOB数据段或索引组织的表溢出数据段指定存储特性。该子句具有大型表的性能分支。应分配存储以最小化额外空间的动态分配。有关更多信息，请参阅storage_clause 。

15.1 Oracle扩展的对象引用

Oracle支持使用对数据库对象的引用。Oracle JDBC为对象引用提供支持：

  • SELECT子句中的列

  • IN或OUT绑定变量

  • Oracle对象中的属性

  • 集合类型对象中的元素

在SQL中，对象引用（ REF ）是强类型的。例如，对EMPLOYEE对象的引用将被定义为EMPLOYEE REF ，而不仅仅是REF 。

选择对象引用时，请注意您只检索指向对象的指针，而不是对象本身。您可以选择将引用具体化为java.sql.Ref实例的可移植性，或将其实现为您事先创建的自定义Java类的实例，这是强类型的。用于对象引用的自定义Java类称为自定义引用类 ，必须实现oracle.jdbc.OracleData接口。

您可以通过结果集或可调用语句对象检索REF实例，并通过REF准备语句或可调用语句对象将更新的REF实例传递回数据库。REF类包括获取和设置基础对象属性值的功能，并获取基础对象的SQL基本类型名称。

自定义引用类包括相同的功能，并且具有强类型的优点。这可以帮助您在编译期间找到编码错误，否则可能在运行时才会发现。

注意：

  • 如果您使用的是oracle.jdbc.对于自定义对象类的OracleData接口，您可能会将OracleData用于相应的自定义引用类。但是，如果您使用的是标准java.sql.用于自定义对象类的SQLData接口，那么您只能使用弱Java类型进行引用。SQLData接口仅用于映射SQL对象类型。

  • 只能通过运行SQL语句在JDBC应用程序中创建和检索REF对象。没有特定于JDBC的功能来创建和检索REF对象。

  • 即使数组（如对象）是结构化类型，也不能引用数组。

目的

对于指定的度量， LISTAGG对ORDER BY子句中指定的每个组内的数据进行LISTAGG ，然后连接度量列的值。

  • 作为单集聚合函数， LISTAGG对所有行进行操作并返回单个输出行。

  • 作为一个组集聚合，该函数对GROUP BY子句定义的每个组进行操作并返回一个输出行。

  • 作为分析函数， LISTAGG根据query_partition_clause中的一个或多个表达式将查询结果集划分为组。

函数的参数遵循以下规则：

  • ALL关键字是可选的，提供语义清晰度。

  • measure_expr是度量列，可以是任何表达式。度量列中的空值将被忽略。

  • delimiter指定用于分隔度量列值的字符串。此子句是可选的，默认为NULL 。

    如果measure_expr的类型为RAW ，则分隔符必须是RAW类型。可以通过指定的分隔符作为一个字符的字符串，可以隐式转换来实现这一RAW ，或由分隔符明确地转换为RAW ，例如，使用UTL_RAW.CAST_TO_RAW功能。

  • order_by_clause确定返回连接值的顺序。仅当ORDER BY列列表实现唯一排序时，该函数才是确定性的。

DISTINCT关键字从列表中删除重复值。

32.3.4分组提示

数据挖掘功能包括SQL函数，如PREDICTION* ， CLUSTER* ， FEATURE*和ORA_DM_* 。GROUPING提示是一个可选提示，适用于对分区模型进行评分时的数据挖掘评分功能。

增强的PREDICTION函数命令格式

此提示导致将输入数据集划分为不同的数据切片，以便在前进到下一个分区之前对每个分区进行整体评分。但是，仍然可以通过分区进行并行处理。数据切片由构建模型时使用的分区键列确定。此方法可用于针对分区模型的任何数据挖掘功能。在对与许多分区关联的大数据进行评分时，该提示可能会产生查询性能提升，但在对大型系统上具有较少分区的大数据进行评分时可能会对性能产生负面影响。通常，如果将提示用于单行查询，则不会有性能提升。

<prediction function> :: = PREDICTION <left paren> / * + GROUPING * / <prediction model> [<comma> <class value> [<comma> <top N>]] USING <mining attribute list> <right paren>

仅给出了PREDICTION函数的语法，但它适用于任何发生PREDICTION ， CLUSTERING和FEATURE_EXTRACTION评分函数的数据挖掘函数。

例32-11例子

SELECT PREDICTION（/ * + GROUPING * / my_model USING *）pred FROM <input table>;
