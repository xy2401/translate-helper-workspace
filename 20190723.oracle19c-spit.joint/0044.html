关于Oracle Flex ASM集群网络

从Oracle Grid Infrastructure 12c第1版（12.1）开始，作为Oracle Flex Cluster安装的一部分，Oracle ASM在Oracle Grid Infrastructure中配置以提供存储服务。

Oracle Flex ASM使Oracle ASM实例能够在与数据库服务器不同的物理服务器上运行。可以群集许多Oracle ASM实例以支持众多数据库客户端。每个Oracle Flex ASM集群都有自己的名称，在企业内部是全球唯一的。

您可以将所有存储要求合并到一组磁盘组中。所有这些磁盘组都由在单个Oracle Flex Cluster中运行的一小组Oracle ASM实例进行管理。

每个Oracle Flex ASM集群都有一个或多个运行Oracle ASM实例的集群节点。

Oracle Flex ASM可以使用与Oracle Clusterware相同的专用网络，也可以使用自己的专用专用网络。每个网络都可以分为PUBLIC ， ASM & PRIVATE ， PRIVATE或ASM 。

可以在安装期间配置Oracle ASM网络，也可以在安装后配置或修改Oracle ASM网络。

关于群集节点上的Oracle Flex ASM群集配置

可以使用以下特征配置Oracle Flex ASM集群节点：

  • 类似于先前发布的Oracle Grid Infrastructure集群成员节点，因为配置了集群节点角色的所有服务器都是对等服务器。

  • 与Oracle ASM磁盘有直接连接。

  • 运行Direct ASM客户端进程。

  • 运行Oracle ASM过滤器驱动程序，其功能之一是为Oracle Flex ASM群集提供群集防护安全性。

  • 通过为Oracle Flex ASM集群配置的全局Oracle ASM侦听器响应委派给它们的服务请求，该集群指定三个Oracle Flex ASM集群成员节点侦听器作为Oracle Flex ASM集群的远程侦听器。

  • 可以为在Oracle ASM集群的节点上运行的数据库客户端提供对Oracle ASM的远程访问以获取元数据，并允许数据库客户端直接对Oracle ASM磁盘执行块I / O操作。运行Oracle ASM服务器和远程数据库客户端的主机必须都是群集节点。

关于使用Oracle IOServer（IOS）配置的Oracle Flex ASM群集

Oracle IOServer实例为Oracle成员群集的节点上的Oracle数据库实例提供Oracle ASM文件访问，这些节点没有与Oracle ASM托管磁盘的连接。IOS使您可以在此类节点上配置Oracle成员群集。在存储群集上，每个节点上的IOServer实例会打开客户端发送其I / O的网络端口。 IOServer实例从客户端接收数据包，并对Oracle ASM磁盘执行适当的I / O，类似于任何其他数据库客户端。在客户端，数据库可以使用直接NFS（dNFS）与IOServer实例进行通信。但是，使用IOServer不需要客户端配置，因此您无需提供服务器IP地址或任何其他配置信息。在配置为通过IOServer访问Oracle ASM文件的节点和集群上，Oracle IOS实例的发现会自动发生。

要安装Oracle成员集群，Oracle域服务集群的管理员使用crsctl命令创建Oracle成员集群，该命令用于创建成员集群清单文件。在Oracle Grid Infrastructure安装期间，如果选择安装Oracle成员集群，则安装程序会提示您输入成员集群清单文件。“成员集群清单”文件中的属性指定是否希望Oracle成员集群通过IOServer实例访问Oracle ASM文件。

关于Oracle卸载选项

您可以使用deinstall命令在Oracle数据库主目录中停止和删除Oracle数据库软件和组件。

您可以使用deinstall删除以下软件：

  • Oracle数据库

  • Oracle Grid Infrastructure，包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）

  • Oracle Real Application Clusters（Oracle RAC）

  • Oracle数据库客户端

安装后，Oracle主目录中提供了deinstall命令。它位于$ORACLE_HOME/deinstall目录中。

deinstall使用Oracle主目录中的信息并使用您提供的信息创建响应文件。您可以使用-checkonly选项运行deinstall命令来使用先前生成的响应文件。您还可以编辑响应文件模板。

如果运行deinstall以删除Oracle Grid Infrastructure安装，则卸载程序会提示您以root用户身份运行deinstall命令。对于群集的Oracle Grid Infrastructure，脚本为rootcrs.sh ，对于独立服务器（Oracle Restart）的Oracle Grid Infrastructure，脚本为roothas.sh 。

注意：

  • 您必须从同一版本运行deinstall命令才能删除Oracle软件。请勿从更高版本运行deinstall命令以从早期版本中删除Oracle软件。例如，不要从19c Oracle主目录运行deinstall命令，以从现有的11.2.0.4 Oracle主目录中删除Oracle软件。

  • 与Oracle数据库12c的第1版（12.1.0.2）开始， roothas.sh脚本替换roothas.pl在Oracle网格基础架构回家甲骨文重新启动脚本， rootcrs.sh脚本替换rootcrs.pl脚本在网格家适用于集群的Oracle Grid Infrastructure。

如果Oracle主目录中的软件未运行（例如，安装失败后），则deinstall无法确定配置，您必须以交互方式或在响应文件中提供所有配置详细信息。

此外，在为Oracle Grid Infrastructure安装运行deinstall之前：

  • 卸载Oracle自动存储管理群集文件系统（Oracle ACFS）并禁用Oracle自动存储管理动态卷管理器（Oracle ADVM）。

  • 如果正在使用网格命名服务（GNS），请通知DNS管理员从DNS中删除子域名条目。

deinstall删除的文件

当您运行deinstall ，如果中央库存（ oraInventory ）除了您要解除配置和删除的主页之外不包含其他已注册的主目录，则deinstall将删除Oracle数据库安装所有者的Oracle基本目录中的以下文件和目录内容：

  • 管理

  • 诊断

警告：

目的

注意：

  • 您必须从同一版本运行deinstall命令才能删除Oracle软件。请勿从更高版本运行deinstall命令以从早期版本中删除Oracle软件。例如，不要从19c Oracle主目录运行deinstall命令，以从现有的11.2.0.4 Oracle主目录中删除Oracle软件。

在没有仲裁磁盘的情况下部署Oracle ASM磁盘组的准则

  • 要使用NFS文件系统，它必须位于支持的NAS设备上。通过以下URL登录My Oracle Support，然后单击“认证”以查找有关支持的NAS设备的最新信息：

  • 直接NFS需要硬安装。硬安装NFS文件管理器可防止在客户端连接超时时可能发生的损坏。如果NFS文件管理器挂起对镜像文件的I / O操作，则数据库和Oracle ASM无法故障转移到幸存的镜像副本。因此，Oracle建议您在NFS存储上部署Oracle ASM磁盘组时使用外部冗余。

  • NFS不支持Oracle ASM筛选器驱动程序和Oracle ACFS以及Oracle ADVM。由于基于NFS的存储的操作系统接口的性质，这些功能不兼容。

  • 存储在NFS上的Oracle ASM磁盘组上的Oracle软件和数据库的性能取决于Oracle服务器和NAS设备之间的网络连接的性能。Oracle建议您使用专用网络连接将服务器连接到NAS设备，该专用网络连接应为千兆以太网或更好。

  • 部署Oracle独立群集配置时，可以在NFS上配置Oracle ASM。

  • 您可以为Oracle Clusterware文件和Grid Infrastructure Management Repository（GIMR）为Oracle ASM磁盘组指定单独的NFS位置。

  • 用于执行安装的用户帐户（ oracle或grid ）必须具有写入权限才能在指定的路径中创建文件。

  • 在NFS上选择Oracle ASM时，无法使用Oracle自动存储管理群集文件系统（Oracle ACFS）进行存储。此群集配置不能用作Oracle Fleet Patching和Provisioning Server。

使用Quorum磁盘部署Oracle ASM磁盘组的准则

  • SAN连接存储或iSCSI连接设备是连接仲裁磁盘的首选方法。如果标准部署要求将NFS用作存储，则使用基于NFS的Oracle ASM仲裁磁盘的软安装和其他Oracle ASM磁盘的硬安装。

  • 您可以使用Direct NFS（dNFS）存储Oracle数据库数据文件。 dNFS不支持软安装，因此您无法将dNFS用于仲裁失败组。或者，使用基于内核的NFS和用于存储在仲裁失败组中的NFS存储的软安装。

  • Oracle ASM中的仲裁失败组功能允许在Oracle ASM磁盘组中使用NFS存储，而无需在仲裁失败组中对NFS存储进行硬装载。此功能对于Oracle Extended Clusters非常有用，其中需要第三个站点来建立仲裁。

关于直接NFS客户端安装到NFS存储设备

Direct NFS Client直接在Oracle软件中集成NFS客户端功能，以优化Oracle和NFS服务器之间的I / O路径。这种集成可以显着提高性能。

Direct NFS Client支持NFSv3，NFSv4，NFSv4.1和pNFS协议来访问NFS服务器。Direct NFS Client还简化了数据库工作负载的NFS客户端配置的性能优化，并在许多情况下实现了自动化。

从Oracle Database 12c第2版开始，启用Direct NFS时，还可以启用Direct NFS调度程序。Direct NFS调度程序将从数据库实例创建的TCP连接数合并到NFS服务器。在大型数据库部署中，使用Direct NFS调度程序可提高可伸缩性和网络性能。并行NFS部署还需要大量连接。因此，建议使用Direct NFS调度程序进行并行NFS部署。

Direct NFS Client可以从操作系统装入条目或oranfstab文件中获取NFS装入点。

直接NFS客户端要求

  • NFS服务器的写入大小值（ wtmax ）必须为32768或更高才能与Direct NFS Client一起使用。

  • 即使您将Direct NFS Client配置为提供文件服务，也必须由操作系统内核NFS客户端和Direct NFS Client挂载NFS挂载点。

    如果Oracle数据库无法使用Direct NFS Client连接到NFS服务器，则Oracle Database将使用操作系统内核NFS客户端连接到NFS服务器。当Oracle数据库无法通过Direct NFS Client连接到NAS存储时，它会在Oracle警报和跟踪文件中记录有关Direct NFS客户端连接错误的信息性消息。

  • 遵循标准指南，以维护操作系统NFS和Direct NFS客户端安装的Oracle数据库文件的完整性。

直接NFS挂载点搜索顺序

Direct NFS Client按以下顺序搜索装入条目：

  1. $ ORACLE_HOME / DBS / oranfstab

  2. 在/ etc / oranfstab

  3. 在/ etc / mtab中

Direct NFS Client使用第一个匹配条目作为安装点。

您可以使用deinstall命令在Oracle数据库主目录中停止和删除Oracle数据库软件和组件。

25.1关于随机森林

Random Forest是Oracle Data Mining使用的分类算法。该算法构建了树的集合 （也称为森林 ）。

该算法构建了许多决策树模型并使用整体进行预测。通过从训练数据集中选择随机样本作为输入来构建单独的决策树。在树的每个节点处，仅选择随机的预测器样本来计算分裂点。这引入了森林中不同树木使用的数据的变化。参数RFOR_SAMPLING_RATIO和RFOR_MTRY用于指定在每个节点处选择的样本大小和预测变量的数量。在运行算法之前，用户可以使用ODMS_RANDOM_SEED设置随机种子值。

25.2建立随机森林

随机森林建立在Oracle Data Mining的现有基础架构和应用程序编程接口（API）之上。

通过在现有API中指定参数来构建模型。使用与现有分类算法相同的SQL查询和API执行评分。Oracle Data Mining实现了经典随机森林算法的变体。此实现支持大数据集。算法的实现在以下方面有所不同：

  • Oracle Data Mining不支持包装，而是提供无需替换的采样

  • 用户可以指定树的深度。树木没有建造到最大深度。

24.1关于随机森林

Random Forest是Oracle Data Mining使用的分类算法。该算法构建了树的集合 （也称为森林 ）。

该算法构建了许多决策树模型并使用整体进行预测。通过从训练数据集中选择随机样本作为输入来构建单独的决策树。在树的每个节点处，仅选择随机的预测器样本来计算分裂点。这引入了森林中不同树木使用的数据的变化。参数RFOR_SAMPLING_RATIO和RFOR_MTRY用于指定在每个节点处选择的样本大小和预测变量的数量。在运行算法之前，用户可以使用ODMS_RANDOM_SEED设置随机种子值。

目的

CONVERT将字符串从一个字符集转换为另一个字符集。

  • char参数是要转换的值。它可以是CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB任何数据类型。

  • dest_char_set参数是char转换为的字符集的名称。

  • source_char_set参数是char存储在数据库中的字符集的名称。默认值是数据库字符集。

CHAR和VARCHAR2的返回值是VARCHAR2 。对于NCHAR和NVARCHAR2 ，它是NVARCHAR2 。对于CLOB ，它是CLOB ，对于NCLOB ，它是NCLOB 。

目标和源字符集参数都可以是文字或包含字符集名称的列。

要在字符转换中完全对应，目标字符集必须包含源字符集中定义的所有字符的表示。如果目标字符集中不存在字符，则会显示替换字符。替换字符可以定义为字符集定义的一部分。

注意：

Oracle不鼓励在当前的Oracle数据库版本中使用CONVERT函数。CONVERT的返回值具有字符数据类型，因此它应该在数据库字符集中或在国家字符集中，具体取决于数据类型。不支持任何不是这两个字符集之一的dest_char_set 。char参数和source_char_set具有相同的要求。因此，该功能的唯一实际用途是纠正存储在错误字符集中的数据。

关于Oracle卸载选项

您可以使用deinstall命令在Oracle数据库主目录中停止和删除Oracle数据库软件和组件。

您可以使用deinstall删除以下软件：

  • Oracle数据库

  • Oracle Grid Infrastructure，包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）

  • Oracle Real Application Clusters（Oracle RAC）

  • Oracle数据库客户端

安装后，Oracle主目录中提供了deinstall命令。它位于$ORACLE_HOME/deinstall目录中。

deinstall使用Oracle主目录中的信息并使用您提供的信息创建响应文件。您可以使用-checkonly选项运行deinstall命令来使用先前生成的响应文件。您还可以编辑响应文件模板。

如果运行deinstall以删除Oracle Grid Infrastructure安装，则卸载程序会提示您以root用户身份运行deinstall命令。对于群集的Oracle Grid Infrastructure，脚本为rootcrs.sh ，对于独立服务器（Oracle Restart）的Oracle Grid Infrastructure，脚本为roothas.sh 。

注意：

  • 您必须从同一版本运行deinstall命令才能删除Oracle软件。请勿从更高版本运行deinstall命令以从早期版本中删除Oracle软件。例如，不要从19c Oracle主目录运行deinstall命令，以从现有的11.2.0.4 Oracle主目录中删除Oracle软件。

  • 与Oracle数据库12c的第1版（12.1.0.2）开始， roothas.sh脚本替换roothas.pl在Oracle网格基础架构回家甲骨文重新启动脚本， rootcrs.sh脚本替换rootcrs.pl脚本在网格家适用于集群的Oracle Grid Infrastructure。

如果Oracle主目录中的软件未运行（例如，安装失败后），则deinstall无法确定配置，您必须以交互方式或在响应文件中提供所有配置详细信息。

  • 卸载Oracle自动存储管理群集文件系统（Oracle ACFS）并禁用Oracle自动存储管理动态卷管理器（Oracle ADVM）。

关于直接NFS客户端安装到NFS存储设备

Direct NFS Client直接在Oracle软件中集成NFS客户端功能，以优化Oracle和NFS服务器之间的I / O路径。这种集成可以显着提高性能。

Direct NFS Client支持NFSv3，NFSv4，NFSv4.1和pNFS协议来访问NFS服务器。Direct NFS Client还简化了数据库工作负载的NFS客户端配置的性能优化，并在许多情况下实现了自动化。

从Oracle Database 12c第2版开始，启用Direct NFS时，还可以启用Direct NFS调度程序。Direct NFS调度程序将从数据库实例创建的TCP连接数合并到NFS服务器。在大型数据库部署中，使用Direct NFS调度程序可提高可伸缩性和网络性能。并行NFS部署还需要大量连接。因此，建议使用Direct NFS调度程序进行并行NFS部署。

Direct NFS Client可以从操作系统装入条目或oranfstab文件中获取NFS装入点。

直接NFS客户端要求

  • NFS服务器的写入大小值（ wtmax ）必须为32768或更高才能与Direct NFS Client一起使用。

  • 即使您将Direct NFS Client配置为提供文件服务，也必须由操作系统内核NFS客户端和Direct NFS Client挂载NFS挂载点。

    如果Oracle数据库无法使用Direct NFS Client连接到NFS服务器，则Oracle Database将使用操作系统内核NFS客户端连接到NFS服务器。当Oracle数据库无法通过Direct NFS Client连接到NAS存储时，它会在Oracle警报和跟踪文件中记录有关Direct NFS客户端连接错误的信息性消息。

  • 遵循标准指南，以维护操作系统NFS和Direct NFS客户端安装的Oracle数据库文件的完整性。

直接NFS挂载点搜索顺序

Direct NFS Client按以下顺序搜索装入条目：

  1. $ ORACLE_HOME / DBS / oranfstab

  2. 在/ var /选择/ ORACLE / oranfstab

  3. 在/ etc / mnttab中

Direct NFS Client使用第一个匹配条目作为安装点。

要启用只读Oracle主目录：

  1. 执行仅限软件的Oracle数据库安装。

  2. 运行roohctl -enable脚本。

  3. 运行Oracle数据库配置助手（Oracle DBCA）以创建数据库。

纯软件数据库安装

  1. 以您希望拥有软件二进制文件的Oracle安装所有者用户帐户（ oracle ）身份登录。

  2. 将Oracle数据库安装映像文件（ db_home.zip ）下载到您选择的目录中。例如，您可以将图像文件下载到/tmp目录。

  3. 创建Oracle主目录并将已下载的映像文件解压缩到此Oracle主目录。例如：

    $ mkdir -p /u01/app/oracle/product/19.0.0/dbhome_1 $ chown oracle：oinstall /u01/app/oracle/product/19.0.0/dbhome_1 $ cd /u01/app/oracle/product/19.0。 0 / dbhome_1 $ unzip -q /tmp/db_home.zip

    注意：

    确保您创建的Oracle主目录路径符合Oracle Optimal Flexible Architecture建议。此外，仅在您创建的此Oracle主目录中解压缩安装映像文件。

  4. 在Oracle主目录中，运行runInstaller命令以启动Oracle数据库安装程序。

  5. 在“选择配置选项”屏幕中，选择“仅设置软件”。

  6. 选择您的安装类型。安装屏幕因您选择的安装选项而异。根据需要响应配置提示。

仅架构帐户和升级EXPIRED密码帐户

在开始升级之前，请确定是否要对密码处于EXPIRED状态且其帐户处于LOCKED状态的默认Oracle数据库帐户使用密码身份验证

在升级到Oracle Database 19c期间，在升级之前未将密码重置（并且设置为EXPIRED状态）并且也设置为LOCKED状态的默认Oracle帐户在升级完成后设置为NO AUTHENTICATION 。

由于此新功能，更改为仅架构帐户的默认帐户将无法用于密码身份验证。此功能的好处是管理员不再需要定期轮换这些Oracle数据库提供的模式的密码。此功能还可以降低使用默认密码侵入这些帐户的攻击者的安全风险。

如果要在升级期间阻止将这些Oracle帐户设置为仅架构帐户，则必须在开始升级之前为该帐户设置有效的强密码，或者在升级后为这些帐户设置有效的强密码，或者在登录升级的Oracle数据库之前解锁帐户。

升级后，管理员还可以为仅架构帐户启用密码身份验证。但是，为了更好的安全性，Oracle建议您将这些帐户保留为仅模式帐户。
