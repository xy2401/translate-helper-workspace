16 Oracle Text Utilities

Oracle Text提供了用于管理和操作Text索引的实用程序。例如，您可以将特定的同义词库加载到索引中，您可以创建自己的知识库以与索引相关联等。本章讨论Oracle Text附带的实用程序。

包括以下主题：

  • 同义词库加载器（ctxload）

  • 实体提取用户字典加载器（ctxload）

  • 知识库扩展编译器（ctxkbtc）

  • 词法编译器（ctxlc）

注意：

Oracle Text附带的实用程序中的API不支持以架构或所有者名称为前缀的标识符。

16.1同义词库加载器（ctxload）

使用ctxload来将同义词库文件导入Oracle Text同义词库表。

导入文件是ASCII平面文件，其中包含同义词，更宽泛的术语，更窄的术语或相关术语的条目，可用于扩展查询。

16.1.2 ctxload语法

ctxload -user username[/password][@sqlnet_address] -name object_name -file file_name [-thes] [-thescase y | n] [-thesdump] [-log file_name ] [-trace] [-drop]

ctxload强制性参数

用户

指定运行ctxload的用户的用户名和密码。

@ sqlnet_address可以立即跟随用户名和密码，以允许登录到远程数据库。sqlnet_address的值是数据库连接字符串。如果将TWO_TASK环境变量设置为远程数据库，则无需为sqlnet_address指定值以连接到数据库。

-name object_name

使用ctxload导入同义词库时，请使用object_name指定要导入的同义词库的名称。

使用object_name标识使用同义词库运算符的查询中的同义词库。

注意：

同义词名称必须是唯一的。如果为同义词库指定的名称与现有同义词库相同，则ctxload将返回错误，并且不会覆盖现有的同义词库。

-file file_name

使用ctxload导入同义词库时，请使用file_name指定包含同义词库条目的导入文件的名称。

使用ctxload导出同义词库时，请使用file_name指定ctxload创建的导出文件的ctxload 。

注意：

如果为同义词库转储文件指定的名称与现有文件相同，则ctxload覆盖现有文件。

ctxload可选参数

-thes

导入同义词库。使用-file参数指定源文件。使用-name指定要导入的同义词库的名称。

-thescase y | ñ

指定y以使用-name指定的名称创建区分大小写的同义词库，并使用-file指定的同义词库导入文件中的条目填充同义词库。如果 - thescase是y （同义词库区分大小写），则ctxload输入同义词库中的术语，与导入文件中显示的完全相同。

默认为 - thescase 是n （不区分大小写的词库）。

注意：

-thescase仅对-thes参数有效。

-thesdump

导出词库。使用-name参数指定要导出的同义词库的名称。使用-file参数指定目标文件。

-log

指定ctxload写入处理期间生成的任何支持国家语言（全球化支持）消息的日志文件的名称。如果未指定日志文件名，则消息将显示在标准输出上。ctxload生成的日志将出现在$ORACLE_HOME/ctx/log目录中。

-trace

使用ALTER SESSION SET SQL_TRACE TRUE启用SQL语句跟踪。此命令捕获跟踪文件中的所有已处理的SQL语句，这些语句可用于调试。跟踪文件的位置取决于操作系统，可以使用DIAGNOSTIC_DEST初始化参数进行修改。

16.2.1 ctxload语法

ctxload -user username[/password][@sqlnet_address] -name entity extraction policy name -file user-dictionary file name [-drop] 从策略中删除用户字典

ctxload强制性参数

用户

指定运行ctxload的用户的用户名和密码。

@ sqlnet_address可以立即跟随用户名和密码，以允许登录到远程数据库。sqlnet_address的值是数据库连接字符串。如果将TWO_TASK环境变量设置为远程数据库，则无需为sqlnet_address指定值以连接到数据库。

-name实体提取策略名称

使用ctxload导入实体提取字典时，请使用object_name指定要与字典关联的实体提取策略。实体提取策略只能有一个用户词典。

-file用户词典文件名

使用file指定包含用户词典的XML文件的名称。

-drop

删除当前与实体提取策略关联的用户词典。

16.2.2创建用户词典时的注意事项

以下是创建用户词典时的一些注意事项：

  • 实体提及区分大小写。它们不能包含任何空字符。

  • 实体类型名称不区分大小写。它们不能包含任何null或逗号字符。

  • 客户将能够为单个实体提及分配两个或更多实体类型。例如，可以为实体“Washington”分配类型“CITY”以及“STATE”类型。

  • 用户词典的内容对其他用户是不可见的。

  • 服务器端数据库字符集提到的实体的最大字节长度为512字节。

  • 服务器端数据库字符集的实体类型名称的最大字节长度为30个字节。

  • 用户定义的实体类型名称必须以字母“x”开头。

16.3知识库扩展编译器（ctxkbtc）

知识库是Oracle Text用于执行主题分析的信息源，例如主题索引，处理ABOUT查询以及使用CTX_DOC包记录主题提取。提供英语和法语的知识库，默认安装。

16.3.1知识库字符集

知识库可以是任何单字节字符集。提供的知识库位于WE8ISO8859P1中。您可以将扩展知识库存储在另一个字符集中，例如US7ASCII。

16.3.2 ctxkbtc语法

ctxkbtc -user uname / passwd
[-name thesname1 [ thesname2 ... thesname16 ] [-revert] [-stoplist stoplistname] [-verbose] [-log filename ]
用户

指定管理员创建扩展知识库的用户名和密码。该用户必须具有ORACLE_HOME目录的写权限。

-name thesname1 [thesname2 ... thesname16]

16.3.3 ctxkbtc使用说明

  • 在运行ctxkbtc之前，必须设置NLS_LANG环境变量以匹配数据库字符集。

  • 发出ctxkbtc的用户必须具有ORACLE_HOME写权限，因为程序将文件写入此目录。

  • 在编译之前，必须加载每个词库 Oracle Text使用thescase Y的“ thescase Y ”选项ctxload 。

  • 运行ctxkbtc两次删除以前的扩展名。

16.3.4 ctxkbtc限制

ctxkbtc程序具有以下限制：

  • 将数据库升级或降级到其他版本时，要使主题索引和相关功能正常工作，Oracle建议您在新环境中重新编译扩展知识库。

  • 在扩展知识库之前，必须终止在其生命周期内调用任何与知识库相关的Text函数的所有服务器进程。

  • 每种安装的每种语言只能有一个用户扩展名。由于用户扩展会影响安装中的所有用户，因此只有CTXSYS用户可以扩展知识库。

  • 在Oracle RAC环境中， ORACLE_HOME可以在多个节点之间共享，也可以每个节点都有自己的ORACLE_HOME 。以下要求适用：

    • 在任何Oracle RAC节点中使用任何与知识库相关的功能之前，必须在Oracle RAC环境中的每个ORACLE_HOME中运行ctxkbtc 。

    • 使用ctxkbtc ，必须在Oracle RAC环境中的每个ORACLE_HOME中使用完全相同的输入同义词库内容。

16.3.5 ctxkbcc对词库术语的约束

条款区分大小写。例如，如果同义词库具有大写的术语，则不会识别文档中以小写形式出现的相同术语。

术语的最大长度为80个字符。

不支持消除歧义的同形异义词。

16.3.6 ctxkbcc对同义词关系的约束

以下约束适用于同义词关系：

  • BTG和BTP与BT相同。 NTG和NTP与NT相同。

  • 只有首选术语可以有BT，NT或RT。

  • 如果一个术语没有USE关系，它将被视为自己的首选术语。

  • 如果一组术语与SYN关系相关，则它们中只有一个可能是首选术语。

  • 现有类别不能成为顶级术语。

  • BT和NT关系中没有循环。

  • 术语可以具有至多一个优选术语并且至多一个BT。一个术语可以有任意数量的NT。

  • 术语的RT不能是术语的祖先或后代。优选术语可以具有任何数量的RT，最多为32。

  • 树的最大高度为16，包括顶级项。

  • 当编写多个叙词表时，一个词库中的顶级术语不应该在另一个词库中具有更广泛的术语。

    注意：

    同义词库编译器容忍一些违反前面规则的行为。例如，如果一个术语有多个BT，那么编译器会忽略它遇到的最后一个BT。

    类似地，现有知识库类别之间的BT仅在警告消息中产生。

    Oracle建议您不要设置具有违规的扩展存储库。使用包含违规的扩展存储库会产生不希望的结果。

16.3.7扩展知识库

通过使用Oracle Text知识库编译一个或多个叙词表来扩展所提供的知识库。扩展信息可以是特定于应用程序的术语和关系。在主题分析期间，知识库的扩展部分将覆盖知识库中存在重叠的任何术语和关系。

在扩展知识库时，Oracle建议将新术语链接到知识库中的一个类别，以便在适当时进行主题验证。

如果新条款与现有类别完全脱节，则新条款中的主题将被证明更少。这样做的结果是较差的精确度和召回与ABOUT查询以及精神质量差和主题突出显示。

通过使现有术语成为新术语的更广泛术语，将新术语与现有术语相关联。

16.3.8扩展知识库的示例

您购买的医疗词库medthes含医学术语的层次结构。词库中的四个最高术语如下：

  • 麻醉和镇痛

  • 抗过敏和呼吸系统药剂

  • 抗炎药，抗风湿药和炎症介质

  • 抗肿瘤和免疫抑制剂

要将这些术语链接到现有知识库，请将以下条目添加到医学词库，以将新术语映射到现有的健康和医学分支：

健康和医药NT麻醉和镇痛NT抗过敏和呼吸系统药剂NT抗炎药，抗风湿药和炎症介质NT抗肿瘤和免疫抑制剂

设置全球化支持语言环境变量以匹配数据库字符集。例如，如果您的数据库字符集是WE8ISO8859P1并且您使用的是美式英语，请按如下方式设置NLS_LANG ：

setenv NLS_LANG AMERICAN_AMERICA.WE8ISO8859P1

假设医学词库位于名为med.thes的文件中， medthes使用ctxload将同义词库加载为medthes ，如下所示：

ctxload -thes -thescase y -name medthes -file med.thes -user ctxsys / ctxsys

要加载的词库链接medthes到知识库，使用ctxkbtc 如下：

ctxkbtc -user ctxsys / ctxsys -name medthes

16.3.9添加特定于语言的知识库

通过为任何单字节空白分隔语言（包括西班牙语）加载您自己的知识库，将主题功能扩展为英语或法语以外的语言。

主题功能包括主题索引， ABOUT查询，主题突出显示，以及使用CTX_DOC PL / SQL包生成主题，要点和主题摘要。

通过添加用户定义的知识库来扩展主题功能。例如，您可以从西班牙语词库创建西班牙语知识库。

要加载特定于语言的知识库，请按照下列步骤操作：

  1. 使用ctxload加载自定义同义词库。

  2. 设置NLS_LANG以使语言部分成为目标语言。charset部分必须是单字节字符集。

  3. 使用ctxkbtc编译加载的同义词库：

ctxkbtc -user ctxsys / ctxsys -name my_lang_thes

此命令从加载的同义词库中编译特定于语言的知识库。要在索引和ABOUT查询期间使用此知识库进行主题分析，请将NLS_LANG语言指定为BASIC_LEXER首选项的THEME_LANGUAGE属性值。

16.3.10添加知识库的限制

以下限制适用于添加知识库：

  • Oracle Text仅提供英语和法语的知识库。您必须为任何其他语言提供自己的同义词库。

  • 您只能为具有单字节字符集的语言添加知识库。您无法为只能以多字节字符集表示的语言创建知识库。如果数据库是多字节通用字符集（如UTF-8），则在编译同义词库时，仍必须将NLS_LANG参数设置为兼容的单字节字符集。

  • 添加知识库最适合以空格分隔的语言。

  • 每种全球化支持语言最多只能有一个知识库。

  • 获取分层查询反馈信息（如更广泛的术语，更窄的术语和相关术语）在英语和法语以外的语言中不起作用。在其他语言中，知识库完全来自您的叙词表。在这种情况下，Oracle建议您直接从您的叙词表中获取分层信息。

16.3.11多个叙词表的优先顺序

当要编译多个叙词表时，优先级由叙述符在编译器的参数中列出的顺序确定，假定首先是最优选的 。用户定义的同义词库始终优先于内置知识库。

16.4词法编译器（ctxlc）

词法编译器（ ctxlc ）是一个命令行实用程序，使您可以创建自己的中文和日语词典（词典）。这样的词典可以从用户提供的单词列表生成，也可以从单词列表与该语言的系统词典合并生成。

ctxlc在当前目录中创建新词典。新词典由三个文件组成， drold.dat ， drolk.dat和droli.dat 。要更改日语或中文的系统词典，请使用这些文件覆盖系统词典。

词法编译器还可以从日语和中文的系统词典生成单词列表，使您可以查看其内容。这些单词列表转到标准输出，因此可以重定向到您选择的文件中。

覆盖系统词典后，需要在查询之前重新创建索引。

ctxlc的16.4.1语法

ctxlc具有以下语法：

ctxlc -ja | -zht [-n] -ics character_set -i input_file ctxlc -ja | -zht -ocs character_set [> output_file ]

ctxload强制性参数

-ja | -zht

指定要修改或创建的词典的语言。 -ja表示日语词典; -zht表示中文词典，对于传统或简体中文都是如此。

-ics character_set

指定由-i input_file表示的输入文件的字符集。 input_file是用于创建新词典的单词列表，一行到一行。

-i input_file

指定包含要在创建新词典时使用的单词的文件。

-ocs character_set

指定要输出的文本文件的字符集。

ctxload可选参数

-n

指定-n以创建仅包含从input_file获取的用户提供的单词的新词典。如果未指定-n ，则新词典包含系统词典与input_file的合并。此外，如果未选择-n ，则会在当前目录中创建名为drolt.dat的文本文件，以便您无需输入另一个ctxlc命令即可检查合并词典的内容。

16.4.2 ctxlc性能注意事项

您可以在词典中添加最多1,000,000个新单词。但是，创建一个非常大的词典可能会降低索引和查询的性能。当词典字符集为UTF-8时，性能最佳。对中文或日文V-gram词法分析师没有性能影响，因为他们不使用词典。

16.4.3 ctxlc使用说明

Oracle建议关于ctxlc的以下实践：

  • 将纯文本字典文件保存在您的环境中以供紧急使用。

  • 将数据库升级或降级到其他版本时，请在新环境中重新编译纯文本字典文件，以便用户词典正常工作。

16.4.4 ctxlc实施例

在此示例中，您将从文件jadict.txt创建一个新的日语词典，该文件是使用JA16EUC字符集的单词列表。因为您没有指定-n ，所以新词典是将jadict.txt与系统日语词典合并的结果。然后用新合并的词汇替换现有的日语词典。 

％ctxlc -ja -ics JA16EUC -i jadict.txt

这会在当前目录中创建新文件：

％ls drold.dat drolk.dat droli.dat drolt.dat

对日本和中国的系统词库文件名为drold xx .dat drolk xx .dat ，并droli xx .dat ，其中xx或者是JA （日语）或ZH （对中国）。重命名这三个新文件并将它们复制到包含系统日语词典的目录中。 

％mv drold.dat droldJA.dat％mv drolk.dat drolkJA.dat％mv droli.dat droliJA.dat％cp * dat $ ORACLE_HOME / ctx / data / jalx

这将系统日语词典替换为旧系统词典和jadict.txt 。

您还可以使用ctxlc来获取系统词典的转储。此示例将中文词典转储到当前目录中名为new_chinese_dict.txt的文件中：

％ctxlc -zh -ocs UTF8> new_chinese_dict.txt

这将在当前目录中使用UTF8字符集创建一个文件new_japanese.dict.txt 。

将应用程序请求与服务器关联。

目的

句法

sword OCIStmtExecute（OCISvcCtx * svchp，OCIStmt * stmtp，OCIError * errhp，ub4 iters，ub4 rowoff，const OCISnapshot * snap_in，OCISnapshot * snap_out，ub4 mode）;

参数

svchp （IN / OUT）

服务上下文句柄。

stmtp （IN / OUT）

声明句柄。它定义了要在服务器上执行的语句和相关数据。如果svchp指向Oracle7服务器，则svchp仅具有版本8.x或更高版本支持的数据类型绑定的语句句柄无效。

errhp （IN / OUT）

当出现错误时，您可以传递给OCIErrorGet()以获取诊断信息的错误句柄。

iters （IN）

对于非SELECT语句，执行此语句的次数等于iters - rowoff 。

对于SELECT语句，如果iters非零，则必须为语句句柄完成定义。执行将iters行提取到这些预定义的缓冲区中，并根据预取行数预取更多行。如果您不知道SELECT语句检索了多少行，则将iters设置为零。

如果对于DDL语句， iters不为1，则此函数返回错误。

注意：

对于数组DML操作，设置iters <= 32767以获得更好的性能。

rowoff （IN）

数组绑定中的数据的起始索引与此多行执行相关。

snap_in （IN）

此参数是可选的。如果提供，则它必须指向类型为OCI_DTYPE_SNAP的快照描述符。必须从先前调用的snap_out参数获取此描述符的内容。如果SQL不是SELECT语句，则忽略描述符。此工具允许Oracle数据库的多个服务上下文查看数据库提交的数据的相同一致快照。但是，即使使用相同的快照，一个上下文中的未提交数据也不会对另一个上下文可见。

snap_out （OUT）

此参数是可选的。如果它被提供，那么它必须指向类型为OCI_DTYPE_SNAP的描述符。此描述符使用不透明表示填充，该表示是当前Oracle数据库系统更改编号（SCN），适合作为后续调用OCIStmtExecute()的snap_in输入。要避免“快照太旧”错误，请不要超过必要时使用此描述符。

模式 （IN）

模式是：

  • OCI_BATCH_ERRORS - 有关此模式的信息，请参阅使用批处理错误模式 。

  • OCI_COMMIT_ON_SUCCESS - 在此模式下执行语句时，如果执行成功完成，则在执行后提交当前事务。

  • OCI_DEFAULT - 在此模式下调用OCIStmtExecute()执行该语句。它还隐式返回有关选择列表的描述信息。

  • OCI_DESCRIBE_ONLY - 此模式适用于希望在执行前描述查询的用户。在此模式下调用OCIStmtExecute()不会执行该语句，但它会返回选择列表描述。为了最大限度地提高性能，Oracle建议应用程序以默认模式执行语句，并使用执行时附带的隐式描述。

  • OCI_EXACT_FETCH - 当应用程序事先知道它正在获取的行数时使用。此模式为Oracle Database 8或更高版本模式关闭预取，并要求在执行调用之前完成定义。使用此mode会在获取所需的行后取消游标，并可能导致服务器端资源使用量减少。

  • OCI_PARSE_ONLY - 此模式允许用户在执行前解析查询。在此模式下执行会解析查询并返回SQL中的解析错误（如果有）。用户必须注意，这涉及到服务器的额外往返。为了最大限度地提高性能，Oracle建议用户以默认模式执行语句，该模式将语句作为捆绑操作的一部分进行分析。

  • OCI_STMT_SCROLLABLE_READONLY - 结果集可滚动所必需的。结果集无法更新。有关此模式的更多信息，请参见关于获取结果 。此模式不能与任何其他模式一起使用。

    OCI_RETURN_ROW_COUNT_ARRAY - 此模式允许用户每次迭代获得DML行OCI_RETURN_ROW_COUNT_ARRAY 。对于非DML的语句，传递此模式是错误的。有关更多信息，请参见语句句柄属性 。此模式可与OCI_BATCH_ERRORS一起使用。

这些模式并不相互排斥;您可以一起使用它们，但OCI_STMT_SCROLLABLE_READONLY除外。

评论

此函数用于执行准备好的SQL语句。使用执行调用，应用程序将请求与服务器相关联。

如果执行SELECT语句，则可以隐式地将选择列表的描述作为响应。此描述在客户端缓冲，用于描述，提取和定义类型转换。因此，仅在执行之后描述选择列表是最佳的。

26.2.2 OCIStmtFetch2（）

从（可滚动）结果集中获取一行。

目的

从（可滚动）结果集中获取一行。建议您使用此fetch调用而不是不推荐的调用OCIStmtFetch() 。

句法

sword OCIStmtFetch2（OCIStmt * stmthp，OCIError * errhp，ub4 nrows，ub2 orientation，sb4 fetchOffset，ub4 mode）;

参数

stmthp（IN / OUT）

这是（可滚动）结果集的语句句柄。

errhp（IN / OUT）

如果发生错误，您可以传递给OCIErrorGet()以获取诊断信息的错误句柄。

nrows（IN）

从当前位置获取的行数。

方向（IN）

可接受的值是：

  • OCI_DEFAULT - 与OCI_FETCH_NEXT具有相同的效果

  • OCI_FETCH_CURRENT - 获取当前行。

  • OCI_FETCH_NEXT - 获取当前位置的下一行。它是默认值（与OCI_DEFAULT具有相同的效果）。用于不可滚动的语句句柄。

  • OCI_FETCH_FIRST - 获取结果集中的第一行。

  • OCI_FETCH_LAST - 获取结果集中的最后一行。

  • OCI_FETCH_PRIOR - 将结果集放在结果集中当前行的上一行。您也可以使用此模式从“上一行”获取多行。

  • OCI_FETCH_ABSOLUTE - 使用绝对定位获取结果集中的行号（由fetchOffset参数指定）。

  • OCI_FETCH_RELATIVE - 使用相对定位获取结果集中的行号（由fetchOffset参数指定）。

fetchOffset（IN）

与orientation参数一起使用的偏移量，用于更改当前行位置。

模式（IN）

OCI_DEFAULT 。

评论

fetch调用与不推荐使用的OCIStmtFetch()调用类似，但添加了fetchOffset参数。它可以在任何语句句柄上使用，无论它是否可滚动。对于不可滚动的语句句柄 ，唯一可接受的orientation值是OCI_FETCH_NEXT ，并忽略fetchOffset参数。

对于新应用程序，建议您使用此调用OCIStmtFetch2() 。

一个fetchOffset与orientation设定为OCI_FETCH_RELATIVE等同于所有的以下内容：

  • OCI_FETCH_CURRENT ，其值为fetchOffset等于0

  • OCI_FETCH_NEXT ，其值fetchOffset等于1

  • OCI_FETCH_PRIOR ，其值fetchOffset等于-1

OCI_ATTR_UB8_ROW_COUNT包含提取的最高绝对行值。

除OCI_FETCH_ABSOLUTE和OCI_FETCH_RELATIVE之外的所有其他定向模式OCI_FETCH_RELATIVE忽略fetchOffset值。

此调用还可用于通过使用OCI_FETCH_LAST然后在OCI_ATTR_CURRENT_POSITION上调用OCIAttrGet()来确定结果集中的行数。但这次通话的响应时间可能很长。如果使用OCI_FETCH_LAST方向将nrows设置为大于1，则认为nrows为1。

返回代码与不推荐使用的OCIStmtFetch() ，只是每次在可滚动语句句柄（或执行）上进行提取时返回带有返回码OCI_NO_DATA OER(1403) ，而不是应用程序请求的所有行都可以牵强。

如果在nrows参数设置为0的情况下调用OCIStmtFetch2() ， OCIStmtFetch2()取消游标。

必须显式取消可滚动语句句柄（即，使用0行获取）或释放可滚动语句句柄以释放可滚动游标的服务器端资源。在接收OER(1403)隐式取消不可滚动的语句句柄OER(1403) 。

使用OCI_ATTR_ROWS_FETCHED查找在上次提取调用中成功提取到用户缓冲区的行数。

  • LAG_DIFF_PERCENT返回当前成员的度量值与先前成员的度量值之间的百分比差异。

  • LEAD返回后一个成员的度量值。

  • LEAD_DIFF返回当前成员的度量值与后一成员的度量值之间的差异。

  • LEAD_DIFF_PERCENT返回当前成员的度量值与后一成员的度量值之间的百分比差异。

lead_lag_clause

指定要评估的层次结构和偏移值。lead_lag_clause的参数如下：

  • HIERARCHY hierarchy_ref指定分析视图中定义的层次hierarchy_ref的别名。

  • OFFSET offset_expr指定解析为数字的calc_meas_expression 。该数字指定从当前成员向前或向后移动的成员数。级别中成员的排序由层次结构使用的属性维度的定义确定。

  • WITHIN LEVEL指定通过向前或向后移动与当前成员具有相同级别深度的成员内的成员的偏移数来定位相关成员。级别中成员的排序由层次结构使用的属性维度的定义确定。

    如果未指定WITHIN LEVEL和ACROSS ANCESTOR AT LEVEL关键字，则WITHIN LEVEL操作是默认操作。

  • WITHIN PARENT指定通过向前或向后移动与当前成员具有相同父级的成员中的成员的偏移数来定位相关成员。

  • ACROSS ANCESTOR AT LEVEL level_ref指定通过导航到由level_ref指定的级别的当前成员的祖先（或者如果没有祖先存在的成员本身）并且注意每个祖先成员（包括成员）的位置来定位相关成员。在其父母内部。level_ref参数是指定层次结构中级别的名称。

    一旦找到祖先成员，导航就向前或向后移动成员内与祖先成员具有相同深度的成员的偏移数量。在找到相关的祖先之后，导航从该成员向下返回层次结构，匹配在向上的路径上记录的父级内的位置（以相反的顺序）。父级中的位置要么是偏离第一个子项还是最后一个子项，具体取决于是否指定了POSITION FROM BEGINNING或POSITION FROM END 。默认值为POSITION FROM BEGINNING 。级别中成员的排序由层次结构使用的属性维度的定义确定。

window_expression

window_expression选择从当前成员开始并且与当前成员具有相同深度的指定范围内的成员集。您可以通过使用WITHIN短语指定层次关系来进一步限制成员的选择。然后对所选测量值执行聚合以产生表达式的单个结果。

window_expression的参数如下：

  • aggregate_function是除COLLECT ， GROUP_ID ， GROUPING ， GROUPING_ID ， SYS_XMLAGG ， XMLAGG和任何多参数函数之外的任何现有SQL聚合函数。还允许用户定义的聚合函数。aggregate函数的参数是calc_meas_expression表达式。使用外部上下文计算这些表达式，并将指定层次结构的成员更改为相关范围中的每个成员。因此，每个相关成员对每个表达式参数进行一次计算。然后使用aggregate_function 。

  • OVER ( window_clause )指定要使用的层次结构以及要考虑的窗口边界。

window_clause参数选择与当前成员相关的成员范围。范围介于preceding_boundary或following_boundary参数指定的成员之间。范围始终在与当前成员相同级别的成员上计算。

window_clause的参数如下：

  • HIERARCHY hierarchy_ref指定分析视图中定义的层次hierarchy_ref的别名。

  • BETWEEN preceding_boundary或following_boundary定义与当前成员相关的成员集。

  • WITHIN LEVEL通过将boundary子句应用于当前级别的所有成员来选择相关成员。这是未指定WITHIN关键字时的默认值。

  • WITHIN PARENT通过将boundary子句应用于与当前成员共享父级的所有成员来选择相关成员。

  • WITHIN ANCESTOR AT LEVEL通过在共享一个祖先当前深度施加边界子句向所有成员中选择的相关成员（或是构件本身）在与当前部件指定的水平。如果当前成员高于指定级别，则窗口表达式的值为NULL 。如果级别不在指定的层次结构中，则会发生错误。

preceding_boundary

preceding_boundary参数定义从当前成员向前并指向边界的指定末尾的指定数量的成员的成员范围。以下参数指定范围：

  • UNBOUNDED PRECEDING开始该级别第一个成员的范围。

  • offset_expr PRECEDING从当前成员向后的offset_expr成员数开始。offset_expr表达式是一个解析为数字的calc_meas_expression 。如果偏移数大于当前成员到级别中第一个成员的成员数，则将第一个成员用作范围的开头。

  • CURRENT MEMBER结束CURRENT MEMBER的范围。

  • offset_expr PRECEDING结束从当前成员向后offset_expr的成员的范围。

  • offset_expr FOLLOWING结束从当前成员向前的offset_expr成员的范围。

  • UNBOUNDED FOLLOWING结束级别中最后一个成员的范围。

following_boundary

following_boundary参数定义从当前成员转发到指定结束范围的指定成员数的成员范围。以下参数指定范围：

  • CURRENT MEMBER开始CURRENT MEMBER的范围。

  • offset_expr FOLLOWING从当前成员开始向前的offset_expr成员的范围。

  • offset_expr FOLLOWING结束从当前成员向前的offset_expr成员的范围。

  • UNBOUNDED FOLLOWING结束级别中最后一个成员的范围。

hierarchy_ref

对分析视图层次结构的引用。hier_alias参数指定分析视图定义中层次结构的别名。您可以使用双引号来转义特殊字符或保留大小写，或两者兼而有之。

可选的attr_dim_alias参数指定分析视图定义中属性维的别名。如果指定的层次结构别名与分析视图中的其他层次结构别名冲突，或者在分析视图定义中多次使用属性维度，则可以使用attr_dim_alias参数来解决歧义。即使名称冲突不存在，您也可以使用attr_dim_alias参数。

rank_expression

分层排名计算根据指定度量值的顺序对指定层次结构的相关成员进行排名，并返回这些结果中当前成员的排名。

分层排名计算在指定的层次结构中定位一组相关成员，根据指定度量值的顺序对所有相关成员进行排名，然后在这些结果中返回当前成员的排名。相关成员是与当前成员处于同一级别的一组成员。您可以选择通过某种层次关系来限制集合，但该集合始终包含当前成员。度量值的排序由calc_meas_order_by_clause的rank_clause 。

rank_function_name

每个分层排名功能基于calc_meas_order_by_clause从1开始为每个相关成员分配订单号。这些函数在处理相同度量值的方式上有所不同。

它们之间的功能和区别如下：

  • RANK ，为相同的度量值分配相同的等级。一组绑定值之后的排名是绑定值加上绑定顺序值;因此，排序可能不是连续的数字。

  • DENSE_RANK ，为相同的度量值分配相同的最小等级。一组绑定值之后的排名总是比绑定值多一个;因此，订购总是有连续的数字。

  • AVERAGE_RANK ，将相同的平均等级分配给相同的值。平均等级值之后的下一个值是相同值的数量加1，该总和除以2加上平均等级值。例如，对于五个值AVERAGE_RANK的系列， AVERAGE_RANK返回AVERAGE_RANK 。对于系列2,12,10,12,17,12，返回的等级为1,2,3,3,3,5。

  • ROW_NUMBER ，它在层次结构成员中分配唯一且连续的值。如果calc_meas_order_by_clause导致相等的值，则结果是非确定性的。

rank_clause

rank_clause定位与当前成员相关的一系列层次结构成员。范围是与当前成员处于同一级别的成员的某个子集。子集由WITHIN子句确定。

WITHIN子句的有效值为：

  • WITHIN LEVEL ，指定相关成员是当前级别的所有成员。如果未指定WITHIN关键字，则这是默认子集。

  • WITHIN PARENT ，指定相关成员与当前成员共享父项

  • WITHIN ANCESTOR AT LEVEL ，指定相关成员是当前级别的所有成员，它们与当前成员共享指定级别的祖先（或自我）。

share_of_expression

share_of_expression表达式计算当前上下文的表达式值与相关上下文中表达式值的比率。表达式是在当前上下文和相关上下文中计算的calc_meas_expression 。share_clause规范确定要使用的相关上下文。

share_clause

share_clause通过将指定层次结构的成员设置为相关成员来修改外部上下文。

share子句的参数如下：

  • HIERARCHY hierarchy_ref指定作为share_of_expression计算的外部上下文的层次hierarchy_ref的名称。

  • PARENT指定相关成员是当前成员的父成员。

  • LEVEL level_ref指定相关成员是层次结构中指定级别的当前成员的祖先（或者是成员本身）。如果当前成员高于指定的级别，则为共享表达式返回NULL 。如果级别不在层次结构中，则会发生错误。

  • MEMBER member_expression指定相关成员是在评估当前上下文中的member_expression后返回的成员。如果指定成员的值为NULL ，则为共享表达式返回NULL 。

member_expression

member_expression求值为指定层次结构的成员。始终可以从外部表达式确定层次结构（由语法强制执行）。member_expression可以是以下之一：

  • level_member_literal是一个求值为层次结构成员的表达式。

  • hier_navigation_expr是一个表达式，它将层次结构的一个成员与另一个成员相关联。

  • CURRENT MEMBER指定由外部上下文确定的层次结构成员。

  • NULL是一种指定不存在的成员的方法。

  • ALL指定每个层次结构的单个最顶层成员。

level_member_literal

level_member_literal是一个解析为层次结构的单个成员的表达式。表达式包含级别的名称和一个或多个成员键。可以通过位置或名称来标识一个或多个成员密钥。如果指定的级别不在上下文层次结构中，则会发生错误。

pos_member_keys

member_key_expr表达式解析为成员的键值。按位置指定时，必须按ALL_HIER_LEVEL_ID_ATTRS字典视图中的顺序给出密钥的所有组件。对于子级别未确定指定级别的层次结构，必须在当前级别的成员键之前提供所有此类子级别的所有成员键值。重复的关键组件仅在它们第一次出现时指定。

当主密钥用于level_member_literal使用指定pos_member_keys短语。您可以使用named_member_keys短语引用备用密钥。

named_member_keys

member_key_expr表达式解析为成员的键值。attr_name参数是属性名称的标识符。如果所有属性名称都不构成指定级别的键或备用键，则会发生错误。

当按名称指定时，必须给出密钥的所有组件，并且所有组件必须以任何顺序使用属性name = value表单。对于未由子级别确定指定级别的层次结构，必须同时使用命名表单提供所有此类子级别的所有成员键值。重复的关键组件仅指定一次。

hier_navigation_expression

hier_navigation_expression表达式从指定的成员导航到层次结构中的其他成员。

hier_ancestor_expression

以指定的级别或深度从指定的成员导航到祖先成员（或成员本身）。深度指定为必须解析为数字的表达式。如果成员位于指定成员上方的某个级别或深度，或者该成员为NULL ，则为表达式值返回NULL 。如果指定的级别不在上下文层次结构中，则会发生错误。

hier_parent_expression

从指定成员导航到父成员。

hier_lead_lag_expression

通过向前或向后移动上下文层次结构中的一些成员，从指定成员导航到相关成员。HIER_LEAD关键字返回后一个成员。HIER_LAG关键字返回先前的成员。

hier_lead_lag_clause

从指定的成员向前或向后导航offset_expr成员数。级别中成员的顺序在属性维度的定义中指定。

hier_lead_lag_clause的可选参数如下：

  • WITHIN LEVEL通过向前或向后移动与当前成员具有相同深度的成员内的offset_expr成员来定位相关成员。级别内成员的排序由属性维度的定义决定。如果既不使用WITHIN也不使用ACROSS关键字，则WITHIN LEVEL操作是默认操作。

  • WITHIN PARENT通过在与当前成员具有相同深度的成员中向前或向后移动offset_expr成员来定位相关成员，但仅考虑与当前成员共享父成员的成员。级别内成员的排序由属性维度的定义决定。

  • WITHIN ACROSS ANCESTOR AT LEVEL通过导航到当前成员的祖先定位相关构件（或到构件本身）在指定的电平，并指出每个祖先构件的位置（包括构件本身）其父项内。一旦找到祖先成员，导航就会向前或向后移动与祖先成员具有相同深度的成员内的offset_expr成员。

    在找到相关祖先之后，导航从该成员向下移动层次结构，匹配在向上的过程中记录的父级内的位置（以相反的顺序）。父母中的位置要么偏离第一个孩子，要么是最后一个孩子，这取决于是否指定了POSITION FROM BEGINNING或POSITION FROM END ，默认为POSITION FROM BEGINNING 。级别内成员的排序由属性维度的定义决定。

qdr_expression

qdr_expression是一个限定数据引用， calc_meas_expression在新上下文中计算指定的calc_meas_expression ，并将层次结构成员设置为新值。

预选赛

限定符通过将指定层次结构的成员设置为评估member_expression得到的成员来修改外部上下文。如果member_expression为NULL ，则qdr_expression选择的结果为NULL 。

av_hier_expression

av_hier_expression执行层次结构导航以定位相关成员的属性值。一个av_hier_expression可以是顶级表达式，而hier_navigation_expression可以仅被用作member_expression参数。

hier_function_name

hier_function_name值如下：

  • HIER_CAPTION ，返回层次结构中相关成员的标题。

  • HIER_DEPTH ，返回的值小于层次结构中相关成员和ALL成员之间的祖先数。ALL成员的深度为0。

  • HIER_DESCRIPTION ，返回层次结构中相关成员的描述。

  • HIER_LEVEL ，以字符串值的形式返回相关成员在层次结构中所属的级别的名称。

  • HIER_MEMBER_NAME ，返回层次结构中相关成员的成员名称。

  • HIER_MEMBER_UNIQUE_NAME ，返回层次结构中相关成员的成员唯一名称。

1.3对象关系模型的主要特征

Oracle数据库将对象类型模型实现为关系模型的扩展，同时继续支持标准关系数据库功能，例如查询，快速提交，备份和恢复，可伸缩连接，行级锁定，读取一致性等。

SQL和各种编程接口和语言（包括PL / SQL，Java，Oracle调用接口，Pro * C / C ++和C＃）已通过扩展来增强，以支持Oracle对象。结果是一个对象关系模型，它提供了对象接口的直观性和经济性，同时保留了关系数据库的高并发性和吞吐量。

1.1 OCI概述

Oracle调用接口（OCI）是一个应用程序编程接口（API），它允许您创建使用函数调用来访问Oracle数据库并控制SQL语句执行和数据访问的所有阶段的应用程序。

OCI支持C和C ++的数据类型，调用约定，语法和语义。

OCI提供：

  • 通过有效使用系统内存和网络连接实现高性能和可扩展性

  • 用于双层客户端/服务器或多层环境中的动态会话和事务管理的一致接口

  • N层身份验证

  • 使用Oracle数据库对象全面支持应用程序开发

  • 访问外部数据库，例如Oracle TimesTen内存数据库和Oracle内存数据库缓存。请参阅Oracle TimesTen内存数据库C开发人员指南 。

  • 无需额外硬件投资即可支持越来越多用户和请求的应用程序

  • 使用C编程语言操作Oracle数据库中的数据和模式的方法，以及动态运行时库（OCI库）形式的标准数据库访问和检索函数库，可以在运行时在应用程序中进行链接。

  • 封装或不透明接口，其实现细节未知

  • 简化的用户身份验证和密码管理

  • 扩展以提高应用程序性能和可伸缩性

  • 一致的事务管理界面

  • OCI扩展，支持客户端访问Oracle对象

  • 与访问Oracle数据库的其他方法相比具有显着优势：

  • 一个或多个SQL语句

  • 变量声明

  • 作业陈述

  • 程序控制声明（IF ...然后...ELSE语句和循环）

1.5程序和非程序要素

1.6.1客户端对象缓存

对象缓存是客户端内存缓冲区，为对象提供查找和内存管理支持。

对象缓存存储和跟踪OCI应用程序从服务器提取到客户端的对象实例。初始化OCI环境时会创建对象缓存。当多个应用程序对同一服务器运行时，每个应用程序都有自己的对象缓存缓存跟踪当前在内存中的对象，维护对象的引用，管理自动对象交换，以及跟踪元属性或键入有关对象的信息。对象缓存为OCI应用程序提供以下功能：

  • 通过减少获取和操作对象所需的客户端/服务器往返次数来提高应用程序性能

  • 通过支持从客户端缓存进行对象交换来增强可伸缩性

  • 通过支持对象级锁定来改进并发性

OCI提供了一组带有扩展的函数，以支持使用SQL SELECT ， INSERT和UPDATE语句进行对象操作。要访问Oracle数据库对象，这些SQL语句使用一组一致的步骤，就像访问关系表一样。OCI提供访问对象所需的以下功能集：

  • 获取对象类型实例和引用

  • 描述Oracle对象类型的选择列表项

OCI还提供了一组函数，使用C风格的指针追逐方案，在通过遍历相应的智能指针或REF将对象提取到客户端缓存后访问对象。该导航界面提供以下功能：

  • 类型操作函数允许宿主语言应用程序操纵Oracle模式的各个属性，例如设置和获取其值以及将其值刷新到服务器。

1.6.5对象类型转换器

对象类型转换器（OTT）实用程序将有关Oracle对象类型的架构信息转换为主机语言变量（如结构）的客户端语言绑定。

OTT将包含有关Oracle模式对象的元数据信息的intype文件作为输入。它生成一个outtype文件以及必须包含在针对对象模式运行的C应用程序中的头文件和实现文件。OCI应用程序和Pro * C / C ++预编译器应用程序都可以包括OTT生成的代码。 OTT是有益的，因为它：

  • 提高应用程序开发人员的工作效率：OTT使您无需编写与架构对象相对应的主机语言变量。

  • 保持SQL作为首选的数据定义语言：通过提供自动映射使用SQL创建的Oracle模式对象来托管语言变量的功能，OTT便于使用SQL作为首选的数据定义语言。这反过来又允许Oracle数据库支持一致的数据模型。

  • 促进对象类型的模式演变：OTT在模式更改时重新生成包含的头文件，允许Oracle应用程序支持模式演变。

通常通过指定intype文件， outtype文件和特定数据库连接从命令行调用OTT。使用Oracle数据库，OTT只能生成可以与OCI程序一起使用或与Pro * C / C ++预编译器程序一起使用的C结构。

1.7简单的Oracle文档访问（SODA）

SODA for C是一个C API，是Oracle Call Interface（OCI）的一部分。

SODA for C实现了简单的Oracle文档访问（SODA）。您可以使用它对任何类型的文档执行创建，读取（检索），更新和删除（CRUD）操作，并且可以使用它来查询JSON文档。您编译使用SODA for C的程序与编译其他OCI程序的方式相同。SODA是一组NoSQL风格的API，允许您在Oracle数据库中创建和存储文档集合，检索它们并查询它们，而无需了解结构化查询语言（SQL）或文档中的数据如何存储在数据库。Oracle数据库支持存储和查询JSON数据。要访问此功能，您需要使用带有特殊JSON SQL运算符的结构化查询语言（SQL）。SODA for C隐藏了SQL / JSON编程的复杂性。

1.8封装接口

OCI调用使用的所有数据结构都以不透明接口的形式封装，这些接口称为句柄。

句柄是指向由OCI库分配的存储区域的不透明指针，该存储区域存储有关SQL或PL / SQL语句的上下文信息，连接信息，错误信息或绑定信息。客户端分配某些类型的句柄，通过定义良好的接口填充这些句柄中的一个或多个，并使用这些句柄向服务器发送请求。反过来，应用程序可以使用访问器函数访问句柄中包含的特定信息。

OCI库管理句柄层次结构。使用这些句柄封装OCI接口对应用程序开发人员有几个好处，包括：

  • 减少必须保留的服务器端状态信息，从而减少服务器端内存使用量

  • 通过消除对全局变量的需求，简化错误报告以及提供OCI变量访问和使用方式的一致性来提高生产率

  • 允许在不影响应用程序的情况下对底层结构进行更改

1.9用户认证和密码管理

OCI为应用程序开发人员提供用户身份验证和密码管

这有以下几种方式支持：

  • OCI允许单个OCI应用程序对多个用户进行身份验证和维护。

  • OCI使应用程序能够更新用户的密码，如果验证尝试返回过期的密码消息，这将特别有用。

OCI支持两种类型的登录会话：

  • 用于会话的登录功能，单个用户使用登录名和密码连接到数据库

  • 单个OCI应用程序通过将登录会话（用户登录到Oracle数据库时创建的会话）与用户会话（用户创建的所有其他会话）分开来对多个会话进行身份验证和维护的机制

还支持特权连接，例如SYSDBA ， SYSOPER ，代理身份验证，外部身份验证等。

1.10提高应用程序性能和可伸缩性的功能

OCI提供了多种功能扩展，以提高应用程序性能和可伸缩性。

通过减少所需的客户端到服务器往返次数来提高应用程序性能，并且通过减少必须在服务器端保留的状态信息量来提高可扩展性。其中一些功能包括：

  • 语句缓存通过缓存重复使用的可执行语句来提高性能

  • 客户端结果缓存以限制到数据库服务器的往返次数

  • 隐式预取SELECT语句结果集以消除描述往返，减少往返次数并减少内存使用

  • 消除打开和关闭光标往返

  • 支持多线程环境

  • 通过连接进行会话复用

  • 对各种配置的一致支持，包括标准的双层客户端/服务器配置，服务器到服务器事务协调和三层事务处理（TP） - 监视器配置

  • 对本地和全局事务的一致支持，包括对XA接口的TM_JOIN操作的支持

  • 通过提供在连接上跨用户集中连接，进程和会话的能力以及消除为全局事务的每个分支创建单独会话的需要，提高了可伸缩性

  • 允许应用程序对多个用户进行身份验证，并允许代表他们启动事务

175.1 DBMS_TSDP_PROTECT概述

使用DBMS_TSDP_PROTECT程序包创建透明的敏感数据保护策略，通过将策略与敏感类型相关联来配置保护，以及启用和禁用已配置的保护。

可以使用DBMS_TSDP_MANAGE包添加敏感类型。

175.2 DBMS_TSDP_PROTECT安全模型

所有程序都以调用者的权利执行。通常，安全性管理员应具有此程序包的EXECUTE特权。

175.5.1 ADD_POLICY过程

此过程创建TDSP策略。

句法

DBMS_TSDP_PROTECT.ADD_POLICY（policy_name IN VARCHAR2，security_feature IN PLS_INTEGER，policy_enable_options IN FEATURE_OPTIONS，policy_apply_condition IN POLICY_CONDITION DEFAULT TSDP $ default_condition）;

参数

表175-3 ADD_POLICY过程参数

参数 描述

POLICY_NAME

正在创建的策略的名称。此标识符的最大长度为M_IDEN 。这遵循Oracle命名约定。

security_feature

与策略关联的Oracle安全功能。允许值：

  • DBMS_TSDP_PROTECT.REDACT

  • DBMS_TSDP_PROTECT.VPD

  • DBMS_TSDP_PROTECT.UNIFIED_AUDIT

  • DBMS_TSDP_PROTECT.FINE_GRAINED_AUDIT

  • DBMS_TSDP_PROTECT.COLUMN_ENCRYPTION

policy_enable_options

使用与security_feature设置对应的参数值对进行初始化

policy_apply_condition

使用必须满足的属性 - 值对进行初始化，以便应用相应的policy_enable_options 。这是一个以Property作为键（ PLS_INTEGER ）的关联数组。

示例： example_policy_condition （ Property ）= property_value 。财产的允许值：

  • DBMS_TSDP_PROPERTY.DATATYPE

  • DBMS_TSDP_PROPERTY.LENGTH

  • DBMS_TSDP_PROPERTY.PARENT_SCHEMA

  • DBMS_TSDP_PROPERTY.PARENT_TABLE

使用说明

要创建TDSP策略，必须在匿名块中包含该过程，该块定义将使用策略和条件的安全功能的类型，以便在启用策略时进行测试。有关更多信息，请参见Oracle数据库安全指南 。

例子

创建策略PARTIAL_MASK_POLICY ：

DECLARE redact_feature_options DBMS_TSDP_PROTECT.FEATURE_OPTIONS; policy_conditions DBMS_TSDP_PROTECT.POLICY_CONDITIONS; BEGIN redact_feature_options（'expression'）：='SYS_CONTEXT（''USERENV''，''SESSION_USER''）=''APPUSER'''; redact_feature_options（'function_type'）：='DBMS_REDACT.PARTIAL'; redact_feature_options（'function_parameters'）：='STR，VVVVVVVVV，VVVVVVVVV，*，1,6'; policy_conditions（DBMS_TSDP_PROTECT.DATATYPE）：='VARCHAR2'; DBMS_TSDP_PROTECT.ADD_POLICY（'PARTIAL_MASK_POLICY'，DBMS_TSDP_PROTECT.REDACT，redact_feature_options，policy_conditions）;结束;

管理自动数据优化的策略

在使用SQL语句创建和更改表时，可以在行，段和表空间粒度级别为ADO指定策略。

通过为ADO指定策略，您可以在数据库中的不同存储层之间自动执行数据移动。这些策略还允许您为每个层指定不同的压缩级别，并控制数据移动的发生时间。

SQL CREATE和ALTER TABLE语句的ILM子句使您可以为ADO创建，删除，启用或禁用策略。 ILM策略子句确定压缩或存储分层策略，并包含其他子句，例如AFTER和ON子句，用于指定策略操作发生时的条件。创建表时，可以为ADO添加新策略。您可以更改表以添加更多策略或启用，禁用或删除现有策略。您可以将策略添加到整个表或表的分区。将ADO策略添加到表的表或分区时，只能为AFTER子句指定一种条件类型。ILM ADO策略被赋予系统生成的名称，例如P1 ， P2 ，等等到P n 。

段级别策略仅执行一次。策略成功执行后，它将被禁用，不会再次进行评估。但是，您可以再次明确启用该策略。行级策略继续执行，并且在成功执行后不会禁用。

可以使用关键字GROUP ， ROW或SEGMENT为一组相关对象或段或行级别指定ADO策略的范围。

可应用于组策略的压缩的默认映射是：

  • 堆表上的COMPRESS ADVANCED映射到索引的标准压缩和LOB段的LOW 。

  • 堆表上的COMPRESS FOR QUERY LOW / QUERY HIGH映射到索引的标准压缩和LOB段的MEDIUM 。

  • 堆表上的COMPRESS FOR ARCHIVE LOW / ARCHIVE HIGH映射到索引的标准压缩和LOB段的HIGH 。

压缩映射无法更改。GROUP只能应用于细分级别策略。存储分层策略仅适用于段级别，不能在行级别指定。

自动数据优化（ADO）支持具有INMEMORY ， INMEMORY MECOMPRESS和NO INMEMORY策略类型的内存中列存储（IM列存储）。

  • 要在In-Memory列存储中启用填充对象， INMEMORY在ADD POLICY子句中包含INMEMORY 。

  • 要增加IM列存储中对象的压缩级别， MEMCOMPRESS在ADD POLICY子句中包含INMEMORY MEMCOMPRESS 。

  • 要明确驱逐从IM列存储中受益最少的对象， INMEMORY在ADD POLICY子句中包含NO INMEMORY 。例如：

  • 如果HEAT_MAP初始化参数设置为ON且ENABLED参数设置为FALSE （ 0 ），则会收集热图统计信息，但ADO不会自动对统计信息执行操作。

  • 如果HEAT_MAP初始化参数设置为OFF并且ENABLED参数设置为TRUE （ 1 ），则不会收集热图统计信息，并且由于ADO不能依赖热图统计信息，因此ADO不执行任何操作。ADO的行为就像ENABLED设置为FALSE 。

EXECUTION MODE

EXECUTION MODE的值控制ADO是在在线还是离线模式下执行。默认为在线（ 2 ）。

EXECUTION INTERVAL

EXECUTION INTERVAL的值确定ADO启动后台评估的频率。默认值为15分钟。

JOB LIMIT

JOB LIMIT的值随时控制ADO作业的最大数量。最大并发ADO作业数计算为（ JOB LIMIT ）*（实例数）*（每个实例的CPU数）。默认值为2 。

POLICY TIME的值确定是否以秒或天指定ADO策略。1表示秒或0表示天（默认值）。

RETENTION TIME

RETENTION TIME的值指定在清除数据之前保留已完成的ADO任务的数据的时间长度。默认值为30天。

TBS PERCENT FREE

属性聚类是一种表级指令，它根据某些列的内容对物理接近的数据进行聚类。将逻辑上属于数据的数据存储在物理上接近的位置可以大大减少要处理的数据量，并可以提高工作负载中某些查询的性能。

13.1关于属性聚类

13.1.1聚类数据的方法

您可以使用以下方法聚类数据：

  • 基于定义了属性聚类的表的一列或多列进行聚类。

  • 基于与定义了属性聚类的表连接的一个或多个列进行聚类。基于连接列的聚类称为连接属性聚类 。表应通过主键 - 外键关系连接，但不必强制执行外键。

    由于星型查询通常限定维层次结构，因此如果事实表基于一个或多个维表的列（属性）进行聚类，则会很有用。使用连接属性群集，您可以将一个或多个维度表与事实表连接，然后按维度层次结构列对事实表数据进行群集。要在一个或多个维表的列上对事实表进行聚类，维表的连接必须位于维表的主键或唯一键上。在星型查询的上下文中加入属性聚类也称为层次聚类，因为表数据按维层次结构聚类 ，每个层次结构由一个有序的层次列列表组成（例如，构成location的nation ， state和city列）层次）。

    注意：与Oracle Table Clusters相比，join属性聚簇表不会存储来自同一数据库块中的一组表的数据。例如，考虑使用维表products连接的属性集群表sales 。sales表将只包含sales表中的行，但行的顺序将基于从products表连接的列的值。在数据移动，直接路径插入和CTAS操作期间将执行适当的连接。

13.1.2属性聚类的类型

属性 clustering是一个用户定义的表指令，它在表中的一列或多列上提供数据聚类。可以在创建或修改表时指定指令。

Oracle数据库提供以下类型的属性集群：

  • 线性排序的属性聚类

  • 具有交织排序的属性聚类

无论使用何种类型的属性聚类，您都可以基于单个表或通过连接多个表（连接属性聚类）来聚类数据。

13.1.2.1带线性排序的属性聚类

线性排序根据指定列的顺序存储数据。这是群集的默认类型。例如，表SALES的(prod_id, channel_id)列上的线性排序首先按prod_id对数据进行排序，然后按channel_id对数据进行排序。已排序的数据存储在磁盘上，群集列的数据非常接近。

可以在通过主键 - 外键关系连接的单个表或多个表上定义线性排序。

使用CLUSTERING ... BY LINEAR ORDER指令，根据指定列的顺序执行属性聚类。

基于列的线性排序的属性聚类最适用于以下场景：

  • 查询在单个表中指定CLUSTERING子句中包含的列的前缀

    例如，如果sales查询通常指定客户ID或客户ID和产品ID的组合，那么您可以使用列顺序cust_id ， prod_id在表中对数据进行集群。

  • CLUSTERING子句中使用的列具有可接受的基数级别

    在“ 属性 - 聚类表的优点 ”中描述的场景中可以获得的潜在数据减少与从列上的谓词获得的数据减少成正比地增加。

线性聚类与区域图相结合在I / O减少方面非常有效。

13.1.2.2具有交织排序的属性聚类

交错排序使用基于Z阶曲线拟合的特殊多维聚类技术。它将多个列属性值（多维数据点）映射到单个一维值，同时保留列值（数据点）的多维局部性。单个表或多个表支持交错排序。与线性排序不同，此方法不需要存在聚类定义的前导列，以实现“ 属性 - 聚簇表的优点 ”中描述的方案的I / O修剪优势。

列可以单独使用，也可以组合成列组。每个单独的列或列组将用于构成群集中的多维数据点之一。分组列由'（'..'）'括起来，并且必须遵循从最粗糙到最精细粒度级别的维度层次结构。例如，（ product_category ， product_subcategory ）。

使用CLUSTERING ... BY INTERLEAVED ORDER指令通过交错排序执行聚类。

交错聚类对于SQL操作最有利，因为多列上的谓词不同。对于维度模型的星型查询通常就是这种情况，其中查询谓词在维度表上并且谓词的数量不同。使用交叉连接属性群集在基于维度表中的列对事实表进行群集的环境中最常见。维度表中的列可能包含层次结构，例如，产品类别和子类别的层次结构。在这种情况下，事实表的聚类将在形成层次结构的维度列上发生。这就是为什么星型模式的连接属性聚类有时被称为分层聚类的原因。例如，如果sales查询指定来自不同维度的列，则可以根据这些维度中的列对sales表中的数据进行集群。

交错聚类与区域映射相结合在星型模式查询的I / O修剪中非常有效。此外，它使您能够使用区域映射为查询提供非常有效的I / O修剪，并增强压缩，因为相同的列值彼此接近并且可以轻松压缩。

13.1.4使用属性聚类的指南

以下是一些定义属性聚簇表时的注意事项：

  • 将属性聚类与区域映射结合使用可以促进区域修剪及其相关的I / O减少。

  • 考虑经常在中到低基数列上使用谓词查询的大型表。

  • 考虑经常被维度层次结构查询的事实表。

  • 对于分区表，请考虑包含与分区键相关的列（以便于区域映射分区修剪）。

  • 对于线性排序，请按前缀到后缀顺序列出列。

  • 将形成维度层次结构的列组合在一起。这构成了一个列组。在每个列组中，按最粗到最细的顺序列出列。

  • 如果有四个以上的维表，请包含最常用过滤器指定的维。将维数限制为两个或三个以获得更好的聚类效果。

  • 考虑在低到中的基数列上使用属性聚类而不是索引。

  • 如果维度表的主键由维度层次结构值组成（例如，主键由年，季度，月，日值组成），请将相应的外键设置为群集列而不是维度层次结构。

13.1.5属性聚簇表的优点

  • 消除与使用索引相关的存储成本

  • 与区域映射结合使用时，可以访问群集区域，而不是执行随机I / O或全表扫描

  • 与以下任何一项结合使用时，可提供I / O减少：

    • Oracle Exadata存储索引

    • Oracle内存最小/最大修剪

    • 区域地图

    属性集群基于用作过滤谓词的属性提供数据集群。由于Exadata存储索引和Oracle内存最小/最大修剪都跟踪存储在每个物理区域中的列的最小值和最大值，因此群集可减少访问数据所需的I / O.

    使用区域映射进行I / O修剪可以显着降低表扫描和索引扫描的I / O成本和CPU成本。

  • 根据星型模式中的维度列启用事实表的群集

    诸如传统表簇之类的技术不提供按其他表的列排序。在星型模式中，大多数查询限定维度表而不是事实表，因此按事实表列进行集群无效。Oracle数据库支持对维度表中的列进行群集。

  • 提高数据压缩率，从而间接提高了表扫描成本

    压缩可以得到改善，因为通过群集，具有相同值的群集列很可能在磁盘上彼此接近，因此数据库可以更容易地压缩它们。

  • 当属性聚类在索引选择条件上时，最小化索引范围扫描操作的表查找和单块I / O操作。

  • 在OLTP应用程序中为符合前缀的查询启用I / O减少，并使用具有线性顺序的属性聚类

  • 通过交叉排序，在聚类列的子集上启用I / O减少，以进行属性聚类

    如果表数据在多个列上排序（如在索引组织表中），则查询必须指定列的前缀以获得I / O节省。相反， BY INTERLEAVED表允许查询在以非前缀顺序指定多个表中的列时受益于I / O修剪。

13.1.7关于何时必须执行属性聚类的指定

执行群集可能很昂贵，因为它涉及在DML操作期间重组表和聚类数据。Oracle数据库不会对常规DML，传统插入，更新和合并强制执行数据集群。

可以以两种方式执行聚类。第一种是对表上的某些DML操作自动执行聚类。这是通过定义作为表元数据的一部分来触发群集的操作来完成的。第二个是明确指定必须按照“ 使用提示控制DML操作的属性群集 ”和“ 在DDL操作期间覆盖属性群集的表级设置”中所述执行群集 。在这种情况下，即使其元数据定义不包括群集，也可以对表执行群集。

作为表定义的一部分，您可以指定在触发以下操作时必须执行属性集群：

  • 直接路径插入操作

    将ON LOAD选项设置为YES以指定必须在直接路径插入操作期间执行属性群集。

  • 数据移动操作

    将ON DATA MOVEMENT选项设置为YES以指定必须在数据移动操作期间执行群集。这包括在线表重新定义和以下分区操作： MOVE ， MERGE ， SPLIT和COALESCE 。

13.2.4.1将属性聚类添加到现有表

当你创建一个具有聚类的表，默认情况下使用区域映射创建。但是，您可以使用WITHOUT ZONEMAP明确地阻止这种ZONEMAP 。这可以由于多种原因来完成，例如想要在群集列上创建区域映射以及与群集列相关的其他列，或者使用特定的区域映射存储选项而不是默认值。

使用ALTER TABLE ...ADD CLUSTERING命令将属性群集添加到当前不使用属性群集的现有表。

以下命令将属性集群添加到SALES事实表。修改后的表将使用基于连接的维度表CUSTOMERS和PRODUCTS交叉聚类。

ALTER TABLE sales ADD CLUSTERING sales JOIN customers ON（sales.cust_id = customers.cust_id）JOIN products ON（sales.prod_id = products.prod_id）BY INTERLEAVED ORDER（（prod_category，prod_subcategory），（country_id，cust_state_province，cust_city））YES ON在没有物质化的ZONEMAP的情况下加载数据运动;

将群集添加到表时，现有数据不会群集。要强制现有数据进行群集，您需要使用ALTER TABLE...移动表的内容ALTER TABLE...MOVE声明。您可以按分区执行此分区。

以下命令将sales表中的数据进行聚类：

ALTER TABLE销售MOVE PARTITION sales_1995 UPDATE INDEXES ALLOW CLUSTERING;
