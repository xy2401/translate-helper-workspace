6.4.1 Instant Client概述

Instant Client的打包方式使得通过消除对Oracle主页的需求，可以非常轻松地部署OCI，Oracle C ++调用接口（OCCI），开放式数据库连接（ODBC）和基于JDBC-OCI的客户应用程序。与在完整客户端安装上运行的同一应用程序相比，使用Instant Client的JDBC OCI应用程序的存储空间要求显着降低。Instant Client共享库仅占完整客户端安装所用磁盘空间的四分之一。

6.4.3 Instant Client的优点

Instant Client的好处如下：

  • 安装涉及复制较少数量的文件。

  • Oracle客户端所需的文件数和总磁盘存储量大大减少。

  • 使用Instant Client部署的应用程序不会丧失功能或性能。

  • 独立软件供应商很容易打包应用程序。

6.4.4 JDBC OCI Instant Client安装过程

可以通过从Oracle Universal Installer中选择Instant Client选项来安装Instant Client库。也可以从Oracle Technology Network网站下载Instant Client库。安装过程如下：

  1. 将Instant Client共享库和Oracle JDBC类库下载并安装到目录（例如instantclient 。

  2. 将库路径环境变量设置为步骤1中的目录。例如，在UNIX系统上，将LD_LIBRARY_PATH环境变量设置为instantclient 。在Microsoft Windows上，设置PATH环境变量以找到instantclient目录。

  3. 将JDBC类库的完整路径名添加到CLASSPATH环境变量中。

完成这些步骤后，您就可以运行JDBC OCI应用程序了。

使用Instant Client时，可以通过JDBC OCI应用程序的库路径环境变量访问OCI和JDBC共享库。在这种情况下，不依赖于ORACLE_HOME ，除了tnsnames.ora文件之外，JDBC OCI不需要ORACLE_HOME提供的其他任何代码和数据文件。

也可以通过选择Instant Client选项从Oracle Universal Installer安装Instant Client。Instant Client文件应始终安装在空目录中。与OTN安装一样，必须将LD_LIBRARY_PATH环境变量设置为Instant Client目录才能使用Instant Client。

如果通过选择“ Admin选项完成了完整的客户端安装，则还会安装Instant Client共享库。完整客户端安装中Instant Client共享库和JDBC类库的位置为：

在Linux或UNIX系统上：

  • libociei.so库位于$ORACLE_HOME/instantclient

  • libclnstsh.so.18.1 ， libocijdbc18.so和libnnz18.so位于$ORACLE_HOME/lib

  • JDBC类库位于$ORACLE_HOME/jdbc/lib

在Microsoft Windows上：

  • oraociei18.dll库位于ORACLE_HOME\instantclient

  • oci.dll ， ocijdbc18.dll和orannzsbb18.dll在ORACLE_HOME\bin

  • JDBC类库位于ORACLE_HOME\jdbc\lib

通过将这些文件复制到其他目录，设置库路径以找到此目录，并将JDBC类库的路径名添加到CLASSPATH环境变量，可以启用运行JDBC OCI应用程序以使用Instant Client。

注意：

  • 要提供本机XA功能，必须复制JDBC XA类库。在UNIX上，此库libheteroxa18.so位于ORACLE_HOME /jdbc/lib 。在Windows上，此库heteroxa18.dll位于ORACLE_HOME \bin 。

  • 必须从同一ORACLE_HOME复制所有库，并且必须将它们放在同一目录中。

  • 在混合平台（如Sparc64）上，如果JDBC OCI驱动程序需要使用Instant Client库，则必须从ORACLE_HOME /instantclient32目录复制libociei.so库。您必须从ORACLE_HOME /lib32目录复制JDBC OCI Instant Client所需的所有其他Sparc64库。

  • 只应在库路径环境变量中指定一组Oracle库。也就是说，如果您有多个包含Instant Client库的目录，则只应在库路径环境变量中指定一个此类目录。

  • 如果您的计算机上有Oracle主目录，则不应同时在库路径环境变量中具有ORACLE_HOME /lib和Instant Client目录，而不管它们在变量中的显示顺序如何。也就是说，只应在库路径环境变量中指定ORACLE_HOME /lib目录（用于非Instant Client操作）或Instant Client目录（用于Instant Client操作）中的一个。

6.4.5 Instant Client的使用

Instant Client是一种部署功能，应该用于运行生产应用程序。对于开发，需要完整安装才能访问演示程序等。通常，使用Instant Client的应用程序可以使用所有JDBC OCI功能，但Instant Client仅用于客户端操作。因此，服务器端外部过程无法使用Instant Client。

6.4.6关于修补Instant Client共享库

Instant Client是一种部署功能，因此重点是减少运行JDBC OCI应用程序所需的文件数量和大小。因此，在Instant Client部署中无法使用修补Instant Client共享库所需的所有文件。需要基于ORACLE_HOME安装来修补Instant Client共享库。opatch实用程序将负责修补Instant Client共享库。

注意：

在Microsoft Windows上，您无法修补共享库。

在ORACLE_HOME环境中应用修补程序后，将表6-1中列出的文件复制到即时客户端目录。

您可以为OCI，OCCI，JDBC和SQL * Plus生成Instant Client ZIP文件，而不是复制单个文件。然后，您可以将ZIP文件复制到目标计算机并解压缩。

opatch实用程序将ORACLE_HOME安装的修补信息存储在libclnstsh.so.18.1 。可以通过以下命令检索此信息：

genezi -v

6.4.8 OCI Instant Client的数据库连接名称

所有不需要ORACLE_HOME或TNS_ADMIN环境变量来查找配置文件的Oracle Net命名方法（例如tnsnames.ora或sqlnet.ora ）都使用Instant Client。特别是，可以使用以下格式指定连接字符串：

  • 形式为Thin的连接字符串：

    host ： port ： service_name

    例如：

    URL = “的jdbc：神谕：OCI：@ example.com：5521：ORCL”

  • 表单的SQL连接URL字符串：

    // host ：[ port ] [/ service_name ]

    例如：

    URL = “的jdbc：神谕：OCI：@ // example.com:5521/orcl”

  • 作为Oracle Net关键字 - 值对。例如：

    url =“jdbc：oracle：oci：@（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = localhost）（PORT = 5521））（CONNECT_DATA =（SERVICE_NAME = orcl）））”

如果设置了TNS_ADMIN环境变量，则需要TNS_ADMIN定位配置文件的命名方法才能继续工作。

也可以看看：

“Oracle数据库网络服务管理员指南” ，了解有关连接格式的更多信息

如果未设置TNS_ADMIN环境变量并且使用TNSNAMES条目（例如inst1 ，则必须设置ORACLE_HOME环境变量，并且配置文件应位于$ORACLE_HOME/network/admin目录中。

注意：

在这种情况下， ORACLE_HOME环境变量仅用于查找Oracle Net配置文件。客户端代码库的其他组件不使用ORACLE_HOME环境变量的值。

不支持空连接字符串。但是，使用空连接字符串的另一种方法是将UNIX系统上的TWO_TASK环境变量或Microsoft Windows上的LOCAL变量设置为tnsnames.ora条目或Oracle Net关键字 - 值对。如果将TWO_TASK或LOCAL设置为tnsnames.ora条目，则必须通过TNS_ADMIN或ORACLE_HOME设置加载tnsnames.ora文件。

例

请考虑数据库服务器上的listener.ora文件包含以下信息：

LISTENER =（ADDRESS_LIST =（ADDRESS =（PROTOCOL = tcp）（HOST = server6）（PORT = 5221）））SID_LIST_LISTENER =（SID_LIST =（SID_DESC =（SID_NAME = rdbms3）（GLOBAL_DBNAME = rdbms3.server6.com）（ORACLE_HOME = /家庭/ DBA / rdbms3 / ORACLE）））

您可以通过以下方式之一连接到此服务器：

url =“jdbc：oracle：oci：@（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = server6）（PORT = 5221））（CONNECT_DATA =（SERVICE_NAME = rdbms3.server6.com）））”

要么：

url =“jdbc：oracle：oci：@ // server6：5221 / rdbms3.server6.com”

或者，您可以将TWO_TASK环境变量设置为任何连接字符串并连接到数据库服务器，而不指定连接字符串以及sqlplus命令。例如，使用以下方法之一设置TWO_TASK环境：

setenv TWO_TASK“（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = server6）（PORT = 5221））（CONNECT_DATA =（SERVICE_NAME = rdbms3.server6.com）））”

要么：

setenv TWO_TASK //server6:5221/rdbms3.server6.com

现在，您可以使用以下URL连接到数据库服务器：

url =“jdbc：oracle：oci：@”

连接字符串也可以存储在tnsnames.ora文件中。例如，请考虑tnsnames.ora文件包含以下内容：

conn_str =（DESCRIPTION =（ADDRESS =（PROTOCOL = tcp）（HOST = server6）（PORT = 5221））（CONNECT_DATA =（SERVICE_NAME = rdbms3.server6.com）））

如果此tnsnames.ora文件位于/home/webuser/instantclient目录中，则可以按如下方式设置TNS_ADMIN环境变量（或Microsoft Windows上的LOCAL ）：

setenv TNS_ADMIN / home / webuser / instantclient

现在，您可以按如下方式连接：

url =“jdbc：oracle：oci：@conn_str”

注意：

TNS_ADMIN环境变量指定tnsnames.ora文件所在的目录。但是， TNS_ADMIN不指定tnsnames.ora文件的完整路径，而是指定目录。

如果此tnsnames.ora文件位于Oracle主目录的/network/server6/home/dba/oracle/network/admin目录中，则可以设置ORACLE_HOME环境变量，而不是使用TNS_ADMIN来定位tnsnames.ora文件。如下：

setenv ORACLE_HOME / network / server6 / home / dba / oracle

现在，您可以连接任何conn_str连接字符串，如前所述。

如果tnsnames.ora可以通过TNS_ADMIN或ORACLE_HOME ，那么TWO_TASK可以设置为：

setenv TWO_TASK conn_str

然后，您可以使用以下URL连接：

url =“jdbc：oracle：oci：@”

6.4.9 OCI Instant Client的环境变量

ORACLE_HOME环境变量不再确定NLS，CORE和错误消息文件的位置。仅OCI应用程序不需要设置ORACLE_HOME环境变量。但是，如果设置了变量，则它不会对OCI驱动程序的操作产生影响。OCI驱动程序始终从数据共享库中获取其数据。如果数据共享库不可用，则仅使用ORACLE_HOME环境变量并假定完整客户端安装。即使不需要设置ORACLE_HOME环境变量，如果已设置，则必须将其设置为标识目录的有效操作系统路径名。

使用Instant Client时，将忽略环境变量ORA_NLS10和ORA_NLSPROFILES33 。

如果未设置ORA_TZFILE变量，则Instant Client将使用数据共享库中较大的timezlrg_n.dat文件，这是默认设置。如果要从数据共享库中使用较小的timezone_n.dat文件，则将ORA_TZFILE环境变量设置为文件名，而不包含任何绝对路径名或相对路径名。那是：

在UNIX系统上：

setenv ORA_TZFILE timezone_ n .dat

在Microsoft Windows上：

设置ORA_TZFILE timezone_ n .dat

在上面的示例中， n是时区数据文件版本号。

如果OCI驱动程序由于数据共享库不可用而未使用Instant Client，则ORA_TZFILE变量（如果已设置）将命名完整路径名，就像在以前的Oracle数据库版本中一样。

如果使用TNSNAMES条目，则如前所述， TNS_ADMIN目录必须包含TNSNAMES配置文件，如果未设置TNS_ADMIN ，则ORACLE_HOME/network/admin目录必须包含Oracle Net Services配置文件。

监控Oracle RAC和Oracle Clusterware

Oracle RAC环境中的自动工作负载存储库

您可以使用自动工作负载存储库来监视与Oracle RAC数据库相关的性能统计信息。

自动工作负载存储库（AWR）每小时自动生成一次性能数据快照，并在工作负载存储库中收集统计信息。在Oracle RAC环境中，每个AWR快照都捕获来自群集中所有活动实例的数据。从同一时间点捕获每个快照集的数据。AWR将所有实例的快照数据存储在同一个表中，并且数据由实例限定符标识。例如， BUFFER_BUSY_WAIT统计信息显示每个实例上的缓冲区等待数。AWR不存储从整个群集聚合的数据。换句话说，为每个单独的实例存储数据。

使用自动数据库诊断监视器（ADDM），您可以分析AWR收集的有关Oracle数据库可能出现的性能问题的信息。ADDM从集群范围的角度提供性能数据，从而使您能够在全球范围内分析性能。在Oracle RAC环境中，ADDM可以使用从所有实例收集的数据分析性能，并以不同的粒度级别呈现它，包括：

  • 分析整个集群

  • 分析特定的数据库实例

  • 分析数据库实例的子集

要执行这些分析，可以在ADDM for Oracle RAC模式下运行ADDM Advisor以执行整个群集的分析;在本地ADDM模式下分析单个实例的性能;或者在部分ADDM模式下分析实例的子集。通过Oracle Enterprise Manager中的Advisor Central或通过DBMS_ADVISOR和DBMS_ADDM PL / SQL包使用顾问程序框架激活ADDM分析。

Oracle RAC的ASH报告概述

灰是Oracle数据库自我管理框架不可或缺的一部分，非常有用诊断Oracle RAC环境中的性能问题。ASH报告统计提供了详细信息 Oracle数据库会话活动。Oracle数据库记录有关的信息所有活动Oracle RAC实例的活动会话，并将此数据存储在系统全局区域（SGA）。连接到数据库并使用CPU的任何会话都被视为活动会话。例外情况是等待属于该事件的事件的会话闲暇等课。

ASH报告通过仅捕获有关活动会话的信息来呈现可管理的数据集。数据量与正在执行的工作直接相关，而不是与系统上允许的会话数相关。

在指定持续时间内收集的ASH统计数据可以放入ASH报告中。每个ASH报告分为多个部分，以帮助您识别ADDM分析中未出现的短期性能问题。特定于Oracle RAC的两个ASH报告部分是Top Cluster Events和Top Remote Instance，如下两节所述。

适用于Oracle RAC的ASH报告：热门群集事件

ASH报告 Top Cluster Events部分是特定于Oracle RAC的Top Events报告的一部分。 “热门群集事件”报告列出了占群集等待类事件中会话活动百分比最高的事件以及受影响实例的实例号。您可以使用此信息来识别哪些事件和实例导致高比例的群集等待事件。

适用于Oracle RAC的ASH报告：顶级远程实例

ASH报告 Top Remote Instance部分是特定于Oracle RAC的Top Load Profile报告的一部分。 “顶级远程实例”报告显示群集等待事件以及占会话活动最高百分比的实例的实例编号。您可以使用此信息来标识导致扩展群集等待期的实例。

Oracle RAC等待事件

分析和解释导致会话等待的原因是确定花费时间的重要方法。

在Oracle RAC中，等待时间归因于反映请求的确切结果的事件。例如，当实例上的会话在全局高速缓存中查找块时，它不知道它是否将接收由另一个实例高速缓存的数据，或者它是否将接收从磁盘读取的消息。全局缓存的等待事件传达精确信息并等待全局缓存块或消息：

  • 总结在一个名为Cluster Wait Class的更广泛的类别中

  • 暂时由占位符事件表示，该事件在等待块时处于活动状态，例如：

    • gc当前阻止请求

    • gc cr阻止请求

  • 在知道请求结果时归因于精确事件，例如：

    • gc当前块3路

    • gc当前块忙

    • gc cr block grant 2-way

总之，Oracle RAC的等待事件传达了对性能分析有价值的信息。它们用于自动数据库诊断监视器（ADDM），以便精确诊断缓存融合的效果。

分析Oracle RAC中缓存融合的效果

访问全局缓存中的块并保持一致性的效果表示为：

  • 该 current和cr块的全局高速缓存服务（GCS）统计信息，例如， gc current blocks received gc cr blocks received ，等等

  • GCS等待事件，对于gc current block 3-way ， gc cr grant 2-way ，等等

缓存融合传输的响应时间由物理互连组件强加的消息传递和处理时间决定 IPC协议和 GCS协议。除偶尔的日志写入外，它不受磁盘I / O因素的影响。缓存融合协议不需要I / O来保证数据文件 缓存一致性和Oracle RAC固有地不会导致比非聚簇实例更多的磁盘I / O.

使用GCS和GES统计分析性能

您可以通过识别所有实例经常使用（ 热 ）的数据块和对象来监视GCS性能。

某些块上的高并发性可以通过GCS等待事件和时间来识别。

gc current block busy等待事件表示对高速缓存数据块的访问被延迟，因为它们在远程或本地高速缓存中忙碌。这可能是由以下任何原因引起的：

  • 这些街区被固定了

  • 这些街区被会议所阻止

  • 这些块被远程实例上的日志写入延迟

  • 同一个实例上的会话已经访问了一个块，该块在实例之间转换，当前会话需要在它后面等待（例如， gc current block busy ）

使用V$SESSION_WAIT视图标识具有争用的对象和数据块。GCS等待事件分别包含p1和p2中块请求的文件和块号。

添加了一个额外的段统计信息gc buffer busy ，可以快速确定繁忙的对象，而无需查询前面提到的V$SESSION_WAIT视图。

AWR基础结构提供活动会话历史记录的视图，该视图还可用于跟踪最近的等待事件及其参数。因此，它适用于热块分析。AWR和Statspack使用的大多数报告工具都包含对象统计信息和群集等待类类别，因此前面提到的视图的采样在很大程度上是不必要的。

注意：

Oracle建议使用ADDM和AWR。但是，Statspack可用于向后兼容。Statspack仅提供报告。您必须在级别7运行Statspack以收集与块争用和段块等待相关的统计信息。

建议在AWR基础架构收集的快照数据上运行ADDM，以获得对全局缓存影响的总体评估。该通报还将识别繁忙对象和SQL最高群集等待时间。

使用GCS统计分析缓存融合传输影响

描述如何通过识别频繁读取和修改的对象以及远程访问强加的服务时间来监视GCS性能。

等待块到达可能构成响应时间的重要部分，就像从磁盘读取可能增加块访问延迟一样，只有缓存融合传输通常比磁盘访问延迟更快。

以下等待事件表明远程高速缓存的块已经发送到本地实例，而没有繁忙，固定或需要日志刷新：

  • gc当前阻止2路

  • gc当前块3路

  • gc cr阻挡2路

  • gc cr block 3-way

gc current blocks received和gc current blocks received到的gc cr blocks received的对象统计信息可以快速识别活动实例共享的索引和表。如前所述，创建ADDM分析通常会指向可能受实例间争用影响的SQL语句和数据库对象。

前面列表中提到的事件的平均等待时间的任何增加都可能由以下事件引起：

  • 高负载：CPU短缺，长时间队列，调度延迟

  • 配置错误：使用公共而非私有互连来实现消息和阻塞流量

如果平均等待时间是可接受的并且不能诊断互连或负载问题，则等待的累积时间通常可归因于需要调整以最小化所访问的块数的一些SQL语句。

V$SQLAREA的列CLUSTER_WAIT_TIME表示单个SQL语句对全局缓存事件的等待时间，并将标识可能需要调整的SQL。

基于等待事件的响应时间分析

AWR和Statspack报告或动态性能视图中报告的显示总时间较长的大多数全局缓存等待事件都是正常的，并且可能将自己显示为没有实际指示问题的顶级数据库时间消费者。

本节介绍在解释性能数据时应注意的频繁等待事件。

如果用户响应时间增加并且等待的时间比例高于全局缓存，则应确定原因。大多数报告都包括按总时间百分比排序的事件明细。

从ADDM报告开始是很有用的，该报告分析了常规收集的性能统计信息，并指出了等待时间最多的对象和SQL，然后转到AWR生成的更详细的报告。 STATSPACK。

Oracle RAC的等待事件包括以下类别：

  • 与块相关的等待事件

  • 与消息相关的等待事件

  • 与争用相关的等待事件

  • 与负载相关的等待事件

与块相关的等待事件

主要的等待事件与块相关的等待是：

  • gc当前阻止2路

  • gc当前块3路

  • gc cr阻挡2路

  • gc cr block 3-way

与块相关的等待事件统计信息表明接收到的块是2路或3路消息的结果，也就是说，块是从需要1条消息和1条传输的资源主机发送的，或者是转发到发送它的第三个节点，需要2个消息和1个块传输。

与消息相关的等待事件

主要的等待事件与消息相关的等待是：

  • gc当前授予2路

  • gc cr grant 2-way

与消息相关的等待事件统计信息表明未收到任何阻止，因为它未在任何实例中缓存。而是给出了全局授权，使请求实例能够从磁盘读取块或修改它。

如果这些事件消耗的时间很长，那么可以假设经常使用的SQL会导致大量磁盘I / O（在cr grant ）或者工作负载会插入大量数据并需要查找并经常格式化新块（在当前授权的情况下）。

与争用相关的等待事件

主要的等待事件与争用相关的等待是：

  • gc当前块忙

  • gc cr块忙

  • gc缓冲区繁忙获取/释放

与争用相关的等待事件统计信息表明收到了一个由另一个节点上的会话固定的块，由于更改尚未刷新到磁盘或由于高并发性而被延迟，因此无法立即发送。当会话已经启动了缓存融合操作时，缓冲区也可能在本地忙，并且当同一节点上的另一个会话正在尝试读取或修改相同数据时，缓冲区正在等待其完成。在全局高速缓存中交换的块的高服务时间可能会加剧争用，这可能是由对相同数据的频繁并发读写访问引起的。

gc current block busy和gc cr block busy等待事件表示发出请求的本地实例未立即接收到当前或一致的读取块。这些事件名称中的忙碌一词表示在远程实例上发送了块。例如，如果Oracle数据库尚未为块更改日志文件编写重做，则无法立即传送块。

与block busy等待事件相比， gc buffer busy事件表示Oracle数据库无法立即授予对存储在本地缓冲区高速缓存中的数据的访问权限。这是因为缓冲区上的全局操作正在等待，并且操作尚未完成。换句话说，缓冲区正忙，所有其他尝试访问本地缓冲区的进程必须等待才能完成。

gc buffer busy事件的存在也意味着存在块争用，这导致多个访问本地块的请求。Oracle数据库必须排队这些请求。Oracle数据库处理队列所需的时间长度取决于块的剩余服务时间。服务时间受任何网络延迟添加的处理时间，远程和本地实例的处理时间以及等待队列的长度的影响。

当提醒这些特定等待具有高影响的性能问题时，应考虑平均等待时间和总等待时间。通常，可以发现互连或加载问题或针对大型共享工作集的SQL执行是根本原因。

与负载相关的等待事件

主要的等待事件与负载相关的等待是：

  • gc当前区块拥挤

  • gc cr阻塞拥挤

与负载相关的等待事件表明GCS中发生了处理延迟，这通常是由高负载，CPU饱和引起的，并且必须通过额外的CPU，负载平衡，卸载处理到不同时间或者新的群集节点。对于所提到的事件，等待时间包括从发起块请求之后会话开始等待直到块到达的整个往返。

10.5在OCI中操纵字符串

字符串操作支持两种类型的数据结构：

  • 原生字符串

目的

lmsgen实用程序将基于文本的消息文件（ .msg ）转换为二进制格式（ .msb ），以便用户提供的Oracle消息和OCI消息可以返回到所需语言的OCI函数。

服务器使用的消息存储在二进制格式文件中，这些文件位于$ORACLE_HOME/product_name/mesg目录中，或者等效于操作系统。使用以下文件名约定，可以存在这些文件的多个版本，每个受支持的语言一个版本：

<PRODUCT_ID> <language_abbrev> .msb

例如，包含法语服务器消息的文件称为oraf.msb ，因为ORA是产品ID（< product_id >）， F是法语的语言缩写（< language_abbrev >）。product_name的值是rdbms，因此它位于$ORACLE_HOME/rdbms/mesg目录中。

句法

LMSGEN text_file product facility [ language ] [-i indir ] [-o outdir ]
  • text_file是消息文本文件。
  • product是product的名称。
  • facility是facility的名称。
  • language是与NLS_LANG参数中指定的语言对应的可选消息语言。如果消息文件未使用语言正确标记，则需要语言参数。
  • indir是指定文本文件位置的可选目录。
  • outdir是指定输出文件位置的可选目录。

输出（ .msb ）文件将在$ORACLE_HOME/product/mesg/目录下生成。

短信文件

短信文件必须遵循以下准则：

  • 以/和//开头的行被视为内部注释，并被忽略。

  • 要使用特定语言标记消息文件，请包含类似于以下内容的行：

＃CHARACTER_SET_NAME = Japanese_Japan。JA16EUC

  • 每条消息包含三个字段：

message_number ， warning_level ， message_text
  • 消息编号在消息文件中必须是唯一的。
  • 目前未使用警告级别。使用0 。
  • 消息文本不能超过511个字节。

以下示例显示了Oracle消息文本文件：

/ Oracle版权所有（c）2006。版权所有。/这是测试us7ascii消息文件＃CHARACTER_SET_NAME = american_america.us7ascii / 00000,00000，“导出未成功终止\ n”00003,00000，“找不到段的存储定义（％lu，％lu）”

选择网关的系统标识符

该网关系统标识符 （SID）是标识网关实例的字母数字字符串。对于要访问的每个SQL Server数据库，您需要一个网关实例，因此需要一个网关SID。SID用作初始化参数文件的文件名的一部分。默认SID是 dg4msql 。

您可以定义网关SID，但使用默认的dg4msql更容易，因为您不需要更改初始化参数文件名。但是，如果要访问两个SQL Server数据库，则需要两个网关SID，一个用于网关的每个实例。如果您只有一个SQL Server数据库并且希望有时使用一组网关参数设置来访问它，而其他时候使用不同的网关参数设置，那么您将需要多个网关SID用于单个SQL Server数据库。

创建数据库链接

连接到Oracle数据库的任何Oracle客户端都可以通过网关访问SQL Server数据。Oracle客户端和Oracle数据库可以驻留在不同的计算机上。网关仅接受来自Oracle数据库的连接。

首次在Oracle会话中使用时，通过数据库链接建立与网关的连接。在此上下文中，连接是指Oracle数据库与网关之间的连接。在Oracle会话结束之前，连接将保持建立状态。另一个会话或用户可以访问相同的数据库链接，并获得与网关和SQL Server数据库的不同连接。

解释输出

表plsql_profiler_data为生成代码的源单元的每一行包含一行。行＃值指定哪个源行。如果该行存在，并且该行中的total_occur值> 0，则执行与该行关联的一些代码。如果该行存在且total_occur值为0，则不会执行与该行关联的代码。如果表中不存在该行，则不会为该行生成任何代码，因此不应在报告中提及该行

如果单个语句的源是在一行上，则为该语句生成的任何代码都将归于该行号。（在某些情况下，例如简单的声明，或者由于优化，不需要代码）。要获取覆盖率信息，应使用PLSQL_OPTIMIZE_LEVEL=1编译单位。

如果语句跨越多行，则为该语句生成的任何代码都将归因于该范围中的某些行，但不保证该范围中的每一行都将具有归属于它的代码。在这种情况下，行＃值集合中将存在间隙。特别是，多行SQL相关语句可能看起来在一行（通常是第一行）。这是因为PL / SQL将处理后的文本文本传递给SQL引擎;因此，就PL / SQL而言，整个SQL语句是一个不可分割的操作。

当多个语句位于同一行时，探查器将组合每个语句的出现次数。如果一条线嵌入了控制流，这可能会造成混淆。例如，如果'then ...'和'else ...'在同一行，则无法确定是否采用了'then'或'else' 。

通常，如果每个语句都在自己的行上，则最容易解释分析器和覆盖率报告。

Oracle Data Guard可确保企业数据的高可用性，数据保护和灾难恢复。

Oracle Data Guard提供了一整套服务，可以创建，维护，管理和监视一个或多个备用数据库，从而使生产Oracle数据库能够在灾难和数据损坏中幸免于难。Oracle Data Guard将这些备用数据库维护为生产数据库的副本。然后，如果生产数据库由于计划内或计划外中断而变得不可用，则Oracle Data Guard可以将任何备用数据库切换到生产角色，从而最大限度地减少与中断相关的停机时间。Oracle Data Guard可与传统的备份，还原和集群技术一起使用，以提供高级别的数据保护和数据可用性。其他Oracle功能（如Oracle Streams和Oracle GoldenGate）也使用Oracle Data Guard传输服务，以便将重做从源数据库高效可靠地传输到一个或多个远程目标。

1.1 Oracle Data Guard配置

Oracle Data Guard配置可以包含一个主数据库和最多三十个目标。

Oracle Data Guard配置的成员由Oracle Net连接，可能在地理上分散。只要Oracle Data Guard配置的成员可以相互通信，它们的位置就没有限制。例如，您可以在与主数据库相同的数据中心中拥有备用数据库，以及在另一个数据中心中使用两个备用数据库。

您可以使用SQL命令行界面或Oracle Data Guard代理接口管理主数据库和备用数据库。代理提供命令行界面（DGMGRL）和集成在Oracle Enterprise Manager云控制中的图形用户界面。

1.1.1主数据库

Oracle Data Guard配置包含一个以主要角色运行的生产数据库（也称为主数据库）。

主数据库是大多数应用程序访问的数据库。

主数据库可以是单实例Oracle数据库，也可以是Oracle Ral Application Clusters（Oracle RAC）数据库。

1.1.2备用数据库

备用数据库是主数据库的事务一致性副本。

使用主数据库的备份副本，您可以创建多达30个备用数据库并将它们合并到Oracle Data Guard配置中。Oracle Data Guard通过从主数据库传输重做数据然后将重做应用于备用数据库来自动维护每个备用数据库。

与主数据库类似，备用数据库可以是单实例Oracle数据库，也可以是Oracle RAC数据库。

备用数据库的类型如下：

  • 物理备用数据库

    提供主数据库的物理上相同的副本，其中磁盘上的数据库结构在逐个块的基础上与主数据库相同。数据库模式（包括索引）是相同的。物理备用数据库通过Redo Apply与主数据库保持同步，Redo Apply恢复从主数据库接收的重做数据并将重做应用于物理备用数据库。

    从Oracle Database 11g第1版（11.1）开始，物理备用数据库可以在打开以进行只读访问时接收和应用重做。因此，可以同时使用物理备用数据库进行数据保护和报告。

    此外，从Oracle Database 11g第2版（11.2.0.1）开始，可以使用物理备用数据库以滚动方式安装符合条件的一次性修补程序，修补程序集更新（PSU）和关键修补程序更新（CPU）。有关此功能的更多信息，请参阅http://support.oracle.com的My Oracle Support说明1265700.1。

  • 逻辑备用数据库

    包含与生产数据库相同的逻辑信息，尽管数据的物理组织和结构可能不同。逻辑备用数据库通过SQL Apply与主数据库保持同步，SQL Apply将从主数据库接收的重做数据转换为SQL语句，然后在备用数据库上执行SQL语句。

    逻辑备用数据库的灵活性使您可以升级Oracle数据库软件（补丁集和新的Oracle数据库版本）并以滚动方式执行其他数据库维护，几乎不会出现停机。从Oracle Database 11g开始，瞬态逻辑数据库滚动升级过程也可以与现有物理备用数据库一起使用。

  • 快照备用数据库

    快照备用数据库是完全可更新的备用数据库。

    与物理或逻辑备用数据库一样，快照备用数据库从主数据库接收和归档重做数据。与物理或逻辑备用数据库不同，快照备用数据库不应用它接收的重做数据。在首先丢弃对快照备用数据库进行的任何本地更新之后，快照备用数据库转换回物理备用数据库之前，不应用快照备用数据库接收的重做数据。

    快照备用数据库最适用于需要物理备用数据库的临时可更新快照的方案。例如，您可以使用Oracle Real Application Testing选项捕获主数据库上的数据库工作负载，然后在快照备用数据库上重放它以进行测试。由于快照备用数据库接收的重做数据在转换回物理备用数据库之前不会应用，因此从主数据库故障中恢复所需的时间与需要应用的重做数据量成正比。

1.4 Oracle Data Guard保护模式

Oracle Data Guard提供三种不同的数据保护模式。

在某些情况下，无论情况如何，企业都不能丢失数据。在其他情况下，数据库的可用性可能比在多次失败的不太可能的情况下的任何潜在数据丢失更重要。最后，某些应用程序始终需要最大的数据库性能，因此如果任何组件发生故障，可以容忍少量数据丢失。以下是可用于以下每种情况的保护模式的简要说明：

最高可用性

此保护模式可提供最高级别的数据保护，而不会影响主数据库的可用性。使用Oracle Data Guard，在恢复这些事务所需的所有重做数据已在内存中接收或写入至少一个同步备用数据库上的备用重做日志（取决于配置）之前，事务不会提交。如果主数据库无法将其重做流写入至少一个同步备用数据库，则它的运行方式就像它处于最高性能模式一样，以保持主数据库可用性，直到它再次能够将其重做流写入同步备用数据库。

此保护模式可确保零数据丢失，但在某些双重故障的情况下，例如备用数据库发生故障后主数据库发生故障。

最高性能

这是默认保护模式。它提供了最高级别的数据保护，而不会影响主数据库的性能。这是通过允许事务在这些事务生成的所有重做数据写入在线日志后立即提交来实现的。重做数据也会写入一个或多个备用数据库，但这是在事务承诺方面异步完成的，因此主数据库性能不会受到将重做数据写入备用数据库的延迟的影响。

与最大可用性模式相比，此保护模式提供的数据保护略少，对主数据库性能的影响最小。

最大保护

此保护模式可确保在主数据库发生故障时不会发生数据丢失。为了提供这种级别的保护，在事务提交之前，必须将恢复事务所需的重做数据写入至少一个同步备用数据库的联机重做日志和备用重做日志。为确保不会发生数据丢失，如果主数据库无法将其重做流写入至少一个同步备用数据库，则主数据库将关闭而不是继续处理事务。

所有三种保护模式都要求使用特定的重做传输选项将重做数据发送到至少一个备用数据库。

1.6 Oracle Data Guard和补充技术

Oracle数据库提供了几种独特的技术，可以补充Oracle Data Guard，与单独使用任何一种解决方案相比，可以使业务关键系统保持更高的可用性和数据保护。

以下列表总结了一些Oracle高可用性技术：

  • Oracle Real Application Clusters（Oracle RAC）

    Oracle RAC允许通过互连链接的多个独立服务器共享对Oracle数据库的访问，从而在故障期间提供高可用性，可伸缩性和冗余。Oracle RAC和Oracle Data Guard一起提供系统级，站点级和数据级保护的优势，从而实现高级别的可用性和灾难恢复，而不会丢失数据：

    • Oracle RAC通过提供从故障中快速自动恢复（例如节点故障和实例崩溃）来解决系统故障。它还为应用程序提供了更高的可伸缩性。

    • Oracle Data Guard通过不共享磁盘的事务一致的主数据库和备用数据库解决站点故障和数据保护问题，从而实现从站点灾难中恢复和数据损坏。

    使用Oracle RAC和Oracle Data Guard的许多不同体系结构都是可能的，具体取决于本地和远程站点的使用以及节点的使用以及逻辑和物理备用数据库的组合。请参阅Oracle Data Guard和Oracle Real Application Clusters for Oracle RAC和Oracle Data Guard集成。

  • Oracle Real Application Clusters One Node（Oracle RAC One Node）

    Oracle RAC One Node为非群集数据库提供增强的高可用性，从而保护它们免受计划内和计划外停机的影响。Oracle RAC One Node提供以下内容：

    • 永远在线的非群集数据库服务

    • 更好地整合数据库服务器

    • 增强服务器虚拟化

    • 完整Oracle RAC的低成本开发和测试平台

    此外，Oracle RAC One Node有助于整合数据库存储，标准化您的数据库环境，并在必要时使您能够升级到完整的多节点Oracle RAC数据库，而无需停机或中断。

    从Oracle Database 11g第2版（11.2.0.2）开始，Oracle Data Guard和Oracle Data Guard代理与Oracle Real Application Clusters One Node（Oracle RAC One Node）完全集成。

  • 闪回数据库

    闪回数据库功能可以从逻辑数据损坏和用户错误中快速恢复。通过允许您及时闪回，可以再次访问可能已被错误更改或删除的先前版本的业务信息。此功能：

    • 无需恢复备份并前滚更改，直至错误或损坏。相反，闪回数据库可以将Oracle数据库回滚到以前的时间点，而无需还原数据文件。

    • 提供延迟重做应用程序的替代方法，以防止用户错误或逻辑损坏。因此，备用数据库可以与主数据库更紧密地同步，从而减少故障转移和切换时间。

    • 在故障转移后，无需完全重新创建原始主数据库。可以将故障主数据库闪回到故障转移之前的某个时间点，并将其转换为新主数据库的备用数据库。

    有关闪回数据库的信息，请参阅“ Oracle数据库备份和恢复用户指南” ，并为存档重做日志文件的应用指定时间延迟，以获取有关重做数据应用的信息。

  • 恢复管理器（RMAN）

    RMAN是一个Oracle实用程序，可简化备份，还原和恢复数据库文件。与Oracle Data Guard一样，RMAN是Oracle数据库的一项功能，不需要单独安装。Oracle Data Guard与RMAN完美集成，允许您：

    • 使用Recovery Manager DUPLICATE命令从主数据库的备份创建备用数据库。

    • 在物理备用数据库而不是生产数据库上进行备份，从而减轻生产数据库上的负载并在备用站点上有效使用系统资源。此外，可以在物理备用数据库应用重做时进行备份。

    • 通过在执行备份后自动删除用于输入的归档重做日志文件来帮助管理归档重做日志文件。

    请参阅使用Recovery Manager创建备用数据库 。

  • Oracle全球数据服务（GDS）

    Oracle全球数据服务（GDS）将Oracle RAC服务模型应用于全局分布式数据库池，为一组提供公共服务的复制数据库提供动态负载平衡，故障转移和集中服务管理。该组数据库可以包括通过Oracle Data Guard，Oracle GoldenGate或任何其他复制技术互连的Oracle RAC和单实例Oracle数据库。

    GDS与Oracle Data Guard代理集成在一起。这允许在Oracle Data Guard代理配置中发生角色转换时，根据需要自动启动和停止特定于角色的全局服务。

    GDS允许为全局服务指定复制滞后限制。如果在给定副本超过滞后限制，则在该副本上暂时停止全局服务，并将新客户端请求路由到满足滞后限制的副本。当复制延迟小于滞后限制时，全局服务将在原始副本上自动重新启动。

属于分片组的碎片通常位于同一数据中心。整个分片组可以完全复制到相同或不同数据中心的一个或多个分片组。

1.8 Oracle Data Guard优点摘要

Oracle Data Guard提供高效，全面的灾难恢复和高可用性解决方案。

Oracle Data Guard提供以下好处：

  • 高可用性

    Oracle Data Guard易于管理的切换和故障切换功能允许主数据库和备用数据库之间的角色转换，从而最大限度地减少主数据库因计划内和计划外中断而停机的时间。

  • 完整的数据保护

    即使面对无法预料的灾难，Oracle Data Guard也可确保零数据丢失。备用数据库提供了防止所有类型的计划外中断的安全措施，包括数据损坏和管理错误。由于从主数据库接收的重做数据在备用数据库中验证，因此主数据库中可能发生的物理损坏不会传播到备用数据库。在备用数据库上执行的其他验证还可以防止逻辑块内损坏和丢失写入损坏传播到备用数据库。同样，存储管理员意外删除文件等管理错误也不会传播到备用数据库。物理备用数据库还可用于通过延迟重做应用或使用闪回数据库回退备用数据库并提取数据的良好副本来防止用户错误。

  • 有效利用系统资源

    使用从主数据库接收的重做数据更新的备用数据库表可用于其他任务，如备份，报告，汇总和查询，从而减少执行这些任务所需的主数据库工作负载，从而节省宝贵的CPU和I / O周期。

  • 灵活的数据保护，以平衡可用性和性能要求

    Oracle Data Guard提供最大程度的保护，最大可用性和最高性能模式，以帮助企业平衡数据可用性与系统性能要求。

  • 自动间隙检测和分辨率

    如果主数据库与一个或多个备用数据库之间的连接丢失（例如，由于网络问题），则无法将主数据库上生成的重做数据发送到这些备用数据库。重新建立连接后，Oracle Data Guard会自动检测丢失的归档重做日志文件（称为间隙），然后Oracle Data Guard会自动将丢失的归档重做日志文件传输到备用数据库。备用数据库与主数据库同步，无需DBA的手动干预。

  • 集中和简单的管理

    Oracle Data Guard代理提供图形用户界面和命令行界面，以在Oracle Data Guard配置中跨多个数据库自动执行管理和操作任务。代理还监视单个Oracle Data Guard配置中的所有系统。

  • 与Oracle数据库集成

    Oracle Data Guard是Oracle数据库企业版的一项功能，不需要单独安装。

  • 自动角色转换

    启用快速启动故障转移后，Oracle Data Guard代理会在主站点发生灾难时自动故障转移到同步备用站点，而不需要DBA进行干预。此外，还会自动通知应用程序角色转换。

自动工作负载存储库（AWR）

每个Oracle数据库中都存在的内置存储库。Oracle Database定期对其所有重要统计信息和工作负载信息进行快照，并将其存储在AWR中。

您可以专门定义可以运行的服务器以及服务可以在数据库中运行的数据库。

缓存融合

Oracle RAC中的无盘高速缓存一致性机制，它直接从保留实例的内存高速缓存向请求实例的内存高速缓存提供块的副本。

基数

要在正常操作期间运行的数据库实例数。

CDB

客户端集群

一个群集，用于向服务器群集通告其名称。

簇

多个互连的计算机或服务器，看起来好像是最终用户和应用程序的一个服务器。

群集配置策略

一个文档，其中只包含系统中定义的每个服务器池的一个定义。

群集配置策略集

定义集群中配置的所有服务器池的名称并包含一个或多个配置策略的文档。

集群数据库

Oracle RAC数据库的通用术语。

集群文件系统

分布式文件系统，是一组服务器，协作为其客户端提供高性能服务。群集文件系统软件处理将请求分发到存储群集组件。

群集就绪服务守护程序（CRSD）

主要Oracle Clusterware进程，执行高可用性恢复和管理操作，例如维护OCR。还管理应用程序资源并以root用户（或基于Mac OS X的系统上的admin组中的用户）运行，并在失败时自动重新启动。

群集同步服务（CSS）

Oracle Clusterware组件，通过提供群集中成员资格的公共视图，发现并跟踪每个节点的成员身份状态。CSS还监视进程运行状况，特别是数据库实例的运行状况。全局入队服务监视器（LMON），一种后台进程，用于监视集群数据库环境的运行状况以及从CSS注册和取消注册。另见OCSSD。

群集时间同步服务

一种时间同步机制，可确保群集中所有节点的所有内部时钟同步。

群集验证实用程序（CVU）

一种工具，用于验证各种Oracle RAC组件，例如共享存储设备，网络配置，系统要求，Oracle Clusterware，组和用户。

承诺结果

在提交事务后从Oracle数据库发送到客户端的消息。这些消息不耐用。

数据库池

数据库云中的一组数据库，提供一组唯一的全局服务并属于某个管理域。将云数据库划分为多个池可简化服务管理并提供更高的安全性，因为每个池可由不同的管理员管理。

分布式事务处理（DTP）

分布式事务的范例，包括XA类型的外部协调事务和分布式SQL类型（Oracle中的数据库链接）内部协调事务。

动态网络

使用DHCP进行IPv4或无状态自动配置（autoconfig）的IPv6网络。

活动经理（EVM）

发布Oracle Clusterware事件的后台进程。当事件发生时，EVM扫描指定的callout目录并运行该目录中的所有脚本。

活动经理守护进程（EVMD）

启动racgevt进程以管理标注的Linux或UNIX事件管理器守护程序。

失败组

故障组是磁盘组中磁盘的子集，由于它们共享硬件，因此可能同时发生故障。故障组用于存储数据的镜像副本。

快速申请通知（FAN）

应用程序可以使用FAN实现快速故障检测，故障后连接池的平衡以及修复故障组件时重新平衡连接池。FAN通知进程使用Oracle数据库在无法访问集群服务器或网络接口出现故障时发布的系统事件。

快速连接故障转移

快速连接故障转移为FAN集成客户端提供高可用性，例如使用JDBC，OCI或ODP.NET的客户端。如果将客户端配置为使用快速连接故障转移，则客户端会自动订阅FAN事件，并可以对数据库UP和DOWN事件做出响应。作为响应，Oracle数据库为客户端提供与提供所请求的数据库服务的活动实例的连接。

文件系统

文件系统是提供对磁盘的结构化访问的软件组件。文件系统向应用程序呈现对象，例如文件。通常使用标准API来定义对文件的访问，该API定义操作系统调用，例如应用程序用于访问文件的打开/关闭和读/写。文件系统通常作为操作系统的组件提供，但可以作为独立的软件组件提供。

强制磁盘写入

在Oracle RAC中，特定数据块一次只能由一个实例修改。如果一个实例修改了另一个实例需要的数据块，那么是否需要强制磁盘写入取决于为块提交的请求类型。

通用并行文件系统（GPFS）

通用并行文件系统（GPFS）是一种共享磁盘IBM文件系统产品，可从同构或异构集群中的所有节点提供数据访问。

全局缓存服务（GCS）

实现缓存融合的过程。它维护全局角色中块的块模式。它负责实例之间的块传输。全局缓存服务采用各种后台进程，例如全局缓存服务进程（LMSn）和全局入队服务守护进程（LMD）。

全局缓存服务进程（LMSn）

管理远程消息的进程。Oracle RAC提供多达10个全局缓存服务进程。

全局缓存服务（GCS）资源

全局资源，用于协调对多个Oracle RAC实例的缓冲区高速缓存中的数据块的访问，以提供高速缓存一致性。

全局数据库名称

从任何其他数据库唯一标识它的数据库的全名。全局数据库名称的格式为database_name . database_domain - 例如： OP.US.FOO.COM

全球动态绩效观点（GV $）

动态性能视图，用于存储有关Oracle RAC集群中所有打开实例的信息。（不仅是本地实例。）相反，标准动态性能视图（ V$ ）仅存储有关本地实例的信息。

全球入队服务（GES）

一种服务，用于协调全局共享的队列。

全球入队服务守护进程（LMD）

管理资源请求以控制对块的访问的资源代理进程。LMD进程还处理死锁检测和远程资源请求。远程资源请求是源自另一个实例的请求。

全球入队服务监测（LMON）

后台LMON进程监视整个群集以管理全局资源。LMON管理实例死亡以及任何失败实例的相关恢复。特别是，LMON处理与全球资源相关的恢复部分。LMON提供的服务也称为群集组服务。

全球服务守护进程（GSD）

从SRVCTL接收请求以执行管理作业任务（例如启动或关闭）的组件。该命令在每个节点上本地执行，结果返回给SRVCTL。默认情况下，GSD安装在节点上。

网格即插即用守护进程（GPNPD）

此过程提供对Grid Plug and Play配置文件的访问，并协调集群节点之间对配置文件的更新，以确保所有节点节点都具有最新的配置文件。

高可用性群集多处理（HACMP）

高可用性群集多处理是基于IBM AIX的高可用性群集软件产品。HACMP有两个主要组件：高可用性（HA）和集群多处理（CMP）。

高可用性

具有冗余组件的系统，即使存在硬件或软件故障，也能提供一致且不间断的服务。这涉及一定程度的冗余。

集线器节点

Oracle Flex Cluster中的节点，与其他服务器紧密连接，可直接访问共享存储。

例

对于Oracle RAC数据库，群集中的每个节点通常都有一个正在运行的Oracle软件的实例，该实例引用该数据库。启动数据库时，Oracle数据库会分配一个称为系统全局区域（SGA）的内存区域，并启动一个或多个Oracle数据库进程。SGA和Oracle数据库进程的这种组合称为实例。每个实例都具有唯一的Oracle系统标识符（SID），实例名称，回滚段和线程ID。

实例囚禁

一种方法，它使用初始化参数来限制实例可以同时用于前台进程的CPU数。

实例成员资格恢复

Oracle RAC使用的方法，保证所有集群成员正常运行或处于活动状态。实例成员资格恢复轮询和仲裁成员资格。通过控制文件未显示心跳或未响应周期性活动查询消息的任何成员都被假定终止。

实例名称

表示实例的名称，用于在群集共享公共服务名称时唯一标识特定实例。实例名称由实例初始化文件initsid.ora的INSTANCE_NAME参数标识。实例名称与Oracle系统标识符（SID）相同。

实例编号

将数据块的范围与特定实例相关联的数字。实例编号使您可以启动实例并确保它使用分配给它的扩展区来进行插入和更新。这将确保它不使用为其他实例分配的空间。

互连

节点之间的通信链路。

密钥库

存储透明数据加密密钥的容器。在以前的版本中，这被称为钱包。

叶节点

与Hub Node松散耦合的服务器，可能无法直接访问共享存储。

逻辑卷管理器（LVM）

描述用于联机磁盘存储管理的Linux或UNIX子系统的通用术语。

进程间通信（IPC）

高速操作系统相关的传输组件。IPC在不同节点上的实例之间传输消息。也称为互连。

主引导记录（MBR）

计算机启动时执行的程序。通常，MBR驻留在本地硬盘的第一个扇区上。程序通过检查分区表以确定用于启动系统的分区来启动启动过程。然后，MBR程序将控制权转移到启动分区的引导扇区，这将继续启动过程。

记忆压力

指示服务器上的可用内存量有限的状态。

公

累积统计量的变化率。

多租户容器数据库

见CDB 。

mutables

非确定性函数，每次调用时都可以更改结果。如果函数结果在重放时发生变化，则可变函数可能导致重放被拒绝。考虑.nextval使用的sequence .nextval和SYSDATE。如果主键是使用来自这些函数调用的值构建的，并且在后来的外键或其他绑定中使用，则在重放时必须返回相同的函数结果。对于授予的Oracle函数调用，Application Continuity在重放时提供可变值替换，以提供不透明的绑定变量一致性。

网络附加存储（NAS）

通过网络连接到服务器的存储。

网络时间协议（NTP）

基于TCP / IP构建的Internet标准协议，可确保在计算机网络中精确同步到计算机时钟的毫秒时间。

网络接口卡（NIC）

插入计算机以将计算机连接到网络的卡。

节点

节点是安装了Oracle RAC和Oracle Clusterware软件的计算机系统。

对象链接管理器（OLM）

Oracle接口，用于将符号链接映射到逻辑驱动器，并在OLM图形用户界面中显示它们。

OCSSD

管理群集同步服务（CSS）守护程序的Linux或UNIX进程。管理集群节点成员资格并以oracle用户身份运行;此过程失败会导致群集重新启动。

Oracle集群文件系统

Oracle提供了两个集群文件系统：OCFS2 for Linux和Oracle ASM集群文件系统（Oracle ACFS）。虽然Oracle ACFS是一个专有文件系统，但OCFS2 for Linux的源代码可供所有GNU的通用公共许可证（GPL）使用。这两个文件系统不兼容。

Oracle集群注册表（OCR）

Oracle RAC配置信息存储库，用于管理有关群集节点列表和实例到节点映射信息的信息。OCR还管理有关自定义应用程序的Oracle Clusterware资源配置文件的信息。

Oracle Clusterware

这是Oracle提供的集群件，用于管理集群数据库处理，包括节点成员资格，组服务，全局资源管理和高可用性功能。

Oracle扩展集群

集群由位于称为站点的多个位置的节点组成。

Oracle Flex Cluster

由集线器节点和其他受支持节点组成的大型集群，其中集线器节点使用当前成员资格算法形成集群，而其他节点将成员资格连接到单个集线器节点。

Oracle Grid Infrastructure

为企业网格体系结构提供基础结构的软件。在集群中，该软件包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）。对于独立服务器，此软件包括Oracle Restart和Oracle ASM。 Oracle数据库将这些基础架构产品组合到一个名为Oracle Grid Infrastructure home （ Grid_home ）的软件安装中。

Oracle网格命名服务守护程序（GNSD）

Oracle网格命名服务是群集mDNS和外部DNS服务器之间的网关。gnsd进程在集群中执行名称解析。

Oracle高可用性服务守护程序（OHASD）

此过程锚定Oracle Clusterware堆栈的下半部分，该堆栈由促进集群操作的进程组成。

Oracle接口配置工具（OIFCFG）

用于非群集Oracle数据库和Oracle RAC数据库的命令行工具，使您能够为组件分配和取消分配网络接口，指导组件使用特定网络接口以及检索组件配置信息。Oracle Universal Installer还使用OIFCFG来识别和显示可用的接口。

Oracle托管文件

一种服务，可根据一些初始化参数自动命名，定位，创建和删除数据库文件，如控制文件，重做日志文件，数据文件等。您可以在主机操作系统支持的传统文件系统之上使用Oracle Managed Files，例如VxFS或ODM。它可以简化数据库管理的许多方面，无需为此类详细信息设计自己的策略。

Oracle Notification Service

用于传递有关所有FAN事件的信息的发布和订阅服务。

Oracle Universal Installer

用于安装Oracle Clusterware，Oracle关系数据库软件和Oracle RAC软件的工具。您还可以使用Oracle Universal Installer启动Database Configuration Assistant（DBCA）。

Oracle XA

一个外部接口，允许由Oracle数据库以外的事务管理器协调全局事务。

PDB

在多租户容器数据库（ CDB ）中，作为非CDB在Oracle Net客户端中显示的模式，模式对象和非复制对象的可移植集合。

可插拔数据库

见PDB 。

策略管理的数据库

您定义为群集资源的数据库。数据库的管理由您配置资源的方式定义，包括数据库可以运行的服务器以及支持预期工作负载所需的数据库实例数。

原始设备

尚未设置文件系统的磁盘驱动器。原始设备用于Oracle RAC，因为它们支持共享磁盘。另请参见原始分区 。

原始分区

物理磁盘的一部分，以尽可能低的级别访问。创建扩展分区并为其分配逻辑分区时，将创建原始分区，而不进行任何格式化。格式化完成后，它将被称为熟化分区。另请参见原始设备。

可恢复的错误

由于外部系统故障而出现的一类错误，与正在执行的应用程序会话逻辑无关。在前台，网络，节点，存储和数据库的计划内和计划外中断之后发生可恢复的错误。应用程序收到错误代码，该错误代码可能使应用程序不知道上次提交的操作的状态。

恢复管理器（RMAN）

一种Oracle工具，可用于备份，复制，还原和恢复数据文件，控制文件和归档重做日志。它包含在Oracle服务器中，不需要单独安装。您可以从操作系统（O / S）提示符运行RMAN作为命令行实用程序，也可以使用基于GUI的Oracle Enterprise Manager备份管理器。

区域

包含被认为彼此接近的数据库客户端和服务器的逻辑边界。

请求

从申请中提交的工作单元。请求通常对应于单个数据库连接上的单个Web请求的SQL和PL / SQL以及其他数据库调用，并且通常由检出和从连接签入数据库连接的调用划分池。

请求边界

请求边界标记应用程序或应用程序服务器借用并返回其数据库连接池的连接。

结果缓存

结果缓存是存储器的一个区域，在SGA或客户端应用程序存储器中，存储数据库查询或查询块的结果以供重用。缓存的行在语句和会话之间共享，除非它们变得陈旧。

运行时连接负载平衡

使Oracle数据库能够基于连接池做出智能服务连接决策，该连接池基于当前工作负载为所请求的应用程序提供最佳服务。JDBC，ODP.NET和OCI客户端与负载平衡顾问集成在一起;您可以使用任何这些客户端环境来提供运行时连接负载平衡。

可扩展性

能够向Oracle RAC应用程序添加其他节点，并显着提高扩展和加速。

安全外壳（SSH）

用于通过网络登录远程计算机的程序。您可以使用SSH在远程系统上执行命令，并将文件从一个系统移动到另一个系统。SSH在不安全的通道上使用强身份验证和安全通信。

服务器控制实用程序（SRVCTL）

服务器管理（SRVM）包含在Oracle RAC中运行Oracle Enterprise Manager所需的组件。 SRVM组件（如Intelligent Agent，全局服务守护程序和SRVCTL）使您能够使用Oracle Enterprise Manager通过开放式客户端/服务器体系结构管理在异构环境中运行的集群数据库。

服务器

没有安装Oracle软件的计算机系统。

服务器集群

运行共享GNS服务器的群集。

服务器组

群集中节点的逻辑分区，分为承载应用程序，数据库或两者的组。服务器组可以是其他服务器组的成员。

服务水平

您可以在Oracle RAC数据库中定义的实体，这些实体使您能够将数据库工作负载分组并将工作路由到分配给提供服务的最佳实例。

会话状态一致性

执行COMMIT后，如果该事务中的状态已更改，则如果会话丢失，则无法重播事务以重新建立该状态。配置应用程序连续性时，应用程序的分类取决于初始设置后的会话状态是动态还是静态，然后是否继续通过请求中的COMMIT操作。

  • 动态 :(默认）如果会话状态更改未通过初始化完全封装，则会话具有动态状态，并且在故障转移时无法在回调中完全捕获。一旦请求中的第一个事务提交，就会在内部禁用故障转移，直到下一个请求开始。这是大多数应用程序应用于请求的默认模式。

  • 静态 :(特殊请求）如果所有会话状态更改（例如NLS设置和PL / SQL包状态）都可以在初始化回调中重复，则会话具有静态。此设置仅用于不更改请求内的会话状态的数据库诊断应用程序。如果请求中存在无法通过回调重新建立的任何非事务状态更改，请不要设置STATIC模式。如果您不确定，请使用DYNAMIC模式。

分享一切

一种数据库体系结构，其中所有实例共享对所有数据的访问权限。

单客户端访问名称（SCAN）

Oracle Database 11g数据库客户端使用SCAN连接到数据库。SCAN可以解析为多个IP地址，反映了处理公共客户端连接的集群中的多个侦听器。

单身人士服务

一次只能在一个实例上运行的服务。通过定义服务的分布式事务属性（DTP）属性，可以强制服务为单例服务。

裂脑综合症

两个或更多实例尝试控制群集数据库的情况。例如，在双节点环境中，一个实例尝试同时管理更新，而另一个实例尝试管理更新。

SQL翻译配置文件

SQL转换概要文件是一个数据库模式对象，它指示非Oracle数据库中的SQL语句如何转换为Oracle，以及Oracle错误代码和ANSI SQLSTATES如何转换为其他供应商的等价物。

系统标识符（SID）

Oracle系统标识符（SID）标识正在运行的Oracle软件的特定实例。对于Oracle RAC数据库，群集中的每个节点都有一个引用数据库的实例。

透明应用程序故障转移（TAF）

对于高可用性环境（例如Oracle RAC和Oracle RAC Guard）的运行时故障转移，TAF指的是应用程序到服务连接的故障转移和重新建立。它使客户端应用程序能够在连接失败时自动重新连接到数据库，并可选择恢复正在进行的SELECT语句。此重新连接从Oracle Call Interface库中自动进行。

虚拟互联网协议（VIP）

分配给驻留在单个服务器，多个域名或多个服务器上的多个应用程序的IP地址，而不是分配给特定的单个服务器或网络接口卡（NIC）。

卷管理员

卷管理器是一个软件组件，用于管理磁盘集合到卷中的映射。

投票盘

管理节点成员资格信息的文件。

39.5添加新的物化视图数据库

使用一个或多个物化视图数据库创建物化视图环境后，可能需要添加新的物化视图数据库。

如果满足以下两个条件，则在尝试对在新的物化视图数据库上创建的实例化视图执行快速刷新时，可能会遇到问题：

  • 新物化视图数据库中的物化视图和其他物化视图数据库中的现有物化视图基于相同的主表。

  • 在新的物化视图数据库中创建新的物化视图时，可以刷新现有的物化视图。

在新的物化视图可以执行其第一次快速刷新之前清除主表的物化视图日志时，会出现问题。如果发生这种情况并且您尝试在新的物化视图数据库上对实例化视图执行快速刷新，则可能会遇到以下错误：

ORA-12004 REFRESH FAST不能用于物化视图materialized_view_name ORA-12034物化视图登录materialized_view_name小于上次刷新

如果您收到这些错误，那么唯一的解决方案是执行新物化视图的完整刷新。要避免此问题，请在创建生成物化视图之前在新的物化视图数据库中创建虚拟物化视图。虚拟物化视图可确保在创建生产物化视图时不会清除物化视图日志。

如果选择在实例化视图数据库中创建虚拟物化视图，请完成以下步骤：

  1. 根据主表创建一个名为dummy_mview的虚拟物化视图。例如，要基于名为sales的主表创建虚拟物化视图，请在新的物化视图数据库中发出以下语句：
    CREATE MATERIALIZED VIEW dummy_mview REFRESH FAST AS SELECT * FROM pr.sales@orc1.example.com WHERE 1 = 0;
  2. 在新的物化视图数据库中创建生产物化视图。
  3. 在新的物化视图数据库中快速刷新生产物化视图。
  4. 删除虚拟物化视图。

172.2.8 PURGE_LOST_DB_ENTRY过程

过程PURGE_LOST_DB_ENTRY清除控制从本地站点恢复数据库的条目。

在提交处理期间发生故障时，自动恢复会始终在事务中涉及的所有站点上解析结果。但是，如果在恢复完成之前销毁或重新创建远程数据库，则永远不会删除用于控制DBA_2PC_PENDING和关联表中的恢复的条目，并且将定期重试恢复。过程PURGE_LOST_DB_ENTRY允许从本地站点删除此类事务。

句法

DBMS_TRANSACTION.PURGE_LOST_DB_ENTRY（xid VARCHAR2）;

参数

表172-5 PURGE_LOST_DB_ENTRY过程参数

参数 描述

XID

必须设置为DBA_2PC_PENDING表中LOCAL_TRAN_ID列的值。

使用说明

警告：

当其他数据库丢失或已重新创建PURGE_LOST_DB_ENTRY， 才应使用。任何其他用途都可能使其他数据库处于不可恢复或不一致的状态。

在自动恢复运行之前，事务可以在DBA_2PC_PENDING显示为状态“收集”，“已提交”或“已准备好”。如果DBA通过使用“commit force”或“rollback force”强制不确定事务具有特定结果，则还可能出现“强制提交”或“强制回滚”状态。自动恢复通常会删除任何这些状态的条目。唯一的例外是当恢复发现强制事务处于与事务中其他站点不一致的状态时;在这种情况下，条目保留在表中， MIXED列的值为“yes”。

但是，在某些情况下，可能无法运行自动恢复。例如，远程数据库可能已永久丢失。即使重新创建它，它也会获得一个新的数据库ID，因此恢复无法识别它（可能的症状是ORA-02062 ）。在这种情况下，DBA可以使用过程PURGE_LOST_DB_ENTRY来清除除“准备”之外的任何状态的条目。DBA不需要特别急于解决这些条目，因为它们没有持有任何数据库资源。

此过程删除有关给定混合结果事务的信息
