1使用消息

本章为您提供有关错误消息的一般信息和有用的提示。本章包含以下主题：

查找消息文档

本节介绍在给出错误消息时如何查找消息文档。

Oracle数据库消息

运行任何Oracle程序时，Oracle数据库都会生成这些消息。本手册介绍了Oracle产品和工具中常见的消息。

特定于产品的消息

这些消息特定于一种产品，并记录在该产品的手册中。

特定于操作系统的消息

这些消息特定于一个操作系统。为每个操作系统保留一系列消息代码编号。例如，ORA-07500到ORA-07999的范围是为VMS操作系统消息保留的;这些消息列在相应的操作系统特定文档中。

消息的前缀表示在何处查找有关消息的信息。例如，本手册中的某些消息具有前缀“ORA”。如果您遇到没有前缀的消息，请首先查看您正在使用的Oracle发行版的手册，然后查看本手册。

消息的准确性

本手册中消息的准确性是我们首要关注的问题。有时，Oracle软件中不会包含对消息的增强功能。如果您在系统上生成的消息与本书中的消息不同，请确保改进的消息文本通常包含在软件的下一个版本中。

消息格式

显示的所有消息都以文本为前缀，指示发出消息的组件。例如，前缀“ORA”表示该消息是由Oracle数据库生成的。本书中消息的位置取决于消息的前缀。

消息代码编号按顺序列出所有消息。要查找消息，请使用消息代码编号。

识别消息中的变量文本

为了帮助您查找和修复错误，Oracle在某些消息中嵌入了对象名称，数字和字符串。这些嵌入变量适当地由字符串 ， 数字或字符表示。例如：

ORA-00020：超出最大进程数 （ 数量 ）

上述消息实际上可能如下所示：

ORA-00020：超出最大进程数（50）

消息堆栈

有时，您可能会看到一个消息堆栈 。这只是在Oracle数据库的不同级别发布的一系列相关消息。

以下消息堆栈是一个通用示例：

ORA-06502：PL / SQL：数字或值错误ORA-06512：在“SCOTT.VALUE_ERR”，第1行ORA-06512：在第1行

以下消息堆栈是VAX / VMS示例：

ORA-01034：Oracle不可用ORA-07625：smsget：$ MGBLSC失败％SYSTEM-W-NOSUCHSEC，没有这样的（全局）部分

在此示例中，请注意堆栈底部的消息由VMS操作系统发出。查看系统中每个级别的消息可以帮助您跟踪原始事件。例如，在这种情况下，Oracle可能因为尚未启动而无法使用，因此没有系统全局区域（SGA）。如果需要Oracle Support Services的帮助，请务必报告堆栈中的所有消息。

联系Oracle支持服务

某些消息建议联系Oracle支持服务部门以报告问题。联系Oracle支持服务时，请提供以下信息：

  • 运行Oracle数据库的操作系统的硬件，操作系统和版本号

  • Oracle数据库的完整版本号（例如，版本9.2.0.1.0或版本9.0.1.1.2）

  • 发生错误时，所有Oracle程序（带有版本号）都在使用中。例如，SQL * Net V2.0或SQL * Forms V3.0

  • 如果您遇到一个或多个错误代码或消息，则按照它们出现的顺序出现确切的代码编号和消息文本

  • 问题严重程度，根据以下代码：

    1 =程序不可用。对运营的关键影响。

    2 =程序可用。操作受到严格限制。

    3 =程序可用，功能有限。对整体运营并不重要。

    4 =客户规避的问题。对操作的影响最小（如果有的话）。

您还需要提供以下内容：

  • 你的名字

  • 您组织的名称

  • 您的Oracle支持ID号

  • 你的电话号码

Oracle异常消息

Oracle广泛的自检有助于检测内部错误。Oracle对Oracle内部异常使用以下catchall消息：

ORA-00600：内部错误代码，参数：[ 数字 ]，[ 数字 ]，[]，[]，[]，[]

实际消息可能如下所示：

ORA-00600：内部错误代码，参数：[1042]，[3]，[upilam]，[]，[]，[]

消息文本后面最多可以跟六个参数，这些参数指示错误的来源和属性。第一个参数是内部错误号。其他参数是各种数字，名称和字符串。空括号可能会被忽略。

除了返回给用户之外，还会将内部错误写入警报文件以及有关导致消息的事件的其他信息。警报文件还列出了由于内部错误而可能生成的所有跟踪文件。有关跟踪和警报文件的说明，请参阅以下部分。

如果收到ORA-00600消息，请将其报告给Oracle支持服务。

跟踪文件

每次Oracle实例启动或在用户进程或后台进程中发生意外事件时，都会创建跟踪文件。跟踪文件的名称包括实例名称，进程名称和Oracle进程号。文件扩展名或文件类型通常是TRC，如果不同，则在特定于操作系统的Oracle文档中记录。跟踪文件的内容可能包括系统全局区域，进程全局区域，管理程序堆栈和寄存器的转储。

两个初始化参数用于指定跟踪文件的位置。

  • BACKGROUND_DUMP_DEST初始化参数指定由Oracle后台进程PMON，DBWR，LGWR和SMON创建的跟踪文件的位置。

  • USER_DUMP_DEST初始化参数指定由用户进程（如SQL * Loader或Pro * C）创建的跟踪文件的位置。

警报文件还描述了发生内部错误时生成的跟踪文件的位置。有关警报文件的说明，请参阅下一节。

在使用跟踪文件诊断问题之前，您可能需要格式化跟踪文件。要格式化跟踪文件，请使用DUMPFMT实用程序，该实用程序在大多数系统上都可用，并在特定于操作系统的Oracle文档中进行了描述。Oracle支持服务可能会要求您提供格式化的跟踪文件以帮助解决问题。

警报文件

警报文件是一个日志文件，用于记录有关内部错误和管理活动（如备份）的信息。发生内部错误时，消息将发送到终端屏幕并写入警报文件。Oracle还会将有关内部错误的其他信息写入警报文件，例如由于错误而生成的任何跟踪文件的位置和名称。

Alert文件的名称是特定于操作系统的。Alert文件的位置与后台进程跟踪文件的位置相同。此位置由BACKGROUND_DUMP_DEST初始化参数指定。有关更多信息，请参阅Oracle操作系统特定的文档。

如果您的系统使用操作员控制台，则Oracle的某些消息可能会显示在控制台上。所有重要消息都写入Alert文件以及操作员控制台。由于非Oracle消息也出现在此控制台上，因此警报文件是跟踪控制台日志而不是跟踪所有Oracle管理活动和错误的更好记录。

警报文件还记录有关管理活动的信息，例如备份和归档联机重做日志文件。

1.5 Oracle JVM的主要组件

本节简要介绍Oracle JVM的主要组件及其提供的一些功能。

Oracle JVM是一个完整的，符合Java 2的环境，用于运行Java应用程序。它通过共享内存堆并直接访问其关系数据，在与数据库内核相同的进程空间和地址空间中运行。此设计可优化内存使用并提高吞吐量。

Oracle JVM为Java对象提供运行时环境。它完全支持Java数据结构，方法分派，异常处理和语言级线程。它还支持所有核心Java类库，包括java.lang ， java.io ， java.net ， java.math和java.util 。

1.5.1图书馆经理

要在Oracle数据库中存储Java类，可以使用loadjava命令行工具，该工具使用SQL CREATE JAVA语句来完成其工作。当由CREATE JAVA {SOURCE | CLASS | RESOURCE}语句，库管理器将Java源，类或资源文件加载到数据库中。不直接访问这些Java模式对象，只有Oracle JVM使用它们。

1.5.2编译器

Oracle JVM包括标准Java编译器。运行CREATE JAVA SOURCE语句时，它会将Java源文件转换为体系结构中立的单字节指令，称为字节码。每个字节码由操作码后跟其操作数组成。生成的Java类文件完全符合Java标准，在运行时提交给解释器。

1.5.3口译员

要运行Java程序，Oracle JVM包括标准的Java 2字节码解释器。解释器和关联的Java运行时系统运行标准Java类文件。运行时系统支持本机方法以及来自主机环境的call-in和call-out。

注意：

您还可以编译Java代码以提高性能。Oracle JVM使用核心Java类库，SQLJ转换器和JDBC驱动程序的本机编译版本。

1.5.4类装载机

响应来自运行时系统的请求，Java类加载器定位，加载和初始化存储在数据库中的Java类。类加载器读取类并生成运行它所需的数据结构。不可变数据和元数据被加载到初始化一次共享内存中。结果，每个会话需要更少的内存。类加载器尝试在必要时解析外部引用。此外，如果源文件可用，则在必须重新编译Java类文件时，类加载器会自动调用Java编译器。

1.5.5验证者

Java类文件是完全可移植的，并且符合明确定义的格式。验证程序可防止无意中使用欺骗性Java类文件，这可能会改变程序流或违反访问限制。Oracle安全性和Java安全性与验证程序一起使用以保护您的应用程序和数据。

1.5.6服务器端JDBC内部驱动程序

JDBC是一种标准，它定义了一组Java类，提供对关系数据的独立于供应商的访问。JDBC类在ODBC和X / Open SQL调用级接口（CLI）之后建模，并提供标准功能，例如同时连接到多个数据库，事务管理，简单查询，对存储过程的调用以及对LONG列数据的流式访问。

使用低级入口点，专门调整的JDBC驱动程序直接在Oracle数据库中运行，从而可以从Java存储过程快速访问Oracle数据。服务器端JDBC内部驱动程序完全符合标准JDBC规范。JDBC驱动程序与数据库紧密集成，支持特定于Oracle的数据类型，全球化字符集和存储过程。此外，客户端和服务器端JDBC API是相同的，这使得分区应用程序变得容易。

1.5.7服务器端SQLJ转换器

SQLJ使您可以在Java程序中嵌入SQL语句。它比JDBC更简洁，对静态分析和类型检查更敏感。SQLJ预处理器本身是一个Java程序，它将嵌入了SQLJ子句的Java源文件作为输入。然后，它将SQLJ子句转换为实现指定SQL语句的Java类定义。Java类型系统确保使用正确的参数调用这些类的对象。

高度优化的SQLJ转换器直接在数据库内运行，使用服务器端内部JDBC驱动程序提供对Oracle数据的运行时访问。SQLJ表单可以包括查询，数据操作语言（DML）语句，数据定义语言（DDL）语句，事务控制语句以及对存储过程的调用。客户端和服务器端SQLJ API完全相同，因此可以轻松地对应用程序进行分区。

1.5.8系统类

在Oracle数据库环境中构成Java实现的重要部分的一组类称为系统类 。这些类在SYS模式中定义，并通过公共同义词为所有用户导出。可以在SYS模式Foot 1以外的模式中定义与其中一个System类同名的类。但是，这是一种不好的做法，因为类的替代版本可能会违反关于该类的语义的假设，这些假设存在于其他系统类或Java虚拟机的底层实现中。甲骨文强烈反对这种做法。

数据定义语言（DDL）语句

数据定义语言（DDL）语句允许您执行以下任务：

  • 创建，更改和删除架构对象

  • 授予和撤销权限和角色

  • 分析表，索引或集群的信息

  • 建立审计选项

  • 添加注释到数据字典

CREATE ， ALTER和DROP命令需要对指定对象的独占访问权限。例如，如果另一个用户在指定的表上具有打开的事务，则ALTER TABLE语句将失败。

GRANT ， REVOKE ， ANALYZE ， AUDIT和COMMENT命令不需要对指定对象的独占访问权限。例如，您可以在其他用户更新表时分析表。

Oracle Database在每个DDL语句之前和之后隐式提交当前事务。

DDL语句是阻塞或非阻塞，两种类型的DDL语句都需要内部结构上的独占锁。

数据操作语言（DML）语句

数据操作语言（DML）语句访问和操作现有模式对象中的数据。这些语句不会隐式提交当前事务。数据操作语言语句是：

  • 呼叫
  • 删除
  • 解释计划
  • 插入
  • 锁表
  • 合并
  • 选择
  • UPDATE

SELECT语句是DML语句的有限形式，因为它只能访问数据库中的数据。它不能操纵存储在数据库中的数据，尽管它可以在返回查询结果之前操纵访问的数据。

仅在动态执行时才在PL / SQL中支持SELECT语句。但是，您可以在PL / SQL代码中使用类似的PL / SQL语句SELECT INTO ，而不必动态执行它。仅当动态执行时，PL / SQL才支持CALL和EXPLAIN PLAN语句。PL / SQL完全支持所有其他DML语句。

交易控制声明

事务控制语句管理DML语句所做的更改。事务控制语句是：

  • 承诺
  • ROLLBACK
  • SAVEPOINT
  • SET TRANSACTION
  • SET CONSTRAINT

PL / SQL支持除COMMIT和ROLLBACK命令的某些形式之外的所有事务控制语句。有关限制的信息，请参阅COMMIT和ROLLBACK 。

会话控制语句

会话控制语句动态管理用户会话的属性。这些语句不会隐式提交当前事务。

PL / SQL不支持会话控制语句。会话控制语句是：

  • 改变会话
  • 设置角色

系统控制声明

单个系统控制语句ALTER SYSTEM动态管理Oracle数据库实例的属性。此语句不隐式提交当前事务，PL / SQL不支持。

目的

topN和cutoff

您可以指定topN和cutoff来限制函数返回的topN数。默认情况下， topN和cutoff都为null，并返回所有要素。

  • topN是N最高价值的特征。如果多个要素具有第N个值，则该函数选择其中一个。

  • cutoff是一个值阈值。仅返回大于或等于cutoff要素。要仅按cutoff过滤，请为topN指定NULL 。

要返回大于或等于cutoff N要素，请同时指定topN和cutoff 。

语法选择

FEATURE_SET可以通过以下两种方式之一对数据进行评分：它可以将挖掘模型对象应用于数据，也可以通过执行构建和应用一个或多个瞬态挖掘模型的分析子句来动态挖掘数据。选择语法或分析语法 ：

  • 语法 - 使用第一种语法使用预定义的模型对数据进行评分。提供特征提取模型的名称。

  • 分析语法 - 使用分析语法在没有预定义模型的情况下对数据进行评分。包括INTO n ，其中n是要提取的mining_analytic_clause数，以及mining_analytic_clause ，它指定是否应为多个模型构建分区数据。mining_analytic_clause支持query_partition_clause和order_by_clause 。（参见“ analytic_clause :: = ” 。）

对于type_name ，请指定要将expr转换为的数据类型。您可以指定以下数据类型：

  • BINARY_DOUBLE

8.6瞬态和通用类型

Oracle数据库有三种通用（即通用编程）SQL数据类型，使您能够动态地封装和访问任何其他SQL类型的类型描述，数据实例和数据实例集，包括对象和集合类型。您还可以使用这三种类型来创建匿名类型，包括匿名集合类型。

这三种SQL类型实现为不透明类型。换句话说，数据库不知道这些类型的内部结构;只有通过实现功能（通常是3GL例程）才能查询它们的数据。Oracle数据库提供了OCI和PL / SQL API来实现这些功能。

在这三种类型中， ANYTYPE是瞬态的， ANYDATA和ANYDATASET不是瞬态的，而是持久的。暂态类型不能持久存储，因为它们的结构对数据库是不透明的。您不能创建瞬态类型的列或使它们成为持久类型的属性。

与Oracle数据库版本12个C开头，版本12.2，可以在活动数据卫士例如，如果创建瞬态类型：

  1. 实时应用正在Active Data Guard上运行，并且

  2. 逻辑待机并不远远落后于主要（通常是秒数）。

1超大型数据库简介

非常大的数据库（VLDB）提出了需要多种策略的管理挑战。分区是VLDB战略的关键组成部分。

现代企业经常运行任务关键型数据库，其中包含数百兆字节，通常是数兆兆字节的数据。这些企业受到超大型数据库（VLDB）的支持和维护要求的挑战，必须设计出应对这些挑战的方法。

分区简介

分区通过将它们细分为更小且更易于管理的部分，为非常大的表和索引提供支持。

分区解决了支持非常大的表和索引的关键问题，方法是将它们分解为更小且更易于管理的分区 ，这些分区对应用程序完全透明。不需要修改SQL查询和数据操作语言（DML）语句来访问分区表。但是，在定义分区之后，数据定义语言（DDL）语句可以访问和操作单个分区，而不是整个表或索引。这就是分区可以简化大型数据库对象的可管理性的方式。

表或索引的每个分区必须具有相同的逻辑属性，例如列名，数据类型和约束，但每个分区可以具有单独的物理属性，例如启用或禁用压缩，物理存储设置和表空间。

分区对许多不同类型的应用程序很有用，特别是管理大量数据的应用程序。OLTP系统通常受益于可管理性和可用性的改进，而数据仓库系统则受益于性能和可管理性。

分区提供了以下优势：

  • 它支持数据管理操作，如数据加载，索引创建和重建，以及分区级别的备份和恢复，而不是整个表。这导致这些操作的时间显着减少。

  • 它提高了查询性能。通常，查询的结果可以通过访问分区的子集而不是整个表来实现。对于某些查询，此技术（称为分区 修剪 ）可以提供性能的数量级增益。

  • 它显着降低了计划停机对维护操作的影响。

    分区维护操作的分区独立性允许您在同一个表或索引的不同分区上执行并发维护操作。您还可以对不受维护操作影响的分区运行并发SELECT和DML操作。

  • 如果将关键表和索引划分为多个分区以减少维护窗口，恢复时间和故障影响，则可以提高任务关键型数据库的可用性。

  • 并行执行提供了优化资源利用率和最小化执行时间的特定优势。查询以及DML和DDL支持并行执行。

分区可以在Oracle数据库中实现更快的数据访问。无论数据库是10 GB还是10 TB数据，分区都可以提高数量级的数据访问。无需对应用程序进行任何修改即可实现分区。例如，您可以将非分区表转换为分区表，而无需修改任何访问该表的SELECT语句或DML语句。您无需重写应用程序代码即可利用分区。

VLDB和分区

分区是管理超大型数据库（VLDB）的有价值策略。

一个非常大的数据库没有最小绝对大小。虽然VLDB是像小型数据库这样的数据库，但在管理VLDB方面存在特定的挑战。这些挑战与针对该规模的系统执行操作的庞大规模和成本效益有关。

几个趋势是数据库大小稳步增长的原因：

  • 很长一段时间以来，系统都是孤立开发的。公司已经开始看到组合这些系统的好处，以实现跨部门分析，同时降低系统维护成本。数据库和应用程序的整合是数据库大小持续增长的关键因素。

  • 许多公司都面临着在最短时间内存储数据的规定。法规通常会导致更多数据存储更长时间。

  • 公司通过扩大销售和运营或通过合并和收购来增长，从而导致生成和处理的数据量增加。同时，依赖于数据库进行日常活动的用户群增加。

分区是管理超大型数据库的关键功能。成长是非常大的数据库分区地址的基本挑战，和分区使分 而 治之的技术用于管理数据库中的表和索引，尤其是那些表和索引增长。分区是一种功能，允许数据库扩展非常大的数据集，同时保持一致的性能，而不会过度增加管理或硬件资源。

分区作为信息生命周期管理的基础

分区为信息生命周期管理（ILM）提供支持。

信息生命周期管理（ILM）是一组用于在整个使用寿命期间管理数据的流程和策略。ILM战略的一个重要组成部分是确定在其生命周期中的任何时刻存储数据的最合适和最具成本效益的媒介：日常运营中使用的较新数据存储在速度最快，可用性最高的存储层上，而不经常访问的旧数据可以存储在较便宜且效率较低的存储层上。较旧的数据也可能不经常更新，因此将数据压缩和存储为只读是有意义的。

Oracle数据库为实施ILM解决方案提供了理想的环境。Oracle支持多个存储层，并且由于所有数据都保留在Oracle数据库中，因此多个存储层对应用程序是透明的，并且数据仍然是安全的。分区提供了基本技术，使表中的数据可以存储在不同的分区中。

虽然企业级系统中最常见的是多个存储层和复杂的ILM策略，但大多数公司和大多数数据库都需要一定程度的信息生命周期管理。在使用分区时，最基本的ILM操作（归档旧数据以及从数据库中清除或删除该数据）可以快几个数量级。

XMLTable将XQuery评估的结果映射到关系行和列。您可以使用SQL将函数返回的结果作为虚拟关系表进行查询。

  • XMLNAMESPACES子句包含一组XML名称空间声明。这些声明由XQuery表达式（计算的XQuery_string ）引用，该表达式计算行，并通过XML_table_column的PATH子句中的XPath表达式XML_table_column ，该表达式计算整个XMLTable函数的列。如果要在COLUMNS子句的PATH表达式中使用限定名称，则需要指定XMLNAMESPACES子句。

16.1关于OCI数据库启动和关闭

OCI函数OCIDBStartup()和OCIDBShutdown()提供启动和关闭Oracle数据库所需的最小接口。

在调用OCIDBStartup()之前，C程序必须连接到服务器并在初步认证模式下启动SYSDBA或SYSOPER会话。此模式是实例未启动时唯一允许的模式，仅用于启动实例。对OCIDBStartup()调用将启动一个服务器实例，而无需安装或打开数据库。要装入和打开数据库，请结束初步认证会话并启动常规SYSDBA或SYSOPER会话以执行相应的ALTER DATABASE语句。

关闭数据库需要活动的SYSDBA或SYSOPER会话。对于除OCI_DBSHUTDOWN_ABORT以外的所有模式，对OCIDBShutdown()进行两次调用：一次通过禁止进一步连接数据库来启动关闭，然后使用适当的ALTER DATABASE命令来卸载和关闭它;另一个调用是通过关闭实例来完成关闭。在特殊情况下，要尽快关闭数据库，请在OCI_DBSHUTDOWN_ABORT模式下调用OCIDBShutdown() ，这相当于SQL * Plus中的SHUTDOWN ABORT 。

这两个功能都需要与服务器的专用连接。如果在通过调度程序将数据库连接到共享服务器时尝试启动或关闭数据库，则会发出ORA-106信号。

OCIAdmin管理句柄C数据类型用于使接口可扩展。OCIAdmin与句柄类型OCI_HTYPE_ADMIN相关联。用于传递一个值OCIAdmin参数， admhp ，是可选的OCIDBStartup()并且不被需要OCIDBShutdown()

句法

目的

REGEXP_SUBSTR通过让您在字符串中搜索正则表达式模式REGEXP_SUBSTR扩展SUBSTR函数的功能。它也类似于REGEXP_INSTR ，但它不返回子字符串的位置，而是返回子字符串本身。如果您需要匹配字符串的内容但不需要它在源字符串中的位置，则此函数很有用。该函数将字符串作为VARCHAR2或CLOB数据返回到与source_char相同的字符集中。

此函数符合POSIX正则表达式标准和Unicode正则表达式指南。有关更多信息，请参阅Oracle正则表达式支持 。

  • source_char是用作搜索值的字符表达式。它通常是字符列，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB 。

  • pattern是正则表达式。它通常是文本文字，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 。它最多可包含512个字节。如果pattern的数据类型与source_char的数据类型不同，则Oracle数据库将pattern转换为source_char的数据类型。有关可以在pattern指定的运算符的列表，请参阅Oracle正则表达式支持 。

  • position是一个正整数，表示Oracle应该开始搜索的source_char的字符。默认值为1，表示Oracle在source_char的第一个字符处开始搜索。

  • occurrence是一个正整数，表示Oracle应该搜索的source_char哪个pattern出现。默认值为1，表示Oracle搜索第一次出现的pattern 。

    如果occurrence大于1，则数据库将搜索第二次出现，从第一次出现pattern后的第一个字符开始，依此类推。此行为与SUBSTR函数不同，后者开始在第一次出现的第二个字符处搜索第二个匹配项。

  • match_parameter是一个文本文字，可让您更改函数的默认匹配行为。此参数的行为与此函数的行为与REGEXP_COUNT 。有关详细信息，请参阅REGEXP_COUNT 。

  • 对于具有子表达式的pattern ， subexpr是从0到9的非负整数，表示函数返回pattern哪个子表达式。此参数与REGEXP_INSTR函数具有相同的语义。有关更多信息，请参阅REGEXP_INSTR 。

使用不区分大小写的密码版本检查帐户

使用这些过程可以确定要升级的Oracle数据库是否具有使用不区分大小写的密码版本的帐户或配置参数。

默认情况下，在Oracle Database 12 c第 2版（12.2）及更高版本中，不会生成或允许10G密码版本。

如果未将SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为允许不区分大小写的版本的许可身份验证协议，并且您不希望将使用不区分大小写的密码版本进行身份验证的用户帐户锁定在数据库之外，则必须识别受影响的帐户，并且确保他们使用区分大小写的密码版本。

示例2-1查找使用不区分大小写（10G）版本的用户帐户

以管理用户身份登录SQL * Plus，并输入以下SQL查询：

来自DBA_USERS的SELECT USERNAME，PASSWORD_VERSIONS;

以下结果显示了帐户的密码版本：

USERNAME PASSWORD_VERSIONS ------------------------------ ----------------- JONES 10G 11G 12C ADAMS 10G 11G CLARK 10G 11G PRESTON 11G BLAKE 10G

在此示例中，使用的每个用户帐户密码验证版本的背景不同：

  • JONES是在Oracle Database 10G中创建的，当SQLNET.ALLOWED_LOGON_VERSION_SERVER参数的设置设置为8时， JONES的密码在Oracle Database 12C重置。因此，此密码重置创建了所有三个版本。11G和12C使用区分大小写的密码。

  • ADAMS和CLARK最初是使用10G版本创建的，然后是从早期版本导入的11G版本。然后，这些帐户密码在11G重置，不推荐使用的参数SEC_CASE_SENSITIVE_LOGON设置为TRUE。

  • BLAKE的密码是使用10G版本创建的，密码尚未重置。因此，用户BLAKE继续使用10G密码版本，该版本使用不区分大小写的密码。

用户BLAKE在升级之前只有10G密码版本：

SQL> SELECT USERNAME，PASSWORD_VERSIONS FROM DBA_USERS; USERNAME PASSWORD_VERSIONS ------------------------------ ----------------- BLAKE 10G

如果在不采取任何进一步操作的情况下升级到新的Oracle数据库版本，则此帐户将无法访问。确保在升级之前未将系统配置为独占模式（通过将SQLNET.ORA参数SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为更宽松的身份验证模式）。
