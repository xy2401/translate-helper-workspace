ALTER LIBRARY

目的

ALTER LIBRARY语句显式重新编译库。显式重新编译消除了对隐式运行时重新编译的需要，并防止了相关的运行时编译错误和性能开销。

注意：

此语句不会更改现有库的声明或定义。要重新声明或重新定义库，请使用带有OR REPLACE子句的“ CREATE LIBRARY ” 。

先决条件

如果库位于SYS架构中，则必须以SYSDBA身份连接。否则，库必须位于您自己的架构中，或者您必须具有ALTER ANY LIBRARY系统特权。

网络接口硬件最低要求

查看这些要求以确保您拥有Oracle Grid Infrastructure群集的最低网络硬件技术。

每个节点的公共网络

公共网络为客户端提供数据库服务访问。公共网络必须满足这些最低要求：

  • 适配器：每个节点必须至少有一个公共网络适配器或网络接口卡（NIC）。

    Oracle支持使用链路聚合，绑定，中继或组合网络来提高带宽和高可用性。

  • 协议：每个公共接口必须支持TCP / IP。

每个节点的专用网络

专用网络（也称为互连）是仅集群成员节点可以访问的网络。他们使用交换机进行连接。专用网络必须满足以下最低要求：

  • 适配器：每个节点必须至少有一个专用网络适配器或网络接口卡（NIC）。

    Oracle建议您使用冗余互连使用配置互连，其中多个网络适配器配置了链路本地范围内的地址，以便为互连提供高可用性IP（HAIP）地址。您可以在安装期间或使用Oracle Interface Configuration Tool（OIFCFG）安装后配置冗余互连使用，以提供更高的带宽和高可用性。

    Oracle还支持使用链路聚合，绑定，中继或组合网络，以提高带宽和高可用性。

  • 协议：使用支持TCP / IP的高速网络适配器和交换机或使用Infiniband的可靠数据报套接字（RDS）的用户数据报协议（UDP）。

    交换机：必须使用支持TCP / IP的互连交换机。 Oracle建议您使用专用交换机。最小交换机速度为1千兆以太网。

局域网技术

Oracle不支持互连的令牌环或交叉电缆。Oracle支持Jumbo Frames和Infiniband。在互连上使用Infiniband时，Oracle支持使用RDS协议。

如果您使用共享物理适配器进行共享以太网VLAN部署，请确保应用标准以太网设计，部署和监视最佳实践，以防止由于常见共享以太网交换机网络事件导致的群集中断和性能下降。

存储网络

Oracle自动存储管理和Oracle Real Application Clusters需要网络连接存储。

Oracle自动存储管理（Oracle ASM）：用于Oracle Clusterware文件的网络接口也用于Oracle ASM。

第三方存储：Oracle建议您为存储配置其他接口。

关于安装后配置文件

运行静默或响应文件安装时，在响应文件中提供有关服务器的信息，否则在图形用户界面安装期间手动提供这些信息。

但是，响应文件不包含配置助手在完成软件安装后需要的用户帐户密码。配置助手使用名为configToolAllCommands的脚本启动。您可以使用密码响应文件以响应文件模式运行此脚本。该脚本使用密码连续运行配置工具以完成配置。

如果保留用于克隆安装的密码文件，则Oracle强烈建议您将密码文件存储在安全位置。此外，如果必须停止安装以修复错误，则可以使用configToolAllCommands和密码响应文件运行配置助手。

configToolAllCommands密码响应文件具有以下语法选项：

  • oracle.crs for Oracle Grid Infrastructure组件或oracle.server for Oracle Database组件，配置助手配置

  • variable_name是配置文件变量的名称

  • value是用于配置的所需值。

命令语法如下：

internal_component_name |变量名=值

例如：

oracle.crs | S_ASMPASSWORD =密码

数据库配置助手需要SYS，SYSTEM和DBSNMP密码才能与Oracle DBCA一起使用。您可能需要指定以下附加密码，具体取决于您的系统配置：

  • 如果数据库使用Oracle自动存储管理（Oracle ASM）进行存储，则必须为S_ASMSNMPPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

  • 如果使用一个或多个可插拔数据库（PDB）创建多租户容器数据库（CDB），则必须为S_PDBADMINPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

Oracle强烈建议您使用密码响应文件来维护安全性：

  • 响应文件的权限应设置为600。

  • 响应文件的所有者应该是安装所有者用户，并将该组设置为中央库存（ oraInventory ）组。

关于具有作业角色分离的Oracle安装

作业角色分离要求您为通过操作系统授权授予的每组系统权限创建不同的操作系统组。

通过Oracle Grid Infrastructure作业角色分离，Oracle ASM具有单独的操作系统组，这些操作系统组为存储层管理的Oracle ASM系统特权提供操作系统授权。此操作系统授权与Oracle数据库操作系统授权分开。此外，Oracle Grid Infrastructure安装所有者还提供操作系统用户授权，以修改Oracle Grid Infrastructure二进制文件。

通过Oracle数据库作业角色分离，每个Oracle数据库安装都具有单独的操作系统组，以提供对该Oracle数据库的系统特权的授权。因此，可以在群集上安装多个数据库，而无需共享系统权限的操作系统授权。此外，每个Oracle软件安装都由单独的安装所有者拥有，以便为修改Oracle Database二进制文件提供操作系统用户授权。请注意，任何Oracle软件所有者都可以启动和停止所有数据库并共享Oracle Grid Infrastructure资源，例如Oracle ASM或虚拟IP（VIP）。作业角色分离配置可实现数据库安全性，并且不会限制用户角色启动和停止各种Oracle Clusterware资源。

您可以选择为存储和数据库层上的所有系统特权创建一个管理用户和一个组以进行操作系统身份验证。例如，您可以将oracle用户指定为所有Oracle软件的安装所有者，并将oinstall指定为其成员被授予Oracle Clusterware所有系统特权的组; Oracle ASM的所有系统权限;服务器上所有Oracle数据库的所有系统权限;以及安装所有者的所有OINSTALL系统权限。该组还必须是Oracle Inventory组。

如果您不想使用角色分配组，那么Oracle强烈建议您至少使用两个组：

网络接口硬件最低要求

查看这些要求以确保您拥有Oracle Grid Infrastructure群集的最低网络硬件技术。

每个节点的公共网络

公共网络为客户端提供数据库服务访问。公共网络必须满足这些最低要求：

  • 适配器：每个节点必须至少有一个公共网络适配器或网络接口卡（NIC）。

    Oracle支持使用链路聚合，绑定，中继或组合网络来提高带宽和高可用性。

  • 协议：每个公共接口必须支持TCP / IP。

每个节点的专用网络

专用网络（也称为互连）是仅集群成员节点可以访问的网络。他们使用交换机进行连接。专用网络必须满足以下最低要求：

  • 适配器：每个节点必须至少有一个专用网络适配器或网络接口卡（NIC）。

    Oracle建议您使用冗余互连使用配置互连，其中多个网络适配器配置了链路本地范围内的地址，以便为互连提供高可用性IP（HAIP）地址。您可以在安装期间或使用Oracle Interface Configuration Tool（OIFCFG）安装后配置冗余互连使用，以提供更高的带宽和高可用性。

    Oracle还支持使用链路聚合，绑定，中继或组合网络，以提高带宽和高可用性。

  • 协议：使用支持TCP / IP的高速网络适配器和交换机或使用Infiniband的可靠数据报套接字（RDS）的用户数据报协议（UDP）。

    交换机：必须使用支持TCP / IP的互连交换机。 Oracle建议您使用专用交换机。最小交换机速度为1千兆以太网。

局域网技术

Oracle不支持互连的令牌环或交叉电缆。Oracle支持Jumbo Frames和Infiniband。在互连上使用Infiniband时，Oracle支持使用RDS协议。

如果您使用共享物理适配器进行共享以太网VLAN部署，请确保应用标准以太网设计，部署和监视最佳实践，以防止由于常见共享以太网交换机网络事件导致的群集中断和性能下降。

存储网络

Oracle自动存储管理和Oracle Real Application Clusters需要网络连接存储。

Oracle自动存储管理（Oracle ASM）：用于Oracle Clusterware文件的网络接口也用于Oracle ASM。

第三方存储：Oracle建议您为存储配置其他接口。

运行静默或响应文件安装时，在响应文件中提供有关服务器的信息，否则在图形用户界面安装期间手动提供这些信息。

如果保留用于克隆安装的密码文件，则Oracle强烈建议您将密码文件存储在安全位置。此外，如果必须停止安装以修复错误，则可以使用configToolAllCommands和密码响应文件运行配置助手。

configToolAllCommands密码响应文件具有以下语法选项：

  • oracle.crs for Oracle Grid Infrastructure组件或oracle.server for Oracle Database组件，配置助手配置

  • variable_name是配置文件变量的名称

  • value是用于配置的所需值。

命令语法如下：

internal_component_name |变量名=值

例如：

oracle.crs | S_ASMPASSWORD =密码

数据库配置助手需要SYS，SYSTEM和DBSNMP密码才能与Oracle DBCA一起使用。您可能需要指定以下附加密码，具体取决于您的系统配置：

  • 如果数据库使用Oracle自动存储管理（Oracle ASM）进行存储，则必须为S_ASMSNMPPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

  • 如果使用一个或多个可插拔数据库（PDB）创建多租户容器数据库（CDB），则必须为S_PDBADMINPASSWORD变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

Oracle强烈建议您使用密码响应文件来维护安全性：

  • 响应文件的权限应设置为600。

  • 响应文件的所有者应该是安装所有者用户，并将该组设置为中央库存（ oraInventory ）组。

关于具有作业角色分离的Oracle安装

作业角色分离要求您为通过操作系统授权授予的每组系统权限创建不同的操作系统组。

通过Oracle Grid Infrastructure作业角色分离，Oracle ASM具有单独的操作系统组，这些操作系统组为存储层管理的Oracle ASM系统特权提供操作系统授权。此操作系统授权与Oracle数据库操作系统授权分开。此外，Oracle Grid Infrastructure安装所有者还提供操作系统用户授权，以修改Oracle Grid Infrastructure二进制文件。

通过Oracle数据库作业角色分离，每个Oracle数据库安装都具有单独的操作系统组，以提供对该Oracle数据库的系统特权的授权。因此，可以在群集上安装多个数据库，而无需共享系统权限的操作系统授权。此外，每个Oracle软件安装都由单独的安装所有者拥有，以便为修改Oracle Database二进制文件提供操作系统用户授权。请注意，任何Oracle软件所有者都可以启动和停止所有数据库并共享Oracle Grid Infrastructure资源，例如Oracle ASM或虚拟IP（VIP）。作业角色分离配置可实现数据库安全性，并且不会限制用户角色启动和停止各种Oracle Clusterware资源。

您可以选择为存储和数据库层上的所有系统特权创建一个管理用户和一个组以进行操作系统身份验证。例如，您可以将oracle用户指定为所有Oracle软件的安装所有者，并将oinstall指定为其成员被授予Oracle Clusterware所有系统特权的组; Oracle ASM的所有系统权限;服务器上所有Oracle数据库的所有系统权限;以及安装所有者的所有OINSTALL系统权限。该组还必须是Oracle Inventory组。

如果您不想使用角色分配组，那么Oracle强烈建议您至少使用两个组：

  • 系统特权组，其成员被授予管理系统特权，包括OSDBA，OSASM和其他系统特权组。

  • 安装所有者组（ oraInventory组），其成员被授予Oracle安装所有者系统特权（ OINSTALL系统特权）。

句法


to_approx_percentile.eps插图的描述

目的

TO_APPROX_PERCENTILE将包含近似百分位数信息，百分位数值和排序规范的详细信息作为输入，并返回一个近似插值，该值将根据排序规范落入该百分位数值。

有关detail ，请指定BLOB类型的详细detail ，该详细信息由APPROX_PERCENTILE_DETAIL函数或APPROX_PERCENTLE_AGG函数创建。

对于expr ，请指定百分位数值，该值必须计算为0到1之间的数值。如果指定ERROR_RATE或CONFIDENCE子句，则百分位数值不适用。在这种情况下，对于expr您必须指定null或0到1之间的数值。但是，该值将被忽略。

对于datatype ，请在详细信息中指定近似百分位数信息的数据类型。这是提供给发起详细信息的APPROX_PERCENTILE_DETAIL函数的表达式的数据类型。有效数据类型为NUMBER ， BINARY_FLOAT ， BINARY_DOUBLE ， DATE ， TIMESTAMP ， INTERVAL YEAR TO MONTH和INTERVAL DAY TO SECOND 。

DESC | ASC

指定插值的排序规范。为降序排序顺序指定DESC ，或为升序排序顺序指定ASC 。ASC是默认值。

ERROR_RATE |置信度

这些条款可让您确定细节百分位评估的准确性。如果指定其中一个子句，则该函数返回一个从0到1的十进制值，而不是返回近似插值，而是表示以下值之一：

  • 如果指定ERROR_RATE ，则返回值表示详细信息的百分位数评估的错误率。

  • 如果指定CONFIDENCE ，则返回值表示指定ERROR_RATE时返回的错误率的置信度。

如果指定ERROR_RATE或CONFIDENCE ，则忽略百分位数值expr 。

目的

NVL允许您使用查询结果中的字符串替换null（作为空白返回）。如果expr1为null，则NVL返回expr2 。如果expr1不为null，则NVL返回expr1 。

参数expr1和expr2可以具有任何数据类型。如果它们的数据类型不同，则Oracle数据库会隐式地将其中一个转换为另一个。如果无法隐式转换它们，则数据库将返回错误。隐式转换实现如下：

  • 如果expr1是字符数据，则Oracle数据库在比较它们之前将expr2转换为expr1的数据类型，并在expr1的字符集中返回VARCHAR2 。

  • 如果expr1是数字，则Oracle数据库确定哪个参数具有最高的数字优先级，隐式地将另一个参数转换为该数据类型，并返回该数据类型。

为集群时间同步设置网络时间协议

使用Oracle Cluster Time Synchronization或网络时间协议（NTP）选项进行时间同步。

Oracle Clusterware要求在所有群集节点上设置相同的时区环境变量。在安装过程中，安装过程会在运行Oracle Universal Installer（OUI）的节点上获取Grid安装所有者的时区（TZ）环境变量设置，并将所有节点上的时区值用作默认TZ环境变量设置适用于Oracle Clusterware管理的所有进程。时区默认用于数据库，Oracle ASM和任何其他托管进程。您有两种时间同步选项：

  • 操作系统配置网络时间协议（NTP），例如chronyd或ntpd

  • Oracle集群时间同步服务

Oracle Cluster Time Synchronization Service专为群集服务器无法访问NTP服务的组织而设计。如果使用NTP，则Oracle Cluster Time Synchronization守护程序（ctssd）将以观察者模式启动。如果您没有NTP守护程序，则ctssd将以活动模式启动并在集群成员之间同步时间，而无需联系外部时间服务器。

注意：

  • 在开始安装Oracle Grid Infrastructure之前，Oracle建议您确保所有节点上的时钟设置为同一时间。

  • 默认情况下，Oracle Linux 7和Red Hat Linux 7上提供的NTP服务是chronyd 。

如果您的服务器上有NTP守护程序，但您无法将它们配置为与时间服务器同步时间，并且您希望使用群集时间同步服务在群集中提供同步服务，则停用并卸载NTP。

当安装程序发现NTP协议未处于活动状态时，群集时间同步服务将以活动模式安装，并在节点之间同步时间。如果发现NTP已配置，则群集时间同步服务将以观察者模式启动，并且群集中的Oracle Clusterware不会执行活动时间同步。

要停用chronyd服务，必须停止现有的chronyd服务，并从初始化序列中禁用它。

    #systemctl stop chronyd #systemctl disable chronyd

使用Oracle Cluster Health Advisor可以提高可用性和性能管理。

Oracle Cluster Health Advisor根据默认模型估算观察输入的预期值，默认模型是基于目标系统正常运行周期的经过训练的校准模型。然后，Oracle Cluster Health Advisor根据观察值和预期值之间的差异对每个输入执行异常检测。如果与特定问题相关的足够输入异常，则Oracle Cluster Health Advisor会发出警告并生成即时针对性诊断和纠正措施。

3.4使用市场篮子数据

马克等篮子数据标识在一组篮子或交易中出售的物品。Oracle Data Mining为市场购物篮分析提供关联挖掘功能。

关联模型使用 Apriori算法生成关联规则，描述如何在组中购买项目。例如，关联规则可以断言购买花生酱的人有80％也可能购买果冻。

市场篮子数据通常是交易性的 。在交易中数据，案例是事务，事务的数据存储在多行中。Oracle Data Mining关联模型可以基于事务数据或单记录案例数据构建。ODMS_ITEM_ID_COLUMN_NAME和ODMS_ITEM_VALUE_COLUMN_NAME设置指定关联规则的数据是否为事务格式。

注意：

关联模型是可以在本机事务数据上构建的唯一模型类型。对于所有其他类型的模型，Oracle Data Mining要求数据以单记录格式呈现。

Apriori算法假设数据是事务性的，并且它有许多缺失值。Apriori将所有缺失值解释为稀疏数据，并且它有自己的处理稀疏数据的本机机制。

为集群域委派配置DNS到网格命名服务

如果您计划将网格命名服务（GNS）与委派域一起使用，那么在安装Oracle Grid Infrastructure之前，请配置您的域名服务器（DNS）以发送到子域GNS服务的GNS名称解析请求，这些请求是集群成员节点。

动态公共网络（DHCP，自动配置）必须使用GNS域委派。静态公共网络（静态地址，手动配置）不需要GNS域委派。
以下是对域委派执行的步骤的概述。您的实际程序可能与此示例不同。

配置DNS以使用委派发送GNS名称解析请求：

  1. 在DNS中，为GNS虚拟IP地址创建一个条目，其中地址使用gns-server.clustername.domainname形式。例如，如果群集名称为mycluster ，域名为example.com，且IP地址为192.0.2.1 ，请创建类似于以下内容的条目：

    mycluster-gns-vip.example.com A 192.0.2.1

    您提供的地址必须是可路由的。

  2. 设置GNS子域转发到GNS虚拟IP地址，以便GNS将地址解析为GNS子域。为此，请为委派域创建类似于以下内容的BIND配置条目，其中cluster01.example.com是您要委派的子域：

    cluster01.example.com NS mycluster-gns-vip.example.com

  3. 使用GNS时，必须在群集中的节点（或系统上提供解决方案信息的文件）上配置resolve.conf ，以包含可解析为公司DNS服务器的名称服务器条目。配置的总超时时间 - 选项尝试（重试）和选项超时（指数退避）的组合应小于30秒。例如，如果xxx.xxx.xxx.42和xxx.xxx.xxx.15是网络中的有效名称服务器地址，请在/etc/resolv.conf提供类似于以下内容的条目：

    选项尝试：2个选项超时：1个搜索cluster01.example.com example.com nameserver xxx.xxx.xxx.42 nameserver xxx.xxx.xxx.15

    /etc/nsswitch.conf控制名称服务查找顺序。在某些系统配置中，网络信息系统（NIS）可能会导致SCAN地址解析出现问题。Oracle建议您将NIS条目放在搜索列表的末尾。例如：

    /etc/nsswitch.conf hosts：files dns nis
请注意，在进行电缆拉力测试时，使用NIS是常见问题的根源，因为主机名和用户名解析可能会失败。

在NAS设备上创建文件以与Oracle自动存储管理一起使用

如果您拥有经过认证的NAS存储设备，则可以在NFS安装目录中创建零填充文件，并将这些文件用作Oracle ASM磁盘组中的磁盘设备。

在安装Oracle Grid Infrastructure 19c期间，Oracle Universal Installer（OUI）可以在您指定的NFS安装目录中创建文件。以下过程说明如何在NFS安装目录中手动创建文件以用作Oracle ASM磁盘组中的磁盘设备：
  1. 如有必要，请在NAS设备上为磁盘组文件创建导出目录。
  2. 将用户切换到root 。
  3. 在本地系统上创建挂载点目录。

    例如：

    #mkdir -p / mnt / oracleasm
  4. 要确保在系统重新启动时装入NFS文件系统，请在装入文件/etc/fstab为文件系统添加一个条目。
  5. 输入类似于以下内容的命令，以在本地系统上挂载NFS：
    #mount / mnt / oracleasm
  6. 选择要创建的磁盘组的名称，并使用磁盘组名称作为目录名称为NFS文件系统上的文件创建目录。
    例如，如果要为销售数据库设置磁盘组：
    #mkdir / mnt / oracleasm / sales1
  7. 使用类似于以下命令在此目录中创建所需数量的零填充文件：
    #dd if = / dev / zero of = / mnt / oracleasm / sales1 / disk1 bs = 1024k count = 1000

    此示例在NFS文件系统上创建1 GB文件。您必须分别创建一个，两个或三个文件以创建外部，普通或高冗余磁盘组。

    注意：

    在同一NAS设备上创建多个零填充文件无法防止NAS故障。而是为每个NAS设备创建一个文件，并使用Oracle ASM技术对其进行镜像。
  8. 输入与以下内容类似的命令，以更改您创建的目录和文件的所有者，组和权限：
    #chown -R grid：asmadmin / mnt / oracleasm＃chmod -R 660 / mnt / oracleasm
    在此示例中，安装所有者是grid ，OSASM组是asmadmin 。
  9. 在Oracle数据库安装期间，编辑Oracle ASM磁盘发现字符串以指定与您创建的文件名匹配的正则表达式。

    例如：

    的/ mnt / oracleasm程序/销售额1 /

关于具有作业角色分离的Oracle安装

作业角色分离要求您为通过操作系统授权授予的每组系统权限创建不同的操作系统组。

通过Oracle Grid Infrastructure作业角色分离，Oracle ASM具有单独的操作系统组，这些操作系统组为存储层管理的Oracle ASM系统特权提供操作系统授权。此操作系统授权与Oracle数据库操作系统授权分开。此外，Oracle Grid Infrastructure安装所有者还提供操作系统用户授权，以修改Oracle Grid Infrastructure二进制文件。

通过Oracle数据库作业角色分离，每个Oracle数据库安装都具有单独的操作系统组，以提供对该Oracle数据库的系统特权的授权。因此，可以在群集上安装多个数据库，而无需共享系统权限的操作系统授权。此外，每个Oracle软件安装都由单独的安装所有者拥有，以便为修改Oracle Database二进制文件提供操作系统用户授权。请注意，任何Oracle软件所有者都可以启动和停止所有数据库并共享Oracle Grid Infrastructure资源，例如Oracle ASM或虚拟IP（VIP）。作业角色分离配置可实现数据库安全性，并且不会限制用户角色启动和停止各种Oracle Clusterware资源。

您可以选择为存储和数据库层上的所有系统特权创建一个管理用户和一个组以进行操作系统身份验证。例如，您可以将oracle用户指定为所有Oracle软件的安装所有者，并将oinstall指定为其成员被授予Oracle Clusterware所有系统特权的组; Oracle ASM的所有系统权限;服务器上所有Oracle数据库的所有系统权限;以及安装所有者的所有OINSTALL系统权限。该组还必须是Oracle Inventory组。

如果您不想使用角色分配组，那么Oracle强烈建议您至少使用两个组：

  • 系统特权组，其成员被授予管理系统特权，包括OSDBA，OSASM和其他系统特权组。

  • 安装所有者组（ oraInventory组），其成员被授予Oracle安装所有者系统特权（ OINSTALL系统特权）。

注意：

要配置网络目录服务（如网络信息服务（NIS））上的安装用户，请参阅目录服务文档。

配置DNS以使用委派发送GNS名称解析请求：

  1. 在DNS中，为GNS虚拟IP地址创建一个条目，其中地址使用gns-server.clustername.domainname形式。例如，如果群集名称为mycluster ，域名为example.com，且IP地址为192.0.2.1 ，请创建类似于以下内容的条目：

    mycluster-gns-vip.example.com A 192.0.2.1

    您提供的地址必须是可路由的。

  2. 设置GNS子域转发到GNS虚拟IP地址，以便GNS将地址解析为GNS子域。为此，请为委派域创建类似于以下内容的BIND配置条目，其中cluster01.example.com是您要委派的子域：

    cluster01.example.com NS mycluster-gns-vip.example.com

    选项尝试：2个选项超时：1个搜索cluster01.example.com example.com nameserver xxx.xxx.xxx.42 nameserver xxx.xxx.xxx.15

    /etc/nsswitch.conf控制名称服务查找顺序。在某些系统配置中，网络信息系统（NIS）可能会导致SCAN地址解析出现问题。Oracle建议您将NIS条目放在搜索列表的末尾。例如：

    /etc/nsswitch.conf hosts：files dns nis

Oracle如何解决同步信息集成问题

使用Oracle数据库网关，您可以访问分布式数据库系统中的任何位置，而无需了解数据的位置或数据的存储方式。

如果客户端程序必须访问或修改多个Oracle数据库上的数据，则可以打开每个数据库的连接。但是，这种方法有几个缺点，包括：

  • 要从数据库连接数据，客户端必须具有允许它的逻辑。

  • 为了保证数据完整性，客户端必须具有事务协调逻辑。

Oracle提供了另一种称为分布式处理的方法，其中客户端连接到一个Oracle数据库，并将加入数据和事务协调的负担转移到该数据库。客户端程序连接的数据库称为本地数据库。除此之外的任何数据库都是远程数据库。客户端程序可以使用数据库链接访问任何远程数据库中的对象。Oracle查询处理器负责连接，其事务引擎负责事务协调。

Oracle用于解决异构连接问题的方法是允许非Oracle系统成为前面描述的场景中的远程节点之一。远程非Oracle系统用作远程Oracle系统。非Oracle系统使用与Oracle系统相同的SQL方言和相同的数据字典结构。通过异构服务访问非Oracle系统。

在大多数情况下，异构服务组件完成的工作对最终用户完全透明。除了少数例外（这些在后面的章节中提到），与访问Oracle系统所需的访问非Oracle系统不需要做任何不同的事情。异构服务组件用作实现Oracle对非Oracle数据库的访问的基础。

Oracle数据库网关与Oracle数据库的异构服务组件配合使用，以访问Oracle Database网关所针对的特定的商用非Oracle系统。例如，您使用Oracle Database Gateway for Sybase访问Sybase数据库。Oracle还提供了一个用于ODBC的Oracle数据库网关，使您可以使用ODBC驱动程序访问非Oracle数据库。

注意：

Oracle不提供Oracle Database Gateway for ODBC所需的ODBC驱动程序。用户必须从其他供应商处获取这些驱动程

Oracle还提供异步信息集成产品。本指南中未讨论这些产品。简而言之，这些产品包括：

  • Oracle GoldenGate

    Oracle GoldenGate提供跨异构系统的数据库事务的实时捕获，转换，路由和交付。该软件可促进高性能，低影响力的数据移动，同时保持各种数据库和平台的低延迟，同时保持事务完整性。

  • 消息网关

    Messaging Gateway支持Oracle数据库和非Oracle消息传递系统之间的通信。

  • 开放系统接口

    Oracle提供了许多开放接口，例如OCI，JDBC和ODBC，使客户能够使用第三方应用程序或编写自己的客户端应用程序来访问Oracle数据库。

您可以在响应文件模式下运行Oracle数据库配置助手（Oracle DBCA），以在系统上配置和启动Oracle数据库。

要在响应文件模式下运行Oracle DBCA，您必须复制和编辑响应文件模板。Oracle在ORACLE_HOME/assistants/dbca目录中提供名为dbca.rsp的响应文件模板。要在响应文件模式下运行Oracle DBCA，必须将-responseFile标志与-silent标志结合使用。您还必须使用图形显示并设置DISPLAY环境变量。

要以响应文件模式运行Oracle DBCA：

  1. 将dbca.rsp响应文件模板从响应文件目录复制到系统上的目录：
    $ cp / directory_path /assistants/dbca/dbca.rsp local_directory

    在此示例中， directory_path是您复制安装二进制文件的目录的路径。

    作为编辑响应文件模板的替代方法，您还可以通过在运行Oracle DBCA时将所有必需信息指定为命令行选项来创建数据库。有关支持的选项列表的信息，请输入以下命令：

    $ $ ORACLE_HOME / bin / dbca -help
  2. 在文本编辑器中打开响应文件：
    $ vi / local_dir /dbca.rsp
  3. 按照文件中的说明编辑文件。

    注意：

    如果未正确配置响应文件，Oracle DBCA将失败。

  4. 以Oracle软件所有者用户身份登录，并设置ORACLE_HOME环境变量以指定正确的Oracle主目录。
  5. 要在响应文件模式下运行Oracle DBCA，请设置DISPLAY环境变量。
  6. 使用以下命令语法使用响应文件以静默或响应文件模式运行Oracle DBCA：
    $ ORACLE_HOME / bin / dbca [-silent] -responseFile / local_dir /dbca.rsp

    在这个例子中：

    • -silent选项表示Oracle DBCA以静默方式运行。

    • local_dir是复制dbca.rsp响应文件模板的目录的完整路径。

关于具有作业角色分离的Oracle安装

作业角色分离要求您为通过操作系统授权授予的每组系统权限创建不同的操作系统组。

通过Oracle Grid Infrastructure作业角色分离，Oracle ASM具有单独的操作系统组，这些操作系统组为存储层管理的Oracle ASM系统特权提供操作系统授权。此操作系统授权与Oracle数据库操作系统授权分开。此外，Oracle Grid Infrastructure安装所有者还提供操作系统用户授权，以修改Oracle Grid Infrastructure二进制文件。

通过Oracle数据库作业角色分离，每个Oracle数据库安装都具有单独的操作系统组，以提供对该Oracle数据库的系统特权的授权。因此，可以在群集上安装多个数据库，而无需共享系统权限的操作系统授权。此外，每个Oracle软件安装都由单独的安装所有者拥有，以便为修改Oracle Database二进制文件提供操作系统用户授权。请注意，任何Oracle软件所有者都可以启动和停止所有数据库并共享Oracle Grid Infrastructure资源，例如Oracle ASM或虚拟IP（VIP）。作业角色分离配置可实现数据库安全性，并且不会限制用户角色启动和停止各种Oracle Clusterware资源。

您可以选择为存储和数据库层上的所有系统特权创建一个管理用户和一个组以进行操作系统身份验证。例如，您可以将oracle用户指定为所有Oracle软件的安装所有者，并将oinstall指定为其成员被授予Oracle Clusterware所有系统特权的组; Oracle ASM的所有系统权限;服务器上所有Oracle数据库的所有系统权限;以及安装所有者的所有OINSTALL系统权限。该组还必须是Oracle Inventory组。

如果您不想使用角色分配组，那么Oracle强烈建议您至少使用两个组：

  • 系统特权组，其成员被授予管理系统特权，包括OSDBA，OSASM和其他系统特权组。

  • 安装所有者组（ oraInventory组），其成员被授予Oracle安装所有者系统特权（ OINSTALL系统特权）。

注意：

要配置网络目录服务（如网络信息服务（NIS））上的安装用户，请参阅目录服务文档。

关于Oracle RAC数据库的侦听器配置

Oracle数据库通过本地侦听器接收连接请求。

本地侦听器代理客户端请求，将请求切换到服务器。侦听器配置有协议地址，配置了相同协议地址的客户端可以向侦听器发送连接请求。建立连接后，客户端和Oracle数据库将直接相互通信。

安装了Oracle Grid Infrastructure后，本地侦听器或默认侦听器位于Grid主目录中。本地侦听器配置为响应数据库连接请求和非数据库连接请求，例如外部过程或Oracle XML数据库（XDB）请求。当数据库启动时，数据库代理程序（ oraagent ，以前称为racgimon ）设置LOCAL_LISTENER参数不需要的Oracle Net服务名称连接描述符。LOCAL_LISTENER的值计算为Grid主侦听器的端点。

您可以在一个listener.ora文件中配置多个Oracle数据库侦听器，每个侦听器都具有唯一的名称。数据库侦听器的多个侦听器配置是可能的，因为每个顶级配置参数都具有侦听器名称的后缀或者是侦听器名称本身。要配置数据库以向多个本地侦听器注册，您必须手动修改LOCAL_LISTENER参数。

注意：

Oracle建议在大多数客户环境中为每个节点仅运行一个侦听器。

对于Oracle RAC数据库，数据库参数REMOTE_LISTENER标识SCAN侦听器。数据库使用这些参数中包含的连接描述信息向本地和SCAN侦听器注册。Oracle Database 11g第2版及更高版本实例仅向SCAN侦听器注册为远程侦听器。升级后的数据库将SCAN侦听器注册为远程侦听器，并继续向所有节点侦听器注册。

警告：

请勿修改Oracle RAC部署的REMOTE_REGISTRATION_ADDRESS参数。默认设置REMOTE_REGISTRATION_ADDRESS参数以防止TNS中毒。有关REMOTE_REGISTRATION_ADDRESS参数的详细信息，请参阅Oracle数据库网络服务参考 。

Oracle RAC数据库的REMOTE_LISTENER参数始终设置为SCAN地址。例如，如果群集的SCAN是myscan ，并且群集的GNS子域是mycluster.example.com ，则REMOTE_LISTENER参数具有以下值：

myscan.mycluster.example.com:1521

注意：

不要将Oracle RAC数据库的REMOTE_LISTENER参数设置为具有单个地址的Oracle Net别名，该地址使用SCAN作为主机名（ HOST= scan ）。

REMAINDER返回n2的余数除以n1 。

此函数将任何数值数据类型或任何可以隐式转换为数字数据类型的非数字数据类型作为参数。Oracle确定具有最高数字优先级的参数，隐式将剩余参数转换为该数据类型，并返回该数据类型。

MOD函数类似于REMAINDER只是它在公式中使用FLOOR ，而REMAINDER使用ROUND 。参见MOD 。

11 Oracle XA库

Oracle XA库是一个外部接口，允许Oracle服务器以外的事务管理器协调全局事务。XA库支持分布式事务中的非Oracle资源管理器。这在多个数据库和资源之间的事务中特别有用。

Oracle XA库的实现符合X / Open分布式事务处理（DTP）软件架构的XA接口规范。Oracle XA库作为Oracle数据库企业版的一部分安装。

Oracle数据库中的异构服务由事务服务和SQL服务组成。

    异构服务组件的事务服务组件使非Oracle系统能够集成到Oracle数据库事务和会话中。

交易服务

异构服务组件的事务服务组件使非Oracle系统能够集成到Oracle数据库事务和会话中。

当您第一次使用Oracle用户会话中的数据库链接访问非Oracle系统时，您将在非Oracle系统中透明地设置经过身份验证的会话。在Oracle用户会话结束时，将关闭非Oracle数据库系统中经过身份验证的会话。

此外，一个或多个非Oracle系统可以参与Oracle分布式事务。当应用程序提交事务时，Oracle的两阶段提交协议访问非Oracle数据库系统以透明地协调分布式事务。即使在非Oracle系统不支持Oracle两阶段提交协议的所有方面的情况下，Oracle也可以（有一些限制）支持与非Oracle系统的分布式事务。

SQL服务

结构化查询语言（SQL）服务处理所有与SQL相关的操作。

SQL服务完成的工作包括：

  • 将Oracle内部SQL相关调用映射到异构服务驱动程序应用程序编程接口（API）。然后，此API由驱动程序映射到非Oracle系统的客户端API。

  • 将Oracle SQL语言中的SQL语句转换为非Oracle系统的SQL方言。

  • 将引用Oracle数据字典表的查询转换为从非Oracle系统的数据字典中提取必要信息的查询。

  • 将数据从非Oracle系统数据类型转换为Oracle数据类型并返回。

  • 通过发出多个查询来获取必要的数据并进行后处理以获得所需的结果，从而补偿非Oracle系统的缺失功能。
