1.77 DB_FILE_MULTIBLOCK_READ_COUNT

DB_FILE_MULTIBLOCK_READ_COUNT指定在顺序扫描期间在一次I / O操作中读取的最大块数。

属性 描述

参数类型

整数

默认值

默认值对应于可以有效执行的最大I / O大小，并且与平台相关

可修改

ALTER SESSION ， ALTER SYSTEM

可在PDB中修改

是

价值范围

依赖于操作系统

基本

没有

DB_FILE_MULTIBLOCK_READ_COUNT是可用于在表扫描期间最小化I / O的参数之一。执行全表扫描所需的I / O总数取决于诸如表的大小，多块读取计数以及是否正在将并行执行用于操作等因素。

此参数的默认值是与可以有效执行的最大I / O大小相对应的值。此值取决于平台，对于大多数平台而言为1MB。

因为参数以块表示，所以它将被设置为一个值，该值等于可以有效执行的最大I / O大小除以标准块大小。请注意，如果会话数量非常大，则会减少多块读取计数值，以避免缓冲区缓存因过多的表扫描缓冲区而泛滥。

即使默认值可能很大，如果不设置此参数，优化程序也不会支持大I / O.仅当您将此参数显式设置为较大值时，它才会支持大I / O.

对于此参数，联机事务处理（OLTP）和批处理环境通常具有4到16的值。DSS和数据仓库环境倾向于从最大化此参数的值中获益最多。如果此参数的值很高，优化器更有可能在索引上选择全表扫描。

最大值是操作系统的最大I / O大小，表示为Oracle块（（最大I / O大小）/ DB_BLOCK_SIZE ）。如果将此参数设置为大于最大值的值，则Oracle将使用最大值。

6.1存储过程和运行时上下文

存储过程是发布到SQL并存储在数据库中以供一般使用的Java方法。要发布Java方法，可以编写调用规范，将Java方法名称，参数类型和返回类型映射到SQL对应项。

与包装器不同，后者增加了另一层执行，调用规范发布了Java方法的存在。因此，当您通过其调用规范调用该方法时，运行时系统将以最小的开销调度该调用。

当客户端应用程序调用时，存储过程可以接受参数，引用Java类并返回Java结果值。

6.1.1功能和程序

函数和过程是命名块，用于封装一系列语句。它们是构建块，可用于构建模块化，可维护的应用程序。

通常，您使用过程来执行操作，使用函数来计算值。因此，您使用void Java方法的过程调用规范和值返回方法的函数调用规范。

只有顶级和包级PL / SQL函数和过程可用作调用规范。使用SQL CREATE FUNCTION ， CREATE PROCEDURE或CREATE PACKAGE语句定义它们时，它们存储在数据库中，供一般使用。

必须显式调用作为函数和过程发布的Java方法。他们可以接受参数并可以从以下方面调用：

  • SQL数据操作语言（DML）语句

  • SQL CALL语句

  • PL / SQL块，子程序和包

6.1.2数据库触发器

数据库触发器是与特定表或视图关联的存储过程。只要给定的DML操作修改表或视图，Oracle数据库就会自动调用触发器。

触发器包含以下部分：

  • 触发事件，通常是DML操作

  • 可选的触发器约束

  • 触发动作

事件发生时，将调用触发器。触发器中的CALL语句通过方法的调用规范调用Java方法来执行操作。

数据库触发器用于实施复杂的业务规则，自动派生列值，防止无效事务，透明地记录事件，审计事务和收集统计信息。

6.1.3对象关系方法

一个 SQL对象类型是一种用户定义的复合数据类型，它封装了一组变量，称为ttributes，带有一组操作，称为方法，可以用Java编写。由该组属性形成的数据结构是public 。但是，作为一种良好的编程习惯，您必须确保应用程序不直接操作这些属性并使用提供的方法集。

您可以为某些现实世界对象创建一个抽象模板作为SQL对象类型。模板仅指定对象在应用程序环境中需要的那些属性和方法。在运行时，使用值填充数据结构时，将创建对象类型的实例。您可以根据需要创建任意数量的实例。

通常，对象类型对应于某个业务实体，例如采购订单。为了容纳可变数量的项，对象类型可以使用VARRAY ，嵌套表或两者。

例如，采购订单对象类型可以包含可变数量的行项目。

目的

BITAND函数将其输入和输出视为位向量;输出是输入的按位AND 。

expr1和expr2的类型为NUMBER ，结果的类型为NUMBER 。如果BITAND任一参数为NULL ，则结果为NULL 。

参数必须在 - （2 （n-1） ）...（（2 （n-1） ） - 1）范围内。如果参数超出此范围，则结果未定义。

结果按几个步骤计算。首先，每个参数A被替换为值SIGN(A)*FLOOR(ABS(A)) 。此转换具有将每个参数截断为零的效果。接下来，每个参数A（现在必须是整数值）被转换为n位二进制补码二进制整数值。使用按位AND运算组合两个位值。最后，得到的n位二进制补码值被转换回NUMBER 。

  • BITAND的当前实现定义n = 128。

  • PL / SQL支持BITAND的重载，输入和结果的类型都是BINARY_INTEGER ，其中n = 32。

3.1分片为分布式分区

Sharding是一种基于跨多个独立物理数据库的数据水平分区的数据库扩展技术。这种配置中的每个物理数据库称为分片。

尽管分片数据库（SDB）看起来像一个单一的数据库应用程序和应用程序开发人员，从数据库管理员的角度来看，它是一组离散Oracle数据库中，每一个被称为碎片 。分片表分布在SDB的所有分片上。每个分片上的表分区与可以在未分片的Oracle数据库中使用的分区没有区别。

上市

模式

指定包含同义词的模式。如果省略schema ，则Oracle数据库假定同义词位于您自己的模式中。
