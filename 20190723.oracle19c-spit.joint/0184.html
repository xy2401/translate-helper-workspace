12 CTX_QUERY包

本章介绍CTX_QUERY PL / SQL包，可用于生成查询反馈，计数命中和创建存储的查询表达式。

CTX_QUERY包中包含以下过程和函数：

名称 描述

BROWSE_WORDS

返回索引中种子词周围的单词。

COUNT_HITS

返回查询的匹配数。

说明

生成查询表达式解析和扩展信息。

HFEEDBACK

生成分层查询反馈信息（更广泛的术语，更窄的术语和相关术语）。

REMOVE_SQE

从SQL表中删除指定的存储查询表达式。

RESULT_SET

执行查询并生成结果集。

RESULT_SET_CLOB_QUERY

执行查询并基于CLOB查询参数生成结果集。

RESULT_SET_DOCUMENT

显式关闭CONTAINS查询游标后，如果查询模板具有<ctx_result_set_descriptor>元素，则保留结果集文档。

STORE_SQE

执行查询并将结果存储在存储的查询表达式表中。

注意：

仅当索引类型为CONTEXT时，才能使用此包。此程序包不支持CTXCAT索引类型。

CTX_QUERY包中的API不支持以架构或所有者名称为前缀的标识符。

12.1 BROWSE_WORDS

此过程使您可以浏览Oracle Text索引中的单词。指定种子词， BROWSE_WORDS返回索引中围绕它的单词，以及包含每个单词的文档数的近似计数。

此功能对于优化查询很有用。您可以识别以下单词：

  • 非选择性单词（文档计数较低的单词）

  • 文档集中拼写错误的单词

语法1：将结果存储在表中

ctx_query.browse_words（
index_name IN VARCHAR2，种子IN VARCHAR2，restab IN VARCHAR2，browse_id IN NUMBER DEFAULT 0，numwords IN NUMBER DEFAULT 10，direction IN VARCHAR2 DEFAULT BROWSE_AROUND，part_name IN VARCHAR2 DEFAULT NULL
）;

语法2：将结果存储在内存中

ctx_query.browse_words（
index_name IN VARCHAR2，种子IN VARCHAR2，resarr IN OUT BROWSE_TAB，numwords IN NUMBER DEFAULT 10，方向IN VARCHAR2 DEFAULT BROWSE_AROUND，part_name IN VARCHAR2 DEFAULT NULL
）;

指定索引的名称。您可以指定schema.name 。必须是本地索引。

指定种子字。在浏览扩展之前，这个词是lexed。令牌表中不需要存在该词。 种子必须是一个单词。使用多个单词作为种子将导致错误。

restab

指定结果表的名称。您可以输入restab作为schema.name 。在调用此过程之前，该表必须存在，并且您必须对表具有INSERT权限。此表必须具有以下架构。

12.2 COUNT_HITS

返回指定查询的命中数。您可以在完全或估计模式下调用COUNT_HITS 。精确模式返回查询的确切命中数。估计模式返回上限估计，但运行速度比精确模式快。

句法

语法1

exec CTX_QUERY.COUNT_HITS（index_name IN VARCHAR2，text_query IN VARCHAR2，exact IN BOOLEAN DEFAULT TRUE，part_name IN VARCHAR2 DEFAULT NULL）返回数字;

语法2

exec CTX_QUERY.COUNT_HITS_CLOB_QUERY（index_name IN VARCHAR2，text_query IN CLOB，精确IN BOOLEAN DEFAULT TRUE，part_name IN VARCHAR2 DEFAULT NULL）返回数字;
index_name

指定索引名称。

text_query

指定查询。

确切

为精确计数指定TRUE 。为上限估计指定FALSE 。

指定FALSE返回不太准确的数字但运行速度更快。如果自上次FULL索引优化以来已更新或删除行，则指定FALSE可能会返回一个过高的数字。在完整模式下进行优化会删除这些错误命中，然后将EXACT设置为FALSE将返回与EXACT设置为TRUE相同的数字。

part_name

指定要查询的索引分区的名称。

笔记

如果查询包含结构化条件，则应使用SELECT COUNT(*) 。

如果索引是用。创建的 TRANSACTIONAL参数，然后COUNT_HITS将包括挂起的rowid以及已同步的rowid。

12.3说明

使用CTX_QUERY.EXPLAIN生成查询表达式的解释计划信息。EXPLAIN计划为Text查询表达式提供了解析树的图形表示。此信息存储在结果表中。

此过程不会执行查询。相反，此过程可以告诉您在输入查询之前如何扩展和解析查询。这对于词干，通配符，同义词库，模糊，soundex或关于查询尤其有用。解析树还显示以下信息：

  • 执行顺序（运算符的优先级）

  • ABOUT查询规范化

  • 查询表达式优化

  • 停止词转换

  • 复合词标记的细分

了解Oracle Text如何评估查询对于优化和调试查询非常有用。您还可以设计应用程序，以便它使用说明计划信息来帮助用户编写更好的查询。

句法

语法1

exec CTX_QUERY.EXPLAIN（
index_name IN VARCHAR2，text_query IN VARCHAR2，explain_table IN VARCHAR2，sharelevel IN NUMBER DEFAULT 0，explain_id IN VARCHAR2 DEFAULT NULL，part_name IN VARCHAR2 DEFAULT NULL
）;

语法2

exec CTX_QUERY.EXPLAIN_CLOB_QUERY（index_name IN VARCHAR2，text_query IN CLOB，explain_table IN VARCHAR2，sharelevel IN NUMBER DEFAULT 0，explain_id IN VARCHAR2 DEFAULT NULL，part_name IN VARCHAR2 DEFAULT NULL）;
index_name

指定要查询的索引的名称。

text_query

指定要用作选择行的条件的查询表达式。

在text_query包含通配符，模糊或soundex运算符时，此过程将查看索引表以确定扩展。

通配符，模糊（？）和soundex（！）表达式反馈不像常规查询那样考虑延迟删除。

explain_table

指定用于存储text_query的解析树表示的表的名称。您必须至少对用于存储EXPLAIN结果的表具有INSERT和DELETE权限。

也可以看看：

Oracle文本结果表中的 “ EXPLAIN表 ” ，以获取有关explain表结构的更多信息。

sharelevel

指定explain_table是否由多个EXPLAIN调用共享。指定0表示独占使用，1表示共享使用。默认值为0（一次性使用）。

指定0时，系统会在下次调用EXPLAIN之前自动截断结果表。

为共享使用指定1时，此过程不会截断结果表。只更新具有相同explain_id结果。如果没有相同explain_id结果，则会将新结果添加到EXPLAIN表中。

explain_id

当多个EXPLAIN调用使用相同的共享EXPLAIN表时，指定一个标识EXPLAIN过程返回的解释结果的名称。默认值为NULL 。

part_name

指定要查询的索引分区的名称。

例

12.4 HFEEDBACK

在英语或法语中，此过程为指定的查询生成分层查询反馈信息（更宽泛的术语，更窄的术语和相关术语）。

更广泛的术语，更窄的术语和相关的术语信息是从知识库中获得的。但是，仅返回索引中的知识库术语作为查询反馈信息。这增加了从HFEEDBACK返回的HFEEDBACK产生对当前索引文档集的命中的机会。

分层查询反馈信息对于向用户建议其他查询术语很有用。

句法

语法1

exec CTX_QUERY.HFEEDBACK（index_name IN VARCHAR2，text_query IN VARCHAR2，feedback_table IN VARCHAR2，sharelevel IN NUMBER DEFAULT 0，feedback_id IN VARCHAR2 DEFAULT NULL，part_name IN VARCHAR2 DEFAULT NULL）;

语法2

exec CTX_QUERY.HFEEDBACK_CLOB_QUERY（index_name IN VARCHAR2，text_query IN CLOB，feedback_table IN VARCHAR2，sharelevel IN NUMBER DEFAULT 0，feedback_id IN VARCHAR2 DEFAULT NULL，part_name IN VARCHAR2 DEFAULT NULL）;
index_name

指定要查询的文本列的索引名称。

text_query

指定要用作选择行的条件的查询表达式。

feedback_table

指定用于存储的表的名称 反馈条款。

也可以看看：

Oracle文本结果表中的 “ HFEEDBACK表 ” ，以获取有关explain表结构的更多信息。

sharelevel

指定feedback_table是否由多个HFEEDBACK调用共享。指定0表示独占使用，1表示共享使用。默认值为0（一次性使用）。

指定0时，系统会在下次调用HFEEDBACK之前自动截断反馈表。

12.6 RESULT_SET

此过程执行XML查询并以XML格式生成结果集。结果集界面可以返回难以在SQL中表达的数据视图。

也可以看看：

有关如何使用结果集接口的详细信息，请参阅Oracle Text Application Developer's Guide

句法

CTX_QUERY.RESULT_SET（starttag IN VARCHAR2 DEFAULT'<b>'，endtag IN VARCHAR2 DEFAULT'</ b>'，radius IN INTEGER DEFAULT 25，max_length IN INTEGER DEFAULT 250 part_name IN VARCHAR2 DEFAULT NULL）;
starttag

指定用于标记查询关键字的开始标记。默认为'<b>' 。

endtag

指定用于标记查询关键字的结束标记。默认为'</ b>' 。

半径

指定要在段中的匹配查询的任一侧显示的字符数。命中查询之前的字符数开始于段中显示的第一个匹配查询的第一个字符。因此，命中查询之后的字符计数开始于特定段上显示的最后一个命中查询的最后一个字符。如果半径重叠，则将两个段合并为一个。每侧显示的字符数可以修改+/-10字符，以便最好地匹配句子或单词的开头或结尾。

值0需要特别注意。指定时，半径设置为自动，并在句子之间变化。显示结果的最佳猜测，尝试匹配完整的句子。请注意，匹配查询每一侧的半径长度很可能会显着不同。

默认值为25 。

max_length

以字符指定结果集输出的最大长度。此值当前上限为当前返回类型CTX_DOC.RESULT_SET (VARCHAR2) 。如果输出长于返回类型VARCHAR2 ，则结果将被截断。max_length的默认值为250 。

如果将max_length值设置为非常低的值，则不会生成任何结果集。例如，如果max_length设置为0或者max_length低于查询令牌本身的长度，则根本不会生成任何结果集。

part_name

指定索引分区名称。如果索引是全局的，则part_name必须为NULL 。如果索引已分区且part_name不为NULL ，则仅针对给定分区计算查询。如果索引已分区且part_name为NULL ，则将针对所有分区评估查询。

输入结果集描述符

结果集描述符是一条XML消息，它描述了为结果集计算的内容。结果集描述符中存在的元素及其出现的顺序用作简单模板，指定要包含在输出结果集中的内容。也就是说，应该有命中的rowid列表，然后是计数，然后是令牌计数，依此类推。元素的属性指定特定操作的参数和选项，例如rowid列表中的命中数，估计值与精确计数等等。

      限制返回的最大构面值数。它默认按降序排序。有效属性值是大于零的正整数。

    • 排序方式

      有效的属性值是值和计数。值使用值本身进行排序，适用于每种数据类型。计数（默认）使用每个组的计数排序。

    • 订购

      订单可以是升序或降序。

12.9 STORE_SQE

此过程创建存储或会话持续时间查询表达式（SQE）。仅存储查询定义。

SQE用于存储查询的定义而不存储任何结果。使用CONTAINS SQL运算符引用查询引用查询的定义。通过这种方式，SQE可以轻松定义长期或经常使用的查询表达式。当您不希望删除未使用或不再需要的SQE的维护开销时，创建会话持续时间SQE非常有用。

支持的运算符

存储的查询表达式支持所有CONTAINS查询运算符。存储的查询表达式还支持可以在查询表达式中使用的所有特殊字符和其他组件，包括其他存储的查询表达式。

特权

允许用户创建和删除它们拥有的存储查询表达式。允许用户使用任何人拥有的存储查询表达式。CTXSYS用户可以为任何用户创建或删除存储的查询表达式。

句法

语法1

CTX_QUERY.STORE_SQE（query_name IN VARCHAR2，text_query IN VARCHAR2持续时间IN NUMBER默认CTX_QUERY.DURATION_SESSION）;

语法2

CTX_QUERY.STORE_SQE_CLOB_SYNTAX（query_name IN VARCHAR2，text_query IN CLOB持续时间IN NUMBER默认CTX_QUERY.DURATION_SESSION）;
query_name

指定要创建的存储查询表达式的名称。

text_query

指定要与query_name关联的查询表达式。

持续时间

可能的值为DURATION_SESSION和DURATION_PERSISTENT 。

  • 当duration设置为DURATION_SESSION ，存储的查询表达式存储在PL / SQL包变量中，可用于会话。

  • 当duration设置为DURATION_PERSISTENT ，存储的查询表达式存储在数据库表中，并可由其他数据库会话引用。

  • query_name名称空间在持久性和持续时间的SQE之间共享。如果您尝试使用已由另一个持久性或会话持续时间SQE使用的名称添加持久性或会话持续时间SQE，则会引发错误。

duration_persistent

存在CLOB查询时，请指定持续时间存储在数据库表中。必须在不再需要时删除此SQE。

  • query_name名称空间在持久性和持续时间的SQE之间共享。如果您尝试使用已由另一个持久性或会话持续时间SQE使用的名称添加持久性或会话持续时间SQE，则会引发错误。

5.1.1使用系统编辑器编写脚本

DEFINE _EDITOR = vi

您可以在用户或站点配置文件中包含编辑器定义，以便在启动SQL * Plus时始终启用它。有关详细信息，请参阅SQL * Plus配置以及DEFINE和EDIT命令。

5.3关于在脚本中放置注释

您可以输入评论在脚本中有三种方式：

  • 使用SQL * Plus REMARK命令进行单行注释。

  • 使用SQL注释分隔符/ * ... * /进行单行或多行注释。

  • 使用ANSI / ISO（美国国家标准协会/国际标准组织）评论 - - 用于单行注释。

    在命令行输入的注释不存储在SQL缓冲区中。

5.3.1使用REMARK命令

使用备注在脚本中单独命令一行，然后是注释在同一条线上。要继续对其他行进行注释，请输入其他REMARK命令。不要在单个SQL命令的不同行之间放置REMARK命令。 

REMARK委员会报告;备注每月运行。; COLUMN LAST_NAME HEADING'LAST_NAME'; COLUMN SALARY HEADING'每月SALARY'格式$ 99,999; COLUMN COMMISSION_PCT HEADING'COMMISSION％'FORMAT 90.90;备注仅包括销售人员; SELECT LAST_NAME，SALARY，COMMISSION_PCT FROM EMP_DETAILS_VIEW WHERE JOB_ID ='SA_MAN';

5.3.2使用/ * ...* /

输入SQL注释分隔符，/ * .. * / ，在脚本中的单独行上，与SQL命令在同一行上，或在PL / SQL块中的行上。

您必须在斜杠（/ *）开头注释后输入一个空格。

注释可以跨越多行，但不能嵌套在另一行中：

/ *佣金报告每月运行一次。* / COLUMN LAST_NAME标题'LAST_NAME'; COLUMN SALARY HEADING'每月SALARY'格式$ 99,999; COLUMN COMMISSION_PCT HEADING'COMMISSION％'FORMAT 90.90;备注仅包括销售人员; SELECT LAST_NAME，SALARY，COMMISSION_PCT FROM EMP_DETAILS_VIEW / *仅包括销售人员。* / WHERE JOB_ID ='SA_MAN';

5.3.3使用 - -

你可以使用ANSI / ISO“ - - ” 风格评论在SQL语句，PL / SQL块或SQL * Plus命令中。由于没有结束分隔符，注释不能跨越多行。

对于PL / SQL和SQL，在一行上的命令之后或在一行上输入注释：

- 每月运行的佣金报告DECLARE  - 用于报告月度销售额的块

对于SQL * Plus命令，如果它们自己在一行上，则只能包含“ - - ”样式注释。例如，这些评论是合法的：

- 将LONG的最大宽度设置为777 SET LONG 777

这条评论是非法的：

SET LONG 777  - 将LONG的最大宽度设置为777

如果输入以下SQL * Plus命令，SQL * Plus会将其解释为注释，并且不执行以下命令：

-  SET LONG 777

5.3.4关于放置评论的说明

SQL * Plus没有SQL或PL / SQL命令解析器。它会扫描每个新语句的前几个关键字，以确定命令类型，SQL，PL / SQL或SQL * Plus。某些位置的注释可能会阻止SQL * Plus正确识别命令类型，从而产生意外结果。以下使用说明可以帮助您更有效地使用SQL * Plus注释：

  1. 不要将注释放在语句的前几个关键字中。例如：

    创建或替换2 / * HELLO * / 3程序HELLO 4 BEGIN 5 DBMS_OUTPUT.PUT_LINE（'HELLO'）; 6结束; 7 /警告：使用编译错误创建的过程。

    注释的位置可防止SQL * Plus将命令识别为命令。当SQL * Plus在注释的开头看到斜杠“/”时，它会将PL / SQL块提交给服务器，它将其解释为“/”语句终止符。移动注释以避免此错误。例如：

    创建或替换程序2 / * HELLO * / 3 HELLO作为4 BEGIN 5 DBMS_OUTPUT.PUT_LINE（'HELLO'）; 6结束; 7 /程序已创建。

  2. 不要在语句终止符（句点，分号或斜杠）之后添加注释。例如，如果输入：

    选择'Y'来自DUAL; - 测试

    您收到以下错误：

    选择'Y'来自DUAL; - 测试*第1行的错误：ORA-00911：无效字符

    SQL * Plus期望语句终止符后没有文本，并且无法处理命令。

  3. 不要将语句终止字符放在注释行的末尾或SQL语句或PL / SQL块中的注释之后。例如，如果输入：

    SELECT *  - 评论;

    您收到以下错误：

    - 第2行的注释*错误：ORA-00923：未找到FROM关键字

    分号被解释为语句终止符，SQL * Plus将部分形成的SQL命令提交给服务器进行处理，从而导致错误。

  4. 不要在SQL语句或PL / SQL块中的注释中使用＆符号'＆'。例如，如果输入脚本，例如：

    SELECT REGION_NAME，CITY / * THIS＆THAT * / from EMP_DETAILS_VIEW WHERE SALARY> 12000;

    SQL * Plus将＆符号“＆”后面的文本解释为替换变量，并提示输入变量的值，并且：

    输入值：old 2：/ * THIS＆THAT * / new 2：/ * THIS * / REGION_NAME CITY ------------------------- ------------------------------美洲西雅图美洲西雅图美洲西雅图欧洲牛津欧洲牛津美洲多伦多6排入选。

    您可以设置DEFINE OFF以防止扫描替换字符。

有关替换和终止字符的更多信息，请参阅SET命令中的DEFINE，SQLTERMINATOR和SQLBLANKLINES。

5.9使用替代变量

假设您要编写一个类似于SALES中的查询来列出具有各种作业的员工，而不仅仅是那些工作是SA_MAN的员工。您可以通过在每次运行命令时将不同的值编辑到WHERE子句中来实现，但有一种更简单的方法。

通过使用替换变量代替WHERE子句中的文本SA_MAN，如果将值写入命令本身，则可以得到相同的结果。

替换变量前面有一个或两个＆符号 （＆）。当SQL * Plus在命令中遇到替换变量时，SQL * Plus会执行命令，就好像它包含替换变量的值，而不是变量本身。

例如，如果变量SORTCOL的值为JOB_ID且变量MYTABLE的值为EMP_DETAILS_VIEW，则SQL * Plus执行命令

SELECT＆SORTCOL，SALARY FROM＆MYTABLE WHERE SALARY> 12000;

好像他们一样

SELECT JOB_ID，SALARY FROM EMP_DETAILS_VIEW WHERE SALARY> 12000;

5.9.1何处以及如何使用替代变量

您可以使用替换变量 SQL和SQL * Plus命令中的任何位置，输入的第一个单词除外。当SQL * Plus遇到一个在命令中未定义的替换变量，SQL * Plus会提示您输入值。

您可以在提示符处输入任何字符串，即使是包含空格和标点符号的字符串。如果包含引用的SQL命令应该在变量周围有引号，并且您没有在那里包含它们，则用户必须在提示时包含引号。

Oracle ASM Configuration Assistant使您可以使用“ 配置 ASM 磁盘 组”选项卡配置或创建Oracle ASM磁盘组。

  • 将磁盘添加到磁盘组并从磁盘组中删除磁盘

  • 添加，删除和列出配额组。

  • 添加，删除和移动文件组。

  • 编辑磁盘组属性

  • 管理磁盘组的模板

在“ 创建磁盘组”页面上，您可以输入磁盘组的名称，选择冗余级别，更改磁盘发现路径以在不同目录中查找磁盘，识别磁盘组中的磁盘，输入磁盘组的名称故障组，并指定故障组是否为仲裁类型。

仲裁失败组是一种特殊类型的故障组，这些故障组中的磁盘不包含用户数据，在确定与用户存储有关的冗余要求时，不考虑仲裁失败组。但是，安装磁盘组时会出现仲裁失败组。

如果已为系统配置了Oracle ASM筛选器驱动程序，则可以将Oracle ASM筛选器驱动程序标签添加到磁盘或从磁盘清除Oracle ASM筛选器驱动程序标签。

您可以选择在创建磁盘组时显示高级选项。高级选项部分允许您设置其他选项，例如磁盘组兼容性设置和分配单元（AU）大小。完成创建磁盘组规范后，单击“ 确定” 。

注意：

磁盘组兼容性设置只能进行。提升磁盘组兼容性设置后，无法恢复到较低的兼容性设置。

注意：

Oracle Database Vault如何处理职责分离

职责分离是限制每个用户的权限， 只给任务，他或她负责，并没有更多的 。

在过去十年中，职责分离的重要性日益增加。对于许多组织而言，职责分离是一个不断发展的新概念。数据库整合，法规遵从和外包只是增加职责分离的几个驱动因素。Oracle Database Vault职责分离通过将安全相关的管理与日常的DBA操作分开来增强安全性。您可以定制Database Vault职责分离实施，以轻松适应当前和未来的业务需求。特别是小型组织在尝试用有限的资源增加其安全性时需要灵活性。

Oracle Database Vault环境中的任务分离

Oracle Database Vault定义了几个主要职责。

这些责任如下：

  • 帐户管理。 帐户管理需要创建，修改和删除用户帐户。DV_ACCTMGR角色提供这些权限。在Oracle Database Vault注册过程中，将创建主要的日常DV_ACCTMGR用户和备份DV_ACCTMGR用户。作为安全措施，您可以保留并维护备份帐户，以防主DV_ACCTMGR帐户所有者忘记密码或离开公司。

  • 安全管理。 安全管理涵盖基本的安全任务，例如创建领域和命令规则，为数据库用户访问设置安全策略，以及授权数据库用户执行允许的作业。安全管理员还运行安全审核报告。DV_OWNER和DV_ADMIN角色提供这些权限。在Oracle Database Vault注册过程中，将创建主要的日常DV_OWNER用户和备份DV_OWNER用户。

    重要：

    作为安全措施，您应该保留并维护备份用户帐户，以防主DV_OWNER帐户所有者忘记密码或离开公司。同样重要的是，您不要失去对已授予DV_OWNER角色的所有用户帐户的访问权限。有没有办法恢复DV_OWNER如果您无法访问（如使用密码丢失或员工离职），以具有任何帐户角色DV_OWNER作用。如果您无法访问DV_OWNER角色，则无法修改任何Database Vault控件或禁用Database Vault。要解决此问题，您可以将数据库恢复到数据库拥有Database Vault所有者帐户的最后一个已知点。

    （可选）您可以合并帐户管理和安全管理职责。

  • 数据库管理。 数据库管理是指管理数据库系统但不访问业务数据。它包括以下操作：

    • 备份操作需要预定义的时间来使用预定义工具执行备份。

    • 调整和监控操作需要持续的性能监控和分析。

    • 修补操作仅在修补发生期间需要临时访问

    Oracle强烈建议您在分离任务的情况下查看数据库管理帐户。不同的数据库管理员可能有不同的职责，需要不同的权限和角色同样，经验丰富的数据库管理员可能拥有更多角色和权限。不考虑向用户授予用户默认的DBA角色，而是考虑为组织中的特定职位和资历定制数据库管理角色。将命名帐户用于日常活动非常重要。SYS和使用SYSDBA管理权限的帐户等帐户应使用特权帐户管理（PAM）系统进行管理，并在使用时签出（并审核）。您还应该使用PAM系统管理备份Oracle Database Vault所有者和帐户管理帐户。在操作系统中，您应该只通过结帐系统使root和oracle帐户可用，因为这些帐户具有强大的权限。

3.5有效时间支持的运算符

Workspace Manager提供关系检查运算符和设置运算符，这些运算符接受两个时间段参数，可用于在查询中应用有效时间过滤器。

如果两个句点之间的关系存在，则关系检查运算符返回整数值1，如果关系不存在，则返回0。提供以下关系检查操作符以获得有效时间支持：

  • WM_OVERLAPS检查两个周期是否重叠。

  • WM_CONTAINS检查第一个句点是否包含第二个句点。

  • WM_MEETS检查第一个周期的结束是否是第二个周期的开始。

  • WM_EQUALS检查两个句点是否相等（即它们的开始和结束时间是否相同）。

  • WM_LESSTHAN检查第一个周期的结束是否小于（ 即早于）第二个周期的开始。

  • WM_GREATERTHAN检查第一个句点的开头是否大于（即晚于）第二个句点的结尾。

集合运算符返回反映两个周期之间关系的周期，如果两个周期没有指定的关系，则返回空值。为有效时间支持提供以下集合运算符：

  • WM_INTERSECTION返回两个句点的交集，即两个句点共有的时间范围。

  • WM_LDIFF返回左侧两个句点之间的差异（即，更早的时间）。

  • WM_RDIFF返回右侧两个句点之间的差异（即稍后的时间）。

您可以使用关系检查运算符作为使用行的wm_valid.validFrom和wm_valid.validTill属性的替代方法。例如，以下两个查询选择有效的数据，这些查询在1991年1月1日有效：

3.6.1查询

针对具有有效时间支持的启用版本的表发出的所有查询都会考虑当前会话的有效时间设置（使用SetValidTime或SetValidTimeFilterON过程设置）。除非查询另有规定（例如，通过使用在所描述的有效时间支持运营商之一算符有效时支持 ），每个查询显示具有重叠的会话有效时间或有效时间有效的时间范围从基本表中的所有行过滤，并满足查询的其他条件。

默认情况下（即，如果未在会话中调用SetValidTime过程或者在没有参数的情况下调用它），则在当前时间有效的所有行都被视为有效，并且有效时间段被视为来自目前的时间没有限制。

父主题： 具有有效时间支持的查询和DML操作

3.6.2数据操作（DML）操作

针对具有有效时间支持的启用版本的表发出的所有DML语句（INSERT，UPDATE和DELETE）都会考虑当前会话的有效时间设置和更新模式。（更新模式由SetWMValidUpdateModeON和SetWMValidUpdateModeOFF过程控制。）DML语句可以影响对有效时间段有效的所有行。

默认情况下（即，如果在会话中未调用SetValidTime过程或者在没有参数的情况下调用它），则当前时间有效的所有行都可能受DML语句的影响，并且所有已修改的行都有效时间范围时间戳设置为从当前时间到更改。

以下部分描述了适用于特定类型的DML操作的其他注意事项。

  • 插入操作

3.6.2.1更新操作

可以对具有有效时间支持的启用版本的表的更新操作进行排序或不排序。

如果未在UPDATE语句中指定对WM_VALID列的更改，则会发生顺序更新操作。在顺序更新操作中，行的WM_VALID.ValidTill值更改为当前会话有效时间范围的ValidFrom时间戳，并创建一个新行，其中WM_VALID周期反映当前会话有效时间范围。顺序更新确保UPDATE语句不会创建重复记录，因为WM_VALID列值不同。

3.7有效时间支持的约束管理

  • 独特的约束

父主题： 工作区管理器有效时间支持

3.7.1参照完整性约束

如果在具有有效时间支持的两个启用版本的表之间存在参照完整性约束，则在强制执行约束时将考虑行的有效时间段。例如，假设DEPARTMENTS表具有MANAGER_ID列，该列是引用EMPLOYEES表中的EMPLOYEE_ID列的外键（即，部门经理必须是现有员工）。如果两个表都启用了版本且具有有效时间支持，并且插入或更新操作将导致新的DEPARTMENTS.MANAGER_ID值，则如果D EPARTMENTS.WM_VALID值不在EMPLOYEES.WM_VALID范围内，则操作将失败正在担任部门经理的员工的价值。（也就是说，如果新部门经理在指定的时间段内不是有效员工，或者在插入或更新操作中默认，则操作将失败。）

如果参照完整性约束中的任何一个或两个表未启用版本且具有有效时间支持，则在强制执行约束时将忽略有效时间段。

父主题： 有效时间支持的约束管理

3.7.2唯一约束

如果具有有效时间支持的启用版本的表中存在唯一约束，则在强制执行约束时将考虑行的有效时间段。例如，假设EMPLOYEES表具有一个具有唯一约束的EMPLOYEE_ID列。如果插入或更新操作将导致新的EMPLOYEE_ID值与现有EMPLOYEE_ID值相同，则如果现有行和插入行的WM_VALID值重叠，则操作将失败。（也就是说，如果新员工和现有员工具有相同的ID号并且他们的行在任何给定时间都有效，则操作将失败。但是，如果两个员工的有效时间段不重叠，则操作将成功。）

父主题： 有效时间支持的约束管理

3.8锁定有效时间支持

如果锁定了具有有效时间支持的启用版本的表中的行，则会在其整个有效时间段内自动锁定该行。在指定的时间段内无法锁定行。

在悲观锁定的工作空间中的任何更新都将锁定从祖先工作空间看到的行，因为更新是在工作空间中执行的。祖先工作空间中的锁定行在其有效时间段内不会进一步更新，只要它们被锁定即可。

有关Workspace Manager锁定的说明，请参阅使用Workspace Manager锁定管理 。
