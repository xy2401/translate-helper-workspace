根据您的区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。

您必须根据区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。如果应用程序没有自己的特定方法来配置区域设置首选项，则用于配置Oracle数据库客户机连接的方法取决于用于连接到数据库的访问API。在为应用程序配置区域设置首选项之前，请检查应用程序文档。

对于使用Oracle调用接口（OCI）连接到Oracle数据库的应用程序，请使用NLS_LANG和其他名称以NLS_开头的客户端设置来设置Oracle数据库会话的区域设置约定和客户端字符集。正确设置NLS_LANG值的字符集部分非常重要。您设置的字符集必须与I / O设备使用的字符集相对应，在Microsoft Windows的情况下，可以是ANSI代码页（用于GUI应用程序），例如WE8MSWIN1252，或OEM代码页（用于控制台模式）应用程序），例如US8PC437。通过这样做，OCI API将收到有关它从应用程序接收的数据字符集的通知。然后，OCI可以正确地将此数据转换为数据库字符集和从数

NLS_LANG和其他NLS设置可以指定为环境变量或Windows注册表设置。环境变量值优先于Registry值。

在Universal Windows上创建新的Oracle主目录时，Oracle Universal Installer会在注册表中为NLS_LANG设置设置默认值。NLS_LANG值基于Windows用户界面的语言，Windows用户界面是Windows菜单项和对话框标签的语言。

警告：

无法正确设置客户端字符集可能会导致数据丢失。

使用Oracle JDBC连接到Oracle数据库的Java应用程序不使用NLS_LANG。相反，Oracle JDBC将运行应用程序的Java VM的缺省语言环境映射到Oracle数据库语言和区域设置。然后，Oracle JDBC使用这些设置配置连接的数据库会话。由于Java在Unicode内部工作，因此客户端字符集始终设置为Unicode。除非应用程序显式更改它，否则将根据运行Java VM的用户操作系统的区域设置来设置Java VM的缺省语言环境。有关配置Java VM缺省语言环境的信息，请查看Java VM文档。

您可以在交互模式下使用OUI来记录响应文件，然后您可以编辑并使用它们来完成静默模式或响应文件模式安装。此方法对于高级或纯软件安装非常有用。

记录响应文件时，您可以完成安装，也可以在OUI开始将软件设置到系统之前退出“摘要”页面上的安装程序。

如果在响应文件模式安装期间使用记录模式，则安装程序会将原始源响应文件中指定的变量值记录到新响应文件中。

OUI在记录响应文件时不保存密码。

    当您运行安装程序以记录响应文件时，它会检查系统以验证它是否满足安装软件的要求。因此，Oracle建议您在完成安装时完成所有必需的安装前任务并记录响应文件。

  2. 确保Oracle软件所有者用户（通常是oracle ）有权创建或写入运行安装程序时指定的Oracle主路径。

  3. 在每个安装屏幕上，指定所需信息。

  4. 当安装程序显示“摘要”屏幕时，请执行以下步骤：

根据您的区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。

您必须根据区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。如果应用程序没有自己的特定方法来配置区域设置首选项，则用于配置Oracle数据库客户机连接的方法取决于用于连接到数据库的访问API。在为应用程序配置区域设置首选项之前，请检查应用程序文档。

对于使用Oracle调用接口（OCI）连接到Oracle数据库的应用程序，请使用NLS_LANG和其他名称以NLS_开头的客户端设置来设置Oracle数据库会话的区域设置约定和客户端字符集。正确设置NLS_LANG值的字符集部分非常重要。您设置的字符集必须与I / O设备使用的字符集相对应，在Microsoft Windows的情况下，可以是ANSI代码页（用于GUI应用程序），例如WE8MSWIN1252，或OEM代码页（用于控制台模式）应用程序），例如US8PC437。通过这样做，OCI API将收到有关它从应用程序接收的数据字符集的通知。然后，OCI可以正确地将此数据转换为数据库字符集和从数

NLS_LANG和其他NLS设置可以指定为环境变量或Windows注册表设置。环境变量值优先于Registry值。

在Universal Windows上创建新的Oracle主目录时，Oracle Universal Installer会在注册表中为NLS_LANG设置设置默认值。NLS_LANG值基于Windows用户界面的语言，Windows用户界面是Windows菜单项和对话框标签的语言。

警告：

无法正确设置客户端字符集可能会导致数据丢失。

使用Oracle JDBC连接到Oracle数据库的Java应用程序不使用NLS_LANG。相反，Oracle JDBC将运行应用程序的Java VM的缺省语言环境映射到Oracle数据库语言和区域设置。然后，Oracle JDBC使用这些设置配置连接的数据库会话。由于Java在Unicode内部工作，因此客户端字符集始终设置为Unicode。除非应用程序显式更改它，否则将根据运行Java VM的用户操作系统的区域设置来设置Java VM的缺省语言环境。有关配置Java VM缺省语言环境的信息，请查看Java VM文档。

注意：

Oracle ACFS的限制和指南

将这些主题作为存储计划的一部分进行查看，以便将Oracle ACFS用于单实例和群集配置。

  • Oracle自动存储管理群集文件系统（Oracle ACFS）提供通用文件系统。

  • 配置Oracle ASM时，只能使用Oracle ACFS。

  • 请注意以下有关在Oracle ACFS上放置Oracle数据库和Oracle Grid Infrastructure文件的一般准则和限制：

    • 您可以在Oracle ACFS上放置Oracle数据库二进制文件，数据文件和管理文件（例如，跟踪文件）。

    • Oracle ACFS不支持使用Oracle数据库数据文件，表空间文件，控制文件和重做日志进行加密。

    • 仅当数据库版本为Oracle Database 11g第2版或更新版本时，才能将Oracle数据库主目录置于Oracle ACFS上。您无法在Oracle ACFS上安装早期版本的Oracle数据库。

    • 对于Oracle Clusterware上的安装，您无法将Oracle Clusterware文件放在Oracle ACFS上。

  • Oracle Restart不支持基于root的Oracle Clusterware资源。因此，如果在Oracle重新启动配置上运行Oracle ACFS，则以下限制适用：

    • 从Oracle Database 18c开始，配置助手不允许在Oracle Restart配置中在Oracle ACFS上创建Oracle数据库主目录。

    • Oracle Restart不支持所有平台上的Oracle ACFS资源。

    • 从Oracle Database 12c开始，Oracle Restart配置不支持Oracle ACFS注册表。

    • 在Linux上，Oracle ACFS提供了一种自动机制，用于在系统重新启动和关闭时加载和卸载驱动程序以及装载和卸载Oracle ACFS文件系统。但是，Oracle ACFS在系统运行时不提供已装入文件系统的自动恢复。除Linux之外，Oracle ACFS不在其他操作系统上提供此自动机制。

    • Oracle Restart配置不支持在Oracle ACFS文件系统上创建Oracle数据文件。Oracle Grid Infrastructure支持在Oracle ACFS文件系统上创建Oracle数据文件以进行群集配置。

  • IBM AIX Workload Partitions（WPAR）不支持Oracle ACFS和Oracle ADVM。

使用DBCA删除Oracle RAC数据库

使用DBCA删除Oracle RAC数据库涉及首先删除数据库，然后删除数据库的初始化参数文件，实例，最佳灵活体系结构（OFA）结构以及数据库的Oracle网络配置。

要使用DBCA删除数据库：

  1. 在其中一个节点上启动DBCA：

    • 从$ORACLE_HOME/bin目录运行dbca命令。

    DBCA显示“操作”页面，显示不同的数据库部署选项。

  2. 选择删除数据库 ，然后单击下一步 。DBCA显示从运行DBCA的Oracle主目录运行的所有Oracle RAC和单实例数据库的列表。
  3. 如果您的用户标识和密码未经过操作系统身份验证，则“列表数据库列表”页面将显示用户名和密码字段。如果出现这些字段，则输入具有SYSDBA权限的用户帐户的用户标识和密码。
  4. 选择要删除的数据库，然后单击“ 完成” 。

    单击“ 完成”后 ，DBCA将显示一个对话框，以确认已将DBCA配置为要删除的数据库和实例。

  5. 单击“ 确定”开始删除数据库及其关联的文件，服务和环境设置，或单击“ 取消”停止操作。

单击“ 确定”后 ，DBCA将继续该操作并删除此数据库的所有关联实例。DBCA还会删除参数文件，密码文件和oratab条目。

此时，您已完成以下操作：

  • 从群集中删除了选定的Oracle RAC数据库

  • 删除分配给Oracle RAC数据库的高可用性服务

  • 删除了Oracle RAC数据库的Oracle Net配置

Oracle Grid Infrastructure和Oracle RAC的存储注意事项

对于所有安装，您必须选择用于Oracle Grid Infrastructure（Oracle Clusterware和Oracle ASM）和Oracle Real Application Clusters（Oracle RAC）数据库的存储选项。

Oracle Clusterware的存储注意事项

Oracle Clusterware投票文件用于监控群集节点状态，Oracle Cluster Registry（OCR）文件包含有关群集的配置信息。您可以在Oracle ASM磁盘组或共享文件系统上存储Oracle Cluster Registry（OCR）和表决文件。您还可以将OCR文件的备份存储在磁盘组中。存储必须共享;任何无法访问绝对多数表决文件（超过一半）的节点都会重新启动。

如果使用在网络文件系统（NFS）上创建的Oracle ASM磁盘组进行存储，请确保遵循“在NFS上配置Oracle ASM磁盘组的准则 ”主题中所述的挂载NFS建议。

Oracle RAC的存储注意事项

Oracle ASM是数据库和恢复文件的受支持存储选项。对于所有安装，Oracle建议您至少创建两个单独的Oracle ASM磁盘组：一个用于Oracle数据库数据文件，另一个用于恢复文件。Oracle建议您将Oracle数据库磁盘组和恢复文件磁盘组放在不同的故障组中。

  • 如果不将Oracle ASM用于数据库文件，则Oracle建议您将数据文件和快速恢复区放置在位于Oracle主目录之外的共享存储中的不同位置，以免硬件故障影响可用性。

  • 如果满足所选存储选项的所有要求，则可以为每种文件类型选择受支持的存储选项的任意组合。

  • 如果计划在共享OCFS2位置安装Oracle RAC主目录，则必须将OCFS2升级到至少1.4.1版，该版本支持共享可写内存映射。

  • 要将Oracle ASM与Oracle RAC一起使用，并且要配置新的Oracle ASM实例，则系统必须满足以下条件：

    • 群集上的所有节点都安装了Oracle Clusterware和Oracle ASM 19c，作为群集安装的Oracle Grid Infrastructure的一部分。

7.2 UCP中基于JMX的管理概述

JMX（Java Management Extensions）是一种Java技术，提供用于管理和监视应用程序，系统对象，设备，面向服务的网络和JVM（Java虚拟机）的工具。在JMX中，给定资源由一个或多个称为MBean（Managed Beans）的Java对象进行检测。MBean由MBean接口和类组成。MBean接口列出了所有公开属性和操作的方法。该类实现此接口并提供已检测资源的功能。

MBean在核心托管对象服务器（称为MBean服务器）中注册，该服务器充当管理代理程序，可以在为Java编程语言启用的大多数设备上运行。JMX代理由MBean服务器和一组用于处理MBean的服务组成，MBean服务器在其中注册MBean。

Oracle RAC的存储注意事项

Oracle ASM是数据库和恢复文件的受支持存储选项。对于所有安装，Oracle建议您至少创建两个单独的Oracle ASM磁盘组：一个用于Oracle数据库数据文件，另一个用于恢复文件。Oracle建议您将Oracle数据库磁盘组和恢复文件磁盘组放在不同的故障组中。

1.2.7 Oracle数据库服务质量（QoS）管理简介

Oracle数据库服务质量（QoS）管理管理跨应用程序共享的资源。

Oracle数据库服务质量（QoS）管理：

  • 需要12.1.0.2+ Oracle Grid Infrastructure

  • 提供关键绩效指标集群范围的仪表板

  • 使用Measure，Monitor，Management Modes逐步进入

Oracle数据库服务质量（QoS）管理调整系统配置，以使应用程序以您的业务所需的性能级别运行。

许多公司正在整合和标准化他们的数据中心计算机系统。这些公司在集群数据库上运行多个应用程序，而不是为每个应用程序使用单独的服此外，将应用程序迁移到Internet还引入了管理开放工作负载的问题。开放式工作负载受到需求激增的影响，这可能导致系统过载。过载系统会导致无法完全预期或计划的新型应用程序故障。要在这种类型的环境中保持应用程序可用并在其目标服务级别内执行，您必须：

  • 池资源

  • 拥有可实时检测性能瓶颈的管理工具

  • 重新分配资源以满足需求变化

Oracle数据库QoS管理可以优雅地响应系统配置和需求的变化，从而避免应用程序性能级别的更多振荡。

Oracle数据库QoS管理监视目标系统上每个工作请求的性能。Oracle数据库QoS管理从工作请求尝试使用数据库服务建立与数据库的连接时开始跟踪工作请求。完成工作请求或响应时间所需的时间是从启动数据请求到完成数据请求的时间。响应时间也称为端到端响应时间或往返时间。通过准确测量响应时间的两个组成部分，Oracle数据库QoS管理可以快速检测系统中的瓶颈。然后，Oracle数据库QoS管理建议重新分配资源以缓解瓶颈 ，从而保留或恢复服务级别。

Oracle数据库QoS管理管理系统上的资源，以便：

  • 当有足够的资源来满足需求时，即使工作负载发生变化，也可以满足应用程序的业务级性能要求

  • 当没有足够的资源来满足需求时，Oracle数据库QoS管理会尝试以不太关键的性能要求为代价来满足更关键的业务性能要求

根据环境，当您使用SCAN使用服务名称连接到Oracle RAC数据库时，会发生以下操作。

  2. 客户端使用以下形式的连接描述符发出数据库连接请求：

    orausr / @ scan_name ：1521 / webapp

    注意：

  4. 当连接请求成功时，客户端连接到承载sales数据库的集群的SCAN侦听器，并且具有提供webapp服务的实例，在此示例中为sales1和sales2 。SCAN侦听器比较实例sales1和sales2的工作负载以及它们运行的节点的工作负载。如果SCAN侦听器确定node2的负载小于node1 ，则SCAN侦听器选择node2并将该节点上的本地侦听器的地址发送回客户端。

  5. 客户端连接到node2上的本地侦听器。本地侦听器启动与数据库连接的专用服务器进程。

  6. 客户端直接连接到node2上的专用服务器进程并访问sales2数据库实例。

  • TimeToLiveConnectionTimeoutCallback接口

6.1 AbandonedConnectionTimeoutCallback接口

AbandonedConnectionTimeoutCallback回调接口用于废弃的连接超时功能。此功能使应用程序能够提供对废弃连接的自定义处理。回调对象使用其逻辑连接代理之一，或者在每个池连接中注册。这使得应用程序可以在池认为放弃特定连接时执行自定义处理。当通用连接池认为放弃借用连接时，将调用handleTimedOutConnection方法。应用程序可以在连接上执行以下操作之一：

  • 完全覆盖池处理过程

  • 调用其他处理操作

  • 假设默认池处理

JDBC应用程序可以在handleTimedOutConnection方法中的废弃连接上调用cancel ， close和rollback方法。

注意：

如果您尝试在同一连接上注册多个AbandonedConnectionTimeoutCallback接口，则会导致异常。此异常可以是池层或java.sql.的UniversalConnectionPoolException java.sql.SQLException ，特定于UCP Adapter的类型，如JDBC，JCA等。

6.2 TimeToLiveConnectionTimeoutCallback接口

TimeToLiveConnectionTimeoutCallback回调接口，用于生存时间（TTL）连接超时功能。这使应用程序能够为TTL超时连接提供自定义处理。

回调对象使用其逻辑连接代理之一，或者在每个池连接中注册。这使得应用程序能够在特定连接的TTL超时时执行自定义处理。

当通用连接池发现借用的连接被TTL超时时，将调用handleTimedOutConnection方法。应用程序可以在连接上执行以下操作之一：

  • 完全覆盖池处理过程

  • 调用其他处理操作

  • 假设默认池处理

JDBC应用程序可以在handleTimedOutConnection方法中的废弃连接上调用cancel ， close和rollback方法。

注意：

如果您尝试在同一连接上注册多个TimeToLiveConnectionTimeoutCallback接口，则会导致异常。此异常可以是池层或java.sql.的UniversalConnectionPoolException java.sql.SQLException ，特定于UCP Adapter的类型，如JDBC，JCA等。

9.9物化视图对对象的支持

物化视图支持可用于包含对象，集合或REF类型列的关系表。这种物化视图称为对象关系物化视图。

对象关系实体化视图所需的所有用户定义类型必须存在于物化视图站点以及主站点。它们必须在两个站点上具有相同的对象类型ID和版本。

话题：

  • 对象，集合或REF类型列

  • 对象表

9.9.1对象，集合或REF类型列

要进行更新，基于包含对象列的表的实例化视图必须选择列作为定义视图的查询中的对象：如果查询仅选择列对象类型的某些属性，则读取实例化视图-只要。

视图定义查询还可以选择集合或REF类型的列。REF可以是基于主键的，也可以是系统生成的键，它们可以是作用域的，也可以是未作用域的。可以将实体化REF列重新调整到物化视图的站点上的不同表 - 例如，到主表的本地物化视图而不是原始远程表。

9.9.2对象表

基于对象表的物化视图称为对象物化观点。这种物化视图本身就是一个对象表。通过将OF type关键字添加到CREATE MATERIALIZED VIEW语句来创建对象实例化视图。例如：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT * FROM HR.Customer_objtab@dbs1;

与普通对象表一样，对象物化视图的每一行都是一个对象实例，因此创建物化视图的视图定义查询必须从主表中选择整个对象：查询不能仅选择对象类型的子集属性。例如，不允许使用以下物化视图：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT CustNo FROM HR.Customer_objtab@dbs1;

您可以通过省略OF type关键字从对象表创建对象关系的物化视图，但这样的视图是只读的：您无法从对象表创建可更新的对象关系物化视图。

例如，以下CREATE MATERIALIZED VIEW语句创建对象表的只读对象 - 关系实例化视图。即使视图定义查询选择了对象类型的所有列和属性，它也不会将它们选为对象的属性，因此创建的视图是对象关系和只读的：

CREATE MATERIALIZED VIEW客户AS SELECT * FROM HR.Customer_objtab@dbs1;

对于基于对象表的对象关系视图和对象实例化视图，如果主对象表的类型不是FINAL ，则实例化视图定义查询中的FROM子句必须包含ONLY关键字。例如：

CREATE MATERIALIZED VIEW customer OF cust_objtyp AS SELECT CustNo FROM ONLY HR.Customer_objtab@dbs1;

否则， FROM子句必须省略ONLY关键字。

在开始安装之前，每个节点上必须至少配置两个接口：一个用于专用IP地址，另一个用于公用IP地址。

在安装过程中，系统会要求您确定Oracle Universal Installer（OUI）在群集节点上检测到的每个网络接口的计划用途。将每个接口标识为公共或私有接口，或者作为您不希望Oracle Grid Infrastructure或Oracle ASM使用的接口。公共和虚拟互联网协议（VIP）地址在公共接口上配置。私有地址在私有接口上配置。

使用以下选项之一配置IP地址：

使用多集群或标准Oracle网格命名服务（GNS）分配动态IP地址

如果选择此选项，则网络管理员将委派GNS解析的子域（标准或多集群）。GNS要求根据您配置区域委派的方式而有所不同。如果您使用区域委派配置GNS，则GNS将被委派为其解析服务请求的域。如果您在没有区域委派的情况下配置GNS，则GNS具有由DNS解析的虚拟IP地址。

对于带区域委派的GNS：

  • 对于IPv4，群集使用的公共网络上运行的DHCP服务

  • 对于IPv6，群集使用的公共网络上运行的自动配置服务

  • DHCP服务器上有足够的地址为每个节点提供一个IP地址，并为群集的单客户端访问名称（SCAN）使用的群集提供三个IP地址

使用现有的GNS配置

从Oracle Grid Infrastructure 12c第1版（12.1）开始，多个集群可以使用单个GNS实例。要将GNS用于多个群集，DNS管理员必须委派一个区域供GNS使用。此外，必须在网络上的某处启动GNS实例，并且GNS实例必须是可访问的（不会被防火墙阻止）。向GNS实例注册的所有节点名称必须是唯一的。

使用DNS或主机文件解析分配静态IP地址

如果选择此选项，则网络管理员会为群集中的每个物理主机名分配固定IP地址，并为Oracle Clusterware管理的VIP分配IP。此外，每个节点都使用基于域名服务器（DNS）的静态名称解析，或者必须更新群集和客户端的主机文件，从而导致SCAN功能受限。选择此选项要求您在修改群集时请求网络管理更新。

对于没有区域委派的GNS，请为群集配置GNS虚拟IP地址（VIP）。要启用Oracle Flex Cluster，您必须至少配置GNS虚拟IP地址。

表达式列表可以出现在比较和成员条件以及查询和子查询的GROUP BY子句中。比较或成员条件中的表达式列表有时称为行值构造函数或行构造函数 。

比较和成员条件出现在WHERE子句的条件中。它们可以包含一个或多个逗号分隔的表达式或一组或多组表达式，其中每个集合包含一个或多个逗号分隔的表达式。在后一种情况下（多组表达式）：

  • 每组都用括号括起来

  • 每个集合必须包含相同数量的表达式

  • 每个集合中的表达式数量必须与比较条件中运算符之前或成员条件中IN关键字之前的表达式数量相匹配。

逗号分隔的表达式列表可以包含不超过1000个表达式。逗号分隔的表达式列表可以包含任意数量的集合，但每个集合可以包含不超过1000个表达式。

错误29391849

升级到Oracle Grid 18.x或19.x完成后，后续升级到Oracle Grid 18.y或19.y会导致Oracle自动存储管理群集文件系统（Oracle ACFS）在第一个节点上脱机正在升级。在全新安装的Oracle Grid 19c或18c上不会发生此问题。

解决方法：

  • 要避免此问题：
    1. 完成从Oracle Grid 12c第1版（12.1.0.2）或Oracle Grid 12c第2版（12.2）到Oracle Grid 18.x或19.x的升级。
    2. 在每个节点上重新启动Cluster Ready Services（CRS），最后重新启动主节点。例如：
      1. 运行acfsutil clusterinfo | grep -i master 。
      2. 以任何顺序在每个非主节点上重新启动CRS。
      3. 最后在主节点上重启CRS。
    3. 然后，从Oracle Grid 18.x或19.x升级到Oracle Grid 18.y或19.y.
  • 如果您已完成从Oracle Grid 12.1.0.2或12.2到Oracle Grid 18.x或19.x的升级，并已开始升级到Oracle Grid 18.y或19.y，并且您看到了Oracle自动存储管理动态卷管理器（Oracle ADVM）卷在已升级的节点上脱机，然后：
    1. 将该节点回滚到Oracle Grid 18.x或19.x.
    2. 在每个节点上重新启动CRS，最后重新启动主节点。
      1. 运行acfsutil clusterinfo | grep -i master 。
      2. 以任何顺序在每个非主节点上重新启动CRS。
      3. 最后在主节点上重启CRS。
    3. 然后，从Oracle Grid 18.x或19.x升级到Oracle Grid 18.y或19.y.

配置开放IPMI驱动程序

在Linux系统上，OpenIPMI驱动程序是支持IPMI的Oracle Clusterware部署的驱动程序。您可以通过手动加载所需模块来动态安装和配置驱动程序。

有关如何为分发配置IPMI的信息，请与Linux分发供应商联系。在Oracle Linux上手动配置Open IPMI驱动程序，如下所示：
  1. 以root身份登录。
  2. 运行以下命令：
    ＃/ sbin / modprobe ipmi_msghandler＃/ sbin / modprobe ipmi_si＃/ sbin / modprobe ipmi_devintf
  3. （可选）运行命令/sbin/lsmod |grep ipmi以确认已加载IPMI模块。例如：
    ＃/ sbin / lsmod | grep ipmi ipmi_devintf 12617 0 ipmi_si 33377 0 ipmi_msghandler 33701 2 ipmi_devintf，ipmi_si

    无论BMC是否存在，您都可以安装模块。

  4. 使用文本编辑器打开/etc/rc.local文件，导航到文件末尾，然后输入类似于以下内容的行，以便在系统重新启动时自动运行步骤2中的modprobe命令：
    #START IPMI ON SYSTEM RESTART / sbin / modprobe ipmi_msghandler / sbin / modprobe ipmi_si / sbin / modprobe ipmi_devintf

    在SUSE Linux Enterprise Server系统上，将上面的modprobe命令添加到/etc/init.d/boot.local 。

  5. 使用以下命令检查以确保Linux系统正在识别IPMI设备：
    ls -l / dev / ipmi0
    如果动态加载IPMI设备，则输出必须类似于以下内容：
    #ls -l / dev / ipmi0 crw ------- 1 root root 253,0 Sep 23 06:29 / dev / ipmi0

    如果确实看到了设备文件输出，则配置了IPMI驱动程序，您可以忽略以下步骤。

    如果您没有看到设备文件输出，则udevd守护程序未设置为自动创建设备文件。继续下一步。

  6. 使用命令grep ipmi /proc/devices确定IPMI设备的设备主grep ipmi /proc/devices 。例如：
    ＃grep ipmi / proc / devices 253 ipmidev
    设备主要编号为253。
  7. 运行mknod命令，使用设备主设备号为IPMI设备创建目录条目和i-node 。例如：
    #mknod / dev / ipmi0 c 253 0x0
    示例中的/dev/ipmi0权限允许设备只能由root访问。只应通过root访问设备，以防止系统漏洞。
