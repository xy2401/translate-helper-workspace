目的

TO_CHAR （number）使用可选的数字格式fmt将n转换为VARCHAR2数据类型的值。值n可以是NUMBER ， BINARY_FLOAT或BINARY_DOUBLE BINARY_FLOAT 。如果省略fmt ，则n将转换为VARCHAR2值，其长度足以保持其有效数字。

如果n为负数，则在应用格式后应用符号。因此TO_CHAR(-1, '$9')返回 - $ 1，而不是$ -1。

有关数字格式的信息，请参阅“ 格式模型 ” 。

'nlsparam'参数指定由数字格式元素返回的这些字符：

  • 十进制字符

  • 组分隔符

  • 当地货币符号

  • 国际货币符号

这个论点可以有这样的形式：

'NLS_NUMERIC_CHARACTERS = '' 危险品 'NLS_CURRENCY = '' 文本 'NLS_ISO_CURRENCY = 领土 '

字符d和g代表十进制字符和组分隔符。它们必须是不同的单字节字符。在引用的字符串中，必须在参数值周围使用两个单引号。货币符号有十个字符。

如果省略'nlsparam'或任何一个参数，则此函数使用会话的默认参数值。

任务1：选择一组自包含的表空间

可传输集中的数据库对象与可传输集外部的数据库对象之间可能存在逻辑或物理依赖关系。您只能传输自包含的表空间集，也就是说，表空间集内的任何数据库对象都不依赖于该表空间集之外的任何数据库对象。

自包含表空间违规的一些示例是：

  • 表空间集内的索引用于表空间集之外的表。

    注意：

    如果表的相应索引在表空间集之外，则不违反。

  • 分区表部分包含在表空间集中。

    要复制的表空间集必须包含分区表的所有分区，或者不包含分区表的任何分区。要传输分区表的子集，必须将分区交换为表。

    有关交换分区的信息，请参见“ Oracle数据库VLDB和分区指南” 。

  • 引用完整性约束指向跨集合边界的表。

    传输一组表空间时，可以选择包含参照完整性约束。但是，这样做会影响一组表空间是否是自包含的。如果您决定不传输约束，则不将约束视为指针。

  • 表空间集内的表包含一个LOB列，指向表空间集外的LOB 。

  • 由用户A注册的XML DB模式（* .xsd）导入由用户B注册的全局模式，并且以下情况属实：用户A的默认表空间是表空间A，用户B的默认表空间是表空间B，只有表空间A包含在表空间集中。

要确定一组表空间是否为自包含，请在Oracle提供的包DBMS_TTS运行TRANSPORT_SET_CHECK过程。您必须已被授予EXECUTE_CATALOG_ROLE角色（最初签名到SYS ）才能运行此过程。

运行DBMS_TTS.TRANSPORT_SET_CHECK过程时，请指定要检查自包含的可传输集中的表空间列表。您可以选择指定是否必须包含约束。对于严格或完全包含，您必须另外将TTS_FULL_CHECK参数设置为TRUE 。

严格或完整的包含检查适用于不仅要捕获可移动集外部的引用，还要捕获进入集合的引用的情况。表空间时间点恢复（TSPITR）就是这样一种情况，其中依赖对象必须完全包含或完全位于可传输集之外。

例如，对包含表t但不包含其索引i的表空间执行TSPITR是违规的，因为索引和数据在传输后将不一致。完整的包含检查可确保外部没有依赖关系或进入可传输集。请参阅“ Oracle数据库备份和恢复用户指南”中的 TSPITR示例。

注意：

可传输表空间的默认设置是检查自包含而不是完全包含。

以下语句可用于确定表空间sales_1和sales_2是否为自包含，并考虑了引用完整性约束（由TRUE表示）。 

EXECUTE DBMS_TTS.TRANSPORT_SET_CHECK（'sales_1，sales_2'，TRUE）;

任务1：选择一组自包含的表空间

可传输集中的数据库对象与可传输集外部的数据库对象之间可能存在逻辑或物理依赖关系。您只能传输自包含的表空间集，也就是说，表空间集内的任何数据库对象都不依赖于该表空间集之外的任何数据库对象。

自包含表空间违规的一些示例是：

  • 表空间集内的索引用于表空间集之外的表。

    注意：

    如果表的相应索引在表空间集之外，则不违反。

  • 分区表部分包含在表空间集中。

    要复制的表空间集必须包含分区表的所有分区，或者不包含分区表的任何分区。要传输分区表的子集，必须将分区交换为表。

    有关交换分区的信息，请参见“ Oracle数据库VLDB和分区指南” 。

  • 引用完整性约束指向跨集合边界的表。

    传输一组表空间时，可以选择包含参照完整性约束。但是，这样做会影响一组表空间是否是自包含的。如果您决定不传输约束，则不将约束视为指针。

  • 表空间集内的表包含一个LOB列，指向表空间集外的LOB 。

  • 由用户A注册的XML DB模式（* .xsd）导入由用户B注册的全局模式，并且以下情况属实：用户A的默认表空间是表空间A，用户B的默认表空间是表空间B，只有表空间A包含在表空间集中。

要确定一组表空间是否为自包含，请在Oracle提供的包DBMS_TTS运行TRANSPORT_SET_CHECK过程。您必须已被授予EXECUTE_CATALOG_ROLE角色（最初签名到SYS ）才能运行此过程。

运行DBMS_TTS.TRANSPORT_SET_CHECK过程时，请指定要检查自包含的可传输集中的表空间列表。您可以选择指定是否必须包含约束。对于严格或完全包含，您必须另外将TTS_FULL_CHECK参数设置为TRUE 。

严格或完整的包含检查适用于不仅要捕获可移动集外部的引用，还要捕获进入集合的引用的情况。表空间时间点恢复（TSPITR）就是这样一种情况，其中依赖对象必须完全包含或完全位于可传输集之外。

例如，对包含表t但不包含其索引i的表空间执行TSPITR是违规的，因为索引和数据在传输后将不一致。完整的包含检查可确保外部没有依赖关系或进入可传输集。请参阅“ Oracle数据库备份和恢复用户指南”中的 TSPITR示例。

注意：

可传输表空间的默认设置是检查自包含而不是完全包含。

以下语句可用于确定表空间sales_1和sales_2是否为自包含，并考虑了引用完整性约束（由TRUE表示）。 

EXECUTE DBMS_TTS.TRANSPORT_SET_CHECK（'sales_1，sales_2'，TRUE）;

创建模式

目的

使用CREATE SCHEMA语句创建多个表和视图，并在单个事务中在您自己的模式中执行多个授权。

要执行CREATE SCHEMA语句，Oracle数据库将执行每个包含的语句。如果所有语句都成功执行，则数据库将提交事务。如果任何语句导致错误，则数据库将回滚所有语句。

注意：

该语句实际上并不创建模式。Oracle Database在您创建用户时自动创建架构（请参阅CREATE USER ）。此语句允许您使用表和视图填充模式，并为这些对象授予权限，而无需在多个事务中发出多个SQL语句。

13.54返回声明

RETURN语句立即结束子程序或包含它的匿名块的执行。

在函数中， RETURN语句为函数标识符指定一个指定的值，并将控制权返回给调用者，其中执行在调用之后立即恢复（可能在调用语句内）。函数中的每个执行路径都必须导致RETURN语句（否则，PL / SQL编译器会发出编译时警告PLW-05005）。

在一个过程中， RETURN语句将控制返回给调用者，其中执行在调用之后立即恢复。

在匿名块中， RETURN语句退出其自己的块和所有封闭块。

子程序或匿名块可以包含多个RETURN语句。

注意：

RETURN语句与函数标题中的RETURN子句不同，后者指定返回值的数据类型。

完成以下步骤以为独立服务器安装Oracle Grid Infrastructure，然后创建由Oracle Restart管理的数据库。

为独立服务器安装Oracle Grid Infrastructure，该服务器安装Oracle Restart和Oracle ASM，并创建一个磁盘组。

在开始安装之前，您应该可以使用网络信息，存储信息以及操作系统用户和组。您还应该准备好运行根脚本或提供自动化根脚本的信息。

  1. 以Oracle Restart软件所有者用户（ oracle ）身份登录。
  2. 下载Oracle Grid Infrastructure以获取独立服务器安装映像文件，创建网格主目录，并在此网格主目录中提取映像文件。

    例如：

    $ mkdir -p /u01/app/oracle/product/19.0.0/grid $ chgrp oinstall /u01/app/oracle/product/19.0.0/grid $ cd /u01/app/oracle/product/19.0.0/ grid $ unzip -q /tmp/grid_home.zip

    注意：

    确保您创建的Grid主目录路径符合Oracle Optimal Flexible Architecture建议。此外，仅在您创建的此Grid主目录中解压缩安装映像文件。
  3. 配置和配置共享磁盘以与Oracle ASM筛选器驱动程序（Oracle ASMFD）一起使用。
    1. 以root身份登录并将$ ORACLE_HOME设置为网格主页位置，并将$ ORACLE_BASE设置为临时位置。
      su root #set ORACLE_HOME = / u01 / app / oracle / product / 19.0.0 / grid #set ORACLE_BASE = / tmp

      您将$ ORACLE_BASE设置为临时位置，以避免在安装Oracle Grid Infrastructure之前在Grid主目录中创建诊断或跟踪文件。

      #cd /u01/app/oracle/product/19.0.0/grid/bin#./asmcmd afd_label DATA1 / dev / rdsk / cXtYdZsA --init＃。/ asmcmd afd_label DATA2 / dev / rdsk / cXtYdZsB --init＃。 / asmcmd afd_label DATA3 / dev / rdsk / cXtYdZsC --init

    • 默认磁盘组名称为DATA。您可以为磁盘组输入新名称，或使用默认名称。

句法


说明instr.eps的说明

目的

INSTR函数搜索string以查找substring 。搜索操作被定义为将substring参数与相同长度的string的子string进行比较以获得相等性，直到找到匹配或者不再有子字符串为止。字符串的每个连续比较子string从前一个比较子字符串的第一个字符开始向右一个字符（用于向前搜索）或向左开始一个字符（用于向后搜索）。如果找到等于substring ，则该函数返回一个整数，指示该子字符串的第一个字符的位置。如果没有找到这样的子字符串，则该函数返回零。

  • position是一个非零整数，指示的字符string ，其中的Oracle数据库开始搜索，也就是说，所述第一字符串的第一个字符的位置到与比较substring 。如果position为负数，则Oracle从string末尾开始向后计数，然后从结果位置向后搜索。

  • occurrence是一个整数，表示Oracle应该搜索的substring中出现的substring string 。值occurrence必须为正。如果occurrence大于1，则该数据库不返回第一匹配但继续进行比较的连续子string ，如上所述，直到匹配号码occurrence已经找到。

INSTR接受并返回由输入字符集定义的字符位置，字符串的第一个字符具有位置1。INSTRB使用字节而不是字符。INSTRC使用Unicode完整字符。INSTR2使用UCS2代码点。INSTR4使用UCS4代码点。

string可以是CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB任何数据类型。 INSTRC ， INSTR2和INSTR4是例外，它们不允许string为CLOB或NCLOB 。

substring可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB 。

返回的值为NUMBER数据类型。

position和occurrence必须是数据类型NUMBER ，或者可以隐式转换为NUMBER任何数据类型，并且必须解析为整数。position和occurrence的默认值都是1，这意味着Oracle开始在string的第一个字符处搜索第一次出现的substring 。无论position的值如何，返回值都相对于string的开头。

      在上面的示例中，有20个可能的插槽，第一个未使用的插槽是4。

    3. 分配所需的管理员用户名和密码，并为标识的插槽启用消息传递。（请注意，对于IPMI v1.5，用户名和密码最多为16个字符）。此外，在通过LAN（通道1）访问ADMIN（级别4）时，设置该插槽的权限级别。例如， username是管理用户名， password是密码：

  6. 使用bmc info命令验证BMC是否可从群集中的远程节点访问和控制。例如，如果node2-ipmi是分配了node2的BMC的IP地址的网络主机名，那么要使用管理员帐户username在node1上验证节点node2上的BMC，请在node1输入以下命令：

    $ ipmitool -H node2-ipmi -U username lan print 1

    系统会提示您输入密码。提供IPMI密码。

  7. 对每个集群成员节点重复此过程。如果每个群集成员节点上的IPMI管理员帐户凭据不相同，则IPMI将在配置期间失败。
