8.8定位器如何提高嵌套表的性能

您可以使用嵌套表定位器来提高检索数据时的性能。

集合类型不直接映射到C ++和Java等语言中的本机类型或结构。使用这些语言的应用程序必须通过Oracle数据库接口（如OCI）访问集合的内容。

通常，当客户端显式或隐式访问嵌套表（通过获取包含对象）时，数据库会将整个集合值返回给客户端进程。出于性能原因，客户可能希望延迟或避免检索集合的全部内容。Oracle数据库通过使用定位器而不是实际的嵌套表值来处理这种情况。当您真正访问集合的内容时，它们会自动转移到客户端。

嵌套表定位器就像集合值的句柄。它通过在检索时包含数据库快照来尝试保留嵌套表的值或复制语义。快照有助于数据库在稍后使用定位器获取集合元素时检索嵌套表值的正确实例化。定位器的范围限定为会话，不能跨会话使用。由于使用了数据库快照，因此如果嵌套表上的更新率较高，则可能会使snapshot too old错误。与LOB定位器不同，嵌套表定位器确实是一个定位器，不能用于修改集合实例。

关于共享SCAN

共享SCAN提供跨多个群集共享SCAN VIP的功能，从而减少部署Oracle群集时必须分配的IP地址数量。

在早期版本的Oracle Clusterware中，SCAN VIP基于每个群集进行配置。使用共享SCAN，在多个集群中使用相同的SCAN，因此这些集群中只有一个运行SCAN侦听器。所有群集的数据库都使用此群集的SCAN VIP来进行所有数据库连接。每个群集都有自己的一组端口，而不是SCAN VIP。使用共享SCAN的群集可以根据需要命名其数据库服务，如果这些群集中的一个或多个配置了相同名称的服务，则不会发生命名冲突。节点VIP使用主机IP地址。

关于共享SCAN

共享SCAN提供跨多个群集共享SCAN VIP的功能，从而减少部署Oracle群集时必须分配的IP地址数量。

11.1 Memory Guard概述

Memory Guard自动监视群集节点，以防止由于内存不足而导致的节点压力。

Memory Guard自动收集Oracle Real Application Clusters（Oracle RAC）环境中每个节点的内存使用情况指标。Memory Guard从Cluster Health Monitor获取信息。如果Memory Guard确定节点内存不足，则Memory Guard将执行以下操作：

  • 阻止在受影响的节点上创建新的数据库会话

  • 在节点上以事务方式停止所有CRS管理的服务，允许节点上的现有工作负载完成并释放其内存

当Memory Guard确定已释放内存压力时，它会恢复与节点的连接，从而允许在该节点上创建新会话。

内存不足可能导致事务失败，或者在极端情况下，节点重新启动会导致应用程序的可用性和资源丢失。

5.1 UCP中标签连接概述

应用程序通常在使用连接之前初始化从连接池检索的连接。初始化各不相同，可能包括简单的状态重新初始化，需要在应用程序代码中进行方法调用或需要通过网络进行往返的数据库操作。这种初始化的成本可能很高。

标记连接使应用程序可以将任意名称/值对附加到连接。应用程序可以从连接池请求与所需标签的连接。通过将特定标签与特定连接状态相关联，应用程序可以从池中检索已初始化的连接，并避免重新初始化的时间和成本。连接标记功能对用户定义的键或值没有任何意义;用户定义的键和值的含义仅由应用程序定义。

连接标记的一些示例包括：角色，NLS语言设置，事务隔离级别，存储过程调用，或者在资源执行工作之前，连接上昂贵且必需的任何其他状态初始化。

连接标签是应用程序驱动的，需要使用两个接口。oracle.ucp.jdbc.LabelableConnection接口用于应用和删除连接标签，以及检索已在连接上设置的标签。oracle.ucp.ConnectionLabelingCallback接口用于创建标签回调，该回调确定是否已存在与请求标签的连接。如果不存在连接，则接口允许根据需要配置当前连接。本章将详细介绍这些接口的方法。

将数据从非Oracle数据库系统复制到Oracle数据库服务器

CREATE TABLE语句允许您将数据从非Oracle数据库复制到Oracle数据库。

要在本地数据库上创建表并从非Oracle表中插入行，请使用以下语法：

CREATE TABLE table_name AS 查询;

以下示例在本地Oracle数据库中创建表emp ，并插入非Oracle数据库的EMP表中的行：

CREATE TABLE table1 AS SELECT * FROM“EMP”@remote_db;

或者，您可以使用SQL * Plus COPY命令将数据从非Oracle数据库复制到Oracle数据库服务器。

在Oracle Solaris系统上，在安装Oracle ASMFD之前，请考虑以下准则：

  • 确保将磁盘标记为SMI或可扩展固件接口（EFI）。

  • 确保磁盘至少有一个代表整个磁盘的片。例如， Slice 2 。

3个伪列

伪列的行为类似于表列，但实际上并不存储在表中。您可以从伪列中进行选择，但不能插入，更新或删除它们的值。伪列也类似于没有参数的函数（参见函数 ）。但是，没有参数的函数通常会为结果集中的每一行返回相同的值，而伪列通常会为每一行返回不同的值。
