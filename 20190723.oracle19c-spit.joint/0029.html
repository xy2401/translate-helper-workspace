启用名称服务缓存守护程序

要允许Oracle Clusterware更好地容忍使用NAS设备或NFS挂载的网络故障，请启用名称服务缓存守护程序（ nscd ）。

要在重新启动系统时检查nscd是否设置为load，请输入命令chkconfig --list nscd 。例如：
#chkconfig --list nscd nscd 0：off 1：off 2：off 3：on 4：off 5：off 6：off

nscd在运行级别3时打开，在运行级别5时关闭。应为运行级别3和运行级别5打开nscd 。

要更改配置以确保运行级别3和运行级别5都启用了nscd ，请以root身份输入以下命令：

#chkconfig --level 35 nscd on

要在当前会话中启动nscd ，请以root身份输入命令：

#service nscd start

要使用新设置重新启动nscd ，请以root身份输入以下命令：

#service nscd restart
对于Oracle Linux 7，请验证是否已启用nscd ，如下所示：
systemctl --all | grep nscd nscd.service已加载活动运行名称服务缓存守护程序

12.39保存

句法

SAV [E] [FILE] file_name [。 分机 ] [ CRE [ATE] | REP [LACE] |附加]]

保存内容 SQL缓冲区在操作系统脚本 。

条款

文件

用于指定以下参数是要为保存的脚本指定的名称的关键字。通常省略此可选关键字。

如果要使用名称文件保存脚本，因为它是命令关键字，您需要将名称文件放在单引号中。

file_name [。 分机 ]

指定脚本您希望保存缓冲区的内容。 

创建

创建具有指定名称的新文件。这是默认行为。 

更换]

替换现有文件的内容。如果该文件不存在，REPLACE将创建该文件。 

附加]

将缓冲区的内容添加到指定文件的末尾。

用法

如果未指定扩展名，则SQL * Plus将采用默认命令文件扩展（通常是SQL）。看到 SET SUF [FIX] {SQL |文字}有关更改此默认扩展的信息。

如果希望以与SAVE命令子句（CREATE，REPLACE或APPEND）相同的名称保存文件，则必须指定文件扩展名。

保存SQL缓冲区的内容时，SAVE会在文件末尾添加包含斜杠（/）的行。

例子

要将缓冲区的内容保存在名为DEPTSALRPT且扩展名为SQL的文件中，请输入

保存DEPTSALRPT

要将缓冲区的内容保存在名为DEPTSALRPT且扩展名为OLD的文件中，请输入

保存DEPTSALRPT.OLD

版本查询伪列

版本查询伪列仅在Oracle闪回版本查询中有效，后者是Oracle闪回查询的一种形式。版本查询伪列是：

  • VERSIONS_STARTSCN和VERSIONS_STARTTIME ：创建行版本时启动系统更改编号（SCN）或TIMESTAMP 。此伪列标识数据首次具有行版本中反映的值的时间。使用此伪列来标识Oracle闪回表或Oracle闪回查询的过去目标时间。如果此伪列为NULL ，则在启动之前创建行版本。

  • VERSIONS_ENDSCN和VERSIONS_ENDTIME ：行版本过期时的SCN或TIMESTAMP 。如果伪列为NULL ，则行版本在查询时是当前的，或者行对应于DELETE操作。

  • VERSIONS_XID ：创建行版本的事务的标识符（ RAW编号）。

  • VERSIONS_OPERATION ：事务执行的操作： I表示插入， D表示删除， U表示更新。版本是插入，删除或更新的行的版本;即， INSERT操作之后的行， DELETE操作之前的行或受UPDATE操作影响的行。

    对于索引键的用户更新，Oracle闪回版本查询可能将UPDATE操作视为两个操作， DELETE加INSERT ，表示为两个版本行，其中D后跟I VERSIONS_OPERATION 。

分层查询运算符

两个运算符PRIOR和CONNECT_BY_ROOT仅在分层查询中有效。

PRIOR

在分层查询中， CONNECT BY condition中的一个表达式必须由PRIOR运算符限定。如果CONNECT BY condition是复合的，那么只有一个条件需要PRIOR运算符，尽管您可以有多个PRIOR条件。PRIOR计算分层查询中当前行的父行的紧接着的表达式。

在将列值与相等运算符进行比较时，最常用的是PRIOR 。（ PRIOR关键字可以位于运营商的任何一侧。）PRIOR使Oracle使用列中父行的值。在CONNECT BY子句中理论上可以使用等号（=）以外的运算符。但是，这些其他运算符创建的条件可能会导致可能组合的无限循环。在这种情况下，Oracle在运行时检测循环并返回错误。有关此运算符的更多信息，请参阅分层查询 ，包括示例。

CONNECT_BY_ROOT

CONNECT_BY_ROOT是一元运算符，仅在分层查询中有效。使用此运算符限定列时，Oracle将使用根行中的数据返回列值。此运算符扩展了分层查询的CONNECT BY [ PRIOR ]条件的功能。

对CONNECT_BY_ROOT的限制

您无法在START WITH条件或CONNECT BY条件下指定此运算符。

查看此信息，以获取针对专用互连使用多个NIC的Oracle RAC系统的rp_filter参数的设置。

使用Oracle Linux内核2.6.31（也包括Oracle Unbreakable Enterprise Kernel 2.6.32），在反向路径过滤中修复了一个错误。作为此更正的结果，使用多个NIC进行专用互连的Oracle RAC系统现在需要对rp_filter参数进行特定设置。此要求也适用于运行Linux内核2.6.32及更高版本的所有Exadata系统。如果没有这些rp_filter参数设置系统，则可以阻止或丢弃互连数据包。

目的

ORA_HASH是一个计算给定表达式的哈希值的函数。此功能对于分析数据子集和生成随机样本等操作非常有用。

  • expr参数确定您希望Oracle数据库计算哈希值的数据。expr表示的数据长度没有限制，通常解析为列名。expr不能是LONG或LOB类型。它不能是用户定义的对象类型，除非它是嵌套的表类型。嵌套表类型的哈希值不依赖于集合中元素的顺序。expr支持所有其他数据类型。

  • 可选的max_bucket参数确定哈希函数返回的最大桶值。您可以指定0到4294967295之间的任何值。默认值为4294967295。

  • 可选的seed_value参数使Oracle能够为同一组数据生成许多不同的结果。Oracle将哈希函数应用于expr和seed_value的组合。您可以指定0到4294967295之间的任何值。默认值为0。

该函数返回NUMBER值。

6.1关于评分和部署

评分是将模型应用于新数据。在Oracle Data Mining中，评分由SQL语言函数执行。

预测函数执行分类，回归或异常检测。群集功能将行分配给群集。特征提取功能将输入数据转换为一组高阶预测变量。DBMS_DATA_MINING PL / SQL包中也提供了评分过程。

部署是指在目标环境中使用模型。构建模型后，面临的挑战就是部署模型以获得最佳结果，并将其维护在生产环境中。 部署可以是以下任何一种：

  • 为批次或分数评分数据实时结果。分数可以包括预测，概率，规则和其他统计数据。

  • 提取模型详细信息以生成报告。例如：聚类规则，决策树规则或属性重要性模型中的属性排名。

  • 通过将挖掘结果合并到应用程序或操作系统中来扩展数据仓库的商业智能基础架构。

  • 将模型从构建它的数据库移动到用于评分的数据库（导出/导入）

Oracle Data Mining支持所有这些部署方案。

注意：

Oracle Data Mining评分操作支持并行执行。什么时候启用并行执行，将多个CPU和I / O资源应用于单个数据库操作的执行。

并行执行提供显着的性能改进，尤其是涉及复杂查询的操作以及通常与决策支持系统（DSS）和数据仓库相关联的大型数据库。

要将ASMLIB配置为首先选择多路径磁盘，请完成以下过程。

  1. 使用文本编辑器打开ASMLIB配置文件/etc/sysconfig/oracleasm 。
  2. 编辑ORACLEASM_SCANORDER变量以提供多路径磁盘的前缀路径。

11.2内存保护架构

Memory Guard实现为作为由Cluster Ready Services（CRS）管理的J2EE容器中的MBean运行的守护程序。

Memory Guard托管在qosmserver资源上，该资源在任何群集节点上运行以实现高可用性。

图11-1 Memory Guard架构


“图11-1 Memory Guard架构”的说明

Cluster Health Monitor将度量标准流发送到Memory Guard，后者提供有关群集节点的内存资源的实时信息。此信息包括以下内容：

  • 可用内存量

  • 当前正在使用的内存量

获取内存资源信息后，Memory Guard将从Oracle Clusterware收集集群拓扑。Memory Guard使用群集拓扑和内存指标来识别具有内存压力的数据库节点。当空闲内存小于某个阈值时，内存被认为是压力。

然后，Memory Guard会在受压节点上以事务方式停止Oracle Clusterware管理的数据库服务。Memory Guard可以缓解内存压力，而不会影响已经运行的会话及其关联的事务。完成后，这些进程使用的内存开始释放并添加到节点上的可用内存池中。当Memory Guard检测到可用内存量超过阈值时，它会重新启动受影响节点上的服务。

当服务在受压节点上停止时，侦听器会将该服务的新连接重定向到为非单例数据库实例提供相同服务的其他节点。但是，对于策略管理的数据库，不会停止最后一个服务实例以确保可用性。

注意：

Memory Guard可以为处于打开状态的数据库启动或停止服务。Memory Guard不管理默认数据库服务，并且在升级或降级数据库时不起作用。

关于orabasetab

orabasetab文件用于根据$ ORACLE_HOME定义基本目录：ORACLE_BASE，ORACLE_BASE_HOME和ORACLE_BASE_CONFIG。

orabasetab文件驻留在ORACLE_HOME/install/orabasetab ，可用于确定ORACLE_HOME是只读还是读/写。它还定义了Oracle主目录的ORACLE_BASE和HOME_NAME。HOME_NAME是ORACLE_HOME的内部名称。

orabasetab文件中的最后一行以$ ORACLE_HOME开头，它定义了$ ORACLE_HOME的目录。最后一行包含四个字段，每个字段用冒号分隔符(:).分隔(:).

  1. 第一个字段匹配当前的$ ORACLE_HOME。

  2. 第二个字段定义当前ORACLE_HOME的ORACLE_BASE。

  3. 第三个字段定义了HOME_NAME，用于在只读ORACLE_HOME中构造ORACLE_BASE_HOME路径。

  4. 第四个字段在读/写ORACLE_HOME中显示N ，在只读ORACLE_HOME中显示Y

在只读ORACLE_HOME中，ORACLE_BASE_HOME路径为ORACLE_BASE/homes/HOME_NAME ，ORACLE_BASE_CONFIG与ORACLE_BASE相同。

在读/写ORACLE_HOME中，ORACLE_HOME，ORACLE_BASE_HOME和ORACLE_BASE_CONFIG都是相同的。

关于以只读模式配置Oracle Home

从Oracle Database 18c开始，您可以通过将只读Oracle主目录部署为软件映像来简化跨多个数据库服务器的软件修补和批量部署。

只读Oracle主目录（ ORACLE_HOME ）可防止在Oracle主目录中创建和修改文件。要以只读模式配置Oracle主目录，首先使用纯软件部署安装Oracle数据库软件，然后在创建侦听器和数据库之前将其配置为只读Oracle主目录。

传统的读写Oracle主目录包含特定于实例的文件，因此如果您对其进行修补，则必须单独修补每个文件。但是，当Oracle主目录为只读时，特定于实例的文件将单独存储在Oracle基本目录（ ORACLE_BASE ）中，而不是Oracle主目录中。由于此配置，您可以使用只读Oracle主目录作为可以跨多个数据库服务器共享的软件映像，因为它只存储静态文件。此选项简化了修补和批量部署，因为当您要将修补程序分发到多个数据库服务器时，只需更新一个Oracle主映像。

除了传统的ORACLE_BASE和ORACLE_HOME目录之外，只读Oracle主目录中还存在以下附加目录：

  • ORACLE_BASE_HOME ：这是ORACLE_BASE目录中的子目录。此目录包含特定于用户的文件，特定于实例的文件和日志文件。

  • ORACLE_BASE_CONFIG ：该目录与ORACLE_BASE目录相同。此目录包含特定于实例的动态文件，例如配置文件。

关于orabasetab

orabasetab文件用于根据$ ORACLE_HOME定义基本目录：ORACLE_BASE，ORACLE_BASE_HOME和ORACLE_BASE_CONFIG。

orabasetab文件驻留在ORACLE_HOME/install/orabasetab ，可用于确定ORACLE_HOME是只读还是读/写。它还定义了Oracle主目录的ORACLE_BASE和HOME_NAME。HOME_NAME是ORACLE_HOME的内部名称。

orabasetab文件中的最后一行以$ ORACLE_HOME开头，它定义了$ ORACLE_HOME的目录。最后一行包含四个字段，每个字段用冒号分隔符(:).分隔(:).

  1. 第一个字段匹配当前的$ ORACLE_HOME。

  2. 第二个字段定义当前ORACLE_HOME的ORACLE_BASE。

  3. 第三个字段定义了HOME_NAME，用于在只读ORACLE_HOME中构造ORACLE_BASE_HOME路径。

  4. 第四个字段在读/写ORACLE_HOME中显示N ，在只读ORACLE_HOME中显示Y

在只读ORACLE_HOME中，ORACLE_BASE_HOME路径为ORACLE_BASE/homes/HOME_NAME ，ORACLE_BASE_CONFIG与ORACLE_BASE相同。

在读/写ORACLE_HOME中，ORACLE_HOME，ORACLE_BASE_HOME和ORACLE_BASE_CONFIG都是相同的。

G Oracle标签安全限制

此Oracle Label Security版本中存在一些限制。

这些限制如下：

  • CREATE TABLE AS SELECT限制

    如果尝试在受Oracle Label Security策略保护的模式中执行CREATE TABLE AS SELECT ，则该语句将失败。

  • 标签标签限制

    标签标签在数据库中的策略中必须是唯一的。在数据库中使用多个策略时，不能在不同的策略中使用相同的数字标签标记。

  • 出口限制

    在Oracle Database 12c第1版（12.1）之前，由于在Oracle Label Security中使用了opaque类型，因此无法导出LBACSYS模式。除了不会导出LBACSYS模式之外，可以使用安装的Oracle Label Security导出整个数据库（参数FULL =Y ）。

    从Oracle Database 12c版开始，此限制已被删除。有关可以从中支持导出的数据库版本的其他详细信息，请参阅完整数据库导出 。

  • Oracle Label Security删除限制

    您无法删除Oracle Label Security，但可以禁用它。请参阅禁用Oracle Label Security 。

  • 共享架构支持限制

    Oracle Internet Directory中定义的用户帐户无法获得单独的Oracle Label Security授权。但是，可以将授权授予目录用户映射到的共享模式。

    可以通过编程方式使用Oracle Label Security函数SET_ACCESS_PROFILE来设置标签授权配置文件，以便在用户通过身份验证并映射到共享模式后使用。Oracle Label Security不会在Oracle Label Security中给予标签授权的用户与实际数据库用户之间强制执行映射。

  • 隐藏列限制

    PL / SQL无法识别对表中隐藏列的引用。将生成编译器错误。

HugePages和透明HugePages配置的限制

查看本节中讨论的HugePages和Transparent HugePages指南。

Oracle建议您禁用Transparent HugePages，因为它们可能会导致访问内存的延迟，从而导致Oracle RAC环境中的节点重新启动，或Oracle Database单实例的性能问题或延迟。Oracle继续建议使用标准的HugePages for Linux。

Red Hat Enterprise Linux 7，SUSE 12和Oracle Linux 7默认启用透明HugePages内存，早期版本的Oracle Linux具有Unbreakable Enterprise Kernel 2（UEK2）内核。默认情况下，在UEK2及更高版本的UEK内核的后续版本中禁用透明HugePages内存。

注意：

虽然在UEK2和更高版本的UEK内核上禁用了透明HugePages，但默认情况下可以在Linux系统上启用Transparent HugePages。

透明Hugepages类似于标准HugePages。但是，虽然标准HugePages在启动时分配内存，但Transparent Hugepages内存使用内核中的khugepaged线程在运行时使用可交换的HugePages动态分配内存。

HugePages使用内存映射文件为大页表分配不可交换的内存。默认情况下不启用HugePages。如果启用HugePages，则应在计算交换空间之前从可用RAM中扣除分配给HugePages的内存。有关更多信息，请参阅您的分发文档以及Oracle Technology Network和My Oracle Support。

在Oracle Grid Infrastructure安装期间，Grid Infrastructure Management Repository（GIMR）配置为使用HugePages。由于Grid Infrastructure Management Repository数据库在群集上安装的所有其他数据库之前启动，如果分配给HugePages的空间不足，则一个或多个数据库的系统全局区域（SGA）可能会映射到常规页面，而不是Hugepages，这会对性能产生不利影响。将HugePages内存分配配置为足够大的大小，以容纳要在群集上安装的所有数据库的SGA大小的总和，以及Grid Infrastructure Management Repository。

目的

GROUPING_ID返回与行相关的GROUPING位向量对应的数字。GROUPING_ID仅适用于包含GROUP BY扩展的SELECT语句，例如ROLLUP或CUBE ，以及GROUPING函数。在具有许多GROUP BY表达式的查询中，确定特定行的GROUP BY级别需要许多GROUPING函数，这会导致繁琐的SQL。 GROUPING_ID在这些情况下很有用。

关于无效的架构对象和数据库升级

运行utlrp.sql以验证无效对象，作为升级测试计划的一部分。

数据库升级后，发布更改可能导致升级后的数据库中的架构对象无效。通常，无效对象在访问或运行时会自行修复。但是，Oracle建议您在修补和升级过程中重新编译数据库中的无效对象，以便在用户遇到这些无效对象之前解决无效对象和任何所需依赖项的问题。

对象验证是一种检查Oracle数据库数据定义语言（DDL）语句的操作。这些语句用于定义数据库结构或模式。验证DDL语句可能需要一些时间才能完成。以下是可能影响对象验证时间的一些常见因素列表：

  • 无效对象的数量

  • CPU类型

  • 处理器速度

  • 系统负载

  • 可用的物理内存

utlrp.sql命令重新编译处于无效状态的所有对象，包括包，过程和类型。它位于$ORACLE_HOME/rdbms/admin目录中。utlrp.sql脚本根据可用的CPU数量（由参数cpu_count标识）自动运行，并根据每个CPU的线程数（由参数parallel_threads_per_cpu标识）进行串行或并行重新编译。在Oracle Real Application Clusters系统（Oracle RAC）上，所有Oracle RAC节点上都添加了并行线程数。

以SYS用户身份运行该命令，或者以授予SYSDBA系统权限的其他用户帐户身份运行该命令。

Oracle建议您在早期版本的Oracle数据库中运行utlrp.sql命令，以重新编译数据库中的任何现有无效对象。特别要确保更新SYS和SYSTEM用户模式无效对象。在升级测试期间，作为升级测试计划的一部分，在升级的Oracle数据库中运行utlrp.sql ， utlrp.sql在升级过程中包括规划重新编译时间。重新编译时间与数据库中无效对象的数量成正比。如果升级导致大量无效对象，则utlrp.sql可能需要很长时间才能运行。

检查SQLNET.ALLOWED_LOGON_VERSION参数行为

从版本10g之前的客户端连接到Oracle数据库失败，错误ORA-28040: No matching authentication protocol 。

从Oracle Database 18c开始， SQLNET.ALLOWED_LOGON_VERSION参数的默认值从Oracle Database 12c（12.2）中的11更改为Oracle Database 18c及更高版本中的12。不推荐使用此参数。

SQLNET.ALLOWED_LOGON_VERSION现在替换为SQLNET.ALLOWED_LOGON_VERSION_SERVER和SQLNET.ALLOWED_LOGON_VERSION_CLIENT参数。如果未在已升级的数据库中显式设置SQLNET.ALLOWED_LOGON_VERSION_SERVER参数，则来自版本10g之前的客户端的连接将失败，并显示错误ORA-28040: No matching authentication protocol 。为了更好的安全性，请检查数据库用户的密码验证程序，然后通过设置SQLNET.ALLOWED_LOGON_VERSION_SERVER和SQLNET.ALLOWED_LOGON_VERSION_CLIENT参数来配置数据库以使用正确的密码验证程序。

如果您在现有数据库中具有受密码保护的角色（安全角色），并且使用默认的SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为12升级到Oracle Database 18c及更高版本，因为这些安全角色仅具有版本10g验证程序，则密码必须由管理员重置每个安全角色，以便安全角色在升级后仍可使用。

在安装期间使用自动SSH配置

要安装Oracle软件，请在所有集群成员节点之间配置安全shell（SSH）连接。

Oracle Universal Installer（OUI）在安装期间使用ssh和scp命令在其他群集节点上运行远程命令并将文件复制到其他群集节点。您必须配置SSH，以便这些命令不会提示输入密码。

注意：

Oracle配置助手使用SSH进行从本地节点到远程节点的配置操作。Oracle Enterprise Manager也使用SSH。 RSH不再受支持。

您可以在安装期间为运行安装的用户帐户从OUI界面配置SSH。自动配置在所有集群成员节点之间创建无密码SSH连接。Oracle建议您尽可能使用自动过程。

要使脚本能够运行，您必须从要使用的任何现有Oracle软件安装所有者的配置文件中删除stty命令，并删除在登录期间触发的其他安全措施，并生成到终端的消息。这些消息，邮件检查和其他显示会阻止Oracle软件安装所有者使用OUI中内置的SSH配置脚本。如果未禁用它们，则必须手动配置SSH，然后才能运行安装。

在极少数情况下，当远程节点关闭SSH连接时，Oracle Clusterware安装可能会在“AttachHome”操作期间失败。要避免此问题，请通过在所有群集节点上的SSH守护程序配置文件/etc/ssh/sshd_config设置以下参数，将timeout wait设置为unlimited：

LoginGraceTime 0
