11升级Oracle Grid Infrastructure

Oracle Grid Infrastructure升级包括升级Oracle Clusterware和Oracle自动存储管理（Oracle ASM）。

Oracle Grid Infrastructure升级可以是滚动升级，其中一部分节点在其他节点保持活动状态时关闭并升级。从Oracle ASM 11g第2版（11.2）开始，Oracle ASM升级可以是滚动升级。

您还可以使用Oracle Fleet Patching和Provisioning升级群集的Oracle Grid Infrastructure。

  • 了解不合适的升级
    通过不正常升级，安装程序会在单独的Oracle Clusterware主目录中安装较新版本。
  • 关于Oracle Grid Infrastructure升级和降级
    您可以将Oracle Grid Infrastructure升级或降级到受支持的版本。
  • Oracle Grid Infrastructure升级选项
    升级到Oracle Grid Infrastructure 19c时，升级到Oracle Flex Cluster配置。
  • Oracle Grid Infrastructure升级的限制
    查看以下信息，了解Oracle Grid Infrastructure安装升级的限制和更改，其中包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）。
  • 准备升级现有Oracle Clusterware安装
    如果您已安装现有Oracle Clusterware，则可以通过执行非现场升级来升级现有群集。您无法执行就地升级。
  • 了解使用批次滚动升级
    您可以批量执行Oracle Grid Infrastructure的滚动升级。
  • 执行Oracle Grid Infrastructure的滚动升级
    查看此信息以执行Oracle Grid Infrastructure的滚动升级。
  • 关于使用Oracle Fleet补丁和配置升级Oracle Grid Infrastructure
    Oracle Fleet Patching and Provisioning（Oracle FPP）是一种用于配置和修补Oracle主目录的软件生命周期管理方法。
  • 将修补程序应用于Oracle Grid Infrastructure
    升级Oracle Grid Infrastructure 19c后，可以通过从My Oracle Support下载来安装单个软件修补程序。
  • 更新Oracle企业管理器云控制目标参数
    升级Oracle Grid Infrastructure后，升级企业管理器云控制目标。
  • 解锁和卸载以前的版本网格主页
    从先前版本升级后，如果要卸载以前版本的Oracle Grid Infrastructure主目录，则必须先更改先前版本Grid主目录的权限和所有权。
  • 升级后检查群集运行状况监视器存储库大小
    如果要将Oracle Grid Infrastructure从使用IPD / OS的先前版本升级到当前版本，请查看Cluster Health Monitor存储库大小（CHM存储库）。
  • 将Oracle Clusterware降级为早期版本
    升级成功或失败后，您可以将Oracle Clusterware还原到以前的版本。
  • 完成安装和升级失败或中断
    如果Oracle Universal Installer（OUI）在您开始升级的节点上退出，或者在确认所有节点上都运行了rootupgrade.sh脚本之前节点重新启动，则升级仍然不完整。
  • 升级Oracle Grid Infrastructure后转换为Oracle Extended Cluster
    升级Oracle Grid Infrastructure后，请查看此信息以转换为Oracle Extended Cluster。Oracle Extended Cluster使您可以在群集上部署Oracle RAC数据库，其中某些节点位于不同的站点中。

使用安装响应文件进行安装后配置

从Oracle Database 12c第2版（12.2）开始，您可以使用在安装期间创建的响应文件来完成安装后配置。

使用-executeConfigTools选项运行安装程序，以在安装Oracle Grid Infrastructure或Oracle数据库后配置配置助手。您可以使用位于$ORACLE_HOME/install/response/ product_timestamp.rsp的响应文件来获取运行配置工具所需的密码。在运行-executeConfigTools命令之前，必须使用所需的密码更新响应文件。

Oracle强烈建议您使用密码响应文件来维护安全性：

  • 响应文件的权限应设置为600。

  • 响应文件的所有者应该是安装所有者用户，并将该组设置为中央库存（ oraInventory ）组。

示例A-1 Oracle Grid Infrastructure的响应文件密码（ grid用户）

grid.install.crs.config.ipmi.bmcPassword = password grid.install.asm。SYSASMPassword = password grid.install.asm.monitorPassword = password grid.install.config.emAdminPassword = password

如果您没有BMC卡，或者您不想启用IPMI，请将ipmi.bmcPassword输入字段留空。

如果您不想启用Oracle Enterprise Manager进行管理，请将emAdminPassword密码字段留空。

示例A-2独立服务器的Oracle Grid Infrastructure响应文件密码（ oracle用户）

oracle.install.asm。SYSASMPassword = password oracle.install.asm.monitorPassword = password oracle.install.config.emAdminPassword = password

如果您不想启用Oracle Enterprise Manager进行管理，请将emAdminPassword密码字段留空。

示例A-3 Oracle数据库的响应文件密码（ oracle用户）

此示例说明了指定用于数据库配置助手的密码。

oracle.install.db.config.starterdb.password。SYS = password oracle.install.db.config.starterdb.password。SYSTEM = password oracle.install.db.config.starterdb.password。DBSNMP = password oracle.install.db.config.starterdb.password。PDBADMIN = password oracle.install.db.config.starterdb.emAdminPassword = password oracle.install.db.config.asm。ASMSNMPPassword = password

您还可以指定oracle.install.db.config.starterdb.password.ALL= 密码为所有数据库用户使用相同的密码。

数据库配置助手需要SYS，SYSTEM和DBSNMP密码才能与Oracle DBCA一起使用。您必须指定以下密码，具体取决于您的系统配置：

  • 如果数据库使用Oracle自动存储管理（Oracle ASM）进行存储，则必须为ASMSNMPPassword变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

  • 如果使用一个或多个可插拔数据库（PDB）创建多租户容器数据库（CDB），则必须为PDBADMIN变量指定密码。如果您未使用Oracle ASM，请将此密码变量的值保留为空。

关于CVU升级验证命令选项

查看有关运行升级验证的此信息。

  • 运行Oracle Universal Installer（OUI），并允许OUI中内置的Cluster Verification Utility（CVU）验证执行系统检查并生成修复脚本。

  • 运行CVU手动脚本cluvfy.sh以执行系统检查并生成cluvfy.sh脚本。

要使用OUI执行安装前检查并生成修复脚本，请像平常一样运行安装。OUI启动CVU，并在安装过程中执行系统检查。如果您认为已完成预安装检查，并且要确认系统配置满足最低安装要求，则选择OUI以执行这些检查尤为合适。

要为CVU使用cluvfy.sh命令行脚本，请导航到解压缩要升级的映像文件的新Grid主目录，其中包含runcluvfy.sh脚本，然后运行命令runcluvfy.sh stage -pre crsinst -upgrade to检查Oracle Clusterware安装是否准备好进行升级。使用-pre crsinst -upgrade选项运行runcluvfy.sh会执行系统检查，以确认群集是否处于从现有集群件安装升级的正确状态。

该命令使用以下语法，其中变量内容以斜体表示：

runcluvfy.sh stage -pre crsinst -upgrade [-rolling] -src_crshome src_Gridhome ] -dest_crshome dest_Gridhome -dest_version dest_release [-fixup] [ -  fixupnoexec] [ - 方法sudo -user user_name [-location dir_path ] [ -  method root] [ -详细]

选项是：

  • -rolling

    使用此选项可验证滚动升级的准备情况。

  • -src_crshome src_Gridhome

    使用此选项指示要升级的源Oracle Clusterware或Grid home的位置，其中src_Gridhome是要升级的主src_Gridhome的路径。

  • -dest_crshome dest_Gridhome

    使用此选项指示升级Grid home的位置，其中dest_ Gridhome是Grid home的路径。

  • -dest_version dest_release

    使用-dest_version选项指示升级的版本号，包括任何补丁集。版本号必须包含五个数字，指定发布到特定于平台的修补程序级别。例如： 19.0.0.0.0 。

  • -fixup [-method sudo -user user_name [-location dir_path ][-method root ]

    使用-fixup选项表示您要为完成所需的任何步骤生成说明，以确保您的群集已准备好进行升级。默认位置是CVU工作目录。

    -fixup -method选项定义运行root脚本的方法。-method标志需要以下选项之一：

    • sudo ：在sudoers列表中以用户身份运行。

    • root ：以root用户身份运行。

    如果选择sudo ，则输入-location选项以在服务器上提供Sudo的路径，并输入-user选项以向用户帐户提供Sudo权限。

  • -fixupnoexec

    如果指定了该选项，则在验证失败时，将生成修正数据，并显示手动执行生成的修正的指令。

  • -verbose

    使用-verbose标志生成单个检查的详细输出。

DROP PLUGGABLE DATABASE

目的

使用DROP PLUGGABLE DATABASE语句删除可插拔数据库（PDB）。PDB可以是传统的PDB，应用程序容器，应用程序种子或应用程序PDB。

删除PDB时，多租户容器数据库（CDB）的控制文件将被修改，以删除对已删除的PDB及其数据文件的所有引用。与删除的PDB关联的存档日志和备份不会被删除。您可以使用Oracle Recovery Manager（RMAN）删除它们，也可以保留它们以防您随后要执行PDB的时间点恢复。

警告：

您无法回滚DROP PLUGGABLE DATABASE语句。

先决条件

您必须连接到CDB。

要删除传统PDB或应用程序容器，当前容器必须是根，必须通过AS SYSDBA或AS SYSOPER进行身份验证，并且必须通常授予您SYSDBA或SYSOPER权限，或者在本地授予您权限。在传统的PDB或应用程序容器中本地删除。应用程序容器必须为空，也就是说，它不能包含应用程序种子或任何应用程序PDB。

要删除应用程序种子，当前容器必须是根目录或应用程序根目录，必须通过AS SYSDBA或AS SYSOPER进行身份验证，并且必须通常授予您SYSDBA或SYSOPER权限，或者在根目录中授予您本地权限。或应用程序根。

要删除应用程序PDB，当前容器必须是根或应用程序根目录，您必须通过AS SYSDBA或AS SYSOPER进行身份验证，并且必须通常授予您SYSDBA或SYSOPER权限，或者在根目录中授予您本地权限。或应用程序root，并在您要删除的应用程序PDB中本地。

要指定KEEP DATAFILES （默认值），必须拔下要删除的PDB。

要指定INCLUDING DATAFILES ，要删除的PDB必须处于装入模式或必须拔下插头。

语义

pdb_name

指定要删除的PDB的名称。你不能放弃种子（ PDB$SEED ）。但是，您可以删除应用程序种子。

保留数据文件

指定KEEP DATAFILES以在删除PDB后保留与PDB关联的数据文件。删除PDB的临时文件，因为不再需要它。这是默认值。

保留数据文件在从一个CDB拔出的PDB插入另一个CDB的情况下可能很有用，两个CDB共享存储设备。

包括数据文件

指定INCLUDING DATAFILES以删除与要删除的PDB关联的数据文件。PDB的临时文件也将被删除。

了解密码区分大小写和升级

默认情况下，Oracle Database 12 c第2版（12.2）及更高版本将升级为独占模式。独占模式不支持不区分大小写的基于密码的身份验证。

当服务器以独占模式运行时，仅具有10G密码版本的帐户将无法访问。

在以前的Oracle数据库发行版中，您可以配置身份验证协议，以便通过设置SEC_CASE_SENSITIVE_LOGON=FALSE来允许不区分大小写的基于密码的身份验证。从Oracle Database 12 c第 2版（12.2）开始，默认的基于密码的身份验证协议配置不包括使用不区分大小写的10G密码版本。默认情况下， SQLNET.ORA参数SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为12 ，这是一种独占模式。在独占模式下配置数据库时，基于密码的身份验证协议要求对要进行身份验证的帐户存在一个区分大小写的密码版本（ 11G或12C ）。此模式不包括使用早期版本中使用的10G密码版本。升级到Oracle Database 12 c第 2版及更高版本后，仅具有不区分大小写的10G密码版本的帐户将无法访问。发生此更改是因为默认情况下服务器以独占模式运行。在独占模式下配置Oracle数据库时，它无法使用旧的10G密码版本来验证客户端。服务器没有用于验证客户端的密码版本。

为了提高安全性，Oracle建议您启用区分大小写的基于密码的身份验证。此设置是默认设置。但是，您可以在升级到新的Oracle数据库版本期间临时禁用区分大小写的身份验证。升级后，您可以决定是否要在实施计划中启用区分大小写的基于密码的身份验证功能来管理密码版本。

在升级之前，Oracle建议您确定对基于密码的默认身份验证协议配置的此更改是否会影响您。执行以下检查：

  • 确保您没有将已弃用的参数SEC_CASE_SENSITIVE_LOGON设置为FALSE。将此参数设置为FALSE可防止使用区分大小写的密码版本（ 11G和12C密码版本）进行身份验证。

当需要翻译或后处理语句时，异构服务和网关会重写SQL语句。

为了以下示例假设非Oracle数据库不支持INITCAP功能。考虑从非Oracle数据库请求以下内容的程序。例如：

SELECT“COLUMN_A”FROM“test”@remote_db WHERE“COLUMN_A”= INITCAP（“COLUMN_B”）;

非Oracle数据库无法识别INITCAP函数，因此Oracle数据库服务器从远程数据库中的表test中获取数据并在本地过滤结果。网关重写SELECT语句，如下所示：

SELECT“COLUMN_A”，“COLUMN_B”FROM“test”@remote_db;

查询结果从网关发送到Oracle，并由Oracle数据库服务器过滤。

如果提供了字符串文字或绑定变量来代替"COLUMN_B" ，如上例所示，则在将SQL命令发送到网关之前，Oracle服务器的异构服务组件将应用INITCAP函数。例如，如果发出以下SQL命令：

SELECT“COLUMN_A”FROM“test”@remote_db WHERE“COLUMN_A”= INITCAP（'jones'）;

以下SQL命令将发送到网关：

SELECT“COLUMN_A”FROM“test”@remote_db WHERE“COLUMN_A”='琼斯';

请考虑以下UPDATE请求：

UPDATE“test”@remote_db SET“COLUMN_A”='new_value'WHERE“COLUMN_A”= INITCAP（“COLUMN_B”）;

在这种情况下，Oracle数据库服务器和网关无法弥补非Oracle端的缺乏支持，因此会发出错误。

如果提供了字符串文字或绑定变量来代替"COLUMN_B" ，如前面的示例所示，则Oracle服务器的异构服务组件将在将SQL命令发送到网关之前应用INITCAP函数。例如，如果发出以下SQL命令：

UPDATE“test”@remote_db SET“COLUMN_A”='new_value'WHERE“COLUMN_A”= INITCAP（'jones'）;

数据类型检查支持远程映射语句

在异构环境中有远程映射语句的数据类型检查支持。

Oracle数据库始终在同构环境中执行数据类型检查和数据类型强制。例如， SELECT * FROM EMP WHERE EMPNO='7934'将返回与SELECT * FROM EMPNO WHERE EMPNO=7934相同的结果。在异构环境中还有对远程映射语句的完整数据类型检查支持。通常，SQL语句中的操作数，无论是列，文字还是绑定变量，都将在内部进行数据类型检查。请考虑以下示例：

SELECT * FROM EMP @LINK WHERE NUMBER_COLUMN ='123'CELECT * FROM EMP @LINK WHERE NUMBER_COLUMN = CHAR_COLUMN; SELECT * FROM EMP @LINK WHERE NUMBER_COLUMN = CHAR_BIND_VARIABLE;

大多数非Oracle数据库不支持数据类型强制，如果先前的语句按原样发送到非Oracle数据库，则它们会失败。Oracle数据库的异构服务组件在将可接受的语句发送到非Oracle数据库之前执行数据类型检查和必要的数据类型强制。

数据类型检查在后处理或远程映射语句上提供一致的行为。请考虑以下两个陈述：

SELECT * FROM EMP @LINK WHERE TO_CHAR（EMPNO）='7933'+'1';

和：

SELECT * FROM EMP @LINK WHERE EMPNO ='7933'+'1';

无论是否在非Oracle数据库中支持TO_CHAR功能，前面的两个语句都提供相同的结果和强制。现在，请考虑以下声明：

SELECT * FROM EMP @LINK WHERE EMPNO ='123abc'+'1';

在强制执行数据类型检查时，Oracle中的强制尝试会生成错误并将其返回，而不会向非Oracle数据库发送任何语句。

总之，无论后处理或远程映射语句如何，都存在一致的数据类型检查和强制行为。

了解密码区分大小写和升级

默认情况下，Oracle Database 12 c第2版（12.2）及更高版本将升级为独占模式。独占模式不支持不区分大小写的基于密码的身份验证。

当服务器以独占模式运行时，仅具有10G密码版本的帐户将无法访问。

在以前的Oracle数据库发行版中，您可以配置身份验证协议，以便通过设置SEC_CASE_SENSITIVE_LOGON=FALSE来允许不区分大小写的基于密码的身份验证。从Oracle Database 12 c第 2版（12.2）开始，默认的基于密码的身份验证协议配置不包括使用不区分大小写的10G密码版本。默认情况下， SQLNET.ORA参数SQLNET.ALLOWED_LOGON_VERSION_SERVER设置为12 ，这是一种独占模式。在独占模式下配置数据库时，基于密码的身份验证协议要求对要进行身份验证的帐户存在一个区分大小写的密码版本（ 11G或12C ）。此模式不包括使用早期版本中使用的10G密码版本。升级到Oracle Database 12 c第 2版及更高版本后，仅具有不区分大小写的10G密码版本的帐户将无法访问。发生此更改是因为默认情况下服务器以独占模式运行。在独占模式下配置Oracle数据库时，它无法使用旧的10G密码版本来验证客户端。服务器没有用于验证客户端的密码版本。

为了提高安全性，Oracle建议您启用区分大小写的基于密码的身份验证。此设置是默认设置。但是，您可以在升级到新的Oracle数据库版本期间临时禁用区分大小写的身份验证。升级后，您可以决定是否要在实施计划中启用区分大小写的基于密码的身份验证功能来管理密码版本。

在升级之前，Oracle建议您确定对基于密码的默认身份验证协议配置的此更改是否会影响您。执行以下检查：

  • 确定您是否拥有仅使用10G不区分大小写的密码身份验证版本的帐户。

  • 确定您是否拥有Oracle Database 11g第 2版（11.2.0.3）数据库或未应用关键修补程序更新CPUOct2012或更高版本更新的早期客户端，以及任何不具有不区分大小写的10G密码版本的帐户。

  • 确保您没有将已弃用的参数SEC_CASE_SENSITIVE_LOGON设置为FALSE。将此参数设置为FALSE可防止使用区分大小写的密码版本（ 11G和12C密码版本）进行身份验证。

使用不区分大小写的版本的帐户选项

如果您的用户帐户只有不区分大小写的10G密码版本，则必须选择以下备选方案之一：

  • 在升级之前，请更新仅具有10G密码版本的每个帐户的密码版本。您可以通过使用10G密码版本使用户密码到期来更新密码版本，并请求这些用户登录到他们的帐户。当他们尝试登录时，服务器会自动更新密码版本列表，其中包括区分大小写的密码版本。

  • 将SQLNET.ORA参数SQLNET.ALLOWED_LOGON_VERSION_SERVER的设置更改为任何非独占模式的设置。例如： SQLNET.ALLOWED_LOGON_VERSION_SERVER=11

SYS_OP_ZONE_ID

句法


说明sys_op_zone_id.eps的说明

目的

SYS_OP_ZONE_ID将rowid作为参数，并返回区域ID。 rowid标识表中的行。区域ID标识包含该行的称为区域的连续磁盘块集。该函数返回NUMBER值。

使用CREATE MATERIALIZED ZONEMAP语句创建区域映射时，将使用SYS_OP_ZONE_ID函数。必须在区域映射的定义子查询的SELECT和GROUP BY子句中指定SYS_OP_ZONE_ID 。

对于rowid ，请指定区域映射的事实表的ROWID伪列。

使用schema和table指定事实表的模式和名称，或使用t_alias指定事实表的表别名。这些参数的规范取决于区域映射的定义子查询中的FROM子句：

  • 如果FROM子句指定事实表的表别名，则还必须在SYS_OP_ZONE_ID指定表别名（ t_alias ）。

  • 如果FROM子句未指定事实表的表别名，则使用table指定事实表的名称。如果事实表位于您自己的模式之外，则可以使用schema限定符。如果省略schema ，则数据库假定事实表在您自己的模式中。如果FROM子句仅指定一个表（事实表），则无需指定schema或table 。

可选的scale参数表示区域图的比例。没有必要指定此参数，因为默认情况下， SYS_OP_ZONE_ID使用正在创建的区域映射的比例。如果您指定了scale ，则它必须与正在创建的区域地图的比例相匹配。有关指定区域地图比例的信息，请参阅CREATE MATERIALIZED ZONEMAP的SCALE子句。

目的

DENSE_RANK计算有序行组中行的等级，并将等级返回为NUMBER 。排名是从1开始的连续整数。最大的排名值是查询返回的唯一值的数量。在发生关系时不会跳过排名值。排名标准具有相同值的行具有相同的排名。此功能对于前N个和下N个报告非常有用。

此函数接受任何数字数据类型作为参数并返回NUMBER 。

  • 作为聚合函数， DENSE_RANK计算由函数的参数相对于给定排序规范标识的假设行的密集等级。函数的参数必须全部计算到每个聚合组中的常量表达式，因为它们标识每个组中的单个行。常量参数表达式和聚合的order_by_clause中的表达式按位置匹配。因此，参数的数量必须相同，并且类型必须兼容。

  • 作为分析函数， DENSE_RANK根据order_by_clause value_exprs的值计算查询返回的每一行相对于其他行的等级。

V$RMAN_ENCRYPTION_ALGORITHMS显示支持的加密算法。 RMAN客户端使用它来验证用户请求的算法。此视图将列出当前版本的AES128，AES192和AES256加密算法。默认算法是AES128。
柱 数据类型 描述

ALGORITHM_ID

数

用于标识算法的编号

ALGORITHM_NAME

VARCHAR2（64）

算法名称（例如，AES128，AES192或AES256）

ALGORITHM_DESCRIPTION

VARCHAR2（64）

算法描述

IS_DEFAULT

VARCHAR2（3）

指示这是否是默认加密算法（ YES ）或否（ NO ）。此值由Oracle数据库设置，并且可能因每个版本而异（即，它不依赖于用户指定的RMAN配置）。

RESTORE_ONLY

VARCHAR2（3）

指示此算法是否可用于仅还原（ YES ）或不用（ NO ）。如果值为NO ，则还允许算法进行备份。此列可用于确定是否已弃用加密算法以进行备份。

CON_ID

数

数据所属容器的ID。可能的值包括：

  • 0 ：此值用于包含与整个CDB相关的数据的行。此值还用于非CDB中的行。

  • 1 ：此值用于包含仅与根有关的数据的行

  • n ：其中n是包含数据的行的适用容器ID
