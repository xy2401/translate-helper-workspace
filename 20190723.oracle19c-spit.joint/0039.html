CURSOR表达式

CURSOR表达式返回嵌套游标。这种表达形式相当于PL / SQL REF CURSOR ，可以作为REF CURSOR参数传递给函数。


说明cursor_expression.eps的说明

在计算游标表达式时，将隐式打开嵌套游标。例如，如果游标表达式出现在选择列表中，则将为查询提取的每一行打开嵌套游标。嵌套游标仅在以下情况下关闭：

  • 嵌套游标由用户显式关闭

  • 将重新执行父光标

  • 父光标已关闭

  • 父光标被取消

  • 在其中一个父游标上获取时出现错误（它作为清理的一部分关闭）

CURSOR表达式的限制

以下限制适用于CURSOR表达式：

  • 如果封闭语句不是SELECT语句，则嵌套游标只能作为过程的REF CURSOR参数出现。

  • 如果封闭语句是SELECT语句，则嵌套游标也可以出现在查询规范的最外面的选择列表中，或者出现在另一个嵌套游标的最外面的选择列表中。

  • 嵌套游标不能出现在视图中。

  • 您不能对嵌套游标执行BIND和EXECUTE操作。

以下示例显示了在查询的选择列表中使用CURSOR表达式：

SELECT department_name，CURSOR（SELECT salary，commission_pct FROM employees e WHERE e.department_id = d.department_id）FROM departments d ORDER BY department_name;

Oracle ASM动态卷管理器概述

Oracle ASM动态卷管理器（Oracle ADVM）为客户端提供卷管理服务和标准磁盘设备驱动程序接口。

文件系统和其他基于磁盘的应用程序将I / O请求发送到Oracle ADVM卷设备，就像向供应商操作系统上的其他存储设备发送I / O请求一样。

Oracle ADVM卷设备由Oracle ASM动态卷构成。可以在每个Oracle ASM磁盘组中配置一个或多个Oracle ADVM卷设备。Oracle ADVM驱动程序将针对Oracle ADVM卷设备的I / O请求映射到位于Oracle ASM磁盘组中的相应Oracle ASM动态卷和磁盘集中的块。Oracle ADVM卷设备导出Oracle ASM卷管理器功能，并确保在异常系统关闭，Oracle ASM实例故障或系统故障时卷镜像保持一致。

Oracle ADVM通过为分配为Oracle ADVM卷文件的Oracle ASM存储提供磁盘驱动程序接口来扩展Oracle ASM。您可以使用Oracle ADVM创建包含文件系统的虚拟磁盘。Oracle ADVM卷上包含的这些文件系统能够支持Oracle数据库文件之外的文件，例如可执行文件，报告文件，跟踪文件，警报日志和其他应用程序数据文件。由于Oracle ADVM卷实际上是Oracle ASM文件，因此它们需要与Oracle ASM文件相同的管理权限。

Oracle自动存储管理群集文件系统（Oracle ACFS）通过Oracle ADVM接口与Oracle ASM进行通信。通过添加Oracle ADVM，Oracle ASM成为用户数据的完整存储解决方案，可满足数据库和非数据库文件的需求。

要将卷添加到Oracle ASM磁盘组，必须将磁盘组属性COMPATIBLE.ASM和COMPATIBLE.ADVM设置为“ '11.2' 。

注意：

动态卷取代传统的设备分区。每个卷都是单独命名的，可以配置为单个文件系统。可以根据需要从Oracle ASM磁盘组存储创建Oracle ADVM卷，并根据需要动态调整大小。这些属性使Oracle ADVM卷比物理设备和相关的分区方案更灵活。

Oracle数据库网关概述

异构数据访问是一个影响很多公司的问题。许多公司运行几个不同的数据库系统。这些系统中的每一个都存储数据，并具有一组针对它运行的应用程序。在一个数据库系统中合并这些数据通常很难 - 很大程度上是因为许多针对一个数据库运行的应用程序可能没有与另一个数据库运行的等效项。在迁移到一个统一数据库系统变得可行之前，各种异构数据库系统必须进行互操作。

Oracle数据库网关提供了从Oracle环境透明地访问驻留在非Oracle系统中的数据的功能。这种透明性消除了应用程序开发人员定制其应用程序以访问来自不同非Oracle系统的数据的需要，从而减少了开发工作并提高了应用程序的移动性。可以使用Oracle和SQL Server的一致Oracle界面开发应用程序。

网关技术由两部分组成：具有连接到非Oracle系统的通用技术的组件，所有非Oracle系统都通用，称为异构服务，以及特定于非Oracle的组件网关连接的系统。异构服务与Oracle Database Gateway代理结合使用，可以从Oracle环境透明地访问非Oracle系统。

关于异构服务技术

异构服务提供了连接到非Oracle系统的通用技术。作为数据库的集成组件，异构服务可以利用数据库的功能，例如强大的SQL解析和分布式优化功能。

异构服务扩展了Oracle SQL引擎，以识别远程非Oracle系统的SQL和过程功能以及获取必要数据字典信息所需的映射。异构服务提供两种类型的转换：将Oracle SQL转换为非Oracle系统的正确方言的能力，以及以本地格式显示非Oracle系统的元数据的数据字典转换。对于没有可用转换的情况，可以使用异构服务的传递功能向非Oracle系统发出本机SQL。

异构服务还维护Oracle与远程非Oracle系统之间的事务协调，例如提供两阶段提交协议以确保分布式事务完整性，即使对于本身不支持两阶段提交的非Oracle系统也是如此。

透明数据加密

透明数据加密（TDE）使您可以加密数据，以便只有授权的收件人才能读取数据。

使用加密来保护可能不受保护的环境中的敏感数据，例如放置在发送到场外存储位置的备份媒体上的数据。您可以加密数据库表中的各个列，也可以加密整个表空间。

要使用透明数据加密，您无需修改应用程序。TDE使您的应用程序能够像以前一样无缝地继续工作。它在将数据写入磁盘时自动加密，然后在应用程序访问数据时自动解密数据。密钥管理是内置的，消除了管理和保护加密密钥的复杂任务。

Oracle数据编辑

Oracle Data Redaction使您可以使用多种编校类型来编辑（掩盖）列数据。

您可以执行的编辑类型如下：

  • 完全编辑。 您编辑列数据的所有内容。返回给查询用户的编辑值取决于列的数据类型。例如， NUMBER数据类型的列用零（ 0 ）编辑，字符数据类型用空格编辑。

  • 部分编辑。 您编辑一部分列数据。例如，您可以使用星号（*）编辑大部分社会安全号码，但最后4位数除外。

  • 常用表达。 您可以在完整和部分编校中使用正则表达式。这使您可以根据数据的搜索模式编辑数据。例如，您可以使用正则表达式来编辑数据中的特定电话号码或电子邮件地址。

  • 随机编辑。 呈现给查询用户的编辑数据在每次显示时都显示为随机生成的值，具体取决于列的数据类型。

  • 没有编辑。 此选项使您可以测试编校策略的内部操作，而不会影响对其上定义了策略的表的查询结果。您可以使用此选项在将编校策略定义应用于生产环境之前对其进行测试。

Data Redaction在运行时执行编校，即用户尝试查看数据的时刻。此功能非常适用于数据不断变化的动态生产系统。在对数据进行编辑时，Oracle数据库能够正常处理所有数据并保留后端参照完整性约束。数据编辑可以帮助您遵守行业法规，如支付卡行业数据安全标准（PCI DSS）和萨班斯 - 奥克斯利法案。
