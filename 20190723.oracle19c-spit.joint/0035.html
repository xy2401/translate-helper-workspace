录制响应文件

您可以在交互模式下使用Oracle Universal Installer（OUI）来记录响应文件，然后您可以编辑并使用它们来完成静默模式或响应文件模式安装。

通过单击“摘要”页面上的“ 保存响应文件” ，可以在安装期间将所有安装步骤保存到响应文件中。您可以稍后使用生成的响应文件进行静默安装。

记录响应文件时，您可以完成安装，也可以在OUI开始将软件设置到系统之前退出“摘要”页面上的安装程序。

如果在响应文件模式安装期间使用记录模式，则安装程序会将原始源响应文件中指定的变量值记录到新响应文件中。

注意：

OUI在记录响应文件时不保存密码。

要记录响应文件：

  1. 完成Oracle数据库客户端安装的完整预安装任务。

    当您运行安装程序以记录响应文件时，它会检查系统以验证它是否满足安装软件的要求。因此，Oracle建议您在完成安装时完成所有必需的安装前任务并记录响应文件。

  2. 确保Oracle软件所有者用户（通常是oracle ）有权创建或写入您在运行安装程序时指定的Oracle主路径。

  3. 在每个安装屏幕上，指定所需信息。

  4. 当OUI显示“摘要”屏幕时，请执行以下步骤：

    1. 单击保存响应文件 。在窗口中，指定新响应文件的文件名和位置。单击“保存”将您输入的响应写入响应文件。

    2. 单击“ 完成”继续安装。

      如果您不想继续安装，请单击“ 取消” 。安装停止，但保留记录的响应文件。

      注意：

      确保您的响应文件名具有.rsp后缀。

  5. 在另一个系统上使用已保存的响应文件之前，请编辑该文件并进行所需的更改。编辑时，请使用文件中的说明作为指导。

目的

RPAD返回expr1 ，右边填充到长度为n字符的expr2 ，根据需要复制多次。此函数对格式化查询的输出很有用。

expr1和expr2都可以是CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB任何数据类型。返回的字符串的VARCHAR2数据类型，如果expr1是字符数据类型， NVARCHAR2如果expr1是一个民族性的数据类型和LOB如果expr1是一个LOB数据类型。返回的字符串与expr1字符集相同。参数n必须是NUMBER整数或可以隐式转换为NUMBER整数的值。

expr1不能为null。如果未指定expr2 ，则默认为单个空白。如果expr1长于n ，则此函数返回expr1中适合n 。

参数n是终端屏幕上显示的返回值的总长度。在大多数字符集中，这也是返回值中的字符数。但是，在某些多字节字符集中，字符串的显示长度可能与字符串中的字符数不同。

1.2.4 Oracle Cluster Health Advisor简介

Oracle Cluster Health Advisor持续监视群集节点和Oracle RAC数据库的性能和可用性问题前兆，以便在问题变得严重之前提供问题的早期警告。

Oracle Cluster Health Advisor已集成到Oracle企业管理器云控制（EMCC）事件管理器中。

Oracle Cluster Health Advisor执行以下操作：

  • 检测节点和数据库性能问题

  • 提供预警警报和纠正措施

  • 支持现场校准以提高灵敏度

在Oracle Database 12c第2版（12.2.0.1）中，Oracle Cluster Health Advisor支持监视Oracle Real Application Clusters（Oracle RAC）的两个关键子系统：数据库实例和主机系统。Oracle Cluster Health Advisor确定并跟踪受监视系统的运行状况。它定期从受监控系统中采集各种关键测量数据。

已经对超过一百个数据库和集群节点问题进行了建模，并且已经确定了指示这些问题的发展或存在的特定操作系统和Oracle数据库度量标准。该信息用于构建基于目标系统的正常操作时段的训练的校准模型。

Oracle Cluster Health Advisor每分钟多次运行分析。Oracle Cluster Health Advisor根据默认模型估计观察输入的预期值。然后，Oracle Cluster Health Advisor根据观察值和预期值之间的差异对每个输入执行异常检测。如果与特定问题相关的足够输入异常，则Oracle Cluster Health Advisor会发出警告并生成即时针对性诊断和纠正措施。

Oracle Cluster Health Advisor模型是保守的，可以防止出现错误警告通知。但是，对于关键生产系统，默认配置可能不够灵敏。因此，Oracle Cluster Health Advisor提供现场模型校准功能，以使用实际生产工作负载数据构成其默认设置的基础，并提高节点和数据库模型的准确性和灵敏度。

Oracle Cluster Health Advisor在Grid Infrastructure Management Repository（GIMR）中存储分析结果，以及诊断信息，纠正措施和度量标准，以便以后进行分类。Oracle Cluster Health Advisor还使用Oracle Clusterware事件通知协议向Enterprise Manager Cloud Control发送警告消息。

您还可以使用Oracle Cluster Health Advisor诊断和分类过去的问题。您可以通过Oracle Enterprise Manager云控制（EMCC）事件管理器或命令行界面CHACTL指定过去的日期。通过在Grid Infrastructure Management Repository（GIMR）中配置Oracle Cluster Health Advisor表空间的保留设置，管理Oracle Cluster Health Advisor查看过去问题的功能。默认保留期为72小时。

从回滚段迁移到自动撤消模式

如果数据库版本早于Oracle Database 11g ，则必须将正在升级的数据库从使用回滚段（手动撤消管理）迁移到自动撤消管理。

自动撤消管理是默认的撤消空间管理模式。UNDO_MANAGEMENT初始化参数指定系统应使用的撤消空间管理模式：

  • 如果UNDO_MANAGEMENT设置为AUTO （或未设置UNDO_MANAGEMENT ），则数据库实例将以自动撤消管理模式启动。

    null UNDO_MANAGEMENT初始化参数默认为Oracle Database 11g第1版（11.1）及更高版本中的自动撤消管理模式。在早期版本中，默认为手动撤消管理模式。升级早期版本时请小心。

  • 如果UNDO_MANAGEMENT设置为MANUAL ，则撤消空间在外部分配为回滚段。

  1. 将UNDO_MANAGEMENT参数设置为UNDO_MANAGEMENT=MANUAL 。
  2. 再次启动实例并运行标准业务周期以获取代表性工作负载。评估工作负载，并计算自动撤消管理所需的撤消表空间的大小。
  3. 标准业务周期完成后，运行以下函数来收集撤消表空间大小，并帮助调整撤消表空间的大小。您需要SYSDBA权限才能运行此功能。
    DECLARE utbsiz_in_MB NUMBER; BEGIN utbsiz_in_MB：= DBMS_UNDO_ADV.RBU_MIGRATION;结束; /

    此函数运行PL / SQL过程，该过程根据系统中回滚段的配置和使用情况提供有关如何调整新undo表空间大小的信息。该函数直接返回大小调整信息。

  4. 创建所需大小的撤消表空间，并通过设置UNDO_MANAGEMENT=AUTO或删除参数来打开自动撤消管理。
  5. 对于Oracle RAC配置，请在所有实例上重复这些步骤。

录制响应文件

您可以在交互模式下使用Oracle Universal Installer（OUI）来记录响应文件，然后您可以编辑并使用它们来完成静默模式或响应文件模式安装。

通过单击“摘要”页面上的“ 保存响应文件” ，可以在安装期间将所有安装步骤保存到响应文件中。您可以稍后使用生成的响应文件进行静默安装。

记录响应文件时，您可以完成安装，也可以在OUI开始将软件设置到系统之前退出“摘要”页面上的安装程序。

如果在响应文件模式安装期间使用记录模式，则安装程序会将原始源响应文件中指定的变量值记录到新响应文件中。

注意：

OUI在记录响应文件时不保存密码。

要记录响应文件：

  1. 完成Oracle数据库客户端安装的完整预安装任务。

    当您运行安装程序以记录响应文件时，它会检查系统以验证它是否满足安装软件的要求。因此，Oracle建议您在完成安装时完成所有必需的安装前任务并记录响应文件。

  2. 确保Oracle软件所有者用户（通常是oracle ）有权创建或写入您在运行安装程序时指定的Oracle主路径。

  3. 在每个安装屏幕上，指定所需信息。

  4. 当OUI显示“摘要”屏幕时，请执行以下步骤：

    1. 单击保存响应文件 。在窗口中，指定新响应文件的文件名和位置。单击“保存”将您输入的响应写入响应文件。

    2. 单击“ 完成”继续安装。

      如果您不想继续安装，请单击“ 取消” 。安装停止，但保留记录的响应文件。

      注意：

      确保您的响应文件名具有.rsp后缀。

  5. 在另一个系统上使用已保存的响应文件之前，请编辑该文件并进行所需的更改。编辑时，请使用文件中的说明作为指导。

SQL标准

Oracle致力于遵守行业认可的标准，并积极参与SQL标准委员会。行业认可的委员会是美国国家标准协会（ANSI）和国际标准化组织（ISO），隶属于国际电工委员会（IEC）。ANSI和ISO / IEC都接受SQL作为关系数据库的标准语言。当这些组织同时发布新的SQL标准时，标准的名称符合组织使用的约定，但标准在技术上是相同的。

SQL如何工作

SQL的优势为所有类型的用户提供了好处，包括应用程序编程人员，数据库管理员，经理和最终用户。从技术上讲，SQL是一种数据子语言。SQL的目的是为关系数据库（如Oracle数据库）提供接口，所有SQL语句都是数据库的指令。在这个SQL中，它与C和BASIC等通用编程语言不同。 SQL的功能包括：

  • 它将数据集处理为组而不是单个单元。

  • 它提供数据的自动导航。

  • 它使用复杂而强大的语句，因此是独立的。流控制语句，例如begin-end，if-then-else，循环和异常条件处理，最初不是SQL和SQL标准的一部分，但它们现在可以在ISO / IEC 9075-4中找到 - 持久性存储模块（SQL / PSM）。Oracle SQL的PL / SQL扩展类似于PSM。

SQL允许您在逻辑级别处理数据。只有在想要操作数据时才需要关注实现细节。例如，要从表中检索一组行，请定义用于过滤行的条件。满足条件的所有行都可以在一个步骤中检索，并且可以作为一个单元传递给用户，另一个SQL语句或应用程序。您无需逐个处理行，也不必担心它们的物理存储或检索方式。所有SQL语句都使用优化器 ，它是Oracle数据库的一部分，用于确定访问指定数据的最有效方法。Oracle还提供了可用于使优化器更好地执行其工作的技术。

SQL为各种任务提供语句，包括：

  • 查询数据

  • 在表中插入，更新和删除行

  • 创建，替换，更改和删除对象

  • 控制对数据库及其对象的访问

  • 保证数据库的一致性和完整性

SQL以一种一致的语言统一所有前面的任务。

所有关系数据库的通用语言

所有主要的关系数据库管理系统都支持SQL，因此您可以将使用SQL获得的所有技能从一个数据库转移到另一个数据库。此外，所有用SQL编写的程序都是可移植的。它们通常可以通过很少的修改从一个数据库移动到另一个数据库。

Oracle ACFS的限制和指南

将这些主题作为存储计划的一部分进行查看，以便将Oracle ACFS用于单实例和群集配置。

  • Oracle自动存储管理群集文件系统（Oracle ACFS）提供通用文件系统。

  • 配置Oracle ASM时，只能使用Oracle ACFS。

  • 请注意以下有关在Oracle ACFS上放置Oracle数据库和Oracle Grid Infrastructure文件的一般准则和限制：

    • 您可以在Oracle ACFS上放置Oracle数据库二进制文件，数据文件和管理文件（例如，跟踪文件）。

    • Oracle ACFS不支持使用Oracle数据库数据文件，表空间文件，控制文件和重做日志进行加密。

    • 仅当数据库版本为Oracle Database 11g第2版或更新版本时，才能将Oracle数据库主目录置于Oracle ACFS上。您无法在Oracle ACFS上安装早期版本的Oracle数据库。

    • 对于Oracle Clusterware上的安装，您无法将Oracle Clusterware文件放在Oracle ACFS上。

  • Oracle Restart不支持基于root的Oracle Clusterware资源。因此，如果在Oracle重新启动配置上运行Oracle ACFS，则以下限制适用：

    • 从Oracle Database 18c开始，配置助手不允许在Oracle Restart配置中在Oracle ACFS上创建Oracle数据库主目录。

    • Oracle Restart不支持所有平台上的Oracle ACFS资源。

    • 从Oracle Database 12c开始，Oracle Restart配置不支持Oracle ACFS注册表。

    • 在Linux上，Oracle ACFS提供了一种自动机制，用于在系统重新启动和关闭时加载和卸载驱动程序以及装载和卸载Oracle ACFS文件系统。但是，Oracle ACFS在系统运行时不提供已装入文件系统的自动恢复。除Linux之外，Oracle ACFS不在其他操作系统上提供此自动机制。

    • Oracle Restart配置不支持在Oracle ACFS文件系统上创建Oracle数据文件。Oracle Grid Infrastructure支持在Oracle ACFS文件系统上创建Oracle数据文件以进行群集配置。

  • IBM AIX Workload Partitions（WPAR）不支持Oracle ACFS和Oracle ADVM。

默认情况下，为Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库安装Oracle Grid Infrastructure时，将自动在每个节点上配置Oracle Cluster Health Advisor。

Oracle Cluster Health Advisor不需要任何其他配置。Grid Infrastructure Management Repository（GIMR）中OCHAD守护程序用户的凭据是安全随机生成的，并存储在Oracle Grid Infrastructure凭据存储中。

当Oracle Cluster Health Advisor检测到Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库实例正在运行时，Oracle Cluster Health Advisor会自动开始监视群集节点。以Grid用户身份登录时使用CHACTL打开数据库监控。

要监视Oracle Real Application Clusters（Oracle RAC）环境：

  1. 要监视数据库，请运行以下命令：
    $ chactl monitor database -db db_unique_name

    Oracle Cluster Health Advisor使用默认模型监控Oracle Real Application Clusters（Oracle RAC）或Oracle RAC One Node数据库的所有实例。即使单实例Oracle数据库与Oracle Real Application Clusters（Oracle RAC）数据库共享同一群集，Oracle Cluster Health Advisor也无法监控单实例Oracle数据库。

    Oracle Cluster Health Advisor会在群集重新启动时保留数据库监控状态，因为Oracle Cluster Health Advisor会将状态信息存储在GIMR中。每个数据库实例都在Oracle Real Application Clusters（Oracle RAC）数据库节点上独立监视，并且在单个节点上运行多个数据库时。

  2. 要停止监视数据库，请运行以下命令：
    $ chactl unmonitor database -db db_unique_name

    Oracle Cluster Health Advisor停止监视指定数据库的所有实例。但是，Oracle Cluster Health Advisor在超出保留期限之前不会删除任何数据或问题。

  3. 要检查所有群集节点和数据库的监视状态，请运行以下命令：
    $ chactl status

    使用–verbose选项可以查看更多详细信息，例如用于节点和每个数据库的模型。

Oracle建议您在开始安装之前禁用Transparent HugePages。

透明HugePages内存与标准HugePages内存不同，因为内核khugepaged线程在运行时动态分配内存。标准HugePages内存在启动时预先分配，并且在运行时不会更改。

注意：

虽然在UEK2和更高版本的UEK内核上禁用了透明HugePages，但默认情况下可以在Linux系统上启用Transparent HugePages。

默认情况下，透明HugePages内存使用SUSE 12，Red Hat Enterprise Linux 7和Oracle Linux 7以及早期版本的Oracle Linux和Unbreakable Enterprise Kernel 2（UEK2）内核。

透明HugePages可能会在运行时导致内存分配延迟。为避免性能问题，Oracle建议您在所有Oracle数据库服务器上禁用透明HugePages。Oracle建议您使用标准HugePages来提高性能。

关于安装期间的字符集选择

在创建数据库之前，请确定要使用的字符集。

创建数据库后，更改其字符集通常在时间和资源方面非常昂贵。此类操作可能需要通过导出整个数据库并将其导回来转换所有字符数据。因此，在安装时仔细选择数据库字符集非常重要。

Oracle数据库使用以下字符集：

  • 存储在SQL字符数据类型（ CHAR ， VARCHAR2 ， CLOB和LONG ）中的数据。

  • 表名，列名和PL / SQL变量等标识符。

  • 存储的SQL和PL / SQL源代码，包括嵌入在此代码中的文本文字。

从Oracle Database 12c第2版（12.2）开始，从通用/事务处理或数据仓库模板创建的数据库的缺省数据库字符集是Unicode AL32UTF8 。

Unicode是通用字符集，支持世界上大多数当前使用的语言。它还支持许多历史脚本（字母表）。Unicode是许多技术的本机编码，包括Java，XML，XHTML，ECMAScript和LDAP。 Unicode非常适合支持互联网和全球经济的数据库。

由于AL32UTF8是一个多字节字符集，因此与单字节数据库字符集（如WE8ISO8859P1或WE8MSWIN1252相比，对字符数据的数据库操作可能会稍微慢WE8MSWIN1252 。与支持该语言的传统字符集相比， AL32UTF8中使用ASCII AL32UTF8集以外的字符的大多数语言的文本的存储空间要求更高。仅当存储在CLOB （字符大对象）列中时，英语数据可能需要更多空间。非字符数据类型（如NUMBER或DATE ）的存储不依赖于字符集。Unicode的普遍性和灵活性通常超过了这些额外的成本。

仅当数据库需要支持单组语言时才考虑使用旧版字符集，并且使用旧版字符集对于满足兼容性，存储或性能要求至关重要。在这种情况下要选择的数据库字符集是连接到此数据库的大多数客户端的字符集。

多租户容器数据库（CDB）的数据库字符集确定以后可以插入哪些数据库。确保为CDB选择的字符集与要插入此CDB的数据库的数据库字符集兼容。如果使用Unicode AL32UTF8作为CDB字符集，则可以在Oracle数据库支持的任何数据库字符集中插入可插拔数据库（PDB）（基于EBCDIC的字符集除外）。

12.11改变

句法

C [HANGE] sepchar 旧 [ sepchar [ new [ sepchar ]]]

更改缓冲区中当前行的第一次出现的指定文本。

条款

sepchar

表示任何非字母数字字符，例如“/”或“！”。使用未出现在旧版或新版中的sepchar 。 

旧

表示您要更改的文本。CHANGE忽略搜索旧的情况 。例如，

CHANGE / aq / aw

找到第一次出现的“aq”，“AQ”，“aQ”或“Aq”并将其更改为“aw”。SQL * Plus完全按照您的指定插入新文本。

如果old以“...”为前缀，则它匹配所有内容，包括第一次出现的旧内容 。如果它以“...”为后缀，则它匹配第一次出现的old以及该行后面的所有内容。如果它包含一个嵌入的“...”，它匹配一切从老通过老以下部分的前半部分。 

新

表示您希望替换旧文本的文本。如果省略new和（可选）第二个和第三个sepchars，CHANGE将从缓冲区的当前行中删除old 。

用法

改变变化缓冲区当前行上第一次出现的指定文本到新指定的文本。当前行在LIST输出中标有星号（*）。

您还可以使用CHANGE修改生成Oracle数据库错误的缓冲区中的行。SQL * Plus将缓冲区的当前行设置为包含错误的行，以便您可以进行修改。

要重新输入整行，可以键入行号，后跟行的新内容。如果指定的行号大于缓冲区中的行数并使用文本后跟数字，则SQL * Plus会在缓冲区末尾的新行中添加文本。如果为行号指定零（“0”）并在文本后跟零，则SQL * Plus会在缓冲区的开头插入该行（该行变为第1行）。

例子

输入3，以便缓冲区的当前行包含以下文本：

3
3 * WHERE JOB_ID在（'CLERK'，'SA_MAN'）

输入以下命令：

更改/秘书/ SH_CLERK /

缓冲区中的文本更改如下：

3 * WHERE JOB_ID在（'SH_CLERK'，'SA_MAN'）

或输入以下命令：

改变''CLERK'，....../ 'SH_CLERK'/

原始行更改为

3 * WHERE JOB_ID在（'SH_CLERK'）

或输入以下命令：

改变/（...）/（ 'SA_MAN'）/

原始行更改为

3 * WHERE JOB_ID在（'SA_MAN'）

您可以使用行号替换整行的内容。这个条目

3 WHERE JOB_ID在（'SH_CLERK'）

导致缓冲区的第二行被替换为

JOB_ID在哪里（'SH_CLERK'）

请注意，输入行号后跟字符串将替换该行，而不管行号后面的文本是什么。例如，

2更改/旧/新/

将更改缓冲区的第二行

2 * C / OLD / NEW /

设置客户端连接的语言和区域设置首选项

根据您的区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。

您必须根据区域设置首选项和I / O设备字符集配置连接到Oracle数据库的客户端应用程序。如果应用程序没有自己的特定方法来配置区域设置首选项，则用于配置Oracle数据库客户机连接的方法取决于用于连接到数据库的访问API。在为应用程序配置区域设置首选项之前，请检查应用程序文档。

对于使用Oracle调用接口（OCI）连接到Oracle数据库的应用程序，请使用NLS_LANG和其他名称以NLS_开头的客户端设置来设置Oracle数据库会话的区域设置约定和客户端字符集。正确设置NLS_LANG值的字符集部分非常重要。您设置的字符集必须与I / O设备使用的字符集相对应，在Microsoft Windows的情况下，可以是ANSI代码页（用于GUI应用程序），例如WE8MSWIN1252，或OEM代码页（用于控制台模式）应用程序），例如US8PC437。通过这样做，OCI API将收到有关它从应用程序接收的数据字符集的通知。然后，OCI可以正确地将此数据转换为数据库字符集和从数

NLS_LANG和其他NLS设置可以指定为环境变量或Windows注册表设置。环境变量值优先于Registry值。

在Universal Windows上创建新的Oracle主目录时，Oracle Universal Installer会在注册表中为NLS_LANG设置设置默认值。NLS_LANG值基于Windows用户界面的语言，Windows用户界面是Windows菜单项和对话框标签的语言。

警告：

无法正确设置客户端字符集可能会导致数据丢失。

使用Oracle JDBC连接到Oracle数据库的Java应用程序不使用NLS_LANG。相反，Oracle JDBC将运行应用程序的Java VM的缺省语言环境映射到Oracle数据库语言和区域设置。然后，Oracle JDBC使用这些设置配置连接的数据库会话。由于Java在Unicode内部工作，因此客户端字符集始终设置为Unicode。除非应用程序显式更改它，否则将根据运行Java VM的用户操作系统的区域设置来设置Java VM的缺省语言环境。有关配置Java VM缺省语言环境的信息，请查看Java VM文档。

录制响应文件

您可以在交互模式下使用OUI来记录响应文件，然后您可以编辑并使用它们来完成静默模式或响应文件模式安装。此方法对于高级或纯软件安装非常有用。

通过单击“摘要”页面上的“ 保存响应文件” ，可以在安装期间将所有安装步骤保存到响应文件中。您可以稍后使用生成的响应文件进行静默安装。

记录响应文件时，您可以完成安装，也可以在OUI开始将软件设置到系统之前退出“摘要”页面上的安装程序。

如果在响应文件模式安装期间使用记录模式，则安装程序会将原始源响应文件中指定的变量值记录到新响应文件中。

注意：

OUI在记录响应文件时不保存密码。

要记录响应文件：

  1. 完成标准安装的预安装任务。

    当您运行安装程序以记录响应文件时，它会检查系统以验证它是否满足安装软件的要求。因此，Oracle建议您在完成安装时完成所有必需的安装前任务并记录响应文件。

  2. 确保Oracle软件所有者用户（通常是oracle ）有权创建或写入运行安装程序时指定的Oracle主路径。

  3. 在每个安装屏幕上，指定所需信息。

  4. 当安装程序显示“摘要”屏幕时，请执行以下步骤：

    1. 单击保存响应文件 。在窗口中，指定新响应文件的文件名和位置。单击“ 保存”将您输入的响应写入响应文件。

    2. 单击“ 完成”继续安装。

      如果您不想继续安装，请单击“ 取消” 。安装停止，但保留记录的响应文件。

    注意：

    确保您的响应文件名具有.rsp后缀。

  5. 在另一个系统上使用已保存的响应文件之前，请编辑该文件并进行所需的更改。编辑时，请使用文件中的说明作为指导。
