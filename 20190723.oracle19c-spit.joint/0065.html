表之间用于一对多关系的连接通过嵌套列表示。

Oracle数据挖掘需要一个单记录格式的案例表，每个记录在一个单独的行中。如果您的部分或全部数据采用多记录格式，每条记录包含多行，该怎么办？如果您希望一个属性表示一系列或一组值，例如学生的考试成绩或客户购买的产品，该怎么办？

  • 获取在线或离线设置的表空间中的所有表空间

  • 开始或结束备份

  • 使表空间中的所有表空间都设置为只读或读写

  • 设置表空间中所有表空间的默认日志记录模式设置为LOGGING或NOLOGGING

  • 将所有表空间放在以强制日志记录模式设置的表空间中，或将它们置于强制日志记录模式之外

  • 调整表空间集的所有数据文件的大小

  • 启用或禁用表空间集的所有数据文件的自动扩展

在将表空间设置为只读之前，必须满足以下条件：

  • 表空间集中的表空间必须在线。

  • 表空间集不得包含任何活动的回滚段。此外，由于无法访问只读表空间集的回滚段，因此Oracle建议您在将表空间设置为只读之前删除回滚段。

  • 表空间集不得涉及开放备份，因为备份结束时会更新表空间集中所有数据文件的头文件。

通过检查嵌入式SQL语句和PL / SQL块的语法和语义，Oracle预编译器可帮助您快速查找和修复编码错误。这个附录说明如何使用SQLCHECK选项来控制检查的类型和范围。

D.1什么是句法和语义检查？

语法规则指定语言元素如何排序以形成有效语句。因此， 语法检查会验证关键字，对象名称，运算符，分隔符等是否正确放置在SQL语句中。例如，以下嵌入式SQL语句包含语法错误：

- 拼写错误的关键字WHERE EXEC SQL DELETE FROM EMP WERE DEPTNO = 20; - 列名称缺少括号COMM和SAL EXEC SQL INSERT INTO EMP COMM，SAL VALUES（NULL，1500）;

语义规则指定如何进行有效的外部引用。因此， 语义检查验证对数据库对象和主机变量的引用是否有效以及主机变量数据类型是否正确。例如，以下嵌入式SQL语句包含语义错误：

- 不存在的表，EMPP EXEC SQL DELETE FROM EMPP WHERE DEPTNO = 20; - 未声明的主机变量，emp_name EXEC SQL SELECT * FROM EMP WHERE ENAME =：emp_name;

SQL语法和语义的规则在Oracle数据库SQL语言参考中定义。

D.2关于控制检查的类型和范围

您可以通过指定来控制检查的类型和范围命令行上的SQLCHECK选项。使用SQLCHECK ，检查的类型可以是语法，语义或两者。检查的范围可以包括数据操作语句和PL / SQL块。但是， SQLCHECK无法检查动态SQL语句，因为它们在运行时才完全定义。

您可以为SQLCHECK指定以下值：

  • 语义| FULL

  • 语法|有限公司| NONE

SEMANTICS和FULL的值是等价的， SYNTAX和LIMITED的值也是等价的。默认值为SYNTAX 。

D.3关于指定SQLCHECK = SEMANTICS

当SQLCHECK=SEMANTICS ，预编译器检查以下语法和语义：

  • INSERT和UPDATE等数据操作语句

  • PL / SQL块

但是，预编译器仅检查远程数据操作语句（使用AT db_name子句的语句）的语法。

预编译器从嵌入式DECLARE TABLE语句中获取语义检查的信息，或者，如果指定选项USERID ，则通过连接到Oracle并访问数据字典来获取。如果在DECLARE TABLE语句中定义了数据操作语句或PL / SQL块中引用的每个表，则无需连接到Oracle。

如果连接到Oracle但在数据字典中找不到某些信息，则必须使用DECLARE TABLE语句提供缺少的信息。如果数据字典定义发生冲突，则DECLARE TABLE定义将覆盖它。

在检查数据操作语句时，预编译器使用Oracle数据库SQL语言参考中的Oracle数据库版本7语法规则集，但使用更严格的语义规则集。因此，当SQLCHECK=SEMANTICS时，为早期版本的Oracle编写的现有应用程序可能无法成功预编译。

预编译新程序时指定SQLCHECK=SEMANTICS 。如果在主机程序中嵌入PL / SQL块，则必须指定SQLCHECK=SEMANTICS 。

默认情况下， PARALLEL_SERVERS_TARGET设置为低于系统允许的最大并行服务器进程数（ PARALLEL_MAX_SERVERS ），以确保每个并行语句将获得所需的所有并行服务器资源，并防止系统使用并行服务器进程过载。

在实例上以默认并行度运行的并发并行用户数取决于实例的内存初始化参数设置。例如，如果设置了MEMORY_TARGET或SGA_TARGET初始化参数，那么concurrent_parallel_users的数量= 4 。如果MEMORY_TARGET设置MEMORY_TARGET或SGA_TARGET ，则检查PGA_AGGREGATE_TARGET 。如果为PGA_AGGREGATE_TARGET设置了值，则concurrent_parallel_users = 2 。如果没有为PGA_AGGREGATE_TARGET设置值，则concurrent_parallel_users = 1 。

请注意，即使已激活语句排队，所有串行（非并行）语句也将立即执行。

默认情况下，所有PDB查询首先在PDB级别通过PDB的PARALLEL_SERVERS_TARGET值进行排队，然后在CDB级别通过CDB的PARALLEL_SERVERS_TARGET值进行排队。如果CDB中的并行服务器已耗尽，则此默认行为可防止PDB中的任何SQL语句降级。

默认情况下，在CDB级别启用并行语句排队，因为CDB具有PARALLEL_SERVERS_TARGET的默认值。您可以使用ALTER SYSTEM将CDALL级别的PARALLEL_SERVERS_TARGET设置为0来禁用CDB级别的并行语句排队。

PDB的PARALLEL_SERVERS_TARGET的默认值是使用上表中的计算和PDB的CPU_COUNT值确定的。

注意：

PDB可以为并行执行服务器设置下限，而不是CDB资源计划中指定的限制。在PDB中设置PARALLEL_SERVERS_TARGET初始化参数，并为CDB资源计划中的PDB指定并行执行服务器限制时，将强制执行这两个限制中的较低者。例如，假设在CDB根目录中将PARALLEL_SERVERS_TARGET初始化参数设置为100。还假设hrpdb将其PARALLEL_SERVERS_TARGET初始化参数设置为50，并且CDB计划具有hrpdb的指令，并将parallel_server_limit设置为70％。在这种情况下，hrpdb的并行执行服务器的限制为50，因为50低于hrpdb的CDB资源计划限制70。

查找和重置使用10G密码版本的用户密码

为了更好的安全性，请查找和重置使用10G密码版本的用户帐户的密码，以便他们使用更安全的密码版本。

查找当前用户的所有密码版本

您可以查询DBA_USERS数据字典视图，以查找为用户帐户配置的所有密码版本的列表。

例如：

来自DBA_USERS的SELECT USERNAME，PASSWORD_VERSIONS; USERNAME PASSWORD_VERSIONS ------------------------------ ----------------- JONES 10G 11G 12C ADAMS 10G 11G CLARK 10G 11G PRESTON 11G BLAKE 10G

PASSWORD_VERSIONS列显示该帐户存在的密码版本列表。10G是指早期不区分大小写的Oracle密码版本， 11G是指基于SHA-1的密码版本， 12C是指基于SHA-2的SHA-512密码版本。

  • 用户jones ：当SQLNET.ALLOWED_LOGON_VERSION_SERVER参数设置为8时，在Oracle Database 12 c版本12.1中重置了此用户的密码。这样就可以创建所有三个密码版本。

  • 用户adams和clark ：这些帐户的密码最初是在Oracle Database 10g中创建的，然后在Oracle Database 11g中重置。Oracle Database 11g软件当时使用的默认SQLNET.ALLOWED_LOGON_VERSION设置为8 。由于默认情况下启用了大小写不敏感，因此它们的密码现在区分大小写， preston的密码也是如此。

  • 用户preston ：此帐户是从以独占模式（ SQLNET.ALLOWED_LOGON_VERSION = 12 ）运行的Oracle Database 11 g数据库导入的。

  • 用户blake ：此帐户仍使用Oracle Database 10 g密码版本。在此阶段，阻止用户blake登录。

重置使用10G密码版本的用户密码

为了更好的安全性，请从所有用户的帐户中删除10G密码版本。在以下过程中，要重置具有10G密码版本的用户的密码，必须暂时放松SQLNET.ALLOWED_LOGON_VERSION_SERVER设置，该设置在允许登录之前控制客户端所需的能力级别。放宽设置使这些用户能够登录并更改其密码，从而生成除10G密码版本之外的新密码版本。之后，您可以将数据库设置为使用独占模式，并确保客户端具有O5L_NP功能。然后用户可以再次重置他们的密码，以便他们的密码版本不再包含10G ，但只有更安全的11G和12C密码版本。

  1. 查询DBA_USERS视图以查找仅使用10G密码版本的用户。
    从DBA_USERS WHERE中选择USERNAME（PASSWORD_VERSIONS ='10G'或PASSWORD_VERSIONS ='10G HTTP'）和USERNAME <>'ANONYMOUS';
  2. 配置数据库，使其不以独占模式运行，如下所示：
    1. 编辑sqlnet.ora文件中的SQLNET.ALLOWED_LOGON_VERSION_SERVER设置，使其比默认值更宽松。例如：
      SQLNET.ALLOWED_LOGON_VERSION_SERVER = 11
    2. 重启数据库。
  3. 使您在查询DBA_USERS视图时找到的用户过期，以查找仅使用10G密码版本的用户。

    您必须使仅具有10G密码版本且没有11G或12C密码版本中的一个或两个的用户到期。

    例如：

    ALTER USER用户username PASSWORD EXPIRE;
  4. 询问您过期密码的用户是否登录。

    当用户登录时，系统会提示他们更改密码。除10G密码版本外，数据库还为其帐户生成缺少的11G和12C密码版本。10G密码版本继续存在，因为数据库正在以许可模式运行。

  5. 确保用户连接的客户端软件具有O5L_NP功能。

    所有Oracle Database 11.2.0.3及更高版本的客户端都具有O5L_NP功能。如果您有早期的Oracle数据库客户端，则必须安装CPUOct2012补丁。

  6. 在所有客户端都具有O5L_NP功能后，将服务器的安全性设置回“独占模式”，如下所示：
    1. 从实例初始化文件中删除SEC_CASE_SENSITIVE_LOGON参数设置，或将SEC_CASE_SENSITIVE_LOGON设置为TRUE 。
      SEC_CASE_SENSITIVE_LOGON = TRUE
    2. 卸下SQLNET.ALLOWED_LOGON_VERSION_SERVER从服务器参数sqlnet.ora文件，或设置的值SQLNET.ALLOWED_LOGON_VERSION_SERVER在服务器sqlnet.ora文件返回到12 ，将其设置为独占模式。
      SQLNET.ALLOWED_LOGON_VERSION_SERVER = 12
    3. 重启数据库。
  7. 找到仍具有10G密码版本的帐户。
    从DBA_USERS中选择USERNAME，其中PASSWORD_VERSIONS类似'％10G％'和USERNAME <>'ANONYMOUS';
  8. 使仍具有10G密码版本的帐户过期。

目的

此声明将创建一个新版本作为现有版本的子版本。版本可以在数据库中具有两个或更多版本的相同可编辑对象。创建版本时，它会立即继承其父版本的所有可编辑对象。以下对象类型是可编辑的：

  • 代名词

  • 视图

  • 功能

  • 程序

  • 包装（规格和正文）

  • 类型（规格和正文）

  • 图书馆

  • 触发

可编辑对象是启用版本的模式中的上述可编辑对象类型之一的对象。在数据库中拥有这些对象的多个版本的能力极大地促进了在线应用程序升级。

上面未列出的所有数据库对象类型都不可编辑。对数据库中的所有版本都可以立即看到对不可编辑的对象类型的更改。

在一个简单的CASE表达式中，Oracle数据库搜索第一个WHEN ...THEN对， expr等于comparison_expr并返回return_expr 。如果没有的WHEN ...THEN对符合这个条件，并且存在ELSE子句，然后Oracle返回else_expr 。否则，Oracle返回null。

在搜索的CASE表达式中，Oracle从左向右搜索，直到找到condition为true的情况，然后返回return_expr 。如果发现没有condition为真，并且存在ELSE子句，则Oracle返回else_expr 。否则，Oracle返回null。

Oracle数据库使用短路评估 。对于简单的CASE表达式，数据库仅在将每个comparison_expr值与expr进行comparison_expr之前对其进行评估，而不是在将任何comparison_expr值与expr进行comparison_expr之前对其进行评估。因此，如果先前的comparison_expr等于expr ，Oracle从不评估comparison_expr 。对于搜索的CASE表达式，数据库会评估每个condition以确定它是否为真，并且如果先前condition为真，则永远不会评估condition 。

对于简单的CASE表达式， expr和所有comparison_expr值必须具有相同的数据类型（ CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 ， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE ），或者必须都具有数字数据类型。如果所有表达式都具有数字数据类型，则Oracle确定具有最高数字优先级的参数，将其余参数隐式转换为该数据类型，并返回该数据类型。

对于简单和搜索的CASE表达式，所有return_expr必须具有相同的数据类型（ CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 ， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE ），或者必须都具有数字数据类型。如果所有返回表达式都具有数字数据类型，则Oracle确定具有最高数字优先级的参数，将其余参数隐式转换为该数据类型，并返回该数据类型。

CASE表达式中的最大参数数为65535。所有表达式都计入此限制，包括简单CASE表达式的初始表达式和可选的ELSE表达式。每个WHEN ......THEN对是两个参数。为避免超出此限制，可以嵌套CASE表达式，以便return_expr本身是CASE表达式。

如果比较的参数具有字符数据类型（ CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 ），则由简单CASE表达式执行的比较是排序规则敏感的。核对确定规则确定要使用的核对。
