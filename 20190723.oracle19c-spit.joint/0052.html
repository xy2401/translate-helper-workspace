FIRST和LAST是非常相似的功能。两者都是聚合和分析函数，它们对一组行中的一组值进行操作，这些行根据给定的排序规范排名为FIRST或LAST 。如果只有一行排名为FIRST或LAST ，则聚合只对一个元素进行操作。

如果省略OVER子句，则FIRST和LAST函数将被视为聚合函数。您可以通过指定OVER子句将这些函数用作分析函数。query_partition_clause是OVER子句中唯一对这些函数有效的部分。如果包含OVER子句但省略query_partition_clause ，则该函数将被视为分析函数，但为分析定义的窗口是整个表。

6.6成本敏感的决策

成本是用户指定的数字，偏向分类。该算法使用正数来惩罚比较便宜的结果更昂贵的结果。数字越大表示成本越高。

该算法使用负数来支持比不太有利的结果更有益的结果。较低的负数表示较高的收益。

所有分类算法都可以使用评分成本。您可以在成本矩阵表中指定成本，也可以在评分时指定内联成本。如果您指定内联成本且模型也具有关联的成本矩阵，则仅使用内联成本。PREDICTION ， PREDICTION_SET和PREDICTION_COST函数支持成本。

只有决策树算法可以使用成本来偏向模型构建。如果要创建包含成本的决策树模型，请创建成本矩阵表，并在模型的CLAS_COST_TABLE_NAME设置中提供其名称。如果在构建模型时指定成本，则在评分时使用用于创建模型的成本矩阵。如果要使用不同的成本矩阵表进行评分，请先删除现有的成本矩阵表，然后添加新的成本矩阵表。

8.2从数据库触发器调用Java

一个数据库触发器是与特定表或视图关联的存储程序。只要数据操作语言（DML）操作影响表或视图，Oracle数据库就会自动运行触发器。

发生触发事件时，触发器将运行PL / SQL块或CALL 声明执行操作。语句触发器运行一次，之前或之后触发事件。对于受触发事件影响的每一行，行触发器运行一次。

在数据库触发器中，您可以使用new和old相关名称来引用更改行的新旧值。在trigger-action块或CALL语句中，列名必须以:new或:old为前缀。

TO_BINARY_DOUBLE

句法


to_binary_double.eps插图的描述

目的

TO_BINARY_DOUBLE将expr转换为双精度浮点数。

  • expr可以是任何计算结果为CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2类型的字符串的表达式， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE类型的数值，或null。如果expr是BINARY_DOUBLE ，则该函数返回expr 。如果expr求值为null，则该函数返回null。否则，该函数将expr转换为BINARY_DOUBLE值。

  • 可选的DEFAULT return_value ON CONVERSION ERROR子句允许您指定此函数返回的值，如果在将expr转换为BINARY_DOUBLE时发生错误。如果在计算expr时发生错误，则此子句无效。return_value可以是表达式或绑定变量，并且必须求值为CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2类型的字符串， NUMBER ， BINARY_FLOAT或BINARY_DOUBLE类型的数值，或null。该函数将return_value到BINARY_DOUBLE在其转换以同样的方式expr到BINARY_DOUBLE 。如果return_value无法转换为BINARY_DOUBLE ，则该函数返回错误。

  • 可选的' fmt '和' nlsparam '参数与TO_NUMBER函数的用途相同。如果指定这些参数，那么expr和return_value （如果指定）必须都是字符串或null。如果其中一个是字符串，则该函数使用fmt和nlsparam参数将字符串转换为BINARY_DOUBLE值。

如果expr或return_value计算为以下字符串，则该函数将按如下方式转换它们：

  • 不区分大小写的字符串' INF '将转换为正无穷大。

  • 不区分大小写的字符串' - INF '将转换为否定标识。

  • 不区分大小写的字符串' NaN '将转换为NaN （不是数字）。

您不能在字符串expr使用浮点数字格式元素（ F ， f ， D或d ）。

从字符串或NUMBER到BINARY_DOUBLE转换可能不准确，因为NUMBER和字符类型使用小数精度来表示数值，而BINARY_DOUBLE使用二进制精度。

从BINARY_FLOAT到BINARY_DOUBLE转换是准确的。

  • 重新编译所有无效对象
    Oracle建议您在安装，修补或升级数据库后运行utlrp.sql脚本，以识别和重新编译无效对象。

  • 升级Oracle数据库后运行OPatch命令
    升级Oracle数据库后，必须从新的Oracle主目录运行OPatch命令。
  • 升级Oracle数据库后，将oratab和脚本设置为指向新的Oracle位置
    您必须将脚本设置为指向新的Oracle主目录位置。
  • 检查PL / SQL包和依赖过程
    在早期版本的Oracle数据库中安装的软件包可能在新版本中不可用，这会影响应用程序。

    从Oracle Database 12 c第2版（12.2）及更高版本开始，您必须手动升级依赖于Oracle维护类型的用户表。
  • 启用新的扩展数据类型功能
    使系统能够利用新的扩展数据类型需要特定的升级操作。
  • 调整并行执行服务器的最小值和最大值
    根据您的环境，您可以减少PARALLEL_MIN_SERVERS参数的默认设置。
  • 关于升级Oracle数据库后的恢复目录升级
    如果使用的恢复目录模式版本早于RMAN客户端所需的版本，则必须对其进行升级。

    如果升级前信息工具指示您在完成数据库升级后升级时区文件，则使用DBMS_DST PL / SQL程序包升级时区文件。

    如果使用DBMS_STATS.CREATE_STAT_TABLE过程创建了统计表，则通过运行DBMS_STATS.UPGRADE_STAT_TABLE升级这些表。
  • 升级Oracle数据库后升级经过外部验证的SSL用户
    如果您是从Oracle9需要升级我第2版（9.2）或Oracle数据库10g第1版（10.1），并且正在使用外部认证的SSL的用户，那么你必须运行SSL外部用户转换（ extusrupgrade ）脚本来升级的用户。

    Oracle数据库配置助手（DBCA）不会对Oracle数据库12c中配置的Oracle XML DB港口和以后升级版本使用摘要身份验证。
  • 升级Oracle数据库后安装Oracle文本提供的知识库
    Oracle数据库升级后，必须重新生成Oracle Text提供的知识库的所有用户扩展。
  • 升级Oracle数据库后更新Oracle Application Express配置
    Oracle Application Express会影响升级过程，具体取决于Oracle Application Express版本和数据库安装类型。
  • 配置外部网络服务的访问控制列表（ACL）
    Oracle数据库12c与更高版本包括细粒度访问控制的UTL_TCP ， UTL_SMTP ， UTL_MAIL ， UTL_HTTP ，或UTL_INADDR包。
  • 检查SQLNET.ALLOWED_LOGON_VERSION参数行为
    从版本10g之前的客户端连接到Oracle数据库失败，错误ORA-28040: No matching authentication protocol 。

Oracle Fleet Patching and Provisioning（Oracle FPP）是一种用于配置和维护Oracle主目录的软件生命周期管理方法。Oracle Fleet Patching and Provisioning支持大规模部署和维护数据库，集群和用户定义的软件类型的标准操作环境。

  • Oracle Fleet补丁和配置客户端

使用DBMS_FILE_TRANSFER包在平台之间转换数据

您可以使用GET_FILE或PUT_FILE的议事DBMS_FILE_TRANSFER包中的数据文件传输过程中的平台之间的数据转换。

当您使用这些过程之一在源平台和目标平台之间移动数据文件时，每个数据文件中的每个块都将转换为目标平台的字节顺序。

本节使用一个示例来描述如何使用DBMS_FILE_TRANSFER包将数据文件转换为其他平台。该示例做出以下假设：

  • GET_FILE过程将传输数据文件。

  • mytable.342.123456789数据文件正在转移到另一个平台。

  • 源平台的字节顺序与目标平台的字节顺序不同。

  • 源数据库的全局名称是dbsa.example.com 。

  • 源数据库和目标数据库都使用Oracle自动存储管理（Oracle ASM）。

注意：

您还可以使用DBMS_FILE_TRANSFER包在具有相同字节顺序的平台之间传输数据文件。

完成以下步骤以通过使用GET_FILE过程传输数据文件来转换数据文件：

  1. 使用SQL * Plus以可以创建目录对象的管理用户身份连接到源数据库。
  2. 创建目录对象以存储要传输到目标数据库的数据文件。

    例如，要为+data/dbsa/datafile目录创建名为sales_dir_source的目录对象，请执行以下SQL语句：

    创建或替换目录sales_dir_source AS'+ data / dbsa / datafile';

    创建目录对象时，指定的文件系统目录必须存在。

  3. 使用SQL * Plus作为管理用户连接到目标数据库，管理用户可以创建数据库链接，创建目录对象，并在DBMS_FILE_TRANSFER包中运行这些过程。
  4. 创建从目标数据库到源数据库的数据库链接。

    源数据库中的已连接用户必须对您在步骤2中创建的目录对象具有读取权限。

  5. 创建目录对象以存储要从源数据库传输的数据文件。

    将在DBMS_FILE_TRANSFER程序包中运行该过程的本地数据库用户必须对目录对象具有写权限。

    例如，要为+data/dbsb/datafile目录创建名为sales_dir_target的目录对象，请运行以下SQL语句：

    创建或替换目录sales_dir_target AS'+ data / dbsb / datafile';
  6. 运行DBMS_FILE_TRANSFER包中的GET_FILE过程以传输数据文件。

    例如，运行以下过程以使用您在步骤4中创建的数据库链接将mytable.342.123456789数据文件从源数据库传输到目标数据库：

    BEGIN DBMS_FILE_TRANSFER.GET_FILE（source_directory_object =>'sales_dir_source'，source_file_name =>'mytable.342.123456789'，source_database =>'dbsa.example.com'，destination_directory_object =>'sales_dir_target'，destination_file_name =>'mytable'）;结束; /

注意：

在此示例中，目标数据文件名是mytable 。Oracle ASM不允许在GET_FILE过程的destination_file_name参数中使用完全限定的文件名表单。

9.3.1运行时连接负载平衡概述

在Oracle Real Application Clusters环境中，连接可以属于提供相关服务的任何实例。在最好的情况下，所有实例都表现同样好，并且从缓存中随机检索连接是合适的。但是，当一个实例比其他实例表现更好时，连接的随机选择效率低下。运行时连接负载平衡功能可以将工作请求路由到提供最佳性能的实例，从而最大限度地减少重新定位工作的需要。

UCP JDBC连接池利用Oracle RAC数据库提供的负载平衡功能。运行时连接负载平衡需要使用Oracle JDBC驱动程序和Oracle RAC数据库。

也可以看看：

“Oracle Real Application Clusters管理和部署指南”

在以下情况下，运行时连接负载平衡很有用：

  • 传统的工作负载平衡不是最佳的

  • 必须路由请求以优化使用群集数据库中的资源

  • 群集内的容量不同，预计会随时间而变化

  • 需要避免将工作发送到慢速，挂起和死区节点

UCP使用Oracle RAC负载平衡咨询。该通报用于平衡Oracle RAC实例之间的工作，并用于确定哪些实例提供最佳性能。应用程序透明地从提供最佳性能的实例接收连接。连接请求很快从已放慢，没有响应或失败的实例转移。

运行时连接负载平衡具有以下优点：

  • 管理池化连接以实现高性能和可扩展性

  • 接收有关路由到数据库实例的工作百分比的连续建议

  • 根据不同的后端节点容量（如CPU容量或响应时间）调整工作分配

  • 快速响应群集重新配置，应用程序工作负载，过度工作节点或挂起的更改

  • 从Oracle RAC负载平衡咨询中接收指标。最常使用与性能良好的实例的连接。与表现不佳的实例的新的和未使用的连接将随着时间的推移而消失。如果未收到分发指标，则使用随机选择选择连接。

9.3.2设置运行时连接负载均衡

运行时连接负载平衡要求正确启用和配置FCF。

此外，您必须为启用了负载平衡的每项服务配置Oracle RAC负载平衡咨询，并提供服务级别目标：

  • 必须将服务目标设置为以下之一：

    • DBMS_SERVICE.SERVICE_TIME

    • DBMS_SERVICE.THROUGHPUT

    可以使用goal参数设置服务目标，并且可以使用clb_goal参数设置连接平衡目标。

  • 连接平衡目标必须设置为SHORT 。例如，

    EXECUTE DBMS_SERVICE.MODIFY_SERVICE（service_name =>'sjob' - ，goal => DBMS_SERVICE.GOAL_THROUGHPUT  - ，clb_goal => DBMS_SERVICE.CLB_GOAL_SHORT）;

    要么

    EXECUTE DBMS_SERVICE.MODIFY_SERVICE（service_name =>'sjob' - ，goal => DBMS_SERVICE.GOAL_SERVICE_TIME  - ，clb_goal => DBMS_SERVICE.CLB_GOAL_SHORT）;

也可以通过调用DBMS_SERVICE.create_service过程来设置连接平衡目标。

注意：

您可以将连接平衡目标设置为LONG 。但是，这对于已关闭的工作负载来说非常有用，也就是说，当完成工作的速率等于启动新工作的速度时。

使用DBMS_FILE_TRANSFER包在平台之间转换数据

您可以使用GET_FILE或PUT_FILE的议事DBMS_FILE_TRANSFER包中的数据文件传输过程中的平台之间的数据转换。

当您使用这些过程之一在源平台和目标平台之间移动数据文件时，每个数据文件中的每个块都将转换为目标平台的字节顺序。

本节使用一个示例来描述如何使用DBMS_FILE_TRANSFER包将数据文件转换为其他平台。该示例做出以下假设：

  • GET_FILE过程将传输数据文件。

  • mytable.342.123456789数据文件正在转移到另一个平台。

  • 源平台的字节顺序与目标平台的字节顺序不同。

  • 源数据库的全局名称是dbsa.example.com 。

  • 源数据库和目标数据库都使用Oracle自动存储管理（Oracle ASM）。

注意：

您还可以使用DBMS_FILE_TRANSFER包在具有相同字节顺序的平台之间传输数据文件。

完成以下步骤以通过使用GET_FILE过程传输数据文件来转换数据文件：

  1. 使用SQL * Plus以可以创建目录对象的管理用户身份连接到源数据库。
  2. 创建目录对象以存储要传输到目标数据库的数据文件。

    例如，要为+data/dbsa/datafile目录创建名为sales_dir_source的目录对象，请执行以下SQL语句：

    创建或替换目录sales_dir_source AS'+ data / dbsa / datafile';

    创建目录对象时，指定的文件系统目录必须存在。

  3. 使用SQL * Plus作为管理用户连接到目标数据库，管理用户可以创建数据库链接，创建目录对象，并在DBMS_FILE_TRANSFER包中运行这些过程。
  4. 创建从目标数据库到源数据库的数据库链接。

    源数据库中的已连接用户必须对您在步骤2中创建的目录对象具有读取权限。

  5. 创建目录对象以存储要从源数据库传输的数据文件。

    将在DBMS_FILE_TRANSFER程序包中运行该过程的本地数据库用户必须对目录对象具有写权限。

    例如，要为+data/dbsb/datafile目录创建名为sales_dir_target的目录对象，请运行以下SQL语句：

    创建或替换目录sales_dir_target AS'+ data / dbsb / datafile';
  6. 运行DBMS_FILE_TRANSFER包中的GET_FILE过程以传输数据文件。

    例如，运行以下过程以使用您在步骤4中创建的数据库链接将mytable.342.123456789数据文件从源数据库传输到目标数据库：

    BEGIN DBMS_FILE_TRANSFER.GET_FILE（source_directory_object =>'sales_dir_source'，source_file_name =>'mytable.342.123456789'，source_database =>'dbsa.example.com'，destination_directory_object =>'sales_dir_target'，destination_file_name =>'mytable'）;结束; /

注意：

1.78 DB_FILE_NAME_CONVERT

DB_FILE_NAME_CONVERT对于创建重复数据库以进行恢复非常有用。它将主数据库上的新数据文件的文件名转换为备用数据库上的文件名。

属性 描述

参数类型

串

句法

DB_FILE_NAME_CONVERT = ' string1 ' , ' string2 ' , ' string4 ' , ... ' , ' string3 ' , ...

哪里：

  • string1是主数据库文件名的模式

  • string2是备用数据库文件名的模式

  • string3是主数据库文件名的模式

  • string4是备用数据库文件名的模式

您可以根据需要使用多对主要和备用替换字符串。您可以使用单引号或双引号。

以下是可接受的示例设置：

DB_FILE_NAME_CONVERT ='/ dbs / t1 /'，'/ dbs / t1 / s'，'dbs / t2 /'，'dbs / t2 / s_'

默认值

没有默认值。

可修改

改变会话

可在PDB中修改

没有

基本

没有

如果将数据文件添加到主数据库，则必须将相应的文件添加到备用数据库。更新备用数据库时，此参数将主数据库上的数据文件名转换为备用数据库上的数据文件名。备用数据库上的文件必须存在且可写，否则恢复过程将因错误而停止。

如果指定奇数个字符串（最后一个字符串没有相应的替换字符串），则在启动期间会发出错误信号。如果要转换的文件名与模式/替换字符串列表中的多个模式匹配，则第一个匹配的模式生效。您可以在此参数中指定的对数量没有限制（除了多值参数的最大长度的硬限制之外）。

将此参数的值设置为两个字符串。第一个字符串是在主数据库的数据文件名中找到的模式。第二个字符串是在备用数据库上的数据文件名中找到的模式。

在表空间时间点恢复期间设置克隆数据库时，还可以使用DB_FILE_NAME_CONVERT重命名克隆控制文件中的数据文件。
