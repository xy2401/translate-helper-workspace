关于Oracle ASM滚动升级

通过Oracle ASM滚动升级，您可以独立升级或修补集群Oracle ASM节点，而不会影响数据库可用性，从而提供更长的正常运行时间。

滚动升级意味着当群集中的一个或多个节点使用不同的软件版本时，群集Oracle ASM环境的某些功能将继续运行。Oracle建议您在执行Oracle Clusterware滚动升级时执行Oracle ASM滚动升级。

要执行滚动升级，必须准备好您的环境。在启动Oracle ASM滚动升级之前，必须将Oracle Clusterware完全升级到下一个修补程序或发行版。此外，您应该以滚动升级方式准备Oracle Clusterware，以确保高可用性和最长的正常运行时间。

请注意，Oracle ASM随Oracle Clusterware for Oracle 11g第2版（11.2）或更高版本一起升级，因为它们都位于Oracle Grid Infrastructure主目录中。

注意：

  • 滚动升级仅适用于集群Oracle ASM实例，您只能对使用Oracle Database 11g或更高版本的环境执行滚动升级。您无法使用此功能从Oracle数据库10g升级到Oracle Database 11g 。

  • 有关在存在Oracle Exadata存储时执行Oracle ASM实例的滚动升级的信息，请参阅Oracle Exadata文档 。

关于Oracle ASM Rolling Patches

您可以在集群Oracle ASM环境中应用修补程序，以便一次将一个节点更新到最新的修补程序级别，而不会影响Oracle ASM集群或使用Oracle ASM进行存储的数据库集群的整体可用性。

ALTER SYSTEM ROLLING PATCH SQL语句使您可以启动和停止滚动补丁。例如：

SQL> ALTER SYSTEM START ROLLING PATCH; SQL> ALTER SYSTEM STOP ROLLING PATCH;

7.1关于动态性能视图

Oracle包含一组基础视图，这些视图由数据库服务器维护，并且可供数据库管理员用户SYS 。这些视图称为动态性能视图，因为它们在数据库打开和使用时不断更新，其内容主要与性能相关。

虽然这些视图似乎是常规数据库表，但它们不是。这些视图提供有关内部磁盘结构和内存结构的数据。您可以从这些视图中进行选择，但永远不能更新或更改它们。

注意：

  • 您可以查询动态性能视图以从中提取信息。但是，仅支持简单查询。如果需要排序，连接， GROUP BY子句等，那么您应该将每个V$视图中的信息复制到一个表中（例如，使用CREATE TABLE ...AS SELECT语句），然后从那些表中查询。

  • 由于V$视图中的信息是动态的，因此无法保证这些视图上的SELECT操作的读取一致性。

catalog.sql脚本包含动态性能视图的视图和公共同义词的定义。您必须运行catalog.sql才能创建这些视图和同义词。安装后，只有用户SYS或具有SYSDBA权限的任何人都可以访问动态性能表。有关运行catalog.sql详细信息，请参见“ Oracle数据库管理员指南” 。

7.1.1 V $视图

实际的动态性能视图由前缀V_$标识。这些视图的公共同义词具有前缀V$ 。数据库管理员和其他用户应该只访问V$对象，而不是V_$对象。

Oracle Enterprise Manager使用动态性能视图，它是访问系统性能信息的主要接口。启动实例后，可以访问从内存中读取的V$视图。从磁盘读取数据的视图要求装入数据库，有些要求打开数据库。

V $视图是CONTAINER_DATA对象。当连接到根的用户查询V $视图时，查询结果将取决于视图用户的CONTAINER_DATA属性。SQL ALTER USER语句的CONTAINER_DATA子句用于设置和修改用户的CONTAINER_DATA属性。

6.3运行Java存储过程

您可以运行Java 存储过程的方式与PL / SQL存储过程相同。通常，对Java存储过程的调用是数据库操作的结果，因为它通常是触发器或SQL DML调用的结果。要调用Java存储过程，必须通过调用规范发布它。

在调用Java存储过程之前，必须将它们加载到Oracle数据库实例中并将它们发布到SQL。加载和发布是单独的任务。许多Java类（仅由其他Java类引用）永远不会发布。

要自动加载Java存储过程，可以使用loadjava工具。它将Java源，类和资源文件加载到系统生成的数据库表中，然后使用SQL CREATE JAVA {SOURCE | CLASS | RESOURCE}语句将Java文件加载到Oracle数据库实例中。您可以从文件系统，流行的Java IDE，Intranet或Internet上载Java文件。

您必须执行以下步骤来创建，加载和调用Java存储过程：

  • 创建或重用Java类

  • 加载和解析Java类

  • 发布Java类

  • 调用存储过程

注意：

要手动加载Java存储过程，可以使用CREATE JAVA语句。例如，在SQL * Plus中，您可以使用CREATE JAVA CLASS语句将Java类文件从本地BFILE和LOB列加载到Oracle数据库中。

6.3.1创建或重用Java类

使用首选Java IDE创建类，或重用符合您要求的现有类。Oracle数据库支持许多Java开发工具和客户端编程接口。例如，Oracle JVM接受在流行的Java IDE中开发的程序，例如Oracle JDeveloper，Symantec Visual Cafe和Borland JBuilder。

在以下示例中，您将创建public类Oscar 。它有一个名为quote()方法，它返回Oscar Wilde的引文。 

公共课奥斯卡{//从奥斯卡王尔德返回引文public static String quote（）{return“除了诱惑，我能抵抗一切。”; }}

将该类保存为Oscar.java 。使用Java编译器编译客户端系统上的.java文件，如下所示：

javac Oscar.java

编译器输出Java二进制文件，在本例中为Oscar.class 。

6.3.2加载和解析Java类

使用loadjava工具，您可以将Java源代码，类和资源文件加载到Oracle数据库实例中，并将它们存储为Java模式对象。您可以从命令行或从应用程序运行loadjava工具，并且可以指定多个选项，包括解析程序。

在以下示例中， loadjava工具使用默认的JDBC OCI驱动程序连接到数据库。您必须指定用户名和密码。默认情况下， Oscar类被加载到您登录的用户的模式中，在本例中为HR 。

创建PFILE

目的

使用CREATE PFILE语句将二进制服务器参数文件或当前的In-Memory参数设置导出到文本初始化参数文件中。创建文本参数文件是获取数据库正在使用的当前参数设置列表的便捷方式，它允许您在文本编辑器中轻松编辑文件，然后使用CREATE SPFILE将其转换回服务器参数文件声明。

成功执行此语句后，Oracle数据库会在服务器上创建一个文本参数文件。在Oracle Real Application Clusters环境中，它将包含所有实例的所有参数设置。它还将包含与服务器参数文件中的参数设置在同一行上出现的任何注释。

关于在CDB中创建文本参数文件的注意事项

在多租户容器数据库（CDB）中创建文本参数文件时，当前容器可以是根或PDB。

  • 如果当前容器是根，则数据库将创建一个包含根的参数设置的文本文件。

  • 如果当前容器是PDB，则数据库将创建一个包含PDB参数设置的文本文件。在这种情况下，您必须指定pfile_name 。

13.53 RESTRICT_REFERENCES Pragma

RESTRICT_REFERENCES pragma声明用户定义的子程序不读取或写入数据库表或包变量。

注意：

不推荐使用RESTRICT_REFERENCES编译指示。Oracle建议使用DETERMINISTIC和PARALLEL_ENABLE而不是RESTRICT_REFERENCES 。

读取或写入数据库表或包变量的子程序很难优化，因为子程序的任何调用都可能产生不同的结果或遇到错误。如果用户定义的子程序中的语句违反了RESTRICT_REFERENCES所做的断言，则PL / SQL编译器在解析该语句时会发出错误消息，除非您指定TRUST 。

通常，此pragma是为函数指定的。如果函数调用过程，则也为这些过程指定此pragma。

  • 该pragma只能出现在包规范或ADT规范中。

  • 只有一个RESTRICT_REFERENCES pragma可以引用给定的子程序。

8.1关于协会

关联是一种数据挖掘功能，可以发现集合中项目共现的概率。共现项目之间的关系表示为关联规则 。

8.1.1关联规则

关联模型的结果是识别数据内关联模式的规则。Oracle Data Mining不支持关联建模的评分操作。

关联规则可以应用如下：

  • 支持 ：这些项目在数据中一起出现的频率是多少？
  • 信心 ：在包含前提的交易中，结果发生的频率。
  • 价值 ：与商品关联相关的商业价值

8.1.2市场篮子分析

关联规则通常用于分析销售交易。例如，注意到在杂货店购买谷物的顾客经常同时购买牛奶。事实上，关联分析发现，85％的包括谷物的结账会议也包括牛奶。这种关系可以表述为以下规则：

谷物意味着牛奶的置信度为85％

这种关联建模的应用称为市场篮子分析 。它对于直接营销，促销和发现业务趋势很有价值。市场购物篮分析也可以有效地用于商店布局，目录设计和交叉销售。

8.1.3关联规则和电子商务

关于并行升级可插拔数据库（PDB）

使用并行技术，您可以升级CDB，然后立即使用并行SQL处理器升级PDB。

容器数据库（CDB）可以包含零个，一个或多个可插拔数据库（PDB）。默认情况下，并行升级实用程序（ catctl.pl ）在同一升级窗口中更新CDB及其所有PDB。并行升级实用程序使用计算机处理单元（CPU）的数量来确定同时升级的最大PDB数。并行升级的PDB数量是通过将并行SQL进程计数（ -n选项）除以并行PDB SQL进程计数（ -N选项）来确定的。

注意：

您必须规划升级窗口，以适应CDB上提供的所有数据库服务的常见停机时间。

可插拔数据库升级语法

dbupgrade [-M] -n [-N]

  • -M指定CDB $ ROOT是保持升级模式，还是在升级完成后可用：

    • 如果使用-M参数运行并行升级实用程序，则升级会将CDB $ ROOT及其所有PDB置于升级模式，这可以减少总升级时间。但是，在CDB及其所有PDB升级之前，您无法启动任何PDB。

    • 如果不使用-M参数运行Parallel Upgrade Utility，则会在正常模式下升级并重新启动CDB $ ROOT，并启动正常的后台进程。升级成功后，只能在读/写模式下打开CDB $ ROOT。所有PDB都处于MOUNT模式。随着每个PDB的升级，您可以将每个PDB联机，而其他PDB仍在升级。

  • -n指定并行PDB升级处理器的数量。

    如果未指定-n的值，则-n的CPU_COUNT值为CPU_COUNT值。

    如果为-n指定了值，则该值用于确定并行SQL进程的数量。最大值是无限的。最小值为4。

  • -N指定升级PDB时要使用的SQL处理器数。最大值为8。最小值为1。如果未指定-N的值，则默认值为2。

  • 同时运行的最大PDB升级是-n的值除以-N的值。

以下是In Parallel PDB升级技术期间的高级操作列表：

  1. 确保您的备份策略已完成。

  2. 运行升级前工具。修复报告的任何问题..升级前工具（ preupgrade.jar ）随新的Oracle数据库版本一起提供。

  3. 运行并行升级实用程序。按顺序，执行以下升级：

    1. 周期1：CDB $ ROOT升级到新的Oracle版本

    2. 循环2到循环x：PDB $ SEED和PDB并行升级，升级循环次数由您使用-n指定的参数设置确定。

  4. 完成升级后的步骤。
