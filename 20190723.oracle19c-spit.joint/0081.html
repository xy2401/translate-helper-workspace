2.4分配频道

目的

ALLOCATE CHANNEL手动分配一个通道 ，这是RMAN和数据库实例之间的连接。必须在RUN块中发出ALLOCATE CHANNEL命令。它仅在发出命令的块中分配通道。

先决条件

必须启动目标实例。

使用说明

手动分配的通道与使用CONFIGURE指定的自动分配的通道不同。自动通道适用于您不手动分配通道的任何RMAN作业。您可以通过在RUN命令中手动分配通道来覆盖自动通道配置，但在使用ALLOCATE CHANNEL指定手动通道后，不能使用BACKUP DEVICE TYPE或RESTORE DEVICE TYPE来使用自动通道。

多个频道

您最多可以分配255个频道;每个通道可以并行读取多达64个文件。您可以通过分配的通道数控制作业内的并行度。同时分配多个通道允许单个作业并行读取或写入多个备份集或磁盘副本，每个通道在单独的备份集或副本上运行。

在备份到磁盘时，指南是为每个输出设备分配一个通道（参见示例2-7 ）。但是，如果RMAN正在写入条带文件系统或ASM磁盘组，则多个通道可以提高性能。备份到磁带时，准则是磁带通道数等于磁带设备数除以双工副本数（参见示例2-8 ）。

Oracle RAC环境中的通道

如果SYS和SYSBACKUP用户的所有Oracle RAC实例的密码相同，则无需将密码放在ALLOCATE或CONFIGURE命令的CONNECT选项中。如果使用user @ database形式的连接字符串，则RMAN会自动使用与RMAN会话启动时用于TARGET连接的密码相同的密码。

架构名称

语义网络的所有者。（ 见表1-1 ）

network_name

语义网络的名称。（ 见表1-1 ）

使用说明

使用三级安全性时，OLS将应用于网络中的每个语义模型。也就是说，标签安全性应用于相关的内部表和所有应用程序表;无需手动将策略应用于现有语义模型的应用程序表。但是，如果在应用OLS策略后需要创建其他模型，则必须使用SEM_OLS.APPLY_POLICY_TO_APP_TAB过程在创建模型之前将OLS应用于应用程序表。

您必须具有以下条件才能执行此过程：SA_POLICY_ADMIN程序包的EXECUTE特权和policy_DBA角色。

在执行此过程之前，您必须执行SEM_RDFSA.APPLY_OLS_POLICY过程，为rdfsa_options参数指定SEM_RDFSA.TRIPLE_LEVEL_ONLY 。

要从应用程序表中删除OLS策略，请使用SEM_OLS.REMOVE_POLICY_FROM_APP_TAB过程。

有关OLS支持的信息，请参阅RDF数据的细粒度访问控制 。

有关语义网络类型和选项的信息，请参阅语义网络 。

例子

以下示例将名为defense的OLS策略应用于MY_SCHEMA.MY_APP_TABLE应用程序表。 

开始sem_ols.apply_policy_to_app_table（policy_name =>'defense'，schema_name =>'my_schema'，table_name =>'my_app_table'）;结束; /

1.334 TDE_CONFIGURATION

TDE_CONFIGURATION用于透明数据加密（TDE）的每PDB配置。在Oracle Database 18c之前，每个PDB将其单独的加密密钥存储在CDB的密钥库（联合模式）中。从Oracle Database 18c开始 ，PDB可以选择将其加密密钥存储在单独的密钥库（隔离模式）中，从而允许通过单独的密钥库密码进行保护。必须设置WALLET_ROOT初始化参数才能使TDE_CONFIGURATION生效。

可以指定以下属性：

  • KEYSTORE_CONFIGURATION属性。此属性是必需的。使用此属性指定的值将配置指定PDB的密钥库类型。可以为此属性指定以下值：

    • FILE ：此值配置钱包密钥库。

    • OKV ：此值配置Oracle Key Vault（OKV）密钥库。

    • HSM ：此值配置硬件安全模块（HSM）密钥库。

    • FILE|OKV ：此值配置从OKV到钱包密钥库的反向迁移。

    • FILE|HSM ：此值配置从HSM到钱包密钥库的反向迁移。

    • OKV|FILE ：此值配置从钱包到OKV密钥库的迁移。

    • HSM|FILE ：此值配置从钱包到HSM密钥库的迁移。

    某些KEYSTORE_CONFIGURATION属性值由单个单词组成，例如， FILE ， OKV和HSM值。其他KEYSTORE_CONFIGURATION属性值由两个由“|”字符分隔的单词组成，这两个单词是值语法的必需部分，例如， FILE|OKV ， FILE|HSM ， OKV|FILE和HSM|FILE值。

    在Oracle Database 18.1之前的Oracle数据库发行版中，使用SQLNET.ENCRYPTION_WALLET_LOCATION参数的METHOD属性在sqlnet.ora中配置了密钥库类型。

  • CONTAINER属性：只有在CDB$ROOT中设置参数时，才能使用此可选属性。仅当CDB$ROOT处于MOUNTED状态时，才能指定CONTAINER属性。使用此属性，您必须指定要为其设置参数的PDB的名称。指定CONTAINER属性时，必须使用分号“;”作为KEYSTORE_CONFIGURATION和CONTAINER属性之间的分隔符。

例子

以下语句为发出语句的打开PDB配置钱包密钥库：

ALTER SYSTEM SET TDE_CONFIGURATION =“KEYSTORE_CONFIGURATION = FILE”SCOPE = BID = SID ='*';

以下语句为处于MOUNTED状态的PDB配置一个OKV密钥库，从该状态发出语句：

ALTER SYSTEM SET TDE_CONFIGURATION =“KEYSTORE_CONFIGURATION = OKV”SCOPE = SPFILE SID ='*';

以下语句为ORCLPDB PDB配置HSM密钥库。要使此语句成功，当CDB$ROOT处于MOUNTED状态时，必须在CDB$ROOT中设置该参数：

ALTER SYSTEM SET TDE_CONFIGURATION =“KEYSTORE_CONFIGURATION = HSM; CONTAINER = ORCLPDB”SCOPE = MEMORY SID ='*';

13.49 PIPELINED条款

指示数据库迭代地返回表函数或多态表函数的结果 。

仅用于表函数，以指定它是流水线的。流水线表函数在处理该行后立即向其调用者返回一行并继续处理行。要向调用者返回一行（但不是控件），该函数使用“ PIPE ROW语句 ” 。

表函数返回集合类型。

多态表函数是一个表函数，其返回类型由参数确定。

您可以在查询的FROM子句中的函数名称之前使用TABLE关键字来查询这两种表函数。例如：

SELECT * FROM TABLE（ function_name （...））

当表函数参数列表或空列表（）出现时， TABLE运算符是可选的。例如：

SELECT * FROM function_name （）

然后，数据库返回由函数生成的行。

PIPELINED选项可以出现在以下SQL语句中：

  • 创建功能声明

  • CREATE PACKAGE声明

  • CREATE PACKAGE BODY声明

  • 您可以指定PIPELINED USING implementation_type子句来预定义包含start，fetch和close操作的接口。实现类型必须实现ODCITable接口，并且必须在创建表函数时存在。此子句对于在外部语言（如C ++和Java）中实现的表函数很有用。

[schema。] implementation_type

实现类型必须是包含ODCIAggregate子程序实现的ADT。如果未指定schema ，则数据库假定实现类型位于模式中。

对PIPELINED的限制

您不能为嵌套函数指定PIPELINED 。

注意：

您无法在数据库链接上运行管道化表函数。原因是流水线表函数的返回类型是SQL用户定义的类型，只能在单个数据库中使用（如Oracle数据库对象关系开发人员指南中所述 ）。虽然流水线表函数的返回类型可能看起来是PL / SQL类型，但数据库实际上将该PL / SQL类型转换为相应的SQL用户定义类型。

随着FOR LOOP语句的每次迭代，其语句运行，其索引递增或递减，并且控制返回到循环的顶部。FOR LOOP语句在其索引达到指定值时结束，或者当循环内的语句将控制转移到循环外或引发异常时结束。

指数

隐含声明的整数变量的名称，该变量是FOR LOOP语句的本地变量。循环外的语句不能引用index 。循环内的语句可以引用index ，但不能更改其值。运行FOR LOOP语句后， index未定义。

[REVERSE] lower_bound .. upper_bound

lower_bound和upper_bound必须求值为数字（参见“ 下界和上界 ” ）。输入FOR LOOP语句时，PL / SQL会对lower_bound和upper_bound一次计算，并将它们存储为临时PLS_INTEGER值，必要时将它们四舍五入为最接近的整数。

如果lower_bound等于upper_bound ，则statements只运行一次。

如果FOR LOOP语句开始运行时lower_bound不等于upper_bound ，则：

  • 如果省略REVERSE ：

    如果lower_bound大于upper_bound ，则statements不会运行，并且控制将转移到FOR LOOP语句之后的语句。

    否则，将lower_bound分配给index ，运行statements ，并且控制返回到循环的顶部，其中index与upper_bound进行比较。如果index小于upper_bound ，则index增加1， statements再次运行，并且控制返回到循环的顶部。当index大于upper_bound ，控制将转移到FOR LOOP语句之后的语句。

  • 如果指定了REVERSE ：

    如果upper_bound小于lower_bound ，则statements不会运行，并且控制将转移到FOR LOOP语句之后的语句。

    否则，将upper_bound分配给index ，运行statements ，并且控制返回到循环的顶部，其中index与lower_bound进行比较。如果index大于lower_bound ，则index减1， statements再次运行，控制返回到循环的顶部。当index小于lower_bound ，控制转移到FOR LOOP语句之后的语句。

标签

标识for_loop_statement的标签（请参阅“ statement :: = ”和“ label ” ）。CONTINUE ， EXIT和GOTO语句可以引用此标签。

标签提高了可读性，尤其是在嵌套LOOP语句时，但LOOP是确保END LOOP语句中的标签与同一LOOP语句开头的标签匹配（编译器不检查）。

6.13在对象视图中定义复杂关系

您可以使用MAKE_REF运算符在对象视图中定义循环引用： view_A可以引用view_B ，而view_B又可以引用view_A 。这允许对象视图合成复杂结构，例如来自关系数据的图形。

例如，在部门和员工的情况下，部门对象当前包括雇员列表。为了节省空间，您可能希望在department对象内部引用employee对象，而不是实现department对象中的所有雇员。您可以构造（固定）对员工对象的引用，然后使用点表示法引用引用以提取员工信息。

由于employee对象已经具有对员工所在部门的引用，因此该模型上的对象视图包含部门视图和员工视图之间的循环引用。

注意：

创建循环引用的两种方法都需要“ 表和类型中演示圆形视图引用 ”中描述的设置。

  • 第二视图后的第一视图

    1. 创建视图A而不参考视图B.

    2. 创建视图B，其中包含对视图A的引用。

    3. 将视图A替换为包含对视图B的引用的新定义。

    请参阅“ 方法1：创建第二个视图后重新创建第一个视图 ”中的示例

  • 使用FORCE关键字的第一个视图

    2. 使用视图A的引用创建视图B.使用视图A时，将对其进行验证和重新编译。

    请参阅“ 方法2：使用FORCE关键字创建第一个视图 ”中的示例

创建SPFILE

目的

使用CREATE SPFILE语句从传统的纯文本初始化参数文件或当前的系统范围设置创建服务器参数文件。服务器参数文件是仅存在于服务器上的二进制文件，从客户端位置调用以启动数据库。

服务器参数文件允许您对各个参数进行持久更改。使用服务器参数文件时，可以在ALTER SYSTEM SET parameter语句中指定新参数值应该是持久的。这意味着新值不仅适用于当前实例，还适用于随后启动的任何实例。传统的纯文本参数文件不允许您对参数值进行持久更改。

服务器参数文件位于服务器上，因此它们允许Oracle数据库自动进行数据库调整，并由Recovery Manager（RMAN）进行备份。

要在启动数据库时使用服务器参数文件，必须使用CREATE SPFILE语句创建它。

Oracle Real Application Clusters环境中的所有实例都必须使用相同的服务器参数文件。但是，如果允许，单个实例可以在此一个文件中具有相同参数的不同设置。特定于实例的参数定义指定为SID.parameter = value ，其中SID是实例标识符。

使用服务器参数文件启动数据库的方法取决于您是创建默认服务器参数文件还是非默认服务器参数文件。有关如何使用服务器参数文件的示例，请参阅“ 创建服务器参数文件：示例 ” 。

关于在CDB中创建服务器参数文件的注意事项

在多租户容器数据库（CDB）中创建服务器参数文件时，当前容器可以是根或PDB。

  • 如果当前容器是根，则您为根中的初始化参数设置的值将用作所有其他容器的默认值。

  • 如果当前容器是PDB，则数据库在内部存储PDB的初始化参数值，而不是在文件中。因此，您无法指定spfile_name 。您为PDB中的初始化参数设置的值是持久的，并覆盖为根中的这些参数设置的任何值。

随后可以使用ALTER SYSTEM语句修改根或PDB的初始化参数值。

语义

spfile_name

此子句允许您为要创建的服务器参数文件指定名称。

如果指定spfile_name ，则Oracle数据库会创建非默认服务器参数文件。

  • 对于spfile_name ，您可以指定传统文件名，Oracle ACFS文件系统中的文件或Oracle Storage Management（Oracle ASM）文件名。

  • 如果在Oracle ACFS文件系统中指定传统文件名或文件，则spfile_name可以包含路径前缀。如果未指定此类路径前缀，则数据库会添加默认存储位置的路径前缀，该位置与平台相关。

  • 如果指定Oracle ASM文件名语法，则数据库将在Oracle ASM磁盘组中创建spfile。

  • 使用非默认服务器参数文件时，必须在STARTUP数据库时在STARTUP命令中指定服务器参数文件名。此规则的例外情况如下：

    • 如果数据库被定义为在Oracle集群的资源，从中发出命令的实例在运行，并且指定spfile_name ，指定FROM PFILE子句，并省略AS COPY子句，则该语句将自动更新SPFILE数据库资源。在这种情况下，您可以启动数据库，而无需按名称引用服务器参数文件。如果发出命令的实例未运行，则必须使用srvctl modify database -d dbname -spfile spfile_path手动更新数据库资源中的SPFILE。

如果省略spfile_name ，则Oracle数据库使用特定于平台的缺省服务器参数filename。如果服务器上已存在此类文件，则此语句将覆盖该文件。使用默认服务器参数文件时，可以启动数据库而无需按名称引用文件。

对 spfile_name的 限制

在连接到PDB时创建服务器参数文件时，无法指定spfile_name 。

pfile_name

指定要从中创建服务器参数文件的传统纯文本初始化参数文件。传统参数文件必须驻留在服务器上。

  • 如果指定pfile_name且传统参数文件不驻留在操作系统上参数文件的缺省目录中，则必须指定完整路径。

  • 如果未指定pfile_name ，则Oracle数据库会在操作系统的缺省目录中查找参数文件，以获取缺省参数filename并使用该文件。如果该文件在预期目录中不存在，则数据库将返回错误。

在Oracle Real Application Clusters环境中，必须先将所有实例参数文件合并到一个文件中，然后在此语句中指定该文件名以创建服务器参数文件。有关完成此步骤的信息，请参阅“ Oracle Real Application Clusters管理和部署指南” 。

作为复制

仅当数据库被定义为Oracle Clusterware中的资源时，此子句才适用。默认情况下，如果同时指定spfile_name和FROM PFILE子句，那么CREATE SPFILE自动语句更新数据库中的资源的SPFILE。您可以指定AS COPY以防止数据库更新数据库资源中的SPFILE。

V$AQ_SUBSCRIBER_LOAD描述了在Oracle RAC环境中的每个实例的延迟方面所有分片队列订户的负载。

    ％sqlldr userid = ctxdemo / ctxdemo control = loader.ctl

    您可以从SQL * Plus创建索引集：

    sqlplus> begin ctx_ddl.create_index_set（'bookset'）; ctx_ddl.add_index（ '查询手册'， '价格'）; ctx_ddl.add_index（ '查询手册'， '发布者'）;结束; /
