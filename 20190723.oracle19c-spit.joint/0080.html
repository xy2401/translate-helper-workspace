Oracle Database Release 19c中的更改

这些是Oracle Data Guard Broker用户指南 （Oracle Database Release 19c中的更改）中的更改。

新功能

  • 您可以将快速启动故障转移目标动态更改为列表中的指定成员，而不会禁用快速启动故障转移。请参阅SET FAST_START FAILOVER TARGET 。

  • 在不影响当前环境的情况下，您可以使用快速启动故障转移的仅观察模式来测试快速启动故障转移的工作方式。请参阅仅观察模式下的配置快速启动故障转移 。

  • 代理配置可以导出并存储在文本文件中，从而使您可以创建当前代理配置的备份。必要时，可以使用导出文件中的元数据重新创建代理配置。新命令EXPORT CONFIGURATION和IMPORT CONFIGURATION用于导出和导入代理配置。请参见场景15：导出和导入代理配置文件 。

  • 新命令可用于设置，修改和显示数据库，远程同步实例或恢复设备中的数据库初始化参数的值。使用这些命令设置的值将直接应用于数据库，而不会存储在代理配置文件中。新命令如下：

    • 编辑数据库（参数）

    • EDIT DATABASE RESET（参数）

    • 编辑FAR_SYNC RESET（参数）

    • EDIT RECOVERY_APPLIANCE（参数）

    • EDIT RECOVERY_APPLIANCE RESET（参数）

    • SET TRACE_LEVEL

    新的子句SET PARAMETER被添加到EDIT FAR_SYNC命令中。

不推荐的功能

  • 此版本中不推荐使用与数据库初始化参数关联的以下属性，并且可能在将来的版本中不支持这些属性： ArchiveLagTarget ， DataGuardSyncLatency ， LogArchiveMaxProcesses ， LogArchiveMinSucceedDest ， LogArchiveTrace ， StandbyFileManagement ， DbFileNameConvert ， LogArchiveFormat ， LogFileNameConvert 。这些属性将不再存储在Broker配置文件中。

    而不是使用EDIT DATABASE...SET PROPERTY命令用于修改已弃用的属性，使用EDIT DATABASE…SET PARAMATER命令修改与已弃用的属性对应的数据库初始化参数。

  • 此版本中不推荐使用InconsistentProperties属性。此属性始终没有任何价值。

  • 不推荐使用以下与逻辑备用数据库相关的属性，并且可能在将来的发行版中不支持这些属性： LsbyMaxEventsRecorded ， LsbyMaxServers ， LsbyMaxSga ， LsbyPreserveCommitOrder ， LsbyRecordAppliedDdl ， LsbyRecordSkipDdl ， LsbyRecordSkipErrors和LsbyParameter 。使用DBMS_LOGSTDBY.APPLY_SET程序配置与合理的备用性质。不推荐使用的属性将不再存储在Broker配置文件中。

不受支持的功能

  • 不支持MaxConnections配置属性。

OnlineArchiveLocation属性重命名为ArchiveLocation ， OnlineAlternateLocation属性重命名为AlternateLocation 。

ArchiveLocation ， AlternateLocation ， StandbyArchiveLocation和StandbyAlternateLocation属性的行为有一些更改。此外，这些属性不会自动从Oracle Database Release 18c版本的元数据文件升级。升级到Oracle Database Release 19c后，必须设置这些属性的值。

目的

SQL / JSON函数JSON_OBJECT将一系列键值对或一个对象类型实例作为其输入。集合类型无法传递给JSON_OBJECT 。

它返回一个JSON对象，该对象包含每个键值对的对象成员。

条目

regular_entry ：使用此子句指定属性键值对。

regular_entry

  • KEY是可选的，并且是为了语义清晰度而提供的。

  • 使用可选的expr将属性键名称指定为区分大小写的文本文字。

  • 使用expr指定属性值。对于expr ，您可以指定任何计算结果为SQL数字文字，文本文字，日期或时间戳的表达式。日期和时间戳数据类型在生成的JSON对象或数组中打印为符合ISO日期格式的JSON字符串。如果expr求值为数字文字，则结果属性值为JSON数值;否则，结果属性值是用双引号括起来的区分大小写的JSON字符串值。

    您可以使用冒号分隔JSON_OBJECT条目。

JSON_OBJECT列条目

在某些情况下，您可能希望让JSON对象键名与表列的名称匹配，以避免在键值表达式中重复列名。例如：

SELECT JSON_OBJECT（'first_name'VALUE first_name，'last_name'VALUE last_name，'email'VALUE email，'hire_date'VALUE hire_date）FROM employees WHERE employee_id = 140; {“first_name”：“Joshua”，“last_name”：“Patel”，“email”：“JPATEL”，“hire_date”：“2006-04-06 06：00：00”}

在这种情况下，您可以使用快捷方式，其中可以将单个列值指定为输入，并从列的名称推断出相应的对象条目键。例如：

SELECT JSON_OBJECT（first_name，last_name，email，hire_date）FROM employees WHERE employee_id = 140; {“first_name”：“Joshua”，“last_name”：“Patel”，“email”：“JPATEL”，“hire_date”：“2006-04-06 06：00：00”}

您可以对列名称使用带引号或非带引号的标识符。如果使用非带引号的标识符，则使用查询中写入的标识符区分大小写的值来生成相应的对象键值。但是，为了引用列值，标识符仍然不区分大小写。例如：

SELECT JSON_OBJECT（eMail）FROM employees WHERE employee_id = 140 {“eMail”：“JPATEL”}

请注意，保留列名称中键入的大写“M”。

句法


插图说明regexp_count.eps

目的

REGEXP_COUNT通过返回模式在源字符串中出现的次数来补充REGEXP_INSTR函数的功能。该函数使用输入字符集定义的字符计算字符串。它返回一个整数，表示pattern的出现次数。如果未找到匹配项，则该函数返回0。

  • source_char是用作搜索值的字符表达式。它通常是字符列，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR ， NVARCHAR2 ， CLOB或NCLOB 。

  • pattern是正则表达式。它通常是文本文字，可以是任何数据类型CHAR ， VARCHAR2 ， NCHAR或NVARCHAR2 。它最多可包含512个字节。如果pattern的数据类型与source_char的数据类型不同，则Oracle数据库将pattern转换为source_char的数据类型。

    REGEXP_COUNT忽略pattern子表达式括号。例如，模式'(123(45))'相当于'12345' 。有关可以在pattern指定的运算符的列表，请参阅Oracle正则表达式支持 。

  • position是一个正整数，表示Oracle应该开始搜索的source_char的字符。默认值为1，表示Oracle在source_char的第一个字符处开始搜索。在找到第一次出现的pattern ，数据库将搜索以第一次出现后的第一个字符开头的第二次出现。

  • match_param是一个文本文字，可让您更改函数的默认匹配行为。您可以为match_param指定以下一个或多个值：

    • 'i'指定不区分大小写的匹配。

    • 'c'指定区分大小写的匹配。

    • 'n'允许句点（。）（匹配任意字符）匹配换行符。如果省略此参数，则句点与换行符不匹配。

    • 'm'将源字符串视为多行。Oracle将插入符号（ ^ ）和美元符号（ $ ）分别解释为源字符串中任何位置的任何行的开始和结束，而不是仅在整个源字符串的开头或结尾处。如果省略此参数，则Oracle会将源字符串视为单行。

    • 'x'忽略空格字符。默认情况下，空白字符与自身匹配。

    如果指定多个矛盾值，则Oracle使用最后一个值。例如，如果指定'ic' ，则Oracle使用区分大小写的匹配。如果指定的字符不是上面显示的字符，则Oracle会返回错误。

    如果省略match_param ，则：

    • 默认区分大小写由NLS_SORT参数的值确定。

    • 句点（。）与换行符不匹配。

    • 源字符串被视为单行。

9.1全球化和Unicode支持概述

OCCI现在可以在所有Oracle支持的多字节和Unicode字符集中启用应用程序开发。完全支持Unicode的UTF16编码。应用程序可以在创建OCCI环境时指定其字符集。采用字符串参数（如SQL语句，用户名，错误消息，对象名等）的OCCI接口已扩展为处理任何字符集中的数据。来自关系表或对象的字符数据可以在任何字符集中。OCCI可用于开发多语言，全局和Unicode应用程序。

9.2指定字符集

初始化OCCI环境时，OCCI应用程序必须指定客户端字符集和客户端国家字符集。客户端characterset指定所有SQL语句，对象/用户名，错误消息以及所有CHAR数据类型（ CHAR ， VARCHAR2 ， LONG ）列/属性的数据的CHAR 。客户端国家字符集指定所有NCHAR数据类型（ NCHAR ， NVARCHAR2 ）列/属性的数据的字符集。

现在提供了一个新的createEnvironment()接口，它接受客户端字符集和客户端国家字符集。这允许OCCI应用程序动态设置字符集信息，与NLS_LANG和NLS_CHAR初始化参数无关。

请注意，如果应用程序将OCCIUTF16指定为客户端字符集（第一个参数），则应用程序应仅使用OCCI的UTF16接口。这些接口采用UString参数类型。

OCCI环境中的字符集仅限客户端。它们表示OCCI应用程序用于与Oracle交互的字符集。创建数据库时指定数据库字符集和数据库国家字符集。在服务器处理数据之前，Oracle会将客户端字符集/国家字符集中的所有数据转换为数据库字符集/国家字符集。

示例9-1如何使用全球化和Unicode支持

环境* env =环境：createEnvironment（“JA16SJIS”，“UTF8”）;

此语句创建一个OCCI Environment其中JA16SJIS作为客户端字符集， UTF8作为客户端国家字符集。

任何有效的Oracle字符集名称（ AL16UTF16除外）都可以传递给createEnvironment() 。可以传递OCCI特定字符串OCCIUTF16 （大写）以指定UTF16作为字符集。 

环境* env = Environment :: createEnvironment（“OCCIUTF16”，“OCCIUTF16”）;环境* env = Environment :: createEnvironment（“US7ASCII”，“OCCIUTF16”）;

9.3全球化和Unicode支持的数据类型

用于支持全球化和使用unicode的数据类型包括：

  • 使用UString数据类型

  • 使用多字节和UTF16数据

  • 使用CLOB和NCLOB数据类型

9.3.1使用UString数据类型

UString是一种数据类型，它使应用程序和OCCI库能够以UTF-16编码传递和接收Unicode数据。UString是使用Oracle的utext数据类型从C ++ STL basic_string模板化的。 

typedef basic_string <utext> UString;

Oracle的utext数据类型是一个2字节的短数据类型，表示UTF-16编码中的Unicode字符。Unicode字符的代码点可以用1个utext或2个utext （2或4个字节）表示。来自欧洲和大多数亚洲脚本的字符在单个文本中表示。Unicode 3.1标准中定义的补充字符用2个utext元素表示。

在Microsoft Windows平台中， UString等同于C ++标准的wstring数据类型。这是因为wchar_t数据类型在这些平台中的类型定义为2字节short ，这与Oracle的utext相同，允许应用程序使用wstring类型变量，其中通常需要UString 。因此，应用程序还可以传递宽字符字符串文字，这些文字是通过在文字前面添加字母“L”来创建的，用于OCCI Unicode API。

OCCI应用程序应将UString数据类型用于UTF16字符UString数据

//使用wstring数据类型绑定Unicode数据//绑定欧元符号，UTF16代码点0x20AC wchar_t eurochars [] = {0x20AC，0x00}; wstring eurostr（eurochars）; stmt-> setUString（1，eurostr）; //通过//传递widechar文字调用Unicode版本的createConnection连接* conn = Connection（L“HR”，L“ password ”，L“”）;

对于JA16SJIS和UTF8等多字节字符JA16SJIS数据，应用程序应使用C ++ string类型。采用string参数的现有OCCI API可以处理任何多字节字符集中的数据。由于使用了string类型，OCCI仅支持多字节strings字节长度语义。

示例9-3使用字符串数据类型绑定UTF8数据

9.3.3使用CLOB和NCLOB数据类型

Oracle提供CLOB和NCLOB数据类型，用于存储和处理大量字符数据。CLOB表示数据库字符集中的数据， NCLOB表示数据库国家字符集中的数据。CLOB和NCLOB可以用作关系表中的列类型和对象类型中的属性。

OCCI Clob类用于处理CLOB和NCLOB数据类型。如果数据库类型是NCLOB ，则在从LOB读取/写入之前，应使用OCCI_SQLCS_NCHAR调用Clob set CharSetForm()方法。

OCCI Clob类支持多字节和UTF16字符集。默认情况下， Clob接口假定数据在客户端字符集中编码（对于CLOB和NCLOB ）。要指定不同的字符集或为NCLOB指定客户端国家字符集，请使用适当的字符集调用setCharSetId()或setCharSetIdUString()方法。可以传递OCCI特定字符串'OCCIUTF16'以指示UTF16作为字符集。

要读取或写入多字节字符集中的数据，请使用带有char缓冲区的现有读取和写入接口。为UTF16数据提供utext缓冲区的新重载接口已作为read() ， write()和writeChunk()方法添加到Clob类中。这些方法的参数和返回值是字节或字符，具体取决于LOB的字符集。

11.2.1关于构建分片键

分片识别应用程序必须识别和构建分片密钥和超级分片密钥，这些密钥是建立与分片数据库的连接所必需的。为实现此目的，分片OracleShardingKey应用程序必须使用OracleShardingKey和OracleShardingKeyBuilder接口。

OracleShardingKeyBuilder使用以下构建器方法来支持具有不同数据类型的复合键：

子键（Object子键，java.sql。SQLTYPE subkeyDataType）

构建器上有多个子subkey方法调用，用于构建复合分片键，其中每个子键可以是不同的数据类型。可以使用oracle.jdbc.定义数据类型oracle.jdbc.OracleType枚举或java.sql.JDBCType 。

11.2.2如何使用分片键从池中检出连接

当从UCP借用连接时，分片识别应用程序可以使用PoolDataSource类中存在的新连接构建器提供分片键和超分片键。如果不存在分片键或不映射到数据库元数据指定的数据类型，则抛出IllegalArgumentException 。以下代码段显示了如何使用分片键检出连接：
PoolDataSource pds = PoolDataSourceFactory.getPoolDataSource（）; ...Connection conn = pds.createConnectionBuilder（）//使用分片键和超级分片键建立连接.shardingKey（shardingKey）.superShardingKey（superShardingKey）.build（）; OracleShardingKey shardKey = pds.createShardingKeyBuilder（）//使用电子邮件地址和客户ID构建复合分片键作为两个分片键.subkey（<email>，OracleType。VARCHAR2）.subkey（<custid>，OracleType。NUMBER）.build（）; OracleShardingKey superShardKey = pds.createShardingKeyBuilder（）//使用客户区域构建超级分片键.subkey（<cust_region>，OracleType。VARCHAR2）.build（）;

注意：

您必须在连接结帐时指定分片键。否则，将错误或异常抛回应用程序。竞争条件也会在连接使用期间导致异常。

11.2.3关于在不提供分片键的情况下检出连接

当您使用UCP数据源连接到分片数据库时，必须通过连接构建器API在连接请求中提供分片键。如果未提供分片键，则会向用户返回异常。

11.2.4关于连接多个碎片查询的碎片目录或协调器

连接到Shard Catalog或Co-ordinator以运行多个分片查询时，建议使用新的PoolDataSource实例创建单独的池。您可以对从协调器服务上创建的数据源检索的连接运行多个分片查询。协调器的连接请求不应在连接构建器API中具有分片键。

11.2.5关于配置每个Shard的连接数

当UCP用于池化分片数据库的连接时，池包含与不同分片的连接。因此，在拉动连接时，为了确保在连接的所有分片中合理使用池容量，UCP使用MaxConnectionsPerShard参数。这是一个全局参数，适用于分片数据库中的每个分片，用于将任何分片的连接总数限制为低于指定的限制。

11.2.6连接检出期间的池连接选择算法

每当通过UCP将新连接创建到分片数据库中的不同分片时，池就会逐步学习并在内部构建分片路由缓存。

路由缓存将分片密钥映射到密钥存在的相应分片。在池中查找具有特定分片键的连接请求的连接时，UCP使用缓存将请求重定向到正确的分片。此功能称为快速路径连接借用，可根据请求的分片键有效地重用池中的连接。此功能还有助于避免转到分片数据库以路由请求。

11.2.7 UCP中的故障转移或重新分配事件处理

在重新分片或故障转移事件之后，尝试使UCP分片路由缓存与服务器上的数据保持同步。通过为数据库上的各种更改订阅ONS通知，缓存保持最新。

2.13创建脚本

目的

使用CREATE SCRIPT命令在恢复目录中创建存储的脚本。存储的脚本是一系列RMAN命令，它们被赋予一个名称并存储在恢复目录中以供以后执行。

也可以看看：

  • Oracle数据库备份和恢复用户指南 ，了解如何使用存储的脚本

  • REPLACE SCRIPT以了解如何更新存储的脚本

先决条件

仅在RMAN提示符下执行CREATE SCRIPT 。RMAN必须连接到目标数据库和恢复目录。必须打开恢复目录数据库。

如果指定了GLOBAL ，则恢复目录中不得存在具有此名称的全局脚本。如果未指定GLOBAL ，则必须不存在具有相同目标数据库相同名称的本地脚本。如果您不满足这些先决条件，则RMAN将返回错误RMAN-20401 。

使用说明

存储的脚本可以是本地脚本或全局脚本。仅为当前目标数据库创建本地脚本，而全局脚本可用于在恢复目录中注册的任何数据库。

允许创建与本地脚本同名的全局脚本，或者与全局脚本同名的本地脚本。

存储脚本中的替换变量

RMAN支持在存储的脚本中使用替换变量。 &1表示放置第一个值的位置， &2表示放置第二个值的位置，依此类推。必须引用特殊字符。

替换变量语法是& integer后跟可选的句点，例如&1.3 。可选句点是变量的一部分，并替换为值，从而使替换文本后面紧跟另一个整数。例如，如果将值mybackup传递给包含替换变量&1.3的命令文件，则替换的结果为mybackup3 。要创建结果mybackup.3 ，请在语法&1..3使用两个句点。

使用替换变量创建存储脚本时，必须在创建时提供示例值。您可以在启动RMAN时USING子句提供这些值（请参阅RMAN ），或在提示时输入它们（请参阅示例2-74 ）。

卸载Oracle Database Vault

您可以从Oracle数据库安装中删除Oracle Database Vault，以用于单实例和Oracle RAC安装。

但是，您无法从多租户环境中的数据库卸载Database Vault。此过程仅适用于旧的非CDB Oracle数据库环境。

卸载过程不会影响初始化参数设置，即使是在安装过程中修改的设置，也不会影响Oracle Label Security。

  1. 以具有SYSDBA管理权限的用户SYS或具有ALTER SYSTEM系统特权的用户身份登录数据库实例。

    例如：

    sqlplus psmith - 或者，sqlplus psmith @hrpdb用于可插拔数据库（PDB）输入密码： password
  2. 确保已禁用回收站。
    SHOW PARAMETER RECYCLEBIN
  3. 如果回收站已打开，则使用以下语句之一禁用它：
    ALTER SYSTEM SET RECYCLEBIN = OFF; ALTER SESSION SET recyclebin = OFF SCOPE = SPFILE;
  4. 以已被授予DV_OWNER或DV_ADMIN角色的用户身份进行连接。

    例如：

    CONNECT leo_dvowner - 或者，CONNECT leo_dvowner @hrpdb输入密码： 密码
  5. 运行以下过程以禁用Oracle Database Vault：
    EXEC DBMS_MACADM.DISABLE_DV;
  6. 以SYS身份连接SYSOPER权限，然后重新启动数据库。

    例如：

    将SYS作为SYSOPER连接- 或者，连接SYS @hrpdb AS SYSOPER输入密码： password SHUTDOWN IMMEDIATE STARTUP

    对于Oracle RAC安装，请关闭然后重新启动每个数据库实例，如下所示：

    $ srvctl stop database -db db_name $ srvctl start database -db db_name
  7. 运行dvremov.sql脚本以删除Oracle Database Vault。

    例如：

SUBMULTISET条件测试指定的嵌套表是否是另一个指定嵌套表的子数据集。

运算符返回一个布尔值。当nested_table1是nested_table1的子nested_table2时，返回TRUE 。当出现下列情况之一时， nested_table1是nested_table1的子nested_table2 ：

  • nested_table1不为null且不包含任何行。即使nested_table2为null，也会返回TRUE因为空多重集是nested_table2的任何非null替换的子nested_table2 。

  • nested_table1和nested_table2不为null， nested_table1不包含null元素，并且nested_table1的每个元素与nested_table1中的相等元素之间存在nested_table2 。

发生以下某种情况时返回NULL ：

  • nested_table1为null。

  • nested_table2为null， nested_table1不为null且不为空。

  • 在将nested_table1和nested_table2每个null元素修改为某个非null值之后， nested_table1是nested_table1的子nested_table2 ，从而实现nested_table1中每个元素与nested_table1中的相等元素的nested_table2 。

13.18游标变量声明

游标变量类似于显式游标，不限于一个查询。

要创建游标变量，请声明预定义类型SYS_REFCURSOR的变量或定义REF CURSOR类型，然后声明该类型的变量。

游标变量的限制

  • 您不能在游标FOR LOOP语句中使用游标变量。

  • 您不能在包规范中声明游标变量。

    也就是说，包不能具有公共游标变量（可以从包外引用的游标变量）。

  • 您不能将游标变量的值存储在集合或数据库列中。

  • 您不能使用比较运算符来测试游标变量的相等性，不等式或无效性。

  • 在服务器到服务器远程过程调用（RPC）中使用游标变量会导致错误。但是，如果远程数据库是通过Procedural Gateway访问的非Oracle数据库，则可以在服务器到服务器RPC中使用游标变量。

    installActions date_time .log

    $ tail -50 installActions date_time .log |更多
