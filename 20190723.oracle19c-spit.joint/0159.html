204.1 DBMS_XMLSTORAGE_MANAGE概述

DBMS_XMLSTORAGE_MANAGE包含在架构注册完成后管理和修改XML存储的过程。

使用此程序包中的子程序可以提高批量加载操作的性能。您可以在执行批量加载过程之前禁用索引和约束，然后再启用它们。

204.2 DBMS_XMLSTORAGE_MANAGE安全模型

由XDB拥有，DBMS_XMLSTORAGE_MANAGE包必须由SYS或XDB创建。 EXECUTE特权被授予PUBLIC 。

此包中的子程序使用当前用户的权限执行。

传递XMLTYPE表

对于XMLType表，必须传递要在其上执行批量加载操作的XMLType表名。对于XMLType列，您必须传递关系表名称和相应的XMLType列名称。

使用clear启用和禁用索引和约束

注意：

如果使用clear设置为TRUE调用DISABLEINDEXESANDCONTRAINTS过程，它将删除有关在过程的早期执行期间存储的XMLTYPE表或列的任何索引或约束信息。

因此，在调用DISABLEINDEXESANDCONTRAINTS过程并将clear设置为TRUE之前，必须确保在表或列上重新启用所有禁用的索引和约束。

理想情况下，建议您在第一次执行时将clear set设置为TRUE 。对于任何后续执行（由于禁用或启用索引时的错误）， clear应设置为FALSE ，即默认值。在批量加载操作之后成功重新启用所有索引和约束后，可以再次调用此过程，并将clear设置为TRUE以进行下一次批量加载操作。

例

以下示例说明了在DISABLEINDEXESANDCONSTRAINTS过程和ENABLEINDEXESANDCONSTRAINTS过程中使用clear 。

首先，在PURCHASEORDER_TAB表的comment元素上添加一个非NULL约束：

ALTER TABLE PURCHASEORDER_TAB ADD CONSTRAINT c1 check（“XMLDATA”。“comment”IS NOT NULL）;

然后，通过调用DISABLEINDEXESANDCONSTRAINTS过程将clear传递为TRUE禁用所有索引和约束：

BEGIN XDB.DBMS_XMLSTORAGE_MANAGE.DISABLEINDEXESANDCONSTRAINTS（USER，'PURCHASEORDER_TAB'，NULL，TRUE）;结束; /

接下来，执行违反ALTER表语句中的约束c1的批量加载操作（例如数据泵导入）。这不会引发错误，因为禁用了约束：

host impdp orexample / orexample directory = dir dumpfile = dmp.txt tables = OREXAMPLE.PURCHASEORDER_TAB content = DATA_ONLY;

注意：要查看禁用的约束和索引，请使用：

SELECT constraint_name，table_name，status FROM all_constraints WHERE owner = user;

最后，尝试使用ENABLEINDEXESANDCONSTRAINTS过程启用约束。它引发了一个错误，因为批量加载操作违反了c1 （非空约束）：

BEGIN XDB.DBMS_XMLSTORAGE_MANAGE.ENABLEINDEXESANDCONSTRAINTS（USER，'PURCHASEORDER_TAB'）;结束; /

要禁用所有索引和约束，请再次使用DISABLEINDEXESANDCONSTRAINTS ，但设置clear= FALSE （因为ENABLEINDEXESANDCONSTRAINTS未能成功完成）。注意：默认情况下， clear = FALSE ，因此我们不需要在下次调用中明确传递它。 

BEGIN xdb。DBMS_XMLSTORAGE_MANAGE.DISABLEINDEXESANDCONSTRAINTS（USER，'PURCHASEORDER_TAB'）;结束; /

然后，删除输入表中的错误行

DELETE FROM purchaseorder_tab p WHERE p.xmldata。“comment”IS NULL;

使用ENABLEINDEXESANDCONSTRAINTS重新启用索引和约束，并成功完成。 

BEGIN xdb。DBMS_XMLSTORAGE_MANAGE.ENABLEINDEXESANDCONSTRAINTS（USER，'PURCHASEORDER_TAB'）;结束; /

89.1 DBMS_JOB安全模型

使用DBMS_JOB不需要特定的系统特权。没有系统权限可用于管理DBMS_JOB 。除了用户拥有的作业之外，不能更改或删除作业。对所有用户都是如此，包括那些被授予DBA权限的用户。

您可以执行用户拥有的程序或明确授予用户EXECUTE 。但是，无法执行通过角色授予用户执行权限的过程。

请注意，一旦作业启动并运行，就没有简单的方法可以停止作业。

89.2 DBMS_JOB操作说明

这些说明描述了停止工作以及使用Oracle Real Application Clusters。

停止工作

请注意，一旦作业启动并运行，就没有简单的方法可以停止作业。

使用Oracle Real Application Clusters

DBMS_JOB支持多实例作业执行。默认情况下，可以在任何实例上执行作业，但只有一个实例将执行作业。此外，您可以通过将作业绑定到特定实例来强制实例绑定。您可以通过为实例关联参数指定实例编号来实现实例绑定。但请注意，在Oracle数据库10g第1版（10.1）中，不建议使用实例绑定。服务亲和力是首选。此概念在DBMS_SCHEDULER包中实现。

以下过程可用于创建，更改或运行具有实例关联的作业。请注意，不指定关联性意味着任何实例都可以运行该作业。

  • DBMS_JOB.SUBMIT

  • DBMS_JOB.INSTANCE

  • DBMS_JOB.CHANGE

  • DBMS_JOB.RUN

DBMS_JOB.SUBMIT

要将作业提交到作业队列，请使用以下语法：

DBMS_JOB.SUBMIT（作业OUT BINARY_INTEGER，IN VARCHAR2，next_date IN DATE DEFAULT SYSDATE，interval IN VARCHAR2 DEFAULT'NULL'，no_parse IN BOOLEAN DEFAULT FALSE，实例IN BINARY_INTEGER DEFAULT ANY_INSTANCE，强制IN BOOLEAN DEFAULT FALSE）;

使用参数instance和 force控制作业和实例关联。instance的默认值为0（零），表示任何实例都可以执行作业。要在特定实例上运行作业，请指定instance值。如果instance值为负数或NULL. Oracle将显示错误ORA-23319 NULL.

force参数默认为false. 如果force为TRUE,任何正整数都可以作为作业实例。如果force为FALSE,则指定的实例必须正在运行，或者Oracle显示错误号ORA-23428.

DBMS_JOB.INSTANCE

要分配特定实例以执行作业，请使用以下语法：

DBMS_JOB.INSTANCE（JOB IN BINARY_INTEGER，实例IN BINARY_INTEGER，强制IN BOOLEAN DEFAULT FALSE）;

此示例中的FORCE参数默认为FALSE.如果实例值为0（零），则会更改作业关联，并且尽管有force值，任何可用实例都可以执行作业。如果INSTANCE值为正且FORCE参数为FALSE,则仅在指定实例正在运行时更改作业关联，或者Oracle显示错误ORA-23428.

如果force参数为TRUE ，则任何正整数都可以作为作业实例并更改作业关联。如果instance值为负或NULL Oracle将显示错误ORA-23319。

DBMS_JOB.CHANGE

要更改与作业关联的用户可定义参数，请使用以下语法：

DBMS_JOB.CHANGE（JOB IN BINARY_INTEGER，IN VARCHAR2 DEFAULT NULL，next_date IN DATE DEFAULT NULL，interval IN VARCHAR2 DEFAULT NULL，实例IN BINARY_INTEGER DEFAULT NULL，强制IN BOOLEAN DEFAULT FALSE）;

此示例中显示了两个参数instance和force, 。instance的默认值为null表示作业关联性不会更改。

force的默认值为FALSE.甲骨文显示错误ORA-23428如果指定的实例没有运行和错误ORA-23319 ，如果instance数是负的。

DBMS_JOB.RUN

DBMS_JOB.RUN的force参数默认为FALSE.如果force为TRUE,实例关联与在前台进程中运行作业无关。如果force为FALSE ，则作业只能在指定实例中的前台运行。如果force为FALSE并且连接的实例是不正确的实例，Oracle会显示错误ORA-23428 。 

DBMS_JOB.RUN（作业IN BINARY_INTEGER，强制进入BOOLEAN DEFAULT FALSE）;

传统数据库用户拥有数据库模式，可以为这些模式创建传统的重量级数据库会话。

    从员工中选择COUNT（*）;

    BEGIN SYS.XS_PRINCIPAL.SET_PASSWORD（'lwuser1'，' password '）;结束; /

关于应用程序角色

应用程序角色是只能授予应用程序用户或其他应用程序角色的角色。应用程序角色为分组应用程序用户提供了一种方法，这些用户必须具有在ACL中标识的公共应用程序特权才能访问应用程序。XS_PRINCIPAL.CREATE_ROLE过程可以创建常规应用程序角色。XS_PRINCIPAL.CREATE_DYNAMIC_ROLE过程可以创建动态应用程序角色（一种类型的应用程序角色）。

应用程序角色在概念上类似于企业角色。企业角色只能授予企业用户，并且授权发生在数据库之外。同样，应用程序角色只能授予应用程序用户或应用程序角色，并且该授权发生在标准数据库授予机制之外。动态角色不能授予应用程序用户或其他应用程序角色，但只能在应用程序会话中作为附加会话调用中的参数启用，如“ 动态应用程序角色 ”中所述 。

常规应用角色

一个常规应用程序角色是您可以授予应用程序用户或其他应用程序角色（常规或动态）的应用程序角色。您可以指定是否要在默认情况下启用常规应用程序角色。

动态应用程序角色

动态应用程序角色是仅在某些情况下启用的应用程序角色，例如，当用户使用SSL登录时，或在特定时间段内，等等。例如，如果在工作日期间为所有连接的应用程序用户授予了某些应用程序权限，则可以使用动态应用程序角色。如果满足该标准，则应用程序将启用这些应用程序角色。

应用程序确定启用动态应用程序角色的条件，但应用程序或应用程序请求时，应用程序或数据库可以评估条件。

  • 当应用程序评估标准时

    如果应用程序评估条件并且应用程序角色满足它，则应用程序（如果它附加到应用程序会话）可以为应用程序用户启用动态应用程序角色。当应用程序从应用程序会话中分离时，将自动禁用动态应用程序角色。

    出于安全原因，您无法在会话期间禁用动态应用程序角色。这一点尤为重要，因为它们可能会推断出负面的应用程

  • 当数据库评估标准时

    如果数据库评估条件并且应用程序角色满足条件，则数据库可以为应用程序用户启用应用程序角色。数据库可以基于两种类型的超时来禁用动态应用程序角色：一次是从上次访问会话时开始，一次是从上一次会话身份验证开始。首次连接会话时，Oracle数据库会检查这些超时。

您不需要事先正式向用户授予动态应用程序角色。无法通过标准启用和禁用API启用或禁用动态应用程序角色。您无法将动态应用程序角色授予其他应用程序角色，但您可以将其他应用程序角色授予动态角色。

86.1 DBMS_INMEMORY_ADMIN概述

该软件包提供用于管理内存中表达式（IM表达式）和内存中快速启动（IM FastStart）区域的接口。

IM表达式

分析查询通常包含在执行期间消耗大量CPU和内存的复杂表达式或计算。使用IME_CAPTURE_EXPRESSIONS标识这些常用（“热”）表达式和IME_POPULATE_EXPRESSIONS以在IM列存储中填充它们。通过使用IM表达式，数据库避免了重复计算并提高了性能。

数据库将IM表达式表示为系统生成的虚拟列。IM虚拟列的名称以SYS_IME 。您还可以使用DBMS_INMEMORY_ADMIN.IME_DROP_ALL_EXPRESSIONS和DBMS_INMEMORY.IME_DROP_EXPRESSIONS删除现有的SYS_IME列。

DBA_IM_EXPRESSIONS视图显示具有INMEMORY属性的SYS_IME列。使用IME_CAPTURE_EXPRESSIONS过程后，您可以查询此视图以查看添加到数据库中不同表的热表达式。

也可以看看：

Oracle Database In-Memory指南了解有关IM表达式的更多信息

IM FastStart区域

IM FastStart区域存储在数据库重新启动时优化IM列存储的填充的数据。由于数据库直接从持久存储中读取列数据而无需压缩或格式化，因此数据库实例重新启动时填充速度会更快。

为IM列存储启用IM FastStart时，必须为IM FastStart区域指定ASSM表空间。表空间将数据存储在名为SYSDBIMFS_LOGSEG$的SecureFiles LOB中。SYSAUX表空间存储元数据。在IM列存储中填充或重新填充数据时，数据库会自动将数据写入IM FastStart区域。您无法手动强制写入。如果将对象指定为NO INMEMORY ，则数据库会将其从IM FastStart区域中删除。

当IM FastStart区域处于空间压力下时，数据库会自动删除最旧的15％的段并继续保存列数据。如果空间不可用，则数据库将停止写入IM FastStart区域。

    数据库为在最近窗口中跟踪的表达式添加隐藏的虚拟列。如果捕获窗口当前处于打开状态，则数据库将考虑当前窗口中跟踪的所有表达式，直到此时为止，然后实现最热门的表达式。要列出在当前窗口中跟踪的表达式， DBA_EXPRESSION_STATISTICS使用SNAPSHOT='WINDOW'查询DBA_EXPRESSION_STATISTICS 。

使用说明

调用此过程时，数据库将查询表达式统计信息存储（ESS），并仅考虑至少部分填充在IM列存储中的表上的表达式。数据库将20个最热门的表达式作为隐藏的虚拟列添加到其各自的表中，前缀为字符串SYS_IME ，并应用默认的INMEMORY列压缩子句。如果在上一次调用期间添加的任何SYS_IME列不再位于最新的前20个列表中，则数据库NO INMEMORY它们标记为NO INMEMORY 。

注意：

在备用数据库上执行IME_CAPTURE_EXPRESSIONS过程无效。

无论属性是INMEMORY还是NO INMEMORY ，表的最大SYS_IME列数为50。达到表的限制后，数据库将不会添加新的SYS_IME列。要为新表达式腾出空间，必须使用IME_DROP_ALL_EXPRESSIONS过程或IME_DROP_EXPRESSIONS过程手动删除SYS_IME列。

每个表的50表达式限制（包括INMEMORY和NO INMEMORY表达式）与数据库的20表达式限制不同，后者仅包含INMEMORY表达式。例如，如果在IM列存储中填充了20个表，则每个表可能都有1 SYS_IME带有INMEMORY属性的SYS_IME列，以及带有NO INMEMORY属性的49个SYS_IME列。

IM表达式和虚拟列存储在称为内存中表达单元（IMEU）的内存中结构中。每个IMEU都链接到父内存压缩单元（IMCU），从中继承压缩特性。

ESS信息存储在数据字典中，并在DBA_EXPRESSION_STATISTICS视图中公开。此视图显示优化程序在ESS中收集的元数据。 IM表达式在DBA_IM_EXPRESSIONS视图中以系统生成的虚拟列（以字符串SYS_IME为前缀） DBA_IM_EXPRESSIONS 。

参数 描述

  • ORA-00600
  • ORA-07445

  • ORA-04031

关于Oracle ASM的自动内存管理

自动内存管理使用MEMORY_TARGET参数自动管理Oracle ASM和数据库实例的内存相关参数。默认情况下，在Oracle ASM实例上启用自动内存管理，即使未明确设置MEMORY_TARGET参数也是如此。对于大多数环境， MEMORY_TARGET使用的默认值是可接受的。这是您必须为完整的Oracle ASM内存管理设置的唯一参数。Oracle强烈建议您对Oracle ASM使用自动内存管理。

一个Oracle ASM实例可以自动增加设定的值MEMORY_TARGET和MEMORY_MAX_TARGET如果引发ORA-04031错误和自动内存管理功能。如果MEMORY_MAX_TARGET已明确设置的值，那么每ORA-04031升高时， MEMORY_TARGET值增加由现有的10％ MEMORY_TARGET值或128 MB，以较高者为准，但不得低于客户指定更大MEMORY_MAX_TARGET值。如果MEMORY_MAX_TARGET没有明确设置，则两个MEMORY_TARGET和MEMORY_MAX_TARGET增加由现有的10％ MEMORY_TARGET值或128 MB，取大值，为最多五个增加。Oracle ASM实例必须重新启动才能使用新的MEMORY_TARGET和MEMORY_MAX_TARGET设置。

如果没有为MEMORY_TARGET设置值，但是您为其他与内存相关的参数设置了值，则Oracle MEMORY_TARGET根据这些内存参数值在内部计算MEMORY_TARGET的最佳值。您也可以增加MEMORY_TARGET动态，达价值MEMORY_MAX_TARGET参数，就像你可以为数据库实例做。

虽然不建议这样做，但可以通过在Oracle ASM参数文件MEMORY_TARGET的值设置为0或运行ALTER SYSTEM SET MEMORY_TARGET =0语句来禁用自动内存管理。禁用自动内存管理时，Oracle将恢复自动共享内存管理和自动PGA内存管理。要恢复到Oracle数据库10g第2版（10.2）功能以手动管理Oracle ASM SGA内存，还要运行ALTER SYSTEM SET SGA_TARGET=0语句。除非指定，否则Oracle ASM实例中自动内存管理参数的行为与Oracle数据库实例中的行为相同。

注意：

  • 对于Linux环境，如果/dev/shm不可用或尺寸不足，则自动内存管理无法工作。有关更多信息，请参阅适用于Linux和基于UNIX的操作系统的Oracle数据库管理员参考 。有关支持自动内存管理的平台的信息，请参见“ Oracle数据库管理员指南” 。

  • Oracle ASM的最小MEMORY_TARGET为1 GB。如果将MEMORY_TARGET设置MEMORY_TARGET较低，则Oracle会自动将MEMORY_TARGET的值MEMORY_TARGET 1 GB。

ASM_DISKGROUPS

ASM_DISKGROUPS初始化参数指定在发出SQL ALTER DISKGROUP ALL MOUNT语句时，Oracle ASM实例在启动时安装的磁盘组名称列表。

除非指定了NOMOUNT启动选项，否则Oracle ASM实例启动过程将执行ALTER DISKGROUP ALL MOUNT 。

ASM_DISKGROUPS参数的默认值是NULL字符串。

ASM_DISKGROUPS参数是动态的。如果使用服务器参数文件（SPFILE），则在大多数情况下不必手动更改ASM_DISKGROUPS的值。成功创建或装入磁盘组后，Oracle ASM会自动将磁盘组添加到参数中。删除磁盘组时，Oracle ASM还会自动从参数中删除磁盘组。但是，SPFILE不会在手动卸载时更新。

注意：

在Oracle Flex ASM配置中，卸载时不会从SPFILE中删除磁盘组。必须始终自动挂载磁盘组，以便远程实例可以访问它们。远程实例是位于与Oracle ASM实例不同的节点上的Oracle数据库，Oracle IOServer或Oracle ASM代理实例。

以下是动态设置ASM_DISKGROUPS参数的示例：

SQL> ALTER SYSTEM SET ASM_DISKGROUPS = DATA，FRA;

使用文本初始化参数文件（PFILE）时，您可以编辑初始化参数文件以添加任何磁盘组的名称，以便在实例启动时自动装入。您必须删除不再需要自动挂载的任何磁盘组的名称。

以下是初始化文件中ASM_DISKGROUPS参数的示例：

ASM_DISKGROUPS = DATA, FRA

注意：

发行ALTER DISKGROUP ......ALL MOUNT或ALTER DISKGROUP ...ALL DISMOUNT命令不会影响ASM_DISKGROUPS的值。

对于Oracle Database 12 c第1版或更高版本，Oracle ASM配置最多支持511个磁盘组。在12 c版本1之前的Oracle数据库版本的Oracle ASM配置最多只能支持63个磁盘组。

ASM_DISKSTRING

ASM_DISKSTRING初始化参数指定以逗号分隔的字符串列表，这些字符串限制Oracle ASM实例发现的磁盘集。

发现字符串可以包含通配符。仅发现与其中一个字符串匹配的磁盘。无法同时发现两次相同的磁盘。

发现字符串格式取决于Oracle ASM库和正在使用的操作系统。支持模式匹配。有关默认模式匹配的信息，请参阅特定于操作系统的安装指南。

例如，在Linux服务器上，要将发现过程限制为仅包含不使用Oracle ASM过滤器驱动程序（Oracle ASMFD）或ASMLIB的Oracle ASM实例的/dev/rdsk/mydisks目录中的磁盘，请设置ASM_DISKSTRING初始化参数：

的/ dev / rdsk / mydisks / *

星号是必需的。

要将发现过程限制为仅包含名称以disk3或disk4结尾的磁盘，可以在Linux系统上按如下方式设置ASM_DISKSTRING ：

ASM_DISKSTRING = '/dev/rdsk/*disk3' ， '/dev/rdsk/*disk4'

的? 字符用作路径的第一个字符时，会扩展到Oracle主目录。根据操作系统的不同，何时使用? 路径中其他位置的字符，它是一个字符的通配符。

ASM_DISKSTRING参数的默认值是NULL字符串。NULL值会导致Oracle ASM搜索Oracle ASM实例具有读写访问权限的系统中所有磁盘的缺省路径。默认搜索路径是特定于平台的。有关默认搜索路径的详细信息，请参阅特定于操作系统的安装指南。

除非群集中的所有Oracle ASM实例都可以通过其自己的发现字符串之一发现磁盘，否则Oracle ASM无法使用磁盘。每个节点上的名称不必相同，但群集中的所有节点都必须能够发现所有磁盘。这可能需要动态更改初始化参数以启用添加新存储。

ASM_POWER_LIMIT

ASM_POWER_LIMIT初始化参数指定磁盘组中磁盘重新平衡的默认功率。

值的范围是0到1024 。默认值为1 。值为0将禁用重新平衡。较高的数值使重新平衡操作能够更快地完成，但可能会导致更高的I / O开销和更多的重新平衡过程。

  • 对于磁盘组ASM兼容性设置为11.2.0.2或更高的磁盘组（例如， COMPATIBLE.ASM = 11.2.0.2 ），重新平衡功率的值的操作范围为0到1024 。

  • 对于磁盘组ASM兼容性设置为小于11.2.0.2的磁盘组，值的操作范围为0到11包括0和11 。如果ASM_POWER_LIMIT的值大于11 ，则对这些磁盘组使用值11 。

您还可以使用SQL ALTER DISKGROUP的POWER子句指定磁盘组中重新平衡操作的DISKGROUP 。REBALANCE声明。POWER子句的允许值范围对于ASM_POWER_LIMIT初始化参数是相同的。如果为ASM兼容性设置为小于11.2.0.2的磁盘组指定的POWER子句的值大于11 ，则会显示警告，并且将使用等于11的POWER值进行重新平衡。

磁盘组中重新平衡操作的功能规范仅影响重新平衡操作，而不影响磁盘组的新分配。

注意：

  • Oracle建议您不要将ASM_POWER_LIMIT初始化参数设置为0 。

  • 您无法在Oracle Exadata或Oracle数据机环境中将ASM_POWER_LIMIT初始化参数设置为0 。

决策树是表示导致类或价值的一系列问题的一种方式。决策树的顶级节点称为根节点;终端节点称为叶节点。决策树通过将数据迭代分割成离散组来增长，其目标是最大化每个分组处的组之间的距离。

决策树模型的一个重要特征是它们是透明的;也就是说，有一些规则可以解释分类。

PL / SQL是SQL的Oracle过程扩展，是一种可移植的高性能事务处理语言。本概述解释了其优点，并简要介绍了其主要特性和架构。

1.1 PL / SQL的优点

PL / SQL与其他编程语言相比具有几个优点。

PL / SQL具有以下优点：

  • 与SQL紧密集成

1.1.1与SQL紧密集成

PL / SQL与SQL（最广泛使用的数据库操作语言）紧密集成。

例如：

  • PL / SQL允许您使用所有SQL数据操作，游标控制和事务控制语句，以及所有SQL函数，运算符和伪列。

  • PL / SQL完全支持SQL数据类型。

    您无需在PL / SQL和SQL数据类型之间进行转换。例如，如果PL / SQL程序从SQL类型VARCHAR2的列中检索值，则它可以将该值存储在VARCHAR2类型的PL / SQL变量中。

    您可以为PL / SQL数据项提供数据库表的列或行的数据类型，而无需显式指定该数据类型（请参阅“ 使用％TYPE属性 ”和“ 使用％ROWTYPE属性 ” ）。

  • PL / SQL允许您运行SQL查询并一次处理一个结果集的行 （请参阅“一次处理一行查询结果集 ” ）。

  • 可以在SQL SELECT语句的WITH子句中声明和定义PL / SQL函数（请参阅Oracle数据库SQL语言参考 ）。

PL / SQL支持静态和动态SQL。 静态SQL是SQL，其全文在编译时是已知的。动态SQL是SQL，其全文在运行时才知道。动态SQL使您可以使应用程序更加灵活和通用。有关更多信息，请参阅PL / SQL静态SQL和PL / SQL动态SQL 。

1.1.2高性能

PL / SQL允许您向数据库发送一个语句块，从而显着减少应用程序和数据库之间的流量。

绑定变量

当您在PL / SQL代码中直接嵌入SQL INSERT ， UPDATE ， DELETE ， MERGE或SELECT语句时，PL / SQL编译器将WHERE和VALUES子句中的变量转换为绑定变量（有关详细信息，请参阅“ 名称解析”）在静态SQL语句中 “ ）。每次运行相同的代码时，Oracle数据库都可以重用这些SQL语句，从而提高性能。

使用动态SQL时，PL / SQL不会自动创建绑定变量，但您可以通过动态SQL明确指定它们来使用它们（有关详细信息，请参阅“ EXECUTE IMMEDIATE语句 ” ）。

子程序

PL / SQL子程序以可执行的形式存储，可以重复调用。由于存储的子程序在数据库服务器中运行，因此通过网络进行的单个调用可以启动大型作业。这种分工可以减少网络流量并缩短响应时间。存储的子程序在用户之间进行缓存和共享，从而降低了内存需求和调用开销。有关子程序的详细信息，请参阅“ 子程序 ” 。

优化

PL / SQL编译器有一个优化器，可以重新排列代码以获得更好的性能。有关优化程序的更多信息，请参阅“ PL / SQL优化程序 ” 。

1.2.1错误处理

PL / SQL使检测和处理错误变得容易。

发生错误时，PL / SQL会引发异常。正常执行停止并控制传输到PL / SQL块的异常处理部分。您不必检查每个操作以确保它成功，就像在C程序中一样。

有关更多信息，请参阅PL / SQL错误处理 。

1.2.2块

PL / SQL源程序的基本单元是块 ，它对相关的声明和语句进行分组。

PL / SQL块由关键字DECLARE ， BEGIN ， EXCEPTION和END 。这些关键字将块分为声明部分，可执行部分和异常处理部分。只需要可执行部分。块可以有标签。

声明是块的本地声明，并且在块完成执行时不再存在，有助于避免变量和子程序的混乱命名空间。

块可以嵌套：因为块是可执行语句，所以只要允许可执行语句，它就可以出现在另一个块中。

您可以将块提交给交互式工具（例如SQL * Plus或Enterprise Manager），也可以将其嵌入Oracle预编译器或OCI程序中。交互式工具或程序运行块一次。该块未存储在数据库中，因此，它被称为匿名块 （即使它具有标签）。

每次将匿名块加载到内存时都会编译它，并且它的编译有三个阶段：

  1. 语法检查：检查PL / SQL语法，并生成解析树。

  2. 语义检查：在解析树上键入检查和进一步处理。

  3. 代码生成

注意：

匿名块是SQL语句。

有关语法详细信息，请参阅“ 阻止 ” 。

对于完整行或部分行的每一列，记录都有一个具有相同名称和数据类型的字段。如果行的结构发生变化，则记录的结构会相应地发生变化。

有关%ROWTYPE语法和语义的详细信息，请参阅“ ％ROWTYPE属性 ” 。有关其用法的更多详细信息，请参阅“ 使用％ROWTYPE属性声明项目 ” 。

1.2.8.4使用％TYPE属性

%TYPE属性允许您声明与先前声明的变量或列具有相同数据类型的数据项（不知道该类型是什么）。

如果引用项的声明发生更改，则引用项的声明会相应更改。在声明用于保存数据库值的变量时， %TYPE属性特别有用。有关%TYPE语法和语义的详细信息，请参阅“ ％TYPE属性 ” 。有关其用法的更多详细信息，请参阅“ 使用％TYPE属性声明项目 ” 。

2.1关于Oracle Instant Client

Oracle Instant Client提供必要的Oracle数据库客户端文件，以创建和运行OCI，OCCI，ODBC和JDBC OCI应用程序。还提供有用的命令行实用程序，包括SQL * Plus，SQL * Loader和Oracle Data Pump。Oracle Instant Client消除了客户端计算机上对Oracle主目录的需求，从而简化了应用程序的部署。

与在完整客户端安装中运行的同一应用程序相比，在Instant Client模式下运行的OCI应用程序的存储空间要求显着降低。最小安装的Instant Client共享库可占用完整客户端安装的不到10％的空间。最后，Instant Client易于安装。

为何使用Instant Client？

  • 安装涉及解压缩一些文件或安装RPM软件包。

  • Oracle数据库客户端所需文件数量和总磁盘存储量显着减少。

  • 使用Instant Client部署的应用程序不会丧失功能或性能。

  • 独立软件供应商很容易打包应用程序。

Instant Client安装中包含README文件。它描述了生成Instant Client的版本，日期和时间以及操作系统。

可以通过多种方式安装Oracle Instant Client。

从Oracle技术网站下载并安装Instant Client库

以下步骤适用于Linux，Unix和Windows环境。

  1. 从Oracle技术网 ，下载并将所需的ZIP包从OTN解压缩到一个空目录，例如instantclient_18_1 ，用于Oracle数据库版本18c，版本18.1。所有Instant Client ZIP安装都需要Basic或Basic Light软件包。

  2. 或者，要安装RPM文件，请按照以下说明操作：

    1. 通过运行su并输入超级用户密码成为root用户。

    2. 切换到要安装的目录。

    3. 卸载任何早期安装。

      #rpm -e package-name

    4. 用yum安装每个包。Yum将自动安装所需的依赖项，例如libaio 。 

      #yum remove package-name #yum install package-name.rpm

      或者，使用rpm安装每个包。

      #rpm -ivh package-name .rpm

    5. 如果要节省磁盘空间，请删除.rpm文件。

    6. 退出root shell。无需重启。

  3. 将操作系统共享库路径环境变量设置为步骤1中的目录。例如，对于Linux或UNIX上的ZIP文件，请将LD_LIBRARY_PATH设置为instantclient_18_1 。
    export LD_LIBRARY_PATH = / opt / oracle / instantclient_18_1：$ LD_LIBRARY_PATH
    使用Linux RPMs，将LD_LIBRARY_PATH设置为：
    export LD_LIBRARY_PATH = / usr / lib / oracle / 18.1 / client64 / lib：$ LD_LIBRARY_PATH

2.6关于Oracle Instant Client Light

Instant Client Light版本的Instant Client进一步减少了客户端安装的磁盘空间要求。

2.6.3安装Oracle Instant Client Light

如何安装Instant Client Light。

Instant Client Light可以通过以下方式之一安装：

  • 安装在Oracle技术网（OTN）上

    以下URL是Oracle Technology Network网站上的Instant Client页面： Oracle Instant Client

    对于Instant Client Light，请将basiclite.zip包下载并解压缩到空的instantclient_12_2目录中。

  • 从Instant Client管理员安装中安装

    从ORACLE_HOME/instantclient/light子目录中，复制libociicus.so （或Windows上的oraociicus12.dll ）。LD_LIBRARY_PATH上的Instant Client目录（Windows上的PATH ）应包含Instant Client Light数据共享库libociicus.so （Windows上的oraociicus12.dll ），而不是更大的OCI Instant Client数据共享库libociei.so （ oraociei12.dll在Windows上的oraociei12.dll ）。

  • 从Oracle Universal Installer安装中安装

    当您从Oracle Universal Installer中选择Instant Client选项时， libociei.so （或Windows上的oraociei12.dll ）安装在安装的基本目录中，这意味着这些文件放在LD_LIBRARY_PATH （Windows上的PATH ）上。

    Instant Light客户端数据共享库libociicus.so （或Windows上的oraociicus12.dll ）安装在基本目录的light子目录中，默认情况下不启用。因此，要在Instant Client Light模式下运行，必须删除或重命名OCI数据共享库libociei.so （或Windows上的Oraociei12.dll ），并且必须将Instant Client Light库从light子目录复制到基本目录的安装。

    例如，如果Oracle Universal Installer已在LD_LIBRARY_PATH （Windows上为PATH ）的my_oraic_12_2目录中安装了Instant Client，则使用以下命令序列在Instant Client Light模式下运行：

    cd my_oraic_12_2 rm libociei.so mv light / libociicus.so。

    注意：

    要确保安装中不存在不兼容的二进制文件，请始终将Instant Client文件复制并安装到空目录中。

2.7关于在Linux或UNIX上修补Oracle Instant Client共享库

由于Instant Client是部署功能，因此运行OCI应用程序所需的文件数量和大小（客户端占用空间）已减少。

因此，在Instant Client部署中无法使用修补Instant Client共享库所需的所有文件。修补需要基于Oracle Home的完整客户端安装。使用opatch实用程序进行修补。

在Oracle主环境中应用修补程序后，将关于Oracle Instant Client中列出的文件复制到即时客户端目录，如从ZIP文件安装，从RPM和Oracle Universal Installer中所述 。

您可以为OCI和OCCI，JDBC和SQL * Plus生成Instant Client zip和RPM文件，而不是复制单个文件，如数据共享库和Zip和RPM文件的再生中所述。然后，您可以将zip和RPM文件复制到目标系统并按照从ZIP文件安装，从RPM和Oracle Universal Installer中的说明解压缩它们。

opatch实用程序将ORACLE_HOME安装的修补信息存储在libclntsh.so 。可以通过以下命令检索此信息：

genezi -v

如果Instant Client部署系统没有genezi实用程序，则可以从ORACLE_HOME /bin 目录中复制它。

注意：

opatch实用程序在Windows上不可用。
