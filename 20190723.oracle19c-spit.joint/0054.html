Oracle Grid Infrastructure升级的限制

查看以下信息，了解Oracle Grid Infrastructure安装升级的限制和更改，其中包括Oracle Clusterware和Oracle自动存储管理（Oracle ASM）。

  • Oracle Grid Infrastructure升级始终是不合适的升级。您无法对现有家庭执行Oracle Grid Infrastructure的就地升级。

  • 拥有早期版本Oracle Grid Infrastructure软件的同一用户必须执行Oracle Grid Infrastructure 19c升级。

  • Oracle ASM和Oracle Clusterware都在Oracle Grid Infrastructure主目录中运行。

  • 升级Oracle Grid Infrastructure时，升级到Oracle Flex Cluster配置。

  • 不要删除Grid home中的目录。例如，不要删除目录Grid_home / OPatch。如果删除目录，则Grid Infrastructure安装所有者无法使用OPatch实用程序修补网格主页，并且OPatch显示错误消息“'checkdir'错误：无法创建Grid_home / OPatch”。

  • 要将现有Oracle Grid Infrastructure安装升级到Oracle Grid Infrastructure 19c，必须首先验证是否需要应用任何强制修补程序才能成功升级。

    Oracle建议您使用Cluster Verification Utility工具（CVU）检查升级现有Oracle Grid Infrastructure或Oracle RAC数据库安装是否需要任何修补程序。有关检查准备情况的步骤，请参阅使用CVU验证Oracle Clusterware升级的准备情况。

  • 在升级完成之前，19c Oracle Grid Infrastructure主页中的软件无法正常运行。在运行最终的rootupgrade.sh脚本并在所有节点上完成升级之前，不支持从新的Grid主目录运行srvctl ， crsctl和其他命令。

    要在Oracle Grid Infrastructure升级期间管理现有早期版本数据库主目录中的数据库，请使用现有数据库主目录中的srvctl 。

  • 要将现有Oracle Clusterware安装升级到Oracle Grid Infrastructure 19c群集，您的版本必须大于或等于Oracle Grid Infrastructure 11g第2版（11.2.0.3）。

关于升级的存储限制

  • 如果当前安装的Oracle Cluster Registry（OCR）和投票文件位置位于原始或块设备上，则必须先将它们迁移到Oracle ASM磁盘组，经过认证的NAS设备或共享文件系统，然后再升级到Oracle Grid Infrastructure 19C。

  • 如果要在Oracle Grid Infrastructure 11g第2版（11.2）之前升级Oracle Grid Infrastructure版本，其中OCR和投票文件位于原始或块设备上，则必须升级到Oracle Grid Infrastructure 12c第1版（12.1），然后移动升级到Oracle Grid Infrastructure 19c之前，Oracle Cluster Registry（OCR）和投票文件到Oracle ASM磁盘组或共享文件系统。

  • 如果您在Oracle Grid Infrastructure 11g第2版（11.2.0.1）上安装了Oracle自动存储管理群集文件系统（Oracle ACFS）文件系统，则可以将Oracle Grid Infrastructure升级到任何更高版本，并且可以利用冗余互连使用并添加一个或者更多私有网络的私有接口，然后必须在每个升级的集群成员节点上重新启动Oracle ASM实例。

关于升级共享网格家园

  • 如果现有的Oracle Clusterware主目录是共享主目录，那么您可以将Oracle Grid Infrastructure的非共享主目录用于Oracle Clusterware和Oracle ASM 19c的集群主目录。

  • 您可以在共享的Oracle Clusterware主目录上执行升级。

关于单实例Oracle ASM升级

  • 在Oracle Grid Infrastructure安装或升级期间，如果本地节点上存在单个实例Oracle ASM版本，则会将其转换为Oracle Flex ASM 19c安装，并且Oracle ASM将在所有节点上的Oracle Grid Infrastructure主目录中运行。

  • 如果单个实例（非群集）Oracle ASM安装位于远程节点上，该节点是本地节点（正在执行Oracle Grid Infrastructure安装或升级的节点）之外的节点，则它仍然是单个实例Oracle ASM安装。但是，在安装或升级期间，将OCR和投票文件放在Oracle ASM上时，将在群集中的所有节点上创建Oracle Flex ASM安装。远程节点上的单实例Oracle ASM安装变得无法运行。

1.3 Oracle JVM概述

Oracle JVM是一个标准的，Java兼容的环境，可以运行任何纯Java应用程序。它与标准JLS和JVM规范兼容。它支持标准Java二进制格式和标准Java API。此外，Oracle数据库遵循标准Java语言语义，包括运行时的动态类加载。

Oracle数据库中的Java引入了以下术语：

  • 会议

    Oracle数据库Java环境中的会话与标准Oracle数据库用法相同。一个会话通常（尽管不一定）受单个用户连接到服务器的限制。作为调用Java代码的用户，您必须在服务器中建立会话。

  • 呼叫

    当用户导致Java代码在会话中运行时，它被称为a 呼叫。可以通过以下不同方式启动呼叫：

    • SQL客户端程序运行Java存储过程。

    • 触发器运行Java存储过程。

    • PL / SQL程序调用Java代码。

    在定义的所有情况下，调用开始，Java，SQL或PL / SQL代码的某些组合运行完成，并且调用结束。

注意：

会话和调用的概念适用于Oracle数据库的所有用途。

与其他Java环境不同，Oracle JVM嵌入在Oracle数据库中，并引入了许多新概念。本节讨论Oracle JVM与典型客户端JVM之间的一些重要差异，基于：

  • Java会话初始化_持续时间和入口点

  • GUI

1.3.1过程域

在标准Java环境中，通过在命令行上发出以下命令，通过解释器运行Java应用程序，其中classname是您希望JVM首先解释的类的名称：

java classname

使用Oracle JVM时，必须将应用程序加载到数据库中，发布接口，然后在数据库会话中运行应用程序。数据库会话是Oracle JVM运行的环境，因此是运行标准客户端JVM的操作系统进程的模拟。

1.3.2 Java会话初始化，持续时间和入口点

标准的基于客户端的Java应用程序声明了一个单一的顶级方法public static void main(String args[]) 。此方法执行一次，Java虚拟机的实例化持续该调用的持续时间。但是，Oracle Java应用程序不限于单个顶级主入口点，并且Oracle JVM实例化的持续时间不是由单个调用确定的，也不是从此入口点退出调用。每个客户端开始一个会话，通过顶级入口点调用其服务器端逻辑模块，并最终结束会话。在整个会话期间，相同的JVM实例保持不变，因此可以在多个顶级入口点的多个调用之间使用静态变量值等数据状态。

已在数据库中加载和发布的类定义通常可用于该数据库中的所有会话。给定会话中的JVM实例以及该JVM实例中的Java数据对象和全局字段值对于会话是私有的。该数据在会话期间存在，并且可以在该会话的生命周期内由多个呼叫使用。但是，这些数据对其他会话都不可见，也不能以任何方式与其他会话共享数据。这类似于在标准客户端Java应用程序中单独调用main方法的方式共享相同的类定义，但在这些调用期间创建和使用的数据是分开的。

1.3.3 GUI

服务器无法提供GUI，但它可以提供驱动它们的逻辑。Oracle JVM仅支持Java Abstract Window Toolkit（AWT）的无头模式。所有Java AWT类都可在服务器环境中使用，并且您的程序可以使用Java AWT功能，只要它们不尝试在服务器上实现GUI即可。

任务2：生成可传输的表空间集

确保您拥有要传输的自包含表空间集后，生成可传输的表空间集。

要生成可传输的表空间集：

  1. 启动SQL * Plus并以管理员或具有ALTER TABLESPACE或MANAGE TABLESPACE系统特权的用户身份连接到数据库。
  2. 将集合中的所有表空间设置为只读。
    ALTER TABLESPACE sales_1只读; ALTER TABLESPACE sales_2只读;
  3. 以具有DATAPUMP_EXP_FULL_DATABASE角色的用户身份运行Data Pump导出实用程序，并在可传输集中指定表空间。
    SQL> HOST $ expdp user_name dumpfile = expdat.dmp directory = data_pump_dir transport_tablespaces = sales_1，sales_2 logfile = tts_export.log密码： password

    您必须始终指定TRANSPORT_TABLESPACES ，它指定使用可传输选项。此示例指定以下附加数据泵参数：

    • DUMPFILE参数指定要创建的结构信息导出转储文件的名称expdat.dmp 。

    • DIRECTORY参数指定指向转储文件的操作系统或Oracle自动存储管理位置的目录对象。必须在调用Data Pump之前创建DIRECTORY对象，并且必须将该目录的READ和WRITE对象权限授予运行Export实用程序的用户。

      在非CDB中，将自动创建目录对象DATA_PUMP_DIR 。对此目录的读写访问权限自动授予DBA角色，从而授予用户SYS和SYSTEM 。

      但是，目录对象DATA_PUMP_DIR不会在PDB中自动创建。因此，在导入PDB时，在PDB中创建目录对象，并在运行Data Pump时指定目录对象。

    • LOGFILE参数指定要为导出实用程序创建的日志文件。在此示例中，日志文件与转储文件在同一目录中创建，但您可以指定用于存储日志文件的任何其他目录。

    • 默认情况下，触发器和索引包含在导出操作中。

    要使用严格包含检查执行传输表空间操作，请使用TRANSPORT_FULL_CHECK参数，如以下示例所示：

    expdp use_name dumpfile = expdat.dmp directory = data_pump_dir transport_tablespaces = sales_1，sales_2 transport_full_check = y logfile = tts_export.log

    在这种情况下，Data Pump导出实用程序会验证可传输集内的对象与可传输集外部的对象之间是否存在依赖关系。如果正在传输的表空间集不是自包含的，则导出失败并指示可传输集不是自包含的。您必须解决这些违规，然后再次运行此任务。

    注意：

    在此示例中，Data Pump实用程序仅用于导出表空间的数据字典结构信息（元数据）。没有卸载实际数据，因此即使对于大型表空间集，此操作也会相对快速地进行。

  4. expdp实用程序在命令行上显示转储文件的名称和路径以及数据文件，如以下示例所示。这些是您需要传输到目标数据库的文件。另外，检查日志文件是否有任何错误。
    ************************************************** ***************************为SYSTEM.SYS_EXPORT_TRANSPORTABLE_01设置的转储文件是：/ u01 / app / oracle / admin / salesdb / dpdump / expdat .dmp ************************************************* *****************************可传输表空间SALES_1所需的数据文件：/u01/app/oracle/oradata/salesdb/sales_101.dbf可传输表空间SALES_2所需的数据文件：/u01/app/oracle/oradata/salesdb/sales_201.dbf
  5. 完成数据泵导出操作后，退出expdp实用程序以返回到SQL * Plus：
    $ EXIT

Oracle Database Release 19c，版本19.1中的更改

适用于Oracle数据库的 Oracle数据库概念 19c版本19.1具有以下更改。

新功能

以下主要功能是此版本中的新功能：

  • 延期插入

    应用程序可以将单行插入流式传输到存储在大型池中的特殊缓冲区中。插入是“延迟”的，因为数据库异步地将这些缓冲区写入数据文件。插入是自动提交的，无法回滚。

    请参阅“ 延迟插入的大型池缓冲区 ” 。

  • 混合分区表

    表可以包含内部和外部分区。内部分区占用数据库（段）内的空间。外部分区使用外部表功能存储在数据库外部。例如，销售表可能包含存储在数据文件中的两个分区和存储在文本文件中的三个分区。

    请参阅“ 分区表的分段 ” 。

    从Oracle Database 19c开始，不支持Oracle Streams。

    请参阅“ Oracle GoldenGate ” 。

Oracle Database Release 18c，版本18.1中的更改

适用于Oracle数据库的 Oracle数据库概念 版本18c，版本18.1进行了以下更改。

新功能

以下主要功能是此版本中的新功能：

  • Memoptimize池

    Oracle数据库支持使用MEMOPTIMIZE FOR READ子句指定的表的高性能读取。缓冲区缓存在SGA中称为memoptimize池的新区域中。

    请参阅“ Memoptimize Pool ” 。

    专用临时表是临时数据库对象，在事务或会话结束时自动删除。私有临时表存储在内存中，仅对创建它的会话可见。私有临时表将临时表的范围限制为会话或事务，从而为应用程序编码提供更大的灵活性，从而简化代码维护并提供更好的即用功能。

    请参阅“ 临时表概述 ” 。

    当数据库认为已将块写入存储时发生丢失写入，但未发生写入。影子表空间使数据库能够捕获大多数丢失的写入，而无需使用备用数据库，也不会出现级联错误。

关于SQL运算符

运算符操作称为操作数或参数的各个数据项。运算符由特殊字符或关键字表示。例如，乘法运算符由星号（*）表示。

如果已安装Oracle Text，则可以在Oracle Text查询中使用SCORE运算符，该运算符是该产品的一部分。您还可以使用内置的文本运算符创建条件，包括CONTAINS ， CATSEARCH和MATCHES 。有关这些Oracle Text元素的更多信息，请参阅Oracle Text Reference 。

一元和二元运算符

两类通用运算符是：

  • unary ：一元运算符仅在一个操作数上运行。一元运算符通常以此格式显示其操作数：

    运算符 操作数

  • binary ：二元运算符在两个操作数上运行。出现二进制运算符，其操作数采用以下格式：

    operand1 运算符 operand2

具有特殊格式的其他运算符接受两个以上的操作数。如果给操作符一个空操作数，则结果始终为null。唯一不遵循此规则的运算符是连接（||）。

运营商优先权

优先级是Oracle数据库在同一表达式中评估不同运算符的顺序。在评估包含多个运算符的表达式时，Oracle会在评估优先级较低的运算符之前评估具有较高优先级的运算符。Oracle在表达式中从左到右评估运算符的优先级相同。

A.1.2.2命名

主机变量名称必须仅包含字母，数字和连字符。他们必须以一封信开头。它们可以是任意长度，但只有前30个字符才有意义。您的编译器可能允许不同的最大长度。

由于Pro * COBOL限制，在与SQLLIB（C例程）交互时，除非正确对齐主机变量的边界，否则可能会出现一些不可预测的结果。有关定义主变量边界对齐的特定信息，请参阅COBOL文档。解决方法可能包括：

  • 使用FILLER手动对齐

  • 使用01级条目强制边界

  • 如果数据源是第三方代码，则使用77级条目或01级条目的临时变量，并将其用作主机变量。

A.1.3包含声明

您可以包含任何文件。预编译Pro * COBOL程序时，每个EXEC SQL INCLUDE语句都将替换为语句中指定的文件的副本。

如果您的系统使用文件扩展名但未指定文件扩展名，则Pro * COBOL预编译器将采用源文件（通常为COB）的默认扩展名。默认扩展名取决于系统。检查Oracle系统特定的文档。

如果系统使用目录，则可以通过指定预编译器选项INCLUDE = path来为包含的文件设置目录路径 。必须使用INCLUDE为非标准文件指定目录路径，除非它们存储在当前目录中。指定目录路径的语法是特定于系统的。检查Oracle系统特定的文档。

A.1.4 MAXLITERAL默认值

例如，如果您的COBOL编译器无法处理长度超过132个字符的字符串文字，请指定“MAXLITERAL = 132”。检查COBOL编译器用户指南。有关MAXLITERAL选项的更多信息，请参阅预编译器选项

数据库内存管理可以对数据挖掘产生重大影响。正确的尺寸程序全局区域（PGA）内存对于模型构建，复杂查询和批处理评分非常重要。从数据挖掘的角度来看， 系统全局区域（SGA）通常不太重要。但是，必须调整SGA的大小以适应实时评分，这会将模型加载到SGA中的共享游标中。在大多数情况下，您可以配置数据库以自动管理内存。为此，请在调整参数MEMORY_TARGET指定总的最大内存大小。通过自动内存管理，Oracle Database可根据需要在SGA和实例PGA之间动态交换内存，以满足处理需求。

大多数数据挖掘算法都可以利用在数据库中启用并行执行。INIT.ORA参数控制并行执行的行为。

17.1关于k -Means

k -Means算法是基于距离的聚类算法，将数据划分为指定数量的聚类。

基于距离的算法依赖于距离函数来测量案例之间的相似性。根据所使用的距离函数将案例分配到最近的簇。

17.1.1 Oracle数据挖掘增强型k -Means

Oracle Data Mining实现了k -Means算法的增强版本，具有以下功能：

  • 距离函数 ：该算法支持欧几里德和余弦距离函数。默认为欧几里德。

  • 可扩展的并行模型构建 ：该算法使用基于Bahmani，Bahman等人的非常有效的初始化方法。 “可扩展的k-means ++。” VLDB捐赠基金会的会议记录5.7（2012）：622-633 。

  • 群集属性 ：对于每个群集，算法返回质心，每个属性的直方图以及描述包含分配给群集的大部分数据的超级盒的规则。质心报告分类属性的模式以及数值属性的均值和方差。

这种k -Means方法避免了构建多个k -Means模型的需要，并提供了始终优于传统k -Means的聚类结果。

17.1.2质心

该 centroid表示集群中最典型的情况。例如，在客户年龄和收入的数据集中，每个群集的质心将是该群集中的平均年龄和平均收入的客户。质心是原型。它不一定描述分配给集群的任何给定案例。

质心的属性值是数值属性的平均值和分类属性的模式。

17.2 k -Means算法配置

了解配置k -means算法。

Oracle Data Mining增强型k -Means算法支持多种构建时设置。所有设置都有默认值。除非您想以某种特定方式影响算法的行为，否则没有理由覆盖默认值。

您可以通过指定以下注意事项来配置k -Means：

  • 集群数量

  • 距离函数。默认距离函数是欧几里德。

17.3 k -Means的数据准备

了解为k -means算法准备数据。

k -Means算法通常需要归一化。自动数据准备执行k -Means的规范化。如果不使用ADP，则必须在创建或应用模型之前规范化数字属性。

当具有简单数据类型（未嵌套）的列中缺少值时， k -Means会将它们随机解释为缺失。该算法用模式替换缺失的分类值，用平均值替换缺失的数值。

当嵌套列中缺少值时， k -Means会将它们解释为稀疏。该算法用零和矢量稀疏分类数据替换稀疏数值数据。
